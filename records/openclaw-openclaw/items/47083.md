## Summary

Keep this PR open: it is a valid stale-token display bug fix direction, but the submitted TUI change still lets stale totals survive in flag-only updates and blocks fresh-null clears, and the external PR lacks after-fix real behavior proof.

## What This Changes

The PR carries `totalTokensFresh` into TUI and Control UI session-token display paths so stale session token totals are hidden.

## Best Possible Solution

Land a repaired, current-main-based fix that carries `totalTokensFresh` through the TUI session-info path, handles stale flag-only and fresh-null cases, keeps Control UI formatting defensive, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a TUI refresh with `totalTokensFresh: false` and no fresh `totalTokens` leaves an old `sessionInfo.totalTokens`, which the footer still formats; a direct Control UI row with `totalTokensFresh: false` and a numeric total also still formats as a count.

## Solution Assessment

No, not as submitted. The direction is the narrow maintainable fix, but the TUI logic must check freshness before value/null guards and be ported to the current `TuiSessionList`/`SessionInfo` type surface.
