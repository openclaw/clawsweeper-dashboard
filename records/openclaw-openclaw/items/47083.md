## Summary

Keep open. The bug remains source-reproducible on current main and the PR targets the right TUI/UI surfaces, but the submitted TUI guard can leave a retained stale count on screen, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds `totalTokensFresh` plumbing to TUI and Control UI session-token display paths to suppress stale token counts.

## Best Possible Solution

Carry freshness through the TUI row contract, clear retained TUI totals when freshness is false while allowing fresh null clears, keep Control UI formatting defensive, and add focused regression coverage plus real behavior proof.

## Reproduction Assessment

Yes, source-level. Gateway session rows emit `totalTokensFresh`, current TUI ignores that flag before footer formatting, and Control UI direct row formatting still ignores a stale flag; I did not run a live TUI/UI session.

## Solution Assessment

No, not as submitted. The fix boundary is right, but the TUI condition must clear retained stale totals and allow fresh null clears before this is the narrow maintainable solution.
