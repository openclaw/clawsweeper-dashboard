## Summary

Keep this PR open: the reported stale-token display path still exists on current main, and the PR is a plausible narrow fix, but the submitted TUI guard leaves stale or fresh-null totals retained and the external real-behavior proof gate is still missing.

## What This Changes

The PR adds `totalTokensFresh` handling to TUI and Control UI session token display paths so stale session totals are not shown.

## Best Possible Solution

Rebase onto current main, carry `totalTokensFresh` through the TUI session list, clear stale totals while allowing fresh null clears, keep Control UI formatting defensive, and add focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows the gateway can emit `totalTokensFresh: false`, current TUI does not carry that flag into `applySessionInfo`, and the footer formats a previously retained `sessionInfo.totalTokens`.

## Solution Assessment

No, not as submitted. The fix boundary is right, but the TUI condition must allow fresh null clears and explicitly avoid retained stale totals before this is the narrow maintainable solution.
