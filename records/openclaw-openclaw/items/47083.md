## Summary

Keep open: the stale-token display bug is still valid from current-main source inspection, but this PR is not merge-ready because the TUI guard can still retain stale totals and the external contributor has not provided real after-fix behavior proof.

## What This Changes

Adds `totalTokensFresh` propagation and display checks for TUI session info and Control UI session-token formatting.

## Best Possible Solution

Land a current-main fix that carries `totalTokensFresh` through the TUI session-info path, clears or suppresses stale totals including flag-only and fresh-null updates, keeps Control UI formatting defensive, and adds focused regression coverage.

## Reproduction Assessment

Yes, from source inspection. A TUI session refresh or patch can retain an old `sessionInfo.totalTokens` when the next row marks totals stale or clears the total, and the footer formats that retained value directly.

## Solution Assessment

No, not as submitted. The direction is narrow, but the TUI guard must check freshness before value/null guards and must handle stale flag-only and fresh-null updates on the current `TuiSessionList` surface.
