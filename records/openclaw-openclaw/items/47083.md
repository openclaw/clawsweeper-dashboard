## Summary

Keep open: the stale-token display bug is valid and current main still has a TUI gap, but this PR is not mergeable as submitted because the TUI guard can leave old totals in state and the external PR lacks after-fix real behavior proof.

## What This Changes

The PR adds totalTokensFresh handling to TUI and Control UI session-token display paths so stale session totals are hidden.

## Best Possible Solution

Rebase onto current main, carry totalTokensFresh through the TUI session-list/session-info types, clear TUI totalTokens when freshness is false while allowing fresh null clears, keep Control UI formatting defensive, and add focused regression coverage.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current TUI receives no totalTokensFresh metadata, applies totalTokens directly, and formats the retained total in the footer. The Control UI list path is partly fixed on current main, but the formatter remains defensive only for null totals.

## Solution Assessment
