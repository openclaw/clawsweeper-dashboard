## Summary

Keep open. The stale-token bug is valid and current `main` only partially covers the surface; this PR still lacks real behavior proof, is conflicting, and its TUI guard would preserve stale totals when the gateway omits `totalTokens` or sends a fresh `null` clear.

## What This Changes

The PR adds `totalTokensFresh` plumbing to TUI and Control UI token display paths so stale session token totals are not shown.

## Best Possible Solution

Thread `totalTokensFresh` through the TUI session row/state contract, clear retained totals when freshness is false while allowing fresh `null` clears, keep Control UI formatting defensive, and reconcile with the active TUI footer work before merge.

## Reproduction Assessment

Yes, source-level. Current `main` emits `totalTokensFresh`, the TUI path does not carry that flag and can retain old totals when a stale row omits `totalTokens`, and the Control UI formatter still ignores a stale flag if a numeric total reaches it.

## Solution Assessment

No, not as submitted. The fix boundary is narrow, but the TUI guard must handle stale omitted totals and fresh-null clears before this is maintainable.
