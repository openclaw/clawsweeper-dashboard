## Summary

Keep open: the PR is aimed at a real stale-token display bug, but the submitted TUI guard still leaves stale counts visible in important update shapes and the external PR lacks after-fix real behavior proof.

## What This Changes

The PR adds `totalTokensFresh` typing and display checks for TUI and Control UI session-token counts.

## Best Possible Solution

Land a current-main-based fix that carries `totalTokensFresh` through the TUI session-info path, clears or suppresses stale totals including flag-only updates, keeps Control UI formatting defensive, and adds focused regression coverage.

## Reproduction Assessment

Yes, from source inspection. A session refresh or patch can leave an old `sessionInfo.totalTokens` in TUI state when the next row marks totals stale without a fresh total, and the footer formats that retained number directly.

## Solution Assessment

No, not as submitted. The direction is narrow and maintainable, but the TUI guard must handle freshness before value/null guards and must clear stale flag-only or fresh-null updates on the current `TuiSessionList` surface.

## Review Findings
