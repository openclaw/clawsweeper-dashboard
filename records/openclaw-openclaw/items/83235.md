## What This Changes

The PR changes Control UI Quick Settings avatar preview routing so local assistant avatars use the existing fetched blob URL or fallback instead of passing `/avatar/<agentId>` to an image tag, with focused UI regression tests.

## Best Possible Solution

Refresh the branch against current main, keep the UI-side bearer/blob/fallback approach, and land only after redacted Quick Settings proof shows no direct `/avatar/main` image render.

## Reproduction Assessment

Yes at source level, though not through a live browser/Tailscale run in this read-only review: current main can hand `/avatar/<agentId>` directly to Quick Settings while the protected avatar route rejects unauthenticated image loads.

## Solution Assessment

Yes for the code shape, but not merge-ready yet: reusing the existing bearer-authenticated blob/fallback path is narrower and safer than query-token image URLs or weakening avatar auth.
