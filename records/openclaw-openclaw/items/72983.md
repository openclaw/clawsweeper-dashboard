## What This Changes

## Best Possible Solution

Amend the PR to use direct escaped text binding for streaming markdown/reasoning, preserve intended whitespace, keep final sanitized markdown rendering, add focused coverage when feasible, and attach real behavior proof.

## Reproduction Assessment

Yes at source level: current main passes active streaming assistant text to `renderGroupedMessage()` with `isStreaming: true`, while markdown still renders through `unsafeHTML`. I did not establish a live browser recording in this read-only review.

## Solution Assessment

No. The plain-text streaming fallback is a reasonable minimal direction, but this implementation should remove template padding, preserve reasoning line breaks, and provide inspectable after-fix proof before merge.

  - confidence: 0.91

Streaming reasoning will collapse its lines until completion; add a streaming thinking style/helper that preserves newlines without template padding.
  - confidence: 0.8
