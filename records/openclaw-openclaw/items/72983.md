## Summary

Keep open: this targets a real source-reproducible web UI bug, but the PR still needs changes before merge because the streaming text binding preserves template whitespace, streamed reasoning loses line breaks, and the external PR has no inspectable after-fix behavior proof.

## What This Changes

The PR changes Control UI grouped chat rendering and chat text CSS so streaming markdown and reasoning render as escaped text while completed messages continue using sanitized markdown HTML.

## Best Possible Solution

Amend this PR so streaming markdown and reasoning use stable escaped text without template padding, preserve reasoning line breaks, keep final sanitized markdown rendering, and add focused grouped-render coverage plus real behavior proof.

## Reproduction Assessment

Yes, at source level: current main passes active streaming messages into `renderGroupedMessage()` with `isStreaming: true` while markdown still renders through `unsafeHTML`. I did not run a live browser recording in this read-only review.

## Solution Assessment

No. The plain-text streaming fallback is a maintainable minimal direction, but this implementation needs direct text binding, whitespace-preserving reasoning rendering, and inspectable after-fix proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Remove template padding from streaming text:** `ui/src/ui/chat/grouped-render.ts:1432`
  - body: The streaming branch places `${markdown}` on an indented standalone line while `.chat-text--streaming` uses `white-space: pre-wrap`. Lit keeps the static template newline and spaces as text, so streaming replies can show a leading blank line or indentation until final markdown rendering; bind the text without surrounding template whitespace or use a helper that emits only the markdown string.
  - confidence: 0.91
- **[P3] Preserve reasoning line breaks while streaming:** `ui/src/ui/chat/grouped-render.ts:1409`
  - body: The PR renders streaming `reasoningMarkdown` as raw text inside `.chat-thinking`, but that class does not preserve whitespace. Since `formatReasoningMarkdown()` joins the label and reasoning lines with `\n`, visible streaming reasoning collapses into one run until final markdown rendering; add a streaming thinking style/helper that preserves newlines without template padding.
  - confidence: 0.8
