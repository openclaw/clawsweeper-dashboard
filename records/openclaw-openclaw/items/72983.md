## Summary

Keep open. Current main still has the streaming `unsafeHTML` root cause and this PR is a useful narrow fix, but PR head still has a visible streaming whitespace bug, collapses streaming reasoning line breaks, and lacks the required changelog entry.

## What This Changes

The PR changes Control UI grouped chat rendering and chat text CSS so streaming assistant markdown and reasoning render as Lit text while completed messages continue using sanitized markdown HTML.

## Best Possible Solution

Amend this PR so streaming markdown and reasoning use stable escaped text without template whitespace, final messages keep sanitized markdown rendering, and focused grouped-render tests plus a changelog entry land before closing #68655.

## Reproduction Assessment

Yes. Current main statically reproduces the risky path because streaming messages set `isStreaming: true` while markdown still renders through `unsafeHTML`; the PR-specific whitespace regression is visible from the indented text binding combined with `white-space: pre-wrap`.

## Solution Assessment

No. The plain-text streaming fallback is a maintainable minimal direction, but this implementation needs direct text binding, whitespace-preserving reasoning styling, tests, and a changelog entry before merge.
