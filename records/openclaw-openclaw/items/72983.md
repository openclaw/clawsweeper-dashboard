## Summary

Keep open. The PR targets a real current-main Control UI streaming markdown bug and is narrow, but the branch still needs changes before merge: the new `pre-wrap` streaming template preserves literal indentation, streaming reasoning loses line breaks, and the user-facing fix lacks the required changelog entry and focused coverage.

## What This Changes

The PR changes Control UI grouped chat rendering and chat text CSS so streaming assistant markdown and reasoning render as Lit text during active streaming, while completed messages continue through sanitized markdown HTML.

## Best Possible Solution

Amend this PR, or replace it with an equivalent focused Control UI patch, so streaming markdown and reasoning update as stable escaped text without visible template whitespace, final messages keep sanitized markdown rendering, and grouped-render coverage plus a changelog entry land before the linked issue is closed.

## Reproduction Assessment

Yes. Current main statically reproduces the risky path because streaming messages set `isStreaming: true` but still render changing markdown through `unsafeHTML`; the PR regression is also reproducible from the head source because `pre-wrap` preserves the indented template text around `${markdown}`.

## Solution Assessment

No. The plain-text streaming fallback is a reasonable minimal direction, but this implementation should bind streaming text without surrounding template whitespace, preserve reasoning newlines with a streaming thinking style/helper, and add targeted tests plus the required changelog entry before merge.
