## Summary

Keep open. Current main still has the streaming markdown `unsafeHTML` path, and this PR is a focused fix candidate, but the latest patch still introduces visible whitespace regressions during streaming and lacks inspectable real behavior proof.

## What This Changes

The PR changes Control UI grouped chat rendering so streaming markdown and reasoning render as escaped plain text with a streaming text style, while completed messages continue using sanitized markdown HTML.

## Best Possible Solution

Amend this PR so streaming markdown and reasoning use stable escaped text without template padding, preserve reasoning line breaks, keep final sanitized markdown rendering, and add focused coverage plus real behavior proof.

## Reproduction Assessment

Yes, at source level: current main passes active streaming messages into `renderGroupedMessage()` with `isStreaming: true` while markdown still renders through `unsafeHTML`. I did not run a live browser recording in this read-only review.

## Solution Assessment

No. The plain-text streaming fallback is a maintainable minimal direction, but this implementation needs direct text binding, whitespace-preserving reasoning rendering, and inspectable after-fix proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Remove template padding from streaming text:** `ui/src/ui/chat/grouped-render.ts:1432`
  - body: The streaming branch puts `${markdown}` on an indented standalone line while `.chat-text--streaming` uses `white-space: pre-wrap`. Lit keeps that static newline and indentation as text, so streaming replies can show a leading blank line or spaces until the final markdown render; bind the markdown without surrounding template whitespace or use a helper that emits only the text.
  - confidence: 0.91
- **[P3] Preserve reasoning line breaks while streaming:** `ui/src/ui/chat/grouped-render.ts:1409`
  - body: The PR streams `reasoningMarkdown` as raw text inside `.chat-thinking`, but that class does not preserve whitespace. Since `formatReasoningMarkdown()` joins reasoning lines with newlines and final markdown rendering uses line breaks, visible streaming reasoning collapses until completion; add a streaming thinking style/helper that preserves newlines without template padding.
  - confidence: 0.8
