## Summary

Keep open. Current main still has the reported streaming markdown path, but this PR needs small fixes because the new streaming text branch preserves template indentation, streaming reasoning collapses newlines, and the user-facing fix lacks a changelog entry.

## What This Changes

The PR changes Control UI grouped chat rendering and chat text CSS so streaming assistant markdown and reasoning render as Lit text while completed messages continue using sanitized markdown HTML.

## Best Possible Solution

Land a focused UI patch that keeps final sanitized markdown rendering, emits escaped streaming text without surrounding template whitespace, preserves reasoning whitespace while streaming, and adds targeted regression tests plus a changelog entry before closing #68655.

## Reproduction Assessment

Yes. Current main statically reproduces the risky path: streaming messages set `isStreaming: true`, but the markdown branches still render through `unsafeHTML(toSanitizedMarkdownHtml(...))`.

## Solution Assessment

No. The plain-text streaming fallback is a reasonable minimal direction, but this implementation needs whitespace-safe text binding, streaming reasoning newline preservation, tests, and changelog coverage before merge.

Lit preserves the surrounding static newline/spaces as DOM text, so streaming replies can show a leading blank line or indentation until the final markdown render replaces them; bind the value through a helper or template shape that emits only the markdown string.
