## Summary

Keep open: the PR targets a real current-main bug, but the proposed guard does not reliably distinguish stripped/unknown fast-path refreshes from authoritative no-web snapshots, so it can both keep stale tools after a real removal and still clear tools after repeated stripped refreshes.

## What This Changes

The PR adds an internal `webToolsFromFastPath` flag, guards web-tool metadata activation for fast-path snapshots, adds runtime regression tests, and updates the changelog.

## Best Possible Solution

Keep the fix in the secrets runtime refresh boundary, but make the preserved web-tool metadata state durable across repeated partial refreshes while still clearing metadata for an authoritative full config that removes or disables the web surface.

## Reproduction Assessment

Yes, at source level. Current main prepares an empty fast-path web-tools snapshot for a stripped config and activates it unconditionally; the PR's remaining failure is reproducible by applying two stripped fast-path refreshes or by removing the web surface with no other SecretRefs.

## Solution Assessment

No. The runtime boundary is the right place, but keying preservation only off `priorSourceConfig` is not enough because that value is overwritten by the first stripped activation and cannot distinguish partial snapshots from real no-web configs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve web metadata only when the fast-path result is non-authoritative:** `src/secrets/runtime.ts:472-474`
  - body: This guard treats every fast-path no-surface snapshot after a prior web surface as something to preserve. That means a real write that removes the web/plugin web surface and has no other SecretRefs leaves stale tools advertised; after the first preserved stripped refresh, `activeSnapshot.sourceConfig` is also stripped, so a second stripped refresh clears the metadata anyway. The fix needs a durable way to distinguish partial/unknown fast-path refreshes from authoritative no-web snapshots.
  - confidence: 0.9
