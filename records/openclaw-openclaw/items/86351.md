## Summary

Keep open: this is a focused active fix for the linked Bun launcher startup failure, but the PR currently broadens the direct-miss detector enough to hide non-module-miss loader/runtime errors and should be narrowed before merge.

## What This Changes

The PR changes `openclaw.mjs` to recognize Bun-shaped direct optional warning-filter import misses and adds launcher e2e coverage for direct optional misses plus transitive import failures.

## Best Possible Solution

Land a narrow launcher classifier that accepts Bun-shaped direct `Cannot find module` misses while still surfacing non-module-miss and transitive failures, with the targeted e2e coverage kept in place.

## Reproduction Assessment

Yes. Current main still requires Node's `ERR_MODULE_NOT_FOUND` for direct misses, while the linked issue shows Bun's released startup failure uses a no-code raw-specifier `Cannot find module` error; I did not run live Bun in this read-only review.

## Solution Assessment

No, not as written. The fix direction is right, but the no-code branch should stay limited to explicit module-not-found signals instead of accepting any same-specifier error.
