## Summary

Keep open. Current main still sends macOS shells with stale `SSH_*` variables and no display env to the no-GUI fallback, and the open closing PR has not merged yet.

## What This Changes

The issue reports that `openclaw dashboard` on macOS falls into the no-GUI SSH fallback when stale `SSH_*` environment variables are present.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81081 once its proof and review are acceptable, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Source inspection on current main shows Darwin with stale `SSH_*` and no `DISPLAY`/`WAYLAND_DISPLAY` returns `ssh-no-display` before reaching the macOS `open` branch.

## Solution Assessment

Yes. Excluding Darwin from the SSH/no-display short-circuit is the narrow maintainable fix while preserving Linux no-display fallback and Windows launch behavior.
