## Summary

Keep open. Current main and the latest release still route stale macOS `SSH_*` environments to the no-GUI fallback, and the open closing PR has not merged yet.

## What This Changes

The issue reports that `openclaw dashboard` on macOS reports `No GUI detected` when a local shell inherits stale `SSH_*` variables.

## Best Possible Solution

Land the narrow resolver/test fix in https://github.com/openclaw/openclaw/pull/81081 once its proof and review are acceptable, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection on current main shows Darwin with any stale `SSH_*` variable and no `DISPLAY`/`WAYLAND_DISPLAY` returns `ssh-no-display` before reaching the macOS `open` branch.

## Solution Assessment

Yes. Excluding Darwin from the SSH/no-display short-circuit is the narrow maintainable fix, while preserving the Linux no-display fallback and existing Windows launch behavior.
