## Summary

Keep open. Current main still short-circuits stale `SSH_*` macOS shells to `ssh-no-display`, and the open fixing PR has not merged yet.

## What This Changes

The issue requests that `openclaw dashboard` on macOS use the native browser-open path even when stale `SSH_*` environment variables are present.

## Best Possible Solution

Review and land the narrow resolver/test fix in https://github.com/openclaw/openclaw/pull/81081 once proof and maintainer review are acceptable, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection on current main shows Darwin with any stale `SSH_*` variable and no `DISPLAY`/`WAYLAND_DISPLAY` returns `ssh-no-display` before reaching the macOS `open` branch.

## Solution Assessment

Yes. Excluding Darwin from the SSH/no-display short-circuit is the narrow maintainable fix, with regression coverage needed for both Darwin stale SSH and Linux preservation.
