## Summary

Keep open. Current main still short-circuits macOS browser opening when stale `SSH_*` variables exist and no display env is set, and an open unmerged PR already references this issue as its implementation candidate.

## What This Changes

Fix dashboard browser launch detection so macOS with lingering `SSH_*` environment variables still uses the native `open` path instead of printing the SSH no-GUI tunnel hint.

## Best Possible Solution

Land or replace PR #67115 with a narrow resolver fix in `src/infra/browser-open.ts` and focused regression tests for macOS stale-SSH behavior plus Linux SSH/no-display preservation.

## Reproduction Assessment

Yes. The source path is deterministic: on `darwin` with any `SSH_*` variable set and no display env, `resolveBrowserOpenCommand()` returns `ssh-no-display`, and `dashboardCommand()` turns that into the reported no-GUI hint.

## Solution Assessment

Yes. The narrowest maintainable fix is in the shared browser-open resolver rather than dashboard output formatting, with tests locking macOS and Linux behavior separately.
