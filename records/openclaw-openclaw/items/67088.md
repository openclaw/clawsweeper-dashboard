## Summary

Keep open. Current main and the latest release still short-circuit macOS shells with stale `SSH_*` variables into the no-GUI fallback, and the open closing PR remains the active implementation path.

## What This Changes

The issue reports that `openclaw dashboard` on macOS shows the remote no-GUI SSH hint when stale `SSH_*` environment variables are present.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81081 after acceptable real macOS behavior proof or maintainer override, then close this issue once the fix reaches main.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: Darwin with stale `SSH_*` variables and no `DISPLAY`/`WAYLAND_DISPLAY` returns `ssh-no-display` before the macOS `open` branch.

## Solution Assessment

Yes. Excluding Darwin from the SSH/no-display short-circuit is the narrow maintainable fix while preserving Linux no-display fallback and Windows launch behavior; merge should wait on the linked PR’s proof/review gate.
