## Summary

Keep open. Current main still takes the macOS stale-SSH path to `ssh-no-display`, and the issue is already paired with open PR #67115 using closing syntax, so it should remain open until that PR is reviewed, landed, or replaced.

## What This Changes

The issue asks OpenClaw to stop falsely reporting no GUI for `openclaw dashboard` on macOS when stale `SSH_*` environment variables are present.

## Best Possible Solution

Review, land, or replace PR #67115 with a narrow resolver fix and regression tests for macOS stale-SSH behavior while preserving Linux no-display SSH handling.

## Reproduction Assessment

Yes. Source inspection gives a deterministic current-main path: on Darwin with any `SSH_*` variable and no `DISPLAY`/`WAYLAND_DISPLAY`, `resolveBrowserOpenCommand()` returns `ssh-no-display` before the macOS `open` branch.

## Solution Assessment

Yes. The narrowest maintainable fix is in `src/infra/browser-open.ts`, with focused coverage in `src/commands/onboard-helpers.test.ts` for macOS stale SSH env variables and preserved Linux behavior.
