## Summary

Keep open, but this PR is not mergeable as-is: current main still uses the 60s pricing fetch timeout, while the PR changes only production code and leaves the focused timeout-warning test expecting 60s.

## What This Changes

This PR changes Gateway model-pricing refresh to use a 15s default fetch timeout with an env override and to skip background pricing refresh when an env disable flag is set.

## Best Possible Solution

Keep models.pricing.enabled as the primary shipped offline/restricted-network control. If maintainers still want a shorter default timeout or env override, rebase this PR onto current main, align it with the existing config contract, update tests/docs/changelog, and keep #72208 focused on lifecycle-aware shutdown.

## Reproduction Assessment

Yes. A static reproduction is high-confidence: the PR changes the timeout warning default to 15s, while the unchanged pricing-cache test still asserts timeout 60s for both OpenRouter and LiteLLM failures.

## Solution Assessment

No. The current patch is not the best mergeable form because current main already has a documented config opt-out, the broader hang needs lifecycle-abort work, and this PR leaves stale test expectations.
