## Summary

Keep this PR open, but it is not mergeable as-is. Current main does not implement the PR's 15s timeout/env-var controls, so it should not be closed as implemented; however, current main already has the documented config opt-out for restricted networks, the linked startup regression was closed as duplicate/fixed-on-main, and the PR leaves the timeout-warning test stale.

## What This Changes

This PR changes Gateway model-pricing refresh to use a 15s default fetch timeout with an env override and to skip background pricing refresh when an env disable flag is set.

## Best Possible Solution

Keep models.pricing.enabled as the primary shipped offline/restricted-network control. If maintainers still want a shorter default timeout or env override, rebase the PR onto current main, align it with the existing config contract, update the timeout-warning tests plus docs/changelog, and decide whether invalid env values should warn or be rejected strictly.
