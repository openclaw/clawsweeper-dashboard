## Summary

Keep open. Current main still keeps the 60s pricing fetch timeout and does not implement the PR's env-based timeout override, but the patch is not merge-ready because the changed default leaves existing timeout-warning tests stale and the new operator-facing env controls are not documented or changelogged.

## What This Changes

The PR changes the gateway model-pricing cache to use a 15s default fetch timeout with a clamped `OPENCLAW_PRICING_FETCH_TIMEOUT_MS` override and adds `OPENCLAW_DISABLE_PRICING_REFRESH=1` to skip background pricing refresh startup work.

## Best Possible Solution

Keep `models.pricing.enabled` as the documented offline/restricted-network opt-out. If maintainers still want a shorter default timeout or env override, rebase the PR onto current main, update the timeout-warning tests, document the env controls, add a changelog entry, and handle malformed timeout values explicitly.

## Reproduction Assessment

Yes. A static reproduction is high-confidence: the PR changes the default timeout to 15s while the unchanged pricing-cache test still asserts `timeout 60s` warning strings for both pricing sources.

## Solution Assessment

No. The current patch is not the best mergeable form because it leaves tests and public docs stale and does not reconcile the new env controls with the existing config-level pricing opt-out.
