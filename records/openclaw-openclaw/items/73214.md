## Summary

Keep open. Current main does not fully implement this PR’s intended shorter timeout or env-var controls, but the submitted patch is not mergeable as-is because it changes the default timeout while leaving the existing timeout-warning test at 60s, and it adds operator-facing controls without the expected release/docs coverage. Current main already has a shipped `models.pricing.enabled` config opt-out, so the branch should be repaired and aligned with that contract before maintainers decide whether the extra env surface is worthwhile.

## What This Changes

This PR changes Gateway model-pricing refresh to use a 15s default per-source fetch timeout with `OPENCLAW_PRICING_FETCH_TIMEOUT_MS` override and `OPENCLAW_DISABLE_PRICING_REFRESH=1` startup skip.

## Best Possible Solution

Keep the shipped `models.pricing.enabled` config opt-out as the primary offline/restricted-network control. If maintainers still want the shorter default timeout and env escape hatches, rebase this PR onto current main, update the stale test expectations, document the new controls, add the changelog entry, and keep the remaining #72208 lifecycle-abort work separate.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: the PR changes the default timeout used by `formatPricingFetchFailure` to 15s, while the current pricing-cache test still asserts `timeout 60s` for both OpenRouter and LiteLLM timeout warnings.

## Solution Assessment

No as submitted. The proposed direction may be acceptable, but the current patch is not the best mergeable form because current main already has a config opt-out and the PR needs tests/docs/changelog alignment before product review of the extra env controls.
