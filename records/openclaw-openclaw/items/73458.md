## Summary

Keep open. The ACP `_meta` direction is compatible, but the PR still has a blocking data-exposure concern because it publishes absolute spawned workspace paths to ACP clients; maintainer approval or redaction is needed before merge.

## What This Changes

The PR adds ACP session lineage `_meta` serialization for session listings and session-info snapshots, plus docs, changelog, and targeted ACP tests.

## Best Possible Solution

Land a rebased ACP `_meta` lineage implementation that exposes stable session lineage identifiers and omits or redacts workspace paths unless maintainers explicitly approve that contract.

## Reproduction Assessment

Not applicable as a feature PR. The high-confidence review path is source inspection against current main plus the PR diff: current ACP list/snapshot metadata lacks lineage while Gateway session rows already carry the proposed source fields.

## Solution Assessment

Unclear. `_meta` is the narrow compatible place for OpenClaw-specific ACP lineage, but including `spawnedWorkspaceDir` is a data-exposure decision; the safer default is lineage identifiers only until maintainer approval.
