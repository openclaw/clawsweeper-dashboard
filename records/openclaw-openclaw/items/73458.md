## Summary

Keep open. The PR targets a real remaining ACP bridge gap on current main, but the proposed patch also exposes `spawnedWorkspaceDir` absolute workspace paths through ACP `_meta` and the live PR is currently dirty against `main`, so it needs maintainer data-exposure review and rebase handling before merge.

## What This Changes

The PR adds a Gateway session-lineage serializer for ACP `_meta`, wires it into session listings and `session_info_update` snapshots, and updates ACP docs, changelog, and targeted tests.

## Best Possible Solution

Use ACP `_meta` for the stable lineage identifiers after rebase, but either omit/redact `spawnedWorkspaceDir` or land it only after an explicit maintainer decision that absolute workspace paths are acceptable in the ACP-visible contract.

## Reproduction Assessment

Not applicable as a feature PR; no bug reproduction is needed. The source-inspection path is high confidence: current ACP list and snapshot metadata omit lineage while Gateway session rows already carry the requested fields.

## Solution Assessment

Unclear. `_meta` is the narrow compatible place for OpenClaw-specific ACP lineage, but including `spawnedWorkspaceDir` is a data-exposure policy decision; the safer default is to keep lineage ids and omit or redact the workspace path.
