## Summary

Keep open. The PR addresses a real ACP bridge gap, but the proposed public ACP metadata includes absolute workspace paths and the branch is currently dirty, so it needs maintainer data-exposure review before merge.

## What This Changes

The PR adds normalized Gateway session lineage fields to ACP `_meta` for session listings and `session_info_update` snapshots, with docs, changelog, and targeted tests.

## Best Possible Solution

Use ACP `_meta` for stable lineage identifiers after rebase, but omit or redact `spawnedWorkspaceDir` unless maintainers explicitly approve absolute workspace paths in the ACP-visible contract.

## Reproduction Assessment

Not applicable as a feature PR. The source-inspection path is high confidence: current ACP list and snapshot metadata omit lineage while Gateway session rows already carry the requested fields.

## Solution Assessment

Unclear. `_meta` is the narrow compatible place for OpenClaw-specific ACP lineage, but including `spawnedWorkspaceDir` is a data-exposure policy decision; the safer default is to keep lineage ids and omit or redact the workspace path.
