## Summary

Keep open: the linked Ollama timeout complaint is plausible, but this PR changes the global implicit LLM idle fallback for every provider instead of using the existing provider-specific timeout path for slow local models.

## What This Changes

The PR changes `src/config/agent-timeout-defaults.ts` so the default LLM idle timeout constant becomes 300 seconds instead of 120.

## Best Possible Solution

Keep the global fallback conservative and address the Ollama regression through provider-scoped timeout behavior or clearer config/docs, with regression coverage and a changelog entry if runtime behavior changes.

## Reproduction Assessment

Unclear: source inspection verifies current main can use a 120-second implicit idle watchdog, but the linked issue lacks logs or a deterministic local Ollama transcript to reproduce the reported nonsensical-output regression end to end.

## Solution Assessment

No: changing `DEFAULT_LLM_IDLE_TIMEOUT_SECONDS` widens the implicit fallback for all providers, while current code, tests, runtime copy, and Ollama docs identify `models.providers.<id>.timeoutSeconds` as the scoped mechanism for slow local models.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Keep the local-model timeout scoped:** `src/config/agent-timeout-defaults.ts:1`
  - body: This constant is the implicit idle watchdog fallback for every provider when no provider request timeout is configured. Please keep the fix scoped to the Ollama/provider-timeout path or get maintainer agreement for the global behavior change with matching docs and tests.
