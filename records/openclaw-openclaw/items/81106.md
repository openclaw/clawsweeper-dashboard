## Summary

This PR should stay open and needs changes before merge: the contributor supplied enough real startup/connectivity proof, but the patch introduces a type-check blocker and weakens the current Teams monitor startup failure contract.

## What This Changes

The PR changes Microsoft Teams webhook startup to create an HTTP server explicitly, bind it to `0.0.0.0`, add startup catch logging, and make small handler return/cast edits.

## Best Possible Solution

Preserve the current fail-fast monitor contract, keep the existing valid local types, and add any startup logging or binding change only when backed by targeted reproduction and lifecycle coverage.

## Reproduction Assessment

Yes for the PR blockers: source inspection shows undefined type names in the proposed signature and a catch block that converts startup failures into a successful null-app result. No independent live reproduction was established for the original Teams startup failure.

## Solution Assessment

No: this patch is not the best way to solve the issue because Express already creates an HTTP server for `app.listen`, and the new catch block weakens the existing fail-fast startup contract. A safer repair should keep failures rejecting while adding narrowly tested logging or binding changes if needed.
