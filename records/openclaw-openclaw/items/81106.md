## What This Changes

The PR changes Microsoft Teams webhook startup to create a Node HTTP server explicitly, bind to `0.0.0.0`, add startup error logging, and make minor handler return/cast edits.

## Best Possible Solution

Preserve monitor startup rejection on bind/listen failures, add targeted startup logging if useful, and back any bind behavior change with focused lifecycle coverage.

No: this patch is not the best fix because Express already creates an HTTP server for `app.listen`, and the catch weakens the existing fail-fast startup contract. A safer fix would keep any logging or binding changes while still rejecting on listen errors.
