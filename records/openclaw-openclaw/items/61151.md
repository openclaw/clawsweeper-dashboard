## Summary

Keep open: the branch fixes the earlier drop-only and sessions_spawn-redaction review comments, but the current patch can still keep an interrupted Anthropic/Kimi tool-call artifact when the transport has already initialized arguments, and the external proof is still not convincing as written.

## What This Changes

The PR changes agent session transcript repair to strip partialJson from retained tool-call blocks, keep sessions_spawn redaction, and add focused regression tests.

## Best Possible Solution

Fix the sanitizer against actual transport completion semantics, add coverage for Anthropic initialized-arguments artifacts plus sessions_spawn redaction, then land only after valid real-behavior proof through an exported sanitizer or replay path.

## Reproduction Assessment

Yes, source-reproducible: current main can persist tool-call blocks with partialJson from OpenAI Responses, and replay passes them through sanitizeToolCallInputs without stripping that field. The PR's remaining failure is also source-reproducible from Anthropic transport initialization plus the new keep-after-strip logic.

## Solution Assessment

No, not yet: the workBlock direction is close for completed OpenAI Responses blocks and sessions_spawn redaction, but treating id/name/arguments as completion is not sufficient for Anthropic interrupted streams. The safer fix should distinguish complete parseable partialJson from incomplete streaming artifacts or use provider-aware repair rules.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Drop incomplete partialJson blocks with initialized arguments:** `src/agents/session-transcript-repair.ts:375-381`
  - body: Anthropic streaming creates toolCall blocks with `arguments: {}` and `partialJson` before any stop event, then deletes `partialJson` only on `content_block_stop`. If that stream is interrupted after id/name/arguments exist, this branch treats the block as complete, deletes `partialJson`, and replays the partial tool call instead of dropping the artifact the PR is meant to remove. Add coverage for that transport shape and only keep blocks whose `partialJson` is actually complete, or drop the incomplete shape.
  - confidence: 0.88
