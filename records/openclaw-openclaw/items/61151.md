## Summary

The PR should stay open because it is a focused fix in the right area, but the current predicate still has a blocking replay-safety bug: it treats OpenAI Responses pipe-delimited ids as proof that a `partialJson` tool call is finalized even though that same shape exists while the stream is still in progress.

## What This Changes

The branch updates session transcript repair to drop interrupted `partialJson` tool-call artifacts, strip redundant `partialJson` from retained OpenAI Responses calls, and add regression coverage for the keep/drop/redact behavior.

## Best Possible Solution

Keep the narrow repair, but gate retained `partialJson` blocks on real completion evidence such as a non-error/non-aborted assistant turn, and add regression coverage for an aborted OpenAI Responses pipe-id partial before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: sanitize an aborted assistant message containing `id: "call_x|fc_y"`, non-null `arguments`, and string `partialJson`; the PR predicate treats it as finalized and strips `partialJson` instead of dropping it.

## Solution Assessment
