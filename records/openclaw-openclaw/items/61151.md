## Summary

Keep open: the branch still converts an interrupted Anthropic tool-call stream into replayable session state by stripping `partialJson` from an initialized block with id/name/`arguments: {}` before `content_block_stop`, and the supplied terminal proof does not cover that shape.

## What This Changes

The PR changes transcript repair and tests so complete tool-call blocks with `partialJson` keep their parsed payload after stripping the streaming field, while incomplete artifacts are dropped and `sessions_spawn` redaction still runs.

## Best Possible Solution

Distinguish finalized OpenAI Responses blocks from initialized Anthropic streaming artifacts, keep `sessions_spawn` redaction after normalization, and prove both shapes through the exported transcript-repair path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current Anthropic streaming creates id/name/`arguments: {}` plus `partialJson` before `content_block_stop`, and the PR sanitizer would strip the marker and keep that interrupted block.

## Solution Assessment

No. The current approach handles the OpenAI finalized-block case but is not yet the best fix because it does not distinguish that case from Anthropic initialized streaming artifacts.
