Keep open: the patch is useful but not merge-ready because the new `partialJson` filter is bypassed by the provider-owned signed-thinking preservation path, so an interrupted Anthropic thinking/tool-use turn can still survive session repair.

## Best Possible Solution

Update transcript repair so non-final `partialJson` artifacts are rejected before any provider-owned signed-thinking preservation path, add signed-thinking regression coverage, and keep finalized OpenAI Responses retention plus `sessions_spawn` payload preservation intact.

Yes. Source inspection gives a high-confidence path: a signed-thinking assistant turn with a tool-call block containing `partialJson` enters the preservation branch before the new `partialJson` filter and is pushed through unchanged.

## Solution Assessment

No. The repair location is right, but the current implementation is incomplete because it handles ordinary assistant turns while missing the signed-thinking preservation path that Anthropic replay can use.
