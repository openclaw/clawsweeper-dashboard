## Summary

Keep open. The branch targets a real current-main transcript-repair gap, but the current patch still has a blocking streaming-shape bug for Anthropic-style tool calls and the supplied proof is too narrow for merge.

## What This Changes

The PR changes agent session transcript repair to strip `partialJson` from retained tool-call blocks, preserve `sessions_spawn` redaction, and add focused regression tests.

## Best Possible Solution

Make transcript repair distinguish finalized OpenAI Responses blocks from incomplete Anthropic-style streaming artifacts, add regression coverage for both shapes, and prove the behavior through an exported sanitizer or replay path.

## Reproduction Assessment

Yes. Source inspection shows current main can retain OpenAI Responses `partialJson` on replay, and the PR's remaining Anthropic failure is source-reproducible from the transport's initialized `arguments` plus `partialJson` lifecycle.

## Solution Assessment

No. The workBlock approach is close for complete OpenAI Responses blocks and `sessions_spawn` redaction, but treating id/name/arguments as completion is not sufficient for interrupted Anthropic streams.
