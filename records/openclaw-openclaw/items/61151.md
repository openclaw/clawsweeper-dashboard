The PR changes `repairToolCallInputs()` and its regression tests to strip `partialJson` from complete tool-call blocks, preserve `sessions_spawn` redaction, and drop incomplete streaming artifacts.

## Best Possible Solution

Distinguish finalized OpenAI Responses blocks from incomplete Anthropic streaming artifacts, keep `sessions_spawn` redaction after normalization, add regression coverage for both shapes, and refresh real behavior proof through the exported sanitizer/replay path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Anthropic streaming creates id/name/`arguments: {}` plus `partialJson` before `content_block_stop`, and replay sanitization would strip `partialJson` and keep that interrupted block under the PR.

## Solution Assessment

No. The direction is useful, but the branch needs provider-aware completion detection before stripping `partialJson`; treating initialized `arguments: {}` as completion is not safe for the reported Anthropic streaming artifact.
