## Summary

This PR is a plausible repair for stale streaming artifacts, but it is not merge-ready because the current patch still keeps an interrupted Anthropic/Kimi tool-call shape after stripping `partialJson`.

## What This Changes

The PR changes session transcript repair to strip `partialJson` from otherwise complete tool-call blocks, preserve `sessions_spawn` redaction, and add regression tests.

## Best Possible Solution

Distinguish finalized OpenAI Responses tool-call blocks from incomplete Anthropic/Kimi streaming artifacts, keep `sessions_spawn` redaction after normalization, and prove both paths through the replay sanitizer.

## Reproduction Assessment

Yes. Source inspection shows Anthropic streaming initializes tool calls with id/name/`arguments: {}` plus `partialJson` and only deletes `partialJson` on `content_block_stop`, while the PR would strip and retain that interrupted shape.

## Solution Assessment

No. The direction is useful, but the patch needs finalized-versus-incomplete detection instead of treating any block with id/name/input or arguments as safe to retain.
