## Summary

Keep open. Current main still has a single fixed italic-style reasoning display path with no per-channel or per-agent display-format setting, and the only linked implementation attempt was closed unmerged after review found incomplete parameter threading; the remaining work is real but needs maintainer product direction because it adds config surface and cross-channel display semantics.

## What This Changes

Request configurable thinking/reasoning block display formatting per channel or agent, especially to make Telegram reasoning easier to read than the current italic rendering.

## Best Possible Solution

Define one small reasoning display-format contract with italic as the compatibility default, clear channel/global precedence, schema/types/docs coverage, and cross-channel regression tests before implementation.

## Reproduction Assessment

Yes for the current limitation by source inspection: the formatter still accepts only text and all current call sites use the fixed italic display. This is a feature request rather than a broken current contract, so no failing runtime reproduction is required.

## Solution Assessment

Unclear until maintainer product direction is confirmed; a new setting could solve it, but the durable fix needs a deliberate display-format contract rather than reusing provider payload `thinkingFormat`.
