## Summary

Keep open. Current main still hardcodes visible reasoning as italic Markdown, exposes reasoning visibility but not display formatting, and the only linked implementation PR is closed and unmerged with known incomplete threading.

## What This Changes

Request a per-channel or per-agent config option to choose how thinking/reasoning blocks are displayed, especially for Telegram readability.

## Best Possible Solution

Define a documented display-format contract with italic as the compatibility default, then thread it through shared formatter, runner/subscribe payloads, Telegram reasoning lanes, config schema/types/help/docs, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main still lacks the requested display-format configuration surface.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is to preserve the current default, avoid hardcoding channel IDs in core, and pass the selected format through every reasoning delivery path.
