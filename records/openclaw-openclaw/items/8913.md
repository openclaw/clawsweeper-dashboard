## Summary

Keep open. Current main still has no per-channel or per-agent reasoning display-format setting; recent raw-reasoning refactors moved formatting closer to channel edges, but Telegram still renders reasoning through the fixed italic helper and the closed implementation PR was not merged.

## What This Changes

Request to add configurable formatting for thinking/reasoning blocks, especially Telegram code/spoiler/italic/blockquote-style rendering.

## Best Possible Solution

Define a small documented reasoning-display-format contract, keep italic as the compatibility default, thread it to the channel delivery edges, and add regression plus real Telegram proof for the selected formats.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks the requested display-format setting while Telegram still formats reasoning with the fixed italic helper.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a narrow format enum with explicit precedence and channel-edge threading, not only changing `formatReasoningMessage`.
