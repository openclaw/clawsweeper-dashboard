## Summary

Keep open. Current main still has a fixed italic reasoning display helper and no public per-channel or per-agent reasoning display-format setting, so the prior discussion and closed unmerged implementation PR do not resolve the requested feature.

## What This Changes

The issue requests configurable thinking/reasoning display formatting, especially for Telegram, with formats such as italic, code, spoiler, or blockquote at channel or agent scope.

## Best Possible Solution

Define a small documented reasoning-display-format contract, preserve italic as the compatibility default, thread the selected value to channel delivery edges without hardcoding Telegram in core, and add regression coverage for default and configured formats.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks a display-format setting while Telegram visible reasoning still formats through the fixed italic helper.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a narrow format enum with compatibility defaults and explicit threading to the channel rendering edge, not ad hoc formatter-only changes.
