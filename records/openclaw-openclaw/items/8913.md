## Summary

Keep open. Current main still hardcodes reasoning display as italic `Reasoning:\n_..._`, exposes visibility settings but no display-format setting, and the related implementation PR was closed unmerged.

## What This Changes

Request a configurable per-channel or per-agent format for visible thinking/reasoning blocks, especially Telegram alternatives to italic text.

## Best Possible Solution

Define a small backward-compatible reasoning display-format contract, keep italic as the default, document precedence, and thread the selected format through all reasoning payload and Telegram lanes with focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `formatReasoningMessage("example")` and Telegram tagged reasoning still resolve to italic `Reasoning:\n_example_`, and the config schemas lack a display-format key.

## Solution Assessment

Unclear until maintainers choose the public config contract. The requested direction is plausible, but the narrow maintainable fix should first define the shared format enum and precedence, then thread it through runner, subscribe, and Telegram paths.
