## Summary

Keep open. Current main still has a fixed italic reasoning display shape, no per-channel or per-agent display-format config, and the only linked implementation attempt was closed unmerged after review found incomplete parameter threading.

## What This Changes

Request to let users configure how reasoning/thinking messages are displayed per channel or agent, especially for Telegram readability.

## Best Possible Solution

Define one small reasoning display-format contract with italic as the compatibility default, clear channel versus agent/global precedence, schema/docs coverage, full pipeline threading, and focused shared plus Telegram regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the formatter and Telegram lane still have no display-format parameter or config path, although I did not need a live Telegram run to prove that missing capability.

## Solution Assessment

Unclear until maintainers choose the product contract. The likely best path is a narrow shared display-format enum plus full pipeline threading, not another Telegram-only patch or reuse of provider compat.thinkingFormat.
