## Summary

Keep open. Current main still hardcodes visible reasoning as `Reasoning:\n_..._`, config/docs expose visibility rather than display format, and https://github.com/openclaw/openclaw/pull/25336 is closed unmerged with review feedback that its threading was incomplete.

## What This Changes

Add a configurable per-channel or per-agent display format for visible thinking/reasoning blocks, motivated by Telegram readability.

## Best Possible Solution

Define a documented reasoning display-format contract with italic as the compatibility default, then thread it through shared formatter, runner payloads, subscribe streaming/message-end replies, Telegram reasoning lanes, config schema/types/help/docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection of current main is high-confidence that the requested display-format configuration is absent and visible reasoning remains fixed italic.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is to preserve the current default, avoid channel IDs in core, and pass the selected format through every reasoning delivery path.
