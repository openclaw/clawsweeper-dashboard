## Summary

Keep open: current main and the latest release still have a fixed italic reasoning formatter and no per-channel or per-agent display-format contract. The earlier linked implementation PR was closed unmerged after review found incomplete threading, so it does not supersede the remaining feature decision.

## What This Changes

The issue requests a configurable visible thinking/reasoning display format, especially for Telegram, with options such as italic, code, spoiler, or blockquote at channel or agent scope.

## Best Possible Solution

Define a small documented reasoning display-format enum with italic as the compatibility default, explicit precedence, schema/types/docs coverage, channel-edge threading, and focused Telegram plus shared formatter regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection of current main and v2026.5.22 gives high confidence that OpenClaw still lacks a display-format key and still renders visible reasoning through fixed italic formatting.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a narrow documented enum with explicit precedence and end-to-end threading, not a low-level formatter-only patch.
