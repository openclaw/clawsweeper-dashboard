## Summary

Keep open: current main and the latest release still expose reasoning visibility controls, but not a configurable display-format contract. The shared formatter and Telegram lane still produce fixed italic `Thinking` messages, and the related implementation PR closed unmerged after review found incomplete runner threading.

## What This Changes

The issue requests configurable display formatting for visible thinking/reasoning blocks, especially Telegram-readable options such as code, spoiler, or blockquote instead of hardcoded italics.

## Best Possible Solution

Define a documented reasoning display-format enum with italic as the compatibility default, explicit precedence, schema/types/docs coverage, runner and channel-edge threading, focused tests, and Telegram-visible proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks the requested display-format key and still formats visible Telegram reasoning through the fixed italic formatter.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a small documented enum with explicit precedence and end-to-end threading, not a local-only formatter parameter.
