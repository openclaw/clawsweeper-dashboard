## Summary

Keep open. Current main still hardcodes visible reasoning as italic Markdown and exposes reasoning visibility but no display-format configuration; the linked implementation PR is closed and unmerged.

## What This Changes

Request a configurable per-channel or per-agent format for visible thinking/reasoning blocks, especially Telegram readability.

## Best Possible Solution

Define a documented display-format contract with italic as the compatibility default, then thread it through shared formatter, runner/subscribe payloads, Telegram reasoning lanes, config schema/types/help/docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main still lacks the requested display-format configuration surface.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is to preserve the current default, avoid hardcoded channel IDs in core, and pass the selected format through every reasoning delivery path.
