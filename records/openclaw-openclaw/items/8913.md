## Summary

Current main and v2026.6.5 still expose only fixed italic reasoning presentation, and no merged or open canonical item now owns the requested configurable format. The feature is aligned with channel ergonomics, but its config ownership, precedence, portable modes, and channel fallback behavior remain product decisions, so this issue is still necessary.

## What This Changes

Add configurable reasoning-message presentation, potentially with per-channel or per-agent selection among italic, code, spoiler, or blockquote formats.

## Best Possible Solution

Define a small typed reasoning-presentation contract with italic as the compatibility default, choose one documented precedence chain, and let channel adapters render supported modes or a deterministic fallback across final, streaming, block-reply, and progress paths.

## Reproduction Assessment

Not applicable: current main consistently implements the documented fixed italic presentation, while this issue requests new selectable presentation modes rather than reporting broken established behavior.

## Solution Assessment

No, the sketched raw config option is not yet the best complete solution; the maintainable path needs a typed portable mode, explicit agent/channel precedence, and channel-owned rendering and fallback across all reasoning delivery paths.
