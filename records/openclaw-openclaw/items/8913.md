## Summary

Current main and the latest release still provide one fixed italic reasoning presentation, while the only implementation PR was closed unmerged and no viable canonical replacement now owns the feature. The request remains useful, but naming, configuration ownership, precedence, portable formats, and unsupported-channel behavior are product decisions, so this issue should remain open for maintainer direction.

## What This Changes

Add a configurable reasoning-block presentation setting, scoped per channel or agent, with alternatives such as italic, code, spoiler, or blockquote.

## Best Possible Solution

Define a small transport-neutral reasoning-presentation enum with current italic output as the compatibility default, choose one canonical configuration owner and explicit precedence, thread it through every reasoning delivery path, and let channel plugins map supported native styles with a documented deterministic fallback.

## Reproduction Assessment

Not applicable: current source and docs clearly establish fixed italic presentation, and the issue requests a new selectable capability rather than reporting broken defined behavior.

## Solution Assessment

No, the example configuration is not yet the best-defined solution; the narrow maintainable path requires a distinct presentation contract, one canonical owner, explicit precedence, complete delivery-path threading, and channel capability fallback.
