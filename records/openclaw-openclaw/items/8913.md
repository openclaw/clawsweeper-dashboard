Keep open: current main and the latest release still expose reasoning visibility controls, but not a configurable reasoning display-format contract. The related implementation PR is closed unmerged and its review identified incomplete threading, so it does not supersede the request.

## What This Changes

Add a configurable per-channel or per-agent display format for visible reasoning/thinking blocks, especially to improve Telegram readability beyond italic Markdown.

## Best Possible Solution

Define a small documented reasoning display-format enum with italic as the compatibility default, explicit precedence, schema/type/docs coverage, channel-edge threading, focused tests, and Telegram-visible proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection on current main gives high confidence that the requested display-format setting is absent and Telegram reasoning still flows through a fixed italic formatter.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a documented enum with explicit precedence and channel-edge implementation, not a local-only formatter parameter.
