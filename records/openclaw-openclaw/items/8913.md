## Summary

Keep open: current main and v2026.5.12 still expose reasoning visibility only and still format visible reasoning through the fixed italic `Reasoning:` path; the related implementation PR was closed unmerged after review found incomplete threading. This remains a feature/config contract decision rather than conservative cleanup.

## What This Changes

Add a configurable per-channel or per-agent display format for visible thinking/reasoning blocks, especially Telegram code/spoiler formatting instead of hardcoded italics.

## Best Possible Solution

Define a documented reasoning display-format contract with italic as the compatibility default, explicit precedence, schema/types/docs coverage, channel-edge threading, focused regression tests, and Telegram-visible proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main still lacks the requested display-format key and Telegram still routes visible reasoning through the fixed italic formatter.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a small documented enum with explicit precedence and channel-edge implementation, not a local-only formatter parameter.
