## Summary

Keep open: current main and the latest release still hardcode reasoning display as italic-style markdown, with no per-channel or per-agent display-format setting. The request remains useful, but it adds config/API surface and needs a maintainer product decision before an automated fix PR should choose the contract.

## What This Changes

The issue requests a configurable per-channel or per-agent format for visible thinking/reasoning blocks, especially to make Telegram reasoning easier to read than the current italic rendering.

## Best Possible Solution

Define a small reasoning display-format contract with italic as the compatibility default, choose precedence, document it, and thread it through shared formatter, embedded runner/subscribe paths, Telegram reasoning lanes, and focused regression tests.

## Reproduction Assessment

Yes at source level: current main still has a fixed formatter and no config/schema/docs path for choosing code, spoiler, blockquote, or another display style. I did not run a live Telegram scenario because the missing capability is directly visible in source.

## Solution Assessment

Unclear as proposed: the user problem is real, but the best fix needs maintainer choice on config scope and precedence. A formatter-only parameter would be too narrow unless runner, subscribe, Telegram, docs, schema, and tests all agree.
