## Summary

Keep open. Current main still has no public reasoning display-format config, Telegram reasoning delivery still calls the fixed italic formatter at the channel edge, and the earlier implementation PR is closed unmerged while the later raw-reasoning refactor only moved formatting responsibility to channel dispatchers.

## What This Changes

Add a configurable per-channel or per-agent display format for thinking/reasoning blocks, especially to make Telegram reasoning easier to read than the current italic rendering.

## Best Possible Solution

Define a small documented reasoning display-format contract with italic as the compatibility default, then thread a prepared selected format through config schema/types/docs, Telegram/channel edge formatting, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks the requested display-format configuration while Telegram visible reasoning still uses the fixed italic formatter.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is to avoid channel-ID special cases in core and pass a selected display-format value to the channel delivery edge that formats visible reasoning.
