## Summary

Keep open: current main and the latest release still do not provide configurable reasoning display formatting, and the earlier implementation attempt was closed unmerged after incomplete threading was identified. This needs maintainer product/config decisions rather than cleanup closure.

## What This Changes

Add a configurable per-channel or per-agent reasoning display format for Telegram and other channel replies.

## Best Possible Solution

Define a documented reasoning display-format contract with italic as the compatibility default, explicit precedence, schema/types/docs coverage, channel-edge threading, focused tests, and Telegram-visible proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main still lacks the requested display-format key and Telegram still routes visible reasoning through the fixed italic formatter.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a small documented enum with explicit precedence and channel-edge implementation, not a local-only formatter parameter.
