## Summary

Keep open: current main still hardcodes italic reasoning display and exposes only reasoning visibility plus provider payload-format config; the related https://github.com/openclaw/openclaw/pull/25336 is closed unmerged and did not become the canonical implementation.

## What This Changes

Add a per-channel or per-agent configuration option to choose how visible reasoning/thinking blocks are rendered, especially for Telegram readability.

## Best Possible Solution

Define one small reasoning display-format contract with italic as the compatibility default, clear channel versus agent/global precedence, docs/schema/type coverage, and focused tests across shared and Telegram-visible paths.

## Reproduction Assessment

Yes, source-level only: current main still routes Telegram reasoning through a shared formatter with no display-format parameter, and tests assert only italic output. This is a missing feature rather than a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the product contract. The maintainable path is a narrow shared display-format contract with explicit precedence and no collision with provider `compat.thinkingFormat`.
