## Summary

Keep open. Current main still has fixed italic reasoning display and no per-channel or per-agent display-format setting; the linked implementation PR was closed unmerged after review noted incomplete threading.

## What This Changes

Request configurable formatting for shown thinking/reasoning blocks, especially Telegram, with options such as italic, code, spoiler, or blockquote.

## Best Possible Solution

Define a small documented reasoning-display-format contract, keep italic as the compatibility default, thread the selected value to channel delivery edges, and cover default plus configured formats in tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks the requested display-format setting while Telegram reasoning still formats through the fixed italic helper.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a narrow format enum with compatibility defaults and explicit threading to channel rendering edges, not a formatter-only change.
