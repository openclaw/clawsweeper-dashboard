## Summary

Keep open. Current main still lacks a per-channel or per-agent reasoning display-format setting, and the only implementation PR linked from the discussion was closed unmerged with incomplete threading called out in review.

## What This Changes

The issue requests a public configuration option to choose how thinking/reasoning blocks are rendered per channel or agent.

## Best Possible Solution

Define a small documented reasoning-display-format contract, keep italic as the compatibility default, thread the selected value to channel delivery edges, and add regression coverage for default and configured formats.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks the requested display-format setting while Telegram reasoning still formats through the fixed italic helper.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a narrow format enum with compatibility defaults and explicit threading to the channel rendering edge, not an ad hoc formatter-only change.
