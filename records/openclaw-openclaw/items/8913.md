## Summary

Keep open: current main and v2026.5.19 still expose reasoning visibility controls, but not a configurable display-format contract. The central formatter and Telegram lane still use the fixed `Thinking` prefix plus italicized lines, and the related implementation PR is closed unmerged with review feedback that it missed important runner threading.

## What This Changes

Request a per-channel or per-agent configuration option for how visible thinking/reasoning blocks render, especially on Telegram.

## Best Possible Solution

Define a documented reasoning display-format enum with italic as the compatibility default, explicit precedence, schema/types/docs coverage, channel-edge threading, focused tests, and Telegram-visible proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks the requested display-format key and still formats visible Telegram reasoning through the fixed italic formatter.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a small documented enum with explicit precedence and channel-edge implementation, not a local-only formatter parameter.
