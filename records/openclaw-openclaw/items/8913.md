## Summary

Keep open. Current main still lacks a per-channel or per-agent reasoning display-format setting, and the only direct implementation attempt was closed unmerged after review found incomplete parameter threading.

## What This Changes

The issue requests configurable formatting for reasoning/thinking blocks per channel or agent so Telegram can use alternatives to italic rendering.

## Best Possible Solution

Define a small documented reasoning-display-format contract, preserve italic as the compatibility default, thread it through the affected channel delivery edges, and add regression coverage plus real Telegram proof for the selected formats.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main still lacks the requested display-format setting while Telegram uses the fixed italic helper.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a narrow format enum with explicit precedence and channel-edge threading, not only changing `formatReasoningMessage`.
