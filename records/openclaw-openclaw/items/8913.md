## Summary

Keep open: current main still lacks a configurable reasoning/thinking display format, and the remaining work is a public config/API decision rather than a narrow bug fix. The related implementation pull request at https://github.com/openclaw/openclaw/pull/25336 was closed unmerged and its review notes identified incomplete threading, so it does not supersede this issue.

## What This Changes

The issue requests a per-channel or per-agent config option to change visible reasoning blocks from the current italic Telegram formatting to alternatives such as code, spoiler, or blockquote.

## Best Possible Solution

Define a small documented reasoning display-format contract with compatibility defaults, explicit precedence, schema/type/docs coverage, channel-edge implementation, focused regression tests, and Telegram-visible proof for any chat rendering change.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main still lacks a display-format key and Telegram still routes visible reasoning through the fixed formatter.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a documented enum with explicit precedence and channel-edge threading, not a local-only formatter parameter.
