## Summary

Keep open. Current main still hardcodes reasoning display as italic `Reasoning:\n_..._` text, exposes only reasoning visibility settings, and has no per-channel, per-agent, or global display-format contract. The related implementation PR #25336 was closed unmerged and its review context called out incomplete parameter threading, so it does not implement or supersede the request.

## What This Changes

Add configuration to choose how thinking/reasoning blocks are displayed per channel or agent, with Telegram-friendly formats such as code, spoiler, or blockquote while preserving the current italic default.

## Best Possible Solution

Define a small reasoning display-format contract, preserve italic as the compatibility default, document precedence, thread the selected format through runner payloads, subscribe streaming, message-end block replies, and Telegram reasoning lanes, then add focused tests and docs for default italic plus the approved alternate formats.

## Reproduction Assessment

Yes. Current-main source and tests give a high-confidence reproduction path: `formatReasoningMessage("example")` and `splitTelegramReasoningText("<think>example</think>Done")` still produce `Reasoning:\n_example_`, and config/schema searches show no accepted display-format key.

## Solution Assessment

Unclear until maintainers choose the public config contract. The requested direction is valid, but the closed PR #25336 was incomplete; the safer implementation is a narrow, documented core display-format option threaded through all reasoning payload paths before channel-specific rendering tests are added.
