## Summary

Keep open: current main and the latest release still expose reasoning visibility, but not a configurable reasoning display-format contract. The central formatter and Telegram reasoning lane remain fixed to the underscore-italic `Thinking` message shape, and the earlier implementation PR was closed unmerged with incomplete parameter threading.

## What This Changes

Request to add configurable reasoning/thinking display formatting per channel or per agent so Telegram reasoning can use a more readable format than fixed italics.

## Best Possible Solution

Define a documented display-format enum with italic as the compatibility default, explicit precedence, config schema/types/docs coverage, runner and channel-edge threading, focused regression tests, and Telegram-visible proof after the contract is chosen.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection of current main and v2026.6.5 confirms the display-format option is still absent and reasoning remains underscore-wrapped where it is rendered.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a small documented enum with explicit precedence and end-to-end threading, not a formatter-only patch.
