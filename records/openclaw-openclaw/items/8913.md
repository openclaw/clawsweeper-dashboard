## Summary

Keep open: current main and v2026.5.12 still expose reasoning visibility only, while the shared formatter and Telegram edge still produce fixed italic `Reasoning:` output. ## What This Changes

Add configurable per-channel or per-agent formatting for visible thinking/reasoning blocks, especially Telegram code, spoiler, or blockquote formatting instead of fixed italics.

## Best Possible Solution

Define a documented reasoning display-format enum with italic as the compatibility default, explicit precedence, schema/types/docs coverage, channel-edge threading, focused tests, and Telegram-visible proof.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a small documented enum with explicit precedence and channel-edge implementation, not a local-only formatter parameter.
