## Summary

Keep open. Current main and the latest release still expose reasoning visibility controls but not a per-channel or per-agent display-format setting, and the only directly linked implementation PR was closed unmerged after review found incomplete parameter threading.

## What This Changes

Request a configurable way to render visible thinking/reasoning blocks per channel or agent, especially for Telegram alternatives to italic text such as code, spoiler, or blockquote formatting.

## Best Possible Solution

Define one small reasoning display-format contract with italic as the compatibility default, clear per-channel versus per-agent precedence, schema/docs coverage, and regression tests across the shared formatter plus Telegram reasoning lanes before implementation.

## Reproduction Assessment

Yes by source inspection for the current behavior: the formatter and Telegram lane still hardcode the visible reasoning shape and accept no display-format input. This is a feature gap rather than a failing bug reproduction.

## Solution Assessment

Unclear as an implementation path until maintainers choose the product contract. The requested direction is plausible, but the best fix needs a narrow owner boundary and full threading rather than a low-level formatter option alone.
