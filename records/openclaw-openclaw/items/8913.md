## What This Changes

## Best Possible Solution

Define one small reasoning display-format contract with italic as the compatibility default, clear global/per-agent/channel precedence, schema/type/docs coverage, and focused tests across the shared formatter plus Telegram-visible paths.

## Reproduction Assessment

Yes, at source level: current main still has a fixed reasoning formatter and no config/schema/docs path for code, spoiler, or blockquote display. A live Telegram run is not needed to prove the requested capability is absent.

## Solution Assessment

Unclear until maintainers choose the product shape: the requested config is plausible, but the best fix needs a named display-format contract that avoids confusion with provider payload `thinkingFormat` and covers all formatter call sites.
