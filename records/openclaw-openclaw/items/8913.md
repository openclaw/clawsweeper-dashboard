## Summary

Keep open. Current main still lacks a configurable reasoning display format, Telegram still routes reasoning through the fixed italic formatter, and the related implementation PR is closed unmerged with review feedback showing incomplete parameter threading.

## What This Changes

Request a per-channel or per-agent configuration option for how visible reasoning blocks render, especially for Telegram readability.

## Best Possible Solution

Define a documented reasoning display-format contract with italic as the compatibility default, then thread the selected format through shared formatter, agent config schema/types/docs, Telegram reasoning lanes, streaming/block reply paths, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection of current main is high-confidence that the requested display-format configuration is absent and visible reasoning still uses the fixed formatter on Telegram.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is to preserve the current default, avoid hard-coding channel IDs in core, and pass a prepared display-format value through every reasoning delivery path.
