## Summary

Keep this issue open for a maintainer product decision. Current main still hardcodes one shared italic reasoning presentation, the config and docs expose visibility rather than display style, and the only implementation attempt was closed unmerged with incomplete runner threading; however, the proposed global string setting needs a clearer portable channel contract before implementation.

## What This Changes

Requests configurable per-channel or per-agent formatting for visible reasoning blocks, including italic, code, spoiler, or blockquote styles.

## Best Possible Solution

Define a small typed reasoning-presentation contract at the shared payload boundary, preserve the current italic presentation as the compatibility default, let each channel map supported native styles or fall back safely, and document an explicitly approved agent-versus-channel precedence model.

## Reproduction Assessment

Yes. Current main provides a high-confidence source reproduction: `formatReasoningMessage(text)` has no presentation input and deterministically emits per-line italic Markdown, while current configuration has no corresponding style setting.

## Solution Assessment

No, not in the raw form proposed. A plain global format string does not model differing channel capabilities or precedence safely; a typed portable presentation mode with channel-owned rendering and fallback behavior is the narrower maintainable direction.
