## Summary

Current main and the latest release still hardcode visible reasoning as italic Markdown, and the only implementation PR closed without merging after review found incomplete parameter threading. The request remains useful, but its config scope, precedence, supported formats, and channel fallback behavior need an explicit product decision before implementation.

## What This Changes

Add a configurable presentation style for visible reasoning messages, with alternatives to the current fixed italic rendering.

## Best Possible Solution

Define one typed, transport-portable reasoning-presentation policy, preserve the current italic output as the compatibility default, document one precedence rule, and let each channel plugin map supported variants or apply an explicit fallback across streaming and final-message paths.

## Reproduction Assessment

Not applicable as a bug reproduction: current source, tests, and v2026.6.5 consistently define italic presentation, while the issue requests a new selectable capability.

## Solution Assessment

No, not exactly as proposed: the user need is valid, but parallel per-agent and per-channel raw-Markdown settings should be replaced by one approved presentation contract with explicit precedence and channel fallbacks.
