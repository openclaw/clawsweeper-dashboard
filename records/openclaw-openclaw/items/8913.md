## Summary

Keep open. Current main still hardcodes italic reasoning display and lacks a per-channel or agent-level display-format config; the prior implementation PR was closed unmerged and does not supersede the requested feature.

## What This Changes

Request configurable formatting for reasoning/thinking blocks per channel or agent, especially Telegram alternatives to italic text.

## Best Possible Solution

Define a backward-compatible reasoning display-format enum and precedence rule, preserve italic as the default, document the config, and thread it through all reasoning delivery paths with focused regression tests.

## Reproduction Assessment

Yes for the observable current behavior: source inspection shows formatReasoningMessage("example") still produces italic Reasoning output, and the config schemas do not expose a display-format key. As a feature request, there is no broken contract to reproduce beyond confirming the requested capability is absent.

## Solution Assessment

Unclear until maintainers choose the public config contract. The narrow maintainable path is a shared display-format option with documented precedence, then consistent threading through core agent and Telegram paths.
