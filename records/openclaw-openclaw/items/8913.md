## Summary

Keep open. Current main still hardcodes visible reasoning as italic Markdown and does not expose a per-channel or per-agent display-format setting; the related implementation PR #25336 is closed and unmerged.

## What This Changes

Request to add configurable reasoning/thinking display formatting so channel replies can use formats such as italic, code, spoiler, or blockquote instead of fixed italics.

## Best Possible Solution

Define a small documented reasoning display-format contract, preserve italic as the default, settle precedence between agent defaults and channel overrides, then thread it through core reasoning payloads, subscribe streaming/message-end paths, and Telegram tests/docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main still produces fixed italic reasoning output and lacks the requested configuration surface.

## Solution Assessment

Unclear until maintainers settle the public config contract. The requested direction is reasonable, but adding a new config option and supported format set is a product/API decision rather than a narrow bug fix.
