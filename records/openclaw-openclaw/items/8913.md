## Summary

Keep open. Current main still has a fixed italic reasoning display path and no per-channel or per-agent display-format setting; the direct implementation attempt was closed unmerged, and the later raw-reasoning refactor moved formatting to channel edges without adding configurability.

## What This Changes

The issue requests a configurable reasoning/thinking display format for channel or agent output, especially Telegram alternatives to italic formatting.

## Best Possible Solution

Define a small documented display-format enum with italic as the compatibility default, explicit precedence, channel-edge threading, and regression coverage for Telegram plus affected shared reasoning paths.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main still lacks the requested accepted config key and Telegram still uses the fixed italic formatter.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a narrow documented enum with explicit precedence and channel-edge integration, not only changing `formatReasoningMessage`.
