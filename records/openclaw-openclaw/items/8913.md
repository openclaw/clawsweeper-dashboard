## Summary

Keep open: current main and v2026.6.6 still expose reasoning visibility, not a configurable reasoning display style, and Telegram reasoning remains formatted through the fixed italic helper. The requested config/display contract needs maintainer product direction before a fix PR should be queued.

## What This Changes

Request configurable per-channel or per-agent formatting for visible thinking/reasoning messages instead of fixed italic Telegram output.

## Best Possible Solution

Define a narrow reasoning display-format contract with italic as the compatibility default and explicit precedence, then update schema, docs, Telegram reasoning lanes, and shared reasoning delivery tests together.

## Reproduction Assessment

Yes for the current behavior: source inspection shows the formatter and Telegram lane still have fixed italic output and no display-format parameter. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Unclear: the requested config is plausible, but OpenClaw's config-surface bar makes the exact per-channel versus per-agent contract a maintainer product decision. A narrower Telegram-only default change may also be considered before adding schema surface.
