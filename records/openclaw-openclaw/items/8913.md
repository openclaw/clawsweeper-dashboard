## Summary

Keep open. Current main still has a fixed italic reasoning display path and no per-channel or per-agent display-format setting; the direct implementation attempt at https://github.com/openclaw/openclaw/pull/25336 was closed unmerged after review found incomplete parameter threading.

## What This Changes

The issue requests configurable per-channel or per-agent formatting for thinking/reasoning blocks, motivated by Telegram readability for long italic reasoning messages.

## Best Possible Solution

Define a small documented reasoning display-format contract, keep italic as the compatibility default, thread the selected format through the channel-edge delivery paths, and add regression coverage plus real Telegram proof for the chosen formats.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection confirms current main still lacks a display-format setting while Telegram uses the fixed italic helper.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a narrow documented enum with explicit precedence and channel-edge threading, not only changing `formatReasoningMessage`.
