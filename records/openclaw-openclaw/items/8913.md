## Summary

Keep open: current main and the latest release still hardcode reasoning display as `Thinking` plus underscore-wrapped italic lines, and no per-channel or per-agent display-format setting exists. The linked implementation PR is closed unmerged, and adjacent leak/truncation reports do not own this configurable-format feature.

## What This Changes

The issue requests a per-channel or per-agent configuration option for reasoning/thinking block display formatting, especially to avoid hardcoded italic Telegram rendering.

## Best Possible Solution

Keep this open for a maintainer-approved display-format contract that preserves italic as the default and threads the chosen format through shared reasoning delivery plus Telegram and other channel lanes.

## Reproduction Assessment

Yes, at source level for the missing capability: current main and v2026.6.6 expose reasoning visibility and thinking depth but no display-format input. I did not run a live Telegram setup because this is a feature gap rather than a broken-current-contract bug.

## Solution Assessment

No, not as an automatic fix. The requested direction is plausible, but the best solution needs maintainer choice on naming, scope, and precedence before adding a new config surface; a low-level formatter-only change would be too narrow.
