## Summary

Keep open. Current main still has fixed italic reasoning display and no public display-format config; the closed implementation PR was not merged, and the remaining work needs a maintainer decision on the public config contract and precedence.

## What This Changes

Add a configurable reasoning/thinking display format for channel or agent replies, especially to improve Telegram readability.

## Best Possible Solution

Define a small backward-compatible reasoning display-format contract, keep italic as the default, document precedence, then thread the selected format through runner, subscribe, message-end, and Telegram paths with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `formatReasoningMessage("example")` and Telegram tagged reasoning still resolve to italic `Reasoning:\n_example_`, while the agent config schemas lack a display-format key.

## Solution Assessment

Unclear until maintainers choose the public config contract. The narrow maintainable direction is to add a shared enum and precedence rule, then thread it consistently through all reasoning delivery paths.
