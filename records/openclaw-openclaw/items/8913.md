## Summary

Keep open: current main and v2026.5.22 still expose reasoning visibility, but not a configurable reasoning/thinking display-format contract. The formatter and Telegram lane still use the fixed Thinking-plus-italic shape, and the linked implementation PR closed unmerged after review found incomplete runner threading.

## What This Changes

Request configurable formatting for visible thinking/reasoning blocks per channel or agent, especially to avoid hardcoded italic Telegram output.

## Best Possible Solution

Define a documented reasoning display-format enum with italic as the compatibility default, explicit precedence, schema/types/docs coverage, runner and channel-edge threading, focused tests, and Telegram-visible proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection of current main and v2026.5.22 gives high confidence that there is no display-format key and visible reasoning still formats through the fixed italic helper.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a small documented enum with explicit precedence and end-to-end threading, not a Telegram-only local formatter parameter.
