## Summary

Keep open: current main and v2026.5.12 still expose reasoning visibility only, while Telegram and other channel edges still route visible reasoning through the fixed italic `Reasoning:` formatter. The closed implementation attempt was not merged and was already identified as incompletely threaded, so this remains a product/config contract decision rather than conservative cleanup.

## What This Changes

Request a configurable per-channel or per-agent display format for visible thinking/reasoning blocks, especially Telegram code/spoiler/italic rendering.

## Best Possible Solution

Define a documented reasoning display-format contract with italic as the compatibility default, explicit precedence, schema/types/docs coverage, channel-edge threading, focused regression tests, and Telegram-visible proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks the requested display-format key and Telegram still routes visible reasoning through the fixed italic formatter.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a small documented enum with explicit precedence and channel-edge implementation, not a local-only formatter parameter.
