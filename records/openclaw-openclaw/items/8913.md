## Summary

Keep open: current main and v2026.5.22 still expose reasoning visibility but no configurable display-format contract, and Telegram visible reasoning still passes through the fixed italic formatter. The prior implementation PR is closed unmerged and its review notes identified incomplete parameter threading, so it does not supersede this request.

## What This Changes

The issue asks for a per-channel or per-agent option to render visible reasoning blocks as italic, code, spoiler, blockquote, or a similar format instead of always italic.

## Best Possible Solution

Define a documented reasoning display-format enum with italic as the compatibility default, explicit precedence, schema/types/docs coverage, runner and channel-edge threading, focused tests, and Telegram-visible proof once the contract is chosen.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection of current main and v2026.5.22 gives high confidence that OpenClaw still lacks a display-format key and still formats visible Telegram reasoning through the fixed italic formatter.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a small documented enum with explicit precedence and end-to-end threading, not a formatter-only patch.
