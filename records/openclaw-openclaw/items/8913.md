## Summary

Keep open. Current main still hardcodes visible reasoning as italic Markdown and lacks the requested per-channel or per-agent display-format setting; the related implementation PR #25336 is closed and unmerged.

## What This Changes

Add a per-channel or per-agent configuration option to choose how visible reasoning blocks are rendered, especially for Telegram.

## Best Possible Solution

Define a documented reasoning display-format contract, keep italic as the compatibility default, decide config precedence, then thread the selected format through core reasoning delivery and Telegram with regression tests and docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main still produces fixed italic reasoning output and lacks the requested configuration surface.

## Solution Assessment

Unclear until maintainers settle the public config contract. The requested direction is reasonable, but adding a new config option and supported format set is not a narrow bug fix.
