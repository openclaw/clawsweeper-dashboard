## Summary

Keep open: current main and the latest shipped release still have fixed italic reasoning rendering and no per-channel or per-agent display-format config. The linked implementation PR was closed unmerged after review found incomplete parameter threading, so the remaining work needs maintainer product/config decisions rather than conservative cleanup closure.

## What This Changes

The issue requests a public configuration option to choose how reasoning/thinking blocks are rendered per channel or agent, especially for Telegram readability.

## Best Possible Solution

Define a documented reasoning display-format contract with italic as the compatibility default, explicit precedence, schema/types/docs coverage, channel-edge implementation, focused regression tests, and Telegram-visible proof for chat rendering changes.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main still lacks the requested display-format key and Telegram still routes visible reasoning through the fixed formatter.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a small documented enum with explicit precedence and channel-edge threading, not a local-only formatter parameter.
