## Summary

Keep open. Current main and the latest release still expose reasoning visibility but not a per-channel or per-agent reasoning display-format setting, and the linked implementation attempt is closed unmerged with incomplete threading called out in review.

## What This Changes

Request to add configurable reasoning/thinking block presentation, especially for Telegram, with formats such as italic, code, spoiler, or blockquote at channel or agent scope.

## Best Possible Solution

Define one small reasoning display-format contract with italic as the compatibility default, document precedence, and thread it through shared agent delivery plus Telegram-owned rendering with focused regression tests.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.10 still have fixed Thinking plus italic-line rendering and no display-format config key. This is a feature gap rather than a failing bug reproduction, so I did not run live Telegram proof.

## Solution Assessment

Unclear until maintainers choose the product contract. The maintainable solution is one narrow display-format contract threaded through config, shared agent delivery, and channel-owned rendering rather than a formatter-only option.
