## Summary

Keep open: current main and v2026.5.20 still expose reasoning visibility but not a configurable reasoning display-format contract. The formatter and Telegram reasoning lane still use the fixed Thinking-plus-italic shape, while the only linked implementation PR closed unmerged after review found incomplete runner threading.

## What This Changes

The issue requests a per-channel or agent-level configuration option to control how visible thinking/reasoning blocks render, especially for Telegram readability.

## Best Possible Solution

Define a documented reasoning display-format enum with italic as the compatibility default, explicit precedence, schema/types/docs coverage, runner and channel-edge threading, focused tests, and Telegram-visible proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main and v2026.5.20 lack the requested display-format key and still format visible Telegram reasoning through fixed italic output.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a small documented enum with explicit precedence and end-to-end threading, not a local-only formatter parameter.
