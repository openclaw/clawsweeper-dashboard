## Summary

Keep open. Current main still hardcodes italic `Reasoning:` output and exposes no per-channel or agent-level display-format config; the related implementation PR was closed unmerged and did not supersede this request.

## What This Changes

The issue requests configurable reasoning/thinking display formatting, especially for Telegram, via per-channel or agent-level options such as italic, code, or spoiler.

## Best Possible Solution

Define a backward-compatible reasoning display-format contract, keep italic as the default, document precedence, and thread the selected format through all core and Telegram reasoning delivery paths with regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence the requested capability is absent: the formatter only accepts text and the config schema exposes reasoning visibility but no display-format key.

## Solution Assessment

Unclear until maintainers choose the public config contract. The narrow maintainable direction is a shared display-format enum with documented precedence, then consistent threading through runner, subscribe, message-end, and Telegram paths.
