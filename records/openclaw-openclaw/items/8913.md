## Summary

Keep open. Current main still lacks a public reasoning display-format config, Telegram reasoning delivery still calls the fixed italic formatter, and the closed implementation PR did not merge; the later raw-reasoning refactor only moved formatting responsibility to channel edges.

## What This Changes

The issue requests a configurable per-channel or per-agent display format for visible thinking/reasoning blocks, especially to improve Telegram readability.

## Best Possible Solution

Define a documented reasoning display-format contract with italic as the compatibility default, then thread a prepared selected format through config schema/types/docs and the channel-edge reasoning delivery paths with focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks the requested display-format configuration while Telegram visible reasoning still uses the fixed italic formatter.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is to avoid core channel-ID special cases, preserve the current default, and pass a selected display-format value to the channel delivery edge that formats visible reasoning.
