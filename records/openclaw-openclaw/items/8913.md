## Summary

Keep open. Current main still lacks a reasoning display-format config, Telegram visible reasoning still routes through the fixed italic formatter, and the linked implementation PR is closed unmerged with incomplete threading feedback.

## What This Changes

The issue requests a configurable reasoning/thinking block display format per channel or agent, especially to improve Telegram readability.

## Best Possible Solution

Define a documented reasoning display-format contract with italic as the compatibility default, then thread a prepared display-format value through config schema/types/docs, Telegram reasoning lanes, channel edge formatting, streaming/block reply paths, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection of current main gives high confidence that the requested display-format configuration is absent and Telegram visible reasoning still uses the fixed formatter.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is to preserve the current default, avoid core channel-ID special cases, and pass a selected display-format value to every reasoning delivery edge.
