## Summary

Keep open. Current main still hardcodes visible reasoning as italic Markdown, exposes reasoning visibility but no display-format setting, and the only implementation PR is closed and unmerged with known incomplete threading.

## What This Changes

Requested a configurable reasoning/thinking display format so channels such as Telegram can render visible reasoning as code, spoiler, blockquote, or another selected style instead of hardcoded italic Markdown.

## Best Possible Solution

Define a documented display-format contract with italic as the compatibility default, then thread it through shared formatter, runner/subscribe payloads, Telegram reasoning lanes, config schema/types/help/docs, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection of current main confirms the requested display-format configuration is absent and visible reasoning formatting remains fixed italic.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is to preserve the current default, avoid hardcoding channel IDs in core, and pass the selected format through every reasoning delivery path.
