## Summary

Keep open. Current main still lacks a public reasoning display-format setting, Telegram reasoning still reaches the fixed italic formatter at the delivery edge, and the only implementation PR for this request is closed unmerged.

## What This Changes

Request configurable per-channel or per-agent reasoning display formatting so visible reasoning can use alternatives such as code, spoiler, blockquote, or italic instead of fixed italics.

## Best Possible Solution

Define a small documented reasoning-display-format contract with italic as the compatibility default, choose precedence for agent/global/channel scopes, thread it to channel delivery edges, and cover Telegram/default behavior with regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks a display-format setting while Telegram visible reasoning still formats through the fixed italic helper.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is to preserve the current default, avoid core channel-ID special cases, and pass the selected display format to the channel delivery edge that renders visible reasoning.
