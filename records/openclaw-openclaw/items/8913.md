## Summary

Keep open: current main still lacks a configurable reasoning display-format contract, and the Telegram path still formats visible reasoning through the fixed italic helper while schemas and docs expose visibility only.

## What This Changes

Add a per-channel or agent-level configuration option for choosing visible reasoning/thinking block formatting such as italic, code, spoiler, or blockquote.

## Best Possible Solution

Define a documented reasoning display-format enum with compatibility defaults, explicit precedence, schema/type/docs coverage, and channel-edge threading across visible reasoning delivery paths.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection confirms no display-format key exists and Telegram still routes visible reasoning through the fixed italic formatter.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a narrow documented enum with explicit precedence and channel-edge formatting, not only adding a parameter to `formatReasoningMessage`.
