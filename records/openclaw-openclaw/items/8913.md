## Summary

Keep open: current main and the latest release still hardcode displayed reasoning as italic `Thinking` messages and expose no per-channel or per-agent display-format setting. The remaining work is a product/config decision rather than a cleanup close.

## What This Changes

Request configurable per-channel or per-agent formatting for displayed thinking/reasoning blocks, especially Telegram code, spoiler, or italic presentation.

## Best Possible Solution

Define one small reasoning display-format contract with italic as the compatibility default, clear channel/global precedence, docs/schema/type coverage, and tests across shared and Telegram-visible paths.

## Reproduction Assessment

Not applicable as a bug reproduction, but source inspection gives high confidence in the current behavior: displayed reasoning is still fixed italic and no display-format parameter is present.

## Solution Assessment

Unclear until maintainers choose the product shape. The narrow implementation should add a single contract and preserve the default, not create competing per-channel ad hoc formatting paths.
