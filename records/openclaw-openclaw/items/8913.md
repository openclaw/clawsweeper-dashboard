## Summary

Current main and the latest release still lack the requested configurable reasoning display format; the shared formatter and Telegram reasoning lane remain fixed to the existing `Thinking` plus italic-line shape, while the only linked implementation attempt is closed unmerged and incomplete.

## What This Changes

The issue requests a per-channel or per-agent configuration option to choose how thinking/reasoning blocks are rendered, especially for Telegram readability.

## Best Possible Solution

Define one small reasoning display-format contract with italic as the compatibility default, choose clear precedence for channel versus agent/global defaults, document it, and thread it through shared agent delivery plus Telegram reasoning lanes with regression tests.

## Reproduction Assessment

Yes, by source inspection: current main still has fixed reasoning display formatting and no display-format config input in the shared formatter, agent params, or Telegram reasoning lane. This is a feature gap rather than a failing bug reproduction.

## Solution Assessment

Unclear until maintainers choose the product contract. The requested direction is plausible, but the best fix needs one narrow format contract threaded through config, shared agent delivery, and channel-owned rendering rather than only adding a low-level formatter option.
