## Summary

Keep open. Current main still lacks any per-channel or per-agent reasoning display-format setting, and the only linked implementation PR was closed unmerged after review found incomplete parameter threading.

## What This Changes

Request a configurable reasoning/thinking block display format for channel output, especially replacing Telegram’s hardcoded italic rendering with code, spoiler, blockquote, or similar formats.

## Best Possible Solution

Define a small documented reasoning display-format contract, preserve italic as the compatibility default, thread the selected value to the channel delivery edges, and add regression coverage for default and configured formats.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks the requested display-format setting while Telegram reasoning still formats through the fixed italic helper.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a narrow format enum with compatibility defaults and explicit threading to the rendering edge, not an ad hoc formatter-only change.
