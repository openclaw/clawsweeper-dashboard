## Summary

Keep open: current main and v2026.5.22 still expose reasoning visibility, but not a configurable reasoning display-format contract. The formatter and Telegram reasoning lane remain fixed to the italic `Thinking` message shape, and the prior implementation PR is closed unmerged with known incomplete threading.

## What This Changes

The issue requests a per-channel or per-agent configuration option for formatting visible thinking/reasoning blocks, especially to improve Telegram readability.

## Best Possible Solution

Define a documented reasoning display-format enum with italic as the compatibility default, explicit precedence, config schema/types/docs coverage, runner and channel-edge threading, focused tests, and Telegram-visible proof after the contract is chosen.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection of current main and v2026.5.22 gives high confidence that OpenClaw still lacks a display-format key and still formats visible Telegram reasoning through the fixed italic formatter.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a small documented enum with explicit precedence and end-to-end threading, not a formatter-only patch.
