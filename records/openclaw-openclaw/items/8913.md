## Summary

Keep open. Current main still lacks a public reasoning display-format setting, Telegram reasoning still reaches the fixed italic formatter, and the closed implementation PR was not merged.

## What This Changes

The issue requests configurable per-channel or per-agent display formatting for visible thinking/reasoning blocks, especially Telegram.

## Best Possible Solution

Define a documented reasoning display-format contract with italic as the compatibility default, then thread the selected value through config schema/types/docs and channel-edge reasoning delivery paths with focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks the requested display-format configuration while Telegram visible reasoning still uses the fixed italic formatter.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is to avoid core channel-ID special cases, preserve the current default, and pass a selected display-format value to the channel delivery edge that formats visible reasoning.
