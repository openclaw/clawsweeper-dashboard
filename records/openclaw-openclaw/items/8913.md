## Summary

Keep open: current main and the latest release still have a fixed italic reasoning display with no per-channel or per-agent display-format setting, and the linked implementation attempt was closed unmerged after review found incomplete threading. This is useful channel ergonomics work, but it adds config/product surface and needs maintainer direction before implementation.

## What This Changes

Request a configurable display format for visible thinking/reasoning blocks, especially Telegram code, spoiler, or blockquote formatting instead of hardcoded italics.

## Best Possible Solution

Define one small reasoning display-format contract with italic as the compatibility default, clear channel versus agent/global precedence, schema/docs coverage, full pipeline threading, and focused shared plus Telegram regression tests.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: the current formatter and Telegram lane have fixed italic output and no display-format parameter or config field. I did not run a live Telegram scenario because this read-only review could establish the missing capability from source and docs.

## Solution Assessment

Unclear until maintainers choose the product shape. The narrow maintainable path is not just adding a formatter option; it should settle naming and precedence, avoid conflicting with provider `thinkingFormat`, and cover every reasoning delivery lane.
