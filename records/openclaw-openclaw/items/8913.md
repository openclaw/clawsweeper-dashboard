## Summary

Keep open. Current main and the latest release still have a fixed `Thinking` plus italic-line reasoning display, no per-channel or per-agent display-format setting, and the only linked implementation attempt is closed unmerged with incomplete threading noted in review.

## What This Changes

Request to add a configurable reasoning/thinking block display format, especially for Telegram, with options such as italic, code, spoiler, or blockquote at channel or agent scope.

## Best Possible Solution

Define one small reasoning display-format contract with italic as the compatibility default, choose clear precedence for channel versus agent/global defaults, document it, and thread it through shared agent delivery plus Telegram reasoning lanes with regression tests.

## Reproduction Assessment

Yes, source-level. Current main and `v2026.6.10` still expose reasoning visibility but not display-format configuration, so this is a feature gap rather than a failing bug reproduction.

## Solution Assessment

Unclear until maintainers choose the product contract. The maintainable path is one narrow display-format contract threaded through config, shared agent delivery, and channel-owned rendering rather than a formatter-only option.
