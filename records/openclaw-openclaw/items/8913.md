## Summary

Keep open: current main still lacks a configurable reasoning display-format contract, and the relevant Telegram/channel-edge paths still call the fixed italic formatter. The prior implementation attempt is closed unmerged and its review identified incomplete parameter threading, so it does not supersede the request.

## What This Changes

The issue requests a per-channel or per-agent configuration option for how visible thinking/reasoning blocks are formatted, especially in Telegram.

## Best Possible Solution

Define a documented reasoning display-format enum with compatibility defaults, explicit precedence, schema/type/docs coverage, and channel-edge threading across visible reasoning delivery paths.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection on current main confirms no display-format config key exists and Telegram reasoning still routes through the fixed italic formatter.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a narrow documented enum with explicit precedence and edge formatting, not only adding another parameter to the low-level formatter.
