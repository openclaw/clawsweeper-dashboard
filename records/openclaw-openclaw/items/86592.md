## Summary

Keep open. Current main still has a source-visible ACP path where the accepted user message is only persisted after `runTurn` completes, and the issue is already paired with an open mergeable fixing PR that uses GitHub closing syntax.

## What This Changes

The issue reports that accepted inbound user turns can be lost from session JSONL when agent or ACP attempts fail, leaving replay and compaction with one-sided transcript state.

## Best Possible Solution

Land or close the linked fixing PR after review; the durable end state is that accepted user turns are persisted before ACP/CLI/internal attempts can fail, without duplicate user rows on successful completion.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: make the ACP `runTurn` call reject on current main and execution rethrows before `persistAcpTurnTranscript` appends the accepted user row. I did not run a live reproduction in this read-only review.

## Solution Assessment

Yes, the best direction is pre-run user-turn persistence with duplicate suppression on later success, which is what the open fixing PR is attempting. Starting a second repair lane would be worse while that mergeable PR remains open.
