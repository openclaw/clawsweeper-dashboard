## Summary

Keep open. Current main partially addresses the CLI/embedded side with pre-attempt user-turn persistence, but the ACP branch still rethrows before its user/assistant transcript helper runs, and the open linked PR is still the active implementation candidate for the remaining session-state fix.

## What This Changes

The issue reports that accepted inbound user turns can be omitted from session JSONL when an agent attempt throws, leaving recovery and compaction with incomplete session history.

## Best Possible Solution

The end state should make accepted user turns durable before any fallible CLI, embedded, ACP, topic, or internal session-effects runtime attempt, while suppressing duplicate user rows after successful persistence.

## Reproduction Assessment

Yes. Source inspection shows an ACP runTurn failure still rethrows before persistAcpTurnTranscript, and this read-only review did not run a live Telegram or ACP reproduction.

## Solution Assessment

Yes. The best fix direction is early approved user-turn persistence with duplicate suppression on later success; the linked PR or an equivalent refreshed implementation should cover the remaining ACP/internal path before this issue closes.
