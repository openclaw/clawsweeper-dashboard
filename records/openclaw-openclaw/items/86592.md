## Summary

Keep open. Current main still has source-visible ACP paths where an accepted turn can throw before transcript persistence appends the user row, and https://github.com/openclaw/openclaw/pull/86764 is the open linked fix candidate for this issue.

## What This Changes

The issue reports accepted inbound user messages being omitted from session transcript JSONL when agent attempts fail, with follow-up evidence that Telegram topic sessions can also miss user rows on successful turns.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/86764 so accepted CLI, ACP, internal/session-effects, and Telegram topic user turns persist before fallible attempts without duplicate rows on success.

## Reproduction Assessment

Yes. Source inspection on current main shows ACP runTurn can throw before persistAcpTurnTranscript or persistAcpDispatchTranscript runs, matching the reported missing user-row failure mode; I did not run live Telegram or ACP in this read-only sweep.

## Solution Assessment

Yes. The maintainable fix direction is to persist the accepted user turn after policy acceptance and before fallible backend execution, then suppress duplicate user rows during later successful assistant persistence.
