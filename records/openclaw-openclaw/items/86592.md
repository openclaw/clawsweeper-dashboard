## Summary

Keep open. Current main has partial embedded-run protection, but source inspection still shows CLI-backed attempts can throw before the post-run transcript persistence block that appends the user turn, so the reported message-loss/session-wedge class is still actionable.

## What This Changes

Ensure inbound user turns are appended to session JSONL even when the agent attempt throws before normal post-run transcript persistence.

## Best Possible Solution

Persist the accepted current user turn before, or failure-safely around, CLI/ACP attempts with idempotency, while keeping assistant persistence post-run and preserving embedded Pi duplicate suppression.

## Solution Assessment

Yes, the narrow fix is to decouple current-user persistence from successful assistant completion for CLI/ACP paths. The implementation should add regression coverage for failure and fallback cases and avoid duplicating embedded Pi user messages.
