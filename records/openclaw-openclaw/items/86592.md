## Summary

Keep open. Current main still has a source-visible ACP failure path where the turn throws before ACP transcript persistence can append the accepted user message, and the issue is already paired with an open fixing PR using GitHub closing syntax.

## What This Changes

The issue asks OpenClaw to persist accepted inbound user turns to session JSONL before fallible agent attempts so failed turns do not disappear from session replay and compaction state.

## Best Possible Solution

Land or replace the active linked fix so accepted CLI, ACP, embedded/internal, and Telegram topic user turns are persisted before fallible attempts without duplicate rows on success.

## Reproduction Assessment

Yes. Source inspection on current main shows ACP attempt failure rethrows before ACP transcript persistence, matching the reported missing user-row failure mode; this read-only review did not run live Telegram or ACP.

## Solution Assessment

Yes. The best fix direction is early accepted-user-turn persistence with later duplicate suppression; the active linked PR or an equivalent refreshed fix should cover ACP/internal and topic/session routing before this closes.
