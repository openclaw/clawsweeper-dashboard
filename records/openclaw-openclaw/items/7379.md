## Summary

Keep open. Current `main` and the latest release still do not route provider quota windows into the per-response `/usage` footer; `/status` and the GitHub Copilot plugin already expose the needed Premium/Chat usage-window data, while the direct fix PR is closed unmerged and the broader footer PR only partially overlaps.

## What This Changes

Request to show GitHub Copilot Premium/Chat quota percentages in the per-response `/usage` footer when provider usage windows are available.

## Best Possible Solution

Add a generic active-provider usage-window summary to the existing `/usage` footer, reusing plugin-owned provider usage hooks and `formatUsageWindowSummary` while omitting quota text on auth ambiguity, timeout, or fetch failure.

## Reproduction Assessment

Yes, at source level. Enabling `/usage tokens` or `/usage full` reaches the reply footer path in `agent-runner.ts`, and current `main` never loads provider usage windows there even though `/status` does.

## Solution Assessment

Yes. The best fix is a generic active-provider usage-window composition point in the reply footer, not GitHub Copilot-specific core text and not a new config surface.
