## Summary

Current main still omits active-provider quota windows from per-response `/usage` footers. The generic provider usage data and formatter already exist, but the reply path only consumes turn tokens, cache counts, and optional cost; the direct fix PR is closed unmerged, and the broader open footer PR is conflicting, lacks real behavior proof, and does not explicitly own this request.

## What This Changes

Add active-provider quota-window percentages, such as GitHub Copilot Premium and Chat remaining percentages, to per-response `/usage` footers.

## Best Possible Solution

At the reply-footer boundary, reuse a cached generic usage snapshot for the active provider and append `formatUsageWindowSummary` output when available, while keeping authentication and vendor endpoint fetching inside provider plugins and leaving the existing footer unchanged on missing or failed quota data.

## Reproduction Assessment

Yes, at source level with high confidence: current main's enabled reply footer has no provider-usage input, while `/status` and the GitHub Copilot plugin already expose the requested quota windows. A live OAuth-backed response was not executed during this read-only review.

## Solution Assessment

Yes. Generic active-provider integration through the existing provider-usage contract is the narrowest maintainable solution; a Copilot-only core fetch or a new configuration option would duplicate policy and cross ownership boundaries.
