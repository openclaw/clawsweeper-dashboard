## Best Possible Solution

Add a generic active-provider usage-window append to `/usage tokens` and `/usage full` reply footers using the existing provider usage loader/formatter, while keeping Copilot-specific fetching inside the plugin and updating focused tests, docs, and changelog.

## Solution Assessment

Yes. The maintainable path is a generic reply-footer bridge to existing provider usage windows, not Copilot-specific policy in core; the prior direct PR demonstrated the shape but is closed unmerged and should be reimplemented against current main.
