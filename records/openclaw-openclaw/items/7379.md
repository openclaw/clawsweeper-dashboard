## Summary

Keep open. Current main still does not add provider quota windows to the per-response `/usage` footer, while `/status` and the GitHub Copilot plugin already expose the Premium/Chat data needed for the requested display.

## What This Changes

Add GitHub Copilot Premium/Chat quota-window percentages to the per-response `/usage` footer.

## Best Possible Solution

Add a generic provider-window summary to the existing reply-footer path, keeping provider-specific quota fetching inside plugins and updating docs, changelog, and regression coverage.

## Reproduction Assessment

Yes, at source level. Enable `/usage tokens` or `/usage full` for a reply with nonzero usage and an active provider that exposes usage windows; current main appends only token/cache/cost/session text and never loads provider usage windows in the reply-footer path.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the generic provider-usage loader plus `formatUsageWindowSummary` in the reply-footer path, not to add GitHub Copilot-specific quota fetching to core reply code.
