## Summary

Keep open. Current main still limits the per-response `/usage` footer to turn token, cache, cost, and session data; provider quota windows remain available only through the separate status/provider-usage path. The direct implementation at https://github.com/openclaw/openclaw/pull/43098 closed unmerged, while https://github.com/openclaw/openclaw/pull/69618 is a broad overlapping footer proposal that does not implement this Premium/Chat quota request.

## What This Changes

Add active-provider premium quota percentages, such as GitHub Copilot Premium and Chat windows, to the per-response `/usage` footer.

## Best Possible Solution

Extend the generic reply usage-footer boundary to append a compact active-provider quota-window summary from the existing provider-usage interfaces, while retaining all GitHub Copilot API fetching and normalization inside its plugin and bounding network or cache latency.

## Reproduction Assessment

Yes at source level. Current main can be traced from `/usage` mode resolution through the token-only footer formatter and final payload append, while the separate `/status` path demonstrates that provider quota windows are available but not passed into replies.

## Solution Assessment

Yes, with the generic owner boundary described here. Reusing provider-usage snapshots and `formatUsageWindowSummary` is narrower and more maintainable than placing Copilot-specific API logic in the core reply runner.
