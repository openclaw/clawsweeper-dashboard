## Summary

Keep open. Current main still has the requested provider quota-window data for `/status` and the GitHub Copilot plugin, but the per-response `/usage` footer only formats token/cache/cost/session data and never loads or appends Premium/Chat usage windows. The direct implementation PR #43098 is closed unmerged, and open #69618 overlaps footer plumbing without claiming this specific quota-window request.

## What This Changes

Request to append GitHub Copilot Premium/Chat quota-window percentages to the per-response `/usage` footer when available.

## Best Possible Solution

Add a generic reply-footer integration that appends the existing provider usage-window summary for the active provider when response usage is enabled, while preserving plugin-owned quota fetching, OAuth/API-key auth rules, and current token/cost/session footer behavior. Document the new surface and add focused regression coverage for the reply-runner footer path.
