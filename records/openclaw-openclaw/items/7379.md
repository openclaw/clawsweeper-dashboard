Keep open. Add GitHub Copilot Premium/Chat provider usage percentages to the per-response `/usage tokens` and `/usage full` footer.

## Best Possible Solution

Add a generic provider-window summary to the reply footer using the existing provider-usage loader and formatter, keeping provider-specific quota fetching inside plugins and updating docs, changelog, and regression coverage.

## Solution Assessment

Fix the per-response `/usage` footer enhancement requested in https://github.com/openclaw/openclaw/issues/7379. When `/usage tokens` or `/usage full` is enabled and the active provider exposes provider usage windows, append a compact generic provider-window summary such as `Premium 86% left · Chat 100% left` to the existing footer. Reuse the existing provider usage plumbing (`loadProviderUsageSummary`, `resolveUsageProviderId`, `formatUsageWindowSummary`) and keep provider-specific fetching inside plugins; do not add GitHub Copilot-specific fetching to core reply code and do not add a new config option. Coordinate with https://github.com/openclaw/openclaw/pull/69618 because it touches nearby footer composition. Stop and report back if the implementation would require a new plugin API or product-policy decision.
