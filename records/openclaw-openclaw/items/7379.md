## Summary

Keep open. Current main still lacks the requested bridge between provider quota windows and the per-response `/usage` footer: the reply path formats token/cache/cost/session data only, while `/status` and the GitHub Copilot plugin already expose the Premium/Chat usage windows.

## What This Changes

Add GitHub Copilot Premium/Chat quota percentages to the per-response `/usage` footer when `/usage tokens` or `/usage full` is enabled.

## Best Possible Solution

Implement a generic reply-footer enhancement that appends `formatUsageWindowSummary` for the active provider when `/usage tokens` or `/usage full` is enabled, keeping provider-specific fetching inside plugin usage hooks and adding focused docs/changelog/test coverage.

## Reproduction Assessment

Yes, by source inspection rather than a live run. Current main's reply footer path never loads provider usage windows, so a Copilot OAuth session with `/usage tokens` or `/usage full` would still only get token/cache/cost/session footer content while `/status` can show Premium/Chat.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse the existing provider usage contracts and formatter in the reply footer with bounded, error-swallowed loading, rather than adding Copilot-specific core behavior.
