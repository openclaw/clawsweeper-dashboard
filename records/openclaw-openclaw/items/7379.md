## Summary

Keep open. Current main still builds the per-response `/usage` footer from turn tokens, cache, cost, and session data only; `/status` already loads provider usage windows and the GitHub Copilot plugin already exposes Premium/Chat windows, so the requested bridge remains missing. The direct implementation PR is closed unmerged, and the broader open footer PR does not close this specific quota-window request.

## What This Changes

Add active provider usage-window percentages, especially GitHub Copilot Premium/Chat quota, to the per-response `/usage` footer.

## Best Possible Solution

Add a generic active-provider usage-window append to the reply footer using the existing provider usage loader and formatter, keeping provider-specific fetching inside plugins and covering docs, changelog, and reply-runner regression tests.

## Reproduction Assessment

Not applicable as a feature request rather than broken documented behavior. Source inspection gives high-confidence evidence that current `/usage` footers do not include provider quota windows while `/status` already does.

## Solution Assessment

Yes. The maintainable solution is a generic active-provider usage-window bridge into the footer, not a Copilot-only special case in core; coordinate with https://github.com/openclaw/openclaw/pull/69618 because it changes nearby footer plumbing.
