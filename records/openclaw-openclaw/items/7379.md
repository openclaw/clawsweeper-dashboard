## Summary

Current main still lacks the requested bridge between provider quota windows and the per-response `/usage` footer. The reply path only formats token/cache/cost/session data, while `/status` already loads generic provider usage windows and GitHub Copilot already exposes Premium/Chat windows through the plugin usage hook.

## What This Changes

Request to show GitHub Copilot Premium/Chat quota percentages in the per-response `/usage` footer when that provider usage data is available.

## Best Possible Solution

Add a generic reply-footer enhancement that appends `formatUsageWindowSummary` output when `/usage tokens` or `/usage full` is enabled and the active provider exposes usage windows, while keeping provider-specific fetching inside plugins.
