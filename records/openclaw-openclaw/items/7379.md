## Summary

Keep open. Current main still builds the per-response `/usage` footer from token/cache/cost/session data only, while `/status` and the GitHub Copilot plugin already expose the provider usage windows needed for Premium/Chat quota text.

## What This Changes

Add provider quota-window text, such as GitHub Copilot Premium/Chat percentage, to the per-response `/usage` footer when available.

## Best Possible Solution

Add a generic provider-window summary to the reply footer using the existing provider-usage loader and formatter, keeping provider-specific quota fetching inside plugins and updating docs, changelog, and regression coverage.

## Reproduction Assessment

Yes, at source level. Enable `/usage tokens` or `/usage full` for a reply with nonzero usage and an active provider that exposes usage windows; current main appends only the token/cache/cost/session footer path.

## Solution Assessment

Yes. The narrow maintainable direction is to reuse the generic provider-usage loader plus `formatUsageWindowSummary` in the reply-footer path, not to add GitHub Copilot-specific quota fetching to core reply code.
