## Summary

The direct implementation PR was closed unmerged, while the broader open footer PR is conflicting, lacks real behavior proof, and does not claim this request, so the feature remains necessary and has no safe canonical replacement.

## What This Changes

Add active-provider quota-window percentages, especially GitHub Copilot Premium and Chat balances, to per-response `/usage` footers.

## Best Possible Solution

Append a compact shared `formatUsageWindowSummary` result for the active provider to enabled reply usage footers, using bounded best-effort loading while retaining all provider-specific endpoint logic and labels in the owning plugin.

## Reproduction Assessment

Yes, at source level. Current `main` has a concrete reply-footer path that formats only token, cache, and cost data, while `/status` separately loads and formats the active provider's quota windows.

## Solution Assessment

Yes. Reusing the generic provider-usage loader and formatter in the reply pipeline is the narrowest maintainable solution; Copilot-specific fetching must remain in its plugin and message delivery must remain best-effort.
