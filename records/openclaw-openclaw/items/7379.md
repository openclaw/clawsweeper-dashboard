## Summary

Keep open. Current main still builds the per-response `/usage` footer from turn token/cache/cost/session data only, while `/status` and the GitHub Copilot plugin already expose reusable Premium/Chat provider usage windows; the direct implementation PR at https://github.com/openclaw/openclaw/pull/43098 is closed unmerged and the broader footer PR at https://github.com/openclaw/openclaw/pull/69618 does not claim this quota-window request.

## What This Changes

The issue requests showing GitHub Copilot Premium/Chat quota percentages in the per-response `/usage` footer.

## Best Possible Solution

Add a generic, bounded provider-usage-window append to the existing `/usage` footer path, reuse plugin-owned `loadProviderUsageSummary` plus `formatUsageWindowSummary`, document the new footer behavior, and keep Copilot-specific quota fetching inside the plugin.

## Solution Assessment
