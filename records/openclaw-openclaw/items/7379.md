## Summary

Keep open. Current main still keeps provider quota windows out of the per-response footer: reply code composes token/cache/cost/session text only, while the status and GitHub Copilot paths already expose the Premium/Chat windows needed for this request.

## What This Changes

Add active-provider quota-window remaining percentages, such as GitHub Copilot Premium/Chat left, to the per-response `/usage` footer.

## Best Possible Solution

Add a generic provider-window summary to the reply footer using the existing provider usage loader and formatter, with timeout/error handling, docs, changelog, and regression coverage.

## Reproduction Assessment

Yes, at source level. Enable `/usage tokens` or `/usage full` for a reply with nonzero usage and an active provider that exposes usage windows; current main only appends the token/cache/cost/session footer path.

## Solution Assessment

Yes. The narrow maintainable direction is to reuse generic provider usage hooks plus `formatUsageWindowSummary` in the reply-footer path, not to add GitHub Copilot-specific fetching to core reply code.
