## Summary

Keep open. Current main still formats the per-response `/usage` footer from token/cache/cost/session data only, while `/status` and the GitHub Copilot plugin already expose the Premium/Chat provider usage windows needed for the requested footer addition.

## What This Changes

Request to append available provider quota-window summaries, especially GitHub Copilot Premium/Chat, to the per-response `/usage` footer.

## Best Possible Solution

Add a generic provider-window summary to the response footer when `/usage tokens` or `/usage full` is enabled, keeping provider-specific fetching inside plugins and updating docs, changelog, and regression coverage.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: enable `responseUsage` for a reply with nonzero model usage and an active provider that has usage windows; current main formats only token/cache/cost/session text and never loads provider usage for that footer.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse `loadProviderUsageSummary` and `formatUsageWindowSummary` in the reply-footer composition path, not to add GitHub Copilot-specific fetching inside core reply code.
