## Summary

Current main and the latest v2026.6.5 release still omit provider quota windows from per-response usage footers. The request remains a narrow, source-reproducible feature; the direct fix PR closed unmerged, while the broader open footer PR is conflicting and does not own this specific behavior.

## What This Changes

Append active provider quota windows, especially GitHub Copilot Premium and Chat percentages, to per-response `/usage` footers.

## Best Possible Solution

Append a compact active-provider `formatUsageWindowSummary` to enabled response usage footers through the generic provider-usage contract, with bounded loading and no Copilot-specific core branch.

## Reproduction Assessment

Yes at source level: current main's reply footer has no provider-usage input, while `/status` loads and formats the same active-provider windows. This establishes the missing behavior without needing a failing runtime test.

## Solution Assessment

Yes. The narrow maintainable direction is to reuse the generic provider-usage contract and formatter while leaving Copilot fetching inside its plugin; a provider-specific core fetch or new configuration surface would be worse.
