## Summary

Keep open. Current main still builds the per-response `/usage` footer from token/cache/cost/session data only, while `/status` and the GitHub Copilot plugin already expose the requested Premium/Chat quota windows; the direct implementation PR at https://github.com/openclaw/openclaw/pull/43098 is closed unmerged, and the open footer-related PRs do not currently own this specific quota-window request.

## What This Changes

Request to add GitHub Copilot Premium/Chat quota percentages to the per-response `/usage` footer.

## Best Possible Solution

Add a generic active-provider usage-window summary to `/usage tokens` and `/usage full`, keeping provider-specific fetching inside plugin hooks and reusing `formatUsageWindowSummary`.

## Solution Assessment
