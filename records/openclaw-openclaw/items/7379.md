## Summary

Keep open. Current main and latest release v2026.6.5 still leave `/usage` reply footers limited to token/cache/cost/session text, while `/status` already loads provider usage windows and the GitHub Copilot plugin already exposes Premium/Chat windows; the direct fix at https://github.com/openclaw/openclaw/pull/43098 is closed unmerged, and https://github.com/openclaw/openclaw/pull/69618 overlaps footer plumbing without owning this quota-window request.

## What This Changes

Request to add GitHub Copilot Premium/Chat quota-window percentages to the per-response `/usage` footer so users do not need to run `/status` separately.

## Best Possible Solution

Add a generic active-provider usage-window summary to `/usage tokens` and `/usage full` reply footers using the existing provider usage loader/formatter, while keeping Copilot-specific fetching inside the GitHub Copilot plugin.

## Reproduction Assessment

Yes, at source level. Current `main` formats `/usage` reply footers from turn token/cache/cost/session data only, while `/status` loads provider usage windows and the Copilot plugin supplies Premium/Chat windows.

## Solution Assessment

Yes. The narrow maintainable solution is a generic reply-footer bridge to existing provider usage windows, not Copilot-specific fetching or policy in core.
