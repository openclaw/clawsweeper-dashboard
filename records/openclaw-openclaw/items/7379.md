## What This Changes

The issue requests adding GitHub Copilot Premium/Chat quota percentages to the per-response `/usage` footer when provider usage data is available.

## Best Possible Solution

Add a generic provider-usage-window append to the existing `/usage` footer path, reusing plugin-owned `loadProviderUsageSummary` and shared `formatUsageWindowSummary` without Copilot-specific policy in core.

## Reproduction Assessment

## Solution Assessment

Yes. The narrow maintainable path is a generic reply-footer enhancement that reuses the existing provider usage loader/formatter, rather than adding Copilot-specific footer logic or waiting on a broader footer redesign.
