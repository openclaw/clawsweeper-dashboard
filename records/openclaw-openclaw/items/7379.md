## Summary

Keep open. Current main still builds the per-response `/usage` footer from token/cache/cost/session data only, while `/status` and the GitHub Copilot plugin already expose the requested Premium/Chat usage windows; the direct implementation PR at https://github.com/openclaw/openclaw/pull/43098 is closed unmerged, and nearby footer PRs do not currently claim this quota-window request.

## What This Changes

Request to append GitHub Copilot Premium/Chat provider quota percentages to the per-response `/usage` footer when available.

## Best Possible Solution

Add a generic active-provider usage-window summary to `/usage tokens` and `/usage full`, keeping provider-specific fetching inside plugin hooks and reusing `formatUsageWindowSummary`.

Implement https://github.com/openclaw/openclaw/issues/7379 as a narrow generic reply-footer enhancement. When `/usage tokens` or `/usage full` is enabled and the active provider has provider usage windows, append a compact `formatUsageWindowSummary` result to the existing per-response usage footer. Keep provider-specific fetching inside existing plugin/provider-usage hooks; do not add Copilot-specific core formatting.
