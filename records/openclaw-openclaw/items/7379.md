## Summary

Keep open. Current main still formats the per-response `/usage` footer from token/cache/cost/session data only, while `/status` and the GitHub Copilot plugin already expose the Premium/Chat provider usage windows needed for this enhancement.

## What This Changes

Add GitHub Copilot Premium/Chat provider quota percentages to per-response `/usage` footers when available.

## Best Possible Solution

Append a compact generic provider-window summary to `/usage tokens` and `/usage full` when the active provider exposes windows, keeping provider-specific fetching inside plugins and updating docs, changelog, and regression coverage.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse the existing generic provider-usage loader and formatter in the reply-footer composition path, not to add GitHub Copilot-specific fetching inside core reply code.
