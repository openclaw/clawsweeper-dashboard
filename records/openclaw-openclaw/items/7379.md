## Summary

Keep open. Current main still lacks the requested bridge from provider quota windows into the per-response `/usage` footer, while `/status` and the GitHub Copilot plugin already expose the needed Premium/Chat data.

## What This Changes

Add GitHub Copilot Premium/Chat provider quota percentage to the `/usage` per-response footer when usage footers are enabled.

## Best Possible Solution

Add a generic reply-footer integration that appends the existing active-provider usage-window summary when `/usage tokens` or `/usage full` is enabled, while keeping provider-specific quota fetching inside plugins.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to enable `responseUsage` in the reply path with mocked turn usage and mocked active provider usage windows; current main only appends token/cache/cost/session text and never loads provider usage for the footer.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse `loadProviderUsageSummary` and `formatUsageWindowSummary` in the reply-footer composition path, not to add GitHub Copilot-specific fetching inside core reply code.
