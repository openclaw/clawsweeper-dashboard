## Summary

Keep open. Current main and the latest release still do not bridge active-provider quota windows into the per-response `/usage` footer; `/status` already formats those windows, and the GitHub Copilot plugin already exposes the requested Premium/Chat quota data.

## What This Changes

Request to include active provider quota windows, especially GitHub Copilot Premium/Chat percentages, in the per-response `/usage` footer.

## Best Possible Solution

Add a generic active-provider usage-window summary to the existing `/usage` footer path, reusing provider usage hooks and `formatUsageWindowSummary` while omitting quota text when auth is unavailable or the fetch fails.

## Reproduction Assessment

Yes, at source level. Enabling `/usage tokens` or `/usage full` reaches the reply footer code, and that path never loads provider usage windows while `/status` does.

## Solution Assessment

Yes. The narrow maintainable solution is a generic active-provider usage-window composition point in the reply footer, not GitHub Copilot-specific core formatting or a new config surface.
