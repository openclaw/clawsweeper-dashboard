## Summary

Keep open. Current main still formats the per-response `/usage` footer from token/cache/cost/session data only, while `/status` and the GitHub Copilot plugin already expose the Premium/Chat provider usage windows needed for this enhancement.

## What This Changes

Add GitHub Copilot Premium/Chat provider quota percentages to the per-response `/usage` footer when that data is available.

## Best Possible Solution

Append a compact generic provider-window summary to the existing `/usage tokens` and `/usage full` footer when the active provider exposes windows, while keeping provider-specific fetching inside plugins and updating docs, changelog, and regression coverage.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: enable `responseUsage` for a reply with nonzero usage and an active provider that has usage windows; current main formats only token/cache/cost/session text and never loads provider usage for that footer.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse `loadProviderUsageSummary` and `formatUsageWindowSummary` in the reply-footer composition path, not to add GitHub Copilot-specific fetching inside core reply code.
