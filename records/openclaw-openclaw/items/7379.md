## Summary

Keep open. Current main still does not bridge provider quota windows into the per-response `/usage` footer: the footer path formats token/cache/cost/session data only, while `/status` and the GitHub Copilot plugin already expose the requested Premium/Chat usage-window data.

## What This Changes

Add GitHub Copilot Premium/Chat quota-window percentages to the per-response `/usage` footer when available.

## Best Possible Solution

Add a generic active-provider usage-window summary to the existing response usage footer, keeping provider-specific fetching inside plugins and updating docs, changelog, and regression coverage.

## Reproduction Assessment

Not applicable as a feature request rather than broken documented behavior. Source inspection gives high-confidence evidence that current `/usage` footers omit provider quota windows while `/status` already displays them.

## Solution Assessment

Yes. The maintainable path is a generic active-provider usage-window bridge into the footer, not a Copilot-only special case in core; coordinate with https://github.com/openclaw/openclaw/pull/69618 because it changes nearby footer plumbing.
