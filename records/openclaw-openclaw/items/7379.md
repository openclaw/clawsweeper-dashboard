## Repair Work Prompt

Implement the `/usage` footer provider-quota enhancement for https://github.com/openclaw/openclaw/issues/7379. Current main builds the footer in `src/auto-reply/reply/agent-runner.ts` with `formatResponseUsageLine`, which only emits token/cache/cost/session data; `/status` already uses `loadProviderUsageSummary` plus `formatUsageWindowSummary`, and the GitHub Copilot plugin already exposes `Premium` and `Chat` windows. Add a generic, provider-agnostic append when `/usage tokens` or `/usage full` is enabled and the active provider has usage windows. Keep provider-specific fetching inside plugins, resolve the active usage provider through existing contracts, use a short timeout, and swallow quota-load errors so replies are never delayed or failed by provider usage fetching. Do not implement the broad `messages.responseFooter` feature or new config options from https://github.com/openclaw/openclaw/pull/69618. Add regression coverage for a Copilot Premium/Chat footer, preserve existing token/cache/cost/session behavior, update `docs/reference/token-use.md`, and add a user-facing changelog entry.
