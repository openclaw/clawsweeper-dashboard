## Summary

Keep open. Current main and latest release still separate provider quota-window display from the per-response `/usage` footer: `/status` can show active-provider Premium/Chat windows, but the reply footer only formats token/cache/cost/session data. The direct fix PR is closed unmerged, and the broader open footer PR is conflicting and does not own this specific quota-window request.

## What This Changes

Add active-provider usage quota windows, especially GitHub Copilot Premium/Chat remaining percentage, to enabled per-response `/usage` footers.

## Best Possible Solution

Append a compact active-provider `formatUsageWindowSummary` result through the generic provider-usage contract when `/usage tokens` or `/usage full` is enabled, with bounded loading, regression tests, and token-use docs updated.

## Reproduction Assessment

Yes, at source level: the current reply footer has no provider-usage input, while `/status` already loads and formats the same active-provider windows. That proves the requested behavior is still missing without needing a live Copilot credential.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse the existing generic provider-usage loader and formatter while leaving Copilot-specific fetching inside the plugin; a Copilot-only core branch or new config option would be the wrong shape.
