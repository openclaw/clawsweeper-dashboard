## Summary

Keep open. Current main and the latest release still keep provider quota-window display out of the per-response `/usage` footer: `/status` can load and format active-provider Premium/Chat windows, but the reply footer still formats only token/cache/cost/session data. The direct implementation PR at https://github.com/openclaw/openclaw/pull/43098 is closed unmerged, and the broader open footer PR at https://github.com/openclaw/openclaw/pull/69618 does not implement this quota-window request.

## What This Changes

Request to append active provider Premium/Chat quota-window percentages to the per-response `/usage` footer when `/usage tokens` or `/usage full` is enabled.

## Best Possible Solution

Extend the generic reply usage footer to append a bounded active-provider usage-window summary through the existing provider-usage contract, without adding provider-specific core logic or a new config option.

## Reproduction Assessment

Yes, at source level: current main's reply footer has no provider-usage input or `loadProviderUsageSummary` call, while `/status` already loads and formats the same active-provider windows. That proves the requested behavior is still missing without needing a live Copilot credential.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse the existing generic provider-usage loader and formatter while leaving Copilot-specific fetching inside the plugin; a Copilot-only core branch or new setting would be the wrong shape.
