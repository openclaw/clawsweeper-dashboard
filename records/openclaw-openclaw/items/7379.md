The issue requests showing GitHub Copilot Premium/Chat quota percentages in the per-response `/usage` footer.

## Best Possible Solution

Add a generic active-provider usage-window append to `/usage tokens` and `/usage full` reply footers using the existing provider usage loader/formatter, while keeping Copilot-specific fetching inside the plugin and covering the reply path with focused tests and docs.

## Solution Assessment

Yes. The maintainable fix is a generic reply-footer bridge to existing provider usage windows, not Copilot-specific policy in core; the closed direct PR demonstrated the shape but should be reimplemented against current main and coordinated with the broader footer PR.
