Keep open. ## What This Changes

The issue requests adding GitHub Copilot Premium/Chat quota percentages to the per-response `/usage` footer when provider usage windows are available.

## Best Possible Solution

Add a generic active-provider usage-window summary to `/usage tokens` and `/usage full` reply footers using the existing provider usage loader/formatter, while keeping Copilot-specific fetching inside the plugin.

## Reproduction Assessment

Yes, at source level. Current main formats `/usage` reply footers from local turn token/cache/cost/session data only, while `/status` loads provider usage windows and the Copilot plugin supplies Premium/Chat windows.

## Solution Assessment

Yes. The narrow maintainable solution is a generic reply-footer bridge to existing provider usage windows, not Copilot-specific fetching or policy in core.
