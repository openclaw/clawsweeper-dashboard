## Summary

Keep open. Current main still formats the per-response `/usage` footer from token/cache/cost/session data only, while `/status` and the GitHub Copilot plugin already expose the Premium/Chat provider usage windows needed for this enhancement.

## What This Changes

Request to append GitHub Copilot Premium/Chat quota percentages to the per-response `/usage tokens` and `/usage full` footer.

## Best Possible Solution

Add a generic provider-window summary to the reply footer using the existing provider-usage loader and formatter, keeping plugin-specific quota fetching inside plugins and updating docs, changelog, and regression coverage.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: enable `/usage tokens` or `/usage full` for a reply with nonzero usage and an active provider that exposes usage windows; current main appends only token/cache/cost/session text.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse the existing generic provider-usage loader and `formatUsageWindowSummary` in the reply-footer path, not to add GitHub Copilot-specific quota fetching to core reply code.
