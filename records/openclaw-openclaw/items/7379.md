## Summary

Current main still does not implement the requested provider quota summary in the per-response `/usage` footer. `/status` has the provider-usage window plumbing and GitHub Copilot exposes Premium/Chat windows, but the reply footer path still formats only local turn/session usage, so the issue remains valid and narrow.

## What This Changes

Add GitHub Copilot premium request percentage, preferably through the generic provider-usage window summary, to the per-response `/usage` footer.

## Best Possible Solution

Add provider-agnostic provider-window summary support to the existing response usage footer path, reusing `loadProviderUsageSummary` and `formatUsageWindowSummary` while keeping provider-specific fetching inside plugins.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: enable `/usage tokens` or `/usage full` with a provider that has usage windows, and current main formats only turn/session usage while `/status` separately loads provider windows.

## Solution Assessment

Yes. The narrow maintainable solution is a generic reply-footer bridge to existing provider usage windows, not Copilot-specific fetching or policy in core.
