## Summary

Keep open. Current main still builds the per-response `/usage` footer from turn token/cache/cost/session data only, while `/status` and the GitHub Copilot plugin already have the provider-usage Premium/Chat window plumbing the issue wants reused.

Add GitHub Copilot Premium/Chat quota percentages to the per-response `/usage` footer when provider usage windows are available.

## Best Possible Solution

Add a generic active-provider usage-window append to the reply footer using the existing provider usage loader and formatter, keep provider-specific fetching inside plugins, and cover the reply-runner path plus token-use docs and changelog.
