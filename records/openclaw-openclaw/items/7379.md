## What This Changes

Request to append GitHub Copilot Premium/Chat quota percentages to the per-response `/usage` footer when provider usage windows are available.

## Best Possible Solution

Add a generic active-provider usage-window append to `/usage tokens` and `/usage full` reply footers using the existing provider usage loader/formatter, keeping Copilot-specific fetching inside the plugin.

## Reproduction Assessment

Yes, at source level. Current main’s reply runner formats and appends only turn token/cache/cost/session data, while `/status` loads provider usage windows and the GitHub Copilot plugin supplies Premium/Chat windows.

## Solution Assessment

Yes. The maintainable path is a generic reply-footer bridge to existing provider usage windows, not Copilot-specific fetching or policy in core.
