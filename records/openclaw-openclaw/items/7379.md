## Summary

Keep open. Current main and the latest v2026.6.5 release still keep provider quota windows out of the per-response `/usage` footer, while `/status` and the GitHub Copilot plugin already expose the requested Premium/Chat window data through the generic provider-usage path; the direct fix PR is closed unmerged and the broader open footer PR is conflicting and not a canonical fix for this request.

## What This Changes

Request to add GitHub Copilot Premium/Chat provider usage percentages to the enabled `/usage` response footer when that quota data is available.

## Best Possible Solution

Add a generic, bounded active-provider usage-window summary to response usage footers through the provider-usage contract, with tests and token-use docs updated, no new config option, and no Copilot-specific fetch logic in core.

## Reproduction Assessment

Yes at source level: current main's reply footer path formats only token/cache/cost/session data, while `/status` loads active-provider usage windows that include Copilot Premium/Chat data.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the generic provider-usage loader and formatter with bounded loading while leaving Copilot-specific fetching inside its plugin; a provider-specific core fetch or new config knob would be the wrong layer.
