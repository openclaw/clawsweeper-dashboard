## Summary

Current main still does not include provider quota windows in per-response `/usage` footers: replies format turn tokens, cache, cost, and session data, while status surfaces separately load generic provider usage and the GitHub Copilot plugin supplies Premium and Chat windows. The direct implementation PR closed unmerged for contributor queue limits, and the broader conflicting footer PR does not specifically own this request.

## What This Changes

Add active-provider quota-window percentages, including GitHub Copilot Premium and Chat usage, to per-response `/usage` footers.

## Best Possible Solution

Extend the existing reply-footer composition point to append a compact `formatUsageWindowSummary` for the active provider, while retaining provider-specific authentication, fetching, and window labels inside provider plugins.

## Reproduction Assessment

Yes, at source level. Current main clearly formats only turn/session usage in the reply footer, while the status path and Copilot plugin expose the requested generic quota windows.

## Solution Assessment

Yes. Reusing the generic provider-usage loader and formatter at the existing reply-footer boundary is narrower and more maintainable than adding Copilot-specific core code or another configuration surface.
