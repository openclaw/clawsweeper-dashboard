## Summary

Keep open: current main still exposes `pty` only as a per-call exec parameter, while the requested `tools.exec.pty` default is absent from schema, public config types, runtime defaults, and docs. The exact related implementation attempt in #45976 was closed unmerged, so there is no shipped or merged fix to close against.

## What This Changes

Add a global or per-agent `tools.exec.pty` default so exec commands can use PTY mode without relying on each tool call to pass `pty: true`.

## Best Possible Solution

If maintainers accept the config surface, add an opt-in default-false `pty` exec config at global and per-agent scopes, preserve per-call override and sandbox disabling, then update docs, config metadata, and regression coverage.

## Reproduction Assessment

Not applicable as a feature request. Current-main source inspection confirms `pty` is available only as an exec call parameter and no `tools.exec.pty` config key is accepted or threaded.

## Solution Assessment

Unclear pending maintainer policy approval. The proposed narrow implementation is plausible, but adding a new command-execution default should be accepted deliberately before a repair PR proceeds.
