## Summary

Keep open. Current main still supports `pty` only as a per-call exec parameter; the config schema, runtime defaults, config resolution, and direct/gateway exec paths do not expose `tools.exec.pty`, and the related implementation PRs were closed unmerged.

## What This Changes

Request to add a global `tools.exec.pty` option so exec commands can default to PTY mode without each tool call specifying `pty: true`.

## Best Possible Solution

If maintainers accept this config surface, implement `tools.exec.pty` end to end while preserving sandbox disabling behavior and per-call override semantics.
