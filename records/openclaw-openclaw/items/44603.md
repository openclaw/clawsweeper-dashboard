## Summary

Keep open. Current main and v2026.5.7 still expose PTY only as a per-call exec parameter, while both prior attempts to add `tools.exec.pty` were closed unmerged, so the requested global config option remains a maintainer/product decision rather than implemented cleanup.

## What This Changes

The issue requests a global `tools.exec.pty` config default so long-running exec commands can stream output without the model passing `pty: true` on every tool call.

## Best Possible Solution

If maintainers approve the config surface, add `tools.exec.pty` consistently across schema, exported types, help/docs, direct exec, gateway approval execution, and focused regression coverage.

## Reproduction Assessment

Not applicable as a feature request. Source and v2026.5.7 inspection show the requested global config key is absent while per-call `pty` is already supported.

## Solution Assessment

Unclear until maintainers approve the new global config option. If accepted, the narrow maintainable path is to use a config fallback in both the main exec run and gateway approval path, plus schema/types/docs/tests.
