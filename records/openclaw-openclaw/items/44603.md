## Summary

Keep open. Current main still only supports per-call `pty`; the exec config schema/types/default resolution do not expose `tools.exec.pty`, and the related implementation PR was closed unmerged with a noted gateway-approval gap.

## What This Changes

The issue requests a global `tools.exec.pty` config default so exec commands can use PTY mode without each tool call explicitly passing `pty: true`.

## Best Possible Solution

If maintainers accept the new config surface, implement `tools.exec.pty` consistently through schema, types, config resolution, gateway approval, direct exec, docs, and focused PTY/default tests while preserving sandbox disabling behavior.
