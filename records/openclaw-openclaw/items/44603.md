## Summary

Keep open: current main and the latest release still only support PTY as a per-call exec parameter, while the requested `tools.exec.pty` default would add a new command-execution config surface that needs maintainer product/security review.

## What This Changes

The issue requests a global `tools.exec.pty` configuration option so exec commands can default to PTY-backed streaming output without the model setting `pty: true` on each call.

## Best Possible Solution

Keep this issue open for a maintainer decision; if approved, add a default-false `tools.exec.pty` setting across schema, types, exec default resolution, direct and gateway-approved execution, docs, and focused tests.

## Reproduction Assessment

Yes, at source level. Current main clearly reads PTY from the per-call exec argument only, while config schema/default plumbing has no `tools.exec.pty` path.

## Solution Assessment

Unclear until maintainers decide the product/security direction. The narrow fallback is plausible, but the best fix must cover schema/types/docs plus both direct and gateway approval execution paths.
