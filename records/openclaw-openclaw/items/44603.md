## Summary

Keep open: current `main` still only honors PTY from the per-call exec parameter, while the requested `tools.exec.pty` config default is absent from the config schema/types/docs and the exact same fix PR was closed unmerged.

## What This Changes

The issue requests a global `tools.exec.pty` setting so exec tool calls can default to PTY mode and stream long-running command output without relying on each model call to pass `pty: true`.

## Best Possible Solution

If maintainers approve the config surface, add `tools.exec.pty` to config types/schema/help/docs, preserve per-call override precedence, thread the default through direct and gateway approval exec paths, and align sandbox-disable warning behavior with https://github.com/openclaw/openclaw/issues/38601.

## Reproduction Assessment

Yes. From source inspection, current `main` does not accept `tools.exec.pty` in config and computes PTY from the per-call parameter only; I did not run a live command because this review is read-only.
