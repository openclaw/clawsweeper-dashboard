## Summary

Keep open: current main and the latest release still only honor PTY from the per-call exec parameter, while the requested global `tools.exec.pty` config default is absent from config schema, types, help labels, and docs. The exact implementation PR and an older same-shape PR are both closed unmerged, so no merged or open canonical fix owns this yet.

## What This Changes

The issue requests a global `tools.exec.pty` config default so long-running exec commands can stream output without the model passing `pty: true` on every tool call.

## Best Possible Solution

If maintainers approve the config surface, add a canonical `tools.exec.pty` field across schema/types/help/docs, thread it through direct and gateway approval exec paths while preserving per-call precedence, and coordinate sandbox warning behavior with the related open issue.
