## Summary

Keep open: current `main` and the latest release still only accept `pty` as a per-call exec argument, not as a `tools.exec` config default. The same-repo implementation attempt is closed unmerged, and the remaining work needs maintainer product/security approval because it adds a command-execution default surface.

## What This Changes

The issue requests a `tools.exec.pty` global config default so long-running exec commands can stream output without relying on each model tool call to pass `pty: true`.

## Best Possible Solution

If maintainers accept the config surface, add a documented opt-in `tools.exec.pty` and matching per-agent default that preserves per-call override, keeps sandbox behavior explicit, and threads the value through both gateway approval and direct execution paths with focused tests.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers choose the product/security direction.
