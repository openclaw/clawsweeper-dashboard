## Summary

Keep open: current main and the latest release still only honor PTY as a per-call exec parameter, while the requested `tools.exec.pty` config default remains absent and the linked implementation PR is closed unmerged.

## What This Changes

Add `tools.exec.pty` to `openclaw.json` so PTY mode can be enabled by default for exec commands instead of relying on each tool call to pass `pty: true`.

## Best Possible Solution

Keep this open for maintainer product/security review; if accepted, add `tools.exec.pty` as an explicit default-false operator setting wired through schema, types, default resolution, direct exec, gateway approvals, docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection shows `pty` is currently only read from the per-call exec argument, while config schema/default plumbing includes `timeoutSec` but no `tools.exec.pty` key.

## Solution Assessment

Unclear until maintainers make the product/security decision. The narrow implementation is plausible, but the best fix must cover schema/types/docs plus both direct and gateway approval execution paths.
