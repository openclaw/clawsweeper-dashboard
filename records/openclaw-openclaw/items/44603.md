## Summary

Keep open. Current main still has only per-call exec `pty`, the exact implementation PR is closed unmerged, and making PTY a global `tools.exec` default is a new command-execution config surface that needs maintainer product and security review before implementation.

## What This Changes

The issue requests a global `tools.exec.pty` configuration default so long-running exec commands can use PTY streaming without relying on every tool call to pass `pty: true`.

## Best Possible Solution

If maintainers approve the config surface, implement `tools.exec.pty` end to end across schema/types, config resolution, exec runtime, docs, and regression tests while preserving explicit per-call overrides and sandbox PTY suppression.
