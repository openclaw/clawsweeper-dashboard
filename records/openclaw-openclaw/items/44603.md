## Summary

Keep open: current main still only supports `pty` as a per-call exec parameter, while the requested `tools.exec.pty` config default is absent from the config schema, public types, and runtime defaults. This is a new exec configuration/product decision rather than an already-fixed bug.

## What This Changes

Request adding `tools.exec.pty` as a global/default exec configuration option so long-running commands can use PTY streaming without relying on each model tool call to pass `pty: true`.

## Best Possible Solution

Make an explicit maintainer product decision on `tools.exec.pty`; if accepted, implement it consistently in config types/schema, exec defaults, gateway approval execution, direct execution, docs, and focused regression coverage while preserving sandbox behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection does confirm current main has only per-call `pty` support and no `tools.exec.pty` config/default path.

## Solution Assessment
