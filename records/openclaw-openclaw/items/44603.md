## Summary

Keep open. Current main still supports `pty` only as a per-call exec parameter, with no `tools.exec.pty` config/default path; adding one is a new global command-execution default that needs maintainer product/security judgment, and the prior implementation PR is closed unmerged.

## What This Changes

Request adding `tools.exec.pty` as a global/per-agent exec default so long-running commands can use PTY streaming without every tool call specifying `pty: true`.

## Best Possible Solution

If maintainers approve the config surface, implement `tools.exec.pty` end to end while preserving sandbox PTY suppression and explicit per-call override semantics.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main accepts per-call `pty` but has no `tools.exec.pty` config/default path.

## Solution Assessment
