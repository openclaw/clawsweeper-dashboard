## Summary

Keep open. Current main still exposes PTY only as a per-call exec parameter, the exact implementation PRs were closed unmerged, and adding a global `tools.exec.pty` default is a new command-execution config/policy surface that needs maintainer product and security review.

## What This Changes

Request to add `tools.exec.pty` as a global exec config default so commands can run in PTY mode without the model passing `pty: true` on every call.

## Best Possible Solution

If maintainers approve the config surface, implement `tools.exec.pty` end to end across schema/types, config resolution, exec runtime, docs, and regression tests while preserving explicit per-call overrides and sandbox PTY suppression.

## Reproduction Assessment

## Solution Assessment
