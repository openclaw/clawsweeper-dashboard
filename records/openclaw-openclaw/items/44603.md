## Summary

Keep open. Current main still exposes `pty` only as a per-call exec parameter, while the runtime defaults, config schema, generated config docs, and exec defaults plumbing do not accept or apply a global `tools.exec.pty` setting; the linked implementation PR was closed unmerged rather than superseded.

## What This Changes

The issue asks for a global `tools.exec.pty` configuration default so long-running exec commands can stream intermediate output without the model setting `pty: true` on every call.

## Best Possible Solution

Maintainers should decide whether `tools.exec.pty` is an accepted config contract; if it is, implement the default through schema, types, `resolveExecToolDefaults`, gateway approval, foreground execution, docs, and focused tests while preserving per-call override and sandbox behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source and docs inspection do give high-confidence evidence that current main lacks a global `tools.exec.pty` default.

## Solution Assessment

Unclear. The requested shape is plausible and prior art exists, but adding a global exec PTY default should be a maintainer-approved config decision and the previous PR missed at least the gateway approval path.
