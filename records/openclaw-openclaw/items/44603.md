## Summary

Keep open. Current main and v2026.5.18 still expose `pty` only as a per-call exec parameter; `tools.exec.pty` is absent from the strict config schema, config types, default resolution, and docs. The linked implementation PR is closed unmerged, so this remains a maintainer product/security decision rather than cleanup-closeable work.

## What This Changes

Allow `tools.exec.pty: true` in OpenClaw config to make PTY mode the default for exec calls unless a tool call overrides it.

## Best Possible Solution

If maintainers approve the policy, add an optional `tools.exec.pty` boolean across schema/types/help/docs, thread global and per-agent defaults into direct and gateway exec paths, preserve per-call override and sandbox no-PTY behavior, and cover the config/default paths with focused tests.

## Reproduction Assessment

Not applicable as a feature/config request. Source inspection gives high confidence that per-call PTY exists, but no global `tools.exec.pty` config/default path exists on current main or v2026.5.18.

## Solution Assessment

Unclear until maintainer product/security review. The maintainable solution is the full config contract plus direct and approval-backed runtime wiring, not only a single fallback expression.
