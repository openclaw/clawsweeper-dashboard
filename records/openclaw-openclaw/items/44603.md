## Summary

Keep open. Current main still supports exec PTY only as a per-call parameter, both exact implementation PRs were closed unmerged, and adding a global PTY default is a new command-execution config policy that needs maintainer product and security review.

## What This Changes

Add a global `tools.exec.pty` default so exec commands can use PTY streaming without requiring each tool call to pass `pty: true`.

## Best Possible Solution

If maintainers approve the config surface, implement `tools.exec.pty` end to end across schema/types, config resolution, gateway/direct exec runtime, docs, and regression tests while preserving explicit per-call overrides and sandbox PTY suppression.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence that per-call `pty` exists, but no `tools.exec.pty` config/default path exists.

## Solution Assessment

Unclear until maintainer product review. The requested direction is plausible, but the maintainable fix must cover the full config contract plus gateway and direct exec paths rather than only adding one fallback expression.
