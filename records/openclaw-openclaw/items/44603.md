## Summary

Keep open: current main still only supports PTY as a per-call exec parameter, while the requested global `tools.exec.pty` default is absent from schema, runtime default threading, docs, and public config types. The related #45976 implementation attempt was closed unmerged, so the requested feature has not shipped.

## What This Changes

The issue requests a global `tools.exec.pty` config default so long-running exec commands can stream intermediate output without requiring each tool call to pass `pty: true`.

## Best Possible Solution

If maintainers accept the config surface, add an opt-in default-false `pty` exec config at the global and per-agent levels, thread it through exec defaults, preserve per-call override and sandbox disabling, and update docs plus regression coverage.

## Reproduction Assessment

Not applicable as a feature request. Current-main source inspection confirms the requested config key is absent and PTY is still controlled only by per-call `params.pty`.

## Solution Assessment

Unclear pending maintainer policy approval. The proposed narrow implementation is plausible, but adding a new `tools.exec` command-execution knob should be accepted deliberately before a repair PR proceeds.
