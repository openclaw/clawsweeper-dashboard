## Summary

Keep open: current main and v2026.6.10 still only honor PTY from the per-call exec parameter, while the requested global `tools.exec.pty` default is absent from config schema, config types, config help, config docs, and exec-default wiring. The two exact implementation PRs are closed unmerged, so no merged or open canonical fix owns the remaining work.

## What This Changes

Add a global `tools.exec.pty` config default so exec commands can use PTY without the model passing `pty: true` on each call.

## Best Possible Solution

If maintainers approve the config surface, add canonical `tools.exec.pty` schema/types/help/docs support, thread it through direct and gateway exec defaults with per-call precedence, and coordinate the sandbox warning behavior with the related issue.

## Reproduction Assessment

Yes. Source inspection shows current main accepts per-call `pty` but has no `tools.exec.pty` schema/type/help/doc path, and direct plus gateway exec paths only read `params.pty`.
