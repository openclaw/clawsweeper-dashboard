## Summary

Keep open: current main and v2026.6.6 still only honor PTY as a per-call exec parameter, while the requested global `tools.exec.pty` default is a new command-execution config surface that needs maintainer product and security review.

## What This Changes

Request to add a `tools.exec.pty` configuration default so exec calls can use PTY mode without relying on each model tool call to pass `pty: true`.

## Best Possible Solution

Keep this issue open for a maintainer product/security decision; if approved, add a default-false `tools.exec.pty` setting across schema, types, default resolution, direct and gateway-approved execution, docs, and focused tests.

## Reproduction Assessment

Yes, at source level. Current main reads PTY from the per-call exec argument only, while config schema/default plumbing and docs have no `tools.exec.pty` path.

## Solution Assessment

Unclear until maintainers decide the product/security direction. The narrow fallback is plausible, but the maintainable fix must cover schema/types/docs plus both direct and gateway approval execution paths.
