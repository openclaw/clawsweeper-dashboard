## Summary

Keep open: current main and v2026.6.6 still only support `pty` as a per-call exec parameter, not as a `tools.exec` config default. The request is real, but it needs maintainer product/security review because it adds a global command-execution default and the only related PR was closed unmerged.

## What This Changes

Request adding a global `tools.exec.pty` default in `openclaw.json` so exec commands can use PTY mode without each model call setting `pty: true`.

## Best Possible Solution

Maintainers should decide the supported knob and semantics first; if accepted, implement it across schema/types, config resolution, gateway approval and local exec paths, docs, and focused fresh-install plus upgrade/override tests.

## Reproduction Assessment

Yes, at source level. Current main exposes per-call `pty` but has no `tools.exec.pty` config schema/type/default forwarding, and the runtime still derives PTY only from `params.pty`.

## Solution Assessment

Unclear as proposed. The issue identifies the right gap, but the one-line fallback is not enough because schema, config resolution, docs, tests, gateway approval, sandbox forcing, and node-host semantics need a deliberate treatment.
