## Summary

Keep open. Current `main` and `v2026.5.12` still expose PTY only as a per-call exec parameter, and the linked implementation PR is closed unmerged; adding a global PTY default is a new command-execution config surface with product and security implications.

## What This Changes

The issue requests a global `tools.exec.pty` configuration option so exec commands can default to PTY streaming without the model passing `pty: true` on each call.

## Best Possible Solution

If maintainers approve the config surface, implement `tools.exec.pty` end to end across config types/schema/help/docs, default resolution, gateway and local exec runtime paths, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that per-call `pty` exists, but current `tools.exec` config schema/default resolution has no `pty` key.

## Solution Assessment

Unclear until maintainer product and security review. The requested direction is plausible, but the maintainable fix must cover the full config contract and both gateway approval and direct exec paths.
