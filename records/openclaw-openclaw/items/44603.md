## Summary

Keep open: current `main` and `v2026.5.22` still expose PTY only as a per-call exec parameter, while the requested `tools.exec.pty` config default is absent from schema, config types, config resolution, docs, and runtime wiring. This remains a concrete feature/config request, but it needs maintainer product/security review before implementation because it changes the default shell-execution mode when the model omits `pty`.

## What This Changes

The issue requests an optional global `tools.exec.pty` setting so exec commands can default to PTY streaming output when the model omits the per-call `pty` parameter.

## Best Possible Solution

If maintainers approve the policy, add an optional `tools.exec.pty` config contract through schema, types/help, config resolution, direct and gateway runtime wiring, docs, and focused tests while preserving per-call override and sandbox no-PTY behavior.

## Reproduction Assessment

Not applicable as a feature/config request. Source and docs inspection on current main give high confidence that OpenClaw currently exposes PTY only as a per-call `exec` parameter, not a global `tools.exec.pty` default.
