## Summary

Keep open: current main still has the Telegram/Feishu `reasoningDefault` fallback gap, and this PR addresses real code paths, but the branch now also introduces a broader `verbose` reasoning mode with at least one cross-surface correctness issue before merge.

## What This Changes

The PR wires Telegram and Feishu reasoning-preview resolution through a new per-agent/global `reasoningDefault` helper, adds tests/docs/changelog, expands `ReasoningLevel` with `verbose`, and adds Telegram stream-preview retry handling.

## Best Possible Solution

Land a narrowed, rebased fallback fix that shares normalized per-agent/global reasoning-default resolution for Telegram and Feishu, while gating or splitting the new `verbose` mode until each advertised surface has correct behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows Telegram and Feishu ignore configured reasoning defaults in their channel preview-resolution paths while the config docs and core directive resolver define per-agent/global `reasoningDefault` precedence.

## Solution Assessment

No. The shared fallback resolver is the right direction for the reported gap, but making `verbose` a global reasoning level without a real stream/rendering fallback on every advertised surface is not the narrowest maintainable fix.
