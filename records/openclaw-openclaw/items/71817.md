Keep open. Current main still ignores configured reasoning defaults in the Telegram and Feishu channel preview paths, while the latest PR addresses the gap without a concrete blocking finding; it should proceed through maintainer/channel-owner review and exact-head validation.

## What This Changes

The PR adds a shared Plugin SDK helper for resolving per-agent then global `reasoningDefault`, uses it in Telegram and Feishu reasoning fallback paths, updates Feishu call sites and channel tests, registers/documents the SDK subpath, updates generated API hash metadata and the changelog, and includes small type/test cleanup from the branch.

## Best Possible Solution

Land the PR after maintainer/channel-owner review and exact-head validation if the new helper is accepted, preserving explicit session overrides, normalized per-agent then global fallback resolution, Feishu's stream-only preview behavior, and aligned tests/docs/changelog/API metadata.

## Reproduction Assessment

Yes. On current main, configure `reasoningDefault: "stream"` per agent or globally with no stored session reasoning level; Telegram resolves `off` and Feishu returns `false` instead of using the configured default.

## Solution Assessment

Yes. The latest PR is a narrow maintainable fix because it keeps session state first, centralizes normalized config fallback resolution, includes the global default, and avoids changing Feishu block-mode semantics.
