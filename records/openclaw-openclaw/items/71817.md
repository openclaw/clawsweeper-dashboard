The PR adds a shared Plugin SDK reasoning-default helper, wires Telegram and Feishu reasoning preview resolution to fall back from session state to per-agent/global `reasoningDefault`, and updates tests, docs, changelog, and SDK export metadata.

## Best Possible Solution

Merge a focused version that keeps session overrides ahead of normalized per-agent/global defaults, preserves Feishu’s stream-only preview semantics, and trims the remaining unnecessary runtime imports.

## Reproduction Assessment

Yes. Current main is statically reproducible by configuring `reasoningDefault: "stream"` with no stored session reasoning level: Telegram resolves `off`, and Feishu preview stays false.

## Solution Assessment

Yes, with non-blocking cleanup. The fallback behavior and fail-closed direction are maintainable; the remaining improvement is to keep the new SDK helper and Telegram test mock on the lightweight import path.
