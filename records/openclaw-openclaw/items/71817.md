## Summary

Keep open. The PR addresses a real current-main Telegram/Feishu reasoning-default gap, but the latest branch still introduces narrow SDK/runtime boundary problems and the exact head has failing plugin dependency/loader contract checks.

## What This Changes

The PR adds a public `plugin-sdk/agent-config-helpers` helper, wires Telegram and Feishu reasoning fallback to configured per-agent/global `reasoningDefault`, updates SDK docs/baseline/changelog and focused tests, plus small type/test cleanups.

## Best Possible Solution

Keep the Telegram/Feishu fallback behavior, but make the shared helper a narrow, cheap SDK/config seam and remove stale broad runtime re-exports before landing with the focused channel and Plugin SDK contract checks green.

## Reproduction Assessment

Yes. Source inspection on current main is enough: with a configured reasoning default and no stored session `reasoningLevel`, Telegram falls through to `off` and Feishu returns false unless the session store says `stream`; the PR's added tests target this same path.

## Solution Assessment

No, not fully. The fallback precedence is the right product direction, but the current implementation widens SDK/runtime imports and is failing plugin dependency/loader contracts, so the branch needs a small boundary cleanup first.
