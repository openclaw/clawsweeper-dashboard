## What This Changes

The PR adds a Plugin SDK reasoning-default helper, wires Telegram and Feishu reasoning preview resolution to fall back to per-agent/global `reasoningDefault`, and updates tests, docs, changelog, and SDK export metadata.

## Best Possible Solution

Merge a focused version that keeps session overrides ahead of normalized per-agent/global defaults, preserves Feishu’s stream-only preview behavior, and trims the remaining unnecessary runtime imports.

## Reproduction Assessment

Yes. On current main, configure `reasoningDefault: "stream"` with no stored session reasoning level: Telegram resolves `off`, and Feishu preview stays false because both helpers only consult session state or hardcoded defaults.

## Solution Assessment

Yes, with cleanup. The shared fallback helper is the right shape, but it should import the lightweight resolver and remove the unused Telegram runtime re-export/test partial mock.
