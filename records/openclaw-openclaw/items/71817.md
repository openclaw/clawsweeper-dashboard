## Summary

Keep open. Current main still has the reported Telegram/Feishu reasoning-default gap, and the PR now covers normalized per-agent plus global fallback resolution, but the patch introduces fail-open behavior when the session store cannot be read, so explicit session overrides do not reliably keep priority.

## What This Changes

The PR adds a Plugin SDK helper for effective `reasoningDefault` resolution, wires Telegram and Feishu reasoning-preview paths to use per-agent/global config defaults, and updates tests, SDK metadata, docs, and changelog.

## Best Possible Solution

Update the branch so config defaults apply only when the session store was read and no valid session override exists; store read failures should preserve the existing fail-closed behavior while keeping normalized per-agent/global fallback resolution.

## Reproduction Assessment

Yes. Current main can be verified by a Telegram or Feishu session with `reasoningDefault: "stream"` and no stored reasoning level; the PR regression can be reproduced by mocking `loadSessionStore` to throw while config default is `stream`.

## Solution Assessment

No, not yet. The shared helper and precedence direction are maintainable, but falling back to config after session-store errors is not the safest implementation of the stated session-override priority.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Fail closed when Feishu session state cannot be read:** `extensions/feishu/src/reasoning-preview.ts:26-30`
  - body: With `reasoningDefault: "stream"` in config, this catch now falls through and returns `true` after `loadSessionStore` throws. That changes the existing fail-closed preview behavior and can expose reasoning previews even though an existing session may have `reasoningLevel: "off"`, so session-store overrides no longer reliably take priority.
  - confidence: 0.86
- **[P2] Do not ignore Telegram session overrides after store errors:** `extensions/telegram/src/bot-message-dispatch.ts:221`
  - body: After a session-store read or parse error, returning `configDefault` means a session that previously set `/reasoning off` can be treated as `on` or `stream`. The PR promises stored session toggles keep priority, so this load-failure path should stay fail-closed or otherwise distinguish an absent level from an unreadable store.
  - confidence: 0.78
