## Review Findings

Full review comments:

- **[P2] Serialize verbose action updates through the draft queue:** `extensions/telegram/src/bot-message-dispatch.ts:507`
  - body: `onReasoningStream` and agent-event callbacks are fire-and-forget, and normal reasoning updates go through `enqueueDraftLaneEvent` to serialize draft mutations. This helper updates `reasoningLane.lastPartialText` and the stream directly, so a queued reasoning update can overwrite or reorder a verbose tool-action line; append these lines through the same draft-lane queue.
  - confidence: 0.84
- **[P3] Key active verbose actions by tool identity:** `extensions/telegram/src/bot-message-dispatch.ts:1127-1128`
  - body: `payload.phase` is `start`/`update`, so every started tool is stored under the same key instead of a tool identity. The end path looks up `payload.itemId ?? payload.name`, which means durations are usually missing and entries stay in the map until the run ends.
  - confidence: 0.86
