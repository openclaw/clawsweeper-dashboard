## Summary

Keep open: current main still has a real documented `reasoningDefault` gap in Telegram and Feishu, and the PR is useful, but the branch now makes the new `verbose` mode emit both streamed and block reasoning before merge.

## What This Changes

The PR changes Telegram and Feishu reasoning-preview resolution to use configured per-agent/global reasoning defaults, adds a plugin SDK helper/subpath with tests/docs/changelog, and introduces a `verbose` reasoning level across config/session/runtime surfaces.

## Best Possible Solution

Merge only after `verbose` is either split out or made a streaming-only reasoning mode with tool summaries, with command/docs/tests updated and the original Telegram/Feishu fallback behavior preserved.

## Reproduction Assessment

Yes. Source inspection on current main shows Telegram and Feishu ignore configured reasoning defaults, and source inspection of the PR head shows `verbose` enables both block and stream reasoning paths.

## Solution Assessment

No. The shared fallback resolver is the right direction, but the current branch should not make `verbose` produce both streamed previews and separate block-mode reasoning, and the newly accepted mode needs matching user-facing command/docs coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep verbose from emitting block reasoning:** `src/agents/pi-embedded-subscribe.ts:134-137`
  - body: The branch makes `verbose` set both `includeReasoning` and `streamReasoning`. The message-end handler uses `includeReasoning` to emit a separate `Reasoning:` block via `onBlockReply`, while `streamReasoning` already drives the live reasoning preview, so Telegram/Feishu verbose sessions can stream reasoning and then enqueue block-mode reasoning too. Please keep block reasoning limited to `on` or explicitly suppress the block emission for `verbose`.
  - confidence: 0.9
- **[P3] Advertise verbose wherever reasoning is surfaced:** `src/auto-reply/thinking.shared.ts:216-217`
  - body: After adding `verbose` to `normalizeReasoningLevel`, `/reasoning verbose` and `sessions.patch` become valid, but the command registry, no-arg/error text, help output, and public docs still list only `on`, `off`, and `stream`. Users cannot discover or select the new mode from native command menus, and status/error guidance is stale; update those surfaces and coverage if `verbose` remains in this PR.
  - confidence: 0.84
