## Summary

Keep open. Current main still has the reported Telegram/Feishu reasoning-default gap, and this PR head addresses the behavior, but two low-risk import-topology/test-mock cleanups remain before normal maintainer review.

## What This Changes

The branch wires Telegram and Feishu reasoning preview resolution to fall back from session state to normalized per-agent/global `reasoningDefault`, adds a public SDK helper/subpath, updates tests/docs/API metadata, and adds a changelog entry.

## Best Possible Solution

Merge a focused version that keeps session overrides ahead of normalized per-agent/global defaults, preserves Feishu’s stream-only preview semantics, and trims the remaining unnecessary runtime imports.

## Reproduction Assessment

Yes. Current main is statically reproducible by configuring `reasoningDefault: "stream"` with no stored session reasoning level: Telegram resolves `off`, and Feishu preview stays false.

## Solution Assessment

Yes, with cleanup. The fallback chain is the narrow maintainable fix; the safer final form imports the resolver from `agent-scope-config` and removes the unused Telegram runtime re-export.
