## Summary

Keep open. Current main still has the Telegram/Feishu config-default gap, and the PR now addresses normalized/global defaults, but the branch introduces an authorization regression in channel preview resolution and is currently conflict-dirty with a broader feature scope than the title describes.

## What This Changes

The PR updates Telegram and Feishu reasoning-preview resolution to use per-agent/global `reasoningDefault`, adds a plugin SDK helper with tests/docs/changelog, and also adds a new `verbose` reasoning mode plus Telegram draft-stream retry handling.

## Best Possible Solution

Update the branch to preserve the existing configured-default authorization gate in Telegram and Feishu, keep Feishu stream-only behavior, add focused unauthorized/default regression tests, and split or explicitly approve the new `verbose` and retry work before merge.

## Reproduction Assessment

Yes from source inspection. Current main ignores config defaults in the Telegram and Feishu channel helpers, and the PR’s new fallback path can be traced to an authorization mismatch without needing a live channel run.

## Solution Assessment

No. The shared normalized/default helper is the right direction, but the channel callers need the same authorization gate as the core resolver, and the broad `verbose`/retry additions should be split or explicitly accepted.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate Telegram config defaults by sender authorization:** `extensions/telegram/src/bot-message-dispatch.ts:203-206`
  - body: Current docs and `get-reply-directives` only apply configured reasoning defaults for owners, authorized senders, or operator-admin contexts. This new Telegram fallback resolves `agents.*.reasoningDefault` without any sender auth state, so a non-authorized turn with no stored session level can resolve to `on`/`stream` here even though the core resolver would force configured defaults to `off`. Please pass the same authorization decision into this channel resolver, or fall back to `off` unless the sender is allowed to use configured reasoning state.
  - confidence: 0.86
- **[P2] Gate Feishu config defaults by sender authorization:** `extensions/feishu/src/reasoning-preview.ts:11-16`
  - body: The Feishu helper now enables previews from config whenever the session store has no explicit level, but it receives only `cfg` and `agentId`, not the owner/authorized/operator decision required by the config contract. That can expose a stream/verbose preview in contexts where configured reasoning defaults should be ignored. Thread the Feishu sender authorization state into this helper and keep unauthorized config fallback false.
  - confidence: 0.84
- **[P3] Document the global reasoning fallback in thinking docs:** `docs/tools/thinking.md:105`
  - body: This page now advertises the expanded reasoning modes, but its resolution order still says per-agent default then `off`, omitting `agents.defaults.reasoningDefault`. The PR’s helper and config docs use per-agent then global default, so this page should match the actual fallback chain.
  - confidence: 0.78
