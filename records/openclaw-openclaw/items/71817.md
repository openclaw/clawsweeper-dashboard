## Summary

Keep open: current main still has a real Telegram/Feishu reasoning-default gap, but this PR is not mergeable as-is because the new `verbose` reasoning mode is advertised as including tool actions while the changed stream path does not enable those summaries unless `/verbose` is also on.

## What This Changes

The PR makes Telegram and Feishu channel reasoning previews fall back to configured `reasoningDefault`, adds a shared plugin SDK helper plus docs/tests/changelog, and expands reasoning visibility with a new `verbose` mode and Telegram stream-preview retry handling.

## Best Possible Solution

Land a narrowed fallback fix, or make `verbose` fully consistent by routing tool action summaries through the reasoning preview wherever the mode is accepted and documented.

## Reproduction Assessment

Yes. Source inspection on current main shows Telegram and Feishu ignore configured reasoning defaults in their channel preview-resolution paths while the config docs define per-agent/global `reasoningDefault` precedence.

## Solution Assessment

No. The shared fallback resolver is the right direction, but the added `verbose` mode is not complete because it is documented as including tool actions without wiring those summaries to the reasoning stream.

## Review Findings

- **[P2] Wire verbose reasoning to tool-action summaries:** `src/agents/pi-embedded-subscribe.ts:144`
