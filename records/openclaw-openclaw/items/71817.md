## Summary

Keep open: current main still has the Telegram/Feishu `reasoningDefault` fallback gap, and the PR’s normalized fallback direction looks useful, but the latest branch also exposes a new `verbose` reasoning mode without wiring the shared directive/session/runtime contract that still only accepts `on|off|stream`.

## What This Changes

The PR wires Telegram and Feishu reasoning-preview fallback through a new plugin SDK helper for per-agent/global `reasoningDefault`, updates tests/docs/exports/changelog, and adds a new Telegram `verbose` reasoning mode/config value.

## Best Possible Solution

Keep the normalized per-agent/global `reasoningDefault` fallback, but either split out `verbose` or wire it through the shared `ReasoningLevel`, directive, session patch, UI/docs, runtime subscriber, and tests before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows Telegram and Feishu preview resolution ignore configured `reasoningDefault` fallbacks, and the PR-introduced `verbose` mode gap is also source-reproducible through the unchanged parser/session/runtime paths.

## Solution Assessment

No, not yet. The shared fallback helper is the right direction, but exposing `verbose` as a config/session mode is incomplete unless the shared reasoning contract is updated or the feature is removed from this PR.
