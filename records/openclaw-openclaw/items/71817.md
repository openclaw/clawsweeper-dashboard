## Summary

Keep open. Current main still misses the Telegram/Feishu channel-side `reasoningDefault` fallback, and the PR now addresses the main behavior with only narrow P3 import-topology cleanup remaining.

## What This Changes

The PR adds a Plugin SDK reasoning-default helper, wires Telegram and Feishu reasoning preview resolution to fall back from session state to per-agent/global `reasoningDefault`, and updates tests, docs, changelog, and SDK export metadata.

## Best Possible Solution

Land the behavior after trimming the import cleanup, preserving session-store overrides, fail-closed store-error behavior, normalized/global config fallback, and Feishu’s stream-only preview boundary.

## Reproduction Assessment

Yes. On current main, a Telegram or Feishu session with no stored `reasoningLevel` and config `reasoningDefault: "stream"` resolves to `off`/false because the channel helpers never read `cfg`.

## Solution Assessment

Yes, with minor cleanup. The proposed fallback chain is the narrow maintainable fix; importing the lighter resolver and removing the unused Telegram re-export would better match the repo’s SDK boundary expectations.
