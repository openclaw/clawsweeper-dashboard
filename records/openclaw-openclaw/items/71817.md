## Summary

Keep open. The underlying main-branch gap is real, and the branch now handles normalized per-agent plus global defaults, but it should not merge until the channel preview fallback preserves the same sender-trust gate that core reply resolution already applies to configured reasoning defaults.

## What This Changes

The PR adds a Plugin SDK helper for resolving configured reasoning defaults, wires Telegram and Feishu reasoning previews to use per-agent/global `reasoningDefault` after session state, and updates tests, changelog, SDK exports, docs, and generated baselines.

## Best Possible Solution

Merge a focused version that keeps session overrides ahead of normalized per-agent/global defaults, preserves Feishu's stream-only preview semantics, and applies configured defaults only for trusted reasoning contexts.

## Reproduction Assessment

Yes. Current main is statically reproducible by configuring `reasoningDefault: "stream"` with no stored session reasoning level: Telegram resolves `off`, and Feishu preview stays false; the branch regression is reproducible by applying that default for an untrusted sender, where core tests say reasoning must remain `off`.

## Solution Assessment

No. The shared normalized fallback is the right direction, but the PR needs the channel preview path to share the core owner/authorized/operator trust gate before it is the narrow maintainable fix.
