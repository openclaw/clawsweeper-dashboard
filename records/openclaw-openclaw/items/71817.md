## Summary

Keep open. The PR addresses a real current-main gap and the latest head incorporates normalized per-agent plus global `reasoningDefault` fallback, but it still changes unreadable session-store paths from fail-closed to config-driven visibility, so stored `off` overrides are not reliably preserved on error.

## What This Changes

The PR adds a Plugin SDK `resolveConfigReasoningDefault` helper, wires Telegram and Feishu reasoning preview resolution to use per-agent/global config defaults, and updates tests, SDK docs, package exports, generated baseline hashes, and the changelog.

## Best Possible Solution

Keep the normalized config-default fallback for absent session levels, but preserve fail-closed behavior when the session store cannot be read and add focused throw-path tests for Telegram and Feishu before merge.

## Reproduction Assessment

Yes. Current main can be checked by configuring `reasoningDefault: "stream"` with no stored session level; the PR regression can be reproduced by mocking `loadSessionStore` to throw while config default is `stream`.

## Solution Assessment

No, not yet. The shared resolver and precedence direction are maintainable, but falling back to config after store errors is not the safest implementation of the promised session-override priority.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Fail closed when Feishu session state cannot be read:** `extensions/feishu/src/reasoning-preview.ts:26-30`
  - body: With `reasoningDefault: "stream"` in config, this catch now falls through and enables previews after `loadSessionStore` or session resolution throws. That changes the existing fail-closed edge case from the Feishu stream-only gate, so an unreadable store can expose reasoning previews despite a persisted `off` override.
  - confidence: 0.88
- **[P2] Preserve Telegram session overrides on store errors:** `extensions/telegram/src/bot-message-dispatch.ts:217-220`
  - body: After a session-store read or parse error, returning `configDefault` can treat a session that previously stored `/reasoning off` as `on` or `stream`. The branch promises stored session toggles keep priority, so this unreadable-store path should stay fail-closed or otherwise distinguish missing state from failed state.
  - confidence: 0.82
