## Summary

Keep open: the branch addresses a real source-reproducible fallback gap, but the latest head is not merge-ready because Telegram cannot accept the new `verbose` default type and several command surfaces remain incomplete.

## What This Changes

The PR updates Telegram and Feishu reasoning-preview resolution to honor per-agent/global `reasoningDefault`, adds a shared plugin SDK helper, extends `reasoningDefault` with `verbose`, and updates related docs, schema, tests, and command surfaces.

## Best Possible Solution

Land a type-clean fallback fix first, and either finish `verbose` as a supported mode across channel, ACP, TUI, and session-event surfaces or split it into a separate product-reviewed PR.

## Reproduction Assessment

Yes, source-reproducible. Current main's Telegram and Feishu preview helpers ignore configured reasoning defaults even though core directive resolution and docs expose per-agent and global `reasoningDefault`.

## Solution Assessment

No, not as currently scoped. The shared resolver is the right direction for the fallback bug, but the branch introduces `verbose` without completing Telegram type/dispatch handling, ACP/TUI affordances, mode-switch events, or real proof for the new behavior.
