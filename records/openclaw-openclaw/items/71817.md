## Summary

Keep open: current main still has the reported Telegram/Feishu config-default gap, and the PR fixes part of it, but the branch exposes configured reasoning defaults in channel preview paths without the authorization gate required by core resolution and docs.

## What This Changes

The PR updates Telegram and Feishu reasoning preview resolution to fall back to per-agent/global `reasoningDefault`, adds a public SDK helper and tests/docs, and expands `/reasoning` with a new `verbose` mode plus Telegram stream-preview retry handling.

## Best Possible Solution

Share or extend one authorized reasoning-default resolver for channel previews so Telegram and Feishu use the same precedence and authorization semantics as core, then split the new `verbose` and retry/backoff work if maintainers want those changes separately.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main hardcodes the channel preview fallback to `off`, while the PR head applies configured defaults in Telegram/Feishu without the core authorization gate. I did not run tests in this read-only review.

## Solution Assessment

No. The fallback direction is right, but the implementation must preserve the existing authorized-sender/owner/operator-admin gate before enabling visible reasoning previews from config.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate Telegram config defaults before exposing reasoning:** `extensions/telegram/src/bot-message-dispatch.ts:203-206`
  - body: The new `configDefault` path is applied without checking whether the Telegram sender is an owner, an authorized sender, or an operator-admin context. Core reasoning resolution and the config docs both gate configured reasoning defaults to those contexts, so this can expose `on`/`stream` reasoning in channel replies that core would otherwise suppress. Please pass the same authorization signal into this resolver before using config defaults.
  - confidence: 0.9
- **[P1] Gate Feishu config defaults before exposing previews:** `extensions/feishu/src/reasoning-preview.ts:11-16`
  - body: This helper now enables Feishu reasoning previews from `agents.*.reasoningDefault` without any authorization input. That bypasses the configured-default gate used by core and documented for `reasoningDefault`, so an unauthorized Feishu context can inherit `stream`/`verbose` from config and receive a reasoning preview. Please require the same authorized/owner/operator-admin check before falling back to config.
  - confidence: 0.88
