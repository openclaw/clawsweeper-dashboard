## Summary

Keep open: current main still lets skill-workshop pass raw Matrix room-style context into the embedded reviewer as a runnable channel, which can reach bundled plugin directory-name validation and produce the reported error.

## What This Changes

skill-workshop should handle Matrix room IDs gracefully or skip cleanly instead of emitting bundled plugin dirName errors after Matrix DM turns.

## Best Possible Solution

Add a skill-workshop-local reviewer channel resolver that uses real channel ids only and falls back to the message provider for Matrix room-style contexts, with regression coverage for colon-containing room ids.

## Reproduction Assessment

Yes, at source level: when the LLM reviewer runs after a Matrix turn with ctx.channelId like room:!room:server, current main forwards that value as messageChannel into embedded-run channel discovery. I did not run the live Matrix E2EE scenario in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to sanitize skill-workshop reviewer channel context like the active-memory embedded-run guard, not to relax bundled plugin dirName validation or add a config knob.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/77636. The skill-workshop LLM reviewer currently passes ctx.channelId, including Matrix room ids such as room:!room:matrix.org, as runEmbeddedPiAgent messageChannel; that can reach channel message-action/public-surface lookup and throw `Bundled plugin dirName must be a single directory`. Add a skill-workshop-local helper, or a small shared helper only if already appropriate, that uses ctx.channelId only when it is a runnable channel id and otherwise falls back to ctx.messageProvider such as matrix. Preserve sessionKey and raw room/thread context; do not relax normalizeBundledPluginDirName and do not treat room ids as plugin ids. Add regression coverage in extensions/skill-workshop/index.test.ts proving a Matrix room-style channelId results in messageChannel: "matrix" or another safe runnable channel value, not the raw room id. Add a CHANGELOG.md entry because this is a user-facing plugin fix.
