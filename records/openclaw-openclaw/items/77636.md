## Summary

Keep open: current main still has a clear source-level path where skill-workshop forwards a scoped Matrix room id as the embedded reviewer channel, which can reach bundled-plugin dirName validation and skip the reviewer.

## What This Changes

The issue reports that enabling skill-workshop in a Matrix DM causes repeated bundled-plugin dirName validation errors and skipped reviewer runs.

## Best Possible Solution

Keep the bundled dirName guard strict, but make skill-workshop pass a runnable channel id to its embedded reviewer, falling back to messageProvider when ctx.channelId is a scoped conversation or room id, with regression coverage and a changelog entry.

## Reproduction Assessment

Yes, source-reproducible: a skill-workshop LLM reviewer run with ctx.channelId set to a Matrix room target like room:!room:server forwards that value as messageChannel, and current embedded-run channel capability probing can hit bundled-plugin dirName validation. I did not run the live Matrix lane in this read-only checkout.

## Solution Assessment

Yes: the narrowest maintainable fix is plugin-owned sanitization or fallback before skill-workshop calls runEmbeddedPiAgent, matching the existing Active Memory scoped-channel guard without weakening core public-surface validation.

## Repair Work Prompt

Fix the skill-workshop Matrix/scoped-channel reviewer crash. Current main forwards ctx.channelId directly as runEmbeddedPiAgent messageChannel in extensions/skill-workshop/src/reviewer.ts; for Matrix DMs and other scoped conversation ids such as room:!room:server or values containing ':' this can reach bundled plugin dirName validation. Keep src/plugins/public-surface-runtime.ts validation strict and do not change Matrix target normalization. Implement a small skill-workshop-side resolver that passes a runnable channel id to embedded reviewer runs: prefer a non-colon explicit channel id when it is a real channel name, otherwise fall back to ctx.messageProvider, preserving messageProvider separately. Add regression coverage in extensions/skill-workshop/index.test.ts or reviewer tests for a Matrix-style room id and ensure runEmbeddedPiAgent receives messageChannel: "matrix" rather than the room id. Add a single-line CHANGELOG.md fix entry under Unreleased because this is user-facing. Stop if the fix would require a new config option or changing core plugin loader security rules.
