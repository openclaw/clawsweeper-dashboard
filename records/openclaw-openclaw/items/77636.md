## Summary

Keep open: current main still forwards skill-workshop's raw Matrix room-style channelId into the embedded reviewer as messageChannel, so the reported bundled-plugin dirName error remains source-reproducible. The closest existing fix is an adjacent Active Memory pattern, not a merged skill-workshop repair.

## What This Changes

The issue asks skill-workshop to handle Matrix room IDs during reviewer runs without crashing bundled plugin dirName validation.

## Best Possible Solution

Add a skill-workshop-local channel resolver before runEmbeddedPiAgent that keeps Matrix room identity in session context but passes a runnable channel id such as matrix as messageChannel, with regression coverage in extensions/skill-workshop/index.test.ts.

## Reproduction Assessment

Yes, at source level: a Matrix turn with messageProvider "matrix" and channelId "room:!...:matrix.org" reaches runEmbeddedPiAgent with that room id as messageChannel, then bundled channel public-surface discovery can hit the dirName validator. I did not execute the live Matrix E2EE scenario in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is in skill-workshop's embedded reviewer channel-context mapping, mirroring the Active Memory guard, not relaxing bundled plugin path validation or adding a user config option.
