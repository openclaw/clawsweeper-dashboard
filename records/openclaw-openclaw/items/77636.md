## Summary

Keep open: current main and the latest v2026.5.7 release still forward skill-workshop’s raw Matrix room-style `channelId` into the embedded reviewer as `messageChannel`, so the reported bundled plugin dirName error remains source-reproducible.

## What This Changes

The issue reports skill-workshop’s LLM reviewer crashing after Matrix DM turns because Matrix room IDs reach bundled plugin directory-name validation.

## Best Possible Solution

Sanitize skill-workshop reviewer channel context before `runEmbeddedPiAgent`, preserving Matrix room/session identity as context but using a real runnable channel id such as `matrix` for embedded-run `messageChannel`.

## Reproduction Assessment

Yes, at source level: a Matrix turn whose hook context has `messageProvider: "matrix"` and `channelId: "room:!...:matrix.org"` reaches `runEmbeddedPiAgent` with that room id as `messageChannel`, then the embedded runner’s channel discovery can hit the bundled dirName validator. I did not execute the live Matrix E2EE scenario in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is a skill-workshop-local channel resolver mirroring the active-memory guard, not relaxing bundled plugin path validation or adding a user config option.
