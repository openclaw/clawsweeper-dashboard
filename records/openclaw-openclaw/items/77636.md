## Summary

Keep open: current main still has the source path described in the issue, with skill-workshop forwarding Matrix room-style context as the embedded reviewer message channel and downstream validation still rejecting colon-containing bundled plugin names.

## What This Changes

Handle Matrix room-style channel IDs in skill-workshop reviewer context so the LLM reviewer falls back cleanly instead of treating them as plugin directory names.

## Best Possible Solution

Add a skill-workshop-local reviewer channel resolver, analogous to active-memory, that uses real runnable channel IDs and falls back to the message provider for Matrix room-style context.

## Reproduction Assessment

Yes, at source level: when ctx.channelId is a Matrix room target like room:!room:server, current main passes it as the embedded reviewer messageChannel and downstream discovery can reach bundled dirName validation. I did not live-run the Matrix E2EE scenario.

## Solution Assessment

Yes: the narrow maintainable fix is plugin-local normalization before skill-workshop calls runEmbeddedPiAgent, not relaxing bundled plugin dirName validation or adding a configuration knob.
