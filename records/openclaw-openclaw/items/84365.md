## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/84365 with a narrow Codex app-server configuration diagnostic. Source review shows `plugins.entries.codex.config.appServer.command` accepts a raw string such as `node C:\\...\\codex.js`; on Windows the shared spawn resolver can treat that whole string as a `.js` entrypoint, causing Node to load `node C:\\...\\codex.js` as one module path. Add validation and/or doctor lint so malformed `appServer.command` values containing executable plus args fail or warn before runtime, with guidance to remove the override for managed startup or use `appServer.command` for the executable and `appServer.args` for arguments. Do not add shell splitting or a broad command parser. Preserve existing legacy `openai-codex/*` route repair behavior and managed Codex startup. Update docs/manifest help if needed, add focused tests for Windows malformed command guidance, and add a changelog Fixes entry if the maintainer lane treats this as user-facing diagnostics.
