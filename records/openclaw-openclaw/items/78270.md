## Summary

Keep this PR open for maintainer/contributor follow-up: the refactor is plausible but the current diff has blocking malformed bundle JSON regressions, a supply-chain-sensitive fs-safe source change, and no after-fix real behavior proof.

## What This Changes

The PR centralizes durable JSON queue and root-bounded JSON-object reads across outbound/session queues, plugin bundle config/discovery, package metadata, package-manager detection, and Pi settings while pinning @openclaw/fs-safe to a GitHub commit.

## Best Possible Solution

Preserve diagnostics for parse/invalid bundle JSON failures, use a reviewed published fs-safe release or explicit maintainer-approved temporary pin, then add real runtime proof before merge.

## Reproduction Assessment

Not applicable for the PR as a refactor, but the review findings are source-reproducible: the changed MCP/LSP loader lines flatten parse/invalid read results that current main lets surface.

## Solution Assessment

No; centralizing the helpers is a reasonable direction, but this implementation should distinguish missing/open failures from malformed JSON and avoid an unapproved GitHub build-source dependency before merge.
