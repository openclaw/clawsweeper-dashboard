## What This Changes

The PR adds inherited tool-deny state to delegated session storage and policy, forwarding it through loopback tool resolution, sessions_spawn, native subagent spawns, ACP spawns, session patching, and focused regression tests.

## Best Possible Solution

Land a reviewed implementation that persists normalized caller-denied tools on delegated child sessions and proves in a real gateway run that a delegated session cannot regain tools removed from the caller surface.

## Reproduction Assessment

## Solution Assessment

Yes conceptually: adding a normalized inherited deny list to child session state and consuming it in the existing subagent policy path is a narrow maintainable fix. Merge should still wait for maintainer review and real behavior proof because this touches delegated tool security.
