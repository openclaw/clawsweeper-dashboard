## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use a durable restart lifecycle signal:** `src/infra/restart.ts:113-115`
  - body: `isGatewayRestartPending()` only covers `pendingRestartPreparing` or an unconsumed SIGUSR1 marker. The run loop consumes that marker before entering restart shutdown, so a deferred reload waking during shutdown can see this helper return false and still start the channel. Gate reload cancellation on state that lasts until old reload work is aborted or drained.
  - confidence: 0.93
- **[P2] Recheck cancellation immediately before startChannel:** `src/gateway/server-reload-handlers.ts:513-518`
  - body: The new restart gate runs before `await params.stopChannel(...)`. If SIGUSR1 restart becomes pending while that await is in flight, the continuation still calls `params.startChannel(name)`, leaving the duplicate-start race open; recheck immediately before `startChannel` and cover that interleaving.
  - confidence: 0.86
