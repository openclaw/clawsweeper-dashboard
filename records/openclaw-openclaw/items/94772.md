## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use a durable restart lifecycle signal:** `src/infra/restart.ts:113-115`
  - body: `isGatewayRestartPending()` only covers pending preparation or an unconsumed SIGUSR1 marker. The run loop consumes that marker before it starts the restart request, so a deferred channel reload that wakes during the actual shutdown window can see this helper return false and still start the channel, leaving the EADDRINUSE race from https://github.com/openclaw/openclaw/issues/79487 open. Gate this on cancellation/restart state that lasts until old reload work is aborted or drained.
  - confidence: 0.93
- **[P2] Recheck cancellation immediately before startChannel:** `src/gateway/server-reload-handlers.ts:513-523`
  - body: The new restart gate runs before `await params.stopChannel(...)`. If SIGUSR1 restart becomes pending while that await is in flight, the continuation still calls `params.startChannel(name)` with no second check, leaving the duplicate-start race open. Add the cancellation check immediately before `startChannel` and cover that interleaving.
  - confidence: 0.86
