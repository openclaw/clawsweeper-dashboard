## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Track restart lifecycle past SIGUSR1 consumption:** `src/infra/restart.ts:113-116`
  - body: `isGatewayRestartPending()` only checks the pre-consumption signal/preparing state, but the SIGUSR1 handler marks the restart handled before it enters the restart shutdown path. A deferred reload that wakes during `server.close()` can therefore see `false` and still run stale channel stop/start work.
  - confidence: 0.92
- **[P2] Recheck cancellation after stopping the channel:** `src/gateway/server-reload-handlers.ts:510-515`
  - body: The new guard runs before `stopChannel`, but that call awaits. If a restart becomes pending while the stop is in flight, this reload still falls through to `startChannel`, leaving the duplicate-start race possible unless cancellation covers the whole stop/start sequence.
  - confidence: 0.84
