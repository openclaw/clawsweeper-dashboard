## Summary

Keep open: the reported gateway/channel restart race is real, but this branch still has blocking lifecycle defects and a guard failure. The process-global abort can poison future reloads after an ignored SIGUSR1, can be cleared by a new lifecycle before old deferred work wakes, does not stop the plugin pre-stop path, and the committed root proof script fails `check-guards`.

## What This Changes

The branch adds a durable channel-reload abort flag exported through the gateway lifecycle runtime, calls it from SIGUSR1 handling, checks it during deferred channel reload/restart, adds regression coverage, and commits a root L2 proof script.

## Best Possible Solution

Use a lifecycle-owned cancellation/drain boundary for config reload work, or a generation-scoped abort signal captured by each reload pass, and land it with tests for ignored SIGUSR1, old/new lifecycle overlap, plugin pre-stop cancellation, clean guards, and redacted live gateway/channel proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Gate reload aborts on accepted restarts:** `src/cli/gateway-cli/run-loop.ts:746`
  - body: This call runs before the code knows whether an external SIGUSR1 is allowed. When `commands.restart=false` or external restart is disabled, the branch below logs and returns without scheduling a restart, but the durable abort flag stays set, so later channel config reloads can be skipped until a real gateway lifecycle recreates handlers.
  - confidence: 0.92
- **[P1] Keep abort state scoped to the old reload lifecycle:** `src/gateway/server-reload-handlers.ts:229`
  - body: Resetting the process-global abort flag in every new `createGatewayReloadHandlers()` can clear cancellation for an old deferred reload while that old async poll is still asleep. If the new lifecycle starts before the old poll wakes, the old reload can see cancellation as false and still run the duplicate stop/start path this PR is trying to prevent.
  - confidence: 0.9
- **[P2] Propagate cancellation before plugin pre-stop:** `src/gateway/server-reload-handlers.ts:336`
  - body: The new cancellation branch returns `void`, so the plugin-reload caller at `stopChannelsBeforePluginReplace` cannot tell that the wait was canceled and continues into the channel pre-stop loop. A SIGUSR1 during deferral can therefore still stop channel runtimes before plugin reload during restart; return a canceled result and bail out before the pre-stop path.
  - confidence: 0.88
- **[P2] Remove the root proof script from the branch:** `proof-79487-l2.mjs:1`
  - body: `check-guards` is failing because `proof-79487-l2.mjs` is a root source-like file outside duplicate-scan coverage. Keep this proof in the PR body or an external artifact, and move any durable regression value into a colocated test so the branch can pass guards.
  - confidence: 0.98
