## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use a restart lifecycle signal that survives handling:** `src/infra/restart.ts:113-115`
  - body: `isGatewayRestartPending()` only reports `pendingRestartPreparing` or an unconsumed SIGUSR1 token. The SIGUSR1 handler marks the token handled before it starts the restart request, and config-reloader stop still does not abort or drain active hot reload work, so a stale reload can see this helper return false while shutdown/restart is still underway and proceed to channel stop/start.
  - confidence: 0.9
- **[P2] Recheck restart cancellation before starting channels:** `src/gateway/server-reload-handlers.ts:520-523`
  - body: The new per-channel guard runs before `await params.stopChannel(...)`, but if restart becomes pending while stop is in flight, the continuation still calls `startChannel(name)`. Add a cancellation check immediately before `startChannel` and cover that interleaving so the stale reload cannot duplicate the restart path's channel start.
  - confidence: 0.86
