## Summary

Keep this PR open, but it is not merge-ready: it addresses a real Feishu card-action invalid-ID path, yet the patch can suppress a valid context reply target and the PR body explicitly lacks live Feishu after-fix proof.

## What This Changes

The PR changes Feishu card-action synthetic messages to detect top-level `card-action-c-*` IDs and set `suppress_reply_target` instead of forwarding `reply_target_message_id`.

## Best Possible Solution

Normalize the card-action reply target first: preserve a valid `context.open_message_id`, suppress only when the selected target is temporary or missing, cover mixed-ID and temporary-only callbacks with focused tests, then attach redacted live Feishu proof.

## Reproduction Assessment

Yes from source, but not from a live Feishu run in this review. Current main forwards card-action `open_message_id` into `replyToMessageId` and no current code filters `card-action-c-*`, while the PR body's live proof section explicitly says no live Feishu workspace was tested.

## Solution Assessment

No. Suppressing invalid temporary IDs is the right direction, but the proposed guard is too broad because it keys off the top-level callback ID instead of the selected reply target and does not preserve valid context IDs.
