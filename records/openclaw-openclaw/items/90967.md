## Summary

Keep this PR open, but it should not merge as-is: it targets a real Feishu card-action reply failure, yet the new guard can suppress a valid context reply target and the external contributor proof is source-reasoning only, not real Feishu behavior.

## What This Changes

The PR changes Feishu card-action synthetic messages so callbacks whose temporary card action IDs are detected set `suppress_reply_target` instead of passing a reply target into downstream reply delivery.

## Best Possible Solution

Preserve the real Feishu context message ID when present, suppress only the selected temporary `card-action-c-*` target, add a focused regression for that callback shape, and attach redacted live Feishu proof before merge.

## Reproduction Assessment

Yes, from source: current main can carry a `card-action-c-*` synthetic reply target through `buildSyntheticMessageEvent`, `parseFeishuMessageEvent`, and `createFeishuReplyDispatcher` into `streaming.start()`. I did not establish a live Feishu reproduction in this read-only review.

## Solution Assessment

No. Suppressing invalid temporary IDs is the right direction, but the patch should base suppression on the selected reply target or preserve the real context ID first so it does not drop valid Feishu reply metadata.
