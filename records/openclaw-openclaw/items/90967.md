## Summary

The PR addresses a real Feishu message-loss path, but the current diff still suppresses a valid `context.open_message_id` whenever the top-level callback ID is temporary, which can degrade reply targeting for card callbacks that include the original message context.

## What This Changes

The branch changes Feishu card-action synthetic message construction to suppress `reply_target_message_id` when the callback `open_message_id` starts with `card-action-c-`.

## Best Possible Solution

Preserve a valid `context.open_message_id` as the reply target, suppress only the selected temporary `card-action-c-*` target, and add a regression test for the mixed valid-context/temporary-top-level callback before merge.

## Reproduction Assessment

Yes from source: current main forwards `event.open_message_id` as `reply_target_message_id` when no context target exists, and the related issue shows Feishu rejects `card-action-c-*` IDs. I did not run a live Feishu workspace, so this is source-reproducible rather than locally reproduced.

## Solution Assessment

No: suppressing based on the top-level callback ID after selecting `context.open_message_id` is not the best fix because it can discard a valid original message target. The narrower solution is to suppress only when the selected reply target itself is temporary.
