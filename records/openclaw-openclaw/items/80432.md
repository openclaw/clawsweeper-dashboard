## Summary

Keep open. Current main still only renders Matrix outbound mention metadata for fully qualified MXIDs and @room, while bare sibling aliases such as @alice remain plain text; the related sibling-roster issues are adjacent but do not supersede this outbound auto-reply gap.

## What This Changes

Render bare sibling Matrix account aliases in agent auto-replies as Matrix m.mentions metadata and matrix.to links so mention-gated sibling bots can detect them.

## Best Possible Solution

Define a Matrix-plugin-owned outbound alias/roster contract with documented source, precedence, and collision behavior, or explicitly document that only qualified Matrix IDs are supported.

## Reproduction Assessment

Yes, at source level. Current formatter ignores bare @alice, tests lock that behavior, and Matrix auto-replies use the same send path; I did not run a live homeserver repro in this read-only review.

## Solution Assessment

Unclear. Auto-deriving aliases from configured Matrix accounts is plausible, but maintainers should first choose implicit roster versus explicit alias support and define collision handling.
