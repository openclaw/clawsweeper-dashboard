## Summary

Keep open: current main and v2026.5.12 still leave bare sibling aliases like `@alice` as plain text, while Matrix auto-replies go through the same formatter; the remaining work is a Matrix roster/alias behavior decision, not cleanup closure.

## What This Changes

The issue requests Matrix auto-replies to resolve bare sibling-account aliases into `matrix.to` anchors and `m.mentions.user_ids` metadata.

## Best Possible Solution

Define a Matrix-plugin-owned sibling mention roster or alias contract for outbound rendering, with explicit collision behavior and matching inbound handling, then cover auto-reply and send-tool paths.

## Reproduction Assessment

Yes, at source level. Current formatter and send tests show qualified MXIDs become `m.mentions` while bare `@name` stays plain text, and Matrix auto-replies use that same send path.

## Solution Assessment

Unclear. Auto-deriving aliases from configured Matrix accounts is plausible, but maintainers should first choose the roster source, alias precedence, and collision semantics.
