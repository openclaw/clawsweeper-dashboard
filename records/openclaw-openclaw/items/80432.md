## Summary

Keep open. Current main and v2026.6.10 still preserve bare sibling names like `@alice` as plain Matrix text with empty `m.mentions`, and the related issues cover adjacent inbound routing or prompt-roster gaps rather than this outbound rendering behavior.

## What This Changes

Request Matrix outbound auto-replies to turn sibling account name mentions like `@b` into `matrix.to` links and `m.mentions` metadata so mention-gated sibling agents trigger.

## Best Possible Solution

Define a Matrix-owned sibling mention alias or roster contract, then feed it into outbound send formatting with coverage for auto-replies and coordination with sibling inbound dispatch policy.

## Reproduction Assessment

Yes, source-reproducible. Current main routes Matrix auto-reply text through `sendMessageMatrix`, and the send test asserts bare `@alice` produces empty `m.mentions` with no Matrix link.

## Solution Assessment

Unclear. Auto-deriving configured sibling aliases is plausible, but maintainers need to decide whether sibling account names are implicitly mentionable or whether Matrix should grow an explicit roster/alias contract.
