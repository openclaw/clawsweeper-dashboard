## Summary

Keep open. Current main still treats bare Matrix localparts like `@alice` as plain text in outbound Matrix messages, so the reported sibling-account mention gap remains; resolving it needs Matrix product/owner judgment on roster source, alias precedence, and collision behavior.

## What This Changes

Request bare `@name` mentions of configured sibling Matrix accounts in agent auto-replies to render as Matrix pills and populate `m.mentions.user_ids`.

## Best Possible Solution

Add a Matrix-plugin-owned outbound sibling mention roster or document full-MXID-only support; if implemented, centralize alias resolution with clear precedence and collision rules across text sends, media captions, edits, polls, and auto-replies.

## Reproduction Assessment

Yes, at source level. The focused path is an agent reply or `sendMessageMatrix` call containing `@alice`, which current formatter/send tests assert remains plain text with empty `m.mentions`.

## Solution Assessment

Unclear. Auto-deriving aliases from configured sibling accounts is plausible, but an explicit roster or full-MXID-only documentation may be safer until Matrix owners decide alias precedence and collision handling.
