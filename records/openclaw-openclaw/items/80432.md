Keep open. Current main still only emits Matrix mention metadata for qualified MXIDs and @room, keeps bare sibling aliases like @alice as plain text, and sends Matrix auto-replies through that same formatter; the remaining work needs a Matrix roster and alias-semantics decision.

## What This Changes

Request Matrix auto-replies to render bare sibling account mentions as matrix.to pills with m.mentions metadata.

## Best Possible Solution

Define a Matrix-plugin-owned outbound roster or alias contract with documented source, precedence, collision handling, and interoperability with inbound sibling-target detection.

## Reproduction Assessment

Yes, at source level. Current formatter and tests show qualified MXIDs become Matrix mention metadata while bare @name remains plain text, and Matrix auto-replies route through that same send path.

## Solution Assessment

Unclear. Auto-deriving aliases from configured sibling Matrix accounts is plausible, but maintainers should first choose implicit roster versus explicit alias config and define collision handling.
