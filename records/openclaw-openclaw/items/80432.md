## Summary

Keep open. Current main only renders qualified Matrix user IDs into outbound mention metadata, while bot-to-bot docs rely on visible mentions; resolving bare sibling account names is a valid Matrix behavior gap but needs owner/product judgment on roster and alias semantics.

## What This Changes

Request Matrix outbound auto-replies to resolve sibling Matrix account @names into matrix.to anchors and m.mentions metadata.

## Best Possible Solution

Add a Matrix-plugin-owned outbound mention roster for configured sibling accounts, with documented alias precedence, collision handling, and regression coverage for auto-reply/send/edit paths; otherwise document that only full MXIDs are supported.

## Reproduction Assessment

Yes, at source level. Current tests and formatter code show qualified MXIDs get m.mentions and matrix.to anchors while bare @names stay plain text, which matches the reported auto-reply symptom.

## Solution Assessment

Unclear. Auto-deriving a sibling roster from Matrix accounts is plausible, but maintainers should decide alias precedence and collision behavior before automation changes mention semantics.
