## Summary

Keep open. Current main supports Matrix mention metadata for fully qualified MXIDs, but the auto-reply send path still treats bare sibling names like `@alice` as plain text, and the linked inbound/team-roster issues do not supersede this outbound rendering gap.

## What This Changes

Render bare sibling Matrix account mentions in agent auto-replies as `matrix.to` anchors with `m.mentions.user_ids` metadata.

## Best Possible Solution

Add a Matrix-owned sibling mention roster or alias resolver, feed it into outbound mention rendering, and document/test the collision and enabled-account semantics.

## Reproduction Assessment

Yes. Source inspection shows auto-replies call `sendMessageMatrix`, and the current send test asserts that a bare `@alice` produces empty `m.mentions` and no `matrix.to` link.

## Solution Assessment

Unclear. Auto-deriving sibling accounts is plausible, but maintainers should choose implicit roster rules versus explicit aliases before implementation because this changes Matrix mention semantics.
