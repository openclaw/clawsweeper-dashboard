## Summary

Keep open. Current main still renders Matrix outbound mentions only for qualified MXIDs and @room, while the requested bare sibling-account alias behavior is not implemented and needs a Matrix owner decision on roster, alias, and collision semantics.

## What This Changes

The issue asks for Matrix auto-replies that contain bare sibling account names like @b to emit matrix.to pills and m.mentions metadata for the sibling account.

## Best Possible Solution

Add a Matrix-plugin-owned outbound mention roster for configured sibling accounts with explicit alias precedence and collision handling, or document that only full MXIDs are supported for outbound pings.

## Reproduction Assessment

Yes, at source level. Current main routes agent replies through the Matrix send formatter, and focused tests assert that bare @alice remains plain text with empty m.mentions.

## Solution Assessment

Unclear. Auto-deriving sibling aliases from configured Matrix accounts is plausible, but maintainers should decide alias precedence, collision behavior, and whether an explicit roster config is needed before changing notification semantics.
