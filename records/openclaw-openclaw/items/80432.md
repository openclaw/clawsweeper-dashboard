## Summary

Keep open. Current main and the latest stable release still convert only qualified Matrix user IDs into `matrix.to` links and `m.mentions`, while bare sibling aliases such as `@alice` remain plain text; the requested sibling-account alias behavior is useful but needs Matrix product/owner direction before implementation.

## What This Changes

Request outbound Matrix auto-replies to resolve bare `@name` references to sibling account MXIDs and emit both `matrix.to` formatted HTML and `m.mentions` metadata.

## Best Possible Solution

Define a Matrix-plugin-owned sibling mention roster or resolver with documented alias source, precedence, and collision behavior, or explicitly document that outbound Matrix mentions require full MXIDs.

## Reproduction Assessment

Yes, at source level. The auto-reply path reaches `sendMessageMatrix`, and current formatter/send tests show qualified MXIDs become mention metadata while bare `@name` tokens remain plain text with empty `m.mentions`.

## Solution Assessment

Unclear. Auto-deriving aliases from configured sibling accounts is plausible, but the maintainable fix needs an explicit decision on roster source, alias precedence, collisions, and whether external aliases are in scope.
