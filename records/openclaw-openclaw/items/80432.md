## Summary

Keep open. Current main and the latest release still render bare sibling names like `@alice` as plain Matrix text with empty `m.mentions`; the remaining work needs a Matrix roster or alias product decision rather than cleanup closure.

## What This Changes

The issue requests Matrix outbound auto-replies to render sibling-account mentions as Matrix pills and populate `m.mentions` so mention-gated sibling bots can receive them.

## Best Possible Solution

Define a Matrix-owned sibling mention roster or alias contract, then feed it into outbound mention rendering with tests for auto-replies and coordinate it with sibling inbound dispatch policy.

## Reproduction Assessment

Yes, source-reproducible. Current main routes Matrix auto-reply text through `sendMessageMatrix`, and the send test asserts bare `@alice` produces empty `m.mentions` with no Matrix link.

## Solution Assessment

Unclear. Auto-deriving configured sibling aliases is plausible, but maintainers need to decide whether sibling account names are implicitly mentionable or whether Matrix should grow an explicit roster/alias contract.
