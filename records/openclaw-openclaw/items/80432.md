## Summary

Keep open. Current main still leaves bare Matrix sibling aliases like `@alice` as plain text, while full MXIDs render as `m.mentions` plus `matrix.to` anchors; deciding whether aliases auto-resolve or require an explicit roster needs Matrix product/owner judgment.

## What This Changes

Request outbound Matrix auto-replies to resolve bare sibling account mentions into Matrix mention metadata and `matrix.to` anchors.

## Best Possible Solution

Add Matrix-plugin-owned outbound sibling mention alias resolution with documented source, precedence, collision behavior, and regression coverage, or document that only full MXIDs are supported.

## Reproduction Assessment

Yes, at source level. The auto-reply/outbound path reaches `sendMessageMatrix`, and current tests assert bare `@alice` keeps empty `m.mentions` while full MXIDs produce mention metadata.

## Solution Assessment

Unclear. Auto-deriving a roster from configured Matrix accounts is plausible, but maintainers should decide alias source, precedence, collision handling, and whether non-OpenClaw aliases need explicit config first.
