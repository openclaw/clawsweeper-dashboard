## Summary

Keep open: the report matches a current source-level bug. `sessions_spawn` builds the task-bearing subagent prompt, but the PI prompt path drops that runtime `extraSystemPrompt` whenever the target agent has `systemPromptOverride`, leaving only the generic initial user message described in the issue.

## What This Changes

Ensure `sessions_spawn(mode="run")` passes the task text into a target sub-agent even when the target agent uses `systemPromptOverride`.

## Best Possible Solution

Preserve subagent runtime context when applying a configured system prompt override, with regression coverage for `systemPromptOverride + sessions_spawn` and without reintroducing duplicate task text in the initial user message.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction path: configure a target agent with `systemPromptOverride`, call `sessions_spawn(mode="run", task="unique token")`, and the override branch omits the `extraSystemPrompt` that contains the unique task. I did not execute a live model turn in this read-only environment.

## Solution Assessment

Yes, the narrow maintainable fix is to compose `extraSystemPrompt` into the override prompt path or an equivalent runtime-context supplement. Re-adding the task to every initial user message would regress the intentional de-duplication tracked by `#72019`.

## Repair Work Prompt

Fix openclaw/openclaw#77950. Current behavior: `sessions_spawn(mode="run", task="...")` builds the task-bearing subagent system prompt in `src/agents/subagent-spawn.ts`, but when the target agent has `agents.list[].systemPromptOverride` or `agents.defaults.systemPromptOverride`, `buildAttemptSystemPrompt` in `src/agents/pi-embedded-runner/run/attempt-system-prompt.ts` bypasses `buildEmbeddedSystemPrompt` and drops `embeddedSystemPrompt.extraSystemPrompt`, so the child only sees the generic initial message from `src/agents/subagent-initial-user-message.ts`. Implement the narrow fix so runtime `extraSystemPrompt` is preserved with configured system prompt overrides, including the subagent task under `## Your Role`. Do not solve this by globally duplicating task text into the initial user message; preserve the #72019 de-duplication behavior. Add focused regression coverage for `systemPromptOverrideText + extraSystemPrompt` and, if feasible, a subagent spawn coverage path proving a unique task token reaches the child prompt when an override is configured. This is a user-facing bug fix, so add a single-line `CHANGELOG.md` fix entry without guessing attribution. Validate with targeted tests first, then the changed gate in Testbox if available.
