## Summary

Keep open. Current main does not implement the requested follow-mode reconnect/backoff behavior; implicit loopback tails can fall back to the local log file, but explicit Gateway URL tails still treat transient transport failures as fatal, and open PR #75059 is already the closing implementation candidate with PR #75372 overlapping the same surface.

## What This Changes

Request auto-reconnect and retry/backoff handling for `openclaw logs --follow` when transient Gateway WebSocket or RPC disconnects occur.

## Best Possible Solution

Review and reconcile #75059 with #75372, then land one focused `logs --follow` retry/backoff implementation that preserves local fallback and fatal-error boundaries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path for explicit Gateway URLs: run `openclaw logs --follow --url ws://127.0.0.1:18789`, trigger a closed or timed-out Gateway transport, and the catch path exits with status 1.

## Solution Assessment

Yes in direction. The narrow maintainable fix belongs in `src/cli/logs-cli.ts` with targeted tests/docs for follow-mode transient retries, while keeping one-shot non-follow RPCs and non-recoverable errors fatal.
