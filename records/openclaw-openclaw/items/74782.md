## Summary

Keep open. Current `main` still exits from the `openclaw logs --follow` polling loop on non-fallback Gateway transport errors, and open PR #75059 is an active closing implementation candidate that should be reviewed or replaced before this issue closes.

## What This Changes

The issue requests that `openclaw logs --follow` retry transient Gateway disconnects with warning/backoff instead of terminating the log tail session.

## Best Possible Solution

Land or adapt one focused CLI fix that retries only transient follow-mode Gateway transport failures with bounded backoff and clear warning output while preserving existing local fallback and fatal handling.

## Reproduction Assessment

Yes. The manual path is to run `openclaw logs --follow`, restart or interrupt the Gateway, and observe the CLI exit; the source-level path is the current explicit-URL test that mocks a closed `GatewayTransportError` and expects `process.exit(1)`.

## Solution Assessment

Yes in direction. The narrow maintainable fix belongs in the logs CLI follow loop with targeted tests/docs/changelog, not as a broad retry change to every one-shot Gateway RPC call.
