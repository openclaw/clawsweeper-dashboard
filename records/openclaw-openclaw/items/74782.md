## Summary

Keep open. Current main still exits the `openclaw logs --follow` polling loop on Gateway transport errors, and open PR #75059 is already linked as a closing fix for this issue.

## What This Changes

Request auto-reconnect with warning/backoff for `openclaw logs --follow` after transient Gateway disconnects instead of terminating the CLI session.

## Best Possible Solution

Review, land, or adapt one focused CLI fix that retries only transient follow-mode Gateway transport failures with bounded backoff and clear warning output while preserving existing local fallback and fatal-error handling.

## Reproduction Assessment

Yes. The manual path is to run `openclaw logs --follow`, restart or interrupt the Gateway, and observe the CLI exit; the code-level path is to mock a closed or timed-out `GatewayTransportError` in the follow loop with an explicit `--url`.

## Solution Assessment

Yes in direction. The narrow maintainable solution belongs in `src/cli/logs-cli.ts` with targeted tests and docs/changelog updates, rather than adding blanket retries to every one-shot Gateway RPC call.
