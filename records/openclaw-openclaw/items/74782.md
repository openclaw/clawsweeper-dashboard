## Summary

Keep open. Current main still exits from the `openclaw logs --follow` polling loop on non-fallback Gateway transport failures, and open PR #75059 explicitly closes this issue while #75372 overlaps the same surface.

## What This Changes

Request `openclaw logs --follow` to auto-reconnect with warning/backoff on transient Gateway disconnects instead of terminating the log tail session.

## Best Possible Solution

Review and land or replace one focused CLI fix that retries transient follow-mode transport failures with bounded backoff while preserving local fallback and fatal handling for non-recoverable errors.

## Reproduction Assessment

Yes. The manual path is to run `openclaw logs --follow`, restart or interrupt the Gateway, and observe the CLI exit; the code-level path is the existing explicit-URL test that mocks a closed `GatewayTransportError` and expects `process.exit(1)`.

## Solution Assessment

Yes in direction. The narrow maintainable fix belongs in `src/cli/logs-cli.ts` follow polling with targeted tests and docs, not in broad retry behavior for every one-shot Gateway RPC.
