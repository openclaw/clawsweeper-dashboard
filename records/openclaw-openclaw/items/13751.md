## Summary

Keep open. Current main still uses contact-backed Feishu sender-name lookup by default and live user enumeration, while the overlapping maintainer PR is open, protected, and not shipped.

## What This Changes

Request to make basic Feishu sender attribution and directory behavior avoid org-wide contact-directory permissions by default.

## Best Possible Solution

Make basic inbound Feishu contact-free by default, fall back quietly to stable sender ids, and gate contact-backed profile lookup plus live user-directory listing behind explicit opt-in with docs and regression tests.

## Reproduction Assessment

Yes, from source inspection rather than a live Feishu replay. With default config, inbound handling calls `resolveFeishuSenderName`, which calls `contact.user.get`, and live peer listing calls `contact.user.list`.

## Solution Assessment

No, current main is not the best final solution because opt-out lookup and stale-scope notice suppression still leave contact-backed access on by default. The safer solution is explicit contact access with contact-free basic inbound messaging.
