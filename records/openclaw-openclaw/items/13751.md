## Summary

Keep open: current main and the latest release still keep Feishu sender-name resolution contact-backed by default, and live peer listing still enumerates the contact directory. The report remains valid and security-sensitive, but the fix changes Feishu permission/default behavior and overlaps assigned maintainer follow-up, so it should not be auto-closed or queued as a competing repair.

## What This Changes

The issue requests making basic Feishu sender-name handling stop depending on `contact:contact.base:readonly` and making live contact-directory lookup explicit rather than default behavior.

## Best Possible Solution

Make basic inbound Feishu operation contact-free by default, quietly fall back to stable sender ids, and require explicit documented opt-in for contact-backed profile and live directory lookup.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.5` default `resolveSenderNames` to true, call `contact.user.get` for sender names, and call `contact.user.list` for live peer listing.

## Solution Assessment

Yes directionally, but the exact policy needs maintainer review. The maintainable path is a Feishu-plugin-owned contact-free default with explicit opt-in contact lookup and tests/docs for upgrade behavior.
