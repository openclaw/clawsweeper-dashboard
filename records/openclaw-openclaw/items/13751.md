## Summary

Keep open: current main and the latest release still perform Feishu contact-directory lookups for sender names by default and for live peer listing. The stale-scope notice is partly suppressed now, but the underlying security/auth-provider concern remains and the related open PRs do not fully supersede this issue.

## What This Changes

The issue requests making basic Feishu sender attribution work without requiring the broad `contact:contact.base:readonly` contact-directory permission.

## Best Possible Solution

Make basic inbound Feishu operation contact-free by default, using safe event or chat-member metadata when available, quietly falling back to stable sender IDs, and requiring explicit documented opt-in for contact-backed profile and live directory lookup.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.20` default `resolveSenderNames` to true, call `contact.user.get` for sender names, and still call `contact.user.list` for live peer listing.

## Solution Assessment

Yes directionally, but it needs maintainer policy and security review. The maintainable path is a Feishu-plugin-owned default that avoids contact directory access for basic inbound messages, with opt-in contact lookup plus tests/docs for upgrade behavior.
