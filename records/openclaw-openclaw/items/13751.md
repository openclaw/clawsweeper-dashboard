## Summary

Keep open. Current main still performs contact-backed Feishu sender-name lookup by default and live user enumeration, while repeated user reports and an overlapping protected maintainer PR show the remaining work is valid but needs maintainer coordination.

## What This Changes

Requests that Feishu basic inbound messaging stop depending on broad contact-directory permission for sender-name resolution, with live directory listing gated separately.

## Best Possible Solution

Make basic inbound Feishu contact-free by default, fall back quietly to stable sender ids, and gate contact-backed profile lookup plus live user enumeration behind explicit opt-in with docs and regression tests.

## Reproduction Assessment

Yes. From source inspection on current main, default inbound handling calls `resolveFeishuSenderName`, that resolver calls `contact.user.get`, and live peer listing calls `contact.user.list`.

## Solution Assessment

No. The current opt-out flag and stale-scope notice suppression reduce noise but still leave broad contact-backed access on default paths; the safer solution is contact-free basic inbound behavior with explicit opt-in for directory-backed lookups.
