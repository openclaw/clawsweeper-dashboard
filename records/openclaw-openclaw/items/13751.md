## Summary

Keep open: current main and the latest release still resolve Feishu sender names through contact APIs by default, and live peer listing still enumerates the contact directory. The report is valid and security-sensitive, but the issue is assigned and overlaps an open maintainer-labeled Feishu display-name PR, so the next step is maintainer coordination rather than auto-close or a competing repair PR.

## What This Changes

Request Feishu sender-name and directory behavior that does not require broad contact-directory permission for basic inbound bot operation.

## Best Possible Solution

Make basic inbound Feishu operation contact-free by default, quietly fall back to stable sender IDs, and require explicit documented opt-in for contact-backed profile and live directory lookup.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.22` default `resolveSenderNames` to true, call `contact.user.get` for sender names, and call `contact.user.list` for live peer listing.

## Solution Assessment

Yes directionally, but the exact policy needs maintainer review. The maintainable path is a Feishu-plugin-owned contact-free default with explicit opt-in contact lookup and tests/docs for upgrade behavior.
