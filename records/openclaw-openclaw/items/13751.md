## Summary

Keep open. Current main still performs Feishu contact-backed sender-name lookup by default and still uses live contact-directory enumeration; the overlapping maintainer PR is open and conflicting rather than a shipped contact-free default.

## What This Changes

The issue asks Feishu sender-name resolution and live directory listing to avoid broad contact-directory API permissions by default.

## Best Possible Solution

Make basic inbound Feishu contact-free by default, fall back quietly to the stable sender id, and put contact-backed profile lookup plus live user-directory listing behind explicit opt-in with docs and regression tests.

## Reproduction Assessment

Yes, from source inspection rather than a live Feishu replay. With default config, inbound handling calls `resolveFeishuSenderName`, which calls `contact.user.get`, and live user directory listing calls `contact.user.list`.

## Solution Assessment

No, current main is not the best final solution because it suppresses one stale-scope notice while still performing contact-backed lookups by default. A maintainable fix should change the default sender metadata path and make contact-directory lookup explicit.
