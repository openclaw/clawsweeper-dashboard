## Summary

Keep open: current main and v2026.6.8 still default Feishu sender-name resolution on and use contact-directory APIs, so the reported sensitive-permission dependency remains. The issue is valid, but the fix changes Feishu permission/default behavior and overlaps an assigned maintainer-owned PR, so it should stay in maintainer review rather than be closed or queued as a competing repair.

## What This Changes

Request to make Feishu sender-name resolution and live user directory lookup avoid default dependence on `contact:contact.base:readonly` for basic bot use.

## Best Possible Solution

Make basic inbound Feishu messages contact-free by default, use verified low-scope event or chat-member metadata when available, fall back quietly to a stable sender id, and require explicit opt-in for contact-backed profile lookup and live user-directory listing with docs and regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main defaults `resolveSenderNames` to true and then calls `client.contact.user.get` for inbound sender names; I did not run a live Feishu tenant in this read-only review.

## Solution Assessment

Yes on direction, but not yet on final implementation shape. The maintainable fix should stay inside the Feishu plugin, make contact-backed lookups explicit, and validate the low-scope replacement path before changing defaults.
