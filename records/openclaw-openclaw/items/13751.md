## Summary

Keep open: current main and the latest release still default to contact-backed Feishu sender-name lookup and live peer listing. The issue remains valid and security-sensitive, and the overlapping assigned PR at https://github.com/openclaw/openclaw/pull/73399 is still open with human-review/security blockers rather than a landed fix.

## What This Changes

Request to make Feishu sender attribution and live user listing stop depending on broad contact-directory permissions by default.

## Best Possible Solution

Make basic inbound Feishu operation contact-free by default, quietly fall back to stable sender IDs, and require explicit documented opt-in for contact-backed profile and live directory lookup with regression coverage.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.22` default `resolveSenderNames` to true, call `contact.user.get` for sender names, and call `contact.user.list` for live peer listing.

## Solution Assessment

Yes directionally, but the exact policy needs maintainer review. The maintainable path is a Feishu-plugin-owned contact-free default with explicit opt-in contact lookup and tests/docs for upgrade behavior.
