## Summary

Keep open. Current main still performs contact-backed Feishu sender-name lookup and live peer enumeration, multiple commenters reported organization/security impact, and the overlapping maintainer path at https://github.com/openclaw/openclaw/pull/73399 is open, protected, and human-review paused rather than shipped.

## What This Changes

Request to make Feishu basic sender attribution and directory behavior avoid contact-directory permissions by default, using stable sender ids or lower-scope metadata instead.

## Best Possible Solution

Make basic inbound Feishu contact-free by default, fall back quietly to stable sender ids, and gate contact-backed profile lookup plus live user-directory listing behind explicit opt-in with docs and regression tests.

## Reproduction Assessment

Yes, from source inspection rather than a live Feishu replay. With default config, inbound handling calls `resolveFeishuSenderName`, which calls `contact.user.get`, and live peer listing calls `contact.user.list`.

## Solution Assessment

No, current main is not the best final solution because stale-scope notice suppression and an opt-out flag still leave contact-backed lookup on by default. The safer path is explicit contact access with contact-free basic inbound messaging.
