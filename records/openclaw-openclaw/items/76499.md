## Summary

Keep open. Current main still misses the exact @homebridge/ciao `Could not find valid addresses for interface` assertion, so the Bonjour plugin will not claim that rejection before the global fatal unhandled-rejection handler exits the gateway.

## What This Changes

The issue asks for the Bonjour ciao classifier to treat `Could not find valid addresses for interface` as a non-fatal, no-recovery warning.

## Best Possible Solution

Add a Bonjour plugin-local `no-valid-addresses` classification, log it as a warning without recovery, add focused regression tests, and keep the core fatal unhandled-rejection policy unchanged.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `@homebridge/ciao@1.3.7` can throw the reported assertion, current Bonjour code does not classify it, and unclassified rejections still reach the fatal global handler; I did not perform a live network-interface repro.

## Solution Assessment

Yes. The proposed plugin-local classifier plus no-recovery warning is the narrowest maintainable fix; changing the global fatal handler or disabling Bonjour by default would be broader than this dependency-specific failure.
