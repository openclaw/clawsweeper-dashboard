## Summary

Keep open. Current main still lacks the requested Bonjour classifier, and an open fixing PR explicitly targets this issue, so the issue should stay open until that PR is reviewed and merged or closed.

## What This Changes

The issue asks Bonjour to classify @homebridge/ciao's no-valid-addresses interface assertion as non-fatal instead of letting it crash the gateway.

## Best Possible Solution

Land a narrow Bonjour plugin fix that adds a `no-valid-addresses` ciao classification, treats it as a warning without futile recovery, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: @homebridge/ciao 1.3.7 can throw the reported assertion, current main does not classify it, and unhandled rejections still exit when plugin handlers return false.

## Solution Assessment

Yes. A plugin-local classifier and no-recovery warning is the narrowest maintainable fix; changing the global fatal handler or disabling Bonjour more broadly would be too broad for this dependency-specific failure.
