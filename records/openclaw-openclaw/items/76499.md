## Summary

Keep open. Current main still misses the exact ciao `Could not find valid addresses for interface` assertion: the Bonjour classifier handles other known ciao failures, but an unclassified rejection still falls through to the global fatal unhandled-rejection path.

## What This Changes

Handle a Bonjour/@homebridge/ciao assertion for interfaces with no ciao-valid address as non-fatal so the Gateway does not exit on an unhandled rejection.

## Best Possible Solution

Add a plugin-local `no-valid-addresses` ciao classification, handle it as a warning without recovery, add focused regression coverage, and leave core unhandled-rejection policy unchanged.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: ciao 1.3.7 can throw the reported assertion from `NetworkManager.getCurrentNetworkInterfaces`, and current main has no classifier branch for that message. I did not live-create an IPv6-only interface, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

Yes, the proposed plugin-local classifier plus no-recovery warning is the narrowest maintainable fix. Changing the global fatal handler or disabling Bonjour by default would be broader than the observed dependency-specific failure.
