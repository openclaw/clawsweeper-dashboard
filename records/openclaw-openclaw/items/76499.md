## Summary

Keep open. Current main still lacks a Bonjour classifier for ciao's `Could not find valid addresses for interface` assertion, and the open linked fix PR should be reviewed or landed before this issue closes.

## What This Changes

Add Bonjour handling for ciao's IPv6-only or address-less interface assertion so it is treated as non-fatal instead of crashing the gateway.

## Best Possible Solution

Review and land #76993, or an equivalent plugin-local classifier with regression coverage and a changelog entry, then close this issue as implemented.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: ciao 1.3.7 can throw the reported assertion, current main does not classify that message, and unclassified rejections reach the fatal global handler. I did not live-create an IPv6-only interface, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

Yes, the proposed plugin-local classifier and warn-only/no-recovery handling is the narrowest maintainable fix. Changing the global unhandled-rejection policy or treating this as a default-enablement issue would be broader than the dependency-specific failure.
