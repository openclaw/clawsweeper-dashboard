## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Move the nested guard before ingress echo:** `src/agents/tools/sessions-send-tool.ts:297-308`
  - body: The ingress echo gateway send runs before the nested sessions_send provenance guard. With ingress echo enabled, a recursive A2A call can still post to the target channel and only then return forbidden, defeating the default loop guard.
  - confidence: 0.91
- **[P2] Keep rejecting thread session targets:** `src/agents/tools/sessions-send-tool.ts:251-257`
  - body: Current main rejects thread-scoped sessions_send targets before starting a run. This branch falls through into baseline capture, echo, and target-run setup, allowing thread session keys for inter-agent coordination despite the documented contract.
  - confidence: 0.86
- **[P2] Preserve inter-session prompt annotation:** `src/agents/tools/sessions-send-tool.ts:392-405`
  - body: The target run receives the raw message instead of the annotated inter-session prompt text that current main uses, dropping the marker that distinguishes tool-routed data from a direct user instruction.
  - confidence: 0.88
- **[P2] Pass the baseline into the A2A flow:** `src/agents/tools/sessions-send-tool.ts:462-477`
  - body: Current main passes a baseline snapshot into the A2A wait path so delayed or fast target runs do not announce stale replies. This branch omits that baseline and reads the latest assistant reply directly.
  - confidence: 0.84
- **[P2] Honor strict relay delivery failures:** `src/agents/tools/sessions-send-tool.a2a.ts:88-95`
  - body: relay.requireDelivery is exposed in config and docs, but relay sends catch and log failures. Since the A2A flow then continues, strict relay delivery cannot block or abort as promised.
  - confidence: 0.88
- **[P2] Include the relay turn in idempotency keys:** `src/agents/tools/sessions-send-tool.a2a.ts:84`
  - body: The relay idempotency key only includes run id, direction, and target index. In mirrorTurns="all", later turns with the same direction reuse the same key, so gateway idempotency can collapse distinct conversation turns.
  - confidence: 0.86
- **[P2] Suppress non-deliverable control tokens:** `src/agents/tools/sessions-send-tool.a2a.ts:119-130`
  - body: Current main filters SILENT_REPLY, HEARTBEAT, REPLY_SKIP, and ANNOUNCE_SKIP before A2A delivery. The relay path mirrors replies before that filter and the announce path only checks ANNOUNCE_SKIP, so control tokens can become visible channel messages.
  - confidence: 0.84
- **[P2] Continue A2A after non-terminal wait timeouts:** `src/agents/tools/sessions-send-tool.ts:523-530`
  - body: Current main treats a non-terminal agent.wait timeout as accepted and starts the A2A follow-up flow so delayed replies still arrive. This branch returns a timeout immediately, dropping that delayed requester delivery path.
  - confidence: 0.87
- **[P2] Preserve the native subagent duplicate-delivery skip:** `src/agents/tools/sessions-send-tool.ts:441-445`
  - body: Current main skips A2A follow-up for waited parent-owned native subagents because the parent already receives the inline reply. This branch only checks the background ACP case, so native subagents can resume duplicate ping-pong delivery.
  - confidence: 0.82
- **[P2] Compare account IDs before blocking bound sends:** `src/infra/outbound/message-action-runner.ts:148-153`
  - body: The new bound-session guard compares only channel and normalized target. For multi-account providers such as Slack, the same target id can exist under different accounts or workspaces, so this can block valid agent sends for unrelated accounts.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The patch changes external A2A/channel delivery and currently weakens recursive relay, prompt-provenance, strict-delivery, and account-boundary protections.

Concerns:

- **[medium] Outbound echo happens before recursive relay guard:** `src/agents/tools/sessions-send-tool.ts:297`
  - body: The nested sessions_send guard is meant to prevent runaway or recursive A2A relays by default, but the PR performs ingress echo first, allowing a blocked recursive relay to cause an external channel post.
  - confidence: 0.91
- **[medium] Inter-session prompt marker is dropped:** `src/agents/tools/sessions-send-tool.ts:393`
  - body: Current main annotates inter-session messages before target execution so the target prompt sees them as tool-routed data; the PR sends the raw message while retaining only metadata, weakening that prompt-safety boundary.
  - confidence: 0.86
- **[medium] Strict relay failures are swallowed:** `src/agents/tools/sessions-send-tool.a2a.ts:88`
  - body: The new relay.requireDelivery option cannot enforce delivery because relay send failures are caught and logged inside the relay helper, so callers continue after a configured strict delivery failure.
  - confidence: 0.88
- **[low] Bound-session guard ignores account identity:** `src/infra/outbound/message-action-runner.ts:148`
  - body: The new outbound guard can treat same-named targets in different provider accounts as the same bound conversation, which can wrongly block legitimate sends or misroute operator expectations in multi-workspace setups.
  - confidence: 0.74
