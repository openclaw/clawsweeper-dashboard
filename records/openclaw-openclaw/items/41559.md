  - body: Current main skips the A2A flow for requester-owned native subagent sends as well as ACP children. This branch only checks the ACP background case, so parent-owned native subagents can re-enter the ping-pong/announce path and produce duplicate parent replies.

  - body: The relay idempotency key only includes run id, direction, and target index. In `mirrorTurns="all"`, later turns with the same direction reuse the same key, so gateway idempotency can collapse distinct conversation turns.
  - confidence: 0.87

  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff changes external A2A/channel delivery and weakens recursive relay, prompt-provenance, strict-delivery, and account-boundary protections.

Concerns:

- **[medium] Outbound echo happens before recursive relay guard:** `src/agents/tools/sessions-send-tool.ts:297`
  - body: The nested `sessions_send` guard is meant to prevent recursive A2A relays by default, but the PR performs ingress echo first, allowing a blocked recursive relay to cause an external channel post.
  - confidence: 0.92
- **[medium] Inter-session prompt marker is dropped:** `src/agents/tools/sessions-send-tool.ts:393`
  - body: Current main annotates inter-session messages before target execution so the target prompt sees them as tool-routed data; the PR sends the raw message while retaining only metadata, weakening that prompt-safety boundary.
  - confidence: 0.9
- **[medium] Strict relay failures are swallowed:** `src/agents/tools/sessions-send-tool.a2a.ts:88`
  - body: The new `relay.requireDelivery` option cannot enforce delivery because relay send failures are caught and logged inside the relay helper, so callers continue after a configured strict delivery failure.
  - confidence: 0.9

  - body: The new outbound guard can treat same-named targets in different provider accounts as the same bound conversation, which can wrongly block legitimate sends in multi-workspace setups.
  - confidence: 0.78
