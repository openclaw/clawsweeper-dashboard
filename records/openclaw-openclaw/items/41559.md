Full review comments:

- **[P1] Move the nested guard before ingress echo:** `src/agents/tools/sessions-send-tool.ts:297-308`
  - body: The ingress echo gateway send runs before the nested `sessions_send` guard. With ingress echo enabled, a recursive A2A call can still post to the target channel and only then return `forbidden`, which defeats the default loop guard's purpose.
  - confidence: 0.91
- **[P2] Preserve inter-session prompt annotation:** `src/agents/tools/sessions-send-tool.ts:393`
  - body: The PR sends the raw `message` into the target run instead of the current main annotated inter-session prompt text. That drops the visible prompt marker that tells the target agent this is tool-routed data, not a direct end-user instruction.
  - confidence: 0.86
- **[P2] Keep rejecting thread session targets:** `src/agents/tools/sessions-send-tool.ts:267-278`
  - body: Current main rejects thread-scoped `sessions_send` targets before starting a run. This branch falls through into baseline capture, echo, and target-run setup, so a thread session key can be used for inter-agent coordination despite the current docs and guard.
  - confidence: 0.84
- **[P2] Honor strict relay delivery failures:** `src/agents/tools/sessions-send-tool.a2a.ts:73-95`
  - body: `relay.requireDelivery` is exposed in config and docs, but relay sends catch and log failures. Since the A2A flow then continues, strict relay delivery cannot block or abort as promised.
  - confidence: 0.88
- **[P2] Include the relay turn in idempotency keys:** `src/agents/tools/sessions-send-tool.a2a.ts:84`
  - body: The relay idempotency key only includes run id, direction, and target index. In `mirrorTurns: "all"`, later turns with the same direction reuse the same key, so gateway idempotency can collapse distinct conversation turns.
  - confidence: 0.86
- **[P2] Preserve baseline reply filtering:** `src/agents/tools/sessions-send-tool.a2a.ts:109-111`
  - body: Current main passes a baseline snapshot into the A2A wait path so delayed or fast target runs do not announce stale replies. This branch reads the latest assistant reply without that fingerprint check, reintroducing stale or duplicate A2A announce behavior.
  - confidence: 0.78
- **[P2] Commit generated config outputs:** `src/config/zod-schema.session.ts:62-80`
  - body: Adding these session config fields changes the computed base config schema and config-doc metadata, but the generated schema artifacts do not contain the new keys. Existing generated-schema/config-doc checks can fail and public config metadata can stay stale.
  - confidence: 0.9
- **[P2] Compare account IDs before blocking bound sends:** `src/infra/outbound/message-action-runner.ts:148-153`
  - body: The new bound-session guard compares only channel and normalized target. For multi-account providers such as Slack, the same target id can exist under different accounts or workspaces, so this can block a valid agent send because another agent is bound to a different account.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `docs/concepts/session-tool.md:130-156`
  - body: This is a user-facing feature/config change, but the PR does not include a `CHANGELOG.md` entry. Add a single-line entry under the active release section with valid contributor credit before this is merge-ready.
  - confidence: 0.82
