## Summary

Keep open. The PR addresses a real Telegram delivery-target bug, but current main moved the active implementation to `extensions/telegram`, while this branch changes obsolete `src/telegram` files and still normalizes too late for parser-first send/chat-type paths.

## What This Changes

The PR adds a Telegram delivery group-id normalizer to old `src/telegram` target helpers and tests bare `group:<numeric>` normalization and send delivery.

## Best Possible Solution

Rework or replace the PR against `extensions/telegram/src/targets.ts` so numeric bare `group:` delivery targets are normalized before topic parsing, with focused regression coverage for send delivery, target normalization, and exec approval chat-type routing.

## Reproduction Assessment

Yes. Source inspection on current main shows `group:-100...` remains prefixed, `group:123456789` can hit the generic colon parser, and send/chat-type paths parse before the helper normalizers run.

## Solution Assessment

No. The idea is aimed at the right bug, but the branch targets obsolete paths and applies normalization after the parser entry point that current send and approval routing use.
