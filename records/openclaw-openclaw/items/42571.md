## Summary

Close as mostly implemented on current main: the useful Telegram announce-delivery behavior is now handled by the plugin-owned session target path and covered by regression tests, while this older branch still edits removed `src/telegram/*` files and leaves the parser path incomplete.

## What This Changes

The PR adds a helper to strip bare `group:<numeric_chat_id>` before Telegram chat-id and lookup-target normalization, plus old `src/telegram` regression tests for direct send delivery.

## Best Possible Solution

Keep the shipped plugin-owned announce target resolution; any future request to accept bare `group:` as a direct Telegram send target should be filed against the current `extensions/telegram` paths with real Telegram proof.

## Reproduction Assessment

Yes. Source inspection of the PR diff gives a high-confidence branch reproduction: `parseTelegramTarget` is not normalized, so positive `group:<id>` inputs are split as a chat id of `group` plus a thread id, and group-prefixed chat type resolution remains wrong.

## Solution Assessment

No. The current plugin-owned session-target implementation is the narrower maintained solution for the announce-delivery bug, while this stale branch edits removed files and leaves an important parse path incomplete.

- **[P2] Normalize parsed delivery targets as well:** `src/telegram/targets.ts:8-12`
  - body: The new helper only runs in `normalizeTelegramChatId` and `normalizeTelegramLookupTarget`, but the send path calls `parseTelegramTarget` first. Apply the same delivery-only normalization before topic/colon parsing if this branch is revived.
