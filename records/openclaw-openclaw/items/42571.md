## What This Changes

The PR adds a Telegram helper to strip bare `group:<numeric_chat_id>` targets in chat-id and lookup normalization and adds old-path regression coverage for normalization and send delivery.

## Best Possible Solution

Land a replacement or repaired PR against `extensions/telegram/src/targets.ts` that normalizes bare numeric `group:` delivery targets at the shared parser/stripper boundary, with send, chat-type, topic, and changelog coverage.
