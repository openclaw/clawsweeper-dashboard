## Summary

Keep open: the underlying Telegram target bug is still source-reproducible on current main, but this PR is not merge-ready because it edits obsolete `src/telegram` paths and does not normalize before the active parser consumes `group:` targets.

## What This Changes

The PR adds a helper and tests under old `src/telegram` paths to strip bare numeric `group:` prefixes in Telegram chat-id and lookup normalization.

## Best Possible Solution

Land a repaired or replacement branch against `extensions/telegram/src` that normalizes bare numeric `group:` targets at the shared strip/parse boundary with send, chat-type/approval, and target-normalization coverage.

## Reproduction Assessment

Yes. Source inspection against `origin/main` shows `parseTelegramTarget` preserves bare `group:` before topic parsing, and send/approval paths use that parser before lookup; I did not run tests in this read-only review.

## Solution Assessment

No. The branch targets removed `src/telegram` files and applies normalization only in chat-id/lookup helpers, while the maintainable fix belongs before parsing in the active Telegram plugin.
