## Summary

Keep open. Current main still has the bare Telegram `group:<numeric>` delivery gap in the active plugin, and this PR is not merge-ready because it targets obsolete `src/telegram/*` paths and only normalizes helper paths rather than the parser that send, target normalization, and approval routing share.

## What This Changes

The PR adds bare `group:<numeric>` stripping to Telegram chat-id and lookup normalization helpers under `src/telegram/*` and adds target/send regression tests there.

## Best Possible Solution

Rework or replace the PR against `extensions/telegram/src` so bare delivery-only `group:<numeric>` targets are stripped before parser topic handling, while preserving explicit `telegram:group:` legacy targets and forum topic suffix behavior across send, messaging target normalization, and approval routing.

## Reproduction Assessment

Yes. The static path is deterministic on current main: `stripTelegramInternalPrefixes("group:-100123")` is expected to preserve the prefix, `parseTelegramTarget` consumes that unnormalized value, and `sendMessageTelegram` resolves `target.chatId` afterward.

## Solution Assessment

No. The proposed direction is useful, but the patch is not the best fix because it edits obsolete paths and misses the parser-level normalization point used by send, target normalization, and chat-type routing.
