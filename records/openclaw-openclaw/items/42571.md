## Summary

Keep this PR open for now, but do not merge it as-is: the reported Telegram `group:` delivery bug is still source-reproducible on current main, while this branch is stale against pre-extension paths, incomplete for `parseTelegramTarget`, and lacks real Telegram behavior proof.

## What This Changes

The PR adds a Telegram helper to normalize bare `group:<numeric_chat_id>` delivery targets in chat-id and lookup-target helpers, plus unit coverage for normalization and mocked send delivery.

## Best Possible Solution

Port a root-level Telegram target fix to `extensions/telegram/src/targets.ts` so strip, parse, normalize, send, and chat-type callers all handle bare `group:` consistently, then add focused regressions and real Telegram proof.

## Reproduction Assessment

Yes, by source inspection: current main only strips `group:` after a Telegram prefix, while send resolution depends on the resulting numeric chat ID and otherwise throws. I did not run live Telegram or tests because this review was required to keep the checkout read-only.

## Solution Assessment

No. Applying a helper only to two normalizers in old `src/telegram` files is not the best fix; the current extension parser/strip layer should be fixed once so `parseTelegramTarget`, chat-type resolution, and all send helpers share the behavior.
