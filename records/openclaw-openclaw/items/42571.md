## Summary

Keep open. The PR is useful source context, but it patches the old `src/telegram/*` location while current main routes Telegram through `extensions/telegram/src/*`, and the proposed normalization is still applied after the parser path used by send, target normalization, and approval routing. The related merged #51688 is relevant prior work, but current main still contains the guarded `group:` stripping behavior and tests that preserve bare `group:` prefixes.

## What This Changes

The PR adds a `normalizeTelegramDeliveryGroupChatId` helper in `src/telegram/targets.ts`, calls it from chat-id and lookup-target normalization, and adds target/send regression tests under `src/telegram/*`.

## Best Possible Solution

Rework or replace the PR against `extensions/telegram/src/targets.ts` so bare delivery-only `group:<numeric>` targets are normalized before topic parsing, with focused coverage for send delivery, messaging target normalization, and exec approval chat-type routing.

## Reproduction Assessment

Yes. The current-main static path is deterministic: the test suite still expects bare `group:-100123` to remain prefixed, `sendMessageTelegram` parses before resolving the chat id, and the generic colon parser can misread `group:123456789` as a topic-qualified target.

## Solution Assessment

No. The proposed helper is aimed at the right user problem, but applying it only in chat-id and lookup normalizers and under obsolete `src/telegram/*` files is not the narrowest maintainable fix for current main.
