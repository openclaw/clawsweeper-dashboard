## Summary

Keep open. This PR is aimed at a real Telegram target-normalization bug, but the branch changes obsolete `src/telegram` files while current main uses `extensions/telegram`, and the proposed normalization still misses the parser entry point used by send and chat-type routing.

## What This Changes

The PR adds a Telegram delivery group-id normalizer under `src/telegram` and tests bare `group:<numeric>` normalization plus send-time delivery.

## Best Possible Solution

Replace or rework the PR against `extensions/telegram/src/targets.ts` so bare numeric `group:` targets are normalized before topic/colon parsing, with regression coverage for delivery, target normalization, and approval chat-type routing.

## Reproduction Assessment

Yes. Source inspection on current main shows bare `group:` targets remain prefixed in the active Telegram parser, and send plus approval paths consume `parseTelegramTarget` before the helper normalization added by this PR would run.

## Solution Assessment

No. The fix direction is valid, but this branch patches obsolete paths and applies normalization too late; the narrow maintainable fix belongs in the active Telegram plugin parser before topic/colon splitting.
