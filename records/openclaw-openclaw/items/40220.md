## Summary

Keep open: current main still writes a local-offset top-level `time` while leaving tslog's nested `_meta.date` as a `Date` that JSON serializes to UTC, so the raw JSON log mismatch remains source-reproducible. The direct fix PR is closed unmerged, and the related open logs-display issue tracks CLI rendering rather than this raw field consistency bug.

## What This Changes

The issue requests consistent raw log timestamp fields when file logs include both `_meta.date` and top-level `time`.

## Best Possible Solution

Patch the file-log serializer to clone `_meta` and set `_meta.date` to the same formatted value as `time`, with focused regression coverage; leave CLI display defaults to https://github.com/openclaw/openclaw/issues/46748.

## Reproduction Assessment

Yes, at source level. Current main formats top-level `time` with a local offset while serializing the unchanged tslog `_meta.date`, whose dependency contract is a `Date` and therefore JSON-renders as UTC.

## Solution Assessment

Yes. Aligning `_meta.date` inside `src/logging/logger.ts` is the narrowest maintainable fix; changing CLI display defaults or timezone configuration would solve a different surface.
