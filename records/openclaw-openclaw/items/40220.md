## Summary

Keep open: current main and the latest release still compute a local-offset top-level `time` while serializing tslog `_meta.date` unchanged, so raw JSON file logs can still expose UTC and local-offset timestamps for the same event. The direct fix PRs are closed unmerged, and https://github.com/openclaw/openclaw/issues/46748 tracks CLI display defaults rather than this raw-field mismatch.

## What This Changes

The issue asks for raw JSON log timestamps to be consistent by aligning `_meta.date` with the configured local-offset `time` field, or otherwise documenting the UTC/local split.

## Best Possible Solution

Patch the file-log serializer to clone `_meta` and serialize `_meta.date` as the already computed `time`, with regression coverage for exact equality; keep CLI display-default behavior in https://github.com/openclaw/openclaw/issues/46748.

## Reproduction Assessment

Yes, at source level rather than by live execution. Current main computes a local-offset top-level `time` while serializing unchanged tslog `_meta.date`, whose `Date` contract JSON-renders as UTC.

## Solution Assessment

Yes. Aligning `_meta.date` inside `src/logging/logger.ts` is the narrowest maintainable fix, while changing `openclaw logs` display defaults belongs to https://github.com/openclaw/openclaw/issues/46748.
