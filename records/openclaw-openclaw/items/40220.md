## Summary

Keep open: current main still serializes a local-offset top-level `time` while preserving the original nested `_meta.date`, so raw JSON logs can show two different timestamp representations. The earlier direct fix PR is closed unmerged, and the open adjacent logs CLI PR targets display defaults rather than this file-log field mismatch.

## What This Changes

The issue asks for raw log timestamp fields to be consistent when `_meta.date` is UTC but top-level `time` is local-offset time.

## Best Possible Solution

Patch the file-log serializer to copy `_meta` and set `_meta.date` to the computed top-level `time`, with regression coverage for that invariant while keeping the separate CLI display work on https://github.com/openclaw/openclaw/issues/46748 separate.

## Reproduction Assessment

Yes, at source level. Current main computes an offset `time` field and serializes the original nested `_meta.date`, which JSON serializes a Date as UTC `Z`; I did not run a live gateway tail in this read-only pass.

## Solution Assessment

Yes. Aligning `_meta.date` inside `src/logging/logger.ts` is the narrowest maintainable fix; changing CLI display defaults or agent timezone configuration would be broader than this raw log-field defect.
