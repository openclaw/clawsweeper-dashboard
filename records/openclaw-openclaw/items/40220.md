## Summary

Keep open: current main still writes a local-offset top-level `time` while preserving tslog's nested `_meta.date`, so raw JSON file logs can still contain two timestamp zones. The direct fix PRs are closed unmerged, and the related open logs issue tracks CLI display defaults rather than raw field consistency.

## What This Changes

The issue reports raw JSON logs where `_meta.date` remains UTC while top-level `time` uses the configured/local timezone offset.

## Best Possible Solution

Update the file-log serializer to clone `_meta` and serialize `_meta.date` as the already computed `time`, with regression coverage asserting equality.

## Reproduction Assessment

Yes, at source level. Current main computes a local-offset top-level `time` while serializing unchanged tslog `_meta.date`, whose dependency contract is a `Date` that JSON-renders as UTC.

## Solution Assessment

Yes. Aligning `_meta.date` inside `src/logging/logger.ts` is the narrowest maintainable fix; changing `openclaw logs` display defaults belongs to https://github.com/openclaw/openclaw/issues/46748.
