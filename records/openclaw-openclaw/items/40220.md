## What This Changes

The issue requests consistent timezone rendering between raw log `_meta.date` and top-level `time` fields.

## Best Possible Solution

Patch the file-log serializer to copy `_meta` and set `_meta.date` to the computed `time`, with regression coverage; leave `openclaw logs` display defaults to https://github.com/openclaw/openclaw/issues/46748.

## Reproduction Assessment

Yes, at source level. Current main computes a local-offset `time` field and serializes the original nested `_meta.date`, which JSON serializes a Date as UTC `Z`.

## Solution Assessment

Yes. Aligning `_meta.date` inside `src/logging/logger.ts` is the narrowest maintainable fix; changing CLI display defaults or agent timezone configuration would be broader than this raw log-field defect.
