## Summary

Keep open: current main and latest release still write a local-offset top-level `time` while preserving tslog's nested `_meta.date`, so raw JSON log lines can still contain two timezone representations. The direct fix PRs are closed unmerged, and https://github.com/openclaw/openclaw/issues/46748 is related but tracks CLI display defaults rather than raw field consistency.

## What This Changes

The issue requests consistent raw log timestamp serialization so `_meta.date` and top-level `time` do not disagree between UTC and the configured/local timezone.

## Best Possible Solution

Patch the file-log serializer to clone `_meta` and serialize `_meta.date` as the already computed `time`, with regression coverage for equality; keep CLI display-default behavior in the separate related issue.

## Reproduction Assessment

Yes, at source level. Current main computes a local-offset top-level `time` while serializing unchanged tslog `_meta.date`, whose dependency contract is a `Date` that JSON-renders as UTC.

## Solution Assessment

Yes. Aligning `_meta.date` inside `src/logging/logger.ts` is the narrowest maintainable fix, while changing `openclaw logs` display defaults belongs to https://github.com/openclaw/openclaw/issues/46748.
