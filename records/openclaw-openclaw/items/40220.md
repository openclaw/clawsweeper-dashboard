## Summary

Keep open: current main and the latest release still add a local-offset top-level `time` while preserving tslog's nested `_meta.date`, so raw JSON file logs can still show two timestamp zones. The direct fix PRs are closed unmerged, and the adjacent open logs issue covers CLI display defaults rather than this raw-field consistency bug.

## What This Changes

The issue asks for raw JSON file logs to use a consistent timezone for `_meta.date` and top-level `time`.

## Best Possible Solution

Update the file-log serializer to clone `_meta` and serialize `_meta.date` as the already computed `time`, with regression coverage asserting equality.

## Reproduction Assessment

Yes, at source level. Current main computes a local-offset top-level `time` while serializing unchanged tslog `_meta.date`, whose dependency contract is a `Date` that JSON-renders as UTC.

## Solution Assessment

Yes. Aligning `_meta.date` inside `src/logging/logger.ts` is the narrowest maintainable fix; the separate CLI display-default behavior should remain with https://github.com/openclaw/openclaw/issues/46748.
