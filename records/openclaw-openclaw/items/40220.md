## Summary

Keep open: current main still writes a local-offset top-level `time` while preserving tslog's nested `_meta.date` Date, so raw JSON file logs can still show the two timestamp representations reported here.

## What This Changes

The issue asks OpenClaw to make file-log `_meta.date` consistent with the top-level local-offset `time` field.

## Best Possible Solution

Patch the file-log serializer to copy `_meta` and set `_meta.date` to the computed `time`, with regression coverage for that invariant while keeping the separate CLI display-default work on https://github.com/openclaw/openclaw/issues/46748 separate.

## Reproduction Assessment

Yes, at source level. Current main computes an offset `time` field and serializes the original nested `_meta.date`, which JSON serializes a Date as UTC `Z`; I did not run a live gateway tail in this read-only pass.

## Solution Assessment

Yes. Aligning `_meta.date` inside `src/logging/logger.ts` is the narrowest maintainable fix; changing CLI display defaults or agent timezone configuration would be broader than this raw log-field defect.
