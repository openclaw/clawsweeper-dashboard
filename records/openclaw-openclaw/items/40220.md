## Summary

Keep open: current main and v2026.5.19 still compute a local-offset top-level `time` while serializing tslog's nested `_meta.date` unchanged, so the raw JSON log mismatch remains. The direct fix PRs are closed unmerged, and the related open logs issue/PR tracks CLI rendering rather than this raw field consistency bug.

## What This Changes

Make raw JSON log `_meta.date` use the same local-offset timestamp as top-level `time` so log timestamps are consistent.

## Best Possible Solution

Update the file-log serializer to clone `_meta` and set `_meta.date` to the already computed `time`, with regression coverage asserting equality; leave `openclaw logs` display defaults to https://github.com/openclaw/openclaw/issues/46748.

## Reproduction Assessment

Yes, at source level. Current main formats top-level `time` with a local offset while serializing the unchanged tslog `_meta.date`, whose dependency contract is a `Date` and therefore JSON-renders as UTC.

## Solution Assessment

Yes. Aligning `_meta.date` inside `src/logging/logger.ts` is the narrowest maintainable fix; changing CLI display defaults or timezone configuration would solve a different surface.
