## Summary

Keep open: current main still writes a local-offset top-level `time` while spreading the original tslog record into the file-log JSON, so nested `_meta.date` remains a UTC `Date` serialization. The closed attempted fix and the open adjacent CLI logs PR do not resolve this raw file-log field mismatch.

## What This Changes

The issue requests consistent timestamp fields in JSON file logs so `_meta.date` and `time` do not show different time zones.

## Best Possible Solution

Patch the file-log serializer to shallow-copy `_meta` and set `_meta.date` to the computed `time`, then add regression coverage for that invariant.

## Reproduction Assessment

Yes, at source level. Current main computes an offset `time` field and serializes the original nested `_meta.date`, which JSON stringifies a Date as UTC `Z`; I did not run a live gateway tail in this read-only pass.

## Solution Assessment

Yes. Aligning `_meta.date` inside the file-log serializer is the narrowest maintainable fix; changing CLI display defaults, UI display, or agent timezone configuration would be broader than this raw log-field defect.
