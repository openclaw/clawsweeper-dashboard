## Summary

Keep open: current main still writes a local-offset top-level `time` while serializing tslog's nested `_meta.date` unchanged, so raw JSON file logs can still contain two timestamp zones; the direct fix PRs are closed unmerged and the related open logs issue tracks CLI display behavior rather than this raw-field consistency bug.

## What This Changes

The issue requests consistent log timestamps so `_meta.date` and top-level `time` no longer show different timezones in JSON file logs.

## Best Possible Solution

Use tslog's `_meta.date` as the timestamp source when present, format it once as the local-offset string, and write that same string to both top-level `time` and a cloned `_meta.date` with regression coverage.

## Reproduction Assessment

Yes, at source level. Current main computes a local-offset top-level `time` while serializing unchanged tslog `_meta.date`, whose dependency contract is a `Date` that JSON-renders as UTC.

## Solution Assessment

Yes. Aligning `_meta.date` inside `src/logging/logger.ts` is the narrowest maintainable fix; changing `openclaw logs` display defaults belongs to https://github.com/openclaw/openclaw/issues/46748.
