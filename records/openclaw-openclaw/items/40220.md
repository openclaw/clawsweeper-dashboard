## Summary

Keep open: current main still computes a local-offset top-level `time` while serializing tslog `_meta.date` unchanged, the direct fix PRs are closed unmerged, and the related open logs item tracks a different CLI display-default problem.

## What This Changes

The issue asks OpenClaw to stop emitting raw JSON logs where `_meta.date` is UTC while the top-level `time` field uses the configured/local timezone.

## Best Possible Solution

Update the file-log serializer to clone `_meta` and set `_meta.date` to the already computed local-offset `time`, with regression coverage asserting equality.

## Reproduction Assessment

Yes, at source level. Current main formats top-level `time` with a local offset while preserving tslog `_meta.date` as a `Date`, which JSON serializes as UTC.

## Solution Assessment

Yes. Aligning `_meta.date` in `src/logging/logger.ts` is the narrowest maintainable fix; CLI display defaults and timezone flags are a separate issue.
