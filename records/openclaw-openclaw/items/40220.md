## Summary

Keep open: current main still writes a local-offset top-level `time` while preserving tslog's nested `_meta.date`, so raw JSON file logs can still contain two timestamp zones. The direct fix PRs were closed unmerged, and the related open logs item/PR is about CLI display defaults rather than raw field consistency.

## What This Changes

Align raw JSON log `_meta.date` with the top-level `time` field so file-log timestamps use one consistent offset-aware value.

## Best Possible Solution

Update the file-log serializer to clone `_meta` and set `_meta.date` to the already computed `time`, with regression coverage asserting equality.

## Reproduction Assessment

Yes, at source level. Current main formats the top-level `time` with a local offset while serializing unchanged tslog `_meta.date`, whose dependency contract is a `Date` that JSON-renders as UTC.

## Solution Assessment

Yes. Aligning `_meta.date` inside `src/logging/logger.ts` is the narrowest maintainable fix; changing CLI display defaults or timezone configuration would solve a different surface.
