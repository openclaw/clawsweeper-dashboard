## Summary

Keep this PR open. It addresses a real ACP mode run cleanup gap on current main, but the proposed fixed 24-hour fallback can close valid long-running or no-timeout ACP runs if the lifecycle terminal event is missed; the broader parent-reset/PID orphan cleanup remains tracked by #68916.

## What This Changes

The PR adds ACP mode run lifecycle cleanup, dispatch-failure runtime cleanup, and regression tests in `src/agents/acp-spawn.ts` and `src/agents/acp-spawn.test.ts`.

## Best Possible Solution

Keep the lifecycle end/error and dispatch-failure cleanup, but make fallback cleanup depend on confirmed terminal state or the effective run timeout plus grace; leave parent reset cleanup, PID-level orphan recovery, and startup reaping to #68916.

## Reproduction Assessment

Yes. A focused code-level reproduction is an ACP `mode: "run"` spawn with `runTimeoutSeconds: 0` or a timeout longer than 24 hours where the lifecycle terminal event is missed; the PR fallback still closes the runtime at the fixed 24-hour deadline.

## Solution Assessment

No. The cleanup direction is narrow and maintainable, but the fallback policy should respect existing timeout semantics instead of using an unconditional wall-clock cap.
