## Summary

Keep open: current main and v2026.6.10 still do not auto-forward Node `agent.request` image attachments to chat, and the remaining request is a new security-sensitive gateway/media delivery contract rather than conservative cleanup.

## What This Changes

Request to add an `agent.request` media auto-forward option so Node-supplied images with `deliver: true` can reach the target IM user without an agent-authored media reply.

## Best Possible Solution

Approve a narrow explicit opt-in protocol/API design with authorization, limits, audit, and agent override semantics, or keep media forwarding on direct send and message-tool surfaces.

## Reproduction Assessment

Yes. Source inspection on current main shows `agent.request` attachments become `images`/`imageOrder` for agent input, while final delivery uses only media present in the agent reply payloads.

## Solution Assessment

Unclear. A narrow explicit opt-in is safer than automatic fallback forwarding, but maintainers must decide whether `agent.request` should become a media relay or whether direct-send/message-tool surfaces remain the supported path.
