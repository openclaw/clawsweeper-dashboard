## Summary

Current main implements substantial foundations for configurable semantic memory search, optional transcript indexing, usage observability, and first-turn startup context, but it still lacks the requested general session handoff, pre-message restart warmup, and agent-facing evaluation metrics. The issue remains useful as a canonical product umbrella, especially given the May 10, 2026 report of repeated cross-session decision rediscovery, but implementation should wait for maintainers to split and define the security-sensitive contracts.

## What This Changes

Add agent performance metrics, broader semantic search, cross-session context handoff, and restart-time context preload.

## Best Possible Solution

Keep the existing search, usage, and startup-context foundations, then split the remaining metrics API, normal-session handoff, and restart warmup into separate security-reviewed product designs with explicit ownership and acceptance criteria.
