## Summary

Current main now covers much of the semantic-search, usage-observability, startup-context, and boot groundwork, but it still does not provide the requested general-session summary handoff or clearly expose task-completion and satisfaction signals as an agent-facing evaluation API. The recent operator report confirms that cross-session recall remains a real product concern, and no narrower active item has replaced this canonical umbrella.

## What This Changes

Request agent-facing performance metrics, broader semantic recall, cross-session context handoff, and restart-time context preloading.

## Best Possible Solution

Retain the current plugin-based recall, usage RPC, bounded startup-context, and BOOT.md mechanisms, then split the unresolved evaluation API, normal-session handoff, and restart lifecycle semantics into focused designs with explicit security and ownership boundaries.
