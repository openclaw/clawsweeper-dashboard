## Summary

Current main implements much of the semantic-search, usage-observability, startup-context, and memory-backend groundwork, but it does not fully provide agent-facing self-evaluation metrics, general OpenClaw session chaining, or an explicit restart-time agent preload before the first user turn. The recent operator report also confirms the remaining cross-session recall problem is still meaningful, so this umbrella should remain open for maintainer scoping rather than be closed as implemented.

## What This Changes

Add agent-facing performance metrics, broader semantic memory search, cross-session context handoff, and restart-time context preload.

## Best Possible Solution

Preserve the current semantic-search and usage infrastructure, then split the unresolved work into focused designs for agent-visible metrics, normal-session summary handoff, and restart lifecycle behavior, each with explicit privacy boundaries and validation criteria.
