## Summary

Keep open. Current `main` now registers `openclaw agent --model`, but the CLI Backends quick start is still invalid because it omits a required session selector, and https://github.com/openclaw/openclaw/pull/74613 is still the open closing PR for the docs fix.

## What This Changes

The issue asks the CLI Backends quick start to use an `openclaw agent` command that runs on current releases and to clarify the `codex-cli` setup path.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/74613, or an equivalent docs-only edit, so the quick start uses `--agent main` and explains when to replace that default agent id.

## Reproduction Assessment

Yes. Current `main` still documents the selector-free quick-start command, while docs and source require `--agent`, `--session-id`, or `--to` before dispatch.

## Solution Assessment

Yes. The linked docs-only fix is the narrowest maintainable solution because `main` is the source-backed default agent id and no runtime CLI or Gateway behavior needs to change.
