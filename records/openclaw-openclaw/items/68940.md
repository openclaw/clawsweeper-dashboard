## Summary

Keep open. Current main has the `openclaw agent --model` flag and bundled `codex-cli/gpt-5.5` backend, but the CLI Backends quick-start still documents a selector-free `openclaw agent` command that the current gateway and embedded paths reject; open PR #74613 is the active closing path.

## What This Changes

The issue asks to correct the CLI Backends quick-start so the documented Codex CLI command works on current releases and explains the backend registration prerequisites accurately.

## Best Possible Solution

Review and land #74613, or an equivalent docs-only change, so the quick start uses a valid selector such as `--agent main` and explains why that target works on a fresh install.

## Reproduction Assessment

Yes. Current main still documents the selector-free quick-start command, and current gateway plus embedded source checks reject that command shape before any model run starts.

## Solution Assessment

Yes. A docs-only correction is the narrowest maintainable fix because `--model` and the bundled `codex-cli` backend already exist; making `openclaw agent` infer a session would be a broader runtime behavior change.
