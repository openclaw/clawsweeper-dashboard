## Summary

Keep open. Current main has the `openclaw agent --model` flag and bundled `codex-cli` backend, but the CLI Backends quick start still documents a selector-free command that current `openclaw agent` paths reject; the open closing PR at https://github.com/openclaw/openclaw/pull/74613 is the active fix path.

## What This Changes

The issue asks OpenClaw to stop documenting a CLI Backends quick-start command that `openclaw agent` rejects and to clarify the valid Codex CLI backend invocation.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/74613, or an equivalent docs-only edit, so the quick start uses `--agent main` and explains when to replace that default agent id.

## Reproduction Assessment

Yes. Current main still documents the selector-free quick-start command, and both the public agent docs and source guards show that command is rejected before a model run starts.

## Solution Assessment

Yes. Adding `--agent main` in docs is the narrow maintainable fix because the runtime already supports `--model` and already defines `main` as the no-explicit-agent default.
