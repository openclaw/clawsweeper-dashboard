## Summary

Keep open. Current main and the latest release still publish a selector-free CLI Backends quick-start command; `--model` is now supported, but the documented command still fails because `openclaw agent` requires `--agent`, `--to`, or `--session-id`, and the linked open PR remains the active docs fix path.

## What This Changes

The issue asks for the CLI Backends quick-start docs to stop showing an `openclaw agent` command that fails on released OpenClaw because the invocation is not accepted as written.

## Best Possible Solution

Refresh and land https://github.com/openclaw/openclaw/pull/74613 so the current Claude CLI quick start uses `openclaw agent --agent main --message "hi" --model claude-cli/claude-sonnet-4-6` and briefly explains the default `main` agent id.

## Reproduction Assessment

Yes. Source and docs inspection gives a high-confidence reproduction path: the CLI Backends quick start omits a selector, while both the documented contract and runtime paths require `--agent`, `--to`, or `--session-id`.

## Solution Assessment

Yes. A docs-only selector fix is the narrow maintainable solution because current main already supports `--model` and already has a default `main` agent id for no-config setups.
