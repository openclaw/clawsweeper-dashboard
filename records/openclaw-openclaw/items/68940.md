## Summary

Keep open. Current main has fixed the original `--model` flag and bundled `codex-cli` registration pieces, but the CLI Backends quick start still documents a selector-free `openclaw agent --message ... --model ...` command that current gateway and embedded agent paths reject. The linked open PR #74613 is the active closing fix path, so this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue reports that the CLI Backends quick start documents an `openclaw agent --message ... --model ...` command that fails because `openclaw agent` requires a session selector and the `codex-cli` backend was not presented with clear zero-config prerequisites.

## Best Possible Solution

Review and land or adjust #74613 so the CLI Backends quick start uses a selector accepted by current CLI behavior, such as `--agent main`, and briefly explains why that default agent id works. If maintainers want selector-free `openclaw agent` quick starts, that should be a deliberate runtime change with tests rather than a docs-only cleanup.

## Reproduction Assessment

Yes. Current main still has a source-level reproduction path: the quick-start docs omit a selector, while both gateway and embedded command paths deterministically throw without `--to`, `--session-id`, or `--agent`.

## Solution Assessment

Yes. The narrow docs fix proposed by #74613 is the best current path because `--model` and `codex-cli` registration are already present on main; making `openclaw agent` infer a default session would be broader runtime behavior.
