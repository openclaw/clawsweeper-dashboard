## Summary

Keep open. Current main has fixed the original `--model` flag and bundled `codex-cli` registration, but the CLI Backends quick-start still documents a selector-free `openclaw agent --message ... --model ...` command that current gateway and embedded paths reject; the linked open PR #74613 remains the active closing path.

## What This Changes

The issue asks for the CLI Backends quick-start docs to stop recommending an `openclaw agent --model` command that fails on released OpenClaw versions and to clarify `codex-cli` registration prerequisites.

## Best Possible Solution

Review and land or adjust #74613 so the CLI Backends quick start uses a selector accepted by current CLI behavior, such as `--agent main`, and explains why that default agent id works.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: the documented quick-start command omits every accepted session selector, while both gateway and embedded agent command paths deterministically throw before running the model.

## Solution Assessment

Yes. The narrow docs correction in #74613 is the best current path because `--model` and bundled `codex-cli` registration already exist on main; changing `openclaw agent` to infer a session would be broader runtime behavior.
