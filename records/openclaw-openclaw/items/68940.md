## Summary

Keep open. Current main fixes the original missing `--model` flag and bundled `codex-cli` registration, but the CLI Backends quick start still documents a selector-free `openclaw agent` command that current gateway and embedded paths reject; #74613 remains the active closing PR.

## What This Changes

The issue asks for the CLI Backends quick start to use a supported `openclaw agent` invocation and clarify when the bundled `codex-cli` backend is available.

## Best Possible Solution

The docs should show a selector-valid quick start, preferably `--agent main`, and briefly explain that `main` is the default agent id when no explicit agent list is configured.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: the documented command omits every accepted selector, and both current command paths throw before running the model when no selector is present.

## Solution Assessment

Yes. A narrow docs correction is the best path because `--model` and bundled `codex-cli` registration already exist on main; making `openclaw agent` infer a session would be broader runtime behavior.
