## Summary

Keep open. Current main has restored `openclaw agent --model`, but the CLI Backends quick-start still documents a selector-free command that current gateway and embedded agent code rejects, and https://github.com/openclaw/openclaw/pull/74613 is still the open closing PR for the narrow docs fix.

## What This Changes

The issue asks for the CLI Backends quick-start docs to stop showing an `openclaw agent --message ... --model codex-cli/...` command that fails on current CLI behavior.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/74613, or an equivalent docs-only edit, so the quick start includes a valid selector such as `--agent main` and explains when to replace it.

## Reproduction Assessment

Yes. Source and docs inspection shows current main still documents a selector-free command, while both gateway and embedded agent paths reject runs without `--to`, `--session-id`, or `--agent`.

## Solution Assessment

Yes. Adding `--agent main` in the docs is the narrow maintainable fix because current main already supports `--model` and already defines `main` as the no-explicit-agent default.
