## Summary

Keep open. Current main has the `openclaw agent --model` flag and bundled `codex-cli` backend, but the CLI Backends quick-start still shows a selector-free command that current gateway and embedded agent paths reject; the linked open PR #74613 is the active closing path.

## What This Changes

The issue asks for the CLI Backends quick-start docs to stop recommending an `openclaw agent --message ... --model ...` command that fails on released CLI behavior.

## Best Possible Solution

Review and land #74613, or an equivalent docs change, so the quick start includes a valid selector such as `--agent main` and explains why that target works.

## Reproduction Assessment

Yes. Current main still documents the selector-free quick-start command, and both the public agent docs and source-level gateway/embedded guards show that command is rejected before a model run starts.

## Solution Assessment

Yes. The linked docs-only fix is the narrowest maintainable solution because the runtime already supports `--model`; changing `openclaw agent` to infer a session would be broader behavior.
