## Summary

Keep open. Current main and v2026.6.8 still have classified cron error observability, transient retry handling, generic recurring backoff, and failure alerts, but not the requested reason-specific recurring cron circuit breaker, Retry-After-aware job guard state, or job-level pause/disable recovery contract.

## What This Changes

Request reason-aware cron guardrails that classify quota, auth, rate-limit, format, timeout, and unknown failures, apply safe backoff or circuit-break policy, persist guard state, and expose it to clients.

## Best Possible Solution

Keep this issue open until maintainers define and implement a core cron guard policy with reason-specific thresholds, durable guard state, client visibility, alerts, docs, and regression coverage coordinated with provider/auth-profile cooldowns.

## Reproduction Assessment

Yes at source level, but not with a live provider reproduction in this review. Current main shows classified error capture and generic recurring retry/backoff without a job-level reason-specific circuit breaker or guard state, and the issue discussion includes production retry-storm reports.

## Solution Assessment

No, current main is only a partial solution. The best path is a maintainer-approved guard policy across cron scheduling, persisted state, public schema, UI/docs, and tests rather than another narrow timer-only change.
