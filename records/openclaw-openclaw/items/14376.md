## Summary

Keep open. Current main and latest v2026.6.8 still only provide classified cron error observability, transient retry handling, generic recurring backoff, and failure alerts; the requested reason-specific recurring cron circuit breaker, Retry-After-aware guard state, and job-level pause/disable recovery contract remain missing.

## What This Changes

The issue requests reason-aware cron guardrails that classify quota, auth, rate-limit, timeout, format, and unknown failures to choose backoff, circuit-break, observability, and alert behavior.

## Best Possible Solution

Define one recurring-cron guard contract that reuses failover classification, retry hints, provider cooldowns, and failure alerts, then add the minimal persisted job state plus protocol, UI, docs, and tests for safe pause, recovery, and operator visibility.

## Reproduction Assessment

Yes, at source level: current main shows classified error capture and transient retry handling followed by generic recurring error backoff, with no reason-specific circuit breaker or job guard state. I did not run a live provider retry-storm reproduction.

## Solution Assessment

Partly. Reusing FailoverReason, retry hints, auth-profile cooldowns, and failure alerts is the right foundation, but maintainers still need to choose thresholds, recovery semantics, config exposure, and provider-versus-job ownership before implementation.
