## Summary

Keep open. Current main and v2026.6.10 still provide classified cron error observability plus generic retry/backoff, but not the requested reason-specific billing/auth/format circuit breaker, Retry-After-aware job guard state, or durable pause/disable policy.

## What This Changes

The issue requests a core cron guard layer that classifies quota, auth, rate-limit, timeout, format, and unknown failures and applies reason-specific backoff, circuit-break, alert, and state behavior.

## Best Possible Solution

Define one maintainer-approved core cron guard policy with reason-specific thresholds, durable job guard state, client visibility, alert behavior, docs, and regression coverage coordinated with existing auth-profile cooldown and retry helpers.

## Reproduction Assessment

Yes, at source level. Current main records lastErrorReason and has transient retry/generic recurring backoff, but recurring cron still lacks a reason-specific job circuit breaker, Retry-After-aware guard state, or durable pause/disable fields.

## Solution Assessment

Unclear as an implementation, but the core scheduler boundary is appropriate. The best fix needs a cohesive policy across timer behavior, persisted state, protocol/UI visibility, docs, and tests rather than another timer-only patch.
