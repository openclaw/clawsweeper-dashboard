## Summary

Keep open. Current main and v2026.6.10 still provide classified cron error observability, transient retry, failure alerts, and generic recurring error backoff, but not the requested reason-specific billing/auth/format circuit breaker, Retry-After-aware job guard state, or durable pause/disable recovery policy.

## What This Changes

Request reason-aware cron guardrails that classify quota, auth, rate-limit, timeout, and format failures and apply appropriate backoff, circuit breaking, alerts, and visible job state.

## Best Possible Solution

Define and implement one maintainer-approved core cron guard policy with reason-specific thresholds, durable guard state, client visibility, alert behavior, docs, and regression coverage coordinated with existing auth-profile cooldown and retry helpers.

## Reproduction Assessment

Yes, at source level. Current main records lastErrorReason and has transient retry/generic recurring backoff, but recurring cron still lacks a reason-specific job circuit breaker, Retry-After-aware guard state, or durable pause/disable fields; no live provider storm was replayed.

## Solution Assessment

No, current main is only a partial solution. The best path is a cohesive cron guard policy across scheduling, persisted state, public schema, UI/docs, and tests rather than another timer-only patch.
