## Summary

Keep open for maintainer product/API/privacy review. Current main and v2026.5.12 still expose only `mention` and `always` group activation, while this request would add new cross-channel ambient observation, LLM routing, state, config, logging, and privacy-sensitive model exposure.

## What This Changes

Request an opt-in contextual group activation mode for Telegram and WhatsApp groups using a peeking/engaged LLM decision state machine, decision logs, schedules, fallbacks, and CLI log viewing.

## Best Possible Solution

Keep this as a maintainer-owned design item: define an explicit opt-in activation/API contract with privacy, retention, cost, observability, and channel-owner responsibilities before accepting a cross-channel implementation.

Not applicable. This is a feature request, and source inspection shows current main still has no contextual mode to reproduce as broken existing behavior.
