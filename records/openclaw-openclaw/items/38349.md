## Summary

Keep open for maintainer product/API/privacy review: current main and the latest release still expose group activation as mention-gated or always-on, while this request asks for a new ambient decision-model activation mode with state, routing, scheduling, logging, and reply-authority semantics across Telegram and WhatsApp.

Add contextual group activation for Telegram and WhatsApp so unmentioned group messages can be observed, model-scored, and transition between peeking and engaged states.

## Best Possible Solution

Define a maintainer-approved contextual activation contract with explicit opt-in, privacy/retention controls, cost limits, channel ownership, plugin/API boundaries, and separate passive-observation versus reply-authority semantics before implementation.

- The open related issue https://github.com/openclaw/openclaw/issues/47167 covers passive skipped-message visibility for archive plugins, but it does not decide whether OpenClaw should initiate replies from observed skipped traffic.
