## Summary

Keep open for maintainer product/API/security review: current main and v2026.5.22 still expose mention-gated or always-on group activation plus ambient room events, but not the requested cross-channel LLM peeking/engaged activation state machine.

## What This Changes

The issue requests a configurable contextual group activation mode for WhatsApp and Telegram using an LLM-driven peeking/engaged state machine instead of explicit mentions only.

## Best Possible Solution

Define a maintainer-approved contextual activation contract with explicit opt-in, privacy and retention controls, cost and rate limits, channel ownership, and separate passive-observation versus reply-authority semantics before implementation.

Not applicable as a bug reproduction. Source and release inspection give a high-confidence current-behavior trace: activation is mention|always, while ambient room events are a separate always-on quiet-context mode rather than the requested contextual state machine.
