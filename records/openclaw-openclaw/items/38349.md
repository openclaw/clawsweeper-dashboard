The issue requests LLM-driven contextual activation for group chats, with peeking/engaged state, model fallbacks, per-group config, decision logs, and CLI log viewing.

Maintain one explicit product-design track for consent-aware group activation: either narrow this to a core/plugin API seam that can safely support LLM peeking, or choose the simpler attention-window behavior as the supported product direction and document why.
