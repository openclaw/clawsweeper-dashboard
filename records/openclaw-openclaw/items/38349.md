## Summary

Keep open: current main and the latest release have ambient room events and mention/always activation, but not the requested LLM peeking/engaged classifier, WhatsApp support, decision-model fallback chain, scheduling, decision logs, or contextualActivation config. This remains a broad privacy-sensitive product decision rather than conservative cleanup.

## What This Changes

The issue requests an LLM-driven contextual activation mode for group chats with peeking and engaged states across WhatsApp and Telegram.

## Best Possible Solution

Adopt one consent-aware group-attention architecture that reuses ambient room events where possible, adds only the minimal generic seam needed by unsupported channels, and introduces a separate classifier only if maintainers approve its privacy, cost, and participation semantics.

## Reproduction Assessment

Not applicable. This issue proposes new group-chat behavior rather than reporting a failure of an existing documented contract.

## Solution Assessment

No. The proposed parallel classifier and peeking/engaged state machine is not yet proven to be the narrowest maintainable solution because current ambient room events already let the main agent quietly observe and decide when to participate on supported channels.
