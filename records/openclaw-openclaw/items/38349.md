## Summary

Current main does not implement ambient LLM-driven group activation, and the linked implementation PR closed without merging. The request remains useful but requires an explicit product and security decision because it would expose otherwise unaddressed group content to model providers and add broad cross-channel config, state, routing, and observability surfaces.

## What This Changes

Requests an opt-in LLM-driven peeking/engaged activation mode for Telegram and WhatsApp group conversations.

## Best Possible Solution

Adopt a narrow activation contract that preserves mention-only secure defaults, lets channel plugins supply explicit continuation signals, stores engagement state in canonical SQLite storage, and permits ambient model classification only after maintainers approve its consent and data-disclosure policy.

## Reproduction Assessment

Not applicable: this issue requests behavior beyond the current explicit and implicit mention contract rather than reporting a failing existing contract.

## Solution Assessment

No. The proposed monolithic peeking/engaged implementation is not yet the best solution because it crosses plugin ownership and introduces broad privacy, configuration, storage, logging, and fallback policy; the narrower attention-window behavior should be evaluated first.
