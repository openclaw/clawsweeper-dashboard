## Summary

Keep open: current main still only supports mention/always group activation, the paired implementation PR is closed unmerged, and contextual peeking remains a broad opt-in feature with privacy, logging, provider-exposure, and channel/plugin ownership decisions unresolved.

## What This Changes

Add an opt-in contextual group activation mode for Telegram and WhatsApp where a lightweight model decides when the bot should join or disengage without explicit mentions.

## Best Possible Solution

Keep mention/always semantics until maintainers approve an explicit contextual activation contract, then implement it in channel-owned runtime, schema, docs, logs, and tests with privacy-conscious defaults.
