## Summary

Keep open for maintainer product/security review: current main and the latest release still only support static mention/always activation plus ambient room-event behavior, while the requested LLM peeking/engaged mode would add new config, model calls over unmentioned group traffic, cross-channel state, and observability policy.

## What This Changes

The issue requests an opt-in contextual group activation mode that uses an LLM-driven peeking/engaged state machine for WhatsApp and Telegram groups without requiring every message to mention the bot.

## Best Possible Solution

Maintainers should first choose a privacy-aware product shape: reuse documented ambient room events where sufficient, decide whether the narrower attention-window proposal is the safer core path, and only add plugin/channel APIs or bundled channel behavior after the security and config contract is explicit.
