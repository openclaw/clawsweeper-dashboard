## Summary

Keep open for maintainer product/API/privacy review: current main and the latest release still expose only `mention`/`always` group activation, while the requested LLM peeking/engaged mode would add new cross-channel behavior, config, model-call, logging, and privacy surfaces.

## What This Changes

Add an opt-in contextual group activation mode for WhatsApp and Telegram that observes unmentioned group messages and uses a lightweight model-driven peeking/engaged state machine to decide when to reply.

## Best Possible Solution

Keep this as a maintainer-owned design item; if accepted, define an opt-in activation contract with privacy/cost documentation and channel-owner implementation, while using https://github.com/openclaw/openclaw/issues/47167 only for the raw observation prerequisite.
