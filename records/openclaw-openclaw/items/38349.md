## Summary

Keep open for maintainer product/API/security review: current main and v2026.5.20 still support mention-gated or always-on group activation plus ambient room events in some channels, but not the requested configurable LLM peeking/engaged state machine across Telegram and WhatsApp.

## What This Changes

The issue requests an opt-in LLM-driven contextual activation mode for Telegram and WhatsApp groups with peeking/engaged state, fallback models, scheduling, decision history, decision logs, and a CLI viewer.

## Best Possible Solution

Define a maintainer-approved contextual activation contract with explicit opt-in, privacy and retention controls, cost/rate limits, channel ownership, and separate passive-observation versus reply-authority semantics before implementation.

## Reproduction Assessment

Not applicable. This is a feature request, and current source, tests, docs, and the latest release show the existing contract intentionally supports mention-gated or always-on activation only.

Unclear. The peeking/engaged design may be useful, but the maintainable path is product and privacy/API design first; ambient room events and inbound_observed are partial adjacent paths, not the full requested feature.
