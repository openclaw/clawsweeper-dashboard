## Summary

Keep open: current main and the latest release still only support mention/always group activation, the paired implementation PR is closed unmerged, and contextual peeking remains a broad opt-in feature with privacy, logging, and channel/plugin ownership decisions unresolved.

## What This Changes

The issue requests a contextual activation mode for WhatsApp and Telegram groups that uses model-driven peeking/engaged decisions, fallbacks, schedules, and decision logs instead of requiring explicit mentions.

## Best Possible Solution

Keep mention/always semantics until maintainers approve an explicit opt-in contextual activation contract, then implement it in channel-owned runtime, schema, docs, logs, and tests with privacy-conscious defaults.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken existing behavior. Source, docs, tests, and latest-release inspection give a high-confidence negative check that contextual activation is not currently supported.

## Solution Assessment

Unclear; the requested design may be useful, but it is not yet the narrowest maintainable solution because privacy, logging, provider exposure, and Telegram/WhatsApp/plugin ownership are unresolved. A maintainer-approved opt-in contract should come before implementation.
