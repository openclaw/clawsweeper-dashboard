## Summary

Keep open. Current main still lacks the requested contextual activation mode: the shared activation contract, Gateway session patch/schema, Telegram config, WhatsApp config, and both bundled channel gates remain built around mention-vs-always behavior, and the linked implementation PR is closed unmerged.

## What This Changes

The issue requests opt-in contextual activation for Telegram and WhatsApp groups, using a lightweight LLM-driven peeking/engaged state machine instead of requiring explicit mentions.

## Best Possible Solution

The desired end state is either a first-class opt-in contextual activation contract across config, sessions/protocol, Telegram/WhatsApp gates, observability, CLI/docs, and tests with privacy-safe defaults, or a typed pre-gating plugin/channel activation API that lets this live outside core without dropping messages too early.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence source verification path: current main only accepts `mention`/`always`, and Telegram/WhatsApp skip unmentioned group messages before any LLM decision path.

## Solution Assessment

Unclear. A direct peeking/engaged state machine may be viable, but the best maintainable approach needs maintainer design review because it changes the trust boundary for unmentioned group traffic; a narrower pre-gating plugin/API seam may be safer if core should stay extension-agnostic.
