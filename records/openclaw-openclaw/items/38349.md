## Summary

Keep open: current main and v2026.6.6 still only provide static mention/always activation plus opt-in ambient room events, not an LLM-driven peeking/engaged group activation state machine. The request is useful context, but it needs maintainer product and security decisions before it is a safe implementation lane.

## What This Changes

Request contextual group activation for WhatsApp and Telegram groups using an LLM decision model, peeking/engaged state, new config, fallbacks, scheduling, decision history, and observability.

## Best Possible Solution

Pick a consent-aware group activation design first, likely by reconciling this broad LLM peeking proposal with the narrower attention-window and early inbound hook work, then implement one opt-in path with clear config, storage, retention, and channel tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection give a high-confidence check that current main lacks contextualActivation and only supports static mention/always activation plus always-on ambient room events.

## Solution Assessment

Unclear. The broad LLM peeking state machine may be useful, but it is not yet proven to be the best core fix compared with a narrower attention window, a passive inbound hook for plugins, or an extension of ambient room-event policy.
