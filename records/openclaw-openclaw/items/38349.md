## Summary

Keep open for maintainer product/API/privacy review: current main and the latest release still support mention-gated or always-on group activation, plus ambient room events for some channels, but not the requested configurable LLM peeking/engaged state machine across Telegram and WhatsApp.

## What This Changes

Request to add an opt-in contextual group activation mode for Telegram and WhatsApp that uses a lightweight decision model and peeking/engaged state to decide when the bot participates without explicit mentions.

## Best Possible Solution

Define a maintainer-approved contextual activation contract with explicit opt-in, privacy and retention controls, cost/rate limits, channel ownership, and separate passive-observation versus reply-authority semantics before implementation.

## Reproduction Assessment

Not applicable. This is a feature request, and current source, tests, docs, and the latest release show the existing contract intentionally supports mention-gated or always-on activation only.

## Solution Assessment

Unclear. The peeking/engaged design may be useful, but the maintainable path is product and privacy/API design first; ambient room events are a partial supported alternative, not the full requested feature.
