## Summary

Keep open for maintainer product/API/privacy review: current main and v2026.5.12 still expose only `mention` and `always` group activation, while the requested contextual peeking mode would add new cross-channel model calls, ambient group-message observation, state, config, logging, and privacy behavior.

## What This Changes

The issue requests an opt-in contextual group activation mode for Telegram and WhatsApp that uses a decision model and peeking/engaged state machine instead of explicit mentions.

## Best Possible Solution

Keep this as a maintainer-owned design item; if accepted, define an explicit opt-in activation contract with privacy/cost documentation and channel-owner implementation, using https://github.com/openclaw/openclaw/issues/47167 only for the raw observation prerequisite.
