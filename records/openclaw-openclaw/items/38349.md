## Summary

Keep open: current main has static `mention`/`always` activation and ambient room events, but it does not implement the requested contextual peeking/engaged state machine, cheap decision model, timeout/disengage logic, fallback chain, or WhatsApp coverage.

## What This Changes

Request to add an opt-in LLM-driven contextual activation mode for WhatsApp and Telegram group chats so the bot can observe unmentioned messages, decide when to engage, and disengage naturally.

## Best Possible Solution

Choose one consent-aware group activation design that either extends ambient room events/plugin ingress or adds a narrow opt-in contextual mode with redacted observability, then implement it consistently across supported channels.
