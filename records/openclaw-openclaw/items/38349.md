## Summary

Keep open: current main and v2026.6.6 still only support static `mention`/`always` group activation plus always-on ambient room events, not an LLM-driven peeking/engaged state machine. The request is useful but needs maintainer product and security direction before any core or plugin API work is safe.

## What This Changes

Requested contextual group activation for group chats where a lightweight model can decide when the agent observes, joins, remains engaged, or disengages without repeated explicit mentions.

## Best Possible Solution

Define a consent-aware, channel-neutral activation design first, likely by combining ambient room events and the early inbound hook/API discussion, then decide whether the LLM peeking policy belongs in a plugin/ClawHub package or a narrow core API.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs checks give high confidence that current main lacks the requested contextual activation mode and only has static activation plus ambient room-event behavior.
