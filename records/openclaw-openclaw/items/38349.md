## Summary

Keep open. Current main still only supports mention-vs-always group activation, the related implementation PR is closed unmerged, and the requested contextual mode needs maintainer product/API/security design before implementation.

## What This Changes

Request a contextual activation mode for Telegram and WhatsApp groups where a lightweight model decides when to engage without explicit mentions, with config, state management, logging, and a CLI viewer.

## Best Possible Solution

Define an explicit opt-in design either as first-class contextual activation across config, protocol, Telegram/WhatsApp gates, observability, CLI/docs, and tests, or as a typed pre-gating channel/plugin activation API with privacy-safe defaults.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence current-state proof: the shared contract and runtime gates only support `mention`/`always` and skip unmentioned Telegram/WhatsApp group messages before any LLM activation decision path.

## Solution Assessment

Unclear. The requested peeking/engaged state machine may be viable, but the best maintainable solution needs maintainer design review because it changes the trust boundary for unmentioned group traffic; a narrower pre-gating plugin/API seam may be safer.
