## Summary

Keep open for maintainer product/API/privacy review: current main and v2026.5.12 still only support mention/always group activation, while the requested LLM-driven peeking/engaged behavior would add new cross-channel config, model calls over ambient group traffic, state, scheduling, and observability. The paired implementation PR was closed unmerged, and the open raw-inbound hook issue is only a prerequisite, not the full activation design.

## What This Changes

Request contextual group activation for WhatsApp and Telegram using an LLM-driven peeking/engaged state machine instead of requiring explicit mentions for every group interaction.

## Best Possible Solution

Keep this as a maintainer-owned design item; if accepted, define an opt-in activation contract with privacy/cost documentation and coordinate with https://github.com/openclaw/openclaw/issues/47167 for the raw observation prerequisite and https://github.com/openclaw/openclaw/issues/74615 for related follow-up-listening semantics.
