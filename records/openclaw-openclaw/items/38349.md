## Summary

Keep open: current main and v2026.6.6 still do not implement the requested contextual LLM peeking/engaged activation mode, and the remaining work is a product/security decision rather than a safe cleanup close.

## What This Changes

The issue requests an opt-in contextual group activation mode that uses an LLM-driven peeking/engaged state machine for group chats without requiring every message to mention the bot.

## Best Possible Solution

Maintainers should decide the product boundary first: either sponsor a narrow plugin/API path for passive observation plus external contextual activation, or design one consent-aware core group activation mechanism that reuses ambient room-event semantics instead of adding per-channel duplicate gating logic.

## Reproduction Assessment

Not applicable. This is a feature request, and source/docs inspection gives a high-confidence check that contextual activation is absent while static `mention`/`always` and ambient room-event modes exist.

## Solution Assessment

Unclear. The broad LLM peeking state machine is not proven to be the best core solution; maintainers should compare it with ambient room events, the narrower attention-window request, and the public skipped-message hook proposal.
