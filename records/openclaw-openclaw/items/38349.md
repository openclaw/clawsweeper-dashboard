## Summary

Keep open for maintainer product/API/privacy review. Current main and v2026.5.19 still expose only mention/always group activation; the newer ambient room-event path partially overlaps the “listen quietly” goal for Telegram/Discord/Slack, but it is not the requested Telegram-plus-WhatsApp contextual peeking/engaged decision-model state machine.

## What This Changes

The issue requests a new contextual group activation mode for Telegram and WhatsApp that lets a model decide when the bot should observe, engage, disengage, and log decisions without explicit mentions.

## Best Possible Solution

Decide whether contextual group participation should be an extension of the existing ambient room-event/message-tool contract or a new activation mode, then specify explicit opt-in config, privacy/retention rules, channel support, model routing, logs, and reply authority before implementation.

## Reproduction Assessment

Not applicable. This is a feature request, and source/docs/release inspection shows the supported contracts are mention/always activation plus ambient room events for a subset of channels, not a contextual peeking state machine.

## Solution Assessment

Unclear. The peeking/engaged direction may be useful, but current main already has an ambient room-event pattern for quiet participation, so the maintainable path is a privacy-reviewed product/API decision before code.
