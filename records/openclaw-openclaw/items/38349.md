## Summary

Keep open for maintainer product/API/privacy review: current main and v2026.5.19 only support static mention-gated or always-on group activation, while this asks for a new multi-channel LLM pre-gate state machine with scheduling, model routing, logs, and reply-authority semantics.

## What This Changes

Request adds opt-in contextual group activation for WhatsApp and Telegram using an LLM-driven peeking/engaged state machine, fallback decision models, schedules, decision logs, and a CLI viewer.

## Best Possible Solution

Define a maintainer-approved, default-off contextual activation contract with privacy/retention controls, cost/rate limits, per-channel ownership, explicit passive-observation versus reply-authority semantics, schemas, docs, and tests before implementation.

## Solution Assessment
