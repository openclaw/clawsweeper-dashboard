## Summary

Current main still does not implement contextual group activation. The shipped contract remains `mention`/`always`, there is no `contextualActivation` config/protocol/logging surface, and Telegram/WhatsApp still skip or buffer unmentioned group messages before any model-driven participation decision. The linked implementation PR is closed and unmerged, so this remains a real product/API design item.

## What This Changes

Add opt-in contextual group activation for Telegram and WhatsApp so a lightweight model can decide when the bot should peek, engage, continue, or disengage in group chats without explicit mentions.

## Best Possible Solution

Define a first-class opt-in contextual activation capability, or a typed pre-gating plugin/channel activation seam, with explicit privacy-safe defaults, model/fallback policy, fail-closed behavior, observability, and Telegram/WhatsApp coverage before accepting implementation work.
