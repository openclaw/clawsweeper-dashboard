## Summary

Keep open: current main only supports mention/always group activation, the linked implementation PR is closed unmerged, and the requested LLM-driven contextual mode is a broad opt-in feature that needs maintainer product and security design rather than cleanup closure.

## What This Changes

Request adds opt-in LLM-driven contextual activation for WhatsApp and Telegram groups with a peeking/engaged state machine, fallback models, scheduling, decision logs, and CLI log viewing.

## Best Possible Solution

Decide whether contextual activation belongs as channel-owned behavior, a new plugin/API seam, or a ClawHub-style plugin, then add opt-in schema/docs and focused WhatsApp/Telegram regression coverage without weakening existing allowlist and mention defaults.

## Reproduction Assessment

## Solution Assessment

Unclear; the requested direction is plausible, but the issue does not settle the product/security boundary for LLM peeking in group chats. A safer path is a maintainer-approved design that preserves group allowlists first and adds an explicit opt-in activation seam before implementation.
