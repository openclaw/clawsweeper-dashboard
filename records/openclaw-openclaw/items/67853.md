## What This Changes

The branch adds a direct memory-wiki fallback for memory-core tools, exports a memory-wiki plugin API subpath, changes plugin tool registry loading, adjusts related memory-wiki/QMD behavior, and adds targeted regression checks.

## Best Possible Solution

Rebase onto current main, preserve compatible channel/active/standalone registry reuse before in-flight degradation, keep the wiki fallback behind the public memory-wiki plugin API, and add redacted runtime proof.

## Reproduction Assessment

Yes, at source level. Current main still depends on registered memory corpus supplements for shared wiki recall, and the PR head visibly returns no plugin registry during an in-flight load before trying compatible active registry reuse.

## Solution Assessment

No. The direct wiki fallback direction is useful, but the resolver change should reuse a compatible loaded registry before degrading on in-flight loads, and the new local gate should use the repo test wrapper.
