## Summary

Keep open: current main still returns only count summaries in primary MCP content for the reported channel tools while placing the useful arrays in `structuredContent`, which matches the reported failure mode and conflicts with MCP compatibility guidance for structured results.

## What This Changes

The issue asks for `messages_read` and `conversations_list` to expose their returned message and conversation objects in primary MCP `content`, not only in `structuredContent`.

## Best Possible Solution

Return a model-visible serialized JSON payload in `content` for these channel MCP data tools while preserving `structuredContent` for structured clients and adding regression coverage for the primary content field.

## Reproduction Assessment

Yes from source inspection: calling `conversations_list` or `messages_read` on current main will put only `conversations: N` or `messages: N` in `content` and the actual arrays in `structuredContent`. I did not run a live MCP client, so this is source-reproducible rather than locally executed.

## Solution Assessment

Yes: the narrow maintainable fix is in the channel MCP result formatting, not in Claude-specific client behavior. Keep the existing structured payloads and add text JSON content for compatibility.
