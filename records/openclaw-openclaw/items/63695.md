## Summary

Keep open. Current main still has the Telegram partial-preview bug this PR targets, but the PR head still has two concrete correctness defects in archived-preview cleanup and active-preview retained lifecycle handling, and the PR API reports the branch as dirty against current main.

## What This Changes

The PR changes Telegram lane delivery to score active and archived answer previews by shared-prefix affinity, clean up stale archived previews, and adds Telegram lane-delivery and dispatch regression tests.

## Best Possible Solution

Keep the affinity-based Telegram-only approach, rebase it onto current main, ensure every consumed archived preview is either deleted or remains tracked for final cleanup, and finalize retained active previews through the same completed lifecycle as edited active previews.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: partial streaming can enqueue multiple archived answer previews during superseded preview and assistant-message rotations, and final delivery consumes the oldest archived preview with `shift()` instead of choosing the active or best matching preview.

## Solution Assessment

No as-is. The proposed affinity selection is the narrow maintainable direction, but the branch needs the archived cleanup and active retained lifecycle fixes before it is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Delete every consumed superseded preview:** `extensions/telegram/src/lane-delivery-text-deliverer.ts:602-603`
  - body: When an archived preview after an unrelated earlier stale preview wins, this `splice` drops all earlier previews from `archivedAnswerPreviews`, but only `selection.staleArchivedPreviews` are deleted. Any earlier `deleteIfUnused: true` preview with score 0 is no longer visible to final cleanup and can remain in Telegram indefinitely, so consumed superseded previews need deletion or must stay tracked.
  - confidence: 0.88
- **[P2] Mark retained active finals complete:** `extensions/telegram/src/lane-delivery-text-deliverer.ts:581-584`
  - body: In the active-selection path, a retained final is treated as delivered but the lane remains `transient`. A later final attempt resets the retain flag for transient lanes, so cleanup can clear or reuse the preview that was intentionally retained; this branch should mark the active preview complete like the edited and regressive-skipped paths.
  - confidence: 0.84
