## Summary

Keep open. The PR targets a real Telegram partial-streaming preview bug on current main, but the head still has narrow preview lifecycle defects that can leak consumed previews and leave retained active finals in the transient state; it also needs the required changelog entry.

## What This Changes

The PR changes Telegram partial final delivery to score active and archived answer previews by shared-prefix affinity, clean up stale archived previews, and add Telegram lane-delivery and dispatch regression tests.

## Best Possible Solution

Keep the Telegram-only affinity selection, repair the consumed-preview cleanup and retained-active lifecycle handling, add a one-line Unreleased changelog entry, then re-run the targeted Telegram tests before merge.

## Reproduction Assessment

Yes. Source inspection of current main shows partial streaming can queue multiple archived answer previews, while final delivery consumes the oldest archived preview with `shift()` instead of selecting the active or best matching preview.

## Solution Assessment

No as-is. The affinity-based direction is the narrow maintainable fix, but this PR must preserve or delete every consumed archived preview and mark retained active finals complete before it is safe to merge.
