## Summary

Keep this PR open. Current main still finalizes Telegram archived answer previews FIFO, so the branch addresses a real remaining partial-preview bug, but the proposed patch should not merge as-is because it can strand consumed stale previews and can leave a retained active final in the transient lifecycle.

## What This Changes

The branch changes Telegram lane delivery to score active and archived answer previews by shared-prefix affinity, clean up stale archived previews, and adds lane-delivery and dispatch regression tests.

## Best Possible Solution

Rebase the PR onto current main, keep the affinity-based selection approach, ensure every consumed superseded preview is either deleted or remains tracked for final cleanup, mark retained active finals complete, and preserve the added lane-delivery plus Telegram dispatch regressions.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: multiple archived answer previews are possible during partial streaming rotations, and final delivery consumes the oldest archived preview with `shift()` instead of choosing the active or best matching preview.

## Solution Assessment

No as-is. Affinity selection is the right narrow direction for this Telegram-only bug, but the current patch needs the archived cleanup and active retained lifecycle fixes before it is the maintainable solution.
