## Summary

Keep open. The implementation looks narrowly scoped, but it changes the default memory-flush policy for large-window models and still needs real behavior proof; I also found a docs mismatch that should be fixed before merge.

## What This Changes

The PR makes memory-core derive the default memory-flush soft threshold from the resolved model context window, plumbs that value through the memory flush resolver, and adds focused unit coverage.

## Best Possible Solution

Land an owner-approved dynamic default with docs/help updated and redacted real after-fix proof from a large-window OpenClaw memory-flush run, while preserving explicit operator overrides.

## Reproduction Assessment

No high-confidence real reproduction is present. Source inspection confirms current main uses a fixed 4000-token default and the flush gate depends on that value, but I did not reproduce a long-lived 1M-token session reaching the bad state.

## Solution Assessment

Unclear as a merge-ready solution. The plumbing is additive and narrow, but the adaptive 70%/900000 default is a policy change that should be documented and backed by real behavior proof before merge.
