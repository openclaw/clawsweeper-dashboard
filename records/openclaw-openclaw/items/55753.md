## Summary

Keep open: the PR addresses a real restart-clobbering path, but the proposed one-time copy changes the Kubernetes ConfigMap/PVC ownership contract and can break the documented redeploy update flow.

## What This Changes

The PR adds `set -eu` and guards the Kubernetes init-container copies of `openclaw.json` and `AGENTS.md` so they only run when the PVC-backed destination files are missing.

## Best Possible Solution

Make the Kubernetes config ownership contract explicit, then implement that contract consistently: either seed once with documented reset/resync guidance, or keep ConfigMap redeploy updates with backup/conflict handling.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main overwrites the PVC-backed files on each init-container run, while the PR guard would skip later ConfigMap copies once those files exist.

## Solution Assessment

No. The file-exists guard is a narrow preservation mechanism, but it is not the best complete fix until maintainers choose and document whether Kubernetes config is ConfigMap-declarative or PVC-owned runtime state.
