## Summary

Keep this PR open, but it is not merge-ready: it fixes a source-reproducible overwrite problem while also changing Kubernetes config ownership, and the branch still lacks real rollout proof plus maintainer-approved redeploy/reset guidance.

## What This Changes

The PR changes the Kubernetes Deployment init container to use `set -eu`, seed `openclaw.json` and workspace `AGENTS.md` only when PVC files are missing, and comment that the PVC becomes the source of truth.

## Best Possible Solution

Define the Kubernetes config ownership contract, then either document seed-once PVC ownership with explicit reset/resync steps and rollout proof, or implement safe reconciliation that preserves runtime edits without silently ignoring ConfigMap updates.

## Reproduction Assessment

Yes for source reproduction: current `main` unconditionally copies ConfigMap files into the PVC on each init-container run, and the PR would skip those copies once files exist. I did not establish a live Kubernetes rollout reproduction.

## Solution Assessment

No, not yet. The seed-once guard is maintainable only if maintainers choose PVC ownership and the branch adds docs/reset guidance plus real rollout proof, or else it needs a safer reconciliation path.
