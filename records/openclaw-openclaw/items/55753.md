## Summary

Keep this PR open: it addresses a real source-visible overwrite path, but the current patch changes Kubernetes ConfigMap redeploy semantics and still lacks real rollout proof.

## What This Changes

The PR changes the Kubernetes Deployment init container so `openclaw.json` and workspace `AGENTS.md` are seeded from the ConfigMap only when the PVC-backed files are missing, with `set -eu` added to the shell script.

## Best Possible Solution

Define the Kubernetes config ownership contract, then either document seed-once PVC ownership with explicit reset/resync steps and rollout proof, or implement safe reconciliation that preserves runtime edits without silently ignoring ConfigMap updates.

## Reproduction Assessment

Yes, source-reproducible: current `main` unconditionally copies ConfigMap files into the PVC on each init-container run, and the PR would skip those copies once files exist. I did not establish a live Kubernetes rollout reproduction.

## Solution Assessment

No, not yet. The seed-once guard is reasonable only if maintainers choose PVC ownership; the branch currently conflicts with documented redeploy behavior and needs docs/reset guidance plus real rollout proof or a safer reconciliation path.
