## Summary

Keep open: the overwrite problem is source-reproducible, but the proposed one-time copy changes the documented Kubernetes ConfigMap redeploy contract and can leave security-sensitive config stale.

## What This Changes

The PR changes the Kubernetes init container to use `set -eu` and copy ConfigMap-backed `openclaw.json` and `AGENTS.md` into the PVC only when destination files are absent.

## Best Possible Solution

Define Kubernetes config ownership, then implement either documented seed-once PVC ownership with resync/reset guidance or ConfigMap reconciliation that preserves local edits with backup or conflict handling.

## Reproduction Assessment

Yes, source-reproducible: current `main` unconditionally copies ConfigMap files into the PVC on each init-container run, and the PR would skip those copies once files exist. I did not establish a real Kubernetes rollout reproduction.

## Solution Assessment

No. The guard preserves PVC edits, but it is not the best complete solution until maintainers decide whether Kubernetes config is declarative ConfigMap state or PVC-owned runtime state.
