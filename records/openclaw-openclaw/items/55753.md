## Summary

Keep open: the overwrite path is real, but the branch changes Kubernetes config ownership without aligned docs, reset/resync guidance, or redacted real rollout proof, so it is not merge-ready.

## What This Changes

The PR changes the Kubernetes deployment init container so `openclaw.json` and `AGENTS.md` are seeded from the ConfigMap only when the PVC files are missing.

## Best Possible Solution

Define the Kubernetes config ownership contract, then either document seed-once PVC ownership with explicit reset/resync steps or implement safe reconciliation that preserves user edits without silently ignoring ConfigMap updates.

## Reproduction Assessment

Yes, source-reproducible: current main copies ConfigMap files over the PVC files on each init-container run, and the PR would skip those copies once the files exist. I did not establish a live Kubernetes rollout reproduction.

## Solution Assessment

No, not yet. The guard is a plausible seed-once implementation only if maintainers choose PVC ownership, but the complete fix needs docs, reset/resync guidance, and real rollout proof.
