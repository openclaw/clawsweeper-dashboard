## Summary

Keep open: the PR addresses a plausible Kubernetes config clobbering problem, but it is not merge-ready because it silently changes the documented ConfigMap redeploy contract and still lacks real Kubernetes rollout proof.

## What This Changes

The PR changes the Kubernetes deployment init container to use `set -eu` and seed `openclaw.json` and `AGENTS.md` from the ConfigMap only when the PVC files are missing.

## Best Possible Solution

Define Kubernetes config ownership, then implement either documented seed-once PVC ownership with reset/resync guidance or safe ConfigMap reconciliation that preserves local edits with backup or conflict handling.

## Reproduction Assessment

Yes, source-reproducible: current `main` unconditionally copies ConfigMap files into the PVC on each init-container run, and the PR would skip those copies once files exist. I did not establish a real Kubernetes rollout reproduction.

## Solution Assessment

No. The guard preserves PVC edits, but it is not the best complete solution until maintainers decide whether Kubernetes config is declarative ConfigMap state or PVC-owned runtime state.
