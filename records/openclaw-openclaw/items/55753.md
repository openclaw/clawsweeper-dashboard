## Summary

Keep open: the source-level data-loss concern is plausible, but the patch changes the Kubernetes config ownership contract by making ConfigMap copies one-time-only, which conflicts with the documented edit-ConfigMap-and-redeploy workflow and needs maintainer direction plus real behavior proof.

## What This Changes

The PR changes the Kubernetes init container script to add `set -eu` and copy `openclaw.json` and `AGENTS.md` from the ConfigMap into the PVC only when the destination files are missing.

## Best Possible Solution

Define the Kubernetes config ownership contract explicitly, then implement and document either seed-once behavior with reset/resync guidance or ConfigMap reconciliation with backup/conflict handling.

## Reproduction Assessment

Yes, source-reproducible: current `main` unconditionally copies ConfigMap files into the PVC on every init-container run, and the PR would skip those copies once the files exist. I did not establish a real Kubernetes rollout reproduction.

## Solution Assessment

No. The one-time guard addresses PVC preservation, but it is not the best complete solution until maintainers choose whether Kubernetes config is declarative ConfigMap state or PVC-owned runtime state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve ConfigMap updates on redeploy:** `scripts/k8s/manifests/deployment.yaml:34-39`
  - body: The new existence guards make ConfigMap copies one-time-only. After the first pod start, later `deploy.sh` runs still apply the ConfigMap and restart the deployment, but updated `openclaw.json` or `AGENTS.md` will not propagate into the PVC, breaking the documented redeploy/update workflow and leaving stale gateway settings in production.
  - confidence: 0.9

## Security Review

Status: needs_attention

  - body: Once `/home/node/.openclaw/openclaw.json` exists, the new guard prevents later ConfigMap edits from reaching the running config file; that file carries gateway bind, auth, and Control UI settings.
