## Summary

Keep open: the PR targets a real skills watcher FD-exhaustion regression and includes useful terminal proof, but the proposed `depth: 1` bound is too shallow for documented grouped skill folders and would regress hot-refresh for `skills/<group>/<skill>/SKILL.md`.

## What This Changes

The PR adds `depth: 1` to the skills Chokidar watcher, asserts that watcher option in the unit test, and adds a changelog entry for the linked EMFILE fix.

## Best Possible Solution

Bound traversal to the shallowest depth that still matches the documented loader contract, including grouped skill folders, and add regression coverage for both grouped `SKILL.md` refresh and deep non-skill tree avoidance.

## Reproduction Assessment

Yes for the review finding by source: a grouped skill at `skills/group/name/SKILL.md` is supported by the loader and docs, while Chokidar `depth: 1` stops before reading that nested skill directory. I did not run a live watcher because this was a read-only review.

## Solution Assessment

No: `depth: 1` is not the best fix because it is narrower than the current skill discovery contract. A safer fix is to use a bound that still watches grouped skills, or derive watcher targets from the same discovery rules, with focused regression coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
