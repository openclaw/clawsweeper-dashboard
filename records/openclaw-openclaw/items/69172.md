## Summary

Keep open: current main still lacks the intended Gemini proxy override behavior, and this PR is not mergeable as-is because the branch weakens private-network request policy and carries unrelated runtime/script regressions.

## What This Changes

The PR branch adds Google/Gemini base URL and API type resolution for web search, embeddings, image generation, and media understanding, plus tests and unrelated changes to pnpm runner, memory runtime exports, plugin metadata, and provider model typings.

## Best Possible Solution

Land a narrower Google-plugin-focused change that centralizes Gemini baseUrl/apiType resolution, preserves explicit private-network policy, keeps native Google suffix normalization from #66445, maps OpenAI-compatible payloads per capability, and leaves unrelated pnpm/memory/runtime surfaces untouched.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the missing behavior: web search is hardcoded to the native Gemini URL, embeddings ignore the endpoint env aliases, and current tests explicitly reject non-Google Gemini base URLs without opt-in.

## Solution Assessment

No. The requested direction is valid, but this PR is not the best implementation because it weakens network policy, regresses current shipped Google URL normalization, and includes unrelated runtime/script changes; a split or replacement PR is safer.
