## Summary

## What This Changes

Bound memory-core dreaming promotion writes so `MEMORY.md` cannot grow past the bootstrap budget and wedge Gateway sessions.

## Best Possible Solution

Review and land PR #74088 or an equivalent memory-core promotion budget that compacts auto-promoted sections while preserving user-authored memory content.

## Solution Assessment

Yes on boundary: the durable fix belongs in memory-core's promotion writer because bootstrap trimming only caps injected prompt context. The open draft PR follows that shape but is not merged yet.
