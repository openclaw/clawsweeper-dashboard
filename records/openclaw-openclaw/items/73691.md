## Summary

Keep open: current `main` still has the memory-core promotion path appending to `MEMORY.md` without a durable size budget, and draft PR #74088 is the explicit closing implementation candidate rather than a merged fix.

## What This Changes

The issue requests bounding or compacting `MEMORY.md` so dreaming promotions cannot grow the bootstrap memory file until Gateway sessions freeze.

## Best Possible Solution

Review and land PR #74088 or an equivalent memory-core fix that enforces a pre-write budget or compaction policy while preserving user-authored `MEMORY.md` content.

## Reproduction Assessment

Yes for the root growth path by source inspection: repeated successful `applyShortTermPromotions` calls with distinct candidates append new sections without a size guard. The full Gateway freeze still needs lock-timeout logs or an end-to-end reproduction.

## Solution Assessment

Yes on boundary: the durable fix belongs in memory-core's promotion writer because bootstrap truncation only caps injected context and does not bound disk growth.
