## Summary

Keep open: current main still has an append-only memory-core promotion writer with no durable MEMORY.md size budget, while bootstrap only trims injected context. PR #74088 is an open, unmerged implementation candidate that uses closing syntax for this issue, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that memory-core dreaming promotions can grow MEMORY.md beyond bootstrap limits, leading to silent truncation and reported Gateway hangs after extended use.

## Best Possible Solution

Land PR #74088 or an equivalent memory-core change that enforces a durable pre-write MEMORY.md budget, preserves user-authored content, compacts or archives only promotion-owned sections, and adds regression coverage for repeated promotion runs near and over the bootstrap cap.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the unbounded file growth: repeated successful applyShortTermPromotions calls with new candidates append new promotion sections to existing MEMORY.md without a size guard. The exact Gateway freeze mechanism is unclear because the issue does not include lock-timeout logs or a focused gateway reproduction.

## Solution Assessment

Is this the best way to solve the issue? Yes on boundary: the durable guard belongs in memory-core's promotion writer, not only in bootstrap truncation. PR #74088 is the active candidate to review because it targets that writer with compaction while preserving user-authored content.
