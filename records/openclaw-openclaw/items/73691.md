## Summary

Keep open: current main still has an unbounded memory-core promotion write to `MEMORY.md`, and open draft PR #74088 is a closing implementation candidate rather than a merged fix.

## What This Changes

The issue requests bounding `MEMORY.md` growth from memory-core dreaming promotions so bootstrap context stays within the per-file limit and Gateway sessions do not freeze.

## Best Possible Solution

Review and land PR #74088 or an equivalent memory-core promotion change that caps or compacts auto-promoted sections while preserving user-authored memory content.

## Reproduction Assessment

Yes for the root growth bug by source inspection: repeated successful `applyShortTermPromotions` calls with distinct candidates append new promotion sections without a size guard. The full Gateway freeze still lacks lock-timeout logs or an end-to-end current-main reproduction.

## Solution Assessment

Yes on boundary: the durable fix belongs in memory-core's promotion writer because bootstrap trimming only caps injected prompt context and does not bound disk growth. The open draft PR follows that shape with compaction, but it has not merged.
