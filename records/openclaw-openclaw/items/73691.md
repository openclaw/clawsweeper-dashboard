## Summary

Keep open: current main still has an unbounded memory-core promotion write to `MEMORY.md`, and PR #74088 is an open draft closing implementation candidate rather than a merged fix.

## What This Changes

The issue requests bounding memory-core dreaming promotions so `MEMORY.md` cannot grow past bootstrap-safe limits and wedge Gateway sessions.

## Best Possible Solution

Land PR #74088 or an equivalent memory-core promotion change that enforces a durable pre-write budget or compaction policy while preserving user-authored `MEMORY.md` content.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the root unbounded growth path by source inspection: repeated successful `applyShortTermPromotions` calls with distinct candidates append new sections without a size guard; the full Gateway freeze still needs lock-timeout logs or an end-to-end reproduction.

## Solution Assessment

Is this the best way to solve the issue? Yes on boundary: the durable fix belongs in memory-core's promotion writer because bootstrap truncation only caps injected context and does not bound disk growth.
