## Summary

Keep open: current main still has the unbounded memory-core promotion write, and the linked draft PR is an open implementation candidate rather than a merged fix.

## What This Changes

The issue requests a bounded or compacting `MEMORY.md` promotion path so memory-core dreaming cannot grow the bootstrap memory file without limit.

## Best Possible Solution

Land PR #74088 or an equivalent memory-core promotion change that enforces a durable pre-write budget while preserving user-authored `MEMORY.md` content.

## Reproduction Assessment

Yes for unbounded growth: repeated successful `applyShortTermPromotions` calls with distinct candidates append new promotion sections without a durable size guard. The full Gateway freeze remains only partially reproducible without the reported lock-timeout logs.

## Solution Assessment

Yes on boundary: the fix belongs in memory-core's promotion writer because bootstrap truncation only caps injected context and does not bound disk growth. The currently linked PR direction is appropriate, but it is still draft and unmerged.
