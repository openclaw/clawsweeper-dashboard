## Summary

Keep open. Current main rules out the direct compaction-state flip theory, but the normal agent tool set still exposes `tts` with guidance that describes automatic audio delivery and the silent reply token without limiting calls to explicit user audio intent. That leaves a plausible current-main path for unsolicited dashboard audio replies and a narrow behavior-hardening fix.

## What This Changes

The issue asks OpenClaw to keep dashboard replies as text unless the user explicitly requests TTS/audio, even after long-session compaction.

## Best Possible Solution

Make the TTS tool and matching prompt/docs/tests explicit that agent-initiated speech requires user audio intent by default, while preserving configured Auto-TTS modes, deliberate TTS commands, directives, and existing media delivery semantics.

## Reproduction Assessment

No. The report has concrete steps, but the exact provider/routing/long-compacted dashboard state was not reproduced here; static review shows compaction does not flip `ttsAuto` and still leaves a plausible `tts` tool path for unsolicited audio.

## Solution Assessment

Yes. The narrow maintainable fix is to harden agent-facing `tts` tool guidance and regression tests rather than changing compaction state or disabling existing TTS workflows.
