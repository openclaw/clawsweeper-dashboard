## Summary

Current main and the shipped package already implement the no-profile path the issue asks for: `google-vertex` ADC auth evidence is resolved directly at runtime, so an auth profile entry is not required.

## What This Changes

The issue reports that `google-vertex` ADC env evidence makes `models list` show configured, but no auth profile is written and runtime requests fail with `No API key found`.

## Best Possible Solution

Keep the shipped direct env-auth evidence fallback for `google-vertex`; env-only ADC should not require writing a portable auth profile.

## Reproduction Assessment

No. Source and package inspection show current main and the v2026.5.7 npm package resolve `google-vertex` ADC auth evidence after an empty profile lookup instead of throwing the reported provider-level error.

## Solution Assessment

Yes. The existing direct runtime fallback is the narrowest maintainable solution because it avoids persisting env-derived local ADC markers into portable profile storage.
