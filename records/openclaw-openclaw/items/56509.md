## Summary

Keep open: current main still has the completed one-shot cron gap, but this PR is not merge-ready because it bundles unrelated changes and introduces a blocking transcript repair regression.

## What This Changes

The branch enables `skipAtIfAlreadyRan` for cron due collection and also bundles WhatsApp append handling, transcript repair, startup secret retry, and exec environment key filtering changes.

## Best Possible Solution

Land or replace this with a cron-only fix that applies the terminal one-shot guard to all non-forced due scans while preserving transient-error retry eligibility, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: persist an enabled `at` job with terminal `lastStatus`, no `runningAtMs`, and a past `nextRunAtMs`, then let the normal timer or non-forced due path evaluate it without `skipAtIfAlreadyRan`.

## Solution Assessment

No. Passing `skipAtIfAlreadyRan` through the cron due scans is the right mechanism, but this PR is not the best vehicle because it is stale, bundled, and has a blocking transcript repair regression.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Rewrite sanitized tool-result IDs before pairing:** `src/agents/session-transcript-repair.ts:439-443`
  - body: When `rawId` only matches through the sanitized-ID fallback, the code stores the unchanged `toolResult` under the original ID. That can emit an assistant tool call with the original ID next to a tool result whose `toolCallId`/`toolUseId` is still sanitized, which strict replay paths can reject; rewrite the result ID before storing it.
  - confidence: 0.91
- **[P3] Report total startup attempts consistently:** `src/gateway/server.impl.ts:470-471`
  - body: The retry loop performs the initial attempt plus three retries, but the warning logs `Attempt 3/3 failed, retrying` before the fourth total attempt. That makes startup diagnostics misleading; log total attempts as four or label the field as retry count.
  - confidence: 0.78
