Full review comments:

- **[P1] Rewrite the paired tool result ID:** `src/agents/session-transcript-repair.ts:446-448`
  - body: When `rawId` only matches through the sanitized-ID fallback, `id` becomes the original tool-call ID but `normalizedToolResult` still carries the sanitized `toolCallId`/`toolUseId`. That can emit an assistant tool call with one ID followed by a paired result with a different ID, which strict replay paths reject; rewrite the result ID fields to the matched `id` before storing it.
  - confidence: 0.88
- **[P2] Keep append read receipts when skipping history:** `extensions/whatsapp/src/inbound/monitor.ts:485-486`
  - body: This early `continue` bypasses `normalizeInboundMessage` and `maybeMarkInboundAsRead` for stale append messages. The existing path marked replayed history as read while suppressing auto-reply, so the fix should suppress pairing without dropping the read-receipt behavior.
  - confidence: 0.82
- **[P3] Clarify the startup retry count:** `src/gateway/server.impl.ts:469-471`
  - body: The retry loop logs `Attempt 3/3 failed, retrying` and then performs a fourth attempt, so operators see another try after the denominator appears exhausted. Log total attempts or label the message as retry count.
  - confidence: 0.78
