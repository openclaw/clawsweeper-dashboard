## Summary

Keep this PR open for maintainer review, but do not merge it as written: it targets the real linked Telegram group `/status` no-reply regression while broadening group control-command authorization in an auth-sensitive path.

## What This Changes

The PR changes Telegram inbound text-command gating to skip access-group enforcement when the effective Telegram command allow-list has no entries.

## Best Possible Solution

Preserve the fail-closed sender/owner command policy and add a narrow authorization signal for explicitly matched configured groups instead of treating an empty sender allow-list as global permission.

## Reproduction Assessment

Yes. The source-level reproduction is a per-group `groupPolicy: "open"` setup with no effective sender allow-list and `/status` from a random sender; current tests expect no reply, while this patch would authorize the command gate.

## Solution Assessment

No. The patch conflates a dedicated allowed group with any empty sender allow-list; the safer fix should carry the explicit group-match fact into the gate or native-command path without weakening open-group command authorization.
