## Summary

Keep this PR open, but do not merge it as written. The fix direction is valid for the linked Telegram group /status regression, but the patch uses an empty effective allowlist as the switch and therefore also authorizes text commands in per-group open groups that current tests and docs keep unauthorized.

## What This Changes

The PR changes Telegram inbound body command gating so access-group command enforcement is skipped whenever the effective command allowlist has no entries.

## Best Possible Solution

Keep the targeted Telegram fix, but narrow it so text-command gating knows when group policy explicitly authorized the chat without a sender allowlist. Preserve per-group open command denial, `commands.allowFrom`, owner/native command checks, and add regression coverage for explicit groups, wildcard groups, open groups, and configured sender allowlists.

## Reproduction Assessment

Yes. The current-main static path reproduces the linked bug: an explicitly configured group can pass group policy with no sender allowlist, then `resolveControlCommandGate` has no configured authorizer and drops `/status`; the PR regression is covered by the existing per-group open `/status` denial test.

## Solution Assessment

No. The PR fixes the symptom by disabling the gate for all empty allowlists, but empty allowlists also occur in open-group configurations where current tests and docs require text commands to remain unauthorized; a safer fix should pass an explicit group-policy authorization fact or add a dedicated explicit-group authorizer.
