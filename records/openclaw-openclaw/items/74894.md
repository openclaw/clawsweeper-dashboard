## Summary

Keep open: the PR is a focused attempt to address the linked Telegram group `/status` no-reply bug, but the diff widens group command authorization when the effective allow-list is empty and still lacks real Telegram proof.

## What This Changes

This PR changes Telegram inbound command gating to skip access-group enforcement when the effective command allow-list has no entries.

## Best Possible Solution

Replace the broad empty-allow-list bypass with a narrow Telegram command authorization fix that preserves explicit group/topic/owner policy and is proven in real Telegram.

## Reproduction Assessment

Yes, source-level rather than live. Current tests show per-group `groupPolicy: "open"` lets normal text through but drops `/status` from an unallowlisted sender, while this PR would authorize that path without distinguishing configured groups.

## Solution Assessment

No. The patch bypasses the gate based solely on an empty effective allow-list; the safer approach is to carry a narrow configured-group authorization fact while preserving sender and owner command checks.
