## Summary

Keep open, but do not merge as-is: the diff targets a real Telegram group command no-reply bug, but it broadens command authorization for empty group allow-lists and lacks after-fix real Telegram proof.

## What This Changes

The PR changes Telegram inbound command gating so access-group enforcement is skipped when the effective command allow-list has no entries.

## Best Possible Solution

Replace the broad empty-allow-list bypass with a narrow Telegram command authorization path that recognizes explicitly configured allowed groups while preserving sender and owner command authorization.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: per-group `groupPolicy: "open"`, no effective sender allow-list, and `/status` from a random sender; current coverage expects no reply, while this PR would authorize the early command gate.

## Solution Assessment

No. The patch treats every empty effective group command allow-list as authorized; the safer fix is to carry an explicit configured-group authorization fact into the command gate without weakening open-group command auth.
