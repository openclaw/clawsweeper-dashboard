## Summary

Keep open. The PR is a focused attempt at the linked Telegram bug, but the diff weakens group slash-command authorization and the external PR still lacks after-fix real Telegram proof.

## What This Changes

The PR changes Telegram inbound command gating so group command access-group enforcement is skipped when the effective command allow-list is empty.

## Best Possible Solution

Implement a focused Telegram command-authorization fix that distinguishes explicitly configured groups from open groups, preserves sender and owner command boundaries, and includes regression coverage plus real Telegram proof.

## Reproduction Assessment

Yes, source-level. Current tests configure a per-group `groupPolicy: "open"` group with no effective command group allow-list and assert normal text replies while `/status` from an unallowlisted sender does not.

## Solution Assessment

No. The PR bypasses command authorization whenever no effective group command allow-list entries exist; the safer solution is to carry an explicit configured-group authorization fact without authorizing arbitrary open-group senders.
