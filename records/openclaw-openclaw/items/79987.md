## Summary

Keep open: the bug is valid and the branch targets the right installer path, but the current patch does not build, lacks real behavior proof, and adds an early success path that bypasses plugin validation and security scanning.

## What This Changes

The branch adds a managed npm plugin version check and a unit test so install mode skips npm install when the on-disk package version is equal to or newer than the requested version.

## Best Possible Solution

Revise the skip path to reuse the existing installed-package validation/security-scan result, keep the linked issue open until merge, and add redacted real wizard or CLI proof for the fixed behavior.

## Reproduction Assessment

Yes, from source inspection: current main rejects an existing managed npm package directory in install mode before npm install, matching the linked Docker wizard report. I did not run a live wizard in this read-only review.

## Solution Assessment

No: the proposed early return is not the safest implementation because it trusts package.json version alone and skips existing lockfile, manifest, plugin-id, dependency-scan, and peer-link checks. The narrower maintainable fix is to detect the compatible existing package, then route through the existing installed-package validation path before returning success.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Guard the nullable semver comparison:** `src/plugins/install.ts:471-475`
  - body: `compareComparableSemver` returns `number | null`, but the new condition compares it directly with `>= 0`. CI is already failing `tsgo:prod` with TS2531 here, so store the comparison and check it is non-null before comparing.
  - confidence: 0.97
- **[P1] Validate the installed package before returning:** `src/plugins/install.ts:479-486`
  - body: This fast path accepts any existing directory with a package.json version greater than or equal to the request and returns success before the normal package-lock metadata check, OpenClaw manifest/plugin-id validation, dependency scan, and peer-link repair. That can mark a stale or invalid managed package as installed; route the skip through the existing installed-package validation/scanning path instead of constructing a success result.
  - confidence: 0.93
