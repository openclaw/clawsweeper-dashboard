## Summary

Keep open: the reported bug is real, but this PR's early cache hit would bypass the plugin install trust path and can also skip OpenClaw correction releases.

## What This Changes

The PR adds an install-mode npm plugin cache hit that skips npm install when the existing package.json version is equal to or newer than the requested version, with focused npm-spec tests.

## Best Possible Solution

Keep the idempotent skip direction, but route cache hits through installed-package validation, managed-root dependency scan, peer-link repair, validated metadata return, and the existing OpenClaw release comparator.

## Reproduction Assessment

Yes, source-level: current main rejects an existing managed npm target in install mode before checking the installed version, and the duplicate-target test asserts that behavior. I did not run tests because this review must keep the checkout read-only.

## Solution Assessment

No. The proposed early return is not the best fix because it bypasses validation/scanning and uses raw semver comparison instead of OpenClaw's release comparator.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Validate cached packages before returning success:** `src/plugins/install.ts:482-489`
  - body: This cache hit returns success after parsing only package.json.version. An existing target with the wrong plugin id, missing OpenClaw entries, incompatible host metadata, scan-blocked contents, or broken peer links would now be accepted; route the cache hit through the existing installed-package validation and scan path before returning.
  - confidence: 0.92
- **[P2] Use the OpenClaw release comparator for cache hits:** `src/plugins/install.ts:477`
  - body: This comparison treats `2026.5.3` as newer than requested `2026.5.3-1` because plain semver sees the correction suffix as a prerelease. OpenClaw correction releases are stable and newer than their base release, so this can incorrectly skip installing a requested correction version.
  - confidence: 0.88
- **[P2] Return validated installed package metadata:** `src/plugins/install.ts:484-487`
  - body: For newer installed versions, the result records the requested npmResolution and `extensions: []` instead of the accepted package's manifest name, version, plugin id, and extension entries. Onboarding or install records can persist stale metadata and later target the wrong artifact.
  - confidence: 0.86

## Security Review
