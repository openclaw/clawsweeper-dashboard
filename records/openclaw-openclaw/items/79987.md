## Summary

Keep open: the reported install bug is valid, but this PR still returns success from a version-only package.json check, bypassing the normal plugin validation and dependency-scan path and returning incomplete install metadata.

## What This Changes

The PR adds an install-mode fast path in managed npm plugin installs to skip npm install when an existing package version is equal to or newer than the requested version, plus unit coverage.

## Best Possible Solution

Keep the version-satisfied skip direction, but route cache hits through the existing installed-package validation, dependency scan, and peer-link repair path, then return metadata from the validated package.

## Reproduction Assessment

Yes, source-level: current main resolves the managed npm target and calls the duplicate-target availability gate before any installed-version comparison, and the current duplicate-install test asserts install mode rejects existing targets.

## Solution Assessment

No. The requested direction is narrow, but this implementation trusts package.json.version alone and should reuse the existing validated installed-package path before returning success.

## Review Findings
