## Summary

Keep open: the PR addresses a real source-reproducible install-mode bug, but the latest fast path accepts an existing managed npm directory after checking only `package.json.version`, bypasses normal plugin validation/security scanning, and persists requested npm metadata instead of the accepted package metadata. The supplied proof is still mocked unit-test output, not real CLI or wizard behavior.

## What This Changes

The PR adds an install-mode fast path in the managed npm plugin installer plus focused tests so an existing package with version greater than or equal to the requested version skips npm install.

## Best Possible Solution

Keep the install-mode cache-hit behavior, but accept an existing managed npm package only after the normal installed-package validation/security path and record metadata derived from the installed package or package-lock, then add real CLI or wizard proof.

## Reproduction Assessment

Yes from source inspection: current main resolves the requested npm metadata, then rejects an existing managed package directory via `ensureInstallTargetAvailableForMode` before comparing installed and requested versions. I did not run the Docker wizard flow in this read-only review.

## Solution Assessment

No. The skip direction is right, but the current PR shortcuts the installer validation/security contract and records requested npm metadata rather than the package it accepted.

## Review Findings

Overall confidence: 0.87

Full review comments:

- **[P1] Validate cached packages before skipping install:** `src/plugins/install.ts:484-490`
  - body: This branch returns success after reading only `package.json.version`. If the target directory contains the wrong package, lacks OpenClaw extension metadata, fails `expectedPluginId`, or would be blocked by the install scan, install mode still records/enables it instead of rejecting. Reuse `installPluginFromInstalledPackageDir` or equivalent validation before returning the cache hit.
  - confidence: 0.89
- **[P2] Persist metadata for the installed package:** `src/plugins/install.ts:489`

Onboarding and update records persist `result.npmResolution`, so later update or audit paths can believe the cached install came from the older artifact and target that older spec again.
  - confidence: 0.85
