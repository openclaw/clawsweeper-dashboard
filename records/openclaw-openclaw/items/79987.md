## Summary

Keep open: the linked bug is source-reproducible on current main, but this patch should not merge until it preserves install validation/security checks, uses OpenClaw-aware version ordering, and includes real behavior proof.

## What This Changes

The PR adds an install-mode fast path that reads an existing npm package version and skips reinstall when it is at least the requested version, with focused unit tests.

## Best Possible Solution

Keep the fast path, but make it validate the existing installed package through the same manifest/id/safety path and compare versions with the OpenClaw-aware comparator before returning success.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: installPluginFromNpmSpec resolves npm metadata, then installPluginFromManagedNpmRoot rejects an existing install target before comparing installed package versions. I did not run a live wizard or Docker reproduction.

## Solution Assessment

No. The direction is narrow, but the current patch returns success too early and uses generic semver ordering where OpenClaw already has a domain-specific comparator.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Validate the cached package before returning:** `src/plugins/install.ts:482`
  - body: This return treats a satisfying package.json version as a completed plugin install, but it skips the existing package manifest/id validation and install safety scan that runs through installPluginFromInstalledPackageDir. A stale or non-plugin directory with a high enough version would now be persisted as installed, so the skip path should reuse that validation or require a trusted install record before returning success.
  - confidence: 0.86
- **[P2] Use the OpenClaw-aware version comparator:** `src/plugins/install.ts:477`
  - body: The skip condition uses generic semver comparison. OpenClaw treats correction versions such as 2026.5.3-1 as stable and newer than 2026.5.3, but generic semver parses them as prereleases and lower, so those installed correction releases would still fall through to the duplicate-target error.
  - confidence: 0.84
