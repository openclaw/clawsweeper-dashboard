## Summary

Keep open: the PR targets a real plugin install bug, but the current patch returns before the normal plugin validation/safety scan and compares versions with the generic semver helper instead of OpenClaw's release-ordering helper. The supplied proof is still unit-test/mock output, not a real CLI or wizard install run.

## What This Changes

The PR adds an install-mode skip path in `src/plugins/install.ts` for existing npm plugin dirs with satisfying `package.json` versions and adds focused npm-spec tests.

## Best Possible Solution

Keep the fast path, but route the cached package through the same manifest/id/safety validation and OpenClaw-aware version comparison before accepting it, then add real CLI or wizard proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main resolves npm metadata, then rejects an existing managed npm target before checking whether the installed version already satisfies the requested version.

## Solution Assessment

No. The direction is narrow, but this implementation returns too early and uses generic semver ordering where OpenClaw already has a domain-specific comparator.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Validate the cached package before returning:** `src/plugins/install.ts:482`
  - body: This early return treats a satisfying `package.json` version as a completed plugin install, but it skips the manifest/id validation and install safety scan normally performed through `installPluginFromInstalledPackageDir`. A stale or non-plugin directory with a high enough version can now be accepted as installed, so reuse the existing validation path before returning success.
  - confidence: 0.86
- **[P2] Use the OpenClaw-aware version comparator:** `src/plugins/install.ts:477`
  - body: The skip condition uses the generic semver comparator. OpenClaw treats correction versions such as `2026.5.3-1` as stable and newer than `2026.5.3`, but generic semver treats them as prereleases, so valid correction installs can still fall through instead of being recognized as satisfying the request.
  - confidence: 0.84
