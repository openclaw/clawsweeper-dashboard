## Summary

Keep open: the PR targets a valid source-proven plugin install bug, but the current patch is not mergeable because the cache hit bypasses existing plugin validation/security scans, returns incomplete metadata, and still has only mock-based proof.

## What This Changes

The PR adds an install-mode fast path in `installPluginFromManagedNpmRoot` that skips npm install when an existing package version is equal to or newer than the requested npm version, plus unit coverage.

## Best Possible Solution

Keep the version-satisfied skip direction, but route cache hits through the existing installed-package validation and scan path and return metadata from the accepted package before requiring live CLI or wizard proof.

## Reproduction Assessment

Yes, source-level: current main computes the managed npm target and calls `ensureInstallTargetAvailableForMode` before any installed-version comparison, and the duplicate-install test asserts install mode rejects existing targets.

## Solution Assessment

No. The cache-hit direction is narrow, but this implementation trusts `package.json.version` alone and fabricates install metadata instead of reusing the validated installed-package path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Validate cached packages before returning success:** `src/plugins/install.ts:482-489`
  - body: This cache hit returns success after parsing only `package.json.version`. An existing target with the wrong plugin id, missing OpenClaw entries, incompatible host metadata, scan-blocked contents, or broken peer links would now be accepted; route it through the existing installed-package validation path before returning.
  - confidence: 0.9
- **[P2] Return validated package metadata on cache hits:** `src/plugins/install.ts:486-487`
  - body: For newer installed versions, the result records the requested `npmResolution` and `extensions: []` instead of the accepted package's actual manifest/version/extensions. Onboarding or update records can persist stale metadata and later target the wrong artifact.
  - confidence: 0.86
