## Summary

Keep this PR open, but do not merge it as-is: the underlying install bug is source-proven on current main, while the proposed skip path bypasses validation/safety scanning and uses the wrong version comparator.

## What This Changes

The PR adds an install-mode shortcut in `src/plugins/install.ts` that returns an existing npm plugin directory when its `package.json` version is greater than or equal to the requested version, with focused npm install tests.

## Best Possible Solution

Keep the fast-path idea, but validate the cached package through the existing installed-package path and compare versions with the OpenClaw-aware comparator before accepting it.

## Reproduction Assessment

Yes from source inspection, though not from a live wizard run. Current main rejects an existing managed npm plugin directory in install mode before checking whether the installed version satisfies the requested version, matching the linked WeCom wizard report.

## Solution Assessment

No. The direction is narrow, but this implementation returns too early and should reuse the established validation/safety scan plus `compareNpmSemver` before deciding that the installed package satisfies the request.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Validate the cached package before returning:** `src/plugins/install.ts:482`
  - body: This early return treats a satisfying `package.json` version as a completed plugin install, but it skips the manifest/id validation and install safety scan normally performed through `installPluginFromInstalledPackageDir`. A stale or non-plugin directory with a high enough version can now be accepted as installed, so route the cached directory through the existing validation path before returning success.
  - confidence: 0.86
- **[P2] Use the OpenClaw-aware version comparator:** `src/plugins/install.ts:477`
  - body: The skip condition uses the generic semver comparator. OpenClaw treats correction versions such as `2026.5.3-1` as stable and newer than `2026.5.3`, but generic semver treats them as prereleases, so valid correction installs can still fall through instead of being recognized as satisfying the request.
  - confidence: 0.84
