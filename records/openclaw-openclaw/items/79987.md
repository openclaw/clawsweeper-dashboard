## Summary

Keep this PR open: it targets a real managed npm install bug, but the current fast path is merge-blocking because it reports success from a version-only package.json check instead of preserving the normal validation, scan, and install-metadata contract.

## What This Changes

The PR adds a managed npm install-mode cache hit that skips reinstalling an already-installed plugin when its package.json version is greater than or equal to the requested version, plus focused unit coverage.

## Best Possible Solution

Keep the version-satisfies skip behavior, but route cache hits through the same installed-package validation and scan contract as normal installs and record metadata for the accepted installed package.

## Reproduction Assessment

Yes, from source inspection: current main reaches the install-mode duplicate-target guard before comparing installed and requested plugin versions. I did not run a live Docker wizard reproduction in this read-only review.

## Solution Assessment

No. The cache-hit direction is narrow, but this implementation is not the best fix because it accepts version-only package directories and records requested metadata instead of validating and recording the installed package.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Validate cached packages before returning success:** `src/plugins/install.ts:482-489`
  - body: This branch reports a successful install after reading only package.json.version. A preexisting target can have the wrong plugin id, no valid OpenClaw extensions, failed host compatibility, bad lockfile provenance, or scan-blocked contents, but install mode would still enable and record it. Reuse the normal installed-package validation and scan path before treating the existing directory as installed.
  - confidence: 0.9
- **[P2] Return installed package metadata on cache hits:** `src/plugins/install.ts:484-488`
  - body: When an installed newer version satisfies an older requested spec, the result carries the requested npmResolution and empty extensions rather than the accepted package's version, extensions, and provenance. The wizard records those fields, so config can persist stale metadata and later update checks can target the wrong artifact.
  - confidence: 0.86
