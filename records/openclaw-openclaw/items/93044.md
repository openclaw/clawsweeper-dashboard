## What This Changes

The issue reports duplicate Control UI webchat rendering of the same agent reply under `dmScope=main`, with one native copy and one relay-labeled copy.

## Best Possible Solution

Control UI should deduplicate adjacent native and relay-labeled projections by stable transcript id, prefer the native copy, preserve distinct ids, and close this issue through the linked merged PR.

## Reproduction Assessment

Yes. The reporter supplied live `dmScope=main` steps, and current source shows selected session-message delivery plus renderer collapse keyed only by role, sender label, and text rather than transcript id.

## Solution Assessment

Yes. The narrow maintainable solution is renderer/history-boundary id dedupe with native-copy preference, not a config change, backend transcript rewrite, or protocol change.
