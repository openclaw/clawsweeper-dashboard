## What This Changes

The issue asks for Control UI WebChat to render each shared-session agent reply once when `dmScope=main`, preferring the native unlabeled copy over the relay-labeled duplicate.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/93065 with an id-aware Control UI renderer dedupe that prefers the native unlabeled copy while preserving distinct transcript ids.

## Reproduction Assessment

Yes, source-supported but not live in this review. The reporter live-reproduced v2026.6.6, and current main still collapses only role/sender/text signatures even though session events carry stable transcript ids.

## Solution Assessment

Yes for direction. The narrow maintainable fix is renderer-level id/messageId dedupe with native-copy preference; backend transcript rewrites or config changes would be broader than the observed UI render bug.
