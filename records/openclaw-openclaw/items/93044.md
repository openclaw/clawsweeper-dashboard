## Summary

Keep open: current main and v2026.6.6 still collapse only identical consecutive display signatures, while this report's raw and relay-labeled copies differ and no open canonical item covers the dmScope=main relay-envelope duplicate.

## What This Changes

The issue asks Control UI WebChat to render each agent reply once when unified dmScope=main delivers both the native and relay-enveloped copy of the same message.

## Best Possible Solution

Add id/messageId-aware dedupe at the Control UI chat render or history boundary, prefer the native un-enveloped row for the same underlying message, and preserve distinct messages with different ids.

## Reproduction Assessment

Yes, source-level but not live: with dmScope=main the Control UI can receive both native chat/history and session-message delivery, and current rendering only collapses identical role/sender/text rows rather than same message id rows.

## Solution Assessment

Yes. The narrow maintainable fix is Control UI id/messageId dedupe with a native-copy preference, not a backend transcript rewrite, config change, or protocol change.
