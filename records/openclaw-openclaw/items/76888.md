Keep open: current main still routes queued orphan-user repair by prepending the older orphaned user text ahead of the latest triggering prompt, which matches the reported stale-reply failure shape and is not implemented, superseded, or stale.

## What This Changes

The issue asks OpenClaw to prevent queued/orphaned user-message repair from letting an older unresolved user turn steer the next visible reply over the latest triggering instruction.

## Best Possible Solution

Keep the issue open and change the embedded orphan-user repair so role-order repair cannot make older orphan text outrank the latest trigger, with focused regression coverage for latest-prompt-wins behavior.
