## Summary

Keep open: current main still has a source-backed orphan-user repair path that prepends an older orphaned user turn ahead of the latest triggering prompt, matching the reported stale-reply shape and conflicting with the documented latest-input steering behavior.

## What This Changes

The issue asks OpenClaw to ensure queued/orphaned user-message repair cannot make an earlier unresolved user turn steer the next reply over the latest user instruction.

## Best Possible Solution

Change the embedded runner's orphan-trailing-user repair so older orphan content cannot outrank the latest trigger, then add regression coverage for latest-prompt-wins while preserving provider-safe role ordering.
