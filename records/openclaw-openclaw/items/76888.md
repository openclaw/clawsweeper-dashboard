## Summary

Keep open: current main still has a source-backed orphan-user repair path that prepends an older orphaned user leaf ahead of the latest prompt, and focused tests currently lock that ordering. Related stale-transcript reports overlap, but the orphan-merge behavior here remains a distinct actionable path on current main.

## What This Changes

The issue asks OpenClaw to prevent queued/orphaned user-message repair from letting an older unresolved user turn steer the reply instead of the latest triggering instruction.

## Best Possible Solution

Make the embedded-runner orphan repair keep the latest triggering instruction authoritative while preserving strict provider role ordering, then update regression tests around merge ordering and trajectory submission.

## Reproduction Assessment

No, not as a high-confidence end-to-end live channel/model reproduction. The source path is clear: current main merges the older orphan leaf before the latest prompt and records that merged prompt on the submitted model-call path.

## Solution Assessment

Yes, the best direction is a narrow embedded-runner repair rather than a new queue mode or config option. The fix should change prompt shaping or pruning so latest-prompt-wins while keeping provider-safe alternation.
