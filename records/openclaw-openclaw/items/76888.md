## Summary

Keep open: current main still contains the orphan-trailing-user repair path that prepends older queued user text ahead of the newest triggering prompt, while the queue docs say the next model call should see the latest user input.

## What This Changes

The issue asks OpenClaw to prevent queued/orphaned user-message repair from causing the assistant to answer an older unresolved user turn instead of the latest triggering instruction.

## Best Possible Solution

Change the embedded runner's orphan-trailing-user repair so older orphan content cannot outrank the latest triggering instruction, then add regression coverage for latest-prompt-wins behavior without adding a queue mode or config option.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established because this read-only review did not run a live channel/model scenario. The source path is reproducible: current main prepends the older orphaned user text before the latest prompt and removes the user leaf, matching the reported failure shape.
