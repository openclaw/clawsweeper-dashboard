## Summary

Keep open: current main still has the orphan-trailing-user repair path that can place older queued user text before the latest triggering prompt, while the queue docs define the next model call as seeing the latest user input.

## What This Changes

The issue reports that queued/orphaned user-message merging can cause a channel-backed run to answer an earlier unresolved user turn instead of the latest prompt.

## Best Possible Solution

Adjust the embedded runner's orphan-trailing-user repair so older orphan content cannot override the latest triggering instruction, then add regression coverage for latest-prompt-wins behavior without adding a new queue mode or config option.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established in this read-only review. The source path is reproducible: current main prepends the older orphaned user text before the latest prompt and removes the leaf, matching the reported failure shape.

## Solution Assessment

Unclear because there is no proposed patch yet. The narrow maintainable direction is to change orphan-merge prompt shaping or pruning in the embedded-runner path so the latest triggering prompt remains authoritative.
