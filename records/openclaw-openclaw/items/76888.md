## Repair Work Prompt

Fix the queued/orphaned user-message merge stale-reply bug reported in https://github.com/openclaw/openclaw/issues/76888. In current main, mergeOrphanedTrailingUserPrompt prepends older orphan user text ahead of the latest triggering prompt, and runEmbeddedAttempt then removes the orphan leaf. Change the embedded runner's orphan-trailing-user repair so the latest triggering user instruction remains authoritative while still preventing provider-invalid consecutive user turns and preserving useful orphan content when appropriate. Coordinate with the adjacent persisted-leaf-removal work in #76968/#76969; do not add a new queue mode, config option, provider-specific policy, or broad transcript rewrite unless the source path proves it is required. Add regression coverage proving latest-prompt-wins for text and structured orphan content, update existing expectations that currently lock in older-text-first ordering, and add a user-facing CHANGELOG.md Fixes entry for this bug.
