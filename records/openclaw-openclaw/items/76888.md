## Summary

Keep open: current main still has the source-backed orphan-user repair path that prepends the older orphaned user text before the latest triggering prompt and submits that merged prompt to the model, while focused tests continue to assert that older-before-newer ordering.

## What This Changes

The issue reports that the embedded runner's orphaned-user merge can cause a queued run to answer an earlier unresolved user message instead of the latest triggering instruction.

## Best Possible Solution

Change the embedded runner's orphan-trailing-user repair so older orphan content cannot outrank the latest trigger while preserving provider-safe role ordering, then update focused regression coverage and the changelog.

## Reproduction Assessment

No, not as a high-confidence end-to-end channel/model run. Source inspection does reproduce the risky current-main path: the orphan leaf is prepended before the latest prompt, the leaf is removed, and the merged prompt is submitted.

## Solution Assessment
