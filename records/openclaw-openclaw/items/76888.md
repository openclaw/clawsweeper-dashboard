## Summary

Keep open: current main still has the source-backed orphan-user repair path that prepends older queued user text before the latest triggering prompt, removes the orphan leaf, and logs the same repair shape reported here.

## What This Changes

The issue reports that queued/orphaned user-message repair can make an embedded runner reply answer an older unresolved user turn instead of the latest triggering instruction.

## Best Possible Solution

Change the embedded runner's orphan-trailing-user repair so older orphan content cannot outrank the latest trigger while preserving provider-safe role ordering, then add focused regression coverage for latest-prompt-wins behavior.

## Reproduction Assessment

No, not for a high-confidence end-to-end live channel/model run. The source path is clear on current main: an older orphaned user leaf is prepended ahead of the latest prompt, removed from the active branch, and logged with the reported repair text.

## Solution Assessment

Unclear because there is no proposed patch yet. The narrow maintainable direction is to adjust prompt shaping or pruning in the embedded-runner orphan repair so the latest triggering prompt remains authoritative without adding a queue mode or config knob.
