## Summary

Keep open: current main still has the orphan-user repair path that prepends older queued user text ahead of the latest triggering prompt, and focused tests lock in that stale-prone ordering while the queue-steering docs say the next model call should see the latest user input.

## What This Changes

The issue reports that queued/orphaned user-message repair can let an earlier unresolved user turn steer the next visible reply over the latest prompt.

## Best Possible Solution

Change the embedded runner's orphan-trailing-user repair so older orphan content cannot outrank the latest trigger, then add regression coverage for latest-prompt-wins while keeping strict provider role ordering intact.

## Reproduction Assessment

No, not for a high-confidence end-to-end live channel/model run in this review. The source path is reproducible: current main prepends the older orphaned user leaf before the latest prompt, removes that leaf, and logs the same repair shape reported here.

## Solution Assessment

Unclear because there is no proposed patch yet. The narrow maintainable direction is to adjust prompt shaping or pruning in the embedded-runner orphan repair so the latest triggering prompt remains authoritative without adding a queue mode or config knob.
