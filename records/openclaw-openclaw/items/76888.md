## Summary

Keep open: current main still removes the orphaned user leaf while prepending its older text ahead of the latest prompt, so the reported stale-reply shape remains source-backed and is not implemented, duplicated, or stale.

## What This Changes

A channel-backed queued run can merge an orphaned earlier user turn into the next prompt and visibly answer that older turn instead of the latest triggering instruction.

## Best Possible Solution

Adjust orphan-trailing-user repair so prior orphan content cannot override the latest triggering instruction, then add focused regression coverage for latest-prompt-wins behavior without adding a queue mode or config knob.
