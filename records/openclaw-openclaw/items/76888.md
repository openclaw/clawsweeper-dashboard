## Summary

Keep open: current main still has a source-backed orphan-user repair path that prepends older queued user text ahead of the latest triggering prompt, matching the reported stale-reply shape.

## What This Changes

The issue reports that queued/orphaned user-message merging can make a channel-backed run answer an earlier unresolved user turn instead of the latest triggering instruction.

## Best Possible Solution

Change the embedded runner's orphan-trailing-user repair so older orphan content cannot outrank the latest trigger, while preserving provider-safe role ordering and adding regression coverage for latest-prompt-wins.
