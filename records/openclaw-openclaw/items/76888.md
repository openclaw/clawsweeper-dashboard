The issue reports stale replies when a queued/orphaned user-message merge lets an earlier unresolved user turn steer the next reply over the latest instruction.

Update the embedded runner's orphan-trailing-user repair so the latest triggering prompt remains authoritative while provider-safe role ordering is preserved, then align regression tests and the changelog with that invariant.
