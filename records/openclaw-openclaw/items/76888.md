## Summary

Keep open: current main still has a source-backed orphan-user repair path that prepends older queued user text ahead of the latest triggering prompt, matching the reported stale-reply shape and conflicting with the documented latest-input steering contract.

## What This Changes

The issue asks OpenClaw to prevent queued/orphaned user-message repair from letting an older unresolved user turn steer the next reply over the latest triggering instruction.

## Best Possible Solution

Change the embedded runner's orphan-trailing-user repair so older orphan content cannot outrank the latest trigger, then add regression coverage proving latest-prompt-wins while preserving provider-safe role ordering.

## Reproduction Assessment

No, not for a high-confidence end-to-end live channel/model run. The source path is reproducible: current main prepends the older orphaned user leaf before the latest prompt, removes that leaf, and records the same repair log shape reported here.

## Solution Assessment

Unclear because there is no proposed patch yet. The narrow maintainable direction is to adjust prompt shaping or pruning in the embedded-runner orphan repair so the latest triggering prompt remains authoritative without adding a new queue mode or config knob.
