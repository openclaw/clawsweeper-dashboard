## Summary

Keep open: current main still prepends older orphaned user text before the latest triggering prompt, matching the reporter's sanitized stale-reply trajectory and conflicting with the queue-steering contract that the next model call sees the latest user input.

## What This Changes

The issue reports that queued/orphaned user-message repair can make a run whose final prompt is the latest instruction visibly answer an earlier unresolved user turn.

## Best Possible Solution

Adjust the embedded runner's orphaned-user repair so preserved older orphan content cannot out-prioritize the latest triggering user instruction, and add regression coverage for latest-prompt-wins behavior without adding a queue mode or config option.

## Reproduction Assessment

No high-confidence live end-to-end reproduction was established. The source path is reproducible on current main: the repair helper prepends older orphan text ahead of the latest prompt and the runtime then removes the orphan leaf before submitting the prompt.

## Solution Assessment

Unclear because there is no proposed patch yet. The narrow maintainable solution is to change orphan-merge prompt shaping or pruning so the latest triggering prompt remains authoritative while preserving queued content and role ordering.
