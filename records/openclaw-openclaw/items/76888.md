## Summary

Keep open: current main still has the source-backed orphan-user repair path that prepends an older orphaned user turn ahead of the latest triggering prompt, matching the reported stale-reply shape and conflicting with the documented latest-input steering behavior.

## What This Changes

The issue reports that queued/orphaned user-message repair can merge an older unresolved user turn into a new run and cause the assistant to answer stale context instead of the latest instruction.

## Best Possible Solution

Adjust the embedded runner's orphan-trailing-user repair so older orphan content cannot outrank the latest trigger, then add regression coverage proving latest-prompt-wins while preserving provider-safe role ordering.

## Reproduction Assessment

No high-confidence end-to-end live channel/model reproduction was established. The source path is reproducible: current main prepends the older orphaned user leaf before the latest prompt, removes that leaf, and records the same repair log shape reported here.

## Solution Assessment

Unclear because there is no proposed patch for this issue yet. The narrow maintainable solution is to change prompt shaping or pruning in the embedded-runner orphan repair so the latest triggering prompt remains authoritative without adding a queue mode or config knob.
