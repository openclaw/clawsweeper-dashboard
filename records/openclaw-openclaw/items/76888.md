## Summary

Keep open: current main still has a source-backed orphan-user repair path that can put an older queued user turn ahead of the latest triggering prompt, matching the reported stale-reply shape and conflicting with the documented latest-input steering behavior.

## What This Changes

The issue reports that queued/orphaned user-message merging can make a channel-backed run answer an earlier unresolved user turn instead of the latest triggering instruction.

## Best Possible Solution

Adjust the embedded runner's orphan-trailing-user repair so older orphan content cannot outrank the latest trigger, while preserving provider-safe role ordering and adding regression coverage for latest-prompt-wins.

## Reproduction Assessment

No, not as a high-confidence end-to-end live channel/model run. The source path is reproducible from current main: an orphaned user leaf is merged ahead of the latest prompt, removed from the active branch, and sent through the normal prompt submission path.

## Solution Assessment

Unclear because there is no proposed patch yet. The narrow maintainable solution is to change prompt shaping or pruning in the embedded-runner orphan repair so the latest triggering prompt remains authoritative without adding a queue mode or config knob.
