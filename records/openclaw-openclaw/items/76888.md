## Label Justifications

- `P1`: A channel-backed agent can visibly answer an older instruction instead of the latest user prompt, which is a broken agent workflow for affected users.
- `impact:session-state`: The failure is rooted in session transcript leaf repair, branch state, and prompt assembly around orphaned user turns.
- `impact:message-loss`: The latest user instruction can be effectively ignored or displaced by older unresolved text in the delivered reply.

## Summary

Keep open: current main and the latest release still preserve older orphaned user text ahead of the latest prompt, and the focused fix attempt for this issue was closed unmerged. The broad stale-reply issues remain relevant context, but this issue still carries a specific embedded-runner/session repair shape that is not implemented on main.

## What This Changes

The issue reports that queued/orphaned user-message repair can let an older unresolved user turn steer the next visible reply instead of the latest triggering prompt.

## Best Possible Solution

Revive, replace, or re-review the focused embedded-runner orphan-repair approach so the runner looks past non-message state, preserves state entries, and keeps the latest active prompt as the answer target without changing queue policy.

## Reproduction Assessment

Yes for the source-level failure shape: current main repairs only a direct user leaf and tests assert older-first orphan merging. No live stochastic channel reproduction was run during this read-only review.

## Solution Assessment

Yes for the focused embedded-runner/session repair direction, with maintainer review of prompt wording and state replay semantics. Downstream reply filtering or a broad final-answer re-anchor would be a weaker fix because it would not repair the model-input/session-state mismatch.
