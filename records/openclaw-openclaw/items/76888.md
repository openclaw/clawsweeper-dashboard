## Summary

Keep open: current main still prepends the older orphaned user turn ahead of the latest prompt in the embedded-run repair path, so the reported stale-reply behavior remains source-plausible and is not already implemented or superseded.

## What This Changes

The issue reports that queued/orphaned user-message repair can let an earlier unresolved user turn steer the next visible reply instead of the latest triggering instruction.

## Best Possible Solution

Change the embedded orphan-repair prompt shaping so the latest triggering instruction remains the controlling user turn, with regression coverage for latest-prompt-wins behavior and no new queue mode or config knob.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established. The source path is reproducible: current main still prepends the older orphaned user text before the latest prompt and existing tests assert that ordering.

## Solution Assessment

Unclear because there is no proposed patch yet. The narrow maintainable solution is to adjust orphan-merge ordering/pruning in the embedded runner while preserving role alternation, not to add a new queue policy.
