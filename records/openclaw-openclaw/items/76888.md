Keep open: the current embedded-run merge path still preserves an older orphaned user turn ahead of the latest prompt, so the reported stale-reply behavior is plausible and not implemented, duplicated, or stale.

## What This Changes

The issue asks OpenClaw to prevent queued/orphaned user-message repair from making a run answer an earlier user turn instead of the latest triggering instruction.

## Best Possible Solution

Adjust orphan-trailing-user repair so prior orphan content cannot override the latest triggering instruction, then add focused regression coverage for the latest-prompt-wins behavior without adding a queue mode or config knob.
