## Summary

Keep open: current main still prepends an older orphaned user leaf ahead of the latest triggering prompt before removing the leaf, which matches the reported stale-reply shape and conflicts with the documented latest-input steering behavior.

## What This Changes

The issue reports that queued/orphaned user-message repair can let an older unresolved user turn steer the next reply instead of the latest triggering prompt.

## Best Possible Solution

Change the embedded runner's orphan-trailing-user repair so older orphan content cannot outrank the latest trigger, while preserving provider-safe role ordering and adding regression coverage for latest-prompt-wins.

## Reproduction Assessment

No for a high-confidence end-to-end live channel/model run. Yes for the source path: current main prepends the older orphaned user leaf before the latest prompt, removes that leaf, and emits the reported repair log shape.

## Solution Assessment

Unclear because there is no proposed patch yet. The narrow maintainable solution is to adjust prompt shaping or pruning in the embedded-runner orphan repair so the latest triggering prompt remains authoritative without adding a queue mode or config knob.
