## Summary

Keep open. Current main still has the reported producer/validator mismatch on the browser CLI media path: browser screenshot and snapshot commands emit home-shortened `MEDIA:~/...` lines, while the shared media parser intentionally rejects `~` paths; the direct prior fix PR is closed unmerged.

## What This Changes

The issue asks for browser screenshot media paths to survive outbound media validation so chat channels receive the captured image.

## Best Possible Solution

Emit absolute paths only in machine-readable browser CLI `MEDIA:` lines, keep home-shortened paths for human-readable output, and preserve shared parser rejection of `~` and traversal paths.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: current browser CLI code emits `MEDIA:~/...` under HOME through `shortenHomePath`, and current parser code/tests reject that form.

## Solution Assessment

Yes. A producer-side fix is the narrowest maintainable path because expanding `~` in the shared parser would weaken an intentional home-dir/traversal guard unless paired with new boundary checks.
