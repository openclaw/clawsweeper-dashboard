## Summary

Keep open. Current main still has the producer/validator mismatch: browser inspect commands emit home-shortened `MEDIA:~/...` lines, while the shared media parser intentionally rejects `~` paths, and the directly related fix PR was closed unmerged.

## What This Changes

Fix browser screenshot and snapshot media delivery so saved browser images are emitted through validator-accepted `MEDIA:` paths instead of home-shortened `~/` paths.

## Best Possible Solution

Update only the browser inspect CLI’s machine-readable `MEDIA:` output to use the absolute `result.path` and `result.imagePath` values, keep human display paths shortened, preserve parser tilde/traversal rejection, add focused regression tests, and add a user-facing changelog fix entry.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: current browser CLI code emits `MEDIA:` lines through `shortenHomePath`, and current parser tests explicitly reject the resulting `~/` form.

## Solution Assessment

No for the issue’s preferred parser-side tilde expansion; producer-side absolute media output is the narrower maintainable fix because the parser’s `~` rejection is an intentional safety guard.
