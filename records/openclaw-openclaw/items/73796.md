## Summary

Keep open. Current main still has the reported producer/validator mismatch: browser CLI media output is home-shortened with `shortenHomePath`, while the shared media parser intentionally rejects `~` paths; the related fix PR was closed unmerged.

## What This Changes

Browser screenshot delivery can fail when browser CLI output emits `MEDIA:~/...` paths that the shared media parser rejects before channel delivery.

## Best Possible Solution

Emit absolute paths only in machine-readable browser `MEDIA:` lines, keep `shortenHomePath` for human-readable output, preserve parser home-dir rejection, and add focused regression tests plus a changelog fix entry.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: current browser CLI code emits `MEDIA:~/...` when paths are under HOME, and current parser tests explicitly reject that form.

## Solution Assessment

Yes for a producer-side repair. Updating browser CLI `MEDIA:` output is narrower and safer than relaxing the shared parser’s intentional home-dir/traversal guard.
