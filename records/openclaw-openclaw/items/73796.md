## Summary

Keep open. Current main still has the producer/validator mismatch: browser CLI `MEDIA:` output is home-shortened, while the shared media parser rejects `~` paths; the direct fix PR #73798 is closed unmerged, and related PR #75762 does not remove the CLI tilde producer.

## What This Changes

Deliver browser screenshots to chat when an agent emits a `MEDIA:` line for the saved screenshot instead of dropping `~/` paths.

## Best Possible Solution

Emit absolute paths only in machine-readable browser `MEDIA:` lines, keep home-dir rejection in the shared parser, and add focused regression tests plus a changelog fix entry.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: current browser CLI code can emit `MEDIA:~/...` for HOME-relative screenshot paths, and current parser code/tests reject that exact form.

## Solution Assessment

Yes. A producer-side absolute-path repair is the narrowest maintainable fix because the screenshot route already returns absolute paths and the parser's tilde rejection is an intentional safety guard.
