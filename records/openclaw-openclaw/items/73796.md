## Summary

Keep open. Current main still has the producer/validator mismatch: browser screenshot and snapshot `MEDIA:` output is shortened with `shortenHomePath`, while the shared parser intentionally rejects `~` paths; #73798 was closed unmerged and #75762 is related but does not cover the CLI tilde producer path.

## What This Changes

The issue reports failed browser screenshot delivery when generated `MEDIA:~/...` paths are rejected by the shared media-source validator.

## Best Possible Solution

Emit absolute paths only for machine-readable browser `MEDIA:` lines, keep `shortenHomePath` for human-readable output, preserve parser tilde rejection, and add focused regression coverage plus a changelog fix entry.

## Reproduction Assessment

Yes. Current source emits `MEDIA:${shortenHomePath(...)}` for browser screenshot/snapshot image paths under HOME, while parser tests reject `MEDIA:~/...`, giving a high-confidence source-level reproduction without needing live Telegram.

## Solution Assessment

Yes for a producer-side repair. Updating browser output to emit absolute `MEDIA:` paths is narrower and safer than expanding `~` in the shared parser, which is currently a deliberate guard.
