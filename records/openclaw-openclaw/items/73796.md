## Summary

Keep open. Current main still has the reported mismatch: browser CLI machine-readable `MEDIA:` output can be home-shortened, while the shared media parser intentionally rejects `~` paths before outbound channel delivery.

## What This Changes

Fix browser screenshot delivery when browser CLI emits `MEDIA:~/...` paths that the shared media parser rejects.

## Best Possible Solution

Emit absolute paths only on browser CLI machine-readable `MEDIA:` lines, keep `shortenHomePath` for human-readable output, and preserve parser rejection of `~` paths.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: current main emits home-shortened browser CLI `MEDIA:` lines, and current parser tests reject the same `MEDIA:~/...` form.

## Solution Assessment

Yes for a producer-side repair. The browser route already returns absolute paths, making browser CLI output changes narrower and safer than relaxing the shared media parser.
