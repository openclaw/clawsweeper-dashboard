## Summary

Keep open. Current main still emits browser screenshot and AI snapshot `MEDIA:` directives through `shortenHomePath`, while the shared parser still rejects `~` as a home-dir/traversal prefix; the related PR #73798 is useful source context but was closed unmerged and does not resolve the issue.

## What This Changes

Browser screenshots emitted as `MEDIA:~/...` should be delivered to chat instead of being rejected before outbound media delivery.

## Best Possible Solution

Keep the shared parser's traversal and home-dir rejection intact, and update the browser CLI so agent-consumed `MEDIA:` lines use the absolute `result.path` and `result.imagePath` values while human-facing path summaries may remain home-shortened. Add focused browser CLI regression tests and a user-facing changelog fix entry.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: the browser CLI shortens absolute HOME paths into `MEDIA:~/...`, and the current parser test suite explicitly rejects `MEDIA:~/Pictures/My File.png`.

## Solution Assessment

Yes. The narrowest maintainable fix is producer-side absolute-path emission in the browser CLI, because the browser route already returns an absolute path and the parser's tilde rejection is intentional security behavior.
