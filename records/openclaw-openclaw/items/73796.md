## Summary

Keep open. Current main still has the reported producer/parser mismatch: browser CLI machine-readable `MEDIA:` lines are home-shortened with `shortenHomePath`, while the shared media parser intentionally rejects `~` paths before outbound delivery can attach them.

## What This Changes

The issue asks for browser screenshot `MEDIA:` paths under the user home directory to deliver as chat attachments instead of being dropped when emitted as `~/...`.

## Best Possible Solution

Keep the parser’s home-dir rejection, emit absolute paths only for machine-readable browser CLI `MEDIA:` lines, preserve shortened paths for human-readable logs, and add focused regression tests plus a changelog fix entry.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: current browser CLI code emits `MEDIA:~/...` for HOME-relative paths, and current parser tests explicitly reject that form.

## Solution Assessment

Yes. A producer-side browser CLI fix is the narrowest maintainable repair; the parser-expansion alternative is broader and security-sensitive, and #75762 does not cover the existing CLI output mismatch.
