## Summary

Keep open. Current main still has the reported producer/validator mismatch: browser CLI screenshot and AI snapshot media lines are HOME-shortened, the shared media parser still rejects `~` paths, and the exact producer-side fix PR is closed unmerged.

## What This Changes

The issue asks OpenClaw to deliver browser screenshots when browser-produced `MEDIA:` lines point at HOME-shortened local paths that the media validator rejects.

## Best Possible Solution

Emit absolute paths only for machine-readable browser CLI `MEDIA:` lines, keep shortened paths for human-readable output, preserve parser home-dir rejection, and add focused regression coverage plus a changelog fix entry.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: a browser screenshot saved under HOME is shortened to `MEDIA:~/...` by the CLI, and the current media parser rejects that directive before delivery.

## Solution Assessment

Yes for a producer-side repair. Updating browser CLI media output is narrower and safer than expanding `~` in the shared parser because current parser tests intentionally block home-dir paths.
