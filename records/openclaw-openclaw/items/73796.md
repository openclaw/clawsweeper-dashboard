## Summary

Keep open. Current main still has the producer/validator mismatch: browser machine-readable `MEDIA:` output uses `shortenHomePath`, the shared parser intentionally rejects `~` paths, and related PR #73798 was closed unmerged.

## What This Changes

The issue requests browser screenshot media delivery to work when generated `MEDIA:` lines use `~/` paths that the shared media parser rejects.

## Best Possible Solution

Emit absolute paths only for machine-readable browser `MEDIA:` lines, keep `shortenHomePath` for human-readable output, preserve parser tilde rejection, and add focused regression coverage plus a changelog fix entry.

## Reproduction Assessment

Yes. Current source emits `MEDIA:${shortenHomePath(...)}` for browser screenshot/snapshot image paths under HOME, while parser tests reject `MEDIA:~/...`; this gives a high-confidence source-level reproduction.

## Solution Assessment

Yes for a producer-side repair. Updating browser output to emit absolute `MEDIA:` paths is narrower and safer than expanding `~` in the shared parser, which is currently a deliberate guard.
