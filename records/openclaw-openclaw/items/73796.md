## Summary

Keep open. Current main still emits home-shortened browser CLI `MEDIA:` lines while the shared media parser rejects `~` paths, and the exact producer-side fix PR was closed unmerged; related PR #75762 overlaps screenshot media hints but does not remove the CLI tilde output.

## What This Changes

The request is to make browser screenshot `MEDIA:` paths deliverable over chat instead of emitting `~/` paths that the shared media validator rejects.

## Best Possible Solution

Emit absolute paths only in machine-readable browser `MEDIA:` lines, keep `shortenHomePath` for human-readable logs, preserve parser home-dir rejection, and add focused regression tests plus a changelog fix entry.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: current browser CLI code prints `MEDIA:${shortenHomePath(...)}` for saved screenshots under HOME, while current parser tests reject `MEDIA:~/...`; no live Telegram run is required to confirm the mismatch.

## Solution Assessment

Yes for a producer-side repair. Emitting absolute paths from the browser CLI is narrower and safer than expanding `~` in the shared parser, because the parser rejection protects against home-dir file exfiltration patterns.
