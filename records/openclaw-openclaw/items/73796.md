## Summary

Keep open. Current main still emits browser screenshot and AI snapshot `MEDIA:` directives through `shortenHomePath`, while the shared parser still treats `~` paths as rejected home-dir/traversal media, so the reported Telegram delivery failure remains valid; the related fix PR was closed unmerged.

## What This Changes

Browser screenshot and snapshot media output should reach chat channels as attachments instead of being dropped when the emitted `MEDIA:` path is home-shortened with `~/`.

## Best Possible Solution

Keep the shared parser's traversal and home-dir rejection intact, update the browser inspect CLI so agent-consumed `MEDIA:` lines use the absolute `result.path` and `result.imagePath` values, and add focused regression tests plus a user-facing changelog fix entry.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: current browser CLI code converts HOME paths to `MEDIA:~/...`, and current parser tests explicitly reject that form.

## Solution Assessment

Yes for a producer-side fix. Emitting absolute paths from the browser CLI is narrower and safer than expanding `~` in the shared parser because the parser's home-dir rejection is an intentional guard.
