## Summary

Keep open: the gateway admin-scope fix is useful, but the branch also reverses the current macOS LaunchAgent PATH policy and still lacks external real behavior proof.

## What This Changes

The branch adds `operator.admin` scopes for `openclaw agent` model override and `/new`/`/reset` gateway calls, updates docs/tests/changelog, and changes macOS LaunchAgent PATH defaults to include user tool directories.

## Best Possible Solution

Keep the focused admin-scope fix for `/new` and `/reset` with regression coverage, while preserving the current canonical macOS LaunchAgent PATH unless daemon owners explicitly change that policy.

## Reproduction Assessment

Yes, source-backed but not live-run: current main sends reset-style `openclaw agent` requests without admin scope, while the gateway reset handler rejects write-scoped `/new` and `/reset` calls with `missing scope: operator.admin`. No live command was run because this is a read-only review.

## Solution Assessment

No. The admin-scope direction is narrow and appropriate, but the macOS PATH default reversal is unrelated to the reset authorization bug and conflicts with current daemon policy.
