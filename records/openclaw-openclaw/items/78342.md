## Summary

Keep open: the admin-scope fix direction is useful and source-backed, but the PR is not correct as-is because it also reverses the current macOS LaunchAgent PATH hardening and still lacks required after-fix real behavior proof.

## What This Changes

The PR adds explicit admin scopes for selected `openclaw agent` gateway calls, updates CLI docs/tests/changelog, and changes macOS LaunchAgent PATH defaults to include user tool directories.

## Best Possible Solution

Keep the focused gateway admin-scope fix with regression coverage, and remove or split the macOS PATH policy reversal unless a daemon owner explicitly approves that separate policy change.

## Reproduction Assessment

Yes, source-backed: current main sends CLI agent requests for `/new` and `/reset` without explicit admin scope, while the gateway rejects reset commands from write-scoped clients with `missing scope: operator.admin`. I did not run live commands because this review is read-only.

## Solution Assessment

No. The admin-scope direction is narrow and appropriate, but the macOS PATH default reversal should be dropped or handled as a separate owner-reviewed policy change before merge.
