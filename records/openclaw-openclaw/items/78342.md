## Summary

Keep open. The `/new` and `/reset` admin-scope fix is still useful on current main, but this branch mixes it with a macOS LaunchAgent PATH policy/security reversal and has no real behavior proof, so it is not mergeable as-is.

## What This Changes

The branch adds admin-scoped `openclaw agent` gateway requests for `/new`, `/reset`, and model overrides, updates docs/tests/changelog, and changes macOS service PATH defaults to include user tool directories.

## Best Possible Solution

Keep the narrow CLI gateway admin-scope fix with focused tests/docs, and leave macOS LaunchAgent PATH canonical unless a separate owner-reviewed policy change approves expanding it.

## Reproduction Assessment

Yes, source-backed but not live-run: current main sends ordinary `/new` and `/reset` `agent` calls with CLI/write scope, while the gateway rejects reset commands without `operator.admin`. The PR still needs real after-fix proof from a real setup.

## Solution Assessment

No. The admin-scope direction is narrow and appropriate, but the PR should drop or split the macOS PATH policy reversal and prove the gateway behavior before merge.
