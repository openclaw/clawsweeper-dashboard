## Summary

Keep open: the narrow admin-scope fix is still useful on current main, but the PR cannot merge as-is because it also reverses the current macOS LaunchAgent PATH policy and still lacks required real behavior proof.

## What This Changes

The PR adds explicit admin scopes for selected `openclaw agent` gateway calls, updates docs/tests/changelog, and changes macOS LaunchAgent PATH defaults to include user tool directories.

## Best Possible Solution

Keep the focused gateway admin-scope fix with regression coverage, and remove or split the macOS PATH policy reversal unless a daemon owner explicitly approves it.

## Reproduction Assessment

Yes, source-backed but not live-run: current main sends `/new` and `/reset` agent calls without explicit admin scope, while the gateway rejects reset commands without `operator.admin`. No live command was run because this is a read-only review.

## Solution Assessment

No. The admin-scope direction is narrow and appropriate, but the PATH default reversal should be dropped or handled as a separate owner-reviewed policy change before merge.
