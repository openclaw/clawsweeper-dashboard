## Summary

Keep open. The branch contains useful Feishu work, but it should not merge as-is because it is stale against current main's generic conversation-binding architecture and has concrete account-scope and boundary regressions. Current main already includes the Feishu ACP/session-binding foundation from #46819 and generic ACP block fallback behavior, but Feishu thread-binding config and lifecycle support still need a clean plugin-owned implementation.

## What This Changes

The PR adds Feishu focus/unfocus and session lifecycle support, Feishu thread-binding config/startup wiring, ACP block delivery changes, session-binding SDK/registry changes, and an ACP child-process environment adjustment.

## Best Possible Solution

Rework the useful Feishu pieces on top of current main's generic plugin conversation-binding seam: expose Feishu threadBindings schema/metadata, account-aware command conversation resolution, current-binding support and lifecycle setters through `conversationBindings`, keep block-delivery behavior covered by targeted Feishu ACP tests, and add docs/changelog without core-to-extension private imports.

## Reproduction Assessment

Yes. The PR branch's regressions are source-reproducible by comparing its fixed channel allowlist and core Feishu deep import with current main's generic binding path; the account-scope bug follows from a multi-account Feishu config where `accounts.<id>.groupSessionScope` differs from the top-level config.

## Solution Assessment

No. The direction is useful, but this implementation is not the best fix because it duplicates Feishu-specific command behavior in core and bypasses the current plugin-owned lifecycle seam instead of extending it cleanly.
