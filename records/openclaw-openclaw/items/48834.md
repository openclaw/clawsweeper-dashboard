## Summary

Keep open. The PR still targets real Feishu gaps, especially non-streaming ACP block delivery and Feishu lifecycle/config coverage, but it should not merge as-is because it is stale against main's generic conversation-binding path and has blocking account-scope and boundary issues.

## What This Changes

The PR adds Feishu focus/unfocus and session lifecycle support, ACP block delivery changes, Feishu thread-binding config/startup wiring, session-binding SDK/registry changes, and an ACP child-process environment adjustment.

## Best Possible Solution

Rework the useful Feishu pieces on top of current main's plugin conversation-binding seam: expose Feishu `threadBindings` schema/metadata, `supportsCurrentConversationBinding`, lifecycle setters, and manager creation through `conversationBindings`; make command conversation resolution account-aware; fix non-streaming ACP block delivery with targeted tests; and add docs/changelog without core deep imports.

## Reproduction Assessment

Yes. The current-main block-delivery gap is source-backed by the Feishu dispatcher early return and a regression test that asserts block suppression; the PR branch's account-scope bug is reproducible with a multi-account Feishu config whose account override changes `groupSessionScope`.

## Solution Assessment

No. The direction is useful, but this implementation is not the best fix because it duplicates channel-specific command behavior in core and deep-imports Feishu internals instead of extending the current generic plugin binding lifecycle surface.
