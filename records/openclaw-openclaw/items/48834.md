## Summary

Keep open: this is useful Feishu parity work, but the branch is not ready because sender-scope detection ignores account-specific Feishu config, core imports Feishu private extension code directly, and the external PR lacks real behavior proof. Current main has also moved this area toward generic conversation-binding APIs, so rebasing onto that path is the safer direction.

## What This Changes

The PR adds Feishu `/focus`, `/unfocus`, and `/session` binding support, changes Feishu ACP block delivery, wires thread-binding lifecycle/config/SDK exports, and strips `CLAUDECODE` from ACP child processes.

## Best Possible Solution

Rebase or replace this with a narrow current-main patch that uses Feishu's channel-owned conversation binding API for `/focus`, `/unfocus`, `/session`, and ACP delivery, with account-scoped config coverage and real Feishu proof.

## Reproduction Assessment

Yes, at source level for the review findings. The PR helper reads only top-level Feishu config while Feishu runtime routing merges account-specific config, so a multi-account `groupSessionScope` override can make `/focus`, `/unfocus`, and `/session` mutate a different conversation id than live routing uses.

## Solution Assessment

No. The safer solution is to rebase onto current main's generic conversation-binding hooks and channel plugin capabilities rather than adding Feishu-specific core imports and duplicate config resolution.

  - confidence: 0.88
