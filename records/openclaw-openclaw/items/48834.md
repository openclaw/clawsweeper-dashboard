## Summary

Keep open: the PR is still trying to address useful Feishu binding gaps, but it is not mergeable as-is because the current branch still has source-level blockers and lacks after-fix real Feishu proof.

## What This Changes

The PR adds Feishu `/focus`, `/unfocus`, and session-binding support, changes Feishu ACP block delivery, adds thread-binding config/lifecycle wiring, and strips `CLAUDECODE` from spawned ACP processes.

## Best Possible Solution

Rebase or replace this with a current-main patch that uses generic conversation-binding APIs, keeps Feishu-specific routing inside the plugin, resolves sender scope from effective account config, and proves `/focus` plus ACP block delivery in a real Feishu setup.

## Reproduction Assessment

Yes, for the review findings at source level: the PR helper reads top-level Feishu config while current routing uses merged account config, and core imports plugin-private Feishu helpers. Real Feishu behavior is not reproduced from PR artifacts.

## Solution Assessment

No. The safer solution is to rebase onto current main's generic conversation-binding APIs and expose Feishu lifecycle behavior through the channel/plugin contract.
