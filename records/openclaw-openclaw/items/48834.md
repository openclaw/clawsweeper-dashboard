## Summary

Keep open: this PR still contains useful Feishu command and ACP delivery work, but it is not mergeable as-is because source-level blockers remain and the external PR lacks real Feishu behavior proof.

## What This Changes

The PR enables Feishu `/focus`, `/unfocus`, and `/session` binding support, changes Feishu ACP block delivery, adds thread-binding config/lifecycle wiring and SDK exports, and strips `CLAUDECODE` from spawned ACP processes.

## Best Possible Solution

Rebase or replace this with a current-main patch that uses generic conversation-binding APIs, keeps Feishu-specific routing inside the plugin, resolves sender scope from effective account config, and proves `/focus` plus ACP block delivery in a real Feishu setup.

## Reproduction Assessment

Yes, for the review findings at source level: the PR helper reads top-level Feishu config while current routing merges account config, and core imports plugin-private Feishu helpers. Real Feishu behavior has not been reproduced from PR artifacts.

## Solution Assessment

No. The safer path is to rebase onto current main's generic conversation-binding APIs and keep Feishu routing/config resolution behind the channel/plugin contract.

  - confidence: 0.88
