## What This Changes

The branch adds Feishu `/focus`, `/unfocus`, and `/session` conversation binding support, Feishu ACP block delivery changes, thread-binding config/lifecycle wiring, SDK exports, and ACP child-process env cleanup.

## Best Possible Solution

Rebase or replace this with a current-main patch that keeps Feishu conversation binding in channel/plugin APIs, uses account-scoped config consistently, and includes real Feishu proof for commands and ACP block delivery.

## Reproduction Assessment

Yes, at source level for the review findings. The PR helper reads only top-level Feishu config while current routing uses merged account config, so a multi-account scope override can make commands mutate a different conversation id than live routing uses.

## Solution Assessment

No. The safer solution is to rebase onto current main's generic conversation-binding APIs and add Feishu plugin-owned lifecycle capabilities instead of duplicating Feishu config resolution in core and importing private plugin files.
