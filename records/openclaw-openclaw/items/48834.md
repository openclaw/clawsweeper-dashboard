## Summary

Keep open. Current main has moved part of this work into generic conversation-binding plumbing, but the full Feishu/ACP behavior is still not complete, and this PR branch is stale enough that merging it as-is would regress existing generic `/focus` behavior and keep a Feishu sender-scope bug in multi-account configs.

## What This Changes

The PR branch adds Feishu support for `/focus`, `/unfocus`, `/session` lifecycle controls, ACP block delivery, Feishu thread-binding config/lifecycle wiring, session-binding registry changes, command metadata updates, and an ACP child-process environment adjustment.

## Best Possible Solution

Rework the useful Feishu pieces on top of current main: keep the generic conversation-binding command flow, add Feishu `threadBindings` schema/metadata and lifecycle setters through the plugin `conversationBindings` seam, make Feishu command conversation resolution account-aware, fix non-streaming ACP block delivery with tests, and update Feishu command/docs copy.

## Reproduction Assessment

Yes. The current-main block-delivery gap is source-backed by the early return in Feishu reply dispatch plus a test that asserts suppression, and the PR branch's sender-scope bug is reproducible by a multi-account Feishu config whose account override differs from top-level `groupSessionScope`.

## Solution Assessment

No. The PR direction is useful, but the implementation is not the best current fix because it duplicates channel-specific command logic and deep-imports Feishu internals instead of using the generic plugin conversation-binding surfaces already present on main.
