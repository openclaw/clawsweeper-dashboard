## Summary

Keep open: current main and v2026.6.9 still choose the last outbound heartbeat payload without excluding reasoning, and https://github.com/openclaw/openclaw/pull/92356 is the open closing PR for the fix path.

## What This Changes

Fix heartbeat payload selection so default heartbeats do not deliver reasoning/thinking payloads as visible replies when `includeReasoning` is false.

## Best Possible Solution

Land one canonical heartbeat selector fix that excludes SDK-classified reasoning from the main reply while preserving the opt-in Thinking delivery path.

## Reproduction Assessment

Yes at source level. A reply result shaped as final text followed by `{ text, isReasoning: true }` makes current main return the reasoning payload because it scans backward and checks only outbound content.

## Solution Assessment

Yes. The narrow maintainable solution is a selector-layer fix that prevents reasoning payloads from becoming the visible heartbeat reply while preserving the separate `includeReasoning` Thinking path.
