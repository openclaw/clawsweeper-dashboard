## Summary

Keep open: current main and the latest release still choose the last outbound heartbeat payload without excluding reasoning, and an open closing PR is already the active fix path. The issue should remain open until https://github.com/openclaw/openclaw/pull/92356 lands or maintainers choose a different canonical fix.

## What This Changes

The issue requests that heartbeat main-reply selection avoid delivering reasoning payloads when `includeReasoning` is false.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/92356 as the canonical fix; after a merge, this issue can close through the linked PR.

## Reproduction Assessment

Yes at source level. A reply result shaped as final `HEARTBEAT_OK` followed by `{ text, isReasoning: true }` is enough for current `resolveHeartbeatReplyPayload` to return the reasoning payload because it scans backward and checks only outbound content.

## Solution Assessment

Yes. The narrow maintainable fix is to change heartbeat main-reply selection so SDK-classified reasoning is not selected as the visible reply while preserving the separate `includeReasoning` Thinking path.
