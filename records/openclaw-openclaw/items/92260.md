## What This Changes

Prevent heartbeat from delivering a reasoning payload as the primary visible reply when reasoning output is disabled.

## Best Possible Solution

Land the linked fix that reuses the shared reasoning classifier, covers the actual heartbeat runner delivery path, and preserves explicit includeReasoning behavior, then close the competing PR.

## Reproduction Assessment

Yes. On current main, an ordered result ending with outbound `isReasoning: true` content deterministically selects that reasoning payload because the reverse scan checks only outbound content.

## Solution Assessment

Yes, at the heartbeat selector boundary. The strongest path is to use the existing reasoning classifier and prove both default suppression and the explicit includeReasoning opt-in path through runner-level coverage.
