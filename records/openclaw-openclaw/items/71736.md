## Summary

Keep open; current main has only a generic descriptor/session-extension foundation, while the requested broader Control UI slot model remains unimplemented and still needs maintainer design.

## What This Changes

Requests a first-class, data-driven Control UI plugin contribution API for chat modes, approval cards, event classifiers, input guards, and status surfaces.

## Best Possible Solution

Keep this as the broad RFC until maintainers approve an additive, versioned, data-only contribution model with gateway projection, host-owned rendering, action authorization, disablement semantics, and fixture coverage for enabled and disabled plugins.

## Reproduction Assessment

Not applicable; this is an RFC/feature request rather than a bug. The high-confidence verification path is source, docs, protocol, UI searches, and issue discussion showing only the generic descriptor foundation on current main.

## Solution Assessment

Unclear. The requested direction is plausible, but the maintainable implementation should be a maintainer-approved host-rendered contract rather than arbitrary browser JavaScript or a broad mutable UI registry.

Status: needs_attention
