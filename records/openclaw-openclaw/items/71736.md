## Summary

Keep open; current main and the latest release have the narrower `registerControlUiDescriptor` and plugin session-action foundation, but not the broader Control UI contribution-slot vocabulary or browser rendering/security contract requested here.

## What This Changes

Request adds first-class plugin SDK, Gateway, and Control UI contribution slots for plugin chat modes, approval cards, event classifiers, input guards, and status surfaces.

## Best Possible Solution

Keep this as a maintainer-owned RFC until OpenClaw chooses a versioned, data-only Control UI contribution model with host-owned rendering, explicit scopes, disablement semantics, and security tests.

## Reproduction Assessment

Not applicable: this is an RFC for new SDK and Control UI contribution surfaces, not a bug report. Source inspection verifies the requested broader slot model is absent from current main and `v2026.6.6`.

## Solution Assessment

Unclear: the RFC identifies a real missing API family, but the best implementation needs maintainer decisions on descriptor vocabulary, host-owned component catalog, browser trust boundary, and protocol/versioning before code should land.
