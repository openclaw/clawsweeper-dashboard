## Summary

Keep open. Current main has the generic host-hook/UI descriptor foundation, but it still lacks the broader Control UI contribution-slot model requested here, and the issue discussion treats the related PRs and panel prototype as partial design inputs rather than a complete implementation.

## What This Changes

Request to add a versioned, data-driven Control UI plugin contribution API for chat modes, approval cards/renderers, event classifiers, input guards, status surfaces, and related host-rendered descriptors.

## Best Possible Solution

Keep this as the broad Control UI contribution-slot tracker until maintainers approve an additive, versioned, data-only descriptor model with gateway projection, host-owned rendering, disablement semantics, action authorization, and fixture coverage.

## Reproduction Assessment

Not applicable; this is an RFC/feature request rather than a bug. The high-confidence verification path is source, docs, protocol, UI search, and issue-discussion review, which shows only the generic descriptor foundation on current main.

## Solution Assessment

Unclear. The requested direction fits OpenClaw's plugin API strategy, but the best implementation needs maintainer-approved scope and should preserve a data-only, host-rendered browser boundary rather than exposing arbitrary plugin JavaScript or an unbounded UI registry.
