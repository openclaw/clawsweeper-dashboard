## Summary

Keep open; current main and v2026.6.8 still expose only the narrower plugin Control UI descriptor/session-action foundation, not the requested contribution-slot registry or browser rendering contract.

## What This Changes

The issue requests data-driven Control UI contribution slots for plugin chat modes, approval cards, event classifiers, input guards, and status surfaces.

## Best Possible Solution

Keep this as a maintainer-owned RFC until OpenClaw chooses a versioned, data-only Control UI contribution model with host-owned rendering, scopes, disablement semantics, and security tests.
