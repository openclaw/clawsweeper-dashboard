## Summary

Keep open; current main and v2026.5.18 only provide a narrow generic Control UI descriptor/session-action foundation, while the requested chat modes, input guards, event classifiers, approval-slot families, status surfaces, browser rendering contract, and security model remain design work.

## What This Changes

The issue requests a new public SDK/protocol/UI contribution model so enabled plugins can declare safe Control UI slots such as chat modes, approval cards, event classifiers, input guards, and status surfaces.

## Best Possible Solution

Keep this as the broad design tracker until maintainers approve a versioned, data-only Control UI contribution model with safe host-owned rendering, Gateway projection, action authorization, disablement semantics, and fixture/client coverage.

## Reproduction Assessment

Not applicable; this is an RFC/feature request rather than a reported runtime failure. Source and release inspection give high confidence that the requested broader slot vocabulary is not present on current main or v2026.5.18.

Unclear; the descriptor-first, host-owned direction fits the plugin architecture, but the exact SDK schema, Gateway projection, browser rendering model, action semantics, versioning, and security boundary need maintainer design approval.
