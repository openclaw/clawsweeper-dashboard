## Summary

Keep open; current main has a useful generic Control UI descriptor foundation, but it still does not implement the broader contribution-slot model requested for chat modes, input guards, event classifiers, status surfaces, approval-card families, or native/panel rendering contracts.

## What This Changes

Request an SDK, gateway, and Control UI contribution-slot model that lets enabled plugins declare safe host-rendered chat modes, approval/status UI, event classifiers, input guards, and related UI descriptors.

## Best Possible Solution

Keep this as the broad design tracker until maintainers approve an additive, versioned, data-only contribution model with gateway projection, host-owned rendering, authorized plugin actions, disablement semantics, and fixture coverage for enabled and disabled plugins.

## Reproduction Assessment

Not applicable; this is an RFC/feature request, not a bug report. The high-confidence verification path is source, docs, tests, protocol schema, UI search, and related discussion showing only the generic descriptor foundation on current main.

## Solution Assessment

Unclear. The requested direction is plausible, but the maintainable implementation should be a maintainer-approved, data-only host-rendered contract rather than arbitrary plugin browser JavaScript or a broad mutable UI registry.
