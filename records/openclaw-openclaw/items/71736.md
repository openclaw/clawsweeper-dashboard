## Summary

Keep open; current main has a narrow generic Control UI descriptor foundation, but the requested broader contribution-slot vocabulary, browser rendering contract, and trust-boundary decisions are still absent and still under active design discussion.

## What This Changes

Request a first-class, data-only Control UI contribution API for plugins to expose chat modes, approval cards, event classifiers, input guards, and status surfaces.

## Best Possible Solution

Keep this as the broad design tracker until maintainers approve a versioned, data-only Control UI contribution model with safe host-owned rendering, Gateway projection, action authorization, plugin disablement semantics, and focused fixture/client coverage.
