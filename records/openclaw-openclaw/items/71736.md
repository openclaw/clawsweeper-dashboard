## Summary

Keep open; current main and v2026.6.1 have a useful generic descriptor/session-action foundation, but not the broader Control UI contribution-slot vocabulary, browser rendering contract, or security model requested here.

## What This Changes

Request to add data-driven Control UI plugin contribution slots for chat modes, approval cards, event classifiers, input guards, status surfaces, and related safe host-rendered descriptors.

## Best Possible Solution

Keep this as a maintainer-owned API/security RFC until the project chooses a versioned, data-only descriptor model, host-owned component catalog, and Gateway/UI rendering contract that can cover the related panel/native-page needs.

## Reproduction Assessment

Not applicable: this is an RFC for new SDK and Control UI contribution surfaces, not a bug report. Source inspection confirms current main only has the narrower descriptor/session-action foundation.

## Solution Assessment

No, the current implementation is not the full solution yet; the best path is a maintainer-approved, versioned host-owned descriptor/rendering model rather than accepting arbitrary plugin UI code or treating generic descriptors as complete.
