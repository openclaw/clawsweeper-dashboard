## Summary

Keep open. Current main has the generic host-hook descriptor foundation, but not the broader Control UI contribution-slot system requested here; the active discussion and follow-up PRs describe partial foundations, not a completed closeout.

## What This Changes

Request to add data-driven plugin Control UI contribution slots for chat modes, approval cards, event classifiers, input guards, status surfaces, and safe host-rendered descriptor projection.

## Best Possible Solution

Keep this issue as the broad Control UI contribution-slot design tracker until maintainers choose an additive, versioned descriptor model with typed schemas, gateway projection, host-owned browser components, plugin disablement behavior, and fixture coverage for enabled/disabled plugins and plugin-originated actions.

## Reproduction Assessment

Not applicable; this is an RFC/feature request rather than a bug. The verification path is source, docs, protocol, UI search, and discussion review against current main, which shows only the generic descriptor foundation.

## Solution Assessment

Unclear. The direction is plausible, but the maintainable first implementation should be a maintainer-approved, manifest-first/data-only slot vocabulary rather than arbitrary browser plugin code or a broad mutable UI registry.
