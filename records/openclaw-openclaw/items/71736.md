## Summary

Keep open; current main has only a narrow Control UI descriptor foundation and related follow-up PRs cover slices, not the broader contribution-slot RFC. The request is a new SDK/Gateway/UI surface that still needs maintainer API and security design.

## What This Changes

Request adds a declarative SDK/Gateway Control UI contribution surface for plugin-provided chat modes, approval cards, event classifiers, input guards, and status surfaces.

## Best Possible Solution

Keep this as the broad design tracker until maintainers approve a versioned, data-only contribution model with safe host-owned rendering, Gateway projection, action authorization, disablement semantics, and fixture coverage.
