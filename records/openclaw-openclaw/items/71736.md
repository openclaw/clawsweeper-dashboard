## Summary

Keep open; current main has a narrow generic Control UI descriptor and session-action foundation, but not the broader contribution-slot vocabulary, browser rendering contract, or security model requested here. The live discussion and related items still treat this as active design work, not cleanup-ready implemented behavior.

## What This Changes

The issue requests a declarative plugin SDK, Gateway, and Control UI contribution surface for chat modes, approval cards, event classifiers, input guards, status surfaces, and safe host-owned rendering.

## Best Possible Solution

Keep this as the broad design tracker until maintainers approve a versioned, data-only Control UI contribution model with safe host-owned rendering, Gateway projection, action authorization, disablement semantics, and fixture/client coverage.
