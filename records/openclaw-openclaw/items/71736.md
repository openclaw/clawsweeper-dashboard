## Summary

Keep open; current main has a useful generic host-hook foundation, session actions, and session-state projection, but it still lacks the broader Control UI contribution-slot model requested for chat modes, input guards, event classifiers, status surfaces, approval families, and panel/native-page rendering contracts.

## What This Changes

Add a data-driven plugin SDK and Control UI contribution-slot surface for plugin chat modes, approval cards, event classifiers, input guards, status surfaces, and safe host-rendered descriptors.

## Best Possible Solution

Keep this as the broad design tracker until maintainers approve an additive, versioned, data-only contribution model with gateway projection, host-owned rendering, authorized plugin actions, disablement semantics, and fixture coverage for enabled and disabled plugins.

## Reproduction Assessment

Not applicable; this is an RFC/feature request rather than a bug report. The high-confidence verification path is source, docs, tests, protocol schema, UI search, and related discussion showing only the generic descriptor foundation on current main.

## Solution Assessment

Unclear. The descriptor-first, host-owned direction is plausible, but the maintainable implementation should be a maintainer-approved versioned contract rather than arbitrary plugin browser JavaScript or a broad mutable UI registry.
