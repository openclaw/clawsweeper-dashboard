## Summary

Keep open: current main and v2026.6.10 still expose only a narrow plugin Control UI descriptor/session-action surface, while the requested broader contribution-slot model remains an SDK, Gateway, Control UI, and browser-security design decision with overlapping open PRs.

## What This Changes

The issue requests a new plugin SDK and Control UI contribution-slot surface for plugin chat modes, approvals, event classifiers, input guards, and status/status-panel rendering.

## Best Possible Solution

Keep this as the maintainer-owned broad RFC and define a versioned host-owned or explicitly sandboxed contribution contract before landing implementation slices.
