## Summary

Keep open; current main and v2026.6.8 only expose the narrower Control UI descriptor/session-action foundation, not the broader plugin contribution-slot vocabulary or browser rendering contract requested here.

## What This Changes

Add data-driven Control UI plugin contribution slots for chat modes, approval cards, event classifiers, input guards, and status surfaces.

## Best Possible Solution

Keep this as a maintainer-owned RFC until OpenClaw chooses a versioned, data-only Control UI contribution model with host-owned rendering, explicit scopes, disablement semantics, and security tests.
