## Summary

Keep open: current main and the latest release still document provider lists, provider-owned catalogs, auth probes, and plugin compatibility records, but not the requested provider/model-family/capability lifecycle matrix or provider deprecation policy. The linked Matrix regression is useful motivation, but it was a plugin-specific issue closed by a targeted fix and does not supersede this broader support-policy proposal.

## What This Changes

Request to publish a machine-readable provider compatibility matrix with lifecycle/deprecation policy and surface it in docs plus doctor or models CLI output.

## Best Possible Solution

Approve the support levels and deprecation windows, then add one provider lifecycle registry that generates public docs and concise doctor/models output while keeping live auth/probe health separate.

## Reproduction Assessment

Not applicable. This is a feature and support-policy proposal; docs and source inspection confirm the requested lifecycle matrix/output is absent, but there is no runtime bug to reproduce.

## Solution Assessment

Unclear until maintainers choose the support guarantees. The maintainable path is a single source-of-truth lifecycle registry feeding docs and CLI output without conflating lifecycle policy with auth/probe health or plugin SDK compatibility records.
