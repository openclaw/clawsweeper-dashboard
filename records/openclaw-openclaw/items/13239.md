## Summary

Keep open: current main and v2026.5.12 still do not publish the requested provider/model-family/capability lifecycle matrix, deprecation windows, generated docs, or doctor/models lifecycle output. The closed Matrix plugin regression raised in discussion is useful motivation, but it was plugin-specific and does not supersede this broader provider support-policy proposal.

## What This Changes

This issue requests a machine-readable provider compatibility matrix with lifecycle/deprecation policy, generated docs, and doctor/models CLI surfacing.

## Best Possible Solution

Define maintainer-approved support tiers, then add a lifecycle registry that generates public docs and concise doctor/models output without mixing live probe health with support policy.
