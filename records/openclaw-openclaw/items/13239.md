## Summary

Keep open. Current main still lacks the requested provider lifecycle compatibility matrix, deprecation windows, tracked issue links, and doctor/models lifecycle output; the linked Matrix plugin regression is useful motivation but not a superseding tracker.

## What This Changes

Request to publish a machine-readable provider/model-family/capability compatibility matrix with support statuses, deprecation policy, generated docs, and doctor output.

## Best Possible Solution

Define maintainer-approved support tiers, then add a registry-backed lifecycle matrix that generates public docs and concise doctor/models output without conflating it with live probes.

## Reproduction Assessment

Not applicable: this is a feature and policy proposal, not a failing current behavior. Source and docs inspection on current main confirms the requested lifecycle matrix, deadlines, tracked issue links, and doctor/models lifecycle output are absent.

## Solution Assessment

Unclear because there is no patch and the central blocker is policy shape. A static lifecycle registry feeding docs and CLI output looks maintainable only after support tiers and deprecation windows are defined.
