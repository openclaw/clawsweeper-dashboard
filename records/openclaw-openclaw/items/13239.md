## Summary

Keep open. Current main still lacks the requested provider lifecycle compatibility matrix and deprecation policy, and recent issue discussion already distinguishes this broader policy request from the closed Matrix plugin regression.

## What This Changes

Publish a machine-readable provider/model-family/capability/status compatibility matrix with deprecation policy, generated docs, and doctor or models CLI output.

## Best Possible Solution

Define the support policy first, then add registry-backed lifecycle metadata that generates public docs and concise doctor/models output without conflating it with live probes.

## Solution Assessment
