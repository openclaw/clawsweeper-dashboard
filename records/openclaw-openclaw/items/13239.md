## Summary

Keep open: current main still has provider catalogs, auth/probe status, model row status metadata, and plugin compatibility/deprecation records, but it does not publish the requested provider x model-family x capability lifecycle matrix, deprecation windows, generated docs page, or doctor/models lifecycle output.

## What This Changes

The issue requests a public, machine-readable provider compatibility matrix with lifecycle/deprecation policy and docs plus CLI surfacing.

## Best Possible Solution

Approve a provider lifecycle policy, then add one source-of-truth registry that generates public docs and concise doctor/models output while keeping live auth/probe health separate.

## Solution Assessment

Unclear until maintainers choose the support guarantees. The maintainable direction is a single lifecycle registry feeding docs and CLI output without conflating lifecycle policy with auth/probe health.
