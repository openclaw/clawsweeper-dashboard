## Summary

Keep open. Current main still lacks the requested provider lifecycle compatibility matrix and deprecation policy, and the discussion treats the closed Matrix plugin regression as motivation rather than a superseding tracker.

## What This Changes

Request to publish a provider/model-family/capability compatibility matrix with lifecycle statuses, deprecation windows, generated docs, and doctor/models CLI output.

## Best Possible Solution

Define maintainer-approved support tiers first, then add a registry-backed lifecycle matrix that generates docs and concise doctor/models output without conflating it with live probes.

## Reproduction Assessment

## Solution Assessment
