## Summary

Keep open. Current main has provider setup docs, provider-owned model catalogs, per-model listing status metadata, plugin-contract deprecation policy, and live auth/probe status, but it still does not publish the requested provider x model-family x capability compatibility matrix with lifecycle deadlines, canonical issue links, generated docs, or doctor/models lifecycle output. The latest provided maintainer-side comment also explicitly kept this open, and the Matrix plugin regression is useful motivation but not a superseding tracker for the broader provider policy request.

## What This Changes

Request to publish a machine-readable provider/model-family/capability compatibility matrix with lifecycle statuses, deprecation windows, docs, and CLI doctor or model-status output.

## Best Possible Solution

Keep this issue open until maintainers approve the provider support policy shape, then ship a lifecycle registry keyed by provider, model family, capability, status, deadline, replacement, and canonical issue link, with generated docs and concise CLI lifecycle surfacing separate from live auth/probe health.

Not applicable as a feature/policy request. The current-main verification path is source and docs inspection, which shows the requested matrix and lifecycle policy are still absent.
