## Summary

Keep open: current main has provider docs, catalogs, auth probes, and plugin deprecation records, but not the requested provider/model-family/capability lifecycle matrix or policy-backed CLI/docs output.

## What This Changes

The issue requests a public provider compatibility matrix with lifecycle statuses, deprecation policy, generated docs, and doctor/models CLI surfacing.

## Best Possible Solution

Approve the support levels and deprecation windows, then add one provider lifecycle registry that generates public docs and concise doctor/models output while keeping live auth/probe health separate.

## Solution Assessment
