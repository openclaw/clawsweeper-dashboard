## Summary

Keep open: current main still lacks the requested provider/model/capability lifecycle matrix, deprecation policy, generated docs, and doctor/models lifecycle output, and the support guarantees need maintainer policy approval before implementation.

## Best Possible Solution

Ship a maintainer-approved provider lifecycle registry keyed by provider, model family, capability, status, deadline, replacement, and canonical issue link, then generate docs and concise CLI output from it while keeping live probe health separate.
