## Summary

Keep open: current main and v2026.6.5 still lack the requested public provider/model-family/capability lifecycle matrix, deprecation policy, generated docs page, and doctor/models lifecycle output. Existing model catalogs and probes cover provider setup, auth health, and some per-model metadata, but they do not settle the broader support-guarantee policy requested here.

## What This Changes

The issue requests a machine-readable provider compatibility matrix with lifecycle statuses, deprecation windows, public docs, and CLI doctor visibility.

## Best Possible Solution

Approve a provider lifecycle policy, add a canonical registry keyed by provider/model family/capability/status/deadline/replacement/canonical issue, generate public docs from it, and surface concise lifecycle state in doctor or models status without conflating it with live auth probes.

## Reproduction Assessment

Not applicable as a feature and product-policy request. Source inspection still confirms the requested matrix, deprecation policy, generated docs page, and doctor/models lifecycle output are absent on current main.

## Solution Assessment

Unclear as an implementation until maintainers choose the support policy. The best direction is a canonical lifecycle registry and generated surfaces, not extending live auth probe buckets into support guarantees by accident.
