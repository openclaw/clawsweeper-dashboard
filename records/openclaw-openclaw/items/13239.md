## Summary

## What This Changes

The issue requests a machine-readable provider compatibility matrix with lifecycle/deprecation policy, plus docs and doctor/models surfacing.

## Best Possible Solution

Define a maintainer-approved provider lifecycle metadata contract, generate public docs from it, and surface concise lifecycle state in doctor/models output while keeping support lifecycle separate from live auth/probe health.
