## Summary

Keep open. Current main still lacks the requested provider x model-family x capability lifecycle matrix and deprecation policy; existing surfaces cover provider setup, provider-owned catalogs, plugin compatibility records, and live auth/probe health instead of support guarantees.

## What This Changes

Request to publish a machine-readable provider compatibility matrix with lifecycle statuses, deprecation windows, docs, and CLI doctor/status output.

## Best Possible Solution

Approve the provider lifecycle semantics first, then add a generated provider lifecycle registry and docs/CLI views that stay separate from live probes.

## Solution Assessment
