## Summary

Keep open: current main still has provider catalogs, auth/probe status, and plugin compatibility/deprecation records, but it does not publish the requested provider x model-family x capability lifecycle matrix, deprecation windows, generated docs page, or doctor/models lifecycle output.

## What This Changes

The issue requests a public, machine-readable provider compatibility matrix with lifecycle statuses, deprecation policy, docs, and CLI doctor output.

## Best Possible Solution

Approve a provider lifecycle policy, then add one source-of-truth registry that generates public docs and concise doctor/models output while keeping live auth/probe health separate.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers choose the support guarantees. The maintainable direction is a single lifecycle registry that feeds docs and CLI output without conflating lifecycle policy with auth/probe health.
