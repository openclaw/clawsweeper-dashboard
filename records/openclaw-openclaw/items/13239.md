## Summary

Keep open: current main still has provider lists, provider-owned catalogs, auth probes, and plugin compatibility deprecation records, but not the requested provider/model-family/capability lifecycle matrix or policy-backed docs/CLI output. The Matrix plugin regression linked in discussion is useful motivation, but it is a closed plugin-specific issue and does not supersede this broader provider support-policy proposal.

## What This Changes

The issue proposes a public machine-readable provider compatibility matrix with lifecycle statuses, deprecation windows, docs, and doctor/models CLI output.

## Best Possible Solution

Approve the support levels and deprecation windows, then add one provider lifecycle registry that generates public docs and concise doctor/models output while keeping live auth/probe health separate.
