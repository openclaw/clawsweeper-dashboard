## Summary

Keep open: current main and v2026.6.10 still provide provider lists, provider-owned catalogs, auth/probe diagnostics, and plugin compatibility policy, but not the requested provider/model-family/capability lifecycle matrix with deprecation windows, tracked issue links, generated docs, or doctor/models lifecycle output.

## What This Changes

Request to publish a machine-readable provider compatibility matrix with lifecycle statuses, deprecation policy, public docs, and doctor output.

## Best Possible Solution

Define a maintainer-approved provider lifecycle metadata contract, generate public docs from it, and surface concise lifecycle state in doctor/models output while keeping support lifecycle separate from live auth/probe health.

## Reproduction Assessment

Not applicable: this is a provider support-policy and reporting feature request, not a failure of existing documented behavior. Source, docs, latest-release, and related-item checks show the requested matrix is still missing.

## Solution Assessment

Yes in direction, but not as a small automated docs-only patch. The maintainable solution is a provider lifecycle registry owned separately from auth probes, model catalog availability, and plugin compatibility records.
