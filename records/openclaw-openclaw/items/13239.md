## Summary

Keep open. Current main still does not publish the requested provider lifecycle compatibility matrix or deprecation policy, and the issue discussion includes recent maintainer-side signal to keep this broader policy proposal open.

## What This Changes

The issue requests a machine-readable provider/model-family/capability compatibility matrix with lifecycle statuses, deprecation windows, generated docs, and CLI doctor visibility.

## Best Possible Solution

Define the provider support policy first, then ship a registry-backed generated docs page and concise doctor/models lifecycle output separate from live auth/probe health.

## Reproduction Assessment

Not applicable as a feature and policy request. Source and docs inspection on current main verifies that the requested matrix, lifecycle deadlines, canonical issue links, and CLI lifecycle output are absent.

## Solution Assessment

Unclear because there is no proposed patch yet and the central blocker is policy shape. The maintainable direction is likely a registry-backed lifecycle surface that reuses existing catalog metadata only where it fits without conflating it with live probes.
