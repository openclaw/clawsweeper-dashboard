## Summary

Keep open. Current main still does not provide the requested provider/model-family/capability compatibility matrix, public deprecation policy, generated docs page, or doctor/models lifecycle output; the existing model catalog status fields are only partial per-model listing metadata and the linked Matrix/plugin-compatibility reports do not supersede this broader policy proposal.

Proposal to publish a machine-readable provider compatibility matrix with lifecycle statuses, deprecation windows, docs output, and doctor/CLI surfacing.

## Best Possible Solution

Define maintainer-approved support tiers, then add a lifecycle registry that generates public docs and concise doctor/models output without conflating policy with live probes.
