## Summary

Keep open: current main still treats Skill Workshop as workspace-scoped, with only a narrow opt-in symlink write-through path, and an open PR only covers the managed-root slice. The requested named/shared/extra-root targeting plus cross-workspace promotion remains security-sensitive product work.

## What This Changes

Request Skill Workshop support for proposing updates to shared or configured skill roots and promoting repeated cross-workspace patterns into shared skill candidates.

## Best Possible Solution

Split the work into reviewed stages: settle or land the managed-root PR if maintainers approve it, then design explicit named writable targets with manual-only shared applies before considering a separate opt-in cross-workspace promotion reviewer.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, the latest release tag, and the open partial PR give high confidence that the requested shared/named/extra-root and cross-workspace flows are not fully available today.

## Solution Assessment

Unclear. Managed-root support in the open PR is a plausible first slice, but the full request needs a target model and security/product decision before implementation should proceed.
