## Summary

Keep open. Current main still implements Skill Workshop as workspace-scoped, and the requested shared-skill targeting plus cross-workspace promotion would expand shared write authority and cross-workspace data flow, so it needs maintainer product/security design rather than cleanup closure.

## What This Changes

Request Skill Workshop support for opt-in shared skill proposals, cross-workspace pattern promotion, and a `promote_to_shared` action.

## Best Possible Solution

Stage this as an opt-in Skill Workshop design with explicit target metadata, separated proposal storage, strict shared-root validation, manual-only shared applies, docs, and tests; defer semantic cross-workspace promotion until privacy/security rules are approved.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Static inspection gives high confidence in the current limitation: source, docs, tests, tool schema, reviewer, and store are all workspace-scoped.

## Solution Assessment

Unclear. Shared targeting is plausible as a narrow v1, but the full A/B/C proposal crosses shared write and cross-workspace privacy boundaries, so maintainers should approve the staged design before a fix PR is queued.
