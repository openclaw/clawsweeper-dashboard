## Summary

Keep open. Current main still limits Skill Workshop proposals, review context, storage, tool actions, and writes to workspace skills, and the requested shared-skill targeting plus cross-workspace promotion would expand write authority and review scope in a security-sensitive bundled plugin that needs maintainer product/security direction first.

## What This Changes

Request opt-in Skill Workshop support for shared-skill targeting, cross-workspace pattern detection, and manual promotion from workspace skills into shared skill candidates.

## Best Possible Solution

Stage this inside `extensions/skill-workshop`: first design explicit target/root metadata, strict shared-root validation, separated proposal storage/audit data, manual-only shared applies, docs, and focused tests; then treat cross-workspace semantic promotion as a follow-up after privacy and retention rules are approved.

## Solution Assessment

Unclear. Shared targeting is plausible, but the proposed A/B/C package crosses shared write and cross-workspace privacy boundaries, so the safer path is to split the design and get maintainer product/security approval before coding.
