## Summary

Keep open. Current main still implements Skill Workshop as a workspace-only proposal, review, storage, and write flow, while the requested shared-skill targeting and cross-workspace promotion would expand shared write authority and cross-workspace data review in a bundled plugin that needs maintainer product/security direction.

## What This Changes

Request opt-in Skill Workshop support for targeting shared skills, detecting similar cross-workspace skill patterns, and promoting workspace skills into shared skill proposals while preserving approval and scanning safeguards.

## Best Possible Solution

Stage this inside `extensions/skill-workshop`: first design shared-target metadata, strict shared-root validation, per-agent opt-in, separated proposal storage/audit data, manual-only shared applies, docs, and focused tests; then handle cross-workspace semantic promotion after privacy rules are approved.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Static inspection gives high confidence in the current limitation: the code, docs, tool schema, reviewer, and store are all workspace-scoped on current main.

## Solution Assessment

Unclear. Opt-in shared targeting is plausible, but the full A/B/C proposal crosses shared write and cross-workspace privacy boundaries, so the safer path is to split the design and get maintainer product/security approval before implementation.

## Security Review

Status: needs_attention

Summary: No patch was reviewed, but the requested feature is security-sensitive because it expands Skill Workshop from per-workspace writes into shared-skill writes and cross-workspace review.

Concerns:

- **[medium] Define shared-skill write authority:** `extensions/skill-workshop/src/skills.ts:38`
  - body: Current Skill Workshop writes only under the active workspace. Allowing an agent to target shared managed skills would let that agent change procedures visible to other agents, so the design needs explicit target metadata, strict root validation, separated store/audit data, and manual-approval-only shared applies.
  - confidence: 0.91
- **[medium] Define cross-workspace reviewer privacy limits:** `extensions/skill-workshop/src/reviewer.ts:183`
  - body: The current reviewer reads only the active workspace's skills and transcript slice. A cross-workspace reviewer could expose workspace-specific procedures or transcript-derived context unless operator opt-in, evidence minimization, and retention limits are specified.
  - confidence: 0.86
- **[low] Separate shared proposal audit state:** `extensions/skill-workshop/src/store.ts:20`
  - body: The current store is keyed by workspace path only. Shared-targeted proposals need distinct target/root identity and provenance so workspace and shared proposals cannot collide or lose audit context.
  - confidence: 0.84
