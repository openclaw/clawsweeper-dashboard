## Summary

Keep open: current main and v2026.5.7 still keep Skill Workshop writes, proposal storage, and reviewer context workspace-scoped, while shared targeting and cross-workspace promotion are new security-sensitive capabilities that need maintainer product direction.

## What This Changes

The issue requests opt-in Skill Workshop support for shared-skill targeting, cross-workspace pattern detection, and a `promote_to_shared` action.

## Best Possible Solution

Stage this as an opt-in Skill Workshop design with explicit target metadata, isolated proposal stores, strict shared-root validation, and manual-only shared applies; defer cross-workspace promotion until privacy and security policy are approved.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and release inspection give high confidence that the current limitation exists: Skill Workshop writes, stores, and reviews only workspace skills today.

## Solution Assessment

Unclear. Shared targeting is plausible, but the full proposal crosses write-authority and cross-workspace data-flow boundaries, so the maintainable solution needs a split design and maintainer product/security approval before implementation.
