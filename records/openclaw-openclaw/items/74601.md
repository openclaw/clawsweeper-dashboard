## What This Changes

Request Skill Workshop support for opt-in shared/custom skill targets and cross-workspace promotion flows.

## Best Possible Solution

Establish a maintainer-approved writable target registry and approval model, then land scoped phases with target-aware records, containment tests, docs, and a separate privacy review for automatic cross-workspace promotion.

## Reproduction Assessment

Yes, for a source-level feature gap: current main creates and applies Skill Workshop proposals under the active workspace and rejects non-workspace skill sources. No runtime bug reproduction is needed because this is a request for new opt-in behavior.

## Solution Assessment

Unclear as a final product path. A managed-root slice is plausible, but the broader request should use a maintainer-approved writable target registry and explicit approval/security semantics rather than an ad hoc shared-root toggle.
