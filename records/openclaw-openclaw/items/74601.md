## Summary

Keep open. Current main still scopes Skill Workshop proposals, reviewer context, proposal storage, and writes to the active workspace, and the requested shared-skill targeting plus cross-workspace promotion would expand a bundled plugin's shared write authority and cross-workspace data flow.

## What This Changes

The issue requests opt-in Skill Workshop support for shared-skill proposals, cross-workspace pattern promotion, and a promote-to-shared action.

## Best Possible Solution

Stage this as an opt-in Skill Workshop design with explicit shared-target metadata, separated proposal storage, strict shared-root validation, manual-only shared applies, docs, and tests; defer semantic cross-workspace promotion until privacy/security rules are approved.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Static inspection gives high confidence in the current limitation: source, docs, tests, tool schema, reviewer, and store are all workspace-scoped.

## Solution Assessment

Unclear. Shared targeting is plausible, but the full A/B/C proposal crosses shared write and cross-workspace privacy boundaries, so the maintainable path is a split design with maintainer product/security approval before implementation.
