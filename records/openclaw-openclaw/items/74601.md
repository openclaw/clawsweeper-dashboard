## Summary

Keep open: current main still keeps Skill Workshop proposals, review context, storage, and writes scoped to one workspace, while the requested shared targeting and cross-workspace promotion would expand write authority beyond the existing safety model.

## What This Changes

Add opt-in Skill Workshop support for shared-skill targeting, cross-workspace pattern promotion, and a manual promote-to-shared action.

## Best Possible Solution

Stage this as an opt-in Skill Workshop design with explicit target metadata, isolated proposal stores, strict shared-root validation, and manual-only shared applies; add cross-workspace promotion only after privacy and security policy are approved.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Static inspection gives high confidence in the current limitation: source, docs, tests, tool schema, reviewer, and store are all workspace-scoped.

## Solution Assessment

Unclear. Shared targeting is plausible, but the full proposal crosses shared write and cross-workspace privacy boundaries, so the maintainable path is a split design with maintainer product/security approval before implementation.
