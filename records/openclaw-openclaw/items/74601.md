## Summary

Keep open: current main and v2026.6.10 still keep Skill Workshop proposals scoped to workspace/project-agent skill writes, while this feature asks for shared, extraDir/named-target, and cross-workspace promotion behavior that needs maintainer product and security review.

## What This Changes

The issue requests Skill Workshop support for targeting shared or configured skill roots and promoting cross-workspace patterns into shared skill candidates.

## Best Possible Solution

Define a maintainer-approved writable target registry with explicit approval text, target-aware proposal records, containment tests, docs, and a separate privacy review before any automatic cross-workspace promotion work lands.

## Reproduction Assessment

Yes, source-level: current main creates Skill Workshop targets under the active workspace and rejects non-workspace skill sources, while tool, protocol, and config schemas lack shared/named target or promotion support.

## Solution Assessment

Unclear as a final product path. A managed-root slice is plausible, but the broader request should use a maintainer-approved writable target registry and explicit approval/security semantics rather than a hard-coded shared-root toggle.
