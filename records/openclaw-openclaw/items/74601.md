## Summary

Keep open: current main and the latest release still keep Skill Workshop’s live writes workspace-scoped except for the narrow trusted-symlink write-through path, while https://github.com/openclaw/openclaw/pull/92733 only covers the managed-root slice. The extraDir/named-target and cross-workspace promotion comments add real remaining scope, and the write authority/privacy model needs maintainer product and security review.

## What This Changes

Request to let Skill Workshop propose, review, and apply shared, named, extra-root, or cross-workspace skill updates instead of only workspace-local skills.

## Best Possible Solution

Stage this behind maintainer-approved design: land or close the managed-root slice on its own merits, then design explicit named writable targets with manual shared-impact approval before separately considering opt-in cross-workspace promotion.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source, docs, release inspection, and the open partial PR give high confidence that the requested shared/named/extra-root and cross-workspace flows are not fully available today.

## Solution Assessment

Unclear. Managed-root support is a plausible first slice, but the full requested direction needs a target model, approval semantics, and cross-workspace privacy policy before implementation should proceed.
