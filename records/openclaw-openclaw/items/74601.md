## Summary

Keep open: current main still scopes Skill Workshop writes, proposals, and reviewer context to one workspace, and the requested shared-skill targeting plus cross-workspace promotion is a new security-sensitive feature that needs maintainer product direction.

## What This Changes

Request opt-in Skill Workshop support for shared-skill proposals, cross-workspace pattern detection, and manual promotion from workspace skills to shared skills.

## Best Possible Solution

Stage this as an opt-in Skill Workshop design with explicit target metadata, separated shared/workspace proposal stores, strict shared-root validation, and manual-only shared applies; defer cross-workspace promotion until privacy and security policy are approved.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and release inspection give high confidence that Skill Workshop writes, stores, and reviews only workspace skills today.

## Solution Assessment

Unclear. Shared targeting and manual promotion are plausible, but the full proposal crosses write-authority and cross-workspace data-flow boundaries, so the maintainable solution needs split design and maintainer product/security approval first.
