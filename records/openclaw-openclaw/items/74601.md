## Summary

Keep open: current main still treats Skill Workshop as a governed workspace-skill flow with only a narrow trusted-symlink escape hatch, while the issue asks for broader shared, named extra-root, and cross-workspace promotion behavior that needs maintainer product and security decisions.

## What This Changes

Request opt-in Skill Workshop support for shared or named writable skill targets, plus cross-workspace promotion of repeated skill patterns.

## Best Possible Solution

Decide the approved writable target model first, then land scoped phases with target-aware approval copy, containment tests, docs, and no automatic cross-workspace reviewer until its privacy rules are settled.

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence current-main check: new proposals target `<workspace>/skills`, updates reject non-writable sources such as managed and extra roots, and the tool has no target or promotion action.
