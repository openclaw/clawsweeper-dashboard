## Summary

Keep open: current main still scopes Skill Workshop proposals, stores, reviewer context, and writes to one workspace, while the requested shared-skill targeting and cross-workspace promotion would expand plugin authority and needs maintainer product/security design.

## What This Changes

Request adds opt-in shared-skill targeting, cross-workspace pattern promotion, and a promote-to-shared action to the Skill Workshop plugin.

## Best Possible Solution

Design this as a staged, opt-in Skill Workshop expansion with explicit target metadata, isolated proposal state, strict shared-root validation, manual-only shared applies, and later cross-workspace promotion after privacy policy is settled.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Static inspection gives high confidence in the current limitation: source, manifest, docs, reviewer, store, and tool schema are all workspace-scoped.

## Solution Assessment

Unclear. Shared targeting is plausible, but the full proposal crosses shared write authority and cross-workspace privacy boundaries, so the safest maintainable path is a split design reviewed by maintainers before implementation.
