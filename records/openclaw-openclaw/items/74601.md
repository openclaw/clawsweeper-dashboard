## Summary

Keep open because current main still scopes Skill Workshop proposals, review context, storage, and writes to the active workspace, while the requested shared-skill targeting and cross-workspace promotion add new config plus security-sensitive product behavior.

## What This Changes

Request opt-in Skill Workshop support for shared-skill targets, cross-workspace pattern promotion, and a `promote_to_shared` action.

## Best Possible Solution

Stage this as an opt-in Skill Workshop design with explicit target metadata, separated proposal storage, strict shared-root validation, manual-only shared applies, docs, and tests; treat semantic cross-workspace promotion as a follow-up after privacy and security rules are approved.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Static inspection gives high confidence in the current limitation: source, docs, tests, tool schema, reviewer, and store are all workspace-scoped.

## Solution Assessment

Unclear. Shared targeting and explicit promotion are plausible, but the full proposal crosses shared-write and cross-workspace privacy boundaries, so the maintainable path is a split design with maintainer product/security approval before implementation.
