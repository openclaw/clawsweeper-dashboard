## Summary

Keep open. Current main still scopes Skill Workshop proposals, reviewer context, proposal storage, and writes to the active workspace, while the requested shared-skill targeting and cross-workspace promotion would expand shared write authority and cross-workspace data flow in a bundled plugin.

## What This Changes

Request opt-in Skill Workshop support for proposing shared-skill updates, detecting repeated patterns across workspaces, and promoting workspace skills into shared skill candidates.

## Best Possible Solution

Stage this in Skill Workshop as an opt-in shared-target design with explicit target/root metadata, separated proposal storage, strict shared-root validation, manual-only shared applies, docs, and tests; handle semantic cross-workspace promotion only after privacy/security rules are approved.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Static inspection gives high confidence in the current limitation: source, docs, tests, tool schema, reviewer, and store are all workspace-scoped.

## Solution Assessment

Unclear. Shared targeting is plausible, but the proposed A/B/C bundle crosses shared write and cross-workspace privacy boundaries, so the maintainable path is a split design with maintainer product/security approval before implementation.
