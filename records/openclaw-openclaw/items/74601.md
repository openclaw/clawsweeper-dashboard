## Summary

Keep open. Current main still scopes Skill Workshop proposals, writes, reviewer context, and proposal storage to the active workspace; the requested shared-skill targeting and cross-workspace promotion are not implemented or superseded and need maintainer product/security direction.

## What This Changes

Request to let Skill Workshop propose changes to shared skills and promote repeated workspace patterns into shared skill candidates.

## Best Possible Solution

Stage this as a split, opt-in Skill Workshop design: shared target/root metadata and manual-only shared applies first, then cross-workspace promotion only after privacy and security rules are approved.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Static inspection gives high confidence in the current limitation: source, docs, tests, tool schema, reviewer, and store are all workspace-scoped.

## Solution Assessment

Unclear. Shared targeting and explicit promotion are plausible, but the full A/B/C proposal crosses shared write authority and cross-workspace privacy boundaries, so maintainer design review should set the scope before implementation.
