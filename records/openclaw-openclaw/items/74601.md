## Summary

Keep open: current main and v2026.5.7 still keep Skill Workshop writes, proposal storage, and reviewer context workspace-scoped, while this request adds new shared-write authority and cross-workspace review policy that needs maintainer product/security direction.

## What This Changes

Allow the Skill Workshop plugin to opt into shared-skill targeting, detect cross-workspace patterns, and promote workspace skills into shared skill proposals.

## Best Possible Solution

Stage this as an opt-in Skill Workshop design with explicit target metadata, isolated proposal stores, strict shared-root validation, and manual-only shared applies; defer cross-workspace promotion until privacy and security policy are approved.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and v2026.5.7 inspection give high confidence that Skill Workshop currently writes, stores, and reviews proposals only against the active workspace.

## Solution Assessment

Unclear. Shared targeting is plausible, but the full proposal crosses write-authority and cross-workspace data-flow boundaries, so the maintainable solution needs a split design and maintainer product/security approval before implementation.
