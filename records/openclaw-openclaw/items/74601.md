## Summary

Keep open. Current main still scopes Skill Workshop writes, proposal storage, review context, and tool actions to the active workspace; the requested shared-skill targeting and cross-workspace promotion remain real feature work and need maintainer product/security direction before implementation.

## What This Changes

Request opt-in Skill Workshop support for proposing shared-skill updates, promoting workspace skills into shared skill candidates, and detecting similar proposals across workspaces.

## Best Possible Solution

Stage this inside the Skill Workshop plugin: first add explicit target/root metadata, separated workspace-vs-shared proposal storage, strict shared-root validation, manual-only shared applies, docs, and focused tests; then treat cross-workspace semantic promotion as a follow-up after privacy and retention rules are approved.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Static inspection gives high confidence in the current limitation because source, docs, tests, tool schema, reviewer, and store are all workspace-scoped.

## Solution Assessment

Unclear. Opt-in shared targeting is plausible, but the full A/B/C proposal crosses shared write and cross-workspace privacy boundaries, so the safer path is a staged maintainer-approved design rather than an immediate automated fix.
