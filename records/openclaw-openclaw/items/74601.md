## Summary

Keep open. Current main still scopes Skill Workshop proposals, reviewer context, storage, and writes to workspace skills only, while the requested shared-skill targeting and cross-workspace promotion would expand a bundled experimental plugin across shared write authority and workspace privacy boundaries that need maintainer product/security direction.

## What This Changes

Request to extend Skill Workshop so opt-in agents can propose shared-skill updates and promote repeated workspace patterns into shared skill candidates.

## Best Possible Solution

Stage this inside the Skill Workshop plugin only after maintainer direction: add explicit target metadata, strict shared-root validation, separated proposal storage/audit data, manual-only shared applies, docs, and focused tests first; handle semantic cross-workspace promotion later once the privacy model is settled.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Static inspection gives high confidence in the current limitation because source, docs, tests, tool schema, reviewer, and store are all workspace-scoped on current main.

## Solution Assessment

Unclear. Shared targeting and an explicit promote action are plausible, but the proposed package combines narrow target expansion with broader cross-workspace review, so the maintainable path should split the work and get maintainer product/security approval first.
