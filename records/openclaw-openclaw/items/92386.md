## Summary

This is a valid current-release WebChat regression that remains present on current main. A focused, proof-backed repair already exists, but that PR is currently conflicted, so the issue should remain open until the repair is refreshed and merged or replaced.

## What This Changes

Prevent new and streaming WebChat messages from overriding a user’s deliberate backscroll position.

## Best Possible Solution

Refresh the canonical repair onto current main, retain its explicit manual follow-release and bottom-only reacquisition state, preserve all existing auto-scroll modes, and land focused regression tests without release-owned changelog edits.

## Reproduction Assessment

Yes, from source inspection with high confidence: incoming updates invoke the scroll scheduler, and current follow state can remain engaged through deliberate upward user input. The reporter also supplies concrete v2026.6.5 steps, although this read-only review did not execute a browser reproduction.

## Solution Assessment

Yes in design: an explicit manual follow lock models user intent more safely than shrinking the shared near-bottom threshold, which also participates in initial-load, content-growth, and smooth-scroll behavior. The existing branch needs a current-main refresh before it is the best landing vehicle.
