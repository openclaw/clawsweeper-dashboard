## Summary

Keep this open: current `main` and the latest release still have the broad near-bottom auto-follow path, and an open closing PR already owns the narrow follow-lock repair.

## What This Changes

The issue reports that WebChat new messages and streaming chunks force-scroll the conversation to the bottom after the user has scrolled up.

## Best Possible Solution

Review and land the focused open follow-lock repair, then close this issue as implemented and resolve the overlapping WebChat scroll PR path.

## Reproduction Assessment

Yes. From source inspection, streaming updates call `scheduleChatScroll`, and current near-bottom mode can keep auto-follow enabled after a deliberate upward scroll that remains inside the broad 450px threshold; I did not run a live browser repro in this read-only review.

## Solution Assessment

Yes in direction. The linked follow-lock repair is the narrowest current path because it changes the central scroll helper, preserves existing auto-scroll modes, and directly prevents streaming from reacquiring follow until the user returns to the bottom.
