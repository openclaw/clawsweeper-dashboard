## Summary

Keep open: current main and latest release still lack the follow-lock behavior needed to preserve WebChat backscroll during streaming, and the open closing PR at https://github.com/openclaw/openclaw/pull/92622 now owns the narrow fix path.

## What This Changes

WebChat should preserve the user's backscroll when new messages or streaming chunks arrive instead of forcing the conversation to the bottom.

## Best Possible Solution

Review and land one focused WebChat follow-lock fix, preferably https://github.com/openclaw/openclaw/pull/92622 if it remains the narrower clean branch, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes from source inspection: current main schedules chat scrolling on streaming changes and has no follow-lock state to keep deliberate upward user scrolls from re-enabling near-bottom auto-follow. I did not run a live WebChat stream in this read-only review.

## Solution Assessment

Yes in direction: the best fix is a narrow state-machine change in the central WebChat scroll helper that preserves existing auto-scroll modes while locking follow after user upward intent. The existing open PR is the current canonical implementation candidate, with the older overlapping PR needing maintainer selection or retirement.
