## Summary

Keep open: current main and v2026.6.8 still have prompt-marker external-content wrapping and blocking hooks, but no structured external-content provenance/action source at the before_tool_call gate. https://github.com/openclaw/openclaw/pull/91800 is an open partial implementation candidate, and the remaining API, security, policy, and compatibility semantics need maintainer review.

## What This Changes

The issue proposes source-aware instruction tracking and tool-execution gating so external content can inform responses but cannot authorize tool calls.

## Best Possible Solution

Define a coordinated provenance and tool-policy contract that tags external content at ingress, preserves enough source context through model/tool decisions, exposes stable hook or approval fields, and enforces risky content-derived actions through explicit operator-controlled policy.

## Reproduction Assessment

Yes at source level, not as a live exploit reproduction. Current main's hook type, event builder, input provenance helper, external-content wrapper, and latest release tag show no external-content action-source field or execution gate.

## Solution Assessment

Unclear as a direct implementation. The security direction is aligned, but a simple action.source denial is too narrow until maintainers define provenance fidelity, plugin API, approval behavior, policy exceptions, and upgrade compatibility.
