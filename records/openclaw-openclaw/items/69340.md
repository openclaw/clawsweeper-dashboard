## Summary

Keep this PR open, but do not merge it as-is. Current main still has the linked hyphenated-token rendering gap, while the one-line guard also makes dash-only long runs copy-sensitive and bypasses the chunking that protects narrow TUI output.

## What This Changes

The PR changes `src/tui/tui-formatters.ts` so `isCopySensitiveToken` treats file-like tokens containing hyphens as copy-sensitive.

## Best Possible Solution

Land a narrower TUI formatter fix that preserves package/entity-style hyphenated identifiers while retaining chunking for punctuation-only or prose-like long tokens, with regression tests and a changelog entry.

## Reproduction Assessment

Yes. With the PR condition, a token such as `-`.repeat(60) matches `FILE_LIKE_RE`, contains `-`, and bypasses `chunkToken`; current tests show generic long tokens are expected to stay within 32-character display segments.

## Solution Assessment

No as written. The maintainable fix should classify genuinely copy-sensitive hyphenated identifiers more narrowly instead of making every long `[A-Za-z0-9._-]` token containing a hyphen unwrapped.
