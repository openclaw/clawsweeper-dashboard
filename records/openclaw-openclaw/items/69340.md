## Summary

Keep this PR open, but do not merge it as written: current main still has the hyphenated-token copy/paste bug, while the proposed one-line guard also disables long-token chunking for punctuation-only hyphen runs.

## What This Changes

The PR changes `src/tui/tui-formatters.ts` so file-like tokens containing hyphens as well as underscores are treated as copy-sensitive during TUI render sanitization.

## Best Possible Solution

Preserve genuine long hyphenated identifiers in TUI output with a narrower classifier, keep dash-only and prose-like long tokens chunkable, and add regression tests for both preserved and still-wrapped cases.

## Reproduction Assessment

Yes. The current-main code path shows long hyphenated package-like tokens are chunked because only `_` is protected, and the PR blocker is reproducible by applying the proposed condition to `"-".repeat(60)`, which then bypasses `chunkToken`.

## Solution Assessment

No as written. The maintainable fix should distinguish real copy-sensitive hyphenated identifiers from punctuation-only or prose-like long tokens instead of treating every `[A-Za-z0-9._-]+` token containing `-` as unbreakable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep generic hyphen runs chunkable:** `src/tui/tui-formatters.ts:82`
  - body: This guard treats every long token containing `-` and only `[A-Za-z0-9._-]` as copy-sensitive. That includes strings like `"-".repeat(60)`, which currently get split by `normalizeLongTokenForDisplay` to protect narrow terminals; after this change they bypass `chunkToken` and can exceed the formatter's 32-character guardrail. Please narrow the hyphen heuristic and add regression coverage for dash-only or prose-like tokens that must still wrap.
  - confidence: 0.88
