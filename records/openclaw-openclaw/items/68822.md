## Summary

Keep open: the feature request is still relevant, but this PR is not merge-ready because it reads an unvalidated top-level config path instead of the resolved agent-scoped memorySearch contract, and it also commits an unrelated runtime log. Current main already covers non-batch concurrency via `remote.nonBatchConcurrency`, but retry/backoff knobs remain hardcoded.

## What This Changes

The PR exports memory embedding retry/concurrency defaults, adds overridable getters, reads proposed `memorySearch.embedding.*` config values in memory-core, and adds a root runtime log file.

## Best Possible Solution

Revise the feature against the existing `agents.defaults.memorySearch` contract: keep or explicitly reconcile `remote.nonBatchConcurrency`, add typed and validated retry/backoff settings through resolver/schema/help/docs, remove the runtime log, add focused tests, and include redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection is enough for the review blockers: current main has a strict agent-scoped memorySearch schema with no `embedding` object, while the PR reads `cfg.memorySearch.embedding.*` and adds a generated log file.

## Solution Assessment

No. Configurable retry/backoff is a reasonable direction, but this patch bypasses the resolved memory config path and proposes keys that users cannot reliably set under the current schema; the safer solution is to add the knobs to `ResolvedMemorySearchConfig` and the validated `agents.*.memorySearch` surface.
