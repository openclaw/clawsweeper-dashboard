## Summary

Keep open. The underlying request is still valid because retry/backoff remains hardcoded on current main, but this PR is not merge-ready: it reads an unvalidated top-level config path, duplicates/replaces the existing non-batch concurrency contract, lacks changelog coverage, and includes an unrelated runtime log file.

## What This Changes

This PR exports memory embedding default constants, adds overridable retry/concurrency getters that read `cfg.memorySearch.embedding.*`, and adds a root `openclaw-2026-04-15.log` file.

## Best Possible Solution

Revise the feature against the existing agent-scoped memory config contract: keep or explicitly reconcile `remote.nonBatchConcurrency`, add typed and validated retry/backoff settings through the resolver/schema/help/docs path, remove the runtime log, add focused config and retry tests, and include a user-facing changelog entry.

## Reproduction Assessment

Yes. Code inspection is enough: current main still passes fixed retry/backoff constants into the embedding retry loop, while the PR's new `cfg.memorySearch.embedding` path is absent from the strict agent-scoped memory config schema.

## Solution Assessment

No. Configurable retry/backoff is a reasonable direction, but this patch bypasses the resolved config path and introduces keys users cannot reliably set; the safer solution is to add the knobs to `ResolvedMemorySearchConfig` and the validated `agents.*.memorySearch` schema.
