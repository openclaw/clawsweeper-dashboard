## Summary

Keep open. This external PR is a plausible feature direction, but the current patch bypasses the validated agent-scoped memory config contract and includes an unrelated generated runtime log; current main only covers the concurrency part through `remote.nonBatchConcurrency`, not retry/backoff tuning.

## What This Changes

The PR exports memory embedding retry/concurrency defaults, adds overridable getters that read proposed `memorySearch.embedding.*` values, and adds a root runtime log file.

## Best Possible Solution

Rework the feature against the existing `agents.defaults.memorySearch` contract: keep `remote.nonBatchConcurrency`, add typed and validated retry/backoff settings to resolved config/docs/tests, remove the log, and include redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection is enough for the review blockers: current main has a strict agent-scoped memorySearch schema with no `embedding` object, while the PR reads `cfg.memorySearch.embedding.*` and adds a generated log file.

## Solution Assessment

No as-is. Configurable retry/backoff is reasonable, but the maintainable path is to extend `ResolvedMemorySearchConfig` and the validated `agents.*.memorySearch` surface instead of adding an untyped top-level config read.
