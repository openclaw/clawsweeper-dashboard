## Summary

Keep open. The earlier getter-call bug was fixed, but this PR still wires new knobs through an untyped, non-canonical config path, commits an unrelated runtime log, and lacks contributor real-behavior proof.

## What This Changes

The PR exports memory embedding retry/concurrency defaults, adds overridable getters reading proposed `memorySearch.embedding.*` config values, and adds a generated root log file.

## Best Possible Solution

Rework this against the existing `agents.defaults.memorySearch` contract: keep `remote.nonBatchConcurrency`, add typed and validated retry/backoff settings to resolved config/docs/tests, remove the log, and include redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection is enough for the review blockers: current main rejects top-level `memorySearch`, resolves agent-scoped memory config through `ResolvedMemorySearchConfig`, and the PR reads a schema-less top-level `memorySearch.embedding.*` path while adding a generated log file.

## Solution Assessment

No as-is. Configurable retry/backoff is reasonable, but the maintainable solution is to extend the validated resolved memorySearch contract rather than adding an untyped top-level config read.
