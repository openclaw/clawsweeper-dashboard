## Summary

Keep open: this active external PR still has blocking config-contract and generated-log issues, and current main only implements non-batch concurrency, not retry/backoff tunability.

## What This Changes

The PR exports embedding retry/concurrency defaults, adds MemoryIndexManager overrides that read proposed `memorySearch.embedding.*` values, and adds a root runtime log file.

## Best Possible Solution

Add typed retry/backoff settings to the existing agent-scoped memorySearch contract, keep the shipped non-batch concurrency path, update docs/tests, remove the log, and include redacted real behavior proof.

## Reproduction Assessment

Yes for the review blockers. Source inspection shows current main has no retry/backoff config fields and the PR reads an unvalidated config path while adding a generated log file.

## Solution Assessment

No as-is. Configurable retry/backoff is reasonable, but the maintainable fix should extend the typed resolved `agents.*.memorySearch` schema/resolver/docs/tests instead of adding untyped top-level `memorySearch.embedding` reads.
