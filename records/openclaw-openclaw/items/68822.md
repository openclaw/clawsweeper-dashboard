## Summary

Keep open: the requested retry/backoff configurability is still useful, but this PR is not merge-ready because it wires the knobs to an invalid/unresolved config path, commits a runtime log, lacks real behavior proof, and is currently conflicting.

## What This Changes

The PR exports embedding retry/concurrency defaults, routes retry/backoff and index concurrency through overridable getters, reads new `memorySearch.embedding.*` values from config, and adds a generated runtime log.

## Best Possible Solution

Extend the validated agent-scoped `memorySearch` schema/resolver/types/help/docs/tests with retry/backoff settings, keep the existing `remote.nonBatchConcurrency` path for concurrency, remove the log, and require redacted real-run proof before merge.

## Reproduction Assessment

Yes. Source inspection on current main and the PR diff shows retry/backoff constants are still hardcoded, while the PR's new config path is not accepted by the strict config contract and bypasses the resolved `this.settings` path.

## Solution Assessment

No as-is. Configurable retry/backoff is reasonable, but the maintainable solution is to extend the existing resolved `memorySearch` contract instead of adding a top-level untyped config branch, and the runtime log must be removed.
