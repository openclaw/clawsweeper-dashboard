## Summary

Keep open. This external PR is still a plausible implementation candidate for the open retry/backoff configurability request, but the current branch is not mergeable as-is because it reads an unvalidated top-level config path, includes an unrelated generated runtime log, and lacks after-fix real behavior proof.

## What This Changes

The PR exports memory embedding retry/concurrency defaults, adds overridable embedding tuning getters, reads proposed `memorySearch.embedding.*` values, and adds a root runtime log file.

## Best Possible Solution

Rework the feature through the existing agent-scoped `memorySearch` contract: keep `remote.nonBatchConcurrency`, add typed and validated retry/backoff settings with docs/tests, remove the log, and include redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection is enough for the review blockers: the PR reads a top-level untyped config path that current main rejects, and the branch adds a generated runtime log.

## Solution Assessment

No as-is. Configurable retry/backoff is reasonable, but the maintainable path is to extend the resolved, validated `agents.*.memorySearch` contract rather than adding an untyped competing config path.
