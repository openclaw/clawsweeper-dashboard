## Summary

Keep open. The linked feature is still meaningful because retry/backoff settings remain hardcoded on current main and v2026.5.12, but this PR is not mergeable as-is because it bypasses the resolved memory config contract, adds an unrelated runtime log, and still lacks real behavior proof.

## What This Changes

The PR exports memory embedding retry/concurrency defaults, adds getter-based overrides in memory-core, reads `cfg.memorySearch.embedding.*`, and adds a generated runtime log.

## Best Possible Solution

Extend the existing agent-scoped `memorySearch` schema, resolver, generated types/help, docs, and tests with typed retry/backoff settings, keep or reuse `remote.nonBatchConcurrency` for concurrency, remove the log, and then land or replace this PR.

## Reproduction Assessment

Yes, for the review blockers and remaining feature gap. Source inspection shows the PR reads an untyped top-level config path while current main accepts only the resolved agent-scoped memory config, and current main/v2026.5.12 still pass hardcoded retry/backoff constants into the embedding retry loop.

## Solution Assessment

No as-is. Configurable retry/backoff is reasonable, but the narrow maintainable fix is to extend the existing resolved `memorySearch` contract instead of adding a competing config path and committing a runtime log.
