## Summary

Keep open. The retry/backoff configurability is still useful, but this PR is not merge-ready because it wires new knobs through a legacy/untyped config path, includes an unrelated generated runtime log, and still lacks real behavior proof.

## What This Changes

The PR exports memory embedding retry/concurrency defaults, adds protected tuning getters, reads `cfg.memorySearch.embedding.*`, and adds a root `openclaw-2026-04-15.log` file.

## Best Possible Solution

Extend the existing agent-scoped `memorySearch` schema, resolver, docs, and tests with typed retry/backoff settings, keep `remote.nonBatchConcurrency` as the concurrency knob, remove the log, and require redacted real behavior proof.

## Reproduction Assessment

Yes, for the review blockers. Source inspection shows the PR reads a legacy/top-level untyped path instead of resolved `agents.*.memorySearch`, and the diff adds a generated runtime log.

## Solution Assessment

No as-is. Configurable retry/backoff is reasonable, but the maintainable path is to extend the resolved, validated `agents.*.memorySearch` contract rather than adding a competing config path.
