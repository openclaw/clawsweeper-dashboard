## Summary

Keep open. Current main still leaves embedding retry attempts/base/max backoff hardcoded, so the PR is not obsolete, but the branch is not merge-ready because it reads a schema-invalid top-level config path, overlaps the already-implemented `memorySearch.remote.nonBatchConcurrency` surface, and adds an unrelated runtime log file.

## What This Changes

The PR exports memory embedding default constants, adds overridable retry/concurrency getters, overrides them in `MemoryIndexManager` to read `memorySearch.embedding.*`, and adds a root `openclaw-2026-04-15.log` file.

## Best Possible Solution

Revise this work around the existing agent-scoped memory config contract: keep `remote.nonBatchConcurrency` as the concurrency knob or explicitly reconcile the naming, add typed and validated retry/backoff settings with docs/help/changelog coverage, remove the log file, and add focused resolver/retry tests while preserving current defaults.

## Reproduction Assessment

Yes. Code inspection on current main is enough: retry attempts/base/max delay are still fixed constants, while the PR's new keys are absent from the strict schema and read from the wrong top-level path.

## Solution Assessment

No. Making retry/backoff tunable is a valid direction, but this patch is not the best implementation because it bypasses resolved config, omits schema/docs/tests/changelog coverage, and duplicates part of the concurrency problem already solved through `remote.nonBatchConcurrency`.
