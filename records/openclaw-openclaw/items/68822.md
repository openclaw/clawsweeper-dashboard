## Summary

Keep open. The feature direction is valid, but the PR still wires the new knobs around the validated agent-scoped memory config path and includes an unrelated runtime log, so it is not merge-ready.

## What This Changes

The PR exports memory embedding default constants, adds overridable retry/concurrency getters that read `cfg.memorySearch.embedding.*`, and adds a root `openclaw-2026-04-15.log` file.

## Best Possible Solution

Add typed, validated retry/backoff settings through the agent-scoped `memorySearch` schema/resolver/docs/tests path, reconcile concurrency with `remote.nonBatchConcurrency`, remove the runtime log, and include a user-facing changelog entry.

## Reproduction Assessment

Yes. Code inspection is enough: the PR reads `cfg.memorySearch.embedding.*`, while current main resolves memory settings from `agents.*.memorySearch` and the strict schema has no `embedding` object.

## Solution Assessment

No. Configurable retry/backoff is reasonable, but this patch bypasses the resolved config contract; the safer solution is to extend the validated agent memory config surface first.
