## Summary

Keep open. Current main and the latest release still show the diagnostics-otel OTLP exporters without a plugin-local OTLPExporterError rejection handler, and an open implementation PR at https://github.com/openclaw/openclaw/pull/81085 already references this issue with closing syntax, so the issue should stay open until that PR lands or is closed.

## What This Changes

The issue requests a diagnostics-otel-owned handler for OTLPExporterError unhandled rejections so telemetry export failures do not crash OpenClaw.

## Best Possible Solution

Land the plugin-local diagnostics-otel handler and regression coverage through https://github.com/openclaw/openclaw/pull/81085, then close this issue when the fix merges and ships.

## Reproduction Assessment

Yes, source-reproducible: diagnostics-otel creates OTLP exporters, current main has no OTLPExporterError handler in that plugin, and the global unhandled-rejection handler exits for unmatched rejection reasons. I did not run a live collector failure during this read-only review.

## Solution Assessment

Yes, the issue’s plugin-local handler direction is the best fit because diagnostics-otel owns the OTLP exporter dependency and the public runtime-env seam already supports plugin-owned benign rejection handling.
