## Summary

Keep open. Current main still lacks diagnostics-otel-owned handling for OTLPExporterError rejections, and the active open fix PR should be reviewed or landed before this crash issue is closed.

## What This Changes

The issue reports that diagnostics-otel OTLP exporter failures can surface as unhandled promise rejections and crash OpenClaw when the collector is unavailable.

## Best Possible Solution

Land the plugin-local diagnostics-otel handler and regression coverage through the linked fix PR, then close this issue once that implementation is on main.

## Reproduction Assessment

Yes, source-reproducible: enable diagnostics-otel OTLP export, trigger an exporter failure such as a non-retryable HTTP response, and current main has no plugin-local handler before the core fatal fallback. I did not run a live collector crash reproduction.

## Solution Assessment

Yes. A diagnostics-otel-owned registerUnhandledRejectionHandler callback is the narrowest maintainable fix because the OTLP dependency and error shape are plugin-owned, while the core rejection handler already provides the generic seam.
