## Summary

Keep open. Current main still lacks a diagnostics-otel handler for OTLPExporterError rejections, and an open linked PR is the active implementation path; this issue should stay open until that PR merges.

## What This Changes

The issue reports that diagnostics-otel OTLP exporter failures can surface as unhandled promise rejections and crash OpenClaw when the collector is unavailable.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81085 if it passes merge gates, then close this issue after the fix is merged.

## Reproduction Assessment

Yes. It is source-reproducible: current main creates OTLP exporters in diagnostics-otel, has no plugin-local OTLPExporterError handler, and the global fallback exits for unmatched unhandled rejections.

## Solution Assessment

Yes. A diagnostics-otel-owned registerUnhandledRejectionHandler callback is the narrowest maintainable fix because the OTLP dependency and error shape are plugin-owned and core already provides a generic handler seam.
