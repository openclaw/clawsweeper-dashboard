## What This Changes

The issue asks diagnostics-otel to suppress OTLPExporterError unhandled rejections from OTLP exporters so telemetry export failures do not crash OpenClaw.

## Best Possible Solution

Review and land the open plugin-local fix, then close this issue after the merged PR proves OTLPExporterError is suppressed only while diagnostics-otel is active.

## Reproduction Assessment

Yes. The source path is clear: enable diagnostics-otel with an OTLP endpoint that returns a non-retryable response or disappears during export, and current main has no diagnostics-otel handler before the core fatal fallback.

## Solution Assessment

Yes. A diagnostics-otel-owned runtime-env rejection handler is the narrowest maintainable solution because the OTLP dependency and error shape are plugin-specific and should not broaden the core transient allowlist.
