## What This Changes

The issue reports that diagnostics-otel OTLPExporterError unhandled rejections can crash OpenClaw when the OTLP collector is unavailable.

## Best Possible Solution

## Reproduction Assessment

Yes, source-reproducible: enable diagnostics-otel with an OTLP/HTTP endpoint that returns a non-retryable response such as HTTP 410, trigger export, and current main has no plugin-local handler before the core fatal fallback. I did not run a live collector crash repro.

## Solution Assessment

Yes, the plugin-local registerUnhandledRejectionHandler direction is the narrowest maintainable fix because diagnostics-otel owns the OTLP dependency and the SDK already exposes a plugin-facing rejection-handler seam.
