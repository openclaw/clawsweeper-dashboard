## What This Changes

Handle diagnostics-otel OTLPExporterError unhandled rejections from unavailable OTLP collectors as non-fatal plugin-owned warnings instead of process-fatal rejections.

## Best Possible Solution

## Solution Assessment

Yes, the plugin-local runtime-env handler in the linked PR is the narrowest maintainable fix because diagnostics-otel owns the OTLP dependency and core already exposes plugin rejection handlers.
