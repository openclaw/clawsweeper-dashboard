## Summary

Keep open. Current main still lacks a diagnostics-otel-owned handler for OTLPExporterError rejections, and the open fix PR at https://github.com/openclaw/openclaw/pull/81085 references this issue with closing syntax, so the issue should remain open until that PR lands or is closed.

## What This Changes

The issue reports that diagnostics-otel OTLP exporter failures can surface as unhandled promise rejections and crash OpenClaw when the OTLP collector is unavailable.

## Best Possible Solution

Land the plugin-local diagnostics-otel suppression and regression coverage through https://github.com/openclaw/openclaw/pull/81085, then close this issue after the PR is merged.

## Reproduction Assessment

Yes. The source path is clear: diagnostics-otel creates OTLP exporters, current main has no plugin-local OTLPExporterError handler, and the global handler exits for unmatched unhandled rejections.

## Solution Assessment

Yes. A diagnostics-otel-owned runtime-env rejection handler is the narrowest maintainable fix because the failure is exporter-specific and should not make core’s global transient allowlist depend on OTLP details.
