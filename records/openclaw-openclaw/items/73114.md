## Summary

Close as duplicate/superseded. The remaining session-store memory problem is already tracked by open issue #55334, including the same unbounded `sessions.json`, duplicated `skillsSnapshot`, ephemeral cron/subagent session, lazy-loading, and cleanup asks; the zombie running-task slice is separately tracked by open issue #73121, which this issue cross-referenced.

## What This Changes

The issue asks OpenClaw to bound Gateway memory growth by pruning or archiving session/task records, deduplicating per-session skill snapshots, reconciling zombie running tasks, and avoiding eager loading of historical session metadata.

## Best Possible Solution

Keep #55334 as the broad session-store memory tracker and #73121 as the focused zombie-task tracker, preserving this report's concrete 471-entry/26.8MB/2.5GB field data by linking it from the close comment.
