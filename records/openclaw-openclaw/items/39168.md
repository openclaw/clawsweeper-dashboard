## Summary

Keep open: the reported tool-dispatch bug is still valid, but this PR is not mergeable against current main because it patches the wrong dependency version, lacks lockfile patch metadata, and can break registered `mcp__...` tool names.

## What This Changes

The PR adds a root pnpm patchedDependency for `@mariozechner/pi-agent-core@0.55.3` and a compiled patch that strips a leading `mcp_` before dependency-level tool lookup.

## Best Possible Solution

Add guarded `mcp_` alias handling in `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts`, with exact registered names first, non-empty unambiguous suffix mapping, regression tests, and a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a streamed `toolCall` named `mcp_exec` with allowed tools containing `exec`; current main has no `mcp_` alias branch, so the name remains unnormalized before exact dependency dispatch.

## Solution Assessment

No. The PR's dependency patch is stale and overbroad; the narrower maintainable fix is a guarded OpenClaw-side normalizer change in the existing embedded runner seam.
