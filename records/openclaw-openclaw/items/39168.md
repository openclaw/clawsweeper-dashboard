## Summary

Keep open: the reported dispatch bug is still source-reproducible on current main, but this branch is not mergeable because it patches a stale compiled dependency version, lacks lockfile patch metadata, and rewrites valid MCP-style tool names.

## What This Changes

The PR adds a root pnpm patch for `@mariozechner/pi-agent-core@0.55.3` that strips a leading `mcp_` prefix before dependency-level tool lookup.

## Best Possible Solution

Add a guarded first-party embedded-runner normalizer: exact registered names first, then non-empty `mcp_` suffix mapping only when the suffix uniquely matches an allowed tool, with regression coverage and no compiled dependency patch.

## Reproduction Assessment

Yes. A source reproduction is a streamed `toolCall` named `mcp_exec` with visible allowed tool names containing `exec`; current main leaves that raw name intact, and pi-agent-core then performs exact-name dispatch.

## Solution Assessment

No. Patching compiled `pi-agent-core@0.55.3` is stale and overbroad; the narrower maintainable fix belongs in OpenClaw's existing stream/tool-call-name normalizer with exact-name precedence.
