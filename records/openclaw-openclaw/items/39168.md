## Summary

Keep open: the reported `mcp_` alias failure is source-reproducible, but this branch is not safe to merge because it patches a stale dependency version, lacks lockfile patch metadata, and rewrites registered MCP-style tool names too broadly.

## What This Changes

The PR adds a root `patchedDependencies` entry and patch file for `@mariozechner/pi-agent-core@0.55.3` to strip `mcp_` before dependency-level tool lookup.

## Best Possible Solution

Add a guarded first-party embedded-runner normalizer: exact registered names first, then non-empty `mcp_` suffix mapping only when the suffix uniquely matches an allowed tool, with regression tests and no compiled dependency patch.

## Reproduction Assessment

Yes. A source reproduction is a streamed `toolCall` named `mcp_exec` with `allowedToolNames` containing `exec`; current main has no `mcp_` alias branch, so the raw name reaches exact dispatch unresolved.

## Solution Assessment

No. Patching compiled `pi-agent-core@0.55.3` is stale and overbroad; the narrow maintainable fix is in OpenClaw's existing dispatch normalizer with exact-name precedence and regression coverage.
