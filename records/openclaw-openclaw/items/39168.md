## Summary

Keep open: the underlying `mcp_exec` alias bug still appears source-reproducible on current `main`, but this PR is not mergeable as written because it patches a stale package/version, uses package-local patch metadata, and rewrites valid exact `mcp__` tool names before lookup.

## What This Changes

The PR adds a root `pnpm.patchedDependencies` entry and a compiled patch for `@mariozechner/pi-agent-core@0.55.3` that strips a leading `mcp_` prefix before tool lookup.

## Best Possible Solution

Add guarded first-party alias normalization in the embedded-runner tool-call normalization path, preserving exact allowed names first and covering live plus replayed `mcp_` aliases with focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `mcp_exec` with only `exec` allowlisted falls through current normalization and then reaches pi-agent-core's exact-name lookup as an unknown tool.

## Solution Assessment

No. Patching a stale compiled dependency is not the best fix; the maintainable solution belongs in OpenClaw's existing pre-dispatch normalization seam with exact-name precedence.
