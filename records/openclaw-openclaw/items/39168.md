## Summary

Keep open: the reported alias bug is still source-reproducible on current main, but this PR is not mergeable as-is because it patches a stale dependency version, omits lockfile patch metadata, and would rewrite registered `mcp__...` tool names.

## What This Changes

This PR adds a root pnpm patch for `@mariozechner/pi-agent-core@0.55.3` to strip a leading `mcp_` from tool call names before dependency-level lookup.

## Best Possible Solution

Add guarded `mcp_` alias handling in the first-party embedded runner normalizer, with exact registered names first, non-empty unambiguous suffix mapping, regression tests, and a changelog entry.

## Reproduction Assessment

Yes. A source reproduction is a streamed `toolCall` named `mcp_exec` with allowed tools containing `exec`; current main has no `mcp_` alias branch before exact dependency dispatch.

## Solution Assessment

No. The PR's dependency patch is stale and overbroad; the narrow maintainable solution is to extend the existing OpenClaw-side normalizer rather than patch compiled dependency output.
