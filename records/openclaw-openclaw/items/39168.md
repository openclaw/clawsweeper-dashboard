## Summary

Keep open, but the PR is not mergeable as written: current main still lacks the specific `mcp_exec` alias path, while this branch patches a stale dependency package/version, misses lockfile metadata, and can break exact `mcp__...` registered tool names.

## What This Changes

The PR adds a root pnpm patch for `@mariozechner/pi-agent-core@0.55.3` that strips a leading `mcp_` before dependency-level tool lookup.

## Best Possible Solution

Add a guarded first-party embedded-runner normalizer: preserve exact registered names first, then map a non-empty single-underscore `mcp_` suffix only when that suffix uniquely matches an allowed tool, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a tool call named `mcp_exec` with `allowedToolNames` containing `exec` is not normalized by current main before exact dependency dispatch.

## Solution Assessment

No. Patching a stale compiled dependency is not the best fix; the narrow maintainable solution belongs in OpenClaw's existing stream-normalization seam with exact-name precedence.
