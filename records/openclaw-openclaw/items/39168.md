The PR adds a root `pnpm.patchedDependencies` entry and a compiled patch for `@mariozechner/pi-agent-core@0.55.3` that strips a leading `mcp_` before tool lookup.

## Best Possible Solution

Yes. Source inspection shows `mcp_exec` is not normalized to an allowed `exec` tool on current main, and the active pi-agent-core dependency dispatches by exact tool name before returning `Tool mcp_exec not found`.

## Solution Assessment

No. Patching a deprecated compiled dependency is not the maintainable fix; the narrow path is OpenClaw's first-party stream normalization seam with exact-name precedence.
