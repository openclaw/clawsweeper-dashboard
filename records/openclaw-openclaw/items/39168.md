## Summary

This PR should stay open as a plausible bug-fix proposal, but the branch is not mergeable: current main still does not normalize `mcp_exec` to `exec`, while this patch targets a stale dependency version, omits lockfile patch metadata, and over-strips exact `mcp__...` tool names.

## What This Changes

The branch adds a root pnpm patch for `@mariozechner/pi-agent-core@0.55.3` that strips a leading `mcp_` before dependency-level tool lookup.

## Best Possible Solution

Extend the existing embedded-runner normalizer to try exact names first, then map a non-empty `mcp_` suffix only when it uniquely matches an allowed tool, with regression coverage for both `mcp_exec` and exact `mcp__...` preservation.

## Reproduction Assessment

Yes. A source reproduction is a streamed tool-call block named `mcp_exec` with `allowedToolNames` containing `exec`; current main's normalizer never derives `exec` from that single-underscore form, so exact dispatch remains unresolved.

## Solution Assessment

No. Patching compiled `pi-agent-core@0.55.3` is stale and overbroad; the safer fix belongs in OpenClaw's first-party dispatch normalizer with exact-name precedence and focused tests.
