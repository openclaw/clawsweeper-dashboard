## Summary

Keep open: the reported `mcp_exec` dispatch failure remains source-reproducible on current main, but this PR is not mergeable because it patches a stale/removed dependency package, omits lockfile patch metadata, and would rewrite exact MCP tool names too broadly.

## What This Changes

The PR adds a root pnpm patched dependency for `@mariozechner/pi-agent-core@0.55.3` to strip a leading `mcp_` prefix before dependency-level tool lookup.

## Best Possible Solution

Add a guarded first-party normalizer in `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts`: exact registered names first, then strip a non-empty single-underscore `mcp_` alias only when the suffix uniquely matches an allowed tool, with regression tests and no compiled dependency patch.

## Reproduction Assessment

Yes. A source reproduction is a streamed or final tool call named `mcp_exec` with `allowedToolNames` containing `exec`; current main leaves that name unchanged before pi-agent-core's exact lookup, which returns `Tool mcp_exec not found`.

## Solution Assessment

No. Patching compiled `@mariozechner/pi-agent-core@0.55.3` is not the right fix for current main; the maintainable fix belongs in OpenClaw's existing stream normalization seam with exact-name precedence.
