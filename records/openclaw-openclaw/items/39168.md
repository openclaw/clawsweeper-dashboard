## Best Possible Solution

Move guarded `mcp_` alias recovery into `normalizeToolCallNameForDispatch` after exact allowed-name matching, delete the package patch, and add regression coverage for `mcp_exec`, `mcp_read`, `mcp_write`, and exact `mcp__...` preservation.
