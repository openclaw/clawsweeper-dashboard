## Summary

Keep open: the reported bug is plausible and source-reproducible, but this PR patches an inactive dependency version and adds overbroad command/tool-dispatch behavior without inspectable real behavior proof.

## What This Changes

This PR adds package-level pnpm patch metadata and a compiled patch for `@mariozechner/pi-agent-core@0.55.3` to strip a leading `mcp_` prefix before tool lookup.

## Best Possible Solution

Implement guarded alias normalization in the first-party embedded-runner stream wrapper, preserving exact allowed names first and covering `mcp_exec`, `mcp_read`, `mcp_write`, and exact `mcp__...` names with focused regression tests.

## Reproduction Assessment

Yes, source-reproducible. Current main does not map single-underscore `mcp_exec` to allowed `exec`, and the active pi-agent-core dependency exact-matches `toolCall.name` before returning `Tool mcp_exec not found`.

## Solution Assessment

No. Patching `@mariozechner/pi-agent-core@0.55.3` is not the best fix for current main, which uses `@earendil-works/pi-agent-core@0.74.0`; the safer solution is guarded first-party stream normalization with exact-name precedence.
