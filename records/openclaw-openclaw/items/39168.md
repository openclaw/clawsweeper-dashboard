## What This Changes

The PR adds package-level pnpm patch metadata and a compiled patch for `@mariozechner/pi-agent-core@0.55.3` to strip a leading `mcp_` prefix before tool lookup.

## Best Possible Solution

Implement guarded alias normalization in the first-party embedded-runner stream normalization path, preserving exact-name precedence and adding focused regression coverage for `mcp_exec`, `mcp_read`, `mcp_write`, and exact `mcp__...` names.

## Reproduction Assessment

Yes, source-reproducible. Current main has no `mcp_exec` to `exec` alias in the active pre-dispatch normalization path, while active pi-agent-core exact-matches `toolCall.name` and returns `Tool mcp_exec not found` on a miss.

## Solution Assessment

No. Patching a compiled, inactive dependency is not the best fix; the narrow maintainable path is the first-party embedded-runner normalization wrapper with exact-name precedence.
