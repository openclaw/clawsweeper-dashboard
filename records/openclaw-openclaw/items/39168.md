## Summary

Keep open: the reported tool-dispatch bug is still source-reproducible on current main, but this PR should not merge because it patches an inactive compiled dependency path, lacks inspectable real behavior proof, and overbroadly rewrites exact MCP tool names.

## What This Changes

The PR adds a package-level pnpm patch for `@mariozechner/pi-agent-core@0.55.3` that strips a leading `mcp_` prefix before tool lookup.

## Best Possible Solution

Implement guarded alias normalization in `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts`, preserving exact allowed names first and adding focused regression tests for `mcp_exec` recovery and exact `mcp__...` preservation.

## Reproduction Assessment

Yes, source-reproducible. Current main has no `mcp_exec` to `exec` alias in the first-party stream normalizer, and the active pi-agent-core dependency exact-matches `toolCall.name` before returning `Tool mcp_exec not found`.

## Solution Assessment

No. Patching obsolete compiled dependency code is not the narrow maintainable fix; the safer solution is guarded first-party normalization with exact-name precedence and regression coverage.
