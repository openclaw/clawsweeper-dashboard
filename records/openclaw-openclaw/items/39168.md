## Summary

Keep open: the reported `mcp_` alias failure is source-reproducible, but this PR is not mergeable because it patches a stale compiled dependency instead of the current first-party runner normalizer.

## What This Changes

The PR adds a root pnpm patchedDependency and patch file for `@mariozechner/pi-agent-core@0.55.3` that strips a leading `mcp_` before dependency-level tool lookup.

## Best Possible Solution

Add guarded `mcp_` alias handling in `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts`, with exact registered names first, non-empty unambiguous suffix mapping, focused regression tests, and a changelog entry.

## Reproduction Assessment

Yes. A streamed `toolCall` named `mcp_exec` with allowed tools containing `exec` still flows through current main without an `mcp_` alias branch, so dependency dispatch would continue to see the unmatched name.

## Solution Assessment

No. Patching a stale compiled dependency is not the narrowest maintainable fix; the existing OpenClaw runner normalizer should handle this alias before pi-agent-core dispatch.
