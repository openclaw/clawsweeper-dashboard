## Label Justifications

- `P1`: The issue reports real Feishu and Telegram agent/channel workflows producing runaway repeated messages until user intervention.
- `impact:message-loss`: The core symptom is duplicated visible channel messages during failed tool-call retry loops.
- `impact:crash-loop`: The agent can become stuck in a runaway repeated tool-call loop, affecting availability of the session.

## Summary

Keep open. Current main and the latest release still have only partial loop safety: rolling tool-loop detection is default-off, validation failures get retry guidance but not a low terminal per-turn cap, and the reported Feishu/Telegram duplicate visible-message path is not fully covered.

## What This Changes

The issue asks OpenClaw to stop repeated failed tool-call loops from sending repeated visible channel messages, especially missing-parameter `exec` retries in Feishu and similar Telegram loops.

## Best Possible Solution

Keep this as the concrete bug thread and land a maintainer-scoped runtime guard for deterministic validation failures and duplicate visible-progress suppression, while leaving broad self-healing architecture to https://github.com/openclaw/openclaw/issues/60767.

## Reproduction Assessment

I did not replay the live channels in this read-only review.

## Solution Assessment
