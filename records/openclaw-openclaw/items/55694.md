## Summary

Keep open. Current main and the latest release still leave the reported no-config Feishu/Telegram visible-message loop only partially mitigated: rolling loop detection is default-off, validation failures are returned as ordinary tool errors with retry guidance, and the remaining default/threshold/channel-noise behavior needs maintainer judgment.

## What This Changes

The issue asks OpenClaw to stop repeated failed tool-call retry loops from sending duplicate channel messages, especially after deterministic validation errors like an `exec` call missing `command`.

## Best Possible Solution

Keep this as the concrete bug thread and land a maintainer-approved guard for deterministic validation-failure loops plus visible-message duplicate suppression, while leaving broader self-healing architecture to https://github.com/openclaw/openclaw/issues/60767.
