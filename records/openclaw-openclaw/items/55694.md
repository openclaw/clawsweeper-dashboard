- `impact:message-loss`: The core symptom is duplicated visible channel messages during failed tool-call retry loops.
- `impact:crash-loop`: The agent can remain stuck in a runaway repeated tool-call loop, affecting session availability.

## Summary

Keep open. Current main has useful partial mitigations, including disabled embedded Pi SDK auto-retry and replay guards for committed message sends, but deterministic validation failures still lack a low terminal per-turn cap and the reported Feishu/Telegram duplicate visible-message loop is not fully covered by default behavior.

## What This Changes

The issue requests limiting failed tool-call retry loops and suppressing repeated visible channel messages when tools fail with deterministic errors such as missing exec command parameters.

## Best Possible Solution

Keep this as the concrete bug thread and land a maintainer-scoped runtime guard for deterministic validation failures plus duplicate visible-message suppression, while leaving broad self-healing architecture to https://github.com/openclaw/openclaw/issues/60767.

## Reproduction Assessment
