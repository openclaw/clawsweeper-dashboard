## Summary

Keep open. The PR is a useful narrow fix with real OpenShift proof for the missing-file default `full/off` case, but the branch still misses existing readable empty or invalid approval stores and the item has a protected maintainer label plus an active member-owned replacement path.

## What This Changes

Adds a no-write `resolveExecApprovals()` fast path for missing `exec-approvals.json` under default no-prompt `full/off`, keeps Mac exec-host socket persistence via an internal override, and adds focused regression tests.

## Best Possible Solution

Land one narrow default `full/off` no-persistence fix that handles missing and readable empty/invalid approval stores while preserving strict prompting, allowlist, socket, symlink, and persisted-state paths; https://github.com/openclaw/openclaw/pull/86964 currently appears more complete unless this branch is repaired.

## Reproduction Assessment

Yes. Current main still calls `ensureExecApprovals()` from `resolveExecApprovals()` and that write path hardens the directory with a non-Windows chmod rethrow; the PR body also includes OpenShift fsGroup after-fix logs, although I did not run a new pod repro in this read-only review.

## Solution Assessment

No, not as written. The missing-file fast path is useful, but the safer complete fix is a no-persistence resolution path that safely handles readable empty/invalid approval files and only returns early after resolving the effective policy to `full/off`.
