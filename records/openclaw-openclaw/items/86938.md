## Summary

Keep open: the patch has useful OpenShift proof and the security-preserving direction is plausible, but it still misses the readable empty/invalid approval-store state that current source treats as empty and the related member PR covers.

## What This Changes

Adds a no-write fast path for missing `exec-approvals.json` when the resolved policy is `full/off`, adds an internal `requireSocket` override for Mac exec-host routing, and expands exec approval store tests.

## Best Possible Solution

Land a narrow no-persistence fix that reads existing readable approval state, returns only after resolving `full/off`, preserves `requireSocket` and unreadable-file fail-closed behavior, and leaves broader chmod tolerance to the separate PVC policy discussion.

## Reproduction Assessment

Yes. Current main and `v2026.5.22` call `ensureExecApprovals()` before default policy resolution and rethrow Unix chmod errors; the PR body also gives OpenShift fsGroup pod proof for the missing-file path.

## Solution Assessment

No. The missing-file fast path is directionally sound, but the best pre-merge solution should also cover readable empty or invalid approvals files without weakening unreadable-file or socket-required fail-closed behavior.
