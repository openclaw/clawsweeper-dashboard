## Summary

Keep open: current main and the latest release only support explicit exec env overrides plus `OPENCLAW_SHELL=exec`; they do not inject the requested session id, session key, or agent id. The request is a plausible core exec ergonomics feature, but it needs maintainer product/security approval because it would make session identity ambient in every spawned command.

## What This Changes

The issue requests automatically exposing the current OpenClaw session id, session key, and agent id in every exec subprocess environment, with explicit tool-call env values taking precedence.

## Best Possible Solution

Define an approved exec subprocess session-context contract first; if accepted, implement it centrally where exec env is built for gateway, sandbox, and node, with user env override precedence, docs, and focused tests.

## Reproduction Assessment

## Solution Assessment
