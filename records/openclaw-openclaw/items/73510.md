## Summary

Current main has partial mitigations, but the central outage contract is still open: diagnostics only releases stale session bookkeeping with no active work, while active stalled runs are classified as stalled/long-running and are not aborted by the heartbeat path.

## What This Changes

The issue requests automatic recovery for stuck gateway sessions plus startup-compaction, trajectory-size, memory-limit, and Feishu heartbeat hardening.

## Best Possible Solution

Keep this as the outage umbrella until #73243 or an equivalent configurable, race-safe active stuck-session abort/reset path lands, then split any still-current storage, Feishu, startup-compaction, or service-limit gaps into narrower follow-ups.

Yes for the remaining central code-path gap: current source and tests show active processing sessions are detected as stalled or long-running but are not recovered by the diagnostics heartbeat. The exact Feishu production outage was not live-reproduced in this read-only review.

## Solution Assessment

No: current main is only a partial solution. The maintainable path is to review and land #73243 or an equivalent diagnostics-owned recovery threshold, then validate the storage and Feishu requests separately.
