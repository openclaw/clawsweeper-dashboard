## Summary

Keep open. Current main has partial mitigations, but the central outage contract is still missing: diagnostics can warn and release stale bookkeeping, while active embedded runs, active reply work, and active lane tasks are still not aborted/reset by the heartbeat path.

## What This Changes

The issue requests automatic recovery for stuck gateway sessions plus startup compaction, trajectory size control, memory-limit documentation, and Feishu probe hardening after a Feishu WebSocket outage on v2026.4.26.

## Best Possible Solution

Keep this as the outage umbrella until #73243 or an equivalent configurable, race-safe active stuck-session abort/reset path lands, then split any still-current Feishu, startup-compaction, trajectory, or service-limit gaps into narrower follow-ups.
