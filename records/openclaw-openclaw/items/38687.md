## Summary

Keep open. The reported NO_PROXY/CIDR media proxy problem is plausible from current source and upstream undici behavior, but this branch is dirty and currently regresses managed proxy/FormData behavior; it also lacks required inspectable real behavior proof.

## What This Changes

The PR changes media-understanding audio/video provider execution to resolve proxy-aware fetch after the backend baseUrl is known and adds target-aware NO_PROXY/CIDR proxy bypass coverage.

## Best Possible Solution

Rebase or rewrite against current proxy-env/proxy-fetch helpers: pass the media backend URL into target-aware NO_PROXY/CIDR logic, preserve resolveEnvHttpProxyAgentOptions and FormData normalization, and remove the undocumented loopback media bypass unless explicitly approved.

## Reproduction Assessment

Yes, from source inspection. Current main resolves the media proxy fetch before the backend baseUrl is known, and upstream undici EnvHttpProxyAgent does not implement OpenClaw's CIDR NO_PROXY extension.

## Solution Assessment

No. The PR moves fetch resolution to the right runner point, but it reimplements proxy matching instead of reusing current shared helpers and regresses current main's proxy options and multipart handling.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Require explicit NO_PROXY before loopback bypass:** `src/infra/net/proxy-fetch.ts:136-140`
  - body: The matcher returns direct fetch for localhost, ::1, and 127.0.0.0/8 before consulting NO_PROXY. The managed proxy docs reserve direct loopback bypass for Gateway control-plane RPC; media provider HTTP should stay under the operator proxy unless the backend is explicitly excluded or maintainers approve a policy change.
  - confidence: 0.89
- **[P2] Preserve ALL_PROXY-backed proxy fetch options:** `src/infra/net/proxy-fetch.ts:204-217`
  - body: This branch gates proxy fetch creation on HTTP(S)-specific proxy env and constructs EnvHttpProxyAgent without the current explicit options. Current main maps ALL_PROXY/all_proxy into EnvHttpProxyAgent options, so ALL_PROXY-only deployments can silently lose media proxy routing here.
  - confidence: 0.87
- **[P2] Keep FormData normalization in the proxy fetch:** `src/infra/net/proxy-fetch.ts:217-220`
  - body: The proxy fetch now passes the raw RequestInit into undici fetch. Current main converts global FormData into undici FormData and strips multipart body headers; dropping that path can re-break the audio transcription multipart uploads this PR is trying to protect.
  - confidence: 0.82
