## Summary

Keep open: the PR targets a real media proxy gap, but the branch is not ready to merge because it duplicates proxy matching now centralized on main, changes loopback egress behavior, and risks dropping current ALL_PROXY-backed proxy routing.

## What This Changes

The PR changes media-understanding to resolve proxy fetches after audio/video backend baseUrl resolution and adds target-aware NO_PROXY/loopback/CIDR bypass tests for media backends.

## Best Possible Solution

Keep the baseUrl-aware media runner direction, but implement the target decision through shared proxy helpers, preserve ALL_PROXY behavior, and require explicit NO_PROXY/CIDR support or an approved documented loopback exception.

## Reproduction Assessment

Yes. Source inspection on current main shows media proxy fetch is resolved before the backend baseUrl is known and then passed to audio/video providers; I did not run tests because this review is read-only.

## Solution Assessment

No. The PR touches the right surface, but the current patch is not the best fix because it hand-rolls proxy matching, changes loopback egress policy, and does not preserve current ALL_PROXY option handling.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep media loopback behind explicit proxy opt-out:** `src/infra/net/proxy-fetch.ts:136-140`
  - body: The new matcher returns direct fetch for localhost, ::1, and 127.0.0.0/8 before checking NO_PROXY. Managed proxy mode documents loopback bypass only for Gateway control-plane traffic, so media transcription to a loopback URL would bypass operator egress controls without an explicit media opt-out.
  - confidence: 0.88
- **[P2] Preserve ALL_PROXY-backed proxy fetches:** `src/infra/net/proxy-fetch.ts:205-216`
  - body: Current main maps ALL_PROXY into explicit EnvHttpProxyAgent options and tests ALL_PROXY-only environments. This branch gates proxy fetch creation on hasEnvHttpProxyConfigured("https") and constructs EnvHttpProxyAgent without those options, so ALL_PROXY-only deployments can stop receiving a proxy fetch for affected callers.
  - confidence: 0.84
