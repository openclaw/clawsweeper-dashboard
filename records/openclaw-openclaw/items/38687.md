## Summary

Keep open: the reported media proxy bug is still source-reproducible on current main, but this PR head is not mergeable as-is because it drops newer proxy-fetch behavior, changes proxy bypass policy, and still lacks inspectable real behavior proof.

## What This Changes

This PR makes media-understanding audio and video provider fetch resolution target-aware using the configured backend baseUrl and adds proxy/NO_PROXY regression coverage.

## Best Possible Solution

Rework the media runner to pass baseUrl into a shared proxy decision that preserves resolveEnvHttpProxyAgentOptions and multipart normalization, then add redacted live STT proof.

## Reproduction Assessment

Yes, source-reproducible: current main resolves the media proxy fetch before the backend baseUrl is available, and undici's matcher does not provide OpenClaw's CIDR NO_PROXY extension by itself. I did not run a live self-hosted STT backend in this read-only review.

## Solution Assessment

No. Moving fetch resolution after baseUrl is the right direction, but this branch duplicates proxy matching and drops current proxy-fetch behavior instead of reusing the shared proxy-env and proxy-fetch contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Require explicit NO_PROXY for media loopback bypass:** `src/infra/net/proxy-fetch.ts:136-140`
  - body: The helper returns direct fetch for localhost, ::1, and 127.0.0.0/8 before reading NO_PROXY. Normal runtime media-provider traffic should stay behind the operator proxy unless a bypass is explicitly configured; this can silently send loopback media requests direct with no opt-out.
  - confidence: 0.9
- **[P2] Preserve EnvHttpProxyAgent option mapping:** `src/infra/net/proxy-fetch.ts:204-216`
  - body: This branch gates on hasEnvHttpProxyConfigured("https") and constructs EnvHttpProxyAgent without the current resolveEnvHttpProxyAgentOptions mapping. Deployments that rely on ALL_PROXY can lose proxy routing for media provider requests after this change.
  - confidence: 0.9
- **[P2] Normalize multipart init before undici fetch:** `src/infra/net/proxy-fetch.ts:217-220`
  - body: The returned wrapper forwards raw init through undiciFetch. Current main converts global FormData to undici FormData and strips stale multipart headers before proxy dispatch, so this can reintroduce multipart upload failures for proxied audio transcription.
  - confidence: 0.84
