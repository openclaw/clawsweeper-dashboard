## Summary

Keep open: source inspection still shows the reported media proxy timing problem on current main, but this PR is not safe to merge as-is because it duplicates stale proxy matching, adds direct loopback media bypass behavior, drops current proxy-agent handling, and lacks inspectable after-fix behavior proof.

## What This Changes

This PR makes media-understanding resolve proxy-aware fetches after audio/video backend baseUrl resolution and adds target-aware NO_PROXY/loopback bypass logic with proxy/media regression tests.

## Best Possible Solution

Rewrite against current proxy-env helpers: compute the media backend URL before resolving fetch, skip the wrapper only for explicit NO_PROXY matches, preserve EnvHttpProxyAgent options and multipart normalization, and cover audio/video regressions.

## Reproduction Assessment

Yes, from source inspection. Current main resolves the media proxy fetch before the backend baseUrl is known, so it cannot suppress the wrapper specifically for a configured NO_PROXY audio/video backend.

## Solution Assessment

No. The PR targets the right runner surface, but the current diff is not the best fix because it duplicates stale proxy logic, adds undocumented loopback media bypass behavior, and can drop current proxy-agent and multipart handling.

  - body: The new matcher returns direct fetch for localhost, ::1, and 127.0.0.0/8 before consulting NO_PROXY.   - confidence: 0.88
- **[P2] Preserve EnvHttpProxyAgent options:** `src/infra/net/proxy-fetch.ts:205-216`
  - body: Current main maps ALL_PROXY/all_proxy into explicit EnvHttpProxyAgent options. This branch gates fetch creation on HTTP(S)-specific proxy detection and constructs `EnvHttpProxyAgent` without those options, so ALL_PROXY-only deployments can lose proxy routing.
  - confidence: 0.86
- **[P2] Keep current multipart normalization:** `src/infra/net/proxy-fetch.ts:217-220`
  - body: Current main normalizes global FormData into undici FormData before proxy dispatch. This branch forwards the raw init object, which can reintroduce multipart upload failures for proxied transcription requests.
  - confidence: 0.81
