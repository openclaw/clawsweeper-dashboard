## Summary

Keep open: the reported media/STT proxy problem is plausible and source-reproducible, but this PR still needs contributor-provided real behavior proof and rework for proxy-policy regressions before it is mergeable.

## What This Changes

The PR makes media audio/video provider execution pass the configured backend baseUrl into proxy-fetch resolution, adds target-aware NO_PROXY matching in proxy-fetch, and extends proxy runner tests.

## Best Possible Solution

Rebase and rework the branch so media execution passes the backend baseUrl into proxy resolution while reusing the current shared proxy-env matcher/options and proxy-fetch FormData normalization.

## Reproduction Assessment

Yes, source-reproducible: current main calls resolveProxyFetchFromEnv before the media backend baseUrl is known, so this runner path cannot apply target-aware NO_PROXY/CIDR handling at that seam. I did not exercise a live self-hosted STT backend in this read-only review.

## Solution Assessment

No. Moving fetch resolution after baseUrl is the right direction, but the implementation should reuse current proxy-env and proxy-fetch contracts instead of duplicating older proxy policy and losing multipart/options behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Require NO_PROXY before direct loopback bypass:** `src/infra/net/proxy-fetch.ts:136-140`
  - body: The matcher returns direct fetch for localhost, ::1, and 127.0.0.0/8 before consulting NO_PROXY. The managed proxy docs limit automatic loopback direct routing to Gateway control-plane traffic, so media provider requests can bypass an operator-configured proxy with no explicit backend opt-out.
  - confidence: 0.9
- **[P2] Preserve EnvHttpProxyAgent options:** `src/infra/net/proxy-fetch.ts:204-216`
  - body: This branch gates on HTTP/HTTPS proxy env and constructs EnvHttpProxyAgent without the current explicit options. Current main maps ALL_PROXY and lower/upper-case proxy precedence through resolveEnvHttpProxyAgentOptions, so ALL_PROXY-only media deployments can silently go direct after this change.
  - confidence: 0.9
- **[P2] Normalize multipart init before undici fetch:** `src/infra/net/proxy-fetch.ts:217-220`
  - body: Current main converts global FormData to undici FormData and strips stale multipart headers before proxy dispatch. This branch forwards raw init through undiciFetch, which can reintroduce multipart upload failures for proxied audio transcription.
  - confidence: 0.84
