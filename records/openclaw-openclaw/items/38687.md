## Summary

Keep open. The reported media proxy bug is real on current main, but the PR branch is not merge-ready because it changes security-sensitive egress behavior and regresses current proxy-env handling rather than reusing the newer shared helpers.

## What This Changes

The PR makes media-understanding resolve proxy fetches after provider baseUrl resolution and adds target-aware NO_PROXY bypass logic plus proxy regression tests for media backends.

## Best Possible Solution

Keep the baseUrl-aware media runner change, but implement the decision in the shared proxy layer: accept an optional target URL, reuse matchesNoProxy and resolveEnvHttpProxyAgentOptions, preserve ALL_PROXY support and current request overrides, and require explicit NO_PROXY or a documented maintainer-approved media opt-out for direct loopback behavior.

## Reproduction Assessment

Yes. Static reproduction is high confidence: current main constructs the media provider fetch before resolving baseUrl, while OpenAI-compatible audio uses the injected fetch for the multipart transcription request. I did not run tests because this was a read-only review.

## Solution Assessment

No. The PR targets the correct bug, but the branch is not the best final fix because it duplicates NO_PROXY semantics, adds an unconditional loopback media bypass, and drops current ALL_PROXY-backed proxy fetch behavior instead of extending the current shared proxy-env helpers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep media loopback behind explicit proxy opt-out:** `src/infra/net/proxy-fetch.ts:136-140`
  - body: The new matcher returns direct fetch for localhost, ::1, and 127.0.0.0/8 before checking NO_PROXY. Managed proxy mode documents loopback bypass only for Gateway control-plane traffic and clears NO_PROXY so normal runtime traffic stays behind the filtering proxy, so media transcription to a loopback URL would bypass operator egress controls without an explicit media opt-out.
  - confidence: 0.88
- **[P2] Preserve ALL_PROXY-backed proxy fetches:** `src/infra/net/proxy-fetch.ts:205-216`
  - body: Current main maps ALL_PROXY into explicit EnvHttpProxyAgent options and has coverage for ALL_PROXY-only environments. This branch gates proxy fetch creation on hasEnvHttpProxyConfigured("https") and then constructs EnvHttpProxyAgent without options, so ALL_PROXY-only deployments stop receiving a proxy fetch for affected callers.
  - confidence: 0.84

## Security Review
