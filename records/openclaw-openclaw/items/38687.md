## Summary

Keep open: the media CIDR NO_PROXY gap is still a plausible source-level bug, but this PR head is stale/conflicting, lacks inspectable real behavior proof, and introduces proxy-policy regressions around loopback, ALL_PROXY, and multipart handling.

## What This Changes

The PR moves media audio/video proxy fetch resolution until after provider baseUrl is known and adds custom NO_PROXY/loopback bypass matching plus regression tests.

## Best Possible Solution

Rework the PR so media providers resolve proxy fetch after baseUrl is known while reusing the shared proxy-env matcher/options and preserving operator proxy policy plus multipart handling.

## Reproduction Assessment

Yes, source-reproducible for the CIDR/target-aware gap: current main resolves the media proxy fetch before baseUrl is known, while Undici's EnvHttpProxyAgent no_proxy matcher does not support CIDR entries. I did not run a live self-hosted STT backend in this read-only review.

## Solution Assessment

No. Moving resolution after baseUrl is the right direction, but this branch should reuse matchesNoProxy/resolveEnvHttpProxyAgentOptions and current FormData normalization without adding an implicit media loopback bypass unless maintainers approve that policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Require NO_PROXY before bypassing loopback media targets:** `src/infra/net/proxy-fetch.ts:136-140`
  - body: The matcher returns direct fetch for localhost, ::1, and 127/8 before reading NO_PROXY. Current network-proxy docs reserve direct loopback bypass for Gateway control-plane RPC and say normal runtime HTTP remains proxied, so a media backend on loopback can bypass an operator proxy with no explicit opt-out.
  - confidence: 0.9
- **[P2] Preserve ALL_PROXY options when resolving proxy fetch:** `src/infra/net/proxy-fetch.ts:204-216`
  - body: Current main maps ALL_PROXY into EnvHttpProxyAgent options before constructing the dispatcher. This branch gates on only HTTP(S) env vars and calls new EnvHttpProxyAgent() without options, so ALL_PROXY-only deployments can lose proxy routing for media provider requests.
  - confidence: 0.91
- **[P2] Keep FormData normalization on proxy fetch requests:** `src/infra/net/proxy-fetch.ts:217-220`
  - body: Current main converts global FormData to Undici FormData and strips stale multipart headers before proxy dispatch. This branch sends raw init through undiciFetch in the proxy path, which can reintroduce multipart upload failures for proxied audio transcription.
  - confidence: 0.82
