## Summary

Keep open. The PR targets a real media proxy gap, but the current head is dirty against main, still has proxy-routing correctness/security blockers, and lacks inspectable real STT behavior proof.

## What This Changes

The PR makes media-understanding audio/video proxy fetch resolution target-aware using the configured backend baseUrl and adds proxy-fetch and media runner tests for NO_PROXY bypass cases.

## Best Possible Solution

Rebase or rewrite against current proxy-env/proxy-fetch helpers, pass backend baseUrl into a target-aware helper for explicit NO_PROXY/CIDR matches, preserve ALL_PROXY and FormData behavior, and remove the undocumented loopback media bypass unless maintainers approve it.

## Reproduction Assessment

Yes, from source inspection. Current main resolves the media proxy fetch before the backend baseUrl is known, so OpenClaw's CIDR NO_PROXY matcher cannot be applied to the configured media backend; I did not run a live STT setup.

## Solution Assessment

No. The PR moves resolution to the right runner point, but duplicates proxy matching and regresses current main's managed proxy contract/options; reusing the shared helpers is the narrower maintainable fix.
