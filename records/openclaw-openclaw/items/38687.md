## Summary

Keep open: the PR targets a real source-reproducible media proxy gap, but the branch is not mergeable as-is because it duplicates stale proxy policy, weakens current proxy-fetch behavior, and lacks inspectable real behavior proof.

## What This Changes

The PR makes media audio/video provider execution resolve proxy fetches after the backend baseUrl is known and adds custom NO_PROXY/CIDR matching plus runner/proxy tests.

## Best Possible Solution

Rework the branch so media execution passes the backend URL into a shared proxy decision that reuses src/infra/net/proxy-env.ts and src/infra/net/proxy-fetch.ts, with no implicit media loopback bypass.

## Reproduction Assessment

Yes, source-reproducible: current main creates the media proxy fetch before the backend baseUrl is known, and undici's EnvHttpProxyAgent does not implement OpenClaw's CIDR NO_PROXY extension. I did not run a live self-hosted STT backend in this read-only review.

## Solution Assessment

No. Moving fetch resolution after baseUrl is the right direction, but the implementation should reuse the current shared proxy-env/proxy-fetch contracts instead of duplicating older proxy policy and losing multipart/options behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Require explicit NO_PROXY for loopback media bypass:** `src/infra/net/proxy-fetch.ts:136-140`
  - body: The new matcher returns direct fetch for localhost, ::1, and 127.0.0.0/8 before consulting NO_PROXY. Current proxy docs limit automatic loopback direct routing to Gateway control-plane traffic, so media provider requests can bypass an operator-configured proxy with no explicit backend opt-out.
  - confidence: 0.9
- **[P2] Preserve EnvHttpProxyAgent option mapping:** `src/infra/net/proxy-fetch.ts:204-216`
  - body: This branch gates on hasEnvHttpProxyConfigured("https") and constructs EnvHttpProxyAgent without the current resolveEnvHttpProxyAgentOptions mapping. Deployments relying on ALL_PROXY can silently send media provider traffic direct after this change.
  - confidence: 0.9
- **[P2] Normalize multipart init before undici fetch:** `src/infra/net/proxy-fetch.ts:217-220`
  - body: Current proxy-fetch converts global FormData to undici FormData and strips stale multipart headers before proxy dispatch. This branch forwards raw init through undiciFetch, which can reintroduce multipart upload failures for proxied audio transcription.
  - confidence: 0.84
