## Summary

Keep open: the PR targets a real, source-reproducible media proxy bug, but the branch is stale/conflicting and still has merge-blocking correctness and security issues plus missing real behavior proof.

## What This Changes

The PR makes media-understanding audio/video proxy fetch resolution target-aware and adds NO_PROXY/CIDR matching coverage for self-hosted media backends.

## Best Possible Solution

Rebase and implement target-aware media proxy selection by reusing the current proxy-env matcher, resolveEnvHttpProxyAgentOptions, multipart normalization, and provider request override flow, then add focused regression tests plus redacted live STT proof.

## Reproduction Assessment

Yes, source-reproducible: current main resolves fetchFn before the configured media backend baseUrl is available, while target-aware NO_PROXY matching exists in proxy-env.ts and is not applied at that runner seam. I did not run a live self-hosted STT backend in this read-only review.

## Solution Assessment

No. Moving proxy resolution after baseUrl is the right direction, but this branch duplicates stale proxy matching and drops current proxy-fetch and media request contracts instead of reusing the shared helpers.

- **[P2] Preserve EnvHttpProxyAgent option mapping:** `src/infra/net/proxy-fetch.ts:204-216`
  - body: The branch checks only HTTPS proxy configuration and constructs EnvHttpProxyAgent without current resolveEnvHttpProxyAgentOptions. After rebase, ALL_PROXY-backed media egress and normalized HTTP/HTTPS fallback behavior can go direct instead of through the configured proxy.
  - confidence: 0.93

  - confidence: 0.9

  - confidence: 0.86
- **[P2] Preserve provider request overrides:** `src/media-understanding/runner.entries.ts:494-503`
  - body: The stale branch resolves only apiKeys, baseUrl, and headers from the media execution context. Current main also propagates request into audio/video providers, so merging this shape would drop configured media request overrides on the same path.
  - confidence: 0.88
- **[P2] Require explicit NO_PROXY for loopback bypass:** `src/infra/net/proxy-fetch.ts:136-140`
