## Summary

Keep open: current main still lacks target-aware media proxy fetch resolution, and this PR moves in a useful direction, but the latest head still has proxy-policy regressions and no inspectable real behavior proof.

## What This Changes

The PR changes media audio/video provider execution to resolve proxy-aware fetch after the backend baseUrl is known and adds local NO_PROXY/loopback matching coverage.

## Best Possible Solution

Rework the branch so media provider fetch resolution receives the backend baseUrl while reusing the shared proxy-env matcher/options and current multipart normalization, then verify with focused tests and redacted real STT proof.

## Reproduction Assessment

Yes, source-reproducible: current main resolves media proxy fetch before the provider baseUrl is known, while the shared NO_PROXY/CIDR matcher needs a target URL. I did not run a live self-hosted STT backend in this read-only review.

## Solution Assessment

No. Moving fetch resolution after baseUrl is the right direction, but the implementation should reuse matchesNoProxy/resolveEnvHttpProxyAgentOptions and preserve current FormData normalization instead of duplicating proxy policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Require NO_PROXY before direct loopback bypass:** `src/infra/net/proxy-fetch.ts:136-140`
  - body: The matcher returns direct fetch for localhost, ::1, and 127.0.0.0/8 before consulting NO_PROXY. Current proxy docs reserve automatic loopback direct routing for Gateway control-plane traffic, so media provider requests can bypass a configured proxy with no explicit operator opt-out.
  - confidence: 0.9
- **[P2] Preserve EnvHttpProxyAgent options:** `src/infra/net/proxy-fetch.ts:204-216`
  - body: Current main maps ALL_PROXY and lowercase/uppercase proxy env into explicit EnvHttpProxyAgent options. This branch gates only on HTTPS-style proxy env and constructs new EnvHttpProxyAgent() without options, so ALL_PROXY-only deployments can silently lose media-provider proxy routing.
  - confidence: 0.91
- **[P2] Normalize multipart init in proxy fetch:** `src/infra/net/proxy-fetch.ts:217-220`
  - body: Current main converts global FormData to undici FormData and strips stale multipart headers before proxy dispatch. This branch forwards raw init through undiciFetch, which can reintroduce multipart upload failures for proxied audio transcription.
  - confidence: 0.84
