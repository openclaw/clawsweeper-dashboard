## Summary

Keep open: the reported media proxy bug is plausible on current main, but this PR is not mergeable as-is because its proxy matcher is stale against the current shared proxy helpers, it changes loopback egress policy, and it lacks inspectable real behavior proof.

## What This Changes

The PR makes media audio/video proxy fetch resolution target-aware by passing backend baseUrl into proxy-fetch logic and adds proxy-fetch/media runner NO_PROXY tests.

## Best Possible Solution

Rebase or rewrite the fix so media audio/video resolves proxy fetch after baseUrl is known, reuses the shared proxy-env NO_PROXY/CIDR matcher and EnvHttpProxyAgent option helper, preserves FormData handling, and avoids an implicit loopback media bypass unless maintainers approve that policy.

## Reproduction Assessment

Yes, source-reproducible: current main passes a generic proxy fetch into media providers before the configured backend baseUrl is known, while Undici does not implement OpenClaw's CIDR NO_PROXY extension. I did not run a live self-hosted STT backend in this read-only review.

## Solution Assessment

No. Moving resolution after baseUrl is the right direction, but the PR duplicates stale proxy matching, loses current ALL_PROXY option handling, and adds an undocumented loopback bypass instead of reusing the current shared helper contract.
