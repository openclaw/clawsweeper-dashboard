## Summary

Keep open: the reported media/STT proxy bug is plausible and source-reproducible, but this PR is not mergeable as-is because it lacks inspectable real behavior proof and introduces concrete proxy-policy regressions in the current proxy-fetch contract.

## What This Changes

The PR makes media audio/video provider proxy fetch resolution target-aware and adds custom NO_PROXY/CIDR matching plus focused runner/proxy tests.

## Best Possible Solution

Pass the media backend baseUrl into a shared proxy decision that reuses current proxy-env/proxy-fetch contracts, preserves multipart/options behavior, and includes redacted live or log proof for the self-hosted STT path.

## Reproduction Assessment

Yes, source-reproducible: current main resolves media proxy fetch before the backend baseUrl is available, and undici's no_proxy matcher does not support CIDR entries. I did not run a live self-hosted STT backend in this read-only review.

## Solution Assessment

No. Moving fetch resolution after baseUrl is the right direction, but this branch duplicates proxy matching and drops current proxy-fetch behavior instead of reusing the shared proxy-env and proxy-fetch contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Require explicit NO_PROXY for loopback media bypass:** `src/infra/net/proxy-fetch.ts:136-140`
  - body: The new matcher returns direct fetch for localhost, ::1, and 127.0.0.0/8 before consulting NO_PROXY. Current proxy docs limit automatic loopback direct routing to Gateway control-plane traffic, so media provider requests can bypass an operator-configured proxy with no explicit backend opt-out.
  - confidence: 0.9
- **[P2] Preserve EnvHttpProxyAgent option mapping:** `src/infra/net/proxy-fetch.ts:204-216`
  - body: This branch gates on hasEnvHttpProxyConfigured("https") and constructs EnvHttpProxyAgent without the current resolveEnvHttpProxyAgentOptions mapping. Deployments relying on ALL_PROXY can silently send media provider traffic direct after this change.
  - confidence: 0.9
- **[P2] Normalize multipart init before undici fetch:** `src/infra/net/proxy-fetch.ts:217-220`
  - body: Current proxy-fetch converts global FormData to undici FormData and strips stale multipart headers before proxy dispatch. This branch forwards raw init through undiciFetch, which can reintroduce multipart upload failures for proxied audio transcription.
  - confidence: 0.84
