## Summary

Keep this PR open because the underlying media proxy bug is still source-reproducible on current main, but the branch is not mergeable as-is: it drops current proxy-fetch contracts, changes egress policy for loopback media targets, and still lacks inspectable real behavior proof.

## What This Changes

The PR changes media-understanding proxy resolution so audio and video provider fetches can bypass proxy wrapping for configured backend URLs covered by loopback or NO_PROXY rules.

## Best Possible Solution

Rework the PR to pass the media backend baseUrl into a shared proxy decision that preserves resolveEnvHttpProxyAgentOptions and multipart normalization, then add redacted live/log proof for the self-hosted STT path.

## Reproduction Assessment

Yes, source-reproducible: current main resolves the media proxy fetch before the backend baseUrl is available, and undici's matcher does not provide OpenClaw's CIDR NO_PROXY extension by itself. I did not run a live self-hosted STT backend in this read-only review.

## Solution Assessment

No. Moving fetch resolution after baseUrl is the right direction, but this branch duplicates proxy matching and drops current proxy-fetch behavior instead of reusing the shared proxy-env and proxy-fetch contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Require explicit NO_PROXY before loopback media bypass:** `src/infra/net/proxy-fetch.ts:136-140`
  - body: The matcher returns direct fetch for localhost, ::1, and 127.0.0.0/8 before consulting NO_PROXY.   - confidence: 0.9

  - body: This branch gates on hasEnvHttpProxyConfigured("https") and constructs EnvHttpProxyAgent without the current resolveEnvHttpProxyAgentOptions mapping. Deployments relying on ALL_PROXY can silently send media provider traffic direct after this change.
  - confidence: 0.9
- **[P2] Normalize multipart init before undici fetch:** `src/infra/net/proxy-fetch.ts:217-220`
  - body: Current proxy-fetch converts global FormData to undici FormData and strips stale multipart headers before proxy dispatch. This branch forwards raw init through undiciFetch, which can reintroduce multipart upload failures for proxied audio transcription.
  - confidence: 0.84
