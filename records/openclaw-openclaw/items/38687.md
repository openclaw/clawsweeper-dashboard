## Summary

Keep open. The reported media proxy bug is source-reproducible, but this PR is stale/conflicting against current main and introduces proxy-routing regressions that should block merge until it is rebased and reworked around the current shared proxy contracts.

## What This Changes

This PR makes media-understanding audio/video fetch resolution target-aware by passing backend baseUrl into proxy-fetch and adding custom NO_PROXY/CIDR matching coverage.

## Best Possible Solution

Rebase and implement the media bypass by reusing current proxy-env matching plus current proxy-fetch option and multipart normalization contracts, then add focused regression tests and redacted live STT proof.

## Reproduction Assessment

Yes, source-reproducible: current main resolves the media fetch before backend baseUrl is available, while OpenClaw's CIDR NO_PROXY extension lives in proxy-env.ts and is not applied at that runner seam. I did not run a live self-hosted STT backend in this read-only review.

## Solution Assessment

No. Moving fetch resolution after baseUrl is the right direction, but this branch duplicates proxy matching and drops current proxy-fetch behavior instead of reusing the shared proxy-env and proxy-fetch contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Require explicit NO_PROXY for loopback media bypass:** `src/infra/net/proxy-fetch.ts:136-140`
  - body: The helper returns direct fetch for localhost, ::1, and 127.0.0.0/8 before reading NO_PROXY. Deployments that intentionally force even local media backend traffic through an operator proxy have no opt-out, so keep loopback under the same explicit bypass policy or document a maintainer-approved exception.
  - confidence: 0.85
- **[P2] Preserve EnvHttpProxyAgent option mapping:** `src/infra/net/proxy-fetch.ts:204-216`
  - body: This branch checks only `hasEnvHttpProxyConfigured("https")` and constructs `EnvHttpProxyAgent` without the current `resolveEnvHttpProxyAgentOptions` mapping. Current main uses that mapping to preserve ALL_PROXY and lower/upper-case env semantics, so ALL_PROXY-only media egress can go direct after this change.
  - confidence: 0.93
- **[P2] Normalize multipart init before undici fetch:** `src/infra/net/proxy-fetch.ts:217-220`
  - body: The returned wrapper forwards raw `init` into `undiciFetch`. Current main converts global FormData to undici FormData and strips stale multipart headers before proxy dispatch, so this can reintroduce multipart upload failures for proxied audio transcription.
  - confidence: 0.9
- **[P2] Reuse the shared NO_PROXY matcher:** `src/infra/net/proxy-fetch.ts:151-156`
  - body: The new local matcher only adds CIDR support, while current main's `matchesNoProxy` also owns OpenClaw's IPv4 wildcard and kept-in-sync undici semantics. A media backend covered by `NO_PROXY=100.64.*` would still receive a proxy fetch here even though the shared contract treats it as a bypass.
  - confidence: 0.82
