## Summary

This PR should stay open: it targets a real media proxy bug, but the current branch still has blocking egress-routing regressions and lacks inspectable real behavior proof.

## What This Changes

The PR makes media audio/video proxy fetch resolution target-aware so self-hosted backend baseUrls covered by NO_PROXY can use direct fetch instead of an env proxy wrapper.

## Best Possible Solution

Keep the target-aware media fetch direction, but implement it through the shared proxy-env matcher/options, preserve multipart normalization and provider request overrides, and require contributor proof from a redacted self-hosted STT run.

## Reproduction Assessment

Yes, source-reproducible: current main resolves the media proxy fetch before the provider baseUrl is known, while the shared NO_PROXY CIDR matcher only works when a target URL is available. I did not run a live self-hosted STT backend in this read-only review.

## Solution Assessment

No. Moving fetch resolution after baseUrl is the right direction, but the branch should reuse matchesNoProxy/resolveEnvHttpProxyAgentOptions and current FormData normalization instead of adding duplicate proxy policy and implicit loopback bypass.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Require NO_PROXY before bypassing loopback media targets:** `src/infra/net/proxy-fetch.ts:136-140`
  - body: The matcher returns direct fetch for localhost, ::1, and 127/8 before reading NO_PROXY. Current proxy docs reserve automatic loopback bypass for Gateway control-plane traffic, so media backends on loopback can bypass a configured proxy with no explicit operator opt-out.
  - confidence: 0.9
- **[P2] Preserve shared proxy agent options:** `src/infra/net/proxy-fetch.ts:204-216`
  - body: Current main maps ALL_PROXY into EnvHttpProxyAgent options before constructing the dispatcher. This branch gates only on HTTP(S) env vars and calls new EnvHttpProxyAgent() without options, so ALL_PROXY-only deployments can lose proxy routing for media provider requests.
  - confidence: 0.91
- **[P2] Keep FormData normalization on proxy fetch requests:** `src/infra/net/proxy-fetch.ts:217-220`
  - body: Current main converts global FormData to Undici FormData and strips stale multipart headers before proxy dispatch. This branch sends raw init through undiciFetch in the proxy path, which can reintroduce multipart upload failures for proxied audio transcription.
  - confidence: 0.82
