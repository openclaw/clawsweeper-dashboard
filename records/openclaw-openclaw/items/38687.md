## Summary

Keep open: the reported proxy/NO_PROXY media bug is still plausible on current main, but this PR should not merge as-is because it regresses proxy routing policy, ALL_PROXY handling, and multipart FormData handling, and it still lacks inspectable real behavior proof.

## What This Changes

The PR moves media audio/video proxy fetch resolution until after provider baseUrl is known, adds custom loopback/NO_PROXY/CIDR bypass matching in proxy-fetch, and adds proxy regression tests.

## Best Possible Solution

Keep the target-aware media proxy resolution direction, but rework it to reuse the shared proxy-env matcher/options, preserve multipart normalization, and require explicit NO_PROXY for direct media bypass unless maintainers approve a broader loopback policy.

## Reproduction Assessment

Yes, source-reproducible: current main resolves the media proxy fetch before baseUrl is known, while Undici's EnvHttpProxyAgent does not implement OpenClaw's CIDR NO_PROXY extension. I did not run a live self-hosted STT backend in this read-only review.

## Solution Assessment

No. Moving fetch resolution after baseUrl is the right direction, but this branch should reuse matchesNoProxy/resolveEnvHttpProxyAgentOptions and current FormData normalization instead of adding duplicate proxy policy and an implicit loopback bypass.
