## Summary

Keep this PR open. Current main still lacks target-aware media-runner proxy selection for configured audio/video backends, but the branch needs maintainer/security follow-up before merge because it duplicates proxy semantics that have since moved on and introduces direct loopback media egress under proxy-enabled runtimes.

## What This Changes

The PR makes media proxy-fetch resolution target-aware, moves audio/video fetchFn selection after backend baseUrl resolution, and adds proxy-fetch plus media-runner regression tests for NO_PROXY-covered audio backends.

## Best Possible Solution

Update the branch against current main by resolving the media provider baseUrl before choosing fetchFn, centralizing target-aware proxy/NO_PROXY logic in the shared proxy-env layer or a single documented helper, preserving current ALL_PROXY support, and making any media loopback bypass explicit, tested, and aligned with the managed-proxy security contract.

## Reproduction Assessment

Yes. The static reproduction path is high confidence: with HTTPS_PROXY set and an audio provider baseUrl covered by NO_PROXY, current main still creates the proxy fetch before it knows the baseUrl and passes that fetch into the OpenAI-compatible STT path. I did not run tests because this was a read-only sweep.

## Solution Assessment

No. The PR is aimed at the right bug, but the current branch is not the best final shape because it duplicates proxy matching semantics, diverges from current main's ALL_PROXY helper, and opens a loopback direct path that conflicts with the documented managed-proxy contract unless explicitly approved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Keep media loopback behind an explicit proxy bypass:** `src/infra/net/proxy-fetch.ts:136-140`
  - body: The new matcher returns direct fetch for localhost, ::1, and 127.0.0.0/8 before checking NO_PROXY. In managed proxy mode OpenClaw clears NO_PROXY so loopback/internal destinations cannot bypass the filtering proxy, and the docs limit direct loopback to Gateway control-plane traffic. This would make normal media transcription traffic bypass operator egress controls without an explicit media-specific opt-in.
  - confidence: 0.86
- **[P2] Preserve ALL_PROXY-backed proxy fetches:** `src/infra/net/proxy-fetch.ts:205`
  - body: Current main builds EnvHttpProxyAgent options with an ALL_PROXY fallback because Undici does not read ALL_PROXY itself. This replacement gates proxy fetch creation on hasEnvHttpProxyConfigured("https"), which ignores ALL_PROXY-only environments, so those deployments would stop receiving the proxy fetch and can route media/OpenRouter requests differently than current main.
  - confidence: 0.74

## Security Review
