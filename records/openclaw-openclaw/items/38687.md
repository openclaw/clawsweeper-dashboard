## Summary

Keep open: current main still lacks target-aware media backend proxy resolution for CIDR-style NO_PROXY entries, but this PR should not merge as-is because it duplicates newer proxy helpers, changes loopback egress policy, drops ALL_PROXY-backed routing, and lacks inspectable real behavior proof.

## What This Changes

The PR makes media-understanding audio/video proxy fetch resolution target-aware using configured backend baseUrls and adds proxy-fetch and media runner tests for NO_PROXY bypass cases.

## Best Possible Solution

Reuse the shared proxy-env matcher/options path, pass the resolved backend baseUrl into target-aware media fetch resolution, preserve FormData and ALL_PROXY behavior, and avoid any new loopback media bypass unless maintainers explicitly approve that policy.

## Reproduction Assessment

Yes, from source inspection. Current main resolves media fetch before the backend baseUrl is known, so OpenClaw's CIDR NO_PROXY matcher cannot be applied to the configured media backend; I did not run a live STT setup in this read-only review.

## Solution Assessment

No. The PR moves resolution to the right runner point, but duplicates proxy matching and regresses the current shared proxy contract; the narrower maintainable fix is to reuse existing proxy-env helpers and keep egress policy unchanged.
