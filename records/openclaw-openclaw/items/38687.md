## Summary

Keep open: current main still has the media proxy bug, but this PR head is stale and would regress current proxy semantics before it is safe to merge.

## What This Changes

The PR makes media audio/video proxy fetch resolution target-aware and adds NO_PROXY matching tests so self-hosted media backends can bypass proxy wrapping.

## Best Possible Solution

Rework the branch against current main so media audio/video resolve proxy fetches after baseUrl using shared proxy helpers, preserving ALL_PROXY, FormData normalization, and explicit NO_PROXY/CIDR-only bypass behavior.

## Reproduction Assessment

Yes, by source inspection: configure media audio with a baseUrl covered by NO_PROXY plus HTTPS_PROXY, and current main still builds fetchFn before the baseUrl is known. I did not run tests because this review was read-only and pnpm is unavailable here.

## Solution Assessment

No. Moving media fetch resolution after baseUrl is the right direction, but this patch duplicates proxy matching, hard-codes loopback bypass, and drops current ALL_PROXY/FormData behavior; the safer fix is to extend the shared proxy layer and preserve current semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep media loopback behind explicit proxy opt-out:** `src/infra/net/proxy-fetch.ts:136-140`
  - body: The new matcher returns direct fetch for localhost, ::1, and 127.0.0.0/8 before checking NO_PROXY. Managed proxy docs only allow direct loopback for Gateway control-plane RPC, while normal runtime HTTP/WebSocket traffic stays behind the configured proxy, so media transcription to loopback would bypass operator egress controls without an explicit opt-out.
  - confidence: 0.9
- **[P2] Preserve current ALL_PROXY-backed proxy fetches:** `src/infra/net/proxy-fetch.ts:204-216`
  - body: Current main builds EnvHttpProxyAgent with resolveEnvHttpProxyAgentOptions(env), including ALL_PROXY/all_proxy fallback. This branch gates on hasEnvHttpProxyConfigured("https") and then constructs EnvHttpProxyAgent() without those options, so ALL_PROXY-only deployments can lose proxy routing for affected callers.
  - confidence: 0.88
- **[P2] Keep FormData normalization in env proxy fetches:** `src/infra/net/proxy-fetch.ts:217-219`
  - body: Current main normalizes global FormData-like bodies before undici dispatch so multipart uploads survive proxy fetch wrapping. The PR head's env proxy fetch spreads init directly into undiciFetch, which can regress the multipart audio upload path this PR is trying to fix when the request still uses a proxy.
  - confidence: 0.78
