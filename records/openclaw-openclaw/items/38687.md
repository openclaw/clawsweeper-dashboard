## Summary

Keep open. The target-aware media proxy bug is still source-reproducible on current main, but this PR has correctness/security blockers and lacks inspectable after-fix real STT proof.

## What This Changes

The PR moves media audio/video proxy fetch resolution after backend baseUrl resolution and adds custom NO_PROXY, loopback, and CIDR bypass logic with proxy tests.

## Best Possible Solution

Rework on current main so media resolves proxy fetch target-aware through the shared proxy-env matcher, bypasses only explicit NO_PROXY matches, preserves EnvHttpProxyAgent options and multipart normalization, then attach redacted live STT proof.

## Reproduction Assessment

Yes, source-reproducible. Current main creates the media fetchFn before backend baseUrl is known, and Undici's EnvHttpProxyAgent does not implement OpenClaw's CIDR NO_PROXY extension for the final fetch path.

## Solution Assessment

No. Passing baseUrl into the decision is the right seam, but this branch duplicates proxy matching and drops current ALL_PROXY and multipart behavior instead of reusing the shared proxy contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Require explicit NO_PROXY before loopback bypass:** `src/infra/net/proxy-fetch.ts:136-140`
  - body: This returns direct fetch for localhost, ::1, and 127.0.0.0/8 before consulting NO_PROXY. The managed-proxy docs limit direct loopback to Gateway control-plane traffic, so media provider traffic should stay on the operator proxy unless explicitly excluded.
  - confidence: 0.89
- **[P2] Preserve env proxy option mapping:** `src/infra/net/proxy-fetch.ts:204-216`
  - body: This gates only on `hasEnvHttpProxyConfigured("https")` and constructs `EnvHttpProxyAgent` with no options, while current main maps ALL_PROXY into explicit agent options for proxy fetches. ALL_PROXY-only media provider deployments would get no `fetchFn` and go direct.
  - confidence: 0.9
- **[P2] Keep multipart normalization on proxy fetches:** `src/infra/net/proxy-fetch.ts:217-220`
  - body: The wrapper forwards raw `init` into `undiciFetch`. Current proxy-fetch normalizes headers and FormData before dispatch, so proxied audio transcription can regress to stale multipart headers or incompatible FormData bodies.
  - confidence: 0.86
