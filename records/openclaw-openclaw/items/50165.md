## Summary

Keep this open: current main still treats the child agent-run terminal outcome as the subagent completion boundary, while child-owned background exec/process work remains a separate scoped registry and can outlive that agent turn.

## What This Changes

Requests that subagent completion/status stay in progress until delegated child work, including long-running background exec/process work, is actually finished.

## Best Possible Solution

Define the lifecycle contract explicitly, then implement one coherent path: either keep subagent completion tied to the child agent run and surface child-owned background processes separately in task/status views, or defer completion through a generic owned-process liveness seam with timeout, cancel, cleanup, and regression coverage.

## Reproduction Assessment

Yes at source level: current main can complete a subagent from the child agent-run outcome while background exec/process sessions remain tracked separately. No live OS reproduction was run in this read-only review.

## Solution Assessment

Unclear. The requested behavior is reasonable, but the best fix depends on whether child-owned background processes are part of subagent completion or a separate active-work/status surface.
