## Summary

Keep open. Current main and the latest release have important descendant announce gating, but the broader state contract is still not settled: subagent task status is finalized at child lifecycle completion before cleanup, descendant delivery, and background-work semantics have fully converged.

## What This Changes

The issue asks OpenClaw to avoid showing delegated subagent work as complete until the real user-relevant work and result pipeline are finished.

## Best Possible Solution

Define one lifecycle contract for subagent-visible status: keep the task/session state active or explicitly `waiting/ready for parent review` until descendants, required completion delivery, and any supported background-work completion semantics have settled, then update registry, task status, and session status together.

## Reproduction Assessment

Yes, source-reproducible for the status-ordering part: current main creates a running subagent task, marks it succeeded on child lifecycle completion, and only afterward can cleanup defer for pending descendants. The background-exec part is reported in comments but still needs a live focused repro.

## Solution Assessment

No, the current descendant announce gating is necessary but not sufficient for the broader issue. The best fix needs a maintainer decision on whether terminal subagent status means child LLM turn ended, parent-review-ready, delivery-complete, or all delegated/background work complete.
