## Summary

Keep this open. Current main still finalizes subagent task completion from the child agent-run terminal outcome, while child-owned background exec/process sessions are tracked separately and can outlive that terminal state; related merged fixes address descendant subagent announce timing and embedded-run cleanup, not this background-process lifecycle contract.

## What This Changes

The issue asks OpenClaw to avoid reporting a subagent as completed until delegated long-running or background work is actually finished.

## Best Possible Solution

Define the lifecycle contract explicitly, then implement one coherent path: either keep subagent completion agent-run authoritative and surface child-owned background processes as separate active work in `/tasks` and `session_status`, or defer subagent finalization through a generic owned-process liveness seam with timeout, cancellation, cleanup, and regression coverage.

## Reproduction Assessment

Unclear: the report lacks focused lifecycle logs, but current code supports a narrow harness where a subagent starts a yielded/background exec process, returns successfully, and the subagent task finalizes before the scoped process exits.

## Solution Assessment

Unclear because no patch is proposed and the right fix depends on maintainer lifecycle policy; a generic owned-work contract is safer than a subagent-only special case.
