## Label Justifications

- `P2`: The report is a real subagent lifecycle reliability bug with limited blast radius and a product-decision blocker, not an emergency outage.
- `impact:session-state`: The core symptom is parent/subagent lifecycle state drifting from whether delegated child work is still active.
- `impact:message-loss`: False completion can cause users or parent agents to stop monitoring before the final result is delivered or acted on.

## Summary

Keep open: current main has stronger subagent task and announce tracking, but it still finalizes subagent completion from the child agent-run outcome rather than all child-owned background work that users may see as part of the delegated task.

## What This Changes

The issue asks OpenClaw to keep subagent/session/task state in progress until delegated work and user-relevant results are actually finished.

## Best Possible Solution

Define one lifecycle contract: either subagent-owned background work keeps the subagent task running until terminal, or it is promoted to a separate tracked task that parent/UI state cannot mistake for completed subagent work.

## Reproduction Assessment

Source-level yes, live no: current main can complete a subagent from waitForAgentRun/completeSubagentRun while background exec/process sessions remain separate and do not keep the subagent run live.

## Solution Assessment

No current implementation fully solves the reported boundary; descendant gating handles nested subagents and #66749 avoids spurious wakes, but neither makes arbitrary child-owned background exec/process work part of subagent completion.
