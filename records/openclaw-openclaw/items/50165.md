## Summary

Keep this open. Current main still finalizes subagent task state from the child agent-run terminal outcome, while background exec/process sessions are managed by a separate scoped process registry and can outlive that agent turn. Related merged work improved descendant subagent announce timing and embedded-run cleanup ordering, but it does not add a generic child-owned process liveness gate or a clearly separate active-work surface for this report.

## What This Changes

The issue requests reliable subagent lifecycle semantics so a subagent is not shown as complete before delegated background work is actually finished.

## Best Possible Solution

Define the lifecycle contract explicitly, then implement one coherent path: either keep agent-run completion authoritative and surface child-owned background exec/process sessions as separate active work in `/tasks` and `session_status`, or defer subagent finalization through a generic owned-process liveness seam with timeout, cancellation, cleanup, and regression coverage.

## Reproduction Assessment

Yes, at code level: a child subagent can start an exec with `background: true` or `yieldMs`, receive a `status: "running"` process session, return successfully, and then have its subagent task finalized from the agent-run outcome while that process remains separately tracked. I did not run a live harness because this review is read-only.

## Solution Assessment

Unclear: no patch is proposed, and the narrowest maintainable fix depends on whether OpenClaw wants background exec/process sessions to block subagent completion or remain a separate active-work channel. A contract decision should come before implementation.
