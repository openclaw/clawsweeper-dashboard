## Summary

Keep this open. Current main and v2026.5.18 still let child-owned background exec/process liveness outlive subagent task finalization; the workaround discussion does not fix the lifecycle contract, and the related exec-liveness PR is still open and unmerged.

## What This Changes

The issue reports that subagent/session status can show completion before delegated child work, especially background exec/process work, is actually finished.

## Best Possible Solution

Define one lifecycle contract, then either surface child-owned background exec/process sessions as separate active work in task/status views or gate subagent finalization through a generic owned-work liveness seam with timeout, cancellation, cleanup, and regression coverage.

## Reproduction Assessment

Yes at source level, but not as a live harness reproduction. A child subagent can start a background exec that returns status="running" and a process sessionId, then the subagent run can be finalized from the agent-run outcome without checking that process session.

## Solution Assessment

Unclear because no patch is proposed and the correct behavior is a runtime contract decision. A coherent owned-work liveness model or separate active-work surfacing is safer than a narrow subagent-only wait hack.
