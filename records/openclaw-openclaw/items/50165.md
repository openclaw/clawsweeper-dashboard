## Summary

Keep this open. Current main still treats the child agent run as the authoritative completion boundary, while yielded/background exec processes are scoped and tracked separately, so the reported lifecycle-contract gap is not closed by the existing task/status work.

## What This Changes

The issue requests that subagent completion/status remain in progress until all user-relevant delegated work, including child-owned background exec/process work, is actually finished.

## Best Possible Solution

Define the lifecycle contract explicitly, then either keep agent-run completion authoritative while surfacing child-owned background processes as separate active work, or add a generic owned-work liveness gate with timeout, cancel, cleanup, and regression coverage.

## Reproduction Assessment

Unclear as a live reproduction: the issue lacks focused logs and I did not run a harness. Source inspection gives a concrete path where a subagent starts a yielded/background exec process, returns successfully, and the subagent task finalizes while the scoped process can remain running.

## Solution Assessment

Unclear because no patch is proposed and current docs make agent-run completion authoritative. A generic owned-work contract or explicit separate active-work surfacing would be safer than a subagent-only special case.
