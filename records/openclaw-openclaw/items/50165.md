## Summary

Keep open. Current main still finalizes subagent completion from the child agent-run outcome while child-owned background exec/process sessions are tracked separately, so the reported lifecycle-contract gap is not closed on main.

## What This Changes

The report asks OpenClaw to avoid showing a delegated subagent as complete while its child-owned background work is still running.

## Best Possible Solution

Define the lifecycle contract explicitly, then implement one coherent path: either surface child-owned background exec/process sessions as separate active work in task/status views, or defer subagent finalization through an owned-work liveness gate with timeout, cancel, cleanup, and regression coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a live run in this review; source inspection shows a concrete path where a subagent can start a background exec, return successfully, and finalize while the scoped process session remains running.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the completion contract; a generic owned-work or active-work status seam is safer than a subagent-only special case.
