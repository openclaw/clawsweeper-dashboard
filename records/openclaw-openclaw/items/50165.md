- `impact:session-state`: The reported false completion means parent/subagent task state can say terminal while delegated child-owned work is still active.

## Summary

Keep open: current main has descendant-subagent announce gating and task/status visibility, but subagent task completion is still agent-run authoritative while child-owned background exec/process sessions are tracked separately and can outlive the child agent turn.

## What This Changes

The issue reports that subagents can be shown as completed before delegated background work or user-visible outputs are actually finished.

## Best Possible Solution

Define one lifecycle contract, then either keep subagent completion agent-run authoritative and surface child-owned process liveness as separate active work, or defer subagent finalization through a generic owned-process liveness seam with timeout and cancellation rules.

## Reproduction Assessment

No live reproduction was run in this read-only review. Source inspection gives a credible path: a subagent can complete from waitForAgentRun while yielded/backgrounded exec work remains in the separate process registry.

## Solution Assessment

Unclear. Current main handles descendant subagent announce timing, but the best fix for child-owned background processes needs maintainers to choose whether those processes block subagent completion or remain separately visible active work.
