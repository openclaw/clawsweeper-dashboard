## Label Justifications

- `P2`: The issue reports a real subagent lifecycle reliability gap with limited blast radius and a product-contract blocker.
- `impact:session-state`: The parent/subagent state can report terminal completion while delegated process work is still active.
- `impact:message-loss`: False completion can cause users or parent agents to stop monitoring before final results or follow-up messages are ready.

## Summary

Keep open: current main has descendant-subagent announce gating and related lifecycle fixes, but it still leaves child-owned background exec/process work as separate state that can outlive the child agent run and make subagent completion appear too early.

## What This Changes

The issue requests reliable subagent completion semantics so delegated work, including long-running background exec or file-producing work, is not reported complete before the user-relevant result is ready.

## Best Possible Solution

Define one lifecycle contract, then either surface child-owned background processes as separate active work or defer subagent terminal state through a generic owned-process liveness seam with bounded timeout and cleanup rules.

## Reproduction Assessment

Yes at source level, not live: current main can finalize a subagent from the child agent-run outcome while background exec/process sessions remain separately tracked and running.

## Solution Assessment

No automatic fix should be selected yet; the correct implementation depends on whether background processes are part of subagent completion or separately surfaced active work.
