## Summary

Keep open: current main has descendant-subagent announce gating and task/status visibility, but the remaining report is still a real lifecycle-contract gap for child-owned background exec/process work that can outlive the child agent turn.

## What This Changes

The issue reports that parent/session state can show a delegated subagent as complete before the user-relevant delegated work or output pipeline is actually finished.

## Best Possible Solution

Define one lifecycle contract, then either keep subagent completion agent-run authoritative and surface child-owned process liveness separately, or defer subagent terminal state through a generic owned-process liveness seam with timeout and cancellation rules.

## Reproduction Assessment

Source-level yes, live no: current main can finalize a subagent from the agent-run wait outcome while background exec/process sessions remain separate. I did not run a live Windows or Linux reproduction in this read-only review.

## Solution Assessment

No: current fixes handle descendant subagent announce timing and status visibility, but not the ambiguous contract for child-owned background process work. The narrowest maintainable solution needs a maintainer decision on that contract first.
