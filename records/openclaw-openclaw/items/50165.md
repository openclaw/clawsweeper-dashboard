## Summary

Keep this open. Current main still finalizes the subagent task from the child agent-run terminal result, while child-owned background exec/process sessions are scoped and tracked separately; that leaves the reported lifecycle contract gap unresolved rather than implemented or stale.

## What This Changes

The issue asks OpenClaw to keep subagents in progress until delegated background or external work is actually complete.

## Best Possible Solution

Define the lifecycle contract and implement one coherent path: either keep agent-run completion canonical while clearly surfacing child-owned processes as separate active work, or add a generic owned-work liveness gate with timeout, cancellation, cleanup, and regression coverage.

## Reproduction Assessment

Unclear as a live reproduction: the issue lacks focused lifecycle logs and I did not run a harness. Source inspection gives a concrete path where a subagent starts a background/yielded exec process, returns successfully, and the subagent task finalizes while the scoped process remains running.

## Solution Assessment

Unclear because no patch is proposed and current docs make agent-run completion authoritative. A generic owned-work contract or explicit separate task/status surfacing would be safer than a subagent-only special case.
