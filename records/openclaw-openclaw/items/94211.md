## Summary

Keep this PR open, but the patch should not merge as written: it fixes the reported duplicate-memory symptom with a broad consecutive text check that can silently drop legitimate assistant content, while current code already exposes narrower lineage and transcript-only signals for the same case.

## What This Changes

The PR changes `src/hooks/bundled/session-memory/transcript.ts` to skip consecutive assistant messages whose extracted text exactly matches the previous assistant text.

## Best Possible Solution

Land one canonical fix that filters only transcript-only or parent-linked cleaned assistant rows, with a regression test and real memory-file proof; do not land broad text-only deduplication.

## Reproduction Assessment

Yes at source level: current main extracts text from each user/assistant JSONL message and the session-memory handler writes that transcript into memory files, but no live provider reproduction was run in this read-only review.

## Solution Assessment

No: the proposed text-equality filter is broader than necessary because the current codebase has transcript-only metadata helpers and transcript rows carry lineage fields that can distinguish cleaned duplicates from legitimate same-text assistant messages.
