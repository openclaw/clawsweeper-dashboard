## What This Changes

The branch adds a `lastAssistantText` guard in session-memory transcript extraction to skip assistant messages whose extracted text matches the previous assistant text.

## Best Possible Solution

Land one canonical consumer-layer session-memory filter that skips actual transcript mirror duplicates, preserves legitimate repeated assistant content, and includes focused regression coverage plus direct after-fix proof.

## Reproduction Assessment

Yes. Current main reads every assistant text row into session-memory output, so a raw thinking row followed by a cleaned transcript row emits duplicate assistant lines at the reader boundary.

## Solution Assessment

No. The consumer layer is plausible, but a plain assistant text latch is broader than the mirror-row bug; metadata or parent-lineage filtering better matches sibling transcript contracts.
