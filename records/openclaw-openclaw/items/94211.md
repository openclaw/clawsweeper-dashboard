## Summary

Keep this PR open, but it is not merge-ready: the text-equality dedupe can drop valid repeated assistant lines from generated session-memory artifacts, and the linked issue still needs one canonical semantic fix chosen from overlapping PRs.

## What This Changes

The PR adds a `lastAssistantText` guard in `getRecentSessionContent()` to skip assistant messages whose extracted text matches the previously stored assistant text.

## Best Possible Solution

Land one canonical session-memory fix that filters only actual raw/cleaned mirror rows, preserves legitimate repeated assistant content, and includes regression coverage plus direct after-fix behavior proof.

## Reproduction Assessment

Yes. At source level, current main extracts every assistant text row from JSONL, and the proposed guard remains armed after user rows, so a transcript with `assistant: OK`, `user: repeat`, `assistant: OK` would lose the second assistant line.

## Solution Assessment

No. The consumer layer is plausible, but plain text equality is too broad; a transcript metadata or parent-lineage filter is narrower and matches existing sibling contracts.
