## Summary

Keep this PR open for now: it targets a real session-memory duplication bug, but the proposed text-only deduplication is broader than the storage artifact and can silently drop valid same-text assistant rows. The branch also only supplies unit-test proof, while a user-visible memory-file fix still needs redacted real `/new` or `/reset` output/log proof before merge.

## What This Changes

The PR adds a `lastAssistantText` guard in `src/hooks/bundled/session-memory/transcript.ts` to skip consecutive assistant messages with identical extracted text.

## Best Possible Solution

Land one canonical consumer-layer fix that filters only transcript-only or parent-linked cleaned assistant rows, with focused regression coverage and redacted real memory-file proof.

## Reproduction Assessment

Yes, at source level. Current main extracts text from every user/assistant JSONL message and writes that transcript into `/new` or `/reset` memory files; I did not run a live thinking-provider session in this read-only review.

## Solution Assessment

No. The PR is a plausible mitigation, but the best fix should use transcript-only metadata or parent-linked lineage instead of suppressing rows solely because their assistant text matches.
