## Summary

Keep open: the linked session-memory bug is real, but this PR is not merge-ready because the text-only guard can drop valid repeated assistant lines and the contributor proof is only test output, not real generated memory-file behavior.

## What This Changes

The PR adds a `lastAssistantText` guard in session-memory transcript extraction to skip assistant rows whose extracted text matches the previous assistant text.

## Best Possible Solution

Land one canonical consumer-layer fix that filters only actual transcript-mirror or raw-cleaned duplicate rows, preserves legitimate repeated assistant replies, adds regression coverage, and includes redacted real memory-file proof.

## Reproduction Assessment

Yes, at source level. Current main extracts and saves every assistant text row from the JSONL reader, so a raw thinking row plus a cleaned mirror row will duplicate in session-memory output; no live provider flow was run in this read-only review.

## Solution Assessment

No. This is a plausible workaround, but text equality is broader than the reported raw/cleaned transcript artifact and the existing codebase already has semantic transcript-only helpers and lineage metadata to target the real duplicate shape.
