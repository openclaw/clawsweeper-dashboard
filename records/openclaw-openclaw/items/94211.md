## Summary

Keep open: this PR targets a real session-memory duplication bug, but the proposed text-only deduplication is broader than the transcript artifact and can suppress valid assistant content. Current main and the latest release still have the underlying bug, and open linked PRs are candidate fixes rather than merged implementations.

## What This Changes

The branch changes `src/hooks/bundled/session-memory/transcript.ts` to remember the previous assistant text and skip consecutive assistant messages with identical extracted text.

## Best Possible Solution

Land one canonical fix that filters only transcript-only or parent-linked cleaned assistant rows, with regression coverage and redacted real memory-file proof.

## Reproduction Assessment

Yes at source level. A JSONL transcript containing a raw assistant text block followed by a transcript-only assistant copy will be read twice by current `getRecentSessionContent()`, but this read-only review did not run a live thinking-provider `/new` or `/reset` flow.

## Solution Assessment

No. The current PR is only a plausible workaround because text equality is broader than the storage artifact; the safer fix is to filter transcript-only metadata or parent-linked cleaned rows.
