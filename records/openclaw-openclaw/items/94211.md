## Summary

## What This Changes

This PR adds a `lastAssistantText` guard in `getRecentSessionContent()` to skip assistant messages whose extracted text matches the previously accepted assistant text.

## Best Possible Solution

Land one canonical consumer-layer session-memory fix that filters only actual duplicate transcript mirror rows, preserves legitimate repeated assistant content, and includes focused regression coverage plus real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main extracts every assistant text row from JSONL, so a raw thinking row followed by a cleaned text row duplicates in memory output; the PR's guard bug is also visible from unchanged `lastAssistantText` state across accepted user rows.

## Solution Assessment

No. The consumer layer is plausible, but plain text equality is too broad and this implementation does not reset across user turns; a metadata or parent-lineage filter is the narrower maintainable fix.
