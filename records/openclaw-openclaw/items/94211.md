## Summary

Keep open: this is a real session-memory bug candidate, but the PR should not merge as-is because the text guard can drop valid repeated assistant replies and the contributor proof is only a test run.

## What This Changes

The PR adds a `lastAssistantText` guard in `getRecentSessionContent()` to skip assistant messages whose extracted text matches the previously stored assistant text.

## Best Possible Solution

Land one canonical session-memory fix that filters only actual raw/cleaned mirror rows, preserves legitimate repeated assistant content, and includes regression coverage plus direct after-fix behavior proof.

## Reproduction Assessment

Yes. At source level, current main extracts every assistant text row from JSONL, and the patch regression is visible because the new `lastAssistantText` state is not cleared across accepted user rows.

## Solution Assessment

No. The consumer layer is plausible, but plain text equality is too broad and this implementation does not reset across user turns; a metadata or parent-lineage filter is narrower and more maintainable.
