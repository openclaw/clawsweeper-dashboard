Keep open: the patch targets a real session-memory duplicate-output bug, but the plain text dedupe guard can drop valid repeated assistant content, the PR only supplies test-output proof, and maintainers still need to choose one canonical fix among overlapping candidates.

## What This Changes

Adds a `lastAssistantText` guard in `src/hooks/bundled/session-memory/transcript.ts` to skip assistant rows whose extracted text matches the previous assistant text.

## Best Possible Solution

Land one canonical session-memory filter that skips only actual raw/cleaned mirror rows, preserves valid repeated assistant text, and includes regression coverage plus direct after-fix memory-output proof.

## Reproduction Assessment

Yes, at source level. Current main appends every assistant text row to session-memory output, so a raw thinking row plus a stripped text row duplicates; I did not run a live thinking-provider flow in this read-only review.

## Solution Assessment

No. The consumer layer is plausible, but this specific plain text-equality guard is too broad; a transcript-mirror or parent-lineage filter is narrower and safer.
