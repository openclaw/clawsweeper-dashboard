## Summary

Keep open: this is a focused candidate fix for a real session-memory bug, but the text-only dedupe is broader than the reported raw/cleaned transcript artifact and can drop valid assistant memory content; proof is also limited to a Vitest run.

## What This Changes

The branch changes session-memory transcript extraction to remember the previous assistant text and skip consecutive assistant messages with identical extracted text.

## Best Possible Solution

Land one canonical consumer-layer fix that filters only actual transcript mirror duplicates while preserving legitimate repeated assistant text and visible gateway-injected rows, with regression coverage and redacted memory-file proof.

## Reproduction Assessment

Yes, at source level. Current main extracts and saves every assistant text row from JSONL, so a raw thinking row plus a cleaned/mirrored assistant row will duplicate in session-memory output.

## Solution Assessment

No. This PR is a plausible workaround, but text equality is broader than the reported transcript artifact; the safer fix is a transcript-mirror or parent-lineage filter with preservation tests.
