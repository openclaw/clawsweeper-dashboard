## Summary

Keep open for maintainer review, but the current patch is not merge-ready: it removes persisted overload cooldown windows that current main uses for auth-profile availability and bounded cooldown probing, and the PR still has no after-fix real overload/fallback proof.

## What This Changes

The PR changes auth-profile failure handling so provider `overloaded` failures increment failure stats without persisting `cooldownUntil`, and updates two overload fallback tests to expect no persisted cooldown.

## Best Possible Solution

Keep overload retry/backoff design on https://github.com/openclaw/openclaw/issues/49376; a replacement should preserve the cooldown/probe contract or explicitly replace it with maintainer-approved, tested behavior.

## Reproduction Assessment

Yes for the PR blocker: current main source and tests show `overloaded` failures persist `cooldownUntil` and use that state for bounded cooldown probes. I did not run live overload traffic in this read-only review.

## Solution Assessment

No. Removing persisted overload cooldown state is not the best fix; the maintainable path is a maintainer-approved retry/backoff contract that either preserves or deliberately replaces the current cooldown/probe behavior.
