## What This Changes

This PR changes auth-profile failure accounting so provider `overloaded` failures increment failure counts and set `cooldownReason` without persisting `cooldownUntil`, with two tests updated to expect repeated overload attempts.

## Best Possible Solution

Preserve the current overload rotation/backoff contract unless maintainers explicitly change it; a safe fix should prevent unwanted overload skips without clearing active cooldown windows or bypassing documented overload controls, and should include real overload/fallback proof.

No. The patch is a plausible mitigation for repeated overload skips, but it is not the best fix as written because it drops the broader overload rotation/backoff contract instead of preserving that contract or making an explicit documented behavior change.
