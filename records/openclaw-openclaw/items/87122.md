## Summary

Keep open, but this PR is not merge-ready: it removes persisted overload cooldown state that current main uses for auth-profile availability and bounded cooldown probes, while the broader overload retry/backoff direction remains on an open canonical product issue and the PR still lacks real behavior proof.

## What This Changes

The PR changes auth-profile failure handling so provider `overloaded` failures no longer persist `cooldownUntil`, and updates overload fallback tests to expect no persisted cooldown.

## Best Possible Solution

Keep overload retry/backoff design on https://github.com/openclaw/openclaw/issues/49376; a replacement should preserve the cooldown/probe contract or explicitly replace it with tested, maintainer-approved behavior.

## Reproduction Assessment

Yes for the PR blocker: source and tests on current main show `overloaded` failures persist `cooldownUntil` and use that state for bounded cooldown probes. I did not run live overload traffic in this read-only review.

## Solution Assessment

No. Removing persisted overload cooldown state is not the best fix; the maintainable path is a maintainer-approved retry/backoff contract that either preserves or deliberately replaces the current cooldown/probe behavior.
