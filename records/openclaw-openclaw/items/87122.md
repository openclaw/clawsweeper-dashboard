## Summary

## What This Changes

The branch changes overloaded auth-profile failures so they record failure metadata without persisting `cooldownUntil`, and updates two tests to expect no persisted overload cooldown.

## Best Possible Solution

No. Clearing `cooldownUntil` for overloaded failures is not the best fix; the maintainable path is a maintainer-approved retry/backoff contract that preserves or deliberately replaces the current cooldown/probe behavior.
