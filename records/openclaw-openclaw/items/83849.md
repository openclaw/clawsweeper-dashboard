## Summary

Keep open: this targets a real Codex/Docker sandbox regression, but the branch can disable Codex native Code Mode for Docker bind aliases while current dynamic-tool routing still withholds Docker shell/file tools, so the reported write path can remain unavailable.

## What This Changes

The PR adds Docker bind host roots to Codex app-server workspace-write policy, gates native Code Mode for Docker alias/shadow bind layouts, exports sandbox bind helpers, updates tests, and adds a changelog entry.

## Best Possible Solution

Mirror writable host roots for host-representable Docker binds, and when aliases or read-only shadows force native Code Mode off, provide an OpenClaw Docker sandbox tool path with regression coverage and real behavior proof of an agent writing through the bind.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main sends only the effective cwd as Codex `writableRoots`, while docs say writable Docker binds remain independent of `workspaceAccess`. I did not run a live Docker/Codex turn in this read-only review.

## Solution Assessment

No, not as-is. Mirroring writable host roots is the right direction, but disabling native Code Mode for container aliases needs a replacement Docker sandbox tool path or the central write workflow remains broken.
