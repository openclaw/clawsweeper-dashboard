## Summary

## What This Changes

The PR adds `ui:build` to the full and CI artifact build profiles after backend dist cleanup, removes now-duplicate prepack UI invocation, and updates docs, release hints, and regression tests for the new one-command full dist contract.

## Best Possible Solution

Land a single build-pipeline change that makes a full source/runtime `dist` complete while keeping backend-only watch/startup profiles separate and retaining release/prepack artifact verification.

## Reproduction Assessment

Yes by source inspection: current main's full and CI artifact build profiles omit `ui:build`, while the backend build cleans `dist` and the Control UI writes to `dist/control-ui`. I did not run the destructive build sequence because this review is read-only.

## Solution Assessment

Yes, if maintainers accept the build-contract change: placing `ui:build` after `tsdown`/runtime postbuild is the narrow implementation for a complete full `dist`. The safer alternative for backend-only automation is to keep using explicit minimal profiles rather than changing this PR's ordering.
