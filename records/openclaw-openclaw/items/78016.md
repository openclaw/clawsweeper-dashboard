## Summary

Keep open: current main still lacks Matrix-owned inbound audio preflight before the room mention gate, and the linked open PR is the active implementation candidate with closing syntax, so the issue should not be closed before that PR lands or is replaced.

## What This Changes

The issue requests Matrix inbound voice messages, especially in `requireMention` rooms, be transcribed before mention gating so the agent can answer spoken prompts.

## Best Possible Solution

Review, repair, and land a Matrix-owned implementation such as https://github.com/openclaw/openclaw/pull/78069 that reuses `transcribeFirstAudio`, preserves existing Matrix media/E2EE behavior, and carries the transcript into mention matching and `BodyForAgent`.

## Reproduction Assessment

Yes, source-level: current main drops unmentioned Matrix room media before download, and the existing Matrix test asserts media downloads are skipped in that path. I did not run a live Synapse/Element voice-note reproduction.

## Solution Assessment

Yes, the narrow maintainable direction is Matrix-plugin-owned audio preflight using the existing shared STT helper. The issue itself should remain open until the linked PR lands or maintainers choose a replacement implementation.
