## Summary

Keep open: current Matrix handling still gates room messages on typed mentions before downloading media or running audio preflight, so Matrix voice notes do not get the same pre-mention STT path that Discord and Telegram have.

## What This Changes

Add Matrix inbound voice-message transcription so audio-only Matrix messages can be understood by the agent like other supported channels.

## Best Possible Solution

Implement a Matrix-plugin-owned audio preflight that reuses the shared STT helper before mention gating, carries the transcript into agent/command text, marks the audio as already transcribed, and preserves existing E2EE media download behavior.

## Reproduction Assessment

Yes for a source-level path: an m.audio Matrix room event without typed bot mention follows the requireMention branch and returns before media download or STT. No live Matrix/Element run was performed in this read-only review.

## Solution Assessment

Yes: the narrowest maintainable fix is Matrix-owned preflight transcription using the existing shared helper and the Discord/Telegram patterns, not a new core media-understanding path or config option.
