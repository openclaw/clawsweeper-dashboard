## Summary

Keep open: the current Matrix handler still does not transcribe inbound Matrix audio before the `requireMention` gate on `main`, and an open contributor PR already references this issue with closing syntax, so the issue should stay open until that PR is reviewed, proved, and either merged or replaced.

## What This Changes

The issue requests Matrix inbound voice messages to be transcribed and used for mention-gated agent dispatch like other channels.

## Best Possible Solution

Review PR https://github.com/openclaw/openclaw/pull/78069 or land an equivalent Matrix-owned preflight that downloads/transcribes eligible audio before mention gating and carries the transcript plus transcribed index into the agent context.

## Reproduction Assessment

No live high-confidence reproduction was run. The current-main source path is reproducible: Matrix decides mention eligibility from text before media download/transcription, so a voice note that only speaks the bot name in a `requireMention` room is skipped before STT can run.

## Solution Assessment

Yes, the best direction is the Matrix-owned preflight already proposed by the linked PR or an equivalent patch using the shared `transcribeFirstAudio` helper. That keeps the fix inside the channel/plugin boundary and avoids core media-understanding or new config changes.
