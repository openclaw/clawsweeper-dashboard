## Summary

Keep open: current main still gates Matrix media before download and does not have a Matrix audio-preflight path, and the open linked implementation candidate at https://github.com/openclaw/openclaw/pull/78069 should be reviewed or replaced before this issue closes.

## What This Changes

Request Matrix voice messages, especially in `requireMention` rooms, be transcribed before mention gating so spoken bot mentions can trigger the agent.

## Best Possible Solution

Land a Matrix-plugin-owned implementation that reuses the shared audio preflight helper before mention gating, preserves existing media/E2EE handling, sets transcript reuse metadata, and documents the STT data flow.

## Reproduction Assessment

No live high-confidence Matrix/Element reproduction was run here. Source inspection gives a clear path: a Matrix room voice message with `requireMention: true` reaches mention gating before media download or STT, so spoken bot mentions cannot be matched.

## Solution Assessment

Yes, the narrow maintainable direction is Matrix-owned audio preflight using the existing shared helper, not a core media-understanding rewrite or a new config option. The issue should stay open while https://github.com/openclaw/openclaw/pull/78069 is reviewed or replaced.
