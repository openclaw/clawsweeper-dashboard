## Summary

Keep open: current main still drops Matrix room voice notes before any audio transcription can feed the mention gate, and the linked implementation PR is open, unmerged, and explicitly closes this issue.

## What This Changes

Matrix voice notes in mention-gated rooms should be transcribed before mention checking so spoken bot names or aliases can trigger the agent and provide transcript text.

## Best Possible Solution

Review, validate, and land a Matrix-plugin-owned audio preflight fix that reuses `transcribeFirstAudio` before mention gating while preserving existing media download and E2EE behavior.

## Reproduction Assessment

Yes, at source level: an `m.audio` room event with only spoken mention content reaches the Matrix `requireMention` gate before any media download or STT path, so current main cannot match the spoken alias. I did not run a live Matrix/Element repro.

## Solution Assessment

Yes: the narrow maintainable fix is Matrix-owned preflight transcription using the existing shared helper, not a core media-understanding rewrite or a new config option.
