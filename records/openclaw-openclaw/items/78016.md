## Summary

Keep open: current main still gates Matrix room messages on text mentions before downloading or transcribing media, and the open closing pull request at https://github.com/openclaw/openclaw/pull/78069 has not landed. The issue should stay open until that PR is merged or the implementation path is replaced.

## What This Changes

Matrix inbound voice notes should be transcribed before mention gating so spoken bot mentions reach the agent in requireMention rooms.

## Best Possible Solution

Land a Matrix-owned preflight implementation that downloads allowed audio before the mention gate, runs the shared audio transcription helper, feeds surfaced transcripts into mention and agent-body context, marks only surfaced transcripts as consumed, and preserves Matrix media/E2EE behavior.

## Reproduction Assessment

Yes, at source level: create a Matrix room with requireMention enabled and send a voice note whose only bot mention is spoken. Current main resolves mentions from text and returns before media download or transcription.

## Solution Assessment

Yes, the maintainable direction is Matrix audio preflight using the existing shared helper; the open closing PR should be reviewed, rebased, proven in a real Matrix setup, and then landed or replaced.
