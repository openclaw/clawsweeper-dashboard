## Summary

Keep open: current main still gates Matrix room media before download/transcription, so a voice-only message whose bot mention exists only in audio cannot pass `requireMention`; the issue is already paired with an open fixing PR and should close only after that or a replacement lands.

## What This Changes

The issue asks for Matrix inbound voice notes to be transcribed before mention gating and agent dispatch, matching other channels.

## Best Possible Solution

Land or replace the linked Matrix-owned preflight implementation so Matrix downloads eligible audio before the mention gate, feeds the transcript into mention detection and `BodyForAgent`, sets `MediaTranscribedIndexes`, and preserves existing Matrix media/E2EE behavior.

## Reproduction Assessment

No live high-confidence reproduction was run, but the current-main source path is high-confidence: Matrix drops mention-gated room media before media download or STT. A focused test can reproduce this with an `m.audio` room event whose spoken transcript contains the bot mention.

## Solution Assessment

Yes, the best direction is Matrix-plugin-owned audio preflight using the existing shared helper, not a core media rewrite or new config surface. Because https://github.com/openclaw/openclaw/pull/78069 is open with closing syntax, the issue should remain open until that PR or a replacement lands.
