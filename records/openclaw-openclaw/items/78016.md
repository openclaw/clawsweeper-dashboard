## Summary

Keep open: current `main` still gates Matrix room messages before any audio download or transcription, and the issue is explicitly paired with an open, unmerged implementation PR. The latest discussion narrows the failing case to Matrix groups with `requireMention: true`, where the mention exists only in the voice transcript.

## What This Changes

The issue requests Matrix inbound voice messages to be transcribed early enough for agent input and spoken mention-gating in rooms.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/78069, or an equivalent Matrix-owned preflight transcription change, so Matrix audio is transcribed after access checks but before mention gating and downstream media understanding can skip duplicate STT.

## Reproduction Assessment

Yes for source-level reproduction: current Matrix code skips `requireMention` room media before download/transcription, and the reporter confirmed the voice-only mention path. No live Matrix/Element reproduction was run in this review.

## Solution Assessment

Yes: the maintainable direction is a narrow Matrix plugin change that reuses the existing `transcribeFirstAudio` preflight contract and keeps core media understanding generic. The open PR is the current implementation candidate, pending its own review/proof blockers.
