## Summary

Keep open: current main still gates Matrix room media before any audio transcription can run, and an open linked PR directly targets this issue with closing syntax, so the issue should remain open until that PR lands or is closed.

## What This Changes

The issue requests Matrix inbound voice notes to be transcribed before agent dispatch, especially in `requireMention` group rooms where the bot is mentioned only in speech.

## Best Possible Solution

Land a Matrix-plugin-owned preflight implementation through the linked PR or a replacement that reuses the shared audio helper, keeps allowlist checks before STT, and marks transcribed media to avoid duplicate STT.

## Reproduction Assessment

Yes at source level: an `m.audio` Matrix room message with `requireMention: true` and no typed caption is evaluated for mentions and skipped before media download or STT can run. No live Matrix/Element run was performed here.

## Solution Assessment

Yes: the best direction is the narrow Matrix-owned preflight path already proposed in the linked PR, not a core media-understanding change or a new config option.
