## Summary

Keep open: current Matrix code saves inbound media, but it lacks the Matrix-specific audio preflight path that comparable channels use so voice-only messages can become transcript text before mention gating and agent dispatch.

## What This Changes

The issue requests inbound Matrix voice-message handling so the agent can transcribe and answer user audio messages.

## Best Possible Solution

Implement Matrix-plugin-owned voice-note preflight that reuses the shared audio transcription helper, updates the agent-facing text when STT succeeds, and keeps existing media download/E2EE behavior intact.

## Reproduction Assessment

No live high-confidence reproduction was run, but the source path is reproducible: Matrix handles audio as media/bodyText and skips unmentioned group media before download, with no Matrix transcript preflight or MediaTranscribedIndexes path. A focused unit test can cover an m.audio DM and a requireMention room voice note whose transcript mentions the bot.

## Solution Assessment

Yes: the narrowest maintainable direction is to add Matrix-owned preflight transcription using the existing shared helper and channel patterns, not to change core media understanding or add a new config option.
