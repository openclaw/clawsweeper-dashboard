## Summary

Keep this open: the report matches a narrow current-main Ollama capability-discovery gap rather than an already-implemented cleanup candidate. OpenClaw has the right downstream vision mapping, but the current `/api/show` request shape appears to use the wrong Ollama API field, which can leave a real vision model resolved as text-only.

## What This Changes

The issue reports that `ollama/qwen3-vl:235b-cloud` is shown as the configured image model but resolves with text-only input, causing image inference and the image tool to reject it.

## Best Possible Solution

Align Ollama native API calls with the upstream `model` field, keep capability-based `vision` mapping, and add focused tests that fail when `/api/show` does not return capabilities for a vision model.

## Repair Work Prompt

Fix the current Ollama image-capability discovery regression reported in https://github.com/openclaw/openclaw/issues/80581. Current main posts `{ name: modelName }` to Ollama native endpoints such as `/api/show`, but the upstream Ollama API documents `model` for `POST /api/show` and `POST /api/pull`; with Ollama 0.21.0 this can prevent `/api/show` capabilities from being read, so `vision` cloud models resolve as text-only and the image tool rejects them. Update the Ollama provider to send the correct `model` field for show discovery, and consider the same contract correction for the nearby pull helper if it uses the same stale shape. Preserve the existing capability-based mapping from `vision` to `input: ["text", "image"]`; do not add a new config option or broad model-name heuristic. Update focused tests in the Ollama provider discovery/model tests to assert the `model` request body and cover a qwen3-vl/cloud vision capability response becoming image-capable. Add a changelog entry because this is user-visible. Validate with `pnpm test extensions/ollama/src/provider-models.test.ts extensions/ollama/provider-discovery.test.ts extensions/ollama/index.test.ts` and, if feasible, the live Ollama image smoke from the provider docs with redacted credentials.
