## Label Justifications

- `P2`: Premature compaction degrades an active session but has limited blast radius and does not make the core runtime unavailable.
- `impact:session-state`: The persisted selection and running session model metadata can diverge after the Telegram picker changes models.
- `impact:auth-provider`: The defect is triggered by provider/model routing state and applies the wrong selected-model metadata to session behavior.

## Summary

Current main still has the synchronization gap: Telegram persists a model override without the pending live-switch marker used by generic and Discord paths, so this remains an actionable bug.

## What This Changes

Ensure a live session refreshes its active model metadata after a model switch so compaction and context usage use the selected model’s context window.

## Best Possible Solution

Set `markLiveSwitchPending: true` in the Telegram picker’s existing model-override call and add focused coverage proving the next active attempt consumes the selected model; avoid registry lookup inside compaction.

## Reproduction Assessment

Yes at the source level: Telegram omits the pending live-switch marker used by generic and Discord switch paths, while compaction reads the running session’s model snapshot. A live 200k-to-1M Telegram run was not executed in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable fix is to reuse the existing pending live-switch contract in the Telegram picker. Re-resolving the registry during every compaction check would duplicate discovery in a hot path.
