## Summary

Keep open. Source inspection confirms the CLI/gateway state-dir mismatch the branch targets, but the patch changes the documented profile-isolation contract and still needs maintainer acceptance plus real behavior proof before merge.

## What This Changes

The PR makes CLI profile state resolution use `<OPENCLAW_HOME>/.openclaw` without appending a profile suffix and adds profile/home-dir tests.

## Best Possible Solution

Define one precedence rule, then align CLI path resolution, gateway/config defaults, CLI help, public docs, and regression tests so users do not unexpectedly share or split credentials, sessions, and config.

## Reproduction Assessment

Yes, by source inspection. Current main derives `<OPENCLAW_HOME>/.openclaw-work` in CLI profile setup while config/gateway defaults derive `<OPENCLAW_HOME>/.openclaw`; I did not run a live CLI/gateway process in this read-only review.

## Solution Assessment

Unclear as-is. The unsuffixed `OPENCLAW_HOME` fix matches current daemon defaults, but it conflicts with existing profile-isolation help/docs unless the precedence contract is explicitly accepted and documented.
