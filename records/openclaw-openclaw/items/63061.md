## Summary

Keep open. Source inspection confirms the CLI/gateway state-dir mismatch the PR targets, but the patch changes the profile isolation contract and still lacks contributor real behavior proof, so it needs maintainer contract approval plus docs/help alignment before merge.

## What This Changes

The PR makes CLI profile state resolution use `<OPENCLAW_HOME>/.openclaw` without appending `-<profile>` when `OPENCLAW_HOME` is explicit, adds a normalized helper, and updates profile tests.

## Best Possible Solution

Settle and document one precedence contract, then align CLI profile env, daemon state/config resolution, help text, docs, and focused tests to that contract.

## Reproduction Assessment

Yes, at source level. Current main derives `<OPENCLAW_HOME>/.openclaw-work` in CLI profile setup while config/gateway defaults derive `<OPENCLAW_HOME>/.openclaw`; no live CLI/gateway process was run in this read-only review.

## Solution Assessment

Unclear as-is. The unsuffixed `OPENCLAW_HOME` fix matches daemon defaults, but it conflicts with documented profile isolation and should land only after the precedence contract and docs are made explicit.
