## Summary

Keep open: this PR addresses a real CLI/gateway path mismatch, but the proposed precedence changes the documented profile isolation boundary under `OPENCLAW_HOME` and needs maintainer contract approval, docs/help alignment, and real behavior proof before merge.

## What This Changes

The branch makes profiled CLI commands use `<OPENCLAW_HOME>/.openclaw` without a profile suffix, adds a normalized explicit-home helper, and updates profile tests.

## Best Possible Solution

Choose one explicit precedence contract, then align code, tests, CLI help, and the multiple-gateway docs: either `OPENCLAW_HOME` owns the physical root and profiles share that state, or profiles keep isolated state/config and the daemon path is made to follow the same profile-derived environment.

## Reproduction Assessment

Yes by source inspection. Current main derives `<OPENCLAW_HOME>/.openclaw-work` in CLI profile setup while config/gateway defaults derive `<OPENCLAW_HOME>/.openclaw`; I did not run a live CLI/gateway process because this cleanup review is read-only.

## Solution Assessment

Unclear as-is. The unsuffixed `OPENCLAW_HOME` fix is plausible and matches daemon defaults, but it conflicts with documented profile isolation and should land only after the precedence contract and docs are made explicit.
