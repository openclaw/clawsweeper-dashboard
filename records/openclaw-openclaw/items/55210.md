## Summary

Keep open. Current main already handles the normal top-level nullable `finish_reason` and explicit stop chunk path, but the streaming error-content/terminal-stop split remains valid and this stale PR would regress the newer usage-aware finalizer if merged as-is.

## What This Changes

The PR changes the Gateway `/v1/chat/completions` SSE writer and tests so streamed chunks use choice-level `finish_reason`, final stop chunks use empty deltas, lifecycle end emits a stop chunk, and streaming errors emit content before a separate stop signal.

## Best Possible Solution

Adapt the useful error-splitting part onto current main while preserving `maybeFinalize()`/usage cleanup; if the overlapping #62070/#59913 repair lands first, close this PR as superseded by that merged fix.

## Reproduction Assessment

Yes. Current main has a source-level and test-backed reproduction: an `agentCommand` rejection in the streaming `/v1/chat/completions` path produces one stop choice whose delta content is `Error: internal error`, and the current test asserts that combined shape.

## Solution Assessment

No, not as this PR is written. The maintainable fix is narrower: keep current main's finalizer and change only the streaming error content/stop sequencing plus focused assertions and changelog coverage.
