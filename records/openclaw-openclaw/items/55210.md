## Summary

Keep open: the PR targets a real Gateway streaming compatibility bug, but the branch is stale/dirty and currently regresses the stream finalizer while carrying a failing error-path assertion.

## What This Changes

The PR changes the Gateway `/v1/chat/completions` stream writer and tests so normal and error streams emit choice-level `finish_reason` with a separate empty terminal stop chunk.

## Best Possible Solution

Adapt the error-content/terminal-stop split onto current main while preserving `maybeFinalize()` cleanup and usage ordering, then land one coordinated Gateway fix with focused tests and a changelog entry.

## Reproduction Assessment

Yes, source-level. Current main’s streaming catch path writes `Error: internal error` with `finishReason: "stop"`, and the current endpoint test asserts that combined shape.

## Solution Assessment

No, not as written. The useful error-splitting behavior should be applied to current main’s finalizer path instead of adding a direct stream-close branch that bypasses cleanup and optional usage output.
