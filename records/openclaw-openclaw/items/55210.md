## Summary

Keep open. The Gateway compatibility bug is still real on current main, but this PR should not merge as-is because it bypasses the newer usage-aware stream finalizer and has a failing test expectation around nullable non-final `finish_reason`.

## What This Changes

This PR changes the Gateway `/v1/chat/completions` streaming writer and tests so content chunks use top-level `finish_reason`, final stop chunks use empty deltas, lifecycle end emits a stop chunk, and streaming errors emit error text before a separate stop signal.

## Best Possible Solution

Apply a small current-main patch that splits streaming error content from the terminal stop chunk while preserving the existing usage-aware finalizer and disconnect cleanup, with focused Gateway tests and a changelog fix entry.

## Reproduction Assessment

Yes. Current main reproduces the remaining error-shape bug through the mocked streaming error path: an `agentCommand` rejection produces one stop choice whose delta content is `Error: internal error`, and the current test asserts that combined shape.

## Solution Assessment

No, not as this PR is written. The safer fix is to keep current main's `maybeFinalize()` path and change only the streaming error handling plus focused assertions needed to split error content from the final stop chunk.

## Review Findings
