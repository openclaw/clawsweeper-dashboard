## Summary

Keep open. The PR targets a real Gateway OpenAI-compat streaming bug, but the branch should not merge as-is because its lifecycle end path bypasses current main's usage-aware finalizer and its added error test contradicts the helper's nullable `finish_reason` behavior.

## What This Changes

This PR changes the Gateway `/v1/chat/completions` streaming helper and tests so streamed chunks use top-level `finish_reason`, terminal stop chunks use empty deltas, lifecycle end emits a stop chunk, and streaming errors emit error text before a separate stop signal.

## Best Possible Solution

Carry forward the error-path split as a small current-main patch: emit the error text as a nullable non-final content chunk, let the existing finalization path emit the empty-delta stop chunk and optional usage chunk, preserve disconnect cleanup, and update the focused streaming tests plus a user-facing changelog fix entry.

## Reproduction Assessment

Yes. Current main reproduces the remaining bug through the existing mocked streaming error path: an `agentCommand` rejection produces one choice containing both `Error: internal error` and `finish_reason: "stop"`, and the current test asserts that old combined shape.

## Solution Assessment

No, not as this PR is written. The narrow maintainable fix is to preserve current main's `maybeFinalize()` path and change only the streaming error handling and tests needed to split error content from the terminal stop chunk.
