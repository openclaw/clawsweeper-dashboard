## Summary

Keep open. The PR targets a real remaining Gateway compatibility bug, but the branch is stale against current main and should not merge as-is because it bypasses the current stream finalizer and adds a failing error-path assertion.

## What This Changes

This PR changes the Gateway `/v1/chat/completions` streaming writer and tests so content chunks keep top-level nullable `finish_reason`, normal end/error paths emit an explicit stop chunk, and streaming error text is split from the terminal stop chunk.

## Best Possible Solution

Apply a small current-main repair that splits streaming error content from the terminal stop chunk while preserving `maybeFinalize()` and the current usage/disconnect semantics.

## Reproduction Assessment

Yes. Current main reproduces the remaining error-shape bug through the mocked streaming error path: an `agentCommand` rejection produces one stop choice whose delta content is `Error: internal error`, and the current test asserts that combined shape.

## Solution Assessment

No. The PR direction is useful, but the safer solution is a narrow patch on current main that changes only streaming error shaping plus focused assertions while keeping the existing finalizer.
