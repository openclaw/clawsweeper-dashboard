## Summary

Keep open: the PR targets a real OpenAI-compatible streaming shape issue, but current main already has the normal choice-level nullable `finish_reason` and usage-gated stop finalizer, while this branch bypasses that finalizer and has a failing added assertion.

## What This Changes

The PR changes the Gateway `/v1/chat/completions` SSE writer and tests so streamed content and error paths emit terminal stop chunks separately from content chunks.

## Best Possible Solution

Update this branch or a replacement on current main so it preserves `maybeFinalize()`, splits streaming error content from the terminal stop chunk, keeps nullable non-final `finish_reason`, and adds a changelog entry.

## Reproduction Assessment

Yes. Current main has a source-level and test-backed reproduction: a streaming `agentCommand` rejection emits one stop choice whose delta content is `Error: internal error`, and the current test asserts that shape.

## Solution Assessment

No, not as written. The branch should keep current main's usage-aware finalizer and adjust only the error content/stop sequencing plus focused assertions and changelog coverage.
