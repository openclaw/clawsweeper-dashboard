## Summary

Keep open. The PR targets a real Gateway streaming compatibility problem, but the branch is conflicting, lacks required real behavior proof, and its direct-close stream handling no longer fits current main's usage-aware finalizer.

## What This Changes

The PR changes `/v1/chat/completions` streaming SSE chunk shaping and tests so content chunks and terminal `finish_reason` chunks are separated for normal and generic error streams.

## Best Possible Solution

Rebase or replace the streaming fix around current main's `maybeFinalize()` flow, preserving nullable intermediate `finish_reason`, terminal stop chunks, optional usage chunks, disconnect cleanup, and focused Gateway coverage.

## Reproduction Assessment

Yes, source-level. Current main still combines generic streaming error content with a stop-marked chunk, and current tests still expect that combined shape; I did not run a live Gateway request in this read-only review.

## Solution Assessment

No, not as written. The fix should carry terminal stop/error state through current main's finalizer instead of closing directly from the lifecycle handler, and the tests should match nullable non-final `finish_reason` behavior.
