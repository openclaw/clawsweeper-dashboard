## Summary

Keep open: the reported Gateway streaming bug is real, but this PR is not merge-ready because it bypasses current main’s stream finalizer, contains a contradictory test assertion, and lacks required real Gateway behavior proof.

## What This Changes

The PR changes `/v1/chat/completions` SSE chunk shaping and tests so content chunks and terminal `finish_reason` chunks are separated for normal and generic error streams.

## Best Possible Solution

Rebase or replace the streaming fix around current main’s `maybeFinalize()` flow, preserving nullable intermediate `finish_reason`, usage chunks, disconnect cleanup, and focused Gateway coverage.

## Reproduction Assessment

Yes, source-level. Current main’s generic streaming error path writes `Error: internal error` through a stop-marked content chunk, and the existing endpoint test still expects that combined shape; I did not run a live Gateway request in this read-only review.

## Solution Assessment

No, not as written. The fix should carry the error and terminal stop state through the current finalizer instead of closing directly from the lifecycle handler.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Route stream end through the current finalizer:** `src/gateway/openai-http.ts:595-611`
  - body: The new lifecycle `phase === "end"` branch closes the SSE response directly. Current main finalization removes the disconnect watcher and emits requested `stream_options.include_usage` usage before `[DONE]`, so bypassing it drops documented usage chunks and cleanup for successful streams.
  - confidence: 0.92
- **[P2] Align the error chunk assertion with nullable finish_reason:** `src/gateway/openai-http.test.ts:889`

  - confidence: 0.9
