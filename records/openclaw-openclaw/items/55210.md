## Summary

## What This Changes

The PR changes the Gateway `/v1/chat/completions` SSE writer and tests so normal and error streams use choice-level `finish_reason` and a separate terminal stop chunk.

## Best Possible Solution

Land a current-main repair that preserves `maybeFinalize()` and usage ordering, splits streaming error content from the terminal stop chunk, updates focused assertions, and adds the required changelog entry; if #62070 lands first, treat this PR as superseded.

## Reproduction Assessment

Yes, source-level. Current main's streaming catch path emits `Error: internal error` with `finishReason: "stop"`, and the current endpoint test asserts that combined shape.

## Solution Assessment

No, not as written. The useful error-splitting behavior should be adapted onto current main while preserving the existing finalizer, disconnect cleanup, and optional usage chunk sequencing.
