## Summary

Keep open: the Gateway SSE compatibility fix is still useful, but this PR is not merge-ready because it bypasses current main's usage-aware stream finalizer, has a test assertion that contradicts the branch's nullable `finish_reason` output, and lacks after-fix real behavior proof.

## What This Changes

The PR changes `/v1/chat/completions` streaming SSE chunks and tests so terminal `finish_reason` is emitted at choice level, adds a stop chunk on stream end, and splits streaming error content from the stop signal.

## Best Possible Solution

Implement the stop/error chunk split inside current main's `maybeFinalize()` flow, preserving nullable intermediate `finish_reason`, optional usage chunks, disconnect cleanup, and focused Gateway tests.

## Reproduction Assessment

Yes, source-level. Current main writes `Error: internal error` through `writeAssistantContentChunk(... finishReason: "stop")`, and the endpoint test asserts that same stop choice contains the error text; I did not run a live Gateway request.

## Solution Assessment

No, not as written. The useful wire-shape fix should be rebased or replaced around current main's usage-aware finalizer instead of closing the SSE response directly.
