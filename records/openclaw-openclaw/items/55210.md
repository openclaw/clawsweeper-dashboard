## Summary

Current main still has a source-level streaming error/finish chunk gap, but this external PR is not mergeable as written: it predates the current `maybeFinalize()` SSE finalizer, bypasses usage/cleanup on normal stream end, and lacks required real behavior proof.

## What This Changes

The PR changes `src/gateway/openai-http.ts` and `src/gateway/openai-http.test.ts` to split streamed content chunks from terminal `finish_reason` chunks for `/v1/chat/completions`.

## Best Possible Solution

Rework the streaming fix around current main's `maybeFinalize()` flow, preserving nullable intermediate `finish_reason`, terminal empty stop chunks, usage chunks, disconnect cleanup, and focused Gateway coverage.

## Reproduction Assessment

Yes, source-level. Current main still combines generic streaming error content with a stop-marked chunk, and the current test suite still expects that combined shape; I did not run a live Gateway request in this read-only review.

## Solution Assessment

No, not as written. The fix should carry stop/error state through current main's finalizer instead of closing directly from the lifecycle handler, and the tests should match nullable non-final `finish_reason` behavior.
