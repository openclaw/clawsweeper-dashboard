## Summary

Keep open: the PR targets a real Gateway compatibility gap, but the branch is stale against current main and still has merge-blocking correctness issues in the stream finalization path and error-chunk test assertion, plus it needs real behavior proof before merge.

## What This Changes

This PR changes the Gateway `/v1/chat/completions` streaming writer and endpoint tests to place `finish_reason` at choice level, emit a terminal stop chunk, and split error content from the stop signal.

## Best Possible Solution

Adapt the useful error-content/stop-signal split onto current main's `maybeFinalize()` flow, preserve nullable `finish_reason` and optional usage chunks, and update the focused Gateway tests around that current contract.

## Reproduction Assessment

Yes, source-level. Current main still combines `Error: internal error` with `finishReason: "stop"`, and the current endpoint test asserts that combined chunk shape; I did not run a live Gateway request.

## Solution Assessment

No, not as written. The intended wire-shape fix is useful, but the PR should be rebased or replaced around current main's usage-aware finalizer instead of closing the SSE response directly.
