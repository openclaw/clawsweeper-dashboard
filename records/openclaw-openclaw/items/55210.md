## Summary

Keep this PR open: the reported streaming chunk-shape problem is still source-reproducible on current main for the error path, but the branch is incorrect against current main's newer usage-aware finalization flow and still lacks real Gateway behavior proof before merge.

## What This Changes

The PR changes `/v1/chat/completions` streaming SSE chunk shaping and tests so content chunks and terminal `finish_reason` chunks are separated for normal and error streams.

## Best Possible Solution

Rework the current-main Gateway SSE finalization path so error content and terminal stop are separate chunks while preserving `maybeFinalize()`, usage chunks, disconnect cleanup, and focused Gateway tests.

## Reproduction Assessment

Yes, source-level. Current main writes `Error: internal error` through `writeAssistantContentChunk(... finishReason: "stop")`, and the endpoint test still expects that stop choice to contain the error text; I did not run a live Gateway request in this read-only review.

## Solution Assessment

No, not as written. The useful fix should be rebased or replaced around current main's `maybeFinalize()` flow, preserving nullable intermediate `finish_reason`, optional usage chunks, and disconnect cleanup.
