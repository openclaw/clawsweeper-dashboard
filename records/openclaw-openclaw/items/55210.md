## Summary

Keep open: this is a real Gateway compatibility bug, but the branch is not mergeable as-is because it bypasses the current stream finalizer and still lacks after-fix real behavior proof.

## What This Changes

The PR changes `/v1/chat/completions` streaming SSE chunk shaping and tests so content chunks and terminal `finish_reason` chunks are separated for normal and error streams.

## Best Possible Solution

Rework the streaming-error and terminal stop-chunk fix against current main’s `maybeFinalize()` flow, keeping nullable intermediate `finish_reason`, usage chunks, disconnect cleanup, and focused Gateway coverage.

## Reproduction Assessment

Yes, source-level. Current main’s rejected streaming command path writes `Error: internal error` through a stop-marked content chunk, and the existing endpoint test still expects that combined shape; I did not run a live Gateway request in this read-only review.

## Solution Assessment

No, not as written. The useful fix should be rebased or replaced around current main’s `maybeFinalize()` path so it preserves usage chunks, disconnect cleanup, and nullable intermediate `finish_reason`.
