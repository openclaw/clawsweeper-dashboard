Full review comments:

- **[P2] Accept homeDir in the runtime schema:** `extensions/codex/src/app-server/config.ts:339-340`
  - body: The resolver now reads `config.homeDir`, but `readCodexPluginConfig` still validates `appServer` with a strict zod object that does not include `homeDir`. Setting `plugins.entries.codex.config.appServer.homeDir` makes the strict parse fail and drops the appServer config before this resolver runs, so only the env fallback works.
  - confidence: 0.94

  - confidence: 0.82
- **[P3] Document the HOME override in the harness docs:** `extensions/codex/openclaw.plugin.json:329-333`
  - body: This adds a public `appServer.homeDir` field and env fallback, but the Codex harness docs still omit both and still say `HOME` is reserved for per-agent isolation. Users need the trusted-local boundary and env fallback documented on the public Codex harness page.
  - confidence: 0.78
