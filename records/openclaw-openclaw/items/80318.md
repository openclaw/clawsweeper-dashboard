## Summary

## What This Changes

The PR adds a Codex app-server `homeDir` config/env override so stdio launches can use a real OS `HOME` while keeping `CODEX_HOME` isolated.

## Best Possible Solution

Land the trusted-local override only after runtime schema, docs, tests, mkdir behavior, and redacted live app-server proof all align around the intended isolation boundary.

## Reproduction Assessment

Yes for the blocking patch bug: the PR head source shows `resolveCodexAppServerRuntimeOptions` reading `config.homeDir` while the strict app-server zod schema omits `homeDir`, so the advertised config key is rejected before resolution. I did not run tests because this review is read-only.

## Solution Assessment

No as written. The explicit override is a plausible narrow direction, but it needs the runtime schema fix, explicit-HOME mkdir preservation, public docs, focused tests, and live proof before it is the maintainable solution.
