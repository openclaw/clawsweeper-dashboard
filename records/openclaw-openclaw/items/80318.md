## Summary

Keep this PR open, but do not merge it as written. The direction matches the linked request, but the patch currently rejects the new config key through the strict runtime schema, weakens explicit HOME handling, lacks public docs, and only provides test output rather than live app-server/local-auth proof.

## What This Changes

The PR adds a Codex app-server `homeDir` config/env override, passes it as `HOME` for stdio launches while keeping `CODEX_HOME` isolated, updates the plugin manifest, and adds a focused auth-bridge test.

## Best Possible Solution

Land a documented, explicit trusted-local HOME override only after schema, manifest, docs, tests, and redacted live app-server proof all agree on the intended isolation boundary.

## Reproduction Assessment

Yes for the current behavior and the PR blocker: source inspection shows current main isolates HOME under `codex-home/home`, and the PR head reads `homeDir` after a strict schema that still omits it. I did not run a live `gh auth status` app-server reproduction in this read-only review.

## Solution Assessment

No as written. The explicit override is a plausible narrow shape, but the implementation must accept the key in the runtime schema, preserve explicit HOME override behavior, update public docs, and include real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Accept homeDir in the runtime config schema:** `extensions/codex/src/app-server/config.ts:335-337`
  - body: Adding `homeDir` to the type, manifest, key list, and resolver is not enough because `readCodexPluginConfig` validates `appServer` with a strict zod object that still omits this key. A config using the advertised `plugins.entries.codex.config.appServer.homeDir` field will make the appServer parse fail and fall back to `{}`, so only the env fallback can work.
  - confidence: 0.94
- **[P3] Only skip mkdir when homeDir supplies HOME:** `extensions/codex/src/app-server/auth-bridge.ts:207-209`
  - body: When both `startOptions.env.HOME` and `homeDir` are present, explicit `env.HOME` still wins, but this condition skips creating that explicit HOME path solely because `homeDir` exists. Track whether `nativeHome` actually came from `homeDir` before skipping `mkdir` so the existing explicit-HOME behavior keeps working.
  - confidence: 0.82
- **[P3] Document the app-server HOME override:** `extensions/codex/openclaw.plugin.json:329-333`
  - body: This adds a public `appServer.homeDir` field and env fallback, but the Codex harness docs still omit both and still say `HOME` is reserved for per-agent isolation. Update the supported `appServer` fields and environment override list so users see the trusted-local boundary and the new knob.
  - confidence: 0.78
