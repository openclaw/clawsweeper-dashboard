## Summary

Keep open: this is a focused PR for the linked Codex local-auth issue, but the patch currently rejects the advertised config key through the strict runtime schema and the supplied proof is only targeted test output.

## What This Changes

The PR adds a Codex plugin `appServer.homeDir` and `OPENCLAW_CODEX_APP_SERVER_HOME_DIR` override so stdio app-server launches can use a configured `HOME` while keeping `CODEX_HOME` isolated.

## Best Possible Solution

Land the trusted-local override only after schema, docs, tests, and redacted real app-server proof all align around the intended isolation boundary.

## Reproduction Assessment

Yes for the blocking patch bug: the PR head source shows a strict `appServer` schema that omits `homeDir` while the resolver reads it, so `plugins.entries.codex.config.appServer.homeDir` is rejected before resolution. I did not run tests in this read-only review.

## Solution Assessment

No as written. The explicit override is a plausible narrow direction, but the implementation must accept the new key in the runtime schema, preserve existing HOME override behavior, update public docs, and include real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Accept homeDir in the runtime config schema:** `extensions/codex/src/app-server/config.ts:338-340`
  - body: Adding `homeDir` to the type, manifest, and resolver does not make `plugins.entries.codex.config.appServer.homeDir` work because `readCodexPluginConfig` validates `appServer` with a strict zod object that still lacks this key. A config using the advertised key makes the whole appServer parse fail and falls back to `{}`, so only the env fallback can work. Please add `homeDir` to `codexPluginConfigSchema` and cover config/env resolution in `config.test.ts`.
  - confidence: 0.94
- **[P3] Only skip mkdir when homeDir supplies HOME:** `extensions/codex/src/app-server/auth-bridge.ts:207-209`
  - body: When both `startOptions.env.HOME` and `homeDir` are present, explicit `env.HOME` wins, but this condition skips creating that explicit HOME path solely because `homeDir` exists. That regresses the existing explicit-HOME behavior; track whether `nativeHome` actually came from `homeDir` before skipping `mkdir`.
  - confidence: 0.82
- **[P3] Document the new app-server HOME override:** `extensions/codex/openclaw.plugin.json:329-333`
  - body: This adds a public `appServer.homeDir` field and env fallback, but the Codex harness docs still omit both and still say `HOME` is reserved for per-agent isolation. Please update the supported `appServer` fields and environment override list so users see the trusted-local boundary and the new knob in the public docs.
  - confidence: 0.78
