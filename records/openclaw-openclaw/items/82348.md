## Summary

The report points to an external OcuClaw npm relay/client that is not present in this repository and appears to be using an older gateway protocol/auth contract that current OpenClaw intentionally rejects.

## What This Changes

The issue reports that the external OcuClaw relay connects downstream glasses but loses its upstream gateway connection after updating to OpenClaw 2026.5.12.

## Best Possible Solution

Update or file against the OcuClaw relay package so it negotiates gateway protocol v4, waits for `connect.challenge`, signs the nonce, and sends current device auth; OpenClaw core should only change if a minimal conforming v4 client is rejected.

## Reproduction Assessment

No, we do not have a high-confidence end-to-end hardware reproduction. Source inspection does confirm that a non-probe client whose protocol range stays at v3 would be rejected by current main and v2026.5.12.

## Solution Assessment

Yes. Closing this repo item is the maintainable path because the implicated implementation is an external relay using an older gateway contract; a new OpenClaw issue should show a valid v4, challenge-auth client failing against current main.
