## What This Changes

The issue requests investigation of an external OcuClaw relay losing its upstream gateway WebSocket connection after updating to OpenClaw 2026.5.12.

## Best Possible Solution

Update or file against the OcuClaw relay/plugin so it negotiates OpenClaw gateway protocol v4 and current challenge-bound device auth; OpenClaw core should only change if a minimal v4 client is rejected.

## Reproduction Assessment

No, we do not have a high-confidence end-to-end hardware reproduction. Source inspection does confirm that the reported protocol-v3 upstream client would be rejected by current main and v2026.5.12 because non-probe clients must negotiate protocol v4.

## Solution Assessment

Yes, closing this repo item is the maintainable path because the implicated code is an external relay/plugin using an older gateway contract. A new OpenClaw issue would be actionable if it shows a valid v4, challenge-auth client failing against current main.
