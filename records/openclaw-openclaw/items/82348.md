## Summary

Close as outside this repository: the report points to an external npm OcuClaw relay using an older gateway protocol/auth contract, while OpenClaw current main and the reported release already document and enforce protocol v4 plus challenge-bound device auth.

## What This Changes

The issue asks OpenClaw core to investigate an external OcuClaw relay/plugin failure after v2026.5.12 where glasses connect to the relay but downstream sockets drop after about 300 ms.

## Best Possible Solution

Update or file against the external OcuClaw relay/plugin so it negotiates gateway protocol v4 and current challenge-bound device auth; OpenClaw core should change only if a minimal valid v4 client is rejected.

## Reproduction Assessment

No, we do not have a high-confidence end-to-end hardware reproduction with the glasses and relay. Source inspection does show that the reported protocol-v3 upstream client would be rejected by current main and v2026.5.12 because non-probe clients must negotiate protocol v4 and send challenge-bound device auth.

## Solution Assessment

Yes, closing this repository item is the maintainable path because the implicated relay/client code is external and using an older gateway contract. A new OpenClaw issue would be actionable if it demonstrates a valid v4, challenge-auth client failing against current main or v2026.5.12.
