## Summary

Keep open. Current main still warns for workspace skill paths whose realpath escapes the configured root, while the exact Markdown-link mechanism in the report is not confirmed by the current loader path.

## What This Changes

The issue asks OpenClaw to avoid noisy escaped skill-path startup warnings for intentional sibling skill references in skill documentation.

## Best Possible Solution

Keep path containment, but define a narrow, tested policy for benign sibling or duplicate skill references that avoids startup noise without weakening audit and doctor diagnostics.

## Reproduction Assessment

Unclear for the reported Markdown-link path: current main scans directories and `SKILL.md` files and parses frontmatter, not Markdown body links. Yes for the broader warning class, because current tests construct workspace symlink escapes and assert warn-level output.

## Solution Assessment

Unclear. The closed warn-to-debug PR is likely too broad; a safer fix would target a proven benign reference pattern while preserving warnings for real root escapes.
