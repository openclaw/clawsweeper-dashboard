## Summary

The exact Markdown-link parsing mechanism from the report is not present in the inspected loader, but the noisy intentional sibling symlink case is addressed on main.

## What This Changes

Allow intentional sibling skill layouts to avoid noisy escaped-path warnings without weakening skill-root containment.

## Best Possible Solution

Keep the explicit `skills.load.allowSymlinkTargets` contract and default warning behavior so intentional sibling layouts are quiet only after the target root is trusted.

## Reproduction Assessment

## Solution Assessment

Yes. The mainline solution is narrower than the closed warn-to-debug PR because it lets trusted sibling skill targets load quietly while keeping warnings for real untrusted containment escapes.
