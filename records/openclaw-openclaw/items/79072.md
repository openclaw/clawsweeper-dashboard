## Summary

Keep open. Current main still has the source-visible stale skills snapshot path that can evaluate a disabled `skills.entries.gh-issues.apiKey` SecretRef, and an open linked PR already owns the narrow fix with closing syntax.

## What This Changes

The issue reports that WhatsApp embedded lane sessions crash when `skills.entries.gh-issues.apiKey` is an unresolved env SecretRef, even after the skill entry is disabled.

## Best Possible Solution

Refresh/review and land https://github.com/openclaw/openclaw/pull/79173, or replace it with the same focused guard and regression test if that branch cannot be merged.

## Reproduction Assessment

Yes, source-level: a persisted snapshot containing `gh-issues`, current config with `skills.entries.gh-issues.enabled:false`, a raw SecretRef apiKey, and no host `GH_TOKEN` reaches `applySkillEnvOverridesFromSnapshot` without a disabled-entry guard.

## Solution Assessment

Yes, the linked PR’s direction is the narrowest maintainable fix: honor the existing enabled-state contract at the skill env injection boundary and cover the stale snapshot case with a regression test.
