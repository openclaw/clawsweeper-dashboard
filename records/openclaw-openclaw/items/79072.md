## Summary

Keep open. Current main still has the source-visible stale snapshot path that can evaluate a disabled skill entry's raw `apiKey` SecretRef, and the linked fixing PR is open rather than merged.

## What This Changes

The issue reports that `skills.entries.gh-issues.apiKey` remains unresolved during WhatsApp embedded sessions, even when the skill entry is disabled.

## Best Possible Solution

Keep disabled skill entries inert in both fresh skill loading and persisted snapshot env injection, with regression coverage for a stale `gh-issues` snapshot carrying a raw SecretRef.

## Reproduction Assessment

Yes, source-level. A persisted snapshot containing `gh-issues`, current config with `skills.entries.gh-issues.enabled:false` and a raw `apiKey` SecretRef, and no host `GH_TOKEN` reaches `applySkillEnvOverridesFromSnapshot` without an enabled guard.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable fix: add the disabled-entry guard at the env-injection boundary and cover the stale snapshot case, rather than adding WhatsApp-specific behavior.
