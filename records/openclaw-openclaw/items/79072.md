## Summary

Keep open. Current main still has a source-visible disabled-snapshot SecretRef failure path, and the linked fix PR at https://github.com/openclaw/openclaw/pull/79173 uses closing syntax but remains open and unmerged.

## What This Changes

The issue reports that WhatsApp embedded agent sessions crash when a gh-issues skill entry keeps an unresolved SecretRef apiKey, including after the entry is set to enabled:false.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/79173; the end state should keep disabled skill entries inert in both fresh skill loading and persisted snapshot env injection.

## Reproduction Assessment

Yes. Source inspection shows a persisted snapshot containing gh-issues plus current config with enabled:false and a raw apiKey SecretRef reaches applySkillEnvOverridesFromSnapshot and strict apiKey normalization without a disabled-entry guard.

## Solution Assessment

Yes. The narrow maintainable solution is to honor enabled:false at the skill env-injection boundary for live entries and persisted snapshots, rather than adding WhatsApp-specific handling.
