Keep open: current main has zh-CN Control UI localization, but the named audit/event/tool labels still bypass the i18n layer and render English or raw event names.

## What This Changes

Request Chinese localization for Gateway Control UI text, especially Audit Log action names, navigation labels, and buttons.

## Best Possible Solution

Add a focused Control UI i18n display-label layer for known tool/action/event names while preserving raw payloads and shared tool-display metadata contracts.

## Solution Assessment

Yes. The narrowest maintainable fix is to route Control UI display-only labels through locale keys instead of rewriting shared JSON or translating arbitrary server payloads.
