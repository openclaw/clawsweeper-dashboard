## Summary

Keep open: current main supports zh-CN in the Control UI, but the specific audit/event/tool labels named here still bypass the i18n layer and render shared English labels or raw event identifiers. The related https://github.com/openclaw/openclaw/issues/52254 covers broader lazy-tab Chinese copy, but it does not fully replace this issue's tool/action/event-label scope.

## What This Changes

Request Chinese localization for the Gateway/Control UI, especially audit log action names, navigation, buttons, and other visible UI text.

## Best Possible Solution

Add a focused Control UI display-label layer for known tool/action/event names through the existing i18n source and generator, while leaving protocol identifiers and shared tool metadata stable.

## Reproduction Assessment

Yes. Source inspection against current origin/main shows zh-CN locale loading works, while tool cards still read English shared metadata and Overview/Debug event logs render raw event identifiers.

## Solution Assessment

Yes. The narrow maintainable direction is locale-backed render-time labels for known display names, not rewriting arbitrary server payloads or changing shared JSON contracts.
