## Summary

Keep open: current main and the latest release support zh-CN in the Control UI, but the specific tool/action/event labels called out here still render English shared metadata or raw identifiers. Related open issues and PRs cover overlapping chat, tab, and Settings localization work, but none fully owns this audit/event/tool-label scope yet.

## What This Changes

Request Chinese localization for Gateway/Control UI labels, especially Audit Log action names plus remaining navigation and button copy.

## Best Possible Solution

Add locale-backed display-only labels for finite tool, action, and event names through the existing Control UI i18n pipeline, while preserving raw protocol identifiers and debug payloads.

## Reproduction Assessment

Yes. Source inspection on current main shows zh-CN loading works, but tool cards still render English shared metadata/hardcoded chrome and event logs render raw event identifiers.

## Solution Assessment

Yes, if the fix stays display-only. The maintainable direction is finite render-time i18n labels, not changing gateway payloads, tool names, shared protocol identifiers, or arbitrary debug payloads.
