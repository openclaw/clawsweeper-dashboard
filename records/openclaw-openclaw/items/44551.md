## Summary

Keep open: current main has broad zh-CN Control UI localization and a browser/setting-based locale path, but the concrete tool/action/event-label examples still come from English shared metadata or raw event names.

## What This Changes

Request Chinese-language Gateway UI support, especially for Audit/Event Log and tool/action labels such as exec, read, cron run, subagents list, and browser.

## Best Possible Solution

Add a focused Control UI i18n layer for user-visible tool/action/event labels while keeping raw payloads and shared tool-display metadata stable.

## Reproduction Assessment

Yes. Source inspection on current main shows zh-CN locale loading works, but tool cards read English shared titles/action labels and event logs render raw event names.

## Solution Assessment

Yes. The narrowest maintainable fix is to route Control UI display labels through locale keys, not to rewrite shared JSON into Chinese or translate arbitrary server payloads.
