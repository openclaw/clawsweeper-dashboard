## Summary

Keep open: current main has zh-CN Control UI support, but the specific tool/action/event labels called out here still bypass display localization. Related localization issues and PRs cover overlapping chat metadata or broader tab copy, but none fully owns this issue's audit/event/tool label scope yet.

## What This Changes

Gateway/Control UI should show Chinese display text for audit logs, tool/action labels such as exec/read/browser, navigation, buttons, and other visible UI elements.

## Best Possible Solution

Add locale-backed display-only labels for known tool, action, and event names through the existing Control UI i18n source and generator, while preserving raw protocol identifiers and shared metadata contracts.
