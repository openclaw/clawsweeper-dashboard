## What This Changes

The issue requests Chinese display support across the Gateway UI, especially audit log event names and tool/action labels such as exec, read, cron run, subagents list, and browser.

## Best Possible Solution

Add display-only zh-CN aliases for a finite set of event, tool, and action labels through the existing i18n/generated-locale pipeline while preserving raw identifiers in protocol payloads and debug details.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.5.19 shows zh-CN support exists, but overview/debug event rows render raw event identifiers and tool cards resolve English catalog labels without locale lookup.

## Solution Assessment

Yes for keeping this open. The maintainable path is a display-alias layer through the existing i18n pipeline, not changing protocol payloads, command names, or arbitrary debug data.
