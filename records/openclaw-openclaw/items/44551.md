## Summary

Keep open: current main and the latest release still render the specific Gateway/Control UI audit, event, and tool/action labels as raw identifiers or English shared metadata. Related localization issues and open PRs cover adjacent zh-CN gaps, but none fully supersedes this audit/tool-label scope.

## What This Changes

Add Simplified Chinese display support for the Gateway/Control UI, especially Audit Log and tool/action labels such as exec, read, cron run, subagents list, and browser.

## Best Possible Solution

Add display-only locale aliases for finite tool, action, and event names through the existing Control UI i18n pipeline, while preserving raw protocol identifiers in payloads, debug data, and commands.

## Reproduction Assessment

Yes, as source-level reproduction: set the Control UI locale to zh-CN and inspect Overview/Debug event logs or chat tool cards. Current code renders entry.event/evt.event directly and uses English shared tool-display metadata, although I did not run a live browser session in this read-only review.

## Solution Assessment

Yes, if the fix stays display-only and finite. The safer maintainable direction is locale-backed aliases at render time, not changing gateway payloads, tool names, commands, or arbitrary debug payloads.
