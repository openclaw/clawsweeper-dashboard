## What This Changes

Request Chinese display support for the Gateway/Control UI, especially Audit Log operation names, tool labels, navigation, and buttons.

## Best Possible Solution

Add display-only zh-CN aliases for finite event, tool, and action labels through the existing Control UI i18n pipeline while preserving raw protocol/debug identifiers and coordinating with the related localization issue.

## Solution Assessment

Yes, keeping this open for maintainer/product review is the best current outcome. The maintainable fix is a finite display-alias layer through i18n, not changing gateway payloads, command names, or arbitrary debug data.
