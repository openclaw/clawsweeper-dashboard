## What This Changes

The issue requests Chinese display support for Gateway/Control UI audit logs, navigation, buttons, and visible labels such as exec, read, cron run, subagents list, and browser.

## Best Possible Solution

Add display-only zh-CN aliases for a finite set of event, tool, and action labels through the existing Control UI i18n/generated-locale pipeline, while preserving raw identifiers in protocol payloads and debug details.

## Reproduction Assessment

Yes, by source inspection rather than a live browser run. With zh-CN selected, the Overview/Debug event logs still render entry.event or evt.event directly, and tool cards still resolve English shared metadata without locale lookup.

## Solution Assessment

Yes for keeping this open and routing it through maintainer review. The narrow maintainable fix is a display-alias layer in the existing i18n pipeline, not translating protocol payloads, command names, or arbitrary debug JSON in place.
