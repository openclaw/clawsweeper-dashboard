## Summary

Keep open: current main and the latest release still render raw Gateway event identifiers and English shared tool/action display labels under zh-CN, so the requested Chinese display support is not implemented or superseded.

## What This Changes

The issue requests Chinese display support in the Gateway/Control UI, especially Audit Log action names such as exec, read, cron run, subagents list, browser, and surrounding UI text.

## Best Possible Solution

Add display-only zh-CN aliases for finite event, tool, and action labels through the existing Control UI i18n pipeline while preserving raw protocol/debug identifiers and coordinating with the broader localization issue.

## Reproduction Assessment

Yes, source-level. Current main renders Overview/Debug event names from entry.event/evt.event directly and resolves shared tool labels from English catalog titles such as Read, Browser, Exec, and Subagents/list.

## Solution Assessment

Yes, keeping this open for maintainer/product review is the best current outcome. The maintainable fix is a finite display-alias layer through i18n, not changing gateway payloads, command names, or arbitrary debug data.
