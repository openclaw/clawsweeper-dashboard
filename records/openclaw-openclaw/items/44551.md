## Summary

Keep open: current main and the latest release still support zh-CN only partially, while the requested Gateway/Control UI audit, event, and tool/action labels still render as raw identifiers or English shared metadata. The related localization issue at https://github.com/openclaw/openclaw/issues/52254 covers adjacent untranslated tabs, but it does not fully supersede this audit/tool-label scope.

## What This Changes

The issue requests zh-CN display support for Gateway/Control UI audit logs, tool/action names, navigation, and buttons.

## Best Possible Solution

Add display-only zh-CN aliases for finite event, tool, and action labels through the existing Control UI i18n pipeline while preserving raw protocol/debug identifiers in payloads and commands.

## Reproduction Assessment

Yes, as source-level reproduction: set the Control UI locale to zh-CN and inspect Overview/Debug event logs or chat tool cards. Current main renders entry.event/evt.event directly and uses English shared tool-display metadata.

## Solution Assessment

Yes, keeping this open is the right outcome. The maintainable fix should be a finite display-alias layer at render time through i18n, not changing gateway payloads, command names, or arbitrary debug data.
