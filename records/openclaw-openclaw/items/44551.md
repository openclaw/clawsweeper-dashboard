## Summary

Keep open: current main and v2026.5.12 still expose raw Gateway event identifiers and English shared tool/action labels in zh-CN. The related open localization issue at https://github.com/openclaw/openclaw/issues/52254 overlaps nearby tab copy, but it does not fully supersede this audit/tool-label scope.

## What This Changes

Request Chinese display support in the Gateway/Control UI, especially Audit Log/tool/action names such as exec, read, cron run, subagents list, browser, plus other UI elements.

## Best Possible Solution

Add display-only zh-CN aliases for finite event, tool, and action labels through the existing Control UI i18n pipeline, while preserving raw identifiers in protocol/debug payloads and coordinating with https://github.com/openclaw/openclaw/issues/52254.

## Reproduction Assessment

Yes. Source inspection on current main shows zh-CN loading exists, but Overview/Debug event logs render entry.event/evt.event directly and the shared tool-display catalog still supplies English labels such as Read, Browser, and Exec.

## Solution Assessment

Yes, keeping this open is the best current outcome. The narrow maintainable solution is a finite display-alias layer at render time through i18n, not changing gateway payloads, command names, or arbitrary debug data.
