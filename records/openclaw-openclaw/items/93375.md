## What This Changes

The issue reports that Telegram polling can enter a silent stopped/crash-loop state after a transient Bot API timeout, with health-monitor restarts failing to recover ingress until the whole gateway process restarts.

## Best Possible Solution

Keep this issue open while maintainers review the linked Telegram/gateway recovery candidates, then close it only after a merged fix proves Telegram ingress resumes after transient Bot API failure or escalates with a clear diagnostic when account-level recovery cannot clear process-local state.

## Reproduction Assessment

The logs map to current Telegram polling lease, deleteWebhook recovery, isolated ingress, and health-monitor restart paths, and multiple commenters reproduced the same stuck-ingress signature on recent releases.

## Solution Assessment

Unclear: the open linked PR is a plausible narrow stale-lease repair, but the issue discussion points to broader gateway-state and diagnostics failures as well. The best fix needs maintainer review of whether to repair the lease only, gate health-monitor reset behavior, surface worker exit details, finish the gateway stale-task fix, or combine those in one bounded recovery change.
