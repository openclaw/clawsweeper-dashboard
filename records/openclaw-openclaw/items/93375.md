## What This Changes

The issue reports Telegram polling entering a silent no-recovery state after a transient Bot API network timeout, with health-monitor restarts failing until the whole gateway process restarts.

## Best Possible Solution

Finish a gateway-owned recovery path that can retire or replace never-settling timed-out channel tasks without duplicate Telegram pollers, then close this issue after the linked fix lands with live or equivalent recovery proof.
