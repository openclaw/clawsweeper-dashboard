## Summary

Keep open: the use case is legitimate and the author linked a companion AICM plugin, but the revised PR still has blocking public API and security issues around raw transcript hooks.

## What This Changes

This PR adds `before_assemble` and `after_assemble` typed plugin hooks, HookRunner methods, embedded-runner call sites, a changelog entry, and a whitespace-only lockfile change.

## Best Possible Solution

Define one guarded assembly extension point, then implement it with conversation and prompt policy classification, honest final-payload semantics, raw-run boundaries, SDK/docs updates, and contract tests, or fold the use case into the context-engine or #61485 path.

## Reproduction Assessment

Yes. Static source review is enough: the PR head exposes `before_assemble.messages` while leaving `before_assemble` out of the current conversation-access policy set, and the runner call order shows `after_assemble` fires before the runtime system prompt is finalized.

## Solution Assessment

No. A guarded assembly extension point may be useful, but this implementation is not the best solution yet because it bypasses one existing safety gate and does not accurately report the final model input.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Gate before_assemble behind conversation access:** `src/plugins/hook-types.ts:958`
  - body: `before_assemble` exposes the full message list and lets the result replace active session state, but the PR only adds `after_assemble` to `CONVERSATION_HOOK_NAMES`. Since `registerTypedHook` enforces `allowConversationAccess` solely from that set, a non-bundled plugin can read and mutate raw conversation history through this hook without the documented opt-in.
  - confidence: 0.94
- **[P2] Report the final system prompt to after_assemble:** `src/agents/pi-embedded-runner/run/attempt.ts:2883-2884`
  - body: The hook is described as observing the final context before the LLM call, but it fires with `systemPromptText` before the non-runtime-only path appends `runtimeContext` to the system prompt. Observability plugins will miss prompt text that is actually visible to the model.
  - confidence: 0.87
- **[P2] Keep assembly hooks out of raw model runs:** `src/agents/pi-embedded-runner/run/attempt.ts:2598-2603`
  - body: Raw model mode resets history and skips the context engine, prompt-build hooks, and `llm_input`, but this new hook runs unconditionally. A registered `before_assemble` handler can therefore inject messages back into a raw model probe that is supposed to bypass normal context assembly.
  - confidence: 0.82
