## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Gate before_assemble as prompt mutation:** `src/plugins/hook-types.ts:177`
  - body: `before_assemble` can replace the model-visible message list, but this registers it only as a conversation hook. With `allowConversationAccess=true` and `allowPromptInjection=false`, a non-bundled plugin can still rewrite or inject model context, bypassing the existing prompt-mutation policy.
  - confidence: 0.93
- **[P2] Pass cloned messages to assembly hooks:** `src/agents/pi-embedded-runner/run/attempt.ts:2605`
  - body: The hook payload passes `activeSession.messages` directly to plugin code. A handler that mutates the array and then returns void or throws still changes live session state outside the returned-result contract; pass cloned normalized messages and apply only validated returned values.
  - confidence: 0.9
- **[P2] Report the final model input to after_assemble:** `src/agents/pi-embedded-runner/run/attempt.ts:2886-2887`
  - body: `after_assemble` reports `systemPromptText` and `promptSubmission.prompt`, but current main composes runtime context and current-turn context into the values observed by the model. Observability plugins would miss context that is actually sent.
  - confidence: 0.88
