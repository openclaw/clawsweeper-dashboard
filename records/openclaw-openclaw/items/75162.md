## Summary

Keep open. Current main does not implement these assembly hooks, and the PR is an active feature/API proposal, but the latest head still changes a raw conversation mutation boundary with source-reproducible security/correctness issues and no after-fix real behavior proof.

## What This Changes

The PR adds `before_assemble` and `after_assemble` hook types, hook-runner methods, embedded-runner call sites, a changelog entry, and a lockfile whitespace change.

## Best Possible Solution

Define one guarded, documented context-assembly mutation/observation contract with explicit prompt policy, cloned and validated message payloads, final model-input semantics, SDK docs/API coverage, regression tests, and real gateway/plugin proof.

## Reproduction Assessment

Yes. Source inspection of PR head reproduces the review findings: `before_assemble` can rewrite model-visible messages without the prompt-injection guard, both assembly events receive live session arrays, and `after_assemble` observes stale prompt fields.

## Solution Assessment

No. The feature direction may be useful, but this patch is not the best mergeable shape until the trust policy, payload cloning, final prompt semantics, docs/API baselines, tests, and real behavior proof are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Gate before_assemble with prompt-injection policy:** `src/plugins/hook-types.ts:177`
  - body: `before_assemble` can replace the model-visible message list, but this change only classifies it as a conversation hook. A non-bundled plugin with `allowConversationAccess=true` and `allowPromptInjection=false` can still rewrite or inject context, bypassing the existing prompt-mutation policy.
  - confidence: 0.92
- **[P2] Pass cloned messages into assembly hooks:** `src/agents/pi-embedded-runner/run/attempt.ts:2605`
  - body: The hook payload hands `activeSession.messages` to plugin code directly. A handler that mutates the array and then returns void or throws still changes session state outside the result contract; use cloned payloads and apply only validated returned messages.
  - confidence: 0.87
- **[P2] Report the final prompt payload to after_assemble:** `src/agents/pi-embedded-runner/run/attempt.ts:2886-2887`
  - body: The event claims to observe the final model input, but it sends `systemPromptText` and `promptSubmission.prompt` before runtime context is folded into the prompt state below. Observability plugins can miss context that is actually visible to the model.
  - confidence: 0.9
