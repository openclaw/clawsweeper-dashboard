## Summary

Keep this PR open, but do not merge it as-is. Current main does not already implement these hooks, and the companion AICM plugin shows real downstream demand, but the patch introduces a public raw transcript mutation/observation API without the existing hook policy gates and with mismatched sequencing/final-context semantics.

## What This Changes

The PR adds `before_assemble` and `after_assemble` plugin hook types, HookRunner methods, embedded-runner call sites around context assembly, and a stray `extensions/aicm` lockfile importer.

## Best Possible Solution

Define one supported assembly or prompt-interception seam with explicit hook policy classification, honest payload semantics, SDK/docs/contract tests, and a changelog entry, then align or consolidate it with #61485 if that path is broader and safer.

## Reproduction Assessment

Yes. Static review is sufficient: current main lacks these hook names, and the latest PR diff adds raw-message hooks without the existing policy classifications, without event evolution between sequential handlers, and without a complete final model-input payload.

## Solution Assessment

No. A guarded core extension point may be appropriate, but this implementation is not the best fix because it bypasses current safety gates and does not accurately implement the stated sequential or final-context behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Gate assembly hooks through hook policies:** `src/plugins/hook-types.ts:106-107`
  - body: The new hooks can read and replace full conversation messages, but the diff only adds their names to the generic hook list. Because they are absent from the prompt-injection and conversation-access hook classifications, a non-bundled plugin can register raw transcript hooks without the existing `allowPromptInjection` and `allowConversationAccess` gates.
  - confidence: 0.93
- **[P2] Pass accumulated messages to each assembly handler:** `src/plugins/hooks.ts:1318-1325`
  - body: `runBeforeAssemble` promises chained transformations, but `runModifyingHook` invokes every handler with the original event. A later plugin therefore sees the unmodified message list and can undo an earlier plugin's deletion or reorder when it returns its own `messages`.
  - confidence: 0.9
- **[P2] Expose the actual model input to after_assemble:** `src/agents/pi-embedded-runner/run/attempt.ts:2878`
  - body: The `after_assemble` event is described as the final context before the LLM call, but the payload only includes `activeSession.messages`. The actual request also includes system/runtime prompt text and image context, so observers will miss part of the model submission.
  - confidence: 0.84
- **[P3] Remove the stale AICM lockfile importer:** `pnpm-lock.yaml:234`
  - body: The PR no longer contains an `extensions/aicm` package, but the lockfile still adds `extensions/aicm: {}` as a workspace importer. That leaves package metadata inconsistent with the workspace package set.
  - confidence: 0.8
