## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Gate before_assemble as prompt mutation:** `src/plugins/hook-types.ts:177`
  - body: `before_assemble` can replace model-visible history, but PR head only lists it under conversation hooks. A non-bundled plugin with `allowConversationAccess=true` and `allowPromptInjection=false` could still rewrite prompt history, bypassing the documented prompt-mutation opt-out.
  - confidence: 0.93
- **[P2] Pass isolated messages into assembly hooks:** `src/agents/pi-embedded-runner/run/attempt.ts:2605`
  - body: The hook payload hands `activeSession.messages` directly to plugin code. A handler can mutate the array or nested content and then return `void` or throw, changing session state outside the returned-result contract; clone and normalize payloads before `before_assemble` and `after_assemble`.
  - confidence: 0.91
- **[P2] Validate replacements before storing messages:** `src/agents/pi-embedded-runner/run/attempt.ts:2623`
  - body: Returned `messages` are cast to `AgentMessage[]` and installed without validation or replay repair. A malformed plugin result can corrupt `activeSession` and fail later outside hook error handling; validate and normalize replacements before assignment.
  - confidence: 0.86
- **[P2] Report the final runtime prompt to after_assemble:** `src/agents/pi-embedded-runner/run/attempt.ts:2886-2887`
  - body: The event claims to observe the final model input, but it reports `systemPromptText` and `promptSubmission.prompt` rather than the runtime-context-aware system prompt and current-turn prompt that current main submits to the model. Observability plugins will miss context actually visible at runtime.
  - confidence: 0.88
