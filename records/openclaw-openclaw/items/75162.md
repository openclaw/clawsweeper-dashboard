## Summary

Keep open. The AICM companion plugin shows a real plugin-author use case, but this PR still introduces a new raw context-assembly API with unresolved security and product-boundary issues: `before_assemble` can read and replace transcript messages without the existing conversation-access gate, the hooks run in raw model mode, and `after_assemble` does not observe the final model-visible prompt state.

## What This Changes

This PR adds `before_assemble` and `after_assemble` plugin hook types, HookRunner methods, embedded-runner call sites, a changelog entry, and a whitespace-only lockfile change.

## Best Possible Solution

Define one guarded assembly/interception extension point, or fold this use case into #61485 or the context-engine path, with explicit policy classification, raw-run boundaries, final-payload semantics, SDK/docs updates, and contract tests.

## Reproduction Assessment

Yes. Static source review of the PR head is enough: the diff exposes raw `before_assemble.messages`, leaves that hook out of `CONVERSATION_HOOK_NAMES`, runs it without an `isRawModelRun` guard, and fires `after_assemble` before the final runtime system prompt is applied.

## Solution Assessment

No. A guarded assembly hook may be useful, but this PR is not the best current implementation because it bypasses an existing safety gate, gives observers stale final-context data, and needs maintainer direction against #61485/context-engine overlap.

  - confidence: 0.88

  - body: Raw model mode skips prompt-build hooks and `llm_input`, but this new `before_assemble` call runs unconditionally. A registered handler can inject session messages into a raw model probe that is supposed to bypass normal context assembly.
  - confidence: 0.83
