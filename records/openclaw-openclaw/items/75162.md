## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Gate before_assemble as prompt mutation:** `src/plugins/hook-types.ts:177`
  - body: `before_assemble` can replace model-visible history, but the latest head only classifies it as a conversation hook. A non-bundled plugin with `allowConversationAccess=true` and `allowPromptInjection=false` could still rewrite context, bypassing the documented prompt-mutation opt-out.
  - confidence: 0.93
- **[P2] Pass cloned messages into assembly hooks:** `src/agents/pi-embedded-runner/run/attempt.ts:2605`
  - body: The hook payload hands `activeSession.messages` directly to plugin code. A handler that mutates the array or nested content and then returns `void` or throws still changes session state outside the returned-result contract; pass cloned, normalized payloads and only apply validated returned messages.
  - confidence: 0.9
- **[P2] Report the final runtime prompt to after_assemble:** `src/agents/pi-embedded-runner/run/attempt.ts:2886-2887`
  - body: `after_assemble` claims to observe the final model input, but it reports `systemPromptText` before runtime context is composed into the temporary system prompt below. Observability plugins will miss context that is actually visible to the model on non-runtime-only submissions.
  - confidence: 0.88
