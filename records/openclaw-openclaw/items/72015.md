## Summary

Keep open. Current main has meaningful mitigations for hard Active Memory timeouts and QMD startup breadth, but it still preserves the core blocking hook and current-session model fallback; draft PR #73667 is the active closing implementation candidate.

## What This Changes

The issue requests safer Active Memory recall and QMD boot initialization behavior so memory work does not block or overload normal gateway replies.

## Best Possible Solution

Review, repair, land, or replace #73667 so Active Memory has a bounded fail-open reply path and QMD startup remains opt-in or explicitly serialized before this issue is closed.

## Reproduction Assessment

Yes, at source level. Enabling Active Memory for an eligible direct session with `config.model` unset still makes `before_prompt_build` await embedded recall and resolve the active session model before fallback.

## Solution Assessment

No. Current main is a partial mitigation rather than the final fix because blocking recall and inherited-model defaults remain; the narrow maintainer path is to finish the linked PR or a replacement.
