## Summary

Keep open. Current main has meaningful partial mitigations for QMD startup laziness, hard timeout racing, and timeout circuit breaking, but it still leaves the central Active Memory behavior reported here: eligible reply-path hooks await an embedded recall run, inherit the current/session model when no plugin model is configured, and keep a broad default hook budget. The open draft PR #73667 is explicitly paired with this issue, so this issue should remain the canonical tracker until that PR or a replacement lands.

## What This Changes

The issue requests safer Active Memory and QMD reliability defaults so memory recall does not block replies, inherit slow models by default, stack concurrent recalls, or overload multi-agent gateways during startup.

## Best Possible Solution

Keep this issue as the canonical tracker until a maintainer-approved PR lands a safer default profile: Active Memory should either be opportunistic/fail-open by default or have a small strict pre-reply budget, inherited slow-model use should be avoided or diagnosed clearly, recall concurrency and cleanup should be bounded, and QMD startup refresh should remain lazy by default with explicit serialized or jittered opt-in behavior for multi-agent deployments.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by enabling Active Memory for an eligible interactive direct session with config.model unset: current main awaits maybeResolveActiveRecall in before_prompt_build, selects the current/session model, and uses a 15s recall timeout plus 30s setup grace. The QMD boot-storm side is partly mitigated by default-off startup refresh, but opt-in startup refresh behavior remains covered by the current startup policy and tests.

## Solution Assessment

No. Current main is only a partial solution because it adds timeout racing, a circuit breaker, and QMD lazy startup defaults while keeping blocking Active Memory recall, inherited model selection, and the broad default hook budget; the narrower maintainable path is to review, repair, land, or supersede PR #73667.
