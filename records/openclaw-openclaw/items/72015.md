## Summary

Keep open. Current main has partial mitigations for QMD startup laziness, hard timeout racing, and timeout circuit breaking, but the central Active Memory behavior remains: eligible replies still await a blocking `before_prompt_build` embedded run, still inherit the current/session model by default, and still use a broad 45s default hook budget. Draft PR #73667 is open, unmerged, and paired with this issue, so the issue should remain open until that PR or a replacement lands.

## What This Changes

Request to make Active Memory fail open or use a short strict deadline, avoid slow inherited models by default, bound recall concurrency, and prevent QMD startup work from overloading multi-agent gateways.

## Best Possible Solution

Keep this as the canonical tracker until #73667 or a replacement lands a maintainable default profile: Active Memory should have a fail-open or very short strict pre-reply budget, slow inherited model use should be avoided or clearly diagnosed, recall concurrency should be bounded, and QMD startup should remain lazy unless explicitly opted into serialized or jittered refreshes.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by enabling Active Memory for an eligible interactive session with `config.model` unset: the hook awaits embedded recall in `before_prompt_build`, inherits the current/session model, and defaults to a 45s hook budget. Recent comments add field reproductions on 2026.4.25 and 2026.4.29, including a single-agent non-QMD case.

## Solution Assessment

No for current main. The current implementation mitigates some failure modes, but it does not change the default reply-path blocking contract or slow-model inheritance; the narrowest path is maintainer review, landing, or superseding of #73667 with those semantics covered.
