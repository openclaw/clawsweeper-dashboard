## Summary

Keep open. Current main has partial mitigations for Active Memory deadlines, a dedicated recall lane, and lazier QMD startup, but the central reliability concern remains: Active Memory is still a blocking pre-reply embedded agent, can still inherit the active chat model, has no nonblocking or single-shot mode, and related focused work is still open or closed unmerged.

## What This Changes

The issue requests safer Active Memory and QMD startup behavior so memory recall does not block replies or overload multi-agent gateways.

## Best Possible Solution

Keep this as the canonical umbrella until maintainers land or replace the focused Active Memory prompt/deadline/lane work, then split the remaining default model, nonblocking/single-shot mode, QMD startup, and status/doctor policy into narrower follow-ups.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main still awaits Active Memory recall inside before_prompt_build, can route recall through a process-wide serial lane, and can inherit the active chat model. I did not run a live current-main multi-agent gateway canary.

## Solution Assessment

No, current main is only a partial solution. The best path needs maintainer-reviewed defaults and API choices for nonblocking or single-shot recall, model selection, QMD startup behavior, and diagnostics rather than another isolated one-line repair.
