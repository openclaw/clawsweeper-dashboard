## Summary

Keep open. Current main has partial QMD startup and timeout mitigations, but Active Memory still blocks `before_prompt_build`, still inherits the active/session model when unset, and an open draft PR already targets this issue as its closing fix.

## What This Changes

The issue asks for safer Active Memory reply-path defaults and QMD startup behavior so memory recall does not block or overload gateways.

## Best Possible Solution

Keep this issue as the canonical tracking item until PR #73667 or a replacement lands a fail-open, low-budget Active Memory path with bounded startup work.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: enable Active Memory for an eligible direct session with `config.model` unset and current main awaits embedded recall in `before_prompt_build` using the current/session model; the issue comments provide matching live logs.

## Solution Assessment

Yes, directionally. The maintainable fix should keep recall opportunistic or tightly budgeted, avoid inheriting slow primary models by default, bound per-agent/session concurrency, and keep QMD startup lazy or serialized unless explicitly opted in.
