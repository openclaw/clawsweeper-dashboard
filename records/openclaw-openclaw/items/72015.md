## Summary

Keep open. Current main has mitigations for hard timeout handling and QMD startup breadth, but Active Memory still awaits embedded recall during prompt construction, still inherits the current session model by default, and the linked closing PR is open and unmerged.

## What This Changes

The issue requests safer Active Memory and QMD startup behavior so memory recall does not block replies or overload multi-agent gateways.

## Best Possible Solution

Review, repair, land, or replace https://github.com/openclaw/openclaw/pull/73667 so Active Memory cannot block user-visible replies beyond a strict bounded budget and QMD startup remains bounded, then close this issue after the fix merges.

## Reproduction Assessment

Yes, at source level. Enable Active Memory for an eligible direct session: the `before_prompt_build` hook awaits embedded recall, model resolution can inherit the current session model, and the embedded run currently has no explicit non-main lane.

## Solution Assessment

No. Current main is a partial mitigation, not the full solution, because it preserves blocking recall and inherited-model defaults; the best path is to finish the linked reliability PR or a replacement with maintainer-approved semantics.
