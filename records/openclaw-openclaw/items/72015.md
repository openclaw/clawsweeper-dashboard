## Summary

Keep open. Current main has partial Active Memory/QMD mitigations, but the central reliability path is still present: Active Memory awaits embedded recall during `before_prompt_build`, can inherit the active chat model, and the recall call still omits a dedicated lane; open PRs already track the direct repair paths.

## What This Changes

The issue asks for Active Memory and QMD startup reliability changes so memory recall cannot block normal replies or overload multi-agent gateways.

## Best Possible Solution

Keep this issue as the canonical reliability tracker until the dedicated Active Memory lane fix lands or is replaced, then split any remaining nonblocking/default-model/QMD startup policy into narrower maintainer-owned follow-ups.

## Reproduction Assessment

Yes, at source level: an eligible Active Memory `before_prompt_build` hook awaits recall, the recall embedded run omits `lane`, omitted embedded lanes resolve to `main`, and command lanes run one active task by default. I did not run a live gateway canary in this read-only review.

## Solution Assessment

No, current main is only a partial solution. The narrow dedicated-lane fix is the best first repair, while broader nonblocking/default-model/QMD startup choices need maintainer policy decisions or narrower follow-up issues.
