## Summary

Keep open. Current main has meaningful mitigations, but the core Active Memory reliability problem remains source-reproducible: eligible replies still await embedded recall during `before_prompt_build`, model resolution still inherits the active/session model when no plugin model is configured, and open draft PR #73667 is the current implementation candidate rather than a merged fix.

## What This Changes

The issue asks OpenClaw to make Active Memory recall fail open under bounded latency and reduce QMD boot/update bursts on multi-agent gateways.

## Best Possible Solution

Ship a maintainer-approved fix that keeps Active Memory within a strict pre-reply budget, avoids slow inherited-model defaults for recall, and keeps QMD startup explicitly bounded before closing this issue.

## Reproduction Assessment

Yes. Source-level reproduction is enabling Active Memory for an eligible direct session with `config.model` unset: the hook awaits `maybeResolveActiveRecall`, which runs an embedded agent and resolves to the current/session model by default.

## Solution Assessment

No. Current main is a partial mitigation, not the final solution, because recall remains an awaited pre-prompt operation and the active model remains the default fallback; the linked draft PR or a replacement needs maintainer review before closure.
