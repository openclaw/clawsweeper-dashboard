## Summary

Keep open. Current main has mitigated the worst QMD startup breadth and Active Memory timeout overrun, but the reply-critical Active Memory hook still awaits embedded recall, still inherits the active/session model when unset, and still has a related open draft closing PR plus a narrower open lane-deadlock report.

## What This Changes

The issue asks for safer Active Memory and QMD startup behavior so memory recall cannot block replies, inherit slow models by default, overrun timeouts, or create multi-agent boot bursts.

## Best Possible Solution

Review, repair, land, or replace the linked Active Memory/QMD reliability PR, and keep the lane-specific deadlock fix aligned with https://github.com/openclaw/openclaw/issues/79026 before closing this issue.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: enable Active Memory for an eligible interactive session, leave config.model unset or use a slow model, and the hook awaits embedded recall on the reply-critical path. I did not run the live multi-agent gateway scenario in this read-only review.

## Solution Assessment

No, current main is not the complete solution. It has useful timeout and QMD startup mitigations, but the blocking hook, inherited-model default, missing dedicated recall lane, and broader default/concurrency policy remain unresolved.
