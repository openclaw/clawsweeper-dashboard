## Summary

Keep open. Current main has partial mitigations for QMD startup laziness, Active Memory hard timeout handling, and timeout circuit breaking, but the central reported behavior remains: eligible Active Memory turns still await a blocking `before_prompt_build` embedded run, still inherit the active/session model by default, and still use a broad default hook budget. Draft PR #73667 is open, unmerged, and explicitly paired with this issue, so the issue should remain open until that PR or a replacement lands.

## What This Changes

Request safer Active Memory and QMD startup defaults so multi-agent gateways do not block replies or overload startup with memory recall and boot refresh work.

## Best Possible Solution

Keep this issue as the canonical tracker until #73667 or a replacement lands a maintainable default profile: Active Memory should either be opportunistic/fail-open by default or have a very small strict pre-reply budget, inherited-model behavior should be made safe or loudly diagnosed, recall concurrency should be bounded, and QMD startup should stay lazy by default with explicit serialized or jittered behavior for opt-in multi-agent refreshes.

## Reproduction Assessment

Yes. A source-level reproduction exists by enabling Active Memory for an eligible interactive session with `config.model` unset: `before_prompt_build` awaits `maybeResolveActiveRecall`, which runs an embedded sub-agent using the current/session or primary model and the default 45s hook budget. The QMD side is reproducible through opt-in `memory.qmd.update.startup` plus multi-agent memory search defaults, though current main has made startup refresh lazy by default.

## Solution Assessment

No. Current main is only a partial solution: it adds lazy QMD startup defaults, hard timeout racing, and a circuit breaker, but it does not make Active Memory nonblocking or short-budgeted by default and does not stop inherited slow model use. The narrowest maintainable path is to review, land, or supersede PR #73667 with those remaining semantics covered.
