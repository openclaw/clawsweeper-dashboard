## Summary

Keep open. Current main mitigates the worst QMD startup breadth and hard-timeout behavior, but Active Memory still blocks `before_prompt_build` on an embedded recall run and still inherits the current/session model when no plugin model is configured; open draft PR #73667 uses closing syntax for this same reliability work.

## What This Changes

The issue asks for safer Active Memory and QMD startup behavior so memory recall does not block replies or overload multi-agent gateways.

## Best Possible Solution

Review, repair, land, or replace #73667 so Active Memory is bounded and fail-open for normal replies while QMD startup remains opt-in or explicitly serialized, then close this after the fix merges.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: enable Active Memory for an eligible direct session with `config.model` unset, and the `before_prompt_build` hook awaits embedded recall using the current/session model before the main reply can proceed.

## Solution Assessment

No, current main is only a partial mitigation: it bounds timeout behavior and narrows QMD startup, but it keeps the blocking recall/default-model semantics that the linked PR is meant to finish.
