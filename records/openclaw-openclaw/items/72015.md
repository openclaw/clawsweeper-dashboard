## Summary

Keep open. Current main has partial mitigations for timeout handling and QMD startup breadth, but Active Memory still awaits embedded recall during prompt construction and still inherits the current/session model when no plugin model is configured; open draft PR #73667 already uses closing syntax for this reliability work.

## What This Changes

Request safer Active Memory and QMD defaults so pre-reply recall and boot work do not stall multi-agent gateways.

## Best Possible Solution

Active Memory should be fail-open for normal replies by default, avoid inherited slow primary models unless explicitly chosen, bound per-agent/session recall concurrency, and keep QMD startup lazy or explicitly serialized before this issue closes.

## Reproduction Assessment

Yes. Enable Active Memory for an eligible direct session with config.model unset: current main still awaits maybeResolveActiveRecall in before_prompt_build and resolves the current session model; the issue comments add live macOS and Linux logs for the same slowdown pattern.

## Solution Assessment

No. Current main is a partial mitigation rather than the final best solution because it preserves blocking recall and inherited-model defaults; the narrow maintainer path is to review, repair, land, or replace #73667 and then close the remaining gap explicitly.
