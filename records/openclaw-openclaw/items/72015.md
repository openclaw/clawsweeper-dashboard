## Summary

Keep open. Current main has meaningful mitigations for hard timeout handling and QMD startup breadth, but Active Memory still awaits embedded recall during prompt construction and still inherits the current/session model when no plugin model is configured; open draft PR #73667 uses closing syntax for this same reliability work.

## What This Changes

The issue requests safer Active Memory and QMD startup behavior for gateways where blocking embedded recall and boot-time QMD initialization can overload replies.

## Best Possible Solution

Review, repair, land, or replace #73667 so Active Memory is bounded/fail-open under normal replies and QMD startup stays opt-in or explicitly serialized, then close this issue after the fix merges.

## Reproduction Assessment

Yes. Source-level reproduction is enabling Active Memory for an eligible direct session with `config.model` unset: `before_prompt_build` awaits `maybeResolveActiveRecall`, and `getModelRef` selects the current session model before fallback.

## Solution Assessment

No. Current main is a partial mitigation rather than the final fix because it preserves blocking recall and inherited-model defaults; the narrow maintainer path is to finish the linked PR or a replacement before closing this issue.
