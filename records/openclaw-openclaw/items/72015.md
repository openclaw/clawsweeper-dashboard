## Summary

Keep open. Current main has partial mitigations for QMD startup laziness, Active Memory hard-deadline racing, and timeout circuit breaking, but eligible Active Memory turns still block prompt construction, still inherit the active/session model by default, and still use a broad 45s default hook budget. Draft PR #73667 remains open and explicitly targets this issue, so this issue should stay open until that PR or a replacement lands.

## What This Changes

The issue requests safer Active Memory and QMD startup defaults so memory recall does not block replies or overload gateway boot on multi-agent or slow-host deployments.

## Best Possible Solution

Keep this issue as the canonical tracker until #73667 or a replacement lands a maintainable default profile: Active Memory should be opportunistic or tightly deadline-bound by default, inherited heavy-model behavior should be made safe or clearly diagnosed, recall concurrency should be bounded, and QMD startup should remain lazy with explicit serialized or jittered opt-in refresh behavior.

## Reproduction Assessment

Yes. A source-level reproduction path exists by enabling Active Memory for an eligible interactive session with `config.model` unset: the `before_prompt_build` hook awaits embedded recall, uses the current/session model, and has a default 45s hook budget. The QMD boot-storm part is partly mitigated on current main because startup refresh is lazy by default.

## Solution Assessment

No for current main as the complete solution. The existing mitigations reduce some failure modes, but the narrow maintainable path is still to review, land, or supersede #73667 with the remaining default-latency and inherited-model semantics covered.
