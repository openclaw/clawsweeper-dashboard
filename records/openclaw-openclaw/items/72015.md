## Summary

Keep open. Current main has useful Active Memory/QMD mitigations, but the central reply-path risk is still source-reproducible and this issue is already paired with open fixing PRs rather than being ready for cleanup close.

## What This Changes

Make Active Memory fail open with stricter recall budgets and safer defaults, and reduce QMD boot work for multi-agent gateways.

## Best Possible Solution

Keep this as the canonical reliability tracker until the open closing PRs land or are replaced, then split remaining nonblocking/default-model/QMD policy into focused follow-ups.

## Reproduction Assessment

Yes, at source level: current main still awaits Active Memory recall in before_prompt_build, the embedded recall omits a lane, omitted lanes resolve to main, and command lanes run one active task by default. I did not run a live gateway canary.

## Solution Assessment

No, current main is only a partial solution. The narrow dedicated-lane fix is the best first repair, while nonblocking mode, default recall model, QMD startup policy, and status/doctor warnings need maintainer product choices or narrower follow-ups.
