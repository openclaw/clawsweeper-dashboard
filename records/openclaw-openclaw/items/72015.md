## Summary

Keep open. Current main has partial mitigations for timeout overrun and QMD startup breadth, but Active Memory still awaits an embedded recall during prompt build and still inherits the current/session model when no plugin model is configured; open draft PR #73667 also uses closing syntax for the same work.

## What This Changes

The issue requests fail-open Active Memory recall and reduced QMD boot work so gateway replies are not delayed by memory initialization or recall subagents.

## Best Possible Solution

Active Memory should be fail-open by default with a small pre-reply budget, explicit or safe fast-model recall behavior, bounded per-agent/session concurrency, and QMD startup kept lazy or serialized for multi-agent gateways.

## Reproduction Assessment

Yes. Enable Active Memory for an eligible direct session with config.model unset: current main still awaits maybeResolveActiveRecall in before_prompt_build and resolves the current/session model before runEmbeddedPiAgent, matching the issue logs.

## Solution Assessment

No. Current main is a partial mitigation rather than the best end state because it preserves blocking recall and inherited-model defaults; the best near-term path is to review, repair, land, or replace open draft PR #73667 and explicitly address the remaining defaults.
