## Summary

Keep open. Current main has partial hard-timeout and QMD startup mitigations, but Active Memory still awaits a blocking embedded recall during prompt construction, still inherits the active session model when no plugin model is set, and this issue is already paired with an open closing PR that has not landed.

## What This Changes

The issue asks OpenClaw to make Active Memory and QMD startup safer for multi-agent gateways by avoiding reply-blocking recall, heavy default model inheritance, soft timeouts, stacked recall work, and boot-time QMD bursts.

## Best Possible Solution

Land or replace the open Active Memory lane/deadline work, keep the existing QMD startup mitigation, and split any cheap-model or nonblocking-default change into an explicit maintainer policy decision.

## Reproduction Assessment

Yes, at source level. Enabling Active Memory for an eligible interactive session still makes before_prompt_build await embedded recall, unset config.model still inherits the session model, and the embedded recall call still omits a lane that defaults to the serial main lane.

## Solution Assessment

No, current main is not a complete solution. The narrow maintainable fix is to land or replace the dedicated-lane and deadline work, while the broader nonblocking/default-model/QMD policy should be decided separately.
