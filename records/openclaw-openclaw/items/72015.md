## Summary

Keep open. Current main has useful mitigations for timeout budgeting, circuit breaking, and QMD startup breadth, but Active Memory still awaits an embedded recall during `before_prompt_build`, still inherits the current/session model when `config.model` is unset, and a linked draft PR still uses closing syntax for this reliability work.

## What This Changes

The issue asks for safer Active Memory and QMD startup behavior so reply-critical paths are not blocked or overloaded on multi-agent gateways.

## Best Possible Solution

Review, repair, land, or replace https://github.com/openclaw/openclaw/pull/73667, incorporate the narrower lane fix tracked at https://github.com/openclaw/openclaw/issues/79026 if still applicable, and close this issue only after the fix merges.

## Reproduction Assessment

Yes, source-level reproduction is clear: enable Active Memory for an eligible direct session, leave `config.model` unset or use a slow model, and the `before_prompt_build` hook awaits an embedded recall before prompt construction can proceed. I did not run a live gateway canary in this read-only review.

## Solution Assessment

No, current main is only a partial mitigation: timeout and QMD startup behavior are improved, but the reply-critical blocking recall and inherited-model default remain. The maintainable path is to finish or replace the linked reliability PR and keep the narrower lane-deadlock work aligned with it.
