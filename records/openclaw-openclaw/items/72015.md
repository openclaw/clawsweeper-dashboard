## Summary

Keep open. Current main has partial Active Memory/QMD mitigations, but the core reliability path is still present: Active Memory awaits recall during prompt construction, can inherit the active chat model, and still calls the embedded runner without a dedicated lane; related open PRs/issues already track the direct repair paths.

## What This Changes

Request to make Active Memory recall and QMD startup safer so memory work does not delay chat replies or overload multi-agent gateway startup.

## Best Possible Solution

Land or replace the dedicated Active Memory lane fix, then split or resolve the remaining nonblocking/default-model/QMD startup policy as narrower maintainer-owned follow-ups before closing this issue.

## Reproduction Assessment

Yes, at source level: an eligible Active Memory before_prompt_build hook awaits recall, the recall embedded run omits lane, omitted embedded lanes resolve to main, and command lanes run one active task by default. I did not run a live gateway canary in this read-only review.

## Solution Assessment

No, current main is only a partial solution. The narrow dedicated-lane fix is the best first repair, while broader nonblocking/default-model/QMD startup choices need maintainer policy decisions or narrower follow-up issues.
