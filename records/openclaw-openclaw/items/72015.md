## Summary

Keep open. Current main has partial mitigations for hard timeout racing and QMD startup breadth, but the central reply-path behavior remains: Active Memory still awaits an embedded recall agent from `before_prompt_build`, still inherits the current/session model when no plugin model is pinned, and still registers a 45s default hook envelope. The linked draft PR #73667 uses closing syntax for this issue and targets the same boundary, so this should remain open until that PR or a replacement lands.

## What This Changes

The issue asks for safer Active Memory and QMD reliability defaults: fail-open or nonblocking recall, cheaper recall-model selection, strict wall-clock budgets, recall concurrency limits, QMD startup throttling or laziness, and operator diagnostics.

## Best Possible Solution

Active Memory should become fail-open for normal replies by default, with a small pre-reply budget or opportunistic injection, no inherited slow primary model unless explicitly configured, bounded per-agent/session recall concurrency, and QMD startup kept lazy or serialized unless operators deliberately opt into broader refresh work.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: enable Active Memory for an eligible direct session with `config.model` unset, then the current hook awaits embedded recall and resolves the current/session model before the main prompt can proceed; the issue comments provide matching live logs.

## Solution Assessment

No. Current main is a partial mitigation rather than the best final solution because it preserves blocking recall and inherited-model defaults; the best near-term path is to review, repair, land, or replace open draft PR #73667, then reassess any remaining model-selection and concurrency gaps.
