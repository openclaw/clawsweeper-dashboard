## Summary

Keep open. Current main has partial mitigations for hard timeout handling and QMD startup breadth, but Active Memory still blocks prompt construction on embedded recall and still inherits the current/session model when no plugin model is configured; open draft PR #73667 is the active closing candidate.

## What This Changes

Make Active Memory fail open with a strict low-latency recall budget, safer model defaults/concurrency, and QMD startup throttling or laziness for multi-agent gateways.

## Best Possible Solution

Finish or replace #73667 so Active Memory is bounded/fail-open by default, avoids inherited slow primary models unless explicitly chosen, limits concurrent recall pressure, and keeps QMD startup lazy or explicitly serialized.

## Reproduction Assessment

Yes. Enable Active Memory for an eligible direct session with config.model unset: current main still awaits maybeResolveActiveRecall in before_prompt_build and resolves the current session model, matching the supplied live macOS and Linux slowdown logs.

## Solution Assessment

No, current main is not yet the complete fix. The hard-timeout and QMD startup mitigations are useful, but the remaining blocking recall and inherited-model defaults need the linked PR or an equivalent replacement before this can close.
