## Summary

Keep open. Current main has useful mitigations for Active Memory hard timeouts and QMD startup breadth, but it still awaits Active Memory recall during `before_prompt_build`, still inherits the current session model when `config.model` is unset, and open draft PR #73667 uses closing syntax for this same reliability work.

## What This Changes

The issue asks to make Active Memory and QMD startup safer by bounding or avoiding blocking recall, avoiding heavy default model inheritance, adding concurrency control, and reducing multi-agent QMD boot bursts.

## Best Possible Solution

Review, repair, land, or replace #73667 so Active Memory is bounded or fail-open for normal replies and QMD startup remains explicitly scheduled, then close this issue after that fix merges.

## Reproduction Assessment

Yes. Source-level reproduction is enabling Active Memory for an eligible direct session with `config.model` unset: `before_prompt_build` awaits `maybeResolveActiveRecall`, and `getModelRef` selects the current session model before fallback.

## Solution Assessment

No. Current main is a partial mitigation because it preserves blocking recall and inherited-model defaults; the maintainable path is to finish the linked PR or a replacement before closing this issue.
