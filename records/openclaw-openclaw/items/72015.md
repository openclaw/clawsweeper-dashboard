## Summary

Keep open. Current main has partial timeout and QMD startup mitigations, but Active Memory still awaits a nested embedded recall in the prompt-build path, still inherits the active session model when no plugin model is set, and live GitHub still has open closing PRs for this reliability family.

## What This Changes

The issue asks for safer Active Memory recall and QMD startup behavior so reply-critical turns are not blocked or overloaded on multi-agent gateways.

## Best Possible Solution

Review, repair, land, or replace the open Active Memory reliability PRs, then close this issue only after the reply-critical recall path is fixed and any broader QMD/default policy is explicitly settled.

## Reproduction Assessment

Yes, at source level. Enabling Active Memory for an eligible interactive session still makes before_prompt_build await maybeResolveActiveRecall, the recall starts an embedded agent without a dedicated lane, and unset config.model still inherits the active session model.

## Solution Assessment

No, current main is only a partial solution. The narrow fix is to land or replace the linked Active Memory lane/deadline work, while the broader nonblocking/default-model/QMD startup choices need maintainer policy review.
