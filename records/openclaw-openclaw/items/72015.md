## Summary

Keep open. Current main has partial hard-timeout and QMD startup mitigations, but Active Memory still awaits recall inside `before_prompt_build`, still inherits the active session model when no plugin model is configured, and still launches the recall embedded run without a dedicated lane; the issue also has an open closing PR at https://github.com/openclaw/openclaw/pull/73667 and a narrower open lane-fix path at https://github.com/openclaw/openclaw/issues/79026 / https://github.com/openclaw/openclaw/pull/80255.

## What This Changes

The issue requests safer Active Memory and QMD startup behavior so memory recall does not block chat replies or overload multi-agent gateways.

## Best Possible Solution

Land or replace the open dedicated-lane fix, then either repair/replace the broader reliability PR or explicitly split the remaining nonblocking/default-model/QMD startup policy into narrower tracked work before closing this issue.

## Reproduction Assessment

Yes, at source level: an eligible Active Memory `before_prompt_build` hook awaits recall, the recall call omits `lane`, omitted embedded lanes resolve to `main`, and command lanes are serial by default. I did not run a live gateway canary in this read-only review.

## Solution Assessment

No, current main is only a partial solution. The narrow dedicated-lane fix should land or be replaced, while broader nonblocking/default-model/QMD startup choices need maintainer product judgment.
