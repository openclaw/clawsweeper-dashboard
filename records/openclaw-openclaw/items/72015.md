## Summary

Keep open. Current main and the latest shipped release still have the reply-critical Active Memory recall path, and the issue is paired with open fixing PRs rather than being ready for cleanup close.

## What This Changes

The issue asks OpenClaw to stop Active Memory/QMD work from blocking normal replies or overloading multi-agent gateways at startup.

## Best Possible Solution

Keep this as the canonical reliability tracker until the open targeted PRs land or are replaced, then split any remaining nonblocking/default-model/single-shot/QMD policy into focused follow-ups.

## Reproduction Assessment

Yes, at source level: eligible before_prompt_build still awaits Active Memory recall, the recall uses an embedded run without a dedicated lane, and omitted embedded lanes default to main. I did not run a live gateway canary.

## Solution Assessment

No current-main solution fully resolves the issue. The best path is to land or replace the narrow lane/deadline fixes first, then make explicit maintainer decisions for nonblocking mode, default model policy, and QMD startup behavior.
