## Summary

Keep open. Current main has partial mitigations for QMD startup laziness and Active Memory timeout racing, but the central reported behavior remains: eligible Active Memory turns still await a blocking pre-prompt embedded run, still inherit the active/session model when no plugin model is set, and still have a broad default 45s hook budget. The open draft PR #73667 is explicitly paired with this issue, so the issue should stay open until that PR or a replacement lands.

## What This Changes

The issue requests safer Active Memory and QMD reliability defaults: fail-open/nonblocking recall, short hard deadlines, safer model selection, recall concurrency limits, QMD startup burst controls, and operator diagnostics.

## Best Possible Solution

Keep this issue as the canonical tracker until #73667 or a replacement lands a maintainable default profile: Active Memory should have a small strict fail-open pre-reply budget or become opportunistic by default, inherited slow-model use should be avoided or loudly diagnosed, recall concurrency and cancellation should be bounded, and QMD startup refresh should remain lazy by default with explicit serialized/jittered opt-in behavior for multi-agent deployments.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: enable Active Memory for an eligible interactive session with config.model unset and current main will await maybeResolveActiveRecall in before_prompt_build, choose the current/session model, and use the 15s timeout plus 30s setup grace. The QMD boot side is partially mitigated now, but opt-in startup refresh across explicitly enabled/default memory-search agents is still covered by current tests.

## Solution Assessment

No. Current main is only a partial solution because it adds hard timeout racing and QMD lazy startup defaults, but keeps the blocking Active Memory hook, inherited model default, and 45s default hook budget. The narrowest maintainable path is to review, land, or supersede the paired draft PR with the remaining semantics covered.
