## Summary

Keep open. Current main has partial mitigations, but the reply-critical Active Memory path still awaits embedded recall, still inherits the current/session model when no plugin model is configured, and still omits a dedicated lane while the embedded runner defaults omitted lanes to `main`.

## What This Changes

The issue asks OpenClaw to make Active Memory recall fail open under reply load, avoid slow inherited recall models, bound timeout/concurrency behavior, and reduce QMD boot work on multi-agent gateways.

## Best Possible Solution

Review, repair, land, or replace https://github.com/openclaw/openclaw/pull/73667 while also accounting for the dedicated-lane fix tracked at https://github.com/openclaw/openclaw/issues/79026, then close this issue after the merged implementation proves bounded fail-open reply behavior and controlled QMD startup.

## Reproduction Assessment

Yes. A high-confidence source path exists: enable Active Memory for an eligible direct session, leave `config.model` unset, and the hook awaits embedded recall while model resolution can inherit the current session model and the embedded call has no dedicated lane.

## Solution Assessment

No. Current main is only a partial mitigation because it preserves blocking recall and inherited-model defaults; the narrow path is to finish or replace the linked PR and include the lane-specific fix instead of closing this issue now.
