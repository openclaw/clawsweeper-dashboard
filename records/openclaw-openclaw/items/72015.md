## Summary

Keep open. Current main and the latest release include meaningful mitigations, but the central reliability problem remains source-reproducible: Active Memory still gates prompt construction on embedded recall, can inherit the active chat model, and uses a single process-wide recall lane while QMD startup policy and safe defaults still need maintainer direction.

## What This Changes

The issue asks OpenClaw to make Active Memory recall and QMD startup safe for multi-agent gateways by preventing blocking reply-path recall, heavy model inheritance, timeout overruns, and startup work bursts.

## Best Possible Solution

Keep this as the canonical reliability tracker until maintainers choose and land a cohesive safe-default policy for blocking versus opportunistic recall, recall model selection, recall lane isolation, QMD startup work, and diagnostics.

## Reproduction Assessment

Yes, at source level: current main still awaits Active Memory recall in before_prompt_build, can inherit the current chat model, and uses a fixed serial recall lane. I did not run a live multi-agent gateway canary in this read-only review.

## Solution Assessment

No single suggested fix is enough. Current main is a partial mitigation, and the durable solution needs a maintainer-owned design for safe defaults, config/API shape, model routing, recall scheduling, and QMD startup behavior.
