## Summary

Keep open. Current main has partial mitigations for the 45s hook budget and QMD boot breadth, but Active Memory still awaits embedded recall during prompt build and still inherits the current/session model when no plugin model is configured; open draft PR #73667 also uses closing syntax for the same work.

## What This Changes

The issue asks to make Active Memory recall fail open/bounded and QMD startup lazy or serialized so memory work cannot delay replies or stampede multi-agent gateways.

## Best Possible Solution

Active Memory should be fail-open for normal replies by default, with a small pre-reply budget or opportunistic injection, no inherited slow primary model unless explicitly configured, bounded per-agent/session recall concurrency, and QMD startup kept lazy or explicitly serialized.

## Reproduction Assessment

Yes. Enable Active Memory for an eligible direct session with config.model unset: current main still awaits the embedded recall in before_prompt_build and resolves the current/session model; the issue comments provide matching live logs across macOS and Linux.

## Solution Assessment

No. Current main is a partial mitigation rather than the final best solution because it preserves blocking recall and inherited-model defaults; the best near-term path is to review, repair, land, or replace open draft PR #73667 and then close the remaining gap explicitly.
