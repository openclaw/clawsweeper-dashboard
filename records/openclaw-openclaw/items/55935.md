## Summary

Keep open. Current main and v2026.6.8 still emit the shared natural-language `Current time:` plus `Reference UTC:` block for heartbeat prompt bodies, and the exact ISO 8601 plus trigger/current-time semantic split remains a maintainer prompt-contract decision rather than an implemented fix.

## What This Changes

The issue requests ISO 8601 `Current time` output for heartbeat prompts and an optional separate heartbeat trigger timestamp.

## Best Possible Solution

Define one shared cron-style prompt timestamp contract, then update `resolveCronStyleNow`, heartbeat trigger/processing-time semantics, sibling callers, tests, and docs together.

## Reproduction Assessment

Yes, via source inspection. Current main still appends the shared natural-language `Current time:` / `Reference UTC:` block to heartbeat runs, with `startedAt` as the timestamp and no separate trigger/current field.

## Solution Assessment

Unclear until maintainers choose the prompt-time contract. The shared helper is the likely implementation point, but the field names and semantics affect heartbeat plus sibling cron/reset/post-compaction surfaces.
