## Summary

Keep open. Current main and v2026.6.6 still emit a localized natural-language `Current time:` line plus non-ISO `Reference UTC:`, and heartbeat still passes the heartbeat start timestamp through that shared helper without a separate trigger timestamp; the merged related PR only split UTC onto its own line and the direct ISO PR was closed unmerged.

## What This Changes

The issue requests ISO 8601 formatting for heartbeat `Current time` and an optional separate `Heartbeat triggered at` timestamp.

## Best Possible Solution

Choose one model-visible timestamp contract for cron-style prompt inserts, then update the shared formatter and all affected caller tests together.

## Reproduction Assessment

Yes. Source inspection shows the current helper still emits localized `Current time` plus `Reference UTC`, and the heartbeat path uses that helper with `startedAt` rather than exposing a separate trigger timestamp.

## Solution Assessment

Unclear. The shared helper is the maintainable implementation location, but maintainers need to decide whether `Current time` should mean processing time, trigger time, or whether both fields should appear.
