## Summary

Keep open. Current main and v2026.6.10 still emit the shared natural-language `Current time:` plus `Reference UTC:` block for heartbeat prompts, and queued wake trigger time is tracked internally but not surfaced as a separate model-visible heartbeat timestamp.

## What This Changes

The issue requests ISO 8601 formatting for heartbeat prompt time context and an optional separate heartbeat trigger timestamp.

## Best Possible Solution

Define one model-visible automation timestamp contract, then update the shared current-time helper, heartbeat wake plumbing, docs if needed, and focused tests so ISO current time and any trigger timestamp appear only where maintainers want that context.

## Reproduction Assessment

Yes at source level. Current main still routes heartbeat prompt bodies through the natural-language `Current time` / `Reference UTC` helper, and the queued wake `requestedAt` value is not passed into the heartbeat runner as a separate trigger timestamp.

## Solution Assessment

Unclear. The requested ISO/split format is plausible, but the best fix needs a maintainer decision because the same helper feeds multiple model-visible prompt paths and overlaps the prompt-cache placement discussion in https://github.com/openclaw/openclaw/issues/71973.
