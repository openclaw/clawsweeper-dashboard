## Summary

Keep open. Current main and the latest release still emit natural-language heartbeat/cron `Current time:` plus `Reference UTC:` lines, and there is still no separate `Heartbeat triggered at` field; the remaining work is a prompt-contract decision rather than already-implemented cleanup.

## What This Changes

Request to change heartbeat/cron-style `Current time` lines to ISO 8601 and optionally add a separate `Heartbeat triggered at` timestamp.

## Best Possible Solution

Decide the shared cron-style timestamp contract, update `resolveCronStyleNow` and all prompt expectations together, and add a heartbeat-only trigger timestamp only if maintainers want that semantic split.

## Reproduction Assessment

Yes for the current prompt shape: source inspection shows current main still emits prose `Current time:` from `startedAt` and no `Heartbeat triggered at` field. This is a requested prompt-contract change rather than a deterministic broken-behavior repro.

## Solution Assessment

Unclear. The shared helper is the likely implementation point for the formatting half, but the trigger-versus-processing-time distinction needs maintainer agreement before choosing the final prompt shape.
