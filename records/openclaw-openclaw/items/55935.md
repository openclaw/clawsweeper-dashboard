## Summary

Keep open. Current main and the latest release still build cron/heartbeat `Current time:` with locale prose plus a non-ISO UTC suffix, heartbeat still passes the run `startedAt` into that shared helper, and there is still no separate heartbeat trigger timestamp field; the related ISO PR is closed unmerged while the open adjacent PRs only partially overlap.

## What This Changes

The issue requests ISO 8601 formatting for heartbeat `Current time` and an optional separate heartbeat trigger timestamp.

## Best Possible Solution

Update the shared cron-style current-time formatter to emit an ISO 8601 user-local timestamp with explicit offset, update all helper consumers/tests together, and add any `Heartbeat triggered at` line only in the heartbeat runner if maintainers want that distinction.

## Reproduction Assessment

Yes. The deterministic reproduction path is source-backed: call `resolveCronStyleNow` or any `appendCronStyleCurrentTimeLine` consumer with a fixed `nowMs` and it produces the old prose-plus-UTC format; heartbeat reaches the same path through `runHeartbeatOnce`.

## Solution Assessment

Yes, the narrow maintainable direction is a shared helper contract change plus targeted caller tests; #42654 and #75025 are not sufficient by themselves because they do not implement the requested ISO `Current time` format and separate heartbeat trigger semantics.
