## Summary

Keep open. Current main still emits a natural-language `Current time:` line from the shared cron-style formatter, heartbeat still passes `startedAt` into that helper with no `Heartbeat triggered at` field, and the merged related PR only separated UTC onto `Reference UTC:` without implementing ISO 8601 local time.

## What This Changes

The issue requests ISO 8601 formatting for heartbeat `Current time` and an optional separate heartbeat trigger timestamp.

## Best Possible Solution

Decide the shared cron-style timestamp contract, update the helper and all direct prompt expectations together, and add a heartbeat-only trigger timestamp only if maintainers want that semantic distinction.

## Reproduction Assessment

Yes for the reported current format. Source inspection of `resolveCronStyleNow` and heartbeat’s call path shows current main still emits prose `Current time:` from `startedAt` and has no `Heartbeat triggered at` field.

## Solution Assessment

Unclear. Changing the shared helper to ISO is likely the narrow implementation point for the formatting half, but the heartbeat trigger/processing-time distinction needs a maintainer decision before choosing the final prompt shape.
