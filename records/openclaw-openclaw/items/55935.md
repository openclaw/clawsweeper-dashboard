## Summary

Keep open. Current main and the latest release still emit the shared natural-language `Current time:` plus `Reference UTC:` block for heartbeat prompts, so the requested ISO 8601 format and separate heartbeat trigger timestamp are not implemented; the merged related work only partially reduced the UTC ambiguity.

## What This Changes

The issue requests ISO 8601 heartbeat current-time output and a separate timestamp for when the heartbeat event was triggered.

## Best Possible Solution

Define one shared cron-style prompt timestamp contract, then update heartbeat trigger/processing semantics, sibling prompt callers, regression tests, and docs together.

## Reproduction Assessment

Yes. Source inspection shows current main still appends the shared natural-language `Current time:` / `Reference UTC:` block to heartbeat runs, using `startedAt` and no separate trigger/current-time fields.

## Solution Assessment

Unclear until maintainers choose the prompt-time contract. The shared helper is the likely implementation point, but field names and semantics affect heartbeat plus sibling cron, reset, post-compaction, and memory-flush surfaces.
