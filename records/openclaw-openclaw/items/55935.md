## Summary

Keep open. Current `main` and the latest release still render heartbeat prompt time through the shared natural-language `Current time:` plus `Reference UTC:` helper, and the queued wake trigger timestamp is stored internally but not passed to the heartbeat runner as a separate model-visible field.

## What This Changes

The issue requests ISO 8601 formatting for heartbeat `Current time` and an optional separate heartbeat trigger timestamp.

## Best Possible Solution

Define the automation timestamp contract first, then update the shared current-time helper, heartbeat wake plumbing, docs, and focused tests so ISO current time and any trigger timestamp appear only in the approved prompt locations.

## Reproduction Assessment

Yes, at source level. Current main still sends heartbeat prompts through the natural-language `Current time` / `Reference UTC` helper, and the stored wake `requestedAt` value is not forwarded to the heartbeat runner.

## Solution Assessment

Unclear. The requested ISO/split format is plausible, but the best fix needs maintainer direction because the same helper feeds multiple prompt paths and overlaps prompt-cache placement work.
