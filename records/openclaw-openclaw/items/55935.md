## Summary

Keep open. Current main and the latest release still emit the shared natural-language `Current time:` plus `Reference UTC:` block for heartbeat prompts, and the wake trigger timestamp is tracked internally but not surfaced to the heartbeat runner as a separate model-visible field.

## What This Changes

The issue asks to make heartbeat `Current time` model-visible text ISO 8601 and optionally add a separate heartbeat trigger timestamp.

## Best Possible Solution

Define one model-visible automation timestamp contract, then update the shared helper, heartbeat wake plumbing, docs if needed, and focused tests so ISO current time and any trigger timestamp appear only where maintainers want that context.

## Reproduction Assessment

Yes, at source level. Current main still routes heartbeat prompt bodies through the natural-language `Current time` / `Reference UTC` helper, and the queued wake `requestedAt` value is not passed into the heartbeat runner as a separate trigger timestamp.

## Solution Assessment

Unclear. The requested ISO/split format is plausible, but the best fix needs a maintainer decision because the same helper feeds multiple model-visible prompt paths and overlaps the prompt-cache placement discussion.
