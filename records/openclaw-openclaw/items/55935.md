## Summary

Keep open. Current main and the latest release still emit the shared natural-language `Current time:` plus `Reference UTC:` block for heartbeat prompts, and the wake trigger timestamp is tracked internally but not carried into the heartbeat run prompt contract.

## What This Changes

The issue requests ISO 8601 heartbeat current-time formatting and a separate heartbeat trigger timestamp in the model-visible heartbeat prompt.

## Best Possible Solution

Define one model-visible timestamp contract for recurring automation, then update the shared current-time helper, heartbeat wake plumbing, and focused tests so ISO current time and any trigger timestamp are added only where maintainers want that context.

## Reproduction Assessment

Yes. Source inspection shows heartbeat prompt bodies still receive the natural-language `Current time` / `Reference UTC` helper block, and the queued wake `requestedAt` value is not passed into the heartbeat runner as a separate trigger timestamp.

## Solution Assessment

Unclear. The requested ISO/split format is plausible, but the best fix needs a maintainer decision because the same helper feeds multiple model-visible prompt paths and overlaps the prompt-cache placement discussion in https://github.com/openclaw/openclaw/issues/71973.
