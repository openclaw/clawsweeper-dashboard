## Summary

Keep open. Current main still emits prose `Current time:` text for heartbeat/cron prompts, has no separate `Heartbeat triggered at` field, and the related merged work only split UTC onto `Reference UTC:` without implementing ISO 8601 local time.

## What This Changes

Change the heartbeat cron-style `Current time:` prompt line to ISO 8601 and optionally add a separate heartbeat trigger timestamp.

## Best Possible Solution

Decide the shared cron-style timestamp contract, update the helper and all prompt expectations together, and add a heartbeat-only trigger timestamp only if maintainers want that semantic split.

## Reproduction Assessment

Yes for the current prompt shape. Source inspection shows current main still emits prose `Current time:` from `startedAt` and has no `Heartbeat triggered at` field.

## Solution Assessment

Unclear. The shared helper is the likely implementation point for the formatting half, but the trigger-versus-processing-time distinction needs a maintainer prompt-contract decision before choosing the final shape.
