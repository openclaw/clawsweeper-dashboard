## Summary

Keep open. Current main and v2026.6.6 still use the shared natural-language `Current time:`/`Reference UTC:` prompt block for heartbeat, and the only direct ISO implementation was closed unmerged; deciding whether to redefine `Current time` or add a separate trigger timestamp is a product-level prompt contract choice.

## What This Changes

Request to make heartbeat prompt time context ISO 8601 and distinguish the heartbeat trigger timestamp from the agent's processing/current time.

## Best Possible Solution

Choose one shared cron-style prompt timestamp contract, then update `resolveCronStyleNow`, heartbeat semantics, sibling prompt tests, and any docs together.

## Reproduction Assessment

Yes, via source inspection. `runHeartbeatOnce` appends the shared current-time block with `startedAt`, and the helper still emits natural-language `Current time:` plus `Reference UTC:` with no separate heartbeat trigger timestamp.

## Solution Assessment

Unclear until maintainers choose the desired prompt-time contract. The central helper is the likely implementation point, but the exact field names and semantics affect multiple prompt surfaces.
