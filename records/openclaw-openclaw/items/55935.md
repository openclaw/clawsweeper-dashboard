## Summary

Keep open. Current main and the latest release still use the natural-language `Current time:` plus `Reference UTC:` helper output, and the heartbeat body still passes the heartbeat `startedAt` value without a separate `Heartbeat triggered at:` field. The merged UTC-split work only reduced local/UTC ambiguity, the ISO-format PR was closed unmerged, and the open helper-refresh PR at https://github.com/openclaw/openclaw/pull/75025 addresses stale timestamps rather than this timestamp contract.

## What This Changes

The issue requests ISO 8601 heartbeat `Current time` output and an optional separate `Heartbeat triggered at` timestamp.

## Best Possible Solution

Decide the shared cron-style timestamp contract, update `resolveCronStyleNow` and all prompt expectations together, and add a heartbeat-only trigger timestamp only if maintainers want that semantic split.

## Reproduction Assessment

Yes for the prompt contract: source inspection shows current main deterministically produces natural-language `Current time:` and `Reference UTC:` from `startedAt`, with no `Heartbeat triggered at:` field. I did not run a live model turn because the issue is about the emitted prompt shape.

## Solution Assessment

Unclear. ISO formatting likely belongs in the shared `resolveCronStyleNow` helper, but whether `Current time` should become processing time and whether heartbeat needs a separate trigger timestamp require maintainer agreement first.
