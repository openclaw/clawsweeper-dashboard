## Summary

Keep open. Current main and the latest release still use the shared natural-language `Current time:` plus non-ISO `Reference UTC:` block, and heartbeat still passes the heartbeat `startedAt` value into that shared line without a separate trigger timestamp; the merged related PR only split UTC onto its own line, while the direct ISO attempt was closed unmerged.

## What This Changes

The issue requests ISO 8601 heartbeat time context and an optional separate `Heartbeat triggered at` timestamp.

## Best Possible Solution

Choose one shared model-visible timestamp contract for cron-style prompt inserts, then update `resolveCronStyleNow`, heartbeat wiring, docs, and affected prompt tests together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: heartbeat builds its body with `appendCronStyleCurrentTimeLine(prompt, cfg, startedAt)`, and that helper still emits the non-ISO `Current time:` block with no separate trigger timestamp.

## Solution Assessment

Unclear until maintainers choose the prompt-time contract. The shared helper is the right implementation boundary, but the exact field names and timestamp semantics need product judgment first.
