## Summary

Keep open. Current main still emits natural-language `Current time:` plus `Reference UTC:` for heartbeat/cron-style prompts, has no `Heartbeat triggered at` field, and the merged related work only separated UTC from the local prose line rather than implementing ISO 8601.

## What This Changes

The issue requests ISO 8601 `Current time` formatting for heartbeat time context and an optional separate `Heartbeat triggered at` timestamp.

## Best Possible Solution

Decide the shared cron-style timestamp contract, update the helper and all prompt expectations together, and add a heartbeat-only trigger timestamp only if maintainers want that semantic split.

## Reproduction Assessment

Yes for the current prompt shape: source inspection shows current main still emits prose `Current time:` from `startedAt` and no `Heartbeat triggered at` field. This is a requested prompt-contract change rather than a deterministic broken-behavior repro.

## Solution Assessment

Unclear. The shared helper is the likely implementation point for the formatting half, but the trigger-versus-processing-time distinction needs maintainer agreement before choosing the final prompt shape.
