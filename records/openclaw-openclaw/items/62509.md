## Summary

Keep open. Current main still runs the installer finalization npm and daemon-status probes without a shell-level timeout, and read-only stub reproductions show either probe can block the installer indefinitely if the child command hangs.

## What This Changes

The issue requests bounded, non-fatal timeouts for install.sh finalization probes, especially `npm prefix -g` and `openclaw daemon status --json`.

## Best Possible Solution

Add a small cross-platform bounded-probe helper in `scripts/install.sh` and use it for non-essential finalization commands so install success is never blocked by npm prefix lookup, binary resolution side effects, or daemon status probing. Cover the behavior with focused shell-stub tests and add a user-facing changelog fix entry.

## Reproduction Assessment

Yes. The full live macOS installer was not run, but the current helper behavior is reproducible with read-only shell stubs: a sleeping `npm prefix -g` or sleeping `openclaw daemon status --json` causes the sourced installer helper to wait until an outer timeout kills it.

## Solution Assessment

Yes. The narrow maintainable fix is to bound only best-effort finalization probes and warn on timeout while preserving install success; changing npm or daemon-status internals alone would not protect the installer from arbitrary child-command hangs.
