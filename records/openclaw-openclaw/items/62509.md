## Summary

Keep open. Current main still has unbounded best-effort installer finalization probes, and shell stubs confirm a hung `npm` or daemon-status child can still block until an outer timeout kills the installer path.

## What This Changes

Bound the installer's best-effort finalization probes so `npm prefix -g` and `openclaw daemon status --json` cannot hang an otherwise successful install.

## Best Possible Solution

Implement a portable bounded-command helper for nonessential installer probes, warn on timeout, preserve install success, and add shell-stub regression coverage plus a changelog fix entry.

## Reproduction Assessment

Yes. Sourcing `scripts/install.sh` with `OPENCLAW_INSTALL_SH_NO_RUN=1` and stubbing either `npm` or the daemon-status command to sleep reproduces the hang path on current main until an outer timeout exits with 124.

## Solution Assessment

Yes. Bounding only the best-effort installer probes is the narrow maintainable fix; changing npm or daemon-status internals alone would not protect the installer from arbitrary child-command hangs.

## Repair Work Prompt

Fix openclaw/openclaw#62509. Current `scripts/install.sh` finalization can hang because `npm_global_bin_dir` runs `npm prefix -g`/`npm config get prefix` directly and `is_gateway_daemon_loaded` runs `"$claw" daemon status --json` directly. Add a portable bounded-command helper for best-effort installer probes on macOS/Linux/WSL; do not depend only on GNU `timeout`. Use it for finalization/diagnostic npm probes and daemon-status checks, warn when a nonessential probe times out, and keep install success independent of those probe results. Cover this in `test/scripts/install-sh.test.ts` by sourcing `install.sh` with `OPENCLAW_INSTALL_SH_NO_RUN=1` and stubbing `npm` and a `claw` command to sleep; assert the probe returns within budget and emits the expected warning/fallback. Add a single-line `CHANGELOG.md` Unreleased Fixes entry for this issue, thanking @rootwhois. Do not change core daemon behavior, and do not claim #73814 is fixed unless the piped-stdin prompt/read path is separately addressed and tested.
