## Summary

Keep open. Current main and the latest release still run the reported installer finalization probes without a timeout, and read-only shell stubs reproduce both hang paths; the related stdin-stealing thread and open fix PR cover a separate installer hang mode, not these unbounded probes.

## What This Changes

The issue requests bounded, nonfatal timeouts around installer finalization probes such as `npm prefix -g` and `openclaw daemon status --json`.

## Best Possible Solution

Add a portable bounded-probe helper for nonessential installer checks, warn on timeout, preserve install success, and cover the npm and daemon-status paths with focused shell-stub tests plus a changelog fix entry.

## Reproduction Assessment

Yes. On current main, source-level stubs that make either `npm` or `openclaw` sleep cause the affected helper to hang until an outer timeout kills it with status 124.

## Solution Assessment

Yes. Bounding the best-effort installer probes is the narrow maintainable fix; changing npm, daemon internals, or the separate pipe-guard work would not protect this finalization path from arbitrary child-command hangs.
