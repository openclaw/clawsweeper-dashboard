## What This Changes

The issue asks for installer finalization probes such as `npm prefix -g` and `openclaw daemon status --json` to be bounded so setup cannot hang indefinitely.

## Best Possible Solution

Add a portable bounded-probe helper for nonessential installer checks, warn on timeout, preserve install success, and cover both npm and daemon-status hang paths with focused shell-stub tests.

Yes. Sourcing current `scripts/install.sh` with `OPENCLAW_INSTALL_SH_NO_RUN=1` and stubbing either `npm` or `openclaw` to sleep makes the affected helper hang until an outer timeout exits 124.

## Solution Assessment

Yes. Bounding the best-effort finalization probes is the narrow maintainable fix; the separate pipe-stdin guard work does not protect this path from arbitrary child-command hangs.

Fix https://github.com/openclaw/openclaw/issues/62509 by bounding nonessential installer finalization probes so setup cannot hang indefinitely. In `scripts/install.sh`, add or reuse a portable bounded-probe helper for `npm_global_bin_dir`/direct `npm prefix -g` lookups and `is_gateway_daemon_loaded`/`openclaw daemon status --json`; warn when a best-effort probe times out, preserve install success, and do not make finalization fail because a probe hangs. Keep the fix scoped to installer probing and tests; do not attempt to solve the separate piped-stdin corruption tracked by https://github.com/openclaw/openclaw/issues/73814 and https://github.com/openclaw/openclaw/pull/82918 except to avoid regressions.
