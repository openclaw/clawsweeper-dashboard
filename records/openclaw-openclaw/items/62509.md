## Summary

Keep open. Current main still runs the reported best-effort installer probes without a timeout, and a read-only shell-stub harness reproduces the hang path for both `npm prefix -g` and `openclaw daemon status --json`.

## What This Changes

The issue asks for bounded installer finalization probes so hung `npm prefix -g` or `openclaw daemon status --json` calls cannot block installation success.

## Best Possible Solution

Add a portable bounded-probe helper for nonessential installer finalization and diagnostic checks, warn on timeout, preserve install success, and cover it with focused shell-stub tests plus a changelog fix entry.

## Reproduction Assessment

Yes. A read-only current-main shell harness can source `scripts/install.sh` with `OPENCLAW_INSTALL_SH_NO_RUN=1`; stubbing either `npm` or the daemon-status command to sleep makes the helper wait until an outer timeout kills it.

## Solution Assessment

Yes. Bounding only the best-effort installer probes is the narrow maintainable fix; changing npm or daemon-status internals alone would not protect the installer from arbitrary child-command hangs.

In `scripts/install.sh`, add a portable bounded-probe helper for nonessential installer finalization and diagnostic probes so a hung `npm prefix -g`, `npm config get prefix`, or `openclaw daemon status --json` cannot block installation indefinitely.
