## Summary

Keep open. Current main still has unbounded best-effort installer finalization probes, and read-only shell stubs confirm a hung `npm` or daemon-status child can block the installer indefinitely.

## What This Changes

The issue asks for installer finalization probes to be bounded so npm/global-bin and daemon status checks cannot hang installation.

## Best Possible Solution

Add a portable bounded-probe helper for non-essential installer finalization and diagnostic checks, warn on timeout, preserve install success, and cover the behavior with focused shell-stub tests plus a changelog fix entry.

## Reproduction Assessment

Yes. A read-only shell harness against current main can source `scripts/install.sh` with `OPENCLAW_INSTALL_SH_NO_RUN=1`; stubbing either `npm` or the `openclaw` daemon-status command to sleep makes the helper wait until an outer timeout kills it.

## Solution Assessment

Yes. Bounding only the best-effort installer probes is the narrow maintainable fix; changing npm or daemon-status internals alone would not protect the installer from arbitrary child-command hangs.

## Repair Work Prompt

Fix openclaw/openclaw#62509. In `scripts/install.sh`, add a small portable bounded-probe helper for non-essential post-install checks so finalization cannot hang indefinitely when `npm prefix -g`, `npm config get prefix`, binary resolution side effects, or `openclaw daemon status --json` block. Use it in the finalization/PATH-warning and daemon-refresh paths while preserving install success and printing a concise warning when a probe times out. Do not rely only on GNU `timeout`; the implementation must work on default macOS, Linux, and WSL. Do not broaden the fix into onboarding or the separate piped-stdin behavior from #73814 unless a focused reproduction proves the same bounded-probe helper covers it. Add regression tests in `test/scripts/install-sh.test.ts` using `OPENCLAW_INSTALL_SH_NO_RUN=1` and shell stubs that sleep, and add a user-facing fix entry to `CHANGELOG.md`.
