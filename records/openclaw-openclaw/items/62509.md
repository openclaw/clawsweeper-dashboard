## Summary

Keep open. Current main and the latest release still run the reported installer probes without a timeout, and read-only shell stubs reproduce both hang paths with an outer timeout.

## What This Changes

The issue requests bounded installer finalization probes so `npm prefix -g` and `openclaw daemon status --json` cannot hang installation indefinitely.

## Best Possible Solution

Add a portable bounded-probe helper for nonessential installer checks, warn on timeout, preserve install success, and cover it with focused shell-stub tests plus a changelog fix entry.

## Reproduction Assessment

Yes. On current main, sourcing `scripts/install.sh` with `OPENCLAW_INSTALL_SH_NO_RUN=1` and stubbing either `npm` or `openclaw` to sleep makes the affected helper wait until an outer timeout kills it with status 124.

## Solution Assessment

Yes. Bounding only the best-effort installer probes is the narrow maintainable fix; changing npm or daemon-status internals alone would not protect the installer from arbitrary child-command hangs.
