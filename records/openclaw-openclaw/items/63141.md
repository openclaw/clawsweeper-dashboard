## Label Justifications

- `P2`: The issue reports a real production continuity problem, but the safest next step is a bounded UX improvement rather than an urgent core-runtime regression fix.
- `impact:session-state`: Oversized parent-fork skips start child sessions without inherited context, making the agent appear to forget recent conversation state.

## Summary

Keep open. Current main and the latest release still handle oversized parent forks by starting the child with isolated context after a log-only skip on the channel/thread path; the requested visible warning, pre-fork compaction, and model-aware threshold sizing are not implemented.

## What This Changes

Request to replace log-only oversized parent-fork skips with automatic compaction, a visible channel warning, or model-aware threshold sizing.

## Best Possible Solution

Add a session-owned, deduped visible channel/thread warning when parent-fork inheritance is skipped, and leave automatic parent compaction plus dynamic cap sizing to separate maintainer-approved compaction/model-budget work.

## Reproduction Assessment

Yes. Source and tests give a high-confidence path: create a parent session above 100,000 tokens, initialize a child with ParentSessionKey, and current main marks the child forked with a fresh isolated session while only logging the skip on the channel/thread path.

## Solution Assessment

Yes for the narrow warning slice. A deduped visible warning is the smallest maintainable next step; automatic parent compaction and dynamic threshold sizing are broader compaction/model-budget policy changes.
