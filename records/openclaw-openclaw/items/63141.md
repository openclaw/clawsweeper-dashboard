## Summary

Keep open. Current main still starts oversized parent-fork channel/thread children with isolated context after a log-only skip; the requested visible warning, pre-fork compaction, and model-aware cap sizing are not implemented.

## What This Changes

Request to auto-compact, visibly warn, or model-scale parent fork sizing when oversized parent sessions are skipped.

## Best Possible Solution

Add a session-owned, deduped visible warning when channel/thread parent inheritance is skipped, leaving automatic parent compaction and model-derived cap sizing for separate design work.

## Reproduction Assessment

Yes. Source and tests provide a high-confidence path: create a parent session above 100,000 tokens, initialize a child with ParentSessionKey, and current main marks the child forked with fresh isolated context while only logging the skip on the channel/thread path.

## Solution Assessment

Yes for the narrow warning slice. A deduped visible warning is the smallest maintainable next step; auto-compaction and dynamic cap sizing are broader compaction/model-budget policy changes that should not be bundled into this repair.
