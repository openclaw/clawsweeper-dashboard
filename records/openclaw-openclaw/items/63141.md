## Summary

Keep open. Current main still skips oversized parent-session inheritance behind an internal 100,000-token cap with only a log line in the channel/thread session-init path; the subagent path has a visible fallback note, but normal channel forks still do not warn users.

## What This Changes

The issue requests auto-compaction, a visible warning, or model-scaled sizing when an oversized parent session prevents child sessions from inheriting context.

## Best Possible Solution

Add a rate-limited, session-owned visible warning for skipped parent inheritance now, while leaving automatic parent compaction and model-derived sizing to separate design work.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: create a parent session entry above 100,000 tokens, initialize a child with ParentSessionKey, and current main marks the child fork handled while starting it isolated and only logging the skip.

## Solution Assessment

Yes for the warning-only slice. A deduped visible warning is the smallest maintainable next step; auto-compaction and dynamic cap sizing touch broader compaction and model-budget policy and should stay out of the narrow repair.
