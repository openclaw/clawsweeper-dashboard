## Summary

Keep open. Current main and the latest release still use the fixed 100,000-token parent-fork cap, log oversized parent skips, and start the child isolated without auto-compaction, model-aware sizing, or a channel-visible warning.

## What This Changes

Request a safer handling path when oversized parent-session inheritance is skipped, such as visible warning, pre-fork compaction, or model-aware sizing.

## Best Possible Solution

Add a session-owned, rate-limited visible warning when parent inheritance is skipped, and leave automatic parent compaction plus model-derived cap sizing for separate design work.

## Reproduction Assessment

Yes. Source and tests show a high-confidence path: create a parent session above 100,000 tokens, initialize a child with ParentSessionKey, and current main starts the child isolated while only logging the skip.

## Solution Assessment

Yes for the narrow warning slice. A deduped visible warning is the smallest maintainable next step; auto-compaction and dynamic cap sizing are broader compaction/model-budget policy changes.
