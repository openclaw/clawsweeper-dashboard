## Summary

Keep open. Current main still skips oversized parent-session inheritance for channel/thread session initialization by logging a warning and starting the child isolated; it does not auto-compact the parent, derive the cap from model context, or emit a channel-visible warning for the reported cold starts.

## What This Changes

Request a warning, auto-compaction, or model-aware sizing when an oversized parent session would otherwise be skipped during child session initialization.

## Best Possible Solution

Add a session-owned, one-time visible warning when parent inheritance is skipped, while leaving automatic parent compaction and model-aware threshold sizing for separate design work.

## Reproduction Assessment

Yes. Existing current-main tests and source provide a high-confidence reproduction path by creating an oversized parent session and verifying the child starts fresh while the skip is only logged; I did not execute tests in this read-only review.

## Solution Assessment

Yes for the narrow warning slice. A rate-limited visible warning is the smallest maintainable next step; auto-compaction and dynamic cap sizing should remain separate because they touch broader compaction and model-budget policy.
