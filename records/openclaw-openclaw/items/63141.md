## Summary

Keep open. Current main and the latest release still start oversized parent-fork child sessions with isolated context after a log-only skip on the channel/thread path; the subagent path already has a visible fallback note, but the reported channel-visible warning, pre-fork compaction, and model-aware cap sizing are not implemented.

## What This Changes

Request to avoid cold child sessions when parent fork context is over budget by auto-compacting, surfacing a visible warning, or model-scaling the fork cap.

## Best Possible Solution

Add a session-owned, deduped visible warning when channel/thread parent inheritance is skipped, while leaving automatic parent compaction and model-derived cap sizing for separate design work.

## Reproduction Assessment

Yes. Source and tests provide a high-confidence path: create a parent session above 100,000 tokens, initialize a child with ParentSessionKey, and current main marks the child forked with fresh isolated context while only logging the skip on the channel/thread path.

## Solution Assessment

Yes for the narrow warning slice. A deduped visible warning is the smallest maintainable next step; auto-compaction and dynamic cap sizing are broader compaction/model-budget policy changes that should not be bundled into this repair.
