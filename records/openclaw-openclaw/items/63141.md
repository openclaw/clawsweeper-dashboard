## Summary

Keep open. Current main still skips oversized parent-session inheritance by using a fixed 100,000-token cap, logging the skip, marking the child handled, and starting isolated; it does not auto-compact, model-scale the cap, or send a channel-visible warning.

## What This Changes

Request auto-compaction, a visible warning, or model-aware sizing when oversized parent-session context is skipped during child/thread session initialization.

## Best Possible Solution

Add a rate-limited, session-owned visible warning for skipped parent inheritance now, while leaving automatic parent compaction and model-derived sizing to separate design work.

## Reproduction Assessment

Yes. A current-main source reproduction is clear: create a parent session entry above 100,000 tokens, initialize a child with ParentSessionKey, and the tested path starts the child isolated while only logging the skip.

## Solution Assessment

Yes for the narrow warning slice. A deduped visible warning is the smallest maintainable next step; auto-compaction and dynamic cap sizing should remain separate because they touch broader compaction and model-budget policy.
