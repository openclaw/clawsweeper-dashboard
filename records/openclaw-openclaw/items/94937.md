## Summary

Current main still has the reported drop points: durable `isReasoning` payloads are emitted, then shared dispatch and outbound normalization suppress them before Telegram can persist them. The issue should stay open because multiple open PRs reference it with closing syntax, including a maintainer-labeled candidate, so the right cleanup is to land or reject one of those PRs rather than close this issue first.

## What This Changes

Telegram `/reasoning on` should persist model thinking as durable Telegram messages instead of dropping generated `type:"thinking"` content while `/reasoning stream` works.

## Best Possible Solution

Review and land one focused Telegram durable-reasoning fix, preferably the maintained candidate at https://github.com/openclaw/openclaw/pull/95051 if its review blockers are cleared, then close this issue with merged commit proof.
