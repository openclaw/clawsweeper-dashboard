## What This Changes

The issue reports that Telegram `/reasoning on` drops `type:"thinking"` reasoning parts instead of rendering them as persistent messages, while `/reasoning stream` still shows the thinking lane.

## Best Possible Solution

Land a narrow Telegram-only durable `isReasoning` delivery fix with regression coverage and redacted Telegram proof, then close this issue after the fixing PR merges.

## Reproduction Assessment

Yes, source-level: current `main` emits `isReasoning` block and final payloads, then the shared dispatcher suppresses both before Telegram can persist them. I did not run a live Telegram/DeepSeek session in this read-only review.

## Solution Assessment

Yes for the issue direction: the best fix is narrow Telegram-owned durable `isReasoning` delivery while preserving non-Telegram suppression, not a provider-specific path or new config option. The stronger candidate path appears to be the shared dispatch gate repair, but it still needs maintainer review and live Telegram proof.
