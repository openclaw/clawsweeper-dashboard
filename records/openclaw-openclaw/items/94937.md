## Summary

Keep open: current main still has a source-reproducible Telegram durable reasoning loss. The agent path can emit `isReasoning` block and final payloads for `/reasoning on`, but the shared dispatch path suppresses those payloads before Telegram can persist them; `/reasoning stream` works through a separate streaming callback.

## What This Changes

The issue reports that Telegram `/reasoning on` loses `type:"thinking"` content instead of persisting it as durable thinking messages, while `/reasoning stream` still shows the same thinking lane.

## Best Possible Solution

Route explicit durable `isReasoning` payloads to Telegram when `/reasoning on` is active while preserving `/reasoning off`, `/reasoning stream`, and generic-channel suppression for channels without reasoning handling.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: `/reasoning on` can emit durable `isReasoning` block/final payloads, and the shared dispatch path suppresses those payloads before Telegram delivery. I did not run a live Telegram session in this read-only review.

## Solution Assessment

Yes, the best fix is a narrow dispatch or channel-capability repair that lets Telegram receive durable reasoning payloads it already knows how to render. A provider-specific DeepSeek workaround or new config option would be the wrong boundary.
