## Best Possible Solution

## Solution Assessment

Yes, the patch is a narrow maintainable fix if maintainers accept a one-shot session marker. A wrapper-only TTS gate is not enough because the speech runtime already gates on `inboundAudio`; the missing part is the active-run metadata handoff.
