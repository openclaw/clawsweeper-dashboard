## Summary

Keep open. Current main and v2026.6.10 still provide sidebar/full-message recovery and JSON/tool disclosures, but not inline show-more collapse for long assistant chat bubbles; the same-scope progressive-disclosure PR was closed unmerged.

## What This Changes

Request to add collapsible/foldable assistant messages in Control UI chat, with per-message show-more toggles and optional global expand/collapse controls.

## Best Possible Solution

Ship a Control UI-owned progressive-disclosure design that preserves the full-message sidebar path, avoids expensive long-markdown rendering before collapse, localizes new copy, and proves long markdown/code-block behavior in focused UI tests plus browser proof.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives a high-confidence current behavior check: long assistant markdown still renders inline without a show-more threshold or per-message collapse state.

## Solution Assessment

Yes, keeping this open for a maintainer-shaped UI feature is the best current path. The existing side reader and JSON/tool disclosures are complementary, but they do not solve the main-thread scrolling problem this request targets.
