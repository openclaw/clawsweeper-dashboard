## Summary

Keep open: current main has adjacent chat compaction controls for tools, JSON, duplicates, history limits, and canvas viewing, but it does not implement the requested long assistant-message fold/show-more behavior.

## What This Changes

The issue requests collapsible or foldable assistant messages in the Control UI chat to reduce scrolling in long conversations.

## Best Possible Solution

Add an accessible Control UI-owned folding state for long assistant markdown messages, likely keeping the newest or streaming response expanded and reusing existing chat action patterns without changing tool/JSON collapse behavior.

## Reproduction Assessment

Not applicable. This is an enhancement request, and source inspection of current main shows ordinary assistant markdown is still fully rendered rather than governed by a failing existing contract.

## Solution Assessment

Unclear until a maintainer chooses the UX policy. The requested direction is plausible, but the narrow maintainable path should define threshold, latest-response behavior, and accessibility before implementation.
