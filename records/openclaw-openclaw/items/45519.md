## Summary

Keep open: current main still lacks fold/show-more state for long assistant markdown in Control UI chat. The remaining work is a user-facing UX enhancement, and the collapse threshold, latest/streaming behavior, code-block handling, and accessibility policy need maintainer product judgment before a fix PR is safe.

## What This Changes

The issue requests foldable long assistant messages in the Control UI chat to reduce scrolling in long conversations.

## Best Possible Solution

Add an accessible Control UI-owned folding state for long assistant markdown after maintainers choose the UX policy, reusing existing chat action/state patterns without changing tool-message, JSON, duplicate-message, or history truncation behavior.

## Reproduction Assessment

Not applicable. This is an enhancement request, and current source inspection shows ordinary assistant markdown renders directly as `.chat-text` rather than violating an existing documented behavior.

## Solution Assessment

Unclear until maintainers choose the UX policy. The fold/show-more direction is plausible, but the implementation should first define threshold, latest/streaming behavior, code-block handling, and accessibility semantics.
