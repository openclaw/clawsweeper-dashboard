## Summary

Keep open. Current main and v2026.6.8 still render assistant markdown directly in the Control UI chat bubble, while the merged full-message reader only adds a side-reader escape hatch for truncated assistant content and the same-scope progressive-disclosure PR was closed unmerged.

## What This Changes

The issue requests collapsible or foldable assistant messages in the Control UI chat to reduce scrolling in long conversations.

## Best Possible Solution

Ship a Control UI-owned progressive-disclosure design that keeps streaming/latest content readable, preserves the full-message sidebar fallback, localizes new copy through the UI i18n source, and proves long markdown/code-block behavior in focused tests plus browser proof.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection shows current main renders assistant markdown directly in `.chat-text` and has no inline collapse threshold or per-message show-more state.

## Solution Assessment

Yes, keeping this open for a maintainer-shaped UI feature is the best current path. The existing side reader is complementary, but it does not solve the main-thread scrolling problem this request targets.
