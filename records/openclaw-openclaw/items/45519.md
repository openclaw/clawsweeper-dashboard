## Summary

Keep open: current main and v2026.6.6 still do not provide inline Show more/Show less folding for long assistant messages. The Control UI now has adjacent mitigations, including the side-reader full-message fetch, JSON/tool collapses, duplicate-message coalescing, and bounded history rendering, but this remains a product/UX feature request rather than implemented behavior.

## What This Changes

The issue requests collapsible or foldable assistant messages in the Control UI chat to reduce scrolling in long conversations.

## Best Possible Solution

Define a Control UI-native progressive-disclosure design for long assistant bubbles, then implement it with accessible per-message state, streaming-safe behavior, mobile layout coverage, and focused renderer/browser tests while preserving the shipped side-reader path.

Not applicable: this is a new UX capability request, not a broken existing-behavior bug. Source inspection of current main and v2026.6.6 shows adjacent mitigations but no inline long-message Show more/Show less path.
