Keep open: current main and v2026.5.20 still render ordinary assistant markdown inline without an inline Show more/Show less folding affordance. The related implementation PR at https://github.com/openclaw/openclaw/pull/51546 targets this feature but is open, conflicting, and previously reviewed as needing proof and availability fixes; https://github.com/openclaw/openclaw/issues/84651 is adjacent full-message-reader work rather than a replacement for inline message folding.

## What This Changes

The issue requests collapsible/foldable assistant messages in the Control UI chat, including automatic collapse for long responses and per-message expand/collapse controls.

## Best Possible Solution

Define the Control UI long-message folding semantics, then land a revised bounded implementation or replacement for https://github.com/openclaw/openclaw/pull/51546 with focused tests and browser proof.
