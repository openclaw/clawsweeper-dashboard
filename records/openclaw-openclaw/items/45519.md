## Summary

Keep open: current main has adjacent collapse behavior for tool output, pure JSON, duplicate messages, and bounded history, but it does not implement collapsible long assistant response bubbles in the Control UI chat.

## What This Changes

Request to add collapsible or foldable assistant messages in the Control UI chat so long responses consume less vertical space.

## Best Possible Solution

Add a deliberate Control UI message-level disclosure design for long assistant responses, reusing the existing chat expansion patterns while keeping streaming/current responses readable and covering markdown, code blocks, JSON, attachments, and mobile layout in focused UI tests.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken documented behavior. Source inspection gives a high-confidence current-main baseline: ordinary assistant markdown is rendered expanded while only tool/JSON/duplicate/history cases have collapse-like behavior.

## Solution Assessment

Yes in direction, but the implementation needs maintainer UX choices before automation should write it. A Control UI-level disclosure around long assistant message content is the right layer; changing gateway transcript/history behavior would be the wrong place for this request.
