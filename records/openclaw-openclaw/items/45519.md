## Summary

Keep open. Current main has collapsible tool/JSON affordances and an “open in canvas” action, but ordinary long assistant markdown still renders inline without a per-message fold/show-more control; the request remains a valid Control UI enhancement that needs maintainer UX choices.

## What This Changes

Request to add collapse/expand controls for long assistant responses in Control UI chat, optionally including collapse-all and expand-all actions.

## Best Possible Solution

Add a Control UI-owned collapse state for long assistant markdown that preserves existing tool/JSON disclosure behavior, keeps streaming/latest content readable, and includes focused UI tests plus changelog/docs if the UX becomes user-facing configuration or help text.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that ordinary assistant markdown currently renders fully inline, while only tool/JSON content has collapse affordances.

## Solution Assessment

Unclear until maintainers choose the UX defaults. The requested Control UI direction is plausible, but threshold, persistence, latest-message behavior, and collapse-all scope should be decided before coding.
