## What This Changes

Requested collapsible or foldable long assistant responses in the Control UI chat to reduce scrolling in long conversations.

## Best Possible Solution

Keep the issue open for maintainer UX direction, then implement one Control UI folding policy in the chat renderer with focused tests for long markdown, streaming, accessibility, and side-reader coexistence.

## Reproduction Assessment

Not applicable: this is a feature request for a new inline folding interaction, not a broken existing contract. Source inspection confirms the requested generic Show more/Show less assistant-message control is absent on current main and v2026.6.6.

## Solution Assessment

Unclear: inline folding is a plausible solution, but it is not yet proven to be the best maintainable policy versus improving the existing side reader or folding only code/tool-heavy blocks. Maintainers should decide the UX contract before implementation.
