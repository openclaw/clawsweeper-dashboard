## Summary

Current main and the latest release provide an assistant-message canvas reader, including on-demand recovery for truncated history, but ordinary long assistant markdown still renders inline without per-message Show more/Show less folding. The closed unmerged implementation at https://github.com/openclaw/openclaw/pull/51546 is not a viable canonical replacement, while the merged full-reader work at https://github.com/openclaw/openclaw/pull/84670 solves a related but distinct truncation problem, so this feature remains a product decision rather than an implemented or superseded item.

## What This Changes

Add collapsible or foldable assistant messages in Control UI chat to reduce scrolling through long responses.

## Best Possible Solution

Keep the existing canvas reader for full-content access, then add per-message inline folding only after maintainers define a content or height threshold, keep streaming and the newest response expanded, and specify accessible Show more/Show less behavior with focused browser tests.

## Reproduction Assessment

Not applicable: this issue requests a new Control UI interaction, and current source inspection confirms that inline folding is absent.

## Solution Assessment

Unclear: inline folding is a reasonable solution, but the exact default-collapse policy and how it should coexist with the newer canvas reader require maintainer UX judgment.
