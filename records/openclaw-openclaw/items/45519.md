## Summary

Keep open: current main and v2026.5.20 still render ordinary assistant markdown inline without a Show more/Show less folding affordance. The related implementation PR at https://github.com/openclaw/openclaw/pull/51546 targets this feature but is still open, conflicting, and previously reviewed as needing proof and availability fixes, while the newer full-message-reader issue at https://github.com/openclaw/openclaw/issues/84651 is adjacent rather than a replacement for inline message folding.

## What This Changes

The issue requests collapsible/foldable assistant messages in the Control UI chat, including per-message expand/collapse and optional expand-all/collapse-all controls.

## Best Possible Solution

Define the Control UI long-message folding semantics, then land a revised bounded implementation or replacement for https://github.com/openclaw/openclaw/pull/51546 that preserves oversized-rendering safety and includes focused tests plus browser proof.

## Reproduction Assessment

Not applicable as a feature request. Source review gives high confidence that current main lacks the requested inline Show more/Show less affordance rather than failing an existing documented behavior.

## Solution Assessment

Unclear until maintainers choose the UX policy. Progressive disclosure is plausible, but the implementation should bound parse/render work, preserve the oversized fallback, and handle streaming/latest-message behavior deliberately.
