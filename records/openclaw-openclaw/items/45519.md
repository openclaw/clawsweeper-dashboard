## Summary

Keep open. Current main and v2026.6.10 still do not provide inline collapse/show-more behavior for long assistant chat bubbles; the merged full-message reader is complementary side-reader work and the same-scope progressive-disclosure PR was closed unmerged.

## What This Changes

Request to add collapsible/foldable assistant messages in the Control UI chat, including default collapse for long responses and optional expand/collapse controls.

## Best Possible Solution

Ship a Control UI-owned progressive-disclosure design that keeps the newest/streaming content readable, preserves the full-message sidebar fallback, localizes new UI copy, and proves long markdown/code-block behavior with focused tests plus browser proof.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection verifies current main renders normal assistant markdown inline and lacks an inline collapse threshold or per-message show-more state.

## Solution Assessment

Yes, keeping this open for a maintainer-shaped UI feature is the best current path. The existing side reader solves truncated-history recovery, but it does not solve the main-thread scrolling problem this request targets.
