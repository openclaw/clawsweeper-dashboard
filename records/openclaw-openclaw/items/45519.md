## What This Changes

The issue requests inline collapsible/foldable assistant responses in Control UI chat, with per-message expand/collapse controls for long messages.

## Best Possible Solution

Keep the current side-reader behavior, then add inline folding only after maintainers define the threshold, latest-message behavior, streaming rules, and accessible Show more/Show less interaction.

## Reproduction Assessment

Not applicable: this is a feature request for a new inline folding interaction, not a broken existing contract. Source inspection confirms the requested inline collapse control is absent on current main and v2026.6.6.

## Solution Assessment

Unclear: inline folding is a plausible solution, but maintainers need to choose the threshold/default behavior and its relationship to the existing canvas reader before implementation.
