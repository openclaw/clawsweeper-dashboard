## Summary

Keep open: current main has tool-message and JSON disclosure UI, plus an open-in-sidebar action, but ordinary long assistant responses still render inline. This is a new Control UI UX feature rather than a broken existing contract.

## What This Changes

Request collapsible assistant responses in Control UI chat, with long messages hidden behind per-message show-more controls and optional global expand/collapse controls.

## Best Possible Solution

Add a Control UI-only disclosure layer for long assistant markdown that preserves streaming/latest responses, keeps accessibility and copy/open-sidebar actions intact, and covers the behavior with grouped-render tests plus a small docs/changelog note.

## Reproduction Assessment

Not applicable: this is a feature request, not a reported regression. Source inspection shows current main renders ordinary assistant markdown inline while only tool and JSON payloads have disclosure controls.

## Solution Assessment

Unclear until maintainers choose the default UX. A narrow Control UI message-disclosure implementation is the likely best path; changing gateway history truncation would solve a different problem.
