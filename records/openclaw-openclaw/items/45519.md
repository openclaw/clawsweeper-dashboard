## Summary

Keep open. Current main has collapsible tool-message rows and collapsible JSON/details blocks, but normal long assistant markdown is still rendered directly into the chat bubble with no message-level collapsed state, threshold, or Show more/less control. The request is concrete and narrow enough for a focused Control UI PR.

## What This Changes

Add collapsible/foldable long assistant messages in the Control UI chat, with per-message expand/collapse and optional all-message controls.

## Best Possible Solution

Implement this in the Control UI chat layer by adding message-level collapsed state for long assistant markdown, a per-message Show more/Show less control, sensible defaults that keep streaming/latest messages expanded, and focused tests for long prose, code blocks, and existing tool/JSON collapse behavior.

## Reproduction Assessment

Not applicable as a runtime bug, but the missing feature is reproducible by source inspection: normal assistant markdown renders fully in `grouped-render.ts`, while only tool and JSON paths have disclosure state.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI-only rendering/state change; no gateway protocol or transcript storage change appears necessary for this request.
