## Summary

Keep open: current main and v2026.6.6 still do not implement the requested inline Show more/Show less collapse for long assistant messages. The Control UI has adjacent mitigations, including an Open in canvas/sidebar action, JSON/tool collapse, duplicate-message coalescing, and a full-message side reader for truncated history, but those do not solve the long-response scrolling UX described here.

## What This Changes

The issue requests a Control UI chat feature that collapses long assistant responses behind per-message expand/collapse controls, with optional global controls.

## Best Possible Solution

Define a Control UI-native progressive-disclosure design for long assistant bubbles, then implement it with accessible per-message state, streaming behavior, mobile layout coverage, and focused renderer/browser tests.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing-behavior bug. Source inspection shows adjacent mitigations but no current long-message inline collapse feature.

## Solution Assessment
