## Summary

Keep open. Current main and v2026.6.6 still render assistant markdown directly in the chat bubble and only provide JSON/details/sidebar full-message affordances, while the related progressive-disclosure PR was closed unmerged and pointed the remaining work back to this issue. The request fits OpenClaw's web frontend ergonomics direction, but threshold, streaming, last-message, code-block, and collapse-all behavior need maintainer UX policy before implementation.

## What This Changes

The issue asks for collapsible/foldable assistant messages in Control UI chat, with long responses collapsed by default and optional expand/collapse controls.

## Best Possible Solution

Ship a Control UI-owned progressive-disclosure design that keeps streaming/latest content readable, preserves the full-message sidebar fallback, localizes new copy through the UI i18n source, and proves long markdown/code-block behavior in focused tests plus browser proof.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection confirms the current behavior: assistant markdown is rendered directly in `.chat-text`, with a sidebar/full-message affordance but no default inline collapse for long assistant messages.

## Solution Assessment

Yes, keeping this open for a maintainer-shaped UI feature is the best current path. The existing sidebar reader is complementary, but it does not solve the main-thread scrolling problem that this request targets.
