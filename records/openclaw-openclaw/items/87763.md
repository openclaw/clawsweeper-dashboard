## Summary

Keep open: current main and the latest release still route model-provider requests through the shared SSRF pinned dispatcher with Node family autoselection applied, and there is no model-fetch family fallback or merged canonical fix. The report is source-supported but still needs live dual-stack/provider proof before a repair direction is chosen.

## What This Changes

The issue reports model-provider fetch timeouts from the SSRF-pinned DNS dispatcher on dual-stack Node 24 systems when autoSelectFamily is enabled.

## Best Possible Solution

Fix the shared pinned-dispatcher family-selection behavior while preserving SSRF host and resolved-IP checks, then prove model fetch plus related web-search or memory guarded-fetch paths on Node 24 dual-stack networking.

## Reproduction Assessment

No high-confidence live reproduction is available yet. Source inspection shows the reported path exists on current main and in v2026.5.27, but the exact timeout depends on the reporter's dual-stack provider/network conditions.

## Solution Assessment

Unclear as proposed: a model-only IPv4/IPv6 fallback would address the reported symptom, but the maintainable fix should first settle the shared SSRF pinned-dispatcher behavior and sibling guarded-fetch surfaces.
