## Summary

Keep open. Current main improves the auth snapshot save path, but the central google-antigravity runtime remains unresolved: OpenClaw still exposes Antigravity-compatible model/provider surfaces while the pinned pi-ai package no longer ships the old Cloud Code Assist stream provider and OpenClaw has no replacement stream for `google-gemini-cli`.

## What This Changes

The issue asks OpenClaw to recover google-antigravity by updating the deprecated Antigravity client version and clearing stale runtime auth snapshots after OAuth refresh.

## Best Possible Solution

Maintainers should choose one direction: restore an owned, tested Antigravity/Cloud Code Assist transport with OAuth refresh recovery, or intentionally retire google-antigravity aliases, docs, tests, and model examples.

## Reproduction Assessment

Yes for source-level reproduction. Current main can still resolve/list google-antigravity models as `api: "google-gemini-cli"`, but neither OpenClaw nor pinned pi-ai 0.71.1 supplies a matching stream transport; live OAuth credentials would be needed for endpoint smoke proof.

## Solution Assessment

No. The issue's suggested `DEFAULT_ANTIGRAVITY_VERSION` bump targets pi-ai 0.70.x code, but current main pins 0.71.1 where that provider file is gone; the maintainable path needs a support-or-retire decision.
