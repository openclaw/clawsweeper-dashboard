## Summary

Keep open. Current main appears to improve the runtime auth snapshot save/cache path, but Antigravity support is still inconsistent: OpenClaw still documents and routes `google-antigravity` while the current `pi-ai` dependency no longer ships the old Cloud Code Assist stream provider and OpenClaw has no clear replacement transport.

## What This Changes

The issue reports that `google-antigravity` stopped working because an old Antigravity User-Agent is rejected and stale runtime credential snapshots prevent recovery after token refresh.

## Best Possible Solution

Maintainers should choose one direction: restore an owned current Antigravity/Cloud Code Assist transport with OAuth refresh coverage, or intentionally retire `google-antigravity` by removing or redirecting aliases, docs, tests, and model examples.

## Reproduction Assessment

Yes for source-level reproduction. Current main still advertises and routes `google-antigravity`, while the visible provider stream path lacks an Antigravity transport and the old upstream provider file is absent from the pinned dependency.

## Solution Assessment

No. The issue’s narrow version-bump proposal no longer fits current main because the upstream file containing the User-Agent constant is gone; the maintainable solution needs a provider support decision plus matching implementation, tests, and docs.
