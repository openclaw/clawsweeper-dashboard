## Summary

Keep open. Current main still exposes and documents `google-antigravity` as a live Google provider surface, but the old pi-ai Antigravity implementation that the report’s version bump targets is no longer present in the pinned dependency, so this needs a maintainer support-or-retire decision rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to restore `google-antigravity` by updating the deprecated Antigravity client version and clearing stale runtime auth snapshots after OAuth refresh.

## Best Possible Solution

Maintainers should either restore a supported, tested Antigravity/Cloud Code Assist transport in the Google plugin with auth-refresh recovery coverage, or intentionally retire the aliases, docs, model examples, and live-test expectations.

## Reproduction Assessment

Yes for source-level reproduction: current main still exposes and documents `google-antigravity`, while the pinned pi-ai package no longer contains the old provider implementation. Live endpoint reproduction would require Google OAuth credentials and was not established here.

## Solution Assessment

No. The suggested `DEFAULT_ANTIGRAVITY_VERSION` bump is not the best current-main fix because pi-ai 0.71.1 no longer ships that provider file; the maintainable path is a support-or-retire decision with matching code, docs, and tests.
