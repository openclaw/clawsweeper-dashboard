## Summary

Keep open. Current main still exposes google-antigravity and google-gemini-cli compatibility surfaces, but the report’s specific dependency-constant fix targets an older pi-ai implementation; the remaining work needs a maintainer decision to support or retire this unofficial transport.

## What This Changes

The issue reports google-antigravity failures from a deprecated Antigravity API version and stale in-memory credential snapshots after token refresh.

## Best Possible Solution

Choose a durable provider direction, then either restore a supported and tested Cloud Code Assist/Antigravity transport with auth-refresh recovery or consistently remove the aliases, docs, model metadata, and tests.

## Reproduction Assessment

No for a full live reproduction in this review; yes for source-level reproduction. Current main still exposes google-antigravity paths while the runtime provider lacks a direct Antigravity stream branch, and live proof would require Google OAuth credentials plus the external endpoint state.

## Solution Assessment

No. The suggested DEFAULT_ANTIGRAVITY_VERSION bump targets pi-ai 0.70.x, but current main pins pi-ai 0.71.1 and needs a support-or-retire decision instead of a narrow constant update.
