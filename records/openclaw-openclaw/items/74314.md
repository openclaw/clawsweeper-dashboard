## Summary

Keep open. Current main still exposes `google-antigravity` as an active Google hook/model/live-test surface, but the pinned `@mariozechner/pi-ai` no longer ships the old `google-gemini-cli`/Antigravity transport and OpenClaw does not provide a replacement stream for `api: "google-gemini-cli"`; the auth snapshot save path looks improved, so the remaining fix is a support-or-retire decision rather than a simple version bump.

## What This Changes

The issue asks to repair `google-antigravity` by moving past the deprecated Antigravity 1.21.9 User-Agent and clearing stale runtime auth snapshots after token refresh.

## Best Possible Solution

Maintainers should either restore an owned, tested Antigravity/Cloud Code Assist transport with OAuth refresh recovery or consistently retire the `google-antigravity` aliases, docs, live tests, and model examples.

## Reproduction Assessment

Yes for source-level reproduction. Current main can still synthesize/list `google-antigravity` models as `api: "google-gemini-cli"`, but neither the pinned pi-ai package nor OpenClaw's provider stream path supplies that transport; live OAuth credentials would be needed for endpoint smoke proof.

## Solution Assessment

No. The issue's suggested `DEFAULT_ANTIGRAVITY_VERSION` bump targets pi-ai 0.70.x, while current main pins 0.71.1 where that provider is gone; the maintainable path is a support-or-retire decision.
