## Summary

Keep open. The report still maps to an active Google provider surface, but current main no longer has the old pi-ai Antigravity transport that the suggested version bump targeted, so the remaining work needs a maintainer support-or-retire decision.

## What This Changes

The issue asks OpenClaw to restore google-antigravity by updating the deprecated Antigravity client version and clearing stale in-memory OAuth credential state after refresh.

## Best Possible Solution

Restore a tested OpenClaw-owned Antigravity/Cloud Code Assist stream and auth-refresh recovery, or intentionally retire the alias, docs, tests, and examples in one provider decision.

## Reproduction Assessment

Yes for source-level reproduction: current main still exposes `google-antigravity` and resolves models onto `api: "google-gemini-cli"`, while the pinned pi-ai package no longer ships that transport and the Google plugin does not provide a replacement stream. Live endpoint proof would require OAuth credentials.

## Solution Assessment

No. The issue’s concrete `DEFAULT_ANTIGRAVITY_VERSION` bump is aimed at pi-ai 0.70.x, but current main uses pi-ai 0.71.1 where that implementation is gone; maintainers need to choose a durable support or retirement path.
