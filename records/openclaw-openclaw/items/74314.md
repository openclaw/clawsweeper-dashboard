## Summary

Keep open. Current main partially improves auth snapshot refresh, but the central google-antigravity surface is still unresolved: OpenClaw still advertises/routes Antigravity while the pinned pi-ai package no longer ships the old Cloud Code Assist stream provider and OpenClaw has no replacement stream.

## What This Changes

The issue reports google-antigravity failures from a deprecated Antigravity User-Agent version and stale runtime OAuth credential snapshots after refresh.

## Best Possible Solution

Maintainers should either restore a Google plugin-owned Antigravity/Cloud Code Assist transport with OAuth refresh coverage, or deliberately retire google-antigravity by removing/redirecting aliases, examples, docs, and tests.

## Reproduction Assessment

Yes for source-level reproduction: current metadata/tests still route google-antigravity models to google-gemini-cli-style Cloud Code Assist behavior, while the pinned upstream package no longer ships that provider and OpenClaw does not replace it. Live reproduction of the exact sandbox rejection still requires credentials.

## Solution Assessment

No. The reported version bump alone is not the best current fix because the upstream provider containing that hardcoded version is gone; the maintainable solution needs a support-or-retire decision first.
