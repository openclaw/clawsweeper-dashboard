## Summary

Keep open. Current main no longer has the old pi-ai Antigravity transport that the version-bump workaround targeted, but it still carries google-antigravity aliases, model-list/test docs, and live-test guidance, so the repository needs an explicit support-or-retire decision rather than cleanup closure.

## What This Changes

The issue asks to repair google-antigravity by updating the deprecated Antigravity API version and preventing stale runtime auth snapshots after OAuth token refresh.

## Best Possible Solution

Resolve the product boundary: either restore a supported, tested Cloud Code Assist/Antigravity transport with auth-refresh recovery, or remove remaining google-antigravity aliases, docs, tests, and examples while pointing users to google-gemini-cli.

## Reproduction Assessment

Yes for source-level reproduction: current main still references google-antigravity in provider aliases, docs, and model tests, while no stream function handles that transport. Live endpoint proof would require Google OAuth credentials.

## Solution Assessment

No. The requested DEFAULT_ANTIGRAVITY_VERSION bump targets an older pi-ai implementation that is absent from current main, so the maintainable path is a deliberate restore-or-retire change.
