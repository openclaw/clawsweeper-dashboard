## Summary

Keep open. Current main partially addresses stale auth snapshots by replacing active runtime snapshots on auth-store save, but the google-antigravity runtime remains unresolved: OpenClaw still has Antigravity compatibility surfaces while the pinned pi-ai package no longer ships the old Cloud Code Assist stream provider and OpenClaw has no replacement stream for `google-gemini-cli`.

## What This Changes

Fix the google-antigravity provider by updating the deprecated Antigravity client version and clearing stale runtime auth snapshots after token refresh.

## Best Possible Solution

Maintainers should choose one direction: restore an owned, tested Antigravity/Cloud Code Assist transport with OAuth refresh recovery, or intentionally retire the google-antigravity aliases, docs, tests, and examples.

## Reproduction Assessment

Yes for source-level reproduction: current main still has google-antigravity compatibility hooks and tests resolving models to `api: "google-gemini-cli"`, while neither OpenClaw nor pinned pi-ai 0.71.1 supplies a matching stream transport. A live OAuth smoke would be needed for endpoint-level proof.

## Solution Assessment

No. The suggested `DEFAULT_ANTIGRAVITY_VERSION` bump targets pi-ai 0.70.x code, but current main pins pi-ai 0.71.1 where that provider file is gone; the maintainable fix needs a support-or-retire decision.
