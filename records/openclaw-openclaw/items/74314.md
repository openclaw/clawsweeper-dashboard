## Summary

Keep open. Current main still has an unresolved product/code boundary for google-antigravity: compatibility docs/tests/model hooks remain, but the pinned pi-ai package no longer ships the Cloud Code Assist stream provider and OpenClaw has no replacement or complete retirement path.

## What This Changes

The issue reports that google-antigravity rejects the old Antigravity client version and that OAuth refresh does not recover from a stale in-memory auth snapshot without restarting the gateway.

## Best Possible Solution

Maintainers should choose one direction: restore an owned current Antigravity/Cloud Code Assist transport with OAuth refresh regression coverage, or fully retire google-antigravity from aliases, docs, tests, model examples, and compatibility metadata.

## Reproduction Assessment

Yes for source-level reproduction: current main can still resolve google-antigravity model metadata, while the pinned pi-ai package no longer contains the old stream provider and OpenClaw has no replacement stream. Live reproduction needs Google OAuth credentials and sandbox access.

## Solution Assessment

No. The reported version bump is not sufficient on current main because the upstream file containing DEFAULT_ANTIGRAVITY_VERSION is gone; the maintainable fix needs a support-or-retire decision first.
