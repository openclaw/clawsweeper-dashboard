## Summary

Keep open. Current main appears to improve the auth snapshot path for OpenClaw-managed OAuth saves, but the central google-antigravity provider problem remains unresolved: the repo still exposes google-antigravity compatibility surfaces while the pinned pi-ai package no longer ships the old Cloud Code Assist stream provider and OpenClaw does not provide a replacement stream for `google-gemini-cli` models.

## What This Changes

The issue asks OpenClaw to restore google-antigravity by updating deprecated Antigravity client-version behavior and clearing stale runtime credential snapshots after token refresh.

## Best Possible Solution

Maintainers should choose one direction: restore an owned current Antigravity/Cloud Code Assist transport with tested OAuth refresh behavior, or intentionally retire google-antigravity by removing/redirecting aliases, docs, tests, and model examples.

## Reproduction Assessment

Yes for source-level reproduction. Current tests and model metadata can still resolve google-antigravity models to `api: "google-gemini-cli"`, but current main has neither the removed upstream pi-ai provider nor an OpenClaw replacement stream for that API.

## Solution Assessment

No. The issue's proposed version bump is not enough on current main because the upstream provider file that contained the Antigravity User-Agent is gone; the maintainable fix needs a support-or-retire decision and, if supported, an owned transport plus auth-cache regression coverage.
