## Summary

Keep open. Current main still treats google-antigravity as an active Google provider compatibility surface, but the pinned pi-ai package no longer includes the old Cloud Code Assist stream provider that the report’s version-bump workaround targeted.

## What This Changes

Fix the google-antigravity provider by moving off the deprecated Antigravity 1.21.9 User-Agent path and recovering from stale runtime auth snapshots after token rotation.

## Best Possible Solution

Maintainers should choose one durable direction: restore a supported, tested Antigravity/Cloud Code Assist transport with auth-refresh recovery, or intentionally retire the google-antigravity aliases, docs, tests, and examples.

## Reproduction Assessment

Yes for source-level reproduction: current main still exposes google-antigravity model/provider paths, while the pinned pi-ai 0.71.1 package no longer ships the old google-gemini-cli/Antigravity stream implementation. Live endpoint proof would require OAuth credentials.

## Solution Assessment

No. The issue’s suggested DEFAULT_ANTIGRAVITY_VERSION bump targets pi-ai 0.70.x code, but current main pins pi-ai 0.71.1 where that provider file is gone; the maintainable solution needs a support-or-retire decision.
