## Summary

Keep open. Current main refreshes saved auth snapshots, but the main google-antigravity runtime problem remains: OpenClaw still has Antigravity alias/model surfaces while pinned pi-ai 0.71.1 no longer ships the old Cloud Code Assist provider and OpenClaw does not provide a replacement google-gemini-cli stream transport.

## What This Changes

The issue asks OpenClaw to restore google-antigravity by updating the deprecated Antigravity client version and clearing stale runtime auth snapshots after OAuth refresh.

## Best Possible Solution

Maintainers should choose one path: restore an owned and tested google-antigravity/Cloud Code Assist transport with OAuth refresh recovery, or retire the Antigravity aliases, model examples, and docs with migration guidance.

## Reproduction Assessment

Yes, for source-level reproduction. Current main still recognizes/list-checks google-antigravity surfaces, but neither OpenClaw nor pinned pi-ai 0.71.1 supplies the old Cloud Code Assist stream provider; live OAuth credentials are still needed for endpoint smoke proof.

## Solution Assessment

No. The suggested pi-ai DEFAULT_ANTIGRAVITY_VERSION bump targets 0.70.x code, while current main pins 0.71.1 where that provider file is gone, so the maintainable solution is a support-or-retire decision.
