## Summary

Keep open. Current main still has the cron/main-session relay-prose behavior this PR targets, so it is not obsolete, but the PR head is not safe to land as-is because it drops internal-only no-relay behavior, blocks operational edits to legacy rich main-session jobs, and is stale against newer exec-event prompt handling on main.

## What This Changes

The PR adds main-session cron systemEvent token validation/redaction, rewrites cron heartbeat prompts away from rich reminder prose, updates agent/tool guidance, and adds focused cron/heartbeat tests.

## Best Possible Solution

Adopt or rebase the remediation, but preserve current-main exec-event prompt hardening, keep no-relay cron wakes strictly internal/HEARTBEAT_OK-only, safely degrade rich legacy main-session jobs at execution while still allowing disable/repair edits, and update docs, tool guidance, and focused regression tests for the new main-session token contract.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: a main-session cron systemEvent is enqueued unchanged, resolveHeartbeatRunPrompt passes it into buildCronEventPrompt, and the resulting heartbeat prompt contains the rich reminder wrapper/relay prose. The PR-specific blockers are also directly reproducible from the PR head source without running tests.

## Solution Assessment

No. The direction is the right narrow product boundary, but this implementation is not the best fix as-is because it regresses no-relay behavior, legacy job operability, and current exec-event prompt handling. A safer fix keeps the token-only main-session cron path while preserving those existing invariants.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Restore internal-only no-relay cron wakes:** `src/infra/heartbeat-events-filter.ts:23`
  - body: `buildCronEventPrompt` now ignores `deliverToUser` and always asks for follow-up unless session context requires it. In no-target or alerts-disabled heartbeat runs, delivery is skipped but the model turn can still be generated and retained, so disabling relay no longer forces the internal-only/`HEARTBEAT_OK` behavior that avoids main-session residue.
  - confidence: 0.91
- **[P1] Preserve current exec event prompt handling:** `src/infra/heartbeat-events-filter.ts:27`
  - body: The PR head changes `buildExecEventPrompt` back to a one-argument helper that says the result is in system messages above. Current main passes the pending exec events into this helper and uses structured completion filtering/truncation, so landing this stale version would lose command-output delivery and missing-output safeguards.
  - confidence: 0.86
- **[P2] Allow operational edits to legacy main jobs:** `src/cron/service/jobs.ts:147`
  - body: `assertSupportedJobSpec` now rejects every main-session job whose stored text is rich reminder prose. Since patch application calls this shared assertion even for unrelated updates, existing legacy jobs can become impossible to disable or repair even though execution is designed to degrade unsafe text to the fallback token.
  - confidence: 0.88
