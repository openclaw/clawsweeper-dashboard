## What This Changes

The PR adds main-session cron systemEvent token validation/redaction, changes cron heartbeat prompts to a minimal wake-token format, updates agent cron guidance, and adds focused tests.

## Best Possible Solution

Land a rebased narrow cron hardening that tokenizes main-session cron text, keeps no-relay and exec response-tool invariants, lets legacy jobs be disabled or repaired, and updates docs/tool guidance.
