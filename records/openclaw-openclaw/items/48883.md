## Summary

Keep open: this targets a real cron/main-session prompt safety problem, but the branch still weakens no-relay cron semantics, is stale against current exec-event handling, blocks benign legacy-job edits, lacks docs alignment, and has only test proof.

## What This Changes

The PR adds main-session cron systemEvent token validation/redaction, changes cron heartbeat prompts to a minimal wake-token format, updates agent cron guidance, and adds focused tests.

## Best Possible Solution

Rebase or replace with a narrow cron/heartbeat hardening that uses token-only main-session cron text, preserves internal-only no-relay and current exec/heartbeat response-tool paths, allows legacy job disable/repair edits, and aligns public docs/tool guidance.

## Reproduction Assessment

Yes. Source inspection on current main shows main-session systemEvent text is accepted unchanged, passed into buildCronEventPrompt, and wrapped with scheduled-reminder relay prose; I did not run a live cron job because this was a read-only review.

## Solution Assessment

No. The direction is appropriate, but this PR is not the best fix as-is because it drops no-relay invariants, regresses current exec prompt handling, blocks legacy job edits, and lacks docs/proof.
