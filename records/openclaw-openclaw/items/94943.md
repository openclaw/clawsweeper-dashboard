## Summary

Keep open: current main and the latest release still ship the compact bundled gog skill, while upstream gogcli now exposes a much broader generated command surface. ClawHub is useful for an authoritative external skill, but it does not by itself resolve the stale file that OpenClaw still packages.

## What This Changes

The issue requests that OpenClaw stop shipping an outdated bundled gog skill and either sync it from official gogcli docs, route to live help, or add release automation.

## Best Possible Solution

OpenClaw should settle the ownership policy for the bundled gog baseline, then either keep a concise bundled skill that points to live gog help/schema and the official generated docs, or move the full authoritative skill to ClawHub with no stale core copy.

## Reproduction Assessment

Yes. Source inspection reproduces the stale scope: current main still advertises only six gog service families in skills/gog/SKILL.md while upstream gogcli's generated docs list many more service and command families.

## Solution Assessment

Unclear as a maintainer policy matter. A one-file skill refresh is mechanically possible, but the best durable fix depends on whether maintainers want OpenClaw core to own a synchronized bundled baseline or point users to ClawHub/upstream generated docs.
