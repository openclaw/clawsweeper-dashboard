## Summary

Keep open: current main and the latest release still ship a compact gog bundled skill that lags the official gogcli command surface, while the exact core-bundle versus ClawHub ownership choice needs maintainer direction after the reporter challenged the earlier ClawHub-only close path.

## What This Changes

The issue asks OpenClaw to refresh or source-sync the bundled `gog` SKILL.md so agents see current gogcli capabilities after OpenClaw updates.

## Best Possible Solution

Maintainers should decide the gog ownership boundary, then either keep a concise bundled baseline updated with official help/schema pointers and the current install metadata, or move authoritative command coverage to a gogcli-owned ClawHub/Git skill.

## Reproduction Assessment

Yes, from source: current main and `v2026.6.8` ship the 116-line bundled gog skill while upstream gogcli documents many more services and generated commands. I did not reproduce a live agent failure because the observable problem is stale skill guidance.

## Solution Assessment

Unclear for the preferred full generated sync: that may be too much core ownership for a fast-moving external CLI. A bounded bundled-skill refresh that points to `gog --help`, `gog schema`, and official docs looks more maintainable if maintainers want to keep the baseline skill in core.
