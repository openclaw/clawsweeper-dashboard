## Summary

Keep open: current main and the latest release still ship the compact bundled gog skill, while upstream gogcli now exposes a much larger generated command surface. The earlier ClawHub suggestion is useful for an authoritative external skill, but it does not by itself resolve the stale file that OpenClaw still packages.

## What This Changes

The issue asks OpenClaw to refresh or source-control-sync the bundled gog skill so agents see the current official gogcli command surface.

## Best Possible Solution

Keep the bundled gog skill accurate as a small baseline that points agents to live gogcli help/schema/generated docs, while any full generated command index lives in gogcli or an installable ClawHub/Git skill.

## Reproduction Assessment

Yes from source inspection: current main and v2026.6.8 ship a 4,574-byte gog quick reference for the older service set, while upstream gogcli generated docs list 680 pages and the missing command families. I did not run a live agent task because the mismatch is in bundled skill content and upstream docs.

## Solution Assessment

Unclear as a product decision: updating the bundled file is a plausible narrow repair, but making core own a generated gogcli command index conflicts with the ClawHub/plugin guidance. The best path is a maintainer-sponsored baseline update plus external source-of-truth coverage, or an explicit decision to keep the core bundle minimal.
