## Summary

Keep open: current main and v2026.6.9 still ship the compact bundled gog skill while upstream gogcli has a much broader generated command reference. Local/ClawHub overrides help individual users, but they do not settle whether OpenClaw core should continue owning an accurate bundled gog baseline.

## What This Changes

The issue asks OpenClaw to refresh or source-link the bundled gog skill so agents are not guided by a stale subset of gogcli commands.

## Best Possible Solution

Keep a small maintainer-owned bundled gog baseline only if it points agents to live `gog --help`, `gog schema`, and generated gogcli docs; keep the full fast-changing command index in gogcli or an installable ClawHub/Git skill.

## Solution Assessment

Unclear as a product decision. Updating the bundled file is a plausible narrow repair, but making core own the full generated gogcli command index conflicts with the ClawHub/plugin guidance unless maintainers sponsor that baseline.
