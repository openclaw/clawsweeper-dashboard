## Summary

Keep open: current `main` and v2026.6.8 still ship the compact bundled `gog` skill, while upstream gogcli exposes a much larger generated command surface. This is source-reproducible, but the right fix needs maintainer policy on whether the bundled skill stays a small baseline, is generated from gogcli, or moves authoritative coverage to ClawHub/Git installs.

## What This Changes

The issue asks OpenClaw to stop shipping a stale bundled `gog` skill by syncing, regenerating, or explicitly deferring to the authoritative gogcli command reference.

## Best Possible Solution

Keep the bundled gog skill as an accurate small baseline that points to live `gog` help/schema/generated docs, or move authoritative full coverage to a ClawHub/Git skill with an explicit maintainer-owned release policy for the bundled baseline.

## Reproduction Assessment

Yes. Current source and the latest release tag both ship a 116-line `skills/gog/SKILL.md` that only advertises Gmail, Calendar, Drive, Contacts, Sheets, and Docs, while upstream gogcli generated docs list 680 generated pages and the missing command families.

## Solution Assessment

Unclear as a product decision. Updating the bundled file is a plausible repair, but AGENTS.md and VISION.md make the durable owner boundary the key decision: compact bundled baseline, generated sync, or ClawHub/Git skill as source of truth.
