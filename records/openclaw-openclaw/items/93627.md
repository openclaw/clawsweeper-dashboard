- **[P1] Track emitted content across dreaming days:** `extensions/memory-core/src/dreaming-phases.ts:1254`
  - body: This skip only applies when the previous ingestion day equals the current one. On the next dreaming day, unchanged daily memory can be re-ingested, `recordShortTermRecalls` advances `lastRecalledAt`, and the light freshness filter can emit the same old work item again. The linked bug explicitly includes repeated output across nights, so the cutoff needs stable emitted/source state rather than day-scoped daily reinforcement.
  - confidence: 0.9
- **[P2] Seed strippedHash for upgraded daily checkpoints:** `extensions/memory-core/src/dreaming-phases.ts:1231`
  - body: Released daily-ingestion checkpoints do not contain `strippedHash`. This unchanged-file branch preserves `undefined` and skips reading the file, so after a managed Light Sleep rewrite changes the fingerprint, the next cycle cannot match a stripped hash and can falsely re-ingest the same content. Seed or compute the hash before relying on it, with an upgrade regression.
  - confidence: 0.82
- **[P2] Preserve user blank lines in stripped hash:** `extensions/memory-core/src/dreaming-phases.ts:1247`
  - body: The new hash drops every blank line before comparison. `buildDailySnippetChunks` records `startLine` and `endLine`, so inserting blank lines in user content can match the old hash, skip re-ingestion, and leave stale evidence locations. Hash the managed-block-stripped content without collapsing unrelated user spacing.
  - confidence: 0.86
