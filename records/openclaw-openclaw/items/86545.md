## Repair Work Prompt

Fix the isolated cron Discord announce SecretRef regression reported in https://github.com/openclaw/openclaw/issues/86545. Current source builds cfgWithAgentDefaults for isolated cron runs and passes that derived config into direct delivery; Discord token resolution only selects the active runtime snapshot when the input config matches the runtime source snapshot, so SecretRef-backed Discord tokens can remain configured_unavailable. Preserve fail-closed SecretRef semantics: do not fall back to DISCORD_BOT_TOKEN when a configured SecretRef is unavailable, do not add a new config option, and do not make EnvironmentFile the required product behavior. Prefer carrying or selecting the active runtime config/source relationship through the cron delivery path, or otherwise make the derived agent-default config still eligible for the active runtime snapshot. Add focused regression coverage for isolated cron announce delivery to Discord with a source config containing a SecretRef and a runtime snapshot containing the resolved token. Update CHANGELOG.md because this is a user-visible delivery/auth regression fix.
