## Summary

Keep open. Current main still matches the reported Slack directory gap: the public directory CLI list commands use config-backed directory methods only, Slack registers token-backed live readers separately, and Slack still inherits the default null self resolver.

## What This Changes

The issue asks OpenClaw to return Slack self, peers, and groups from live Slack directory reads when valid Slack bot/app/user tokens are configured.

## Best Possible Solution

Add a generic directory CLI path that prefers live list methods when available while preserving config-backed fallback, then add Slack-owned current-account identity support and focused regressions for peers, groups, and self.

## Reproduction Assessment

Yes. The issue provides exact CLI commands, and current-main source inspection shows those commands still use config-backed list methods and the default null self resolver instead of Slack's token-backed live directory methods.

## Solution Assessment

Yes. The narrow maintainable fix is a generic live-reader preference in the CLI plus Slack-owned self resolution; moving Slack token policy or Slack IDs into core would be broader than needed.
