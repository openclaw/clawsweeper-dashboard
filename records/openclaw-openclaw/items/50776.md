## Summary

Keep open. Current main still matches the reported Slack directory gap: the public directory CLI calls config-backed list methods, Slack registers token-backed live readers separately, and Slack inherits the default null self resolver. The related merged Slack user-token work in #28103 is adjacent, but it does not connect these CLI commands to live Slack directory reads or current-account identity.

## What This Changes

Slack directory commands should return workspace peers, groups, and current user when valid Slack bot/app/user tokens are configured.

## Best Possible Solution

Add a live-aware generic directory CLI path that prefers live list methods when present and preserves config-backed fallback, then add Slack-owned current-account identity support with focused regressions for peers, groups, and self.

## Reproduction Assessment

Yes. The issue gives exact CLI commands, and current-main source inspection shows those commands still call config-backed list methods and the default self resolver rather than Slack's token-backed live readers.

## Solution Assessment

Yes. The narrow maintainable solution is a generic CLI live-reader preference plus Slack-owned self resolution; moving Slack-specific token policy or IDs into core would be broader than needed.

## Repair Work Prompt

Fix the Slack directory CLI gap tracked by https://github.com/openclaw/openclaw/issues/50776. Current main routes `openclaw directory peers list` and `openclaw directory groups list` through config-backed `plugin.directory.listPeers`/`listGroups`, while Slack registers token-backed `listPeersLive`/`listGroupsLive` separately; `openclaw directory self --channel slack` also returns the adapter default because Slack has no explicit self resolver. Implement a generic directory CLI path that prefers live list methods when present and falls back to config-backed list methods. Add Slack-owned current-account identity resolution using the effective Slack user token or bot token without exposing raw token/profile data beyond `ChannelDirectoryEntry`. Keep core extension-agnostic: no Slack IDs, Slack token defaults, or Slack-only recovery policy in core. Add focused CLI and Slack tests for live preference, config fallback, peers, groups, and self. Add a user-facing `CHANGELOG.md` fix entry crediting @pjaillon; preserve existing #28103 behavior and do not change dependency versions.
