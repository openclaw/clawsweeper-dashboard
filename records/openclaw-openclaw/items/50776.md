## Summary

Keep open. Current main still matches the reported Slack directory gap: the CLI list commands use only config-backed directory methods, Slack exposes token-backed live readers separately, and Slack still falls through to the default null self resolver.

## What This Changes

Slack directory CLI commands should use valid Slack token-backed directory data for self, peers, and groups instead of returning empty or null results.

## Best Possible Solution

Make the directory CLI prefer live list readers when available with config fallback, then add Slack-owned current-account identity support and focused regressions for peers, groups, and self.

## Reproduction Assessment

Yes. The issue gives exact CLI commands, and current-main source inspection shows those commands still route to config-backed list methods and the default null self resolver instead of Slack's live token-backed readers.

## Solution Assessment

Yes. A generic live-aware CLI path plus Slack-owned self resolution is the narrow maintainable fix; moving Slack token policy or Slack IDs into core would be broader than needed.
