## Summary

Keep open. Current `main` still matches the reported Slack directory gap: the directory CLI reads config-backed list methods, Slack exposes token-backed live readers separately, and Slack still inherits the default null self resolver.

## What This Changes

Slack directory commands should return workspace peers, groups, and current user when valid Slack bot/app/user tokens are configured.

## Best Possible Solution

Add a live-aware generic directory CLI path that prefers live list methods when present and preserves config fallback, then add Slack-owned current-account identity support for `directory self`.

## Reproduction Assessment

Yes. The issue provides exact CLI commands, and current-main source inspection shows those commands still call config-backed list methods and the default null self resolver rather than Slack's token-backed live readers.

## Solution Assessment

Yes. The narrow maintainable fix is generic live-reader preference in the CLI plus Slack-owned self resolution; moving Slack token policy, Slack IDs, or provider-specific behavior into core would be broader than needed.

## Repair Work Prompt

Fix the Slack directory CLI gap tracked by https://github.com/openclaw/openclaw/issues/50776. Current main routes `openclaw directory peers list` and `openclaw directory groups list` through config-backed `plugin.directory.listPeers` / `listGroups`, while Slack registers token-backed `listPeersLive` / `listGroupsLive` separately; `openclaw directory self --channel slack` returns the adapter default because Slack has no explicit self resolver. Implement a generic directory CLI path that prefers live list methods when present and falls back to config-backed list methods. Add Slack-owned current-account identity resolution using the effective Slack read token or bot token, returning only the existing `ChannelDirectoryEntry` shape and not raw token/profile details. Keep core extension-agnostic: no Slack IDs, Slack token defaults, or Slack-only recovery policy in core. Preserve #28103 behavior and do not fold in the broader #42847 `userTokenReadOnly` policy discussion. Add focused CLI and Slack tests for live preference, config fallback, peers, groups, and self. Add a user-facing `CHANGELOG.md` fix entry crediting @pjaillon; do not change dependency versions.
