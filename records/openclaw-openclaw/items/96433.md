Keep open for normal maintainer review. The patch is a narrow and correct bug fix for video base64 limit rounding, with source-level reproduction and sufficient terminal proof; the only remaining blocker I saw is unrelated required CI failures in `extensions/lmstudio`.

## What This Changes

The PR changes video max-base64 limit resolution to use `estimateBase64Size(maxBytes)` and adds boundary regression tests.

## Best Possible Solution

Merge the focused helper fix with its regression test after required CI is green, keeping the raw-byte cap and global base64 cap semantics unchanged.

## Reproduction Assessment

Yes, at source level. Current main rounds the configured raw byte cap down while the caller rounds the actual payload size up to base64 quanta, so a raw size at a non-multiple-of-three cap can be rejected even though it fits `maxBytes`.

## Solution Assessment

Yes. Reusing `estimateBase64Size` in the resolver is the narrowest maintainable fix because it makes both sides of the comparison use the same base64 sizing primitive without adding config or changing the global cap.
