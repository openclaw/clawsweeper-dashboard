## Summary

Keep open. Current main has some fail-soft message-action discovery behavior, but the embedded Telegram prompt/reaction path still reads the run config and can reach strict non-env SecretRef token resolution before model output; the issue also has an open closing PR (#75445), so it should stay open until that PR or an equivalent fix lands.

## What This Changes

Embedded Telegram/Discord reply runs should not crash during prompt/tool discovery when channel credentials are configured with non-env SecretRefs.

## Best Possible Solution

Land #75445 or a replacement that makes optional Telegram/Discord discovery use inspect/fail-soft behavior or a resolved runtime snapshot while keeping real send/probe authentication strict.

## Reproduction Assessment

Yes. The issue gives concrete 2026.4.29 steps and sanitized logs, and current source still shows an embedded-run path from params.config into Telegram account/token resolution for prompt capabilities and reaction guidance.

## Solution Assessment

Unclear. Fail-soft optional discovery is the right boundary, but the complete fix must explicitly cover Telegram prompt capabilities, reaction guidance, and message-action discovery without hiding real transport auth failures.
