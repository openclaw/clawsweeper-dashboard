## Summary

Close. Current main now resolves embedded reply-run configs through the active runtime snapshot and scoped channel SecretRef targets before prompt/tool discovery, so Telegram/Discord discovery no longer receives the unresolved non-env SecretRef shape that caused the pre-model crash.

## What This Changes

Embedded Telegram/Discord reply runs crash before model output when channel credentials are configured as non-env SecretRefs in source config.

## Best Possible Solution

Keep the current main runtime/scoped SecretRef resolution path and treat the older channel-actions PR as superseded unless maintainers want extra coverage from it.

## Reproduction Assessment

No against current main. The original 2026.4.29 failure path is well described, but current source and regression tests show reply runs now receive a runtime/scoped resolved config before embedded prompt prep reads channel discovery hooks.

## Solution Assessment

Yes. Resolving active/scoped channel secrets before the embedded run reaches prompt/tool discovery fixes the raw-config leak while preserving strict token handling for real send and probe paths.
