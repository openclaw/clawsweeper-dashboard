## Summary

Keep open. ## What This Changes

Embedded Telegram/Discord reply-run discovery should not crash when channel credentials are backed by non-env SecretRefs.

## Best Possible Solution

Yes. The issue provides concrete 2026.4.29 steps and sanitized logs, and origin/main still shows a static Telegram path from embedded prompt prep into strict non-env SecretRef token resolution.

## Solution Assessment

Unclear. Fail-soft optional discovery is the right boundary, but the complete fix must cover Telegram message-tool capabilities, reaction guidance, and channel-action discovery without hiding real transport authentication failures.
