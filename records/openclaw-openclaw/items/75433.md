## Summary

Keep open. The report is current and source-backed: current `main` has partial Discord/message-action discovery hardening, but the embedded Telegram prompt capability and reaction-guidance path still passes `params.config` into hooks that can synchronously resolve a non-env SecretRef and throw before model output. An open unmerged closing PR, #75445, is already attached, so this issue should stay open until that PR or an equivalent broader fix lands.

## What This Changes

The issue requests that embedded Telegram/Discord reply prompt and action discovery stop crashing when channel credentials are configured through non-env SecretRefs that are resolved in the active gateway runtime snapshot.

## Best Possible Solution

Review #75445 and land it only if it covers the full failure surface, including Telegram prompt capabilities and reaction guidance; otherwise expand the fix so optional discovery uses the resolved runtime snapshot or fails soft without weakening strict credential handling for real send/probe paths.

## Reproduction Assessment

Yes. The issue gives concrete 2026.4.29 steps and sanitized logs, and current source still shows a static reproduction path from embedded prompt prep through Telegram prompt/reaction hooks into strict non-env SecretRef token resolution.

## Solution Assessment

Unclear. A channel-actions-only fix is not sufficient for the source-proven Telegram prompt/reaction hooks; the maintainable solution should cover all optional prompt/discovery paths while preserving strict runtime authentication failures on actual transport operations.
