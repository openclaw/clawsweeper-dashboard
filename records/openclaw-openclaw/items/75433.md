## Summary

Keep open. Current main still has a Telegram embedded prompt-prep path from raw run config into strict non-env SecretRef token resolution, and the open closing PR has not merged yet.

## What This Changes

The issue requests that embedded Telegram/Discord reply prompt and discovery paths stop crashing when channel credentials are backed by non-env SecretRefs.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/75445 with a fix that makes optional prompt/discovery paths use resolved runtime data or fail-soft inspection while preserving strict credential failures for real send/probe paths.

## Reproduction Assessment

Yes by source inspection and the reporter’s logs. Configure Telegram with a non-env SecretRef botToken and trigger an embedded reply run; current main still routes prompt capabilities and reaction guidance into strict token resolution before model output.

## Solution Assessment

Yes for the direction, but not yet completed on main. Optional prompt/discovery should degrade safely or use an active runtime snapshot; the linked PR is the current implementation path and should be reviewed or replaced before closing this issue.
