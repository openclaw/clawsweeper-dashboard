## Summary

Keep open. Current main partially protects message-action discovery, but embedded Telegram prompt-prep still passes raw run config into prompt capability and reaction guidance hooks that resolve strict non-env SecretRefs before model output, and #75445 is an open unmerged closing PR that should be reviewed before this issue closes.

## What This Changes

Embedded Telegram/Discord reply runs should not crash during optional prompt/tool discovery when channel credentials are backed by SecretRefs that are available in the active runtime snapshot.

## Best Possible Solution

Make optional embedded prompt/discovery paths use a resolved runtime snapshot or fail-soft read-only capability inspection, with regression coverage for Telegram prompt capabilities, Telegram reaction guidance, and Discord/Telegram message-action discovery.

## Reproduction Assessment

Yes, from source inspection rather than a live gateway run. Current embedded prompt prep passes raw params.config into Telegram prompt capability and reaction guidance hooks, which resolve accounts/tokens and can still throw on strict non-env SecretRefs before model output.

## Solution Assessment

Unclear. Fail-soft optional discovery or use of the resolved runtime snapshot is the right boundary, but the open PR needs coverage review for Telegram prompt capabilities and reaction guidance rather than only channel-actions discovery.
