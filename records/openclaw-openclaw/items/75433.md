## Summary

Keep open. Current main still has embedded prompt/reaction preparation paths that pass raw channel config into Telegram prompt hooks, where non-env SecretRefs can reach strict credential resolution before model output; the issue is also explicitly paired with open PR #75445, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

Embedded Telegram and Discord reply-run prompt/action discovery should not crash when channel credentials are configured as resolved runtime SecretRefs rather than plaintext/env config values.

## Best Possible Solution

Land a narrow fix that either uses the active resolved runtime config snapshot for embedded discovery or treats unresolved credential-only discovery as unavailable, while keeping real send/probe credential handling strict. Coverage should include Telegram prompt capabilities, Telegram reaction guidance, Discord/Telegram message-action discovery, channel-level credentials, and account-scoped credentials.

## Reproduction Assessment

Yes. The issue gives concrete setup steps and sanitized logs, and current main source still shows raw-config embedded prompt paths that can synchronously resolve Telegram credentials before model output.

## Solution Assessment

Unclear whether the open PR is sufficient without review. The right direction is a narrow discovery/prompt-boundary fix, not a broad weakening of SecretRef strictness or a startup-time migration.
