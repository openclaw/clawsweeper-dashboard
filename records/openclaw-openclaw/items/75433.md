## Summary

Keep open. The report is current, has an open unmerged closing PR, and current main still shows a Telegram embedded prompt-prep path that can read raw config and reach strict non-env SecretRef token resolution before model output.

## What This Changes

Embedded Telegram/Discord reply runs should not crash during optional prompt and tool discovery when channel credentials are non-env SecretRefs already resolved by the active runtime.

## Best Possible Solution

Land #75445 or an equivalent fix that makes optional Telegram prompt capabilities, reaction guidance, and message-action discovery use read-only inspection or a resolved runtime snapshot while keeping real send/probe authentication strict.

## Reproduction Assessment

Yes. The issue provides concrete 2026.4.29 steps and sanitized logs, and current source still routes embedded prompt prep through params.config into Telegram account/token resolution that throws for non-env SecretRefs.

## Solution Assessment

Unclear. Fail-soft optional discovery is the right boundary, but current main still lacks a guard on the Telegram prompt/reaction paths, so the linked PR needs coverage review before this can close.
