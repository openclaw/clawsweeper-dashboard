## Summary

Keep open. The report is current, source-backed, and has open closing PR #75445; current main still contains embedded prompt-prep paths that pass raw `params.config` into Telegram prompt/reaction helpers that can reach strict non-env SecretRef token resolution, so closing would be premature.

## What This Changes

Embedded Telegram and Discord reply runs should not fail before model output when channel credentials are backed by non-env SecretRefs that are resolved in the active gateway runtime snapshot.

## Best Possible Solution

Review #75445 and land it only if it covers the full raw-config failure surface, or adjust the fix so embedded prompt capabilities, reaction guidance, and message-action discovery either use the resolved runtime snapshot or fail soft only for optional discovery while preserving strict SecretRef behavior for real send/probe paths.

## Reproduction Assessment

Yes. The issue gives concrete 2026.4.29 steps and sanitized logs, and current main still shows a static source path from embedded prompt prep through Telegram prompt/reaction helpers into strict non-env SecretRef token resolution.

## Solution Assessment

Unclear. Making optional discovery tolerant is a maintainable direction, but the complete solution must cover both message-action discovery and the separate Telegram prompt capability/reaction guidance paths without hiding real runtime authentication failures.
