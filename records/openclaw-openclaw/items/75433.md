## Summary

Keep open. The report is current, source-backed, and paired with open closing PR #75445; target main still has the Telegram embedded prompt capability and reaction-guidance path passing raw config into strict non-env SecretRef token resolution, so cleanup closure would be premature.

## What This Changes

Embedded Telegram/Discord reply-run prompt and message-action discovery should not crash before model output when channel credentials are configured through non-env SecretRefs.

## Best Possible Solution

Preserve strict SecretRef behavior for real send/probe/auth paths, but make embedded prompt capability, reaction guidance, and message-action discovery use the resolved active runtime snapshot or fail soft when optional discovery sees unresolved non-env SecretRefs; add focused Telegram and Discord regression coverage for channel-level and account-level SecretRefs.

## Reproduction Assessment

Yes. The issue gives concrete 2026.4.29 steps and sanitized logs, and target main still has a static path from embedded prompt prep through Telegram prompt/reaction helpers into strict non-env SecretRef token resolution.

## Solution Assessment

Unclear. The current main implementation is not complete for the reported Telegram prompt/reaction path, and open PR #75445 should be reviewed for whether it covers more than message-action discovery before this can be considered solved.
