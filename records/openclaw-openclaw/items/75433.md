Keep open. Current main still has a Telegram embedded prompt-prep path that can synchronously reach strict token resolution from the run config, and this issue is already paired with open closing PR #75445 rather than merged implementation evidence.

## What This Changes

The issue requests that embedded Telegram/Discord reply prompt discovery use resolved runtime channel credentials or fail soft when raw non-env SecretRefs are unavailable for discovery.

## Best Possible Solution

Keep this issue open until #75445 or a successor lands and proves that embedded prompt capabilities, reaction guidance, and message-action discovery either receive resolved runtime config or treat unresolved discovery-only credentials as unavailable without aborting the agent run.
