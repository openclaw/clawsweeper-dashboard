## What This Changes

The issue reports that embedded Telegram/Discord reply runs crash before model output when raw non-env SecretRef channel credentials reach prompt or action discovery instead of the resolved runtime snapshot.

## Best Possible Solution

Review and land #75445 or an equivalent narrow fix that makes embedded prompt/action discovery use the resolved runtime config or tolerate unresolved credential-only discovery without weakening strict SecretRef handling for real send/probe paths, with focused Telegram and Discord regression tests.
