## Summary

Keep open. Current main still has a Telegram embedded prompt path that reads raw `params.config` and can reach strict non-env SecretRef token resolution before model output, and the linked closing PR #75445 remains open and unmerged.

## What This Changes

Embedded Telegram/Discord reply prompt and discovery paths should not crash when channel credentials are configured through non-env SecretRefs that are resolved by the active gateway runtime.

## Best Possible Solution

Review and land #75445 or an equivalent fix that makes optional Telegram/Discord discovery use read-only inspection or a resolved runtime snapshot while preserving strict auth failures for real send and probe paths.

## Reproduction Assessment

Yes. The issue provides concrete 2026.4.29 steps and sanitized logs, and current main still shows `params.config` flowing into Telegram prompt hooks that can synchronously resolve a non-env SecretRef in strict mode.

## Solution Assessment

Yes for the requested direction: fail-soft optional discovery or resolved runtime snapshots are narrower than weakening SecretRef resolution globally. It is still unclear whether the open PR fully covers Telegram `agentPrompt` hooks until it is reviewed.
