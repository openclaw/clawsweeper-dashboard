## Summary

Keep open. The report is current, source-backed, and explicitly paired with open PR #75445; current main still routes raw run config through Telegram prompt/reaction hooks that can reach strict non-env SecretRef token resolution before model output.

## What This Changes

Embedded Telegram/Discord reply runs should use resolved runtime credentials or degrade optional discovery gracefully when channel credentials are backed by non-env SecretRefs.

## Best Possible Solution

Review and land or adjust #75445, or an equivalent fix, so optional prompt capabilities, reaction guidance, and message-action discovery either use the resolved runtime snapshot or fail soft without weakening strict send/probe authentication failures.

## Reproduction Assessment

Yes. The issue gives concrete 2026.4.29 steps and sanitized logs, and current main still has a static source path from embedded prompt prep through Telegram prompt/reaction helpers into strict non-env SecretRef token resolution.

## Solution Assessment

Unclear until the open PR is reviewed. Making optional discovery tolerant is a maintainable direction, but the final fix needs to cover the prompt/reaction hooks as well as message-action discovery without masking real runtime auth failures.
