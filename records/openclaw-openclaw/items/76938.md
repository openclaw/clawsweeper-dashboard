## Summary

Keep open. Current main has materially improved compaction summaries and a default-on narrow post-compaction identical-tool guard, but the reported Telegram amnesia/looping behavior is not fully resolved: active-turn resume, delivery continuity, and broader saturated-session loop handling remain covered by open related reports.

## What This Changes

The issue reports that long Telegram direct/group sessions lose task and topic context, then loop or answer irrelevantly after compaction/truncation markers appear.

## Best Possible Solution

Define and land a shared compaction continuity contract that preserves pending user asks, current-turn identity, visible reply obligations, and loop-stop behavior across Telegram and other channels.

## Reproduction Assessment

No high-confidence exact Telegram reproduction is available from current main. Source inspection and related open reports show adjacent compaction resume, delivery, and loop paths still exist, but the issue lacks attached session logs and the exact marker was not found in repo source.

## Solution Assessment

Unclear as a single fix. The current summary quality guard and post-compaction loop guard are useful but too narrow; the maintainable path should be scoped with the open active-turn-loss, retry-identity, and saturated-session guard reports.
