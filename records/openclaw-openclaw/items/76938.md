## Summary

Keep open. Current main has compaction safeguards and retry plumbing, but the live Telegram report against v2026.5.2 is not proven fixed and I found no post-report compaction change that resolves the described context loss.

## What This Changes

The issue asks OpenClaw to preserve key decisions, active tasks, recent exchanges, language, and topic after long-session compaction instead of degrading into amnesia or repeated irrelevant replies.

## Best Possible Solution

Add a focused regression that replays a long Telegram direct/group transcript through compaction and verifies the compacted transcript or retry prompt preserves the current ask, language/topic, key decisions, and recent turns before tuning the narrow failing path.
