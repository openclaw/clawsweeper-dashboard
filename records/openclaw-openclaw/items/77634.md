## Best Possible Solution

Validate the Discord probe on Windows with Node 24, then avoid or defer nonessential startup `/users/@me` probes while preserving explicit deep status diagnostics and regression coverage.

## Reproduction Assessment

No high-confidence live reproduction is available from this review. Source inspection confirms the exact current-main `/users/@me` timeout path, but I did not reproduce Windows/Node 24 event-loop starvation with a real Discord bot token.

## Solution Assessment
