## Summary

## What This Changes

The PR changes `readBrowserVersion` in the browser plugin to run Chrome `--version` with a 10-second timeout instead of 2 seconds.

## Best Possible Solution

Preserve the public helper's fast default and apply the longer cold-cache budget explicitly to the doctor Chrome MCP readiness path, with focused coverage and redacted macOS terminal proof.

## Reproduction Assessment

Yes for source reproduction: current main uses a 2-second synchronous `--version` probe that returns null on timeout, and doctor turns null into the reported warning. I did not reproduce the macOS Gatekeeper cold-cache delay live.

## Solution Assessment

No as written: the doctor bug is real, but changing the SDK-exported helper's global timeout is broader than the reported failure. A doctor-specific or explicit opt-in timeout is the safer maintainable fix.
