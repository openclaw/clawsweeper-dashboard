## Label Justifications

- `P1`: The issue breaks a real Slack Agent/Assistant DM workflow by fragmenting conversation context across sibling sessions.
- `impact:session-state`: The central failure is session fan-out that splits one continuous Slack DM conversation into multiple assistant-thread sessions.
- `impact:message-loss`: Messages route away from the user's expected visible DM session, making context and replies effectively misdelivered.

## Summary

Keep open. Current main and the latest release still preserve Slack assistant DM threads as thread-scoped sessions, and the linked closing PR remains open rather than merged.

## What This Changes

The issue requests that Slack Agent/Assistant DM messages from the same user route into the existing DM session instead of spawning a new session for each assistant thread.

## Best Possible Solution

Review and land or replace the linked Slack routing fix with a maintainer-approved assistant-DM collapse contract, aligned tests, and docs, then close this issue from the merged PR.

## Reproduction Assessment

Yes. Source inspection shows current main passes Slack DM `assistantThreadTs` into session-key suffixing, and the adjacent test expects the thread-scoped result.

## Solution Assessment

Unclear as final product direction. Collapsing assistant-DM sessions addresses the root cause, but maintainers need to decide whether the behavior should be default, opt-in, or preserved.
