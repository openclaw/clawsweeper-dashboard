## Summary

Keep open. Current main has adjacent Gateway/session performance hardening, but the exact ACP task-maintenance close-failure loop and multi-minute event-loop saturation still lack source, test, release, or live-run proof of a fix.

## What This Changes

The issue reports a 2026.4.27 Gateway regression where ACP task maintenance repeatedly fails to close terminal or orphaned sessions, session write locks are held for minutes, and the Node event loop saturates while idle.

## Best Possible Solution

Root-cause the current-main ACP task-maintenance path and make cleanup bounded and recoverable with close-failure backoff/quarantine, bounded session-store work, preserved active bindings, regression tests, and a live idle Gateway validation.

## Reproduction Assessment

Yes for the originally reported bad build: the issue gives exact checkout/build/restart steps and a strong five-signal log fingerprint. Current-main reproduction is unclear because this read-only review did not run a live Gateway, but the matching maintenance paths and warning strings still exist.

## Solution Assessment

No patch is proposed, and closing is not the best way to solve it. The narrow maintainable next step is a targeted live reproduction or bisect followed by an ACP task-maintenance/session-store fix tied to the confirmed current-main root cause.
