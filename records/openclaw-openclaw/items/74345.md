## What This Changes

Investigate and fix a 2026.4.27 Gateway regression where ACP task-maintenance cleanup loops, session-write-lock holds, and event-loop saturation make transports and embedded model calls time out.

## Best Possible Solution

Root-cause the current-main ACP task-maintenance path and make cleanup bounded and recoverable: avoid unbounded synchronous session-store work in hot sweeps, add per-session close failure backoff or quarantine where appropriate, preserve active persistent bindings, and add regression coverage plus a live idle Gateway validation with stale/orphaned ACP metadata.
