## Summary

## What This Changes

The issue reports a 2026.4.27 gateway regression where ACP task-maintenance cleanup loops, session-write locks are held for minutes, embedded model calls time out, and Telegram/Discord transports stall at idle.

## Best Possible Solution

Make ACP task-maintenance cleanup bounded and idempotent: avoid reattempting the same failing ACP close every sweep without cooldown, cap or yield long cleanup work, keep session-store scans out of the hot maintenance path where practical, safely clear/quarantine stale metadata when the backend cannot close it, and preserve separate tracking for process-tree orphan cleanup if needed.

Current-main reproduction remains unclear because this read-only review did not start a live gateway service, but current source still exposes the reported maintenance path.

## Solution Assessment

No patch is proposed here. A narrow repair in ACP task-maintenance is the best next approach, but it should be validated against repeated close failures and large session-store scans before claiming the production idle stall is fixed.
