## What This Changes

The PR changes `/status` and status-summary context-window resolution to ignore stale persisted session `contextTokens` unless the persisted model metadata matches the resolved status model or fallback provenance.

## Best Possible Solution

Apply one consistent persisted-context trust rule across the status renderer and summary rows, preserve known fallback/runtime-alias behavior, add the missing no-runtime-metadata regression, and land only after targeted tests plus lint/type checks are clean.
