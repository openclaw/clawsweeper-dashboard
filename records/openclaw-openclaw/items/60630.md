Full review comments:

- **[P1] Preserve loopback browser-control fetches:** `extensions/browser/src/browser/client-fetch.ts:190-194`
  - body: `fetchBrowserJson` supports absolute browser-control URLs such as `http://127.0.0.1:...`, and current main passes `policy: { allowPrivateNetwork: true }` for that trusted control-plane fetch. This PR switches to `fetchWithSsrFGuard` without that policy, so loopback/private base URLs are blocked before the browser request is sent.
  - confidence: 0.93
- **[P2] Forward abort signals through guarded browser fetch:** `extensions/browser/src/browser/client-fetch.ts:190-191`
  - body: `fetchWithSsrFGuard` composes cancellation from its top-level `signal` option and then overwrites the fetch init signal when `timeoutMs` is set. Passing `ctrl.signal` only inside `init` means `fetchBrowserJson(..., { signal })` can stop cancelling promptly and wait for the timeout instead.
  - confidence: 0.91
- **[P2] Drop the retired Telegram SDK facade:** `src/plugin-sdk/telegram.ts:25-26`
  - body: Current main no longer has `src/plugin-sdk/telegram.ts` or an `openclaw/plugin-sdk/telegram` export; it only exposes narrower Telegram subpaths. Reintroducing the plugin-specific facade for tests revives a retired boundary and creates another source of truth for Telegram contract access.
  - confidence: 0.88
- **[P2] Use the routing facade for account IDs:** `extensions/telegram/src/setup-contract.ts:1`
  - body: `setup-contract.ts` is imported by Telegram runtime setup, but this change imports account-id helpers from the broad `openclaw/plugin-sdk/setup` barrel. Matrix uses `openclaw/plugin-sdk/routing` for the same helpers, which preserves the lighter startup and lazy-loading boundary.
  - confidence: 0.87
