## Summary

Keep open: current main still falls back from Gateway transport and timeout failures into embedded agent execution, and the requested gateway-only hard-fail diagnostic mode is not implemented. The open draft PR at https://github.com/openclaw/openclaw/pull/78507 targets the flag/env path but is not merged and still needs review/proof.

## What This Changes

Request louder gateway-to-embedded fallback diagnostics and an opt-out hard-fail mode for `openclaw agent` diagnostic and CI probes.

## Best Possible Solution

Land or replace the open draft PR with an additive gateway-only no-fallback mode, focused regression coverage, and public CLI/docs coverage, while deciding broader warning/prefixing behavior separately.

## Reproduction Assessment

Yes, at source level: current main catches Gateway transport and timeout failures and invokes embedded fallback, with focused tests asserting that behavior. I did not run a live Gateway probe in this read-only review, but recent issue comments provide live macOS and WSL2 before-fix examples.

## Solution Assessment

Yes for the additive no-fallback mode: it is the narrowest way to make diagnostic probes fail closed without breaking default fallback compatibility. The optional louder default warning or embedded stderr prefixing should remain a separate maintainer UX decision.
