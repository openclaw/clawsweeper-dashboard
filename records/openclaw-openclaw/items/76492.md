## Summary

Keep open: current main still automatically runs the embedded agent after Gateway transport failures or final-response timeouts, and there is still no hard-fail diagnostic mode for probes that must prove Gateway behavior.

## What This Changes

Add a loud, fail-closed diagnostic mode for `openclaw agent` so Gateway probes can opt out of embedded fallback.

## Best Possible Solution

Preserve the current default fallback, add an explicit Gateway-only hard-fail control with clearer fallback diagnostics, and split any deeper stale Gateway task cleanup into a focused lifecycle fix if needed.

## Reproduction Assessment

Yes, at source level: current main catches Gateway transport/timeouts in `agentCliCommand`, logs the fallback diagnostic, and invokes the embedded agent. I did not run a live Gateway/Codex probe in this read-only review.

## Solution Assessment

Yes, the narrow maintainable direction is an additive `--no-fallback` or equivalent hard-fail control while keeping default compatibility. Flipping the default fallback or prefixing all embedded output should be treated as separate product/logging choices.
