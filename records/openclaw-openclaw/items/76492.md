Keep open. Current `main` and latest release still fall back from Gateway transport or timeout failures into embedded execution, and the requested hard-fail diagnostic mode is not present; the linked draft PR is the current implementation candidate but still needs maintainer product review.

## What This Changes

Request a louder Gateway-to-embedded fallback diagnostic and an opt-in hard-fail mode for `openclaw agent` diagnostic or CI probes.

## Best Possible Solution

Review, land, or replace the focused diagnostic-mode PR while preserving the current interactive fallback default unless maintainers explicitly approve a broader behavior change.

## Reproduction Assessment

Yes, at source level: forcing a Gateway transport error or timeout makes `agentCliCommand` invoke the embedded agent and report fallback metadata/stderr diagnostics. The issue discussion also includes live before-fix auth-mismatch and timeout data, but this read-only review did not run a live Gateway repro.

## Solution Assessment

Unclear as a final product choice: an additive no-fallback diagnostic mode is a narrow plausible fix, but the flag/env contract and any exit-code or JSON-shape changes need maintainer approval. Changing the default fallback behavior would be riskier than preserving it and adding an explicit operator-selected hard-fail mode.
