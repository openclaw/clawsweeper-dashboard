## Summary

Keep open: current main still falls back from Gateway transport and timeout failures into embedded execution, the requested no-fallback diagnostic mode is not on main, and the open draft PR at https://github.com/openclaw/openclaw/pull/78507 remains the active fix path.

## What This Changes

Add louder diagnostics and a hard-fail no-fallback mode for `openclaw agent` so Gateway diagnostic probes are not masked by embedded fallback success.

## Best Possible Solution

Land or replace the open draft PR with an additive Gateway-only fail-closed mode, focused regression coverage, docs, and real diagnostic proof while preserving default fallback compatibility unless maintainers choose a broader UX change.

## Reproduction Assessment

Yes, source-reproducible: current main catches Gateway timeout and transport failures and runs embedded fallback, with focused tests asserting that behavior. The issue also has live WSL2 and macOS repro comments, but I did not run a live Gateway probe in this read-only review.

## Solution Assessment

Yes for the narrow direction: an additive no-fallback flag/env mode is the smallest maintainable diagnostic fix without changing the interactive fallback default. Broader warning copy, JSON shape, and stderr prefixing remain maintainer product decisions.
