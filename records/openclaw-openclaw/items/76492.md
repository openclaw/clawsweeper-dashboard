## Summary

Keep open. Current `main` and latest release still fall back to embedded execution for Gateway timeout/transport failures, the active `openclaw agent` command has no hard-fail diagnostic option, and the linked implementation PR is still open, draft, and conflicting rather than merged.

## What This Changes

The issue requests a louder and optionally disable-able embedded fallback path for `openclaw agent` Gateway diagnostic probes.

## Best Possible Solution

Keep embedded fallback as the default interactive behavior, and add a maintainer-approved hard-fail diagnostic mode for Gateway-only probes with docs and real failure-path proof.

## Reproduction Assessment

Yes, at source level: Gateway timeout and eligible transport errors route to embedded fallback in `agentCliCommand`, and tests assert that behavior. I did not run a live Gateway repro in this read-only review.

## Solution Assessment

Unclear as a final product contract: an additive no-fallback mode is the narrow likely solution, but the CLI flag/env shape needs maintainer approval because fallback behavior and env surfaces are compatibility-sensitive.
