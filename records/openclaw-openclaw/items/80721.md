## Summary

Keep open: current main still lacks `--dry-run` on `openclaw config unset`, while the sibling `set` and `patch` commands expose dry-run validation. Exact duplicate search did not find a canonical replacement issue or PR, and the implementation looks narrow enough for a focused config CLI repair.

## What This Changes

The issue requests a `--dry-run` option for `openclaw config unset` so operators can validate an unset operation without writing `openclaw.json`.

## Best Possible Solution

Add unset dry-run through the shared config mutation validation path, preserving current resolved-snapshot and `unsetPaths` semantics while documenting the new CLI surface.

## Reproduction Assessment

Yes. Source inspection on current main shows `config unset` has no `--dry-run` option and its handler has no no-write validation branch.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse or factor the existing delete-operation dry-run machinery rather than adding a separate validator that could drift from set/patch behavior.
