## Summary

Keep open: current main and v2026.6.10 still expose raw-first `web_fetch` with provider fallback only, while the requested provider-first/provider-only behavior is a new operator security/config policy that needs maintainer approval.

## What This Changes

Request to add an explicit `tools.web.fetch.mode` enum for raw-first, provider-first, and provider-only execution of canonical `web_fetch`.

## Best Possible Solution

Have maintainers decide the provider-routing policy first; if accepted, implement an opt-in enum that preserves raw-first by default, separates provider-first from provider-only, updates cache identity, docs, schema/help, and focused runtime/config tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config-policy request. Source inspection confirms the current state: no `tools.web.fetch.mode` exists, and provider execution is fallback-only after the raw fetch path or extraction failure.

## Solution Assessment

Unclear until maintainers confirm the product and security policy. If accepted, an enum is the right shape because it distinguishes call ordering from whether raw fallback is allowed while keeping the current default unchanged.
