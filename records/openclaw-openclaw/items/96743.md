## Summary

Keep open: current main and v2026.6.10 still expose raw-first `web_fetch` with provider fallback only, while the requested provider-first/provider-only mode is a new security-sensitive public config and provider-routing policy that needs maintainer direction.

## What This Changes

Request adds an opt-in `tools.web.fetch.mode` enum for raw-first, provider-first, and provider-only canonical `web_fetch` execution.

## Best Possible Solution

If maintainers accept the policy, implement an opt-in enum that preserves raw-first by default, includes mode in cache identity, updates schema/help/docs, and tests raw-first, provider-first, and provider-only behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config-policy request. Source inspection gives high-confidence current behavior: no `tools.web.fetch.mode` exists, and current main plus v2026.6.10 are raw-first with provider fallback.

## Solution Assessment

Unclear until maintainers confirm the product and security contract. If accepted, an enum is likely cleaner than booleans because it separates call order from raw-fallback policy while preserving the default.
