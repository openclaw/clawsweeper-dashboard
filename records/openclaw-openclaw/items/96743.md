## Summary

Keep open: current main and v2026.6.10 still use raw-first `web_fetch` with provider fallback only, while the requested provider-first/provider-only mode is a new security-sensitive config and routing policy that needs maintainer direction.

## What This Changes

Add an opt-in `tools.web.fetch.mode` setting so canonical `web_fetch` can run raw-first, provider-first, or provider-only.

## Best Possible Solution

Have maintainers decide the web_fetch egress policy first; if accepted, implement an opt-in enum that preserves raw-first by default and updates runtime ordering, cache keys, schema/help/docs, and focused config/runtime tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config-policy request. Source inspection gives high confidence about current behavior: no `tools.web.fetch.mode` exists and provider execution remains fallback-only.

## Solution Assessment

Unclear until maintainers confirm the product and security policy. If accepted, an enum is a maintainable shape because it separates ordering from whether raw fallback is allowed while preserving the current default.
