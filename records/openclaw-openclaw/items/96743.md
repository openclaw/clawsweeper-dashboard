## Summary

Keep open: current main still implements canonical `web_fetch` as raw-first with provider fallback only, while provider-first/provider-only execution would add a security-sensitive public config and provider-routing policy that needs maintainer direction.

## What This Changes

Add an explicit `tools.web.fetch.mode` option to choose raw-first, provider-first, or provider-only execution for canonical `web_fetch`.

## Best Possible Solution

If maintainers accept the direction, add an opt-in enum that preserves raw-first as the default, updates schema/help/docs/cache identity, and tests provider-first/provider-only ordering and failure behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config-policy request. Source inspection confirms current main has no mode field and provider execution remains fallback-only under documented branches.

## Solution Assessment

Unclear until maintainers confirm the product and security policy. If accepted, an enum is likely the narrowest maintainable shape because it separates provider ordering from whether raw fallback is allowed.
