## Summary

Keep open: current main and v2026.6.10 still only support raw-first `web_fetch` with provider fallback, while provider-first/provider-only is a new security-sensitive config policy needing maintainer direction.

## What This Changes

Request to add an explicit `tools.web.fetch.mode` enum so canonical `web_fetch` can run raw-first, provider-first, or provider-only while preserving the current default.

## Best Possible Solution

Have maintainers decide the provider-routing policy first; if accepted, implement an opt-in enum that preserves raw-first by default and updates runtime ordering, cache identity, schema/help, docs, and focused tests.

## Reproduction Assessment

Yes for the current-state source check, though this is not a bug repro. `runWebFetch` still takes the raw guarded HTTP path first, and the config schema/docs do not expose a provider-first or provider-only mode.

## Solution Assessment

Unclear until maintainers confirm the product and security policy. If accepted, an enum is the cleanest shape because it separates provider call order from whether raw fallback is allowed.
