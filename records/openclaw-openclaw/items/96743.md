## Summary

Keep open: current main and latest release still expose only raw-first `web_fetch` with provider fallback, while provider-first/provider-only execution would add a new security-sensitive public config policy that needs maintainer direction.

## What This Changes

Request adds an explicit `tools.web.fetch.mode` enum so canonical `web_fetch` can run raw-first, provider-first, or provider-only while preserving the current default.

## Best Possible Solution

Have maintainers decide the provider-routing policy first; if accepted, implement an opt-in enum that keeps raw-first as default and updates runtime ordering, cache identity, schema/help metadata, docs, and focused config/runtime tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config-policy request. Source inspection confirms the current state: no `tools.web.fetch.mode` exists, and provider execution is fallback-only after raw fetch failures or extraction fallback paths.

## Solution Assessment

Unclear until maintainers confirm the product and security policy. If accepted, an enum is a good shape because it separates provider call order from whether raw fallback is allowed while preserving the current default.
