## Summary

Keep open: current main and v2026.6.10 still only support raw-first `web_fetch` with provider fallback, and the requested provider-first/provider-only behavior is a new public config and egress-policy contract that needs maintainer product/security direction.

## What This Changes

Add an opt-in `tools.web.fetch.mode` enum so canonical `web_fetch` can run raw-first, provider-first, or provider-only.

## Best Possible Solution

Keep this issue open for maintainer product/security direction; if accepted, implement the opt-in enum across schema, runtime ordering, cache identity, docs, and focused tests while preserving raw-first as the default.

## Reproduction Assessment

Not applicable as a bug reproduction. Current main and v2026.6.10 source inspection confirm the requested `tools.web.fetch.mode` is absent and provider execution is fallback-only.

## Solution Assessment

Unclear until maintainers confirm the policy. If accepted, the proposed enum is likely the best shape because it separates provider-first ordering from provider-only fail-closed behavior while preserving the existing raw-first default.
