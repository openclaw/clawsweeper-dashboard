## Summary

Keep this PR open: it targets a real Discord guild reply regression, but the new scoped action map is incomplete for some allowlisted Discord admin actions and the external PR only provides unit-test proof, not real Discord behavior proof.

## What This Changes

The PR changes the shared message-tool schema builder to include optional property groups only when the active action list contains related message actions, adds a Discord regression test, and adds a changelog entry.

## Best Possible Solution

Land a corrected schema-group map that removes unrelated send fields while preserving every advertised action's consumed parameters, with focused allowlist regression tests and redacted live Discord guild proof.

## Reproduction Assessment

Yes, source-level: current main uses the full merged schema for any non-send-only action list, so a Discord guild action set can expose unrelated send fields. I did not run a live Discord guild reproduction in this read-only review.

## Solution Assessment

No, not yet. The scoping direction is the right fix shape, but the action-to-schema map needs to cover required params for allowlisted admin actions before this is mergeable.
