- **[P1] Pass includeBestEffort through scoped schemas:** `src/agents/tools/message-tool.ts:855`
  - body: `buildScopedMessageToolSchemaProps` narrows `options` to `includePresentation`, `includeDeliveryPin`, and `extraProperties`, then passes it to `buildSendSchema`, whose parameter type requires `includeBestEffort`.   - confidence: 0.95

  - body: The new channel-target group omits `emoji-list`, `emoji-upload`, `sticker-upload`, and `voice-status`, but the Discord handlers read `guildId`, `userId`, or `roleIds` for those actions. With a narrowed allowlist, the action remains advertised while required params that current main exposes disappear from the schema, making calls invalid or impossible for the model to form reliably.
  - confidence: 0.9
- **[P1] Keep name available for scoped renameGroup:** `src/agents/tools/message-tool.ts:826-835`
  - body: `renameGroup` is in the target group but not the channel-management group, so scoped schemas for iMessage/Microsoft Teams group rename actions lose the `name` field that current main exposes through `buildChannelManagementSchema`.
