## Summary

Keep open: this PR targets a real P1 isolated-cron cleanup gap, but the added flag is wired to each inner CLI prompt attempt. Current cron execution can retry after an interim acknowledgement, so this placement can close provider-native live-session state before the continuation prompt runs, and the provided proof still does not exercise the changed cron path.

## What This Changes

The branch adds `cleanupCliLiveSessionOnRunEnd` to isolated cron CLI `runCliAgent` requests and asserts that propagated flag in the session-key isolation test.

## Best Possible Solution

Move CLI live-session cleanup to a completion boundary that runs once after final cron execution, with focused CLI interim-retry coverage and redacted real cron process cleanup proof before merging.

## Solution Assessment

No, not as written: passing the existing cleanup flag is the right general direction, but the cleanup boundary should be the outer cron-run completion point or be proven safe for CLI interim retries. The submitted dummy-process terminal proof is not enough to clear the real behavior gate.
