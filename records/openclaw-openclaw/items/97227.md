## Summary

Keep open: the cleanup direction is valid, but the patch wires cleanup to each inner CLI prompt attempt, which can close the live Claude session before the existing interim-ack retry finishes; the submitted proof also still does not show after-fix cleanup from a real isolated cron run.

## What This Changes

## Best Possible Solution

Move CLI live-session cleanup to an outer cron-run completion boundary or otherwise prove it only fires after the final internal prompt attempt, then add CLI interim-retry coverage and redacted real cron cleanup proof before merge.

Yes at source level: current main and `v2026.6.10` omit the live-session cleanup flag from isolated cron CLI runs, and the PR's new line is inside a function that can be called twice for one cron execution. I did not run a live repeated-cron process-count reproduction in this read-only review.

## Solution Assessment
