## Summary

Keep this PR open: it is an active draft maintainer PR that expands the managed proxy contract, but current review found a Node-core HTTPS-proxy routing bug plus a remaining docs mismatch before merge.

## What This Changes

This PR adds HTTPS managed proxy URL support, `proxy.tls.caFile`/`--proxy-ca-file`, proxy CA propagation across validation/runtime/plugin paths, docs, generated config hashes, tests, and changelog.

## Best Possible Solution

Keep the scoped proxy-endpoint CA design, add proxy-host SNI for the managed Node `ProxyAgent`, correct the CA-trust enablement command, then finish targeted proxy validation and maintainer merge gates.

## Reproduction Assessment

Not applicable; this is a feature/config PR rather than a bug report. Current-main source and docs show managed proxy URLs are HTTP-only today, so the PR expands an existing contract.

## Solution Assessment

No, not as currently patched. The overall design is maintainable, but Node-core HTTP destinations through named HTTPS proxy endpoints need proxy SNI, and the CA-trust docs command still does not enable the feature it describes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Set SNI for HTTPS proxy agents:** `src/infra/net/proxy/proxy-lifecycle.ts:316-318`
  - body: For HTTPS managed proxy URLs, Node-core plain HTTP requests go through `proxy-agent` to `http-proxy-agent`, which calls `tls.connect()` with the constructor options and does not derive `servername` from the proxy host. Since this options object only passes `ca`, named enterprise proxies such as `proxy.corp.example` can fail TLS/SNI verification for `http://` destinations even though HTTPS destinations and APNs use paths that set SNI. Add `servername` from the proxy URL host when constructing the managed `ProxyAgent`.
  - confidence: 0.82
- **[P3] Show the CA config in the enablement command:** `docs/security/network-proxy.md:236-238`
  - body: This CA-trust section tells readers to prefer `proxy.tls.caFile`, but the bash command immediately below still configures an `http://` proxy URL and never sets `proxy.tls.caFile`. Readers following it will not enable the scoped HTTPS proxy CA path the section describes, so update the command example to set the HTTPS proxy URL and CA file.
  - confidence: 0.88
