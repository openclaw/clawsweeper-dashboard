## What This Changes

The PR adds HTTPS managed forward-proxy endpoints plus `proxy.tls.caFile` CA trust across config, CLI validation, runtime proxy routing, plugin fetch paths, docs, baselines, tests, and changelog.

## Best Possible Solution

Keep the scoped proxy-endpoint CA design, add proxy-host SNI to the managed Node `ProxyAgent` path, fix the CA-trust docs command, then undraft after the targeted proxy/docs gates and required CI are green.

## Reproduction Assessment

Not applicable; this is a feature/config PR rather than a bug report. Source and docs on current main show managed proxy URLs are HTTP-only today, so the PR intentionally expands the contract.

## Solution Assessment

No, not as currently patched. The overall direction is maintainable, but the current head needs proxy SNI for named HTTPS proxy endpoints and a docs command that actually enables the CA setting it describes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Set SNI for HTTPS proxy agents:** `src/infra/net/proxy/proxy-lifecycle.ts:316-318`
  - body: For HTTPS managed proxy URLs, Node-core plain HTTP requests go through `proxy-agent` to `http-proxy-agent`, which forwards the constructor options into `tls.connect()` without deriving `servername` from the proxy host. Since this options object only includes `ca`, named enterprise proxies can fail TLS/SNI verification for `http://` destinations even though HTTPS destinations and APNs use paths that set SNI. Add `servername` from the proxy URL host when constructing the managed `ProxyAgent`.
  - confidence: 0.86
- **[P3] Show the CA config in the enablement command:** `docs/security/network-proxy.md:236-238`
  - body: This CA-trust section tells readers to prefer `proxy.tls.caFile`, but the command immediately below still configures an `http://` proxy URL and never sets `proxy.tls.caFile`. Readers following it will not enable the scoped HTTPS proxy CA path the section describes, so update the command example to set the HTTPS proxy URL and CA file.
  - confidence: 0.9
