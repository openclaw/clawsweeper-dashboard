## Summary

Keep open: this is a draft MEMBER PR with the protected maintainer label, and current head still has two concrete blockers in the HTTPS proxy path and CA-trust docs.

## What This Changes

The PR adds HTTPS managed proxy endpoints plus scoped `proxy.tls.caFile` and `--proxy-ca-file` support across config, validation, Node/undici/APNs/plugin routing, docs, generated baselines, tests, and changelog.

## Best Possible Solution

Keep the scoped proxy-endpoint CA design, add proxy-host SNI to the managed Node `ProxyAgent` path, fix the CA-trust enablement command, then undraft after targeted proxy/docs checks and required CI pass.

## Reproduction Assessment

Not applicable for the feature request itself; current main source and docs show managed proxy URLs are HTTP-only today. The remaining SNI blocker is source-reproducible from PR head plus upstream `proxy-agent` and `http-proxy-agent` source.

## Solution Assessment

No, not as currently patched. The overall direction is maintainable, but the current head needs proxy-host SNI for named HTTPS proxy endpoints and docs that actually enable the CA setting being described.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Set SNI for HTTPS proxy agents:** `src/infra/net/proxy/proxy-lifecycle.ts:316-321`
  - body: For HTTPS managed proxy URLs, plain HTTP destinations go through `proxy-agent` to `http-proxy-agent`, which forwards these options into `tls.connect()` without deriving `servername` from the proxy host. Since this object only passes `ca`, named enterprise proxies can fail certificate/SNI verification; include `servername` from the proxy URL host when constructing the managed `ProxyAgent`.
  - confidence: 0.86
- **[P3] Show the CA config in the enablement command:** `docs/security/network-proxy.md:235-238`
  - body: This CA-trust section tells readers to prefer `proxy.tls.caFile`, but the command below still configures an `http://` proxy URL and never sets `proxy.tls.caFile`. Readers following it will not enable the scoped HTTPS proxy CA path, so update the command example to set the HTTPS proxy URL and CA file.
  - confidence: 0.9
