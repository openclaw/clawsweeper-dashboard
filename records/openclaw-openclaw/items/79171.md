## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Register inherited proxy TLS before resetting undici:** `src/infra/net/proxy/proxy-lifecycle.ts:533`
  - body: In inherited child processes, this loads `OPENCLAW_PROXY_CA_FILE` and passes it only into `bootstrapNodeHttpStack()`. `forceResetGlobalDispatcher()` gets proxy CA data from active managed proxy state via `addActiveManagedProxyTlsOptions()`, but this path never registers that state, so inherited fetch/undici clients still build an `EnvHttpProxyAgent` without the private CA and fail against HTTPS proxy endpoints. Register the inherited proxy URL/TLS state before the reset, or teach the undici helper to read the inherited CA, and add a child-routing regression.
  - confidence: 0.87
