## Summary

Keep open: this is a draft, maintainer-labeled feature/config PR and the latest head still has a Node-core HTTPS-proxy SNI gap plus one stale CA-trust docs command before merge.

## What This Changes

This PR adds HTTPS managed forward-proxy URLs, scoped proxy CA-file config/CLI support, runtime CA propagation across proxy validation, Node/undici routing, APNs tunnels, plugin fetch paths, docs, generated baselines, tests, and changelog.

## Best Possible Solution

Keep the scoped proxy-endpoint CA design, add proxy-host SNI to the managed Node ProxyAgent path, update the CA-trust enablement command, then undraft and run the targeted proxy/docs gates.

## Reproduction Assessment

Not applicable; this is a feature/config PR rather than a bug report. Current main source and docs show managed proxy URLs are HTTP-only today, so the PR expands the existing contract.

## Solution Assessment

No, not as currently patched. The design direction is maintainable, but the Node-core HTTP path through named HTTPS proxy endpoints needs proxy SNI and the CA-trust docs command needs to enable the CA setting it describes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Set SNI for HTTPS proxy agents:** `src/infra/net/proxy/proxy-lifecycle.ts:316-318`
  - body: For HTTPS managed proxy URLs, Node-core plain HTTP requests go through proxy-agent to http-proxy-agent, which calls tls.connect() with the constructor options and does not derive servername from the proxy host. Since this options object only passes ca, named enterprise proxies can fail TLS/SNI verification for http:// destinations even though HTTPS destinations and APNs use paths that set SNI. Add servername from the proxy URL host when constructing the managed ProxyAgent.
  - confidence: 0.84
- **[P3] Show the CA config in the enablement command:** `docs/security/network-proxy.md:236-238`
  - body: This CA-trust section tells readers to prefer proxy.tls.caFile, but the bash command immediately below still configures an http:// proxy URL and never sets proxy.tls.caFile. Readers following it will not enable the scoped HTTPS proxy CA path the section describes, so update the command example to set the HTTPS proxy URL and CA file.
  - confidence: 0.88
