## Summary

Keep open. Current main still lacks parser, status payload, UI, and docs support for a skill-provided configuration action; this is a core contract/product decision rather than something an external skill can fully solve by itself.

## What This Changes

Add optional SKILL.md metadata for a skill configuration URL/button and render that action in the Web Dashboard Skills list.

## Best Possible Solution

Define a documented skill configuration-action contract, validate/sanitize URLs, carry it through `skills.status`, render it in the Skills UI, and cover parser/status/UI/docs behavior with focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence verification that the requested parser, status payload, UI action, and docs contract are absent on current main.

## Solution Assessment

Unclear until maintainers choose the public contract shape. If the two requested fields are accepted, the narrow path is metadata parsing to `skills.status` to Control UI rendering; otherwise a generalized skill action model is safer.
