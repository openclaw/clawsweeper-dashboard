## Summary

Keep open. Current main still lacks end-to-end support for the requested skill configuration action, and this cannot be closed as ClawHub-only because external skills can host a UI but cannot make OpenClaw parse, transport, and render new dashboard metadata without a core contract.

## What This Changes

Request to add optional `configUrl` and `configButton` SKILL.md metadata fields and show a configuration button for matching skills in the Web Dashboard.

## Best Possible Solution

Define a small documented skill configuration-action contract, then parse validated metadata, include it in skill status/UI types, render a safe external action, and cover parser/status/UI/docs behavior with focused tests.

## Reproduction Assessment

Not applicable as a feature request. Static current-main inspection gives high-confidence verification that the requested parser, status payload, UI action, and docs contract are absent.

## Solution Assessment

Unclear until maintainers choose the public contract shape. If `configUrl`/`configButton` are accepted, extending the existing skill metadata to status to Control UI path is the narrowest maintainable implementation; otherwise a generalized skill action list is the safer alternative.
