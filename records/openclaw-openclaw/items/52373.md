## Summary

Keep this issue open. Current main still lacks the requested SKILL.md configuration-link contract end to end: no `configUrl`/`configButton` metadata fields, no status payload fields, no Control UI configuration action, and no docs entry; this is not closeable as ClawHub-only because external skills can host the page but cannot make OpenClaw parse, transport, and render new dashboard metadata without a core contract.

## What This Changes

Add optional `configUrl` and `configButton` SKILL.md frontmatter metadata so skills can expose a configuration entry point in the OpenClaw Web Dashboard Skills list.

## Best Possible Solution

Implement a small, documented, backwards-compatible core/dashboard contract: parse optional validated `metadata.openclaw.configUrl` and `configButton`, include them in skill status/UI types, render a safe external configuration action in the Skills list or detail view, and cover parser/status/UI/docs behavior with focused tests.

## Reproduction Assessment

Not applicable as a feature request, but there is high-confidence verification of the missing behavior through static current-main inspection: exact field search plus parser, status, UI, and docs checks all show the contract is absent.

## Solution Assessment

Yes, if maintainers accept the requested field names, the narrowest maintainable path is to extend the existing skill metadata/status/UI pipeline rather than adding skill-specific core behavior. The main design guardrail is URL safety and whether a generalized action list should be chosen instead before shipping a public contract.
