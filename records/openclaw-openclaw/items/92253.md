## Summary

This PR addresses a real usability gap, but the current implementation also converts an explicit empty target list into enable-all, which is a breaking configuration semantic and should block merge until repaired. The supplied runtime output confirms a healthy gateway and memory search, but it does not prove that omission selected multiple configured agents.

## What This Changes

The PR makes Active Memory derive its target-agent list from global configured agents when its plugin list is absent or empty, and adds normalization tests for default-all and explicit non-empty allowlists.

## Best Possible Solution

Default only an absent `agents` property to configured agent IDs, preserve an explicit empty array as no agents, document the three states, and prove agent-specific behavior after configuration refresh.

## Reproduction Assessment

Yes for the configuration behavior: source inspection deterministically shows both the current empty default and the proposed empty-to-all result. The report does not establish that default-all is an already-defined expected contract, so this is not a reproduced bug.

## Solution Assessment

No in its current form; deriving targets for an omitted property is plausible, but using normalized length conflates omission with an explicit empty allowlist and is not upgrade-safe.
