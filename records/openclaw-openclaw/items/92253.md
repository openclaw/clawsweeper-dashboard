## Summary

The Active Memory targeting inconsistency is valid, but this branch converts both omitted and explicitly empty allowlists into broad activation across every configured agent; that compatibility-sensitive default needs correction or explicit maintainer approval, and the supplied proof does not demonstrate the changed targeting on the final PR head.

## What This Changes

The PR defaults omitted or empty Active Memory agent allowlists to all configured agents, falls back to `main` without an agent list, and updates tests, manifest help, and documentation.

## Best Possible Solution

Distinguish omission from an explicitly empty allowlist: apply the maintainer-approved default only when `config.agents` is absent, preserve `[]` as targeting no agents, and add direct fresh-install and upgrade-path proof for the chosen omitted-field behavior.

## Reproduction Assessment

Yes. Current source deterministically normalizes omitted or empty targeting to `[]`, and both the command and prompt-hook gates reject agents not present in that list.

## Solution Assessment

No, not as written. Central normalization is the appropriate owner boundary, but collapsing an explicit empty allowlist into broad activation changes a user setting and is not required to fix omitted-field behavior.
