## What This Changes

## Best Possible Solution

Land only if maintainers approve omitted `config.agents` as the configured-agents default and final-head runtime proof covers omitted, explicit-empty, and explicit allowlist configs; otherwise revise toward a narrower default-main, setup, docs, or doctor repair.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.6.9` shows omitted Active Memory `config.agents` normalizes to `[]`, and both command and prompt-build gates reject agents unless the allowlist is explicit.

## Solution Assessment

Unclear as a final product decision. Central normalization is the clean layer if maintainers want omission to mean all configured agents, but a narrower default-main fallback or setup/doctor repair would solve the linked single-agent symptom with less upgrade blast radius.
