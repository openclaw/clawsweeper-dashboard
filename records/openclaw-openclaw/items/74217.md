## Summary

Keep open. Current main still sends heartbeat runs through the auto-reply session-usage persistence path without a run-kind guard, so the heartbeat provider/model can overwrite shared user-session runtime fields that status then displays.

## What This Changes

Prevent heartbeat or internal async-completion runs using `agents.defaults.heartbeat.model` from making a normal WhatsApp/direct session status look switched to that heartbeat model.

## Best Possible Solution

Thread run-kind-aware session accounting through the auto-reply persistence path so heartbeat/background turns preserve the prior user-session runtime model while normal turns and fallback changes still update it.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure a shared-session heartbeat model, run a heartbeat through the WhatsApp/direct auto-reply path, and inspect the session entry or `session_status`; current main still writes the heartbeat provider/model through `persistSessionUsageUpdate`.

## Solution Assessment

Yes. The narrow maintainable solution is to apply the existing preserve-runtime-model intent to auto-reply session accounting based on heartbeat/background run kind, rather than weakening status display globally.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/74217. Current main still lets heartbeat/async auto-reply runs persist their heartbeat `providerUsed`/`modelUsed` into the shared user session because `runReplyAgent` calls `persistRunSessionUsage` without run-kind information and `persistSessionUsageUpdate` writes `modelProvider`/`model` unconditionally. Thread `isHeartbeat` or a small run-kind/accounting option through `persistRunSessionUsage`/`persistSessionUsageUpdate` so heartbeat/background runs preserve the prior runtime `model`, `modelProvider`, and `contextTokens` while still recording safe usage/cost/system-prompt data as intended. Normal user turns, explicit model overrides, and real fallback transitions must continue to update runtime model fields. Add focused regression coverage for a shared WhatsApp/direct-style heartbeat using `agents.defaults.heartbeat.model` with an existing default runtime model, and update `CHANGELOG.md`. Do not change unrelated heartbeat delivery, silence, provider normalization, or global `session_status` selection semantics.
