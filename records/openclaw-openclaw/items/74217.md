## Summary

Keep open. Current main has docs/changelog/helper coverage claiming heartbeat runtime-model preservation, but the WhatsApp/gateway auto-reply path still resolves `heartbeat.model` into the active run, reports that provider/model in run metadata, and persists it through the generic session usage writer that status reads.

## What This Changes

Prevent heartbeat/internal async runs using `agents.defaults.heartbeat.model` from overwriting the user-facing session runtime model/status.

## Best Possible Solution

Thread heartbeat/background run kind into auto-reply session accounting so heartbeat model overrides never overwrite normal user-turn runtime model fields, while optionally recording separate labeled background diagnostics for status.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a shared-session `heartbeat.model`, run a heartbeat, then inspect the session entry/status; current main still persists the heartbeat run metadata through the generic auto-reply usage writer.

## Solution Assessment

Yes. The narrow maintainable fix is run-kind-aware persistence in the auto-reply session accounting path, preserving ordinary user/fallback runtime model updates while excluding heartbeat/background-only runs from the user-facing model fields.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/74217. In the gateway auto-reply path, heartbeat/internal background turns that use `agents.defaults.heartbeat.model` must not overwrite the user-facing session entry `model`, `modelProvider`, or context-window fields that `/status` and `session_status` use for normal user turns. Keep the existing behavior that ordinary user turns and real fallback/runtime model changes update those fields. Thread `isHeartbeat` or a broader run kind from `runReplyAgent`/`runAgentTurnWithFallback` into `persistRunSessionUsage`/`persistSessionUsageUpdate`, or add an equivalent generic seam, and preserve/skip runtime model fields for heartbeat/background-only runs. Do not change heartbeat model selection itself, fallback routing, explicit `/model` overrides, or the status preference for valid normal runtime models. Add a regression that exercises the auto-reply heartbeat path, not only `src/agents/command/session-store.ts`, and update `CHANGELOG.md` because the fix is user-facing.
