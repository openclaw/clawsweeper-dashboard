## Summary

Keep open. Current main still has a source-level path where a heartbeat run resolves `agents.defaults.heartbeat.model` and the auto-reply runner persists that run provider/model into the shared session entry that `session_status` displays.

## What This Changes

The issue asks for heartbeat/internal async runs using `agents.defaults.heartbeat.model` to stop overwriting the user-facing session runtime model/status.

## Best Possible Solution

Thread heartbeat-aware preservation through auto-reply session accounting so heartbeat turns can update usage/task metadata without replacing the shared session runtime provider/model/context window.

## Reproduction Assessment

Yes, at source level. A heartbeat with `agents.defaults.heartbeat.model` enters auto-reply with `isHeartbeat: true`, resolves the heartbeat provider/model, and then persists those values through `persistSessionUsageUpdate` without a heartbeat guard.

## Solution Assessment

Yes. The narrow maintainable solution is to carry the existing preserve-runtime intent into `persistRunSessionUsage`/`persistSessionUsageUpdate`, not to mask only the status display after session state has been polluted.

## Repair Work Prompt

Fix the heartbeat runtime-model bleed in auto-reply session accounting. A heartbeat/internal run using `agents.defaults.heartbeat.model` must not overwrite the shared user session's persisted `modelProvider`, `model`, or runtime context-window identity, while normal user turns must continue to update runtime model and usage state. Carry a heartbeat/preserve-runtime discriminator from `runReplyAgent` into `persistRunSessionUsage` and `persistSessionUsageUpdate`, or an equivalent narrow seam, and add regression coverage proving heartbeat usage persistence preserves the prior runtime model and that `session_status` still displays the preserved user-session model. Do not solve this by only changing `session_status`, and do not change unrelated heartbeat delivery, `HEARTBEAT_OK`, or channel-visible output behavior. Add a user-facing changelog fix entry crediting the reporter/contributor only if a human credit is known.
