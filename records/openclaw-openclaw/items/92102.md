## Summary

Keep open: current `main` and the latest release still expose the status-only path that can render indexed builtin vectors as `unknown`, and the linked open fix PR is the correct issue lifecycle owner until it merges or is rejected.

## What This Changes

The issue asks for plain `openclaw memory status` to report the vector store as ready when indexed chunks already exist, instead of showing unknown until sqlite-vec is initialized by a deeper probe.

## Best Possible Solution

Review and land the linked memory-core fix PR if its implementation holds up, then close this issue after the fix reaches `main`.

## Reproduction Assessment

Yes. Source inspection shows the status-only path can leave `this.vector.available` unset and the CLI then renders that as `Vector store: unknown`; I did not run the full reporter environment.

## Solution Assessment

Yes. The narrow maintainable fix is to infer builtin store readiness from existing indexed chunks while preserving the fast path that avoids sqlite-vec and embedding-provider probes; the linked PR targets that shape.
