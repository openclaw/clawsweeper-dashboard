## Summary

The bug remains present on current `main`: plain status returns an unprobed `storeAvailable` value even when indexed chunks exist. An open, mergeable PR explicitly closes this issue with a narrow guarded fallback and regression coverage, so repository policy requires keeping the issue open until that PR is reviewed and merged or closed.

## What This Changes

Make plain `openclaw memory status` report an indexed builtin vector store without forcing the sqlite-vec or embedding probes used by deep status.

## Best Possible Solution

Land a narrow memory-core change that reports persisted indexed-store readiness on plain status, keeps semantic/live extension readiness unprobed, and retains deep status as the authoritative runtime probe.

## Reproduction Assessment

Yes. Current `main` has a concrete source-level reproduction: create or reopen a status-purpose manager for a database with indexed chunks, skip the vector probe, and `status().vector.storeAvailable` remains undefined.

## Solution Assessment

Yes, with the linked PR’s guarded shape: reuse the existing indexed-chunk signal only for plain cached status, without initializing embeddings or replacing deep status as the live availability probe.
