## What This Changes

The issue reports that plain `openclaw memory status` can show `Vector store: unknown` for an already indexed builtin vector store because the status-only CLI path does not probe or infer sqlite-vec readiness.

## Best Possible Solution

Review and land one canonical memory-core fix that derives fast-path store readiness from persisted vector metadata without probing embeddings, then let the linked closing reference resolve this issue.

## Reproduction Assessment

Yes. Source inspection shows plain `memory status` skips vector probing, `status()` returns an undefined `storeAvailable` when the probe has not run, and the CLI renders that undefined state as `Vector store: unknown`.

## Solution Assessment

Yes. Keeping this issue open while the linked implementation PR is reviewed is the narrow maintainable path; a fix should use persisted vector metadata rather than loading sqlite-vec or embeddings on the fast path.
