## Reproduction Assessment

## Solution Assessment

Yes, the narrowest likely solution is to keep the plain status path cheap and infer only store readiness from existing indexed chunks, while leaving semantic vector readiness unprobed. The active fix path is the open linked PR rather than a new duplicate fix lane.
