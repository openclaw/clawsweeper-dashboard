## Summary

Close as superseded: current main still lacks the requested `agents.list[].memorySearch.qmd.command` field, but the same remaining capability is already centralized in the open per-agent memory/QMD override PR, which supports `agents.list[].memory.qmd.*` including `command`.

## What This Changes

Request to let each agent choose its own QMD executable or wrapper for memory search by adding `agents.list[].memorySearch.qmd.command`.

## Best Possible Solution

Keep one implementation track on https://github.com/openclaw/openclaw/pull/64647 or an equivalent `agents.list[].memory.qmd.*` overlay, rather than adding a second executable override under `memorySearch`.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection does confirm current main rejects the requested `memorySearch.qmd.command` path while the canonical PR tracks the per-agent QMD override surface.

## Solution Assessment

Yes for cleanup: closing this as superseded centralizes the remaining work on the open PR that already targets per-agent `memory.qmd.*` overrides. The maintainable solution should avoid creating two competing per-agent QMD command config paths.
