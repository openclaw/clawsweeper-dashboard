## Summary

Keep open: the reported Feishu/Lark DM behavior is source-reproducible on current main and is also present in the reported v2026.5.12 release, but the current code, docs, and tests explicitly encode the Codex harness direct-chat message-tool default, so changing it needs maintainer product judgment rather than cleanup closure.

## What This Changes

The issue requests that Codex/GPT-backed Feishu/Lark DMs keep using the visible streaming/final-reply path unless the user explicitly opts into message-tool-only delivery.

## Best Possible Solution

Make the Codex direct-chat delivery policy explicit for Feishu/Lark: either keep and document the existing message-tool default with diagnostics, or change the resolver/harness/channel policy and tests so Feishu/Lark DMs remain automatic unless configuration opts into tool-only delivery.

## Reproduction Assessment

Yes, source-level. Current main has a direct-chat Codex harness path where unset `messages.visibleReplies` resolves to `message_tool_only` and suppresses automatic final delivery; I did not run a live Feishu DM reproduction.

## Solution Assessment

Unclear. A code change is not the best next step until maintainers decide whether the Codex message-tool default should apply to Feishu/Lark DMs; after that, the fix shape is narrow in the harness/default resolver plus docs/tests.
