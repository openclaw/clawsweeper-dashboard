## Summary

Keep open: current main still source-reproduces the reported Feishu/Lark DM visible-reply suppression for Codex/GPT-backed normal turns, but the current docs and tests intentionally encode Codex direct chats as message-tool-first when `messages.visibleReplies` is unset, so this needs maintainer product judgment rather than cleanup closure.

## What This Changes

The issue requests restoring Feishu/Lark DM streaming and visible final replies for Codex/GPT-backed sessions when `messages.visibleReplies` is unset.

## Best Possible Solution

Make an explicit maintainer decision for external direct-chat Codex defaults, then implement either a Feishu/Lark compatibility exception or keep the current contract with clearer diagnostics and live Feishu proof.

## Reproduction Assessment

Yes at source level, but not by live Feishu run. A Codex-backed direct Feishu context with unset `messages.visibleReplies` inherits the harness `message_tool` default, resolves to `message_tool_only`, and suppresses normal final-text delivery, matching `queuedFinal=false, replies=0`.

## Solution Assessment

Unclear until maintainers decide the product boundary. The narrow code path is understood, but changing Codex direct-chat defaults or adding a Feishu/Lark exception is a policy decision rather than an automatic bug fix.
