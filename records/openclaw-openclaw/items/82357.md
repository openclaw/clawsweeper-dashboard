## Summary

Keep open: the report matches a real current policy path where Codex harness defaults can suppress automatic direct-chat final replies, but current docs and tests intentionally codify that default, so this needs maintainer compatibility/product judgment rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to keep Feishu/Lark DM replies from Codex/GPT-backed agents on the normal streaming visible-reply path unless the user explicitly opts into message-tool-only delivery.

## Best Possible Solution

Decide whether Codex-backed Feishu/Lark DMs should remain message-tool-only by default or regain automatic visible streaming when unset; then update the core policy/tests/docs together so operators get one clear contract.

## Reproduction Assessment

Yes for the core behavior: current source and tests show that an unset direct-chat Codex harness default resolves to `message_tool_only` and suppresses automatic final delivery. A live Feishu/Lark WebSocket run was not performed.

## Solution Assessment

Unclear. Restoring automatic Feishu DM delivery may be the right compatibility move, but current main intentionally documents and tests the opposite Codex direct-chat default, so this is not a narrow bug fix until maintainers choose the policy.
