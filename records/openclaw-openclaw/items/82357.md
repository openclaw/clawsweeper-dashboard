## Summary

Keep open: current main still makes unset Codex-backed direct chats inherit the Codex harness `message_tool` visible-reply default, which can suppress normal Feishu/Lark DM final replies and matches the reported `queuedFinal=false, replies=0` shape. This is not cleanup-closeable because the behavior is currently documented and tested, while the report asks for a Feishu DM compatibility policy decision.

## What This Changes

The issue asks OpenClaw to keep Feishu/Lark DM replies from Codex/GPT-backed agents on the visible streaming reply path unless message-tool-only behavior is explicitly configured.

## Best Possible Solution

Maintainers should decide whether Codex-backed Feishu/Lark DMs get a narrow direct-chat compatibility exception or keep the current tool-only default with clearer diagnostics, then require focused regression coverage and redacted live Feishu proof for the chosen path.

## Reproduction Assessment

Yes at source level: an unset direct chat on the Codex harness resolves to `message_tool_only`, suppresses automatic final delivery, and current tests assert the resulting no-final-reply path. I did not establish a live Feishu/Lark tenant reproduction in this read-only review.

## Solution Assessment

Unclear as an automated fix: forcing automatic visible replies for Codex-backed Feishu DMs would change the documented Codex direct-chat default. The safer next step is a maintainer policy decision before changing the delivery contract.
