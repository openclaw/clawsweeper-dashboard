## Summary

Keep open: current main and the reported shipped release intentionally default unset Codex direct-chat visible replies to the `message` tool, and source-level coverage shows final text is suppressed when the agent does not call that tool. The report is credible, but changing that Feishu DM/Codex default is a maintainer product decision rather than a conservative cleanup close or safe automated fix.

## What This Changes

The issue asks OpenClaw to keep Feishu/Lark DM replies on the automatic streaming visible-reply path for Codex/GPT-backed agents unless the user explicitly opts into message-tool-only delivery.

## Best Possible Solution

Maintainers should decide whether Codex unset direct chats should remain message-tool-only globally, be overridden for Feishu DMs, or receive stronger doctor/migration guidance, then update the harness/source-delivery policy, docs, and regression coverage together.

## Reproduction Assessment

Yes, at source level: current-main coverage shows a direct chat with a Codex harness default and unset `messages.visibleReplies` resolves to `message_tool_only`, returns `queuedFinal=false`, and does not call `sendFinalReply`. I did not live-reproduce it against a Feishu tenant.

## Solution Assessment

Unclear until maintainers choose the desired default. The local workaround is correct, but flipping the Codex default or adding a Feishu DM exception is a product/compatibility decision rather than a narrow bug fix.
