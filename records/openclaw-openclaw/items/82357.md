## Summary

Keep open: current main still routes unset Codex direct-chat visible replies through the message-tool-only policy that can suppress ordinary Feishu DM final replies, while the related Feishu PR only covers explicit text commands and intentionally leaves ordinary chat turns unchanged.

## What This Changes

The issue asks OpenClaw to restore Feishu/Lark DM visible streaming replies for Codex/GPT-backed ordinary turns when `messages.visibleReplies` is unset.

## Best Possible Solution

Maintainers should decide whether Codex-backed Feishu/Lark DMs should regain automatic visible streaming when unset or keep the documented message-tool-only default, then update policy, tests, docs, and Feishu proof together.

## Reproduction Assessment

Yes, at source level: current main resolves unset Codex direct-chat Feishu turns to `message_tool_only`, and the dispatch path suppresses ordinary final replies before Feishu logs `queuedFinal` and reply counts. A live Feishu/Lark WebSocket run was not performed here.

## Solution Assessment

Unclear. Restoring automatic Feishu DM delivery may be the right compatibility fix, but current docs and tests intentionally encode the opposite Codex direct-chat default, so maintainers need to choose the policy before implementation.
