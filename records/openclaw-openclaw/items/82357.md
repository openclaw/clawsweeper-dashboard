## Summary

Keep open: current main still has the reported behavior by design. Codex direct-chat defaults can suppress automatic Feishu/Lark final replies when `messages.visibleReplies` is unset, and changing that for Feishu/Codex DMs needs maintainer product judgment rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to restore automatic streaming/visible Feishu/Lark DM replies for Codex/GPT-backed agents unless the user explicitly opts into message-tool-only delivery.

## Best Possible Solution

Maintainers should decide whether external Feishu/Lark DMs should be exempt from Codex's documented message-tool direct-chat default, or keep the default and improve Feishu/operator guidance around `messages.visibleReplies: "automatic"`.

## Reproduction Assessment

Yes at source level, but not from a live Feishu account in this review. A Codex-backed direct chat with unset `messages.visibleReplies` resolves to `message_tool_only`, and ordinary final text without a message-tool send is suppressed, producing the reported `queuedFinal=false, replies=0` shape.

## Solution Assessment

Unclear. Restoring automatic replies for this surface may be right for Feishu DMs, but changing the Codex default or adding a Feishu exception is a maintainer privacy/UX decision rather than a narrow mechanical bug fix.
