## Summary

Keep open: current main still has the Codex direct-chat policy path that can suppress ordinary Feishu DM final replies when `messages.visibleReplies` is unset, and the remaining decision is a maintainer compatibility/product-policy choice rather than cleanup closure.

## What This Changes

The issue reports that Codex/GPT-backed Feishu/Lark DM replies stop using visible streaming replies unless `messages.visibleReplies` is set to `automatic`.

## Best Possible Solution

Maintainers should decide whether Codex-backed Feishu/Lark DMs should keep the documented message-tool-only default or regain automatic visible streaming when unset, then update policy, tests, docs, and Feishu proof together.

## Reproduction Assessment

Yes for the core behavior: current source and tests show that an unset direct-chat Codex harness default resolves to `message_tool_only` and suppresses ordinary automatic final delivery. A live Feishu/Lark WebSocket reproduction was not performed here.

## Solution Assessment

Unclear. Restoring automatic Feishu DM delivery may be the right compatibility fix, but current main intentionally documents and tests the opposite Codex direct-chat default, so maintainers need to choose the policy before implementation.
