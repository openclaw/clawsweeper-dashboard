## Summary

Keep open: current main still makes unset Codex-harness direct source replies use the message tool, which matches the Feishu/Lark DM `queuedFinal=false, replies=0` shape, but changing that behavior is a product/default-compatibility decision rather than conservative cleanup.

## What This Changes

Requests restoring Feishu/Lark DM Codex/GPT replies to the legacy streaming and visible final-reply path unless the operator explicitly configures message-tool-only delivery.

## Best Possible Solution

Keep the documented Codex direct-chat default unless maintainers approve a Feishu DM compatibility exception or migration; if approved, scope it narrowly and update tests plus operator docs.

## Reproduction Assessment

Yes, at source level: an unset `messages.visibleReplies` direct chat through the Codex harness selects `message_tool`, resolves to `message_tool_only`, and suppresses automatic final delivery. I did not run a live Feishu tenant reproduction.

## Solution Assessment

Unclear: the requested default change would address the reporter's Feishu DM workaround, but it conflicts with the current documented Codex direct-chat contract. The maintainable path is a maintainer decision on default compatibility, not an automatic bug-fix patch.
