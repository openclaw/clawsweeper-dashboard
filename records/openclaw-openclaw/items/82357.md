## Summary

Keep open: current `main` and the shipped 2026.5.12 code still make unset Codex-backed direct chats prefer `message_tool_only`, which can suppress Feishu/Lark DM final replies and match the reported `queuedFinal=false, replies=0` shape. This needs maintainer policy review rather than cleanup closure because the behavior is documented and tested as the Codex direct-chat default, while the report asks for Feishu DM compatibility with the old visible streaming path.

## What This Changes

The issue reports Feishu/Lark DM replies from Codex/GPT-backed agents bypassing visible streaming/final reply delivery unless `messages.visibleReplies` is set to `automatic`.

## Best Possible Solution

Make an explicit maintainer policy decision for Codex-backed Feishu/Lark DMs: either keep the tool-only default with clearer diagnostics/docs, or add a narrow Feishu/direct compatibility path with regression coverage and redacted live Feishu proof.

## Reproduction Assessment

Yes at source level: an unset direct chat on the Codex harness resolves to `message_tool_only`, suppresses automatic final delivery, and current tests assert the resulting `queuedFinal=false` path. I did not run a live Feishu/Lark tenant reproduction.

## Solution Assessment

Unclear as an automated fix: restoring automatic visible streaming for Codex-backed Feishu DMs would change or override the documented Codex direct-chat default. The narrow command PR is not enough for ordinary replies, so maintainers should decide the compatibility boundary first.
