## Summary

Keep this PR open. Current main still has the dispatch-side tool-availability gap that this PR is trying to fix, but the proposed patch is not merge-ready because its sandbox availability check does not use the same runtime policy session key as the embedded runner.

## What This Changes

The PR extends dispatch-time message-tool availability checks to include group, sandbox, and subagent tool policies and adds focused dispatch regression tests for group and sandbox policy removal.

## Best Possible Solution

Update this PR or a narrow replacement so dispatch computes message-tool availability from the same effective runtime policy inputs as the embedded runner, especially the derived runtime policy session key used for sandbox and subagent checks, then keep the focused group/sandbox tests and add coverage for the missed external direct-chat non-main sandbox shape.
