## Summary

Keep open: the branch is directionally useful, but it still resolves sandbox tool availability with the dispatch session key while embedded execution applies sandbox policy through the runtime policy key, so visible fallback can remain suppressed.

## What This Changes

The PR extends reply-dispatch message-tool availability checks to group, sandbox, and subagent tool policies and adds fallback regression tests.

## Best Possible Solution

Align dispatch sandbox availability with the same runtime policy session key used by embedded execution, keep the existing pre-resolved/group fallback behavior, add focused sandbox coverage and changelog, then validate the changed reply-policy surface.

## Reproduction Assessment

Yes. A direct external turn can dispatch through a main session while `resolveRuntimePolicySessionKey` derives a sandboxed non-main key; with sandbox `tools.allow` omitting `message`, the PR can still classify `message` as available because it checks the dispatch key.

## Solution Assessment

No as written. Adding effective policy inputs is the right direction, but sandbox availability should come from the runtime policy key or a shared effective-availability helper rather than `acpDispatchSessionKey`.
