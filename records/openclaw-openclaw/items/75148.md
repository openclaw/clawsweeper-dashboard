## Summary

The PR should remain open because current main still has a source-reproducible sandbox-policy mismatch, but this branch still checks sandbox availability with the dispatch session key instead of the runtime policy key, so it is not correct as-is.

## What This Changes

The PR adds group, sandbox, and subagent policy inputs to dispatch-time `message` tool availability and adds fallback regression tests in `dispatch-from-config.test.ts`.

## Best Possible Solution

Compute dispatch-time `message` availability from the same effective policy inputs as embedded execution, using the runtime policy session key for sandbox policy and keeping current main's group, subagent, and inherited-policy behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: dispatch can keep `message_tool_only` because current main omits sandbox policy, while embedded execution applies sandbox tool policy using `runtimePolicySessionKey`.

## Solution Assessment

No as written. Adding sandbox availability is the right direction, but the PR must use the runtime policy session key or a shared effective-availability helper instead of `acpDispatchSessionKey`.
