## Summary

Keep this PR open: the fallback bug is real and current main only partially covers it, while the PR branch still resolves sandbox availability against the dispatch session key instead of the runtime policy key used by embedded execution.

## What This Changes

The PR adds group, sandbox, and subagent policy inputs to dispatch-time `message` tool availability and adds group/sandbox fallback regression tests.

## Best Possible Solution

Compute dispatch-time `message` availability from the same effective policy inputs as embedded execution, using the runtime policy session key for sandbox policy while preserving current main's group, subagent, and inherited-policy handling.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: dispatch can keep `message_tool_only` because it omits sandbox policy on current main, while embedded execution applies sandbox `tools.allow` to the runtime policy session key and can remove `message`.

## Solution Assessment

No as written. Extending dispatch availability toward the real tool-policy pipeline is the right direction, but the PR must use the runtime policy session key or a shared effective-availability helper for sandbox policy.
