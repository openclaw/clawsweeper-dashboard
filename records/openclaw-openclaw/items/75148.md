## Summary

Keep open. The PR is aimed at a real remaining fallback gap, but the patch still checks sandbox availability against the dispatch session key instead of the runtime policy session key used by embedded execution, so one user-visible fallback path can remain suppressed.

## What This Changes

The PR adds group, sandbox, and subagent tool-policy inputs to dispatch-side `message` tool availability and adds group/sandbox fallback tests.

## Best Possible Solution

Compute dispatch-side `messageToolAvailable` from the same effective runtime policy key used by embedded runs, preserve the existing group/subagent handling, add focused sandbox coverage, and document the user-visible fix in the changelog.

## Reproduction Assessment

Yes. A direct external turn that shares `agent:main:main` can derive a sandboxed per-channel runtime policy key, while this PR’s sandbox check still uses the unsandboxed main dispatch key.

## Solution Assessment

No as written. Extending availability to sandbox policy is the right direction, but it must use `resolveRuntimePolicySessionKey` or a shared effective-availability helper rather than `acpDispatchSessionKey`.
