## What This Changes

The PR adds an installer no-op path that skips npm install when the detected local OpenClaw version matches the registry target, with `--force` and `OPENCLAW_FORCE_INSTALL` overrides.

## Best Possible Solution

Rebase and implement the optimization as an npm-provenance no-op: verify the npm-managed package or npm-global binary, skip only the npm install operation, preserve shared finalization, and add focused tests plus redacted real installer proof.

## Reproduction Assessment

Yes for the PR-head blockers: source inspection shows the skip trusts `resolve_openclaw_version()`, and a reviewer reports a fake-PATH reproduction. This is not a current-main bug reproduction because current main does not have the skip path.

## Solution Assessment

No. The safer path is to verify npm-managed provenance, skip only the npm install operation, and continue through the existing finalization path rather than returning from `main`.
