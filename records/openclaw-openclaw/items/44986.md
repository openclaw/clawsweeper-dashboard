## Summary

Keep open for maintainer review: current main still lacks the requested installer skip/force behavior, but this branch is not mergeable as-is because the skip trusts generic PATH-based version resolution, exits before installer finalization, and lacks contributor real behavior proof.

## What This Changes

The branch updates scripts/install.sh to add an npm-install no-op when the detected installed version matches the registry target, plus --force and OPENCLAW_FORCE_INSTALL overrides.

## Best Possible Solution

Implement the optimization only after verifying the npm-managed package or npm-global binary matches the target, skip only the npm install step, preserve finalization and --verify, and add focused installer coverage plus redacted real output.

## Reproduction Assessment

Yes for the PR-head blockers: source shows the skip uses a generic resolver that can prefer PATH, and a MEMBER review reproduced the false skip with a fake PATH binary. This is not a current-main bug reproduction because current main has no skip path.

## Solution Assessment

No. The current PR is not the best way to solve the request because it bases an npm-install no-op on generic version resolution and exits before the normal installer finalization path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Verify the npm-managed install before skipping:** `scripts/install.sh:1989-1990`
  - body: The skip compares the target registry version with resolve_openclaw_version(), which can return any openclaw earlier on PATH before checking npm's global package tree. A wrapper or different install with the same semver can make the installer report up to date even when the npm-managed package is absent or stale, so base the skip on the npm global package or a verified npm-global binary.
  - confidence: 0.93
- **[P2] Keep finalization after a skipped install:** `scripts/install.sh:2340-2341`
  - body: Returning from main when INSTALL_SKIPPED=1 bypasses finalization. The install step can no-op, but the installer should still run duplicate-install diagnostics, gateway refresh/restart handling, and explicit --verify behavior after a skipped install.
  - confidence: 0.88
