## What This Changes

The PR adds `--force` / `OPENCLAW_FORCE_INSTALL`, a `semver_equal` helper, and an npm-install skip path when the detected local OpenClaw version equals the resolved target version.

## Best Possible Solution

Rebase and implement the skip as an npm-provenance no-op that bypasses only the npm install step, preserves shared finalization, adds focused installer coverage, and includes redacted real installer proof.

## Reproduction Assessment

Yes for the PR-head blockers: source inspection shows the skip trusts `resolve_openclaw_version()`, and a MEMBER review reports a fake-PATH reproduction. This is not a current-main bug reproduction because current main does not have the skip path.

## Solution Assessment

No. The safer solution is to verify npm-managed provenance, skip only the npm install operation, and continue through the existing finalization path rather than returning from `main`.
