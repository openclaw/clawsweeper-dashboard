## Summary

Keep this PR open: it addresses the real installer no-op request from #44700, but the proposed skip can trust an unrelated `openclaw` earlier on `PATH` and incorrectly skip the npm-managed install.

## What This Changes

The PR adds an installer `--force`/`OPENCLAW_FORCE_INSTALL` override and skips npm install when the resolved target version matches the currently detected OpenClaw version.

## Best Possible Solution

Keep the optimization, but make the skip depend on the npm-managed OpenClaw package for the active npm environment, preserve installer finalization, and add focused shell tests plus docs/changelog coverage.

## Reproduction Assessment

Yes. The existing review comment gives a concrete fake-PATH reproduction, and current source confirms the resolver prefers `command -v openclaw` before checking the npm global package tree.

## Solution Assessment

No. Comparing the target version against the generic resolved CLI version is not the narrowest maintainable fix; the skip should verify the npm-managed package itself or that the resolved binary belongs to the npm global install being updated.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Verify the npm-managed package before skipping install:** `scripts/install.sh:1990`

A wrapper or different install with the same semver can therefore make the npm installer return "already up to date" even when the npm-managed `openclaw` package is absent or stale. Base this check on the npm global package/bin being updated instead.
