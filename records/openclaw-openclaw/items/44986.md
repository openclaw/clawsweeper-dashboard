## Summary

Keep open: current main still lacks the requested installer no-op optimization, and this PR is the canonical linked implementation, but the proposed patch is not merge-ready because it can falsely skip npm installation, bypasses finalization work, and lacks real behavior proof.

## What This Changes

The PR adds an installer version check that skips npm installation when the detected local OpenClaw version matches the target version, with `--force` and `OPENCLAW_FORCE_INSTALL` overrides.

## Best Possible Solution

Implement the optimization by verifying the npm-managed package or npm-global binary before skipping only the npm install step, preserve finalization and `--verify`, and add focused installer coverage plus redacted real installer output.

## Reproduction Assessment

Yes for the PR-head blockers: source shows the skip uses a generic resolver that can prefer PATH, and a MEMBER review reproduced the false skip with a fake PATH binary. This is not a current-main bug reproduction because current main has no skip path.

## Solution Assessment

No. The current PR is not the best solution because it bases an npm-install no-op on a generic CLI resolver and returns before finalization; the safer fix should prove the npm-managed install is current and keep post-install diagnostics running.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Verify the npm package before skipping install:** `scripts/install.sh:1989-1990`
  - body: The skip compares the target registry version with `resolve_openclaw_version()`, which can return any `openclaw` earlier on PATH before checking npm's global package tree. A wrapper or different install with the same semver can make the installer report "already up to date" even when the npm-managed package is absent or stale, so base the skip on the npm global package or a verified npm-global binary.
  - confidence: 0.93
- **[P2] Keep finalization on the skip path:** `scripts/install.sh:2340-2342`
  - body: Returning from `main` when `INSTALL_SKIPPED=1` bypasses finalization. The install step can no-op, but the installer should still run duplicate-install/PATH diagnostics, gateway refresh/restart handling, and explicit `--verify` behavior after a no-op install.
  - confidence: 0.88
