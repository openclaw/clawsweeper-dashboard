## Summary

Keep open. Current main still does not implement the requested installer no-op optimization, and this PR remains the canonical implementation thread, but the PR head still has a real false-skip correctness bug, bypasses finalization on skipped installs, lacks real behavior proof, and is merge-conflicting.

## What This Changes

The PR adds an npm install skip to `scripts/install.sh` when the detected installed version equals the resolved target version, plus `--force` and `OPENCLAW_FORCE_INSTALL` overrides.

## Best Possible Solution

Implement the optimization by checking the npm-managed package or npm-global binary identity, continue installer finalization and verify on no-op installs, then update installer docs and focused installer tests.

## Reproduction Assessment

Yes for the PR-head blocker: a MEMBER review reports a fake PATH `openclaw` with matching semver caused the branch to skip while the npm global root had no OpenClaw package, and the diff shows that generic resolver drives the skip. This is not a current-main bug reproduction.

## Solution Assessment

No. The current PR compares the target version with a generic CLI resolver and exits before finalization; the maintainable solution should prove the npm-managed install is current and keep the post-install checks running.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Verify the npm-managed package before skipping install:** `scripts/install.sh:1989-1990`
  - body: The skip path compares the target version with `resolve_openclaw_version()`, which can return any `openclaw` found earlier on PATH before checking npm's global package tree. A wrapper or different install with the same semver can make the installer return "already up to date" even when the npm-managed package is absent or stale, so base this skip on the npm global package or verified npm-global binary.
  - confidence: 0.93
- **[P2] Run finalization when skipping the npm install:** `scripts/install.sh:2340-2341`
  - body: Returning from `main` here skips the existing finalization path, including loaded gateway refresh and explicit `--verify` execution. The install step can no-op, but the installer should still run the post-install diagnostics and verification the user requested.
  - confidence: 0.87
