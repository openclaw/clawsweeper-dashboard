The PR adds a local-version skip to the npm path in `scripts/install.sh`, plus `--force` and `OPENCLAW_FORCE_INSTALL` overrides.

## Best Possible Solution

Rebase and implement the skip as an npm-provenance no-op that bypasses only the npm install command, preserves shared finalization, adds focused installer coverage, and includes redacted real installer proof.

## Reproduction Assessment

Yes for the PR-head blockers: source inspection shows the skip trusts PATH-based version resolution, and a reviewer reports a fake-PATH reproduction. This is not a current-main bug reproduction because current main does not have the skip path.

## Solution Assessment

No. The requested optimization is reasonable, but the current implementation should verify npm-managed provenance and preserve shared finalization instead of returning early from `main`.
