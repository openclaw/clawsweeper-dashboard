## Summary

Keep open: current main still lacks the requested installer skip/force behavior, and the linked feature request was closed in favor of this PR. The current patch should not merge as-is because the skip trusts a generic `openclaw` found on `PATH`, bypasses finalization on the skip path, and lacks external real behavior proof.

## What This Changes

The branch changes `scripts/install.sh` to add `--force`/`OPENCLAW_FORCE_INSTALL` and skip npm install when the detected local OpenClaw version equals the resolved target version.

## Best Possible Solution

Keep the optimization only if the skip verifies the npm-managed package or npm-global binary identity, continues finalization and verification behavior, documents the new flag/env surface, and adds focused installer regression coverage.

## Reproduction Assessment

Yes for the PR-introduced blocker: a maintainer review gives a concrete fake-PATH reproduction, and the diff shows the skip is driven by `resolve_openclaw_version`. This is PR-head behavior, not a current-main bug.

## Solution Assessment

No. Comparing the target version to a generic CLI resolver is not the narrowest maintainable npm install skip; the check should be based on the npm-managed package or a verified npm-global binary and should not bypass final verification/finalization.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Verify the npm-managed package before skipping install:** `scripts/install.sh:1987-1990`
  - body: The skip path compares the target version with `resolve_openclaw_version()`, which can return any `openclaw` found earlier on `PATH` before checking npm's global package tree. A wrapper or different install with the same semver can make the npm installer return "already up to date" even when the npm-managed package is absent or stale, so base this skip on the npm global package/bin being updated.
  - confidence: 0.93
- **[P2] Keep finalization on the skip path:** `scripts/install.sh:2340-2342`
  - body: Returning from `main` when `INSTALL_SKIPPED=1` bypasses finalization, including duplicate-install diagnostics, gateway refresh, footer links, and `--verify`. That makes `bash scripts/install.sh --verify` silently skip the requested verification when the version already matches, so the skip should continue through finalization instead of returning here.
  - confidence: 0.86
