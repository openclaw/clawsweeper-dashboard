## Summary

Keep this PR open, but do not merge it as-is. The optimization addresses a real redundant-install request, but the current patch can skip the official npm install based on an arbitrary PATH or OPENCLAW_BIN executable and still returns before normal installer finalization.

## What This Changes

The PR changes scripts/install.sh to add --force/OPENCLAW_FORCE_INSTALL and skip the npm install path when the resolved local version matches the target registry version.

## Best Possible Solution

Keep the no-op install optimization only if it is based on npm-managed package evidence, such as reading the global package.json or verifying the resolved binary belongs to the npm global bin, and let skipped installs continue through the normal finalization path with focused installer tests for fake PATH/no-package and skipped-finalization cases.

## Reproduction Assessment

Yes. The high-confidence reproduction is to put a fake same-version openclaw earlier on PATH while npm root -g has no openclaw package; the reviewer reported that this PR still skips, and the resolver order in current main explains why.

## Solution Assessment

No. The PR solves a real inefficiency, but the current approach trusts a general executable resolver for a package-integrity decision and skips more of the installer than the package download/install step.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Verify the npm package before skipping install:** `scripts/install.sh:1990`
  - body: The skip guard accepts resolve_openclaw_version(), which can come from OPENCLAW_BIN or any openclaw earlier on PATH before npm package metadata. A wrapper or stale binary that prints the target semver can set INSTALL_SKIPPED=1 even when the npm global package is missing, so the installer never repairs the official package.
  - confidence: 0.94
- **[P2] Keep finalization on skipped installs:** `scripts/install.sh:2340-2342`
  - body: Returning when INSTALL_SKIPPED=1 bypasses shared finalization, including duplicate-install diagnostics, gateway refresh, doctor/onboarding checks, and the final resolver path. Same-version reruns should skip only the npm install work and then continue through finalization.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The installer diff introduces a supply-chain/integrity concern because the no-op decision can trust an arbitrary PATH or OPENCLAW_BIN executable instead of the npm-managed package.

Concerns:

- **[medium] Skip can trust an arbitrary PATH binary:** `scripts/install.sh:1990`

For the official installer path, that can leave the npm package missing or unrepaired.
