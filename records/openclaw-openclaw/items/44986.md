## Summary

Keep open: the installer optimization is useful, but the current PR is not merge-ready because the skip trusts a PATH-resolved binary, exits before shared finalization, conflicts with current installer changes, and lacks real installer proof.

## What This Changes

The PR adds `--force`/`OPENCLAW_FORCE_INSTALL` and an npm install skip path when the detected local OpenClaw version equals the resolved target version.

## Best Possible Solution

Rebase and implement the optimization as an npm-provenance no-op that skips only the npm install operation while preserving shared finalization, focused installer coverage, and redacted real installer proof.

## Reproduction Assessment

Yes for the PR-head blockers: source inspection shows the skip trusts `resolve_openclaw_version()`, and a MEMBER review reports a fake-PATH reproduction. This is not a current-main bug reproduction because current main does not have the skip path.

## Solution Assessment

No. The safer solution is to verify npm-managed provenance, skip only the npm install operation, and continue through the existing finalization path rather than returning from `main`.
