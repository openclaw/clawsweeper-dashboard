## Summary

Keep open: the branch still has a concrete correctness blocker because the skip can be satisfied by a PATH-shadowed binary instead of the npm-managed package, and the external PR lacks after-fix real behavior proof.

## What This Changes

The branch adds an npm install skip in `scripts/install.sh` when the detected local version equals the target version, plus `--force` and `OPENCLAW_FORCE_INSTALL` overrides.

## Best Possible Solution

Keep the optimization only if the skip verifies the npm-managed package or npm-global bin identity, preserves installer finalization, updates public docs, and adds focused installer regression coverage.

## Reproduction Assessment

Yes for the PR-introduced blocker: maintainer review reports a fake PATH `openclaw` with matching semver caused the branch to skip while npm had no OpenClaw package, and the diff shows that generic resolver drives the skip. This is a PR-head reproduction, not a current-main bug.

## Solution Assessment

No. Comparing the target version with a generic CLI resolver is not the narrowest maintainable solution for an npm install skip; the check should be based on the npm-owned package or a verified npm-global binary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Verify the npm-managed package before skipping install:** `scripts/install.sh:1987-1990`
  - body: The skip path compares the target version with `resolve_openclaw_version()`, which can return any `openclaw` found earlier on PATH before checking npm's global package tree. A wrapper or different install with the same semver can make the npm installer return "already up to date" even when the npm-managed package is absent or stale, so base this skip on the npm global package/bin being updated.
  - confidence: 0.93
