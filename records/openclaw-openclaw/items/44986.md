## Summary

Keep open: the feature is useful, but the PR still has blocking installer correctness issues, lacks real behavior proof, and current main does not implement the skip path.

## What This Changes

The PR adds an `install.sh` npm-install skip when the detected local OpenClaw version matches the target, with `--force` and `OPENCLAW_FORCE_INSTALL` overrides.

## Best Possible Solution

Implement the optimization by verifying the npm-managed package or npm-global binary matches the target, skip only the npm install step, preserve finalization and `--verify`, and add focused installer coverage plus redacted real output.

## Reproduction Assessment

Yes for the PR-head blockers: source shows the skip uses a generic resolver that can prefer PATH, and a MEMBER review reproduced the false skip with a fake PATH binary. This is not a current-main bug reproduction because current main has no skip path.

## Solution Assessment

No. The current PR is not the best solution because it bases an npm-install no-op on generic version resolution and exits before finalization; the safer fix should prove npm-managed provenance and keep post-install diagnostics running.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Verify the npm-managed install before skipping:** `scripts/install.sh:1989-1990`
  - body: The skip compares the target registry version with `resolve_openclaw_version()`, which can return any `openclaw` earlier on PATH before checking npm's global package tree. A wrapper or different install with the same semver can make the installer report up to date even when the npm-managed package is absent or stale, so base the skip on the npm global package or a verified npm-global binary.
  - confidence: 0.93
- **[P2] Keep finalization after a skipped install:** `scripts/install.sh:2340-2342`
  - body: Returning from `main` when `INSTALL_SKIPPED=1` bypasses finalization. The install step can no-op, but the installer should still run duplicate-install diagnostics, gateway refresh/restart handling, and explicit `--verify` behavior after a skipped install.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: Needs attention: the installer skip weakens package provenance by trusting a PATH-resolved binary before confirming the npm-managed package is current.

  - body: The skip path can accept any `openclaw` earlier on PATH with a matching version before checking the npm global package, so a shadow wrapper can prevent reinstalling the intended npm package while reporting success.
