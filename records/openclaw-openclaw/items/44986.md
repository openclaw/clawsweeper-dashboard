## Summary

Keep open: the requested installer optimization is not on current main, but this PR is not merge-ready because the skip path trusts PATH-resolved version output for npm install provenance, exits before shared finalization, is currently conflicting, and lacks real behavior proof.

## What This Changes

This PR adds an `install.sh` version check plus `--force`/`OPENCLAW_FORCE_INSTALL` to skip npm installs when the detected local OpenClaw version matches the target.

## Best Possible Solution

Rebase and implement the skip as an npm-provenance check that bypasses only the npm install step, preserves finalization, adds focused installer coverage, and includes redacted real installer proof.

## Reproduction Assessment

Yes for the PR-head blockers: source inspection shows the skip trusts PATH-based version resolution, and a MEMBER review reports a fake-PATH reproduction. This is not a current-main bug reproduction because current main does not have the skip path.

## Solution Assessment

No. The safer solution is to verify npm-managed provenance, skip only the npm install step, and keep the existing finalization path rather than returning from `main`.
