## Summary

Keep open: the installer optimization is not on current main, and the PR still needs rework because the skip path trusts a PATH-resolved binary, exits before shared finalization, lacks real behavior proof, and currently conflicts with main.

## What This Changes

The PR adds `--force`/`OPENCLAW_FORCE_INSTALL` and a `scripts/install.sh` skip path that avoids npm install when the detected local version equals the resolved registry version.

## Best Possible Solution

Rebase and implement the optimization as an npm-provenance no-op: verify the npm-managed package or npm-global binary, skip only the npm install operation, preserve shared finalization, and add focused tests plus redacted real installer proof.

## Reproduction Assessment

Yes for the PR-head blockers: source inspection shows the skip trusts `resolve_openclaw_version()`, and a MEMBER review reports a fake-PATH reproduction. This is not a current-main bug reproduction because current main does not have the skip path.

## Solution Assessment

No. The safer solution is to verify npm-managed provenance, skip only the npm install operation, and continue through the existing finalization path rather than returning from `main`.
