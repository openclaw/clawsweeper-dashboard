## What This Changes

The PR adds `--force` / `OPENCLAW_FORCE_INSTALL` and skips the npm install path in `scripts/install.sh` when the detected local OpenClaw version matches the resolved target version.

## Best Possible Solution

Rebase and implement the skip as an npm-provenance no-op that bypasses only the npm install step, preserves finalization, adds focused installer coverage, and includes redacted real installer proof.

## Reproduction Assessment

Yes for the PR-head blockers: source inspection shows the skip trusts PATH-based version resolution, and a MEMBER review reports a fake-PATH reproduction. This is not a current-main bug reproduction because current main does not have the skip path.

## Solution Assessment

No. The safer solution is to verify npm-managed provenance, skip only the npm install step, and keep the existing finalization path rather than returning from `main`.
