## What This Changes

The PR hides daemon runtime last-exit status and reason details from status text when the service runtime reports `status: "running"`.

## Best Possible Solution

Refresh the formatter-level guard on current main so running services suppress stale last-exit details while non-running services keep signal labels and diagnostics, with regression coverage and a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: systemd runtime can return `status: "running"` with `ExecMainStatus`/`ExecMainCode`, and current status formatting prints those fields into the CLI output.

## Solution Assessment

No, not as submitted. Formatter-level suppression is the right boundary, but the branch needs to be refreshed around current `formatLastExitStatus(...)` behavior and should add focused regression coverage plus the required changelog entry.
