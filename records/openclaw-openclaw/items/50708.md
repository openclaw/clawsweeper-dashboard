## What This Changes

The PR changes daemon runtime status formatting to omit last exit status and reason whenever the runtime status is `running`.

## Best Possible Solution

Land a refreshed formatter-level fix that hides stale systemd exit details only from running human status output while preserving raw runtime fields, stopped/failed diagnostics, focused coverage, and live systemd proof.

## Reproduction Assessment

Yes, at source level: current main reads systemd active state plus exit fields and the shared formatter prints those fields whenever present. I did not establish a live Linux/systemd restart reproduction in this read-only review.

## Solution Assessment

Partly yes: formatter-level suppression is the right boundary because diagnostics can still access raw runtime fields. The branch still needs to preserve `formatLastExitStatus` for non-running services and provide real systemd proof.
