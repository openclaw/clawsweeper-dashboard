## What This Changes

This PR suppresses daemon runtime `lastExitStatus` and `lastExitReason` details in human-readable status output when the service runtime is running.

## Best Possible Solution

Land a current-main formatter-level guard that hides stale last-exit status and reason only for running services while preserving raw runtime data, stopped/failed diagnostics, signal labels, regression coverage, and the changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: systemd can report `status: "running"` with retained `ExecMainStatus`/`ExecMainCode`, and current status formatting prints those fields.

## Solution Assessment

No, not as submitted. Formatter-level suppression is the right boundary, but the branch needs to be refreshed around current `formatLastExitStatus(...)` behavior and paired with regression coverage plus the required changelog entry.
