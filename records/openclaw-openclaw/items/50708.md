## What This Changes

The PR changes daemon runtime status formatting to hide last-exit status and reason when the service runtime status is running.

## Best Possible Solution

Land a formatter-level guard that hides last-exit details only for running services while preserving raw systemd runtime fields, non-running diagnostics, signal labels, regression coverage, and the changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `readSystemdServiceRuntime` can return `status: "running"` with stale `lastExitStatus`/`lastExitReason`, and `formatRuntimeStatus` currently prints those fields unconditionally.

## Solution Assessment

No, not as submitted. Formatter-level suppression is the right boundary, but the branch should be rebased around current `formatLastExitStatus(...)` and include the required regression coverage and changelog entry.
