## What This Changes

The PR updates `src/daemon/runtime-format.ts` so formatted runtime output omits `lastExitStatus` and `lastExitReason` when `runtime.status` is `running`.

## Best Possible Solution

Refresh the PR to preserve `formatLastExitStatus`, add focused formatter/systemd coverage, and attach live Linux systemd proof showing running status omits stale exit details while stopped/failed output still reports diagnostics.

## Reproduction Assessment

Yes, at source level: current main reads systemd active state plus `ExecMainStatus`/`ExecMainCode` and the shared formatter prints those exit fields whenever present. I did not establish a live Linux/systemd restart reproduction in this read-only review.

## Solution Assessment

Partly yes: formatter-level suppression is the right boundary because it leaves raw runtime diagnostics available to other flows. The branch still needs to preserve non-running signal-aware formatting and provide live systemd proof.
