## Summary

Keep open. Current main still has the stale systemd-status rendering path, but this external PR is not merge-ready because its hunk predates the current signal-label formatter and the PR body explicitly lacks live Linux systemd proof.

## What This Changes

The PR adds a running-state guard in daemon runtime status formatting so `openclaw status` omits stale systemd last-exit details while a service is running.

## Best Possible Solution

Refresh the formatter on current main so running systemd status hides stale last-exit fields while stopped/failed services keep `formatLastExitStatus(...)` diagnostics, with focused formatter coverage and live systemd proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `readSystemdServiceRuntime` can return `status: "running"` with parsed `ExecMainStatus` and `ExecMainCode`, and `formatRuntimeStatus` currently prints those fields into daemon status output.

## Solution Assessment

No, not as submitted. Formatter-level suppression is the right boundary, but the branch should guard the existing `formatLastExitStatus(...)` helper call and add focused coverage plus real systemd proof.
