## Summary

Keep open: the linked bug is still present on current main and the PR targets the right user-facing status formatter, but the branch is stale/conflicting, regresses signal-aware last-exit formatting for stopped/failed services, and still lacks required live systemd proof.

## What This Changes

The PR changes `formatRuntimeStatus` to suppress `lastExitStatus` and `lastExitReason` whenever the service runtime status is `running`.

## Best Possible Solution

Land a refreshed formatter-level fix that hides stale systemd exit details only from running human status output while preserving raw runtime fields, stopped/failed signal-aware diagnostics, focused coverage, and live systemd proof.

## Reproduction Assessment

Yes, at source level: current main reads systemd active state plus exit fields and the shared formatter prints those fields whenever present. I did not run a live Linux/systemd restart in this read-only review.

## Solution Assessment

Partly yes: formatter-level suppression is the right boundary because raw runtime diagnostics remain available. The current PR still needs to preserve `formatLastExitStatus` for non-running services and provide live systemd proof.
