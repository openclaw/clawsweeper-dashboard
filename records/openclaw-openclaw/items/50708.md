## Summary

Keep open: the stale systemd exit-status problem is still present on current main, and this PR targets the right user-visible bug, but the submitted patch is stale against the current formatter and still needs a narrow refresh plus regression coverage and changelog.

## What This Changes

This PR changes daemon runtime status formatting so `openclaw status` hides last-exit status and reason details when the service runtime is running.

## Best Possible Solution

Land a formatter-level guard that hides last-exit status and reason only for running services while preserving raw runtime fields, non-running diagnostics, signal labels, regression coverage, and the changelog entry.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: systemd runtime can return `status: "running"` with `ExecMainStatus`/`ExecMainCode`, and current `formatRuntimeStatus` prints those fields into `openclaw status`. I did not run a live systemd restart scenario in this read-only review.

## Solution Assessment

No, not as submitted. Formatter-level suppression is the right boundary, but the branch should be refreshed around current `formatLastExitStatus(...)` behavior and include regression coverage plus the required changelog entry.
