## What This Changes

The PR changes daemon runtime status formatting so running services omit `lastExitStatus` and `lastExitReason` from the visible status line.

## Best Possible Solution

Apply the running-status suppression in the current formatter while preserving `formatLastExitStatus`, keep raw runtime fields available for diagnostics, add focused regression coverage, and attach redacted live Linux/systemd proof.

## Reproduction Assessment

Yes, at source level: current main can return `status: "running"` with parsed systemd exit fields, and the shared formatter prints those fields in the gateway status path. I did not run a live Linux/systemd restart reproduction in this read-only review.

## Solution Assessment

Partly yes: formatter-level suppression is the right visible-output boundary, but this branch must be refreshed to preserve `formatLastExitStatus` and prove the real systemd restart path.
