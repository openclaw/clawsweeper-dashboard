## Summary

Keep open. Current main still exposes stale systemd exit details, but the PR is stale against the current formatter and needs a narrow repair to preserve signal labels plus the required changelog and regression coverage.

## What This Changes

The PR changes daemon runtime status formatting to hide last-exit status and reason when the service runtime status is running.

## Best Possible Solution

Land a formatter-level guard that hides last-exit details only for running services while preserving raw systemd runtime fields, non-running diagnostics, signal labels, tests, and changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `readSystemdServiceRuntime` can return `status: "running"` with stale `lastExitStatus` and `lastExitReason`, and `formatRuntimeStatus` currently prints those fields into status output.

## Solution Assessment

No, not as submitted. Formatter-level suppression is the right boundary, but the branch should be rebased around current `formatLastExitStatus(...)` behavior and include regression coverage plus the required changelog entry.
