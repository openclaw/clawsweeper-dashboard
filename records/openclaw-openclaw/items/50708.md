## Summary

Keep open: current main still exposes systemd `ExecMainStatus`/`ExecMainCode` through the status formatter, so the PR is not obsolete, but the branch is not merge-ready because it regresses current signal-aware last-exit formatting and lacks live systemd proof.

## What This Changes

The PR changes daemon runtime status formatting to suppress last-exit status and reason details when `runtime.status` is `running`.

## Best Possible Solution

Refresh this PR so running status suppresses stale exit details only in the formatter, non-running diagnostics keep signal labels, focused tests cover both paths, and live Linux systemd proof is attached.

## Reproduction Assessment

Yes, at source level: current main reads systemd active state plus `ExecMainStatus`/`ExecMainCode`, maps active units to `running`, and the status printer formats those exit fields. I did not perform a live Linux/systemd restart in this read-only review.

## Solution Assessment

Partly yes: formatter-level suppression is the right boundary because raw runtime diagnostics remain available to restart and onboarding flows. The current branch still needs to preserve `formatLastExitStatus`, add focused coverage, and provide live proof.
