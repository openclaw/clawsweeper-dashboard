## What This Changes

The PR changes the plugin SDK transcript-stem resolver to skip entries whose `sessionId` normalizes empty during QMD slug fallback, adds a regression test, and adds a changelog entry.

## Best Possible Solution

Land the narrow guard and regression test through the existing automerge or maintainer gate once exact-head checks are green.

## Reproduction Assessment

Yes. Current main calls the QMD stem normalizer with `entry.sessionId` during fallback, and existing store cleanup can leave metadata rows without `sessionId`, so a mixed store can throw before returning the valid slug match.

## Solution Assessment

Yes. The patch reuses the existing optional string normalizer, skips only missing or empty ids during the fallback path, and preserves exact/file matching and ambiguous-match behavior.
