## Summary

Keep open: the feature direction is plausible, but PR head still leaves `before_agent_run` hook-block run metadata without `prepStages`, and the current-head proof refresh is focused test output rather than refreshed real runtime proof.

## What This Changes

This PR adds optional PI embedded-run prep stage timing snapshots to embedded attempt results, run metadata, and `agent_end` hook events, with docs, changelog, and focused coverage.

## Best Possible Solution

Expose the optional telemetry consistently on every terminal embedded-run meta path, keep the hook type/docs explicitly optional and diagnostic, and require current-head real runtime proof from a plugin-backed embedded run.

## Reproduction Assessment

Not applicable as a current-main bug because this is additive observability. The PR defect is source-reproducible on head: the `before_agent_run` hook-block branch returns run metadata without `prepStages` while adjacent terminal branches include it.

## Solution Assessment

No as submitted: reusing the existing stage tracker and frozen snapshot is a narrow maintainable direction, but the patch must cover the hook-block metadata path and refresh current-head real behavior proof before merge.
