## What This Changes

The issue reports stale persisted session rows staying in `running` after the backing run is gone, causing queued messages to stop processing across restarts and `sessions.abort`.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/84157 as the focused implementation path, coordinating with overlapping stale-session work rather than closing this issue before a fix lands.

## Reproduction Assessment

Yes at source level. Seed a persisted session row with `status: "running"` and no active abort controller; current `sessions.abort` can report `no-active-run` without clearing the row, and restart recovery skips the same row unless `abortedLastRun` is already true.

## Solution Assessment

Yes. The narrow maintainable fix is stale persisted-row reconciliation in the `sessions.abort` no-active-run path and restart recovery path, with guards for live work; the open closing PR is the current candidate.
