## What This Changes

The issue asks OpenClaw to clear persisted session rows stuck in `status: "running"` when the backing run is gone, including `sessions.abort` no-active-run cases and gateway restarts.

## Best Possible Solution

Keep this issue open while https://github.com/openclaw/openclaw/pull/84157 is reviewed, then close it only after a merged fix covers the stale abort and restart reconciliation paths or the residual scope is split into a canonical follow-up.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed a persisted row with `status: "running"` and no active abort controller, then `sessions.abort` can return no-active-run without clearing it, and startup recovery skips markerless running rows.

## Solution Assessment

Yes for the narrow fix direction. The linked PR is the right implementation candidate to review because it targets the abort no-active-run and markerless restart seams; broader heartbeat/TTL policy can remain separate if maintainers want it.
