## Summary

Keep open. Current main still has the stale persisted-row gap: `sessions.abort` can report no active run without healing the persisted row, restart recovery still requires `abortedLastRun`, and this issue is paired with an open closing PR that should be reviewed or rejected before the issue is closed.

## What This Changes

The issue asks OpenClaw to clear or recover persisted session rows that remain `status: "running"` after the backing run has died, including through `sessions.abort` and gateway restart.

## Best Possible Solution

Land or revise the linked stale-row reconciliation PR so `sessions.abort` heals no-active-run rows and restart recovery safely handles markerless persisted `running` rows without clearing genuinely active sessions.

## Reproduction Assessment

Yes, source-level with high confidence. Seed a persisted session row with `status: "running"` and no active abort controller; current main can return `no-active-run` without updating the row, and restart recovery skips the same row unless `abortedLastRun` is already true.

## Solution Assessment

Yes. The narrow maintainable solution is stale persisted-row reconciliation in the abort and restart recovery ownership paths; the open closing PR is the current implementation candidate to review against that boundary.
