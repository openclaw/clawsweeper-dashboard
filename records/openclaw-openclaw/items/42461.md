## What This Changes

The issue requests that `openclaw tui` automatically connect to the active local Gateway runtime port, such as `48789`, instead of defaulting to `18789`.

## Best Possible Solution

Review and land or supersede https://github.com/openclaw/openclaw/pull/73338 with lock-backed port metadata consumed only for default local TUI resolution, preserving explicit `--url` and remote config precedence.

## Reproduction Assessment

Yes, source-reproducible. The issue gives concrete `openclaw gateway --port 48789` then `openclaw tui` steps, and current main still resolves the default local URL without active runtime-port metadata.

## Solution Assessment

Yes at the direction level. Persisting the resolved runtime port in the existing Gateway lock and consuming it only for default local TUI resolution is the narrow maintainable fix, and the open linked PR is already attempting that path.
