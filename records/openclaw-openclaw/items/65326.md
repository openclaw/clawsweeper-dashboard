## Summary

Keep open. Current main only partially supports the reported Google Chat Workspace Add-ons flow: message add-on payloads and auth diagnostics exist, but lifecycle add-on wrappers still fail payload validation and setup/docs still do not carry or explain the required numeric appPrincipal; related open PRs cover subsets and need maintainer consolidation.

## What This Changes

The issue asks OpenClaw's Google Chat plugin to support Workspace Add-ons app-url authentication/setup and native unwrapping of Add-ons chat event envelopes without an external proxy.

## Best Possible Solution

Consolidate the open Google Chat PRs into plugin-local parser and setup/docs fixes with regression tests and real proof, while preserving explicit appPrincipal binding unless maintainers approve an auth-policy change.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: submit an Add-ons envelope with `commonEventObject.hostApp: "CHAT"`, `authorizationEventObject.systemIdToken`, and only `chat.addedToSpacePayload` or `chat.removedFromSpacePayload`; current main skips unwrapping and reaches `400 invalid payload`.

## Solution Assessment

No. Current main is incomplete; the narrow maintainable path is to normalize the supported Add-ons payload union and wire/document the existing appPrincipal field, not to auto-accept principals by pattern without review.
