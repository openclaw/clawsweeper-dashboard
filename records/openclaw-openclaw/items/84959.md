The issue asks OpenClaw to clarify node pairing through Tailscale Serve when the Gateway is loopback-bound, especially the need for `--tls` on the node command.

Document the supported Serve node path in the node and Tailscale/remote docs: use `openclaw node run --tls --host <magicdns> --port 443` for Tailscale Serve, and reserve plaintext `.ts.net` guidance for direct Tailnet binds.
