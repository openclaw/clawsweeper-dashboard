## What This Changes

The issue asks OpenClaw to add a hard bound to the gateway `agentRunCache` terminal snapshot map under sustained agent run fan-out.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77973, or replace it with an equivalent narrow gateway patch that caps `agentRunCache` after inserts while preserving existing TTL and wait semantics.

## Reproduction Assessment

Yes. From source inspection, emitting many unique terminal lifecycle events within the 10-minute TTL reaches `recordAgentRunSnapshot`, where fresh entries are not pruned and each unique run ID is inserted into the Map.

## Solution Assessment

Yes. The narrow maintainable fix is a hard max-entry cap next to the existing TTL pruning with regression coverage through the lifecycle event path; the issue should stay open while the linked implementation PR is reviewed or replaced.
