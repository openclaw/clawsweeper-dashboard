## Summary

Keep open: current main still has the TTL-only `agentRunCache` growth path, and an open same-author implementation PR already closes this issue and should be reviewed or replaced before the issue is closed.

## What This Changes

The issue requests a hard bound for gateway `agentRunCache` so sustained agent run fan-out cannot retain every terminal snapshot within the 10-minute TTL.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/77973, or replace it with an equivalent narrow gateway patch that caps `agentRunCache` after inserts while preserving existing TTL pruning and `agent.wait` semantics.

## Reproduction Assessment

Yes. Source inspection shows many unique lifecycle `end` events within the 10-minute TTL reach `recordAgentRunSnapshot`, where each fresh run ID is inserted into a TTL-only Map with no size cap.

## Solution Assessment

Yes. A hard max-entry cap enforced after insertion is the narrow maintainable fix because it composes with the existing TTL prune without changing lifecycle wait semantics.
