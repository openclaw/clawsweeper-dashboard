## Summary

Keep this open: current main and the latest release still have a TTL-only gateway `agentRunCache` without a hard size cap, while the linked closing PR is still open and unmerged.

## What This Changes

The issue requests a hard bound on `agentRunCache` so terminal agent-run snapshots cannot grow with sustained run fan-out for the full TTL window.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77973, or an equivalent small gateway fix that caps the terminal snapshot cache while preserving `agent.wait` semantics.

## Reproduction Assessment

Yes, source-reproducible: unique lifecycle terminal events with fresh run IDs flow through `recordAgentRunSnapshot`, which currently stores each snapshot with only TTL pruning. I did not run a stress loop because this review is read-only.

## Solution Assessment

Yes, a hard cap on the existing in-process cache plus regression coverage is the narrowest maintainable fix. The open linked PR owns that repair, so the issue should stay open until that PR or an equivalent fix lands.
