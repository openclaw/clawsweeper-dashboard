## Summary

Keep this open: current main and the latest release still have a TTL-only gateway `agentRunCache` with no hard size cap, and the open linked PR is still the active closing candidate rather than a merged fix.

## What This Changes

Request a hard max-size cap for the gateway `agentRunCache` so sustained lifecycle fan-out cannot retain every terminal run snapshot from the last 10 minutes.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77973, or an equivalent small gateway patch that caps the terminal snapshot cache while preserving `agent.wait` semantics and regression coverage.

## Reproduction Assessment

Yes. Source inspection shows unique lifecycle terminal events with fresh run IDs flow into `recordAgentRunSnapshot`, which currently stores each snapshot with only TTL pruning and no hard cap.

## Solution Assessment

Yes. A hard cap on the existing in-process terminal-snapshot cache is the narrow maintainable fix; the issue should stay open until the linked PR or an equivalent fix lands.
