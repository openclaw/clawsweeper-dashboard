## Summary

Keep this open: current main and the latest release still have a TTL-only gateway `agentRunCache` with no hard size cap, and https://github.com/openclaw/openclaw/pull/77973 is still the open candidate fix rather than a merged implementation.

## What This Changes

The issue requests a hard bound for the gateway `agentRunCache` so sustained agent-run fan-out cannot retain every terminal snapshot in the 10-minute TTL window.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77973, or an equivalent focused gateway patch that caps the terminal snapshot cache while preserving `agent.wait` semantics and adding regression coverage.

## Reproduction Assessment

Yes. Source inspection shows unique lifecycle terminal events with fresh run IDs flow into `recordAgentRunSnapshot`, which currently stores each snapshot with only TTL pruning and no hard cap.

## Solution Assessment

Yes. A hard cap on the existing in-process terminal-snapshot cache is the narrow maintainable fix; the issue should stay open until the linked PR or an equivalent fix lands.
