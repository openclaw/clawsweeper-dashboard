## Summary

Keep open: current main still has the TTL-only `agentRunCache` with no max-entry cap, and #77973 is an open closing PR for the same fix, so this should remain paired until that PR lands or is replaced.

## What This Changes

Request a hard bound on the gateway `agentRunCache` so sustained agent-run fan-out cannot retain every terminal snapshot from the last 10 minutes.

## Best Possible Solution

Land #77973 or an equivalent narrow gateway patch that enforces a hard entry cap after cache inserts, preserves the existing TTL and wait semantics, and adds regression coverage plus the required changelog entry.

## Reproduction Assessment

Yes, from source inspection: many unique terminal lifecycle events within 10 minutes reach `recordAgentRunSnapshot`, where TTL pruning removes none of the fresh entries and the map grows one entry per run. I did not run a live stress reproduction in this read-only review.

## Solution Assessment

Yes, a hard max-entry enforcement point next to the existing TTL pruning is the narrow maintainable fix. The issue should stay open while the linked PR is reviewed rather than spawning a competing repair lane.
