## Summary

Keep this open: current `main` and the latest release still have a TTL-only `agentRunCache` with no hard cap, and the linked closing PR is open but unmerged.

## What This Changes

Request a hard bound on gateway `agentRunCache` so sustained agent run fan-out cannot retain every terminal snapshot from the last ten minutes.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77973 or an equivalent small gateway fix that caps the terminal snapshot cache while preserving `agent.wait` behavior.

## Reproduction Assessment

Yes, source-reproducible: unique lifecycle terminal events with fresh run IDs flow through `recordAgentRunSnapshot`, which currently stores each snapshot with only TTL pruning. I did not run a stress loop because this review is read-only.

## Solution Assessment

Yes, a small hard cap on the existing in-process cache plus regression coverage is the narrowest maintainable fix. The right path is to review the linked PR or land an equivalent focused patch, not queue a second fix PR.
