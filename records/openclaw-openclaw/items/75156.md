## Summary

Keep this open: current main still has the block-streaming media accounting path that can mark mixed text+media as already sent before final dedupe strips the attachment, and PR #75180 is the open closing implementation candidate rather than a reason to close this issue before merge.

## What This Changes

Telegram direct-chat replies that combine visible text with a `MEDIA:` attachment should deliver both parts on the first send.

## Best Possible Solution

Land PR #75180 or an equivalent narrow fix that tracks only media actually delivered by block streaming while preserving duplicate suppression for true media-only sends.

## Reproduction Assessment

Yes. The reporter provided Telegram DM steps, and current main has a source-backed path where mixed block media is recorded as sent and later stripped from the final payload while text remains.

## Solution Assessment

Yes. The narrow maintainable fix belongs in generic block-reply sent-media accounting or delivery confirmation, not in Telegram transport, because Telegram already supports captioned media sends.
