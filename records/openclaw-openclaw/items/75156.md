## Summary

Keep this open: current main still records mixed text+media block payload media as already sent before final reply media dedupe, and the linked closing PR is still open rather than merged.

## What This Changes

The issue asks for Telegram first replies containing visible text plus a `MEDIA:` attachment to deliver both parts together instead of dropping the attachment on the first send.

## Best Possible Solution

Review and land #75180, or an equivalent narrow fix that tracks only media actually delivered by block streaming while preserving duplicate suppression for media-only sends.

## Reproduction Assessment

Yes. The reporter provided concrete Telegram DM steps, and current main still has a source-level path where mixed text+media block payload media is recorded as sent and later stripped from the final payload.

## Solution Assessment

Yes. The narrow fix belongs in generic block-reply media accounting or delivery confirmation rather than Telegram transport, because Telegram delivery already supports captioned media sends.
