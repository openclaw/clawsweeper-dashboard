## Summary

Keep this open: current main still has the block-reply media accounting path that can mark mixed text-plus-media block payloads as sent before final reply dedupe, and open PR #75180 is the active closing implementation candidate rather than a reason to close the issue before merge.

## What This Changes

Fix Telegram DM delivery so the first assistant reply containing visible text plus a MEDIA attachment sends both parts instead of dropping the attachment.

## Best Possible Solution

Land PR #75180 or an equivalent source-level fix that makes block reply sent-media tracking delivery-aware while preserving duplicate suppression for media that actually shipped.

## Reproduction Assessment

Yes. The reporter gave live Telegram DM steps, and current main has a source-level path where a mixed text-plus-media block reply can be recorded in sentMediaUrls and then have its final attachment stripped by dedupe.

## Solution Assessment

Yes. The narrow maintainable fix belongs in generic block reply media accounting, not in Telegram transport or generated dist files, because Telegram delivery already supports captioned media payloads.
