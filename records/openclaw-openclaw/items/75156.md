## Summary

Keep this open: current main still has the premature block-reply media accounting path, and PR #75180 is the open closing implementation candidate rather than a reason to close this issue before merge.

## What This Changes

Telegram direct-chat replies that combine visible text with a `MEDIA:` attachment should deliver both together instead of dropping the attachment on the first send.

## Best Possible Solution

Land PR #75180 or an equivalent narrow fix that tracks only media actually delivered by block streaming while preserving duplicate suppression for true media-only sends.

## Reproduction Assessment

Yes. The reporter provided live Telegram DM steps, and current main still has a source path where mixed block media is recorded and later stripped from the final payload.

## Solution Assessment

Yes. The narrow fix belongs in generic block-reply sent-media accounting and delivery confirmation, not in Telegram transport, because Telegram delivery already supports captioned media.
