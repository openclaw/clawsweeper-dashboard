## Summary

Keep this open: current main still has the source-level media-accounting path described by the report, and #75180 is an open, unmerged closing PR for the same bug.

## What This Changes

The issue asks for Telegram direct replies containing both visible text and a `MEDIA:` attachment to deliver both on the first send.

## Best Possible Solution

Review and land #75180, or an equivalent narrow generic block-reply fix that records only media actually delivered by streaming while preserving duplicate suppression for media-only sends.

## Reproduction Assessment

Yes. The reporter provided concrete Telegram DM steps, and current main still has a source-level path where mixed text+media block payload media is recorded as sent and later stripped from the final payload.

## Solution Assessment

Yes. The narrow fix belongs in generic block-reply media accounting or delivery confirmation rather than Telegram transport, because the Telegram delivery path already supports captioned media sends.
