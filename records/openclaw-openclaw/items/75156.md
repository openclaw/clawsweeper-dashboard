## Summary

Keep open because current main still has the reported block-streaming media accounting path, and open PR #75180 is the active unmerged implementation candidate with closing syntax for this issue.

## What This Changes

Ensure Telegram direct replies containing visible text plus a `MEDIA:` attachment deliver both the text and attachment on the first send.

## Best Possible Solution

Review and land PR #75180, or an equivalent narrow patch, so block reply sent-media tracking only dedupes media that was actually delivered while preserving media-only duplicate suppression and adding regression coverage for mixed text-plus-media block replies.

## Reproduction Assessment

Yes. The reporter provided live Telegram DM steps and local validation, and current main gives a source-level reproduction path: mixed text-plus-media block payloads can enter `sentMediaUrls`, then final payload media dedupe strips the same attachment before delivery.

## Solution Assessment

Yes. The fix should be in the generic block reply media accounting boundary, not Telegram transport, because Telegram can send captioned media and the strip happens before the transport delivery path.
