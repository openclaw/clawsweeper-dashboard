## Summary

Keep this open: current main still records media from mixed text+media block replies before final payload dedupe, and #75180 is an open closing PR rather than a merged fix.

## What This Changes

Telegram direct-chat replies that combine visible text with a `MEDIA:` attachment should deliver both text and the attachment on the first send.

## Best Possible Solution

Land #75180 or an equivalent narrow fix that tracks only media actually delivered by block streaming while preserving duplicate suppression for media-only sends.

## Reproduction Assessment

Yes. The reporter provided live Telegram direct-chat steps, and current main has a high-confidence source path where mixed block media is recorded and later stripped from the final payload.

## Solution Assessment

Yes. The right fix boundary is shared block-reply sent-media accounting, not Telegram transport, because Telegram delivery already supports text captions on media replies.
