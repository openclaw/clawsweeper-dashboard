## Summary

Keep this open: current main still has the source-level mixed text-plus-media media-accounting path that can strip the attachment from the final payload, and #75180 is an open closing PR rather than a merged fix.

## What This Changes

The issue reports that Telegram can deliver visible text while dropping a same-reply `MEDIA:` attachment on the first assistant response.

## Best Possible Solution

Review and land #75180, or an equivalent narrow fix that tracks only media actually delivered by block streaming while preserving duplicate suppression for real media-only sends.

## Reproduction Assessment

Yes. The reporter provided concrete Telegram DM steps, and current main still has a source-level path where mixed text-plus-media block payload media is recorded as sent and later stripped from the final payload.

## Solution Assessment

Yes. A delivery-aware block-reply media accounting fix is the narrowest maintainable direction because Telegram transport already supports captioned media and the bug is upstream in final payload dedupe.
