## Summary

Keep this open. Current main still has the reported block-stream/final-dedupe path, and open PR #75180 is the active implementation candidate with closing syntax, so the issue should remain open until that PR lands or is replaced.

## What This Changes

Fix Telegram direct-chat delivery so a first assistant reply containing visible text plus a `MEDIA:` attachment sends both the text/caption and the attachment.

## Best Possible Solution

Review and land PR #75180, or replace it with an equivalent narrow fix that makes block reply sent-media accounting delivery-aware and adds regression coverage for mixed text-plus-media replies plus media-only duplicate suppression.

## Reproduction Assessment

Yes. ## Solution Assessment

Yes. The maintainable fix belongs in the generic block reply sent-media accounting boundary with regression tests; changing Telegram transport would address a downstream symptom while Telegram already supports captioned media sends.
