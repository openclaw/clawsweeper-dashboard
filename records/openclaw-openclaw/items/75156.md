## Summary

Keep this open: current main still contains the mixed block-media accounting path, and PR #75180 is the open closing implementation candidate rather than a reason to close this issue before merge.

## What This Changes

A Telegram DM bug report asks for assistant replies containing visible text plus `MEDIA:` attachments to deliver both the text and attachment on the first send.

## Best Possible Solution

Land PR #75180 or an equivalent narrow fix that tracks only media actually delivered by block streaming while preserving duplicate suppression for media-only sends.

## Reproduction Assessment

Yes. The reporter provided live Telegram DM steps, and current main has a high-confidence source path where mixed block media is recorded and later stripped from the final payload.

## Solution Assessment

Yes. The narrow fix belongs in generic block-reply sent-media accounting and dispatch confirmation, not in Telegram transport, because Telegram media delivery already supports text captions.
