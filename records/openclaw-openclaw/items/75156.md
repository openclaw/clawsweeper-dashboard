## Summary

Keep this open: current main still has the block-media accounting path that can strip final attachments, and PR #75180 is an open unmerged closing implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests that Telegram direct replies containing visible text plus a `MEDIA:` attachment deliver both parts on the first send.

## Best Possible Solution

Review and land PR #75180, or an equivalent narrow fix, so block reply dedupe records only media that was actually delivered while preserving media-only duplicate suppression.

## Reproduction Assessment

Yes. The reporter provided live Telegram DM steps, and current main has a source path where mixed `MEDIA:` block replies are parsed, enqueued, recorded as sent media, and later stripped from the final payload.

## Solution Assessment

Yes. The narrow fix belongs in generic block-reply media accounting rather than Telegram transport, because the Telegram delivery code already supports captions on media replies.
