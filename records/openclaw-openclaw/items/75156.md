## Summary

Keep this open: current main still has the source path that can mark mixed text+media block replies as sent media before final payload dedupe, and the linked closing PR is still open rather than merged.

## What This Changes

The issue reports that Telegram direct chats can drop the attachment from a first assistant reply when visible text and a `MEDIA:` attachment are emitted together.

## Best Possible Solution

Review and land #75180, or an equivalent narrow fix that tracks only media actually delivered by block streaming while preserving duplicate suppression for media-only sends.

## Reproduction Assessment

Yes. The reporter provided concrete Telegram DM steps and current main has a source-level reproduction path where a mixed text+media block payload is recorded as sent media and later filtered from the final payload.

## Solution Assessment

Yes. The narrow fix belongs in generic block-reply media accounting or delivery confirmation rather than Telegram transport, because the Telegram delivery code already supports captioned media sends.
