## Summary

Keep this open. Current main still has the reported block-stream/final-dedupe path that can record mixed text-plus-media block replies as already sent, and open PR #75180 is the active implementation candidate with closing syntax for this issue.

## What This Changes

The issue requests that Telegram direct chats deliver both visible text and a MEDIA attachment from the first assistant reply when both appear in the same response.

## Best Possible Solution

Review and land PR #75180 or an equivalent narrow fix that makes block reply sent-media tracking delivery-aware, preserves media-only duplicate suppression, and adds regression coverage for mixed text-plus-media block replies with final payload dedupe.

## Reproduction Assessment

Yes. The reporter provided live Telegram DM steps and local validation, and current source gives a high-confidence unit path: enqueue a mixed text-plus-media block reply, observe it enter sentMediaUrls, then let final payload dedupe strip that media while leaving text.

## Solution Assessment

Yes. The best fix belongs in generic block-reply sent-media accounting rather than Telegram transport, because Telegram delivery can send captioned media and the strip occurs before final Telegram send.
