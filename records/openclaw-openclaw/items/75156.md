## Summary

Keep this open: current main still has the source-level mixed text+media dedupe path that can strip the attachment, and #75180 is an open closing PR rather than a merged fix.

## What This Changes

The issue reports that Telegram direct chat can drop the attachment from the first assistant reply when text and MEDIA appear in the same assistant reply.

## Best Possible Solution

Review and land #75180, or an equivalent narrow fix that tracks only media actually delivered by block streaming while preserving duplicate suppression for confirmed media-only sends.

## Reproduction Assessment

Yes. The reporter provided concrete Telegram DM steps, and current main still has a source-level path where mixed text+media block payload media is recorded as sent and later stripped from the final payload.

## Solution Assessment

Yes. The narrowest maintainable fix belongs in generic block-reply media accounting or delivery confirmation rather than Telegram transport, because Telegram delivery already supports captioned media sends.
