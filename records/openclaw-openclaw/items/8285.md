## Summary

Keep open. Current main still lacks a supported cross-channel text acknowledgment that sends before agent processing and then continues normal dispatch; the related Feishu work is open, unmerged, and channel-specific.

## What This Changes

The issue requests a configurable pre-processing text acknowledgment, such as `messages.ackText`, for Slack, WhatsApp, WebChat, Feishu, and similar messaging channels.

## Best Possible Solution

Design a supported acknowledgment-text contract with scope and channel/account overrides or a documented non-terminal hook, then update config types, schema/help, public docs, channel docs, and regression coverage across the affected messaging surfaces.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for missing supported behavior. Source, schema, docs, hook tests, and related PR inspection give high confidence that current main still lacks the requested ack-text surface.

## Solution Assessment

Unclear. The proposed `messages.ackText` shape is plausible, but the maintainable fix needs maintainer API direction because it spans core config, plugin hooks, and multiple channel reply lifecycles.
