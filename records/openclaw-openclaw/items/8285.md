## Summary

Keep open: current main and the latest release still lack a supported cross-channel text acknowledgement config or send-and-continue pre-processing hook. The existing reaction, status reaction, typing, and terminal before_dispatch surfaces are adjacent but do not satisfy the requested behavior, and related channel-specific items are not merged replacements.

## What This Changes

The issue requests configurable pre-processing text acknowledgments for messaging channels before agent processing starts.

## Best Possible Solution

Keep this open for maintainer product/API direction on a supported default-off `messages.ackText` surface or a documented non-terminal pre-processing hook that can send interim text and continue normal dispatch.

## Reproduction Assessment

Yes for source-level reproduction: current main and v2026.6.8 lack `messages.ackText`, and `before_dispatch` is terminal rather than send-and-continue. This is not a failing-bug reproduction path because the issue requests a new capability.

## Solution Assessment

Unclear until maintainers choose the API shape. A default-off `messages.ackText` option is plausible, but a non-terminal plugin hook may better match the repository’s lean-core/plugin-boundary policy.
