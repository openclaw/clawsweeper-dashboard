## Summary

Keep open. Current main still lacks a supported cross-channel pre-processing text acknowledgement; reactions, typing indicators, preview/progress streaming, and terminal dispatch hooks do not satisfy the requested send-text-and-continue contract.

## What This Changes

The issue requests a configurable text acknowledgement that is sent before agent processing continues, similar to existing acknowledgement reactions.

## Best Possible Solution

Add a supported cross-channel acknowledgement contract, either messages.ackText with scope/override semantics or a documented non-terminal hook, with schema/help/docs and channel regression coverage updated together.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks messages.ackText and a supported send-text-and-continue hook.

## Solution Assessment

Unclear. The requested behavior is valid, but maintainers need to choose between a cross-channel config contract and a plugin/API hook; #47961 is useful Feishu-specific source work, not the complete solution.
