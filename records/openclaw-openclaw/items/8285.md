## Summary

Keep open: current main and v2026.6.6 still lack a supported cross-channel text acknowledgement before agent processing; existing reactions, typing indicators, status reactions, and terminal hooks do not provide the requested send-and-continue text path.

## What This Changes

The issue requests a configurable pre-processing text acknowledgment before agent work begins on messaging channels.

## Best Possible Solution

Choose a maintainer-approved default-off design, either messages.ackText with scope and channel overrides or a documented non-terminal pre-processing hook, then update config types, schema/help, docs, channel docs, and regression tests.

## Reproduction Assessment

Yes at source level: current config/schema/docs and dispatch hooks do not expose a supported send-and-continue acknowledgement text path. No live transport run is needed to prove the requested capability is absent.

## Solution Assessment

Unclear until maintainers choose the product/API shape; messages.ackText is plausible, but a plugin/channel hook may be the better long-term boundary for cross-channel and channel-specific behavior.
