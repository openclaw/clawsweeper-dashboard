## Summary

Keep open. Current main still has supported ack reactions, typing indicators, terminal synthetic-reply hooks, and final dispatch hooks, but it does not expose a supported cross-channel text acknowledgement that can send and then continue normal agent processing; the related Feishu PR is open and only covers one channel.

## What This Changes

The issue requests a supported cross-channel config or hook that sends short acknowledgement text before normal agent processing continues.

## Best Possible Solution

Keep this as a core/product or plugin-SDK feature request and design a supported acknowledgement-text contract with scope/channel overrides, docs, schema/help updates, and regression coverage; coordinate the Feishu-only PR as a partial channel implementation rather than treating it as the canonical fix.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that the requested supported ack-text surface is still missing from current main.

## Solution Assessment

No current implementation is the complete solution. The requested direction is plausible, but maintainers still need to choose the cross-channel config and/or plugin-SDK hook shape before implementation.
