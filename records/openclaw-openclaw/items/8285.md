## Summary

Keep open. Current main still lacks a supported cross-channel pre-processing text acknowledgement: message config/schema expose reactions and status reactions, docs describe reaction/typing/preview signals, and the closest hook path is terminal rather than send-and-continue.

## What This Changes

Request to add configurable text acknowledgement before agent processing, similar to `ackReaction`, for messaging channels such as Slack, WhatsApp, WebChat, and Feishu.

## Best Possible Solution

Add a supported cross-channel pre-processing acknowledgement contract, either `messages.ackText` with scope/override semantics or a documented non-terminal hook, with schema/help/docs and channel regression coverage updated together.

## Reproduction Assessment

Not applicable as a runtime bug report; this is a feature request. Source inspection gives high-confidence verification of the missing capability because `ackText` is absent from strict messages config and `before_dispatch` cannot send interim text while continuing model dispatch.

## Solution Assessment

Unclear. The requested behavior is valid, but maintainers need to choose between a cross-channel config contract and a plugin/API hook; Slack preview progress and the Feishu-only PR are useful partial paths, not the complete solution.
