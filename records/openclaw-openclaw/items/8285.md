## Summary

Keep open. Current main still lacks a supported cross-channel text acknowledgment that sends before agent processing and then continues normal dispatch; the related Feishu PR is open, unmerged, channel-specific, and does not supersede the broader API/config request.

The issue requests a configurable text acknowledgment, such as `messages.ackText`, that is sent before agent processing across messaging channels.

## Best Possible Solution

Design a supported acknowledgment-text contract with scope and per-channel/account overrides, or a documented non-terminal hook, then update config types, strict schema/help, public docs, channel docs, and representative channel tests.
