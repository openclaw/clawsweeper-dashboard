## Summary

Keep open. Current main still has reaction/status/typing signals and a terminal before_dispatch hook, but no supported cross-channel text acknowledgement config or send-and-continue hook; the related Feishu PR is open and only covers one channel.

## What This Changes

Requests configurable pre-processing text acknowledgements, similar to ackReaction, before agent processing across messaging channels.

## Best Possible Solution

Define a supported cross-channel ackText config or a documented non-terminal preprocessing hook, then update config types, schema/help, docs, channel behavior, and regression coverage together.

## Reproduction Assessment

Not applicable as a feature request rather than broken documented behavior. Source inspection against current main shows the requested ackText config and send-and-continue hook are absent.

## Solution Assessment

No current workaround is the best solution: prompting the agent is unreliable, ackReaction/typing are not text, and before_dispatch is terminal. The maintainable direction is a deliberate config or plugin-API contract with channel-specific semantics.
