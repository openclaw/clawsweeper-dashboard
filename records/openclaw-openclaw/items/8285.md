## Summary

Keep open. Current main and the latest shipped tag still lack a supported cross-channel text acknowledgment before agent processing; existing reactions, status reactions, typing indicators, and terminal dispatch hooks do not satisfy the requested send-and-continue contract.

## What This Changes

The issue requests a configurable pre-processing text acknowledgment, such as `messages.ackText`, for Slack, WhatsApp, WebChat, Feishu, and similar messaging channels.

## Best Possible Solution

Define a supported cross-channel acknowledgment-text contract or non-terminal pre-processing hook, then align config types, strict schema/help, public docs, channel behavior, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Current source, strict schema, docs, shipped-tag inspection, and related-item review give high confidence that the supported text-acknowledgment contract is still missing.

## Solution Assessment

Unclear pending product/API decision. The proposed `messages.ackText` shape is plausible, but maintainers should choose the shared config, per-channel override, or hook boundary before implementation.
