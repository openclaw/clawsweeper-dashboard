## Summary

Keep open. Current main still lacks a supported cross-channel text acknowledgement that sends before agent processing and then continues normal dispatch; the only related implementation in context is open PR #47961, which is Feishu-only and not merged.

## What This Changes

Request to add configurable pre-processing acknowledgement text, similar to ack reactions, across messaging channels such as Slack, WhatsApp, WebChat, and Feishu.

## Best Possible Solution

Design a supported acknowledgement-text contract with scope and channel/account overrides or a documented non-terminal hook, then update config types, schema/help, docs, and regression coverage while coordinating the Feishu-only PR as a partial channel implementation.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that the requested supported ack-text surface is still missing from current main.

## Solution Assessment

Unclear. The proposed `messages.ackText` shape is plausible, but the best maintainable fix needs maintainer API direction because it crosses config, plugin hooks, and multiple channel reply lifecycles.
