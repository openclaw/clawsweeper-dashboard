## Summary

Keep open. Current main still lacks a supported cross-channel text acknowledgment before agent processing; existing reactions, typing indicators, and terminal dispatch hooks do not satisfy the requested configurable send-and-continue behavior.

## What This Changes

The issue requests a configurable pre-processing text acknowledgment such as `messages.ackText`/`ackTextScope` for messaging channels before normal agent processing begins.

## Best Possible Solution

Design a supported acknowledgment-text contract with scope and channel/account overrides, or a documented non-terminal pre-processing hook, then update config types, strict schema/help, public docs, channel docs, and representative messaging tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for behavior that current config and hook contracts do not expose. Source, docs, and the open Feishu-specific PR give high confidence the cross-channel surface remains missing.

## Solution Assessment

Unclear. The proposed `messages.ackText` shape is plausible, but the maintainable fix needs maintainer API direction because it spans config, plugin hooks, and channel reply lifecycles.
