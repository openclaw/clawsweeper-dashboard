## Summary

Keep open. Current main still lacks a supported cross-channel text acknowledgment before agent processing, and the related Feishu PR is open, channel-specific, and not a replacement for the requested config or plugin API contract.

## What This Changes

The issue requests configurable acknowledgment text, such as `messages.ackText`, that sends before agent processing begins across messaging channels.

## Best Possible Solution

Design a supported acknowledgment-text contract with scope and channel/account overrides, or a documented non-terminal pre-processing hook, then update config types, strict schema/help, public docs, channel docs, and representative messaging tests.

Not applicable as a bug reproduction: this is a feature request for behavior that current config and hook contracts do not expose. Source, docs, and the open Feishu-specific PR give high confidence the cross-channel surface remains missing.

## Solution Assessment
