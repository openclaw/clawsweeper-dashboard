## Summary

Keep open. Current main still lacks a supported cross-channel pre-processing text acknowledgement; the existing mechanisms cover reactions, typing/progress drafts, and terminal dispatch hooks, but not the requested send-text-and-continue contract.

## What This Changes

Request to add configurable acknowledgement text before agent processing on messaging channels, analogous to the existing ack reaction settings.

## Best Possible Solution

Define a supported cross-channel acknowledgement contract with schema/help/docs and channel regression coverage, while treating #47961 as Feishu-specific input rather than the complete fix.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks messages.ackText and a supported send-text-and-continue hook.

## Solution Assessment

Unclear. The requested behavior is valid, but maintainers need to choose between a cross-channel config contract and a plugin/API hook; a Feishu-only delayed ack would not satisfy the reported Slack, WhatsApp, and WebChat scope.
