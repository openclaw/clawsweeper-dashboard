## Summary

Keep open. Current main still lacks a supported cross-channel text acknowledgment that sends before agent processing and then continues normal dispatch; https://github.com/openclaw/openclaw/pull/47961 is open, unmerged, Feishu-specific, and does not supersede the broader request.

## What This Changes

Request a supported cross-channel pre-processing text acknowledgment, such as `messages.ackText`, that sends before agent processing and then continues normal dispatch.

## Best Possible Solution

Design a supported acknowledgment-text contract with scope and per-channel/account overrides, or a documented non-terminal hook, then update config types, strict schema/help, public docs, channel docs, and representative channel tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for behavior current config and hook contracts do not provide. Source, schema, docs, hook tests, and related PR inspection give high confidence the requested surface is still missing on current main.

## Solution Assessment

Unclear. The proposed `messages.ackText` config is plausible, but the best maintainable fix needs maintainer API direction because it spans core config, plugin hooks, and channel reply lifecycles.
