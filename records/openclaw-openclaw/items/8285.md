## Summary

Keep open. Current main and v2026.5.22 still lack a supported text acknowledgement that sends before agent processing and then continues normal dispatch; existing reaction, status-reaction, typing, and terminal hook paths do not satisfy the requested cross-channel contract.

## What This Changes

Request a cross-channel `messages.ackText`-style configuration or equivalent hook that sends a short visible text acknowledgement before agent processing begins.

## Best Possible Solution

Define one supported cross-channel acknowledgement-text contract or non-terminal send-and-continue hook, then align config types, strict schema/help, public docs, channel behavior, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source, strict schema, docs, latest-release inspection, and related-item checks give high confidence that the supported text-acknowledgement contract is absent.

## Solution Assessment

Unclear pending product/API decision. The proposed `messages.ackText` shape is plausible, but maintainers should choose the shared config or hook boundary before implementation so channel-local work does not create incompatible behavior.
