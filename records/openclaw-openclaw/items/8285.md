## Summary

Keep open. Current main still does not provide a supported configurable text acknowledgement that sends before agent processing and then continues normal dispatch; the existing reaction, status-reaction, typing, progress-draft, and terminal hook paths are adjacent but do not satisfy the requested cross-channel contract.

## What This Changes

The issue requests a configurable pre-processing acknowledgement text message, similar to `ackReaction`, for channels such as Slack, WhatsApp, WebChat, and Feishu.

## Best Possible Solution

Define one supported cross-channel acknowledgement-text contract or non-terminal send-and-continue hook, then align config types, strict schema/help, public docs, channel behavior, and focused regression tests around that boundary.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source, strict schema, docs, latest-release inspection, and related-item checks give high confidence that the supported text-acknowledgement contract is absent.

## Solution Assessment

Unclear pending product/API decision. The proposed `messages.ackText` shape is plausible, but maintainers should choose the shared config or hook boundary before implementation so channel-local work does not create incompatible behavior.
