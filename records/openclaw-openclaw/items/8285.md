## Summary

Keep open: current main and the latest release still lack a supported cross-channel text acknowledgement before agent processing; existing reactions, typing indicators, streaming progress, and terminal hooks do not provide the requested configurable send-and-continue text path.

## What This Changes

Request support for automatically sending a configurable text acknowledgment before agent processing starts.

## Best Possible Solution

Choose a maintainer-approved default-off design, either messages.ackText with scope and channel/account overrides or a documented non-terminal pre-processing hook, then update config types, schema/help, docs, channel docs, and regression tests.

## Reproduction Assessment

Yes at source level: current config types, strict schema, docs, and dispatch hooks do not expose a supported send-and-continue acknowledgement text path. No live transport run is needed to prove the requested capability is absent.

## Solution Assessment

Unclear until maintainers choose the product/API shape; messages.ackText is plausible, but a plugin/channel hook may be the better long-term boundary for cross-channel and channel-specific behavior.
