## Summary

Keep open. Current main still only supports reactions, status reactions, typing indicators, final reply dispatch, and a terminal `before_dispatch` hook; it does not provide the requested cross-channel text acknowledgment config or send-and-continue hook. The related Feishu PR is open and channel-specific, so it is not a substitute for this broader API/product request.

## What This Changes

The issue requests a supported cross-channel text acknowledgment before agent processing, similar to existing acknowledgment reactions.

## Best Possible Solution

Keep this as a maintainer-owned feature request and choose either a supported `messages.ackText` contract with scope and overrides or a documented non-terminal pre-processing hook, then update config types, schema/help, public docs, channel docs, and regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source, schema, docs, and hook tests give high confidence that current main still lacks the requested supported text-acknowledgment contract.

## Solution Assessment

Unclear pending product decision. The proposed `messages.ackText` shape is plausible, but maintainers should choose the cross-channel config/API boundary before implementation rather than treating this as a narrow bug fix.
