## Summary

Keep open. Current main still lacks a supported cross-channel text acknowledgment that sends before agent processing and then continues normal dispatch; the related Feishu work at https://github.com/openclaw/openclaw/pull/47961 is open, unmerged, and channel-specific.

## What This Changes

Request a configurable `messages.ackText`/scope-style text acknowledgment or equivalent non-terminal hook before messaging-channel agent work starts.

## Best Possible Solution

Design a supported acknowledgment-text contract with scope and channel/account overrides or a documented non-terminal hook, then update config types, schema/help, docs, channel docs, and regression coverage across the relevant messaging surfaces.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for a missing supported behavior. Source, schema, docs, and hook-test inspection give high confidence that current main still lacks the requested ack-text surface.

## Solution Assessment

Unclear. The proposed `messages.ackText` shape is plausible, but the maintainable fix needs maintainer API direction because it spans core config, plugin hooks, and multiple channel reply lifecycles.
