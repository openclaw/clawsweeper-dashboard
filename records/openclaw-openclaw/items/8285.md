## Summary

Keep open. Current main still does not provide a supported cross-channel text acknowledgment before agent processing, and the only related implementation is the open Feishu-specific https://github.com/openclaw/openclaw/pull/47961 rather than the requested config/API contract.

## What This Changes

Request a configurable pre-processing text acknowledgment, similar to ack reactions, that can send before normal agent processing across Slack, WhatsApp, WebChat, Feishu, and similar messaging channels.

## Best Possible Solution

Design a supported acknowledgment-text contract with scope and channel/account overrides, or a documented non-terminal pre-processing hook, then update config types, strict schema/help, public docs, channel docs, and representative messaging tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for behavior the current config and hook contracts do not expose. Source, docs, tests, and the still-open Feishu-specific PR give high confidence the requested cross-channel surface remains missing.

## Solution Assessment

Unclear. The proposed messages.ackText shape is plausible, but the best maintainable fix needs maintainer API direction because it spans config, plugin hooks, and channel reply lifecycles.
