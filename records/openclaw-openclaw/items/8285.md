## Summary

Keep open. Current main still lacks a supported cross-channel text acknowledgment that sends before agent processing and then continues normal dispatch; the linked Feishu PR is open, unmerged, and channel-specific.

## What This Changes

The issue requests configurable automatic acknowledgment text, similar to `messages.ackReaction`, before longer agent processing on Slack, WhatsApp, WebChat, Feishu, and other messaging channels.

## Best Possible Solution

Design a supported acknowledgment-text contract with scope and channel/account overrides, or a documented non-terminal hook, then update config types, strict schema/help, public docs, channel docs, and regression coverage across representative messaging channels.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for behavior current config and hook contracts do not provide. Source, schema, docs, hook tests, and related PR inspection give high confidence the requested surface is still missing on current main.

## Solution Assessment

Unclear. The proposed `messages.ackText` config is plausible, but the best maintainable fix needs maintainer API direction because it spans core config, plugin hooks, and channel reply lifecycles.
