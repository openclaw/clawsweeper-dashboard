## Summary

Keep open. Current main still lacks a supported cross-channel pre-processing text acknowledgement: config and schema expose reactions/status reactions but no ackText-style field, docs are reaction/typing-only, and before_dispatch remains terminal rather than a send-and-continue hook. The open Feishu-only PR #47961 is useful related work but does not satisfy the requested Slack/WhatsApp/WebChat/core contract.

## What This Changes

The issue requests configurable text acknowledgement before agent processing begins, analogous to ack reactions and scoped across messaging channels.

## Best Possible Solution

Keep this open for maintainer product/API follow-up. The desired end state is either a supported messages.ackText configuration with scope and per-channel/account override semantics, or a documented non-terminal pre-processing hook that can send interim text and continue normal model dispatch, with config schema/help, public docs, and channel regression tests updated together.

## Reproduction Assessment

Not applicable as a runtime bug report; this is a feature request. There is high-confidence source verification of the missing capability because messages.ackText is absent from the strict config schema and before_dispatch cannot send interim text while continuing model dispatch.

## Solution Assessment

Unclear. The requested behavior is valid, but the best implementation requires maintainer choice between a cross-channel config contract and a plugin/API hook; the current Feishu-only PR is useful partial work but not the complete solution.
