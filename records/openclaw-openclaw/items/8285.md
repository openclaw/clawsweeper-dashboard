## Summary

Keep open. Current main still lacks a supported cross-channel pre-processing text acknowledgement; the existing implementation covers reactions, typing indicators, preview/progress streaming, and terminal dispatch hooks, but not a send-text-and-continue acknowledgement contract.

## What This Changes

The issue requests configurable acknowledgement text before agent processing starts across messaging channels.

## Best Possible Solution

Add a supported cross-channel acknowledgement contract, either messages.ackText with scope and override semantics or a documented non-terminal hook, with schema/help/docs and channel regression coverage updated together.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence the requested messages.ackText/send-and-continue path is absent from current main.

## Solution Assessment

Unclear. The requested behavior is valid, but maintainers need to choose between a cross-channel config contract and a plugin/API hook; #47961 is useful Feishu-specific source work, not the complete solution.
