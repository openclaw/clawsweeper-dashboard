## Summary

Keep open. Current main still lacks a supported cross-channel text acknowledgment contract before agent processing, and the related Feishu PR is open, channel-specific, and not a substitute for the requested Slack/WhatsApp/WebChat-style configuration or hook.

## What This Changes

The issue requests configurable text acknowledgments, such as `messages.ackText`, before normal agent processing starts across chat channels.

## Best Possible Solution

Keep this as a maintainer-owned core/product feature request: add either a supported `messages.ackText` contract with scope and channel/account overrides, or a documented non-terminal pre-processing hook, with config types, schema/help, public docs, channel docs, and regression coverage updated together.
