## Summary

Keep open. Current main still lacks a supported cross-channel text acknowledgement contract before agent processing, and the related Feishu PR is open, channel-local, and not a substitute for the requested Slack/WhatsApp/WebChat/core behavior.

## What This Changes

The issue requests a configurable text acknowledgement that can be sent before normal agent processing continues across messaging channels.

## Best Possible Solution

Define a supported cross-channel acknowledgement-text contract, either as `messages.ackText` with scope/timing overrides or a documented non-terminal pre-processing hook, then update config types, schema/help, docs, channel behavior, and regression coverage together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence current-main check that the requested `ackText` config/API does not exist and existing hooks are terminal or final-delivery oriented.

## Solution Assessment

Unclear until a maintainer makes the product/API decision. The requested config shape is plausible, but the maintainable fix should be a unified core/channel contract rather than only a channel-local hardcoded message.
