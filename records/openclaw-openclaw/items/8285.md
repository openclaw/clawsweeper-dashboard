## Summary

Keep open: current main and v2026.6.6 still lack a supported cross-channel text acknowledgement before agent processing, and the existing reaction, typing, status, and terminal hook paths do not satisfy the requested send-and-continue behavior.

## What This Changes

The issue requests a configurable pre-processing text acknowledgement, such as `messages.ackText`, for messaging channels before the agent starts long-running work.

## Best Possible Solution

Choose a maintainer-approved default-off design, either `messages.ackText` with scope and channel overrides or a documented non-terminal pre-processing hook, then update config types, schema/help, docs, channel docs, and regression tests.

## Reproduction Assessment

Yes at source level: current config/schema/docs and dispatch hooks do not expose a supported send-and-continue acknowledgement text path; no live transport run was needed to prove the capability is absent.

## Solution Assessment

Unclear until maintainers choose the product/API shape; the proposed `messages.ackText` config is plausible, but a plugin/channel hook may be the better long-term boundary.
