## Summary

Keep open. Current main and the latest release still lack a supported cross-channel acknowledgement text contract that sends before agent processing and then continues normal dispatch; existing reactions, typing indicators, status reactions, terminal hooks, and open channel-specific PRs do not supersede the broader product/API decision.

## What This Changes

The issue requests configurable cross-channel acknowledgement text, such as `messages.ackText`, before agent processing begins.

## Best Possible Solution

Define one supported cross-channel acknowledgement-text contract or a documented non-terminal send-and-continue hook, then align config types, strict schema/help, public docs, channel behavior, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source, strict schema, docs, latest-release inspection, and related-item checks give high confidence that the supported text-acknowledgement contract is absent.

## Solution Assessment

Unclear pending product/API decision. The proposed `messages.ackText` shape is plausible, but maintainers should choose the shared config or hook boundary before implementation so channel-local work does not create incompatible behavior.
