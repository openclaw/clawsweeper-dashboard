## Summary

Keep open: current main and v2026.6.10 still lack a supported cross-channel text acknowledgement that sends before agent processing and then continues normal dispatch. The gap is source-reproducible, but the fix needs a maintainer product/API decision rather than an automatic cleanup close.

## What This Changes

Request a configurable pre-processing text acknowledgement for messaging channels, similar to ack reactions but sent as a visible message before agent work starts.

## Best Possible Solution

Choose a maintainer-approved, default-off acknowledgement-text or send-and-continue hook contract, then update config types, strict schema/help, public docs, channel docs, and focused regression coverage.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.10 expose reactions, typing indicators, progress drafts, terminal hooks, and final replies, but no supported send-and-continue acknowledgement-text path. No live transport run is needed to prove the requested capability is absent.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. messages.ackText is plausible, but a non-terminal plugin or channel hook may be the cleaner long-term seam for cross-channel and channel-specific behavior.
