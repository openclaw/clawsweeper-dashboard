## Summary

Keep open. Current main and the latest release still do not provide a supported cross-channel text acknowledgement that sends before agent processing and then continues normal dispatch; existing reactions, typing indicators, status reactions, and terminal hooks cover adjacent UX but not the requested contract.

## What This Changes

The issue requests configurable cross-channel text acknowledgements sent before agent processing starts, similar to existing acknowledgement reactions.

## Best Possible Solution

Define one supported cross-channel acknowledgement-text contract or non-terminal preprocessing hook, then align config types, strict schema/help, public docs, channel behavior, and focused regression tests.
