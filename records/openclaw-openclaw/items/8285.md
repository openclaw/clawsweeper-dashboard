## Summary

Keep open. Current main and the latest shipped tag still lack a supported cross-channel text acknowledgement that sends before agent processing and then continues normal dispatch; the open Feishu PR and Telegram issue overlap the UX gap but do not supersede the broader config/API decision.

## What This Changes

The issue requests a configurable pre-processing text acknowledgement, similar to `messages.ackReaction`, for Slack, WhatsApp, WebChat, Feishu, and similar messaging channels.

## Best Possible Solution

Define a first-class cross-channel acknowledgement-text contract or a documented non-terminal pre-processing hook, then align config types, strict schema/help, public docs, channel behavior, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source, strict schema, public docs, shipped-tag inspection, and related-item inspection give high confidence that the supported text-acknowledgement contract is absent.

## Solution Assessment
