## Summary

Keep open. Current main and v2026.5.22 still lack a supported text acknowledgement that sends before agent processing and then continues normal dispatch; reactions, typing indicators, progress drafts, and terminal hooks only cover adjacent UX.

## What This Changes

Request a configurable cross-channel pre-processing acknowledgement text message, similar to ackReaction, for Slack, WhatsApp, WebChat, Feishu, and related messaging channels.

## Best Possible Solution

Define one supported cross-channel acknowledgement-text contract or non-terminal send-and-continue hook, then align config types, strict schema/help, public docs, channel behavior, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source, strict schema, docs, related-item checks, and v2026.5.22 inspection give high confidence that the supported text-acknowledgement contract is absent.

## Solution Assessment

Unclear pending product/API decision. The proposed messages.ackText shape is plausible, but maintainers should choose the shared config or hook boundary before implementation so channel-local work does not create incompatible behavior.
