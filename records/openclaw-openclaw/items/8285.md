## Summary

Current main and v2026.6.5 still lack a supported cross-channel text acknowledgment that sends before agent processing and then continues normal dispatch. Existing reactions, typing indicators, transport acknowledgments, and the terminal dispatch hook cover adjacent behavior, while all located implementation PRs closed unmerged.

## What This Changes

Add configurable cross-channel acknowledgment text that is sent before agent processing while allowing normal dispatch to continue.

## Best Possible Solution

Define one generic interim-feedback lifecycle in core or the plugin SDK, with explicit timing and scope semantics and channel capabilities for sending, editing, or cleaning up visible status text.

## Reproduction Assessment

Not applicable as a bug reproduction; source and release inspection provide a high-confidence capability-gap check showing no supported visible acknowledgment-text contract.

## Solution Assessment

No, the proposed messages.ackText shape is only one plausible interface; a shared interim-feedback lifecycle is more maintainable than channel-local timers and keeps transport receipt acknowledgment semantically separate.
