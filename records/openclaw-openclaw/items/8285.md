## Summary

Keep open. Current main still lacks a supported cross-channel text acknowledgement before agent processing: config/schema expose reactions and status reactions, docs describe reaction/typing signals, and the closest hook path is terminal rather than send-and-continue.

## What This Changes

Request a configurable cross-channel text acknowledgement before agent processing, similar to ackReaction, for Slack, WhatsApp, WebChat, Feishu, and other messaging channels.

## Best Possible Solution

Add a supported cross-channel pre-processing acknowledgement contract, either messages.ackText with scope and per-channel/account overrides or a documented non-terminal plugin hook, with schema/help/docs and channel regression coverage updated together.

## Reproduction Assessment

Not applicable as a runtime bug report; this is a feature request. Source inspection gives high-confidence verification of the missing capability because ackText is absent from strict messages config and before_dispatch cannot send interim text while continuing model dispatch.

## Solution Assessment

Unclear. The requested behavior is valid, but the best implementation needs maintainer choice between a cross-channel config contract and a plugin/API hook; the current Feishu-only PR is useful partial work, not the complete solution.
