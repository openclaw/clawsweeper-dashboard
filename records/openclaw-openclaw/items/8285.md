## Summary

Current main and the latest release still lack a supported cross-channel text acknowledgment that sends before processing and then continues normal agent dispatch. Related implementation PRs closed without merging, so the request remains valid but needs a maintainer product/API decision.

## What This Changes

Add a supported cross-channel text acknowledgment that is sent before long-running agent processing begins.

## Best Possible Solution

Define one generic non-terminal interim-message lifecycle with channel capability, scope, cancellation, dedupe, cleanup, and failure semantics, then expose the smallest approved core or plugin-facing configuration and validate it across representative channels.

## Reproduction Assessment

Yes, from source inspection: current main has reactions, typing, durable inbound acknowledgments, and terminal pre-dispatch handling, but no supported outbound text acknowledgment that continues into normal agent processing.

## Solution Assessment

No, the proposed config fields alone are not yet the best proven solution; maintainers should first choose the canonical lifecycle and ownership boundary so the implementation does not duplicate channel behavior or create competing delivery semantics.
