## Summary

Keep open. Current main and the latest shipped tag still lack a supported cross-channel text acknowledgment before agent processing; the existing ack/status reactions, typing indicators, and terminal `before_dispatch` hook do not provide the requested send-and-continue text contract.

## What This Changes

The issue requests a configurable pre-processing text acknowledgment such as `messages.ackText`/`ackTextScope` for messaging channels.

## Best Possible Solution

Define a supported cross-channel acknowledgment-text contract or non-terminal preprocessing hook, then update config types, schema/help, public docs, channel behavior, and focused regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source, schema, docs, release, and related-item inspection give high confidence that current main lacks the requested supported text-acknowledgment contract.

## Solution Assessment

Unclear pending product/API decision. The proposed `messages.ackText` shape is plausible, but maintainers should choose the cross-channel config or hook boundary before implementation.
