## Summary

Keep open: the PR targets a real source-reproducible fake-IP model transport failure, but the current diff applies the fake-IP SSRF exemption to every model fetch target and redirect instead of scoping it to the configured provider host.

## What This Changes

The PR adds a constant fake-IP SSRF policy to model provider guarded fetch, adds provider transport policy tests, and records a changelog fix.

## Best Possible Solution

Use a host-scoped fake-IP exemption derived from `model.baseUrl`, while preserving explicit `allowPrivateNetwork` only for operator-trusted private endpoints.

## Reproduction Assessment

Yes, from source inspection. Current main sends model provider requests through `fetchWithSsrFGuard` without RFC2544 or IPv6 ULA flags unless private networking is explicitly enabled, while the SSRF guard blocks those DNS answers by default.

## Solution Assessment

No. The PR targets the right transport surface, but the current policy is not host-scoped; a narrower provider-host fake-IP exemption is the safer maintainable direction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Scope fake-IP exemptions to the provider host:** `src/agents/provider-transport-fetch.ts:316-320`
  - body: The same policy is passed to `fetchWithSsrFGuard` for every model fetch, and the guard reuses `params.policy` while following redirects. That means SDK-supplied URLs or cross-origin redirects whose DNS resolves to 198.18/fc00 now pass the resolved-IP check even when they are not the configured provider host; please scope the fake-IP exemption to `model.baseUrl` or an equivalent provider-host policy.
  - confidence: 0.86
