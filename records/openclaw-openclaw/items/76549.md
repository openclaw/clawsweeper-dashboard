## Summary

Keep open. The fake-IP model-transport failure is source-reproducible, and the updated PR addresses the earlier changelog and broad `allowedHostnames` issues, but the current patch still applies fake-IP exemptions to every model fetch URL and redirect instead of scoping them to the configured provider host.

## What This Changes

The PR changes model provider HTTP fetch policy to exempt RFC2544 and IPv6 ULA fake-IP DNS ranges, adds provider transport regression tests, and records the user-facing fix in the changelog.

## Best Possible Solution

Scope the fake-IP-only SSRF exemption to the configured model provider base host while preserving explicit `allowPrivateNetwork` for operator-trusted private endpoints.

## Reproduction Assessment

Yes, from source inspection. Current main sends model provider requests through `fetchWithSsrFGuard` without RFC2544 or IPv6 ULA policy flags unless private networking is explicitly enabled, while the SSRF guard blocks those DNS answers by default.

## Solution Assessment

No. The PR targets the right surface and uses the safer fake-IP-only flags, but it should scope that exemption to `model.baseUrl` or an equivalent provider-host policy rather than applying it to every URL and redirect handled by the model fetcher.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Scope fake-IP exemptions to the provider host:** `src/agents/provider-transport-fetch.ts:316-320`
