## Summary

Keep open. The PR targets a real model-provider fake-IP failure path, but the proposed `allowedHostnames` fix broadens the SSRF trust boundary beyond the reported RFC2544/IPv6-ULA proxy case and is missing the required user-facing changelog entry.

## What This Changes

The PR adds a `model.baseUrl`-derived `allowedHostnames` SSRF policy to `buildGuardedModelFetch` and merges it with the existing `allowPrivateNetwork` provider request policy.

## Best Possible Solution

Add a hostname-scoped fake-IP-only provider transport exemption that permits RFC2544 and IPv6 ULA DNS answers without bypassing loopback, RFC1918, link-local, or metadata ranges, then add regression coverage and a changelog entry.

## Reproduction Assessment

Yes, from source inspection. Current main sends model provider requests through the guarded fetch without fake-IP exemptions, the model request schema exposes no fake-IP knobs, and SSRF tests show RFC2544/IPv6 ULA answers are blocked unless explicitly exempted.

## Solution Assessment

No. The PR changes the right `buildGuardedModelFetch` surface, but `allowedHostnames` is broader than the reported fake-IP failure because it skips all resolved-IP private/special-use checks for that hostname.

## Review Findings
