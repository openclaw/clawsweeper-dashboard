## Summary

Keep open. The fake-IP provider-transport failure is source-reproducible, but this PR should not merge as-is because it uses the broad `allowedHostnames` SSRF bypass for every configured model provider hostname, and it also lacks the required user-facing changelog entry.

## What This Changes

The PR adds a model `baseUrl` hostname-derived SSRF policy to `buildGuardedModelFetch` and merges it with the existing `allowPrivateNetwork` policy before guarded provider fetches.

## Best Possible Solution

Add a hostname-scoped fake-IP-only exemption for model provider transport that covers RFC2544 and IPv6 ULA answers while preserving loopback, RFC1918, metadata, and other special-use blocks, then add regression coverage and a changelog entry.

## Reproduction Assessment

Yes. Current main sends model provider requests through the guarded fetch without fake-IP SSRF flags, while the SSRF tests show 198.18.0.0/15 and fc00::/7 are blocked unless explicitly exempted.

## Solution Assessment

No. The PR changes the right model transport surface, but using `allowedHostnames` is broader than the reported fake-IP failure because the current SSRF guard treats it as a complete resolved-IP private/special-use bypass for that hostname.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve resolved-IP checks for provider hostnames:** `src/agents/provider-transport-fetch.ts:316-322`
  - body: The new `allowedHostnames` policy makes `resolvePinnedHostnameWithPolicy` skip all private/special-use DNS answer checks for the configured `model.baseUrl` host. That fixes fake-IP ranges, but it also allows that provider hostname to resolve to RFC1918, loopback, metadata, or other blocked addresses, so please use a narrower fake-IP exemption that keeps ordinary private/metadata answers blocked unless `allowPrivateNetwork` is explicit.
  - confidence: 0.83
- **[P3] Add the provider transport changelog entry:** `src/agents/provider-transport-fetch.ts:316-322`
  - body: This is a user-facing `fix` for model API calls on fake-IP proxy setups, and repo policy requires a `CHANGELOG.md` entry for that surface. The PR currently changes only the transport source file, so release notes would miss the behavior change.
  - confidence: 0.78
