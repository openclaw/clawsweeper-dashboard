## Summary

Keep open. Current main has scoped Gateway/announce idempotency and bounded orphan-recovery tombstones, but it still does not provide a stable sub-agent operation identity for lost-ack recovery or arbitrary downstream side effects; the linked ClawHub skill is useful production guidance, not a replacement for a core reliability contract.

## What This Changes

Add idempotency or recovery identity for sub-agent timeout/retry paths so successful lost-ack POSTs are not repeated as duplicate comments or actions.

## Best Possible Solution

Define a stable sub-agent operation identity that survives timeout/recovery retries, propagate it through OpenClaw-owned side-effecting Gateway/tool calls, and document where external services must still supply their own idempotency keys.

## Reproduction Assessment

No high-confidence end-to-end reproduction is available from the issue context alone. Source inspection does show the current gap: retries only dedupe when the same key is reused, while spawn and orphan resume paths generate fresh random keys and do not expose a stable operation identity.

## Solution Assessment

Unclear. The proposed directions are plausible, but the best core fix needs maintainer choice between stable run/action identity propagation, endpoint-level dedupe, and plugin/tool guidance for external services.
