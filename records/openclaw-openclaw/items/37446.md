## Summary

Keep open. Current main has exact-key Gateway dedupe and a stable completion-handoff path, but the broader lost-ack duplicate side-effect gap remains because native subagent spawn, orphan resume, and outbound send fallbacks can still use fresh idempotency identities.

## What This Changes

Request stable idempotency/recovery semantics so subagent timeout retries do not repeat already-succeeded side-effecting POST actions.

## Best Possible Solution

Define a stable subagent operation/action identity that survives timeout and recovery retries, propagate it through OpenClaw-owned side-effecting Gateway/tool/outbound paths, and document where external services still need their own idempotency keys.

## Reproduction Assessment

No, not as a high-confidence live lost-ack reproduction in this read-only pass. Source inspection shows the current-main gap: exact-key Gateway dedupe works when the same key is reused, while spawn, orphan recovery, and outbound fallback paths can still mint fresh keys.

## Solution Assessment

Unclear. The proposed idempotency-key, last-N check, and session-scoped dedupe approaches are plausible, but the maintainable fix should be a shared retry/recovery identity contract rather than content matching in one caller.
