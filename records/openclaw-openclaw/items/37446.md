## Summary

Keep open: current main and the shipped message-tool idempotency fix only cover part of the duplicate-send family, while the broader sub-agent logical-action retry identity and committed-result recovery problem remains. The posted ClawHub skill is useful operational guidance, but it does not replace a core recovery contract for side-effecting sub-agent work.

## What This Changes

The issue requests durable idempotency or recovery semantics so sub-agent timeout retries do not repeat already-committed POST-style side effects.

## Best Possible Solution

Define a maintainer-approved logical-action idempotency and committed-result recovery contract for sub-agent side effects, then implement it at the gateway/tool delivery boundary with replay tests and redacted real workflow proof.
