## What This Changes

The issue asks OpenClaw to prevent duplicate side-effecting API posts when sub-agent timeout recovery retries after a successful but unacknowledged POST.

## Best Possible Solution

Define a stable sub-agent operation or action identity that survives timeout and orphan recovery for OpenClaw-owned side effects, and document where external APIs still require their own idempotency keys.
