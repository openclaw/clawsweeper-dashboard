## Summary

Keep open. Current main has exact-key Gateway dedupe and several narrower sub-agent announce fixes, but the broader lost-ack idempotency gap remains source-visible because native spawn and orphan-recovery paths can still mint fresh idempotency identities for retried side effects.

## What This Changes

The issue requests retry-safe idempotency for sub-agent side effects so timeout recovery does not duplicate POST-style actions after a lost acknowledgement.

## Best Possible Solution

Define a stable sub-agent operation/action identity that survives timeout and recovery retries, use it for OpenClaw-owned side-effecting Gateway/tool delivery, and document where external APIs still need their own idempotency keys.

## Reproduction Assessment

No, not as a high-confidence live lost-ack reproduction. Source inspection does show the current-main path: Gateway dedupe is exact-key scoped while native spawn and orphan recovery can still generate fresh keys.

## Solution Assessment

Unclear. The requested idempotency-key approaches are plausible, but the maintainable fix should be a shared retry/recovery identity contract rather than a content match or one-off caller shim.
