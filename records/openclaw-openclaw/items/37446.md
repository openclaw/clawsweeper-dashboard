## Summary

Keep open. Current main has scoped Gateway, announce, and hook dedupe, but the sub-agent spawn/recovery path still lacks a stable retry identity for side-effecting work; the linked ClawHub skill is useful operational guidance, not a replacement for a core reliability contract.

## What This Changes

Request stable idempotency or recovery semantics so sub-agent timeout recovery does not repeat successful side-effecting POST actions.

## Best Possible Solution

Define and document a stable sub-agent operation or action identity that survives timeout/recovery retries, propagate it through OpenClaw-owned side-effecting paths, and document where external services must still provide their own idempotency keys.

## Reproduction Assessment

No high-confidence end-to-end reproduction is available from the issue context alone. Source inspection does show the current gap: current main dedupes reused Gateway agent keys, while sessions_spawn and orphan resume create fresh random identities.

## Solution Assessment

Unclear. The proposed directions are plausible, but maintainers need to choose the core contract between stable run/action identity propagation, endpoint-level dedupe, and plugin/tool guidance for external services.
