## Summary

Keep open. Current main has same-key Gateway dedupe and a fixed sub-agent announce storm path, but the broader lost-ack problem remains source-visible because several sub-agent/recovery/outbound paths can still mint a fresh idempotency identity for a retried side effect.

## What This Changes

The issue requests reliable idempotency for sub-agent timeout recovery so successful-but-unacknowledged side-effecting POST-like actions are not replayed as duplicates.

## Best Possible Solution

Define a stable sub-agent operation/action identity that survives timeout and recovery retries, propagate it through OpenClaw-owned side-effecting Gateway/tool paths, and document where external services still need their own idempotency keys.

## Reproduction Assessment

No, not as a high-confidence live lost-ack reproduction. Source inspection shows the current-main gap: Gateway dedupe works for the same key, while spawn, orphan recovery, and outbound fallback paths can still generate fresh keys.

## Solution Assessment

Unclear. The proposed idempotency-key, last-N check, and session-scoped dedupe approaches are plausible, but the maintainable fix should be a shared retry/recovery contract rather than content matching in one caller.
