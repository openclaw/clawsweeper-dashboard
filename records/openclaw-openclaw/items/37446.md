## Summary

Keep open. Current main and v2026.5.20 still have exact-key Gateway idempotency while subagent spawn, control, and orphan-recovery paths mint fresh keys, so the lost-ack duplicate side-effect gap remains and needs a maintainer contract decision.

## What This Changes

The issue asks for stable idempotency across subagent timeout/recovery retries so successful-but-unacknowledged POST side effects are not repeated.

## Best Possible Solution

Define a stable subagent operation identity that survives spawn retry, steer/restart, and orphan resume for OpenClaw-owned side effects, and document where external APIs must still provide their own idempotency keys.
