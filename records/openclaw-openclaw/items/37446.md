## Summary

Keep open. Current main still has same-key Gateway dedupe, but several sub-agent and recovery paths can mint fresh idempotency identities, so the lost-ack duplicate side-effect problem is not fully resolved by the existing implementation or by the linked ClawHub workaround.

## What This Changes

The issue requests an idempotency or recovery contract that prevents sub-agent timeout retries from repeating already-successful side-effecting POST/API actions.

## Best Possible Solution

Define a stable sub-agent operation/action identity that survives timeout and recovery retries, propagate it through OpenClaw-owned side-effecting Gateway/tool paths, and document where external services still require their own idempotency keys.

## Reproduction Assessment

No, not as a high-confidence live lost-ack reproduction. Source inspection does show the current-main gap: Gateway dedupe works for the same key, while spawn/recovery/message-action paths can still generate fresh keys.

## Solution Assessment

Unclear. The proposed idempotency-key, last-N check, and session-scoped dedupe approaches are plausible, but the maintainable fix should be a shared retry/recovery contract rather than content matching in one caller.
