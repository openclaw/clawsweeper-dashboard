## What This Changes

Request to prevent duplicate side-effecting API posts when sub-agent timeout recovery retries work whose first POST may have succeeded but lost its acknowledgement.

## Best Possible Solution

Define a stable sub-agent operation or action identity that survives timeout and recovery retries for OpenClaw-owned side effects, and document where external APIs still require their own idempotency keys.
