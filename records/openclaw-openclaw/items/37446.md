## Summary

Keep open: current main and v2026.6.10 have useful partial idempotency coverage for Gateway/message-tool sends, but they still do not define the durable logical sub-agent action identity needed to prevent duplicate external/API side effects after a successful commit with a lost acknowledgement.

## What This Changes

The issue requests durable idempotency for sub-agent timeout recovery so retried POST or side-effect actions do not duplicate after lost acknowledgements.

## Best Possible Solution

Define a maintainer-approved logical-action idempotency and replay contract for OpenClaw-owned sub-agent side effects, then implement it through the narrow subagent/Gateway/tool owner paths with regression and real retry proof.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection gives a medium-confidence path: sessions_spawn has no logical idempotency input, sub-agent spawns create fresh child keys, and Gateway dedupe only works when the same key is supplied.

## Solution Assessment

No, current main is only a partial mitigation. The best fix needs a durable logical-action identity and replay/claim semantics before changing retry behavior broadly.
