## Summary

Keep open. Current main still has only scoped same-key Gateway/send dedupe; native sub-agent spawn and orphan recovery generate fresh identities, so timeout/recovery replay safety still needs a maintainer-owned reliability contract.

## What This Changes

Request a stable idempotency mechanism so sub-agent timeout recovery does not repeat side-effecting POST/API actions after a lost acknowledgement.

## Best Possible Solution

Define and document a stable sub-agent operation/action identity that survives timeout and recovery retries, then propagate it through OpenClaw-owned side-effecting Gateway/tool paths while documenting where external services still need their own idempotency keys.

## Reproduction Assessment

No, not as a high-confidence end-to-end lost-ack reproduction. Source inspection does show the gap: same-key Gateway dedupe exists, while native spawn and recovery paths still generate fresh keys.

## Solution Assessment

Unclear. The proposed idempotency-key, last-N check, and session-scoped dedupe approaches are plausible, but maintainers need to choose the shared retry/recovery contract before implementation.
