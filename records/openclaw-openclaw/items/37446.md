## Summary

Keep open. Current main has exact-key Gateway dedupe and stable keys for subagent completion announces, but native subagent spawn, steer, and orphan recovery still create fresh agent idempotency keys, so the reported lost-ack duplicate side-effect gap is not fully resolved.

## What This Changes

The issue requests stable idempotency or recovery identity for sub-agent timeout retries so successful side-effecting API posts are not repeated after an acknowledgement is lost.

## Best Possible Solution

Define a stable subagent operation identity that survives spawn retry, steer/restart, and orphan resume for OpenClaw-owned side effects, and document where external APIs must still provide their own idempotency keys.

## Reproduction Assessment

No, not with a high-confidence live lost-ack reproduction. Source inspection does show that Gateway dedupe is exact-key scoped while subagent spawn, steer, and orphan recovery can still generate fresh keys.

## Solution Assessment

Unclear. The proposed idempotency-key directions are plausible, but the maintainable fix should be a shared retry/recovery identity contract rather than caller-local content matching or a narrow last-N check.
