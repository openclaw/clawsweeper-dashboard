## Summary

Keep open. Current main and the latest release still have exact-key Gateway dedupe while sub-agent spawn/resume paths can mint fresh identities, so the central durable operation-identity gap is not implemented; the related ClawHub skill and earlier announce-storm fix are useful context but do not close this core contract question.

## What This Changes

The issue requests durable idempotency for sub-agent timeout/recovery retries so successful side-effecting POSTs are not repeated after a lost acknowledgement.

## Best Possible Solution

Define a stable sub-agent operation/action identity that survives timeout and recovery retries for OpenClaw-owned side effects, and document where external APIs still require their own idempotency keys.

## Reproduction Assessment

No, not with a high-confidence live lost-ack reproduction. Source inspection does show the current-main path: Gateway dedupe is exact-key scoped while native spawn and orphan recovery can generate fresh keys.

## Solution Assessment

Unclear. The proposed idempotency-key approaches are plausible, but the maintainable fix should be a shared retry/recovery identity contract rather than a one-off content match or caller shim.
