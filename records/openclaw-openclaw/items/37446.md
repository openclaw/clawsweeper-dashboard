## Summary

Keep open. Current main has exact-key Gateway idempotency and some outbound/message dedupe work, but the sub-agent retry/recovery paths still mint fresh operation keys, so the reported lost-ack duplicate side-effect gap is not resolved.

## What This Changes

The issue requests idempotent sub-agent timeout recovery so retries after lost acknowledgements do not repeat side-effecting API posts.

## Best Possible Solution

Define a stable sub-agent operation identity that survives spawn retry, steer/restart, and orphan resume for OpenClaw-owned side effects, and document where external APIs must still provide their own idempotency keys.

## Reproduction Assessment

No high-confidence live lost-ack reproduction is attached. Source inspection does show the current-main path: Gateway dedupe is exact-key scoped while sub-agent spawn, control, and orphan recovery can still generate fresh keys.

## Solution Assessment

Unclear. The proposed idempotency-key approaches are plausible, but the maintainable fix should be a shared retry/recovery identity contract rather than one-off last-N or content matching.
