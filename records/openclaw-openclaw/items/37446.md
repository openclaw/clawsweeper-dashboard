## Summary

Keep open: current main has exact-key gateway dedupe and a merged message-tool partial mitigation, but sub-agent spawn still creates a fresh per-attempt key and there is no durable logical-action claim/committed-result recovery for sub-agent side effects after a lost acknowledgement.

## What This Changes

The issue requests idempotent timeout recovery for sub-agent POST side effects so a committed-but-unacknowledged action is not posted again on retry.

## Best Possible Solution

Define a core logical-action identity and committed-result recovery contract for sub-agent side effects, then cover gateway, message-tool, and subagent retry paths with focused tests plus live workflow proof.

## Reproduction Assessment

No live high-confidence reproduction was established in this read-only pass. Source inspection does show the core gap: gateway dedupe requires the same idempotency key, while sub-agent spawn and successful replay paths can create fresh identities.

## Solution Assessment

Unclear as an implementation plan: idempotency keys are the right primitive, but maintainers still need to choose the logical-action identity, result replay semantics, and expiry before a safe fix lands.
