## Summary

Keep open: current main and v2026.6.5 only cover the narrower `message(action=send)` lost-ack retry path when the same logical key is reused; they do not provide the broader sub-agent logical-action idempotency contract for replacement runs or arbitrary external POST side effects.

## What This Changes

The issue asks OpenClaw to prevent duplicate side effects when sub-agent timeout recovery retries a POST that may already have committed.

## Best Possible Solution

Define a narrow logical-operation identity for supported sub-agent recovery paths, persist and propagate it through side-effecting tool contracts, and require each owner to provide durable claim/result semantics or documented preflight checks.

## Reproduction Assessment

No high-confidence reproduction for the arbitrary external POST case. Source inspection shows why the broader contract is still missing, but the report does not identify a concrete current-main tool, endpoint, command, or controlled lost-ack sequence for that path.

## Solution Assessment

No, the proposed options are only partial. The best direction is a scoped logical-operation contract with durable server/tool-owner cooperation, not a universal transport header or last-N heuristic.
