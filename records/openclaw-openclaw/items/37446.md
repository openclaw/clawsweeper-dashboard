## Summary

Keep open. Current main still dedupes Gateway agent calls by exact idempotency key while subagent spawn, control, and orphan-recovery paths mint fresh keys, so the reported lost-ack duplicate side-effect gap is not clearly fixed and needs a maintainer contract decision.

## What This Changes

The issue asks OpenClaw to prevent sub-agent timeout/retry recovery from reposting side-effecting API calls by carrying a durable idempotency identity.

## Best Possible Solution

Define a stable subagent operation identity that survives spawn retry, steer/restart, and orphan resume for OpenClaw-owned side effects, and document where external APIs must still provide their own idempotency keys.

## Reproduction Assessment

No high-confidence live reproduction is attached. Source inspection gives a medium-confidence current-main path: Gateway dedupe is exact-key scoped while subagent spawn, control, and orphan recovery can still generate fresh keys for the same logical recovery flow.

## Solution Assessment

Unclear. The proposed idempotency-key approaches are plausible, but maintainers should choose a shared retry/recovery identity contract rather than one-off last-N or content matching.
