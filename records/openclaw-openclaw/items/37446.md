## Summary

Keep open. Current main still has exact-key Gateway dedupe, but sub-agent spawn and orphan-resume paths can mint fresh idempotency keys, so the reported lost-ack/retry identity gap is not implemented or safely closeable.

## What This Changes

The issue requests stable idempotency or recovery identity for sub-agent timeout retries so already-succeeded side-effecting API posts are not repeated.

## Best Possible Solution

Define a stable sub-agent operation or action identity that survives timeout and orphan recovery for OpenClaw-owned side effects, and document where external APIs still require their own idempotency keys.

## Reproduction Assessment

No, not with a high-confidence live lost-ack reproduction. Source inspection does show the current-main path: Gateway dedupe is exact-key scoped while native spawn and orphan recovery can still generate fresh keys.

## Solution Assessment

Unclear. The proposed idempotency-key approaches are plausible, but the maintainable solution should be a shared retry/recovery identity contract rather than a one-off content match or caller shim.
