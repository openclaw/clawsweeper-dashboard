## Summary

Keep open. Current main has same-key Gateway dedupe and replay-invalid guards for some known successful mutating tool paths, but sub-agent spawn/recovery and message-action fallback paths still mint fresh identities and do not provide a stable lost-ack idempotency contract for external POST side effects.

## What This Changes

The issue asks OpenClaw to prevent sub-agent timeout or lost-ack recovery from re-POSTing the same side-effecting API action.

## Best Possible Solution

Define a stable sub-agent operation or action identity that survives timeout and recovery retries, propagate it through OpenClaw-owned side-effecting Gateway/tool paths, and document where external APIs still need their own idempotency keys.

## Reproduction Assessment

No, not as a high-confidence live lost-ack reproduction. Source inspection shows same-key dedupe exists, while several sub-agent and message-action recovery paths still generate fresh keys when the caller did not provide a stable one.

## Solution Assessment

Unclear. The proposed idempotency-key, last-N check, and session-scoped dedupe approaches are plausible, but the maintainable fix should be a shared retry/recovery contract rather than ad hoc content matching in one path.
