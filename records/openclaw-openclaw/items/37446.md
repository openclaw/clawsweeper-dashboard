## Summary

Keep open. Current main has Gateway idempotency and exact-key dedupe, plus the linked ClawHub multi-agent deployment skill is useful operational guidance, but the core sub-agent recovery paths still generate fresh operation keys, so the lost-ack duplicate side-effect gap remains unresolved.

## What This Changes

The issue asks OpenClaw to prevent duplicate side-effecting sub-agent API posts after timeout recovery by carrying stable idempotency or recovery identity.

## Best Possible Solution

Define a stable sub-agent operation identity that survives spawn retry, steer/restart, and orphan resume for OpenClaw-owned side effects, and document where external APIs must still provide their own idempotency keys.

## Reproduction Assessment

No, not with a high-confidence live lost-ack reproduction. Source inspection does show the current-main path: Gateway dedupe is exact-key scoped while sub-agent spawn, steer, and orphan recovery can still generate fresh keys.

## Solution Assessment

Unclear. The proposed idempotency-key approaches are plausible, but the maintainable solution should be a shared retry/recovery identity contract rather than a one-off content match or caller shim.
