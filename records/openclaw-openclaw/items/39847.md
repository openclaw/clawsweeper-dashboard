## Summary

Keep open: current main and v2026.5.12 still parse outbound reply text before any inbound-metadata stripping, and the open closing PR owns the fix path but remains unmerged.

## What This Changes

Strip model-echoed inbound metadata before outbound channel delivery so internal context JSON does not become visible chat text.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/50520 or a clean replacement that strips leading inbound metadata at the shared outbound payload planner, preserves directive/media parsing, and includes sanitizer/outbound regression tests plus real channel proof.

Yes. A high-confidence source reproduction is to pass `normalizeReplyPayloadsForDelivery` a payload beginning with generated inbound metadata; current main sends that text into `parseReplyDirectives` before any inbound metadata stripping.

## Solution Assessment

Yes. The shared outbound payload planner before directive parsing is the narrow maintainable fix point; channel-specific compiled-bundle patches would leave other delivery paths uncovered.
