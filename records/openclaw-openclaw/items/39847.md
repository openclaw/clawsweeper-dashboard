## Summary

Keep open: current main and v2026.6.6 still parse raw outbound payload text before inbound-metadata stripping, and the canonical repair at https://github.com/openclaw/openclaw/pull/50520 is still open and conflicting.

## What This Changes

The issue requests stripping echoed inbound metadata before outbound delivery so internal context blocks cannot be sent to channel users.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/50520, or replace it with an equivalent narrow source-level outbound sanitizer fix that keeps directive parsing and metadata stripping covered by tests.

## Reproduction Assessment

Yes, at source level: current main still feeds raw payload text into parseReplyDirectives, and downstream outbound sanitization does not remove inbound metadata blocks. I did not run a live Discord repro because this review is read-only.

## Solution Assessment

Yes: stripping leading inbound metadata at the shared outbound payload-plan boundary before directive parsing is the narrow maintainable fix because delivery projections flow through that plan. A Discord-only compiled-bundle patch would miss sibling delivery paths and be overwritten by updates.
