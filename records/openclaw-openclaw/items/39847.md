## Summary

Keep open: current main and the latest release still parse outbound reply text without inbound metadata stripping, and the canonical closing PR remains open instead of merged.

## What This Changes

The issue asks OpenClaw to strip echoed inbound metadata before outbound channel delivery so internal conversation and sender context cannot become visible messages.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/50520 or a clean replacement that strips generated leading inbound metadata at the shared outbound payload planner before directive parsing, with focused regression tests and redacted real channel proof.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to pass `createOutboundPayloadPlan` or `normalizeReplyPayloadsForDelivery` a text payload beginning with generated inbound metadata; current main parses that text before inbound metadata stripping.

## Solution Assessment

Yes. The shared outbound payload planner before directive parsing is the narrow maintainable fix point; channel-specific or compiled-bundle patches leave other delivery paths uncovered.
