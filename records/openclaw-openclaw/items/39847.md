## What This Changes

The issue requests stripping echoed OpenClaw inbound metadata from outbound delivery so internal context is not sent to chat channels.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/50520 or a replacement that strips leading inbound metadata at the shared outbound payload plan boundary, covers generated metadata labels, and preserves directive/media parsing.

Yes. The shared outbound payload planner is the narrow maintainable fix point; per-channel or earlier partial sanitizers leave other planner callers and generated-label cases uncovered.
