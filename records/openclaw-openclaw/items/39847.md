## What This Changes

Strip echoed inbound metadata from outbound reply payload text before channel delivery and directive parsing so internal context is not visible in Discord or other channels.

## Best Possible Solution

The end state should be a narrow source-level outbound-plan sanitizer with regression coverage, landed through the existing linked PR or an equivalent replacement.

## Reproduction Assessment

## Solution Assessment

Yes. Stripping leading inbound metadata at the shared outbound payload-plan boundary before directive parsing is the narrow maintainable fix; adapter-specific or compiled-bundle patches would miss sibling paths and be overwritten by updates.
