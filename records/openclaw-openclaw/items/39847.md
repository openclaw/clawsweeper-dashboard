## Summary

Keep open: current main still sends outbound payload text through reply-directive parsing and channel delivery without calling the existing inbound metadata stripper, and open PR #50520 is already the canonical closing fix candidate.

## What This Changes

Strip echoed inbound metadata blocks at the shared outbound delivery normalization boundary before channel sends.

## Best Possible Solution

Land one canonical source-level fix, likely #50520, that sanitizes outbound text at the shared payload-plan boundary and adds regression coverage for fixed and generated metadata labels plus directives/media.

Yes, from source inspection: a payload containing `Conversation info (untrusted metadata):` plus fenced JSON reaches `createOutboundPayloadPlanEntry`, `parseReplyDirectives`, and channel delivery projection without `stripInboundMetadata`.
