## Summary

Keep open: current main and v2026.5.12 still parse outbound reply text before any inbound-metadata stripping, while the open closing PR remains unmerged and should carry the fix path.

## What This Changes

Strip AI-facing inbound metadata from outbound payload normalization before channel delivery so Discord and other channels cannot display echoed internal context.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/50520 or a clean replacement that strips leading inbound metadata at the shared outbound payload planner, preserves directive and media parsing, and covers generated metadata labels with tests plus real channel proof.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to pass normalizeReplyPayloadsForDelivery a payload beginning with generated inbound metadata; current main sends that raw text into parseReplyDirectives before any inbound metadata stripping.

## Solution Assessment

Yes. The shared outbound payload planner before directive parsing is the narrow maintainable fix point; channel-specific compiled-bundle patches would leave other delivery paths uncovered.
