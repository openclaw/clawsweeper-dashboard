## Summary

Keep open: current main and v2026.6.6 still parse outbound reply payload text without inbound-metadata stripping, and the open linked repair PR has not merged.

## What This Changes

The issue asks OpenClaw to strip echoed inbound metadata from outbound reply payloads before channel delivery parses or sends them.

## Best Possible Solution

Refresh and land the canonical repair at https://github.com/openclaw/openclaw/pull/50520, keeping the fix at the outbound payload-plan boundary with focused regression coverage, then close this issue after merge.

## Reproduction Assessment

Yes at source level: current main parses outbound `payload.text` before any inbound-metadata stripping and then projects that text for delivery. I did not run tests because this cleanup review was read-only.

## Solution Assessment

Yes, the best fix shape is to strip leading inbound metadata in `src/infra/outbound/payloads.ts` before directive parsing, not in individual channel adapters. The linked PR follows that boundary but needs refresh, review, and proof before landing.
