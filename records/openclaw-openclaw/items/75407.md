## Summary

Keep open. Current main only partially mitigates restricted Telegram networks with sticky IPv4 and one pinned-IP retry; the requested configurable or discoverable fallback IP list is still absent and needs maintainer egress-policy direction.

## Best Possible Solution

Keep the work in the Telegram plugin after maintainer-approved network policy; if approved, add validated fallback-IP configuration or discovery, preserve TLS/SNI and proxy precedence, document it, and cover configured-IP plus timeout-forced fallback paths with focused tests.
