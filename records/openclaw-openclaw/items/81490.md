## Summary

Keep open: current `main` still lacks a durable paused-session route binding, so a yielded parent can remain inactive while completion delivery falls back to an `agent` handoff on the route and risks creating a new run.

## What This Changes

The issue requests that subagent completion handoffs resume the yielded parent session on the same external route instead of spawning a fresh run and overwriting the route session pointer.

## Best Possible Solution

Persist a bounded yielded-session binding on the route entry, or an equivalent resolver contract, and make completion handoffs prefer that paused parent before normal route delivery can allocate a new run.

## Reproduction Assessment

Yes. The source path is high-confidence: yielded parents are marked paused/inactive, current route entries have no paused-session binding, inactive embedded sessions are not queueable, and completion delivery can fall through to a route `agent` call.

## Solution Assessment

Yes. The narrow maintainable direction is to preserve and resolve the yielded route/session association; queue-first completion routing alone does not address the route pointer overwrite reported here.
