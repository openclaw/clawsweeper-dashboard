## Summary

Keep open: current main still has a credible source-backed gap between the documented `sessions_yield` contract and completion routing for yielded parents, and the related open draft PR does not prove or directly fix the paused route-to-session binding described here.

## What This Changes

The issue asks for subagent completion announcements to resume the yielded parent session on the same external route instead of starting a fresh route run and overwriting the session-store pointer.

## Best Possible Solution

Persist a bounded yielded/paused session binding for route entries and make completion or inbound inter-session delivery prefer that session before normal dormant direct fallback.

## Reproduction Assessment

Yes, source-reproducible with high confidence. The reporter gives live transcript/store evidence, and current main shows `sessions_yield` parks the parent while completion delivery can still fall through to direct route handoff without a durable paused-session binding.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve and resolve a bounded paused-session binding for yielded route entries, while keeping existing direct fallback for truly dormant sessions.
