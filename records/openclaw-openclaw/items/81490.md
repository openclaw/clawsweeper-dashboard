## Summary

Keep open: current main still lacks a durable paused-session binding for yielded parent routes. The reporter's Telegram/Bedrock evidence matches the current source path: `sessions_yield` marks the run paused, inactive embedded runs are not queueable, and completion delivery can fall through to a direct gateway agent handoff that resolves the route without knowing it should resume the parked session. The adjacent queue-first work in https://github.com/openclaw/openclaw/pull/76927 and the merged origin-routing fix in https://github.com/openclaw/openclaw/pull/80242 do not cover the missing paused route/session reuse contract.

## What This Changes

The issue requests that subagent completion handoffs resume a yielded parent session on its external route instead of starting a fresh run and overwriting the route session entry.

## Best Possible Solution

Persist a bounded paused-session binding for yielded route entries, make completion delivery and gateway session reuse honor it before direct fallback can allocate a new session, and clear or expire the marker after wakeup.

## Reproduction Assessment

Yes, at source level and with reporter-provided live evidence. Current main exits `sessions_yield` as a paused inactive run, queueing that session returns `no_active_run`, and the completion path can then use direct route handoff without a paused-session binding.

## Solution Assessment

Yes. A generic bounded paused-session binding is the narrow maintainable fix; a Telegram-specific special case or queue-order-only change would not address the route/session pointer overwrite.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/81490. Current main can spawn a fresh route run instead of resuming a yielded parent after subagent completion because yielded sessions are marked paused/inactive but the route store does not persist or resolve a bounded paused session binding. Implement a generic, non-Telegram-specific fix: when an embedded run exits from `sessions_yield` with `livenessState: "paused"`/`yielded: true`, persist a bounded paused-for-subagent-completion marker on the route/session entry; have subagent completion activity resolution and gateway session reuse prefer that existing session while the marker is valid; clear the marker after successful wake and prune/ignore it after expiry. Preserve direct fallback for truly dormant non-paused requester sessions, do not add a public config option, and account for adjacent completion queue work in https://github.com/openclaw/openclaw/pull/76927.
