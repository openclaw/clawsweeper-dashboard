## Summary

Keep open: current main and v2026.6.8 still route yielded subagent completions through active-run steering or direct requester-agent handoff, and neither source nor tests show a durable paused-session route binding that resumes the yielded parent session id.

## What This Changes

The issue asks OpenClaw to resume the yielded parent session for subagent completion announcements instead of starting a fresh run on the same external route.

## Best Possible Solution

Land a maintainer-owned yielded-session resume contract that durably binds the external route to the paused parent session and delivers the child completion as that session's next event.

## Reproduction Assessment

Yes at source level: current main still makes yielded requesters inactive for embedded steering and then falls through to a route-key direct agent handoff. I did not run a live current-main Telegram/Bedrock reproduction.

## Solution Assessment

No, current main is not the best solution because it has no durable paused-session route binding and can hand completion to a fresh requester-agent run. The safer fix is a shared lifecycle contract for yielded parent resumption, not another one-off direct fallback.
