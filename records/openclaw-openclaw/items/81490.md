## Summary

Keep open: current main still has a credible gap between the documented `sessions_yield` contract and route/session resolution after a yielded parent becomes inactive. The related open draft at https://github.com/openclaw/openclaw/pull/76927 improves completion queueing, but this issue describes a distinct route-to-paused-session binding problem that is not implemented on main.

## What This Changes

The issue requests that subagent completion announces resume the yielded parent session for an external route instead of starting a fresh run that overwrites the route's session-store entry.

## Best Possible Solution

Persist and resolve an explicit bounded paused-session binding for yielded route entries, then route completion/inbound inter-session handoffs to that paused session before normal dormant direct fallback.

## Reproduction Assessment

Yes, at source level and with reporter-provided live evidence. Current main marks yielded parents as paused/inactive, completion wake fails for inactive embedded runs, and direct fallback can allocate a fresh route session when the stored session is no longer fresh.

## Solution Assessment

Yes, the paused-session route binding direction is the narrowest maintainable fix. Queue-first completion routing alone is not sufficient for the reported route/session-store overwrite case.

## Repair Work Prompt

Fix the yielded-parent subagent completion route bug reported at https://github.com/openclaw/openclaw/issues/81490. Current main can treat a completion announce for an inactive yielded requester as a dormant direct handoff; after session freshness expires, gateway route resolution may allocate a fresh session id and orphan the paused parent. Implement a bounded persisted paused-session binding or equivalent canonical route resolver behavior so a run that exits with `livenessState: "paused"` and `yielded: true` can be resumed by completion/inbound inter-session delivery before normal dormant direct fallback. Preserve existing behavior for truly dormant completed sessions, including the direct external delivery coverage in `src/agents/subagent-announce-delivery.test.ts`. Add focused regression coverage for route/session-store reuse after `sessions_yield`, and update docs or changelog if the user-visible contract text changes. Do not merge or depend on the draft queue-first PR unless the final patch also proves the paused route binding case.
