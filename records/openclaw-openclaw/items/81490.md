## Summary

Keep open: current main still lacks a durable paused-session route binding for yielded external parents. The reporter’s live evidence matches the source path: yielded runs are marked paused/inactive, inactive embedded runs cannot be queued, and the gateway route resolver can allocate a new session when the stored route session is no longer considered reusable.

## What This Changes

Ensure subagent completion handoffs resume a yielded external parent session instead of creating a new session on the same route.

## Best Possible Solution

Persist and honor a bounded paused-session binding for yielded route entries, then route subagent completion handoffs to that paused session before normal freshness-based direct fallback can create a new run.

## Reproduction Assessment

Yes, at source level and with reporter-provided live evidence. Current main marks yielded parents paused/inactive, inactive embedded queues return `no_active_run`, and gateway direct fallback can allocate a fresh route session when the stored entry is no longer fresh.

## Solution Assessment

Yes, the suggested paused-route/session binding is the narrow maintainable fix. Queue-first completion routing alone does not solve the route pointer overwrite when the yielded parent is inactive or freshness-expired.

Implement a bounded paused-session binding or equivalent canonical resolver owned by the session store/gateway/announce-delivery path: when an embedded run exits with `livenessState: "paused"` and `yielded: true`, persist enough route/session state to let subagent completion delivery target that paused session before freshness-based direct fallback allocates a new session.
