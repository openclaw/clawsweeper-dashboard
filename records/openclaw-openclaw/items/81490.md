## Summary

Keep open: current main still has a source-backed gap between the documented `sessions_yield` contract and completion announce routing for inactive yielded parents. The adjacent draft at https://github.com/openclaw/openclaw/pull/76927 changes completion queue ordering, and the closed swallowed-error fix at https://github.com/openclaw/openclaw/issues/82306 addresses a different cleanup failure path, but neither provides a durable route-to-paused-session binding for this issue.

## What This Changes

The issue reports that subagent completion announces can start a fresh route run instead of resuming the yielded parent session waiting on `sessions_yield`.

## Best Possible Solution

Persist a bounded paused-session binding for yielded route entries and resolve completion or inbound inter-session delivery to that paused session before normal dormant direct fallback.

## Reproduction Assessment

Yes. The reporter gives concrete live transcript and session-store evidence, and current main still shows yielded parents become inactive while completion dispatch can take a direct route handoff without a persisted paused-session binding.

## Solution Assessment

Yes. A bounded paused-session route binding is the narrowest maintainable fix because it preserves direct fallback for dormant sessions while making `sessions_yield` completion delivery honor the existing wait contract.
