## Summary

Keep open: current main still has the reported gap between the documented `sessions_yield` completion contract and route/session reuse for a yielded external parent. The reporter's live Telegram/Bedrock evidence matches current source: yielded runs are marked paused, inactive embedded runs cannot be queued, and the direct gateway path can allocate a new session for the same route.

## What This Changes

The issue asks OpenClaw to resume the yielded parent session for subagent completion handoffs instead of starting a fresh run on the parent's external route.

## Best Possible Solution

Persist and resolve an explicit bounded paused-session binding for yielded route entries, then route subagent completion handoffs to that session before normal direct fallback can create a fresh run.

## Reproduction Assessment

Yes, at source level and with reporter-provided live evidence. Current main marks yielded parents as paused/inactive, inactive embedded runs cannot be queued, and direct route fallback can allocate a new session id when the stored route session is not reusable.

## Solution Assessment

Yes. The narrow maintainable fix is a yielded-route/session binding or equivalent paused-session resolver; queue-first completion routing alone does not address the reported route pointer overwrite.
