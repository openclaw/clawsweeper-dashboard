## Summary

Keep open: current main still lacks a durable paused-session binding for a yielded external-route parent, so subagent completion delivery can still fall back into a normal `agent` call that may create or overwrite the route session instead of resuming the yielded parent. The reporter's live Telegram/Bedrock evidence is consistent with the current source, and the later proposed fix commits referenced on the issue are not contained in current main or any local branch/tag.

## What This Changes

Ensure subagent completion announces resume the yielded parent session bound to an external route instead of creating a fresh route session.

## Best Possible Solution

Persist a bounded paused-session binding for yielded route entries and have completion delivery/gateway route resolution prefer that session before normal direct fallback can allocate a fresh run.

## Reproduction Assessment

Yes. Current main shows the documented yielded-wait contract, the paused/inactive embedded parent state, no durable paused-session route binding, and a direct fallback path that can allocate a new session; the reporter also provided live Telegram/Bedrock transcript and store evidence.

## Solution Assessment

Yes. A bounded paused-session binding or equivalent resolver is the narrow maintainable fix; queue-first completion routing alone does not address the reported route pointer overwrite.
