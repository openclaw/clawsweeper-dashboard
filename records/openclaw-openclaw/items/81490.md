## Label Justifications

- `P1`: The report describes a broken subagent orchestration workflow where completed child work can orphan or misroute the yielded parent session for real external-channel users.
- `impact:session-state`: The central failure is route/session pointer drift that can leave the yielded parent session orphaned while another run owns the route.
- `impact:message-loss`: The expected parent continuation or reply can be missed or misdelivered when the completion handoff enters the wrong session.

## Summary

Keep open: the report describes a supported core sessions_yield/subagent completion workflow, current main does not provide enough live or provenance-backed proof to close the exact route-to-fresh-run symptom as fixed, and related yielded-parent delivery/session-state work remains open for maintainer judgment.

## What This Changes

The issue asks OpenClaw to resume the yielded parent session for subagent completion handoffs on the original external route instead of starting a new run and overwriting the route's session pointer.

## Best Possible Solution

Define and test one yielded-parent completion wake contract that preserves the requester route/session binding, re-enters the intended parent deterministically, and avoids competing raw child fallback delivery paths.

## Reproduction Assessment

No high-confidence current-main reproduction was established. The original report has detailed v2026.5.7 live evidence, and the current source path is clear, but main has changed enough that the exact fresh-run symptom needs live or focused regression proof before closure or repair.

## Solution Assessment

Unclear. Persisting an explicit pausedSessionId is plausible, but the better maintainable fix may be a shared yielded-parent completion/wake contract across the open sessions_yield cluster rather than a one-off route field.
