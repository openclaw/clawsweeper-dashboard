## Summary

Keep open: current main still has a credible source-backed gap between the documented `sessions_yield` contract and completion announce routing for inactive yielded parents. The related draft PR at https://github.com/openclaw/openclaw/pull/76927 addresses completion queue ordering, but this report adds live route/session-store evidence that the paused route can be rebound to a fresh run instead of the yielded parent session.

## What This Changes

The issue reports that a subagent completion on an external route starts a new parent-route run instead of resuming the yielded parent session.

## Best Possible Solution

Persist and resolve a bounded paused-session binding for yielded route entries, then route completion/inbound inter-session messages to that paused session before normal dormant direct fallback.

## Reproduction Assessment

Yes. The reporter gives concrete live transcript/store evidence, and current source shows the yielded parent becomes inactive while completion delivery can fall through to a direct route run without any durable paused-session binding.

## Solution Assessment

Yes. The suggested paused-session route binding is the narrowest maintainable direction, provided it is bounded and preserves existing direct fallback for truly dormant completed sessions.
