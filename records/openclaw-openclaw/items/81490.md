## Summary

Keep open: current main still has a source-backed gap between the documented `sessions_yield` contract and route/session resolution for yielded parents. The related draft PR improves completion queue ordering, but this report describes a distinct route rebinding failure that current main does not appear to persistently guard against.

## What This Changes

The issue reports that subagent completion announces on an external route start a fresh parent run instead of resuming the yielded parent session.

## Best Possible Solution

Persist a bounded yielded-parent session binding on the route and have completion/inbound inter-session delivery prefer that paused session before normal dormant direct fallback.

## Reproduction Assessment

Yes. The reporter gives concrete live transcript/store evidence, and current main source shows the yielded parent becomes inactive while completion delivery can fall through to a direct route run without a durable paused-session binding.

## Solution Assessment

Yes. A bounded paused-session route binding is the narrowest maintainable fix, as long as it preserves direct fallback for genuinely dormant completed sessions.
