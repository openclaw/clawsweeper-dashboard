## Summary

Keep open: current main still lacks a durable paused-session binding for yielded route sessions, so a subagent completion can fall through to the normal route `agent` handoff and allocate a fresh session when the route entry is stale. Related open PRs address queue ordering or wrong-session routing, but none close this route-to-paused-session gap.

## What This Changes

The issue requests that subagent completion handoffs resume the yielded parent session bound to an external route instead of starting a fresh route session.

## Best Possible Solution

Persist a bounded paused-session binding for yielded route entries and have completion or inbound inter-session handoffs prefer that paused session before normal stale-route session rotation.

## Reproduction Assessment

Yes, at source level and with reporter-provided live evidence. Current main parks yielded parents, removes the active embedded run, lacks a paused-session route pointer, and can rotate stale route entries to a new session id.

## Solution Assessment

Yes, the paused-session route binding is the narrowest maintainable direction. Queue-first completion routing or active-wake fallthrough alone does not address the stale route sessionId overwrite described here.
