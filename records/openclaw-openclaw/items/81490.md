## Summary

Keep open: current main still lacks the durable paused-session binding needed to resume a yielded external parent, and the later commits that implement that idea are not on any local branch/tag containing current main.

## What This Changes

The issue reports that subagent completion announcements routed to an external parent create a new route run instead of resuming the yielded parent session.

## Best Possible Solution

Persist a bounded paused-session binding for yielded route entries, prefer it in completion announce and gateway reuse paths, then clear or expire it after the parent is successfully woken.

## Reproduction Assessment

Yes, at source level with reporter-provided live evidence. Current main parks sessions_yield runs as paused/inactive, cannot queue to inactive embedded runs, and has no durable paused-session binding before direct fallback can allocate a fresh route session.

## Solution Assessment

Yes. The narrow maintainable fix is a yielded route/session binding or equivalent paused-session resolver; queue-first completion routing alone does not address the reported route pointer overwrite.
