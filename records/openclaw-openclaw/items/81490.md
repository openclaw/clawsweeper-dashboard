## Summary

Keep open: current main still does not preserve a yielded route-to-session binding for subagent completion handoffs. The reporter's live Telegram/Bedrock evidence matches the current source shape: `sessions_yield` parks the parent, inactive embedded runs cannot be queued, and completion delivery can continue into the direct `agent` path without a paused-session pointer.

## What This Changes

Ensure subagent completion resumes a yielded parent session on the same external route instead of starting a fresh route-bound run.

## Best Possible Solution

Persist and resolve an explicit bounded paused-session binding for yielded external route entries, then route subagent completion handoffs to that session before normal direct fallback can create a fresh run.

## Reproduction Assessment

Yes, at source level and with reporter-provided live evidence. Current main parks yielded runs, cannot queue inactive embedded runs, and lacks a paused-session route binding before direct fallback.

## Solution Assessment

Yes. A bounded yielded-route/session binding is the narrow maintainable direction; queue-first completion routing alone does not address the reported route pointer overwrite.
