## Summary

Keep open: current main and v2026.6.8 still resolve yielded subagent completion delivery through active-run wake or a route-key direct agent handoff, with no durable paused-session route binding that would resume the yielded parent session id.

## What This Changes

A bug report that subagent completion on an external route starts a fresh parent-route run instead of resuming the yielded parent session, overwriting the session-store pointer.

## Best Possible Solution

Land a maintainer-owned yielded-session resume contract that durably binds an external route to the paused parent session and re-enters that session for the child completion event.

## Reproduction Assessment

Yes, at source level: current main still makes yielded requesters inactive for embedded steering and then falls through to a route-key direct agent handoff. I did not run a live Telegram/Bedrock replay.

## Solution Assessment

Unclear: the suggested paused-session route binding is plausible, but the best fix needs maintainer agreement on the durable yield/resume lifecycle contract and window semantics.
