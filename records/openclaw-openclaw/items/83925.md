The issue asks for consistent plain-text `openclaw node status` hint output across not-loaded, missing-unit, and stopped service states.

## Best Possible Solution

Normalize the plain-text node status hint stream in `runNodeDaemonStatus` and add regression coverage for missing-unit and stopped states without refactoring the broader gateway status printer.
