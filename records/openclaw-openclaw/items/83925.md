## What This Changes

The issue reports that `openclaw node status` emits recovery hints to stdout for the not-loaded service path but to stderr for missing-unit and stopped service paths.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/84100; the end state should normalize node-status hint routing with focused stdout/stderr regression coverage.

## Reproduction Assessment

Yes for a source-level reproduction: current main visibly routes not-loaded hints through `defaultRuntime.log` and missing-unit/stopped hints through `defaultRuntime.error`. I did not run real service states in this read-only review.

## Solution Assessment

Yes: the narrow maintainable solution is to move only the follow-up runtime hint lines to stdout, preserve terse diagnostics on stderr, and add focused stream-capture coverage. The active linked PR follows that shape, pending review and proof.
