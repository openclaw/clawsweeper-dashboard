## Summary

Close as superseded. The branch addresses a real current-main `agents add` workspace-default bug, but it still broadens the peer-level suggestion to custom shared workspace configurations that current main intentionally preserves, while open PR #71941 tracks the same linked bug with the narrower behavior and stronger coverage.

## What This Changes

The PR adds and exports a peer-workspace suggestion helper, uses it as the interactive `agents add` workspace prompt default for brand-new agents, and adds helper tests.

## Best Possible Solution

Keep one canonical implementation for #71889, preferably #71941: land a narrow wizard-prefill fix that changes the documented default/setup workspace case, preserves #59789 custom shared-base behavior, includes wizard regression tests, and carries the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: setup writes `agents.defaults.workspace`, `agents add` uses `resolveAgentWorkspaceDir` for the prompt default, and the resolver appends non-default ids under that fallback while docs expect a peer-level path.

## Solution Assessment

No. The wizard-only direction is reasonable, but this branch applies the sibling suggestion to too many workspace shapes; the safer solution is the narrower #71941-style gate that only overrides the documented default/setup workspace case.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve custom shared workspace defaults:** `src/agents/agent-scope-config.ts:204`
  - body: When `agents.defaults.workspace` is an intentional shared base, current main resolves new non-default agents below that base and has regression coverage for it. This helper instead derives a sibling of the default agent workspace for every new agent, and `agents add` then persists that suggestion, so accepting the prompt silently bypasses the configured shared root. Gate the peer-level suggestion to the documented setup/default workspace case or defer to `resolveAgentWorkspaceDir` for custom roots.
  - confidence: 0.84
