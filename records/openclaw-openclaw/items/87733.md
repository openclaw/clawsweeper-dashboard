## Summary

Keep open. Current main and v2026.6.8 still define isolated gateways and one-domain operator scopes; the open session-tool PR is a narrower routing proposal and does not resolve the requested bridge protocol, signed peer identity, trust-root, or cross-gateway scope projection.

## What This Changes

The issue requests a documented OpenClaw cross-gateway federation protocol with bridge events, signed peer identity, and tenant-scoped operator-scope mapping.

## Best Possible Solution

Keep this as the broad federation architecture tracker; if maintainers accept the direction, split it into bridge protocol, gateway identity/trust-root, scope projection, and upgrade/security proof sub-issues while leaving narrower session-tool routing on its own track.

## Reproduction Assessment

Not applicable: this is an FRD for new federation primitives, not a report of broken current behavior. Source and docs inspection show the requested bridge, identity, and scope-projection contract is absent on current main and v2026.6.8.

## Solution Assessment
