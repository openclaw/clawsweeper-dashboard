## What This Changes

The issue asks for provider/auth diagnostics or repair guidance when `openai/gpt-5.5` is selected after OpenAI/Codex unification but provider-level `openai` config still points at a different backend.

## Best Possible Solution

Keep canonical `openai/gpt-*` routing, and add or bless one diagnostic path that exposes the effective model, runtime, provider/backend, auth profile, and session override state for canonical mixed-provider configs without restoring live `openai-codex` routes.
