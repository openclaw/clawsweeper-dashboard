## What This Changes

The issue asks OpenClaw to diagnose and prevent post-upgrade OpenAI/Codex routing drift when canonical `openai/gpt-5.5` is paired with provider-level OpenAI config that points at a non-subscription backend.

## Best Possible Solution

Add a narrow doctor/status diagnostic for canonical OpenAI GPT routes whose provider-level OpenAI config/auth/backend cannot satisfy ChatGPT/Codex OAuth routing, then track route-inspection and session-override UX separately if maintainers want those commands.
