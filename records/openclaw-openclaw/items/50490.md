## Summary

Keep open: current main and v2026.5.22 still derive Feishu group hard mention gating from config-only `requireMention`, while `/activation` only persists `groupActivation` for the later reply path, so the reported behavior remains source-reproducible.

## What This Changes

The issue reports that `/activation mention` in Feishu group chats confirms success but does not make the bot ignore later unmentioned group messages.

## Best Possible Solution

Resolve effective Feishu group activation before the hard mention gate, using persisted `/activation` state as a per-session override over config defaults, with regression tests for both `mention` and `always` modes.

## Reproduction Assessment

Yes, at source level: `/activation mention` persists `sessionEntry.groupActivation`, while Feishu's pre-dispatch group gate still reads only config-derived `requireMention`. I did not run a live Feishu group in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to make Feishu honor the existing group activation session state before its hard mention gate. Adding a new config knob or relying on model-level `NO_REPLY` would leave the existing `/activation` contract inconsistent.
