## Summary

Keep open. Current main and v2026.5.22 still have the reported Feishu gap: `/activation mention` persists session activation state, but Feishu group admission still decides dispatch from config-derived `requireMention` before that session state can hard-stop non-mention messages.

## What This Changes

The issue reports that Feishu group `/activation mention` confirms but does not stop replies to unmentioned group messages when config allows non-mention replies.

## Best Possible Solution

Make Feishu group admission honor the current session `groupActivation` before dispatch, so `mention` hard-gates non-mentions while preserving existing config defaults and history recording.

## Reproduction Assessment

Yes, at source level. With Feishu `groupPolicy: "open"` or `requireMention: false`, `/activation mention` updates `sessionEntry.groupActivation`, but a later unmentioned group event still passes Feishu admission because `requireMention` is resolved only from Feishu config.

## Solution Assessment

Yes, the narrow maintainable fix is in Feishu group admission/session lookup, not another config knob. Relying on prompt-level `NO_REPLY` is the behavior that is failing here.
