## Summary

Keep open. Current main and the latest release still have the reported Feishu gap: `/activation mention` persists group activation state, but Feishu group admission still decides whether to dispatch from config-derived `requireMention` before that session state can hard-stop unmentioned messages.

## What This Changes

The issue asks for Feishu group `/activation mention` to make the bot stop responding to ordinary unmentioned group messages after the mode is switched.

## Best Possible Solution

Make the Feishu plugin honor persisted group activation at its group admission gate, using config-derived `requireMention` as the default and session `groupActivation` as the per-session override.

## Reproduction Assessment

Yes, at source level. With Feishu `groupPolicy: "open"` or `requireMention: false`, `/activation mention` updates `sessionEntry.groupActivation`, but a later unmentioned group event still passes Feishu admission because `requireMention` is resolved only from config.

## Solution Assessment

Yes. The narrow maintainable fix is in Feishu group admission/session lookup, not a new config option or relying on prompt-level `NO_REPLY`.
