- `impact:message-loss`: Feishu group messages are routed for visible replies when the session has been switched to mention-only activation, so channel delivery/routing behavior is wrong.

## Summary

Keep open. Current main and the latest release still route Feishu group mention gating from config-derived `requireMention`, while `/activation mention` only persists session activation that is consumed later in the shared reply path.

## What This Changes

The issue asks Feishu group chat to honor `/activation mention` so unmentioned group messages stop triggering replies after the command succeeds.

## Best Possible Solution

Implement the Feishu-owned bridge so effective persisted group activation is resolved before the hard mention gate, with config-derived `requireMention` as the default and focused coverage for normal, broadcast, and topic-scoped group sessions.

## Reproduction Assessment

Yes, with source-level confidence. Configure Feishu so unmentioned group messages are admitted, send `/activation mention`, then send an unmentioned group message; current source still decides Feishu admission from config-only `requireMention` before persisted activation can gate it.

## Solution Assessment

Yes. The narrow maintainable fix is to resolve the effective Feishu route/session early enough to fold persisted activation into the hard mention gate; prompt-only or docs-only changes would leave the successful command disconnected from runtime behavior.
