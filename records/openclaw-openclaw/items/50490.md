## Summary

Keep open. Current main still has the Feishu-specific gap: `/activation mention` persists session activation state, but Feishu group admission computes the hard mention gate from config-only `requireMention`; the linked fix candidate is still open and unmerged.

## What This Changes

The issue requests that Feishu group chats honor `/activation mention` so ordinary unmentioned group messages stop triggering replies after the mode is changed.

## Best Possible Solution

Review, repair, and land or replace https://github.com/openclaw/openclaw/pull/87709 so Feishu plugin admission enforces persisted group activation before non-mention group messages enter the agent path, while preserving config fallback and command recovery.
