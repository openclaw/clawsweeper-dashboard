## Summary

Keep open. Current main still has the Feishu-specific gap described in the discussion: `/activation mention` persists session activation state, but Feishu’s pre-dispatch group gate still decides non-mention admission from config-derived `requireMention`.

## What This Changes

The issue reports that Feishu group chats keep responding to non-mention messages after `/activation mention` switches the session to mention-only activation.

## Best Possible Solution

Resolve effective Feishu group activation before the hard mention gate, using config `requireMention` as the default and persisted `/activation` state as the per-session override.

## Reproduction Assessment

Yes, from source inspection rather than a live Feishu run: `/activation mention` writes `sessionEntry.groupActivation`, while the Feishu gate still uses config-only `requireMention` before dispatch. The report’s steps map directly onto those code paths.

## Solution Assessment

Yes. The narrow maintainable fix is to make Feishu’s group admission consult the effective session activation state before dispatch, not to add another config knob or rely on model self-filtering.
