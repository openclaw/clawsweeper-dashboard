## Summary

Keep open. Current main still has the Feishu-specific gap: `/activation mention` stores session activation state, but Feishu group ingress still gates ordinary messages from config-derived `requireMention` before that persisted activation state is consulted; the linked closing PR remains open and unmerged.

## What This Changes

The issue reports that Feishu group `/activation mention` appears to succeed but does not stop the bot from answering non-mention group messages.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/87709, or an equivalent Feishu plugin fix that reads the effective persisted group activation before ordinary group messages enter the agent path while preserving config fallback and command recovery.

## Reproduction Assessment

Yes. Source inspection shows the reproducible path: `/activation` writes `sessionEntry.groupActivation`, but Feishu group admission uses config-derived `requireMention` and does not read that persisted activation before dispatch.

## Solution Assessment

Yes. Keeping this issue open is the right cleanup outcome because the active fix is still an open PR; the maintainable code fix belongs in the Feishu plugin admission path with tests for persisted activation overriding static config.
