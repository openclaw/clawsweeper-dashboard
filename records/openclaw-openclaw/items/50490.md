## Summary

Keep open. Current main and the latest release still derive Feishu group admission from config-only `requireMention`, while `/activation mention` persists session state that Feishu does not enforce; https://github.com/openclaw/openclaw/pull/87709 is the open unmerged fix candidate.

## What This Changes

Feishu group `/activation mention` should make later non-mention group messages stop entering the agent path.

## Best Possible Solution

Review, repair, and land or replace https://github.com/openclaw/openclaw/pull/87709 so Feishu plugin admission enforces persisted group activation before non-mention group messages enter the agent path while preserving config fallback and command recovery.

## Reproduction Assessment

Yes. Source inspection shows `/activation` persists `groupActivation`, while Feishu group admission still uses config-derived `requireMention` before dispatch; no live Feishu tenant was exercised in this read-only pass.

## Solution Assessment

Yes. The maintainable fix belongs in Feishu plugin group admission or an equivalent plugin-owned helper; current main and the latest release are not fixed, and the open PR should be reviewed, repaired, landed, or replaced before this issue closes.
