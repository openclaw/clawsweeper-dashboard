## Summary

Keep open for maintainer/product review. Current main still has no Telegram bot profile-photo sync, and Quick Settings assistant uploads are browser-local UI overrides, so automatic upload needs an explicit opt-in, per-account source-of-truth design.

## What This Changes

The issue requests bidirectional synchronization between the Web UI assistant avatar and the Telegram bot avatar.

## Best Possible Solution

Add an opt-in Telegram plugin profile-sync feature backed by a gateway-persisted avatar source with explicit per-account targeting, documented manual fallback, and no automatic upload of browser-local Control UI overrides.

## Reproduction Assessment

Not applicable as a feature request. Source inspection shows current main intentionally treats Quick Settings assistant uploads as browser-local overrides, while Telegram exposes no profile-photo sync action or watcher in this repo.

## Solution Assessment

Unclear. The proposed automatic avatars/avatar.jpg watcher is not the safest design because the current Web UI upload is local-only; an explicit opt-in Telegram plugin setting around a persisted avatar source is the narrower maintainable direction.
