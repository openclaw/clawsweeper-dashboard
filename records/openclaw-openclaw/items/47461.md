## Summary

Keep open: current main still treats Control UI assistant avatar uploads as browser-local OpenClaw UI state, while Telegram has no registered bot profile-photo update action or opt-in policy; this remains a product/security/API decision rather than a stale or already-implemented request.

## What This Changes

Requested bidirectional synchronization so changing the assistant avatar in the Web UI updates the Telegram bot profile photo.

## Best Possible Solution

Define a Telegram-owned opt-in profile update flow, preferably using the existing channel action vocabulary where it fits, with account/config gating, docs, tests, and live Telegram proof; otherwise document that Web UI assistant avatars intentionally do not mutate Telegram bot profiles.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and tests show the current behavior is a missing capability: Control UI assistant avatar overrides are browser-local, and Telegram has no registered bot profile-photo action.

## Solution Assessment

No. The proposed file watcher plus setChatPhoto is not the best settled path because the current avatar source is browser-local UI state and the Telegram profile mutation needs an explicit opt-in Telegram plugin/API design.
