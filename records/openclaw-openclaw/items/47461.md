## Summary

Keep open: current main still treats Control UI assistant avatars as local OpenClaw identity state, while the Telegram plugin exposes no bot profile-photo action or opt-in policy surface. Prior Telegram avatar PRs show the idea is real but were closed unmerged, so this remains a maintainer product/API decision rather than cleanup.

## What This Changes

Request to make Telegram bot avatars sync bidirectionally with Web UI avatar changes, including an automatic push from OpenClaw avatar state to Telegram.

## Best Possible Solution

Define an explicit Telegram-owned, opt-in bot profile-photo update flow with config/account gating, docs, tests, and real Telegram proof, or document that Web UI assistant avatars intentionally do not mutate Telegram bot profiles.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs show the current behavior is a missing capability: Control UI avatar overrides are browser-local, and Telegram has no registered bot profile-photo action.

## Solution Assessment

No; the proposed file watcher plus setChatPhoto is not the best settled solution because the UI avatar source is browser-local and Telegram bot profile photos use setMyProfilePhoto, so maintainers should design an explicit opt-in Telegram plugin flow first.
