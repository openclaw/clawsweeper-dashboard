## Summary

Keep open for maintainer/product review. Current main still lacks Telegram bot profile-photo sync, and automatic upload needs privacy, opt-in, and multi-account decisions because Quick Settings currently uses a browser-local Control UI override rather than an explicit Telegram profile setting.

## What This Changes

Request bidirectional Telegram bot avatar synchronization so Web UI avatar changes update the Telegram client and Telegram-side avatar changes remain reflected in OpenClaw.

## Best Possible Solution

Add an opt-in Telegram plugin profile-sync feature that uses Telegram bot profile-photo methods from a gateway-persisted avatar source, supports explicit per-account targeting, documents the BotFather/manual fallback, and never uploads browser-local overrides unless the user deliberately promotes them.

## Reproduction Assessment

Not applicable as a feature request. Current-main source inspection shows no Telegram profile-photo action or watcher, while the Quick Settings image path stores a browser-local Control UI override.

## Solution Assessment

Unclear. The requested automatic file-watch plus setChatPhoto approach is not the safest or most accurate path because Telegram exposes bot profile-photo methods and the current UI override is local; an opt-in Telegram plugin design around a gateway-persisted avatar is the more maintainable direction.
