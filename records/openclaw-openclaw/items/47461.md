## What This Changes

The issue requests bidirectional synchronization between OpenClaw/Web UI assistant avatar changes and the Telegram bot/client profile photo.

## Best Possible Solution

Define an explicit Telegram-owned, opt-in bot profile-photo update flow with config/policy gating, docs, and real Telegram proof, or document that Web UI avatar overrides intentionally do not mutate Telegram bot profiles.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection shows the current product has local OpenClaw avatar state and no Telegram bot profile-photo action, so this is a missing capability rather than broken documented behavior.

## Solution Assessment

No; a file watcher on avatars/avatar.jpg that calls setChatPhoto is not yet the best settled solution. If maintainers want this capability, it should be a designed Telegram plugin action/config flow with explicit operator gating and real Telegram API proof.
