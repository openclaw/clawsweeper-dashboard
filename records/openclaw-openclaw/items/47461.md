## Summary

Keep open: current main still has no Web UI-to-Telegram bot profile-photo sync, and the request is a new Telegram identity-management capability rather than broken existing behavior. The current Control UI assistant avatar override is explicitly browser-local, so an implementation needs maintainer product/API judgment and an explicit operator gate before any fix lane should proceed.

## What This Changes

The issue asks for Web UI avatar changes to update the Telegram bot profile photo automatically so Telegram clients show the new avatar.

## Best Possible Solution

Define an explicit Telegram-owned, opt-in bot profile-photo update path with clear account/operator gating and docs, or decide that Web UI avatar overrides remain local and out of scope for Telegram profile mutation.

## Reproduction Assessment

No, not as a bug reproduction. Source inspection confirms the requested sync path is absent, while current main intentionally treats Control UI assistant avatar overrides as browser-local state.

## Solution Assessment

No, the proposed watcher on `avatars/avatar.jpg` plus `setChatPhoto` is not the best settled fix. Telegram bot profile mutation should use a designed `setMyProfilePhoto`-style action or config-backed flow with explicit operator gating if maintainers approve it.
