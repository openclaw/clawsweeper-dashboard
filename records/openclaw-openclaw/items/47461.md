## Summary

Keep open: current main and the latest release still keep Control UI assistant avatar uploads browser-local, and Telegram has no OpenClaw profile-photo update action; the remaining work is a security-sensitive product/API decision, not an implemented bug fix.

## What This Changes

The issue requests bidirectional synchronization so a Web UI assistant avatar change updates the Telegram bot profile shown in Telegram clients.

## Best Possible Solution

Decide whether OpenClaw should support Telegram bot-profile avatar mutation at all; if yes, design it as a Telegram-owned opt-in action/config flow with docs, tests, and live Telegram proof, otherwise document that BotFather remains the supported profile-photo path.

## Reproduction Assessment

Not applicable as a bug reproduction. Source, docs, tests, and the latest release show this is a missing capability: Web UI assistant avatar overrides are browser-local and Telegram has no registered bot profile-photo action.

## Solution Assessment

No. The proposed file watcher plus setChatPhoto is not the best settled path because setChatPhoto is a chat-photo API and any bot profile mutation needs explicit Telegram plugin/API policy and opt-in security design.
