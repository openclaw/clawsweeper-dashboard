## Summary

The request is useful, but it is Telegram-specific avatar automation rather than a missing OpenClaw core contract. Current main already has generic agent avatar resolution/serving, the plugin API can support a plugin-owned service/tool with config access, and the remaining Telegram Bot API call should live in a ClawHub plugin or skill instead of adding automatic public bot-profile mutation to core.

## What This Changes

The issue requests bidirectional Telegram bot avatar synchronization so Web UI avatar changes automatically update the Telegram client avatar.

## Best Possible Solution

Publish this as a ClawHub plugin or skill that explicitly opts in, reads the configured agent avatar, validates Telegram-compatible JPG/static profile-photo input, and calls `setMyProfilePhoto` with the operator's bot token.

## Reproduction Assessment

Not applicable. This is a feature request, not a broken existing behavior report; source inspection gives a high-confidence absence check for built-in Telegram bot profile-photo sync on current main.

## Solution Assessment

No. The proposed Gateway/Control UI file watcher and `setChatPhoto` call are not the best core solution; the maintainable path is optional plugin-owned automation using Telegram `setMyProfilePhoto` with explicit operator opt-in.
