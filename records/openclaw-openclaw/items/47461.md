## Summary

Keep open: current main still does not implement Telegram bot profile-photo synchronization, and the request is a new Telegram/Web UI identity feature rather than a broken existing contract. The Control UI avatar override is documented as browser-local, while the Telegram action/config/docs surface exposes message, reaction, sticker, and topic actions but no bot profile-photo action.

## What This Changes

Request bidirectional synchronization between OpenClaw/Web UI avatar changes and the Telegram bot profile photo.

## Best Possible Solution

Design a Telegram-owned explicit profile-photo update path, likely through the plugin action surface with config gating and docs, while preserving the browser-local Control UI avatar boundary unless maintainers approve server-side identity sync.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs are enough to show this is a requested sync capability that current main does not promise or expose.

## Solution Assessment

No. The proposed file watcher plus chat-photo update is not the narrowest maintainable path; a safer direction is an explicit Telegram plugin profile-photo action or sync policy approved by maintainers.
