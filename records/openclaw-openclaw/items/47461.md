## Summary

Keep open: current main still treats Control UI uploaded assistant avatars as browser-local OpenClaw identity state, while Telegram exposes no bot profile-photo action or config gate; prior bot-avatar PRs were closed unmerged, so this remains a maintainer product/security/API decision.

## What This Changes

The issue requests two-way synchronization between OpenClaw Web UI assistant avatar changes and the Telegram bot profile photo.

## Best Possible Solution

Define an explicit Telegram-owned, opt-in bot profile-photo update flow with config/account gating, docs, tests, and real Telegram proof, or document that Web UI assistant avatars intentionally do not mutate Telegram bot profiles.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs show the current behavior is a missing capability: Control UI assistant avatar overrides are browser-local, and Telegram has no registered bot profile-photo action.

## Solution Assessment

No. The proposed file watcher plus setChatPhoto is not the best settled solution because the UI avatar source is browser-local and Telegram bot profile photos use setMyProfilePhoto, so maintainers should design an explicit opt-in Telegram plugin flow first.
