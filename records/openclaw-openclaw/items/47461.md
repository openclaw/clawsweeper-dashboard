## What This Changes

Request to add two-way sync between the Web UI assistant avatar and the Telegram bot profile photo so users do not need to update the bot manually in BotFather.

## Best Possible Solution

Define a Telegram-plugin-owned, default-off explicit bot profile-photo update flow using `setMyProfilePhoto`, with account scoping, authorization/audit behavior, docs, focused tests, and live Telegram proof before any automatic UI handoff.

## Reproduction Assessment

No as a bug reproduction; this is a missing feature rather than a broken current contract. Source inspection gives a high-confidence absence check: Control UI avatar uploads stay browser-local and Telegram actions do not include a bot profile-photo operation.

## Solution Assessment

No, the proposed file watcher plus `setChatPhoto` is not the best settled solution. A maintainable path would be an explicit Telegram-plugin-owned `setMyProfilePhoto` action or sync policy after maintainers decide the security and UI boundary.
