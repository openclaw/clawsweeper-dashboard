Keep open: current main still lacks Telegram bot profile/avatar synchronization, and this is a new Telegram/Web UI identity feature that needs product and operator-control decisions rather than a broken existing contract.

The issue requests bidirectional synchronization between the Web UI assistant/avatar setting and the Telegram bot avatar shown in Telegram clients.

## Best Possible Solution

Design an explicit Telegram-owned bot profile update path with a clear config/action gate and docs, while preserving the browser-local Control UI avatar boundary unless maintainers approve server-side identity sync.
