## Summary

Keep open: current main still has no Telegram bot profile-photo sync, and the request is a new Telegram/Web UI identity-management feature that needs maintainer product/API design rather than conservative cleanup. Two prior related Telegram bot-avatar PRs attempted an explicit action but were closed unmerged, so there is no implemented or canonical merged fix yet.

## What This Changes

The issue requests two-way avatar sync so changing the assistant/bot avatar in the Web UI updates the Telegram bot profile automatically.

## Best Possible Solution

Define a Telegram-owned, opt-in profile-photo update path with a clear operator gate and docs, using the bot-profile API rather than watching browser-local UI overrides by default.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection against current main shows the requested sync path is absent, while docs and UI code define Control UI assistant avatar overrides as browser-local rather than a Telegram profile contract.

## Solution Assessment

No. The proposed file watcher plus setChatPhoto is not the best settled fix because Web UI overrides are intentionally local and Telegram provides bot-profile-specific methods; maintainers should first choose an explicit Telegram-owned action/config design or decide this remains out of scope.
