## Summary

Keep open. Current main does not implement Telegram bot profile-photo synchronization, and the request is plausible now that Telegram Bot API has bot profile-photo methods, but the requested automatic sync crosses product/privacy boundaries because the current Quick Settings assistant image is a Control UI override rather than a clear gateway-owned Telegram profile setting.

## What This Changes

Request automatic bidirectional synchronization between OpenClaw/Web UI assistant avatar changes and the Telegram bot profile image.

## Best Possible Solution

Design an opt-in Telegram plugin feature that syncs a gateway-persisted assistant avatar to selected Telegram account(s) using the bot-profile API, documents BotFather/manual fallback, and avoids uploading browser-local-only overrides unless the user explicitly promotes them to gateway config.

## Reproduction Assessment

Not applicable as a feature request. A high-confidence current-main check shows no Telegram bot profile-photo action or sync path, while Quick Settings avatar uploads currently produce a Control UI override.

## Solution Assessment

Unclear. The requested direction is understandable, but the proposed setChatPhoto endpoint is not the right bot-profile primitive and automatic default sync is not obviously the safest behavior; an opt-in Telegram plugin profile-sync design would be safer.
