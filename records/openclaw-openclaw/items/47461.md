## Summary

Keep open: current main still has no Telegram bot profile-photo sync, and this is a new Telegram/Web UI identity feature that needs maintainer product/API design rather than conservative cleanup.

## What This Changes

Request to synchronize Telegram bot profile photos with Web UI assistant avatar changes, ideally in both directions.

## Best Possible Solution

Add a Telegram-owned bot profile update path only after maintainers define the sync direction, operator gate, upload limits, and docs, while keeping browser-local Control UI overrides local unless explicitly promoted.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection against current main shows the requested sync path is absent, and current docs define Control UI assistant avatars as browser-local rather than a Telegram profile contract.

## Solution Assessment

No. The proposed file watcher plus setChatPhoto is not the best settled fix because Telegram now has bot-profile-specific methods while setChatPhoto targets chat photos, and OpenClaw still needs an explicit Telegram-owned action/config gate before crossing the browser-local avatar boundary.
