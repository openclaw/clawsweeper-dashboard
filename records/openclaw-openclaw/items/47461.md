## Summary

Keep open: current main does not implement Telegram bot profile-photo synchronization, and this is a feature/product design request rather than a broken existing contract. The current Control UI avatar override is explicitly browser-local, while the Telegram plugin action/config surface has no profile-photo or set-profile capability.

## What This Changes

Request to synchronize assistant avatar changes from the Web UI back to the Telegram bot avatar automatically.

## Best Possible Solution

Define a Telegram-owned, explicit profile-photo update path that uses Telegram's bot profile-photo API and preserves the documented browser-local Control UI avatar boundary unless maintainers decide to add server-side identity sync.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs are enough to show the requested sync path is not part of current main: Control UI overrides are browser-local and Telegram exposes no profile-photo action.

## Solution Assessment

No. The proposed file watcher plus `setChatPhoto` is not the best fix because `setChatPhoto` targets chat photos and the Control UI override is local; a maintainable solution belongs in the Telegram plugin with an explicit profile-update surface and maintainer-approved sync policy.
