## Summary

Keep open: current main and the latest release still keep Control UI assistant avatar uploads browser-local and Telegram has no bot profile-photo update action or opt-in policy. The request is useful, but it is a new Telegram profile-mutation capability with product/security/API decisions, not an implemented bug fix or a safe automated repair target.

## What This Changes

Request Web UI assistant avatar changes to automatically synchronize to the Telegram bot avatar shown in Telegram clients instead of requiring manual BotFather changes.

## Best Possible Solution

Define a Telegram-plugin-owned, default-off bot profile-photo update flow with explicit account authorization, config/docs, focused tests, and live Telegram proof; only then consider a separate UI bridge from assistant avatar state.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and release inspection give a high-confidence absence check: Control UI avatar overrides are browser-local and Telegram has no registered bot profile-photo update action.

## Solution Assessment

No. A file watcher plus setChatPhoto is not the best settled path because setChatPhoto targets chat photos, while bot profile mutation should be a Telegram-owned opt-in action using the correct Bot API method and security policy.
