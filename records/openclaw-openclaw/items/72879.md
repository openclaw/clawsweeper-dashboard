## Summary

Keep this open. Current main still has a native Google path that can serialize Gemini `functionCall` parts without `thoughtSignature` when the in-memory tool-call block lacks one, and the issue discussion includes recent native Gemini 3 thinking/tool-calling reproductions after the earlier fixes.

## What This Changes

This issue asks OpenClaw to fix a Gemini native Google tool-calling regression where replayed `functionCall` parts can be sent without the required `thoughtSignature`.

## Best Possible Solution

Add a narrow native Google replay repair that preserves real same-model signatures byte-for-byte, recovers or merges signatures when native streaming provides them, and handles unsigned transferred/current replay history in the provider-specific way Google accepts. Keep cross-provider real-signature stripping intact and add focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. The issue includes concrete native Gemini 3 thinking/tool-calling reproduction steps, and current source inspection shows an unsigned native Google `functionCall` payload can still be built when the assistant tool-call block lacks `thoughtSignature`.

## Solution Assessment

Yes. Keeping this open and routing a focused provider fix is the narrowest maintainable path; a broad transcript sanitizer would risk corrupting provider-specific replay semantics.
