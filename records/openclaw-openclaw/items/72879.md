## Summary

Keep this open. Current main still has the native Google serializer path that can send a same-model Gemini `functionCall` without `thoughtSignature` when the stored assistant tool-call block lacks that field, and the issue has a recent native Gemini 3 thinking/tool-calling reproduction that matches Google's documented 400 validation rule.

## What This Changes

The issue reports a Gemini native Google provider regression where tool-calling replay can send `functionCall` parts without `thoughtSignature`, causing Google API 400 errors in thinking mode.

## Best Possible Solution

Add a narrow native Google replay repair that preserves real same-model signatures byte-for-byte, recovers or merges signatures when native responses contain them, and handles unsigned transferred/current-turn function-call history in the provider-scoped Google path without reusing real signatures across provider/model boundaries. Add regression coverage and update transcript-hygiene docs/changelog if behavior changes.

Yes. The issue includes a concrete native `google-generative-ai` Gemini 3 thinking/tool-calling reproduction, and current source inspection shows an unsigned same-model `functionCall` payload can still be built when the assistant tool-call block lacks `thoughtSignature`.

## Repair Work Prompt

Fix the native Google Gemini `thoughtSignature` replay regression tracked by https://github.com/openclaw/openclaw/issues/72879. Current `extensions/google/transport-stream.ts` only serializes `thoughtSignature` on `functionCall` parts when the stored assistant tool-call block already has `block.thoughtSignature`; current native Google replay policy sanitizes/repairs ordering but does not recover or substitute missing function-call signatures. Implement a provider-scoped fix for `google-generative-ai`/native Google replay that preserves real same-provider+same-model signatures byte-for-byte, recovers/merges signatures when native responses provide them, and handles unsigned transferred or deterministic current-turn function-call history according to Google’s documented Gemini 3 current-turn signature requirements. Do not blindly reuse real `thoughtSignature` values across provider/api/model boundaries, and keep `src/agents/transport-message-transform.ts` cross-provider stripping intact. Add focused unit tests for native Google serialization/parsing/replay repair, including same-model signed replay, unsigned same-model/current-turn repair behavior, cross-provider stripping, and parallel-call behavior. Update `docs/reference/transcript-hygiene.md` if the provider replay rule changes, and add a user-facing `CHANGELOG.md` fix entry crediting the issue reporter/commenters, e.g. `Thanks @M00zyx` and `Thanks @yuxiaoyang2007-prog`.
