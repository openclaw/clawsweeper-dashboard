## Summary

Keep open. Current main has partial Gemini tool-call signature round-tripping, but it still can replay a native Google `functionCall` without `thoughtSignature` when the stored assistant tool-call block is unsigned; the issue has recent matching reproduction details and the linked fix PR is still open.

## What This Changes

Native Google Gemini replay should avoid `thought_signature` 400s by preserving or repairing required tool-call signatures during thinking/tool-calling turns and cross-provider replay.

## Best Possible Solution

Land a focused provider-scoped Google replay fix that preserves real same-model signatures byte-for-byte, uses Google's documented sentinel only for unsigned transferred current-turn tool-call history, keeps cross-provider real-signature stripping intact, and adds deterministic replay tests plus docs/changelog coverage.

## Reproduction Assessment

Yes. The issue includes a concrete native `google-generative-ai` Gemini 3 thinking/tool-calling reproduction, and current-main source inspection shows the outbound native serializer still emits an unsigned `functionCall` when the stored block lacks `thoughtSignature`.

## Solution Assessment

Yes, keeping this open while reviewing the linked repair is the best current path. The narrowest maintainable fix is a Google-specific replay repair, not closing the issue before the open implementation candidate has merged and shipped.
