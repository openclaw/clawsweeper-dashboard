## Summary

Keep this open. Current main still has a native Google path that can serialize Gemini `functionCall` parts without `thoughtSignature` when the in-memory tool-call block lacks one, while the report includes a recent native Gemini 3 tool-calling reproduction and Google's current contract requires current-turn function-call signatures or documented dummy signatures for transferred history. citeturn0search0

## What This Changes

The issue requests a fix for a native Google Gemini tool-calling replay regression where missing or dropped `thought_signature` metadata causes 400 errors during thinking/tool-use turns, especially after fallback or replay history contamination.

## Best Possible Solution

Add a narrow native Google replay repair that preserves real same-model signatures byte-for-byte, recovers or merges signatures when the native response contained them, and handles unsigned transferred/deterministic current-turn function-call history with Google's documented dummy-signature behavior. Keep cross-provider real-signature stripping intact and document the provider-specific transcript rule.

## Reproduction Assessment

Yes. The issue includes a concrete native Google Gemini 3 thinking/tool-calling reproduction, and current source inspection shows an unsigned `functionCall` payload can still be built when the assistant tool-call block lacks `thoughtSignature`.

## Solution Assessment

Yes for keeping this open and routing a focused fix PR. The narrowest maintainable path is provider-scoped native Google signature repair plus regression coverage, not a broad transcript sanitizer or cross-provider real-signature reuse.
