## What This Changes

The issue reports a Gemini thinking/tool-call replay regression where OpenClaw can resend function-call history without the required `thoughtSignature`, causing Google or Google-compatible providers to return 400 errors.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/75713 or an equivalent focused patch that preserves real Gemini tool-call signatures across shared replay normalization, native Google serialization, and OpenAI-compatible Google serialization, using the sentinel only for genuinely unsigned legacy history.
