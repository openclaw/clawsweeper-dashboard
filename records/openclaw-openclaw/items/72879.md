## Summary

Keep open. Current `main` preserves native Google tool-call signatures when they are already present, but the same serializer can still replay a native Gemini `functionCall` without `thoughtSignature` when the stored block is unsigned, which matches the recent 2026.4.27 regression evidence in the discussion.

## What This Changes

Native Google Gemini 3 tool-calling turns can fail with a 400 when OpenClaw replays a `functionCall` part without the required `thoughtSignature`.

## Best Possible Solution

Add a provider-scoped native Google replay repair that preserves real same-model signatures byte-for-byte, handles unsigned transferred current-turn function-call history according to Google's Gemini 3 rules, and keeps cross-provider real-signature stripping intact. Add deterministic regression tests for native serialization, parser round-trip, unsigned current-turn repair, and cross-provider boundaries.

## Reproduction Assessment

Yes. The discussion gives a concrete native `google-generative-ai` Gemini 3 thinking/tool-calling reproduction, and current `main` source inspection shows the serializer still emits an unsigned `functionCall` when the stored tool-call block lacks `thoughtSignature`.

## Solution Assessment

Yes for keeping this open and routing a focused fix PR. The narrowest maintainable solution is a native Google replay/signature repair with targeted tests, not another broad transcript sanitizer or reuse of real signatures across provider boundaries.
