## Summary

Keep open: current main still has the native Google missing-signature replay shape, and active PR #75713 is an implementation candidate rather than a reason to close before merge.

## What This Changes

The issue reports that native Google Gemini thinking/tool-calling turns can replay `functionCall` history without required `thoughtSignature` metadata, causing Google 400 errors after the earlier sanitizer fix.

## Best Possible Solution

Land or reject PR #75713 after focused review; the end state should prevent native Google replay from sending validator-breaking unsigned current-turn Gemini tool calls, with regression coverage and a changelog entry.

## Reproduction Assessment

Yes. The issue includes concrete Gemini thinking/tool-calling reproduction steps, and current-main source inspection shows an unsigned stored native Google tool call can still serialize without `thoughtSignature`; I did not run a live Gemini request in this read-only sweep.

## Solution Assessment

Yes. Keeping this open while PR #75713 is reviewed, landed, or closed is the narrowest maintainable path; closing before a merged native Google replay repair would lose the active regression tracker.
