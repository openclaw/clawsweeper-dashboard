## Summary

Keep open. Current main still has the raw session-memory persistence bug, but this PR should not merge as-is because the sanitizer misses known token families, can elide valid short replies, can strip prose examples, and still lacks real after-fix hook-run proof.

## What This Changes

The PR adds a `session-memory` sanitizer, wires it into transcript extraction and memory-file writes, and adds sanitizer/handler regression tests.

## Best Possible Solution

Reuse or align with the shared model-token and assistant-visible tool-call sanitizers, keep session-memory-specific elision only for empty, NO_REPLY, or structurally malformed turns, and require redacted proof from a real hook run.

## Reproduction Assessment

Yes, at source level. A session JSONL assistant message containing `<tool_call>...</tool_call><|im_end|>` is returned unchanged by current main, appended to session content, and written under `## Conversation Summary`.

## Solution Assessment

No. Sanitizing before persistence is the right direction, but this implementation duplicates narrower sanitizer logic and can still miss known markers or remove legitimate short/prose content; shared sanitizer reuse or parity is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reuse the shared model-token sanitizer:** `src/hooks/bundled/session-memory/sanitize.ts:20-21`
  - body: The hard-coded token regex still lets template markers OpenClaw already recognizes elsewhere survive, including `[INST]`, `<s>`, `<start_of_turn>`, `<|reserved_special_token_42|>`, and harmony markers. Those can still be persisted under `## Conversation Summary`, so this path should reuse or match the shared sanitizer coverage instead of maintaining a narrower allowlist.
  - confidence: 0.92
- **[P2] Keep valid short replies after token stripping:** `src/hooks/bundled/session-memory/sanitize.ts:92-95`
  - body: The `strippedRatio > 0.5 && cleaned.length < 80` branch drops concise valid replies when one leaked token dominates the character count; for example `2+2=4<|im_end|>` cleans to useful text but is replaced with `[malformed turn elided]`. Short factual replies are useful memory context, so only elide when the remainder is structurally garbage.
  - confidence: 0.9
- **[P2] Guard tool-call stripping against prose examples:** `src/hooks/bundled/session-memory/sanitize.ts:32`
  - body: The open-ended `<tool_call>` regex removes everything from any `<tool_call>` mention through the closing tag or end of string, so ordinary prose such as `Use <tool_call> to invoke tools.` can be collapsed or elided. Reuse the shared tag-aware stripping behavior or check payload shape/code regions before deleting text.
  - confidence: 0.9
