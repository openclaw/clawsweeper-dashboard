## Summary

Keep open: the underlying session-memory bug is source-reproducible on current main, but this PR still has sanitizer correctness problems and lacks accepted after-fix real hook proof from the contributor's setup.

## What This Changes

The branch adds a session-memory sanitizer, wires it into transcript reads and memory writes, and adds handler and sanitizer regression tests.

## Best Possible Solution

Align the session-memory path with the existing shared assistant text/token sanitizers, preserve legitimate short and technical prose, keep focused regression coverage, and require redacted after-fix hook proof from a real run.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main reads a session JSONL assistant message containing `<tool_call>...</tool_call><|im_end|>` unchanged and writes it under `## Conversation Summary`.

## Solution Assessment

No. Sanitizing at transcript and write boundaries is the right shape, but this implementation should reuse or match the shared sanitizer behavior and avoid eliding useful short replies or normal prose about tool tags.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reuse the shared model-token sanitizer:** `src/hooks/bundled/session-memory/sanitize.ts:20-21`
  - body: The new token regex only removes selected literals, so formats OpenClaw already strips elsewhere, including `[INST]`, `<s>`, `<start_of_turn>`, `<|reserved_special_token_42|>`, and harmony markers, can still be persisted and re-injected on the next `/new`. Align this path with the shared sanitizer coverage instead of keeping a separate allowlist.
  - confidence: 0.92
- **[P2] Keep valid short replies after stripping tokens:** `src/hooks/bundled/session-memory/sanitize.ts:92-95`
  - body: The `strippedRatio > 0.5 && cleaned.length < 80` branch still elides concise valid replies when a single leaked token dominates the character count; for example `2+2=4<|im_end|>` becomes useful text but is replaced with `[malformed turn elided]`. Short factual replies are useful memory context, so keep the cleaned reply unless the remainder is structurally garbage.
  - confidence: 0.9
- **[P2] Guard tool-call stripping against prose examples:** `src/hooks/bundled/session-memory/sanitize.ts:32`
  - body: The open-ended `<tool_call>` pattern removes everything from any `<tool_call>` mention through end-of-string, so ordinary prose such as `Use <tool_call> to invoke tools.` can collapse or be elided. Reuse the shared tag parser or check payload shape and code regions before stripping.
  - confidence: 0.9
