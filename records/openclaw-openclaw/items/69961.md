## Summary

Keep open. The underlying session-memory bug is still present on current main, but this PR still has merge-blocking sanitizer correctness issues and lacks after-fix real OpenClaw hook proof showing a cleaned memory file written by the real path.

## What This Changes

The branch adds a session-memory sanitizer, uses it during transcript extraction and memory-file writes, and adds handler/sanitizer regression tests.

## Best Possible Solution

Reuse or match the shared assistant-visible/token sanitizer behavior, keep session-memory-specific elision only for truly empty, NO_REPLY, or structural-garbage turns, and require redacted proof from a real hook run writing a clean memory file.

## Reproduction Assessment

Yes, at source level. A session JSONL assistant message containing `<tool_call>...</tool_call><|im_end|>` is returned unchanged by current main, appended to sessionContent, and written under `## Conversation Summary`.

## Solution Assessment

No. Sanitizing transcript and write boundaries is the right direction, but this implementation duplicates narrower sanitizer logic and can still persist missed tokens or elide legitimate short/prose content.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reuse the shared model-token sanitizer:** `src/hooks/bundled/session-memory/sanitize.ts:20-21`
  - body: The hard-coded token regex still lets template markers OpenClaw already recognizes elsewhere survive, including `[INST]`, `<s>`, `<start_of_turn>`, `<|reserved_special_token_42|>`, and harmony tokens. Those can still be written under `## Conversation Summary` and re-injected, so this path should reuse or match the shared sanitizer coverage instead of maintaining a narrower allowlist.
  - confidence: 0.92
- **[P2] Keep valid short replies after token stripping:** `src/hooks/bundled/session-memory/sanitize.ts:92-95`
  - body: The `strippedRatio > 0.5 && cleaned.length < 80` branch still drops concise valid replies when a single leaked token dominates the character count; for example `2+2=4<|im_end|>` cleans to useful text but is replaced with `[malformed turn elided]`. Short factual replies are useful memory context, so only elide when the remainder is structurally garbage.
  - confidence: 0.9
- **[P2] Guard tool-call stripping against prose examples:** `src/hooks/bundled/session-memory/sanitize.ts:32`
  - body: The open-ended `<tool_call>` regex removes everything from any `<tool_call>` mention through the closing tag or end of string, so ordinary prose such as `Use <tool_call> to invoke tools.` can be collapsed or elided. Reuse the shared tag-aware stripping behavior or check payload shape/code regions before deleting text.
  - confidence: 0.9
