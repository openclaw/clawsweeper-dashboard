## Summary

Keep open. Current main still persists raw session transcript content, and this PR targets a real bug, but the proposed sanitizer is not merge-ready because it misses existing token formats, leaves truncated tool-call XML, can elide valid short replies, and lacks the required changelog entry.

## What This Changes

The PR adds a session-memory sanitizer, wires it into transcript reading and memory persistence, and adds tests for leaked chat-template tokens, raw tool_call XML, NO_REPLY, and malformed turns.

## Best Possible Solution

Consolidate the open approaches into one focused session-memory fix that reuses or exports the existing broad special-token sanitizer, adds session-memory-specific raw and truncated tool-call cleanup plus precise malformed-turn elision, preserves valid short content, and includes the changelog entry.

## Reproduction Assessment

Yes. A synthetic session JSONL message containing `<tool_call>...</tool_call><|im_end|>` or a known special token reproduces the current-main persistence path because the transcript reader returns raw text and the handler writes it under `## Conversation Summary`. The PR findings have small string-level repros such as `2+2=4<|im_end|>` and an unterminated `<tool_call>` block.

## Solution Assessment

No. Sanitizing at the transcript/read and handler/write boundaries is the right shape, but this implementation is not the best mergeable form because it duplicates narrower token logic than the existing helper and has avoidable false-positive and truncation gaps.

  - body: The new allowlist only covers a subset of the chat-template markers already handled in `external-content.ts`, including missing Mistral `[INST]`, Phi `<s>`, GPT-OSS/harmony markers, Gemma turns, and reserved-token variants.   - body: This regex only removes `<tool_call>` blocks that reach a closing `</tool_call>`. If the provider/parser leak is cut mid-stream after `<tool_call>`, the malformed fragment survives sanitization and can still be persisted into the memory file, leaving the poisoning loop open for the parser-race class this PR is defending against.
  - confidence: 0.87
- **[P2] Preserve short replies after token cleanup:** `src/hooks/bundled/session-memory/sanitize.ts:83-86`

Short factual replies are common memory context, so this drops legitimate history instead of only removing the scaffolding token.
  - confidence: 0.9

  - body: This is a user-facing bug fix: it changes what session-memory persists and how future `/new` sessions recover from poisoned context.
