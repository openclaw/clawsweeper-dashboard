## Summary

Keep open. The reported bug is still present on current main and this PR targets the right session-memory path, but the patch still has sanitizer correctness issues and lacks real after-fix behavior proof from an OpenClaw hook run.

## What This Changes

The PR adds a session-memory sanitizer wired into transcript extraction and memory-file writes, plus sanitizer and handler tests for chat-template tokens, raw tool_call XML, role-label lines, and NO_REPLY elision.

## Best Possible Solution

Use shared model-token and assistant-visible tool-call stripping semantics at the session-memory read/write boundaries, limit elision to empty, NO_REPLY, or structurally malformed turns, and require redacted proof from a real hook run before merge.

## Reproduction Assessment

Yes, at source level. A JSONL message containing `<tool_call>...</tool_call><|im_end|>` is extracted unchanged by current main and written under `## Conversation Summary` by the handler.

## Solution Assessment

No. Sanitizing before persistence is the right boundary, but this implementation should reuse or match existing sanitizer contracts and avoid short-reply/prose false positives before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reuse the shared token sanitizer:** `src/hooks/bundled/session-memory/sanitize.ts:20-21`
  - body: The hard-coded token regex still lets known markers survive, including `[INST]`, `<s>`, `<start_of_turn>`, `<|reserved_special_token_42|>`, and harmony markers. Those can still be persisted under `## Conversation Summary`, so this path should reuse or match the shared token sanitizer instead of maintaining a smaller allowlist.
  - confidence: 0.92
- **[P2] Keep valid short replies after token stripping:** `src/hooks/bundled/session-memory/sanitize.ts:92-95`
  - body: The `strippedRatio > 0.5 && cleaned.length < 80` branch drops concise valid replies whenever one leaked token dominates the character count; for example `2+2=4<|im_end|>` cleans to useful text but is replaced with `[malformed turn elided]`. Short factual replies are useful memory context, so only elide when the remainder is structurally garbage.
  - confidence: 0.9
- **[P2] Avoid stripping prose mentions of tool_call:** `src/hooks/bundled/session-memory/sanitize.ts:32`
  - body: The open-ended `<tool_call>` regex removes everything from any literal `<tool_call>` mention through a closing tag or end of string, so ordinary prose like `Use <tool_call> to invoke tools.` is collapsed or elided. Reuse the shared tag-aware stripping behavior or require payload shape before deleting text.
  - confidence: 0.9
