## Summary

Keep this PR open: the linked session-memory poisoning bug is valid and source-reproducible on current main, but this patch still has blocking correctness issues and lacks the required redacted real behavior proof from an after-fix OpenClaw hook run.

## What This Changes

The PR adds a session-memory sanitizer, wires it into transcript extraction and memory writes, and adds sanitizer and handler tests for chat-template tokens, tool_call XML, role-marker lines, and NO_REPLY elision.

## Best Possible Solution

Revise the PR to use or align with the shared token and assistant-visible tool-call sanitizers at the session-memory read/write boundaries, preserve valid concise turns, add focused regressions, and post redacted real hook proof.

## Reproduction Assessment

Yes, at source level. A session JSONL assistant message containing `<tool_call>...</tool_call><|im_end|>` is extracted unchanged on current main, appended by getRecentSessionContent, and written under `## Conversation Summary` by the handler.

## Solution Assessment

No. Sanitizing before persistence is the right boundary, but this implementation should reuse or match the established token/tool-call sanitizers, avoid corrupting prose examples, and preserve concise cleaned replies.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Reuse the established token sanitizer:** `src/hooks/bundled/session-memory/sanitize.ts:20-21`
  - body: The hard-coded token regex is narrower than the repository's existing special-token coverage, so markers such as `[INST]`, `</s>`, `<start_of_turn>`, harmony tokens, and `<|reserved_special_token_N|>` can still be persisted into session memory. Reuse/export the shared sanitizer or align this helper with that established list.
  - confidence: 0.93
- **[P2] Require a tool-call payload before stripping XML:** `src/hooks/bundled/session-memory/sanitize.ts:32`
  - body: This regex treats any literal `<tool_call>` as the start of a block and strips through the close tag or end of turn. That corrupts legitimate summaries like `Use <tool_call> to invoke tools.`, while the shared assistant-visible sanitizer already preserves prose examples and strips only tool-call-shaped payloads.
  - confidence: 0.92
- **[P2] Keep concise cleaned replies:** `src/hooks/bundled/session-memory/sanitize.ts:92-95`
  - body: After token cleanup, a valid short reply such as `2+2=4<|im_end|>` becomes `2+2=4`, but the ratio/length heuristic marks it skipped and replaces it with `[malformed turn elided]`. Short factual replies are useful memory context, so elide only empty, NO_REPLY, or structurally malformed remnants.
  - confidence: 0.9
