## Summary

Keep open; the PR is useful but current head still leaves the expensive secrets snapshot getter on the memory-tool registration path and includes unresolved changelog conflict markers, so it should not merge as-is.

## What This Changes

The PR changes plugin tool resolution to build the initial runtimeConfig from config-only snapshots and adds an Unreleased changelog entry for the #76295 latency fix.

## Best Possible Solution

Route both the initial runtimeConfig and live getRuntimeConfig callback through cheap config-only snapshots while preserving refresh semantics, then cleanly record the fix and regression coverage.

## Reproduction Assessment

Yes, source-reproducible rather than live-timed here. Current main deep-clones the active secrets snapshot, and PR head still passes that getter to ctx.getRuntimeConfig where memory-core invokes it during tool registration.

## Solution Assessment

No. The initial config-only snapshot direction is good, but the patch must also move the live getter off getActiveSecretsRuntimeSnapshot(), resolve the changelog markers, and add focused regression coverage before it is the maintainable fix.
