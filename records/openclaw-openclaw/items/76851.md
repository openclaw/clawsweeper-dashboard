## Summary

The PR should stay open because the latest diff still performs a clone-heavy secrets runtime snapshot read during plugin tool construction, so it does not deliver the claimed latency fix.

## What This Changes

## Best Possible Solution

Use config-only runtime/source snapshots for the one-shot `runtimeConfig`, keep `ctx.getRuntimeConfig()` live for callbacks, and add focused regression coverage plus timing proof for the latency fix.
