## Summary

Keep open: the PR targets a real hot-path regression, but it is not mergeable because the live plugin-tool config getter can still deep-clone the full secrets snapshot during tool registration, and the changelog contains unresolved conflict markers.

## What This Changes

The PR attempts to avoid full secrets snapshot cloning during plugin tool resolution by using config-only runtime snapshots for the initial runtime config and adding a changelog entry.

## Best Possible Solution

Route both the initial runtimeConfig and the live getRuntimeConfig callback through cheap config-only runtime snapshots, preserve live refresh semantics, add focused regression coverage, and resolve the changelog conflict cleanly.

## Reproduction Assessment

Source-reproducible, not live-timed here. Current main deep-clones the active secrets snapshot, and the PR still leaves that getter reachable from bundled memory tool registration during core-plugin-tools prep.

## Solution Assessment

No. The initial config-only snapshot direction is good, but the patch needs to move the live getter off getActiveSecretsRuntimeSnapshot() and remove the changelog conflict markers before it is a maintainable fix.
