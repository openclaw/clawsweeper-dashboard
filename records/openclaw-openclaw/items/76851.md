## Summary

Keep this PR open for normal maintainer validation; the latest branch addresses the earlier review concern by avoiding the clone-heavy secrets snapshot for the one-shot plugin runtime config, but it still needs targeted tests and timing proof before merge.

## What This Changes

The PR changes plugin tool construction to derive the one-shot `runtimeConfig` from config-only runtime snapshot helpers while preserving the live runtime-config getter, and adds a changelog fix entry.

## Best Possible Solution

Merge this PR or an equivalent patch after targeted integration tests, formatting, and live or trace-log timing confirm the prep stage no longer pays the full secrets snapshot clone.

Yes for the source-level regression: current main calls `getActiveSecretsRuntimeSnapshot()` during plugin tool construction, and that getter clones the active secrets snapshot. I did not remeasure the reported 8.3s live latency in this read-only pass.

## Solution Assessment

Yes, with a validation caveat. The branch uses config-only snapshot helpers for the one-shot `runtimeConfig` and preserves the documented live getter for long-lived callbacks, which is the narrow maintainable direction.
