## Summary

Keep open: this is a focused, valid iMessage bug-fix PR. Current main strips leading corruption markers in the in-memory echo cache but not in the persisted echo cache, and the PR aligns those normalizers with focused persisted-cache coverage.

## What This Changes

The PR extracts iMessage echo text corruption-marker stripping into a shared monitor helper and applies it to the persisted echo cache, with a focused persisted-cache regression test.

## Best Possible Solution

Land the shared leaf helper and persisted-cache normalization after CI, preserving leading-only stripping and avoiding any new plugin or core surface.

## Reproduction Assessment

Yes. Current main source shows the persisted cache compares text after only CRLF cleanup and trim, so a leading NUL/replacement/BOM marker fails to match the clean stored send while the in-memory cache strips it.

## Solution Assessment

Yes. A shared leaf helper is the best small fix because the persisted and in-memory echo caches need identical normalization and importing either cache into the other would create an owner-cycle risk.
