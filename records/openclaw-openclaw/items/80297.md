## Summary

Keep open: current main has useful label/displayName support, but the requested readable-title path is still incomplete for unlabeled Control UI dashboard sessions and the exact `sessions.patch.displayName` API does not exist.

## What This Changes

The issue requests readable session names in the Control UI session picker via `displayName` in `sessions.list`, a way to set it with `sessions.patch`, and picker fallback away from raw `sessionKey`.

## Best Possible Solution

Make the picker show a stable human title for unlabeled dashboard sessions by reusing label/displayName where present and a bounded derived-title path where absent, with protocol, UI types, docs, and tests aligned if `displayName` becomes patchable.

## Reproduction Assessment

Yes, source-reproducible: implicit Control UI sessions are generated as unlabeled `agent:<agent>:dashboard:<uuid>` keys, the UI does not request derived titles, and the picker falls back to the raw key when label/displayName are absent.

## Solution Assessment

No, current main is only a partial solution: label/displayName rows render well, but unlabeled dashboard rows and the requested `displayName` patch contract still need a maintainer API/UX decision.
