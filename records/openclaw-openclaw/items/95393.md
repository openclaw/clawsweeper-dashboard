## Summary

Keep open: this PR targets a real current and shipped doctor false-positive path, but the new skipped-probe guard is too broad because it can bypass existing local-provider setup guidance when no local model path is configured.

## What This Changes

Adds an early return for skipped gateway memory probes in local-provider doctor diagnostics and a regression test for an explicit local GGUF model with a skipped probe.

## Best Possible Solution

Narrow the skipped-probe return so configured local models do not get a false readiness warning, while missing local model setup still reports guidance and timeout or checked failures still warn.

## Reproduction Assessment

Yes, at source level: current main passes skipped gateway memory probes into noteMemorySearchHealth, and the local branch warns without checking skipped while sibling provider branches already treat the same shape as neutral. I did not run a live command because this review is read-only.

## Solution Assessment

No, not as written: the renderer is the right layer, but the skipped return should not run before preserving the local-provider missing-model check. The safer fix suppresses the false readiness warning only for configured local models and keeps setup guidance for missing local config.
