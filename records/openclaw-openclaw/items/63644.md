## Summary

Keep open. Current main still lacks the viewport/body/chat-shell parts of the iPhone Safari layout fix, and the PR remains a useful source patch, but the current branch is not merge-ready because its iOS textarea font-size override targets a stale selector and the user-facing fix is missing the required changelog entry.

## What This Changes

The PR changes Control UI CSS to lock the root/app/chat shell to viewport-aware heights, hide body overflow, remove the chat-shell min-height, and add an iOS WebKit textarea font-size override for input-focus zoom.

## Best Possible Solution

Land one clean Control UI fix that preserves the viewport/body/chat-shell scrollbar improvements, applies the 16px iOS input rule to the current composer and other affected text inputs without reintroducing a viewport zoom cap, adds focused CSS assertions and a changelog entry, and coordinates the overlapping input-zoom work with #64651 and #64673.

## Reproduction Assessment

Yes. The PR provides iPhone Safari steps and screenshots, and source inspection on current main still shows both the viewport/body layout gap and an active chat textarea below the iOS 16px focus threshold; live iOS verification remains the missing end-to-end proof.

## Solution Assessment

No. The direction is maintainable after moving away from `maximum-scale=1`, but the current branch misses the active textarea selector and lacks the release-note/test guardrails needed for a user-facing Control UI fix.
