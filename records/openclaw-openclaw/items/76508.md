## Summary

Keep open: current main still rejects community plugins, and this PR's manifest opt-in is incomplete because real manifest parsing would drop the new field before the registry check sees it.

## What This Changes

The PR adds a manifest opt-in for community plugin `openKeyedStore` access with a lower namespace cap, updates docs/changelog/tests, and changes `sessions_send` A2A announce reply delivery.

## Best Possible Solution

If maintainers approve the feature direction, wire the opt-in through the real manifest parser and typed contract model, add parser/registry coverage and docs, then split or document the `sessions_send` change.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a workspace plugin manifest with `contracts.usesKeyedStore: true` would pass through `normalizeManifestContracts`, which currently preserves only known list-valued contract keys, so the PR's registry check would not see the opt-in.

## Solution Assessment

No. The direction is plausible, but the patch is incomplete until the manifest parser, typed contract helpers, manifest docs, and regression coverage all agree; the `sessions_send` behavior change also needs separate handling.
