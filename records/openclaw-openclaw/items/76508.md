## Summary

Keep open: current main still has the bundled-only keyed-store gate, and this PR's manifest opt-in does not work through the real manifest parser yet. The feature may be useful, but the parser/test gap should block merge until fixed.

## What This Changes

This PR adds a `contracts.usesKeyedStore` opt-in, docs, changelog, and runtime tests for community plugin keyed-store access, and also changes A2A `sessions_send` announce reply delivery.

## Best Possible Solution

If maintainers approve the feature direction, preserve the manifest field through the real parser/registry path with regression coverage, then split or document the unrelated `sessions_send` fix before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a workspace plugin manifest with `contracts.usesKeyedStore: true` would be normalized by `normalizeManifestContracts`, which currently preserves only known string-list contract keys, leaving `record.contracts?.usesKeyedStore` absent for the PR's registry check.

## Solution Assessment

No. The proposed direction is plausible, but the patch is incomplete because it does not wire the new manifest contract through parsing and coverage; the unrelated `sessions_send` change should be split out or documented.
