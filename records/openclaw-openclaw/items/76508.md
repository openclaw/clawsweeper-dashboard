## What This Changes

This PR adds a manifest opt-in, docs, changelog text, and tests for community plugin `openKeyedStore` access, and also changes A2A `sessions_send` announce reply delivery.

## Best Possible Solution

If maintainers approve the SDK direction, preserve `contracts.usesKeyedStore` through manifest normalization with parser/registry coverage, then split or document the `sessions_send` change before merge.

Yes. Source inspection gives a high-confidence path: a community plugin manifest with `contracts.usesKeyedStore: true` is normalized by `normalizeManifestContracts`, which currently drops unknown boolean fields before `registry.ts` checks `record.contracts?.usesKeyedStore`.
