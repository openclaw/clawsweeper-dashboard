## Summary

Keep open. Current main and v2026.6.5 still pass the `before_prompt_build` prompt into Active Memory message mode without a cap or slim-intent field, and the linked fix at https://github.com/openclaw/openclaw/pull/92099 is open but unmerged.

## What This Changes

The issue requests a native Active Memory cap or slim intent boundary so `queryMode: "message"` cannot pass a full assembled request envelope into the hidden recall runner.

## Best Possible Solution

Review, prove, and either land or reject https://github.com/openclaw/openclaw/pull/92099; after a fix merges, close this issue with the merge commit and release or main-only provenance.

Unclear as a final product choice. The linked PR is a plausible bounded fix, but maintainers still need to choose whether the durable contract should be a new `messageMaxChars` config option or a slim current-user-intent hook field.
