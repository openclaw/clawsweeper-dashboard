The issue asks for Active Memory `queryMode: "message"` to bound or slim the latest-user input so the hidden recall runner does not receive full assembled request envelopes.

## Best Possible Solution

Keep this issue open until the linked candidate PR is repaired and landed, or replaced by one owner-approved Active Memory input-boundary fix with realistic Telegram/direct-session proof.
