## Summary

Keep open. Current main and v2026.6.6 still pass the assembled `event.prompt` into Active Memory message mode without a cap or slim-intent field, and the closing fix PR remains open and conflicting.

## What This Changes

Active Memory message-mode recall should not send the full assembled prompt envelope to the hidden memory sub-agent; it needs a bounded or slim latest-user-intent input.

## Best Possible Solution

Review and resolve the linked Active Memory fix path, choose the durable bounded-input contract, then close this issue only after a fix merges with large-envelope proof.

## Reproduction Assessment

Yes, source-reproducible. Current main passes `event.prompt` into Active Memory message mode and `buildQuery()` returns it unchanged; the reporter also supplied v2026.6.5 logs showing 500K-character query input.

## Solution Assessment

Unclear as a final product choice. Adding a `messageMaxChars` cap is a plausible bounded fix, but maintainers still need to decide whether the better contract is a new plugin config key or a slimmer hook/runtime intent field.
