## Summary

Keep open. Current main and v2026.6.8 still pass the assembled prompt into Active Memory recall, message mode returns that value unchanged, and the open closing pull request is not merged, so the issue still needs maintainer review.

## What This Changes

The issue asks for Active Memory message-mode recall to stop forwarding full assembled prompt envelopes by using a bounded latest-message input, a slim user-intent field, or equivalent logged behavior.

## Best Possible Solution

Land or replace the linked Active Memory fix with a maintainer-approved current-user-intent boundary that keeps message mode bounded, preserves useful recall, and exposes effective query diagnostics.

## Reproduction Assessment

Yes. Source inspection shows current main passes event.prompt into Active Memory and message mode returns it unchanged; the reporter also supplied logs with very large queryChars and bounded searchQueryChars, but this read-only review did not run a live Telegram reproduction.

## Solution Assessment

Unclear as a final product choice. The fix belongs in Active Memory's current-message boundary, but maintainers still need to choose between a config cap, a slim-intent hook contract, or an equivalent bounded behavior.
