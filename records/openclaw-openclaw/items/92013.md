## Summary

Keep open. Current main and the latest release still let Active Memory `queryMode: "message"` pass the hook prompt into the hidden recall run without a message cap or slim-intent field, and the linked closing fix PR is open but unmerged.

## What This Changes

The issue asks Active Memory message mode to stop sending full assembled request envelopes to the hidden recall runner by using a latest-message cap, slim intent input, or equivalent bounded behavior with diagnostics.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/92099 so message mode receives a bounded current-user intent without losing current-turn semantics, then close this issue after merge provenance is available.

## Reproduction Assessment

Yes. Source inspection shows current main passes `event.prompt` into Active Memory message mode and `buildQuery()` returns it unchanged; the reporter also supplied v2026.6.5 logs showing 500K-character query input with bounded search query size.

## Solution Assessment

Unclear. The issue direction is valid, but maintainers still need to choose whether the best durable fix is a new plugin config cap, a slim hook intent field, or a smaller change that preserves current-turn semantics without adding unnecessary config surface.
