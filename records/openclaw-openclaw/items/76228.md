## Summary

Current main and shipped release code now implement the reported configured-subagent discovery and spawn behavior: configured `agents.list[]` targets are passed through the shared allowlist policy, listed by `agents_list`, and accepted by `sessions_spawn` when allowlisted.

The issue requests that custom agents configured in `agents.list` appear in `agents_list` and be accepted by `sessions_spawn` when included in `subagents.allowAgents`.

## Best Possible Solution

Keep the shipped shared target-policy implementation and ask for a new current-release repro only if configured allowlisted agents are still omitted or rejected.
