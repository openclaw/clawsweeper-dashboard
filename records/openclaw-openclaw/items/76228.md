## Summary

Close as implemented: current main and the shipped v2026.6.6 release now resolve custom configured agents through the same configured-agent allowlist path for both `agents_list` and native `sessions_spawn`.

## What This Changes

The issue reports that a custom agent listed in `agents.list` and allowed via `subagents.allowAgents` was omitted from `agents_list` and rejected by `sessions_spawn(agentId=...)`.

## Best Possible Solution

Keep the shipped configured-agent allowlist behavior; ask for a fresh report only if the same config still fails on v2026.6.6 or newer.

## Reproduction Assessment

No, not against current main: the current source path and regression tests show a configured allowlisted custom agent is included and accepted. The original failure was reported against older 2026.4.x builds, while v2026.6.6 contains the fixed behavior.

## Solution Assessment

Yes. The maintainable solution is the existing configured-agent registry plus `subagents.allowAgents` policy path, not a new runtime registration requirement or manual agent directory rule.
