## Summary

Keep open: this PR targets a valid linked bug, but it is not mergeable because it breaks the status handler import, misses the common swallowed pricing-fetch failure path, and reads CLI-local status state instead of gateway-owned pricing state.

## What This Changes

The branch adds a model-pricing bootstrap failure ledger and wires optional `modelPricing` data into health/status JSON plus human-readable health and status output.

## Best Possible Solution

Keep the additive health/status direction, but preserve the status RPC import, record failures where catalog fetch errors are currently swallowed, and render gateway-owned pricing degradation through health, `openclaw status`, and gateway status with focused regression coverage.

## Reproduction Assessment

Yes. CI reproduces the TypeScript failure for the deleted `getStatusSummary` import, and source inspection shows catalog fetch errors are caught before the new outer failure ledger can see them.

## Solution Assessment

No. The direction is narrow, but this implementation should record failures at the per-source catch points or shared gateway pricing state and have status output consume gateway-owned state rather than CLI-local process state.
