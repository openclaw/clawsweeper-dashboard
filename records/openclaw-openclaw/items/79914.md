## Summary

Keep open: this PR addresses a real linked bug, but the current head is not mergeable because it drops a required status import and misses the common catalog-fetch failure path it is meant to surface.

## What This Changes

The PR adds process-local model-pricing bootstrap failure tracking and exposes optional `modelPricing` health/status JSON plus CLI health/status text.

## Best Possible Solution

Keep the additive health/status direction, but record failures at the per-source catch points or shared pricing cache state and render gateway-owned pricing state in health, `openclaw status`, and gateway status with focused regression coverage.

## Reproduction Assessment

Yes for the review findings: PR head leaves `getStatusSummary` undefined in `server-methods/health.ts`, and catalog fetch failures are caught inside `refreshGatewayModelPricingCache` before the new outer ledger can observe them. I did not run a live gateway during this read-only review.

## Solution Assessment

No, this is not the best way to solve the issue yet. The fix should preserve the status RPC import, record the swallowed fetch failures where they occur, and consume gateway-owned pricing health instead of a CLI-local summary.
