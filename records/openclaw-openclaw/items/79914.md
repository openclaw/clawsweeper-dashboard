## Summary

Keep open: the direction is useful, but the patch currently fails typecheck, misses the swallowed catalog-fetch failure path it is meant to surface, reads process-local state for normal status output, and still lacks sufficient exact real-behavior proof.

## What This Changes

The PR adds an in-memory model-pricing failure ledger and exposes optional degraded pricing state through health/status types, health output, status overview rows, and cached gateway health snapshots.

## Best Possible Solution

Keep the additive degraded-pricing direction, but record structured failures at the per-source catalog fetch catches, clear only after a successful no-failure refresh, preserve the status RPC import, and render gateway-owned pricing state in health, status, and gateway status with focused coverage.

## Reproduction Assessment

Yes. Source inspection shows a high-confidence path: make OpenRouter or LiteLLM catalog fetch reject, `refreshGatewayModelPricingCache` catches and resolves, and the PR's outer catch ledger never records degradation; PR-head CI also reproduces the missing `getStatusSummary` import.

## Solution Assessment

No. The solution should keep the additive health/status warning, but it needs to record failures at the actual catch boundary and consume gateway-owned state for user-facing status surfaces rather than duplicating process-local CLI state.
