## Summary

Keep open: this PR targets a real shipped Skill Workshop UI bug, but it is not merge-ready because it can bypass Gateway's configured-default fallback for default/global sessions and the contributor proof is still test-only.

## What This Changes

The branch derives a Skill Workshop agent id from the current UI session and sends it with list, inspect, apply, and reject proposal RPCs.

## Best Possible Solution

Land one canonical Control UI fix that passes `agentId` only for parsed agent-scoped sessions, leaves default/global requests unscoped for Gateway fallback, covers all proposal actions, and includes redacted live proof.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.6 send unscoped Skill Workshop proposal RPCs from the UI while Gateway and CLI already support agent-scoped proposal workspaces. I did not run a live Control UI plus Gateway repro.

## Solution Assessment

No. Threading the existing optional `agentId` contract through the UI is the right layer, but this implementation should omit `agentId` for default/global sessions instead of forcing an explicit fallback value.
