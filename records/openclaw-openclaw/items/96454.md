## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Detach attach signal handlers on child completion:** `src/cli/attach-cli.ts:175-176`
  - body: The attach command registers process-level SIGINT/SIGTERM handlers for every spawned harness but never removes them when the child exits or errors. In a long-lived or test-hosted CLI process, a completed attach run can leave stale handlers that swallow later SIGINTs or send SIGTERM to an already-finished child; use the shared child-process bridge pattern or equivalent detach-on-exit/error cleanup.
  - confidence: 0.86
- **[P2] Validate the grant expiry before formatting it:** `src/cli/attach-cli.ts:88-95`
  - body: `attach.grant` response validation omits `expiresAtMs`, but the command immediately formats it with `new Date(grant.expiresAtMs).toISOString()`. A malformed or version-skewed response can still throw here instead of using the intended boundary error path, so include `expiresAtMs` in the validation and regression test.
  - confidence: 0.78
