## Summary

Keep open: current main still rejects and prunes the documented explicit `main` binding shape, the latest release contains the same behavior, and the linked closing PR is still open rather than merged.

## What This Changes

The issue reports that route bindings targeting the implicit default agent `main` are rejected at config load and removed by `doctor --fix` when `agents.list` is populated.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89419, or a replacement, with exact `main` preservation across schema validation, doctor pruning, route resolution, and configured binding compilation.

## Reproduction Assessment

Yes, source-reproducible: current main still validates and prunes `bindings[].agentId` against only `agents.list`, while the documented ACP example routes fallback traffic to unlisted `main`. I did not run tests because this review is read-only.

## Solution Assessment

Yes, preserving exact `main` is the right narrow fix, but it needs to cover schema, doctor repair, route resolution, and configured binding compilation. A schema-only or doctor-only repair would leave sibling routing behavior inconsistent.
