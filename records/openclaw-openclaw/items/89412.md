## Summary

Keep open: current main and the latest release still reject or prune explicit `main` route bindings when `agents.list` is populated, and https://github.com/openclaw/openclaw/pull/89419 is the open candidate fix rather than a reason to close this issue early.

## What This Changes

The issue requests allowing explicit implicit-default `main` route bindings when `agents.list` is populated and preserving those bindings during `doctor --fix`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89419, or an equivalent replacement, with consistent exact-`main` preservation across schema validation, doctor pruning, and route resolution plus focused regression proof.

## Reproduction Assessment

Yes. Current source still validates and prunes `bindings[].agentId` against only `agents.list`, so a config with non-empty `agents.list` and `agentId: "main"` follows the reported failure path; I did not run the reproducer because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve exact `main` consistently in config schema validation, doctor pruning, and route resolution while still rejecting genuinely missing agent ids.
