## Summary

Keep open: current `main` and the latest release still reject and prune explicit `main` bindings when `agents.list` is populated, and the linked closing PR is open rather than merged.

## What This Changes

The issue reports that route bindings to the implicit default agent `main` are rejected during config validation and removed by `doctor --fix` when `agents.list` is non-empty.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/89419 or a replacement that preserves exact `main` across schema validation, doctor repair, route resolution, and configured-binding compilation with focused regression and real doctor/routing proof.

## Reproduction Assessment

Yes. This is source-reproducible on current `main`: both the schema and doctor predicates omit `DEFAULT_AGENT_ID` when `agents.list` is non-empty, and no tests were run because this review is read-only.

## Solution Assessment

No current-main solution has landed yet. The best fix is the narrow exact-`main` preservation path across validation, doctor repair, and sibling route consumers; a schema-only repair would be too narrow.
