## What This Changes

The PR adds Harmony-format plain-text tool-call parsing to `src/plugin-sdk/tool-payload.ts` and expands parser unit coverage for bare and delimited Harmony tool-call text.

## Best Possible Solution

Finish this as a narrow LM Studio/parser fix that handles Harmony consistently in parsing, stream suppression, assistant-visible stripping, focused tests, changelog, and real runtime proof.

Yes. The parser-level failure is source-reproducible on current main because the shared parser only accepts `[` openings and the LM Studio recovery path delegates to that parser; I did not execute the reporter's command in this read-only review.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Buffer Harmony text before promoting it:** `src/plugin-sdk/tool-payload.ts:260-264`
  - body: Once this fallback can parse `commentary to=... code` blocks, the LM Studio stream wrapper still only buffers possible plain-text tool calls whose streamed text starts with `[`. A streamed Harmony delta will be flushed as ordinary text before the final `done` message is promoted, unlike the existing bracketed path, so users can still see the raw tool-call payload. Please extend the LM Studio prefix/buffering path and add a Harmony stream regression test that asserts no `text_delta` is emitted.
  - confidence: 0.88
- **[P2] Strip Harmony blocks from visible assistant text:** `src/plugin-sdk/tool-payload.ts:112-113`
  - body: The PR makes Harmony blocks parseable, but `stripPlainTextToolCallBlocks` still returns early unless the text contains a bracketed `[tool]` marker. Assistant-visible cleanup already calls that helper, so any Harmony tool block that reaches fallback or persisted text surfaces will still be delivered literally. Please update the stripper guard and add visible-text coverage for a standalone Harmony block.
  - confidence: 0.8
- **[P3] Add the required changelog entry:** `CHANGELOG.md:108`
  - body: This is a user-facing LM Studio fix for runs that can currently exit with zero tool invocations, so repo policy requires a single-line `CHANGELOG.md` entry under the active release section.
  - confidence: 0.93
