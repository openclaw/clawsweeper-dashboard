## Summary

Keep open: current main and the latest shipped release still use the existing compaction summarization path by default, while the requested running Session Memory compact basis would add new session-state storage, config, privacy, and ownership decisions. Existing context-engine hooks and compaction-provider seams are adjacent and useful, but they do not settle whether this should become a core default, opt-in engine behavior, or plugin-owned strategy.

## What This Changes

The issue proposes maintaining a running Session Memory summary and reusing it for auto-compaction so each compact does not require a fresh large summarization model call.

## Best Possible Solution

Decide the ownership boundary first, then implement a guarded opt-in running-summary design with explicit privacy settings, natural-break thresholds, and regression coverage for auto and manual compaction paths.
