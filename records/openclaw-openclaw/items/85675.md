## Summary

Keep open: current main still uses the built-in compaction summarization path, while the proposed running Session Memory compact basis would add new default session-state storage, config, and privacy semantics that need maintainer product/architecture review.

## What This Changes

Request to maintain a running per-session summary and reuse it for auto-compaction so OpenClaw can avoid a fresh per-compact summarization LLM call.

## Best Possible Solution

Decide the ownership boundary first, then implement the running summary either as an opt-in core/context-engine feature or a plugin with explicit privacy settings, natural-break thresholds, and regression coverage for auto/manual compaction paths.

## Reproduction Assessment

Not applicable: this is a feature/performance proposal rather than a reported broken existing behavior. Source inspection confirms current main still summarizes through the existing compaction path, but there is no failing reproduction to run.

## Solution Assessment

Unclear: the proposal is plausible, but the best implementation depends on whether maintainers want this in core defaults or as a context-engine/plugin strategy. A safe design needs privacy, upgrade, and compatibility decisions before code work.
