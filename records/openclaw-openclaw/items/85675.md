## Summary

Keep open: current main still has the standard compaction summarization path and only command/reset session-memory hooks, while the requested running Session Memory compact basis would add default session-state storage, update scheduling, privacy policy, and compaction behavior that need maintainer product and architecture review.

## What This Changes

The issue requests a running per-session memory layer that can be reused as the basis for auto-compaction so compaction does not need a separate summarization LLM call each time.

## Best Possible Solution

Decide the ownership and privacy contract first, then implement the running summary as an explicit core/context-engine capability or plugin strategy with natural-break thresholds, permission boundaries, and regression coverage for automatic and manual compaction paths.
