  - body: `search.messages` can query workspace-visible content under a user token, but this branch does not accept a target channel or apply the current Slack read-target allowlist before searching. Constrain search to approved targets or make broader workspace search an explicit maintainer-approved contract.
  - confidence: 0.9
- **[P1] Keep advanced Slack scopes out of the default manifest:** `extensions/slack/src/setup-shared.ts:42-45`
  - body: Bookmarks, canvases, and reminders are runtime opt-in action groups, but the default setup manifest asks every new Slack install for their read/write bot scopes. Keep the base manifest least-privilege or move these scopes behind an explicit opt-in setup path.
  - confidence: 0.88
- **[P2] Include user-token-only accounts in search discovery:** `extensions/slack/src/message-actions.ts:11-15`
  - body: Search discovery checks `userTokenSource` only after the account list has already been filtered to bot-token accounts, so a user-token-only account that can execute search is never advertised. Build the search-capable account set before the bot-token filter or special-case search discovery.
  - confidence: 0.87
- **[P2] Parse numeric postAt strings as epoch seconds:** `extensions/slack/src/action-runtime.ts:214-217`
  - body: `postAt` accepts strings, but digit-only Unix-second strings still fall through to `Date.parse`, so serialized payloads like `"1725000000"` throw even though Unix seconds are supported. Parse numeric strings before date parsing or narrow the schema/docs to numbers only.
  - confidence: 0.88
