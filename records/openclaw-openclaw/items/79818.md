Overall confidence: 0.91

Full review comments:

- **[P1] Authorize Slack search read targets:** `extensions/slack/src/action-runtime.ts:577-583`
  - body: The new search action takes a free-form query and calls Slack search.messages with user-token read options, but never requires an authorized channel target or calls assertSlackReadTargetAllowed. Current Slack read paths authorize channelId before reading messages, files, pins, or reactions, so this can return workspace-visible results outside OpenClaw channel allowlists unless maintainers explicitly approve that new contract.
  - confidence: 0.92
- **[P1] Keep advanced Slack scopes out of the default manifest:** `extensions/slack/src/setup-shared.ts:42-45`
  - body: The advanced action groups are opt-in, but the default setup manifest now asks every new Slack install for bookmarks, canvases, and reminders read/write scopes. That broadens consent and token authority before users enable those actions; keep the base manifest least-privilege or add an explicit setup opt-in with docs and tests.
  - confidence: 0.9
- **[P2] Include user-token-only accounts in search discovery:** `extensions/slack/src/message-actions.ts:11-13`
  - body: Action discovery filters out every account without a bot token before checking whether search has a user token. A user-token-only read/search account with actions.search enabled therefore will not advertise search even though the runtime requires a user token for that action.
  - confidence: 0.84
- **[P2] Parse numeric epoch strings before Date.parse:** `extensions/slack/src/action-runtime.ts:215-217`
  - body: The schema accepts postAt as a string and the error says Unix seconds are supported, but digit-only strings such as "1725000000" go through Date.parse and throw. Parse numeric strings as epoch seconds before falling back to date-string parsing.
  - confidence: 0.88
