Full review comments:

- **[P2] Keep user-token-only Slack search discoverable:** `extensions/slack/src/message-actions.ts:13`
  - body: `listSlackMessageActions` filters out accounts without a bot token before checking `isActionEnabledWithUserToken`, so an account configured with only `userToken` and `actions.search: true` never advertises `search`. That conflicts with the runtime/docs contract that search is a user-token read action; keep user-token-capable accounts in the search discovery path or explicitly change the contract and tests.
  - confidence: 0.9
- **[P2] Parse numeric postAt strings as epoch seconds:** `extensions/slack/src/action-runtime.ts:215-217`
  - body: `postAt` is exposed as `number | string`, but string values are sent straight to `Date.parse`; a common Unix-seconds payload like `"1725000000"` throws even though Slack `chat.scheduleMessage` expects epoch seconds. Parse digit-only strings before date parsing or narrow the schema/docs to reject them deliberately.
  - confidence: 0.83
