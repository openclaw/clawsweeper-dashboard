## Summary

Keep open. Current main still has the source-visible onboarding prompt mismatch, and open closing PRs #74407 and #74891 already target the same bug, so this issue should stay open until maintainers land or replace one canonical fix.

## What This Changes

The issue asks `openclaw onboard` to stop prompting for OpenAI or ElevenLabs API keys after listing the local `openai-whisper` skill as keyless.

## Best Possible Solution

Land or replace one narrow `setupSkills` fix that separates keyless dependency installation from keyed credential setup, with mixed local/API regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: with missing local `openai-whisper` plus missing env-gated `openai-whisper-api` or `sag`, run `openclaw onboard` and continue through skill setup; current main can show the keyless install option and then prompt for the keyed missing skill.

## Solution Assessment

No for current main. The maintainable fix should either scope credential prompts to the keyed skill path the user opted into or clearly separate dependency installation from credential setup before prompting.
