## Summary

Keep open. Current main still has the source-visible onboarding mismatch, and two open PRs already use closing syntax for this issue, so the issue should remain open until maintainers land or replace one canonical fix.

## What This Changes

The issue requests that `openclaw onboard` stop asking for OpenAI Whisper or ElevenLabs API keys after showing a keyless local Whisper dependency option.

## Best Possible Solution

Review and land or replace one narrow fix that separates keyless dependency installation from keyed credential setup, with regression coverage for mixed local/API skill status.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with local `openai-whisper` missing and a keyed missing skill such as `openai-whisper-api` or `sag`, `openclaw onboard` can list the keyless install option and then prompt for a key from the later all-missing-skills env loop.

## Solution Assessment

No for current main. The maintainable fix should either gate credential prompts to the keyed skill path the user opted into or add a clear credential phase that cannot be confused with the keyless local dependency install.
