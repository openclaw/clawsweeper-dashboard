## Summary

Keep open. Current main still has the source-visible onboarding prompt mismatch, and open closing PRs #74407 and #74891 already target the same bug, so this issue should stay open until one fix lands or is replaced.

## What This Changes

The report asks onboarding not to request OpenAI Whisper or ElevenLabs API keys after showing the local `openai-whisper` skill as keyless.

## Best Possible Solution

The onboarding skill setup should clearly separate dependency installation from credential setup and only ask for secrets on the keyed path the user actually chose, with mixed local/API regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: with missing local `openai-whisper` plus a missing env-gated skill such as `openai-whisper-api` or `sag`, run `openclaw onboard` and continue through skill setup; current main can show a keyless install option and then prompt for a keyed missing skill.

## Solution Assessment

No for current main. The maintainable fix should either gate credential prompts to the selected keyed skill path or add an unmistakable credential phase before prompting, rather than leaving the current all-missing-skills env loop unchanged.
