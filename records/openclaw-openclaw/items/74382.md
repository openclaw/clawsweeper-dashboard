## Summary

Keep open. Current main still has the reported mixed keyless/keyed onboarding behavior, and open closing PRs #74407 and #74891 already target the issue, so this issue should stay open until maintainers land or replace one canonical fix.

## What This Changes

The issue asks for `openclaw onboard` to stop asking for OpenAI Whisper or ElevenLabs API keys in a flow that first advertises the selected local Whisper skill as not needing an API key.

## Best Possible Solution

Land or replace one narrow `setupSkills` fix that clearly separates dependency installation from credential setup and adds mixed keyless/keyed regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: have missing local `openai-whisper` plus missing env-gated `openai-whisper-api` or `sag`, then run `openclaw onboard` and proceed through skill setup.

## Solution Assessment

No for current main. The narrow maintainable solution belongs in `setupSkills`, either by scoping credential prompts to selected/eligible keyed skills or by clearly separating keyed credential setup with regression coverage.
