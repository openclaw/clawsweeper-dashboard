## Summary

Keep open. Current main still presents missing skill dependency installation separately from a later credential loop over all missing `primaryEnv` skills, while the bundled local `openai-whisper` skill remains keyless and the API/TTS skills remain separate keyed skills. The issue is also paired with open closing PRs #74407 and #74891, so it should not be closed before one lands or is explicitly replaced.

## What This Changes

The issue asks `openclaw onboard` to avoid asking for OpenAI Whisper or ElevenLabs API keys after presenting the local `openai-whisper` skill as not requiring an API key.

## Best Possible Solution

Keep this issue open until one linked PR lands or is replaced. The desired end state is a narrow `setupSkills` change with regression coverage: local `openai-whisper` remains keyless, credential prompts are clearly scoped to actual env-gated skills, and skipped binary-backed skills do not receive surprise API-key prompts unless that is an explicit product choice.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to make skill status contain missing keyless `openai-whisper` plus missing env-gated `openai-whisper-api` or `sag`; current `setupSkills` can show the install list first and then prompt for every missing `primaryEnv` skill.

## Solution Assessment

Unclear until PR review. The narrowest maintainable fix appears to be selected-skill gating plus clear credential-phase copy in `src/commands/onboard-skills.ts`, with focused tests in `src/commands/onboard-skills.test.ts`; copy-only clarification may not fully address the skipped-install case described by #74891.
