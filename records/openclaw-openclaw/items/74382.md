## What This Changes

The issue reports that `openclaw onboard` can show keyless local `openai-whisper` setup and then ask for OpenAI or ElevenLabs API keys for separate keyed skills.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/74891 with a narrow `setupSkills` fix that gates credential prompts to selected/configured skills while preserving required prompts for non-presented skills.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with missing local `openai-whisper` plus missing `openai-whisper-api` or `sag` credentials, `openclaw onboard` can show the keyless install option and then prompt for the separate keyed skill because the credential loop scans all missing skills.

## Solution Assessment

Yes for issue handling. Keeping this issue open until the linked canonical fix lands is the narrowest maintainable path; the code fix should live in `setupSkills`, not in the individual skill manifests.
