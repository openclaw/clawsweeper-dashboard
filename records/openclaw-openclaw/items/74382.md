## What This Changes

## Best Possible Solution

Land or replace the linked selection-gated `setupSkills` fix so onboarding only prompts for credentials for skills the user intentionally selected, while preserving legitimate env-only credential prompts.

## Reproduction Assessment

Yes, at source level: current `setupSkills` prompts every missing `primaryEnv` skill after dependency selection, while the local and API-backed Whisper skill metadata create the reported keyless/keyed mix. I did not run interactive onboarding because this review is read-only.

## Solution Assessment

Yes, the best fix is to gate the shared `setupSkills` credential prompt by the user's selected setup choices instead of special-casing Whisper metadata or relying only on explanatory copy.
