## Summary

Keep open. Current main still prompts for credentials from the full missing env-gated skill set after showing keyless install options, and open closing PRs #74407 and #74891 already target the same bug.

## What This Changes

Stop `openclaw onboard` from making keyless local Whisper setup look like it requires OpenAI or ElevenLabs API keys.

## Best Possible Solution

Land one canonical `setupSkills` fix that separates dependency install selection from credential setup and adds mixed keyless/keyed regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists when `openai-whisper` is missing alongside missing env-gated skills such as `openai-whisper-api` or `sag`: the local keyless skill appears in the install picker, then the full missing env-gated set is prompted for credentials.

## Solution Assessment

No for current main. The narrow maintainable fix belongs in `setupSkills`, either by gating credential prompts to the intended selected skill path or by clearly separating the credential phase with regression tests.
