## Summary

Keep open. Current main still has the reported onboarding mismatch: `setupSkills` shows keyless missing binary skills in the install picker, then later prompts for every missing env-gated skill from the full missing set. The issue also has two open closing PRs, #74407 and #74891, so the right cleanup path is to review and land or replace one of those candidates rather than close the issue early.

## What This Changes

`openclaw onboard` should avoid asking for OpenAI or ElevenLabs skill API keys in a way that appears attached to the keyless local `openai-whisper` install option.

## Best Possible Solution

Keep the issue open until one linked PR lands or is replaced. The desired end state is a narrow onboarding fix that keeps local `openai-whisper` keyless, scopes credential prompts to the skill the user is actually configuring for bin-backed skills, preserves env-only prompts where appropriate, and adds focused regression coverage for mixed keyless and keyed skill status.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists on current main: make skill status include missing keyless `openai-whisper` plus missing env-gated `openai-whisper-api` or `sag`, then run `setupSkills`; the install picker is shown first and the later credential loop still scans the full missing set.

## Solution Assessment

Unclear until a linked PR is reviewed. Current main is not the best solution because it conflates install selection with later credential prompting; the narrow maintainable fix should combine selected-install gating with clear credential-phase copy and regression tests.
