## Summary

Current main still has the reported onboarding behavior, and two open PRs already reference this issue with closing syntax, so the issue should stay open until a fix is reviewed and merged.

## What This Changes

The issue asks onboarding to avoid prompting for OpenAI/ElevenLabs API keys when the selected keyless skill was advertised as not needing a key.

## Best Possible Solution

Land #74891 or an equivalent narrow behavior fix that tracks selected dependency installs and skips credential prompts for unselected bin-backed skills, with regression coverage in `src/commands/onboard-skills.test.ts`.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is clear: mixed missing `openai-whisper`, `openai-whisper-api`, and `sag` entries make `setupSkills` show keyless dependency options, then the post-install loop prompts for every missing keyed skill because it iterates over all `missing` entries.

## Solution Assessment

No, current main is not the best solution because it still conflates the install selection phase with the credential phase. The narrow maintainable fix is to gate credential prompts to skills the user selected or env-only skills that are actually in scope.
