## Summary

Keep open. Current main still has the reported onboarding UX problem: the install picker can show the keyless local `openai-whisper` skill, then the same `setupSkills` pass prompts for missing `primaryEnv` credentials on separate skills such as `openai-whisper-api` and `sag` without gating that prompt to the user's selected install choices. The issue also has open unmerged closing PRs #74407 and #74891, so it should remain open until one lands or is replaced.

## What This Changes

The issue asks for `openclaw onboard` to stop asking for OpenAI Whisper or ElevenLabs API keys in a way that contradicts the keyless local `openai-whisper` install hint.

## Best Possible Solution

Keep this issue open until a linked PR lands. The desired end state is a narrow `setupSkills` fix with regression coverage: local `openai-whisper` remains keyless, credential prompts are clearly scoped to the actual env-gated skills, and skipped binary-backed skills do not receive surprise API-key prompts unless that is an explicit product choice.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: make skill status contain missing keyless `openai-whisper` plus missing env-gated `openai-whisper-api` or `sag`; current `setupSkills` can show the install list first and then prompt for every missing `primaryEnv` skill.

## Solution Assessment

Unclear until one linked PR is reviewed. Current main is not the best solution because it prompts from the full missing set; the narrow maintainable fix should gate credential prompts by the selected install path where appropriate and add clarifying credential-phase copy plus focused tests.
