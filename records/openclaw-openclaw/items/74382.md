## Summary

Keep open. Current main still has the reported onboarding mismatch: `setupSkills` can list keyless local `openai-whisper` in the dependency install picker, then the later credential loop prompts for every missing env-gated skill without tying those prompts to the user’s selected installs. The issue also has open closing PRs #74407 and #74891, so it should remain open until one lands or is replaced.

## What This Changes

The issue reports that `openclaw onboard` presents local `openai-whisper` as not requiring an API key but later asks for OpenAI or ElevenLabs API keys during the same skills setup flow.

## Best Possible Solution

Keep this issue open until a linked fix lands. The desired end state is a narrow `setupSkills` change with regression coverage: local `openai-whisper` remains visibly keyless, credential prompts are clearly scoped to env-gated skills, and skipped binary-backed skills do not get surprise API-key prompts unless maintainers intentionally choose that product behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists by making skill status contain missing keyless `openai-whisper` plus missing env-gated `openai-whisper-api` or `sag`; current `setupSkills` can show the install list first and then prompt for every missing `primaryEnv` skill.

## Solution Assessment

Unclear until a linked PR is reviewed. The narrowest maintainable fix appears to be selected-skill gating plus clear credential-phase copy in `src/commands/onboard-skills.ts`, with focused tests in `src/commands/onboard-skills.test.ts`; copy-only clarification may not cover the skipped-install case targeted by #74891.
