## What This Changes

The issue reports that `openclaw onboard` advertises keyless local OpenAI Whisper setup but later asks for an API key for an API-backed skill path.

## Best Possible Solution

Land or replace the linked selection-gated `setupSkills` fix so onboarding only prompts for credentials that match the user's chosen skill setup, while preserving legitimate env-only skill prompts.

## Reproduction Assessment

Yes, at source level: current `setupSkills` prompts every missing skill with `primaryEnv` after the dependency picker, while the bundled local Whisper and API-backed Whisper skills create the keyless/keyed mix from the report. I did not run interactive onboarding because this review is read-only.

## Solution Assessment

Yes: the narrow maintainable fix is to gate the shared `setupSkills` credential prompt by the user's setup selection, rather than special-casing Whisper metadata or relying only on explanatory copy.
