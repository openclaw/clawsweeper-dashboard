## Summary

Keep open. Current main still has the reported onboarding UX problem: `setupSkills` shows the install multiselect first, then prompts for every missing env-gated skill from the full `missing` list. Open PRs #74407 and #74891 both explicitly target this issue, so this should stay open until one lands or is replaced.

## What This Changes

The issue asks for `openclaw onboard` to stop asking for OpenAI Whisper or ElevenLabs API keys after presenting a keyless local `openai-whisper` dependency option.

## Best Possible Solution

Keep this issue open until a linked PR lands. The desired end state is a narrow `setupSkills` fix with regression coverage: local `openai-whisper` remains keyless, `OPENAI_API_KEY` and `ELEVENLABS_API_KEY` prompts are clearly scoped to the actual env-gated skills, and skipped binary-backed skills do not receive surprise credential prompts unless maintainers intentionally choose that UX.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to make skill status include missing keyless `openai-whisper` plus missing env-gated `openai-whisper-api` or `sag`; current `setupSkills` can show the install list first and then prompt for every missing `primaryEnv` skill.

## Solution Assessment

Unclear until PR review. The narrowest maintainable fix appears to be selected-skill gating plus clear credential-phase copy in `src/commands/onboard-skills.ts`, with focused tests in `src/commands/onboard-skills.test.ts`; copy-only clarification may not cover the skipped-install variant described by #74891.
