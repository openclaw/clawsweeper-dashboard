## Summary

Keep open. Current main still has the reported onboarding behavior, and open closing PRs #74407 and #74891 already target this issue, so it should remain open until maintainers land or replace one canonical fix.

## What This Changes

The issue reports that `openclaw onboard` lists keyless `openai-whisper` dependency setup but later prompts for OpenAI or ElevenLabs API credentials.

## Best Possible Solution

Onboarding should distinguish keyless dependency installation from env-gated credential setup, with regression coverage for mixed local Whisper, OpenAI Whisper API, and ElevenLabs skill states.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: with missing local `openai-whisper` plus missing env-gated `openai-whisper-api` or `sag`, `setupSkills` can show the keyless install picker and then prompt from the full missing env-gated skill set.

## Solution Assessment

No for current main. The narrow maintainable fix belongs in `setupSkills`, either by gating credential prompts to the configured/selected skill path or by explicitly separating the credential phase with tests for the mixed keyless/keyed case.
