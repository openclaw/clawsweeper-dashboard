## Summary

Keep open: current main still sends WhatsApp outbound text/caption payloads without Baileys mention metadata, and #73961 is an open, unmerged implementation PR that uses closing syntax for this issue. The right cleanup is to leave this issue open while maintainers review that PR or an equivalent replacement.

## What This Changes

The issue requests native WhatsApp outbound mention support so messages containing @phone-number text populate Baileys mention metadata and trigger WhatsApp group mention notifications.

## Best Possible Solution

Keep this issue open until #73961 or an equivalent scoped WhatsApp plugin change lands. The end state should add typed outbound mention resolution, participant/LID-aware JID mapping, Baileys mention metadata for supported text/caption sends, focused unit coverage, and user-facing docs/changelog updates without moving WhatsApp-specific policy into core.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to send outbound WhatsApp text containing an @number through createWebSendApi or sendMessageWhatsApp on current main; the code constructs a plain { text } payload and the existing test asserts that exact behavior, so native mention notifications cannot be triggered from metadata.

## Solution Assessment

Yes, reviewing and landing the open narrow WhatsApp plugin PR is the best path. Closing this issue before that PR merges would be premature, and queueing another repair PR would duplicate the active implementation vehicle.
