## Summary

Keep open: the feature is useful and current main still lacks Copilot `/models` catalog discovery, but the PR head remains unsafe because the authenticated discovery fetch still uses an `allowedHostnames` SSRF policy that current main treats as bypassing private-address checks. The changelog also still splits Unreleased into two `### Fixes` sections.

## What This Changes

The PR adds GitHub Copilot `/models` discovery during provider catalog resolution, forwards Copilot IDE and configured headers, merges Copilot implicit headers, guards the discovery fetch, filters Gemini discoveries, and adds tests plus changelog entries.

## Best Possible Solution

Keep the provider-owned additive discovery design, but use hostname scoping that still enforces resolved-address private checks unless `request.allowPrivateNetwork` is true, then fold the Copilot fix bullet into the existing Unreleased fixes section.

## Reproduction Assessment

Yes. ## Solution Assessment

No. Auto-discovery belongs in the Copilot plugin, but the mergeable form should use hostname scoping that preserves private-network blocking by default and should keep one canonical Unreleased `### Fixes` section.
