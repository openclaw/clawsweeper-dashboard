## Summary

Keep this PR open: the Copilot model discovery feature is still meaningful because current main only returns the static/empty catalog from the runtime catalog path, but the latest PR head still has a P1 security blocker in the authenticated discovery fetch and a changelog structure issue.

## What This Changes

The PR adds GitHub Copilot `/models` discovery, merges discovered models and Copilot IDE headers into provider catalog resolution, custom-merges configured headers, adds focused tests, and updates the changelog.

## Best Possible Solution

Keep the provider-owned additive discovery design, but make discovery preserve the same private-network opt-in contract as provider runtime fetches and keep the changelog under one canonical Unreleased section set.

## Reproduction Assessment

Yes. From source inspection of the PR head, set `models.providers.github-copilot.baseUrl` to a loopback, LAN, or split-DNS host without `request.allowPrivateNetwork`; discovery sends the Copilot bearer token with an `allowedHostnames` policy that current main treats as skipping private-address checks.

## Solution Assessment

No. Auto-discovery is a good provider-local direction, but the mergeable form should use hostname scoping that still checks resolved private addresses unless `request.allowPrivateNetwork` is true, then fold the changelog bullets into the existing sections.
