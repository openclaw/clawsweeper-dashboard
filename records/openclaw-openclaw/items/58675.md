## Summary

Keep open. The PR is useful and current main does not implement Copilot `/models` discovery, but the latest head still has merge blockers around authenticated discovery SSRF policy, auto-advertising Gemini models covered by the open transport bug, a stale raw-fetch guard exception, and the required changelog entry.

## What This Changes

The PR adds GitHub Copilot provider model discovery from `/models`, merges discovered models into the catalog, preserves IDE headers, and adds focused discovery and merge tests.

## Best Possible Solution

Keep the provider-owned additive discovery design, but merge the configured provider request policy into discovery fetches, block or correctly route Gemini discoveries until #74159 is fixed, remove the stale guard exception, and add the changelog entry.

## Reproduction Assessment

Yes. On the PR head, configure `models.providers.github-copilot.baseUrl` to loopback or LAN without `request.allowPrivateNetwork` and discovery will call that URL with the Copilot token under an `allowedHostnames` policy; a mocked `/models` response containing a Gemini preview ID reproduces the unsupported registration path.

## Solution Assessment

No. The feature belongs in the Copilot plugin, but this implementation is not the best mergeable form until it preserves the private-network opt-in contract and avoids advertising models that current Copilot transports cannot run.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require private-network opt-in for discovery fetches:** `extensions/github-copilot/discovery.ts:91`
  - body: `baseUrl` can come from `models.providers.github-copilot.baseUrl`, but the discovery fetch sends the Copilot bearer token while using an `allowedHostnames` policy. Current SSRF code treats that as skipping private-network checks, so loopback/LAN/private configured discovery targets can receive the token without `request.allowPrivateNetwork`; thread the provider request opt-in into the policy instead.
  - confidence: 0.92
- **[P2] Avoid auto-registering unsupported Gemini discoveries:** `extensions/github-copilot/discovery.ts:64`
  - body: Discovered models use `inferApiType`, which maps non-Claude IDs such as Gemini to `openai-responses`. Related #74159 shows Copilot Gemini IDs returned by `/models` fail on `/responses`, so discovery can advertise Gemini entries that fail or fall back at runtime until the Gemini transport/header path is fixed or those IDs are filtered.
  - confidence: 0.86
- **[P3] Remove the stale raw-fetch exception:** `scripts/check-no-raw-channel-fetch.mjs:30`
  - body: The discovery helper now uses `fetchWithSsrFGuard`, but this PR still adds `github-copilot/discovery.ts:78` to the raw-fetch allowlist. Keeping the exception can hide a future raw `fetch` in this provider file, so drop the allowlist entry.
  - confidence: 0.84

  - body: This is a user-facing `feat` that changes the Copilot model catalog by returning discovered models, but the PR does not update `CHANGELOG.md`. OpenClaw policy requires a changelog entry before merge.
