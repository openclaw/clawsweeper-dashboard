## Summary

Keep open: current main still lacks Copilot `/models` discovery, and the PR remains useful, but the latest head still has a security blocker because the authenticated discovery fetch uses an `allowedHostnames` policy that bypasses private-network checks.

## What This Changes

The PR adds GitHub Copilot `/models` catalog discovery with Copilot IDE/configured headers, discovered-model filtering, implicit header merging, focused tests, and changelog entries.

## Best Possible Solution

Keep the provider-local additive discovery design, but use hostname scoping that still checks resolved private addresses unless `request.allowPrivateNetwork` is true, then collapse the duplicate changelog heading before merge.

## Reproduction Assessment

Yes. Source inspection of the PR head shows the discovery request sends the Copilot bearer token under an `allowedHostnames` policy, and current main shows that policy skips private-network checks for matching hostnames.

## Solution Assessment

No. Auto-discovery is a reasonable provider-owned direction, and the Gemini filter helps with #74159, but the mergeable solution should preserve the private-network opt-in contract by using hostname allowlisting without bypassing resolved-address checks.
