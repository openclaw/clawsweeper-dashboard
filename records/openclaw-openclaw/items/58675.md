## Summary

Keep open. Current main still has no Copilot `/models` discovery, but this PR is not mergeable because the discovery fetch can still send a Copilot bearer token to configured private-network targets without the normal opt-in, and the changelog has duplicate active headings.

## What This Changes

This PR adds GitHub Copilot `/models` discovery during provider catalog resolution, merges discovered models and Copilot/configured headers into implicit provider config, adds focused tests, and updates the changelog.

## Best Possible Solution

Keep the provider-local additive discovery design, but use host scoping that still validates resolved private addresses unless `request.allowPrivateNetwork` is true, then fold the changelog bullets into the existing Unreleased sections.

## Reproduction Assessment

Yes. Source inspection is enough: on the PR head, configure `models.providers.github-copilot.baseUrl` to a loopback, LAN, or split-DNS host without `request.allowPrivateNetwork`; discovery still uses `allowedHostnames` while sending the Copilot bearer token, and current main skips private-network checks for allowed hostnames.

## Solution Assessment

No. Provider-owned auto-discovery is the right boundary, but this patch should not merge until hostname scoping preserves the private-network opt-in contract and the changelog structure is repaired.
