## Summary

Keep open: current `main` still has the disallowed `payload.model` path returning an early error before the isolated run starts, and there is no rejection-point hook warning with the rejected model and allowlist hint. The existing hook path does queue a generic non-ok system event, so the fix should improve diagnostics/visibility rather than restore silent fallback by default.

## What This Changes

The issue asks OpenClaw to make cron and `/hooks/agent` isolated-run model allowlist rejections visible instead of appearing as silently dropped runs.

## Best Possible Solution

Keep the documented fail-closed model validation, but add explicit warning/log diagnostics and regression coverage so cron and hook callers can discover the rejected model and allowed-model guidance immediately.

## Reproduction Assessment

Yes: source inspection gives a high-confidence reproduction path using an isolated cron or `/hooks/agent` payload with a `model` outside `agents.defaults.models`. The code returns before runner execution, which explains the missing LLM call and transcript.

## Solution Assessment

Yes for the minimum fix: make the existing explicit validation failure loud and visible in cron/hook diagnostics. Restoring fallback-to-default should stay out of an automated bug fix unless a maintainer makes that product choice.
