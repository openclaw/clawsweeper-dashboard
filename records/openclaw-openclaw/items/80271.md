## Summary

Keep open: the narrow QQBot storage override is plausible, but this PR is not mergeable because it carries a very large unrelated diff, removes Telegram QA infrastructure, regresses native device identity handling, has a protected `security` label, and lacks real behavior proof.

## What This Changes

The PR claims to add `QQBOT_DATA_DIR` support for QQBot data and media path helpers, while also changing thousands of unrelated CI, docs, app, protocol, and UI files.

## Best Possible Solution

Resubmit a narrow QQBot-only change with docs or config-contract coverage, focused tests for unset/empty/tilde env paths, and redacted terminal or live QQBot proof; keep unrelated CI, Telegram QA, native identity, protocol, app, and UI changes out of this PR.

## Reproduction Assessment

Yes for the narrow premise by source inspection: current main reads no `QQBOT_DATA_DIR` and builds QQBot storage paths from `getHomeDir()`. No live QQBot run or after-fix behavior proof is provided.

## Solution Assessment

No. The narrow helper direction may be acceptable after a maintainer-owned config decision, but this PR is not the best solution because it bundles unrelated regressions and deletes QA/identity behavior outside the QQBot path surface.
