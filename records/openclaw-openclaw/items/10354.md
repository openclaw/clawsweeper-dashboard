Clarify that the message tool’s optional channel argument selects a messaging provider or channel plugin id rather than a destination conversation, channel, or user id, with enum support considered only if it remains plugin-aware.

## Best Possible Solution

Land the linked description-and-schema-test repair while keeping channel optional, and consider an enum separately only if it can derive from lightweight registered channel/plugin discovery without constraining external plugins.

## Solution Assessment
