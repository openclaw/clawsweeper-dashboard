## Summary

## What This Changes

Request to add tool-schema metadata clarifying that the shared message tool `channel` parameter expects a messaging provider or channel plugin id, not a destination conversation/channel/user id.

## Best Possible Solution

Add a focused TypeBox description for `channel`, keep it optional, and only add enum metadata if it is derived from the same plugin-aware channel id source used by routing.

## Solution Assessment

Yes for the description portion: schema metadata plus a focused assertion is the narrowest maintainable repair. A static enum is not the best default unless it is generated from plugin-aware channel ids.
