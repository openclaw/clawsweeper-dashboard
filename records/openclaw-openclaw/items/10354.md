## Summary

Keep open. Current main still exposes the shared message tool `channel` parameter without description or enum metadata, the linked implementation PRs are closed unmerged, and canonical search did not find an open replacement PR that owns this exact schema gap.

## What This Changes

The issue requests clearer tool-schema guidance for the message tool `channel` parameter so agents pass a provider/channel plugin id such as `slack` rather than a destination conversation id.

## Best Possible Solution

Add a focused schema description for `channel`, keep it optional, and add enum metadata only if it can be derived from plugin-aware deliverable channel ids without loading broad channel runtimes.

## Reproduction Assessment

Yes for the schema gap: source inspection against current main shows `createMessageTool().parameters` still includes `channel` from `buildRoutingSchema()` as a bare optional string. I did not run a live Slack setup, but the reported schema omission is directly visible in source.

## Solution Assessment

Yes for the description portion. The narrow maintainable fix is TypeBox schema metadata plus a focused assertion; a static enum should be skipped unless it is generated from plugin-aware channel ids.
