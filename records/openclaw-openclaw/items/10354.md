## Summary

Keep open: current main and v2026.5.12 still expose the shared message tool `channel` parameter as a bare optional string, and the two implementation PRs for this issue are closed unmerged.

## What This Changes

Request to add TypeBox schema guidance, and possibly plugin-aware enum metadata, to the message tool `channel` parameter so agents pass provider ids instead of destination ids.

## Best Possible Solution

Add a focused TypeBox description for `channel`, keep it optional, add a schema assertion, and only add enum metadata if it is dynamically derived from registered deliverable channel ids.

## Reproduction Assessment

Yes for the schema gap: current-main source inspection shows the shared message tool exposes `channel` as a bare optional string. A live Slack or Discord workspace is not needed to verify the missing tool metadata.

## Solution Assessment

Yes for the description portion: schema metadata plus a focused assertion is the narrowest maintainable repair. A hard-coded static enum is not the best default unless it is generated from plugin-aware channel ids.
