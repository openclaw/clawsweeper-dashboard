## Summary

Keep open. Current main still exposes the shared message tool `channel` parameter as a bare optional string, so the requested schema guidance remains unimplemented; prompt guidance and runtime fallback paths do not add the missing TypeBox metadata, and the two linked implementation PRs are closed unmerged.

## What This Changes

Request to add a TypeBox description, and possibly enum metadata, to the message tool `channel` parameter so agents understand it expects a channel plugin/provider id rather than a destination id.

## Best Possible Solution

Land a narrow schema/test update that keeps `channel` optional, adds provider/plugin-id wording, and adds enum metadata only if it is derived from the plugin-aware channel list.
