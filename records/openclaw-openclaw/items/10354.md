## What This Changes

Request to add provider-name description and possible enum guidance to the message tool `channel` parameter so agents do not pass destination channel IDs there.

## Best Possible Solution

Review and land the linked schema/test PR if description-only closure is acceptable, then track registry-derived enum support separately only if maintainers still want it.

## Reproduction Assessment

Yes. Source inspection on current main shows the `channel` parameter lacks the requested schema metadata, and the channel-selection path can still reject destination IDs as unknown channels.

## Solution Assessment

Yes for the description gap: a narrow TypeBox description plus focused schema assertion is the maintainable repair. The enum portion needs maintainer direction because channel/plugin availability is dynamic.
