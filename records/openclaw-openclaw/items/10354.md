## Summary

Keep open. Current main still exposes the shared message tool `channel` parameter without schema metadata, while routing/docs treat it as a provider or channel plugin id rather than a destination id.

## What This Changes

Request to add message tool schema guidance, and optionally enum metadata, for `channel` so agents pass provider/channel plugin ids instead of Slack/Discord destination ids.

## Best Possible Solution

Add a focused TypeBox description for `channel`, keep it optional, and add a schema assertion; only add enum metadata if it is derived from the plugin-aware channel id source.

## Solution Assessment

Yes for the description portion: schema metadata plus a focused assertion is the narrowest maintainable repair. A static enum is not the best default unless generated from plugin-aware channel ids.
