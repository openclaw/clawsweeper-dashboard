## Repair Work Prompt

Fix openclaw/openclaw#10354. Current main defines the shared message tool `channel` schema as `Type.Optional(Type.String())` without description or enum metadata. Add a TypeBox description in `buildRoutingSchema()` explaining that `channel` is the messaging provider or channel plugin id, for example `slack` or `discord`, not the destination conversation/channel/user id. Keep the parameter optional and do not change runtime routing behavior. Add a focused assertion in `src/agents/tools/message-tool.test.ts` that the generated message tool schema exposes the new `channel.description`. Do not add a hard-coded static enum unless it is derived from the plugin-aware channel ids already used at runtime; deferring enum support is acceptable. Add a single-line user-facing `CHANGELOG.md` entry for the active version, crediting `@carrotRakko` if the changelog section expects contributor credit. Validate with the targeted message-tool test and targeted oxfmt check; run `pnpm check:changed` in Testbox before handoff if the fix expands beyond the message-tool schema/test/changelog surface.
