## What This Changes

Add provider-id guidance, and possibly plugin-aware enum metadata, to the shared `message` tool `channel` parameter.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/82868 with a narrow schema/test update that adds provider-id guidance and derives any enum from deliverable registered channel ids.

## Solution Assessment

Yes for the description portion: schema metadata plus focused regression coverage is the narrowest maintainable repair. Any enum support should be plugin-aware and derived from registered deliverable channel ids, not hard-coded.
