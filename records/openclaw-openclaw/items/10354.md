- `impact:message-loss`: The reported failure mode can make agents pass destination ids as provider ids, causing `Unknown channel` errors and failed tool-based sends.

## Summary

Keep open: current main and the latest release still expose the shared message tool `channel` parameter as an undescribed optional string, and two open linked pull requests already carry the active fix path.

## What This Changes

The issue requests clearer schema metadata for the message tool `channel` parameter so agents pass provider ids like `slack` or `discord` instead of destination channel/conversation ids.

## Best Possible Solution

Review and land one maintainer-preferred linked pull request for the narrow schema/test change, then close the duplicate linked pull request if it is superseded; any enum should be registry-derived rather than hard-coded.

## Reproduction Assessment

Yes for source-level reproduction: current main exposes `channel` as a bare optional string, and invalid explicit channel values still have an `Unknown channel` path. I did not run a live Slack or Discord workspace reproduction.

## Solution Assessment

Yes: adding provider-oriented schema metadata plus focused regression coverage is the narrow maintainable fix. A static enum would be weaker than a registry-derived list because deliverable channels include registered plugin ids.
