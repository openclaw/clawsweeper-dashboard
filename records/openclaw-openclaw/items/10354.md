## Summary

Keep open. Current main still exposes the shared message tool `channel` parameter as `Type.Optional(Type.String())` without description or enum metadata, and the linked implementation PRs are closed unmerged.

## What This Changes

The issue requests clearer schema metadata for the message tool `channel` parameter so agents pass a messaging provider or channel plugin id rather than a destination conversation/channel/user id.

## Best Possible Solution

The shared message tool schema should keep `channel` optional, add provider/channel-plugin-id wording, and add enum metadata only if it can be derived from the plugin-aware deliverable channel registry.

## Reproduction Assessment

Yes. Source inspection against current main shows the message tool schema still comes from `buildRoutingSchema()`, where `channel` has no description or enum metadata.

## Solution Assessment

Yes for the description portion. The narrow maintainable fix is TypeBox schema metadata plus a focused assertion; a hard-coded enum should be skipped unless it is generated from plugin-aware channel ids.
