## Summary

Keep open. Current main and the latest release still expose the message tool `channel` parameter as an undescribed optional string, and the open linked PR is a fix candidate rather than an implemented fix.

## What This Changes

Request to add model-facing schema guidance, and possibly an enum, for the message tool `channel` parameter so agents pass provider ids instead of conversation or channel ids.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84589 for the description/test fix if accepted, and split registry-derived enum support into separate maintainer-approved work only if needed.

## Reproduction Assessment

Yes, source-level. Current main exposes `channel` as an undescribed optional string while outbound routing interprets it as a provider/channel id and still throws `Unknown channel` for invalid explicit values without a valid fallback.

## Solution Assessment

Yes for the description portion: TypeBox metadata plus a focused schema assertion is the narrow maintainable fix. The enum portion is unclear and should be a maintainer decision, preferably derived from registered channel/plugin ids if pursued.
