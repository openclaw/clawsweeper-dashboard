Fix https://github.com/openclaw/openclaw/issues/10354 with a narrow message-tool schema/test change. In `src/agents/tools/message-tool.ts`, keep `channel` optional and add a TypeBox description explaining that it expects a messaging provider or channel plugin id such as `slack` or `discord`, not a destination channel, user, or conversation id. Do not add a hard-coded static enum; only add enum metadata if it is derived from registered deliverable channel ids without loading broad plugin runtimes.
