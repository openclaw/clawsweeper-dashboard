## Summary

Keep open: current main still has a source-reproducible path where Telegram outbound image replies use default image optimization, which can require sharp even though the packaged root manifest does not declare sharp as a runtime dependency.

## What This Changes

The issue reports Telegram MEDIA image replies failing when optional sharp is missing or resolves incorrectly in OpenClaw 2026.5.2.

## Best Possible Solution

Make Telegram image replies deliver without a manual sharp shim, either by ensuring packaged installs include a resolvable runtime image backend or by falling back to raw/document upload when optimization is unavailable.

## Reproduction Assessment

Yes, source-reproducible: a Telegram MEDIA final reply enters loadWebMedia with optimizeImages enabled, which reaches media-understanding-core's dynamic sharp import. In a packaged install without root sharp, that path matches the reporter's missing-sharp failure.

## Solution Assessment

No current-main fix is present yet. The narrow maintainable repair is to keep the existing media safety limits while preventing optional optimization dependency failures from dropping the Telegram reply.
