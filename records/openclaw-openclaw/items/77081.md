## Summary

Keep open: current main still has a source-visible path where Telegram image MEDIA delivery aborts when the sharp-backed optimizer is unavailable, matching the reported macOS failure and the later Linux message-tool confirmation.

## What This Changes

The issue asks for Telegram image MEDIA replies and message-tool sends to deliver valid local images even when optional sharp image optimization is missing or unresolved.

## Best Possible Solution

Make outbound image loading tolerate an unavailable optimizer by returning valid in-limit images raw or as Telegram documents, while keeping sharp for resizing/conversion and adding regression coverage for missing-sharp media sends.

## Reproduction Assessment

Yes, by source inspection and supplied logs: a Telegram image MEDIA path reaches `loadWebMedia`, the missing sharp import makes optimization throw, and Telegram then sends the empty-response fallback. I did not run a live Telegram send in this read-only review.

## Solution Assessment

No, current main is not the best solution because optional optimization can abort basic image delivery. The narrow maintainable fix is a raw/document fallback for already-valid images, plus dependency-metadata cleanup only if package acceptance proves it is needed.
