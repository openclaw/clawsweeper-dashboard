## Summary

## What This Changes

The PR changes the Amazon Bedrock provider wrapper and tests so direct `amazon.nova-*` models bypass the no-cache wrapper and receive Bedrock Converse cache points through `onPayload`.

## Best Possible Solution

Land provider-owned Nova prompt caching that normalizes direct, regional, and global Nova IDs, preserves explicit retention/no-cache semantics, avoids unsupported 1-hour TTL unless proven for Nova, updates docs/tests, and includes redacted live Bedrock proof.

## Solution Assessment

No as submitted. The Amazon Bedrock provider wrapper is the right boundary, but this branch still needs Nova ID normalization, explicit retention propagation, docs alignment, and live Bedrock proof before it is the best fix.
