## Best Possible Solution

- **[P1] Preserve Nova cacheRetention options:** `extensions/amazon-bedrock/register.sync.runtime.ts:692`
  - body: The new Nova branch always calls `injectBedrockCachePoints(payloadRecord, "short")`.   - confidence: 0.91

  - body: `isNovaModel()` only matches IDs beginning with `amazon.nova-`, but the linked reports use forms such as `global.amazon.nova-2-lite-v1:0`, and Bedrock model routing commonly includes regional/global prefixes. Those models will continue through the old no-cache path, leaving Nova prompt caching only partially fixed.
  - confidence: 0.88

  - body: This patch changes user-visible Bedrock cache behavior, but the current prompt-caching docs still say non-Anthropic Bedrock models are forced to `cacheRetention: "none"`. Update the docs with the supported Nova IDs and retention behavior so users do not follow stale guidance after merge.
