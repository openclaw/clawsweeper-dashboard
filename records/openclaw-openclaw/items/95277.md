- **[P1] Preserve Nova cacheRetention overrides:** `extensions/amazon-bedrock/register.sync.runtime.ts:712`
  - body: This branch injects Nova cache points with hard-coded `"short"`, but the normal extra-param wrapper still drops direct Nova `cacheRetention` because the runner gate only accepts Anthropic, Google, or prompt-cache-key-capable families. A user with `params.cacheRetention: "none"` or `"long"` for a Nova Bedrock model would no longer get the configured behavior, so teach the runner/provider path Nova eligibility and cover `none`, `short`, and `long`.
  - confidence: 0.88

  - body: `isNovaModel()` only matches IDs that start with `amazon.nova-`, but Bedrock discovery preserves system-defined inference-profile IDs such as `us.amazon...` and `global.amazon...`, and the linked report used `global.amazon.nova-2-lite-v1:0`. Those models will still miss the Nova branch and fall back to the old no-cache wrapper, so normalize the Bedrock region/profile prefix and add tests for those IDs.
  - confidence: 0.86
