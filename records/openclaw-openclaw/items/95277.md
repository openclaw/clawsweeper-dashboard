## Summary

Keep open: current main and v2026.6.8 still force direct Nova Bedrock models through the no-cache path, but this PR is not merge-ready because it hard-codes Nova cache injection to short retention, only matches direct `amazon.nova-*` IDs, leaves public docs stale, and lacks live Bedrock proof.

## What This Changes

The PR updates the bundled Amazon Bedrock plugin to treat direct `amazon.nova-*` models as prompt-cache eligible, inject Bedrock Converse cache points for Nova, and update existing Bedrock tests that expected Nova to be forced no-cache.

## Best Possible Solution

Land provider-owned Nova prompt caching that normalizes supported Nova IDs, preserves explicit retention semantics, keeps unsupported Bedrock families no-cache, updates docs/tests, and includes redacted live AWS proof.
