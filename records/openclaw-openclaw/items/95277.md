## What This Changes

The PR changes the Amazon Bedrock provider plugin and tests to skip the no-cache wrapper for direct Amazon Nova model IDs and inject Bedrock Converse cache points for Nova payloads.

## Best Possible Solution

Land a provider-owned Nova cache implementation that normalizes Bedrock Nova IDs, preserves explicit `cacheRetention` values through the runner, keeps unsupported Bedrock families no-cache, updates docs/tests, and includes redacted live Bedrock proof.

## Reproduction Assessment

Yes. Source inspection shows the normal runner still returns `undefined` for direct Nova `cacheRetention`, while the PR branch injects Nova cache points with hard-coded `"short"` and only matches IDs beginning with `amazon.nova-`.

## Solution Assessment

No. The provider wrapper is the right ownership boundary for Nova payload shaping, but the implementation also needs the upstream cache-retention gate, regional/system profile ID matching, docs, and live AWS proof to be the best fix.

- **[P1] Preserve Nova cacheRetention overrides:** `extensions/amazon-bedrock/register.sync.runtime.ts:712`
  - body: This branch injects Nova cache points with a hard-coded `"short"`, but the normal runner still drops direct Nova `cacheRetention` because `resolveCacheRetention` only allows Anthropic, Google, or prompt-cache-key families. A user with `params.cacheRetention: "none"` for a Nova Bedrock model would no longer have that explicit no-cache setting honored once this path runs, so add Nova to the retention eligibility gate and test `none`/`short`/`long` through the runner.
  - confidence: 0.88
- **[P2] Recognize regional Nova profile IDs:** `extensions/amazon-bedrock/register.sync.runtime.ts:95`
  - body: `isNovaModel()` only matches IDs that start with `amazon.nova-`, but Bedrock discovery keeps system-defined inference-profile IDs such as `us.amazon...` and `global.amazon...` as model IDs, and the linked reports include `global.amazon.nova-2-lite-v1:0`. Those models will still miss the Nova branch and fall back to the old no-cache wrapper, so strip the Bedrock region/profile prefix before matching and cover at least `global.`/`us.` Nova IDs in tests.
  - confidence: 0.86
