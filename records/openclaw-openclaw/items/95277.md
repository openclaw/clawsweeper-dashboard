## Summary

Keep open: this PR is the active candidate for Nova prompt caching, but it is not merge-ready because it hard-codes cache injection to short retention, misses AWS-documented regional/global Nova IDs, leaves public docs stale, and lacks live Bedrock Nova proof.

## What This Changes

The PR changes the Amazon Bedrock provider and tests so direct `amazon.nova-*` models bypass the no-cache wrapper and receive Bedrock `cachePoint` injection via `onPayload`.

## Best Possible Solution

Centralize supported Nova eligibility, cover direct/regional/global IDs, preserve explicit retention including opt-out, update docs/tests, and attach redacted live Nova proof before merge.

## Solution Assessment

No. The provider-owned direction is plausible, but the best fix also updates the shared retention family gate, handles all supported Nova ID forms, preserves opt-out semantics, and aligns docs with the new behavior.
