## Summary

Keep this PR open as the active candidate for Nova prompt caching, but it is not merge-ready: the patch still bypasses explicit Nova cache-retention choices, misses regional/global Nova IDs, leaves public docs contradicting the new behavior, and has no live Bedrock Nova proof.

## What This Changes

The PR changes the Amazon Bedrock provider and tests so direct `amazon.nova-*` models bypass the no-cache wrapper and receive Bedrock `cachePoint` injection through `onPayload`.

## Best Possible Solution

Centralize supported Nova eligibility, cover direct/regional/global IDs, preserve explicit retention including opt-out, avoid unsupported TTL assumptions, update docs/tests, and attach redacted live Nova proof before merge.
