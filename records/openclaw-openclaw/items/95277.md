## Summary

Keep open: this is a useful Bedrock Nova prompt-caching feature PR, but the current patch is incomplete because it hard-codes Nova retention, misses regional/global Nova IDs, leaves public docs stale, and still lacks live Bedrock proof.

## What This Changes

The branch changes the Amazon Bedrock plugin runtime and tests to treat direct `amazon.nova-*` IDs as cache-eligible and inject Bedrock Converse cache points instead of forcing `cacheRetention: "none"`.

## Best Possible Solution

Land provider-owned Nova prompt caching that normalizes supported Nova IDs, preserves explicit retention semantics, keeps unsupported Bedrock families no-cache, updates docs/tests, and includes redacted live AWS proof.

## Reproduction Assessment

No. The provider wrapper is a plausible boundary, but the best fix also needs Nova ID normalization, retention propagation, docs alignment, and live Bedrock proof.
