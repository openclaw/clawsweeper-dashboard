- **[P1] Preserve Nova cacheRetention overrides:** `extensions/amazon-bedrock/register.sync.runtime.ts:692`
  - body: The new Nova branch always calls `injectBedrockCachePoints(payloadRecord, "short")`. Those models will continue through the old no-cache path, leaving Nova prompt caching only partially fixed.
  - confidence: 0.88

  - body: The patch changes user-visible provider behavior, but the current prompt-caching docs still say non-Anthropic Bedrock models are forced to `cacheRetention: "none"`. Update the docs with the supported Nova IDs and retention modes.
