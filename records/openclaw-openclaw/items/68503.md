## What This Changes

The PR filters and groups routine `status --all` gateway log-tail diagnostics and changes gateway diagnostics to return `null` instead of an arbitrary last-log fallback when no curated error matches.

## Best Possible Solution

Land a rebased status-diagnosis cleanup that filters routine noise while preserving actionable OAuth/gateway failures and current restart-log behavior, with focused tests and redacted real `status --all` proof.

## Reproduction Assessment

Yes for the review blockers: source inspection shows timestamped embedded OAuth failures are skipped before parsing, grouped lane waits keep the first value while saying `last`, and the diagnostics test targets the old helper. No live command was run during this read-only review.

## Solution Assessment

No. The direction is reasonable, but this branch needs a rebase, targeted correctness fixes, and real after-fix `status --all` proof before it is the narrow maintainable solution.

Full review comments:

- **[P1] Mock the restart-log helper used by diagnostics:** `src/daemon/diagnostics.test.ts:17-19`
  - body: Current main's `diagnostics.ts` imports `resolveGatewayLogPaths` from `./restart-logs.js`, but this added test mocks `./launchd.js`. After rebase, the mocked paths will not drive the implementation under test, so the stderr/stdout cases can read default paths instead of fixture paths.
  - confidence: 0.93
- **[P2] Preserve timestamped embedded auth failures:** `src/commands/status-all/gateway.ts:75-76`
  - body: This skip runs before the embedded OAuth parser, so a timestamped line like `2026-... Embedded agent failed before reply: OAuth token refresh failed...` is discarded because its stripped content does not start with `[`. That hides an actionable auth-refresh outage that current main still surfaces in the log tail.
  - confidence: 0.9
- **[P2] Make the lane-wait label match the grouped value:** `src/commands/status-all/gateway.ts:181-184`
  - body: `addGroup` increments the count but keeps the first `base` string, while this summary labels the duration as `last`. If waits progress from 5s to 45s, the output still says `last 5s ×2`; update the grouped base for newer occurrences or label it as the first observed wait.
  - confidence: 0.88
