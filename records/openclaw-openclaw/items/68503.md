## Summary

Keep open: the PR is useful but not ready to merge because current main has not implemented the central fallback-removal/noise-reduction behavior, and the branch still has correctness issues plus missing real behavior proof.

## What This Changes

The branch expands `status --all` gateway log-tail filtering/grouping and changes gateway diagnostics to return `null` instead of an arbitrary last-log fallback when no curated error matches.

## Best Possible Solution

Land a rebased status-diagnosis cleanup that filters routine noise while preserving actionable OAuth/gateway failures and uses the current restart-log contracts.

## Reproduction Assessment

Yes. Source inspection gives high-confidence reproduction paths for the PR findings: timestamped embedded OAuth failures are skipped before parsing, grouped lane waits keep the first base while saying `last`, and the diagnostics test is tied to the old launchd helper instead of current main's restart-log helper.

## Solution Assessment

No. The direction is reasonable, but this branch needs a rebase, targeted correctness fixes, and real after-fix `status --all` proof before it is the narrow maintainable solution.

  - confidence: 0.93

That hides an actionable auth-refresh outage that current main still surfaces in the log tail.
  - confidence: 0.9

  - body: `addGroup` increments the count but keeps the first `base` string, while this summary labels the duration as `last`. If waits progress from 5s to 45s, the output still says `last 5s ×2`; update the grouped base for newer occurrences or label it as the first observed wait.
  - confidence: 0.88
