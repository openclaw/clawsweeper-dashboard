## Summary

Keep open: current main still rejects Telegram direct-message ACP configured bindings before they can register or match, and https://github.com/openclaw/openclaw/pull/68537 remains the focused open draft fix path.

## What This Changes

The issue requests support for top-level Telegram ACP configured bindings that target direct messages, not only group forum topics.

## Best Possible Solution

Land or replace the draft PR with a narrow Telegram binding-adapter change that accepts positive direct-chat peer ids, keeps bare negative group ids rejected, updates current docs, and adds regression coverage.

## Reproduction Assessment

Yes, source-reproducible. Configure a top-level Telegram ACP binding with `match.peer.kind: "direct"` and a positive numeric `match.peer.id`; current main sends that id to the Telegram binding adapter, which returns `null` before the rule can be registered or matched.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Telegram channel adapter's configured-binding normalization and inbound matching, with docs and tests; the generic ACP registry already delegates channel-specific conversation ids to that adapter.
