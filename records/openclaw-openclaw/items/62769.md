## Summary

Keep open: current main still drops Telegram direct-message ACP configured bindings before route registration, and the existing draft implementation remains the active fix path.

## What This Changes

Support top-level Telegram ACP configured bindings for direct-message peer IDs in addition to group forum topics.

## Best Possible Solution

Land or replace the draft PR with a narrow Telegram binding adapter change that accepts positive direct-chat ids, keeps whole-group bare negative ids rejected, and updates focused docs/tests.

## Reproduction Assessment

Yes. Configure a top-level Telegram ACP binding with `match.peer.kind: "direct"` and a positive `match.peer.id`; current main passes that id to the Telegram binding adapter, which returns `null` before the rule can be registered.

## Solution Assessment

Yes. The narrow maintainable fix belongs in Telegram-specific configured-binding normalization and inbound matching, with regression coverage and docs for direct-chat ACP bindings.
