## Summary

Keep open: current main still rejects positive Telegram direct-chat IDs in ACP configured binding normalization/matching, docs remain topic-scoped, and the related fix PR is still open as a draft rather than merged.

## What This Changes

Support top-level Telegram ACP configured bindings whose match peer is a direct DM, so DM messages route into persistent ACP sessions instead of the native runtime.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/68537 with a narrow Telegram bindings-adapter change that accepts plain positive DM IDs, preserves topic/group rejection rules, and updates focused docs/tests.

## Reproduction Assessment

Yes, by source inspection. Configure a top-level Telegram ACP binding with match.peer.kind 'direct' and a positive match.peer.id; current main passes that ID into configured-binding compilation, where the Telegram adapter returns null before registration.

## Solution Assessment

Yes for the requested direction, but it is not implemented on main. The maintainable fix belongs in Telegram configured-binding normalization and inbound matching, with coverage for plain positive DM IDs and rejection of invalid topic-shaped IDs.
