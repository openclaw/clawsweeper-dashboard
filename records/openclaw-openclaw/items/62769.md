## Summary

Keep open: current main still drops Telegram direct-message ACP configured bindings before route resolution, and the open draft implementation at https://github.com/openclaw/openclaw/pull/68537 remains the active fix path.

## What This Changes

The request is to allow Telegram top-level ACP bindings to match direct-message peer ids instead of only forum-topic ids.

## Best Possible Solution

Land or replace the draft PR with a narrow Telegram adapter change that accepts positive direct-chat peer ids, keeps bare negative group ids rejected, updates docs, and adds regression coverage.

## Reproduction Assessment

Yes. Configure a top-level Telegram ACP binding with `match.peer.kind: "direct"` and a positive numeric `match.peer.id`; current main sends that id to the Telegram binding adapter, which returns `null` before the rule can be registered or matched.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Telegram channel adapter's configured-binding normalization and inbound matching, not in the generic ACP registry or route compiler.
