## Summary

Current main still rejects Telegram direct-message ACP configured bindings in the Telegram binding adapter, while the docs still describe persistent Telegram ACP bindings as forum-topic scoped. The related draft PR #68537 remains the active implementation candidate, so this issue should stay open until that PR or a replacement lands.

## What This Changes

Support top-level Telegram ACP configured bindings for direct-message peer ids so Telegram DMs can create or resume persistent ACP sessions.

## Best Possible Solution

Land a rebased #68537 or a narrow replacement that accepts positive Telegram direct-chat peer ids in the Telegram configured-binding adapter, preserves forum-topic behavior, updates Telegram/ACP docs, adds focused regression coverage, and includes a changelog entry.

## Reproduction Assessment

Yes. Configure a top-level Telegram ACP binding with match.peer.kind "direct" and a positive numeric match.peer.id; current main routes that id through the Telegram binding adapter, which returns null before the binding can match.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Telegram plugin's configured-binding normalization and matching path, with tests around the adapter and inbound DM route, rather than in the generic ACP registry.
