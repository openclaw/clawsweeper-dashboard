## Summary

Keep open: current main and v2026.6.1 still reject plain positive Telegram direct-chat IDs for configured ACP bindings, and the open draft fix path at https://github.com/openclaw/openclaw/pull/68537 has not merged.

## What This Changes

The issue requests support for configured Telegram ACP bindings in direct-message conversations, not only group/forum topics.

## Best Possible Solution

Finish or replace the open draft with a narrow Telegram-plugin change that compiles and matches `peer.kind: "direct"` ACP bindings, preserves topic behavior, and adds direct-chat regression coverage plus docs.

## Reproduction Assessment

Yes, by source inspection: a Telegram DM configured binding with a positive peer id reaches the configured-binding resolver as a direct conversation, then the Telegram ACP normalizer/matcher rejects it. I did not run tests because this review is read-only.

## Solution Assessment

Yes, the maintainable direction is to extend the existing Telegram configured-binding provider for direct peers rather than add a new config surface. The open draft PR is the right place to review that implementation once it has proof.
