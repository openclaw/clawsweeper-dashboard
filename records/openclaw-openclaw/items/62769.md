## Summary

Keep open: current main still rejects Telegram direct-message ACP configured bindings, and the open draft PR that targets this behavior is not merged.

## What This Changes

Support top-level Telegram ACP configured bindings for direct-message peers in addition to group forum topics.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/68537 with a narrow Telegram adapter change that accepts plain positive DM ids, preserves topic/group guards, and updates focused docs/tests.

## Reproduction Assessment

Yes. Configure a top-level Telegram ACP binding with `match.peer.kind: "direct"` and a positive `match.peer.id`; current main passes that id to Telegram configured-binding compilation, where the adapter returns null before the rule can register.

## Solution Assessment

Yes for the requested direction, but not done on main. The maintainable fix belongs in Telegram configured-binding normalization and inbound matching, with coverage for positive plain DM ids and rejection of invalid topic-shaped ids.
