## Summary

Current main still rejects Telegram direct-message ACP configured bindings in the Telegram adapter, and the already-linked draft PR #68537 remains the active implementation candidate for the same behavior.

## What This Changes

Support top-level Telegram `type: "acp"` configured bindings for direct-message peers as well as group/forum-topic conversations.

## Best Possible Solution

Land a rebased #68537 or narrow replacement that accepts positive Telegram direct-chat peer ids in the Telegram configured-binding adapter, preserves forum-topic behavior, updates docs, and adds focused regression coverage.

## Reproduction Assessment

Yes. Configure a top-level Telegram ACP binding with `match.peer.kind: "direct"` and a positive numeric `match.peer.id`; current main routes that id through the Telegram adapter, which returns null before the binding can match.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Telegram plugin's configured-binding normalization and matching path rather than the generic ACP registry or core routing layer.
