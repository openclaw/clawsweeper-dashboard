- `impact:message-loss`: The reported messages are routed to the wrong runtime path instead of the configured ACP binding.
- `impact:session-state`: The requested behavior is about creating or resuming the intended persistent ACP session for Telegram DM conversations.

## Summary

Keep open: current main and the latest release still only compile Telegram ACP configured bindings from topic-shaped group IDs, while the related direct-chat fix remains an open draft PR with prior review blockers.

## What This Changes

Support top-level Telegram ACP bindings whose `match.peer` is a direct chat ID so Telegram DMs can route to persistent ACP sessions.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/68537 with a narrow Telegram bindings-adapter change that accepts plain positive DM IDs, preserves topic/group rejection rules, and updates focused docs/tests.

## Reproduction Assessment

Yes, by source inspection. Configure a top-level Telegram ACP binding with `match.peer.kind: "direct"` and a positive `match.peer.id`; current main passes that ID into Telegram configured-binding compilation, where the adapter returns null before registration.

## Solution Assessment

Yes for the requested direction, but it is not implemented on main. The maintainable fix belongs in Telegram configured-binding normalization and inbound matching, with coverage for plain positive DM IDs and rejection of invalid topic-shaped IDs.
