## Summary

Keep this PR open because stale direct-bridge gateway fallback remains useful on current main, but the submitted patch is not merge-ready: it fail-opens unavailable Codex PreToolUse, conflicts with the current no-policy fallback contract, and has only test-command proof.

## Best Possible Solution

Keep the stale direct-bridge gateway fallback idea, but preserve unknown unavailable PreToolUse and PermissionRequest fail-closed semantics, keep the explicit no-policy noop marker, and add real OpenClaw/Codex proof.

Yes for the patch defect at source level: the PR changes unavailable PreToolUse to empty successful output, and Codex treats empty successful PreToolUse output as non-blocking. I did not run a live OpenClaw/Codex reproduction in this read-only review.

## Solution Assessment

No. Falling back from a stale direct bridge to the gateway is a plausible narrow fix, but fail-opening unknown PreToolUse relay outages is not the safest maintainable solution.
