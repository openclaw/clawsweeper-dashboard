## Summary

Keep open: the stale direct-bridge fallback direction is useful, but this branch also changes unavailable PreToolUse from deny-by-default to fail-open, which can bypass OpenClaw tool policy when the relay or gateway is unavailable.

## What This Changes

The PR changes native hook relay CLI fallback behavior so stale or unusable direct bridge records fall through to the gateway path, and changes unavailable PreToolUse fallback to return a no-op response.

## Best Possible Solution

Keep the stale direct-bridge-to-gateway fallback, but preserve unknown/unavailable PreToolUse as deny-by-default unless a generated command explicitly proves no local PreToolUse policy existed and production config tests cover that path.

## Reproduction Assessment

Yes, source-reproducible: current main fails closed for unavailable PreToolUse, while the PR diff makes that path return no-op output that Codex treats as no blocking decision.

## Solution Assessment

No. Falling back from stale direct bridge records to the gateway is a good repair direction, but making unknown PreToolUse relay failure fail-open is not the safest maintainable default.
