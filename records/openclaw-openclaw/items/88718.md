## Summary

Keep open, but this PR is not merge-ready: the stale-bridge gateway fallback is a plausible bugfix direction, while the PreToolUse fail-open change would bypass a permission-sensitive hook failure path and the external PR still lacks real behavior proof.

## What This Changes

The PR makes stale or unusable direct native hook bridge errors fall through to gateway `nativeHook.invoke`, changes unavailable PreToolUse fallback to no-op output, and updates relay CLI tests.

## Best Possible Solution

Preserve the useful direct-bridge resilience only with fail-closed unknown PreToolUse behavior, explicit no-policy markers where intended, focused regression coverage, and redacted real setup proof; route broader Codex native hook delivery policy through the active canonical PRs.

## Reproduction Assessment

Yes, source-reproducible: the PR diff returns no-op output for unavailable PreToolUse, and Codex source treats empty exit-0 stdout as no blocking decision. I did not run a live Codex setup in this read-only review.

## Solution Assessment

No. Falling back from stale direct bridge records may be a useful repair, but making unknown PreToolUse relay failure fail open is not the safest or most maintainable fix.
