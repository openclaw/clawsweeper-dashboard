## Summary

Keep this PR open. ## Best Possible Solution

## Repair Work Prompt

Treat https://github.com/openclaw/openclaw/pull/91296 as the source PR. Fix the built-in Control UI caller regression introduced by routing supervised git `update.run` through managed-service handoff: when the Gateway response is `ok: true` and `result.reason === "managed-service-handoff-started"`, the UI must not show a skipped warning or clear update tracking. It should mark an update handoff as pending, poll `update.status` after reconnect even if the immediate response lacks `after.version`, and surface final sentinel failures. Keep the Gateway protocol shape and service handoff behavior unchanged, do not add config/options, and do not edit CHANGELOG.md. Add focused tests for the new handoff-started response while preserving existing ok/error/skipped behavior.
