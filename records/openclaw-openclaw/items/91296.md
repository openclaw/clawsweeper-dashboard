## Summary

Keep this PR open: the runtime change is a plausible and likely right fix for supervised git-source updates, but it expands the public `update.run` response contract without updating docs and still lacks after-fix real behavior proof from the live self-update path.

## What This Changes

The PR routes supervised git-source `update.run` calls through the managed service handoff path while leaving unsupervised git/dev updates on the existing in-process path, with focused regression coverage.

## Best Possible Solution

Merge after documenting supervised git-source handoff in the Gateway/update docs and adding redacted after-fix live self-update proof, while preserving the existing in-process path for unsupervised git/dev runs.

## Reproduction Assessment

Yes from source for the routing bug: current main sends supervised git installs through `runGatewayUpdate`, and that runner installs dependencies and rebuilds in the git root. I did not live-reproduce the transient module-resolution failure in this read-only review.

## Solution Assessment

Mostly yes, but not complete: reusing the managed service handoff is the right owner path for supervised Gateway updates because it already runs the CLI outside the live process. The PR still needs docs/API alignment and after-fix live proof before it is the best merge-ready solution.
