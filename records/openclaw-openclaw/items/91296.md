## Summary

## What This Changes

The PR routes supervised git-source gateway `update.run` calls through the detached managed-service update handoff, extends CLI managed-service stop/restart handling to git updates, and updates focused tests and docs.

## Best Possible Solution

Land the PR after maintainer acceptance of the documented handoff response contract and service-lifecycle risk, keeping unsupervised git updates on the existing in-process path.

## Reproduction Assessment

## Solution Assessment

Yes, this appears to be the best fix shape: it reuses the existing managed-service handoff owner boundary and closes the helper-side gap by stopping the service before git rebuilds. A different in-process guard would still leave the supervisor relaunch window harder to reason about.
