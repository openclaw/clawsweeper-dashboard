## Summary

Keep open: this PR still addresses a current-main `openclaw status`/gateway status auth-resolution hang, but it is missing the required changelog entry for a user-facing CLI fix.

## What This Changes

The PR adds a bounded timeout fallback around status gateway probe auth resolution and a unit test for a never-settling auth lookup.

## Best Possible Solution

Land the bounded auth-resolution timeout with a single-line Unreleased changelog entry, keeping the fix scoped to status scan probe auth rather than changing broader gateway auth policy.

## Reproduction Assessment

Yes. At the unit seam, mock `resolveGatewayProbeAuthResolution` with a never-settling promise and call `resolveGatewayProbeSnapshot({ cfg: {}, opts: { timeoutMs: 1 } })`; current main can block before probing, while the PR falls back to empty auth and records the timeout warning.

## Solution Assessment

Yes, with the changelog fix. Bounding the pre-probe auth await in the status snapshot helper is the narrow maintainable place to prevent status scans from hanging while preserving the existing gateway probe and local RPC fallback behavior.
