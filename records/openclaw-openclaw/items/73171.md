## Summary

Keep open: the PR contains a useful status-auth robustness idea, but it conflicts with current main, lacks real behavior proof, and the timeout race does not cancel the underlying SecretRef/file/exec resolver, so it is not merge-ready.

## What This Changes

## Best Possible Solution

## Solution Assessment

No. The timeout fallback is a plausible mitigation, but the best fix should bound or cancel the actual credential resolver and prove real status behavior rather than only racing a mock promise.
