## Summary

Keep open. Current main still lacks the PR's bounded auth-resolution fallback, and the shipped local RPC fallback for #73535 runs only after a probe result exists, so it does not cover a stalled auth lookup before probing. The patch is narrow and functionally plausible, but it still needs the required user-facing changelog entry before merge.

## What This Changes

The PR adds a timeout-bounded fallback around gateway probe auth resolution for status scans and a focused unit test for a never-settling auth lookup.

## Best Possible Solution

Land the bounded auth-resolution timeout if maintainers want status scans to keep moving when SecretRef or credential lookup stalls, with a changelog entry and no broader gateway/auth policy changes. Keep the existing local RPC fallback as the separate post-probe safety net for loopback false negatives.

## Reproduction Assessment

Yes. At the unit seam, mock resolveGatewayProbeAuthResolution with a never-settling promise and call resolveGatewayProbeSnapshot({ cfg: {}, opts: { timeoutMs: 1 } }); current main waits before probeGateway, while the PR expects empty auth and a timeout warning.

## Solution Assessment

Yes, with a small process fix. Wrapping the auth-resolution await is the narrowest maintainable place to bound this pre-probe stall without changing probeGateway or callGateway behavior; the PR should add the required changelog entry before merge.
