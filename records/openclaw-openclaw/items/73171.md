## Summary

Keep open. Current main still awaits gateway probe auth resolution without a timeout, and this PR still addresses that focused status hang path, but the external PR needs real behavior proof plus a rebase/fresh CI before it is mergeable.

## What This Changes

## Best Possible Solution

Rebase the bounded auth-resolution timeout onto current main's local status RPC fallback and land it after real behavior proof plus fresh focused status validation.

I did not run the test because this was a read-only review.

## Solution Assessment

Yes, for auth lookup stalls. Bounding only the pre-probe auth await is the narrowest maintainable fix, but it should be rebased onto the current local RPC fallback and proven in a real status/gateway run.
