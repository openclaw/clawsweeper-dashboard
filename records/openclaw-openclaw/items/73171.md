## Summary

Keep open. Current main still awaits gateway probe auth resolution before probing, while this PR adds the missing bounded fallback and focused coverage; the remaining merge blocker is the required user-facing changelog entry.

## What This Changes

The PR wraps status gateway probe auth resolution in a timeout fallback that probes without configured credentials when auth lookup stalls, and adds a unit test for the never-settling auth path.

## Best Possible Solution

Land the bounded auth-resolution timeout with a single-line Unreleased changelog entry, keeping the change scoped to status scan auth lookup rather than broadening gateway auth policy.

## Reproduction Assessment

Yes. At the unit seam, mock resolveGatewayProbeAuthResolution with a never-settling promise and call resolveGatewayProbeSnapshot({ cfg: {}, opts: { timeoutMs: 1 } }); current main never reaches probeGateway, while the PR falls back to empty auth and records a timeout warning.

## Solution Assessment

Yes, with the changelog fix. Bounding the pre-probe auth await is the narrowest maintainable place to prevent status scans from hanging when auth lookup stalls, and it preserves the existing post-probe local RPC fallback.
