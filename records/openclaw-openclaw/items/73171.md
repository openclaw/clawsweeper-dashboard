## What This Changes

The PR adds a timeout-raced fallback around status scan gateway probe auth resolution and a unit test for a never-resolving auth promise.

## Best Possible Solution

Rebase onto current main, bound the actual command-secret/gateway-auth resolution path with cancellation or resolver-owned timeouts, preserve configured auth when it resolves within the user budget, and add real status/gateway proof.

## Reproduction Assessment

Unclear: the PR test simulates a never-resolving auth promise, but no real slow SecretRef/provider status run is shown. Source inspection also shows current status command secret resolution happens before this helper, so the central stall path is not fully reproduced.

## Solution Assessment

No. The direction is plausible, but the current patch is too late and too broad; the best fix should bound or cancel the actual command auth resolver and avoid unauthenticated probes when configured auth can still resolve within the status budget.
