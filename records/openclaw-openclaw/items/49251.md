## Summary

Request to queue API-limit-blocked prompts, make the waiting state visible, and automatically resume processing when capacity becomes available.

## Best Possible Solution

Keep this as the canonical design tracker until OpenClaw has a SQLite-backed capacity-wait queue that preserves prompt payloads, exposes waiting status, and replays them after provider/model capacity returns.
