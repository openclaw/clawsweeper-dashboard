## What This Changes

Request to preserve prompts when all configured models are rate-limited, show them as waiting, and automatically resume once capacity returns.

## Best Possible Solution

Keep this as the canonical product/design tracker and implement one scheduler-backed durable rate-limit pause/resume path using existing session suspension, provider Retry-After parsing, and SQLite-backed state rather than adding sidecar queue files.
