## Summary

Keep open: current main handles model fallback, cooldown summaries, and quota suspension, but it still does not durably preserve API-limit-blocked prompts, show them as waiting, and replay them when capacity returns.

## What This Changes

Request to queue prompts when all current and fallback models are blocked by API limits, make the waiting state visible, and resume automatically once capacity is available.

## Best Possible Solution

Keep this as the canonical design tracker for a SQLite-backed capacity-wait queue that preserves pending prompts, exposes waiting status, and replays them when provider/model capacity returns.

## Reproduction Assessment
