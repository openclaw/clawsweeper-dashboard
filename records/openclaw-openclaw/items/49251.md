## Summary

Keep open: current main and the latest release still keep follow-up prompt queues in process-local state, while provider cooldown exhaustion only records quota suspension metadata; the open related PRs cover useful pieces but no merged canonical fix implements the full API-limit wait, visibility, and automatic resume behavior.

## What This Changes

Issue requests preserving prompts in a visible queue and automatically resuming them when all available models are blocked by API limits.

## Best Possible Solution

Land or sponsor one core SQLite-backed durable prompt/follow-up queue design that records waiting prompts, exposes a waiting state to users, and wakes/retries from provider cooldown recovery without adding JSONL sidecar state.

## Reproduction Assessment

No high-confidence live provider-limit reproduction was run. Source inspection shows current main has process-local follow-up queues and quota-suspension metadata, but no durable prompt queue or provider-capacity wake/replay path.

## Solution Assessment

No, the JSONL-per-session suggestion in the discussion is not the best fit for this repository because AGENTS.md directs runtime state toward SQLite. The maintainable solution is a core-owned persisted queue/recovery design aligned with existing state stores and provider cooldown semantics.
