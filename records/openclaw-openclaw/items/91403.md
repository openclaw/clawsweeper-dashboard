Keep this PR open, but it is not merge-ready: the branch fixes the direct `openai-completions` adapter path while the embedded/channel runtime still uses the boundary-aware OpenAI completions transport, where a stop-only stream can still end as `done` with empty content. ## What This Changes

## Best Possible Solution
