## Summary

## What This Changes

The PR adds a direct `streamOpenAICompletions` empty-`stop` guard and adapter-level regression tests that route empty stop results to the existing error path.

## Best Possible Solution

Define one completions terminal-normalization rule for both the direct adapter and boundary-aware transport: treat provider-emitted no-output `stop` as error/retry, preserve transform-stripped intentional empty turns, and cover both paths with tests and active-path proof.

## Solution Assessment

No. The current branch is a plausible adapter mitigation, but the best fix is a shared, maintainer-approved terminal normalization that covers the active transport path and distinguishes provider no-output from output stripped by OpenClaw transforms.
