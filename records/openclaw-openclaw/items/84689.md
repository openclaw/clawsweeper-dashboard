## Label Justifications

- `P1`: The PR addresses provider credential leakage into generated `models.json`, which affects auth and security-sensitive agent context.
- `merge-risk: 🚨 compatibility`: Changing whether existing plaintext `models.json` keys are preserved can affect upgraded custom-provider setups that only authenticate from that file.
- `merge-risk: 🚨 auth-provider`: The patch changes persisted provider `apiKey` values from runtime plaintext to marker values, which can affect provider credential resolution paths.
- `merge-risk: 🚨 security-boundary`: Generated `models.json` can be agent-readable prompt context, so preserving or stripping plaintext keys is a concrete credential-boundary decision.

## Summary

Keep open for maintainer review. The patch is focused and has sufficient writer-path proof, but it preserves existing plaintext `models.json` keys in merge mode and overlaps with competing sanitizer PRs, so maintainers need to choose the canonical credential policy before merge.

## Best Possible Solution
