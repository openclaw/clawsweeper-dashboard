- `P1`: The PR addresses provider credential leakage into generated models.json, a high-impact auth/security workflow bug.

## Summary

This PR is still useful and not closeable: current main still writes provider apiKey values into models.json, while the branch adds a focused serialization sanitizer with regression coverage and sufficient writer-path proof. The remaining blocker is maintainer policy and duplicate selection, not a concrete patch defect.

## What This Changes

The branch replaces newly resolved provider apiKey values with the non-secret SecretRef marker before writing models.json, preserves merge-mode existing plaintext keys for compatibility, adds planner regression tests, and applies a small OpenRouter object-spread lint cleanup.

## Best Possible Solution

Land one canonical models.json sanitizer with an explicit policy for existing plaintext keys, then close or supersede the competing PRs and keep docs/tests aligned with that chosen behavior.
