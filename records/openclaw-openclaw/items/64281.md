## Summary

Keep open. The PR targets a useful subagent orchestration gap, but the current branch is conflicting and still has blocking package, lifecycle, security, and real-behavior-proof gaps before it can be considered for merge.

## What This Changes

The PR adds `sessions_delegate` and `sessions_delegate_batch` tools with tool catalog/display registration, spawn-and-wait execution, batch fan-out, context-engine spawn preparation, and delegate cleanup/output-capture lifecycle wiring.

## Best Possible Solution

Rebase onto current main and implement a reviewed synchronous delegation contract that reuses current spawn context, redacts persisted inputs, closes the cleanup race, documents/tests the new tool surface, and includes live proof.

## Reproduction Assessment

Yes for source-level review: the PR head and current main expose the package, sanitizer, and lifecycle mismatches without needing to run the app. This is a feature PR rather than a current-main bug reproduction.

## Solution Assessment

No. A synchronous delegate primitive may be the right direction, especially relative to https://github.com/openclaw/openclaw/issues/76145, but this patch is not the safest implementation until it rebases and fixes the security, lifecycle, and proof gaps.

Full review comments:

- **[P2] Import TypeBox from the declared package:** `src/agents/tools/sessions-delegate-tool.ts:1`
  - body: This new tool imports `Type` from `@sinclair/typebox`, but current main declares and uses `typebox` for agent tool schemas. After rebase, this will fail clean module resolution unless it uses the repo's declared package or adds an approved dependency change.
  - confidence: 0.95
- **[P1] Redact delegate attachment content before persistence:** `src/agents/tools/sessions-delegate-tool.ts:51-56`
  - body: `sessions_delegate` accepts inline `attachments[].content`, but transcript sanitization only redacts attachment payloads for `sessions_spawn`. Persisted delegate tool inputs can therefore retain raw attachment text or secrets in session history.
  - confidence: 0.94
- **[P1] Register the output gate before child completion can race:** `src/agents/tools/sessions-delegate-tool.ts:191`
  - body: The output-capture gate is registered only after `spawnSubagentDirect` returns, while run registration starts completion tracking before later spawn hook work returns. A fast child can finish and cleanup can delete the run/transcript before the delegate tool has a gate to wait on.
  - confidence: 0.89
- **[P2] Release the output gate when output reading fails:** `src/agents/tools/sessions-delegate-tool.ts:220-221`
  - body: `signalOutputCaptured(runId)` runs only after `readChildOutput` succeeds. If the history fallback throws, delegate cleanup waits for the full gate timeout even though the child already finished; wrap the output read in `try/finally`.
  - confidence: 0.83
