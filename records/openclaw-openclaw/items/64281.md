  - body: `sessions_spawn` carries inherited allow/deny tool policy into child runs, and current `spawnSubagentDirect` applies those lists to the child session patch.   - confidence: 0.93

  - body: The new schema accepts inline `attachments[].content`, but transcript sanitization currently redacts attachment payloads only for `sessions_spawn`. Persisted delegate tool-call inputs can retain raw text or secrets unless the new tool names share the redaction path or the tool avoids storing raw content.
  - confidence: 0.95
- **[P1] Create the cleanup gate before the child can finish:** `src/agents/tools/sessions-delegate-tool.ts:189-191`
  - body: The cleanup gate is created only after `spawnSubagentDirect` returns, while child run tracking and post-registration work happen inside the spawn path. A fast child can reach lifecycle cleanup before the gate exists and delete the transcript/run entry before `sessions_delegate` reads output.
  - confidence: 0.9

  - body: `signalOutputCaptured(runId)` runs only after `readChildOutput` succeeds. If the history fallback throws, lifecycle cleanup waits for the full gate timeout even though the child is done; release the gate in a `finally` block around output reading.
  - confidence: 0.84
- **[P2] Use the current TypeBox import:** `src/agents/tools/sessions-delegate-tool.ts:1`
  - body: Current main imports `Type` from `typebox` and only declares the `typebox` dependency. After rebasing, this new file should use the current package name or intentionally add dependency metadata so the tool compiles.
  - confidence: 0.82
- **[P2] Classify delegate tools as mutating:** `src/agents/tool-catalog.ts:175-192`
  - body: These catalog entries introduce tools that spawn child sessions and can delete transcripts, but the mutation classifier is not extended for `sessions_delegate` or `sessions_delegate_batch`. Failed delegate calls can be handled like read-only failures instead of preserving safeguards used for side-effectful tools.
