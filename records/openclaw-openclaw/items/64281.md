## Summary

Keep open. The PR proposes a useful but new core delegation feature that is still absent from current main, but the branch is stale/conflicting and still has concrete security and lifecycle blockers plus no after-fix real behavior proof.

## What This Changes

The PR adds core `sessions_delegate` and `sessions_delegate_batch` tools, tool catalog/display registration, delegate cleanup/output-capture lifecycle changes, and focused tests.

## Best Possible Solution

Rebase onto current main, define the synchronous delegation contract, integrate output capture before child completion can race, extend transcript redaction, update docs/changelog/tests, and require real behavior proof from a live run.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main lacks the delegate tools, the PR import conflicts with the current declared TypeBox package, delegate attachments are outside the current sanitizer, and the output gate is registered after child completion can already be waiting/cleaning up.

## Solution Assessment

No. A synchronous delegate primitive may be the right direction, especially for the foreground/background lifecycle discussion in https://github.com/openclaw/openclaw/issues/76145, but this patch is not the safest implementation until it rebases and fixes the security, lifecycle, and proof gaps.

  - confidence: 0.83
