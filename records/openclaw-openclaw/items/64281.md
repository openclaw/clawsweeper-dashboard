Keep open, but this PR is not merge-ready. The proposed synchronous delegate tools are still absent from current main, but the PR head has source-visible security, lifecycle, policy, dependency, and proof gaps that need a rebase and maintainer review before the core agent API can land.

## Best Possible Solution

Full review comments:

- **[P1] Preserve inherited tool restrictions for delegates:** `src/agents/tools/sessions-delegate-tool.ts:164-175`
  - body: `sessions_spawn` now carries inherited allow/deny tool policy into child runs, but this delegate path drops those fields before calling `spawnSubagentDirect`. A parent session with restricted tools can therefore delegate a child with broader tool access; pass the inherited allowlist and denylist through the delegate options and cover it in tests.
  - confidence: 0.92
- **[P1] Redact delegate attachments before persistence:** `src/agents/tools/sessions-delegate-tool.ts:51-56`
  - body: `sessions_delegate` accepts inline `attachments[].content`, while current transcript sanitization only redacts attachment payloads for `sessions_spawn`. Persisted delegate tool-call inputs can retain raw text or secrets unless the new tool names share the redaction path or avoid storing raw content.
  - confidence: 0.95
- **[P1] Create the output gate before child cleanup can run:** `src/agents/tools/sessions-delegate-tool.ts:191`
  - body: The cleanup gate is created only after `spawnSubagentDirect` returns, but child completion tracking starts inside the spawn path after `registerSubagentRun`. A fast child can finish cleanup before the gate exists, allowing transcript/run deletion before `readChildOutput` can read the result.
  - confidence: 0.9
- **[P2] Release the output gate in a finally block:** `src/agents/tools/sessions-delegate-tool.ts:220-221`
  - body: `signalOutputCaptured(runId)` runs only after `readChildOutput` succeeds. If the history fallback throws, lifecycle cleanup waits for the full gate timeout even though the child is already done; wrap output reading with `try/finally`.
  - confidence: 0.84
- **[P2] Import TypeBox from the declared package:** `src/agents/tools/sessions-delegate-tool.ts:1`
  - body: The repo declares and imports `typebox`, but this new file imports `@sinclair/typebox`. Without a dependency metadata change, clean installs can fail module resolution for the new tool.
  - confidence: 0.94
- **[P2] Classify delegate tools as mutating:** `src/agents/tool-catalog.ts:179-188`
  - body: These new tools spawn child sessions and can delete transcripts, but the mutation classifier is not updated for `sessions_delegate` or `sessions_delegate_batch`. Failed delegate calls can be handled like read-only failures instead of preserving the side-effect safeguards used for `sessions_spawn`.
  - confidence: 0.86
