## Summary

Keep open. Current main still lacks the proposed synchronous delegate tools, and the branch contains concrete correctness/security blockers plus missing contributor real-run proof; the later user comment confirms demand but does not resolve those blockers.

## What This Changes

The PR adds `sessions_delegate` and `sessions_delegate_batch` agent tools, registers their catalog/display entries, and modifies subagent lifecycle cleanup/output capture for delegate runs.

## Best Possible Solution

Rebase onto current main and land a reviewed core delegation contract that reuses current spawn/context plumbing, redacts delegate attachments, classifies the new tools across policy/mutation surfaces, and includes focused tests plus redacted real-run proof.

## Reproduction Assessment

Yes. Static source inspection against the PR head and current main is high-confidence: main lacks the new tools, and the PR still has source-visible dependency, redaction, mutation-policy, and cleanup-gate failure paths.

## Solution Assessment

No. A synchronous delegate primitive may be the right direction, but this patch is not the safest implementation until it rebases and fixes the security, lifecycle, dependency, policy, and proof gaps.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Redact delegate attachment content before persistence:** `src/agents/tools/sessions-delegate-tool.ts:51-56`
  - body: `sessions_delegate` accepts inline `attachments[].content`, but transcript sanitization on current main only redacts attachment payloads for `sessions_spawn`. Persisted delegate tool inputs can therefore retain raw text or secrets in session history; extend the redaction path to the new delegate tool names or avoid persisting raw content.
  - confidence: 0.95
- **[P1] Register the output gate before child completion can race:** `src/agents/tools/sessions-delegate-tool.ts:191`
  - body: The output-capture gate is created only after `spawnSubagentDirect` returns, but spawning registers the run before post-registration hooks and lifecycle work finish. A fast child can complete cleanup before this gate exists, letting cleanup delete the transcript before `readChildOutput` can read it.
  - confidence: 0.9
- **[P2] Import TypeBox from the declared package:** `src/agents/tools/sessions-delegate-tool.ts:1`
  - body: This file imports `Type` from `@sinclair/typebox`, but current main declares and uses the `typebox` package. Without an approved dependency metadata change, clean installs can fail module resolution for the new tool.
  - confidence: 0.94
- **[P2] Release the output gate when output reading throws:** `src/agents/tools/sessions-delegate-tool.ts:220-221`
  - body: `signalOutputCaptured(runId)` runs only after `readChildOutput` succeeds. If the history fallback throws, cleanup waits for the full gate timeout even though the child is already done; wrap the output read in `try/finally`.
  - confidence: 0.84
- **[P2] Classify delegate tools as mutating:** `src/agents/tool-catalog.ts:179-187`
  - body: These new tools spawn child sessions and can delete transcripts, but the mutation classifier is not updated for `sessions_delegate` or `sessions_delegate_batch`. Failed delegate calls can be treated like read-only failures in replay/error handling instead of preserving the same side-effect safeguards as `sessions_spawn`.
  - confidence: 0.86
