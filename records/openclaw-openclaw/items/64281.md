## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks the requested synchronous delegate tools, while the PR head still has build, transcript-redaction, and lifecycle race blockers on a new built-in agent execution path.

## What This Changes

The PR adds `sessions_delegate` and `sessions_delegate_batch`, registers them in core tool/catalog/display surfaces, and changes subagent cleanup to support delegate runs without auto-announce.

## Best Possible Solution

Rebase onto current main, use the existing context-engine spawn preparation path, add a lifecycle-safe output capture design, extend transcript redaction plus docs/changelog, then run the focused delegate and subagent lifecycle tests plus build/changed gates.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: clean module resolution fails on the undeclared `@sinclair/typebox` import, the sanitizer excludes the new delegate tool name, and the child completion waiter can start before the PR registers its output gate.

## Solution Assessment

No. The core primitive may be useful, but this patch is not the safest maintainable implementation until it rebases and closes the build, redaction, cleanup synchronization, and gate-release gaps.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Import TypeBox from the declared package:** `src/agents/tools/sessions-delegate-tool.ts:1`
  - body: This new file imports `Type` from `@sinclair/typebox`, but the repo declares and consistently imports the `typebox` package. A clean build/typecheck will fail module resolution unless this uses `typebox` or an approved dependency change is made.
  - confidence: 0.96
- **[P1] Redact delegate attachments before persistence:** `src/agents/tools/sessions-delegate-tool.ts:51-56`
  - body: `sessions_delegate` accepts inline `attachments[].content`, but transcript repair only redacts attachment payloads for `sessions_spawn`. Persisted tool-call inputs for delegate calls can therefore retain raw attachment text or secrets in history.
  - confidence: 0.94
- **[P1] Register the output gate before child completion can start:** `src/agents/tools/sessions-delegate-tool.ts:191`
  - body: The output-capture gate is created only after `spawnSubagentDirect` returns, while run registration starts completion tracking before later spawn hook work returns. A fast child can finish and cleanup can delete the run/transcript before the delegate tool has any gate to wait on.
  - confidence: 0.89
- **[P2] Release the output gate when output reading fails:** `src/agents/tools/sessions-delegate-tool.ts:220-221`
  - body: `signalOutputCaptured(runId)` runs only after `readChildOutput` succeeds. If the history fallback throws, delegate cleanup waits for the full gate timeout even though the child has already finished; wrap the read in `try/finally`.
  - confidence: 0.83
