## Summary

Keep open: this is a plausible core agents feature, but the current patch still drops inherited tool policy, can persist raw delegate attachment content, races cleanup/output capture, lacks after-fix real behavior proof, and is dirty against current main.

## What This Changes

The branch adds synchronous single and batch delegate agent tools, registers them in the core tool catalog/display config, and changes subagent lifecycle cleanup to return child output as a tool result.

## Best Possible Solution

Land only after a rebased core delegation contract preserves inherited tool policy, redacts persisted delegate inputs, synchronizes result capture before cleanup can run, classifies delegate tools as side-effectful, updates docs/tests, and includes redacted real-run proof.

## Reproduction Assessment

Yes for source-level PR review: current main lacks these tools, and the PR head shows the policy, redaction, cleanup-gate, dependency, and mutation-classification gaps by static inspection. I did not run live behavior because this review is read-only and the PR provides no real-run proof.

## Solution Assessment

No. A synchronous delegate primitive may be a valid direction, but this patch is not the safest implementation until it is rebased onto current subagent contracts and the security, lifecycle, dependency, and proof gaps are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Pass inherited tool policy to delegates:** `src/agents/tools/sessions-delegate-tool.ts:164-175`
  - body: `sessions_spawn` carries inherited allow/deny tool policy into child runs, but this delegate path drops those fields before calling `spawnSubagentDirect`. A restricted parent session can therefore delegate to a child with broader tool access; pass the inherited allowlist and denylist through and cover it in tests.
  - confidence: 0.93
- **[P1] Redact delegate attachment inputs:** `src/agents/tools/sessions-delegate-tool.ts:51-56`
  - body: The new tool accepts inline `attachments[].content`, while transcript sanitization only redacts attachment payloads for `sessions_spawn`. Persisted delegate tool-call inputs can retain raw text or secrets unless the new tool names share the redaction path or avoid storing raw content.
  - confidence: 0.95
- **[P1] Register the output gate before spawning:** `src/agents/tools/sessions-delegate-tool.ts:191`
  - body: The cleanup gate is created only after `spawnSubagentDirect` returns, but child completion tracking starts inside the spawn path before that return. A fast child can reach cleanup with no gate and delete the transcript/run entry before `sessions_delegate` reads output.
  - confidence: 0.9
- **[P2] Release the output gate in finally:** `src/agents/tools/sessions-delegate-tool.ts:220-221`
  - body: `signalOutputCaptured(runId)` runs only after `readChildOutput` succeeds. If the history fallback throws, lifecycle cleanup waits for the full gate timeout even though the child is done; release the gate in a `finally` block around output reading.
  - confidence: 0.84
- **[P2] Import TypeBox from the declared package:** `src/agents/tools/sessions-delegate-tool.ts:1`
  - body: The repo declares and consistently imports `typebox`, but this new file imports `@sinclair/typebox` without dependency metadata changes. Clean installs can fail module resolution for the new tool; use the declared package or add the dependency intentionally.
  - confidence: 0.94
- **[P2] Mark delegate tools as mutating:** `src/agents/tool-catalog.ts:179-188`
  - body: These new tools spawn child sessions and can delete transcripts, but the mutation classifier is not updated for `sessions_delegate` or `sessions_delegate_batch`. Failed delegate calls can be handled like read-only failures instead of preserving the safeguards used for side-effectful tools.
  - confidence: 0.86
