## Summary

Keep open. Current main still lacks `sessions_delegate` and `sessions_delegate_batch`, but the PR head is not landable: it imports an undeclared TypeBox package, does not extend transcript redaction for delegate inline attachments, and can still race child cleanup before delegate output is captured.

## What This Changes

The PR adds synchronous `sessions_delegate` and batch delegation tools, registers them in the agent tool catalog/display surfaces, and changes subagent lifecycle cleanup to support delegate runs without auto-announce.

## Best Possible Solution

Keep this as a candidate only after maintainer API approval, then rebase onto current main, use the existing context-engine spawn preparation path, fix the import and transcript redaction, make output capture synchronization happen before lifecycle cleanup can run, release gates in all paths, and add docs/changelog plus focused lifecycle/tool tests.

## Reproduction Assessment

Yes. Static review gives a high-confidence reproduction path for the blockers: clean module resolution fails on the undeclared `@sinclair/typebox` import, sanitizer matching excludes the new delegate tool name, and the gate is visibly registered after `spawnSubagentDirect` returns.

## Solution Assessment

No. The proposed tool direction may be useful, but this implementation is not the safest maintainable fix until it is rebased on current main and closes the build, redaction, cleanup-race, and gate-release gaps.
