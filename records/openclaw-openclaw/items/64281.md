## Summary

Keep open. The feature is not on current main and may be useful, but this branch still needs maintainer review because it adds a core agent API with unresolved security, cleanup, mutation-classification, rebase, and real-behavior-proof gaps.

## What This Changes

The PR adds `sessions_delegate` and `sessions_delegate_batch` agent tools, registers them in core tool/display surfaces, and adds delegate cleanup/output-capture lifecycle plumbing and tests.

## Best Possible Solution

Land only after a rebased core delegation contract preserves inherited tool restrictions, redacts or avoids persisted delegate attachment content, synchronizes cleanup before child transcript deletion, classifies delegate tools as side-effectful, updates focused tests/docs, and includes redacted real-run proof.

## Reproduction Assessment

Yes for source-level PR review: current main lacks these tools, and the latest PR head shows the policy, redaction, output-capture, dependency, and mutation-classification gaps by static inspection. I did not run live behavior because this review is read-only and the PR provides no real-run proof artifact.

## Solution Assessment

No. A synchronous delegate primitive may be a valid direction, but this patch is not yet the safest implementation until it is rebased onto current subagent contracts and the security, lifecycle, dependency, and proof gaps are resolved.

- **[P1] Pass inherited tool policy to delegates:** `src/agents/tools/sessions-delegate-tool.ts:164-175`
  - body: `sessions_spawn` carries inherited allow/deny tool policy into child runs, but this delegate path omits those fields when calling `spawnSubagentDirect`. A restricted parent session can therefore create a delegated child with broader tool access; pass the inherited allowlist and denylist through and cover it in tests.
  - confidence: 0.93
- **[P1] Redact delegate attachment inputs:** `src/agents/tools/sessions-delegate-tool.ts:51-56`
  - body: The new schema accepts inline `attachments[].content`, while current transcript sanitization only redacts attachment payloads for `sessions_spawn`. Persisted delegate tool-call inputs can retain raw text or secrets unless the new tool names share the redaction path or avoid storing raw content.
  - confidence: 0.95
- **[P1] Register the output gate before spawning:** `src/agents/tools/sessions-delegate-tool.ts:191`
  - body: The cleanup gate is created only after `spawnSubagentDirect` returns, but child completion tracking starts inside the spawn path. A fast child can reach cleanup before the gate exists and delete the transcript/run entry before `sessions_delegate` reads output.
  - confidence: 0.9
- **[P2] Release the output gate in finally:** `src/agents/tools/sessions-delegate-tool.ts:220-221`
  - body: `signalOutputCaptured(runId)` runs only after `readChildOutput` succeeds. If the history fallback throws, lifecycle cleanup waits for the full gate timeout even though the child is done; release the gate in a `finally` block around output reading.
  - confidence: 0.84
- **[P2] Import TypeBox from the current package:** `src/agents/tools/sessions-delegate-tool.ts:1`
  - body: Current main declares and imports `typebox`, while this new file imports `@sinclair/typebox`. After rebasing onto current main, use the current package name or intentionally restore the dependency metadata.
  - confidence: 0.82
- **[P2] Mark delegate tools as mutating:** `src/agents/tool-catalog.ts:179-188`
  - body: These tools spawn child sessions and can delete transcripts, but the mutation classifier is not extended for `sessions_delegate` or `sessions_delegate_batch`. Failed delegate calls can be handled like read-only failures instead of preserving safeguards used for side-effectful tools.
  - confidence: 0.86
