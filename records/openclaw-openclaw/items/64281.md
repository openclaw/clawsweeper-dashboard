## Summary

Keep open: the feature is still useful and not implemented on current main, but this PR is not mergeable as-is because it still has security, lifecycle, and package-contract blockers, plus no real behavior proof.

## What This Changes

The branch adds synchronous `sessions_delegate` and `sessions_delegate_batch` agent tools, registers them in tool catalogs/display metadata, and changes subagent cleanup/output-capture lifecycle handling.

## Best Possible Solution

Land a rebased, owner-reviewed delegate API that extends current spawn/cleanup contracts, captures output before cleanup can race, redacts persisted delegate inputs, updates docs/changelog/tests, and includes live proof.

## Reproduction Assessment

Yes. Static source review against current main and the PR head is high-confidence: current main lacks the delegate tools, and the import, sanitizer, and lifecycle-race blockers are visible in code. I did not run a live OpenClaw delegation scenario.

## Solution Assessment

No. A synchronous delegate primitive may be the right direction, especially near the related foreground/background subagent reports, but this patch needs rebase, lifecycle integration, redaction, dependency alignment, and live proof before it is the maintainable solution.

  - body: This new tool imports `Type` from `@sinclair/typebox`, but current main declares and uses `typebox` for agent tool schemas. After rebase, clean installs will not have the imported package unless the PR uses the declared package or adds an approved dependency change.

Persisted delegate tool inputs can retain raw attachment text or secrets in session history, so the new tool must reuse or extend the sanitizer before merge.
  - confidence: 0.94
- **[P1] Register the output gate before child completion can race:** `src/agents/tools/sessions-delegate-tool.ts:191`
  - body: The output-capture gate is registered only after `spawnSubagentDirect` returns, while the child run is registered before later spawn-hook work returns. A fast child can complete and trigger delete-cleanup before this gate exists, leaving `sessions_delegate` with no run entry or transcript to read.
  - confidence: 0.9
- **[P2] Release the output gate when output reading fails:** `src/agents/tools/sessions-delegate-tool.ts:220-221`
  - body: `signalOutputCaptured(runId)` runs only after `readChildOutput` succeeds. If the history fallback throws, delegate cleanup waits for the full gate timeout even though the child already finished; wrap the output read in `try/finally`.
  - confidence: 0.83
