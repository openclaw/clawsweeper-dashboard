## Summary

Keep open. Current main still does not expose the proposed synchronous delegate tools, but this PR is not ready to merge because the branch adds a new core delegation API with unresolved security, lifecycle, dependency, and proof gaps.

## What This Changes

The PR adds `sessions_delegate` and `sessions_delegate_batch` tools plus catalog/display registration, delegate output capture, and no-announce cleanup handling for subagent runs.

## Best Possible Solution

Rebase onto current main, settle the synchronous delegation contract, integrate output capture with the current lifecycle before cleanup can race, extend transcript redaction, update docs/changelog/tests, and add redacted real-run proof.

## Reproduction Assessment

Yes. Static source inspection against the PR head and current main is high-confidence: main lacks the new tools, while the PR still has source-visible dependency, redaction, and cleanup-gate failure paths.

## Solution Assessment

No. A synchronous delegate primitive may be the right direction, but this patch is not the safest implementation until it rebases and fixes the security, lifecycle, dependency, and proof gaps.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Import TypeBox from the declared package:** `src/agents/tools/sessions-delegate-tool.ts:1`
  - body: This new tool imports `Type` from `@sinclair/typebox`, but current main declares and consistently imports the `typebox` package for schemas. After rebase, this can fail clean module resolution unless it uses the declared package or adds an approved dependency change.
  - confidence: 0.95
- **[P1] Redact delegate attachments before persisting transcripts:** `src/agents/tools/sessions-delegate-tool.ts:51-56`
  - body: `sessions_delegate` accepts inline `attachments[].content`, but the transcript sanitizer only treats `sessions_spawn` attachments as redactable. Persisted delegate tool inputs can therefore keep raw attachment text or secrets in session history.
  - confidence: 0.94
- **[P1] Register the output gate before child completion can race:** `src/agents/tools/sessions-delegate-tool.ts:191`
  - body: The output-capture gate is registered only after `spawnSubagentDirect` returns, but that call registers the run and can start child completion cleanup before returning. A fast child can hit cleanup before any gate exists, letting cleanup delete the transcript before the delegate reads output.
  - confidence: 0.9
- **[P2] Release the output gate when output reading throws:** `src/agents/tools/sessions-delegate-tool.ts:220-221`
  - body: `signalOutputCaptured(runId)` runs only after `readChildOutput` succeeds. If the history fallback throws, delegate cleanup waits for the full gate timeout even though the child already finished; wrap the read in `try/finally`.
  - confidence: 0.84
