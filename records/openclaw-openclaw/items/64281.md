## Summary

Keep open: current main still does not implement same-turn `sessions_delegate` tools, but this PR is not merge-ready because it is a new core agent delegation API with source-visible security, lifecycle, dependency, and policy blockers plus no real behavior proof.

## What This Changes

The PR adds synchronous `sessions_delegate` and parallel `sessions_delegate_batch` agent tools, registers catalog/display metadata, adds tests, and changes subagent cleanup/output-capture lifecycle paths.

## Best Possible Solution

Rebase onto current main and land a reviewed core delegation contract that preserves inherited tool policy, redacts delegate inputs, synchronizes output capture before cleanup, updates mutation classification/docs, and includes redacted real-run proof.

## Reproduction Assessment

Yes for source-level PR review: current main lacks the requested tools, and the PR head has source-visible redaction, inherited-policy, output-gate, mutation-classification, and dependency gaps. I did not run live behavior because this review is read-only and the PR provides no real-run proof.

## Solution Assessment

No. A synchronous delegation primitive may be a valid direction, but this patch is not the safest implementation until it rebases onto current main and fixes the security, lifecycle, policy, and dependency issues with real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve inherited tool restrictions for delegates:** `src/agents/tools/sessions-delegate-tool.ts:164-175`
  - body: `sessions_spawn` now carries inherited allow/deny tool policy into child runs, but the delegate path drops those fields before calling `spawnSubagentDirect`. A parent session with restricted tools can therefore delegate a child with broader tool access; pass the inherited allowlist/denylist through the delegate options and tests.
  - confidence: 0.92
- **[P1] Redact delegate attachments before persistence:** `src/agents/tools/sessions-delegate-tool.ts:51-56`
  - body: `sessions_delegate` accepts inline `attachments[].content`, but the existing transcript sanitizer only redacts attachment payloads for `sessions_spawn`. Persisted delegate tool-call inputs can retain raw text or secrets unless the new tool names share the same redaction path or avoid storing raw content.
  - confidence: 0.95
- **[P1] Register the output gate before spawning can complete:** `src/agents/tools/sessions-delegate-tool.ts:191`
  - body: The cleanup gate is created only after `spawnSubagentDirect` returns, but child completion tracking starts inside the spawn path. A fast child can finish cleanup before the gate exists, allowing deletion of the transcript/run entry before `readChildOutput` can read it.
  - confidence: 0.9
- **[P2] Signal the output gate in a finally block:** `src/agents/tools/sessions-delegate-tool.ts:220-221`
  - body: `signalOutputCaptured(runId)` runs only after `readChildOutput` succeeds. If the history fallback throws, lifecycle cleanup waits for the full gate timeout even though the child is already done; wrap output reading with `try/finally`.
  - confidence: 0.84
- **[P2] Import TypeBox from the declared package:** `src/agents/tools/sessions-delegate-tool.ts:1`
  - body: The repo declares and imports `typebox`, but this new file imports `@sinclair/typebox`. Without an approved dependency metadata change, clean installs can fail module resolution for the new tool.
  - confidence: 0.94
- **[P2] Classify delegate tools as mutating:** `src/agents/tool-catalog.ts:179-188`
  - body: These new tools spawn child sessions and can delete transcripts, but the mutation classifier is not updated for `sessions_delegate` or `sessions_delegate_batch`. Failed delegate calls can be handled like read-only failures instead of preserving the side-effect safeguards used for `sessions_spawn`.
  - confidence: 0.86
