## Summary

Keep this open: current main still lacks the requested Dashboard-embedded noVNC/RFB viewer and agent-requested full-desktop takeover flow, while the plugin/MCP path only covers part of the agent-control story. The remaining Dashboard embedding, permission, token, and security model needs maintainer product/security design rather than conservative cleanup.

## What This Changes

Request to add a Dashboard-embedded noVNC/RFB viewer and later an agent-initiated full-desktop takeover flow with permission/token handoff.

## Best Possible Solution

Settle a permissioned desktop preview/control architecture: keep plugin-owned VNC tools where they fit, and add a reviewed Control UI/gateway seam only if OpenClaw should embed interactive desktop sessions.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main check that the Dashboard noVNC/RFB viewer and request_desktop_takeover flow are not present.

## Solution Assessment

Unclear/no: a VNC-only full takeover path is not yet proven to be the best maintainable solution. The safer direction needs product and security design around visual preview, semantic observation, permissions, tokens, and plugin/core boundaries.
