## Summary

Close as ClawHub/plugin work: the useful VNC desktop-control capability is optional integration surface, and current OpenClaw plugin APIs cover agent tools, prompt hooks, approvals, plugin-owned HTTP routes, session state, and Control UI descriptors. The requested stock Dashboard takeover remains a new product/security choice, not conservative core cleanup.

## What This Changes

Request to add a Dashboard-embedded noVNC viewer and an agent-requested full-desktop takeover flow with temporary VNC control tokens.

## Best Possible Solution

Publish this as a ClawHub plugin that owns the VNC/RFB client, optional browser-hosted viewer route, screenshot/click/type tools, prompt/approval hooks, credentials, and security docs; reopen core only for a specific plugin API gap that blocks that implementation.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main check that the requested stock Dashboard noVNC/RFB viewer and request_desktop_takeover flow are not present.

## Solution Assessment

Yes for cleanup: the maintainable route is a standalone ClawHub plugin using existing tools, hooks, HTTP routes, session state, and approval seams. Adding full-desktop takeover to core Dashboard would be a separate product/security decision, not the conservative fix for this item.
