## Summary

Keep this open. Current main still does not have a Dashboard-embedded VNC/RFB viewer or an agent-requested full-desktop takeover flow, and the newer plugin-path comment usefully covers part of the tool surface but not the full Dashboard/security design this issue asks for.

## What This Changes

Request a Dashboard-embedded noVNC/RFB viewer with click/type support and a later agent-requested full-desktop takeover permission/token flow.

## Best Possible Solution

Define a permissioned desktop preview/control architecture first, keeping VNC/RFB as plugin-owned where possible and extending the Control UI/plugin API only for the missing embedded viewer and approval surfaces.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken current behavior. Source inspection gives a high-confidence current-main check that the requested Dashboard VNC/RFB viewer and request_desktop_takeover flow are absent.

## Solution Assessment

No, a direct core VNC takeover patch is not clearly the best solution yet. The narrow maintainable path is a designed split between plugin-owned VNC/RFB tools, existing Codex/Peekaboo/cua desktop-control paths, and any missing Control UI permission/embedding API.
