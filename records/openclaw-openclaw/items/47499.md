Keep this open: current main has related sandbox noVNC observer and desktop-control integrations, but it still does not provide the requested Dashboard-embedded VNC viewer or agent-requested full-desktop takeover flow. The remaining work is a product and security design decision, not conservative cleanup.

## Best Possible Solution

Define a narrow permissioned preview/control design first, reusing existing sandbox noVNC, Codex Computer Use, Peekaboo/CUA, and node observer contracts where possible instead of adding an arbitrary broad VNC proxy.
