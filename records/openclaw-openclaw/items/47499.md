## Summary

Keep this open: current main has related desktop-control pieces, but it does not implement the requested Dashboard-embedded VNC viewer or an agent-requested full-desktop takeover permission flow. The remaining work is a broad product and security design decision, not a narrow cleanup close.

## What This Changes

Request an embedded noVNC-style Dashboard viewer plus a future agent-triggered full-desktop takeover flow with permission and token handoff.

## Best Possible Solution

Settle the Dashboard and gateway security boundary first, then pursue a narrow permissioned preview/control design that reuses existing Codex Computer Use, Peekaboo/CUA, node, and sandbox observer contracts instead of adding a broad arbitrary VNC proxy.

## Reproduction Assessment

Not applicable as a feature request. Source inspection does confirm the Control UI does not currently include the requested VNC/remote-desktop viewer surface.

## Solution Assessment

No, the VNC-only direction is not yet clearly the best solution. The current evidence points to a needed maintainer design decision around visual preview, semantic desktop observation, permission prompts, token handling, and whether this belongs in core or a plugin/API extension.
