## Summary

Keep open. Current main has adjacent sandbox noVNC observer and desktop-control provider paths, but it still lacks the requested Control UI embedded VNC viewer and dashboard-mediated full-desktop takeover flow; the linked implementation PR is closed unmerged and the remaining work needs maintainer product/security design.

## What This Changes

The issue requests a Control UI embedded VNC/noVNC viewer and a future permissioned agent full-desktop takeover flow.

## Best Possible Solution

Define a maintainer-approved desktop-control design that keeps agent actions on semantic provider paths and adds any Control UI viewer only as an authenticated, consent-gated visual preview with short-lived credentials, tests, and docs.

## Reproduction Assessment

Not applicable as a feature request. The current-main check is high confidence: docs and source searches show adjacent sandbox/desktop-control paths but no embedded Control UI VNC viewer or dashboard-mediated takeover RPC.

## Solution Assessment

No for a straight VNC-first takeover as proposed. The maintainable direction is a product/security design that separates visual preview from semantic desktop-control actions and reuses existing permissioned provider paths where possible.
