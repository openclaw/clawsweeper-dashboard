## Summary

Keep open. Current main still lacks the requested Control UI embedded VNC viewer and dashboard-mediated full-desktop takeover flow; adjacent sandbox noVNC and Codex/Peekaboo desktop-control paths do not cover this dashboard product/security request.

## What This Changes

The issue requests a Dashboard-embedded noVNC viewer plus a future agent-requested full desktop takeover flow with user permission and temporary control tokens.

## Best Possible Solution

Define a maintainer-approved desktop-control design that keeps agent actions on semantic/provider-owned paths and adds any Control UI viewer only as an authenticated, consent-gated visual preview with short-lived credentials, tests, and docs.

## Reproduction Assessment

Not applicable; this is a feature request, not a bug report. The current-main check is high confidence: source and docs show sandbox noVNC observer support and separate desktop-control provider paths, but no Control UI VNC viewer or dashboard-mediated takeover RPC.

## Solution Assessment

No for a straight VNC-first takeover as proposed. The maintainable path is a product/security design that separates visual preview from semantic desktop-control actions and reuses Codex, Peekaboo, or MCP provider boundaries where appropriate.
