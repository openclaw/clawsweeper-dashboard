Keep this open: current main has adjacent noVNC observer, Codex Computer Use, PeekabooBridge, and screen.snapshot pieces, but it still does not provide a Dashboard-embedded VNC viewer or an agent-requested full-desktop takeover permission flow. The issue requests a Dashboard-embedded VNC viewer plus a future permissioned agent full-desktop takeover flow.

## Best Possible Solution

Settle the Dashboard and gateway security boundary first, then pursue a narrow permissioned preview/control design that reuses existing Codex Computer Use, PeekabooBridge/CUA, node screen commands, and sandbox observer contracts where appropriate.
