Full review comments:

- **[P2] Re-apply search after stale rename successes:** `ui/src/ui/chat/session-controls.ts:753-755`
  - body: When a rename becomes stale because another edit starts before it resolves, this branch only patches the cached row label. Under an applied server search, a row renamed from a matching label to a non-matching label can stay visible until another reload, so the picker no longer reflects `sessions.list` search semantics.
  - confidence: 0.9
- **[P2] Label rename controls with row context:** `ui/src/ui/chat/session-controls.ts:973-974`
  - body: Every per-row icon-only rename button exposes the same accessible name, and the inline rename input/save controls are not tied to the row text. Screen-reader and voice-control users cannot tell which session each control edits before activating it.
  - confidence: 0.88
- **[P2] Skip unchanged rename patches:** `ui/src/ui/chat/session-controls.ts:735-737`
  - body: Saving an unchanged normalized label still calls `sessions.patch`; the gateway patch path refreshes `updatedAt`, so an accidental Save can bump the session in Recent without changing its name. Compare with the current row label and close the editor without patching when there is no label change.
  - confidence: 0.86
