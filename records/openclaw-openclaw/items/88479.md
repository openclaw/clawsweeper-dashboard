## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Re-apply search after stale rename successes:** `ui/src/ui/chat/session-controls.ts:755`
  - body: When an older rename succeeds after another edit has made its request stale, this branch only patches the cached row label. If the applied picker search matched the old label but not the new one, the row stays visible under a search that sessions.list would no longer return, until another reload happens. Reload or re-filter the current applied query instead of only mutating the cached label.
  - confidence: 0.87
- **[P2] Skip unchanged rename saves before patching:** `ui/src/ui/chat/session-controls.ts:735-737`
  - body: Saving the inline editor without changing the normalized label still calls sessions.patch. The gateway patch projection touches updatedAt for accepted patches, so a no-op save can bump that session in recent ordering even though the label did not change. Compare against the current row label and close the editor without patching when there is no effective change.
  - confidence: 0.84
- **[P2] Label rename buttons with their row:** `ui/src/ui/chat/session-controls.ts:974`
  - body: Every icon-only rename button currently exposes the same accessible name, "Rename session". In a multi-session picker, screen-reader and voice-control users cannot tell which row each action targets before activating it. Include the row label or key in the button's accessible label, or otherwise associate the control with the row text.
  - confidence: 0.92
