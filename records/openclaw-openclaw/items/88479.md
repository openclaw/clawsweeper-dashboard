## Summary

Keep this PR open, but it still needs a small session-state fix before merge: unchanged saves on displayName-only picker rows can still write a custom label and bump recency even though the visible name did not change.

## What This Changes

The PR adds inline per-row session renaming to the Control UI chat session picker, plus related UI state, styles, i18n strings, and chat-view regression tests.

## Best Possible Solution

Use one visible-label helper for both editor seeding and no-op detection, while preserving explicit clear-on-empty semantics and the existing sessions.patch write path.

## Reproduction Assessment

Yes. Source inspection shows a displayName-only picker row opens the editor with the visible label, but the no-op check compares against stored row.label only, so unchanged Save still reaches sessions.patch.

## Solution Assessment

No. Inline rename is a plausible UX, but the implementation should compare against the same visible label it edits before it is the best session-state-safe fix.
