## Summary

Keep open. Current main still treats Telegram group sessions as agent-scoped and applies the shared session visibility plus A2A policy gates to both list and send paths; related open work covers adjacent send-only or broader policy models, but does not settle the Telegram same-group cross-agent contract.

## What This Changes

Request consistent cross-agent Telegram group session listing and `sessions_send` between two agents sharing the same group.

## Best Possible Solution

Define the Telegram same-group cross-agent access contract explicitly, then either keep the documented `visibility=all` plus A2A requirement with clearer diagnostics or add a tested opt-in policy that grants same-group coordination without broad transcript exposure.

## Reproduction Assessment

Unclear for the exact asymmetric one-way report because the effective policy config is missing and no live Telegram repro was run. Source inspection does show the current cross-agent Telegram group list/send path is gated by session visibility and A2A policy on current main.

## Solution Assessment

Unclear. Automatically allowing same-group cross-agent access is not clearly the safest fix; maintainers should choose between explicit existing config, a narrower opt-in Telegram/group policy, or diagnostics that explain why the target is hidden.
