## Summary

This remains a distinct, security-sensitive WhatsApp identity issue. The shipped ghost-chat fix handles outbound phone-to-LID selection, but current main can still accept a stale reverse mapping as another contact’s identity before session routing; the exact production collision needs a controlled two-contact reproduction.

## What This Changes

Prevent WhatsApp LID/phone identity confusion from misrouting messages, contaminating unrelated sessions, and stranding final-delivery state.

## Best Possible Solution

Establish one validated WhatsApp identity before session selection, fail closed when LID and phone evidence conflict, and add a two-contact regression covering ingress isolation, outbound target selection, and pending-delivery recovery.

## Reproduction Assessment

No, not at high confidence. Current source provides a concrete way for a stale reverse mapping to resolve a LID as another phone before session routing, but the complete two-contact production collision has not been reproduced.

## Solution Assessment

No, the earlier outbound-only LID fix is not sufficient for this incident. The narrow maintainable solution must validate identity before session routing and verify the coupled delivery-state behavior.
