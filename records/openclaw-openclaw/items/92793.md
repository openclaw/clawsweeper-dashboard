## Summary

Keep open: the patch is not merge-ready because it only mitigates one stale DM config path while the linked Feishu dynamicAgentCreation bug still needs account-aware bindings, the comment-flow caller misses the new refreshed-config result mode, and the supplied proof is unit-test-only.

## What This Changes

The PR makes Feishu dynamic-agent creation consult `runtime.config.current()` for existing direct-user bindings and re-resolves Feishu DM routes when that returns a fresher config, with unit coverage.

## Best Possible Solution

Land a Feishu plugin fix that makes dynamic-agent bindings account-aware, updates both DM and comment callers to consume refreshed configs, preserves default-account legacy behavior, and includes redacted live routing proof.

## Reproduction Assessment

Yes. Source inspection shows Feishu named-account messages route with an `accountId`, generated dynamic bindings omit `match.accountId`, and core routing ignores account-less bindings for non-default accounts; I did not run a live Feishu tenant reproduction in this read-only review.

## Solution Assessment

No. The refreshed-config lookup is a plausible partial mitigation, but the maintainable fix must make dynamic binding lookup/writes account-aware and update every caller that can receive a refreshed config.
