## Summary

Keep open: this is a useful partial Feishu stale-config mitigation, but it is not merge-ready because it still does not fix the named-account dynamic-agent routing bug it claims to close, misses the sibling comment flow, and has only unit-test proof.

## What This Changes

The PR makes Feishu dynamic-agent creation consult `runtime.config.current()` for existing direct-user bindings, re-resolves DM routes with a fresher config, and adds focused unit coverage.

## Best Possible Solution

Land a Feishu plugin fix that passes account id into dynamic-agent lookup and binding writes, preserves default-account legacy behavior, updates DM and comment callers, and includes focused tests plus redacted live routing proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Feishu passes `accountId` into route resolution, dynamic bindings omit it, and routing tests assert omitted `accountId` matches only the default account; I did not run a live Feishu tenant.

## Solution Assessment

No. The refreshed-config lookup is a plausible partial mitigation, but the best fix must make dynamic binding lookup and writes account-aware and update every caller that can receive a refreshed config.
