## Summary

Keep open. Current main still has terminal chat-event handling, Web Push controls, and the public `agent_end` plugin hook, but none provides the requested opt-in browser-local completion sound for Control UI agent turns.

## What This Changes

Request an optional completion or success sound in the OpenClaw Control UI when an agent turn finishes.

## Best Possible Solution

Add an off-by-default Control UI setting that plays a short built-in browser-local chime on successful active-run terminal chat events, with storage, settings UI, docs, and focused tests; leave custom sound URLs for a separate maintainer decision unless they fit cleanly.

## Reproduction Assessment

Yes. Code inspection gives a high-confidence reproduction path: emit a final `chat` event for the active Control UI run and the current code clears run state without any sound, audio, notification, or persisted completion-sound setting path.

## Solution Assessment

Yes. The narrowest maintainable fix is Control UI-owned local playback tied to successful terminal chat events, not a backend Web Push change and not the existing `agent_end` plugin hook.
