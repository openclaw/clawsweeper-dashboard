## Summary

Keep open. Current main still has terminal chat handling, manual Web Push controls, and the public `agent_end` hook, but none provides the requested opt-in browser-local completion sound for Control UI agent turns.

## What This Changes

Request an optional Control UI completion/success sound when an agent turn finishes, with a settings toggle and possible custom sound configuration.

## Best Possible Solution

Add an off-by-default Control UI setting that plays a short built-in local chime only when the active chat run reaches a successful final state, with docs, changelog, and focused UI/storage tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: emit a final `chat` event for the active run and the UI clears the run state, while settings and notification code have no completion-sound path.

## Solution Assessment

Yes. The narrowest maintainable fix is Control UI-owned local playback tied to terminal chat events; the existing `agent_end` hook and Web Push controls are useful but do not provide this background-tab audio cue.
