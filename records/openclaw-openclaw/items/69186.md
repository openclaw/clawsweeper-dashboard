## Summary

Keep open. Current main still handles successful terminal chat events and Web Push settings, but it does not provide the requested opt-in browser-local completion sound for Control UI agent turns.

## What This Changes

Add an optional audible completion/success notification in the Control UI when an agent turn finishes.

## Best Possible Solution

Add an off-by-default Control UI setting that plays a short built-in browser-local chime only when the active chat run reaches a successful final state, with docs, changelog, and focused storage/UI/chat tests.

## Reproduction Assessment

Yes. Emit a final `chat` event for the active Control UI run: current code clears the active run and flushes/reloads chat state, while settings and notification surfaces have no completion-sound path.

## Solution Assessment

Yes. The narrowest maintainable fix is Control UI-owned local audio tied to successful terminal chat events; `agent_end` hooks and Web Push are useful adjacent features but do not provide the requested background-tab chime.

## Repair Work Prompt

Implement this issue as a narrow Control UI feature. Add an off-by-default browser-local completion sound setting for successful agent turn completion in the Control UI. Keep the first implementation local and built-in: do not add backend push behavior, a new plugin API, remote custom sound fetching, or provider/channel-specific logic unless a maintainer explicitly asks. Wire playback to the active chat run reaching `state: "final"`, avoid playing for aborted/error/unowned side-result terminal events, and handle browser autoplay/playback failures without surfacing noisy errors. Persist the setting through `UiSettings`, expose it in the existing Settings/Notifications or Appearance area, update docs at `docs/web/control-ui.md`, and add a user-facing `CHANGELOG.md` entry.
