## Summary

Keep open. Current main still has terminal chat-event handling, manual Web Push subscribe/test controls, and the public `agent_end` plugin hook, but none of those implements the requested opt-in browser-local completion chime or automatic turn-complete notification in the Control UI.

## What This Changes

Request adding an optional Control UI sound or notification when an agent finishes generating a response.

## Best Possible Solution

Add an off-by-default Control UI setting that plays a short built-in browser-local chime when the active chat run reaches a successful final state. Keep it UI-owned, catch playback failures, document the behavior, add focused UI/storage tests, and defer custom remote sound URLs unless maintainers agree on a privacy policy.

## Reproduction Assessment

Yes. Code inspection against current main is enough: final chat events are handled and active run state is cleared, while settings, notification, docs, and search results show no completion-sound preference or playback path.

## Solution Assessment

Yes. The narrowest maintainable fix is a Control UI-owned browser setting tied to terminal chat events; relying on the existing `agent_end` plugin hook alone would not solve the requested WebChat/background-tab audio cue.
