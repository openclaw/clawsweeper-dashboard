The branch adds gateway disk discovery for `{session}.checkpoint.<uuid>.jsonl` transcripts, threads discovered checkpoints through session row/list/get/branch/restore APIs, and adds regression coverage for missing and malformed checkpoint metadata.

## Best Possible Solution

## Solution Assessment

Yes. Gateway-side disk discovery behind the existing session APIs is the maintainable boundary, and the latest head fixes the previous partial-metadata failure by separating projectable and fully usable checkpoint metadata.
