## Summary

Keep open: the latest head addresses the prior timestamp-normalization feedback, but it still has a restore consistency bug for disk-discovered checkpoints, so the PR should not merge as-is.

## What This Changes

The branch adds gateway disk discovery of `{session}.checkpoint.<uuid>.jsonl` files for session rows and compaction list/get/branch/restore, plus focused regression tests.

## Best Possible Solution

Keep the gateway disk-discovery approach, but preserve disk-discovered checkpoints through restore and cover that path with a focused regression test.

## Reproduction Assessment

Yes. Source inspection shows current main list/get/row projection consume only `entry.compactionCheckpoints`, so a metadata-missing session with valid checkpoint files reproduces the empty API/UI state; I did not execute a live gateway repro in this read-only pass.

## Solution Assessment

No, not as-is. Gateway-side disk discovery is the right boundary, but restore must keep disk-derived checkpoint history consistent with metadata-backed restore behavior.
