## Summary

Keep open: the PR fixes the source-backed disk-checkpoint discovery gap, but the session-row preview path now repeats a synchronous scan of every session-directory filename for each listed row, which should be indexed before merge.

## What This Changes

The PR adds gateway discovery of `{session}.checkpoint.<uuid>.jsonl` transcript files for session row previews and compaction list/get/branch/restore APIs when stored checkpoint metadata is missing or incomplete.

## Best Possible Solution

Keep the disk-discovery approach, but build a per-directory checkpoint preview index once per session-list context and land after the focused gateway tests and changed checks stay green.

## Reproduction Assessment

Yes. Source inspection shows current main reads checkpoint rows and compaction APIs from `entry.compactionCheckpoints` only, so a session with valid disk checkpoint files and missing metadata returns no checkpoints.

## Solution Assessment

Not quite. Gateway-side disk discovery is the right boundary, but the row-preview implementation should index checkpoint filenames once per directory before this is the best merge shape.

## Review Findings

Full review comments:

- **[P2] Index checkpoint filenames once per directory:** `src/gateway/session-utils.ts:434-435`
  - body: `discoverDiskCompactionCheckpointPreviewsSync` is called while building each session row, but it loops over the entire cached filename list every time. Build a per-directory index in the row context and have each row look up its checkpoint base instead.
