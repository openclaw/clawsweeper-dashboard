## Summary

Keep open. This is a focused bug-fix PR for the linked open issue; current main is still metadata-only for compaction checkpoints, while the patch targets the implicated gateway paths with regression coverage and sufficient terminal proof.

## What This Changes

The PR adds gateway-side discovery of `{session}.checkpoint.<uuid>.jsonl` transcript files and wires it into session rows, compaction list/get, branch/restore, and regression tests.

## Best Possible Solution

Land this gateway-side reconciliation after maintainer review so historical disk checkpoint files, row summaries, list/get, branch, and restore all use one consistent checkpoint view.

## Reproduction Assessment

Yes. On current main, a session-store entry with no `compactionCheckpoints` but valid matching checkpoint transcript files will still produce empty checkpoint row/list data because the gateway reads only stored metadata.

## Solution Assessment

Yes. The PR fixes the server-side source of truth behind the existing gateway APIs and validates discovered transcript files before returning full checkpoint objects, which is narrower than a UI-only workaround.
