## Best Possible Solution

Refresh the branch against current main, preserve the current agent-scoped compaction API behavior, and land only after maintainers accept disk-discovered checkpoints as the recovery source for historical sessions.

## Reproduction Assessment

Yes. Current main's session row and compaction API paths only project persisted `compactionCheckpoints`, so a session with valid checkpoint files on disk but no metadata reproduces the empty-list behavior by source inspection.

## Solution Assessment

Mostly yes, pending maintainer acceptance and rebase. The PR fixes the implicated gateway path with focused tests, but the final solution must preserve current main's agent-scoped handlers and intentionally accept disk files as canonical recovery evidence.
