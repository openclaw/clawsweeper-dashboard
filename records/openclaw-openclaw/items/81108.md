## Summary

Keep open. Current main still has the metadata-only checkpoint behavior reported by the linked issue, while this PR is a focused fix with sufficient terminal proof, green relevant CI, and no blocking review findings.

## What This Changes

The PR adds gateway-side discovery of disk checkpoint transcript files when session metadata lacks `compactionCheckpoints`, wiring that into session rows, compaction list/get, branch/restore, and regression tests.

## Best Possible Solution

Land this PR or an equivalent gateway-side reconciliation that validates disk checkpoint files and keeps rows, list/get, branch, and restore behavior consistent for historical sessions.

## Reproduction Assessment

Yes. On current main, a session entry with no `compactionCheckpoints` but valid checkpoint transcript files will produce empty checkpoint row/list data because the gateway reads only stored metadata.

## Solution Assessment

Yes. Server-side disk discovery behind the existing gateway API is the narrow maintainable fix; a UI-only workaround would still leave get, branch, and restore inconsistent.
