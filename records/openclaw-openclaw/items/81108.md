## What This Changes

The branch discovers `.checkpoint.<uuid>.jsonl` files from disk for gateway session rows and compaction list/get/branch/restore paths, routes branch/restore through the session accessor, and adds gateway regression coverage.

## Best Possible Solution

Land this after a maintainer explicitly accepts runtime discovery/healing for legacy checkpoint files; otherwise move the repair into a doctor/migration path and keep runtime reads metadata-first.

## Reproduction Assessment

Yes, source inspection plus the linked report give a high-confidence path: current main's compaction list/get/branch/restore APIs only read stored checkpoint metadata, so a session with valid disk checkpoint files but no `compactionCheckpoints` metadata returns empty checkpoint results.

## Solution Assessment

Yes, if maintainers accept runtime legacy discovery as the desired repair path; the latest shape is no longer a gateway-side mutation bypass because branch/restore still go through the session accessor. The safer alternative is an explicit doctor/migration rebuild instead of runtime healing.
