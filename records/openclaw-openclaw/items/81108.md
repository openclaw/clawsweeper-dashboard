The PR adds gateway-side disk discovery for `{session}.checkpoint.<uuid>.jsonl` transcripts and wires discovered checkpoints into session rows, compaction list/get, branch/restore, and focused tests.

## Best Possible Solution

Yes. A current-main session entry with no `compactionCheckpoints` but valid sibling checkpoint transcript files will return empty checkpoint row/list data because the gateway reads stored metadata only; I verified this from source rather than by running a live failing setup.

## Solution Assessment

Yes. Server-side discovery behind the existing gateway APIs is the narrow maintainable fix; a UI-only workaround would leave get, branch, and restore inconsistent.
