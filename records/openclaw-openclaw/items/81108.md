## What This Changes

The PR adds gateway discovery of valid `{session}.checkpoint.<uuid>.jsonl` files for session rows and compaction list/get/branch/restore when persisted checkpoint metadata is missing or incomplete.

## Best Possible Solution

Land the disk-discovery approach after normal maintainer review and merge gates, keeping the projectable-vs-fully-usable checkpoint helper split to protect API recovery from malformed stored metadata.

## Reproduction Assessment

Yes, source-backed. Current main's row and compaction APIs read only `entry.compactionCheckpoints`, so a session with valid disk checkpoint files but missing checkpoint metadata returns no usable checkpoints.

## Solution Assessment

Yes. Gateway-side disk discovery behind the existing session APIs is the right boundary, and the latest head validates fully usable stored checkpoint metadata before dedupe on the API path.
