## What This Changes

The issue asks to rename OpenClaw's internal `cron` scheduler subsystem to `scheduler` or another less ambiguous name across CLI, tools, config, state, session keys, and docs with a migration alias.

## Best Possible Solution

Maintainers should choose the canonical name and deprecation policy first, then land an additive alias/migration plan that keeps existing `cron` jobs and integrations working while new docs and UI use the chosen scheduler name.
