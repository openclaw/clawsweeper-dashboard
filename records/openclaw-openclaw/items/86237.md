## What This Changes

Request to rename OpenClaw's cron-named scheduler surfaces to `scheduler` or another clearer canonical name while keeping a temporary compatibility path.

## Best Possible Solution

Choose the canonical name and compatibility contract first, then implement an additive `scheduler` surface with tested `cron` aliases, doctor-backed config/state migration, docs updates, and explicit upgrade behavior across CLI, Gateway methods, tools, plugin SDK, and session/state handling.

## Solution Assessment

Unclear until maintainers choose the naming and compatibility policy. If the rename is accepted, the alias-first migration shape discussed in the issue is the maintainable direction, but it is too broad and compatibility-sensitive for an autonomous cleanup patch.
