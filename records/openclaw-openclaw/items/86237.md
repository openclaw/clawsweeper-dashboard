## Summary

## What This Changes

Rename OpenClaw's internal `cron` scheduler surface to a less ambiguous name such as `scheduler`, with compatibility aliases and migration.

## Best Possible Solution

Maintainers should first choose the canonical name and deprecation policy, then implement an additive `scheduler` surface with tested `cron` aliases, doctor-backed migration, docs updates, and explicit upgrade behavior across CLI, Gateway methods, tools, plugin SDK, and state/session handling.

## Solution Assessment
