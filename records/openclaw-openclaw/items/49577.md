## Summary

Keep open. Current main still has component-level liveness, timestamps, plugin-supplied channel warnings, and gateway probe warnings, but I did not find the generic inbound-receipt to agent-dispatch health signal requested here. The linked #72724 PR is still recorded as open and unmerged with closing wording, so repository policy says to keep this issue open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to make false-healthy gateway/channel states visible when running, connected, or process-alive checks pass but inbound messages are not reaching the agent path, including confusing multi-gateway process cases.

## Best Possible Solution

Review and land #72724, or replace it with an equivalent generic core implementation that surfaces inbound receipt, dispatch progress, session-turn creation, processing outcomes, no-recent-agent-activity, and multi-listener warnings without hard-coding Feishu or any other channel plugin internals.

## Reproduction Assessment

No. The operational report is concrete, but I did not have a live Feishu gateway setup; the missing generic status telemetry is reproducible by inspecting the current status aggregation and channel issue collection paths.

## Solution Assessment

Yes. The maintainable direction is a plugin-agnostic gateway/status telemetry layer, with channel plugins contributing only channel-specific diagnostics; per-channel false-healthy fixes would be narrower than the reported failure mode.
