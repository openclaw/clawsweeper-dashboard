## Summary

Keep open: current main still rewrites the secret-bearing systemd EnvironmentFile from generated state-dir `.env` entries without reading, merging, or backing up existing operator entries, and the open closing PR #76885 has not merged.

## What This Changes

This issue requests non-destructive upgrade and doctor handling for the managed systemd EnvironmentFile, preserving operator keys and adding backup/notification on rewrite.

## Best Possible Solution

Make systemd env-file staging non-destructive by preserving unrelated existing keys, defining conflict precedence, backing up or clearly reporting content-changing rewrites, and adding regression coverage for doctor/update staging.

Yes. Source-reproducible: create a state-dir `.env`, prepopulate `gateway.systemd.env` with an extra provider key, then run the systemd staging path; current main rewrites the file from `dotenvVars` only.

Yes. The narrow maintainable fix belongs in the systemd EnvironmentFile writer and its doctor/update staging tests; #76885 is the active implementation candidate, but this issue should not close before a fix merges.
