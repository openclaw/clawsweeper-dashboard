## Summary

Keep open: current main still has a source-reproducible destructive rewrite path for `gateway.systemd.env`, and the linked fix PR is open but not merged.

## What This Changes

Preserve operator-customized `gateway.systemd.env` keys during upgrade, doctor repair, or service restaging instead of replacing the file with bundled/default `.env` entries.

## Best Possible Solution

Review and land PR #76885 or an equivalent narrow fix that merges existing env-file keys, preserves permissions, backs up before content-changing rewrites, and adds regression coverage.

## Reproduction Assessment

Yes. Source reproduction is high confidence: create state-dir `.env` with one key and an existing `gateway.systemd.env` with extra operator keys, then `stageSystemdService` will rewrite the env file from only `.env`-derived entries.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the systemd env-file staging writer and tests, without changing provider auth precedence or broader service-install policy.
