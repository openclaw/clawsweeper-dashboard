## Summary

Keep open: current main still overwrites `gateway.systemd.env` from generated state-dir `.env` entries without reading, merging, or backing up existing operator contents, and #76885 is an open unmerged closing PR for the same bug.

## What This Changes

Prevent upgrade/doctor systemd staging from overwriting operator-added secrets in `~/.openclaw/gateway.systemd.env`.

## Best Possible Solution

Land or revise #76885, or a narrow replacement, so systemd env-file staging is non-destructive and covered by regression tests for doctor/update restaging.

## Reproduction Assessment

Yes, source-reproducible: prepopulate `gateway.systemd.env` with extra provider keys, create a state-dir `.env`, then call the systemd staging path reached by doctor/update. The current writer serializes only `dotenvVars` and overwrites the env file, so extra operator keys are lost.

## Solution Assessment

Yes, the narrow maintainable fix belongs in `writeSystemdGatewayEnvironmentFile` and its systemd staging tests. #76885 is the active implementation candidate, but this issue should remain open until a fix merges.
