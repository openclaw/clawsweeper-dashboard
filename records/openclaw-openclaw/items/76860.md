## Summary

Keep open: current main still rewrites `gateway.systemd.env` from generated state-dir `.env` values without reading, merging, or backing up existing operator entries, and the open closing PR #76885 has not merged.

## What This Changes

Preserve operator-customized `gateway.systemd.env` contents during upgrade or `doctor` systemd restaging instead of silently reverting the file to generated defaults.

## Best Possible Solution

Make systemd env-file staging non-destructive by merging existing operator keys, preserving generated-key precedence deliberately, backing up before content-changing writes, and adding regression coverage for doctor/update restaging.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: prepopulate `gateway.systemd.env` with an operator-only key, provide generated state-dir `.env` entries, and call `stageSystemdService`; the current writer emits only `dotenvVars`.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the systemd env-file writer plus focused staging tests; #76885 is the active implementation candidate, but this issue should not close before a fix merges.
