## Summary

Keep open. Current main still lacks the requested opt-in Gateway Responses built-in tool-call exposure, and the issue is paired with the open closing implementation PR at https://github.com/openclaw/openclaw/pull/75075, so it should not be closed before that PR or an equivalent fix lands.

## What This Changes

The issue requests a default-off `gateway.http.endpoints.responses.exposeBuiltInToolCalls` option that emits built-in agent tool calls as Responses `function_call` output items in JSON and SSE responses.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75075 or an equivalent narrow implementation that keeps the feature default-off, separates audit output from client-tool delegation, and updates config schema, docs, changelog, and JSON/SSE regression coverage.

## Reproduction Assessment

Yes, at source level. Current main emits run-scoped built-in `tool/start` events, but the `/v1/responses` JSON and SSE paths only serialize assistant messages plus caller-provided `pendingToolCalls`, and the requested config key is absent from types/schema/docs.

## Solution Assessment

Unclear until the active PR is reviewed. A default-off Responses-shaped `function_call` audit item is a plausible narrow solution, but maintainers should verify redaction, shell/tool-name semantics, and wire-format scope before merge.
