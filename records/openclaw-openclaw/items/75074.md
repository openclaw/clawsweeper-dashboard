## Summary

Keep open. Current main still lacks the requested `/v1/responses` built-in tool-call exposure path, and open closing PR #75075 is the active implementation candidate, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Request an opt-in `gateway.http.endpoints.responses.exposeBuiltInToolCalls` flag that surfaces built-in agent tool calls in `/v1/responses` JSON and SSE output.

## Best Possible Solution

Land #75075 or an equivalent default-off implementation that captures built-in tool start events in JSON and SSE while keeping client-tool delegation distinct and documented.

## Reproduction Assessment

Yes. A source-level reproduction is clear on current main: built-in `stream: "tool"` start events are emitted, but `/v1/responses` only serializes assistant messages and caller-provided client-tool `function_call` items.

## Solution Assessment

Unclear until #75075 is reviewed. A default-off Responses-shaped `function_call` audit item is a plausible narrow solution, but maintainers should verify wire-format and argument-redaction semantics before merge.
