## Summary

Keep open. Current main still lacks the requested opt-in `/v1/responses` built-in tool-call exposure, while open closing PR #75075 is the active implementation candidate and should be reviewed or replaced before this issue is closed.

## What This Changes

Request an opt-in `gateway.http.endpoints.responses.exposeBuiltInToolCalls` setting that surfaces built-in tool starts as Responses `function_call` output items in JSON and SSE.

## Best Possible Solution

Land reviewed PR #75075 or an equivalent narrow implementation that stays default-off and updates config schema, docs, changelog, JSON behavior, and SSE regression coverage.

## Reproduction Assessment

Yes, at source level. Current main emits run-scoped built-in `stream: "tool"` start events, but `/v1/responses` only serializes assistant messages and caller-provided `pendingToolCalls`.

## Solution Assessment

Unclear until the active PR is reviewed. A default-off Responses-shaped `function_call` audit item is a plausible narrow solution, but maintainers should verify argument exposure, redaction, output ordering, and wire-format semantics before merge.
