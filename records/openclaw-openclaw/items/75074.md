## Summary

Keep open. Current main still lacks the requested opt-in Gateway Responses surface for built-in tool calls, and the issue is paired with the open closing implementation PR at https://github.com/openclaw/openclaw/pull/75075, so it should stay open until that PR or an equivalent fix lands.

## What This Changes

Request to add a default-off `gateway.http.endpoints.responses.exposeBuiltInToolCalls` option that emits built-in tool starts as Responses `function_call` output items in JSON and SSE responses.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75075 or an equivalent narrow implementation that keeps the feature default-off, start-only, run-scoped, sanitized, and covered in config schema, docs, JSON, and SSE tests.

## Reproduction Assessment

Yes, at source level. Current main emits run-scoped built-in tool-start events, but `/v1/responses` serializes assistant output and caller-provided client-tool calls only, and the requested config key is absent from types, schema, and docs.

## Solution Assessment

Unclear until maintainer API review. A default-off Responses-shaped `function_call` audit item is a plausible narrow direction, but the linked PR needs rebase and review for redaction, naming, and wire-format scope before this issue can close.
