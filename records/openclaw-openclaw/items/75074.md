## Summary

Keep open. Current main still lacks the requested opt-in `/v1/responses` built-in tool-call exposure path, and open closing PR #75075 is the active implementation candidate, so this issue should not be closed before that PR or an equivalent fix lands.

## What This Changes

Add an opt-in `gateway.http.endpoints.responses.exposeBuiltInToolCalls` setting so `/v1/responses` can include built-in agent tool invocations as `function_call` output items in JSON and SSE responses.

## Best Possible Solution

Land reviewed PR #75075 or an equivalent narrow implementation that keeps the feature default-off, separates audit items from client-tool delegation, and updates config schema, docs, changelog, and JSON/SSE regression coverage.

## Reproduction Assessment

Yes, at source level. Current main emits built-in `stream: "tool"` start events, but `/v1/responses` only serializes assistant messages and caller-provided `pendingToolCalls`.

## Solution Assessment

Unclear until the open PR is reviewed. A default-off Responses-shaped `function_call` audit item is a plausible narrow solution, but maintainers should verify argument exposure, redaction, and wire-format semantics before merge.
