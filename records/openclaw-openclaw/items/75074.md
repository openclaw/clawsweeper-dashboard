## Summary

Keep open. Current main still lacks the requested opt-in `/v1/responses` built-in tool-call exposure path, and open closing PR #75075 is the active implementation candidate, so this issue should not be closed before that PR or an equivalent change lands.

## What This Changes

The issue requests a default-off `gateway.http.endpoints.responses.exposeBuiltInToolCalls` option that surfaces built-in agent tool starts as Responses `function_call` output items in JSON and SSE.

## Best Possible Solution

Land reviewed PR #75075 or an equivalent narrow implementation that keeps the feature default-off, separates audit items from client-tool delegation, and updates config schema, docs, changelog, and JSON/SSE regression coverage.

## Reproduction Assessment

Yes, at source level. Current main emits run-scoped built-in `tool` start events, but `/v1/responses` only serializes assistant messages and caller-provided pending tool calls.

## Solution Assessment

Unclear until the open PR is reviewed. A default-off Responses-shaped `function_call` audit item path is a plausible narrow solution, but maintainers should verify redaction and streaming index semantics before merge.
