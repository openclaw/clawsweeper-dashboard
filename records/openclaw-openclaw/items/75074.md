## Summary

Keep open. Current main still lacks the requested default-off built-in tool-call exposure path, and open closing PR #75075 is the active implementation candidate, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests an opt-in `/v1/responses` flag that surfaces built-in agent tool calls as Responses `function_call` output items in JSON and SSE responses.

## Best Possible Solution

Land a reviewed implementation through #75075 or an equivalent narrow PR that preserves the default-off contract, captures built-in start events in JSON and SSE output, keeps client-tool delegation distinct, and documents the audit surface.

## Reproduction Assessment

Yes. A source-level reproduction is clear on current main: built-in tool start events are emitted by the embedded runner, but `/v1/responses` only serializes assistant messages and caller-provided client tool calls.

## Solution Assessment

Unclear until #75075 is reviewed. A default-off, Responses-shaped `function_call` audit item is a plausible narrow solution, but maintainers should verify the security and wire-format details before merge.
