## Summary

Keep open. Current main still lacks the requested default-off `/v1/responses` flag and built-in tool-call capture path, and open closing PR #75075 is the active implementation path, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for a default-off Gateway config flag that makes `/v1/responses` include built-in tool starts as Responses-shaped `function_call` output items in JSON and SSE responses.

## Best Possible Solution

Review and land #75075, or an equivalent narrow implementation, with the default-off config flag, JSON and SSE coverage, client-tool collision/dedupe coverage, docs, changelog, and an explicit security review of argument exposure. Tool results, trace digests, and built-in delegate mode should stay separate follow-ups.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to run `/v1/responses` on current main with an agent turn that invokes a built-in tool: built-in `stream: "tool"` events exist, but the HTTP handler has no config flag or capture branch, so saved output remains assistant-only unless caller-provided client `pendingToolCalls` exist.

## Solution Assessment

Unclear until #75075 is reviewed. A default-off Responses-shaped `function_call` audit item is a plausible narrow fix, but maintainers should verify the security and wire-format details before merge.
