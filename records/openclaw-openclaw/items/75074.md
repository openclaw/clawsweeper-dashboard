## Summary

Keep open. Current main still lacks the requested opt-in built-in tool-call exposure, and open closing PR #75075 is the active implementation candidate, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Request to add a default-off `gateway.http.endpoints.responses.exposeBuiltInToolCalls` option that serializes built-in agent tool starts as Responses `function_call` output items in JSON and SSE.

## Best Possible Solution

Review and land PR #75075 or an equivalent narrow default-off implementation with config schema, docs, changelog, JSON/SSE coverage, and clear argument-exposure semantics.

## Reproduction Assessment

Yes, at source level. Current main emits built-in `stream: "tool"` start events, but `/v1/responses` only serializes assistant messages and caller-provided `pendingToolCalls`.

## Solution Assessment

Unclear until the open PR is reviewed. A default-off Responses-shaped `function_call` audit item is a plausible narrow direction, but maintainers should verify redaction and protocol semantics before accepting the new config contract.
