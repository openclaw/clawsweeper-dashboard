## Summary

Keep open. Current main still lacks the requested `/v1/responses` built-in tool-call exposure path, and open closing PR #75075 is the active implementation candidate, so this issue should not be closed before that PR or an equivalent fix lands.

## What This Changes

The issue requests a default-off `/v1/responses` config flag that surfaces built-in agent tool calls as Responses `function_call` output items in JSON and SSE responses.

## Best Possible Solution

Land reviewed PR #75075 or an equivalent narrow implementation that preserves the default-off contract, captures built-in start events in JSON and SSE output, keeps client-tool delegation distinct, and documents the audit surface.

## Reproduction Assessment

Yes. A source-level reproduction is clear on current main: built-in `stream: "tool"` start events are emitted, but `/v1/responses` only serializes assistant messages and caller-provided client tool calls.

## Solution Assessment

Unclear until #75075 is reviewed. A default-off Responses-shaped `function_call` audit item is a plausible narrow solution, but maintainers should verify argument exposure/redaction and wire-format semantics before merge.
