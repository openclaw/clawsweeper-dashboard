## Summary

Keep open. Current main still lacks the requested `/v1/responses` built-in tool-call exposure path, and open closing PR #75075 is the active implementation candidate, with #75107 and the m13v comment adding useful corroborating context.

## What This Changes

The issue asks for a default-off gateway config flag that surfaces built-in agent tool start events as Responses `function_call` output items in JSON and SSE `/v1/responses` responses.

## Best Possible Solution

Land reviewed PR #75075 or an equivalent narrow implementation that keeps the flag default-off, preserves client-tool delegation behavior, documents the contract, and adds regression coverage for JSON and SSE output.

## Reproduction Assessment

Yes. Source inspection on current main shows `/v1/responses` only serializes client `pendingToolCalls` and assistant messages, while the embedded runner already emits run-scoped built-in `stream: "tool"` start events.

## Solution Assessment

Unclear until #75075 is reviewed. A default-off Responses-shaped `function_call` audit path is a plausible narrow solution, but maintainers should verify redaction, ordering, and compatibility semantics before merge.
