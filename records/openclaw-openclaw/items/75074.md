## Summary

Keep open because current main still lacks the requested opt-in built-in tool-call exposure, and open closing PR #75075 is already the active implementation candidate.

## What This Changes

Request an opt-in `/v1/responses` config flag that includes built-in tool-call audit items in JSON and SSE output.

## Best Possible Solution

Land reviewed PR #75075 or an equivalent narrow implementation that keeps the feature default-off, documents the audit surface, and preserves client-tool delegation semantics.

## Reproduction Assessment

Yes. A source-level reproduction is clear on current main: built-in tool start events are emitted by the embedded runner, but `/v1/responses` serializes only assistant messages and caller-provided client tool calls.

## Solution Assessment

Unclear until #75075 is reviewed. A default-off Responses-shaped `function_call` audit item is a plausible narrow solution, but maintainers should verify the security and wire-format details before merge.
