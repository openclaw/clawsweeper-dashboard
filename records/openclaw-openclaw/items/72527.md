## What This Changes

The issue reports that downgrading from OpenClaw 2026.4.x to 2026.3.2 can leave newer `openclaw.json` keys and value shapes that break config loading without actionable downgrade guidance.

## Best Possible Solution

Keep this as the canonical downgrade-config issue, and revise or replace the linked PR so invalid-config diagnostics and config snapshots can surface newer-version remediation before deciding any automatic doctor rewrites.

## Reproduction Assessment

Yes by source inspection. A future-stamped config with unknown keys or changed value shapes fails validation before the newer-version warning path, and the snapshot path also returns invalid before warning.

## Solution Assessment

No for the currently linked PR as written. Updating only `warnIfConfigFromFuture` misses invalid downgraded configs; the maintainable fix should share downgrade remediation with invalid-config and snapshot diagnostics while keeping doctor repair claims precise.
