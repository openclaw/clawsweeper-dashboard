## Summary

Keep this issue open. Current `main` has partial downgrade protection and docs, but the reported invalid downgraded config can still fail schema validation before the future-version warning path runs, and the active closing PR #72535 remains open and unmerged.

## What This Changes

The issue asks for clearer warning and repair behavior when `openclaw.json` was written by a newer OpenClaw version and then opened by an older downgrade target.

## Best Possible Solution

Review and land, expand, or replace #72535 so newer-version guidance appears even for invalid downgraded configs, and decide whether doctor should rewrite known downgrade-incompatible shapes or list exact manual edits.

## Reproduction Assessment

Yes. The issue gives concrete 2026.4.24 to 2026.3.2 steps and current-main source inspection shows invalid downgraded values can hit invalid-config handling before newer-version warning logic runs.

## Solution Assessment

Unclear. A warning-copy-only patch helps, but the narrower maintainable fix likely needs invalid-load/snapshot warning coverage and an explicit doctor repair policy for known value-format changes.
