## Summary

Keep this issue open. Current main has partial downgrade guardrails, but the reported invalid downgraded config can still miss the newer-version warning because invalid config paths return or throw before `warnIfConfigFromFuture`, and the active closing PR #72535 is still open in the provided context.

## What This Changes

The issue asks for downgrade-aware warnings and repair guidance when a newer OpenClaw writes `openclaw.json` that an older OpenClaw cannot parse.

## Best Possible Solution

Review and land, expand, or replace #72535 so newer-version guidance appears even for invalid downgraded configs, then define whether doctor should rewrite known downgrade-incompatible shapes or list exact manual edits.

## Reproduction Assessment

Yes. The issue gives concrete 2026.4.24 to 2026.3.2 steps, and current-main source inspection shows a future-stamped but schema-invalid config can hit invalid-config handling before the newer-version warning runs.

## Solution Assessment

Unclear. Warning-copy-only helps, but the narrow maintainable fix likely needs invalid-load/snapshot warning coverage plus an explicit doctor policy for known value-format downgrade repairs.
