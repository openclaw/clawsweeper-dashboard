Keep this issue open. Current main still shows partial downgrade protection rather than the full requested repair path, and open PR #72535 is the active closing implementation candidate, so cleanup should wait for that PR to be reviewed, expanded, or replaced.

## What This Changes

The issue reports that downgrading from OpenClaw 2026.4.x to 2026.3.2 can leave newer `openclaw.json` keys and value formats invalid for the older binary without clear warning or repair guidance.

## Best Possible Solution

Review and land, expand, or replace PR #72535 so newer-version guidance is visible even when config validation fails, doctor either safely rewrites or clearly enumerates downgrade-incompatible value formats, and all automated config rewrites continue using structured serialization.

## Reproduction Assessment

Yes. The report gives concrete version steps and doctor errors, and current-main source inspection shows invalid downgraded config values reach invalid-config handling before future-version warning logic runs.

## Solution Assessment

Unclear. A warning-copy-only fix is useful but likely incomplete unless it also covers invalid load/snapshot reporting and deliberately defines which downgrade rewrites doctor can safely perform.
