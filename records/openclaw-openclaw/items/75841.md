## Summary

Keep open. Current main still lacks the requested host-owned run-budget config and structured harness termination path; existing safeguards cover opt-in loop detection, unknown-tool protection, wall-clock timeouts, or CLI output size, and related open work is intentionally narrower.

## What This Changes

Request adds configurable harness-level `maxTurnsPerRun` and `retryBudget` defaults/overrides with structured termination when a run exceeds turn or error budgets.

## Best Possible Solution

Define a runtime-owned run-budget policy, add schema/types/docs for default and per-agent inheritance, enforce it before further model or tool continuation, and emit a structured trajectory termination event across supported harnesses.
