## Summary

Keep open. Current main does not implement the requested harness-level `maxTurnsPerRun` or `retryBudget`; the closest existing feature is optional tool-loop detection, but it is disabled by default, pattern/no-progress based, and does not provide the requested hard per-dispatch termination or `harness_terminate` trajectory entry.

## What This Changes

Add configurable harness-level per-run turn caps and retry budgets, with per-agent overrides and trajectory/error reporting when a cap fires.

## Best Possible Solution

Define a host-owned run-budget contract in the agent runtime layer, resolve it from documented global and per-agent config before harness execution, enforce it before subsequent model/tool cycles, record a structured trajectory termination event, and make plugin/native harness behavior explicit in the SDK/docs.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the requested config keys and trajectory event are absent, and the only nearby guard is opt-in pattern-based loop detection rather than a hard dispatch budget.

## Solution Assessment

Unclear. Harness-level enforcement is the right class of solution, but the exact config placement and cross-runtime API coverage should be approved before implementation so a PI-only fix does not create misleading safety semantics.
