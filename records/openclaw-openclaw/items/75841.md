## Summary

Keep open. Current main still lacks host-owned `maxTurnsPerRun` / `retryBudget` config and the requested terminal trajectory event; the closest shipped guards are opt-in tool-loop detection and a narrow Claude CLI JSONL output cap, while the related open PRs cover different, narrower failure modes.

## What This Changes

Add configurable harness-level per-dispatch turn and retry budgets with per-agent overrides, forced termination, trajectory recording, and nonzero exit behavior.

## Best Possible Solution

Define a public host-owned run-budget policy in the runtime/harness contract, enforce it before further model or tool continuation, emit a terminal trajectory event, and document inheritance across supported runtimes.

## Reproduction Assessment

Yes. Static inspection of current main is enough: the requested config keys and termination event are absent, the relevant schemas are strict, and the nearest guard is opt-in pre-tool loop detection.

## Solution Assessment

Unclear. Harness-level enforcement is the right class of fix, but the exact public config location and cross-runtime coverage need maintainer approval before implementation.
