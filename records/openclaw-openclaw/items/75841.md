## Summary

Keep open. Current main still lacks the requested host-owned `maxTurnsPerRun` / `retryBudget` config and `harness_terminate` trajectory event; the closest shipped safeguards are opt-in tool-loop detection and unrelated retry or output caps, while related #75924 is explicitly narrower and does not supersede this per-dispatch budget request.

## What This Changes

Add configurable harness-level max-turn and retry-budget enforcement per dispatch, with structured trajectory termination when a cap fires.

## Best Possible Solution

Define a host-owned run-budget policy in the runtime/harness contract, enforce it before model and tool continuation across supported runtimes, emit structured trajectory termination, and document default/per-agent inheritance.

## Reproduction Assessment

Yes. Static source inspection on current main confirms the requested config keys, termination event, and dispatch-wide budget gate are absent, while the current schemas are strict.

## Solution Assessment

Unclear. Harness-level enforcement is the right class of solution, but the exact public config shape, default values, and runtime boundary need maintainer approval before implementation.
