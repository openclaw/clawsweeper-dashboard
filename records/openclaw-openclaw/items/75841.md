## Summary

Keep open. Current main still has only optional pattern-based tool-loop detection and no config-driven host-owned `maxTurnsPerRun` or `retryBudget` contract; the related open PRs address compaction anchoring or a narrower error-cascade detector, not this requested hard run budget.

## What This Changes

Add configurable harness-level turn and retry budgets that terminate a dispatch, write a structured trajectory event, and allow per-agent overrides.

## Best Possible Solution

Define a host-owned run-budget policy in the runtime/harness plan, enforce it before model and tool continuation, record a structured trajectory termination, and document exactly which runtimes inherit the guarantee.

## Reproduction Assessment

Yes. Static inspection on current main is enough: the requested config keys and trajectory event are absent, and the runner/hook paths show only opt-in pattern detection rather than a hard per-dispatch budget.

## Solution Assessment

Unclear. Harness-level enforcement is the right class of solution, but the exact public config placement and cross-runtime API coverage need maintainer approval before implementation.
