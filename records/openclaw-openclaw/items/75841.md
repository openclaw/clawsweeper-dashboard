## Summary

Keep open. Current main still lacks host-owned `maxTurnsPerRun` / `retryBudget` config and the requested `harness_terminate` trajectory event; the closest guardrails are opt-in or narrower tool-loop/output protections rather than a per-dispatch harness budget.

## What This Changes

Request adds config-driven harness-level turn and retry caps with per-agent overrides and structured termination when a dispatch exceeds them.

## Best Possible Solution

Define a host-owned run-budget policy in the runtime contract, enforce it before model/tool continuation across supported harnesses, emit a structured trajectory termination, and document inheritance/overrides.

## Reproduction Assessment

Yes. Static verification against current main is enough: the requested config keys and termination event are absent, and the nearest in-tree guards are opt-in or narrower than the reported harness-level cap.

## Solution Assessment

Unclear. Harness-owned enforcement is the right class of fix, but the public config shape and cross-runtime boundary should be settled by maintainers before implementation.
