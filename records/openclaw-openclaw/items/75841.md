## Summary

Keep open. Current main still lacks the requested host-owned `maxTurnsPerRun` / `retryBudget` contract and `harness_terminate` trajectory event; the closest shipped behavior is opt-in pattern-based tool-loop detection plus a narrow unknown-tool safety net, and related open work explicitly does not implement a hard per-dispatch run budget.

## What This Changes

The issue requests configurable harness-level turn and retry budgets with per-agent overrides, enforced outside the agent prompt/tool environment and recorded in trajectory output.

## Best Possible Solution

Define a host-owned run-budget policy in the runtime/harness contract, validate and document it, enforce it before model and tool continuation, and emit a structured trajectory termination across every runtime that claims the guarantee.

## Reproduction Assessment

Yes. Static inspection on current main is enough: the requested config keys and termination event are absent, and the current guard path is pattern-based rather than a hard per-dispatch budget.

## Solution Assessment

Unclear. Harness-level enforcement is the right direction, but the exact public config placement and cross-runtime API contract need maintainer approval before a narrow implementation can be safely attempted.
