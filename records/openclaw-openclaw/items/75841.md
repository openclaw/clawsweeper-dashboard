## Summary

Keep open. Current main still lacks host-owned `maxTurnsPerRun` / `retryBudget` config and the requested trajectory termination event; the existing guardrail is opt-in pattern-based tool-loop blocking, while the related open items cover narrower compaction anchoring or error-cascade detection rather than a hard per-dispatch budget.

## What This Changes

The issue requests configurable harness-level turn and retry budgets with per-agent overrides and structured trajectory termination when a cap fires.

## Best Possible Solution

Define a host-owned run-budget policy in the runtime/harness contract, enforce it before model and tool continuation, emit a structured trajectory termination, and document inheritance across supported runtimes.

## Reproduction Assessment

Yes. Static inspection of current main is enough: the requested keys and event are absent, the schema is strict, and the existing loop guard is disabled by default and pre-tool only.

## Solution Assessment

Unclear. Harness-level enforcement is the right class of fix, but the exact public config placement and cross-runtime API coverage need maintainer approval before implementation.
