## Summary

Keep open. Current main still lacks the requested host-owned run-budget config and structured harness termination path; existing loop guards, timeouts, and CLI output limits are narrower surfaces, and related #75923/#75924 explicitly do not cover per-turn caps or retry budgets.

## What This Changes

The issue requests configurable harness-level `maxTurnsPerRun` and `retryBudget` limits, with defaults/per-agent overrides, that stop runaway tool loops and emit a trajectory termination event.

## Best Possible Solution

Define a documented host-owned run-budget policy in the shared harness/runtime contract, enforce it before further model or tool continuation across supported runtimes, and emit a structured trajectory termination result with clear per-agent inheritance.

## Reproduction Assessment

Yes. Source inspection of current main confirms the proposed config keys and termination event are absent, the relevant schemas are strict, and the main harness/embedded prompt paths do not enforce a run-budget gate.

## Solution Assessment

Unclear. Harness-level enforcement is the right class of solution, but the exact config contract, default limits, exit/result behavior, and cross-runtime coverage need maintainer design before implementation.
