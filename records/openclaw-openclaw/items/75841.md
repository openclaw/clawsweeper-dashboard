## Summary

Keep open. Current main still lacks the requested host-owned run-budget config and structured harness termination path; the closest safeguards are opt-in tool-loop detectors and Claude CLI JSONL output limits, and the related open work is intentionally narrower.

## What This Changes

Request adds configurable harness-owned `maxTurnsPerRun` and `retryBudget` limits with per-agent overrides, trajectory termination events, and non-zero dispatch failure when a cap fires.

## Best Possible Solution

Define a host-owned run-budget policy in the runtime/harness contract, enforce it before further model or tool continuation, emit structured trajectory termination, and document inheritance across supported runtimes.

## Reproduction Assessment

Yes. Source inspection of current main gives a high-confidence reproduction: the requested config keys and termination event are absent, the relevant schemas are strict, and the embedded prompt path lacks a run-budget gate.

## Solution Assessment

Unclear. Harness-level enforcement is the right class of solution, but the exact config placement, defaults, exit behavior, and cross-runtime coverage need maintainer design before implementation.
