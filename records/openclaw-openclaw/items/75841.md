## Summary

Keep open. Current main still lacks the requested host-owned turn-cap/retry-budget config and structured harness termination path; existing safeguards are narrower loop detectors and Claude JSONL output limits, while related #75924 explicitly excludes this broader contract.

## What This Changes

Requests new `openclaw.json` agent run-budget fields for harness-level `maxTurnsPerRun` and per-error/per-dispatch retry limits.

## Best Possible Solution

Define a host-owned run-budget policy in the agent runtime/harness contract, enforce it before further model or tool continuation, emit structured trajectory termination, and document inheritance across supported runtimes.

## Reproduction Assessment

Yes for the feature gap: source inspection of current main shows the requested config keys and termination event are absent, and the strict schemas would reject the proposed fields. I did not run a live looping agent because this review is read-only and `pnpm` is unavailable in the checkout.

## Solution Assessment

Unclear. Harness-level enforcement is the right class of solution, but the public config placement, defaults, non-zero exit semantics, and cross-runtime behavior need maintainer design before a safe implementation.
