## Summary

Keep open. Current main still lacks the requested host-owned `maxTurnsPerRun` / `retryBudget` config and `harness_terminate` trajectory event; the closest safeguards are opt-in tool-loop detection and CLI-output limits, while related #75924 is explicitly narrower and does not supersede this per-dispatch budget request.

## What This Changes

Add configurable harness-level max-turn and retry-budget enforcement per dispatch, with structured trajectory termination when a cap fires.

## Best Possible Solution

Define a host-owned run-budget policy in the runtime/harness contract, enforce it before model/tool continuation across supported runtimes, emit a structured trajectory termination event, and document default/per-agent inheritance.

## Reproduction Assessment

Yes. Static inspection of current main is enough: the requested config keys and trajectory event are absent, the relevant schemas are strict, and the current prompt/tool paths do not contain a dispatch-wide budget gate.

## Solution Assessment

Unclear. Harness-level enforcement is the right class of fix, but the exact config surface, default values, and runtime boundary need maintainer approval before a safe implementation PR.
