## Summary

Keep open. Current main still lacks the requested host-owned `maxTurnsPerRun` / `retryBudget` config and `harness_terminate` trajectory event; the nearest safeguards are narrower loop-detection and CLI-output guards, and the related open work does not supersede the requested per-dispatch budget.

## What This Changes

Add configurable harness-level per-run turn and retry budgets with default and per-agent overrides plus structured termination reporting.

## Best Possible Solution

Define a host-owned run-budget policy in the runtime/harness contract, enforce it before model and tool continuation, emit a structured trajectory termination, and document inheritance across supported runtimes.

Yes. Static inspection of current main is enough: the requested config keys and termination event are absent, the relevant schemas are strict, and current loop detection is opt-in and pre-tool-call scoped.

## Solution Assessment

Unclear. Harness-level enforcement is the right class of fix, but the exact public config placement and cross-runtime API coverage need maintainer approval before implementation.
