## Summary

Keep open. Current main still lacks the requested host-owned run-budget config and structured harness termination path; existing tool-loop detection and Claude live output caps are narrower safeguards, and related #75923/#75924 explicitly exclude per-turn caps and retry budgets.

## What This Changes

Request adding `maxTurnsPerRun` and `retryBudget` config with per-agent overrides, enforced by the harness to terminate runaway tool-call dispatches and record a `harness_terminate` trajectory event.

## Best Possible Solution

Define a host-owned run-budget policy shared across supported harnesses, enforce it before further model/tool continuation, emit structured termination metadata, and document default/per-agent inheritance.
