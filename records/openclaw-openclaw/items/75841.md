## Summary

Keep open. Current main still lacks the requested host-owned run-budget config, structured `harness_terminate` trajectory event, and runtime contract coverage for both PI-owned and plugin-owned harness loops; existing tool-loop safeguards are opt-in/narrower and the related open work explicitly excludes this per-run cap.

## What This Changes

Add config-driven harness-level limits for per-run tool-calling turns and retry budgets, with structured termination events and per-agent overrides.

## Best Possible Solution

Define a public run-budget policy in the agent runtime/harness contract, enforce it before further model or tool continuation across supported runtimes, emit documented trajectory/failure results, and document default/per-agent inheritance.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence evidence that the requested config keys, termination event, and host-owned run-budget gate are absent on current main.

## Solution Assessment

Unclear. Harness-owned enforcement is the right class of solution, but the exact public config shape, defaults, non-zero result semantics, and cross-runtime coverage need maintainer design before implementation.
