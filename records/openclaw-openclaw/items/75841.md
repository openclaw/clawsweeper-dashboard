## Summary

Keep open. Current main still lacks the requested host-owned run-budget config and structured harness termination, while the existing safeguards are opt-in loop detectors or backend output caps that do not terminate a dispatch after a total turn or retry budget.

## What This Changes

Request adding host-owned `maxTurnsPerRun` and `retryBudget` config with per-agent overrides, enforced by the agent harness with structured termination.

## Best Possible Solution

Define a core runtime/harness run-budget policy, enforce it before further model or tool continuation across supported harnesses, emit a structured trajectory termination event, and document default/per-agent inheritance.

## Reproduction Assessment

Yes for the absence of the requested capability: source inspection of current main shows the proposed config keys, termination event, and dispatch-level budget gate are not present. This is a feature request rather than a broken documented behavior.

## Solution Assessment

Unclear. Harness-level enforcement is the right class of solution, but the exact public config contract, defaults, runtime coverage, and termination semantics need maintainer design before a safe implementation PR.
