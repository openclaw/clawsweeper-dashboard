## Summary

Keep open. Current main has useful adjacent work in deterministic group session keys, dashboard-created sessions, delete-transcript archival, compaction restore, and memory search, but it still does not implement the RFC's first-class multi-session architecture, archive/restore/pin/tag lifecycle, or unified public knowledge-base contract.

## What This Changes

The issue proposes shared model inference, isolated channel/task sessions, shared public knowledge/RAG, and WebUI session lifecycle controls.

## Best Possible Solution

Keep this as a maintainer-owned architecture thread or split it into reviewable slices: canonical session identity, lifecycle storage/API, Control UI lifecycle actions, and shared memory/RAG behavior.
