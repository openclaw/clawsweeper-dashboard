## Summary

Keep open. Current main has adjacent routing isolation, Google Chat group route-key handling, session create/delete/compaction UI, and memory retrieval work, but the RFC's first-class archive/restore/pin/tag lifecycle, shared public knowledge/RAG contract, and full multi-session product policy are still not implemented and need maintainer scoping.

## What This Changes

The issue requests a multi-session architecture with shared LLM inference, isolated per-channel/task sessions, shared public knowledge/RAG, new session lifecycle APIs, and WebUI create/archive/restore/pin management.

## Best Possible Solution

Keep this as the maintainer-owned architecture thread or split it into reviewable slices for canonical session identity, lifecycle storage/API, Control UI actions, and shared memory/RAG policy.

Not applicable: this is a product and architecture RFC, not a single executable bug. Source inspection gives high confidence that current main lacks the requested lifecycle API/UI and shared public knowledge contract, while the narrow Google Chat group-key follow-up appears source-covered.
