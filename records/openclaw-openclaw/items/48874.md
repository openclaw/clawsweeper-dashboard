## Summary

Keep open: current main has adjacent routing, session CRUD, transcript archive files, compaction checkpoints, UI filtering, and memory/wiki pieces, but the RFC's first-class archive/restore/pin lifecycle, canonical session identity policy, WebUI management model, and shared knowledge privacy rules remain unresolved.

## What This Changes

Define a multi-session architecture with shared model inference, isolated per-channel/task sessions, WebUI create/archive/restore/pin management, and a shared public knowledge/RAG layer.

## Best Possible Solution

Keep this as the canonical maintainer design thread and split future work into approved tracks for session identity/routing, archive/restore UI and API, and memory visibility/security defaults.
