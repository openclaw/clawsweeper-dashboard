## Summary

Keep open: current main has useful adjacent pieces for scoped routing, session creation, transcript archiving, and memory lookup, but the RFC's central first-class session lifecycle, configurable main-key routing, archive/restore/pin model, all-channel binding policy, and shared-memory privacy boundary are still unresolved maintainer product/security work.

## What This Changes

The issue requests a multi-session architecture with isolated channel sessions, shared model/memory layers, canonical session keys, archive/restore/pin lifecycle, and WebUI management.

## Best Possible Solution

Keep this as a maintainer RFC and split sponsored work into narrow designs for main-key/session binding, archive/restore/pin storage and migration, WebUI lifecycle controls, and memory/RAG privacy boundaries.
