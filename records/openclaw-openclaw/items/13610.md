## Summary

Keep open: current main and v2026.5.20 still provide the shipped SecretRef baseline for env/file/exec only, while native AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager support remains unresolved and security-sensitive.

## What This Changes

Request native external secrets-manager providers for OpenClaw credential resolution, including AWS Secrets Manager, Vault, Azure Key Vault, and GCP Secret Manager.

## Best Possible Solution

Keep this as the native secrets-manager provider umbrella until maintainers approve a security-reviewed SecretRef provider contract, then split AWS, Vault, Azure, GCP, and other backends into focused provider/plugin follow-ups with live backend proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source, docs, release inspection, and live PR state give high confidence that native AWS/Vault/Azure/GCP SecretRef sources are absent and the supported contract remains env/file/exec.

## Solution Assessment

Unclear until maintainers choose the contract. A plugin or ClawHub provider seam aligns with OpenClaw's lean-core architecture, but secret-provider execution is security-sensitive and needs explicit product/secops approval before implementation.
