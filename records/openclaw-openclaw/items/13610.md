## Summary

Keep open: current main and latest release v2026.5.19 still expose the shipped SecretRef baseline for env/file/exec only, while native AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager support remains unresolved; the related provider-seam PR is open but not merged.

## What This Changes

The issue requests native external secret-manager integrations for OpenClaw configuration secrets, including AWS Secrets Manager, Vault, Azure Key Vault, Google Cloud Secret Manager, and formalized environment-variable references.

## Best Possible Solution

Keep this as the native secrets-manager provider umbrella until maintainers approve a security-reviewed provider contract, then split AWS, Vault, Azure, GCP, and other backends into focused provider/plugin follow-ups with redacted live backend proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source, docs, live PR state, and latest-release inspection give high confidence that native AWS/Vault/Azure/GCP SecretRef sources are absent and the supported contract remains env/file/exec.

## Solution Assessment

Unclear until maintainers choose the contract. A plugin or ClawHub provider seam aligns with OpenClaw's lean-core architecture, but secret-provider execution is security-sensitive and needs explicit product/secops approval before implementation.

## Risks / Open Questions

- There are multiple open implementation directions, so maintainers still need to choose whether native vendors land as first-party providers, plugin-owned SecretRef sources, bundled resolver plugins, or ClawHub-published providers.
- Plugin-owned or vendor-native secret resolution changes activation, source ownership, dependency policy, redaction, caching, disablement, audit, and failure semantics for credential materialization.
- Exec providers can wrap Vault, cloud CLIs, or custom resolvers today, but that is not equivalent to native provider source types, provider-specific schema, resolver dispatch, CLI UX, backend batching, or live vendor tests.
