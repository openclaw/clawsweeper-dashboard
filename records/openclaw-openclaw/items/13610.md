## Summary

Keep open: current main has the shipped SecretRef baseline and documented exec-wrapper path, but native AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager support still requires a security-reviewed provider/API decision.

## What This Changes

Request native external secrets-manager integrations and config syntax for AWS Secrets Manager, Vault, Azure Key Vault, Google Cloud Secret Manager, and formalized environment-variable secrets.

## Best Possible Solution

Keep this as the native secrets-manager provider umbrella until maintainers approve a security-reviewed SecretRef or credential-provider contract, then split AWS, Vault, Azure, GCP, and similar backends into focused provider/plugin follow-ups with live backend proof.
