## Summary

Keep open. Current main and the latest shipped release still provide only the env/file/exec SecretRef baseline, while the requested native AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager support remains unresolved and maintainer discussion explicitly kept this as the umbrella for first-class provider work.

## What This Changes

Add native external secrets-manager integrations for AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, Google Cloud Secret Manager, and formalized environment-variable references.

## Best Possible Solution

Keep this as the native secret-provider umbrella until maintainers choose a reviewed plugin SDK seam for external providers or explicitly decide to own AWS/Vault/Azure/GCP providers in core with aligned schemas, resolver, CLI, docs, tests, and live provider proof.

## Reproduction Assessment

Not applicable as a feature/security architecture request. The high-confidence verification path is current-main and v2026.5.7 source/docs inspection showing SecretRef sources and providers are still env, file, and exec only.

## Solution Assessment

Unclear pending maintainer and secops direction. Native provider support is a reasonable goal, but the maintainable route should be decided between a plugin-provider seam like https://github.com/openclaw/openclaw/pull/72548 and explicit core-owned providers before implementation.
