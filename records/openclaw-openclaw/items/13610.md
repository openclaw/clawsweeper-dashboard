## Summary

Keep open. Current main and the latest release still ship only the env/file/exec SecretRef baseline, native cloud/Vault providers are still absent, and https://github.com/openclaw/openclaw/pull/72548 is only an open proposed plugin seam.

## What This Changes

Add native external secrets-manager support for AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, Google Cloud Secret Manager, and formalized environment-variable secret references.

## Best Possible Solution

Keep this as the native secrets-manager umbrella until maintainers choose and land either a reviewed plugin-provider API seam or explicit first-class core providers with schema, resolver, CLI, docs, audit, caching, reload, and tests aligned.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. The high-confidence verification path is current-main source and docs inspection showing SecretRef sources, provider config, schema validation, resolver dispatch, and gateway wire schema still limited to env/file/exec.

## Solution Assessment

Unclear pending maintainer and secops direction. The maintainable path is likely to review a generic plugin-provider seam such as https://github.com/openclaw/openclaw/pull/72548, then ship AWS/Vault/Azure/GCP implementations as reviewed plugins or explicitly accept them as core-owned providers.
