## Summary

Current main now provides the generic integration boundary that this request needed: plugins can publish manifest-owned SecretRef resolver presets, and that path has been validated with a real HashiCorp Vault plugin. The remaining AWS, Azure, GCP, and additional Vault implementations are optional vendor integrations that belong in plugins and ClawHub rather than as native core providers.

## What This Changes

Add first-class AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, Google Cloud Secret Manager, and environment-variable secret integrations with caching, rotation-friendly loading, graceful errors, and audit logging.

## Best Possible Solution

Keep the generic SecretRef transport and manifest integration contract in core, and publish AWS, Vault, Azure, and GCP implementations as independently maintained plugins discoverable through ClawHub.
