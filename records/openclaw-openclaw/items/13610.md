## Summary

Keep open because current main and the latest release still only support the shipped SecretRef baseline for env, file, and exec providers; native AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager support remains unresolved security-sensitive provider work.

## What This Changes

The issue requests native integration with external secrets managers such as AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, Google Cloud Secret Manager, plus formalized environment-variable secret references.

## Best Possible Solution

Keep this as the native secrets-provider umbrella until maintainers either land a reviewed plugin SDK seam for vendor providers or explicitly decide to own AWS/Vault/Azure/GCP providers in core with aligned schemas, CLI, docs, tests, and live proof.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is current-main and v2026.5.7 source/docs inspection, which shows supported SecretRef/provider sources remain env, file, and exec only.

## Solution Assessment

Unclear pending maintainer and secops direction. Native provider support is a reasonable goal, but the maintainable path should be decided between a plugin-provider seam and explicit core-owned providers before implementation.
