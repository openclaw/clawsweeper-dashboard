## Summary

Keep open: current main and the latest shipped release still support the shipped SecretRef baseline for env/file/exec only, while native AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager support remains unresolved and security-sensitive.

## What This Changes

Request native external secrets manager support so OpenClaw config can resolve credentials from AWS Secrets Manager, Vault, Azure Key Vault, Google Cloud Secret Manager, and formalized environment-variable references.

## Best Possible Solution

Keep this issue as the native secrets-manager umbrella until maintainers approve a security-reviewed provider/plugin contract, then split AWS, Vault, Azure, GCP, and other backends into focused provider follow-ups with redacted live backend proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection gives a high-confidence current-main and v2026.5.12 check that native cloud/Vault SecretRef sources are absent and the supported contract remains env/file/exec.

## Solution Assessment

Unclear pending maintainer direction. A plugin or ClawHub provider seam appears more aligned with OpenClaw's plugin-agnostic architecture than bundling every vendor directly in core, but the public contract and security boundaries still need approval.
