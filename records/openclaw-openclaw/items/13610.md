## Summary

Keep open: the shipped SecretRef baseline covers env/file/exec, but current main still does not include native AWS/Vault/Azure/GCP provider sources or the pluggable provider seam. The issue remains a useful umbrella because the related provider-seam work is still open and needs security/product review.

## What This Changes

Add native external secrets-manager integrations for OpenClaw configuration secrets, including AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, Google Cloud Secret Manager, and formalized environment-variable refs.

## Best Possible Solution

Keep this as the native secrets-manager umbrella while maintainers review a security-audited provider plugin seam such as https://github.com/openclaw/openclaw/pull/72548, then move vendor providers to plugins/ClawHub unless maintainers explicitly choose first-class core providers.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection gives a high-confidence current-main check that SecretRef provider sources remain env/file/exec only.

## Solution Assessment

Unclear until maintainers choose the contract. A plugin-provider seam appears more aligned with the repo’s plugin architecture than adding one-off native core providers, but the secret-resolution boundary needs explicit security review.
