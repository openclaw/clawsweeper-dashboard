## Summary

Keep open: current main and latest release v2026.5.19 still expose the shipped SecretRef baseline for env/file/exec only, while native AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager support remains unresolved and security-sensitive. The open pluggable-provider PR is directly relevant, but it is not merged and does not close this umbrella yet.

## What This Changes

The issue requests native external secrets-manager support with provider-specific config syntax, lazy/cached resolution, graceful failure, rotation support, and audit logging for AWS, Vault, Azure, GCP, and formalized env references.

## Best Possible Solution

Keep this issue as the native secrets-manager provider umbrella until maintainers approve a security-reviewed provider/plugin contract, then split AWS, Vault, Azure, GCP, and other backends into focused provider follow-ups with redacted live backend proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source and latest-release inspection give high confidence that native AWS/Vault/Azure/GCP SecretRef sources are absent and the supported contract remains env/file/exec.

## Solution Assessment

Unclear pending maintainer direction. A plugin or ClawHub provider seam aligns better with OpenClaw's plugin-lean architecture than bundling every vendor directly in core, but the public contract and security boundaries still need approval.
