## Summary

Keep open. Current main still ships the env/file/exec SecretRef baseline, while native AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager provider sources remain unimplemented; the issue discussion also explicitly keeps this as the umbrella beyond the shipped baseline.

## What This Changes

Request native external secrets-manager provider support for AWS, Vault, Azure, GCP, and formalized env references in OpenClaw configuration.

## Best Possible Solution

Keep this as the native secrets-manager provider umbrella until maintainers choose a reviewed plugin SDK seam for external providers or explicitly decide to own AWS/Vault/Azure/GCP providers in core with aligned schemas, resolver, CLI, docs, and tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is current-main source and docs inspection showing SecretRef sources, provider config, schema validation, resolver dispatch, and gateway wire schema still limited to env/file/exec.

## Solution Assessment

Unclear pending maintainer and secops direction. Native provider support is reasonable, but the maintainable path should be decided between a plugin-provider seam like https://github.com/openclaw/openclaw/pull/72548 and explicit core-owned providers before implementation.
