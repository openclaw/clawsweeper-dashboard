## Summary

Keep open: current main has the shipped SecretRef baseline for env/file/exec, but it still lacks native AWS Secrets Manager, Vault, Azure Key Vault, and Google Cloud Secret Manager sources, and the related plugin/provider contract work is still open.

## What This Changes

Request to add native external secrets-manager integration and config syntax for providers such as AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, Google Cloud Secret Manager, and env refs.

## Best Possible Solution

Keep this as the native secrets-manager umbrella while maintainers choose and land a security-reviewed provider extension strategy, preferably a plugin/API seam plus focused provider implementations or explicit first-class core providers.

## Reproduction Assessment

Not applicable as a bug reproduction; this is feature/API work. Source inspection gives a high-confidence current-main check that the accepted SecretRef sources and provider schemas remain env/file/exec-only.

## Solution Assessment

Unclear until maintainers choose the public contract. The existing exec-provider baseline is a useful workaround, but the requested native/provider-owned AWS/Vault/Azure/GCP surface needs a security-reviewed product and API decision.
