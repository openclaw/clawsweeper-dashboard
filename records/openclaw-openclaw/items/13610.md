## Summary

Close as ClawHub/plugin work: current main has the generic SecretRef baseline plus a plugin-managed exec provider integration seam, so AWS/Vault/Azure/GCP implementations no longer need to stay as an OpenClaw core-native umbrella unless a narrower missing core API is found.

## What This Changes

The issue requests native external secrets-manager integrations such as AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, Google Cloud Secret Manager, and formalized env references.

## Best Possible Solution

Keep the generic SecretRef and plugin-integration seam in core, and publish provider-specific AWS, Vault, Azure, GCP, 1Password, or Bitwarden integrations through ClawHub/plugin repositories unless a narrower core API gap is demonstrated.

## Solution Assessment
