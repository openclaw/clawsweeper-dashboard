## Summary

Keep open. Current main has the shipped env/file/exec SecretRef baseline, but native AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager sources are still not implemented, and the discussion explicitly keeps this as the umbrella for first-class provider work beyond that baseline.

## What This Changes

Request native integration with external secrets managers such as AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager for OpenClaw configuration secrets.

## Best Possible Solution

Keep this as the native secrets-manager provider umbrella until maintainers choose a reviewed plugin SDK seam or explicitly decide to own AWS/Vault/Azure/GCP providers in core.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is current-main source and docs inspection showing SecretRef sources, provider config, schema validation, resolver dispatch, and gateway wire schema still limited to env/file/exec.

## Solution Assessment

Unclear pending maintainer and secops direction. The maintainable path is likely a plugin-provider seam like https://github.com/openclaw/openclaw/pull/72548 followed by provider plugins/ClawHub, unless maintainers explicitly choose core-owned native providers.
