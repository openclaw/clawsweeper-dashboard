## Summary

Keep open. Current main still ships only the env/file/exec SecretRef baseline, while native AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager support remains unimplemented and the related plugin-provider seam is still an open maintainer/security decision.

## What This Changes

Request native integration with external secrets managers such as AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager for OpenClaw secrets resolution.

## Best Possible Solution

Keep this as the native secrets-provider umbrella until maintainers choose either a reviewed plugin SDK provider seam with provider implementations outside core or explicit core-owned AWS/Vault/Azure/GCP providers with aligned contracts, docs, and tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is current-main source and docs inspection, which shows supported SecretRef/provider sources remain env, file, and exec only.

## Solution Assessment

Unclear pending maintainer and secops direction. Native vendor-provider support is a reasonable goal, but the maintainable path is either a reviewed plugin-provider seam or an explicit decision to own these providers in core.
