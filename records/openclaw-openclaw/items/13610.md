## Summary

Keep open. Current main still supports only env/file/exec SecretRef providers, while native AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager support remains unresolved security-sensitive provider architecture work.

## What This Changes

Request native external secrets-manager support for OpenClaw configuration, including AWS Secrets Manager, Vault, Azure Key Vault, Google Cloud Secret Manager, caching, fallback behavior, and audit logging.

## Best Possible Solution

Keep this as the native secrets-provider umbrella until maintainers choose a reviewed plugin SDK seam for external providers or explicitly decide to own AWS/Vault/Azure/GCP providers in core with aligned schemas, CLI, docs, and tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is current-main source and docs inspection, which shows supported SecretRef/provider sources remain env, file, and exec only.

## Solution Assessment

Unclear pending maintainer and secops direction. Native provider support is a reasonable goal, but the maintainable path should be decided between a plugin-provider seam and explicit core-owned providers before implementation.
