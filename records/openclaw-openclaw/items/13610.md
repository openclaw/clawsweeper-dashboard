## Summary

Keep open: current main provides the SecretRef foundation and trusted exec-plugin presets, but still lacks first-class AWS, Vault, Azure, and GCP integrations; the permanent provider trust and extension model also remains an active maintainer decision.

## What This Changes

Add native integrations for AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager, including resolution, caching, rotation, and audit behavior.

## Best Possible Solution

Choose and document one generic, security-reviewed provider boundary, then implement AWS, Vault, Azure, and GCP as separately maintained plugins through that boundary while keeping core limited to shared SecretRef lifecycle and contracts.

## Reproduction Assessment

Not applicable: this requests new vendor-provider capabilities rather than reporting broken established behavior.

## Solution Assessment

No, adding four one-off native core integrations is not the best maintainable default; the preferred path is a narrow generic plugin contract with explicit trust and lifecycle rules, followed by vendor plugins.
