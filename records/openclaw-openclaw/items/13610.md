## Summary

Keep open. Current main and v2026.5.7 ship the env/file/exec SecretRef baseline, but the requested native AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager integrations remain unresolved, and a maintainer comment explicitly kept this as the umbrella after https://github.com/openclaw/openclaw/pull/26155.

## What This Changes

Request native external secrets-manager providers for AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, Google Cloud Secret Manager, and formal environment-variable references in OpenClaw config.

## Best Possible Solution

Keep this as the native secret-provider umbrella until maintainers choose a reviewed plugin SDK seam for external providers or explicitly decide to own AWS/Vault/Azure/GCP providers in core with aligned schemas, resolver, CLI, docs, and tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is current-main and v2026.5.7 source/docs inspection showing SecretRef sources and providers are still env, file, and exec only.

## Solution Assessment

Unclear pending maintainer and secops direction. Native provider support is a reasonable goal, but the maintainable path should be decided between a plugin-provider seam like https://github.com/openclaw/openclaw/pull/72548 and explicit core-owned providers before implementation.
