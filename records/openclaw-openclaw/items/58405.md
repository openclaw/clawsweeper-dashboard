## Best Possible Solution

Define request `skills` as an ephemeral narrowing selector over the selected agent's effective configured allowlist, reject or ignore outside names by contract, then prove allowed, outside, empty, and omitted cases with tests and redacted live gateway output.

## Reproduction Assessment

## Solution Assessment

No. The feature direction is useful, but replacing the configured final allowlist is not the safest contract; the request field should narrow or reject names outside the effective agent allowlist unless maintainers explicitly approve wider semantics.
