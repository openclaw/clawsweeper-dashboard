## Summary

Keep open; current main still lacks the requested `/v1/responses` `skills` field, but this PR should not merge as-is because the request override can widen configured agent skill allowlists and the user-facing feature lacks the required changelog entry.

## What This Changes

The PR adds an optional OpenClaw-specific `skills` array to `/v1/responses`, threads it into agent command execution as an ephemeral skill snapshot override, and documents the request field.

## Best Possible Solution

Implement `skills` as an ephemeral narrowing selector by default: rebuild and avoid persisting the run snapshot, but intersect with or reject names outside the effective configured agent allowlist, then add a changelog entry and targeted tests.

## Reproduction Assessment

Yes. Configure an agent with `agents.list[].skills: ["safe"]`, keep another workspace skill available, and send `/v1/responses` with `skills` naming the other skill; the PR path uses the request filter as the full snapshot filter.

## Solution Assessment

No. The feature shape is reasonable, but replacing the configured allowlist is not the narrowest maintainable contract; the request field should be constrained by the effective agent allowlist unless maintainers explicitly approve elevation semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Keep request skills within the agent allowlist:** `src/agents/agent-command.ts:505-507`
  - body: When `skills` is present, this path uses the request body as the whole skill filter, allowing a caller to select skills outside `agents.list[].skills`. Since the documented agent allowlist is the final set and snapshot membership drives skill env injection, intersect with the effective agent filter or reject names outside it.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `src/gateway/open-responses.schema.ts:218`
  - body: This is a user-facing `/v1/responses` feature touching runtime code, but the PR does not update `CHANGELOG.md`. Add an active-version changelog entry so the merge gate and release notes stay aligned.
  - confidence: 0.92

## Security Review

Status: needs_attention

Summary: The diff can widen a restricted agent's skill set from the request body, so it needs API/security policy attention before merge.

Concerns:

- **[high] Request skills can bypass agent allowlists:** `src/agents/agent-command.ts:505`

Since skill snapshots feed prompt construction and per-skill env override application, this can expose skills and skill-specific host env config that the selected agent's allowlist was meant to withhold.
