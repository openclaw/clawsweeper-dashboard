## Summary

Keep open. Current main has ordered reactive model fallback, cooldown/probe recovery, and provider usage display surfaces, but no config or routing path that switches models based on token, quota, or cost thresholds; the open pace-aware rate-limit tracker overlaps but does not supersede this broader cross-provider failover request.

## What This Changes

The issue requests configurable token or cost threshold based automatic failover across ordered primary and fallback model providers, with optional reverse failover and notifications.

## Best Possible Solution

Define a core usage-aware routing policy before candidate execution, fed by provider-plugin usage hints and local token/cost accounting where available, while preserving the existing reactive cooldown/error fallback path and documenting config, failback, and notification behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The implementation gap is high-confidence from source and docs: OpenClaw exposes ordered reactive fallback and usage/status surfaces, but no threshold policy participates in model selection.

## Solution Assessment

Unclear until maintainers define the product and config contract. The likely maintainable direction is core routing policy with provider-plugin usage inputs, because core owns generic agent execution/failover while provider plugins own usage snapshot collection.
