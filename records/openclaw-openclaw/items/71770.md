## Summary

Keep open. Current main has ordered, error-driven model fallback plus provider usage visibility, but it still lacks a token, quota, or cost threshold policy that proactively selects fallback models; the related items are adjacent rather than terminal replacements.

## What This Changes

The issue requests configurable token or cost thresholds that automatically switch from a primary model/provider to ordered fallbacks, optionally fail back, and notify users.

## Best Possible Solution

Define a usage-aware model-routing policy that consumes explicit provider quota windows plus local token/cost accounting, preserves current error-driven fallback defaults, and documents config, hysteresis, failback, and notification behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection give high confidence that current main lacks a proactive token/cost threshold model-routing policy while supporting adjacent fallback and usage visibility surfaces.

## Solution Assessment

Unclear until maintainers define the product and config contract. The maintainable path is a core routing policy with explicit usage freshness, accounting, hysteresis, failback, and notification semantics rather than ad hoc checks in the existing fallback loop.
