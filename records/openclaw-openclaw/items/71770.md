## Summary

Keep open. Current main has ordered reactive model fallback, cooldown/probe recovery, provider usage snapshots, and local cost summaries, but no config or model-selection path that switches providers based on token, quota, or cost thresholds; the related pace-aware rate-limit item overlaps but does not supersede this broader routing request.

## What This Changes

Add configurable token or cost threshold based automatic failover across ordered primary and fallback model providers, with optional reverse failover and notifications.

## Best Possible Solution

Define one core usage-aware model-routing policy before candidate execution, fed by provider-plugin usage hints and local token/cost accounting where available, while preserving the existing reactive cooldown/error fallback path and documenting config, failback, and notification behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The implementation gap is high-confidence from current source and docs: OpenClaw exposes ordered reactive fallback and usage/status surfaces, but no threshold policy participates in model selection.

## Solution Assessment

Unclear until maintainers define the product and config contract. The likely maintainable direction is core routing policy with provider-plugin usage inputs, because OpenClaw owns the generic agent loop and failover while provider plugins own usage snapshot collection.
