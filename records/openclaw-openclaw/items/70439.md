## Summary

Keep open. Current main still lacks a Signal audio-preflight transcript path before mention gating, while the shared helper and peer-channel implementations already exist and the issue discussion explicitly kept this gap open after the Signal media-metadata fix.

## What This Changes

Request Signal voice-note preflight transcription, preferably through the existing provider-backed audio pipeline, with documentation for media variables and audio format conversion limits.

## Best Possible Solution

Wire Signal voice-note handling into the existing provider-backed transcribeFirstAudio flow before mention gating, then mark the preflighted media so the main media-understanding pass does not double-transcribe it.

## Reproduction Assessment

Yes, at source level. A Signal group with requireMention can return before attachments are fetched, and the current Signal mention-gating test still asserts skipped audio is recorded as <media:audio> without transcript context.

## Solution Assessment

Yes for the provider-backed path. Reusing transcribeFirstAudio inside Signal is the narrow maintainable fix; a new arbitrary shell-command hook would be broader and security-sensitive.
