## Summary

Keep open. Current main and the latest shipped release still lack the Signal audio-preflight path requested here, even though related Signal MIME/context fixes have already landed.

## What This Changes

Request a Signal/channel audio-preflight hook so Signal voice notes can be transcribed before the agent sees `<media:audio>`.

## Best Possible Solution

Add a Signal-owned provider-backed audio preflight that reuses `transcribeFirstAudio` before mention gating, carries transcript state downstream, and defers arbitrary shell-command hooks.

## Reproduction Assessment

Yes, at source level. A Signal group with `requireMention` and an audio-only message reaches the no-mention return before attachments are fetched or transcribed, and the current test suite asserts the skipped `<media:audio>` pending-history behavior.

## Solution Assessment

Yes for the narrow provider-backed path. Reusing `transcribeFirstAudio` and the existing `tools.media.audio` contract is maintainable; a Signal-specific shell-command hook is broader and security-sensitive.

## Repair Work Prompt

Fix the Signal audio-preflight gap tracked by https://github.com/openclaw/openclaw/issues/70439. Add a Signal-owned provider-backed preflight using the existing `transcribeFirstAudio` helper after Signal sender/group access checks but before mention gating can skip audio-only group messages. Fetch only the needed audio attachments for preflight, use the transcript when evaluating mention patterns, thread `Transcript` and `MediaTranscribedIndexes` into the finalized inbound context, and avoid double transcription downstream. Do not add `channels.signal.audio.preflight_cmd` or any arbitrary shell-command config surface. Preserve existing media MIME fallback behavior and owner-boundary rules for extension code. Add or update focused Signal regression tests for audio-only mention-gated groups, inbound context transcript/index state, and the existing skipped-placeholder behavior where preflight is unavailable. Add a changelog entry because this changes user-visible Signal voice-note behavior.
