## Summary

Keep open. Current main still lacks a Signal audio-preflight transcript path, and source shows mention-gated Signal group audio can be skipped before attachments are fetched while peer channels already use provider-backed preflight transcription.

## What This Changes

Request to add Signal audio preflight support so inbound voice notes are transcribed before the agent sees the `<media:audio>` placeholder.

## Best Possible Solution

Add a Signal-owned provider-backed audio preflight using the existing `transcribeFirstAudio` and `tools.media.audio` path before mention gating, while deferring arbitrary shell hooks for separate product/security review.

## Reproduction Assessment

Yes, at source level. A Signal group with `requireMention` and an audio-only attachment reaches the no-mention return before attachments are fetched, and the current test suite asserts the skipped `<media:audio>` pending-history behavior.

## Solution Assessment

Yes for the provider-backed subset. Reusing the existing shared preflight helper is the narrow maintainable path; adding a Signal-specific shell command hook would be broader and security-sensitive.
