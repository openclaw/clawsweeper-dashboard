## Summary

Keep open. Current main has Signal audio metadata and general media-understanding support, but Signal still lacks the pre-mention audio preflight path used by other bundled channels, so mention-gated group voice notes can be skipped before transcription.

## What This Changes

Request a Signal/channel audio preflight hook so Signal voice notes can be transcribed before agent handling, matching Telegram, Discord, and WhatsApp behavior.

## Best Possible Solution

Add a Signal-owned provider-backed audio preflight using the existing shared helper before mention gating, then thread the transcript and transcribed media index into the inbound context while deferring any shell-command hook design.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a Signal group with `requireMention` and an audio-only message can hit the no-mention return before attachments are fetched, while the pass-through path forwards media metadata without `Transcript` or `MediaTranscribedIndexes`.

## Solution Assessment

Yes for the narrow provider-backed path. Reusing `transcribeFirstAudio` in the Signal handler is the smallest maintainable fix; adding arbitrary command execution is a broader security/product decision.

## Repair Work Prompt

Fix the Signal voice-note audio preflight gap for this issue. Implement a Signal-owned provider-backed preflight using the existing `transcribeFirstAudio` helper from `openclaw/plugin-sdk/media-runtime`; do not add `channels.signal.audio.preflight_cmd`, arbitrary shell execution, new dependency overrides, or a broad product/config design. The fix should run after Signal access checks and media-size/attachment safeguards but before mention/command gating can skip an audio-only group message, so a voice note transcript can participate in mention detection and agent-facing body construction. Thread the successful transcript into the inbound context (`Body`/`BodyForAgent` as appropriate, `Transcript`, and `MediaTranscribedIndexes` for the first transcribed audio) so downstream media understanding does not double-transcribe. Preserve existing Signal attachment limits, `ignoreAttachments`, access policy, pending-history behavior, and group allowlist semantics. Add focused Signal tests for DM audio preflight, mention-gated group audio preflight, failure fallback to `<media:audio>`, and no double-transcription marker; update docs/changelog if behavior is user-facing.
