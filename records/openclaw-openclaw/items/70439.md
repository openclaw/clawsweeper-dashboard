## Summary

Keep open. Current main and the latest release still lack the Signal audio preflight path used by other bundled channels, so mention-gated Signal voice notes can be dropped before transcription despite the shared audio contract.

## What This Changes

Request to add Signal audio preflight so voice notes are transcribed before they reach the agent, with an optional pluggable command-hook variant.

## Best Possible Solution

Add a Signal-owned provider-backed audio preflight using the existing shared helper before mention gating, then pass transcript and transcribed-media index state downstream while deferring arbitrary shell-command hooks.

## Reproduction Assessment

Yes, source-level: a Signal group with requireMention and an audio-only message reaches the no-mention return before attachments are fetched, while the later inbound context has media metadata but no preflight transcript state. I did not run a live Signal setup in this read-only review.

## Solution Assessment

Yes for the narrow provider-backed path: reusing transcribeFirstAudio in the Signal handler matches Telegram/Discord/WhatsApp and the existing media contract. The shell-command preflight_cmd variant is not the best first fix because it adds command execution and config policy surface.
