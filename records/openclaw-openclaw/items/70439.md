## Summary

Keep open. Current main still has the Signal voice-note preflight gap described by the reporter and the maintainer follow-up: Signal now preserves audio media metadata, but it still does not invoke the shared audio preflight helper or mark a transcript before mention/command handling.

## What This Changes

Request to add Signal voice-note audio preflight so inbound audio can be transcribed before the agent sees it, matching Telegram, Discord, and WhatsApp behavior.

## Best Possible Solution

Implement a Signal-owned provider-backed audio preflight using the existing shared helper after Signal access checks and attachment limits but before mention/command gating, then pass the transcript into body, command, mention handling, and MediaTranscribedIndexes so shared media understanding does not double-transcribe.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a mention-gated Signal group audio-only message can return at the no-mention branch before attachments are fetched, and pass-through Signal messages currently dispatch media metadata without Transcript or MediaTranscribedIndexes.

## Solution Assessment

Yes, the narrow provider-backed Signal hook is the best current fix. A generic shell preflight command would be a larger security/product design and is not needed to close the observable parity gap.
