## Summary

Keep open. Current main still has the Signal mention gate before attachment fetch/transcription, and the issue comments correctly identify that Signal has media metadata now but still lacks the Telegram/Discord/WhatsApp-style audio preflight path.

## What This Changes

Request a Signal audio-preflight path so inbound voice notes can be transcribed before the agent sees a `<media:audio>` placeholder, with a shell-command hook discussed as an alternate flavor.

## Best Possible Solution

Add a Signal-owned provider-backed preflight using the existing `transcribeFirstAudio` helper before mention gating, then pass transcript and transcribed-media index state downstream while deferring arbitrary shell-command hooks.

## Reproduction Assessment

Yes. Source-level reproduction is clear: a Signal group with `requireMention` and an audio-only message reaches the no-mention return before attachments are fetched, leaving no transcript path available.

## Solution Assessment

Yes for the narrow provider-backed path. Reusing `transcribeFirstAudio` and the existing `tools.media.audio` contract is the maintainable fix; adding a Signal-specific shell command hook is broader and riskier.
