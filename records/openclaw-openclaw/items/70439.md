## Summary

Keep open. Current `main` still leaves Signal without the provider-backed audio preflight path that the discussion asks for, and the existing Signal mention gate can skip audio-only group messages before attachments are fetched or transcribed.

## What This Changes

Request Signal audio preflight so voice notes are transcribed before the agent sees them, with either existing media transcription or a configurable external hook.

## Best Possible Solution

Add a Signal-owned provider-backed preflight using the existing `transcribeFirstAudio` helper before mention gating, then thread transcript and transcribed-media index state downstream while deferring arbitrary shell-command hooks.

## Reproduction Assessment

Yes at source level. A Signal group with `requireMention` and an audio-only message reaches the no-mention return before attachments are fetched, and current tests assert the skipped `<media:audio>` pending-history behavior.

## Solution Assessment

Yes for the narrow provider-backed path. Reusing `transcribeFirstAudio` and the existing `tools.media.audio` contract is the maintainable fix; a Signal-specific shell-command hook is broader and security-sensitive.
