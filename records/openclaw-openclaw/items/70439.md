## Summary

Keep open: current main still lacks Signal audio-preflight parity, and the requested external command flavor adds security/product questions that should not be auto-repaired without maintainer direction.

## What This Changes

Expose a Signal audio-preflight or configurable transcription hook so inbound Signal voice notes can become transcript text before reaching the agent.

## Best Possible Solution

Add a Signal-owned preflight path that reuses the shared tools.media.audio transcription helper before mention gating, and defer any arbitrary command hook until maintainers approve the security model.

## Reproduction Assessment

Yes, source-level: the Signal no-mention path records `<media:audio>` placeholder history and returns before attachment fetch/transcription, while sibling channels call shared preflight. I did not run a live Signal plus STT scenario.

## Solution Assessment

Partly: the Signal parity fix should reuse the shared audio-preflight helper before mention gating. The external preflight_cmd flavor is not yet the best first fix because it adds command-execution and config surface that need maintainer approval.
