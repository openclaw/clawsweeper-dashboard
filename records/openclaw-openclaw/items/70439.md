## Summary

Keep open. Current main still lacks a Signal audio-preflight transcript path comparable to Telegram, Discord, and WhatsApp, while the existing discussion and source both support a focused provider-backed Signal repair.

## What This Changes

Request a Signal audio-preflight hook so inbound voice notes can be transcribed before the agent receives placeholder-only audio messages.

## Best Possible Solution

Add a Signal-owned provider-backed audio preflight that reuses `transcribeFirstAudio` and `tools.media.audio`, then leave arbitrary shell command hooks for a separate reviewed design.

## Reproduction Assessment

Yes, at source level. A Signal group with `requireMention` and an audio-only attachment can hit the no-mention return before attachment fetch, and the current test suite asserts the resulting `<media:audio>` pending-history behavior.

## Solution Assessment

Yes for the provider-backed subset. Reusing the existing shared preflight helper is the narrow maintainable path; the shell-command hook variant should remain separate because it changes the security model.
