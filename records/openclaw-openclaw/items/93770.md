## Label Justifications

- `P1`: The reported release regression makes Telegram agent replies unreadable for affected Desktop/Web clients, breaking a real channel workflow.
- `impact:message-loss`: Readable Telegram reply content is replaced by unsupported-message placeholders, which is effectively lost message content for affected users.

## Summary

Current main already solves the user problem: Telegram rich messages are explicit opt-in through `channels.telegram.richMessages`, while the default path sends standard Telegram HTML/text. The fix is current-main-only after `v2026.6.8`, so affected release users still need a newer build or patch release.

## What This Changes

Request to add a Telegram configuration option that disables Bot API `sendRichMessage` and falls back to readable plain/HTML text delivery.

## Best Possible Solution

Keep the current-main behavior: standard Telegram HTML/text delivery by default, Bot API rich messages only through explicit `channels.telegram.richMessages: true`, and ship that fix in the next patch release.

## Reproduction Assessment

Yes for the reported released behavior: `v2026.6.8` source still routes text through `sendRichMessage`, and multiple related reports show unsupported-message rendering. Current main no longer reproduces that default path because rich messages are opt-in.

## Solution Assessment

Yes. The current-main fix is better than adding a separate disable flag because the broadly compatible `sendMessage`/HTML path is the default and Bot API rich messages remain available only through explicit opt-in.
