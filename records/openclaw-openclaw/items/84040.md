## Summary

Keep open: current main still accepts a well-formed unknown `agentId` when `allowAgents` contains `*`, and the behavior is security-sensitive because it can create sessions and agent homes under unintended identities. The fix should be maintainer-reviewed because current docs, types, and tests also describe `*` as broadly allowing targets, so changing it to registered-only is a compatibility and product-policy decision.

## What This Changes

The issue asks OpenClaw to reject unknown `sessions_spawn` agent IDs when wildcard subagent targeting is enabled, instead of auto-provisioning default-configured agent homes.

## Best Possible Solution

Define the wildcard contract explicitly, then enforce registered-target validation by default across native spawn, ACP spawn, and discovery, with a documented compatibility path only if maintainers approve it.

## Reproduction Assessment

Yes. Source inspection shows the wildcard branch returns ok before configured-agent membership is checked, and current tests expect wildcard and allowlisted-but-unconfigured targets to be accepted; I did not run a live spawn because this review is read-only.

## Solution Assessment

Partly. Registry validation before provisioning is the right safety boundary, but the proposed default-on strictness and opt-out flag need maintainer approval because current docs, types, and tests define `*` as broadly permissive.
