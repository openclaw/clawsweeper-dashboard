## Summary

Keep open. Current main still logs per-skill sandbox copy failures without removing the resolved destination, and the downstream Claude skills plugin still warns when snapshot skill files are missing. The proposed cleanup is scoped and appears functionally correct, but it should gain the required release note and focused regression coverage before landing, and maintainers should keep it coordinated with #74200.

## What This Changes

This PR adds best-effort removal of the resolved per-skill sandbox destination after `syncSkillsToWorkspace` fails to copy that skill.

## Best Possible Solution

Land this scoped failed-copy cleanup after adding release-note and regression coverage, or explicitly preserve the same per-skill cleanup guarantee inside the broader #74200 sync rewrite.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to make `fsp.cp` fail after creating the destination during sandbox skill sync, then build/use the skill snapshot and observe the Claude plugin missing-file warning path; this review verified the path by source inspection rather than executing a write-heavy repro.

## Solution Assessment

Yes for the narrow warning bug. Removing only the resolved per-skill sandbox destination after a copy failure is the smallest maintainable fix; the safer landing path is to add a targeted test and keep the behavior coordinated with #74200.
