## What This Changes

The branch adds best-effort removal of the resolved per-skill sandbox destination after `syncSkillsToWorkspace` catches a copy failure.

## Best Possible Solution

Land the scoped cleanup after redacted real behavior proof is added, with focused regression coverage if the branch is refreshed and coordination so https://github.com/openclaw/openclaw/pull/74200 preserves the same failed-copy cleanup guarantee.

## Reproduction Assessment

Yes, by source inspection rather than a live run. Current main can resolve a sandbox skill destination, have `fsp.cp` fail, leave that destination behind, and then the Claude plugin warns when a snapshot points at a missing `SKILL.md`.

## Solution Assessment

Yes for the narrow bug. Removing only the resolved per-skill sandbox destination after a failed copy is confined and maintainable; the broader non-destructive sync policy belongs in https://github.com/openclaw/openclaw/pull/74200 or a follow-up.
