## Summary

Keep open. Current main still catches per-skill sandbox copy failures in `syncSkillsToWorkspace` by logging only, while the downstream Claude skills plugin does warn on missing snapshot skill files. The PR's cleanup is narrow and appears functionally correct, but it still needs maintainer handling for regression coverage and coordination with the related open sync rewrite in #74200.

## What This Changes

This PR adds best-effort removal of the resolved per-skill sandbox destination after `syncSkillsToWorkspace` fails to copy that skill.

## Best Possible Solution

Land the scoped failed-copy cleanup or carry the same per-skill cleanup guarantee into the broader sync rewrite, with a focused regression test that leaves a partial destination and verifies it is absent after sync failure.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to make `fsp.cp` fail after the destination begins to exist during sandbox skill sync, then build/use the skill snapshot and observe the Claude plugin's missing-file warning path; this review verified the path by source inspection rather than running a write-heavy repro.

## Solution Assessment

Yes for the narrow warning bug. Removing only the resolved per-skill sandbox destination after a copy failure is the smallest maintainable fix; adding a focused test and coordinating with #74200 would make it safer to land.
