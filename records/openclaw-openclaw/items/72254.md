## Summary

Keep open: current main still lacks failed-copy cleanup and this PR is a focused valid fix, but the external contributor still needs to provide real after-fix behavior proof before merge.

## What This Changes

The branch removes a per-skill sandbox destination after `syncSkillsToWorkspace` fails to copy it, so incomplete skill directories are not exposed to later skill snapshots.

## Best Possible Solution

Land the scoped cleanup after contributor-supplied real behavior proof is added, ideally with focused regression coverage and with the same guarantee preserved if the broader sync rewrite proceeds.

## Reproduction Assessment

Yes, by source inspection rather than a live run. Current main can leave a destination that was created before `fsp.cp` fails, and the Claude plugin then warns when the resulting snapshot points at a missing `SKILL.md`.

## Solution Assessment

Yes for the narrow bug. Removing only the resolved per-skill sandbox destination after a failed copy is the smallest maintainable change; the broader non-destructive sync policy belongs in https://github.com/openclaw/openclaw/pull/74200 or a follow-up.
