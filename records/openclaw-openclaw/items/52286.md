## Summary

Keep open: current main still has the Telegram forum-topic routing gap for file/media sends when the model explicitly targets the current sender's positive Telegram user ID. Same-chat topic auto-threading exists, and related discussion correctly identified the reply-router/media inconsistency, but the current send path does not remap a sender-user target back to the originating supergroup topic.

## What This Changes

The issue requests that Telegram forum-topic file/media sends stay in the originating chat topic instead of being delivered to the sender's DM when the message tool target is the Telegram user ID.

## Best Possible Solution

Keep this as the canonical Telegram forum-topic file/media routing bug and implement a narrow same-conversation attachment route or Telegram-specific guard that redirects accidental current-sender user-id media sends back to the originating group topic while preserving an explicit way to DM a user intentionally.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Telegram forum-topic turn with currentChannelId/OriginatingTo set to the negative supergroup id, currentThreadTs/MessageThreadId set, SenderId set to a positive user id, and message(action=send, media=..., target=<SenderId>); current main preserves that target and resolves no topic thread.

## Solution Assessment

Unclear. Extending reply delivery to handle attachments would avoid target guessing and may be the cleanest end state, but a narrower Telegram guard may be safer; maintainer judgment is needed before changing DM-versus-topic semantics.
