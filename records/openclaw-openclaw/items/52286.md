## Summary

Keep open: current main still preserves an explicit Telegram sender-user target as a direct chat, so a forum-topic file/media send can bypass the originating topic even though same-chat targets are auto-threaded.

## What This Changes

Telegram forum-topic file/media sends via the message tool should stay in the originating topic instead of being delivered to the sender's Telegram DM.

## Best Possible Solution

Keep this as the canonical Telegram forum-topic file/media routing issue and add a narrow same-conversation attachment path or Telegram current-sender target guard that preserves intentional DM behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Telegram forum-topic turn with currentChannelId set to the supergroup and currentThreadTs set, followed by message(action=send, media=..., target=<positive SenderId>), preserves the sender target and resolves no thread.

## Solution Assessment

Unclear. Same-chat auto-threading and final media reply routing are useful, but the remaining sender-user target case needs maintainer intent rules before rewriting or guarding targets safely.
