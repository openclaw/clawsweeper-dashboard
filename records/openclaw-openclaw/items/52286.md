## Summary

Keep open: current main still preserves an explicit Telegram sender-user target as a direct chat, so a message-tool file/media send from a forum topic can bypass the originating topic. The open stale-thread fallback PR is adjacent but does not cover this sender-user-ID routing path.

## What This Changes

The issue asks for file/media sends from Telegram forum topics to stay in the originating topic instead of being sent to the sender’s DM.

## Best Possible Solution

Keep this as the canonical Telegram forum-topic file/media routing issue and implement a narrow same-conversation attachment route or Telegram-specific current-sender guard that preserves an explicit way to intentionally DM a user.

## Reproduction Assessment

Yes. A high-confidence source path is a Telegram forum-topic turn with currentChannelId set to the negative supergroup and currentThreadTs set, followed by message(action=send, media=..., target=<positive sender user id>); current main preserves that target, resolves no thread, and treats it as a direct chat.

## Solution Assessment

No. Current main’s same-chat auto-threading is correct for explicit group/topic targets, but it does not solve the sender-user-ID case; the safer fix needs a deliberate same-conversation attachment path or Telegram intent guard rather than broad target rewriting.
