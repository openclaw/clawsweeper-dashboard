## Summary

Keep open. Current `main` does not ship the PR's shared `replyToMode: "auto"` feature or its core followup plumbing. The shared reply-mode type/schema still stop at `off|first|all|batched`, queued followups still carry only a single `messageId`, collect-mode prompts are still the generic single-block form, and followup delivery still lacks the per-message ID reconciliation/multi-tag split path described in the PR and discussion.
