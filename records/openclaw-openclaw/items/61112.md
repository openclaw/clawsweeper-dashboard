Keep open, but the patch is not merge-ready: current main still has the markdown sidebar localization gap, while this PR targets an older sidebar shape and hand-edits generated locale bundles.

This PR adds `markdownSidebar.*` keys to six locale files and wires the older markdown sidebar title, close tooltip, raw-text label, and empty state through `t(...)`.

## Best Possible Solution

Refresh against current main by localizing every current markdown/canvas sidebar chrome string through the English locale source, regenerating all shipped locale bundles and metadata, and updating focused tests without changing rendering or sanitization behavior.

## Reproduction Assessment

Yes, by source inspection rather than a live browser run. `origin/main` still renders the sidebar chrome as literal English strings, and the current markdown sidebar test asserts two of those English labels.

## Solution Assessment

No, not as currently patched. Locale-backed sidebar chrome is the right direction, but the branch must cover the current sidebar surface and use the generated-locale workflow.
