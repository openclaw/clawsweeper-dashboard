## Summary

Keep open: the localization gap is still real, but this branch is stale against current main, would leave current markdown/canvas sidebar chrome hardcoded, and bypasses the generated locale workflow; it also lacks after-fix real behavior proof.

## What This Changes

The PR adds `markdownSidebar.*` keys to six locale bundles and changes four older markdown sidebar labels in `markdown-sidebar.ts` to use `t(...)`.

## Best Possible Solution

Refresh the localization on current main by moving every current markdown/canvas sidebar chrome string into the English locale source, regenerating shipped locale bundles plus `.i18n` metadata, and preserving rendering, sandbox, canvas URL, and raw-text behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows the markdown/sidebar chrome still renders English labels directly, and the current markdown sidebar test asserts two of those labels.

## Solution Assessment

No, not as currently patched. Locale-backed sidebar chrome is the right direction, but the branch must be refreshed for the current sidebar surface and generated-locale workflow.
