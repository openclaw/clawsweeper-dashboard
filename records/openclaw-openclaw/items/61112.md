## Summary

Keep this PR open. The reported localization gap still exists on current main, but the branch is stale: it localizes an older four-label sidebar shape and manually edits generated locale bundles instead of refreshing the current markdown/canvas sidebar through the Control UI i18n pipeline.

## What This Changes

The PR adds `markdownSidebar.*` locale entries and routes the old markdown sidebar title, close tooltip, raw-text action, and empty state through `t(...)`.

## Best Possible Solution

Land a refreshed localization change against current main that adds English source keys for every current markdown/canvas sidebar chrome label, wires `markdown-sidebar.ts` through `t(...)`, regenerates all shipped locale bundles and `.i18n` metadata, and updates focused tests without changing rendering, sanitization, or canvas behavior.

## Reproduction Assessment

Yes. Source inspection on current main is enough to reproduce the issue: switch to a non-English Control UI locale and open markdown/sidebar output, and the sidebar chrome still comes from hardcoded English strings that the current test also asserts.

## Solution Assessment

No, not as currently patched. Locale-backed sidebar chrome is the right direction, but the branch targets an older sidebar and bypasses the generated-locale workflow; a current-main refresh is the narrower maintainable fix.
