## Summary

Keep open. Current main still has the two-phase approval no-route bug in the shared pending-approval helper, while this PR remains useful but is stale against the refactored current path and overlaps the broader open #56581 consolidation.

## What This Changes

This PR adds a two-phase guard to exec approval no-route fast-fail behavior and adds gateway server-method tests for two-phase pending versus single-phase fast-fail behavior.

## Best Possible Solution

Adapt the intended guard at `handlePendingApprovalRequest`, keep single-phase no-route requests fast-failing, and add focused regression coverage for a two-phase exec approval with no approval clients, no accepted forwarding delivery, and no turn-source route. Maintainers should decide whether that lands as a narrow update from this PR or as part of #56581.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `exec.approval.request` with `twoPhase: true`, no approval clients, no accepted delivery, and no turn-source route; current main reaches the no-route expiry branch before sending the two-phase accepted response.

## Solution Assessment

Yes for the PR's original base, but not as-is for current main. The proposed behavior is the narrow maintainable fix, but after the refactor it belongs in `src/gateway/server-methods/approval-shared.ts` with updated regression coverage.
