## Summary

Keep open. The reported approval bug is still present on current main, but this PR is stale because the no-route branch moved from the old inline exec handler into the shared approval helper; it needs a small rebase plus changelog before merge.

## What This Changes

The PR changes exec approval handling and tests so two-phase exec approvals remain pending when no approval client or forwarding route is connected, while single-phase approvals still fast-fail.

## Best Possible Solution

Land the guard in `handlePendingApprovalRequest` on current main so two-phase no-route approvals return `accepted` and wait until timeout, while single-phase no-route requests still expire immediately, with focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes, source-reproducible. On current main, `exec.approval.request` with `twoPhase: true`, no approval clients, no turn-source route, and no delivery reaches the shared no-route branch and expires before the two-phase accepted response.

## Solution Assessment

No, not as currently diffed. The guard concept is correct, but current main centralizes the branch in `approval-shared.ts`, and the PR also needs the required changelog entry.
