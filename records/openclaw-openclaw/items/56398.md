## What This Changes

The PR adds a two-phase guard to exec approval no-client fast-fail logic and adds regression coverage for two-phase pending versus single-phase fast-fail approvals.

## Best Possible Solution

Move the no-route two-phase behavior into `handlePendingApprovalRequest`, preserve single-phase fast-fail behavior, and add shared-helper/server-method regression coverage around the current approval routing rules.

Yes, source-level. Current main sends agent exec approvals as `twoPhase: true`, passes them into `handlePendingApprovalRequest`, and that helper expires no-client/no-route requests before returning the two-phase `accepted` response.

## Solution Assessment

No, not as patched. The requested behavior is narrow, but the fix must be rebased into `approval-shared.ts` because the current runtime path no longer uses the old inline branch changed by this PR.
