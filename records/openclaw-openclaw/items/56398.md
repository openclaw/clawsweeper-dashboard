## Summary

Keep open: the reported bug is still source-reproducible on current main, but this branch changes an older inline fast-fail path and no longer fixes the current shared helper path.

## What This Changes

The PR adds a `!twoPhase` guard to the old exec approval no-route fast-fail branch and adds a regression test for two-phase no-client approvals.

## Best Possible Solution

Move the two-phase no-route guard into `handlePendingApprovalRequest` and add focused shared-helper coverage that preserves single-phase fast-fail while keeping no-client two-phase requests pending until timeout or approval.

## Reproduction Assessment

Yes. Source inspection shows agent exec requests send `twoPhase: true`, current exec approval handling delegates to `handlePendingApprovalRequest`, and that helper expires the no-client/no-delivery path before the two-phase accepted response.

## Solution Assessment

No, not as patched. The `!twoPhase` guard is the right behavior, but it needs to be applied in the shared helper now used by current main, not in the stale inline exec branch changed by this PR.
