## Summary

Keep open: the reported bug is real, but this stale-base patch changes the old inline exec approval branch while current main runs the no-route decision through the shared helper. The PR also needs inspectable real behavior proof before merge.

## What This Changes

The PR adds a guard so two-phase gateway exec approval requests stay pending without connected approval clients, plus a regression test preserving single-phase fast-fail behavior.

## Best Possible Solution

Rebase the concept into `handlePendingApprovalRequest` so two-phase no-route approvals return `accepted` and remain registered until timeout, with focused exec/shared/plugin approval coverage and without importing unrelated scope from https://github.com/openclaw/openclaw/pull/56581.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `exec.approval.request` sends `twoPhase: true`, current main delegates to `handlePendingApprovalRequest`, and the helper expires no-route requests before the two-phase accepted response.

## Solution Assessment

No, not as patched. The `!twoPhase` guard is the right idea, but the current-main implementation needs it in the shared helper rather than the stale inline exec branch.
