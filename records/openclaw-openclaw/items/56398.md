## Summary

Keep open: the reported approval bug is real, but this PR is stale against current main because the no-route fast-fail now lives in the shared approval helper, and the PR also lacks the required changelog entry.

## What This Changes

The PR adds a guard so two-phase exec approvals do not fast-fail when no approval clients are connected and adds tests for pending two-phase versus single-phase fast-fail behavior.

## Best Possible Solution

Update `handlePendingApprovalRequest` so two-phase no-route approvals remain pending until timeout while single-phase approvals still fast-fail, then add focused regression coverage and an Unreleased changelog entry.

## Solution Assessment

No, not as-is. The guard concept is right, but current main needs it in `approval-shared.ts` rather than the stale inline `exec-approval.ts` branch, plus a changelog entry.
