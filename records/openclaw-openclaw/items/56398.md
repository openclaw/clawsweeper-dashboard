## Summary

Keep open: the reported approval bug is source-reproducible, but this PR is stale against current main and lacks after-fix real behavior proof; current main routes the no-route decision through the shared approval helper, while the patch changes the old inline exec branch.

## What This Changes

The PR adds a two-phase guard to exec approval no-route fast-fail behavior and adds regression coverage for two-phase pending versus single-phase fast-fail requests.

## Best Possible Solution

Move the no-route guard into `handlePendingApprovalRequest` so two-phase no-route approvals return `accepted` and remain registered until timeout, with focused exec/shared/plugin coverage preserving single-phase fast-fail semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: agent exec registration sends `twoPhase: true`, current exec handlers delegate to `handlePendingApprovalRequest`, and that helper expires no-route approvals before it reaches the two-phase accepted response.

## Solution Assessment

No, not as patched. The `!twoPhase` guard is the right concept, but current main needs it in the shared helper rather than the stale inline exec branch changed by this PR.
