## Summary

Keep open: current main and v2026.6.10 still lack LINE native sticker sending and STICKER directive support, while inbound stickers are only converted to text context; the remaining scope needs maintainer product boundary.

## What This Changes

Request LINE sticker send and receive support, including AI-selected sticker replies and a catalog/skill path for package and sticker IDs.

## Best Possible Solution

Keep this issue as the canonical tracker, land a focused LINE-plugin sticker send path if maintainers sponsor it, track inbound metadata separately, and keep catalog/editor content ClawHub-first unless explicitly promoted.

## Reproduction Assessment

Yes, at source level: current main has no LINE sticker carrier, no STICKER directive parser, and no LINE sticker send helper. I did not perform a live LINE account test in this read-only review.

## Solution Assessment

No, not as originally scoped. A core ReplyPayload sticker field plus bundled catalog/editor is broader than needed; a LINE-owned channelData send path with separate inbound metadata and ClawHub catalog decisions is safer.
