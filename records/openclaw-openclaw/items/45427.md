## What This Changes

Request a trusted sender display-name or identity field in the inbound metadata envelope so agents do not infer recipients from untrusted conversational context.

## Best Possible Solution

Keep this issue as the canonical trusted inbound identity discussion, then land one approved contract that uses stable provider IDs or configured principals rather than trusting raw conversational display names.
