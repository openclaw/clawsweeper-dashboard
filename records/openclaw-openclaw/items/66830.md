## Summary

Keep this open: current main and the latest shipped release still compute the preflight and memoryFlush gates from `reserveTokensFloor`, while the exact replacement fix is still open and conflicting at https://github.com/openclaw/openclaw/pull/74010.

## What This Changes

The issue asks for preflight compaction and memoryFlush thresholds to honor the effective `agents.defaults.compaction.reserveTokens` budget instead of treating `reserveTokensFloor` as the full reserve.

## Best Possible Solution

Refresh and land https://github.com/openclaw/openclaw/pull/74010, or an equivalent narrow patch that computes the effective reserve once and feeds it into both compaction gates with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows both threshold gates still subtract `reserveTokensFloor`, while the config contract defines `reserveTokens` as the user-facing headroom knob.

## Solution Assessment

Yes. The narrow effective-reserve helper approach in https://github.com/openclaw/openclaw/pull/74010 matches the existing config contract without adding a new public setting or product policy.
