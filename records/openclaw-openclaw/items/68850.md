## Summary

Keep open: current main still can surface synthetic `webchat:g-agent` display names, but this PR is not merge-ready because it filters explicit labels too, lacks after-fix real UI proof, and now conflicts with the current session-controls module split.

## What This Changes

The branch filters `webchat:g-agent-...` session names from Control UI selector labels and adds helper/chat view tests for fallback labels.

## Best Possible Solution

Port a displayName-only synthetic-name filter into `ui/src/ui/chat/session-controls.ts`, keep explicit labels authoritative, and cover both hidden synthetic display names and preserved synthetic-shaped labels.

## Reproduction Assessment

Yes, source-level: current main's `resolveSessionDisplayName` returns a non-key `displayName`, and `resolveSessionScopedOptionLabel` routes selector rows into that resolver. I did not run a browser repro because this review is read-only and avoids commands that create artifacts.

## Solution Assessment

No, not as submitted. The narrow maintainable fix is a displayName-only filter in the current `session-controls.ts` owner module, with explicit `label` values preserved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve explicit labels when filtering synthetic names:** `ui/src/ui/app-render.helpers.ts:883-895`
