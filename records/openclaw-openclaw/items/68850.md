## Best Possible Solution

Rebase onto current main, apply the synthetic-name filter only to transport-derived `displayName` values in the shared session display path, preserve explicit `label` values, and add proof that the selector behavior works in the Control UI.

## Solution Assessment

No, not as submitted. The maintainable fix is a displayName-only filter in the current shared session-display path, with explicit labels preserved.
