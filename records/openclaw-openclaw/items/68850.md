## Summary

Keep open: current main still exposes synthetic `webchat:g-agent-...` display names in the Control UI selector, and this PR is useful but needs changes before merge because it filters explicit labels and targets an older helper location.

## What This Changes

The PR adds a synthetic transport-name filter and tests so webchat `g-agent` session display names do not appear as Control UI session selector labels.

## Best Possible Solution

Port the visual cleanup to `ui/src/ui/chat/session-controls.ts`, suppress only synthetic transport-derived `displayName` values, keep explicit `label` values authoritative, add regression coverage, and add the required changelog entry.

## Reproduction Assessment

Yes. A static reproduction on current main is a session row with key `agent:control-tower:main` and `displayName: "webchat:g-agent-control-tower-main"`; the current selector path routes that display name into the visible option before fallback.

## Solution Assessment

No, not as submitted. The intended UI-only cleanup is narrow and maintainable, but the submitted patch also clears explicit labels and needs to be ported from the old helper body to the current session-control module.
