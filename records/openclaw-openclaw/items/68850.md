## Summary

Keep open: current main still has a source-level path for synthetic `displayName` values to appear, but this PR is not merge-ready because it filters explicit labels, conflicts with the current session-label modules, and lacks real behavior proof.

## What This Changes

The PR filters synthetic `webchat:g-agent-...` display names from Control UI session labels and adds helper/chat selector tests.

## Best Possible Solution

Land a rebased patch against `session-display.ts` and `session-controls.ts` that suppresses only transport-derived `displayName` values while preserving explicit labels, with regression coverage for both cases.

## Reproduction Assessment

Yes, source-level: current main delegates a non-key `displayName` into `resolveSessionDisplayName`, which returns it before fallback, so a `webchat:g-agent-...` display name would be shown. I did not run a browser repro in this read-only review.

## Solution Assessment

No, not as submitted. The narrow maintainable fix is to port the change to the current shared resolver path and filter only transport-derived `displayName` values while leaving `label` authoritative.

## Review Findings

Overall confidence: 0.91
