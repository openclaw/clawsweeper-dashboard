## Summary

Keep open: current main still has the synthetic `webchat:g-agent-...` selector-label problem, but this PR is not ready to merge because the branch is dirty against current main and the patch discards explicit session labels that happen to match the synthetic pattern. The existing Codex review comment flagged the same label-preservation issue, and the related #51124 is adjacent Telegram label cleanup rather than a replacement for this webchat synthetic-name fix.

## What This Changes

The PR adds a synthetic transport-name detector and tests so webchat `g-agent` transport names are hidden from Control UI session selector labels while keeping session selection semantics unchanged.

## Best Possible Solution

Port the useful visual cleanup to the current session-control module, filter only transport-derived `displayName` values, keep explicit `label` values authoritative, add regression coverage for both hidden synthetic display names and preserved synthetic-shaped labels, and leave the broader Telegram cleanup to #51124.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a session row whose key is `agent:control-tower:main` or `agent:control-tower:subagent:<id>` and whose `displayName` is `webchat:g-agent-control-tower-main`; current main routes that non-key displayName straight into the visible selector label.

## Solution Assessment

No, not as submitted. The intended display cleanup is narrow and maintainable, but the PR should filter `displayName` only, preserve `label`, and be rebased or replaced against the current `ui/src/ui/chat/session-controls.ts` layout.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve explicit labels when filtering synthetic display names:** `ui/src/ui/app-render.helpers.ts:883-895`
  - body: The new synthetic-name filter is applied to `row.label` as well as `row.displayName`. `label` is the operator-authored session label stored by `sessions.patch`, so a deliberate label like `webchat:g-agent-control-tower-main` would now be replaced with fallback text and lose its disambiguation. Filter transport-derived `displayName` only and keep `label` authoritative.
  - confidence: 0.9
