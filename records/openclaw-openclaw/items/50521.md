## Summary

Keep this PR open. Current main still lacks explicit null-byte validation in the shared archive entry path helper, while the PR is a narrow security hardening change with focused tests and no blocking review findings.

## What This Changes

The PR adds an `entryPath.includes("\0")` guard in `validateArchiveEntryPath()` and adds two archive-path regression cases for null bytes in filenames and directory components.

## Best Possible Solution

Land this PR or an equivalent minimal patch that rejects null bytes inside `validateArchiveEntryPath()` before normalization and keeps the focused regression tests in the archive-path test file.

## Reproduction Assessment

Yes. Current main can be reproduced by calling the archive-entry validation predicates with `safe.txt\0.malicious` or `dir\0evil/file.txt`; source inspection and a read-only Node probe show those examples are not rejected today.

## Solution Assessment

Yes. The proposed guard is placed at the shared validation boundary before normalization, covers both ZIP and TAR extraction callers, and adds focused tests without changing valid archive entries.
