## Summary

Keep open. Current main still lacks explicit null-byte archive-entry validation, and this PR is a narrow, correct hardening patch with focused regression coverage and no discrete review finding.

## What This Changes

The PR adds an explicit null-byte rejection in `validateArchiveEntryPath()` and two archive-path tests for null bytes in filenames and directory components.

## Best Possible Solution

Land this PR or an equivalent minimal patch that rejects null bytes in the shared archive-entry validator before normalization and keeps the focused regression tests.

## Reproduction Assessment

Yes. Current main can be reproduced by evaluating the shared archive-entry validation logic with `safe.txt\0.malicious` or `dir\0evil/file.txt`; both pass the validator today while lower-level Node filesystem APIs reject them.

## Solution Assessment

Yes. The proposed guard is the narrowest maintainable fix because it sits at the shared validation boundary used by ZIP and TAR extraction and does not change valid archive entries.
