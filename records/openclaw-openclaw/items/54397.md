## Summary

Keep open. Current main has meaningful partial session management, but it still does not implement the requested first-class topic/session management lifecycle end to end: Control UI lacks create/import/export/archive-restore topic workflows, CLI only has listing/cleanup/trajectory export, and the Gateway session contract still lacks requested topic metadata such as description, tags, createdAt, and message count.

## What This Changes

Add first-class topic/session management UI and lifecycle support across Control UI, TUI, CLI, and Gateway session metadata/import/export/archive operations.

## Best Possible Solution

Keep this as the umbrella tracker, but split the work into maintainer-reviewed slices: Control UI topic/session creation and switching around existing session RPCs, a Gateway contract decision for topic metadata, CLI lifecycle commands if accepted, and a separate design for import/export plus deleted-session archive restore.
