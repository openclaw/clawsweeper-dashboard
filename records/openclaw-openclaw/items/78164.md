## Summary

Keep open: this draft, maintainer-labeled experimental runtime PR still has a source-backed event-forwarding regression before it is merge-ready.

## What This Changes

The PR adds opt-in Node worker isolation for agent attempts, worker build/config/docs plumbing, parent-side context-engine initialization before CLI compaction, and file-locked session-store writes across runtime isolates.

## Best Possible Solution

Keep the worker runtime opt-in, preserve the existing parent event bus contract from the worker bridge, add the required changelog entry, and then have maintainers decide when the draft experiment is ready.

## Reproduction Assessment

Yes for the review finding: source inspection shows the worker bridge handles agentEvent messages by invoking only the stripped callback while current parent subscribers listen through emitAgentEvent/onAgentEvent state. I did not run a live Control UI or subagent stream in this read-only review.

## Solution Assessment

No. The experimental direction is plausible, but the bridge should re-emit worker events through the parent-process event bus with runId/session context while keeping the callback for bookkeeping.
