## Summary

Keep open: current main still has the source-reproducible gateway lifecycle race, and the linked fix candidate at https://github.com/openclaw/openclaw/pull/79510 is still open and unmerged.

## What This Changes

The issue reports a gateway race where a deferred channel hot reload can overlap with a SIGUSR1 in-process restart and double-start the Telegram webhook listener.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/79510; this issue should close only after a merged fix cancels or fences deferred channel reload work during gateway shutdown/restart.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: defer a channel hot reload behind active work, trigger a SIGUSR1/gateway-tool restart before the deferral drains, and the old reload path can still call `startChannel` while the new gateway lifecycle starts channels.

## Solution Assessment

Yes. The narrow maintainable fix is to make deferred reloads cancellation-aware at the reloader/close boundary; a process-wide channel single-flight would be broader and would mask stale lifecycle work instead of stopping it.
