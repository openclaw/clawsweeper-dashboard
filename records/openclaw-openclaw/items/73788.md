  - body: Current main rejects plugin tools that are not listed in `contracts.tools`. This PR registers `memory_refresh`, but the manifest still declares only the three existing memory tools, so real plugin loading will skip the new tool even though the direct mock tests pass.

  - body: This adds a fourth agent-visible memory-lancedb tool, but the memory-lancedb docs still list only recall, store, and forget. Document search-only mode, replace mode, returned IDs, and audit behavior so operators can discover the new contract.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: Needs attention: the new local refresh audit log relies on default filesystem modes.

Concerns:

- **[medium] Audit log uses default filesystem permissions:** `extensions/memory-lancedb/index.ts:1038`
