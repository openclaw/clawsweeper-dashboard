## Summary

Keep open. Current main still lacks `memory_refresh`, so this contributor feature PR is not obsolete; however, the proposed patch is not merge-ready because it still creates the refresh audit log with default filesystem permissions and omits the required docs and changelog follow-through for a user-facing plugin tool.

## What This Changes

The PR adds a `memory_refresh` tool to the bundled `memory-lancedb` plugin with search-only conflict preview, serialized replace/forget operations, rollback/audit behavior, and mocked tests for the new paths.

## Best Possible Solution

Keep the implementation in the `memory-lancedb` plugin if maintainers accept the feature shape, but make audit persistence private by construction, document the new tool and audit behavior, add the required changelog entry, and then run the targeted plugin tests plus the changed gate before merge.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. The current-main gap is high-confidence from source and docs: only three memory-lancedb tools are registered and documented, while the PR adds a fourth with mocked coverage for its main branches.

## Solution Assessment

Unclear as submitted. A plugin-owned `memory_refresh` tool is the right boundary, but the branch needs the permission fix plus docs/changelog follow-through, and maintainers should confirm the replace/new-ID contract before merge.
