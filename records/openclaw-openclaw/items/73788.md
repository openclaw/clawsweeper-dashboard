## Summary

Keep open. Current main still lacks the requested `memory_refresh` tool, and the closed predecessor #43498 was not merged, so this PR remains the active implementation candidate. The current head is not merge-ready because the new audit log relies on default filesystem permissions and the user-facing tool still lacks docs and changelog follow-through.

## What This Changes

The PR modifies the bundled `memory-lancedb` plugin and tests to add a `memory_refresh` tool with search-only conflict preview, per-ID serialized replace behavior, rollback/audit logging, and mocked coverage.

## Best Possible Solution

Keep the feature in the `memory-lancedb` plugin if maintainers accept the replace contract, but make audit persistence private by construction, document the new tool and audit behavior, add the required changelog entry, and then rerun targeted plugin tests plus the changed gate before merge.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. The current-main gap is high-confidence from source and docs, and the PR head shows the new tool registration plus mocked coverage for search, replace, rollback, inheritance, and audit paths.

## Solution Assessment

Unclear as submitted. A plugin-owned `memory_refresh` tool is the right boundary, but the permission fix, docs/changelog follow-through, and explicit maintainer acceptance of the new-ID replace contract are needed before this is the best maintainable solution.
