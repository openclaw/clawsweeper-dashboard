## Summary

Keep open. The feature is plausible in the memory-lancedb plugin, but the PR is not merge-ready because the new runtime tool is not declared in the manifest contract, the audit log uses default filesystem permissions, and the user-facing docs/changelog are missing.

## What This Changes

The PR adds a `memory_refresh` tool to `memory-lancedb` with search-only conflict preview, atomic replace with rollback, per-memory locking, audit logging, and mocked tests.

## Best Possible Solution

Keep the feature plugin-owned if maintainers accept it, but merge only after the manifest declaration, private audit-log modes, docs, changelog, and a registry/contract-level test are added.

## Reproduction Assessment

Not applicable as an end-user bug reproduction. The main merge blocker is source-verifiable: current main rejects undeclared plugin tools, while the PR registers `memory_refresh` without updating the manifest and tests only a direct mock API path.

## Solution Assessment

No as submitted. A `memory-lancedb` owned refresh tool is the right boundary, but the implementation must satisfy the manifest-first tool contract and private local-state conventions before it is maintainable.
