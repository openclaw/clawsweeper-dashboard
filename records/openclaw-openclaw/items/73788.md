  - body: `mkdir(..., mode)` does not tighten an existing `~/.openclaw/memory` directory, and this chmod only runs when the audit file did not already exist.   - body: `stat(auditLogPath)` and `open(auditLogPath, "a", 0o600)` both follow symlinks. Because the audit path is predictable, a preexisting symlink can receive refresh metadata; check with `lstat`/`O_NOFOLLOW` or an existing safe-file helper before writing.
  - confidence: 0.9
- **[P2] Align docs with best-effort preserved-id refresh:** `docs/plugins/memory-lancedb.md:259-281`
  - body: The docs still say `memory_refresh` atomically replaces and creates a fresh id, but the code preserves the original id and explicitly documents best-effort process-mutex semantics. Update this section so users do not rely on nonexistent storage-level atomicity or fresh-id behavior.
