## Summary

Keep open. The PR is useful source work, but it is not merge-ready against current main because the new `memory_refresh` tool is registered without the required manifest contract, so current plugin loading would reject it; audit-log permissions, docs, and changelog follow-through also remain unresolved.

## What This Changes

The PR adds a bundled `memory-lancedb` `memory_refresh` tool with search-only conflict preview, serialized replace/rollback behavior, `memory_forget` locking, and mocked tests.

## Best Possible Solution

Keep the feature in the memory-lancedb plugin if maintainers accept the replace contract, but add the manifest declaration, private audit-log modes, user docs, changelog, and a registry/contract-level test before merge.

## Reproduction Assessment

Not applicable as a bug reproduction. The merge blocker is source-verifiable: current main rejects undeclared plugin tools, while the PR registers `memory_refresh` without updating the manifest and tests only a direct mock API path.

## Solution Assessment

No as submitted. A plugin-owned `memory_refresh` tool is the right boundary, but the branch must satisfy OpenClaw's manifest-first tool contract and user-facing release/docs requirements before it is the maintainable solution.
