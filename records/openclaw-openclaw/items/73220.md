## Summary

Keep open. Current main still lacks the requested diagnostic and this PR targets the right mapper, but the submitted helper would produce misleading hints for nonnumeric and exclusive-bound Zod issues and needs changelog coverage before merge.

## What This Changes

The branch adds numeric min/max text to mapped Zod config validation issues and a regression test for `session.agentToAgent.maxPingPongTurns`.

## Best Possible Solution

Land a narrow mapper change that enriches only numeric-origin bounds, handles inclusive and exclusive constraints accurately, preserves nonnumeric length/size messaging, adds focused regression coverage, and records the diagnostics fix in `CHANGELOG.md`.

## Reproduction Assessment

Yes. Source inspection shows `session.agentToAgent.maxPingPongTurns: 10` reaches a `.max(5)` schema while current main returns the raw Zod message through `mapZodIssueToConfigIssue` without a maximum-bound hint.

## Solution Assessment

No, not as submitted. `mapZodIssueToConfigIssue` is the right surface, but the helper should gate on numeric origins, respect `inclusive === false`, and include the required changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Restrict numeric hints to numeric origins:** `src/config/validation.ts:497-509`
  - body: Zod uses `too_big` and `too_small` for string length, array/set size, and file size checks too. Because this helper checks only the issue code and numeric bound, existing config constraints like `z.string().max(50)` would get wording such as `maximum allowed value is 50`; gate the hint to numeric origins such as `number` or `int`.
  - confidence: 0.9

  - confidence: 0.9
