## Summary

Keep open. The PR targets the right config-validation mapper for the linked diagnostics bug, but the submitted helper is not merge-ready because it would mislabel non-numeric Zod length/size constraints, misstate exclusive numeric bounds already used by current positive-number config fields, and lacks the required changelog entry.

## What This Changes

The PR adds numeric min/max hints to mapped Zod config validation errors and regression coverage for the `session.agentToAgent.maxPingPongTurns` ceiling case.

## Best Possible Solution

Land a narrow config-layer mapper change that enriches only numeric-value bounds, handles inclusive and exclusive constraints accurately, preserves allowed-values behavior, adds focused mapper coverage, and records the user-facing diagnostics fix in `CHANGELOG.md`.

## Reproduction Assessment

Yes. Current main has a source-level reproduction by validating `session.agentToAgent.maxPingPongTurns: 10`, which hits the `.max(5)` schema while the mapper returns the raw Zod message without a maximum-bound hint; the PR regressions are also traceable from Zod v4.4.1 issue contracts and current positive-number config schemas.

## Solution Assessment

No, not as submitted. `mapZodIssueToConfigIssue` is the right narrow surface, but the helper needs to gate on numeric origins and branch on `inclusive === false`, then add the missing changelog line.
