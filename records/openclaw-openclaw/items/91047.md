## Summary

Keep this open: current main and the latest release still resolve plugin session-extension patching and projection through the mutable active plugin registry, and the linked open fix PR should remain the closure path until it merges.

## What This Changes

The issue requests pinning the plugin session-extension registry so `sessions.pluginPatch` and `pluginExtensions` projection survive agent/subagent plugin registry churn.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/91208, or replace it with an equivalent pinned session-extension registry fix, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes from source, though I did not run the live WebSocket gateway flow. Current main reads session-extension patching and projection from the mutable active registry while only channel and HTTP-route surfaces are pinned.

## Solution Assessment

Yes. A pinned session-extension registry is the narrow owner-boundary fix; adding a downstream fallback inside `sessions.pluginPatch` would leave projection and lifecycle ownership split.
