## Summary

Keep open. Current main still drops `cwd` on the native `sessions_spawn` path, but this PR’s attachment override would break cross-agent native spawns by writing attachments under the caller workspace while the child still runs in the target agent workspace.

## What This Changes

The PR forwards `sessions_spawn.cwd` into native subagent workspace context and adds an attachment workspace override for inline attachments.

## Best Possible Solution

Compute the final child workspace once, reuse it for spawned metadata and attachment materialization, preserve target-agent workspace selection for cross-agent native spawns, and add focused regression tests plus the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a native `sessions_spawn` from one allowed agent to another with inline attachments and a caller workspace or `cwd`: the PR writes attachments under `ctx.workspaceDir`, while the child still resolves to the target agent workspace.

## Solution Assessment

No. Forwarding `cwd` into native same-agent workspace inheritance is the right direction, but the attachment change should use the resolved child workspace rather than raw caller context.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Resolve the child workspace before materializing attachments:** `src/agents/subagent-spawn.ts:635`

Compute the final child workspace first and pass that same value to both attachment materialization and spawned metadata.
