## Summary

Keep open: the reported native `sessions_spawn.cwd` bug is still present on current `main`, but this PR is not mergeable as-is because it routes `cwd` through inherited context instead of explicit spawn state and then writes attachments to a workspace that can differ from the child runtime workspace.

## What This Changes

The PR forwards parsed `sessions_spawn.cwd` into the native subagent context workspace and adds a context-workspace override for inline attachment materialization.

## Best Possible Solution

Thread `cwd` as explicit native spawn state, resolve the final child workspace once, use that same resolved workspace for attachments and spawned metadata, and add focused same-agent plus cross-agent regression coverage.

## Reproduction Assessment

Yes, at source level: current main reads `sessions_spawn.cwd` and ACP forwards it, while native `SpawnSubagentParams` cannot receive it and cross-agent native resolution drops inherited workspace context. I did not run a live native subagent canary in this read-only review.

## Solution Assessment

No. The PR's context-workspace shortcut is not the best fix because explicit `cwd` should be carried as its own spawn parameter and resolved once before both runtime metadata and attachment materialization use it.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Pass cwd as explicit native spawn state:** `src/agents/tools/sessions-spawn-tool.ts:315`
  - body: Putting `cwd` into `ctx.workspaceDir` still fails cross-agent native spawns: `spawnSubagentDirect` discards that context workspace before resolving the child workspace when `agentId` targets another agent. Add `cwd` to `SpawnSubagentParams` and pass it to `resolveSpawnedWorkspaceInheritance` as `explicitWorkspaceDir` so explicit user intent beats target-agent fallback like ACP does.
  - confidence: 0.9
- **[P1] Use the resolved child workspace for attachments:** `src/agents/subagent-spawn.ts:635`
  - body: Forwarding raw `ctx.workspaceDir` into attachment materialization can write files under the requester/cwd workspace while the cross-agent child still runs in the target agent workspace, leaving relative attachment paths missing at runtime. Resolve the final child workspace once and pass that same value to attachment materialization and spawned metadata.
  - confidence: 0.92
