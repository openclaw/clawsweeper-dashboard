## Summary

Keep open. The PR addresses a real native `sessions_spawn.cwd` gap on current `main`, but the branch is not merge-ready because its attachment follow-up uses the raw requester workspace before the final child workspace policy is resolved, which breaks cross-agent spawns with inline attachments; it also needs the required user-facing changelog entry.

## What This Changes

This PR forwards `sessions_spawn.cwd` into native subagent workspace inheritance and adds a workspace override path for inline subagent attachment materialization.

## Best Possible Solution

Compute the final child workspace once in the native subagent path, use that same value for child metadata and attachment materialization, preserve the existing target-agent workspace behavior for cross-agent spawns, add focused regression coverage, and add a single-line `CHANGELOG.md` fix entry crediting the contributor.

## Reproduction Assessment

Yes. Current main can be checked statically: native `sessions_spawn` reads `cwd` but does not pass it into `spawnSubagentDirect`; the PR regression is also reproducible by a cross-agent native spawn with inline attachments, where files are written under the requester workspace while the child resolves relative paths under the target workspace.

## Solution Assessment

No. Forwarding `cwd` toward native subagent workspace inheritance is the right direction, but using raw `ctx.workspaceDir` for attachments is not the narrow maintainable fix; attachment materialization should consume the already-resolved child workspace instead.
