## Summary

Close as superseded: the underlying `sessions_spawn.cwd` bug is still real on current main, but this PR’s approach is functionally incorrect and the remaining work is now tracked by the canonical open issue.

## What This Changes

The PR attempts to make native `sessions_spawn(runtime="subagent")` honor `cwd` by forwarding it as the spawned workspace and by overriding attachment materialization workspace.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/79840 as the canonical tracker and land a fresh focused fix that threads `cwd` as explicit native spawn state, resolves the final child workspace once, and uses that same workspace for metadata and attachments.

## Reproduction Assessment

Yes, at source level: current main reads `sessions_spawn.cwd` and ACP forwards it, while native `SpawnSubagentParams` and the native call path still drop it before workspace inheritance. I did not run a live native subagent canary in this read-only review.

## Solution Assessment

No. The PR’s context-workspace shortcut is not the best fix because explicit `cwd` should be carried as its own spawn parameter and resolved once before both spawned metadata and attachment materialization use it.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Pass cwd as explicit native spawn state:** `src/agents/tools/sessions-spawn-tool.ts:315`
  - body: Putting `cwd` into `ctx.workspaceDir` still fails cross-agent native spawns because `spawnSubagentDirect` discards that inherited workspace when `agentId` targets another agent. Add `cwd` to `SpawnSubagentParams` and pass it to `resolveSpawnedWorkspaceInheritance` as explicit spawn state.
  - confidence: 0.9
- **[P1] Use the resolved child workspace for attachments:** `src/agents/subagent-spawn.ts:635`
  - body: Forwarding raw `ctx.workspaceDir` into attachment materialization can write files under a different workspace than the child actually runs in for cross-agent spawns, leaving relative attachment paths missing at runtime. Resolve the final child workspace once and use it for both attachments and spawned metadata.
  - confidence: 0.92
