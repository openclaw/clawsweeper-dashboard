## Summary

Keep open. The native `sessions_spawn.cwd` gap is real, but the current PR head creates a cross-agent attachment workspace mismatch and still needs the required user-facing changelog entry before merge.

## What This Changes

The PR forwards `sessions_spawn.cwd` into native subagent workspace context and adds an attachment workspace override for inline attachments.

## Best Possible Solution

Resolve the final native child workspace once, reuse it for spawned metadata and attachment materialization, preserve the cross-agent target-workspace policy, and document the user-facing fix in the changelog.

## Reproduction Assessment

Yes. A static reproduction is a cross-agent native `sessions_spawn` with inline attachments plus an inherited workspace or `cwd`: the PR writes attachments under `ctx.workspaceDir`, while the child still runs from the target agent workspace.

## Solution Assessment

No. Forwarding `cwd` for the intended native same-agent inheritance path is plausible, but the patch should not pass raw caller context into attachment materialization after the final child workspace policy diverges for cross-agent spawns.

## Review Findings
