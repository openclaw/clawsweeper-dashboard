## Summary

## What This Changes

Request an explicit trusted workspace alias or memory-root config so workspace-only file tools and memory flushes can write through an intentional symlinked memory directory without disabling path-escape protection.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/90986 after product/security approval or choose a narrower memory-root/filesystem-policy design, then close this issue after the accepted implementation merges.

## Reproduction Assessment
