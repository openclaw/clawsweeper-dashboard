## What This Changes

The issue asks OpenClaw to define a consistent skill applicability and loading prompt contract across the native OpenClaw harness and the Codex harness.

## Best Possible Solution

Define the permanent OpenClaw/Codex skill-contract boundary, including prompt layer, lifetime, applicability rules, and loading mechanism, then update native prompt code, Codex app-server wiring, parity tests, and docs together.

## Reproduction Assessment

Yes. Current main and v2026.6.10 provide a high-confidence source-level reproduction: native OpenClaw emits Skills rules in the system prompt, while Codex receives the OpenClaw skill snapshot through turn-scoped collaboration developer instructions alongside Codex-native skills behavior.

## Solution Assessment

No implementation path is confirmed yet. Upstream Codex has its own native skill-instruction contract, so the best solution needs a maintainer decision about whether OpenClaw should align with Codex, wrap it, or keep a documented OpenClaw overlay.
