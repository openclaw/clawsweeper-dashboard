## Summary

Keep open: this is a maintainer-authored, protected issue, and current main plus the latest release still show the native OpenClaw skill rules and the Codex app-server OpenClaw skill overlay living in different prompt layers alongside Codex-native skill handling. No merged PR has settled the cross-harness skill selection/loading contract.

## What This Changes

The issue asks OpenClaw to define and align skill selection and loading prompt semantics between the native OpenClaw and Codex app-server harnesses.

## Best Possible Solution

Have maintainers choose the permanent cross-harness skill contract, then update native prompt construction, Codex app-server wiring, parity tests, and docs together.

## Reproduction Assessment

Yes. Current source gives a high-confidence reproduction: native OpenClaw emits a Skills section in the system prompt, while Codex receives the OpenClaw skills snapshot through turn-scoped collaboration developer instructions and can also have Codex-native skill instructions and selected-skill user fragments.

## Solution Assessment

Unclear for implementation. The issue identifies a real divergence, but the best fix depends on a maintainer product decision about which prompt layer and loading rules should be authoritative across harnesses.
