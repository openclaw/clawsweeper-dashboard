## What This Changes

The PR changes CLI missing-command diagnostics so `openclaw commands` points users to the runtime `/commands` slash command and adds focused CLI regression tests.

## Best Possible Solution

If this UX should land, rebase it as a typed built-in runtime-slash classification in `src/cli/run-main-policy.ts`, preserve known-plugin allowlist behavior, and cover both policy and CLI exit paths.

## Reproduction Assessment

Yes, by source inspection. Current main defines `/commands` as a built-in chat command but has no CLI policy branch for built-in runtime slash commands, while this PR still preserves the misleading allowlist path when `plugins.allow` is configured.

## Solution Assessment

No. The PR points at the right diagnostic, but it should not preserve the original allowlist failure and should return a typed policy result instead of parsing user-facing message text.
