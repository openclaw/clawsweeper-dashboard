## Summary

Keep open. The PR addresses a valid CLI diagnostic problem, but it is not mergeable as-is: current main already removed the broad `plugins.allow` false-positive path, while this branch still preserves that stale message for `commands` and uses human-readable diagnostic text as control flow.

## What This Changes

The branch adds a hard-coded built-in `/commands` runtime-slash diagnostic in `src/cli/run-main.ts`, adds CLI tests for that path, and updates Matrix ACP spawn target assertions.

## Best Possible Solution

Rework this as a typed built-in runtime-slash classification in `src/cli/run-main-policy.ts`, preserving plugin-owned allowlist diagnostics only for real plugin-owned roots and covering both policy and CLI exit paths.

## Reproduction Assessment

Yes, by source inspection. Current main registers `commands` as `/commands` but lacks a built-in runtime-slash diagnostic, and the PR diff shows the allowlist-preservation and string-matching paths that would affect `openclaw commands`.

## Solution Assessment

No. The PR targets the right UX, but the maintainable fix should return a structured policy result instead of hard-coding a map in `run-main.ts` and parsing display text.
