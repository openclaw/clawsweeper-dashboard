## What This Changes

The branch replaces before_tool_call and abort object-spread wrappers with descriptor-preserving execute wrappers and hides plugin MCP tools whose `parameters` schema cannot be read.

## Best Possible Solution

Resolve the conflicts, consolidate or sequence the overlapping schema-quarantine PRs, and land one maintainer-approved fail-closed MCP plugin-tool schema policy with descriptor-preserving wrappers aligned to the shared runtime schema quarantine path.

## Reproduction Assessment

## Solution Assessment

Mostly yes. Descriptor-preserving wrappers are the right layer for the eager getter bug, but hiding unreadable-schema MCP tools is a compatibility policy decision that should be coordinated with https://github.com/openclaw/openclaw/pull/89493 and the newer wrapper/schema-quarantine PRs.
