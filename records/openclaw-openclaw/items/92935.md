The issue reports that generated local images fail through documented Control UI/WebChat `[embed ref="..."]` canvas embeds even though the same files render through `MEDIA:/data/.openclaw/media/outbound/...`.

## Best Possible Solution

Keep this open until the Control UI canvas/embed path is proven against current `main`; the right fix should keep iframe access on scoped `pluginSurfaceUrls.canvas` capabilities, refresh with `node.pluginSurface.refresh`, and leave the assistant-media ticket route unchanged.

## Solution Assessment
