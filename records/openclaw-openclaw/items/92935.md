## Summary

Current main and shipped code already implement the supported Control UI paths: `[embed ...]` renders hosted canvas documents through scoped capability URLs, and generated files/images use structured media or `MEDIA:` delivery rather than bare unauthenticated canvas URLs.

## What This Changes

The issue asks Control UI/WebChat to render generated image files consistently when sent via `[embed]`/canvas or MEDIA, without requiring a MEDIA outbound workaround.

## Best Possible Solution

Keep the current contract: use structured media or `MEDIA:` for generated file attachments, and use `[embed ...]` for hosted Control UI canvas documents loaded through scoped capability URLs.

## Reproduction Assessment

No high-confidence current-main failing reproduction remains. The provided curl evidence checks the bare canvas URL, and current source/tests show that path should return 401 while WebChat uses a scoped capability URL for embeds.

## Solution Assessment

Yes. The shipped implementation preserves the gateway auth boundary for canvas content while routing generated images/files through the assistant-media attachment path.
