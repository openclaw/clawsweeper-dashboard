## Summary

Keep this open for maintainer review. Current main has the intended scoped canvas capability path, and the reporter's bare 401 evidence from OpenClaw 2026.5.6 matches a fixed/expected auth boundary, but current source still has unresolved reliability edges when `pluginSurfaceUrls.canvas` is missing, unscoped, or expires.

## What This Changes

The issue asks for generated PNGs to render or download reliably through documented Control UI `[embed ref="..."]` canvas embeds instead of requiring the `MEDIA:` outbound workaround.

## Best Possible Solution

Preserve scoped canvas capability URLs as the supported auth model, then make the Control UI path either never emit raw protected canvas iframe URLs under gateway auth or refresh/reconnect before capability expiry, with targeted tests and live Control UI proof.

## Reproduction Assessment

Yes, source-reproducible for the unresolved edge: current main can still return a raw protected canvas path when no scoped surface URL is present, and gateway tests prove bare canvas routes return 401. No live current-main browser reproduction has confirmed the reporter's exact setup yet.

## Solution Assessment

No, the `MEDIA:` workaround is not the best solution for documented embeds, and the old unmerged refresh PR was not the right final shape. The best fix should stay on the scoped plugin-surface contract and address missing or expired canvas surface URLs deliberately.
