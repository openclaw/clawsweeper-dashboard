## Summary

Keep open. ## What This Changes

The issue asks for generated local images/files to render consistently in Control UI/WebChat when sent through canvas/embed, instead of requiring the working MEDIA outbound route workaround.

## Best Possible Solution

Keep the canvas auth boundary intact, then either confirm the supported self-closing embed/MEDIA contract with current-release proof or make a narrow maintainer-approved parser/docs change for the shorthand users are actually emitting.

Unclear. Weakening canvas auth is not the right fix; the maintainable path is a maintainer decision on whether to clarify the current MEDIA/self-closing embed contract or accept a narrow shorthand syntax with tests.
