## Summary

This PR is superseded: current main already contains the request-header compatibility change and changelog entry, while the PR discussion and follow-up #76880 show the remaining Google Chat auth regression is not resolved by this request-only patch.

## What This Changes

The PR adds a Google Chat Gaxios request interceptor that converts non-native request headers to `Headers` and adds unit coverage for object-header normalization and passthrough.

## Best Possible Solution

Keep the current-main request normalizer, close this obsolete PR branch, and fix the remaining Google Chat auth regression in a follow-up that preserves or normalizes response headers under #76880.

## Reproduction Assessment

Yes for source-level reproduction, but no live Google Chat run was performed here. The current app-url auth path reaches `OAuth2Client.verifyIdToken()`, google-auth-library reads `res?.headers.get()`, and the PR discussion plus #76880 report that exact failure after the request-header normalizer landed.

## Solution Assessment

No. The request-header normalizer is useful but incomplete; the narrow maintainable fix needs to address the response headers returned to google-auth, not merge this already-superseded request-only patch.
