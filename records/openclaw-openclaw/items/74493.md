## Summary

Keep open for maintainer review. Current main still has the source-level reset path, and the PR head appears to address it with focused coverage and no blocking correctness or security finding from this read-only review.

## What This Changes

This PR changes Claude CLI auth-epoch hashing for token-shaped credentials and token auth profiles to ignore rotating token material, adds a Claude null-read fallback, bumps the auth-epoch version, adds regression tests, and updates the changelog.

## Best Possible Solution

Land this PR or an equivalent narrow patch after CODEOWNER review and required checks, keeping parser hardening or richer Claude account identity as separate follow-up work unless maintainers choose to expand the scope.

## Reproduction Assessment

Yes. Current main has a high-confidence source-level reproduction: a missing `refreshToken` parses as a token credential, token-shaped credentials are hashed with rotating token material, and same-version auth-epoch mismatches reset reusable CLI sessions.

## Solution Assessment

Yes, with maintainer contract review. Identity-only hashing for the remaining token paths plus a version bump and Claude null-read fallback is the narrowest maintainable fix for rotation-induced resets; parser cache hardening would be defense-in-depth rather than required for the reported symptom.
