## Summary

Keep open for normal PR validation and landing. Current main still has the reported auth-epoch drift path, and this active PR is a focused fix with no blocking code-review or security finding from this read-only pass.

## What This Changes

The PR bumps `CLI_AUTH_EPOCH_VERSION` to 5, hashes Claude CLI token credentials and token auth-profile credentials by identity instead of token material, adds regression tests, and updates `CHANGELOG.md`.

## Best Possible Solution

Land this focused auth-epoch fix after routine CI and maintainer checks; keep parser hardening or local credential file-presence behavior in separate follow-ups such as https://github.com/openclaw/openclaw/pull/80265.

## Reproduction Assessment

Yes. Source inspection on current main shows the parser can produce token-shaped Claude credentials, token encoders include rotating token material, and same-version auth-epoch drift resets reusable CLI sessions; I did not run the live macOS race.

## Solution Assessment

Yes. Identity-only token hashing plus an epoch-version bump is the narrow maintainable fix, and the latest head preserves true credential absence as `undefined` without adding parser churn.
