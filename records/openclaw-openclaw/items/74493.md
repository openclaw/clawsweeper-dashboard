## Summary

Keep open for normal PR landing: current main still has the reported auth-epoch drift, and this PR is a focused fix with sufficient terminal proof, regression tests, no blocking findings, and no security or supply-chain concerns.

## What This Changes

The branch bumps the CLI auth-epoch version, hashes Claude CLI token-shaped credentials and token auth-profile credentials by stable identity fields, adds regression coverage, and updates the changelog.

## Best Possible Solution

Land this focused auth-epoch fix after routine maintainer approval; keep parser hardening and the identity-less local credential file-presence work in separate follow-ups such as https://github.com/openclaw/openclaw/pull/80265.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a partial Claude credential can parse as `type: "token"`, current main hashes the rotating token material, and same-version auth-epoch drift invalidates reusable CLI sessions. I did not run the live macOS Keychain race.

## Solution Assessment

Yes. Identity-only hashing at the auth-epoch layer plus an epoch-version bump is the narrow maintainable fix, and the latest head preserves true credential absence as `undefined` rather than hiding logout/removal.
