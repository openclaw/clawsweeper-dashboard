## Summary

Keep open for maintainer review: current main still has the auth-epoch drift path, and this PR appears to address it with targeted encoder/version changes, regression coverage, and sufficient terminal proof.

## What This Changes

The PR changes CLI auth-epoch encoding to ignore rotating Claude/token material, adds a Claude null-read fallback, bumps the epoch version, adds regression tests, and updates the changelog.

## Best Possible Solution

Land this PR or an equivalent narrow patch after maintainer review and exact-head checks, leaving parser hardening or other-provider null fallbacks to separate follow-up work.

## Reproduction Assessment

Yes, by source inspection. Current main can parse a partial Claude credential as `type: "token"`, hash the rotating token material, and reset reusable sessions when the same-version epoch changes.

## Solution Assessment

Yes. Identity-only token hashing plus a version bump and Claude null-read fallback is the narrowest maintainable fix for the observed auth-epoch resets without changing parser behavior.
