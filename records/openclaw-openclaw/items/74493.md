## Summary

Keep open: the Claude partial-read fix is plausible and has proof, but the PR still drops token material from all token auth-profile epochs, so same-profile static token swaps can reuse old CLI sessions contrary to the documented session boundary.

## What This Changes

The PR makes Claude token-shaped credentials and token auth profiles identity-only in CLI auth-epoch hashing, bumps `CLI_AUTH_EPOCH_VERSION` to 5, and adds regression coverage plus a changelog entry.

## Best Possible Solution

Land a narrowed auth-epoch fix that keeps Claude OAuth partial reads and token refreshes stable, bumps the epoch version, and preserves invalidation for true static-token changes or token-only account switches.

## Reproduction Assessment

Yes, source-level. Current main can parse a partial Claude OAuth credential as token-shaped, hash rotating token material into the auth epoch, and reset same-version reusable sessions when that hash changes.

## Solution Assessment

No. Identity-only hashing is the right direction for the Claude partial-read path, but applying it to every token auth profile is broader than the current docs contract; keep token invalidation unless a stable identity boundary is proven.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Preserve static-token invalidation:** `src/agents/cli-auth-epoch.ts:112-123`
