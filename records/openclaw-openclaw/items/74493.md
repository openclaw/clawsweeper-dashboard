## Summary

Keep open: the main token-hashing fix is well targeted and the supplied proof is adequate, but the PR still needs maintainer/security judgment because it makes true missing Claude local credentials hash as a stable synthetic identity instead of invalidating auth state.

## What This Changes

The PR changes Claude CLI and token auth-profile auth-epoch hashing to use identity-only material, adds a Claude null-read fallback, bumps the epoch version to 5, and adds regression tests plus a changelog entry.

## Best Possible Solution

Keep the identity-only token/profile hashing and epoch-version bump, but handle full null reads with an explicit approved policy, such as a stale-cache grace for transient parse failures without making durable credential absence indistinguishable from a valid identity-less Claude credential.

## Reproduction Assessment

Yes, at source level. Current main can parse a partial Claude credential as `type: "token"`, hashes token material into the epoch, and same-version epoch drift resets reusable sessions; I did not reproduce the live macOS keychain timing.

## Solution Assessment

No, not quite. Identity-only hashing is the narrow maintainable fix, but the unconditional synthetic null fallback is not the best landing shape unless maintainers approve the credential-absence policy and coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Preserve invalidation for absent Claude credentials:** `src/agents/cli-auth-epoch.ts:167-173`
  - body: The new fallback hashes a synthetic Claude OAuth identity when the credential read returns null. That covers transient keychain parse failures, but null also represents a removed or missing credential; with a same-version binding, resolving the same provider-keyed epoch can keep reusing an old Claude session after local auth disappears instead of invalidating on auth-state loss. Please distinguish transient read failure from true absence, or get an explicit policy decision with coverage.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff has no supply-chain change, but the synthetic Claude null fallback can weaken auth-state invalidation when credentials are truly absent.

  - body: Hashing a synthetic Claude OAuth identity for null reads can keep the same auth epoch after local credential removal or unreadable credential state, allowing a reusable CLI session to survive an auth-state loss that current main would invalidate.
