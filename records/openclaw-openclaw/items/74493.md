## Summary

Keep this PR open for maintainer review and final CI. Current main still has the reported Claude token auth-epoch reset path, while the PR head implements the intended narrow fix with regression coverage and I did not find a blocking correctness issue in the diff.

## What This Changes

This PR changes CLI auth-epoch hashing so Claude token-shaped credentials, token auth profiles, and null Claude credential reads no longer hash rotating token material, bumps the epoch version to 5, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land this PR or an equivalent narrow patch after maintainer/CODEOWNER review and green required checks, keeping any parser hardening or richer Claude account identity extraction as a separate follow-up unless maintainers want to expand scope.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to return a Claude CLI credential with type "token", change only credential.token, and compare resolveCliAuthEpoch; current main changes the epoch, and same-version epoch mismatches reset sessions with reason "auth-epoch".

## Solution Assessment

Yes, with a maintainer contract decision. Identity-only auth-epoch hashing plus a version bump and focused regression tests is the narrowest maintainable fix for token-rotation resets; parser hardening is a safer follow-up, not required for the reported symptom.
