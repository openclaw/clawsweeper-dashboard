## Summary

Keep open for normal maintainer landing. Current main still has the reported token-material auth-epoch drift, and the latest PR head contains a focused fix with no blocking review findings found.

## What This Changes

This PR updates CLI auth-epoch encoding so Claude token-shaped credentials and token auth-profile credentials use identity fields, bumps the auth-epoch version, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land this focused auth-epoch fix after normal maintainer checks, while keeping file-presence drift or parser hardening in separate follow-ups if maintainers want that defense in depth.

## Reproduction Assessment

Yes. Source inspection on current main shows partial Claude credentials can parse as token credentials, token encoders include rotating token material, and same-version epoch drift resets reusable CLI sessions.

## Solution Assessment

Yes. Identity-only hashing for token-shaped credentials plus an epoch-version bump is the narrow maintainable fix and matches the existing OAuth/Gemini precedent while preserving true credential absence invalidation.
