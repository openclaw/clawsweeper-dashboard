## Summary

Keep open: current main still hashes Claude token material into the CLI auth epoch, while this PR provides a narrow fix with focused regression coverage and no blocking review findings found in this pass.

## What This Changes

The PR changes Claude CLI token and absent-credential auth-epoch hashing to identity-only behavior, drops token material from token auth-profile hashes, bumps the epoch version to 5, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land this PR or an equivalent narrow patch after required checks, keeping parser hardening and richer Claude identity extraction as separate follow-up work unless maintainers choose to expand scope.

## Reproduction Assessment

Yes, by source inspection. Current main hashes `credential.token` for Claude token credentials and token auth profiles, the Claude parser can fall through to token shape when `refreshToken` is missing, and same-version epoch drift invalidates reusable CLI sessions.

## Solution Assessment

Yes. Identity-only token hashing with an auth-epoch version bump and the null-read fallback is the narrow maintainable fix for the reported rotation/reset path; parser hardening is a plausible follow-up, not required for this symptom.
