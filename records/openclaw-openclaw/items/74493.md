## Summary

Keep open: source inspection confirms current main can flip the Claude CLI auth epoch on token-shaped or absent credential reads, and this PR is a focused fix with regression coverage and supplied real behavior proof; merge should still wait for maintainer review and clean required checks.

## What This Changes

The PR bumps the CLI auth epoch version, changes Claude CLI and token auth-profile epoch hashing to ignore rotating token material with a Claude null-credential fallback, adds regression tests, and updates the changelog.

## Best Possible Solution

Land this PR or an equivalent narrow patch after maintainer review and green required checks, leaving parser hardening or richer Claude account identity extraction for separate follow-up work.

## Reproduction Assessment

Yes, by source inspection. Current main can return a Claude `type: "token"` credential when `refreshToken` is missing, hashes the rotating token material, and resets reusable sessions when the same-version epoch changes.

## Solution Assessment

Yes. Identity-only token hashing plus an auth epoch version bump is the narrow maintainable fix for token-rotation resets, and the null fallback addresses the reviewer-confirmed full parse-failure shape flip without changing parser behavior.
