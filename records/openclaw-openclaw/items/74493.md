## Summary

Keep this PR open for maintainer handling. Current main still hashes rotating Claude token material, while the PR applies a narrow identity-only auth-epoch fix with focused regression coverage and no blocking review findings.

## What This Changes

The PR changes CLI auth-epoch hashing to ignore rotating Claude token material, adds a null-safe Claude credential fallback, bumps the auth-epoch version to 5, adds regression tests, and updates the changelog.

## Best Possible Solution

Land this PR or an equivalent narrow patch after maintainer acceptance, keeping parser hardening or richer Claude account identity extraction as separate follow-up work.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to return a Claude CLI `type: "token"` credential, change only `credential.token`, and compare `resolveCliAuthEpoch`; current main changes the epoch, and same-version epoch drift resets the reusable session.

## Solution Assessment

Yes, subject to maintainer acceptance of the token identity contract. Identity-only auth-epoch hashing plus a version bump and focused regression tests is the narrowest maintainable fix for token-rotation resets; parser hardening is not required for the reported symptom.
