## Summary

Keep open. Current main still hashes rotating Claude token-shaped credential material, while this PR head appears to apply the focused identity-only hashing fix with regression coverage and no blocking review findings; it still needs maintainer approval and merge handling.

## What This Changes

The PR changes CLI auth-epoch hashing to ignore rotating Claude token material, add a null-safe Claude credential fallback, bump the epoch version to 5, add regression tests, and update the changelog.

## Best Possible Solution

Land this PR or an equivalent narrow patch after maintainer acceptance, keeping parser hardening or richer Claude account identity extraction as separate follow-up work unless maintainers choose to expand scope.

Yes. A high-confidence source-level reproduction is to return a Claude CLI `type: "token"` credential, change only `credential.token`, and compare `resolveCliAuthEpoch`; current main changes the epoch, and same-version epoch drift resets the reusable session.

## Solution Assessment

Yes, subject to maintainer acceptance of the token identity contract. Identity-only auth-epoch hashing plus a version bump and focused regression tests is the narrowest maintainable fix for token-rotation resets; parser hardening is a follow-up, not required for the reported symptom.
