## Summary

Keep open. The PR addresses a real source-reproducible Claude CLI auth-epoch bug, but the current diff also makes all token auth-profile token swaps invisible to session reuse, which conflicts with the documented static-token invalidation boundary.

## What This Changes

The PR changes CLI auth-epoch hashing for Claude token-shaped credentials and token auth profiles, bumps `CLI_AUTH_EPOCH_VERSION`, adds regression tests, and updates the changelog.

## Best Possible Solution

Land a narrowed auth-epoch fix that keeps Claude OAuth partial reads and token refreshes stable, bumps the epoch version, and preserves generic static-token profile invalidation unless a stable Claude-derived identity boundary is explicit.

## Reproduction Assessment

Yes, source-level. Current main can parse a partial Claude OAuth credential as token-shaped, hash rotating token material into the auth epoch, and reset same-version reusable sessions when that hash changes.

## Solution Assessment

No, not as currently proposed. Identity-only Claude token-shaped local hashing is the right direction, but applying it to every token auth profile is broader than the documented static-token contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve static-token auth-profile invalidation:** `src/agents/cli-auth-epoch.ts:117-123`
