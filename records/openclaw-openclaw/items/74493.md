## Summary

Keep open. The core identity-only token hashing direction is well supported, but the added Claude `null` fallback now treats a true missing local credential the same as a valid identity-less OAuth credential, which can preserve a reusable session after credential removal/logout instead of invalidating on auth-state loss.

## What This Changes

The PR changes CLI auth-epoch hashing for Claude CLI token-shaped credentials, token auth-profile credentials, and the Claude local missing-credential fallback, bumps the epoch version to 5, adds regression tests, and updates `CHANGELOG.md`.

## Best Possible Solution

Keep the identity-only token hashing and version bump, but change the Claude null-read handling so transient parse failures do not churn sessions while true credential absence still invalidates or is covered by an explicit maintainer-approved policy and regression test.

## Reproduction Assessment

Yes from source inspection. Current main can parse partial Claude credentials as `type: "token"`, token encoders include rotating token material, and same-version auth-epoch drift resets reusable CLI sessions; I did not rerun the live macOS keychain timing race.

## Solution Assessment

No, not quite. The identity-only encoder change is the right narrow fix, but the synthetic null fallback should not make true missing Claude credentials indistinguishable from a valid identity-less local credential without a deliberate policy and coverage for logout/removal.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Preserve invalidation when Claude credentials are truly absent:** `src/agents/cli-auth-epoch.ts:167-173`
  - body: This fallback runs after `readClaudeCliCredentialsCached` was called with `allowKeychainPrompt: false`, so `null` is also the normal result for a keychain-only/no-file setup or after the local Claude credential is removed. Hashing a synthetic OAuth credential makes that true absence produce the same local fingerprint as a valid identity-less Claude OAuth read, so a same-version binding can keep reusing the old Claude session after logout/removal instead of getting the current `auth-epoch` reset. Please distinguish transient parse failure from real absence, or keep absence as `undefined` and cover the intended behavior with a regression test.
  - confidence: 0.78
