## Summary

Keep this PR open for normal maintainer review and merge. Current main still has the reported auth-epoch drift, while the PR is a focused fix with regression coverage and no concrete review finding from this pass.

## What This Changes

The PR changes Claude CLI and token auth-profile auth-epoch hashing to ignore rotating token material, adds a Claude absent-credential fallback, bumps the auth-epoch version, updates tests, and adds a changelog entry.

## Best Possible Solution

Land this PR or an equivalent narrow patch once required checks and maintainer review pass, keeping parser hardening or richer Claude account identity extraction as separate follow-up work.

## Reproduction Assessment

Yes. Source inspection on current main shows the parser can produce token-shaped Claude credentials, both relevant token encoders include rotating token material, and same-version epoch drift resets reusable CLI sessions.

## Solution Assessment

Yes. Identity-only token hashing with an auth-epoch version bump and targeted regression tests is the narrow maintainable fix; parser hardening can remain defense-in-depth follow-up work.
