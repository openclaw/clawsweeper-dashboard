## What This Changes

The branch changes CLI auth-epoch hashing so Claude token-shaped credentials and token auth-profile credentials use identity fields instead of token material, bumps the auth-epoch version, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land this focused auth-epoch fix after normal maintainer checks; keep file-presence or parser-hardening work in separate follow-ups such as https://github.com/openclaw/openclaw/pull/80265 if maintainers still want it.

## Reproduction Assessment

Yes. Source inspection on current main shows partial Claude credentials can parse as token credentials, token encoders include rotating token material, and same-version epoch drift resets reusable CLI sessions; I did not run the live macOS race.

## Solution Assessment

Yes. Identity-only hashing for token-shaped credentials plus an epoch-version bump is the narrow maintainable fix and matches the existing OAuth/Gemini precedent while preserving true credential absence invalidation.
