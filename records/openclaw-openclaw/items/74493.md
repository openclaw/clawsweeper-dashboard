## Summary

Keep open: current main still has the token-inclusive auth-epoch behavior reported in the linked issue, and this PR is a narrow bug fix with no blocking review findings from this pass.

## What This Changes

The PR changes CLI auth-epoch hashing for Claude token credentials and token auth-profile credentials, bumps the auth epoch version to 5, adds regression tests, and updates the changelog.

## Best Possible Solution

Land this PR or an equivalent narrow patch after normal maintainer review and CI, leaving parser hardening or richer Claude account identity extraction to a separate follow-up if maintainers want it.

## Reproduction Assessment

Yes, by source inspection: return a Claude CLI `type: "token"` credential, change only `credential.token`, and current main changes the auth epoch. Same-version epoch drift then invalidates reusable sessions with `auth-epoch`, and the linked issue plus review comments add empirical macOS keychain-race evidence.

## Solution Assessment

Yes. Identity-only token hashing, a version bump, a null-read fallback, and focused regression tests are the narrow maintainable fix for token-rotation auth-epoch resets without changing parser behavior or session consumers.
