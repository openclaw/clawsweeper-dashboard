## Summary

Keep open: current main still contains the auth-epoch token-rotation reset path, and this PR is a focused fix with matching regression coverage and no concrete review findings from the diff.

## What This Changes

This PR changes Claude CLI auth-epoch hashing for token-shaped credentials, token auth profiles, and null local reads, bumps `CLI_AUTH_EPOCH_VERSION` to 5, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land this PR or an equivalent narrow patch that keeps token rotation and transient Claude keychain read failures from changing auth epochs, while leaving parser hardening or richer Claude account identity extraction as a follow-up unless maintainers choose to expand scope.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to mock a Claude CLI `type: "token"` credential, change only `credential.token`, compare `resolveCliAuthEpoch`, and then observe that same-version epoch mismatch maps to `invalidatedReason: "auth-epoch"` in session reuse.

## Solution Assessment

Yes, with maintainer acceptance of the identity-only token contract. The patch fixes the reset loop at the auth-epoch boundary, bumps the version for migration, and adds focused tests; parser hardening is useful defense-in-depth but not required for this symptom.
