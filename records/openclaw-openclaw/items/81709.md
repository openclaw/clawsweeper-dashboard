## Summary

Keep this PR open: the no-run-id direction is reasonable, but the current patch introduces contributor-specific fallback text into generic user-visible task delivery and still lacks real behavior proof.

## What This Changes

The PR rewrites background task terminal and follow-up copy to remove run IDs and ACP fallback wording, then updates task policy and registry tests.

## Best Possible Solution

Use neutral user-facing background work copy without run-id or ACP boilerplate, keep focused formatter and registry coverage, and add one redacted real delivery sample before merge.

## Reproduction Assessment

Yes for the introduced PR bug: the PR head formatter returns hard-coded `Moeed` fallback strings, and current delivery code sends formatter output to channel messages or session events. I did not run tests because this review is read-only.

## Solution Assessment

No, not as written: removing raw run metadata is a reasonable fix direction, but generic task delivery must not include contributor-specific instructions. The narrower maintainable fix is neutral fallback copy or no fallback instruction, plus real delivery proof.
