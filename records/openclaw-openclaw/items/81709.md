## Summary

Keep open: the PR is a plausible cleanup for raw task completion copy, but it cannot merge as-is because fallback task messages hard-code a contributor name into generic delivery text and the external PR has no real behavior proof.

## What This Changes

This PR rewrites task terminal and blocked-follow-up messages to use “Background work” copy, remove run-id snippets and the ACP fallback title, and updates task policy/registry tests.

## Best Possible Solution

Keep the no-run-id/user-facing copy direction, but make fallback wording generic and verify the real terminal or chat delivery path with redacted after-fix output before merge.

## Reproduction Assessment

Yes for the introduced PR bug: the diff shows `formatTaskTerminalMessage` returning strings containing `Moeed`, and current delivery code sends that formatter output to requester channels or session events. I did not run tests because this review is read-only.

## Solution Assessment

No, not as written: removing raw run metadata is a reasonable direction, but generic OpenClaw task delivery must not address one contributor by name. The narrower fix is to use neutral fallback copy or omit the instruction entirely.
