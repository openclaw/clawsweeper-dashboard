## Summary

This PR should stay open: the intended cleanup of raw background task notices is plausible, but the current patch introduces contributor-specific fallback text into generic task delivery, and the external PR still has only unit-test output instead of real after-fix behavior proof.

## What This Changes

The PR changes background task terminal and follow-up wording in `src/tasks/task-executor-policy.ts` and updates task policy/registry tests to remove run-id snippets and ACP fallback wording.

## Best Possible Solution

Keep the user-facing no-run-id wording direction, replace the personal-name fallback with neutral task-delivery copy, and require redacted real terminal or chat output showing the changed behavior before merge.

## Reproduction Assessment

Yes for the introduced PR bug from source inspection: the diff adds contributor-specific `Moeed` fallback strings, and current task delivery sends formatter output to requester channels or session events. I did not run tests because this review is read-only.

## Solution Assessment

No, not as written: removing raw run metadata is a reasonable direction, but generic OpenClaw task delivery must not address one contributor by name. The narrower fix is neutral fallback wording or omitting the instruction when no summary is present.
