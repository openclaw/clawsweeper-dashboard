## Summary

Close as implemented: the defaults-only heartbeat inheritance bug is fixed in current main and shipped in v2026.5.7; zero-config default-agent heartbeat is now documented/tested as default-on policy, and the remaining mixed defaults-plus-per-agent policy question is tracked by https://github.com/openclaw/openclaw/issues/78710 and https://github.com/openclaw/openclaw/issues/46101.

## What This Changes

The issue asks heartbeat enablement to respect `agents.defaults.heartbeat` instead of falling back to only the default agent across defaults-only, zero-config, and mixed per-agent configurations.

## Best Possible Solution

Keep the shipped defaults-only inheritance fix, and route mixed per-agent inheritance or warning changes through https://github.com/openclaw/openclaw/issues/78710 and https://github.com/openclaw/openclaw/issues/46101.

## Reproduction Assessment

No for the defaults-only bug: current source and regression tests show `agents.defaults.heartbeat` enables and schedules both `main` and `ops`. The mixed defaults-plus-per-agent behavior is source-reproducible, but current docs/tests define it as an allowlist policy and it is tracked separately.

## Solution Assessment

Yes for closing this issue: current main fixes the actionable defaults-only inheritance mismatch in both status and scheduler paths, and v2026.5.7 shipped it. A broader inheritance-policy change should be handled in the narrower open follow-up rather than keeping this mixed historical issue open.
