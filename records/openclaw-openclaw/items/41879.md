## Summary

Current `main` fixes the reported defaults-only multi-agent heartbeat bug: `agents.defaults.heartbeat` now enables all configured agents in both the summary helper and scheduler. The remaining zero-config default-agent behavior is still present, but it matches the current heartbeat docs and regression tests, so it should not keep this defaults-inheritance issue open.

## What This Changes

The issue requests heartbeat enablement to respect `agents.defaults.heartbeat` across configured agents and to stop implicitly enabling heartbeat for the default agent when no heartbeat config exists.

## Best Possible Solution

Keep the shipped defaults-only inheritance fix and leave any broader default-on or empty-config opt-out policy change to a focused heartbeat-disable issue rather than reopening this defaults-scope bug.

## Reproduction Assessment

Yes. Current-main source and tests show the defaults-only non-default-agent bug no longer reproduces, while the zero-config default-agent fallback still reproduces as documented default-on behavior.

## Solution Assessment

Yes for closing this issue. The current implementation fixes the defaults-inheritance bug narrowly; changing zero-config default-on semantics would be a separate product behavior change, not a cleanup repair for this issue.
