## Summary

Keep open for maintainer judgment: current main fixes the defaults-only inheritance behavior, but the remaining request to disable heartbeats when no heartbeat config exists conflicts with current docs and tests that define default-agent heartbeats as enabled by default.

## What This Changes

The issue asks for `isHeartbeatEnabledForAgent` to honor `agents.defaults.heartbeat` across agents and to disable heartbeats for every agent when no heartbeat config exists.

## Best Possible Solution

Keep the defaults-only inheritance behavior on main, and have a maintainer explicitly decide whether zero-config heartbeats should remain the documented default or become opt-in with coordinated docs and tests.

## Reproduction Assessment

Yes for the remaining zero-config behavior: source inspection shows current main still enables the default agent when no heartbeat config exists. The defaults-only inheritance bug no longer reproduces from source and test inspection because defaults now enable all listed agents.

## Solution Assessment

No for an automatic fix. Disabling zero-config heartbeats would conflict with current docs/tests and should be a maintainer product decision, while the narrow defaults-inheritance fix is already present on main.
