## Summary

Current main implements the actionable defaults-only heartbeat bug: `agents.defaults.heartbeat` now enables all configured agents for summaries and scheduling, with regression coverage. The no-config default-agent fallback remains documented and tested default-on behavior, so that part is not an outstanding bug against the current contract.

## What This Changes

Request to make heartbeat enablement respect `agents.defaults.heartbeat` across agents and stop default-agent heartbeat when no heartbeat config exists.

## Best Possible Solution

Keep the shipped defaults-as-global implementation; any switch from default-on heartbeat to default-off should be tracked separately as a product-policy change.

## Reproduction Assessment

No. The defaults-only multi-agent failure no longer reproduces from source inspection: current tests and scheduler code cover both `main` and `ops` under `agents.defaults.heartbeat`; the no-config fallback still exists but is documented default-on behavior.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix for the source-defined bug: summaries and scheduling honor global defaults while preserving explicit per-agent scoping and the documented default-on policy.
