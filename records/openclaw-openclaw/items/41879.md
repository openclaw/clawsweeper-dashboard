## Summary

Close as implemented on current main: the actionable `agents.defaults.heartbeat` inheritance bug is fixed in the helper, scheduler, and regression coverage; the zero-config default-agent heartbeat behavior is now documented and tested as the product’s default-on policy rather than a remaining bug in this tracker.

## What This Changes

The issue asks for heartbeat enablement to honor `agents.defaults.heartbeat` across agents and to stop enabling the default agent when no heartbeat config exists.

## Best Possible Solution

Keep the current defaults-only inheritance fix, and handle any future zero-config/default-on behavior change as a separate maintainer product-policy item.

## Reproduction Assessment

No for the actionable defaults-inheritance bug: current-main source and regression tests show defaults-only configs enable and schedule both `main` and `ops`. The no-config fallback still exists, but current docs and tests define it as default-on behavior disabled with `0m`, not a failing current contract.

## Solution Assessment

Yes for the actual bug: fixing both the enablement helper and scheduler resolver is the narrow maintainable solution, with regression coverage. Changing zero-config behavior would be a separate product-policy change.
