## Summary

Current main and the shipped release now honor `agents.defaults.heartbeat` for defaults-only multi-agent configs; the remaining default-agent heartbeat behavior with no explicit config is documented as current product behavior rather than a source-level bug.

## What This Changes

The issue asks OpenClaw to respect `agents.defaults.heartbeat` in heartbeat enablement and to stop implicitly enabling heartbeat for the default agent when no heartbeat config is present.

## Best Possible Solution

Keep the shipped global-defaults fix; use https://github.com/openclaw/openclaw/issues/46101 for the remaining doctor-warning work around mixed default/per-agent scoping.

## Reproduction Assessment

No. The defaults-only multi-agent failure no longer reproduces from source inspection and checked-in tests; the no-config default-agent fallback is still present, but current docs describe heartbeat as enabled by default and configurable with `0m` to disable.

## Solution Assessment

Yes. The merged implementation fixes the documented defaults-only scope without changing the documented explicit-per-agent scoping rule or the default heartbeat policy.
