Keep this PR open: it addresses a real webchat `/tts audio` rendering gap and supplies real behavior proof, but the latest head still broadens `chat.history` by preserving small non-assistant base64 audio blocks when no inline budget is passed.

## Solution Assessment

No, not yet. Structured audio normalization plus CSP support is a plausible narrow fix, but the history sanitizer needs to keep the inline-audio preservation scoped to assistant playback and retain redaction for other roles.
