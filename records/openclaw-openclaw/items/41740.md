## What This Changes

The issue asks for Discord/native exec approval Gateway clients to pass the resolved local self-signed TLS fingerprint into GatewayClient.

## Best Possible Solution

Refresh and land https://github.com/openclaw/openclaw/pull/80204 as the shared bootstrap TLS fingerprint fix, then close this issue from merged commit proof.

## Reproduction Assessment

Yes, at source level: bootstrap consumers construct GatewayClient without tlsFingerprint, while GatewayClient only enables the pinned self-signed WSS trust path when that option is present. I did not run a live Discord flow in this read-only review.

## Solution Assessment

Yes, the best fix direction is shared bootstrap TLS fingerprint resolution and pass-through; a Discord-only fix would leave ACP/MCP and other native approval bootstrap clients inconsistent.
