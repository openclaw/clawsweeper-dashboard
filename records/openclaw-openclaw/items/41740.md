## Summary

Keep open: current main and the latest release still omit TLS fingerprint propagation from bootstrap-created approval Gateway clients, and the related open fix at https://github.com/openclaw/openclaw/pull/80204 is still the canonical implementation path.

## What This Changes

Discord exec approvals should pass the local Gateway TLS fingerprint when connecting to a local self-signed WSS Gateway.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80204 or an equivalent shared bootstrap TLS fingerprint propagation fix, then close this issue with merge proof for native approval clients against a local self-signed WSS Gateway.

## Reproduction Assessment

Yes. Source inspection shows the Discord approval path reaches shared bootstrap-created Gateway clients that cannot receive tlsFingerprint, while GatewayClient only enables pinned self-signed WSS handling when that option is supplied.

## Solution Assessment

Yes for the shared bootstrap direction. Another Discord-only patch would miss ACP/MCP/native approval clients, while the related open PR moves the existing callGateway TLS precedence into a shared resolver.
