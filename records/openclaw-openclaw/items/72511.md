## Summary

Keep open: the PR targets a valid transcript-growth bug, but the current branch is not mergeable because the role-plus-content replay key can drop legitimate repeated turns and the guard misses the current file-state JSONL rewrite path on main.

## What This Changes

The PR adds SHA-256 identity deduplication to transcript rewrite suffix replay plus regression tests for duplicate user messages, compaction entries, and bootstrap custom entries.

## Best Possible Solution

Refactor replay idempotency into a shared helper used by both SessionManager and TranscriptFileState rewrites, with an identity narrow enough to preserve distinct turns and real JSONL regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main's rewrite loops re-append every suffix entry, and the PR regression is source-reproducible because two same-role, same-content but distinct messages share the new key and the later one is skipped. I did not run Vitest because pnpm is not installed in this checkout.

## Solution Assessment

No. The rewrite boundary is a reasonable hardening point, but this PR's identity key is too broad and the guard is not applied to the persisted file-state rewrite path now used on main.
