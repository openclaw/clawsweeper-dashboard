## Summary

Keep open. Current main still drops or genericizes Feishu/Lark custom sticker context before the agent body is built, while the Feishu docs advertise sticker receive support; this remains a valid bundled Feishu plugin behavior gap rather than a cleanup close.

## What This Changes

The issue requests preserving Feishu/Lark bracket-style custom stickers by converting them to Unicode emoji equivalents or visible text in agent-visible inbound messages.

## Best Possible Solution

Preserve verified custom-sticker labels in the Feishu plugin inbound parser, keep unknown stickers visible as stable bracket/text fallback, and add focused parser tests for mixed text plus custom sticker metadata, standalone stickers, rich-text emotion elements, Chinese labels, and unknown-label fallback behavior.

## Reproduction Assessment

Unclear for the exact live Feishu payload because the report does not include raw event JSON. Source-level reproduction is clear: current main returns only `parsed.text` for text messages and `<media:sticker>` for standalone sticker messages.

## Solution Assessment

Yes for the implementation boundary: this should be solved in the Feishu plugin parser, not core. The exact mapping policy is unclear until maintainers or the reporter confirm Feishu's raw custom-sticker fields, so label/text preservation should come before broad Unicode conversion.
