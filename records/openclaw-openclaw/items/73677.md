## Summary

Keep this issue open. Current main still does not preserve Feishu/Lark custom sticker labels in the agent-visible inbound body: text messages keep only `text`, standalone sticker messages become a generic sticker media marker, and existing rich-text emotion handling only helps for supported `post` emotion elements. The recent maintainer mentions also make this active Feishu plugin follow-up rather than cleanup material.

## What This Changes

The issue asks OpenClaw to preserve Feishu/Lark custom sticker meaning in inbound agent-visible text, either by mapping stickers to Unicode emoji or by retaining bracket-text labels such as `[OK]` and `[完成]`.

## Best Possible Solution

Implement this inside the Feishu plugin inbound parser after confirming the raw payload shape: preserve verified custom-sticker labels or stable bracket-text fallbacks, keep unknown sticker metadata visible as text, and add focused tests for mixed text plus sticker metadata, standalone sticker messages, rich-text emotion elements, Chinese labels, and unknown-label fallback behavior.

## Reproduction Assessment

Unclear for a high-confidence live Feishu reproduction because the issue does not include raw event JSON. Source-level reproduction is clear: text events return only `parsed.text`, standalone sticker events render as `<media:sticker>`, and that parsed content is used directly for the agent body.

## Solution Assessment

Yes, the narrow maintainable boundary is the Feishu plugin parser, not OpenClaw core. A verified label/text-preservation fallback is safer than broad Unicode conversion until the plugin has a trusted Feishu sticker-name mapping source.
