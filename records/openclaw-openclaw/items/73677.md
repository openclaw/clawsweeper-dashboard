## Summary

Keep open: current main still treats standalone Feishu/Lark stickers as generic media placeholders, while rich-text emotion rendering is a separate path. The issue is recent, Feishu plugin owners were already pulled into the discussion, and the remaining blocker is confirming the real custom-sticker payload plus the bracket-text versus Unicode mapping policy.

## What This Changes

Request Feishu/Lark custom stickers such as [OK] and [完成] be preserved for agents as Unicode equivalents or bracket-text labels.

## Best Possible Solution

Capture a real Feishu custom-sticker event, then update the Feishu parser to preserve verified sticker labels in agent context before considering any maintained Unicode mapping.

## Reproduction Assessment

Unclear for the exact mixed `好的[OK]` case: source inspection shows standalone `message_type: "sticker"` events become `<media:sticker>`, but the bracket-label loss needs raw Feishu event JSON. That is enough to keep this open, not enough for a high-confidence automated bug fix.

## Solution Assessment

Unclear: the Feishu parser is the right boundary, but Unicode mapping needs payload and product confirmation. The safer first implementation target is verified bracket-text preservation while keeping media attachment behavior.
