## Summary

Keep open: current main still reduces Feishu/Lark sticker messages to generic media context, and the issue is recent with Feishu plugin owners already pulled into the discussion. The remaining uncertainty is the real custom-sticker payload and whether OpenClaw should preserve bracket text first or maintain a Unicode mapping.

## What This Changes

The issue requests preserving Feishu/Lark custom sticker semantics by converting custom sticker markers into Unicode emoji or bracket-text for the agent.

## Best Possible Solution

Capture a real Feishu custom-sticker payload, then update the Feishu parser to preserve verified text labels in agent context before considering a maintained Unicode mapping.

## Reproduction Assessment

Unclear for the exact reported mixed `好的[OK]` case: current source gives a clear standalone `message_type: "sticker"` path to `<media:sticker>`, but the bracket-label loss needs raw Feishu event JSON. The source path is enough to keep this open, not enough for a high-confidence automated bug fix.

## Solution Assessment

Unclear on mapping details: the correct boundary is the Feishu plugin parser/body path, but Unicode conversion needs product and payload confirmation. The safer first step is verified bracket-text preservation while keeping the media attachment behavior.
