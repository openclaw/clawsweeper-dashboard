## Summary

Keep open: current main still reduces standalone Feishu/Lark sticker messages to generic media context and no duplicate or merged fix surfaced. The remaining work needs real Feishu payload evidence plus a maintainer/product choice on preserving bracket text versus maintaining a Unicode mapping.

## What This Changes

The issue requests that Feishu/Lark custom stickers such as `[OK]`, `[完成]`, and `[调皮]` be preserved for the agent as Unicode emoji or text instead of disappearing from context.

## Best Possible Solution

Capture a real Feishu custom-sticker event payload, then update the Feishu parser to preserve verified sticker labels as text before considering any owner-approved Unicode mapping.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for the exact mixed `好的[OK]` report because raw Feishu event JSON is missing; source inspection does show standalone sticker events are reduced to `<media:sticker>` and label-like fields are not extracted.

## Solution Assessment

Is this the best way to solve the issue? Unclear: the Feishu plugin parser is the right boundary, but the maintainable first step is payload-backed bracket/text preservation rather than guessing a Unicode mapping table.
