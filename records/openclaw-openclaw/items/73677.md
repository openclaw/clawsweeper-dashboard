## Summary

Keep open: current main still treats standalone Feishu/Lark sticker messages as generic media and does not preserve custom bracket sticker text for the agent. The remaining work needs real Feishu payload evidence plus an owner/product choice on bracket-text preservation versus a Unicode mapping table.

## What This Changes

The issue asks the Feishu/Lark channel to convert custom bracket-style stickers into Unicode emoji or a textual representation visible to the agent.

## Best Possible Solution

Capture real Feishu custom-sticker payloads, then update the Feishu parser to preserve verified sticker labels as agent-visible text before adding any owner-approved Unicode mapping.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for the exact mixed `好的[OK]` report because raw Feishu event JSON is missing; source inspection does show standalone sticker events still become `<media:sticker>` without label extraction.

## Solution Assessment

Is this the best way to solve the issue? Unclear: the Feishu parser is the right boundary, but a payload-backed text-preservation contract is safer than guessing a Unicode mapping table for localized custom stickers.
