## Summary

Keep open. Current main still does not preserve Feishu/Lark custom sticker labels in the agent-visible body: sticker messages are reduced to media placeholders, while related emoji reaction work does not cover inbound sticker content.

## What This Changes

Request to make inbound Feishu/Lark custom stickers visible to the agent by converting bracket-style stickers to Unicode emoji or text labels.

## Best Possible Solution

Fix this in the Feishu plugin after collecting a real custom-sticker payload: preserve verified sticker text labels in inbound parsing while keeping media attachments, then add focused parser and body tests.

## Reproduction Assessment

Unclear: there is a high-confidence source-level path showing standalone `sticker` messages become `<media:sticker>`, but the report does not include raw Feishu payload JSON for the mixed text plus bracket-sticker case.

## Solution Assessment

Yes on boundary, unclear on mapping details: the maintainable solution belongs in `extensions/feishu`, with bracket-text preservation from verified payload fields before any Unicode mapping policy is added.
