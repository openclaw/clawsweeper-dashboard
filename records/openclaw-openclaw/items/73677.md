## Summary

Keep open. Current main still does not preserve Feishu/Lark custom sticker labels for agent-visible inbound content, and the issue is recent with Feishu plugin owners already brought into the discussion.

## What This Changes

Request to convert Feishu/Lark bracket-style custom stickers into Unicode emoji or text so agents can see the missing inbound context.

## Best Possible Solution

Fix this in the Feishu plugin parser after collecting a real custom-sticker payload: preserve verified text-like sticker labels in inbound content, keep the existing media attachment path, and add focused parser/body tests.

## Reproduction Assessment

Unclear: the current source gives a high-confidence path for standalone `sticker` messages becoming `<media:sticker>`, but the mixed text-plus-custom-sticker label-loss case needs raw Feishu event JSON for high-confidence reproduction.

## Solution Assessment

Yes on boundary, unclear on mapping details: the maintainable fix belongs in `extensions/feishu`, with bracket-text preservation first and Unicode mapping only after owner confirmation.
