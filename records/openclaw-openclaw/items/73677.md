## Summary

Keep open. Current main still does not reliably preserve Feishu/Lark custom sticker labels in the agent-visible inbound body: text events return only `parsed.text`, standalone sticker events become a generic media placeholder, and the existing rich-text emotion renderer only helps when Feishu supplies supported emotion fields.

## What This Changes

Preserve Feishu/Lark custom bracket-style stickers in inbound messages by converting them to Unicode emoji equivalents or stable text labels.

## Best Possible Solution

Implement the behavior in the Feishu plugin inbound parser: preserve verified sticker labels or bracket-text fallbacks from raw event payloads, keep unknown stickers visible as stable text, and add focused tests for mixed text plus sticker metadata, standalone sticker messages, rich-text emotion elements, Chinese bracket labels, and unknown-label fallback behavior.

## Reproduction Assessment

Unclear for the exact live Feishu event because the report does not include raw payload JSON. Source-level reproduction is clear: current main returns only `parsed.text` for text messages and `<media:sticker>` for standalone sticker messages.

## Solution Assessment

Yes, a Feishu plugin parser fix is the right boundary and should not require a core channel API change. The maintainable path is label/text preservation first, with Unicode mapping only for verified sticker names.
