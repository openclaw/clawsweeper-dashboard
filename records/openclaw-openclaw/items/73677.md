## Summary

Keep open. Current main still reduces standalone Feishu/Lark sticker messages to generic media placeholders and does not recover bracket labels before building the agent-visible body.

## What This Changes

The issue asks the Feishu/Lark channel to preserve custom sticker intent by converting bracket-style stickers such as [OK] or [完成] to Unicode emoji or text in the agent-visible message body.

## Best Possible Solution

Fix this in `extensions/feishu` after collecting a real custom-sticker payload: preserve verified text-like sticker labels in the inbound parser while keeping the existing media attachment path, then add focused parser/body tests.

## Reproduction Assessment

Unclear: there is a high-confidence source-level path showing standalone `sticker` messages become `<media:sticker>`, but the issue does not include the raw Feishu payload for the [OK]/[完成] label-loss case. That missing sample blocks high-confidence live reproduction.

## Solution Assessment

Yes on boundary, unclear on mapping details: the maintainable fix belongs in the Feishu plugin parser, not core. The safer first step is bracket-text preservation from verified payload fields, with Unicode mapping only after owner confirmation.
