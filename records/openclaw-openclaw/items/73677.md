## Summary

Keep open: current main still reduces Feishu/Lark `sticker` messages to generic media context, and the issue is recent with Feishu plugin owners already pulled into the timeline.

## What This Changes

The issue asks OpenClaw to preserve Feishu/Lark custom sticker meaning by converting bracket-style stickers such as `[OK]` or `[完成]` to Unicode emoji or text visible to the agent.

## Best Possible Solution

Fix this inside the Feishu plugin parser after capturing a real custom-sticker payload: preserve verified bracket labels as text first, keep the media attachment path, and add focused parser/body tests; Unicode mapping can follow once mappings are confirmed.

## Reproduction Assessment

Unclear: source gives a high-confidence standalone path where `message_type: "sticker"` becomes `<media:sticker>`, but the reported mixed `好的[OK]` label-loss case still needs raw Feishu event JSON or a live sample.

## Solution Assessment

Yes on boundary, unclear on mapping details: the maintainable fix belongs in `extensions/feishu`, with bracket-text preservation before guessing Unicode equivalents.
