## Summary

Keep this issue open. Current main still preserves Feishu/Lark stickers only as generic media placeholders, with no implemented path that carries custom bracket-style sticker labels or Unicode equivalents into the agent body.

## What This Changes

The issue requests preserving Feishu/Lark custom sticker meaning by converting bracket-style stickers such as [OK], [完成], and [调皮] into Unicode emoji or bracket-text for the agent.

## Best Possible Solution

Fix this in the bundled Feishu plugin inbound parser after collecting a real custom-sticker event sample: preserve verified text-like labels as bracket text, keep the existing media attachment path, and add focused parser/body tests.

## Reproduction Assessment

Unclear: there is a high-confidence source-level path showing `message_type: sticker` becomes `<media:sticker>`, but the report lacks raw Feishu event JSON for the exact [OK]/[完成] label-loss case. That missing payload blocks a high-confidence live reproduction.

## Solution Assessment

Yes on boundary, unclear on mapping details: the maintainable fix belongs in `extensions/feishu`, not core. The safest approach is verified bracket-text preservation first, with Unicode conversion only after a stable Feishu sticker-name source and ownership policy are confirmed.
