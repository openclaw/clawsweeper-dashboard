## Summary

Keep this open. Current main still loses Feishu/Lark custom-sticker meaning before the agent body is built, and the issue is recent with maintainer/plugin-owner attention rather than stale cleanup material.

## What This Changes

Request that Feishu/Lark inbound custom stickers such as [OK] and [完成] be preserved for the agent as Unicode emoji or bracket-text instead of being dropped.

## Best Possible Solution

Implement this in the Feishu plugin inbound parser after confirming the raw custom-sticker payload shape: preserve a verified label or bracket-text fallback for custom stickers, keep unknown sticker metadata agent-visible, and add focused parser/body tests for mixed text plus custom stickers, standalone stickers, Chinese labels, rich-text emotion elements, and unknown-label fallback behavior.

## Reproduction Assessment

Unclear for a high-confidence live Feishu reproduction because the issue lacks raw event JSON or logs. Source-level reproduction is clear from current main: text parsing only keeps `parsed.text`, sticker messages become `<media:sticker>`, and that parsed content is used directly for the agent body.

## Solution Assessment

Yes, the best boundary is the bundled Feishu plugin parser rather than OpenClaw core. The safest first fix is verified text preservation; broad Unicode conversion should wait for a maintained Feishu sticker-name mapping source.
