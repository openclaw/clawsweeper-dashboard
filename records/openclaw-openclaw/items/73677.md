## Summary

Keep open: current main still loses Feishu/Lark custom-sticker semantics by treating `sticker` messages as generic media, and the issue is recent with Feishu plugin owners already pulled into the timeline.

## What This Changes

Request to preserve Feishu/Lark custom sticker meaning by converting bracket-style stickers to Unicode emoji or visible text in agent-visible inbound content.

## Best Possible Solution

Fix this inside the Feishu plugin parser after capturing a real custom-sticker payload: preserve verified bracket labels as text first, keep the media attachment path, and add focused parser/body tests; Unicode mapping can follow once the mapping policy is confirmed.

## Reproduction Assessment

Unclear: current source gives a high-confidence path for standalone `message_type: "sticker"` events becoming `<media:sticker>`, but the reported mixed `好的[OK]` label-loss case needs raw Feishu event JSON for a high-confidence reproduction.

## Solution Assessment

Yes on boundary, unclear on mapping details: the maintainable fix belongs in `extensions/feishu`, with bracket-text preservation before guessing Unicode equivalents.
