## Summary

Keep this open. Current main still does not preserve Feishu/Lark custom sticker labels in agent-visible inbound text: plain text parsing only returns the text field, standalone stickers become a generic media marker, and the existing rich-text emotion handling only helps when Feishu sends supported post emotion fields.

## What This Changes

The issue requests preserving Feishu/Lark custom sticker context by converting stickers to Unicode emoji or bracket-text representations in agent-visible inbound messages.

## Best Possible Solution

Implement this inside the Feishu plugin inbound parser after confirming raw event shapes: preserve verified custom-sticker labels as bracket text or a stable textual fallback, add Unicode mapping only from a trusted table, and cover mixed text, standalone sticker, rich-text emotion, Chinese label, and unknown-label cases with focused tests.

## Reproduction Assessment

Unclear for a high-confidence live reproduction because the issue lacks raw Feishu event JSON. Source-level reproduction is clear for current main: text events return only `parsed.text`, standalone stickers render as `<media:sticker>`, and that parsed content is passed directly into the agent body.

## Solution Assessment

Yes, the narrow maintainable boundary is the Feishu plugin inbound parser, not OpenClaw core. A verified text-preservation fallback is safer than broad Unicode conversion until maintainers confirm the payload fields and mapping source.
