## Summary

Keep open: current main still treats Feishu/Lark stickers as generic media and does not preserve the bracket-style label or provide a Unicode mapping, while the exact mixed text-plus-sticker payload still needs plugin-owner confirmation.

## What This Changes

Request to convert Feishu/Lark custom stickers such as `[OK]`, `[完成]`, and `[调皮]` into Unicode emoji or preserved text for agent context.

## Best Possible Solution

Capture a real Feishu custom-sticker payload, then update the Feishu parser to preserve verified sticker text in agent context before considering a maintained Unicode mapping.

## Reproduction Assessment

Unclear. Source inspection shows standalone `message_type: "sticker"` currently becomes `<media:sticker>`, but the reported mixed `好的[OK]` loss needs the raw Feishu event payload to reproduce with high confidence.

## Solution Assessment

Unclear. The Feishu parser is the right boundary, but bracket-text preservation is safer than inventing Unicode mappings until maintainers confirm real payload fields and mapping policy.
