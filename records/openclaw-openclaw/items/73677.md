## Summary

Keep this open. Current main still does not preserve Feishu/Lark custom bracket-style sticker labels in the agent-visible inbound body: text messages return only `parsed.text`, standalone sticker messages become a generic media placeholder, and the agent body uses that parsed content without a later sticker-label recovery layer.

## What This Changes

Request to preserve Feishu/Lark custom bracket-style stickers in inbound messages by converting them to Unicode emoji or text labels for the agent.

## Best Possible Solution

Implement this inside the Feishu plugin inbound parser by preserving verified sticker labels or stable text fallbacks from raw event payloads first, then add Unicode mappings only for documented and tested sticker names. Cover mixed text plus custom sticker metadata, standalone sticker messages, rich-text emotion elements, Chinese bracket labels, and unknown-label fallback behavior.

## Reproduction Assessment

Unclear for the exact live Feishu event because the issue does not include raw payload JSON. Source-level reproduction is clear: current main returns only `parsed.text` for text messages, renders standalone sticker messages as `<media:sticker>`, and skips empty text when no media is resolved.

## Solution Assessment

Yes, a Feishu plugin parser fix is the right maintainable direction and should not require a core channel API change. The safer alternative to broad Unicode conversion is preserving raw verified labels first, with a documented mapping table added only after Feishu payload behavior is confirmed.
