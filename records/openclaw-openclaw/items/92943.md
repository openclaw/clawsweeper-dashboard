Overall correctness: patch is incorrect

Full review comments:

- **[P2] Keep replacement refresh tied to dirty cleanup:** `extensions/memory-core/src/memory/manager.ts:664`
  - body: `refreshDatabaseHandleIfExternallyReplaced()` returns true only the first time a replaced DB is observed. Calling it here means the real `memory_search` route, which searches before checking status, updates `indexFileSignature` before `refreshIndexIdentityDirty()` can see `refreshedDatabase === true`; a manager that was dirty from missing identity can then keep `this.dirty`/retry state and start another sync even though the replacement DB is valid. Let the identity-refresh path own the reopen or carry a pending-refresh flag into the cleanup path, and cover search-first recovery in the regression test.
  - confidence: 0.92
