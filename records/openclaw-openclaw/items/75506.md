## Summary

Keep open. Current main still has a credible CLI warning path for Telegram in packaged installs: control-plane/doctor lookup can evaluate the full bundled Telegram channel plugin while explicitly bypassing staged runtime-dependency preparation, and that full plugin path still reaches bare `grammy` imports.

## What This Changes

The issue requests that CLI commands stop emitting missing `grammy` warnings when Telegram runtime dependencies are correctly staged outside the package root.

## Best Possible Solution

Keep the issue open and make CLI/control-plane paths use manifest, setup, or other read-only Telegram surfaces without importing send/monitor runtime code; true Telegram runtime execution should load through the prepared bundled runtime root.

## Reproduction Assessment

Yes. The issue gives concrete package-install commands and dependency state, and current source still shows a configured Telegram doctor/full-load path that bypasses staged runtime deps while reaching bare `grammy` imports.

## Solution Assessment

No current fix is present. The narrow maintainable fix is to short-circuit read-only/doctor adapter lookup before full-plugin fallback and move Telegram send/monitor value imports behind runtime-only lazy boundaries where needed.
