## Summary

Keep open. Current main still has the credible packaged-install warning path: a bundled-channel full plugin load disables runtime-deps staging and then reaches Telegram modules with value imports from `grammy`, while the root package still does not declare `grammy`.

## What This Changes

Request that unrelated CLI/control-plane commands stop warning about missing Telegram `grammy` runtime dependencies when those deps are already staged externally.

## Best Possible Solution

Keep CLI metadata, doctor, completion, and cron paths on manifest or narrow public artifacts, and make Telegram runtime-only modules lazy or load them only through the prepared runtime-deps root when real Telegram execution is needed.

## Reproduction Assessment

Yes. The issue gives concrete commands and install conditions, and current main still contains the no-runtime-deps bundled loader plus static Telegram `grammy` value imports needed to reproduce the warning when `grammy` is absent from the package root.

## Solution Assessment

No current fix is present. The narrowest maintainable repair is to avoid importing Telegram runtime modules from unrelated control-plane paths, rather than copying Telegram runtime dependencies into the global OpenClaw package root.
