## Summary

Keep open. Current main still has a credible packaged-install warning path for Telegram: the bundled channel fallback can full-load the Telegram plugin with runtime dependency staging explicitly disabled, and Telegram's full plugin still has static value imports that resolve `grammy` from the package root instead of the staged runtime-deps root.

## What This Changes

The issue asks for CLI/control-plane commands to stop warning about missing Telegram `grammy` modules when Telegram bundled runtime dependencies are already staged externally.

## Best Possible Solution

Keep this open and make CLI metadata/doctor/control-plane paths use lightweight Telegram artifacts instead of full runtime imports; when a real Telegram runtime load is needed, it should go through the prepared bundled runtime root rather than package-root bare import resolution.

## Reproduction Assessment

Yes. A packaged install with Telegram configured, external staged runtime deps present, and no package-root `grammy` should reproduce through `openclaw doctor` or any path that calls `getBundledChannelPlugin("telegram")` with the current full-load fallback.

## Solution Assessment

Yes, the issue's requested direction is the maintainable fix: avoid importing Telegram send/monitor runtime modules for unrelated CLI metadata and use the staged runtime-deps root only for true runtime execution.
