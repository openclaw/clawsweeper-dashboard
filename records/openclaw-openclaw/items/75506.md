## Summary

Keep open. Current main still has a credible package-install warning path: bundled channel discovery/full-load code explicitly bypasses runtime-dependency staging, while the Telegram channel plugin still statically reaches modules with bare `grammy` imports. The report is concrete, recent, unprotected, and narrow enough for a focused repair PR.

## What This Changes

The issue asks OpenClaw CLI commands such as `cron list`, `doctor`, and `completion --help` to stop warning about missing Telegram `grammy` modules when Telegram runtime dependencies are already staged externally.

## Best Possible Solution

Keep the issue open and make CLI metadata/control-plane paths avoid importing Telegram runtime modules that require staged dependencies; when Telegram runtime code is genuinely needed, load it through the prepared bundled runtime root instead of relying on package-root bare imports.

## Reproduction Assessment

Yes. The issue gives concrete commands and environment details, and current source still shows the same static import and skipped-staging path needed to reproduce it in a packaged install with external Telegram runtime deps.

## Solution Assessment

No current fix is present. The narrow maintainable direction is to preserve lightweight CLI/plugin discovery paths and move Telegram `grammy` value imports behind runtime-only lazy boundaries, or otherwise use the existing prepared runtime root only for true runtime execution.
