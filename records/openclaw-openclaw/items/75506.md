## Summary

Keep open. Current main still has the same credible packaged-install warning path: core channel lookup full-loads bundled Telegram with runtime dependency staging disabled, while Telegram's full channel surface still statically reaches `grammy` through runtime-heavy imports.

## What This Changes

The issue reports noisy CLI warnings about missing Telegram `grammy` modules after updating to OpenClaw 2026.4.29, despite staged bundled plugin runtime dependencies being present.

## Best Possible Solution

Keep metadata and diagnostic CLI paths on lightweight manifest/public-artifact surfaces, and make Telegram runtime-only modules lazy or load them through the prepared bundled runtime root only when real Telegram execution is needed.

## Reproduction Assessment

Yes. The issue gives concrete commands and package-install conditions, and current main still contains the full-load, no-runtime-deps staging path plus static Telegram `grammy` value imports needed to reproduce the warning when `grammy` is absent from the package root.

## Solution Assessment

No current fix is present. The narrowest maintainable solution is to keep CLI control-plane lookups from importing Telegram runtime modules, rather than copying Telegram dependencies into the global package root or making unrelated CLI commands perform dependency repair.
