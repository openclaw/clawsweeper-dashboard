## Summary

Keep open. Current main still contains the credible packaged-install warning path reported here: CLI/doctor control-plane lookup can full-load the bundled Telegram channel with runtime dependency staging disabled, while Telegram still has value imports from `grammy` that are only declared as plugin-local staged runtime dependencies. The related macOS confirmation strengthens the report, and the adjacent open runtime-deps issue appears related but not a clean duplicate for this CLI warning surface.

## What This Changes

The issue asks OpenClaw CLI commands to stop warning about missing Telegram `grammy` runtime dependencies when those dependencies are already staged under the bundled plugin runtime-deps root.

## Best Possible Solution

Keep CLI metadata, doctor, completion, and scheduler paths on manifest or narrow public-artifact surfaces, and load Telegram runtime modules through the prepared bundled runtime root only when actual Telegram execution needs them. Avoid making `grammy` a root OpenClaw dependency unless maintainers explicitly choose that broader dependency-ownership change.

## Reproduction Assessment

Yes. The report gives concrete packaged-install commands, and current main still has the source path needed to reproduce when Telegram is configured and `grammy` is absent from the package root but present in staged runtime deps.

## Solution Assessment

No current fix is present. The narrowest maintainable repair is to prevent unrelated CLI/control-plane lookups from importing Telegram runtime-heavy modules, rather than copying plugin-local dependencies into the global package root.
