## Summary

Keep open: the Telegram fix idea matches the linked bug, but the patch currently broadens a shared plugin SDK contract inconsistently, carries unrelated localized docs that this repo does not maintain, and lacks required real Telegram proof.

## What This Changes

The branch adds a shared `suppressReply` reply-payload flag for Telegram native plugin commands, returns early before the Telegram empty-response fallback, and also adds several Simplified Chinese docs pages.

## Best Possible Solution

Land a focused plugin-command silent-result contract that is either clearly Telegram-scoped or consistently honored by native command adapters, with regression coverage, documented semantics, real Telegram proof, and no localized docs in this branch.

## Reproduction Assessment

Yes, source-reproducible rather than live-reproduced: current main normalizes undefined plugin command results to `{}` and the Telegram native command path then sends the empty-response fallback, with an existing test asserting that behavior. The PR still needs real after-fix Telegram proof.

## Solution Assessment

No, not yet: a silent plugin-command result is a plausible direction, but this patch should not add a shared SDK field unless the semantics are documented and honored consistently or explicitly scoped to Telegram. The unrelated localized docs should be removed from the branch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Define suppressReply consistently across native adapters:** `src/auto-reply/reply-payload.ts:49-52`
  - body: Adding `suppressReply` to the shared `ReplyPayload` makes it part of the public plugin command result contract, but only the Telegram adapter honors it. Discord native plugin commands still treat an empty payload as non-renderable and send `⚠️ Command produced no visible reply.`, so a plugin returning `{ suppressReply: true }` would still get a fallback outside Telegram. Either scope this as Telegram-only data or make the native command adapters honor the same contract with tests/docs.
  - confidence: 0.9
- **[P2] Remove the localized docs from this Telegram fix:** `docs/zh-CN/index.md:1`
  - body: This branch adds `docs/zh-CN/**` pages, but the scoped docs policy says localized docs are generated/maintained in the separate docs publish repo and should not be added under `docs/<locale>/**` here. These pages are also unrelated to the Telegram suppress-reply fix, so they need to be removed from this PR.
  - confidence: 0.97
