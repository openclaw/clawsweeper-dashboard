## Summary

This PR remains necessary because current main still does not expose the built-in read tool through direct invoke, and the patch is internally consistent with strong live proof and no actionable review defect; merge should remain a maintainer decision because it adds a permanent config surface and intentionally expands authenticated Gateway access to host filesystem data.

## What This Changes

The branch adds dual-key opt-in exposure of the built-in read coding tool through Gateway HTTP and SDK direct-invoke paths, together with config schema/help, security auditing, tests, and operator documentation.

## Best Possible Solution

Land a read-only, default-off direct-invoke capability only after gateway/security owners accept the host-read trust model and the broad factory dependency; retain dual-key consent, canonical filesystem policy, shared hook execution, source-aware auditing, and built-in precedence over same-named plugins.
