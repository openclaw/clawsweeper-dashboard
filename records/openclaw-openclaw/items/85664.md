## Summary

Keep open: current main and the latest release do not expose the built-in `read` coding tool through Gateway direct invoke, and this PR is a coherent narrow implementation. The PR adds a dual-key `gateway.tools.directInvoke.hostFsRead` opt-in that materializes the built-in `read` tool for Gateway HTTP `/tools/invoke` and SDK RPC `tools.invoke`, with matching config types, schema/help metadata, audit findings, tests, and docs.

## Best Possible Solution

Land the narrow read-only direct-invoke path only after the docs deny list is corrected and a gateway/security owner explicitly accepts the dual-key host-read contract.
