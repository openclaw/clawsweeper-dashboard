Overall confidence: 0.9

Full review comments:

- **[P1] Preserve QMD root replacement reattach coverage:** `extensions/memory-core/src/memory/qmd-manager.ts:1698`
  - body: The sibling memory watcher adds a parent-directory watcher because a native recursive watcher remains tied to the old inode after `rm -rf <root> && mkdir <root>`. This QMD path only stores the main watcher, so after a collection root is replaced it can silently stop seeing new files and stop driving QMD sync.
  - confidence: 0.9
- **[P2] Mock the platform before asserting native watch routing:** `extensions/memory-core/src/memory/qmd-manager.test.ts:672`
  - body: The implementation only takes the native path on `darwin` or `win32`, but this test never overrides `process.platform`. On Linux CI it will fall back to chokidar and fail the `nativeWatchMock` assertions, so the darwin/native cases need a platform mock or helper.
  - confidence: 0.96
- **[P2] Remove the unused native-watch test local:** `extensions/memory-core/src/memory/qmd-manager.test.ts:992`
  - body: `tsconfig.extensions.json` enables `noUnusedLocals`, and `initialStats` is assigned but never read in this new test. That will fail typechecking until the local is removed or used.
  - confidence: 0.95
