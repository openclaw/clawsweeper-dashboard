## Summary

Keep open: current main still lacks the requested lifecycle API and the paired issue was closed in favor of this PR, but the patch needs rebase/API/security fixes and contributor proof before maintainer review can proceed.

## What This Changes

The PR adds a `plugin_updated` lifecycle hook, updater callback, CLI dispatch after plugin updates, tests/docs, and a formatter `maxBuffer` increase.

## Best Possible Solution

Rebase the feature onto current plugin hook/update surfaces, emit `plugin_updated` only after successful real npm/marketplace/ClawHub/git updates, load only the updated plugin runtimes, and include focused tests/docs plus redacted real behavior proof.

## Reproduction Assessment

Yes for review purposes. Source and PR-diff inspection show current main lacks `plugin_updated`, while the PR diff shows stale hook placement, missing git-source coverage, and broad runtime loading without needing a live plugin install.

## Solution Assessment

No, not as written. The feature direction is plausible, but the implementation should be rebased onto `hook-types.ts`, include git update events, preserve current update persistence, and use scoped runtime loading.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Load only updated plugin runtimes for update hooks:** `src/cli/plugins-update-command.ts:197`
  - body: The new dispatch path loads the full plugin registry before calling targeted `plugin_updated` handlers. During a single-plugin update this can execute unrelated plugin runtime code, so load only the updated plugin IDs before dispatching.
  - confidence: 0.84
- **[P2] Register plugin_updated in hook-types.ts:** `src/plugins/types.ts:1425-1454`
  - body: Current main moved typed hook names, event/context types, and handler maps into `src/plugins/hook-types.ts`, with `types.ts` re-exporting that surface. Adding `plugin_updated` only to the stale `types.ts` contract will not register the hook correctly after rebase.
  - confidence: 0.82
- **[P2] Include git plugin updates in the event source:** `src/plugins/types.ts:1945`
  - body: Current main supports updating git-installed plugins, but the new public event source type only allows npm, marketplace, and ClawHub. Include `git` in the lifecycle payload and add coverage so git plugin upgrades can dispatch the hook too.
  - confidence: 0.78

## Security Review
