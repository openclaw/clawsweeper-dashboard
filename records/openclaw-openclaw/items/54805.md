## Summary

Keep this PR open. Current main still lacks a `plugin_updated` lifecycle hook, but this PR is dirty against current main and the proposed diff needs changes before it is a maintainable plugin-contract update.

## What This Changes

The PR adds a typed `plugin_updated` hook, emits update lifecycle events from the plugin updater, dispatches them from `openclaw plugins update`, updates plugin docs/tests, and tweaks a generated-module formatter buffer.

## Best Possible Solution

Refresh the feature onto current main as a narrow plugin contract change: define `plugin_updated` in `src/plugins/hook-types.ts`, emit events only for real successful plugin updates, persist install records through the current ledger path, load only the updated plugin IDs before targeted dispatch, document the public hook behavior, and keep the unrelated script buffer change separate unless it is still needed.

## Reproduction Assessment

Yes. Source and patch inspection are enough: current main lacks the hook, while the PR head shows the new dispatch path and its broad registry load before targeted hook execution.

## Solution Assessment

No, not as written. The feature direction is sound, but the current PR must be ported to the active hook contract file, current install-ledger update flow, and scoped runtime loading before merge.
