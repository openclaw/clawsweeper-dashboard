## Summary

Keep open: the feature direction is plausible and current main still lacks `plugin_updated`, but this branch is not mergeable as written because it targets stale hook contracts, omits git-source lifecycle coverage, broadly loads plugin runtimes, and still lacks required real behavior proof.

## What This Changes

The PR adds a typed `plugin_updated` lifecycle hook for `openclaw plugins update`, dispatches it from the CLI, documents it, adds tests, and increases a formatter `spawnSync` buffer.

## Best Possible Solution

Rebase onto current `hook-types.ts` and managed install-record flows, emit scoped targeted events only for actual updates across npm, ClawHub, marketplace, and git sources, then land after API/security review and real behavior proof.

## Reproduction Assessment

Yes for the review findings: source and PR-head inspection show current main lacks `plugin_updated`, while the PR writes a stale hook surface, excludes git sources, and loads all plugin runtimes before targeted dispatch. I did not run a live plugin update because this read-only review found blocking source-level issues.

## Solution Assessment

No, not as written. The feature should be implemented through current `hook-types.ts`, the managed install-record update path, git-aware event typing, and scoped runtime loading rather than the stale branch shape.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Load only updated plugin runtimes:** `src/cli/plugins-update-command.ts:197`
  - body: The new dispatch path calls `ensurePluginRegistryLoaded({ scope: "all" })` before invoking targeted `plugin_updated` handlers. During a single-plugin update this can load and execute unrelated plugin runtime code; pass the updated plugin IDs through the scoped loader before invoking handlers.
  - confidence: 0.84
- **[P2] Register plugin_updated in hook-types:** `src/plugins/types.ts:1432-1462`
  - body: Current main exports hook names, event/context types, and handler maps from `src/plugins/hook-types.ts`, with `types.ts` re-exporting that surface. Adding `plugin_updated` only to the old `types.ts` hook block will not register the hook correctly after rebase.
  - confidence: 0.82
- **[P2] Include git in update hook events:** `src/plugins/types.ts:1951`
  - body: Current `updateNpmInstalledPlugins` supports `record.source === "git"`, and this PR passes `record.source` into the lifecycle event, but the public event type excludes `"git"`. Add git to the source union and cover git updates so valid git-installed plugin updates can dispatch a truthful event.
  - confidence: 0.78
