## Summary

Keep this PR open: current main still lacks a `plugin_updated` lifecycle hook, but the proposed patch needs changes before merge because it targets a stale hook contract surface, loads too much plugin runtime during update dispatch, and misses the required changelog entry.

## What This Changes

This PR adds a typed `plugin_updated` lifecycle hook, updater event emission, CLI dispatch after plugin updates, hook runner tests/docs, and a formatter `maxBuffer` tweak.

## Best Possible Solution

Land a refreshed, narrow lifecycle change: define `plugin_updated` in `src/plugins/hook-types.ts`, emit events only for real successful plugin updates, persist through the installed-plugin index path, load only updated plugin IDs before targeted dispatch, and update docs plus `CHANGELOG.md`.

## Reproduction Assessment

Yes. Source and diff inspection are enough: current main lacks the hook, while the PR diff shows the stale contract placement and broad registry load without needing a live plugin install.

## Solution Assessment

No, not as written. The feature direction is sound, but the safest implementation is the narrower scoped-load and active-contract version described above.
