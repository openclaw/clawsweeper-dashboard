## Summary

Keep open. Current main still lacks a `plugin_updated` lifecycle hook, but this PR is stale against the active hook contract and plugin update pipeline and needs targeted repairs before merge.

## What This Changes

This PR adds a typed `plugin_updated` hook, emits update lifecycle events from plugin updates, dispatches them from the CLI, adds hook/update tests and docs, and increases a formatter subprocess buffer.

## Best Possible Solution

Land a rebased, narrow lifecycle change that defines `plugin_updated` in `src/plugins/hook-types.ts`, emits events only for real successful npm/ClawHub/marketplace/git plugin updates, persists through the installed-plugin index flow, loads only updated plugin IDs before targeted dispatch, and updates docs plus `CHANGELOG.md`.

## Reproduction Assessment

Yes for review purposes. Source and PR-diff inspection are enough: current main lacks the hook, while the PR diff shows stale hook contract placement and broad runtime loading without needing a live plugin install.

## Solution Assessment

No, not as written. The feature direction is sound, but the current diff should be rebased onto `hook-types.ts`, preserve current install-record/update semantics, include git update events, and scope runtime loading to updated plugin IDs.
