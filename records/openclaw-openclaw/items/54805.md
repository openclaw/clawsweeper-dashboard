## Summary

Keep this PR open: the feature is still useful and not implemented on current main, but the branch is stale against the active plugin hook contract and plugin install-record paths, has concrete correctness/security blockers, and still lacks contributor real-behavior proof.

## What This Changes

The PR adds a typed `plugin_updated` lifecycle hook for `openclaw plugins update`, CLI dispatch, docs, tests, and a formatter buffer adjustment.

## Best Possible Solution

Rebase onto current `hook-types.ts` and managed install-record paths, emit scoped targeted events only for actual updates across npm, ClawHub, marketplace, and git sources, then land after maintainer API/security review and real behavior proof.

## Reproduction Assessment

Yes for review purposes: source and diff inspection show current main lacks `plugin_updated`, while the PR writes a stale hook surface, omits git-source coverage, and loads all plugin runtimes before targeted dispatch. I did not run a live plugin update because this is a read-only review and the blocking gaps are visible in source.

## Solution Assessment

No, not as written. The feature direction is plausible, but the implementation should use current `hook-types.ts`, preserve the managed install-record path, include git updates, and load only updated plugin runtimes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Scope update-hook runtime loading:** `src/cli/plugins-update-command.ts:197`
  - body: The new dispatch path calls `ensurePluginRegistryLoaded({ scope: "all" })` before targeted `plugin_updated` dispatch. During a single-plugin update this can load and execute unrelated plugin runtime code; pass the updated plugin IDs through the scoped loader before invoking handlers.
  - confidence: 0.84
- **[P2] Register plugin_updated on the active hook surface:** `src/plugins/types.ts:1432-1462`
  - body: Current main exports hook names, event/context types, and handler maps from `src/plugins/hook-types.ts`, with `types.ts` re-exporting that surface. Adding `plugin_updated` only to the old `types.ts` block will not register the hook correctly after rebase.
  - confidence: 0.82
- **[P2] Include git updates in the lifecycle source:** `src/plugins/types.ts:1951`
  - body: Current `updateNpmInstalledPlugins` supports `record.source === "git"`, and the PR passes `record.source` into the lifecycle event, but the public event type excludes `"git"`. Add git to the source union and cover git updates so valid git-installed plugin updates can dispatch a truthful event.
  - confidence: 0.78
