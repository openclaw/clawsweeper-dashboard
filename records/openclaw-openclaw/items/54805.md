## Summary

Keep open: current main still lacks the requested `plugin_updated` lifecycle hook, and the linked feature issue was paired with this PR, but the branch is not merge-ready against current plugin hook/update architecture.

## What This Changes

This PR adds a typed `plugin_updated` hook with update-pipeline emission, CLI dispatch, tests, docs, and a formatter `spawnSync` buffer increase.

## Best Possible Solution

Land a rebased lifecycle hook that defines `plugin_updated` in `hook-types.ts`, emits only for real successful npm, ClawHub, marketplace, and git plugin updates, dispatches only to updated plugin IDs after persistence, and updates docs, changelog, and focused tests.

## Reproduction Assessment

Yes for review purposes. Source and PR-diff inspection show current main lacks the hook, while the branch adds it in stale contract placement, uses broad runtime loading, and omits git from the public event source.

## Solution Assessment

No, not as written. The feature direction is maintainable, but the implementation should be rebased onto current hook/update seams and scoped so unrelated plugin runtime code is not executed.
