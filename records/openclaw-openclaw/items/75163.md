## Summary

Keep open. Current main still misses provider-qualified alias resolution: `lmstudio-moe/MoE` is not matched as the configured `MoE` alias before parsing, and #75198 is the open unmerged closing PR for this exact defect.

## What This Changes

The issue requests that TUI/Control UI mid-session model switches resolve configured aliases to canonical provider model IDs before session persistence.

## Best Possible Solution

Land or adapt #75198 so provider-qualified aliases resolve canonically in shared model selection before `sessions.patch` persists the override, with resolver and session-patch regression coverage.

## Solution Assessment

Yes. Fixing the shared resolver/session-patch path is the narrowest maintainable solution because the TUI/Control UI model picker and model slash-command paths persist through that boundary.
