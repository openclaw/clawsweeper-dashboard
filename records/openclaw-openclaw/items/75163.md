## Summary

Keep open. Current main still has the source-level provider-qualified alias path, and #75198 is an open, unmerged closing PR for the same session-patch resolver defect.

## What This Changes

The issue reports that Control UI/TUI session model switching can persist a provider-qualified display alias instead of resolving it to the canonical provider/model ID.

## Best Possible Solution

Land or adapt #75198 so shared model selection resolves provider-qualified configured aliases before sessions.patch persists overrides, with resolver and sessions.patch regression coverage.

## Reproduction Assessment

Yes, source-level. With alias MoE configured for lmstudio-moe/qwen3.6-35b-a3b, a sessions.patch model value of lmstudio-moe/MoE misses the bare alias index and resolves as provider lmstudio-moe with model MoE.

## Solution Assessment

Yes. Fixing the shared resolver used by sessions.patch is the narrowest maintainable solution because it covers the Control UI picker and other session-patch model switch callers without LM Studio-specific logic.
