## Summary

Keep open. Current main still takes the provider-qualified picker value through the shared resolver path without resolving the suffix alias, and the issue is already paired with open closing PR #75198.

## What This Changes

TUI and Control UI mid-session model switching should canonicalize provider-qualified display aliases before persisting session model overrides.

## Best Possible Solution

Land or adapt PR #75198's shared resolver approach with regression coverage for both direct alias resolution and `sessions.patch` persistence.

## Reproduction Assessment

Yes. Configure `agents.defaults.models["lmstudio-moe/qwen3.6-35b-a3b"].alias = "MoE"`, then call `sessions.patch` with `model: "lmstudio-moe/MoE"`; current source shows the value is parsed as model `MoE` after the whole-string alias lookup misses.

## Solution Assessment

Yes. Resolving provider-qualified aliases at the shared model resolver/session-patch boundary is the narrowest maintainable fix because TUI, Control UI, slash commands, SDK callers, and other session patch users share that path.
