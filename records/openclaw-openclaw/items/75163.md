## Summary

Keep open. Current main still resolves aliases only against the whole raw model string before provider/model parsing, so provider-qualified alias input can still flow through as the suffix model ID, and the issue is already paired with open closing PR #75198.

## What This Changes

Resolve provider-qualified display aliases during TUI and Control UI session model switches before persisting the session override.

## Best Possible Solution

Review and land or adapt PR #75198's shared resolver fix with regression coverage for provider-qualified aliases at both the resolver and `sessions.patch` persistence boundary.

## Reproduction Assessment

Yes. Configure a model alias such as `agents.defaults.models["lmstudio-moe/qwen3.6-35b-a3b"].alias = "MoE"`, then drive the shared resolver or `sessions.patch` with `model: "lmstudio-moe/MoE"`; current source shows it misses alias resolution and parses the suffix as model `MoE`.

## Solution Assessment

Yes. Fixing provider-qualified alias canonicalization in the shared model resolver/session patch path is the narrowest maintainable solution because TUI, Control UI, slash-command, SDK, and direct session patch callers all converge there.
