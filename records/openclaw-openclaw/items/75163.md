## Summary

Keep open. Current main still resolves only bare aliases before parsing, so `lmstudio-moe/MoE` can reach the provider-facing model as `MoE`; open PR #75198 is the linked unmerged closing fix.

## What This Changes

The issue reports that Control UI/TUI mid-session model switches can persist provider-qualified display aliases instead of canonical model IDs.

## Best Possible Solution

Land or adapt #75198 so provider-qualified aliases resolve canonically before `sessions.patch` persists the model override, with resolver and session-patch regression coverage.

## Reproduction Assessment

Yes. Configure `MoE` as the alias for `lmstudio-moe/qwen3.6-35b-a3b`, then patch `model: "lmstudio-moe/MoE"`; source inspection shows the alias lookup misses and parsing preserves `MoE` as the provider-facing model.

## Solution Assessment

Yes. Fixing the shared resolver/session-patch path is the narrowest maintainable solution because the Control UI model picker and model slash-command paths persist through that boundary.
