## Summary

Keep open. Current main still source-reproduces the provider-qualified alias failure path, and #75198 is the open unmerged closing PR for the same session-switch resolver defect.

## What This Changes

The issue asks for Control UI/TUI session model switches to resolve configured display aliases to canonical provider/model IDs before persisting session overrides.

## Best Possible Solution

Land or adapt #75198 so shared model selection canonicalizes provider-qualified aliases before sessions.patch persists overrides, with resolver and sessions.patch regression coverage.

## Reproduction Assessment

Yes, source-level. With alias MoE configured for lmstudio-moe/qwen3.6-35b-a3b, a sessions.patch value of lmstudio-moe/MoE misses the bare alias index and resolves to provider lmstudio-moe, model MoE.

## Solution Assessment

Yes. Fixing the shared resolver used by sessions.patch is the narrowest maintainable path because it covers the Control UI picker and any other session-patch model switch caller without LM Studio-specific logic.
