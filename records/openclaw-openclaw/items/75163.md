## Summary

Keep open. Current main still leaves the provider-qualified alias path unresolved: bare aliases are indexed and resolved, but `provider/Alias` is parsed as a provider/model ref and can preserve the display alias as the provider-facing model id. Open PR #75198 is the paired closing fix candidate, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for TUI and Control UI mid-session model switches to resolve configured display aliases to canonical provider model ids before persisting session overrides.

## Best Possible Solution

Review and land or adapt PR #75198 so provider-qualified aliases resolve canonically at the shared resolver/session-patch boundary with regression coverage.

## Reproduction Assessment

Yes. Configure alias `MoE` for `lmstudio-moe/qwen3.6-35b-a3b`, then patch a session with `model: "lmstudio-moe/MoE"`; current source shows the alias lookup misses and parsing preserves `MoE` as the provider-facing model id.

## Solution Assessment

Yes. Fixing the shared resolver/session-patch boundary is the narrowest maintainable solution because TUI, Control UI, slash commands, SDK callers, and other session patch users share that path.
