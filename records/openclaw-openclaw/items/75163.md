## Summary

Keep open. Current main still resolves aliases only against the whole raw model string before provider/model parsing, so a provider-qualified alias such as `lmstudio-moe/MoE` can still flow through session patching as provider `lmstudio-moe` and model `MoE`; #75198 is the open closing PR for this shared resolver defect.

## What This Changes

The issue asks OpenClaw to canonicalize aliased model selections during TUI/Control UI session switches before persisting the session model override.

## Best Possible Solution

Review and land or adapt #75198 so provider-qualified aliases resolve canonically at the shared resolver/session-patch boundary with regression coverage.

## Reproduction Assessment

Yes. Configure alias `MoE` for `lmstudio-moe/qwen3.6-35b-a3b`, then patch a session with `model: "lmstudio-moe/MoE"`; current source shows the alias lookup misses and the parsed model remains `MoE`.

## Solution Assessment

Yes. Fixing the shared resolver/session-patch boundary is the narrowest maintainable path because Control UI, terminal TUI, slash commands, and SDK callers share that session patch route.
