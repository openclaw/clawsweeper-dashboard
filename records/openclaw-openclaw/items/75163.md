## Summary

Keep open. Current main still lets a provider-qualified alias such as `lmstudio-moe/MoE` miss the configured bare alias lookup and then persist `MoE` as the provider-facing model, and #75198 is the open unmerged closing PR for this exact defect.

## What This Changes

The issue asks for mid-session Control UI/TUI model switches to resolve configured display aliases to canonical provider model IDs before persisting session overrides.

## Best Possible Solution

Land or adapt #75198 so provider-qualified aliases resolve canonically in shared model selection before `sessions.patch` persists the override, with resolver and session-patch regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction is clear: configure `MoE` as the alias for `lmstudio-moe/qwen3.6-35b-a3b`, then send `sessions.patch` with `model: "lmstudio-moe/MoE"`; current main misses the alias and parses `MoE` as the provider model.

## Solution Assessment

Yes. Fixing the shared resolver/session-patch path is the narrowest maintainable solution because both the Control UI picker and slash-command model changes persist through that boundary.
