## Summary

Keep open. Current main at be276843 still resolves aliases only against the whole raw model string before provider/model parsing, and open PR #75198 is the paired unmerged fix candidate for this session-switch defect.

## What This Changes

The issue asks for TUI and Control UI mid-session model switches to resolve configured display aliases to canonical provider/model IDs before storing session overrides.

## Best Possible Solution

Review and land or adapt PR #75198 so provider-qualified aliases resolve canonically at the shared resolver/session-patch boundary with regression coverage.

## Reproduction Assessment

Yes. Configure alias `MoE` for `lmstudio-moe/qwen3.6-35b-a3b`, then patch a session with `model: "lmstudio-moe/MoE"`; current source shows the alias lookup misses and the string parses as provider `lmstudio-moe`, model `MoE`.

## Solution Assessment

Yes. A shared resolver/session-patch fix is the narrowest maintainable solution because TUI commands, Control UI controls, slash commands, SDK callers, and direct session patch users all share that path.
