## Summary

Keep open. Current main still has the provider-qualified alias failure path in source, and the open unmerged PR #75198 is already the closing implementation candidate for this exact sessions.patch resolver defect.

## What This Changes

The issue asks Control UI session model switching to resolve configured display aliases such as `MoE` to canonical provider model IDs before persisting session model overrides.

## Best Possible Solution

Land or adapt #75198 so provider-qualified aliases resolve canonically in shared model selection before `sessions.patch` persists overrides, with resolver and gateway regression coverage.

## Reproduction Assessment

Yes, source-level. With alias `MoE` configured for `lmstudio-moe/qwen3.6-35b-a3b`, a `sessions.patch` model value of `lmstudio-moe/MoE` misses the bare alias index and parses as provider `lmstudio-moe`, model `MoE`.

## Solution Assessment

Yes. Fixing the shared resolver used by `sessions.patch` is the narrowest maintainable solution because it covers the Control UI picker and any other session-patch model switch caller without LM Studio-specific logic.
