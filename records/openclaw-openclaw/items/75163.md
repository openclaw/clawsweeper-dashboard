## Summary

Keep open. Current main still has the provider-qualified alias resolver gap in the sessions.patch model path, and open PR #75198 is already the active closing implementation for this issue.

## What This Changes

The issue asks Control UI/TUI model switching to resolve configured display aliases to canonical provider/model IDs before persisting session model overrides.

## Best Possible Solution

Review and land or adapt PR #75198 so shared model resolution canonicalizes provider-qualified aliases before sessions.patch persists user model overrides, with resolver and sessions.patch regression coverage.

## Reproduction Assessment

Yes, source-level. With alias MoE configured for lmstudio-moe/qwen3.6-35b-a3b, a sessions.patch value of lmstudio-moe/MoE misses the bare alias key and falls through as provider lmstudio-moe with model MoE.

## Solution Assessment

Yes. Fixing the shared resolver used by sessions.patch is the narrow maintainable solution because it covers the Control UI picker and other session-patch callers without LM Studio-specific logic.
