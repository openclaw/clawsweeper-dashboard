## Summary

Keep open. Current main still routes `provider/Alias` through whole-string alias lookup and then provider/model parsing, so the reported session-switch failure is not implemented on main; the issue is also paired with open closing PR #75198, so it should remain open until that PR or an equivalent shared resolver fix lands.

## What This Changes

The issue asks OpenClaw to resolve TUI/Control UI mid-session model selections from display aliases to canonical provider/model IDs before persisting session model overrides.

## Best Possible Solution

Land or adapt PR #75198's shared resolver approach so `provider/Alias` resolves only when the provider prefix matches the configured alias target, with regression coverage for resolver behavior and session override persistence.

## Reproduction Assessment

Yes. Configure `agents.defaults.models["lmstudio-moe/qwen3.6-35b-a3b"].alias = "MoE"`, call `sessions.patch` with `model: "lmstudio-moe/MoE"`, and current main will treat `MoE` as the raw model suffix rather than resolving it to the canonical configured model ID.

## Solution Assessment

Yes. The best fix is at the shared model resolver/session-patch boundary; a TUI-only workaround would leave Control UI, slash-command, SDK, and other `sessions.patch` callers with divergent alias behavior.
