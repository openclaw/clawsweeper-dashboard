## Summary

Keep open. Current main still has a source-level path where a provider-qualified alias can miss the alias index and be parsed as a provider/model pair with the alias as the provider-facing model; #75198 is the open unmerged closing PR for this exact defect.

## What This Changes

The issue asks for mid-session Control UI/TUI model switches to canonicalize aliased provider/model selections before persisting session overrides.

## Best Possible Solution

Land or adapt #75198 so provider-qualified aliases resolve canonically in shared model selection before session overrides are persisted, with resolver and session-patch regression coverage.

## Reproduction Assessment

Yes, source-level. Configure alias `MoE` for `lmstudio-moe/qwen3.6-35b-a3b`, then patch `model: "lmstudio-moe/MoE"`; current resolver code misses the bare alias entry and parses `MoE` as the provider-facing model.

## Solution Assessment

Yes. Fixing the shared model resolver used by `sessions.patch` is the narrowest maintainable solution because both the Control UI picker and model command paths delegate through that gateway boundary.
