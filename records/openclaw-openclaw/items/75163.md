## Summary

Keep open. Current main still resolves aliases only against the whole raw model string, so `provider/Alias` can miss the configured alias target, and open PR #75198 already links this issue as the active fix candidate.

## What This Changes

The issue asks for mid-session TUI and Control UI model switching to resolve configured display aliases to canonical provider/model IDs before persisting session overrides.

## Best Possible Solution

Land or adapt #75198's shared resolver approach so `provider/Alias` resolves only when the provider prefix matches the configured alias target, with regression coverage in both model selection and `sessions.patch` persistence.

## Reproduction Assessment

Yes. Configure `agents.defaults.models["lmstudio-moe/qwen3.6-35b-a3b"].alias = "MoE"` and call `sessions.patch` with `model: "lmstudio-moe/MoE"`; current source shows the alias lookup misses and parses `MoE` as the raw model segment.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared model resolver and session-patch boundary; a TUI-only workaround would leave Control UI, slash-command, SDK, and other `sessions.patch` callers divergent.
