## Summary

Keep open: current main still lacks provider-qualified alias resolution at the shared session model patch boundary, and open PR #75198 already closes this issue with the narrow resolver fix.

## What This Changes

The issue requests that mid-session model switches resolve configured display aliases such as `MoE` to canonical provider model IDs before persisting session overrides.

## Best Possible Solution

Review and land or adapt PR #75198, keeping the fix in the shared model resolver/session-patch path with regression coverage for `provider/Alias` canonicalization.

## Reproduction Assessment

Yes. Configure `agents.defaults.models["lmstudio-moe/qwen3.6-35b-a3b"].alias = "MoE"` and patch a session with `model: "lmstudio-moe/MoE"`; current source shows the whole-string alias lookup misses and parses `MoE` as the model suffix.

## Solution Assessment

Yes. Resolving provider-qualified aliases in the shared resolver used by `sessions.patch` is the narrowest maintainable fix because TUI, Control UI, and other session patch callers share that boundary.
