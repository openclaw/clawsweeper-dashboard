## Summary

Keep open: current main still lets provider-qualified display aliases miss the shared alias resolver on the `sessions.patch` path, and #75198 is the active unmerged closing PR for the same defect.

## What This Changes

Resolve TUI/Control UI mid-session model selections from display aliases to canonical provider/model IDs before persisting the session override.

## Best Possible Solution

Review and land or adapt #75198 so shared model selection canonicalizes provider-qualified aliases before session overrides are persisted.

## Reproduction Assessment

Yes, source-level. Configure alias `MoE` for `lmstudio-moe/qwen3.6-35b-a3b`, then patch `model: "lmstudio-moe/MoE"`; current main misses the bare alias entry and parses `MoE` as the provider-facing model.

## Solution Assessment

Yes. Fixing the shared resolver used by `sessions.patch` is the narrow maintainable fix because the Control UI picker delegates through that gateway boundary, and #75198 already targets that path.
