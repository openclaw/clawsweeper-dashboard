## Summary

Keep open. Current main still has the source-level provider-qualified alias miss, and #75198 is the open unmerged closing PR for this exact resolver/session-patch defect.

## What This Changes

The issue requests that Control UI/TUI mid-session model switches resolve configured display aliases to canonical provider/model IDs before persisting session overrides.

## Best Possible Solution

Land or adapt #75198 so provider-qualified aliases resolve canonically in the shared model-selection path before sessions.patch persists session overrides.

## Reproduction Assessment

Yes, source-level. A value such as lmstudio-moe/MoE is compared against aliasIndex.byAlias using the full string, while configured aliases are indexed as bare alias text, so it falls through to provider/model parsing instead of canonicalizing.

## Solution Assessment

Yes. Fixing the shared resolver used by sessions.patch is the narrowest maintainable solution because it covers the Control UI picker and other session-patch callers without LM Studio-specific logic.
