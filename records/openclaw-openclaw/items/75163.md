## What This Changes

Fix Control UI session model switching so provider-qualified display aliases resolve to canonical provider/model IDs before being persisted.

## Best Possible Solution

Review and land or adapt #75198 so shared model resolution canonicalizes provider-qualified aliases before sessions.patch persists model overrides, with resolver and sessions.patch regression coverage.

Yes, source-level. With MoE configured as an alias for lmstudio-moe/qwen3.6-35b-a3b, a sessions.patch model value of lmstudio-moe/MoE misses the bare alias key and resolves as provider lmstudio-moe with model MoE.

## Solution Assessment

Yes. Fixing the shared resolver used by sessions.patch is the narrow maintainable solution because it covers the Control UI picker and any other session-patch caller without LM Studio-specific logic.
