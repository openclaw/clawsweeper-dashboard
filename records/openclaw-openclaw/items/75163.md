## Summary

Keep open. Current main still misses provider-qualified aliases at the shared resolver/session-patch boundary, and open PR #75198 is the paired closing fix rather than a reason to close early.

## What This Changes

The issue asks for Control UI/TUI model switching to resolve configured display aliases such as `lmstudio-moe/MoE` to canonical provider/model IDs before persisting the session override.

## Best Possible Solution

Review and land or adapt PR #75198 so provider-qualified aliases resolve canonically at the shared resolver/session-patch boundary with regression coverage.

## Reproduction Assessment

Yes. Configure alias `MoE` for a canonical LM Studio model and patch a session with `model: "lmstudio-moe/MoE"`; current source shows the alias lookup misses and the value is parsed as provider `lmstudio-moe`, model `MoE`.

## Solution Assessment

Yes. Fixing the shared resolver/session-patch boundary is the narrowest maintainable path because Control UI, terminal TUI, slash commands, and SDK callers all converge there.
