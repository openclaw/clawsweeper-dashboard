## Summary

Keep open. Current main still misses provider-qualified alias resolution before session model overrides are persisted, and #75198 is the open unmerged PR that directly targets this bug.

## What This Changes

The issue requests that TUI/Control UI mid-session model switches resolve configured display aliases to canonical provider model IDs before persisting the session override.

## Best Possible Solution

Review and land or adapt #75198 so provider-qualified aliases resolve canonically in the shared model resolver before sessions.patch persists model overrides.

## Reproduction Assessment

Yes, source-level. Configure alias MoE for lmstudio-moe/qwen3.6-35b-a3b, then patch model: "lmstudio-moe/MoE"; current main misses the bare alias entry and parses MoE as the provider-facing model.

## Solution Assessment

Yes. Fixing the shared model resolver used by sessions.patch is the narrowest maintainable solution because both picker and slash-command model changes pass through that gateway boundary.
