## Summary

Keep open. Current main still resolves provider-qualified display aliases as direct provider/model refs, and open PR #75198 is already the active closing implementation path for the `sessions.patch` resolver gap.

## What This Changes

The issue asks for TUI/Control UI mid-session model changes to resolve configured aliases to canonical provider/model IDs before persistence.

## Best Possible Solution

Review and land or adapt #75198 so shared model resolution canonicalizes provider-qualified aliases before session model overrides are persisted, with resolver and `sessions.patch` regression coverage.

## Reproduction Assessment

Yes, source-level. With alias `MoE` configured for `lmstudio-moe/qwen3.6-35b-a3b`, a `sessions.patch` model value of `lmstudio-moe/MoE` misses the bare alias key and resolves as provider `lmstudio-moe` with model `MoE`.

## Solution Assessment

Yes. The narrow maintainable fix is in the shared resolver used by `sessions.patch`, not LM Studio-specific UI handling; open PR #75198 follows that direction.
