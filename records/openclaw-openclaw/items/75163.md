## Summary

Keep open. Current main still misses provider-qualified alias resolution on the shared session model-switch path, and #75198 is the open unmerged closing PR for this exact defect.

## What This Changes

Resolve provider-qualified configured model aliases during Control UI/TUI session model switches before `sessions.patch` persists the override.

## Best Possible Solution

Review and land or adapt #75198 so provider-qualified aliases resolve canonically in shared model selection before session model overrides are validated and persisted.

## Reproduction Assessment

Yes from source inspection. Configure `MoE` as the alias for `lmstudio-moe/qwen3.6-35b-a3b`, then send `sessions.patch` with `model: "lmstudio-moe/MoE"`; current main misses the bare alias lookup and parses `MoE` as the provider-facing model suffix.

## Solution Assessment

Yes. Fixing provider-qualified alias handling in the shared resolver/session-patch boundary is the narrowest maintainable solution because both the Control UI picker and other session model-switch paths go through that boundary.
