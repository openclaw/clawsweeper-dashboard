## Summary

Keep open. Current main still has the provider-qualified alias resolution gap, and https://github.com/openclaw/openclaw/pull/75198 is the active open closing PR for the same resolver/session-patch defect.

## What This Changes

The issue reports that Control UI/TUI model switching can persist a provider-qualified display alias instead of the canonical provider model ID.

## Best Possible Solution

Review and land or adapt https://github.com/openclaw/openclaw/pull/75198 so shared model resolution canonicalizes provider-qualified aliases before sessions.patch persists user model overrides.

## Reproduction Assessment

Yes, source-level. With alias MoE configured for lmstudio-moe/qwen3.6-35b-a3b, sessions.patch model value lmstudio-moe/MoE misses the bare alias key and resolves as provider lmstudio-moe with model MoE.

## Solution Assessment

Yes. Fixing the shared resolver used by sessions.patch is the narrow maintainable path because it covers Control UI/TUI model switching without LM Studio-specific logic.
