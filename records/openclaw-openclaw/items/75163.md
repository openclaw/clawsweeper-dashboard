## Summary

Keep open. Current main still has the source-level provider-qualified alias path that can persist `lmstudio-moe/MoE` as model `MoE`, and #75198 is the open unmerged closing PR for the same resolver/session-patch defect.

## What This Changes

The issue asks for TUI/Control UI mid-session model switches to resolve display aliases to canonical provider model IDs before persisting session overrides.

## Best Possible Solution

Land or adapt #75198 so provider-qualified aliases resolve canonically in shared model selection before session overrides are persisted, with resolver and sessions.patch regression coverage.

## Reproduction Assessment

Yes, source-level. With alias `MoE` configured for `lmstudio-moe/qwen3.6-35b-a3b`, a `sessions.patch` value of `lmstudio-moe/MoE` misses the bare alias index and resolves to provider `lmstudio-moe`, model `MoE`.

## Solution Assessment

Yes. Fixing the shared resolver used by `sessions.patch` is the narrowest maintainable solution because it covers the Control UI picker and any other session-patch model switch caller without LM Studio-specific logic.
