## Summary

Keep open. Current main still lets a provider-qualified display alias miss the configured alias index and flow into provider/model parsing, while open PR #75198 is the paired unmerged fix candidate.

## What This Changes

Resolve TUI/Control UI mid-session model switches so configured display aliases are persisted as canonical provider/model IDs instead of raw alias text.

## Best Possible Solution

Review and land or adapt PR #75198 so provider-qualified aliases resolve canonically at the shared resolver/session-patch boundary with regression coverage.

## Solution Assessment

Yes. Fixing shared model resolution before session persistence is the narrowest maintainable path because UI selections, slash commands, session_status(model=...), and direct sessions.patch calls share that boundary.
