## Summary

Keep open. Current main still exact-matches the full raw model string against the alias index, so `lmstudio-moe/MoE` falls through as provider `lmstudio-moe` and model `MoE`; open PR #75198 targets this defect but is not merged.

## What This Changes

Resolve Control UI/TUI mid-session model switches from provider-qualified display aliases to canonical model IDs before persisting session overrides.

## Best Possible Solution

Land or adapt #75198 so provider-qualified aliases resolve canonically in the shared model resolver and session patch persistence has regression coverage.

## Reproduction Assessment

Yes, source-level. With alias `MoE` configured for `lmstudio-moe/qwen3.6-35b-a3b`, `sessions.patch` value `lmstudio-moe/MoE` misses the bare alias index and resolves as provider `lmstudio-moe`, model `MoE`.

## Solution Assessment

Yes. Fixing shared model resolution before `sessions.patch` persists overrides is the narrowest maintainable solution because it covers the Control UI picker without LM Studio-specific logic.
