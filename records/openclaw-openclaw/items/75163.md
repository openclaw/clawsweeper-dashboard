## Summary

Keep open. Current main still resolves only whole-string aliases before provider/model parsing, so a provider-qualified display alias such as `lmstudio-moe/MoE` can still flow through the shared session patch boundary as model `MoE`; open PR #75198 is the paired closing fix candidate.

## What This Changes

The issue asks for TUI and Control UI mid-session model switches to resolve display aliases to canonical provider/model IDs before persisting session model overrides.

## Best Possible Solution

Review and land or adapt PR #75198 so provider-qualified aliases canonicalize at the shared resolver/session-patch boundary with regression coverage.

## Reproduction Assessment

Yes. Configure alias `MoE` for `lmstudio-moe/qwen3.6-35b-a3b`, then patch a session with `model: "lmstudio-moe/MoE"`; current source shows the alias lookup misses and the value parses as provider `lmstudio-moe` with model `MoE`.

## Solution Assessment

Yes. The shared resolver/session-patch boundary is the narrowest maintainable fix because Control UI, terminal TUI commands, slash-command paths, and SDK callers all share it.
