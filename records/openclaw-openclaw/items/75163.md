## Summary

Keep open. Current main at 9e983ef still resolves only whole-string aliases before provider/model parsing, so a provider-qualified alias such as `lmstudio-moe/MoE` can still flow through `sessions.patch`; open PR #75198 is the paired closing fix candidate.

## What This Changes

The issue reports that TUI/Control UI session model switching can persist a provider-qualified display alias instead of the canonical model ID.

## Best Possible Solution

Review and land or adapt PR #75198 so provider-qualified aliases resolve canonically at the shared resolver/session-patch boundary with regression coverage.

## Reproduction Assessment

Yes. Configure alias `MoE` for `lmstudio-moe/qwen3.6-35b-a3b`, then patch a session with `model: "lmstudio-moe/MoE"`; current source shows it resolves as provider `lmstudio-moe` and model `MoE` after the alias lookup misses.

## Solution Assessment

Yes. The shared resolver/session-patch boundary is the narrowest maintainable fix because TUI commands, Control UI controls, slash commands, SDK callers, and other session patch users all share that path.
