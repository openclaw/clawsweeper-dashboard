## Summary

Keep open. Current main still resolves aliases only against the full raw model string, so `lmstudio-moe/MoE` misses a configured bare `MoE` alias and can persist `MoE` as the provider-facing model; #75198 is the open unmerged PR that targets this exact defect.

## What This Changes

The issue requests Control UI/TUI mid-session model switching to resolve configured display aliases to canonical provider/model IDs before persisting session model overrides.

## Best Possible Solution

Review and land or adapt #75198 so provider-qualified aliases resolve canonically in shared model selection before `sessions.patch` persists overrides, with resolver and session-patch regression coverage.

## Reproduction Assessment

Yes. Configure `MoE` as the alias for `lmstudio-moe/qwen3.6-35b-a3b`, then patch `model: "lmstudio-moe/MoE"`; source inspection shows the alias lookup misses and parsing preserves `MoE` as the provider-facing model.

## Solution Assessment

Yes. Fixing shared model alias resolution before session patch persistence is the narrowest maintainable path because both the Control UI picker and model-switch surfaces flow through that resolver boundary.
