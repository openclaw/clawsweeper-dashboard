## Summary

Keep open. Current main still misses provider-qualified alias resolution before session persistence, and #75198 is the open unmerged closing PR for the same resolver/session-patch defect.

## What This Changes

Resolve TUI/Control UI mid-session model aliases to canonical provider model IDs before `sessions.patch` persists the session override.

## Best Possible Solution

Land or adapt #75198 so provider-qualified aliases resolve canonically in shared model selection before session overrides are persisted, with resolver and session-patch regression coverage.

## Reproduction Assessment

Yes, source-level. Configure alias `MoE` for `lmstudio-moe/qwen3.6-35b-a3b`, then patch `model: "lmstudio-moe/MoE"`; current code misses the bare alias entry and parses `MoE` as the provider-facing model.

## Solution Assessment

Yes. Fixing the shared model resolver used by `sessions.patch` is the narrowest maintainable solution because both the Control UI picker and model slash-command flow delegate through that gateway boundary.
