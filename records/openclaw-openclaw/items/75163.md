## Summary

Keep open. Current main still resolves only bare aliases before provider/model parsing, so a value like `lmstudio-moe/MoE` falls through as provider `lmstudio-moe` with provider-facing model `MoE`; open PR #75198 is the paired closing fix candidate.

## What This Changes

Resolve TUI and Control UI mid-session model switches containing configured aliases to canonical provider/model IDs before persisting session overrides.

## Best Possible Solution

Review and land or adapt PR #75198 so provider-qualified aliases resolve canonically at the shared model-selection and session-patch boundary with regression coverage.

## Reproduction Assessment

Yes. Configure alias `MoE` for `lmstudio-moe/qwen3.6-35b-a3b`, then send `sessions.patch` with `model: "lmstudio-moe/MoE"`; source inspection shows the alias lookup misses and parsing preserves `MoE` as the provider-facing model.

## Solution Assessment

Yes. Fixing the shared resolver/session-patch path is the narrowest maintainable solution because UI, TUI slash commands, `session_status(model=...)`, and direct `sessions.patch` calls all pass through that boundary.
