## Summary

Keep open. Current main still routes `sessions.patch` model updates through the shared resolver without handling `provider/Alias`, and the open linked PR at https://github.com/openclaw/openclaw/pull/75198 is the active implementation path for this same Control UI/TUI model-switch bug.

## What This Changes

The issue asks for Control UI/TUI mid-session model switching to persist canonical provider/model IDs when a user selects an aliased model.

## Best Possible Solution

Land a narrow resolver/session-patch fix equivalent to https://github.com/openclaw/openclaw/pull/75198, with coverage for both shared model resolution and gateway session persistence.

## Reproduction Assessment

Yes, from source inspection rather than a live run: `lmstudio-moe/MoE` will not match the bare `MoE` alias key and then parses as provider `lmstudio-moe` with model `MoE`, which fails when only the canonical configured model is allowed.

## Solution Assessment

Yes, the linked PR's direction is the narrow maintainable fix: canonicalize provider-qualified aliases in shared model selection before `sessions.patch` persists the override.
