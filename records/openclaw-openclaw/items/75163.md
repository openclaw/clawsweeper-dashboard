## Summary

Keep open. Current main still resolves aliases only against the whole raw model string before provider/model parsing, while `sessions.patch` is the shared persistence boundary for model switches; open PR #75198 is the paired unmerged fix candidate and should not be bypassed by closing this issue early.

## What This Changes

The issue asks for TUI and Control UI mid-session model switches to resolve configured display aliases to canonical provider/model IDs before persistence.

## Best Possible Solution

Review and land or adapt PR #75198 so provider-qualified aliases resolve canonically at the shared resolver/session-patch boundary with regression coverage.

## Reproduction Assessment

Yes. Configure alias `MoE` for `lmstudio-moe/qwen3.6-35b-a3b`, then patch a session with `model: "lmstudio-moe/MoE"`; current source shows it resolves as provider `lmstudio-moe` and model `MoE` after the alias lookup misses.

## Solution Assessment

Yes. Fixing the shared resolver/session-patch boundary is the narrowest maintainable solution because TUI commands, Control UI controls, slash commands, SDK callers, and other session patch users all share that path.
