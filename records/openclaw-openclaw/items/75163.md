## Summary

Keep open. Current main still misses the provider-qualified alias case in the shared resolver, and open PR #75198 is already the linked closing fix candidate for this exact session-switch path.

## What This Changes

The issue reports that TUI/Control UI model switching can persist a provider-qualified display alias instead of the canonical provider model id.

## Best Possible Solution

Review and land or adapt PR #75198 so provider-qualified aliases resolve canonically at the shared model resolver/session patch boundary with regression coverage.

## Reproduction Assessment

Yes. Configure alias MoE for lmstudio-moe/qwen3.6-35b-a3b and patch a session with model lmstudio-moe/MoE; current source shows the alias lookup misses and slash parsing preserves MoE as the provider-facing model id.

## Solution Assessment

Yes. Fixing the shared resolver/session-patch boundary is the narrow maintainable solution because TUI, Control UI, slash commands, and other session patch callers share that path.
