## Summary

Keep open: current main and v2026.6.6 still leave the Control UI/TUI session patch path without provider-qualified alias canonicalization, and the linked fixing PR is still open rather than merged.

## What This Changes

Control UI/TUI model switching should resolve display aliases or provider-qualified alias values to canonical model IDs before persisting session overrides.

## Best Possible Solution

Refresh, review, and land or replace https://github.com/openclaw/openclaw/pull/75198 with a shared resolver/session-patch canonicalization fix that preserves exact provider/model refs and includes regression coverage.

## Reproduction Assessment

Yes. Source inspection shows Control UI sends the selected model directly to `sessions.patch`, and current main resolves only exact whole-string aliases before parsing provider/model, so `provider/Alias` is not canonicalized.

## Solution Assessment

Yes. The maintainable fix belongs in shared model-resolution canonicalization before session persistence; the current issue should stay open until the linked fix is refreshed and merged or replaced.
