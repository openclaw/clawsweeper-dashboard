## Summary

Keep open. Current main still resolves bare aliases but not provider-qualified display aliases such as `lmstudio-moe/MoE`, and the linked open PR #75198 is the correct closing candidate rather than a reason to close this before merge.

## What This Changes

TUI and Control UI model switching should resolve configured aliases to canonical provider model IDs before persisting session overrides.

## Best Possible Solution

Review and land or adapt PR #75198 so provider-qualified aliases resolve canonically at the shared resolver/session-patch boundary with focused regression coverage.

## Reproduction Assessment

Yes. Configure alias `MoE` for `lmstudio-moe/qwen3.6-35b-a3b`, then switch or patch the session to `lmstudio-moe/MoE`; current source shows the alias lookup misses and parsing preserves `MoE` as the provider-facing model id.

## Solution Assessment

Yes. Resolving provider-qualified aliases in the shared resolver/session-patch path is the narrowest maintainable fix because TUI, Control UI, slash commands, and SDK callers all share that boundary.
