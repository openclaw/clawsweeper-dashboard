## Summary

Keep open. Current main still has the provider-qualified alias resolution gap in the shared model resolver used by `sessions.patch`, and open PR #75198 is already the active closing implementation path for this issue.

## What This Changes

The issue requests that Control UI/TUI mid-session model switches resolve configured display aliases to canonical provider/model IDs before persisting session overrides.

## Best Possible Solution

Keep this issue open until #75198 is reviewed and either landed or replaced with an equivalent shared resolver fix plus resolver and sessions.patch regression coverage.

## Reproduction Assessment

Yes, source-level. With alias `MoE` configured for `lmstudio-moe/qwen3.6-35b-a3b`, a `sessions.patch` model value of `lmstudio-moe/MoE` misses the bare alias key and resolves as provider `lmstudio-moe` with model `MoE`.

## Solution Assessment

Yes. Fixing the shared resolver used by `sessions.patch` is the narrow maintainable solution because it covers the Control UI picker and other session-patch callers without LM Studio-specific behavior.
