## Summary

Keep open. Current main still has the provider-qualified alias resolution gap, and open PR #75198 is already the active closing implementation path for this exact `sessions.patch` model resolver defect.

## What This Changes

The issue asks Control UI/TUI model switching to resolve provider-qualified display aliases to canonical model IDs before persisting session overrides.

## Best Possible Solution

Review and land or adapt #75198 so shared model resolution canonicalizes provider-qualified aliases before `sessions.patch` persists user model overrides, with resolver and session-patch regression coverage.

## Reproduction Assessment

Yes, source-level. With alias `MoE` configured for `lmstudio-moe/qwen3.6-35b-a3b`, a `sessions.patch` model value of `lmstudio-moe/MoE` misses the bare alias key and resolves as provider `lmstudio-moe` with model `MoE`.

## Solution Assessment

Yes. Fixing the shared resolver used by `sessions.patch` is the narrow maintainable path because it covers the Control UI picker and other session-patch callers without LM Studio-specific logic.
