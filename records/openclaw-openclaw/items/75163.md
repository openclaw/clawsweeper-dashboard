## Summary

Keep open. Current main still has the provider-qualified alias resolver gap, and the open closing PR at https://github.com/openclaw/openclaw/pull/75198 is already the active fix path for the same `sessions.patch` model-switch defect.

## What This Changes

The issue reports that Control UI/TUI mid-session model switching can persist a provider-qualified display alias instead of the canonical model id.

## Best Possible Solution

Review and land or adapt https://github.com/openclaw/openclaw/pull/75198 so shared model resolution canonicalizes provider-qualified aliases before `sessions.patch` persists user model overrides.

## Reproduction Assessment

Yes, at source level. With alias `MoE` configured for `lmstudio-moe/qwen3.6-35b-a3b`, `sessions.patch` receives `lmstudio-moe/MoE`, misses the bare alias key, and resolves to provider `lmstudio-moe` with model `MoE`.

## Solution Assessment

Yes. Fixing the shared resolver used by `sessions.patch` is the narrowest maintainable path because it covers Control UI/TUI model switching without LM Studio-specific logic.
