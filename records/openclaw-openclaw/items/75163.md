## Summary

Keep open. Current main still misses the provider-qualified alias resolver case, and the open unmerged PR #75198 is already the canonical closing implementation for this issue.

## What This Changes

The issue requests resolving provider-qualified display aliases selected in the Control UI/TUI to canonical provider/model IDs before session overrides are persisted.

## Best Possible Solution

Land or adapt #75198 so shared model selection resolves provider-qualified aliases canonically before session overrides are stored, with resolver and sessions.patch regression coverage.

## Reproduction Assessment

Yes, source-level. With alias "MoE" configured for a canonical lmstudio-moe model, the current resolver checks "lmstudio-moe/MoE" as a full alias key, misses the bare alias entry, and parses the raw display alias as the model id.

## Solution Assessment

Yes. Fixing the shared resolver used by sessions.patch is the narrowest maintainable solution because it covers the Control UI picker and other session model-switch callers without LM Studio-specific logic.
