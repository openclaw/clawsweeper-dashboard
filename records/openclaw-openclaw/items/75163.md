## Summary

Keep open. Current main still has the provider-qualified alias gap in the shared session model resolver, and open PR #75198 is already the active closing implementation path for this issue.

## What This Changes

The issue requests that Control UI/TUI mid-session model switches resolve configured display aliases to canonical provider/model IDs before persisting session model overrides.

## Best Possible Solution

Review and land or adapt #75198 so sessions.patch canonicalizes provider-qualified aliases through the shared resolver before storing user model overrides.

## Reproduction Assessment

Yes, source-level. With alias MoE configured for lmstudio-moe/qwen3.6-35b-a3b, a sessions.patch value of lmstudio-moe/MoE misses the bare alias key and resolves as provider lmstudio-moe with model MoE.

## Solution Assessment

Yes. Fixing the shared resolver used by sessions.patch is the narrow maintainable path because it covers the Control UI picker and any other session-patch caller without provider-specific LM Studio logic.
