## Summary

Keep open. Current main still resolves aliases only as whole raw strings before provider/model parsing, so a provider-qualified alias can still persist as the raw alias model, and open PR #75198 is the paired closing fix candidate.

## What This Changes

Resolve provider-qualified model aliases to canonical provider/model IDs during TUI and Control UI session model switches.

## Best Possible Solution

Review and land or adapt PR #75198 so provider-qualified aliases resolve canonically at the shared resolver/session-patch boundary with regression coverage.

## Reproduction Assessment

Yes. Configure alias `MoE` for `lmstudio-moe/qwen3.6-35b-a3b`, then patch a session with `model: "lmstudio-moe/MoE"`; current source shows the alias lookup misses and parsing preserves `MoE` as the provider-facing model id.

## Solution Assessment

Yes. The shared resolver/session-patch boundary is the narrowest maintainable fix because TUI commands, Control UI controls, slash commands, SDK callers, and other session patch users share that path.
