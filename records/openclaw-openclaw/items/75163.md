## Summary

Keep open. Current main still has the source-level provider-qualified alias failure path, and #75198 is the open unmerged closing PR for the same resolver/session-patch defect.

## What This Changes

The issue reports that Control UI/TUI mid-session model switches should resolve display aliases to canonical provider model IDs before persisting the session override.

## Best Possible Solution

Review and land or adapt #75198 so provider-qualified aliases resolve canonically in the shared resolver before sessions.patch persists user model overrides.

## Reproduction Assessment

Yes, source-level. With alias "MoE" configured for lmstudio-moe/qwen3.6-35b-a3b, a sessions.patch model value of lmstudio-moe/MoE misses the bare alias index and resolves to provider lmstudio-moe, model MoE.

## Solution Assessment

Yes. Fixing the shared resolver used by sessions.patch is the narrowest maintainable solution because it covers the Control UI picker and any other session-patch model switch caller without adding provider-specific LM Studio logic.
