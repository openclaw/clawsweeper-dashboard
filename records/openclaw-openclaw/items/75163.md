## Summary

Keep open. Current main still misses the provider-qualified alias case in the shared session model resolver, and open PR #75198 is already the linked closing implementation candidate for this issue.

## What This Changes

The issue requests that mid-session model switching resolve configured display aliases to canonical provider/model IDs before session persistence.

## Best Possible Solution

Land or adapt PR #75198's shared resolver approach so `provider/Alias` resolves only when the provider prefix matches the configured alias target, with regression coverage for both the resolver and `sessions.patch` persistence boundary.

## Reproduction Assessment

Yes. Configure an allowed model alias such as `agents.defaults.models["lmstudio-moe/qwen3.6-35b-a3b"].alias = "MoE"`, then send `sessions.patch` with `model: "lmstudio-moe/MoE"`; current main parses `MoE` as the raw model suffix instead of resolving the alias.

## Solution Assessment

Yes. The shared resolver/session-patch boundary is the best fix location because TUI, Control UI, slash-command, and SDK callers all flow through `sessions.patch`; a TUI-only workaround would leave other session switch paths divergent.
