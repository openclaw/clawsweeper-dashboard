## Summary

Keep open. Current main still has the Control UI/TUI to `sessions.patch` canonicalization gap, and the linked implementation path at https://github.com/openclaw/openclaw/pull/75198 is still open and unmerged.

## What This Changes

Control UI/TUI model switching should resolve aliased or display model selections to canonical provider/model IDs before session overrides reach providers.

## Best Possible Solution

Review and land or adapt https://github.com/openclaw/openclaw/pull/75198 so shared model resolution canonicalizes provider-qualified aliases before `sessions.patch` persists user model overrides, while checking the newer provider/display-value repro.

## Reproduction Assessment

Yes, at source level. With alias `MoE` configured for `lmstudio-moe/qwen3.6-35b-a3b`, current main indexes `MoE` as `moe` but looks up `lmstudio-moe/moe`, then parses the value as provider `lmstudio-moe` with model `MoE`.

## Solution Assessment

Yes. Fixing the shared resolver before `sessions.patch` persists user overrides is the narrow maintainable solution; the linked PR is the current candidate, but it should be reviewed against the newer display-value repro too.
