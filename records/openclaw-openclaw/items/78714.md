## What This Changes

Codex provider usage should honor proxy environment variables when fetching ChatGPT quota usage data.

## Best Possible Solution

Land the narrow loader fix through the linked PR or an equivalent replacement, with regression coverage and redacted real proxy usage proof, then close this issue through the merged fix.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? I did not establish a live blocked-direct/proxy-enabled network repro in this environment.

## Solution Assessment

Is this the best way to solve the issue? Yes: using the existing `resolveProxyFetchFromEnv(env)` only when no explicit fetch override is provided is the narrow maintainable fix. The best path is to review and land the linked PR or an equivalent patch after proof and validation.
