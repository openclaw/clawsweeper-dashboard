## Summary

Keep open. Current main and latest release still expose `skill_workshop` as create-new plus single-skill update, with no multi-target patch action or create-time guard for update-shaped proposal content.

## What This Changes

The issue asks for Skill Workshop MCP handling that avoids creating a new sibling skill when proposal content is intended to patch existing skills.

## Best Possible Solution

Define the Skill Workshop contract so `create` cannot silently accept clearly update-shaped proposals, or add an explicit structured patch/update path, then update the agent tool, Gateway/schema, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection shows `action=create` targets a new skill name through `proposeCreateSkill`, while `action=update` accepts one `skill_name`; no current path scans create proposal content for existing `skills/<name>/...` targets. I did not run a live MCP/apply flow because this review is read-only.

## Solution Assessment

Unclear. The report identifies a real gap, but maintainers should choose whether the best fix is a small create-time guard/docs clarification or a broader multi-target patch API.
