## Summary

Keep open. Current main and the latest release still have a single-target Skill Workshop contract: `create` records a new skill target, `update` targets one existing skill, and there is no create-time guard or multi-skill patch action for update-shaped proposal content.

## What This Changes

The issue asks Skill Workshop to prevent or explicitly support update-shaped multi-skill proposals instead of letting `action=create` create a new sibling skill.

## Best Possible Solution

Define the public Skill Workshop contract so update-shaped proposal content cannot silently land as a new sibling skill, then cover the chosen guard or patch API in the agent tool, Gateway schema, docs, prompt guidance, and regression tests.

## Reproduction Assessment

Yes. Source inspection shows `action=create` records and applies the requested new skill target, while `action=update` accepts only one existing `skill_name`; I did not run a live MCP flow because this review is read-only.

## Solution Assessment

Unclear. The report identifies a real workflow gap, but maintainers need to choose whether the best fix is a conservative create-time guard/docs update or a broader structured multi-target patch API.
