## Summary

Keep open. The linked bug is real and the PR touches the right Windows scheduled-task surface, but the diff breaks the existing scheduled-task command parsing/fallback contract and has no after-fix Windows proof.

## What This Changes

The PR changes the generated Windows Scheduled Task batch script to launch the node process with `start "" /b` instead of running the command line directly.

## Best Possible Solution

Keep the fix scoped to Windows scheduled-task launching, but preserve the parseable service command contract and prove `install`, `start`, `status`, `stop`, and `restart` on Windows after the launcher change.

## Reproduction Assessment

Yes. The linked issue provides concrete Windows 11 steps, and current source still routes `openclaw node install` through `node.cmd` plus the shared interactive Scheduled Task path; I did not run a live Windows login session in this read-only review.

## Solution Assessment

No. The PR targets the right file, but wrapping the stored command without updating `readScheduledTaskCommand` and fallback launch handling is not the narrow maintainable fix.
