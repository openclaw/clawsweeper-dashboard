## Summary

Current main and the latest stable release already implement the central safe-defaults request: key-free web_search providers, including Parallel Search Free, are selectable but are no longer auto-detected on unconfigured installs.

## What This Changes

The issue asks OpenClaw to stop unconfigured web_search from routing to Parallel.ai/Parallel Search Free without explicit user opt-in.

## Best Possible Solution

Keep the shipped explicit-opt-in behavior and treat duplicate PRs that only remove the old key-free default as superseded by the merged fix.

## Reproduction Assessment

No on current main: the previous source-level failure mode is now covered by regression tests, and current source returns no provider unless a key-backed provider is configured or a key-free provider is explicitly selected.

## Solution Assessment

Yes: the merged fix changes the provider-selection boundary directly by requiring an implicit credential signal for auto-detect while preserving explicit key-free provider selection.
