## Summary

Close as implemented: current main and v2026.6.8 require explicit selection for keyless managed web_search providers, so the actionable hosted Parallel default no longer reproduces; the personal-attribution claims are not source evidence.

## What This Changes

The issue asks OpenClaw to stop routing unconfigured managed web_search traffic to Parallel.ai via the keyless parallel-free provider.

## Best Possible Solution

Keep the current explicit-selection behavior for keyless hosted web_search providers and treat redundant follow-up branches against that policy separately.

## Reproduction Assessment

No for current main: source inspection and current tests show unconfigured managed web_search has no provider and keyless Parallel is only explicit. The older source-level behavior described in the previous ClawSweeper comment has since been changed.

## Solution Assessment

Yes. The current fix is broader than changing one provider's order because it prevents all keyless managed providers from implicit selection while preserving explicit opt-in.
