## Summary

Current main and release v2026.6.8 already solve the actionable OpenClaw concern: managed web_search no longer auto-selects Parallel Search (Free) or other key-free providers when no API-backed provider is configured.

## What This Changes

The issue asks OpenClaw to stop unconfigured web_search from routing through Parallel Search (Free) or another third-party key-free provider without explicit user choice.

## Best Possible Solution

Keep the shipped key-free opt-in behavior from https://github.com/openclaw/openclaw/pull/93616 and handle the now-redundant open follow-up PR separately.

## Reproduction Assessment

No: the former source-level behavior no longer reproduces on current main. Current runtime tests assert keyless providers are not auto-selected without explicit provider configuration.

## Solution Assessment

Yes: the merged fix changed the generic web_search selection path, setup/docs, and regression tests for all key-free providers instead of applying a Parallel-only workaround.
