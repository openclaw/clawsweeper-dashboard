## Summary

Close as implemented: the actionable OpenClaw behavior is fixed on current main and present in the latest release, because keyless managed web_search providers including `parallel-free` are selectable but not implicitly chosen when no provider is configured.

## What This Changes

The issue asks OpenClaw to stop routing unconfigured `web_search` calls to Parallel's hosted keyless provider without explicit user opt-in.

## Best Possible Solution

Keep the shipped explicit-selection behavior for keyless web_search providers and handle the now-redundant open PR separately if maintainers want cleanup.

## Reproduction Assessment

No, not against current main: source and tests show unconfigured managed web_search no longer selects keyless providers such as `parallel-free`. The original zero-config behavior is traceable through the earlier merged Parallel Search PR and previous ClawSweeper review, but the current checkout and latest release no longer match it.

## Solution Assessment

Yes. The current solution is the narrow maintainable fix: keep keyless providers available, but require explicit `tools.web.search.provider` selection instead of adding a new config flag or removing the provider.
