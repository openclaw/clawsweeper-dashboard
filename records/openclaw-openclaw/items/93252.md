## Summary

Close: current main and v2026.6.8 already keep key-free managed web_search providers, including Parallel Search (Free), explicit opt-in instead of auto-selecting them when no API-backed provider is configured.

## What This Changes

The issue reports that zero-config web_search can route to Parallel.ai without explicit user configuration and asks that this traffic not be rerouted unexpectedly.

## Best Possible Solution

Keep the shipped explicit-selection behavior for key-free web_search providers unless maintainers deliberately choose a different default policy later.

## Reproduction Assessment

Yes. The original problem is source-reproducible from the old keyless auto-detect path, and current source/tests now prove unconfigured managed web_search does not select keyless providers such as parallel-free.

## Solution Assessment

Yes. The shipped fix is the narrow maintainable solution: keep key-free providers available for explicit selection while preventing unexpected hosted traffic from zero-config provider resolution.
