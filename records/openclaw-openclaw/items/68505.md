## What This Changes

This PR keeps REM preview body headings at `###` for inline daily notes and lowers them only when writing standalone REM reports, with focused memory-core tests and a changelog entry.

## Best Possible Solution

Land the writer-level REM-only heading normalization after the contributor adds real OpenClaw behavior proof and normal merge checks pass.

## Reproduction Assessment

## Solution Assessment

Yes for the patch shape. The writer is the output-mode boundary, so lowering only REM report headings preserves inline nesting and avoids changing the shared preview generator globally.
