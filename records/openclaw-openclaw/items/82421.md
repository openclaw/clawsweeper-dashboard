## Summary

Keep open: the shared-helper fix is directionally correct, but this PR lacks required after-fix real behavior proof and the patch is incomplete because it can drop styled Markdown IR trailing whitespace and leaves Matrix's inline runtime chunker unaligned.

## What This Changes

This PR trims the final residual chunk in the shared text chunking helper and adds a regression test for final-chunk trailing whitespace.

## Best Possible Solution

Land one aligned fix that trims plain final text chunks while preserving Markdown IR styled whitespace and keeping the Matrix runtime copy plus guardrail in sync.

## Reproduction Assessment

Yes. Source inspection on current main shows over-limit text such as `a b ` can leave `remaining` as `b ` and `src/shared/text-chunking.ts:31` pushes it unchanged; I did not run tests because this review is read-only.

## Solution Assessment

No. Trimming the shared final chunk is the right core direction, but this branch should also preserve Markdown IR styled whitespace, align Matrix's runtime copy, and provide real after-fix proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve styled final whitespace in Markdown IR chunks:** `src/shared/text-chunking.ts:31`
  - body: Trimming the residual chunk here also changes `chunkMarkdownIR`, because `chunkText` delegates to this helper before IR maps chunk length back to style spans. For a fenced code block ending in a styled newline, such as a final chunk `89\n`, this becomes `89` and truncates the code_block span; keep the IR path preserving styled trailing whitespace while trimming plain outbound text.
  - confidence: 0.86
- **[P2] Align the Matrix runtime chunker with the shared fix:** `src/shared/text-chunking.ts:31`
  - body: Matrix's lightweight `runtime-api.ts` carries an inline `chunkTextForOutbound` that still pushes the final residual `remaining` unchanged, and the guardrail test snapshots that inline code. After this change, shared/plugin SDK chunking trims the final tail but Matrix runtime imports can still emit `beta ` for `alpha beta ` with limit 8, so the linked bug remains in that runtime path.
  - confidence: 0.78
