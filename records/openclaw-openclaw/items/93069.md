## Summary

Keep open: current `main` and `v2026.6.9` still accept or advertise `diagnostics.otel.protocol: "grpc"`, while diagnostics-otel only starts OTLP exporters for `http/protobuf`; the linked fix candidate is open but unmerged.

## What This Changes

The issue asks OpenClaw to make `diagnostics.otel.protocol: "grpc"` either a working runtime path or a rejected/repairable config value rather than an accepted but ignored setting.

## Best Possible Solution

Review and land a clean version of `https://github.com/openclaw/openclaw/pull/93087` with the schema/docs/doctor migration alignment, unless maintainers choose the larger real gRPC exporter implementation instead.

## Reproduction Assessment

Yes. Source inspection on current main shows `grpc` accepted or advertised by config surfaces while the runtime exits before OTLP startup for non-`http/protobuf`.

## Solution Assessment

Unclear until maintainers choose the compatibility direction. The narrow repair is to reject `grpc` in canonical config/docs with a doctor migration; implementing real gRPC export is a larger feature path.
