## Summary

Keep open: current main and `v2026.6.8` still accept or advertise `diagnostics.otel.protocol: "grpc"`, while the diagnostics-otel runtime only starts OTLP export for `http/protobuf`. The open fixing PR at https://github.com/openclaw/openclaw/pull/93087 is the right candidate path, but it is unmerged and currently reported as conflicting, so this issue remains necessary.

## What This Changes

Align `diagnostics.otel.protocol` so `grpc` either has real gRPC exporter support or is rejected consistently across validation, help, docs, runtime behavior, and tests.

## Best Possible Solution

Land a focused fix that either removes `grpc` from the canonical config contract with doctor repair for shipped configs, or deliberately implements real gRPC exporters with matching dependencies, docs, and tests.

## Reproduction Assessment

Yes. Source inspection reproduces the mismatch: schema/types/help/docs accept or advertise `grpc`, while the diagnostics-otel runtime returns before exporter startup for any protocol other than `http/protobuf`.

## Solution Assessment

Yes, the issue is valid and the narrowest maintainable path is to reject `grpc` with a doctor migration unless maintainers intentionally choose the larger real gRPC exporter implementation.
