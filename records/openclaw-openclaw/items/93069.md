## Summary

Keep open: current main and the latest release still accept and advertise `diagnostics.otel.protocol: "grpc"`, while the diagnostics-otel runtime returns before exporter startup for non-`http/protobuf`; the linked fix candidate is still open and should land or be replaced before this issue closes.

## What This Changes

The issue requests making `diagnostics.otel.protocol` validation, help, docs, and diagnostics-otel runtime behavior agree for the currently accepted `grpc` value.

## Best Possible Solution

Land or replace the linked fix with a migration-safe HTTP/protobuf-only contract, unless maintainers explicitly choose the larger gRPC exporter implementation.

## Reproduction Assessment

Yes. Source inspection shows the config schema/type/help accept or advertise `grpc`, while the diagnostics-otel runtime returns before exporter construction for any protocol other than `http/protobuf`.

## Solution Assessment

Yes, aligning the public config contract with runtime support is the right fix; the narrowest maintainable path appears to be rejecting `grpc` with doctor/upgrade handling unless maintainers intentionally choose to implement gRPC export.
