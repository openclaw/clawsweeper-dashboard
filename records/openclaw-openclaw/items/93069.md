## Summary

Keep open: current main and the latest release still accept or advertise `diagnostics.otel.protocol: "grpc"`, while the diagnostics-otel runtime only starts OTLP exporters for `http/protobuf`. The open linked fix PR is the right candidate path, but it has not merged yet, so this issue remains necessary.

## What This Changes

The issue requests that the accepted and documented `diagnostics.otel.protocol` values match the diagnostics-otel runtime behavior for gRPC.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/93087 lands or is replaced; the preferred end state is a single supported protocol contract with doctor migration for shipped `grpc` configs, or a deliberate full gRPC exporter implementation.

## Reproduction Assessment

Yes. Source inspection on current main shows the schema/type/help/reference paths accept or advertise `grpc`, while the diagnostics-otel runtime returns before exporter startup for any protocol other than `http/protobuf`; I did not run tests because this was a read-only checkout review.

## Solution Assessment

Yes, if maintainers accept the compatibility tradeoff. Rejecting `grpc` with a doctor migration is the narrow maintainable fix under current runtime support; implementing real gRPC export remains a larger feature alternative.
