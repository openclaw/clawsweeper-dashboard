## Summary

Keep open: current `main` and the latest stable release still accept or advertise `diagnostics.otel.protocol: "grpc"`, while the diagnostics-otel runtime only starts exporters for `http/protobuf`. The linked fixing PR is open and unmerged, so this issue should remain open until that PR, or an equivalent migration-aware fix, lands.

## What This Changes

Align the documented and validated `diagnostics.otel.protocol` contract with the diagnostics OTel runtime's actual protocol support.

## Best Possible Solution

Land a narrow fix that either rejects unsupported `grpc` with schema/help/docs updates plus doctor migration for shipped configs, or implements real gRPC exporters end to end; keep this issue open until that fix merges.

## Reproduction Assessment

Yes. Source inspection shows current `main` accepts and advertises `grpc` in config surfaces while the diagnostics-otel runtime emits `unsupported_protocol` and returns for any protocol other than `http/protobuf`.

## Solution Assessment

Yes, aligning the contract is the right fix, but a schema-only rejection is not the best landing shape. The maintainable fix needs either doctor migration for the shipped `grpc` config value or a maintainer decision to implement real gRPC exporter support.
