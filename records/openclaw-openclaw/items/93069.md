- `impact:other`: The concrete impact is an observability configuration foot-gun where a validated protocol can disable telemetry export at runtime.

## Summary

Keep open: current main and the latest release still accept and advertise `grpc` for `diagnostics.otel.protocol`, while the diagnostics-otel runtime returns before starting export for any protocol other than `http/protobuf`; no linked or searched PR currently fixes that mismatch.

## What This Changes

The issue asks to make `diagnostics.otel.protocol` validation, help/reference docs, handwritten docs, and diagnostics-otel runtime support agree for `grpc`.

## Best Possible Solution

Align on one supported contract: either reject `grpc` in schema/types/help/docs with clear validation and doctor/upgrade handling, or implement real gRPC exporter support across plugin dependencies, runtime, tests, and docs.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: the schema/type/help/reference paths accept or advertise `grpc`, while the diagnostics-otel service returns before exporter startup for any protocol other than `http/protobuf`.

## Solution Assessment

Unclear until maintainers choose the config contract. Rejecting `grpc` looks like the smaller repair, but because `grpc` is already in the shipped schema, the best fix needs explicit compatibility or doctor handling.
