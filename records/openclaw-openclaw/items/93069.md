## Summary

Keep open: current main and the latest release still accept and advertise `diagnostics.otel.protocol: "grpc"`, while the diagnostics-otel runtime only starts exporters for `http/protobuf`. The linked fix candidate is still open and currently conflicting, so this issue should close only after that PR or an equivalent fix merges.

## What This Changes

The issue asks OpenClaw to align the `diagnostics.otel.protocol` config contract with diagnostics-otel runtime support so `grpc` either works or is rejected with clear upgrade-safe handling.

## Best Possible Solution

Land a narrow upgrade-safe fix that either rejects `grpc` with matching type/schema/help/docs changes plus `openclaw doctor --fix` migration, or implements real gRPC exporters with dependencies, runtime support, and tests.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main accepts and advertises `grpc`, while the diagnostics-otel service emits `unsupported_protocol` and returns before exporter startup for any protocol other than `http/protobuf`.

## Solution Assessment

No, not as a closed item yet. The best path is to keep this issue open until the linked fix PR lands or maintainers choose an equivalent contract-alignment patch with doctor/upgrade handling.
