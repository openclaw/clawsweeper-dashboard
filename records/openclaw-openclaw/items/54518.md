## Summary

Keep open. Current main and v2026.5.22 still make host filesystem confinement opt-in through a boolean `tools.fs.workspaceOnly`, tests preserve out-of-workspace access when it is unset or false, and the related filesystem-policy PRs are either closed unmerged or still open for security/product review.

## What This Changes

The issue requests hard per-agent filesystem confinement so agents cannot read or mutate files outside their own workspace.

## Best Possible Solution

Land an owner-reviewed filesystem isolation contract that aligns defaults, path-rule semantics, read/write/edit/apply_patch, local media, exec/process scope, sandbox behavior, subagent inheritance, docs, and upgrade proof.

## Solution Assessment

No for a quick default flip. A durable fix needs a maintainer-approved security contract for defaults, ACL shape, exec/process scope, filesystem-adjacent tools, and upgrade compatibility.
