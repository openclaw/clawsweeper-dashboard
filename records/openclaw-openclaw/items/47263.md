  - body: `finish()` now records the acknowledgement unconditionally, but users can still jump away from page 0 through the progress dots because those buttons do not check the new gate.   - confidence: 0.9
- **[P1] Preserve env and launchd auth when gating node mode:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:147`
  - body: The fallback here only checks `openclaw.json`, but current macOS gateway auth also comes from `OPENCLAW_GATEWAY_TOKEN`, `OPENCLAW_GATEWAY_PASSWORD`, and the launchd snapshot. After an onboarding-version bump, existing local installs that use those sources can be disconnected until users rerun onboarding even though the gateway is already configured.
  - confidence: 0.86
- **[P2] Keep readiness probes on the injectable gateway connection:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:202`
  - body: Switching attach/start/readiness checks to `probeLocalGatewayHealth` bypasses `self.connection`, so the existing `setTestingConnection` seam no longer drives these paths and tests that fake gateway readiness fall through to a real socket/channel. Keep the shared injectable connection, or inject the probe helper with equivalent test coverage.
  - confidence: 0.84

  - body: Returning nil for `.debug` bundle identifiers makes both `CLIInstaller.install()` and the manual command fall back to `latest`, which changes tester/developer builds from a bundle-version target to a moving remote install target. If that behavior is intended, make it explicit and gated; otherwise keep a pinned version for debug builds too.
  - confidence: 0.78

  - body: The manual install path only copies the command, while the visible recheck button appears only for Command Line Tools preflight failures. If a user runs the copied command in Terminal, `cliInstalled` may not refresh and Next can stay disabled until they leave and re-enter the page.
