Full review comments:

- **[P1] Keep health probes local and credential-safe:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:415-419`
  - body: `GatewayEndpointStore.localConfig()` can resolve `gateway.bind=custom` or `tailnet` hosts and returns token/password values from config, env, or launchd. Building a fresh `GatewayChannelActor` from that config means these automatic process-manager probes can send gateway credentials to a non-loopback `ws://` endpoint during startup; keep this probe on a local no-secret path or explicitly validate the endpoint before sending auth.
  - confidence: 0.93
- **[P1] Preserve env and launchd auth in the node gate:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:45-49`
  - body: The new onboarding gate only treats installs as configured when `openclaw.json` has wizard or gateway auth data. Current gateway resolution also accepts `OPENCLAW_GATEWAY_TOKEN` and the saved launchd snapshot, so upgraded local installs using those supported auth sources will now disconnect node mode until users rerun onboarding.
  - confidence: 0.87
- **[P2] Block acknowledgement bypass before finishing:** `apps/macos/Sources/OpenClaw/OnboardingView+Actions.swift:61-64`
  - body: `finish()` now writes the security acknowledgement unconditionally, but the progress dots can still move a first-run user off page 0 without ticking the checkbox because only `Next` checks `canAdvance`. Route dot navigation and finish through the same acknowledgement invariant before persisting this flag.
  - confidence: 0.91
- **[P2] Preserve the injected gateway connection in readiness waits:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:380`
  - body: Current main has tests and debug hooks that inject a `GatewayConnection` to drive gateway readiness deterministically. The new `probeLocalGatewayHealth` bypasses `self.connection`, so those paths depend on a real local socket instead of the injected test/session contract.
  - confidence: 0.82
