## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Block page-dot navigation until acknowledgement:** `apps/macos/Sources/OpenClaw/OnboardingView+Layout.swift:115-116`
  - body: The new acknowledgement only disables Next. These dots can jump a fresh user past page 0, and `finish()` then persists the acknowledgement as true without the toggle ever being selected. Disable forward dot navigation while this gate is active or stop recording acknowledgement unless it was actually accepted.
  - confidence: 0.99
- **[P1] Include env and launchd auth in node-mode upgrade detection:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:44-49`
  - body: This upgrade gate only inspects `openclaw.json`, but local gateway credentials are also valid from `OPENCLAW_GATEWAY_TOKEN`/`PASSWORD` and the launchd snapshot. Those existing installs are classified as unconfigured and node mode is disconnected until onboarding is rerun; use the canonical endpoint/auth resolver or pass all supported sources into the predicate.
  - confidence: 0.97
- **[P1] Keep process-manager health probes on a trusted local endpoint:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:415-419`
  - body: `localConfig()` can resolve a custom or tailnet host and includes token/password credentials, so the new automatic startup polling may send local gateway secrets to a non-loopback endpoint, including over `ws://`. Build this process-manager probe from a loopback-only endpoint or explicitly validate transport and host before attaching credentials.
  - confidence: 0.95
