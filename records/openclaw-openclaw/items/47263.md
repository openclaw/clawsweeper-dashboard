## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.97

Full review comments:

- **[P1] Require acknowledgement before persisting completion:** `apps/macos/Sources/OpenClaw/OnboardingView+Actions.swift:58`
  - body: The progress dots can jump past page 0, while `finish()` always stores the acknowledgement. A fresh user can therefore finish without selecting the toggle; validate acknowledgement in `finish()` and prevent forward navigation across the gate.
  - confidence: 0.99
- **[P2] Add an in-page recheck for manual CLI installs:** `apps/macos/Sources/OpenClaw/OnboardingView+Monitoring.swift:43-48`
  - body: `Next` remains disabled until `cliInstalled` changes, but a Terminal installation is checked only on page entry or after the built-in installer. Users following the displayed manual command are stranded unless they leave the page or restart onboarding; add a visible recheck or bounded polling path.
  - confidence: 0.99
- **[P1] Recognize all supported auth sources before disconnecting node mode:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:145-147`
  - body: This fallback calls `hasExistingLocalSetup(root:)`, which only sees `openclaw.json`, while `GatewayEndpointStore.localConfig()` also accepts environment and launchd snapshot credentials. Existing installations using those supported sources will be disconnected until onboarding is rerun; use canonical resolved setup state.
  - confidence: 0.98
- **[P1] Constrain process-manager probes to a safe local endpoint:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:414-421`
  - body: `GatewayEndpointStore.localConfig()` can resolve custom or tailnet hosts and supplies token/password credentials, so this automatic startup probe can send secrets to a non-loopback `ws://` endpoint. Use a process-manager-owned loopback endpoint or validate host and TLS before attaching credentials.
  - confidence: 0.95
- **[P2] Preserve the injected readiness connection:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:201`
  - body: These readiness loops now call a newly constructed channel instead of `self.connection`, so `setTestingConnection` no longer controls attach and wait behavior and the existing fake-connection tests attempt a real endpoint. Keep the probe behind the injected connection or add a dedicated injectable probe dependency.
  - confidence: 0.97
