## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep process health probes on a local-only auth path:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:415-419`
  - body: `probeLocalGatewayHealth()` now builds a fresh channel from `GatewayEndpointStore.localConfig()`, which can resolve tailnet/custom hosts and includes token/password. These process-manager probes run automatically during attach/start/wait, so startup can transmit gateway credentials to a non-loopback bind instead of checking the local process; keep this probe on the injected/shared local connection or force a loopback/no-secret health path.
  - confidence: 0.91
- **[P1] Gate page-dot jumps on the security acknowledgement:** `apps/macos/Sources/OpenClaw/OnboardingView+Layout.swift:115-124`
  - body: The new acknowledgement only disables the Next button. The progress dots still set `currentPage` with only the wizard lock applied, and `finish()` then persists the acknowledgement unconditionally, so a first-run user can skip the notice by clicking a later dot and finishing onboarding.
  - confidence: 0.93
- **[P2] Preserve env-backed auth when gating node mode:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:145-147`
  - body: This fallback only inspects `openclaw.json` through `hasExistingLocalSetup(root:)`. Current macOS gateway resolution also treats `OPENCLAW_GATEWAY_TOKEN` / `OPENCLAW_GATEWAY_PASSWORD` and launchd snapshots as valid local auth, so upgraded installs using those sources can have node mode disconnected until they rerun onboarding.
  - confidence: 0.86
- **[P2] Add a recheck path after copied CLI installs:** `apps/macos/Sources/OpenClaw/OnboardingView+Pages.swift:744-751`
  - body: The manual install path only copies the command while `Next` remains disabled until `cliInstalled` refreshes. The refreshes happen on page entry or after the in-app installer path, so a user who installs in Terminal has no explicit recheck on this page and may have to leave the page or trigger an unnecessary reinstall.
  - confidence: 0.84
- **[P2] Do not silently force debug installs to latest:** `apps/macos/Sources/OpenClaw/GatewayEnvironment.swift:98-103`
  - body: Returning `nil` for every `.debug` bundle makes the installer fall back to `openclaw@latest` even when a debug build has a bundle version. Because the installer executes a remote install script, this should be an explicit development opt-in or a pinned versioned install path rather than an implicit bundle-id side effect.
  - confidence: 0.78
