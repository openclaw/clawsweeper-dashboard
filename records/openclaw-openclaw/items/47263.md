  - body: `finish()` now always persists the security acknowledgement, but users can still jump away from page 0 with the progress dots because dot navigation only checks the wizard lock. A first-run user can skip the checkbox, land on the final page, and have the acknowledgement recorded anyway.
  - confidence: 0.92
- **[P1] Keep local probes from sending secrets off-host:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:415-419`
  - body: This new probe builds a `GatewayChannelActor` from `GatewayEndpointStore.localConfig()`, which can resolve tailnet/custom hosts and includes token/password credentials. Startup health polling can therefore send gateway credentials to a non-loopback or plain-WS endpoint before the app has proven it is talking to a local process.
  - confidence: 0.9
- **[P2] Preserve env-backed auth before blocking node mode:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:147`
  - body: The upgrade fallback only checks `openclaw.json` through `hasExistingLocalSetup(root:)`, but the macOS gateway endpoint resolver treats `OPENCLAW_GATEWAY_TOKEN`, `OPENCLAW_GATEWAY_PASSWORD`, and launchd snapshot secrets as valid local auth. Users relying on those existing sources can have node mode disconnected until they rerun onboarding.
  - confidence: 0.87
- **[P2] Add a same-page recheck for manual installs:** `apps/macos/Sources/OpenClaw/OnboardingView+Pages.swift:747-749`
  - body: The manual install path copies a command, but `Next` remains disabled until `cliInstalled` refreshes and this page has no CLI recheck button unless Command Line Tools are missing. Users who complete the copied install in Terminal can stay stuck on the page until they leave and re-enter onboarding.
  - confidence: 0.84
