Full review comments:

- **[P1] Gate finish on the security acknowledgement:** `apps/macos/Sources/OpenClaw/OnboardingView+Actions.swift:62`
  - body: `finish()` now records the security acknowledgement unconditionally, but the page dots can still jump past page 0 because they only honor the wizard lock. A first-run user can skip the checkbox, finish onboarding, and still persist the acknowledgement as true.
  - confidence: 0.93
- **[P1] Preserve env-backed local auth before blocking node mode:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:147`
  - body: The new onboarding gate falls back only to `openclaw.json` via `hasExistingLocalSetup`, but current endpoint resolution also accepts `OPENCLAW_GATEWAY_TOKEN` / `OPENCLAW_GATEWAY_PASSWORD` and launchd snapshot credentials for local gateways. Existing installs that rely on those auth sources can be disconnected until the user reruns onboarding even though the gateway remains configured.
  - confidence: 0.88
- **[P1] Keep local health probes loopback-only:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:414-419`
  - body: `probeLocalGatewayHealth()` uses `GatewayEndpointStore.localConfig()`, which can resolve tailnet or custom hosts and includes local token/password values. Because `GatewayChannelActor` sends those credentials in the connect frame, startup polling can transmit local gateway secrets to a non-loopback endpoint.
  - confidence: 0.86
- **[P2] Restore the injected readiness path:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:380`
  - body: Replacing readiness checks with `probeLocalGatewayHealth()` bypasses the existing `setTestingConnection` injection used by `GatewayProcessManagerTests`. The PR's own readiness test can no longer drive `waitForGatewayReady` with its fake connection and instead depends on a real local socket.
  - confidence: 0.82
- **[P2] Add a recheck path for manual CLI installs:** `apps/macos/Sources/OpenClaw/OnboardingView+Pages.swift:747-749`
  - body: The manual install path only copies a command while `Next` stays disabled until `cliInstalled` refreshes. If the user runs the copied command in Terminal, this page has no visible recheck button unless they leave and re-enter the page or trigger the app installer path.
  - confidence: 0.8
