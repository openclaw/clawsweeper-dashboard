## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep local health probes from sending secrets remotely:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:415-419`
  - body: `GatewayEndpointStore.localConfig()` can resolve `gateway.bind=tailnet` or `custom` to a non-loopback URL and include config/env/launchd token or password values. Because `GatewayChannelActor` sends those credentials in the `connect` frame before `health`, the new startup polling path can leak local gateway secrets to a remote `ws://` endpoint. Constrain this process-manager probe to loopback/no secrets or reuse the existing injected connection path safely.
  - confidence: 0.94
- **[P1] Preserve env and launchd auth when gating node mode:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:45-49`
  - body: This gate only inspects `openclaw.json`, but current local gateway auth can also come from `OPENCLAW_GATEWAY_TOKEN`, `OPENCLAW_GATEWAY_PASSWORD`, or the launchd snapshot. Upgraded users with those valid auth sources will have node mode disconnected until they rerun onboarding, even though the gateway is configured.
  - confidence: 0.9
- **[P2] Require acknowledgement before every finish path:** `apps/macos/Sources/OpenClaw/OnboardingView+Actions.swift:61-63`
  - body: `finish()` now records the security acknowledgement unconditionally, while the page dots can still jump past page 0 because only `canAdvance` is blocked. A first-run user can navigate to the final page and finish without ticking the acknowledgement, making the new gate ineffective.
  - confidence: 0.92
- **[P2] Avoid implicit latest installs for debug builds:** `apps/macos/Sources/OpenClaw/GatewayEnvironment.swift:98-103`
  - body: Returning `nil` for `.debug` bundle identifiers makes both `CLIInstaller.install()` and the copied command fall back to `openclaw@latest`. Since the installer path executes a network-fetched shell script, debug builds should not silently switch to a moving package without an explicit opt-in or pinned version.
  - confidence: 0.82
- **[P2] Add a recheck path for manual CLI installs:** `apps/macos/Sources/OpenClaw/OnboardingView+Pages.swift:744-748`
  - body: The manual path only copies the install command, but `Next` remains disabled until `cliInstalled` refreshes. Users who run the copied command in Terminal have no in-page way to recheck success unless the CLT preflight branch is active, so the onboarding page can remain stuck.
  - confidence: 0.86
