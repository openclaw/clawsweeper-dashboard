- **[P1] Enforce acknowledgement across page-dot navigation:** `apps/macos/Sources/OpenClaw/OnboardingView+Actions.swift:62`
  - body: `finish()` now persists the security acknowledgement unconditionally, but the progress dots can still jump a first-run user past page 0. That lets onboarding finish without the checkbox ever being selected, making the warning ineffective.
  - confidence: 0.89
- **[P1] Preserve env and launchd auth in the node-mode gate:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:147`
  - body: The fallback only checks `openclaw.json` via `hasExistingLocalSetup(root:)`, but current gateway auth resolution also accepts `OPENCLAW_GATEWAY_TOKEN`, `OPENCLAW_GATEWAY_PASSWORD`, and launchd snapshots. Upgraded local installs using those sources will disconnect node mode until onboarding is rerun.
  - confidence: 0.88
- **[P1] Keep process health probes loopback-safe and reusable:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:415-419`
  - body: `probeLocalGatewayHealth()` creates a fresh `GatewayChannelActor` from `GatewayEndpointStore.localConfig()` on each polling attempt and passes token/password into it. That bypasses the shared/injected connection path and needs a loopback-safe, reusable probe contract before startup loops use it repeatedly.
  - confidence: 0.82
- **[P2] Add a same-page recheck for manual CLI installs:** `apps/macos/Sources/OpenClaw/OnboardingView+Pages.swift:747-749`
  - body: The new manual copy path does not refresh `cliInstalled` after the user installs in Terminal, while `canAdvance` keeps Next disabled until that state flips. Users can get stuck on the CLI page unless they leave and re-enter onboarding.
  - confidence: 0.86
- **[P2] Keep debug CLI installs version-pinned:** `apps/macos/Sources/OpenClaw/GatewayEnvironment.swift:99-103`
  - body: Returning `nil` for `.debug` bundle identifiers makes the installer and displayed dev command fall back to `latest`. That turns debug/dev onboarding into an unpinned network installer path, which is not a good default for a code-execution setup flow.
  - confidence: 0.8
