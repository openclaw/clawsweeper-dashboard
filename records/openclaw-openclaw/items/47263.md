  - confidence: 0.91

  - body: `finish()` now persists `openclaw.onboardingSecurityAcknowledged` unconditionally, but the progress dots still jump pages without checking `canAdvance`. A first-run user can skip page 0, reach the final page, and mark the security notice as accepted without ticking the checkbox.
  - confidence: 0.92
- **[P1] Preserve env and launchd auth in node-mode startup:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:147`
  - body: This gate falls back only to `OnboardingWizardModel.hasExistingLocalSetup(root:)`, which inspects `openclaw.json`. Current endpoint resolution also treats `OPENCLAW_GATEWAY_TOKEN`, `OPENCLAW_GATEWAY_PASSWORD`, and launchd snapshot credentials as valid local auth sources, so upgraded installs using those sources can have node mode disconnected until they rerun onboarding.
  - confidence: 0.86
- **[P2] Keep debug CLI installs pinned or explicit:** `apps/macos/Sources/OpenClaw/GatewayEnvironment.swift:99-103`
  - body: Returning `nil` for `.debug` bundle identifiers makes `CLIInstaller.install()` and the manual command fall back to `latest`. That changes debug/tester builds from a bundle-version pin to a moving remote install-script target without an explicit opt-in.
  - confidence: 0.82
- **[P2] Preserve the injected readiness connection:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:380`
  - body: `waitForGatewayReady()` now bypasses `self.connection`, so `setTestingConnection()` no longer controls this path even though the PR head still has a test expecting a fake `GatewayConnection` to drive readiness. Keep the probe injectable or update the test seam before merge.
  - confidence: 0.87
- **[P2] Add a manual CLI install recheck:** `apps/macos/Sources/OpenClaw/OnboardingView+Pages.swift:747-749`
  - body: The new manual install path only copies a command while `Next` stays disabled until `cliInstalled` is refreshed.   - confidence: 0.84
