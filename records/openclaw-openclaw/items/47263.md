- **[P1] Keep local health probes loopback-safe:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:415-419`
  - body: `localConfig()` can resolve `tailnet` or `custom` hosts and returns token/password credentials, and `GatewayChannelActor` sends those credentials in the connect frame before the health request. These process-manager probes are now automatic attach/start checks, so a local startup probe can repeatedly authenticate to a non-loopback endpoint instead of proving the local launchd Gateway is ready.
  - confidence: 0.9
- **[P1] Preserve env-backed auth before blocking node mode:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:145-147`
  - body: This fallback only inspects `openclaw.json`, but existing local installs may get gateway token/password from `OPENCLAW_GATEWAY_TOKEN`, `OPENCLAW_GATEWAY_PASSWORD`, or the launchd snapshot. After an onboarding version bump those users can be disconnected from node mode even though the app's normal gateway config resolver would still authenticate successfully.
  - confidence: 0.86
- **[P1] Require acknowledgement before finishing:** `apps/macos/Sources/OpenClaw/OnboardingView+Actions.swift:61-63`
  - body: `finish()` writes the new acknowledgement key unconditionally, while page-dot navigation still jumps directly to later pages without consulting `canAdvance`. A first-run user can skip the checkbox, navigate to the last page, and persist the acknowledgement anyway, so the new security gate is not enforceable.
  - confidence: 0.9
- **[P2] Add a same-page CLI recheck:** `apps/macos/Sources/OpenClaw/OnboardingView+Pages.swift:747-750`
  - body: The manual install path only copies the command, but `Next` remains disabled until `cliInstalled` refreshes; the normal manual-install case has no recheck button on this page. Users who run the copied command successfully in Terminal are stuck until they leave and re-enter onboarding or restart it.
  - confidence: 0.84
