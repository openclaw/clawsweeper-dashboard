Full review comments:

- **[P1] Block dot navigation before acknowledging security:** `apps/macos/Sources/OpenClaw/OnboardingView+Actions.swift:62`
  - body: `finish()` now persists the security acknowledgement unconditionally, but the progress dots still let a first-run user jump away from page 0 without checking the box and then finish onboarding. Gate all page transitions, or only persist this key after an explicit acknowledgement.
  - confidence: 0.92
- **[P2] Keep node mode aware of env and launchd auth:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:147`
  - body: This fallback only checks `openclaw.json`, while current macOS local auth can come from `OPENCLAW_GATEWAY_TOKEN`, `OPENCLAW_GATEWAY_PASSWORD`, or the launchd snapshot. Upgraded local installs using those sources can be disconnected until they rerun onboarding even though their gateway auth is valid.
  - confidence: 0.88
- **[P2] Preserve injected gateway health checks:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:380`
  - body: `waitForGatewayReady` now calls `probeLocalGatewayHealth`, which creates a fresh channel and ignores the existing `setTestingConnection` seam. The current GatewayProcessManager tests drive readiness through that injected connection, so this path becomes dependent on a real local socket under tests.
  - confidence: 0.86
- **[P2] Add a manual install recheck path:** `apps/macos/Sources/OpenClaw/OnboardingView+Pages.swift:747`
  - body: The CLI page disables Next until `cliInstalled` is true, but after a user copies and runs the manual install command there is no visible recheck on this page unless the CLT preflight is failing. Add a refresh/recheck action for the manual path so users are not stuck after a successful terminal install.
  - confidence: 0.84
- **[P2] Avoid implicit latest installs for debug builds:** `apps/macos/Sources/OpenClaw/GatewayEnvironment.swift:102`
  - body: Returning `nil` for `.debug` bundle identifiers makes the install flow fall back to `openclaw@latest` instead of the app bundle version. That silently changes developer/tester builds to a moving remote install target; keep a pinned version or require an explicit opt-in for latest.
  - confidence: 0.78
