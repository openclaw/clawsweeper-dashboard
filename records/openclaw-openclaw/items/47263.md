## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep local health probes on loopback:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:415-419`
  - body: `probeLocalGatewayHealth` uses `GatewayEndpointStore.localConfig()`, which can resolve `gateway.bind=custom` or `tailnet` and current env/launchd auth. Because `GatewayChannelActor` sends token/password during connect, startup polling can transmit local gateway credentials to a configured non-loopback `ws://` endpoint. Build this process-manager probe from loopback or validate before sending auth.
  - confidence: 0.91
- **[P1] Enforce acknowledgement before finishing:** `apps/macos/Sources/OpenClaw/OnboardingView+Actions.swift:62`
  - body: `finish()` now persists `openclaw.onboardingSecurityAcknowledged` unconditionally, but the progress dots can jump away from page 0 without going through `canAdvance`. A first-run user can reach the final page and persist acknowledgement without ticking the checkbox; gate all page transitions or only write this key after explicit acknowledgement.
  - confidence: 0.9
- **[P1] Preserve env and launchd auth in node-mode gating:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:45-49`
  - body: The new node-mode gate only loads `openclaw.json` and calls `hasExistingLocalSetup(root:)`. Current endpoint resolution also treats `OPENCLAW_GATEWAY_TOKEN`, `OPENCLAW_GATEWAY_PASSWORD`, and launchd snapshot auth as valid local gateway auth, so upgraded users using those sources can have node mode disconnected until they rerun onboarding.
  - confidence: 0.88
- **[P2] Add a recheck path for manual CLI installs:** `apps/macos/Sources/OpenClaw/OnboardingView+Pages.swift:747-748`
  - body: The manual path only copies the install command while the CLI page's Next button requires `cliInstalled`. Since readiness is refreshed only on page entry or after `installCLI()`, a user who installs in Terminal stays stuck until they leave and re-enter the page; add a recheck button or lightweight polling for the manual path too.
  - confidence: 0.86
- **[P2] Preserve the injected readiness connection:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:202`
  - body: Switching readiness checks from `self.connection.requestRaw` to `probeLocalGatewayHealth` bypasses the DEBUG `setTestingConnection` hook used by existing GatewayProcessManager tests. Keep the probe injectable or route through the existing connection abstraction so readiness tests do not depend on a real local socket.
  - confidence: 0.82
