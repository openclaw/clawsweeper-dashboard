## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep local probes from using configurable remote hosts:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:415-419`
  - body: `probeLocalGatewayHealth()` now builds a fresh `GatewayChannelActor` from `GatewayEndpointStore.localConfig()` and passes token/password into it. `localConfig()` can resolve `gateway.bind=tailnet` or `custom` plus `ws://`, and the channel sends those credentials in the connect auth frame before the health request, so startup polling can leak gateway credentials to a non-loopback endpoint. Keep process-manager probes on loopback without shared credentials, or reject non-loopback/unencrypted auth before connecting.
  - confidence: 0.94
- **[P1] Gate all onboarding navigation on the security acknowledgement:** `apps/macos/Sources/OpenClaw/OnboardingView+Layout.swift:123-124`
  - body: The acknowledgement gate only disables the Next button through `canAdvance`; the page-dot buttons still set `currentPage` directly and are disabled only by the wizard lock. A first-run user can jump to a later page, finish onboarding, and `finish()` will persist the acknowledgement without the checkbox ever being selected. Route dot navigation through the same guard or disable non-zero dots until acknowledged.
  - confidence: 0.91
- **[P2] Add a recheck path for manual CLI installs:** `apps/macos/Sources/OpenClaw/OnboardingView+Pages.swift:747-750`
  - body: On the new CLI page, Next is disabled until `cliInstalled` is true, but the manual path only copies `devLinkCommand` and does not refresh after the user runs it in Terminal. Because refreshes only happen on page entry or the in-app installer paths, a successful manual install leaves the user stuck on the page until they leave and re-enter onboarding. Add an always-visible recheck/refresh action or equivalent polling after copy/manual install.
  - confidence: 0.88
- **[P2] Preserve injected gateway connections in readiness waits:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:380`
  - body: `waitForGatewayReady()` now calls `probeLocalGatewayHealth()` instead of the `self.connection` wrapper, so DEBUG tests that inject a fake `GatewayConnection` no longer drive readiness; the remaining `GatewayProcessManagerTests` still sets a test connection and expects the wait to succeed. Keep the injected connection path for tests, or add a probe dependency that tests can supply without opening a real local socket.
  - confidence: 0.82
