  - confidence: 0.93
- **[P1] Keep process-manager health probes off remote credential paths:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:415-419`
  - body: This creates a GatewayChannelActor from localConfig() and passes token/password. localConfig() can resolve tailnet/custom hosts and ws://, while GatewayChannelActor sends auth in the connect frame, so startup polling can leak gateway credentials outside loopback.
  - confidence: 0.9
- **[P1] Preserve env and launchd auth when gating node mode:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:45-49`
  - body: The new onboarding gate only inspects openclaw.json via hasExistingLocalSetup(root:). Existing local installs whose auth lives in OPENCLAW_GATEWAY_TOKEN, OPENCLAW_GATEWAY_PASSWORD, or the launchd plist will be disconnected after an onboarding-version bump even though GatewayEndpointStore still treats those as valid auth sources.
  - confidence: 0.86
- **[P2] Avoid implicit latest installs for debug builds:** `apps/macos/Sources/OpenClaw/GatewayEnvironment.swift:99-103`
  - body: Returning nil for .debug bundle identifiers flows into CLIInstaller.install() as latest, which is then executed through the network install script. Debug/test builds should stay pinned or require an explicit opt-in for moving latest code.
  - confidence: 0.82
