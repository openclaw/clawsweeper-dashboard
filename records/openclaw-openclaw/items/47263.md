## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Gate page-dot navigation on acknowledgement:** `apps/macos/Sources/OpenClaw/OnboardingView+Layout.swift:116`
  - body: The new security acknowledgement only blocks `Next`, but these page dots still jump to later pages whenever the wizard lock allows it. A first-run user can skip page 0, reach the final page, and `finish()` then persists the acknowledgement without the checkbox ever being set, so the new security gate is bypassable.
  - confidence: 0.93
- **[P1] Preserve env-backed node-mode setup:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:147`
  - body: This fallback only checks `openclaw.json` through `hasExistingLocalSetup(root:)`, but the macOS gateway endpoint store also treats `OPENCLAW_GATEWAY_TOKEN`, `OPENCLAW_GATEWAY_PASSWORD`, and launchd snapshots as valid local auth sources. Upgraded installs using those sources can now have node mode disconnected until onboarding is rerun despite having a usable gateway config.
  - confidence: 0.88
- **[P2] Keep gateway readiness checks injectable:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:380`
  - body: `waitForGatewayReady` now calls `probeLocalGatewayHealth` directly, so `setTestingConnection` no longer affects this path. The existing Swift test still injects a fake `GatewayConnection` and expects readiness to succeed without a real local socket, so this change breaks the deterministic test seam and likely the test itself.
  - confidence: 0.89
- **[P2] Add a manual CLI install recheck:** `apps/macos/Sources/OpenClaw/OnboardingView+Pages.swift:748`
  - body: The manual install path only copies the command while `Next` stays disabled until `cliInstalled` is refreshed. Since refresh happens on page entry or after the built-in installer/preflight path, users who run the copied command in Terminal can remain stuck on this page unless they leave and re-enter onboarding.
  - confidence: 0.86
- **[P2] Avoid implicit latest installs in debug builds:** `apps/macos/Sources/OpenClaw/GatewayEnvironment.swift:102`
  - body: Returning `nil` for `.debug` bundle identifiers flows into `CLIInstaller.install()` as `latest`, which then executes the remote install script. That makes debug/test builds pull a moving CLI version without an explicit opt-in or pin, which is a supply-chain regression for real developer machines.
  - confidence: 0.78
