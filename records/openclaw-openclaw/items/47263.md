- **[P1] Gate page-dot navigation on the acknowledgement:** `apps/macos/Sources/OpenClaw/OnboardingView+Layout.swift:116`
  - body: The new security notice only blocks `Next`, but the page dots still allow a first-run user to jump past page 0 and reach `finish()`, which writes the acknowledgement key. Treat this gate as a page-transition invariant so the acknowledgement cannot be bypassed through the progress dots.
  - confidence: 0.94
- **[P2] Add a recheck path for manual CLI installs:** `apps/macos/Sources/OpenClaw/OnboardingView+Pages.swift:747-749`
  - body: After copying the manual install command, nothing refreshes `cliInstalled` while the user stays on this page, but `Next` is disabled until that state flips. Add an explicit recheck or polling path so a successful Terminal install can advance without leaving and re-entering onboarding.
  - confidence: 0.9
- **[P2] Preserve env and launchd auth in node-mode gating:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:147`
  - body: `shouldConnectNodeMode()` now uses only the JSON root to decide whether an existing local setup can keep running, but the gateway endpoint resolver accepts `OPENCLAW_GATEWAY_TOKEN`, `OPENCLAW_GATEWAY_PASSWORD`, and launchd snapshots as valid local auth. Upgraded users with those auth sources can be disconnected until they rerun onboarding.
  - confidence: 0.88
- **[P2] Keep readiness probes on the injected connection:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:380`
  - body: The new readiness probe constructs a fresh `GatewayChannelActor` directly, bypassing `self.connection` and the DEBUG `setTestingConnection` hook. Existing gateway process tests and future deterministic readiness checks can no longer drive these paths with a fake session and will fall through to a real local socket.
  - confidence: 0.86
