- **[P1] Keep local probes from sending auth off-loopback:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:414-418`
  - body: `probeLocalGatewayHealth()` uses `GatewayEndpointStore.localConfig()`, which can resolve custom or tailnet hosts and include token/password auth; `GatewayChannelActor` then sends those secrets in the connect frame during an automatic startup probe. Keep process-manager health checks on the existing configured/loopback-safe path, or explicitly reject non-loopback and avoid sending secrets.
  - confidence: 0.9
- **[P1] Gate completion on explicit acknowledgement:** `apps/macos/Sources/OpenClaw/OnboardingView+Actions.swift:62-63`
  - body: `finish()` now writes the security acknowledgement unconditionally, but progress dots can still jump past page 0 without using `canAdvance`. A first-run user can skip the checkbox, finish onboarding, and persist the acknowledgement anyway, so all page transitions and completion need to enforce the acknowledgement state.
  - confidence: 0.92
- **[P1] Preserve env-backed auth for node startup:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:145-147`
  - body: `shouldConnectNodeMode()` falls back only to `OnboardingWizardModel.hasExistingLocalSetup(root:)`, which inspects `openclaw.json`. Existing local installs that keep `OPENCLAW_GATEWAY_TOKEN`, `OPENCLAW_GATEWAY_PASSWORD`, or launchd-stored auth will be treated as unconfigured and node mode will disconnect until onboarding is rerun.
  - confidence: 0.86
- **[P2] Keep readiness probes injectable:** `apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:377-380`
  - body: Switching `waitForGatewayReady()` and attach/start loops away from `self.connection` bypasses the DEBUG `setTestingConnection` hook that current tests use to fake gateway health. This makes gateway readiness depend on a real local socket in tests and drops the configured connection contract those paths already exercise.
  - confidence: 0.84
- **[P2] Add a manual CLI recheck path:** `apps/macos/Sources/OpenClaw/OnboardingView+Pages.swift:744-748`
  - body: The manual install path copies a command, but `Next` stays disabled until `cliInstalled` refreshes; the new refreshes only run on page entry or after the in-app installer. A user who installs from Terminal while staying on this page has no visible way to recheck and advance.
  - confidence: 0.82
- **[P2] Keep debug installs pinned or explicitly opted in:** `apps/macos/Sources/OpenClaw/GatewayEnvironment.swift:98-102`
  - body: Returning nil for `.debug` bundle identifiers makes downstream CLI install and the displayed command use `latest`. That changes debug/tester builds from the bundle version to a moving remote installer target without an explicit opt-in or integrity proof.
  - confidence: 0.78
