## Summary

Keep open. ## What This Changes

The PR adds an opt-in deferred Gateway activation mode with a token-gated loopback control port, CLI env-variable opt-in, activation-time config revalidation, supervised-lock recovery changes, and Gateway/CLI tests.

## Best Possible Solution

Keep deferred activation opt-in only if parked health preserves the same effective bind/listen contract as active Gateway startup, or explicitly rejects unsupported bind modes with docs and maintainer approval for the new env/control surface.

## Reproduction Assessment

Yes from source inspection: with deferred activation enabled, the parked placeholder always listens on 127.0.0.1 even when normal Gateway startup would resolve LAN, container auto, custom, or IPv6 loopback listen hosts. I did not run a live socket repro in this read-only review.

## Solution Assessment

No; the proposed implementation is coherent but not the best fix until parked mode shares the normal Gateway listen-host resolution or explicitly rejects unsupported bind modes. The new env/control surface also needs maintainer product and security-boundary sign-off.
