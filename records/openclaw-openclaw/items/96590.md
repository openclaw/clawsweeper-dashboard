## Summary

Keep open: the deferred activation idea is useful, but the patch is not merge-ready because the parked Gateway placeholder binds only IPv4 loopback, so it does not preserve the configured Gateway-port health/listen contract for LAN/container/custom-bind deployments while parked.

## What This Changes

The PR adds an opt-in deferred Gateway activation mode with a token-gated loopback control port, CLI env opt-in, activation-time config rechecks, supervised recovery updates, and Gateway/CLI tests.

## Best Possible Solution

Keep the feature opt-in, but make the parked Gateway health listener preserve the same configured bind/listen contract as active startup or narrow deferred mode to loopback-only with explicit validation and docs.

## Reproduction Assessment

Yes from source inspection: with deferred activation enabled, the parked placeholder always listens on 127.0.0.1 even when normal Gateway startup would resolve a LAN, auto/container, or IPv6 loopback listen set.

## Solution Assessment

No; the proposed implementation is coherent but not the best fix until deferred parking shares normal Gateway listen-host resolution or explicitly rejects unsupported bind modes.
