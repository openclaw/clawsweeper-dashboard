## What This Changes

Add configurable rate limiting, throttling, retry/backoff, token or cost budgets, observability, alerts, and CLI controls for external API calls.

## Best Possible Solution

Keep one canonical design tracker and approve a phased, opt-in model-provider dispatch policy first, backed by canonical SQLite accounting and existing status surfaces; retain channel delivery throttles and specialized pace or per-agent budget behavior in their narrower owner-specific trackers unless the design establishes a shared generic seam.
