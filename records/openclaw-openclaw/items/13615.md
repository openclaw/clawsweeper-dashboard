## Summary

Keep open. Current main has adjacent reactive provider cooldown/fallback handling, gateway auth rate limiting, Codex usage-limit readouts, and plugin-specific retry/rate-limit handling, but it still lacks the requested configurable outbound API limiter with queueing, token/cost budgets, warning/status UX, and `openclaw limits` commands.

## What This Changes

Request configurable rate limiting, throttling/backoff, budget controls, observability, and CLI status/set/reset commands for external API calls.

## Best Possible Solution

Keep this as the canonical maintainer-scoped design tracker for an opt-in outbound policy layer, starting with model-provider dispatch and explicitly deciding how tools, channels/plugins, persistence, warnings, 429 backoff, status UX, and budgets fit.
