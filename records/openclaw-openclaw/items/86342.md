## What This Changes

The issue asks OpenClaw to stop user-visible `MissingAgentHarnessError` failures when plugin cache or registry reload paths clear agent harness registrations around Discord, cron, or gateway agent turns.

## Best Possible Solution

Keep this issue open as the canonical remaining registry lifecycle item, then review or replace https://github.com/openclaw/openclaw/pull/87046 with a fix that stabilizes harness availability without masking truly missing explicit runtimes.

## Reproduction Assessment

Yes at source level, but not as a live timing repro in this read-only review. Current main still has harness-clearing plugin reload paths plus fail-closed explicit plugin-runtime selection, while the exact `claude-cli` alias throw is already fixed.

## Solution Assessment

Unclear as a final fix. Atomic Map restore alone would not settle empty-scope reload semantics, and a retry-only fix could hide real missing-runtime configuration errors; the best path is a lifecycle fix with explicit maintainer-approved preservation rules.
