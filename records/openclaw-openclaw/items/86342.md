## What This Changes

This issue reports `MissingAgentHarnessError` during Discord/cron dispatch when plugin loader cache cycles clear and restore the process-wide agent harness registry.

## Best Possible Solution

Keep this as the canonical lifecycle bug and resolve it through maintainer-approved, ownership-aware harness preservation semantics with focused loader/gateway coverage and real gateway proof.

Yes at source level: current main has deterministic empty-scope paths that clear the harness registry without restoration, and selection still reads that global registry before throwing. I did not run a live gateway timing repro in this read-only review.

No single proposed option is proven best yet. The narrow fix should choose explicit harness lifecycle ownership semantics rather than blindly restoring all prior harnesses or masking real missing-runtime states with retries.
