## Summary

Keep open: current main and the latest release still have global agent-harness registry clearing during plugin loader cache/reset paths, and the Discord dispatch path can surface the same MissingAgentHarnessError from a queued lane. The exact production timing window was not reproduced in this read-only pass, and the safest fix boundary needs maintainer review because retrying MissingAgentHarnessError can mask real missing-runtime configuration errors.

## What This Changes

Investigate a reported race where plugin loader cache clearing can temporarily unregister agent harnesses and surface MissingAgentHarnessError during Discord lane dispatch.

## Best Possible Solution

Keep the issue open and fix the runtime lifecycle so active harness availability is stable across plugin cache cycles, with any bounded retry preserving fail-closed behavior for truly missing explicit runtimes and regression coverage for Discord lane dispatch.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established in this read-only pass. Source inspection does show the relevant failure path: plugin runtime state can be cleared and explicit harness selection throws MissingAgentHarnessError when the runtime is not registered.

## Solution Assessment

Unclear. The report's atomic Map restore suggestion removes one narrow clear/restore concern, but the broader loader lifecycle and retry-vs-fail-closed boundary need maintainer choice before a safe fix is selected.
