Keep open: current main and v2026.6.8 still have the process-global harness clear/restore and empty-plugin-scope clear paths, and the open candidate PR is still unresolved around ownership semantics and mergeability.

## What This Changes

The issue asks OpenClaw to prevent `MissingAgentHarnessError` during plugin cache/reload cycles by making harness registry lifecycle changes atomic or adding bounded readiness/retry behavior.

## Best Possible Solution

Keep this as the canonical lifecycle bug, settle ownership-aware harness preservation semantics, then land loader/gateway/registry tests plus patched-gateway proof.

## Reproduction Assessment

Yes at source level: current main still has empty-scope loader and gateway paths that clear the process-global harness registry, and registry restore still clears then repopulates the map. I did not run a live gateway timing repro in this read-only review.

## Solution Assessment

No single proposed option is proven best yet: atomic restore does not settle empty-scope ownership, retry-only behavior can hide real missing runtimes, and readiness gating is broader than the observed path. The best fix needs maintainer-approved lifecycle semantics.
