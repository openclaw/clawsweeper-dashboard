## Summary

This PR is still aimed at the right Feishu P1 regression, but it is not merge-ready: the new runtime seeding uses the gateway startup channel runtime, which is only a lightweight `runtimeContexts` surface for bundled channels, while Feishu inbound handling needs the full channel runtime. The supplied proof is still mocked/containerized test output rather than redacted live Feishu or gateway evidence.

## What This Changes

The branch forwards Feishu gateway `channelRuntime` into `monitorFeishuProvider`, seeds Feishu runtime from either a plugin-shaped runtime or that channel runtime, and adds mocked startup regression coverage.

## Best Possible Solution

Keep this PR open, but require a fix that seeds Feishu from the full `PluginRuntime` used by inbound dispatch and add redacted live Feishu or gateway proof before merge.

## Reproduction Assessment

Yes from source inspection: current gateway startup gives bundled Feishu a startup `channelRuntime` containing only `runtimeContexts`, while Feishu handlers require `channel.debounce` and `channel.inbound.run`. I did not run a live Feishu reproduction in this read-only review.

## Solution Assessment

No. Forwarding `ctx.channelRuntime` into `setFeishuRuntime` is not the best fix because bundled startup uses a partial runtime; the safer repair is to seed Feishu from the existing full `PluginRuntime` path or explicitly route full channel runtime before account monitors register handlers.
