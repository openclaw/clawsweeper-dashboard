## What This Changes

The PR adds a plugin-packaging recovery classifier and routes selected config validation and gateway startup errors away from the generic `openclaw doctor --fix` hint toward publisher update/reinstall or disable/uninstall guidance.

## Best Possible Solution

Use one shared packaging-aware recovery decision across config validation, foreground gateway startup, and daemon lifecycle start/restart preflight while preserving the doctor hint for normal invalid configs.

## Reproduction Assessment

## Solution Assessment
