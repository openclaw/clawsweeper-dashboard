## Summary

Keep open. Current main still lacks an injected plugin-runtime gateway accessor while the requested Gateway RPCs already exist for normal Gateway clients; this is a new public SDK seam with security, replay, and type-contract implications.

## What This Changes

The issue requests a plugin SDK runtime surface that lets plugins call selected read-only host Gateway RPCs such as `status` and `models.list`.

## Best Possible Solution

Add a narrow documented `api.runtime.gateway` namespace for read-only status/model queries, with allowlisting, redaction rules, exported public types, and replay snapshot semantics.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken existing behavior. Source inspection confirms the requested runtime surface is absent while the underlying host RPCs are present.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade looks safer than exposing a broad `GatewayClient`, but maintainers should settle allowlisting, redaction, public types, and replay behavior first.
