## Summary

Keep open: current main and `v2026.6.6` still lack the requested plugin runtime accessors for host `status`/`models.list`, and the existing Gateway dispatcher is intentionally limited to authenticated plugin HTTP request scopes. The related broad inter-plugin issue and the closed raw-dispatch PR do not supersede this narrower security-sensitive SDK design item.

## What This Changes

Request a plugin SDK/runtime surface for plugins to invoke selected host Gateway RPCs such as `status` and `models.list`.

## Best Possible Solution

Keep this open as a focused plugin SDK design item and add narrow read-only Gateway accessors, or an allowlisted request surface, with caller attribution, authorization, replay behavior, public types, docs, and contract tests.

## Reproduction Assessment

Yes, at source level: current main exposes plugin Gateway method registration and host `status`/`models.list` handlers, but `PluginRuntime` has no normal plugin Gateway caller and the only SDK dispatcher is restricted to authenticated plugin HTTP routes.

## Solution Assessment

Unclear. Curated read-only accessors look safer than a broad raw request API, but maintainers still need to choose the allowlist, caller identity, authorization, public type surface, and replay contract.
