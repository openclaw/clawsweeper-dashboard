## Summary

Keep open. Current main still lacks an injected `api.runtime.gateway` or equivalent plugin-runtime request facade even though the requested `status` and `models.list` Gateway handlers exist; resolving this is a new public SDK capability with security and replay-contract decisions.

## What This Changes

Add a plugin-sdk runtime surface that lets plugins invoke selected read-only host Gateway RPCs such as `status` and `models.list`.

## Best Possible Solution

Add a narrow documented `api.runtime.gateway` read-only facade with explicit method allowlisting, status redaction rules, public response types, replay snapshot semantics, tests, docs, and a changelog entry.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request for a new SDK capability. Source inspection on current main confirms the injected plugin runtime lacks the requested Gateway accessor while the underlying Gateway handlers exist.

## Solution Assessment

Unclear until maintainer API review. Curated read-only accessors look safer than exposing a generic Gateway client, but allowlisting, redaction, public types, and replay behavior need to be settled first.
