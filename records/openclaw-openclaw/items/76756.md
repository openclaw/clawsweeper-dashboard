## Summary

Keep open. Current main still has no injected plugin runtime surface for curated host gateway RPC reads, while the underlying `status` and `models.list` gateway methods exist and are callable by normal gateway clients; this is a new public SDK/API seam with security and replay semantics, not cleanup closure material.

## What This Changes

Request a plugin-sdk `PluginRuntime` gateway accessor so plugins can read host runtime status and model catalog data without exposing arbitrary gateway RPC calls.

## Best Possible Solution

Add a narrow documented `api.runtime.gateway` read-only facade for status and model catalog reads, with allowlisting, redaction rules, exported response types, and replay snapshot semantics.

## Reproduction Assessment

Not applicable; this is a feature request rather than a failing existing contract. Source inspection confirms the requested runtime surface is absent while the host RPCs are present.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade appears safer than exposing `GatewayClient` or arbitrary `request()`, but the allowlist, redaction, type exports, and replay contract need to be settled together.
