## Summary

Keep open: current main and v2026.6.10 still lack ordinary plugin runtime access to the existing status/models.list Gateway RPCs. The remaining work is a public Plugin SDK and security-boundary decision, not conservative cleanup.

## What This Changes

The issue requests a public plugin SDK surface for plugins to call curated host Gateway RPCs such as status and models.list.

## Best Possible Solution

Add a narrow read-only PluginRuntime.gateway contract only after maintainer approval of allowlist, caller attribution, sensitive-field behavior, replay snapshot semantics, public types, docs, and contract tests.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.10 shows existing status/models.list Gateway handlers and plugin Gateway method registration, but no ordinary PluginRuntime gateway caller; the only dispatcher is reserved for authenticated plugin HTTP routes.

## Solution Assessment

Unclear until maintainers choose the public SDK shape. Curated read-only accessors look safer than a raw Gateway request API, but the allowlist, caller identity, sensitive-field policy, exported types, and replay behavior need product/security review.
