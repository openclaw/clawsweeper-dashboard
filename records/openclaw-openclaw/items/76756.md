## Summary

Keep open. Current main still has the underlying Gateway RPCs but no public PluginRuntime gateway accessor, and adding one is a new SDK contract with redaction, allowlist, type-export, and replay semantics to settle.

## What This Changes

The issue requests a public plugin SDK surface that lets plugins call selected read-only host Gateway RPCs such as status and models.list.

## Best Possible Solution

Add a narrow documented api.runtime.gateway namespace for read-only status/model queries, with allowlisting, redaction rules, exported public types, and replay snapshot semantics.

## Reproduction Assessment

Not applicable; this is a feature request rather than a bug reproduction. Source inspection on current main confirms the requested runtime surface is absent while status and models.list exist as Gateway RPCs.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade looks safer than exposing a broad GatewayClient/request surface, but maintainers should settle allowlisting, redaction, public types, and replay behavior first.
