## Summary

Keep open. Current main still lacks the requested injected `api.runtime.gateway` accessor even though gateway clients can call `status` and `models.list`; this is a new public plugin SDK seam with redaction, replay, and type-export decisions, so it needs maintainer API review rather than cleanup closure.

## What This Changes

The issue requests a public plugin runtime surface for invoking selected read-only host gateway RPCs such as `status` and `models.list`.

## Best Possible Solution

Add a narrow documented `api.runtime.gateway` namespace for read-only status/model queries, with explicit allowlisting, redaction rules, exported public response types, and replay snapshot semantics aligned with #76758 and #76759.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing public contract. Source inspection confirms the requested injected runtime surface is absent on current main while the underlying gateway RPCs exist.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade looks safer than exposing the broad `GatewayClient`, but maintainers should settle allowlisting, redaction, type exports, and replay behavior first.
