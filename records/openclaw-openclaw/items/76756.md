## Summary

Keep open. Current main still lacks an injected plugin-runtime gateway accessor while the requested Gateway RPCs already exist and are callable by normal Gateway clients; this is a new public SDK seam with redaction, replay, and type-export decisions rather than cleanup-closeable work.

## What This Changes

The issue requests a curated `PluginRuntime` gateway accessor or allowlisted request method for read-only host RPCs such as `status` and `models.list`.

## Best Possible Solution

Add a narrow documented `api.runtime.gateway` surface for read-only status/model queries, with explicit allowlisting, redaction rules, exported public types, and replay snapshot semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request rather than a broken existing contract. Source inspection on current main confirms the requested runtime surface is absent while the underlying Gateway RPC handlers exist.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade appears safer than exposing a broad `GatewayClient`, but maintainers should settle allowlisting, redaction, public types, and replay behavior first.
