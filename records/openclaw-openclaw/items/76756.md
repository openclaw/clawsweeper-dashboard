## Summary

Keep open. Current main still lacks an injected plugin-runtime Gateway accessor, while the requested `status` and `models.list` RPCs exist; adding this is a new public SDK seam with API, redaction, allowlist, type-export, and replay-contract decisions still unresolved.

## What This Changes

Request a plugin SDK runtime surface that lets plugins invoke curated read-only host Gateway RPCs such as `status` and `models.list`.

## Best Possible Solution

Design and document a narrow `api.runtime.gateway` facade with allowlisted read-only accessors, redacted status behavior, public response types, and deterministic replay snapshot semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request for a new public SDK capability. Source inspection on current main confirms the runtime lacks a Gateway accessor while the underlying `status` and `models.list` RPC handlers exist.

## Solution Assessment

Unclear until maintainer API review. Curated read-only accessors look safer than exposing a generic Gateway client, but maintainers need to settle allowlisting, redaction, public types, and replay behavior first.
