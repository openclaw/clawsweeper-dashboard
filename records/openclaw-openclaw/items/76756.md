## Summary

Keep open. Current main still lacks an injected plugin-runtime Gateway accessor while the requested `status` and `models.list` Gateway RPCs exist, so this is a public SDK/API design item with security and replay semantics rather than cleanup-closeable work.

## What This Changes

Expose a plugin SDK runtime Gateway surface so plugins can call curated read-only host RPCs such as `status` and `models.list`.

## Best Possible Solution

Add a narrow documented `api.runtime.gateway` facade for read-only status/model queries, with allowlisted methods, redaction rules, exported public types, regression tests, and replay snapshot semantics aligned with the related follow-ups.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request for a new SDK seam. Source inspection on current main confirms the requested runtime surface is absent while the underlying Gateway handlers exist.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade appears safer than exposing `GatewayClient.request`, but maintainers should decide allowlisting, redaction, type exports, and replay behavior first.
