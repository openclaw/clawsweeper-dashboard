## Summary

Keep open. Current main still lacks a PluginRuntime gateway accessor while the requested `status` and `models.list` RPCs exist for Gateway clients; this is a new public SDK seam with allowlist, redaction, replay, and type-export decisions.

## What This Changes

The issue requests a read-only `PluginRuntime` gateway API so native plugins can read host status and model catalog RPC data.

## Best Possible Solution

Add a narrow documented `api.runtime.gateway` surface for read-only status/model queries with explicit allowlisting, redaction defaults, exported public types, and replay snapshot semantics.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source inspection on current main confirms the requested runtime surface is absent while the underlying Gateway RPCs are present.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade appears safer than exposing a broad `GatewayClient`, but maintainers need to settle allowlisting, redaction, replay behavior, and public types.
