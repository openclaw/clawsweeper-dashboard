## Summary

Keep open. Current main still lacks a plugin runtime gateway accessor, and this is a public SDK/security contract request rather than stale cleanup.

## What This Changes

Add a plugin SDK runtime gateway surface so plugins can read selected host gateway RPCs such as `status` and `models.list`.

## Best Possible Solution

Add a narrow documented `api.runtime.gateway` read-only facade with allowlisting, redacted status behavior, exported public types, regression tests, and replay semantics aligned with #76758 and #76759.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken documented behavior. Source inspection on current main confirms the requested runtime surface is absent while the underlying gateway RPCs exist.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade looks safer than exposing a broad GatewayClient, but the final shape should settle allowlisting, redaction, replay snapshots, and public type exports.
