## Summary

Keep open. Current main still lacks an injected plugin runtime gateway accessor while the host `status` and `models.list` RPCs exist for gateway clients, and the requested change is a new public SDK seam that needs maintainer API, redaction, and replay-policy review.

## What This Changes

Request a plugin-sdk runtime surface that lets plugins call allowlisted read-only host gateway RPCs such as `status` and `models.list`.

## Best Possible Solution

Add a narrow documented `runtime.gateway` namespace for read-only status/model queries, with allowlisting, redaction, public response types, tests, and replay snapshot semantics aligned with #76758 and #76759.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing OpenClaw contract. Source inspection confirms the requested runtime surface is absent while the underlying host RPCs are present.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade appears safer than exposing a broad `GatewayClient`, but maintainers should decide allowlisting, redaction, type exports, and replay behavior first.
