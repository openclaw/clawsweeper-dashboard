## Summary

Keep open. Current main still has the underlying Gateway `status` and `models.list` RPCs, but the public plugin runtime exposes no `runtime.gateway` accessor, so this remains a valid SDK feature request that needs maintainer API review.

## What This Changes

Add a public plugin runtime gateway surface so plugins can call allowlisted read-only host RPCs such as `status` and `models.list`.

## Best Possible Solution

Add a narrow, documented plugin runtime gateway namespace for read-only status/model queries, with allowlisting, redaction, exported public types, and replay semantics coordinated with #76758 and #76759.

## Reproduction Assessment

Not applicable; this is a feature request. Source inspection confirms current main exposes the underlying host RPCs to Gateway clients but not through the plugin runtime SDK surface.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade appears safer than exposing a broad `GatewayClient`, but the implementation must settle redaction and deterministic replay behavior first.
