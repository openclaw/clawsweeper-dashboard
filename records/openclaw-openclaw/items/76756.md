## Summary

Keep open. Current main still lacks an injected plugin runtime gateway accessor while the host `status` and `models.list` RPCs exist for gateway clients; the remaining work is a new public SDK seam with redaction and replay-design implications.

## What This Changes

Request adding a curated plugin SDK runtime surface that lets plugins call read-only host gateway RPCs such as `status` and `models.list`.

## Best Possible Solution

Add a narrow documented plugin runtime gateway namespace for read-only status/model queries, with method allowlisting, redaction rules, exported public types, and replay snapshot semantics.

## Reproduction Assessment

Not applicable; this is a feature/API request rather than a broken existing contract. Source inspection on current main confirms the requested `runtime.gateway` surface is absent while the underlying gateway RPC handlers exist.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade looks safer than exposing a broad `GatewayClient` or generic request surface, but maintainers should settle allowlisting, redaction, type exports, and replay behavior first.
