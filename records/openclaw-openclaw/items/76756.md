## Summary

Keep open. Current main still lacks an injected plugin-runtime gateway accessor while the requested Gateway RPCs exist for normal Gateway clients; this is a new public SDK contract with API-shape, redaction, and replay implications.

## What This Changes

Add a plugin-runtime SDK surface that lets plugins invoke curated read-only host Gateway RPCs such as `status` and `models.list`.

## Best Possible Solution

Add a narrow documented `runtime.gateway` facade for read-only status/model queries, with allowlisting, redaction rules, exported public types, and replay snapshot semantics.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken existing behavior. Source inspection confirms the requested runtime surface is absent while the underlying Gateway RPCs are present.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade appears safer than exposing a broad `GatewayClient`, but maintainers need to settle allowlisting, redaction, public types, and replay behavior first.
