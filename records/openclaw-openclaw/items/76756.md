## Summary

Keep open. Current main still lacks an injected plugin runtime gateway accessor, and the request is a public SDK/security/replay contract decision rather than a cleanup-close candidate.

## What This Changes

Request to add a curated `PluginRuntime` gateway surface so plugins can read host RPCs such as `status` and `models.list`.

## Best Possible Solution

Add a narrow documented `runtime.gateway` facade for read-only status/model queries, with allowlisting, redaction, exported public types, and replay snapshot semantics.

## Reproduction Assessment

Not applicable; this is a feature request. Source inspection confirms the runtime surface is absent while the underlying host RPCs exist and are used by other clients.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade is safer than exposing a broad `GatewayClient`, but the final contract should settle redaction and deterministic replay behavior first.

## Security Review

Status: needs_attention

Summary: The requested API would expose host runtime state to plugins, so it needs explicit allowlist and redaction rules.
