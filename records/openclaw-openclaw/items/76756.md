## Summary

Keep open: current main still lacks general plugin runtime access to host Gateway RPCs like `status` and `models.list`; the newer `plugin-sdk/gateway-method-runtime` path is reserved for authenticated plugin HTTP routes and does not solve plugin tool/runtime reads.

## What This Changes

Request to add a safe plugin SDK/runtime surface for plugins to invoke selected host Gateway RPCs such as `status` and `models.list`.

## Best Possible Solution

Define one maintainer-approved read-only Gateway accessor contract, or explicitly fold this into the canonical Gateway-call API issue, with allowlisted methods, plugin identity, exported types, docs, tests, and replay/snapshot semantics.

## Reproduction Assessment

Yes, at source level. Current `PluginRuntime` lacks a general `gateway` caller surface, and the only current dispatch helper rejects callers outside entitled authenticated plugin HTTP-route scope.

## Solution Assessment

Unclear. Curated read-only accessors look safer than a raw dispatcher, but maintainers still need to choose the public SDK shape and reconcile it with the existing reserved HTTP-route helper and https://github.com/openclaw/openclaw/issues/39000.
