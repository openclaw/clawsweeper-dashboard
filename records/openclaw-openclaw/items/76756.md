## Summary

Keep open. Current main still lacks an injected plugin runtime accessor for host gateway RPCs, while the `status` and `models.list` gateway handlers exist; adding that path is a new public SDK seam with security and replay-design implications.

## What This Changes

Request a public plugin-sdk runtime gateway accessor so plugins can call read-only host RPCs such as `status` and `models.list`.

## Best Possible Solution

Add a narrow, documented `api.runtime.gateway` surface for approved read-only status/model queries, with explicit allowlisting, redaction, exported response types, tests, and replay snapshot semantics.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source inspection confirms the requested runtime surface is absent while the underlying gateway RPCs are present.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade appears safer than exposing a broad `GatewayClient`, but maintainers should settle allowlisting, redaction, type exports, and replay behavior first.
