## Summary

Keep open. Current main still lacks a plugin runtime gateway accessor even though `status` and `models.list` exist for gateway clients, and the requested surface is a new public SDK/security/replay contract that needs maintainer API review.

## What This Changes

Add a plugin SDK runtime surface that lets plugins call curated read-only host gateway RPCs such as `status` and `models.list`.

## Best Possible Solution

Add a narrow documented `runtime.gateway` read-only facade with explicit allowlisting, redacted status semantics, exported public types, replay behavior, SDK baseline coverage, and plugin docs.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source inspection gives a high-confidence absence check: the runtime surface is missing while the underlying gateway methods exist.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade looks narrower and safer than exposing `GatewayClient.request`, but it should be designed together with the linked replay and type-export follow-ups.
