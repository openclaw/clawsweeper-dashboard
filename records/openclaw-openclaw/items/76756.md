## Summary

Keep open. Current main still lacks an injected plugin-runtime Gateway accessor while the requested Gateway RPCs already exist; resolving this would add a new public SDK seam with security, type-export, and replay semantics decisions.

## What This Changes

The issue requests a public plugin-sdk runtime surface for plugins to call selected read-only host Gateway RPCs such as `status` and `models.list`.

## Best Possible Solution

Add a narrow documented `api.runtime.gateway` read-only facade with explicit method allowlisting, status redaction rules, exported public response types, and replay snapshot semantics aligned with the related open design issues.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request for a new SDK capability. Source inspection confirms current main lacks the requested runtime surface while the underlying Gateway RPC handlers exist.

## Solution Assessment

Unclear until maintainer API review. Curated read-only accessors look safer than exposing a generic Gateway client, but maintainers need to settle allowlisting, redaction, public types, and replay behavior first.
