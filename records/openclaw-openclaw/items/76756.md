## Summary

Keep open. Current main still lacks the requested injected plugin runtime gateway accessor; the underlying `status` and `models.list` gateway RPCs exist, but the plugin API currently exposes registration of plugin-owned methods and separate gateway client helpers rather than a curated read-only `runtime.gateway` contract.

## What This Changes

Add a plugin SDK runtime surface that lets plugins call curated read-only host gateway RPCs such as `status` and `models.list`.

## Best Possible Solution

Add a narrow documented `runtime.gateway` facade with allowlisted redacted read-only status/model access, public types, contract tests, docs, and replay semantics aligned with #76758 and #76759.

## Reproduction Assessment

Not applicable; this is a feature request. Source inspection on current main confirms the requested injected `runtime.gateway` surface is absent while the underlying gateway RPCs are implemented and used by the TUI.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade appears safer than exposing a broad `GatewayClient` or generic request method, but redaction, allowlisting, exported types, and replay behavior should be specified first.

## Security Review

Status: needs_attention

Summary: Any implementation must preserve gateway authorization/redaction boundaries for host runtime state exposed to plugins.

Concerns:

- **[medium] Preserve gateway status redaction for plugins:** `src/gateway/server-methods/health.ts:126`
  - body: The current `status` RPC derives `includeSensitive` from connected client scopes; an in-process plugin runtime accessor would bypass that client auth path unless it intentionally returns redacted output or enforces an equivalent trusted policy.
  - confidence: 0.84
- **[medium] Keep plugin gateway calls allowlisted:** `src/gateway/server-methods-list.ts:31`
  - body: A generic runtime request helper would be able to name any gateway method unless it enforces a small read-only allowlist; the method table includes mutating surfaces such as config, sessions, and update operations.
