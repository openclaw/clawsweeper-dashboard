## Summary

Keep open. Current main still lacks an injected plugin runtime gateway accessor, while the requested host RPCs exist for gateway clients; this needs public SDK, redaction, and replay-contract review rather than cleanup closure.

## What This Changes

Request to add a curated or allowlisted PluginRuntime gateway accessor so plugins can read host status and model catalog RPCs.

## Best Possible Solution

Add a documented, backwards-compatible api.runtime.gateway read-only facade after maintainer review, preserving method allowlisting, status redaction, replay snapshot semantics, and public type exports tracked by #76758 and #76759.

## Reproduction Assessment

Not applicable; this is a feature request. Source inspection on current main confirms PluginRuntime has no gateway namespace while status and models.list exist for gateway clients.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade looks narrower than exposing a broad GatewayClient, but the final design needs explicit allowlist, redaction, and replay behavior.
