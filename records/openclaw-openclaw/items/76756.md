## Summary

Keep open: current main still lacks the requested ordinary plugin runtime accessor for curated host Gateway RPCs. The existing `plugin-sdk/gateway-method-runtime` helper is deliberately reserved for authenticated plugin HTTP routes, so this remains a focused plugin SDK and security-boundary design item.

## What This Changes

Request to add a curated, read-only PluginRuntime Gateway accessor for host RPCs such as `status` and `models.list`.

## Best Possible Solution

Keep this as the narrow host-RPC plugin SDK design item and add a curated read-only Gateway accessor only after maintainer security/product approval of allowlist, auth, public types, and replay semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows no ordinary `PluginRuntime.gateway` accessor while `status` and `models.list` exist as host Gateway methods, and the only in-process dispatcher rejects callers outside the authenticated plugin HTTP-route contract.

## Solution Assessment

Unclear until maintainers choose the public SDK shape. Curated read-only accessors look safer than exposing a raw Gateway request API, but the allowlist, caller identity, sensitive-field policy, exported types, and replay/frozen-snapshot behavior need product/security review.
