## Summary

Keep open. Current main still lacks a plugin runtime gateway accessor: the host RPCs exist and normal gateway clients can call them, but the injected plugin runtime exposes only curated runtime helpers plus plugin-owned gateway method registration.

## What This Changes

The issue requests a new plugin SDK runtime surface that lets plugins call a curated set of read-only host gateway RPCs such as status and models.list.

## Best Possible Solution

Add a narrow documented plugin runtime gateway namespace for allowlisted read-only status/model queries, with redaction, public types, and deterministic replay semantics aligned with #76758 and #76759.

## Reproduction Assessment

Not applicable; this is a feature request. Source inspection confirms the requested runtime surface is absent while the underlying gateway methods exist.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade looks safer than exposing a broad GatewayClient, but the final design should settle redaction and replay behavior first.
