## Summary

Keep open: current main and the latest release still lack the requested ordinary `PluginRuntime` host RPC accessor, and the existing dispatcher is intentionally reserved for authenticated plugin HTTP routes. This remains a focused plugin SDK and security-boundary design item, not stale cleanup or an implemented feature.

## What This Changes

The issue requests a curated plugin SDK surface that lets plugins call existing read-only host Gateway RPCs such as `status` and `models.list`.

## Best Possible Solution

Keep this as the canonical narrow host-RPC accessor design item and add curated read-only accessors, or an allowlisted request surface, only after maintainers accept the security, type, and replay contract.

## Reproduction Assessment

Yes. Source inspection shows current `PluginRuntime` has no `gateway` accessor, while `dispatchGatewayMethod` is restricted to authenticated plugin HTTP routes with the manifest contract.

## Solution Assessment

Unclear. Curated read-only accessors look safer than a broad raw request API, but maintainers still need to choose the allowlist, caller identity, authorization, public type surface, and replay contract.
