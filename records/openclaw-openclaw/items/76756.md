## Summary

Keep open. Current main still lacks a plugin runtime gateway accessor, and the requested read-only host RPC surface is a new public SDK capability with security and replay-contract choices rather than an implemented cleanup target.

## What This Changes

Request a PluginRuntime gateway namespace or allowlisted request API so plugins can read host status and model catalog RPCs.

## Best Possible Solution

Add a narrow documented api.runtime.gateway read-only facade for status/model catalog with exported types, allowlisted methods, redacted defaults, tests, docs, and replay snapshot semantics coordinated with #76758 and #76759.

## Reproduction Assessment

Not applicable; this is a feature/API request rather than a broken documented contract. Source inspection confirms the requested runtime gateway surface is absent while the target RPC handlers exist.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade looks safer than exposing GatewayClient directly, but the final design should settle allowlisting, redaction, and deterministic replay behavior first.
