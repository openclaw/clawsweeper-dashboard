## Summary

Keep open. Current main still lacks an injected plugin runtime gateway accessor, while the host `status` and `models.list` RPCs exist for gateway clients; because this is a new public SDK seam with redaction and replay implications, it needs maintainer API review rather than cleanup closure.

## What This Changes

Request adding a plugin runtime gateway surface so plugins can read host status and model catalog RPCs.

## Best Possible Solution

Add a narrow documented plugin runtime gateway namespace for read-only status/model queries, with method allowlisting, redaction rules, exported public types, and replay snapshot semantics aligned with the related follow-ups.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source inspection confirms the requested runtime surface is absent while the underlying gateway RPCs are present.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade looks safer than exposing a broad `GatewayClient`, but maintainers should settle allowlisting, redaction, type exports, and replay behavior first.
