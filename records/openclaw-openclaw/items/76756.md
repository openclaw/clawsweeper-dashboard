## Summary

Keep open: current main still lacks a plugin-runtime Gateway accessor, and the remaining work is a public SDK/Gateway contract decision rather than stale cleanup.

## What This Changes

The issue requests a public plugin-sdk Gateway accessor so plugins can call allowlisted host RPCs such as status and models.list.

## Best Possible Solution

Design and document a narrow api.runtime.gateway facade with allowlisted read-only accessors, redacted status behavior, public response types, and replay snapshot semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a request for a new public SDK capability. Source inspection on current main confirms the requested host RPCs exist but no PluginRuntime Gateway caller is exposed.

## Solution Assessment

Unclear until maintainer API review. Curated read-only accessors look safer than exposing an arbitrary Gateway client, but maintainers need to settle allowlisting, redaction, response typing, and replay behavior first.
