## Summary

Keep open. Current main still has no `PluginRuntime.gateway` or equivalent in-process plugin runtime accessor for `status` and `models.list`; the existing gateway dispatch helper is reserved for authenticated plugin HTTP routes and does not cover the reported runtime use case.

## What This Changes

Add a curated plugin SDK runtime gateway surface so plugins can read host `status` and `models.list` RPCs without broad gateway access.

## Best Possible Solution

Keep this as a focused SDK design item and land a narrow read-only runtime gateway API only after maintainers define auth/redaction, replay, and public type contracts.

## Reproduction Assessment

Yes, source inspection is enough: current `PluginRuntime` has no gateway accessor, while the only SDK gateway dispatch helper rejects callers outside entitled plugin HTTP request scopes.

## Solution Assessment

Unclear as an implementation path: curated read-only accessors are the safer direction, but maintainers still need to choose accessor versus generic allowlisted request and define redaction/replay semantics.
