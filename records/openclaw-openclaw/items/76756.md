## Summary

Keep open. Current main still lacks a plugin-runtime Gateway accessor while `status` and `models.list` exist as Gateway RPCs; this is a new public SDK seam with redaction, allowlist, type-export, and replay-contract decisions rather than cleanup-closeable work.

## What This Changes

The issue requests a plugin SDK surface that lets plugins call curated read-only host Gateway RPCs such as `status` and `models.list`.

## Best Possible Solution

Add a narrow documented `api.runtime.gateway` facade for read-only status/model catalog queries, backed by existing Gateway handlers with explicit scopes, redaction, replay snapshot semantics, and public type exports.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request for a missing public SDK contract. Source inspection on current main confirms the requested plugin runtime surface is absent while the underlying Gateway RPC handlers exist.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade appears safer than exposing `GatewayClient` or arbitrary Gateway requests, but the allowlist, redaction, exported types, and replay behavior need an explicit contract first.

Status: needs_attention

Summary: The requested SDK surface would expose host runtime state to in-process plugins, so the design needs an explicit allowlist and redaction policy.
