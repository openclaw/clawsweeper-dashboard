## Summary

Keep open. Current main still lacks a plugin runtime gateway accessor: the host RPCs exist and the TUI can call them, but plugin runtime injection exposes only curated helpers plus plugin-owned gateway method registration.

## What This Changes

Request to add a plugin SDK runtime surface for selected read-only host gateway RPCs such as status and models.list.

## Best Possible Solution

Add a narrow documented plugin runtime gateway namespace for read-only status/model queries, with allowlisting, redaction, exported public types, and replay semantics aligned with #76758 and #76759.

## Reproduction Assessment

Not applicable; this is a feature request. Source inspection confirms the requested runtime surface is absent while the underlying gateway methods exist.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade looks safer than exposing a broad GatewayClient, but the final shape should settle redaction and deterministic replay behavior first.
