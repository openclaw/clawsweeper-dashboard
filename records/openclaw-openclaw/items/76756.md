## Summary

Keep open. Current main still lacks an injected plugin runtime gateway accessor, while the host `status` and `models.list` RPCs exist; this is a new public SDK seam with redaction and replay implications that needs maintainer API review.

## What This Changes

Request a plugin-sdk runtime surface that lets plugins call curated read-only host gateway RPCs such as `status` and `models.list`.

## Best Possible Solution

Add a narrow documented `runtime.gateway` namespace for read-only status and model queries, with explicit allowlisting, redaction, public response types, and replay snapshot semantics.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken existing behavior. Source inspection confirms the requested runtime surface is absent while the underlying gateway RPCs are present.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade looks safer than exposing a broad `GatewayClient`, but maintainers should settle allowlisting, redaction, type exports, and replay behavior first.
