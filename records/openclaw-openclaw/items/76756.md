## Summary

Keep open. Current main still lacks an injected plugin-runtime Gateway accessor, while the underlying `status` and `models.list` Gateway RPCs exist; adding this would be a new public SDK seam with security, type-export, and replay-contract decisions still unresolved.

## What This Changes

Request a plugin SDK runtime gateway accessor for read-only host RPCs such as `status` and `models.list`.

## Best Possible Solution

Design and document a narrow `api.runtime.gateway` read-only facade with explicit allowlisting, redaction rules, public response types, and replay snapshot semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request for a new SDK capability. Source inspection confirms current main lacks the requested runtime surface while the underlying Gateway RPC handlers exist.

## Solution Assessment

Unclear until maintainer API review. Curated read-only accessors look safer than exposing a generic Gateway client, but maintainers need to settle allowlisting, redaction, public types, and replay behavior first.

## Security Review

Status: needs_attention

Summary: The requested SDK surface would expose host runtime state to in-process plugins and needs an explicit trust and redaction policy before implementation.
