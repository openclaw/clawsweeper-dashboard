## Summary

Keep open. Current main still has the reported SDK gap: host `status` and `models.list` RPCs exist and normal Gateway clients can call them, but the injected `PluginRuntime` has no curated gateway accessor; adding one is a new public SDK contract with security, replay, and type-export decisions.

## What This Changes

Request to add a plugin SDK runtime surface for read-only host Gateway RPCs such as `status` and `models.list`.

## Best Possible Solution

Add a narrow documented `api.runtime.gateway` namespace for read-only status/model queries, with allowlisting, redaction rules, exported public types, and replay snapshot semantics aligned with #76758 and #76759.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken existing behavior. Source inspection confirms the requested runtime surface is absent while the underlying host RPCs are present.

## Solution Assessment

Unclear until maintainer API review. A curated read-only facade looks safer than exposing a broad Gateway client, but maintainers should settle allowlisting, redaction, type exports, and replay behavior first.
