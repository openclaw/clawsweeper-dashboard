## Summary

Keep open: current main still creates native `sessions_spawn` child sessions with a UUID key, documents native resume as ACP-only, and no duplicate or merged PR implements a native warm-reuse contract.

## What This Changes

Request adding a warm-reuse path for native `sessions_spawn` delegations so repeated subagent calls can avoid cold-start cache cost.

## Best Possible Solution

Define an explicit native subagent reuse contract that preserves isolated UUID sessions as the default and covers key selection, lifecycle cleanup, concurrency, docs, schema/types, and focused regression tests.

## Reproduction Assessment

Yes for the source-level gap. A native `sessions_spawn` call reaches `spawnSubagentDirect`, which still creates a UUID-backed child session key and receives no native reuse, pool, or resume selector from the tool path.

## Solution Assessment

Unclear. The requested direction is valid, but choosing opt-in reuse, automatic pooling, or main-session routing requires maintainer product/API judgment before implementation.
