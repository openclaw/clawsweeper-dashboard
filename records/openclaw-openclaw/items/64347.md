## Summary

Keep open: current main still has the reported native warm-reuse gap. Source, docs, and tests show native `sessions_spawn` creates isolated UUID-backed child sessions while resume support remains ACP-only.

## What This Changes

Requests a native `sessions_spawn` warm-reuse or pooling path so repeated delegations to the same subagent can avoid a fresh cold-start session each time.

## Best Possible Solution

Define an explicit native subagent reuse contract that keeps isolated UUID sessions as the default, then wire the chosen API through tool schema, config/types if needed, docs, session key selection, cleanup, and focused tests.

## Reproduction Assessment

Yes. A native `sessions_spawn` call reaches `spawnSubagentDirect`, which currently builds a UUID-backed child session key and receives no native reuse or resume parameter from the tool path.

## Solution Assessment

Unclear. The issue’s opt-in key, automatic pool, and main-session routing ideas are plausible, but maintainers need to choose the isolation, history, locking, archive cleanup, and thread semantics before implementation.
