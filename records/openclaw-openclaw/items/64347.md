## Summary

Keep open: current main still implements native subagent spawns as fresh UUID-backed isolated sessions, while resume/reuse behavior remains ACP-only and no native warm-reuse contract exists.

## What This Changes

Request native `sessions_spawn` warm reuse, pooling, or a reuse key so repeated delegations to the same agent can avoid cold-start context costs.

## Best Possible Solution

Define an explicit native subagent reuse contract that keeps isolated UUID sessions as the default, then wire the chosen API through tool schema, config/types if needed, docs, key selection, cleanup, locking, and focused tests.

## Reproduction Assessment

Yes for the source-level gap. A native `sessions_spawn` call reaches `spawnSubagentDirect`, which still creates a UUID-backed child session key and receives no native reuse, pool, or resume selector from the tool path.

## Solution Assessment

Unclear. The issue's opt-in key, automatic pool, and main-session routing options are plausible, but maintainers need to choose isolation, history sharing, locking, archive cleanup, and thread semantics before implementation.
