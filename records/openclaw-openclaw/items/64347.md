## Summary

Keep open. Current main still creates native subagent sessions with fresh UUID-backed keys, treats resume as ACP-only, and exposes no native warm reuse, pool, or stable child-session selector in code, config, docs, or tests. Related merged PRs #64468 and #72331 fixed adjacent bootstrap-marker and ACP-field behavior, not this native reuse contract.

## What This Changes

Add native warm-reuse, pooling, or stable session-key support for `sessions_spawn` so repeated delegations can avoid a fresh cold subagent session each time.

## Best Possible Solution

Define an explicit native subagent reuse contract, then implement it through the tool schema, config/types, child-session key selection, docs, and focused tests. The desired end state should keep isolated fresh sessions as the default while allowing deliberate warm reuse with clear history-sharing semantics.

## Reproduction Assessment

Yes. The source-level reproduction is the current native `runtime="subagent"` path: the tool ignores `resumeSessionId` for subagent runtime, passes no reuse argument, and `spawnSubagentDirect` constructs a new UUID-backed child session key on each call.

## Solution Assessment

Unclear. The issue's proposed options are plausible, but the best fix needs maintainer design around isolation, history sharing, concurrency locking, cleanup, and thread-bound session behavior before choosing opt-in reuse, automatic pooling, or main-session routing.
