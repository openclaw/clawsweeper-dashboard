## What This Changes

The issue reports that OpenClaw 2026.4.29 gateway idles at roughly one saturated CPU core because multiple background timers repeatedly process session/trajectory data and filesystem state.

## Best Possible Solution

Keep this open until a maintainer captures a current-main or 2026.4.29 timer profile that names the hot callback, then fix that owner-owned loop narrowly: reuse cached snapshots, make idle work event-driven, add in-flight guards/backoff, or bound filesystem/session reads in the responsible gateway, memory-core, or plugin status path.
