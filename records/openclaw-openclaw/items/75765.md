## Summary

Keep open. The PR is draft, carries the protected `maintainer` label, and the proposed structured heartbeat path is not merge-ready because the new tool schema uses provider-fragile union enums and the public Plugin SDK export change is missing generated API-baseline follow-through.

## What This Changes

The PR adds a `heartbeat_respond` tool, plumbs its structured outcome through Codex and embedded runner telemetry/payloads, updates heartbeat delivery handling, and documents/changelogs the new heartbeat response contract.

## Best Possible Solution

Keep the structured heartbeat design on the PR, but make the tool schema provider-portable, regenerate the Plugin SDK API baseline if the new exports are intentional, capture the promised rollout evidence, and then run focused heartbeat/Codex tests plus the changed gate before maintainer review.

## Reproduction Assessment

Yes for the review findings: the schema issue is reproducible by inspecting the new `Type.Union` enum construction against the existing `stringEnum` contract, and the SDK issue is reproducible by comparing the public export diff with the missing generated baseline hash. There is no separate user bug reproduction because this is a feature PR.

## Solution Assessment

No, not yet. The overall approach is plausible, but the mergeable solution should use the established flat enum helper for tool parameters and keep the public SDK baseline in sync before this becomes a stable heartbeat contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Use flat enums in the heartbeat tool schema:** `src/agents/tools/heartbeat-response-tool.ts:13-20`
  - body: `outcome` and `priority` are built with `Type.Union([...Type.Literal])`, which serializes to nested `anyOf`. This tool is exposed through the normal model tool path for heartbeat runs, and the existing schema helper explicitly avoids that shape because some providers reject it. Use `stringEnum`/`optionalStringEnum` so the new tool remains portable across providers.
  - confidence: 0.87
- **[P2] Regenerate the Plugin SDK API baseline:** `src/plugin-sdk/agent-harness-runtime.ts:75-78`
  - body: This diff adds public exports from `openclaw/plugin-sdk/agent-harness-runtime`, but it does not include an updated `docs/.generated/plugin-sdk-api-baseline.sha256`. The SDK API check tracks this exported surface, so intentional new exports should be paired with `pnpm plugin-sdk:api:gen` output.
  - confidence: 0.75
