## Summary

Keep open. The async-reader direction is reasonable, but the PR currently drops the transcript header validation that `SessionManager.open()` provided before feeding entries into CLI hook history and export-session data.

## What This Changes

The PR removes synchronous gateway session-reader exports and converts CLI history/reseed, export-session, and one gateway live test to async transcript parsing APIs.

## Best Possible Solution

Keep the async surface removal, but add a small validated transcript-reader helper that preserves the old header contract before returning migrated entries, with focused malformed-transcript tests.

## Reproduction Assessment

Yes. A JSONL transcript whose first parsed record is not a `type: "session"` header is rejected by the current `SessionManager.open()` path but accepted by the PR’s direct `parseSessionEntries()` replacements.

## Solution Assessment

No. Replacing sync reads is the right direction, but direct parsing is not the narrowest safe replacement until it validates the transcript header contract that callers currently rely on.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve transcript validation before loading hook history:** `src/agents/cli-runner/session-history.ts:165-167`
  - body: `SessionManager.open()` currently goes through `loadEntriesFromFile()`, which returns no entries unless the first parsed record is a valid session header. This replacement calls `parseSessionEntries()` directly and then filters messages, so a corrupt or non-session JSONL file under the resolved sessions directory can now be included in `llm_input`/`agent_end` history instead of being dropped. Validate the header before returning entries.
  - confidence: 0.88
- **[P2] Validate exported transcripts before building session data:** `src/auto-reply/reply/commands-export-session.ts:130-131`
  - body: The export path also relied on `SessionManager.open()` to validate the file before deriving `header`, `entries`, and `leafId`. The new helper accepts parsed entries even when the file is not a valid session transcript, which can export malformed or non-session JSONL as session data. Check the first parsed entry/header before building the export payload.
  - confidence: 0.84
