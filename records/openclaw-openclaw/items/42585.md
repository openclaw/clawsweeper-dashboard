## What This Changes

The PR changes auth-profile session override resolution to canonicalize legacy OAuth `provider:default` auto overrides to non-default OAuth aliases and adds regression coverage.

## Best Possible Solution

Re-port only identity-safe canonicalization onto the current resolver, preserving newer auth/session semantics and remapping no-identity defaults only when the alias is unique or otherwise provably equivalent.

## Reproduction Assessment

Yes, at source level. On current main, a non-cooldown auto `provider:default` override on the ordinary non-new-session path remains `current` and is persisted/returned without canonicalizing to an available OAuth alias.

## Solution Assessment

No, not as currently proposed. The canonicalization direction is useful, but it should be re-ported onto current main and tightened so no-identity defaults do not remap to arbitrary OAuth aliases.
