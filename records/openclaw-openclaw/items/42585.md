## What This Changes

The PR adds resolver logic and regression tests to canonicalize legacy auto OAuth `provider:default` session overrides to non-default OAuth aliases.

## Best Possible Solution

Re-port only identity-safe canonicalization onto current main, keep the newer resolver semantics intact, and remap no-identity defaults only when the alias is unique or otherwise provably equivalent.

## Reproduction Assessment

Yes, at source level. On current main, a non-cooldown auto `provider:default` override can remain unchanged on the ordinary non-new-session path because the resolver does not canonicalize legacy OAuth defaults before returning or persisting the next profile.

## Solution Assessment

No, not as submitted. The canonicalization direction is useful, but the branch is stale and needs a current-main re-port plus stricter alias-safety rules and real runtime/session proof.
