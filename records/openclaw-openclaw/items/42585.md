## What This Changes

The PR changes auth-profile session override resolution to canonicalize legacy OAuth `provider:default` auto overrides to non-default OAuth aliases and adds regression tests.

## Best Possible Solution

Re-port identity-safe canonicalization onto the current resolver, preserve current auth/session semantics, and only remap no-identity defaults when the alias is unique or otherwise provably equivalent.

## Reproduction Assessment

Yes, at source level. Configure a store with `openai-codex:default` plus a matching alias, set an auto session override to the legacy default, and current main's normal existing-session path returns the current override without canonicalizing.

## Solution Assessment

No, not as currently proposed. The canonicalization idea is useful, but the patch should be re-ported onto current main and tightened so no-identity defaults do not remap to arbitrary OAuth aliases.
