Keep open. Current main still has the legacy-default drift, but this stale PR should not merge as-is because it would regress current auth alias and explicit user-override safeguards in credential routing.

## What This Changes

The PR adds resolver-side canonicalization of automatic OAuth `provider:default` session overrides to non-default OAuth aliases plus regression tests for identity matching, non-OAuth defaults, fallback behavior, and rotation progress.

## Best Possible Solution

Port the canonicalization into current `src/agents/auth-profiles/session-override.ts` while preserving the no-source fast path, `resolveProviderIdForAuth`, explicit user override handling, cooldown behavior, and rotation progress; then add current-harness tests and a changelog entry.
