## Summary

Keep open. Current main still makes timeout-shaped embedded run aborts fallback-eligible, and an open linked implementation PR still has closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to stop model fallback when an embedded run aborts because its own timeout budget expired.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/62682, or an equivalent narrow fix that carries terminal run-budget abort context to the fallback boundary while preserving retryable provider-timeout fallback.

## Reproduction Assessment

Yes. A high-confidence source reproduction is an embedded run with a configured timeout and fallback candidates: scheduleAbortTimer calls abortRun(true), and current fallback classification keeps timeout-shaped aborts eligible for later candidates.

## Solution Assessment

Yes. The narrow maintainable solution is to distinguish terminal run-budget aborts from retryable provider/model timeout failures at the fallback boundary; the linked open PR is the active implementation path.
