## Summary

Keep open. Current main still lacks a terminal run-timeout signal at the model-fallback boundary, and the open closing PR at https://github.com/openclaw/openclaw/pull/62682 is the paired implementation candidate, so this issue should close only after that PR or a replacement lands.

## What This Changes

The issue asks OpenClaw to skip model fallback when an embedded run is aborted by its own timeout budget while preserving fallback for provider/model failures.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/62682, coordinating with https://github.com/openclaw/openclaw/pull/52365 if maintainers also want a proactive cron-deadline guard.

## Reproduction Assessment

Yes. A high-confidence source reproduction is an embedded run with a short `timeoutMs` and configured fallbacks: the embedded timer aborts with a `TimeoutError`, current fallback logic treats timeout-shaped aborts as retryable, and failover policy can rotate or fall back on `timedOut`.

## Solution Assessment

Yes. The narrow maintainable fix is to carry terminal run-abort context to the fallback boundary and embedded failover policy while preserving fallback for provider/model timeouts; the open PR at https://github.com/openclaw/openclaw/pull/62682 implements that direction.
