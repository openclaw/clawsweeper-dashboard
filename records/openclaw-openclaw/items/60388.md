## Summary

Keep open. Current main still treats timeout-shaped embedded run aborts as fallback-eligible, and the open linked implementation PR is still unmerged with closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to stop model fallback when an embedded run exhausts its own timeout budget.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/62682, or an equivalent narrow fix that carries terminal run-budget abort context to the fallback boundary while preserving provider-timeout fallback.

## Reproduction Assessment

Yes. A high-confidence source reproduction is an embedded run with a configured timeout and fallback candidates: scheduleAbortTimer calls abortRun(true), the private controller aborts with a TimeoutError, and current fallback logic continues timeout-shaped failures through remaining candidates.

## Solution Assessment

Yes. The narrow maintainable fix is to distinguish run-budget terminal timeout from provider/model timeout at the fallback decision path; the linked open PR is the current implementation candidate.
