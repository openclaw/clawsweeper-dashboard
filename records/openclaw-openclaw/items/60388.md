## Summary

Keep open. Current main still treats timeout-shaped run aborts as eligible for model fallback, and open PR #62682 is already paired with this issue using closing syntax, so this should remain open until that PR is reviewed and merged or closed.

## What This Changes

Stop model fallback when an embedded or cron run aborts because its own timeout budget expired, while preserving fallback for provider/model failures.

## Best Possible Solution

Land a narrow fix like PR #62682 that preserves provider-timeout fallback but stops the chain when the run controller's own terminal timeout has fired, with focused unit coverage for both paths.

## Reproduction Assessment

Yes. The source-level path is high-confidence: the embedded timeout timer aborts with `TimeoutError`, current fallback logic treats timeout-shaped aborts as retryable, and existing tests still prove timeout errors advance to fallbacks.

## Solution Assessment

Yes. The maintainable fix is to carry terminal abort context to the fallback boundary and classify run-budget timeouts separately from provider/model timeout failures.
