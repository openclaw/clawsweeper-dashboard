## Summary

Keep open. Current main still lets run-budget timeout-shaped aborts flow into model fallback, and https://github.com/openclaw/openclaw/pull/62682 is an open closing PR for the same bug, so this issue should stay open until that PR is landed, replaced, or closed.

## What This Changes

The issue asks OpenClaw to stop trying fallback models when an embedded or cron run is aborted by its own timeout budget.

## Best Possible Solution

Land or replace the linked terminal-abort fix so run-budget exhaustion is classified as terminal while ordinary provider/model timeouts remain fallback-eligible.

## Reproduction Assessment

Yes. A high-confidence source reproduction is an embedded run with a configured timeout and fallback candidates: `scheduleAbortTimer` aborts with a `TimeoutError`, current fallback logic treats timeout-shaped aborts as retryable, and existing coverage still expects timeout errors to fall back.

## Solution Assessment

Yes. The narrow maintainable direction is to carry terminal abort or run-budget context to the fallback boundary and keep provider/model timeout failures retryable; https://github.com/openclaw/openclaw/pull/62682 is the current implementation path.
