## Summary

Keep open. Current main still treats timeout-shaped run aborts as eligible for the model fallback path, and open PR #62682 is explicitly paired with this issue using closing syntax, so the issue should remain open until that implementation is reviewed and merged or closed.

## What This Changes

The issue asks OpenClaw to skip model fallback when an embedded agent run exceeds its own configured timeout budget.

## Best Possible Solution

Land a narrow fix like PR #62682 that carries terminal abort context into the fallback boundary and adds focused coverage proving run-budget timeouts stop immediately while provider timeout failures can still use configured fallbacks.

## Reproduction Assessment

Yes. The source path is high-confidence: the embedded timeout timer calls `abortRun(true)`, that creates an AbortError with a `TimeoutError` cause, and current fallback logic treats timeout-shaped aborts as retryable.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix: classify the run controller's terminal timeout separately at the fallback boundary rather than disabling all timeout-based fallback behavior.
