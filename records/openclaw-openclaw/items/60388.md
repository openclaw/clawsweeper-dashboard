- `impact:auth-provider`: The issue concerns model fallback, auth-profile rotation, and provider/model routing decisions after timeout-shaped failures.

## Summary

Keep open. Current main and the latest shipped tag still let run-budget timeout-shaped aborts flow into model fallback, and https://github.com/openclaw/openclaw/pull/62682 is still the open implementation path with closing syntax for this issue.

## What This Changes

Stop model fallback from trying additional candidates when an embedded or cron run exhausts its own timeout budget, while preserving fallback for provider/model failures.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/62682, or an equivalent narrow fix that carries terminal run-budget abort context to the fallback boundary while preserving provider-timeout fallback.

## Reproduction Assessment

Yes. A high-confidence source reproduction is an embedded or cron run with a configured timeout and fallback candidates: scheduleAbortTimer calls abortRun(true), the private controller aborts with TimeoutError, and current fallback logic keeps timeout-shaped failures eligible for remaining candidates.

## Solution Assessment

Yes. The maintainable fix is to distinguish terminal run-budget aborts from retryable provider/model timeouts at the fallback decision boundary; the open linked PR is the active implementation path but this issue should not close before a fix merges.
