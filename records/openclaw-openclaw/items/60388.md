- `P2`: The issue is a bounded fallback-routing bug that wastes model calls and lane time without making the core runtime unusable.
- `impact:auth-provider`: The affected behavior is provider, profile, and model fallback routing after timeout classification.

## Summary

Keep open. Current main and v2026.5.22 still lack terminal run-budget abort context at the model-fallback boundary, and https://github.com/openclaw/openclaw/pull/62682 remains the open implementation path.

## What This Changes

Stop model fallback from trying additional candidates after an embedded or cron run exhausts its own timeout budget.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/62682, or an equivalent narrow fix that carries terminal run-budget abort context to the fallback boundary while preserving provider-timeout fallback.

## Reproduction Assessment

Yes. Source inspection provides a high-confidence path: the run-budget timer calls `abortRun(true)`, current failover policy can escalate timeout failures to `fallback_model`, and `runWithModelFallback` has no terminal abort input to stop the candidate loop.

## Solution Assessment

Yes. The best fix is the narrow fallback-boundary distinction requested here; the linked open PR is the active path, so this issue should stay open until that or an equivalent change merges.
