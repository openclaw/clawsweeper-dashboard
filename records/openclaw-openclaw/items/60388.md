## Summary

Keep open. Current main still treats timeout-shaped run aborts as fallback-eligible, and the open implementation candidate at https://github.com/openclaw/openclaw/pull/62682 is linked with closing syntax, so this issue should remain open until that PR is merged or closed.

## What This Changes

The issue asks OpenClaw to stop cascading through model fallbacks when an embedded run is aborted by its own configured timeout budget.

## Best Possible Solution

Review and land a narrow fix like https://github.com/openclaw/openclaw/pull/62682 that preserves provider-timeout fallback while stopping the chain when the runner's own terminal timeout has fired.

## Reproduction Assessment

Yes. A high-confidence source reproduction is an embedded run with a configured timeout and fallback candidates: the run timer aborts with `TimeoutError`, current fallback logic treats timeout-shaped aborts as retryable, and existing tests still expect timeout errors to advance to a fallback model.

## Solution Assessment

Yes. The maintainable solution is to carry terminal abort context to the fallback boundary and classify run-budget exhaustion separately from provider/model timeout failures; the paired open PR is the current implementation path.
