## Label Justifications

- `P2`: The issue is a bounded but real fallback-routing bug that wastes model calls and lane time without making the core runtime unusable.
- `impact:auth-provider`: The affected behavior is provider/profile/model fallback routing after timeout classification.

## Summary

Keep open. Current main still treats run-budget timeout-shaped embedded aborts as fallback-eligible at the model fallback boundary, and the active implementation path is the open closing PR https://github.com/openclaw/openclaw/pull/62682.

## What This Changes

The issue requests that embedded run timeout-budget aborts fail immediately instead of cascading through auth-profile and model fallback candidates.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/62682, or an equivalent narrow fix that carries terminal run-budget abort context to runWithModelFallback while preserving fallback for provider/model timeouts.

## Reproduction Assessment

Yes. A high-confidence source reproduction is an embedded or cron run with a configured timeout and fallback candidates: the run-budget timer aborts with a TimeoutError, while current runWithModelFallback has no terminal abort context and continues timeout-shaped failures through later candidates.

## Solution Assessment

Yes. The narrow maintainable solution is to distinguish terminal run-budget aborts from retryable provider/model timeouts at the fallback boundary; the linked open PR is the active path, so this issue should remain open until that fix or an equivalent merges.
