## Summary

Keep open. Current main still lacks terminal run-budget abort context at the model fallback boundary, and the open closing PR is the active implementation path rather than a reason to close this issue before merge.

## What This Changes

The issue requests that embedded or cron run-budget timeouts fail immediately instead of triggering model/profile fallback attempts with nearly no time remaining.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/62682, or an equivalent narrow fix that carries terminal run-budget abort context to the fallback boundary while preserving provider-timeout fallback.

## Reproduction Assessment

Yes. A high-confidence source reproduction is an embedded or cron run with a configured timeout and fallback candidates: the run-budget timer aborts with a TimeoutError, while current runWithModelFallback has no terminal abort context and continues timeout-shaped failures through later candidates.

## Solution Assessment

Yes. The narrow maintainable fix is to distinguish terminal run-budget aborts from retryable provider/model timeouts at the fallback boundary; the linked open PR is the active path, so this issue should remain open until it or an equivalent change merges.
