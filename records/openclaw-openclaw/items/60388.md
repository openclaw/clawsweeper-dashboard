## Summary

Keep open. Current main still lets timeout-shaped embedded run aborts remain fallback-eligible, and https://github.com/openclaw/openclaw/pull/62682 is still open with closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to stop walking model fallback candidates when an embedded run is aborted by its own configured timeout budget.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/62682, or an equivalent narrow fix that carries terminal run-budget abort context to the fallback boundary while preserving provider-timeout fallback.

## Reproduction Assessment

Yes. A high-confidence source reproduction is an embedded run with a configured timeout and fallback candidates: scheduleAbortTimer calls abortRun(true), the private controller aborts with a TimeoutError, and current fallback logic continues timeout-shaped failures through remaining candidates.

## Solution Assessment

Yes. The narrow maintainable solution is to distinguish terminal run-budget aborts from retryable provider/model timeouts at the fallback boundary; the linked open PR is the active implementation path but still needs PR resolution before this issue can close.
