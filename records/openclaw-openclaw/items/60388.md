## Summary

Keep open. Current main still treats timeout-shaped embedded run aborts as fallback-eligible, and the active fix path is the still-open https://github.com/openclaw/openclaw/pull/62682 with closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to stop model fallback when an embedded run exhausts its own timeout budget.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/62682, or an equivalent narrow fix that carries terminal run-budget abort context to the model-fallback boundary while preserving retryable provider timeout fallback.

## Reproduction Assessment

Yes. A high-confidence source reproduction is an embedded run with a configured timeout and fallback candidates: the timeout timer calls abortRun(true), the private controller aborts with a TimeoutError, and current fallback logic keeps timeout-shaped failures eligible for later candidates.

## Solution Assessment

Yes. The maintainable fix is to distinguish terminal run-budget aborts from retryable provider/model timeouts at the fallback boundary; the linked open PR is the active implementation path but must land before this issue can close.
