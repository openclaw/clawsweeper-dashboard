## Summary

The intended bug is valid on current main, but this draft PR is not merge-ready because the diff appears to regress the lazy catalog path, lacks required real behavior proof, and has no changelog entry for the user-facing fix.

## What This Changes

The PR changes auto-reply thinking default resolution so global `thinkingDefault` falls through to the shared model resolver, then adds a regression test for per-model `params.thinking` precedence.

## Best Possible Solution

Update auto-reply thinking resolution so per-model `params.thinking` can override the global default while preserving per-agent overrides and the no-catalog-load path when the global default wins; include focused regression coverage, changelog, and real behavior proof.

## Reproduction Assessment

Yes from source inspection, though I did not run tests in this read-only review. Current main returns the global `agentCfg?.thinkingDefault` before reaching the shared resolver whose implementation and tests define per-model `params.thinking` as higher precedence than the global default.

## Solution Assessment

No. Dropping the global-default short-circuit fixes the precedence bug in one case, but it is not the narrowest maintainable solution because it calls `resolveThinkingCatalog()` before the shared resolver can return the global default; the fix should preserve lazy catalog behavior when no per-model override exists.
