## Summary

Keep open. Current main still stores sandbox backend registrations in a module-local map while plugin code registers through the SDK, and the same reporter already has an open PR for this exact registry-scope bug.

## What This Changes

The issue asks for plugin-registered sandbox backends to remain visible when `requireSandboxBackendFactory` resolves the configured backend during a sandboxed turn.

## Best Possible Solution

Land the existing PR or an equivalent narrow fix that shares sandbox backend registrations across plugin loader module instances and adds duplicate-module regression coverage.
