## Summary

Current main and the shipped v2026.5.28 release now use a process-global sandbox backend registry, so plugin registration and runtime lookup no longer depend on the same module instance.

## What This Changes

The issue reports that a custom sandbox backend plugin registers successfully during gateway startup but `requireSandboxBackendFactory` cannot find it during an agent turn.

## Best Possible Solution

Keep the shipped global sandbox backend registry and treat older same-fix PRs as superseded by current main and v2026.5.28.

## Reproduction Assessment

No current-main failing reproduction remains. Source inspection shows the old module-local registry has been replaced by a `globalThis` singleton, and the same implementation is present in v2026.5.28.

## Solution Assessment

Yes. A process-global `Symbol.for` registry is the narrow owner-boundary fix for split module graphs because plugin SDK registration and runtime lookup share the same process state without adding config or plugin-specific policy.
