## Summary

Close: current main and shipped releases already use a process-global sandbox backend registry, so plugin-registered sandbox backends are visible to runtime lookup instead of failing at requireSandboxBackendFactory.

## What This Changes

The issue reports custom sandbox backend plugins registering during gateway startup but failing at message time because requireSandboxBackendFactory cannot see the backend.

## Best Possible Solution

Keep the shipped process-global sandbox backend registry; any remaining custom-backend failure should be filed as a new current-version report with plugin and gateway proof.

## Reproduction Assessment

Yes, source-level for the original failure. Historical source used a module-local backend registry while plugin registration and runtime lookup could run through split module graphs; current main was not live-reproduced because the registry is now process-global.

## Solution Assessment

Yes. The shipped fix changes the core registry owner to a Symbol.for global singleton, which is the narrow maintainable solution and avoids plugin-specific workarounds.
