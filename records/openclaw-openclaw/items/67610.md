## Summary

Current main and a stable release already share sandbox backend registrations through a process-global registry, so the reported plugin-registration lookup failure is fixed.

## What This Changes

The issue reports that a plugin-registered sandbox backend is visible during gateway startup but unavailable to `requireSandboxBackendFactory` during agent turns.

## Best Possible Solution

Keep the shipped process-global sandbox backend registry; treat the still-open same-shape contributor PR as separate superseded-PR cleanup.

## Reproduction Assessment

Yes, source-level against older builds: the old registry was a module-local Map while plugin loading could run through a separate module graph. Current main no longer reproduces that path because the registry is backed by `globalThis` and `Symbol.for`.

## Solution Assessment

Yes. The merged fix is the narrow generic core repair: share the existing registry across module instances instead of adding plugin-specific config, fallback, or loader policy.
