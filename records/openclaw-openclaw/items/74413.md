## Summary

Keep open. Current main still lacks the requested WhatsApp first-use/hello-world guidance in both the WhatsApp setup flow and the cited docs, and #75082 is an open, unmerged closing PR rather than a shipped fix.

## What This Changes

Request a post-WhatsApp-setup cue and docs example that show how to use the WhatsApp channel after it is connected.

## Best Possible Solution

Land #75082 or an equivalent narrow patch that adds manual WhatsApp first-message guidance to the channel docs and setup completion flow, plus a CLI send example, while avoiding automatic outbound sends unless maintainers explicitly approve that behavior.

Yes. Source and docs inspection against current main reproduces the gap: WhatsApp setup finishes after DM access, the WhatsApp channel page ends at pairing approval, and the message CLI examples lack a WhatsApp hello-world send path.

## Solution Assessment

Yes. A manual try-it path in docs and onboarding is the narrowest maintainable direction; automatic test sends would broaden behavior and raise privacy and destination-selection concerns.
