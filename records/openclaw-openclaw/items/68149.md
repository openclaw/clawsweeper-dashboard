## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Pass the env when launching the fallback task:** `src/daemon/schtasks.ts:916`
  - body: This PowerShell `/Run` fallback calls `launchFallbackTaskScript` with `params.scriptPath`, but that helper expects a `GatewayServiceEnv` and resolves the script path from it. Passing a file path string can throw `Missing HOME` instead of launching the fallback when `Start-ScheduledTask` succeeds but the task never starts.
  - confidence: 0.91
- **[P1] Await and pass the env for Startup fallback launch:** `src/daemon/schtasks.ts:1047`
  - body: The create fallback also passes `scriptPath` to `launchFallbackTaskScript` and does not await it, so the access-denied Startup-folder path can report success while the async launch rejects or resolves the wrong task script path. Keep the existing env-based launch contract here.
  - confidence: 0.9
- **[P1] Quote the task script for cmd.exe:** `src/daemon/schtasks.ts:977`
  - body: The PowerShell task action builds `cmd.exe` arguments as `/d /s /c '<scriptPath>'`. `cmd.exe` does not treat single quotes as path quoting, so script paths with spaces or command metacharacters can fail or be parsed as extra commands; reuse the existing cmd quoting semantics for both create and update paths.
  - confidence: 0.88
- **[P2] Remove the idle flag from the ONLOGON fallback:** `src/daemon/schtasks.ts:1027-1028`
  - body: The `schtasks` fallback creates `/SC ONLOGON` while also passing `/I 0`. Microsoft documents `/I` as valid only with `ONIDLE` and requiring a 1-999 value, so fallback installs can fail instead of registering the gateway task.
  - confidence: 0.96
- **[P1] Continue uninstall cleanup after PowerShell removal:** `src/daemon/schtasks.ts:1105`
  - body: Returning immediately after `Unregister-ScheduledTask` skips deletion of the Startup-folder launcher and generated task script below. Machines that previously used the fallback can keep auto-starting after uninstall, so remove the task and still run artifact cleanup.
  - confidence: 0.97
