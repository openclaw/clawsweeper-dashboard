## Summary

Keep this PR open. Current main still uses the existing schtasks plus Startup-folder fallback contract, so the PR is not obsolete, but the PR head still has blocking regressions in the PowerShell task action, schtasks fallback arguments, and uninstall cleanup path.

## What This Changes

This PR changes `src/daemon/schtasks.ts` to prefer PowerShell ScheduledTasks cmdlets for Windows gateway auto-start, set no execution time limit and battery settings, require administrator privileges, and keep schtasks as a fallback.

## Best Possible Solution

Keep the PowerShell ScheduledTasks direction only if it preserves a valid cmd.exe task action, keeps the schtasks fallback command valid, always removes Scheduled Task plus Startup-folder/script artifacts on uninstall, and updates Windows docs/tests/changelog for the chosen administrator and fallback policy.

## Reproduction Assessment

Yes. The blocking findings can be reproduced with focused tests or mocks that assert the generated PowerShell `New-ScheduledTaskAction` argument, simulate PowerShell uninstall success with existing fallback artifacts, and exercise the schtasks fallback create args.

## Solution Assessment

No. PowerShell ScheduledTasks may be the right direction for the 72-hour and battery settings problem, but this patch is not merge-ready until the command quoting, fallback flags, uninstall cleanup, docs, and tests are corrected.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Quote the task script for cmd.exe, not PowerShell:** `src/daemon/schtasks.ts:977`
  - body: The PowerShell action stores `/d /s /c '${safeScriptPath}'`; PowerShell converts the doubled quotes into literal single quotes, and `cmd.exe` does not use single quotes for command quoting. The primary PowerShell install/update path can register a task that cannot launch the generated script, so build this argument with the existing cmd quoting semantics instead.
  - confidence: 0.88
- **[P2] Remove the idle flag from the ONLOGON fallback:** `src/daemon/schtasks.ts:1027-1028`
  - body: The schtasks fallback appends `/I 0` while also using `/SC ONLOGON`. `/I` is only valid for `ONIDLE` schedules and valid values are 1-999, so machines that need the fallback can reject task creation instead of installing the gateway task.
  - confidence: 0.96
- **[P1] Run uninstall cleanup after PowerShell task removal:** `src/daemon/schtasks.ts:1105`
  - body: Returning immediately after `Unregister-ScheduledTask` skips the cleanup below that deletes the Startup-folder launcher and generated gateway script. If a previous fallback launcher exists, uninstall can leave auto-start behavior behind.
  - confidence: 0.97
