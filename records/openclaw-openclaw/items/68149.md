## Summary

Keep open. The PR is a plausible Windows ScheduledTasks direction, but the current head still has blocking command/fallback/cleanup regressions, lacks after-fix real Windows proof, and changes the documented no-admin Startup-folder fallback policy without an explicit maintainer decision.

## What This Changes

The PR changes `src/daemon/schtasks.ts` to prefer PowerShell ScheduledTasks cmdlets for Windows auto-start, set task lifetime and battery settings, require administrator privileges, and keep `schtasks` as a fallback.

## Best Possible Solution

Land a narrower Windows ScheduledTasks update only after preserving valid cmd quoting, valid fallback args, uninstall artifact cleanup, and an explicitly approved fallback policy with live Windows proof.

## Reproduction Assessment

Yes for the review blockers: source inspection of PR head gives concrete reproduction targets around generated PowerShell action arguments, `schtasks` fallback create args, and uninstall cleanup after simulated PowerShell success. A live Windows run is still missing for the claimed end-to-end behavior.

## Solution Assessment

No. PowerShell ScheduledTasks may be the right mechanism for execution limit and battery settings, but this branch is not the best mergeable solution until it preserves existing cleanup/fallback contracts and has an approved policy for admin-only installation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Quote the task script for cmd.exe:** `src/daemon/schtasks.ts:977`
  - body: The registered action passes `/d /s /c '<scriptPath>'` through PowerShell. `cmd.exe` does not use single quotes for path quoting, so paths with spaces or cmd metacharacters can fail or be parsed as extra commands; build this argument with the existing cmd quoting semantics in both create and update paths. ([learn.microsoft.com](https://learn.microsoft.com/en-us/windows-server/administration/windows-commands/cmd))
  - confidence: 0.88
- **[P2] Remove the idle flag from the ONLOGON fallback:** `src/daemon/schtasks.ts:1027-1028`
  - body: The fallback creates `/SC ONLOGON` while also passing `/I 0`. Microsoft documents `/I` as valid only with `ONIDLE` and requiring a 1-999 value, so fallback installs can fail instead of registering the gateway task. ([learn.microsoft.com](https://learn.microsoft.com/en-us/windows-server/administration/windows-commands/schtasks-create?utm_source=openai))
  - confidence: 0.96
- **[P1] Continue uninstall cleanup after PowerShell removal:** `src/daemon/schtasks.ts:1105`
  - body: Returning immediately after `Unregister-ScheduledTask` skips deletion of the Startup-folder launcher and generated task script below. Machines that previously used the fallback can keep auto-starting after uninstall, so remove the task and still run artifact cleanup.
  - confidence: 0.97
