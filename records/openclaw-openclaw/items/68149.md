Overall confidence: 0.93

Full review comments:

- **[P1] Keep the documented Startup-folder fallback:** `src/daemon/schtasks.ts:954-959`
  - body: This preflight throws on every non-admin Windows install before `schtasks` can fail into the documented Startup-folder fallback. Current docs and tests require access-denied task creation to install a per-user login item and start the gateway immediately, so this is a user-visible compatibility regression unless maintainers explicitly change that policy.
  - confidence: 0.9
- **[P1] Pass the env when launching fallback tasks:** `src/daemon/schtasks.ts:917-935`
  - body: Both new run fallback branches call `launchFallbackTaskScript(params.scriptPath)`, but that helper expects a `GatewayServiceEnv` and reads the task script plus inline env from it. Passing a file path string breaks the fallback path when the task run is accepted but no process starts.
  - confidence: 0.93
- **[P1] Await and pass the env for Startup fallback launch:** `src/daemon/schtasks.ts:1047`
  - body: The create fallback passes `scriptPath` to `launchFallbackTaskScript` and does not await it, so the access-denied Startup-folder path can report success while launching the wrong path or dropping an async rejection. Keep the existing env-based, awaited launch contract here.
  - confidence: 0.91
- **[P1] Use cmd-safe quoting for PowerShell task actions:** `src/daemon/schtasks.ts:977`
  - body: The PowerShell task action builds `cmd.exe` arguments as `/d /s /c '<scriptPath>'`. `cmd.exe` does not use single quotes as path quoting, so paths with spaces or command metacharacters can fail or be parsed as extra commands; reuse the existing cmd quoting semantics for create and update actions.
  - confidence: 0.9
- **[P2] Remove the idle flag from ONLOGON fallback:** `src/daemon/schtasks.ts:1027-1028`
  - body: The `schtasks` fallback creates `/SC ONLOGON` while also passing `/I 0`. Microsoft documents `/I` as valid only with `ONIDLE` and a 1-999 value, so fallback installs can fail instead of registering the gateway task.
  - confidence: 0.96
- **[P1] Continue uninstall cleanup after PowerShell removal:** `src/daemon/schtasks.ts:1105`
  - body: Returning immediately after `Unregister-ScheduledTask` skips deletion of the Startup-folder launcher and generated task script below. Machines that previously used the fallback can keep auto-starting after uninstall, so remove the task and still run artifact cleanup.
  - confidence: 0.96
