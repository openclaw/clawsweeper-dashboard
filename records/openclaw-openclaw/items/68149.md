## Summary

Keep open: current main still uses the schtasks/Startup-folder Windows gateway path, and the PR is not safe to merge because its PowerShell path introduces blocking command quoting, fallback, and uninstall cleanup regressions.

## What This Changes

This PR changes Windows gateway Scheduled Task management in `src/daemon/schtasks.ts` to prefer PowerShell ScheduledTasks cmdlets, with schtasks kept as a fallback.

## Best Possible Solution

Keep the PowerShell ScheduledTasks direction only after fixing cmd quoting, valid fallback args, uninstall cleanup, and the chosen Startup-folder policy across code, tests, docs, and changelog.

## Reproduction Assessment

Yes. The blocking findings can be reproduced with focused unit tests or mocks that assert the generated `New-ScheduledTaskAction` argument, schtasks fallback create args, and uninstall cleanup after simulated PowerShell success.

## Solution Assessment

No. PowerShell ScheduledTasks may be a good direction for execution limits and battery settings, but this patch is not the narrowest maintainable fix until it preserves valid command construction, fallback behavior, cleanup, and policy/docs alignment.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Quote the task script for cmd.exe:** `src/daemon/schtasks.ts:977`
  - body: The PowerShell action stores `/d /s /c '${safeScriptPath}'`; after PowerShell unescapes it, cmd.exe sees single quotes, which are not command quoting. New and updated tasks can fail to launch, and paths with cmd metacharacters are parsed incorrectly, so build this argument with the existing cmd quoting semantics in both create and update paths.
  - confidence: 0.88
- **[P2] Remove the idle flag from the ONLOGON fallback:** `src/daemon/schtasks.ts:1027-1028`
  - body: The schtasks fallback appends `/I 0` while creating `/SC ONLOGON`. Microsoft documents `/i` as valid only for `ONIDLE` and requiring 1-999, so fallback installs can fail with invalid parameters instead of installing the gateway task.
  - confidence: 0.96
- **[P1] Continue uninstall cleanup after PowerShell removal:** `src/daemon/schtasks.ts:1105`
  - body: Returning after `Unregister-ScheduledTask` skips deletion of the Startup-folder launcher and generated gateway script below. Systems that previously used the fallback can keep auto-starting after uninstall, so continue to artifact cleanup after removing the task.
  - confidence: 0.97
