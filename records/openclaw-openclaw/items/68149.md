## Summary

This PR still needs maintainer handling because the PowerShell ScheduledTasks direction is plausible, but the branch changes established Windows install/uninstall policy and still has blocking functional regressions in command construction, fallback creation, and cleanup after PowerShell removal.

## What This Changes

The PR changes `src/daemon/schtasks.ts` to prefer PowerShell ScheduledTasks cmdlets for Windows gateway auto-start, set task lifetime and battery settings, require administrator privileges, and keep `schtasks` as a fallback.

## Best Possible Solution

Land a narrower Windows ScheduledTasks fix only after preserving valid cmd quoting, valid schtasks fallback args, uninstall artifact cleanup, and an explicitly approved install fallback policy with matching tests and docs.

## Reproduction Assessment

Yes for the review blockers: source inspection of the PR head gives focused reproduction targets for unit tests or mocks around generated PowerShell action arguments, schtasks fallback create args, and uninstall cleanup after simulated PowerShell success.

## Solution Assessment

No. PowerShell ScheduledTasks may be the right mechanism for execution limit and battery settings, but this branch is not yet the best mergeable solution because it changes fallback policy and introduces concrete command/fallback/cleanup regressions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Quote the task script for cmd.exe:** `src/daemon/schtasks.ts:977`
  - body: The registered action passes `/d /s /c '${safeScriptPath}'` through PowerShell. PowerShell removes the doubled quotes into single quotes, but `cmd.exe` does not use single quotes for path quoting, so paths with spaces or cmd metacharacters can fail to launch; build this argument with the existing cmd quoting semantics in both create and update paths.
  - confidence: 0.88
- **[P2] Remove the idle flag from the ONLOGON fallback:** `src/daemon/schtasks.ts:1027-1028`
  - body: The schtasks fallback creates `/SC ONLOGON` while also passing `/I 0`. `schtasks /Create` only accepts the idle option for ONIDLE schedules and requires a 1-999 value, so fallback installs can fail instead of registering the gateway task.
  - confidence: 0.96
- **[P1] Continue uninstall cleanup after PowerShell removal:** `src/daemon/schtasks.ts:1105`
  - body: Returning immediately after `Unregister-ScheduledTask` skips deletion of the Startup-folder launcher and generated gateway script below. Machines that previously used the fallback can keep auto-starting after uninstall, so remove the task and still run artifact cleanup.
  - confidence: 0.97
