## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Seed sync from the installed plugin records:** `src/gateway/server-methods/update.ts:45`
  - body: `updateNpmInstalledPlugins` gets its targets from `config.plugins.installs`, but this passes the gateway runtime config directly instead of loading the persisted installed-plugin index like the CLI path does. In the normal externalized-plugin setup, there are no install records here, so the reported `codex` drift still has nothing to update.
  - confidence: 0.92
- **[P1] Persist plugin updates through the install index:** `src/gateway/server-methods/update.ts:73`
  - body: Writing `npmResult.config` directly bypasses the existing install-record commit path, which strips `plugins.installs` from user config, writes the installed-plugin index, and refreshes the registry. If this branch ever changes plugins, it can leave plugin source state split between `openclaw.json` and the installed-plugin index.
  - confidence: 0.9
- **[P1] Fail the update before restarting on plugin sync errors:** `src/gateway/server-methods/update.ts:164-172`
  - body: The CLI treats post-core plugin-update errors as an update failure before restart, but this gateway path catches sync exceptions or records `postUpdate.plugins.status: "error"` while leaving `result.status` as ok. That can restart the updated gateway with exactly the stale or incompatible plugin set this change is meant to prevent.
  - confidence: 0.88
