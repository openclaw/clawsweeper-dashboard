## Best Possible Solution

Keep doctor repair as the first stale-route hint, but make the reply also cover persistent or non-stale missing Codex auth by naming the `models auth login --provider openai-codex` fallback or by distinguishing stale route state before selecting doctor-only copy.

## Solution Assessment

No, not as written. Doctor is the right repair for stale Codex route/session state, but the shared error signature also covers normal missing Codex auth, so the maintainable fix should preserve both recovery paths or add a more specific detector.
