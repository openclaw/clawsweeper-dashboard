## Summary

Keep this PR open: it documents a useful queue/streaming gotcha that is not fully covered on current main, but the current patch still over-promises that `interrupt` can stop an in-flight tool immediately.

## What This Changes

The PR adds `docs/concepts/queue.md` guidance about steer with partial/block streaming, non-streaming fallback, tool-boundary timing, and per-session queue troubleshooting.

## Best Possible Solution

Land a docs-only update that keeps the partial/block streaming gotcha while describing `interrupt` as aborting/replacing the active run, with actual tool cancellation depending on runtime/tool abort support.

## Reproduction Assessment

Not applicable; this is a docs-only PR, not a failing runtime bug report. The relevant claims are source-checkable against current queue docs, steering docs, streaming docs, and the queue helper.

## Solution Assessment

No; the docs direction is useful, but this patch is not the best final wording because it promises immediate in-flight tool stopping. The safer solution is to describe interrupt as abort-and-replace while noting cancellation depends on runtime/tool abort support.
