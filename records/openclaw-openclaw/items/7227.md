## Summary

Keep open. Current main has partial mitigation for the original Peekaboo path: the bundled Peekaboo skill now targets a standalone `peekaboo` binary and the macOS app can host PeekabooBridge so UI automation can reuse `OpenClaw.app` TCC permissions. But the broader node/TCC concern is not clearly closed out: macOS headless node `system.run` still defaults to local node execution unless `OPENCLAW_NODE_EXEC_HOST=app` is set, bundled skills are still allowed by default, PeekabooBridge defaults to enabled in the app, and the permissions UI does not add the kind of prominent security warning requested here. That is not strong enough evidence for a conservative close.
