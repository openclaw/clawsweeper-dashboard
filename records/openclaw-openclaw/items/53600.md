## What This Changes

The issue requests a systematic analysis or instrumentation plan for reasoning slowness in a native OpenClaw 2026.3.13 setup on a constrained Hetzner VPS, covering context assembly, memory search, compaction, subagents, Telegram, hooks, auth fallback, and Node/OS pressure.

## Best Possible Solution

Scope a bounded native-Linux gateway performance investigation that records queue wait, context assembly duration, token estimation/truncation, memory query embedding/search, compaction trigger/duration, subagent spawn/announce, Telegram delivery, hook execution, auth decisions, heap/RSS/GC/event-loop pressure, then publish operator guidance and split confirmed hot spots into focused optimization follow-ups.
