## What This Changes

Control console session list should show full local date/time in `YYYY-MM-DD HH:mm` format instead of only relative or time-only labels.

## Best Possible Solution

Render the Sessions Updated column with a tested local full date-time formatter while preserving the existing sessions API, row data, and timestamp-based sorting.

## Reproduction Assessment

Yes. Source inspection on current main shows a non-null `updatedAt` flows through `formatRelativeTimestamp(row.updatedAt)` and is rendered directly in the Sessions Updated cell.

## Solution Assessment

Yes. A narrow Control UI formatting change is the maintainable solution because the row already carries `updatedAt`; there is no evidence that the sessions API, storage, or sorting contract needs to change.
