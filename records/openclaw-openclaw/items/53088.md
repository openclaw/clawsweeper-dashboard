## Summary

Keep open. Current main still renders the Control UI Sessions Updated column from the relative timestamp formatter, and open PR #53445 still references this issue with closing syntax, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that the Control UI session list show a full local date/time in the Updated column instead of relative-only time text.

## Best Possible Solution

Land #53445 if acceptable after review/rebase, or apply the same narrow Control UI change on main: add a tested local full date-time formatter, render the Sessions Updated column as local `YYYY-MM-DD HH:mm`, and preserve the existing sessions API, row data, and sorting behavior.

Yes. Source inspection on current main shows a non-null `updatedAt` flows through `formatRelativeTimestamp(row.updatedAt)` and is rendered directly in the Updated cell, reproducing the relative-only display path.

## Solution Assessment

Yes. A narrow UI formatting change is the best maintainable fix because the row already carries `updatedAt`; there is no evidence that the sessions API or storage contract needs to change.
