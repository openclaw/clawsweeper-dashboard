## Summary

Keep open. Current main still renders the Control UI Sessions Updated column as relative timestamp text rather than the requested full local date/time, and an open unmerged implementation PR already tracks this exact UI change.

## What This Changes

The issue requests that the Control UI Sessions list show full local date/time, such as `2026-03-20 14:30`, instead of only relative time in the Updated column.

## Best Possible Solution

Land #53445 if acceptable after review/rebase, or make the same narrow Control UI change on main: format the Sessions Updated column as local `YYYY-MM-DD HH:mm`, preserve the existing sessions API and sorting behavior, and cover the formatter/rendering path with focused UI tests.

## Reproduction Assessment

Yes. On current main, a Sessions row with a non-null `updatedAt` flows through `formatRelativeTimestamp(row.updatedAt)` and renders that string directly in the Updated cell, which reproduces the relative-only display from source inspection.

## Solution Assessment

Yes. A narrow UI formatting change is the best maintainable fix because the row already carries `updatedAt`; there is no evidence that the sessions API or storage contract needs to change.
