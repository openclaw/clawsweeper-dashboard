## Summary

Keep open: current main still lacks a configurable Control UI browser title, and this PR contains the right core idea, but the branch is stale/conflicting and still needs two narrow integration fixes before merge.

## What This Changes

The PR adds a `gateway.controlUi.title` config key, includes it in Control UI bootstrap JSON, applies it to `document.title`, and adds bootstrap tests for configured and unset titles.

## Best Possible Solution

Land a rebased or replacement implementation that fully exposes `gateway.controlUi.title`, documents the config surface, serves it through authenticated bootstrap JSON, applies it on initial load and post-auth refreshes, preserves `OpenClaw Control` when unset, and composes with dynamic per-agent titles.

## Reproduction Assessment

Not applicable: this is a new feature/config option rather than a bug report. Source inspection of current main confirms the baseline still has a static title and no `gateway.controlUi.title` config or bootstrap field.

## Solution Assessment

Mostly yes: carrying an operator-configured string through the existing config and authenticated bootstrap path into `document.title` is the narrow maintainable shape. The branch still needs the public config surface completed and title application integrated with runtime bootstrap refreshes.
