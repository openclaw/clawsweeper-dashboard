## Summary

Keep open: current main still lacks configurable Control UI title support, and this branch is a plausible narrow implementation, but it is incomplete because the new strict config option is not wired into the public config metadata, docs, baseline, and gateway/config coverage.

## What This Changes

The PR adds optional `gateway.controlUi.title`, sends it in the Control UI bootstrap JSON, applies it to `document.title`, and adds UI bootstrap tests for configured and default titles.

## Best Possible Solution

Land the bootstrap-based `gateway.controlUi.title` support with complete config metadata, public docs, config baseline updates, and focused gateway/config/UI coverage, without server-side HTML title rewriting.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config PR. Source inspection against current main shows `gateway.controlUi.title` is still absent from the strict schema and bootstrap payload, while the Control UI still ships a fixed default title.

## Solution Assessment

Mostly yes: passing a plain config string through the existing bootstrap JSON and assigning `document.title` is the narrow maintainable path. The missing piece is completing the public config contract and coverage around the new option.
