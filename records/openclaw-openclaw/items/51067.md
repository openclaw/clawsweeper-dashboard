Keep open: this is a useful static Control UI title feature, but the PR is not merge-ready because it adds the raw schema/bootstrap/browser plumbing without the public config metadata, docs/baseline updates, and current-main rebase needed for a new `gateway.controlUi.*` option.

## What This Changes

The PR adds optional `gateway.controlUi.title` plumbing from config to Control UI bootstrap JSON and assigns it to `document.title` in the browser.

## Best Possible Solution

Land a rebased implementation that treats `gateway.controlUi.title` as a documented public config option and carries it through authenticated bootstrap JSON into `document.title` without server-side HTML interpolation.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config PR. Source inspection of current main shows the Control UI title is static, `gateway.controlUi.title` is not accepted by the schema, and the bootstrap payload has no title field.

## Solution Assessment

Mostly yes: using the existing authenticated bootstrap JSON response and assigning `document.title` in the browser is the narrow maintainable direction. The branch still needs the full public config contract, focused coverage, docs/baseline updates, and a rebase over current bootstrap behavior.
