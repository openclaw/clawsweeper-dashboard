## Summary

Keep open: current main still lacks the static Control UI title configuration, and this PR contains the right central feature, but the live diff is not merge-ready because it leaves the public config contract incomplete and applies the title in a bootstrap path current refreshes can skip.

## What This Changes

The PR adds `gateway.controlUi.title` and passes it through the gateway bootstrap response to set the Control UI browser tab title, with UI tests for configured and default cases.

## Best Possible Solution

Land a rebased implementation that exposes and documents `gateway.controlUi.title`, carries it through the authenticated bootstrap JSON path, and applies it independently of identity refreshes while preserving the default title when unset.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config PR. Source inspection confirms current main still has the static title path and lacks the custom static title config/bootstrap/UI wiring.

## Solution Assessment

Mostly yes: the authenticated bootstrap payload plus browser-side `document.title` assignment is the narrow maintainable direction. The patch is not merge-ready until it completes the public config/docs/test contract and moves title application outside identity-only refresh logic.
