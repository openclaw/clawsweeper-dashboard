## Summary

Keep open: the static configurable Control UI title is still absent from current main, and this PR is useful but not merge-ready because it has not completed the public config/docs contract and needs to be rebased around current bootstrap refresh behavior.

## What This Changes

The PR adds a `gateway.controlUi.title` option, includes it in the Control UI bootstrap payload, assigns it to `document.title`, and adds bootstrap-loader tests.

## Best Possible Solution

Land a rebased implementation that documents and exposes `gateway.controlUi.title` as a public config option, carries it through the authenticated bootstrap JSON path, and applies it independently of identity refreshes while preserving the default title when unset.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config PR. Source inspection confirms current main still uses a static title and lacks the config, bootstrap payload, and UI assignment paths for a custom static title.

## Solution Assessment

Mostly yes: using the authenticated bootstrap JSON payload plus a browser-side `document.title` assignment is the narrow maintainable direction. The current patch is not merge-ready until it completes config metadata/docs/tests and moves the title write outside identity-only refresh logic.
