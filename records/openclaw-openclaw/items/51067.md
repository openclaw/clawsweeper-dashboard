## Summary

Keep open: current main still does not implement a configurable Control UI browser title, and this PR remains the canonical static-title implementation path. The branch is useful but not merge-ready because the config surface is incomplete and the UI write is placed where an authenticated runtime bootstrap refresh can skip it.

The PR adds optional `gateway.controlUi.title`, passes it through the Control UI bootstrap payload, applies it to `document.title`, and adds bootstrap-controller tests.

## Best Possible Solution

Land a rebased or replacement patch that makes `gateway.controlUi.title` a documented config field, carries it through authenticated bootstrap runtime data, applies it outside identity-only refreshes, and preserves `OpenClaw Control` when unset.
