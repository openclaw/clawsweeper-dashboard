## Summary

Keep open: current main still lacks the static Control UI title config, and this PR remains the canonical implementation for that static-title request, but the patch is not merge-ready because it leaves config metadata/docs/tests incomplete and applies the title only inside identity bootstrap updates.

## What This Changes

The branch adds optional `gateway.controlUi.title` wiring through config schema/types, the Control UI bootstrap payload, browser title assignment, and bootstrap controller tests.

## Best Possible Solution

Land a rebased implementation that exposes and documents `gateway.controlUi.title`, carries it through the authenticated bootstrap JSON path, applies it independently of identity refreshes, preserves the default title when unset, and composes with dynamic agent-title work.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config PR. Source inspection on current main confirms the static title path and the absence of `gateway.controlUi.title` config/bootstrap/UI wiring.

## Solution Assessment

Mostly yes: using the authenticated bootstrap payload plus a browser-side `document.title` assignment is the narrow maintainable path. The patch is not merge-ready until it completes the config/docs/test contract and moves title application outside identity-only refresh logic.
