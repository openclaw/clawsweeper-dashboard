## Summary

Keep this PR open: it has the protected `maintainer` label, the current head still has source-proven blocking regressions, and the PR body/comments do not show after-fix live Canvas/A2UI behavior proof.

## What This Changes

This PR moves Canvas/A2UI into a bundled experimental Canvas plugin, removes vendored/generated A2UI and protocol/client surfaces, and adds plugin-owned Canvas config, HTTP/WS routes, node policies, docs, and migration tests.

## Best Possible Solution

Keep the Canvas plugin migration under maintainer review, fix the source-proven regressions, add focused tests for those contracts, and require real Canvas/A2UI runtime proof before merge.

## Reproduction Assessment

Yes for the review findings: source inspection shows the snapshot path drops the existing image sanitization parameter, and the WebSocket upgrade path always dispatches plugin upgrades with `gatewayAuthSatisfied: false`. No live Canvas end-to-end proof was established for the broader refactor.

## Solution Assessment

No. The refactor direction may be reasonable, but the current implementation is not the best merge candidate until it preserves Canvas snapshot sanitization and authorizes gateway-auth plugin WebSocket upgrades with the same auth context as HTTP routes.

## Review Findings

  - body: The plugin version of `canvas.snapshot` returns `imageResultFromFile(...)` without forwarding image sanitization limits from config.   - confidence: 0.9
