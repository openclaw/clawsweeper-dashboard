## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Pass gateway auth into plugin WebSocket dispatch:** `src/gateway/server-http.ts:869-871`
  - body: `handlePluginUpgrade` is still called with `gatewayAuthSatisfied: false`, while gateway-authenticated plugin upgrade routes require that flag and scope context. Gateway-authenticated plugin WebSocket routes are therefore unreachable after this change.
  - confidence: 0.94
- **[P1] Provide a capability reissue path after token expiry:** `src/gateway/server/ws-connection/message-handler.ts:1328-1329`
  - body: Plugin surface capability tokens are minted only during `connect` with a finite expiry, but the Canvas refresh RPC path is removed. A long-lived connected node that idles past the TTL cannot recover its Canvas/A2UI URL without reconnecting.
  - confidence: 0.91
- **[P2] Preserve Canvas snapshot image sanitization limits:** `extensions/canvas/src/tool.ts:217-221`
  - body: The moved Canvas snapshot path returns `imageResultFromFile(...)` without forwarding configured image sanitization limits. Users relying on `agents.defaults.imageMaxDimensionPx` can again produce oversized provider image payloads.
  - confidence: 0.94
- **[P2] Isolate hosted media resolver exceptions per plugin:** `src/media/web-media.ts:78-79`
  - body: A thrown error from any registered `hostedMediaResolver` now bubbles out of `loadWebMedia`, which can break all media loading for unrelated URLs. Catch and log resolver faults, then continue to the next resolver.
  - confidence: 0.86
- **[P2] Preserve built-in node subcommands when loading plugin CLI:** `src/plugins/register-plugin-cli-command-groups.ts:78-83`
  - body: When `parentPath[0]` matches the primary command, this branch removes plugin command names before overlap checks. A plugin registering under `openclaw nodes` with a built-in name can replace the built-in command instead of being skipped.
  - confidence: 0.87
- **[P2] Require node capability only for the route that handles request:** `src/gateway/server/plugins-http/route-capability.ts:20-27`
  - body: Filtering all matching routes for the first `nodeCapability` route can force capability auth even when a more-specific matched route without node capability will handle the request. Capability gating should follow the effective dispatch route.
  - confidence: 0.84
- **[P2] Honor route-specific node capability TTL at connect time:** `src/gateway/server/ws-connection/message-handler.ts:1326-1329`
  - body: The connect handshake mints plugin capabilities from surface names only, so any `nodeCapability.ttlMs` declared on a plugin HTTP route is dropped until later request auth. First-use token lifetimes should respect the route contract.
  - confidence: 0.84
- **[P3] Add a user-facing changelog entry:** `docs/refactor/canvas.md:10`
  - body: This PR changes Canvas distribution, config/protocol compatibility, CLI behavior, plugin SDK seams, and dependency ownership, but `CHANGELOG.md` has no current Canvas/A2UI/plugin-surface entry.
  - confidence: 0.82
