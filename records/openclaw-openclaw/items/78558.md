## Summary

Keep open: this protected, broad PR still has source-proven blocking regressions in WebSocket auth, Canvas snapshot sanitization, plugin test loading, hosted media resolution, nested CLI collision handling, and node-capability route selection, plus missing real behavior proof.

## What This Changes

The branch moves Canvas from core into a bundled experimental plugin, removes vendored/generated A2UI and legacy Canvas protocol/config compatibility, and adds generic plugin seams for CLI, HTTP, WebSocket, media, surface URLs, and node capability auth.

## Best Possible Solution

Land a repaired Canvas plugin refactor that preserves current auth, image limits, command collision behavior, route-specific capability semantics, and Canvas availability, with changelog coverage and live paired-node or maintained-lane Canvas/A2UI proof.

## Reproduction Assessment

Yes for the review findings: PR-head source inspection gives high-confidence reproduction paths for the WebSocket auth, snapshot sanitization, missing helper, media resolver, nested CLI, and route-capability regressions. I did not live-run paired-node Canvas/A2UI because this review is read-only and the PR proof does not cover that runtime path.

## Solution Assessment

No. Moving Canvas behind generic plugin seams may be a maintainable direction, but this patch is not the best merge candidate until the blocking regressions, mergeability, changelog, and live behavior proof gaps are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Pass gateway auth into plugin WebSocket dispatch:** `src/gateway/server-http.ts:869-871`
  - body: `handlePluginUpgrade` is still called with `gatewayAuthSatisfied: false`, while gateway-authenticated plugin upgrade routes are rejected unless that flag and auth/scope context are present. Gateway-authenticated plugin WebSocket routes are therefore unreachable after this change.
  - confidence: 0.94
- **[P2] Preserve Canvas snapshot image sanitization limits:** `extensions/canvas/src/tool.ts:217-221`
  - body: The moved Canvas snapshot path returns `imageResultFromFile(...)` without forwarding the configured image sanitization limits. Users relying on `agents.defaults.imageMaxDimensionPx` can again produce oversized provider image payloads.
  - confidence: 0.94
- **[P2] Add the managed NPM plugin helper:** `src/plugins/installed-plugin-index-records.test.ts:17`
  - body: This test imports `./test-helpers/managed-npm-plugin.js`, but the helper file is absent from the PR-head tree. The suite will fail at module load before the assertions run.
  - confidence: 0.93
- **[P2] Isolate hosted media resolver exceptions per plugin:** `src/media/web-media.ts:78-79`
  - body: A thrown error from any registered `hostedMediaResolver` now bubbles out of `loadWebMedia`, which can break all media loading for unrelated URLs. Catch and log resolver faults, then continue to the next resolver.
  - confidence: 0.86
- **[P2] Preserve built-in node subcommands when loading plugin CLI:** `src/plugins/register-plugin-cli-command-groups.ts:78-83`
  - body: When `parentPath[0]` matches the primary command, this branch removes plugin command names before overlap checks. A plugin registering under `openclaw nodes` with a built-in name can replace the built-in command instead of being skipped.
  - confidence: 0.87
- **[P2] Derive node capability auth from the dispatched route:** `src/gateway/server/plugins-http/route-capability.ts:20`
  - body: Filtering all matching routes for the first `nodeCapability` route can force capability auth even when a more-specific matched route without node capability will handle the request. Capability gating should follow the effective dispatch route.
  - confidence: 0.84
- **[P3] Add a user-facing changelog entry:** `docs/refactor/canvas.md:10`
  - body: This PR changes Canvas distribution, config/protocol compatibility, CLI behavior, plugin SDK seams, and production dependency ownership, but the PR files do not include `CHANGELOG.md`. Repo policy requires user-facing behavior changes to be recorded before merge.
  - confidence: 0.82
