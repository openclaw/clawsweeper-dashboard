## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Pass gateway auth context into plugin WebSocket upgrades:** `src/gateway/server-http.ts:869-871`
  - body: The upgrade path always calls plugin WebSocket dispatch with `gatewayAuthSatisfied: false`. Any plugin route registered with `auth: "gateway"` is blocked by the plugin upgrade handler, so authenticated plugin WebSocket endpoints cannot be served and fall through to the generic gateway WebSocket path.
  - confidence: 0.92
- **[P2] Preserve Canvas snapshot image sanitization limits:** `extensions/canvas/src/tool.ts:217-221`
  - body: The plugin Canvas snapshot path now returns `imageResultFromFile(...)` without forwarding configured image sanitization limits. Current main passes those limits into the snapshot result, so users relying on `agents.defaults.imageMaxDimensionPx` can again produce oversized provider payloads.
  - confidence: 0.94
- **[P2] Add the managed npm plugin test helper:** `src/plugins/installed-plugin-index-records.test.ts:17`
  - body: These tests import `./test-helpers/managed-npm-plugin.js`, but the helper file is absent from the PR head. The affected plugin tests will fail during module loading before their assertions run.
  - confidence: 0.9
- **[P2] Advertise Canvas host only after routes register:** `src/gateway/server-runtime-config.ts:126-129`
  - body: `canvasHostEnabled` is now derived from config before Canvas plugin route registration is known. Because gateway plugin load errors are recorded non-fatally, clients can receive a `canvasHostUrl` even when the Canvas plugin failed to register its handlers, turning the advertised Canvas URL into a 404 or failed upgrade.
  - confidence: 0.86
- **[P3] Add the user-facing changelog entry:** `docs/refactor/canvas.md:12`
  - body: This PR changes Canvas distribution, config migration, CLI behavior, and plugin SDK surfaces, but `CHANGELOG.md` is not changed in the PR file list. OpenClaw policy requires a changelog entry for user-facing feature or behavior changes before merge.
  - confidence: 0.84
