## Summary

Keep this PR open: it is a protected maintainer-labeled, merge-dirty XL refactor, and current PR-head inspection still shows blocking node refresh, plugin WebSocket auth, Canvas image handling, route matching, CLI conflict, and live-proof gaps.

## What This Changes

This PR moves Canvas runtime ownership into a bundled experimental plugin, removes vendored A2UI, adds generic plugin HTTP/WebSocket/media/CLI/surface URL seams, and updates native clients, docs, SDK baselines, Docker, CI, and package metadata.

## Best Possible Solution

Rebase and land a repaired Canvas plugin refactor that preserves node refresh authorization, gateway WebSocket auth, image sanitization, resolver isolation, CLI conflict rules, route-specific capability semantics, changelog coverage, and live paired-node proof.

## Reproduction Assessment

Yes for the review findings: PR-head source inspection shows node refresh methods are registered and called but not node-role authorized, plugin WebSocket dispatch cannot satisfy gateway-auth routes, and Canvas snapshots omit existing image-sanitization limits. I did not run live paired-node Canvas/A2UI in this read-only review.

## Solution Assessment

No. Moving Canvas behind generic plugin seams may be the right direction, but this patch is not the best merge candidate until the auth, refresh, runtime, changelog, rebase, and real behavior proof blockers are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Allow node clients to refresh plugin surface URLs:** `src/gateway/method-scopes.ts:35-39`
  - body: `node.pluginSurface.refresh` and the legacy Canvas refresh alias are registered handlers and native clients now call them, but neither method is in `NODE_ROLE_METHODS`. A real node connection will be rejected with `unauthorized role: node` before it can refresh an expired Canvas/plugin surface URL.
  - confidence: 0.96
- **[P1] Pass gateway auth into plugin WebSocket dispatch:** `src/gateway/server-http.ts:869-871`
  - body: The upgrade path always calls plugin WebSocket dispatch with `gatewayAuthSatisfied: false`, so any plugin route using `auth: "gateway"` can never pass the dispatcher auth gate even after the request was authenticated by the gateway.
  - confidence: 0.94
- **[P2] Reject blocked plugin upgrades instead of falling through:** `src/gateway/server/plugins-http.ts:189-193`
  - body: When a matched plugin upgrade route fails gateway-auth preconditions, returning `false` makes the caller treat it as unhandled and continue into the generic gateway WebSocket handler. Blocked plugin upgrade routes should terminate the upgrade with an auth failure.
  - confidence: 0.9
- **[P2] Preserve Canvas snapshot image limits:** `extensions/canvas/src/tool.ts:217-221`
  - body: The moved Canvas snapshot tool returns `imageResultFromFile(...)` without forwarding the configured image sanitization limits, so users relying on `agents.defaults.imageMaxDimensionPx` can again send oversized provider image payloads.
  - confidence: 0.94
- **[P2] Isolate hosted media resolver failures per plugin:** `src/media/web-media.ts:77-79`
  - body: A thrown exception from any hosted media resolver now bubbles out of `loadWebMedia`, so one plugin resolver can break media loading for unrelated URLs. Catch/log resolver faults and continue to the next resolver.
  - confidence: 0.86
- **[P2] Preserve built-in nested CLI commands:** `src/plugins/register-plugin-cli-command-groups.ts:78-83`
  - body: This branch removes command groups before overlap checks when the parent path matches the primary command, letting a plugin under `openclaw nodes` replace built-in node subcommands such as `status`. Nested plugin commands should still respect existing-command conflict handling.
  - confidence: 0.87
- **[P2] Apply capability auth to the dispatched route:** `src/gateway/server/plugins-http/route-capability.ts:20-27`
  - body: Selecting the first matching route with `nodeCapability` can force capability auth even when a more-specific matched route without capability auth will handle the request. Capability gating should follow the effective dispatch route/order.
  - confidence: 0.84
- **[P2] Honor route-specific TTL when minting URLs:** `src/gateway/server/ws-connection/message-handler.ts:1325-1329`
  - body: Connect-time plugin surface URLs are minted from `{ surface }` only, so any `nodeCapability.ttlMs` declared on the HTTP route is ignored and the default TTL is used for first-use tokens.
  - confidence: 0.84
- **[P2] Retry Canvas host initialization after transient failures:** `extensions/canvas/src/http-route.ts:26-40`
  - body: `hostHandlerPromise ??=` caches a rejected initialization promise, so a transient Canvas host startup failure keeps every later HTTP request and upgrade failing until process restart. Clear the cached promise on rejection so later requests can retry.
  - confidence: 0.83
- **[P3] Add a user-facing changelog entry:** `docs/refactor/canvas.md:10`
  - body: This PR changes Canvas distribution, config/protocol compatibility, CLI behavior, plugin SDK seams, and dependency ownership, but the PR head does not add a current `CHANGELOG.md` entry for those user-facing changes.
  - confidence: 0.82
