## Summary

Keep open: this protected, maintainer-labeled XL refactor is not closeable by cleanup automation, and the current head still has source-proven blockers in Canvas snapshot sanitization and plugin WebSocket auth.

## What This Changes

This PR moves Canvas/A2UI into a bundled experimental Canvas plugin, removes vendored/generated A2UI and app-server protocol surfaces, adds plugin-owned HTTP/WS routes and config migration, and introduces nested plugin CLI registration.

## Best Possible Solution

Fix the snapshot sanitization and WebSocket auth regressions, add focused regression coverage plus a required changelog entry, complete live Canvas/A2UI proof, and keep the ownership move under maintainer review.

## Reproduction Assessment

Yes for the review findings: source inspection at the live PR head shows the snapshot path drops the existing image-sanitization parameter and plugin WebSocket upgrades are always dispatched with `gatewayAuthSatisfied: false`. I did not run a live Canvas node reproduction for the broader refactor.

## Solution Assessment

No. The refactor direction may be reasonable, but this patch is not the best merge candidate until it preserves current Canvas image limits, authenticates gateway-auth plugin WebSocket upgrades, records the user-facing change, and demonstrates the live runtime path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Authorize plugin WebSocket upgrades before dispatch:** `src/gateway/server-http.ts:870-872`
  - body: The upgrade path always calls plugin WS dispatch with `gatewayAuthSatisfied: false`. Any plugin WebSocket route registered with `auth: "gateway"` is blocked by `createGatewayPluginUpgradeHandler` and falls through to the generic gateway WebSocket path instead of the plugin endpoint.
  - confidence: 0.91
- **[P2] Preserve Canvas snapshot image sanitization:** `extensions/canvas/src/tool.ts:217-221`
  - body: The plugin version of `canvas.snapshot` returns `imageResultFromFile(...)` without forwarding the configured image sanitization limits. Current main passes those limits into the snapshot result, so users relying on `agents.defaults.imageMaxDimensionPx` can again produce oversized provider payloads.
  - confidence: 0.93
- **[P3] Add the required changelog entry:** `docs/refactor/canvas.md:11`
  - body: This PR changes user-visible Canvas distribution/config/CLI behavior and adds nested plugin CLI capability, but the diff does not update `CHANGELOG.md`. OpenClaw policy requires user-facing changes like this to be recorded before merge.
  - confidence: 0.84
