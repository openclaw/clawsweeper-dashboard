## Summary

Keep open: current head still has blocking regressions in plugin WebSocket upgrade auth, Canvas snapshot image sanitization, and plugin test imports, and the protected `maintainer` label plus broad Canvas ownership move require explicit maintainer handling.

## What This Changes

This PR moves Canvas/A2UI from core into a bundled experimental Canvas plugin, removes vendored/generated A2UI surfaces, and adds generic plugin CLI, HTTP, WebSocket, hosted-media, and config-migration seams.

## Best Possible Solution

Preserve Canvas compatibility while moving ownership only after fixing the WebSocket auth context, snapshot sanitization, missing helper and changelog gaps, then proving the live Canvas/A2UI path on a real paired node or equivalent maintained lane.

## Reproduction Assessment

Yes for the review findings: source inspection of the current PR head shows the missing WebSocket auth context, dropped snapshot sanitization argument, and missing test helper import. I did not run a live Canvas node reproduction for the broader refactor.

## Solution Assessment

No. The refactor direction may be maintainable, but this patch is not the best merge candidate until it preserves current Canvas limits, wires authenticated plugin WebSocket upgrades, restores the helper/changelog requirements, and includes live behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Pass auth context into plugin WebSocket upgrades:** `src/gateway/server-http.ts:869-871`
  - body: The upgrade path always calls plugin WebSocket dispatch with `gatewayAuthSatisfied: false`. Any plugin route registered with `auth: "gateway"` is blocked by `createGatewayPluginUpgradeHandler`, so authenticated plugin WebSocket endpoints are skipped instead of being served.
  - confidence: 0.92
- **[P2] Preserve Canvas snapshot image sanitization:** `extensions/canvas/src/tool.ts:217-221`
  - body: The plugin `canvas.snapshot` path returns `imageResultFromFile(...)` without forwarding the configured image sanitization limits. Current main passes those limits into the snapshot result, so users relying on `agents.defaults.imageMaxDimensionPx` can again produce oversized provider payloads.
  - confidence: 0.94
- **[P2] Add the extracted managed plugin helper:** `src/plugins/installed-plugin-index-records.test.ts:17`
  - body: These tests now import `./test-helpers/managed-npm-plugin.js`, but the helper file is absent from the current head. The affected plugin test files will fail during module loading before their assertions run.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `docs/refactor/canvas.md:11`
  - body: This PR changes user-visible Canvas distribution/config/CLI behavior and adds nested plugin CLI capability, but it does not update `CHANGELOG.md`. OpenClaw policy requires user-facing changes like this to be recorded before merge.
  - confidence: 0.84
