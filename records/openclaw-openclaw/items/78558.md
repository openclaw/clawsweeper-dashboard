## Summary

Keep open: this protected, broad Canvas/plugin refactor still has current PR-head blocking regressions in plugin WebSocket auth, Canvas media limits, a missing test helper, Canvas host advertisement, hosted-media isolation, and nested CLI command shadowing, with insufficient live Canvas/A2UI runtime proof.

## What This Changes

This PR moves Canvas into a bundled experimental plugin, removes vendored/generated A2UI and protocol surfaces, adds plugin SDK seams for nested CLI/HTTP/WebSocket/media/node capability/config migration, and updates related docs, CI, Docker, scripts, and dependency pins.

## Best Possible Solution

Land a repaired Canvas plugin refactor that preserves current auth, media limits, command collision behavior, and Canvas availability semantics, includes changelog coverage, and has live paired-node or maintained-lane Canvas/A2UI proof.

## Reproduction Assessment

Yes for the review findings: PR-head source inspection gives high-confidence reproduction paths for the WebSocket auth, media limit, missing helper, Canvas advertisement, hosted-media resolver, and nested CLI regressions. I did not live-run paired-node Canvas/A2UI because this review is read-only and the PR proof does not cover that runtime path.

## Solution Assessment

No. Moving Canvas behind generic plugin seams may be a maintainable direction, but this patch is not the best merge candidate until the blocking regressions are fixed and the live Canvas/A2UI path is proven.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Pass gateway auth into plugin WebSocket dispatch:** `src/gateway/server-http.ts:869-871`
  - body: `handlePluginUpgrade` is always called with `gatewayAuthSatisfied: false`, while `createGatewayPluginUpgradeHandler` rejects matched routes with `auth: "gateway"` unless that flag and auth/scope context are present. Gateway-authenticated plugin WebSocket routes are therefore unreachable after this change.
  - confidence: 0.94
- **[P2] Preserve canvas snapshot sanitization limits:** `extensions/canvas/src/tool.ts:217-221`
  - body: The moved Canvas snapshot path returns `imageResultFromFile(...)` without forwarding the configured image sanitization limits. Current main passes those limits, so users relying on `agents.defaults.imageMaxDimensionPx` can again produce oversized provider image payloads.
  - confidence: 0.94
- **[P2] Add the managed NPM plugin helper:** `src/plugins/installed-plugin-index-records.test.ts:17`
  - body: This test imports `./test-helpers/managed-npm-plugin.js`, but the helper file is absent from the PR head. The suite will fail at module load before the assertions run.
  - confidence: 0.93
- **[P2] Advertise Canvas only after plugin routes load:** `src/gateway/server.impl.ts:1328`
  - body: `canvasHostEnabled` is now forwarded from config rather than actual Canvas route registration success. Because gateway plugin load errors are non-fatal by default, clients can receive `canvasHostUrl` for a Canvas plugin that failed to register its handlers, producing dead Canvas URLs instead of preserving the previous `Boolean(canvasHost)` behavior.
  - confidence: 0.88
- **[P2] Isolate hosted media resolver failures:** `src/media/web-media.ts:78-79`
  - body: `resolveHostedPluginMediaUrl` awaits each optional plugin resolver without a guard, so an unexpected throw from one resolver rejects `loadWebMedia` for unrelated inputs. Catch and log resolver faults, then continue to the next resolver.
  - confidence: 0.84
- **[P2] Avoid replacing built-in nested CLI commands:** `src/plugins/register-plugin-cli-command-groups.ts:78-83`
  - body: When `parentPath[0]` matches the primary command, this branch removes the plugin's command names before overlap checks. A plugin registering under `openclaw nodes` with a name like `status` can replace the built-in node command instead of being skipped as an overlap.
  - confidence: 0.87
- **[P3] Add a user-facing changelog entry:** `docs/refactor/canvas.md:10`
  - body: This PR changes Canvas distribution, config migration, CLI behavior, plugin SDK surfaces, and a production dependency override, but the diff has no `CHANGELOG.md` entry. Repo policy requires user-facing behavior changes to be recorded before merge.
  - confidence: 0.85
