## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Pass gateway auth into plugin upgrades:** `src/gateway/server-http.ts:870`
  - body: `handlePluginUpgrade` always receives `gatewayAuthSatisfied: false`, so any plugin WebSocket route declaring `auth: "gateway"` is rejected even after the gateway has authenticated the request, and the matched endpoint can fall through to the generic WebSocket path.
  - confidence: 0.95
- **[P2] Fail closed on blocked plugin upgrades:** `src/gateway/server/plugins-http.ts:190-192`
  - body: When a matched plugin upgrade route requires gateway auth, this handler logs and returns `false`. The caller treats that as unhandled and continues into the generic gateway WebSocket path, so the matched plugin endpoint is not rejected at its auth boundary.
  - confidence: 0.91
- **[P1] Scope node capabilities to the owning route:** `src/gateway/plugin-node-capability.ts:244-250`
  - body: Capability authorization is keyed only by the plugin-provided surface string. Two routes or plugins that reuse a surface name can share a token and bypass the intended owner or TTL policy for the route being accessed.
  - confidence: 0.9
- **[P2] Isolate hosted media resolver failures:** `src/media/web-media.ts:77-78`
  - body: A thrown error from one `hostedMediaResolver` escapes `loadWebMedia`, which can break unrelated media loads. Optional plugin-owned resolvers should be isolated per resolver and skipped unless they return a handled URL.
  - confidence: 0.86
- **[P2] Keep Crabbox on standard capacity by default:** `.crabbox.yaml:3`
  - body: Changing the repo default from `standard` to `beast` makes routine Crabbox sync/test commands request the largest owned AWS class, while current guidance keeps the default conservative and requires explicit escalation for oversized lanes.
  - confidence: 0.88
- **[P2] Restore proof redaction guidance:** `.github/pull_request_template.md:40`
  - body: The PR template drops the reminder to redact IP addresses, API keys, phone numbers, and non-public endpoints from external proof. That weakens the proof gate and conflicts with current contributor privacy guidance.
  - confidence: 0.84
