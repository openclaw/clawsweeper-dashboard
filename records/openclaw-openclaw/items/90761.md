Full review comments:

- **[P1] Verify SSH host keys before forwarding:** `apps/android/app/src/main/java/ai/openclaw/app/gateway/SshTunnelManager.kt:119`
  - body: This starts the tunnel with `StrictHostKeyChecking=no`, so a spoofed or changed SSH host is accepted before gateway traffic is forwarded through it. Add a pinned/TOFU host-key flow or split SSH tunneling out for maintainer security review before merging.
  - confidence: 0.96
- **[P1] Keep stored operator tokens on the device-token path:** `apps/android/app/src/main/java/ai/openclaw/app/NodeRuntime.kt:2739`
  - body: Passing the stored operator token as `GatewayConnectAuth.token` makes `GatewaySession` classify it like an explicit shared token instead of the stored device-token path, so stored scopes and device-token retry behavior can be lost for operator sessions. Preserve the current no-explicit-token call shape and let `GatewaySession` load the stored role token.
  - confidence: 0.93
- **[P1] Do not mark unavailable providers as ready:** `apps/android/app/src/main/java/ai/openclaw/app/ui/ProvidersModelsScreen.kt:312-313`
  - body: `unconfigured` and `unreachable` are setup/connectivity failure states, but this change counts them as ready. That hides provider setup failures in `readyProviderCount` and the providers UI instead of surfacing the existing attention state.
  - confidence: 0.9
- **[P1] Keep Android version metadata monotonic:** `apps/android/app/build.gradle.kts:68-69`
  - body: Current main is already at `versionCode = 2026060201` and `versionName = "2026.6.2"`; this branch lowers both to `20260602` and `2026.6.1`. Android upgrades and release uploads require the versionCode to keep increasing, so this can block users from upgrading current-main builds.
  - confidence: 0.95
