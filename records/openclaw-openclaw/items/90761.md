Full review comments:

- **[P1] Verify SSH host keys before forwarding:** `apps/android/app/src/main/java/ai/openclaw/app/gateway/SshTunnelManager.kt:119`
  - body: This disables JSch host-key verification for the new tunnel. Current OpenClaw SSH tunnel code and docs require strict host-key checking, and JSch only rejects unknown/changed hosts on the yes/ask paths; with `no`, the Android app can forward gateway traffic through an unverified SSH server.
  - confidence: 0.95
- **[P1] Keep stored device tokens on the device-token path:** `apps/android/app/src/main/java/ai/openclaw/app/NodeRuntime.kt:2738`
  - body: Returning the stored operator token as `token` makes `GatewaySession` treat it as an explicit shared token instead of loading it from `DeviceAuthStore` as a stored device token. That bypasses the current auth-source/scopes contract covered by `GatewayBootstrapAuthTest` and can break operator reconnects after bootstrap handoff.
  - confidence: 0.9
- **[P1] Do not mark unavailable providers as ready:** `apps/android/app/src/main/java/ai/openclaw/app/ui/ProvidersModelsScreen.kt:312-313`
  - body: `unconfigured` and `unreachable` are not usable provider states, but this adds them to the ready set. Together with the home-screen `modelsCount == 0` guard, an unavailable catalog can hide the existing No ready providers attention row even when no provider can actually serve a model.
  - confidence: 0.88
- **[P1] Keep Android version metadata monotonic:** `apps/android/app/build.gradle.kts:68-69`
  - body: This downgrades the app from current main's `versionCode = 2026060201` and `versionName = "2026.6.2"` to `20260602` / `2026.6.1`. Android uses versionCode to prevent downgrades, so this can block upgrades and Play uploads from current builds.
