Full review comments:

- **[P1] Reject changed SSH host keys before forwarding:** `apps/android/app/src/main/java/ai/openclaw/app/gateway/SshTunnelManager.kt:157`
  - body: With `StrictHostKeyChecking` set to `ask`, JSch calls `promptYesNo` when a known host key changes; this implementation always returns true, so a changed SSH host key can still establish the tunnel and forward gateway traffic. Return false for changed keys or fail closed until Android shows an explicit host-key change approval flow.
  - confidence: 0.93
- **[P1] Keep stored operator tokens on the device-token path:** `apps/android/app/src/main/java/ai/openclaw/app/NodeRuntime.kt:2770`
  - body: Passing `storedToken` as `token` turns the stored operator device token into an explicit token; `GatewaySession` treats explicit tokens as shared-token auth, while current main intentionally lets the session select its stored token as `DEVICE_TOKEN`. This can change existing Android operator reconnect behavior and should preserve the current null-token handoff.
  - confidence: 0.91
- **[P2] Preserve unclosed reasoning-only recovery:** `src/shared/text/reasoning-tags.ts:133`
  - body: Current tests expect a fully wrapped but unclosed reasoning tag to recover the visible answer text; changing this branch to return an empty string will regress malformed local-model outputs that otherwise display nothing. Keep the recovery behavior or update the sanitizer contract and tests intentionally.
  - confidence: 0.86
