  - confidence: 0.91

  - body: Current tests expect a fully wrapped but unclosed reasoning tag to recover the visible answer text; changing this branch to return an empty string will regress malformed local-model outputs that otherwise display nothing.   - confidence: 0.86

  - body: The Settings toggle updates `MainViewModel.showThinkingText`, but `ChatMarkdown` creates a separate `SecurePrefs` and remembers the initial boolean, so toggling it while chat is mounted will not reliably show or hide reasoning text. Pass the collected state into chat rendering instead of reading a one-time preference here.
  - confidence: 0.84
