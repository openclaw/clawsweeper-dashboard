  - body: With `StrictHostKeyChecking` set to `ask`, JSch calls `promptYesNo` when a known host key changes; this implementation always returns true, so a changed SSH host key can still establish the tunnel and forward gateway traffic.   - confidence: 0.93
- **[P1] Keep stored operator tokens on the device-token path:** `apps/android/app/src/main/java/ai/openclaw/app/NodeRuntime.kt:2770`
  - body: Passing `storedToken` as `token` turns the stored operator device token into an explicit token; `GatewaySession` treats explicit tokens as shared-token auth, while current main intentionally lets the session select its stored token as `DEVICE_TOKEN`. Preserve the current null-token handoff and keep the existing test contract.
  - confidence: 0.91
- **[P2] Preserve unclosed reasoning-only recovery:** `src/shared/text/reasoning-tags.ts:133`
  - body: Current tests expect a fully wrapped but unclosed reasoning tag to recover the visible answer text; changing this branch to return an empty string will regress malformed local-model outputs that otherwise display nothing. Keep the recovery behavior or update the sanitizer contract and tests intentionally.
  - confidence: 0.86
- **[P2] Observe the thinking toggle from chat state:** `apps/android/app/src/main/java/ai/openclaw/app/ui/chat/ChatMarkdown.kt:110`
  - body: The Settings toggle updates `MainViewModel.showThinkingText`, but `ChatMarkdown` creates a separate `SecurePrefs` and remembers the initial boolean, so toggling it while chat is mounted will not reliably show or hide reasoning text. Pass the collected state into chat rendering instead of reading a one-time preference here.
  - confidence: 0.84

  - body: Current Android connection metadata appends `-dev` for debug builds so gateway logs do not look like release clients. Returning raw `VERSION_NAME` here silently drops that marker from gateway connect metadata; keep the existing suffix or update the contract deliberately.
