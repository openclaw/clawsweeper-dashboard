## Summary

Keep open. Current main still does not implement the requested bounded, metadata-validated skill-loader cache: direct loader paths still scan roots and read/parse `SKILL.md` on each call, while the author-linked PR #59547 is closed and unmerged.

## What This Changes

Request to add a bounded in-memory cache for direct validated skill loads while preserving root containment, symlink safety, and conservative invalidation.

## Best Possible Solution

Add the cache in the skill-loading layer with direct-load-only entries, metadata validation on hits, root/symlink rechecks, scanned-root pruning, bounded process growth, and regression tests for invalidation and safety.

## Reproduction Assessment

Yes for the structural behavior: repeated calls to `loadWorkspaceSkillEntries` or `buildWorkspaceSkillSnapshot` against unchanged roots still execute the synchronous scan/stat/read/parse path on current main. No high-confidence latency reproduction is available because the report does not include benchmark data.

## Solution Assessment

Unclear pending maintainer acceptance of the performance tradeoff, but the proposed direct-load cache is a reasonable narrow approach if it remains bounded and safety-revalidating. Higher-level session snapshots alone are not enough because direct loader entry points still exist.
