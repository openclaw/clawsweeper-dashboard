## Summary

Keep open: this draft, maintainer-labeled PR still has a verified P2 RTT contract regression in the Convex credential path, so it should stay with the author/maintainers before merge.

## What This Changes

The PR rewrites channel, agent, plugin, media, and model-catalog hot paths for lower allocation, adds Slack live QA plus Convex Telegram RTT support, and updates related workflow, tests, docs, and changelog entries.

## Best Possible Solution

Preserve the RTT sampling contract for Convex credential mode, either by adding Convex leasing to the RTT sampling harness or by making the live package runner emit equivalent sample and percentile stats before the broader performance PR proceeds.

## Reproduction Assessment

Yes. Source inspection of the latest PR head shows `--credential-source convex` selects the live Telegram package harness, while the sample loop and p50/p95 result fields live only on the RTT driver path.

## Solution Assessment

No. Routing Convex RTT through the one-shot live package runner is not the narrowest maintainable fix because it drops the existing sampling contract; Convex support should preserve the same RTT output semantics.
