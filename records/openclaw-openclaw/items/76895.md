Keep open: this draft, maintainer-labeled PR still has a concrete correctness blocker in the Convex Telegram RTT path on the latest head, where Convex mode bypasses the sampling harness that produces the expected warm-sample stats.

## What This Changes

The PR trims allocation-heavy channel, agent, plugin, media, and model-catalog paths, adds Slack live QA, adds Convex credential-source support for Telegram RTT, and updates related docs, workflow, tests, and changelog entries.

## Best Possible Solution

Preserve the RTT sampling contract for Convex credential mode by adding Convex lease support to the RTT sampling path or giving the live package runner equivalent sample-loop and stats output before continuing the broader performance review.

## Reproduction Assessment

Yes. Source inspection of the latest head shows `--credential-source convex` selects the live Telegram package harness, while the `--samples`, `--sample-timeout-ms`, warm sample array, and p50/p95 stats are only implemented by the separate RTT driver path.

## Solution Assessment

No. Routing Convex RTT through the one-shot live package runner is not the narrowest maintainable solution because it drops the existing sampling contract; the Convex credential support should live on the RTT sampling path or preserve the same result fields.
