## Summary

Keep open: this is a draft, maintainer-labeled MEMBER PR with XL scope, and current main does not already contain the representative hot-path changes. The sampled diff has one concrete correctness issue in the new Convex RTT route: it selects the live Docker harness, which does not honor the RTT sample-count/sample-timeout contract.

## What This Changes

The PR replaces many channel/core array-transform hot paths with direct loops and adds Convex credential-source support plus docs, changelog, and test updates for Telegram RTT/QA harnesses.

## Best Possible Solution

Keep this as a maintainer-owned draft until the Convex RTT path preserves the RTT sample contract or is explicitly split/renamed, then merge only with exact-head changed-gate and RTT benchmark or live proof.

## Reproduction Assessment

Yes for the review finding by source inspection: `pnpm rtt ... --credential-source convex --samples N` selects the live Docker harness while the warm-sample loop and stats remain in the RTT driver path. No live credentials are needed to see that the CLI sample contract is not preserved in that route.

## Solution Assessment

No. The broad hot-path rewrites may be directionally valid, but the Convex RTT support should either use an RTT driver that can lease credentials or update the live harness/results/docs so the advertised sample contract remains true.
