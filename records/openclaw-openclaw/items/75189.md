## Summary

Keep open. Current main still injects the broad workspace bootstrap set into ordinary sessions, has no `bootstrapTier` or constrained local-model default, and `doctor` only warns for truncation or near configured limits rather than the large-under-budget prompt pressure reported here.

## What This Changes

Requests a leaner default bootstrap context, tier, template trim, or doctor warning so small/mid local models keep structured tool dispatch reliable.

## Best Possible Solution

Ship one maintainer-approved constrained-local-model bootstrap path that reduces ordinary-session prompt pressure, preserves critical guidance, aligns schema/help/docs, and adds targeted doctor guidance.

## Reproduction Assessment

Yes. The issue provides concrete vLLM/local-model steps, prompt-size measurements, an `openclaw agent` command, and JSONL checks for missing versus present `toolCall` events; current main still has the broad ordinary-session bootstrap shape needed to validate it, though I did not live-rerun Hermes/vLLM.

## Solution Assessment

Unclear as a product choice. Lowering only the character caps would mitigate the symptom, but the maintainable solution likely needs coordinated tier/default/template/doctor behavior so local-model users can reduce prompt context without silently losing expected workspace guidance.
