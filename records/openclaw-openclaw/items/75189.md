## Summary

Keep open. Current main has partial prompt-pressure mitigations, but it still lacks the requested constrained bootstrap default, bootstrap tier/profile control, template trim/reordering, or small-model doctor warning for large but under-budget prompts.

## What This Changes

The issue requests reducing default workspace bootstrap prompt pressure so small and mid local models keep reliable structured tool dispatch.

## Best Possible Solution

Ship a maintainer-approved constrained bootstrap path: tier/profile or shorter defaults/templates, tool and safety guidance early in generated files, schema/docs/UI alignment, and doctor guidance for loopback small-model setups with large injected bootstrap context.

## Reproduction Assessment

Yes. The issue provides concrete install/config/model steps, an `openclaw agent` command, prompt-size reports, and session JSONL checks for missing versus present toolCall/toolResult events; current main still has the ordinary-session broad bootstrap path needed to validate that failure shape.

## Solution Assessment

Unclear. Lowering only `bootstrapMaxChars` would mitigate the report, but the maintainable fix likely needs coordinated tier/default/template/doctor behavior so constrained local-model users can shrink prompt context without silently losing expected workspace guidance.
