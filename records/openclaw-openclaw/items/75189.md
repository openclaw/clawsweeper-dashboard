## Summary

Keep open. Current main reduced the historical bootstrap caps and added `localModelLean`, but normal sessions still load the full default bootstrap file set, no `bootstrapTier` exists, doctor only warns near configured truncation budgets, and the related implementation PRs are still unmerged.

## What This Changes

The issue asks OpenClaw to reduce, tier, or warn on large default workspace bootstrap context so small and mid-sized local models keep reliable structured tool dispatch.

## Best Possible Solution

Ship one maintainer-approved constrained-local-model bootstrap path: tiered or lower default context, shorter generated templates with operational rules preserved early, schema/help/docs alignment, and doctor guidance when local-model sessions exceed a practical prompt budget.

## Reproduction Assessment

Yes. The issue provides concrete install/config/model steps, prompt-size measurements, an `openclaw agent` command, and JSONL checks for missing versus present `toolCall`/`toolResult` events; current main still has the ordinary-session bootstrap shape needed to validate it.

## Solution Assessment

Unclear. Lowering only `bootstrapMaxChars` would mitigate the symptom, but the maintainable fix likely needs coordinated tier/default/template/doctor behavior so constrained local-model users can reduce prompt context without silently losing expected workspace guidance.
