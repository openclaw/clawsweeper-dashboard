## Summary

Keep open. Current main has lowered the old reported bootstrap budget, but it still injects the broad workspace bootstrap set for ordinary sessions, has no `bootstrapTier` config, and only warns in doctor when files are near or over configured bootstrap budgets rather than when constrained local models face large prompt pressure.

## What This Changes

The issue reports that default workspace bootstrap size and verbose generated bootstrap files make small and mid local models stop emitting structured tool calls, and requests a lower-budget, tiered, trimmed, or doctor-guided mitigation.

## Best Possible Solution

Ship one coherent constrained-local-model bootstrap path: a maintainer-approved tier/profile or lower-budget default, schema/help/docs/UI alignment, seeded templates that keep operational tool and safety guidance early, and doctor guidance for loopback local models with large injected bootstrap context.

## Reproduction Assessment

Yes. The issue gives concrete setup steps, an `openclaw agent` command, systemPromptReport sizes, and JSONL checks for missing versus present `toolCall`/`toolResult` events; current main still has the ordinary-session full-bootstrap path needed to exercise the same failure shape.

## Solution Assessment

Unclear. Lowering one budget alone is a partial mitigation and can trim important instructions; the maintainable fix likely needs coordinated tier/default/template/doctor behavior, and the open related PRs need maintainer choice before this can be called solved.
