## Summary

Keep open. Current main has partially reduced bootstrap budgets, but the reported small/local-model prompt-pressure problem is not solved: ordinary main sessions still inject the broad bootstrap set, no `bootstrapTier` config exists on main, doctor only warns near configured truncation limits, and related implementation PRs remain open for maintainer choice.

## What This Changes

The issue asks OpenClaw to reduce or tier default workspace bootstrap prompt size, trim generated bootstrap content, or warn local small-model users when prompt pressure can break tool dispatch.

## Best Possible Solution

Ship one coherent constrained-local-model bootstrap path: maintainer-approved tier/profile or lower-budget semantics, schema/help/docs/UI alignment, seeded templates that keep operational tool and safety guidance early, and doctor guidance for loopback local models with large injected bootstrap context.
