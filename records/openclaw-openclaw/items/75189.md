## Summary

Keep open. Current main lowered the runtime bootstrap caps from the reported 20,000/150,000 values to 12,000/60,000, but ordinary sessions still inject the full generated bootstrap set under those caps, there is no `bootstrapTier` config, and doctor still warns only for truncation or near-limit pressure rather than constrained local-model prompt pressure.

## What This Changes

The issue asks OpenClaw to reduce or tier default workspace bootstrap context so small and mid local models keep reliable structured tool dispatch.

## Best Possible Solution

Ship a maintainer-approved constrained-local-model bootstrap path that reduces ordinary-session workspace context, preserves critical operational rules early, updates schema/help/docs, and adds doctor guidance for large local-model prompts.

## Reproduction Assessment

Yes, there is a high-confidence reproduction procedure in the issue: run a tool-warranted prompt against the listed vLLM/Hermes-style local setup, then compare `systemPromptReport` and JSONL `toolCall` events before and after reducing bootstrap context. I did not rerun that live model path, but current main still has the ordinary-session bootstrap shape needed for it.

## Solution Assessment

Unclear. A one-line default reduction might mitigate the symptom, but the best maintainable fix likely needs coordinated tier/default/template/doctor behavior so constrained local-model users can reduce prompt pressure without silently losing expected workspace guidance.
