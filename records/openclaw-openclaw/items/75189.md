## Summary

Keep open. Current main reduced the older release defaults and has an experimental local-model tool-surface trim, but it still does not implement the requested small/mid-model bootstrap-pressure fix: normal sessions load the broad bootstrap set, no `bootstrapTier` config exists, the generated `AGENTS.md` remains large with `Tools` late in the file, and doctor only warns about configured bootstrap budget pressure rather than model-aware prompt pressure.

## What This Changes

The issue requests reducing, tiering, or warning on default workspace bootstrap prompt content so small and mid-sized local models keep reliable structured tool dispatch.

## Best Possible Solution

Define a supported constrained-local-model bootstrap path with schema/help/docs/tests: a maintainer-approved bootstrap profile or tier, clear default behavior for fresh installs, preserved high-priority operational guidance under truncation, and doctor guidance when a configured local model is likely to exceed prompt-pressure thresholds while keeping richer bootstrap context opt-in.

## Reproduction Assessment

Yes. The issue provides concrete setup steps, an `openclaw agent` command, systemPromptReport sizes, and session JSONL checks for missing versus present `toolCall`/`toolResult` events; current main still contains the broad ordinary-session bootstrap path needed to exercise the same failure shape.

## Solution Assessment

Unclear. A lower `bootstrapMaxChars` is a narrow mitigation, but the maintainable fix likely needs coordinated tier/default/doctor/template behavior so constrained local models get a lean path without surprising existing users who rely on full bootstrap context.
