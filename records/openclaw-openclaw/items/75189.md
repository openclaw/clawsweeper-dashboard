## Summary

Keep open: current main and the latest release still use the large default bootstrap budgets, generated bootstrap templates remain large enough to recreate the reported prompt-pressure condition, no `bootstrapTier` exists on main, and doctor still warns only for truncation or near configured limits rather than small-local-model prompt pressure. The closest mitigations are still open maintainer/product-decision work, so this should not be closed as implemented.

## What This Changes

The issue asks OpenClaw to reduce or control verbose default workspace bootstrap prompt content because the current default can degrade tool dispatch on small and mid-size local models.

## Best Possible Solution

A maintainer should choose one coordinated prompt-pressure strategy: leaner generated bootstrap content or defaults, an explicit bootstrap tier if accepted, and doctor guidance for constrained local models, with upgrade-safe tests and documentation.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the source preconditions: current main still uses the reported bootstrap budgets, injects large generated bootstrap files, and exposes the same system-prompt-report diagnostics; this review did not rerun the live small-model vLLM tool-dispatch scenario.

## Solution Assessment

Is this the best way to solve the issue? No single best implementation is confirmed yet: lowering defaults, trimming templates, tiered loading, and doctor warnings are all plausible, but the repo's config/default policy makes the final shape a maintainer product decision.
