## Summary

Keep open. Current main still injects the broad workspace bootstrap set into ordinary sessions, has no `bootstrapTier` config, and doctor still warns on truncation or near configured limits rather than large-under-budget local-model prompt pressure; related tier/template PRs are open but not merged.

## What This Changes

The issue asks OpenClaw to reduce or warn about default workspace bootstrap prompt size for small and mid local models so tool calls are not silently displaced by long system prompts.

## Best Possible Solution

Ship one maintainer-approved constrained-local-model bootstrap path with tiered or lower default context, shorter or prioritized generated templates, aligned schema/help/docs, and doctor guidance for local-model prompt pressure.

## Reproduction Assessment

Yes, with source-backed confidence and reporter-supplied live evidence. The issue gives concrete install/config/model steps, prompt-size measurements, an `openclaw agent` command, and JSONL checks for missing versus present `toolCall` events; this review confirmed current main still has the ordinary-session bootstrap shape needed to validate it.

## Solution Assessment

Unclear. Lowering only `bootstrapMaxChars` would mitigate the symptom, but the narrow maintainable fix likely needs coordinated tier/default/template/doctor behavior so constrained local-model users can reduce prompt context without silently losing expected workspace guidance.
