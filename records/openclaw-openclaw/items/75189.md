## Summary

Keep open. Current main has lowered bootstrap caps and has an experimental lean local-model tool mode, but ordinary main sessions still inject the broad bootstrap file set, there is no `bootstrapTier` config surface on main, the default templates remain large, and doctor only warns when files approach configured truncation budgets rather than when local-model prompt pressure is likely to degrade tool dispatch.

## What This Changes

The issue requests reducing, tiering, or warning on default workspace bootstrap context so small and mid-sized local models keep reliable structured tool dispatch.

## Best Possible Solution

Define one supported local-model prompt-pressure path: a maintainer-approved bootstrap profile or tier, schema/help/docs/tests, clear default behavior for new installs, template priority that preserves operational tool rules under truncation, and doctor guidance for constrained loopback/local models while keeping richer bootstrap context opt-in.

## Reproduction Assessment

Yes. The issue gives concrete setup steps, an `openclaw agent` command, systemPromptReport sizes, gateway/session JSONL checks for `toolCall` and `toolResult`, and a before/after config mitigation; current main still has the broad ordinary-session bootstrap path needed to validate the same failure shape.

## Solution Assessment

Unclear. Lowering `bootstrapMaxChars` alone is a useful mitigation, but the maintainable fix likely needs coordinated tier/default/doctor/template behavior so local-model users get a supported prompt-pressure profile without surprising existing full-context users.
