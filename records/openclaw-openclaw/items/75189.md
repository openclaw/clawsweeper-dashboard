## Summary

Keep open. Current main has partial prompt-pressure mitigations, but ordinary main sessions still inject the broad bootstrap set, there is no tier/profile config, and doctor still only warns on truncation or near-limit bootstrap budgets rather than local-model prompt pressure.

## What This Changes

The issue requests smaller or tiered default bootstrap context, or a model-aware doctor warning, to prevent small and mid-sized local models from silently missing structured tool dispatch under verbose bootstrap prompts.

## Best Possible Solution

Ship one documented constrained-bootstrap path for weaker local models: a tier/profile or lower-budget default where appropriate, generated templates with operational tool and safety guidance early, aligned config/docs/tests, and doctor guidance when local-model runs carry large injected bootstrap context.

## Reproduction Assessment

Yes. The issue provides a concrete install, provider setup, agent command, systemPromptReport sizes, and session JSONL checks for missing versus present toolCall/toolResult events; current main still has the broad normal-session bootstrap path needed to exercise that failure shape.

## Solution Assessment

Unclear. A one-line lower bootstrapMaxChars default would reduce prompt size, but the safer maintainable fix is a coordinated tier/profile plus template and doctor behavior so constrained local-model users can shrink prompt context without losing expected workspace guidance silently.
