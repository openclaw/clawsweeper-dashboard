## Summary

Keep open. Current main lowered the old defaults but still injects the broad generated bootstrap set into ordinary sessions, has no `bootstrapTier` config, and doctor only warns near configured bootstrap budgets rather than local-model prompt pressure; related PRs are open but not landed.

## What This Changes

The issue asks OpenClaw to reduce or profile default workspace bootstrap prompt injection so small and mid-sized local models keep reliable structured tool dispatch.

## Best Possible Solution

Ship one coherent local-model prompt-pressure path: a documented bootstrap tier/profile or equivalent default policy, schema/help/docs/tests, template priority for operational rules, and doctor guidance for constrained loopback/local models while leaving richer bootstrap context opt-in.

## Reproduction Assessment

Yes. The issue gives concrete vLLM setup, an `openclaw agent` command, systemPromptReport sizes, and JSONL checks for missing `toolCall`/`toolResult`; current main still has the broad ordinary-session bootstrap path needed to exercise the same failure shape.

## Solution Assessment

Unclear. A lower cap is a useful mitigation, but the maintainable fix likely needs coordinated tier/default/doctor/template behavior so small-model users get a lean prompt without silently dropping expected workspace context.
