## Summary

Keep open. Current main still injects the broad bootstrap file set into ordinary sessions, lacks a tiered/minimal bootstrap config, and only warns on truncation or near-limit budget pressure rather than the local-model prompt-size failure reported here.

## What This Changes

The issue asks OpenClaw to reduce or warn about default workspace bootstrap prompt size so small/mid local models keep reliable structured tool dispatch.

## Best Possible Solution

Ship one maintainer-approved constrained-local-model bootstrap path that reduces ordinary-session prompt pressure, preserves critical guidance, aligns schema/help/docs, and adds doctor guidance for local-model prompt-size risk.

## Reproduction Assessment

Yes. The issue provides concrete vLLM/local-model steps, prompt-size measurements, an `openclaw agent` command, and JSONL checks for missing versus present `toolCall` events; current main still has the broad ordinary-session bootstrap shape needed to validate it, though I did not live-rerun Hermes/vLLM.

## Solution Assessment

Unclear as a product choice. Lowering only the character caps would mitigate the symptom, but the maintainable solution likely needs coordinated tier/default/template/doctor behavior so local-model users can reduce prompt context without silently losing expected workspace guidance.
