Keep open. Current main partially lowers bootstrap budgets, but ordinary sessions still inject the broad workspace bootstrap set, no `bootstrapTier` config exists on main, doctor remains budget/truncation based, and related implementation PRs are still open for maintainer choice.

## What This Changes

The issue asks OpenClaw to reduce, tier, trim, or warn about verbose default workspace bootstrap prompt context so small and mid-sized local models keep reliable structured tool dispatch.

## Best Possible Solution

Ship one coherent constrained-local-model bootstrap path: maintainer-approved tier/profile or lower-budget semantics, schema/help/docs/UI alignment, seeded templates that keep operational tool and safety guidance early, and doctor guidance for loopback local models with large injected bootstrap context.

Yes. The issue gives concrete setup steps, an `openclaw agent` command, systemPromptReport sizes, and session JSONL checks for missing versus present `toolCall`/`toolResult` events; current main still has the ordinary-session bootstrap path needed to validate the same failure shape.

## Solution Assessment
