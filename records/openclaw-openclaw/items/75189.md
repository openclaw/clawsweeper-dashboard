## Summary

Keep open. Current main still has the prompt-shape conditions behind the report: ordinary sessions inject the full workspace bootstrap set, there is no `bootstrapTier`, and doctor still only warns on truncation or near-limit budget pressure rather than local-model prompt pressure.

## What This Changes

The issue asks OpenClaw to reduce or warn about default workspace bootstrap prompt pressure so small and mid-sized local models keep structured tool dispatch reliable.

## Best Possible Solution

Ship a maintainer-approved constrained-local-model bootstrap path with aligned tier/default/template/doctor behavior, schema/help/docs coverage, and regression tests around ordinary-session bootstrap filtering and local-model prompt-pressure warnings.

## Reproduction Assessment

Yes. The issue provides concrete install/config/model steps, prompt-size measurements, an `openclaw agent` command, and JSONL checks; current-main source still has the full ordinary-session bootstrap shape needed to validate it, though I did not run live vLLM/Hermes here.

## Solution Assessment

Unclear. Lowering only `bootstrapMaxChars` would mitigate one symptom, but the narrow maintainable fix likely needs coordinated tier/default/template/doctor behavior so constrained local-model users reduce prompt context without silently losing expected workspace guidance.
