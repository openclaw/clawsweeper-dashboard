## Summary

Keep open. Current main has reduced the historical default bootstrap cap and added an experimental local-model tool trim, but ordinary sessions still load a broad default bootstrap set, there is no `bootstrapTier` knob, and doctor still warns on configured budget/truncation pressure rather than small-model prompt pressure.

## What This Changes

The issue asks OpenClaw to reduce or tier default bootstrap prompt content so small and mid local models keep reliable structured tool dispatch.

## Best Possible Solution

Ship one maintainer-approved constrained-local-model bootstrap path: tiered or lower default context, shorter generated templates with operational rules preserved early, schema/help/docs alignment, and doctor guidance when local-model sessions exceed a practical prompt budget.

## Reproduction Assessment

Yes. The issue provides concrete install/config/model steps, an `openclaw agent` command, prompt-size measurements, and JSONL checks for missing versus present `toolCall`/`toolResult` events; current main still has the ordinary-session bootstrap shape needed to validate it.

## Solution Assessment

Unclear. Lowering only `bootstrapMaxChars` would mitigate the symptom, but the maintainable fix likely needs coordinated tier/default/template/doctor behavior so constrained local-model users can reduce prompt context without silently losing expected workspace guidance.
