## Summary

Keep open. Current main still injects the broad workspace bootstrap set into ordinary sessions, the defaults remain far above the reported small-model budget, there is no `bootstrapTier` or small-model prompt-pressure doctor warning on main, and the related implementation PRs are still open rather than merged.

## What This Changes

Reduce default workspace/bootstrap prompt pressure for small and mid local models so tool-dispatch instructions remain reliable.

## Best Possible Solution

Ship a maintainer-approved constrained-local-model bootstrap path that reduces ordinary-session prompt pressure, preserves critical tool and safety guidance, aligns schema/help/docs/tests, and coordinates the open tier/template/doctor work.

## Reproduction Assessment

Yes. The issue provides concrete vLLM/Hermes steps, prompt-size measurements, an `openclaw agent` command, and JSONL checks for missing versus present `toolCall` events; current main still has the broad ordinary-session bootstrap shape needed to validate it, though I did not live-rerun vLLM.

## Solution Assessment

Unclear as a product choice. Lowering only the character caps would mitigate the symptom, but the durable fix likely needs coordinated tier/default/template/doctor behavior so local-model users can reduce prompt context without silently losing expected workspace guidance.
