## Summary

Keep open: current main and v2026.6.6 still use the 20,000 per-file and 60,000 total bootstrap budgets, still inject the ordinary-session bootstrap file set from verbose templates, and still lack a merged bootstrapTier or small-local-model bootstrap pressure warning. The related open work at https://github.com/openclaw/openclaw/pull/22439 and https://github.com/openclaw/openclaw/pull/78031 covers pieces of the problem, but nothing has landed that resolves this issue yet.

## What This Changes

The issue asks OpenClaw to reduce or warn about default workspace bootstrap prompt pressure because verbose generated files plus the 20,000-character per-file budget can degrade tool dispatch on small and mid local models.

## Best Possible Solution

Keep this as the focused prompt-size reliability issue until maintainers land or deliberately replace a mitigation that reduces default bootstrap pressure or warns constrained local-model users, with upgrade-safe defaults and local-model proof.

## Reproduction Assessment

Yes, source-reproducible with high confidence: the issue provides concrete vLLM/Hermes commands and session-jsonl evidence, and current main still has the same default budget and ordinary-session bootstrap shape. This review did not run the live small-model setup.

## Solution Assessment

Unclear: existing per-agent bootstrap caps and localModelLean are useful workarounds, but the repository fix still needs maintainer choice between changing defaults, compacting templates, adding tiered loading, and adding a doctor warning.
