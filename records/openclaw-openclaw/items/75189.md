## Summary

Keep open. Current main reduces some prompt pressure with lower bootstrap caps and experimental local-model tool trimming, but ordinary chat sessions still inject the broad bootstrap set, no `bootstrapTier` config exists, and doctor only warns on configured budget/truncation pressure rather than small-model prompt pressure.

## What This Changes

The issue asks for lower or tiered workspace bootstrap prompt defaults and diagnostics so small local models keep reliable structured tool dispatch.

## Best Possible Solution

Ship one maintainer-approved constrained-local-model bootstrap path: documented tier/profile behavior or lower budgets, shorter generated templates with tool/safety guidance early, schema/help/docs alignment, and doctor guidance for loopback local models with oversized injected bootstrap context.

## Reproduction Assessment

Yes. The issue gives concrete install/config/model steps, an `openclaw agent` command, prompt-size measurements, and session JSONL checks for missing versus present `toolCall`/`toolResult` events; current main still has the ordinary-session bootstrap shape needed to validate that failure.

## Solution Assessment

Unclear. Lowering only `bootstrapMaxChars` would be a partial mitigation, while the maintainable fix likely needs coordinated tier/default/template/doctor behavior so constrained local-model users can reduce prompt context without silently losing expected workspace guidance.
