## Summary

Keep open. Current main has partial prompt-pressure mitigations, but normal sessions still inject a broad bootstrap set with 12,000-char per-file and 60,000-char total defaults, and there is still no tier/profile default or local-model doctor warning that resolves the reported small-model tool-dispatch failure.

## What This Changes

The issue asks OpenClaw to reduce or tier default bootstrap context, trim generated bootstrap templates, or warn/fix when small local models receive oversized prompts that degrade tool dispatch.

## Best Possible Solution

Ship one maintainer-approved constrained bootstrap path: a documented tier/profile or lower-budget default for local-model users, shorter generated templates with operational tool guidance early, schema/help/docs alignment, and doctor guidance when loopback local models receive large injected bootstrap context.

## Reproduction Assessment

Yes. The issue provides a concrete install/version/model setup, an openclaw agent command, systemPromptReport sizes, and session JSONL checks for missing versus present toolCall/toolResult events; current main still has the ordinary-session full-bootstrap path needed to validate that failure shape.

## Solution Assessment

Unclear. Lowering one cap would be a narrow mitigation, but the maintainable fix likely needs coordinated tier/default/template/doctor behavior so constrained local-model users can shrink prompt context without silently losing expected workspace guidance.
