## Summary

Keep open. Current main still injects the broad workspace bootstrap set into ordinary sessions, has no `bootstrapTier` config, and doctor still warns on truncation or near configured limits rather than local-model prompt pressure; related tier/template PRs are open but not merged.

## What This Changes

The issue asks OpenClaw to reduce, tier, trim, or warn on large default workspace bootstrap context so small and mid local models keep reliable structured tool dispatch.

## Best Possible Solution

Ship one maintainer-approved constrained-local-model bootstrap path with tiered or lower default context, shorter generated templates that preserve operational rules early, aligned schema/help/docs, and doctor guidance for local-model prompt pressure.

## Reproduction Assessment

Yes. The issue provides concrete install/config/model steps, prompt-size measurements, an `openclaw agent` command, and JSONL checks for missing versus present `toolCall`/`toolResult` events; current main still has the ordinary-session bootstrap shape needed to validate it.

## Solution Assessment

Unclear. Lowering only `bootstrapMaxChars` would mitigate the symptom, but the narrow maintainable fix likely needs coordinated tier/default/template/doctor behavior so constrained local-model users can reduce prompt context without silently losing expected workspace guidance.
