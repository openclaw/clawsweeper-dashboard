## Summary

Keep open. Current main has mitigations, but normal chat sessions still inject the broad bootstrap file set, there is no `bootstrapTier` config on main, and doctor only warns for configured-limit pressure rather than small-model prompt pressure.

## What This Changes

The issue requests a smaller or tiered default bootstrap prompt path, shorter generated bootstrap content, or doctor guidance so small/mid local models keep reliable tool dispatch.

## Best Possible Solution

Ship one maintainer-approved constrained-local-model bootstrap path with aligned tier/default/template/doctor behavior, tests, docs, and changelog coverage.

## Reproduction Assessment

Yes. The issue provides concrete setup, model/backend details, prompt-size measurements, an `openclaw agent` command, and session JSONL checks for missing versus present `toolCall`/`toolResult` events; current main still has the normal-session bootstrap shape needed to validate it.

## Solution Assessment

Unclear. Lowering only `bootstrapMaxChars` would mitigate part of the bug, but the maintainable solution likely needs coordinated tier/default/template/doctor behavior so constrained local-model users can reduce prompt pressure without losing expected workspace guidance.
