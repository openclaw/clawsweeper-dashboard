## Summary

Keep open. Current main has partial mitigations, but normal main sessions still load the broad bootstrap set, the default bootstrap budgets remain far above the reporter's small-model target, there is no `bootstrapTier` config on main, and doctor only warns around configured truncation pressure rather than local-model prompt pressure.

## What This Changes

Request to reduce default bootstrap prompt pressure for small and mid local models through lower caps, trimmed templates, tiered bootstrap loading, or model-aware doctor guidance so structured tool dispatch remains reliable.

## Best Possible Solution

Define one supported constrained-local-model path: maintainer-approved bootstrap tier or profile semantics, schema/help/docs/tests, clear new-install default behavior, template priority that preserves operational tool rules under truncation, and doctor guidance for loopback local models with large prompt context.

## Reproduction Assessment

Yes. The issue gives concrete setup steps, an `openclaw agent` command, systemPromptReport sizes, and session JSONL checks for `toolCall`/`toolResult`; current main still has the broad ordinary-session bootstrap path needed to validate the same failure shape.

## Solution Assessment

Unclear. Lowering `bootstrapMaxChars` alone is a useful mitigation, but the maintainable fix likely needs coordinated tier/default/doctor/template behavior rather than one standalone cap change.
