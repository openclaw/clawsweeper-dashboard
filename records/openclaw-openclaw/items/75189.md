## Summary

Keep open. Current main has partial prompt-budget mitigation, but the reported small-model reliability problem is not resolved: ordinary sessions still load the broad bootstrap set, the default caps remain far above the requested constrained-model target, there is no `bootstrapTier` config on main, and doctor only warns around configured bootstrap budget pressure.

## What This Changes

The issue requests reducing default workspace bootstrap prompt pressure for small and mid local models by lowering bootstrap caps, trimming templates, adding tiered bootstrap loading, or adding model-aware doctor guidance.

## Best Possible Solution

Define one supported constrained-local-model path: maintainer-approved bootstrap tier/profile semantics, schema/help/docs/tests, clear default or doctor-recommended behavior for loopback local models, template ordering that preserves operational tool and safety rules under truncation, and focused regression coverage around ordinary-session bootstrap selection and doctor guidance.

## Reproduction Assessment

Yes. The issue provides concrete setup, command, prompt-size report, and JSONL checks for missing `toolCall`/`toolResult` events; current main still has the broad ordinary-session bootstrap path needed to exercise the same failure shape.

## Solution Assessment

Unclear. Lowering `bootstrapMaxChars` alone would reduce prompt pressure but may also truncate load-bearing rules; the safer maintainable fix likely combines tier/default/doctor/template behavior rather than a standalone cap change.
