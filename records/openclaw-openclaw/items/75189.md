## Summary

Keep open. Current main has partial mitigations for local-model prompt pressure, but it still does not implement a constrained bootstrap default, bootstrap tier/profile selection, shortened default templates, or a local-model-aware doctor warning for large but under-budget bootstrap context.

## What This Changes

The issue asks OpenClaw to reduce default workspace bootstrap prompt pressure for small and mid local models by lowering bootstrap budgets, trimming generated bootstrap content, adding tier/profile control, or warning through doctor.

## Best Possible Solution

Ship one maintainer-approved constrained-local-model bootstrap path: documented tier/profile semantics or lower-budget defaults, shorter generated templates with operational tool guidance early, schema/help/docs alignment, and doctor guidance when loopback local-model runs carry a large injected bootstrap context.

## Reproduction Assessment

Yes. The report gives a concrete OpenClaw version, local vLLM/model setup, an `openclaw agent` command, prompt-size metadata, and session JSONL evidence for missing versus present toolCall/toolResult events; current main still has the normal-session broad bootstrap path needed to validate that failure shape.

## Solution Assessment

Unclear. Lowering only `bootstrapMaxChars` would reduce prompt pressure, but the maintainable fix likely needs coordinated tier/profile/default/template/doctor behavior so constrained local-model users can shrink context without unexpectedly losing important workspace guidance.
