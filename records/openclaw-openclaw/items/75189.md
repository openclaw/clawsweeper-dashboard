## Summary

Keep open. Current main has partial prompt-pressure mitigations, including lower bootstrap caps, optional-file skipping, and `localModelLean`, but ordinary main sessions still load the broad bootstrap set by default, the generated templates remain large, `bootstrapTier` is absent, and doctor only warns near configured bootstrap budgets rather than local-model prompt pressure.

## What This Changes

Request to reduce default/generated bootstrap prompt pressure so small and mid local models keep reliable structured tool dispatch instead of hallucinating tool use.

## Best Possible Solution

Define one supported constrained-local-model bootstrap path with maintainer-approved tier/default semantics, schema/help/docs/tests, template priority that preserves operational rules under truncation, and doctor guidance for loopback local models with large bootstrap context while keeping richer bootstrap content opt-in.

## Reproduction Assessment

Yes. The issue provides concrete install/config steps, an `openclaw agent` command, systemPromptReport sizes, and session JSONL checks for `toolCall`/`toolResult`; current main still contains the ordinary-session full-bootstrap path needed to validate the same failure shape.

## Solution Assessment

Unclear. Lowering `bootstrapMaxChars` alone is a plausible mitigation, but the maintainable fix likely needs coordinated tier, default, doctor, and template behavior rather than a standalone cap change.
