## Summary

Keep open. Current main has lowered bootstrap caps and an experimental local-model tool-surface trim, but the reported prompt-pressure problem is not resolved: normal main sessions still load the broad bootstrap set, the generated templates remain large, there is no `bootstrapTier` config, and doctor only warns on configured budget pressure rather than local-model prompt pressure.

## What This Changes

The issue asks OpenClaw to reduce or profile default workspace bootstrap prompt injection so small and mid local models keep reliable structured tool dispatch.

## Best Possible Solution

Define one supported local-model prompt-pressure path: maintainer-approved bootstrap profile or tier semantics, schema/help/docs/tests, clear default behavior for new installs, template priority that preserves operational rules under truncation, and doctor guidance for constrained loopback/local models while leaving richer bootstrap context opt-in.

## Reproduction Assessment

Yes. The issue gives concrete setup steps, an `openclaw agent` command, systemPromptReport sizes, and session JSONL checks for `toolCall`/`toolResult`; current main still contains the broad ordinary-session bootstrap path needed to validate the same failure shape.

## Solution Assessment

Unclear. A lower `bootstrapMaxChars` is a useful mitigation, but the maintainable fix likely needs coordinated tier/default/doctor/template behavior rather than one standalone cap change.
