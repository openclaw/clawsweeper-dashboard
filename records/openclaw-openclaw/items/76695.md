## Summary

Keep open. Current main still has the reported Skills dialog lifecycle bug: the shared helper calls `showModal()` directly from a Lit ref callback without an `isConnected` guard or fallback.

## What This Changes

The issue requests that the Control UI Skills detail dialog defer or guard `showModal()` so skill details open reliably instead of throwing when the dialog is not connected.

## Best Possible Solution

Update the shared Skills dialog opener to wait until the dialog is connected or fall back safely, then add regression coverage for the disconnected `showModal()` lifecycle.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: setting `detailKey` or `clawhubDetailSlug` renders a dialog whose Lit ref calls `showModal()` immediately. A live browser run was not performed in this read-only review.

## Solution Assessment

Yes, the narrow maintainable repair is in `ui/src/ui/views/skills.ts`: make `showDialogWhenClosed` connection-aware and cover that lifecycle in `ui/src/ui/views/skills.test.ts`.
