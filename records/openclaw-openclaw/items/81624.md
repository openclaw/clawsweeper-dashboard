## Summary

Keep open: this is a fresh, detailed Feishu WebSocket multi-account group-reply bug report against v2026.5.7, and current main still routes the behavior through the same Feishu account-specific monitor and reply-dispatch paths without an evident merged fix or canonical duplicate.

## What This Changes

All configured Feishu WebSocket accounts should send group replies when their own bot is mentioned instead of only one account completing final delivery.

## Best Possible Solution

Fix the Feishu plugin path so a mentioned group event uses the receiving account all the way through routing, reply dispatch, and outbound send, with regression coverage for two or more WebSocket accounts in the same group.

## Reproduction Assessment

No high-confidence current-main reproduction was run. The issue provides detailed live logs and the source path maps cleanly to Feishu WebSocket multi-account dispatch, but proving the failure needs real Feishu apps/bots in a shared group or a new focused regression test.

## Solution Assessment
