## Summary

Current main and v2026.5.7 already solve the observable stale auth-profile problem by re-resolving and persisting a replacement when the current session auth profile is unusable; this PR's finalAuthProfileId approach is now a stale and risky re-port that drops current auth/runtime safeguards.

## What This Changes

The PR propagates a final auth-profile id from embedded runs and persists it to the session auth-profile override after successful fallback rotation.

## Best Possible Solution

Keep the shipped session override re-resolution behavior and avoid merging the stale finalAuthProfileId branch that weakens current command and runtime-auth paths.

## Reproduction Assessment

Yes, at source level. Current main covers the stale-profile path with a regression test that resolves a cooldowned session authProfileOverride to a healthy secondary profile and persists it.

## Solution Assessment

No for this PR. The current-main shipped resolver fix is the narrower maintainable solution; the PR branch reintroduces stale command and runner code while trying to add a broader result contract.
