## Summary

Keep open. The requested auth-profile persistence is still useful and not present on current main, but this branch is not mergeable as-is and the diff regresses command ingress/model override and provider runtime-auth behavior.

## What This Changes

The PR adds `finalAuthProfileId` to embedded run results and rewrites command/runner code and tests to persist the winning fallback auth profile into the session override while preserving a user-pinned override source.

## Best Possible Solution

Rework the intended fix as a narrow current-main patch: add a final winning auth-profile result to the embedded runner, persist it through the existing `src/agents/agent-command.ts` session update path, keep the `allowModelOverride` ingress contract intact, and preserve the current runtime-auth controller and plugin hooks.

## Reproduction Assessment

Yes. A focused typecheck or command test run should reproduce the missing `allowModelOverride` field in the PR test, and static inspection reproduces the stale diff by comparing the PR branch against current-main ingress/runtime-auth code.

## Solution Assessment

No. The product direction is reasonable, but this branch rewrites stale command and runner implementations; the safer solution is a small rebase/replacement on current main that keeps existing security and runtime-auth seams.
