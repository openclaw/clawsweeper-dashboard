## Summary

Keep open: the auth-profile persistence goal is plausible, but this branch is not mergeable because it forks a stale command implementation, removes the model-override authorization guard, drops generic provider runtime-auth handling, and lacks real after-fix behavior proof.

## What This Changes

The PR propagates a final successful auth profile from embedded runs and persists it back to the session auth-profile override after fallback rotation.

## Best Possible Solution

Re-apply the final-auth-profile persistence in the current `src/agents/agent-command.ts` and embedded-runner/auth-controller structure, preserving ingress authorization and generic provider runtime-auth behavior, then add focused regression coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes for the PR blockers: source inspection shows the branch removes the current `allowModelOverride` ingress/run guard while the type contract still requires it. The underlying fallback-persistence bug is source-plausible, but I did not run live tests because this was a read-only review.

## Solution Assessment

No. Persisting the winning auth profile is the right direction, but this branch solves it by forking stale command and runner code instead of patching the current shared implementation with the existing auth and runtime contracts intact.
