## Summary

Keep open, but do not merge this branch as-is. The underlying auth-profile persistence fix still appears useful because current main has no `finalAuthProfileId` result channel, but the PR head is dirty and rewrites current command ingress/runtime-auth code in ways that drop existing authorization and provider-auth behavior.

## What This Changes

The PR adds `finalAuthProfileId` to embedded run result types, persists the winning fallback auth profile into session overrides, changes embedded runner auth handling, replaces the `src/commands/agent.ts` re-export with a full command implementation, and adds focused command tests.

## Best Possible Solution

Rework the idea as a narrow current-main patch: propagate the final successful auth profile from the embedded runner or attempt-execution layer, persist it into `authProfileOverride` while preserving user-pinned source semantics, and leave the current `allowModelOverride` ingress boundary and generic provider runtime-auth controller intact.

## Reproduction Assessment

Yes. Static inspection reproduces the blockers: the PR test omits a still-required `allowModelOverride` field, and the PR branch removes current-main ingress and runtime-auth contracts. I did not run tests because this review was required to stay read-only.

## Solution Assessment

No. The product direction is reasonable, but this branch is not the safest implementation because it carries a stale wholesale command rewrite and runtime-auth changes. A small patch on current main is the maintainable route.
