## Summary

Keep this PR open because current main still lacks the final-auth-profile persistence path, but the branch is not mergeable as-is: it replays stale command and runner implementations that weaken ingress model-override authorization, drop trusted one-off model overrides, and bypass current runtime-auth wrapping.

The PR adds a final auth-profile id to embedded-run results, persists that winner into the session auth override after fallback, and adds regression coverage for fallback winner/source preservation.

## Best Possible Solution

Port only the final winning auth-profile id through the current embedded-runner result and session-store contracts, preserving current authorization/runtime-auth behavior, then add focused regression coverage and redacted real behavior proof.

Yes, at source level: current main tracks lastProfileId and marks it successful, but neither EmbeddedPiRunResult nor updateSessionStoreAfterAgentRun has a final-profile field/write. I did not run a live fallback because this review is read-only.

## Solution Assessment
