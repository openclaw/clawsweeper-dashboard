## Summary

Keep open: the fallback auth-profile persistence bug is still source-reproducible on current main, but this PR is not safe to merge because it ports the fix through a stale command implementation, drops current override authorization/runtime-auth behavior, has failing checks, and still lacks contributor real behavior proof.

## What This Changes

The PR adds `finalAuthProfileId` to embedded run result types, tries to persist it into session `authProfileOverride`, and updates command tests around fallback auth-profile persistence.

## Best Possible Solution

Port final auth-profile persistence into the current `src/agents/agent-command.ts`, `src/agents/command/session-store.ts`, and embedded-runner result contracts while preserving existing model override and runtime-auth behavior, then add focused regression coverage and redacted real behavior proof.

## Reproduction Assessment

Yes, source-reproducible: current main tracks and marks `lastProfileId` successful in the embedded runner, but does not return that id or write it into the session `authProfileOverride`. I did not run a live fallback because this review is read-only.

## Solution Assessment

No. The intended fix direction is right, but replacing `src/commands/agent.ts` with a stale full command implementation is not the narrowest maintainable solution on current main.
