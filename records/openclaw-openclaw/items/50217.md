## What This Changes

The PR adds `finalAuthProfileId` to embedded run results and writes that winning profile back to the session auth-profile override after fallback rotation.

## Best Possible Solution

Port only the winning auth-profile propagation and session override write onto current main, preserving model-override authorization, explicit override application, provider runtime-auth wrapping, and focused regression coverage.

## Reproduction Assessment

Yes, at source level: current main tracks an active auth profile during embedded auth rotation but does not expose it in the run result or persist it through the session-store update path. I did not perform a live multi-profile fallback run in this read-only review.

## Solution Assessment

No. The intended persistence fix is the right direction, but this branch is stale and drops current auth, model override, and runtime-auth contracts; a narrow current-main port is the maintainable solution.
