## Summary

Keep open: the underlying auth fallback persistence bug still appears valid on current main, but this PR is not safe to merge because it carries a stale command implementation, drops current override/auth boundaries, has type/CI failures, and lacks required real behavior proof.

## What This Changes

The branch adds `finalAuthProfileId` to embedded run results and rewrites `src/commands/agent.ts` from a re-export into an in-file command implementation that persists that field into session auth overrides.

## Best Possible Solution

Port the final-profile persistence into the current `src/agents/agent-command.ts` and `src/agents/command/session-store.ts` architecture, preserve existing model override/runtime auth behavior, add focused regression coverage, and attach redacted live or terminal proof.

## Reproduction Assessment

Yes, source-reproducible: current main marks the winning `lastProfileId` successful but neither returns it from the embedded runner nor writes it into the session store. I did not run a live fallback because this review is read-only.

## Solution Assessment

No. The intended fix direction is right, but replacing `src/commands/agent.ts` with a stale full command implementation is not the maintainable way to land it on current main.
