## Summary

Keep open: current main still lacks final auth-profile propagation into session persistence, but this PR is not safe to merge as-is because it replays stale command and runner implementations that regress authorization and runtime-auth behavior, and the external real-behavior proof gate is still missing.

## What This Changes

The PR propagates a final successful auth profile from embedded runs and persists it into the session auth override after fallback rotation.

## Best Possible Solution

Port only the winning auth-profile id through the current embedded-runner result and session-store contracts, preserving current authorization and runtime-auth behavior, then add focused regression coverage and redacted real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main tracks lastProfileId and marks it successful, but does not expose it on EmbeddedPiRunResult or write it to session authProfileOverride after the run.

## Solution Assessment

No. The intended fix direction is right, but replacing current command and embedded-runner logic with stale full implementations is not the narrowest maintainable solution; a small port through current contracts is safer.
