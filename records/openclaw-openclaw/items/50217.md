## Summary

Keep open: current main still lacks the requested final-profile persistence behavior, but this PR should not merge as-is because it carries a broad stale command rewrite that drops current ingress and runtime-auth contracts.

## What This Changes

The PR adds final-auth-profile result plumbing and session override persistence after successful auth-profile fallback, with tests for rotated-profile and user-pinned source handling.

## Best Possible Solution

Rework the idea as a narrow current-main patch that returns the successful final auth profile from the embedded runner or attempt layer, persists it to `authProfileOverride` while preserving user-pinned source semantics, and leaves existing ingress authorization and provider runtime-auth contracts intact.

  - body: The runner removes the generic `prepareProviderRuntimeAuth` flow and replaces it with provider-specific token handling. Providers or plugins that implement `prepareRuntimeAuth`, including SDK/IMDS-style paths covered on current main, would stop receiving runtime credentials.
  - confidence: 0.84

  - body: `AgentCommandIngressOpts` still requires `allowModelOverride: boolean`, but this new test call omits it. The branch should pass the flag or intentionally change the type and runtime boundary in the same patch.
