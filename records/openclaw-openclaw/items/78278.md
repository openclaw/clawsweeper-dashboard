## Summary

Keep open: this protected-label PR targets a real Telegram native-command gap, but the patch has a default-resolution regression and still lacks after-fix real behavior proof.

## What This Changes

The PR prefixes Telegram’s native `/think` picker with the current thinking level, threads session/default thinking state into that title, updates tests, and adds a changelog entry.

## Best Possible Solution

Keep the PR open and align the picker display with the same thinking-default resolver contract used by gateway session rows, with a per-model regression test and real Telegram/native-command proof.

## Reproduction Assessment

Yes, at source level: current main formats the Telegram `/think` picker without current-level text even though `/think` without an argument is documented to show the current level. I did not run a live Telegram bot in this read-only review.

## Solution Assessment

No. The UI direction is right, but the helper should preserve the existing per-agent/per-model/global resolver ordering and the PR needs after-fix real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Respect per-model thinking defaults:** `extensions/telegram/src/bot-native-commands.ts:271-275`
  - body: This returns `agents.defaults.thinkingDefault` before `resolveThinkingDefault()` can apply the existing per-model-over-global contract. In configs with both a global default and `agents.defaults.models[provider/model].params.thinking`, Telegram will display the wrong current level compared with gateway session rows, so add that case and let the shared resolver own the ordering.
  - confidence: 0.89
