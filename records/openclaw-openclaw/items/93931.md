Keep open: current main still leaves the exact generic `LLM request failed.` wrapper unclassified for failover, and two open linked PRs claim the fix path. The issue should remain open until maintainers land one canonical classifier fix or reject that direction.

## What This Changes

## Best Possible Solution

Land one narrow shared classifier fix with regression coverage, preferably the exact-match path in https://github.com/openclaw/openclaw/pull/94062, then close this issue and duplicate fix attempts.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main has the exact `LLM request failed.` wrapper, lacks a classifier for that exact string, and cron forwards `payload.fallbacks` into the model fallback runner. I did not run a live LM Studio cron job in this read-only review.

## Solution Assessment

Yes, the best fix direction is a narrow shared classifier plus regression test for the exact bare wrapper. The suggested configurable error-string allowlist would add unnecessary config surface, and a broad substring match risks classifying permanent format errors as transient.
