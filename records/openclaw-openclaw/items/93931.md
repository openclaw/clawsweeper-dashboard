## Summary

Keep open: current `main` still leaves the exact generic `LLM request failed.` text unclassified for failover, while cron docs promise configured model fallbacks should apply when a selected job model fails. Three open PRs already claim this issue as their closing target, so the issue should remain open until one canonical fix lands.

## What This Changes

The issue asks OpenClaw to engage configured cron `payload.fallbacks` when a local provider primary model fails with the generic `LLM request failed.` error.

## Best Possible Solution

Land one narrow fix that classifies the exact generic local-provider availability wrapper as transient for retry and model fallback, preserves permanent schema-rejection variants, and adds focused regression coverage.

## Reproduction Assessment

Yes. The reporter provided concrete LM Studio cron steps and run-log evidence, and current source shows the exact generic wrapper is not classified into the failover path; I did not run a live LM Studio instance in this read-only review.

## Solution Assessment

Yes, the narrow maintainable direction is a classifier and regression-test fix, not a new user allowlist config. Among the open candidates, the best shape should cover the bare generic wrapper without treating schema-rejection variants as transient.
