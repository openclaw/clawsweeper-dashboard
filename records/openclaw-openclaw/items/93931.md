## Summary

Keep open: current `main` and the latest release still lack a transient classification for the bare `LLM request failed.` wrapper, and the issue is already linked by open closing PRs. The best cleanup is to keep this canonical report open while maintainers review the exact-match fix in https://github.com/openclaw/openclaw/pull/94062 and decide what to do with the broader duplicate fix in https://github.com/openclaw/openclaw/pull/94011.

## What This Changes

Cron jobs configured with `payload.fallbacks` are reported to skip fallback models when a local-provider primary fails with the generic `LLM request failed.` error.

## Best Possible Solution

Land one narrow exact-match classifier and regression test for the bare generic wrapper, preferably through https://github.com/openclaw/openclaw/pull/94062, then let the merged PR close this issue.

## Reproduction Assessment

Yes from source and issue evidence, though not from a live LM Studio cron run. Current main defines and returns the exact generic wrapper, but the shared classifier does not map it to a transient failover reason.

## Solution Assessment

Yes for the linked exact-match PR path. It fixes the bare wrapper in the shared classifier without adding config surface or catching colon-suffixed schema/format variants.
