## Summary

Keep open: current main has backend and SDK access to pending exec approval records, but the requested `openclaw approvals pending` CLI command, CLI tests, and CLI docs are still missing; the related implementation PRs were closed unmerged.

## What This Changes

The issue requests a CLI command like `openclaw approvals pending` to list pending exec approval requests with command details, table output, JSON output, and optional watch mode.

## Best Possible Solution

Add `openclaw approvals pending` in the existing approvals CLI using `exec.approval.list`, with table output, `--json`, focused CLI tests, docs, and a changelog entry; defer `--watch` unless maintainers explicitly include it.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source and docs inspection give a high-confidence gap: the CLI only registers `get`, `set`, and `allowlist`, while the backend list RPC already exists.

## Solution Assessment

Yes. The narrowest maintainable solution is CLI-only reuse of `exec.approval.list` under the existing `operator.approvals` scope, with sanitized display and `--watch` deferred unless explicitly scoped.
