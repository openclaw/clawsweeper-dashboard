## Summary

Keep open. Current main has the backend `exec.approval.list` RPC and security-scoped pending-record handling, but it still does not expose the requested `openclaw approvals pending` CLI command, CLI coverage, or CLI docs.

## What This Changes

Request to add `openclaw approvals pending` with table, JSON, and watch modes for inspecting pending exec approval requests and command details.

## Best Possible Solution

Add a focused one-shot `openclaw approvals pending` table/JSON command backed by the existing `exec.approval.list` RPC, with CLI tests and docs; defer `--watch` and dashboard behavior until scoped.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main shows the backend listing RPC exists while the requested CLI/docs/tests surface is still absent.

## Solution Assessment

Yes for the narrow first pass: reuse `exec.approval.list` from `src/cli/exec-approvals-cli.ts` rather than adding a second pending-approval protocol method. `--watch` should remain a follow-up unless maintainers explicitly include live update semantics.
