## Summary

Current main has the authorized backend API for enumerating pending exec approvals but still lacks the requested CLI surface. Keep this issue open while the paired implementation at https://github.com/openclaw/openclaw/pull/83440 is repaired and reviewed; that PR is open, maintainer-labeled, and currently conflicting, so it cannot yet establish an implemented or safely superseded outcome.

## What This Changes

Add an `openclaw approvals pending` command that shows pending exec approval details and supports scriptable JSON output, with watch mode proposed as an optional extension.

## Best Possible Solution

Land a focused `approvals pending` command backed by the existing `exec.approval.list` method, with safe table and JSON output; scope approval resolution and watch behavior separately unless maintainers explicitly approve them as part of the same CLI contract.

## Reproduction Assessment

Yes, at source level: current main implements `exec.approval.list`, while `registerExecApprovalsCli` has no pending/list runtime subcommand, tests, or documentation. No runtime execution was needed to establish that product-surface gap.

## Solution Assessment

Yes for a one-shot list command that reuses `exec.approval.list`; adding another protocol method would duplicate an existing capability. The linked PR's additional resolve aliases and the proposed watch mode require explicit security and product-scope decisions.
