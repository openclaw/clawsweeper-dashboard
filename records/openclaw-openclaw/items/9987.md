## Summary

Current main and v2026.6.5 still lack a pending-approvals CLI command, while the gateway already provides the required list operation. The open linked implementation PR now owns this work, but its expanded list-and-resolve scope and protected maintainer/security signals require maintainer review before the issue can close.

## What This Changes

Add an `openclaw approvals pending` CLI command with human-readable command details and machine-readable output, with optional live updates.

## Best Possible Solution

Ship one canonical CLI client over `exec.approval.list` with concise table and JSON output, preserve existing approval visibility and authorization, and include resolution or watch behavior only after explicit maintainer scoping.

## Reproduction Assessment

Yes, at source level: current main has `exec.approval.list`, but the approvals CLI, tests, and docs have no pending/list runtime command. This is a clearly verifiable capability gap rather than a runtime bug reproduction.

## Solution Assessment

Reusing `exec.approval.list` is the best and narrowest implementation boundary. The proposed `--watch` mode and the linked PR's approval-resolution commands require separate product and security-boundary judgment.
