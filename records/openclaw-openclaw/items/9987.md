## What This Changes

Request to add an `openclaw approvals pending` CLI command that lists pending exec approval requests with command details and scripting-friendly output.

## Best Possible Solution

Land a narrow CLI implementation that reuses the existing `exec.approval.list` backend through the correct approval-runtime authority path, provides sanitized table and JSON output, adds focused CLI/docs coverage, and treats `--watch` as follow-up unless maintainers explicitly include it.

## Reproduction Assessment

Yes. Source inspection of current main shows the approvals CLI only registers policy-oriented `get`, `set`, and `allowlist` commands while the backend `exec.approval.list` RPC exists.

## Solution Assessment

Yes for the narrow direction, but not enough is implemented to close. The maintainable solution is a CLI wrapper over the existing approval list RPC with authorization, sanitization, tests, docs, and live proof handled in an implementation PR.
