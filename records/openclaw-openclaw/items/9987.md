## Summary

Keep open: current main and v2026.6.10 still expose the backend pending-approval list RPC but not the requested `openclaw approvals pending` CLI/docs surface, and the related implementation PRs are unmerged or partial candidates.

## What This Changes

Add a CLI command such as `openclaw approvals pending` to list pending exec approval requests with command details, JSON output, and possible watch mode.

## Best Possible Solution

Land a narrow CLI implementation that reuses `exec.approval.list`, preserves the existing approval visibility authority, renders sanitized table and JSON output, adds focused CLI/docs coverage, and defers `--watch` unless maintainers scope it in.

## Reproduction Assessment

Yes. Source inspection of current main shows the approvals CLI only registers policy-oriented `get`, `set`, and `allowlist` commands while the backend `exec.approval.list` RPC already exists.

## Solution Assessment

Yes for the narrow direction, but it is not implemented on main. The maintainable path is a CLI wrapper over the existing RPC with authorization, sanitization, tests, docs, and live proof rather than a new protocol method.
