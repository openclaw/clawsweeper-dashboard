## Summary

Keep open. Current main and the latest release still have the backend `exec.approval.list` RPC, but they do not expose the requested `openclaw approvals pending` CLI command, CLI coverage, or approvals docs; the related implementation PRs were closed unmerged and the broader Web dashboard issue does not supersede this narrower CLI request.

## What This Changes

Request a new `openclaw approvals pending` CLI command to list pending exec approval requests with command details plus JSON/watch-oriented output.

## Best Possible Solution

Add a focused one-shot `openclaw approvals pending` table/JSON command backed by `exec.approval.list`, with CLI tests and docs; defer `--watch` until maintainers define live-update behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main and `v2026.5.12` shows the backend listing RPC exists while the requested CLI/docs/tests surface is still absent.

## Solution Assessment

Partly: the narrow first pass should reuse `exec.approval.list` from the existing approvals CLI instead of adding another pending-approval protocol method. `--watch` should remain a follow-up unless maintainers explicitly include live update semantics.
