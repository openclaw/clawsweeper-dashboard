## Summary

Keep open. Current main still has only the backend `exec.approval.list` RPC for pending exec approvals; the requested `openclaw approvals pending` CLI command, CLI coverage, and approvals CLI docs are still absent, and the prior implementation PRs were closed unmerged.

## What This Changes

Request to add an `openclaw approvals pending` command that lists pending exec approval requests with command details, plus JSON/watch modes.

## Best Possible Solution

Add a focused one-shot `openclaw approvals pending` table/JSON command backed by `exec.approval.list`, with CLI tests and docs; defer `--watch` until maintainers define live-update behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main shows the backend listing RPC exists while the requested CLI/docs/tests surface is still absent.

## Solution Assessment

Yes for the narrow first pass: reuse `exec.approval.list` from the existing approvals CLI instead of adding another pending-approval protocol method. `--watch` should remain a follow-up unless maintainers explicitly include live update semantics.
