## Summary

Keep open. Current main still has the backend `exec.approval.list` API, but the requested user-facing `openclaw approvals pending` CLI command, CLI tests, and CLI docs are absent, and the related implementation PRs are closed unmerged.

## What This Changes

Add an `openclaw approvals pending` CLI command to list live pending exec approval requests with command details, plus JSON and optional watch output.

## Best Possible Solution

Add a focused one-shot `openclaw approvals pending` table/JSON command backed by the existing `exec.approval.list` method, and track `--watch` or Web dashboard recovery separately.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that the backend list API exists but the requested CLI/docs/tests surface is missing on current main.

## Solution Assessment

Yes for the narrow first pass: reuse `exec.approval.list` rather than adding another protocol method. The safer scope is table output plus `--json`, with `--watch` left to a follow-up unless maintainers explicitly include it.
