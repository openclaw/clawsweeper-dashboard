## Summary

Keep this issue open: current main and the latest release expose backend pending-approval listing but still lack the requested CLI surface, and the directly related implementation PR remains open, conflicting, and unmerged.

## What This Changes

Add an `openclaw approvals pending` command that shows pending exec requests and supports machine-readable output, with optional live watching.

## Best Possible Solution

Expose a narrowly authorized, read-only `approvals pending` command over the existing `exec.approval.list` method with table and `--json` output; treat approval resolution and `--watch` as separately scoped follow-ups unless maintainers explicitly approve them.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request, and source inspection conclusively confirms that the requested CLI capability is absent while its backend list API exists.

## Solution Assessment

The narrow list-plus-JSON direction is the best first solution because it reuses the existing gateway contract; bundling mutation aliases or live watching requires additional security and UX decisions.
