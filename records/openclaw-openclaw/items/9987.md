## What This Changes

The issue requests an `openclaw approvals pending` CLI command that lists pending exec approval requests with command details and scriptable output.

## Best Possible Solution

Add a focused one-shot `openclaw approvals pending` or `list` command backed by the existing `exec.approval.list` RPC, preserving the operator approval runtime boundary, with sanitized table output, stable JSON, focused CLI tests, and docs; handle `--watch` and resolution commands only after maintainer/security scoping.

## Solution Assessment

Partly. Reusing existing `exec.approval.list` is the right implementation direction, but the full proposal needs maintainer/security scoping for authority, output exposure, and `--watch`; the open PR should be repaired or replaced before this issue can close.
