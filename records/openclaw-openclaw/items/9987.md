## Summary

Current main provides the authorized gateway API for pending exec approvals but still lacks the requested CLI and documentation; an open protected PR owns the implementation, yet it conflicts with main and broadens the feature into approval mutation, so this issue should remain open for explicit maintainer review.

## What This Changes

Add `openclaw approvals pending` to list active exec approval requests with command details, table and JSON output, and potentially live watching.

## Best Possible Solution

Land a refreshed implementation that reuses `exec.approval.list` for one-shot table and JSON output, preserves display sanitization and `operator.approvals`, defers `--watch`, and includes resolution commands only if maintainers explicitly approve the broader scope.

## Reproduction Assessment

Yes, at high confidence from current-main source: the approvals command registration lacks a pending/list subcommand while `exec.approval.list` already exists behind `operator.approvals`. This is a feature-gap reproduction rather than a failing runtime bug.

## Solution Assessment

Partly. Reusing `exec.approval.list` for table and JSON output is the narrowest maintainable solution, but `--watch` and the linked PR's approval-resolution commands should be separate product and security decisions.
