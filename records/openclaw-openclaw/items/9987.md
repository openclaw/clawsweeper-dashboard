## Summary

Keep open: current main still has no user-facing CLI command or docs/tests for listing pending exec approvals, although the backend `exec.approval.list` RPC exists. An open draft maintainer PR now adds a related `openclaw approvals list --gateway` surface, so the remaining action is maintainer coordination rather than closing or queuing a competing repair.

## What This Changes

Add a CLI command such as `openclaw approvals pending` to list currently pending exec approval requests with command details and scriptable output.

## Best Possible Solution

Land or adapt the open draft related PR’s CLI list surface, or add a focused one-shot command backed by `exec.approval.list`, with table output, `--json`, docs, changelog, and CLI regression coverage; scope `--watch` separately unless maintainers include it deliberately.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence in the current gap: the backend pending-approval list RPC exists, but current main has no corresponding CLI command, CLI tests, or approvals docs entry.

## Solution Assessment

Yes for the implementation direction: reusing `exec.approval.list` is the narrow maintainable path and avoids a duplicate protocol method. The open question is whether maintainers want the active draft PR’s `list --gateway` shape, the requested `pending` spelling, and `--watch` in the same landing slice.
