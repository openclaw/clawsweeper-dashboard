## Summary

Keep open: current main and v2026.6.10 still have the backend pending exec approval list RPC but do not expose the requested `openclaw approvals pending` CLI/docs surface, and the open candidate PRs are not merged or merge-ready.

## What This Changes

Request to add an `openclaw approvals pending` CLI command with human-readable and JSON output for pending exec approval requests, with `--watch` proposed as an option.

## Best Possible Solution

Land a narrow CLI implementation that reuses `exec.approval.list`, uses the established approval-runtime authority path, renders sanitized table and JSON output, adds focused CLI/docs coverage, and defers `--watch` unless maintainers explicitly scope it in.

## Reproduction Assessment

Yes. Source inspection of current main shows the approvals CLI only registers `get`, `set`, and `allowlist`, while the backend `exec.approval.list` RPC already exists.

## Solution Assessment

Yes for the narrow direction, but it is not implemented on main. The maintainable path is a CLI wrapper over the existing RPC with approval-runtime authority, sanitized output, tests, docs, and live proof rather than a new protocol method.
