## Summary

Current main has the authorized backend API and UI support for listing pending exec approvals, but it still lacks the requested user-facing CLI command, CLI coverage, and CLI documentation; the two linked implementation PRs closed unmerged, and no viable open replacement owns the work.

## What This Changes

Add an `openclaw approvals pending` CLI command to show pending exec approval details, including scriptable JSON output and optionally live watch behavior.

## Best Possible Solution

Add a focused one-shot `openclaw approvals pending` command backed by `exec.approval.list`, with sanitized table output and stable `--json`; defer `--watch` until its lifecycle semantics are explicitly approved.

## Reproduction Assessment

Yes, at source level. Current main exposes `exec.approval.list`, while `registerExecApprovalsCli` has no pending-list command and the CLI tests and docs contain no such surface.

## Solution Assessment

Yes for a one-shot table and JSON command that consumes the existing authorized RPC. Adding another gateway method or blocking the initial feature on watch mode would create unnecessary protocol and product complexity.
