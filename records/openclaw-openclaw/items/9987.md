## What This Changes

The issue requests an `openclaw approvals pending` CLI command that lists pending exec approval requests with command details and scriptable output.

## Best Possible Solution

Land a reviewed implementation, likely through https://github.com/openclaw/openclaw/pull/83440 or a successor, that adds one-shot pending/list output backed by `exec.approval.list`, focused CLI tests, and public docs while deferring watch mode unless maintainers explicitly include it.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and release inspection show backend RPC support exists while the requested CLI/docs/tests surface is absent from current main and `v2026.5.12`.

## Solution Assessment

Partly: a one-shot `openclaw approvals pending` command using the existing `exec.approval.list` RPC is the narrow maintainable path. Watch mode and approval-resolution UX need explicit maintainer/security scope decisions.
