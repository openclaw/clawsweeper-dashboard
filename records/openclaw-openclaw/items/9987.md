## Summary

Keep open. Current main still has the backend pending-list RPC, but the `approvals` CLI/docs/tests surface only exposes policy `get`, `set`, and `allowlist`; the previous implementation PRs were closed unmerged, so there is no shipped user-facing pending command yet.

## What This Changes

The issue requests an `openclaw approvals pending` CLI command to list pending exec approvals with command details, JSON output, and optional watch behavior.

## Best Possible Solution

Add a focused one-shot `openclaw approvals pending` table/JSON command backed by `exec.approval.list`, with CLI tests and docs; track `--watch` or the broader Web dashboard separately.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main has backend pending-list plumbing but lacks the requested CLI/docs/tests surface.

## Solution Assessment

Yes for the narrow first pass: reuse `exec.approval.list` and keep command visibility behind `operator.approvals`. Adding a second protocol method or bundling the Web dashboard would be a less focused first step.
