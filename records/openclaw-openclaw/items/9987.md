## Summary

Keep this issue open: current main and the latest release expose the backend `exec.approval.list` RPC, but they still do not expose the requested `openclaw approvals pending` CLI surface. An open candidate PR now targets the same area, but it is not merged and has review/proof blockers, so this issue remains useful as the canonical request.

## What This Changes

Add an `openclaw approvals pending` CLI command to list pending exec approval requests, including command details and machine-readable output.

## Best Possible Solution

Land a narrow CLI implementation that reuses the existing `exec.approval.list` backend through the correct approval-runtime authority path, provides sanitized table and JSON output, adds focused CLI/docs coverage, and treats `--watch` as a follow-up unless maintainers explicitly include it.

## Reproduction Assessment

Yes. Source inspection of current main shows the approvals CLI only registers policy-oriented `get`, `set`, and `allowlist` commands while the backend `exec.approval.list` RPC already exists.

## Solution Assessment

Yes for the narrow direction, but not by adding duplicate protocol surface. The maintainable solution is a CLI wrapper over the existing approval list RPC with authorization, sanitization, tests, docs, and live proof handled in the implementation PR.
