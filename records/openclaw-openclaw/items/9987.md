## What This Changes

The issue requests a CLI command such as `openclaw approvals pending` to list currently pending exec approval requests with command details, JSON output, and optional watch behavior.

## Best Possible Solution

Land a narrow CLI implementation that reuses the existing `exec.approval.list` backend through the correct approval-runtime authority path, provides sanitized table and JSON output, adds focused CLI/docs coverage, and leaves `--watch` as a scoped follow-up unless maintainers explicitly include it.

## Reproduction Assessment

Yes. Source inspection of current main shows the approvals CLI only registers policy-oriented `get`, `set`, and `allowlist` commands while the backend `exec.approval.list` RPC already exists.

## Solution Assessment

Yes for the narrow direction, but it is not implemented yet. The maintainable solution is a CLI wrapper over the existing approval-list plumbing with the correct approval-runtime authority, sanitization, tests, docs, and live proof.
