## Summary

Keep open. Current main and v2026.6.6 have the backend `exec.approval.list` API, but neither exposes the requested `openclaw approvals pending` CLI command, CLI tests, or CLI docs; the exact implementation PRs I found are closed unmerged.

## What This Changes

The issue requests an `openclaw approvals pending` CLI command to list pending exec approval requests with command details, JSON output, and optional watch behavior.

## Best Possible Solution

Add a focused one-shot `openclaw approvals pending` command that reuses `exec.approval.list`, renders sanitized table output plus raw `--json`, updates CLI docs/tests, and leaves `--watch` for an explicit follow-up unless maintainers scope it in.

## Reproduction Assessment

Not applicable as a runtime bug, but the missing feature is source-reproducible: the backend list RPC exists while the CLI registers no pending subcommand. I did not run the CLI because this review is read-only.

## Solution Assessment

Yes for the core shape: reusing `exec.approval.list` is narrower and safer than adding another protocol method or pending store. The product decision is whether `--watch` and any approve/deny actions belong in the first CLI surface.
