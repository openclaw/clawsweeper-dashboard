## Summary

Keep open. Current main and the latest release still lack the requested user-facing `openclaw approvals pending` CLI/docs/tests surface; the backend listing RPC exists, and an open implementation PR now owns the remaining work.

## What This Changes

Add a CLI command such as `openclaw approvals pending` to list active exec approval requests with command details and scriptable output.

## Best Possible Solution

Land a reviewed implementation, likely through https://github.com/openclaw/openclaw/pull/83440 or a successor, that adds one-shot pending/list output backed by `exec.approval.list`, focused CLI tests, and public docs while keeping resolve/watch behavior explicitly scoped.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and release inspection show the backend RPC exists while the requested CLI/docs/tests surface is absent from current main and `v2026.5.19`.

## Solution Assessment

Yes for the narrow first pass: `openclaw approvals pending` should reuse `exec.approval.list` instead of adding another protocol method. Resolve/respond and `--watch` should remain explicit maintainer/security scope rather than being required to close the original listing request.
