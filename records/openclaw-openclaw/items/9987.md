## What This Changes

The issue requests a CLI command to list pending exec approval requests, including command details plus JSON output and a possible watch mode.

## Best Possible Solution

Land a reviewed implementation, likely through https://github.com/openclaw/openclaw/pull/83440 or a successor, that adds one-shot pending/list output backed by `exec.approval.list`, focused CLI tests, and public docs while keeping resolve/watch behavior explicitly scoped.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and release inspection show the backend RPC exists while the requested CLI/docs/tests surface is absent from current main and v2026.5.20.

## Solution Assessment

Partly: a one-shot `openclaw approvals pending` or `list` command using the existing `exec.approval.list` RPC is the narrow maintainable path. Resolve and watch behavior need explicit maintainer/security scope before becoming part of the first public CLI surface.
