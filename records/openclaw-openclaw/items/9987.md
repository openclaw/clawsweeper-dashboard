## Summary

Keep open. Current main still has the backend `exec.approval.list` RPC, authorization scope, and server-method tests, but it does not expose the requested `openclaw approvals pending` CLI command or docs; the new open implementation PR is the right place to finish review before this issue closes.

## What This Changes

Requested a CLI command to list pending exec approval requests with command details, JSON output, and optional watch behavior.

## Best Possible Solution

Add a focused `openclaw approvals pending` CLI backed by the existing `exec.approval.list` RPC with table/JSON output, tests, and public docs; keep watch mode and approval-resolution scope explicit maintainer decisions.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and release inspection show the backend listing RPC exists while the requested CLI/docs/tests surface is still absent on current main and `v2026.5.12`.

## Solution Assessment

Partly: the requested CLI is the right user-facing shape, but the first implementation should reuse `exec.approval.list` in the existing approvals CLI and defer `--watch` unless maintainers define live-update behavior.
