## Summary

Keep this issue open. Current main still has the backend `exec.approval.list` API and tests, but the user-facing `openclaw approvals pending` CLI command, CLI tests, and CLI docs are still missing; the two direct implementation PRs were closed unmerged.

## What This Changes

The issue requests `openclaw approvals pending` to list pending exec approval requests with command details, JSON output, and optional watch behavior.

## Best Possible Solution

Add a narrow `openclaw approvals pending` command backed by the existing `exec.approval.list` gateway method, with table and JSON output plus docs/tests; scope `--watch` separately if maintainers want it.

## Reproduction Assessment

Yes. Source inspection on current main shows the approvals CLI has `get`, `set`, and `allowlist` but no `pending` subcommand, while the gateway backend already has `exec.approval.list`.

## Solution Assessment

Yes, if implemented narrowly through the existing approvals-scoped list API. Adding a new parallel protocol method would be a worse fit unless maintainers explicitly choose a protocol expansion.
