## What This Changes

The issue requests a CLI command such as `openclaw approvals pending` to list active exec approval requests, including command details and scriptable output.

## Best Possible Solution

Land a narrow CLI implementation that reuses the existing approval-list authority path, renders sanitized table and JSON output, adds focused CLI/docs coverage, and defers `--watch` unless maintainers scope it in.

## Reproduction Assessment

Yes. Source inspection of current main shows `openclaw approvals` only registers policy-oriented `get`, `set`, and `allowlist` commands while the backend `exec.approval.list` RPC already exists.

## Solution Assessment

Yes for the one-shot list portion, but it is not implemented on main. The maintainable solution is a CLI wrapper over the existing approval-list authority path with authorization, sanitization, tests, docs, and live proof rather than a second pending-approval protocol/store.
