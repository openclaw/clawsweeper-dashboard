## Summary

Keep this open. Current main already has backend pending exec approval listing support, but the requested user-facing `openclaw approvals pending` command, CLI tests, and approvals CLI docs are still missing; the two related implementation PRs were closed unmerged.

## What This Changes

Request to add an `openclaw approvals pending` CLI command that lists currently pending exec approval requests with command details, JSON output, and optional watch mode.

## Best Possible Solution

Add `openclaw approvals pending` in `src/cli/exec-approvals-cli.ts` backed by the existing `exec.approval.list` gateway method, with table output, `--json`, focused CLI tests, docs, and changelog coverage; defer `--watch` unless maintainers deliberately include it in the first pass.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The missing behavior is high-confidence from source/docs inspection: the CLI registers no `pending` subcommand while the backend list method already exists.

## Solution Assessment

Yes, the narrowest maintainable solution is to reuse `exec.approval.list` from the existing approvals CLI surface. A new protocol method or broader approval-system refactor is not needed for the first implementation.
