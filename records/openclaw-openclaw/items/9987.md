## Summary

Keep open. Current main and the latest release still have only the backend pending-approval listing RPC, not the requested `openclaw approvals pending` CLI, and the closed implementation PRs plus the broader Web dashboard request do not supersede this narrower CLI feature.

## What This Changes

The issue requests a CLI command to list pending exec approval requests with command details, JSON output, and optional watch behavior.

## Best Possible Solution

Add a focused one-shot `openclaw approvals pending` command backed by `exec.approval.list`, with table and JSON output, CLI tests, and public docs; defer live watch behavior until explicitly scoped.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main and `v2026.5.12` shows the backend listing RPC exists while the requested CLI/docs/tests surface is still absent.

## Solution Assessment

Partly: the requested CLI is the right user-facing shape, but the first implementation should reuse `exec.approval.list` in the existing approvals CLI and leave `--watch` out unless maintainers define live-update behavior.
