## Summary

Keep open: this PR targets a real source-reproducible memory-core batch resilience gap, but the current patch regresses generic batch timeout retries, lacks real behavior proof, and is conflicting with current main.

## What This Changes

The PR adds a memory-core retry-with-split helper for exhausted transport-failed embedding batches, routes text and structured batch embedding through it, and adds policy tests.

## Best Possible Solution

Rebase or replace this with a focused memory-core patch that preserves current retry classification, adds split-after-exhausted-transport fallback for text and structured batches, and includes regression coverage plus redacted real indexing proof.

## Reproduction Assessment

Yes, at source level. Mock a multi-item batch so the retry loop exhausts on `TypeError("fetch failed")` while single-item calls succeed; current main's batch wrappers call the retry loop directly and have no split fallback.

## Solution Assessment

No, not as written. Retry-plus-split is the right narrow direction, but the branch drops generic `timed out` retry coverage and still needs real indexing proof before merge.
