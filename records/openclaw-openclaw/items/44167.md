## Summary

Keep open. The PR still targets a real memory-core batch retry gap, but it is not merge-ready because the branch regresses the existing generic timeout retry path, has only mock/unit proof, and is currently conflicting against main.

## What This Changes

The PR adds a split-capable memory embedding batch retry helper, wires it into text and structured batch embedding paths, broadens transport retry classification, and adds focused policy tests.

## Best Possible Solution

Rebase or replace this with a narrow memory-core patch that preserves the current retry classifier, including generic `timed out`, adds split-after-exhausted-transport-retry fallback for text and structured batches, and includes regression coverage plus redacted real indexing proof.

## Reproduction Assessment

Yes, at source level. Mock a multi-item embedding batch to keep throwing `TypeError("fetch failed")` after retries while single-item calls succeed; current main retries and then throws because no split fallback is invoked.

## Solution Assessment

No, not as written. Retry-plus-split is the right narrow direction, but this branch must preserve generic batch timeout retries and provide real behavior proof before merge.
