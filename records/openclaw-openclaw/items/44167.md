## Summary

Keep this PR open. Current main and latest release still lack transport-error retry classification and exhausted-batch splitting for memory-core embeddings, and the paired bug remains open; the PR appears functionally scoped, but it still needs the required changelog entry and maintainer choice against the broader overlapping #75317.

## What This Changes

The PR changes memory-core embedding retry policy to classify transient transport errors as retryable, split failed text and structured batches after retry exhaustion, and adds focused policy tests.

## Best Possible Solution

Land this PR or an equivalent scoped memory-core change after adding a single-line changelog entry; if maintainers prefer the broader #75317 approach, make that the canonical PR and close this one only after the replacement path is explicit.

## Solution Assessment

Yes for the batch reindex failure this PR targets: it reuses the existing retry loop, keeps provider/config/schema behavior unchanged, and limits splitting to transport-classified multi-item batch failures. It does not fully solve the live single-query failure described in #71784, which belongs in a separate or broader follow-up.
