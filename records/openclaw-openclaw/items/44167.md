## Summary

Keep open. Current main still lacks the exhausted multi-item batch split fallback, and this PR still needs rebase/review because it regresses generic batch timeout retries and has only mocked test evidence rather than after-fix real behavior proof.

## What This Changes

This PR changes memory-core batch embedding retry policy to classify transport errors separately, split multi-item batches after exhausted transport retries, and add focused policy tests.

## Best Possible Solution

Rebase or replace this with a narrow memory-core patch that preserves the current retry classifier, adds split-after-exhausted-transport-retry fallback for text and structured batches, and includes regression coverage plus redacted real indexing proof.

## Reproduction Assessment

Yes, at source level. Mock a multi-item embedding batch to keep throwing `TypeError("fetch failed")` after retries while single-item calls succeed; current main retries and then throws because no split fallback is invoked.

## Solution Assessment

No, not as written. Retry-plus-split is the right narrow direction, but this branch must preserve generic timeout retries and provide real behavior proof before merge.
