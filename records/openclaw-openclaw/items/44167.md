## What This Changes

The PR adds memory-core transport retry classification, recursive split-after-retry fallback for text and structured embedding batches, retry log wording updates, and policy tests.

## Best Possible Solution

Land or replace a focused memory-core patch that keeps current transport and timeout retry classification while adding exhausted multi-item split fallback for both text and structured embedding batches, with regression coverage and real indexing proof.

## Reproduction Assessment

Yes, at source level. Current main calls `runMemoryEmbeddingRetryLoop` directly for text and structured batches and has no split helper, so a mocked multi-item `TypeError("fetch failed")` after retry exhaustion would throw instead of split.

## Solution Assessment

No, not as written. Retry-plus-split is the right narrow direction, but this branch needs to preserve generic `timed out` retry behavior and add real behavior proof before merge.
