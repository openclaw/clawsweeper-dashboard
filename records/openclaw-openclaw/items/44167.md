## Summary

Keep open. The PR still targets a real remaining memory-core bug because current main retries transport errors but does not split exhausted multi-item embedding batches, while this branch is not mergeable as written because it drops existing local timeout retry behavior and still has only mocked/unit proof.

## What This Changes

The PR changes memory-core embedding batch calls to retry transient transport failures, split failed multi-item text and structured batches after retry exhaustion, update retry logging, and add focused policy tests.

## Best Possible Solution

Land or replace a current-main memory-core patch that preserves the broadened retry classifier, adds exhausted transport split fallback for text and structured embedding batches, and includes focused regression coverage plus real behavior proof.

## Reproduction Assessment

Yes, at source/unit level. Mocking a provider so multi-item batches keep throwing `TypeError("fetch failed")` after retries while single-item calls succeed exercises current main's retry-then-throw path because no split helper exists.

## Solution Assessment

No, not as currently written. The retry-plus-split direction is the narrow memory-core fix, but the branch should be rebased on current main and preserve generic `timed out` retry behavior before merge.
