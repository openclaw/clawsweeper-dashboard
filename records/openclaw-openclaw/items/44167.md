## Summary

Keep open. The bug remains partly valid because current main still lacks an exhausted-transport split fallback for memory embedding batches, but this external PR cannot merge as-is due to a timeout retry regression, dirty merge state, and missing real behavior proof.

## What This Changes

The PR changes memory-core embedding batch retries to classify transport failures, recursively split exhausted multi-item text/structured batches, update retry logging, and add policy tests.

## Best Possible Solution

Land or replace a current-main memory-core patch that preserves generic timeout retry classification, adds exhausted transport split fallback for text and structured batches, includes regression coverage, and has real indexing proof.

## Reproduction Assessment

Yes, at source level. Current main's text and structured batch wrappers call `runMemoryEmbeddingRetryLoop` directly, so a mocked multi-item `TypeError("fetch failed")` after retry exhaustion would throw instead of splitting; I did not run tests because this was a read-only review.

## Solution Assessment

No, not as written. The retry-plus-split direction is the narrow memory-core fix, but this branch should preserve generic `timed out` retry behavior, resolve conflicts, and add real behavior proof before merge.
