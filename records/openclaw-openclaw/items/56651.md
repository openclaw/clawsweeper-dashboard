## Summary

Keep open. Current main and the latest shipped release still only expose adjacent handshake/health-monitor knobs, while gateway channel startup and plugin loading still have unbounded startup waits rather than the requested gateway-level announce, connection, and plugin-init timeout contract.

## What This Changes

The issue requests configurable gateway announce, provider connection, and plugin initialization timeouts so stalls fail fast with structured diagnostics instead of leaving the gateway unresponsive.

## Best Possible Solution

Define a gateway-owned startup and connection fast-fail contract with schema, docs, structured logging, restart interaction, and focused tests, while leaving sessions_send A2A timeout policy to the narrower linked tracker.

## Reproduction Assessment

Yes for the missing gateway timeout contract by source inspection: current schema, types, docs, and startup paths lack the requested bounded waits. No live stalled-provider or plugin-validation reproduction was run for the exact production hang.

## Solution Assessment

Unclear as proposed. Configurable timeouts are directionally useful, but the maintainable fix should split gateway-owned startup/connection fast-fail behavior from plugin/channel-owned connection policy and the separate sessions_send A2A timeout tracker.
