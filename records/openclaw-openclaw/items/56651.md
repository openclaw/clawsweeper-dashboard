## Summary

Keep this open. The issue requests configurable gateway announce, connection, and plugin initialization timeouts to avoid indefinite gateway stalls.

## Best Possible Solution

Keep this open for maintainer timeout-policy review: close the `sessions_send` announce gap through an agent/session-owned contract, keep provider and MCP timeouts in their owner namespaces, and add bounded gateway/plugin lifecycle timeouts only if maintainers approve that public config surface.
