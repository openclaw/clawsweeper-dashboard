## Summary

Keep open. Current main and v2026.5.20 still expose only adjacent handshake, subagent announce, MCP, and channel health-monitor timeouts; the requested gateway-owned announce, connection, and plugin-init timeout contract is still missing and needs maintainer policy review.

## What This Changes

## Best Possible Solution

Define a gateway-owned startup and connection fast-fail contract with schema, docs, structured logging, restart interaction, and focused tests, while leaving sessions_send A2A delivery timeout policy to https://github.com/openclaw/openclaw/issues/59339.

## Solution Assessment

Unclear as proposed. Configurable timeouts are directionally useful, but the maintainable fix should split gateway-owned startup/connection fast-fail behavior from plugin/channel-owned connection policy and the separate sessions_send A2A timeout tracker.
