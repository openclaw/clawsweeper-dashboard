## Summary

Keep open. Current main still only has adjacent mitigations such as pre-auth handshake timeout, channel health-monitor restart knobs, and subagent announce timeout configuration; it does not expose the requested gateway-level announce, provider-connection, or plugin-init timeout contract, and startup/bootstrap paths still contain unbounded pre-handoff awaits.

## What This Changes

Request to add gateway-level configurable announce, provider connection, and plugin initialization timeouts with structured fast-failure behavior.

## Best Possible Solution

Define a bounded gateway-owned startup fast-fail contract, keep provider/channel connection budgets and `sessions_send` A2A announce policy in their owner surfaces, and align config schema, docs, and focused tests.

## Reproduction Assessment

Yes for the missing behavior by source inspection: current schema/types/docs omit the requested gateway keys, and startup paths await pre-handoff plugin/channel phases without a generic timeout. I did not establish a live stalled-provider reproduction in this read-only review.

## Solution Assessment

Unclear as proposed. Configurable timeouts are directionally useful, but the maintainable solution should split gateway-owned fast-fail behavior from provider/channel-owned connection policy and the separate A2A announce timeout path.
