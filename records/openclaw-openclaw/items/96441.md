## What This Changes

The issue reports Ollama Cloud/OpenAI-compatible second-turn tool-call replay sending `tool_calls.function.arguments` as a JSON object instead of the required JSON string, causing provider 400 errors.

## Best Possible Solution

Land one narrow fix, preferably the proof-positive path in https://github.com/openclaw/openclaw/pull/96474, that keeps native Ollama `/api/chat` object replay separate from OpenAI-compatible JSON-string replay.
