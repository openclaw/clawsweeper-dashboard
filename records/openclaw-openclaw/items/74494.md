## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Close the mobile menu through state:** `ui/src/ui/app-render.helpers.ts:503-508`
  - body: The close button removes the `.open` class directly, so the overlay can visually close while `chatMobileControlsOpen` and `aria-expanded` still say it is open. Use `setChatMobileControlsOpen(false, ...)` so Escape, outside-click, tab-change, and focus restoration stay synchronized.
  - confidence: 0.93
- **[P2] Keep the shared mobile session controls:** `ui/src/ui/app-render.helpers.ts:528-554`
  - body: This manually renders the mobile session select and bypasses the shared session-control renderer that current main uses for the agent filter, session notice, model select, and thinking select. Preserve the shared path when rebasing so multi-agent mobile users do not lose the agent filter.
  - confidence: 0.82
- **[P2] Keep mobile refresh disabled during active runs:** `ui/src/ui/app-render.helpers.ts:559`
  - body: The desktop refresh button is disabled while chat is sending, has an active run, or is streaming, but the new mobile refresh button only checks loading and disconnected. That can trigger a manual refresh during an active run and reintroduce the race current main avoids.
  - confidence: 0.88
- **[P2] Scope expanded-card scrolling to mobile:** `ui/src/styles/chat/tool-cards.css:20-23`
  - body: This changes `.chat-tool-card--expanded` globally from unbounded visible content to a capped nested scroller, so desktop expanded tool cards also get clipped. Scope the overflow containment to the mobile overflow case or another mobile-only selector.
  - confidence: 0.9
- **[P2] Avoid disabling page zoom:** `ui/index.html:5`
  - body: Adding `maximum-scale=1.0` to the global viewport meta can prevent pinch zoom throughout the Control UI on mobile browsers and webviews. Keep the safe-area and keyboard viewport behavior if needed, but do not cap user zoom.
  - confidence: 0.84
- **[P3] Localize the new settings copy:** `ui/src/ui/app-render.helpers.ts:498-510`
  - body: The overlay adds English-only settings title and close label text, while the visible select labels are also hard-coded. Route the new strings through the existing Control UI locale keys so non-English users do not get mixed-language chat controls.
  - confidence: 0.8
