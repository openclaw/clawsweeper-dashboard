## Summary

Keep open, but do not merge as-is: current main still lacks several requested behaviors, while this stale broad branch mixes useful cron/Feishu fixes with correctness regressions, skipped tests, and unrelated security-sensitive Docker/secret-scan changes.

## What This Changes

The PR broadens cron main-session announce delivery and heartbeat routing, switches some isolated cron delivery to direct outbound, adds Feishu `union_id` target handling, introduces Docker host-state media path rewriting, and changes tests, secret scanning, and Docker artifacts.

## Best Possible Solution

Split the useful work into focused follow-ups: one cron PR for main-session announce delivery and heartbeat target propagation, one Feishu plugin PR for `on_`/`union_id` routing, and a separate Docker/media proposal only if maintainers accept the new environment contract.

## Reproduction Assessment

Yes. Source inspection on current main is enough to reproduce the remaining behavior gaps: main announce delivery is rejected, main heartbeat routing uses `target: "last"`, and Feishu `on_` targets resolve as `user_id`.

## Solution Assessment

No. The requested fixes are useful, but this PR is too broad and stale to be the best merge vehicle; narrower replacement PRs that preserve current cron delivery hardening are safer.
