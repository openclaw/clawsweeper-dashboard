Keep the useful direction, but split it into focused follow-ups: a cron PR for main-session announce delivery and heartbeat override propagation with CLI/account/failure-destination coverage, a Feishu plugin PR for `on_`/`union_id` target handling, and a separate Docker/media proposal only if maintainers accept the environment variable contract. Yes. Source inspection on current main is enough to reproduce the remaining requested behavior: main announce delivery is rejected or cleared, the CLI blocks the requested flags, heartbeat routing uses `target: "last"`, and Feishu `on_` targets fall through to `user_id`.
