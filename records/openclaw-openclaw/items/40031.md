## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Gate main announce delivery on explicit input:** `src/cli/cron-cli/register.cron-add.ts:221-247`
  - body: `--channel` defaults to `last`, so `hasDeliveryTarget` is truthy even when the user did not request delivery. Main system-event jobs therefore get `delivery.mode="announce"` by default, and `--no-deliver` can still be overridden by the default channel.
  - confidence: 0.9
- **[P2] Omit undefined heartbeat override fields:** `src/cron/service/timer.ts:1077-1083`
  - body: The override includes `to` and `accountId` even when they are undefined, and the heartbeat runner spreads it over configured heartbeat defaults. Main cron announce jobs that rely on configured default recipients or accounts can lose those defaults and route incorrectly or be skipped.
  - confidence: 0.84
- **[P2] Keep structured cron payloads on the direct path:** `src/cron/isolated-agent/delivery-dispatch.ts:499-503`
  - body: This gate can send structured payload jobs through the text finalization path when `deleteAfterRun`, follow-up, or silent-token conditions are true. That path only forwards synthesized text, so media or channel-data cron outputs can be dropped.
  - confidence: 0.86
- **[P2] Fail required announce delivery failures:** `src/cron/isolated-agent/delivery-dispatch.ts:409-418`
  - body: When `runSubagentAnnounceFlow` reports no delivery, the non-best-effort path still returns `status: "ok"` with an error string. Cron scheduling can treat a required delivery failure as a successful run and delete one-shot jobs without delivering the message.
  - confidence: 0.86
- **[P2] Narrow secret scanning suppressions:** `.pre-commit-config.yaml:86-93`
  - body: These whole-file exclusions hide future real secrets in docs, fixtures, and tests. Keep suppressions to exact false-positive lines or a minimal baseline cleanup instead of excluding entire tracked files.
  - confidence: 0.82
- **[P2] Remove the ad hoc Docker build surface:** `Dockerfile.cron-fix:20-21`
  - body: This Dockerfile is unrelated to the cron/Feishu feature and runs a mutable remote installer from bun.sh on an unpinned base image. Drop it from this PR or move accepted Docker work to the established build path.
  - confidence: 0.84
