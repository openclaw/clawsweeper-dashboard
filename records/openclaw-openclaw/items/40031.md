## Summary

Keep open. Current main has absorbed much of the isolated cron direct-delivery behavior, but it still rejects main-session announce delivery, lacks Feishu `on_`/`union_id` send-target handling, and lacks the Docker host-state media-path rewrite; the PR also needs a narrower rebase because it mixes unrelated CI, secrets, Docker, and test-skip churn with the feature work.

## What This Changes

The PR changes cron main-session announce delivery and heartbeat routing, isolated cron outbound delivery, Feishu target normalization, Docker media path handling, secret-scan configuration, tests, and an added cron-fix Dockerfile/script.

## Best Possible Solution

Keep the useful product direction, but split it into focused follow-ups: one cron PR for main-session announce delivery and heartbeat override propagation with CLI/account/failure-destination coverage, one Feishu plugin PR for `on_`/`union_id` target handling, and a separate Docker/media proposal only if maintainers accept `OPENCLAW_HOST_STATE_DIR` as a supported contract. Drop the already-landed isolated direct-delivery work and unrelated CI/secrets/test-skip changes.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the missing behavior: creating or patching main-session announce delivery is rejected or cleared, the CLI blocks the requested flags, heartbeat routing ignores delivery overrides, and Feishu `on_` targets resolve as `user_id`.

## Solution Assessment

No. The requested fixes are valid, but this branch is not the best merge vehicle because it combines several independent product changes with unrelated secrets baseline, Dockerfile, skipped-test, and CI-trigger churn; focused PRs would be safer and easier to validate.
