Overall confidence: 0.9

Full review comments:

- **[P2] Gate main announce delivery on explicit flags:** `src/cli/cron-cli/register.cron-add.ts:221-223`
  - body: `--channel` defaults to `last`, so treating any non-empty `opts.channel` as a delivery target makes main system-event jobs opt into announce delivery even when the user did not request it, and can also defeat `--no-deliver`. Track whether the user explicitly passed `--channel`, `--to`, or `--announce` before setting announce mode.
  - confidence: 0.88
- **[P2] Keep structured cron payloads on direct delivery:** `src/cron/isolated-agent/delivery-dispatch.ts:499-503`
  - body: This gate ignores `deliveryPayloadHasStructuredContent`, so structured media/channel payloads fall back to the announce path whenever `deleteAfterRun` or descendant-followup guards fire. The announce path only carries text, so payloads that current main sends directly can be dropped.
  - confidence: 0.86
- **[P2] Preserve heartbeat defaults when fields are omitted:** `src/cron/service/timer.ts:1079-1083`
  - body: The heartbeat override includes `to` and `accountId` even when they are undefined, and the wake runner later spreads the override over configured heartbeat defaults. Main cron announce jobs that rely on a configured recipient or account can lose those defaults and route incorrectly or be skipped.
  - confidence: 0.82
- **[P2] Restore forum-topic delivery coverage:** `src/cron/isolated-agent.direct-delivery-forum-topics.test.ts:18`
  - body: The PR changes direct cron delivery routing while skipping the forum-topic regression test that covers threaded Telegram delivery on this path. Fix or replace the assertion instead of disabling coverage for behavior this branch changes.
  - confidence: 0.86
- **[P2] Scope secret-scan exclusions narrowly:** `.pre-commit-config.yaml:86-93`
  - body: These whole-file detect-secrets exclusions can hide future real credentials in docs, tests, and fixtures. Keep suppressions to exact false-positive lines or a minimal baseline cleanup instead of excluding entire tracked files.
  - confidence: 0.82
- **[P2] Remove the ad hoc Docker build path:** `Dockerfile.cron-fix:21`
  - body: `Dockerfile.cron-fix` is unrelated to the cron/Feishu feature and downloads and executes Bun from a mutable URL while using an unpinned base image. Drop it from this PR or move accepted Docker work to the established build path.
  - confidence: 0.84
