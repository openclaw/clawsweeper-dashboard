Full review comments:

- **[P1] Fail cron runs when announce delivery fails:** `src/cron/isolated-agent/delivery-dispatch.ts:409-411`
  - body: When announce delivery returns false or throws and best-effort is not enabled, this branch records the cron run as `status: "ok"`. That can make a one-shot or delete-after-run job look successful even though no user message was delivered.
  - confidence: 0.9
- **[P2] Preserve direct delivery for structured payloads:** `src/cron/isolated-agent/delivery-dispatch.ts:499-503`
  - body: The direct-delivery gate no longer forces structured payloads down the direct path. If `deleteAfterRun`, silent/interim heuristics, or descendant checks force announce, media or channel payloads can be dropped because announce only sends text.
  - confidence: 0.86
- **[P2] Gate main announce delivery on explicit options:** `src/cli/cron-cli/register.cron-add.ts:221-228`
  - body: `opts.channel` is defaulted, so treating any non-empty channel as a requested delivery target makes main system-event jobs opt into announce delivery even when the user did not ask for it, and can defeat `--no-deliver`.
  - confidence: 0.88
- **[P2] Preserve heartbeat defaults when fields are omitted:** `src/cron/service/timer.ts:1079-1083`
  - body: The heartbeat override includes `to` and `accountId` even when they are undefined. The wake runner later spreads this over configured heartbeat defaults, so default recipients or accounts can be erased.
  - confidence: 0.82
- **[P2] Restore forum-topic delivery coverage:** `src/cron/isolated-agent.direct-delivery-forum-topics.test.ts:18`
  - body: The PR changes direct cron delivery routing while skipping the forum-topic regression test that covers threaded Telegram delivery on this path. Fix or replace the assertion instead of disabling the coverage.
  - confidence: 0.86
- **[P2] Scope secret scanning exclusions narrowly:** `.pre-commit-config.yaml:86-93`
  - body: The new whole-file exclusions can hide future real credentials in docs, tests, and fixtures. Suppress only exact false-positive lines or keep the baseline cleanup out of this feature branch.
  - confidence: 0.82
- **[P2] Remove the ad hoc Docker build surface:** `Dockerfile.cron-fix:21`
  - body: This new Dockerfile is unrelated to the cron and Feishu changes and executes a mutable remote Bun installer from an unpinned base image. Drop it from this PR or move accepted Docker work to the established build path.
  - confidence: 0.84
