  - body: This gate no longer forces direct delivery for `deliveryPayloadHasStructuredContent` or thread-bound payloads. If a structured cron result also hits `deleteAfterRun`, descendant-follow-up, or silent-reply conditions, it falls into `deliverViaAnnounce`, which only carries text and can drop media/channel payloads that current direct delivery preserves.
  - confidence: 0.9
- **[P1] Fail non-best-effort announce delivery errors:** `src/cron/isolated-agent/delivery-dispatch.ts:410-418`
  - body: When `runSubagentAnnounceFlow` returns false or throws, non-best-effort delivery still returns `status: "ok"` with `delivered: false`. Scheduler success handling can then treat a one-shot or delete-after-run job as successful even though no message was delivered, so required delivery failures need to return an error status.
  - confidence: 0.92
- **[P1] Ignore default channel when enabling main announce:** `src/cli/cron-cli/register.cron-add.ts:221-248`
  - body: `--channel` defaults to `last`, so `hasDeliveryTarget` is true even when the user did not request announce delivery. That makes main system-event jobs resolve to announce by default and can make `--no-deliver` ineffective for main jobs, changing existing cron behavior during upgrade.
  - confidence: 0.88

  - body: Production secret path validation now skips Windows ACL verification whenever `VITEST` and `OPENCLAW_SECRETS_RUNTIME_TEST` are set. A runtime process with those env vars would bypass the explicit `allowInsecurePath` policy, which is too broad for a cron/Feishu delivery PR.
  - confidence: 0.82
