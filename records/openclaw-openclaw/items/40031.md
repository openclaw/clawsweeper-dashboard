- **[P2] Gate main announce delivery on explicit user input:** `src/cli/cron-cli/register.cron-add.ts:221-236`
  - body: `--channel` defaults to `"last"`, so treating any non-empty `opts.channel` as a delivery target makes main system-event jobs opt into announce delivery even when the user did not request it. It also makes `--no-deliver` ineffective because the default channel still forces announce mode.
  - confidence: 0.88
- **[P2] Preserve announce safeguards for text cron replies:** `src/cron/isolated-agent/delivery-dispatch.ts:467-470`
  - body: The new direct-delivery gate can route text-only resolved deliveries around the announce path's descendant and interim-reply handling. That can send incomplete parent text instead of waiting for or promoting subagent output.
  - confidence: 0.84
- **[P2] Fail non-best-effort runs when announce delivery fails:** `src/cron/isolated-agent/delivery-dispatch.ts:397-405`
  - body: When `runSubagentAnnounceFlow` reports a failed send, the non-best-effort path can still return an `ok` cron result with only an error field. That lets one-shot jobs be treated as successful even though delivery failed.
  - confidence: 0.82
- **[P2] Preserve heartbeat defaults when fields are omitted:** `src/cron/service/timer.ts:1077-1081`
  - body: The heartbeat override includes `to` and `accountId` even when they are undefined, and the wake runner later spreads that over the configured heartbeat. Main cron announce delivery that relies on configured defaults can lose the recipient or account.
  - confidence: 0.82
- **[P2] Restore forum-topic delivery coverage:** `src/cron/isolated-agent.direct-delivery-forum-topics.test.ts:18`
  - body: This PR changes direct cron delivery routing while skipping the forum-topic regression test that covers threaded Telegram delivery on this path. Fix or replace the assertion instead of disabling coverage for the behavior being changed.
  - confidence: 0.86
- **[P2] Scope secret scanning exclusions narrowly:** `.pre-commit-config.yaml:81-92`
  - body: The new detect-secrets exclusions suppress whole docs, tests, fixtures, and broad line patterns. That can hide future real credentials; keep suppressions to exact false-positive lines or a minimal baseline cleanup.
  - confidence: 0.82
- **[P2] Remove the ad hoc Docker build surface:** `Dockerfile.cron-fix:21`
  - body: `Dockerfile.cron-fix` is unrelated to the cron/Feishu feature and downloads and executes Bun from a mutable URL while using an unpinned base image. Drop it from this PR or move accepted Docker work to the established build path.
  - confidence: 0.84
