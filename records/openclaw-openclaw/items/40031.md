## Summary

Keep open: current main still lacks important parts of the requested cron/Feishu behavior, but this broad branch is not mergeable as-is because it also introduces skipped tests and security-sensitive infrastructure churn.

## What This Changes

The PR changes cron main-session announce and heartbeat delivery, isolated cron outbound dispatch, Feishu target normalization, Docker media path rewriting, secret-scanning config/baseline files, tests, a new Dockerfile, and a PR-check script.

## Best Possible Solution

Split the useful work into focused follow-ups: one cron PR for main-session announce delivery and heartbeat target propagation, one Feishu plugin PR for `on_`/`union_id` routing, and a separate Docker/media proposal only if maintainers accept the new environment contract.

## Reproduction Assessment

Yes. Source inspection on current main is enough to reproduce the remaining requested behavior gaps: main announce delivery is rejected, main heartbeat routing uses `target: "last"`, and Feishu `on_` targets resolve as `user_id`.

## Solution Assessment

No. The requested fixes are useful, but this PR is too broad and stale to be the best merge vehicle; narrower replacement PRs that preserve current cron delivery hardening are safer.
