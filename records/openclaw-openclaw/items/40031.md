Full review comments:

- **[P2] Gate main announce delivery on explicit user input:** `src/cli/cron-cli/register.cron-add.ts:221-247`
  - body: `--channel` defaults to `last`, so this treats a default option as an explicit delivery target. Main system-event jobs can get `delivery.mode="announce"` by default, and `--no-deliver` can still be overridden by the default channel.
  - confidence: 0.9
- **[P2] Omit undefined heartbeat override fields:** `src/cron/service/timer.ts:1077-1083`
  - body: The override includes `to` and `accountId` even when they are undefined. The heartbeat runner merges this over configured heartbeat defaults, so cron jobs that rely on default recipients or accounts can lose those defaults and route incorrectly or be skipped.
  - confidence: 0.84
- **[P2] Keep structured cron payloads on the direct path:** `src/cron/isolated-agent/delivery-dispatch.ts:499-503`
  - body: This gate sends structured media/channel payloads through announce whenever `deleteAfterRun`, follow-up, or silent-token conditions are true. The announce path only forwards synthesized text, so structured cron output can be dropped where it previously used direct outbound delivery.
  - confidence: 0.86
- **[P2] Fail required announce delivery failures:** `src/cron/isolated-agent/delivery-dispatch.ts:409-418`
  - body: When `runSubagentAnnounceFlow` reports no delivery, the non-best-effort path still returns `status: "ok"` with an error string. That makes failed required delivery look successful to scheduling and can delete one-shot jobs without delivering the message.
  - confidence: 0.86
- **[P2] Restore cron forum-topic regression coverage:** `src/cron/isolated-agent.direct-delivery-forum-topics.test.ts:18`
  - body: This PR changes direct cron delivery routing while skipping the forum-topic delivery regression test that exercises that routing surface. Update the expectation or fix the routing behavior instead of disabling coverage for the changed path.
  - confidence: 0.78
- **[P2] Narrow secret scanning suppressions:** `.pre-commit-config.yaml:86-93`
  - body: These whole-file exclusions hide future real secrets in docs, fixtures, and tests. Keep suppressions to exact false-positive lines or a minimal baseline cleanup instead of excluding entire tracked files.
  - confidence: 0.82
- **[P2] Remove the ad hoc Docker build surface:** `Dockerfile.cron-fix:20-21`
  - body: This Dockerfile is unrelated to the cron/Feishu feature and runs a mutable remote installer from bun.sh on an unpinned base image. Drop it from this PR or move accepted Docker work to the established build path.
  - confidence: 0.84
