  - body: `--channel` defaults to `"last"`, so using any non-empty `opts.channel` as a delivery target makes main system-event jobs opt into announce delivery even when the user did not request it. It also makes `--no-deliver` ineffective because the default channel still forces announce mode.
  - confidence: 0.88

  - body: At this point `params.resolvedDelivery.ok` is already true, so the new condition routes every text-only resolved delivery through the direct path. That bypasses the announce path's descendant/interim handling and can send incomplete parent replies instead of waiting for or promoting subagent output.
  - confidence: 0.86
- **[P2] Preserve heartbeat defaults when fields are omitted:** `src/cron/service/timer.ts:1077-1081`
  - body: The heartbeat override includes `to` and `accountId` even when they are undefined, and the wake runner later spreads that over the configured heartbeat. Main cron announce delivery that relies on a configured default recipient or account can lose those defaults and route incorrectly or be skipped.
  - confidence: 0.82
- **[P2] Restore forum-topic delivery coverage:** `src/cron/isolated-agent.direct-delivery-forum-topics.test.ts:18`
  - body: The PR changes direct cron delivery routing while skipping the forum-topic regression test that covers threaded Telegram delivery on this path. Fix or replace the assertion instead of disabling the coverage for the behavior being changed.
  - confidence: 0.86
- **[P2] Scope secret scanning exclusions narrowly:** `.pre-commit-config.yaml:81-92`
  - body: These whole-file detect-secrets exclusions can hide future real credentials in docs, tests, and fixtures. Keep suppressions to exact false-positive lines or a minimal baseline cleanup instead of excluding entire tracked files.
