## Summary

Keep open: this is a focused candidate fix for the canonical open loop-breaker issue, and current main plus v2026.6.8 still have the per-signature global-breaker bug. I found no blocking patch defect, but merge still needs maintainer acceptance of the stricter fail-closed semantics for users who enabled loop detection.

## What This Changes

The PR changes agent tool-loop detection so the global circuit breaker counts repeated no-progress outcomes across the recent session history instead of only the current tool signature, with detector and before-tool-call wrapper tests.

## Best Possible Solution

Land the detector-level session-global breaker fix after maintainer approval of the stricter documented fail-closed behavior; leave broader volatile-output and consecutive-error loop work to their separate issues.

## Reproduction Assessment

Yes. Current main and v2026.6.8 compare the global threshold against a per-tool/args no-progress streak, while the wrapper calls this detector from shared session history before every tool call.

## Solution Assessment

Yes. The detector is the right layer because callers already supply scoped session history; the PR preserves the other per-pattern detectors and adds focused detector plus wrapper regression coverage.
