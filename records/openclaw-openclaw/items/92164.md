## Summary

## What This Changes

The branch changes memory-core keyword planning/search and regression tests so trigram CJK tokens use LIKE terms and empty MATCH results retry the LIKE fallback.

## Best Possible Solution

Land a bounded CJK keyword fix only after maintainers accept the proof override or get real memory_search proof, and after the expanded LIKE path is either latency-proven on a non-trivial index or adjusted to avoid unbounded blocking scans.

## Reproduction Assessment
