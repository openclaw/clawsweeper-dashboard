## Summary

Keep open: current main has several useful pieces, but it does not fully implement the requested doctor-level consolidated verdict or deterministic recovery guidance for mixed LaunchAgent/runtime/RPC states.

## What This Changes

The issue requests clearer `openclaw doctor` and gateway diagnostics when LaunchAgent service state, runtime process state, entrypoint drift, and duplicate channel pollers produce mixed signals.

## Best Possible Solution

Add a small shared gateway diagnosis model for doctor/status that reports service loaded state, runtime PID/status, RPC/read-probe state, service config drift, duplicate services or pollers, and deterministic next steps, with docs for any JSON contract.

## Reproduction Assessment

Yes, from source inspection rather than a live macOS run. A failed doctor status RPC can print `Gateway not running`, then the later daemon pass can print a running runtime summary when launchd/service state is still running.

## Solution Assessment

No. Current main has useful partial diagnostics, but the issue asks for a consolidated doctor verdict and ordered recovery guidance that are not yet present.
