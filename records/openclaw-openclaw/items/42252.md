## Summary

Keep open: current main and the latest shipped tag still split Gateway RPC liveness, supervisor/runtime state, service-config drift, and channel warnings across separate doctor steps, so the requested consolidated health verdict and deterministic recovery guidance are still missing.

## What This Changes

Request clearer `openclaw doctor` and gateway diagnostics for mixed LaunchAgent, runtime, service-config drift, and duplicate-poller states.

## Best Possible Solution

Define a shared gateway diagnostics summary carrying service-loaded, runtime PID, RPC probe, service-config drift, listener/port, and recent channel-conflict facts, then render one final healthy/degraded/down verdict with deterministic next steps.

## Reproduction Assessment

Yes, at source level. Make the status RPC fail with `gateway closed` while the supervisor service remains loaded and `service.readRuntime()` reports running, and doctor can emit `Gateway not running.` before the later daemon step reports `Runtime: running`.

## Solution Assessment

Yes, the requested direction is best solved as a shared diagnostics contract and final verdict renderer. Isolated wording fixes have only addressed narrow duplicate panels and would leave service, runtime, RPC, and channel-conflict facts split across doctor steps.
