## What This Changes

The issue requests clearer doctor and gateway diagnostics for mixed LaunchAgent, runtime, RPC, entrypoint drift, and duplicate Telegram poller states.

## Best Possible Solution

Define a shared gateway diagnostics summary carrying service-loaded, runtime PID, RPC probe, service-config drift, port/listener, and recent channel-conflict facts, then render one final healthy/degraded/down verdict with deterministic next steps.

## Reproduction Assessment

Yes, at source level. Make the status RPC fail with `gateway closed` while the service remains loaded and `service.readRuntime()` reports running, and doctor can emit `Gateway not running.` before a later `Runtime: running` note.

## Solution Assessment

Yes. The maintainable solution is a shared diagnostics contract and final verdict renderer; isolated message tweaks would leave the same split-brain facts scattered across doctor steps.
