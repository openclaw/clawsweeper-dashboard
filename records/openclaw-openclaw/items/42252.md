## Best Possible Solution

Define a maintainer-approved consolidated doctor/gateway diagnosis model that combines service loaded state, runtime PID state, RPC probe state, service config drift, and recent duplicate-instance/channel hints into one final action-ordered verdict without adding long-lived compatibility shims.
