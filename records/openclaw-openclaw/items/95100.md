## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Forward the run context into usage loading:** `src/auto-reply/reply/agent-runner.ts:2295-2299`
  - body: `loadProviderUsageSummary` resolves auth profiles and plugin usage hooks from the supplied `agentDir`/`workspaceDir`, and the existing `/status` path forwards those values. This new reply-footer call only passes `config`, so `/usage full` can read the default agent/plugin context instead of the active run's context, which means non-default agents or workspace plugin installs can show missing or wrong quota windows. Pass `followupRun.run.agentDir` and `followupRun.run.workspaceDir` here and cover that path.
  - confidence: 0.9
- **[P3] Clear the quota timeout after the race settles:** `src/auto-reply/reply/agent-runner.ts:2301-2304`
  - body: The new timeout promise is never cleared when `loadProviderUsageSummary` wins the race. That leaves a ref'd timer alive for up to two seconds on every successful `/usage full` reply, unlike the existing `/status` path which stores and clears the timeout in `finally`. Mirror that cleanup or use the shared timeout helper so active sessions do not accumulate unnecessary timers.
  - confidence: 0.84
