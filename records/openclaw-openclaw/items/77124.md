## Summary

Keep open: current main still contains the trajectory cleanup timeout path, and the open closing PR #77154 has not merged, so this should remain the tracker until that fix lands.

## What This Changes

Large runtime trajectory files can make `pi-trajectory-flush` time out while the queued trajectory writer continues draining and stalls the gateway event loop.

## Best Possible Solution

Review and land #77154, or an equivalent bounded trajectory-runtime fix, then close this after the fix reaches main while leaving broader event-loop/session issues to their own trackers.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection on current main shows the reported timeout/queued-flush path, and the issue includes diagnostic logs for the observed 50 MiB stall.

## Solution Assessment

Yes for the linked bounded trajectory-runtime direction: capping or stopping trajectory writes before large flush work is narrower than adding only a timeout env var, which would not stop the background cleanup work.
