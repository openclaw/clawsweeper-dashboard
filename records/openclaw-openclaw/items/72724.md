## Summary

Needs changes before merge: the PR is a useful fix path for gateway delivery visibility, but the latest status predicate can still warn on healthy resolver-handled traffic that completes without creating an agent turn.

## What This Changes

The PR adds gateway inbound delivery diagnostics/status warnings, delivery exporter metrics, provider allowlist normalization, tests, and a changelog entry.

## Best Possible Solution

Keep the telemetry direction, but classify completed terminal no-agent-turn outcomes before warning so status only flags actual missing handoff or stale in-flight dispatches.

## Reproduction Assessment

Yes, source-level: PR-head status warns on dispatchStarted > 0 with no turnsCreated, while the PR emits dispatchStarted before the generic replyResolver and emits session.turn.created only inside runAgentTurnWithFallback.

## Solution Assessment

No as currently written: the telemetry/status approach is maintainable, but the warning predicate should treat completed processed no-agent-turn outcomes as terminal handled traffic rather than degraded delivery.

- **[P2] Gate turn warnings on terminal handled dispatches:** `src/commands/status-all/diagnosis.ts:321-327`
