Full review comments:

- **[P2] Sanitize new OTel delivery labels:** `extensions/diagnostics-otel/src/service.ts:1388-1390`
  - body: These new handlers pass `evt.channel` and `evt.source` directly into OTel metric attrs, while adjacent message telemetry uses `lowCardinalityAttr` to redact and bound label values. Since these diagnostic fields are plain strings, custom or malformed values can bypass the exporter label policy and create high-cardinality or unredacted attrs. Please wrap channel and source with `lowCardinalityAttr` in the received, dispatch-started, and dispatch-completed handlers.
  - confidence: 0.9

  - confidence: 0.86

  - body: This PR adds new Prometheus and OpenTelemetry delivery/turn metrics, but the public metric tables in `docs/gateway/prometheus.md` and `docs/gateway/opentelemetry.md` still list only the older message metrics. Operators use those tables as the metric contract, so please add the new metric names and labels there.
