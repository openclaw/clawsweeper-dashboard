## Summary

Keep open. Current main still joins all eligible assistant text blocks before embedded channel delivery, and the related open issues cover adjacent delivery/audience problems rather than a merged selector-policy fix.

## What This Changes

The issue requests a deterministic delivery-boundary policy so multi-text assistant messages send one public reply instead of a concatenated near-duplicate while preserving transcript-oriented surfaces.

## Best Possible Solution

Add an explicit delivery-boundary selector that leaves shared transcript extraction as concatenation by default, then cover same-phase multi-text assistant messages in embedded delivery tests and preserve TUI, gateway preview, and chat-history behavior.

## Reproduction Assessment

Yes. A synthetic assistant message with two eligible same-phase `text` blocks is enough to trace the current source path from the extractor join into `handleMessageEnd` delivery without private Slack data.

## Solution Assessment

Unclear until maintainers choose the selector rule. The narrow maintainable direction appears to be a delivery-only selector rather than changing the shared extractor default.
