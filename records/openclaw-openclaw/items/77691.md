## Summary

This can close because current OpenClaw source and docs do not document `acpx ... sessions cleanup`; the implemented cleanup command is `openclaw sessions cleanup`, while direct `acpx` session command naming belongs to the upstream ACPX package.

## What This Changes

The report asks for either an `acpx sessions cleanup` command or docs that point users to `acpx sessions prune` instead.

## Best Possible Solution

Keep the OpenClaw command documented as `openclaw sessions cleanup`; any direct `acpx` alias or prune/cleanup naming change should be handled in the upstream ACPX package unless a specific stale OpenClaw docs URL is provided.

## Reproduction Assessment

No: current OpenClaw docs and CLI source document `openclaw sessions cleanup`, and the current acpx docs section does not show an `acpx ... sessions cleanup` command. The direct `acpx` command list in the report is upstream ACPX behavior rather than an OpenClaw CLI command.

## Solution Assessment

Not applicable for this repository: adding a direct `acpx sessions cleanup` alias would be an ACPX package change, while OpenClaw already has its own cleanup command and docs aligned to it.
