## Summary

Keep open: current main still has a source-reproducible FAL image-generation dispatch bug where the reported model string is parsed as provider `fal-ai`, so the request never reaches FAL auth resolution.

## What This Changes

The issue asks for the FAL image-generation provider to resolve `FAL_KEY` or auth-profile credentials and generate with `fal-ai/flux/dev`.

## Best Possible Solution

Normalize FAL image model selection so `image_generate` does not misroute the displayed FAL model id, and add regression coverage plus a changelog entry for the user-facing fix.

## Reproduction Assessment

Yes from source inspection: pass `model: "fal-ai/flux/dev"` to `image_generate`, and current main parses it as provider `fal-ai`, producing the reported registered-provider error before auth resolution. I did not run a live FAL call in this read-only review.

## Solution Assessment

Yes, a narrow fix in media model-ref normalization or image tool FAL model selection is the best path; changing core auth behavior alone would not address the observed `fal-ai` provider lookup failure.
