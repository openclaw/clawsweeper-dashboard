## Summary

Keep open: current main still has a source-reproducible FAL image-generation routing bug where the reported model string is parsed as provider `fal-ai`, so the request fails before FAL auth resolution can run.

## What This Changes

FAL image generation should route `fal-ai/flux/dev` to the bundled `fal` provider and use `FAL_KEY` or valid auth-profile credentials instead of failing as an unregistered provider.

## Best Possible Solution

Normalize FAL image model selection so `image_generate model=fal-ai/flux/dev` resolves to the registered `fal` provider, or make the tool consistently expose provider-prefixed FAL refs, then add regression coverage and a changelog entry.

## Reproduction Assessment

Yes. From source inspection, passing `model: "fal-ai/flux/dev"` to `image_generate` parses provider `fal-ai`, then current main emits the reported unregistered-provider error before reaching FAL auth.

## Solution Assessment

Yes. The narrow maintainable fix is in media model-ref normalization or image tool FAL model selection; changing generic auth behavior alone would not address the observed `fal-ai` provider lookup failure.
