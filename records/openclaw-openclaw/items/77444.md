## Summary

Keep open: current main still has a source-reproducible FAL image-generation dispatch failure where `model: "fal-ai/flux/dev"` is parsed as provider `fal-ai`, so the request fails before the `fal` provider can resolve `FAL_KEY`.

## What This Changes

The issue reports that FAL image generation using `fal-ai/flux/dev` fails to resolve FAL auth and is rejected as an unregistered image-generation provider.

## Best Possible Solution

Normalize image-generation model selection so exact listed FAL model ids route to the registered `fal` provider before auth resolution, with regression coverage and a changelog entry.

## Reproduction Assessment

Yes, from source inspection: pass `model: "fal-ai/flux/dev"` to `image_generate`, and current main parses it as provider `fal-ai`, producing the reported registered-provider error before FAL auth resolution.

## Solution Assessment

Yes, the narrow maintainable fix is generic media/image model-ref normalization or tool selection for exact listed provider model ids; changing FAL auth alone would not address the observed provider lookup failure.
