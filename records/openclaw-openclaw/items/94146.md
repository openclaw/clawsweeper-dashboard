## Label Justifications

- `P2`: This is a normal-priority skills runtime bug with limited blast radius to skills configured by a source-specific `skillKey`.
- `impact:session-state`: The issue is caused by persisted skill snapshot state losing metadata needed for later runtime behavior.
- `impact:auth-provider`: The issue can prevent configured skill API keys or env credentials under `skills.entries.<skillKey>` from being injected at runtime.

## Summary

Keep open: current main and the latest release still drop the resolved skill key before snapshot env injection, and open linked PRs with closing syntax are already candidate fixes.

## What This Changes

The issue requests preserving the resolved `skillKey` in `SkillSnapshot.skills[]` and using it for snapshot env override lookup so `skills.entries.<skillKey>` API keys/env values are injected.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/94163 or an equivalent fix that updates runtime and session snapshot shapes, uses `skillKey ?? name`, refreshes old snapshots, and then closes duplicate fix PRs.

## Reproduction Assessment

Yes. Source inspection shows current main builds snapshots without the resolved `skillKey` and then resolves snapshot env config by display `name`, so config under `skills.entries.<skillKey>` is missed when the display name differs.

## Solution Assessment

Yes. Persisting optional `skillKey`, resolving snapshot env overrides by `skillKey ?? name`, and refreshing old persisted snapshots is the narrow owner-boundary fix; the remaining work is to land one linked candidate.
