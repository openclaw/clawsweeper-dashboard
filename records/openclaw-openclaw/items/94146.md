## Label Justifications

- `P2`: This is a normal-priority runtime bug with limited blast radius to skills configured by a source-specific skillKey.
- `impact:session-state`: The bug is caused by session skill snapshot metadata losing the key needed for later runtime behavior.
- `impact:auth-provider`: The miss can prevent configured skill API keys or env credentials under skills.entries.<skillKey> from being injected at runtime.

## Summary

Keep open: current main and the latest release still drop the resolved skill key before snapshot env injection, and two live open PRs already use closing references for this issue.

## What This Changes

The issue reports that SkillSnapshot entries omit the resolved skillKey, causing snapshot-based env override lookup to miss config stored under skills.entries.<skillKey>.

## Best Possible Solution

Land one narrow linked fix that persists optional skillKey in new snapshots, uses skill.skillKey ?? skill.name for snapshot env lookup, and covers producer, consumer, and old-snapshot compatibility paths.

## Reproduction Assessment

Yes. Source inspection shows current main builds snapshots without skillKey and then resolves snapshot env config by skill.name, while loading/status paths use metadata.skillKey; I did not run tests because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to persist an optional resolved skillKey in SkillSnapshot and use it for snapshot env override lookup while keeping old snapshots safe; the issue should stay open until one linked PR lands.
