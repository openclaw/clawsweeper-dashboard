## Summary

Keep open: current main still drops the resolved skill key before snapshot env injection, and two open linked PRs use closing syntax for this issue, so the issue should remain open until one fix merges.

## What This Changes

Fix snapshot skill env override resolution when a skill display name differs from `metadata.openclaw.skillKey`.

## Best Possible Solution

Land one narrow linked fix that persists optional `skillKey`, uses it for snapshot env override lookup, safely handles older snapshots, and includes focused producer/consumer/upgrade coverage.

## Reproduction Assessment

Yes. From source inspection, current main builds snapshots without the resolved `skillKey` and then resolves snapshot env config by display `name`, so config under `skills.entries.<skillKey>` is missed when the display name differs.

## Solution Assessment

Yes. Persisting an optional resolved `skillKey` in new snapshots and using it for snapshot env lookup is the narrow owner-boundary fix; the open work is to land one linked candidate, not create a separate repair lane.
