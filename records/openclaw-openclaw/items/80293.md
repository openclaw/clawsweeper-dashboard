## Summary

## What This Changes

The branch applies outbound thread-id resolution before plugin-owned message action dispatch, adds a threaded sticker-action regression test, and declares Slack's typebox runtime dependency.

## Best Possible Solution

Land the focused dispatcher change and regression test while preserving current main's Slack manifest pins; keep the Telegram-visible proof label so maintainers can request a short topic-routing demo if needed.

## Reproduction Assessment

Yes, from source inspection: current main resolves outbound threadId for send and poll but plugin-owned actions go straight to gateway/local dispatch without that helper. I did not run a failing local repro because this was a read-only review.

## Solution Assessment

Yes. Reusing resolveAndApplyOutboundThreadId once at the plugin-action boundary is the narrowest maintainable fix because it matches existing send/poll behavior before both gateway and local dispatch.
