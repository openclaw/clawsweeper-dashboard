## Best Possible Solution

Land the focused dispatcher fix that reuses `resolveAndApplyOutboundThreadId` for plugin actions before gateway/local dispatch, with regression coverage and current main dependency pins preserved.

Yes, source-reproducible: current main applies outbound thread resolution for send and poll but not for plugin-owned actions, while Telegram sticker dispatch consumes `params.threadId` as `messageThreadId`. I did not execute the failing path because this review was read-only.

## Solution Assessment
