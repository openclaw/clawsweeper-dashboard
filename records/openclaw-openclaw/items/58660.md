## Summary

Keep open: current main has generic install, uninstall, stale-config cleanup, and channel replacement primitives, but I did not find a current-main path that maps or removes a still-installed legacy `openclaw-weixin` when installing a differently named WeChat plugin. The remaining decision is policy-sensitive because automatic cleanup by similar name could remove explicitly installed third-party plugin code/config.

## What This Changes

The issue asks OpenClaw to handle the WeChat plugin rename/migration from `openclaw-weixin` to `openclawwechat`, including cleanup or clearer upgrade guidance when both plugin ids are present.

## Best Possible Solution

Keep this open for a maintainer decision on the migration policy. The safest end state is explicit opt-in replacement metadata or a non-destructive install/doctor warning plus WeChat migration docs, with destructive cleanup left to explicit uninstall or doctor repair paths rather than guessed from similar names.

## Reproduction Assessment

Unclear: the report includes concrete versions and install output, but the exact `openclawwechat` package and old/new mixed install fixture are not in this repository. Source inspection does show the current installer is keyed to the resolved plugin id and does not migrate a separate live legacy id.

## Solution Assessment

No for automatic same/similar-name cleanup as proposed. A maintainable solution should use explicit plugin-owned replacement metadata, warnings, or documented uninstall/migration steps so OpenClaw does not guess which third-party plugin should be removed.
