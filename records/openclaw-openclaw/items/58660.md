## Summary

Keep open: current main supports explicit uninstall and same-id replacement, but it still has no `openclaw-weixin` to `openclawwechat` migration or safe different-id cleanup path. The remaining work is a product/security decision around explicit replacement metadata, doctor guidance, or plugin-owned migration docs rather than an automatic same-name cleanup bug fix.

## What This Changes

The issue asks OpenClaw to handle migration or cleanup when installing `openclawwechat` while the legacy `openclaw-weixin` plugin remains installed.

## Best Possible Solution

Define an explicit opt-in replacement or migration story, likely through catalog/plugin metadata plus doctor or docs guidance, and keep destructive different-id cleanup behind a maintainer-approved path.

## Reproduction Assessment

No, not at high confidence. Source inspection shows current main has no different-id replacement cleanup, but the exact external `openclawwechat` and legacy `openclaw-weixin` install state was not recreated.

## Solution Assessment

No for automatic same/similar-name deletion as proposed. The safer maintainable direction is explicit replacement metadata or doctor/docs guidance, because guessing by plugin name could remove a user-selected third-party plugin.
