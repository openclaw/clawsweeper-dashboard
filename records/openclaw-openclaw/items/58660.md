## Summary

Keep open: current main handles explicit uninstall and same-id install replacement, but it does not define or apply a legacy-id migration from `openclaw-weixin` to `openclawwechat`; automatic removal of a separately installed plugin needs maintainer product/security judgment.

## What This Changes

Request automatic cleanup and migration guidance when installing `openclawwechat` over an older `openclaw-weixin` plugin leaves stale plugin state and SDK compatibility errors.

## Best Possible Solution

Add an explicit opt-in replacement story, likely plugin-owned metadata/docs plus an install or doctor warning, and keep destructive cleanup behind explicit uninstall or a maintainer-approved migration path.

## Reproduction Assessment

No, not at high confidence. The report has concrete versions and install output, and source inspection shows current main lacks different-id cleanup, but this review did not install the external package or recreate the exact old/new plugin state.

## Solution Assessment

No for automatic same/similar-name deletion as proposed. The maintainable path is explicit replacement metadata or diagnostics plus documented uninstall/doctor cleanup, because guessing from plugin names could remove a user-selected third-party plugin.
