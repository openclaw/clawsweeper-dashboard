## Summary

Keep open: current main documents and catalogs `@tencent-weixin/openclaw-weixin` as the supported WeChat plugin and has same-id install/uninstall cleanup, but it still has no safe different-id migration from `openclaw-weixin` to `openclawwechat`. Automatic similar-name deletion would be a product/security-sensitive installer policy change, not a small bug cleanup.

## What This Changes

The issue requests a safer migration path when installing `openclawwechat` over a legacy `openclaw-weixin` install, including old-plugin cleanup, ID mapping, and upgrade guidance.

## Best Possible Solution

Define an explicit opt-in replacement or migration path, likely through catalog/plugin metadata plus doctor or docs guidance, and keep destructive different-id cleanup behind maintainer-approved metadata.

## Reproduction Assessment

No, not as a high-confidence live repro. Source inspection shows current main lacks different-id replacement cleanup, but the external `openclawwechat` package and legacy install residue were not recreated.

## Solution Assessment

No for automatic same/similar-name deletion as proposed. The safer maintainable direction is explicit replacement metadata, doctor repair, or plugin-owned migration docs because guessing by plugin name could remove a user-selected install.
