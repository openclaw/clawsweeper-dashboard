## Summary

Keep open: current main and the latest release now document and catalog the supported WeChat identity as `openclaw-weixin`, and current installer/doctor code can repair same-id or trusted official package installs, but there is still no trusted cross-id migration or cleanup from the reporter's legacy `openclawwechat`/`openclaw-weixin` residue. The remaining decision is product/security policy: whether OpenClaw should add explicit replacement metadata, doctor repair, and migration docs instead of deleting similar-looking plugin directories.

## What This Changes

The issue requests a safe migration path for installs where legacy `openclaw-weixin` and newer `openclawwechat` plugin identities conflict or leave stale plugin state.

## Best Possible Solution

Add a maintainer-approved migration design: explicit trusted replacement metadata or a doctor-owned repair for the Weixin identity transition, plus user docs that avoid heuristic deletion of unrelated plugins.

## Reproduction Assessment

No, not as a high-confidence live repro. Source inspection shows current main and `v2026.5.22` still key replacement cleanup by exact plugin id, but I did not recreate the legacy external package/residue state in a live install.

## Solution Assessment

No for the proposed automatic similar-name deletion. The safer direction is explicit, trusted replacement metadata plus doctor/uninstall-backed cleanup and docs, after maintainers decide whether this migration belongs in core catalog policy.
