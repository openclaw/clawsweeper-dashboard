## Summary

Keep open: current main supports WeChat through `@tencent-weixin/openclaw-weixin` and cleans up replaced installs only when the plugin id is unchanged, but it still has no trusted cross-id migration from `openclaw-weixin` to `openclawwechat`. The reporter and SonicBotMan both identify the same name/identity confusion; resolving it would need maintainer-approved replacement metadata, doctor repair, or docs rather than automatic similar-name deletion.

## What This Changes

The issue requests safer installation or migration handling when installing `openclawwechat` while an older differently named `openclaw-weixin` plugin remains installed.

## Best Possible Solution

Add a maintainer-approved, explicit replacement path if this should be core-owned: trusted catalog/package metadata identifying old and new ids, a safe doctor or uninstall-backed cleanup flow, and Weixin migration docs that avoid deleting unrelated plugins by name guesswork.

## Reproduction Assessment

No, not as a high-confidence live repro. Source inspection shows current main lacks different-id replacement cleanup for `openclaw-weixin` to `openclawwechat`, but the external package and legacy residue were not recreated against current main.

## Solution Assessment

No for automatic same/similar-name deletion as proposed. The safer maintainable direction is explicit replacement metadata plus doctor/uninstall-backed repair or plugin-owned migration docs because name guessing could remove a valid user install.
