## Summary

Keep open: the PR targets a real direct-session/DM tool-policy gap, but it still needs real behavior proof and has blocking policy-resolution regressions against current main. The companion Feishu group slice at https://github.com/openclaw/openclaw/pull/53496 was closed pending channel-owner direction, which reinforces that this permission-affecting path needs maintainer review rather than cleanup closure.

## What This Changes

The branch adds direct/DM-aware tool-policy resolution, Feishu DM `tools`/`toolsBySender` schema support, empty sender-override fallback behavior, generated config baseline updates, and regression tests.

## Best Possible Solution

Rebase onto current main, preserve `messageProvider` in all direct and group `resolveToolsBySender` calls, update the active config-baseline SHA artifact, keep focused regression tests, and attach redacted real Feishu/direct-session runtime proof.

## Reproduction Assessment

Yes, source-reproducible. Current main only parses group/channel session keys in this resolver path and Feishu DM config only accepts `enabled`/`systemPrompt`, so the requested direct/DM policy behavior is absent without running tests.

## Solution Assessment

No, not as-is. The direction addresses a real gap, but the current branch drops provider-scoped sender matching and uses obsolete generated baseline artifacts; the safer path is a current-main refresh with those contracts preserved.
