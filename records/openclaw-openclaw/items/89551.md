## Summary

Keep open: current main and `v2026.6.8` still ignore the hook-level session-memory model value during LLM slug generation, and the live closing PR should be reviewed before this issue closes.

## What This Changes

The issue requests that `hooks.internal.entries.session-memory.model` be honored when LLM-based session-memory slugs choose their provider/model.

## Best Possible Solution

Keep this issue open while maintainers review https://github.com/openclaw/openclaw/pull/95049; if that PR lands, close this issue as implemented with the merge commit and release provenance.

## Reproduction Assessment

Yes from source inspection: configure `session-memory` with `llmSlug: true` and a `model` value; current main reads the hook config but still calls the slug generator without that model, and the generator resolves the agent default.

## Solution Assessment

Yes for the narrow code path: thread the hook-level model value into slug model resolution while preserving default-model fallback. The product/compatibility decision is whether `session-memory.model` should become the supported public hook contract.
