## Summary

Keep open: current `main` and the latest release still ignore a hook-level `session-memory.model` value during LLM slug generation, and all linked fix PRs are closed unmerged. The remaining blocker is the public config/provider-routing contract, not source ambiguity.

## What This Changes

The issue asks for LLM slug generation in the bundled `session-memory` hook to honor a hook-level model setting instead of always using the agent default model.

## Best Possible Solution

Maintainers should choose the slug model-selection contract explicitly; if hook-level `model` is accepted, resolve it through the alias-aware model-selection path, document the option, and add focused handler/generator tests plus real hook-path proof.

## Reproduction Assessment

Yes from source inspection: a hook entry can contain `model`, but the session-memory handler never forwards it and the generator resolves only the agent default model. I did not run a live `/reset`, so the proof is source-level rather than live-provider proof.

## Solution Assessment

Unclear: forwarding a hook value is a small plausible repair, but it may not be the best permanent product surface. The safer path is a maintainer decision between hook-level model selection and the broader lightweight slug-model policy before code lands.
