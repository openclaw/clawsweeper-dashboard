## Summary

Keep open: current main still ignores the session-memory hook-level model value, and https://github.com/openclaw/openclaw/pull/95049 is the active open fix candidate. The issue should close only after that PR or a replacement lands with the routing behavior and regression coverage.

## What This Changes

The issue reports that `hooks.internal.entries.session-memory.model` is ignored when LLM session-memory slugs are generated, causing slug generation to use the agent default model instead of the hook-level override.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95049 or a narrow replacement that forwards the hook model into slug generation, resolves provider/model refs through existing model-selection helpers, and adds focused regression coverage.

## Reproduction Assessment

Yes by source inspection: configure the session-memory hook with `llmSlug: true` and a `model` value, then the current handler never forwards that model while the slug generator resolves the agent default. I did not run a live `/reset`, so this is source-reproducible rather than locally reproduced.

## Solution Assessment

Yes, if maintainers accept `session-memory.model` as the supported hook-level contract: the narrow fix is to thread that value into the slug generator and preserve default-model fallback. If they do not want that contract, the broader product direction belongs with https://github.com/openclaw/openclaw/issues/33962 instead.
