## Summary

Keep open: current main still ignores a session-memory hook-level model value for LLM slug generation. The source path is clear, but because `session-memory.model` is not documented as a supported option today, maintainers should first decide whether to make that config key public or clarify that slug generation always uses the agent default.

## What This Changes

The issue requests that `hooks.internal.entries.session-memory.model` override the model used when `llmSlug` generates memory filename slugs.

## Best Possible Solution

Decide whether `hooks.internal.entries.session-memory.model` is a supported public option; if yes, resolve it through existing model-selection helpers, pass it into slug generation, fall back to the agent default when absent, and cover the behavior in docs and focused tests.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: enable `llmSlug`, set a hook-level model value, and the handler still calls the generator without that value while the generator resolves the agent default. I did not run a live provider call in this read-only review.

## Solution Assessment

Unclear until maintainers confirm the config contract. If the key should be public, the narrow maintainable fix is to resolve and pass an optional hook model override into the existing slug-generator path with fallback to the current agent-default behavior.
