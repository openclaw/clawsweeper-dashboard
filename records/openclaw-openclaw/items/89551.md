## Summary

Keep open: current `main` and the latest release still ignore a hook-level `session-memory.model` value during LLM slug generation, and the linked fix PRs are closed unmerged. The remaining blocker is product/config-contract ownership because `model` is not documented today as a supported `session-memory` hook option.

## What This Changes

The issue asks OpenClaw to honor a session-memory hook-level `model` value when choosing the provider/model for LLM slug generation.

## Best Possible Solution

Maintainers should choose the slug model-selection contract explicitly: either support and document `hooks.internal.entries.session-memory.model` with focused tests, or fold this into a generic lightweight-task/slug-model policy instead of adding a hook-specific config surface.

## Reproduction Assessment

Yes from source inspection: if a hook entry contains `model`, current main never forwards it and the generator resolves the agent default model internally. I did not run a live `/reset`, but the code path is direct and covered enough for source-level reproduction.

## Solution Assessment

Unclear: passing the hook value through is a small plausible fix, but it may not be the best product surface. The maintainable path is to decide whether the supported contract is a hook-level override or the broader lightweight slug-model policy already tracked separately.
