## Summary

Keep open. Current main still rejects existing `<qqmedia>` local files outside the guarded OpenClaw media roots, and unrestricted auto-mirroring would change a deliberate local-file upload security boundary.

## What This Changes

Request that QQ Bot copy existing local `<qqmedia>` files from outside its media directory into `.openclaw/media/qqbot/` before sending.

## Best Possible Solution

Define a maintainer-approved QQ Bot staging policy: keep the current guarded media-root requirement, or add a constrained mirror helper with canonicalization, symlink/traversal protection, size checks, docs, and regression tests.

## Reproduction Assessment

Yes. The current source trace from `<qqmedia>` parsing to `resolveOutboundMediaPath` shows an existing workspace file outside OpenClaw media roots reaches the rejection path, and the platform test covers outside-file blocking.

## Solution Assessment

No for unrestricted auto-mirroring. A safer solution needs an explicit allowlist or staging contract rather than copying any existing readable local path into QQ Bot upload storage.
