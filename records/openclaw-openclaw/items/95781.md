## What This Changes

The issue reports that outbound transcript mirror appends ignore configured `session.store` when callers pass `config` but no explicit `storePath`.

## Best Possible Solution

Land a narrow transcript append fallback fix that preserves explicit `storePath` priority, honors `config.session.store` when omitted, and covers fixed and `{agentId}` templated stores.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: configure `session.store`, create outbound session metadata in that store, then send a mirrored outbound reply whose append caller passes `config` but no `storePath`. Current main still falls back to the default store at `src/config/sessions/transcript.ts:399`.

## Solution Assessment

Yes, the best fix is to centralize configured-store fallback inside transcript append rather than adding store-path plumbing to every outbound caller. The linked PR is the current candidate, but this issue should remain open until that candidate is reviewed and merged or replaced.
