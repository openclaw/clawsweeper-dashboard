## Label Justifications

- `P2`: This is a normal-priority correctness bug affecting custom session.store users without breaking channel delivery or core availability.
- `impact:data-loss`: Mirrored assistant replies can be skipped or written to the wrong store, losing transcript bookkeeping for affected custom-store setups.
- `impact:session-state`: The affected behavior corrupts session transcript fidelity for outbound mirror entries.

## Summary

Keep open: current main and the latest release still resolve transcript mirror appends through the default session store when callers omit storePath, and the linked open closing PR is the right place to finish the fix before this issue closes.

## What This Changes

The issue requests that outbound transcript mirror appends honor configured session.store, including {agentId} store templates, when callers pass config without an explicit storePath.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95782 if its PR review remains clean, then close this issue as implemented.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: the append helper ignores config.session.store when storePath is omitted, while the outbound mirror callers pass config without storePath. I did not execute a runtime repro because this review is read-only.

## Solution Assessment

Yes, the narrowest maintainable fix is to resolve the configured store in the central transcript append fallback rather than adding storePath plumbing to each outbound caller. The linked PR appears to take that central-boundary approach and adds focused regression coverage.
