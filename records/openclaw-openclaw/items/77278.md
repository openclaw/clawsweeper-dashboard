## Summary

Keep open: current main and v2026.6.10 still have exec approval decisions and TUI elevated-level controls, but no TUI masked sudo-password prompt or secure password-to-stdin handoff. The remaining work is a security-sensitive product decision, not cleanup-close material.

## What This Changes

Add a TUI-only masked sudo password prompt after exec approval so the password can be passed to sudo without entering model-visible context.

## Best Possible Solution

Define a TUI-only sudo-auth contract that keeps the password out of model context, durable Gateway state, logs, transcripts, and approval records, then implement it through the existing exec approval/runtime boundary with focused tests and docs.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks the requested masked sudo prompt and password-to-stdin handoff.

## Solution Assessment

Unclear until maintainers choose the security contract. The TUI-only direction is plausible, but the password must bypass approval resolve payloads, durable state, logs, transcripts, and model-visible tool results.
