## Summary

Keep open. Current main and the latest release still lack a TUI-only masked sudo password prompt or password-to-stdin handoff; the request remains a security-sensitive feature that needs maintainer product/security direction before implementation.

## What This Changes

Request a TUI-only interactive sudo password prompt that runs after exec approval and passes the password directly to sudo stdin without exposing it to the model.

## Best Possible Solution

Define a TUI-only sudo-auth contract that keeps the password out of model context, durable Gateway state, logs, and approval records, then implement it through the existing exec approval boundary with targeted protocol, TUI, exec runtime, docs, and regression coverage.

## Solution Assessment
