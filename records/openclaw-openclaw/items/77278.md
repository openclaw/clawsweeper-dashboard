## Summary

Keep open. Current main and the latest release still have exec approval and elevated-mode controls, but no TUI masked sudo-password prompt or secure password-to-stdin handoff; this remains a security-sensitive feature decision.

## What This Changes

Request a TUI-only masked sudo password prompt after exec approval that pipes the password to sudo without exposing it to the model.

## Best Possible Solution

Define a TUI-only sudo-auth contract that keeps the password out of model context, durable Gateway state, logs, transcripts, and approval records, then implement it through the existing exec approval boundary with focused tests and docs.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main lacks the requested masked sudo prompt and password-to-stdin handoff.

## Solution Assessment

Unclear until maintainers choose the security contract. The TUI-only direction is plausible, but the password must bypass approval payloads, durable state, logs, transcripts, and model-visible tool results.
