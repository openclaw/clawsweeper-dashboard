Request a TUI-only inline, masked sudo password prompt after exec approval that can feed sudo stdin without exposing the password to the model.

## Best Possible Solution

Define a TUI-only sudo-auth contract that keeps the password out of model context, durable Gateway state, logs, and approval records, then implement it through the existing exec approval boundary with targeted protocol, TUI, exec runtime, and docs coverage.
