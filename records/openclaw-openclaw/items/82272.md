## Summary

Keep open for normal PR review and merge handling. The patch targets a real regression with a source-backed root cause, sufficient after-fix terminal proof, and no blocking correctness or security findings from this review.

## What This Changes

Adds an embedded GatewayRequestContext builder/test and installs it at agentCliCommand entry so local and embedded fallback agent runs can dispatch subagent completion announces in process.

## Best Possible Solution

Land a focused fix that gives embedded local agent runs a valid gateway request context while leaving the dispatcher scope contract intact.

## Reproduction Assessment

Yes. Current main's --local path does not install a fallback context, while the subagent announce path dispatches agent in-process and the dispatcher throws when no scope or fallback exists; the linked issue provides concrete commands and logs.

## Solution Assessment

Yes. Installing an embedded fallback context at the CLI entry is the narrowest fix for local and embedded fallback runs without weakening dispatchGatewayMethodInProcessRaw; a direct agentCliCommand regression test would further strengthen coverage but is not a blocking design issue.
