  - body: The new fallback only runs inside the bare plugin-id `officialExternalPlan` branch. The linked regression and current docs tell users to run `openclaw plugins install @openclaw/searxng-plugin`, but scoped package specs are rejected by `resolveOfficialExternalInstallPlanBeforeNpm` and still go through the lower npm-only `officialNpmTrust` path, so that command will keep failing on the reservation package.   - confidence: 0.94
- **[P2] Keep the expected plugin id on ClawHub fallback:** `src/cli/plugins-install-command.ts:1300-1305`
  - body: The official npm install path enforces `expectedPluginId`, but the new ClawHub fallback calls `installPluginFromClawHub` without the same guard. If the fallback artifact resolves to a different manifest id, the installer can persist that id instead of rejecting the package identity mismatch this recovery path is meant to avoid. Pass `expectedPluginId: officialExternalPlan.pluginId` and cover a mismatch case.
  - confidence: 0.88
