## Summary

Keep this PR open: current main does not implement namespaced `/models` grouping, and the PR is useful but still incomplete because Telegram's interactive model-list path can bypass the grouped text and its pagination callbacks cannot preserve a provider/group scope.

## What This Changes

The branch adds `/models <provider>/<group>` parsing, grouped text output for namespaced provider catalogs, scoped listings, and colocated auto-reply command tests.

## Best Possible Solution

Finish the namespace grouping feature so text and interactive channel surfaces behave consistently: either make the channel adapter and Telegram callbacks namespace-aware, or intentionally bypass interactive rendering for grouped and scoped `/models` views. The finished PR should include Telegram coverage for grouped catalogs and scoped pagination plus docs and changelog updates for the new syntax.

## Reproduction Assessment

Yes. A high-confidence static reproduction is Telegram `/models litellm` on the PR branch with multiple namespaced groups: the command can return provider-only interactive channel data before grouped text is built, and scoped pagination cannot represent the group because callbacks encode only provider and page.

## Solution Assessment

No. The text grouping implementation is close, but the current PR path is not the best complete solution until namespace scope is preserved through the interactive adapter/callback path or the grouped/scoped cases deliberately use text-only rendering.
