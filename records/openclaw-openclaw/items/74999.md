## Summary

Keep this PR open: current main does not implement namespaced `/models` grouping, and the PR still has a blocking Telegram/native-command path where interactive model-list rendering can bypass or lose the new namespace scope before merge.

## What This Changes

The PR adds namespaced `/models <provider>/<group>` parsing, grouped text output for aggregator-style catalogs, scoped model listings, and colocated tests for that behavior.

## Best Possible Solution

Finish the namespace grouping feature with a consistent channel story: grouped and scoped views should either preserve the prefix through Telegram/native callbacks and pagination, or deliberately render as text-only command drilldowns. The final PR should also update slash-command docs and add the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on the PR head: Telegram `/models litellm` with multiple namespaced groups can return `buildModelsListChannelData` before grouped text is built, and scoped pagination cannot encode the group prefix.

## Solution Assessment

No. The text fallback implementation is close, but the PR is not the best complete fix until interactive channel rendering either becomes namespace-aware or is bypassed for grouped/scoped views; docs and changelog also need to match the new command behavior.
