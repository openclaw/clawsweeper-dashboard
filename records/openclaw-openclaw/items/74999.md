## Summary

Keep this PR open: current main does not implement namespaced `/models` grouping, but the proposed patch still breaks the Telegram interactive path and is missing release/docs follow-through for a user-facing command change.

## What This Changes

The PR adds `/models <provider>/<group>` parsing, grouped namespaced model listings for aggregator catalogs, scoped text listings, and colocated auto-reply tests.

## Best Possible Solution

Keep the namespace grouping feature, but make grouped/scoped listings preserve the prefix across channel rendering and pagination, then document and changelog the new command syntax.

## Reproduction Assessment

Yes. A high-confidence static reproduction is Telegram `/models litellm` with multiple namespaced groups: PR head can return provider-only interactive channel data before group text; scoped pagination then loses `litellm/deepseek` because callbacks encode only `litellm` and page.

## Solution Assessment

No. The text grouping helpers are a reasonable start, but the proposed implementation is not complete until interactive rendering either becomes namespace-aware or is bypassed for grouped/scoped listings.
