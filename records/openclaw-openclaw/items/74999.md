## Summary

Keep this PR open: current main does not implement namespaced `/models` grouping, and PR head still has a blocking interactive-channel path where Telegram can return provider-only model list data before the new grouped/scoped text path runs.

## What This Changes

The PR adds `/models <provider>/<group>` parsing, grouped namespaced model listings, and colocated auto-reply tests while preserving `/models <provider> all` as the flat listing escape hatch.

## Best Possible Solution

Keep the feature, but make grouped/scoped listings preserve namespace state across Telegram/channel rendering, then add the user-facing docs and changelog entry before merge.

## Reproduction Assessment

Yes. Source inspection of PR head shows Telegram `/models litellm` with multiple namespaced groups can return provider-only interactive data before the grouped text path, and scoped pagination cannot encode `litellm/deepseek`.

## Solution Assessment

No. The text grouping helpers are a reasonable start, but the PR is not complete until interactive rendering preserves the namespace scope or is deliberately skipped for grouped/scoped views, with docs and changelog updated.
