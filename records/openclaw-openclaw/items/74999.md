## Summary

Keep open: the feature direction is useful and current main does not implement scoped namespaced `/models` browsing, but this PR still has a blocking Telegram/channel rendering bug and lacks after-fix real behavior proof.

## What This Changes

The PR adds `/models <provider>/<group>` parsing, grouped text output for namespaced aggregator model catalogs, and focused auto-reply tests.

## Best Possible Solution

Keep the feature direction, but preserve namespace scope through channel rendering and pagination or deliberately bypass interactive adapters for grouped/scoped views, then document `/models <provider>/<group>`.

## Reproduction Assessment

Yes. Source inspection of PR head gives a high-confidence path: Telegram `/models litellm` with multiple namespaced groups can return `buildModelsListChannelData` before grouped text is built, and callback pagination cannot encode the group.

## Solution Assessment

No. The grouping helpers are a reasonable start, but the maintainable fix must preserve namespace scope across the channel adapter/callback path or skip interactive rendering for scoped/grouped views.

## Review Findings

Overall confidence: 0.9
