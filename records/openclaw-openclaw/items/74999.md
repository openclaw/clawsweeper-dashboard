## Summary

## What This Changes

The PR adds `/models <provider>/<group>` parsing and grouped text rendering for namespaced provider catalogs, with focused command-handler tests.

## Best Possible Solution

Preserve namespace scope through `/models` channel rendering and callback pagination, or deliberately make grouped/scoped views text-only, then document the syntax and attach redacted real behavior proof.

## Reproduction Assessment

Yes for the blocking PR bug by source inspection. On PR head, Telegram/channel rendering can return before grouped text is built, and current main's adapter/callback contract only carries provider/page state.

## Solution Assessment

No. The text grouping helpers are a reasonable start, but the maintainable solution must keep namespace scope through interactive rendering or explicitly bypass that path for grouped/scoped listings.
