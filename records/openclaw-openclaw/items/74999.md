## Summary

Keep open: the PR is useful, but PR head still returns interactive channel data before the new grouped text path, so Telegram can miss the grouping behavior and scoped pagination still has no namespace-preserving callback path.

## What This Changes

The PR adds `/models <provider>/<group>` parsing and grouped text output for namespaced model catalogs, with auto-reply tests for grouped and scoped listings.

## Best Possible Solution

Keep the feature, but make grouping/scoped rendering happen before channel adapters or extend the adapter/callback contract to carry namespace scope, then add docs and a changelog entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction is Telegram `/models litellm` with multiple namespaced groups on PR head: the command can return `buildModelsListChannelData` before the grouped text path, and scoped interactive pagination has only provider/page callbacks.

## Solution Assessment

No. The text helpers are a reasonable start, but the implementation is not complete until interactive rendering preserves or intentionally bypasses namespace scope and the user-facing syntax is documented.
