## Summary

Keep this PR open: it adds useful scoped `/models` behavior that current `main` does not have, but the current patch still has a Telegram/channel rendering bug and lacks real after-fix behavior proof.

## What This Changes

The PR adds `/models <provider>/<group>` parsing, groups namespaced aggregator model catalogs, and adds focused auto-reply tests for grouped and scoped listings.

## Best Possible Solution

Preserve namespace scope through the channel adapter/callback path or intentionally use text-only output for grouped/scoped views, update slash-command docs, then require real Telegram proof before merge.

## Reproduction Assessment

Yes. Source inspection of PR head gives a high-confidence path: Telegram `/models litellm` with multiple namespaced groups can return interactive channel data before group text, and callback pagination only encodes provider/page.

## Solution Assessment

No. The feature direction is reasonable, but this patch should preserve `modelPrefix` through channel rendering and pagination or deliberately bypass the interactive adapter for grouped/scoped views, and it also needs docs plus real behavior proof.
