## Summary

Keep this PR open, but it is not ready to merge: the feature direction is useful, yet PR head can still bypass the new grouped/scoped text on Telegram and the external PR still lacks real after-fix behavior proof.

## What This Changes

The PR adds `/models <provider>/<group>` parsing, grouped text output for namespaced model catalogs, and focused command-handler tests.

## Best Possible Solution

Implement the namespace drilldown with scoped state preserved through channel rendering and pagination, or make grouped/scoped views deliberately text-only, then document the new syntax.

## Reproduction Assessment

Yes for the blocking PR bug. Source inspection of PR head shows Telegram/channel rendering can return before grouped text is built, and the existing Telegram callback contract has no model-prefix field to preserve scoped pagination.

## Solution Assessment

No. The text grouping helpers are a reasonable start, but the maintainable solution must preserve namespace scope through the channel adapter/callback path or intentionally bypass interactive rendering for grouped/scoped listings, with docs for the new syntax.
