## Summary

Keep open: this is a focused gateway availability fix with useful real-world evidence, but the current classifier can strip more OpenClaw ExecStart overrides than the PR’s stated scope because it treats the Node runtime as the `node` subcommand.

## What This Changes

The PR adds systemd drop-in cleanup after service staging so stale OpenClaw ExecStart overrides are removed while unrelated drop-in settings are preserved.

## Best Possible Solution

Keep the drop-in cleanup, but classify the OpenClaw subcommand explicitly so only gateway/node service overrides are stripped, with regression coverage for a Node-launched non-target OpenClaw command.

## Reproduction Assessment

Yes from source and supplied live output: current main rewrites only the main unit, while systemd drop-ins can reset and override ExecStart, and the PR body shows a stale override launching `2026.5.12` after a `2026.5.18` update.

## Solution Assessment

No, not quite: cleaning stale managed drop-ins is the right direction, but the implementation should identify the OpenClaw subcommand instead of matching `node` anywhere in the joined command text.
