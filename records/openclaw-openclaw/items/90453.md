The PR adds memory-core safe-reindex synchronization, stable status snapshots, post-swap metadata rewriting, and regression coverage for search/status reads during full reindex.

## Best Possible Solution

Land one canonical memory-core reindex concurrency fix that keeps search/status off the temp DB, with the wait semantics explicitly accepted and coordinated with adjacent metadata/stale-handle fixes.

## Solution Assessment
