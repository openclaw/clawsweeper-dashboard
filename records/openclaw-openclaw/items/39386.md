## Summary

Keep open. The branch targets a real current-main gateway delivery gap, but the proposed fallback can permanently cache a missing spawnedBy lookup after an early child event, so later parent delivery can still be suppressed until subscription churn clears the cache; the branch is also dirty against current main and must preserve the SerializedEventPayload raw-send path. The maintainer label and member author also make this explicit maintainer-owned work, not a cleanup-close candidate.

## What This Changes

The branch changes gateway node subscription routing to walk spawnedBy lineage on direct session-key misses, adds a focused gateway regression test, and records the fix in the changelog.

## Best Possible Solution

Land a rebased gateway-local spawnedBy fallback that caches only safe positive lineage results, preserves current raw node payload serialization, and covers the early-miss then later-lineage-visible case.

## Reproduction Assessment

Yes from source inspection and the linked logs: current main only looks up exact node subscription keys, so a child session key without a direct subscriber returns before spawnedBy lineage can be considered. I did not run tests because this review was read-only.

## Solution Assessment

No as currently patched: resolving upward through spawnedBy is a reasonable narrow fix, but permanent negative caching makes the implementation unsafe for early child events and the branch needs a clean rebase onto current payload serialization.

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Avoid caching missing spawnedBy lineage:** `src/gateway/server-node-subscriptions.ts:157`
  - body: If the first child event arrives before `loadSessionEntry(current)` returns a `spawnedBy`, this stores `null` for that child key. Later events then hit the cache and never re-read the session entry, so a parent subscriber can keep missing the spawned run until subscription churn clears the cache. Avoid caching negative lineage results, or invalidate them when lineage is written, and cover early-miss-then-lineage-visible in the test.
  - confidence: 0.9
