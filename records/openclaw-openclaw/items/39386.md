## Label Justifications

- `P2`: This is a normal-priority gateway delivery bugfix with limited blast radius but real node fanout contract risk.
- `merge-risk: 🚨 compatibility`: Existing parent-session node subscribers would start receiving spawned child-session events they did not directly subscribe to.
- `merge-risk: 🚨 message-delivery`: The changed subscription manager controls gateway node event delivery, and unresolved-lineage caching can keep child-session events suppressed.

## Summary

Keep open: current main still uses exact-key gateway node subscription fanout, and this maintainer-labeled draft is a plausible focused fix, but it is not merge-ready because unresolved lineage misses are cached as `null` and can preserve the same child-session message loss after a late `spawnedBy` write or transient session-load miss.

## What This Changes

The PR adds a `spawnedBy` ancestor fallback to gateway node subscription delivery, adds a focused unit test, and edits `CHANGELOG.md`.

## Best Possible Solution

Land a refreshed gateway fix that explicitly accepts descendant fanout or chooses child subscription propagation, avoids negative caching or adds invalidation, covers late lineage, and preserves current payload serialization.

## Reproduction Assessment

Yes, source-level. Current main registers exact `chat.subscribe` keys and `sendToSession` fans out only to that exact key, while subagent spawn persists `spawnedBy` on the child session.

## Solution Assessment

No as submitted. Walking `spawnedBy` at send time is an acceptable narrow fix compared with spawn-time subscription copying, but the implementation must avoid negative caching and needs explicit gateway-owner acceptance of descendant fanout.
