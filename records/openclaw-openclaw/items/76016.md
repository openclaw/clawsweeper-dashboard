## Summary

Keep this PR open. Current main still rejects native memory-wiki markdown links that include the `.md` relative path, and the proposed one-line validator expansion matches that bug, but the branch needs the required changelog entry before it is release-ready.

## What This Changes

This PR adds `page.relativePath` to memory-wiki `broken-wikilink` valid targets so `.md` relative markdown links can resolve during lint.

## Best Possible Solution

Land the narrow valid-target expansion with a focused regression test and a single-line changelog entry, while leaving #70986 to track the broader resolver and extraction improvements.

Yes. Source inspection gives a high-confidence path: native `formatWikiLink` emits `[title](relativePath.md)`, `extractWikiLinks` keeps that `.md` target, and current `collectBrokenLinkIssues` validates only extension-stripped forms.

## Solution Assessment

Yes for the code change. Adding `page.relativePath` is the narrowest maintainable fix for `.md` relative targets; the PR should be completed with release-note and regression coverage rather than expanded into #70986.
