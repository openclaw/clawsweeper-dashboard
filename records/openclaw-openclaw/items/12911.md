## Summary

Keep open. Current main has browser Talk for registered realtime voice providers, but xAI/Grok is still not registered or documented as available for realtime voice, and maintainer comments plus #73032's stated scope leave the `/chat` provider surface unimplemented.

## What This Changes

Request adds a Grok/xAI-powered Live voice loop to the `/chat` UI with browser speech input, spoken agent summaries, and continuous voice replies.

## Best Possible Solution

Implement xAI/Grok as a realtime voice provider on the plugin surface, wire it into `/chat` Talk through the existing provider resolver or an additive contract extension, and update focused tests plus xAI/Control UI docs.

## Reproduction Assessment

Not applicable as a feature request rather than a bug report. The missing capability is source-verifiable on current main because `/chat` Talk resolves registered realtime voice providers while xAI registers speech and realtime transcription but not realtime voice.

## Solution Assessment

Yes, keeping this open is the best current resolution. The narrow maintainable direction is provider-plugin work in `extensions/xai`, not closing this as implemented or treating #73032's `/voiceclaw/realtime` adapter as sufficient for `/chat`.
