## Summary

Keep open. Current main has generic browser Talk for `/chat`, but the requested xAI/Grok realtime voice provider is still absent from the Talk provider surface, and the related open xAI PR explicitly targets a different `/voiceclaw/realtime` path.

## What This Changes

Request to add an xAI/Grok-powered Live voice loop in `/chat` with browser microphone input, in-chat spoken agent responses, transcription back into user messages, and text fallback.

## Best Possible Solution

Keep this as the tracker for a maintainer-approved xAI `RealtimeVoiceProviderPlugin` that works with `/chat` Talk, updates provider docs and focused Talk tests, and includes redacted live browser Talk proof with Grok voice.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main cannot satisfy xAI/Grok `/chat` Talk because xAI is not registered as a realtime voice provider and the docs still mark realtime voice as unexposed.

## Solution Assessment

Yes, keeping this open is the best current outcome. The maintainable implementation path is an xAI provider on the existing Talk realtime provider-plugin surface, with any transport-contract changes settled by maintainers before implementation.
