## Summary

Keep open. Current main has generic browser Talk in `/chat`, but xAI/Grok is still absent from the realtime voice provider surface, the xAI plugin/docs still mark realtime voice as unexposed, and the related open work at https://github.com/openclaw/openclaw/pull/73032 targets a different `/voiceclaw/realtime` surface rather than `/chat` Talk.

## What This Changes

Request to add a `/chat` Live/Talk voice loop powered by xAI/Grok, including browser mic input, spoken assistant summaries, transcript handoff, and Live-mode UI controls.

## Best Possible Solution

Keep this as the tracker for a maintainer-approved xAI `RealtimeVoiceProviderPlugin` that works with `/chat` Talk, registers through `extensions/xai`, updates provider docs and focused tests, and includes redacted live browser Talk proof with Grok voice.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main cannot satisfy xAI/Grok `/chat` Talk because xAI is not registered as a realtime voice provider and docs still mark realtime voice as unexposed.

## Solution Assessment

Yes, keeping this open is the best current outcome. The maintainable implementation should use the existing Talk/realtime voice provider surface, with any xAI transport-contract changes settled before implementation.
