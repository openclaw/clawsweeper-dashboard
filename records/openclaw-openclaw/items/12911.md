## Summary

Keep open. Current main implements generic browser Talk in `/chat`, but the requested xAI/Grok realtime voice provider is still not registered or exposed, and the related open xAI work targets a different `/voiceclaw/realtime` surface.

## What This Changes

Add xAI/Grok-powered Live Voice Mode to the `/chat` UI so browser microphone input and agent audio can run as a continuous voice loop without a phone call.

## Best Possible Solution

Implement xAI/Grok as a registered `RealtimeVoiceProviderPlugin` for `/chat` Talk, reusing the existing provider contract where compatible and adding only an additive transport seam if xAI requires it.
