Keep open. Current main has generic browser Talk in `/chat`, but xAI/Grok realtime voice is still not registered or exposed on the Talk realtime provider surface, and the linked open xAI work explicitly targets `/voiceclaw/realtime` rather than this `/chat` provider path.

Request xAI/Grok Live Voice Mode in the `/chat` UI so browser microphone input and spoken agent responses can loop through the chat experience.

## Best Possible Solution

Keep this as the tracker for a maintainer-approved xAI `RealtimeVoiceProviderPlugin` for `/chat` Talk, with matching xAI manifest/docs/UI/tests and an explicit decision on sharing code with the adjacent `/voiceclaw/realtime` work.
