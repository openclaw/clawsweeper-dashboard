## What This Changes

The issue requests a `/chat` Live voice mode that uses xAI/Grok realtime voice for browser microphone input, spoken agent summaries, transcription, and continued voice conversation.

## Best Possible Solution

Keep this open as the tracker for an xAI `RealtimeVoiceProviderPlugin` on the existing Talk realtime provider seam, with docs and focused `/chat` Talk tests once maintainers confirm the provider/transport direction.

## Reproduction Assessment

Yes, source-level: current main exposes browser and relay Talk through registered realtime voice providers, while xAI has no realtime voice provider registration and the docs explicitly say realtime voice is not exposed.

## Solution Assessment

Yes, the maintainable path is the existing provider-plugin seam rather than a new `/chat`-specific or `/voiceclaw/realtime` stack; the remaining product decision is which xAI transport shape to support and how to prove it live.
