## Summary

Keep open. Current main has generic browser Talk in `/chat`, but the requested xAI/Grok realtime voice provider is still not registered or exposed, and the adjacent open xAI work targets `/voiceclaw/realtime` or voice-call rather than completing the `/chat` provider-plugin path.

## What This Changes

Add an xAI/Grok-backed Live Voice Mode to the `/chat` UI so users can have a continuous spoken conversation with the agent in the browser.

## Best Possible Solution

Implement xAI/Grok as a provider-owned `RealtimeVoiceProviderPlugin` for `/chat` Talk, reusing the existing provider contract where compatible and adding only a documented additive transport seam if xAI requires it.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug. Source inspection on current main verifies that the requested xAI/Grok realtime voice provider for `/chat` Talk is still missing.

## Solution Assessment
