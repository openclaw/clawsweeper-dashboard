## Summary

Keep open. Current main has generic /chat Talk support through registered realtime voice providers, but xAI/Grok is still documented and implemented as speech/STT/search/media only, with no realtime voice provider registration for the /chat Talk surface.

## What This Changes

The issue requests a Grok/xAI-backed Live voice loop in the /chat UI with browser microphone input, transcription, spoken assistant replies, and continued conversational turns.

## Best Possible Solution

Implement xAI/Grok as a registered RealtimeVoiceProviderPlugin for /chat Talk, using the existing provider contract when possible and adding only an additive seam if xAI's transport requires it.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. The missing capability is source-verifiable because /chat Talk resolves registered realtime voice providers while xAI registers speech and realtime transcription but not realtime voice.

## Solution Assessment

Yes: keeping this open is the best current outcome. The narrow maintainable path is xAI provider-plugin work for /chat Talk, while #73032 is adjacent /voiceclaw/realtime work and not a replacement.
