## Summary

Keep open. Current main has generic browser Talk for `/chat`, but the requested xAI/Grok realtime voice provider is still absent from the Talk provider surface, and the related open xAI work explicitly targets `/voiceclaw/realtime` rather than the `/chat` provider-plugin path.

## What This Changes

Add an xAI/Grok-powered live voice loop to the `/chat` UI so users can talk with the agent from web chat.

## Best Possible Solution

Keep this as the tracker for a maintainer-approved xAI `RealtimeVoiceProviderPlugin` that works with `/chat` Talk, updates provider docs and focused Talk/xAI tests, and includes redacted live browser Talk proof with Grok voice.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current `main` cannot satisfy xAI/Grok `/chat` Talk because xAI is not registered as a realtime voice provider and docs still mark realtime voice as unexposed.

## Solution Assessment

Yes, keeping this open is the best current outcome. The maintainable path is an xAI provider on the existing realtime voice/Talk provider surface, with any transport-contract changes settled by maintainers before implementation.
