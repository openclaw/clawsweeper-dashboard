## Summary

Keep open. Current main has generic browser Talk in `/chat`, but xAI/Grok realtime voice is still not registered on the Talk realtime provider surface, and the live discussion explicitly says the adjacent `/voiceclaw/realtime` work does not close this `/chat` request.

## What This Changes

Add Grok/xAI live voice mode to the `/chat` web UI so users can speak with the agent through the browser instead of using a separate phone call.

## Best Possible Solution

Implement xAI/Grok as a provider-owned `RealtimeVoiceProviderPlugin` for `/chat` Talk, with any shared protocol code or additive transport seam decided before coding.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug. Source inspection gives high confidence that current main cannot satisfy xAI/Grok `/chat` Talk because xAI is not registered as a realtime voice provider.

## Solution Assessment

Yes, keeping it open is the best current outcome. The maintainable path is an xAI realtime voice provider on the existing Talk provider-plugin surface, with an additive transport seam only if xAI cannot fit the current browser-session or Gateway relay contracts.
