## Summary

Keep open. Current main has generic browser Talk in `/chat`, but the requested xAI/Grok realtime voice provider is still not registered or documented for that Talk path, and the related open xAI work targets other voice surfaces rather than closing this feature request.

## What This Changes

Request to add Grok/xAI Live Voice Mode to the `/chat` UI as a real-time browser voice conversation loop.

## Best Possible Solution

Implement xAI/Grok as a provider-owned `RealtimeVoiceProviderPlugin` for `/chat` Talk, with a documented transport seam only if xAI cannot fit the existing browser-session or Gateway relay contracts.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug. Source inspection gives high confidence that current main cannot satisfy the Grok `/chat` Talk request because xAI is not registered as a realtime voice provider.

## Solution Assessment

Yes, keeping it open is the best current outcome. The maintainable path is an xAI realtime voice provider on the existing Talk provider-plugin surface, or a maintainer-approved additive transport seam if xAI needs one.
