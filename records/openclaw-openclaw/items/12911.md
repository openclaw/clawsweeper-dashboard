## Summary

Keep open. Current main has generic `/chat` Talk support through registered realtime voice providers, but xAI/Grok is still not registered or documented as a realtime voice provider, and maintainer comments explicitly kept this as the `/chat` xAI provider tracker.

## What This Changes

Request to add a `/chat` Live voice loop powered by xAI/Grok realtime voice, including browser mic input, in-chat audio output, transcription, and Talk-style conversation continuity.

## Best Possible Solution

Implement xAI/Grok as a registered `RealtimeVoiceProviderPlugin` for `/chat` Talk, reusing the existing provider contract where possible and adding only an additive transport seam if xAI requires it.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. The missing capability is source-verifiable because `/chat` Talk resolves registered realtime voice providers while xAI registers speech and realtime transcription but not realtime voice.

## Solution Assessment

Yes, keeping this open is the best current outcome. The narrow implementation path is xAI provider-plugin support for `/chat` Talk, while #73032 remains adjacent `/voiceclaw/realtime` work rather than a replacement.
