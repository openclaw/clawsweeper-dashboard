## Summary

Keep open. Current main has generic browser Talk in `/chat`, but xAI/Grok realtime voice is still not registered or exposed for that surface, the xAI docs still mark realtime voice as unexposed, and the related open PR explicitly targets a different `/voiceclaw/realtime` path.

## What This Changes

The issue requests a `/chat` Live voice loop that uses xAI/Grok voice for browser microphone input, in-chat assistant audio, and continuing spoken turns.

## Best Possible Solution

Keep this as the tracker for a maintainer-approved xAI `RealtimeVoiceProviderPlugin` that works with `/chat` Talk, or make an explicit ClawHub/core scope decision before implementation.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence current main cannot satisfy xAI/Grok `/chat` Talk because xAI is not registered as a realtime voice provider and docs still mark realtime voice as unexposed.

## Solution Assessment

Yes for keeping it open. The maintainable path should use the existing Talk/realtime voice provider registry, with any xAI transport or ClawHub-versus-bundled decision settled before implementation.
