## Summary

Keep open. Current main has browser Talk for registered realtime voice providers, but xAI/Grok still lacks a realtime voice provider registration/manifest contract, and maintainer discussion plus #73032 leave the /chat surface unresolved.

## What This Changes

The issue requests a Grok/xAI-backed live voice loop in the /chat UI, including spoken assistant summaries, browser microphone input, transcription, and continued voice conversation.

## Best Possible Solution

Implement xAI/Grok as a registered realtime voice provider for /chat Talk, extending the provider-plugin contract only if the upstream session shape requires it, and update focused tests plus xAI/Control UI docs.

## Reproduction Assessment

Not applicable as a feature request rather than a bug report. The missing capability is source-verifiable on current main because /chat Talk resolves registered realtime voice providers while xAI registers speech and realtime transcription but not realtime voice.

## Solution Assessment

Yes, keeping this open is the best current outcome. The narrow maintainable path is provider-plugin work in extensions/xai, while #73032's /voiceclaw/realtime adapter is adjacent but not sufficient for /chat Talk.
