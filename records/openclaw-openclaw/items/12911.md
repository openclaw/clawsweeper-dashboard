## Summary

Keep open. Current `main` has generic browser Talk in `/chat`, but xAI/Grok realtime voice is still not registered or exposed on the Talk realtime provider surface, and the linked xAI work is an open `/voiceclaw/realtime` PR that explicitly does not implement this `/chat` provider path.

## What This Changes

The issue requests a Grok/xAI Live Voice Mode in `/chat` that plays agent speech in-browser, accepts browser microphone input, transcribes it, and continues the chat loop.

## Best Possible Solution

Keep this as the tracker for a maintainer-approved xAI `RealtimeVoiceProviderPlugin` for `/chat` Talk, with matching xAI manifest/docs/UI/tests and an explicit routing decision for any shared code with the adjacent `/voiceclaw/realtime` work.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence current `main` cannot satisfy xAI/Grok `/chat` Talk because xAI is not registered as a realtime voice provider and docs still mark realtime voice unexposed.

## Solution Assessment

Yes for keeping it open. The maintainable path is to use the existing Talk realtime voice provider registry, with maintainer judgment on xAI transport shape and how to coordinate with the adjacent `/voiceclaw/realtime` PR.
