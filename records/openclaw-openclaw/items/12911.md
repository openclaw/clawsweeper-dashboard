## Summary

Keep open. Current main has generic browser Talk in `/chat`, but the requested xAI/Grok realtime voice provider is still absent from the provider-plugin surface, and the maintainer comments plus #73032 explicitly leave the `/chat` work unresolved.

## What This Changes

Request an xAI/Grok-backed live browser voice loop in `/chat`, with spoken agent summaries and microphone replies continuing the chat conversation.

## Best Possible Solution

Implement xAI/Grok as a registered `RealtimeVoiceProviderPlugin` for `/chat` Talk in `extensions/xai`, extending the provider contract only if xAI's session transport requires it, with focused tests and xAI/Control UI docs.

## Reproduction Assessment

Not applicable as a feature request rather than a bug report. The missing capability is source-verifiable on current main because `/chat` Talk resolves registered realtime voice providers while xAI registers speech and realtime transcription but not realtime voice.

## Solution Assessment

Yes, keeping this open is the best current outcome. The narrow maintainable path is provider-plugin work in `extensions/xai`, while #73032's `/voiceclaw/realtime` adapter is adjacent but not sufficient for `/chat` Talk.
