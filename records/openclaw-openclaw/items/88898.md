## Summary

Keep open: current main still leaks `isError` tool-result progress through the reply dispatch path when `messages.suppressToolErrors` is enabled, and this PR is a focused fix with sufficient live terminal proof and no blocking findings.

## What This Changes

The PR adds a dispatch-time guard and regression test so `messages.suppressToolErrors` suppresses error tool-result progress callbacks before source callback or channel delivery while preserving the final reply.

## Best Possible Solution

Land this patch or an equivalent dispatch-site guard after normal maintainer merge handling, while keeping broader warning-routing and final-only policy work in the existing open issues.

## Reproduction Assessment

Yes. Current-main source shows `dispatchReplyFromConfig` can receive an `isError` tool-result payload and forward or deliver it without checking `messages.suppressToolErrors`; the PR test covers that path.

## Solution Assessment

Yes. The dispatch-site guard is the narrowest maintainable fix because terminal warning generation already honors the config and channel adapters should not infer product policy from tool-result text.
