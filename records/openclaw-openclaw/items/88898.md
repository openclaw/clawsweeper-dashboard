## What This Changes

The branch suppresses `isError: true` tool-result progress delivery in `dispatchReplyFromConfig` when the existing `messages.suppressToolErrors` option is enabled, and adds a focused regression test.

## Best Possible Solution

Land the dispatch-site guard with the focused regression test, preserving default behavior and using the existing config contract rather than adding a new option.

## Reproduction Assessment

Yes, by source inspection and supplied live-output proof: current main lacks a suppressToolErrors gate in the `onToolResult` progress path, while the PR head shows the error case drops `sendToolResult` and preserves final delivery.

## Solution Assessment

Yes. The dispatch path is the right layer because final warning payload suppression already exists elsewhere; moving the guard into channel adapters would duplicate policy and miss the reply-options callback leak.
