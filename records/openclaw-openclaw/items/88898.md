## What This Changes

## Best Possible Solution

Land this narrow dispatch-site guard after current-head checks pass; optional Telegram live proof can add transport confidence, but no code repair is indicated.

## Reproduction Assessment

## Solution Assessment

Yes, this is the best narrow fix: the remaining leak is in the dispatch-site progress path, and guarding before the reply-options callback avoids both dispatcher and channel-owned progress delivery without adding a new config/API surface.
