## What This Changes

The PR adds a `messages.suppressToolErrors` guard in shared reply dispatch so `isError` tool-result progress is dropped before source-callback or dispatcher delivery, plus a focused regression test.

## Best Possible Solution

Land the narrow shared dispatch guard after maintainer review, while keeping broader `all | final | off` warning policy and warning-routing decisions in the existing issues.

## Reproduction Assessment

Yes, at source level. Current main forwards `isError` tool-result progress before applying `messages.suppressToolErrors`, and the PR proof exercises the suppressed-error, non-error, and disabled-config paths.

## Solution Assessment

Yes. The dispatch-owned progress path is the right layer because filtering only final warning payloads or only channel adapters would miss the source callback and duplicate policy across transports.
