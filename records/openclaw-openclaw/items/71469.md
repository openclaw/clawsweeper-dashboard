## Summary

Keep open: this is a scoped Google plugin OAuth bug fix, and current main still has the catch-only URL fallback that can leave users without a copyable authorization URL when browser launch silently fails.

## What This Changes

The PR prints the Gemini CLI OAuth authorization URL before attempting browser launch and adds a focused local-login ordering regression test.

## Best Possible Solution

Land this scoped Google plugin OAuth fix with its regression test, and track sibling-provider opener behavior as separate follow-up work.

## Reproduction Assessment

Yes. Source inspection shows current main only logs the URL on thrown openUrl failure while the shared browser opener can resolve false, and the linked Windows report gives after-patch real setup proof for the same failure mode.

## Solution Assessment

Yes. Printing the URL before the best-effort opener is the narrowest maintainable fix because it preserves the callback server, state validation, and token exchange instead of changing shared opener semantics.
