## Summary

Keep open: the branch targets a real source-reproducible memory-flush gap, but the PR still has a P2 correctness issue because the fallback also runs for byte-size-only snapshots, and its proof does not demonstrate the patched MiniMax/WebChat path.

## What This Changes

The PR adds an `estimatePromptTokensFromSessionTranscript` fallback inside `runMemoryFlushIfNeeded` when the session log usage snapshot lacks prompt tokens.

## Best Possible Solution

Guard the fallback to the stale/unknown or near-threshold transcript-usage path, add focused regression coverage for missing usage and byte-size-only snapshots, then verify with a redacted MiniMax/WebChat runtime run.

## Reproduction Assessment

Yes, by source inspection: a non-CLI session with missing or stale `totalTokens` and no transcript usage records leaves `tokenCountForFlush` undefined on current main. I did not run a live MiniMax/WebChat reproduction in this read-only review.

## Solution Assessment

No. Reusing `estimatePromptTokensFromSessionTranscript` is the right direction, but this PR should only invoke it when memory flush intentionally requested transcript usage, not for byte-size-only snapshots.
