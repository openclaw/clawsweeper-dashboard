## Summary

Keep open: the reported memory-flush bug is source-reproducible, but this PR still has a concrete guard bug and the supplied proof does not show the patched MiniMax/WebChat behavior running after the fix.

## What This Changes

The PR adds a transcript-token estimator fallback in `runMemoryFlushIfNeeded` when transcript usage prompt tokens are unavailable.

## Best Possible Solution

Gate the estimator fallback to the stale/unknown or near-threshold transcript-usage path, add focused regression coverage, and require redacted after-fix runtime proof before merge.

## Reproduction Assessment

Yes, source-reproducible: create a non-CLI session whose `totalTokens` is missing or stale and whose transcript has no usage records; current main leaves `tokenCountForFlush` undefined because memory flush never calls the existing transcript estimator. I did not run a live MiniMax/WebChat reproduction.

## Solution Assessment

No. Reusing `estimatePromptTokensFromSessionTranscript` is the right direction, but this PR invokes it whenever `sessionLogSnapshot` exists, including byte-size-only reads where `includeUsage` is intentionally false.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
