## Summary

This is a useful lifecycle-extension idea, but it does not need a new OpenClaw core skill phase contract: current main keeps skills as prompt-discovered instruction packs, while plugin hooks already cover prompt injection, final-answer revision, turn-end observation, and tool-error observation for a ClawHub plugin.

## What This Changes

The issue requests phase-aware skill behavior so some skills can trigger after task completion, after errors, or before responses rather than only through initial prompt keyword matching.

## Best Possible Solution

Keep the core skill contract as an AgentSkills-compatible prompt catalog and implement phase-aware procedural capture as a ClawHub plugin using the existing lifecycle hooks.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior. Source and docs confirm current main has prompt-discovered skills and plugin lifecycle hooks, but no post-task skill phase contract.

## Solution Assessment

No: adding core skill trigger phases is not the narrowest current fit. A ClawHub plugin can use before_prompt_build, before_agent_finalize, agent_end, and after_tool_call to implement the requested behavior without expanding core skill metadata.
