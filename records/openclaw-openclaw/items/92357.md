## Summary

The reported mixed-result loss is credible, but this patch also removes the existing all-below-threshold fallback for keyword-backed rows that overlap vector results, and its new test exercises behavior already present on current main rather than the changed manager filter. The PR should remain open for a narrow correction plus real after-fix search proof.

## What This Changes

The PR changes memory hybrid-search filtering to retain keyword-only matches when vector and keyword chunks do not overlap, and adds helper-level coverage.

## Best Possible Solution

Preserve the existing all-below-threshold fallback for every keyword-backed merged row, add keyword-only rows beside strict results only for the mixed non-overlap case, and cover both branches through `MemoryIndexManager.searchHybrid()`.

## Reproduction Assessment

Yes at source level: a strict vector result plus a non-overlapping keyword-only result reproduces the reported early-return loss on current main, while an all-below-threshold overlapping result reproduces the patch regression. No real CJK/trigram run was supplied.

## Solution Assessment

No. The manager is the correct boundary, but filtering only `vectorScore === 0` removes the existing overlapping-keyword fallback; the narrow fix must preserve that branch and add manager-level tests.
