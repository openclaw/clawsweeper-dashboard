## What This Changes

The PR moves service-mode stale gateway cleanup behind effective bind resolution and a `/healthz` probe, then adds regression tests for the healthy-listener early exit and systemd exit 78 path.

## Best Possible Solution

Land the early healthy-listener exit if maintainers accept the existing-listener ownership tradeoff, with normal gateway startup validation before merge.

## Solution Assessment

Yes, with maintainer acceptance of the availability tradeoff. Moving stale cleanup behind the existing `/healthz` ownership check matches the documented supervised-lock contract, but maintainers should explicitly accept that the already-healthy listener stays in control.
