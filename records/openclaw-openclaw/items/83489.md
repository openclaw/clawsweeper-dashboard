## What This Changes

The branch moves service-mode stale gateway PID cleanup behind bind resolution and a `/healthz` probe, then adds regression coverage for leaving an existing healthy gateway in control, including systemd exit 78.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes, with a maintainer-owned tradeoff. Moving stale cleanup behind a `/healthz` check matches the existing supervised-lock contract, but maintainers should explicitly accept that a healthy existing listener stays in control even when a duplicate service unit is stale or older.
