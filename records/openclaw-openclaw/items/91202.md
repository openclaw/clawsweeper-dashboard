## Summary

Keep open: current main and v2026.6.8 have a partial required-platform-package check, but they still inherit platform/optional npm config pollution and do not validate the nested Codex optional dependency version or vendor executable before install/update success.

## What This Changes

The issue requests hardening managed Codex plugin install/update so host-native optional dependencies are included, validated, and repaired before macOS update/startup loops.

## Best Possible Solution

Land a maintainer-approved repair that scrubs managed npm platform/optional config pollution and validates the host-required Codex optional package, version, and executable before install/update state is persisted, with stale-index and launchd hardening split or explicitly scoped.

## Reproduction Assessment

Yes, at source level. The safe npm install env preserves the reported platform/optional npm config keys, and upstream Codex requires the host-native optional package before launching.

## Solution Assessment

No, current main is only a partial solution. The maintainable fix needs install/update-time env scrub plus Codex optional-dependency, version, and vendor-binary validation rather than relying only on recorded lockfile entries.
