## Summary

Keep open: current main and v2026.6.6 still do not scrub platform/arch/optional npm config for managed installs or validate the nested host-native Codex optional package before recording the managed Codex plugin install as successful.

## What This Changes

The issue requests managed npm/install update hardening so OpenClaw validates Codex's host-native optional dependency and avoids updater/install state that leaves the Codex app-server broken at runtime.

## Best Possible Solution

Keep this open and scope the central managed npm/Codex verifier first, with explicit maintainer decisions or follow-ups for updater handoff/backoff and installed-index migration behavior.

## Reproduction Assessment

Yes. The source path is clear: managed npm install preserves the relevant parent npm config, current verification only checks the top-level package, and upstream Codex throws when the host-native optional package is missing; live macOS arm64 was not rerun.

## Solution Assessment

Partly. Env scrub plus post-install validation of the Codex native package and vendor binary is the right owner-boundary fix, but launchd/backoff and installed-index precedence should be split or maintainer-scoped instead of bundled into an unfocused patch.
