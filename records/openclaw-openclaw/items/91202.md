## Label Justifications

- `P2`: The issue reported a real managed Codex install/update reliability failure with platform-specific blast radius.
- `impact:crash-loop`: The missing Codex native package left the app-server failing at runtime while update/restart automation could repeatedly restart the Gateway.

## Summary

Close: current main and v2026.6.8 implement the central reliability requirement by declaring Codex platform packages and making managed npm installs retry or roll back instead of persisting missing host-native optional packages.

## What This Changes

The issue asks OpenClaw to validate Codex native optional dependencies, reject bad managed npm plugin installs, and harden related macOS update/install recovery paths.

## Best Possible Solution

Keep the shipped required-platform-package validation and rollback path; handle any post-v2026.6.8 macOS updater or stale-index recurrence as a new focused report with fresh logs.

## Reproduction Assessment

Yes, source-level. Upstream Codex maps macOS arm64 to @openai/codex-darwin-arm64 and throws the reported missing optional dependency error when that package/vendor binary is absent.

## Solution Assessment

Yes. The shipped fix moves detection to managed plugin install/update time using manifest-declared required platform packages and rollback, which is the maintainable boundary for the reported runtime failure.
