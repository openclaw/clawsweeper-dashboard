## Summary

Keep open: current main still has the reported source-level gap. Plugin slash commands are dispatched through the auto-reply/chat path, but `openclaw agent` local/gateway runs and embedded local TUI still hand slash text to the agent runner without a plugin-command preflight; gateway-backed TUI already uses `chat.send`, so the repair should be narrower than the full title.

## What This Changes

The issue requests dispatching plugin-registered slash commands on `openclaw agent` local/gateway paths and embedded TUI before unmatched messages are sent to the LLM.

## Best Possible Solution

Add a shared command-dispatch preflight for agent local/gateway ingress and embedded local TUI that reuses existing auto-reply command semantics, authorization, session metadata, and fallthrough behavior.

## Reproduction Assessment

Yes, at source level. Current main shows plugin commands dispatch through `chat.send`/auto-reply, while agent local/gateway and embedded TUI pass slash text directly to `agentCommandFromIngress` or the agent RPC without calling that dispatcher.

## Solution Assessment

Yes, with scope control. The narrow maintainable fix is to reuse the existing command dispatcher before agent execution, preserve gateway scopes and owner checks, and leave unmatched slash/plain messages on the existing LLM path.
