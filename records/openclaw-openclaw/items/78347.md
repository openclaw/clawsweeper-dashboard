## Summary

Keep open: current main still has the plugin command dispatcher for chat/channel paths, but `openclaw agent` local/gateway runs and embedded TUI hand slash text to the agent runner without that preflight. I found no duplicate or merged fix that owns the remaining CLI/TUI dispatch gap.

## What This Changes

The issue asks for plugin-registered slash commands to dispatch from `openclaw agent` local/gateway and embedded TUI instead of being sent to the LLM.

## Best Possible Solution

Add one shared command-dispatch preflight for agent local/gateway ingress and embedded TUI that reuses existing auto-reply command semantics while leaving unmatched messages on the current agent path.

## Reproduction Assessment

Yes, at source level. Current main shows plugin commands dispatch through the auto-reply/chat pipeline, while the agent CLI and embedded TUI paths pass the slash text into `agentCommandFromIngress` or `runAgentAttempt` without invoking that dispatcher.

## Solution Assessment

Yes, with authorization care. The maintainable fix is a shared pre-agent dispatch hook using the existing command pipeline, not a separate plugin-command parser, while preserving gateway scopes and falling through for non-matching commands.
