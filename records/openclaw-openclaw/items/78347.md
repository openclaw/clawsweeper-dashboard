Keep open: current main and the latest release still forward CLI/Gateway/TUI slash text into agent send paths, while plugin command execution remains wired through the auto-reply/channel command stack. The issue is real, but the fix crosses CLI, Gateway, TUI, plugin-command authorization, and reply-context boundaries, so it needs maintainer direction before implementation.

## What This Changes

Issue requests dispatching plugin-registered slash commands from the agent CLI local/gateway paths and TUI instead of forwarding slash text to the LLM.

## Best Possible Solution

Add a shared pre-agent plugin-command dispatch boundary for CLI local, Gateway agent RPC, and TUI input that reuses the existing dispatcher while making authorization, session, channel, and reply context explicit.
