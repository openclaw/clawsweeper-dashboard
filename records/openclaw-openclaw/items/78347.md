## Summary

Keep open: current main still dispatches plugin slash commands through chat/channel paths, but the agent CLI local/gateway paths and embedded TUI path still pass slash text into the agent runner without the shared command preflight. The report is source-reproducible, and the only related live PR is an open auth-hardening PR rather than a merged fix for this dispatch gap.

## What This Changes

Plugin-registered slash commands should dispatch from `openclaw agent` local/gateway and embedded TUI inputs instead of being sent to the LLM as ordinary prompts.

## Best Possible Solution

Add a shared pre-agent command dispatch hook for agent local/gateway ingress and embedded TUI that reuses the existing command pipeline, falls through for non-matches, and carries explicit owner/scope context from the caller boundary.

## Reproduction Assessment

Yes, at source level. Current main shows plugin commands dispatch in the shared chat/channel pipeline, while `openclaw agent` local/gateway and embedded TUI send the same slash text into `agentCommandFromIngress()` or `runAgentAttempt()` without invoking that dispatcher.

## Solution Assessment

Yes, with authorization care. The maintainable direction is a shared pre-agent dispatch hook using the existing command pipeline, but a maintainer should choose the exact insertion point and caller auth context before implementation.
