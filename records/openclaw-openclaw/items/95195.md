## Best Possible Solution

Land one canonical cached macOS product-version fix for all runtime prompt OS metadata paths, while either keeping the runtime-only label private or explicitly accepting the additive OS-summary metadata field.

## Reproduction Assessment

Yes from source inspection: current main sends `os.type()` plus `os.release()` into the three runtime prompt paths, and the linked issue provides macOS 26.5.1 versus Darwin 25.5.0 host evidence. I did not run a live Tahoe host in this read-only review.

## Solution Assessment

Mostly yes: reusing the cached OS-summary helper for runtime prompt labels is the right layer and covers all three prompt callers. The remaining question is whether the additive `osLabel` field should be public metadata or kept private to prompt construction.
