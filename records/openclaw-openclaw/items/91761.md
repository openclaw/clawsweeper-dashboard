## What This Changes

The issue reports that Docker `workspaceAccess: "rw"` sessions on v2026.6.5 still advertise host-only bundled skill paths in startup context instead of `/workspace/.openclaw/sandbox-skills/skills/...`.

## Best Possible Solution

Audit the actual Docker `workspaceAccess: "rw"` first-turn and command/context prompt paths on current main and v2026.6.5, then route any remaining host-path prompt surface through the existing materialized-sandbox-skills helper with focused sandbox boundary coverage.

## Solution Assessment
