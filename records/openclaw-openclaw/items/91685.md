## Summary

Keep open for maintainer review: the patch is a focused resolver-level mitigation with sufficient real Matrix proof and no discrete code-review finding, but it intentionally changes shipped keyless/channel-only implicit cron behavior from inherited delivery to fail-closed refusal.

## What This Changes

The PR makes keyless or channel-only implicit isolated cron delivery refuse targets inherited from the shared agent-main bucket, passes sessionTarget-derived keys into resolution, and adds regression coverage for resolver, run-layer wiring, dispatch cleanup, and awareness behavior.

## Best Possible Solution

Land this resolver-level refusal only after maintainers accept the fail-closed compatibility policy, while keeping broader restart replay fences, TTLs, or room/session gates on the canonical Matrix issue.

## Reproduction Assessment

Yes for the source path: current main and v2026.6.6 still fall back to the main session entry for implicit last-target delivery, and the PR body supplies after-fix Matrix/Synapse proof. I did not rerun the live Matrix scenario in this read-only review.

## Solution Assessment

Yes as a narrow mitigation if maintainers accept fail-closed behavior; the resolver is the common boundary for dispatch, preview, and failure notifications. If compatibility for existing keyless crons must remain the default, the safer alternative is persisted creator-session provenance plus a migration design.
