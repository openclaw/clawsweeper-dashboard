## Summary

Keep open: the patch looks correct on the reviewed code paths, but it intentionally changes existing keyless/channel-only implicit cron delivery from shared-main fallback delivery into fail-closed delivery-target errors, so this PR still needs maintainer acceptance of that compatibility policy. It also addresses only the keyless resolver slice of the broader Matrix restart replay report at https://github.com/openclaw/openclaw/issues/91613.

## What This Changes

The branch makes keyless implicit isolated cron delivery return `ok:false` when it would inherit the shared agent-main last target, passes sessionTarget-derived keys into delivery resolution, and cleans up deleteAfterRun sessions when delivery is refused.

## Best Possible Solution

Land the resolver-level refusal only after maintainers explicitly accept the fail-closed compatibility policy, while keeping broader restart replay fences, TTLs, or room/session gates tracked on the canonical Matrix issue.

## Reproduction Assessment

Yes for the source path: current main falls back from missing per-job/thread context to the main session entry for `delivery.channel=last`, and existing tests expect that inherited route. I did not run the live Matrix reproduction in this read-only review.

## Solution Assessment

Yes as a narrow mitigation if maintainers accept fail-closed behavior; the resolver is the common boundary for dispatch, preview, and failure notification. If compatibility for existing keyless crons must remain the default, the safer alternative is persisted creator-session provenance plus migration design.
