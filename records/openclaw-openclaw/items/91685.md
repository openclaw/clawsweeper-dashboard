## Summary

Keep open: this PR contains a meaningful, current fix for ambiguous cross-conversation cron delivery and is not implemented on current main. It still needs maintainer acceptance of the intentional fail-closed behavior for existing keyless or channel-only implicit cron jobs before merge.

## What This Changes

The branch changes isolated cron delivery resolution so keyless implicit deliveries inherited only from the shared agent-main last target return `ok:false`, and adds resolver, run-layer, and awareness regression coverage.

## Best Possible Solution

Land a maintainer-approved fail-closed resolver path for ambiguous keyless inherited cron targets, or replace it with an explicit persisted-provenance design before merge.

## Reproduction Assessment

Yes. Source inspection shows current main can resolve a keyless implicit cron through the shared `mainEntry` and then dispatch only if the resolver returns `ok:true`; I did not run the live Matrix reproduction locally.

## Solution Assessment

Conditional. Returning `ok:false` from the resolver is a narrow owner-boundary fix that existing consumers honor, but the compatibility break for keyless and channel-only crons needs maintainer acceptance before calling it the best permanent solution.
