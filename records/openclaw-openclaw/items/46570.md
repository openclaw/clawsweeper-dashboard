## Summary

Keep open: current main still has the provider-backed hybrid path where keyword-only memory-file hits can be filtered out when any strict vector/session result clears the threshold, and the related repair PR at https://github.com/openclaw/openclaw/pull/73976 is open but not merge-ready.

## What This Changes

The issue reports that `openclaw memory search` indexes memory files but hybrid search results only return session transcript matches.

## Best Possible Solution

Land or replace the provider-backed hybrid fallback fix so merged results preserve keyword-backed memory-file hits in mixed memory/session searches, with regression coverage for both the mixed case and existing no-provider FTS-only recall.

## Reproduction Assessment

Yes, at source level. With default minScore 0.35 and 0.7/0.3 hybrid weights, a keyword-only memory-file hit can fall below threshold, and current provider-backed hybrid code returns strict hits immediately when any session/vector result passes.

## Solution Assessment

Yes for keeping this issue open. A narrow memory-core hybrid selection/fallback fix plus mixed memory/session regression coverage is the maintainable path, while the open repair PR is an active branch to refresh or replace rather than proof this is resolved.
