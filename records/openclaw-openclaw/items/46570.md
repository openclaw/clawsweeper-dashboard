## Summary

Keep open. Current main still has a source-reproducible builtin/provider-backed hybrid selection bug where memory-file keyword hits can be filtered out whenever any session/vector result clears the strict score threshold; the related QMD fix and the open per-keyword FTS PR do not cover this mixed-source selection gap.

## What This Changes

The issue requests that `openclaw memory search` return indexed memory-file matches instead of only session transcript results for exact memory-file content.

## Best Possible Solution

Add source-aware preservation or diversification for keyword-backed memory-file hits in the builtin provider-backed hybrid selection path, with mixed memory/session regression coverage and no QMD or FTS-only behavior churn.

## Reproduction Assessment

Yes, at source level. With default hybrid weights, a keyword-only memory hit can fall below `minScore`; if a session/vector hit passes strict filtering, current main returns before preserving the memory keyword result.

## Solution Assessment

Yes. A narrow builtin hybrid selection fix plus mixed memory/session coverage is the maintainable path; lowering thresholds or adding a config option would be broader than needed.
