## Summary

Keep open: current main and v2026.5.22 still have the provider-backed hybrid path where one full-query FTS lookup and an early strict-results return can suppress keyword-backed memory-file hits when session/vector results pass first; the open related repair PR owns the likely fix but is conflicting and previously needed changes.

## What This Changes

The issue reports that indexed memory files do not appear in `openclaw memory search` results while session transcript results do.

## Best Possible Solution

Refresh or replace the related memory-core repair so provider-backed hybrid search preserves keyword-backed memory-file hits in mixed memory/session results while keeping the existing FTS-only fallback behavior intact.

## Reproduction Assessment

Yes, at source level. With default `minScore` 0.35 and 0.7/0.3 hybrid weights, a keyword-only memory-file hit can fall below threshold while the current provider-backed hybrid code returns strict session/vector hits immediately.

## Solution Assessment

Yes for keeping this issue open. The narrow maintainable fix is to refresh or replace the open memory-core repair with mixed memory/session regression coverage, not to close this issue before the repair lands.
