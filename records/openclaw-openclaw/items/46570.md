## Summary

Keep open. Current main still has the builtin mixed-source ranking gap: memory files and session transcripts are searched together, then the merged results are sliced by score, while only the QMD backend has an explicit memory/session retention guard.

## What This Changes

The issue reports that local hybrid memory search indexes memory files but default search results only return session transcript hits.

## Best Possible Solution

Add a focused builtin memory-core regression for mixed memory and session search, then make builtin ranking retain a relevant memory-file hit without changing defaults or weakening the existing FTS fallback behavior.

## Reproduction Assessment

Yes at source level, but not as a live run. The reporter's CLI path is concrete, and current main's builtin manager still slices merged mixed-source results by score without the QMD memory/session retention guard.

## Solution Assessment

Yes, the narrow maintainable fix is in builtin ranking or merge selection plus regression coverage. Asking users to force `corpus=memory` or changing default source configuration would avoid the symptom but would not satisfy the documented default memory recall contract.
