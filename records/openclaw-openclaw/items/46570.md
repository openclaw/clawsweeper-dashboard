## Summary

Keep open. Current main still has a mixed memory/session hybrid search path where strict session hits can be returned before keyword-backed memory-file hits are preserved, and the adjacent open repair PR at https://github.com/openclaw/openclaw/pull/73976 is not merged and does not yet cover this selection behavior.

## What This Changes

The issue asks for `openclaw memory search` to return indexed memory-file matches instead of only session transcript hits when both corpora are enabled.

## Best Possible Solution

Fix the memory-core provider-backed hybrid selection path so exact keyword-backed memory-file hits remain visible when session transcript hits also pass strict filtering, with mixed-source regression coverage.

## Reproduction Assessment

Yes, at source level. With memory and sessions enabled, default hybrid weights can leave a keyword-only memory hit below `minScore` while session vector hits pass, and the manager returns those strict hits before the memory keyword fallback can run.

## Solution Assessment

Yes. A narrow memory-core selection/ranking fix plus mixed memory/session regression coverage is the maintainable path; lowering global thresholds or adding a new config knob would be broader than needed.
