## Summary

Keep open. Current main still has a builtin/provider-backed hybrid search path where strict session hits can be returned before keyword-backed memory-file hits are preserved; the QMD path has separate source diversification, but builtin fallback and explicit `memory.backend: "builtin"` remain affected, and the adjacent open repair PR at https://github.com/openclaw/openclaw/pull/73976 does not cover this selection failure.

## What This Changes

The issue reports that hybrid memory search with session indexing returns session transcript hits while indexed memory-file matches never appear.

## Best Possible Solution

Add source-aware keyword preservation to the builtin provider-backed hybrid selection path, with mixed memory/session regression coverage where session vector hits pass `minScore` and exact memory-file keyword hits would otherwise be filtered out.

## Reproduction Assessment

Yes, at source level. With builtin/provider-backed hybrid search, default weights can leave exact keyword-only memory hits below `minScore` while session vector hits pass, and the manager returns the strict hits before preserving the memory keyword result.

## Solution Assessment

Yes. A narrow builtin hybrid selection/ranking fix plus mixed memory/session coverage is the maintainable path; lowering global thresholds or adding a new config knob would be broader than needed.
