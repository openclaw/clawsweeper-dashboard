## Summary

Keep open. Current main still has a source-level builtin memory-search gap: the default CLI path searches enabled memory and session sources together, then the builtin backend slices mixed results by score without the QMD backend's memory/session retention guard.

## What This Changes

The issue reports that default local hybrid memory search returns session transcript hits while omitting matching indexed memory files.

## Best Possible Solution

Add a focused builtin memory-core regression for mixed memory/session results, then make builtin result selection retain a relevant memory-file hit without changing source defaults or weakening existing FTS fallback behavior.

## Reproduction Assessment

Yes, at source level rather than as a live CLI run. The reporter's path is concrete, and current main still routes default CLI search through mixed-source builtin ranking that can slice out memory-file hits.

## Solution Assessment

Yes, the narrow maintainable fix is in builtin ranking or final selection plus regression coverage. Requiring `corpus=memory` or changing default source configuration would avoid the symptom but would not satisfy the documented default memory recall contract.
