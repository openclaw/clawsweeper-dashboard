## Summary

Keep open. Current main and v2026.5.12 still have a builtin hybrid search path where strict session transcript hits can return before lower-scored memory-file keyword hits are preserved, and the open adjacent memory-search PR is not a merged fix for this exact mixed-source behavior.

## What This Changes

This issue requests that hybrid memory search return indexed memory-file matches instead of only session transcript matches.

## Best Possible Solution

Fix the builtin hybrid selection so memory-file keyword hits are retained or diversified when session transcript hits already pass strict filtering, with regression coverage for mixed memory plus sessions.

## Solution Assessment

Yes. A narrow memory-core ranking/selection fix is the maintainable path; filtering with corpus=memory is only a workaround for the default mixed-source hybrid behavior users expect.
