## Summary

Keep open. Current main still drops lower-scored keyword-only memory-file hits in provider-backed hybrid search when any strict result, such as a session vector hit, passes `minScore`; the related open PR is adjacent but does not close this issue.

## What This Changes

Make memory search return indexed memory-file matches in hybrid local searches instead of returning only session transcript results.

## Best Possible Solution

Preserve source-aware keyword hits in the builtin provider-backed hybrid selection path even when other strict results exist, with mixed memory/session regression coverage and no changes to QMD or no-provider FTS-only semantics.

## Reproduction Assessment

Yes, at source level. Default hybrid scoring can put exact keyword-only memory hits below `minScore`, and current selection returns strict session/vector hits before preserving those memory hits.

## Solution Assessment

Yes. A narrow source-aware keyword-preservation fix in provider-backed hybrid selection plus mixed memory/session tests is the maintainable path; lowering thresholds or adding a config knob would be broader than needed.
