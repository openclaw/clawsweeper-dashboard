## Summary

Keep open. Current main still has a provider-backed hybrid path that returns strict session/vector hits before preserving lower-scored memory-file keyword hits, so the reported memory-file recall failure remains source-reproducible.

## What This Changes

The issue requests that `openclaw memory search` return indexed memory-file matches in hybrid mode instead of only session transcript hits.

## Best Possible Solution

Add source-aware keyword preservation to the builtin provider-backed hybrid result selection path, with mixed memory/session regression coverage for session vector hits plus exact memory-file keyword hits.

## Reproduction Assessment

Yes, at source level. With default hybrid weights, a keyword-only memory hit can score below `minScore`; when a session vector hit passes, the manager returns strict hits before preserving the memory keyword result.

## Solution Assessment

Yes. A narrow builtin hybrid selection fix plus mixed memory/session coverage is the maintainable path; lowering thresholds or adding a config knob would be broader than needed.
