## What This Changes

Issue reports that indexed `memory/*.md` content does not appear in `openclaw memory search` results while session transcript results do appear in local hybrid memory search.

## Best Possible Solution

Refresh or replace the related memory-core repair so provider-backed hybrid search preserves keyword-backed memory-file hits in mixed memory/session results while keeping FTS-only fallback behavior intact.

## Reproduction Assessment

Yes, at source level. With default `minScore` 0.35 and 0.7/0.3 hybrid weights, current main can return strict session/vector hits before preserving lower-scored keyword-backed memory-file hits.

## Solution Assessment

Yes for keeping this issue open. The narrow maintainable fix is to refresh or replace the open memory-core repair with mixed memory/session regression coverage, not to close this issue before the repair lands.
