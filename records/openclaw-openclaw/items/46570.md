## Summary

Keep open: current `main` and the latest release include adjacent memory-search fixes, but the mixed `memory` + `sessions` hybrid path can still return strict session/vector hits before relaxed keyword-backed memory-file hits are considered.

## What This Changes

The issue reports that `openclaw memory search` indexes `memory/*.md` files but returns only session transcript results instead of matching memory-file chunks.

## Best Possible Solution

Fix memory-core hybrid selection so keyword-backed memory-file hits can surface even when strict session/vector results exist, with regression coverage for mixed `memory` and `sessions` sources under default scoring.

## Reproduction Assessment

Yes, at source level. With `memory` and `sessions` enabled, default hybrid scoring can place an exact memory-file keyword hit below `minScore`, and `manager.search` returns strict session/vector results before relaxed keyword-backed rows are considered.

## Solution Assessment

No, closing is not the best solution because the central mixed-source behavior still has a source-level gap. The best fix belongs in `extensions/memory-core/src/memory/manager.ts` plus a mixed memory/session regression test.
