## Summary

Keep open: current main still has the mixed-source hybrid search failure shape where strict session hits can return before relaxed keyword-backed memory-file hits are considered.

## What This Changes

Make `openclaw memory search` return indexed memory-file matches instead of only session transcript results in mixed-source hybrid mode.

## Best Possible Solution

Update the memory-core manager ranking/selection logic so default all-source hybrid search can surface relevant memory-source lexical hits even when session chunks also clear strict scoring, with regression coverage for the mixed-source case.

## Reproduction Assessment

Yes. The issue provides a concrete CLI reproduction, and current source inspection shows a deterministic code path where a keyword-backed memory hit can be filtered out once any strict session hit exists.

## Solution Assessment

Yes. A narrow `MemoryIndexManager.search` fix with manager-level regression coverage is the maintainable path; a CLI-only filter would not fix default mixed-source search behavior.
