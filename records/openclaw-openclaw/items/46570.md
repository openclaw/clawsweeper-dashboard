## Summary

Keep open. Current main still has a source-backed mixed memory/session hybrid path where a memory-file keyword hit can be omitted when session vector hits already satisfy the strict score filter.

## What This Changes

The issue asks for `openclaw memory search` to return indexed memory-file matches instead of only session transcript results in local hybrid search.

## Best Possible Solution

Fix builtin hybrid selection so memory-file keyword hits are retained or diversified when session transcript hits pass strict filtering, with regression coverage for memory plus sessions under default hybrid weights.

## Reproduction Assessment

Yes at source level, but not by a live CLI run. The current builtin hybrid path can return strict session vector hits before the keyword fallback preserves a lower-scored exact memory-file hit.

## Solution Assessment

Yes. A narrow memory-core ranking fix is the maintainable path; `corpus=memory` is only a workaround and the adjacent FTS fan-out PR does not cover mixed-source starvation.
