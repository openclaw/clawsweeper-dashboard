## Summary

Keep open. Current main and the latest release still have a source-backed builtin hybrid path where strict session hits can return before lower-scored memory-file keyword hits are preserved.

## What This Changes

The issue reports that hybrid memory search indexes memory files but returns only session transcript results for exact content present in memory files.

## Best Possible Solution

Fix builtin hybrid selection so memory-file keyword hits are retained or diversified when session transcript hits pass strict filtering, with regression coverage for mixed memory plus sessions.

## Reproduction Assessment

Yes at source level, but not by a live CLI run. The current builtin hybrid path can return strict session vector hits before preserving a lower-scored exact memory-file keyword hit.

## Solution Assessment

Yes. A narrow memory-core ranking fix is the maintainable path; filtering to memory-only is a workaround, not the default hybrid behavior users expect.
