## Summary

Keep open: current main still has the mixed-source hybrid search path where strict session hits can be returned before lower-scoring keyword-backed memory-file hits are considered, and the overlapping repair PR is still open rather than merged.

## What This Changes

The issue asks for `openclaw memory search` to return indexed memory-file matches instead of only session transcript hits in hybrid/local search.

## Best Possible Solution

Fix the shared memory-core manager ranking/selection logic so mixed-source hybrid search can surface relevant memory-file lexical hits even when session chunks also meet the strict score, with regression coverage for the mixed-source case.

## Reproduction Assessment

Yes, from source inspection rather than a live CLI run. A memory-file keyword hit can remain below the strict score while a session hit passes it, and the manager returns the strict session set before the relaxed keyword-backed memory path runs.

## Solution Assessment

Yes. A narrow manager-level ranking/selection fix is the maintainable path because a CLI-only source filter would not fix the shared `memory_search` mixed-corpus behavior.
