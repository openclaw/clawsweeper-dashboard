## Summary

Keep open. Current main still has the mixed-source hybrid search failure shape: memory and session chunks are searched together, but strict session hits can return before relaxed keyword-backed memory hits are considered.

## What This Changes

The issue requests that `openclaw memory search` return indexed `MEMORY.md` and `memory/*.md` matches instead of only session transcript matches in mixed memory-plus-session search.

## Best Possible Solution

Update `MemoryIndexManager.search` selection/ranking so default all-source hybrid search can surface relevant memory-source lexical hits even when session chunks also clear strict scoring, while preserving explicit `sources` filters and QMD behavior.

## Reproduction Assessment

Yes. The issue gives a concrete CLI reproduction, and current code supports a deterministic regression test by seeding an exact memory-file phrase plus session chunks that produce strict hits, then asserting at least one `source: "memory"` result is returned.

## Solution Assessment

Yes. A narrow manager-level ranking/selection fix with regression coverage is the best path; a CLI-only corpus filter would be an incomplete workaround because default mixed search would remain broken.
