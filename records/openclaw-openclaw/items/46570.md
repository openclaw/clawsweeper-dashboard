## Summary

Keep open. Current main and the latest release still have a source-backed builtin hybrid path where strict session hits can return before lower-scored memory-file keyword hits are preserved; the adjacent open memory-search PR does not close this issue and targets a different hybrid recall gap.

## What This Changes

The issue reports that hybrid memory search indexes memory files but returns only session transcript results even for exact phrases in memory/*.md files.

## Best Possible Solution

Fix builtin hybrid selection so exact memory-file keyword hits are retained or source-diversified before strict session transcript hits consume the result window, with regression coverage for mixed memory and sessions.

## Reproduction Assessment

Yes at source level, but not by a live CLI run. The current builtin hybrid path can return strict session vector hits before preserving a lower-scored exact memory-file keyword hit.

## Solution Assessment

Yes. A narrow memory-core ranking fix is the maintainable path; filtering to memory-only is a workaround, not the default hybrid behavior users expect.
