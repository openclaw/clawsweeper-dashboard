## Summary

Current main still has the reported hybrid-search gap: FTS-only mode retries zero-hit multi-word queries per keyword, but provider-backed hybrid mode does not. The open replacement PR is the canonical repair path, so this issue remains necessary until that work lands or is rejected.

## What This Changes

Fix hybrid memory search so indexed Markdown memory files can be returned for multi-word queries instead of yielding only session transcript matches.

## Best Possible Solution

Land a refreshed version of the canonical bounded per-keyword fallback in the existing memory-core search path, with regression coverage showing provider-backed hybrid search returns matching memory-file chunks without unbounded SQLite fan-out.

## Reproduction Assessment

Yes at source level. A multi-word query that gets zero results from the strict FTS query receives per-keyword recovery in FTS-only mode but not in provider-backed hybrid mode on current main.

## Solution Assessment

Yes. Reusing the established fallback through one bounded helper is narrower and more maintainable than adding a separate memory-file search path or changing source-specific ranking.
