## Summary

Keep open. Current main still has a hybrid memory-search selection path that can return strict session transcript hits before preserving lower-scored exact memory-file keyword hits, and the open adjacent PR is not a merged fix for this mixed-source behavior.

## What This Changes

Request that hybrid memory search return indexed memory-file matches instead of only session transcript results in mixed memory/session searches.

## Best Possible Solution

Fix the builtin hybrid selection so keyword-backed memory-file hits are retained or diversified when session transcript hits already satisfy strict filtering, with regression coverage for mixed memory plus sessions.

## Reproduction Assessment

Yes, at source level. With mixed memory and session sources, default hybrid scoring can leave an exact memory-file keyword hit below minScore while session vector hits pass, and current manager code returns strict hits before preserving that memory keyword result.

## Solution Assessment

Yes. A narrow memory-core ranking/selection fix is the maintainable path; filtering searches to memory only is a workaround, and the adjacent per-keyword FTS PR does not by itself solve the strict mixed-source return behavior.
