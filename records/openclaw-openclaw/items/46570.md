## Summary

Keep open. Current main still has a source-reproducible builtin memory-search hybrid selection gap where a strict session/vector result can cause lower-scored memory-file keyword hits to be dropped before the keyword-preservation fallback runs; the related QMD coverage and the open per-keyword FTS PR do not cover this mixed-source builtin path.

## What This Changes

The issue reports indexed memory files being omitted from `openclaw memory search` results while session transcript hits are returned in hybrid local search.

## Best Possible Solution

Add source-aware preservation or diversification for keyword-backed memory-file hits in the builtin provider-backed hybrid selection path, with mixed memory/session regression coverage and no QMD or FTS-only behavior churn.

## Reproduction Assessment

Yes, at source level. With default hybrid weights, a keyword-only memory hit can fall below `minScore`; if any session/vector result passes strict filtering, current main returns before preserving the memory keyword result.

## Solution Assessment

Yes. A narrow builtin hybrid selection fix plus mixed memory/session regression coverage is the maintainable path; lowering thresholds or adding a config option would be broader than needed.
