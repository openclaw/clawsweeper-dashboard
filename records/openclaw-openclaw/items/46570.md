## Label Justifications

- `P2`: The bug makes indexed long-term memory unreliable for affected hybrid queries but is bounded to the memory retrieval subsystem.
- `impact:session-state`: Omitted memory-file matches cause persisted agent context to be absent from recall.

## Summary

Current main still reproduces the reported source-level gap: zero-hit multi-word queries receive per-keyword recovery in FTS-only mode, while provider-backed hybrid mode performs only the strict full-query FTS lookup. The issue remains necessary because the canonical repair has not merged.

## What This Changes

Fix hybrid memory search so indexed memory-file matches are returned instead of only session transcript results.

## Best Possible Solution

Refresh the canonical repair against current memory-core, share one capped per-keyword fallback between FTS-only and provider-backed hybrid retrieval, and prove memory-file recall plus the actual maximum number of executed FTS probes.

## Reproduction Assessment

Yes, at source level. A zero-hit multi-word FTS query receives per-keyword recovery only when no embedding provider is active; provider-backed hybrid search lacks the same recovery.

## Solution Assessment

Yes in direction. A shared bounded helper in the existing memory-core search path is narrower and more maintainable than adding source-specific search or a new configuration surface, but the stale branch and execution-bound test need refresh.
