## What This Changes

The issue reports that indexed `memory/*.md` matches are missing from `openclaw memory search` results when session transcript results are present.

## Best Possible Solution

Land a narrow memory-core fix that preserves keyword-backed memory-file hits in mixed memory/session hybrid results while preserving no-provider FTS and QMD behavior.

## Reproduction Assessment

Yes, at source level. With default minScore 0.35 and 0.7/0.3 hybrid weights, a keyword-only memory-file hit can fall below threshold, and current provider-backed hybrid code returns strict results immediately when session/vector hits pass.

## Solution Assessment

Yes. A narrow builtin hybrid selection fix plus mixed memory/session regression coverage is the maintainable path; the open repair PR is an active branch to refresh or replace, not proof the issue is resolved.
