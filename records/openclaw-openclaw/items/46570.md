## Summary

Keep open. Current main still has the builtin provider-backed hybrid path that can return strict session transcript hits before preserving lower-scored exact memory-file keyword hits, and the related QMD fix plus open per-keyword FTS PR do not fully cover this mixed-source builtin selection bug.

## What This Changes

The issue asks for `openclaw memory search` to return indexed `memory/*.md` matches instead of only session transcript results in local hybrid search.

## Best Possible Solution

Retain or diversify keyword-backed memory-file hits in builtin hybrid search when session transcript hits already satisfy strict filtering, with focused mixed memory-plus-session regression coverage.

## Reproduction Assessment

Yes, at source level. With `sources` including memory and sessions, default hybrid scoring can leave an exact memory-file keyword hit below `minScore` while session vector hits pass, and current builtin manager code returns strict hits before preserving that memory hit.

## Solution Assessment

Yes. A narrow memory-core ranking/selection fix is the maintainable path; QMD diversification and the adjacent per-keyword FTS PR are useful context but do not by themselves solve the builtin mixed-source strict-return behavior.
