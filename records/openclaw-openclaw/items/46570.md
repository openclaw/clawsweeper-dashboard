Keep open: current main and v2026.5.22 still allow provider-backed builtin hybrid search to return strict session/vector hits before preserving lower-scored keyword-backed memory-file hits. ## What This Changes

The issue reports that `openclaw memory search` returns session transcript hits but not indexed `memory/*.md` matches in local hybrid mode.

## Best Possible Solution

Add source-aware preservation or diversification for keyword-backed memory-file hits in the provider-backed builtin hybrid path, with mixed memory/session regression coverage and no QMD or no-provider FTS behavior churn.

## Reproduction Assessment

Yes, at source level. With default hybrid weights, a keyword-only memory hit can fall below `minScore`; if any session/vector result passes strict filtering, current main returns before preserving the memory keyword result.

## Solution Assessment

Yes. A narrow builtin hybrid selection fix plus mixed memory/session regression coverage is the maintainable path; lowering thresholds or adding a config option would be broader than needed.
