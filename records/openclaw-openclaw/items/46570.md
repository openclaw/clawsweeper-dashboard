## Summary

Keep open: current main and the latest release still have a builtin hybrid-search selection path where strict session/vector hits can return before lower-scored keyword-backed memory-file hits are preserved, and the related repair PR is open but not merge-ready.

## What This Changes

The issue reports that `openclaw memory search` returns session transcript matches but omits indexed `memory/*.md` file matches.

## Best Possible Solution

Refresh or replace the related memory-core repair so provider-backed hybrid search preserves keyword-backed memory-file hits in mixed memory/session results while keeping the existing FTS-only fallback behavior intact.

## Reproduction Assessment

Yes, at source level. With default minScore 0.35 and 0.7/0.3 hybrid weights, a keyword-only memory-file hit can fall below threshold, and current provider-backed hybrid code returns strict hits immediately when any session/vector result passes.

## Solution Assessment

Yes for keeping this open. A narrow memory-core hybrid selection/fallback fix plus mixed memory/session regression coverage is the maintainable path; the open repair PR is an active branch to refresh or replace, not proof the issue is resolved.
