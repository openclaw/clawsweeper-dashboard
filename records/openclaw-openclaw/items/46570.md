## Summary

Keep open. Current main still has a source-level mixed-corpus hybrid path where keyword-backed memory-file hits can be excluded when higher-scored session transcript hits satisfy the strict score threshold first.

## What This Changes

The issue requests that hybrid memory search return indexed memory-file matches instead of only session transcript matches.

## Best Possible Solution

Fix the memory-core hybrid selection path so exact keyword-backed memory-file hits remain visible when session transcript hits also pass strict filtering, with regression coverage for mixed memory plus sessions.

## Reproduction Assessment

Yes, at source level. With default hybrid weights, a keyword-only memory-file hit can score below minScore while session vector hits pass, and the manager returns those strict hits before the keyword-preservation fallback can include the memory result.

## Solution Assessment

Yes. A narrow memory-core selection/ranking fix plus mixed-source regression coverage is the maintainable solution; the adjacent per-keyword FTS PR is useful but does not yet merge or fully cover this behavior.
