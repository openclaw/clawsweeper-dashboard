## Summary

Keep open. Current main still has a source-backed mixed memory/session hybrid path where exact memory-file keyword hits can be omitted when session vector hits already satisfy the strict score filter, and the related open repair PR only addresses adjacent provider-backed FTS fan-out.

## What This Changes

`openclaw memory search` should return matches from indexed `memory/*.md` files instead of surfacing only session transcript hits.

## Best Possible Solution

Fix builtin hybrid mixed-source ranking so exact or strong memory-file keyword hits are retained when session transcript vector hits also pass strict filtering, with regression coverage for memory plus sessions under default hybrid weights.

## Reproduction Assessment

Yes by source inspection, but not by a live CLI run. With default hybrid weights, a memory-file keyword-only hit can score below `minScore` while session vector hits pass strict filtering, causing current main to return the strict session set before the keyword fallback can preserve the memory hit.

## Solution Assessment

Yes, a narrow memory-core ranking fix is the right direction. The `corpus=memory` tool option is only a workaround, and the adjacent open FTS fan-out PR does not by itself guarantee default mixed-source memory-file results.
