## Summary

Keep open: current main still has the mixed-source hybrid search failure shape where strict session hits can return before relaxed keyword-backed memory-file hits are considered. The overlapping #73976 PR is open and not merged, so it is not a basis to close this issue yet.

## What This Changes

The issue reports that `openclaw memory search` returns session transcript matches but omits indexed `memory/*.md` matches for exact memory-file content.

## Best Possible Solution

Update the memory-core manager selection logic so default mixed-source hybrid search can surface relevant memory-file lexical hits even when session chunks also pass strict scoring, with regression coverage for the mixed-source case.

## Reproduction Assessment

Yes, by source inspection. In a memory+sessions configuration, a memory-file keyword hit can be present in `keywordResults` but omitted because any strict session hit makes `MemoryIndexManager.search` return before the relaxed keyword-backed path runs.

## Solution Assessment

Yes. A narrow `MemoryIndexManager.search` ranking/selection fix with manager-level regression coverage is the maintainable solution; a CLI-only source filter would not fix the shared `memory_search` mixed-corpus behavior.
