## Summary

Keep this open. Current main has strong memory-search improvements, but the builtin hybrid path can still rank and slice mixed memory/session results without the source-diversity guard that the QMD path already applies.

## What This Changes

The issue asks for `openclaw memory search` to return indexed memory-file matches when session transcript indexing is also enabled.

## Best Possible Solution

Add builtin mixed-source result preservation equivalent to the QMD behavior, with focused regression coverage for memory and session sources in the builtin hybrid path.

## Reproduction Assessment

No, not as a live run in this read-only pass. Source inspection gives a credible reproduction path: enable both `memory` and `sessions`, produce high-scoring session vector hits plus an exact memory-file FTS hit, and the builtin path can slice by score without preserving the memory source.

## Solution Assessment

Yes, the narrow maintainable fix is to bring the existing QMD mixed-source retention behavior to the builtin manager rather than add a new option or change session-memory configuration semantics.
