## Summary

Keep open: current main and the latest release still have a source-reproducible builtin hybrid ranking gap where strict session/vector results can return before lower-scored memory-file keyword hits are preserved.

## What This Changes

The issue asks for hybrid memory search to return indexed `memory/*.md` hits instead of only session transcript results when both sources are indexed.

## Best Possible Solution

Add source-aware preservation or diversification for keyword-backed memory-file hits in the builtin provider-backed hybrid path, with mixed memory/session regression coverage and no QMD behavior churn.

## Reproduction Assessment

Yes, at source level. With default hybrid weights, a keyword-only memory hit can fall below `minScore`; if any session/vector result passes strict filtering, current main returns before preserving the memory keyword result.

## Solution Assessment

Yes. A narrow builtin hybrid selection fix plus mixed memory/session regression coverage is the maintainable path; lowering thresholds or adding a config option would be broader than needed.
