## Summary

Keep open. Current main still has a source-level builtin hybrid selection path where exact memory-file keyword hits can be dropped whenever higher-scored session transcript hits satisfy the strict threshold, and the adjacent open repair PR does not yet merge or fully cover that mixed-source behavior.

## What This Changes

The issue reports that indexed memory files do not appear in `openclaw memory search` results when session transcript results are present.

## Best Possible Solution

Fix the builtin hybrid ranking/selection path so keyword-backed memory-file hits are retained or diversified when session transcript hits already satisfy strict filtering, with regression coverage for mixed memory plus sessions.

## Reproduction Assessment

Yes, at source level. With builtin provider-backed hybrid search, a keyword-only memory hit can score below the default strict threshold while session vector hits pass, and the manager returns the strict session hits before the keyword-preservation fallback can include the memory result.

## Solution Assessment

Yes. A narrow memory-core selection fix plus focused mixed-source regression coverage is the maintainable path; the related per-keyword FTS PR is useful adjacent work but is not a merged fix for this mixed-source return behavior.
