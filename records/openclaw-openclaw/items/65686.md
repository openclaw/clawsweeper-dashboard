## Summary

Keep open. Current main still lacks an operator-configurable cron outbound payload policy for `full` / `last` / `none`; existing final-text collapse is channel-owned and existing `delivery.mode="none"` only covers runner fallback suppression, not last-only payload mapping.

## What This Changes

The issue requests a configurable cron delivery policy that can send all resolved payloads, only the last resolved payload, or no channel message while preserving run records and UI output.

## Best Possible Solution

Add a typed cron outbound payload policy with default `full`, apply `last` after existing payload resolution, and make `none` suppress runner outbound delivery without falling back to synthesized text.

## Reproduction Assessment

Yes, at source level: current main preserves multiple successful payloads for non-final-preference channels and has no accepted `cron.deliveryPayloads` key. I did not run a live channel cron job in this read-only review.

## Solution Assessment

Unclear until maintainers choose the config contract. The requested full/last/none mapping is narrow and plausible, but the public surface likely needs a decision on global versus per-job policy.
