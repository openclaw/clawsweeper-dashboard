## Summary

Keep open: current main has opt-in generic tool-loop diagnostics, but it does not provide the requested default, model-visible same-turn reread nudge for first-class read-tool loops, and the intervention/default semantics need maintainer product judgment.

## What This Changes

The issue requests a warning-only guard that detects repeated same-turn first-class read-tool rereads before they burn context or stall progress.

## Best Possible Solution

If maintainers accept the product behavior, extend the existing before-tool-call loop-detection path with a narrow, bounded, model-visible warning for repeated read paths, plus focused tests and docs that keep normal debugging workflows intact.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers decide the default and warning visibility; the existing before-tool-call loop-detection boundary looks like the right implementation layer if the behavior is accepted.
