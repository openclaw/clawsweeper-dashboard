## Summary

Keep open. Current main and the latest release still lack a default same-turn, first-class `read` reread guard; the only rolling detector that catches repeated `read` calls is opt-in, while the default guard only handles immediate post-compaction repeats.

## What This Changes

Request a warning-only same-turn guard that detects repeated first-class `read` calls against the same file or small file set before the agent burns tokens without progress.

## Best Possible Solution

Add a bounded read-aware detector in the existing loop-detection/before-tool-call path that warns on repeated same-path or small path-set reads only when there is no nearby edit, write, test, or broadened context progress.

## Reproduction Assessment

No live replay was established. Source inspection confirms current main lacks a default first-class `read` reread nudge, and the only same-args `read` detector is behind `tools.loopDetection.enabled`.

## Solution Assessment

Unclear until maintainers decide policy. The best implementation layer appears to be the existing loop-detection and before-tool-call pipeline, not a separate read-tool-only subsystem.
