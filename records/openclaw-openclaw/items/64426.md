## Summary

Keep open: current main and v2026.6.8 still provide only opt-in generic rolling tool-loop detection, while the first-class read wrapper has no default same-turn reread nudge. This remains the canonical read-tool proposal, and the remaining decision is product/runtime policy around defaults, thresholds, warning text, and false-positive tolerance.

## What This Changes

Add a warning-only guard that detects repeated same-turn first-class `read` calls over the same file or small file set and nudges the agent before token burn stalls progress.

## Best Possible Solution

Extend the existing before-tool-call loop-detection path with a narrowly scoped first-class `read` reread nudge after maintainers choose the default behavior, thresholds, and warning text.

## Reproduction Assessment

No high-confidence live reproduction is established. Source inspection shows current main can execute repeated first-class `read` calls without a default read-specific guard, but the exact webchat loop was not replayed.

## Solution Assessment

Unclear until maintainers decide the UX policy. The existing before-tool-call detector is the likely technical seam, but read-specific defaults, warning text, and thresholds should be chosen deliberately.
