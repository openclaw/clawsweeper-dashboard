## Summary

Keep open: current main and v2026.6.10 still provide only opt-in generic rolling tool-loop detection for repeated read calls, while the first-class read path has no default same-turn reread nudge. The remaining decision is product/runtime policy around default behavior, thresholds, warning text, and false-positive tolerance.

## What This Changes

The issue requests a warning-only same-turn guard for repeated first-class read-tool rereads that consume tokens without progress.

## Best Possible Solution

Extend the existing before-tool-call loop-detection seam with a narrowly scoped warning-only first-class read reread nudge after maintainers choose default behavior, thresholds, and warning text.

## Reproduction Assessment

No high-confidence live reproduction is established. Source inspection confirms the missing default guard: repeated first-class read calls are only warned when the opt-in generic detector is enabled.

## Solution Assessment

Unclear until maintainers decide the runtime UX policy. The best technical shape appears to reuse the before-tool-call detector rather than adding a separate read-wrapper history store, but default behavior and thresholds need owner choice.
