## Summary

Keep open. Current main still has only opt-in, generic exact-argument tool-loop detection around first-class `read`, so it does not implement the requested same-turn path-set/reread-ratio guard or transcript-visible nudge; the sibling shell-readback proposal at https://github.com/openclaw/openclaw/issues/63352 is also still open.

## What This Changes

Add a warning-only same-turn guard for first-class `read` calls that detects repeated rereads of the same file or small path set before token burn stalls a run.

## Best Possible Solution

Extend the existing loop-safety design with a maintainer-approved read-aware detector, path-set/reread-ratio heuristics, a clear soft nudge surface, focused tests, and docs for defaults and tuning.

## Reproduction Assessment

Yes, at source level. Current main routes first-class `read` through the generic hook, but only exact-argument generic detection exists, it is disabled by default, and there is no read-aware path-set/reread-ratio guard.

## Solution Assessment

Unclear. The warning-only direction is plausible, but maintainers still need to decide default enablement, thresholds, config shape, warning visibility, and coordination with the sibling exec-readback proposal.
