## Summary

Keep open. Current `main` still lacks a read-aware, model-visible same-turn reread guard; the existing guard is disabled by default, matches generic exact tool+argument repeats, and warning-level detections are diagnostics-only. The related #63352 remains a complementary exec-runtime proposal rather than a replacement for the first-class `read` path.

## What This Changes

The issue requests a warning-only same-turn reread guard for the first-class `read` tool so repeated reads of the same small file set nudge the model before wasting context.

## Best Possible Solution

Add a core read-aware guard that normalizes per-run file identities, detects repeated same-file and small-file-set rereads without nearby progress, returns a soft model-visible nudge, emits diagnostics, and documents/tests the warning-only behavior without changing exec policy by default.

## Reproduction Assessment

Yes for the missing product behavior: current source and tests show repeated `read` calls only hit the generic exact-params detector when opt-in loop detection is enabled, and warning-level results are diagnostic-only. The live UI transcript is not available in the repo, so focused synthetic tests should model the reported reread pattern.

## Solution Assessment

Yes. A narrow guard in the core agent/read-tool loop is the best fit because the failure mode is in the first-class `read` path, not an optional plugin surface; reusing existing history and diagnostic plumbing is appropriate, but read identity and model-visible nudging need dedicated logic.
