## Summary

Keep open. Current main has adjacent opt-in tool-loop detection that can warn on exact repeated `read` calls, but the default path is still off and the existing detectors do not cover the reported small-file-set reread ratio with no edit/test progress; #63352 is related but only tracks the exec-side form.

## What This Changes

Add a soft same-turn guard or nudge for repeated first-class `read` tool calls that burn tokens without edits, tests, or broadened context.

## Best Possible Solution

Extend the existing tool-loop detection surface, with maintainer-approved default behavior, to cover first-class `read` reread loops and align the shared read/exec policy with #63352.

## Reproduction Assessment

No live replay was established, but source inspection gives a high-confidence current-main path for the gap: default loop detection is off, and the tests explicitly allow repeated `read` calls without config.

## Solution Assessment

No, closing as implemented would be premature. The existing opt-in detector is useful groundwork, but the requested default/soft read-reread guard and small-file-set heuristics still need maintainer direction.
