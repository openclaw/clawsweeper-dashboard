## Summary

Keep open. Current main and the latest release have generic opt-in tool-loop detection plus a narrow post-compaction guard, but they still do not provide the requested default same-turn, first-class `read` path-set/reread-ratio nudge before token burn stalls a run.

## What This Changes

Requests a warning-only same-turn guard that detects repeated first-class `read` calls over the same small file set and nudges the agent before a run burns tokens without progress.

## Best Possible Solution

Extend the existing tool-loop/before-tool-call surface with a maintainer-approved read-specific same-turn heuristic and visible nudge, preferably without adding a new config surface unless the default behavior needs an explicit opt-in.

## Reproduction Assessment

No high-confidence live reproduction is available from this review. Source inspection does show the requested default read-specific guard is absent, but the concrete webchat/control-ui loop itself still needs live or harness proof.

## Solution Assessment

Unclear as an implementation path because this is a proposal, not a patch. The existing generic opt-in detector is a useful base, but it is not the best complete solution for the requested default read-specific in-turn nudge without a maintainer policy decision.
