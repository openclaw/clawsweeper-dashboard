## Summary

Keep open. Current main has adjacent exact-argument rolling detection and a post-compaction guard, but it still does not implement the requested same-turn first-class read guard for repeated reads over the same small file set without edit/test progress.

## What This Changes

The issue requests a warning-only same-turn guard that detects repeated first-class `read` calls over the same file or small file set before token burn or a stalled turn.

## Best Possible Solution

Define a maintainer-approved read-loop policy, then extend the existing detector or add a read-specific detector for same-turn file identity and small-file-set rereads with focused tests and docs, coordinated with https://github.com/openclaw/openclaw/issues/63352.

## Reproduction Assessment

Yes, at source level. Current main still ignores repeated `read` calls by default and, when enabled, only detects exact tool+argument repetition or two-pattern ping-pong rather than the reported small-file-set reread ratio with no edit/test progress.

## Solution Assessment

Unclear. A warning-only read guard is plausible, but the best implementation depends on maintainer decisions about enablement, thresholds, warning delivery, and how it should align with the exec-side proposal.
