## Summary

Keep open: current `main` and v2026.6.8 still provide only opt-in generic rolling tool-loop detection, while the first-class `read` path has no default same-turn reread nudge. The remaining decision is runtime/product policy around default behavior, thresholds, warning text, and false-positive tolerance.

## What This Changes

The issue requests a warning-only same-turn guard for repeated first-class `read` tool calls that reread the same small file set without progress.

## Best Possible Solution

Extend the existing before-tool-call loop-detection seam with a narrowly scoped first-class `read` reread nudge after maintainers choose the default behavior, thresholds, and warning text.

## Reproduction Assessment

No high-confidence live reproduction is established. Source inspection shows current main can execute repeated first-class `read` calls without a default read-specific guard, but the exact webchat loop was not replayed.

## Solution Assessment

Unclear until maintainers decide the UX policy. The existing before-tool-call detector is the likely technical seam, but read-specific defaults, warning text, and thresholds should be chosen deliberately.
