## Summary

Keep open. Current main still has only opt-in generic loop detection keyed by exact tool plus full params, so it does not satisfy the requested first-class read same-turn path-set/reread-ratio guard or in-turn soft nudge; this needs maintainer product review because it changes core runtime heuristics and warning behavior.

## What This Changes

Request a warning-only same-turn guard for first-class `read` calls that detects repeated rereads of the same file or small file set before token burn stalls a run.

## Best Possible Solution

Add a maintainer-approved read-aware detector under `tools.loopDetection` that normalizes read paths, tracks same-turn path sets and reread ratios, emits a clear soft nudge, and ships focused tests/docs coordinated with the sibling exec-reread proposal.

## Reproduction Assessment

Yes, at source level. Current main records first-class `read` calls only through generic exact-argument loop detection, so changed offsets/windows or a repeated small path set evade the requested normalized read reread guard.

## Solution Assessment

Unclear. A warning-only read-aware guard is plausible, but maintainers still need to decide thresholds, default behavior, config shape, warning visibility, and how it should share policy with the sibling exec-readback detector.
