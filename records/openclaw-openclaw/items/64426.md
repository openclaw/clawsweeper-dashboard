## Summary

Keep open. Current main still exposes first-class `read` calls to the generic tool-loop detector, but the implemented detector is opt-in, exact-argument based, diagnostic-only at warning level, and does not model same-turn reread ratio across a small file set with no edit/test progress.

## What This Changes

Request a warning-only same-turn guard for repeated first-class `read` calls across a small file set before token/context burn.

## Best Possible Solution

Extend the existing loop-safety design with read-aware path-set/reread-ratio heuristics, a transcript-visible soft nudge or synthetic warning behavior, focused tests, and updated loop-detection docs.

## Reproduction Assessment

Yes, at source level. Current main can observe first-class `read` through the shared hook, but only exact-argument generic detection exists and no default read-aware reread guard is present.

## Solution Assessment

Unclear. The requested warning-only direction is plausible, but maintainers still need to decide default behavior, thresholds, config shape, warning visibility, and alignment with the exec-readback proposal.
