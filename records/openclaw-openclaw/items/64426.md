## Summary

Keep open: current main and v2026.6.10 still leave rolling tool-loop detection disabled by default, and the first-class read tool has no read-specific same-turn reread nudge. The linked cross-turn duplicate was closed in favor of this issue, so this remains the canonical place for the first-class read policy decision.

## What This Changes

Requests a warning-only same-turn guard that detects repeated first-class read calls over the same small path set before token burn stalls a run.

## Best Possible Solution

Extend the existing before-tool-call loop-detection seam with a read-aware warning after maintainer policy approval, then cover the default/opt-in behavior with focused tests, docs, and live proof.

## Reproduction Assessment

No high-confidence live reproduction is established. Source inspection confirms the missing default guard: repeated first-class read calls are only warned when the opt-in generic detector is enabled.

## Solution Assessment

Unclear until maintainers decide the runtime UX policy. The best technical shape appears to reuse the before-tool-call detector rather than adding a separate read-wrapper history store, but default behavior and thresholds need owner choice.
