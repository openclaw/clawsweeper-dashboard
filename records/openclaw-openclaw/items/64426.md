## Summary

Keep open. Current main still only has generic, opt-in tool-loop detection around first-class `read`; it does not implement the requested read-aware same-turn path-set/reread-ratio guard or an in-turn soft nudge, and the linked exec-readback sibling remains open.

## What This Changes

The issue proposes a warning-only same-turn guard for repeated first-class `read` calls over the same file or small file set to reduce token burn and stalled turns.

## Best Possible Solution

Extend the existing loop-safety design with a maintainer-approved read-aware detector, path normalization, small path-set/reread-ratio heuristics, a clear soft-nudge surface, focused tests, and docs for defaults and tuning.

## Reproduction Assessment

Yes, at source level. Current main routes first-class `read` through the shared hook, but only generic exact-argument detection exists; there is no read-aware path-set/reread-ratio guard or synthetic in-turn warning result.

## Solution Assessment

Unclear. A warning-only read-aware guard is plausible, but maintainers still need to decide thresholds, default enablement, config shape, warning visibility, and coordination with the sibling exec-readback proposal.
