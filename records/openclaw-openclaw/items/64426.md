## Summary

Keep open. Current main still has generic, opt-in loop detection around first-class `read`, but it does not implement a read-aware same-turn path-set/reread-ratio guard or an in-turn soft nudge; the related exec-readback proposal is also still open.

## What This Changes

The issue requests a warning-only same-turn reread detector for the first-class `read` tool to reduce token burn from repeated reads of the same small file set.

## Best Possible Solution

Extend the existing loop-safety design with a maintainer-approved read-aware detector, path normalization, path-set/reread-ratio heuristics, a clear soft nudge surface, focused tests, and docs for defaults and tuning.

## Reproduction Assessment

Yes, at source level. Current main routes first-class `read` through the shared hook, but only generic exact-argument detection exists and there is no read-aware path-set/reread-ratio guard or synthetic in-turn warning result.

## Solution Assessment

Unclear. A warning-only read-aware guard is plausible, but maintainers still need to decide thresholds, default enablement, config shape, warning visibility, and coordination with the sibling exec-readback proposal.
