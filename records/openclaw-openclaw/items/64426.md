## Summary

Keep open. Current main still only has generic, opt-in tool-loop detection for first-class `read`; it does not implement the requested read-aware same-turn path-set/reread-ratio guard or in-turn soft nudge.

## What This Changes

Request adds a warning-only same-turn reread guard for the first-class `read` tool to reduce repeated file reads and token burn.

## Best Possible Solution

Extend `tools.loopDetection` with a maintainer-approved read-aware detector that normalizes paths, tracks same-turn path sets and reread ratios, emits a clear soft nudge, and ships focused tests plus docs for defaults and tuning.

## Reproduction Assessment

Yes, at source level. Current main routes first-class `read` through the shared hook, but only generic exact-argument detection exists and there is no read-aware path-set/reread-ratio guard or synthetic in-turn warning result.

## Solution Assessment

Unclear. A warning-only read-aware guard is plausible, but maintainers still need to decide thresholds, default enablement, config shape, warning visibility, and coordination with the sibling exec-readback proposal.
