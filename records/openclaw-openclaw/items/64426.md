## Summary

Keep open. Current main still has only generic, opt-in tool-loop detection around first-class `read`; it does not implement a read-aware same-turn path-set/reread-ratio guard or an in-turn soft nudge, and the related shell-readback proposal remains open.

## What This Changes

Requested a warning-only same-turn guard that detects repeated first-class `read` calls over the same file or small path set before token burn stalls progress.

## Best Possible Solution

Extend the existing loop-safety design with a maintainer-approved read-aware detector, path-set/reread-ratio heuristics, a clear soft nudge surface, focused tests, and docs for defaults and tuning.

## Reproduction Assessment

Yes, at source level. Current main routes first-class `read` through the shared hook, but only generic exact-argument loop detection exists and there is no read-aware path-set/reread-ratio guard or synthetic in-turn warning result.

## Solution Assessment

Unclear. The warning-only direction is plausible, but maintainers still need to decide default enablement, thresholds, config shape, warning visibility, and coordination with the sibling exec-readback proposal.
