## Summary

Keep open. Current main still routes first-class `read` through generic exact-argument tool-loop detection; it does not implement the requested same-turn path-set/reread-ratio guard or an in-turn soft nudge for repeated reads.

## What This Changes

The issue proposes a warning-only same-turn reread guard for the first-class `read` tool to reduce repeated file rereads and token burn.

## Best Possible Solution

Extend `tools.loopDetection` with a maintainer-approved read-aware detector that normalizes paths, tracks same-turn path sets and reread ratios, emits a clear soft nudge, and ships focused tests plus docs for defaults and tuning.

## Reproduction Assessment

Yes, at source level. Current main routes first-class `read` through generic exact-argument loop detection, and there is no normalized path-set/reread-ratio guard or model-visible reread nudge in the read path.

## Solution Assessment

Unclear. A warning-only read-aware guard is plausible, but maintainers still need to decide thresholds, default enablement, config shape, warning visibility, and coordination with the sibling exec-readback proposal.

## Risks / Open Questions

- Normalized paths and debug markers need privacy-safe telemetry rules before they become visible in logs or transcripts.
- The first-class read guard should stay aligned with the open exec-readback proposal at https://github.com/openclaw/openclaw/issues/63352.
