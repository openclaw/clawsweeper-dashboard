## Summary

Keep open. Current main still has only opt-in generic exact-argument tool-loop detection around first-class `read`; it does not implement a read-aware same-path/small-path-set reread guard or an in-turn soft nudge, and the related exec-readback proposal remains open.

## What This Changes

Request to add a warning-only same-turn guard for repeated first-class `read` calls against the same file or small file set.

## Best Possible Solution

Add a maintainer-approved read-aware detector under the existing loop-safety surface, with normalized read-path/path-set heuristics, a clear soft nudge contract, focused coverage, and docs/config alignment with the exec-readback sibling.

## Reproduction Assessment

Yes, at source level. Current main routes first-class `read` through the shared hook, but the detector only sees generic full-params patterns and has no read-aware path-set/reread-ratio or synthetic warning-result path.

## Solution Assessment

Unclear. A warning-only guard is plausible, but maintainers still need to decide thresholds, default enablement, config shape, warning visibility, and coordination with the exec-readback proposal.
