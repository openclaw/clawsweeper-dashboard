## Summary

Keep open. Current main still lacks a read-aware, model-visible reread guard; existing loop detection is opt-in, exact-params based, and warning-level results only emit diagnostics while the tool call proceeds.

## What This Changes

Request to add a warning-only same-turn reread detector for first-class `read` calls that catches repeated file or small-path-set reads without progress and nudges the agent.

## Best Possible Solution

Ship a core agent read-tool guard that normalizes per-run read file identities, detects repeated same-file and small-file-set rereads without nearby progress, and returns a soft model-visible nudge with diagnostics.

## Reproduction Assessment

Yes for the missing product behavior: focused tests can repeatedly invoke wrapped `read` with the same path or small path set on current main and observe that no default read-specific nudge is returned. The live incident itself is not replayable from repo artifacts.

## Solution Assessment

Yes, the requested warning-only read-aware core guard is the narrowest maintainable direction. The existing generic detector can provide plumbing, but it is not sufficient as the user-facing fix.
