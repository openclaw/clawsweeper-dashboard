## Summary

Keep open: current main and the latest release still only provide opt-in generic rolling tool-loop detection, while the first-class `read` path has no default same-turn reread nudge. The remaining blocker is maintainer product policy around default behavior, thresholds, warning text, and false-positive tolerance.

## What This Changes

The issue proposes a warning-only same-turn repeated-reread detector for the first-class `read` tool path.

## Best Possible Solution

Extend the existing before-tool-call loop-detection seam with a narrowly scoped first-class `read` reread nudge after maintainers choose default behavior, thresholds, and warning text.

## Reproduction Assessment

No high-confidence live reproduction is established. Source inspection shows repeated first-class `read` calls run through only the opt-in generic detector by default, but the specific webchat loop was not replayed.

## Solution Assessment

Unclear until maintainers decide the runtime UX policy. The existing before-tool-call detector is the likely seam, but read-specific defaults, warning text, and thresholds should be chosen deliberately.
