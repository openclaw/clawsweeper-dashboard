## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.85

Full review comments:

- **[P2] Decode every quoted-pair escape:** `src/media/fetch.ts:161-165`
  - body: The splitter treats `\;` inside a quoted filename as escaped so it does not split the parameter, but `unquoteContentDispositionValue` only consumes `\"` and `\\`. A valid header like `filename="quarter\; summary.txt"` will therefore return `quarter\; summary.txt` instead of `quarter; summary.txt`, leaving the same filename-mangling class this parser is meant to fix. Consume the backslash for any quoted-pair character and add that case to the new table test.
  - confidence: 0.88
