## What This Changes

The issue reports that the webchat Control UI can leave the left status panel in a thinking/in-progress state after a streamed LLM response has already finished.

## Best Possible Solution

Make terminal run status and refreshed session state win over stale abortability after completion, while preserving Stop/Queue controls for truly active runs.

Yes, the maintainable fix is narrow: separate true active-run abortability from stale session-row abortability for the status/context UI, then add a completed-turn regression test.
