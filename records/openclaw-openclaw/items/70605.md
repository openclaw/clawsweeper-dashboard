## Summary

Keep open. Current main still forwards struck-through context content, and this PR is a plausible narrow fix, but the proposed raw regex pass would also remove literal examples inside Markdown code spans and fences.

## What This Changes

The PR updates `sanitizeContextFileContentForPrompt()` and its tests to remove Markdown and HTML strikethrough content from injected context files before prompt assembly.

## Best Possible Solution

Revise the sanitizer so it removes rendered strikethrough content while preserving Markdown code fences and inline code, using the existing code-region helper or a Markdown-aware token pass, then extend the focused prompt tests around those boundaries.

## Reproduction Assessment

Yes. The current PR regex can be reproduced with a direct string check: inline code and fenced code containing `~~literal example~~` lose their literal content even though those regions should be preserved as prompt context.

## Solution Assessment

No. A raw regex pass over the entire context file is not the best fix because it cannot distinguish rendered strikethrough from literal code examples; masking code regions before stripping is the narrower maintainable path.
