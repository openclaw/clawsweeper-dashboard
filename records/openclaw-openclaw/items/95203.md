## What This Changes

The PR adds a Crabbox wrapper guard that rejects raw Blacksmith Testbox `tbx_...` reuse when the matching Crabbox-owned SSH key is missing, while preserving slug reuse and covered owned-key reuse.

## Best Possible Solution

Land the narrow wrapper guard after maintainer review and a successful remote changed-gate or owner-approved equivalent proof, preserving slug reuse and owned-key reuse as the PR does.

## Reproduction Assessment

Yes from source: current main accepts `run --provider blacksmith-testbox --id tbx_...` without checking the Crabbox key path before delegating to Crabbox, while Crabbox later runs Blacksmith with that key path. I did not run the live Blacksmith path in this read-only review.

## Solution Assessment

Yes; this is the narrow wrapper-owned fix for the observed failure mode, and it preserves the sibling slug and owned-key paths. A future upstream Crabbox validation hook could be cleaner, but it is not required for this focused OpenClaw wrapper guard.
