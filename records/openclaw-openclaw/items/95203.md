## What This Changes

The PR adds a Blacksmith Testbox `--id tbx_...` key-existence guard to `scripts/crabbox-wrapper.mjs` and covers missing-key, existing-key, and slug-reuse behavior in wrapper tests.

## Best Possible Solution

Keep the wrapper guard if maintainers want only Crabbox-created reusable Testbox ids to work through OpenClaw's Crabbox wrapper, then land after focused tests and a remote proof gate pass.

## Reproduction Assessment

Yes from source: current main lets `run --provider blacksmith-testbox --id tbx_*` proceed without checking for the expected Crabbox key, while Crabbox later uses `<user-config>/crabbox/testboxes/<id>/id_ed25519`. I did not run the live Blacksmith path in this read-only review.

## Solution Assessment

Yes; the guard is narrow, wrapper-owned, and preserves slug reuse plus owned-key reuse. A first-class Crabbox validation hook would be cleaner long term, but this is a maintainable repo-local fix for the observed failure mode.
