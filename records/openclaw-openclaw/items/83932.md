The issue asks OpenClaw to give native Codex runtime compaction the same user-visible, status, or Gateway-log observability that users see for Pi/embedded compaction.

## Best Possible Solution

Add a narrow normal Gateway info log for Codex app-server `contextCompaction` start/end on the existing event path, while preserving `notifyUser: false` as the default for chat notices and leaving broader native `/compact` status work to the existing related issues.

## Solution Assessment
