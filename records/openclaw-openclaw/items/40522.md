## Summary

Keep this PR open. Current main still falls through `GET /` and `HEAD /` when `gateway.controlUi.basePath` is set, which leads to the generic gateway `Not Found` response; the PR remains a focused and reasonable implementation candidate for that missing redirect.

## What This Changes

The PR adds a read-method redirect from gateway root `/` to the configured Control UI `basePath`, preserves query strings, and adds routing tests for root redirect and non-read fallthrough.

## Best Possible Solution

Land or reapply the narrow routing change in the Control UI classifier so read requests to `/` redirect to `${basePath}/${search}` when a non-root `gateway.controlUi.basePath` is configured, while non-read root requests continue to fall through to plugin or webhook handlers.

## Reproduction Assessment

Yes. A focused source-level reproduction is `classifyControlUiRequest({ basePath: "/openclaw", pathname: "/", search: "", method: "GET" })` on current main, which returns `not-control-ui` and then falls through to the gateway 404 path.

## Solution Assessment

Yes. The PR targets the smallest maintainable surface: the existing Control UI routing classifier plus colocated unit tests, without changing plugin routing, auth, or broader server behavior.
