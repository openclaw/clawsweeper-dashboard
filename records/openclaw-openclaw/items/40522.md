## Summary

Keep open: current main and the latest release still fall through gateway root `/` when `gateway.controlUi.basePath` is set, and this PR is a narrow, correct classifier fix. The remaining merge blocker is missing contributor-supplied real gateway proof.

## What This Changes

The PR redirects GET/HEAD requests for gateway root `/` to the configured Control UI basePath and adds classifier tests for redirect, query preservation, and POST fallthrough.

## Best Possible Solution

Refresh the branch if needed, keep the classifier-owned root-to-basePath redirect with focused tests, and merge after redacted real gateway proof demonstrates the redirect on a running gateway.

## Reproduction Assessment

Yes by source inspection. On current main and `v2026.5.12`, `basePath: "/openclaw"`, `pathname: "/"`, and `GET` return `not-control-ui`, after which the gateway fallback emits `404 Not Found`.

## Solution Assessment

Yes. The classifier is the narrow maintainable fix point because redirect responses are already handled by `handleControlUiHttpRequest`, plugin routes run earlier, and the PR preserves non-read root request passthrough.
