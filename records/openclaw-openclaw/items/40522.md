## What This Changes

This PR adds a Control UI classifier redirect from gateway root `/` to the configured basePath for read requests, with tests for redirect, query preservation, and write-method fallthrough.

## Best Possible Solution

Land or reapply a current-main-compatible classifier fix that redirects GET/HEAD `/` to `<basePath>/`, preserves query strings and non-read fallthrough, and includes focused regression tests plus real gateway proof.

## Reproduction Assessment

Yes by source inspection: `basePath: "/openclaw"`, `pathname: "/"`, and `method: "GET"` still return `not-control-ui` on current main, after which the gateway can fall through to `Not Found`. I did not run artifact-producing tests or a live gateway during this read-only review.

## Solution Assessment

Yes for the scoped basePath-root behavior: the classifier is the narrowest maintainable fix point because redirect handling is already centralized in the Control UI HTTP handler. It should not be presented as the complete fix for every historical root-404 report in the linked issue.
