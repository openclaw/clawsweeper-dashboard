## Summary

Keep open: the patch targets a real current-main routing gap, but the external PR still lacks after-fix real gateway proof, is merge-conflicting, and changes ownership of the gateway root read path when a Control UI basePath is configured.

## What This Changes

The PR redirects gateway root GET/HEAD requests to the configured Control UI basePath and adds classifier tests for redirect, query preservation, and POST fallthrough.

## Best Possible Solution

Land a rebased classifier-level root redirect only after maintainers accept the root-path policy and real gateway proof shows GET `/`, query-preserving GET `/`, and POST `/` fallthrough with a configured basePath.

## Reproduction Assessment

Yes, from source inspection: current main classifies `basePath=/openclaw` plus `pathname=/` as `not-control-ui`, and the gateway fallback returns plain `Not Found` if earlier stages do not handle it. I did not run a live gateway in this read-only review.

## Solution Assessment

Mostly yes: putting the redirect in the Control UI classifier is narrow and the proposed tests cover GET, query preservation, and POST fallthrough. It still needs a rebase, real gateway proof, and maintainer acceptance of the root-path compatibility tradeoff.
