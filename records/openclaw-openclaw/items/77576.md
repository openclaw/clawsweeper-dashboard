## Summary

Keep open: this is a concrete regression with current-main source evidence and an open linked fix PR that closes this issue, so the issue should stay open until that PR lands or is explicitly replaced.

## What This Changes

Telegram group/topic session replies are reported to route to WebChat instead of Telegram when stale WebChat origin metadata is present.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/77611 so explicit persisted Telegram delivery routes win over stale WebChat origin metadata while preserving WebChat no-inherit and group visible-reply policy.

## Reproduction Assessment

Yes, at source level: current `deliveryContextFromSession` can seed routing from stale WebChat channel/origin metadata even when the stored delivery context is Telegram. I did not run a live Telegram group/topic reproduction in this read-only review.

## Solution Assessment

Yes: keeping this issue open behind the linked fix PR is the best path, because the PR targets the implicated helper and adds focused regression coverage. The remaining validation gap is real Telegram proof before merge.
