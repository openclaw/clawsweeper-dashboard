## Summary

Keep open: current main still has the stale WebChat-origin delivery-context precedence hazard, and https://github.com/openclaw/openclaw/pull/77611 is an open closing PR for this exact regression. Cleanup policy should leave this issue open until that PR lands or is explicitly rejected.

## What This Changes

Telegram group/topic session replies should route back to Telegram even when the session has stale WebChat origin metadata.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77611, or close that PR with maintainer rationale and keep this issue as the canonical bug until an equivalent fix is merged.
