## Summary

Keep open: current main still has a source-level route-precedence hazard around stale WebChat origin metadata, multiple users report the behavior on v2026.5.5, and the linked fix PR at https://github.com/openclaw/openclaw/pull/77611 is still open rather than merged.

## What This Changes

The issue asks to fix a regression where Telegram group/topic sessions with stale WebChat origin metadata send generated replies to WebChat instead of back to Telegram.

## Best Possible Solution

Update and land https://github.com/openclaw/openclaw/pull/77611, or an equivalent focused fix, so explicit Telegram deliveryContext/lastRoute data wins over stale WebChat origin metadata while preserving the WebChat no-inherit tests.

## Reproduction Assessment

## Solution Assessment
