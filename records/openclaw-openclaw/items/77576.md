Keep open: the report describes a current, user-visible regression against the documented channel-routing contract, and current main does not show a later fix for this exact v2026.5.3-1 behavior. ## What This Changes

The issue reports Telegram group/topic agent replies being routed to WebChat instead of delivered back to the originating Telegram group after updating to 2026.5.3-1.

## Best Possible Solution

Add a focused regression that simulates a Telegram group topic turn with an existing session whose origin is webchat but deliveryContext/lastRoute are Telegram, then make the Telegram-originated reply path prefer live Telegram routing while preserving the WebChat no-inherit test.
