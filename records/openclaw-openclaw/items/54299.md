## Summary

Keep open: current main and the latest release still conditionally omit `client_secret` and use a fixed `localhost:8085` Google OAuth callback. The right fix needs maintainer security/product policy because Google treats installed-app `client_secret` as optional while recommending a random available loopback port. ([developers.google.com](https://developers.google.com/identity/protocols/oauth2/native-app))

## What This Changes

The issue asks OpenClaw to stop silently running Google OAuth token exchange without a client secret and to avoid a hardcoded OAuth redirect port.

## Best Possible Solution

Define OpenClaw's supported Google OAuth credential modes and loopback redirect policy, then enforce that contract in the Google plugin with focused tests and docs.

## Reproduction Assessment

Yes, by source inspection. Current main and `v2026.5.12` accept an env client ID without a secret, conditionally omit `client_secret` during token exchange, and bind the local callback helper to port 8085.

## Solution Assessment

Unclear. Throwing on missing `client_secret` is not obviously the best fix because Google documents installed-app secrets as optional, while the fixed callback port conflicts with loopback guidance; maintainers should choose the contract before code changes. ([developers.google.com](https://developers.google.com/identity/protocols/oauth2/native-app))
