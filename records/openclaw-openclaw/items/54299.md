## Summary

Keep open: current main and v2026.5.18 still allow Google Gemini CLI OAuth to run without a client secret and still build the callback around localhost:8085, but the correct fix is a maintainer security/product decision because Google documents installed-app secrets as optional while recommending a random loopback port.

## What This Changes

The issue asks OpenClaw to reject or warn when Google OAuth runs without client_secret and to make the Gemini CLI OAuth callback port configurable or dynamic instead of fixed at localhost:8085.

## Best Possible Solution

Define OpenClaw's supported Google Gemini CLI OAuth client modes and loopback redirect contract, then enforce and document that policy in the Google plugin with focused tests.

## Reproduction Assessment

Yes, by source inspection. Current main and v2026.5.18 accept an env OAuth client ID without a paired secret, conditionally omit client_secret, and bind the local callback helper to port 8085.

## Solution Assessment

Unclear. Throwing on missing client_secret is not obviously the best fix because Google's installed-app guidance treats client_secret as optional, while the fixed callback port should be reviewed against random-port loopback guidance and OpenClaw's current manual fallback.
