## Summary

Keep open: current main and v2026.5.18 still have the reported Google Gemini CLI OAuth behavior, and the remaining choice is security/product policy rather than conservative cleanup.

## What This Changes

The issue asks OpenClaw to stop silently running Google Gemini CLI OAuth without a paired client secret and to remove or configure the fixed localhost:8085 redirect listener.

## Best Possible Solution

Define the supported Gemini CLI OAuth credential modes and loopback redirect policy, document them, and enforce the chosen contract in the Google plugin with focused coverage for env credentials, token body construction, callback binding, and manual fallback.

## Reproduction Assessment

Yes, by source inspection. Current main and v2026.5.18 accept an env client ID without a paired secret, omit client_secret when absent, and bind the callback listener to port 8085 before falling back to manual mode on listen failures.

## Solution Assessment

Unclear. Throwing on a missing client_secret may conflict with installed-app OAuth semantics, and changing the loopback port needs a redirect URI compatibility plan rather than a one-line code cleanup.
