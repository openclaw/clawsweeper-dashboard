## Summary

Keep open: current main and the latest release still conditionally omit `client_secret` and use a fixed `localhost:8085` Google OAuth callback. The right fix needs maintainer security/product policy because Google's native-app OAuth guidance treats installed apps as unable to keep secrets while also recommending a loopback listener on an available random port. ([developers.google.com](https://developers.google.com/identity/protocols/oauth2/native-app?utm_source=openai))

## What This Changes

The issue asks OpenClaw to stop silently omitting the Google OAuth `client_secret` and to avoid a hardcoded `localhost:8085` redirect port.

## Best Possible Solution

Define OpenClaw's supported Google OAuth credential modes and loopback redirect policy, then enforce that contract in the Google plugin with tests and docs.

## Reproduction Assessment

Yes, by source inspection. Current main and `v2026.5.18` still accept an env client ID without a secret, conditionally omit `client_secret` from token requests, and bind the OAuth callback helper to port 8085.

## Solution Assessment

Unclear. Throwing when `client_secret` is missing is not automatically the best fix because Google's native-app guidance treats installed-app secrets as non-confidential, while the fixed callback port does need an explicit redirect policy decision. ([developers.google.com](https://developers.google.com/identity/protocols/oauth2/native-app?utm_source=openai))
