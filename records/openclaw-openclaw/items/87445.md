## Summary

Keep open: current main and `v2026.6.6` still expose the model-facing `subagents` tool as list-only, and the merged timeout fix explicitly left manual kill/reap behavior out of scope. Existing owner-scoped kill primitives make this bounded, but the mutating tool API and stale-reaper policy still need maintainer approval.

## What This Changes

The issue requests model-facing `subagents` tool actions for targeted child-run recovery: `kill` and optional `reap-stale`.

## Best Possible Solution

Approve a narrow owner-scoped `subagents` control API that reuses existing kill primitives, documents the schema, and tests targeted kill, parent wake, and stale-skip behavior; otherwise keep the tool status-only and document operator recovery.

## Reproduction Assessment

Yes, at source level: current main and `v2026.6.6` have a list-only `subagents` action enum and unsupported-action fallback, so the requested `kill` and `reap-stale` tool actions are absent. I did not run a live ghost-child chain in this read-only pass.

## Solution Assessment

Unclear until maintainers choose the API shape. Reusing `killControlledSubagentRun` is the narrow implementation path, but `reap-stale` thresholds and parent completion semantics are product and safety decisions.
