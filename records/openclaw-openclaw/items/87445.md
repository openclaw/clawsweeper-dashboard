## Summary

Keep open: current main and v2026.6.6 still expose the model-facing `subagents` tool as list-only, while https://github.com/openclaw/openclaw/pull/87489 fixed timeout prevention and explicitly left manual kill/reap out of scope. Existing internal/admin kill primitives make the implementation path bounded, but a mutating model-facing API and stale-reaper policy still need maintainer approval.

## What This Changes

The issue requests `subagents` tool actions for targeted kill and optional stale reaping so stuck child runs can be terminated without restarting the gateway.

## Best Possible Solution

Approve a narrow owner-scoped `subagents` control API that reuses existing kill primitives, documents the schema, and tests targeted kill, parent wake, and stale-skip behavior; otherwise keep the current visibility-only tool and document operator recovery.

## Reproduction Assessment

Yes for the missing tool surface: current main and v2026.6.6 have a list-only `subagents` action enum and unsupported actions error. I did not run a live ghost-child chain in this read-only pass.

## Solution Assessment

Unclear until maintainers choose the API shape. Reusing `killControlledSubagentRun` and related registry termination primitives is the narrow path, but `reap-stale` thresholds and parent completion semantics need product and safety approval.
