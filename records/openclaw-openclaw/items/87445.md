## Summary

Keep open: current main and the latest release still expose the model-facing `subagents` tool as list-only, while related timeout and task-cleanup work does not implement `kill` or `reap-stale`. This remains a valid core recovery request, but exposing a mutating model-facing control surface needs maintainer safety and product direction before a repair lane.

## What This Changes

The issue requests `subagents` tool actions to kill a targeted stuck child run and optionally reap stale child runs in bulk.

## Best Possible Solution

Maintainers should approve a bounded model-facing control API or choose an operator-only alternative; any implementation should reuse existing ownership-scoped kill primitives and define parent wake, cascade, stale-age, and task-cleanup semantics.

## Reproduction Assessment

Yes for the missing tool surface: current main and v2026.6.9 both expose only `subagents(action=list)` and return an unsupported-action error for anything else. I did not run a live ghost-child chain in this read-only pass.

## Solution Assessment

Unclear until maintainers choose the safety model. Reusing `killControlledSubagentRun` and related registry termination primitives is the narrow technical base, but target aliases, cascade behavior, `reap-stale` policy, and parent completion semantics need approval first.
