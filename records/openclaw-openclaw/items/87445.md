## Summary

Keep open: current main and the latest release still expose the model-facing `subagents` tool as list-only. Existing kill primitives and adjacent timeout/task-cleanup work help the underlying recovery story, but the requested `kill`/`reap-stale` tool surface remains a product and safety decision.

## What This Changes

Request to add model-facing `subagents` actions for killing a targeted child run and optionally reaping stale child runs in bulk.

## Best Possible Solution

If maintainers approve the direction, implement a narrow ownership-scoped `kill` action using the existing control primitives, and decide separately whether bulk `reap-stale` belongs in the model tool or operator maintenance.

## Reproduction Assessment

Yes for the missing tool surface: current main and `v2026.6.9` both show a list-only `subagents` action enum and unsupported-action fallback. I did not run a live ghost-child chain in this read-only pass.

## Solution Assessment

Unclear until maintainers choose the API shape. The existing ownership-scoped kill primitives are a plausible base, but exposing mutating `kill` or `reap-stale` to models needs product and safety approval first.
