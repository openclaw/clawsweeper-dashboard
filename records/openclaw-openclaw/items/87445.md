## Summary

Keep open: current main and v2026.6.9 still expose the model-facing `subagents` tool as list-only. Related timeout and task-row work reduces adjacent failure modes, but adding mutating `kill` or `reap-stale` actions remains an unresolved product/safety decision.

## What This Changes

Request to add `kill` and optional `reap-stale` actions to the model-facing `subagents` tool so stuck child runs can be terminated without restarting the gateway.

## Best Possible Solution

Maintainers should approve a bounded model-facing control API or choose an operator-only alternative; any implementation should reuse the existing ownership-scoped kill primitives and cover parent wake, task cleanup, docs, and regression tests.

## Reproduction Assessment

Yes for the missing tool surface: current main and v2026.6.9 both have a list-only `subagents` action enum and unsupported-action branch. I did not run a live ghost-child chain in this read-only pass.

## Solution Assessment

Unclear until maintainers choose the API shape. Reusing `killControlledSubagentRun` and related registry termination primitives is the narrow technical path, but target aliases, cascade behavior, `reap-stale` policy, and parent completion semantics need product and safety approval first.
