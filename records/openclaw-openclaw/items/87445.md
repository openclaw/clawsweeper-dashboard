## Summary

Keep open: current `main` and latest release still expose the model-facing `subagents` tool as list-only, while the related timeout fix explicitly left manual kill/reap behavior out of scope. Existing internal kill primitives and task-cancel wiring make the implementation path plausible, but exposing a mutating model-facing kill/reap API still needs maintainer product/safety approval.

## What This Changes

The issue requests adding `subagents` tool actions to kill a stuck child run and optionally reap stale child runs.

## Best Possible Solution

Define a narrow, owner-scoped `subagents` control API that reuses existing kill primitives, documents the schema, tests parent wake and stale-target safety, and coordinates with killed-task finalization before exposing bulk stale reaping.

## Reproduction Assessment

Yes for the missing tool surface: current source and the latest release both show the model-facing `subagents` action enum is still list-only. I did not live-reproduce a ghost child chain, but the requested API absence is source-reproducible.

## Solution Assessment

Unclear until maintainers choose the API shape. Reusing `killControlledSubagentRun` and `killAllControlledSubagentRuns` is likely the narrow implementation path, but stale reaping, model authorization, parent wake semantics, and task cleanup need approval.
