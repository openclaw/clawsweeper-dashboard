## Summary

Keep open: current main and v2026.6.6 still expose the model-facing `subagents` tool as list-only. The merged timeout fix reduced ghost-run risk but explicitly left manual kill/reap out of scope; current admin kill primitives are useful implementation building blocks, not the requested model-facing API.

## What This Changes

The issue requests adding `kill` and optional `reap-stale` actions to the model-facing `subagents` tool so stuck child runs can be terminated without a full gateway restart.

## Best Possible Solution

Maintainers should choose whether recovery stays operator-only or becomes an owner-scoped model-facing `subagents` control API; if approved, reuse the existing kill primitives and add focused schema, parent-wake, cascade, stale-skip, and docs coverage.

## Reproduction Assessment

Yes for the missing tool surface: current main and v2026.6.6 have a list-only `subagents` action enum and unsupported actions error. I did not run a live ghost-child chain in this read-only pass.

## Solution Assessment

Unclear until maintainers choose the API shape. Reusing `killControlledSubagentRun` and related registry termination primitives is the narrow implementation path, but `reap-stale` policy and parent completion semantics need product and safety approval first.
