## Summary

Keep open: current main and latest release v2026.6.8 still expose the model-facing `subagents` tool as list-only. Existing internal/admin kill primitives and the merged timeout fix reduce the recovery gap, but they do not implement the requested `kill` or `reap-stale` actions, and exposing that mutating API still needs maintainer product/safety approval.

## What This Changes

The issue requests model-facing `subagents` tool actions for killing one stuck child run and bulk reaping stale child runs.

## Best Possible Solution

Keep this open for maintainer product/safety review; if approved, implement a bounded `subagents` control API using existing owner-scoped kill primitives plus schema, docs, and regression coverage.

## Reproduction Assessment

Yes for the missing tool surface: current main and v2026.6.8 have a list-only `subagents` action enum and unsupported-actions branch. I did not run a live ghost-child chain in this read-only pass.

## Solution Assessment

Unclear until maintainers choose the API shape. Reusing `killControlledSubagentRun` and related registry termination primitives is the narrow technical path, but `reap-stale` policy and parent completion semantics need product and safety approval first.
