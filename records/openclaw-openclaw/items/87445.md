## Summary

Keep open: current main and latest release v2026.6.6 still expose the model-facing `subagents` tool as list-only. Internal/admin kill primitives and the merged timeout fix reduce the recovery gap, but they do not implement the requested `kill` or `reap-stale` tool actions, and the remaining change needs maintainer product/safety approval.

## What This Changes

Request to add model-facing `subagents(action="kill")` and optional `subagents(action="reap-stale")` controls for stuck child runs.

## Best Possible Solution

Maintainers should choose whether recovery stays operator-only or becomes an owner-scoped model-facing `subagents` control API; if approved, reuse the existing kill primitives and add schema, target resolution, parent wake, stale-skip, docs, and tests together.

## Reproduction Assessment

Yes for the missing tool surface: current main and v2026.6.6 have a list-only `subagents` action enum and unsupported-actions path. I did not run a live ghost-child chain in this read-only pass.

## Solution Assessment

Unclear until maintainers choose the API shape. Reusing `killControlledSubagentRun` and related registry termination primitives is the narrow technical path, but `reap-stale` policy and parent completion semantics need product and safety approval first.
