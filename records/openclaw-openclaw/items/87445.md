## Summary

Keep open: current `main` and the latest release still expose the model-facing `subagents` tool as list-only, while the requested targeted recovery surface remains missing. There are internal kill/cancel primitives and a CLI task-cancel workaround, so this should be handled as a bounded core tool-surface/product decision rather than closed as already implemented.

## What This Changes

The issue requests `subagents` tool actions for killing a targeted child run and optionally reaping stale child runs without restarting the gateway.

## Best Possible Solution

Add a narrow, owner-scoped `subagents` control surface only after maintainer API approval, reusing the existing subagent-control and registry termination primitives, documenting the schema, and testing parent wake/termination semantics plus stale-target safety; otherwise document `openclaw tasks cancel` as the supported operator escape hatch.

## Reproduction Assessment

Yes for the missing tool surface: current source and the latest release show the `subagents` action enum is only `list`. I did not reproduce a live ghost child chain; the operational failure mode is supported by the issue report and related subagent-control code paths.

## Solution Assessment

Unclear until maintainers choose the API shape. Reusing `killControlledSubagentRun`/`killAllControlledSubagentRuns` is likely the narrow implementation path, but `reap-stale` thresholds and parent wake behavior need explicit product/runtime approval.
