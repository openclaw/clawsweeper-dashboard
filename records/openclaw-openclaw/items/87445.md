## Summary

Keep open: current main and v2026.6.9 still expose the model-facing `subagents` tool as list-only. The existing internal/admin kill primitives and the merged timeout fix are useful adjacent work, but they do not implement the requested `kill` or `reap-stale` tool actions, and that mutating model-facing API still needs maintainer product and safety review.

## What This Changes

Request exposing model-facing `subagents` actions to kill a target child run and optionally bulk-reap stale active child runs.

## Best Possible Solution

Keep this open for maintainer product and safety review; if approved, expose a bounded owner-scoped control API on the existing subagent kill primitives with schema, docs, and regression coverage.

## Reproduction Assessment

Yes for the missing tool surface: current main and v2026.6.9 both expose only `subagents(action="list")` and return an unsupported-action error otherwise. I did not run a live ghost-child chain in this read-only pass.

## Solution Assessment

Unclear until maintainers choose the API shape. The existing owner-scoped kill primitives are the likely implementation base, but exposing them to model calls requires product and safety decisions for targeting, cascade, stale detection, and parent wake semantics.
