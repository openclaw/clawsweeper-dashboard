## Summary

Keep open. The context-engine startup fix is supported by current main source, but the bundled tool-registry change is incomplete: its reload branch only runs when no registry exists, while the reported #76598 path can return a loaded-but-empty registry.

## What This Changes

The branch adds selected context-engine slot plugins to the gateway startup plan, adds plugin-tool cold-load diagnostics/fallback code, updates focused tests, and adds changelog bullets for #76576 and #76598.

## Best Possible Solution

Merge the context-engine startup-plan change with its regression tests, and either correctly handle loaded-empty tool registries for #76598 or remove that bundled claim before landing.

## Reproduction Assessment

Yes. On current main, a selected external context-engine plugin without `activation.onStartup` is omitted from startup plugin IDs, and agent startup loads only those IDs before resolving the context engine.

## Solution Assessment

No as submitted. The selected context-engine hook is the narrow maintainable fix for #76576, but the bundled #76598 fallback does not cover loaded-empty registries and should be fixed or removed.
