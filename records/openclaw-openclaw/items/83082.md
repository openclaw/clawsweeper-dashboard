## What This Changes

The PR changes three usage/context Markdown docs lines to say `/usage` footers apply to normal final replies, not explicit `message(action: send)` output such as `messages.groupChat.visibleReplies: "message_tool"`.

## Best Possible Solution

Land the docs clarification if current message-tool semantics remain the product contract; if maintainers instead accept runtime footer delivery for message-tool sends, update these docs with that behavior in the same change.

## Reproduction Assessment

Yes. Source inspection shows the usage footer is appended to final reply payloads after the run, while visible `message(action=send)` output uses the outbound message-tool path and message-tool-only finals are suppressed.

## Solution Assessment

Yes. For this PR, documenting the current final-reply-only footer contract is the narrowest maintainable fix; changing message-tool footer delivery is a separate runtime/product path with privacy concerns.
