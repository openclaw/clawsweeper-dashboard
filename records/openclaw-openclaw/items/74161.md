## Summary

## What This Changes

The PR changes the skills prompt so exactly-one-skill selection tells agents to use the exact `<location>` from `<available_skills>` instead of guessing or hard-coding a skill file path.

Complete the narrow prompt hardening by applying the exact-`<location>` / never-guess rule to every skills selection path, add focused prompt coverage for both branches, and record the user-facing agents fix in the changelog while leaving the broader sandbox path-resolution issues on their existing tracks.
