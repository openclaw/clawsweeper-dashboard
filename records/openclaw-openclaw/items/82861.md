The PR adds a fresh `model-run-<uuid>` session id and matching explicit session key to each gateway `infer model run` request, with a regression test requiring distinct session routing across invocations.

## Solution Assessment

Yes for the code path: using the existing explicit-session helper with a fresh model-run UUID is the narrowest maintainable fix for isolating one-shot gateway probes. The remaining blocker is proof and validation, not a different design.
