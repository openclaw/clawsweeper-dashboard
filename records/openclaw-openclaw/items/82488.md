## Summary

Keep open: the false-positive target is real, but this PR would skip test-named files in deep plugin audits without preserving runtime-reachable imports, and the posted proof is still unit/tooling output rather than a real security-audit run.

## What This Changes

The PR passes `excludeTestFiles: true` into the shared deep code-safety scan helper and adds a regression test that plugin audits request that option.

## Best Possible Solution

Build the deep plugin audit scan set from manifest-entry runtime reachability before excluding unrelated test fixtures, then prove the result with focused tests and real audit output.

## Reproduction Assessment

Yes. Source inspection shows current main includes test files in deep plugin scans by default, and the linked issue provides concrete Discord false-positive examples; the patch risk is also source-reproducible from the scanner and caller contracts.

## Solution Assessment

No, not as written. Passing the existing exclusion option is the right direction, but the deep audit must preserve manifest-entry runtime import coverage before skipping test-like paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve runtime graph coverage when excluding tests:** `src/security/audit-extra.async.ts:256`
