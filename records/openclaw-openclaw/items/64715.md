## Summary

Keep this open. Current main still routes Telegram outbound controls through inline keyboards only; the send/action/message-tool surfaces and docs do not expose Telegram native reply-keyboard rows or keyboard-removal options, and the related closed items are about inline buttons or callback behavior rather than this separate capability.

## What This Changes

The issue requests native Telegram reply keyboard support through the agent/message outbound send surface, including rows/buttons, remove keyboard, resize_keyboard, and one_time_keyboard.

## Best Possible Solution

Add an additive Telegram plugin-owned reply-keyboard/reply_markup path for outbound sends, with docs and tests, while preserving existing inline-button behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows the exposed Telegram send/action/message-tool route only maps controls to inline keyboards and has no reply-keyboard or keyboard-removal fields.

## Solution Assessment

Yes. The narrow maintainable fix is additive Telegram plugin support for native reply keyboards; a broader portable keyboard contract should wait unless maintainers explicitly choose that direction.
