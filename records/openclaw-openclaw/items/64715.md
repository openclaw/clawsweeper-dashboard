## Summary

Keep this issue open: current main still exposes Telegram inline keyboards through the message/agent send path but has no native reply-keyboard or remove-keyboard surface, and the related closed items cover inline buttons/callback handling rather than this separate capability.

## What This Changes

Add native Telegram reply keyboard and remove-keyboard support, including rows/buttons, resize_keyboard, and one_time_keyboard, to the agent/message send surface.

## Best Possible Solution

Implement an additive Telegram plugin-owned reply_markup path for outbound sends, with typed reply keyboard rows, keyboard removal, resize_keyboard, one_time_keyboard, docs, and tests preserving existing inline keyboard behavior.

## Solution Assessment

Yes for the narrow direction. The maintainable fix is Telegram-owned additive schema/runtime/docs support; broader portable presentation semantics should be left out unless maintainers explicitly choose that product direction.
