## Summary

Keep this open. Current main still exposes Telegram outbound controls as inline keyboards only, the message/action/tool docs and schemas do not include native reply-keyboard or keyboard-removal fields, and the related closed items cover inline buttons or callback handling rather than this separate Telegram reply-keyboard capability.

## What This Changes

Add native Telegram reply keyboard and keyboard-removal support to the agent/message outbound send surface.

## Best Possible Solution

Add an additive Telegram plugin-owned reply-keyboard/reply_markup path for outbound sends, with docs and tests for keyboard rows, keyboard removal, resize_keyboard, and one_time_keyboard while preserving existing inline-button behavior.

## Reproduction Assessment

Yes, for the capability gap. Source inspection on current main shows the outbound Telegram send/action/message-tool path only builds inline keyboards, and exact searches plus docs/tests show no native reply keyboard or keyboard-removal surface.

## Solution Assessment

Yes. The narrow maintainable path is an additive Telegram-specific send/action/schema surface; a broader portable persistent-keyboard presentation contract can wait unless maintainers explicitly choose that direction.
