## Summary

Keep this issue open. Current main still only exposes Telegram inline keyboards through the message/agent send path; native reply keyboards and keyboard removal remain absent, and the related closed issues cover inline-button/callback behavior rather than this separate capability.

## What This Changes

Add native Telegram reply keyboard and remove-keyboard support, including rows/buttons, resize_keyboard, and one_time_keyboard, to the agent/message send surface.

## Best Possible Solution

Implement this as an additive Telegram plugin-owned reply_markup path with typed reply keyboard rows, keyboard removal, docs, and regression tests while preserving existing inline-keyboard behavior.
