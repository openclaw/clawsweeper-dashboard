## Summary

Keep this open. Current main still exposes Telegram outbound interactive controls as inline keyboards only, the message/tool schema and docs do not include native reply-keyboard or keyboard-removal fields, and the exact GitHub search did not find an open implementation PR that supersedes this request.

## What This Changes

Add native Telegram reply keyboard and keyboard-removal support to the agent/message send surface, including rows/buttons, resize_keyboard, one_time_keyboard, and remove keyboard semantics.

## Best Possible Solution

Add an additive Telegram plugin-owned reply keyboard/reply_markup path for outbound sends, with docs and tests for reply keyboard rows, keyboard removal, resize_keyboard, and one_time_keyboard while preserving existing inline-button behavior.

## Reproduction Assessment

Yes for the capability gap. Source inspection on current main shows the outbound Telegram send/action/message-tool path only builds inline keyboards, and the relevant docs omit native reply keyboard and removal fields.

## Solution Assessment

Yes. The best path is an additive Telegram-specific send/action/schema surface; a broader portable presentation contract can be deferred unless maintainers intentionally choose that API direction.
