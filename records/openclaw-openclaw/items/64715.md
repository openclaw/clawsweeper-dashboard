Keep this open. ## What This Changes

Request first-class Telegram native reply keyboard and keyboard-removal support through the agent/message outbound send surface.

## Best Possible Solution

Add an additive Telegram plugin-owned reply-keyboard/reply_markup path for outbound sends, with docs and tests for keyboard rows, keyboard removal, resize_keyboard, and one_time_keyboard while preserving existing inline-button behavior.

## Solution Assessment

Yes. The narrow maintainable solution is an additive Telegram-specific reply-keyboard send/action/schema surface; a broader portable persistent-keyboard contract can wait unless maintainers explicitly choose it.
