  - body: Current main changed `resolveExecDetail` to take an options object for `ToolDetailMode` and to handle both `exec` and `bash`; this patch changes the second argument into a number and only threads it through the older exec path.   - body: The new WebChat `/verbose limit` branch patches the session but returns only content. `dispatchSlashCommand` refreshes the active session snapshot only when the result action is `refresh`, so tool cards keep rendering with the previous limit until another refresh or navigation happens.
  - confidence: 0.86
- **[P2] Clear the TUI limit when session info omits it:** `src/tui/tui-session-actions.ts:180-183`
  - body: This update only applies `verboseLimit` when the incoming entry has the field, and `loadHistory` falls back to the previous session's value. A session switch or null-cleared patch can therefore keep rendering tool cards with a stale limit instead of the default.
  - confidence: 0.88
- **[P2] Consume invalid verbose limit arguments before validation:** `src/auto-reply/reply/directives.ts:45-60`
  - body: The parser only consumes digits after `limit`, so `/verbose limit abc` leaves `abc` in the cleaned message body instead of letting the limit handler return the intended invalid-limit error. The directive parser should consume the argument token or the handler should validate leftover limit text so messaging surfaces do not send the bad argument to the agent.
  - confidence: 0.78
