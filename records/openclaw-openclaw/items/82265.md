## Summary

Keep this open: current main still exact-matches only the bundled `memory-lancedb` slot, and the linked open PR is the active implementation path for the reported `memory-lancedb-pro` compatibility gap.

## What This Changes

The issue asks active-memory to use LanceDB-style `memory_recall` defaults when the configured memory slot is `memory-lancedb-pro`.

## Best Possible Solution

Review the linked PR as the immediate fix candidate, or replace it with a capability-driven default that selects the slot owner’s registered recall tools without hardcoding future LanceDB-family ids.

## Reproduction Assessment

Yes, source-level: with `plugins.slots.memory` set to `memory-lancedb-pro` and no explicit `config.toolsAllow`, current main falls through to `memory_search` and `memory_get`. I did not run a live Docker deployment because this review is read-only, but the allowlist path is direct.

## Solution Assessment

Unclear: adding `memory-lancedb-pro` to a small slot allowlist is a narrow compatibility fix, but current docs only promise automatic `memory_recall` for `memory-lancedb`; capability-driven selection would be more durable if maintainers want to support plugin families generically.
