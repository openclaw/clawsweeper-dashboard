## Summary

Keep open. Current main still hardcodes `session-memory` artifacts under `<workspace>/memory/` with a date-and-slug filename, and the related open implementation at https://github.com/openclaw/openclaw/pull/40618 only covers an `outputDir` subset and still has unresolved review blockers.

## What This Changes

The issue requests configurable `session-memory` output paths and filename templates so saved session artifacts can follow workspace conventions.

## Best Possible Solution

Define a backward-compatible `session-memory` artifact-template contract with validated workspace-relative path and filename placeholders, preserve the current default output, and decide separately whether daily files or cleanup belong in this change.

## Reproduction Assessment

Yes. Source inspection shows the handler always writes through `<workspace>/memory` with a fixed date-and-slug filename, and current tests/docs still assert that fixed output shape.

## Solution Assessment

Unclear until maintainers choose the config scope. Configurable hook-owned templates are the right direction, but the existing open PR only covers `outputDir` and the requested daily/cleanup behavior should not be folded in without approval.
