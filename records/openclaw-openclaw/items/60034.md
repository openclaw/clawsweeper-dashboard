## Summary

Keep open. Current main and the latest release still write session-memory artifacts to a fixed workspace memory directory with fixed date-plus-slug basenames, while the related outputDir PRs are closed unmerged and the remaining path/filename/daily/cleanup contract needs maintainer product scope.

## What This Changes

The issue requests configurable session-memory output paths and filenames so generated memory files can follow workspace conventions instead of the fixed flat memory directory layout.

## Best Possible Solution

Define a backward-compatible session-memory artifact-template contract with root-safe workspace-relative output handling, preserving the current default while explicitly deciding whether daily files and cleanup belong in scope.

## Reproduction Assessment

Yes, at source level: current main still hardcodes the session-memory output directory to workspace memory and derives a fixed date-plus-slug basename. I did not run a live /new or /reset because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the config contract; configurable templates are plausible, but the requested daily and cleanup behavior should not be bundled without explicit scope approval.
