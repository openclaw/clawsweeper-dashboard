## Summary

Keep open. Current main and v2026.6.10 still write `session-memory` artifacts to a fixed flat `memory/` layout, the prior output-directory PRs are closed unmerged, and the requested template/config behavior needs maintainer product direction rather than conservative cleanup.

## What This Changes

The issue requests configurable `session-memory` output directories and filename templates so generated memory artifacts can follow workspace conventions instead of always landing in the flat `memory/` tree.

## Best Possible Solution

Define a backward-compatible, workspace-safe session-memory artifact contract first, then implement only the accepted path/template behavior while keeping daily output and cleanup as explicit scoped decisions.
