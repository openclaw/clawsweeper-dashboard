## Summary

Keep open. Current main and the latest release still hardcode the bundled `session-memory` hook to write direct children of `<workspace>/memory/` with a `YYYY-MM-DD-slug.md` filename, and the requested configurable output path and filename templates are not implemented or documented.

## What This Changes

The issue asks for configurable output path and filename templates for the bundled `session-memory` hook so generated memory files can follow workspace conventions instead of a flat `memory/YYYY-MM-DD-slug.md` layout.

## Best Possible Solution

Add a backward-compatible `session-memory` session artifact template contract with validated workspace-relative output path and filename placeholders, while preserving the current default output and leaving daily-output/cleanup behavior for separate maintainer scope.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main always builds `<workspace>/memory`, always writes `${dateStr}-${slug}.md`, and the tests/docs still expect that fixed root-level filename shape.

## Solution Assessment

Yes, for a narrowed repair. Explicit hook configuration is the maintainable way to support workspace conventions; parsing freeform `AGENTS.md`, adding a daily writer, or cleaning directories should remain separate scope unless maintainers approve it.
