## Summary

Keep open. Current main and the latest release still write `session-memory` artifacts to a fixed workspace `memory/` directory with fixed date-plus-slug filenames, and the related output-directory PRs were closed unmerged; the remaining work needs maintainer product judgment on template scope and filesystem-safety rules.

## What This Changes

The issue requests configurable `session-memory` output path and filename templates so `/new` and `/reset` memory artifacts can follow workspace memory layout conventions.

## Best Possible Solution

Define a backward-compatible, root-safe session-memory artifact-template contract that preserves the current default flat output and scopes daily output and cleanup as separate maintainer-approved behavior.

## Reproduction Assessment

Yes, at source level. Current main still hardcodes the session-memory output directory and date-plus-slug basename, with docs and tests asserting the fixed layout; no live `/new` or `/reset` run was needed for this read-only review.

## Solution Assessment

Unclear until maintainers choose the config contract. Configurable templates are plausible, but the proposed daily output and empty-directory cleanup behavior should not be bundled without explicit scope and filesystem-safety decisions.
