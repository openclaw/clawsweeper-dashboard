## Summary

Keep open. Current main and the latest release still hardcode `session-memory` artifacts under `<workspace>/memory/` with a fixed date-and-slug filename, while the related open implementation only covers an `outputDir` subset and still has unresolved review/proof blockers.

## What This Changes

The issue requests configurable `session-memory` output paths and filename templates instead of fixed files under `memory/`.

## Best Possible Solution

Define a backward-compatible session-memory artifact-template contract with root-safe workspace-relative path handling, preserve the current default output, and decide separately whether daily files or cleanup belong in scope.

## Reproduction Assessment

Yes, at source level. Current main still writes through `<workspace>/memory` with a fixed date-and-slug basename, and current tests/docs assert that fixed output shape.

## Solution Assessment

Unclear until maintainers choose the config surface. Configurable hook-owned templates are plausible, but the existing open PR only implements `outputDir` and the requested daily/cleanup behavior should not be folded in without approval.
