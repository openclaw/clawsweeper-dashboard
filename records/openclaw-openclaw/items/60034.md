## Summary

Keep open. Current main and v2026.5.18 still hardcode `session-memory` writes under `<workspace>/memory/` with fixed date-and-slug filenames, and the open related PRs are partial or blocked rather than a merged solution.

## What This Changes

The issue requests configurable `session-memory` output paths and filename templates so generated artifacts can follow workspace conventions instead of the fixed flat `memory/` layout.

## Best Possible Solution

Define a backward-compatible session-memory artifact-template contract with root-safe workspace-relative path and filename handling, preserve the current default output, and decide daily-file and cleanup behavior as explicit scope choices.

## Reproduction Assessment

Yes, at source level. Current `main` still hardcodes `<workspace>/memory` and fixed dated basenames, and current tests/docs assert that output shape.

## Solution Assessment

Unclear until maintainers choose the config surface. Configurable hook-owned templates are plausible, but the existing `outputDir` PR is only a subset and the requested daily/cleanup behavior should not be folded in without approval.
