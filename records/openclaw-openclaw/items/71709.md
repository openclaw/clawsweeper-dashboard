## Summary

Keep open. Current `main` still routes the session-memory slug helper through the embedded runner without the existing helper-local auth-profile failure policy, while Anthropic extra-usage text still classifies as billing and can flow into shared auth-profile cooldown state.

## What This Changes

The issue reports that session-memory LLM slug generation can turn a transient Claude CLI HTTP 400 extra-usage error into a shared five-hour billing cooldown for the whole auth profile.

## Best Possible Solution

Make the slug generator a helper-local embedded run that falls back to a timestamp slug on failures while leaving shared auth-profile health untouched; keep broader lightweight-model or lane-congestion work in #33962.

## Reproduction Assessment

Yes. The issue includes production logs, and current source gives a deterministic path: an Anthropic extra-usage error in `generateSlugViaLLM` still classifies as billing and, without the local policy option, can reach shared auth-profile failure marking.

## Solution Assessment

Yes. The narrowest maintainable fix is to pass `authProfileFailurePolicy: "local"` from `src/hooks/llm-slug-generator.ts`; weakening the global billing matcher would risk regressing the intentional #61608 fallback behavior.
