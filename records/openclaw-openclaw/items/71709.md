## Summary

Keep open. Current main and v2026.5.7 still route enabled LLM slug generation through the embedded runner without helper-local auth-profile failure isolation, so a billing-classified slug helper failure can still persist shared profile-disabled state.

## What This Changes

The issue asks OpenClaw to prevent LLM slug-generator failures from disabling the shared claude-cli auth profile and taking down all agents on that profile.

## Best Possible Solution

Pass authProfileFailurePolicy: "local" from generateSlugViaLLM and add focused regression coverage, while preserving normal shared billing cooldown behavior for user-facing runs.

## Reproduction Assessment

Yes, at source level. With llmSlug enabled, current main calls runEmbeddedPiAgent on temp:slug-generator without local auth policy, the reported extra-usage text classifies as billing, and shared billing failures still write the long profile disable window.

## Solution Assessment

Yes. Reusing the existing authProfileFailurePolicy: "local" seam in the slug helper is the narrowest maintainable fix and avoids weakening provider billing failover globally.
