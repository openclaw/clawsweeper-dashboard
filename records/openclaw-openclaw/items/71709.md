## Summary

Keep open. Current main still leaves the session-memory slug generator on the shared embedded-run auth failure path, so the reported Claude CLI extra-usage error can still become a profile-wide billing disable instead of staying helper-local.

## What This Changes

Prevent session-memory LLM slug-generator failures from marking the shared Claude CLI auth profile as billing-disabled for five hours.

## Best Possible Solution

Make `generateSlugViaLLM` opt into helper-local auth failure policy and add regression coverage, while keeping global billing classification unchanged.

## Reproduction Assessment

Yes. The issue includes production logs, and current source provides a high-confidence path: default session-memory LLM slugging calls the embedded runner without local policy, the reported text maps to billing, and billing writes a five-hour `disabledUntil`.

## Solution Assessment

Yes. Passing `authProfileFailurePolicy: "local"` from the slug helper is the narrowest maintainable fix because the existing seam was built for hidden helper runs and avoids weakening provider billing failover globally.
