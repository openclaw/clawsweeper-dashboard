## Summary

Keep open. Current main still leaves the session-memory slug helper on the shared embedded-run auth-failure path, so the reported Claude CLI extra-usage rejection can still become a profile-wide billing cooldown instead of staying helper-local.

## What This Changes

The issue asks for slug-generator/internal-helper provider failures, especially HTTP 400 extra-usage errors, to avoid disabling the shared Claude CLI auth profile for five hours.

## Best Possible Solution

Make `generateSlugViaLLM` pass `authProfileFailurePolicy: "local"` and add regression coverage, while preserving global Anthropic extra-usage billing classification for user-facing failover.

## Reproduction Assessment

Yes. The report includes production logs, and current source still shows the full path: default slugging calls the embedded runner without local policy, the reported text maps to billing, and billing records a five-hour profile disable.

## Solution Assessment

Yes. Reusing the existing helper-local auth failure policy is the narrowest maintainable fix; globally weakening extra-usage billing classification could regress normal fallback behavior.
