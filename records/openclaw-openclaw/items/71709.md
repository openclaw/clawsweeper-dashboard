## Summary

Keep open. Current main has lowered the default exposure by making LLM filename slugs opt-in, but the opt-in slug-generator still runs through shared auth-profile failure handling, so a billing-classified Claude CLI helper error can still disable the whole profile instead of staying helper-local.

## What This Changes

The issue asks for session-memory slug-generation failures to be scoped to the internal helper lane instead of marking the shared Claude CLI auth profile as a billing failure.

## Best Possible Solution

Make `generateSlugViaLLM` pass `authProfileFailurePolicy: "local"` and cover it with a focused regression test, while preserving global billing classification for normal user-facing failover.

## Reproduction Assessment

Yes, at source level. With `llmSlug: true`, current main calls the embedded slug helper without local auth policy, the reported extra-usage text maps to billing, and billing writes a shared disabled profile window.

## Solution Assessment

Yes. The narrowest maintainable fix is to mark the hidden slug-generator run as helper-local, matching the existing active-memory pattern and avoiding a broad rollback of the billing classifier.
