## Summary

Keep open. Current main only mitigates default installs by making LLM slug generation opt-in; with `llmSlug: true`, the slug helper still uses the shared embedded-runner auth-failure policy, so the reported Anthropic extra-usage billing classification can still poison the shared profile cooldown state.

## What This Changes

The issue asks OpenClaw to prevent optional LLM slug-generation failures from disabling a shared auth profile and blocking unrelated agents.

## Best Possible Solution

Make the slug helper pass `authProfileFailurePolicy: "local"` and add focused regression coverage, while keeping billing classification and shared cooldown behavior intact for normal user-facing runs.

## Reproduction Assessment

Yes, at source level. With `llmSlug: true`, the slug helper can still run on `temp:slug-generator`, the reported extra-usage text classifies as `billing`, and the helper does not opt into the local policy that would suppress shared auth-state writes.

## Solution Assessment

Yes. Reusing the existing `authProfileFailurePolicy: "local"` seam in `generateSlugViaLLM` is the narrowest maintainable fix and avoids weakening provider billing failover globally.
