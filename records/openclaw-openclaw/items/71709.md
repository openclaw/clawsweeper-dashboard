## Summary

Keep open. Current main still routes the session-memory slug helper through shared embedded-run auth failure persistence: the helper does not pass the existing local failure policy, while the reported Anthropic extra-usage text still classifies as billing and billing failures still write a long profile-wide disabled state.

## What This Changes

The issue asks for internal session-memory slug-generation failures to stay lane-local instead of marking the shared `anthropic:claude-cli` auth profile as billing-disabled for five hours.

## Best Possible Solution

Keep global billing classification for real user-facing provider failures, but make `generateSlugViaLLM` opt into the existing helper-local auth-profile failure policy and add regression coverage proving billing-classified slug failures do not write shared cooldown state.

## Reproduction Assessment

Yes. Current source gives a high-confidence reproduction path: default session-memory LLM slugging calls the embedded runner without local auth failure policy, the reported extra-usage text classifies as billing, and billing persists a five-hour `disabledUntil`.

## Solution Assessment

Yes. Passing `authProfileFailurePolicy: "local"` from the slug helper is the narrowest maintainable fix; weakening the Anthropic extra-usage billing classifier globally would risk regressing intentional same-turn fallback behavior.
