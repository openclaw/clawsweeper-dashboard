## Summary

Keep open. Current main mitigates the default path by making LLM slugs opt-in, but with `llmSlug: true` the slug helper still uses the shared embedded-runner auth failure policy, and the reported Anthropic extra-usage text still resolves to `billing` and can persist a profile-wide cooldown.

## What This Changes

The issue asks for slug-generation helper failures to stay lane-local instead of marking the shared auth profile as billing-disabled for the cooldown window.

## Best Possible Solution

Pass `authProfileFailurePolicy: "local"` from `generateSlugViaLLM` and add focused regression coverage while preserving shared billing cooldown behavior for normal user-facing runs.

## Reproduction Assessment

Yes, at source level. With `llmSlug: true`, current main calls the slug embedded run without local auth policy; the reported extra-usage text classifies as `billing`, and shared billing reasons still reach `markAuthProfileFailure`.

## Solution Assessment

Yes. Reusing the existing `authProfileFailurePolicy: "local"` seam in the slug helper is the narrowest maintainable fix and avoids weakening global provider billing failover.
