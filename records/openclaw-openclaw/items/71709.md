## Summary

Keep open. Current main only mitigates default installs by making LLM slugs opt-in; when `llmSlug: true` is enabled, the slug helper still runs through the shared embedded-runner auth failure policy, and the reported Anthropic extra-usage text still classifies as `billing`, which can persist a profile-wide disabled window.

## What This Changes

The issue requests that session-memory LLM slug-generator failures stay helper-local instead of marking a shared auth profile billing-disabled for every agent.

## Best Possible Solution

Pass `authProfileFailurePolicy: "local"` from `generateSlugViaLLM` and cover it with focused regression tests, while preserving billing classification and shared cooldown behavior for normal user-facing runs.

## Reproduction Assessment

Yes, at source level. With `llmSlug: true`, current main calls `runEmbeddedPiAgent` on `temp:slug-generator` without local auth policy; the reported extra-usage text classifies as `billing`, and shared billing failures still write the long profile disable window.

## Solution Assessment

Yes. Reusing the existing `authProfileFailurePolicy: "local"` seam in the slug helper is the narrowest maintainable fix and avoids weakening provider billing failover globally.
