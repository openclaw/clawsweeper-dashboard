## Summary

Keep open. Current main still leaves the session-memory slug-generator embedded run on the shared auth-profile failure path, while the reporter's observed Anthropic extra-usage text is still billing-classified and can still reach the five-hour shared cooldown path. No open PR referencing this issue was found.

## What This Changes

The issue requests that low-priority session-memory slug generation failures stay helper-local instead of marking the shared `anthropic:claude-cli` auth profile as billing-disabled for five hours.

## Best Possible Solution

Keep the global billing classifier intact and mark the slug-generator embedded run as helper-local with `authProfileFailurePolicy: "local"`, backed by focused regression coverage for the run parameters and existing failure-policy helper behavior.

## Reproduction Assessment

Yes. The report provides production logs, and current source gives a deterministic source-level reproduction path: a billing-classified Anthropic extra-usage prompt failure from `generateSlugViaLLM` can reach shared auth-state marking because the helper omits the local policy.

## Solution Assessment

Yes. The narrowest maintainable fix is to scope only the slug-generator embedded run with the existing local failure-policy seam; changing the global billing matcher would risk undoing the intentional Anthropic extra-usage failover behavior from #61608.
