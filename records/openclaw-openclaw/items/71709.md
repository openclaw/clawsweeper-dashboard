## Summary

Keep open. Current main still routes session-memory slug generation through the shared embedded-run auth-failure path, so the reported Claude CLI extra-usage error can still become a profile-wide billing disable instead of staying helper-local.

## What This Changes

The issue asks for session-memory slug-generator failures to stay helper-local instead of disabling the shared Claude CLI auth profile for five hours.

## Best Possible Solution

Make `generateSlugViaLLM` pass `authProfileFailurePolicy: "local"` and add regression coverage showing slug-helper billing/auth failures do not update shared auth-profile failure state.

## Reproduction Assessment

Yes. The issue includes production logs, and current main provides a source-level reproduction path: default session-memory LLM slugging calls the embedded runner without local policy, the reported text classifies as billing, and billing writes a five-hour `disabledUntil`.

## Solution Assessment

Yes. Reusing the existing local helper policy is the narrowest maintainable fix because it leaves the intentional Anthropic extra-usage billing matcher in place for user-facing fallback while preventing hidden helper runs from poisoning shared auth health.
