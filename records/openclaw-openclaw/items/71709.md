## Summary

Keep open. Current main mitigates the default exposure by making LLM filename slugs opt-in, but the opt-in slug helper still calls the shared embedded-run auth failure path, so a billing-classified Claude CLI extra-usage error can still disable the shared auth profile instead of staying helper-local.

## What This Changes

The issue asks for session-memory LLM slug-generation failures to stay lane-local and not mark a shared Claude CLI auth profile as billing-disabled for five hours.

## Best Possible Solution

Set authProfileFailurePolicy: "local" for generateSlugViaLLM and add regression coverage proving billing-classified slug-helper failures do not update shared auth-profile state, while preserving normal user-turn billing failover.

## Reproduction Assessment

Yes, at source level: enable hooks.internal.entries.session-memory.llmSlug, have the slug helper hit a Claude CLI extra-usage error, and the current code classifies it as billing before the shared embedded-run auth state writer can create a five-hour disabledUntil. I did not run a live Claude CLI replay in this read-only review.

## Solution Assessment

Yes. Reusing the existing authProfileFailurePolicy: "local" seam from hidden helper runs is the narrowest maintainable fix because it avoids weakening global billing classification for normal user-facing turns.
