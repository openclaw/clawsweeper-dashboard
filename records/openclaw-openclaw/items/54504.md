## Summary

Keep open: current main still lacks a label-backed subagent role-file loader, and the request remains a real native subagent profile gap rather than an already-implemented cleanup item. The feature should wait for maintainer product/security direction because the proposed frontmatter can affect prompt authority, tool policy, model routing, and auditability.

## What This Changes

The issue requests persistent lightweight per-role subagent instruction files that sessions_spawn can auto-load by label, with optional frontmatter for model and tool policy.

## Best Possible Solution

Keep this issue open as the product/security design item for native subagent role profiles, then implement one audited loader with explicit spawn arguments winning over role-file defaults, safe/narrowing frontmatter semantics, and visible provenance for loaded role or memory files.
