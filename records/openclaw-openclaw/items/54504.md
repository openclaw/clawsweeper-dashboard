## Summary

Keep this issue open: current main still has no label-based workspace/user `subagents/<label>.md` resolver for native `sessions_spawn`, and the requested frontmatter model/tool behavior needs maintainer API and security-policy design before implementation.

## What This Changes

Request persistent workspace/user Markdown role profiles for native `sessions_spawn` keyed by label, including frontmatter model and tool overrides.

## Best Possible Solution

If maintainers accept the feature, add a native role-profile resolver for `sessions_spawn` with workspace-before-user precedence, explicit-parameter precedence, safe frontmatter validation, conservative tool policy, docs, and focused regression tests.
