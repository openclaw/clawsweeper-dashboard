## Summary

Keep open: current main still lacks automatic workspace/user `subagents/<label>.md` role profiles for native `sessions_spawn`, including Markdown body injection and frontmatter-driven model/tool semantics. Adjacent issues cover agent-level prompts, bootstrap filtering, and subagent operating principles, but none supersede this narrower per-label role-file contract.

## What This Changes

Request to add persistent per-role Markdown instruction files for native `sessions_spawn` so `label` can resolve role instructions plus optional model/tool frontmatter.

## Best Possible Solution

Design an additive native role-profile resolver if maintainers accept the feature, with workspace-before-user precedence, explicit spawn parameters winning over frontmatter, safe model validation, tool-policy narrowing by default, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence current-main check that native `sessions_spawn` does not auto-load per-label role files or apply frontmatter model/tool metadata.
