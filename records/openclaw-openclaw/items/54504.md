## Summary

Keep open: current main still does not auto-load workspace/user `subagents/<label>.md` role files for native `sessions_spawn`, and the remaining request is a new role-profile/API and capability-policy feature rather than an implemented or duplicate cleanup item.

## What This Changes

The issue requests persistent Markdown role files for `sessions_spawn` subagents so label-specific instructions and optional model/tool metadata can be reused without full agent entries.

## Best Possible Solution

Design an additive native role-profile resolver for `sessions_spawn` if maintainers accept the feature, with workspace-before-user precedence, explicit parameter precedence over frontmatter, safe model validation, tool-policy narrowing by default, docs, and focused tests.
