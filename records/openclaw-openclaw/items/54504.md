## Summary

Keep open: current main and the latest release have subagent labels, model/thinking overrides, target-agent routing, context modes, attachments, and plugin prompt hooks, but not a persistent label-backed role-file loader with frontmatter, memory, precedence, or audit semantics. The added live-workflow comment reinforces that this is still a product/security design surface rather than cleanup-ready stale work.

## What This Changes

The issue requests persistent workspace/user-scoped role instruction files for `sessions_spawn` so a label can load prompt, model, tool, and memory context without a full `agents.list` entry.

## Best Possible Solution

Design a native subagent-role contract, or a plugin hook extension that can support it, with explicit precedence, privilege narrowing, file-size limits, loaded-file auditability, and a clear memory/state owner.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction of the missing feature: current spawn params, prompt builder, config types, and docs have no label-backed role-file loader or frontmatter parser. No live runtime repro was needed because this is a feature gap, not a reported failing existing contract.

## Solution Assessment

Unclear: the file convention is a plausible user-facing shape, but the best fix needs maintainer decisions on core versus plugin ownership, precedence, privilege narrowing, audit output, and memory write-back before implementation.
