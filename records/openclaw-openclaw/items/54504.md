## Summary

Keep open. Current main still lacks the requested lightweight `sessions_spawn` role-file convention; existing support covers explicit spawn parameters, full agent config/defaults, inline attachments, skills, and broad subagent policy, but not automatic per-label Markdown role loading with frontmatter-derived model/tool controls.

## What This Changes

Request support for persistent per-role Markdown instruction files with optional YAML frontmatter that native `sessions_spawn` loads by label from workspace-scoped and user-scoped subagent directories.

## Best Possible Solution

Keep this open for maintainer product/API design. If accepted, add a native resolver in the `sessions_spawn` and `spawnSubagentDirect` path for workspace-then-user `subagents/<label>.md` files, with explicit spawn parameters taking precedence over frontmatter, constrained frontmatter parsing, safe tool-policy semantics, docs, and focused tests for precedence, invalid metadata, model overrides, tool-policy narrowing, run-mode/session-mode behavior, and unchanged behavior when no role file exists.
