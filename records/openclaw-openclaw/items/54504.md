## Summary

Keep open: current main and v2026.6.8 still lack a native label-backed subagent role-file loader, and the open related PR only covers a narrower cross-agent role-context symptom.

## What This Changes

Request support for persistent per-role subagent Markdown files loaded by `sessions_spawn(label=...)`, with optional model/tool frontmatter and memory/provenance behavior.

## Best Possible Solution

Define one maintainer-approved native subagent role-profile contract with lookup precedence, frontmatter schema, explicit spawn-argument precedence, narrowing-only tool behavior, loaded-file audit metadata, and memory semantics.

## Reproduction Assessment

Yes, at source level: current main accepts `label` but has no code path that resolves `<workspace>/subagents/<label>.md` or a user-scoped equivalent before the child run starts.

## Solution Assessment

Unclear until maintainers choose the product and security boundary; the file convention is plausible, but prompt authority, tool narrowing, model override, memory, and audit semantics must be settled first.
