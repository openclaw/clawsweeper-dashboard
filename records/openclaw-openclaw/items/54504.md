## Summary

Keep open: current main and v2026.6.10 still treat `sessions_spawn.label` as display/prompt metadata only, with no label-backed role file lookup or frontmatter policy; the remaining work is a new trusted subagent profile surface that needs maintainer product and security direction.

## What This Changes

Request persistent label-named subagent role files that `sessions_spawn` can auto-load for role prompt text, frontmatter model/tool defaults, precedence, memory, and audit metadata.

## Best Possible Solution

Define a maintainer-approved subagent role-profile contract covering lookup precedence, explicit spawn-argument overrides, narrowing-only tool policy, model/provider limits, memory ownership, and loaded-file audit metadata before implementation.

## Reproduction Assessment

Yes, at source level. Current main accepts `label` but has no path that resolves `subagents/<label>.md`, parses frontmatter, applies role defaults, or injects a role-file body before the child run starts.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. The proposed file convention is plausible, but prompt authority, tool narrowing, model overrides, memory, and audit semantics must be settled first.
