## Summary

Keep open: current main and v2026.6.8 still have labels, model/thinking overrides, target-agent routing, context modes, attachments, and plugin prompt hooks, but no persistent label-backed subagent role-file loader with frontmatter, memory, precedence, or audit semantics. The live workflow comment adds concrete product evidence that this remains a useful multi-agent ergonomics and security-design request.

## What This Changes

Request persistent per-role markdown instruction files for `sessions_spawn` subagents, with optional frontmatter for model/tool defaults and workspace/user precedence.

## Best Possible Solution

Define a native subagent-role contract, or a plugin hook extension that can support it, with explicit precedence, privilege narrowing, file-size limits, loaded-file auditability, and memory/state ownership.

## Reproduction Assessment

Yes at source level, but not as a failing existing-behavior bug: current spawn params, prompt construction, config types, and docs have no label-backed role-file loader or frontmatter parser. No live runtime repro was needed because the issue is a missing feature surface.

## Solution Assessment

Unclear until maintainers decide the product and security boundary. The requested file convention is plausible, but the maintainable solution needs explicit core-versus-plugin ownership, precedence, privilege narrowing, audit output, and memory write-back semantics.
