## Summary

Keep open: current main and the latest release still lack a label-backed native subagent role-file loader, and the requested prompt/model/tool frontmatter behavior is a product and security design surface rather than an already-implemented cleanup item.

## What This Changes

The issue requests persistent lightweight markdown role files for `sessions_spawn` labels, including body injection plus frontmatter-driven model and tool defaults for ephemeral native subagents.

## Best Possible Solution

Keep this as the product/security design item for native subagent role profiles, then implement one audited loader only after maintainers choose the core API and precedence model.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.6.8` shows `sessions_spawn` accepts `label`, but no code path resolves `<workspace>/subagents/<label>.md` or a user-scoped equivalent before the child run starts.

## Solution Assessment

Unclear. The requested convention is plausible, but the best fix needs maintainer decisions on trust boundaries, explicit-argument precedence, narrowing-only tool semantics, model override authority, memory behavior, and audit output.
