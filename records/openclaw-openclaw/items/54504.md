## Summary

Keep open: current main and v2026.6.10 still lack label-backed subagent role-file lookup, frontmatter parsing, or role-file prompt injection; this is a real feature gap but needs maintainer product and security direction before implementation.

## What This Changes

Request support for persistent workspace/user-scoped Markdown role files that `sessions_spawn(label=...)` can load as subagent instructions with optional model/tool frontmatter.

## Best Possible Solution

Define a maintainer-approved subagent role-profile contract covering lookup precedence, explicit-argument overrides, narrowing-only tool policy, model/provider limits, memory ownership, and loaded-file audit metadata before implementation.

## Reproduction Assessment

Yes, at source level. Current main accepts `label` but has no path that resolves `subagents/<label>.md`, parses frontmatter, applies role defaults, or injects a role-file body before the child run starts.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. The proposed file convention is plausible, but prompt authority, tool narrowing, model overrides, memory, and audit semantics must be settled first.
