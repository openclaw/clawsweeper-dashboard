## Summary

Keep open: current `main` still treats `sessions_spawn.label` as display/runtime metadata and does not auto-load workspace or user `subagents/<label>.md` role profiles. The request is a real feature gap, but it needs maintainer product and security review before implementation because it defines trusted instruction-file precedence plus model/tool frontmatter semantics.

## What This Changes

The issue requests lightweight persistent role instruction files for ephemeral `sessions_spawn` subagents, including optional frontmatter for model/tool behavior and workspace/user precedence.

## Best Possible Solution

Approve a native `sessions_spawn` role-profile design first, then implement a single audited loader that treats explicit spawn arguments as authoritative, constrains file frontmatter to safe/narrowing semantics, and documents how it differs from full `agents.list` entries and ClawHub skills.

## Reproduction Assessment

Yes, source-level: `sessions_spawn(label="researcher")` currently has no code path that resolves `subagents/researcher.md` or `~/.openclaw/subagents/researcher.md`. This is a feature gap rather than a failing existing contract.

## Solution Assessment

Unclear. The proposed convention is plausible, but the best solution depends on maintainer choices for file precedence, auditability, model/tool frontmatter, memory write-back, and how this should coexist with full `agents.list` profiles.
