## What This Changes

The issue requests persistent lightweight per-role subagent instruction files auto-loaded by `sessions_spawn(label=...)`, with markdown prompt content and frontmatter-driven model/tool metadata.

## Best Possible Solution

Define one maintainer-approved native subagent role-profile contract, including lookup precedence, frontmatter schema, explicit spawn-argument precedence, narrowing-only tool behavior, loaded-file audit metadata, and whether any pieces belong in core versus plugin API extensions.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers choose the product and security boundary; the file convention is plausible, but prompt authority, tool narrowing, model override, memory, and audit semantics must be settled first.
