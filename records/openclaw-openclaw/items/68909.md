## Summary

Keep open: current main still has the documented Linux system-service status gap, but this PR is not merge-ready because the latest diff can misclassify missing gateway system units, choose node units as the gateway runtime, and hide unrelated stale listeners; after-fix real behavior proof is also still missing.

## What This Changes

The PR adds system-bus probing for gateway status, cgroup-aware stale supervisor filtering, system-unit status output, hint wording changes, and focused daemon/CLI tests.

## Best Possible Solution

Land a focused daemon/status fix that probes real gateway system units, leaves node units as auxiliary context, preserves user-bus diagnostics unless a gateway system unit is observed, and limits cgroup exemptions to recognized OpenClaw sibling services.

## Reproduction Assessment

Yes, at source level. Current docs support /etc/systemd/system gateway units, while current main still reads runtime through systemctl --user and stale-PID filtering has no cgroup-aware sibling-service distinction.

## Solution Assessment

No, not as currently implemented. The system-bus and cgroup-aware direction is appropriate, but the patch needs narrower gateway-unit selection, preserved user-bus diagnostics, safer cgroup exemptions, and real after-fix proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve user-bus errors until a gateway system unit is observed:** `src/daemon/systemd.ts:715-722`
  - body: When user systemd is unavailable, a reachable system bus with only missing units makes this branch return `stopped`/`scope: "system"` and drop the original no-medium detail. User-scoped installs then lose the linger/XDG_RUNTIME_DIR guidance; only switch to system scope after a real gateway unit is observed.
  - confidence: 0.88
- **[P1] Keep node units out of primary gateway selection:** `src/daemon/systemd-system-probe.ts:160-164`
  - body: Because `buildGatewayCandidates` includes node units, this loop can choose `openclaw-node-host.service` as primary when no gateway unit is active, so status reports the gateway as running with the node PID. Filter primary selection to gateway unit names and keep node units auxiliary.
  - confidence: 0.9
- **[P2] Limit cgroup exemptions to known sibling services:** `src/cli/daemon-cli/restart-health.ts:134-135`
  - body: Any two different `*.service` cgroups are treated as safe, so an OpenClaw listener under an unrelated service such as `user@1000.service` is hidden from `staleGatewayPids`. Restrict the distinct shortcut to known OpenClaw gateway/node sibling service pairs.
  - confidence: 0.86
