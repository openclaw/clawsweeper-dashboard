## Summary

Keep open. The reported headless Linux/systemd status failure is credible against current main, but this PR is not merge-ready because the proposed fallback can select a node-host unit as the gateway runtime and the cgroup filter can hide unrelated stale gateway listeners.

## What This Changes

The PR adds a system-bus systemd runtime probe, cgroup-aware restart-health PID filtering, status/hint output changes, and regression tests for headless Linux system-level gateway/node services.

## Best Possible Solution

Keep the system-bus fallback direction, but separate gateway-primary selection from auxiliary node-unit reporting, narrow cgroup-based stale-PID suppression to known safe OpenClaw sibling services, preserve current restart-health version checks, and add the required changelog and focused regression tests.

## Reproduction Assessment

Yes. The current-main issue follows from readSystemdServiceRuntime returning unknown on user-bus failure and restart-health then classifying non-owned gateway listener PIDs without cgroup context; the PR regressions can be reproduced with mocked node-host-only systemUnits and with gateway/runtime cgroups that differ because the listener is under an unrelated service such as user@1000.service.

## Solution Assessment

No. The system-bus fallback is the right narrow approach, but this implementation over-selects node units as gateway runtime and over-suppresses stale listeners; the safer fix is to probe node services only as context and only suppress explicitly recognized sibling OpenClaw service pairs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restrict primary selection to gateway units:** `src/daemon/systemd-system-probe.ts:160-164`
  - body: `pickPrimaryGatewayUnit` iterates the same candidate list used for probing, but that list now includes `openclaw-node-host.service` and `openclaw-node.service`. If the user bus is unavailable and only the node-host unit is loaded/running, gateway status will be reported as running with the node PID/cgroup, masking a missing or stopped gateway unit. Keep node units in `systemUnits` as auxiliary context, but exclude them from primary gateway selection.
  - confidence: 0.91
- **[P2] Limit suppression to known sibling services:** `src/cli/daemon-cli/restart-health.ts:132-135`
  - body: `classifyCgroupRelationship` treats any two different `*.service` cgroups as distinct, so a stale gateway listener under an unrelated service such as `user@1000.service` is removed from `staleGatewayPids`. That can make restart health time out instead of reporting the duplicate. Restrict the shortcut to known safe OpenClaw gateway/node sibling unit pairs or otherwise prove both service names are safe siblings.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `src/daemon/systemd.ts:697-713`
  - body: This is a user-facing `fix(daemon)` change to gateway status/restart diagnostics, but the PR does not update `CHANGELOG.md`. OpenClaw policy requires changelog coverage for user-facing fix/feat/perf changes before merge.
  - confidence: 0.82
