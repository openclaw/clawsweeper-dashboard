## Summary

Keep open. The PR is aimed at a real Linux systemd gateway-status false positive and now includes useful terminal evidence, but source review still finds blocking gateway-status correctness issues and the exact headless system-level scenario remains unproven after the patch.

## What This Changes

The PR adds a Linux system-bus fallback and cgroup-aware stale-process filtering for gateway status on hosts running OpenClaw as system-level systemd services.

## Best Possible Solution

Keep the fallback direction, but require gateway-only primary runtime selection, cgroup dedup limited to known safe OpenClaw sibling service pairs, and redacted after-fix proof from the targeted system-level systemd deployment.

## Reproduction Assessment

Yes for the PR-introduced blockers: source inspection shows node units are included in the same candidate order used by pickPrimaryGatewayUnit, and any differing service cgroup is treated as distinct. The original host bug has contributor evidence, but I did not live-reproduce the exact target deployment here.

## Solution Assessment

No. The system-bus fallback is the right direction, but this implementation should not let node units become the gateway runtime and should not treat every different service cgroup as a safe sibling.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Restrict primary selection to gateway units:** `src/daemon/systemd-system-probe.ts:158-162`
  - body: `pickPrimaryGatewayUnit` iterates the same candidate list that includes `openclaw-node-host.service` and `openclaw-node.service`. If the user bus is unavailable and only node-host is active, `readSystemdServiceRuntime` will return a running gateway runtime with the node PID, masking a missing or stopped gateway. Keep node units in `systemUnits`, but do not allow them to become the top-level gateway runtime.
  - confidence: 0.91
- **[P2] Do not suppress unrelated service listeners:** `src/cli/daemon-cli/restart-health.ts:348-349`
  - body: `classifyCgroupRelationship` treats any two different `*.service` names as safely distinct. A conflicting OpenClaw gateway listener under another service would be dropped from `staleGatewayPids`, so restart/status may stop warning about the real port conflict. Limit the distinct shortcut to explicitly safe OpenClaw gateway/node sibling services.
  - confidence: 0.86
