## Summary

Keep open: the reported Linux system-level systemd status bug still exists on current main, but this PR needs contributor rework and real after-fix proof before merge because the branch can misclassify node units as the gateway, drop useful user-bus diagnostics, and over-exempt unrelated service cgroups.

## What This Changes

The PR adds a Linux systemd system-bus runtime probe, system-unit status output, `/proc/<pid>/cgroup` parsing, cgroup-aware stale supervisor filtering, and related daemon/CLI tests.

## Best Possible Solution

Land a narrowed systemd-status fix that probes real gateway system units first, keeps node units auxiliary, preserves user-bus diagnostics unless a gateway system unit is observed, limits cgroup exemptions to known OpenClaw sibling services, and includes redacted real systemd proof.

## Reproduction Assessment

Yes, at source level: current main documents Linux system-level gateway units while `readSystemdServiceRuntime` still depends on `systemctl --user`, and restart-health has no cgroup attribution. I did not run a live headless systemd host in this read-only review.

## Solution Assessment

No, not as currently implemented. The system-bus and cgroup-aware direction is appropriate, but the patch needs narrower gateway-unit selection, safer diagnostics, constrained cgroup exemptions, current-main reconciliation, and real after-fix proof.

Full review comments:

- **[P1] Preserve user-bus errors until a gateway system unit is observed:** `src/daemon/systemd.ts:715-722`
  - body: When the user bus fails, this branch returns a stopped system-scoped missing unit whenever the system bus is reachable, even if no OpenClaw gateway unit was observed. That drops the original no-medium or linger diagnostic for user-systemd installs, so only switch to system scope after finding a real gateway unit.
  - confidence: 0.88
- **[P1] Keep node units out of gateway primary selection:** `src/daemon/systemd-system-probe.ts:160-164`
  - body: `pickPrimaryGatewayUnit` selects from the same candidate list that includes `openclaw-node-host.service` and `openclaw-node.service`. If only node-host is active, gateway status can report the gateway running with the node PID/cgroup, masking a missing or stopped gateway unit.
  - confidence: 0.9
- **[P2] Limit cgroup exemptions to known OpenClaw sibling units:** `src/cli/daemon-cli/restart-health.ts:132-135`
  - body: Any two different `*.service` cgroups are treated as distinct, so an unrelated listener under another service can be removed from `staleGatewayPids`. Restrict the exemption to recognized gateway/node sibling service pairs.
  - confidence: 0.86
