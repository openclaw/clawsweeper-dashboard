## Summary

Keep open: current main still lacks the system-bus/cgroup handling, and the PR addresses a real gateway-status failure mode, but the proposed patch has correctness blockers and lacks after-fix real behavior proof from a real system-level systemd host.

## What This Changes

The PR adds system-bus probing, system-unit status output, `/proc/<pid>/cgroup` parsing, cgroup-aware stale PID filtering, hint wording updates, and focused tests for gateway status on headless Linux systemd hosts.

## Best Possible Solution

Land a narrowed daemon/status fix that probes real gateway system units, preserves user-bus diagnostics unless a gateway system unit is observed, and limits cgroup exemptions to recognized OpenClaw gateway/node sibling service pairs.

## Reproduction Assessment

Yes. A high-confidence source/mocked reproduction exists for both the original current-main failure path and the PR regressions: user-bus no-medium with no system unit, node-host-only system units, and unrelated service cgroups in stale-listener filtering.

## Solution Assessment

No. The system-bus and cgroup-aware direction is appropriate, but this patch broadens fallback and dedup decisions too far; the safer fix should separate gateway-primary units from auxiliary node units and preserve historical user-bus guidance unless a real gateway system unit is found.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve user-bus errors when no system unit exists:** `src/daemon/systemd.ts:715-722`
  - body: When the user bus fails but the system bus is reachable, missing units still make `systemctl show` return successfully, leaving `outcome.units` empty. This branch then reports a stopped system-scoped missing unit and drops the original no-medium detail, so user-systemd installs lose the linger/XDG_RUNTIME_DIR guidance they need. Only switch to system scope after observing a real gateway system unit.
  - confidence: 0.88
- **[P1] Keep node units out of primary gateway selection:** `src/daemon/systemd-system-probe.ts:160-164`
  - body: `pickPrimaryGatewayUnit` searches the same candidate list used for discovery, and that list includes `openclaw-node-host.service` and `openclaw-node.service`. If only the node-host unit is active, gateway status will report the gateway as running with the node-host PID/cgroup, masking a missing or stopped gateway unit. Keep node units as auxiliary `systemUnits` only.
  - confidence: 0.9
- **[P2] Limit cgroup exemptions to known sibling services:** `src/cli/daemon-cli/restart-health.ts:134-135`
  - body: This treats any two different `*.service` cgroups as safely distinct. A stale OpenClaw listener running under an unrelated service such as `user@1000.service` would be removed from `staleGatewayPids`, preventing status/restart cleanup from reporting the conflict. Restrict the exemption to recognized OpenClaw gateway/node sibling service pairs.
  - confidence: 0.86
