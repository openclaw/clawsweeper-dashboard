## Summary

Keep open. The PR addresses a real current-main Linux gateway-status gap, but the proposed patch is not merge-ready because its system-bus fallback can hide user-bus diagnostics, its primary runtime selector can report node-host as the gateway, and its cgroup exemption is broader than the stale-listener safety model supports.

## What This Changes

The PR adds a Linux systemd system-bus runtime probe, new runtime scope/unit/cgroup fields, status output for system units, a `/proc/<pid>/cgroup` parser, cgroup-aware stale PID filtering, hint wording updates, and focused tests.

## Best Possible Solution

Land a narrow daemon/status fix that probes real system-level gateway units without reclassifying user-systemd failures, keeps node units as auxiliary context rather than the primary gateway runtime, and only suppresses stale PID warnings for explicitly recognized OpenClaw gateway/node sibling service pairs. Rebase the work onto current main and include a user-facing changelog entry.

## Reproduction Assessment

Yes. The PR findings are reproducible with mocked `systemctl` and port/cgroup inputs: user-bus no-medium plus all system units `not-found`, only `openclaw-node-host.service` active, and runtime/listener cgroups of `openclaw-host-gateway.service` versus `user@1000.service`. The original reported issue also has concrete host evidence in the PR body, and current main lacks the proposed system-bus/cgroup implementation.

## Solution Assessment

No. The system-bus and cgroup-aware direction is appropriate, but this diff is too broad in the fallback and dedup decisions; the safer solution is to split gateway-primary candidates from auxiliary node units and preserve historical user-bus diagnostics unless a real gateway system unit is observed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve user-bus errors when no system unit exists:** `src/daemon/systemd.ts:715-722`
  - body: When the user bus fails but the system bus is reachable, `systemctl show` returns success for missing units, leaving `outcome.units` empty but `systemBusAvailable` true. This branch then reports a stopped system-scoped missing unit and drops the original `systemctl --user` no-medium detail, so user-systemd installs lose the linger/XDG_RUNTIME_DIR guidance they need. Only switch to system scope after observing a real gateway system unit, or keep the original unknown runtime detail.
  - confidence: 0.88
- **[P1] Keep node units out of primary gateway selection:** `src/daemon/systemd-system-probe.ts:160-164`
  - body: `pickPrimaryGatewayUnit` searches the same candidate list used for discovery, and that list includes `openclaw-node-host.service` and `openclaw-node.service`. If only the node-host unit is active, `openclaw gateway status` will report the gateway as running with the node-host PID/cgroup, masking a missing or stopped gateway. Use a gateway-only primary list and keep node units only in the auxiliary `systemUnits` output.
  - confidence: 0.9
- **[P2] Limit cgroup exemptions to known sibling services:** `src/cli/daemon-cli/restart-health.ts:134-135`
  - body: This treats any two different `*.service` cgroups as safely distinct. A real stale OpenClaw listener running under a generic service such as `user@1000.service` would then be removed from `staleGatewayPids`, preventing status/restart cleanup from reporting the conflict. Restrict the exemption to known OpenClaw gateway/node sibling unit pairs, or require both cgroups to match recognized OpenClaw service names.
  - confidence: 0.86
