## Summary

Keep open. Current main still lacks the system-bus runtime probe and cgroup-aware stale-PID filtering this PR is trying to add, but the PR cannot merge as-is because the daemon/status changes broaden system-scope selection too far and the external proof gate is still missing.

## What This Changes

The PR adds daemon status support for system-scoped systemd units, cgroup-based duplicate supervisor filtering, system-unit status output, hint wording changes, and focused tests.

## Best Possible Solution

Land a narrowed daemon/status fix that probes real gateway system units, leaves node units as auxiliary context, preserves user-bus diagnostics unless a gateway system unit is observed, and limits cgroup exemptions to recognized OpenClaw sibling services.

## Reproduction Assessment

Yes, at source level. Current main is documented to support system-service deployments but readSystemdServiceRuntime only uses the user bus, while stale-PID filtering has no cgroup-aware sibling-service distinction.

## Solution Assessment

No, not as currently implemented. The system-bus and cgroup-aware direction is appropriate, but this patch should be narrowed so system scope is selected only from gateway units and stale-listener exemptions apply only to known safe OpenClaw sibling services.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve user-bus errors when no system unit exists:** `src/daemon/systemd.ts:715-722`
  - body: When the user bus fails but the system bus is reachable, missing units still make systemctl show return successfully, leaving outcome.units empty. This branch then reports a stopped system-scoped missing unit and drops the original no-medium detail, so user-systemd installs lose the linger/XDG_RUNTIME_DIR guidance they need. Only switch to system scope after observing a real gateway system unit.
  - confidence: 0.88
- **[P1] Keep node units out of primary gateway selection:** `src/daemon/systemd-system-probe.ts:160-164`
  - body: pickPrimaryGatewayUnit searches the same candidate list used for discovery, and that list includes openclaw-node-host.service and openclaw-node.service. If only the node-host unit is active, gateway status will report the gateway as running with the node-host PID/cgroup, masking a missing or stopped gateway unit. Keep node units as auxiliary systemUnits only.
  - confidence: 0.9
- **[P2] Limit cgroup exemptions to known sibling services:** `src/cli/daemon-cli/restart-health.ts:134-135`
  - body: This treats any two different *.service cgroups as safely distinct. A stale OpenClaw listener running under an unrelated service such as user@1000.service would be removed from staleGatewayPids, preventing status/restart cleanup from reporting the conflict. Restrict the exemption to recognized OpenClaw gateway/node sibling service pairs.
  - confidence: 0.86
