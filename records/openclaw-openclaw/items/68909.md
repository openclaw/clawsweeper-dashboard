## Summary

Keep this PR open: it targets a valid daemon/status bug that is not implemented on current main, but the current patch still has blocking correctness findings and lacks required after-fix real behavior proof from a system-scoped Linux/WSL systemd setup.

## What This Changes

The PR adds a Linux systemd system-bus runtime fallback, system-unit status output, `/proc` cgroup parsing, and cgroup-aware stale supervisor filtering for gateway status.

## Best Possible Solution

Land a narrowed daemon/status fix that probes real gateway system units, keeps node units as auxiliary context, preserves user-bus diagnostics unless a gateway system unit is observed, limits cgroup exemptions to recognized OpenClaw sibling services, and includes redacted real systemd proof.

## Reproduction Assessment

Yes, at source level: current docs support system-level Linux units, while current main still probes runtime through user systemd and stale-PID filtering has no cgroup attribution. I did not run a live headless systemd host in this read-only review.

## Solution Assessment

No, not as currently implemented. The system-bus and cgroup-aware direction is appropriate, but the patch should be narrowed to gateway primary units, known sibling cgroups, preserved user-bus diagnostics, and real after-fix proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve user-bus errors until a gateway system unit is observed:** `src/daemon/systemd.ts:715-722`
  - body: When the user bus fails, `systemctl show` against missing system units can still return code 0 with no reportable units. This branch then reports a stopped system-scoped missing unit and drops the original no-medium/linger guidance, so user-systemd installs get the wrong diagnostic. Only switch to system scope after observing a real gateway unit.
  - confidence: 0.88
- **[P1] Keep node units out of gateway primary selection:** `src/daemon/systemd-system-probe.ts:160-164`
  - body: The candidate order used by `pickPrimaryGatewayUnit` includes `openclaw-node-host.service` and `openclaw-node.service`. If only node-host is active, gateway status reports the gateway as running with the node PID/cgroup, masking a stopped or missing gateway. Keep node units as auxiliary context only.
  - confidence: 0.9
- **[P2] Limit cgroup exemptions to known OpenClaw sibling units:** `src/cli/daemon-cli/restart-health.ts:134-135`
  - body: Any two different `*.service` cgroups are treated as distinct, so an unrelated listener under `user@1000.service` or another service would be removed from `staleGatewayPids`. Restrict the exemption to recognized gateway/node sibling service pairs.
  - confidence: 0.86
