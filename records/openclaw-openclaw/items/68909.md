## Summary

Keep open: the reported system-scoped Linux gateway status problem is valid, but this PR still needs real after-fix host proof and has blocking runtime-selection and stale-listener classification defects before merge.

## What This Changes

The branch adds system-bus systemd probing, system-unit status output, cgroup parsing, cgroup-aware restart-health filtering, hint wording updates, and mocked daemon/CLI tests for Linux gateway status.

## Best Possible Solution

Land one coordinated systemd-status fix that probes real gateway system units first, keeps node units auxiliary, preserves user-bus diagnostics unless a gateway system unit is observed, limits cgroup exemptions to known OpenClaw sibling services, and includes redacted real systemd proof.

## Reproduction Assessment

Yes, at source level: current main documents system-level Linux gateway units, but readSystemdServiceRuntime still depends on systemctl --user and restart-health has no cgroup attribution. I did not run a live headless systemd host in this read-only review.

## Solution Assessment

No, not as currently implemented. The system-bus and cgroup-aware direction is appropriate, but the patch needs narrower gateway-unit selection, safer fallback semantics, constrained cgroup exemptions, current-main reconciliation, and real after-fix proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve user-bus errors until a gateway unit is observed:** `src/daemon/systemd.ts:715-722`
  - body: When the user bus fails, this branch can return a system-scoped stopped state merely because the system bus is reachable. That drops the original no-medium or linger diagnostic for user-systemd installs, so only switch to system scope after an actual gateway unit is found.
  - confidence: 0.9
- **[P1] Keep node units out of gateway primary selection:** `src/daemon/systemd-system-probe.ts:160-164`
  - body: pickPrimaryGatewayUnit iterates candidates that include openclaw-node.service and openclaw-node-host.service, so a host with only the node unit active can make gateway status report the node PID/cgroup as the gateway runtime. Node units should stay auxiliary context, not primary gateway candidates.
  - confidence: 0.92
- **[P2] Limit cgroup exemptions to known OpenClaw sibling units:** `src/cli/daemon-cli/restart-health.ts:132-135`
  - body: This returns distinct for any two different service cgroup names, so stale-listener detection can be suppressed for conflicting OpenClaw listeners under unrelated services such as user@1000.service. Restrict the shortcut to explicitly safe gateway/node sibling-service combinations.
  - confidence: 0.86
