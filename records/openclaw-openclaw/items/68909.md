## Summary

Keep open: the reported systemd system-unit status bug is still present on current main, but this branch has blocking correctness issues and lacks after-fix real-environment proof.

## What This Changes

The PR adds Linux systemd system-bus status probing, system-unit status output, `/proc` cgroup parsing, and cgroup-aware stale supervisor filtering for gateway status.

## Best Possible Solution

Land a narrowed systemd status fix that probes real gateway system units as primary, keeps node units auxiliary, preserves user-bus diagnostics until a gateway unit is found, limits cgroup exemptions to known OpenClaw sibling services, and includes redacted real Linux/WSL proof.

## Reproduction Assessment

Yes, at source level: current main documents Linux system services while `readSystemdServiceRuntime` still probes `systemctl --user` and restart health has no cgroup attribution. I did not run a live headless systemd host in this read-only review.

## Solution Assessment

No, not as currently implemented. The system-bus and cgroup-aware direction is appropriate, but the branch needs narrower gateway-unit selection, preserved diagnostics, constrained cgroup exemptions, and real after-fix proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve user-bus diagnostics until a gateway system unit is found:** `src/daemon/systemd.ts:715-722`
  - body: When the user bus fails, this branch returns a stopped system-scoped missing unit whenever the system bus is reachable, even if no OpenClaw gateway unit was observed. That drops the original no-medium or linger diagnostic for user-systemd installs, so only switch to system scope after finding a real gateway unit.
  - confidence: 0.88
- **[P1] Keep node units out of gateway primary selection:** `src/daemon/systemd-system-probe.ts:160-164`
  - body: `pickPrimaryGatewayUnit` selects from the same candidate list that includes `openclaw-node-host.service` and `openclaw-node.service`. If only node-host is active, gateway status can report the gateway running with the node PID/cgroup, masking a missing or stopped gateway unit.
  - confidence: 0.9
- **[P2] Limit cgroup exemptions to known OpenClaw services:** `src/cli/daemon-cli/restart-health.ts:134-135`
  - body: Any two different `*.service` cgroups are treated as distinct, so an unrelated listener under another service can be removed from `staleGatewayPids`. Restrict the exemption to recognized gateway/node sibling service pairs.
  - confidence: 0.86
