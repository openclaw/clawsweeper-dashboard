## What This Changes

The PR adds system-bus systemd runtime probing, system-unit status output, `/proc/<pid>/cgroup` parsing, cgroup-aware stale PID filtering, hint wording, and mocked daemon/CLI tests for Linux gateway status.

## Best Possible Solution

Land a coordinated systemd-status fix that probes real gateway system units first, keeps node units auxiliary, preserves user-bus diagnostics unless a gateway system unit is observed, limits cgroup exemptions to known OpenClaw sibling services, and includes redacted real systemd proof.

## Reproduction Assessment

Yes, at source level: current main documents Linux system-level gateway units while `readSystemdServiceRuntime` still depends on `systemctl --user`, and restart-health has no cgroup attribution. I did not run a live headless systemd host in this read-only review.

## Solution Assessment

No, not as currently implemented. The system-bus and cgroup-aware direction is appropriate, but the patch needs narrower gateway-unit selection, safer diagnostics, constrained cgroup exemptions, current-main reconciliation, and real after-fix proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve user-bus errors until a gateway system unit is observed:** `src/daemon/systemd.ts:715-722`
  - body: When the user bus fails, this branch returns a stopped system-scoped missing unit whenever the system bus is reachable, even if no OpenClaw gateway unit was observed. That drops the original no-medium or linger diagnostic for user-systemd installs, so only switch to system scope after an actual gateway unit is found.
  - confidence: 0.9
- **[P1] Keep node units out of gateway primary selection:** `src/daemon/systemd-system-probe.ts:160-164`
  - body: `pickPrimaryGatewayUnit` iterates candidates that include `openclaw-node.service` and `openclaw-node-host.service`, so a host with only the node unit active can make gateway status report the node PID/cgroup as the gateway runtime. Node units should remain auxiliary context, not primary gateway candidates.
  - confidence: 0.92
- **[P2] Limit cgroup exemptions to known OpenClaw sibling units:** `src/cli/daemon-cli/restart-health.ts:132-135`
  - body: This returns `distinct` for any two different service cgroups, including unrelated services like `user@1000.service`, which can suppress stale gateway listener reporting. Restrict the exemption to recognized gateway/node sibling service pairs so conflicting listeners still surface.
  - confidence: 0.88
