## Summary

Keep open. The PR addresses a valid supported Linux system-service status bug, but current main does not implement the system-bus runtime fallback or cgroup-aware stale PID filtering, and the proposed patch introduces two blocking correctness regressions in the fallback selection and stale-listener filtering paths.

## What This Changes

Adds a system-bus systemd runtime probe, system-level unit status output, cgroup-aware stale PID filtering, hint wording changes, and daemon/CLI tests for headless Linux gateway status.

## Best Possible Solution

Land a rebased implementation that probes supported system-level gateway units when the user bus is unavailable, keeps node units as auxiliary context, limits cgroup dedup suppression to known safe OpenClaw gateway/node sibling units, and preserves current restart-health semantics.

## Reproduction Assessment

Yes. The reported host shape is concrete, current main shows the user-bus-only failure and PID-only stale-listener logic, and both the original bug and the PR regressions can be reproduced with mocked systemctl output, listener PIDs, and cgroup readers.

## Solution Assessment

No. The approach is directionally right, but this patch should be narrowed so the primary runtime can only be a gateway unit and cgroup-based suppression only applies to known safe sibling OpenClaw units.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restrict primary runtime selection to gateway units:** `src/daemon/systemd-system-probe.ts:160-164`
  - body: `pickPrimaryGatewayUnit` walks `buildGatewayCandidates`, but that list includes node units. If only `openclaw-node-host.service` is active, `gateway status` can report the gateway as running with the node PID/cgroup, masking a stopped or missing gateway. Keep node units in the auxiliary `systemUnits` list, but select the primary runtime from gateway unit names only.
  - confidence: 0.9
- **[P2] Narrow cgroup dedup to known sibling units:** `src/cli/daemon-cli/restart-health.ts:132-135`
  - body: This treats any two different service cgroups as distinct. Since port classification marks any command containing `openclaw` as a gateway listener, an old or foreign OpenClaw listener under another service can disappear from `staleGatewayPids`. Limit the bypass to known safe gateway/node sibling units.
  - confidence: 0.86
