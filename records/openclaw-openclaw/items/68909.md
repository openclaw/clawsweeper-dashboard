## Summary

Keep open: current main still lacks the documented system-service status behavior, but this PR should not merge as-is because it lacks after-fix real behavior proof and still has blocking daemon/status regressions.

## What This Changes

The PR adds a system-bus fallback for daemon runtime status, cgroup-based duplicate supervisor filtering, system-unit status output, hint wording updates, and mocked tests.

## Best Possible Solution

Land a narrowed daemon/status fix that probes real gateway system units, leaves node units as auxiliary context, preserves user-bus diagnostics unless a gateway system unit is observed, and limits cgroup exemptions to recognized OpenClaw sibling services.

## Reproduction Assessment

Yes, at source level. Current docs support system-level Linux gateway units, while current main still reads runtime through the user systemd bus and stale-PID filtering has no cgroup-aware sibling-service distinction.

## Solution Assessment

No, not as currently implemented. The system-bus and cgroup-aware direction is appropriate, but this patch needs to narrow gateway selection, preserve user-bus failure diagnostics, and constrain cgroup exemptions before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve user-bus errors when no system unit exists:** `src/daemon/systemd.ts:715-722`
  - body: When the user bus fails but the system bus is reachable, missing system units make this return a stopped system-scoped missing unit and drop the original no-medium detail. User-systemd installs then lose the linger/XDG_RUNTIME_DIR guidance they need, so only switch to system scope after observing a real gateway system unit.
  - confidence: 0.88
- **[P1] Keep node units out of primary gateway selection:** `src/daemon/systemd-system-probe.ts:160-164`
  - body: pickPrimaryGatewayUnit searches the same candidate list used for discovery, and that list includes openclaw-node-host.service and openclaw-node.service. If only the node-host unit is active, gateway status will report the gateway as running with the node-host PID/cgroup, masking a missing or stopped gateway unit.
  - confidence: 0.9
- **[P2] Limit cgroup exemptions to known sibling services:** `src/cli/daemon-cli/restart-health.ts:134-135`
  - body: This treats any two different *.service cgroups as safely distinct. A stale OpenClaw listener running under an unrelated service such as user@1000.service would be removed from staleGatewayPids, preventing status/restart cleanup from reporting the conflict.
  - confidence: 0.86
