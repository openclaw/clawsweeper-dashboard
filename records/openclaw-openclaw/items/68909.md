## Summary

Keep open. The PR targets a real current-main daemon/status gap, but the proposed patch is not merge-ready because the system-bus fallback can hide user-systemd diagnostics, node units can become the primary gateway runtime, and cgroup dedup is broader than the OpenClaw sibling-service case it is meant to exempt.

## What This Changes

The PR adds a systemd system-bus fallback to gateway runtime status, surfaces system unit runtime fields, adds cgroup-aware stale-PID filtering, adjusts systemd-unavailable hints, and adds focused tests.

## Best Possible Solution

Land a narrower daemon/status fix that probes real system-level gateway units after user-bus failure, keeps node units as auxiliary context, limits cgroup exemptions to recognized OpenClaw gateway/node sibling services, and preserves existing user-systemd guidance unless a gateway system unit is actually observed.

## Reproduction Assessment

Yes. The original report can be reproduced with a failed systemctl --user probe, active system-level openclaw-host-gateway/openclaw-node-host units, and sibling service cgroups; the review blockers are reproducible with mocked systemctl and cgroup inputs against the PR code.

## Solution Assessment

No. The system-bus plus cgroup-aware direction is appropriate, but this diff makes fallback and dedup decisions too broadly; the safer path is to separate gateway-primary unit selection from auxiliary node context and constrain cgroup suppression to known OpenClaw service pairs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve user-bus errors when no system unit exists:** `src/daemon/systemd.ts:715-722`
  - body: When the user bus fails but the system bus is reachable, missing units can still make the probe report systemBusAvailable with no OpenClaw units. This branch then returns a stopped system-scoped missing unit and drops the original systemctl --user no-medium detail, so user-systemd installs lose the linger/XDG_RUNTIME_DIR guidance they need. Only switch to system scope after observing a real gateway system unit.
  - confidence: 0.88
- **[P1] Keep node units out of primary gateway selection:** `src/daemon/systemd-system-probe.ts:160-164`
  - body: pickPrimaryGatewayUnit uses the same candidate list that includes openclaw-node-host.service and openclaw-node.service. If only the node-host unit is active, gateway status can report the gateway as running with the node PID/cgroup, masking a missing or stopped gateway. Use a gateway-only primary list and keep node units only as auxiliary systemUnits context.
  - confidence: 0.9
- **[P2] Limit cgroup exemptions to known sibling services:** `src/cli/daemon-cli/restart-health.ts:132-135`
  - body: This treats any two different *.service cgroups as safely distinct. A stale OpenClaw listener under another service, such as user@1000.service, would be removed from staleGatewayPids and the conflict would no longer be reported. Restrict the exemption to recognized OpenClaw gateway/node sibling unit pairs.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `CHANGELOG.md:15`
  - body: This is a user-facing fix, but the PR does not update CHANGELOG.md. Add a single-line Fixes entry under Unreleased so the PR satisfies the repo's changelog policy before merge.
  - confidence: 0.92
