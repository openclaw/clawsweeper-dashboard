## Summary

Keep open: the reported Linux systemd system-unit status bug is source-reproducible, but this PR is not merge-ready because real after-fix host proof is missing, the branch conflicts with current main, and the patch still has gateway-runtime selection and stale-listener filtering defects.

## What This Changes

The PR adds a Linux systemd system-bus runtime probe, system-unit status output, `/proc/<pid>/cgroup` parsing, cgroup-aware stale supervisor filtering, and daemon/CLI tests.

## Best Possible Solution

Land a coordinated systemd-status fix that probes real gateway system units first, keeps node units auxiliary, preserves user-bus diagnostics unless a gateway system unit is observed, limits cgroup exemptions to known OpenClaw sibling services, rebases on current restart-health behavior, and includes redacted real systemd proof.

## Reproduction Assessment

Yes, at source level: current main documents Linux system-level gateway units while `readSystemdServiceRuntime` still depends on `systemctl --user`, and restart-health has no cgroup attribution. I did not run a live headless systemd host in this read-only review.

## Solution Assessment

No, not as currently implemented. The system-bus and cgroup-aware direction is appropriate, but the patch needs narrower gateway-unit selection, safer diagnostics, constrained cgroup exemptions, current-main reconciliation, and real after-fix proof.

Full review comments:

- **[P1] Preserve user-bus errors until a gateway system unit is observed:** `src/daemon/systemd.ts:715-722`
  - body: When the user bus fails, this branch returns a stopped system-scoped missing unit whenever the system bus is reachable, even if no OpenClaw gateway unit was observed. That drops the original no-medium or linger diagnostic for user-systemd installs, so only switch to system scope after finding a real gateway unit.
  - confidence: 0.88
- **[P1] Keep node units out of gateway primary selection:** `src/daemon/systemd-system-probe.ts:160-164`
  - body: `pickPrimaryGatewayUnit` iterates the combined candidate list, which includes `openclaw-node-host.service` and `openclaw-node.service`. If only a node-host system unit is active, gateway status can report the gateway as running with the node-host PID/cgroup; choose the primary from gateway unit names only and keep node units as auxiliary context.
  - confidence: 0.9
- **[P2] Limit cgroup exemptions to known OpenClaw sibling units:** `src/cli/daemon-cli/restart-health.ts:132-135`
  - body: This treats any differing `*.service` cgroup names as safely distinct, so a conflicting gateway listener under an unrelated service such as `user@1000.service` can be removed from `staleGatewayPids`. Limit the shortcut to recognized gateway/node sibling pairs or keep the conservative stale-PID report.
  - confidence: 0.84
