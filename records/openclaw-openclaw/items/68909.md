## Summary

Keep open: the underlying daemon/status bug is still valid on current main, but this PR lacks after-fix real behavior proof and still has blocking correctness risks in the system-bus fallback and cgroup dedup logic.

## What This Changes

Adds a Linux systemd system-bus fallback for gateway status, system-unit status output, `/proc` cgroup parsing, and cgroup-aware stale supervisor filtering.

## Best Possible Solution

Land a narrowed daemon/status fix that probes real gateway system units, keeps node units auxiliary, preserves user-bus diagnostics unless a gateway system unit is observed, constrains cgroup exemptions to known OpenClaw sibling services, and includes redacted real systemd proof.

## Reproduction Assessment

Yes, at source level: current main documents system-level Linux units while `readSystemdServiceRuntime` still probes `systemctl --user` and `inspectGatewayRestart` has no cgroup attribution. I did not run a live headless systemd host in this read-only review.

## Solution Assessment

No, not as currently implemented. The system-bus and cgroup-aware direction is appropriate, but this patch should be narrowed to gateway primary units, known sibling cgroups, preserved user-bus diagnostics, and real after-fix proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve user-bus errors until a gateway system unit is observed:** `src/daemon/systemd.ts:715-722`
  - body: When the user bus fails, this branch returns a stopped system-scoped missing unit whenever the system bus is reachable, even if no OpenClaw gateway unit was observed. That drops the original no-medium/linger diagnostic for user-systemd installs, so only switch to system scope after finding a real gateway unit.
  - confidence: 0.88
- **[P1] Keep node units out of gateway primary selection:** `src/daemon/systemd-system-probe.ts:160-164`
  - body: `pickPrimaryGatewayUnit` selects from the same candidate list that includes `openclaw-node-host.service` and `openclaw-node.service`. If only node-host is active, gateway status can report the gateway running with the node PID/cgroup, masking a missing or stopped gateway unit.
  - confidence: 0.9
- **[P2] Limit cgroup exemptions to known OpenClaw sibling units:** `src/cli/daemon-cli/restart-health.ts:134-135`
  - body: Any two different `*.service` cgroups are treated as distinct, so an unrelated listener under another service can be removed from `staleGatewayPids`. Restrict the exemption to recognized gateway/node sibling service pairs.
  - confidence: 0.86
