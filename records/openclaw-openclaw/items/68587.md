## Summary

Keep open: current main still has a source-backed `openclaw mcp serve` stdio safety gap, and https://github.com/openclaw/openclaw/pull/68800 remains an open closing PR rather than a merged fix. The SDK-backed tool registration path should answer `tools/list`, but startup/config output can still pollute stdout before strict clients receive valid JSON-RPC.

## What This Changes

The issue asks for `openclaw mcp serve` to return `tools/list` as a normal JSON-RPC request/response so strict MCP clients such as Hermes Agent can discover tools.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/68800 with a fix that routes or suppresses MCP serve startup output before config preflight and proves strict stdio `tools/list` behavior.

## Reproduction Assessment

Yes, at source level. `openclaw mcp serve` is documented as stdio, preAction/config preflight can run before the command action, and current main lacks MCP-specific stdout routing before that path.

## Solution Assessment

Yes in direction, but not complete until the linked PR or an equivalent replacement lands. The narrow maintainable fix is to reserve stdout before startup/config output and add strict stdio proof for `tools/list`.
