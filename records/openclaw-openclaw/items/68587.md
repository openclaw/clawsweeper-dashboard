## Summary

Keep open: current main still has a source-backed `openclaw mcp serve` stdout-safety gap before CLI/config preflight, and an open closing PR remains the active fix path rather than a reason to close this issue early.

## What This Changes

The issue reports that `openclaw mcp serve` breaks strict stdio MCP clients during `tools/list` discovery because the response stream is not a clean request/response JSON-RPC flow.

## Best Possible Solution

Review, repair, or replace https://github.com/openclaw/openclaw/pull/68800 so `openclaw mcp serve` routes or suppresses startup output before config preflight and has strict stdio `tools/list` regression proof.

## Reproduction Assessment

Yes, at source level: `openclaw mcp serve` is documented as stdio MCP, current startup can run config preflight before the MCP action, and that preflight can write warning text to stdout when not suppressed. I did not run a live Hermes client or raw stdio session in this read-only review.

## Solution Assessment

Yes in direction: the narrow maintainable fix is to reserve stdout before CLI/config preflight and assert a clean `tools/list` request/response path. The issue should stay open until the linked PR lands or is replaced by an equivalent fix.
