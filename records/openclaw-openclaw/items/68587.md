## What This Changes

The issue asks for `openclaw mcp serve` to answer MCP `tools/list` as a normal JSON-RPC request/response for strict stdio clients such as Hermes Agent.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/68800 with a fix that reserves stdout before CLI/config preflight and includes strict raw stdio `tools/list` proof.

## Reproduction Assessment

Yes, at source level. `openclaw mcp serve` is documented as stdio, current startup/preflight can run before command action handling, and current main lacks MCP-specific stdout routing before that path.

## Solution Assessment

Yes in direction, but not complete while the linked PR is open and conflicting. The narrow maintainable solution is to reserve stdout before startup/config output, keep the SDK-backed `tools/list` request handler path, and prove strict stdio behavior.
