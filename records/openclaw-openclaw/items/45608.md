## Summary

Keep open: current main still archives or rotates the outgoing session before any reset-time agentic memory-flush turn, and the existing fallback paths are either fire-and-forget hooks or raw session-memory capture. The related filename-read mismatch was fixed, and the open synthesis PR overlaps the hook fallback, but neither implements the requested curated pre-reset flush for `/new`, `/reset`, daily, and idle rollover.

## What This Changes

Request a silent agentic memory flush before session reset or rollover destroys the current session, reusing the existing pre-compaction memory-flush mechanism.

## Best Possible Solution

Define a bounded, opt-in reset-lifecycle memory flush contract that reuses the existing memory-flush runner where safe, covers manual/daily/idle reset consistently, and documents timeout, config, sandbox, and failure behavior.

## Reproduction Assessment

Yes for the source-level gap: a stale daily or idle session reaches `initSessionState`, which creates a new session and archives the previous transcript without invoking `runMemoryFlushIfNeeded`. I did not run a live gateway reproduction because this review is read-only.

## Solution Assessment

Unclear as a final implementation. Reusing the existing memory-flush primitive is plausible, but the maintainable solution needs explicit reset-lifecycle policy, config shape, timeout/cancellation, and safety semantics before code should land.
