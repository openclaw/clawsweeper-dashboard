- `impact:data-loss`: The report is about session context being silently lost across reset boundaries when no curated memory save occurs first.

## Summary

Keep open: current main and v2026.5.19 still archive/reset outgoing sessions without a curated agentic memory-flush turn before teardown. The existing raw session-memory hook, startup memory loading, and related open PRs are useful partial paths, but the requested silent tool-capable reset-time flush still needs maintainer product and security decisions.

## What This Changes

The issue requests a silent agentic memory flush before manual, daily, and idle session resets so durable memory can be curated before the outgoing session is archived.

## Best Possible Solution

Define a guarded reset-time memory-flush contract for manual, daily, and idle rollovers, with explicit defaults/config, timeout and failure behavior, sandbox/tool policy, retention/redaction rules, docs, and focused tests coordinated with the related open PRs.

## Reproduction Assessment

Not applicable as a strict bug reproduction; this is a feature request. Source inspection gives high confidence that current main and v2026.5.19 replace/archive outgoing sessions without invoking an agentic memory-flush turn first.

## Solution Assessment

Unclear as a final implementation. Reusing the existing memory-flush primitive is plausible, but maintainers still need to decide blocking versus background behavior, config shape, reset coverage, tool/sandbox policy, and retention semantics.
