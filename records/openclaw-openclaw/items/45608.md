## Summary

Keep open. Current main has reset archival, lifecycle hooks, startup memory preload, and the mechanical session-memory fallback, but it still does not run an outgoing agentic memory-flush turn before manual, daily, or idle reset replaces the session.

## What This Changes

Request a silent agentic memory flush before manual, daily, and idle session resets so durable memories can be curated before the outgoing session is replaced.

## Best Possible Solution

Land a bounded reset-time flush design that covers manual, daily, and idle reset, keeps session-memory as the raw audit fallback, and explicitly defines timeout, observability, sandbox, and approval behavior.

## Reproduction Assessment

Yes. A source-level trace shows the memory flush only runs in the normal reply path, while bare reset commands and daily/idle rollover replace the session through reset hooks and session lifecycle hooks without an outgoing flush turn.

## Solution Assessment

Unclear. Reusing the existing memory-flush primitive is plausible, but the final implementation should be reviewed through open PR #45643 with explicit reset policy, timeout, auditability, and security boundaries.
