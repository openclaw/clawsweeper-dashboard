## Summary

Keep open: current main still rotates explicit and automatic reset sessions without running the agentic memory-flush path first. The related slugged-file startup visibility gap is partly addressed, but the central reset-time agentic flush remains a product and security decision.

## What This Changes

The issue requests an agentic memory flush before manual and automatic session resets so durable memory can be curated before session state is replaced.

## Best Possible Solution

Design a maintainer-approved reset-lifecycle memory preservation path that covers explicit and automatic rollovers while keeping latency, consent, sandbox, and plugin boundaries explicit.

## Reproduction Assessment

Yes, from source inspection: reset rollover and memory-flush execution are separate current-main paths, so explicit or stale reset can rotate the session before any agentic pre-reset write is invoked. I did not run an end-to-end reset because this is a read-only review.

## Solution Assessment

Unclear: reusing the existing memory-flush turn is plausible, but blocking reset, background cleanup, copying existing compaction summaries, or extending lifecycle hooks have different latency and security tradeoffs.
