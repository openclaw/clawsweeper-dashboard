## Summary

Keep open. Current main has reset-time archival, lifecycle hooks, and a mechanical session-memory fallback, but it still does not run an outgoing agentic memory-flush turn before manual, daily, or idle reset replaces the session.

## What This Changes

The issue requests a silent agentic memory flush before `/new`, `/reset`, daily reset, and idle reset destroy the outgoing session.

## Best Possible Solution

Keep this as the canonical reset-time curated memory thread, then land a bounded implementation that flushes the outgoing session for manual, daily, and idle reset while preserving session-memory as the raw audit fallback.

## Reproduction Assessment

Yes. A source-level trace shows bare `/new` and `/reset` return after reset hooks, while daily and idle rollover happen inside `initSessionState`; the only production `runMemoryFlushIfNeeded` call remains in the normal reply runner.

## Solution Assessment

Unclear for a final implementation because current main has no reset-flush path yet. Reusing the existing memory-flush primitive is plausible, but the accepted solution should define reset reasons, config, timeout, failure visibility, and security policy before landing.
