## Summary

Keep open: current main still has compaction-time agentic memory flush and mechanical/background reset-memory capture, but no reset, daily, or idle rollover path gives the outgoing session a silent tool-enabled agent turn before teardown.

## What This Changes

Request to run a silent, tool-enabled memory flush before `/new`, `/reset`, daily, and idle session resets archive or replace the outgoing session.

## Best Possible Solution

Define one opt-in reset-memory lifecycle policy covering manual, daily, and idle resets, with explicit timeout, sandbox/tool policy, audit/error behavior, and fresh-install plus upgrade/default tests.

## Reproduction Assessment

Yes at source level: stale or explicit reset through `initSessionState` mints a new session and archives the previous transcript, while the only agentic memory flush call remains in the post-session-init reply runner; no live test was run because this review was read-only.

## Solution Assessment

Unclear until maintainers choose the lifecycle contract; reusing the existing memory-flush primitive is plausible, but blocking vs background execution, config shape, and security/audit bounds need a product decision.
