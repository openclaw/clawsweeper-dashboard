- `impact:data-loss`: The report is about session context being silently lost across reset boundaries when no curated memory save occurs first.

## Summary

Keep open: current main and v2026.5.20 still reset or archive outgoing sessions without an awaited, tool-capable agentic memory-flush turn against that outgoing session. The raw session-memory hook, startup memory prelude, and related open PRs are useful partial continuity paths, but the requested reset-time agent turn still needs maintainer product and security decisions.

## What This Changes

The issue requests a pre-reset agentic memory flush before manual, daily, and idle session resets so durable memory can be written before the outgoing session is destroyed.

## Best Possible Solution

Define one guarded reset-memory contract for manual, daily, and idle rollovers, including config/defaults, timeout and failure behavior, sandbox/tool policy, retention/redaction rules, startup trust boundaries, docs, and focused tests coordinated with https://github.com/openclaw/openclaw/pull/48235 and https://github.com/openclaw/openclaw/pull/50584.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high confidence that current main and v2026.5.20 reset or archive outgoing sessions without an agentic memory-flush turn against the outgoing session.

## Solution Assessment

Unclear as a final implementation. Reusing the existing memory-flush primitive is plausible, but maintainers still need to decide reset coverage, blocking versus background behavior, config shape, tool/sandbox policy, and retention semantics.
