## Summary

Keep open. Current main and the latest release still render URL-bearing tool output as text in the Control UI and native WebChat paths, and the only directly related implementation PR is closed unmerged.

## What This Changes

Request to render URLs inside Control UI and native WebChat tool output cards as clickable links while preserving safe formatting.

## Best Possible Solution

Add a shared escape-first http/https URL renderer for Control UI tool outputs across every tool-card surface, plus equivalent native WebChat URL rendering if that scope remains accepted.

## Reproduction Assessment

Yes. Source inspection on current main shows URL-bearing tool output flows through Lit text bindings, `getTruncatedPreview`, and SwiftUI `Text`, so anchors are not produced without needing to run the UI.

## Solution Assessment

Yes. A shared escape-first link renderer is the narrow maintainable direction; the closed prior PR is useful prior art but under-covers expanded, raw, grouped, sidebar, and native WebChat paths.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/8812 by making http/https URLs in tool output cards clickable without changing tool data semantics. Implement an escape-first URL-aware renderer in the Control UI tool-card/helper layer, preserve JSON/plain-text formatting, support safe external anchors only (`target="_blank"` and `rel="noopener noreferrer"`), handle trailing punctuation/whitespace, and cover expanded output, raw details, collapsed previews, inline short cards, grouped tool-message paths, and sidebar/raw views. Use https://github.com/openclaw/openclaw/pull/8880 as prior art, but do not copy its partial coverage blindly. If native WebChat is kept in scope, add equivalent SwiftUI URL rendering with AttributedString or Link handling rather than plain Text; otherwise document the deferral in the PR body. Add focused regression tests and a CHANGELOG.md entry for the user-visible Control UI/WebChat improvement. Do not use unsafeHTML unless all arbitrary tool text is escaped before link creation, and do not linkify non-http schemes.
