Keep open. Current main and the latest release still render URL-bearing tool output as plain text in the relevant Control UI and native WebChat paths, and the linked implementation attempts are closed unmerged prior art rather than shipped fixes.

## What This Changes

Request to render URLs inside Control UI tool output cards, and possibly native WebChat tool results, as clickable links instead of plain text.

## Best Possible Solution

Implement safe URL-aware rendering across the Control UI tool-output paths, and decide or split the native WebChat scope before accepting a fix PR.

## Reproduction Assessment

Yes, at source level. Current main renders URL-bearing Control UI tool output through text bindings and native WebChat through plain strings, with no anchor-producing tool-output renderer.

## Solution Assessment

No, current main has not solved the issue. The maintainable path is a safe URL-aware renderer that preserves formatting and escaping, plus a maintainer scope decision for native WebChat.
