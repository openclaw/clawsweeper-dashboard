## Summary

Keep open: current main and v2026.6.10 still render URL-bearing tool output as plain escaped text in Control UI and native WebChat, and the two linked implementation PRs are closed unmerged prior art rather than shipped fixes.

## What This Changes

Request to render http(s) URLs inside Control UI and possible native WebChat tool output cards as safe clickable links instead of plain text.

## Best Possible Solution

Implement escape-first URL-aware rendering across Control UI tool-output paths, then either add native WebChat attributed link support or explicitly split that scope into a separate issue.

## Reproduction Assessment

Yes. At source level, current main renders URL-bearing Control UI tool output through Lit text bindings and native WebChat through plain SwiftUI `Text`, so no browser/app run is needed to prove the missing link renderer.

## Solution Assessment

Unclear as a product decision. The Control UI fix shape is clear, but maintainers should decide whether native WebChat stays in this issue before another implementation attempts to close the broader request.
