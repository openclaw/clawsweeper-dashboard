## Summary

Keep open: the target checkout still renders URL-bearing tool output as plain text in the Control UI and native WebChat paths, and the related implementation attempt is closed unmerged rather than shipped.

## What This Changes

Request to render http/https URLs inside Control UI and, if applicable, native WebChat tool-result cards as clickable links instead of plain text.

## Best Possible Solution

Add an escape-first safe http/https link renderer across all Control UI tool-output surfaces, and either add equivalent native WebChat Link/AttributedString handling or explicitly split native follow-up scope.

## Reproduction Assessment

Yes for source-level current behavior: Control UI tool cards bind output text directly, and native WebChat renders tool output through SwiftUI Text. I did not run a browser or native UI session in this read-only review, but the current render path is clear.

## Solution Assessment

Yes. The maintainable direction is an escape-first tool-output renderer that only emits safe http/https anchors and avoids routing arbitrary tool output through generic markdown or unescaped unsafeHTML.
