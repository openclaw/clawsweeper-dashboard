## Summary

Keep open. Current main still renders URL-bearing tool outputs as text/code in Control UI tool cards and as plain SwiftUI Text in native WebChat; closed PR #8880 is useful prior art but was not merged or shipped.

## What This Changes

The issue requests clickable URL rendering for tool output cards in Control UI and native WebChat.

## Best Possible Solution

Add a shared safe URL-aware renderer for tool output, reuse it across Control UI tool-card surfaces, and add native WebChat URL rendering if that surface remains in scope.

## Reproduction Assessment

Yes. Source inspection shows a tool result containing `https://example.com` flows through Lit text/code bindings in Control UI and plain SwiftUI `Text` in native WebChat, so URLs remain non-clickable on current main.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix, but it should be implemented as an escape-first, http/https-only renderer with coverage for each tool-output surface rather than raw HTML substitution.
