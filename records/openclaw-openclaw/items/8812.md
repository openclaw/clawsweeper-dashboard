## Summary

Keep open. Current main still renders URL-bearing tool output as escaped text/code in Control UI and plain SwiftUI text in native WebChat, while the directly related implementation remains closed unmerged.

## What This Changes

Request clickable http/https links for URLs inside Control UI tool-output cards and, if applicable, native WebChat tool results.

## Best Possible Solution

Add a shared escape-first http/https URL renderer across Control UI tool-output surfaces, with equivalent SwiftUI AttributedString or Link rendering if native WebChat remains in scope.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence because URL-bearing output flows through text/code bindings, getTruncatedPreview, direct card.outputText rendering, and SwiftUI Text on current main.

## Solution Assessment

Yes. The narrow maintainable path is shared escape-first URL rendering applied to every tool-output surface, not ad hoc regex HTML injection in one card path.
