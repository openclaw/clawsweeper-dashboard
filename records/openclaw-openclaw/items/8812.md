## Summary

Keep open: current main and the latest release still render URL-bearing tool output as escaped/plain text in Control UI and native WebChat, and the linked implementation attempts are closed unmerged prior art rather than shipped fixes.

## What This Changes

The issue requests clickable URL rendering for URL-like values in Control UI and WebChat tool output cards.

## Best Possible Solution

Implement an escape-first URL-aware renderer across Control UI tool-output paths, then either add equivalent native WebChat link rendering or explicitly split that scope.

## Reproduction Assessment

Yes, at source level: current main renders URL-bearing Control UI tool output through Lit text bindings and native WebChat through plain SwiftUI `Text`. I did not run browser or app proof because this read-only review can prove the missing link renderer from source.

## Solution Assessment

Unclear as a product decision: the Control UI fix shape is clear, but maintainers should decide whether native WebChat remains in scope before another implementation. A safe renderer should preserve raw formatting, escape text first, and only produce safe http/https anchors.
