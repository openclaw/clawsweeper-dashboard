## Summary

Keep open: current main still renders URL-bearing tool output as plain text in the affected Control UI and native WebChat paths, and the only linked implementation attempt is closed unmerged rather than shipped.

## What This Changes

Request auto-linkification of URL values in Control UI tool-output cards and, if applicable, native WebChat tool-result rendering.

## Best Possible Solution

Implement escape-first http/https URL rendering across all Control UI tool-output surfaces, with native WebChat handled in the same change only if it can use a small Link or AttributedString path.

## Reproduction Assessment

Yes, for source-level current behavior: Control UI tool output is rendered as Lit text bindings and native WebChat tool output as SwiftUI Text. I did not run a browser/native UI repro in this read-only sweep, but the current-main render path is clear.

## Solution Assessment

Yes. The maintainable fix is to escape arbitrary tool output first, linkify only safe http/https URLs in the tool-output renderers, and avoid routing arbitrary tool output through generic markdown or unescaped unsafeHTML.
