## Summary

Keep open: current main still renders URL-bearing tool output as plain text in Control UI tool-card paths and native WebChat, while normal markdown linkification does not cover those tool-card renderers. The related implementation at https://github.com/openclaw/openclaw/pull/8880 was closed unmerged, so it remains useful prior art rather than shipped behavior.

## What This Changes

Add safe clickable http/https links for URL strings displayed in Control UI and, if kept in scope, native WebChat tool-output cards.

## Best Possible Solution

Add an escape-first, safe-scheme URL renderer across the Control UI tool-output surfaces, with equivalent SwiftUI link rendering if native WebChat remains in scope, and cover the paths with focused regression tests.

## Reproduction Assessment

Yes, by source inspection on current main. Control UI binds tool-output strings as text in expanded, preview, inline, and grouped card paths, and native WebChat renders formatted tool results through plain SwiftUI Text.

## Solution Assessment

Yes. The narrow maintainable solution is an escape-then-linkify renderer for safe http/https URLs across the tool-card paths; applying markdown wholesale or using unsafe regex HTML in one view would be incomplete and riskier.
