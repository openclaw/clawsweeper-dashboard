## Summary

Keep open: current main still renders URL-bearing tool output as plain text in Control UI tool-card paths and native WebChat, while normal markdown linkification does not cover those tool-card renderers; the related implementation at https://github.com/openclaw/openclaw/pull/8880 was closed unmerged.

## What This Changes

Requests clickable http/https links for URLs in Control UI and WebChat tool output cards while preserving JSON/plain-text structure.

## Best Possible Solution

Add an escape-first, safe-scheme URL renderer across Control UI tool-output surfaces, and include or separately track equivalent SwiftUI URL rendering if native WebChat remains in scope.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: render a tool result containing an http/https URL and the Control UI tool-card DOM/native WebChat output will use text renderers rather than anchors or Link/AttributedString spans.

## Solution Assessment

Yes. The narrow maintainable solution is escape-then-linkify safe http/https URLs in the tool-output renderers; applying markdown wholesale or injecting regex-built HTML in only one view would be incomplete and riskier.
