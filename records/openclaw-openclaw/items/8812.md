## Summary

Keep open: current main still renders URL-bearing tool output as text in the Control UI tool-card paths and native WebChat, and the related implementation was closed unmerged rather than shipped.

## What This Changes

Request URL strings in tool output cards to render as clickable safe links in Control UI and, if applicable, native WebChat.

## Best Possible Solution

Add an escape-first, safe-scheme URL renderer across Control UI tool-output surfaces, and add equivalent SwiftUI URL rendering if native WebChat remains in scope, with focused coverage.

## Reproduction Assessment

Yes, source-reproducible on current main. Control UI binds tool-output strings as text in expanded, preview, inline, and grouped paths, and native WebChat renders formatted tool results through plain SwiftUI Text.

## Solution Assessment

Yes. The narrow maintainable solution is an escape-then-linkify renderer for safe http/https URLs across the tool-card paths; applying markdown wholesale or injecting regex-built HTML in only one view would be incomplete and riskier.
