## Summary

Keep open. Current main still renders URL-bearing tool output through plain Lit text/code bindings in Control UI and SwiftUI Text in native WebChat, and PR #8880 was closed unmerged.

## What This Changes

The issue requests clickable http/https URL rendering inside Control UI and native WebChat tool-output cards while preserving JSON/plain-text formatting.

## Best Possible Solution

Implement a shared escape-first, http/https-only URL renderer for tool output, apply it across Control UI tool-output surfaces, and add native SwiftUI AttributedString/Link handling if WebChat remains in scope.

## Reproduction Assessment

Yes, by source inspection: a tool card containing an https URL flows through Lit text/code bindings in Control UI and through SwiftUI Text in WebChat. No live UI test was run in this read-only review, but the current render paths are clear.

## Solution Assessment

Yes. The requested direction is the narrow maintainable solution if implemented as an escape-first safe renderer rather than broad markdown rendering or direct regex-to-HTML injection.
