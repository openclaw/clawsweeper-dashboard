## Summary

Keep open: current main still renders URL-bearing tool output as text in the Control UI tool-card paths and native WebChat, and the related implementation at https://github.com/openclaw/openclaw/pull/8880 was closed unmerged.

## What This Changes

Request auto-linkified http/https URLs in Control UI tool output cards and, where applicable, native WebChat tool result cards.

## Best Possible Solution

Add an escape-first, safe-scheme URL renderer across Control UI tool-output surfaces, with equivalent SwiftUI link rendering if native WebChat remains in scope, and cover those paths with focused tests.

## Reproduction Assessment

Yes, by source inspection on current main. Control UI binds tool-output strings as text in expanded, preview, inline, and grouped paths, and native WebChat renders formatted tool results through plain SwiftUI Text.

## Solution Assessment

Yes. The narrow maintainable direction is escape-then-linkify rendering for safe http/https URLs across the tool-card paths; applying markdown wholesale or injecting regex HTML in one view would be incomplete and riskier.
