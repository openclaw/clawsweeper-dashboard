## Summary

Keep open. Current main and the latest release still render URL-bearing tool output as escaped/plain text in Control UI tool-card paths and native WebChat, while the directly related implementation at https://github.com/openclaw/openclaw/pull/8880 is closed unmerged.

## What This Changes

Request auto-linkifying http/https URLs inside Control UI tool output cards and native WebChat tool-result text.

## Best Possible Solution

Add a shared escape-first http/https URL renderer for all Control UI tool-output surfaces, plus equivalent native WebChat URL rendering if that scope remains accepted.

## Reproduction Assessment

Yes. Source inspection on current main shows URL-bearing tool output flows through Lit text bindings, getTruncatedPreview, markdown code fences, and SwiftUI Text, so anchors are not produced without needing to run the UI.

## Solution Assessment

Yes. A shared escape-first link renderer is the narrow maintainable direction; the closed prior implementation is useful prior art but under-covered expanded, raw, grouped, sidebar, and native WebChat paths.
