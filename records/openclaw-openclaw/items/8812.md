## Summary

Keep open: current main still binds URL-bearing tool output as plain text in Control UI tool cards and native WebChat, while the related implementation at https://github.com/openclaw/openclaw/pull/8880 was closed unmerged.

## What This Changes

Request to render URLs inside Control UI and WebChat tool output cards as safe clickable links.

## Best Possible Solution

Add safe URL-aware rendering for each tool-output card surface, plus the SwiftUI equivalent if kept in scope, without changing the tool-output protocol.

## Reproduction Assessment

Yes, by source inspection on current main. Control UI binds tool-output strings as text in expanded, preview, inline, and grouped card paths, while native WebChat uses plain SwiftUI Text for formatted tool results.

## Solution Assessment

Yes. The narrow maintainable approach is an escape-then-linkify renderer for safe http/https URLs across tool-card paths; applying markdown or unsafe regex HTML in only one view would be incomplete and riskier.
