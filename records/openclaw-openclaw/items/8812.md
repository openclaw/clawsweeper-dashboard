## Summary

Current main still renders URL-bearing tool output as escaped/plain text in the Control UI and as plain SwiftUI text in native WebChat, so this feature request remains valid and should stay open.

## What This Changes

The issue requests clickable http/https URLs inside tool output cards in the Control UI and WebChat while preserving JSON/plain-text formatting safely.

## Best Possible Solution

Add a safe URL-aware renderer for Control UI tool output across expanded, collapsed, inline, raw/sidebar paths, with focused tests, and make an explicit scope call on equivalent native WebChat rendering.

- A naive Control UI implementation that injects HTML into arbitrary tool output without escaping first would create an XSS risk; the renderer should escape text and allow only safe schemes.
- The Control UI has several tool-output paths, so fixing only expanded cards, only previews, or only the sidebar would leave the user-visible request partially unresolved.
- The issue names native WebChat as in scope, so maintainers should confirm whether the first fix must include SwiftUI Link/AttributedString handling or whether web UI support is enough initially.
