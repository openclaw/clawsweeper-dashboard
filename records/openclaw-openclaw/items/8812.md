## Summary

Keep open. Current main and v2026.6.6 still render URL-bearing tool output as plain text in the relevant Control UI and native WebChat paths, and the linked implementation PRs are closed unmerged prior art rather than shipped fixes.

## What This Changes

The issue requests clickable URL rendering inside tool output cards for Control UI, with possible native WebChat coverage.

## Best Possible Solution

Implement safe URL-aware rendering across the Control UI tool-output paths, and add equivalent native WebChat URL spans only if maintainers keep that scope in this issue.

## Reproduction Assessment

Yes, at source level. Current main renders URL-bearing tool output through Lit text bindings and SwiftUI Text, with no anchor-producing helper in the tool-card path.

## Solution Assessment

No, current main has not solved the issue. The best path is a safe URL-aware renderer that preserves formatting and escaping, with a maintainer scope decision for native WebChat.
