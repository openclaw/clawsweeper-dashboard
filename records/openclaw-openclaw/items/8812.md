## Summary

Keep open. Current main still renders Control UI tool output through text/code bindings rather than a URL-aware renderer, native WebChat still displays formatted tool output with plain SwiftUI Text, and the only directly related implementation PR, #8880, was closed unmerged.

## What This Changes

Render URL strings in Control UI and applicable native WebChat tool output cards as clickable links while preserving safe formatting.

## Best Possible Solution

Add a small URL-aware renderer for untrusted tool output that preserves JSON/plain-text layout, trims common trailing punctuation, only links http/https URLs, and is reused across the Control UI tool-output paths; add equivalent SwiftUI URL rendering for native WebChat if that surface remains in scope.

## Reproduction Assessment

Yes. A tool card whose outputText contains a value such as "https://example.com" flows through text/code bindings in Control UI and plain SwiftUI Text in native WebChat, so source inspection gives a high-confidence reproduction path without needing a live browser run.

## Solution Assessment

Yes for the direction, but not for the old partial PR as-is. The maintainable solution is a safe shared renderer with focused coverage across all tool-card surfaces; PR #8880 is useful prior art but only covered collapsed/inline Control UI output and was not merged.
