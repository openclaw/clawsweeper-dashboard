## Summary

Keep open. Current main and the latest release still render URL-bearing tool output as escaped/plain text in Control UI tool-card paths and as plain SwiftUI Text in native WebChat; the related implementation PR is closed unmerged and remains useful prior art.

## What This Changes

Request to render http/https URLs inside Control UI and native WebChat tool output cards as safe clickable links instead of plain text.

## Best Possible Solution

Add a shared escape-first URL-aware renderer for Control UI tool outputs across expanded, raw, collapsed, inline, grouped, and sidebar views, with equivalent SwiftUI URL rendering if WebChat remains in scope.

## Reproduction Assessment

Yes. Source inspection on current main shows URL-bearing tool output flows through Lit text bindings/getTruncatedPreview and SwiftUI Text, so anchors are not produced without running the UI.

## Solution Assessment

Yes. The narrow maintainable direction is a shared escape-first http/https link renderer; reviving https://github.com/openclaw/openclaw/pull/8880 unchanged would under-cover expanded/raw/grouped and WebChat paths.
