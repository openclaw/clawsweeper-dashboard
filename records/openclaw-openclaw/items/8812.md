## What This Changes

Request to render URLs inside Control UI and WebChat tool output cards as safe clickable links.

## Best Possible Solution

Review and land a safe URL-aware renderer for the selected tool-card surfaces, with WebChat/sidebar scope decided before closing this issue.

## Reproduction Assessment

Yes, at source level. URL strings are currently bound as text in Control UI tool-card blocks and SwiftUI Text in native WebChat, while only normal markdown content uses the sanitized markdown linkifier.

## Solution Assessment

Yes for the direction, but the implementation still needs scope clarity. The maintainable solution is a safe renderer that preserves JSON/plain text and defines whether native WebChat and non-expanded tool-card paths are in scope.
