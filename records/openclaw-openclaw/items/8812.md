## Summary

Current main still leaves the requested behavior unresolved: Control UI tool-card output and native WebChat tool-result output render URL strings through text-only paths, while the normal markdown linkifier is separate from those cards. The earlier related PR is useful prior art, but it was closed unmerged and is not shipped behavior.

## What This Changes

Request to render URLs inside Control UI and native WebChat tool-output cards as safe clickable links.

## Best Possible Solution

Implement one safe URL-aware tool-output rendering helper for Control UI and apply it consistently across all tool-card surfaces; add native WebChat URL rendering only if that scope remains desired.

## Reproduction Assessment

Yes, at source level: a tool result containing https://example.com will reach text-only tool-card and native ToolResultCard render paths, while normal chat markdown linkification is routed elsewhere. I did not run a browser or native UI session in this read-only review.

## Solution Assessment

Yes, the best fix direction is a narrow safe linkifier for tool-output rendering, not reusing raw unsafeHTML over unescaped tool output. The implementation should preserve JSON/plain-text formatting, link only safe http/https URLs, and cover every visible tool-output path.
