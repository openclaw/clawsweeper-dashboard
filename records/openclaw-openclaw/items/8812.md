## What This Changes

Request to render URL strings in Control UI and possibly native WebChat tool output cards as clickable links.

## Best Possible Solution

Review and land a safe URL-aware renderer, or split the remaining WebChat/sidebar scope, then close this issue only after the accepted implementation reaches main.

## Reproduction Assessment

## Solution Assessment

Yes for the direction, but it is not implemented on main yet. The maintainable path is a safe URL-aware renderer that preserves formatting, escapes arbitrary text before link creation, and has accepted scope for Control UI versus native WebChat.
