## Summary

Keep open. Current main and the latest release still render URL-bearing tool output as escaped/plain text in Control UI tool cards and native WebChat, while the two linked implementation attempts are closed unmerged and do not own the remaining work.

## What This Changes

Request to make URLs inside Control UI and possible native WebChat tool output cards render as safe clickable links.

## Best Possible Solution

Decide the Control UI versus native WebChat scope, then implement a shared safe URL-aware tool-output renderer with focused coverage across every affected display path.

## Reproduction Assessment

Yes, at source level. Current main renders tool output through Lit text bindings and SwiftUI `Text`, while the markdown renderer that linkifies normal chat text is bypassed by tool-card output paths.

## Solution Assessment

Unclear as requested because the scope spans Control UI and possible native WebChat. The maintainable path is a safe renderer with escaping, safe-scheme filtering, URL-boundary handling, and explicit scope coverage rather than a broad regex/HTML injection patch.

- Maintainers still need to decide whether native WebChat remains in scope or whether this should be narrowed to Control UI first.
