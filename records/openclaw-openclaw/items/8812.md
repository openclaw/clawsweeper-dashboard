## Summary

Keep open. Current main still renders URL-bearing tool outputs as text/code rather than clickable links, and the prior fixing PR was closed unmerged, so there is no shipped implementation to close against.

## What This Changes

Request automatic linkification of http/https URLs in Control UI tool output cards and, if applicable, native WebChat tool result cards.

## Best Possible Solution

Add an escape-first, http/https-only URL renderer shared across Control UI tool-output surfaces, plus equivalent native WebChat URL rendering if that surface remains in scope, with focused tests and a changelog entry.

## Reproduction Assessment

Yes for the observable UI gap. Source inspection shows URL strings flow through Lit text/code bindings in Control UI and SwiftUI `Text` in native WebChat, so current main has no clickable-link path for tool outputs.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix, provided it is implemented as an escape-first safe renderer rather than broad markdown rendering or unsafe regex-to-HTML substitution.
