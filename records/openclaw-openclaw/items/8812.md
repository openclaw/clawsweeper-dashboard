## Summary

Keep open: current main still renders URL-bearing tool output as plain text in Control UI and native WebChat, and the linked implementation PR #8880 was closed unmerged rather than shipped.

## What This Changes

Auto-linkify http/https URLs inside Control UI and WebChat tool-output cards while preserving formatting and safety.

## Best Possible Solution

Add safe URL-aware rendering for all tool-output card surfaces, plus the native SwiftUI equivalent if WebChat remains in scope, without changing the tool-output protocol.

## Reproduction Assessment

Yes, by source inspection on current main. Control UI binds tool-output strings as text in expanded, preview, inline, and grouped card paths, while native WebChat uses plain SwiftUI Text for formatted tool results.

## Solution Assessment

Yes. The maintainable fix is an escape-then-linkify renderer for safe http/https URLs across every tool-card path; injecting regex-generated HTML into only one view would be incomplete and risky.
