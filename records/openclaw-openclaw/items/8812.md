## Summary

Keep open: current main and latest release v2026.6.5 still render URL-bearing tool output through plain-text paths, while both related implementation PRs were closed without merging.

## What This Changes

Request clickable HTTP(S) links in tool output cards across Control UI and, if applicable, native WebChat.

## Best Possible Solution

Add a reusable, safely escaped HTTP(S)-only URL renderer that preserves JSON/plain-text formatting and copy behavior across the agreed Control UI tool-result views, then either include equivalent native WebChat handling or track that scope explicitly as a separate follow-up.

## Reproduction Assessment

Yes. Current-main source provides a high-confidence reproduction path: tool-result values bypass the normal markdown renderer and remain text bindings in both Control UI and native WebChat.

## Solution Assessment

No, the proposed simple regex replacement is not the best complete solution. The maintainable approach is a safe URL-aware rendering contract covering every selected tool-result path while preserving escaping, formatting, punctuation handling, and copied text.
