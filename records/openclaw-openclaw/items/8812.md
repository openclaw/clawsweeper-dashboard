## Summary

Keep open: current main still renders URL-bearing tool outputs as text in the Control UI tool-card paths and native WebChat, while the prior implementation at https://github.com/openclaw/openclaw/pull/8880 was closed unmerged.

## What This Changes

Request to render http/https URLs inside Control UI tool output cards, and possibly native WebChat tool results, as clickable links.

## Best Possible Solution

Add an escape-first, safe-scheme URL renderer across Control UI tool-output surfaces, and add equivalent SwiftUI URL rendering for native WebChat if that surface remains in scope.

## Reproduction Assessment

Yes. This is source-reproducible on current main: Control UI binds tool-output strings as text in expanded, preview, inline, and grouped paths, and native WebChat renders formatted tool results through plain SwiftUI Text.

## Solution Assessment

Yes. The narrow maintainable solution is an escape-then-linkify renderer for safe http/https URLs across the tool-card paths; applying markdown wholesale or injecting regex-built HTML in only one view would be incomplete and riskier.
