## Summary

Keep open: current main and the latest release still render URL-bearing tool output as plain text in the Control UI tool-card paths and native WebChat, and the two linked implementation PRs are closed unmerged prior art rather than shipped fixes.

## What This Changes

Issue requests clickable hyperlink rendering for URLs inside Control UI and WebChat tool output cards.

## Best Possible Solution

Keep this issue as the canonical tracker, decide the Control UI versus native WebChat scope, then implement an escape-safe URL-aware renderer with focused coverage for the affected tool-card paths.

## Reproduction Assessment

Yes, at source level: current main renders Control UI tool output through text bindings/code blocks and native WebChat through plain SwiftUI `Text`. I did not launch either client in this read-only review.

## Solution Assessment

Unclear until maintainers confirm cross-client scope, but the best direction is a safe renderer that preserves text/JSON formatting and only creates safe URL anchors. The closed PRs are useful prior art, not a complete current solution.
