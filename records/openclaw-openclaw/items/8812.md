## Summary

Keep open. Current main and v2026.5.12 still render URL-bearing tool output as escaped text or code in Control UI and as plain SwiftUI Text in native WebChat; the directly related implementation at https://github.com/openclaw/openclaw/pull/8880 was closed unmerged, so there is no shipped fix to close against.

## What This Changes

Request to render URL strings inside Control UI and WebChat tool output cards as safe clickable links.

## Best Possible Solution

Add a shared escape-first http/https URL renderer for all Control UI tool-output surfaces, with equivalent native WebChat URL rendering if that scope remains accepted.

## Reproduction Assessment

Yes. Source inspection on current main shows URL-bearing tool output flows through Lit text bindings, code/pre blocks, getTruncatedPreview, and SwiftUI Text, so anchors are not produced without needing to run the UI.

## Solution Assessment

Yes. The maintainable fix is a shared escape-first link renderer applied across expanded output, raw details, previews, inline cards, grouped tool paths, and sidebar/raw views, with native WebChat handled through SwiftUI URL-capable text rendering if kept in scope.
