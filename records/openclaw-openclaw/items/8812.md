## Summary

Keep open: current main still renders URL-bearing tool output as text in the Control UI tool-card paths and native WebChat, while the related implementation PR at https://github.com/openclaw/openclaw/pull/8880 was closed unmerged.

## What This Changes

The issue requests clickable URL rendering for tool output cards in Control UI and, if applicable, native WebChat.

## Best Possible Solution

Add safe URL-aware rendering for each relevant tool-output surface, preserving JSON/plain-text formatting and keeping native WebChat scope explicit.

## Reproduction Assessment

Yes, by source inspection on current main. Control UI binds tool-output strings as text in expanded, preview, inline, raw, and grouped card paths, while native WebChat uses plain SwiftUI Text for formatted tool results.

## Solution Assessment

Yes. The narrow maintainable approach is an escape-first renderer for safe http/https URLs across tool-card paths; applying markdown wholesale or unsafe regex HTML in only one view would be incomplete and riskier.

## Risks / Open Questions

- A naive HTML-injection implementation would turn arbitrary tool output into an XSS surface; the renderer needs escape-first handling and safe http/https-only anchors.
- Control UI has multiple visible tool-output surfaces, so fixing only one preview path would leave expanded, raw-details, grouped, sidebar, or native views inconsistent.
- Native WebChat was requested only if applicable, so maintainers should decide whether to include it in the same change or track it separately.
