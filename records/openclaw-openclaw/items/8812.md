## Summary

Keep open: current main still renders URL-bearing tool output as escaped text in the Control UI tool-card paths and native WebChat, and the related implementation PR was closed unmerged.

## What This Changes

The issue asks for URLs in tool output cards to render as clickable links in Control UI and, if applicable, native WebChat.

## Best Possible Solution

Add an escape-first, safe-scheme URL renderer across Control UI tool-output surfaces, and include or separately track equivalent SwiftUI URL rendering if native WebChat remains in scope.

## Reproduction Assessment

Yes, source-reproducible on current main. Control UI binds tool-output strings as text in expanded, preview, inline, and grouped paths, and native WebChat renders formatted tool results through plain SwiftUI Text.

## Solution Assessment

Yes. The narrow maintainable solution is escape-then-linkify rendering for safe http/https URLs across the tool-card paths; applying markdown wholesale or injecting regex-built HTML in only one view would be incomplete and riskier.
