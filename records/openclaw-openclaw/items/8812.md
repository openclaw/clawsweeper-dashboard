## Summary

Keep open. Current main and the latest release still render URL-bearing tool output as escaped text/code in Control UI and plain SwiftUI Text in native WebChat, while the directly related implementation at https://github.com/openclaw/openclaw/pull/8880 was closed unmerged.

## What This Changes

Request clickable URL rendering for URLs shown inside Control UI and native WebChat tool output cards.

## Best Possible Solution

Add a shared escape-first http/https URL renderer for all Control UI tool-output surfaces, and add equivalent SwiftUI URL-capable rendering if native WebChat remains in scope.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence. URL-bearing output flows through Lit text/code bindings, getTruncatedPreview, direct card.outputText rendering, and SwiftUI Text, so anchors are not produced on current main.

## Solution Assessment

Yes, the narrow maintainable solution is a shared escape-first URL renderer applied across the tool-output render paths, with native WebChat handled through AttributedString or Link if kept in scope.
