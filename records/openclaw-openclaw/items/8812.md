## Summary

Keep open: current main still renders URL-bearing tool output as escaped/plain text in the affected Control UI card paths and native WebChat, while the linked implementation attempt was closed unmerged.

## What This Changes

Add safe auto-linkification for URLs displayed inside Control UI tool output cards and, if kept in scope, native WebChat tool result cards.

## Best Possible Solution

Implement an escape-first, safe-scheme URL renderer for all Control UI tool-output surfaces, and either add equivalent native WebChat URL rendering or split native follow-up scope deliberately.

## Reproduction Assessment

Yes. Source inspection on current main shows Control UI binds tool-output strings as text in expanded, preview, inline, raw, and grouped paths, and native WebChat renders formatted tool results through plain SwiftUI Text.

## Solution Assessment

Yes. The narrow maintainable solution is to escape arbitrary text first, linkify only safe http/https URLs in the tool-output renderers, and avoid routing tool output through generic markdown wholesale or using unescaped unsafeHTML.
