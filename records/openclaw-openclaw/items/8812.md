## Summary

Keep open: current main still renders URL-bearing tool output as plain text in Control UI and native WebChat, and the prior implementation at https://github.com/openclaw/openclaw/pull/8880 was closed unmerged rather than shipped.

## What This Changes

The issue requests clickable URL rendering for URL-bearing tool outputs in Control UI and native WebChat.

## Best Possible Solution

Add safe URL-aware rendering for all tool-output card surfaces, plus the native SwiftUI equivalent, without changing the tool-output protocol.

## Reproduction Assessment

Yes, by source inspection on current main. Control UI binds tool-output strings as text in expanded, preview, inline, and grouped card paths, while native WebChat uses plain SwiftUI Text for formatted tool results.

## Solution Assessment

Yes. The maintainable fix is an escape-then-linkify renderer for safe http/https URLs across each tool-card path; regex-generated HTML in only one view would be incomplete and risky.
