## Summary

Keep open: current main still renders URL-bearing tool output as plain text in the affected Control UI and native WebChat paths, and the linked implementation attempt was closed unmerged.

## What This Changes

The issue requests clickable http/https URLs inside tool output cards in Control UI and, where applicable, native WebChat.

## Best Possible Solution

Implement an escape-first, safe-scheme URL renderer for all Control UI tool-output surfaces, and either add equivalent native WebChat URL rendering or deliberately split the native scope into a follow-up.

## Reproduction Assessment

Yes for the current behavior from source inspection: current main binds Control UI tool output as Lit text and native WebChat tool results as SwiftUI Text, so URL values are not anchors. This is an enhancement request rather than a broken documented contract.

## Solution Assessment

Yes. The maintainable direction is to escape arbitrary tool output first, linkify only safe http/https URLs in the tool-output renderers, and avoid routing arbitrary tool output through generic markdown or unescaped unsafeHTML.
