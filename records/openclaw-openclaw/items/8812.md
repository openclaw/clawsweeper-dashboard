## Summary

Keep open: current main still renders URL-bearing tool output as escaped/plain text in the Control UI tool-card paths and native WebChat, while the only linked implementation PR was closed unmerged.

## What This Changes

Request auto-linkification of URLs in Control UI and native WebChat tool output cards while preserving safe rendering and JSON/plain-text formatting.

## Best Possible Solution

Add an escape-first, safe-scheme URL renderer across Control UI tool-output surfaces, and add equivalent SwiftUI URL rendering for native WebChat or explicitly split that scope into a tracked follow-up.

## Reproduction Assessment

Yes, source-reproducible on current main d832ad214ce7fd18babc947f228cf192f56fc83b. Control UI binds tool-output strings as text in expanded, preview, inline, and grouped paths, and native WebChat renders formatted tool results through plain SwiftUI Text.

## Solution Assessment

Yes. The narrow maintainable solution is escape-then-linkify safe http/https URLs in the tool-output renderers; applying markdown wholesale or injecting regex-built HTML in only one view would be incomplete and riskier.
