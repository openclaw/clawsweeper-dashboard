Keep open. Current main and the latest release still render URL-bearing tool output as plain text in the relevant Control UI and native WebChat paths, while the two linked implementation PRs are closed unmerged prior art.

## What This Changes

The issue requests automatic hyperlink rendering for URL strings inside Control UI and native WebChat tool output cards.

## Best Possible Solution

Implement safe URL-aware rendering across the Control UI tool-output paths, and add native WebChat URL spans only if maintainers keep that scope in this issue.

## Reproduction Assessment

Yes, at source level: current main renders URL-bearing Control UI tool output through text bindings and native WebChat through plain strings, with no anchor-producing tool-output renderer. I did not run a browser or app repro because this read-only review already has direct source proof.

## Solution Assessment

No, current main has not solved the issue. The best path is a safe URL-aware renderer that preserves formatting and escaping, with a maintainer scope decision for native WebChat.
