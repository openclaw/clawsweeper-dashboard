## Summary

Keep open: current main still renders URL-bearing tool output as text in the Control UI tool-card paths and native WebChat, and the only linked fix PR is closed unmerged.

## What This Changes

The issue asks for URL strings inside tool output cards to become safe clickable links in Control UI and WebChat.

## Best Possible Solution

Add a shared safe URL-aware renderer for Control UI tool output, with equivalent SwiftUI URL rendering if WebChat stays in scope, and cover every tool-output display path with focused tests.

## Reproduction Assessment

Yes. Source inspection on current main shows URL-bearing tool output is passed through plain text render paths in Control UI and SwiftUI Text in WebChat; I did not need a browser run to confirm the render path lacks anchors.

## Solution Assessment

Yes, with guardrails. The maintainable path is a safe URL-aware renderer that preserves JSON/plain-text formatting and avoids raw HTML injection; the simple regex/unsafeHTML approach from the old PR should be treated only as prior art.
