## Summary

Current main still renders URL-bearing tool output as plain text in the relevant Control UI and native WebChat paths, so the feature request remains valid and should stay open.

## What This Changes

Request to render URLs inside tool output cards as safe clickable links in Control UI and WebChat instead of plain escaped text.

## Best Possible Solution

Add a safe URL-aware renderer for tool output, cover each Control UI card state with focused tests, and decide whether native WebChat should receive equivalent Link or AttributedString rendering in the same change.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: URL-bearing tool output is rendered through Control UI tool-card text bindings and native SwiftUI Text, bypassing the markdown renderer that linkifies ordinary chat text. I did not run the UI because this was a read-only review, but the render paths are direct.

## Solution Assessment

Unclear as an exact implementation choice: the requested behavior is useful, but a regex-plus-HTML patch would be risky unless it escapes first and handles URL boundaries. The maintainable direction is a safe structured renderer for Control UI tool-card states, with native WebChat handled deliberately if it stays in scope.
