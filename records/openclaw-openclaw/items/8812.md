## Summary

Keep open: current main still renders URL-bearing tool output as escaped/plain text in Control UI tool-card paths and native WebChat, and the linked implementation PRs are closed unmerged prior art rather than shipped fixes.

## What This Changes

Request to render http(s) URLs inside Control UI and possible native WebChat tool output cards as safe clickable links.

## Best Possible Solution

Implement safe URL-aware rendering across the Control UI tool-output paths, then add or split native WebChat URL spans based on maintainer scope decision.

## Reproduction Assessment

Yes, for the current behavior at source level: current main renders URL-bearing Control UI tool output through text bindings and native WebChat through plain strings. I did not run a browser or app repro because the read-only source path already proves the missing link renderer.

## Solution Assessment

Unclear as a product decision: the Control UI fix shape is clear, but maintainers should decide whether WebChat remains in scope before another automated implementation. The safest implementation is a shared escape-first URL renderer for tool output, with native attributed links only if that scope stays attached.
