## Summary

Keep open: current main still renders URL-bearing tool output as plain text in Control UI tool-card paths and native WebChat, and related PR #8880 was closed unmerged rather than shipped.

## What This Changes

Request auto-linkifies URL values and bare http(s) URLs inside tool output cards in Control UI and applicable WebChat views.

## Best Possible Solution

Implement safe URL-aware rendering for all tool-output card surfaces, plus native SwiftUI URL rendering if WebChat remains in scope, without changing the tool-output protocol.

## Reproduction Assessment

Yes. Source inspection on current main shows Control UI binds tool-output strings as text in expanded, preview, and inline paths, grouped rendering bypasses markdown, and WebChat uses plain SwiftUI Text for tool results.

## Solution Assessment

Yes. The maintainable direction is a shared safe renderer that escapes untrusted output first, linkifies only safe http/https URLs, handles punctuation and whitespace, and covers every tool-card path rather than a one-view regex replacement.
