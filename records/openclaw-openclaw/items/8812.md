## Summary

Keep open. Current main and release v2026.6.5 still display URL-bearing tool results as plain text, and both related implementation PRs closed without merging; the remaining work needs a maintainer decision on whether native WebChat belongs in the same acceptance boundary.

## What This Changes

Add safe clickable HTTP(S) links to URL-bearing tool output in Control UI and possibly native WebChat.

## Best Possible Solution

Use one escape-first, HTTP(S)-only URL-aware text renderer across every accepted Control UI tool-output route, preserving literal formatting and copy text, and explicitly decide whether native WebChat receives equivalent attributed links in the same change.

## Reproduction Assessment

Yes. Current main routes URL-bearing tool results through direct Lit text bindings in Control UI and plain SwiftUI Text in native WebChat, which cannot produce clickable links.

## Solution Assessment

No, the proposed regex-to-HTML replacement is not the safest or most maintainable solution. The implementation should use an escape-first safe-scheme renderer shared across all in-scope output views, with native WebChat scope decided before coding.
