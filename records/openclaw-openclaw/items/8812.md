## Summary

Keep open. Current main still renders URL-bearing tool outputs as escaped text/code in Control UI tool cards and plain SwiftUI Text in native WebChat, while the directly related PR #8880 was closed unmerged.

## What This Changes

Request to render URLs inside Control UI and WebChat tool output cards as clickable links while preserving JSON/plain-text formatting.

## Best Possible Solution

Add a safe shared URL-aware renderer for tool output, reuse it across Control UI tool-card surfaces, and add equivalent native WebChat URL rendering if that surface remains in scope.

## Reproduction Assessment

Yes. Source inspection shows a tool result containing `https://example.com` flows through Lit text/code bindings in Control UI and plain SwiftUI `Text` in native WebChat, so URLs remain non-clickable without needing a live browser run.

## Solution Assessment

Yes for the requested direction, but the implementation should be safer than a raw regex replacement. The narrow maintainable fix is an escape-first, http/https-only renderer with focused coverage across each tool-output surface.
