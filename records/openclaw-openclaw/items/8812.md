## Summary

Keep open: current main and the latest release still render URL-bearing tool output as plain text in Control UI tool cards and native WebChat, while the linked implementation attempts are closed unmerged prior art.

## What This Changes

Request that Control UI and possibly native WebChat render URLs inside tool output cards as safe clickable new-tab links.

## Best Possible Solution

Implement an escape-first URL renderer across Control UI tool-output paths, then confirm or split native WebChat scope before landing.

## Reproduction Assessment

Yes, at source level. Current main renders URL-bearing Control UI tool output through Lit text bindings and native WebChat through SwiftUI Text/plain String formatting, so no clickable link renderer is present.

## Solution Assessment

Unclear as a product decision. The Control UI fix shape is clear, but maintainers should decide whether native WebChat remains in scope before another automated implementation.
