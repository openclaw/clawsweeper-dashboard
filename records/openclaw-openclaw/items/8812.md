## Summary

Keep open: current main still renders URL-bearing tool output as text in the affected Control UI tool-card paths and native WebChat, and the linked implementation attempt at https://github.com/openclaw/openclaw/pull/8880 was closed unmerged.

## What This Changes

Request to render URLs embedded in Control UI and WebChat tool-result output as clickable links while preserving JSON/plain-text formatting.

## Best Possible Solution

Add an escape-first, safe-scheme URL renderer across Control UI tool-output surfaces, and add equivalent SwiftUI URL rendering for native WebChat or explicitly split that scope into a tracked follow-up.

## Reproduction Assessment

Yes. Source inspection on current main shows Control UI binds tool-output strings as text in expanded, preview, inline, raw, and grouped paths, and native WebChat renders formatted tool results through plain SwiftUI Text.

## Solution Assessment

Yes. The narrow maintainable solution is escape-then-linkify safe http/https URLs in the tool-output renderers; applying markdown wholesale or injecting regex-built HTML in one view would be incomplete and riskier.
