## Summary

Keep open: the latest shipped release and current main metadata still show a Discord packaging mismatch. The published `openclaw@2026.5.2` tarball omits `dist/extensions/discord/**`, while the published fallback `@openclaw/discord@2026.5.2` package is source-only TypeScript, so this remains a real packaging regression rather than stale support.

## What This Changes

The issue reports that upgrading to `openclaw@2026.5.2` breaks Discord startup because root-package sidecar metadata points at missing Discord dist files and the fallback Discord npm package lacks compiled JavaScript.

## Best Possible Solution

Make the release artifact contract consistent: preferably publish `@openclaw/discord` with compiled runtime entries and package metadata/tests that prove startup can load it, or intentionally restore Discord compiled artifacts to the root package.

## Reproduction Assessment

Yes for the shipped release: streaming the npm tarballs shows `openclaw@2026.5.2` missing `dist/extensions/discord/**` while `@openclaw/discord@2026.5.2` contains TypeScript sources only. Current-main source still has the same root-package exclusion and source-entry Discord package metadata, but I did not run a gateway boot because this review is read-only.

## Solution Assessment

The report's suggested directions are valid. Given the externalized-plugin metadata on current main, the narrower maintainable fix is to make the official Discord npm plugin ship compiled runtime artifacts and add pack/postpublish checks, unless maintainers choose to roll Discord back into the root package for this release line.

Fix the Discord packaging regression reported at https://github.com/openclaw/openclaw/issues/76405. The observable bug is that `openclaw@2026.5.2` excludes `dist/extensions/discord/**` while sidecar metadata and/or startup paths still expect Discord runtime artifacts, and `@openclaw/discord@2026.5.2` ships source-only TypeScript with `openclaw.extensions: ["./index.ts"]`. Add regression coverage that proves there are no dangling bundled runtime sidecar paths for omitted root-package plugins and that the Discord plugin npm pack contains loadable JS entries, not only TS source.
