## Summary

Keep open. Current main still shows the same Discord distribution mismatch: the core sidecar baseline names Discord dist artifacts that the root package excludes, while the external Discord package metadata does not yet declare built runtime entries.

## What This Changes

The issue asks OpenClaw to fix a Discord packaging regression where the sidecar manifest points at omitted core dist artifacts and the external `@openclaw/discord` fallback is not a usable built runtime package.

## Best Possible Solution

Align the Discord distribution contract so exactly one shipped path is authoritative: either include the required Discord runtime sidecars in the core tarball or publish `@openclaw/discord` with built JS runtime entries declared in package metadata, with release checks covering that path.

## Reproduction Assessment

Yes by source inspection, though not by live package execution in this read-only review. Current main still lists Discord sidecar dist paths while excluding that dist tree, and the external package metadata still lacks built runtime entry declarations.

## Solution Assessment

Unclear between the two suggested implementation choices, but the direction is right. The maintainable fix is to choose the intended Discord distribution path and add package/tarball regression coverage so the sidecar manifest, npm package, and release checks cannot drift again.
