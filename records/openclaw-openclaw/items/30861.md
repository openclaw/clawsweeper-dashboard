## Summary

Keep open. Current main and the latest release show the active-agent breadcrumb and switch feedback path, but the Dashboard header still does not render workspace context even though the Gateway agent list already exposes workspace metadata.

## What This Changes

The issue requests a persistent Dashboard header indicator for the active agent and workspace, with switch feedback to reduce wrong-agent or wrong-workspace actions in multi-agent workflows.

## Best Possible Solution

Add a minimal workspace breadcrumb or badge beside the existing active-agent label using agents.list workspace metadata, without adding a new Gateway contract or bundling the optional dropdown/message-prefix ideas.

## Reproduction Assessment

Yes for the source-level feature gap: current main renders only the active-agent label in the Dashboard header while agents.list already carries workspace metadata. This is not a failing bug repro, but the missing UI path is high-confidence from source inspection.

## Solution Assessment

Yes for a narrowed first pass. Extending the existing DashboardHeader context to include workspace text is the smallest maintainable solution; the optional dropdown, color/icon system, and message-prefix setting should remain separate unless maintainers choose to expand scope.
