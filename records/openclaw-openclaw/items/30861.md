## Summary

Keep open. Current main and v2026.5.20 still render only the active-agent label in the Dashboard header, while the Gateway already exposes agent workspace metadata, so this remains a focused Control UI feature gap rather than stale or superseded work.

## What This Changes

Show persistent active-agent and workspace context in the Control UI Dashboard chat header, with switch feedback, to reduce wrong-agent and wrong-workspace interactions.

## Best Possible Solution

Add a minimal workspace badge or breadcrumb beside the existing active-agent label using agents.list workspace metadata, with responsive truncation and a full-path tooltip, while deferring optional dropdown, color/icon, and message-prefix settings.

## Reproduction Assessment

Yes for a source-level feature-gap check: current main resolves and renders only the active-agent label, while Gateway agent summaries already include workspace metadata. This is not a strict bug reproduction because the requested workspace indicator is a missing UI capability.

## Solution Assessment

Yes for the narrowed first pass. Extending the existing DashboardHeader context to include a workspace label/path from agents.list is the smallest maintainable solution; optional quick-switch and message-prefix settings should remain separate product work.
