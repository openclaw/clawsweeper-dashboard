## Summary

Current main still implements only the active-agent breadcrumb and switch feedback; the persistent workspace indicator requested for multi-agent safety is still missing from the Dashboard header even though `agents.list` already exposes workspace metadata.

## What This Changes

The issue requests a persistent Dashboard header indicator for the active agent and workspace, with synchronized switch feedback, to reduce wrong-agent and wrong-workspace interactions.

## Best Possible Solution

Add a minimal workspace breadcrumb or badge beside the existing active-agent label using `agents.list` workspace metadata, with responsive truncation and tooltip behavior, while deferring the optional dropdown/message-prefix/color work.

## Reproduction Assessment

Yes for a source-level feature-gap check: current main resolves and renders only the active-agent label, while Gateway agent summaries already carry workspace metadata. This is not a strict bug reproduction because the requested workspace indicator is a missing UI capability.

## Solution Assessment

Yes for the narrowed first pass. Extending the existing `DashboardHeader` context to include a workspace label/path from `agents.list` is the smallest maintainable solution; optional quick-switch and message-prefix settings should be separate product work.
