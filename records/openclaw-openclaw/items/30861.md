## Summary

Keep open: current main persistently shows the active agent in the Dashboard header, but it still does not show the active workspace there; the only current workspace path is in a separate chat rail that disappears at narrower widths.

## What This Changes

Request to persistently show the active agent and workspace in the Control UI Dashboard header and keep that context synchronized across agent switches, reloads, and reconnects.

## Best Possible Solution

Extend the existing Dashboard header context to include the active agent workspace, truncate with a full-path tooltip, and prove switch plus reload/reconnect behavior without adding a new setting or target store.

## Reproduction Assessment

Yes, source-level. Current main deterministically returns and renders only agentLabel in the persistent header, while workspace is shown only in a separate rail hidden at max-width 1120px.

## Solution Assessment

Yes, with a narrow scope. Extending the existing header resolver and component from canonical session/agents.list state is more maintainable than adding message prefixes, a new setting, or a second routing state store.
