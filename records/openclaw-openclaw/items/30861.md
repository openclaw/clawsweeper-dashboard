## What This Changes

The issue requests a persistent Control UI chat header context showing the active agent and workspace, synchronized across switches, reloads, and reconnects while staying minimal for single-agent users.

## Best Possible Solution

Add a compact Control UI header context backed by the same selected session and workspace state used for chat routing, with switch/reload/reconnect tests and a minimal single-agent presentation.
