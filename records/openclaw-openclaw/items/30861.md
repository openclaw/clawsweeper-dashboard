## Summary

Current main still does not render the requested workspace context in the Dashboard header. The active-agent label and switch feedback exist, and the Gateway already exposes workspace metadata, so this remains a focused, actionable Control UI enhancement rather than a close candidate.

## What This Changes

Request to persistently show the active agent and workspace in the Dashboard chat/header so multi-agent users can avoid wrong-agent or wrong-workspace interactions.

## Best Possible Solution

Add a minimal workspace badge or breadcrumb beside the existing active-agent label using agents.list workspace metadata, with responsive truncation and a full-path tooltip; defer optional dropdown, color/icon, and message-prefix settings.

## Reproduction Assessment

Yes for a source-level feature-gap check: current main resolves and renders only the active-agent label, while Gateway agent summaries already include workspace metadata. This is not a strict bug reproduction because the requested workspace indicator is a missing UI capability.

## Solution Assessment

Yes for the narrowed first pass. Extending the existing DashboardHeader context to include a workspace label/path from agents.list is the smallest maintainable solution; optional quick-switch and message-prefix settings should remain separate product work.
