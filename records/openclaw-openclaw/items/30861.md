## Summary

Current main partially covers the request with an active-agent breadcrumb and switch toast, but the Dashboard header still does not carry or render the active workspace even though Gateway already exposes it. This remains a focused, queueable Control UI enhancement rather than a close candidate.

## What This Changes

Request to show the active agent and workspace persistently in the Dashboard/chat header, with switch feedback and optional quick-switch or message-prefix affordances for multi-agent safety.

## Best Possible Solution

Add a minimal workspace badge or breadcrumb beside the existing active-agent label using agents.list workspace metadata, with responsive truncation and a full-path tooltip; defer quick-switch, color/icon, and message-prefix settings unless maintainers explicitly expand scope.

## Reproduction Assessment

Yes for a source-level feature-gap check: current main resolves and renders only the active-agent label, while Gateway agent summaries already include workspace metadata. This is a missing UI capability rather than a broken existing contract.

## Solution Assessment

Yes for the narrowed first pass. Extending resolveDashboardHeaderContext and DashboardHeader to include a workspace label/path from agents.list is the smallest maintainable solution; optional dropdown, color/icon, and message-prefix settings should remain separate product work.
