## Best Possible Solution

Add a narrow Control UI header context that derives active agent and workspace from existing session/agent state, updates on switches and reloads, and leaves optional prefix/dropdown behavior for separate product work.

## Solution Assessment

Yes for a narrow first fix: reuse existing `AgentsListResult.workspace` and session workspace root data in the Control UI header. The optional prefix setting and quick-switch dropdown are not needed for the first maintainable solution.
