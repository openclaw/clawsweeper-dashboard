## Summary

Current main and the latest release still render only the active-agent label in the Dashboard header, not the requested workspace context. The Gateway already exposes workspace metadata through `agents.list`, so this remains a narrow, actionable Control UI enhancement rather than a close candidate.

## What This Changes

Request a persistent Dashboard chat/header indicator for the active agent and workspace, with switch feedback to reduce wrong-agent and wrong-workspace mistakes in multi-agent workflows.

## Best Possible Solution

Add a minimal workspace badge or breadcrumb beside the existing active-agent header label using `agents.list` workspace metadata, with responsive truncation and a full-path tooltip; leave optional switcher, color/icon, and message-prefix settings for separate follow-up work.

## Reproduction Assessment

Yes for a source-level feature-gap check: current main resolves and renders only the active-agent label, while Gateway agent summaries already include workspace metadata. This is not a strict bug reproduction because the requested workspace indicator is a missing UI capability.

## Solution Assessment

Yes for the narrowed first pass. Extending the existing DashboardHeader context to include a workspace label/path from `agents.list` is the smallest maintainable solution; optional quick-switch and message-prefix settings should stay separate.
