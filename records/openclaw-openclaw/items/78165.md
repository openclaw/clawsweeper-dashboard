Keep open: current main still routes WhatsApp startup through the plugin path that uses Baileys multi-file auth state without the linked signal-key yielding fix, and the only PR that directly addressed this was closed unmerged.

## Best Possible Solution

Fix the WhatsApp plugin auth/bootstrap path so large Baileys signal-key rehydrate work yields or is offloaded without changing the generic channel manager, while leaving the full auth-store replacement to https://github.com/openclaw/openclaw/issues/9544.
