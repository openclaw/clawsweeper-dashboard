## Summary

This PR should stay open: it addresses a real docs defect that is still present on current main and in the latest release, but the new runnable helper snippet needs a small token-safety guard before merge.

## What This Changes

The PR updates the bundled Notion skill docs and changelog to replace shell-local multiline `curl` examples with `NOTION_API_KEY` setup guidance and a file-backed `python3` helper flow.

## Best Possible Solution

Keep the docs repair, add a small helper guard that rejects paths outside `/v1/...`, and then land it as the focused fix for the linked Notion allowlist issue.

## Reproduction Assessment

Yes. Current main still contains the shell-local `NOTION_KEY=$(cat ...)` and multiline `curl` examples, and the exec-approval docs/tests explain why that command shape misses strict allowlists.

## Solution Assessment

No, not as written. The docs-only direction is the right narrow fix, but the runnable helper should validate the `/v1/...` API path before composing the URL and attaching the bearer token.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:
