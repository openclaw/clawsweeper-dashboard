## Summary

Keep open. The PR fixes the right docs problem and supplies sufficient docs-only behavior proof, but the current helper snippet can send the Notion bearer token to a malformed non-Notion host because it builds the URL before validating the API path.

## What This Changes

The PR updates the bundled Notion skill docs and changelog to replace shell-local multiline `curl` examples with `NOTION_API_KEY` config guidance and a file-backed `python3` helper flow.

## Best Possible Solution

Land the docs-only repair after adding a small `/v1/` path guard before constructing the helper request and attaching the Authorization header.

## Reproduction Assessment

Yes. Current main contains the `NOTION_KEY=$(cat ...)` setup and multiline `curl` examples, and current exec docs/tests show those shell constructs are rejected in strict allowlist paths before any Notion API request is needed.

## Solution Assessment

No, not as written. The docs-only direction is the narrowest maintainable fix, but the helper should reject paths outside `/v1/...` before it builds the URL and sends the bearer token.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
