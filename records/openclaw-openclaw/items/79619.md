## Summary

Keep open. The PR is on the right docs surface and has sufficient docs proof, but the latest head still adds a runnable helper that can attach `NOTION_API_KEY` to a malformed host when `api_path` is not restricted.

## What This Changes

The PR rewrites `skills/notion/SKILL.md` to replace shell-wrapped `curl` examples with `NOTION_API_KEY` setup guidance and a file-backed `python3` helper, plus a `CHANGELOG.md` entry.

## Best Possible Solution

Land the docs repair after adding a small guard that rejects helper paths outside `/v1/...`, without changing runtime exec policy, Notion API behavior, or adding a bundled helper file.

## Reproduction Assessment

Yes. Current main contains the `NOTION_KEY=$(cat ...)` plus multi-line `curl` docs pattern, and the exec approval docs/tests show why that command shape fails strict allowlist parsing before any Notion API call is needed.

## Solution Assessment

No, not as written. The docs-only direction is the narrowest maintainable fix, but the runnable helper should validate `api_path` as a `/v1/...` relative Notion API path before constructing the request with the bearer token.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Validate the helper path before adding the token:** `skills/notion/SKILL.md:71`
  - body: The helper appends arbitrary `api_path` text to `https://api.notion.com` and then adds `Authorization: Bearer {api_key}`. A copied helper called with a malformed value such as `.evil.example/v1/search` would target a different host with the Notion token, so reject paths that do not start with `/v1/` before creating the request.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff is docs-only, but the added runnable helper handles a secret and needs path validation before it attaches the Notion token.

  - body: The helper concatenates an unchecked CLI path into the request URL and sends the `Authorization` header. If a user copies it and passes a malformed non-absolute path, the token can be sent to a non-Notion host.
