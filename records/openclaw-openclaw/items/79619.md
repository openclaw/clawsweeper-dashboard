## Summary

Keep open. The PR is a focused docs repair with sufficient after-fix proof, but the current head still adds a runnable helper that concatenates an unvalidated API path before attaching the Notion bearer token.

## What This Changes

The PR updates the bundled Notion skill docs and changelog to replace shell-local multiline `curl` examples with `NOTION_API_KEY` config guidance and a file-backed `python3` helper flow.

## Best Possible Solution

Land the docs repair after adding a small `/v1/...` path guard to the helper snippet, without changing runtime exec policy or Notion API behavior.

## Reproduction Assessment

Yes. Current main contains the allowlist-hostile `NOTION_KEY=$(cat ...)` plus multiline `curl` docs pattern, and current exec approval docs/tests cover why those shell constructs fail under strict allowlist paths.

## Solution Assessment

No, not as written. The docs-only direction is the narrowest maintainable fix, but the helper should reject paths that do not start with `/v1/` before constructing the request and attaching the token.
