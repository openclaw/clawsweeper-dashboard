## Summary

Keep open: the docs repair is correctly scoped and the supplied proof is enough for a docs-only change, but current head still adds a runnable helper that constructs a token-authenticated URL from an unchecked CLI path.

## What This Changes

The PR updates the bundled Notion skill markdown and changelog to replace shell-local multiline `curl` examples with `NOTION_API_KEY` config guidance and a file-backed `python3` helper flow.

## Best Possible Solution

Land the docs repair after adding a small guard that rejects `api_path` values outside `/v1/...` before constructing the request, without changing runtime exec policy or Notion API behavior.

## Reproduction Assessment

Yes. Current main contains the `NOTION_KEY=$(cat ...)` plus multiline `curl` docs pattern, and the linked report describes that exact shape failing under strict allowlist settings; no live Notion workspace is needed to verify the docs defect.

## Solution Assessment

No, not as written. The docs-only direction is the narrowest maintainable fix, but the helper should validate that `api_path` is a `/v1/...` relative Notion API path before attaching the token.
