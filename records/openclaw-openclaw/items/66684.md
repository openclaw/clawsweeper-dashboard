## Summary

Keep open. The original root `Dockerfile` Bun curl-to-bash hit was fixed and shipped by https://github.com/openclaw/openclaw/pull/74359, but current `main` and `v2026.5.7` still build the optional sandbox-common image by executing remote Bun and Homebrew installer scripts; current `main` also has a copyable Bun curl-to-bash Docker docs example. The cited credential-pattern examples look stale or benign from the checked lines.

## What This Changes

The issue asks maintainers to triage static security-scan hits for remote script execution and credential-pattern matches in repository code and docs.

## Best Possible Solution

Keep this open and narrow it to Docker supply-chain hardening: replace or explicitly approve the sandbox-common Bun/Homebrew installer sources, update the current Docker docs example, and treat the credential-pattern hits as false positives unless fuller scan evidence appears.

## Reproduction Assessment

Yes for the actionable remote-installer part: source inspection on current main and `v2026.5.7` shows sandbox-common still executes remote Bun and Homebrew installer scripts by default. No high-confidence reproduction exists for the credential-pattern claims because the cited AGENTS line is stale and the CHANGELOG/iMessage snippets are benign text examples.

## Solution Assessment

No patch is proposed in this issue. The merged root Dockerfile fix was the right narrow solution for the originally cited file, but the remaining sandbox-common and docs surfaces need a maintainer-approved source, digest, or explicit-risk policy before implementation.
