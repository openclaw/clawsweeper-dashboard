## Summary

This PR should stay open: the identity-line config direction is plausible, but the submitted patch leaves the new public config fields blocked by the gateway config mutation allowlist and still lacks after-fix real behavior proof beyond tests.

## What This Changes

The PR adds default and per-agent `identityLine`/`identityMode` config fields, resolves them into the main agent system prompt, centralizes the default identity line, and adds resolver and prompt tests.

## Best Possible Solution

If maintainers accept the contract, land a documented identity-line config surface that includes schema/types, default and per-agent resolution, prompt/cache wiring, gateway config mutation allowlist coverage, and explicit subagent scoping.

## Reproduction Assessment

Yes for the PR blocker: source inspection shows gateway `config.apply` and `config.patch` reject changed paths not in `ALLOWED_GATEWAY_CONFIG_PATHS`, and the patch adds identity config paths without extending that list. For the feature behavior itself, the PR does not include a real patched OpenClaw run.

## Solution Assessment

No as submitted. The main-agent config approach is plausible and the subagent scope has been clarified, but the patch should update the gateway mutation guard and still needs contributor real behavior proof plus maintainer acceptance of the public prompt contract.
