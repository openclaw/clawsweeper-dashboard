## Summary

Keep open: this is a plausible feature PR for the open identity-line request, but the patch still has a concrete gateway allowlist bug, lacks after-fix real behavior proof, and needs maintainer acceptance of the new public prompt config contract.

## What This Changes

The PR adds default and per-agent `identityLine`/`identityMode` config, resolves it into the main agent system prompt and cache key, centralizes the default identity text, and adds resolver/prompt tests.

## Best Possible Solution

If maintainers accept the direction, land one identity-line contract covering schema/types/docs, configured prompt resolution, prompt cache behavior, gateway config mutation allowlist coverage, focused tests, and explicit subagent scope.

## Reproduction Assessment

Yes for the PR blocker: source inspection shows `gateway config.patch` and `config.apply` reject changed paths not in `ALLOWED_GATEWAY_CONFIG_PATHS`, and the PR adds identity config paths without extending that list. The feature's real runtime behavior still lacks after-fix OpenClaw proof.

## Solution Assessment

No as submitted. The main-agent config approach is plausible and the subagent scope has been clarified in discussion, but the patch needs gateway allowlist coverage, real behavior proof, docs/config alignment, and maintainer acceptance of the public prompt contract.
