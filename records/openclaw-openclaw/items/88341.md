## Label Justifications

- `P1`: The issue reports live Discord channel output exposing internal tool-progress text during active agent runs, affecting a real channel workflow.
- `impact:security`: The visible lines can expose internal tool commands, local paths, and runtime scaffolding to shared Discord recipients.

## Summary

Keep open: current main still has Discord paths that intentionally preserve raw tool-progress text in progress drafts and tool replies, while the report shows those progress-shaped lines becoming visible Discord messages; the remaining work needs a narrower fix than the closed sanitizer PR because some visible progress is documented behavior.

## What This Changes

The issue requests fixing Discord outbound/progress-draft delivery so internal tool-progress and session-history trace lines do not appear as visible channel messages during active agent runs.

## Best Possible Solution

Isolate the real leaking Discord path, then add a narrow sanitizer or progress-draft rule that suppresses internal trace shapes such as `Session History` without removing supported compact progress rows or final reply delivery.

## Reproduction Assessment

No live high-confidence Discord reproduction was run. Source inspection and current mocked tests show raw progress text is still rendered into Discord draft messages and preserved for tool replies, matching the reported leak class enough to keep this open.

## Solution Assessment

Unclear as an implementation direction: the broad sanitizer attempted in https://github.com/openclaw/openclaw/pull/88494 was closed as too broad. The best fix is narrower and needs maintainer confirmation of which progress shapes are supported versus internal-only.
