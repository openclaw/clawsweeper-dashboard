## Summary

This is a focused and technically sound fail-closed mitigation for backend/auth substitution, but it intentionally turns `/btw` into a hard error for CLI-runtime aliases, does not provide the same-runtime behavior requested by the linked issue, and lacks the required real configured-session proof. The protected `maintainer` label and compatibility policy require explicit maintainer approval before merge.

## What This Changes

The PR blocks `/btw` direct-provider fallback for models routed through CLI runtime aliases and adds regression coverage that direct auth and streaming are not invoked.

## Best Possible Solution

Make side-question execution an explicit harness capability: use an isolated same-runtime thread/turn when supported, fail closed when the selected harness cannot support it, and never silently cross to different provider credentials or transport.

## Reproduction Assessment

Yes. The linked `agentRuntime.id: claude-cli` configuration and current fallback path provide a high-confidence source reproduction of the backend divergence, and the new test exercises the fail-closed branch; no live reproduction was performed in this review.

## Solution Assessment

Yes as a bounded safety mitigation, but not as the complete resolution of the linked issue. Failing closed is preferable to silently switching backend/auth, while the durable solution is a harness-owned same-runtime side-question capability.
