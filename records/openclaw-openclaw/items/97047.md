## What This Changes

The issue reports that DeepSeek V4 transcript entries with nonzero token usage and `usage.cost.total: 0` make Control UI Spend show zero instead of estimating cost from configured pricing.

## Best Possible Solution

Land one focused fix that treats known-priced, nonzero-token recorded-zero usage as estimateable, clears stale zero breakdowns, refreshes durable usage-cost caches, and covers both Spend summaries and session logs with regression tests.

## Reproduction Assessment

Yes for a source-level reproduction: a transcript entry with DeepSeek V4 provider/model, nonzero usage, and `usage.cost.total: 0` is parsed as a real zero and skips flat-pricing estimation on current main. I did not run a live DeepSeek API call in this read-only review.

## Solution Assessment

Yes on the fix boundary: the maintainable repair belongs in session cost normalization and log loading, not in the Control UI display layer or a DeepSeek-only UI workaround. The linked PRs still need maintainer selection so the landed fix covers stale breakdown and cache behavior consistently.
