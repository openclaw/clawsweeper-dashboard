## What This Changes

Fix Slack upload-file so agent-scoped mediaLocalRoots/mediaReadFile survive the bundled custom invoke path.

## Best Possible Solution

Yes. A source-level reproduction is to dispatch Slack upload-file through createSlackActions with options.invoke and media roots on the ChannelMessageActionContext; current main forwards only ctx.toolContext to the custom invoke.

## Solution Assessment

Yes. The narrow maintainable fix is to augment the Slack action context once in createSlackActions before both invoke paths, while leaving the local-media guard intact.
