## Summary

Keep open: current main still invalidates `claude-cli` reuse with `reason=system-prompt`, logs the reset, and reseeds a fresh CLI prompt, but it does not surface the reset to the active conversation or expose the requested structured reset event.

## What This Changes

The issue requests an in-band and queryable notice when a `claude-cli` session is reset after a system-prompt change and transcript replay continues in a new CLI session.

## Best Possible Solution

Define a typed CLI-session-reset lifecycle event that preserves transcript ordering, gives active agents and channels a concise reset explanation, and exposes safe correlation metadata for diagnostics.

## Reproduction Assessment

Yes, at source level. Current main invalidates CLI reuse with `reason=system-prompt`, logs that reset, and can build a reseed prompt, while the searched paths do not emit the requested active-conversation or queryable reset notice.

## Solution Assessment

Unclear as an implementation shape. The requested direction is valid, but maintainers should choose the durable event, transcript, channel, and metadata-redaction contract before a fix PR hard-codes one surface.
