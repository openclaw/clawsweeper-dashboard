## Summary

Keep open. Current main already records subagent timeouts and can surface a partial-progress fallback, but the requested proactive checkpointing, named-deliverable verification, and resume/follow-up behavior is not implemented and needs maintainer design rather than cleanup closure.

## What This Changes

Request improved handling for long subagent runs that time out, including checkpoints, explicit partial-progress state, deliverable verification, and resume support.

## Best Possible Solution

Define a bounded subagent completion contract that carries checkpoint/progress metadata, optional named deliverable expectations, verification results, and resume guidance through the subagent registry, announce flow, and task ledger.

## Reproduction Assessment

No high-confidence live reproduction was established. The provided SQLite examples are concrete, and source inspection confirms the missing deliverable-verification/checkpoint/resume mechanisms, but I did not run a long subagent timeout scenario on current main.

## Solution Assessment

Unclear. The suggested direction is reasonable, but the maintainable fix should start with an explicit runtime contract for checkpoint and deliverable metadata before adding automatic resume or spawn-time warnings.
