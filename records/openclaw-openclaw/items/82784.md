## Summary

Keep open. Current main has post-timeout partial-progress reporting, but it does not implement proactive checkpointing, named-deliverable verification, or a resume/follow-up contract for long subagent runs.

## What This Changes

The issue requests runtime support for long-running subagent checkpointing, named-deliverable verification, explicit partial/resume state, and timeout-risk warnings.

## Best Possible Solution

Define a bounded subagent/task completion contract that carries checkpoint metadata, optional named deliverable expectations, verification results, and resume guidance through registry, announce, and task-ledger surfaces.

## Reproduction Assessment

No high-confidence live reproduction was established. The provided local run rows are concrete, and source inspection shows current main lacks the requested deliverable-verification, checkpoint, and resume mechanisms.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable solution should start with an explicit runtime contract rather than ad hoc prompt wording, timeout bumps, or isolated status text changes.
