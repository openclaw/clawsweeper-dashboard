## Summary

Keep open. Current main still only records timeout status and a partial-progress fallback; the broader checkpointing, named-deliverable verification, and resume guidance requested here are not implemented and need maintainer design rather than cleanup closure.

## What This Changes

The issue requests more reliable long-running subagent handling with proactive checkpoints, named artifact verification, clearer timeout metadata, and resume/follow-up guidance.

## Best Possible Solution

Define a bounded subagent completion contract that carries checkpoint metadata, optional deliverable expectations, verification results, and resume guidance through the registry, announce flow, and task ledger.
