## What This Changes

Request to let the sandbox browser connect to externally managed CDP/noVNC endpoints instead of requiring OpenClaw to create and manage a Docker browser container.

## Best Possible Solution

Define a first-class opt-in external sandbox-browser mode or browser-capable backend contract with validated endpoints, explicit trust-boundary docs, Docker-default regression coverage, and real containerized proof.

## Solution Assessment

Unclear: the minimal endpoint config is plausible, but maintainers should first choose between explicit endpoint fields and a browser-capable backend contract with a documented trust model.
