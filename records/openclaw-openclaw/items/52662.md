## What This Changes

The issue requests external CDP/noVNC browser sandbox endpoints so browser automation can run without a Docker daemon.

## Best Possible Solution

Design a reviewed sandbox-browser backend or endpoint contract that supports externally managed browsers while preserving the current relay auth, SSRF, noVNC, and explicit operator-control guarantees.

## Reproduction Assessment

Yes, from source inspection for the central gap: current main still has a Docker-specific sandbox-browser lifecycle and rejects browser support for backends without browser capability. I did not run a live Kubernetes, Compose, or sidecar deployment repro in this read-only review.

## Solution Assessment

Unclear. The proposed external endpoint fields are plausible, but the best fix needs maintainer design around backend ownership, CDP relay auth, SSRF/noVNC policy, and config contract rather than an automatic config shortcut.
