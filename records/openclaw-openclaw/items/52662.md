## Summary

Keep open. Current main still has no external CDP/noVNC mode for sandbox browsers: the runtime path is Docker-managed, the strict browser sandbox config lacks the requested endpoint fields, and non-Docker backends still reject browser sandboxes.

## What This Changes

The issue requests an opt-in external CDP/noVNC browser sandbox mode so non-Docker sandbox deployments can use browser automation without a Docker daemon.

## Best Possible Solution

Define an explicit opt-in external sandbox-browser mode or backend capability that validates endpoints, preserves Docker as the default, documents the security contract, and covers config parsing plus runtime bridge/noVNC behavior.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a broken existing contract. Static current-main source and docs give high confidence that the requested external sandbox-browser endpoint contract is absent.

## Solution Assessment

Unclear. The proposed minimal config could unblock containerized deployments, but the maintainable solution needs sandbox/browser owner judgment because it crosses backend capability design and external CDP/noVNC security policy.
