## Summary

Keep open. Current main still has no external CDP/noVNC sandbox-browser mode, and the existing code/docs continue to make Docker the only browser-capable sandbox backend while SSH/OpenShell remain browser-unsupported.

## What This Changes

Request external CDP/noVNC endpoint support for sandboxed browser automation so deployments can use a browser managed outside Docker.

## Best Possible Solution

Define an explicit opt-in external sandbox-browser mode or backend capability that validates endpoint URLs, preserves the current Docker default, documents noVNC/CDP security constraints, and covers both config parsing and runtime bridge behavior.

## Reproduction Assessment

Not applicable. This is a feature/config request, and current source plus docs are enough to show the requested external sandbox-browser endpoint contract is not present on main.

## Solution Assessment

Unclear. The proposed minimal config could unblock users, but the maintainable solution should be chosen by sandbox/browser owners because it crosses backend capability design and external browser/noVNC security policy.
