## Summary

Keep open. Current main still treats sandbox browser lifecycle as Docker-managed, rejects browser support on non-browser-capable sandbox backends, and has no external CDP/noVNC endpoint config; this is a concrete feature request that needs maintainer product and security design rather than cleanup closure.

## What This Changes

The issue requests configurable externally managed browser CDP/noVNC endpoints so sandboxed browser automation can run without OpenClaw managing a Docker browser container.

## Best Possible Solution

Design a backend-agnostic sandbox browser capability or a tightly guarded external endpoint mode, then update config types/schema/help/docs and focused sandbox browser tests while preserving Docker as the default.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that the requested external endpoint mode is not implemented on current main.

## Solution Assessment

Unclear. The proposed minimal endpoint fields are plausible, but maintainers should decide whether the durable fix is direct external endpoint config or a backend-owned browser capability contract.
