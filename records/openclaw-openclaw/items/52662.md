## Summary

Keep open. Current main and v2026.5.22 still keep sandbox browser lifecycle Docker-managed, non-Docker sandbox backends still reject browser sandboxes, and the existing host remote-CDP profile support does not provide the requested sandbox-browser/noVNC backend mode. The remaining work is a security-sensitive feature and config/API decision, not a shipped fix or duplicate.

## What This Changes

Request support for an opt-in externally managed sandbox browser via configured CDP/noVNC endpoints so browser automation can run without Docker-managed browser lifecycle.

## Best Possible Solution

Define a first-class opt-in external sandbox-browser mode or browser-capable backend contract with validated endpoints, explicit trust-boundary docs, Docker-default regression coverage, and real containerized proof.

## Reproduction Assessment

Not applicable: this is a feature/config request rather than broken behavior under an existing contract. Source, schema, docs, and release inspection give high confidence that the requested external sandbox-browser endpoint mode is absent.

## Solution Assessment

Unclear: the minimal endpoint config is plausible, but maintainers should first choose between explicit external endpoint fields and a browser-capable backend contract with a documented trust model.
