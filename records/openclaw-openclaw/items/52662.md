## Summary

Keep open: current main still does not provide an externally managed sandbox browser mode, and the remaining work is a security-sensitive feature/design change rather than conservative cleanup.

## What This Changes

The issue requests configurable external CDP/noVNC sandbox browser endpoints so non-Docker sandbox backends can use externally managed browser services.

## Best Possible Solution

Add an explicit, validated sandbox browser seam that supports externally managed CDP/noVNC only when configured, preserves Docker defaults, updates config/schema/help/docs, and covers endpoint/auth/SSRF/noVNC behavior with focused regression tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature request. Source inspection gives high-confidence verification that current main has no external sandbox browser endpoint config and still routes sandbox browser lifecycle through Docker-only paths.

## Solution Assessment

Unclear. The proposed endpoint fields are a plausible narrow unblocker, but the maintainable solution should first define validated endpoint semantics and the backend/browser capability boundary instead of adding a raw passthrough.
