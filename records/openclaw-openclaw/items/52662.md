## Summary

Keep open. Current main still has no external CDP/noVNC mode for sandbox browsers: the sandbox browser lifecycle remains Docker-managed, non-Docker sandbox backends still reject browser support, and the existing host-level remote CDP profile support does not satisfy sandbox-target browser requests.

## What This Changes

Request to add an opt-in sandbox-browser mode that connects to externally managed CDP/noVNC endpoints instead of requiring Docker to create and manage the browser container.

## Best Possible Solution

Define an explicit opt-in external sandbox-browser mode or backend capability that validates CDP/noVNC endpoints, preserves Docker as the default, documents the security contract, and covers config parsing plus runtime bridge behavior.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a broken existing contract. Static current-main source and docs give high confidence that the requested external sandbox-browser endpoint contract is absent.

## Solution Assessment

Unclear. The proposed minimal config may unblock containerized deployments, but the best maintainable solution needs sandbox/browser owner judgment because it crosses backend capability design and external CDP/noVNC security policy.
