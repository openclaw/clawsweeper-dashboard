## Summary

Keep open. Current main and v2026.5.22 still keep sandbox browser lifecycle Docker-managed, non-Docker sandbox backends still reject browser sandboxes, and ordinary remote CDP profile support does not provide the requested sandbox-browser/noVNC external backend mode.

## What This Changes

The issue requests a browser sandbox mode that can attach to externally managed CDP/noVNC endpoints instead of requiring OpenClaw to manage a Docker browser container.

## Best Possible Solution

Design and land a first-class external sandbox-browser or backend capability with explicit endpoint config, auth/SSRF/noVNC semantics, docs, and regression tests while preserving Docker as the default.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source, docs, and release inspection give high confidence that the requested external sandbox-browser endpoint mode is absent on current main.

## Solution Assessment

Unclear as proposed: the central problem is valid, but the two-field endpoint shape needs maintainer product and security design before implementation. A named external browser backend or sandbox-browser capability contract may be safer than a narrow Docker bypass.
