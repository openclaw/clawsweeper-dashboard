## Summary

Keep open. Current main and the latest release still have Docker-managed sandbox browser lifecycle only, no external CDP/noVNC sandbox-browser config contract, and non-Docker backends still reject browser sandboxes; the remaining work needs maintainer product and security review.

## What This Changes

Request external CDP/noVNC endpoint support so sandbox browser automation can use an externally managed browser instead of requiring Docker lifecycle management.

## Best Possible Solution

Define an explicit opt-in external sandbox-browser mode or backend capability that validates endpoints, preserves Docker as the default, documents the trust boundary, and covers config parsing, runtime bridge behavior, and upgrade-safe tests.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a broken existing contract. Static current-main and v2026.5.19 source checks show the requested external sandbox-browser endpoint contract is absent.

## Solution Assessment

Unclear. The minimal endpoint config could unblock containerized deployments, but the safest maintainable solution needs maintainer choice between a narrow external mode and a fuller backend browser capability with explicit security policy.
