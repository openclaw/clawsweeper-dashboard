## Summary

Keep open. Current main still has no external CDP/noVNC mode for sandbox browsers: the runtime path remains Docker-managed, the public sandbox browser config/schema lacks the requested endpoint fields, and non-Docker backends still reject browser sandboxes.

## What This Changes

Request opt-in external CDP/noVNC endpoints so sandbox browser automation can be managed outside Docker for containerized or non-Docker sandbox deployments.

## Best Possible Solution

Define an explicit opt-in external sandbox-browser mode or backend capability that validates endpoints, preserves Docker as the default, documents the security contract, and covers config parsing plus runtime bridge behavior.

Not applicable: this is a feature/config request, not a broken existing contract. Static current-main source and docs give high confidence that the requested external sandbox-browser endpoint contract is absent.

## Solution Assessment

Unclear. The proposed minimal config could unblock containerized deployments, but the maintainable solution needs sandbox/browser owner judgment because it crosses backend capability design and external CDP/noVNC security policy.
