## Summary

Keep open. Current main still requires Docker-managed sandbox browser lifecycle and has no external CDP/noVNC endpoint contract, so this is a real feature/design request rather than cleanup-closeable stale work.

## What This Changes

The issue requests an external CDP/noVNC endpoint mode so sandbox browser automation can run without a Docker-managed browser container.

## Best Possible Solution

Design a guarded sandbox browser endpoint/backend contract, then update config types/schema/help/docs and focused sandbox browser tests while preserving Docker as the default path.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing contract. Source inspection gives high confidence that current main lacks the requested external endpoint mode and still uses Docker-managed sandbox browser startup.

## Solution Assessment

Unclear. The requested minimal endpoint config is plausible, but maintainers should first choose between direct external endpoint fields and a backend-owned browser capability contract because this touches sandbox, browser-control, CDP, and noVNC security boundaries.
