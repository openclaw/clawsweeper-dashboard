## Summary

Keep open. Current main still lacks the non-interactive `paste-token --token` path and `invalid_type` valid-type diagnostics, while this PR appears functionally correct; the remaining blocker is maintainer/secops acceptance of an argv-based provider-token input path.

## What This Changes

The PR adds `models auth paste-token --token <value>`, non-interactive token validation, `validTypes` in invalid auth-profile warnings, and matching docs, changelog, and tests.

## Best Possible Solution

Land this PR or an equivalent maintainer patch if argv-token input is acceptable; otherwise choose a safer non-interactive credential input design while preserving the `validTypes` diagnostic and regression coverage.

## Reproduction Assessment

Yes. Source inspection of current main shows no `--token` option, an always-interactive token prompt, and invalid auth-profile warnings that omit `validTypes`; I did not run the CLI in this read-only checkout.

## Solution Assessment

Yes, subject to maintainer policy approval. The PR is narrow, keeps the interactive path intact, validates the new non-interactive path, and documents argv exposure; stdin or secret-reference input would be safer if argv tokens are not acceptable.
