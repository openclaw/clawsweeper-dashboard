## Summary

Keep open for maintainer/security review: current main still lacks the non-interactive `paste-token` path and `validTypes` warning detail, and this PR implements the focused behavior with proof, but the argv secret-input policy needs explicit acceptance before merge.

## What This Changes

This PR adds `models auth paste-token --token <value>`, validates scripted token input, adds `validTypes` to invalid auth-profile warnings, and updates docs, changelog, and focused tests.

## Best Possible Solution

Land this PR or an equivalent maintainer patch if argv-token input is acceptable; otherwise use a safer non-interactive credential design while preserving the `validTypes` diagnostic and regression coverage.

## Reproduction Assessment

Yes. Source inspection of current main shows no `--token` option, an always-interactive token prompt, and invalid auth-profile warnings that omit `validTypes`; I did not run the CLI in this read-only checkout.

## Solution Assessment

Yes, subject to maintainer/security policy approval. The implementation is narrow, keeps the interactive path intact, validates the new non-interactive path, and documents argv exposure; stdin or secret-reference input would be safer if argv tokens are not acceptable.
