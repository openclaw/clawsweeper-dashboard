## Summary

Keep open: current main still lacks the non-interactive `paste-token --token` path and the `validTypes` diagnostic, and this PR now addresses the earlier validation feedback with sufficient terminal proof. ## What This Changes

The PR adds `models auth paste-token --token <value>`, threads it into token persistence with validation, adds `validTypes` to invalid auth-profile warnings, and updates docs, changelog, and focused tests.

## Best Possible Solution

Land this PR or an equivalent maintainer patch if argv-token input is acceptable; otherwise use a safer non-argv non-interactive credential path while preserving the `validTypes` diagnostic and regression coverage.

## Reproduction Assessment

Yes. Source inspection of current main shows no `--token` option, an always-interactive token prompt, and invalid auth-profile warnings without `validTypes`; I did not run the CLI because this review is read-only.

## Solution Assessment

Yes, subject to maintainer/security policy approval. The implementation is narrow and tested, but stdin, SecretRef, or token-file input would be safer if argv secrets are not acceptable.
