## Summary

Keep open; current main still hardcodes the phone realtime barge-in detector values and exposes no voice-call config/docs surface for these knobs, while the related prior issues cover earlier interruption bugs rather than detector configurability.

## What This Changes

Request to expose voice-call phone realtime barge-in detector sensitivity as plugin config instead of fixed runtime constants.

## Best Possible Solution

Add a plugin-owned `realtime.bargeIn` config block with backward-compatible defaults, pass the values into `RealtimeMulawSpeechStartDetector`, and document/test the noise-vs-responsiveness tradeoff.

## Reproduction Assessment

Yes for the config gap: source inspection shows fixed detector values and no accepted voice-call config fields for them. No live phone call was run to reproduce the reported false-positive threshold.

## Solution Assessment

Yes directionally: plugin-owned realtime config is the right boundary for this tuning. Maintainers still need to approve the public field shape and defaults before a fix PR.
