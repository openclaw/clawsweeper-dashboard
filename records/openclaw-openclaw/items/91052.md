## Summary

Keep open; current main and v2026.6.8 still hardcode the voice-call realtime local MuLaw speech detector and expose no requested barge-in config knobs. The related open provider-VAD PR partially overlaps but does not implement this config request and is not merged.

## What This Changes

Request to expose voice-call realtime barge-in sensitivity and enablement settings in plugin config instead of relying on hardcoded local MuLaw speech-start detector values.

## Best Possible Solution

Add a plugin-owned realtime barge-in config block with backward-compatible defaults, bounded validation, handler wiring, focused tests, and docs after maintainers decide how it should relate to provider-VAD handling.

## Reproduction Assessment

Yes for the source-level config gap: current main and v2026.6.8 hardcode the detector values and the strict realtime config schema has no tuning fields. No live phone call was run to reproduce the reporter's exact production false-positive threshold.

## Solution Assessment

Yes directionally: the right boundary is plugin-owned voice-call realtime config, not core config. The exact public fields, bounds, defaults, and interaction with provider speech-start handling need maintainer product approval first.
