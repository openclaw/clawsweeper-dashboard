## Summary

Keep open; current main and the latest release still hardcode the phone realtime local barge-in detector values, and the voice-call schema, plugin metadata, docs, and tests do not expose the requested tuning knobs. The related older barge-in reports addressed earlier interruption bugs, but they do not supersede this configurable-detector request.

## What This Changes

Request to expose voice-call phone realtime barge-in detector sensitivity as plugin configuration.

## Best Possible Solution

Add a plugin-owned `realtime.bargeIn` config block with backward-compatible defaults, validation bounds, metadata/docs, and focused tests, then pass those values into `RealtimeMulawSpeechStartDetector`.

## Reproduction Assessment

Yes for the config gap: source inspection on current main shows fixed detector values and no accepted voice-call config/docs surface for them. No live phone-call run was performed to reproduce the reporter's exact noise threshold.

## Solution Assessment

Yes directionally: a plugin-owned realtime barge-in config surface is the right boundary for this tuning. The exact public fields, defaults, and validation bounds still need maintainer product approval before implementation.
