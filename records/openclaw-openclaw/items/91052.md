## Summary

Keep open; current main and the latest release still hardcode the phone realtime local speech-start detector and expose no voice-call config or docs surface for the requested tuning knobs. The request is source-reproducible and useful, but it expands the public plugin config surface, so it needs maintainer product approval before implementation.

## What This Changes

Request to make the phone voice-call realtime barge-in detector configurable instead of hardcoding RMS and consecutive-chunk sensitivity.

## Best Possible Solution

Add a plugin-owned realtime barge-in config block with backward-compatible defaults, bounded numeric validation, handler wiring into RealtimeMulawSpeechStartDetector, focused tests, and user docs explaining the noise-versus-interruption tradeoff.

## Reproduction Assessment

Yes for the source-level config gap: current main has fixed detector values and a strict realtime config schema with no barge-in tuning fields. No live phone call was run to reproduce the reporter's exact production false-positive threshold.

## Solution Assessment

Yes directionally: a plugin-owned realtime barge-in config surface is the maintainable boundary. The exact public field shape, defaults, and validation bounds still need maintainer approval before implementation.
