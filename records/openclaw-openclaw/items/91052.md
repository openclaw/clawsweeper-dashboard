## Summary

Keep open; current main and latest release still hardcode the phone realtime barge-in detector values, and no accepted voice-call config/docs surface exists for these knobs.

## What This Changes

Request to expose configurable voice-call realtime local barge-in sensitivity for RMS threshold, loud/quiet chunk counts, and optional barge-in enablement.

## Best Possible Solution

Add a plugin-owned `realtime.bargeIn` config block with bounded values, backward-compatible defaults, handler wiring, focused tests, and docs explaining responsiveness versus noise/echo tolerance.

## Reproduction Assessment

Yes for the configurability gap: source inspection shows fixed detector values and no accepted voice-call config fields for them. No live phone call was run to reproduce the reporter's exact false-positive threshold.

## Solution Assessment

Yes directionally: plugin-owned realtime config is the right boundary for deployment-specific telephony tuning. Maintainers still need to approve the public field shape, defaults, and validation policy before a fix PR.
