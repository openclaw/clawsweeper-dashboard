## What This Changes

The PR adds a gateway startup warning when host approvals clamp the requested global exec security, extracts exec target resolution helpers, adds focused tests, and updates troubleshooting docs to point at `openclaw exec-policy`.

## Best Possible Solution

Keep the startup warning, but make the host-ownership predicate match runtime session semantics so gateway-owned main sessions warn while node-owned and always-sandboxed scopes stay quiet.

## Reproduction Assessment

Yes. Source inspection shows the PR suppresses the warning for `non-main`, while current runtime treats no/main session as not sandboxed, so gateway-owned exec can still be clamped without this diagnostic.

## Solution Assessment

No. A startup warning is the right layer, but the current predicate is too broad; the maintainable fix is to mirror runtime gateway ownership instead of suppressing every `non-main` sandbox configuration.
