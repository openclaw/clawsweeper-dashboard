## Summary

Keep this PR open: the startup warning is useful and the contributor supplied real terminal proof, but the current head still suppresses the warning for `non-main` sandbox mode even though main/no-session exec remains gateway-owned.

## What This Changes

The PR adds a gateway startup warning for host-approval clamps on global `tools.exec` security, extracts exec target resolution helpers, adds focused tests, and updates troubleshooting docs to point at `exec-policy`.

## Best Possible Solution

Keep the startup warning, but make sandbox ownership match runtime session semantics: warn for gateway-owned main/no-session paths under `non-main` and stay quiet for node-owned or always-sandboxed paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `agents.defaults.sandbox.mode="non-main"`, `tools.exec.host=auto`, `tools.exec.security=full`, and host approvals `allowlist` make the PR helper return no warning while current runtime treats no-session/main exec as gateway-owned.

## Solution Assessment

No. A startup warning is the right layer, but the current predicate is too broad; the maintainable fix is to mirror runtime main/non-main ownership instead of treating every `non-main` sandbox config as startup-wide sandbox availability.
