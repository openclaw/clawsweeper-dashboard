## Summary

Keep open: the PR is useful but not merge-ready because the new startup warning suppresses a real gateway-owned case when sandbox mode is `non-main`, so some users can still miss the host-approvals clamp this PR is meant to surface.

## What This Changes

The PR adds a gateway startup warning for host approval clamps, factors exec target resolution into a shared helper, and updates troubleshooting docs to point at `openclaw exec-policy`.

## Best Possible Solution

Keep the startup warning, but make its host-ownership predicate match runtime semantics so gateway-owned main sessions still warn while node-owned and always-sandboxed scopes stay quiet.

## Reproduction Assessment

Yes by source inspection for the PR-introduced false negative: `non-main` sandbox mode is treated as startup-wide sandbox availability, while current runtime only sandboxes non-main sessions. I did not run local tests because this review is read-only.

## Solution Assessment

No. A startup warning is the right layer, but the current predicate is too broad; the maintainable fix is to mirror runtime gateway ownership instead of suppressing every `non-main` sandbox configuration.
