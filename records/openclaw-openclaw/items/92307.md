## Summary

Keep open: the PR still suppresses the startup clamp warning for `tools.exec.host=auto` with `sandbox.mode="non-main"`, even though current runtime treats main/no-session exec as gateway-owned and can still be clamped by host approvals.

## What This Changes

The PR adds a gateway startup warning for host-approval exec security clamps, extracts exec target resolution helpers, updates focused tests, and refreshes troubleshooting docs to point at `openclaw exec-policy`.

## Best Possible Solution

Keep the startup warning, but make its host-ownership predicate mirror runtime session semantics: warn for gateway-owned main/no-session auto exec under `non-main`, and suppress only node-managed or fully sandbox-owned paths.

## Reproduction Assessment

Yes. Source inspection shows the PR resolves `non-main` as startup-wide sandbox availability, while current main resolves no-session/main exec as not sandboxed and therefore gateway-owned.

## Solution Assessment

No. A startup warning is the right layer, but the predicate is too broad; the narrow maintainable fix is to mirror runtime gateway ownership before suppressing the warning.
