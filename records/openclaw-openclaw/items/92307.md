## Summary

The proof now directly demonstrates the intended gateway-host case, but the patch misclassifies `host=auto` when sandbox is available: runtime ignores local host approvals there, while the new startup check still reports them as clamping security and recommends changing them. This concrete sibling-path defect should be fixed before merge.

## What This Changes

Adds a one-time gateway startup warning when host approvals appear to reduce requested global exec security, shares the comparison with exec-policy reporting, adds focused tests, and updates troubleshooting remediation.

## Best Possible Solution

Compute or carry the effective runtime host into the startup diagnostic, suppress the warning whenever `auto` resolves to sandbox, and add focused coverage for both `auto`→sandbox and `auto`→gateway behavior while preserving the proven explicit-gateway case.

## Reproduction Assessment

Yes. Current main has a high-confidence source and test path where `host=auto` plus available sandbox ignores local approvals, while the proposed warning’s changed guard still treats those approvals as effective.

## Solution Assessment

No, not yet. Reusing the reporting snapshot is reasonable for explicit gateway scope, but startup must also use the runtime’s sandbox-aware effective-host resolution before claiming a clamp or recommending approval changes.
