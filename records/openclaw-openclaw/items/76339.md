## Summary

Keep this issue open. Current main still has the source-visible Linux gateway service-shape regression, and the linked closing PR #76346 is open/unmerged, so the issue should remain open until that fix lands or is replaced.

## What This Changes

The issue asks `openclaw gateway install --force` to stop regenerating Linux systemd gateway units with version-manager Node/PATH entries and inline gateway token state after an upgraded install.

## Best Possible Solution

Review and land or replace #76346 so install, doctor, and audit share one canonical Linux gateway service plan, then file a narrower follow-up if inline-token or RPC-timeout behavior remains after the runtime/PATH fix.

## Reproduction Assessment

Yes for the core service-shape regression by source inspection: current main can select a supported version-manager Node for `ExecStart` and inject its bin dir into service PATH while audit flags that shape. No live current-main repro was established for the inline-token and CPU-bound RPC-timeout parts.

## Solution Assessment

Yes in direction: preferring a supported system Node for managed Linux gateway services, while keeping a version-manager fallback when no supported system Node exists, is the narrowest maintainable fix. The issue should not close until the open closing PR lands or an equivalent replacement is merged.
