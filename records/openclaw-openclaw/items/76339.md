## Summary

Keep this open. Current main still has a source-visible install/audit disagreement for the version-manager Node/PATH service shape, and the linked PR #76346 is still the open implementation path for the central regression; the inline-token and RPC-timeout symptoms need explicit follow-up proof.

## What This Changes

The issue asks for `openclaw gateway install --force` to regenerate the repaired Linux systemd gateway service shape after upgrade, without version-manager Node/PATH metadata or inline gateway-token persistence.

## Best Possible Solution

Land or supersede PR #76346 with focused Linux version-manager install coverage, then verify whether the inline-token and RPC-timeout symptoms remain as separate follow-ups.

## Reproduction Assessment

Yes for the core Node/PATH service-shape regression by source inspection: current main can still select a supported version-manager `process.execPath` and write that path into the managed service. ## Solution Assessment

Yes for the Node/PATH portion: making gateway service install prefer supported system Node is the narrow maintainable fix and matches doctor/audit expectations. The token and RPC symptoms should not be hidden by that fix unless regression coverage proves they are resolved too.
