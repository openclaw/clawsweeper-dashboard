## Summary

Keep this open. Current main still has the source-visible gateway install path where a supported version-manager `process.execPath` can be selected for the managed service, and the provided GitHub context shows open PR #76346 with closing syntax for this issue.

## What This Changes

The issue reports that `gateway install --force` on an upgraded Linux v2026.5.2 host regenerates a systemd unit with version-manager Node/PATH and inline gateway token metadata, followed by audit warnings and RPC timeout.

## Best Possible Solution

Review and land or supersede PR #76346 for the system-Node install path, then verify whether the inline-token and RPC-timeout symptoms need a separate follow-up.

## Reproduction Assessment

Yes for the core service-shape regression by source inspection: a supported nvm/fnm/volta `process.execPath` can still become service ExecStart and PATH on current main. The inline-token and CPU-bound RPC-timeout portions remain unclear without host/live reproduction.

## Solution Assessment

Unclear for the whole issue, but PR #76346 is the narrow likely solution for the Node/PATH portion. This should stay open until that PR lands or is replaced and the remaining token/timeout symptoms are checked.
