## Summary

Keep open: this maintainer-labeled draft has a blocking picker regression because the new picker renders raw session rows instead of the existing filtered session-option groups, so hidden or wrong-target sessions can become clickable chat switch targets.

## What This Changes

This PR replaces the chat session select/search row with a popover-backed session picker, moves search and pagination into picker-local state, updates responsive styles, and adds focused picker tests.

## Best Possible Solution

Keep the picker-local search and pagination model, but render switchable rows through the shared session-option filtering and label-disambiguation semantics before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: seed the picker with other-agent, global/unknown, cron, and subagent rows, then the PR's raw `rows = result?.sessions` rendering will offer rows current main filters out. I did not run browser tests because this review is read-only.

## Solution Assessment

No, not as written: moving search state into the picker is the right direction, but the picker must preserve the existing session-option filter and disambiguation rules before its rows become clickable switch targets.
