## Summary

Keep open. Current main still misses the reported Feishu nested post-format fallback shape, and there is an open PR with closing syntax plus another overlapping parser PR, so this should stay open until a corrected parser/test fix lands.

## What This Changes

Feishu's message reader should extract text, title, and link content from post-format interactive-card fallbacks with nested element rows instead of returning `[Interactive Card]`.

## Best Possible Solution

Land one focused Feishu-plugin parser and regression-test fix that handles bounded one-level fallback rows, top-level/header titles, text/link nodes, and existing locale/post fallthrough without broadening the change.

## Reproduction Assessment

Yes for the main parser failure at source level: a mocked `getMessageFeishu()` response with `msg_type: "interactive"` and `body.content` set to `{ title, elements: [[{ tag: "text", text: "..." }]] }` follows current main to `[Interactive Card]`. The newer inbound thread empty-body scenario is not separately reproduced from the provided log.

## Solution Assessment

Yes in direction: a narrow Feishu-plugin-local parser/test update is the maintainable fix. The exact implementation should be selected from the open PRs or a replacement that preserves current fallthrough behavior and avoids unbounded traversal.
