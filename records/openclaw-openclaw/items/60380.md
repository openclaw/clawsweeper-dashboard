## Summary

Keep open. Current main still cannot extract the reported Feishu nested post-format fallback shape, and the issue is paired with an open closing PR plus overlapping parser work that should be consolidated before closure.

## What This Changes

The issue requests Feishu message reads and quoted cards to extract text from post-format fallback interactive-card content with nested rows, text/link nodes, and titles.

## Best Possible Solution

Land one focused Feishu-plugin parser fix that handles bounded one-level fallback rows, title/header text, `text`/`a` nodes, and existing fallthrough behavior, while coordinating with https://github.com/openclaw/openclaw/pull/60383, https://github.com/openclaw/openclaw/pull/73203, and the API-readback work at https://github.com/openclaw/openclaw/issues/78289.

## Reproduction Assessment

Yes at source level, but not with a live tenant in this review. A mocked `getMessageFeishu()` response with `msg_type: "interactive"` and `body.content` set to `{ title, elements: [[{ tag: "text", text: "..." }]] }` reaches current main’s parser path and returns `[Interactive Card]`.

## Solution Assessment

Yes in direction: the maintainable fix is local to the Feishu plugin parser and focused tests. The final implementation should be selected from, or replace, the open parser PRs while preserving current fallthrough behavior and avoiding unbounded traversal.
