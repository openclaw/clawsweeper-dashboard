## Summary

Keep open. Current main still drops the reported Feishu top-level nested `elements` fallback shape, and the open paired implementation PR remains the right path to review before this issue can close.

## What This Changes

Make Feishu interactive-card parsing extract useful title, text, and link content from post-format fallback payloads when reading quoted or fetched card messages.

## Best Possible Solution

Land or revise the Feishu-plugin parser and regression tests so nested top-level fallback elements, titles, and link/text nodes are normalized without regressing existing element-array fallthrough.

## Reproduction Assessment

Yes. A focused `getMessageFeishu()` unit test can mock `client.im.message.get` with `msg_type: "interactive"` and `body.content` set to `{"title":"...","elements":[[{"tag":"text","text":"..."}]]}`; current main ignores the nested array and `parsePostContent()` cannot resolve the elements-only payload.

## Solution Assessment

Yes. A narrow Feishu-plugin-local parser and regression-test fix is the right direction; the open PR or a replacement should resolve the known parser-control-flow and real-proof blockers before merge.
