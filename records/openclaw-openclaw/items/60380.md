Keep open: current main still misses the reported Feishu fallback-card parser shape, and an open closing PR already owns the direct fix path but is not merged.

## What This Changes

The issue reports Feishu interactive-card readback losing fallback post-format card text when `elements` contains nested rows with `tag: "text"`/`tag: "a"` nodes and a top-level title.

## Best Possible Solution

Land one bounded Feishu plugin parser/readback fix that handles nested fallback rows and known text/link/title shapes, preserves existing template and post fallbacks, adds regression tests, and resolves the linked PR or a maintainer-chosen replacement.

## Reproduction Assessment

Yes, at source level. A `getMessageFeishu()` response with `msg_type: "interactive"` and `body.content` containing `{ title, elements: [[{ tag: "text", text: "..." }]] }` reaches current main's parser, where the nested row is ignored and `tag: "text"` is unsupported.

## Solution Assessment

Yes in direction, but not closed yet. The narrow fix belongs in the Feishu plugin parser/readback path, with the open closing PR reviewed, repaired, or replaced as the consolidated fix.
