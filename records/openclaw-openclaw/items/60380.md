## Summary

Keep open. Current main still misses the reported Feishu fallback shape, and an open PR with closing syntax already targets this issue, so closing before a corrected implementation lands would be premature.

## What This Changes

Fix Feishu interactive-card reads so post-format fallback content with top-level title and nested `elements` rows is extracted instead of returning `[Interactive Card]`.

## Best Possible Solution

Land a focused Feishu-plugin parser and regression-test fix through https://github.com/openclaw/openclaw/pull/60383 or a maintainer-selected replacement that handles top-level titles, bounded nested fallback rows, text/link extraction, and preserves existing fallthrough behavior.

## Reproduction Assessment

Yes, at source level. A mocked `getMessageFeishu()` response with `msg_type: "interactive"` and `body.content` set to the reported `{ title, elements: [[{ tag: "text", text: "..." }]] }` shape follows current main to `[Interactive Card]`.

## Solution Assessment

Yes, the narrow Feishu-plugin parser/test boundary is the right solution. The final implementation should avoid unbounded traversal and preserve later candidate fallthrough before this issue closes.
