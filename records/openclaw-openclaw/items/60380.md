## Summary

Keep open. Current main still drops the reported Feishu fallback shape, and open PR #60383 is the matching implementation candidate with closing syntax.

## What This Changes

The issue asks Feishu's interactive-card reader to extract text from post-format fallback payloads with top-level titles and nested element arrays.

## Best Possible Solution

Land or revise the Feishu-plugin parser/test update so it normalizes top-level fallback `elements` arrays, includes title/header text, and reuses the rich-post renderer for text, links, and media placeholders where practical.

## Reproduction Assessment

Yes. A focused `getMessageFeishu` unit test can mock `client.im.message.get` with `msg_type: "interactive"` and the issue's top-level `{ title, elements: [[...]] }` body content; current control flow returns the interactive-card fallback instead of extracted text.

## Solution Assessment

Yes. A narrow Feishu-plugin-local parser and regression-test patch is the best way to solve this owner-specific parsing gap; no core API or cross-plugin change appears necessary.
