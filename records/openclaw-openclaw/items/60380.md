## What This Changes

The issue reports that Feishu interactive card reads lose text when the Feishu API returns post-format fallback content with top-level `title` and nested `elements` arrays.

## Best Possible Solution

Review and land PR https://github.com/openclaw/openclaw/pull/60383, or a maintainer-selected replacement, with a focused Feishu parser/test fix for top-level fallback `title` plus nested `elements` text/link/media placeholders.
