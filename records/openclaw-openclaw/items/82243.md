## Summary

This is a valid source-level bug report, but it should stay open because https://github.com/openclaw/openclaw/pull/82245 is an open implementation PR with closing syntax for this issue. Current main still treats `agents.defaults.models` runtime policy entries as exact model matches, while provider wildcard visibility is implemented in the separate model-selection path.

## What This Changes

The issue asks for `agents.defaults.models["provider/*"].agentRuntime` to affect runtime policy resolution consistently with provider wildcard model visibility.

## Best Possible Solution

Review and land or deliberately close https://github.com/openclaw/openclaw/pull/82245, then close this issue once main has the wildcard runtime-policy behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows the runtime-policy resolver compares model entries against the concrete model id, so a `vllm/*` entry cannot match `vllm/qwen-local`; the reporter also supplied the focused failing-test expectation.

## Solution Assessment

Yes. Adding literal `provider/*` fallback in the model-map runtime-policy resolver, after exact model entries and before provider-level fallback, is the narrow fix and matches the already-merged visibility contract.
