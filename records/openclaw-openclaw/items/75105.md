## Summary

Keep open. Current main has the model-level `compat.thinkingFormat` surface and a shared DeepSeek V4 replay wrapper, but custom OpenAI-compatible providers still do not get that wrapper from model compat alone; the linked implementation PR was closed unmerged.

## What This Changes

Allow per-model compatibility config to trigger DeepSeek-style `reasoning_content` replay handling for custom OpenAI-compatible providers.

## Best Possible Solution

Add a model-compat-driven DeepSeek V4 replay wrapper path for custom OpenAI-compatible providers, reusing the shared wrapper and documenting the supported config key.

## Reproduction Assessment

Yes. A source-level reproduction is a custom `openai-completions` provider with a DeepSeek V4 model, `compat.thinkingFormat: "deepseek"`, thinking enabled, and replayed assistant history missing `reasoning_content`; current main renders DeepSeek thinking format but does not install the payload wrapper unless a provider plugin does it.

## Solution Assessment

Yes. The model-level opt-in direction is the narrow maintainable fix; prefer wiring the existing `compat.thinkingFormat === "deepseek"` surface into the generic wrapper path, with an alias only if maintainers want the requested naming.
