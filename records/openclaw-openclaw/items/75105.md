## Summary

Keep open. Current main has the shared DeepSeek V4 replay helper and provider-specific fixes for DeepSeek, OpenRouter, Venice, and opencode-go, but it still does not install that helper from model-level compat metadata for arbitrary OpenAI-compatible providers such as Requesty.

## What This Changes

Request adds a model-level compat opt-in so custom OpenAI-compatible providers can apply the DeepSeek `reasoning_content` replay fix without hard-coded provider/model IDs.

## Best Possible Solution

Add a generic OpenAI-completions replay wrapper keyed by `model.compat.thinkingFormat === "deepseek"`, reusing the shared DeepSeek V4 helper and leaving existing provider-specific wrappers intact.

## Reproduction Assessment

Yes. A source-level reproduction is a custom `openai-completions` provider/model with `compat.thinkingFormat: "deepseek"`, thinking enabled, and replayed assistant history missing `reasoning_content`; current main renders DeepSeek thinking format but does not install the replay wrapper from that compat value.

## Solution Assessment

Yes. The model-level opt-in is the narrow maintainable direction, and using the existing `thinkingFormat` compat surface is safer than adding another key for the same behavior.
