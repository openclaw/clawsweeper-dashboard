## Summary

Keep this issue open. Current main still passes provider-startup config into Discord native command and Slack slash-command route resolution, and the issue is paired with open fix PR https://github.com/openclaw/openclaw/pull/39617, which should land, be retargeted, or be replaced before this is closed.

## What This Changes

Discord, Telegram, and Slack native slash commands should route DMs using the configured `session.dmScope` instead of falling back to the main session.

## Best Possible Solution

Retarget or replace the linked PR with a narrow plugin-owned fix that refreshes route-sensitive config for Discord and Slack native commands, preserves Telegram’s existing fresh-runtime path, and adds DM routing regression coverage.

## Reproduction Assessment

Yes, by source inspection. `resolveAgentRoute` derives direct-message session keys from `cfg.session.dmScope`, while current Discord and Slack native command paths still pass captured startup config into that route-resolution path.

## Solution Assessment

Yes for the issue direction. The narrow maintainable solution is to refresh route-sensitive config at the Discord and Slack plugin command boundaries and leave Telegram’s already-fresh native command path intact.
