## Summary

Keep this issue open. Current main still passes provider-startup config into Discord native command and Slack slash-command route resolution, while Telegram already uses a fresh runtime config path and an open paired fix PR should be reviewed, retargeted, or replaced before this issue is closed.

## What This Changes

Native Discord, Telegram, and Slack slash commands should resolve DM sessions using the current `session.dmScope` instead of a startup config snapshot.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/39617 with a narrow extension-owned fix that refreshes route-sensitive config for Discord and Slack native commands, preserves Telegram’s fresh-runtime path, and adds focused DM routing regression coverage.

## Reproduction Assessment

Yes, by source inspection. `resolveAgentRoute` derives DM session keys from `cfg.session.dmScope`, and current Discord and Slack native command paths still pass captured startup config into that route-resolution path.

## Solution Assessment

Yes for the issue direction, but not by closing this issue yet. The maintainable solution is a narrow Discord/Slack extension fix with regression coverage, while Telegram’s already-fresh native command path should be left intact.
