## Summary

Keep open: current main still leaves Discord native-command and Slack slash-command route resolution tied to startup config while the routing contract depends on `cfg.session.dmScope`. Telegram has already moved to fresh runtime config, and open PR #39617 references this issue with closing syntax, so the issue should stay open until that PR is reviewed, rebased, replaced, or merged.

## What This Changes

Native command and slash command handling for Discord, Telegram, and Slack DMs should resolve session targets using fresh runtime config so non-default `session.dmScope` values are respected.

## Best Possible Solution

Land a narrow channel-owned fix on current main that refreshes route-sensitive runtime config for Discord native command and Slack slash command route resolution, preserves Telegram's existing fresh-config path, and adds focused DM regression coverage for non-main `session.dmScope`.

## Reproduction Assessment

Yes. The issue gives concrete DM steps, and current source verification shows `resolveAgentRoute` consumes `cfg.session.dmScope` while Discord and Slack command paths still pass startup config into route resolution.

## Solution Assessment

Yes. Refreshing runtime config at the Discord/Slack command-handling boundary is the narrowest maintainable fix; because PR #39617 is already open with closing syntax, the correct path is to review, rebase, replace, or merge that implementation rather than close this issue now.
