## Summary

Keep this issue open. Current main still routes Discord native command interactions and Slack slash commands through provider-startup config objects, while Telegram already reloads runtime config before native command route resolution; the issue is also paired with open fix PR https://github.com/openclaw/openclaw/pull/39617 and should wait for that corrected implementation to land or be replaced.

## What This Changes

Native slash commands in Discord, Telegram, and Slack DMs should respect `session.dmScope` routing the same way normal DM messages do.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/39617 with a narrow plugin-owned fix that refreshes route-sensitive config for Discord and Slack native commands, preserves Telegram’s existing fresh-runtime path, and adds DM routing regression coverage.

## Reproduction Assessment

Yes, by source inspection. `resolveAgentRoute` derives direct-message session keys from `cfg.session.dmScope`, and current Discord and Slack native command paths still pass provider-startup config into that route-resolution path.

## Solution Assessment

Yes for the issue direction, but the paired branch needs correction. The narrow maintainable fix is to refresh route-sensitive config at the current Discord and Slack plugin command boundaries while keeping Telegram’s already-fresh path intact.
