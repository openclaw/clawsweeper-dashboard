## Summary

Keep this open: current main still has source-backed Discord and Slack command paths that pass provider-startup config into route resolution while `resolveAgentRoute` depends on `session.dmScope`. Telegram appears already corrected, and the issue is paired with the open closing PR at https://github.com/openclaw/openclaw/pull/39617, so it should not close before that PR is landed, replaced, or closed.

## What This Changes

Discord, Telegram, and Slack slash/native commands should route DMs using the current `session.dmScope` instead of a provider-startup config snapshot.

## Best Possible Solution

Retarget or replace the linked PR with a narrow current-plugin fix for Discord and Slack route-sensitive config refresh, keep Telegram’s already-fresh path intact, and add focused DM session-key regression coverage.

## Reproduction Assessment

Yes. Source inspection shows Discord native command routing and Slack slash routing still pass captured config into `resolveAgentRoute`, which reads `session.dmScope`; this was not live-verified against real Discord or Slack transports.

## Solution Assessment

Yes for the fix direction, but not by closing this issue now. Refreshing route-sensitive config in the affected channel-owned command handlers is the narrow maintainable solution, preferably through the linked PR after retargeting or replacement.
