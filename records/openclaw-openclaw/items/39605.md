## Summary

Keep this issue open. Current main still routes Discord native commands and Slack slash commands through provider-startup config objects even though route resolution depends on `session.dmScope`; Telegram already uses a fresh runtime-config path, and open PR #39617 uses closing syntax for the same bug.

## What This Changes

Discord, Telegram, and Slack native command handling should route DMs using the current `session.dmScope` instead of a startup config snapshot.

## Best Possible Solution

Land a narrow channel-owned fix that refreshes route-sensitive runtime config for Discord native commands and Slack slash commands, preserves Telegram’s existing fresh-config path, and adds DM regression coverage for non-main `session.dmScope`.

## Reproduction Assessment

Yes. The issue gives concrete config/restart/DM `/status` steps, and source inspection confirms the remaining Discord and Slack command paths pass captured startup config into route resolution while `resolveAgentRoute` consumes `session.dmScope`.

## Solution Assessment

Yes. Refreshing runtime config at the native-command handling boundary is the narrowest maintainable fix; because #39617 is already open with closing syntax, the immediate path is normal PR review rather than closing this issue.
