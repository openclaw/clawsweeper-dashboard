## Summary

Current main still routes Discord native commands and Slack slash commands through provider-startup config while Telegram already reloads runtime config before native command route resolution. The issue should stay open because the bug remains partially unresolved and is paired with open fixing PR https://github.com/openclaw/openclaw/pull/39617 using closing syntax.

## What This Changes

Native Discord, Telegram, and Slack slash command routing should respect `session.dmScope` for DM sessions instead of falling back to the main session.

## Best Possible Solution

The end state should be a narrow plugin-owned Discord and Slack fix that refreshes route-sensitive config before native command route resolution, preserves Telegram's already-fresh path, and adds DM session-key regression coverage.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path. Current main still passes captured provider-startup config into Discord native-command and Slack slash-command route resolution, while `resolveAgentRoute` derives the DM session key from `cfg.session.dmScope`.

## Solution Assessment

Yes, the reported fix direction is the narrowest maintainable path. Refresh route-sensitive config in the affected plugin command handlers and add focused regression tests, rather than changing the global routing contract.
