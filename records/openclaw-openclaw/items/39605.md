## Summary

Keep this issue open. Current main still routes Discord native command interactions and Slack slash commands through startup config objects while route resolution depends on `session.dmScope`, and open PR #39617 uses closing syntax for this bug.

## What This Changes

Native slash commands in Discord, Telegram, and Slack should route DMs using the current `session.dmScope` instead of a startup config snapshot.

## Best Possible Solution

Land a narrow channel-owned fix that reloads route-sensitive runtime config before Discord and Slack native/slash route resolution, keeps Telegram's fresh path intact, and adds DM regression coverage.

## Reproduction Assessment

Yes. The issue gives concrete DM steps, and source inspection confirms the remaining Discord and Slack command paths pass startup config into route resolution while `resolveAgentRoute` consumes `session.dmScope`.

## Solution Assessment

Yes. Refreshing runtime config at the Discord and Slack command-handling boundary is the narrowest maintainable fix; the immediate path is to review, rebase, replace, or merge #39617.
