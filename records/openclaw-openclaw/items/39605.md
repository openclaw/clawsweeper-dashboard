## Summary

Keep this issue open: current main still has source-backed Discord and Slack command paths that pass provider-startup config into route resolution, while the linked closing PR is still open, conflicting, and aimed at removed legacy channel files. Telegram’s active native-command path already reloads runtime config before route resolution, so the remaining work is narrower than the original report.

## What This Changes

Slash/native commands in Discord, Telegram, and Slack DMs should respect `session.dmScope` routing instead of falling back to the main session.

## Best Possible Solution

Retarget or replace the linked PR with a narrow fix in `extensions/discord` and `extensions/slack` that refreshes route-sensitive runtime config, keep Telegram’s already-fresh native-command path intact unless new evidence shows a gap, and add focused regression coverage plus redacted real channel proof.

Yes, at source level: current Discord native command routing and Slack slash routing still pass provider-startup config into `resolveAgentRoute`, which reads `session.dmScope`. No live Discord or Slack transport run was performed in this read-only review.

## Solution Assessment

Yes for the fix direction, but not by closing this issue now. The maintainable path is refreshing route-sensitive config in the active Discord and Slack command handlers, then landing or replacing the linked PR after tests and proof.
