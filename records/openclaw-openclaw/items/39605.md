## Summary

Keep this issue open. Current main still passes provider-startup config into Discord native command routing and Slack slash command routing while `resolveAgentRoute` derives DM session keys from `session.dmScope`; Telegram already uses a fresh runtime config path. The issue is also paired with open fixing PR https://github.com/openclaw/openclaw/pull/39617 using closing syntax, so it should stay open until that PR lands, is retargeted, or is replaced.

## What This Changes

Native Discord, Telegram, and Slack slash commands should respect `session.dmScope` routing in DMs instead of falling back to `agent:main:main`.

## Best Possible Solution

Review or replace the linked PR with a narrow current-plugin fix for Discord and Slack route-sensitive config refresh, preserve Telegram’s already-fresh path, and add focused DM session-key regression coverage.
