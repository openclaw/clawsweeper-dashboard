## Label Justifications

- `P2`: This is a real session-routing bug in major messaging channels, but it is limited to native/slash commands under non-default DM scope settings.
- `impact:session-state`: Affected commands can read or operate on a different direct-message session than ordinary messages in the same conversation.

## Summary

Keep open: current main still leaves the central Discord and Slack slash/native command routing issue unresolved, while Telegram already uses a fresh runtime-config path. The issue is also paired with an open closing PR, so the correct cleanup path is to review, refresh, replace, or close that PR before changing this issue state.

## What This Changes

Discord, Telegram, and Slack native slash commands are reported to route direct messages without honoring `session.dmScope`, causing commands to use the wrong agent session key.

## Best Possible Solution

The end state should be a narrow channel-owned fix that resolves Discord and Slack native-command DM targets with fresh runtime config, keeps Telegram's already-fresh path intact, and adds regression coverage for non-main `session.dmScope` values.

## Reproduction Assessment

Yes at source level: set `session.dmScope` to a non-main value, then compare ordinary DM routing with native/slash command routing. Current Discord and Slack source still routes native commands through captured config, while `resolveAgentRoute` derives DM session keys from that config.

## Solution Assessment

Yes: refreshing route-sensitive config inside the channel-owned native/slash command paths is the narrowest maintainable fix. Changing `resolveAgentRoute` globally would be broader and compatibility-sensitive because other callers intentionally pass config snapshots.
