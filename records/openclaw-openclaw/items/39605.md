## Label Justifications

- `P2`: This is a real session-routing bug in major messaging channels, but it is limited to native/slash commands under non-default DM scope settings.
- `impact:session-state`: Affected commands can read or operate on a different direct-message session than ordinary messages in the same conversation.

## Summary

Keep open: current main and the latest release still leave the central Discord and Slack slash/native command routing problem unresolved, while Telegram already has the fresh-runtime-config pattern. The issue is also paired with an open closing PR, so it should stay open until that PR is refreshed, replaced, merged, or explicitly closed.

## What This Changes

Native slash commands in Discord, Telegram, and Slack DMs should respect non-main session.dmScope routing instead of targeting the shared main session.

## Best Possible Solution

Review and refresh or replace the linked implementation PR so Discord and Slack resolve native/slash DM targets with fresh runtime config, add focused non-main dmScope regression coverage, and close this issue only after that fix merges.

## Reproduction Assessment

Yes, at source level: configure a non-main session.dmScope and compare ordinary DM routing with Discord/Slack native command routing. The route resolver uses cfg.session.dmScope, and those command paths still pass provider-startup config.

## Solution Assessment

Yes: refreshing route-sensitive runtime config inside the Discord and Slack native/slash command paths is the narrowest maintainable fix. Changing resolveAgentRoute globally would be broader because callers intentionally pass specific config snapshots.
