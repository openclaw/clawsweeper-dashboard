## Summary

Keep this open: current main still has source-backed Discord and Slack slash/native command paths that route with provider-captured config while route resolution depends on `session.dmScope`; Telegram’s native command path has already moved to a fresh runtime config. The issue is also paired with the open closing PR at https://github.com/openclaw/openclaw/pull/39617, so it should stay open until that PR is retargeted/replaced, landed, or closed.

## What This Changes

Discord, Telegram, and Slack slash/native commands should route DM sessions according to `session.dmScope` instead of stale command-handler config.

## Best Possible Solution

Retarget or replace https://github.com/openclaw/openclaw/pull/39617 with a narrow current-plugin fix for Discord and Slack route-sensitive config refresh, leave Telegram’s already-fresh path intact, and add focused DM session-key regression coverage.

## Reproduction Assessment

Yes. Source inspection shows Discord native command routing and Slack slash routing pass captured config into `resolveAgentRoute`, which reads `session.dmScope`; this was not live-verified against real Discord or Slack transports.

## Solution Assessment

Yes for the fix direction. Refreshing route-sensitive config in the affected channel-owned command handlers is the narrow maintainable solution; the current paired PR needs retargeting to `extensions/*` and should avoid changing Telegram unless a new Telegram regression appears.
