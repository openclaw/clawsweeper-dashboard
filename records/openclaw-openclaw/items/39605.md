## Label Justifications

- `P2`: This is a real session-routing bug in major channels, but it is limited to native/slash commands under non-default DM scope settings.
- `impact:session-state`: Affected commands can read or operate on a different direct-message session than ordinary messages in the same conversation.

## Summary

Keep open: current main and the latest release only partially address the report. Telegram native commands now reload runtime config before route resolution, but Discord and Slack still pass provider-startup config into native/slash routing, and the open closing PR owns the fix but is currently conflicting and edits former core paths.

## What This Changes

Discord, Telegram, and Slack slash/native commands should respect `session.dmScope` in DMs the same way ordinary messages do.

## Best Possible Solution

Refresh Discord and Slack native/slash route-sensitive config through the current plugin runtime-config seam, keep Telegram’s already-fresh path intact, and add regression tests for non-main DM session keys.

## Reproduction Assessment

Yes at source level: configure `session.dmScope` as `per-channel-peer`, then compare ordinary DM routing with native/slash command routing. Current Discord and Slack command paths still pass provider-startup `cfg` into route resolution; I did not run live channel accounts in this read-only review.

## Solution Assessment

Yes: a dispatch-time refresh of route-sensitive config in the channel-owned command paths is the narrowest maintainable fix. The open PR should be refreshed to current `extensions/*` files and scoped to the still-affected Discord and Slack paths plus regression coverage.
