## Summary

Keep this open. Current main still routes Discord native command interactions and Slack slash commands through provider-startup config objects while `resolveAgentRoute` derives DM session keys from `cfg.session.dmScope`; Telegram already uses a fresh runtime config path, and open PR #39617 is paired with this issue.

## What This Changes

Native Discord, Telegram, and Slack slash commands should route DMs using the current `session.dmScope` instead of a provider-startup config snapshot.

## Best Possible Solution

Review and land or replace PR #39617 with a narrow channel-owned fix for Discord and Slack route-sensitive config refresh, preserving Telegram's already-fresh path and adding DM regression coverage.

## Reproduction Assessment

Yes, by source inspection. The issue's DM steps match current main because `resolveAgentRoute` depends on `cfg.session.dmScope` while Discord and Slack native command paths still pass captured startup config objects.

## Solution Assessment

Yes. Refreshing route-sensitive runtime config at the Discord and Slack command-handling boundary is the narrowest maintainable fix; because #39617 is already open with closing syntax, this issue should wait for that PR to land or be replaced.
