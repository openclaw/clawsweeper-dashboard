## Summary

Keep this open. Current main still passes startup config into Discord native-command and Slack slash-command route resolution while `resolveAgentRoute` reads `cfg.session.dmScope`; Telegram already has the fresh runtime-config path, and open PR #39617 is paired with this bug.

## What This Changes

The issue asks Discord, Telegram, and Slack native slash commands in DMs to respect `session.dmScope` instead of routing to the main session.

## Best Possible Solution

Land or replace a narrow channel-owned fix that refreshes route-sensitive runtime config for Discord native commands and Slack slash commands, keeps Telegram's existing fresh path, and adds DM regression coverage for non-main `session.dmScope`.

## Reproduction Assessment

Yes, by source inspection. The issue gives concrete DM steps, and current main still routes Discord and Slack native command paths through captured startup config while `resolveAgentRoute` derives direct-message session keys from `cfg.session.dmScope`.

## Solution Assessment

Yes. Refreshing runtime config at the Discord and Slack command-handling boundary is the narrowest maintainable fix; because #39617 already uses closing syntax, the immediate path is normal PR review rather than closing this issue early.
