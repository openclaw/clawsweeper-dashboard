## Summary

Keep this issue open: current main still has source-backed Discord and Slack command paths that route with provider-startup config while `resolveAgentRoute` depends on `session.dmScope`. Telegram appears already corrected, and the issue is paired with the open closing PR at https://github.com/openclaw/openclaw/pull/39617, so it should stay open until that PR is landed, retargeted, replaced, or closed.

## What This Changes

The issue asks for Discord, Telegram, and Slack native slash commands in DMs to respect `session.dmScope` routing instead of falling back to the agent main session.

## Best Possible Solution

Retarget or replace the linked PR with a current `extensions/discord` and `extensions/slack` fix, preserve Telegram’s fresh runtime-config path, and add focused regression coverage plus redacted real channel proof.

## Reproduction Assessment

Yes. Source inspection shows Discord native command routing and Slack slash routing still pass startup config into `resolveAgentRoute`, which reads `session.dmScope`; this was not live-verified against real Discord or Slack transports.

## Solution Assessment

Yes for the fix direction, but not by closing this issue now. Refreshing route-sensitive runtime config in the affected channel-owned command handlers is the narrow maintainable solution, and the open linked PR needs retargeting from legacy files before it can be that solution.
