## Summary

Keep this issue open. Current main still has source-reproducible Discord and Slack native command paths that resolve DM slash-command sessions with provider-startup config, while `resolveAgentRoute` derives direct-message session keys from `session.dmScope`; Telegram’s active native-command path already reloads runtime config, and the linked closing PR is open, conflicting, and still targets obsolete `src/*` channel paths.

## What This Changes

Discord, Telegram, and Slack native slash commands should respect `session.dmScope` when selecting DM session keys.

## Best Possible Solution

Retarget or replace the linked PR with a narrow active-plugin fix for Discord and Slack command routing, keep Telegram’s fresh-config path unchanged unless new evidence shows a gap, and add focused regression coverage for scoped DM session keys.

## Reproduction Assessment

Yes, at source level: current Discord and Slack native command paths still pass provider-startup config into route resolution, and `resolveAgentRoute` reads `session.dmScope` from that config. I did not run a live transport reproduction in this read-only review.

## Solution Assessment

Yes for the requested fix direction, but not by closing this issue now. The maintainable solution is to refresh route-sensitive config in the active Discord and Slack command handlers, add regression coverage, and then review, retarget, replace, or close the linked PR.
