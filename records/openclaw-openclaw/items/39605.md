## Summary

Keep this issue open. Current main still has a source-backed Discord and Slack native-command routing gap, Telegram already reloads runtime config for native commands, and the issue is paired with an open closing PR that should be reviewed or replaced before closure.

## What This Changes

Slash/native commands in Discord, Telegram, and Slack DMs should respect `session.dmScope` the same way normal text messages do.

## Best Possible Solution

Retarget or replace the open fix with current plugin-owned Discord and Slack route-config refreshes, keep Telegram's fresh runtime path intact, and add focused native-command DM session-key regression coverage.

## Reproduction Assessment

Yes, source-backed. `resolveAgentRoute` uses `cfg.session.dmScope`, and current Discord native-command plus Slack slash-command paths pass provider-startup config into route resolution; I did not run a live channel transport repro.

## Solution Assessment

No, the issue is not solved on current main. Reloading route-sensitive config is the right fix direction, but the open PR needs to move from removed `src/*` channel files to current `extensions/discord` and `extensions/slack` code and add proof/tests.
