## Summary

Keep this issue open. Current main still routes Discord native commands and Slack slash commands through startup config objects while route resolution depends on `session.dmScope`; Telegram already has the fresh runtime-config pattern, and open PR #39617 uses closing syntax for this bug.

## What This Changes

Slash/native command handlers in Discord, Telegram, and Slack should route DMs using the current `session.dmScope` instead of a startup config snapshot.

## Best Possible Solution

Land a narrow channel-owned fix that refreshes route-sensitive runtime config for Discord native commands and Slack slash commands, preserves Telegram's existing fresh-config path, and adds DM regression coverage for non-main `session.dmScope`.

## Reproduction Assessment

Yes. The issue gives concrete DM steps, and source inspection confirms the remaining Discord and Slack command paths pass startup config into route resolution while `resolveAgentRoute` consumes `session.dmScope`.

## Solution Assessment

Yes. Refreshing runtime config at the Discord and Slack command-handling boundary is the narrowest maintainable fix; because #39617 is already open with closing syntax, the immediate path is normal PR review rather than closing this issue.
