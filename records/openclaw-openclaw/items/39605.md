## Label Justifications

- `P2`: This is a real session-routing bug in major messaging channels, but it is limited to native/slash commands under non-default DM scope settings.
- `impact:session-state`: Affected commands can read from or operate on a different direct-message session than ordinary messages in the same conversation.

## Summary

Keep open: current main still leaves Discord and Slack native/slash command routing on provider-startup config while Telegram already has the fresh runtime-config pattern; the issue also has an open closing PR that should be reviewed, refreshed, replaced, or closed before this issue is resolved.

## What This Changes

The issue reports that Discord, Telegram, and Slack native/slash command DMs route through the main session instead of respecting non-main `session.dmScope`.

## Best Possible Solution

Review and refresh or replace https://github.com/openclaw/openclaw/pull/39617 so Discord and Slack resolve native/slash DM route targets from current runtime config, keep Telegram's fresh path intact, and add focused regression proof.

## Reproduction Assessment

Yes, at source level: configure a non-main `session.dmScope` and compare Discord/Slack native slash routing with the route contract that reads `cfg.session.dmScope`. I did not run a live transport reproduction in this read-only review.

## Solution Assessment

Yes, the best fix shape is a narrow channel-owned refresh of route-sensitive runtime config in the Discord and Slack native/slash command paths, plus regression coverage. Changing `resolveAgentRoute` globally would be broader because callers intentionally pass explicit config snapshots.
