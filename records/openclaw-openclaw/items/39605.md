## Summary

Keep this issue open. Current main still routes Discord native interactions and Slack slash commands through provider-startup config objects, while `session.dmScope` is the documented route/session-key contract; the issue is also paired with open fix PR https://github.com/openclaw/openclaw/pull/39617, so it should wait for a corrected implementation to land or be replaced.

## What This Changes

The issue reports that Discord, Telegram, and Slack native slash commands in DMs should respect `session.dmScope` instead of falling back to the main session.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/39617 with a narrow plugin-owned fix that refreshes route-sensitive config for Discord and Slack native commands, preserves Telegram’s existing fresh-runtime path, and adds DM routing regression coverage.

## Reproduction Assessment

Yes, by source inspection. `resolveAgentRoute` derives DM session keys from `cfg.session.dmScope`, and current Discord and Slack native command paths still pass captured provider-startup config into that route-resolution path.

## Solution Assessment

Yes for the issue direction, but not the linked branch as-is. The narrow maintainable fix is to refresh route-sensitive config in the current `extensions/discord` and `extensions/slack` native-command boundaries while leaving Telegram’s already-fresh path intact.
