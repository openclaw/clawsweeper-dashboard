## What This Changes

Discord, Telegram, and Slack slash/native commands in DMs should respect `session.dmScope` routing instead of falling back to the shared main session.

## Best Possible Solution

Retarget or replace https://github.com/openclaw/openclaw/pull/39617 with a narrow plugin-owned fix that refreshes route-sensitive config for Discord and Slack slash/native routing, preserves Telegram's fresh path, and adds focused DM session-key regression coverage.

## Reproduction Assessment

Yes, by source inspection against current main. With a non-main `session.dmScope`, `resolveAgentRoute` computes peer-scoped DM sessions only if the native/slash path passes a fresh config, and current Discord and Slack still pass provider-startup config.

## Solution Assessment

Yes for the requested direction, but not for closing this issue now. The narrow maintainable solution is to refresh route-sensitive config in the current Discord and Slack plugin paths with regression tests, while the linked open PR is reviewed, retargeted, replaced, or closed first.
