## Summary

Keep this issue open. Current main still passes startup config into Discord native-command and Slack slash-command route resolution while `resolveAgentRoute` depends on `session.dmScope`; Telegram already uses the fresh runtime-config pattern, and open PR #39617 uses closing syntax for this bug.

## What This Changes

Discord, Telegram, and Slack native slash commands should route DM command targets using the configured `session.dmScope` instead of falling back to the main session.

## Best Possible Solution

Review and land PR #39617, or replace it with the same narrow channel-owned fix plus Discord and Slack DM regression coverage while leaving Telegram’s fresh-config path intact.

## Reproduction Assessment

Yes. The issue gives concrete DM steps, and source inspection confirms Discord and Slack native command paths still pass startup config into route resolution while `resolveAgentRoute` consumes `session.dmScope`.

## Solution Assessment

Yes. Refreshing route-sensitive runtime config at the native-command handling boundary is the narrowest maintainable fix; because #39617 is already open with closing syntax, this issue should stay open until that implementation or a replacement merges.
