## Summary

Keep this issue open. Current main still routes Discord native commands and Slack slash commands through startup config objects while `session.dmScope` is the documented session-key input; Telegram already uses a fresh runtime config path, and open PR https://github.com/openclaw/openclaw/pull/39617 still references this issue with closing syntax.

## What This Changes

Discord, Telegram, and Slack slash/native commands should respect non-main `session.dmScope` routing in DMs instead of falling back to `agent:main:main`.

## Best Possible Solution

Land, retarget, or replace https://github.com/openclaw/openclaw/pull/39617 with a narrow channel-owned fix that refreshes route-sensitive config for Discord and Slack native command routing and adds DM session-key regression coverage.

## Reproduction Assessment

Yes, by source inspection. With a non-main `session.dmScope`, `resolveAgentRoute` computes a peer-scoped DM session only if the native command path passes a fresh config object, and current Discord and Slack paths still pass startup config.

## Solution Assessment

Yes for the requested direction, but not for closing this issue now. The narrow maintainable solution is a channel-owned config refresh for Discord and Slack slash/native routing plus regression coverage, while the linked open PR must be reviewed, replaced, or closed first.
