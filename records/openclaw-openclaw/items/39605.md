## Label Justifications

- `P2`: This is a real session-routing bug in major messaging channels, but it is limited to native/slash commands under non-default DM scope settings.
- `impact:session-state`: Affected commands can read from or operate on a different direct-message session than ordinary messages in the same conversation.

## Summary

Keep open: current main still leaves Discord native command and Slack slash command route resolution tied to provider/startup config snapshots, while Telegram already has the fresh runtime-config pattern; the issue is also paired with an open closing PR that should be refreshed, reviewed, replaced, or closed before this issue resolves.

## What This Changes

Native slash commands in Discord, Telegram, and Slack DMs are reported to ignore non-default `session.dmScope` routing and use the main session instead of the same DM session as normal messages.

## Best Possible Solution

Refresh, review, or replace https://github.com/openclaw/openclaw/pull/39617 so Discord and Slack native/slash command route resolution uses fresh route-sensitive config, Telegram's fresh path remains intact, and focused regression proof covers non-main DM scopes.

## Reproduction Assessment

Yes, at source level: `resolveAgentRoute` reads `cfg.session.dmScope`, while Discord and Slack command paths still pass captured startup config into route resolution. I did not run live transport proof in this read-only review.

## Solution Assessment

Yes, the best fix shape is a narrow channel-owned refresh of route-sensitive config in the native/slash command routing paths rather than changing `resolveAgentRoute` globally, because other callers intentionally pass explicit config snapshots.
