## Summary

Keep this issue open. Current main and the latest release still route Discord native commands and Slack slash commands through provider-startup config while `resolveAgentRoute` derives DM session keys from `session.dmScope`; Telegram’s active path already reloads runtime config, and the linked closing PR remains open and conflicting.

## What This Changes

The issue reports that Discord, Telegram, and Slack native slash commands in DMs use the main session instead of the session selected by `session.dmScope`.

## Best Possible Solution

Review, retarget, replace, or close the linked PR, with the desired end state being a narrow active-plugin fix for Discord and Slack route-sensitive config freshness plus focused regression coverage; close this issue only after that work lands.

## Reproduction Assessment

Yes, at source level: current Discord and Slack native/slash command paths still pass provider-startup config into `resolveAgentRoute`, which reads `session.dmScope`. I did not run a live transport reproduction in this read-only review.

## Solution Assessment

Yes for the requested fix direction, but not by closing this issue now. The maintainable path is to refresh route-sensitive config in the active Discord and Slack command handlers, keep Telegram’s already-fresh path unchanged unless new evidence shows a gap, and add regression coverage.
