## Summary

Keep this issue open. Current main still leaves Discord native command and Slack slash command route resolution tied to startup config while `resolveAgentRoute` depends on `cfg.session.dmScope`; Telegram already reloads runtime config, and open PR #39617 uses closing syntax for this bug.

## What This Changes

Make Discord, Telegram, and Slack native slash-command DM routing honor current `session.dmScope` instead of falling back to the main session.

## Best Possible Solution

Land a narrow channel-owned fix that refreshes route-sensitive runtime config for Discord native commands and Slack slash commands, preserves Telegram's existing fresh-config path, and adds DM regression coverage for non-main `session.dmScope`.

## Reproduction Assessment

Yes. The issue gives concrete DM steps, and source inspection confirms `resolveAgentRoute` consumes `cfg.session.dmScope` while the Discord and Slack command paths still pass startup config into route resolution.

## Solution Assessment

Yes. Refreshing runtime config at the Discord and Slack command-handling boundary is the narrowest maintainable fix; because PR #39617 is already open with closing syntax, the immediate path is normal PR review rather than closing this issue.
