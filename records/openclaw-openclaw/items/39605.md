## Summary

Keep this issue open. Current main still routes Discord native commands and Slack slash commands through provider-startup config objects while `resolveAgentRoute` derives DM session keys from `session.dmScope`; Telegram’s active native-command path already reloads runtime config, and the linked closing PR is still open/conflicting and targets removed legacy paths.

## What This Changes

Discord, Slack, and Telegram slash/native commands in DMs are reported to ignore `session.dmScope` and use the main session instead of the scoped DM session.

## Best Possible Solution

Retarget or replace the linked PR with a narrow active-plugin fix for `extensions/discord` and `extensions/slack`, keep Telegram’s fresh-config path unchanged unless new evidence shows a gap, and add focused regression coverage plus redacted real channel proof.
