## Label Justifications

- `P2`: This is a real session-routing bug in major channels, but it is limited to native/slash commands under non-default DM scope settings.
- `impact:session-state`: Affected commands can read or operate on a different direct-message session than ordinary messages in the same conversation.

## Summary

Keep open: current main and the latest release still leave the central Discord and Slack native/slash command routing problem unresolved, while Telegram has already moved to a fresh runtime-config path. The issue is also paired with an open closing PR, so it should stay open until that PR is refreshed, replaced, merged, or closed.

## What This Changes

Native Discord, Telegram, and Slack slash commands should respect `session.dmScope` when routing DMs instead of using provider-startup config snapshots.

## Best Possible Solution

Refresh Discord and Slack native/slash route-sensitive config through the current extension runtime-config seam, keep Telegram's fresh path, add focused regression coverage for non-main DM scopes, and resolve the linked PR path.

## Reproduction Assessment

Yes at source level: set `session.dmScope` to a non-main value, then compare ordinary DM routing with native/slash command routing. I did not run live channel accounts in this read-only review, but current Discord and Slack source still passes provider-startup config into route resolution.

## Solution Assessment

Yes: refreshing route-sensitive config inside the channel-owned native/slash command paths is the narrowest maintainable fix. Changing `resolveAgentRoute` globally would be broader and compatibility-sensitive because callers intentionally pass config snapshots in other contexts.
