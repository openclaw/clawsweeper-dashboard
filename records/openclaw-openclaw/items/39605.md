## Summary

Keep this issue open. Current main still has Discord and Slack native command paths passing provider-startup config into route resolution, while the issue is paired with open fix PR https://github.com/openclaw/openclaw/pull/39617 and should wait for a corrected implementation to land or be replaced.

## What This Changes

Native slash commands in Discord, Telegram, and Slack DMs should respect `session.dmScope` when choosing the target session.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/39617 with a narrow plugin-owned fix that refreshes route-sensitive config for Discord and Slack native commands, preserves Telegram’s existing fresh-runtime path, and adds DM routing regression coverage.

## Reproduction Assessment

Yes, by source inspection. `resolveAgentRoute` derives session keys from `cfg.session.dmScope`, and current Discord and Slack native command paths still pass captured startup config into that route-resolution path.

## Solution Assessment

Yes for the issue direction, but the paired branch needs correction. The narrow maintainable fix is to refresh route-sensitive config at the current Discord and Slack plugin command boundaries and keep Telegram’s already-fresh path intact.
