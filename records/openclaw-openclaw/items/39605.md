## Summary

Keep this issue open: current main still has source-backed Discord and Slack command paths that route with provider-startup config while `resolveAgentRoute` depends on `session.dmScope`. Telegram appears already corrected, and the issue is paired with the open closing PR at https://github.com/openclaw/openclaw/pull/39617, so it should not be closed before that PR is landed, replaced, or closed.

## What This Changes

Ensure Discord, Telegram, and Slack native slash command DM routing respects the current `session.dmScope` instead of a startup config snapshot.

## Best Possible Solution

Retarget or replace the linked PR with a narrow `extensions/discord` and `extensions/slack` config-refresh fix, keep Telegram’s already-fresh path intact, and add focused regression coverage plus redacted real channel proof.

## Reproduction Assessment

Yes, at source level. Discord native command routing and Slack slash routing still pass provider-startup config into `resolveAgentRoute`, which reads `session.dmScope`; this was not live-verified against real Discord or Slack transports.

## Solution Assessment

Yes for the requested fix direction, but not by closing this issue now. The maintainable solution is refreshing route-sensitive runtime config in the active channel-owned command handlers, then landing or replacing the linked PR with tests and proof.
