## Summary

Keep open: current main still has a source-backed WhatsApp path where rapid media messages bypass inbound debounce and active-run steering carries only text, so later forwarded-image attachments can be lost from the LLM turn.

## What This Changes

Ensure rapid forwarded WhatsApp messages with multiple images deliver all images to the LLM, either through WhatsApp media coalescing or media-aware steering.

## Best Possible Solution

Implement a WhatsApp-owned rapid media burst path that preserves all local media as MediaPaths/MediaTypes, unless maintainers explicitly choose to extend steering to carry structured media payloads.

## Reproduction Assessment

Yes, at source level: two rapid WhatsApp image messages can be traced through media debounce rejection, single-message media context, and text-only active steering. I did not perform a live WhatsApp forward-burst run against current main.

## Solution Assessment

Unclear until maintainer review: WhatsApp-local coalescing is likely the narrowest owner-boundary fix, but changing default media timing versus extending core steer semantics is a product/runtime decision.
