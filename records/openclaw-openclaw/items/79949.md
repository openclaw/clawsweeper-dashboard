## Summary

Keep open: current main still has a source-backed path where rapid WhatsApp image messages bypass inbound debounce, only singular WhatsApp media context is built, and active-run steering sends text only while suppressing a follow-up turn when steering succeeds.

## What This Changes

Request WhatsApp multi-image forwarded bursts to deliver all images to the LLM, either by coalescing rapid media messages or by making active-run steering carry media attachments.

## Best Possible Solution

Implement a WhatsApp-owned rapid media burst path that preserves all local media as MediaPaths and MediaTypes in one inbound context, or explicitly extend active-run steering to carry media payloads if maintainers want the generic queue contract to own this.

## Reproduction Assessment

Yes, at source level: two rapid WhatsApp image messages can be traced through media debounce rejection, singular WhatsApp media context construction, and text-only active steering. I did not perform a live WhatsApp forward-burst run against current main.

## Solution Assessment

Unclear until maintainer review: WhatsApp-local coalescing appears to be the narrowest owner-boundary fix, but media-capable steering would better match the generic queue docs if maintainers want that contract expanded.
