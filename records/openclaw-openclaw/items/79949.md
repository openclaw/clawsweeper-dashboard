## Summary

Keep open: current main still has a source-backed WhatsApp path where rapid image messages bypass debounce and active-run steering carries only text, so later images can still miss the LLM turn. The related attempted fixes are closed and unmerged, and no canonical open replacement was found in the provided context or live PR checks.

## What This Changes

Forwarded WhatsApp image bursts should preserve every image for the LLM turn, either by WhatsApp media coalescing or by media-aware active-run steering.

## Best Possible Solution

Add a WhatsApp-owned media-burst path that accumulates rapid same-sender media into MediaPaths/MediaTypes before reply dispatch, unless maintainers explicitly choose to extend steering to carry structured media.

## Reproduction Assessment

Yes at source level: two rapid WhatsApp image messages can be traced through media debounce rejection, single-item flushes, singular WhatsApp media context, and text-only active steering. I did not run a live WhatsApp forwarded-image burst.

## Solution Assessment

Unclear: WhatsApp-local coalescing appears to be the narrowest owner-boundary fix, but the issue also exposes a generic queue.mode steer media contract gap that maintainers may want to address instead.
