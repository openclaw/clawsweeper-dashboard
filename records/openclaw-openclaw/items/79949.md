## Summary

Keep open: current main still has a source-backed path where rapid WhatsApp media messages bypass inbound debounce and active-run steering accepts only text, so later forwarded images can be omitted from the LLM turn.

## What This Changes

The issue requests WhatsApp handling that coalesces multiple rapidly forwarded image messages, or otherwise preserves media attachments when queue steering handles overlapping inbound messages.

## Best Possible Solution

Add an owner-bounded WhatsApp media burst path that preserves all local media as multi-attachment context, unless maintainers explicitly choose a generic media-capable steering contract.

## Reproduction Assessment

Yes, at source level: two rapid WhatsApp image messages can be traced through media debounce rejection, singular WhatsApp media context, and text-only active steering. I did not run a live WhatsApp forward-burst test against current main.

## Solution Assessment

Unclear until maintainer review: WhatsApp-local coalescing appears to be the narrowest owner-boundary fix, but extending queue steering to carry media would change a broader runtime contract.
