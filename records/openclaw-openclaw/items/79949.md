## Summary

Keep open: current main still has the reported source path where rapid WhatsApp media messages bypass inbound debounce, active-run steering carries only text, and WhatsApp inbound context exposes only singular media fields.

## What This Changes

The issue requests preserving all images from rapid multi-image WhatsApp forwards in one agent turn or by media-aware steering.

## Best Possible Solution

Preserve every rapid WhatsApp image by adding a WhatsApp-owned media burst path that fills MediaPaths/MediaTypes, or by explicitly extending steer semantics to carry media attachments.

## Reproduction Assessment

Yes, at source level: two rapid WhatsApp image messages can be traced through media debounce rejection, singular WhatsApp media context, and text-only active steering. I did not perform a live WhatsApp forward-burst run against current main.

## Solution Assessment

Unclear until maintainer review: WhatsApp-local coalescing looks like the narrowest owner-boundary fix, but media-aware steering would solve the same loss generically and changes a core queue contract.
