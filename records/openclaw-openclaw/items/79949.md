## Summary

Keep open. Current main and v2026.6.10 still steer active follow-up messages through a text-only embedded queue path, and the open linked PR is the candidate fix rather than a reason to close the issue before merge.

## What This Changes

The issue requests that rapid WhatsApp multi-image messages preserve every image in one agent turn, either through WhatsApp media coalescing or by carrying steered media attachments into the running turn.

## Best Possible Solution

Review, update, and land https://github.com/openclaw/openclaw/pull/94217 or the same narrow steer-queue image threading, while leaving broader WhatsApp batching/config decisions to the adjacent tracking issues.

## Reproduction Assessment

Yes, at source level. Current main lets FollowupRun carry images and AgentSession.steer accept images, but the active embedded steer branch passes only text and text-only options into the queue.

## Solution Assessment

Yes for the narrow bug path. Threading existing FollowupRun images through the embedded steer queue is more maintainable than adding new WhatsApp batching/config behavior for the observed attachment loss.
