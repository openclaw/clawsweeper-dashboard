## Summary

Keep open: current main and shipped v2026.5.12 still have a source-backed WhatsApp path where rapid media messages bypass coalescing and active-run steering injects only text, so later forwarded images can still be lost from the LLM turn.

## What This Changes

The issue requests WhatsApp media coalescing or media-aware steering so rapid forwarded image messages are all included in the LLM turn.

## Best Possible Solution

Implement a WhatsApp-owned media burst path that preserves every rapid local media item as MediaPaths/MediaTypes in one inbound context, unless maintainers explicitly choose to expand the generic steering contract to media.

## Reproduction Assessment

Yes, at source level: two rapid WhatsApp image messages can be traced through media debounce rejection, single-item flushing, singular WhatsApp media context, and text-only active steering. I did not run a live WhatsApp forward-burst session.

## Solution Assessment

Unclear: WhatsApp-local coalescing is likely the narrowest owner-boundary fix, but maintainers should decide whether the intended contract is channel-local media grouping or generic media-aware steering.
