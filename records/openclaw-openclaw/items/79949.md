## Summary

Keep open: current main still has a source-backed path where rapid WhatsApp media messages bypass inbound debounce and active-run steering can only carry text, so later image attachments can be lost instead of reaching the LLM.

## What This Changes

The issue requests preserving all images from rapid WhatsApp multi-image forwards in one or otherwise visible LLM turn instead of losing later attachments during active-run handling.

## Best Possible Solution

Preserve all rapid WhatsApp image attachments by adding a WhatsApp-owned media burst path that emits MediaPaths/MediaTypes, or by explicitly extending queue steering to carry structured media if maintainers choose the generic contract.

## Reproduction Assessment

Yes, at source level: two rapid WhatsApp image messages bypass media debounce, WhatsApp currently carries only singular media fields into the agent context, and active steer accepts only text. I did not perform a live WhatsApp forward-burst run against current main.

## Solution Assessment

Unclear until maintainer review: WhatsApp-local media coalescing appears to be the narrowest owner-boundary fix, but making active-run steering media-aware would solve the alternate path generically.
