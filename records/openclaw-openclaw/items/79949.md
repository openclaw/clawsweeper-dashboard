- `impact:message-loss`: The reported second and later WhatsApp images are not surfaced to the LLM turn in steer mode.

## Summary

Keep open. Current main and v2026.6.8 still drop the media part of steered WhatsApp image bursts because the steer queue path carries text-only options, and the open linked PR is the active candidate fix rather than a reason to close before merge.

## What This Changes

Ensure rapid WhatsApp multi-image forwards in steer queue mode deliver every image attachment to the agent turn, either through WhatsApp media coalescing or media-aware steer injection.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/94217 if it passes PR review, or apply the same narrow existing-API fix so steered followup images reach AgentSession.steer without adding new WhatsApp config.

## Reproduction Assessment

Yes, at source level. WhatsApp media bypasses text debounce, FollowupRun can carry images, but the active steer path passes only text and text-only options into the embedded queue, so steered media attachments have nowhere to go.

## Solution Assessment

Yes for the narrow bug path. Threading existing FollowupRun images through the existing embedded steer queue and AgentSession.steer API is narrower than adding new WhatsApp batching/config for the observed attachment loss.
