## Summary

Keep open: current main and the latest shipped release still have a source-backed WhatsApp path where rapid media messages bypass coalescing and active-run steering carries only text, so later images can still be lost from the LLM turn.

## What This Changes

Request that WhatsApp preserve all images from rapid forwarded multi-image messages, either by coalescing media bursts or by steering media attachments into an active turn.

## Best Possible Solution

Implement a WhatsApp-owned media burst path that preserves every local media item as MediaPaths/MediaTypes in one inbound context, or explicitly add a documented media-capable steering contract if maintainers choose the generic queue fix.

## Reproduction Assessment

Yes, at source level: two rapid WhatsApp image messages can be traced through media debounce rejection, single-item flushing, singular WhatsApp media context, and text-only active steering. I did not run a live WhatsApp forward-burst session.

## Solution Assessment

Unclear: WhatsApp-local coalescing is likely the narrowest owner-boundary fix, but maintainers should decide whether the intended contract is channel-local media grouping or generic media-aware steering.
