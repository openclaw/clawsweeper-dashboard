## Summary

Keep open: current main and the latest release still have a source-backed WhatsApp path where rapid image messages bypass media coalescing and active-run steering only forwards text, so later images can still be lost before reaching the LLM.

## What This Changes

The issue requests that rapid multi-image WhatsApp forwards preserve all images in one LLM turn, or otherwise carry media attachments through active-run steering.

## Best Possible Solution

Add a WhatsApp-owned media burst path that preserves every local media item as MediaPaths and MediaTypes in one inbound context, unless maintainers intentionally choose and document a broader media-capable steering contract.

## Reproduction Assessment

Yes, at source level: two rapid WhatsApp image messages can be traced through media debounce rejection, single-item flushing, singular WhatsApp media context, and text-only active steering. I did not run a live WhatsApp forward-burst session.

## Solution Assessment

Unclear: WhatsApp-local coalescing is likely the narrowest owner-boundary fix, but maintainers should decide whether the intended contract is channel-local media grouping or generic media-aware steering.
