## Summary

Keep open: current main still has a source-backed path where rapid WhatsApp media messages bypass inbound debounce, WhatsApp contexts carry only singular media fields, and active-run steering passes only prompt text into the running LLM turn.

## What This Changes

The issue reports that rapid forwarded WhatsApp image messages are not all delivered to the LLM turn.

## Best Possible Solution

Define the intended runtime contract, then preserve all rapid WhatsApp image attachments either with a WhatsApp-owned media burst buffer or with a generic media-aware steering payload that keeps attachment context through the active run.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. Source inspection plus the reporter's logs give a credible current-main reproduction path: two rapid WhatsApp image messages bypass debounce, and the second message has no media-bearing steer path into the active run.

## Solution Assessment

Unclear until maintainer review. WhatsApp-local coalescing is likely the narrowest owner-boundary fix, but generic media-capable steering may also be appropriate because related active-run media metadata issues exist.
