## Summary

Keep open: current main still has a source-backed path where WhatsApp image messages bypass debounce and active-run steering carries only text, so rapid forwarded images can lose later attachment context; the related fix PRs are closed and unmerged.

## What This Changes

The issue asks for rapid multi-image WhatsApp forwards to preserve all images, either by WhatsApp media coalescing or by media-aware queue steering.

## Best Possible Solution

Preserve every rapid WhatsApp image by adding a WhatsApp-owned media burst path or an explicit media-aware steering contract, with regression coverage and redacted live WhatsApp proof that all images reach the agent.

## Reproduction Assessment

Yes, at source level: two rapid WhatsApp image messages can be traced through media debounce rejection, singular WhatsApp media context, and text-only active steering. I did not perform a live WhatsApp forward-burst run against current main.

## Solution Assessment

Unclear until maintainer review: WhatsApp-local coalescing is likely the narrowest owner-boundary fix, but changing default media timing versus extending core steer semantics is a product/runtime decision.
