## Summary

Keep open: current main still has a source-backed WhatsApp path where rapid media messages bypass inbound coalescing, flush as single-message turns, and active-run steering only passes text, so later forwarded images can still miss the LLM turn.

## What This Changes

The issue asks for WhatsApp forwarded media bursts to preserve all images in one LLM turn, or for active-run steering to carry media attachments instead of text only.

## Best Possible Solution

Implement a WhatsApp-owned media burst path that preserves every rapid local media item as MediaPaths/MediaTypes for one inbound turn, unless maintainers explicitly choose a generic media-aware steering contract.

## Reproduction Assessment

Yes, at source level: two rapid WhatsApp image messages can be traced through media debounce rejection, single-item flushing, singular WhatsApp media context, and text-only active steering. I did not run a live WhatsApp forward-burst session.

## Solution Assessment

Unclear: WhatsApp-local coalescing looks like the narrowest owner-boundary fix, but maintainers should decide whether the intended contract is channel-local media grouping or generic media-capable steering.
