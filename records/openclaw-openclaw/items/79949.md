## Summary

Keep open: current main still has the source-backed failure path described by the reporter. WhatsApp media bypasses inbound debounce, the WhatsApp context exposes only singular media fields, and active-run steering still queues text rather than structured attachments.

## What This Changes

The issue requests that rapid multi-image WhatsApp forwards reach the LLM as all attached images, either through WhatsApp media coalescing or media-aware queue steering.

## Best Possible Solution

Implement a WhatsApp-owned short media burst path that preserves all local media as MediaPaths/MediaTypes, or deliberately extend steering with a media-capable contract if maintainers want this solved generically.

## Reproduction Assessment

Yes at source level: two rapid WhatsApp image messages can be traced through media debounce rejection, singular WhatsApp media context, and text-only active steering. I did not perform a live WhatsApp forwarded-image burst against current main.

## Solution Assessment

Unclear as an implementation choice: WhatsApp-local coalescing is likely the narrowest owner-boundary fix, but maintainers should choose it over broader media-aware steering before changing runtime behavior.
