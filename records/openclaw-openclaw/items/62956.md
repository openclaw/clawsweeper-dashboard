## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Sanitize raw chat error logging:** `src/tui/tui-event-handlers.ts:380-382`
  - body: This new debug log writes `evt.errorMessage` directly. `logDebug` sends messages to the file logger path, and gateway chat errors can be raw `String(err)` provider payloads, so this can persist sensitive error details instead of using the existing formatted/sanitized path.
  - confidence: 0.78
- **[P2] Preserve elapsed for plain error rerenders:** `src/tui/tui.ts:676-678`
  - body: This branch clears `errorElapsed` when the previous status is the plain `error` label because it only treats `error:` labels as still-error rerenders. A connection-status rerender while `activityStatus` remains `error` will therefore drop the elapsed suffix the PR is meant to preserve.
  - confidence: 0.89
- **[P2] Keep optimistic binding until a run is claimed:** `src/tui/tui.ts:581-582`
  - body: When the stale timeout fires during `waiting` before the first gateway event, `activeChatRunId` can still be null and `pendingOptimisticUserMessage` is the only state that lets that first event be marked local. Clearing it here can misclassify a slow-start local turn as external and trigger avoidable history reloads.
  - confidence: 0.84
