  - confidence: 0.88

  - body: When `activityStatus` remains plain `"error"`, `lastActivityStatus` is also plain `"error"`, which does not start with `"error:"`; a connection-status rerender then clears `errorElapsed` and drops the suffix. Treat plain error as an error state when deciding whether to preserve the cached elapsed value.
  - confidence: 0.89
- **[P2] Keep optimistic binding until a run is claimed:** `src/tui/tui.ts:582`
  - body: If the stale timeout fires during `waiting` before the first chat event, `activeChatRunId` can still be null and `pendingOptimisticUserMessage` is the only local-run binding. Clearing it here can make the eventual run look external and trigger avoidable history reload behavior.
  - confidence: 0.82

  - body: This branch calls `loadHistory()` while `state.activeChatRunId` is still the same run. `loadHistory()` clears and rebuilds the chat log, so a sparse but still-live run can lose in-flight output without terminal completion evidence.
  - confidence: 0.84
