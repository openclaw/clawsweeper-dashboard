## Summary

Keep open. Current main still provides fresh isolated cron run sessions plus retention-based pruning, but it does not expose the requested completion-based ephemeral cleanup mode for recurring cron jobs or hook mappings; the open hook-cleanup tracker covers only part of this request, and the earlier closed cron issue covered fresh-per-run behavior rather than immediate disposal.

## What This Changes

Add an opt-in ephemeral session mode for cron jobs and hook mappings that cleans up completed run-owned sessions immediately instead of waiting for retention pruning.

## Best Possible Solution

Add an explicit opt-in lifecycle policy for cron and hook agent runs that deletes only the completed run-owned session after terminal completion, preserves cron run logs, defines transcript archive/delete semantics, and coordinates with hook cleanup and live-session safeguards.

## Reproduction Assessment

Not applicable as a strict bug reproduction, but source inspection is high-confidence: the current cron and hook contracts have no `ephemeral` field, and cleanup still flows through retention or narrow `deleteAfterRun` paths.

## Solution Assessment

Unclear until maintainers define lifecycle semantics. An explicit opt-in policy is the right direction, but transcript handling, failure behavior, and live-session protections need product approval before implementation.
