## Summary

Keep open. Current main and v2026.5.22 still only provide retention-based cron cleanup and strict hook mapping config; there is no opt-in completion-based disposal mode, and related items cover hook-only cleanup or broader retention rules rather than the combined cron and hook lifecycle decision.

## What This Changes

Request an opt-in ephemeral session mode for one-shot cron and hook-triggered agent runs that removes session entries and defines transcript handling on completion.

## Best Possible Solution

Define one additive opt-in lifecycle policy for cron and hook one-shot runs that deletes only completed run-owned session rows, preserves cron run logs, documents transcript archive/delete behavior, and coordinates with existing hook and session-retention work.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high confidence that current main cannot express the requested completion-based cron or hook cleanup mode.

## Solution Assessment

Unclear until maintainers choose lifecycle semantics. The maintainable direction is an explicit opt-in cleanup policy rather than overloading cron.sessionRetention, generic session.maintenance, or cron deleteAfterRun.
