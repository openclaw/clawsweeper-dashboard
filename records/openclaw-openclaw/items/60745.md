## Summary

Keep open. Current main still has retention-based cron session pruning, adjacent `deleteAfterRun` cleanup for one-shot jobs, and hook dispatch through isolated cron-style runs, but it does not expose the requested opt-in completion-based cleanup mode for cron jobs or hook mappings.

## What This Changes

The issue requests an opt-in ephemeral lifecycle mode that removes or archives one-shot cron and hook sessions when their run completes instead of waiting for retention cleanup.

## Best Possible Solution

Define one additive opt-in lifecycle policy for cron and hook one-shot runs that deletes or archives only completed run-owned session rows, preserves cron run logs, documents transcript behavior, and coordinates with existing hook and cron retention settings.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high confidence that current main cannot express the requested completion-based cron or hook cleanup mode.

## Solution Assessment

Unclear until maintainers choose lifecycle semantics. The maintainable direction is an explicit opt-in cleanup policy rather than overloading `cron.sessionRetention`, generic `session.maintenance`, or `deleteAfterRun`.
