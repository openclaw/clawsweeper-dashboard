- `impact:session-state`: The request directly changes lifecycle, persistence, and cleanup behavior for cron and hook-triggered session rows.

## Summary

Keep open. Current main still has retention-based cron cleanup and adjacent deleteAfterRun behavior, but it does not expose an opt-in completion-based ephemeral cleanup mode for cron jobs or hook mappings.

## What This Changes

Request an ephemeral session mode that removes one-shot cron and hook-triggered session entries, with defined transcript handling, when the agent run completes.

## Best Possible Solution

Define one additive opt-in lifecycle policy for cron and hook one-shot runs that deletes only completed run-owned session rows, preserves cron run logs, documents transcript archive/delete behavior, and coordinates with the existing hook and cron lifecycle trackers.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high confidence that current main cannot express the requested completion-based cron or hook cleanup mode.

## Solution Assessment

Unclear until maintainers choose lifecycle semantics. The maintainable direction is an explicit opt-in cleanup policy rather than overloading cron.sessionRetention, generic session.maintenance, or cron deleteAfterRun.
