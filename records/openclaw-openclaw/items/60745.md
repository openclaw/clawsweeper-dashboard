## Summary

Keep open. Current main has adjacent retention and one-shot cleanup primitives, but it does not provide the requested explicit ephemeral mode for recurring cron runs or hook mappings, and the hook portion is only partially covered by the separate open hook-cleanup issue.

## What This Changes

The issue requests an ephemeral session mode for cron jobs and hook-triggered agent runs so completed one-shot sessions can be removed immediately rather than waiting for retention pruning.

## Best Possible Solution

Design an additive lifecycle contract that covers cron run-session cleanup and hook session cleanup separately, with schema/CLI/docs/tests and explicit transcript retention behavior.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection confirms the requested ephemeral fields and hook post-completion cleanup path are absent on current main.

## Solution Assessment

Unclear. A simple boolean may be enough for cron run cleanup, but hook cleanup and transcript handling need maintainer-defined lifecycle semantics before code changes.
