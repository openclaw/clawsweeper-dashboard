## Summary

Keep open. Current main still has fresh isolated cron runs and retention-based pruning, but it does not expose the requested completion-based `ephemeral` cleanup mode for recurring isolated cron jobs or hook mappings.

## What This Changes

Add an opt-in ephemeral cleanup mode for isolated cron jobs and hook mappings that removes completed run session entries immediately instead of waiting for retention timers.

## Best Possible Solution

Add an explicit, opt-in lifecycle policy scoped to run-owned isolated cron sessions and hook agent mappings, preserve cron run logs, and define transcript archive/delete semantics while coordinating the hook-wide policy with https://github.com/openclaw/openclaw/issues/10253.

## Reproduction Assessment

Yes, source inspection is enough for this feature gap: current main lacks `ephemeral` in the cron and hook config surfaces, and the cleanup paths still use retention timers or narrower `deleteAfterRun` behavior.

## Solution Assessment

Unclear until maintainers choose the public lifecycle semantics. The requested opt-in completion-cleanup primitive is a maintainable direction if it is limited to run-owned isolated sessions and does not reuse destructive cleanup for live session bindings.
