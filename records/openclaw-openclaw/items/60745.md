## Summary

Keep open. Current main still has fresh isolated cron run records plus retention pruning, but it does not expose the requested completion-based ephemeral cleanup mode for cron jobs or hook mappings.

## What This Changes

The issue requests an opt-in ephemeral session mode that deletes or archives completed cron and hook-triggered session entries immediately after an agent turn completes.

## Best Possible Solution

Add a public opt-in lifecycle policy that deletes only the exact completed run-owned session entry, preserves cron run logs, and documents transcript archive/delete behavior for cron and hook sessions.

## Reproduction Assessment

Yes, as a source-level feature gap. Current main exposes retention-based cron cleanup and strict hook mappings, but no ephemeral cron or hook cleanup field and no hook post-run deletion/archive finalizer.

## Solution Assessment

Unclear until maintainers choose the lifecycle semantics. The maintainable direction is an explicit opt-in cleanup policy rather than broadening deleteAfterRun or relying only on generic retention timers.
