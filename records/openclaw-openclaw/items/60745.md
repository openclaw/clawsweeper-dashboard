## Summary

Keep open. Current main still provides retention-based cleanup for cron run sessions and adjacent deleteAfterRun cleanup, but it does not expose an opt-in completion-based ephemeral cleanup mode for recurring cron jobs or hook mappings.

## What This Changes

Request an opt-in ephemeral session lifecycle for cron and hook agent runs that removes completed session entries and handles transcript cleanup immediately after completion.

## Best Possible Solution

Define one additive opt-in lifecycle policy for cron and hook runs that deletes only completed run-owned session rows, preserves cron run logs, documents transcript archive/delete behavior, and coordinates with https://github.com/openclaw/openclaw/issues/10253.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high confidence current main cannot express the requested ephemeral cron or hook cleanup mode and still relies on retention or adjacent deleteAfterRun behavior.

## Solution Assessment

Unclear until maintainers choose lifecycle semantics. The maintainable direction is an explicit opt-in cleanup policy rather than overloading cron.sessionRetention, generic session.maintenance, or cron deleteAfterRun.
