## Summary

Keep open. Current main still exposes retention-based cron cleanup and adjacent `deleteAfterRun` cleanup, but it does not provide the requested public completion-based `ephemeral` mode for recurring isolated cron jobs or hook mappings.

## What This Changes

Request an opt-in ephemeral session mode for cron jobs and hook mappings that removes completed one-shot session entries and transcript state immediately instead of waiting for retention pruning.

## Best Possible Solution

Add a public opt-in `ephemeral` cleanup policy for isolated cron jobs and hook agent mappings, delete only the exact completed run-owned session entry after terminal completion, preserve cron run logs, and document transcript archive/delete behavior.

## Reproduction Assessment

Yes. A source-level reproduction is enough: current strict cron and hook config surfaces omit `ephemeral`, hook mappings reject unknown fields, and completed isolated cron cleanup still flows through retention or narrow `deleteAfterRun` paths.

## Solution Assessment

Yes, the requested completion-cleanup primitive is the right maintainable direction if it is opt-in and scoped to owned isolated sessions. The safer alternative is not to broaden existing `deleteAfterRun` cleanup to arbitrary live session targets.
