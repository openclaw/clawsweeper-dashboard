## Summary

Keep open. Current main still exposes retention-based cron cleanup and strict hook mapping config, but no opt-in completion-based session disposal mode; the open hook-specific related issue does not cover the cron half of this request.

## What This Changes

Request an opt-in ephemeral lifecycle mode for cron and hook-triggered agent runs that removes or archives completed one-shot sessions immediately instead of waiting for retention.

## Best Possible Solution

Define one additive opt-in lifecycle policy for cron and hook one-shot runs that deletes only completed run-owned session rows, preserves cron run logs, documents transcript archive/delete behavior, and coordinates with existing hook and cron lifecycle trackers.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high confidence that current main cannot express the requested completion-based cron or hook cleanup mode.

## Solution Assessment

Unclear until maintainers choose lifecycle semantics. The maintainable direction is an explicit opt-in cleanup policy rather than overloading cron.sessionRetention, generic session.maintenance, or cron deleteAfterRun.
