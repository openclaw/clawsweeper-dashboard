## What This Changes

The issue reports isolated cron agent runs being aborted by the pre-execution watchdog even while setup/context phases report progress.

## Best Possible Solution

Land an adjusted watchdog-layer fix that treats known setup progress after runner start as progress while preserving the runner-start/no-phase stall guard, then close this issue from the merged PR.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an isolated cron agentTurn that reports only setup phases after runner start remains in waiting_for_execution until the pre-execution watchdog fires.

## Solution Assessment

No, not as the exact predicate proposed in the issue, because runner_entered alone must still keep the guard armed. The best fix remains in the watchdog transition, but it should distinguish runner-start from later setup progress.
