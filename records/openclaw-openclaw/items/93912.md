## Summary

Keep open: current main and the latest release still classify setup-like cron agent phases as pre-execution and only clear the 60-second guard for execution-stage progress or firstModelCallStarted. The linked closing PR is still open, so this issue should wait for a merged watchdog fix rather than be closed now.

## What This Changes

The issue asks for isolated cron setup/context phase progress to clear the pre-execution watchdog while preserving the no-progress runner-start guard.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/93914 or an equivalent narrow watchdog-layer fix that counts setup progress without removing the true no-progress runner-start guard.

## Reproduction Assessment

Yes, source-reproducible: current main maps setup phases to pre_execution and only clears the guard for execution-stage progress or firstModelCallStarted. Current regression coverage also expects context_engine after runner_entered to hit the pre-execution stalled path.

## Solution Assessment

Yes in direction: the maintainable fix belongs in the cron watchdog progress transition, not provider-specific behavior or a new config option. The current open PR still needs to land, or be replaced by an equivalent reviewed fix, before this issue can close.
