## Summary

Keep open. Current main has useful partial controls for isolated cron runs, lightweight context, model/fallback selection, timeouts, and tool allow-lists, but it still lacks the requested first-class scoped/script-limited mode with script registry, dry-run cost preview, per-job token/spend caps, daily budget aborts, and planned-vs-actual run logs. The existing maintainer discussion also explicitly kept this open and treated #38248 as narrower cost-guard work, not a replacement for this broader tracker.

## What This Changes

The issue requests a first-class scoped/script-limited agent mode with script allowlists, cron-first execution, token/spend caps, dry-run previews, minimal context, and structured run logs.

## Best Possible Solution

Design a native scoped automation mode that adds explicit script-only execution and budget/accounting contracts across cron payload/schema, hook mappings, CLI/UI/docs, isolated-run enforcement, and run-history telemetry while reusing any narrower cost-guard work from #38248.

Not applicable. This is a feature request, not a broken existing behavior report; source and docs inspection show the current partial cron/hooks controls rather than a failing current-main reproduction path.

## Solution Assessment

No. The existing controls are useful but are not the best complete solution for the requested scoped/script-limited mode because the missing script registry, dry-run cost preview, token/spend ceilings, daily abort policy, and planned-vs-actual logs require a designed product/API change.
