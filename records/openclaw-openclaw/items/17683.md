## Summary

Keep open: current main still only has partial cron/hook controls, and the requested scoped/script-limited automation mode would add new public protocol/config, cost-budget enforcement, dry-run planning, persisted script policy, and run accounting that are not implemented.

## What This Changes

Request for a cost-controlled scoped agent mode with admin-approved scripts, cron-first execution, token/spend ceilings, fresh-context jobs, dry-run cost previews, and structured run logs.

## Best Possible Solution

Design one native scoped automation mode across cron/hook protocol, config, CLI/UI/docs, script policy, cost-budget admission, dry-run planning, and run accounting, coordinated with https://github.com/openclaw/openclaw/issues/18160 and https://github.com/openclaw/openclaw/issues/42475.

## Reproduction Assessment

Not applicable. This is a feature request, and current-main source inspection gives high confidence that the requested first-class scoped/script-limited mode is absent rather than a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the product and security shape. The requested direction is plausible, but treating toolsAllow, direct exec, or budget enforcement alone as the complete solution would leave meaningful parts of the request unsolved.
