## Summary

Keep open. Current main still has partial cron and hook controls, but not a first-class scoped/script-limited mode with script registry, dry-run cost preview, per-job token/spend ceilings, daily abort policy, or planned-vs-actual run logs.

## What This Changes

The issue requests a scoped, cron-first agent execution mode with admin-approved scripts, hard budget controls, minimal context, and transparent run logging.

## Best Possible Solution

Design one native scoped automation mode that adds explicit script-only execution and budget/accounting contracts across cron payload/schema, hook mappings, CLI/UI/docs, isolated-run enforcement, and run-history telemetry.

## Reproduction Assessment

Not applicable. This is a feature request rather than a broken existing behavior report; source and docs inspection show a high-confidence current-main feature gap, not a failing reproduction path.

## Solution Assessment

Unclear until maintainers choose the product shape. The requested direction is plausible, but the durable implementation needs one coordinated cron/hooks/config/accounting design rather than treating the current tools allow-list or the cost-only tracker as complete.
