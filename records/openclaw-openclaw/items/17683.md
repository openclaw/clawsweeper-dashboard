## Summary

Keep open: current main still has only partial isolated cron controls, while the requested first-class scoped/script-limited mode remains a broader product and security design that related direct-exec, budget, and schema issues cover only in pieces.

## What This Changes

The issue requests a native scoped/script-limited automation mode that bounds scheduled or hook-triggered agent execution by scripts, tools, context, token budgets, spend ceilings, dry-run planning, and structured run logs.

## Best Possible Solution

Design one native scoped automation mode that reuses the narrower direct-exec and gateway-budget work while adding clear script policy, budget admission, dry-run planning, run accounting, CLI/schema/docs, and security review.

Not applicable as a bug reproduction; this is a feature/product request. Source inspection gives high confidence that current main lacks a first-class scoped/script-limited mode rather than violating an existing contract.

## Solution Assessment

Unclear until maintainers choose the product and security shape. Existing isolated cron sessions, lightContext, toolsAllow, timeouts, model/fallback selection, and post-run cost telemetry solve only parts of the requested mode.
