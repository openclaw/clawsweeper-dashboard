## Summary

Keep open: current main and the latest release still provide only partial cron, hook, lightweight-context, tool allow-list, timeout, model/fallback, and post-run usage controls, while the requested scoped/script-limited mode remains a broader product and security design. ## What This Changes

The issue requests a first-class scoped/script-limited agent mode with admin-approved scripts, deterministic cron-style execution, token and spend ceilings, dry-run cost previews, minimal context, and structured run transparency.

## Best Possible Solution

Design one native scoped automation mode that coordinates the direct-exec and gateway-budget tracks while adding script policy, budget admission, dry-run planning, run accounting, CLI/schema/docs, and security review.
