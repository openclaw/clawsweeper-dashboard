## Summary

Keep open. Current main still has only partial cron/hook controls for isolated agent runs; it does not provide a first-class scoped/script-limited mode with an admin script registry, dry-run/cost preview, token or spend ceilings, budget-abort policy, or planned-vs-actual run logs, and the maintainer discussion already treats #38248 as only a narrower cost-guard component.

## What This Changes

Request a scoped, script-limited agent mode for predictable cron-first automation with whitelisted scripts, token/spend controls, minimal context, dry-run planning, and structured execution logs.

## Best Possible Solution

Design this as a native scoped automation mode across cron payload/schema, hook mappings, CLI/UI/docs, isolated-run accounting, and tests, while sharing the cost-guard model with #38248 instead of duplicating budget logic.

## Reproduction Assessment

Not applicable. This is a feature request for a new scoped/script-limited execution mode, and source inspection confirms current main has only partial cron/hook controls rather than the requested mode.

## Solution Assessment

Unclear until maintainers settle the product/API design. The requested direction is coherent, but the maintainable solution should be a first-class mode with shared cost-guard primitives, not a one-off script or token flag added in isolation.
