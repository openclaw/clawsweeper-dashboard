## Summary

The issue requests a first-class scoped/script-limited automation mode with whitelisted scripts, cron-first execution, fresh context, token/spend ceilings, dry-run cost preview, and structured run logs.

## Best Possible Solution

Design one native scoped automation mode that reuses the direct-exec and gateway-budget tracks while adding script policy, budget admission, dry-run planning, run accounting, CLI/schema/docs, and security review.
