## Summary

Keep open: current main still has only partial cron controls for lightweight context, timeouts, model/fallback selection, and tool allow-lists, while the requested first-class scoped/script-limited mode remains unimplemented. The live discussion also explicitly keeps this as the broader product/API tracker, with spend-budget enforcement coordinated through the separate open gateway budget item.

## What This Changes

Request a first-class scoped/script-limited automation mode with vetted scripts, cron-first execution, fresh context, token/spend ceilings, dry-run cost previews, and structured run logs.

## Best Possible Solution

Design one native scoped automation mode across cron/hook protocol, CLI/config/docs, script policy, budget admission, dry-run planning, and run accounting, sharing the spend-budget mechanism with https://github.com/openclaw/openclaw/issues/42475.

Not applicable. This is a feature request, and source inspection gives high confidence that current main lacks a first-class scoped/script-limited mode rather than breaking an existing contract.

## Solution Assessment

Unclear until maintainers choose the product and security shape. Current isolated cron sessions, light context, toolsAllow, timeouts, and post-run cost telemetry solve only parts of the requested mode.
