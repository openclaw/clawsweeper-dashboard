## Summary

Keep open. Current main has partial controls for cron-triggered isolated agent runs, lightweight context, timeout, model selection, and tool allow-lists, but it does not implement the requested first-class scoped/script-limited mode with an admin script registry, dry-run cost preview, per-job token/spend ceilings, daily budget aborts, or planned-vs-actual run logs; a maintainer comment also explicitly kept this open and treated #38248 as narrower cost-guard work.

## What This Changes

The issue requests a scoped, script-limited automation mode for cron-first, cost-controlled agent execution with whitelisted scripts, hard token/spend budgets, minimal context, dry-run previews, and structured logs.

## Best Possible Solution

Keep this as the broader scoped automation product/API tracker and design a native mode that combines admin-defined script-only execution, dry-run/cost preview, per-job token/spend caps, daily ceilings, graceful budget aborts, and planned-vs-actual run logs while coordinating shared cost guards with #38248.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing behavior report. Source inspection gives high confidence that the requested first-class scoped/script-limited surfaces are not present on current main.

## Solution Assessment

Yes for triage: keeping it open for a designed native mode is the best path. Closing as implemented would be incorrect because current cron/hook controls are useful but materially narrower than the requested script registry, budgets, dry-run preview, and run-log contract.
