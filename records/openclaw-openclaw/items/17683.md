## Summary

Keep open: current main still provides only partial cron controls such as isolated runs, lightweight context, tool allow-lists, timeouts, and post-run usage telemetry, while the requested scoped/script-limited mode remains unimplemented. The live discussion already kept this as the broader product/API tracker separate from the narrower gateway budget request at https://github.com/openclaw/openclaw/issues/42475.

## What This Changes

The issue requests a first-class scoped/script-limited automation mode with admin-defined scripts, deterministic cron execution, minimal context, token/spend ceilings, dry-run cost preview, and structured run transparency.

## Best Possible Solution

Design one native scoped automation mode across cron/hook protocol, config/CLI/docs, script policy, budget admission, dry-run planning, and run accounting, sharing the spend-budget mechanism with https://github.com/openclaw/openclaw/issues/42475.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection gives high confidence that current main lacks a first-class scoped/script-limited mode rather than breaking an existing contract.

## Solution Assessment

Unclear until maintainers choose the product and security shape. The direction is plausible, but current toolsAllow, isolated cron sessions, light context, and post-run cost telemetry each solve only part of the requested mode.
