## Summary

Keep open. Current main still has only partial cron/hook controls, while the requested scoped/script-limited mode spans new public payload/config shape, persisted script policy, budget enforcement, dry-run cost preview, and planned-vs-actual accounting that are not implemented and require maintainer design.

## What This Changes

The issue requests a first-class scoped/script-limited automation mode for predictable cron/script execution with strict tool/script scope, token/spend ceilings, cron-first runs, minimal context, and transparent run logs.

## Best Possible Solution

Design one native scoped automation mode across cron/hook protocol, config, CLI/UI/docs, isolated-run accounting, host-exec policy, and shared budget enforcement, coordinating with the narrower direct-exec and budget trackers.
