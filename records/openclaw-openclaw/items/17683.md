## Summary

Keep open: current main has useful partial controls for isolated cron execution, tool allow-lists, lightweight context, timeouts, delivery/run diagnostics, and post-run cost telemetry, but it still lacks the requested first-class scoped/script-limited mode and the issue discussion has already kept this as the broader product/API tracker.

## What This Changes

Request a first-class scoped/script-limited automation mode for cron or hook-triggered agent runs with admin-approved scripts, dry-run cost previews, token and spend ceilings, fresh context, and structured run logs.

## Best Possible Solution

Design one native scoped automation mode across cron/hook protocol, config/CLI/docs, script policy, budget admission, dry-run planning, and run accounting, sharing the spend-budget mechanism with https://github.com/openclaw/openclaw/issues/42475.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection gives high confidence that current main lacks a first-class scoped/script-limited mode rather than breaking an existing contract.

## Solution Assessment

Unclear until maintainers choose the product and security shape. The direction is plausible, but current toolsAllow, direct exec approvals, light context, and cost telemetry each solve only part of the requested mode.
