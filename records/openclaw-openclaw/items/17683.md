## Summary

Keep open: current main and v2026.6.5 provide useful partial controls such as command cron, isolated runs, light context, tool allow-lists, timeouts, delivery, and post-run cost telemetry, but they still do not implement the requested first-class scoped/script-limited mode with an admin script registry, dry-run cost preview, per-job token/spend ceilings, daily budget aborts, and hook-level scoped execution semantics. The open gateway budget tracker at https://github.com/openclaw/openclaw/issues/42475 is related but does not supersede this broader product and security design.

## What This Changes

The issue requests a first-class scoped/script-limited agent mode with cron-first execution, script whitelisting, token and spend ceilings, dry-run planning, minimal context, and structured run transparency.

## Best Possible Solution

Design one native scoped automation contract that coordinates cron command payloads, hook dispatch, script policy, budget admission, dry-run planning, run accounting, CLI/schema/docs, and security review.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/product request. Source inspection gives high confidence that current main lacks the requested first-class scoped/script-limited mode even though command cron has shipped.

## Solution Assessment

Unclear until maintainers choose the public product and security contract. Existing command cron, isolated cron sessions, lightContext, toolsAllow, timeouts, model/fallback selection, and cost telemetry solve only parts of the requested mode.
