## Summary

Keep open. Current main has useful cron and hook controls, including isolated runs, light context, model/fallback selection, timeouts, and tool allow-lists, but it still does not provide the requested first-class scoped/script-limited mode with a script registry, dry-run cost preview, token/spend ceilings, daily budget aborts, or planned-vs-actual logs.

## What This Changes

Request a native scoped/script-limited automation mode with admin-whitelisted scripts, dry-run cost preview, token and spend ceilings, cron-first execution, fresh context, and structured run logs.

## Best Possible Solution

Design a native scoped automation mode that adds explicit script-only execution and budget/accounting contracts across cron payload/schema, hook mappings, CLI/UI/docs, isolated-run enforcement, and run-history telemetry.

## Reproduction Assessment

Not applicable. This is a feature request, not a broken existing behavior report; source and docs inspection show partial current controls rather than a failing current-main reproduction path.

## Solution Assessment

No. The existing controls are useful but are not a complete solution because script registry, dry-run cost preview, per-job token/spend ceilings, daily abort policy, and planned-vs-actual logs require a designed product/API change.
