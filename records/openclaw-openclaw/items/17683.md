## Summary

Keep open: current main still has partial cron/hook controls, but the requested scoped/script-limited mode would add new public protocol/config, persisted script policy, pre-dispatch budget enforcement, dry-run planning, and run accounting that are not implemented.

## What This Changes

This issue requests a first-class scoped/script-limited automation mode with whitelisted scripts, cron-first execution, token/spend ceilings, dry-run cost preview, minimal context, and structured run logs.

## Best Possible Solution

Design one native scoped automation mode across cron/hook protocol, config, CLI/UI/docs, script policy, cost-budget admission, dry-run planning, and run accounting, coordinated with the narrower direct-exec and budget trackers.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection gives high confidence that current main lacks the requested first-class scoped/script-limited mode rather than breaking an existing contract.

## Solution Assessment

Unclear until maintainers choose the product and security shape. The direction is plausible, but toolsAllow, direct exec, or budget enforcement alone would leave meaningful parts of the requested mode unsolved.
