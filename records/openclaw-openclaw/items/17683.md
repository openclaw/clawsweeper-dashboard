## Summary

Keep open. Current main has partial cron controls, isolated fresh-session runs, model/fallback/timeout/light-context/tool allow-list options, and usage reporting, but it still lacks the requested first-class scoped/script-limited mode with admin script registry, dry-run/cost preview, per-job token or spend ceilings, daily budget aborts, and planned-vs-actual run logs.

## What This Changes

The issue requests a scoped/script-limited agent mode for predictable cron-style execution with explicit script allow-lists, token and spend ceilings, dry runs, minimal context, and structured execution logs.

## Best Possible Solution

Keep this as the broad scoped automation product/API tracker and coordinate the design with https://github.com/openclaw/openclaw/issues/42475, https://github.com/openclaw/openclaw/issues/18160, https://github.com/openclaw/openclaw/issues/64721, and https://github.com/openclaw/openclaw/issues/13617 so cost admission, direct execution, resource limits, dry-run behavior, and logging share one contract.

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks the requested first-class scoped/script-limited mode and only provides partial cron controls.

## Solution Assessment

Unclear until maintainers settle the product and security contract. The best direction is a native, coordinated scoped automation mode, not a schema-only or cron-only patch that duplicates the open direct-exec, budget, and dry-run trackers.
