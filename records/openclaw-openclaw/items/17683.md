## Summary

Keep open. Current main and v2026.6.8 have useful partial primitives for cron command jobs, per-job tool allow-lists, lightweight context, timeouts, hook-triggered runs, pre-model plugin blocking, and usage telemetry, but they still do not provide the requested first-class scoped/script-limited automation mode.

## What This Changes

This issue requests a scoped/script-limited automation mode with admin-defined scripts, cron-first execution, strict tool and context limits, token/spend ceilings, dry-run cost preview, and structured run transparency.

## Best Possible Solution

Keep this open as the umbrella product/API tracker, then design shared scoped-automation primitives across cron, hooks, command jobs, tool policy, budget admission, context handling, run receipts, docs, and upgrade tests.

Not applicable as a bug reproduction: this is a feature/product request. Source inspection confirms current main has partial cron, hook, tool, context, and usage primitives but no first-class scoped/script-limited mode contract.

## Solution Assessment
