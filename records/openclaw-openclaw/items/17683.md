## Summary

Current main provides several building blocks for predictable automation, including isolated agent jobs, direct command jobs, light context, tool allow-lists, timeouts, bounded output, and post-run telemetry. It still lacks the requested unified scoped mode with an admin-owned script registry, script-only policy, pre-execution cost preview, token or spend enforcement, daily ceilings, and explicit persistence semantics; the narrower open budget tracker does not supersede that broader product and security design.

## What This Changes

Add a native scoped/script-limited agent mode for cron-triggered, script-whitelisted, low-context execution with token and spend limits, dry-run previews, and structured run accounting.

## Best Possible Solution

Define one canonical scoped-job contract shared by cron and hook ingress that references an admin-owned script registry, uses fresh isolated context by default, applies existing tool and execution policy, performs explicit pre-dispatch budget reservation or enforcement, and records both estimated and actual usage without creating a parallel agent runtime.

## Solution Assessment

Unclear as a final API: the desired outcomes are valid, but maintainers must choose the script trust model, budget enforcement guarantees, accounting semantics, and whether cron and hooks consume one shared scoped-job contract.
