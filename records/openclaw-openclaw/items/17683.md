## Summary

Keep open: current main still only has partial cron and isolated-agent controls, not the requested first-class scoped/script-limited mode. Existing maintainer discussion also keeps this as the broader product/security tracker while related budget and cron tool-policy work remains open separately.

## What This Changes

The issue requests a scoped agent mode with admin-defined scripts, deterministic cron-style execution, hard token/spend limits, minimal context, dry-run planning, and structured run accounting.

## Best Possible Solution

Design one native scoped automation mode across cron/hook protocol, CLI/config/docs, script policy, budget admission, dry-run planning, and run accounting, sharing budget mechanics with https://github.com/openclaw/openclaw/issues/42475 and coordinating cron tool policy with https://github.com/openclaw/openclaw/issues/12736.

## Solution Assessment

Unclear until maintainers choose the product and security shape. Existing isolated cron sessions, light context, toolsAllow, timeouts, and post-run cost telemetry solve only parts of the requested mode.
