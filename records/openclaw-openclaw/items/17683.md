## Summary

Keep open. Current main still has partial cron/hook controls and post-run usage accounting, but it does not implement the requested first-class scoped/script-limited mode with script registry, dry-run cost preview, token/spend ceilings, daily abort policy, or planned-vs-actual run logs.

## What This Changes

The issue requests a scoped agent mode for deterministic cron/script execution with admin script allow-lists, token/spend ceilings, dry-run cost previews, minimal context, and transparent run logs.

## Best Possible Solution

Design a native scoped automation mode that adds explicit script-only execution and budget/accounting contracts across cron payload/schema, hook mappings, CLI/UI/docs, isolated-run enforcement, and run-history telemetry while reusing narrower cost-guard work where applicable.

Not applicable. This is a feature request rather than broken existing behavior; source and docs inspection show current partial cron/hooks controls but no failing current-main reproduction path.

## Solution Assessment

No. The current partial controls are useful, but they are not the best complete solution because the requested scoped/script-limited mode needs an explicit product/API design and new enforcement/accounting contracts.
