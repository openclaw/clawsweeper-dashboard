## Summary

Keep open: current main has partial cron and isolated-agent controls, but it still lacks the requested first-class scoped/script-limited mode with admin script registry, dry-run cost preview, pre-run budget admission, and planned-vs-actual run accounting. The live discussion and related open budget/tool-policy issues also support keeping this as a broader product/security design tracker.

## What This Changes

Request for a scoped/script-limited agent mode with cron-first execution, admin-approved scripts, token/spend ceilings, minimal context, and transparent run logs.

## Best Possible Solution

Design one native scoped automation mode across cron/hook protocol, CLI/config/docs, script policy, budget admission, dry-run planning, and run accounting, sharing spend-budget mechanics with https://github.com/openclaw/openclaw/issues/42475 and coordinating cron tool policy with https://github.com/openclaw/openclaw/issues/12736.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main lacks a first-class scoped/script-limited mode rather than violating an existing contract.

## Solution Assessment

Unclear until maintainers choose the product and security shape. Existing isolated cron sessions, light context, toolsAllow, timeouts, and post-run cost telemetry solve only parts of the requested mode.
