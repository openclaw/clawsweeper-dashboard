## Summary

Keep open. Current main still only has partial cron and hook controls; the requested scoped/script-limited mode would add new script policy, budget enforcement, dry-run preview, and run accounting semantics that are not implemented and need maintainer design.

## What This Changes

Requests a first-class scoped/script-limited automation mode for cron and hook-triggered work with whitelisted scripts, token and spend ceilings, minimal context, dry-run previews, and structured logs.

## Best Possible Solution

Design one native scoped automation mode across cron and hook protocol, config, CLI/UI/docs, isolated-run accounting, and host-exec policy, coordinating direct-exec work with https://github.com/openclaw/openclaw/issues/18160 and budget enforcement with https://github.com/openclaw/openclaw/issues/42475.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection on current main gives high confidence that the requested scoped/script-limited mode is absent rather than a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the product and security shape. The requested direction is plausible, but the maintainable solution needs one coordinated cron/hooks/config/accounting/host-exec design rather than treating toolsAllow, direct exec, or budget pieces as complete independently.
