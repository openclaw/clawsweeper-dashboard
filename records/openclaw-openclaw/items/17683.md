## Summary

Keep open: current main has partial cron controls, tool allow-lists, lightweight isolated runs, timeouts, model/fallback selection, and post-run cost telemetry, but it still lacks the requested first-class scoped/script-limited mode. The live discussion also keeps this as broader product/API work, with cost-budget enforcement tracked separately in https://github.com/openclaw/openclaw/issues/42475.

## What This Changes

The issue requests a first-class scoped/script-limited agent mode with whitelisted scripts, cron-first execution, token/spend caps, dry-run planning, minimal context, and structured run logs.

## Best Possible Solution

Design one native scoped automation mode across cron/hook protocol, config/CLI/docs, script policy, budget admission, dry-run planning, and run accounting, sharing spend-budget semantics with https://github.com/openclaw/openclaw/issues/42475.
