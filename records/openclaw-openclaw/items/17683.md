## Summary

Keep open. Current main has useful partial cron controls, but it still lacks the requested first-class scoped/script-limited mode with an admin script registry, dry-run cost preview, token/spend ceilings, and budget-abort semantics.

## What This Changes

The issue requests a scoped/script-limited automation mode for predictable cron-style execution, tool/script allow-listing, fresh context, cost controls, and run transparency.

## Best Possible Solution

Design one native scoped automation mode across cron/hook protocol, config, CLI/UI/docs, isolated-run accounting, and host-exec policy, coordinating with https://github.com/openclaw/openclaw/issues/18160, https://github.com/openclaw/openclaw/issues/42475, https://github.com/openclaw/openclaw/issues/41484, and https://github.com/openclaw/openclaw/issues/41346.

## Reproduction Assessment

Not applicable. This is a feature request rather than a broken existing behavior report, and source/docs inspection gives high confidence that current main lacks the requested scoped/script-limited mode.

## Solution Assessment

Unclear until maintainers choose the product shape. Existing controls such as `toolsAllow`, light context, model selection, and timeout are useful partial pieces, but the maintainable solution needs a coordinated cron/hooks/config/accounting/security design.
