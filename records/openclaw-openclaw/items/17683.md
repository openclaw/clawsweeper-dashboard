## Summary

Keep open: current main still has useful partial cron, hook, light-context, timeout, model/fallback, and tool allow-list controls, but it does not implement the requested first-class scoped/script-limited automation mode with an admin script registry, dry-run cost preview, token or spend ceilings, budget aborts, or planned-vs-actual run logs.

## What This Changes

The issue requests a scoped/script-limited automation mode with whitelisted scripts, cron-first execution, token and spend ceilings, dry-run planning, and structured execution logs.

## Best Possible Solution

Design one native scoped automation mode across cron and hook protocol, config, CLI/UI/docs, script policy, cost-budget admission, dry-run planning, and run accounting, coordinated with the narrower direct-exec and gateway-budget trackers.

## Reproduction Assessment

Not applicable. This is a feature request, and current-main source inspection gives high confidence that the requested first-class scoped/script-limited mode is absent rather than a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the product and security shape. The requested direction is plausible, but treating toolsAllow, direct exec, or budget enforcement alone as the complete solution would leave meaningful parts of the request unsolved.
