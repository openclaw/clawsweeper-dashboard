## Summary

Keep open. Current main has partial automation controls, but it still does not implement the requested first-class scoped/script-limited mode with an admin script registry, script-only execution, dry-run cost preview, per-job token/spend ceilings, daily budget aborts, or planned-vs-actual run logs. The issue discussion also explicitly kept this open as the broader scoped automation tracker, with #38248 only covering a narrower cost-guard piece.

## What This Changes

The issue requests a scoped/script-limited agent mode for cron-style automation with admin-defined scripts, bounded tools, fresh context, token and spend ceilings, dry-run cost previews, and structured execution logs.

## Best Possible Solution

Keep this as the broader scoped automation tracker and design a native mode across cron payload/schema, hook mappings, CLI/UI/docs, isolated-run accounting, and tests: admin-defined script-only execution, optional dry-run/cost preview, per-job token/spend caps, daily ceilings, graceful budget aborts, and structured planned-vs-actual run logs, coordinated with #38248.
