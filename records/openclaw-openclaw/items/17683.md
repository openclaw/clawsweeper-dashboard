## Summary

Keep open. Current main has useful partial cron and hook controls, but it still lacks the requested first-class scoped/script-limited mode: no admin script registry, script-only execution contract, dry-run cost preview, per-job token or spend ceilings, daily budget abort, or planned-vs-actual run log surface. The existing maintainer comment also explicitly kept this open as the broader tracker while treating #38248 as a narrower cost-guard component.

## What This Changes

Request a scoped automation mode for cron and hook-driven agent runs with whitelisted scripts, deterministic execution, token and spend ceilings, dry-run previews, fresh context, and structured run transparency.

## Best Possible Solution

Keep this as the umbrella tracker for a designed native scoped automation mode across cron payload/schema, hook mappings, CLI/UI/docs, isolated-run accounting, and tests, with #38248 providing or sharing the narrower spend-guard primitive.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is contract inspection on current main, which shows the requested scoped-mode fields and behaviors are absent from cron, hooks, CLI, docs, and isolated-run accounting.

## Solution Assessment

No, the existing partial controls are not the best complete solution because they only limit model/context/tools/timeout in ordinary isolated agent turns. The maintainable solution is an explicit native mode with script registry, dry-run/cost preview, budget enforcement, and structured planned-vs-actual logs.
