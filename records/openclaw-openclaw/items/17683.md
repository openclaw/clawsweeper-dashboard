## Summary

Current main partially addresses predictable automation with isolated agent turns, light context, strict tool allow-lists, bounded direct command jobs, timeouts, and run telemetry. It still lacks the requested unified policy with an admin-owned script registry, enforceable per-job and daily budgets, preflight cost estimates, explicit state-persistence controls, and planned-versus-actual audit records; the open cost-budget tracker covers only one part of that broader design.

## What This Changes

Requests a first-class scoped automation mode with script-only execution, hard cost and token limits, cron-first runs, minimal context, and structured execution transparency.

## Best Possible Solution

Compose the existing bounded command-job and isolated-agent primitives into one additive scoped-run contract, adding only an approved admin script registry, a shared gateway budget gate with auditable decisions, explicit persistence semantics, and planned-versus-actual run records.

## Reproduction Assessment

Not applicable; this requests a new execution and policy capability rather than reporting a failure of an established contract. Current-source inspection provides high-confidence evidence of which requested components exist and which remain absent.

## Solution Assessment

No, not as a separate parallel agent runtime. The maintainable approach is to reuse current cron command jobs, isolated light-context runs, and tool policies, then design the missing registry, budget, persistence, and audit contracts at their existing owner boundaries.
