## Summary

Keep open: current main still has partial cron controls for isolated runs, light context, timeouts, model selection, and tool allow-lists, but it does not provide the requested first-class scoped/script-limited mode with admin scripts, dry-run cost preview, per-job token/spend ceilings, daily budget aborts, or planned-vs-actual run logs. The live discussion and narrower related budget tracker also support keeping this as the broader product/security design item.

## What This Changes

Request a scoped, script-limited agent mode for predictable cron-driven automation with whitelisted scripts, token/spend ceilings, minimal context, dry-run previews, and execution logs.

## Best Possible Solution

Design one native scoped automation mode across cron/hook protocol, CLI/config/docs, script policy, budget admission, dry-run planning, and run accounting, coordinating narrower trackers such as https://github.com/openclaw/openclaw/issues/42475 instead of duplicating policy.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main lacks a first-class scoped/script-limited mode rather than violating an existing contract.

## Solution Assessment

Unclear until maintainers choose the product and security shape. Existing isolated cron sessions, light context, per-job toolsAllow, timeouts, model/fallback selection, and post-run cost telemetry solve only parts of the requested mode.
