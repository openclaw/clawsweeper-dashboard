## Summary

Keep open. Current main still provides only partial cron/hook controls, while the requested scoped/script-limited automation mode, cost guardrails, dry-run preview, and planned run accounting remain unimplemented and need maintainer design.

## What This Changes

Request a scoped/script-limited agent mode for cron-style automation with admin-defined scripts, bounded tools, token/spend ceilings, fresh context, dry-run cost preview, and structured run logs.

## Best Possible Solution

Design one native scoped automation mode across cron/hook protocol, config, CLI/UI/docs, isolated-run accounting, and host-exec policy, sharing cost-budget work with https://github.com/openclaw/openclaw/issues/42475 and direct-exec design with https://github.com/openclaw/openclaw/issues/18160.

## Reproduction Assessment

Not applicable. This is a feature request rather than a broken existing behavior report, and source inspection gives high confidence that current main lacks the requested scoped/script-limited mode.

## Solution Assessment

Unclear until maintainers choose the product shape. The requested direction is plausible, but the maintainable solution needs one coordinated cron/hooks/config/accounting/security design rather than treating toolsAllow, cost budgets, or direct exec as complete on their own.
