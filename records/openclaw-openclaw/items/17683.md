## Summary

Keep open: current main has useful partial cron controls for isolated execution, lightweight context, model/fallback selection, timeouts, and tool allow-lists, but it still lacks the requested first-class scoped/script-limited automation mode. Prior live review comments also kept this as the broader product/API tracker, with spend-budget enforcement coordinated through the separate open gateway budget item.

## What This Changes

Request a first-class scoped/script-limited automation mode with admin-defined scripts, cron-first execution, dry-run/cost previews, token/spend ceilings, minimal context, and structured run logs.

## Best Possible Solution

Design one native scoped automation mode across cron/hook protocol, CLI/config/docs, script policy, budget admission, dry-run planning, and run accounting, sharing spend-budget mechanics with https://github.com/openclaw/openclaw/issues/42475.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection gives high confidence that current main lacks a first-class scoped/script-limited mode rather than violating an existing contract.

## Solution Assessment

Unclear until maintainers choose the product and security shape. Existing isolated cron sessions, light context, toolsAllow, timeouts, and post-run cost telemetry solve only parts of the requested mode.
