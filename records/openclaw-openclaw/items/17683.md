## Summary

Keep open. Current main has useful partial cron and hook controls, but it still lacks the requested first-class scoped/script-limited mode, including an admin script registry, script-only execution contract, dry-run cost preview, per-job token/spend ceilings, daily budget aborts, and planned-vs-actual run logs; the existing maintainer/Codex discussion also keeps this as the broader tracker while treating #38248 as narrower cost-guard work.

## What This Changes

Request a first-class scoped/script-limited agent mode with whitelisted scripts, deterministic cron triggers, fresh context, token/spend ceilings, dry-run cost preview, and structured run logs.

## Best Possible Solution

Keep this as the umbrella tracker for a designed native scoped automation mode across cron payload/schema, hook mappings, CLI/UI/docs, isolated-run accounting, and tests, with #38248 sharing the spend-guard primitive.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source/docs contract inspection on current main, which shows the requested scoped-mode fields and behaviors are absent from cron, hooks, CLI, run logs, and isolated-run accounting.

## Solution Assessment

Unclear until maintainers decide the API. The existing partial controls are not a full solution because they only bound model/context/tools/timeouts and record usage after a run; the maintainable path is an explicit native mode rather than prompt conventions.
