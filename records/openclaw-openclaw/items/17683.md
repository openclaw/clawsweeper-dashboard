## Summary

Keep open: current main has useful cron controls, run logs, tool allow-lists, lightweight context, model selection, and timeout support, but it still lacks the requested first-class scoped/script-limited mode with persisted script policy, dry-run cost preview, per-job/daily budget enforcement, and planned-vs-actual accounting.

## What This Changes

The issue requests a scoped/script-limited agent mode for predictable cron-style automation with vetted scripts, bounded tools/context, token or spend ceilings, dry-run previews, and structured run logs.

## Best Possible Solution

Design one native scoped automation mode across cron/hook protocol, config, CLI/UI/docs, script policy, cost-budget admission, dry-run planning, and run accounting, coordinated with the narrower related trackers.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection gives high confidence that current main lacks a first-class scoped/script-limited mode rather than breaking an existing contract.

## Solution Assessment

Unclear until maintainers choose the product and security shape. The direction is plausible, but current toolsAllow, direct exec, and budget-tracking pieces each solve only part of the requested mode.
