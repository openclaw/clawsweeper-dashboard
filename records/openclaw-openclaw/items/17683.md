## Summary

Keep open: current main has useful isolated cron controls, but it still lacks the requested first-class scoped/script-limited automation mode; the related direct-exec and gateway-budget issues cover narrower pieces, not the full product/security design.

## What This Changes

Request a scoped/script-limited agent mode with admin-approved scripts, deterministic cron-style execution, minimal context, token/spend ceilings, dry-run cost preview, and structured run logs.

## Best Possible Solution

Design one native scoped automation mode across cron/hook protocol, CLI/config/docs, script policy, budget admission, dry-run planning, and run accounting while coordinating the open direct-exec cron and gateway-budget trackers.

## Solution Assessment
