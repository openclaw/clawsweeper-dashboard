## Summary

Keep open. Current main still has only partial isolated cron/hook controls, while the requested scoped/script-limited mode spans new cron/hook payloads, persisted script policy, token/spend budgets, dry-run cost preview, and structured planned-vs-actual accounting that are not implemented and need maintainer design.

## What This Changes

The issue requests a first-class scoped automation mode with admin-defined scripts, token and spend ceilings, cron-first execution, minimal context, dry-run previews, and structured run logs.

## Best Possible Solution

Design one native scoped automation mode across cron/hook protocol, config, CLI/UI/docs, isolated-run accounting, and host-exec policy, sharing budget work with https://github.com/openclaw/openclaw/issues/42475 and direct-exec work with https://github.com/openclaw/openclaw/issues/18160.

## Solution Assessment
