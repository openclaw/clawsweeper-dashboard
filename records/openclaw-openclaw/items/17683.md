## Summary

Current main and the latest release provide useful building blocks—isolated cron turns, explicit command jobs, lightweight context, per-job tool allow-lists, timeouts, usage telemetry, and durable run history—but they do not provide the requested first-class script registry, pre-dispatch token/spend enforcement, dry-run cost preview, or planned-versus-actual audit contract. The open budget-admission tracker covers only one slice, so this broader product item remains necessary but needs maintainer decomposition and API/security decisions before implementation.

## What This Changes

Add a scoped agent mode with approved-script and tool restrictions, deterministic cron execution, fresh per-job context, token and spend ceilings, dry-run cost previews, and structured run auditing.

## Best Possible Solution

Keep isolated cron, command, hook, and run-log paths canonical; decompose this umbrella into reusable approved contracts, with https://github.com/openclaw/openclaw/issues/42475 owning generic pre-model-call budget admission and separate design work for admin-managed script policy plus preview and planned-versus-actual auditing.

## Solution Assessment

No, not as one new parallel agent mode. The narrowest maintainable direction is to add reusable script-policy, model-admission, preview, and audit primitives to the existing isolated cron and hook execution path after maintainers define their contracts.
