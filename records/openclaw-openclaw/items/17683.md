## Summary

Keep open: current main still provides only partial cron controls, while the requested first-class scoped/script-limited mode remains unimplemented. The live discussion also explicitly keeps this as the broader product/API tracker, separate from the open gateway budget tracker.

## What This Changes

Request a first-class scoped/script-limited automation mode with whitelisted scripts, cron-first isolated runs, token/spend ceilings, dry-run cost preview, minimal context, and structured run logs.

## Best Possible Solution

Design one native scoped automation mode across cron/hook protocol, config/CLI/docs, script policy, budget admission, dry-run planning, and run accounting, sharing the spend-budget mechanism with https://github.com/openclaw/openclaw/issues/42475.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection gives high confidence that current main lacks a first-class scoped/script-limited mode rather than breaking an existing contract.

## Solution Assessment

Unclear until maintainers choose the product and security shape. The direction is plausible, but current isolated cron sessions, light context, toolsAllow, timeouts, and post-run cost telemetry solve only parts of the requested mode.
