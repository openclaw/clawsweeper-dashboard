## Summary

Keep open: current main and v2026.6.5 now cover an important slice with cron command payloads, light context, tool allow-lists, timeouts, delivery, and cost observability, but the requested first-class scoped/script-limited mode still lacks an admin script registry, dry-run cost preview, per-job token/spend ceilings, daily budget aborts, and hook-level scoped execution semantics. The open gateway budget tracker is related but does not supersede this combined product and security design.

## What This Changes

The issue requests a scoped, script-limited agent mode for predictable cron-first automation with script whitelisting, minimal context, token/spend ceilings, dry-run planning, and structured run transparency.

## Best Possible Solution

Design one native scoped automation contract that coordinates cron command payloads, hook dispatch, script policy, budget admission, dry-run planning, run accounting, CLI/schema/docs, and security review.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/product request. Source inspection gives high confidence that current main lacks the requested first-class scoped/script-limited mode even though direct command cron has shipped.

## Solution Assessment

Unclear until maintainers choose the product and security contract. Existing command cron, isolated cron sessions, lightContext, toolsAllow, timeouts, model/fallback selection, and cost telemetry solve only parts of the requested mode.
