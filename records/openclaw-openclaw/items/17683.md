## Summary

Keep open: current main and the latest tagged release still have useful partial cron controls, but not the requested first-class scoped/script-limited mode. Prior review discussion and the open direct-exec, gateway-budget, and cron-resource-field trackers support keeping this as a broader product and security design item.

## What This Changes

The issue requests a scoped agent mode for scheduled/scripted automation with admin-defined scripts, bounded tool scope, hard token/spend controls, minimal context, dry-run planning, and structured run logs.

## Best Possible Solution

Design one native scoped automation mode across cron/hook protocol, CLI/config/docs, script policy, budget admission, dry-run planning, and run accounting while coordinating https://github.com/openclaw/openclaw/issues/18160, https://github.com/openclaw/openclaw/issues/42475, and https://github.com/openclaw/openclaw/issues/64721.
