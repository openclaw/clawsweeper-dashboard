## Summary

Keep open: current main still has useful partial controls for isolated cron jobs, hook-triggered agent runs, lightweight context, timeouts, and per-job tool allow-lists, but not the requested first-class scoped/script-limited mode. ## What This Changes

The request asks for a first-class scoped/script-limited agent mode with admin-defined scripts, cron-first execution, token/spend ceilings, dry-run cost previews, minimal context, and structured run transparency.

## Best Possible Solution

Design one native scoped automation mode across cron/hook protocol, CLI/config/docs, script policy, budget admission, dry-run planning, and run accounting, coordinating the narrower related trackers instead of duplicating policy in separate paths.
