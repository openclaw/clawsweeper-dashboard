## Summary

Keep open for normal PR review: current main still treats every post-core repair warning as fatal, while the PR aligns the fatal decision with the existing structured repair failure and smoke-check contracts. I found no blocking correctness or security findings.

## What This Changes

The PR changes post-core plugin convergence to keep repair warnings visible while setting fatal status only from failed repair plugin ids or payload smoke failures, with regression tests.

## Best Possible Solution

Land the focused convergence fix after normal CI and maintainer review so warnings stay visible while restart-blocking status comes only from failed repair ids and payload smoke failures.
