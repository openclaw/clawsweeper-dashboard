## Summary

Keep open for normal maintainer review; I found no blocking patch findings. Current main makes warning-only convergence fatal, while the existing repair contract already separates `failedPluginIds` from soft warnings, and the PR’s terminal proof covers the soft-warning and hard-failure cases.

## What This Changes

The PR changes post-core plugin convergence to mark only failed repair plugin IDs or payload smoke failures as fatal, with regression tests for soft warnings and failed repair IDs.

## Best Possible Solution

Land the focused convergence fix after normal CI and maintainer review, then let the linked issue close after merge.
