## Summary

Keep this PR open: current main still marks any post-core plugin convergence warning as fatal, while the PR is a focused fix that keeps soft warnings visible and preserves hard stops for failed plugin IDs and payload smoke failures.

## What This Changes

The PR changes post-core plugin convergence so repair warnings only mark the result errored when repair reports failed plugin IDs or payload smoke checks fail, and adds regression coverage for both paths.

## Best Possible Solution

Land the focused predicate change once required checks complete and maintainers accept the repair-warning contract, with optional full update/restart smoke proof for extra release confidence.

## Solution Assessment

Yes, this is a narrow maintainable fix because the repair contract already distinguishes failed plugin IDs and the payload smoke check still blocks unloadable plugin payloads.
