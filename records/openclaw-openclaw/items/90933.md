## Summary

Keep open: the PR directly addresses the linked CLI lane-busy bug with a focused queue, Gateway, and CLI path, and I did not find a blocking patch defect. ## What This Changes

The branch adds a command-queue wait rejection hook, propagates session-lane-busy details through Gateway final responses, and renders structured busy errors in the agent CLI with focused regression tests.

## Best Possible Solution

Land the bounded queue/Gateway/CLI fix after maintainers accept busy-as-terminal CLI semantics and the relevant gate proof is green; keep the linked issue open until merge.

## Solution Assessment

Yes, this appears to be the best bounded fix if maintainers accept fail-fast busy semantics: queue owns wait facts, Gateway translates only CLI-tracked lane-busy failures, and the CLI renders the result.
