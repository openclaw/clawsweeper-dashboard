## Summary

Keep open. Current main still lacks any cron `promptCacheKey` field and still derives OpenAI `prompt_cache_key` from the isolated run session id, so the reported cache-miss behavior remains valid; the optional model-override diagnostic path is already covered separately on main.

## What This Changes

Request to let cron `agentTurn` payloads set a stable prompt cache key independent of isolated per-run session ids.

## Best Possible Solution

Add a validated `promptCacheKey` cron `agentTurn` field and carry it as an OpenAI prompt-cache override while keeping isolated transcript/session semantics unchanged.

## Reproduction Assessment

Yes. Static tracing is high-confidence: isolated cron creates a fresh session id, passes it into the embedded run, and the OpenAI transport uses that id as `prompt_cache_key`; live billing data would only quantify the impact.

## Solution Assessment

Yes. An explicit cache key is the narrowest maintainable fix because it stabilizes provider cache routing without reusing a persistent cron transcript session.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/76218. Add a validated `promptCacheKey` field for cron `agentTurn` payloads so isolated cron jobs can reuse a stable OpenAI prompt cache key without reusing the transcript session. Expected boundary: update the cron protocol schema, cron runtime types, normalization/persist/patch handling as needed, isolated cron execution forwarding, embedded agent stream options, and OpenAI transport request builders. Preserve `sessionTarget` semantics and do not make isolated cron reuse prior conversation history. Preserve provider compatibility stripping: direct OpenAI/allowed Azure/OpenAI-compatible Responses and Completions paths may receive the override only when current policy allows prompt cache keys; unsupported proxy-like providers must still strip or omit it. Update docs for cron jobs and prompt caching, add a user-facing changelog entry, and keep the already-fixed cron payload model diagnostic behavior unchanged.
