## Summary

Keep open: current main and v2026.6.9 still build OpenAI-compatible memory embedding SSRF policy from the embedding base URL only, while the open closing PR is the active fix path and still needs maintainer/security acceptance.

## What This Changes

Requests that memory-core OpenAI-compatible embeddings keep SSRF blocking by default while honoring an explicit configured private-network trust policy for private endpoints.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/94344; if accepted, keep provider-level `request.allowPrivateNetwork` as the canonical trust surface with strict defaults and omitted/false/true regression coverage.

## Reproduction Assessment

Yes. ## Solution Assessment

Unclear until maintainer/security acceptance, but the open linked PR is the best visible direction because it reuses the existing provider request-policy surface instead of adding a second memory-only trust knob.
