## Summary

Keep open: current main and v2026.6.9 still build the OpenAI-compatible memory embedding SSRF policy from the embedding base URL only, while the open linked PR is the active candidate fix that still needs maintainer/security acceptance.

## What This Changes

Allow memory-core OpenAI-compatible embedding requests to use explicitly configured private endpoints through an explicit SSRF-safe trust policy.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/94344; if accepted, keep provider-level request.allowPrivateNetwork as the canonical memory embedding trust surface with strict omitted/false/true coverage.
