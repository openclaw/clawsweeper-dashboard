## Summary

Keep this open: current main has SecretRef-backed configured credential resolution, but not the requested agent-callable `secret.get`/`secret.has` broker, and the remaining design needs a maintainer trust-boundary decision before implementation is safe.

## What This Changes

Add a generic agent-callable secret broker surface for named secret lookup over backends such as Bitwarden, macOS Keychain, 1Password, and Vault.

## Best Possible Solution

Define a trusted secret capability contract with metadata-only `has`/`list`, guarded `get`, redacted or non-persistent results, and owner-trusted session gating before choosing bundled backend code or an external plugin API path.
