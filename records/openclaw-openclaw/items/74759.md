## Summary

Keep this open. Current main supports configured SecretRef resolution, but it does not provide agent-callable `secret.get`/`secret.has`/`secret.list` tools, and the plugin seams needed for safe redaction and pre-tool trust gating are still bundled-only.

## What This Changes

The issue requests a first-class secret broker plugin/API for `secret.get`, `secret.has`, and `secret.list` over backends such as Bitwarden, macOS Keychain, 1Password, and Vault.

## Best Possible Solution

Define a trusted secret capability contract with metadata-only `has`/`list`, redacted or non-persistent `get` results, and owner-trusted session gating before implementing either a bundled backend or an external plugin API extension.

## Reproduction Assessment

Not applicable as a feature and security-design request. The gap is high-confidence by source inspection: current main exposes configured SecretRef resolution but no agent-callable `secret.*` broker tools.

## Solution Assessment

Unclear. The proposed API is plausible, but the safest maintainable path depends on a maintainer decision between an in-tree trusted plugin and an external plugin API extension with explicit redaction and scope contracts.

## Security Review

Status: needs_attention

Summary: The request is security-sensitive because it proposes agent-callable tools that can return live credential material before external redaction and trusted-session contracts exist.

Concerns:

- **[high] Define redacted secret-result semantics before get:** `src/plugins/registry.ts:379`
  - body: A `secret.get` tool would intentionally return credential material. External plugin tools currently produce ordinary tool output, while agent tool-result middleware is bundled-only, so a trusted redacted or non-persistent result shape is needed before implementation.
  - confidence: 0.93
- **[high] Gate secret tools by owner-trusted session scope:** `src/plugins/registry.ts:1606`
  - body: The requested scope guard needs a pre-tool trusted-session check so secret tools are hidden or blocked outside approved owner contexts rather than relying only on transcript redaction after execution.
  - confidence: 0.89
- **[medium] Choose bundled trust or plugin API trust first:** `docs/plugins/sdk-overview.md:177`
  - body: Current trusted policy and result-redaction seams are bundled-only. Shipping this as an external plugin without a new trust contract would leave key safety controls unavailable.
  - confidence: 0.9
