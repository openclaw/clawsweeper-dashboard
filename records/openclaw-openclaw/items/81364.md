## Summary

Keep open: the latest PR head fixes the earlier blocked-download bypass, but it still treats pending/stale clean ClawHub trust records as acknowledgement-required even though the documented PR contract says those states should warn and continue. Because this is a maintainer-labeled, security-sensitive plugin install policy PR, it should stay open for author or maintainer follow-up rather than cleanup close.

## What This Changes

This PR adds exact-release ClawHub trust checks and acknowledgement plumbing for plugin installs/updates, docs/tests, a demo script, and compatibility fixes in plugin runtime, Matrix tests, gateway lint, and prompt snapshots.

## Best Possible Solution

Land the trust gate after separating warning-only `review-recommended` states from acknowledgement-required risk states, while preserving unconditional blocked/malicious failures and the existing local scanner.

## Reproduction Assessment

Yes. Source inspection of the PR head shows `review-recommended` trust states flow into the acknowledgement/failure branch, and the added tests explicitly expect stale/pending clean records to require confirmation.

## Solution Assessment

No. The overall trust-check direction is appropriate, but the implementation should let pending/stale clean trust records warn and continue, with acknowledgement reserved for actual risky non-blocked states.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Let warning-only ClawHub trust states continue:** `src/plugins/clawhub.ts:778-791`
  - body: `review-recommended` is used for pending/stale clean trust records, but after logging the warning this branch still prompts or returns `clawhub_risk_acknowledgement_required`. That makes non-interactive installs and chat/repair installs fail even though the PR docs say these records should warn without requiring acknowledgement. Return success after logging warning-only states and keep acknowledgement for `review-required`.
  - confidence: 0.92
