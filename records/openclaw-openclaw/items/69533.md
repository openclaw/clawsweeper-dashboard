## Summary

Keep open. Current main still hardcodes the Dream Diary narrative system prompt and prompt builder, while the memory-core config contract, docs, and Dreams UI do not expose a prompt/template field; related #65097 is useful cluster context but does not fully replace this custom-template request.

## What This Changes

Request adding a user-customizable Dream Diary prompt/template with placeholders and Settings/Dreaming UI support.

## Best Possible Solution

Add a memory-core-owned Dream Diary prompt/template config contract under plugins.entries.memory-core.config.dreaming, preserve the current prompt as the default, document placeholders, and thread the resolved value through narrative generation and the Dreams/settings surface.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature request. The gap is high-confidence from source inspection because the schema rejects prompt/template keys and the narrative path never reads them.

## Solution Assessment

Yes, with scope adjustment. The maintainable direction is a memory-core plugin config contract rather than a top-level core shortcut, with defaults preserved and placeholders explicitly documented.
