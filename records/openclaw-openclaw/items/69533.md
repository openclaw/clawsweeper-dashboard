## Summary

Keep open. Current main still hardcodes the Dream Diary narrative system prompt and prompt builder, while the memory-core config schema, resolver, docs, and Dreams UI expose no prompt/template field; related #65097 is useful cluster context but is not a complete duplicate of this custom-template request.

## What This Changes

Add a user-configurable Dream Diary prompt/template for memory-core dreaming output, expose it in settings/UI, and support documented placeholders such as date, time, count, and theme.

## Best Possible Solution

Add a memory-core-owned prompt/template contract under plugins.entries.memory-core.config.dreaming, preserve the current prompt as the default, document a small placeholder set, thread the resolved config through light/REM/deep narrative generation, expose it in the existing Dreams/settings surface, and cover backend/UI/docs behavior.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature request. The gap is high-confidence from source inspection because the schema rejects prompt/template keys and the narrative generation path never reads them.

## Solution Assessment

Yes, with scope adjustment. The maintainable solution is a memory-core plugin config contract rather than a top-level core shortcut, with defaults preserved and placeholders explicitly documented.
