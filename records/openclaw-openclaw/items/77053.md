## Summary

Keep open: the latest head resolved the changelog conflict and the inference preload path now carries the TTL, but the shared LM Studio native load helper is still used by embedding warmups that never receive the new provider setting.

## What This Changes

The PR adds provider-local `models.providers.lmstudio.params.ttlSeconds` handling for LM Studio native model-load preloads, with tests, docs, and a changelog entry.

## Best Possible Solution

Keep the provider-local TTL feature, but make the load contract consistent by applying the validated setting to every LM Studio native load path or explicitly narrowing the feature and docs to inference-only behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature PR. The remaining blocker is source-reproducible from the latest PR head: the embedding warmup caller still omits `ttlSeconds`.

## Solution Assessment

No, the current patch is close but incomplete. The narrow maintainable fix is to share the same validated provider TTL with the embedding warmup native load call while keeping OpenAI-compatible inference payloads unchanged.
