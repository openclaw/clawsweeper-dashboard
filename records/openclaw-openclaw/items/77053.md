## Summary

Keep this PR open because the feature is in the right LM Studio plugin boundary but the current patch is incomplete: the provider-level TTL only reaches inference preload and not the existing embedding warmup load path.

## What This Changes

The PR adds an opt-in LM Studio `models.providers.lmstudio.params.ttlSeconds` setting, sends it as `ttl` to native model load requests, updates preload dedupe/tests, docs, and changelog.

## Best Possible Solution

Thread the validated provider-level TTL through every LM Studio native load caller, especially memory embedding warmup, while keeping OpenAI-compatible request payloads unchanged.

## Reproduction Assessment

Not applicable as a feature PR rather than a user bug report. The remaining defect is source-reproducible because the shared native load helper is used by both inference and embeddings, while the PR only passes `ttlSeconds` from the inference stream wrapper.

## Solution Assessment

No, the patch is close but not complete. The LM Studio plugin is the right place for this opt-in behavior, but a provider-level TTL should also reach the embedding warmup load path or the documented scope should be narrowed.
