## Summary

Keep open: the provider-local TTL feature is close, but the latest PR head still wires the new setting only through inference preloads while LM Studio embedding warmups continue using the shared native load helper without `ttlSeconds`.

## What This Changes

The PR adds an opt-in `models.providers.lmstudio.params.ttlSeconds` setting that passes `ttl` to LM Studio native model-load requests, with docs, changelog, and LM Studio tests.

## Best Possible Solution

Keep the provider-local TTL feature, but apply the validated setting consistently to every LM Studio native load path, including embedding warmup, while keeping OpenAI-compatible inference and embedding payloads unchanged.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature PR. The remaining blocker is source-reproducible from the latest PR head: the embedding warmup caller still omits `ttlSeconds`.

## Solution Assessment

No, the current patch is not yet the best complete solution. The narrow maintainable fix is to share the same validated provider TTL with the embedding warmup native load call or explicitly narrow the public contract to inference-only behavior.
