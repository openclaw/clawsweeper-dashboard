## Summary

Keep open: the feature belongs in the LM Studio plugin boundary, but the current PR only applies `ttlSeconds` to inference preload while existing embedding warmups still call the same native load helper without the provider TTL.

## What This Changes

The PR adds `models.providers.lmstudio.params.ttlSeconds` for LM Studio native model-load requests, updates preload dedupe/tests/docs, and adds a changelog entry.

## Best Possible Solution

Thread the validated LM Studio provider TTL through every native load caller covered by the provider-level config, especially memory embedding warmup, while keeping OpenAI-compatible inference payloads unchanged.

## Reproduction Assessment

Not applicable as a feature request, but the review finding is source-reproducible: the shared native load helper is used by both inference and embeddings, and the PR only threads `ttlSeconds` through the inference stream wrapper.

## Solution Assessment

No, the current patch is close but incomplete. The narrower maintainable fix is to keep the TTL handling in the LM Studio plugin and pass the same validated provider config into embedding warmup or explicitly narrow the documented scope.
