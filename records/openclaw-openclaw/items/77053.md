## Summary

Keep open: latest head resolves the changelog conflict and implements opt-in TTL for LM Studio inference preloads, but the same provider setting still does not reach LM Studio embedding warmups that already use the native load helper.

## What This Changes

The PR adds an opt-in LM Studio `models.providers.lmstudio.params.ttlSeconds` config that sends `ttl` in native model-load requests, updates local-model docs/changelog, and adds LM Studio preload tests.

## Best Possible Solution

Keep the provider-local TTL feature, but make it internally consistent before merge by applying the validated opt-in TTL to every LM Studio native load path or explicitly limiting the docs and changelog to inference-only behavior.

Not applicable as a bug reproduction because this is a feature PR. The remaining blocker is source-reproducible: latest head passes `ttlSeconds` from inference preloads, while the embedding warmup still calls the same load helper without it.

## Solution Assessment
