## Summary

Keep open: current main and v2026.5.22 still lack a native direct-parent-only mid-run send path for sub-agents; the available `sessions_send` opt-in is broader and does not enforce parent-only, same-tree, no-chaining authority.

## What This Changes

Request a native parent-scoped way for sub-agents to send mid-run updates or questions only to their direct parent session.

## Best Possible Solution

Add a narrow parent-send API or trusted `parent` resolver that derives the direct parent from stored spawn/session metadata, enforces same-tree direct-parent-only no-chaining behavior, and updates policy, visibility, docs, and regression tests after maintainer API and security review.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing behavior report. Source inspection on current main and v2026.5.22 gives high confidence that no native parent-only send tool or trusted parent alias exists while default sub-agent policy still denies raw `sessions_send`.

## Solution Assessment

Yes at the direction level for a dedicated parent-scoped core API or trusted parent alias; no for broadly re-enabling raw `sessions_send`. Maintainers still need to choose the public shape and exact same-tree/no-chaining enforcement rules before implementation.
