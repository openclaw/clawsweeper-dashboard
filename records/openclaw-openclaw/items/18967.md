## Summary

Keep this open. Current main still lacks a trusted parent-scoped send path for native sub-agents; the existing announce chain, ACP-only parent streaming, and broad `sessions_send` opt-ins do not satisfy the requested parent-only, same-tree, no-chaining API.

## What This Changes

The issue requests a parent-only way for sub-agents to send mid-run updates or questions to the session that spawned them.

## Best Possible Solution

Add a narrow core parent-send path, preferably `sessions_send_parent` or a trusted `sessionKey: "parent"` resolver, backed by spawn/session metadata, docs, and regression coverage.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source inspection of current main gives high confidence that native sub-agents still lack a parent-only send tool or alias.

## Solution Assessment

Yes for a narrow parent-scoped core API; no for broadly re-enabling `sessions_send`. The maintainable solution should derive the parent server-side and reject missing-parent, sibling, ancestor, cross-agent, and forwarded targets.
