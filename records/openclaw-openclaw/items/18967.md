## Summary

Keep open: current main and the latest release still lack a trusted child-to-direct-parent mid-run send path; existing parent completion handoff, ACP-only streaming, and broad `sessions_send` opt-ins are adjacent but do not implement the requested parent-only, same-tree, no-chaining API.

## What This Changes

The issue requests a native parent-scoped send mechanism so sub-agents can send mid-run progress updates or questions only to their direct parent session.

## Best Possible Solution

Define one narrow parent-only send contract backed by canonical sub-agent metadata, keep broad `sessions_send` opt-ins separate, and update policy, resolver, delivery, docs, and security regression tests together.

## Reproduction Assessment

Yes, at source level. Current main still denies `sessions_send` to sub-agents by default, has no `sessions_send_parent` tool, and does not resolve a trusted `parent` target from sub-agent metadata.

## Solution Assessment

Yes in direction, but not as a broad ordinary `sessions_send` opt-in. The maintainable path is a dedicated parent-only operation or trusted parent selector backed by active run metadata and security tests.
