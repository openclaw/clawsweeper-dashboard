## Summary

Keep open: current main still lacks a native direct-parent-only mid-run send path for sub-agents, and the requested capability changes cross-session authority in a way that needs maintainer product and security review.

## What This Changes

The issue requests a parent-scoped `sessions_send` capability so a sub-agent can send mid-run updates or questions only to the direct parent session that spawned it.

## Best Possible Solution

Add a narrow parent-send API or trusted `parent` resolver that derives the direct parent from trusted spawn/session metadata, enforces same-tree parent-only no-chaining behavior, and updates policy, session resolution, visibility checks, docs, and regression tests after maintainer API and security review.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken existing behavior. Source inspection on current main gives high confidence that no native parent-only send tool or trusted parent alias exists while default sub-agent policy still denies raw `sessions_send`.

## Solution Assessment

Yes at the direction level for a dedicated parent-scoped core API or trusted parent alias; no for broadly re-enabling raw `sessions_send`. Maintainers still need to choose the public shape and exact same-tree/no-chaining enforcement rules before implementation.
