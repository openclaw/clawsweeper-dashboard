## Summary

Keep open: current main still lacks a trusted child-to-direct-parent mid-run send path. The existing completion handoff, ACP-only parent streaming, parent-to-child send safeguards, and broad `sessions_send` opt-ins are adjacent but do not implement the requested parent-only, same-tree, no-chaining API.

## What This Changes

Add a safe parent-scoped `sessions_send` or `sessions_send_parent` mechanism so sub-agents can send mid-run progress or questions only to their direct parent session.

## Best Possible Solution

Define one narrow parent-only send contract derived from canonical active sub-agent metadata, keep broad `sessions_send` opt-in separate, and update policy, resolver/tool surface, delivery behavior, docs, and security regression tests together.

## Reproduction Assessment

Yes. Source inspection is a high-confidence reproduction path: current main still denies ordinary `sessions_send` to sub-agents by default, has no `sessions_send_parent` tool, and does not resolve a trusted `parent` target from sub-agent metadata.

## Solution Assessment

Yes in direction, but not as a broad opt-in to ordinary `sessions_send`. The maintainable solution is a dedicated parent-only operation or trusted parent selector backed by active run metadata and security tests.
