- `impact:session-state`: The request changes how active sub-agent sessions communicate with their direct parent session during a run.

## Summary

Keep open: current main still lacks a native direct-parent-only mid-run send path for sub-agents; existing broad opt-ins and ACP/announce paths are adjacent but do not enforce the requested parent-only, same-tree, no-chaining contract.

## What This Changes

Request a parent-scoped session-send mechanism so sub-agents can send mid-run progress or questions only to the direct parent session that spawned them.

## Best Possible Solution

Add a narrow parent-send API or trusted `parent` resolver that derives the direct parent from spawn/session metadata, enforces same-tree parent-only no-chaining behavior, and updates policy, visibility, docs, and regression tests after maintainer API and security review.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing behavior report. Source inspection on current main gives high confidence that no native parent-only send tool or trusted parent alias exists while default sub-agent policy still denies raw `sessions_send`.

## Solution Assessment

Yes at the direction level for a dedicated parent-scoped core API or trusted parent alias; no for broadly re-enabling raw `sessions_send`. Maintainers still need to choose the public shape and exact same-tree/no-chaining enforcement rules before implementation.
