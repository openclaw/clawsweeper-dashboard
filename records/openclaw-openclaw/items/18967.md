## Summary

Keep open: current main and the latest shipped release still lack a native direct-parent-only mid-run send path for sub-agents. Existing completion announce delivery, `sessions_yield`, `subagents` steering, and broad opt-in access to raw `sessions_send` are useful adjacent primitives, but they do not provide the requested server-derived parent target with same-tree and no-chaining guarantees.

## What This Changes

The issue requests a parent-scoped way for sub-agents to send mid-run updates or questions only to the session that spawned them.

## Best Possible Solution

Add a narrow core parent-send path that derives the direct parent from trusted spawn/session metadata, enforces same-tree parent-only no-chaining behavior, and updates policy, resolver/send behavior, docs, and regression tests.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection on current main and `v2026.5.12` gives high confidence that no native parent-only send tool or trusted parent alias exists while default sub-agent policy still denies `sessions_send`.

## Solution Assessment

Yes for a dedicated parent-scoped core API or trusted parent alias; no for broadly re-enabling raw `sessions_send`. The maintainable path should derive the direct parent server-side and enforce same-tree, parent-only, no-chaining behavior.
