## Summary

Keep open: current main and v2026.6.10 still lack a trusted child-to-direct-parent mid-run send path; the code only has default sub-agent denial, ordinary `sessions_send`, completion handoff, ACP parent streaming, and broader opt-ins that are not parent-only.

## What This Changes

Request a parent-scoped way for sub-agents to send mid-run progress updates or questions only to their direct parent session.

## Best Possible Solution

Define one narrow parent-only send contract derived from canonical active sub-agent metadata, and update policy, resolver, delivery, docs, and security regression tests together.

## Reproduction Assessment

Yes, at source level. Current main still denies `sessions_send` to sub-agents by default, has no `sessions_send_parent` tool, and does not resolve a trusted `parent` target from sub-agent metadata.

## Solution Assessment

Yes in direction, but not as a broad opt-in to ordinary `sessions_send`. The maintainable solution is a dedicated parent-only operation or trusted parent selector backed by active run metadata and security tests.
