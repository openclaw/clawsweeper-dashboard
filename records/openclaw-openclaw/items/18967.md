## Summary

Keep this open: current main still lacks a trusted parent-scoped send path for sub-agents, and the remaining work is an authorization-sensitive feature rather than cleanup.

## What This Changes

Request a parent-scoped mechanism that lets sub-agents send mid-run updates or questions only to their direct parent session.

## Best Possible Solution

Add a narrow core parent-send path, preferably sessions_send_parent or a trusted sessionKey="parent" resolver, derived server-side from spawn/session metadata and covered by policy, docs, and regression tests.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source inspection of current main shows no native parent-only send tool or alias, and default sub-agent policy still removes sessions_send.

## Solution Assessment

Yes for a dedicated parent-scoped core API; no for broadly re-enabling raw sessions_send. The maintainable design should derive the direct parent server-side and reject missing-parent, sibling, ancestor, cross-agent, and forwarded targets.
