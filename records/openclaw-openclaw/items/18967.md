## Summary

Keep open: current main and the latest release still do not provide a native parent-scoped mid-run send path for sub-agents; existing behavior is completion announce delivery, ACP-only parent streaming, parent-to-child control sends, or broad raw `sessions_send` opt-in.

## What This Changes

Request a safe parent-only mechanism for sub-agents to send mid-run updates or questions to the direct parent session.

## Best Possible Solution

Add a narrow parent-send API or trusted `parent` resolver that derives the direct parent from stored spawn/session metadata, enforces same-tree direct-parent-only no-chaining behavior, and updates policy, visibility, docs, and regression tests after maintainer security and API review.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing behavior report. Source inspection gives high confidence that no native parent-only send tool or trusted parent alias exists while default sub-agent policy still denies raw `sessions_send`.

## Solution Assessment

Yes at the direction level for a dedicated parent-scoped core API or trusted `parent` alias; no for broadly re-enabling raw `sessions_send`. Maintainers still need to choose the public shape and exact same-tree/no-chaining enforcement rules before implementation.
