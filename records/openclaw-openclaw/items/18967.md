## Summary

Keep open: current main and the latest shipped tag still lack a native direct-parent-only mid-run send path for sub-agents. Existing announce delivery, ACP-only parent streaming, parent-to-child `sessions_send`, and broad opt-in exposure are adjacent but do not provide a server-derived child-to-parent target with same-tree and no-chaining guarantees.

## What This Changes

Request a parent-scoped sub-agent send mechanism so a child can send progress updates or questions only to its direct parent session.

## Best Possible Solution

Add a narrow core parent-send path that derives the direct parent from trusted spawn/session metadata, enforces same-tree parent-only no-chaining behavior, and updates policy, resolver/send behavior, docs, and regression tests.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken existing behavior. Source inspection on current main and `v2026.5.19` gives high confidence that no native parent-only send tool or trusted parent alias exists while default sub-agent policy still denies raw `sessions_send`.

## Solution Assessment

Yes for a dedicated parent-scoped core API or trusted parent alias; no for broadly re-enabling raw `sessions_send`. Maintainers still need to choose the public shape and exact same-tree/no-chaining enforcement rules before implementation.
