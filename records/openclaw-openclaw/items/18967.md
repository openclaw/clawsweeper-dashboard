## Summary

Keep open. Current main still only offers announce-chain completion, ACP-only parent streaming, and broad config opt-in for `sessions_send`; it does not provide a trusted direct-parent send API with parent-only, same-tree, and no-chaining enforcement.

## What This Changes

Allow sub-agents to send mid-run progress updates or questions only to their direct parent session through a native parent-scoped session-send path.

## Best Possible Solution

Add a dedicated `sessions_send_parent` tool or a `sessionKey=parent` resolver backed by trusted spawn/session metadata, with explicit rejection of missing-parent, sibling, ancestor, and cross-agent targets, no automatic forwarding beyond the direct parent, and focused docs plus regression tests.
