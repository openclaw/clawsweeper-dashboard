## Summary

Keep open: current main still lacks a trusted direct-parent-only mid-run send path for native sub-agents. The existing announce chain, ACP-only parent streaming, and broad opt-in `sessions_send` access do not satisfy the requested parent-only, same-tree, no-chaining security boundary.

## What This Changes

Request a parent-scoped `sessions_send` capability so sub-agents can send progress updates or questions only to their direct parent session.

## Best Possible Solution

Add one core-owned direct-parent send operation that derives the parent from canonical session metadata, rejects missing-parent, sibling, ancestor, unrelated, and cross-agent targets, never forwards automatically beyond one hop, and includes policy, resolver, visibility, nested-sub-agent, and docs coverage.

## Reproduction Assessment

Yes at the source-contract level: current main and v2026.6.5 both show that sub-agents deny `sessions_send` by default and that the available opt-in exposes the general tool rather than a parent-only operation.

## Solution Assessment

Yes in architectural direction: deriving the direct parent from trusted spawn/session metadata is narrower and safer than broadly enabling `sessions_send`. The permanent model-facing shape still needs maintainer approval.
