## Summary

Keep open: current main still lacks a native direct-parent-only send path for sub-agents. The existing broad opt-in path can expose `sessions_send`, but it does not derive the direct parent or enforce the same-tree, parent-only, no-chaining constraints requested here.

## What This Changes

Request a native parent-scoped send path so sub-agents can send mid-run updates or questions only to their direct parent session.

## Best Possible Solution

Add a narrow core parent-send path that derives the direct parent from trusted spawn/session metadata, rejects missing-parent, sibling, ancestor, and cross-agent targets, avoids automatic forwarding beyond that parent, and updates policy, resolver/send behavior, docs, and regression tests.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection on current main gives high confidence that no native parent-only send tool or trusted parent alias exists while default sub-agent policy still denies `sessions_send`.

## Solution Assessment

Yes for a dedicated parent-scoped core API or trusted parent alias; no for broadly re-enabling raw `sessions_send`. The maintainable path should derive the direct parent server-side and enforce same-tree, parent-only, no-chaining behavior.
