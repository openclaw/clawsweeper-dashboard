## Summary

Keep this open. Current main still has announce-chain delivery, ACP-only parent streaming, and broad session visibility knobs, but no trusted parent-scoped send API that derives the direct parent and enforces parent-only, same-tree, no-chaining behavior.

## What This Changes

Request a native way for sub-agents to send mid-run updates or questions directly to their parent session only.

## Best Possible Solution

Add a narrow parent-scoped send surface that derives the direct parent from trusted spawn/session metadata, rejects siblings/ancestors/cross-agent targets, avoids forwarding beyond that parent, and updates policy, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection on current main provide high-confidence evidence that the requested feature is absent.

## Solution Assessment

Unclear until maintainers choose the API shape. A dedicated parent-scoped tool or trusted `parent` resolver is safer than broadly re-enabling `sessions_send` because current visibility scopes are not direct-parent-only.
