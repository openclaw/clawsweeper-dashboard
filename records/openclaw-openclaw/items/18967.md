## Summary

Keep this open: current main still does not provide a native parent-scoped send path for sub-agents, and the request is an authorization-sensitive feature rather than conservative cleanup.

## What This Changes

Request a parent-only sub-agent send path, such as `sessions_send_parent` or `sessionKey: "parent"`, for mid-run updates and questions to the spawning session.

## Best Possible Solution

Add a narrow core parent-send path that derives the direct parent server-side from trusted spawn/session metadata, rejects missing-parent, sibling, ancestor, and cross-agent targets, avoids automatic forwarding beyond that parent, and updates policy, docs, and regression tests.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source inspection on current main shows no native parent-only send tool or trusted `parent` alias, while default sub-agent policy still removes `sessions_send`.

## Solution Assessment

Yes for a dedicated parent-scoped core API; no for broadly re-enabling raw `sessions_send`. The maintainable direction should derive the direct parent from trusted metadata and enforce same-tree, parent-only, no-chaining behavior.
