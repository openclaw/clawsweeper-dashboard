## Summary

Keep this open: current main still lacks a trusted parent-scoped send path for sub-agents, and the remaining work is an authorization-sensitive feature rather than cleanup.

## What This Changes

Add a native parent-only send path so sub-agents can send updates or questions to the direct session that spawned them without targeting arbitrary sessions.

## Best Possible Solution

Add a narrow core parent-send path, preferably `sessions_send_parent` or a trusted `sessionKey: "parent"` resolver, backed by spawn/session metadata, no forwarding, docs, and regression coverage.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source inspection of current main shows no native parent-only send tool or alias.

## Solution Assessment

Yes for a dedicated parent-scoped core API; no for broadly re-enabling raw `sessions_send`. The maintainable solution should derive the direct parent server-side and reject missing-parent, sibling, ancestor, cross-agent, and auto-forwarded targets.
