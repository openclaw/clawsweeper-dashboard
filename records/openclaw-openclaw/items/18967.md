## Label Justifications

- `P2`: This is a recurring, source-backed sub-agent coordination limitation, but it is a bounded enhancement rather than an urgent regression or unusable runtime.
- `impact:session-state`: The requested capability would move mid-run questions and progress between linked child and parent sessions during an active sub-agent run.
- `impact:security`: The feature creates a new session authorization edge whose parent-only, same-tree, no-chaining restrictions are essential.

## Summary

Keep open: current main still lacks a trusted child-to-direct-parent mid-run send path. Completion announcements, ACP-only parent streaming, parent-to-child controls, and broad `sessions_send` opt-in settings are adjacent behavior, not the requested parent-only same-tree API.

## What This Changes

The issue requests a native parent-scoped mechanism for sub-agents to send mid-run progress updates or questions only to their direct parent session.

## Best Possible Solution

Define one purpose-built parent-send contract that derives the direct parent from canonical active sub-agent metadata, rejects stale/missing/cross-tree targets, does not auto-forward, and adds policy, resolver, delivery, docs, and regression coverage.

## Reproduction Assessment

Yes, at source level. Current main still denies `sessions_send` to sub-agents by default, has no `parent` resolver or `sessions_send_parent` tool, and exposes only broader opt-in session-send configuration.

## Solution Assessment

Yes in direction, but not as a broad opt-in to ordinary `sessions_send`. The maintainable solution is a dedicated parent-only operation or trusted parent selector backed by active run metadata and security tests.
