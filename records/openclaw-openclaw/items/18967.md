## Summary

Keep this open. Current main still has announce-chain delivery, ACP-only parent streaming, and broad opt-in ways to expose `sessions_send`, but it does not provide the requested trusted parent-scoped send tool or `sessionKey: "parent"` resolver.

## What This Changes

Request a native parent-scoped session send path so sub-agents can send mid-run updates or questions only to their direct parent session without arbitrary session access or human relay.

## Best Possible Solution

Add a narrow parent-send path, preferably `sessions_send_parent` or a `sessionKey: "parent"` resolver, backed by trusted sub-agent registry/session metadata. It should allow only the direct parent in the same agent tree, fail closed when no parent exists, avoid forwarding beyond that parent, and update policy, resolver, tool, docs, and regression tests together.

## Reproduction Assessment

Not applicable as a runtime bug report. The current-main source inspection is high confidence: default sub-agent policy denies `sessions_send`, no parent-send tool or parent alias exists, and docs describe only ACP parent streaming plus announce-chain delivery.

## Solution Assessment

No, the current workaround is not the best solution for this issue. Broadly re-enabling `sessions_send` through config can work for trusted setups, but the requested maintainable fix is a parent-only API that derives and enforces the parent target internally.
