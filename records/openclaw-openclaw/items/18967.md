## Summary

Keep open: current main and v2026.5.20 still do not provide a native direct-parent-only mid-run send path for sub-agents; announce-chain delivery, ACP-only parent streaming, and broad opt-in raw `sessions_send` access are adjacent but do not enforce the requested parent-only, same-tree, no-chaining contract.

## What This Changes

The issue requests a parent-only way for sub-agents to send mid-run updates or clarifying questions to their direct parent session.

## Best Possible Solution

Add a narrow parent-send path that derives the direct parent from trusted spawn/session metadata, enforces same-tree parent-only no-chaining behavior, and updates policy, resolver/send behavior, docs, and regression tests after maintainer API and security review.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken existing behavior. Source inspection on current main and v2026.5.20 gives high confidence that no native parent-only send tool or trusted parent alias exists while default sub-agent policy still denies raw `sessions_send`.

## Solution Assessment

Yes at the direction level for a dedicated parent-scoped core API or trusted parent alias; no for broadly re-enabling raw `sessions_send`. Maintainers still need to choose the public shape and exact same-tree/no-chaining enforcement rules before implementation.
