## Summary

Keep this open: current main still lacks a trusted parent-scoped send path for sub-agents, and the requested change is an authorization-sensitive feature rather than maintainer cleanup.

## What This Changes

The issue requests a parent-only sub-agent send mechanism, such as `sessions_send_parent` or a trusted `sessionKey: "parent"` alias.

## Best Possible Solution

Add a narrow core parent-send path that derives the direct parent server-side from trusted spawn/session metadata, rejects missing-parent, sibling, ancestor, and cross-agent targets, avoids automatic forwarding beyond that parent, and updates policy, docs, and regression tests.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source inspection of current main shows no native parent-only send tool or alias, and default sub-agent policy still removes `sessions_send`.

## Solution Assessment

Yes for a dedicated parent-scoped core API; no for broadly re-enabling raw `sessions_send`. The maintainable design should derive the direct parent from trusted metadata and enforce same-tree, parent-only, no-chaining behavior.
