## Summary

Keep this open. Current main still lacks a trusted parent-scoped send path; ACP parent streaming and broad opt-in `sessions_send` exposure do not satisfy the requested parent-only, same-tree, no-chaining API.

## What This Changes

Request a native parent-scoped session send path so sub-agents can send mid-run updates or questions only to their direct parent session.

## Best Possible Solution

Add a narrow parent-send path in core, preferably `sessions_send_parent` or a `sessionKey: "parent"` resolver, backed by trusted sub-agent registry/session metadata and regression coverage.

## Reproduction Assessment

Not applicable as a runtime bug report. Source inspection is high confidence: current main denies default sub-agent `sessions_send`, has no parent-send tool or parent alias, and documents only ACP parent streaming plus announce-chain delivery.

## Solution Assessment

No, the current workaround is not the best way to solve the issue. Broadly re-enabling `sessions_send` can work for trusted setups, but it does not enforce the requested direct-parent-only authority.
