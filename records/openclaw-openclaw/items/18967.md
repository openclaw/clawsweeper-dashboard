## Summary

Keep open: current main still lacks the requested parent-scoped send path, and the remaining work changes sub-agent cross-session authority, so it needs maintainer API and security design rather than cleanup closure.

## What This Changes

The issue requests a parent-only send API so a sub-agent can message its direct parent session mid-run without arbitrary cross-session targeting.

## Best Possible Solution

Add a narrow core parent-send path that derives the direct parent from trusted spawn/session metadata, rejects missing-parent, sibling, ancestor, and cross-agent targets, and avoids automatic forwarding beyond that parent.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source inspection on current main shows no native parent-only send tool or trusted parent alias while default sub-agent policy still removes sessions_send.

## Solution Assessment

Yes for a dedicated parent-scoped core API or trusted parent alias; no for broadly re-enabling raw sessions_send. The maintainable path should derive the direct parent server-side and enforce same-tree, parent-only, no-chaining behavior.
