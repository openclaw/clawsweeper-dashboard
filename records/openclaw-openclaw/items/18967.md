## Summary

Keep this open: current main still lacks a trusted native parent-scoped send path for sub-agents, and the remaining work is an authorization-sensitive feature rather than stale cleanup.

## What This Changes

Request a parent-only `sessions_send` mechanism so a sub-agent can send progress or questions to its direct parent session without targeting arbitrary sessions.

## Best Possible Solution

Add a narrow core parent-send path, preferably `sessions_send_parent` or a trusted `sessionKey: "parent"` resolver, backed by spawn/session metadata, docs, and regression coverage.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source inspection of current main shows no native parent-only send tool or alias.

## Solution Assessment

Yes for a dedicated parent-scoped core API; no for broadly re-enabling `sessions_send`. The maintainable solution should derive the direct parent server-side and reject missing-parent, sibling, ancestor, cross-agent, and forwarded targets.
