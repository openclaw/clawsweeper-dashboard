## Summary

Keep open: current main still lacks a native direct-parent-only send path for sub-agents, and the requested change affects cross-session authority, so it needs maintainer product and security design rather than cleanup closure.

## What This Changes

Request a parent-only sub-agent session send capability so child agents can send mid-run updates or questions directly to the session that spawned them.

## Best Possible Solution

Add a narrow core parent-send path that derives the direct parent from trusted spawn/session metadata, rejects missing-parent, sibling, ancestor, and cross-agent targets, avoids automatic forwarding beyond that parent, and updates policy, resolver/send behavior, docs, and regression tests.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection on current main shows no native parent-only send tool or trusted parent alias while default sub-agent policy still removes `sessions_send`.

## Solution Assessment

Yes for a dedicated parent-scoped core API or trusted parent alias; no for broadly re-enabling raw `sessions_send`. The maintainable path should derive the direct parent server-side and enforce same-tree, parent-only, no-chaining behavior.
