## Summary

Keep open: current main and the latest release still lack a native direct-parent-only mid-run send path for sub-agents, and the existing raw `sessions_send` opt-in remains broader than the requested same-tree, no-chaining authority.

## What This Changes

Request a parent-scoped `sessions_send_parent` tool or trusted `sessionKey: "parent"` alias so sub-agents can send progress or questions only to their direct parent session.

## Best Possible Solution

Add a narrow parent-send API or trusted `parent` resolver that derives the direct parent from stored spawn/session metadata, enforces same-tree direct-parent-only no-chaining behavior, and updates policy, visibility, docs, and regression tests after maintainer API and security review.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing behavior report. Source inspection on current main gives high confidence that no native parent-only send tool or trusted parent alias exists while default sub-agent policy still denies raw `sessions_send`.

## Solution Assessment

Yes at the direction level for a dedicated parent-scoped core API or trusted parent alias; no for broadly re-enabling raw `sessions_send`. Maintainers still need to choose the public shape and exact same-tree/no-chaining enforcement rules before implementation.
