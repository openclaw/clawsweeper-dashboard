## Summary

Keep open: current main and the latest release still do not provide a native direct-parent-only mid-run send path for sub-agents; the available paths are completion announce delivery, ACP-only parent streaming, and broader raw `sessions_send` opt-ins that do not enforce the requested same-tree/no-chaining authority.

## What This Changes

Request a parent-scoped `sessions_send` capability so sub-agents can send progress updates or questions only to the direct parent session that spawned them.

## Best Possible Solution

Add a narrow parent-send API or trusted `parent` resolver that derives the direct parent from stored spawn/session metadata, enforces same-tree direct-parent-only no-chaining behavior, and updates policy, visibility, docs, and regression tests after maintainer API and security review.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing behavior report. Source inspection gives high confidence that current main lacks a native parent-only send tool or trusted parent alias while default sub-agent policy still denies raw `sessions_send`.

## Solution Assessment

Yes at the direction level for a dedicated parent-scoped core API or trusted parent alias; no for broadly re-enabling raw `sessions_send`. Maintainers still need to choose the public shape and exact same-tree/no-chaining enforcement rules before implementation.
