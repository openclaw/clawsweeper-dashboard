## Summary

Keep open: current main and v2026.6.5 still provide completion announces, ACP-only parent streaming, and broad opt-in access to `sessions_send`, but no trusted direct-parent-only mid-run send capability. The requested authority boundary remains distinct from the broader configuration workaround and requires maintainer product and security judgment.

## What This Changes

Add a trusted mechanism for sub-agents to send mid-run updates or questions only to their direct parent session.

## Best Possible Solution

Add one core-owned direct-parent send operation derived from canonical session metadata, reject sibling, ancestor, unrelated, and cross-agent targets, never forward automatically beyond one hop, and add focused policy, resolution, visibility, nested-sub-agent, and documentation coverage.

## Reproduction Assessment

Yes at the source-contract level. Current main and v2026.6.5 both show that sub-agents deny `sessions_send` by default and that the available opt-in exposes the general tool rather than a parent-only operation.

## Solution Assessment

Yes in architectural direction: resolving the direct parent from trusted spawn/session metadata is narrower and safer than broadly enabling `sessions_send`. The permanent model-facing shape—dedicated tool or reserved `parent` target—still requires maintainer approval.
