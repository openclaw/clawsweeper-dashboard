## Summary

Keep open: current main still lacks a native parent-scoped send path for sub-agents, and the remaining work changes cross-session authority rather than being a cleanup-close candidate.

## What This Changes

The issue requests a parent-scoped way for sub-agents to send mid-run updates or questions only to their direct parent session.

## Best Possible Solution

Add a narrow core parent-send path that derives the direct parent from trusted spawn/session metadata, rejects missing-parent, sibling, ancestor, and cross-agent targets, and does not auto-forward beyond that parent.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source inspection on current main shows no native parent-only send tool or trusted parent alias while default sub-agent policy still removes sessions_send.

## Solution Assessment

Yes for a dedicated parent-scoped core API or trusted parent alias; no for broadly re-enabling raw sessions_send. The maintainable path should derive the direct parent server-side and enforce same-tree, parent-only, no-chaining behavior.
