## Summary

Keep this open: current main still lacks a trusted parent-scoped send path for sub-agents, and the remaining work is an authorization-sensitive feature that needs maintainer design.

## What This Changes

Request a native parent-scoped `sessions_send` path so sub-agents can send mid-run updates or questions only to their direct parent session.

## Best Possible Solution

Add a narrow core parent-send path, preferably `sessions_send_parent` or a trusted `sessionKey: "parent"` resolver, backed by spawn/session metadata, no forwarding, docs, and regression coverage.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source inspection of current main shows no native parent-only send tool or trusted `parent` alias.

## Solution Assessment

Yes for a dedicated parent-scoped core API; no for broadly re-enabling raw `sessions_send`. The maintainable solution should derive the direct parent server-side and reject missing-parent, sibling, ancestor, cross-agent, and auto-forwarded targets.
