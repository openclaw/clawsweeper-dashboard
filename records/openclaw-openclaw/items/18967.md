## Summary

Keep open: current main and v2026.5.12 still lack a native direct-parent-only mid-run send path for sub-agents. The existing announce chain, ACP-only parent streaming, parent-to-child `sessions_send`, and broad sub-agent `sessions_send` opt-in are adjacent but do not provide a server-derived parent target with same-tree and no-chaining guarantees.

## What This Changes

Request adds a native parent-scoped send mechanism so sub-agents can send mid-run updates or questions only to the session that directly spawned them.

## Best Possible Solution

Add a narrow core parent-send path that derives the direct parent from trusted spawn/session metadata, enforces same-tree parent-only no-chaining behavior, and updates policy, resolver/send behavior, docs, and regression tests.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken existing behavior. Source inspection on current main and v2026.5.12 gives high confidence that no native parent-only send tool or trusted parent alias exists while default sub-agent policy still denies raw `sessions_send`.

## Solution Assessment

Yes for a dedicated parent-scoped core API or trusted parent alias; no for broadly re-enabling raw `sessions_send`. Maintainers still need to choose the public shape and security rules before implementation.
