## Summary

Current main still has adjacent broad `sessions_send` visibility/allowlist paths and ACP parent streaming, but no trusted child-to-direct-parent mid-run send API that enforces parent-only, same-tree, and no-chaining behavior.

## What This Changes

Request for a parent-scoped sub-agent send mechanism that lets a sub-agent send mid-run updates or questions only to its direct parent session.

## Best Possible Solution

Define one narrow parent-only send contract derived from canonical active sub-agent metadata, keep broad `sessions_send` opt-ins separate, and update policy, resolver, delivery, docs, and security regression tests together.

## Reproduction Assessment

Yes, at source level. Current main still denies `sessions_send` to sub-agents by default, has no `sessions_send_parent` tool, and does not resolve a trusted `parent` target from sub-agent metadata.

## Solution Assessment

Yes in direction, but not as a broad opt-in to ordinary `sessions_send`. The maintainable solution is a dedicated parent-only operation or trusted parent selector backed by active run metadata and security tests.
