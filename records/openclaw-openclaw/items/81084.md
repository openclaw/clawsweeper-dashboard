## Summary

Keep open: current main still routes Microsoft Teams channel replies into thread-qualified sessions and has no supported channel-memory opt-out. This is a valid feature/config request, but it needs maintainer agreement on the policy shape before implementation.

## What This Changes

The issue requests a Teams channel/session policy that lets channel-bound agents keep canonical channel memory instead of always using per-thread sessions.

## Best Possible Solution

Add a Teams-owned session routing policy that defaults to per-thread isolation, supports an explicit channel-memory mode for channel-bound agents, and updates schema, docs, and focused regression tests without weakening the existing isolation default.

## Reproduction Assessment

Yes for the current behavior: source inspection shows channel replies are routed through a helper that appends a Teams thread id when present, and no config branch exists to keep the canonical channel key. This reproduces the missing opt-out, not a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the policy contract: the requested direction is reasonable if it stays Teams-owned and default-safe, but config naming, scope, and precedence need product review before implementation.
