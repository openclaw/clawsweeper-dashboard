## Summary

Keep open: current main still routes Microsoft Teams channel replies into thread-qualified sessions with no supported channel-memory opt-out, and the remaining work is a new Teams routing policy/config decision rather than an already-fixed cleanup item.

## What This Changes

The issue requests an opt-in Microsoft Teams channel session-routing policy that keeps channel-bound agents on the canonical channel session instead of splitting replies by Teams thread.

## Best Possible Solution

Add a Teams-owned session routing policy that defaults to thread isolation, supports an explicit channel-memory mode for channel-bound agents, and updates types, schema, docs, and focused regression tests.

## Reproduction Assessment

Yes for the current behavior: source inspection shows channel replies are routed through a helper that appends a Teams thread id when present, and no config branch exists to keep the canonical channel key. This reproduces the missing opt-out, not a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the policy contract: the requested direction is reasonable if it stays Teams-owned and default-safe, but the config name, scope, and precedence need product review before implementation.
