## Summary

Keep open: current main still routes Microsoft Teams channel replies into thread-qualified sessions and does not expose a supported channel-memory opt-out; this is a real feature/config policy request rather than an implemented cleanup item.

## What This Changes

Request a Microsoft Teams session-routing policy that lets channel-bound agents keep canonical channel memory instead of always forking channel replies into per-thread sessions.

## Best Possible Solution

Add a Teams-owned session routing policy that defaults to thread isolation, supports an explicit channel-memory mode for channel-bound agents, and updates the schema, docs, and focused regression tests.

## Reproduction Assessment

Yes for the current behavior: source inspection shows Teams channel replies are routed through a helper that appends a thread id when present, and the schema/docs expose no opt-out. This reproduces the missing capability rather than a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the policy contract: the requested direction is reasonable if it stays Teams-owned and default-safe, but the config name, scope, and precedence need product review before implementation.
