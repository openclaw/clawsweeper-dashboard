## Summary

Keep open: current main and the latest release still route Microsoft Teams channel thread replies into thread-qualified session keys with no supported channel-memory opt-out, so this is a real feature/config policy request rather than completed cleanup.

## What This Changes

Request a Microsoft Teams session-routing policy that lets channel-bound agents keep a canonical channel session instead of always splitting replies by Teams thread.

## Best Possible Solution

Add a Teams-owned session-routing policy that defaults to thread isolation, supports an explicit channel-memory mode for channel-bound agents, and updates schema, docs, and focused regression tests without weakening the existing bleed fix.

## Reproduction Assessment

Yes for the current behavior: source and tests show Teams channel thread replies are routed to thread-qualified session keys, and schema/docs expose no channel-memory opt-out. This reproduces the missing feature/config path, not a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the policy contract: the requested opt-in direction is reasonable if default-safe and Teams-owned, but the name, scope, and precedence need product review before implementation.
