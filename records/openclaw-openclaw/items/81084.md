## Summary

Keep open: current main still routes Microsoft Teams channel replies into thread-qualified session keys with no supported channel-memory opt-out, so this remains a real feature/config policy request rather than cleanup.

## What This Changes

Request an opt-in Microsoft Teams session-routing policy so channel-bound agents can keep canonical channel memory instead of per-thread sessions.

## Best Possible Solution

Add a Teams-owned session routing policy that defaults to thread isolation, supports an explicit channel-memory mode for channel-bound agents, and updates schema, docs, generated metadata, and focused regression coverage.

## Reproduction Assessment

Yes for the current behavior: source and tests show Teams channel replies append a thread-qualified session key, and the schema/docs expose no opt-out. This reproduces the missing capability, not a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the policy contract: an opt-in Teams-owned routing policy is likely the right direction, but the exact scope and precedence need product review before implementation.
