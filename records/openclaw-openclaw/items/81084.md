## Summary

Keep open: current main still routes Microsoft Teams channel-thread replies into thread-qualified sessions and does not expose a supported channel-memory opt-out. This is a feature/configuration policy request, not an already-fixed cleanup item.

## What This Changes

Request a Microsoft Teams session-routing policy/config option so channel-bound agents can keep replies on the canonical channel session instead of always splitting by Teams thread.

## Best Possible Solution

Add a Teams-owned session-routing policy that defaults to thread isolation, supports an explicit channel-memory mode for channel-bound agents, and updates schema, docs, and focused regression coverage.

## Reproduction Assessment

Yes for the current behavior: source inspection shows channel replies pass through a helper that appends a Teams thread id when present, and the strict Teams config schema exposes no opt-out branch. This reproduces the missing policy surface, not a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the policy contract. The requested direction is reasonable if it stays Teams-owned and default-safe, but the config name, scope, and precedence need product review before implementation.
