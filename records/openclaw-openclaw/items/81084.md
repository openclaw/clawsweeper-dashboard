## Summary

Keep open: current main and the latest release still route Microsoft Teams channel replies through thread-suffixed session keys, while the requested opt-out is a new Teams session-routing policy/config surface that remains unimplemented.

## What This Changes

The issue requests a Microsoft Teams channel session-routing policy so channel-bound agents can opt out of per-thread session keys.

## Best Possible Solution

Add a maintainer-approved Teams-owned policy that keeps thread isolation as the default while allowing explicit channel-session routing for selected channel-bound agents, with handler, feedback, config, docs, and regression coverage aligned.
