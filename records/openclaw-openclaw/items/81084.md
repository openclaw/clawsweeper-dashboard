## Summary

Keep open: current main still has no supported Microsoft Teams session-routing opt-out for channel-bound agents, and the remaining request is a new Teams-owned config/product policy rather than a cleanup close. The related nested-key fix is already merged, but it only normalizes malformed `:thread:` suffixes and does not provide canonical channel memory.

## What This Changes

Request a Teams channel session policy that lets selected channel-bound agents keep replies on the canonical channel session instead of always using thread-qualified sessions.

## Best Possible Solution

Add a Teams-owned session routing policy that defaults to thread isolation, supports global/team/channel precedence, and updates schema/types/docs plus focused regression tests without weakening existing cross-thread isolation.

## Reproduction Assessment

Yes for the current behavior: source inspection and existing tests show Teams channel replies derive thread-qualified session keys from `conversationMessageId` or `replyToId`. This proves the missing opt-out, not a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the product contract: the requested direction is maintainable if it remains Teams-owned and default-preserving, but the config name, scope, and precedence need review first.
