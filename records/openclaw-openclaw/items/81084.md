## Summary

Keep open: current main still routes Microsoft Teams channel thread replies into thread-qualified sessions with no supported channel-memory opt-out, so this remains a feature/config policy request rather than cleanup.

## What This Changes

Request a supported Microsoft Teams session-routing option so channel-bound agents can keep one canonical channel session instead of splitting replies into per-thread sessions.

## Best Possible Solution

Add a Teams-owned session routing policy that defaults to current thread isolation, supports an explicit channel-memory mode for channel-bound agents, and updates schema, docs, and focused regression coverage.

## Reproduction Assessment

Yes for the current behavior: source inspection shows channel replies are always routed through the thread-session helper when a Teams thread id is present, and no config branch can keep the canonical channel key. I did not establish a live Teams fresh-room reproduction in this read-only pass.

## Solution Assessment

Unclear until maintainers choose the policy contract. The requested direction is maintainable if it stays Teams-owned and default-safe, but the config name, scope, and precedence need product review first.
