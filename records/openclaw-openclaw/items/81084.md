## Summary

Keep open: current main still routes Microsoft Teams channel-thread replies into thread-qualified sessions and does not expose a supported channel-memory opt-out. This is a valid feature/config request, but it needs maintainer product agreement before an autonomous fix PR is safe.

## What This Changes

The issue requests a Microsoft Teams channel-session routing policy that lets channel-bound agents keep canonical channel memory instead of always forking replies into thread-qualified sessions.

## Best Possible Solution

Add a Teams-owned, default-preserving session routing policy that supports an explicit channel-memory mode for configured channel-bound agents, with schema, docs, and focused regression coverage.

## Reproduction Assessment

Yes for the current behavior: source inspection shows Teams channel replies are always routed through the thread-session helper when a thread id is present, and the schema/docs expose no opt-out. This proves the missing capability, not a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the policy contract. The requested direction is maintainable if it stays Teams-owned, default-safe, and explicit about global/team/channel precedence.
