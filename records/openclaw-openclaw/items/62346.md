## Summary

Keep open. The PR is not merge-ready because it changes a helper that is not used by the current production replay path, and the proposed drop-all behavior can discard valid assistant content for an `incomplete-text` classification.

## What This Changes

The PR changes `sanitizeThinkingForRecovery` so `incomplete-text` recovery drops the last assistant turn with `prefill: false`, and updates the focused unit test expectation.

## Best Possible Solution

Retarget the fix to the active runtime replay path, avoid assistant prefill for genuinely unfinished turns, preserve meaningful visible assistant content, and leave the broader linked recovery-state issue open for follow-up work.

## Reproduction Assessment

Yes for the PR defects: source search reproduces that the changed helper is test-only, and the classifier/drop-all edge case follows directly from `assessLastAssistantMessage`. The broader aborted-run inconsistency is supported by the PR comments but still needs focused live or integration reproduction after a runtime fix.

## Solution Assessment

No. The current PR is not the best fix because it changes an inactive helper and would be too broad if wired in as-is; a safer fix should operate where provider replay messages are built and distinguish unfinished fragments from valid assistant output.
