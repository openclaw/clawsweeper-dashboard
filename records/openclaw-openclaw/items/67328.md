Keep open. ## What This Changes

The PR adds representable `z.string()` pipes to transform-backed config fields, filters empty any-schema union branches in the Control UI analyzer, updates generated baseline hashes and changelog text, and adds JSON Schema regression tests.

## Best Possible Solution

Land this branch or an equivalent narrow fix after rebasing onto current main and passing the merge gates, preserving the schema pipes, analyzer metadata alignment, generated baseline updates, and focused regression tests.

## Reproduction Assessment

Yes, from source inspection rather than an executed local repro. Current main and `v2026.5.18` still pass raw transform branches into `toJSONSchema({ unrepresentable: "any" })`, and Zod 4.4.3 maps unrepresentable transforms to empty `{}` schema nodes.

## Solution Assessment

Yes. Piping the transform outputs through `z.string()` matches the existing `DiscordIdSchema` pattern and Zod's pipe JSON Schema behavior, while the UI analyzer filter is conservative defense-in-depth for any future typed-plus-empty union branches.
