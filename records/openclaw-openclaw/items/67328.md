## Summary

Keep open: this is a valid, focused bug-fix PR and current main still has the unrepresentable schema transforms it addresses. The proposed schema fix is narrow and matches the existing local Zod pipe pattern, but the analyzer fallback still needs one small metadata-key alignment before maintainer landing.

## What This Changes

The PR adds representable `z.string()` pipes to `meta.lastTouchedAt` and Docker `setupCommand`, filters empty any-schema branches in the Control UI analyzer, updates generated baseline hashes, adds a schema regression test, and records a changelog entry.

## Best Possible Solution

Keep the schema-level pipe fixes, align analyzer any-schema metadata handling with the renderer, refresh generated baselines if needed, and land after focused config/UI validation.

## Reproduction Assessment

Yes. Current main still has the unpiped transform branches, base schema generation uses Zod `unrepresentable: "any"`, and Zod 4.4.3 emits empty schemas for unrepresentable transforms while pipes resolve through the output schema.

## Solution Assessment

Mostly yes. Piping the transform outputs through `z.string()` is the narrow maintainable fix and matches an existing local pattern, but the analyzer fallback should also classify `tags` and `x-tags` as metadata before landing.
