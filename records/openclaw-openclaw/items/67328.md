## What This Changes

The branch makes `meta.lastTouchedAt` and Docker `setupCommand` transform outputs JSON-Schema-representable with `.pipe(z.string())`, adds a Control UI union fallback for empty any-schema branches, updates generated hash baselines, adds a regression test, and records the user-visible fix in the changelog.

## Best Possible Solution

Land the schema-output fix after aligning analyzer any-schema metadata handling with the renderer and keeping the focused config/UI regression coverage green.

## Reproduction Assessment

Yes. Source inspection shows current main still has unpiped transform branches under the config schema's `unrepresentable: "any"` JSON Schema generation mode, and the PR discussion supplies before/after terminal schema output for the affected field.

## Solution Assessment

Mostly yes. Piping transform outputs through `z.string()` is the narrow maintainable fix and matches an existing local schema pattern; the UI fallback should also treat `tags` and `x-tags` as metadata before merge.
