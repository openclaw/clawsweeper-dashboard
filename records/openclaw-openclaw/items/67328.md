The PR pipes `meta.lastTouchedAt` and Docker `setupCommand` transforms through `z.string()`, filters empty any-schema union branches in the Control UI analyzer, updates config docs hashes/changelog, and adds `lastTouchedAt` schema regression coverage.

## Best Possible Solution

Land the narrow schema/UI fix after reverting Plugin SDK hash churn, aligning analyzer metadata handling with the renderer, and keeping focused config/UI validation green.

## Reproduction Assessment

Yes. Current main still has unpiped transform branches under the config schema path that uses `unrepresentable: "any"`, and the PR discussion supplies before/after terminal schema output for the same fields.

## Solution Assessment

Mostly yes. Piping string-output transforms through `z.string()` is the narrow maintainable fix, but the branch should drop unrelated Plugin SDK hash churn and make the defensive UI filter metadata-consistent.
