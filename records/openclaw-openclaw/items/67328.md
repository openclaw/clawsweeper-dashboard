## What This Changes

This PR makes transform-backed `meta.lastTouchedAt` and Docker `setupCommand` config fields representable in public JSON Schema, switches base config schema generation to Zod input mode, updates Control UI union analysis/tests, generated baselines, and changelog text.

## Best Possible Solution

Preserve accepted transform-backed input shapes in the public schema only if maintainers affirm input-mode as the config schema contract; otherwise replace the broad mode switch with a narrower representation fix that leaves existing schema consumers stable.

## Reproduction Assessment

Yes. Current main has transform-backed fields fed into Zod JSON Schema generation with `unrepresentable: "any"`, and Zod’s contract plus the UI analyzer code explain how that yields untyped `{}` branches that the form marks unsupported; I did not run a browser repro in this read-only review.

## Solution Assessment

Unclear. The field-level `.pipe(z.string())` repair is narrow and consistent with the dependency contract, but the PR now relies on a schema-wide `io: "input"` contract change that should be explicitly approved or narrowed before merge.
