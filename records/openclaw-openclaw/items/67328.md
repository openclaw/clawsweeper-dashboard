## What This Changes

The PR makes `meta.lastTouchedAt` and Docker `setupCommand` transforms JSON-Schema-representable, filters empty any-schema union branches in the Control UI analyzer, and updates tests, baseline hashes, and changelog.

## Best Possible Solution

Land the schema-output fix after aligning analyzer metadata handling with the renderer and letting the latest CI run finish.

## Reproduction Assessment

Yes. Current main has transform branches that Zod cannot represent under the config schema's `unrepresentable: "any"` mode, and the PR discussion provides before/after terminal JSON Schema output for the affected field.

## Solution Assessment

Mostly yes. Piping the transform outputs through `z.string()` is the narrow local fix and matches the existing `DiscordIdSchema` pattern, while the defensive UI filter should also align its metadata keys with the renderer.
