## Summary

Keep open. Current main still has the unrepresentable config-schema transforms and generated empty schema nodes, so this PR is still relevant; the branch is a narrow plausible fix but should address the concrete review cleanup before merge.

## What This Changes

This PR pipes the `meta.lastTouchedAt` and Docker `setupCommand` Zod transforms to representable string outputs, updates the generated base config schema, adds a `lastTouchedAt` JSON Schema regression test, and filters empty union branches in the Control UI analyzer.

## Best Possible Solution

Land a narrow version of this PR after resolving the review comments: make both transform outputs representable in JSON Schema, keep the UI analyzer fallback metadata-consistent, regenerate the current base schema, add regression coverage for both affected fields, and record the user-facing Control UI fix in the changelog.

## Reproduction Assessment

Yes. Current main reproduces the underlying schema input with `meta.lastTouchedAt` as `anyOf: [{ type: "string" }, {}]` and Docker `setupCommand: {}`, and the Control UI analyzer currently treats untyped union branches as unsupported.

## Solution Assessment

Mostly yes. Piping transform outputs to `z.string()` is the narrowest source-level fix and matches the existing `DiscordIdSchema` pattern, but the PR should also make the tests non-vacuous, cover `setupCommand`, align analyzer metadata keys, and add the required changelog entry.
