## What This Changes

The PR makes transform-backed config fields emit representable string JSON Schema, filters empty any-schema union branches in the config-form analyzer, updates generated baseline hashes, and adds regression tests for `meta.lastTouchedAt` and Docker `setupCommand`.

## Best Possible Solution

Land this branch or an equivalent narrow fix after the typecheck gate is green, preserving the schema pipe fixes, analyzer metadata alignment, generated baselines, and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main plus Zod 4.4.3 source gives a high-confidence path: unpiped transforms are passed to `toJSONSchema({ unrepresentable: "any" })`, which yields empty schema branches under `anyOf` that the analyzer cannot classify as renderable.

## Solution Assessment

Yes. Piping the transform outputs through `z.string()` matches the existing `DiscordIdSchema` pattern and fixes the schema contract at the source, while the analyzer filter is a conservative fallback for future empty any-schema branches.
