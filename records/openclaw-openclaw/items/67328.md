## Summary

Keep open: the core schema fix is well supported, but the branch still has unrelated Plugin SDK baseline hash churn that should not be part of this config/UI fix, and the new UI fallback still uses analyzer metadata rules that disagree with the renderer.

## What This Changes

The PR makes the `meta.lastTouchedAt` and Docker `setupCommand` transforms JSON-Schema-representable, adds a Control UI empty-branch union filter, updates generated hash files and changelog, and adds a `lastTouchedAt` schema regression test.

## Best Possible Solution

Land the narrow config/UI schema fix after reverting the unrelated Plugin SDK baseline hash changes, aligning analyzer metadata handling with the renderer, and running focused config/UI/schema validation.

## Reproduction Assessment

Yes. Current main source plus Zod's JSON Schema contract show that these transform branches are unrepresentable under `unrepresentable: "any"`, and the PR discussion supplies before/after terminal schema output for the same path.

## Solution Assessment

Mostly yes. Piping the transform outputs through `z.string()` is the narrow source-level fix and follows the existing pattern, but the branch should not carry unrelated Plugin SDK baseline hashes and the defensive UI filter should share the renderer's metadata classification.
