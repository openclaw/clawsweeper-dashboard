## Summary

This should stay open: current main still has the reported unrepresentable config transform paths, and the PR is a plausible narrow fix, but merge should wait for real Control UI proof plus a small analyzer consistency cleanup.

## What This Changes

This PR pipes the `meta.lastTouchedAt` and Docker `setupCommand` Zod transform outputs to strings, adds a Control UI union filter for empty any-schema branches, updates one schema regression test, and records the user-visible fix in the changelog.

## Best Possible Solution

Land a narrow version that keeps the transform output pipes, makes the analyzer any-schema metadata handling consistent with the renderer, and attaches after-fix Control UI proof for both affected fields.

## Reproduction Assessment

Yes, source-reproducible. Current main still passes unpiped transform schemas for `meta.lastTouchedAt` and `setupCommand` into the `toJSONSchema(..., unrepresentable: "any")` path consumed by the Control UI schema.

## Solution Assessment

Yes for the source-level direction: piping transform outputs to `z.string()` matches the existing Discord ID pattern and keeps runtime output unchanged. The UI fallback should still align metadata handling before merge, and the contributor needs to supply real Control UI proof.
