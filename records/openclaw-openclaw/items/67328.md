## Summary

Keep open: this is a focused, valid fix for a live config JSON-Schema/Control UI bug with sufficient real behavior proof; one small analyzer metadata cleanup remains before landing.

## What This Changes

The PR makes the `meta.lastTouchedAt` and Docker `setupCommand` Zod transform outputs representable in JSON Schema, filters empty union branches in the Control UI analyzer, updates generated baseline hashes, adds schema coverage, and records a changelog entry.

## Best Possible Solution

Land the schema-output fix after aligning analyzer metadata handling with the renderer and running focused config/UI validation.

## Reproduction Assessment

Yes. Current main has the unrepresentable transform shapes in both affected fields, and the PR discussion supplies before/after terminal schema output showing the empty `{}` branch on main and typed branches after the fix.

## Solution Assessment

Mostly yes. Adding `.pipe(z.string())` is the narrow source-level fix and follows the existing `DiscordIdSchema` pattern, but the defensive UI fallback should also classify `tags` and `x-tags` as metadata.
