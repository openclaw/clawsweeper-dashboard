## Summary

Keep open: current main still has the reported schema-rendering bug and this PR is a useful narrow fix, but the analyzer-side fallback still has one metadata consistency issue to address before landing.

## What This Changes

The PR adds representable `z.string()` pipes to the `meta.lastTouchedAt` and Docker `setupCommand` transforms, filters empty any-schema union branches in the Control UI analyzer, updates generated baseline hashes, adds regression coverage, and records a changelog entry.

## Best Possible Solution

Land the schema-output fix after aligning analyzer metadata-only branch handling with the renderer and refreshing the branch against current main.

## Reproduction Assessment

Yes. Current main still has the unpiped transform branches and uses Zod JSON Schema generation with `unrepresentable: "any"`; the PR body also provides before/after terminal schema output from real checkouts.

## Solution Assessment

Mostly yes. Piping transform outputs through `z.string()` is the narrow maintainable fix and matches an existing local pattern, but the analyzer fallback should also classify `tags` and `x-tags` as metadata before landing.
