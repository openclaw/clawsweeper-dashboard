## Summary

This PR is a plausible fix for the Control UI unsupported-schema regression, but the latest diff still needs maintainer handling before merge because it changes the public config schema contract to Zod input shapes and leaves Docker setupCommand editing routed through the JSON-textarea path.

## What This Changes

The PR makes transform-backed config schema fields representable by piping transform outputs, switches base config schema generation to Zod input mode, adjusts Control UI schema analysis, updates generated baselines/changelog, and adds regression tests for meta.lastTouchedAt and Docker setupCommand.

## Best Possible Solution

Land after maintainers explicitly accept the input-shaped public schema contract and setupCommand either renders as a normal command string or is intentionally documented and tested as JSON editing.

## Reproduction Assessment

Yes. Source inspection on current main shows transform-backed config fields are passed to Zod JSON Schema generation with unrepresentable:any, and Zod documents that transforms are unrepresentable and become {} under that mode.

## Solution Assessment

No, not yet. The targeted .pipe changes are reasonable, but the broad io:"input" schema contract change and setupCommand JSON-textarea behavior need maintainer approval or a narrower UI/schema repair before this is the best merge shape.
