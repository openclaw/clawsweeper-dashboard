## Summary

Keep open: this is a focused, valid bug-fix PR for the Control UI config schema path, but the new defensive union filter still uses analyzer metadata rules that are narrower than the renderer's rules. Real behavior proof and the security pass are otherwise adequate.

## What This Changes

The PR pipes `meta.lastTouchedAt` and Docker `setupCommand` Zod transforms through `z.string()`, adds a JSON Schema regression test and changelog entry, and filters empty any-schema union branches in the Control UI analyzer.

## Best Possible Solution

Land the narrow schema-output fix after aligning analyzer metadata handling with the renderer and letting the final relevant CI checks complete.

## Reproduction Assessment

Yes. The current-main source path is high-confidence: Zod transforms are passed to `toJSONSchema(..., unrepresentable: "any")`, which produces empty `{}` schema branches that the Control UI analyzer cannot render, and the PR body includes before/after terminal schema output.

## Solution Assessment

Mostly yes. Adding `.pipe(z.string())` is the narrow source-level fix and matches the existing `DiscordIdSchema` pattern; the defensive UI fallback should also treat `tags` and `x-tags` as metadata before merge or as a small repair-lane follow-up.
