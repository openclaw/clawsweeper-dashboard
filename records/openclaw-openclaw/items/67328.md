Full review comments:

- **[P2] Handle setupCommand before exposing input-mode unions:** `src/config/schema-base.ts:233`
  - body: Adding `io: "input"` makes the public schema expose `setupCommand` as `string | string[]`. The current renderer treats array/string unions as a complex union and uses a JSON textarea, so an existing command string is shown as JSON and ordinary edits like `apt-get update` fail JSON parsing and reset. Either keep the output-shaped schema for this public form path or add explicit setupCommand string-or-array handling before relying on input-mode schema for the UI.
