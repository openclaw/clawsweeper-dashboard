## Summary

Keep open. This is a focused PR for a real current-main config schema/UI bug, and the main fix direction matches the Zod JSON Schema contract; one low-priority analyzer metadata edge remains from the newly added any-schema filter.

## What This Changes

The PR adds representable string output pipes for `meta.lastTouchedAt` and Docker `setupCommand`, adds a Control UI union filter for empty any-schema branches, extends config schema coverage, and records the user-visible fix in the changelog.

## Best Possible Solution

Land this fix after aligning the analyzer any-schema metadata handling with the renderer and keeping focused regression coverage for the affected schema paths.

## Reproduction Assessment

Yes. Source inspection plus Zod v4.4.3's JSON Schema contract show that current-main transforms become `{}` under `unrepresentable: "any"`, and the current analyzer rejects untyped union branches.

## Solution Assessment

Yes, with a small polish item. Declaring transform outputs with `.pipe(z.string())` is the narrow schema-level fix and matches an existing local pattern; the new analyzer fallback should also treat `tags` and `x-tags` as metadata-only any-schema keys.
