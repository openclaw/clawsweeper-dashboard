## What This Changes

The PR makes transform-backed config fields representable in generated schema output, switches base config schema generation to Zod input mode, updates generated baseline hashes, adds regression tests, and adjusts Control UI schema analysis.

## Best Possible Solution

Land a narrow fix only after maintainer signoff on input-mode public schema generation, with Control UI coverage that proves both meta.lastTouchedAt and Docker setupCommand render and edit through the intended form controls.

## Reproduction Assessment

Yes. Current main has transform-backed config fields passed to Zod JSON Schema generation with unrepresentable:any, and Zod documents that transforms are unrepresentable and become {} under that mode.

## Solution Assessment

No, not yet. Typing the transforms is a good direction, but the latest input-mode public schema change needs maintainer acceptance and setupCommand's form editing path should be aligned before merge.
