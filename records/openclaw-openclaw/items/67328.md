## Summary

Keep open: the PR addresses a real current-main Control UI/config schema bug and the supplied proof/CI look adequate, but one small analyzer metadata mismatch remains in the new defensive filter.

## What This Changes

The PR makes `meta.lastTouchedAt` and Docker `setupCommand` transforms JSON-Schema-representable, filters empty any-schema union branches in the config form analyzer, updates generated baseline hashes, adds a regression test, and records a changelog entry.

## Best Possible Solution

Land the narrow schema-output fix after aligning analyzer metadata handling with the renderer and keeping the branch green.

## Reproduction Assessment

Yes. Current main has the unpiped transform branches and the config schema uses Zod's `unrepresentable: "any"` mode, which the upstream Zod contract maps to `{}` for transforms; I did not execute local schema generation because this pass is read-only.

## Solution Assessment

Mostly yes. Piping transform outputs through `z.string()` is the narrow maintainable fix and matches the existing `DiscordIdSchema` pattern, but the defensive UI filter should also classify tag-only metadata branches the same way the renderer does.

## Review Findings

Overall correctness: patch is correct
