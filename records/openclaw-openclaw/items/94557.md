## Summary

Keep open: current main and v2026.6.9 still accept model-id normalization only from plugin manifests, while the requested provider-config key remains absent from the strict provider schema. The direct fix PR was closed unmerged, and the remaining choice is a compatibility-sensitive provider/config product decision rather than conservative cleanup.

## What This Changes

The issue requests a new `models.providers.<provider>.modelIdNormalization` config field so native providers routed through Cloudflare AI Gateway REST or Unified endpoints can send provider-prefixed wire model IDs.

## Best Possible Solution

Maintainers should decide whether endpoint-specific model-id normalization belongs in generic provider config or in the Cloudflare plugin bridge, then implement the chosen path with schema/types/docs, catalog and picker coverage, and redacted live Cloudflare proof.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.9 have a strict provider config schema without `modelIdNormalization`, while normalization policies are collected only from plugin manifests. I did not run a credentialed Cloudflare request in this read-only review.

## Solution Assessment

Unclear. The proposed config knob is plausible and narrow, but maintainers need to decide whether the durable fix is generic provider-config normalization or Cloudflare plugin-owned REST bridge behavior.
