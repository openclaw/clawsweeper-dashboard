## Summary

Keep open: current main and the latest release still lack provider-config `modelIdNormalization`, and the direct implementation PR is closed unmerged. The request is source-reproducible, but it adds public provider/config surface and needs maintainer product review.

## What This Changes

The issue requests accepting `modelIdNormalization.prefixWhenBare` under `models.providers.<provider>` so configured native providers can send Cloudflare AI Gateway's required provider-prefixed wire model IDs.

## Best Possible Solution

Keep this issue open until maintainers choose the generic config versus Cloudflare bridge boundary, then land a revised implementation with schema/types/docs alignment, catalog propagation, focused tests, and redacted live or terminal proof.

## Reproduction Assessment

Yes, source-level: current main has a strict provider config schema without `modelIdNormalization`, and normalization policies are collected from plugin manifests rather than `models.providers.<provider>` config. I did not run a credentialed Cloudflare request in this read-only review.

## Solution Assessment

Unclear. The requested config knob is plausible and narrow, but maintainers still need to decide whether the durable boundary is a generic provider config contract or Cloudflare provider-owned bridge behavior.
