## Summary

Keep open: current main and v2026.6.9 still lack provider-config `modelIdNormalization`, the direct candidate PR was closed unmerged, and the remaining work needs maintainer direction on whether this is a generic provider-config contract or part of the broader Cloudflare AI Gateway bridge.

## What This Changes

Request to let operators set `modelIdNormalization.prefixWhenBare` under `models.providers.<provider>` so native providers routed through Cloudflare AI Gateway can send provider-prefixed wire model IDs.

## Best Possible Solution

Keep this issue open until maintainers choose the provider-config versus Cloudflare-bridge boundary, then land a repaired implementation with schema/types/docs alignment, cfg-backed propagation, regression coverage, and redacted Cloudflare proof.

## Reproduction Assessment

Yes, at source level: current main has a strict provider config schema without `modelIdNormalization`, and normalization policies are collected from manifests rather than `models.providers.<provider>` config. I did not run a credentialed Cloudflare request in this read-only review.

## Solution Assessment

Unclear. The config knob is a plausible narrow mechanism, but the best permanent fix needs maintainer choice across generic config surface, Cloudflare bridge ownership, docs/types, propagation, and live proof.
