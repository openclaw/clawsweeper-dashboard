## What This Changes

The issue requests an opt-in `modelIdNormalization.prefixWhenBare` setting under `models.providers.<provider>` so native providers routed through Cloudflare AI Gateway can send provider-prefixed wire model IDs.

## Best Possible Solution

Keep this issue open until maintainers choose the provider-config versus Cloudflare-bridge boundary, then land or revise the linked PR with schema/types/docs, focused normalization coverage, and redacted Cloudflare proof.
