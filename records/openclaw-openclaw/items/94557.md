## Summary

Keep open: current main and v2026.6.9 still have manifest-only model-id normalization and no accepted `models.providers.<provider>.modelIdNormalization` config field, while the direct fix PR closed unmerged after review found remaining contract and proof gaps.

## What This Changes

The issue requests a provider-configured `modelIdNormalization.prefixWhenBare` option so native providers can send Cloudflare AI Gateway REST/Unified `author/model` IDs without changing the selected OpenClaw provider.

## Best Possible Solution

Maintainers should choose whether endpoint-specific model-id normalization is a generic provider-config contract or Cloudflare/provider-owned behavior, then implement the chosen path with schema/types, docs, focused normalization coverage, and redacted live Cloudflare proof.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.9 have a strict provider config schema with no `modelIdNormalization` field, while normalization policies are collected from plugin manifests. I did not run a credentialed Cloudflare request in this read-only review.

## Solution Assessment

Unclear. The proposed config knob is plausible and narrow, but the best permanent fix may be provider-owned Cloudflare REST/bridge support rather than a new generic config surface.
