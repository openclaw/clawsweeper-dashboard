## Summary

Keep open: current main and v2026.6.9 still lack provider-config `modelIdNormalization`, and the direct candidate PR was closed unmerged. The request is source-reproducible, but it adds provider/config surface and still needs maintainer product review before another implementation attempt.

## What This Changes

Request to accept `modelIdNormalization.prefixWhenBare` under `models.providers.<provider>` so configured native providers can send provider-prefixed model IDs through Cloudflare AI Gateway REST-style endpoints.

## Best Possible Solution

Keep this issue open until maintainers choose the provider-config versus Cloudflare-bridge boundary, then land a focused implementation with schema/types/docs alignment, catalog/picker coverage, and redacted Cloudflare proof.

## Reproduction Assessment

Yes, source-level: current main has a strict provider config schema without `modelIdNormalization`, and normalization policies are collected from plugin manifests rather than `models.providers.<provider>` config. I did not run a credentialed Cloudflare request in this read-only review.

## Solution Assessment

Unclear. The requested config knob is plausible and narrow, but maintainers still need to confirm whether generic provider-config normalization or Cloudflare-provider bridge behavior is the best permanent boundary.
