## Summary

Keep open. The reported Telegram MEDIA/reply-tag bug is still source-reproducible on current main, but this PR is not merge-ready because it lacks real Telegram proof, is stale against the current extension-based Telegram path, and the parser change can corrupt unrelated reply whitespace.

## What This Changes

The PR changes reply directive parsing, outbound reply normalization, Telegram reply delivery wiring, and Telegram tests so inline MEDIA directives and reply tags are normalized before Telegram sends.

## Best Possible Solution

Port the parser-order fix to current main by stripping only reply tags before media extraction, preserving ordinary whitespace, keeping Telegram delivery on the extension outbound-plan path, and adding focused tests plus redacted live Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main splits MEDIA directives before reply tags are stripped, while media extraction only recognizes line-start `MEDIA:`, so a same-line reply tag can hide the media directive.

## Solution Assessment

No. The goal is valid, but the patch should be ported to the current `extensions/telegram` delivery path and narrowed so the pre-media step strips reply tags without normalizing unrelated whitespace.
