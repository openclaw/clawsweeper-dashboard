## What This Changes

The branch changes reply directive parsing, shared outbound reply normalization, legacy Telegram delivery/native command wiring, and Telegram delivery tests so inline MEDIA and reply tags are normalized before Telegram sends.

## Best Possible Solution

Port a current-main fix that strips only reply tags before MEDIA extraction without broad whitespace normalization, updates extension-era Telegram delivery coverage, and includes real Telegram proof of reply-tagged media delivery.

## Reproduction Assessment

Yes, at source level. Current main parses MEDIA before stripping reply tags, and MEDIA extraction only recognizes lines whose trimmed start is `MEDIA:`, so a leading reply tag can hide the media token.

## Solution Assessment

No. The bug target is valid, but this PR should be replaced or repaired with a current-main extension port that strips reply tags for the media pre-parse path without normalizing unrelated reply whitespace.
