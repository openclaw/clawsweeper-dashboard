## Summary

Keep open: current main still has the parser-order bug for reply-tagged MEDIA lines, but this PR is not merge-ready because the parser change can corrupt whitespace, the Telegram delivery edits target stale pre-plugin paths, and required real Telegram proof is still missing.

## What This Changes

The PR changes reply directive parsing, shared outbound reply normalization, Telegram bot delivery wiring, and Telegram delivery tests to deliver reply-tagged inline MEDIA directives as Telegram media.

## Best Possible Solution

Apply a narrow parser-order fix on current main that strips only reply tags before MEDIA extraction, add current extensions/telegram coverage, and attach redacted live Telegram proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: parseReplyDirectives extracts media before removing reply tags, while splitMediaFromOutput only recognizes line-leading MEDIA tokens.

## Solution Assessment

No. The parser-order fix is the right direction, but this PR normalizes whitespace too broadly and edits stale Telegram delivery paths; the safer fix is a narrow parser change plus current plugin tests and live proof.
