## Summary

Keep open: the PR targets a real Telegram reply-media bug, but it is not mergeable as-is because it still needs real Telegram proof and the parser change would apply directive whitespace normalization to untagged reply text.

## What This Changes

The branch changes shared reply directive parsing, outbound reply normalization, Telegram dispatch/native command delivery, and Telegram delivery tests so inline MEDIA and reply tags are normalized before sending replies.

## Best Possible Solution

Carry forward the narrow parser fix on current main by stripping reply tags before MEDIA extraction without normalizing untagged text, add focused parser and Telegram delivery regression coverage, then require redacted live Telegram proof before merge.

## Reproduction Assessment

Yes, from source inspection: current main still splits MEDIA before stripping reply tags, while the media splitter only recognizes MEDIA at the start of the trimmed line. I did not run tests because this review must keep the checkout read-only.

## Solution Assessment

No. The intended ordering fix is right, but this implementation should be reworked so only reply-tag stripping happens before MEDIA extraction and existing whitespace preservation remains intact.
