## Summary

Keep open: current main still has the same-line reply-tag plus MEDIA parser gap, but this PR is not merge-ready because it lacks required real Telegram proof, is stale against the moved Telegram extension path, and introduces a whitespace-preservation regression.

## What This Changes

The branch changes reply directive parsing, outbound reply normalization, old Telegram delivery wiring, message/native command dispatch options, and Telegram delivery tests so inline MEDIA directives and reply tags are normalized before Telegram replies are sent.

## Best Possible Solution

Port a narrow current-main fix that strips only reply tags before media extraction, preserves tag-free whitespace, updates focused extensions/telegram and outbound coverage, and includes redacted live Telegram proof.

## Reproduction Assessment

Yes. Source inspection shows current main parses MEDIA before stripping reply tags and only extracts MEDIA at the trimmed start of a line, so a same-line reply tag plus MEDIA directive remains text; no test was run because this review is read-only.

## Solution Assessment

No. The parser-order goal is valid, but this branch should be narrowed to strip only reply tags before media extraction, preserve tag-free whitespace, and be ported to the current extensions/telegram path.
