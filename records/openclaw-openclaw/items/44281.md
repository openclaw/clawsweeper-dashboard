## Summary

Keep open: this PR targets a valid Telegram reply-media bug, but it still needs real Telegram proof and its current parsing change introduces a whitespace regression before it can be merged.

## What This Changes

The PR changes reply directive parsing, outbound payload normalization, Telegram reply delivery/dispatch, and Telegram tests so inline MEDIA directives and reply tags are normalized before Telegram sends.

## Best Possible Solution

Adapt the fix to current main's extensions/telegram delivery and shared outbound payload plan, strip reply tags before MEDIA extraction without normalizing tag-free text, add focused regression coverage, and attach redacted real Telegram proof.

## Reproduction Assessment

Yes, from source inspection: current main parses MEDIA before stripping reply tags, while media extraction only recognizes lines beginning with MEDIA. A reply like [[reply_to:42]] MEDIA:./photo.jpg therefore remains ordinary text instead of delivering media.

## Solution Assessment

No. The current PR direction fixes the ordering issue too broadly by normalizing all raw reply text; the safer solution is to strip only reply tags, or otherwise preserve tag-free formatting, before media extraction.

## Review Findings
