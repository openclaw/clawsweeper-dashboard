## What This Changes

The PR changes reply directive parsing, outbound reply normalization options, old Telegram bot delivery wiring, and Telegram delivery tests for inline `MEDIA:` directives combined with reply tags.

## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main parses media before removing reply tags, while media extraction only recognizes line-leading `MEDIA:` tokens.

## Solution Assessment

No. The parser-order fix is the right direction, but this PR applies it through unconditional directive whitespace normalization and stale Telegram paths; a narrower parser fix plus current plugin tests is safer.
