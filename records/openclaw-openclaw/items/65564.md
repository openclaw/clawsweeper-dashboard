## Summary

Keep open. Current main fixes the original stale `sessionFile` carryover, but the remaining same-key isolated heartbeat rotation path still overwrites the stored entry without archiving the prior primary transcript; open PR #73809 is the active narrow fix candidate.

## What This Changes

The issue asks for heartbeat `isolatedSession` runs to use fresh transcript history when rotating session IDs, without reusing or orphaning the previous heartbeat transcript file.

## Best Possible Solution

Land #73809 or an equivalent narrow change that captures the prior `isolatedSessionKey` entry before overwrite, archives its transcript with `reason: "reset"` and `restrictToStoreDir: true`, and keeps trajectory/doctor classification and memory-ingestion hardening in separate follow-up work.

## Reproduction Assessment

Yes. The issue provides a concrete two-run `heartbeat.isolatedSession: true` reproduction, and current main source confirms the same-key rotation path can still overwrite the old entry without adding the previous transcript to the archival set.

## Solution Assessment

Yes. The narrow archival fix proposed in the discussion and #73809 is the best current direction because it preserves the already-fixed fresh transcript behavior and only adds missing lifecycle cleanup for the prior same-key heartbeat transcript.
