## What This Changes

The issue reports that `heartbeat.isolatedSession` rotates to a new session id while reusing or orphaning the prior transcript file, causing heartbeat context and token growth.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/73809 or an equivalent narrow reset-archive change, while keeping trajectory classification and memory-ingestion hardening in their separate follow-ups.

## Reproduction Assessment

Yes, at source level. Current main still force-rotates isolated heartbeat sessions under the same key and overwrites the store entry while only archiving stale nested-key removals.

## Solution Assessment

Yes. Capturing the previous same-key entry before overwrite and archiving it with the existing store-directory-restricted reset helper is the narrowest maintainable fix for the remaining behavior.
