## Summary

Keep open: current main and v2026.6.6 still lack the same-key heartbeat transcript reset archival path. The original stale `sessionFile` carryover and the generic generated-path rollover case are fixed, but the remaining heartbeat-specific orphaning behavior is still owned by https://github.com/openclaw/openclaw/pull/73809.

## What This Changes

Ensure heartbeat isolated-session runs use fresh transcript state when the session id rotates, instead of reusing or orphaning transcript files from prior runs.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/73809 or an equivalent narrow same-key reset archival change, while keeping generic rollover and memory-ingestion hardening as separate concerns.

## Reproduction Assessment

Yes. At source level, current main force-rotates an existing isolated heartbeat entry, overwrites the same key, and only archives stale nested keys rather than the previous same-key transcript.

## Solution Assessment

Yes. Capturing the previous same-key heartbeat entry before overwrite and archiving it through the existing store-directory-restricted reset archival path is the narrow maintainable fix; generic merge hardening alone is not enough.
