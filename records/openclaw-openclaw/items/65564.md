## Summary

Keep open because current main fixes the original stale `sessionFile` carryover, but the same-key isolated heartbeat rotation path still lacks archival for the prior transcript; the discussion and open PR #73809 identify the remaining narrow fix.

## What This Changes

Make `heartbeat.isolatedSession` runs avoid accumulating prior heartbeat transcript state when each heartbeat rotates to a fresh session.

## Best Possible Solution

Land #73809 or an equivalent narrow change that archives the previous same-key isolated heartbeat transcript as a reset while keeping trajectory/doctor and memory-ingestion hardening in separate follow-ups.

## Reproduction Assessment

Yes. The issue gives a concrete two-run `heartbeat.isolatedSession: true` reproduction, and current source confirms the same-key rotation path can overwrite the old store entry without adding that prior transcript to the archival set.

## Solution Assessment

Yes. The proposed reset-archive direction is the narrowest maintainable fix because it preserves the already-fixed fresh transcript behavior and only adds missing cleanup for the previous same-key heartbeat transcript.
