## Summary

Keep open: the original stale generated `sessionFile` carryover is fixed, but current `main` and `v2026.6.6` still lack the same-key heartbeat reset archival path, so isolated heartbeat rotation can still leave the previous transcript orphaned. The active narrow fix candidate is https://github.com/openclaw/openclaw/pull/73809.

## What This Changes

The issue asks for `heartbeat.isolatedSession` rotations to use fresh transcripts without reusing or orphaning prior heartbeat transcript files.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/73809 or an equivalent narrow same-key reset archival change, while keeping generic rollover and memory-ingestion hardening as separate follow-ups.

## Reproduction Assessment

Yes, at source level. With `isolatedSession: true` and an existing `<base>:heartbeat` entry, current main force-rotates the session id, overwrites that same key, and only archives stale nested keys.

## Solution Assessment

Yes, the best known fix is the narrow same-key archival path in https://github.com/openclaw/openclaw/pull/73809. The merged cron/session-store fixes remove stale path carryover but do not archive the previous heartbeat transcript on same-key rotation.
