## Summary

Keep open: current `main` fixed the original stale `sessionFile` carryover, but the normal same-key isolated heartbeat rotation path still overwrites the stored entry without archiving the prior primary transcript. Open PR #73809 remains the narrow active fix candidate.

## What This Changes

Ensure `heartbeat.isolatedSession` uses a fresh heartbeat transcript per run without reusing or orphaning the prior transcript.

## Best Possible Solution

Review and land #73809, or an equivalent narrow change that captures the prior entry at the canonical isolated heartbeat key before overwrite and archives its transcript as a reset while leaving trajectory/doctor and memory-ingestion hardening to separate follow-ups.

## Reproduction Assessment

Yes. The issue gives concrete two-run `heartbeat.isolatedSession: true` steps, and current source confirms the same-key rotation path can still overwrite the old store entry without archiving the prior transcript.

## Solution Assessment

Yes. The narrow reset-archive fix is the best current path because it preserves the already-fixed fresh transcript behavior and only adds missing lifecycle cleanup for the prior same-key heartbeat transcript.
