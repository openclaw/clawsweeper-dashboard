## Summary

Keep open: current main fixes the original stale `sessionFile` carryover, but the remaining same-key isolated heartbeat rotation still overwrites the store entry without archiving the previous transcript; an open narrow fix PR already targets that gap.

## What This Changes

The issue requests that `heartbeat.isolatedSession` runs use a fresh transcript and stop carrying or orphaning prior heartbeat transcript files when the heartbeat session id rotates.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/73809 or an equivalent narrow reset-archive change, while keeping trajectory classification and memory-ingestion hardening in separate follow-ups.

## Reproduction Assessment

Yes, at source level. With `isolatedSession: true` and an existing `<base>:heartbeat` store entry, current main force-rotates the session id, overwrites the store entry, and only archives stale nested keys, leaving the previous same-key transcript unarchived.

## Solution Assessment

Yes. Capturing the previous same-key entry before overwrite and archiving it through the existing store-directory-restricted `archiveRemovedSessionTranscripts(..., reason: "reset")` path is the narrow maintainable fix.
