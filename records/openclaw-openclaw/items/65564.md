## Summary

Keep open because current main fixes the original stale `sessionFile` carryover, but the same-key isolated heartbeat rotation path still overwrites the stored entry without archiving the prior primary transcript; open PR #73809 remains the active narrow fix candidate.

## What This Changes

The issue requests that `heartbeat.isolatedSession: true` stop reusing or orphaning prior heartbeat transcript history when each heartbeat run rotates to a fresh session.

## Best Possible Solution

Land #73809 or an equivalent narrow change that captures the prior entry at the canonical isolated heartbeat key before overwrite, archives its transcript as a reset with store-dir containment, and keeps trajectory/doctor and memory-ingestion hardening in separate follow-ups.

## Reproduction Assessment

Yes. The issue gives a concrete two-run `heartbeat.isolatedSession: true` reproduction, and current main source confirms the same-key rotation path can still overwrite the old store entry without adding the prior transcript to the archival set.

## Solution Assessment

Yes. The narrow reset-archive fix is the best current path because it preserves the already-fixed fresh transcript behavior and only adds missing lifecycle cleanup for the prior same-key heartbeat transcript.
