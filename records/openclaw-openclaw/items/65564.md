## Summary

Keep open: current main and the latest release fixed the original stale `sessionFile` carryover and the generic generated-path rollover case, but the heartbeat same-key isolated rotation path still overwrites the store entry without archiving the previous transcript. The active narrow fix path is https://github.com/openclaw/openclaw/pull/73809.

## What This Changes

The issue requests that `heartbeat.isolatedSession` use a genuinely fresh transcript when `sessionId` rotates, without reusing or orphaning the prior transcript file.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/73809 or an equivalent narrow same-key reset archival change, while keeping generic rollover and memory-ingestion hardening as separate concerns.

## Reproduction Assessment

Yes, at source level. With `isolatedSession: true` and an existing `<base>:heartbeat` entry, current main force-rotates the session id, overwrites that same key, and only archives stale nested keys.

## Solution Assessment

Yes. The narrow maintainable fix is to capture the previous same-key heartbeat entry before overwrite and archive it through the existing store-directory-restricted reset archival path; generic merge hardening alone is not enough.
