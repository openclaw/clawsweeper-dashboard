## Summary

Keep open: current main fixed the original stale `sessionFile` carryover, but the same-key isolated heartbeat rotation path still overwrites the stored heartbeat entry without archiving the prior primary transcript; https://github.com/openclaw/openclaw/pull/73809 is the active narrow fix candidate.

## What This Changes

The issue reports that `heartbeat.isolatedSession` rotates `sessionId` while transcript state is reused or orphaned, causing heartbeat context and session artifacts to accumulate.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/73809 or an equivalent narrow change that archives the prior same-key isolated heartbeat transcript as a reset, while leaving doctor trajectory handling and memory-ingestion hardening to separate follow-ups.

## Reproduction Assessment

Yes, at source level. The issue gives concrete two-run steps, and current main still shows the forced-new isolated heartbeat path overwriting the same store key while only archiving stale nested-key removals.

## Solution Assessment

Yes. Capturing the previous same-key entry before overwrite and using the existing store-directory-restricted reset archive helper is the narrowest maintainable fix for the remaining behavior.
