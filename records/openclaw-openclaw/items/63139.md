## Summary

Keep open. Current main still invokes `before_model_resolve` from inside each embedded fallback attempt, so the `runId`/model context mitigation does not stop an unconditional router from reapplying the same override on every fallback candidate.

## What This Changes

The issue requests that `before_model_resolve` either run once per logical agent run or expose retry metadata so routing plugins do not defeat model fallback.

## Best Possible Solution

OpenClaw should make the hook contract explicit, then cover it with a regression test for a router override plus cross-provider model fallback.

## Reproduction Assessment

Yes. The source-level path is high confidence: configure multiple model fallbacks and a hook that always returns the same provider/model, then the current `runWithModelFallback` -> `runEmbeddedPiAgent` flow re-runs the hook for each candidate before model resolution.

## Solution Assessment

Unclear. Hoisting the hook best matches the reporter's logical-run expectation, while adding `attempt`/`isFallbackRetry` plus docs is the narrower compatible contract for plugins that intentionally observe each candidate.
