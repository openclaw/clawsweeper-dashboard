## Summary

Keep open: the requested ACP session tool work is not on current main and may be useful, but this PR is not merge-ready because the cancel path bypasses session visibility policy, the fire-and-forget send path can silently target non-ACP sessions, generated protocol artifacts are stale, and contributor real behavior proof is missing.

## What This Changes

The branch adds `acp_send` and `sessions_cancel` agent tools, a `sessions.cancel` gateway RPC/schema/scope entry, and tool display metadata for ACP session lifecycle control.

## Best Possible Solution

Rework the PR so cancellation uses the same session resolution and visibility/A2A policy boundary as other session tools, `acp_send` fails fast for non-ACP targets, generated protocol artifacts and method metadata are updated, and the contributor provides redacted real ACP runtime proof.

## Reproduction Assessment

Source-reproducible for the blocking review findings: the new cancel tool and server handler can be inspected directly and do not apply the existing session visibility guard. The broader feature itself is not a bug report with a current-main failing reproduction path.

## Solution Assessment

No. The direction may be useful, but the implementation should reuse the established session visibility boundary and ACP target validation before exposing a new tool/RPC surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate cancellations with session visibility:** `src/agents/tools/sessions-cancel-tool.ts:31-36`
  - body: `sessions_cancel` forwards the model-provided `sessionKey` directly to `sessions.cancel`, and the server handler only checks that the key is non-empty before calling `acpManager.cancelSession`. Unlike `sessions_send` and `sessions_history`, this bypasses `tools.sessions.visibility` and `tools.agentToAgent.allow`, so a model that learns another ACP session key can cancel work it should not control.
  - confidence: 0.93
- **[P2] Reject non-ACP targets before dispatch:** `src/agents/tools/acp-send-tool.ts:218-240`
  - body: `acp_send` accepts any visible session key and starts a generic `agent` run with `deliver:false`, but only ACP sessions have the callback path this tool relies on. Passing a normal or native subagent session will return `accepted` while the reply has no ACP callback route back to the caller, so the output can disappear instead of failing fast.
  - confidence: 0.84
- **[P2] Commit the generated protocol models:** `src/gateway/protocol/schema/sessions.ts:119-124`
  - body: Adding `SessionsCancelParamsSchema` changes the Gateway wire contract, but the PR does not include regenerated protocol outputs. The `pnpm protocol:check` CI job regenerated Swift `SessionsCancelParams` models and failed on drift, so the generated protocol artifacts need to be refreshed and committed with the schema change.
  - confidence: 0.9
