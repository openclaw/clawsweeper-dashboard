## Summary

Keep this PR open: current main still lacks the proposed ACP send/cancel surface, so the contribution is not obsolete, but the branch is not merge-ready because it introduces a model-facing cancellation path without the existing session visibility guard, can silently drop non-ACP sends, has current-main protocol/dependency incompatibilities, is merge-conflicting, and still lacks inspectable real ACP runtime proof.

## What This Changes

The PR adds model-facing `acp_send` and `sessions_cancel` tools plus a `sessions.cancel` gateway RPC/schema/scope entry and tool display metadata for ACP session lifecycle control.

## Best Possible Solution

Rebase and rework the feature through existing session resolution and visibility policy, ACP-only target validation, supported gateway error codes/current TypeBox imports, regenerated protocol artifacts, focused tests, and contributor-supplied redacted real ACP runtime proof.

Yes for the review blockers: PR-head source inspection shows the direct cancel-key forwarding and generic non-ACP send path, and live PR status shows protocol/check failures. I did not run local tests because this is a read-only review.

## Solution Assessment

No. The requested capability may be useful, but the current patch is not the narrow maintainable merge path until it aligns with existing session visibility policy, ACP target validation, protocol generation, and current dependency contracts.
