## Summary

Keep open: the feature is not implemented on current main, and the PR still has blocking correctness and security issues around ACP session authorization plus missing real behavior proof.

## What This Changes

The PR adds `acp_send`, `sessions_cancel`, display metadata, and a Gateway `sessions.cancel` protocol/schema/scope/handler for ACP session lifecycle control.

## Best Possible Solution

Rework the feature on current main so ACP-only tools reuse current session resolution, visibility, and A2A policy checks, validate ACP-backed targets before callback-dependent sends, and include focused tests plus real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: follow `sessions_cancel` from raw `sessionKey` input to `sessions.cancel` and then to `AcpSessionManager.cancelSession()`, and follow `acp_send` from any visible non-ACP target to an internal `agent` dispatch with `deliver: false`.

## Solution Assessment

No. The feature direction may be useful, but this implementation is not the narrowest safe solution because it puts ACP-specific control behind raw-key dispatch and does not verify ACP metadata before callback-dependent sends.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Enforce visibility before cancelling ACP sessions:** `src/agents/tools/sessions-cancel-tool.ts:32-34`
  - body: `sessions_cancel` forwards the caller-provided `sessionKey` directly to `sessions.cancel`, and the new handler calls `acpManager.cancelSession()` without requester visibility or A2A policy context. Resolve and guard the target like `sessions_send` before invoking cancel, otherwise a model that learns another ACP session key can cancel a hidden run.
  - confidence: 0.92
- **[P1] Use the current TypeBox import in the new tools:** `src/agents/tools/acp-send-tool.ts:2`
  - body: Current main imports schemas from `typebox` and no longer declares `@sinclair/typebox`. The new tool files import `@sinclair/typebox`, so the branch will fail module resolution after rebasing; update both new tool imports to `typebox`.
  - confidence: 0.9
- **[P2] Reject non-ACP targets before dispatching:** `src/agents/tools/acp-send-tool.ts:216-239`
  - body: `acp_send` resolves any visible session and dispatches an internal `agent` run with `deliver: false`, but only ACP sessions have the callback path this tool relies on. Validate the resolved target is ACP-backed before sending so normal/subagent sessions fail fast instead of returning `accepted` while losing the reply.
  - confidence: 0.86
