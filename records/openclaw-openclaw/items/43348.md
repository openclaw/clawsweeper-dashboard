## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Use the declared typebox import:** `src/agents/tools/acp-send-tool.ts:2`
  - body: Both new tool files import `@sinclair/typebox`, but this repo declares and consistently imports `typebox`. With pnpm's direct-dependency model, these imports can fail resolution when compiled; use `from "typebox"` or add an intentional direct dependency.
  - confidence: 0.93
- **[P1] Gate cancellation through session visibility:** `src/agents/tools/sessions-cancel-tool.ts:26-31`
  - body: `sessions_cancel` forwards the caller-provided `sessionKey` directly to `sessions.cancel`. The server handler only validates a non-empty key before cancelling, so a model that learns another session key can cancel a run hidden by session visibility or A2A policy. Resolve the key and run the same visibility guard used by the other session tools before calling the gateway.
  - confidence: 0.92
- **[P1] Classify new session-control tools before exposing them:** `src/agents/openclaw-tools.ts:187-192`
  - body: Registering `acp_send` and `sessions_cancel` makes them available to normal tool resolution, but the PR does not extend the HTTP `/tools/invoke` default deny list or ACP control-plane classifier. Add both names to those safety lists and tests before exposing them.
  - confidence: 0.9
- **[P2] Reject non-ACP targets before fire-and-forget dispatch:** `src/agents/tools/acp-send-tool.ts:223-231`
  - body: `acp_send` resolves any visible session and dispatches an `agent` run with `deliver: false`. If the target is a normal/native subagent session, there is no ACP callback path to deliver the reply, so the tool can report `accepted` while dropping the result; validate ACP session metadata before sending.
  - confidence: 0.86
- **[P2] Register the tools in the core catalog:** `src/agents/openclaw-tools.ts:187-192`
  - body: The new tools are constructed directly but are absent from `CORE_TOOL_DEFINITIONS`. Profile and group expansion is generated from that catalog, so normal coding/messaging profiles and explicit allowlists can omit or warn on these tools; add catalog/profile/docs coverage with registration.
  - confidence: 0.82
- **[P2] Commit generated protocol outputs:** `src/gateway/protocol/schema/protocol-schemas.ts:210`
  - body: Adding `SessionsCancelParams` to `ProtocolSchemas` changes generated protocol artifacts, but the PR does not include the regenerated JSON/Swift outputs. `pnpm protocol:check` already fails with drift for the Swift generated models, so clients would stay stale.
  - confidence: 0.96
