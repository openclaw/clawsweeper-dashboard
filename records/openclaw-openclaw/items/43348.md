## Summary

Keep open: current main does not implement the requested ACP lifecycle tools, and no canonical replacement owns the work, but this PR is not merge-ready because it lacks real behavior proof and has source-visible authorization, protocol, and dependency blockers.

## What This Changes

The PR adds model-facing `acp_send` and `sessions_cancel` tools, display metadata, and a `sessions.cancel` gateway RPC/schema/scope entry for ACP session lifecycle control.

## Best Possible Solution

Rework the ACP lifecycle tool surface so cancellation preserves existing session visibility/A2A boundaries, `acp_send` rejects non-ACP targets, protocol artifacts are regenerated, and a real ACP run proves callback delivery and cancellation.

## Reproduction Assessment

Yes for the review blockers: PR-head source inspection shows direct cancel-key forwarding, an undefined gateway error code, a non-ACP fire-and-forget path, stale TypeBox imports, and missing generated protocol artifacts. I did not run tests because this review is read-only.

## Solution Assessment

No. The capability may be useful, but this patch is not the narrow maintainable solution until it routes through the existing authorization boundary, validates ACP-only targets, syncs protocol clients, and includes real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use a defined gateway error code:** `src/gateway/server-methods/sessions.ts:465`
  - body: `ErrorCodes.INTERNAL_ERROR` is not part of the current gateway protocol error-code set, so this handler will not typecheck against current main. Use an existing code such as `UNAVAILABLE`, or add a new protocol code consistently before referencing it.
  - confidence: 0.96
- **[P1] Enforce visibility before cancelling sessions:** `src/agents/tools/sessions-cancel-tool.ts:31-34`
  - body: `sessions_cancel` forwards a caller-provided key directly to `sessions.cancel`, unlike existing session tools that resolve targets and enforce `tools.sessions.visibility` and A2A policy first. A model that learns another ACP session key could interrupt a session it should not control.
  - confidence: 0.93
- **[P2] Reject non-ACP targets before dispatch:** `src/agents/tools/acp-send-tool.ts:218-224`
  - body: `acp_send` accepts any visible session and starts an internal run with `deliver: false`, but only ACP sessions have the callback path this tool depends on. Validate the resolved target as ACP-only so normal or native subagent sessions fail fast instead of losing replies.
  - confidence: 0.87
- **[P2] Regenerate protocol artifacts:** `src/gateway/protocol/schema/sessions.ts:119-124`
  - body: Adding `SessionsCancelParamsSchema` changes the gateway protocol contract, but the generated Swift protocol outputs are not included and `protocol:check` is designed to catch that drift. Regenerate and commit the generated artifacts before merge.
  - confidence: 0.96
- **[P2] Import TypeBox from the current package:** `src/agents/tools/acp-send-tool.ts:2`
  - body: The new tools import `@sinclair/typebox`, while current main imports from the direct `typebox` package and the repo's tools follow that package. Rebase the imports to the current package so strict dependency resolution stays stable.
  - confidence: 0.84
