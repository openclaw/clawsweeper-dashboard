## Summary

Keep open, but the patch is not ready: current main still lacks the proposed ACP tool/RPC surface, while this PR has security, build/protocol, and message-loss blockers plus no real behavior proof.

## What This Changes

The branch adds model-facing ACP follow-up and cancellation tools, display metadata, and a new `sessions.cancel` gateway RPC/schema/scope entry.

## Best Possible Solution

Keep the feature path open, but rework it around the existing session visibility/A2A boundary, ACP-only target validation, per-session nested lanes, synchronized protocol artifacts, and real ACP runtime proof.

## Reproduction Assessment

Yes for the review blockers: PR-head source inspection and current-main contracts show the raw cancel-key forwarding, undefined error code, non-ACP fire-and-forget path, stale TypeBox import, global nested lane use, and missing generated protocol artifacts. I did not run tests because this review is read-only.

## Solution Assessment

No. The capability may be useful, but this patch is not the best implementation until it preserves existing authorization, rejects non-ACP targets, uses current lane/protocol patterns, and includes real behavior proof.

Full review comments:

- **[P1] Enforce visibility before cancelling sessions:** `src/agents/tools/sessions-cancel-tool.ts:31-34`
  - body: `sessions_cancel` forwards the caller-provided key directly to `sessions.cancel`, unlike the existing session tools that first resolve the target and enforce `tools.sessions.visibility` plus A2A policy. A model that learns another ACP session key could interrupt a session it should not control.
  - confidence: 0.94
- **[P1] Use a defined gateway error code:** `src/gateway/server-methods/sessions.ts:465`
  - body: `ErrorCodes.INTERNAL_ERROR` is not in the current gateway protocol error-code set, so this handler will not typecheck against current main. Use an existing code such as `UNAVAILABLE`, or add a new protocol code consistently before referencing it.
  - confidence: 0.96
- **[P2] Scope acp_send nested runs by session:** `src/agents/tools/acp-send-tool.ts:224`
  - body: Current main fixed ACP/session message loss from the shared `nested` lane by using `resolveNestedAgentLaneForSession(...)`. This new tool hard-codes `AGENT_LANE_NESTED`, so long ACP sends can again block unrelated sessions behind one global lane.
  - confidence: 0.9
- **[P2] Reject non-ACP targets before dispatch:** `src/agents/tools/acp-send-tool.ts:218-224`
  - body: `acp_send` accepts any visible session and starts an internal run with `deliver: false`, but only ACP sessions have the callback path this tool depends on. Validate the resolved target as ACP-only so normal or native subagent sessions fail fast instead of losing replies.
  - confidence: 0.88
- **[P2] Regenerate protocol artifacts:** `src/gateway/protocol/schema/sessions.ts:116-124`
  - body: Adding `SessionsCancelParamsSchema` changes the gateway protocol contract, but the generated Swift protocol outputs are not included and `protocol:check` is already failing on the PR. Regenerate and commit the generated artifacts before merge.
  - confidence: 0.96
- **[P2] Import TypeBox from the current package:** `src/agents/tools/acp-send-tool.ts:2`
  - body: The new tools import `@sinclair/typebox`, while current main imports from the direct `typebox` package and only declares that package. Rebase the imports so strict dependency resolution remains stable.
  - confidence: 0.88
