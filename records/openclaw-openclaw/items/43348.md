## Summary

Keep open. Current main does not already provide these new ACP tools, and this PR still has blocking build, security, policy, and behavior issues plus missing real behavior proof before it can be merged.

## What This Changes

Adds core agent tools `acp_send` and `sessions_cancel`, a `sessions.cancel` gateway RPC/schema/scope, and display overrides for ACP session lifecycle control.

## Best Possible Solution

Land an owner-reviewed ACP session-control feature that reuses the shared session access helpers, updates the tool catalog/protocol/security/docs/tests together, rejects invalid ACP targets, and includes redacted real ACP send/cancel proof.

## Reproduction Assessment

Yes for the review findings by source inspection: the PR imports an undeclared package, bypasses the existing session visibility pattern for cancellation, and exposes new high-risk tools without extending the existing safety lists. I did not run tests because this is a read-only review.

## Solution Assessment

No. The direction may be useful, but this implementation needs to align with current session visibility, tool catalog, HTTP deny, ACP approval, protocol, and proof requirements before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the declared typebox import:** `src/agents/tools/acp-send-tool.ts:2`
  - body: Both new tool files import `@sinclair/typebox`, but this repo declares and consistently imports `typebox`. With pnpm's direct-dependency model, these new imports are undeclared and can fail build/runtime resolution as soon as the files are compiled; switch them to `from "typebox"` or add an intentional dependency.
  - confidence: 0.93
- **[P1] Gate session cancellation through session visibility:** `src/agents/tools/sessions-cancel-tool.ts:26-31`
  - body: `sessions_cancel` forwards the caller-provided `sessionKey` directly to `sessions.cancel`. The server handler only requires a non-empty key before calling `acpManager.cancelSession`, so a model that learns another ACP session key can cancel a run that `tools.sessions.visibility` or A2A policy should hide. Resolve the key and run the same visibility guard used by the other session tools before calling the gateway.
  - confidence: 0.92
- **[P1] Classify new session-control tools before exposing them:** `src/agents/openclaw-tools.ts:187-192`
  - body: Registering `acp_send` and `sessions_cancel` makes them available to the normal tool resolution path, but the PR does not extend the existing HTTP `/tools/invoke` default deny list or ACP control-plane classifier. That reopens the same class of high-risk session orchestration surface already guarded for `sessions_spawn` and `sessions_send`; add both names to the safety lists and tests before exposing them.
  - confidence: 0.89
- **[P2] Reject non-ACP targets before dispatching acp_send:** `src/agents/tools/acp-send-tool.ts:223-231`
  - body: After resolving visibility, `acp_send` dispatches an `agent` run with `deliver: false` for any session key. If the target is a normal/native subagent session, there is no ACP callback path to deliver the reply, so the tool can report `accepted` while dropping the result. Check the resolved session entry for ACP metadata and fail fast for non-ACP targets.
  - confidence: 0.86
- **[P2] Register the tools in the core catalog:** `src/agents/openclaw-tools.ts:187-192`
  - body: The tools are constructed directly, but they are absent from `CORE_TOOL_DEFINITIONS`. Local onboarding defaults to the `coding` profile and `group:sessions` is generated from that catalog, so these new tools will be filtered out of normal coding/messaging profiles and explicit allowlists can warn as unknown. Add catalog/profile/docs coverage with the registration.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces new session-control surfaces without carrying forward the existing session visibility and HTTP/ACP safety contracts.

Concerns:

- **[high] Cancellation bypasses session visibility:** `src/agents/tools/sessions-cancel-tool.ts:26`
  - body: The new `sessions_cancel` tool sends arbitrary keys to `sessions.cancel`, and the new gateway handler only validates that a key exists before calling the ACP manager. That lets a tool caller cancel a session it should not be able to see if it learns the key.
  - confidence: 0.92
- **[high] New high-risk tools miss existing safety lists:** `src/agents/openclaw-tools.ts:187`
