## What This Changes

The PR adds `acp_send` and `sessions_cancel` agent tools, a `sessions.cancel` gateway RPC/schema/scope entry, and tool display metadata.

## Best Possible Solution

Keep the feature direction, but rebase and update the patch to reuse session visibility/ACP target contracts, regenerate protocol outputs, use current `typebox` imports, add focused coverage, and include redacted real ACP runtime proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Enforce session visibility before cancelling:** `src/agents/tools/sessions-cancel-tool.ts:32-34`
  - body: `sessions_cancel` forwards the caller-provided key directly to `sessions.cancel`, so a model that learns another ACP session key can cancel it without the `tools.sessions.visibility` and `tools.agentToAgent.allow` checks used by the existing direct session tools. Resolve the target through the session helpers and apply a cancel/send-style guard before invoking the gateway method.
  - confidence: 0.93
- **[P2] Regenerate protocol artifacts for sessions.cancel:** `src/gateway/protocol/schema/sessions.ts:119-123`
  - body: Adding `SessionsCancelParamsSchema` changes the gateway protocol contract, but the generated Swift protocol model outputs are not committed. The existing `pnpm protocol:check` CI job fails with diffs adding `SessionsCancelParams`, so native clients stay out of sync until those artifacts are refreshed.
  - confidence: 0.95
- **[P2] Reject non-ACP targets before dispatch:** `src/agents/tools/acp-send-tool.ts:218-224`
  - body: `acp_send` resolves any visible session and dispatches an internal run with `deliver: false`, but only ACP sessions have the callback path this tool relies on. Validate the resolved key as ACP before dispatch so normal or native subagent sessions fail fast instead of accepting a message whose reply is never surfaced.
  - confidence: 0.86
- **[P2] Use the current TypeBox import:** `src/agents/tools/acp-send-tool.ts:2`
  - body: Current main imports schemas from the direct `typebox` dependency, while these new tool files import `@sinclair/typebox`, which is not declared in the current root dependency surface. Rebase the files to use `typebox` like the neighboring tools before this can compile reliably on current main.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The proposed cancel tool introduces a session-control action that bypasses existing model-facing session visibility policy.

Concerns:

- **[medium] Cross-session ACP cancellation bypass:** `src/agents/tools/sessions-cancel-tool.ts:32`
