## Summary

This PR should stay open: current main does not already provide the proposed ACP send/cancel surface, and the branch still has blocking correctness, protocol, security, and real-behavior-proof gaps.

## What This Changes

The PR adds model-facing `acp_send` and `sessions_cancel` tools, a `sessions.cancel` gateway RPC/schema/scope entry, and tool display metadata for ACP session lifecycle control.

## Best Possible Solution

Rebase and rework the feature through existing session resolution/visibility policy, ACP-only target validation, supported gateway error codes, regenerated protocol artifacts, focused tests, and contributor-supplied redacted real ACP runtime proof.

## Reproduction Assessment

Yes for the review blockers: PR-head source inspection shows the direct cancel-key forwarding and generic non-ACP send path, and live PR status shows protocol/check failures. I did not run local tests because this is a read-only review.

## Solution Assessment

No. The requested capability may be useful, but the current patch is not the narrow maintainable merge path until it aligns with existing session visibility policy, ACP target validation, protocol generation, and current dependency contracts.

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use a defined gateway error code:** `src/gateway/server-methods/sessions.ts:465`
  - body: `ErrorCodes.INTERNAL_ERROR` is not part of the current gateway protocol error-code set, so this handler does not typecheck against current main. Use an existing code such as `UNAVAILABLE`, or add a new protocol code consistently through the contract before referencing it.
  - confidence: 0.96
- **[P1] Enforce session visibility before cancelling:** `src/agents/tools/sessions-cancel-tool.ts:32-34`
  - body: `sessions_cancel` forwards the caller-provided key directly to `sessions.cancel`, while existing session tools resolve targets and apply `tools.sessions.visibility`/A2A policy first. A model that learns another ACP session key could interrupt a session it should not control.
  - confidence: 0.93
- **[P2] Reject non-ACP targets before dispatch:** `src/agents/tools/acp-send-tool.ts:218-224`
  - body: `acp_send` accepts any visible session and starts an internal run with `deliver: false`, but only ACP sessions have the callback path this tool depends on. Validate the resolved target as ACP-only so normal or native subagent sessions fail fast instead of losing replies.
  - confidence: 0.87
- **[P2] Regenerate protocol artifacts:** `src/gateway/protocol/schema/sessions.ts:119-123`
  - body: Adding `SessionsCancelParamsSchema` changes the gateway protocol contract, but the generated Swift protocol outputs are not included and the live `pnpm protocol:check` job fails. Regenerate and commit the generated artifacts before merge.
  - confidence: 0.96
- **[P2] Use the current TypeBox package:** `src/agents/tools/acp-send-tool.ts:2`
  - body: The new tools import `@sinclair/typebox`, while current main uses the direct `typebox` package and only declares `typebox` in package metadata. Rebase the new tool imports to the current package so strict dependency resolution stays stable.
  - confidence: 0.84
