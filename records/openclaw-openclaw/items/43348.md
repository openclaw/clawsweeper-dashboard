## Summary

Keep open. The feature is not implemented on current main, and this PR still has blocking source-backed issues around ACP cancellation authorization, current dependency imports, and non-ACP target handling.

## What This Changes

The PR adds `acp_send` and `sessions_cancel` agent tools, display metadata, and a `sessions.cancel` Gateway protocol/schema/scope/handler backed by `AcpSessionManager.cancelSession()`.

## Best Possible Solution

Rework the PR on current main so ACP-only tools reuse existing session resolution, visibility, and A2A policy checks; validate ACP metadata before callback-dependent sends; use current `typebox` imports; and add focused regression tests plus required docs/changelog updates.

## Reproduction Assessment

Yes. The cancellation issue is source-reproducible by following `sessions_cancel` from raw `sessionKey` input to `sessions.cancel` and then to `AcpSessionManager.cancelSession()` without the guard used by `sessions_send`; the `acp_send` issue is source-reproducible by targeting a visible non-ACP session and seeing the PR dispatch with `deliver: false` anyway.

## Solution Assessment

No. The feature direction may be useful, but the proposed implementation is not the safest maintainable path because it exposes ACP session control through raw-key dispatch instead of reusing existing session access helpers and ACP metadata checks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Enforce visibility before cancelling ACP sessions:** `src/agents/tools/sessions-cancel-tool.ts:32-34`
  - body: `sessions_cancel` forwards the caller-provided `sessionKey` directly to `sessions.cancel`, whose handler only requires a non-empty key before calling `acpManager.cancelSession()`. A caller that learns another ACP session key can cancel work that `tools.sessions.visibility` or A2A policy should hide, so resolve and guard the target like `sessions_send` before invoking cancel.
  - confidence: 0.92
- **[P1] Use the current TypeBox import:** `src/agents/tools/acp-send-tool.ts:2`
  - body: Current main imports schemas from `typebox` and `package.json` no longer declares `@sinclair/typebox`. These new tool files import `@sinclair/typebox`, so the branch will fail module resolution after rebasing to current main.
  - confidence: 0.9
- **[P2] Reject non-ACP targets before dispatching:** `src/agents/tools/acp-send-tool.ts:218-238`
  - body: `acp_send` resolves any visible session and dispatches an internal `agent` run with `deliver: false`, but only ACP sessions have the callback path this tool relies on. Targeting a native/subagent session can return `accepted` while losing the reply, so validate that the resolved target is ACP-backed before sending.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/agents/openclaw-tools.ts:187-192`
  - body: This adds user-facing agent tools and a Gateway protocol method, but the PR does not update `CHANGELOG.md`. OpenClaw policy requires user-facing feature changes to have a changelog entry before merge.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff introduces a new ACP session-control path that can cancel by raw session key without requester visibility authorization.

Concerns:

- **[high] Unscoped ACP session cancellation:** `src/agents/tools/sessions-cancel-tool.ts:32`

This can let an agent cancel another ACP session if the key appears in context or prior tool output.
