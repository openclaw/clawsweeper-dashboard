## Summary

Keep this PR open for human review: current main still does not implement the requested ACP tools, but the branch remains unsafe because it exposes raw-key ACP cancellation, accepts non-ACP send targets, uses stale TypeBox imports, and lacks real behavior proof.

## What This Changes

The PR adds `acp_send`, `sessions_cancel`, display metadata, and a Gateway `sessions.cancel` protocol/schema/scope/handler for ACP session lifecycle control.

## Best Possible Solution

Rework the feature on current main so ACP-only tools reuse current session resolution, visibility, and A2A policy checks, validate ACP-backed targets before callback-dependent sends, and include focused tests plus redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `sessions_cancel` forwards raw `sessionKey` input to `sessions.cancel` and then `AcpSessionManager.cancelSession()`, while `acp_send` can dispatch any visible non-ACP target with `deliver: false`.

## Solution Assessment

No. The feature direction may be useful, but this branch is not the narrowest safe implementation because ACP session control and callback-dependent sends need target resolution, visibility checks, ACP validation, and proof on current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Enforce visibility before cancelling ACP sessions:** `src/agents/tools/sessions-cancel-tool.ts:32-34`
  - body: `sessions_cancel` forwards the caller-provided `sessionKey` directly to `sessions.cancel`, and the new handler calls `acpManager.cancelSession()` without requester visibility or A2A policy context.   - confidence: 0.9

  - body: `acp_send` resolves any visible session and dispatches an internal `agent` run with `deliver: false`, but only ACP sessions have the callback path this tool relies on. Validate the resolved target is ACP-backed before sending so normal/subagent sessions fail fast instead of returning `accepted` while losing the reply.
  - confidence: 0.86
