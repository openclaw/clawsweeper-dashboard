Full review comments:

- **[P1] Use a defined gateway error code:** `src/gateway/server-methods/sessions.ts:465`
  - body: `ErrorCodes.INTERNAL_ERROR` is not part of the current gateway protocol error-code set, so this handler will not typecheck against current main. Use an existing code such as `UNAVAILABLE`, or add a new protocol code consistently through the contract before referencing it.
  - confidence: 0.96
- **[P1] Enforce visibility before cancelling sessions:** `src/agents/tools/sessions-cancel-tool.ts:30-34`
  - body: `sessions_cancel` forwards the caller-provided key directly to `sessions.cancel`, while existing session tools resolve targets and apply `tools.sessions.visibility` and A2A policy first. A model that learns another ACP session key could interrupt a session it should not control.
  - confidence: 0.93
- **[P2] Reject non-ACP targets before dispatch:** `src/agents/tools/acp-send-tool.ts:218-224`
  - body: `acp_send` accepts any visible session and starts an internal run with `deliver: false`, but only ACP sessions have the callback path this tool depends on. Validate the resolved target as ACP-only so normal or native subagent sessions fail fast instead of losing replies.
  - confidence: 0.87
- **[P2] Regenerate protocol artifacts:** `src/gateway/protocol/schema/sessions.ts:116-123`
  - body: Adding `SessionsCancelParamsSchema` changes the gateway protocol contract, but the generated Swift protocol outputs are not included and the live `pnpm protocol:check` job fails. Regenerate and commit the generated artifacts before merge.
  - confidence: 0.96
- **[P2] Import TypeBox from the current package:** `src/agents/tools/acp-send-tool.ts:2`
  - body: The new tools import `@sinclair/typebox`, while current main uses the direct `typebox` package and only declares `typebox` in package metadata. Rebase the new tool imports to the current package so strict dependency resolution stays stable.
  - confidence: 0.84
