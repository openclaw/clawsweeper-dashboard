Full review comments:

- **[P1] Enforce visibility before cancelling ACP sessions:** `src/agents/tools/sessions-cancel-tool.ts:32-34`
  - body: `sessions_cancel` accepts any session key and sends it straight to `sessions.cancel`; the gateway handler also only checks that the key is non-empty before calling `acpManager.cancelSession`. A model that has seen another ACP key could interrupt a session that `tools.sessions.visibility` or `tools.agentToAgent.allow` should hide.
  - confidence: 0.91
- **[P2] Reject non-ACP targets before fire-and-forget send:** `src/agents/tools/acp-send-tool.ts:218-224`
  - body: `acp_send` resolves any visible session and starts an internal agent run with `deliver: false`, but it never verifies the target is actually ACP-backed. Normal sessions will accept the message while no ACP callback exists to surface the reply, so output can disappear.
  - confidence: 0.86
- **[P2] Use the current TypeBox import path:** `src/agents/tools/acp-send-tool.ts:2`
  - body: Current main has migrated schema imports to `typebox` and no longer has `@sinclair/typebox` in the source tree. Adding new files with the old import path will fail against current main unless the branch is rebased and updated.
  - confidence: 0.82
