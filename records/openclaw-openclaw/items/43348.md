## Summary

Keep this PR open, but it should not merge as-is: the proposed tools add useful ACP lifecycle surface, yet the diff has blocking policy, protocol-generation, and current-main compatibility issues, and the external contributor has not provided real behavior proof.

## What This Changes

The PR adds `acp_send` and `sessions_cancel` agent tools plus a new `sessions.cancel` gateway RPC, schema exports, method scope entry, and tool display metadata.

## Best Possible Solution

Keep the feature direction, but update the patch to reuse the existing session visibility helpers for cancel, reject non-ACP `acp_send` targets before dispatch, regenerate protocol artifacts, use current `typebox` imports, add focused coverage, and include redacted real ACP runtime proof.

## Reproduction Assessment

Yes for the review blockers: source inspection shows the cancel path bypasses established session-tool visibility checks, `acp_send` lacks an ACP target guard, and CI shows `pnpm protocol:check` fails on generated protocol artifacts. Real end-to-end ACP behavior is not reproduced because the PR lacks runnable proof in this read-only review.

## Solution Assessment

No. The proposed feature is plausible, but the current diff is not the narrowest maintainable solution until it aligns with the existing visibility contract, ACP key contract, generated protocol outputs, and current dependency surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Enforce session visibility before cancelling:** `src/agents/tools/sessions-cancel-tool.ts:32-34`
  - body: `sessions_cancel` passes the raw tool argument directly to `sessions.cancel`, so a model that learns another ACP session key can cancel that run without the `tools.sessions.visibility` and `tools.agentToAgent.allow` checks used by `sessions_send` and `sessions_history`. Resolve the target through the existing session helpers and apply a cancel/send-style visibility guard before invoking the gateway method.
  - confidence: 0.93
- **[P2] Regenerate protocol artifacts for sessions.cancel:** `src/gateway/protocol/schema/sessions.ts:119-123`
  - body: Adding `SessionsCancelParamsSchema` changes the gateway protocol contract, but the generated protocol files were not committed. The existing CI log for `pnpm protocol:check` fails with diffs for the Swift gateway models, so clients will be out of sync until the generated artifacts are updated.
  - confidence: 0.95
- **[P2] Reject non-ACP acp_send targets before dispatch:** `src/agents/tools/acp-send-tool.ts:218-224`
  - body: `acp_send` resolves any visible session and starts an internal agent run with `deliver: false`, but only ACP sessions have the callback path this tool relies on. If the target is a normal or native subagent session, the run can be accepted while the reply is never surfaced, so validate the resolved target as ACP before dispatching.
  - confidence: 0.86
- **[P2] Use the current TypeBox import:** `src/agents/tools/acp-send-tool.ts:2`
  - body: Current main has migrated agent and protocol schemas to import from `typebox`; these new files import `@sinclair/typebox`, which is not part of current main's direct dependency surface. After rebasing, these imports should use `typebox` like the neighboring tool files or the new tools can fail to compile.
  - confidence: 0.82
