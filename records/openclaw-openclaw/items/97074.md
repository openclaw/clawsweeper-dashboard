## Summary

Keep open: current main still sends Feishu plain replies as `post` messages with one raw `tag: "md"` element, and the reviewed tests do not cover single-newline materialization for that renderer.

## What This Changes

The issue requests Feishu post+md outbound markdown normalization so single newlines render as visible line breaks instead of spaces.

## Best Possible Solution

A focused Feishu plugin change should normalize single newlines in outbound post markdown without altering code fences, existing blank paragraphs, native mentions, or streaming-card markdown behavior unless explicitly covered.

## Reproduction Assessment

Yes at source level: current outbound post payloads pass the original markdown string directly into a Feishu `tag: "md"` element, and existing tests only assert no body rewrite for mentions. A live Feishu client proof has not been run in this read-only review.

## Solution Assessment

Yes, the best fix is narrow Feishu post-message normalization in `send.ts`, with tests for plain sends and edits. Changing all channel chunking, renderMode defaults, or switching all messages to cards would be broader than the reported defect.
