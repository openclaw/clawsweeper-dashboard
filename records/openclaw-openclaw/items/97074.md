## What This Changes

The issue reports that Feishu post markdown messages render single newlines as spaces, making multiline agent output cramped.

## Best Possible Solution

Land one canonical Feishu-owned fix that normalizes post+md paragraph breaks before post-limit enforcement, preserves code/list/card/comment behavior, and includes focused tests plus redacted live Feishu proof.

## Reproduction Assessment

Yes at source level: current main sends ordinary Feishu text unchanged into a post md element through send and edit paths. I did not establish a live Feishu client failure in this read-only review.

## Solution Assessment

Yes in shape, but not by closing this issue yet: the maintainable fix belongs in the Feishu post send/edit payload path with chunk-limit awareness, not in global markdown handling. The open linked PRs need maintainer selection and live proof before the issue can close.
