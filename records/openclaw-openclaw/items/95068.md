## Summary

Keep open: current main and v2026.6.10 still render envelope and system-event dates in ISO order and expose no `agents.defaults.dateFormat`; accepting the request would add a new public config surface and needs maintainer product direction.

## What This Changes

Request to add `agents.defaults.dateFormat` so human-facing envelope and system-event dates can render as ISO, UK, US, or locale order while machine-readable timestamps stay ISO.

## Best Possible Solution

Maintainers should first define the date-display contract; if accepted, implement it once in the shared timestamp formatter, thread it through envelope and system-event callers, update schema/help/docs/tests, and keep machine-readable timestamps and filenames ISO.

## Reproduction Assessment

Not applicable as a bug repro. Source inspection confirms current main has no `agents.defaults.dateFormat` schema entry and still renders envelope/system-event dates through ISO-order formatters.

## Solution Assessment

Unclear until maintainers confirm the product contract. A shared date-format option is a plausible implementation path, but ISO-only behavior, locale behavior, or a narrower scope may be safer than adding another config key.
