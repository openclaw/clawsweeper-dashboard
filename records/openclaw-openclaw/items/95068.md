## What This Changes

Request adds an `agents.defaults.dateFormat` option for ISO, UK, US, or locale date order in human-readable envelope and system-event timestamps while keeping machine-readable timestamps and filenames ISO.

## Best Possible Solution

Maintainers should decide the date-display contract first; if accepted, implement it once in the shared timestamp formatter, thread it through envelope and system-event callers, update schema/help/docs/tests, and keep machine-readable timestamps and filenames ISO.

## Reproduction Assessment

Not applicable as a bug repro. Source inspection gives high confidence that current main has no `agents.defaults.dateFormat` key and still renders envelope/system-event dates through YYYY-MM-DD formatters.

## Solution Assessment

Unclear until maintainers confirm the product contract. A shared date-format option is a plausible implementation path, but adding a new config key and changing model-visible timestamp shape may not be the best default direction.
