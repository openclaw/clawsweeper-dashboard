## What This Changes

The issue requests a new `agents.defaults.dateFormat` option for UK, US, ISO, or locale-style date ordering in human-readable envelope and system-event timestamps.

## Best Possible Solution

Maintainers should first decide the date-display contract; if accepted, implement it once in the shared formatter and thread it through envelope/system-event callers, schema/help/docs, and focused tests while keeping filenames and machine-readable timestamps ISO.
