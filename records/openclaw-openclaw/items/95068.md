## Summary

Keep open: current main and v2026.6.8 still hard-code envelope and system-event date order as YYYY-MM-DD, but adding `dateFormat` would be a new public config surface and needs maintainer product direction.

## What This Changes

The issue requests an `agents.defaults.dateFormat` option for ISO, UK, US, or locale-style date display in message envelopes and system-event lines.

## Best Possible Solution

Maintainers should decide the date-display contract first; if accepted, implement it once in the shared timestamp formatter, thread it through envelope and system-event callers, update schema/help/docs, and keep machine-readable timestamps and filenames ISO.

## Reproduction Assessment

Not applicable as a bug repro. Source inspection confirms current main has no `agents.defaults.dateFormat` schema entry and still renders envelope/system-event dates through YYYY-MM-DD formatters.

## Solution Assessment

Unclear until maintainers confirm the product contract. A shared date-format option is a plausible clean implementation path, but ISO-only behavior, locale behavior, or a narrower scope may be safer than adding another config key.
