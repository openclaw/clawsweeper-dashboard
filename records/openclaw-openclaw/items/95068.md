## Summary

Keep open: current main and v2026.6.10 still render the requested envelope and system-event date portions as YYYY-MM-DD and the config schema has no dateFormat key, but accepting this would add a new public config surface and needs maintainer product direction first.

## What This Changes

Request to add an agents.defaults.dateFormat option for ISO, UK, US, or locale date ordering in human-readable envelope and system-event timestamp displays.

## Best Possible Solution

Maintainers should first define whether prompt-facing display timestamps stay ISO-only or get a constrained date-format enum; if accepted, implement it once in the shared timestamp formatter, thread it through envelope and system-event callers, update schema/help/docs/tests, and keep filenames plus timestampUtc/audit data ISO.

## Reproduction Assessment

Not applicable as a bug repro. Source inspection confirms current main has no agents.defaults.dateFormat schema entry and still renders envelope and system-event dates through YYYY-MM-DD formatters.

## Solution Assessment

Unclear until maintainers confirm the product contract. A shared constrained enum in the existing timestamp formatter is the cleanest technical shape if accepted, but locale/custom formats may be less safe than preserving ISO-only prompt timestamps.
