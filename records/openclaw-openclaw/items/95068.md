## Summary

Keep open: current main still hardcodes ISO-style dates for the requested envelope and system-event display paths, and the remaining work is a new core config surface that needs maintainer product approval rather than conservative cleanup.

## What This Changes

The issue requests a new `agents.defaults.dateFormat` option for human-readable envelope and system-event date display while keeping machine-readable timestamps ISO.

## Best Possible Solution

If maintainers accept the config surface, add one shared display-date formatter for envelope and system-event timestamps, wire it through schema/types/help/docs/tests, and explicitly keep filenames, audit logs, and `timestampUtc` unchanged.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives a high-confidence current-main check that envelope and system-event dates are assembled in ISO-style order without a date-format setting.

## Solution Assessment

Unclear until maintainers choose the product contract. A shared formatter is the maintainable implementation shape if approved, but adding a global `dateFormat` key needs config-surface review first.
