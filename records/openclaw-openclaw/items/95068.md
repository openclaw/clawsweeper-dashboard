## Summary

Keep open: current main and v2026.6.9 still have no configurable date-format setting and the shared envelope/system-event formatter remains ISO-ordered, but accepting this would add a new public `agents.defaults` config key and change model-visible timestamp text, so it needs maintainer product direction.

## What This Changes

The issue requests an `agents.defaults.dateFormat` option for ISO, UK, US, or locale-style date display in message envelopes and system-event lines while keeping machine-readable timestamps ISO.

## Best Possible Solution

Maintainers should first define the date-display contract; if accepted, implement it once in the shared timestamp formatter, thread it through envelope and system-event callers, update schema/help/docs/tests, and keep machine-readable fields and filenames ISO.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection confirms current main has no `agents.defaults.dateFormat` schema entry and still renders the relevant timestamp dates through ISO-order helpers.

## Solution Assessment

Unclear until maintainers confirm the product contract. A shared date-format option is a plausible implementation path, but ISO-only behavior or a narrower locale policy may be safer than adding another config key.
