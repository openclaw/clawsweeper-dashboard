## Summary

Keep open. Current main still uses Feishu block fallback text as a delta append while partial replies use snapshot merging, and the same-author closing PR for this exact bug is still open rather than merged.

## What This Changes

Fix Feishu streaming cards so overlapping block payloads and partial snapshots merge instead of repeating reply text.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74146, or replace it with an equivalent Feishu-local dispatcher/test fix, while leaving broader delivery variants on the separate tracker.

## Reproduction Assessment

Yes, at source level. Current main has a focused dispatcher test that drives `onPartialReply("hello")`, then block payload `"lo world"`, and expects the duplicated `hellolo world` close payload.

## Solution Assessment

Yes. Switching the Feishu block fallback update to the existing snapshot merge path is the narrow maintainable fix because it reuses `mergeStreamingText` and stays inside the Feishu plugin boundary.
