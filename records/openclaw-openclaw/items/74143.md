## Summary

Keep open. Current main still appends Feishu block fallback text as a delta while partial snapshots use overlap-aware merging, and #74146 is an open same-author closing PR for this exact dispatcher/test fix.

## What This Changes

The issue reports duplicated Feishu streaming-card content when partial snapshot text and block fallback text overlap.

## Best Possible Solution

Land or replace #74146 with a Feishu dispatcher/test change that routes block fallback updates through overlap-aware snapshot merging while preserving block-only fallback close behavior.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: the dispatcher test sends partial `hello`, then block `lo world`, and expects the duplicated close payload `hellolo world`.

## Solution Assessment

Yes. Reusing the existing Feishu snapshot merge path for block fallback updates is the narrowest maintainable direction because it stays inside the Feishu plugin and avoids changing the generic streaming contract.
