## Summary

Keep open. Current main still sends Feishu block fallback text through the delta append path, and the same-author closing PR #74146 is open rather than merged, so this issue should remain open for PR review or an equivalent fix.

## What This Changes

The issue requests deduping Feishu streaming-card content when overlapping block payloads and partial snapshots arrive in the same reply.

## Best Possible Solution

Keep the issue open until PR #74146, or an equivalent Feishu dispatcher/test fix, lands and preserves block-only fallback delivery while using the overlap-aware snapshot merge.

## Reproduction Assessment

Yes. Current main has a deterministic source-level reproduction: `onPartialReply` sends `hello`, `deliver({ kind: "block" })` sends `lo world`, and the existing test expects `hellolo world`.

## Solution Assessment

Yes. Routing Feishu block fallback updates through the existing snapshot merge semantics is the narrowest maintainable direction because it stays inside the Feishu plugin dispatcher/tests and reuses `mergeStreamingText`.
