## Summary

Keep open. Current main still routes Feishu block fallback text through the delta append path while partial snapshots use overlap-aware merging, and the same-author closing PR #74146 is still open and unmerged.

## What This Changes

Fix Feishu streaming-card duplication when overlapping block payloads and partial snapshots are both mirrored into the card stream.

## Best Possible Solution

Review and land or replace #74146 with a Feishu-local dispatcher/test change that routes block fallback updates through overlap-aware snapshot merging while preserving block-only close fallback behavior.

## Reproduction Assessment

Yes. Current main has a source-level reproduction in the Feishu dispatcher test: partial text `hello` followed by block text `lo world` currently closes as `hellolo world`.

## Solution Assessment

Yes. Reusing the existing snapshot merge path for Feishu block fallback updates is the narrowest maintainable fix because it stays inside the Feishu plugin and does not change the generic streaming contract.
