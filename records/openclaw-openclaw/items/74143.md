## Summary

Keep open. Current main still appends Feishu block fallback text as a delta while partial snapshots use overlap-aware merging, and the linked closing PR #74146 is open and unmerged.

## What This Changes

Prevent Feishu streaming-card replies from duplicating overlapping block payload text that also arrived through partial snapshot updates.

## Best Possible Solution

Land or replace #74146 with a Feishu dispatcher/test change that routes block fallback updates through overlap-aware snapshot merging while preserving block-only fallback close behavior.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: the dispatcher test sends `onPartialReply` text `hello`, then block text `lo world`, and expects the duplicated `hellolo world` close payload.

## Solution Assessment

Yes. Reusing the existing Feishu snapshot merge path for block fallback updates is the narrowest maintainable direction because it stays inside the Feishu plugin and avoids changing the generic streaming contract.
