## Summary

Keep open. Current main still routes Feishu block fallback streaming text through delta concatenation while partial snapshots use overlap-aware merging, and the open same-author fix PR is linked with closing syntax for this issue.

## What This Changes

The issue requests fixing Feishu streaming-card duplication when overlapping block payloads and partial snapshots are both streamed.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74146, or replace it with an equivalent Feishu-local dispatcher/test patch, while keeping broader final/stale delivery work on https://github.com/openclaw/openclaw/issues/77685.

## Reproduction Assessment

Yes, at source level. Current main has a focused path where `onPartialReply` sends `hello`, a block payload sends `lo world`, and the existing Feishu dispatcher test expects `hellolo world`; I did not run a live Feishu tenant in this read-only review.

## Solution Assessment

Yes. Routing the Feishu block fallback update through the existing snapshot merge path is the narrow maintainable fix because it stays inside the plugin and reuses the helper already used for partial snapshots.
