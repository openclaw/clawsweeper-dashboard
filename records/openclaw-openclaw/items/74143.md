## Summary

Current main still routes overlapping Feishu streaming-card block fallback text through the delta append path, and the same-author closing PR #74146 is open rather than merged, so this issue should remain open for PR review or replacement.

## What This Changes

The issue requests overlap-aware merging for Feishu streaming-card block fallback payloads when they overlap with partial snapshot updates.

## Best Possible Solution

Keep the issue open until https://github.com/openclaw/openclaw/pull/74146, or an equivalent Feishu dispatcher/test fix, lands and preserves block-only fallback delivery while using the existing overlap-aware snapshot merge.

## Reproduction Assessment

Yes. Current main has a deterministic source-level reproduction: the Feishu dispatcher test sends partial `hello` followed by block `lo world` and currently expects `hellolo world`, matching the reported overlap duplication.

## Solution Assessment

Yes. Routing Feishu block fallback updates through the existing snapshot merge semantics is the narrowest maintainable fix because it stays inside the Feishu plugin dispatcher/tests and reuses `mergeStreamingText`.
