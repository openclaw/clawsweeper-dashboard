## Summary

Keep open. The Feishu overlap duplication still exists on current main and in the latest release, and open PR #74146 explicitly closes this issue but has not merged.

## What This Changes

The issue requests that Feishu streaming-card block payload text be merged overlap-aware with partial snapshots instead of raw-concatenated when the payloads overlap.

## Best Possible Solution

Keep this issue open until PR #74146, or an equivalent narrow Feishu plugin dispatcher/test fix, lands. The desired end state is for block fallback text to use the existing overlap-aware snapshot merge while preserving block-only fallback delivery and adding regression coverage for overlapping partial snapshots plus block payloads.

## Reproduction Assessment

Yes. Current main has a deterministic source-level reproduction: partial `hello` followed by block `lo world` is expected by the Feishu dispatcher test to close as `hellolo world`, matching the reported duplicated overlap.

## Solution Assessment

Yes. Routing Feishu block fallback updates through the existing snapshot merge semantics is the narrowest maintainable fix because it reuses `mergeStreamingText` and keeps the change inside the Feishu plugin dispatcher/tests.
