## Summary

Keep open. Current main still routes Feishu block fallback text through the delta append path while partial replies use the overlap-aware snapshot merge path, and the linked same-author PR #74146 is open and unmerged.

## What This Changes

The issue asks Feishu streaming-card block fallback updates to merge overlapping block payload text with partial snapshots instead of appending duplicate text.

## Best Possible Solution

Land or replace #74146 with a Feishu dispatcher and regression-test change that routes block fallback updates through overlap-aware snapshot merging while preserving block-only close behavior.

## Reproduction Assessment

Yes. Current main has a deterministic source-level reproduction: `onPartialReply` sends `hello`, `deliver({ kind: "block" })` sends `lo world`, and the existing test expects the duplicated `hellolo world` result.

## Solution Assessment

Yes. Reusing the existing Feishu snapshot merge semantics for block fallback updates is the narrowest maintainable fix because it stays inside the Feishu plugin and avoids changing the generic channel streaming contract.
