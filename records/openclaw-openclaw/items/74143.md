## Summary

Keep open. Current `main` still routes Feishu block fallback text through the delta concatenation path, the focused dispatcher test still expects the duplicated `hellolo world` result, and the linked closing PR remains open and unmerged.

## What This Changes

The issue reports duplicated Feishu streaming-card content when overlapping block payloads are concatenated after partial snapshot updates.

## Best Possible Solution

Keep this issue open until PR #74146, or an equivalent narrow Feishu dispatcher/test fix, lands. The desired end state is for block fallback text to use the existing overlap-aware snapshot merge while preserving block-only fallback delivery and adding regression coverage for overlapping partial snapshots plus block payloads.

## Reproduction Assessment

Yes. Current `main` has a deterministic source-level reproduction: partial `hello` followed by block `lo world` is still expected by the Feishu dispatcher test to close as `hellolo world`, matching the reported overlap duplication.

## Solution Assessment

Yes. Routing Feishu block fallback updates through existing snapshot merge semantics is the narrowest maintainable fix because it reuses `mergeStreamingText` and stays inside the Feishu plugin dispatcher/tests.
