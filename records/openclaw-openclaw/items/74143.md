## Summary

Keep open. Current main still routes Feishu block fallback text through the delta concatenation path, the current dispatcher test still expects the duplicated `hellolo world` close text, and open PR #74146 explicitly closes this issue but has not merged, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that Feishu streaming-card block fallback updates merge overlapping text with partial snapshot updates instead of appending them as raw deltas.

## Best Possible Solution

Keep this issue open until PR #74146, or an equivalent narrow Feishu plugin dispatcher/test fix, lands. The desired end state is for block fallback text to use the existing overlap-aware snapshot merge while preserving block-only fallback delivery and adding regression coverage for overlapping partial snapshots plus block payloads.

## Reproduction Assessment

Yes. Current main has a deterministic source-level reproduction: partial `hello` followed by block `lo world` is still expected by the Feishu dispatcher test to close as `hellolo world`, matching the reported overlap duplication.

## Solution Assessment

Yes. Routing Feishu block fallback updates through the existing snapshot merge semantics is the narrowest maintainable solution because it reuses `mergeStreamingText` and stays inside the Feishu plugin dispatcher/tests.
