## Summary

Keep open. Current main still sends Feishu block fallback text through the delta append path, the focused dispatcher test still expects duplicated overlap text, and the linked closing PR remains open rather than merged.

## What This Changes

The issue requests that Feishu streaming-card block payloads merge overlap-aware with partial snapshots instead of duplicating overlapping text.

## Best Possible Solution

Keep this issue open until PR #74146, or an equivalent Feishu dispatcher and regression-test fix, lands. The desired end state is for block fallback text to use the existing snapshot merge semantics while preserving block-only fallback delivery on idle close.

## Reproduction Assessment

Yes. Current main has a deterministic source-level reproduction: partial "hello" followed by block "lo world" is still expected by the Feishu dispatcher test to close as "hellolo world".

## Solution Assessment

Yes. Routing Feishu block fallback updates through the existing snapshot merge path is the narrowest maintainable fix because it stays inside the Feishu plugin dispatcher and reuses mergeStreamingText.
