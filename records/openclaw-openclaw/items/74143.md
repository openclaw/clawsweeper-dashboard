## Summary

Keep open. Current main still appends Feishu block fallback text as a delta while partial snapshots use overlap-aware merging, and https://github.com/openclaw/openclaw/pull/74146 remains open with closing syntax for this issue.

## What This Changes

The issue reports duplicate Feishu streaming-card text when partial snapshot updates overlap with block-delivery payloads.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74146, or replace it with an equivalent Feishu-local dispatcher/test patch, while keeping broader final/stale delivery work on https://github.com/openclaw/openclaw/issues/77685.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: the Feishu dispatcher test sends partial text `hello`, then block text `lo world`, and expects the duplicated close payload `hellolo world`; I did not run a live Feishu tenant in this read-only review.

## Solution Assessment

Yes. Routing only the Feishu block fallback update through the existing snapshot merge path is the narrow maintainable solution because it stays inside the plugin and reuses the helper already used by partial snapshots.
