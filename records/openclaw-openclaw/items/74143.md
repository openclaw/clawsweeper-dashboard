## Summary

Keep open. Current main still appends Feishu block fallback text in delta mode while partial replies use snapshot merging, and the same-author closing PR for this issue is still open rather than merged.

## What This Changes

The issue requests fixing Feishu streaming card text duplication when overlapping block fallback payloads are appended as deltas alongside partial snapshot updates.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74146, or replace it with an equivalent Feishu-local dispatcher/test fix, while leaving broader final/stale delivery work on https://github.com/openclaw/openclaw/issues/77685.

## Reproduction Assessment

Yes, at source level. Current main has a focused dispatcher test that drives `onPartialReply("hello")`, then block payload `"lo world"`, and expects the duplicated `hellolo world` close payload.

## Solution Assessment

Yes. Switching the Feishu block fallback update to the existing snapshot merge path is the narrow maintainable fix because it reuses `mergeStreamingText` and stays inside the plugin boundary.
