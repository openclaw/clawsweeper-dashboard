## Summary

Keep open. Current main still routes overlapping Feishu streaming-card block fallback text through the delta append path, the focused dispatcher test still expects the duplicated `hellolo world` result, and the linked PR that explicitly closes this issue remains open and unmerged.

## What This Changes

The issue requests that Feishu streaming-card block fallback payloads merge overlap-aware with partial snapshots instead of appending duplicated text.

## Best Possible Solution

Keep this issue open until PR #74146, or an equivalent narrow Feishu dispatcher/test fix, lands. The desired end state is for block fallback text to use the existing overlap-aware snapshot merge while preserving block-only fallback delivery and adding regression coverage for overlapping partial snapshots and block payloads.

## Reproduction Assessment

Yes. Current main has a deterministic source-level reproduction: the Feishu dispatcher test sends partial `hello` followed by block `lo world` and expects the streaming card to close as `hellolo world`, matching the reported overlap duplication.

## Solution Assessment

Yes. Routing Feishu block fallback updates through the existing snapshot merge semantics is the narrowest maintainable fix because it stays inside the Feishu plugin dispatcher/tests and reuses `mergeStreamingText`.
