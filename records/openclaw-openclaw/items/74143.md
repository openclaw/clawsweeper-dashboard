## Summary

Current main still routes Feishu streaming-card block fallback text through the delta append path, and the same-author closing PR #74146 remains open, so this issue should stay open for that PR or an equivalent fix.

## What This Changes

Request Feishu streaming-card block payloads to merge overlap with partial snapshot text instead of appending duplicate content.

## Best Possible Solution

Keep the issue open until #74146 or an equivalent Feishu dispatcher/test fix lands, preserving block-only fallback delivery while using overlap-aware merge semantics for block payloads.

## Reproduction Assessment

Yes. Current main has a deterministic source-level reproduction: the dispatcher test sends partial `hello` followed by block `lo world` and expects the duplicated `hellolo world` result.

## Solution Assessment

Yes. Routing Feishu block fallback updates through the existing snapshot merge path is the narrowest maintainable fix because it stays inside the Feishu plugin and reuses `mergeStreamingText`.
