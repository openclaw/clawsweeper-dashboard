## Summary

Keep open. Current main still routes Feishu block fallback text through append-only delta mode while partial snapshots use the overlap-aware snapshot path, and same-author PR #74146 is open with closing syntax for this issue.

## What This Changes

The issue requests deduping Feishu streaming-card content when overlapping block payloads and partial snapshots arrive in the same reply.

## Best Possible Solution

Keep the issue open until PR #74146 lands, is closed as superseded, or maintainers apply an equivalent Feishu dispatcher/test fix that preserves block-only fallback delivery while using overlap-aware snapshot merging.

## Reproduction Assessment

Yes. Current main has a deterministic source-level reproduction: `onPartialReply` sends `hello`, `deliver({ kind: "block" })` sends `lo world`, and the existing test expects `hellolo world`.

## Solution Assessment

Yes. Routing Feishu block fallback updates through the existing snapshot merge semantics is the narrowest maintainable direction because it stays inside the Feishu plugin dispatcher/tests and reuses `mergeStreamingText`.
