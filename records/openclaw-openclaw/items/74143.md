## Summary

Keep open. Current main still appends Feishu streaming-card block fallback text as a delta while partial snapshots use overlap-aware merging, and the linked closing PR #74146 remains the active implementation path rather than a reason to close this issue before merge.

## What This Changes

Feishu streaming cards should merge overlapping block payloads and partial snapshots instead of appending duplicated text.

## Best Possible Solution

Review and land or replace #74146 with a Feishu dispatcher/test change that routes block fallback updates through overlap-aware snapshot merging while preserving block-only fallback close behavior.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: the dispatcher test sends partial text "hello", then block text "lo world", and expects the duplicated close payload "hellolo world".

## Solution Assessment

Yes. Reusing the existing Feishu snapshot merge path for block fallback updates is the narrowest maintainable fix because it stays inside the Feishu plugin and avoids changing the generic streaming contract.
