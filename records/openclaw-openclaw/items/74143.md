## Summary

Keep open. Current main still routes Feishu streaming-card block fallback text through the delta concatenation path, the focused dispatcher test still encodes duplicated overlap output, and the linked closing PR remains open rather than merged.

## What This Changes

The issue requests Feishu streaming-card block payloads to use overlap-aware snapshot merging when they overlap with partial reply snapshots.

## Best Possible Solution

Keep this issue open until PR #74146, or an equivalent narrow Feishu dispatcher/test fix, lands. The desired end state is for block fallback text to use the existing overlap-aware snapshot merge while preserving block-only fallback delivery and adding regression coverage for overlapping partial snapshots plus block payloads.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: the Feishu dispatcher test sends partial `hello` followed by block `lo world` and currently expects duplicated `hellolo world` output.

## Solution Assessment

Yes. Routing Feishu block fallback updates through the existing snapshot merge semantics is the narrowest maintainable fix because it stays in the Feishu plugin dispatcher/tests and reuses `mergeStreamingText`.
