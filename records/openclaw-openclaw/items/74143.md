## Summary

Keep open. Current main still routes Feishu block fallback text through the delta append path, and the same-author closing PR #74146 is open rather than merged.

## What This Changes

Fix Feishu streaming-card duplication when overlapping block payloads and partial snapshots are combined.

## Best Possible Solution

Review and land PR #74146, or an equivalent Feishu dispatcher/test fix, while preserving block-only fallback delivery and routing overlap cases through snapshot merge semantics.

## Reproduction Assessment

Yes. Current main has a deterministic source-level reproduction: partial `hello` followed by block `lo world` currently closes the streaming card with `hellolo world` in the existing test.

## Solution Assessment

Yes. Switching the Feishu block fallback update to the existing snapshot merge path is the narrowest maintainable solution because it stays plugin-local and reuses `mergeStreamingText`.
