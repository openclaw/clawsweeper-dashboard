## Summary

Keep open. Current main still sends Feishu streaming-card block fallback text through the delta concatenation path, the focused dispatcher test still expects duplicated overlap text, and the linked closing PR remains open and unmerged.

## What This Changes

Change Feishu streaming-card block fallback updates so overlapping block payloads are merged with partial snapshot text instead of concatenated and duplicating content.

## Best Possible Solution

Keep this issue open until PR #74146, or an equivalent narrow Feishu dispatcher/test fix, lands. The desired end state is for block fallback text to use the existing overlap-aware snapshot merge while preserving block-only fallback delivery and adding regression coverage for overlapping partial snapshots plus block payloads.

## Reproduction Assessment

Yes. A deterministic source-level reproduction exists on current main: the Feishu dispatcher test sends partial `hello` followed by block `lo world` and expects the streaming card to close as `hellolo world`, matching the reported overlap duplication.

## Solution Assessment

Yes. Routing Feishu block fallback updates through the existing snapshot merge semantics is the narrowest maintainable fix because it stays in the Feishu plugin dispatcher/tests and reuses `mergeStreamingText`.
