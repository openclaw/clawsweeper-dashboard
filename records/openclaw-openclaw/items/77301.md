## Summary

Keep open. The report is a current memory-core performance/storage proposal rather than a stale or already-implemented bug: main has hybrid search and sqlite-vec KNN, but it still does not set `cache_size`, still stores JSON embeddings in `chunks.embedding`, and the requested storage/query changes need maintainer scoping and measurement.

## What This Changes

Request three memory-core SQLite optimizations: set a per-connection cache-size PRAGMA, evaluate an FTS5-prefiltered vector rerank path, and remove or replace duplicate JSON embedding storage.

## Best Possible Solution

Split this into measured memory-core work items: benchmark cache-size behavior, compare FTS-prefilter rerank against the existing sqlite-vec KNN hybrid path, and only reduce embedding duplication after preserving fallback and migration behavior.

## Reproduction Assessment

No high-confidence latency reproduction was established because this read-only review did not create a large SQLite memory store. Source inspection does confirm the current schema, storage, and query paths behind the requested optimizations.

## Solution Assessment

No, not as one direct repair. The three suggestions have different tradeoffs, and the TEXT embedding column is still an active fallback dependency, so the maintainable path is scoped benchmarking plus separate migration-safe changes.
