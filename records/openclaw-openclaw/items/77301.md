## Summary

Keep open: current main already covers part of the request with hybrid BM25/vector merging and sqlite-vec KNN, but the per-connection cache-size default and duplicate embedding storage remain unresolved performance, memory-budget, and schema-compatibility decisions.

## What This Changes

The issue requests three memory-core SQLite optimizations: set a larger per-connection SQLite page cache, change recall to FTS-prefiltered vector reranking, and remove or replace duplicate JSON TEXT embedding storage.

## Best Possible Solution

Keep this as maintainer-owned performance/schema design and split any accepted work into benchmark-backed PRs for cache sizing, recall strategy, and embedding storage migration/backcompat.

## Reproduction Assessment

No high-confidence live performance reproduction was run. Source inspection does reproduce the current code state for the missing `cache_size` PRAGMA and active duplicate/fallback TEXT embedding storage, but not the claimed latency improvement.

## Solution Assessment

Unclear: applying the request exactly is not yet proven to be the best solution because current main already has hybrid merge plus sqlite-vec KNN, while the remaining cache and schema changes need benchmark, memory-budget, recall-quality, and migration review.
