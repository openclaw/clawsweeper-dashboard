## Summary

Keep open: current main already has sqlite-vec KNN and documented parallel BM25/vector hybrid merging, but the requested cache-size PRAGMA, FTS-prefiltered rerank strategy, and duplicate TEXT embedding storage cleanup remain unresolved design/performance tradeoffs.

## What This Changes

Requests memory-core SQLite performance and schema optimizations: a larger per-connection page cache, FTS-prefiltered vector reranking, and removal or replacement of duplicate JSON TEXT embedding storage.

## Best Possible Solution

Keep this as a benchmark-backed memory-core design task, then split any accepted cache, recall, and schema changes into focused PRs with migration and fallback coverage.

## Reproduction Assessment

No high-confidence performance reproduction was run against a large current-main store. Source inspection does verify the missing cache_size PRAGMA and active JSON TEXT embedding fallback, while also showing that parallel hybrid search and sqlite-vec KNN already exist.

## Solution Assessment

Unclear: the proposal is plausible, but the safest implementation needs benchmark evidence and maintainer decisions on RAM budget, recall-quality tradeoffs, and schema compatibility before code changes.
