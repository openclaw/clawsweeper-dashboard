## Summary

Keep open: current main still lacks the requested SQLite cache-size default, uses parallel keyword/vector hybrid merge rather than FTS-prefiltered vector reranking, and keeps JSON TEXT embeddings as active fallback storage, so this remains a design and benchmark task rather than a cleanup-close candidate.

## What This Changes

The issue requests three memory-core SQLite optimizations: set a larger per-connection cache_size, change hybrid recall to FTS-prefiltered vector reranking, and remove or replace duplicate JSON TEXT embedding storage.

## Best Possible Solution

Keep this as a benchmark-backed memory-core design task that decides cache budget, retrieval semantics, and schema/backcompat before changing defaults.

## Reproduction Assessment

No high-confidence performance reproduction was run. Source inspection on current main confirms the requested code-level gaps: no cache_size PRAGMA, no FTS-prefiltered vector rerank, and active JSON TEXT embedding fallback storage.
