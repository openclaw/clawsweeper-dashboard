Full review comments:

- **[P1] Keep non-string helpers out of the string SDK subpath:** `src/plugin-sdk/string-coerce-runtime.ts:17-22`
  - body: This adds number, boolean, record, and generic unique helpers to `openclaw/plugin-sdk/string-coerce-runtime`, even though numeric coercion already has a dedicated SDK path.   - confidence: 0.9
- **[P2] Use a deterministic comparator for shared string sorting:** `src/shared/string-normalization.ts:20`
  - body: `localeCompare` depends on host locale/ICU, but this helper now feeds plugin IDs, labels, allowlists, and config/cache normalization. Replaced call sites often used code-unit `.toSorted()` ordering, so punctuation, case, or diacritics can order differently across machines; use a locale-independent comparator here.
  - confidence: 0.88
- **[P2] Preserve record-only Google Meet provider config:** `extensions/google-meet/src/config.ts:7`
  - body: Importing the SDK `asRecord` changes the previous local helper from array-rejecting to array-accepting. `resolveProvidersConfig` iterates `Object.entries(raw)`, so an array under `realtime.providers` can create numeric provider IDs instead of being ignored; use a strict record helper for config roots.
  - confidence: 0.86
- **[P2] Keep memory extra paths string-only:** `src/agents/memory-search.ts:243-246`
  - body: `normalizeStringEntries` stringifies every value, but `memorySearch.extraPaths` is a path/read allowlist. This can turn invalid values such as `false` or objects into path strings before memory path resolution; preserve the previous string-only trim/dedupe boundary here.
  - confidence: 0.8
