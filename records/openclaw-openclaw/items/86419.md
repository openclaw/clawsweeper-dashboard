Full review comments:

- **[P1] Keep non-string helpers out of the string SDK subpath:** `src/plugin-sdk/string-coerce-runtime.ts:18-22`
  - body: This public Plugin SDK subpath now exports number and boolean coercion helpers, and the API baseline changes with it. Because third-party plugins can import this path, landing this turns a string helper barrel into a broader primitive contract without an explicit SDK decision; keep these helpers in a more accurately named/narrow subpath or leave them internal until the contract is approved.
  - confidence: 0.84
- **[P2] Preserve record-only Google Meet provider config:** `extensions/google-meet/src/config.ts:7`
  - body: Importing the shared `asRecord` here changes the object guard used by `resolveProvidersConfig`: the shared helper accepts arrays, while the removed local helper rejected them. A malformed array under the providers config would now be iterated as numeric provider IDs instead of being ignored, changing setup/runtime behavior for invalid existing config.
  - confidence: 0.82
- **[P2] Keep memory extra paths string-only:** `src/agents/memory-search.ts:243-247`
  - body: `normalizeStringEntries` stringifies every entry with `String(value)`, so malformed `memorySearch.extraPaths` values such as `false` or objects can become path strings like `false` or `[object Object]`. That output feeds memory path resolution and can silently broaden the memory-read scope instead of preserving the prior string-only list behavior.
  - confidence: 0.78
- **[P2] Use deterministic ordering in the shared sort helper:** `src/shared/string-normalization.ts:19-20`
  - body: `localeCompare` depends on the host locale/ICU data, while several replaced call sites used default code-unit `.toSorted()` ordering. Shared normalization helpers should produce the same plugin/config/cache lists on every host, so use a locale-independent comparator here.
  - confidence: 0.86
