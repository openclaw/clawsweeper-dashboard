## Summary

Keep open. Current main still has no provider-agnostic cron final-output boundary for untagged assistant prose, and the maintainer discussion explicitly rejected a DeepSeek-specific sanitizer or marker/emoji stripping as the solution.

## What This Changes

The issue requests that Telegram cron delivery stop forwarding DeepSeek's untagged preamble text after the 2026.3.13 to 2026.4.5 behavior change.

## Best Possible Solution

Define a generic isolated-cron final-output contract that channels can trust without model IDs, emojis, prose heuristics, or unsupported cron payload fields, then add regression coverage for an unphased preamble-plus-report Telegram cron output.

## Reproduction Assessment

Yes, by source inspection rather than a live run. Current main still forwards a single unphased assistant text value through the cron final-visible-text path, which matches the reported DeepSeek preamble-plus-report shape.

## Solution Assessment

No, the suggested marker or emoji sanitizer is not the best fix. The safer path is the maintainer-requested generic cron final-output boundary after tracing the 2026.3.13 to 2026.4.5 delivery-path change.
