## Label Justifications

- `P1`: This is a reported upgrade regression in Telegram cron delivery where unwanted assistant preamble text reaches real users consistently for the affected DeepSeek jobs.
- `impact:message-loss`: Internal or unphased assistant text is being routed into channel-visible Telegram messages, corrupting the intended user-facing delivery.

## Summary

Keep open. Current main still preserves legacy unphased assistant text as visible output and Telegram cron announce delivery can collapse text-only output to that same value, while the live maintainer discussion explicitly asks for a provider-agnostic cron final-output contract instead of a DeepSeek filter or marker sanitizer.

## What This Changes

Fix isolated Telegram cron delivery so untagged DeepSeek preamble/narration is excluded from the final user-facing message.

## Best Possible Solution

Define a generic isolated-cron final-output contract that channels can trust without model IDs, emojis, prose heuristics, or unsupported cron payload fields, then add regression coverage for an unphased preamble-plus-report Telegram cron output.

## Reproduction Assessment

Yes, by source inspection rather than a live run: current main falls back to unphased assistant text, Telegram opts into finalAssistantVisibleText, and cron collapses text-only announce delivery to that value.

## Solution Assessment

No, the suggested sanitizer/outputStartsWith/enforceFinalTag direction is not the best fix. The maintainable path is the maintainer-requested provider-agnostic cron final-output boundary after tracing the upgrade regression.
