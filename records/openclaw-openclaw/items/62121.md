## Summary

Keep open. Current main has phase-tagged commentary suppression and Telegram cron final-visible-text collapse, but plain untagged DeepSeek preamble still falls through as assistant-visible text, and the maintainer discussion explicitly left this for a generic cron final-output contract rather than a heuristic sanitizer.

## What This Changes

Prevent DeepSeek plain untagged preamble text from being delivered by Telegram isolated cron jobs after the 2026.3.13 to 2026.4.5 upgrade.

## Best Possible Solution

Define a provider-agnostic isolated-cron final-output contract that channels can trust without model IDs, emojis, or prose guessing, while preserving phase-aware final_answer handling and adding a regression for one unphased preamble-plus-report Telegram cron output.

## Reproduction Assessment

Yes. The reporter gave a consistent version/model/channel reproduction, and current source supports a synthetic repro: one unphased assistant text containing preamble plus the report remains the visible text used by Telegram cron delivery.

## Solution Assessment

No. The suggested outputStartsWith or emoji sanitizer is not the best fix; the safer direction is the maintainer-requested generic final-output boundary after tracing the 2026.3.13 to 2026.4.5 delivery-path change.
