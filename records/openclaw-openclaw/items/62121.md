## Summary

Keep open. Current main still treats untagged assistant prose as visible text, and the issue discussion explicitly asks for a generic cron final-output contract rather than a DeepSeek-specific or marker-based sanitizer.

## What This Changes

The issue reports a 2026.3.13 to 2026.4.5 regression where DeepSeek cron output delivered to Telegram includes untagged preamble text before the intended report.

## Best Possible Solution

Define a provider-agnostic isolated-cron final-output contract that channels can trust without model IDs, emojis, or prose heuristics, with regression coverage for one unphased preamble-plus-report Telegram cron output.

## Reproduction Assessment

Yes, by source inspection rather than a live run. An unphased assistant message containing preamble plus report is returned as visible text and then used by Telegram cron delivery when final-visible-text collapse is enabled.

## Solution Assessment

No, the suggested marker or emoji sanitizer is not the best fix. The safer path is the maintainer-requested generic cron final-output boundary after tracing the 2026.3.13 to 2026.4.5 delivery-path change.
