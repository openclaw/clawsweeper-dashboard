## Summary

Keep open: current main still wires BlueBubbles `systemPrompt` only through group config, while the requested per-DM and account-scoped prompt surfaces are absent. This is valid, narrow BlueBubbles feature-parity work rather than an implemented, stale, or duplicate cleanup.

## What This Changes

Request BlueBubbles per-DM `systemPrompt` support, with optional account-wide prompt support, after the prior merged change only added per-group prompts.

## Best Possible Solution

Implement BlueBubbles-owned `direct.<chatGuidOrHandle>.systemPrompt` with exact and `"*"` fallback, update docs/generated config metadata, and split account-wide prompt support unless maintainers choose its precedence rules now.

Yes for Option A: mirroring WhatsApp/Telegram-style `direct.*.systemPrompt` is the narrowest maintainable fix inside the BlueBubbles plugin boundary. Account-wide prompt support is useful but should be intentionally scoped because it needs precedence rules across groups and DMs.
