Keep open: current main still lacks BlueBubbles DM-scoped system prompt support, and an open PR already uses closing syntax to address this exact issue. The correct cleanup path is to review and land or close that paired PR rather than closing this issue early.

## What This Changes

Request BlueBubbles `direct.<chatGuidOrHandle>.systemPrompt` support for per-DM prompt injection, with optional account-wide prompt support as a broader complement.

## Best Possible Solution

Land a BlueBubbles-owned `direct.<chatGuidOrHandle>.systemPrompt` implementation with wildcard fallback, generated metadata/docs updates, and regression coverage; split account-wide prompts unless maintainers deliberately define precedence in the same change.

## Solution Assessment
