## Summary

Keep open: current main still only supports BlueBubbles `systemPrompt` for groups, and the linked open PR #77086 is the active closing implementation for the missing per-DM `direct.*.systemPrompt` behavior.

## What This Changes

The issue requests BlueBubbles per-DM `systemPrompt` config, with optional account-wide prompt support, after the prior group-only implementation left direct messages unsupported.

## Best Possible Solution

Review and land #77086 or an equivalent BlueBubbles-owned `direct.<handle>.systemPrompt` implementation with docs and generated metadata updates; split account-wide prompts unless maintainers deliberately include precedence rules in the same change.

## Reproduction Assessment

Yes, at source level: current main has group-only schema/runtime wiring and an existing test that asserts DMs omit `GroupSystemPrompt`. I did not run tests because this review is read-only, but the current code path is clear.

## Solution Assessment

Yes for the direct-message option: mirroring WhatsApp/Telegram `direct.*.systemPrompt` with wildcard fallback is the narrowest maintainable fix. Account-wide prompt support should be separately scoped or given explicit precedence rules.
