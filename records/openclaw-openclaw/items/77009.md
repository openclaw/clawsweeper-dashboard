## Summary

Keep open: current main still lacks BlueBubbles per-DM or account-level prompt config, and an open closing PR already targets the direct-message part of the request. This should stay open until that PR is reviewed and either merged or replaced.

## What This Changes

Request BlueBubbles per-DM `systemPrompt` support via a `direct.*.systemPrompt` config bucket, with optional account-wide prompt support as a follow-up to the earlier groups-only change.

## Best Possible Solution

Review and land #77086 if it cleanly adds `direct.<chatGuidOrHandle>.systemPrompt` with wildcard fallback, docs, generated metadata, and regression tests; handle account-wide prompt support only with an explicit follow-up decision.

## Reproduction Assessment

Yes, at source level: the BlueBubbles schema exposes prompt config only under `groups`, runtime only reads it when `isGroup` is true, and tests assert DMs omit `GroupSystemPrompt`. I did not run tests because this review is read-only.

## Solution Assessment

Yes for keeping this issue open and using the linked direct-message PR as the implementation path. The direct bucket mirrors existing WhatsApp/Telegram-style config, while account-wide prompt support should be separate unless maintainers intentionally define its precedence.
