## Summary

Keep open: current main still only supports BlueBubbles `systemPrompt` for groups, and #77086 is an open closing PR for the missing per-DM behavior, so this issue should stay open until that PR is reviewed and merged or rejected.

## What This Changes

Request BlueBubbles per-DM `direct.<handle>.systemPrompt` support, with optional account-wide prompt support, after the earlier group-only implementation.

## Best Possible Solution

Review and land #77086 if its direct-chat prompt implementation, wildcard precedence, docs, metadata, tests, and changelog are sound; treat account-wide prompt support as a deliberate follow-up unless maintainers want that scope now.

## Reproduction Assessment

Yes, at source level: current main has no BlueBubbles `direct` prompt schema and the runtime sets `GroupSystemPrompt` only for group messages. I did not run tests because this review is read-only, but the existing test suite asserts the DM omission.

## Solution Assessment

Yes for the direct-chat option: mirroring the WhatsApp and Telegram `direct.<id>.systemPrompt` pattern is the narrowest maintainable fix. Account-wide prompt support is broader and should be included only with explicit precedence rules.
