## Summary

Keep open: current main still only supports BlueBubbles `systemPrompt` for groups, and the linked fix PR is still open with closing syntax, so this issue should track that review rather than be closed early.

## What This Changes

The issue asks BlueBubbles to support per-DM `direct.<chatGuidOrHandle>.systemPrompt` with wildcard fallback, optionally plus account-wide prompt support.

## Best Possible Solution

Review and land #77086 or an equivalent BlueBubbles-owned `direct.<handle>.systemPrompt` implementation with wildcard fallback; treat account-wide prompts as a deliberate follow-up unless precedence is defined in the same change.

## Reproduction Assessment

Yes at source level: current main exposes BlueBubbles `groups.*.systemPrompt` but no `direct` or account-level prompt field, and runtime sets `GroupSystemPrompt` only for `isGroup` messages. I did not run tests because this review is read-only.

## Solution Assessment

Yes for Option A: mirroring the existing WhatsApp/Telegram direct prompt pattern is the narrowest maintainable solution. Option B should be included only with clear precedence rules or split into a separate follow-up.
