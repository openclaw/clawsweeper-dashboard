## Summary

Keep open: current main still only accepts and forwards BlueBubbles `systemPrompt` for groups, not DMs or accounts, and open PR #77086 is the active implementation path rather than a reason to close before merge.

## What This Changes

Add BlueBubbles per-DM `direct.<handle>.systemPrompt` support, with optional account-wide prompt support, to close the gap left by group-only prompt handling.

## Best Possible Solution

Land PR #77086 or an equivalent narrow BlueBubbles change that adds `direct.*.systemPrompt` schema, runtime resolution, docs, metadata, and tests; split account-wide prompts unless maintainers choose the precedence rules.

## Reproduction Assessment

Yes, at source level: a BlueBubbles DM reaches `finalizeInboundContext` with `isGroup` false, and current main assigns `GroupSystemPrompt` to `undefined`. The existing monitor test also asserts that DM omission; no tests were executed in this read-only review.

## Solution Assessment

Yes for Option A: `direct.<handle>.systemPrompt` with wildcard fallback mirrors WhatsApp and Telegram and stays within BlueBubbles-owned config/runtime code. Account-wide support should be deliberate follow-up scope unless maintainers want it now.
