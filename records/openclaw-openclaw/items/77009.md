## Summary

Keep open: current main still lacks BlueBubbles per-DM `systemPrompt` support, and the issue is now paired with open PR #77086 using closing syntax, so it should remain open until that PR is reviewed and either merged or closed.

## What This Changes

Request BlueBubbles `direct.<id>.systemPrompt` and optionally account-wide `systemPrompt` support after the prior group-only BlueBubbles prompt work left direct messages uncovered.

## Best Possible Solution

Review PR #77086 and land or close it; the narrow end state should add BlueBubbles-owned `direct.<id>.systemPrompt` with wildcard fallback, docs, generated metadata, and regression coverage, while account-wide prompts remain a deliberate follow-up unless maintainers choose otherwise.

## Reproduction Assessment

Not applicable as a bug reproduction, but the missing feature is source-reproducible: current main only resolves `GroupSystemPrompt` from `account.config.groups` for group messages, and tests assert DMs omit it. I did not run tests because this was a read-only review.

## Solution Assessment

Yes for the direct-chat half: adding `channels.bluebubbles.accounts.<acct>.direct.<id>.systemPrompt` mirrors WhatsApp/Telegram and is the narrowest plugin-owned fix. Account-wide support should be separately scoped or given explicit precedence rules.
