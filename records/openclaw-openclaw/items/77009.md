## Summary

Keep open. Current main still has group-only BlueBubbles systemPrompt support, no direct/account prompt schema, and tests/docs explicitly show DMs omit it; the related merged PR addressed only the group portion of #60665.

## What This Changes

The issue requests BlueBubbles per-DM systemPrompt configuration, preferably via a `direct.<id>.systemPrompt` map with wildcard fallback.

## Best Possible Solution

Add a BlueBubbles `direct` prompt map with exact and `*` fallback resolution for DM turns, update docs/tests/changelog/generated config metadata, and leave account-level prompt support for a separate maintainer decision.

## Reproduction Assessment

Yes. Source inspection shows the schema lacks a BlueBubbles `direct` prompt map and the inbound DM path sets no `GroupSystemPrompt`; the existing test at `extensions/bluebubbles/src/monitor.test.ts:671` asserts that behavior.

## Solution Assessment

Yes for the direct-map part. Mirroring WhatsApp/Telegram-style `direct.<id>.systemPrompt` inside the BlueBubbles plugin is the narrowest maintainable fix; account-wide prompt support should not be bundled in unless a maintainer asks for that broader surface.
