## Summary

Keep open: current source already resolves literal per-account Telegram botToken strings, but Telegram still lacks Discord’s active runtime snapshot selection pattern for SecretRef-backed named account tokens; the public report also includes live-looking Telegram bot tokens that need rotation/redaction.

## What This Changes

The issue asks for Telegram named accounts to resolve their own bot tokens at runtime so multi-account polling does not call Telegram with an undefined token.

## Best Possible Solution

First rotate/redact the exposed Telegram tokens, then implement a Telegram-owned runtime-snapshot/token-status fix that mirrors the Discord pattern while preserving existing literal botToken behavior.

## Reproduction Assessment

Partially: source inspection gives a high-confidence reproduction path for SecretRef-backed named account tokens missing active runtime snapshot selection. I did not reproduce the exact live plain-token botundefined path, and current source appears to resolve literal account botToken strings.

## Solution Assessment

Yes for the source-proven part: the narrow maintainable fix belongs in the Telegram plugin, reusing the Discord runtime snapshot selection pattern and adding regression tests. A broad core migration or new config option is not needed.
