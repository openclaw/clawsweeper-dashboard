Full review comments:

- **[P2] Preserve numeric exec exit signals in the fingerprint:** `src/agents/tool-loop-detection.ts:234`
  - body: `ExecProcessOutcome.exitSignal` can be a string, number, or null, and foreground exec passes that value into result details. With failed output removed from the hash, `stringField(details.exitSignal)` collapses numeric signals to `null`, so different signal failures with the same status/exitCode/timedOut/failureKind can merge into one no-progress streak and trip the breaker incorrectly. Normalize non-null string or numeric signals and add coverage for that case.
