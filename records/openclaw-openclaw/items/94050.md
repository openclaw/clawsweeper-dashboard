## Summary

Keep this PR open, but do not merge it as written: it fixes the linked failed-exec hash direction only by also treating changing completed exec output as no progress, which can make active sessions hit critical loop blocking.

## What This Changes

The branch removes output text from completed and failed exec result hashes and updates loop-detection tests to expect global breaker escalation when exec output varies.

## Best Possible Solution

Split exec outcome hashing by status: keep completed output as a progress signal, strip volatile failed output, preserve focused regression tests, and require direct after-fix runtime proof before merge.
