## What This Changes

The PR changes exec loop detection so failed exec outcomes hash only status, exitCode, and timedOut while completed exec output remains part of the hash.

## Best Possible Solution

Keep the failed-output volatility fix, but include stable failed-exec fields such as failureKind and exitSignal in the hash and add regression coverage for distinct failure metadata.

## Reproduction Assessment

## Solution Assessment

No. Stripping volatile failed output is the right narrow direction, but the best implementation should preserve stable failed-exec fields instead of hashing only status, exit code, and timeout.
