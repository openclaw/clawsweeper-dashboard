## Summary

Keep open: the linked bug is still real, but this PR is not merge-ready because it strips output from completed exec hashes as well as failed exec hashes, and the supplied proof is still test/typecheck output only.

## What This Changes

The PR removes output text from completed and failed exec result hashes and updates loop-detection tests to expect critical/global escalation when exec output varies.

## Best Possible Solution

Split exec outcome hashing by status: preserve completed output as a progress signal, strip volatile failed output, update the regression tests, and require direct runtime proof.

## Reproduction Assessment

Yes, source-level. Current main hashes completed/failed exec output into `resultHash`, and `getNoProgressStreak` breaks when hashes differ; I did not run live reproduction because this review is read-only.

## Solution Assessment

No. Stripping volatile output is a valid direction for failed exec loops, but this branch also strips completed output even though current tests and docs treat changing completed output as progress.
