## Summary

Keep this PR open, but it is not merge-ready: the linked exec no-progress bug is valid, yet this branch changes completed and failed exec hashing together and can turn changing successful output into a critical/global loop block.

## What This Changes

The PR removes output text from completed and failed exec result hashes and updates loop-detection tests to expect critical/global escalation when exec output varies.

## Best Possible Solution

Split exec outcome hashing by status: keep completed output as a progress signal, strip volatile failed output, preserve focused tests, and require direct after-fix runtime proof.

## Solution Assessment

No. Stripping volatile output is a valid direction for failed exec loops, but this branch also strips completed output even though current tests and docs treat changing completed output as progress.
