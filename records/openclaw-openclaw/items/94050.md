## Summary

## What This Changes

The PR removes output text from completed and failed exec result hashes and updates loop-detection tests to expect critical/global escalation when exec output varies.

## Best Possible Solution

Split exec outcome hashing by status: keep completed output as a progress signal, strip volatile failed output, preserve focused regression tests, and require direct after-fix runtime proof before merge.

## Solution Assessment
