## What This Changes

Prevent read-only Telegram status, allowlist, and default-target metadata paths from resolving file-backed `botToken` SecretRefs while keeping runtime token resolution strict.

## Best Possible Solution

Resolve #74833 first: either close it as superseded by the v2026.4.29/current-main implementation if maintainers accept the shipped code, or land an equivalent reviewed patch, then close this issue as implemented with the release evidence.

## Reproduction Assessment

Yes. The issue gives a concrete file-backed Telegram `botToken` SecretRef config plus `openclaw status`, and current `extensions/telegram/src/shared.test.ts` covers the same metadata path by resolving `allowFrom` and `defaultTo` without touching the token.

## Solution Assessment

Yes. The config-only `resolveAccessorAccount` is the narrow maintainable fix because status metadata can read merged account config without dereferencing secrets, while `resolveTelegramToken` remains strict on runtime paths.
