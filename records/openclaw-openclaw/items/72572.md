## Summary

Keep open: current main still rejects the reported symlinked `~/.openclaw` approvals directory, and open closing PRs #72650 and #72713 remain the implementation candidates to review or consolidate.

## What This Changes

The issue asks OpenClaw to allow a trusted first-level `~/.openclaw` symlink for exec approvals while continuing to reject deeper symlink traversal.

## Best Possible Solution

Land one security-reviewed implementation that permits only a validated first-level `.openclaw` symlink under the trusted home boundary while preserving deeper traversal and destination-file guards.

## Reproduction Assessment

Yes, from source inspection. With `.openclaw` as a symlink under the resolved home directory, `openclaw exec-policy preset yolo` reaches `saveExecApprovals()`, and the current path walker throws before writing the approvals file.

## Solution Assessment

Yes in direction, but it is not implemented on current main. A narrow first-level `.openclaw` exception with target validation is safer than broadly following symlinks or realpathing the full approvals path.
