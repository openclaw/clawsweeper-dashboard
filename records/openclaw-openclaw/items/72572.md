## Summary

Keep open: current main still rejects the reported symlinked `~/.openclaw` approvals directory, and open closing PRs #72650 and #72713 are the active implementation candidates to review or consolidate.

## What This Changes

Allow `openclaw exec-policy preset yolo` to work when the first-level `~/.openclaw` directory is a trusted user-owned symlink while preserving deeper exec-approvals symlink protections.

## Best Possible Solution

Land one security-reviewed implementation that permits only a validated first-level `~/.openclaw` symlink under the trusted home boundary while keeping deeper traversal and `exec-approvals.json` destination symlinks blocked.

## Reproduction Assessment

Yes, from source inspection. `openclaw exec-policy preset yolo` reaches `saveExecApprovals()`, and current main still throws when `.openclaw` is a symlink below the resolved home directory.

## Solution Assessment

Yes in direction, but it is not implemented on current main. A narrow first-level `.openclaw` exception with target validation is safer than broadly following or realpathing the whole approvals path.
