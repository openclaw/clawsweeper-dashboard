Keep open: current main and `v2026.6.8` still reject the reported symlinked default `~/.openclaw` approvals directory, and the two linked closing PRs are still open rather than merged.

## What This Changes

The issue asks OpenClaw to allow a trusted first-level `~/.openclaw` symlink for exec approvals while continuing to reject deeper unsafe symlink traversal.

## Best Possible Solution

Land one security-reviewed fix that allows only a validated first-level `~/.openclaw` symlink while preserving deeper symlink and destination-file protections, then close this issue after merge.

## Reproduction Assessment

Yes, from source inspection. `exec-policy preset` writes approvals through `saveExecApprovals()`, and current main still rejects a symlinked `.openclaw` child by not passing the fs-safe root-child symlink option and by `lstatSync`-rejecting the approvals directory.

## Solution Assessment

Yes to keeping the issue open, but not to starting another fix PR. The best path is to review and converge the linked candidate PRs into one narrow security-reviewed implementation.
