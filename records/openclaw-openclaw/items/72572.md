## Summary

Keep open: current main still rejects the reported symlinked `~/.openclaw` approvals directory, and two open closing PRs already target the remaining fix.

## What This Changes

Allow `openclaw exec-policy preset yolo` to work when the default `~/.openclaw` directory is a trusted first-level symlink while preserving deeper exec-approvals symlink hardening.

## Best Possible Solution

Review and land one implementation that permits exactly one validated first-level `.openclaw` symlink under the home boundary while continuing to reject deeper traversal and symlinked `exec-approvals.json` writes.

## Reproduction Assessment

Yes. The source path from `exec-policy preset` to `saveExecApprovals()` still reaches the symlink-component walker, and the current store test explicitly expects `.openclaw` symlink rejection below the trusted home root.

## Solution Assessment

Yes in direction, but current main has not implemented it. The safest fix is a narrow first-level `.openclaw` exception with current-user ownership and non-group/other-writable target validation, not broad symlink following.
