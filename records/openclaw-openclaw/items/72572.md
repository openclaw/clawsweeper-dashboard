## What This Changes

The issue asks OpenClaw to support a trusted first-level `~/.openclaw` symlink for exec approvals while preserving deeper symlink hardening.

## Best Possible Solution

Land one narrow, security-reviewed current-main fix that accepts only a validated immediate `.openclaw` symlink and preserves all deeper traversal and approval-file write protections.

## Reproduction Assessment

Yes, from source inspection. The default approvals path reaches `assertNoExecApprovalsSymlinkParents()` without the root-child symlink allowance and then hits the final `lstatSync` symlink rejection before approvals can be written.

## Solution Assessment

Yes for the requested direction, but not by starting a third branch. The best path is to review, rebase, and land one existing linked PR or a narrow replacement that preserves current hardening.
