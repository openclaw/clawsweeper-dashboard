## Summary

Keep open: current main still rejects the reported symlinked `~/.openclaw` approvals directory, and two open closing PRs remain the implementation candidates.

## What This Changes

The issue requests support for a validated first-level `~/.openclaw` symlink when `openclaw exec-policy preset yolo` writes `exec-approvals.json`, while preserving deeper symlink rejection.

## Best Possible Solution

Land one security-reviewed implementation that permits only a validated first-level `.openclaw` symlink under the trusted home boundary while keeping deeper symlinks and symlinked approval-file writes blocked.

## Reproduction Assessment

Yes. Source inspection shows `openclaw exec-policy preset yolo` reaches `saveExecApprovals()`, and current main still rejects a symlinked `.openclaw` directory before writing `exec-approvals.json`.

## Solution Assessment

Yes in direction, but it is not implemented on current main. A narrow first-level `.openclaw` symlink exception with ownership and permission validation is safer than broadly following symlinks or realpathing the full approvals path.
