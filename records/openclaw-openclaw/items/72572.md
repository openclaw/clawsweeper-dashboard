## Summary

Keep open: current main still rejects the reported symlinked `~/.openclaw` approvals directory, and open closing PRs #72650 and #72713 remain the implementation candidates to review or consolidate.

## What This Changes

The issue requests that `openclaw exec-policy preset yolo` accept a validated first-level `~/.openclaw` symlink while continuing to reject unsafe deeper symlink traversal.

## Best Possible Solution

Land one security-reviewed implementation that permits only a validated first-level `.openclaw` symlink under the trusted home boundary while preserving deeper traversal and destination-file protections.

## Reproduction Assessment

Yes, from source inspection. With `.openclaw` as a symlink under the resolved home directory, `exec-policy preset yolo` reaches `saveExecApprovals()`, and the current path walker throws before writing the approvals file.

## Solution Assessment

Yes in direction, but it is not implemented on current main. A narrow first-level `.openclaw` exception with target validation is safer than broadly following symlinks or realpathing the full approvals path.
