## Summary

Keep open: the reported Slack direct outbound bug is credible, but this PR currently breaks the outbound normalization contract and does not include after-fix real behavior proof.

## What This Changes

The branch adds Slack direct outbound payload normalization for Slack button directives and a regression test for `slackPlugin.outbound.normalizePayload`.

## Best Possible Solution

Thread config and account context into the outbound normalization seam or move the gated Slack compile into an existing context-aware delivery hook, then prove a real direct Slack send renders Block Kit and add the required changelog entry.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main's direct outbound bridge only calls a plugin normalizer when the adapter exposes one, while the Slack direct adapter lacks the context-aware interactive compile used by the message/reply path. I did not run a live Slack cron delivery in this read-only review.

## Solution Assessment

No. The PR targets the right boundary, but it is not the best current implementation because the added normalizer depends on `cfg` and `accountId` fields that the core normalizePayload contract does not pass.
