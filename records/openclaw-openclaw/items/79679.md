## Summary

Keep open. Current main still defaults Telegram streaming to `partial` and short text finals can be finalized by editing the existing preview message, while `progress` mode is now an opt-in path that sends the final answer normally but does not implement the requested URL-specific or default fresh-send behavior. The Telegram maintainer notes also make the single-preview streaming contract intentional, so changing URL finals needs maintainer UX/product judgment rather than conservative cleanup.

## What This Changes

The issue requests more reliable Telegram link previews for URL-containing assistant replies by using fresh final sends, or a setting for that behavior, when partial streaming would otherwise finish through message edits.

## Best Possible Solution

Maintainers should choose the Telegram URL-final delivery policy explicitly, either documenting `progress` mode as the supported preview-card-friendly mode or adding a narrow URL-final policy while preserving `linkPreview: false` and proving the behavior in a live Telegram DM.

## Reproduction Assessment

No high-confidence live reproduction of Telegram preview-card loss was established. Source inspection does show the relevant current-main path: default/partial Telegram streaming can introduce the final URL through `editMessageText` rather than a fresh `sendMessage`.

## Solution Assessment

Unclear. Fresh-sending URL-containing partial finals is plausible, but the current codebase has an intentional single-preview streaming contract and an existing `progress` mode workaround, so maintainers should choose the policy before implementation.
