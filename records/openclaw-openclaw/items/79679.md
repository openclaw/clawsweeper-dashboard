## Summary

Keep open. Current main still has the reported Telegram partial-stream path where short final text replies are finalized through the edited preview message, and changing URL-containing finals to fresh sends is a Telegram delivery policy/product decision rather than a conservative cleanup close.

## What This Changes

The issue requests more reliable Telegram link previews for assistant replies by avoiding edited streaming finalization for URL-containing messages or adding a URL-final delivery mode.

## Best Possible Solution

Add a Telegram-owned URL-final delivery policy, documented and tested, that preserves partial streaming for ordinary text while using fresh final delivery for URL replies when previews are enabled.

## Reproduction Assessment

No, not as a live end-to-end failure. Source inspection gives a clear current-main path where partial-mode short finals are edited into the preview message, but there is no live Telegram proof here that representative URLs fail to receive preview cards on current main.

## Solution Assessment

Unclear. Fresh-send for URL-containing finals is plausible, but maintainers should decide whether this should be a default heuristic, an opt-in setting such as a URL-final mode, or documentation around using progress mode for preview-sensitive workflows.
