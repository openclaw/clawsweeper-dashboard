## Summary

Keep open: current main and the latest release still finalize ordinary Telegram partial-stream text replies by editing the active preview, and the requested URL fresh-send behavior needs live Telegram proof plus maintainer approval for changing that lifecycle.

## What This Changes

The issue requests more reliable Telegram link preview cards for URL-containing assistant replies when partial streaming currently delivers the final text through edited preview messages.

## Best Possible Solution

Decide the intended Telegram URL-final policy, then implement either a documented supported workaround or a tested URL-aware final-delivery exception with live Telegram proof.

## Reproduction Assessment

No: there is not a high-confidence live reproduction from this review. Source inspection confirms the reported partial-stream edit path still exists on current main, but the missing Telegram preview card outcome needs live Telegram proof.

## Solution Assessment

Unclear: a URL-aware fresh final send is plausible, but it is not clearly the best default because progress mode already uses normal final delivery and maintainer notes intentionally preserve single-message partial streaming. Maintainers should choose the URL-final policy before implementation.
