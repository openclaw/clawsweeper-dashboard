## Summary

Keep open. Current main still uses the documented Telegram partial-streaming path that finalizes short text replies by editing the preview message in place, while the requested URL-specific fresh final send would change an intentional Telegram streaming policy and needs maintainer product judgment plus live Telegram proof.

## What This Changes

The issue asks for Telegram URL-containing final replies to reliably get link preview cards when partial streaming currently delivers them through edited preview messages.

## Best Possible Solution

Have the Telegram owners choose a URL-final policy, then either document the existing progress/off workaround or implement an approved opt-in/default mode with focused tests and real Telegram proof.

## Reproduction Assessment

No live high-confidence reproduction was established. Source inspection confirms the reported mechanism: partial-mode text finals are finalized through an edited preview message, but the preview-card failure depends on Telegram runtime behavior that still needs live proof.

## Solution Assessment

Unclear. Fresh-sending URL-containing finals may be the right UX for previews, but it is not a narrow bug fix because it changes the current Telegram one-message streaming contract; the safer path is maintainer product review first.
