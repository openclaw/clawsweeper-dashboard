## Summary

Keep open: current main still finalizes text-only Telegram partial-stream replies through the preview/edit path, and there is no URL-aware fresh-send policy for link-preview reliability. The source path is clear, but the reported Telegram card failure still needs live proof and a maintainer UX decision before changing final delivery semantics.

## What This Changes

Request a Telegram streaming delivery policy so final assistant replies containing URLs can reliably produce link preview cards when previews are enabled.

## Best Possible Solution

Define a Telegram-owned URL-finalization policy, likely a fresh final send for URL-containing text finals when previews are enabled, with focused regression coverage and live Telegram proof.

## Reproduction Assessment

No, not as a high-confidence live reproduction from this review. Source inspection shows the reported partial-stream path exists on current main, but Telegram preview-card generation needs a real bot/client proof with crawlable URLs.

## Solution Assessment

Unclear. Fresh-sending URL-containing finals is plausible, but it changes Telegram streaming lifecycle behavior enough that maintainers should choose the policy before implementation.
