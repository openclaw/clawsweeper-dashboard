## Summary

Keep open. Current main still has the reported Telegram partial-stream path: short text finals are finalized by editing the live preview message, while the more reliable fresh-send behavior exists only for normal/progress delivery and needs a maintainer decision before changing defaults or adding a URL-specific mode.

## What This Changes

Request a Telegram delivery policy that sends URL-containing final assistant replies as fresh messages when link previews are enabled instead of finalizing partial streaming through edited preview messages.

## Best Possible Solution

Add a Telegram-owned, tested URL-final delivery policy or documented mode that preserves partial streaming for ordinary text while using fresh sendMessage for URL finals when previews are enabled, with docs and changelog coverage.

## Reproduction Assessment

No, not as a live end-to-end failure. Source inspection gives a high-confidence path for partial mode using edited preview messages, but there is no live Telegram bot/crawl proof here that representative URLs fail to receive cards on current main.

## Solution Assessment

Unclear. Fresh-send for URL-containing finals is a plausible implementation direction, but maintainers should first choose whether this should be a new opt-in setting, a default heuristic, or documentation around the existing progress-mode workaround.
