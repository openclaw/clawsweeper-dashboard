## Summary

This should stay open: current main still finalizes normal Telegram partial-stream text replies by editing the active preview message, and I did not find a current-main URL-aware fresh-send fallback or a canonical open replacement item for the link-preview reliability problem.

## What This Changes

The issue requests reliable Telegram link preview cards for URL-containing assistant replies when Telegram partial streaming edits are enabled.

## Best Possible Solution

Decide the intended Telegram partial-stream URL-final policy, then implement either a tested URL-aware fresh-send cleanup path or document `progress`/`off` as the supported workaround, with live Telegram proof for URL preview behavior.

## Reproduction Assessment

No; source inspection confirms the reported partial-stream edit path still exists, but I did not establish a live Telegram reproduction of preview cards failing on current main. The issue body’s runtime notes are plausible evidence, not a complete current-main repro.

## Solution Assessment

Unclear; a URL-aware fresh final send is plausible, but it is not obviously the best default because `progress` mode already sends the final answer normally and URL-specific fallback behavior changes partial-mode semantics. Maintainers should choose the product policy before a fix PR.
