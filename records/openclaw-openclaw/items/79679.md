## Summary

Keep open. Current main still finalizes normal Telegram `partial` streaming replies by editing the preview message, while the fresh-final behavior that would improve URL cards is either the broader `progress` mode workaround or a new URL-specific delivery policy that needs Telegram product/owner judgment.

## What This Changes

Request reliable Telegram link previews for URL-containing assistant replies by using a fresh final send or adding a URL-final delivery option when partial streaming is enabled.

## Best Possible Solution

Decide whether to document `progress` mode as the supported link-preview workaround or add a narrow URL-final fresh-send policy after Telegram owner approval and live Telegram proof.

## Reproduction Assessment

No. Source inspection gives a clear current-main path for partial-mode final replies being edited in place, but I did not establish a live Telegram DM/client reproduction of preview cards failing after edits.

## Solution Assessment

Unclear. Existing `progress` mode is the safest current workaround for fresh final delivery, while a URL-specific fresh-send/default change in `partial` mode needs maintainer product approval because it changes intentional Telegram streaming semantics.
