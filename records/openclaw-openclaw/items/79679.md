## Summary

Keep open. Current main still finalizes Telegram partial-stream text replies by editing the preview message, so the reported link-preview reliability tradeoff remains unresolved and needs an explicit Telegram streaming product decision.

## What This Changes

Request URL-containing Telegram assistant finals to use a fresh send, or a new setting, instead of finalizing through edited partial-stream previews.

## Best Possible Solution

Decide whether partial streaming should gain a URL-aware fresh-final policy; if accepted, implement it in Telegram lane delivery with focused tests and live Telegram proof, otherwise document progress/off mode as the supported link-preview-heavy workaround.

## Reproduction Assessment

No high-confidence live reproduction in this review. Source inspection confirms partial mode finalizes text through the edited preview path, but the missing preview card itself depends on live Telegram client/crawler behavior.

## Solution Assessment

Unclear. A fresh-send URL final may address unfurl reliability, but it conflicts with the current one-preview streaming policy; progress/off mode is the existing lower-risk workaround until owners decide the product behavior.
