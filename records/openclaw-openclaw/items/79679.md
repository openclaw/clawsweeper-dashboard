## Summary

Keep open. Current main still finalizes Telegram partial-stream text by editing the preview message in place, while link-preview config only controls disabling previews; making URL-containing finals switch to a fresh send is a new Telegram delivery policy with UX and security tradeoffs.

## What This Changes

Request reliable Telegram link preview cards for URL-containing assistant replies when partial streaming currently introduces the final URL through edited preview messages.

## Best Possible Solution

Have maintainers choose an explicit Telegram URL-finalization policy, then implement it narrowly in the Telegram delivery layer with docs and regression tests, while preserving `progress` mode as the existing fresh-final workaround and accounting for URL preview security tradeoffs.

## Reproduction Assessment

No high-confidence live reproduction yet. Source inspection does confirm the reported OpenClaw path: partial-mode text finals reuse the active Telegram preview via edit rather than always issuing a fresh `sendMessage`.

## Solution Assessment

Unclear as an automatic fix. A fresh-send URL final mode may be the right solution, but the default behavior versus opt-in setting needs maintainer judgment because it changes Telegram streaming UX and URL-preview security posture.
