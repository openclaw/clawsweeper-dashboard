## Summary

Keep open. Current main still documents and implements Telegram partial-stream finals as in-place edited preview messages, while URL-specific fresh final delivery is not present and would need a Telegram UX/product decision.

## What This Changes

The issue requests more reliable Telegram link-preview delivery for URL-containing assistant replies when partial streaming finalizes through edited messages.

## Best Possible Solution

Add a Telegram-owned URL-final delivery policy, likely opt-in or explicitly documented, that uses fresh sendMessage for URL final replies when previews are enabled while preserving partial streaming for ordinary text.

## Reproduction Assessment

No, not for the live Telegram preview-card failure. Source inspection gives a high-confidence path for partial mode finalizing short URL-containing text through edited preview messages, but there is no live Telegram bot/crawler proof here that representative URLs fail on current main.

## Solution Assessment

Unclear. Fresh-send for URL-containing finals is plausible, but maintainers should first choose whether this should be a default heuristic, a new setting, or documentation around the existing progress-mode workaround.
