## Summary

Keep open. Current main still finalizes partial Telegram assistant text by editing the preview message, while the requested URL-specific fresh-send behavior is not implemented and would change an intentional Telegram streaming policy.

## What This Changes

Request reliable Telegram link-preview cards for URL-containing assistant replies when partial streaming finalizes through edited preview messages.

## Best Possible Solution

Decide the Telegram UX policy, then either document `streaming.mode: "progress"` as the supported fresh-final workaround or add a narrowly scoped URL-final delivery option backed by real Telegram proof.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established. Source inspection does confirm that partial-mode final URL text is delivered through the edited preview path, but the actual Telegram preview-card reliability needs live Telegram proof.

## Solution Assessment

Unclear. A URL-specific fresh-send mode may solve the observed preview issue, but it conflicts with the current intentional one-message preview streaming behavior; `progress` mode is the existing fresh-final alternative.
