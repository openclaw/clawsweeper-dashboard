## What This Changes

The issue requests more reliable Telegram link previews for URL-containing assistant replies when partial streaming finalizes replies through edited preview messages.

## Best Possible Solution

Decide the intended Telegram URL-final policy, then implement a tested URL-aware fresh-send cleanup path or document progress/off as the supported workaround, with live Telegram proof.

## Solution Assessment

Unclear; a URL-aware fresh final send is plausible, but it is not obviously the best default because progress mode already uses normal final delivery and a URL-specific fallback changes partial-mode semantics. Maintainers should choose the policy before implementation.
