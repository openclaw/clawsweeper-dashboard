## Summary

Keep open. Current main still uses the Telegram partial-stream edited-preview path for short text finals, and the requested reliable link-preview behavior would change Telegram delivery policy or add a URL-final mode rather than close as already implemented.

## What This Changes

Make Telegram partial-streamed assistant replies with URLs produce reliable Telegram link preview cards, potentially by sending URL-containing finals as fresh messages or adding a URL-final delivery mode.

## Best Possible Solution

Add a Telegram-owned, tested URL-final delivery policy that preserves partial streaming for ordinary text while using fresh sendMessage for URL finals when previews are enabled.

## Reproduction Assessment

No, not for the live link-card failure. Source inspection gives a high-confidence current-main path where partial-mode text finals are introduced through an edited preview message, but there is no live Telegram crawl/card proof here.

## Solution Assessment

Unclear. Fresh-send URL finals are a plausible fix, but maintainers should choose between a default heuristic, an opt-in config mode, or documentation around progress/off modes before implementation.
