## Summary

Keep open. Current main still has the reported Telegram partial-streaming shape: link previews default to enabled, but ordinary text finals in partial mode are finalized through the edited preview message, and the requested URL-specific fresh-send behavior is neither implemented nor owned by a duplicate issue or PR.

## What This Changes

The issue asks OpenClaw to make Telegram URL-containing final replies use a preview-card-friendly delivery path when partial streaming currently finalizes via edited messages.

## Best Possible Solution

Have the Telegram owner decide a scoped URL-aware finalization policy, backed by live Telegram proof, that preserves one-message streaming for normal text while using fresh final delivery only when it materially improves enabled link previews.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection confirms the relevant current-main path introduces ordinary partial-mode final text through editMessageText rather than a fresh sendMessage, but the Telegram preview-card failure itself was not live-verified.

## Solution Assessment

Unclear. A fresh-send URL final path may solve the UX, but it conflicts with the recorded Telegram streaming policy to avoid extra final bubbles unless edits fail, so the best fix needs owner/product approval before implementation.
