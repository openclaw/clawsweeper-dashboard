## Summary

Keep open. Current main confirms the reported Telegram partial-stream path finalizes short text replies by editing the preview message, while reliable URL-card delivery through a fresh final send would change the documented Telegram streaming policy and likely needs owner/product approval plus live Telegram proof.

## What This Changes

The issue asks for Telegram URL-containing assistant finals to avoid edit-only preview finalization, either by fresh-sending those finals or by adding a URL-final delivery setting.

## Best Possible Solution

Have Telegram owners decide the URL-final policy: likely keep partial streaming edit-in-place by default, document `progress` or `off` as the fresh-final workaround, and only add a URL-aware final-send option with live Telegram proof and focused regression tests.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection does confirm that partial-mode text finals are produced through an edited preview message, but the missing Telegram card depends on live Telegram crawl/edit behavior and URL-specific conditions.

## Solution Assessment

No, not as an automatic cleanup fix. The suggested fresh-send behavior is plausible, but it changes the intentionally documented one-preview Telegram streaming policy and should be decided by the channel owners before implementation.
