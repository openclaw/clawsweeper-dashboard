## Summary

Keep open: current main still intentionally finalizes ordinary Telegram partial-stream answer text through the preview edit path, and there is no URL-aware fresh-send policy or option that would make preview-card generation more reliable.

## What This Changes

Requested Telegram delivery behavior that makes link-preview cards reliable for URL-containing final assistant replies when partial streaming uses edited preview messages.

## Best Possible Solution

Maintainers should decide the Telegram URL-final policy, then either document a supported workaround such as progress/off mode or implement a Telegram-owned URL-aware final-delivery exception with focused tests and redacted live Telegram proof.

## Reproduction Assessment

No high-confidence live reproduction is available in this pass. Source inspection confirms current main can introduce final URL text through an edited partial-stream preview, but the missing preview-card outcome depends on Telegram crawl and client behavior.

## Solution Assessment

Unclear: URL-aware fresh final send is plausible, but it is not clearly the best default because maintainer notes intentionally preserve one visible preview message and `progress` mode already sends the final answer as a normal message.
