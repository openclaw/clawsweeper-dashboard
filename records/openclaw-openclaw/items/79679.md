## Summary

Keep open: current main and the latest release still finalize ordinary Telegram partial-stream text replies by editing the active preview, while the requested URL fresh-send/link-preview behavior is not implemented and needs live Telegram proof plus maintainer approval because it changes an intentional one-preview-message lifecycle.

## What This Changes

The issue requests reliable Telegram link-preview cards for URL-containing assistant replies when partial streaming finalizes by editing preview messages.

## Best Possible Solution

Decide the intended Telegram URL-final policy, then implement either a documented supported workaround such as progress mode or a tested URL-aware final-delivery exception with live Telegram proof.

## Reproduction Assessment

No high-confidence live reproduction yet. Source inspection confirms the partial-stream edit path still exists, but the missing Telegram preview-card outcome needs live Telegram proof because URL crawlability and client behavior can vary.

## Solution Assessment

Unclear: URL-aware fresh final send is plausible, but it is not clearly the best default because progress mode already uses normal final delivery and maintainer notes intentionally preserve single-message partial streaming.
