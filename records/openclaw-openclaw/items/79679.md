## Summary

Keep open. Current main still routes short Telegram partial-stream final answers through the existing edited preview message, while the requested URL-specific fresh-send behavior is not implemented and needs a maintainer product decision before changing Telegram delivery semantics.

## What This Changes

The issue requests more reliable Telegram link previews for URL-containing streamed assistant replies, likely by sending final URL replies as fresh messages instead of relying on edited preview messages.

## Best Possible Solution

Maintainers should decide a Telegram URL-final delivery policy, then implement it narrowly in the draft/finalization path while preserving linkPreview: false and proving the real Telegram preview behavior.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection does confirm that partial-mode URL finals can be finalized through an edited Telegram preview message instead of a fresh sendMessage.

## Solution Assessment

Unclear. Fresh-sending URL-containing finals is plausible, but the safest design likely needs maintainer choice between changing partial-mode defaults, adding an opt-in setting, or recommending existing progress/off modes for fresh final delivery.
