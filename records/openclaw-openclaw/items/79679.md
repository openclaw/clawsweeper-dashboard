## Summary

Keep open for Telegram owner/product follow-up: current main still finalizes partial-streamed text by editing the preview message, and the requested URL-sensitive fresh-send behavior is not implemented and would change an intentional one-visible-message streaming policy.

## What This Changes

Request a Telegram delivery mode that sends URL-containing final assistant replies as fresh messages so Telegram link preview cards render more reliably.

## Best Possible Solution

Have Telegram owners decide whether to add a Telegram-local opt-in URL final mode, or document/use progress/off mode as the supported way to preserve fresh final sends for previews.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection confirms the reported precondition: partial-mode finals can be introduced through an edited Telegram preview, but I did not run a real Telegram probe proving card generation failure on current main.

## Solution Assessment

Unclear. Fresh-sending URL finals is plausible, but it conflicts with the current intentional one-preview-message policy; the narrower maintainable path needs Telegram owner approval for an opt-in URL final mode or a documented progress/off workaround.
