## Summary

Keep open. Current main still finalizes short Telegram partial-stream text replies by editing the preview message, docs and tests lock that behavior, and a URL-specific fresh-send mode would change the recorded Telegram streaming policy.

## What This Changes

The issue requests reliable Telegram link previews for URL-containing final assistant replies when partial streaming currently introduces the URL through an edited preview message.

## Best Possible Solution

Live-prove the Telegram client behavior, then either document/use progress mode for URL-heavy workflows or add an owner-approved Telegram streaming policy for URL finals that preserves the default one-message edit behavior.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection proves URL-containing final text can be finalized through editMessageText with link previews enabled, but the Telegram preview-card failure itself still needs live proof on current main.

## Solution Assessment

Unclear. Fresh-send-on-URL may improve preview reliability, but it conflicts with the current Telegram streaming decision unless maintainers explicitly approve a new mode or default exception.
