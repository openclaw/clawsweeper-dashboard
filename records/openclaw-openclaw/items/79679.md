## Summary

Keep open: current main and v2026.6.8 still finalize ordinary Telegram partial-stream text replies through the preview edit path, and there is no URL-aware fresh-send policy for link-preview reliability. The remaining work needs live Telegram proof plus maintainer direction because it would change the intentional one-preview lifecycle or add new delivery policy.

## What This Changes

The issue requests reliable Telegram preview cards for URL-containing assistant replies when partial streaming currently finalizes by editing a preview message.

## Best Possible Solution

Decide the intended Telegram URL-final policy, then either document an existing supported workaround such as progress/off mode or implement a Telegram-owned URL-aware final-delivery exception with focused tests and redacted live Telegram proof.

## Reproduction Assessment

No high-confidence live reproduction is available in this pass. Source inspection confirms the partial-stream edit-finalization path still exists, but the missing Telegram preview-card outcome needs live Telegram proof because URL crawlability and client behavior can vary.

## Solution Assessment

Unclear: URL-aware fresh final send is plausible, but it is not clearly the best default because progress mode already uses normal final delivery and maintainer notes intentionally preserve single-message partial streaming.
