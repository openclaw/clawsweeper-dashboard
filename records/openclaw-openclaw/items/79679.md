## Summary

Keep open. Current main still finalizes partial-streamed Telegram text replies by editing the preview message, while the requested URL-specific fresh final delivery is a visible Telegram UX/product decision rather than a conservative cleanup close.

## What This Changes

The issue requests more reliable Telegram link preview cards for URL-containing assistant replies when Telegram partial streaming finalizes messages via edits.

## Best Possible Solution

Decide at the Telegram product boundary whether link-preview-sensitive replies should use documented progress mode, an opt-in URL-final delivery mode, or a live-proven edit-path improvement.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection confirms the reported partial-streaming path finalizes text by editing the preview, but I did not run a live Telegram proof that preview cards fail on current main.

## Solution Assessment

Unclear. Fresh-sending URL finals might improve previews, but it changes Telegram's maintained one-message streaming behavior; the safer path is maintainer direction backed by live Telegram proof.
