## Summary

Keep open. Current main still finalizes short Telegram partial-stream assistant replies by editing the preview message in place, and changing URL-containing finals to fresh sends or adding a URL-specific mode needs Telegram maintainer product judgment.

## What This Changes

Request reliable Telegram link preview cards for URL-containing assistant replies when Telegram partial streaming is enabled.

## Best Possible Solution

Add a Telegram-owned URL-final delivery policy or documented mode that preserves partial streaming for ordinary text while using fresh sendMessage for URL finals when previews are enabled.

## Reproduction Assessment

No, not as a live end-to-end Telegram preview-card failure. Source inspection gives a high-confidence path for partial mode editing final URL text into an existing preview message, but there is no live Telegram bot/crawler proof here that representative URLs fail on current main.

## Solution Assessment

Unclear. Fresh-send for URL-containing finals is plausible, but maintainers should choose whether this should be an opt-in setting, a default heuristic, or documentation around progress/off streaming modes.
