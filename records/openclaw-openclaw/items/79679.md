## Summary

Keep open: current main still documents and implements partial-mode finalization through an edited Telegram preview message, while URL-specific fresh-send behavior would change an intentional Telegram streaming policy and has no merged fix or canonical duplicate.

## What This Changes

Requests a Telegram delivery policy or setting that makes URL-containing final replies use fresh delivery, or otherwise improves link preview reliability, when partial streaming is enabled.

## Best Possible Solution

Have the Telegram owner choose and document a URL-final delivery policy, likely an opt-in mode or carefully scoped exception that preserves partial streaming for ordinary text while fresh-sending URL finals when previews are enabled.

## Reproduction Assessment

No high-confidence live reproduction of missing Telegram preview cards was established. Source inspection does give a clear current-main path where partial-mode final text is finalized through editMessageText instead of a fresh sendMessage.

## Solution Assessment

Unclear. Fresh-send for URL-containing finals is a plausible fix direction, but it conflicts with the current one-preview-message streaming policy unless maintainers approve a default exception or new URL-final mode.
