## What This Changes

The branch adds optional provenance metadata to embedded context files, tags bootstrap context entries, renders provenance comments in the system prompt, and adds prompt/bootstrap tests.

## Best Possible Solution

If maintainers accept Phase 1, rebase it into `buildProjectContextSection()`, preserve stable/dynamic cache semantics and prompt sanitization, normalize all rendered metadata, and leave memory freshness plus orchestration refresh as follow-ups.

## Reproduction Assessment

Yes for the PR-specific blockers: current-main source shows project context now flows through `buildProjectContextSection()` with a stable/dynamic cache split, while the PR head renders provenance in the old inline loop and interpolates `volatile` raw.

## Solution Assessment

No: the direction may be useful, but this branch is not the best merge shape until it targets the current prompt helper/cache design, validates all rendered metadata, and receives maintainer agreement on the RFC boundary.
