## Summary

Keep this PR open: current main does not implement context-file provenance, and the PR head is still dirty against the current prompt-cache/context rendering architecture with one prompt-integrity gap and a missing required changelog entry.

## What This Changes

The PR adds optional `EmbeddedContextFile.provenance`, tags bootstrap context files, renders `ctx:provenance` prompt comments, and adds provenance guidance/tests.

## Best Possible Solution

If maintainers accept Phase 1, rebase it onto the shared project-context helper, cover both stable and dynamic context rendering, normalize all rendered provenance attributes, add focused tests and the required changelog entry, and leave the broader RFC open for later phases.

## Reproduction Assessment

Yes for the PR-specific blockers: current main and PR-head source show the stale inline rendering path, dirty merge state, and raw `volatile` interpolation. No runtime test was needed to identify those source-level issues.

## Solution Assessment

No: the feature direction may be useful, but this branch is not the best merge shape until it is rebased through current prompt helpers, validates/coerces provenance metadata at render time, and has maintainer agreement on the RFC boundary.
