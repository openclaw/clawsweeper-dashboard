## What This Changes

The PR adds optional provenance metadata to `EmbeddedContextFile`, tags bootstrap context files, renders `ctx:provenance` comments and guidance in the system prompt, and adds focused tests.

## Best Possible Solution

If maintainers accept Phase 1, rebase the implementation onto `buildProjectContextSection()`, preserve provenance across stable and dynamic context, normalize rendered metadata, add focused prompt-shape coverage, and keep later memory/auto-refresh phases in separate follow-ups.

## Solution Assessment
