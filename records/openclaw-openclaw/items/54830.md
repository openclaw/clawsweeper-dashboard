## Summary

Keep open: current main still lacks context provenance metadata, and this PR remains the active implementation candidate, but it needs maintainer RFC/API judgment, a rebase onto the current prompt-cache structure, contributor real-behavior proof, and two source fixes before merge.

## What This Changes

The PR extends embedded context files with optional provenance metadata, tags bootstrap context entries, renders `ctx:provenance` comments in the system prompt, and adds prompt/bootstrap tests.

## Best Possible Solution

If maintainers accept Phase 1, rebase it into `buildProjectContextSection()`, preserve stable/dynamic cache semantics and prompt sanitization, normalize all rendered metadata, and split memory freshness plus orchestration refresh into follow-ups.

Yes from source inspection for the PR-specific blockers: current main renders project context through `buildProjectContextSection()` with a stable/dynamic cache split, while the PR diff renders provenance in the old inline loop and leaves `volatile` unnormalized. I did not run a live prompt assembly proof.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Move provenance rendering into the project-context helper:** `src/agents/system-prompt.ts:625-627`
  - body: The branch adds tags in the old inline context-file loop, but current main renders both stable and dynamic project context through `buildProjectContextSection()` around the cache boundary.
