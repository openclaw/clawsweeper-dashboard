## Summary

Keep open: current main still lacks context provenance metadata, and this PR is the active implementation candidate, but it needs a rebase into the current prompt-context helper, real behavior proof, and maintainer agreement on the linked RFC/API direction before merge.

## What This Changes

The branch adds optional `provenance` metadata to `EmbeddedContextFile`, tags bootstrap context files, renders `ctx:provenance` comments in the system prompt, and adds prompt/bootstrap tests.

## Best Possible Solution

If maintainers accept Phase 1, rebase the implementation into `buildProjectContextSection()`, preserve stable/dynamic cache semantics and prompt sanitization, validate rendered metadata, and split memory freshness plus orchestration refresh into follow-ups.

## Reproduction Assessment

Yes, for the PR-specific blockers: source inspection shows current main renders project context through `buildProjectContextSection()` with a cache split, while the PR diff modifies the old inline loop and leaves one rendered metadata field unnormalized. I did not run live prompt assembly.

## Solution Assessment

No: the feature direction may be useful, but this branch is not the best merge shape until it targets the current helper/cache design, normalizes all rendered metadata, and receives maintainer agreement on the RFC boundary.
