## Summary

Keep this PR open. Current main still lacks context provenance metadata, but the PR is not merge-ready because it targets an older prompt assembly layout and still leaves a runtime prompt-integrity gap in the rendered provenance tag.

## What This Changes

The PR adds optional `EmbeddedContextFile` provenance metadata, attaches session-start/volatile provenance to bootstrap context files, renders `ctx:provenance` comments and guidance in the agent system prompt, and adds focused tests.

## Best Possible Solution

Rebase the provenance work onto the current `buildProjectContextSection()` layout, validate all rendered provenance fields at runtime, cover both stable and dynamic context files in tests, and keep #54373 as the design anchor until maintainers accept or reject Phase 1.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes: current main can be checked by inspecting the flat `EmbeddedContextFile` type and path/content-only bootstrap and prompt rendering, and the PR defect is visible at the raw `volatile` interpolation line on the PR head.

## Solution Assessment

Is this the best way to solve the issue? No: the concept is plausible, but this branch is not the best merge shape until it is rebased onto current prompt helpers and the provenance renderer coerces or validates every attribute value.
