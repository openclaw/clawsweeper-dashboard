## Summary

Keep open: current main still lacks the requested context-provenance metadata, and this PR remains a plausible Phase 1 implementation for the linked RFC, but it cannot merge as-is because it is stale against the current project-context helper/cache split, still has an unnormalized prompt metadata field, and lacks real behavior proof.

## What This Changes

This PR extends `EmbeddedContextFile` with optional provenance metadata, tags bootstrap context entries, renders `ctx:provenance` comments and guidance in the agent system prompt, and adds focused prompt/bootstrap tests.

## Best Possible Solution

If maintainers accept Phase 1, rebase onto `buildProjectContextSection()`, preserve the stable/dynamic cache semantics and context sanitization, normalize provenance at render time, and leave memory freshness plus auto-refresh phases as separate follow-ups.
