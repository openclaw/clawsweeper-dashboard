Keep open. Current main still lacks this context-provenance feature, but this PR is dirty against current main and leaves a prompt-rendered metadata validation gap before it can merge.

## What This Changes

The PR adds optional `EmbeddedContextFile.provenance`, tags bootstrap context files, renders `ctx:provenance` comments/guidance in the system prompt, and adds focused tests.

## Best Possible Solution

Rebase the branch onto current stable/dynamic context rendering, validate all provenance fields at prompt-render time, add the required changelog/test coverage, and keep #54373 as the broader design anchor.
