## Summary

Keep open: current main still lacks context provenance metadata and this PR remains the open Phase 1 implementation candidate, but it is currently not mergeable because it targets an older prompt rendering shape, has a prompt-comment validation issue, and lacks real behavior proof.

## What This Changes

The PR adds optional provenance metadata to embedded context files, attaches it to bootstrap context assembly, renders `ctx:provenance` prompt comments/guidance, and adds focused bootstrap and prompt tests.

## Best Possible Solution

If maintainers accept Phase 1, rebase it into `buildProjectContextSection()`, preserve stable/dynamic cache semantics, validate or coerce every rendered metadata field, add focused tests, and leave memory freshness plus deterministic refresh to narrower RFC follow-ups.

## Reproduction Assessment

Yes for the PR-specific blockers: source inspection shows current main renders context through `buildProjectContextSection()` with stable/dynamic cache splitting, while the PR modifies the old inline loop and leaves `volatile` unvalidated. No live prompt assembly was run.

## Solution Assessment

No. The feature direction may be useful, but this branch is not the best merge shape until it targets the current helper/cache design, validates rendered metadata, and has maintainer agreement on the RFC boundary.
