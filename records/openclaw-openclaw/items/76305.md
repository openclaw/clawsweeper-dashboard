Keep open. Current main still has the sanitizer starvation path, and the issue is now paired with open PR #76324 using closing syntax, so the right cleanup point is after that PR is reviewed and merged or rejected.

## What This Changes

OpenAI Responses SSE sanitizer should preserve valid split SSE frames so embedded direct OpenAI API-key runs stream normally without a bypass.

## Best Possible Solution

Review and land the linked focused sanitizer PR, or replace it with an equivalent narrow fix plus regression coverage for split readable SSE frames while preserving malformed-frame filtering.

## Reproduction Assessment

Yes. The reporter provided a live bypass isolation, and a source-level Web Streams check against the current sanitizer logic leaves the sanitized read pending when a valid SSE frame is split before the first boundary.

## Solution Assessment

Yes. Fixing the sanitizer pull loop and adding regression coverage is the narrow maintainable solution; a permanent bypass would only avoid the broken wrapper and lose the malformed-frame guard.
