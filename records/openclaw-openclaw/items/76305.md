## Summary

Keep open. Current main still contains the sanitizer behavior that can return from `pull()` without emitting a chunk for split-but-valid SSE frames, and open PR #76324 is the linked fix candidate, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that OpenAI Responses embedded streams hang because `sanitizeOpenAISdkSseResponse()` can starve split SSE frames, while bypassing the sanitizer restores normal output.

## Best Possible Solution

Review and land PR #76324, or an equivalent focused sanitizer change with regression coverage, then close this issue through the merged fix.

## Reproduction Assessment

Yes, at source level. The issue includes live bypass evidence, and a read-only Web Streams check against the current sanitizer logic reproduces a pending read when the first chunk lacks a complete SSE boundary.

## Solution Assessment

Yes. The narrow maintainable fix is to make the sanitizer keep pulling until it can emit a complete readable SSE block or reaches EOF; a permanent bypass would only hide the broken wrapper behavior.
