## Summary

Keep open. Current main still has the one-read SSE sanitizer path that can return from `pull()` without emitting a chunk, and the issue is already paired with open fix PR #76324, so it should close only after that PR or an equivalent fix lands.

## What This Changes

The issue reports that OpenAI Responses SSE sanitization can starve split event-stream frames, causing embedded OpenAI runs to hang until timeout.

## Best Possible Solution

Land #76324 or an equivalent maintainer fix that keeps draining upstream SSE until a complete readable event can be emitted, while preserving malformed-frame filtering and adding split-frame regression coverage.

## Reproduction Assessment

Yes. The reporter and a second commenter provided live and minimal Web Streams reproductions, and current main still has the one-read `pull()` implementation that can return without enqueueing before a complete SSE boundary exists.

## Solution Assessment

Yes. Fixing `sanitizeOpenAISdkSseResponse()` to continue reading until it can enqueue a complete readable frame or the upstream closes is the narrow maintainable solution; a permanent bypass would remove the malformed-frame guard instead of repairing it.
