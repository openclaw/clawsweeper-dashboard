## Summary

Keep open. Current main still has the sanitizer pull path that can return without emitting a complete split SSE frame, and this issue is already paired with open fix PR #76324, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that OpenAI Responses SSE sanitization can starve split embedded streams, causing OpenAI embedded runs to hang until timeout unless the sanitizer is bypassed.

## Best Possible Solution

Land #76324 or an equivalent sanitizer fix that keeps draining upstream SSE until a complete readable frame can be emitted while preserving malformed-frame filtering and adding split-frame regression coverage.

## Reproduction Assessment

Yes. The reporter provided a live bypass isolation, and current-main source inspection plus an in-memory Web Streams check reproduce the pending read on a split valid SSE frame.

## Solution Assessment

Yes. Fixing the sanitizer pull/drain behavior in `src/agents/provider-transport-fetch.ts` is the narrow maintainable solution; a permanent bypass would avoid the broken wrapper instead of preserving filtering safely.
