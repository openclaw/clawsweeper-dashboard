## Summary

Keep open. Current main still has the sanitizer starvation path, and open PR #76324 already targets this issue with closing syntax, so the issue should track that review until a fix lands.

## What This Changes

The issue reports embedded OpenAI Responses streams stalling because the SSE sanitizer can stop after partial chunks before the OpenAI SDK sees valid downstream events.

## Best Possible Solution

Land #76324 or an equivalent fix that keeps draining upstream SSE until a complete readable frame can be emitted, while preserving malformed-frame filtering and adding split-frame regression coverage.

## Reproduction Assessment

Yes. The reporter provided a live bypass isolation, and a current-main Web Streams probe mirroring the sanitizer reproduces the pending read on a split valid SSE frame.

## Solution Assessment

Yes. Fixing the sanitizer drain behavior in `src/agents/provider-transport-fetch.ts` is the narrow maintainable solution; a permanent bypass would only avoid the broken wrapper.
