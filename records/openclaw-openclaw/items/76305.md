## Summary

Keep open. Current main still has the one-shot SSE sanitizer pull path that can return without emitting bytes for split valid frames, and the issue is already paired with open fix PR https://github.com/openclaw/openclaw/pull/76324, so it should close only after that PR or an equivalent fix lands.

## What This Changes

Embedded OpenAI Responses streams can stall when the SSE sanitizer receives a valid SSE event split across chunks, while bypassing the sanitizer restores normal output.

## Best Possible Solution

Land or rebase https://github.com/openclaw/openclaw/pull/76324, or an equivalent current-main fix, so the sanitizer drains until a complete readable SSE block is emitted while preserving malformed-frame filtering and regression coverage.

## Reproduction Assessment

Yes. The reporter provided live bypass/recovery evidence, another user confirmed the same failure on 2026.5.7, and a source-level Node Web Streams check mirroring current main timed out on a split readable SSE frame.

## Solution Assessment

Yes. Fixing the pull/drain behavior in `src/agents/provider-transport-fetch.ts` is the narrow maintainable path; a permanent sanitizer bypass would avoid the bug by removing the protective filtering layer.
