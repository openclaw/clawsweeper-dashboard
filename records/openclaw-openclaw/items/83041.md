## What This Changes

The PR adds optional `requiresRestart` restart-sentinel metadata for config apply/patch writes, formats pending/completed notices from it, and adds gateway/infra regression coverage for hot-reload and legacy-sentinel cases.

## Best Possible Solution

Land this PR after exact-head required checks pass, then close the linked issue and treat https://github.com/openclaw/openclaw/pull/93863 as superseded if this lands.

## Reproduction Assessment

Yes. Current main source still builds successful config-patch sentinel summaries from kind/status/mode without a restart-required discriminator, so a `config-patch` ok payload produces the misleading notice; I did not run a live gateway repro in this read-only review.

## Solution Assessment

Yes. Carrying the reload-plan restart requirement into the sentinel payload is the narrow maintainable fix; a formatter-only rewrite would mislabel hot-reloaded config writes and miss completed-startup wording.
