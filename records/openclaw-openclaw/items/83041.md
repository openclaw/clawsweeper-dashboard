## What This Changes

The PR adds an optional restart-sentinel `requiresRestart` stat, derives config.apply/config.patch restart wording from the gateway reload plan, formats startup-consumed sentinels as completed, and adds focused regression tests.

## Best Possible Solution

Land this PR after exact-head required checks pass, then close https://github.com/openclaw/openclaw/issues/46797 and treat https://github.com/openclaw/openclaw/pull/93863 as superseded if it has no unique remainder.

## Reproduction Assessment

Yes. Source inspection on current main shows config.patch writes serialize only mode/root and the formatter builds the reported ok-style summary from kind/status/mode; no local tests were run in this read-only review.

## Solution Assessment

Yes. Carrying the reload-plan restart requirement into the sentinel payload is the maintainable fix because the formatter needs state to distinguish restart-required, hot-reloaded, pending, and completed config writes.
