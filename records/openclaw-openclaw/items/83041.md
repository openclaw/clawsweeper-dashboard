## What This Changes

The PR adds an optional restart-sentinel `requiresRestart` stat, derives config apply/patch sentinel wording from the reload plan, formats startup-consumed sentinels as completed, and adds focused regression tests.

## Best Possible Solution

Land this PR after exact-head required checks pass, then resolve https://github.com/openclaw/openclaw/issues/46797 and treat the narrower formatter-only PR as superseded.

## Reproduction Assessment

Yes. Source inspection on current main shows config.patch writes produce config-patch ok sentinel payloads without restart-required metadata, and the formatter renders them from kind/status/mode.

## Solution Assessment

Yes. Carrying the reload-plan restart requirement into the sentinel payload is the narrow maintainable fix; a formatter-only rewrite would mislabel hot-reloaded config writes and miss completed-startup wording.
