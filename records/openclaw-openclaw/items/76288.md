## What This Changes

The PR threads the manual `cron.run` acknowledgement `runId` through queued cron execution, finished cron events, JSONL-backed `cron.runs` entries, the Gateway protocol schema, and a gateway regression test.

## Best Possible Solution

Yes. Current main returns a manual acknowledgement id from `enqueueRun()` but does not pass it into the queued execution or preserve it through run-log parsing; the PR's added gateway assertion comparing `cron.run` and `cron.runs` is the right focused reproduction path.
