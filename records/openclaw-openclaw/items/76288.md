## Summary

Keep open. The PR now covers the user-facing cron history gap and the earlier generated/hook/changelog misses, but it also changes manual cron task ledger IDs to `manual:...`, which bypasses existing cron task recovery that only parses `cron:...` execution IDs.

## What This Changes

The PR threads manual `cron.run` acknowledgement IDs into cron execution events, JSONL-backed run history, Gateway protocol/schema outputs, plugin hook typing, generated Swift models, regression coverage, SDK baseline hashes, and the changelog.

## Best Possible Solution

Land the runId history propagation while preserving cron task recovery semantics by separating the public manual acknowledgement ID from the internal task execution ID, or by adding equivalent maintenance recovery coverage for manual IDs.

## Reproduction Assessment

Yes. Current main proves the original issue by source inspection, and the PR’s added gateway assertion is a focused reproduction for the user-facing gap; the new blocker is reproducible by simulating a manual cron task that restarts before task finalization and then enters task maintenance.

## Solution Assessment

No, not as submitted. The user-facing propagation is the right narrow direction, but reusing the manual acknowledgement ID as the task ledger run ID is not the safest implementation because it bypasses existing recovery code.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve recoverable cron task IDs:** `src/cron/service/ops.ts:512`
  - body: Passing the manual acknowledgement ID into `createRunningTaskRun` changes manual cron task records from the recoverable `cron:<jobId>:<startedAt>` form to `manual:...`. `parseCronExecutionId()` in task maintenance only recovers active cron tasks with `cron:` IDs, so a gateway restart before `tryFinishManualTaskRun` can leave the task unrecoverable and mark it lost even when cron history has a terminal entry. Keep the task ledger ID as the cron execution ID and carry the manual ID separately, or extend recovery for manual IDs.
  - confidence: 0.87
