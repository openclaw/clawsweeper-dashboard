## Summary

Keep open: current main has plugin HTTP routes, plugin agent-event subscriptions, per-run context, internal restart-drain counters, and diagnostics/liveness telemetry, but it does not provide the requested versioned busy-state/lease snapshot or a clearly sufficient public SDK activity contract for external container lifecycle controllers.

## What This Changes

The issue requests a plugin/SDK-supported busy-state and lease signal for external lifecycle controllers managing on-demand OpenClaw containers.

## Best Possible Solution

Keep this as a maintainer design item: define the smallest public, sanitized activity/lease contract plugins need, then implement the busy-state capability as an official external or explicitly approved bundled plugin with versioned logs, authenticated read-only routes, docs, and tests.

## Reproduction Assessment

Not applicable: this is a new plugin/API capability request rather than a broken documented behavior. Source inspection confirms current main lacks the requested busy-state marker, lease recommendation model, and controller-facing snapshot surface.

## Solution Assessment

Unclear as proposed: a plugin-owned implementation matches the architecture, but maintainers still need to decide the minimal public activity contract and whether this belongs in core, an official external plugin, or ClawHub.
