## Summary

Keep open: current `main` and `v2026.6.10` still lack the requested busy-state marker, lease recommendation contract, `sessions.activity` API, activity statechange event, and plugin-provided controller read paths. The request remains a product/API decision because existing Plugin SDK hooks and gateway session status are useful ingredients, not the aggregate container lease signal.

## What This Changes

The issue requests a plugin-provided busy/lease signal so external lifecycle controllers can keep OpenClaw containers warm while work is active and release them after idle grace.

## Best Possible Solution

Settle a sanitized runtime/session activity contract first, then implement the busy-state lease projection as plugin-owned code with narrow SDK additions only where current hooks are insufficient.

## Reproduction Assessment

Not applicable: this is a feature/API proposal, not a current-main bug report with steps to reproduce. Source and release searches give high-confidence evidence that the requested marker, lease actions, activity RPCs, and activity event are absent today.

## Solution Assessment

Unclear: plugin-first ownership is aligned, but implementing the lease plugin before settling the shared activity model could create a parallel source of truth. The safer shape is a canonical sanitized activity contract plus a plugin-owned lease projection.
