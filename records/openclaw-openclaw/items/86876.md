## Summary

Keep open: current main and v2026.6.8 still have partial plugin hooks and agent event streams, but no plugin-provided busy-state or lease snapshot contract; the remaining work is a product/API decision around SDK event coverage, auth/redaction, and plugin distribution.

## What This Changes

Request an official plugin/SDK-based busy-state and lease-snapshot signal for external controllers managing on-demand OpenClaw containers.

## Best Possible Solution

Define the missing sanitized activity model, likely with https://github.com/openclaw/openclaw/issues/39127 as a foundation, then keep the busy-state implementation plugin-owned unless maintainers explicitly approve core bundling and public HTTP/log contracts.

## Reproduction Assessment

## Solution Assessment

Unclear; plugin-first is aligned, but an official bundled plugin plus new HTTP/log/config surfaces is not automatically the best product shape. The safer path is to settle the shared activity API and security contract before implementing the plugin.
