## Summary

Keep open: current main and v2026.6.9 still lack the requested plugin-provided busy-state marker, lease recommendation model, and authenticated busy-state endpoints. The request remains a product/API decision because related session-activity and Plugin SDK issues overlap but do not replace the container controller-facing contract.

## What This Changes

Request for an official plugin/SDK busy-state snapshot and lease recommendation surface for external on-demand container lifecycle controllers.

## Best Possible Solution

Settle a small sanitized runtime/session activity contract first, likely alongside the per-session activity API work, then implement the busy-state lease signal as plugin-owned code with narrow SDK additions only where current hooks are insufficient.

## Reproduction Assessment

Not applicable: this is a feature/API proposal rather than a reproducible bug. Source and release searches give high-confidence evidence that the requested marker, lease contract, HTTP route, and session activity API are absent today.

## Solution Assessment

Unclear: plugin-first is aligned, but shipping an official bundled plugin plus HTTP/log/config contracts before settling the shared activity model would risk a parallel source of truth. The safer path is a canonical sanitized activity contract plus a plugin-owned lease projection.
