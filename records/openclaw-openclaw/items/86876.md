Keep open: current main and the latest release still lack the requested plugin-provided busy-state and lease snapshot contract. The existing Plugin SDK has useful building blocks, but the missing aggregate activity model, auth/redaction rules, and distribution decision need maintainer product/API review.

Request a plugin-provided busy/idle and lease recommendation signal exposed through structured logs, authenticated read-only HTTP routes, and docs for external container lifecycle controllers.

## Best Possible Solution

Settle the sanitized activity event model and route/auth contract first, then implement the busy/lease snapshot as plugin-owned functionality unless maintainers explicitly approve official bundling.
