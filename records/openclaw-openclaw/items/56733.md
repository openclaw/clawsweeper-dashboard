Keep open. ## What This Changes

The issue reports a WSL2/systemd Gateway state where the process stays active while HTTP, provider, and channel work silently time out until a manual restart.

## Best Possible Solution

Keep this as field evidence until maintainers either land and live-prove a supported systemd watchdog recovery path or isolate and fix the current-main blocking path that causes the Gateway freeze.
