## Summary

## What This Changes

The issue reports a live Gateway process whose Node event loop freezes during idle periods, causing all outbound provider and channel HTTP requests to time out until manual restart.

## Best Possible Solution

Keep this issue as the broader gateway-freeze tracker until maintainers either land a proven supervisor recovery path or narrow the remaining work to a specific reproducible event-loop-blocking trigger.

## Reproduction Assessment

No live high-confidence reproduction was run against current main. The remaining gap is source-reproducible: current in-process health snapshots and HTTP probes require the event loop to run, and generated systemd units still lack a merged watchdog heartbeat.

## Solution Assessment

Unclear. A systemd watchdog is a plausible recovery layer for Linux services, but maintainers still need to settle default supervision behavior and prove it on a real Linux/systemd service; separate root-cause fixes may still be needed for specific blocking plugins or providers.
