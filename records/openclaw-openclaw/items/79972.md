## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve the existing SDK exports:** `src/plugin-sdk/session-store-runtime.ts:3-22`
  - body: `plugin-sdk/session-store-runtime` is a documented public subpath, and current main exports helpers such as `loadSessionStore`, `resolveStorePath`, `resolveAndPersistSessionFile`, and the store mutation helpers. Replacing them with SQLite-only names will make existing third-party or bundled plugin imports fail on upgrade; keep these exports wired as compatibility/deprecated adapters while adding the new frontier/delta helpers.
  - confidence: 0.9
- **[P2] Use a durable rewrite marker for transcript cursors:** `src/config/sessions/transcript-store.sqlite.ts:582-585`
  - body: The delta cursor is invalidated only when `baseCreatedAt`, `eventCount`, or `lastSeq` move backwards, but `replaceSqliteSessionTranscriptEvents` rewrites all rows with a single `Date.now()` timestamp. A same-millisecond rewrite with the same count and last seq can therefore return `mode: "append"` with no events even though the transcript changed. Add a rewrite generation/version, stable frontier identity, or content/first-event marker that changes on every replace.
  - confidence: 0.86
