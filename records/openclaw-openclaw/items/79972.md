## What This Changes

The PR adds public SQLite transcript frontier/delta helpers to `plugin-sdk/session-store-runtime` along with transcript-store tests, API baseline updates, and broad database-first runtime refactor changes.

## Best Possible Solution

Rework the replay seam as an additive SDK change after or with the SQLite base refactor, preserving existing exports, using a durable rewrite invalidation token, documenting the public contract, and supplying redacted real runtime proof.

## Reproduction Assessment

Not applicable as a current-main bug reproduction: this is a feature/API PR. The review blockers are source-reproducible from the PR head and the current public SDK contract.

## Solution Assessment

No: the direction is plausible, but this patch is not the safest merge shape because it breaks documented SDK exports and its cursor cannot reliably detect same-session rewrites. A narrower additive seam with compatibility adapters and a durable rewrite marker is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve existing SDK exports:** `src/plugin-sdk/session-store-runtime.ts:3-22`
  - body: `plugin-sdk/session-store-runtime` is a documented public subpath, but this patch replaces exports such as `loadSessionStore`, `resolveStorePath`, `resolveAndPersistSessionFile`, and the store mutation helpers. Existing third-party or bundled plugin imports can fail on upgrade; keep these as compatibility/deprecated adapters while adding the new frontier/delta helpers.
  - confidence: 0.9
- **[P2] Use a durable rewrite marker for transcript cursors:** `src/config/sessions/transcript-store.sqlite.ts:582-585`
  - body: Delta invalidation only checks `baseCreatedAt`, `eventCount`, and `lastSeq`, while full replacement rewrites rows with one `Date.now()` timestamp. A same-millisecond rewrite with the same count and last seq can return `append` with no events even though the transcript changed; add a rewrite generation, stable frontier identity, or equivalent marker that changes on every replace.
  - confidence: 0.86
