## Summary

Keep open: current main does not implement write-time markdown code-fence linting or auto-fix, and the two linked implementation PRs were closed unmerged. This is a feature/config/policy request rather than a stale bug cleanup because the proposed default would rewrite user-authored workspace markdown.

## What This Changes

The issue requests a write-time lint and auto-fix pass for markdown code fences in workspace documents, with language inference, warn mode, and a per-write skip option.

## Best Possible Solution

Decide the product policy first, then implement a narrow markdown-code-fence utility through the relevant OpenClaw-owned workspace write paths with focused tests and docs, or add a documented write-hook seam if maintainers want this outside core.

## Reproduction Assessment

## Solution Assessment
