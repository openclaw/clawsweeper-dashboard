## Summary

Keep open: current main still lacks write-time Markdown code-fence linting or auto-fix in workspace write/edit paths, and the linked implementation attempts were closed unmerged after parser-correctness concerns. Because the request would rewrite persisted Markdown by default and add config plus per-write skip semantics, it needs maintainer product direction before a fix PR.

## What This Changes

Add write-time linting for workspace Markdown files that can auto-fence unfenced code, infer missing fence language tags, warn instead of fixing via config, and skip per write.

## Best Possible Solution

Keep workspace writes byte-preserving unless maintainers approve a parser-backed, explicitly configured Markdown lint path with regression coverage for nested fences, write/edit behavior, and upgrade safety.

## Reproduction Assessment

Not applicable; this is a request for new write-time Markdown rewriting/config behavior, not a bug in an existing documented contract. Source inspection shows current write/edit paths preserve caller content and no code-fence lint path exists.

## Solution Assessment

Unclear; a write-path helper could satisfy the request, but autofix-by-default with heuristic language inference is not clearly the safest maintainable policy. A parser-backed opt-in or warn-first mode would be safer if maintainers accept the feature.
