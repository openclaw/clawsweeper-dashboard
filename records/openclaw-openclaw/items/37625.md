## Summary

The capability is absent from current main and has no canonical replacement, but both implementation attempts closed unmerged after concrete content-corruption concerns. Keep this issue open for an explicit maintainer decision on whether OpenClaw should ever rewrite user-authored workspace Markdown automatically.

## What This Changes

Add write-time detection and automatic correction of unfenced or untagged code blocks across workspace Markdown, with warn-only configuration and a per-write bypass.

## Best Possible Solution

If maintainers approve the capability, provide an opt-in CommonMark-aware lint or formatting action with previewable changes, then consider write-path integration only after defining a canonical writer and proving false-positive safety on representative workspace documents.

## Reproduction Assessment

## Solution Assessment
