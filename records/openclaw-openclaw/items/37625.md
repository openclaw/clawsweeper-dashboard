## Summary

Keep open. Current main still lacks the requested write-time Markdown code-fence linting/auto-fix behavior: the requested utility/config/skip symbols are absent, the existing Markdown fence helper only parses existing fence spans, and the central workspace write paths still pass caller content directly to file writes. The related implementation PRs #37628 and #41207 were closed unmerged, so they are prior art rather than shipped fixes.

## What This Changes

Add write-time linting and auto-fix for Markdown code fences in workspace Markdown files, with language inference, warn mode, per-write skip support, and focused tests.

## Best Possible Solution

Keep this open until maintainers either implement a central Markdown code-fence lint/fix utility with sh/ts/python/json inference, change records, warn/auto-fix configuration, per-write skip support, and coverage across current workspace write paths, or explicitly decide that OpenClaw should not mutate workspace Markdown at write time.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is high confidence: exact symbol searches plus source inspection of the Markdown helper and write paths show the requested lint/fix hook is still absent.

## Solution Assessment

Unclear. A central helper integrated into the current write surfaces is the maintainable direction if the feature is accepted, but maintainers should first decide whether write-time mutation is default-on, warn-only, or opt-in because the requested default could alter intentional Markdown.
