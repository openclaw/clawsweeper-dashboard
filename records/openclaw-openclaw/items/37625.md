## What This Changes

## Best Possible Solution

Keep workspace writes byte-preserving unless maintainers approve a parser-backed, preferably opt-in or warn-first Markdown lint policy with coverage for nested fences, no-op preservation, write/edit paths, and upgrade safety.

## Reproduction Assessment

Not applicable; this is a request for new write-time Markdown rewriting/config behavior, not a bug in an existing documented contract. Source inspection shows current write/edit paths preserve caller content and no code-fence lint hook exists.

## Solution Assessment

Unclear; a write-path helper could satisfy the request, but heuristic auto-fix as the default is not clearly the safest policy. A parser-backed opt-in or warn-first mode would be safer if maintainers accept the feature.
