## Summary

Keep open. Current main still has the gateway assistant-media transcript path that can persist fallback text as a supplemental assistant message, and the issue is already paired with open closing PRs, so it should stay open until one canonical fix lands.

## What This Changes

The issue reports that Control UI/WebChat can duplicate assistant replies when the assistant-media delivery path stores a second `gateway-injected` assistant message after the normal final answer.

## Best Possible Solution

Land one canonical gateway fix that only appends supplemental agent-run assistant-media transcript entries when resolved non-text media content exists, while preserving normal model transcripts and non-agent command/TTS fallback behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an agent-run final or block payload with visible text plus a stale or unembeddable media URL: the media-bearing guard passes, the media builder returns no media message, and the append path can persist fallback text with an `:assistant-media` idempotency key.

## Solution Assessment

Yes. Guarding the supplemental append on resolved media content is the narrowest maintainable direction; #73962, #74502, and #75481 already target that path, so maintainers should review and land one canonical PR rather than open another repair lane.
