## Summary

Keep open: the IRC lone-surrogate bug is real and the patch is focused, but the current guard also stops decoding valid adjacent surrogate-pair `\u` escapes, changing existing outbound text behavior.

## What This Changes

## Best Possible Solution

Keep the decoder-stage IRC fix, but decode adjacent escaped high/low surrogate pairs as one valid astral character and leave only unpaired surrogate escapes literal, with regression coverage.

## Solution Assessment

No. The fix belongs in `decodeLiteralEscapes`, but the current implementation is too broad because it preserves valid escaped surrogate pairs instead of only guarding unpaired surrogate output.
