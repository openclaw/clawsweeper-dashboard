## Summary

This PR still fixes a real IRC decoder bug, but the refreshed patch has a remaining compatibility regression for escaped emoji pairs that follow another `\uXXXX` escape, so it should stay open for a narrow repair rather than be closed.

## What This Changes

The PR changes the IRC literal-escape decoder to preserve lone surrogate-range `\uXXXX` escapes while decoding valid surrogate-pair escapes, and adds protocol tests for those cases.

## Best Possible Solution

Keep the decoder-stage fix, but implement `\u` escape handling as a left-to-right scan that decodes valid high/low surrogate pairs wherever they appear and preserves only unpaired surrogate escapes literally.
