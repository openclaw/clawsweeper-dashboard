## Summary

The central runtime fix is still useful and not on main, but the PR remains blocked by a concrete test bug: the new regex detects every surrogate code unit, including valid astral-character pairs that the tests expect to allow.

## What This Changes

The PR changes IRC literal \u escape decoding so valid surrogate pairs become astral characters, unpaired surrogate-range escapes remain literal text, and adds IRC protocol regression tests.

## Best Possible Solution

Keep the decoder-stage runtime fix, repair the tests to detect only unpaired surrogates, and land after focused protocol validation proves valid pairs decode while lone surrogate escapes stay literal.
