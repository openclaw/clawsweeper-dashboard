## What This Changes

The issue requests that Google Chat approval-card clicks treat `APPROVAL_NOT_FOUND` as terminal by consuming the stale card token instead of retrying gateway resolution.

## Best Possible Solution

Review and land the linked Google Chat approval-card fix, or replace it with an equivalent narrow handler change that consumes `APPROVAL_NOT_FOUND` tokens while preserving retry for transient gateway failures.

## Reproduction Assessment

Yes at source level: a Google Chat card click whose `resolveApprovalOverGateway` call throws `APPROVAL_NOT_FOUND` takes the generic catch path and releases the token for retry. I did not run live Google Chat or tests because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable solution is to keep generic gateway failures retryable while completing the Google Chat card binding when the shared approval-not-found classifier matches.
