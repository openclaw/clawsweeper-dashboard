## Summary

Keep open: current main and v2026.6.6 already carry cron job IDs and running markers through the data path, and the CLI exposes a computed running status, but the Control UI cron list and agent cron panel still do not visibly render both the stable ID and in-progress state.

## What This Changes

The issue requests that each scheduled cron task visibly show whether it is running and include an identifier.

## Best Possible Solution

Render the existing cron job id and a running indicator derived from state.runningAtMs in the Control UI cron list and agent cron panel, while leaving custom create-time IDs to the separate custom-ID issue.

## Reproduction Assessment

Yes. Source inspection reproduces the gap by rendering a CronJob with id and state.runningAtMs: the backend and typed UI data carry both fields, but the Control UI cron list and agent cron panel render last completed status, next, and last timestamps instead.

## Solution Assessment

Yes. The best fix is a narrow presentation change that reuses existing id/runningAtMs data and preserves lastRunStatus as the last completed result, rather than adding new scheduler state or custom-ID creation here.
