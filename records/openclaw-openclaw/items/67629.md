## Summary

Current main still exposes cron job IDs and running state through the data path, but the Control UI cron list and agent cron panel do not visibly render them, so this remains an actionable UI feature request.

## What This Changes

Request that each timed cron task visibly show a stable identifier and whether it is currently running.

## Best Possible Solution

Render the existing job.id and a running indicator derived from the current cron state in the Control UI cron list and agent cron panel, while leaving create-time custom IDs to https://github.com/openclaw/openclaw/issues/65636 and https://github.com/openclaw/openclaw/pull/66023.

## Reproduction Assessment

Yes, source-level. Render a cron job object with id and state.runningAtMs; current main carries both fields through the data path, but the UI renderers only show name, schedule, last status, next, and last data.

## Solution Assessment

Yes. A narrow Control UI presentation change is the best fit because the core model, gateway projection, and CLI already expose the needed data; custom ID creation should stay with the existing canonical custom-ID issue and PR.
