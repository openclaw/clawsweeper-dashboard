## Best Possible Solution

Keep this issue as the broader transport investigation, land or reject the focused Discord REST fix separately, and require a minimal redacted wrapper-vs-vanilla repro for the remaining probe/Vertex/Slack paths before changing shared fetch behavior.

## Reproduction Assessment

No. The report and later comments give strong affected-host evidence, and current source shows parser paths that would fail on raw gzip bytes, but maintainer live testing did not reproduce the Discord probe failure on current main with Node v26.

## Solution Assessment
