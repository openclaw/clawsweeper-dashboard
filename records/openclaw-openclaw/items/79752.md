Keep open: current main still has the reported parsing-sensitive fetch and stream paths, no merged fix or duplicate owner item was found, and maintainer discussion explicitly asks for more wrapper-vs-vanilla and Vertex raw-stream evidence before choosing a transport fix.

## What This Changes

The issue reports raw gzip-compressed HTTP/SSE response bodies reaching JSON or text parsers in OpenClaw 2026.5.7, affecting Discord probing/delivery and possibly Anthropic Vertex streaming.

## Best Possible Solution

Keep this open while maintainers collect a minimal wrapper-vs-vanilla Discord repro and a separate Anthropic Vertex raw-SSE proof, then apply the narrowest confirmed transport fix rather than a generic decompression wrapper.

## Reproduction Assessment

No, we do not have a high-confidence current-main live reproduction. The source paths and dependency code make the Vertex raw-SSE failure plausible, and the issue has concrete v2026.5.7 artifacts, but the Discord side did not reproduce for a maintainer on current main with Node 26.

## Solution Assessment

Unclear. A generic response-decompression wrapper is not the best current fix; the safer path is a targeted `Accept-Encoding: identity` or transport fix only for a confirmed raw-stream/raw-fetch path.
