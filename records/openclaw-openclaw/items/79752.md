## Summary

Keep open: current main still contains the reported raw-body parser paths for Discord and Anthropic Vertex, and the follow-up comment adds concrete Linux/Node 24 Vertex evidence that the gzip/SSE failure is broader than the original macOS Discord report.

## What This Changes

The issue reports gzip-encoded HTTP/SSE response bodies reaching Discord JSON parsing and provider SSE parsing without being decompressed.

## Best Possible Solution

Fix the HTTP/SSE transport boundary so compressed response bodies are either decoded before raw parsers consume them or compression is disabled for the affected raw-byte parser paths, then prove Discord and Anthropic Vertex with live redacted runs plus focused regression tests.

## Reproduction Assessment

No, this review does not have a high-confidence live current-main reproduction. The source path is clear if gzip bytes reach the current parsers, and the issue/comments provide concrete v2026.5.7 live evidence, but current-main Discord/Vertex live proof is still missing.

## Solution Assessment

Unclear. Forcing Accept-Encoding: identity may be the fastest safe mitigation for raw SSE clients such as Anthropic Vertex, but the broader Discord/custom-fetch side should be scoped against live evidence before choosing between a shared transport fix, local identity headers, or an upstream pi-ai fix.
