## Summary

Current main still reuses the per-turn budget as the Codex app-server startup ceiling and does not expose a separate startup timeout knob, while this PR is a focused implementation of the missing behavior; the branch is currently dirty against main and needs maintainer review/rebase before merge.

## What This Changes

The PR adds optional Codex `appServer.startupTimeoutMs` config, manifest, docs, changelog, resolver support, startup call-site wiring, and focused regression tests.

## Best Possible Solution

Land this PR or an equivalent narrow change after rebasing onto current main, keeping startup timeout, per-turn timeout, and app-server request timeout as separate documented controls.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: with a never-resolving app-server startup and `params.timeoutMs = 30 * 60_000`, current main gives startup the full turn budget because `withCodexStartupTimeout` receives `params.timeoutMs`.

## Solution Assessment

Yes. The proposed direction is the narrow maintainable fix: add a resolver-backed optional `startupTimeoutMs`, expose it in manifest/docs, and use it only for the startup wrapper while preserving turn and request timeout semantics.
