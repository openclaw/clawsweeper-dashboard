## Summary

Keep open: current main still syncs official plugin installs through the supervised `openclaw update` path, but a raw `npm install -g openclaw@latest` package lifecycle only runs postinstall cleanup/migration and does not update `@openclaw/qqbot`; changing that would require maintainer policy judgment about npm postinstall side effects.

## What This Changes

The issue asks for `@openclaw/qqbot` to auto-update with OpenClaw when users upgrade via direct `npm install -g openclaw@latest`.

## Best Possible Solution

Define the update contract explicitly: either document that manual package-manager upgrades require a follow-up plugin update, or add a supervised post-package plugin sync outside unsafe raw npm postinstall behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: direct npm postinstall runs cleanup and registry migration only, while official plugin sync is wired to `openclaw update`.

## Solution Assessment

Unclear. Auto-updating plugins from npm postinstall is not clearly the best solution without maintainer policy review; the safer maintainable path may be docs plus the existing supervised updater/plugin-update commands.
