## Summary

Keep open: current main syncs trusted official plugins during the supervised `openclaw update` path, but direct `npm install -g openclaw@latest` still runs package postinstall work without plugin updates. Changing that would require maintainer policy judgment because npm postinstall plugin syncing would add networked package work and user-state mutation.

## What This Changes

Request that direct global npm upgrades also auto-update the official `@openclaw/qqbot` plugin to the matching OpenClaw release.

## Best Possible Solution

Define the update contract clearly: either document that manual package-manager upgrades require an explicit plugin update afterward, or add a supervised plugin-sync path outside unsafe raw npm postinstall behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction for the direct npm path: package postinstall runs migration/package hygiene only, while official plugin syncing is wired into `openclaw update`.

## Solution Assessment

No proposed patch is present, and auto-updating plugins from npm postinstall is not clearly the best fix. A safer maintainable direction needs maintainer choice between docs clarification and a supervised updater-owned sync path.
