## Summary

Keep open: current `main` appears to sync `qqbot` through the supervised `openclaw update` path, but a bare `npm install -g openclaw@latest` still only runs package postinstall work and does not run plugin updates. That leaves the reported direct-npm upgrade behavior unresolved and requires maintainer policy judgment before changing package lifecycle side effects.

## What This Changes

Requests that direct global npm upgrades also update the installed official `@openclaw/qqbot` plugin to the new OpenClaw release line.

## Best Possible Solution

Clarify and implement the intended update contract: either document that manual npm upgrades require `openclaw plugins update --all`, or add a deliberately supervised plugin-sync path outside unsafe raw npm postinstall behavior.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction for the direct npm path: package postinstall runs migration and package hygiene only, while plugin syncing is only wired into `openclaw update`. I did not run a live global npm upgrade in this read-only review.

## Solution Assessment

No proposed patch is present, and auto-updating plugins from npm postinstall is not clearly the best fix without maintainer policy review. The safer direction may be to keep plugin updates in `openclaw update` and make the manual npm fallback docs/changelog wording explicit.
