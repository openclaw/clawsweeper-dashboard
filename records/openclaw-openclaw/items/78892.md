## Summary

Keep open: current main still scopes official-plugin syncing to the supervised `openclaw update` path, while raw package-manager postinstall only runs packaged cleanup and registry migration. The remaining question is whether manual `npm install -g openclaw@latest` should gain networked plugin-update side effects or whether the docs should clarify the contract.

## What This Changes

Requests that manual `npm install -g openclaw@latest` upgrades also update installed official `@openclaw/qqbot` plugin records to the current OpenClaw release line.

## Best Possible Solution

Define the upgrade contract explicitly: either add a supervised post-package official-plugin sync that is safe for package-manager upgrades, or document that manual package-manager upgrades require `openclaw update` or a follow-up `openclaw plugins update --all`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: direct npm postinstall runs cleanup and registry migration only, while official plugin sync is wired to `openclaw update`; I did not run a live global npm upgrade.

## Solution Assessment

Unclear. Auto-updating plugins from npm postinstall may not be the safest maintainable solution without maintainer policy review; documentation plus the existing supervised updater/plugin-update commands may be preferable.
