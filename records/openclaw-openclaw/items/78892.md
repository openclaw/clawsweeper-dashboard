## Summary

Keep open: current main still syncs official plugins through the supervised `openclaw update` path, while a direct `npm install -g openclaw@latest` only runs package postinstall work and does not update `@openclaw/qqbot`. The remaining question is a maintainer policy decision: document the manual npm limitation, or add a supervised follow-up sync outside unsafe raw postinstall side effects.

## What This Changes

Request that direct npm global upgrades also update the official `@openclaw/qqbot` plugin to the matching OpenClaw release.

## Best Possible Solution

Clarify the update contract: either document that manual package-manager upgrades require `openclaw plugins update --all`, or add an intentional supervised plugin-sync step that is not hidden inside raw npm postinstall.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction for the direct npm path: package postinstall runs package hygiene and registry migration, while official plugin sync is only wired into `openclaw update`.

## Solution Assessment

Unclear. Auto-updating plugins from npm postinstall is not clearly the best fix because it would add networked package execution and user-state mutation to package lifecycle scripts; a docs clarification or supervised update path is safer.
