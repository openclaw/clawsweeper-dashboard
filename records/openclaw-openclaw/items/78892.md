## Summary

Keep open: current main still syncs official plugin installs through `openclaw update`, but direct `npm install -g openclaw@latest` only runs the package postinstall cleanup/migration path and does not update `@openclaw/qqbot`. Closing would need a maintainer decision on whether raw npm postinstall should gain networked plugin-update side effects or whether the supported path should remain documented/manual.

## What This Changes

Request that raw global npm upgrades keep the official `@openclaw/qqbot` plugin version aligned with the upgraded OpenClaw host package.

## Best Possible Solution

Define the upgrade contract explicitly: document that manual package-manager upgrades require `openclaw update` or `openclaw plugins update`, or add a supervised post-package sync path that avoids unsafe surprise work in raw npm postinstall.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: `npm install -g openclaw@latest` runs the package postinstall script, while official plugin sync is only wired through `openclaw update`.

## Solution Assessment

Unclear. Auto-updating plugins from raw npm postinstall may be convenient, but it adds package lifecycle side effects; a maintainer should choose between explicit documentation/manual update guidance and a supervised sync mechanism.
