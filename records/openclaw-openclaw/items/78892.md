## Summary

Keep open: current main still leaves raw `npm install -g openclaw@latest` upgrades outside the official plugin sync path, and changing npm postinstall to update plugins needs maintainer policy judgment.

## What This Changes

The issue asks for raw package-manager upgrades to automatically sync an installed official `@openclaw/qqbot` plugin to the new OpenClaw host version.

## Best Possible Solution

Define the upgrade contract explicitly: keep official plugin sync in supervised `openclaw update`, or design a guarded post-package sync path and document the manual package-manager follow-up.

## Reproduction Assessment

Yes. Source inspection shows direct npm postinstall runs packaged cleanup and registry migration, while official plugin sync is invoked from `openclaw update`; a live global upgrade was not run in this read-only review.

## Solution Assessment

Unclear. Auto-updating plugins from npm postinstall is not plainly the best fix because it would add package lifecycle side effects; maintainers should choose between documented manual follow-up and a supervised sync design.
