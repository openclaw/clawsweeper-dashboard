## Summary

Keep open: current main still syncs official plugins only through the supervised `openclaw update` path, while a raw package-manager upgrade runs package postinstall cleanup/registry migration without plugin updates. The remaining decision is whether `npm install -g openclaw@latest` should gain networked plugin-update side effects or whether the manual-upgrade contract should be clarified.

## What This Changes

The issue requests that the official `@openclaw/qqbot` plugin be auto-synced when OpenClaw is upgraded with `npm install -g openclaw@latest`.

## Best Possible Solution

Define the upgrade contract explicitly: either add a supervised, safe post-package official-plugin sync for package-manager upgrades, or document that manual package-manager upgrades require `openclaw update` or `openclaw plugins update --all`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: direct npm postinstall runs cleanup and registry migration only, while official plugin sync is wired to `openclaw update`; I did not run a live global npm upgrade.

## Solution Assessment

Unclear. Auto-updating plugins from npm postinstall may not be the safest maintainable solution without maintainer policy review; documentation plus the existing supervised updater/plugin-update commands may be preferable.
