Overall confidence: 0.9

Full review comments:

- **[P2] Honor the parent JSON flag:** `src/cli/program/register.status-health-sessions.ts:184`
  - body: The subcommand passes only `Boolean(opts.json)`, so `openclaw sessions --json explain <key>` emits text while sibling sessions flows merge parent options. Merge `parentOpts?.json` so existing shared flag placement keeps machine-readable output.
  - confidence: 0.91
- **[P2] Load the resolved store path:** `src/commands/sessions.ts:118`
  - body: `loadSessionStore(opts.store)` reads the raw CLI value before `resolveStorePath` expansion. Store values using `~` or `{agentId}` work in list/cleanup through `resolveSessionStoreTargets`, but explain can miss them unless it loads the normalized target path.
  - confidence: 0.9
- **[P2] Pass the selected agent into key resolution:** `src/commands/sessions.ts:119-123`
  - body: `resolveGatewaySessionStoreTarget` is called without `opts.agent`, so unqualified keys like `main` resolve against the default agent. `openclaw sessions --agent work explain main` can therefore inspect or miss the wrong session.
  - confidence: 0.9
- **[P2] Pick the freshest matching session entry:** `src/commands/sessions.ts:126`
  - body: `target.storeKeys.find(...)` uses canonical/input-first order, while the gateway lookup contract chooses the freshest matching entry. If canonical and legacy or case-variant keys coexist, explain can report stale state for the same session.
  - confidence: 0.88
- **[P2] Keep tool diagnostics tied to the inspected store:** `src/commands/sessions.ts:145-165`
  - body: After selecting `matchedKey` from the loaded store, the PR recomputes subagent capabilities and effective tools with `target.canonicalKey` only. Custom stores or legacy-key collisions can make the reported role/tools describe a different session than the entry being explained.
  - confidence: 0.86
- **[P2] Report model reasons only when they win:** `src/commands/sessions.ts:217-223`
  - body: `resolveSessionModelRef` ignores a provider override without a model override and prefers explicit overrides over runtime metadata, but the reason flags are based on field presence. That makes the diagnostic chain claim inputs affected resolution when they did not.
  - confidence: 0.86
