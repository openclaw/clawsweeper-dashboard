## Summary

Keep open: this is a recent external feature PR with a focused gateway protocol change that current main still does not implement. The implementation shape looks sound; the remaining actionable issue is a stale proof-script header that overstates whitespace validation.

## What This Changes

Adds optional `originAgent` support to `chat.inject`, persists it as a separate injected-assistant attribution field, regenerates Swift bindings, and adds proof scripts.

## Best Possible Solution

Land the additive server-side attribution contract after aligning the proof comment and maintainer API review, while keeping `provider`/`model` immutable and handling UI rendering in a follow-up.

## Reproduction Assessment

Not applicable: this is an additive feature PR, not a bug report. Source inspection confirms current main rejects `originAgent` under `additionalProperties: false` and persists injected assistant rows only under the existing OpenClaw sentinel.

## Solution Assessment

Mostly yes: a separate `originAgent` field is safer than overloading `provider` because replay filtering depends on the provider/model sentinel. The narrow safer alternative is only to fix the proof-script wording before maintainer API review.
