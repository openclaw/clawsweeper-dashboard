## Summary

Keep open: current main still has no `originAgent` support, and the related open `chat.inject` PRs do not supersede this attribution field. The sentinel-preserving design looks sound; the only concrete finding is a low-severity proof-script comment mismatch plus normal maintainer protocol coordination.

## What This Changes

Adds an optional `originAgent` parameter to `chat.inject`, persists it as trimmed display attribution on injected assistant transcript entries, regenerates Swift protocol models, and adds proof scripts.

## Best Possible Solution

Land the additive attribution field after fixing the proof comment and coordinating the Gateway protocol shape, while keeping `provider` and `model` as immutable replay sentinels.

## Reproduction Assessment

Not applicable: this is an additive feature PR rather than a bug reproduction. Source inspection confirms current main rejects `originAgent` via `additionalProperties: false` and persists only the existing provider/model sentinel.

## Solution Assessment

Mostly yes: a separate `originAgent` field is safer than overloading `provider` because replay filtering depends on the provider/model sentinel. The maintainable path is to fix the proof comment and coordinate overlapping `chat.inject` API work before merge.
