## Summary

Keep open. The sentinel-preserving design is the right direction and the supplied terminal proof is sufficient, but the PR still expands the Gateway protocol without public docs and retains a misleading proof-script comment.

## What This Changes

The PR adds optional `chat.inject` `originAgent` input, persists it as separate display-attribution metadata while preserving provider/model sentinels, regenerates Swift bindings, and adds proof scripts.

## Best Possible Solution

Land a documented additive `chat.inject.originAgent` contract that keeps provider/model immutable for replay filtering, then track UI rendering separately.

## Reproduction Assessment

Not applicable: this is a new Gateway protocol feature, not a broken current-main behavior. Source inspection confirms current main has no accepted or persisted `originAgent` field and keeps injected rows under the OpenClaw replay sentinel.

## Solution Assessment

Mostly yes: a separate `originAgent` field is safer than overloading `provider` because replay filtering depends on the provider/model sentinel. The merge path still needs public docs, the proof-comment correction, and maintainer API approval.
