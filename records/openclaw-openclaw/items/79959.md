The PR adds optional `originAgent` support to `chat.inject`, persists it as display attribution on injected assistant transcript rows, regenerates Swift protocol models, and adds proof scripts.

## Best Possible Solution

Land the additive server-side attribution field after aligning the proof script and maintainer API approval, while keeping `provider`/`model` immutable and tracking UI rendering separately.

## Reproduction Assessment

Not applicable: this is a feature PR, not a bug reproduction. Source inspection confirms current main has no accepted or persisted `originAgent` field and keeps injected rows under the existing OpenClaw replay sentinel.

## Solution Assessment

Mostly yes: a separate `originAgent` field is safer than overloading `provider` because replay filtering depends on the provider/model sentinel. The runtime shape is sound; the proof-script wording and maintainer API approval are the remaining gaps.
