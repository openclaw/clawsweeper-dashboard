## Summary

## What This Changes

The PR adds an optional `originAgent` field to `chat.inject`, threads it through the Gateway handler, persists it as a separate transcript attribution field, regenerates Swift protocol bindings, and adds proof scripts.

## Best Possible Solution

Land the additive server-side attribution field after correcting the proof-script wording and maintainer protocol approval, keeping `provider` and `model` immutable for replay filtering.

## Reproduction Assessment

Not applicable: this is a feature PR, not a current bug reproduction. Source inspection confirms current main rejects `originAgent` as an extra `chat.inject` parameter and stores injected rows under the existing OpenClaw sentinel.

## Solution Assessment

Yes for the server-side contract: a separate `originAgent` field is safer than overloading `provider` because replay filtering depends on the provider/model sentinel. The remaining issue is proof wording, not the implementation shape.
