## Summary

Keep open: the attribution goal is plausible, but the patch stores display attribution in a field that current replay code uses as a transcript-only sentinel, and the external PR has no after-fix real behavior proof.

## What This Changes

The PR adds optional `originAgent` to `chat.inject` parameters and threads it into injected assistant transcript persistence as the stored `provider` value.

## Best Possible Solution

Preserve `provider: "openclaw"` for gateway-injected transcript-only rows and add a separate validated display-attribution field or an explicit transcript-only marker, with schema/docs/tests updated together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with the PR, `chat.inject` plus `originAgent: "hermes"` persists `provider: "hermes"`, while current replay filtering only drops `provider: "openclaw"` and `model: "gateway-injected"` rows.

## Solution Assessment

No. Overwriting `provider` is not the narrowest maintainable solution because that field is already part of the runtime replay contract; a display-only attribution field or deliberate transcript-only marker would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Preserve the transcript-only provider sentinel:** `src/gateway/server-methods/chat-transcript-inject.ts:94`
  - body: Gateway-injected rows are deliberately stored as `provider: "openclaw"` plus `model: "gateway-injected"` so replay filtering drops them before the next model call. With `originAgent: "hermes"`, this line stores `provider: "hermes"`, so the injected UI note no longer matches the filter and can be sent back to the next provider, duplicating transcript-only content and risking the ordering failures documented in the replay filter.
  - confidence: 0.96
- **[P2] Treat blank originAgent as absent:** `src/gateway/protocol/schema/logs-chat.ts:69`
  - body: `Type.String` accepts `""` and whitespace, and the persistence path uses `??`, so a blank `originAgent` can persist an empty attribution value instead of the documented default. Validate this as non-empty, or trim and normalize blanks to unset before it reaches transcript persistence.
  - confidence: 0.86

## Security Review
