## Summary

Keep open. Current `main` still builds assistant answer payloads before generated tool-error warning payloads, and the focused regression tests still assert that reported user-visible order.

## What This Changes

The issue requests that generated tool-error warnings be shown before or inline with the assistant response instead of being appended after the response.

## Best Possible Solution

Stage parsed assistant answer payloads and warning-suppression state first, then emit any generated tool-error warning before assistant answer payloads with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `buildEmbeddedRunPayloads` with assistant success text plus a mutating `lastToolError`, and current tests show the assistant payload before the generated warning.

## Solution Assessment

Yes, with care. The requested ordering change is the narrow maintainable fix, but the implementation should stage parsed answer payloads before emission so acknowledgement detection keeps working.
