## Summary

This should stay open: current main still builds terminal reply payloads with assistant text before the tool-error warning, and existing tests assert that confusing order for mutating tool failures.

## What This Changes

Reorder user-facing tool-error warnings so failed tool notifications appear before or inline with the assistant response instead of after it.

## Best Possible Solution

Keep the existing warning policy, suppression, and duplicate detection, but emit any required warning payload before the final assistant answer payloads and update regression coverage plus changelog.

## Reproduction Assessment

Yes, from source: constructing payloads with assistant text plus a failed mutating tool follows the tested path where the answer is payload 0 and the error warning is payload 1. I did not run Vitest in this read-only review.

## Solution Assessment

No, the suggested simple block move is not quite the best fix because warning policy depends on parsed assistant text and failure acknowledgement. A safer fix should precompute parsed answer payloads and acknowledgement state, then insert the warning before those answer payloads only when the existing policy says it should be shown.
