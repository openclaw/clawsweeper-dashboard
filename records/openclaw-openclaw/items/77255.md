## Summary

Keep open. Current main still appends the generated mutating-tool warning after assistant answer payloads, and the focused tests currently assert that reported order.

## What This Changes

The issue asks for tool error warnings from failed mutating tools to appear before or inline with the assistant response instead of after it.

## Best Possible Solution

Refactor the payload builder so assistant answer payloads and acknowledgement flags are computed first, then emit any required tool-error warning before the assistant answer payloads while preserving suppression and dedupe rules.

## Reproduction Assessment

Yes. Source inspection on current main shows a direct reproduction path: pass assistant success text plus a mutating `lastToolError` to `buildEmbeddedRunPayloads`, and the warning is appended after the answer; existing unit expectations encode that order.

## Solution Assessment

Partly. The requested warning-before-answer outcome is the right narrow fix, but the literal block move is not the safest implementation because the warning policy depends on parsed assistant answer state.
