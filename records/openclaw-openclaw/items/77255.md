## Summary

Keep open. Current main still appends generated tool-error warnings after assistant answer payloads, and existing tests assert that order for mutating tool failures.

## What This Changes

The issue asks for failed-tool warnings to appear before or inline with the assistant response instead of being appended after it.

## Best Possible Solution

Keep the issue open and refactor the payload builder to compute assistant payloads and acknowledgement flags first, then emit any required tool-error warning before assistant answer payloads while preserving suppression and dedupe rules.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: pass assistant success text with a mutating `lastToolError` to `buildEmbeddedRunPayloads`, and current tests expect the answer before the warning; tests were not executed because this review is read-only.

## Solution Assessment

No for the literal suggested move, yes for the requested ordering. The safer implementation is to parse/compute assistant payloads and acknowledgement state before final output ordering, then place the generated warning before assistant answer payloads only when the existing policy says to show it.
