## Summary

Keep open. Current main still emits assistant answer payloads before generated tool-error warning payloads, and focused tests still lock in that user-visible order for mutating and exec tool failures.

## What This Changes

The issue asks for tool-error warnings generated during an assistant run to appear before or inline with the assistant response instead of after it.

## Best Possible Solution

Stage parsed assistant reply payloads and acknowledgement state first, then emit any generated tool-error warning before assistant answer payloads with focused regression coverage for mutating and exec failures.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: call `buildEmbeddedRunPayloads` with assistant success text and a mutating or exec `lastToolError`, and current tests assert the assistant payload before the warning payload.

## Solution Assessment

Yes, with care. Moving warning emission before visible assistant text is the narrow fix, but the implementation should still parse/stage answer text first so failure acknowledgement and duplicate-warning suppression keep working.
