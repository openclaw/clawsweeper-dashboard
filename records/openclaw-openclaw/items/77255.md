## Summary

Keep open. Current main still appends visible mutating tool-error warnings after assistant answer payloads, and the focused unit expectations currently assert that reported order.

## What This Changes

Request that embedded-run tool error warnings appear before or inline with the assistant response instead of after it.

## Best Possible Solution

Refactor the payload builder to compute parsed assistant answer payloads and warning-policy inputs first, then emit any visible tool-error warning before assistant answer payloads while preserving all current suppression and dedupe rules.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: pass assistant success text plus a mutating lastToolError into buildEmbeddedRunPayloads, and the warning is appended after the answer; the existing unit test asserts that order.

## Solution Assessment

Yes, with one implementation caveat. The requested ordering change is the narrowest maintainable fix, but the code should not simply move the block before parsing answers because the warning policy depends on acknowledgement and duplicate checks from the parsed answer text.
