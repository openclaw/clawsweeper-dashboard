## Summary

Keep open. Current main still appends the generated mutating tool-error warning after assistant answer payloads, and existing unit expectations encode that after-answer order.

## What This Changes

The issue asks for tool-error warning payloads to appear before or inline with assistant responses instead of after the final answer.

## Best Possible Solution

Build pending assistant payloads and acknowledgement state first, then emit any required tool-error warning before assistant answer payloads while preserving current suppression, dedupe, verbosity, cron, exec, and `sessions_send` rules.

## Reproduction Assessment

Yes, from source inspection. Passing assistant success text plus a mutating `lastToolError` into `buildEmbeddedRunPayloads` follows the existing test path that returns the answer first and the warning second; tests were not executed because this review is read-only.

## Solution Assessment

The requested direction is right, but the literal suggested move is not the safest implementation. The fix should separate parsing/acknowledgement collection from final emission order so current warning suppression behavior is preserved.
