## Summary

Keep open. Current main still appends generated tool-error warnings after assistant answer payloads, and current tests assert that reported user-visible order for mutating tool failures.

## What This Changes

The issue requests that generated tool-error warnings be emitted before or inline with assistant response text instead of after it.

## Best Possible Solution

Refactor the payload builder to compute parsed assistant payloads and failure-acknowledgement state first, then insert any required tool-error warning before emitting the assistant answer payloads.

## Reproduction Assessment

Yes. Source inspection and current tests give a high-confidence path: pass assistant success text plus a mutating `lastToolError` into `buildEmbeddedRunPayloads`, and current expectations put the assistant text before the generated warning.

## Solution Assessment

Yes, with care. The requested direction is the right narrow fix, but the implementation should stage parsed assistant payloads rather than moving the warning block ahead of acknowledgement detection.
