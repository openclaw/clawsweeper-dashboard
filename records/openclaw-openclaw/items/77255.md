## Summary

Keep open. Current main still builds the affected final payload list with assistant answer text before the generated tool-error warning, and existing tests assert that user-visible order for mutating tool failures.

## What This Changes

The issue asks for embedded-run tool error warnings to appear before or inline with the assistant response instead of after it.

## Best Possible Solution

Refactor the payload builder to compute assistant payloads and acknowledgement flags first, then emit any required tool-error warning before assistant answer payloads while preserving existing suppression and dedupe rules.

## Reproduction Assessment

Yes. Source inspection plus existing unit expectations provide a high-confidence reproduction path: pass assistant success text with a mutating `lastToolError` to `buildEmbeddedRunPayloads` and the warning is appended after the answer; tests were not executed because this review is read-only.

## Solution Assessment

The requested direction is right, but the literal suggested move is not the safest implementation. The fix should preserve the current warning policy by parsing answer text before final output ordering.

## Repair Work Prompt

Fix openclaw/openclaw#77255. In current main, `buildEmbeddedRunPayloads` appends mutating tool-error warnings after assistant answer payloads when the assistant text does not acknowledge the failure. Change only the embedded runner payload assembly so a generated warning that should be shown is delivered before assistant answer payloads, while preserving existing behavior for assistant error replies, explicit mutating-failure acknowledgement, duplicate warning text, sessions_send suppression, exec/detail suppression, and `suppressToolErrors` / `suppressToolErrorWarnings`. Do not naively move the `lastToolError` block if that loses `hasUserFacingFailureAcknowledgement`; parse or stage answer payloads first if needed, then order the final append. Add focused regression coverage in the payload tests for an edit/write failure where the warning precedes assistant success text, and keep acknowledgement suppression covered.
