## Summary

Keep open. Current main still appends generated tool-error warnings after assistant answer payloads, and the colocated tests assert that user-visible order for mutating tool failures.

## What This Changes

The issue asks for generated tool-error warnings to appear before or inline with the assistant response when a tool fails during a run.

## Best Possible Solution

Refactor the payload builder to compute parsed assistant payloads and acknowledgement flags first, then emit any required tool-error warning before assistant answer payloads while preserving existing suppression and dedupe rules.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: pass assistant success text plus a mutating lastToolError to buildEmbeddedRunPayloads, and current main appends the warning after the answer; existing tests assert that order.

## Solution Assessment

The requested direction is right, but the literal suggested move is not the safest implementation. The fix should preserve the current warning policy by parsing answer text before final output ordering.
