## Summary

Keep open. Current main still appends the generated tool-error warning after assistant answer payloads, and the existing tests assert that after-answer order for mutating tool failures.

## What This Changes

The issue asks for embedded-run tool error warnings to appear before or inline with assistant response text instead of after the final answer.

## Best Possible Solution

Refactor the payload builder to compute assistant payloads and warning-policy flags first, then emit any required tool-error warning before assistant answer payloads while preserving current suppression and dedupe rules.

## Reproduction Assessment

Yes. Source inspection on current main shows a high-confidence path: call `buildEmbeddedRunPayloads` with assistant success text plus a mutating `lastToolError`, and the warning is appended after the answer; existing tests encode that order.

## Solution Assessment

Yes on direction, but not as a direct block move. The maintainable fix is to separate assistant payload parsing from final output emission so the warning can move earlier without losing acknowledgement and duplicate-warning policy.
