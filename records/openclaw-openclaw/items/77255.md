## Summary

Keep open. Current main still emits assistant answer payloads before appending the synthetic tool-error warning, and nearby tests currently assert that after-answer order for mutating tool failures.

## What This Changes

Request that tool error warnings be shown before or inline with the assistant response instead of appended after it.

## Best Possible Solution

Refactor the payload builder so it computes assistant payloads and warning-policy flags first, then emits any required tool-error warning before final assistant answer payloads.

## Reproduction Assessment

Yes. Source inspection provides a high-confidence path: call `buildEmbeddedRunPayloads` with assistant success text and a mutating `lastToolError`, and the current builder appends the warning after the answer; tests were not run because this review is read-only.

## Solution Assessment

Yes, but not as a literal early block move. The maintainable fix is to parse or stage assistant payloads first so suppression and dedupe decisions remain intact, then order the warning before the final answer payloads.
