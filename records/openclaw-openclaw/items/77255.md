## Summary

Keep open. Current main still appends assistant answer payloads before generated tool-error warning payloads, and the focused tests currently assert that user-visible order.

## What This Changes

The issue asks for tool failure warnings to appear before or inline with the assistant response instead of after the response text.

## Best Possible Solution

Stage parsed assistant answer payloads and warning-suppression state first, then emit any generated tool-error warning before assistant answer payloads with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `buildEmbeddedRunPayloads` with assistant text plus a mutating or exec-like `lastToolError`, and current tests assert the assistant payload before the generated warning.

## Solution Assessment

Yes. The narrow maintainable fix is to reorder generated warning emission while staging parsed assistant text first so acknowledgement and duplicate-warning checks keep working.
