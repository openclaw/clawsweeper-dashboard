## Summary

Keep open. Current main still emits assistant answer payloads before the generated mutating-tool error warning, and existing unit expectations encode that order.

## What This Changes

The issue requests showing mutating tool error warnings before or inline with assistant response text in embedded run terminal replies.

## Best Possible Solution

Refactor the payload builder to compute assistant payloads and warning-policy flags before emission, then insert any generated tool-error warning before assistant answer payloads while preserving existing suppression and dedupe rules.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: pass assistant success text plus a mutating `lastToolError` to `buildEmbeddedRunPayloads`, and current main appends the warning after the answer; tests were not run in this read-only review.

## Solution Assessment

Yes on the requested direction, but not as a literal block move. The maintainable fix should separate payload preparation from final ordering so acknowledgement and duplicate-warning suppression remain intact.
