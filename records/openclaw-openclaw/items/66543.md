## Summary

Keep open. The reported cron/model-ID bug is valid and this PR moved in the right direction by avoiding blanket cron normalization, but the current branch still misses two concrete runtime paths: allow-any model configs can accept the corrupted primary ref before repair runs, and fallback candidates still bypass the new repair path.

## What This Changes

The PR adds alias-preserving compacted slash-ref repair in shared model selection and focused cron/model tests while leaving cron payload normalization trim-only.

## Best Possible Solution

Use one alias-safe resolution-layer repair: exact alias lookup first, then compact accidental whitespace only for clear slash-form provider/model refs before accepting uncataloged allow-any refs, and share that behavior across cron primary overrides and fallback candidate resolution while keeping cron normalization trim-only.

## Reproduction Assessment

Yes. The static current-main and PR paths show a high-confidence reproduction: a spaced slash ref can parse, be marked allowed under allow-any config, and return before the PR's compacted retry; payload fallbacks are passed through and later resolved by a helper that does not compact.

## Solution Assessment

No. The current PR direction is close, but the best fix needs to run the compacted slash-form retry before allow-any success and route fallback candidates through the same alias-preserving repair path.
