## Summary

Keep open: the underlying cron/model-ID bug is valid, but the current PR head still misses allow-any primary resolution and production fallback resolution, and it lacks after-fix real behavior proof.

## What This Changes

The PR branch adds an alias-preserving compacted retry for whitespace-corrupted provider/model refs in shared model selection and adds focused cron/model-selection tests.

## Best Possible Solution

Keep cron normalization trim-only, resolve exact aliases first, compact only failed slash-form refs before allow-any acceptance, and route primary plus fallback candidates through the same repair path with focused tests and live proof.

## Reproduction Assessment

Yes, from source inspection rather than an executed repro. Current main preserves internal spaces in cron payload model/fallback values, primary selection passes the raw override to shared resolution, and fallback strings use a raw parser without the PR's compacting repair.

## Solution Assessment

No. The alias-preserving direction is right, but the PR runs compacting too late for allow-any primary refs and does not apply the same repair to production fallback candidates.
