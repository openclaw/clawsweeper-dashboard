## Summary

Keep open: the reported bug is valid, but the current PR head still misses the allow-any primary path and production fallback repair, and the external PR only provides test-level verification.

## What This Changes

The PR adds an alias-preserving compacted retry for whitespace-corrupted provider/model refs in shared model selection and tests that cron normalization keeps internal spaces for alias compatibility.

## Best Possible Solution

Keep cron normalization trim-only, resolve exact aliases first, compact only clear slash-form provider/model refs before allow-any acceptance, and route primary plus fallback candidates through the same shared repair with focused tests.

## Reproduction Assessment

Yes from source inspection. Current main preserves internal spaces in cron payload model/fallback values, passes the primary override to shared resolution, and resolves fallback strings through a raw parser that does not compact spaced slash-form refs.

## Solution Assessment

No. The PR is the right general direction, but the compacted retry runs too late for allow-any configs and is not used by production fallback candidate resolution.
