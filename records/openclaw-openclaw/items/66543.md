## Summary

Keep this PR open: the reported cron/model-ref bug is still source-reproducible on current main and not shipped as fixed, but the current PR head remains incomplete and lacks required real behavior proof.

## What This Changes

This PR adds an alias-preserving compacted retry for slash-form model refs in shared model selection and tests cron normalization/pass-through behavior for spaced primary and fallback model strings.

## Best Possible Solution

Land an alias-preserving resolver repair that keeps exact alias matching first, compacts only clear slash-form refs for primary and fallback candidates, and includes focused tests plus real cron/provider proof.

## Reproduction Assessment

Yes, by source inspection: current main trims but preserves internal whitespace in cron payload model/fallback strings, then passes primary refs to resolveAllowedModelRef and fallback refs to resolveModelRefFromString. I did not run a live OpenRouter cron job.

## Solution Assessment

No. The resolution-time, alias-preserving direction is the right boundary, but this PR must repair slash refs before allow-any success and route fallback candidates through the same repair path.
