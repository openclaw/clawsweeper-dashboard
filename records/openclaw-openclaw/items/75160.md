## What This Changes

The PR updates `src/gateway/tools-invoke-http.test.ts` to mock plugin-backed `memory_search`/`memory_get`, record tool-factory contexts, and assert `/tools/invoke` retries memory slot tools with plugin tools enabled while ordinary core tool invokes remain plugin-disabled.

## Best Possible Solution

Merge the focused regression coverage after normal review and CI, preserving the existing production fallback on main. A real-plugin integration smoke can remain a separate follow-up if maintainers want broader memory-slot proof.

## Reproduction Assessment

Yes. The PR creates a deterministic unit reproduction: `memory_search` is a cataloged core id, the mocked factory hides memory tools when plugin tools are disabled, and the test verifies the request retries with contexts `[true, false]` and succeeds.

## Solution Assessment

Yes. Because the runtime fallback already exists on main, adding narrow regression coverage in the existing gateway HTTP invoke test is the most maintainable fix for this PR’s stated gap; broader integration coverage would be additive rather than required.
