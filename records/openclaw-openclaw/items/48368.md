## Summary

Keep open: current main does not implement the recommend-reset guard path, but this draft stacked feature PR is not mergeable as-is because it targets the old compaction hook path, has prompt-data and docs blockers, and lacks external real behavior proof.

## What This Changes

The branch adds a loop-aware compaction guard stack with `compaction.guard` config, transcript-tail scoring, guarded compaction instructions, post-compaction validation, and internal recommend-reset diagnostics.

## Best Possible Solution

Rebase or consolidate the guard stack onto current `pi-hooks` compaction code, align config/docs with the active guard semantics, wrap transcript-derived data as untrusted, and tie recommendations to checkpoint branch/restore recovery before merge.

## Reproduction Assessment
