## Summary

Keep open. Current main still lacks the requested compaction guard/recommend-reset path, so this PR is not obsolete, but the draft stacked branch is not merge-ready: it targets stale `pi-extensions` surfaces while current main uses `pi-hooks`, and its docs say the guard has no runtime effect while later commits enable runtime diagnostics.

## What This Changes

The PR adds compaction guard config/schema/docs, transcript-tail detector and scorer helpers, safeguard instruction augmentation, a post-compaction validator, recommend-reset diagnostic wiring, and tests for severe failed guarded compactions.

## Best Possible Solution

Rebase or consolidate the loop-aware compaction guard stack onto current `pi-hooks` and `compaction-hooks` surfaces, make docs/schema match the enabled behavior, and define recommend-reset as an internal diagnostic that points operators toward checkpoint branch/restore rather than automatic reset or unclear session loss.

## Reproduction Assessment

Not applicable as a user-facing bug reproduction. There is a high-confidence source-level review path: current main lacks the schema/runtime/diagnostic pieces, while this draft PR attempts to add them on stale paths and with conflicting docs.

## Solution Assessment

No, not as currently proposed. The conservative diagnostic direction is reasonable, but the implementation needs a current-main rebase and a reset/restore semantics decision before it is the narrowest maintainable solution.
