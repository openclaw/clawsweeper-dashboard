## Summary

Keep open. Current main still lacks both requested public fallback config surfaces: explicit embedded compaction models remain exact, and Lossless `summaryModel` remains plugin/context-engine config without a fallback chain; the related implementation PRs are closed unmerged partial attempts.

## What This Changes

Request an ordered fallback model chain for embedded compaction and Lossless/LCM summary-model compaction when the primary summarizer is rate-limited, unavailable, or times out.

## Best Possible Solution

Have maintainers decide the public fallback contract, then implement embedded compaction in core and keep Lossless summary-model fallback in the Lossless/context-engine plugin or a generic SDK seam if one is missing.

## Reproduction Assessment

Yes for the source-level config gap: current main lacks the requested fallback fields, and explicit compaction models bypass the fallback wrapper. A live quota or timeout repro is not required to show that the feature is missing.

## Solution Assessment

Unclear; an explicit compaction fallback list is plausible for embedded compaction, but the Lossless half crosses into plugin/context-engine ownership and needs maintainer API direction.
