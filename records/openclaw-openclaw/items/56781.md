## Label Justifications

- `P2`: This is a useful resilience improvement for compaction and summary-model failures with limited blast radius and no emergency regression evidence.
- `impact:session-state`: Failed compaction can leave transcripts and context state growing without a successful summary checkpoint.
- `impact:auth-provider`: The requested fallback chain changes model/provider routing and credential selection when a primary summarizer is unavailable.

## Summary

Keep open. Current main and the latest release still do not provide a public fallback list for explicit compaction models or context-engine/plugin `summaryModel` calls, and the earlier compaction-only PR was closed unmerged in favor of this broader contract decision.

## What This Changes

The issue requests fallback model chains for `agents.defaults.compaction.model` and Lossless Claw `plugins.entries.lossless-claw.config.summaryModel`.

## Best Possible Solution

Approve the fallback contract first, then land a narrow implementation for explicit compaction fallback plus either a generic plugin LLM fallback seam or a clearly plugin-owned Lossless `summaryModel` fallback option, with regression coverage for auth routing, session state, and exact-model defaults.

## Reproduction Assessment

Yes for the missing capability by source inspection: current config/schema/docs preserve a single exact compaction model, and plugin LLM completion exposes one requested model. I did not run a live provider outage reproduction in this read-only review.

## Solution Assessment

Unclear. The requested fallback behavior is reasonable, but the best fix depends on maintainer API judgment about explicit compaction fallback semantics and whether Lossless summary fallback should be plugin-owned or exposed through a generic runtime contract.
