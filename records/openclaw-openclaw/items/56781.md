## Label Justifications

- `P2`: This is a normal-priority reliability improvement for compaction and summary model routing with limited but real session impact.
- `impact:session-state`: Unavailable compaction or summary models can leave sessions growing without successful summary state.
- `impact:auth-provider`: The requested fallback chain directly affects provider/model routing and auth profile behavior for summary calls.

## Summary

Keep open: current main only retries compaction through the session fallback chain when no explicit compaction model is set; the requested compaction-specific fallback config and Lossless summaryModel fallback contract remain missing and need maintainer API/product judgment.

## What This Changes

Request adding fallback model chains for explicit compaction model overrides and Lossless Claw summaryModel so summarization can recover from rate limits, 500s, or timeouts.

## Best Possible Solution

Design one canonical fallback contract that covers explicit compaction models and context-engine summary models, preserves auth/session invariants, and documents upgrade behavior.

## Reproduction Assessment

Yes for the source-level capability gap: current main bypasses the fallback wrapper when `agents.defaults.compaction.model` is set, and plugin LLM completion accepts one model override. I did not run a live provider 429/timeout scenario.

## Solution Assessment

Unclear. The requested direction is reasonable, but maintainers still need to decide whether the fallback contract belongs in compaction config, plugin LLM runtime, Lossless plugin config, or a shared model-selection helper.
