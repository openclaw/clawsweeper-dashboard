## Summary

Keep open. Current main and the latest v2026.5.22 release still expose only a single exact explicit compaction model and a single plugin LLM completion model target, while the earlier compaction-only PR was closed unmerged in favor of this broader API/product decision.

## What This Changes

Request to add configurable fallback model chains for explicit compaction summarization and Lossless/LCM summaryModel calls.

## Best Possible Solution

Approve the fallback contract first, then land a narrow implementation for explicit compaction fallback plus either a generic plugin LLM fallback seam or a clearly plugin-owned Lossless summary fallback option, with focused auth-routing and session-state tests.

## Reproduction Assessment

Yes, by source inspection for the missing capability: current config/schema/docs preserve a single exact compaction model, and plugin LLM completion exposes one requested model. I did not run a live provider-outage reproduction in this read-only review.

## Solution Assessment

Unclear. The requested fallback behavior is reasonable, but the best implementation depends on maintainer API judgment about explicit compaction fallback semantics and whether Lossless summary fallback should be plugin-owned or exposed through a generic runtime contract.
