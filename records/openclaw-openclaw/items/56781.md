## Summary

Keep open. Current main still lacks an explicit compaction fallback config and a plugin-runtime fallback contract for Lossless Claw-style `summaryModel` use, while the related compaction-only PR is still open and blocked.

## What This Changes

Request fallback chains for `agents.defaults.compaction.model` and Lossless Claw `summaryModel` so summary work can retry configured secondary models.

## Best Possible Solution

Approve the public fallback contract, land a narrow compaction implementation, and keep Lossless Claw summary fallback plugin-owned unless a generic runtime fallback seam is needed.

## Reproduction Assessment

Yes for the missing capability by source inspection: the schema and type expose only a single compaction model, explicit compaction bypasses fallback, and plugin LLM completion accepts one model. No live outage repro was run in this read-only review.

## Solution Assessment

Unclear. The direction is reasonable, but the exact config shape and core-versus-plugin boundary need maintainer choice; the current related PR is not the best merge path as-is because it is blocked and only covers the compaction slice.
