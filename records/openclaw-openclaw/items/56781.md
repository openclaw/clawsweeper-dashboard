## Summary

Keep open. Current main still lacks the requested `agents.defaults.compaction.fallbacks` and Lossless `summaryModelFallbacks` config surfaces; embedded compaction only uses the existing session fallback chain when no explicit compaction model is set, and the related implementation PRs are closed unmerged partial attempts.

## What This Changes

Request to add ordered fallback model chains for core compaction and Lossless/LCM summary models when the primary summarizer is rate-limited, unavailable, or times out.

## Best Possible Solution

Have maintainers choose a core embedded-compaction fallback contract and keep Lossless summary-model failover in the Lossless/context-engine plugin unless a generic core SDK gap is identified.

## Reproduction Assessment

Yes for the requested config gap by source inspection: the fallback fields are absent, and explicit compaction models bypass the fallback wrapper. I did not run a live quota or timeout repro because this is a feature/config request rather than a strict existing-behavior bug.

## Solution Assessment

Unclear; an explicit core compaction fallback list is plausible for embedded compaction, but the Lossless half belongs at the plugin/context-engine boundary unless a core SDK gap is proven.
