- `impact:session-state`: The request is about failed compaction allowing sessions and transcripts to keep growing without a successful summary.

## Summary

Keep open. Current main and the latest release still lack an explicit fallback config for `agents.defaults.compaction.model` and a plugin-runtime fallback list for Lossless Claw-style `summaryModel` calls; the related compaction-only PR at https://github.com/openclaw/openclaw/pull/52012 is open, conflicting, and does not cover the plugin summary side.

## What This Changes

The issue requests configurable fallback model chains for compaction summaries and Lossless Claw-style plugin summary model calls when the primary model is unavailable.

## Best Possible Solution

Approve the fallback contract first, then land a narrow current-main implementation for explicit compaction fallback and either a generic plugin LLM fallback seam or a clearly plugin-owned Lossless `summaryModel` fallback option.

## Reproduction Assessment

Yes for the missing capability by source inspection: the schema/type expose only a single explicit compaction model, tests preserve no-fallback behavior for that override, and plugin LLM completion accepts one model. No live provider outage repro was run in this read-only review.

## Solution Assessment

Unclear. The requested direction is reasonable, but the best fix depends on maintainer API judgment about explicit compaction fallback semantics and whether Lossless summary fallback should be plugin-owned or exposed through a generic runtime contract.
