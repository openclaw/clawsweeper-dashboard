## Summary

Keep open: current main only walks model fallbacks for compaction when no explicit compaction model is configured, and the requested compaction-specific fallback array plus Lossless/context-engine summary-model fallback contract are still missing. The newer duplicate was closed in favor of this issue, while the referenced implementation PRs are closed unmerged, so this remains the canonical product/API decision point.

## What This Changes

The issue requests ordered fallback model chains for explicit compaction models and Lossless/context-engine summary models when the configured summarizer is rate-limited, unavailable, or times out.

## Best Possible Solution

Design one canonical fallback contract that covers explicit compaction models and context-engine summary models, preserves auth/session invariants, and documents upgrade behavior.

## Reproduction Assessment

Yes for the source-level capability gap: current main bypasses fallback traversal when `agents.defaults.compaction.model` is set, and context-engine/plugin LLM completion accepts one model override. I did not run a live provider 429/timeout scenario.

## Solution Assessment

Unclear. The requested direction is reasonable, but maintainers still need to decide whether the fallback contract belongs in compaction config, plugin LLM runtime, Lossless plugin config, or a shared model-selection helper.
