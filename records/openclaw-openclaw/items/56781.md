## Summary

Keep open. Current main only retries compaction through the session fallback chain when no explicit compaction model is configured; the requested `agents.defaults.compaction.fallbacks` config surface is absent, and the Lossless `summaryModelFallbacks` half likely belongs to the external plugin unless maintainers identify a missing core SDK/API seam.

## What This Changes

The issue requests ordered fallback model chains for embedded compaction and Lossless/context-engine summary model selection when the primary summarizer model is unavailable.

## Best Possible Solution

Keep this open for maintainer product/API review: decide the core compaction fallback contract separately from the Lossless plugin contract, and only add core config if maintainers want explicit compaction fallback lists in OpenClaw itself.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path for the requested gap: the fallback config fields are absent, and explicit compaction models bypass the fallback wrapper. I did not run a live provider-failure scenario because this is a feature/config request rather than a strict existing-behavior bug.

## Solution Assessment
