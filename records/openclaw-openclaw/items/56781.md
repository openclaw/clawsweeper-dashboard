## Summary

Keep open. Current main and the latest release still lack a public fallback list for explicit compaction models and for Lossless-style `summaryModel` plugin LLM calls; the related compaction-only PR is now closed unmerged and points back to this issue as the canonical API/product decision.

## What This Changes

The issue requests fallback model chains for `agents.defaults.compaction.model` and `plugins.entries.lossless-claw.config.summaryModel`.

## Best Possible Solution

Approve the fallback contract first, then land a narrow implementation for explicit compaction fallback plus either a generic plugin LLM fallback seam or a clearly plugin-owned Lossless `summaryModel` fallback option.

## Reproduction Assessment

Yes for the missing capability by source inspection: current config/schema/docs preserve a single exact compaction model, and plugin LLM completion exposes one requested model. No live provider outage reproduction was run in this read-only review.

## Solution Assessment

Unclear. The requested fallback behavior is reasonable, but the best fix depends on maintainer API judgment about explicit compaction fallback semantics and whether Lossless summary fallback should be plugin-owned or exposed through a generic runtime contract.
