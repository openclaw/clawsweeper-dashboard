Keep open: current main and v2026.6.6 still lack the requested explicit compaction fallback array and Lossless/context-engine summary-model fallback contract. ## What This Changes

Request to add ordered fallback model chains for explicit compaction summaries and Lossless/context-engine summary model calls.

## Best Possible Solution

Design one canonical fallback contract for explicit compaction models and context-engine summary calls, with schema/docs/tests that preserve auth, session, and truncation behavior.

## Reproduction Assessment

Yes for the source-level capability gap: current main bypasses fallback traversal when `agents.defaults.compaction.model` is set, and plugin LLM completion exposes one model override. I did not run a live provider 429/timeout scenario.

## Solution Assessment

Unclear. The requested direction is reasonable, but maintainers still need to decide whether the durable contract belongs in compaction config, plugin LLM runtime params, Lossless plugin config, or a shared model-selection helper.
