## Summary

Keep open. Current main partially covers implicit compaction fallback through the active session fallback chain, but it still lacks the requested explicit compaction fallback config and the Lossless Claw `summaryModel` fallback remains plugin-owned; https://github.com/openclaw/openclaw/pull/52012 is an open implementation candidate for only the core compaction-config slice.

## What This Changes

The issue requests fallback model chains for core compaction and the Lossless Claw summary model so transient model failures can try secondary models before compaction gives up.

## Best Possible Solution

Settle the public core compaction fallback API, review or replace https://github.com/openclaw/openclaw/pull/52012 for that slice, and keep Lossless Claw summary fallback in the plugin unless a generic SDK helper is needed.

## Reproduction Assessment

Yes. Source inspection on current main shows the strict compaction config has no fallback-list key, explicit `agents.defaults.compaction.model` bypasses `runWithModelFallback`, and only runtime `modelFallbacksOverride` plumbing exists.

## Solution Assessment

Unclear. The direction is reasonable, but the exact public config shape and the core-versus-plugin boundary need maintainer choice; the current implicit fallback behavior may be intentionally narrower than the requested explicit chain.
