## Summary

Keep open: current main and v2026.6.8 still do not expose explicit fallback arrays for either `agents.defaults.compaction.model` or Lossless/context-engine summary calls, and every related implementation PR is closed unmerged or only partial.

## What This Changes

The issue requests ordered fallback model chains for `agents.defaults.compaction` and Lossless/context-engine `summaryModel` use when the primary summarization model is unavailable.

## Best Possible Solution

Define one maintainer-approved fallback contract for embedded compaction and Lossless/context-engine summary calls, then implement it with schema/types/docs, model-ref collection, auth-routing coverage, exhaustion behavior, and focused tests.

## Reproduction Assessment

Yes for the source-level capability gap: current main has no compaction fallback array, explicit compaction models still bypass fallback traversal, and plugin/context-engine LLM completion exposes one model override. I did not run a live provider 429/timeout scenario because this is read-only feature triage.

## Solution Assessment

Unclear: ordered fallback arrays are plausible, but maintainers still need to choose whether the durable contract is embedded-only `compaction.fallbacks`, a plugin LLM fallback parameter, Lossless plugin config, or a unified summary-fallback helper.
