## Summary

Keep open: the manifest metadata direction is plausible, but the PR currently drops configured hook timeout policy, broadens lazy CLI plugin loading, and still lacks contributor real behavior proof before merge.

## What This Changes

The PR adds request-only `permissions.conversationAccess` manifest parsing, registry preservation, docs/tests/API baseline updates, loader hook-policy normalization, and a memory-slot append to primary CLI plugin loading.

## Best Possible Solution

Land a narrow manifest permission metadata contract only after preserving existing hook timeout policy, keeping raw conversation access explicitly host-approved, and removing or tightly scoping the unrelated memory-slot CLI loading change.

## Reproduction Assessment

Yes. Source inspection against PR head shows configured hook timeout fields are discarded before `createApi`, and any non-empty primary-command activation plan gets the configured/default memory slot appended.

## Solution Assessment

No. Request-only manifest metadata is a reasonable safer direction, but this patch should preserve existing hook policy fields and avoid widening CLI plugin load scopes for unrelated commands.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve configured hook timeouts:** `src/plugins/loader.ts:189-196`
  - body: `resolvePluginLoaderHookPolicy` only copies the two allow flags before passing policy to `createApi`. Current hook policy also carries `timeoutMs` and `timeouts`, so configs such as `plugins.entries.<id>.hooks.timeouts.agent_end` will stop overriding hook budgets after this patch.
  - confidence: 0.91
- **[P2] Scope the memory-slot append to memory commands:** `src/plugins/cli-registry-loader.ts:81`
  - body: This path runs for any primary command whose activation plan returns a plugin id. Since `plugins.slots.memory` defaults to `memory-core`, unrelated commands such as `googlemeet` will load memory plugin code too, widening the activation planner's scoped CLI loading.
  - confidence: 0.88
