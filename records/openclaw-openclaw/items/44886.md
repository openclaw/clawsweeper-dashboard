## Summary

Keep open. Current main and v2026.6.5 still accept Telegram DM-level tools/toolsBySender config while the enforcement path derives only group/channel policy and reads group config; the related repair PR is closed unmerged, so this remains the canonical security-relevant report.

## What This Changes

The issue asks whether Telegram account direct.<peer>.tools should enforce per-DM tool permissions and, if so, to fix the schema/runtime mismatch.

## Best Possible Solution

Implement direct-session tool-policy enforcement in the canonical shared policy/channel-plugin path with Telegram account, exact peer, wildcard, topic fallback, sender override, and group-isolation coverage; if maintainers decide it is not supported, remove or clarify the accepted direct tools config surface.

## Reproduction Assessment

Yes at source level: a Telegram direct session key is rejected by the group/channel session parser and the remaining tool resolver reads only group configuration, while Telegram direct config accepts tools. I did not run a live Telegram account in this read-only review.

## Solution Assessment

Yes in direction: enforcement should live in the canonical shared policy/channel-plugin path, not an independent Telegram-only shim. The closed related PR is useful context but not a current merge path.
