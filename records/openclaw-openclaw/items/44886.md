Keep open. Current main and the latest release still accept Telegram direct-message tool-policy config while the enforcement path remains group/channel-scoped, and the related repair PR was closed unmerged rather than landing a fix.

## What This Changes

The issue asks whether Telegram `direct.<peer>.tools` is intended to enforce per-DM-user tool permissions and, if so, to repair the enforcement gap.

## Best Possible Solution

Keep this as the canonical tracker until Telegram direct-message tool policy is either wired into the shared resolver with regression coverage or the accepted config surface is deliberately removed or documented as unsupported.

## Reproduction Assessment

Yes. Source inspection reproduces the mismatch: Telegram accepts and resolves `direct` config for DMs, but mounted tool policy is resolved only through group/channel policy paths.

## Solution Assessment

Unclear until maintainers choose the intended contract. The narrow maintainable fix is a direct-session-aware policy path with Telegram coverage, or a deliberate schema/docs removal if per-DM tool policy is not supported.
