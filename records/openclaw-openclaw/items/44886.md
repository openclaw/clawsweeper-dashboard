## Summary

Keep open. Current main and v2026.6.6 still accept Telegram direct-message tool-policy config while the mounted tool-policy path remains group/channel-scoped, and the only direct repair PR is closed unmerged with this issue left as the canonical tracker.

## What This Changes

The issue asks whether Telegram `direct.<peer>.tools` config should enforce per-DM-user tool permissions, or otherwise be clarified as unsupported.

## Best Possible Solution

Keep this as the canonical tracker until Telegram direct-message tool policy is either wired into the shared tool-policy path with regression coverage, or the accepted config surface is deliberately removed or documented as unsupported.

## Reproduction Assessment

Yes. Source inspection reproduces the mismatch: Telegram accepts and resolves `direct` config for DMs, but mounted tool policy is resolved only through group/channel policy paths.

## Solution Assessment

Unclear until maintainers choose the intended contract. The narrow maintainable fix is a direct-session-aware policy path with Telegram coverage, or a deliberate schema/docs removal if per-DM tool policy is not supported.
