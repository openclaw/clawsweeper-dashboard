## Summary

Keep open. Current main and v2026.6.5 still accept Telegram DM-level tool policy while the enforcement path recognizes only group/channel sessions and group policy configuration; the linked repair at https://github.com/openclaw/openclaw/pull/53530 closed unmerged, so this remains the canonical security-relevant report.

## What This Changes

Ensure Telegram direct-message peer tool policies are enforced consistently with direct-message prompt overrides.

## Best Possible Solution

Add direct-session awareness to the canonical shared policy resolver, expose the necessary typed channel-plugin seam, and cover Telegram account scope, exact peer, wildcard, thread fallback, sender overrides, and group isolation.

## Reproduction Assessment

Yes at source level: a Telegram `direct` session key is rejected by the group-context parser, and the remaining policy path reads group surfaces rather than the accepted direct map. This read-only review did not run a live Telegram account.

## Solution Assessment

Yes in direction: the narrow maintainable fix is direct-session support in the canonical resolver plus a typed Telegram adaptation, rather than another independent enforcement path. The closed unmerged PR demonstrates one approach but is not a viable landing branch.
