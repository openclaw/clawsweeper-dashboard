## Summary

Keep open: the PR targets a real, source-reproducible opencode-go watchdog bug, but the patch applies boundary-event liveness statelessly and can shorten the first-event timeout before a real provider delta; it also lacks external-contributor real behavior proof.

## What This Changes

The PR adds `text_end`, `thinking_end`, `toolcall_start`, and `toolcall_end` to the opencode-go idle-timer progress predicate and adds a regression test for post-delta boundary-event liveness.

## Best Possible Solution

Use a post-delta gated implementation: distinguish provider deltas from boundary-liveness events, refresh on boundaries only after a real delta, preserve the first-event timer before then, and add redacted after-fix wrapper proof.

## Reproduction Assessment

Yes, source-reproducible. Current main forwards boundary events but only re-arms idle liveness on delta events, and the linked canonical issue includes a focused failing timing sequence; I did not execute tests because this review is read-only.

## Solution Assessment

No. The fix belongs in the opencode-go provider wrapper, but this implementation applies boundary liveness statelessly; the safer maintainable shape preserves the first-event timer until a real provider delta has arrived.
