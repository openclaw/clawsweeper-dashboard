## Summary

Keep this PR open: the Gateway startup optimization is well motivated and has benchmark proof, but the new skip gate misses at least one shipped MSTeams doctor state migration source, so it can change upgrade behavior for users with only feedback-learning legacy files.

## What This Changes

The PR adds a lightweight legacy-state detector so `openclaw gateway run` can skip the doctor state-migration preflight when no known legacy inputs are present, with expanded config-guard tests.

## Best Possible Solution

Keep the Gateway startup performance win only after the legacy-input detector is authoritative for all shipped doctor state migrations, preferably through a plugin-owned detection seam or covered regression cases.

## Reproduction Assessment

Yes for the review finding by source inspection: a Gateway run with only MSTeams `*.learnings.json` legacy files can reach the new skip gate without any detector returning true, while current main's doctor contract detects those files.

## Solution Assessment

No; gating Gateway preflight is a plausible performance fix, but the best fix must preserve the existing doctor migration coverage before skipping the full preflight.
