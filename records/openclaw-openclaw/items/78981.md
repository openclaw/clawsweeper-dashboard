Full review comments:

- **[P1] Guard user workspace clears:** `src/agents/sandbox/user-backend.ts:227`
  - body: When `workspaceAccess` is `rw`, `remoteWorkspaceDir` is the configured target-user `workspaceDir`, and this clear command removes every child before upload. Because `normalizeAbsoluteUserPath` accepts broad values like `~` and `/`, a misconfigured backend can erase the target user's home or root-level contents; reject home/root and require a dedicated child workspace before deleting.
  - confidence: 0.9
- **[P2] Use exec preflight in the user backend:** `src/agents/sandbox/user-backend.ts:141-145`
  - body: SSH and OpenShell call `buildValidatedExecRemoteCommand` so unresolved placeholders and malformed generated shell are rejected before remote launch. The new user backend calls the quote-only builder directly, so commands that fail closed on sibling backends can run through `su`; use the validated builder and add a parity test.
  - confidence: 0.84

  - body: `CHANGELOG.md` is release-owned in this repo, and normal PRs should carry release-note context in the PR body or commit message instead. Keeping this entry in the branch creates release-generation churn and a stale conflict surface.
