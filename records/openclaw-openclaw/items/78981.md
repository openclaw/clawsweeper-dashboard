## Summary

## What This Changes

The PR adds a built-in `user` sandbox backend that runs sandboxed tool execution through `su`, with config, SDK exports, tests, docs, generated baselines, and a changelog entry.

## Best Possible Solution

If core accepts a first-class local-user backend, land it only after a dedicated safe workspace guard, resolved branch conflicts, owner security review, and docs/tests that match the approved trust boundary; otherwise keep the feature tracked under the linked design issue or a plugin-first prototype.

## Reproduction Assessment

Not applicable for the original feature request, but the blocking destructive-clear bug is source-reproducible: with `workspaceAccess: "rw"` and `sandbox.user.workspaceDir` set to `~` or `/`, the new backend reaches the clear command at `src/agents/sandbox/user-backend.ts:227`.

## Solution Assessment

No: the feature direction may be useful, but this patch is not mergeable as-is because the workspace clear path is unsafe and the same-host privilege-switching boundary still needs maintainer/security approval.
