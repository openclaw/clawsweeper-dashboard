- **[P1] Guard target-user workspace clears:** `src/agents/sandbox/user-backend.ts:227`
  - body: With `workspaceAccess: "rw"`, `targetDir` can be the configured target-user `workspaceDir`, and this command clears every child before upload.   - confidence: 0.93

  - body: Current SSH and OpenShell execution validates model-generated shell with `buildValidatedExecRemoteCommand` before remote launch, but the new user backend calls the quote-only builder directly. That lets malformed shell that sibling backends reject run through `su`, so use the same validated construction and parity tests.
  - confidence: 0.88
- **[P2] Register current-main user workdir and skills handling:** `src/agents/sandbox/backend.ts:87-90`
  - body: Current main uses backend workdir resolvers, backend cwd validation, and materialized sandbox skills so prompt, media, workspace-prep, and exec callers see the sandbox path without starting or bypassing the backend. The new `user` registration only installs a factory and manager, and the backend never refreshes the materialized skills workspace, so rebase and add the same contracts before merge.
  - confidence: 0.84
