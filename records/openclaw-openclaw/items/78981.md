## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Guard target-user workspace clears:** `src/agents/sandbox/user-backend.ts:227`
  - body: With `workspaceAccess: "rw"`, `targetDir` can be the configured target-user `workspaceDir`, and this command clears every child before upload. Because the resolver accepts broad paths such as `~` and `/`, a misconfigured backend can delete the target user's home or root-level contents; require a contained workspace/root guard before deleting.
  - confidence: 0.93
- **[P2] Validate exec commands before su launch:** `src/agents/sandbox/user-backend.ts:141-145`
  - body: Current SSH/OpenShell execution validates model-generated shell with `buildValidatedExecRemoteCommand` before remote launch, but the new user backend calls the quote-only builder directly. That lets malformed placeholders or unbalanced shell that sibling backends reject run through `su`, so use the same validated construction and parity tests.
  - confidence: 0.88
- **[P2] Register the user backend workdir resolver:** `src/agents/sandbox/backend.ts:87-90`
  - body: Current main uses backend `resolveWorkdir` registrations so `ensureSandboxWorkspaceForSession` can report the sandbox `containerWorkdir` without starting a backend. The new `user` registration only installs a factory and manager, so after rebasing this backend needs a resolver or explicit coverage showing resolverless user sandboxes do not break prompt, media, and workspace-prep callers.
  - confidence: 0.78
- **[P2] Materialize sandbox skills into the user workspace:** `src/agents/sandbox/user-backend.ts:202-215`
  - body: Current main materializes eligible skills into `.openclaw/sandbox-skills` for writable sandboxes, and SSH/OpenShell copy that directory into remote backends before exec. The user backend only seeds workspace and agent directories, so prompt paths for managed or plugin skills can point at files that do not exist for the target user.
  - confidence: 0.82
- **[P1] Keep user helpers out of the public SDK until approved:** `src/plugin-sdk/sandbox.ts:31-48`
  - body: These user-backend-specific helpers would become public plugin SDK contract for privilege-switching execution. Remove the new SDK exports until maintainers approve this as a stable API, or include explicit API direction, docs, and compatibility tests for the new contract.
  - confidence: 0.8
- **[P3] Drop the release-owned changelog entry:** `CHANGELOG.md:9`
  - body: `CHANGELOG.md` is release-generated in this repo, so normal feature PRs should carry release-note context in the PR body or squash message instead. Remove this entry and keep the user-visible behavior summary in the PR metadata.
  - confidence: 0.9
