## Summary

Keep open: the PR fixes a real TUI live-progress gap, but the branch still has a blocking edge case where commentary for a finalized run can render after the final answer.

## What This Changes

The PR moves TUI pending run binding before chat send resolution and renders Codex/OpenAI commentary-phase assistant agent events as transient progress with focused tests and a changelog entry.

## Best Possible Solution

Merge after assistant commentary rendering is limited to pending, active, or otherwise live non-finalized chat runs while preserving finalized-run support for late tool events.

## Reproduction Assessment

Yes by source inspection. Current main can drop early commentary because pendingChatRunId is set after sendChat resolves and handleAgentEvent has no assistant-commentary branch; the PR head also has a source-reproducible late-finalized-run regression path.

## Solution Assessment

No as proposed. The implementation direction is narrow, but assistant commentary must be rejected for finalized runs instead of sharing the finalized-run allowance that exists for late tool events.
