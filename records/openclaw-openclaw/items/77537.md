## Summary

Keep open: the patch addresses a real TUI commentary-progress gap, but it still lets finalized-run assistant commentary render after the final answer. A narrow repair should block finalized commentary while preserving late tool events.

## What This Changes

The branch moves TUI pending run tracking before `sendChat` resolves and renders commentary-phase assistant agent events as transient progress, with focused TUI tests and a changelog entry.

## Best Possible Solution

Land the narrow TUI change after adding a finalized-run guard and regression test, keeping final answers and history commentary-free while preserving existing late-tool handling.

## Reproduction Assessment

Yes, by source inspection. Current main can drop early commentary because pending run tracking is assigned after `sendChat` resolves, and the PR head can render finalized-run commentary if an assistant commentary agent event arrives after a chat final.

## Solution Assessment

No as currently proposed. The direction is right, but assistant commentary must be rejected for finalized runs while finalized-run support remains available for tool events.
