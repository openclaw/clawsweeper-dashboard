## Summary

Keep open. The PR is a focused and useful TUI fix, but it still has a merge-blocking late-event edge: assistant commentary for a finalized run can be accepted and appended after the final answer.

## What This Changes

The PR moves TUI pending run tracking before `sendChat` resolves and renders commentary-phase assistant agent events as live progress, with focused tests and a changelog entry.

## Best Possible Solution

Land the TUI fix after adding a finalized-run guard for assistant commentary plus a regression test, while preserving finalized-run tool events.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main can drop early commentary, and PR head can render late commentary after finalization because finalized runs are accepted before the assistant-progress renderer.

## Solution Assessment

No as proposed. The direction is narrow and maintainable, but assistant commentary should only render before run finalization; finalized-run support should remain tool-only.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Keep finalized-run support for tool events, but reject finalized assistant commentary.
