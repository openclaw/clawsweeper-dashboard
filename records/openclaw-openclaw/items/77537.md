## Summary

Keep open: the PR addresses a real source-visible TUI gap and is now clean/mergeable with CI completed, but it introduces a finalized-run handling regression that should block merge until commentary progress is ignored after final answers.

## What This Changes

The PR moves pending run tracking earlier and renders commentary-phase assistant agent events as transient TUI progress, with targeted tests and a changelog entry.

## Best Possible Solution

Merge once commentary rendering is gated so finalized runs cannot reopen assistant progress, while preserving late tool-event handling and final/history suppression.

## Reproduction Assessment

Yes. Source inspection shows current main can drop early commentary because pending run tracking is delayed and unknown agent events return; the PR discussion also includes a source-backed local TUI smoke for the proposed behavior.

## Solution Assessment

No as proposed. The direction is narrow and maintainable, but the implementation should skip assistant commentary rendering for finalized runs while still allowing late tool updates.
