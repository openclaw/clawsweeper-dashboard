## Summary

Keep open: the PR addresses a real TUI progress-rendering gap and has adequate real-behavior proof, but the current head still lets late finalized-run commentary reopen visible assistant output after the final answer.

## What This Changes

The PR moves `pendingChatRunId` assignment before `sendChat` resolves, renders commentary-phase assistant agent events as transient TUI progress, and adds focused tests plus a changelog entry.

## Best Possible Solution

Merge after commentary rendering is limited to pending, active, or otherwise non-finalized chat runs while preserving finalized-run support for late tool events.

## Reproduction Assessment

Yes by source inspection. Current main can drop early commentary because the pending run id is set after `sendChat` resolves and `handleAgentEvent` has no assistant-commentary branch; I did not run a live TUI repro.

## Solution Assessment

No as proposed. The implementation direction is narrow, but finalized-run acceptance for tool events should not also allow assistant commentary to render after the final answer.

## Review Findings

Overall correctness: patch is incorrect
