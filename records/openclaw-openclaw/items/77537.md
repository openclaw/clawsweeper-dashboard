## Summary

Keep open. The PR has a useful narrow direction and the body includes terminal smoke proof, but the latest diff still allows late commentary-phase assistant events to render after a run has already finalized.

## What This Changes

The PR moves TUI pending chat run tracking earlier and renders commentary-phase assistant agent events as transient TUI progress while keeping final answers/history commentary-free.

## Best Possible Solution

Merge a narrow TUI fix that binds early commentary to pending runs, renders it only while the chat run is live, rejects finalized-run assistant commentary, and preserves existing late tool-event handling.

## Reproduction Assessment

Yes, by source inspection. Current main can drop early commentary because the pending run id is assigned after `sendChat` resolves, and the PR's remaining late-final commentary bug follows from the changed event filter plus `ChatLog.updateAssistant` behavior.

## Solution Assessment

No as proposed. The direction is narrow and maintainable, but assistant commentary must be skipped for finalized runs while retaining finalized-run support for tool updates.
