## Summary

Keep this PR open, but it still has a blocking correctness issue: the branch handles the main early-commentary gap, yet still lets assistant commentary render after a run has finalized, which can make progress text visible as a new assistant message.

## What This Changes

The PR moves TUI pending run tracking earlier and renders Codex/OpenAI commentary-phase assistant agent events as transient TUI progress with focused tests and a changelog entry.

## Best Possible Solution

Land a narrow TUI fix that binds early commentary to the live run, renders it only before finalization, keeps final answers/history commentary-free, and preserves late tool events.

## Reproduction Assessment

Yes, by source inspection. Current main can drop early commentary before `pendingChatRunId` is set, and the PR head can render late finalized-run commentary because finalized runs are accepted before assistant commentary rendering.

## Solution Assessment

No as proposed. The direction is narrow and appropriate, but assistant commentary needs a finalized-run guard while retaining the existing late tool-event behavior.

## Review Findings
