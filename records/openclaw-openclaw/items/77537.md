## Summary

Keep open. The PR targets a real current-main TUI gap and has useful normal-path proof, but the current head still lets finalized-run assistant commentary reach the progress renderer, which can append commentary after the final answer.

## What This Changes

The PR moves TUI pending chat run tracking before `sendChat` resolves and adds assistant commentary-phase progress rendering with tests and a changelog entry.

## Best Possible Solution

Land a narrow TUI progress fix that binds early commentary to the live run, renders it only before finalization, keeps final answers/history commentary-free, and preserves late tool events.

## Reproduction Assessment

Yes, by source inspection. Current main can drop early commentary because pending run tracking is assigned after `sendChat` resolves, and the PR's late-final commentary edge follows from the changed event filter plus `ChatLog.updateAssistant` behavior.

## Solution Assessment

No as currently proposed. The direction is narrow and maintainable, but assistant commentary must be rejected for finalized runs while retaining the existing late-tool-event behavior.
