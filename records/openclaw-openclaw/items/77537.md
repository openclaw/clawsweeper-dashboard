## Summary

Keep open. The PR addresses a real TUI gap, but the current patch still lets assistant commentary for a finalized run enter the progress renderer, which can create visible assistant output after the final answer has already completed.

## What This Changes

The branch moves `pendingChatRunId` assignment before `sendChat` resolves and renders commentary-phase assistant agent events as transient TUI progress, with focused tests and a changelog entry.

## Best Possible Solution

Merge after commentary rendering is limited to pending, active, or non-finalized session runs, with a regression test proving late finalized commentary is ignored while late finalized tool events still work.

## Reproduction Assessment

Yes. Source inspection shows current main can drop pre-resolution commentary because `pendingChatRunId` is set after `sendChat`, and the PR head has a clear finalized-run commentary path that would call `updateAssistant` after finalization.

## Solution Assessment

No as proposed. The implementation direction is narrow, but the finalized-run allowance needed for tool events should not also allow assistant commentary to render after the chat final.
