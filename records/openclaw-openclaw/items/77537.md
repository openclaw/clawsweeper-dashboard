## Summary

The PR addresses a real TUI gap, but it should stay open because the current diff can render a late finalized-run commentary event as a new visible assistant message after the run has already completed.

## What This Changes

The PR renders Codex/OpenAI commentary-phase assistant agent events as transient TUI progress and moves pending run tracking earlier in message send handling.

## Best Possible Solution

Merge after commentary rendering is limited to pending, active, or non-finalized session runs, with a regression test proving late finalized commentary is ignored while finalized tool updates still work.

## Reproduction Assessment

Yes. Source inspection shows current main can drop pre-resolution commentary, and the PR head has a clear late-finalized commentary sequence that would call `updateAssistant` after `finalizeAssistant` removed the streaming run.

## Solution Assessment

No as proposed. The implementation direction is narrow, but the commentary renderer needs a finalized-run guard instead of sharing the finalized-run allowance that exists for late tool events.
