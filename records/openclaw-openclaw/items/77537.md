## Summary

## What This Changes

The branch moves pending TUI chat run tracking before `sendChat` resolves and renders commentary-phase assistant agent events as transient TUI progress with tests and a changelog entry.

## Best Possible Solution

Land a narrow TUI fix that binds early commentary to the live run, renders it only before finalization, keeps final answers/history commentary-free, and preserves late tool events.

## Reproduction Assessment

Yes, by source inspection. Current main can drop early commentary because pending run tracking is assigned after `sendChat` resolves, and the PR's remaining finalized-run commentary edge follows from the changed event filter plus `ChatLog.updateAssistant` behavior.

## Solution Assessment

No as proposed. The direction is narrow, but assistant commentary must be rejected for finalized runs while retaining the existing late-tool-event behavior.
