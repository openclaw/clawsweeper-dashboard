## Summary

Keep this PR open, but do not merge the current head: the useful TUI fix direction is still blocked by a finalized-run commentary edge that can append progress text after the final answer.

## What This Changes

The PR moves TUI pending run tracking before `chat.send` resolves and renders commentary-phase assistant agent events as live progress, with focused tests and a changelog entry.

## Best Possible Solution

Land a narrow TUI fix that binds early commentary to the live pending run, renders commentary only before finalization, preserves late tool events, and adds a regression test for post-final commentary.

## Reproduction Assessment

Yes, by source inspection. Current main can drop early commentary because the pending run id is assigned after `sendChat` resolves, and the PR head can leak late commentary because finalized runs are accepted before commentary rendering.

## Solution Assessment

No as proposed. The direction is narrow and maintainable, but the implementation needs a finalized-run guard for assistant commentary while retaining finalized-run support for tool events.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Block finalized commentary before rendering progress:** `src/tui/tui-event-handlers.ts:535`

Keep finalized-run support for tool events, but do not render finalized assistant commentary.
