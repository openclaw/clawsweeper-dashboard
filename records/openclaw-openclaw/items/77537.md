## Summary

Keep open: the PR addresses a real source-visible TUI progress gap, but the current head still lets late commentary after a finalized run reopen an assistant progress entry, so it needs a narrow fix before merge.

## What This Changes

The branch moves TUI pending-run tracking before `chat.send`, renders `phase:"commentary"` assistant agent events as transient TUI progress, adds TUI handler tests, and records a changelog entry.

## Best Possible Solution

Merge after commentary rendering is gated to pending, active, or non-finalized session runs while preserving finalized-run tool updates and final/history commentary suppression.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence event sequence: main can drop pre-resolution commentary, and the PR head can render a late finalized-run assistant commentary event after `finalizeAssistant` has removed the streaming component.

## Solution Assessment

No as proposed. The implementation direction is narrow and maintainable, but commentary progress should be blocked for finalized runs while leaving the existing late-tool-event exception intact.
