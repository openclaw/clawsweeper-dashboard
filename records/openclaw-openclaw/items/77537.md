## Summary

Keep open because the PR is a focused and useful TUI commentary-progress fix, but PR-head source still lets finalized-run assistant commentary render after the final answer as a new visible assistant message.

## What This Changes

The PR changes TUI run tracking and event handling so commentary-phase assistant agent events render as transient progress while final chat output remains clean.

## Best Possible Solution

Land the PR after commentary rendering is limited to active, pending, or session-tracked runs, while finalized-run support remains available for late tool events only.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main can drop early commentary before `pendingChatRunId` is set, and PR head can render assistant commentary for a finalized run because finalized runs pass the known-run filter before commentary rendering.

## Solution Assessment

No as proposed. The approach is narrow and maintainable, but it needs a guard or split known-run predicate so finalized runs are accepted for tool events without accepting finalized assistant commentary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject finalized commentary before rendering progress:** `src/tui/tui-event-handlers.ts:539-540`

Gate commentary to active, pending, or session-tracked runs while preserving finalized-run support for tool events.
