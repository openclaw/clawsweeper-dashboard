## Summary

Keep open. This PR fixes a real source-reproducible TUI commentary-progress gap, but the current patch still allows assistant commentary for finalized runs to render after the final answer and append visible commentary output.

## What This Changes

The PR moves TUI pending run tracking before `chat.send` resolves and renders commentary-phase assistant agent events as transient TUI progress, with focused tests and a changelog entry.

## Best Possible Solution

Land this after gating assistant commentary to non-finalized active, pending, or session runs, while preserving finalized-run tool events and adding a regression test.

## Reproduction Assessment

Yes, by source inspection. Current main can drop early commentary because pending run tracking is assigned after `sendChat` resolves and assistant commentary agent events have no TUI rendering path; the PR-head late-final leak follows from the finalized-run filter and `ChatLog` behavior.

## Solution Assessment

No as proposed. The direction is narrow and maintainable, but assistant commentary should be rejected for finalized runs before rendering progress while late tool events remain accepted.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject finalized commentary before rendering progress:** `src/tui/tui-event-handlers.ts:539-540`
  - body: Because `isKnownRun` includes finalized runs before calling `renderAssistantProgressEvent`, an assistant commentary event that arrives after the final chat event can call `updateAssistant` after finalization removed the streaming run, appending commentary as visible output. Keep finalized-run support for tool events, but reject finalized assistant commentary.
