## Summary

Keep open: the branch addresses a real TUI progress gap, but PR head still lets finalized-run commentary render after finalization, which can leak progress text as a new visible assistant message.

## What This Changes

The PR renders Codex/OpenAI commentary-phase assistant agent events as transient TUI progress and moves pending chat run tracking before `chat.send` resolves.

## Best Possible Solution

Land this after gating assistant commentary to active, pending, or non-finalized session runs, while preserving finalized-run tool events and adding regression coverage.

## Reproduction Assessment

Yes. Current-main source inspection shows the original drop path, and PR-head source inspection shows a concrete late-final leak path: send a final chat event for a run, then a commentary-phase assistant agent event for the same finalized run.

## Solution Assessment

No as proposed. The direction is narrow and maintainable, but the safest fix is to reject finalized assistant commentary before rendering progress while keeping finalized-run tool events accepted.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject finalized commentary before rendering progress:** `src/tui/tui-event-handlers.ts:539-540`

Keep finalized-run support for tool events, but skip finalized assistant commentary.
