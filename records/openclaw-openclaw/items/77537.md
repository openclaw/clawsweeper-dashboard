## Summary

Keep open. The PR addresses a real TUI commentary-progress gap that is still present on current main, but the current patch can render commentary after a run is finalized and append it as visible assistant output.

## What This Changes

The PR moves pending TUI run tracking before chat send resolves, renders commentary-phase assistant agent events as transient progress, adds focused TUI tests, and updates the changelog.

## Best Possible Solution

Land this after gating assistant commentary to non-finalized active, pending, or session runs, preserving finalized-run tool events and adding regression coverage for the late-final case.

## Reproduction Assessment

Yes, by source inspection. Current main can drop early commentary because pending run tracking is assigned after sendChat resolves and assistant commentary agent events have no TUI rendering path; PR head also has a clear late-final leak through finalizedRuns plus ChatLog.updateAssistant.

## Solution Assessment

No as proposed. The direction is narrow and maintainable, but assistant commentary should be rejected for finalized runs before rendering progress while late tool events remain accepted.
