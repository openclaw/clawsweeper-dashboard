## Summary

Keep open: the PR addresses a real TUI commentary-progress gap, but the current PR head can still render late commentary after a run has finalized, which would leak progress text into visible assistant output.

## What This Changes

The PR renders commentary-phase assistant agent events as transient TUI progress, moves pending chat run tracking before `chat.send` resolves, and adds focused TUI tests plus a changelog entry.

## Best Possible Solution

Land after commentary renders only for active, pending, or session-tracked runs while preserving late tool events after finalization.

## Reproduction Assessment

Yes. Source inspection shows current main can drop early commentary before `pendingChatRunId` is set, and PR-head source shows the finalized-run commentary leak path through `renderAssistantProgressEvent` and `ChatLog.updateAssistant`.

## Solution Assessment

No as proposed. The approach is narrow, but it needs a guard that rejects finalized-run assistant commentary before progress rendering while still allowing finalized-run tool events.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject finalized commentary before rendering progress:** `src/tui/tui-event-handlers.ts:539-540`

That appends commentary as visible assistant output; keep finalized-run support for tool events but gate commentary to active, pending, or session-tracked runs.
