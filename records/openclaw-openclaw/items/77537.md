## Summary

Keep open: the PR fixes a real TUI commentary-progress gap, but the current branch can still render assistant commentary after a run is finalized, which can append commentary as visible chat output after the final answer.

## What This Changes

The branch moves TUI pending run tracking before `sendChat` resolves and renders commentary-phase assistant agent events as live progress, with focused TUI tests and changelog text.

## Best Possible Solution

Land the TUI fix after adding a finalized-run guard for assistant commentary and a regression test, while preserving finalized-run tool events.

## Reproduction Assessment

Yes, by source inspection. Current main can drop early commentary because pending run tracking is assigned after `sendChat` resolves and assistant commentary stream events have no rendering path; the PR’s remaining late-final edge follows from the finalized-run filter and `ChatLog.updateAssistant` behavior.

## Solution Assessment

No as proposed. The direction is narrow and maintainable, but assistant commentary should render only before run finalization; finalized runs should remain accepted for tool events only.
