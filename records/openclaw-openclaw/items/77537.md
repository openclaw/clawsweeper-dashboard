## Summary

Keep open: this PR addresses a real source-reproducible TUI progress gap and includes usable terminal proof, but the latest head still has a P2 correctness blocker because finalized-run commentary can be rendered after the chat final.

## What This Changes

The branch moves TUI pending-run binding ahead of chat send resolution, renders Codex/OpenAI commentary-phase assistant agent events as transient TUI progress, and adds focused tests plus a changelog entry.

## Best Possible Solution

Merge after commentary rendering is limited to pending, active, or non-finalized session runs, while preserving finalized-run support for late tool events.

## Reproduction Assessment

Yes, source-reproducible. Current main binds `pendingChatRunId` after `sendChat` resolves, so early Codex/OpenAI commentary can be dropped before any visible chat event; the PR head also has a clear late-finalized commentary rendering path.

## Solution Assessment

No as proposed. The implementation direction is narrow, but `renderAssistantProgressEvent` must not run for finalized runs; keep the finalized-run allowance scoped to tool updates.
