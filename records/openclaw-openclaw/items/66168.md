## Summary

Keep open: current main still lacks the requested MCP Apps gateway protocol surface, but this PR is not merge-ready because it regresses current MCP loopback owner-token filtering and hook-based tool execution protections.

## What This Changes

The PR adds MCP Apps gateway WebSocket methods, HTTP resource handlers, `_meta.ui` schemas, an in-process resource registry, docs, and tests.

## Best Possible Solution

Rebase the feature onto current main and implement the MCP Apps protocol through the existing token-derived owner context, owner-only policy filtering, and shared before-tool-call execution path, with focused HTTP and WS regression tests.

## Reproduction Assessment

Yes. The reproduction path is code-level and testable: current `src/gateway/mcp-http.test.ts` already exercises spoofed owner headers, owner-only filtering, hook blocking, and abort propagation, while the PR head changes those paths and adds a new direct WS tool-call path.

## Solution Assessment

No. The feature direction may be valid, but this branch is not the safest implementation until it preserves current MCP loopback security semantics instead of adding direct execution and stale owner-context handling.

## Review Findings

Overall confidence: 0.91

Full review comments:

- **[P1] Derive owner status from bearer tokens:** `src/gateway/mcp-http.request.ts:139-140`
  - body: Current main derives `senderIsOwner` from separate owner and non-owner MCP loopback bearer tokens and ignores spoofed owner headers. This branch keeps owner status header-derived, so a caller with a non-owner loopback bearer can self-declare owner and reach owner-only tools.
  - confidence: 0.9
- **[P1] Filter owner-only tools before caching:** `src/gateway/mcp-http.runtime.ts:63-64`
  - body: Current main applies `applyOwnerOnlyToolPolicy` before caching and schema generation. This branch caches the unfiltered tool list, allowing non-owner MCP callers to list and execute owner-only tools such as `cron`, `gateway`, or `nodes`.
  - confidence: 0.93
- **[P1] Run before-tool hooks on HTTP calls:** `src/gateway/mcp-http.handlers.ts:126`
  - body: The modified HTTP `tools/call` handler executes the tool directly. That skips current before-tool-call hooks used for blocking, parameter rewriting, approval handling, and abort propagation before loopback tool execution.
  - confidence: 0.91
- **[P1] Run before-tool hooks on WS calls:** `src/gateway/server-methods/mcp.ts:146`
  - body: The new `mcp.tools.call` handler also calls `tool.execute` directly. Since this creates a second gateway tool execution surface, it needs the same before-tool-call hook and abort behavior as the loopback path before merge.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The PR adds MCP tool execution surfaces while regressing current owner-token filtering and hook-based execution protections.

Concerns:

- **[high] Owner status is header-derived:** `src/gateway/mcp-http.request.ts:139`
  - body: The PR head validates a bearer token but derives owner status from `x-openclaw-sender-is-owner`; current main derives owner status from separate owner/non-owner bearer tokens, so the PR would make owner status spoofable on the loopback surface.
  - confidence: 0.9
- **[high] Owner-only tools can leak through the cache:** `src/gateway/mcp-http.runtime.ts:63`
  - body: The PR head caches and builds schemas from unfiltered tools without `applyOwnerOnlyToolPolicy`, allowing non-owner MCP callers to see or invoke tools current main filters out.
  - confidence: 0.93
- **[high] HTTP tool calls bypass hooks:** `src/gateway/mcp-http.handlers.ts:126`
  - body: The modified HTTP handler directly executes tools, skipping hook-based blocks, parameter rewrites, approvals, and abort propagation that current main applies before execution.
  - confidence: 0.91
- **[high] WS tool calls bypass hooks:** `src/gateway/server-methods/mcp.ts:146`
  - body: The new WebSocket handler executes tools directly, creating a second tool execution surface without the current before-tool-call protections.
  - confidence: 0.88
