## Summary

Keep this PR open for maintainer review: current main still lacks the proposed MCP Apps gateway methods/resources, but the branch is an XL public gateway/tool-execution feature with high-confidence security regressions, merge conflicts, and no after-fix real behavior proof.

## What This Changes

The PR adds MCP Apps WebSocket methods, HTTP resource handlers, `_meta.ui` schemas, an in-process `ui://` resource registry, docs, and tests for gateway-side MCP Apps support.

## Best Possible Solution

Rebase the feature onto current main and implement the MCP Apps protocol through token-derived owner context, owner-only tool filtering, shared hook/abort-aware execution, focused regression tests, and contributor-provided real MCP client proof.

## Reproduction Assessment

Yes for the review defects: source inspection of the PR head and current-main tests show the owner-token, owner-only filtering, hook blocking, and abort-propagation paths that the branch bypasses. The requested MCP Apps feature itself is not a bug reproduction case.

## Solution Assessment

No. The feature direction may be valid, but this branch is not the safest implementation until it preserves current MCP loopback security semantics and resolves the public gateway/resource boundary with maintainers.
