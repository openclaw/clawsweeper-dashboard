## Summary

Keep this PR open for maintainer review: current main does not implement the MCP Apps gateway surface, but this branch is not safe to merge because it regresses current MCP loopback security semantics and lacks real gateway-client behavior proof.

## What This Changes

The PR adds gateway MCP Apps WebSocket methods, HTTP resources/list/read handling, a ui:// resource registry, _meta.ui schemas, docs, and tests.

## Best Possible Solution

Rebase the feature onto current main and implement it through token-derived owner context, owner-only filtering, shared hook/abort-aware execution, scoped resource semantics, focused regression tests, and real MCP client proof.

## Reproduction Assessment

Yes for the review defects: source inspection of the PR head and current-main code/tests shows the owner-token, owner-only filtering, hook blocking, and abort-propagation paths that the branch bypasses. The requested MCP Apps protocol itself is a feature, not a bug reproduction case.

## Solution Assessment

No. The feature direction may be valid, but this branch is not the narrowest maintainable solution until it preserves current MCP loopback security semantics and resolves the gateway/resource boundary with maintainers.
