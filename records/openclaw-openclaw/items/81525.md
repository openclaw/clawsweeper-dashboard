## Summary

Keep open. Current main and v2026.6.8 still trust configured `input: ["image"]` metadata for media-understanding provider/model selection before any catalog-narrowing validation, and the remaining fix needs a maintainer decision on provider/config capability precedence.

## What This Changes

The issue requests validation and clearer diagnostics when media-understanding selects a user-declared image-capable model whose real provider or catalog capability is text-only.

## Best Possible Solution

Choose and document a capability-precedence contract, then implement focused validation and diagnostics so catalog-known text-only models are not silently selected as media-understanding image models.

## Reproduction Assessment

Yes for the source-level selection path: a configured provider model with `input` containing `image` is accepted and prioritized without a catalog cross-check. I did not run a live NVIDIA request in this read-only review, so the external provider rejection remains based on the reporter's logs.

## Solution Assessment

Unclear as an implementation direction because the issue lists several viable safety nets. The best fix needs a maintainer-approved capability precedence policy before choosing rejection, startup warning, runtime fallback, or an explicit override mechanism.
