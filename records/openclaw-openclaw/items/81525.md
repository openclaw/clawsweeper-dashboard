## Summary

Keep open. Current main still trusts user-declared `models.providers.*.models[].input` for media-understanding image routing without a catalog cross-check, and failed summaries still drop the provider/model and capability detail needed to diagnose the mismatch.

## What This Changes

Request validation and clearer diagnostics when media-understanding routes images to user-declared vision models whose actual provider or catalog capability is text-only.

## Best Possible Solution

Add maintainer-approved catalog-aware validation or warnings that preserve intentional overrides, and keep failed media-understanding summaries actionable by reporting the attempted provider/model and capability details.

## Reproduction Assessment

Yes for the OpenClaw-side path: current source still selects configured image rows before catalog checks, and the bundled NVIDIA Kimi catalog row is text-only. I did not live-call NVIDIA, so the external rejection is supported by the reporter's log rather than a fresh network reproduction.

## Solution Assessment

Unclear. Strictly refusing catalog-narrower config could break intentional custom overrides, so a warning plus fuller failed-attempt diagnostics is the safer first implementation unless maintainers choose stricter catalog authority.
