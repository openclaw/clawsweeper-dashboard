## Summary

Keep open. Current main and v2026.5.12 still trust configured image-capability rows before any bundled-catalog cross-check and still collapse failed media-understanding diagnostics, so the reported validation and observability gap remains unresolved.

## What This Changes

The issue asks OpenClaw to validate or clearly diagnose image media-understanding routing when configured provider metadata incorrectly declares text-only models as image-capable.

## Best Possible Solution

Adopt a catalog-aware warning or skip policy for configured image models whose bundled catalog row is narrower, preserve the attempted provider/model and full capability reason in failed media-understanding logs, and document the precedence rule.

## Reproduction Assessment

Yes for the OpenClaw-side current-main path: configured image rows are selected from `models.providers.*.models[].input` before catalog validation, and failed summaries drop the attempted model plus diagnostic tail. The external NVIDIA rejection was not live-called, but the reporter's logs and the bundled text-only catalog row support that last step.

## Solution Assessment

Unclear. Preserving failed-attempt diagnostics is a narrow bug fix, but changing capability authority between user config and bundled catalog needs maintainer policy judgment to avoid breaking intentional custom-provider overrides.
