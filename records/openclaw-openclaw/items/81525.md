## Summary

Keep open. Current main still trusts user-declared image capability metadata for configured media-understanding models, and failed media-understanding summaries still drop the attempted provider/model detail and truncate the richer capability error; the related open issues cover adjacent observability or broader validation policy, but not this exact report end to end.

## What This Changes

The issue asks OpenClaw to validate configured image-model capability declarations before auto-routing images and to make failed media-understanding diagnostics actionable.

## Best Possible Solution

Adopt a maintainer-approved catalog-aware warning or validation policy for configured image models, and preserve the attempted provider/model plus full capability reason in failed media-understanding summaries.

## Reproduction Assessment

Yes for the OpenClaw-side path: current source selects user-declared image-capable configured rows before catalog checks, NVIDIA Kimi remains cataloged as text-only, and the failed summary truncates the richer runtime error. The live NVIDIA rejection itself was not rerun in this read-only review.

## Solution Assessment

Unclear. A warning plus fuller failed-attempt diagnostics is a narrow maintainable step, but refusing or overriding user-declared capability metadata requires a maintainer decision about catalog authority versus operator overrides.
