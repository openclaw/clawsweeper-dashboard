## Summary

Keep open. Current main and v2026.6.8 still trust explicit configured `input: ["image"]` metadata before checking the known catalog row, and the remaining fix needs a maintainer decision on configured capability precedence.

## What This Changes

The issue requests validation and clearer diagnostics so media-understanding does not route images to user-declared image-capable models that known catalogs advertise as text-only.

## Best Possible Solution

Define the configured-model capability precedence contract, then prevent known catalog text-only rows from being auto-selected for image understanding unless maintainers intentionally provide an explicit override path, while preserving resolved provider/model capability detail in failure logs.

## Reproduction Assessment

Yes, at source level. Current main selects and registers configured models based on explicit `input: ["image"]` before checking the known catalog row, while the reported NVIDIA catalog entry remains text-only.

## Solution Assessment

Unclear. The requested guardrail is valid, but maintainers need to decide whether explicit configured capabilities are authoritative, subordinate to known catalog rows, warning-only, or require an opt-in override before code changes are safe.
