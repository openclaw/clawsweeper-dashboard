## Summary

Keep open. Current main still trusts user-declared image capability before catalog validation and still compresses failed media-understanding diagnostics, so the reported validation and observability gap remains unresolved.

## What This Changes

Validate or warn when configured media-understanding image model capabilities conflict with catalog/runtime capabilities, and make image failure diagnostics actionable.

## Best Possible Solution

Adopt a maintainer-approved catalog-aware warning or validation policy for configured image models, and preserve attempted provider/model plus full capability detail in failed media-understanding summaries.

## Reproduction Assessment

Yes for the OpenClaw-side current-main path: source inspection shows configured image rows are selected before catalog checks, and failed summaries truncate the detailed capability error. I did not live-call NVIDIA, so that rejection portion relies on the reporter's log.

## Solution Assessment

Unclear. Strictly refusing catalog-narrower user config may be too aggressive because configured rows currently override discovered/catalog metadata; warning plus clearer failed-attempt diagnostics is the safer first design point unless maintainers choose strict catalog authority.
