## Summary

Keep open. Current main still trusts user-declared image capability metadata for media-understanding routing and still truncates the richer unsupported-image diagnostic in failed summaries; deciding whether bundled catalog metadata should constrain user config needs maintainer product judgment.

## What This Changes

The issue asks OpenClaw to validate or diagnose media-understanding image routing when `models.providers.*.models.*.input` declares image support for a text-only model.

## Best Possible Solution

Add a maintainer-approved catalog-aware diagnostic and richer failed media-summary output while preserving user-config authority unless maintainers explicitly choose stricter override semantics.

## Reproduction Assessment

Yes for the OpenClaw-side path: current source still selects user-declared image-capable config rows and the NVIDIA manifest marks the reported model text-only. I did not live-call NVIDIA, so the provider rejection itself relies on the reporter's log plus that catalog mismatch.

## Solution Assessment

Unclear. Improving the failed warning is a narrow maintainable fix, but changing whether bundled catalog metadata constrains user-declared provider capabilities needs an explicit maintainer product decision.
