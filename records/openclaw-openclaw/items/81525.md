## Summary

Keep open. Current main still trusts configured image-capable model rows before catalog validation and still compresses failed media-understanding diagnostics, so the capability-mismatch and opaque-warning concerns remain unresolved rather than cleanup-closeable.

## What This Changes

Request validation and clearer diagnostics for media-understanding image routing when configured model input metadata conflicts with actual or bundled-catalog capabilities.

## Best Possible Solution

Decide the config-versus-catalog authority contract, then add either catalog-aware warnings or a guarded skip/refusal path plus failed-summary diagnostics that preserve provider, model, and resolved input details.

## Reproduction Assessment

Yes for the OpenClaw-side path: current source selects configured `input: ["image"]` rows before catalog validation, the bundled NVIDIA row is text-only, and the failed summary formatter can drop the detailed error. I did not live-call NVIDIA in this read-only review.

## Solution Assessment

Unclear. A hard catalog refusal is not clearly the safest fix because current behavior treats user config as authoritative; a maintainer-approved warning/diagnostic path may be the narrower first step.
