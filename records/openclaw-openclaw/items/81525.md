## Summary

Keep open. Current main still treats user-declared `input: ["text", "image"]` as authoritative for media-understanding routing and still truncates the failed-attempt diagnostic, so the reported validation/observability gap remains unresolved and needs maintainer policy judgment.

## What This Changes

The issue requests catalog/probe validation, deterministic selection, or more actionable diagnostics when media-understanding routes images through user-declared vision models.

## Best Possible Solution

Add maintainer-approved catalog-aware validation or warnings that preserve intentional overrides, and at minimum stop truncating failed-attempt provider/model and capability details.

## Reproduction Assessment

Yes for the OpenClaw-side current-main path: source inspection shows configured image rows are selected before catalog checks, and the bundled NVIDIA Kimi row is text-only. I did not live-call NVIDIA, so the provider rejection portion relies on the reporter's log.

## Solution Assessment

Unclear. Hard-refusing catalog-narrower user config may not be the best fix because current behavior intentionally lets configured rows override discovered/catalog rows; a warning plus clearer failed-attempt diagnostics is the safer first design point unless maintainers choose strict catalog authority.
