## Summary

Keep open. Current main still trusts configured image-capability metadata before any bundled catalog cross-check, the bundled NVIDIA Kimi entry is still text-only, and failed media-understanding summaries still drop the attempted provider/model detail that would make this diagnosable.

## What This Changes

The issue requests validation and clearer diagnostics when media-understanding routes images to user-declared vision models whose actual or bundled-catalog capabilities are text-only.

## Best Possible Solution

Adopt a maintainer-approved catalog-aware warning or validation policy for configured image models, and preserve full failed-attempt provider/model capability details in media-understanding warnings.

## Reproduction Assessment

Yes for the OpenClaw-side path: current source shows configured image rows are selected before catalog checks, failed attempts carry provider/model detail, and the warning formatter truncates that detail. I did not live-call NVIDIA, so that final provider rejection is supported by the reporter's logs rather than a fresh live run.

## Solution Assessment

Unclear. A hard refusal of catalog-narrower user config may be too aggressive for custom providers; a warning plus clearer failed-attempt diagnostics is the safer first step unless maintainers decide bundled catalog metadata should be authoritative.
