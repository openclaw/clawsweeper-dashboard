## Label Justifications

- `P2`: This is a normal-priority media-understanding/provider routing bug with limited but real impact for custom provider configurations.
- `impact:message-loss`: When image understanding selects a text-only model, the inbound image description can be lost before it reaches the agent context.
- `impact:auth-provider`: The issue concerns provider/model capability metadata and media-understanding routing decisions.

## Summary

Keep open. Current main and v2026.6.6 still trust explicit configured `input: ["image"]` metadata before checking the known catalog row, and the remaining fix needs a maintainer decision on configured capability precedence.

## What This Changes

The issue asks OpenClaw to validate or diagnose media-understanding image routing when user-configured model capability metadata disagrees with bundled provider catalog capabilities.

## Best Possible Solution

Define the configured-model capability contract, then prevent known catalog text-only rows from being auto-selected for image understanding unless maintainers intentionally provide an explicit override path, while preserving resolved provider/model capability detail in failure logs.

## Reproduction Assessment

Yes, at source level. A configured provider model with `input: ["text", "image"]` is selected and registered before catalog validation, while the NVIDIA catalog row for the reported model remains text-only.

## Solution Assessment

Unclear. The requested guardrail is valid, but maintainers need to decide whether explicit configured capabilities are authoritative, subordinate to known catalog rows, or require an opt-in override before an automated fix changes routing behavior.
