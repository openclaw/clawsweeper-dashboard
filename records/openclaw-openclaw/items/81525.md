## Summary

Keep open. Current main still trusts configured `input: ["image"]` metadata for media-understanding routing and still truncates failed diagnostics, while deciding whether bundled catalog metadata should constrain user overrides is a product/config contract decision.

## What This Changes

Request validation and clearer diagnostics when media-understanding selects user-declared image-capable models whose actual or bundled catalog capabilities are text-only.

## Best Possible Solution

Keep the issue open for a maintainer decision, then add catalog-aware diagnostics or clearer rejected-image logging without changing config override authority unless that policy is deliberately approved.

## Reproduction Assessment

Yes for the OpenClaw-side path: source inspection shows configured image metadata is selected before catalog checks, the reported NVIDIA catalog row is text-only, and failed summaries strip the useful error tail. I did not live-call NVIDIA, so the external rejection itself is supported by the reporter's logs.

## Solution Assessment

Unclear. A hard catalog refusal is not clearly the best fix because current behavior gives configured model rows authority; a maintainer-approved warning, skip, or diagnostic path is the safer first design point.
