## What This Changes

The issue requests changing `createWebhookInFlightLimiter` so `maxTrackedKeys` is a hard active-key cap that rejects new keys instead of evicting in-flight counters.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84719 or an equivalent patch with regression coverage, then close this issue once the fix is on main.

## Reproduction Assessment

Yes. Source inspection against current main shows the reported `maxTrackedKeys: 1` sequence can insert key `b`, prune active key `a`, and make `release('a')` return because the counter is gone; I did not run tests because this review is read-only.

## Solution Assessment

Yes. Rejecting brand-new keys before insertion when the active key table is full is the narrow maintainable fix while preserving existing-key acquisition up to `maxInFlightPerKey`.
