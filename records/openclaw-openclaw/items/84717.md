## Summary

## What This Changes

The issue asks for `createWebhookInFlightLimiter` to preserve active counters and reject brand-new keys when `maxTrackedKeys` is full.

## Best Possible Solution

Review and update https://github.com/openclaw/openclaw/pull/84719, or an equivalent narrow patch, then close this issue after the active-counter preservation fix lands with regression coverage.

## Reproduction Assessment

Yes from source inspection. Acquiring `a`, then acquiring `b` with `maxTrackedKeys: 1` sets `b` and prunes the oldest key before `release('a')`, which then becomes a no-op because `a` is missing.

## Solution Assessment

Yes for the code shape: checking active-key capacity before inserting a brand-new key is the narrow fix that preserves active counters. It still needs maintainer acceptance of rejecting new keys at capacity because the helper is shared SDK behavior.
