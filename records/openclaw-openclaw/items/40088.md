## Summary

Keep open: current main only partially addresses the report. The watcher now logs errors and watches the memory directory directly, but default configs still have no periodic fallback, and configured interval sync still does not mark memory files dirty before scanning.

## What This Changes

The issue asks OpenClaw to keep memory_search indexes fresh when chokidar stops firing by adding watcher health/error handling and a periodic fallback.

## Best Possible Solution

Land a focused memory-core reliability fix that preserves the directory watcher work, makes interval sync actually scan memory files when configured, and settles the default interval or health-check policy explicitly.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a live chokidar-death reproduction, but yes at the source level: if watcher events stop, default intervalMinutes is 0 and interval sync does not set dirty, so changed memory files are not scanned.

## Solution Assessment

Is this the best way to solve the issue? Unclear until owner review decides the default interval policy; the narrow code direction should at least make configured interval sync effective and keep watcher error logging.
