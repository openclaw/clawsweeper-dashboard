## Summary

Keep open: current main still has the stale-index failure mode. The watcher error handler has since landed, but the default sync interval is still disabled and interval sync still depends on the watcher having already marked memory dirty; the two direct fix PRs were closed unmerged.

## What This Changes

The issue asks for memory search indexing to remain fresh when chokidar stops delivering file-change events.

## Best Possible Solution

Land a narrow memory-core fix that gives default users a lightweight periodic rescan fallback, makes interval ticks bypass the watcher-only dirty gate without forcing full reindex, preserves explicit opt-out, and keeps the existing watcher error logging.

## Reproduction Assessment

Yes from source inspection: if chokidar stops delivering events, `dirty` remains false; with the default `intervalMinutes: 0` there is no fallback, and even a configured interval currently calls sync without marking memory dirty. I did not run a multi-hour live gateway reproduction because this review is read-only.

## Solution Assessment

Yes, the best repair is the existing-config fallback approach from the prior PRs: use a conservative non-zero default interval and mark memory dirty before interval sync so the normal file listing/hash comparison catches drift. A heartbeat file is more complex and not necessary for the reported stale-index path.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/40088 on current main. Current main still defaults `agents.defaults.memorySearch.sync.intervalMinutes` to `0` in `src/agents/memory-search.ts`, and `extensions/memory-core/src/memory/manager-sync-ops.ts` interval ticks call `sync({ reason: "interval" })` without setting the memory dirty flag, so a dead watcher leaves memory files stale. Preserve the existing watcher `error` handler from #73357. Implement the remaining fallback narrowly: choose a conservative non-zero default interval (prior PRs used 15 minutes), preserve explicit `intervalMinutes: 0` as opt-out, make interval sync trigger a lightweight file listing/hash comparison rather than a full reindex, and avoid leaking `dirty` for session-only or memory-disabled configs. Add regression coverage in `src/agents/memory-search.test.ts` and `extensions/memory-core/src/memory/manager.watcher-config.test.ts`. Do not change QMD watcher semantics or recursive memory behavior beyond what this bug requires. Add a user-facing `CHANGELOG.md` fix entry.
