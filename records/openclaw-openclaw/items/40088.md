## What This Changes

The issue reports that memory_search can read from a stale memory index after watcher events stop, and requests watcher health handling or a periodic sync fallback.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/90019; close this issue only after the chosen fix or maintainer decision settles the default periodic memory freshness behavior.

## Reproduction Assessment

Yes, at source level for the remaining fallback gap: unset intervalMinutes still resolves to 0, and the interval sync path does not mark memory dirty before the dirty-gated memory sync. The exact multi-hour watcher outage was not live-reproduced, and current main partially mitigates zero-result searches by force-syncing on an empty first search.

## Solution Assessment

Unclear until maintainer review: the linked periodic fallback is the narrow source fix for watcher silence, but defaulting it on is a compatibility-sensitive product decision. The shipped zero-result search retry is useful but not a complete answer for general stale indexes.
