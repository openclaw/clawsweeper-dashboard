## Summary

Keep open: current main only covers part of the watcher failure by logging watcher errors, while the default still has no periodic fallback and configured interval sync can still skip memory files when the watcher stopped setting the dirty flag.

## What This Changes

The issue asks OpenClaw to keep memory_search indexes fresh when the chokidar memory watcher stops delivering file events.

## Best Possible Solution

Add a bounded memory sync fallback that defaults interval sync on, marks memory dirty before interval scans so existing hash-diff indexing runs, preserves explicit 0 opt-out, and covers the watcher-stopped path with focused tests.

## Reproduction Assessment

Yes. Source inspection shows a high-confidence reproduction path: after a clean sync, if no watcher events fire, dirty remains false; the interval callback calls sync without dirty/force, and runSync skips memory-file enumeration.

## Solution Assessment

Yes. The narrowest maintainable fix is the earlier unmerged shape: keep the watcher, add a periodic incremental fallback, set dirty before interval sync, and test defaults plus explicit opt-out without forcing full reindexes.
