## Summary

Keep open. Current main has partial watcher hardening, but the reported fallback gap remains: the default interval is still disabled and an interval tick still does not make memory-file sync pass the dirty gate when the watcher has stopped firing.

## What This Changes

The issue asks for memory_search to keep its index fresh when the chokidar watcher silently stops, by adding watcher health/error handling and a periodic fallback sync.

## Best Possible Solution

Add the missing lightweight periodic safety net for builtin memory sync, preserve explicit `intervalMinutes: 0` opt-out, and keep it separate from the broader recursive-memory work in the open follow-up PR.

## Reproduction Assessment

Yes, source-reproducible. With default config `intervalMinutes` resolves to `0`; even when configured, the interval path calls sync without setting `dirty`, and memory-file sync is gated on `this.dirty`.

## Solution Assessment

Yes, the prior proposed direction is the narrow fix: make the interval fallback active by default or otherwise provide a safety net, and ensure interval sync performs a lightweight file scan instead of relying on watcher-set dirty state.
