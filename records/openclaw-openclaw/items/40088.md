## Summary

Keep open: current main only partially addresses the watcher resilience path. The watcher now logs errors and watches the memory directory directly, but the default interval sync is still disabled and configured interval ticks still do not make memory files eligible for a lightweight rescan, so the stale-index fallback remains unresolved.

## What This Changes

The issue asks for memory_search to keep its index fresh when the chokidar watcher stops delivering file events, preferably with watcher health/error handling and a periodic fallback.

## Best Possible Solution

Land a narrow memory-core resilience fix that preserves explicit intervalMinutes: 0 opt-out, makes interval ticks perform a lightweight hash-scan instead of a full reindex, and keeps the existing watcher error logging and direct directory watching.

## Reproduction Assessment

No: I do not have a high-confidence live reproduction of chokidar stopping after long uptime. Source inspection does give a clear current-main reproduction of the fallback gap: with dirty false, an interval sync calls sync({ reason: "interval" }) but does not sync memory files.

## Solution Assessment

Yes: the closed replacement PRs point at the narrowest maintainable repair, but the implementation should be rebased onto current main and keep explicit intervalMinutes: 0 as an opt-out rather than forcing full reindexes.

## Repair Work Prompt

Fix the memory_search stale-index fallback from https://github.com/openclaw/openclaw/issues/40088. Current main still defaults agents.defaults.memorySearch.sync.intervalMinutes to 0 and ensureIntervalSync calls sync({ reason: "interval" }) without making memory files eligible for a lightweight hash-scan, so a dead chokidar watcher leaves memory files stale. Use the closed unmerged PRs https://github.com/openclaw/openclaw/pull/42042 and https://github.com/openclaw/openclaw/pull/65755 as prior art, but implement against current main. Preserve explicit intervalMinutes: 0 as an opt-out, do not force a full reindex on every interval, preserve session-only interval behavior, and do not broaden memory-core/plugin boundaries. Add focused regression coverage for default interval resolution and interval sync scanning memory files when the watcher did not mark dirty; keep the existing watcher error handler coverage or add it if current tests still lack it. Add a changelog entry if the final change is user-facing.
