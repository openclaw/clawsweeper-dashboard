## Summary

Keep open: current main has partial watcher hardening, but the default interval fallback is still disabled and configured interval syncs still skip memory files when no watcher event set the dirty flag, so the stale-index failure can remain.

## What This Changes

The issue asks for memory_search to avoid stale indexes when chokidar stops firing by adding watcher health/error handling and a periodic reindex fallback.

## Best Possible Solution

Built-in memory should have a tested resilience path where missed watcher events are caught by a lightweight periodic file hash comparison without forcing full re-embedding.

## Reproduction Assessment

Partly: source inspection gives a high-confidence reproduction of the fallback gap because interval sync runs without `dirty` and `runSync` then skips memory files. I did not live-reproduce chokidar's long-uptime silent stop.

## Solution Assessment

No: the current main implementation is only a partial fix because watcher error logging/direct directory watching does not address the default no-fallback path or the configured interval no-op. A maintainer should decide the default cadence, then add focused interval fallback coverage.
