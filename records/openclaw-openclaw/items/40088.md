## Summary

Keep open: current main has partial watcher hardening, but the source-visible fallback gap remains. If chokidar stops delivering events after the dirty flag has been cleared, the default interval is still disabled and even a configured interval calls `sync()` without marking memory dirty, so `syncMemoryFiles()` is skipped.

## What This Changes

The issue reports that `memory_search` can return stale results after the memory-core chokidar watcher stops firing and requests watcher health/fallback sync handling.

## Best Possible Solution

Keep the already-merged watcher hardening, then add a bounded periodic fallback that performs an incremental file-state scan when watcher events stop while preserving explicit opt-out.

## Reproduction Assessment

Yes, at source level. Current main shows that after watcher events stop, no default interval runs and a configured interval does not mark memory dirty, so the memory-file sync gate can skip indexing changed files.

## Solution Assessment

Yes, the narrow maintainable fix is to reuse the existing interval setting as an incremental fallback: set a conservative default, preserve explicit zero opt-out, mark memory dirty before interval sync, and add focused regression tests.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/40088 on current main. Keep the existing watcher hardening from the merged directory-watch and awaitWriteFinish-removal work; do not revert those changes. Implement a periodic fallback for builtin memory search using the existing `agents.defaults.memorySearch.sync.intervalMinutes` setting: default it to a conservative non-zero value such as 15 while preserving explicit `intervalMinutes: 0` opt-out, and update `ensureIntervalSync()` so interval ticks trigger an incremental memory-file scan even when the chokidar watcher has stopped firing. Prefer setting the memory dirty flag only when the memory source is enabled, then calling `sync({ reason: "interval" })`; do not use `force: true` or full atomic reindex for the periodic fallback. Add focused coverage in `src/agents/memory-search.test.ts` for the default/override behavior and in `extensions/memory-core/src/memory/manager.watcher-config.test.ts` for interval sync marking dirty when watcher events are absent. Add a concise changelog fix entry because this is user-visible memory freshness behavior.
