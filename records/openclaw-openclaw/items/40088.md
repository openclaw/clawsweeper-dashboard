## Summary

Keep open: current main and v2026.6.8 still have no default interval fallback, and interval sync can skip memory files unless watcher events set the dirty flag. The linked open PR is the current fix candidate, but it still needs maintainer judgment on the default-on periodic sync tradeoff.

## What This Changes

Request a memory_search resilience fix so missed watcher events do not leave the SQLite memory index stale, likely through interval fallback or watcher health handling.

## Best Possible Solution

Land a maintainer-approved fix that preserves explicit intervalMinutes: 0 opt-out, lets interval sync rescan memory when watcher events are missed, and documents the default or opt-in upgrade behavior.

## Reproduction Assessment

Yes, at source level: current main disables the interval fallback by default and the interval path does not mark memory dirty before syncing, so missed watcher events can leave the index stale. I did not reproduce the reporter's multi-hour chokidar outage live.

## Solution Assessment

Unclear as a product decision: the linked PR is a narrow technical fix for the fallback path, but maintainers still need to decide whether default-on periodic sync is acceptable for existing users or should remain opt-in.
