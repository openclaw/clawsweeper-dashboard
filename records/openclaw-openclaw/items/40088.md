## Summary

Keep open: current main still has no default periodic memory sync fallback and interval sync still skips memory files when the watcher has not set `dirty`, so the stale-index path remains source-reproducible. The prior fix PRs are closed unmerged, and the open recursive-memory follow-up overlaps search freshness but does not cover this dead-watcher fallback.

## What This Changes

The issue asks for memory_search to keep indexes fresh or warn when the chokidar watcher stops delivering file-change events.

## Best Possible Solution

Land a narrow memory-core fix that turns the existing interval sync into a lightweight default fallback, preserves explicit `intervalMinutes: 0` opt-out, marks memory dirty before non-force interval sync, and adds regression coverage.

## Reproduction Assessment

Yes, source-reproducible on current main. With default config, interval sync is disabled, search sync depends on `dirty`, and interval sync would not mark memory dirty even if enabled, so a watcher that silently stops leaves modified files out of the index.

## Solution Assessment

Yes, the narrowest maintainable fix is to reuse the existing interval sync path as a lightweight safety net rather than forcing full reindexes or adding a new watcher heartbeat subsystem first. The repair should keep explicit opt-out and avoid changing unrelated QMD recursive-search work.

## Repair Work Prompt

Fix the memory_search stale-index fallback reported at https://github.com/openclaw/openclaw/issues/40088. Use the existing `agents.defaults.memorySearch.sync.intervalMinutes` path as a lightweight fallback: make the unset default non-zero, preserve explicit `intervalMinutes: 0` opt-out, and ensure interval sync marks memory files dirty before calling non-force `sync({ reason: "interval" })` so `syncMemoryFiles` can compare hashes after watcher events stop. Do not use `force: true` for the interval path, do not add a new config/env option, and do not fold unrelated recursive/QMD watcher changes from https://github.com/openclaw/openclaw/pull/72748 into this fix. Add focused regression tests for the default/explicit override behavior and for interval sync after a dead watcher; keep the existing watcher error-handler coverage green. Add a user-facing fix changelog entry if this repair branch is prepared for landing.
