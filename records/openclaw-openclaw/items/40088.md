## Summary

Current main only partially addresses the report. Memory-core now logs watcher errors and has related watcher reliability fixes, but the stale-index fallback is still missing: the default sync interval remains disabled and a configured interval tick still does not mark memory files dirty before the sync gate.

## What This Changes

The issue reports that memory_search can stop updating when chokidar stops delivering events, leaving the SQLite memory index stale without a periodic fallback.

## Best Possible Solution

Keep the issue open and decide whether memory-core should ship a non-zero default fallback; at minimum, configured interval sync should actively rescan memory files without forcing a full reindex.

## Reproduction Assessment

Yes for the fallback gap from source inspection: interval sync remains disabled by default, and configured interval ticks still do not satisfy the dirty gate for memory-file sync. The exact long-running chokidar silent-stop trigger was not live-reproduced in this read-only review.

## Solution Assessment

Unclear without maintainer judgment on the default cadence. Marking configured interval sync dirty is the narrow code fix, while changing intervalMinutes from 0 to 15 is likely the product/performance decision needed to solve the default-config report.
