## Summary

Keep open: current main and v2026.6.10 still leave the reported stale-index fallback unresolved. Main has partial watcher hardening, but unconfigured memory search still defaults interval sync to disabled and interval sync does not force a memory rescan after missed watcher events; the open linked PR is the candidate fix path.

## What This Changes

The issue asks for memory_search to avoid stale indexes when memory file watcher events stop, using watcher health/error handling and/or a nonzero periodic sync fallback.

## Best Possible Solution

Review and repair the open linked PR, or replace it with an equivalent maintainer-sponsored fix, so missed watcher events trigger a bounded memory rescan with an explicit decision on 30-minute default versus opt-in interval sync.

## Reproduction Assessment

No live high-confidence reproduction was run for the multi-hour watcher outage. Source inspection does show the failure path: if watcher events are missed, dirty remains false, default interval sync is disabled, and search does not force a rescan once indexed content already exists.

## Solution Assessment

Unclear as a default-on change: the manager interval/config resolver surfaces are the right narrow place to fix this, but maintainers still need to accept the default periodic fallback or keep it opt-in with documented remediation.
