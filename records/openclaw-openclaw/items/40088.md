## Summary

Keep open: current main has partial watcher hardening, but the fallback gap that leaves memory_search stale after watcher events stop is still present in source.

## What This Changes

The issue asks OpenClaw to keep the memory_search index fresh when the chokidar-backed memory watcher stops delivering file-change events.

## Best Possible Solution

Land a focused memory-core fix that makes the existing interval fallback actually rescan memory files, gives default users a bounded safety net, and preserves the current direct-directory watcher and error-handler improvements.

## Reproduction Assessment

Yes at source level: if the watcher stops while `dirty` is false, current main has `intervalMinutes` disabled by default, and even a configured interval calls sync without setting `dirty`. I did not run a multi-hour live chokidar failure reproduction.

## Solution Assessment

Yes, the narrowest maintainable path is to repair the existing sync fallback contract: make interval ticks trigger a lightweight file comparison and keep explicit opt-out behavior. A heartbeat-file probe can remain a later enhancement unless the interval fallback proves insufficient.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/40088 on current main. Current main still defaults `agents.defaults.memorySearch.sync.intervalMinutes` to 0 and `ensureIntervalSync()` calls `sync({ reason: "interval" })` without setting the memory dirty flag, so a dead chokidar watcher leaves memory files unscanned. Implement the bounded fallback shape from the closed unmerged repair attempts: make interval sync perform a lightweight memory-file comparison when memory files are enabled, choose a safe nonzero default interval unless an explicit config value including 0 is set, and preserve explicit `intervalMinutes: 0` opt-out. Keep the existing direct-directory watcher and watcher error handler; do not reintroduce `memory/**/*.md` glob watching or chokidar `awaitWriteFinish` polling. Add/update regression coverage in `src/agents/memory-search.test.ts` and `extensions/memory-core/src/memory/manager.watcher-config.test.ts`; include a single-line changelog fix entry if the final change is user-visible.
