## Summary

Keep open: current main only partially addresses memory watcher resilience; default periodic sync remains disabled and configured interval sync can still skip memory files when the watcher has stopped marking the index dirty.

## What This Changes

The issue asks for `memory_search` indexes to remain fresh when the chokidar watcher stops delivering events, with fallback syncing, health checks, or warning logs.

## Best Possible Solution

Keep the issue open until memory-core has a maintainer-approved resilience path that preserves memory index freshness after watcher stalls, with focused tests for the chosen fallback or heartbeat behavior and a changelog entry.

## Reproduction Assessment

No high-confidence live reproduction was established for the multi-hour chokidar stall. Source inspection does show the missing default fallback and dirty-gated interval path that would keep the index stale after a silent watcher stop.

## Solution Assessment

Unclear. The current main changes are useful partial fixes, but a complete solution still needs a maintainer decision between a nonzero periodic fallback, watcher heartbeat/restart behavior, or another bounded freshness mechanism.
