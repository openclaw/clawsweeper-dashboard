## Summary

Keep open: this PR is a new, broad feature proposal that adds a home-control plugin plus a first-class kiosk UI, and it still needs product/security review, contributor real behavior proof, and fixes for source-level review findings.

## What This Changes

The PR adds a bundled Home Assistant WebSocket bridge plugin, a `#/kiosk` Lit dashboard for a Wagner Way wall-tablet view, plan documentation, and focused tests.

## Best Possible Solution

Move the generic Home Assistant bridge toward an external ClawHub plugin, and only add core UI/API seams after maintainer design for plugin-owned dashboards; remove household-specific defaults and require live proof before reconsidering merge.

## Reproduction Assessment

Yes for the patch findings: PR-head source inspection shows undeclared `KioskWagnerWay` fields that should fail strict TypeScript. No for the live feature path: the PR body explicitly says the real gateway plus Home Assistant user test is still unchecked.

## Solution Assessment

No. Bundling a household-specific `#/kiosk` dashboard and home-control bridge into core is not the narrowest maintainable path; the safer direction is an external plugin plus a generic Control UI extension discussion if existing descriptors are insufficient.
