## Summary

Keep open: current main and the latest release still have topic-only Telegram ACP configured binding behavior, but this draft PR is not merge-ready because the new positive-id path ignores `match.peer.kind` and contributor real behavior proof is still missing. ## What This Changes

The PR broadens Telegram configured ACP binding normalization and matching so positive direct-chat ids can target ACP bindings, extracts the Telegram binding adapter for focused tests, and updates gateway configuration docs.

## Best Possible Solution

Land one canonical Telegram direct-message ACP binding fix that only activates `direct` or legacy `dm` peer bindings, preserves topic/group behavior, includes focused tests/docs, and has redacted real Telegram proof.

## Reproduction Assessment

Yes. By source inspection, current main passes Telegram DMs into configured binding resolution as a positive peer id with no parent, while the current Telegram ACP normalizer and matcher only accept topic-shaped ids.

## Solution Assessment

No. The branch is a plausible fix direction, but the best fix must gate the direct path on `match.peer.kind: direct` or legacy `dm`, preserve topic behavior, and resolve the overlapping proof-backed implementation before merge.
