## Summary

Keep open: the PR targets a valid Telegram ACP DM binding gap, but it is still draft/conflicting, lacks after-fix real Telegram proof, and the diff introduces two merge-blocking issues around positive topic-shaped IDs and stale docs placement.

## What This Changes

The branch changes Telegram configured ACP binding normalization/matching to accept positive direct-chat IDs, exports the Telegram bindings adapter for focused tests, adds adapter tests, and updates one gateway configuration reference line.

## Best Possible Solution

Rebase the PR, keep the Telegram-owned direct-chat binding support, preserve rejection of positive topic-shaped IDs, update the current Telegram/ACP/channel docs, and add redacted real Telegram proof before merge.

## Reproduction Assessment

Yes by source inspection. A top-level Telegram ACP binding with `match.peer.kind: "direct"` and a positive peer id reaches configured binding resolution on current main, but the Telegram adapter returns `null` and the compiler skips the rule.

## Solution Assessment

No as submitted. The Telegram adapter is the right boundary, but the patch must keep positive `chatId:topic:topicId` shapes rejected, move docs to the current pages, and provide real Telegram proof.
