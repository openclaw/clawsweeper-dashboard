## What This Changes

The PR changes Telegram's configured ACP binding adapter plus docs/tests so top-level ACP bindings can compile and match plain positive direct-chat IDs in addition to forum topic IDs.

## Best Possible Solution

Land one refreshed canonical Telegram direct-peer ACP binding implementation that only matches direct/dm peers, preserves forum-topic behavior, updates docs/tests, and includes redacted live Telegram ACP proof.

## Reproduction Assessment

Yes, by source inspection: Telegram DMs enter configured-binding resolution as direct peers with no parent, while current main and v2026.6.6 only compile topic-shaped Telegram ACP IDs. I did not run tests because this review is read-only and local tests can create artifacts.

## Solution Assessment

No as submitted: the PR targets the right Telegram binding provider, but direct matching should be gated on direct or legacy dm peer kinds, then refreshed with real Telegram proof before it is the best fix.
