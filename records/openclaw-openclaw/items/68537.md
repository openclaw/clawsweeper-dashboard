## Summary

Keep open: current main still rejects direct-chat Telegram configured ACP bindings, so this PR targets real remaining work. The proposed patch is not merge-ready as-is because the extracted adapter loses TypeScript contextual typing, and the docs/changelog need to be rebased onto the current docs structure.

## What This Changes

The branch extracts Telegram ACP binding matching into an exported adapter, accepts positive direct-chat IDs for configured ACP bindings while preserving topic matching and bare negative group rejection, adds adapter tests, and updates one ACP binding docs sentence.

## Best Possible Solution

Land a rebased Telegram-channel fix, or a narrow replacement PR if the source branch is not practical to update, that keeps the fix inside the Telegram bindings adapter, annotates the exported adapter type, preserves direct/topic/group-rejection coverage, updates the current public docs surfaces, and adds the required changelog entry. After that lands, the related #62769 issue can be closed as implemented.

## Reproduction Assessment

Yes. Configure a top-level Telegram ACP binding with `match.peer.id: "123456789"`; current main passes that id to the Telegram provider, `parseTelegramTopicConversation` cannot compile it without a parent, `normalizeTelegramAcpConversationId` returns null, and the registry skips the binding.

## Solution Assessment

No as-is. The Telegram channel adapter is the right boundary for the fix, but the current patch needs a typed adapter extraction, current-doc alignment, and changelog coverage before it is the narrow maintainable solution.
