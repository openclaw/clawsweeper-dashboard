## What This Changes

The branch changes Telegram configured ACP binding normalization and matching to accept positive direct-chat IDs, adds adapter tests, and updates one gateway configuration reference line.

## Best Possible Solution

Rebase or replace this PR with a narrow Telegram binding adapter change that accepts plain positive direct-chat IDs, preserves rejection of whole-group and positive topic-shaped IDs, updates the current Telegram/ACP docs, and includes redacted real Telegram proof.

## Reproduction Assessment

Yes. By source inspection, a top-level Telegram ACP binding with match.peer.kind direct and a positive match.peer.id reaches configured binding resolution on current main, but the Telegram adapter returns null and the compiler skips the rule.

## Solution Assessment

No as submitted. The Telegram adapter is the right boundary, but the patch must keep positive chatId:topic:topicId shapes rejected, move docs to the current user-facing pages, and provide real Telegram proof.
