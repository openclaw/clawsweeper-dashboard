## Summary

Keep open: this PR targets a real Gateway display path, but it is not ready because the patch suppresses a raw user-visible label and the contributor has not provided after-fix real behavior proof.

## What This Changes

The PR changes Gateway session-row display-name projection to ignore `origin.label` when it equals the heartbeat fallback sender string.

## Best Possible Solution

Land a scoped Gateway row-projection guard that ignores synthetic or stale heartbeat-origin metadata while preserving legitimate labels named `heartbeat`, with focused coverage and redacted real behavior proof.

## Reproduction Assessment

Yes, at source level: a current-main row with no `displayName` or `label` and `origin.label: "heartbeat"` still projects that value through `buildGatewaySessionRow`, and the Control UI label resolver can render it. I did not run a live WebChat session, and current main prevents new heartbeat system events from writing this metadata.

## Solution Assessment

No, not as currently written. The row-projection surface is reasonable for stale metadata, but raw string suppression should be replaced with a predicate tied to synthetic heartbeat provenance or stale origin shape.
