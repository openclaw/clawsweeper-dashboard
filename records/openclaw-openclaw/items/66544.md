## What This Changes

The PR changes `src/gateway/session-utils.ts` so `buildGatewaySessionRow` ignores `origin.label` when it equals the heartbeat fallback sender string.

## Best Possible Solution

Land a scoped Gateway row-projection hardening that ignores only synthetic or stale heartbeat origin metadata, preserves legitimate labels named `heartbeat`, and covers list and live row callers.

## Reproduction Assessment

Yes, source-level: current main still falls through to `origin.label`, so a row with no `displayName` or `label` and `origin.label: "heartbeat"` will display that label. I did not execute tests in this read-only review.

## Solution Assessment

No. The shared row builder is the right surface, but filtering every literal `heartbeat` is too broad; the fix should key off synthetic or stale heartbeat provenance instead.
