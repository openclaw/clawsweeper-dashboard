## Summary

Keep open: this is a plausible stale Gateway display-name hardening path, but the current patch should not merge because it suppresses legitimate labels named `heartbeat` and the PR lacks after-fix real behavior proof.

## What This Changes

The PR changes `src/gateway/session-utils.ts` so `buildGatewaySessionRow` ignores `origin.label` when it equals the heartbeat fallback sender string.

## Best Possible Solution

Land an equivalent scoped Gateway row-projection hardening that ignores only synthetic or stale heartbeat origin metadata, preserves legitimate labels named `heartbeat`, and covers both list and live row callers.

## Reproduction Assessment

Yes, as a source-level path: a row with no `displayName` or `label` and `origin.label: "heartbeat"` still returns that origin label from current-main row projection. I did not execute tests in this read-only sweep.

## Solution Assessment

No. The shared row projection is the right surface, but a raw string filter is overbroad; the fix should identify synthetic or stale heartbeat provenance instead of suppressing every matching label.
