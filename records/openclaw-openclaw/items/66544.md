## Summary

Keep this PR open: it targets a real Gateway/WebChat display-name path, but the current patch suppresses every origin label equal to `heartbeat` and still lacks contributor after-fix real behavior proof.

## What This Changes

The branch changes `buildGatewaySessionRow` to ignore `origin.label` when it equals the heartbeat fallback sender string.

## Best Possible Solution

Land a scoped Gateway row-projection guard that ignores only synthetic or stale heartbeat origin metadata, preserves legitimate labels named `heartbeat`, and includes focused coverage plus real behavior proof.

## Reproduction Assessment

Yes, at source level: a current-main session row with no `displayName` or `label` and `origin.label: "heartbeat"` still displays that origin label. I did not run live WebChat or tests in this read-only review.

## Solution Assessment

No. The shared row projection is the right surface for stale display metadata, but a raw string filter is overbroad because direct conversation labels can legitimately be `heartbeat`.
