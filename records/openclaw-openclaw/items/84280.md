## Summary

Keep open: the Windows SIGUSR1 bug is still present on current main and v2026.6.9, but this branch is not merge-ready because the new unmanaged win32 path can terminate or falsely validate the active listener before a replacement restart is proven, and the supplied proof does not exercise that changed path.

## What This Changes

The PR guards Windows SIGUSR1 signaling and routes unmanaged Windows gateway restart through the scheduled-task restart helper, with focused unit tests.

## Best Possible Solution

Keep the SIGUSR1 guard, but make Windows unmanaged restart either preserve or preflight the selected PID and prove a replacement before success, or fail with install guidance without touching the running gateway.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.9 send SIGUSR1 to a verified unmanaged gateway PID, while the Node v22.19 Windows signal contract excludes SIGUSR1 and makes SIGTERM unsuitable for graceful cross-process restart. I did not run a live unmanaged Windows reproduction in this read-only pass.

## Solution Assessment

No. The SIGUSR1 guard is useful, but calling triggerOpenClawRestart in the unmanaged win32 branch is not the best fix until it protects or preflights the selected listener and proves the changed branch in a real Windows setup.
