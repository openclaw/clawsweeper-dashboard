- `merge-risk: 🚨 compatibility`: The PR changes the restart mechanism for an existing CLI path on Windows and may alter how unmanaged setups behave during upgrade.

## Summary

Keep this PR open, but it needs changes before merge: the current-main bug is real, yet the proposed Windows unmanaged path can report success without actually restarting the running gateway.

## What This Changes

This PR changes Windows unmanaged gateway restart handling to use the scheduled-task restart path instead of sending SIGUSR1, and adds tests for the Windows guard and restart path.

## Best Possible Solution

Windows unmanaged restart should reach the running gateway through an authenticated in-process restart request, or fail clearly when that setup cannot be restarted safely without a service manager.

## Reproduction Assessment

Yes at source level: current main sends SIGUSR1 through `process.kill` from the unmanaged restart path, and that signal path is not a Windows process-control mechanism. I did not run a live Windows repro in this read-only review.

## Solution Assessment

No. The PR avoids the unknown-signal throw, but `triggerOpenClawRestart()` is a scheduled-task handoff and is not a reliable unmanaged restart mechanism from the branch reached when Windows service-manager registration is missing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Route unmanaged Windows restarts to the running gateway:** `src/cli/daemon-cli/lifecycle.ts:216`
  - body: `restartGatewayWithoutServiceManager` only runs after `runServiceRestart` has determined the Windows Scheduled Task/startup entry is missing. Calling `triggerOpenClawRestart()` here does not signal the verified listener; it only launches the scheduled-task handoff helper, which can return success after spawning a script even when no task exists. The existing gateway can stay healthy, so the post-restart check can pass against the same process and report `restarted` without restarting anything. This path needs to request restart from the running gateway or fail clearly instead of treating the scheduled-task handoff as the unmanaged restart mechanism.
  - confidence: 0.88
