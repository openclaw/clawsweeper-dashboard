## Summary

Keep open: the PR targets a real Windows unmanaged gateway restart bug, but the proposed Windows branch can still harm availability and the supplied proof does not exercise the changed no-service-manager path.

## What This Changes

The PR guards Windows SIGUSR1 signaling and routes unmanaged Windows gateway restart through the scheduled-task restart handoff, with focused lifecycle and process tests.

## Best Possible Solution

Keep the SIGUSR1 guard, but make the Windows unmanaged restart path preserve or preflight the selected PID and prove a replacement before reporting success, or fail with install guidance without touching the running gateway.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.9 send SIGUSR1 to a verified unmanaged gateway PID, while the Node Windows signal contract excludes SIGUSR1 and makes SIGTERM unsuitable for graceful cross-process restart. I did not run a live unmanaged Windows reproduction in this read-only pass.

## Solution Assessment

No. The SIGUSR1 guard is useful, but calling triggerOpenClawRestart in the unmanaged win32 branch is not the best fix until it protects or preflights the selected listener and proves the changed branch in a real Windows setup.
