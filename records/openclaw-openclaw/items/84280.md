## Summary

Keep open: the Windows SIGUSR1 problem is real, but this patch can report a successful unmanaged Windows restart without restarting the discovered gateway process, because the scheduled-task helper does not target that PID and the post-check only proves a listener is still healthy. The PR also needs real Windows behavior proof before merge.

## What This Changes

The PR adds a Windows SIGUSR1 guard in gateway process signaling, routes Windows unmanaged gateway restart through the existing scheduled-task restart helper, and adds focused unit tests.

## Best Possible Solution

Revise the Windows unmanaged path so it either asks the running gateway itself to restart and verifies the targeted process behavior, or fails with a clear unsupported-path message instead of using a non-targeted scheduled-task handoff.

## Reproduction Assessment

No live Windows repro was run, but source inspection gives a clear current-main failure path: unmanaged restart on win32 reaches process.kill(pid, "SIGUSR1"), while Node’s Windows signal contract does not support SIGUSR1 as an external signal.

## Solution Assessment

No. Guarding SIGUSR1 is useful, but routing an unmanaged listener through the scheduled-task supervisor path is not the best fix because it does not target or verify the discovered PID.
