## Summary

Keep open: current main and the latest release still have the unmanaged Windows SIGUSR1 failure, but this PR is not merge-ready because its Windows handoff can stop the active unmanaged listener before a replacement gateway is proven and the supplied proof does not exercise that changed path.

## What This Changes

The PR adds a Windows SIGUSR1 guard and routes unmanaged Windows gateway restarts through `triggerOpenClawRestart()` with focused unit tests.

## Best Possible Solution

Keep the SIGUSR1 guard, but make the win32 unmanaged restart path preflight or protect the handoff before cleanup, or fail without stopping the active listener, then add redacted real Windows proof for that unmanaged path.

## Reproduction Assessment

Yes, source-level: current main and `v2026.6.8` send `SIGUSR1` to a verified unmanaged gateway PID, while Node's Windows signal contract makes `SIGUSR1` invalid and `SIGTERM` unsafe for this cross-process restart substitute. I did not run a live Windows unmanaged restart in this read-only review. ([nodejs.org](https://nodejs.org/api/process.html?utm_source=openai))

## Solution Assessment

No. The SIGUSR1 guard is useful, but routing the unmanaged win32 branch through `triggerOpenClawRestart()` is not the best fix until the handoff is proven before cleanup or the path fails without stopping the active listener.
