## Summary

Keep this PR open. The Windows SIGUSR1 bug is real on current main and the latest release, but the proposed unmanaged Windows fallback still has an availability bug and the supplied proof covers the managed Scheduled Task path, not the changed unmanaged path.

## What This Changes

The PR guards Windows SIGUSR1 signaling and routes unmanaged Windows gateway restart through the existing scheduled-task restart helper with focused unit tests.

## Best Possible Solution

Keep the SIGUSR1 guard, but change the Windows unmanaged restart path to protect or preflight the active listener and prove a replacement restart before reporting success, then add redacted real Windows proof for that unmanaged path.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.8 send SIGUSR1 to a verified unmanaged gateway PID, and the Node Windows signal contract makes SIGUSR1 invalid for that cross-process path. I did not run a live unmanaged Windows restart in this read-only review.

## Solution Assessment

No. The SIGUSR1 guard is useful, but routing the unmanaged win32 branch through triggerOpenClawRestart() is not the best fix until the handoff is protected and the changed path is proven in a real Windows setup.
