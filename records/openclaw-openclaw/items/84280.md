## Summary

Keep this PR open, but do not merge it as-is: it fixes a real Windows SIGUSR1 gap, yet the proposed unmanaged Windows handoff can terminate the verified gateway listener before a replacement is proven, and the supplied real proof covers only the managed Scheduled Task path.

## What This Changes

The PR adds a Windows SIGUSR1 guard and reroutes unmanaged Windows gateway restart through the existing scheduled-task restart helper, with focused lifecycle and gateway-process tests.

## Best Possible Solution

Keep the Windows SIGUSR1 guard, but make the unmanaged Windows restart path protect or avoid terminating the current listener until a replacement handoff is proven, then require redacted real Windows proof for that exact no-service-manager path.

## Reproduction Assessment

Yes from source inspection: the PR calls triggerOpenClawRestart() from the unmanaged Windows branch, and that helper calls stale-process cleanup without protecting the verified target PID. I did not run a live Windows reproduction in this read-only review.

## Solution Assessment

No. Rerouting away from SIGUSR1 is the right direction, but using the existing helper unchanged is not the best fix because that helper is designed to clean stale listeners before supervised restart and can kill the unmanaged listener first.
