## Summary

Keep open: current main and the latest release still send SIGUSR1 for the unmanaged Windows gateway restart path, but this PR is not merge-ready because the new win32 branch can stop the active listener before the detached scheduled-task handoff is guaranteed, and the supplied proof does not exercise that unmanaged branch.

## What This Changes

The PR adds a Windows SIGUSR1 guard and routes unmanaged Windows gateway restarts through the existing scheduled-task restart helper with focused unit tests.

## Best Possible Solution

Keep the SIGUSR1 guard, but make the Windows unmanaged path preflight or guarantee the correct handoff before any listener cleanup, or fail clearly while leaving the running gateway alive, then add redacted real Windows proof for that no-service-manager path.

## Reproduction Assessment

Yes by source inspection, not live: current main and `v2026.6.8` send SIGUSR1 to a verified unmanaged gateway PID, while Node's Windows signal contract makes SIGUSR1 invalid and SIGTERM unsafe for this cross-process restart substitute. ([nodejs.org](https://nodejs.org/api/process.html))

## Solution Assessment

No. The SIGUSR1 guard is useful, but routing the unmanaged win32 branch through `triggerOpenClawRestart()` is not the best fix until the handoff is proven before cleanup or the path fails without stopping the active listener.
