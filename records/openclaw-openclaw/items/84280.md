## Summary

Keep open: the SIGUSR1 guard is useful, but the Windows unmanaged restart branch is not merge-ready because it calls a helper that can clean the selected listener before proving a replacement, and the supplied proof does not exercise that changed unmanaged path.

## What This Changes

The PR routes Windows unmanaged gateway restart through the existing scheduled-task restart helper instead of SIGUSR1, adds a Windows SIGUSR1 guard, and adds focused unit tests.

## Best Possible Solution

Keep the SIGUSR1 guard, but make the Windows unmanaged restart path either preserve or preflight the selected PID until a replacement is proven, or fail with install guidance while leaving the active gateway alive; then add real Windows proof for that exact path.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.9 send SIGUSR1 to a verified unmanaged gateway PID, while Node's Windows signal contract excludes SIGUSR1 and makes cross-process SIGTERM unsuitable for graceful restart. I did not run a live unmanaged Windows gateway in this read-only pass.

## Solution Assessment

No. The SIGUSR1 guard is useful, but calling triggerOpenClawRestart from the unmanaged win32 branch is not the best fix until it protects or preflights the selected listener and proves the changed branch in a real Windows setup.
