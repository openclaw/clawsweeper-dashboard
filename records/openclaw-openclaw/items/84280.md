## Summary

Keep open: this PR targets a real source-reproducible Windows SIGUSR1 restart bug, but the current diff can report success after killing an unmanaged gateway before proving the Windows scheduled-task or fallback handoff can relaunch it, and the supplied Windows proof only covers the managed service path.

## What This Changes

The PR adds a Windows SIGUSR1 guard and routes unmanaged Windows gateway restart through `triggerOpenClawRestart()` with focused unit tests.

## Best Possible Solution

Keep the SIGUSR1 guard, but make win32 unmanaged restart validate a scheduled-task or fallback relaunch path before terminating the verified PID, or fail clearly without touching the running gateway, then prove that unmanaged Windows path in a real environment.

## Reproduction Assessment

Yes, by source inspection and Node's signal contract: current main sends `SIGUSR1` to an unmanaged gateway PID, while Windows does not support that signal. I did not run a live Windows reproduction in this read-only review.

## Solution Assessment

No. The SIGUSR1 guard is good, but directly using `triggerOpenClawRestart()` from the unmanaged Windows path is not safe until the relaunch handoff is proven before the listener can be killed.
