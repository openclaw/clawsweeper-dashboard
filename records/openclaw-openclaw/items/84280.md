## Summary

Keep open: current main and the latest release still use SIGUSR1 on the unmanaged gateway restart path, but this PR is not merge-ready because the new Windows handoff can clean or validate the selected listener before a replacement is proven, and the supplied proof covers the managed Scheduled Task path rather than the changed unmanaged branch.

## What This Changes

The branch adds a Windows SIGUSR1 guard and routes unmanaged Windows gateway restart through `triggerOpenClawRestart()`, with focused lifecycle and gateway-process tests.

## Best Possible Solution

Keep the SIGUSR1 guard, but make the Windows unmanaged restart path preserve or preflight the selected PID and prove a replacement before success, or fail with install guidance without touching the running gateway.

## Reproduction Assessment

Yes, source-level: current main and `v2026.6.9` send `SIGUSR1` to a verified unmanaged gateway PID, while Node's Windows signal contract excludes safe cross-process SIGUSR1/SIGTERM restart. I did not run a live unmanaged Windows reproduction in this read-only review.

## Solution Assessment

No. The SIGUSR1 guard is useful, but calling `triggerOpenClawRestart()` in the unmanaged Windows branch is not the best fix until it protects or preflights the selected listener and proves the changed branch in a real Windows setup.
