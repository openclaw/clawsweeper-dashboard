## Summary

Keep open: the Windows SIGUSR1 bug is real, but this PR is not merge-ready because the proposed unmanaged Windows path can stop the running gateway before proving a replacement starts, and the supplied proof does not exercise that changed path.

## What This Changes

The PR adds a Windows SIGUSR1 guard and reroutes unmanaged `openclaw gateway restart` on Windows through the scheduled-task restart handoff, with focused unit tests.

## Best Possible Solution

Keep the SIGUSR1 guard, but make unmanaged Windows restart either validate a supervised handoff before listener cleanup or fail clearly while leaving the gateway alive, then prove the actual unmanaged Windows flow.

## Reproduction Assessment

Yes by source inspection: current main sends `SIGUSR1` to an unmanaged gateway PID, and the Node Windows signal contract makes that path invalid. I did not run a live Windows repro in this read-only review.

## Solution Assessment

No. The guard is useful, but routing unmanaged Windows restart through `triggerOpenClawRestart()` reuses a helper that performs listener cleanup before the detached handoff is proven.
