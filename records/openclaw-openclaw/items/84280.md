## Summary

This PR addresses a real Windows restart bug, but it is not merge-ready because the new unmanaged Windows branch can route through stale-listener cleanup before proving a replacement, and the supplied proof still covers the managed Scheduled Task path rather than the changed unmanaged path.

## What This Changes

The branch guards Windows SIGUSR1 signaling and routes unmanaged Windows gateway restarts through the existing Scheduled Task restart helper with focused unit tests.

## Best Possible Solution

Keep the SIGUSR1 guard, but change the unmanaged win32 restart path to protect the selected listener or fail before cleanup, then prove the exact unmanaged Windows path with redacted terminal output, logs, or a recording.

## Reproduction Assessment

Yes for the original bug from source: current main and v2026.6.9 send SIGUSR1 to a verified unmanaged gateway PID, while Node's Windows signal contract makes that path invalid. I did not establish a live unmanaged Windows repro in this read-only review.

## Solution Assessment

No, not yet. The SIGUSR1 guard is useful and the Scheduled Task handoff is safer than SIGTERM, but the current PR should not route an unmanaged listener through cleanup without proving that the selected PID is safely replaced.
