## Summary

Keep open: the SIGUSR1 bug is real, but the current PR still has an availability blocker because the Windows unmanaged path can terminate the live gateway before proving that the scheduled-task/fallback handoff will relaunch it.

## What This Changes

The PR adds a Windows SIGUSR1 guard in gateway process signaling and routes the no-service-manager gateway restart path to triggerOpenClawRestart() on win32, with focused unit tests.

## Best Possible Solution

Keep the SIGUSR1 guard, but make Windows unmanaged restart validate a supervised handoff before terminating the listener or fail clearly while leaving the gateway alive, then prove the actual unmanaged Windows flow.

## Reproduction Assessment

Yes. Current main's unmanaged restart path sends SIGUSR1 through process.kill, and Node's Windows signal contract makes that path invalid; I did not run a live Windows repro in this read-only review.

## Solution Assessment

No. The guard is useful, but routing the unmanaged Windows path through triggerOpenClawRestart() reuses a helper that kills verified listeners before an asynchronous scheduled-task/fallback script proves relaunch success.
