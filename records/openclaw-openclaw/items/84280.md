## Summary

Keep open: current main and the latest release still have the Windows SIGUSR1 unmanaged-restart bug, but this PR is not merge-ready because the new win32 unmanaged branch can route through stale-process cleanup before a replacement gateway is proven, and the supplied proof covers the managed Scheduled Task path rather than the changed unmanaged path.

## What This Changes

This PR adds a Windows SIGUSR1 guard, routes unmanaged Windows gateway restart through `triggerOpenClawRestart()`, and adds focused tests for both paths.

## Best Possible Solution

Keep the SIGUSR1 guard, but change the Windows unmanaged restart path to protect or preflight the selected listener, fail safely if the handoff cannot be guaranteed, and add redacted real Windows proof for that exact path.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: current main and `v2026.6.8` send SIGUSR1 to a verified unmanaged gateway PID, while Node documents Windows signal limitations; I did not run a live unmanaged Windows restart.

## Solution Assessment

No. The SIGUSR1 guard is useful, but routing the unmanaged win32 branch through `triggerOpenClawRestart()` is not the best fix until the selected listener is protected or preflighted and the changed path is proven in a real Windows setup.
