## Summary

Keep open: current main still routes every disconnected Control UI render to the login gate, and open PR #61345 remains the active unmerged implementation candidate for preserving the dashboard during reconnects.

## What This Changes

Request that the Control UI preserve prior dashboard/chat content during transient gateway disconnects by disabling active controls and offering an explicit reconnect path instead of immediately replacing the whole UI with the login gate.

## Best Possible Solution

Review and land PR #61345, or an equivalent Control UI patch that sends only true login-required auth/config/device-auth failures to the login gate while keeping the dashboard shell visible during reconnect states.

## Reproduction Assessment

Yes. On current main, `connectGateway` sets `host.connected = false` during reconnect/close paths, and `renderApp` immediately returns the login gate for any `!state.connected` before the dashboard or chat branch can render.

## Solution Assessment

Yes. The narrow maintainable solution is to distinguish login-required failures from transient reconnect states in the Control UI gate, then rely on existing disconnected-control behavior inside the dashboard instead of changing gateway protocol semantics.
