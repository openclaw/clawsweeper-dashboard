## Summary

Keep open: current main still routes every disconnected Control UI render through the login gate, and the active fix path is the open related PR rather than closing this issue early.

## What This Changes

The issue requests keeping prior Control UI content readable during gateway disconnects instead of replacing the whole UI with the login gate.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/61345, or an equivalent narrow Control UI change that sends only true login-required failures to the login gate while preserving the dashboard shell during reconnects.

## Reproduction Assessment

Yes, at source level. Current main sets `connected = false` during reconnect/close paths, and `renderApp` immediately returns the login gate for any disconnected state.

## Solution Assessment

Yes, the narrow maintainable direction is to distinguish login-required auth/config/device failures from transient reconnect states at the Control UI gate. The open related PR follows that shape but still needs normal PR review and proof before merge.
