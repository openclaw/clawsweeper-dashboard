## Summary

Current `main` still renders the login gate for every disconnected Control UI state, so the requested behavior remains unresolved. The related open PR #61345 is the focused implementation candidate, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Request the Control UI keep the readable dashboard/chat shell available during gateway disconnects, disable active controls, and route to the login gate only when re-authentication is needed.

## Best Possible Solution

Land PR #61345 or an equivalent Control UI change that keeps the shell readable during reconnects while retaining the login gate for high-confidence auth/config/device-auth failures.

## Reproduction Assessment

Yes. On current `main`, any path that sets `state.connected` to false reaches the `renderApp` gate and replaces the dashboard with `renderLoginGate`, which matches the reported workflow disruption.

## Solution Assessment

Unclear. The open PR’s state split looks like the right narrow direction, but maintainers should verify whether it fully covers the requested disabled-controls and reconnect-button UX before closing this issue.
