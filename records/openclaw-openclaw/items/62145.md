## Summary

Keep open: current main still replaces the Control UI with the login gate for every disconnected state, and the related implementation candidate remains open and unmerged.

## What This Changes

The issue requests keeping prior Control UI messages visible during transient gateway disconnects instead of replacing the whole app with the login gate.

## Best Possible Solution

Review and land PR #61345, or an equivalent Control UI change that routes only true login-required failures to the login gate while keeping the dashboard shell visible for transient reconnects.

## Reproduction Assessment

Yes. A source-level reproduction is clear: force `state.connected` false through gateway connect/close and `renderApp` returns the login gate before rendering dashboard content; no browser/live test was run in this read-only pass.

## Solution Assessment

Yes. Distinguishing login-required auth/config/device-auth failures from transient reconnect states in the UI gate is the narrow maintainable direction, with existing dashboard controls already able to disable themselves when disconnected.
