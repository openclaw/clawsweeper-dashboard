## Best Possible Solution

Rebase a scoped negative cache that only covers the intended agent-turn slow path, keep read-only and full-catalog retry semantics intact, and add real runtime proof plus focused agent/gateway tests.

## Reproduction Assessment

Yes, at source level. Mock `ModelRegistry.getAll()` to return `[]` and call non-read-only `loadModelCatalog()` twice; current main clears the shared promise after the first empty result, so the second call can re-enter discovery.

## Solution Assessment

No, not as-is. A bounded negative cache may be useful, but this branch needs a scoped rebase around current read-only and gateway full-catalog retry behavior plus real runtime proof.
