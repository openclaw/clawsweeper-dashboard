The PR adds a 60-second negative cache for empty `loadModelCatalog()` results and unit tests for reuse, bypass, concurrent stale loads, and retry after expiry.

## Best Possible Solution

## Reproduction Assessment

Yes, at source level. Mock `ModelRegistry.getAll()` to return `[]` and call non-read-only `loadModelCatalog()` twice; current main clears the shared promise after the first empty result, so the second call can re-enter discovery.

No, not as-is. A bounded cache may be useful, but this branch predates the current read-only and gateway full-catalog retry contracts and needs a scoped rebase plus real runtime proof.
