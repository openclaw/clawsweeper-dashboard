This PR adds a 60-second negative cache for empty `loadModelCatalog()` results plus unit coverage for reuse, bypass, stale concurrent loads, and retry expiry.

## Best Possible Solution

Rebase a scoped negative cache that preserves side-effect-free read-only `models.list` and immediate retry for empty full-catalog requests, then add focused tests and redacted real runtime proof.

## Reproduction Assessment

Yes, at source level. Current main still clears the shared non-read-only catalog promise after an empty result, so two immediate non-read-only loads can re-enter discovery.

## Solution Assessment

No, not as-is. A bounded negative cache may be useful, but this branch must be rebased around current read-only and full-catalog retry contracts before it is the narrow maintainable fix.
