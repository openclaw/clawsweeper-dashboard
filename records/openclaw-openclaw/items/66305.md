## What This Changes

The branch adds a 60-second negative cache for empty `loadModelCatalog()` results and regression tests for reuse, bypass, stale concurrent loads, and retry after expiry.

## Best Possible Solution

If maintainers still want a bounded core negative cache, port it narrowly onto current `main` behind the non-read-only path, preserve full-catalog retry semantics, add focused coverage for both contracts, and include redacted real runtime proof.

## Reproduction Assessment

Yes, at source level. Mock `ModelRegistry.getAll()` to return `[]` and call non-read-only `loadModelCatalog()` twice; current `main` clears `modelCatalogPromise` after the first empty result, so the second call re-enters discovery.

## Solution Assessment

No, not as-is. The bounded negative cache may be useful, but this branch predates current read-only and gateway full-catalog retry contracts and must be rebased and scoped before it is mergeable.
