## Summary

Keep open. Current main and v2026.6.6 still lack the WebChat/Control UI inline-button delivery, rendering, callback RPC, and settled capability/config contract, while related reports were consolidated into this issue rather than replacing it.

## What This Changes

Request to add WebChat/Control UI inline-button support across button delivery, callback handling, capability/config semantics, and Control UI rendering.

## Best Possible Solution

Design an additive WebChat interactive-control path around shared `presentation` actions, authenticated `chat.callback` RPC, sanitized final/history projection, and Control UI rendering; defer webhook routing and avoid retired WebChat config shapes unless maintainers approve them.

## Reproduction Assessment

Yes at source level: the current schemas, Gateway method catalog, config schema, WebChat projection, and Control UI content model all lack the requested inline-button and callback path.

## Solution Assessment

No, not exactly as originally proposed; the safer path is typed `MessagePresentation` actions over authenticated Gateway RPC, not legacy shared `buttons` fields or a parallel webhook endpoint in v1.
