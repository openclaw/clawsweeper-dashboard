## What This Changes

The issue requests WebChat/Control UI inline button support, including a WebChat capability/config path, final chat-event controls, a callback RPC, and Control UI rendering.

## Best Possible Solution

Implement an additive WebChat interactive-control path through the shared presentation contract, preserve sanitized final-only controls through chat events/history, render them in Control UI, and start with scoped same-session callback routing while leaving webhook delivery for a separate security-reviewed design.

## Solution Assessment
