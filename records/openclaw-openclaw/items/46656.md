## Summary

Keep open. Current main still has no WebChat interactive button contract: chat sends/events have no buttons or callback method, WebChat config only exposes history truncation, and Control UI message normalization/rendering has no button item. The closed duplicate and Telegram webhook discussion do not supersede this, so this remains the canonical WebChat tracker.

## What This Changes

Request to add WebChat/Control UI inline buttons, callback RPC handling, and capability/config support for agent messages.

## Best Possible Solution

Define an additive WebChat interaction contract around shared `presentation`: carry sanitized final-only button blocks through chat events/history, render them in Control UI with disabled/clicked state, and add a scoped callback path that initially routes selected values back into the same session while keeping webhook routing as a separate hardened design.

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection on current main gives high confidence that the requested WebChat button/callback/config path is absent.

## Solution Assessment

No concrete patch is proposed, so there is no implementation to approve as-is. The requested direction is valid, but the maintainable solution should align with the existing shared `presentation` contract and receive maintainer protocol/security scoping before implementation.
