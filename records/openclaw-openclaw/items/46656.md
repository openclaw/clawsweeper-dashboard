## What This Changes

The issue requests WebChat/Control UI inline button support, including a WebSocket button payload, callback RPC, configuration capability, and built-in UI rendering.

## Best Possible Solution

Implement an additive WebChat interactive-control path through the shared presentation contract, preserve sanitized final-only controls through chat events/history, render them in Control UI, and start with a scoped same-session callback route while keeping webhook delivery for a separate security-reviewed design.

## Reproduction Assessment

## Solution Assessment

No concrete patch is proposed, so there is no implementation to approve. The requested direction is valid, but the maintainable path should extend shared presentation/control semantics instead of adding a Telegram-shaped buttons field plus webhook routing in one step.
