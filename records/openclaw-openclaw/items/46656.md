## Summary

Keep open. Current main and v2026.6.6 still lack the requested WebChat inline-button event, callback RPC, config, and Control UI rendering path; the closed related reports correctly consolidate into this issue rather than superseding it.

## What This Changes

Request adds WebChat/Control UI inline buttons, callback handling, and capability/config support for portable message actions.

## Best Possible Solution

Implement an additive WebChat interactive path using MessagePresentation typed actions, sanitized final-only event/history projection, Control UI button rendering/state, and an authenticated scoped chat.callback RPC; keep webhook routing as a separate hardened design.

## Reproduction Assessment

Yes at source level. Current main exposes the portable presentation contract and Telegram rendering, but the WebChat protocol, config, final projection, and Control UI content model lack button/callback support.

## Solution Assessment

Unclear as proposed. The desired capability is valid, but the best fix should use shared typed presentation actions and an authenticated Gateway RPC, not a legacy buttons payload or webhook-first HTTP surface.
