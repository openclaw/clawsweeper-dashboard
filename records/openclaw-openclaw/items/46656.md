Keep open. Current main and the latest release still lack the requested WebChat/Control UI inline-button contract, callback RPC, capability/config path, and Control UI renderer; https://github.com/openclaw/openclaw/issues/54720 was already closed as the narrower duplicate, while the broader notification issue does not replace this WebChat-specific tracker.

## What This Changes

The issue requests inline button support for WebChat/Control UI, including a config capability, WebSocket button payloads, a callback RPC, and UI rendering.

## Best Possible Solution

Implement an additive WebChat interactive-control path based on shared `presentation`: preserve sanitized final-only controls through chat events/history, render them in Control UI, and route clicks through a scoped same-session callback before separately designing webhook routing.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, release, and related-issue inspection give high confidence that WebChat button rendering, callback RPC, and config support are still absent on current main.

## Solution Assessment

No concrete patch is proposed, so there is no implementation to approve. The requested direction is valid, but the maintainable solution should use the shared `presentation` contract and a scoped callback design rather than bolting Telegram-shaped `buttons` onto WebChat alone.
