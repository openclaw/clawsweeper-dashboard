Keep open. ## What This Changes

Request WebChat/Control UI support for inline buttons, including capability configuration, final chat payload projection, a callback RPC, optional webhook routing, and Control UI rendering.

## Best Possible Solution

Implement an additive WebChat interactive-control path using the shared presentation-style contract: preserve sanitized final-only controls through chat events/history, render them in Control UI, and route button clicks through a scoped same-session callback before separately deciding webhook routing.

## Solution Assessment
