Summary: The issue has no patch, but the requested WebChat callback path and optional webhook routing are security-sensitive and need design review before implementation.

  - body: A WebChat button click must bind the authenticated operator, sessionKey, messageId, block/action identity, allowed scope, expiry, idempotency, double-click behavior, and rate limiting before it can safely enter the agent/session path.
