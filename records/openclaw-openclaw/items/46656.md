## Summary

Keep open. Current main still lacks the requested WebChat/Control UI inline-button path: no WebChat button payload in the chat protocol, no `chat.callback` RPC, no WebChat capability config, and no Control UI message item or renderer for buttons.

## What This Changes

Request WebChat/Control UI support for inline message buttons, including a capability config path, WebSocket payload propagation, callback RPC, and UI rendering.

## Best Possible Solution

Implement an additive WebChat interactive-control path based on the shared `presentation` contract, preserve sanitized final-only controls through chat events/history, render them in Control UI, and route clicks through a scoped session-local callback path before considering webhook routing.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence absence check: WebChat chat schemas, methods, config, and UI content types do not carry or render button callbacks.

## Solution Assessment

No patch is proposed, so there is no implementation to approve as-is. The maintainable direction is shared `presentation` support for WebChat plus a scoped callback contract; direct webhook routing should wait for explicit security and product scoping.
