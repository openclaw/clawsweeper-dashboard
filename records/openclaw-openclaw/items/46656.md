## Summary

Keep open. Current main and v2026.6.6 still lack WebChat/Control UI inline-button delivery, rendering, callback RPC, and a settled capability/config contract; related reports were consolidated into this tracker rather than replacing it.

## What This Changes

Request to add full WebChat/Control UI inline-button support, including structured button delivery, Control UI rendering, callback handling, and capability/config semantics.

## Best Possible Solution

Design an additive WebChat interactive-control path around the shared `presentation` contract, authenticated `chat.callback` RPC, sanitized final/history projection, and Control UI rendering, while deferring webhook routing and avoiding a retired WebChat config surface unless maintainers explicitly approve it.

## Reproduction Assessment

Yes. Source inspection shows the current protocol, Gateway method catalog, Control UI config schema, WebChat docs, and UI message renderer all lack the requested inline-button and callback path.

## Solution Assessment

No, not exactly as originally proposed. The best path is the later refined shape: typed `MessagePresentation` actions over an authenticated Gateway RPC, not legacy shared `buttons` fields or a parallel webhook endpoint in v1.
