## Summary

Keep open. Current main still lacks the requested WebChat/Control UI inline-control path: no button or presentation payload in the WebSocket chat contract, no scoped callback RPC, no WebChat capability/config surface, and no Control UI message renderer for clickable controls; the narrower WebChat duplicate at https://github.com/openclaw/openclaw/issues/54720 is already closed in favor of this canonical tracker.

## What This Changes

Request WebChat/Control UI support for inline message buttons, callback routing, and a capability/config surface for interactive chat controls.

## Best Possible Solution

Add an additive WebChat interactive-control path through the shared presentation contract, persist and broadcast sanitized final-only controls, render them in Control UI, and initially route clicks through a scoped same-session callback while keeping webhook routing as a separate hardened decision.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection give high confidence that WebChat button rendering, callback RPC, and config support remain absent on current main.

## Solution Assessment

No concrete patch is proposed, so there is no implementation to approve. The requested direction is valid, but the maintainable path should use the shared presentation/control contract and a scoped callback design rather than copying Telegram-shaped buttons into WebChat alone.
