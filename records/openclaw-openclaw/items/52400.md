The PR raises `session.agentToAgent.maxPingPongTurns` from 5 to 20 across runtime clamping, config validation, generated schema/help metadata, docs, and helper tests while keeping the default at 5.

## Best Possible Solution

Adopt one maintainer-approved path for #52382, then update the runtime clamp, Zod schema, public config types/help, generated schema metadata, tracked config-doc hash, public docs, focused tests, and changelog together while preserving the default unless maintainers choose otherwise.

## Reproduction Assessment

Yes. Current main statically reproduces the limit: schema validation rejects values above 5, `resolvePingPongTurns` clamps to 5, and the A2A loop consumes that resolved value.

## Solution Assessment

No, not as currently proposed. Raising the cap is a plausible narrow fix, but this branch needs a rebase plus docs, generated-artifact, gateway-guide, and changelog corrections before it is the safest maintainable solution.
