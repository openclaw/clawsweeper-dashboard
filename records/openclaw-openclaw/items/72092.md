## Summary

Keep this PR open: current main and the latest release still have the Bedrock `aws-sdk` media-auth failure, while this patch is a focused fix with no blocking correctness finding found. The remaining decision is maintainer acceptance of the bounded auth-provider risk and the fact that live proof covers image, with audio/video covered by runner-level tests.

## What This Changes

The PR lets media-understanding image, audio, and video paths execute `aws-sdk` auth providers without a static API key and adds Bedrock-focused regression coverage.

## Best Possible Solution

Land this focused auth-mode fix once maintainers accept the bounded auth-provider risk, preserving strict missing-key errors for non-`aws-sdk` modes and avoiding empty-key persistence.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.6.9` shows the media paths call `requireApiKey` on empty `aws-sdk` auth before the AWS SDK can resolve credentials, and the linked issue/comments provide live failure reports.

## Solution Assessment

Yes. This is the narrowest maintainable shape I found: use the existing keyless execution contract for resolved `aws-sdk` auth instead of adding config or weakening `requireApiKey` globally.
