## Summary

Keep open: this is a real P1 Bedrock media auth fix with useful live image proof, but the current PR still routes the new empty-key execution path around the shared transient retry loop, so it needs a narrow repair before merge.

## What This Changes

The PR allows Bedrock `aws-sdk` empty-key auth through media image, MiniMax fallback, and audio/video execution paths, adds `allowEmptyKey` rotation support, and adds media regression tests.

## Best Possible Solution

Keep the `aws-sdk`-only empty-key allowance, but represent the empty sentinel as a normal single execution key inside the rotation helper so retry, rotation, and error handling semantics stay shared.

## Reproduction Assessment

Yes. Source inspection shows current main documents keyless Bedrock `aws-sdk` auth while the media image and audio/video paths still force `requireApiKey`; the PR diff also clearly shows the remaining retry bypass.

## Solution Assessment

No, not as-is. The `aws-sdk` carve-out is the right narrow fix direction, but the empty-key execution path should use the existing rotation/retry loop instead of returning before it.
