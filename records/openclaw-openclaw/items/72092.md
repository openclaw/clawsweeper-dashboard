## Summary

Keep open: the PR addresses a valid Bedrock aws-sdk media-auth regression and now has useful live image proof, but it is not merge-ready because the new empty-key path bypasses the existing transient retry loop for audio/video provider execution.

## What This Changes

The PR lets media image, MiniMax fallback, audio, and video paths pass an empty aws-sdk auth sentinel through to Bedrock and adds an `allowEmptyKey` option plus regression tests.

## Best Possible Solution

Keep the aws-sdk-only empty-key allowance, but route the empty sentinel through the normal rotation/retry loop and land after targeted regression coverage plus the supplied Bedrock proof.

## Reproduction Assessment

Yes. Source inspection on current main shows Bedrock `aws-sdk` media auth reaches strict `requireApiKey`, while docs and the chat path define keyless SDK-chain auth as supported.

## Solution Assessment

No, not as-is. The aws-sdk carve-out is the right narrow direction, but `allowEmptyKey` should reuse the existing transient retry loop instead of returning before it.
