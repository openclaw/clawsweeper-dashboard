## Summary

Keep open: this PR addresses a valid current-main Bedrock aws-sdk media-auth bug, but it should not merge until the empty-key execution path goes through the existing transient retry loop and the remaining Bedrock media proof gap is handled.

## What This Changes

The PR allows keyless `aws-sdk` auth through Bedrock image, MiniMax fallback, audio, and video media paths, adds an `allowEmptyKey` rotation option, and adds media regression tests.

## Best Possible Solution

Keep the aws-sdk-only empty-key allowance, represent the empty key as a single execution key inside the normal rotation/retry loop, add an empty-key transient retry regression, and land after redacted live Bedrock audio/video proof or an explicit maintainer proof override.

## Reproduction Assessment

Yes. Current main documents Bedrock `aws-sdk` as keyless while the image and audio/video media paths still require a literal key before provider execution; the PR discussion also includes live before/after image output for the same failure mode.

## Solution Assessment

No, not as-is. The aws-sdk-only carve-out is the right narrow direction, but the empty-key audio/video path should preserve the existing retry behavior before this is merge-ready.
