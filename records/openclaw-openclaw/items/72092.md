## Summary

This PR is still useful and should stay open: current main still rejects Bedrock `aws-sdk` media auth before the SDK credential chain can run, and the branch fixes the image plus audio/video runner paths with the current keyless execution mechanism. ## What This Changes

The PR lets media-understanding image and audio/video execution proceed keyless for providers whose resolved auth mode is `aws-sdk`, with regression tests for image, audio, and video paths.

## Best Possible Solution

Yes, source inspection gives a high-confidence reproduction path: configure Bedrock with `auth: "aws-sdk"` and no literal API key, then current main reaches `requireApiKey` in the image or media runner path before SDK credentials can resolve. PR comments also include live Bedrock before/after output for the image failure.

## Solution Assessment

Yes for the scoped image/audio/video bug: using the existing keyless `kind: "none"` runner path is cleaner than passing an empty key through rotation. It is not complete for the linked issue if maintainers expect the later PDF reproductions to be fixed by the same closing PR.
