## Summary

Keep open: current main still rejects keyless Bedrock media auth, and this PR is the current viable fix candidate with no blocking code-review findings; the remaining decision is maintainer acceptance of the auth-provider behavior change.

## What This Changes

The PR teaches media-understanding image and audio/video auth resolution to allow `aws-sdk` providers to execute without a static API key and adds Bedrock regression coverage.

## Best Possible Solution

Land this focused image/audio/video fix after maintainers accept keyless `aws-sdk` media auth, and track PDF separately unless maintainers require that scope before closing the canonical issue.

## Reproduction Assessment

Yes. Source inspection on current main shows `aws-sdk` auth resolves without a literal API key and the image/audio/video media paths then reach `requireApiKey`, which throws before AWS SDK credentials can be used.

## Solution Assessment

Yes for the image/audio/video scope. The PR reuses the existing keyless `kind: "none"` runner path and limits the image carve-out to `aws-sdk`, rather than broadening `requireApiKey` globally.
