## Best Possible Solution

Keep the narrow `aws-sdk`-only empty-key allowance, route that sentinel through the same retry machinery as normal keys, and land after targeted tests plus redacted live Bedrock image/audio/video proof.

Yes. Source-level reproduction is high confidence: current docs and AWS SDK source support keyless Bedrock credential-chain auth, while current main still calls static-key guards in media image and audio/video paths before provider execution.

## Solution Assessment
