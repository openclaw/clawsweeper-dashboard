## Summary

Keep open: this PR targets a valid Bedrock `aws-sdk` media-auth bug, but the current patch still needs a small retry-behavior fix and external real Bedrock media proof before merge.

## What This Changes

The PR lets media-understanding image, MiniMax fallback, and audio/video execution pass an empty API key for resolved `aws-sdk` auth and adds mocked Bedrock media regressions.

## Best Possible Solution

Keep the narrow `aws-sdk`-only empty-key allowance, make empty-key execution participate in the existing retry loop, then land after targeted tests and redacted live Bedrock media proof.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main documents Bedrock `aws-sdk` as keyless but the media image and audio/video paths still call static-key guards before provider execution. I did not run a live Bedrock call in this read-only review.

## Solution Assessment

No, not as-is: the auth bypass is the right narrow direction, but the empty-key rotation path should preserve the existing transient retry behavior and needs real Bedrock media proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Route empty-key execution through the retry loop:** `src/agents/api-key-rotation.ts:58`
  - body: In the empty-key branch added here, `return await params.execute("")` exits before `resolveTransientProviderRetryOptions` and the same-key retry loop. Treat the empty key as the single key in the existing loop and add a regression for transient retry with `allowEmptyKey`.
  - confidence: 0.88
