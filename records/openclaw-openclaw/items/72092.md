## Summary

Keep open: the PR fixes the right Bedrock aws-sdk media-auth gap and now has live image-path proof, but the new empty-key execution branch bypasses the existing transient read retry behavior for audio/video provider calls.

## What This Changes

The PR allows aws-sdk-mode empty credentials through media-understanding image and audio/video auth paths, adds an `allowEmptyKey` rotation option, and adds image plus audio/video regression coverage.

## Best Possible Solution

Keep the aws-sdk-only empty-key allowance, route empty-key execution through the normal rotation/retry loop, preserve strict behavior for every other auth mode, and land with focused regression coverage plus the supplied Bedrock proof.

## Reproduction Assessment

Yes. Current main source plus the linked live reports show media paths call strict API-key guards even though Bedrock docs and chat runtime support `auth: "aws-sdk"` with no static key.

## Solution Assessment

No, not as-is. The aws-sdk-only carve-out is the right narrow direction, but `allowEmptyKey` should preserve the existing transient read retry loop before this is merge-ready.

## Review Findings

- **[P2] Route empty-key execution through transient retries:** `src/agents/api-key-rotation.ts:59-60`
