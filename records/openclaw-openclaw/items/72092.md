## What This Changes

The PR lets media-understanding image, MiniMax fallback, and audio/video provider execution use an empty `apiKey` for `aws-sdk` auth and adds mocked regression coverage.

## Best Possible Solution

Land this PR, or an equivalent narrow fix, after conflict resolution and redacted real Bedrock media proof confirms keyless `aws-sdk` image and audio/video execution reaches the provider path.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: configure `amazon-bedrock` media auth as `aws-sdk` with no `apiKey`; current main reaches strict `requireApiKey` in image/fallback and audio/video before provider execution, while the Bedrock docs say no API key is required.

## Solution Assessment

Yes for the code path. The PR mirrors the existing text-runtime missing-key allowance, preserves strict errors for other auth modes, and adds an empty-key rotation hook so the audio/video path does not drop the AWS SDK sentinel.
