## Summary

Keep open. Current main still has the Bedrock media `aws-sdk` static-key guard, and the PR head appears to fix the image and audio/video paths with green CI, but this external PR still lacks real Bedrock media proof before merge.

## What This Changes

The PR lets media image, MiniMax fallback, and audio/video execution use an empty key for `aws-sdk` auth, adds an empty-key rotation path, and adds media regression tests.

## Best Possible Solution

Land this PR or an equivalent narrow fix after redacted real Bedrock media proof shows keyless `aws-sdk` auth reaches provider execution while non-`aws-sdk` modes still fail closed.

## Reproduction Assessment

Yes, source-level. Configure `amazon-bedrock` media auth as `aws-sdk` with no static `apiKey`; current main reaches strict `requireApiKey` in image and audio/video media paths even though the Bedrock docs say no `apiKey` is required.

## Solution Assessment

Yes for the code direction. The PR mirrors the existing simple-completion missing-key allowance, keeps `requireApiKey` strict for other auth modes, and addresses the earlier empty-string rotation gap; merge should wait for real behavior proof.
