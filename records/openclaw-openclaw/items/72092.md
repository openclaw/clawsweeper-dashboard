## Summary

Keep open: the PR addresses a real Bedrock aws-sdk media-auth bug with useful proof, but the latest diff still runs the new empty-key execution path outside the existing transient retry loop, so Bedrock audio/video reads can fail after one transient 500, timeout, or network error where static-key media calls retry.

## What This Changes

The PR lets media-understanding image, MiniMax fallback, audio, and video paths pass an empty aws-sdk credential sentinel for Bedrock-style SDK-chain authentication and adds regression coverage.

## Best Possible Solution

Land the Bedrock aws-sdk media-auth fix after the empty-key sentinel uses the existing executeWithApiKeyRotation retry loop and focused tests cover empty aws-sdk, static-key, and non-aws-sdk missing-key behavior.

## Reproduction Assessment

## Solution Assessment

Mostly yes, but not quite merge-ready. The resolved auth.mode guard is narrow and matches the chat-path contract, but allowEmptyKey should enter the existing transient retry loop instead of calling execute directly.
