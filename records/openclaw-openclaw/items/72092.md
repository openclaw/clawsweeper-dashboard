## Summary

Keep this PR open: it targets a real Bedrock media auth bug and the overall fix direction is narrow, but the empty-key execution branch still bypasses the existing transient retry loop and the supplied proof is helper-level terminal output rather than live Bedrock media execution.

## What This Changes

The PR adds an aws-sdk empty-key allowance for Bedrock media image, MiniMax fallback, and audio/video execution paths, plus media regression tests and an allowEmptyKey option in API-key rotation.

## Best Possible Solution

Keep the aws-sdk-only empty-key allowance, route the empty-key case through the normal rotation/retry loop, and land only after targeted tests plus redacted live Bedrock media proof or an explicit maintainer proof override.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence because current main documents Bedrock aws-sdk as keyless while the media image and audio/video paths still call requireApiKey before provider execution.

## Solution Assessment

No, not as-is. The aws-sdk-only carve-out is the right narrow fix direction, but the empty-key execution path should preserve the existing transient retry behavior before this is merge-ready.
