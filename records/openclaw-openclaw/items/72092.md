## Summary

Keep open. This is a focused fix for a current-main Bedrock media auth bug and the latest PR checks are green, but the external PR still lacks after-fix real behavior proof from a real Bedrock setup.

## What This Changes

The PR lets media image, MiniMax fallback, and audio/video execution paths pass an empty key for `aws-sdk` auth, adds an `allowEmptyKey` rotation hook, and adds mocked regression coverage.

## Best Possible Solution

Land this PR, or an equivalent narrow fix, after the contributor adds redacted real Bedrock media proof showing the keyless `aws-sdk` path reaches provider execution.

## Reproduction Assessment

Yes. Configure `amazon-bedrock` media auth with `auth: "aws-sdk"` and no static key; current main reaches `requireApiKey` in the image and audio/video media paths, which source-inspection shows will throw before provider execution.

## Solution Assessment

Yes for the code path. The PR aligns media-understanding with the existing chat `aws-sdk` missing-key allowance, preserves strict `requireApiKey` behavior for other auth modes, and addresses the earlier empty-string rotation gap.
