## What This Changes

This PR lets media-understanding image, MiniMax fallback, and audio/video execution paths use keyless `aws-sdk` auth, adds an `allowEmptyKey` rotation path, and adds mocked regression coverage.

## Best Possible Solution

I did not run a live Bedrock media request in this read-only review.

## Solution Assessment

Yes for the code path. The PR keeps `requireApiKey` strict for other modes, mirrors the existing chat missing-key allowance, and uses a narrow `allowEmptyKey` hook so the earlier empty-string rotation failure is not preserved.
