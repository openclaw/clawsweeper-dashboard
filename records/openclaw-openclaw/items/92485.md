## Summary

This PR addresses a real Windows supervisor-launch gap, but it is not merge-ready: the new hardcoded PATH walker duplicates the repository’s canonical PATHEXT-aware resolver and can change which executable an existing Windows setup launches. Related PRs remain open, but none is currently a safe proof-positive replacement, so this contribution should stay open for a focused repair rather than close as a duplicate.

## What This Changes

The branch makes the supervisor resolve non-allowlisted bare Windows commands by walking PATH for `.cmd`, `.bat`, or `.com` files and adds filesystem-backed unit tests.

## Best Possible Solution

Consolidate the fix into one canonical PR that reuses the shared PATH/PATHEXT-aware executable resolver, preserves deliberate package-manager shim handling, adds executable-precedence regression coverage, and includes redacted real Windows proof of the supervisor launching a non-allowlisted npm shim.

## Reproduction Assessment

Yes, at source level: the supervisor only appends `.cmd` for a fixed allowlist, so other npm-installed CLI shims remain bare before spawn. The PR body also reports a Windows reproduction, although the after-fix runtime evidence is not independently inspectable.

## Solution Assessment

No, not as written. The narrow maintainable fix is to reuse the existing PATHEXT-aware resolver rather than introduce a second resolver with different extension and precedence rules.
