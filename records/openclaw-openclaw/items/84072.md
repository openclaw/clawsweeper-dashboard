## Summary

Keep open: the patch targets a real fallback-latency problem, but the current implementation trips a shared provider/model circuit for every FailoverError, runs that skip before fail-closed harness checks, leaves circuit state unbounded, and still lacks real behavior proof outside mocked unit coverage.

## What This Changes

The branch adds an in-memory provider/model circuit breaker to model fallback, a `circuit_open` failover status, test reset support, and focused unit coverage.

## Best Possible Solution

Add a bounded, scoped circuit after harness policy and alongside auth-profile cooldown semantics, tripping only on safe transient provider/model-wide failures and backed by redacted real gateway/session proof.

## Reproduction Assessment

Yes for the review findings by source inspection: the PR records every known FailoverError into a provider/model-only circuit and checks that circuit before harness and auth cooldown policy. I did not run tests because this review is read-only.

## Solution Assessment

No. A circuit breaker may be a good direction, but this implementation needs narrower failure classification, preserved fail-closed ordering, scoped/bounded state, and real behavior proof before it is the maintainable fix.
