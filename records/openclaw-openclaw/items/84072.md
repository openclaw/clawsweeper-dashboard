## What This Changes

The PR adds an in-memory provider/model circuit breaker to model fallback, introduces a `circuit_open` failover reason mapped to HTTP 503, and adds a focused unit test for skipping a tripped candidate.

## Best Possible Solution

The end state should be a rebased, bounded, maintainer-approved transient-failure circuit that preserves auth-profile/session/probe scope and is proven through redacted real runtime failure evidence.

## Solution Assessment

No. The circuit-breaker direction may be useful, but this implementation is too broad and unbounded; the safer fix is a scoped transient-failure circuit integrated with the existing auth-profile cooldown and probe contract.
