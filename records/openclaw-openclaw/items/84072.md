## Summary

Keep open: the circuit-breaker idea is useful, but this branch is not safe to merge because it adds a provider/model-wide circuit before current auth-profile cooldown/probe logic, records every known failover reason into that circuit, keeps unbounded module-level state, and is now conflicting with newer main fallback work.

## What This Changes

The PR adds an in-memory per-provider/model model-fallback circuit breaker, a new `circuit_open` failover reason/status, and focused unit coverage.

## Best Possible Solution

Keep the useful circuit-breaker direction, but rework it as a bounded circuit that trips only on maintainer-approved transient provider/model-wide failures, preserves current auth-profile/session/model probe semantics, and is proven with a redacted real gateway/session failure path.
