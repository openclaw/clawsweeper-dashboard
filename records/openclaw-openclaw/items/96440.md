## Summary

Keep this PR open for normal maintainer merge review: current main and the latest release still strip only ESC before emitting BEL-delimited OSC-8 links, while the PR adds a focused terminal-core sanitizer with regression tests and sufficient terminal-output proof.

## What This Changes

The PR adds C0/C1 control-character stripping to `formatTerminalLink()` in terminal-core and adds focused tests for OSC-8 and plain fallback output.

## Best Possible Solution

Land the focused terminal-core sanitizer with its regression tests; track broader TUI OSC-8 hardening separately if maintainers want a full terminal-link sweep.

## Reproduction Assessment

Yes. Current main strips only ESC before interpolating the URL into a BEL-delimited OSC-8 sequence in `packages/terminal-core/src/terminal-link.ts`, so a BEL in label or URL can terminate the generated link segment.

## Solution Assessment

Yes for the terminal-core formatter. Sanitizing at `formatTerminalLink()` covers the exported shared formatter and current docs-link caller without reusing the broader `sanitizeTerminalText()` behavior that would also strip ANSI sequences and alter printable residue.
