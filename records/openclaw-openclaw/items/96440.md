## Summary

Keep open for maintainer merge judgment: the patch appears correct and has sufficient terminal proof, but it intentionally changes public terminal-link output by stripping C0/C1 controls that current main and the latest release still emit.

## What This Changes

This PR adds a terminal-core sanitizer that strips C0/C1 control bytes from OSC-8 labels, URLs, and fallback output, with terminal-link and docs-link regression tests.

## Best Possible Solution

Merge the formatter-boundary sanitizer if maintainers accept that terminal links should never emit C0/C1 controls; otherwise narrow the sanitizer to the specific OSC delimiter/control bytes and update the tests accordingly.

## Reproduction Assessment

Yes. Source inspection shows current main removes ESC but not BEL even though BEL delimits the generated OSC-8 sequence; I did not execute a local failing probe because this review kept the checkout read-only.

## Solution Assessment

Yes. Sanitizing in formatTerminalLink is the right boundary because direct callers and formatDocsLink/plugin-SDK paths share it; the only maintainer choice is whether the C0/C1 scope is acceptable for compatibility.
