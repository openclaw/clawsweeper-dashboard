## Summary

Keep open: current main still strips only ESC from terminal links, while this PR now strips C0/C1 controls and preserves explicit empty fallback semantics. After checking the formatter, caller surface, tests, release behavior, and prior ClawSweeper finding, I found no blocking patch defect.

## What This Changes

The PR adds terminal-core control-character stripping for OSC-8 labels, URLs, and plain fallback text, with focused regression coverage for terminal links and docs-link fallback output.

## Best Possible Solution

Merge the boundary-level terminal-link sanitizer with the empty-fallback regression preserved, if maintainers consciously accept the C0/C1 output sanitization change.

## Reproduction Assessment

Yes. Source inspection shows current main removes ESC but not BEL even though BEL delimits the generated OSC-8 sequence; the PR body also includes a direct Node probe showing the fixed output.

## Solution Assessment

Yes. Sanitizing at formatTerminalLink is the right boundary because direct callers and formatDocsLink/plugin-SDK paths share it; caller-side sanitizing would be incomplete, and reusing sanitizeTerminalText would strip whole ANSI sequences beyond this PR's compatibility-preserving goal.
