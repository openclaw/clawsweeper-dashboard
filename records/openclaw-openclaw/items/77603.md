## Summary

Keep open: current main still has the reported agent-path preload asymmetry, and source inspection shows cached descriptor-backed tools can bypass the cold-load branch that would otherwise warm the runtime registry.

## What This Changes

The issue reports that the native agent plugin-tool resolver does not preload the standalone plugin tool registry before calling `resolvePluginTools`, unlike catalog and MCP paths.

## Best Possible Solution

Add the same standalone registry preload to the shared agent plugin-tool helper using the exact context, allowlist, denylist, gateway-binding, and auth inputs passed to `resolvePluginTools`, then cover the agent path with a regression test.

## Reproduction Assessment

Yes from source inspection: current main shows the agent helper calling `resolvePluginTools` without the preload used by catalog and MCP, and the resolver can return cached descriptor-backed tools before the cold-load branch runs. I did not execute a live gateway/agent scenario in this read-only pass.

## Solution Assessment

Yes, the suggested catalog-pattern preload is the narrowest maintainable repair for the asymmetry. A broader resolver invariant change can remain follow-up work unless the regression test shows other callers are still exposed.

## Repair Work Prompt

Fix the agent-path plugin tool registry preload bug tracked by https://github.com/openclaw/openclaw/issues/77603. In `src/agents/openclaw-plugin-tools.ts`, mirror the catalog/MCP pattern by calling `ensureStandalonePluginToolRegistryLoaded` before `resolvePluginTools`, using the same resolved plugin-tool inputs, `pluginToolAllowlist`, `pluginToolDenylist`, `allowGatewaySubagentBinding`, and `hasAuthForProvider` callback that resolution receives. Avoid changing plugin policy semantics, optional-tool allowlist behavior, or the deeper `resolvePluginTools` cold-load algorithm unless a regression test proves the helper-level preload is insufficient. Update `src/agents/openclaw-tools.browser-plugin.integration.test.ts` or adjacent focused agent coverage so it verifies the preload is called before resolution and forwards the same params; keep existing runtime-config getter behavior covered. Add a user-facing `CHANGELOG.md` fix entry because this affects plugin tools visible to agents. Validate with targeted agent/plugin tests and the changed gate if promoted.
