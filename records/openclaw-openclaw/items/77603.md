## Summary

Keep open: current main still has the reported agent-path asymmetry, and source inspection shows cached plugin descriptors can satisfy selection before the cold-load fallback installs the runtime registry needed for execution.

## What This Changes

Preload the standalone plugin tool registry in agent-path plugin tool resolution before resolving plugin tools, matching catalog and MCP behavior.

## Best Possible Solution

Preload the standalone plugin tool registry inside resolveOpenClawPluginToolsForOptions using the same resolved inputs passed to resolvePluginTools, then add regression coverage for call order and cached-descriptor execution.

## Reproduction Assessment

Yes: source-reproducible on current main. The agent resolver skips preload while cached descriptor resolution can return tools before the cold-load branch installs the runtime registry those tools need at execution.

## Solution Assessment

Yes: mirroring the catalog/MCP preload in resolveOpenClawPluginToolsForOptions is the narrowest maintainable fix because it covers both OpenClaw and Pi callers without changing plugin selection policy.
