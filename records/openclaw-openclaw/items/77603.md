## Summary

Keep open: current `main` still has the reported agent-path asymmetry, and source inspection shows a plausible current failure mode when cached plugin tool descriptors satisfy selection before the cold-load fallback can install the runtime registry used for execution.

## What This Changes

The issue asks for the agent-path plugin tool resolver to preload the standalone plugin tool registry before resolving plugin tools, matching catalog and MCP consumers.

## Best Possible Solution

Preload the standalone plugin tool registry in `resolveOpenClawPluginToolsForOptions` using the same resolved context, allowlist, denylist, gateway binding, and auth availability inputs that are passed to `resolvePluginTools`, then add focused regression coverage for call order and cached-descriptor execution.

## Reproduction Assessment

Yes, source-reproducible: current `main` still has the agent path skip preload while cached descriptor resolution can return before cold-loading the runtime registry. A live invocation was not run in this read-only review.

## Solution Assessment

Yes, mirroring the catalog/MCP preload is the narrowest maintainable fix because it uses an existing helper and does not change plugin selection or policy semantics.
