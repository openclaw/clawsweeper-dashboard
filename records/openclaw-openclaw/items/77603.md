## Summary

Keep open: current `main` still has the reported agent-path preload asymmetry, and the cached-descriptor path can return selected plugin tools before the cold-load fallback installs the runtime registry needed for execution.

## What This Changes

The issue asks for the agent-path plugin tool resolver to preload the standalone plugin tool registry consistently with the catalog and MCP plugin-tool consumers.

## Best Possible Solution

Preload the standalone plugin tool registry inside `resolveOpenClawPluginToolsForOptions` with the same resolved context and policy inputs passed to `resolvePluginTools`, then cover the agent-path cached-descriptor execution case.

## Reproduction Assessment

Yes, source-reproducible: current `main` still has the agent resolver skip preload while catalog and MCP preload before the same downstream resolver. I did not run a live plugin invocation in this read-only review.

## Solution Assessment

Yes, mirroring the catalog/MCP preload pattern is the narrowest maintainable fix because it uses the existing helper and should not change plugin selection, policy, or configuration semantics.
