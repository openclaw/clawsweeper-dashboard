## Summary

Keep open: current main still has the reported agent-path asymmetry, and the cached-descriptor fast path can return plugin tool wrappers before the cold-load fallback installs the runtime registry needed for execution.

## What This Changes

The issue requests preloading the standalone plugin tool registry in the agent plugin-tool resolver so it matches the catalog and MCP consumers.

## Best Possible Solution

Preload the standalone plugin tool registry inside `resolveOpenClawPluginToolsForOptions` using the same context, allowlist, denylist, gateway binding, and auth inputs passed to `resolvePluginTools`, then add focused regression coverage for the agent call order and cached-descriptor execution path.

## Reproduction Assessment

Yes, source-reproducible: current main still has the agent resolver call `resolvePluginTools` without a preload, while cached descriptor handling can empty `runtimePluginIds` and skip the cold-load fallback. A live plugin invocation was not run in this read-only review.

## Solution Assessment

Yes, mirroring the catalog/MCP preload in the agent helper is the narrowest maintainable fix because it uses an existing helper and preserves plugin selection policy semantics.
