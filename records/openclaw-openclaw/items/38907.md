## Summary

Current main and v2026.6.5 no longer use the failing first-prompt lifecycle from v2026.3.2: ACP sessions now have explicit protocol-session mapping, atomic runtime metadata persistence, and regression coverage proving assistant chunks are emitted before end-turn.

## What This Changes

Fix ACP bridge sessions that fail on the first prompt because ACP session metadata is unavailable, producing an echo and end-turn without assistant chunks.

## Best Possible Solution

Keep the current separation between ACP protocol-session mapping in the translator and atomic ACP runtime metadata ownership in the control plane; track thread-child or resume-specific metadata failures in their narrower reports rather than reopening this obsolete basic-client failure.

## Reproduction Assessment

Yes for v2026.3.2: the supplied CLI sequence reaches source branches with the exact missing-ACP-metadata failure. Current main and v2026.6.5 instead contain the replacement initialization path and focused chunk-delivery coverage.

## Solution Assessment

Yes; explicit protocol-session mapping plus atomic metadata persistence at the ACP control-plane boundary is the narrow maintainable solution and avoids inferring ACP state from session-key strings.
