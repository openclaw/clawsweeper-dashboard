## Summary

Keep this PR open, but do not merge the current patch: it fixes the reported Telegram duplicate path at the shared helper level while also letting Google Chat suppress its fallback when its native handler will not accept a null-source request.

## What This Changes

The PR changes `shouldSuppressForwardingFallback` so a missing `turnSourceChannel` no longer blocks native fallback suppression and adds one helper-level regression test for that null-channel case.

## Best Possible Solution

Scope null-source suppression to Telegram or to native handlers that can actually handle unknown-source requests, preserve Google Chat fallback delivery, add sibling regression coverage, then attach redacted live Telegram proof.

## Reproduction Assessment

Yes, at source level: current main exits suppression on null `turnSourceChannel`, while Telegram native matching can still accept the same request through account/session binding. I did not establish a live Telegram reproduction in this read-only review.

## Solution Assessment

No. The PR is a plausible Telegram fix, but the shared-helper shape is not the best mergeable solution because it also affects Google Chat, whose native handler rejects null-source approval requests.
