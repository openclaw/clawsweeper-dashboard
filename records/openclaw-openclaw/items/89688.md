## Summary

Keep open: the Telegram duplicate-approval bug is real, but this patch is not merge-ready because the shared null-channel suppression change can drop Google Chat approval prompts and the contributor provided helper-level validation only, not real Telegram proof.

## What This Changes

The PR changes shared approver-restricted native approval fallback suppression to treat a missing `turnSourceChannel` as unknown instead of a mismatch, and adds a regression test for the Telegram null-channel case.

## Best Possible Solution

Gate fallback suppression on the same channel-specific native eligibility that will actually deliver the approval, or narrow the null-channel allowance to Telegram-compatible handling, then add live Telegram proof and sibling regression coverage.

## Reproduction Assessment

Yes at source level: current main exits fallback suppression on null `turnSourceChannel` before checking Telegram native account eligibility, while Telegram native matching can still accept by account/session. The patch's Google Chat regression is also source-reproducible from the shared helper plus Google Chat's turn-source-only native handler gate.

## Solution Assessment

No for the submitted patch: the helper-level direction is plausible for Telegram, but this exact shared condition is too broad for Google Chat. The safer fix is to align suppression with each channel's native `shouldHandle` or target resolver before suppressing fallback.
