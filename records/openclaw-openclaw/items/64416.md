## Summary

Keep open: the PR targets a real ACP/subagent completion-delivery problem, but it still has blocking correctness and prompt-boundary security issues, lacks after-fix real behavior proof, and overlaps the broader open completion-delivery gate work.

## What This Changes

The PR changes ACP/subagent completion delivery by adding task-level spawnedBy metadata, preferring live completion output in completion announces, suppressing a farewell hook, appending ACP capability-normalization guidance, and updating related tests.

## Best Possible Solution

Consolidate on a durable completion-delivery ownership path: keep structured internal completion events authoritative, escape child output before plain prompt interpolation, persist or derive parent lineage across restart, and reconcile this PR with the broader delivery-gate work before merge.

## Reproduction Assessment

Yes for the PR defects: source inspection gives high-confidence paths by placing an internal-context delimiter in child output and by restoring a parent-spawned ACP task from SQLite after restart. I did not run live ACP/channel reproduction because this was a read-only review.

## Solution Assessment

No. The PR is aimed at the right completion-delivery area, but prompt guidance plus an unpersisted task field is not the narrowest maintainable solution while the broader delivery-gate design and the current-main Discord duplicate fix overlap the same behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Escape completion text before building steer messages:** `src/agents/subagent-announce.ts:209`
  - body: For completion announces, this interpolates child findings into the plain steer message before structured internalEvents are attached. If the child output contains an OpenClaw internal-context delimiter, the parent turn can treat the body as already wrapped and skip the authoritative internal event, so escape the text or build the prompt only from the sanitized internal-event formatter.
  - confidence: 0.88
- **[P2] Persist spawnedBy before relying on terminal routing:** `src/tasks/task-registry.ts:1027`
  - body: The new silent-wake branch depends on latest.spawnedBy, but the SQLite task row, select, and upsert paths on main do not persist that field. After a gateway restart, restored ACP tasks lose the lineage and fall back to the visible requester-delivery path this branch is trying to avoid.
  - confidence: 0.91
- **[P3] Align the format expectation with the guard wording:** `src/agents/subagent-announce.format.e2e.test.ts:561`
  - body: The implementation says the silent-reply guard applies when ANY condition holds, but this changed format assertion still expects ALL. Update the assertion so the format test proves the intended prompt wording.
  - confidence: 0.86
