## Summary

Keep open. Source inspection shows this PR is not mergeable as-is because it introduces a prompt-boundary security issue and a non-durable routing decision, and it still lacks after-fix live ACP/channel proof while overlapping https://github.com/openclaw/openclaw/pull/65242.

## What This Changes

The PR changes ACP/subagent completion delivery by adding task-level parent lineage, silent parent wake routing, live-output preference, farewell suppression, ACP turn prompt guidance, and related tests.

## Best Possible Solution

Consolidate on one durable completion-delivery ownership path that keeps structured events authoritative, sanitizes child output before prompt interpolation, persists or derives parent lineage across restart, and reconciles this branch with https://github.com/openclaw/openclaw/pull/65242 before merge.

## Reproduction Assessment

Yes for the PR defects by source inspection: child output containing an internal-context delimiter can bypass structured-event prepending, and restored task rows lose the new spawnedBy routing field. I did not run a live ACP/channel reproduction in this read-only review.

## Solution Assessment

No. The PR is aimed at a valid delivery bug, but raw prompt interpolation, non-persisted task lineage, and overlap with the broader delivery-gate PR make this branch unsafe as-is.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Escape completion text before composing steer messages:** `src/agents/subagent-announce.ts:207-208`
  - body: This inserts child findings into the plain trigger message before `internalEvents` are attached. If the child output contains an OpenClaw internal-context delimiter, `prependInternalEventContext()` treats the body as already wrapped and skips the authoritative event, so sanitize the text or keep it only in structured-event formatting.
  - confidence: 0.9
- **[P2] Persist spawnedBy before using it for terminal routing:** `src/tasks/task-registry.ts:1028-1029`
  - body: The new silent-wake branch depends on `latest.spawnedBy`, but the default SQLite task store still does not persist or restore that field. After a gateway restart, restored ACP tasks lose the lineage and fall back to visible requester delivery.
  - confidence: 0.92
- **[P3] Update the format expectation for the ANY guard:** `src/agents/subagent-announce.format.e2e.test.ts:561`
  - body: The implementation now says the silent-reply guard applies when ANY condition holds, but this format expectation still asserts the older ALL wording. Update the assertion so the test proves the intended prompt contract.
  - confidence: 0.86
