## Summary

Keep this PR open. Current main does not fully implement the PR's intended ACP completion-delivery behavior, but the proposed patch still has concrete blockers: untrusted completion text is interpolated into a steer message before internal events are attached, task-level `spawnedBy` is not persisted through the default SQLite task store, and the related broader CompletionDeliveryGate PR is still open rather than shipped.

## What This Changes

The PR changes ACP/subagent completion delivery by adding task-level `spawnedBy` metadata, preferring live completion output in announce prompts, adding a silent parent wake branch, suppressing a farewell hook, and appending ACP capability-normalization guidance for ACP-backed turns.

## Best Possible Solution

Revise or consolidate this work so completion delivery has one durable coordination path: preserve structured internal events as authoritative, escape any child output used in plain prompts, persist or derive the parent lineage needed after restart, align prompt-format tests, and add regression coverage for delimiter injection plus restored ACP task delivery.

## Reproduction Assessment

Yes for the PR defects: static inspection gives a high-confidence reproduction path by putting `<<<BEGIN_OPENCLAW_INTERNAL_CONTEXT>>>` in child output and by restoring a task from SQLite after restart. The broader duplicate-delivery behavior is also backed by the linked Discord ACP report, but I did not run live channel reproduction in this read-only review.

## Solution Assessment

No. The PR is aiming at the right ACP completion-delivery problem, but prompt-only guidance plus an unpersisted task field is not the narrowest maintainable solution while #65242 proposes a shared delivery gate for the competing delivery paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Escape completion text before building steer messages:** `src/agents/subagent-announce.ts:209`
  - body: For completion announces, this interpolates `findings` directly into the steer message. Child output is untrusted; if it includes an internal-context delimiter, `prependInternalEventContext()` can treat the body as already wrapped and skip the real structured `internalEvents`. Escape the delimiter before building the visible prompt, or build this text from the sanitized internal-event formatter.
  - confidence: 0.88
- **[P2] Persist spawnedBy before relying on it for delivery:** `src/tasks/task-registry.ts:1027`
  - body: The new silent-wake branch depends on `latest.spawnedBy`, but the default SQLite task row/schema/select/upsert paths still omit that field. After a gateway restart, restored ACP task records lose the metadata and fall back to visible requester delivery, reintroducing the duplicate/noisy completion path this branch is meant to avoid.
  - confidence: 0.91
- **[P3] Align the format expectation with the new guard wording:** `src/agents/subagent-announce.format.e2e.test.ts:561`
  - body: The implementation now says the silent-reply guard applies when `ANY` condition holds, but this changed expectation still looks for `ALL`. The format e2e test will not match the proposed prompt shape, so update the assertion to the new wording.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff introduces a prompt-boundary concern by placing unescaped child completion text in a steer message before structured internal events are attached.
