## Summary

Keep open: the PR targets a real ACP/subagent completion-delivery problem, but current head still has blocking correctness and security issues, lacks after-fix real behavior proof, and overlaps the broader open completion-delivery gate work.

## What This Changes

The PR changes ACP/subagent completion delivery guidance, adds task-level parent-spawn routing metadata, prefers live completion output over cached chatter, suppresses a farewell path, adds ACP turn prompt guidance, and updates related tests.

## Best Possible Solution

Consolidate on one durable completion-delivery ownership path that keeps structured completion events authoritative, sanitizes child output before prompt interpolation, persists or derives parent lineage across restart, and reconciles this branch with https://github.com/openclaw/openclaw/pull/65242 before merge.

## Reproduction Assessment

Yes for the PR defects by source inspection: child output containing an internal-context delimiter can bypass structured-event prepending, and a restored task row loses the new spawnedBy routing field. I did not run a live ACP/channel reproduction in this read-only review.

## Solution Assessment

No. The PR is aimed at the right behavior, but prompt guidance plus non-persisted task lineage is not the narrowest maintainable fix while the broader delivery-gate PR overlaps the same user-visible paths.
