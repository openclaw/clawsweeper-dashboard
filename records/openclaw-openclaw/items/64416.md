## Summary

Keep open. This PR targets a real ACP/subagent completion-delivery problem, but the branch is not correct as-is: it interpolates untrusted completion output before structured internal events, relies on task metadata that is not persisted through the SQLite task store, still lacks real behavior proof, and overlaps the broader open delivery-gate work at https://github.com/openclaw/openclaw/pull/65242.

## What This Changes

The PR adds ACP/subagent completion-delivery prompt guidance, task-level spawnedBy routing metadata, silent parent wake behavior, live-output preference, farewell suppression, and related tests.

## Best Possible Solution

Consolidate on one durable completion-delivery ownership path: keep structured completion events authoritative, sanitize child output before prompt interpolation, persist or derive parent lineage across restart, and reconcile this branch with https://github.com/openclaw/openclaw/pull/65242 before merge.

## Reproduction Assessment

Yes for the PR defects: source inspection gives high-confidence paths by placing an internal-context delimiter in child output and by restoring a parent-spawned ACP task from SQLite after restart. I did not run live ACP/channel reproduction in this read-only review.

## Solution Assessment

No. The PR is aimed at the right completion-delivery area, but prompt guidance plus an unpersisted task field is not the narrowest maintainable solution while current main and the broader delivery-gate PR overlap the same behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Escape completion text before building steer messages:** `src/agents/subagent-announce.ts:207-208`
  - body: For completion announces, this interpolates child findings into the plain steer message before structured internalEvents are attached. If the child output contains an OpenClaw internal-context delimiter, the parent turn can treat the body as already wrapped and skip the authoritative internal event, so escape the text or build the prompt only from the sanitized internal-event formatter.
  - confidence: 0.88
- **[P2] Persist spawnedBy before relying on terminal routing:** `src/tasks/task-registry.ts:1029`
  - body: The new silent-wake branch depends on latest.spawnedBy, but the SQLite task row, select, and upsert paths do not persist that field. After a gateway restart, restored ACP tasks lose the lineage and fall back to the visible requester-delivery path this branch is trying to avoid.
  - confidence: 0.91
- **[P3] Align the format expectation with the guard wording:** `src/agents/subagent-announce.format.e2e.test.ts:561`
  - body: The implementation now says the silent-reply guard applies when ANY condition holds, but this changed format assertion still expects ALL. Update the assertion so the format test proves the intended prompt wording.
  - confidence: 0.86
