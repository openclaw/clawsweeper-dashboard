Keep open: the branch targets a real ACP/subagent completion-delivery problem, but it is not merge-ready because it introduces prompt-boundary handling risk, loses the new routing metadata across task-store restore, has a stale format expectation, lacks live behavior proof, and overlaps active completion-delivery work.

This PR adds ACP/subagent completion-delivery guidance, task-level spawnedBy routing metadata, silent parent wake behavior, live-output preference, farewell suppression, and related tests.

## Best Possible Solution

Consolidate on one durable completion-delivery ownership path that keeps structured completion events authoritative, escapes child output before prompt interpolation, persists or derives parent lineage across restart, and reconciles this branch with https://github.com/openclaw/openclaw/pull/65242.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Escape completion text before building steer messages:** `src/agents/subagent-announce.ts:207-208`
  - body: For completion announces, this injects child findings directly into the trigger message before structured internalEvents are attached. If the child output contains an OpenClaw internal-context delimiter, the parent turn can treat the body as already wrapped and skip the authoritative internal event, so escape the text or build the prompt only from the sanitized internal-event formatter.
  - confidence: 0.9
- **[P2] Persist spawnedBy before routing terminal delivery:** `src/tasks/task-registry.ts:1029`
  - body: The new silent-wake branch depends on latest.spawnedBy, but the SQLite task row, select, schema, and upsert paths do not persist that field. After a gateway restart, restored ACP tasks lose the lineage and fall back to the visible requester-delivery path this branch is trying to avoid.
  - confidence: 0.91
- **[P3] Update the format assertion to match the guard wording:** `src/agents/subagent-announce.format.e2e.test.ts:561`
  - body: The implementation now says the silent-reply guard applies when ANY condition holds, but this changed format assertion still expects ALL. Update the assertion so the format test proves the intended prompt wording.
  - confidence: 0.86
