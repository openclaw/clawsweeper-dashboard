## Summary

Keep open: the PR targets a real ACP/subagent completion-delivery problem, but the proposed patch still has a concrete prompt-boundary security issue, an unpersisted delivery-routing field, and overlap with the broader open #65242 CompletionDeliveryGate approach. Current main does not already implement this PR's requested ACP normalization or shared delivery gate, so this needs revision or consolidation rather than cleanup closure.

## What This Changes

The PR changes ACP/subagent completion delivery by adding task-level spawnedBy metadata, preferring live completion output for completion announces, suppressing a farewell hook, appending ACP capability-normalization guidance, and updating related tests.

## Best Possible Solution

Consolidate on one durable completion-delivery coordination path: keep structured internal completion events authoritative, escape any child output before plain prompt interpolation, persist or derive parent lineage across restart, and cover duplicate-delivery, delimiter-injection, and restored ACP-task cases with focused tests.

## Reproduction Assessment

Yes for the PR defects: static inspection gives a high-confidence reproduction path by placing the internal-context delimiter in child output and by restoring a parent-spawned ACP task from SQLite after restart. The broader duplicate Discord delivery behavior also has concrete steps in #60780, but I did not run live channel reproduction in this read-only review.

## Solution Assessment

No. The PR is aimed at the right ACP completion-delivery problem, but prompt guidance plus an unpersisted task field is not the narrowest maintainable fix while #65242 proposes a shared gate across the competing delivery paths.

Overall confidence: 0.9

Full review comments:

- **[P1] Escape completion text before building steer messages:** `src/agents/subagent-announce.ts:209`
  - body: For completion announces, the PR interpolates child findings into the steer message before the structured internal event is attached. If child output contains an OpenClaw internal-context delimiter, the parent turn can treat the plain body as already wrapped and skip the authoritative internalEvents; escape those delimiters or build the prompt only from the sanitized internal-event formatter.
  - confidence: 0.88
- **[P2] Persist spawnedBy before relying on terminal routing:** `src/tasks/task-registry.ts:1027`
  - body: The new silent-wake path depends on `latest.spawnedBy`, but the SQLite task row/schema/select/upsert paths on main do not persist that field. After a gateway restart, restored ACP tasks lose the lineage and fall back to visible requester delivery, reintroducing the duplicate/noisy path this branch is trying to avoid.
  - confidence: 0.91
- **[P3] Align the format expectation with the guard wording:** `src/agents/subagent-announce.format.e2e.test.ts:561`
  - body: The implementation now says the silent-reply guard applies when `ANY` condition holds, but the changed format expectation still looks for `ALL`. Update the assertion to match the proposed prompt shape so the e2e format check proves the intended wording.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The PR introduces a prompt-boundary concern by placing unescaped child completion text in a steer message before structured internal events are attached.

Concerns:
