## Summary

Keep open: this PR still has blocking prompt-boundary and task-lineage persistence defects, lacks after-fix real ACP/channel proof, and overlaps the broader open CompletionDeliveryGate work.

## What This Changes

The branch changes ACP/subagent completion delivery by adding parent lineage to task records, routing some parent-spawned ACP completions through silent heartbeat wakes, preferring live completion output in prompts, adding ACP capability guidance, and updating related tests.

## Best Possible Solution

Land one consolidated completion-delivery path that keeps structured events authoritative, sanitizes child output before prompt interpolation, preserves parent lineage across restart, reconciles with https://github.com/openclaw/openclaw/pull/65242, and includes redacted live ACP/channel proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: child output containing an internal-context delimiter can bypass structured-event prepending, and restored task rows lose the new `spawnedBy` routing field. I did not run live ACP/channel reproduction in this read-only review.

## Solution Assessment

No. The PR points at a valid delivery bug, but raw prompt interpolation, non-persisted lineage, merge conflicts, and overlap with the broader delivery-gate PR make this branch unsafe as-is.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Escape completion text before composing steer messages:** `src/agents/subagent-announce.ts:207-208`
  - body: This inserts child findings into the plain trigger message before `internalEvents` are attached. If the child output contains an OpenClaw internal-context delimiter, `prependInternalEventContext()` treats the body as already wrapped and skips the authoritative event, so sanitize the text or keep it only in structured-event formatting.
  - confidence: 0.9
- **[P2] Persist spawnedBy before routing terminal updates:** `src/tasks/task-registry.ts:1028-1029`
  - body: The new silent-wake branch depends on `latest.spawnedBy`, but the default SQLite task store still does not persist or restore that field. After a gateway restart, restored ACP tasks lose the lineage and fall back to visible requester delivery.
  - confidence: 0.92
- **[P3] Align format coverage with the ANY guard:** `src/agents/subagent-announce.format.e2e.test.ts:561`
  - body: The implementation now says the silent-reply guard applies when ANY condition holds, but this format expectation still asserts the older ALL wording. Update the assertion so the test proves the intended prompt contract.
  - confidence: 0.86
