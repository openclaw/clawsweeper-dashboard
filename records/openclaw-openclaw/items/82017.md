## Summary

Keep open because current main has ACP backend selection and sandbox backends, but not the requested per-spawn execution placement/profile/resource contract; the related open threads are narrower lifecycle, sandbox, observability, or policy follow-ups rather than a merged canonical implementation.

## What This Changes

The issue requests a new execution placement abstraction so spawned subagents and ACP sessions can select local, container, Kubernetes, or remote-worker backends with per-spawn resource profiles.

## Best Possible Solution

Keep this as an architecture umbrella until maintainers define a versioned execution placement contract, then split implementation across config/schema, sessions_spawn parameters, ACP/runtime APIs, worker lifecycle/status, cleanup, docs, and tests.
