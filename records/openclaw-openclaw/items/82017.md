## Summary

Keep open because current main still has only local subagent dispatch, ACP backend selection, and tool-sandbox backends; it does not have the requested per-spawn execution placement/profile/resource contract, and the related threads are narrower open follow-ups rather than a merged canonical implementation.

## What This Changes

Request a pluggable execution backend and resource-profile layer for spawned subagents and ACP sessions, including Kubernetes/container/remote-worker placement selected from config, bindings, or each sessions_spawn call.

Keep this as an architecture umbrella until maintainers define a versioned execution placement contract, then split implementation across config/schema, sessions_spawn parameters, ACP/runtime APIs, worker lifecycle/status, cleanup, docs, and tests.
