## Summary

Keep open because current main still has local subagent dispatch, ACP backend selection, and tool-sandbox backends, but no per-spawn execution placement/resource-profile contract for Kubernetes, containers, or remote workers.

## What This Changes

The issue requests pluggable execution backends and resource profiles for spawned subagents and ACP sessions, including Kubernetes/container/remote worker placement.

## Best Possible Solution

Keep this as an architecture umbrella until maintainers define a versioned execution placement contract, then split implementation across config/schema, sessions_spawn parameters, ACP/runtime APIs, worker lifecycle/status, cleanup, docs, and tests.
