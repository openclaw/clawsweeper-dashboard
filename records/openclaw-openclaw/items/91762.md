## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Gate trusted lifecycle privateData behind an explicit plugin grant:** `src/infra/diagnostic-events.ts:1206-1208`
  - body: `onTrustedDiagnosticEvent` is exported from the public diagnostic-runtime subpath, so any plugin can subscribe to `privateData.clientContext`. Current private diagnostic data is granted only to bundled/trusted official diagnostics services through `ctx.internalDiagnostics` in `src/plugins/services.ts`, and this PR's premise is that `clientContext` is withheld from public subscribers. Please keep this behind an explicit service/capability grant, or get maintainer approval for making this private attribution visible to every plugin.
  - confidence: 0.84
- **[P2] Forward clientContext through the TypeScript SDK:** `packages/gateway-protocol/src/schema/agent.ts:246`
  - body: Adding `clientContext` to the Gateway `agent` schema makes it a supported run parameter, but the TypeScript SDK still type-limits `AgentRunParams` and `buildAgentParams` allowlists fields without forwarding `clientContext`. SDK-based gateway clients will silently drop the attribution bag, so the feature only works for raw RPC or Swift clients; add the SDK field, forwarding, and a client test, or document the feature as raw-RPC-only.
  - confidence: 0.82
- **[P3] Measure the context limit in bytes:** `src/infra/diagnostic-client-context.ts:126`
  - body: `CLIENT_CONTEXT_MAX_BYTES` is enforced with `JSON.stringify(normalized).length`, which counts UTF-16 code units rather than encoded bytes. A caller can exceed the advertised 8 KiB cap with non-ASCII content; use `Buffer.byteLength(..., 'utf8')` or rename the cap to a character limit.
  - confidence: 0.86
