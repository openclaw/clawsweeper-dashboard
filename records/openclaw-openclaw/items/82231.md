## Summary

Keep open. This is the canonical current OpenAI Codex OAuth failure: current main still has a local-expiry status path that can report OAuth as healthy while the Codex app-server first-request path forces a real token refresh and can surface the reported 401.

## What This Changes

The issue asks for fresh openai-codex browser/device-code OAuth credentials to work on the first agent request and for models status to stop reporting unusable credentials as ok.

## Best Possible Solution

Fix the OpenAI Codex OAuth runtime so fresh browser/device-code credentials survive the first Codex app-server refresh, and make status/probe output surface unusable tokens accurately.
