## Summary

Keep open. The report is a fresh current-version OpenAI Codex OAuth failure, and current main still has a status path that can report OAuth as healthy from local expiry metadata while the Codex app-server runtime can force a real refresh and surface the reported 401.

## What This Changes

The issue reports that fresh openai-codex OAuth login is accepted by status checks, but the first real runtime request fails with a 401 token validation error.

## Best Possible Solution

Fix the OpenAI Codex OAuth runtime so browser/device-code credentials are usable by the first Codex app-server request, and make status/probe output surface unusable tokens accurately.

## Reproduction Assessment

No high-confidence live reproduction was run. The reporter provides concrete steps and source inspection confirms the status-only health path and the forced runtime refresh path that can produce the reported mismatch.

## Solution Assessment

Unclear; no patch is proposed. The narrow maintainable direction is to investigate the forced Codex app-server refresh/login-token contract and add regression coverage for this 401 validation failure, not to close it as one of the older external-CLI OAuth reports.
