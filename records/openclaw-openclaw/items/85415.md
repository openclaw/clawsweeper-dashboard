## Summary

Keep open. The image-runtime fix is useful, but the PR is not merge-ready because the new Codex docs say the image timeout is not reduced by preparation work while Codex still applies the same setting as the whole dynamic-tool-call watchdog.

## What This Changes

## Best Possible Solution
