## Summary

Keep open for maintainer handling: this protected, member-authored PR has a focused source-level fix and targeted tests for the image timeout budget collapse, but it deliberately changes the timeout contract from a shared setup+request budget to phase-level budgets, so maintainer approval of that behavior should precede merge.

## What This Changes

The branch changes media-understanding image timeout handling so setup and provider request timeouts have distinct phase errors and the provider request receives the full configured timeout, with focused tests for request, setup, and slow-setup behavior.

## Best Possible Solution

Land the focused implementation if maintainers want phase-level setup and request budgets; otherwise preserve a separate whole-operation watchdog while still giving the provider request its full configured timeout.

## Solution Assessment
