## Summary

Keep open: current main and v2026.5.12 still have the reasoning-marker/cron-selector gap, and this PR contains useful fallback and recovery intent that is not fully superseded. The branch is stale against the current cron outcome helpers, overlaps newer open work, and still lacks contributor-supplied real behavior proof.

## What This Changes

The PR propagates `isReasoning` through embedded-runner payload output and updates cron isolated announce selection, fallback, recovery logic, and helper tests to avoid delivering reasoning text.

## Best Possible Solution

Land one canonical current-main fix that preserves `isReasoning` through embedded payload construction and excludes reasoning from cron summary, output, delivery, synthesized fallback, and post-error recovery, backed by redacted live cron announce proof.

## Reproduction Assessment

Yes, at source level. Current main builds reasoning reply items, drops the marker when returning payloads, and cron selectors can choose text using content and `isError` checks only; I did not run a live cron channel reproduction in this read-only pass.

## Solution Assessment

No as-is. Propagating and filtering `isReasoning` is the right direction, but this branch must be rebased onto the current payload map and cron outcome helpers, then backed by real behavior proof.
