## Summary

Keep open: the change is aimed at a real privacy-sensitive cron announce bug, but this branch is stale against current `main`, lacks after-fix real behavior proof, and overlaps open canonical work at https://github.com/openclaw/openclaw/pull/73363 plus https://github.com/openclaw/openclaw/issues/73186.

## What This Changes

The PR propagates `isReasoning` into embedded runner result payloads and filters cron announce summary, output, delivery, and recovery paths to avoid delivering reasoning text.

## Best Possible Solution

Land one canonical current-main fix that preserves `isReasoning` through embedded payload construction and excludes reasoning from cron summary, output, delivery, synthesized fallback, and post-error recovery, backed by redacted live cron announce proof.

## Reproduction Assessment

Yes, at source level. Current `main` creates reasoning payload items, drops or ignores the marker before cron outcome selection, and can select nonempty reasoning text for summary, delivery, or recovery checks.

## Solution Assessment

No as-is. Propagating and filtering `isReasoning` is the right direction, but this branch needs to be rebased onto the current payload map and `resolveCronPayloadOutcome()` helpers, then proven in a real cron announce run.
