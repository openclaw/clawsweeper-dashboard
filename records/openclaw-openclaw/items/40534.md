## Summary

Keep open: this PR targets a real cron announce reasoning leak, but it is not merge-ready against current main and still lacks contributor-supplied real behavior proof. The overlapping open issue and PR should inform the canonical fix, but they do not make this broader no-fallback/no-recovery branch obsolete yet.

## What This Changes

The PR propagates `isReasoning` from embedded runner payloads and filters cron announce helper, fallback, and recovery selection so reasoning payloads are not delivered.

## Best Possible Solution

Land one canonical current-main fix that preserves `isReasoning` through embedded payload construction and treats reasoning as non-user-visible across cron summary, output, delivery payloads, synthesized fallback, first-text fallback, and post-error recovery, with redacted real cron announce proof.

## Reproduction Assessment

Yes, at source level. Current main creates reasoning payload items, drops or ignores `isReasoning`, and cron announce selectors/recovery checks still choose text using content and `isError` checks only.

## Solution Assessment

No as-is. Propagating and filtering `isReasoning` is the right direction, but this branch must be rebased onto current `resolveCronPayloadOutcome()` and `pickDeliverablePayloads()` paths and backed by real behavior proof.
