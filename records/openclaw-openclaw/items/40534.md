## Summary

## What This Changes

The PR propagates `isReasoning` into embedded-runner payloads and filters cron announce summary, delivery, fallback, and post-error recovery paths, with helper tests.

## Best Possible Solution

Land one canonical current-main fix that preserves `isReasoning` through embedded payload construction and treats reasoning as non-user-visible across cron summary, output, delivery payloads, synthesized fallback, and post-error recovery, backed by redacted live cron announce proof.

## Reproduction Assessment

Yes, at source level. Current main creates reasoning payload items, drops the marker before returning payloads, and cron selectors can choose and deliver nonempty text using only content and `isError` checks.

## Solution Assessment

No as-is. Propagating and filtering `isReasoning` is the right direction, but this stale branch must be rebased onto the current cron outcome helpers and backed by real behavior proof before merge.
