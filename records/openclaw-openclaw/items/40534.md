## Best Possible Solution

Land one canonical cron announce fix that preserves `isReasoning`, suppresses reasoning across current summary/output/delivery/error-recovery selectors, adds focused regression coverage, and includes redacted real behavior proof from a thinking-enabled cron run.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: reasoning is marked internally, the marker is dropped before returned payloads, and cron selectors can still choose non-error reasoning text for announce delivery or post-error recovery.

## Solution Assessment

No as-is. Propagating and filtering `isReasoning` is the right direction, but the stale branch must be rebased onto the current `resolveCronPayloadOutcome()` path and cover `pickDeliverablePayloads()` plus the moved post-error recovery check.
