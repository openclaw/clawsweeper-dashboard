## Summary

Keep open: current main still has the reasoning-marker gap in the embedded payload and cron announce selection path, while this external PR is stale against the refactored cron helpers and lacks required real behavior proof.

## What This Changes

The PR propagates `isReasoning` from embedded-runner payloads and tries to exclude reasoning text from cron announce summary, delivery, fallback, and post-error recovery paths.

## Best Possible Solution

Land one current-main fix that preserves `isReasoning` through embedded payload construction and treats reasoning as non-user-visible across cron summary, output, delivery payloads, synthesized fallback, and post-error recovery, backed by redacted live cron announce proof.

## Reproduction Assessment

Yes, at source level. Current main creates reasoning payload items, drops the marker before returning payloads, and cron selectors can choose and deliver nonempty text using only content and `isError` checks.

## Solution Assessment

No as-is. Propagating and filtering `isReasoning` is the right direction, but this branch must be rebased onto the current cron outcome helpers and backed by real behavior proof before merge.

  - confidence: 0.84

  - confidence: 0.82
