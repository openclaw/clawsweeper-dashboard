## Summary

Keep open: the branch addresses a real reasoning leak, but current main moved cron outcome selection into shared helpers, so the stale caller-local edits are incomplete and the external PR lacks real cron proof.

## What This Changes

The PR propagates `isReasoning` through embedded run payloads, filters reasoning in older cron payload helpers and fallback status logic, and adds helper tests.

## Best Possible Solution

Land one rebased canonical fix that preserves `isReasoning` and excludes reasoning from summary, output, delivery, synthesized fallback, and post-error recovery in the shared cron outcome path, with regression tests and redacted real cron announce proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `buildEmbeddedRunPayloads()` creates reasoning items, drops `isReasoning` in the returned map, and `resolveCronPayloadOutcome()`/`pickDeliverablePayloads()` select text without a reasoning check; I did not run a live cron job.

## Solution Assessment

No. The branch's direction is right, but the maintainable fix must target shared `resolveCronPayloadOutcome()` and multi-payload selectors on current main instead of older caller-local fallback logic.
