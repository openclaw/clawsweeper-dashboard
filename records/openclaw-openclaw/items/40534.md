## Summary

Keep open: the branch is aimed at a real reasoning-leak bug, but it is stale against current main and would still miss the shared cron outcome and multi-payload delivery paths. It also lacks the required after-fix real behavior proof for an external PR.

## What This Changes

The PR propagates `isReasoning` on embedded run payloads, filters reasoning in older isolated-cron payload helpers and caller fallback/error-recovery logic, and adds helper tests.

## Best Possible Solution

Land one rebased canonical fix that preserves `isReasoning` and excludes reasoning from summary, output, delivery payloads, synthesized fallback, and post-error recovery in the shared cron outcome path, with regression tests and redacted real cron announce proof.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction path on current main: reasoning text is marked internally, the returned payload drops that marker, and shared cron selectors choose text using content and `isError` checks only. I did not run a live cron job.

## Solution Assessment

No as-is. The branch direction is right, but the maintainable fix needs to target current main's shared `resolveCronPayloadOutcome()` and multi-payload selectors rather than older caller-local fallback logic.
