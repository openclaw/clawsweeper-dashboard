## Summary

Keep open: this PR targets a real privacy-sensitive cron announce bug, but the branch is stale/conflicting and does not update the current `resolveCronPayloadOutcome()` and `pickDeliverablePayloads()` paths. The overlapping https://github.com/openclaw/openclaw/pull/73363 and https://github.com/openclaw/openclaw/issues/73186 should be handled as the canonical path, but the stricter “reasoning is never deliverable” behavior here is still useful review context rather than a safe cleanup close.

## What This Changes

The PR propagates `isReasoning` from embedded-runner payload construction, filters reasoning payloads from older cron announce helper/fallback paths, and adds helper tests.

## Best Possible Solution

Land one canonical fix that preserves `isReasoning` through embedded payloads and excludes reasoning from cron summary, output, deliverable, synthesized fallback, first-text fallback, and post-error recovery decisions, with focused tests and redacted real cron proof.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: reasoning text is marked internally, the returned payload drops the marker, and cron selectors/recovery checks still select text using content and `isError` checks only.

## Solution Assessment

No as-is. Propagating and filtering `isReasoning` is the right direction, but this branch must be rebased onto the current `resolveCronPayloadOutcome()` and `pickDeliverablePayloads()` paths and backed by real behavior proof before it is the narrow maintainable fix.
