## Summary

Keep open: the PR is aimed at a real cron announce reasoning leak, but it is stale against current main's shared outcome resolver and still needs canonical branch handling with https://github.com/openclaw/openclaw/pull/73363 plus real behavior proof before merge.

## What This Changes

The PR propagates `isReasoning` through embedded run payloads, adds it to the result type, filters reasoning from older cron selection and fallback logic, and adds helper tests.

## Best Possible Solution

Land one rebased canonical fix that preserves `isReasoning` and suppresses reasoning in shared cron summary, output, delivery selection, synthesized fallback, and post-error recovery, with focused regression tests and redacted real cron announce proof.

## Reproduction Assessment

Yes. Source inspection on current main shows reasoning payloads are created, the marker is dropped in the returned payload map, and cron outcome/delivery selectors choose text without reasoning checks; I did not run a live cron job.

## Solution Assessment

No as-is. The branch direction is right, but the maintainable fix must move the checks into current main's shared `resolveCronPayloadOutcome()` and multi-payload selector paths before merge.
