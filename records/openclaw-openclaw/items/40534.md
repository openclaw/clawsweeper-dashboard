## Summary

Keep open: the reasoning-leak bug is still source-reproducible on current main and this PR contains useful stricter behavior, but it is not merge-ready because it targets pre-refactor cron outcome code, lacks real behavior proof, and overlaps https://github.com/openclaw/openclaw/pull/73363.

## What This Changes

The PR adds `isReasoning` to embedded-runner payload output/types and updates isolated cron announce helper and fallback logic with tests so reasoning payloads are not delivered.

## Best Possible Solution

Land one canonical current-main fix that preserves `isReasoning` through embedded payload construction and suppresses reasoning across cron summary, output, all deliverable payloads, synthesized fallback, and post-error recovery, then close the related open issue after merge.

## Reproduction Assessment

Yes, at source level. Current main creates reasoning payload items, drops the `isReasoning` marker before returning payloads, and cron selectors choose non-error deliverable text without reasoning checks.

## Solution Assessment

No, not in the current PR shape. The fix direction is right, but it needs a current-main port through `resolveCronPayloadOutcome()` and `pickDeliverablePayloads()` plus live cron announce proof before it is the best merge path.
