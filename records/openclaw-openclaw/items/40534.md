## Summary

Keep open: the bug is real, but this PR is stale against current main and incomplete as-is; it updates older caller-local cron logic while current main uses the shared outcome and multi-payload delivery helpers, and it overlaps https://github.com/openclaw/openclaw/pull/73363 without real behavior proof.

## What This Changes

The branch propagates `isReasoning` through embedded runner payloads, filters reasoning from cron payload selection and fallback status logic, and adds focused helper tests.

## Best Possible Solution

Land one canonical rebased fix that preserves `isReasoning` and suppresses reasoning in shared cron summary, output, deliverable payloads, synthesized fallback, and post-error recovery, with regression tests and redacted live announce proof.

## Reproduction Assessment

Yes. Source inspection on current main shows reasoning text is marked internally, the returned payload drops that marker, and cron outcome/delivery selectors choose text using content and `isError` checks only; I did not run a live cron job.

## Solution Assessment

No as-is. The direction is right, but the fix must move into current main's shared `resolveCronPayloadOutcome()` and multi-payload selector paths before this PR can safely merge.
