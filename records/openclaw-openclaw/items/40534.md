## Summary

Keep open. Current main still has the cron announce reasoning leak, and this PR contains useful source work, but the branch is dirty against main and misses the newer shared cron outcome boundary.

## What This Changes

The PR propagates `isReasoning` from embedded-runner payload construction, filters reasoning in cron helper selection plus older run-level fallback/recovery paths, and adds helper regression tests.

## Best Possible Solution

Land one canonical rebased fix in the shared cron outcome path that preserves `isReasoning`, keeps reasoning non-user-visible in summary/output/delivery and error recovery, preserves real error fallback, and adds the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: reasoning payloads are created, the returned payload map drops the marker, and `resolveCronPayloadOutcome()` selects text using helpers that only understand `isError`.

## Solution Assessment

No. The PR fixes important older call sites, but current main centralizes final and interim cron behavior in `resolveCronPayloadOutcome()` and has a newer multi-payload selector that this branch does not update.
