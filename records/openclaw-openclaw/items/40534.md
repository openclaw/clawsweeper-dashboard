## Summary

Keep open. The underlying cron announce reasoning leak still exists on current main, and this PR contains useful source work, but it is not correct as-is against the current shared cron outcome path and overlaps with newer open #73363, which disagrees on reasoning-only fallback behavior.

## What This Changes

The PR preserves `isReasoning` on embedded-runner payload results, adds it to the embedded result type, filters reasoning payloads from cron announce helper selection plus older caller fallback/recovery logic, and adds helper regression tests.

## Best Possible Solution

Land one canonical rebased fix in the shared cron outcome path: propagate `isReasoning` from `buildEmbeddedRunPayloads`, include it in the cron payload type, skip reasoning in summary/output, single-payload and multi-payload delivery, and post-error recovery, then add regression coverage for mixed reasoning plus final text, reasoning-only, and error-then-reasoning-only announce cases with the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: reasoning payloads are created with `isReasoning`, the return map drops the marker, and `resolveCronPayloadOutcome()` then selects payloads using helpers that only understand `isError`.

## Solution Assessment

No. This PR fixed important older call sites, but current main centralizes final and interim cron delivery through `resolveCronPayloadOutcome()` and has a multi-payload selector that the PR does not update.
