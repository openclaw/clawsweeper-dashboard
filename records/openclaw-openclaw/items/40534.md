## What This Changes

The PR propagates `isReasoning` from embedded runner payloads, filters reasoning payloads out of cron announce selection/fallback/error-recovery paths, and adds helper tests.

## Best Possible Solution

Land one canonical fix that preserves `isReasoning` through embedded payloads and excludes reasoning from shared cron summary, output, delivery, fallback, and error-recovery selectors, with focused tests and redacted real cron proof.

## Reproduction Assessment

Yes. Source inspection on current main shows reasoning payloads are created, the returned payload mapping drops the marker, and cron selectors/recovery checks can select non-error reasoning text; I did not run a live cron job.

## Solution Assessment

No as-is. Propagating and filtering `isReasoning` is the right direction, but the branch must be rebased onto the current `resolveCronPayloadOutcome()` and `pickDeliverablePayloads()` paths before it is the narrow maintainable fix.
