## Summary

Keep open: the branch is a plausible fix direction for a real cron announce reasoning leak, but it is now conflicting, targets older code shapes, and still lacks after-fix real cron announce proof. Current main remains source-reproducible, and the overlapping work in https://github.com/openclaw/openclaw/pull/73363 is also still open rather than merged.

## What This Changes

The PR propagates `isReasoning` on embedded-runner payloads and filters reasoning payloads from cron announce summary, delivery, fallback, and post-error recovery selection paths.

## Best Possible Solution

Land one canonical current-main fix that preserves `isReasoning` through embedded payload construction and treats reasoning as non-user-visible across cron summary, output, delivery payloads, synthesized fallback, and post-error recovery, backed by redacted real cron announce proof.

## Reproduction Assessment

Yes, at source level. Current main creates reasoning payload items, drops the marker before returning payloads, and cron selectors can choose and deliver nonempty text using only content and `isError` checks.

## Solution Assessment

No as-is. Propagating and filtering `isReasoning` is the right direction, but this branch must be rebased onto the current cron outcome helpers and backed by real behavior proof before merge.
