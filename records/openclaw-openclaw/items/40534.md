## Summary

Keep open: the bug is real, but this branch is stale against current main and incomplete for the shared cron outcome and multi-payload delivery path; it also lacks after-fix real cron proof and overlaps https://github.com/openclaw/openclaw/pull/73363.

## What This Changes

The PR propagates `isReasoning` through embedded run payloads, updates isolated-cron payload selection and fallback checks to skip reasoning, and adds helper tests.

## Best Possible Solution

Land one rebased canonical fix that preserves `isReasoning` and excludes reasoning from summary, output, direct delivery payloads, synthesized fallback, and post-error recovery in `resolveCronPayloadOutcome()`, with regression tests and redacted real cron announce proof.

## Reproduction Assessment

Yes. Source inspection on current main shows reasoning payloads are created, the returned mapper drops the marker, and cron selectors choose text/delivery using only content and `isError`; I did not run a live cron job.

## Solution Assessment

No. The branch direction is right, but the maintainable fix must target current main's shared `resolveCronPayloadOutcome()` and `pickDeliverablePayloads()` paths instead of older caller-local fallback logic.
