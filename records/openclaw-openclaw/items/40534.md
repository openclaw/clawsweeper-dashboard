## Summary

Keep open: the underlying bug is still source-reproducible on current main, but this PR is stale against the refactored cron outcome path and would not fully suppress reasoning in multi-payload announce delivery after a rebase. It also lacks required real behavior proof and overlaps https://github.com/openclaw/openclaw/pull/73363, so this needs human canonical-branch handling before merge.

## What This Changes

This PR propagates `isReasoning` through embedded runner payloads, filters reasoning from older cron announce selectors and fallback status logic, and adds focused helper tests.

## Best Possible Solution

Land one canonical rebased fix that preserves `isReasoning` and suppresses reasoning across shared cron summary, output, deliverable payloads, synthesized fallback, and post-error recovery, with regression tests and redacted live announce proof.

## Reproduction Assessment

Yes. Source inspection on current main shows reasoning text is marked internally, then the returned embedded payload drops that marker, and cron announce selectors choose text using content and `isError` checks only; I did not run a live cron job.

## Solution Assessment

No as-is. The direction is right, but the fix must move into current main's shared `resolveCronPayloadOutcome()` and `pickDeliverablePayloads()` paths and be reconciled with https://github.com/openclaw/openclaw/pull/73363 before merge.
