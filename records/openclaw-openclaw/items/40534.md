## Summary

Keep open: current main still has the reasoning-marker/drop-and-select path, and this PR is the right direction but is stale/conflicting, targets older cron fallback code, and lacks contributor-supplied real behavior proof.

## What This Changes

The PR propagates `isReasoning` through embedded runner payloads and filters reasoning payloads from cron announce summary, delivery, fallback, and post-error recovery selection.

## Best Possible Solution

Land one current-main fix that preserves `isReasoning` through embedded payloads and treats reasoning as non-user-visible across cron summary, output, delivery payloads, synthesized fallback, first-text fallback, and post-error recovery, with real cron announce proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: reasoning text is marked internally, the returned payload drops the marker, and current cron selectors/recovery checks still choose text using content and `isError` checks only.

## Solution Assessment

No as-is. Propagating and filtering `isReasoning` is the right direction, but this branch must be rebased onto the current `resolveCronPayloadOutcome()` and `pickDeliverablePayloads()` paths and backed by real behavior proof.
