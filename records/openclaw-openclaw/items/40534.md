## What This Changes

The PR propagates `isReasoning` through embedded-runner payloads and filters reasoning payloads from cron announce summary, delivery, fallback, and recovery selection paths.

## Best Possible Solution

Land one canonical current-main fix that preserves `isReasoning` through embedded payload construction and treats reasoning as non-user-visible across cron summary, output, delivery payloads, synthesized fallback, first-text fallback, and post-error recovery, with redacted real cron announce proof.

## Reproduction Assessment

Yes, at source level. Current main creates reasoning payload items, drops or ignores the marker before cron outcome resolution, and cron selectors can choose nonempty reasoning text by checking only content and `isError`.

## Solution Assessment

No as-is. Propagating and filtering `isReasoning` is the right direction, but this branch must be rebased onto the current `resolveCronPayloadOutcome()` and `pickDeliverablePayloads()` paths and backed by real behavior proof.
