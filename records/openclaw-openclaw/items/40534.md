## Summary

Keep open: current main still has the reasoning-leak source path, but this PR is stale, conflicting, lacks real behavior proof, and misses the newer `resolveCronPayloadOutcome()` delivery paths that now own the affected logic.

## What This Changes

The PR propagates `isReasoning` from embedded-runner payload construction and filters reasoning payloads from cron announce delivery selection.

## Best Possible Solution

Land one current-main fix that preserves `isReasoning` through embedded payloads and excludes reasoning from cron summary, output, delivery, synthesized fallback, first-text fallback, and post-error recovery decisions.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: reasoning text is marked internally, the returned payload drops the marker, and cron selectors/recovery checks still select text using content and `isError` checks only.

## Solution Assessment

No as-is. Propagating and filtering `isReasoning` is the right direction, but this branch must be rebased onto the current `resolveCronPayloadOutcome()` and `pickDeliverablePayloads()` paths and backed by real behavior proof.
