## Summary

Keep open: current main has manual browser profiles, `profile="user"` attachment, and session-scoped tab cleanup, but it still lacks first-class subagent browser ownership, explicit parent-browser attachment, auth-state mismatch reporting, and `/agent-browser` owner visibility.

## What This Changes

The issue requests first-class browser session ownership and attachment semantics for subagents, including dashboard visibility and mismatch reporting when logged-in browser state is expected.

## Best Possible Solution

Design an explicit browser-session ownership contract covering per-agent profile binding, opt-in parent-browser inheritance/attachment, auth-state mismatch reporting, and dashboard owner visibility, coordinated with the broader browser isolation plan.

## Reproduction Assessment

Yes for the capability gap: current source shows manual browser profile selection and subagent session lineage, but no first-class browser attachment or owner metadata. This is not a strict broken-contract bug reproduction because the requested behavior is a new product/API contract.

## Solution Assessment

Unclear as an implementation path: the requested direction is valid, but a hidden default inheritance change would be risky. The best path is explicit API/config/runtime ownership plus dashboard visibility and security review.
