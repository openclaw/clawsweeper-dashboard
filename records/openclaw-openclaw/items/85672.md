## Summary

Keep open: current main and v2026.6.9 still expose manual browser profile selection plus subagent lineage, but not a first-class browser ownership, parent-attachment, or dashboard auth-mismatch contract for subagents.

## What This Changes

The issue requests explicit subagent browser ownership or inheritance semantics, routing controls, and operator-visible `/agent-browser` ownership state for logged-in browser workflows.

## Best Possible Solution

Define a maintainer-approved, explicit opt-in browser ownership model covering per-agent profile binding, parent-browser attach/inherit, fail-closed mismatch reporting, and dashboard owner visibility.

## Reproduction Assessment

Yes for the capability gap: current source shows manual per-call browser profile selection and subagent lineage, but no first-class browser ownership, inheritance, or dashboard owner/auth metadata. This is not a strict broken-contract bug reproduction.

## Solution Assessment

Unclear as an implementation path: the direction is valid, but hidden inheritance would be risky. The best path is an explicit opt-in API/config/runtime ownership model with dashboard visibility and security review.
