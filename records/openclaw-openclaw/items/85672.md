## Summary

Keep open: current main has manual browser profile selection and tracked tab cleanup, but subagent spawning does not carry a first-class browser profile/session ownership contract, explicit parent-browser attachment, auth-state mismatch reporting, or dashboard visibility. The related browser-isolation issue at https://github.com/openclaw/openclaw/issues/37487 overlaps, but it does not fully own this logged-in subagent attachment case.

## What This Changes

Requests first-class subagent browser session ownership, explicit parent-browser inheritance or attachment, routing controls, and dashboard visibility for logged-in browser workflows.

## Best Possible Solution

Design an explicit browser-session ownership contract covering per-agent profile binding, opt-in parent-browser inheritance/attachment, auth-state mismatch reporting, and dashboard owner visibility, coordinated with the broader browser isolation work.

## Reproduction Assessment

Not applicable as a strict bug reproduction: the expected behavior is a new first-class browser ownership and attachment contract. Source inspection does confirm current main lacks a sessions_spawn browser session/profile parameter and only offers manual browser-tool profile selection.

## Solution Assessment

Unclear: a hidden default inheritance change would be risky. The maintainable path needs explicit product and security design for dedicated versus inherited browser behavior, auth-state checks, and dashboard visibility.
