## Summary

Keep open: current main still has manual browser profiles and `profile="user"` attachment, but subagent spawn/session state does not carry a first-class browser ownership or parent-attachment contract, and browser profile status does not expose owner/auth mismatch visibility.

## What This Changes

Request first-class subagent browser ownership, attachment/inheritance controls, routing defaults, and dashboard visibility for logged-in browser sessions.

## Best Possible Solution

Define a maintainer-approved browser ownership contract covering per-agent default profile binding, explicit parent-browser attach/inherit, fail-closed auth-state mismatch reporting, and dashboard owner visibility.

## Reproduction Assessment

Yes for the capability gap: current source shows manual per-call browser profile selection and subagent lineage, but no first-class browser ownership/attachment or dashboard owner metadata. This is not a strict broken-contract bug reproduction.

## Solution Assessment

Unclear as an implementation path: the direction is valid, but hidden default inheritance would be risky. The best path is an explicit opt-in API/config/runtime ownership model with dashboard visibility and security review.
