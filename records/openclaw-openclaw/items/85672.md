## Label Justifications

- `P2`: This is a normal-priority feature/product gap for multi-agent browser workflows with limited but real workflow impact.
- `impact:session-state`: The issue is about subagents drifting from the intended logged-in browser/session state and lacking explicit ownership metadata.
- `impact:security`: The requested inheritance/attachment model involves logged-in browser cookies and access to private or SSO-protected sites.

## Summary

Keep open: current main supports manual named browser profiles and logged-in `profile="user"` attachment, but subagent spawning only carries session lineage, transcript context, workspace, model/tool policy, and channel routing. The requested first-class browser ownership, explicit parent-browser attachment, auth-state mismatch reporting, and `/agent-browser` ownership visibility are not implemented, and the related browser-isolation issue at https://github.com/openclaw/openclaw/issues/37487 overlaps without fully owning this logged-in subagent attachment problem.

## What This Changes

Add first-class browser session ownership/routing for subagents so they can intentionally attach to a parent’s logged-in browser or get a clearly separate dedicated browser.

## Best Possible Solution

Design a browser-session ownership contract that covers per-agent profile binding, explicit parent-browser inheritance/attachment, auth-state mismatch reporting, and dashboard owner visibility, coordinated with https://github.com/openclaw/openclaw/issues/37487.

## Reproduction Assessment

Not applicable as a strict bug reproduction: the expected behavior is a new first-class browser ownership and attachment contract. Source inspection does confirm current main lacks a `sessions_spawn` browser session/profile parameter and only offers manual browser-tool profile selection.

## Solution Assessment

Unclear: a narrow hidden default change would be risky here. The maintainable solution needs explicit product/security design for inherit versus dedicated behavior, auth-state checks, and dashboard visibility.
