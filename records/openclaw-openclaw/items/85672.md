## Summary

Keep open: current main and the latest release still provide manual browser profile selection, subagent session isolation, and tab cleanup, but not first-class subagent browser ownership, explicit parent-browser attachment, or dashboard owner/auth-state visibility.

## What This Changes

The issue requests first-class browser session ownership, explicit inheritance or attachment semantics, and dashboard observability for subagents using logged-in agent-browser sessions.

## Best Possible Solution

Define an explicit, maintainer-approved browser ownership model covering per-agent profile binding, opt-in parent-browser attach/inherit, mismatch reporting, and dashboard owner visibility.

## Reproduction Assessment

Yes for the capability gap: current source shows manual browser profile selection, subagent context modes, and profile status, but no first-class browser ownership, inheritance, or dashboard owner/auth metadata. This is not a strict broken-contract bug reproduction.

## Solution Assessment

Unclear as an implementation path: the direction is valid, but hidden inheritance would be risky. The best path is an explicit opt-in API/config/runtime ownership model with dashboard visibility and security review.
