## Summary

Keep open: current main and v2026.6.8 still provide manual browser profiles, `profile="user"` existing-session attach, and session-scoped tab cleanup, but they do not provide first-class subagent browser ownership, explicit parent-browser inheritance/attachment, auth-state mismatch reporting, or `/agent-browser` owner visibility.

## What This Changes

Request first-class browser session ownership and attachment semantics so subagents can explicitly inherit a logged-in browser or receive an isolated, visible browser session.

## Best Possible Solution

Design an explicit browser-session ownership contract covering per-agent profile binding, opt-in parent-browser inheritance/attachment, auth-state mismatch reporting, and dashboard owner visibility, coordinated with the broader isolation plan.

## Reproduction Assessment

Yes for the source-level capability gap: current main exposes manual browser profile selection and subagent lineage, but no first-class browser attachment or owner metadata. This is not a strict broken-contract bug reproduction because the expected behavior is a new product/API contract.

## Solution Assessment

Unclear as a concrete implementation path: the direction is valid, but silent default inheritance would be risky. The best path is explicit API/config/runtime ownership plus dashboard visibility and security review.
