## Label Justifications

- `P2`: The issue affects long-running parent/subagent UX across surfaces, but the scope is bounded to yielded sessions rather than a core runtime outage.
- `impact:session-state`: The missing yielded-parent state makes a session appear idle/done/frozen while it is actually waiting on child completion.
- `impact:message-loss`: Connected parent surfaces receive no intermediate wait/progress or streamed child events during the native subagent wait.
- `impact:security`: The requested full native stream relay crosses visibility and redaction boundaries for child tool output and other sensitive progress data.

## Summary

Keep open: current main still has no parent-visible wait/progress contract for native subagent `sessions_yield`, and the related open items only cover parts of the requested behavior.

## What This Changes

The issue requests a parent-visible waiting/progress state and eventual native subagent stream forwarding while a yielded parent session waits for spawned subagent completion.

## Best Possible Solution

Define a generic yielded-parent wait state that Gateway, Control UI, and TUI can render as waiting, then handle full native `streamTo="parent"` in the security-reviewed design tracked by https://github.com/openclaw/openclaw/issues/47597.

## Reproduction Assessment

Yes, at source level: native `sessions_spawn` ignores `streamTo`, `agent.wait` only resolves terminal snapshots, and current UI/session lifecycle types have no yielded-parent wait state. I did not run a live Control UI or TUI session in this read-only pass.

## Solution Assessment

Unclear until maintainers choose the lifecycle and streaming contract. The narrow first step is a visible yielded-parent wait state; full native stream forwarding should remain a separate security-reviewed API decision.
