## Summary

Keep open. Current main still has the reported source-level gap: native subagent spawns do not relay parent streaming, yielded parent lifecycle state is not persisted as a visible waiting/paused state, and both TUI and Control UI terminal handlers clear active run state instead of rendering a yielded-child wait.

## What This Changes

The issue asks for parent-session progress or waiting visibility, and ideally native subagent output streaming, while `sessions_yield` waits for spawned subagents.

## Best Possible Solution

Define a generic yielded-parent lifecycle/wait contract, render that requester-scoped wait in Control UI and TUI, and keep full native `streamTo: "parent"` streaming under the separate API design item.

## Reproduction Assessment

Yes, at source level. A native subagent spawn followed by `sessions_yield` leaves the parent run terminal/idle while child completion is monitored through `agent.wait`, and current Control UI/TUI handlers do not render a yielded-child wait state.

## Solution Assessment

Unclear until maintainers choose the lifecycle and visibility contract. A minimal waiting indicator is the narrowest useful fix, while full native subagent streaming should remain behind the separate API/redaction-policy decision.
