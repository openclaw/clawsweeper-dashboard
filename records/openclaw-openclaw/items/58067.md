## Summary

Keep open. Current main still lacks a parent-visible wait/progress path for native subagent `sessions_yield`: native `streamTo` is ACP-only and ignored, `agent.wait` is terminal-only, yielded lifecycle metadata is not persisted as a visible paused state, and the TUI/Control UI paths have no yielded-child wait state.

## What This Changes

Request a parent-visible progress indicator and eventual native subagent stream forwarding while a main session yields waiting for spawned subagents.

## Best Possible Solution

Define a yielded-parent lifecycle state consumed by Control UI and TUI for a clear waiting indicator, while keeping full native child stream forwarding in the separate `streamTo` design with explicit redaction and verbosity rules.

## Reproduction Assessment

Yes, at source level. A native `sessions_spawn` plus `sessions_yield` path emits yielded metadata but current main does not persist/render a parent wait state, and native child progress is only observed through terminal `agent.wait` completion.

## Solution Assessment

Unclear until maintainers choose the lifecycle and streaming contract. The narrow maintainable first step is a yielded-parent wait indicator; full native stream forwarding should stay with the separate `streamTo` API and redaction-policy discussion.
