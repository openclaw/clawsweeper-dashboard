## Label Justifications

- `P1`: Long native subagent waits can make parent-session surfaces appear dead during an active delegated workflow.
- `impact:session-state`: The issue is about parent session state becoming dormant/ambiguous while child work is still active.
- `impact:message-loss`: Parent-attached surfaces receive no intermediate progress or streaming messages during the wait, even though child work continues.

## Summary

Keep open. Current main and latest release still only partially cover yielded-subagent visibility: native `streamTo` is ACP-only/ignored, yielded metadata is terminal lifecycle metadata rather than a parent-visible waiting contract, and the remaining work overlaps but is not fully owned by https://github.com/openclaw/openclaw/issues/47597 or https://github.com/openclaw/openclaw/issues/56856.

## What This Changes

Show parent-session progress or streaming in Control UI and TUI while a `sessions_yield` parent waits for native subagent completion.

## Best Possible Solution

Define a parent-visible yielded-wait/progress contract for native subagents, including redaction and delivery semantics, then update TUI and Control UI to render the waiting state and optional progress consistently.

## Reproduction Assessment

Yes, at source level. Current main ignores `streamTo` for native subagents, records yielded waits as terminal paused metadata, and has no TUI or Control UI branch that keeps a parent-visible waiting/progress state after the parent turn ends.

## Solution Assessment

No, extending `streamTo` or adding an unconditional heartbeat without a policy is not the safest standalone fix. The maintainable solution needs an explicit native progress contract covering visibility, redaction, truncation, verbosity, and delivery before UI work lands.
