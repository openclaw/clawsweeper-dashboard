## Label Justifications

- `P1`: A yielded parent session can appear frozen across primary user surfaces during active subagent work, breaking a real agent workflow until completion or manual investigation.
- `impact:session-state`: The report is about parent session state becoming dormant or indistinguishable from idle while child work remains active.
- `impact:message-loss`: Intermediate progress and waiting-state events are not surfaced to attached parent-session clients, so user-visible delivery is suppressed or delayed.

## Summary

Keep open. Current main still only partially covers yielded-subagent visibility: native `streamTo` is ACP-only/ignored, yielded metadata is emitted only as terminal lifecycle metadata, and TUI/Control UI still lack a shared parent-visible yielded waiting or native child-progress contract.

## What This Changes

The issue asks for attached parent-session surfaces to show a progress/waiting indicator, and ideally native child streaming, while `sessions_yield` waits for spawned subagents.

## Best Possible Solution

Add a structured parent-yield wait lifecycle/session state consumed by both TUI and Control UI, then handle native child stream forwarding as an explicit redacted, policy-backed extension.

## Reproduction Assessment

Yes, at source level. A native `sessions_spawn` plus `sessions_yield` path ignores native `streamTo`, exposes yielded/liveness data only at terminal lifecycle boundaries, and current TUI/Control UI contracts do not represent a yielded parent wait state.

## Solution Assessment

Unclear until maintainers choose the event contract. The narrow maintainable first step is a parent-visible yielded wait state; full native child streaming should remain a separate redaction- and verbosity-backed design, aligned with https://github.com/openclaw/openclaw/issues/47597.
