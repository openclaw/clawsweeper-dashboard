## Summary

Keep open. Current main still has the reported source-level gap: native subagent spawns ignore parent streaming, subagent completion is monitored through terminal `agent.wait` state, and Control UI/TUI clear the parent run to idle/final states rather than rendering a requester-scoped yielded-child wait. The overlapping lifecycle PR is still open and does not by itself implement the visible Control UI/TUI progress indicator.

## What This Changes

The issue asks for Control UI and TUI sessions to show progress or streamed child output while a parent session is yielded waiting on a native subagent.

## Best Possible Solution

Land or replace the yielded-session lifecycle contract from https://github.com/openclaw/openclaw/pull/75662 if maintainers accept it, then add a generic requester-scoped subagent-wait indicator in Control UI and TUI while keeping full native `streamTo` forwarding in https://github.com/openclaw/openclaw/issues/47597.

## Reproduction Assessment

Yes, at source level. A native subagent spawn followed by `sessions_yield` makes the parent run end/idle while child completion is monitored through `agent.wait`, and the current Control UI/TUI handlers have no requester-scoped child-wait state to render.

## Solution Assessment

Unclear until maintainers choose the lifecycle/UI contract. A minimum yielded-parent waiting indicator is the narrowest useful direction, while full native child streaming should stay with the separate `streamTo` API and redaction-policy discussion.
