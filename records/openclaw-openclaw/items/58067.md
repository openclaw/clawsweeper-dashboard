## Label Justifications

- `P1`: A yielded parent session can appear frozen across primary user surfaces during active subagent work, breaking a real agent workflow until completion or manual investigation.
- `impact:session-state`: The issue is about parent session state becoming dormant or indistinguishable from idle while child work is still active.
- `impact:message-loss`: Intermediate progress and completion-related events are not surfaced to attached parent-session clients, so user-visible delivery is suppressed or delayed.

## Summary

Keep open. Current main and the latest stable release still lack a parent-visible wait/progress path for native `sessions_yield` subagent waits: native `streamTo` is ACP-only, yielded lifecycle metadata is not persisted as a paused/waiting session state, and both TUI and Control UI still lack a yielded-parent wait contract.

## What This Changes

The issue asks for parent-session surfaces to show waiting/progress or native subagent streaming while `sessions_yield` waits on spawned subagents.

## Best Possible Solution

Define a yielded-parent lifecycle state that Control UI and TUI render as waiting, and keep full native child stream forwarding under the separate `streamTo` design with explicit redaction and verbosity policy.

## Reproduction Assessment

Yes, at source level. A native `sessions_spawn` plus `sessions_yield` path ignores native `streamTo`, waits through terminal `agent.wait`, and current session/UI contracts do not expose a yielded waiting state.

## Solution Assessment

Unclear until maintainers choose the event contract. The narrow maintainable first step is a parent-visible yielded wait state; full native child streaming should remain a separate policy-backed `streamTo` design.
