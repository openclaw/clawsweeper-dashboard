## Summary

Keep open. Current main still treats native subagent waits as a parent turn ending plus a background `agent.wait`, and neither the Control UI nor TUI has a requester-scoped waiting/progress state to show while the child run is active.

## What This Changes

The issue asks for parent-session visibility during `sessions_spawn` plus `sessions_yield`, at least a waiting/progress indicator and ideally native child streaming.

## Best Possible Solution

Add a generic requester-scoped native subagent waiting/progress signal that starts when the parent yields with active child work and clears on child completion, error, kill, pause, or parent resume, then render that state in Control UI and TUI while leaving full native `streamTo` design to the separate open feature request.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: spawn a native subagent, call `sessions_yield`, and observe the parent session after the parent final event clears the active run while child wait/lifecycle state is not relayed to that parent surface. I did not run a live Control UI/TUI session because this review is read-only.

## Solution Assessment

Yes for the minimum fix: a requester-scoped wait/progress lifecycle signal is the narrowest maintainable repair. Full native child output streaming is not the best first step here because it needs explicit visibility, redaction, and verbose-policy decisions tracked by the separate streaming issue.
