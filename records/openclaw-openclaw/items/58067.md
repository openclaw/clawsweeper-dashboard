## Label Justifications

- `P1`: A yielded parent session can appear frozen across primary user surfaces during active subagent work, breaking a real agent workflow until completion or manual investigation.
- `impact:session-state`: The issue is about parent session state becoming dormant or indistinguishable from idle while child work remains active.
- `impact:message-loss`: Intermediate progress and waiting-state events are not surfaced to attached parent-session clients, so user-visible delivery is suppressed or delayed.

## Summary

Keep open. Current main still treats native subagent `streamTo` as ACP-only and lacks a parent-visible yielded waiting state for TUI/Control UI, while related open items cover only slices of the problem.

## What This Changes

The issue asks for parent-session surfaces to show progress or stream subagent output while `sessions_yield` waits for a native subagent.

## Best Possible Solution

Define a yielded-parent lifecycle state that Control UI and TUI render as waiting, and keep full native child stream forwarding behind a separate policy-backed `streamTo` design.

## Reproduction Assessment

Yes, at source level. A native `sessions_spawn` plus `sessions_yield` path ignores native `streamTo`, waits through terminal `agent.wait`, and current session/UI contracts do not expose a yielded waiting state.

## Solution Assessment

Unclear until maintainers choose the event contract. The narrow maintainable first step is a parent-visible yielded wait state; full native child streaming should remain a separate redaction- and verbosity-backed `streamTo` design.
