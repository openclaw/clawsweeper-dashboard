## Summary

Keep open. Current main still has no requester-scoped waiting/progress signal for a parent session that yields while native subagent work continues: native `streamTo` remains ignored, subagent completion monitoring waits on `agent.wait`, and both Control UI/TUI clear the active parent run on terminal events instead of showing a child-wait state. A related lifecycle PR is open, but it is unmerged and does not by itself add the visible Control UI/TUI progress indicator.

## What This Changes

The issue asks for Control UI and TUI to show progress, and ideally child streaming, while `sessions_yield` waits for native subagent completion.

## Best Possible Solution

Land or replace the yielded-session lifecycle contract from https://github.com/openclaw/openclaw/pull/75662 if maintainers accept that shape, then add a generic requester-scoped subagent-wait indicator in Control UI and TUI while leaving full native `streamTo` behavior to https://github.com/openclaw/openclaw/issues/47597.

## Reproduction Assessment

Yes at source level. A native subagent spawn followed by `sessions_yield` makes the parent run end/idle while child completion is monitored through `agent.wait`, and current Control UI/TUI handlers have no requester-scoped child-wait state to render.

## Solution Assessment

Yes for the minimum fix: carry a generic yielded-parent waiting state to the attached surfaces and clear it on child completion, error, kill, pause, or parent resume. Full native child streaming is not the best first fix here because it belongs to the separate native `streamTo` API/policy discussion.
