## Summary

Keep open. Current main still has the reported source-level gap: native subagent runs do not forward progress to the parent session, yielded parent runs collapse to terminal/idle state for the visible surfaces, and the related lifecycle and streaming work remains open.

## What This Changes

Requests a visible parent-session wait/progress state, and optionally native subagent stream forwarding, while `sessions_yield` waits for a spawned child run.

## Best Possible Solution

Define the yielded-parent lifecycle contract, land or replace the paused-state work in https://github.com/openclaw/openclaw/pull/75662, then add a requester-scoped wait indicator in Control UI and TUI while keeping full native stream forwarding in https://github.com/openclaw/openclaw/issues/47597.

## Reproduction Assessment

Yes, at source level. A native `sessions_spawn` followed by `sessions_yield` ends the parent-visible run while child completion is monitored through terminal `agent.wait`, and the current Control UI/TUI code has no requester-scoped yielded-child wait state to render.

## Solution Assessment

Unclear until maintainers choose the lifecycle and streaming contract. A yielded-parent waiting indicator is the narrowest useful fix, while full native subagent streaming should stay with the separate `streamTo` API and redaction-policy discussion.
