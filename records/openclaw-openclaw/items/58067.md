## Label Justifications

- `P1`: A yielded parent session can appear frozen across primary user surfaces during active subagent work, breaking a real multi-agent workflow until completion or manual investigation.
- `impact:session-state`: The report is about parent session state becoming dormant or indistinguishable from idle while child work remains active.
- `impact:message-loss`: Intermediate progress and yielded-wait events are not surfaced to attached parent-session clients, so user-visible delivery is suppressed or delayed.

## Summary

Keep open. Current main and v2026.5.22 still only partially cover yielded-subagent visibility: native `streamTo` remains ACP-only/ignored, yielded metadata is terminal-only, and neither TUI nor Control UI has a shared parent-visible yielded-wait state; the related open items cover slices but do not supersede this whole issue.

## What This Changes

Show a progress, waiting, or streaming indication in Control UI and TUI while a parent session is yielded waiting for subagent work.

## Best Possible Solution

Define one shared yielded-parent wait contract that TUI and Control UI can render, then separately add native child-progress forwarding only with explicit redaction, verbosity, and delivery semantics.

## Reproduction Assessment

Yes, at source level. Native `sessions_spawn` plus `sessions_yield` still drops native `streamTo`, only carries yielded data on terminal lifecycle snapshots, and current TUI/Control UI contracts do not represent a yielded parent wait state.

## Solution Assessment

No single narrow UI spinner or raw native stream relay is enough. The maintainable fix needs a shared yielded-wait state first, with native streaming treated as a security- and product-reviewed extension.
