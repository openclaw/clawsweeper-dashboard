## Summary

Keep open: current main still lacks a persistent `agents.defaults` usage-footer default, and live GitHub state shows open implementation candidates already targeting the same feature.

## What This Changes

The issue requests a persistent config default for the per-response `/usage` footer so new sessions can start with `off`, `tokens`, or `full` usage display.

## Best Possible Solution

Land or replace one accepted typed `agents.defaults` usage-footer contract with schema/types/help/docs, runtime inheritance, explicit-off and clear-to-inherit semantics, and focused regression coverage.

## Reproduction Assessment

Yes for the capability gap, though this is a feature request rather than a broken existing contract. Source inspection on current main shows no `agents.defaults` usage-footer key, and reply, slash-command, gateway, and docs paths still treat `responseUsage` as session-local.

## Solution Assessment

Yes for the direction, but the final key name and override semantics need maintainer choice. The maintainable path is to review and land or replace the open implementation PRs rather than close this issue early or start a duplicate repair lane.
