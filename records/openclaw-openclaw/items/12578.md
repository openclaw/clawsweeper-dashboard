## What This Changes

Requests a configurable inherited default for per-session response token-usage footers.

## Best Possible Solution

Add one canonical `agents.defaults` response-usage mode beneath explicit session overrides, with distinct explicit-off and clear-to-inherit semantics and focused config, command, gateway, reply, documentation, fresh-session, and upgrade coverage.

Yes. Current main and v2026.6.5 resolve, mutate, and patch `responseUsage` only on session entries, while the agent-default schema exposes no inherited usage key.

## Solution Assessment

The inherited-default direction is appropriate, but the broad linked PR is not currently the best landing shape; maintainers should choose one canonical key and preserve explicit-off versus clear-to-inherit semantics in a focused implementation.
