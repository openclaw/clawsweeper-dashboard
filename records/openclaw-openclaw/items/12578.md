## Summary

Keep open: current main and the latest release still treat usage footer mode as session-local state, while the requested inherited `agents.defaults` usage default remains unimplemented and the linked implementation PRs are still open.

## What This Changes

Add a persistent config default for `/usage` response footers so new sessions can start with `off`, `tokens`, or `full` without a per-session command.

## Best Possible Solution

Land, narrow, or replace one accepted implementation that adds a validated `agents.defaults` usage-footer default, preserves session overrides including explicit `off`, updates schema/help/docs, and adds focused regression coverage.

## Reproduction Assessment

Yes for the capability gap: current-main source inspection shows no inherited usage-footer key, and runtime/docs still treat `responseUsage` as session-local. This is a feature/config request, not a broken existing contract.

## Solution Assessment

Unclear for a specific implementation. The direction belongs in core, but maintainers still need to choose the stable public key and explicit-off versus clear-to-inherit semantics before this should close.
