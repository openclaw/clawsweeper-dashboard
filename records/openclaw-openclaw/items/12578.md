## Summary

Keep open: current main still lacks a persistent `agents.defaults` usage-footer default, and there are open implementation PRs with closing/related syntax for this feature.

## What This Changes

The issue requests a persistent config default for `/usage off|tokens|full` so new sessions can show token or full usage footers without manually running `/usage` each time.

## Best Possible Solution

Land or replace one accepted typed `agents.defaults` usage-footer contract with schema/types/help/docs, runtime inheritance, explicit-off versus clear-to-inherit semantics, and focused regression coverage.

## Reproduction Assessment

Yes for the capability gap: source inspection on current main shows no inherited usage-footer key, and the reply, slash-command, gateway, tests, and docs paths still treat `responseUsage` as session-local. This is a feature request rather than a broken existing contract.

## Solution Assessment

Yes for the direction, but not yet for a specific implementation. The maintainable path is to choose one public config contract and review, land, or replace the open implementation PRs rather than close this issue early.
