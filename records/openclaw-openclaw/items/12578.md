## Summary

Keep open: current main still lacks a persistent `agents.defaults` usage-footer default, and an open closing PR is already an implementation candidate rather than a reason to close the issue before merge.

## What This Changes

The request is to add an `agents.defaults` usage-footer default so new sessions can start with `off`, `tokens`, or `full` without running `/usage` each time.

## Best Possible Solution

Land one accepted typed `agents.defaults` usage-footer contract with schema, types, generated help, docs, and regression coverage for inherited default, explicit off, clear-to-inherit, and per-session overrides.

## Reproduction Assessment

Yes for the capability gap by source inspection: current main has no `agents.defaults` usage-footer key, and reply/command/gateway paths still rely on session-local `responseUsage`. No live run is needed to see that the requested config surface is absent.

## Solution Assessment

Yes, a typed inherited `agents.defaults` key is the narrow maintainable direction, but the exact public key and override semantics remain a maintainer decision. The existing open PRs should be reviewed or replaced with a smaller accepted implementation before closing this issue.
