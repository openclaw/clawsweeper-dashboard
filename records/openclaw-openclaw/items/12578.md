## Summary

Keep open: current `main` and latest `v2026.6.6` still only resolve the `/usage` footer from per-session `responseUsage`, while the narrow implementation at https://github.com/openclaw/openclaw/pull/89762 and broader implementation at https://github.com/openclaw/openclaw/pull/69618 are still open and unmerged.

## What This Changes

Add a persistent config default for the per-response `/usage` footer so new sessions can start with `off`, `tokens`, or `full` without running `/usage` each time.

## Best Possible Solution

Keep this issue open as the canonical default-tracking issue, then review and land https://github.com/openclaw/openclaw/pull/89762 or an accepted replacement with one stable config key, explicit-off semantics, schema/help/docs, and focused regression tests.

## Reproduction Assessment

Yes, at source level for the missing capability: current schema/types lack a response-usage default and the reply path resolves only session-local `responseUsage`. This is a feature request rather than a broken existing contract, so no failing live repro is required to keep it open.

## Solution Assessment

Unclear until maintainers choose the public config owner. A small, focused default implementation with explicit-off semantics is the best shape; the broader footer PR may be useful but is not necessary to satisfy this exact issue.
