## Summary

Keep open: current main still lacks a persistent `agents.defaults` usage-footer default, and open implementation PRs already reference the same remaining feature work.

## What This Changes

Add a persistent agent-defaults usage-footer setting so new sessions can default to `off`, `tokens`, or `full` without rerunning `/usage`.

## Best Possible Solution

Land or replace one accepted typed `agents.defaults` usage-footer contract with schema/types/help/docs, runtime inheritance, explicit-off and clear-to-inherit semantics, and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection is enough for the capability gap: current main has no agent-defaults usage-footer key, and the reply, command, gateway, and docs paths still treat `responseUsage` as session-local.

## Solution Assessment

Yes for the direction, but the final key name and override semantics need maintainer choice. The maintainable path is to review and land or replace the open implementation PRs rather than close this issue early.
