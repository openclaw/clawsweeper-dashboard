## Summary

Keep open: current main and the latest shipped release still only support session-local `responseUsage`, while the requested inherited `agents.defaults` usage-footer default remains unmerged and is covered by open implementation PRs.

## What This Changes

The issue requests a persistent config default for `/usage off|tokens|full` so new sessions can inherit per-response usage footer behavior.

## Best Possible Solution

Land, narrow, or replace one accepted implementation that adds a validated `agents.defaults` usage-footer default, preserves session overrides including explicit `off` and clear-to-inherit behavior, updates schema/help/docs, and adds focused regression coverage.

## Reproduction Assessment

Yes for the capability gap: current-main source inspection shows no inherited usage-footer key, and runtime/docs still treat `responseUsage` as session-local. This is a feature/config request, not a broken existing contract.

## Solution Assessment

Unclear for a specific implementation. The direction belongs in core config, but maintainers still need to choose one stable public key and define explicit-off versus clear-to-inherit semantics before this should close.
