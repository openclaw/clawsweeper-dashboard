## Summary

Keep open: current main still has only per-session `responseUsage`, and an open closing PR already targets the inherited default semantics.

## What This Changes

Request to add a persistent config default for `/usage off|tokens|full` so new sessions can inherit the response usage footer mode.

## Best Possible Solution

Land one accepted typed `agents.defaults` usage-footer contract with schema, generated help, docs, and regression coverage for inheritance, explicit `off`, clear-to-inherit, and per-session overrides.

## Reproduction Assessment

Yes for the reported capability gap. Source inspection on current main shows no inherited usage-footer default in `agents.defaults`, while reply formatting and command/gateway mutation still read and write only session-local `responseUsage`.

## Solution Assessment

Yes, an inherited typed `agents.defaults` key is the narrow maintainable direction. The remaining product decision is the public key shape and override semantics, so maintainers should review or close the existing implementation PRs first.
