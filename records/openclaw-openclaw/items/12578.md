## Summary

Keep open: current main still only supports session-local `responseUsage`, while the requested persistent defaults key remains absent and open implementation candidates already cover the requested behavior.

## What This Changes

Request adding a persistent `agents.defaults` usage-footer config option so new sessions default to `off`, `tokens`, or `full` without running `/usage` each time.

## Best Possible Solution

Land one stable typed core config contract, update schema/types/generated help/docs, and add regression coverage for inheritance, explicit `off`, clearing, and per-session overrides.

## Reproduction Assessment

Not applicable as a feature request, but source inspection on current main confirms the requested inherited config default is absent. The observable gap is that the reply path and `/usage` command still read/write only session-local `responseUsage`.

## Solution Assessment

Yes. A typed `agents.defaults` usage mode is the narrow maintainable direction, but maintainers should resolve the existing open implementation candidates before starting parallel work.
