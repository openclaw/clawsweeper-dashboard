## Summary

Keep open: current main still lacks a persistent usage-footer default, and open implementation PRs already cover the requested behavior.

## What This Changes

Add a persistent `agents.defaults` usage-footer default so new sessions can start with `/usage` display set to `off`, `tokens`, or `full`.

## Best Possible Solution

Land one accepted typed core config contract for inherited usage footers, with schema/types/generated help/docs and regression coverage for inheritance, explicit off, clearing, and per-session overrides.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection does verify the requested default key is absent and usage footer state is still read from session-local `responseUsage`.

## Solution Assessment

Yes in principle: an inherited typed `agents.defaults` key is the narrow maintainable direction. Maintainers should resolve the existing implementation PRs first and choose one stable public key while preserving explicit `off` versus clear-to-inherit behavior.
