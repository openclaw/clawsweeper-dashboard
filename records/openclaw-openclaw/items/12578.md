## Summary

Keep open: current main still only supports session-local `responseUsage`, and the requested persistent `agents.defaults` usage-footer default remains unmerged in open implementation PRs.

## What This Changes

Request adds an `agents.defaults` usage footer default so new sessions can start with `off`, `tokens`, or `full` without running `/usage` manually.

## Best Possible Solution

Land one accepted typed `agents.defaults` usage-footer contract with schema/types/generated help/docs and regression coverage for inherited default, explicit `off`, clear-to-inherit, and per-session override behavior.

## Reproduction Assessment

Yes for the reported capability gap. Source inspection on current main shows no inherited usage-footer default in `agents.defaults`, while reply formatting and command/gateway mutation still read and write only session-local `responseUsage`.

## Solution Assessment

Yes, a typed inherited `agents.defaults` key is the narrow maintainable direction. The remaining decision is the public key shape and override semantics, so maintainers should review or close the existing implementation PRs first.
