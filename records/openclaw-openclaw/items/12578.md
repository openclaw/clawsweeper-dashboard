## Summary

Keep open: current main still supports only session-local `responseUsage`, while open PRs target the inherited config default and maintainers still need to settle the public key and explicit-off inheritance semantics.

## What This Changes

Request adding a persistent config default under `agents.defaults` so new sessions can inherit the `/usage` response-usage footer mode instead of requiring `/usage tokens|full` per session.

## Best Possible Solution

Land, narrow, or replace one accepted implementation that adds a single validated inherited `agents.defaults` usage-footer default, preserves per-session overrides including explicit off, updates schema/help/docs, and adds focused regression coverage.

## Reproduction Assessment

Yes for the source-level capability gap: current main has no inherited usage-footer key under `agents.defaults`, and runtime/docs still treat `responseUsage` as session-local. This is a feature/config request rather than a broken existing contract.

## Solution Assessment

Unclear which exact public key is best until maintainers choose the config contract and explicit-off semantics. Keeping this issue open while reviewing or replacing the linked implementation PRs is the right current path.
