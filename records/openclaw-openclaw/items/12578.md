## Summary

Keep open: current main still lacks the requested inherited usage-footer default, and the issue is linked by closing syntax to open implementation work that should land, be narrowed, or be closed before this issue closes.

## What This Changes

Add a config default for new sessions so the per-response `/usage` footer can start as `off`, `tokens`, or `full` without manually running `/usage` each session.

## Best Possible Solution

Land, narrow, or replace one accepted core config implementation that adds a validated inherited usage-footer default, preserves session override semantics, updates schema/help/docs, and adds focused regression coverage.

## Reproduction Assessment

Yes for the source-level capability gap: current main has no inherited usage-footer key in agent defaults, and runtime/docs still treat `responseUsage` as session-local. This is a feature/config request rather than a broken existing contract.

## Solution Assessment

Unclear for the exact implementation path. A core config default is the right surface, but maintainers still need to choose the public key and define explicit-off versus clear-to-inherit semantics before this should close.
