## Summary

Keep open: current main still lacks a persistent `agents.defaults` usage-footer default, and live GitHub state still has open implementation candidates targeting the same feature.

## What This Changes

Request adds a config default so new sessions can inherit `/usage off|tokens|full` instead of requiring a per-session command.

## Best Possible Solution

Land or replace one accepted typed `agents.defaults` usage-footer contract with schema/types/help/docs, runtime inheritance, explicit-off and clear-to-inherit semantics, and focused regression coverage.

## Reproduction Assessment

Yes for the capability gap: source inspection on current main shows no inherited usage-footer key, and the reply, slash-command, gateway, tests, and docs paths still treat `responseUsage` as session-local. This is a feature request rather than a broken existing contract.

## Solution Assessment

Yes for the direction, but not yet for a specific implementation. The maintainable path is to choose one stable config contract and land or replace the open implementation PRs rather than close this issue early.
