## What This Changes

The issue requests a new persistent config default for `/usage` footer display across new sessions.

## Best Possible Solution

Land or replace one accepted typed `agents.defaults` usage-footer contract with schema/types/help/docs, runtime inheritance, explicit-off and clear-to-inherit semantics, and focused regression coverage.

Yes for the capability gap: source inspection on current main shows no inherited usage-footer key, and runtime/docs still treat `responseUsage` as session-local. This is a feature request rather than a broken existing contract.

## Solution Assessment

Unclear for a specific implementation. The direction is maintainable, but maintainers still need to choose one stable public config key and preserve explicit off versus inherit/clear semantics before this should close.
