## Summary

Keep open: current main still only supports session-local `responseUsage`, while the requested inherited `agents.defaults` usage-footer default remains unmerged and is already covered by open implementation PRs.

## What This Changes

The issue requests a persistent config default for per-response usage footer mode so new sessions can start with `/usage`-style token or full usage display enabled.

## Best Possible Solution

Land, narrow, or replace one accepted core config implementation that adds a validated inherited usage-footer default, preserves per-session override semantics, updates docs/schema/help, and adds focused regression coverage.

## Reproduction Assessment

Yes for the source-level capability gap: current main has no inherited usage-footer key in agent defaults, and runtime/docs still treat `responseUsage` as session-local. This is a feature/config request rather than a broken existing contract.

## Solution Assessment

Unclear for a specific implementation. The direction belongs in core config, but maintainers still need to choose one public key and define explicit-off versus clear-to-inherit semantics before this should close.
