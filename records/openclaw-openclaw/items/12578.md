## Summary

Keep open: current main still lacks an inherited usage-footer default, the runtime and docs keep `responseUsage` session-local, and the linked implementation PR at https://github.com/openclaw/openclaw/pull/69618 is open, unmerged, and conflicting.

## What This Changes

The issue requests a persistent config default for the `/usage` response footer so new sessions can inherit `off`, `tokens`, or `full` instead of requiring `/usage` each session.

## Best Possible Solution

Land, narrow, or replace one accepted implementation that adds a validated inherited `agents.defaults` usage-footer default, preserves session overrides including explicit `off`, updates schema/help/docs, and adds focused regression coverage.

## Reproduction Assessment

Yes for the source-level capability gap: current main has no inherited usage-footer key in agent defaults, and runtime/docs still treat `responseUsage` as session-local. This is a feature/config request rather than a broken existing contract.

## Solution Assessment

Unclear which exact implementation is best until maintainers choose the public key and explicit-off semantics. Keeping this issue open is the right current outcome because an open PR still references it with closing syntax and the behavior is not implemented on main.
