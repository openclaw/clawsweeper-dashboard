Keep open. Current main still lacks an inherited `agents.defaults` usage-footer default, and the open closing PR is an implementation candidate rather than a reason to close the issue before merge.

## What This Changes

The issue asks for a persistent `agents.defaults` usage-display default so new sessions can inherit `/usage off|tokens|full` behavior without a per-session command.

## Best Possible Solution

Review and either land a maintainer-approved implementation such as https://github.com/openclaw/openclaw/pull/69618 or replace it with a narrower core config change that adds one canonical inherited usage-footer default with explicit override semantics.

## Reproduction Assessment

Not applicable as a bug reproduction, but yes as source evidence: current `main` has session-local `responseUsage` paths and no accepted `agents.defaults` usage key.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable shape is one canonical `agents.defaults` key with schema/types/help/docs and explicit session override behavior, not another ad hoc session fallback.
