## Summary

Keep open: current main and v2026.5.19 still treat `responseUsage` as session-local state, no inherited `agents.defaults` usage-footer default exists, and open implementation PRs already target the remaining behavior.

## What This Changes

Add a persistent config default for `/usage` response-footer mode so new sessions can inherit `off`, `tokens`, or `full` without per-session slash commands.

## Best Possible Solution

Land, narrow, or replace one accepted implementation that adds a single validated inherited `agents.defaults` usage-footer default, preserves per-session overrides including explicit off, updates schema/help/docs, and adds focused regression coverage.

## Reproduction Assessment

Yes for the source-level capability gap: current main has no inherited usage-footer key under `agents.defaults`, and runtime/docs still treat `responseUsage` as session-local. This is a feature/config request rather than a broken existing contract.

## Solution Assessment

Unclear which exact public key is best until maintainers choose the config contract and explicit-off semantics. Keeping this issue open while reviewing the linked implementation PRs is the right current path.
