## Summary

Current main and v2026.6.5 still support only session-local usage-footer preferences, so the requested persistent default remains necessary. Keep the issue open while maintainers choose between the focused implementation at https://github.com/openclaw/openclaw/pull/89762 and the broader, currently dirty implementation at https://github.com/openclaw/openclaw/pull/69618; the focused PR appears closer to the issue’s scope, but its public config location and override semantics require an explicit product decision.

## What This Changes

Add a configurable default that enables `off`, `tokens`, or `full` response-usage footers for sessions without an explicit per-session preference.

## Best Possible Solution

Adopt one focused canonical implementation—preferably by narrowing and reviewing https://github.com/openclaw/openclaw/pull/89762—after maintainers choose the stable config owner, define explicit `off` versus clear-to-inherit behavior, align all mode read sites, document precedence, and add focused fresh-session and override coverage.

## Reproduction Assessment

Yes at the source level. Current main has no matching config field, reply rendering reads only session `responseUsage`, and `/usage` only mutates that session entry; this read-only review did not execute a live chat reproduction.

## Solution Assessment

The requested persistent default is the right product direction, but the issue’s suggested field is not yet proven to be the best permanent API. The best solution is the focused implementation after maintainers settle config ownership and explicit-off/clear-to-inherit semantics, rather than merging the much broader footer bundle unchanged.
