## Summary

Keep open: current main and v2026.6.5 still provide only session-local response-usage footer state, while the open closing PR remains the active implementation candidate and the permanent config/inheritance contract needs maintainer review.

## What This Changes

Add a configurable inherited default for the per-session response usage footer mode.

## Best Possible Solution

Adopt one canonical `agents.defaults` response-usage key, resolve effective mode as explicit session override then inherited default, retain a distinct clear-to-inherit operation, and cover fresh sessions, explicit off, clearing, and per-session overrides in focused tests and docs.

## Reproduction Assessment

Yes at source level: current main and v2026.6.5 lack the requested agent-default field, and the reply path demonstrably resolves usage mode only from session entries.

## Solution Assessment

The focused inherited-default design is the best direction, but the broad linked PR is not automatically the best landing shape; maintainers should accept or extract the narrow config/session contract with explicit override semantics.
