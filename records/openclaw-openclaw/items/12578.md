## Summary

Keep open: current main still lacks a persistent `agents.defaults` usage-footer default, and the open implementation PRs that cover this behavior have not landed.

## What This Changes

The issue requests a config default for `/usage off|tokens|full` so new sessions can inherit a per-response usage footer mode.

## Best Possible Solution

Land one accepted typed `agents.defaults` usage-footer contract with schema/types/generated help/docs and regression coverage for inherited default, explicit off, clear-to-inherit, and per-session overrides.

## Reproduction Assessment

Yes for the reported capability gap. Source inspection on current main shows no inherited usage-footer default in `agents.defaults`, while reply formatting and command/gateway mutation still read and write only session-local `responseUsage`.

## Solution Assessment

Yes, a typed inherited `agents.defaults` key is the narrow maintainable direction. The remaining decision is the public key shape and override semantics, so maintainers should review the existing implementation PRs before closing this issue.
