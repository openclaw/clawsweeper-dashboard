## Summary

Keep open: current main still does not expose a persistent `agents.defaults` usage-footer default, and an open closing PR is already the implementation candidate rather than a reason to close the issue early.

## What This Changes

Request a config default such as `agents.defaults.usageDefault` or `agents.defaults.responseUsage` so new sessions can inherit `/usage off|tokens|full` display behavior.

## Best Possible Solution

Land one accepted typed `agents.defaults` usage-footer contract with schema, types, generated help, docs, and regression coverage for inherited default, explicit off, clear-to-inherit, and per-session overrides.

## Reproduction Assessment

Yes for the capability gap by source inspection: current main has no `agents.defaults` usage-footer key, and reply/command/gateway paths still rely on session-local `responseUsage`. No live failure run is needed to see that the requested config surface is absent.

## Solution Assessment

Yes, a typed inherited `agents.defaults` key is the narrow maintainable direction, but the exact public key and override semantics remain a maintainer decision. The existing open PRs should be reviewed or replaced with a smaller accepted implementation before closing this issue.
