## Summary

Keep open: current main still lacks an inherited usage-footer default, docs still describe `/usage` as session-local, and an open closing PR already targets this behavior but has not landed.

## What This Changes

The issue requests a new config default so new sessions can start with `/usage` footers set to `off`, `tokens`, or `full` without running `/usage` manually each time.

## Best Possible Solution

Land, narrow, or replace one accepted implementation that adds a validated inherited `agents.defaults` usage-footer default, preserves per-session overrides including explicit `off`, updates schema/help/docs, and adds focused regression coverage.

## Reproduction Assessment

Yes for the source-level capability gap: current main has no inherited usage-footer key under `agents.defaults`, and runtime/docs still treat `responseUsage` as session-local. This is a feature/config request rather than a broken existing contract.

## Solution Assessment

Unclear which exact implementation is best until maintainers choose the public key and explicit-off semantics. Keeping this issue open is the right current outcome because an open PR references it with closing syntax and the behavior is not implemented on main.
