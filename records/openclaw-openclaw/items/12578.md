## Summary

Keep open: current main and v2026.5.22 still only support session-local `responseUsage`, and the open closing PR already owns a possible implementation path that maintainers need to review.

## What This Changes

The issue requests a persistent config default for `/usage` response footers so new sessions can inherit `off`, `tokens`, or `full` without running the slash command each time.

## Best Possible Solution

Review, narrow, land, or replace the linked implementation with one accepted config contract that adds a validated inherited `agents.defaults` usage-footer default, preserves session overrides, updates schema/help/docs, and adds focused regression coverage.

## Reproduction Assessment

Yes for the source-level capability gap: current main has no inherited usage-footer key under `agents.defaults`, and runtime/docs still treat `responseUsage` as session-local. This is a feature/config request rather than a broken existing contract.

## Solution Assessment

Unclear which exact public key is best until maintainers choose the config contract and explicit-off semantics. Keeping this issue open while reviewing or replacing the linked implementation PR is the right current path.
