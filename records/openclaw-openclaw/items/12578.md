## Summary

Keep open: current main still lacks a persistent usage-footer default, and open unmerged PRs already cover the requested behavior.

## What This Changes

The issue requests a config default for `/usage off|tokens|full` so new sessions can inherit token or full usage-footers without rerunning the slash command.

## Best Possible Solution

Land one accepted typed core config contract for inherited usage footers, with schema/types/generated help/docs and regression coverage for inheritance, explicit off, clearing, and per-session overrides.

## Reproduction Assessment

Yes for the reported gap. Source inspection on current main shows the requested config key is absent and the reply path still reads only session-local `responseUsage`.

## Solution Assessment

Yes, an inherited typed `agents.defaults` key is the narrow maintainable direction. Maintainers should resolve the existing implementation PRs first and choose one stable public key while preserving explicit `off` versus clear-to-inherit semantics.
