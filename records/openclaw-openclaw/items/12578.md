## Summary

Keep open: current main still has no persistent usage-footer default, and the remaining work is actively covered by open implementation candidates, especially https://github.com/openclaw/openclaw/pull/89762 and the broader closing PR https://github.com/openclaw/openclaw/pull/69618.

## What This Changes

Add a persistent config default for `/usage off|tokens|full` so new sessions can inherit a usage footer mode instead of requiring per-session `/usage` commands.

## Best Possible Solution

Keep this issue open until maintainers land one canonical implementation that adds a single documented usage-footer default, preserves explicit session overrides, updates schema/help/docs, and covers reply, command, gateway, and TUI behavior.

## Reproduction Assessment

Yes, at source level: current main has no usageDefault/responseUsageDefault key, the strict agent defaults schema lacks responseUsage, and reply rendering reads only session responseUsage state. I did not run a live chat scenario because the missing config path is clear from source and docs.

## Solution Assessment

No, closing or queueing a separate fix is not the best path. The maintainable path is to review and land a canonical existing implementation candidate, or choose a narrower replacement only if maintainers reject the current config owner or override semantics.
