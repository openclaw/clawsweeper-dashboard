## Summary

Keep this issue open: current main and v2026.6.6 still only support session-local `responseUsage`, while open implementation candidates such as https://github.com/openclaw/openclaw/pull/89762 and https://github.com/openclaw/openclaw/pull/69618 have not merged.

## What This Changes

The issue requests a persistent default for the `/usage` response footer so new sessions can inherit `off`, `tokens`, or `full` without running `/usage` manually each time.

## Best Possible Solution

Keep this issue open until one focused implementation lands or is rejected, with one stable public config key, explicit-off versus inherit semantics, schema/help/docs coverage, and regression tests across chat, gateway, TUI, and reply rendering.

## Reproduction Assessment

Yes. Source inspection shows the current schema/types do not expose an inherited usage-footer default, and the reply, slash-command, and gateway paths still read or mutate only session-local `responseUsage`.

## Solution Assessment

Yes, keeping it open is the right issue-triage outcome because current main and the latest release do not implement the requested default and the related PRs are still open/unmerged. The best implementation path is narrower than the broad footer PR unless maintainers explicitly choose that broader API surface.
