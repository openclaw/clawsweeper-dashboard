## Summary

Keep open: current main still lacks a persistent `agents.defaults` usage-footer default, and an open unmerged closing PR already tracks the requested behavior, so this should stay open until maintainers resolve the implementation candidates.

## What This Changes

Add a configuration default under `agents.defaults` so new sessions can inherit the `/usage off|tokens|full` response-usage footer mode.

## Best Possible Solution

Add one stable typed `agents.defaults` usage-footer key, update schema/types/generated help/docs, resolve inherited defaults across reply/TUI/gateway/session paths, and cover inherited default, explicit off, clear-to-inherit, and per-session override behavior with focused tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the schema/types do not accept a usage-footer default and reply formatting reads only session-local `responseUsage`.

## Solution Assessment

Yes. A typed inherited `agents.defaults` key with explicit override semantics is the narrowest maintainable core direction, but maintainers should review or close the existing implementation PRs before a separate repair lane starts.
