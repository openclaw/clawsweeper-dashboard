## Summary

Keep open: current main still has only session-local `responseUsage`, the shipped token-use docs still describe `/usage` as per-session, and the linked implementation candidates are still open rather than merged.

## What This Changes

Add a persistent `agents.defaults` usage-footer option so new sessions can default to `off`, `tokens`, or `full` without running `/usage` in every session.

## Best Possible Solution

Land, narrow, or replace one accepted implementation that adds a single validated inherited `agents.defaults` usage-footer default, preserves explicit session overrides, updates schema/help/docs, and adds focused regression coverage.

## Reproduction Assessment

Yes for the source-level capability gap: current main has no inherited usage-footer key under `agents.defaults`, and runtime/docs still treat `responseUsage` as session-local. This is a feature/config request rather than a broken existing contract.

## Solution Assessment

Unclear which exact public key is best until maintainers choose the config contract and explicit-off semantics. Keeping this issue open while reviewing or replacing the linked implementation PRs is the right current path.
