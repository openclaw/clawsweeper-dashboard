## Summary

## What This Changes

The issue asks OpenClaw to stop jiti-loaded plugin entries from re-evaluating OpenClaw's own built dist chunks by adding an OpenClaw package native boundary to the plugin loader jiti options.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/88384, then close this issue once the native OpenClaw package boundary is on main.

## Reproduction Assessment

Yes, at source level. Current main and the latest release still lack `nativeModules`, the plugin module loader can force jiti transform mode, and jiti v2.7.0 source confirms `nativeModules` is the force-native boundary.

## Solution Assessment

Yes. The linked narrow PR changes the shared plugin-loader jiti option helper and adds focused regression coverage instead of adding a parallel loader path.
