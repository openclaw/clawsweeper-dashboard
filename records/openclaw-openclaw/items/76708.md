## Summary

Keep open. Current main still routes `session_status({sessionKey:"current"})` through the requester/sandbox session key, and the linked draft PR #76995 is the active implementation candidate that should be reviewed or closed before resolving this issue.

## What This Changes

The issue reports that `session_status({sessionKey:"current"})` in Telegram direct chats can resolve to a stale channel policy session instead of the live main run.

## Best Possible Solution

Review and land or close #76995; the end state should pass the live run session identity to `session_status current` without weakening sandbox policy identity.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: build tools with a Telegram default direct `sandboxSessionKey` while the live run key is `agent:main:main`, then call `session_status` with `sessionKey:"current"` against a store containing both rows.

## Solution Assessment

Yes. The narrow maintainable fix is to thread live run session identity separately from the sandbox/policy key and add regression coverage for a stale Telegram direct row plus a live main row.
