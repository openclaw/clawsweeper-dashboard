## Summary

Keep open. Current main still has the reported source-level path, and an open draft PR, #76995, uses closing syntax for this issue, so the issue should remain open until that PR is reviewed and merged or replaced.

## What This Changes

The issue asks for `session_status({sessionKey:"current"})` in Telegram direct chats to report the live `agent:main:main` run/model instead of a stale Telegram default direct session row.

## Best Possible Solution

Review and land #76995 or an equivalent narrow fix that threads the live run session identity separately from the sandbox/policy key for status reporting.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to build tools with a Telegram default direct `sandboxSessionKey` while the live run is `agent:main:main`, then call `session_status` with `sessionKey: "current"` against a store containing both rows.

## Solution Assessment

Yes. The narrow maintainable solution is to separate live current-session identity from sandbox/tool-policy identity; the linked draft PR follows that direction but still needs normal review before this issue closes.
