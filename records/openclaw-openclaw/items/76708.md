## Summary

Keep open. Current main still has a source-level path where `session_status({sessionKey:"current"})` resolves through the sandbox/requester session key, and #76995 is an open draft PR with closing syntax that should be reviewed or closed before this issue is resolved.

## What This Changes

The issue asks for `session_status({sessionKey:"current"})` in Telegram direct runs to report the live `agent:main:main` session/model instead of a stale Telegram direct session.

## Best Possible Solution

Review and land or close #76995; the end state should thread live run session identity to `session_status current` without weakening sandbox policy identity.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: build tools with a Telegram default direct `sandboxSessionKey` while the live run key is `agent:main:main`, then call `session_status` with `sessionKey:"current"` against a store containing both rows.

## Solution Assessment

Yes. The narrow maintainable solution is to pass the live run session key separately from the sandbox/policy key and add regression coverage for a stale Telegram direct row plus a live main row.
