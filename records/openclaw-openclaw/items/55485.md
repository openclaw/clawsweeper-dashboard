## Summary

Keep open. The direction is useful, but the PR is not mergeable as-is because the branch introduces SSRF policy controls while leaving policy-agnostic caches and partially wired remote-media paths.

## What This Changes

The PR adds opt-in SSRF policy config for web_fetch, web_search citation redirects, browser/config schema surfaces, outbound remote media, generated config docs, and focused tests.

## Best Possible Solution

Keep the current main web_fetch cache-safe implementation, then land a rebased security-reviewed patch for citationRedirect schema/runtime/cache identity and either complete remoteMedia enforcement across all outbound media senders or a narrower documented config surface.

## Reproduction Assessment

Yes. The reproduction path is code-based: the PR diff threads policy-sensitive fetch/redirect/media settings while the relevant cache keys and some send paths omit the same policy before reuse or loading.

## Solution Assessment

No. The direction is maintainable, but the PR is not the best current form until policy-aware cache identity and complete media routing are in place.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Include SSRF policy in the web_fetch cache key:** `src/agents/tools/web-fetch.ts:549`
  - body: The branch forwards `params.ssrfPolicy` to guarded fetches, but the cache lookup is still keyed only by URL, extract mode, and max chars. A permissive run can populate the cache and a later stricter run can return that cached content before the SSRF guard runs, so the cache key needs a stable policy fingerprint or policy-sensitive cache bypass.
  - confidence: 0.93
- **[P2] Add citation redirect policy to Gemini cache identity:** `extensions/google/src/gemini-web-search-provider.ts:225`
  - body: The branch threads `citationRedirectSsrFPolicy` into redirect resolution, but Gemini search caching is still provider/query/count/model based. A permissive citation policy can cache resolved citations that stricter later calls reuse without reapplying the stricter redirect policy.
  - confidence: 0.88
- **[P2] Wire remoteMedia policy through every media sender:** `src/config/zod-schema.session.ts:188-193`
  - body: The new `messages.remoteMedia.ssrfPolicy` config is global, but the branch only forwards it through a subset of senders. Paths such as Slack and Telegram still load outbound media without this override, so the accepted config behaves differently by channel.
  - confidence: 0.86
