## Summary

Keep open. The PR is still useful, but current main only covers the web_fetch part safely; the branch adds broader SSRF policy surfaces while leaving policy-agnostic caches and incomplete remote-media routing, so it should not be closed or merged without maintainer/security review.

## What This Changes

The PR adds opt-in SSRF policy config for web_fetch, Gemini citation redirect resolution, browser/config schema surfaces, and outbound remote media, plus tests and legacy web-search normalization for citationRedirect.

## Best Possible Solution

Keep the current main web_fetch cache-safe implementation, then land the remaining source-level pieces in one reviewed patch: citationRedirect schema/types/docs and legacy-normalizer preservation, policy-aware Gemini redirect resolution and cache identity, and either consistent messages.remoteMedia enforcement across every outbound media path or a narrower documented config surface.

## Reproduction Assessment

Yes. The review path is code-based: the branch diff adds policy controls while the relevant cache keys and some media send paths omit policy, so a permissive run can populate cached or channel-specific results that later strict settings do not revalidate.

## Solution Assessment

No. The direction is maintainable, but the proposed patch is not the best current form because it introduces security controls without complete cache isolation or channel coverage; a smaller rebased patch with policy-aware cache keys and complete wiring would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Include SSRF policy in the web_fetch cache key:** `src/agents/tools/web-fetch.ts:549`
  - body: The patch forwards `params.ssrfPolicy` to guarded fetches, but the cache lookup is still keyed only by URL, extract mode, and max chars. A permissive run can populate the cache and a later stricter run can return that cached content before the SSRF guard runs. Add a stable policy fingerprint to the cache key or skip cache reuse across policies.
  - confidence: 0.93
- **[P2] Add citation redirect policy to Gemini cache identity:** `extensions/google/src/gemini-web-search-provider.ts:225`
  - body: This threads `citationRedirectSsrFPolicy` into redirect resolution, but Gemini search caching is still provider/query/count/model only. If a permissive citation policy resolves and caches citations, later strict calls can reuse those URLs without reapplying the stricter redirect policy.
  - confidence: 0.88
- **[P2] Wire remoteMedia policy through every media sender:** `src/config/zod-schema.session.ts:188-193`
  - body: The new `messages.remoteMedia.ssrfPolicy` config is global, but the branch only forwards it through a subset of senders. Paths such as Slack and Telegram still load outbound media without an SSRF override, so the accepted config silently behaves differently by channel.
  - confidence: 0.86
