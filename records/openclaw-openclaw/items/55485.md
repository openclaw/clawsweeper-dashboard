## Summary

Keep open; this PR is not safe to merge in its current form. Current main already has cache-separated web_fetch SSRF opt-in handling, while the PR’s broader SSRF plumbing still leaves policy-sensitive cache and outbound-media paths incomplete.

## What This Changes

The PR adds opt-in SSRF policy plumbing for web_fetch, Gemini citation redirects, and outbound remote media, with related config schema, generated docs, and focused tests.

## Best Possible Solution

Keep current main’s cache-safe web_fetch implementation, then land a rebased security-reviewed patch that adds citationRedirect schema/runtime/cache identity and either fully routes remoteMedia policy through outbound media or narrows the documented config surface.

## Reproduction Assessment

Yes. The reproduction path is code-based: policy-sensitive fetch, redirect, and media settings are introduced while the relevant cache identities or outbound-media paths can reuse or load content without the same policy.

## Solution Assessment

No. The direction is maintainable, but this PR is not the best current solution until policy-aware cache identity and complete or deliberately scoped media routing are in place.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Include SSRF policy in the web_fetch cache key:** `src/agents/tools/web-fetch.ts:549`
  - body: The branch forwards `params.ssrfPolicy` to guarded fetches, but cache lookup is still keyed only by URL, extract mode, and max chars. A permissive run can populate the cache and a later stricter run can return that cached content before the SSRF guard runs, so include a stable policy fingerprint or bypass cache for policy-sensitive fetches.
  - confidence: 0.93
- **[P2] Add citation redirect policy to Gemini cache identity:** `extensions/google/src/gemini-web-search-provider.ts:225`
  - body: Passing `citationRedirectSsrFPolicy` into redirect resolution makes citations policy-sensitive, but Gemini search caching is still provider/query/count/baseUrl/model based. Cached citations from a permissive policy can be reused by stricter calls without reapplying redirect checks.
  - confidence: 0.88
- **[P2] Wire remoteMedia policy through every media sender:** `src/config/zod-schema.session.ts:188-193`
  - body: The schema accepts a global `messages.remoteMedia.ssrfPolicy`, but the branch forwards it through only a subset of outbound media loaders. Channels such as Slack and Telegram still load remote media without this override, making the accepted config channel-dependent.
  - confidence: 0.86

## Security Review

Status: needs_attention

Concerns:

- **[high] Policy-agnostic caches can bypass stricter SSRF settings:** `src/agents/tools/web-fetch.ts:549`
  - body: The branch applies permissive SSRF policy during web_fetch and Gemini citation redirect resolution without including the full policy in the relevant cache identities, allowing later stricter calls to receive previously cached results.
