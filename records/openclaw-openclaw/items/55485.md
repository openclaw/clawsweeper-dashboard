## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Include SSRF policy in the web_fetch cache key:** `src/agents/tools/web-fetch.ts:549`
  - body: The branch forwards `params.ssrfPolicy` into guarded fetches, but cache hits are still keyed without that policy. A permissive run can populate the cache and a later stricter run can return cached content before the SSRF guard runs, so the cache key needs a stable policy component or policy-sensitive bypass.
  - confidence: 0.93
- **[P2] Add citation redirect policy to Gemini cache identity:** `extensions/google/src/gemini-web-search-provider.ts:225`
  - body: The branch threads `citationRedirectSsrFPolicy` into redirect resolution, but Gemini search caching still ignores that policy. Resolved citations from a permissive run can be reused by stricter later calls without reapplying redirect SSRF checks.
  - confidence: 0.88
- **[P2] Wire remoteMedia policy through every media sender:** `src/config/zod-schema.session.ts:188-193`
  - body: The new `messages.remoteMedia.ssrfPolicy` setting is global, but not every outbound media sender is wired to pass it into media loading. Channels that omit the option will still enforce default SSRF behavior, creating channel-dependent config behavior.
  - confidence: 0.86
