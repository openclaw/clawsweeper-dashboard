## What This Changes

The PR adds opt-in SSRF policy config/schema/runtime plumbing for web_fetch, Gemini citation redirects, remote outbound media, and legacy web-search normalization.

## Best Possible Solution

Rebase onto current main, keep the cache-safe web_fetch implementation, and land the citationRedirect and remoteMedia policy surfaces only with policy-aware cache identity plus complete sender coverage.

## Reproduction Assessment

Yes. The reproduction path is code-based: permissive SSRF policy can populate web_fetch or Gemini search caches, then stricter calls can read policy-agnostic cached data before the guard reruns; remoteMedia inconsistency is visible from send paths that omit the policy option.

## Solution Assessment

No. The direction is reasonable, but this PR is not the best current form until it preserves current main's cache isolation, adds citation redirect policy to Gemini cache identity, and completes remote-media routing.
