## Summary

Keep this issue open. Current main still does not accept SecretRef objects for the reported phone-number and sender-identity channel fields, and the open PR that explicitly closes this issue has not merged yet.

## What This Changes

The issue requests backward-compatible SecretRef support for phone-number and sender-identity fields in channel configs so PII values can be kept out of plaintext `openclaw.json`.

## Best Possible Solution

Review and either land or request changes on PR #73822. The desired end state is a channel/plugin-owned implementation that accepts both plain strings and SecretRef objects for phone-number and sender-identity fields, resolves refs before routing/allowlist/outbound checks, updates generated metadata/docs/redaction/status surfaces, and adds focused schema/runtime/target-registry tests.
