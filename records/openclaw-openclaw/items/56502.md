## What This Changes

Allow phone-number and equivalent sender-identifier channel config fields to accept SecretRef values while keeping existing plaintext strings backward-compatible.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73822, or an equivalent replacement, with additive SecretRef support for the affected identifier fields while preserving plain strings and the visible wildcard guard for public-open DM policy.

## Solution Assessment

Yes in direction. The maintainable fix is channel/plugin-owned SecretRef support through schemas, contracts, runtime resolution, metadata, and docs while keeping plain strings backward-compatible and preserving public-open access-control semantics.
