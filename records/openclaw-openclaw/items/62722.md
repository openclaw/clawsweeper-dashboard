## What This Changes

The PR changes `src/gateway/server-channels.ts` so `stopChannel` passes `AbortSignal.abort()` to `stopAccount` when no stored controller exists.

## Best Possible Solution

Rebase the one-line fix onto current main, add focused `server-channels` regression coverage for explicit account stop with no stored controller/task, and provide redacted real shutdown proof before merge.

## Reproduction Assessment

Yes at source level: calling `stopChannel(channel, accountId)` for a configured plugin with `stopAccount` but no stored controller/task reaches the fallback and current main passes a non-aborted signal. I did not run a live gateway/plugin shutdown in this read-only review.

## Solution Assessment

Yes, this is the narrowest apparent code fix: `AbortSignal.abort()` makes the no-controller fallback match the already-aborted primary path after `abort?.abort()`. It still needs rebase, regression coverage, compatibility acceptance, and real behavior proof before merge.
