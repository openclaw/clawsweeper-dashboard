## Summary

Keep open. Current main still has the non-aborted fallback, the Gateway stop hook contract passes that signal into plugin cleanup, and the proposed one-line runtime change is the narrow fix; the only merge blocker I found is the required changelog entry for a user-facing Gateway fix.

## What This Changes

The PR changes `src/gateway/server-channels.ts` so `stopChannel` passes an already-aborted `AbortSignal` to `plugin.gateway.stopAccount` when no stored account abort controller exists.

## Best Possible Solution

Land the one-line fallback change after adding the required `CHANGELOG.md` entry and re-running the targeted Gateway channel manager tests on the final head.

## Reproduction Assessment

Yes. A static reproduction exists on current main: stop an account with a plugin `stopAccount` hook but no stored abort controller, and the hook receives a fresh non-aborted fallback signal from `src/gateway/server-channels.ts`.

## Solution Assessment

Yes. `AbortSignal.abort()` is the narrowest maintainable fix because it only changes the no-controller fallback while preserving the existing stored-controller path and the Gateway stop hook contract.
