## What This Changes

The PR changes `stopChannel` so a `stopAccount` hook without an active account controller receives `AbortSignal.abort()` instead of a never-aborted signal.

## Best Possible Solution

Rebase the narrow fallback change onto current main, add a focused gateway regression test for explicit stop without a controller, and require redacted real shutdown proof before merge.

## Reproduction Assessment

Yes at source level: current main reaches `abort?.signal ?? new AbortController().signal` when `stopChannel(channel, accountId)` calls a `stopAccount` hook without a stored controller. I did not run a live gateway shutdown in this read-only review.

## Solution Assessment

Yes, this appears to be the narrowest code fix because `AbortSignal.abort()` makes the fallback match the already-aborted primary path after `abort?.abort()`. It still needs rebase, regression coverage, compatibility acceptance, and real behavior proof before merge.
