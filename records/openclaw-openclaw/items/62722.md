## What This Changes

The PR replaces the no-controller `stopChannel` fallback signal passed to `stopAccount` with `AbortSignal.abort()` in `src/gateway/server-channels.ts`.

## Best Possible Solution

Rebase the fallback change onto current `stopChannel`, cover the no-controller stop hook path with focused regression proof or equivalent validation, and attach redacted real shutdown evidence before merge.

## Reproduction Assessment

Yes, source-reproducible: current main passes a fresh non-aborted fallback signal to `stopAccount` when no controller exists, while abort-aware helpers such as `waitUntilAbort` only complete immediately for an already-aborted signal. I did not run a live gateway/plugin repro in this read-only review.

## Solution Assessment

Yes. Replacing the fallback with `AbortSignal.abort()` is the narrow maintainable fix because the existing-controller path already calls `abort?.abort()` before handing the signal to the stop hook.
