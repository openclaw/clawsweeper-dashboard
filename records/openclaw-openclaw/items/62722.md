The PR replaces the `stopChannel` no-controller fallback signal passed to `gateway.stopAccount` with `AbortSignal.abort()` in `src/gateway/server-channels.ts`.

## Best Possible Solution

Land the fallback change after rebasing onto current main and adding either focused no-controller stop hook coverage or redacted runtime proof that shutdown cleanup no longer waits on a never-aborted signal.

## Reproduction Assessment

Yes, source-reproducible: current main passes a fresh non-aborted fallback signal to `stopAccount` when no controller exists, while abort-aware helpers complete only when the signal is already aborted. I did not run a live gateway/plugin shutdown repro.

## Solution Assessment

Yes. Replacing the fallback with `AbortSignal.abort()` is the narrow maintainable fix because the existing-controller path already calls `abort?.abort()` before handing the signal to the stop hook.
