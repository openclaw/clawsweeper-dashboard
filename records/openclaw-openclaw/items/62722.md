## What This Changes

The PR changes `src/gateway/server-channels.ts` so `stopChannel` passes an already-aborted `AbortSignal` to `plugin.gateway.stopAccount` when no stored account abort controller exists.

## Best Possible Solution

Land the one-line fallback change with a single-line changelog entry and final Gateway validation; use a narrow credited replacement if the source branch cannot be refreshed.

## Reproduction Assessment

Yes. A static reproduction exists on current main: stop an account for a plugin with `stopAccount` but no stored abort controller or running task, and the hook receives `new AbortController().signal` with `aborted === false`.

## Solution Assessment

Yes. `AbortSignal.abort()` is the narrowest maintainable fix because it changes only the no-controller fallback while preserving the existing stored-controller path and plugin Gateway context contract.
