## What This Changes

The PR changes `src/gateway/server-channels.ts` so `stopAccount` receives `AbortSignal.abort()` when `stopChannel` has no stored controller.

## Best Possible Solution

Land the fallback change after the branch is refreshed, focused gateway validation passes, and redacted real behavior proof shows the shutdown path no longer waits on a never-aborted signal.

## Reproduction Assessment

Yes from source inspection, not from an executed runtime repro. Current main can pass a fresh non-aborted signal to `stopAccount` when no controller exists, while abort-aware helpers wait unless `signal.aborted` is already true.

## Solution Assessment

Yes. Replacing the fallback with `AbortSignal.abort()` is the narrow maintainable fix because it makes the no-controller path match the existing-controller path after `abort?.abort()`.
