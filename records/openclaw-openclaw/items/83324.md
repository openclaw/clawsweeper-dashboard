## Summary

This PR looks like a focused and reasonable fix for the lightweight help-path root cause, but it should stay open for protected-label human review and landing rather than cleanup closure.

## What This Changes

The PR moves gateway, doctor, status, and health CLI action/runtime modules behind lazy imports so subcommand help can render through metadata-only registration paths.

## Best Possible Solution

Land this lazy-import fix after protected-label review and clean gates, then keep https://github.com/openclaw/openclaw/issues/83228 as the follow-up place for any remaining rootless Podman runtime crashes or silent output.

## Reproduction Assessment

## Solution Assessment

Yes for the reviewed root cause: deferring action/runtime imports until command execution is the narrowest maintainable fix for subcommand help import crashes without changing CLI semantics.
