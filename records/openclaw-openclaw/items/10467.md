## Summary

Keep open. Current main still exposes `sessions_spawn` through one fixed `subagent` lane, with no caller-selected lane or per-lane subagent concurrency configuration.

## What This Changes

Add multi-lane concurrency for `sessions_spawn`, including an optional lane parameter and per-lane subagent concurrency configuration.

## Best Possible Solution

Add a validated additive `lane` field that defaults to `subagent`, thread it through native and ACP spawns, and support compatible per-lane subagent concurrency config with schema, docs, and focused tests.

## Reproduction Assessment

Yes, by source inspection. Current main has no `sessions_spawn.lane` schema field, native and ACP paths hard-code `subagent`, and docs/config expose one scalar subagent cap.

## Solution Assessment

Yes. The narrow maintainable path is an additive lane parameter backed by the existing string-lane queue, preserving current defaults while adding bounded validation, config, docs, and tests.
