## Summary

Keep open. Current main and the latest release still expose `sessions_spawn` through one fixed `subagent` queue lane, with no caller-selected `lane` parameter or per-lane subagent concurrency config; related open issues cover per-agent, per-parent, or output lifecycle concerns but do not supersede this exact request.

## What This Changes

Request adding optional named lanes to `sessions_spawn`, with independent per-lane subagent concurrency and the current `subagent` lane as the default.

## Best Possible Solution

Add a validated optional `lane` field that defaults to `subagent`, thread it through native and ACP spawns, and support compatible per-lane subagent concurrency config with schema, docs, and focused tests.

## Reproduction Assessment

Yes, by source inspection. Current main has no `sessions_spawn.lane` schema field, native and ACP paths hard-code `subagent`, and docs/config expose one scalar subagent cap.

## Solution Assessment

Yes. The narrow maintainable path is an additive lane parameter backed by the existing string-lane queue, preserving current defaults while adding bounded validation, config, docs, and tests.
