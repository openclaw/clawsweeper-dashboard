## Summary

Keep open. Current main and the latest shipped tag still expose one fixed `subagent` lane for `sessions_spawn`; the tool schema has no lane field, native and ACP dispatch hard-code that lane, and config/docs still describe one scalar sub-agent concurrency cap.

## What This Changes

Request adds caller-selected lanes to `sessions_spawn` plus per-lane sub-agent concurrency configuration so independent sub-agent groups do not share one global queue.

## Best Possible Solution

Add a validated `sessions_spawn.lane` field defaulting to `subagent`, forward it through native and ACP dispatch, and support compatible per-lane subagent concurrency config with docs and regression tests.

## Reproduction Assessment

Yes. The missing capability is source-reproducible: current main has no `sessions_spawn.lane` schema field, both native and ACP paths hard-code `subagent`, and config/docs expose only one scalar sub-agent cap.

## Solution Assessment

Yes. The narrow maintainable path is an additive lane parameter backed by the existing string-lane queue, preserving the current default and adding focused native/ACP/config/docs coverage.
