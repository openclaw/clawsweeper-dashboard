Request to add named lanes to `sessions_spawn`, defaulting to `subagent`, with per-lane sub-agent concurrency configuration.

## Best Possible Solution

Add a validated `sessions_spawn.lane` field defaulting to `subagent`, forward it through native and ACP dispatch, and support compatible per-lane subagent concurrency config with docs and regression tests.

Yes. This is source-reproducible on current main: the tool has no `lane` field, both native and ACP paths hard-code `subagent`, and docs/config still expose one scalar sub-agent cap.

## Solution Assessment

Yes. The narrow maintainable path is an additive lane parameter backed by the existing string-lane queue, with validation, default fallback, compatible per-lane config, and focused coverage.
