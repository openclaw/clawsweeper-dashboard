## Summary

Keep open. Current main still has one fixed `subagent` lane for `sessions_spawn`: the tool schema has no lane field, both native and ACP dispatch hard-code the subagent lane, and config/docs still expose only one scalar sub-agent concurrency cap.

## What This Changes

The issue requests an optional `sessions_spawn.lane` parameter plus per-lane sub-agent concurrency configuration while preserving the current `subagent` default.

## Best Possible Solution

Add a validated additive `sessions_spawn.lane` field defaulting to `subagent`, thread it through native and ACP dispatch, and support compatible per-lane subagent concurrency config with schema, docs, and regression coverage.

## Reproduction Assessment

Yes. The missing capability is source-reproducible on current main: there is no `sessions_spawn.lane` schema field, native and ACP paths hard-code `subagent`, and docs/config expose one scalar sub-agent cap.

## Solution Assessment

Yes. The narrow maintainable path is an additive lane parameter backed by the existing string-lane queue, preserving the current default while adding focused native/ACP/config/docs coverage.
