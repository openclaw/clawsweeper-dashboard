## Summary

Keep open. Current main still lacks caller-selectable `sessions_spawn` lanes: the tool schema has no `lane` input, native and ACP dispatch remain pinned to `subagent`, and config/docs/tests still model one scalar sub-agent concurrency cap.

## What This Changes

Request to add caller-selectable lanes to `sessions_spawn` with compatible per-lane sub-agent concurrency configuration.

## Best Possible Solution

Add a validated `sessions_spawn.lane` field defaulting to `subagent`, forward it through native and ACP dispatch, and support compatible per-lane subagent concurrency config with docs and regression tests.

## Reproduction Assessment

Yes. This is source-reproducible on current main: `sessions_spawn` has no lane field, both dispatch paths hard-code `subagent`, and config/docs/tests still describe one scalar sub-agent lane cap.

## Solution Assessment

Yes. The narrow maintainable direction is an additive lane parameter backed by the existing string-lane queue, with strict lane validation, default fallback, and compatible per-lane config handling.
