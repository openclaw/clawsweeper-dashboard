## Best Possible Solution

Add a validated additive `sessions_spawn.lane` field that defaults to `subagent`, threads through native and ACP spawns, and configures bounded per-lane subagent concurrency while preserving current defaults.

## Reproduction Assessment

Yes, by source inspection. Current main and v2026.6.5 lack a `sessions_spawn.lane` schema field, both native and ACP dispatch are hard-coded to `subagent`, and config/docs expose only one scalar subagent cap.

## Solution Assessment

Yes. The narrow maintainable solution is an additive lane parameter backed by the existing named-lane queue, with bounded validation, compatible config defaults, docs, and focused tests.
