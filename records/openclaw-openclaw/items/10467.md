## Summary

Keep open. Current main still lacks a caller-selectable `sessions_spawn` lane, while the prior maintainer-side comments and current source inspection agree that this is a real bounded feature gap rather than stale, implemented, duplicate, or ClawHub-only work.

## What This Changes

The issue requests an optional `sessions_spawn.lane` parameter with per-lane sub-agent concurrency configuration.

## Best Possible Solution

Implement a backward-compatible lane API that defaults to `subagent`, validates lane names, forwards the resolved lane through native and ACP spawn paths, applies compatible per-lane sub-agent concurrency on startup/reload, and updates schema/help/docs/changelog plus regression tests.

## Reproduction Assessment

Yes. Static inspection on current main reproduces the gap: no tool schema field exists, no execute path reads a lane, and both native and ACP child dispatches still send the fixed `subagent` lane.

## Solution Assessment

Yes. An additive `sessions_spawn.lane` parameter is the narrowest maintainable direction because the Gateway queue already supports arbitrary string lanes and configurable concurrency; adding a separate scheduler would be broader than needed.
