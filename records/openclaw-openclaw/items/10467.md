## Label Justifications

- `impact:session-state`: The request changes queue assignment, fairness, and concurrency isolation for spawned agent sessions.

## Summary

Current main still has exactly one fixed sub-agent queue lane, but the requested free-form lane API and independent concurrency map would expand model-facing and configuration contracts while potentially bypassing the existing global safety budget. Keep the issue open for an explicit maintainer decision on the bounded scheduling model; related open issues cover per-target and per-parent limits but do not supersede workload-class isolation.

## What This Changes

Add an optional lane selector to `sessions_spawn` and independently configurable per-lane concurrency, defaulting existing calls to the current `subagent` lane.

## Best Possible Solution

Define one bounded scheduling model with an aggregate sub-agent ceiling and a single canonical isolation key; only then add an additive spawn input and config shape with deterministic validation, native/ACP parity, reload behavior, docs, and focused tests.

## Reproduction Assessment

Yes at source level. Current main exposes no lane argument and hard-codes both native and ACP spawns to the global sub-agent lane, which proves the missing capability rather than a broken existing contract.

## Solution Assessment

No, not as currently specified. Workload isolation is a reasonable goal, but arbitrary independent lanes need an aggregate resource budget and a maintainer-selected relationship to per-agent and per-parent limits before they are the narrowest maintainable solution.
