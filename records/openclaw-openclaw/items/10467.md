## Summary

Keep open. Current main still exposes `sessions_spawn` as a single fixed `subagent` queue lane, with no caller lane parameter or per-lane sub-agent concurrency config.

## What This Changes

Add optional multi-lane support to `sessions_spawn`, including independent per-lane concurrency settings while preserving the default `subagent` lane.

## Best Possible Solution

Implement a backward-compatible `lane` field that defaults to `subagent`, validates lane names, forwards the resolved lane through native and ACP spawns, and applies per-lane concurrency config with docs and regression coverage.

## Reproduction Assessment

Yes for the feature gap. Static source inspection on current main shows no `lane` field in the tool schema, no execute-path forwarding, and hard-coded `subagent` dispatch for both native and ACP child runs.

## Solution Assessment

Yes. The existing command queue already supports named lanes, so an additive `sessions_spawn.lane` plus compatible per-lane config is the narrowest maintainable direction while preserving current callers.
