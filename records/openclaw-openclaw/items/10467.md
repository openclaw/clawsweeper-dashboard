## What This Changes

Request an additive sessions_spawn lane parameter plus compatible per-lane sub-agent concurrency configuration.

## Best Possible Solution

Add a validated sessions_spawn.lane surface that defaults to subagent, forward it through native and ACP spawns, and add compatible per-lane subagent concurrency config plus docs/tests while preserving existing callers.

## Solution Assessment

Yes. An additive lane parameter backed by the existing string-based command queue is the narrowest maintainable direction, provided existing callers continue to default to subagent and lane names/config are bounded.
