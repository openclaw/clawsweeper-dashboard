## Summary

Keep open. Current main still routes `sessions_spawn` through one fixed `subagent` lane, and source, config, docs, and tests still expose only scalar sub-agent concurrency.

## What This Changes

Request an optional `sessions_spawn.lane` parameter with independent per-lane sub-agent concurrency config while defaulting to `subagent`.

## Best Possible Solution

Add a backward-compatible `lane` field, validate and normalize lane names, forward it through native and ACP spawns, support per-lane concurrency config, and update docs, schema, tests, and changelog.

## Reproduction Assessment

Yes for the feature gap. Static source inspection on current main shows no `lane` field in the tool schema, no execute-path forwarding, and both native and ACP child dispatches hard-code the `subagent` lane.

## Solution Assessment

Yes. An additive `sessions_spawn.lane` parameter that defaults to `subagent` is the narrowest maintainable direction because the command queue already supports arbitrary named lanes and existing callers can remain unchanged.
