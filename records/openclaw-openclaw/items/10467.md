## Summary

Keep open. Current main still exposes `sessions_spawn` through one fixed `subagent` lane, and the recent maintainer-side comments are still accurate: the tool schema, native/ACP dispatch, config schema, docs, and tests have not gained caller-selected lanes or per-lane subagent limits. Related items are adjacent rather than superseding this request.

## What This Changes

Request adds caller-selectable lanes to `sessions_spawn` plus per-lane subagent concurrency configuration.

## Best Possible Solution

Add an additive lane-aware `sessions_spawn` contract that defaults to `subagent`, validates lane names, forwards through native and ACP spawn paths, and keeps existing callers compatible.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence verification path: the current schema, spawn params, dispatch calls, config schema, docs, and tests still expose one fixed `subagent` lane.

## Solution Assessment

Yes. The existing command queue already supports named lanes, so exposing a validated `sessions_spawn.lane` plus compatible per-lane subagent config is the narrow maintainable direction while preserving current defaults.
