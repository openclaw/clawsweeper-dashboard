Add optional multi-lane concurrency support to `sessions_spawn`, including native/ACP lane forwarding and compatible per-lane subagent concurrency configuration.

## Best Possible Solution

Add a validated additive `lane` field that defaults to `subagent`, thread it through native and ACP spawns, and support compatible per-lane subagent concurrency config with schema, docs, and focused tests.

## Solution Assessment

Yes. The narrow maintainable solution is an additive lane parameter backed by the existing named-lane queue, preserving current defaults while adding bounded validation, config, docs, and regression tests.
