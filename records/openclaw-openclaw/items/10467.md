## Summary

Keep open. Current main still lacks caller-selectable `sessions_spawn` lanes, and both maintainer discussion plus current source/docs/tests show the feature remains a bounded OpenClaw gap rather than implemented, duplicate, or ClawHub-only work.

## What This Changes

Add an optional `sessions_spawn.lane` parameter with compatible per-lane sub-agent concurrency configuration and default fallback to the existing `subagent` lane.

## Best Possible Solution

Implement a backward-compatible lane API that defaults to `subagent`, validates lane names, forwards the resolved lane through native and ACP spawn paths, applies per-lane sub-agent concurrency at startup/reload, and updates schema, docs, changelog, and regression tests.

## Reproduction Assessment

Yes. Static source inspection on current main reproduces the gap: no tool schema field exists, no execute path reads a lane, and both native and ACP child dispatches still send the fixed `subagent` lane.

## Solution Assessment

Yes. An additive `sessions_spawn.lane` parameter is the narrowest maintainable direction because the Gateway queue already supports arbitrary string lanes and independent concurrency caps.
