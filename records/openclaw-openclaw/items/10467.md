## Summary

Keep open. Current main and v2026.5.22 still lack caller-selected `sessions_spawn` lanes; native and ACP spawned sessions are both pinned to the fixed `subagent` lane, and no open PR owns the exact requested fix.

## What This Changes

Request an additive `sessions_spawn.lane` parameter, independent subagent queue lanes, and configurable per-lane subagent concurrency while preserving the default `subagent` lane.

## Best Possible Solution

Add a validated additive `lane` field that defaults to `subagent`, thread it through native and ACP subagent spawns, and support compatible per-lane subagent concurrency with schema, docs, and focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main and v2026.5.22 expose no `sessions_spawn.lane`, both native and ACP spawn paths hard-code the `subagent` lane, and config remains a scalar subagent concurrency cap.

## Solution Assessment

Yes. The narrow maintainable solution is an additive lane parameter backed by the existing named-lane queue, preserving current defaults while adding bounded validation, config, docs, and regression tests.
