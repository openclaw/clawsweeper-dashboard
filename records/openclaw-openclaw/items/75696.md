## Summary

Keep open. Current main has a narrow Codex app-server MCP elicitation bridge, but it only handles Codex-tagged MCP tool approvals; unsupported Computer Use app-control elicitations still fall through to a decline response, and the Computer Use troubleshooting docs do not yet describe this Discord/channel-routed approval-denied symptom.

## What This Changes

The issue reports that Codex Computer Use app-control approvals fail closed in an OpenClaw Discord/Codex harness session even though the same Computer Use plugin works in direct Codex Desktop chat.

## Best Possible Solution

Keep the issue open and land a docs-first fix that explicitly documents the channel-routed Computer Use app-control approval limitation and troubleshooting path. A runtime bridge should wait for a redacted upstream payload and maintainer review of the permission boundary before broadening MCP elicitation handling.
