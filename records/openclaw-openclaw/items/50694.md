## Summary

Keep open. Current main still collapses the token-based memory-flush gate when the reserve floor consumes the context window, while this PR contains a focused implementation for that bug; it is not closeable because the PR head is currently dirty/unmergeable and its latest available check run is not clean.

## What This Changes

The PR adds a shared context-aware memory-flush threshold resolver, uses it in preflight and runtime memory-flush gates, expands regression coverage for oversized reserve floors and small context windows, and adds a changelog entry for the linked bug.

## Best Possible Solution

Rebase or apply an equivalent narrow patch that keeps one shared context-aware threshold helper for preflight and runtime memory-flush gates, preserves the regression tests and changelog credit, then close #50611 after the fix lands.

## Reproduction Assessment

Yes. On current main, contextWindowTokens=200000, reserveTokensFloor=200000, and softThresholdTokens=5000 produce threshold=0 in resolveMemoryFlushGateState, which returns null and prevents the token-based memory-flush gate from firing.

## Solution Assessment

Yes, pending rebase and green checks. The latest PR branch is a better direction than the original fixed-percentage clamp because it uses the existing Pi prompt-budget boundary, centralizes the calculation, and applies the same threshold to diagnostics and runtime gating.
