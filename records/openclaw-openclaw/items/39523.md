## Summary

Keep open. Current main still does not implement Feishu channel/account `mediaLocalRoots` config support, and this PR is not ready to merge because its new fallback does not work in the normal outbound path and its advertised `"any"` mode conflicts with the current host-read safety contract.

## What This Changes

The PR adds `mediaLocalRoots` to the Feishu shared config schema and makes `sendMediaFeishu` fall back to account config when no runtime roots are passed.

## Best Possible Solution

A Feishu-owned replacement or rework should add the config key to top-level/account schemas, generated metadata, plugin manifest/docs as appropriate, define how configured roots combine with runtime-scoped media access, route any supported `"any"` mode through `mediaReadFile`/host-read capability, and add focused regression tests for direct sends, action sends, and reply-dispatcher paths.

## Reproduction Assessment

Yes for code-level reproduction. Configure Feishu account `mediaLocalRoots: "any"` and send a local path through the PR's `sendMediaFeishu` path: the PR forwards `localRoots: "any"` without a readFile override, while current `loadWebMedia` rejects that with `unsafe-bypass`; ordinary outbound sends also pass injected roots, so config fallback is bypassed.

## Solution Assessment

No. The PR is directionally useful but not the best fix because it adds a schema field without docs/metadata/tests, uses nullish fallback where normal callers already pass roots, and forwards `"any"` outside the shared host-read safety helper.

  - confidence: 0.9
