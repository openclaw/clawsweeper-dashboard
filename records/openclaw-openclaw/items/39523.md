## Summary

Keep open: current main still lacks Feishu `mediaLocalRoots` config support, but this PR is not mergeable because it forwards `"any"` outside the host-read media contract and lets injected runtime roots shadow the configured account value.

## What This Changes

The PR adds a Feishu `mediaLocalRoots` config schema field and makes `sendMediaFeishu` fall back from the runtime parameter to account config, including `"any"`.

## Best Possible Solution

A Feishu-owned update should expose the config surface across schema, generated metadata, docs, tests, and changelog, then load local media through the shared outbound host-read media contract with explicit root precedence.

## Reproduction Assessment

Yes for the review findings. Source inspection shows the PR's configured `"any"` path reaches `loadWebMedia` without `readFile` and that regular outbound sends already provide runtime roots that suppress the account-config fallback.

## Solution Assessment

No. The PR is directionally useful, but the safer path is to use the shared outbound media load/readFile contract, define config-versus-runtime precedence, and update generated config surfaces before exposing the setting.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Route `any` through host-read media loading:** `extensions/feishu/src/media.ts:459`
  - body: `loadWebMedia` rejects `localRoots: "any"` unless a `readFile` override is also provided. This change forwards configured `"any"` directly from `sendMediaFeishu`, so the advertised setting fails with `unsafe-bypass` instead of using the shared safe helper path.
  - confidence: 0.95
- **[P2] Respect configured roots in the main send path:** `extensions/feishu/src/media.ts:450`
  - body: Regular Feishu outbound sends already pass `ctx.mediaLocalRoots` into this function. Because the fallback uses `mediaLocalRoots ?? account.config?.mediaLocalRoots`, those injected roots suppress configured account roots or `"any"`, so the new config still does not work in the main flow this PR targets.
  - confidence: 0.9
- **[P2] Regenerate config metadata for the new key:** `extensions/feishu/src/config-schema.ts:165`
  - body: Adding the schema field without updating generated config metadata leaves the Feishu public config surface out of sync; current generated metadata has `mediaMaxMb` followed by `httpTimeoutMs` with no `mediaLocalRoots`. The generated metadata/docs check should be updated with the schema change.
  - confidence: 0.82
