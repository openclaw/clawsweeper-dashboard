## Summary

Keep open: current main still lacks Feishu config-backed media roots, but this PR should not merge as-is because its `"any"` path misses the host-read safety contract and its fallback is shadowed in the main outbound flow.

## What This Changes

The PR adds a Feishu `mediaLocalRoots` config field and makes `sendMediaFeishu` fall back to the resolved account config when runtime roots are not supplied.

## Best Possible Solution

A Feishu-owned rework should add the config surface across schema, metadata, docs, and tests, route local media through the shared host-read media contract, and define root precedence explicitly.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configured `mediaLocalRoots: "any"` would be forwarded to `loadWebMedia` without `readFile` and hit the `unsafe-bypass` guard, while regular outbound sends already pass injected roots that suppress the config fallback.

## Solution Assessment

No. The PR is directionally useful, but the safer fix is to use the shared outbound media load/readFile contract and settle config-vs-runtime root precedence before exposing the setting.

## Review Findings

Overall correctness: patch is incorrect

  - body: `loadWebMedia` rejects `localRoots: "any"` unless a `readFile` override is also provided. This change forwards configured `"any"` directly from `sendMediaFeishu`, so the advertised setting fails closed with `unsafe-bypass` instead of using the shared safe helper path.
  - confidence: 0.95

  - confidence: 0.9
