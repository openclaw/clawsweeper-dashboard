## Summary

Keep open. The PR addresses a real Feishu local-media config gap that still exists on current main, but the patch is not ready to merge because it mishandles the security-sensitive `"any"` local filesystem mode and its config fallback is bypassed in the normal outbound path.

## What This Changes

The PR adds `mediaLocalRoots` to Feishu shared config and makes `sendMediaFeishu` fall back to the account config when runtime roots are not provided.

## Best Possible Solution

A Feishu-owned rework should add the config key across schema, manifest/docs/generated metadata as needed, route configured local media through the shared host-read media helper contract, define how configured roots combine with runtime-scoped roots, and cover direct send, outbound action, and reply dispatcher paths with focused regression tests.

## Reproduction Assessment

Yes. A code-level reproduction is clear from the PR diff and current main: configured `mediaLocalRoots: "any"` would be forwarded to `loadWebMedia` without a `readFile` override and hit the unsafe-bypass guard, while normal outbound sends already pass injected roots so the account-config fallback is not used.

## Solution Assessment

No. The PR is directionally useful, but the safest maintainable solution is to use the shared outbound media load/readFile contract and explicitly resolve config-vs-runtime root precedence instead of forwarding `"any"` directly from `sendMediaFeishu`.
