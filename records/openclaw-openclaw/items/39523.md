## Summary

Keep open: current main still lacks Feishu config-backed `mediaLocalRoots`, but this PR is not safe to merge as-is because the advertised `"any"` path fails the shared media loader guard and the config fallback is skipped in normal outbound sends.

## What This Changes

The PR adds a Feishu `mediaLocalRoots` config key and makes `sendMediaFeishu` fall back to account config, including forwarding `"any"` to `loadWebMedia`.

## Best Possible Solution

A Feishu-owned rework should add schema, generated metadata, docs, and tests while loading media through the shared outbound media/readFile contract with explicit root precedence.

## Reproduction Assessment

Yes. Source inspection gives high-confidence paths: configured `mediaLocalRoots: "any"` reaches `loadWebMedia` without `readFile` and fails `unsafe-bypass`, and normal outbound sends already supply non-empty `ctx.mediaLocalRoots` so account config fallback is skipped.

## Solution Assessment

No. The PR is directionally useful, but the narrow maintainable fix needs the shared outbound media host-read contract plus an explicit policy for config-vs-runtime root precedence.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Route any through the host-read media contract:** `extensions/feishu/src/media.ts:459`
  - body: `loadWebMedia` rejects `localRoots: "any"` unless a `readFile` override is also provided. This change forwards configured `"any"` directly, so the advertised all-path setting fails with `unsafe-bypass` instead of using the shared outbound media/readFile path.
  - confidence: 0.95
- **[P2] Let configured roots participate before injected defaults:** `extensions/feishu/src/media.ts:450`
  - body: Regular Feishu outbound sends already pass non-empty `ctx.mediaLocalRoots` into this function. With `mediaLocalRoots ?? account.config?.mediaLocalRoots`, configured account roots or `"any"` are skipped in the main flow the PR is trying to fix.
  - confidence: 0.9
- **[P2] Update generated config metadata with the new key:** `extensions/feishu/src/config-schema.ts:165`
  - body: Adding a runtime schema field without updating generated config metadata/docs leaves the public config surface unaware of `channels.feishu.mediaLocalRoots` and should trip the config docs generation check. Please update the generated/docs surface with the schema change.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff changes persistent local filesystem media access and advertises an all-path mode without routing through the shared host-read safety contract.

Concerns:

- **[medium] All-path local media mode bypasses the safe helper path:** `extensions/feishu/src/media.ts:459`
  - body: The PR allows `mediaLocalRoots: "any"` in Feishu config and forwards it directly to `loadWebMedia` without `mediaReadFile` or `hostReadCapability`; current code fails closed, but the design should be routed through the shared outbound media helper before merge.
  - confidence: 0.9
- **[medium] Persistent root precedence is not defined:** `extensions/feishu/src/media.ts:450`
  - body: The patch introduces account-level filesystem roots while normal sends already carry agent-scoped runtime roots. Without an explicit precedence policy, a future fix could accidentally broaden local file access beyond the intended agent-scoped boundary.
