## Summary

Keep open. Current main still lacks Feishu `mediaLocalRoots` config support, and this PR is directionally useful, but it should not merge as-is because it forwards `"any"` outside the shared host-read media contract and the fallback is shadowed in the normal outbound path.

## What This Changes

The PR adds Feishu `mediaLocalRoots` to channel/account config and makes `sendMediaFeishu` fall back from runtime roots to account config, including `"any"`.

## Best Possible Solution

A Feishu-owned rework should add the config surface across schema, manifest/metadata, docs, and changelog, route local media through the shared outbound host-read helper, and define account-config versus runtime-root precedence explicitly.

## Reproduction Assessment

Yes. Source inspection shows configured `mediaLocalRoots: "any"` in the PR reaches `loadWebMedia` without `readFile` and trips `unsafe-bypass`; normal outbound sends already pass `ctx.mediaLocalRoots`, so the config fallback is not used there.

## Solution Assessment

No. The current PR is in the right area, but the maintainable fix should use the shared outbound media load/read contract and settle root precedence before exposing persistent local roots.
