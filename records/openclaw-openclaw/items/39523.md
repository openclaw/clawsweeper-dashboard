## Summary

Keep this PR open: current main still lacks Feishu account-level media roots, but the proposed patch is not merge-safe because it does not use the shared host-read contract and the main send path would still prefer injected runtime roots over config.

## What This Changes

The PR adds a Feishu `mediaLocalRoots` config field and makes `sendMediaFeishu` fall back to account config, including the `"any"` mode, when loading media.

## Best Possible Solution

A Feishu-owned update should expose the config only after routing local media through the shared outbound media access/readFile contract, defining root precedence, and updating metadata, docs, tests, and changelog.

## Reproduction Assessment

Yes for the review findings from source inspection: `loadWebMedia` rejects `localRoots: "any"` without `readFile`, and the main Feishu outbound path already passes `ctx.mediaLocalRoots`, which suppresses the account-config fallback.

## Solution Assessment

No. The direction is useful, but the narrow maintainable fix is to use the shared outbound media access/readFile contract and explicitly define config-versus-runtime root precedence before exposing the setting.

## Review Findings

  - body: `loadWebMedia` rejects `localRoots: "any"` unless a `readFile` override is also provided. This change forwards configured `"any"` directly from `sendMediaFeishu`, so the advertised setting fails with `unsafe-bypass` instead of enabling local sends through the shared safe helper path.
  - confidence: 0.95

  - body: Regular Feishu outbound sends already pass `ctx.mediaLocalRoots` into this function. Because the fallback uses `mediaLocalRoots ?? account.config?.mediaLocalRoots`, injected runtime roots suppress configured account roots or `"any"`, so the new config still does not work in the main flow this PR targets.
  - confidence: 0.9

  - body: Adding the schema field without updating generated config metadata leaves the Feishu public config surface out of sync; the current generated Feishu metadata still has `mediaMaxMb` followed by `httpTimeoutMs` with no `mediaLocalRoots`.
  - confidence: 0.82
