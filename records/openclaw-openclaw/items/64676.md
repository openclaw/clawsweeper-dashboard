## Summary

Keep open. Current `main` still regenerates the active agent `models.json` on gateway startup and preserves only selected `apiKey`/`baseUrl` fields for matching providers, so durable user pruning of provider model arrays remains an unresolved product/config contract.

## What This Changes

The issue requests that gateway restarts stop regenerating per-agent `models.json` in a way that restores removed model entries after a user edits the file.

## Best Possible Solution

Maintainers should define the ownership contract, then either preserve pruned model arrays with focused regression coverage or document/route durable pruning through the supported config `--replace` path.

## Reproduction Assessment

Yes, from source inspection. Gateway startup still calls `ensureOpenClawModelsJson`, the planner still rewrites changed contents in default merge mode, and matching providers do not preserve an existing pruned `models` array.

## Solution Assessment

Unclear. Preserving direct deletions from `models.json` is not the narrowest safe code-only bug fix until maintainers decide whether matching provider model arrays are editable source of truth or regenerated catalog output.
