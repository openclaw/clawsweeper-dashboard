## Summary

Keep open. Current main still routes gateway startup through `models.json` generation, and merge mode can restore implicit/catalog model entries that a user manually removed; the docs describe that generated-state behavior, but the requested manual-pruning persistence contract remains undecided.

## What This Changes

The issue requests that gateway restarts preserve user-edited per-agent `models.json` model lists instead of regenerating default or cached catalog entries.

## Best Possible Solution

Maintainers should define the `models.json` ownership contract: support direct pruning with durable deletions and a restart regression, or keep it generated and provide a clear config/CLI path for persistent model-list replacement.

## Reproduction Assessment

Yes. The issue's restart steps map to the inspected startup path: gateway prewarm calls `ensureOpenClawModelsJson`, the planner rewrites on content differences, and merge mode appends unseen implicit models.

## Solution Assessment

Unclear. Preserving direct edits may be the desired user-facing behavior, but current docs and code also consistently treat `models.json` as generated merge state, so maintainer product/API judgment should choose the contract before implementation.
