## Summary

Keep open. Current main still regenerates the active agent models registry during gateway startup and refreshes matching provider model arrays from generated/configured provider data, while current docs describe that as generated merge behavior; the remaining question is a product/config contract decision, not an autoclose candidate.

## What This Changes

The issue asks OpenClaw to preserve direct per-agent `models.json` pruning instead of restoring regenerated/default provider model arrays after gateway restart.

## Best Possible Solution

Maintainers should define whether matching provider `models` arrays in per-agent `models.json` are editable source of truth or generated catalog output, then either preserve pruned arrays with focused regression coverage or route durable pruning through the supported config `--replace` path and docs.

## Reproduction Assessment

Yes, from source inspection. Gateway startup still calls `ensureOpenClawModelsJson`, the planner still rewrites changed merge-mode contents, and the merge helper preserves selected auth/baseUrl fields but not an existing pruned provider `models` array.

## Solution Assessment

Unclear. Preserving direct deletions from `models.json` may be the desired UX, but it changes the documented generated-catalog contract and should be decided before a code-only repair.
