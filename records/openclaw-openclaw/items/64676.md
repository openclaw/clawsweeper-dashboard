## Summary

Keep open. Current main still can regenerate the active agent `models.json` during gateway startup, and preserving manual model-list pruning would change the documented generated-registry merge contract rather than closing as already fixed.

## What This Changes

The issue requests that manually edited per-agent `models.json` model lists persist across gateway restarts instead of being regenerated from defaults or catalog state.

## Best Possible Solution

Maintainers should define whether per-agent `models.json` model arrays are user-owned overrides or generated registry state, then either support durable pruning through an explicit config flow with regression coverage or document direct edits as non-authoritative.

## Reproduction Assessment

Yes, source-reproducible. Gateway startup still reaches `ensureOpenClawModelsJson`, and the planner/merge helpers can regenerate provider model arrays when the existing file differs.

## Solution Assessment

Unclear. Preserving manual deletions may be desirable, but it is not a narrow bug fix until maintainers decide whether generated `models.json` should treat model arrays as source-authoritative.
