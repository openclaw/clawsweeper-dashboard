## Summary

Keep open. Current main still prepares and may rewrite the active agent `models.json` during gateway startup, and current docs describe generated merge precedence rather than durable manual pruning of provider model arrays, so the remaining work is a maintainer product/config-contract decision.

## What This Changes

The issue requests that per-agent `models.json` edits, especially pruned provider model arrays, persist across gateway restarts.

## Best Possible Solution

Maintainers should decide whether per-agent `models.json` is editable source of truth or generated registry state; if pruning is supported, preserve deletions with focused gateway-startup regression coverage, otherwise route durable pruning through documented `models.providers` / `models.mode` config flows.

## Reproduction Assessment

Yes, source-reproducible. Gateway startup calls `ensureOpenClawModelsJson`, the planner defaults to merge mode and writes changed generated contents, and the merge helpers do not preserve a matching provider's pruned `models` array.

## Solution Assessment

Unclear. Directly preserving deletions may be the desired behavior, but it is not the narrowest safe code-only fix until maintainers decide whether model arrays in `models.json` are user-owned configuration or refreshed catalog output.
