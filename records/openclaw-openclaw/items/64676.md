## Summary

Keep open. Current main still routes gateway startup through `models.json` preparation and merge mode can regenerate provider model arrays from config/catalog data, while preserving only selected credential and endpoint fields. The docs describe that generated merge contract, but the issue asks whether hand-pruned per-agent model lists should persist, so this needs maintainer product/API judgment rather than cleanup closure.

## What This Changes

The issue asks gateway restarts to preserve user-edited per-agent `agent/models.json` model lists instead of restoring removed model entries from defaults, runtime state, or generated catalog data.

## Best Possible Solution

Maintainers should define the ownership contract for per-agent `models.json`. If manual pruning is supported, add a durable deletion/tombstone or explicit replace mechanism with a gateway-restart regression; if it remains generated state, align docs, doctor/config help, and setup flows around `models.providers`, `agents.defaults.models`, and the intended merge/replace path.

## Reproduction Assessment

Yes. The source-level path is high confidence: restart prewarm can call `ensureOpenClawModelsJson`, the planner compares regenerated contents to the existing file, and merge helpers can append implicit/catalog models that were manually removed.

## Solution Assessment

Unclear. Preserving direct edits may be a valid user-facing fix, but current docs and code also support treating the file as regenerated catalog state; a maintainer should choose the contract before an automated code change.
