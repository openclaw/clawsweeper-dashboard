## Summary

Keep open. Current main still regenerates the active agent `models.json` on gateway startup and merge mode can restore catalog models a user removed; the remaining issue is the intended product contract for direct per-agent model-list pruning.

## What This Changes

The issue requests that direct edits to an agent's `models.json`, especially pruning provider models, survive gateway restarts instead of being regenerated from defaults or catalog state.

## Best Possible Solution

Maintainers should define whether per-agent `models.json` is editable source of truth or generated registry state, then either preserve durable pruning with regression coverage or route pruning through documented config and replace workflows.

## Reproduction Assessment

Yes, from source inspection. The restart steps map to gateway startup calling `ensureOpenClawModelsJson`, the planner rewriting changed contents, and merge mode appending unseen implicit/catalog models.

## Solution Assessment

Unclear. Preserving pruned model arrays would be a new or clarified ownership contract because current docs describe generated merge behavior and only selected fields as agent-file-wins.
