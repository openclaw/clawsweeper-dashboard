## Summary

Keep open. Current main still regenerates the active agent `models.json` during gateway startup and can restore catalog/config models that a user removed, while current docs describe that file as generated merge state rather than a durable manual pruning surface.

## What This Changes

Request that gateway restarts preserve user-pruned per-agent `models.json` model lists instead of restoring config/catalog/default models.

## Best Possible Solution

Maintainers should define the `models.json` ownership contract, then either support durable per-agent pruning with regression coverage or route persistent pruning through the documented config allowlist/replace workflow.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: startup prewarm calls `ensureOpenClawModelsJson`, the planner rewrites changed contents in merge mode, and the merge helper appends unseen implicit/catalog models.

## Solution Assessment

Unclear. Preserving pruned model arrays could be the narrow fix if that is the intended contract, but current docs describe generated merge-state behavior, so maintainers should choose the contract before implementation.
