## Summary

Keep open. Current main still regenerates the active agent `models.json` during gateway startup, and merge mode can restore catalog models that a user removed; the remaining question is the intended product contract for direct per-agent model-list pruning.

## What This Changes

The issue asks for gateway restarts to preserve user-pruned per-agent `models.json` model lists instead of regenerating default or catalog entries.

## Best Possible Solution

Maintainers should define the `models.json` ownership contract, then either support durable per-agent model pruning with regression coverage or route persistent changes through the documented config/replace workflow.

## Reproduction Assessment

Yes. The issue's restart steps map to current source: gateway startup calls `ensureOpenClawModelsJson`, the planner rewrites changed contents, and merge mode appends unseen implicit/catalog models.

## Solution Assessment

Unclear. The current implementation is coherent with the docs' generated merge-state model, but it does not satisfy the requested direct-edit persistence behavior, so maintainers need to choose the contract first.
