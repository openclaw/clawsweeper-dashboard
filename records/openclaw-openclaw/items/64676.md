## Summary

Keep open. Current main still prepares and can rewrite the active agent `models.json` during gateway startup, and merge mode still refreshes matching provider model arrays from generated config/catalog data instead of treating a manually pruned array as authoritative.

## What This Changes

Request durable persistence for direct per-agent `models.json` edits so gateway restarts do not restore generated/default model entries.

## Best Possible Solution

Maintainers should decide whether per-agent `models.json` model arrays are user-owned configuration or generated catalog state, then either preserve deletions with regression coverage or document and route durable pruning through supported config flows.

## Reproduction Assessment

Yes, source-reproducible. Gateway startup still calls the models.json preparation path, the planner still writes regenerated contents when they differ, and the merge helpers still do not preserve a matching provider's pruned `models` array.

## Solution Assessment

Unclear. Preserving manual deletions may be the desired product outcome, but it is not a narrow bug-only fix until maintainers decide whether `models.json` model arrays are source-authoritative or generated registry output.
