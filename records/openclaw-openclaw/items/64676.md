## Summary

Keep open. Current main still reaches the `models.json` regeneration path during gateway startup, and the merge path still refreshes matching provider model arrays instead of treating direct deletions as durable per-agent user intent.

## What This Changes

The issue asks for per-agent `models.json` edits, especially pruning provider model arrays, to persist across gateway restarts instead of being regenerated from default/catalog state.

## Best Possible Solution

Maintainers should define the model-array ownership contract; if pruning is supported, preserve deletions with focused gateway-startup regression coverage, otherwise route durable pruning through the documented config `--replace` path and make that guidance explicit.

## Reproduction Assessment

Yes, from source inspection. Gateway startup calls `ensureOpenClawModelsJson`, which reads existing `models.json` and rewrites changed plans; merge mode regenerates providers and does not preserve a pruned `models` array for matching provider IDs.

## Solution Assessment

Unclear. Preserving direct deletions may be the desired UX, but current docs treat other provider fields as generated catalog data and destructive config changes as an explicit `--replace` flow, so maintainers need to choose the contract first.
