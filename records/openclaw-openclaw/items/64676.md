## Summary

Keep open. Current main still prepares and can rewrite the active agent `models.json` during gateway startup, and the merge path still refreshes provider model arrays from config/catalog data while preserving only selected secret/base URL fields. The remaining question is whether user-pruned per-agent model arrays are durable user config or generated catalog state, so this needs maintainer product/API judgment rather than cleanup closure.

## What This Changes

The issue requests persistent user edits to per-agent gateway `models.json` model lists across gateway restarts.

## Best Possible Solution

Maintainers should decide the source-of-truth contract for per-agent `models.json`: preserve user-pruned arrays with focused startup regression coverage, or document and route durable pruning through supported `models.providers` / `agents.defaults.models` / `models.mode: "replace"` workflows.

## Reproduction Assessment

Yes, from source inspection. Gateway startup still calls `ensureOpenClawModelsJson`, the planner still rewrites changed generated contents in merge mode, and matching provider merges do not preserve an existing pruned `models` array.

## Solution Assessment

Unclear. Preserving direct deletions from `models.json` may be the right user-facing behavior, but current docs treat other provider fields as regenerated catalog/config data, so maintainers should decide the contract before a code-only repair.
