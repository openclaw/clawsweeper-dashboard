- `impact:auth-provider`: The affected `models.json` surface controls provider model availability and routing metadata.

## Summary

Keep open. Current main and v2026.5.18 still regenerate the active agent `models.json` during gateway startup and can restore catalog/config model entries, while current docs describe that generated merge contract rather than promising durable manual pruning.

## What This Changes

The issue requests that manual pruning of per-agent `agent/models.json` survive gateway restarts instead of being regenerated from defaults/catalog data.

## Best Possible Solution

Maintainers should codify the ownership contract for per-agent `models.json`: preserve user-pruned model arrays with regression coverage if they are user-owned, or keep the generated contract and route durable pruning through documented config flows.

## Reproduction Assessment

Yes, source-reproducible. Gateway startup still calls `ensureOpenClawModelsJson`, the planner still writes regenerated contents when they differ, and the merge helpers still do not preserve a pruned matching-provider `models` array.

## Solution Assessment

Unclear. Preserving manual deletions may be the right user outcome, but current docs describe generated/merged registry state, so the maintainable fix depends on a maintainer contract decision.
