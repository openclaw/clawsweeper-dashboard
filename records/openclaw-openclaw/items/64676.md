## Summary

Keep open. Current main still reaches `models.json` regeneration during Gateway startup, and merge mode can restore catalog models that were removed from a per-agent file; current docs describe that generated-state behavior, so durable direct pruning needs maintainer product/API judgment.

## What This Changes

The issue requests that manual edits to per-agent `agent/models.json`, especially pruning a provider to a single model, survive Gateway restarts instead of being regenerated from defaults or catalog state.

## Best Possible Solution

Maintainers should define the ownership contract for per-agent `models.json`: either support durable model-list pruning with regression coverage, or make the documented config/replace workflow the supported path for persistent model-list changes.

## Reproduction Assessment

Yes. The issue's restart steps map to current source: Gateway startup calls `ensureOpenClawModelsJson`, the planner rewrites changed generated contents, and merge mode appends unseen implicit/catalog models.

## Solution Assessment

Unclear. Preserving pruned model arrays may be useful, but current implementation and docs treat most provider fields as generated catalog state, so the maintainable fix depends on the intended contract.
