## Summary

Keep open. Current main still regenerates the active agent `models.json` during gateway startup and merge mode still treats matching provider model arrays as refreshed catalog/config output, while docs describe that generated precedence instead of promising durable manual pruning.

## What This Changes

The issue requests durable per-agent model-list edits in agent `models.json` across gateway restarts.

## Best Possible Solution

Maintainers should define the source-of-truth contract: either preserve user-pruned per-agent model arrays with focused regression coverage, or document and route durable pruning through supported `models.providers`, `agents.defaults.models`, and `models.mode` flows.

## Reproduction Assessment

Yes, source-reproducible. Gateway startup still calls the models.json preparation path, the planner still writes regenerated contents on differences, and the merge helpers do not preserve a matching provider's pruned `models` array.

## Solution Assessment

Unclear. Directly preserving deletions may be the desired outcome, but it is not the narrowest safe code-only fix until maintainers decide whether `models.json` model arrays are user-owned configuration or regenerated catalog state.
