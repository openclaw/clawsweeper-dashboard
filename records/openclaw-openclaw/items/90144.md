## Summary

Keep open: the PR targets a real subagent completion handoff failure, but it is not merge-ready because the new queue path marks the completion delivered without waking the idle requester, the external contributor has not supplied real after-fix proof, and the branch is currently conflicting.

## What This Changes

The PR adds a `durable_queue` subagent announcement fallback that enqueues pending direct completion handoffs into the requester's system-event inbox and treats that path as durable in the harness runtime.

## Best Possible Solution

Land the durable queue fallback only after it wakes or otherwise guarantees requester consumption, makes the new delivery-path contract intentional, rebases cleanly, and includes real after-fix proof.

## Reproduction Assessment

## Solution Assessment

No as submitted. The queue fallback is aimed at the right branch, but treating the queue write as durable needs a requester wake or equivalent consumption guarantee before retry is suppressed.
