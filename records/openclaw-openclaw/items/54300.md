## Summary

Keep open. Current main still has the Docker label-template and thread-ownership raw-path hardening gaps, while the TLS/private-WS and Control UI flags are documented break-glass behavior that needs maintainer security/product judgment before changing.

## What This Changes

The issue asks to harden Docker label inspection, thread-ownership forwarder URL construction, and gateway/control UI break-glass controls.

## Best Possible Solution

Keep this open for security triage: land narrow validation/encoding fixes for Docker label reads and thread-ownership URLs, then separately decide whether the documented break-glass controls need a new policy or configuration model.

## Reproduction Assessment

Yes for source-level reproduction. Static inspection shows the Docker template interpolation and raw thread-ownership URL segments, and a local URL constructor check confirms dot segments can escape the intended ownership route.

## Solution Assessment

Unclear for the grouped issue as a whole. Label validation and encodeURIComponent are narrow maintainable fixes for the first two findings, but replacing documented break-glass controls with CIDR allowlists or hiding operator guidance needs maintainer security/product approval.
