Status: needs_attention

Summary: The issue itself is security-sensitive, and all three reported hardening surfaces remain visible in current main and v2026.6.6.

Concerns:

- **[low] Docker label is interpolated into a Go template:** `src/agents/sandbox/docker.ts:239`
  - body: `readDockerContainerLabel` embeds the label into the `docker inspect -f` template without validating the helper boundary, leaving a template-breakout class of input unguarded if a caller ever accepts external labels.

  - body: The Slack thread-ownership forwarder URL inserts `channelId` and `threadTs` directly into the path, so path metacharacters are not confined to a single segment before the internal request.
  - confidence: 0.9

  - body: With `allowPrivateWs` enabled, the gateway validator accepts non-IP hostnames and the tests bless that behavior, which keeps the break-glass policy broader than private/loopback literals.
