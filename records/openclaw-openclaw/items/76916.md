## Summary

Keep open: current main still lacks a first-class Dreams Deep promotion review-queue/off policy, while the managed Deep path still appends durable promotions to `MEMORY.md`; this is a feature/config-policy request needing maintainer design alignment.

## What This Changes

Request adds a review-queue/off mode and related Dreams phase controls so Deep promotion can propose changes without autonomously editing curated memory files.

## Best Possible Solution

Add a schema-backed, documented Dreams promotion policy that can stage proposals without editing `MEMORY.md`, then align source gates, scheduling, status/UI, and docs around that contract.

## Reproduction Assessment

Yes for the source-visible gap, not as a failing bug reproduction. Current main has no review-queue/off promotion mode, and the managed Deep path still calls the writer that appends to `MEMORY.md`.

## Solution Assessment

Unclear until maintainers choose the public contract. A review-queue/off promotion policy is plausible, but schema shape, defaults, scheduling semantics, and docs need design approval first.
