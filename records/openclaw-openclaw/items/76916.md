Keep open: current main still lacks a first-class Dreams Deep promotion review-queue/off policy, and the remaining work is a feature/config-policy decision rather than conservative cleanup.

## What This Changes

Requests a Dreams Deep promotion review-queue/off mode and related scheduling, source-gating, and documentation alignment so Dreams can run without autonomous `MEMORY.md` edits.

## Best Possible Solution

Add a documented, schema-backed Dreams promotion policy that can stage proposals without editing `MEMORY.md`, then align scheduling, source gates, docs, tests, and status/UI around that contract.
