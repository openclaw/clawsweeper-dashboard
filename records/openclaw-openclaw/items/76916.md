## Summary

Current main still lacks a Deep promotion review-queue/off policy, and the remaining work changes memory-core write semantics and public config shape, so this should stay open for maintainer direction.

## What This Changes

Request a Dreams mode that can run analysis and produce reviewable promotion proposals without autonomously editing MEMORY.md.

## Best Possible Solution

Define the accepted memory-core Dreams no-autowrite contract first, then implement it in the bundled plugin with aligned SDK types, docs, and regression tests.

## Reproduction Assessment

Yes, from source inspection: current main has no promotion mode, the managed cron is derived from Deep enablement, Light ignores the parsed sources list, and Deep promotion writes MEMORY.md. I did not run a live runtime scenario because this was a read-only review.

## Solution Assessment

Unclear: a review-queue mode may be the right endpoint, but the issue asks for a new product/config contract across several Dreams surfaces. The maintainable path is to choose that policy explicitly before implementation.
