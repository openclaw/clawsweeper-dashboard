## Summary

Keep open: current main still has no first-class Deep promotion review queue and the existing Dreams path still writes qualifying promotions to `MEMORY.md`; because the fix changes public config and durable-memory policy, it needs maintainer design alignment.

## What This Changes

This issue requests a review-queue/off policy for Dreams Deep promotion, plus source gating, independent phase scheduling, and clearer write-surface documentation so Dreams can run without autonomously editing `MEMORY.md`.

## Best Possible Solution

Add a documented, schema-backed Dreams promotion policy that can stage proposals without editing `MEMORY.md`, then align source gating, scheduling, status/UI, docs, and tests around that contract.

## Reproduction Assessment

Yes. Source inspection on current main shows no review-queue/off promotion mode and shows the managed Deep path calling the writer that appends qualified promotions to `MEMORY.md`; existing tests also assert that behavior.

## Solution Assessment

Unclear until maintainers choose the public contract. A review-queue/off policy is a plausible direction, but the exact schema, defaults, source-gating semantics, and phase scheduling behavior need product design approval before a narrow implementation PR.
