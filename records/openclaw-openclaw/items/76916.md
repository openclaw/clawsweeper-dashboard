## Summary

Keep open: current main still lacks a first-class Dreams Deep promotion mode that stages review proposals or disables promotion without disabling the managed sweep, while the Deep path still appends qualified promotions to `MEMORY.md`. The docs now clarify much of the write surface, but the requested promotion-policy/config contract remains a valid product design item.

## What This Changes

Add a Dreams Deep promotion policy/review-queue mode that stages proposals instead of autonomously writing `MEMORY.md`, with related source-gating, scheduling, and write-surface alignment.

## Best Possible Solution

Add a documented, schema-backed Dreams promotion policy that can stage Deep proposals without touching `MEMORY.md`, then align source gates, managed sweep semantics, docs, status, and UI around that contract.

## Reproduction Assessment

Yes for the observable current behavior: source inspection shows no review-queue/off promotion mode, and the managed Deep path still calls the `MEMORY.md` writer. This is a feature/config-policy gap rather than a failing existing contract.

## Solution Assessment

Unclear until maintainers choose the public contract. A review-queue/off policy is plausible, but the schema, defaults, phase scheduling semantics, and docs need maintainer design approval before implementation.
