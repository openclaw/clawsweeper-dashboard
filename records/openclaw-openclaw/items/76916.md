## Summary

Keep open: current main still lacks a first-class review-queue/off policy for Dreams Deep promotion, and the runtime/docs still make Deep the path that appends durable promotions to `MEMORY.md`. This is a valid feature/config-policy request that needs maintainer design alignment rather than cleanup closure.

## What This Changes

Add a Dreams Deep promotion review-queue/off mode, align Light source gates and independent phase scheduling, and document each phase's write surface.

## Best Possible Solution

Add a documented, schema-backed Dreams promotion policy that can stage proposals without editing `MEMORY.md`, then align phase source gates, scheduling, docs, and status/UI around that contract.

## Reproduction Assessment

Yes. Source inspection on current main shows no review-queue/off promotion mode, and the managed Deep path still calls the writer that appends promotion sections to `MEMORY.md`; the related Light `sources` concern is also visible in source.

## Solution Assessment

Unclear until maintainers choose the public contract. A review-queue/off promotion policy is a plausible direction, but the exact schema, defaults, scheduling semantics, and docs need design approval before a narrow PR.
