## Summary

Keep open: current main still lacks a first-class review-queue/off policy for Dreams Deep promotion, while the documented and tested Deep path still promotes durable candidates into `MEMORY.md`. This is a valid feature/config-policy request that needs maintainer design alignment rather than cleanup closure.

## What This Changes

The issue requests a Dreams Deep promotion mode that stages reviewable proposals instead of autonomously editing `MEMORY.md`, plus related Light source gating, independent phase scheduling, and write-surface documentation.

## Best Possible Solution

Add a documented, schema-backed Dreams promotion policy that can stage Deep proposals without editing `MEMORY.md`, then align source gates, scheduling semantics, tests, status/UI, and docs around that contract.

## Reproduction Assessment

Yes. Source inspection on current main shows no review-queue/off promotion mode, the managed Deep trigger still writes promotions into `MEMORY.md`, and `runLightDreaming` still does not gate daily/session ingestion by configured sources.

## Solution Assessment

Unclear until maintainers choose the public contract. A review-queue/off promotion policy is a plausible direction, but schema, defaults, scheduling behavior, and migration/docs expectations need design approval before implementation.
