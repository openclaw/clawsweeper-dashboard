## Summary

Keep open: current main still lacks a review-queue/off policy for Dreams Deep promotion, and the managed Deep path continues to promote directly into `MEMORY.md`; this is a valid feature/config-policy request that needs maintainer design alignment.

## What This Changes

Request a Dreams review-queue/off promotion mode, source-gating fixes, scheduling alignment, and write-surface docs so Dreams can run without autonomous `MEMORY.md` edits.

## Best Possible Solution

Add a documented, schema-backed Dreams promotion policy in memory-core that can stage Deep proposals without editing `MEMORY.md`, then align source gating, scheduling, docs, tests, and status/UI around that contract.

## Reproduction Assessment

Yes, source-reproducible. Current main has no review-queue/off promotion config, while the managed Deep path calls `applyShortTermPromotions` and the writer appends to workspace `MEMORY.md`.

## Solution Assessment

Unclear until maintainers choose the public contract. A review-queue/off promotion policy is a plausible direction, but schema, defaults, phase scheduling, source semantics, and docs/UI exposure need owner sign-off before implementation.
