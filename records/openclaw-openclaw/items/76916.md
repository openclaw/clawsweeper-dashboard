## Summary

Keep open: current main still lacks a first-class review-queue/off promotion policy, while the managed Dreams sweep derives scheduling from Deep and appends accepted promotions to `MEMORY.md`. This is a valid feature/config-policy request that needs maintainer design alignment.

## What This Changes

Request a Dreams Deep promotion policy that can stage or skip proposed durable-memory updates, plus source-gated Light ingestion, independent phase scheduling, and clearer write-surface docs.

## Best Possible Solution

Add a documented, schema-backed Dreams promotion policy that can stage proposals without editing `MEMORY.md`, then align source gates, managed scheduling, docs, status/UI, and regression coverage around that contract.

## Reproduction Assessment

Yes. Source inspection on current main shows no review-queue/off promotion policy, and the managed Deep path still calls the writer that appends accepted promotions to `MEMORY.md`; the Light `sources` concern is also visible in source.

## Solution Assessment

Unclear until maintainers choose the public contract. A review-queue/off policy is plausible, but schema names, defaults, scheduling semantics, UI/status behavior, and docs need design approval before a narrow PR.
