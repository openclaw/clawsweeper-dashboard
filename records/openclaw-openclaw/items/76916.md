## Summary

Keep open: current main still has Deep promotion append durable candidates to `MEMORY.md` and exposes no review-queue/off promotion policy. The docs now clarify the write surface, but they also confirm phase policy remains internal rather than a user-facing control.

## What This Changes

Request adding a Dreams Deep promotion review-queue/off mode, source gating, independent phase scheduling, and docs so Dreams can run without autonomous `MEMORY.md` rewrites.

## Best Possible Solution

Add a documented, schema-backed Dreams promotion policy that can stage Deep proposals without editing `MEMORY.md`, then align source gates, scheduling semantics, status/UI, tests, and docs around that contract.

## Reproduction Assessment

Yes. Source inspection on current main shows a managed dreaming event ranks candidates and calls `applyShortTermPromotions`, whose writer appends to workspace `MEMORY.md`, while the public config/schema has no review-queue or off promotion mode.

## Solution Assessment

Unclear until maintainers choose the public contract. A review-queue/off promotion policy is a plausible direction, but the schema, defaults, light-only scheduling behavior, source gates, and docs need design approval first.
