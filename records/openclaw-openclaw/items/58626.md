## Summary

Keep open. Current main handles current-package bundled path aliases and the older `extensions/<id>` layout, but it still does not recognize a removed old-version pnpm/npm package path like `openclaw@2026.3.28.../dist/extensions/acpx`, so the reported gateway-start failure remains source-reproducible.

## What This Changes

The issue asks OpenClaw to recover stale versioned absolute bundled plugin paths after package-manager upgrades so gateway startup and `doctor --fix` do not leave config pointing at removed package directories.

## Best Possible Solution

Teach `doctor --fix` to identify missing old-version OpenClaw package bundled plugin paths by package layout and plugin id, then remove or rewrite only proven redundant bundled paths while preserving strict errors for custom paths.

## Reproduction Assessment

Yes, by source inspection. A removed old-version bundled path in `plugins.load.paths` still reaches `discoverFromPath()` and becomes a config error, while the current doctor matcher only recognizes aliases derived from the active package root.

## Solution Assessment

Yes. The narrow maintainable fix is to extend the existing doctor bundled-load-path repair for old versioned package roots, not to weaken all missing-path validation or mutate config during gateway startup.
