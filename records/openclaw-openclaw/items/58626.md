## Summary

Keep open. Current `main` has partial bundled plugin path cleanup, but it still does not cover the reported upgrade case where `plugins.load.paths` points at a missing `openclaw@<old-version>` package directory; that path remains a config validation error before stale plugin cleanup can proceed.

## What This Changes

The issue requests automatic or `doctor --fix` repair for versioned absolute bundled plugin paths left in user config after OpenClaw package-manager upgrades.

## Best Possible Solution

Extend `doctor --fix` with a conservative bundled-plugin repair that recognizes missing old-version OpenClaw package paths by package layout plus plugin id, removes the redundant bundled load-path entry when the current bundled plugin exists, and keeps strict errors for non-bundled or ambiguous custom paths.

## Reproduction Assessment

Yes. A config entry like the reported old pnpm global `openclaw@2026.3.28.../node_modules/openclaw/dist/extensions/acpx` path no longer exists, so current source still emits `plugin path not found` and maps it to `plugins.load.paths`; the current doctor alias map does not include other package-version roots.

## Solution Assessment

No, current main is not the best solution for this report. The narrow maintainable fix is in the doctor bundled load-path repair path, not startup-time config mutation or weakening all missing-path validation.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/58626. Current `doctor --fix` only removes bundled plugin load-path aliases derived from the current package root; it does not repair missing `plugins.load.paths` entries that point at a previous versioned global package root such as `.../.pnpm/openclaw@2026.3.28.../node_modules/openclaw/dist/extensions/acpx`. Add a conservative doctor repair that recognizes missing old-version OpenClaw bundled plugin paths by package layout plus plugin id, removes the redundant `plugins.load.paths` entry when the same bundled plugin is available in the current install, and preserves strict errors for non-bundled or ambiguous custom paths. Keep the fix in doctor/plugin cleanup rather than adding startup-time config mutation. Do not weaken missing-path validation for arbitrary third-party plugins. Add focused regression coverage for pnpm/npm old-version package roots and for non-bundled missing paths that must remain errors. Add a user-facing changelog entry with appropriate non-bot reporter credit.
