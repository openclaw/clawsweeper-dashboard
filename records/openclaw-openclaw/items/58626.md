## Summary

Keep open. The earlier ClawSweeper review still matches current `main` and the latest shipped tag: OpenClaw repairs current-package bundled path aliases and the old same-package `extensions/<id>` layout, but not a removed old-version package root such as `openclaw@2026.3.28.../dist/extensions/acpx`.

## What This Changes

The issue asks OpenClaw to recover stale versioned bundled plugin paths in config after npm/pnpm global upgrades so gateway startup does not fail on `plugins.load.paths`.

## Best Possible Solution

Extend the existing doctor bundled-load-path repair to detect removed old-version OpenClaw package roots by layout and plugin id, while keeping gateway startup validation strict for custom missing paths.

## Reproduction Assessment

Yes, by source inspection. A removed old-version bundled path in `plugins.load.paths` is not recognized by the current-package alias scanners, falls through to `discoverFromPath()`, and is surfaced as a `plugins.load.paths` config error.

## Solution Assessment

Yes. The narrow maintainable fix is to extend the existing doctor repair for versioned bundled package roots, not to weaken all missing-path validation or mutate config during gateway startup.
