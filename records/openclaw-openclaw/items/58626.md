## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/58626. `openclaw gateway start` can fail after npm/pnpm upgrades when `plugins.load.paths` still contains a stale absolute bundled plugin path under a removed old OpenClaw package root, for example an old `.pnpm/openclaw@.../node_modules/openclaw/dist/extensions/acpx` path. Extend the existing doctor bundled-plugin load-path repair and preview warning to detect old-version OpenClaw bundled package roots by layout and plugin id, then remove or rewrite only those redundant bundled entries to the current packaged bundled source. Keep gateway startup validation strict and do not remove arbitrary missing custom or third-party plugin paths. Preserve existing same-package legacy/current-path behavior and add regression coverage for the old-version package-root case. Add a user-facing changelog entry unless maintainer policy says landing will handle it.
