## Summary

Keep open. Current main still has separate status, sessions, model-auth, channels, logs, and service diagnostics, but no unified `openclaw fleet status` command or equivalent fleet JSON payload covering agents, auth health, channel health, stuck sessions, and recent errors.

## What This Changes

Add an `openclaw fleet status` command, including JSON output, that summarizes multi-agent health, provider/auth state, channel status, stuck sessions, and recent errors.

## Best Possible Solution

Define a supported read-only fleet observability schema that composes the existing status, sessions, model-auth, channel, service, and log collectors with explicit redaction guarantees and focused text/JSON tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for feature absence: source, docs, and tests show separate observability surfaces, and no `openclaw fleet status` command or equivalent fleet JSON payload exists on current main.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the command/API contract; a top-level fleet command is plausible, but the safest implementation should first settle schema boundaries, collector reuse, and sensitive-output redaction.
