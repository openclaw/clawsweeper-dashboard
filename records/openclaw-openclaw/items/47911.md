## Summary

Keep open. Current main still has separate observability surfaces for status JSON, per-agent sessions, model auth, channel probes, gateway logs, services, and agent summaries, but it does not provide the requested unified fleet command or an equivalent single JSON payload for fleet-wide auth, channel, service, session, stuck-run, and recent-error state. Related #50066 and #50072 are useful context, but both are closed and neither shipped the requested core surface.

## What This Changes

Request a new read-only `openclaw fleet status` command, including `--json`, that summarizes multi-agent health, provider auth, channel health, stuck sessions, service state, and recent errors.

## Best Possible Solution

Define one supported read-only fleet observability schema that reuses the existing status, sessions, models/auth, channels, logs, and service collectors with explicit redaction guarantees and tests. The final CLI surface can be a new `fleet status` namespace or an expanded `status --all --json`, but it should avoid duplicated collectors and document the stable JSON contract.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing behavior is verifiable with high confidence by inspecting the current CLI catalogs, status schema, docs, and split command surfaces, none of which provide the requested unified fleet command or payload.

## Solution Assessment

Unclear until maintainers choose the product/API surface. The request is coherent, but the maintainable solution needs a short design pass around command naming, stable JSON fields, reuse of existing collectors, and sensitive-output redaction.
