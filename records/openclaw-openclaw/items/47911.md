## Summary

Keep open. Current main has separate status, sessions, model-auth, channel-status, and log-tail diagnostics, but it still lacks a unified `openclaw fleet status` command or equivalent fleet JSON payload covering agents, auth, channel health, stuck-session state, and recent-error aggregation.

## What This Changes

Request a new `openclaw fleet status` command with human and JSON output aggregating multi-agent health, sessions, auth profiles, channel health, stuck work, and recent errors.

## Best Possible Solution

Define a supported read-only fleet observability surface that reuses existing status, session, model-auth, channel, and log collectors with documented redaction and stable JSON fields.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction, but yes for feature absence: the current command catalog, CLI docs, status schema, and separate sessions/models/channels surfaces show no unified fleet command or equivalent payload.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the product/API surface; the requested direction is coherent, but the maintainable fix should first settle command naming, schema boundaries, collector reuse, and redaction policy.
