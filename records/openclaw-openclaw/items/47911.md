## Summary

Close as ClawHub/plugin scope: the request is a useful multi-agent observability feature, but current OpenClaw policy keeps optional fleet dashboards and orchestration layers out of core while the plugin SDK already provides CLI, hook, gateway, and service extension points to build it externally.

## What This Changes

The issue requests a new `openclaw fleet status` command, including JSON output, that aggregates multi-agent health, auth, channel, session, and recent-error status.

## Best Possible Solution

Keep core focused on reusable status/session/channel/auth primitives and implement fleet observability as a ClawHub plugin or external dashboard; open a narrow SDK issue only if a specific missing field blocks that plugin.

## Reproduction Assessment

Not applicable; this is a feature request, not a bug report. Source and docs search confirms current main lacks an `openclaw fleet status` command while exposing lower-level status primitives that can be aggregated externally.

## Solution Assessment
