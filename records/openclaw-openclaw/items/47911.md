## Summary

Keep open. Current main still has separate status, sessions, model-auth, channel-status, and logs surfaces, but no unified `openclaw fleet status` command or equivalent JSON payload that aggregates fleet-wide agent health, auth/profile state, stuck-session classification, channel health, and recent errors. Related observability items cover signal/logging slices or plugin-shaped tracing, but they do not supersede this core CLI/status product question.

## What This Changes

The issue requests a unified `openclaw fleet status` command with human and JSON output for multi-agent health, auth, channel, session, and recent-error visibility.

## Best Possible Solution

Define one supported read-only fleet observability schema that reuses existing status, sessions, models/auth, channels, logs, and service collectors with clear redaction guarantees and documented JSON fields.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction, but yes for feature absence: current-main source and docs show separate observability surfaces and no unified fleet command or equivalent payload.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the product/API boundary; a maintainable solution should settle command naming, JSON schema stability, collector reuse, and redaction policy before implementation.
