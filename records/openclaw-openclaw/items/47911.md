## Summary

Keep open. Current main still lacks a unified `openclaw fleet status` command or equivalent JSON surface spanning agents, model auth, channel health, stuck-session state, and recent-error aggregation; the request is coherent but needs maintainer product and redaction design before implementation.

## What This Changes

Request a new `openclaw fleet status` command, including `--json`, that aggregates multi-agent health, auth, channel, stuck-session, and recent-error state.

## Best Possible Solution

Define a supported read-only fleet observability schema that reuses existing status, sessions, models/auth, channels, service, and log collectors with explicit redaction guarantees and tests for human and JSON output.
