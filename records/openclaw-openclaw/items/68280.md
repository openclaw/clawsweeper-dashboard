## Summary

Keep open: this is a plausible bug-fix PR for gateway probe/status auth, but the current head still misses inferred local SecretRef auth when `gateway.auth.mode` is unset, and the explicit URL credential-forwarding path needs security/maintainer review before merge.

## What This Changes

The PR propagates gateway probe auth `failureReason` through daemon, status, and gateway-status probe paths and adds regression tests so unresolved required local auth can fail locally instead of attempting anonymous loopback probes.

## Best Possible Solution

Merge only after fail-fast decisions use the effective credential/auth-surface contract, URL override credential forwarding matches the documented no-fallback policy, inferred SecretRef cases have regression coverage, and the contributor adds redacted real CLI/gateway proof.

## Reproduction Assessment

Yes, source-reproducible. With unset `gateway.auth.mode` and an unresolved configured local `gateway.auth.token` SecretRef, the PR head returns before producing `failureReason`, so the anonymous loopback probe path can still run.

## Solution Assessment

No, not yet. The narrow maintainable fix should reuse the effective credential plan or interactive auth-surface result, and explicit override targets should not inherit implicit local credentials unless maintainers explicitly approve that policy.
