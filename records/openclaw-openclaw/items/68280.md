## Summary

Keep this PR open: it addresses a real gateway probe-auth diagnostic problem and now includes usable live CLI proof, but the latest head still violates the explicit URL credential contract and can fail or forward stored credentials on override paths.

## What This Changes

The PR adds gateway probe auth failure reasons, short-circuits selected status/probe paths when local auth cannot be resolved, and adds focused regression coverage for SecretRef-backed gateway auth.

## Best Possible Solution

Keep fail-fast for true local shared-secret probe paths, but make explicit URL overrides use only explicit CLI credentials or no credentials and derive local fail-fast from the effective gateway auth contract.

## Reproduction Assessment

Yes for the review blockers: source inspection of the PR head shows explicit targets still resolve local auth, daemon RPC URL overrides still receive daemon auth, and unset auth mode bypasses fail-fast despite current main defaulting it to token. I did not run tests because this review is read-only.

## Solution Assessment

No. The PR addresses the right local diagnostic class, but the implementation needs to preserve the documented explicit URL credential contract and use effective shared-secret auth semantics before it is the best fix.
