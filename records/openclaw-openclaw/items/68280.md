## Summary

Keep open. The PR is a plausible fix for the local probe-auth failure class, and related closed reports explicitly point to it, but the current head is not merge-ready because the fail-fast guard still keys off raw `gateway.auth.mode`, real behavior proof is missing, and explicit URL credential forwarding needs policy/security review.

## What This Changes

The PR adds `failureReason` propagation through gateway probe/status paths so unresolved required local auth can return a local failure instead of attempting an anonymous loopback WebSocket probe.

## Best Possible Solution

Merge only after fail-fast decisions use the effective credential plan, explicit URL override credential handling matches the documented contract, regression tests cover inferred SecretRefs, and the contributor adds redacted real CLI/gateway proof.

## Reproduction Assessment

Yes, source-reproducible. With unset `gateway.auth.mode` and an unresolved configured local `gateway.auth.token` SecretRef, the PR head returns `undefined` from the fail-fast path before producing `failureReason`, so the anonymous probe path can still run.

## Solution Assessment

No, not merge-ready. The narrow fix should reuse the effective credential/auth-surface contract instead of raw `gateway.auth.mode`, and explicit override credential forwarding should be resolved before changing these probe paths.
