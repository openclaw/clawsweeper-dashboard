## Summary

Keep open. The final PR head still misses the inferred local-auth SecretRef case, lacks the required changelog entry for a user-facing CLI fix, and leaves explicit URL/RPC credential-forwarding policy unresolved.

## What This Changes

The PR adds local gateway probe/status `failureReason` propagation so unresolved required auth can return a synthetic local failure instead of attempting an anonymous loopback probe.

## Best Possible Solution

Merge only after local fail-fast uses the effective credential plan, explicit override credential behavior matches the documented contract, regression tests cover inferred SecretRefs, the changelog is updated, and exact-head checks complete.

## Reproduction Assessment

Yes, source-reproducible. With unset `gateway.auth.mode`, a configured unresolved local `gateway.auth.token` SecretRef can still be the effective local auth path, but the PR head returns before producing `failureReason`; I did not run tests because this review was read-only.

## Solution Assessment

No, not merge-ready. The fail-fast direction is narrow, but the implementation should use effective credential planning instead of raw `gateway.auth.mode`, preserve the explicit URL credential contract, and include the required changelog entry.
