## Summary

Keep open: the local SecretRef fail-fast direction is useful and proof-backed, but latest head still leaves a daemon `--url` credential-boundary bug where stored local gateway auth can be forwarded to an override target.

## What This Changes

The PR adds failureReason-based fail-fast handling for unresolved local gateway probe auth across daemon/status probe paths and regression coverage, while tightening explicit gateway status URL auth resolution.

## Best Possible Solution

Preserve local-loopback unresolved-auth fail-fast, but make every explicit URL override use only CLI-supplied credentials or no auth, with daemon-status regression coverage for resolved and unresolved local credentials.
