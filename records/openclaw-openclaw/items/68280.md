## Summary

Keep open. Current main still probes after unresolved local auth, and this PR is close to the right behavior, but it is not merge-ready because the new token-mode helper test is env-dependent, the user-facing CLI fix lacks a changelog entry, and explicit URL credential forwarding still needs maintainer/security policy confirmation.

## What This Changes

The PR propagates local gateway probe auth failure reasons, turns unresolved required local credentials into failed local status/probe results, preserves explicit override probing, and adds focused gateway/status regression tests.

## Best Possible Solution

Ship the local-loopback fail-fast behavior after the credential-forwarding policy is settled, with env-isolated regression coverage and a single Unreleased changelog bullet for the CLI behavior change.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main resolves auth warnings then still calls probeGateway, while the PR head converts unresolved required local auth into a failed local probe result. The introduced test flake is also reproducible by running the new helper test with OPENCLAW_GATEWAY_TOKEN set.

## Solution Assessment

No, not as written. The fail-fast design is narrow and maintainable, but the PR should first isolate the ambient env test, add the required changelog entry, and get maintainer/security agreement on explicit URL credential forwarding.
