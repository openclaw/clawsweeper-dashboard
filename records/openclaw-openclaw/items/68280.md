## Summary

Keep open. Current main still lacks the fail-fast local probe auth behavior, and the PR is a plausible fix path, but the latest head still needs maintainer/security handling before merge because explicit URL override credential forwarding conflicts with the documented `--url` auth contract, CI has a failing `check` run, and the branch has a concrete env-dependent test flake plus a missing user-facing changelog entry.

## What This Changes

The PR adds `failureReason` propagation to local gateway probe auth resolution, short-circuits local status/probe calls when required local credentials are unresolved, and adds focused gateway/status regression tests.

## Best Possible Solution

Keep the fail-fast `failureReason` design if maintainers agree with the local-loopback boundary, but settle the explicit URL credential policy first, isolate the new tests from ambient gateway auth env, add a changelog entry, and rerun the focused gateway/status tests plus the changed gate before merge.

## Reproduction Assessment

Yes. The product issue is reproducible by source path: current main resolves local auth warnings but still calls `probeGateway`, while the PR head turns unresolved required local auth into a synthetic failed probe result. The introduced test flake is also reproducible by running the new helper test with `OPENCLAW_GATEWAY_TOKEN` set, which makes `failureReason` undefined.

## Solution Assessment

Unclear as the final merge shape. The local-loopback fail-fast contract is narrow and maintainable, but the PR should not merge until the explicit URL credential-forwarding policy is approved or changed, and the test/changelog blockers are fixed.
