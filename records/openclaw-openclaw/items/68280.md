## Summary

## What This Changes

The PR threads gateway probe auth failure reasons through daemon, status, and gateway-status paths so unresolved required local auth can return a structured local failure instead of making anonymous loopback probes, with focused regression tests.

## Best Possible Solution

Update the PR so fail-fast decisions use the effective credential plan or interactive auth result, align explicit URL overrides with the documented no-fallback credential contract, rebase, and add redacted real CLI/gateway proof.

## Reproduction Assessment

Yes, source-reproducible: with unset gateway.auth.mode and an unresolved configured local gateway.auth.token SecretRef, the PR head returns before producing failureReason, so the anonymous loopback probe can still run.

## Solution Assessment

No, not yet. The narrow maintainable fix should derive local fail-fast from effective credential resolution rather than raw authMode and should keep explicit URL overrides from inheriting implicit local credentials.
