## Summary

Keep open: the PR is aimed at a real gateway probe/auth bug, but the latest diff still conflicts with the documented explicit-URL credential contract and lacks real failure-path proof.

## What This Changes

The PR threads a gateway probe auth `failureReason` through status and gateway-status paths so unresolved local credentials can fail before a WebSocket probe.

## Best Possible Solution

Keep the local fail-fast behavior, but base it on the effective shared-secret credential plan and ensure explicit URL overrides use only explicit credentials or a documented missing-explicit-credentials error.

## Reproduction Assessment

Yes. Source inspection of the latest diff shows explicit URL targets still resolve local auth, daemon RPC URL overrides still receive resolved daemon auth, and raw unset auth mode still bypasses fail-fast before the effective configured auth surface is considered.

## Solution Assessment

No. The PR has the right general direction, but the narrow maintainable fix needs a single credential contract that distinguishes local targets from explicit overrides and derives fail-fast from effective auth rather than raw mode alone.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Keep explicit gateway-status URLs off local auth:** `src/commands/gateway-status/helpers.ts:177-179`
  - body: `--url` is documented as never reusing config/env credentials, but this still maps `kind: "explicit"` to the local auth surface. If `gateway.auth.token`, `gateway.auth.password`, or the matching env var resolves, the probe can attach local credentials to a user-supplied URL; explicit targets should use only `--token`/`--password` or fail with the documented missing-explicit-credentials diagnostic.
  - confidence: 0.94
- **[P1] Do not forward daemon auth to RPC URL overrides:** `src/cli/daemon-cli/status.gather.ts:602-604`
  - body: The override branch skips the local failure reason, but the probe call still passes `daemonProbeAuth` to `gateway.probeUrl`. When `--rpc-url` points outside the local daemon and local config/env auth resolves, `openclaw daemon status` can send the operator's local gateway token or password to that explicit endpoint.
  - confidence: 0.92
- **[P2] Derive local fail-fast from effective auth:** `src/gateway/probe-auth.ts:178-181`
  - body: Returning early whenever `gateway.auth.mode` is unset misses configs where `gateway.auth.token` or `gateway.auth.password` is configured, including unresolved SecretRefs. Current docs and resolver behavior treat those as active local auth inputs, so this path can still fall through to an anonymous probe instead of the fail-closed SecretRef diagnostic.
  - confidence: 0.86
