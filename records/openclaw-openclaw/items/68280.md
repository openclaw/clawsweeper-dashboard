## Summary

Keep open. Current main still lacks the local-loopback auth fail-fast behavior, and the PR is useful, but the latest head is not merge-ready because one new regression test is ambient-env dependent, the user-facing CLI behavior has no changelog entry, and explicit URL credential forwarding still needs maintainer/security policy sign-off.

## What This Changes

The PR adds `failureReason` propagation to gateway probe auth resolution, short-circuits local status/probe paths when required local loopback credentials are unresolved, and adds focused gateway/status regression tests.

## Best Possible Solution

Merge a version that fail-fasts only known local-loopback credential-required probes, does not send stored local credentials to untrusted explicit URL overrides without an approved policy, isolates the new tests from ambient env, and records the user-facing CLI fix in the changelog.

## Reproduction Assessment

Yes. The product behavior is reproducible by source path: current main resolves probe auth warnings and still probes, while the PR head returns a synthetic failed probe when `failureReason` is set. The introduced test flake is reproducible by running the new helper test with `OPENCLAW_GATEWAY_TOKEN` set.

## Solution Assessment

Unclear as the final merge shape. The local-loopback `failureReason` design is narrow and maintainable, but the explicit URL credential policy must be settled before this is the best fix; the env isolation and changelog gaps are mechanical.
