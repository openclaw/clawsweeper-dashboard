## Summary

Keep open: the current PR head still couples explicit URL override probes to stored local gateway auth and lacks real failure-path proof, so it needs credential-contract/security fixes before merge.

## What This Changes

The PR adds gateway probe auth failure reasons through probe/status/daemon status paths, short-circuits selected probes when auth cannot be resolved, and adds focused regression tests.

## Best Possible Solution

Keep fail-fast only for true local/effective shared-secret auth, make explicit URL overrides use explicit credentials or no stored credentials, and add redacted real CLI/log proof for the unresolved SecretRef failure path.

## Reproduction Assessment

Yes. Source inspection of the PR head shows explicit targets still resolve local auth, daemon RPC URL overrides still receive daemon auth, and unset auth mode bypasses fail-fast despite current config defaulting it to token.

## Solution Assessment

No. The PR addresses the right failure class, but the maintainable fix needs one credential contract that separates true local targets from explicit overrides and derives fail-fast from effective shared-secret auth semantics.
