## Summary

Keep open: the PR targets a real gateway auth diagnostic gap, but the latest diff still conflicts with the documented URL override credential contract and lacks live proof of the changed unresolved-SecretRef fail-fast path.

## What This Changes

The PR changes gateway probe/status auth resolution so unresolved local SecretRef credentials can return a structured fail-fast probe result instead of attempting an anonymous WebSocket connect.

## Best Possible Solution

Keep the local loopback fail-fast for effective shared-secret auth, but make explicit URL overrides use only explicit credentials and add tests plus redacted real failure-path proof for unresolved SecretRefs.

## Reproduction Assessment

Yes for the patch defects: source inspection of PR head shows explicit URL targets still resolve local auth and daemon URL overrides still forward daemon auth. The original unresolved-SecretRef failure path still lacks live reproduction proof in the PR body.

## Solution Assessment

No. The local fail-fast direction is useful, but the safest solution should reuse the existing URL override credential contract and derive fail-fast from effective shared-secret auth rather than raw mode checks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep explicit URLs out of local credential resolution:** `src/commands/gateway-status/helpers.ts:214-220`
  - body: `docs/gateway/remote.md` says CLI `--url` overrides never reuse implicit config/env credentials and require explicit `--token` or `--password`. This branch still resolves `kind: "explicit"` through the local auth surface, then fails `--url wss://prod.example/ws` on a local SecretRef diagnostic when no token resolves; when the SecretRef does resolve, the same path can send local credentials to the override target. Use only explicit credentials for explicit URLs, or return a missing-explicit-credentials error.
  - confidence: 0.95
- **[P1] Do not forward daemon auth to URL overrides:** `src/cli/daemon-cli/status.gather.ts:599-605`
  - body: When `probeUrlOverride` is set, this skips the local fail-fast but still passes `daemonProbeAuth?.token/password` into `probeGatewayStatus`. That helper forwards the values to `probeGateway` and `callGateway`, so a user-supplied override URL can receive daemon config/env credentials even though URL overrides are documented not to reuse implicit credentials.
  - confidence: 0.89
- **[P2] Derive fail-fast from effective auth:** `src/gateway/probe-auth.ts:183-185`
  - body: This guard treats an unset `gateway.auth.mode` as open, but `resolveGatewayAuth()` defaults unset mode to token and the credential planner allows unset-mode token SecretRefs to win. A config with `gateway.auth.token` as an unresolved SecretRef and no explicit mode can still fall through to an anonymous probe instead of failing fast.
  - confidence: 0.82
