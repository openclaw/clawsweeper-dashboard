## Summary

Keep this PR open; it is not merge-ready. The fail-fast direction is useful, but the latest head still violates the explicit URL credential contract in two paths and the supplied proof does not show the unresolved-SecretRef failure path that the PR changes.

## What This Changes

The PR threads gateway probe auth failure reasons through status, gateway-status, and daemon status paths so unresolved local probe credentials can stop before a WebSocket probe.

## Best Possible Solution

Keep the local/effective shared-secret fail-fast behavior, but make explicit URL overrides use only explicit credentials or no credentials and derive local fail-fast from the effective gateway auth contract.

## Reproduction Assessment

Yes. Source inspection of the PR head shows explicit URL targets still resolve local auth, daemon RPC URL overrides still receive resolved daemon auth, and unset auth mode is treated as open despite current config/auth code defaulting it to token.

## Solution Assessment

No. The PR addresses the right class of failure, but the best fix needs a single credential contract that separates local targets from explicit overrides and uses effective auth semantics instead of raw mode checks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep explicit gateway-status URLs off local auth:** `src/commands/gateway-status/helpers.ts:176-180`
  - body: `--url` is documented as not falling back to stored credentials, but this still classifies `kind: "explicit"` as the local auth surface. A user-supplied URL can receive `gateway.auth.*`/`OPENCLAW_GATEWAY_*` credentials, and the new SecretRef fail-fast can also block an explicit target because unrelated local config is broken.
  - confidence: 0.93
- **[P1] Do not forward daemon auth to RPC URL overrides:** `src/cli/daemon-cli/status.gather.ts:601-605`
  - body: When `--url` overrides the daemon status probe target, the code skips the local fail-fast but still forwards `daemonProbeAuth` into the overridden URL. That lets an arbitrary or copied override endpoint receive locally resolved daemon gateway credentials instead of requiring explicit `--token`/`--password`.
  - confidence: 0.9
- **[P2] Derive local fail-fast from effective auth:** `src/gateway/probe-auth.ts:183-185`
  - body: The new guard treats an unset `gateway.auth.mode` as an open gateway, but the current config contract defaults unset mode to token. A config with an unset mode and an unresolved `gateway.auth.token` SecretRef can still fall through to probing without the configured credential.
  - confidence: 0.84
