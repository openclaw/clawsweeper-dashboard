## Summary

Keep open: the local SecretRef fail-fast behavior is useful and the PR now has adequate CLI proof, but the latest patch still couples explicit URL overrides to local auth state and can forward stored credentials to user-supplied targets, which conflicts with the documented override contract and prior review concerns.

## What This Changes

The PR adds gateway probe/status failureReason plumbing so local auth-required probes short-circuit when required gateway credentials cannot be resolved, with focused regression tests and CLI proof.

## Best Possible Solution

Keep the local loopback fail-fast, but make explicit gateway and daemon URL overrides use only explicitly supplied credentials or no credentials, with tests covering unresolved and resolved local SecretRefs for both default local and override paths.

## Reproduction Assessment

Yes. Source inspection of the PR merge commit shows an explicit target with unresolved local SecretRef diagnostics returns `failureReason`, and the probe runner then skips the network probe entirely.

## Solution Assessment

No. The local loopback fail-fast is the right direction, but applying local auth state to explicit override URLs is not the narrowest maintainable solution and conflicts with the documented override contract.

Full review comments:

- **[P1] Keep explicit URL overrides off local SecretRefs:** `src/commands/gateway-status/helpers.ts:213-220`
  - body: This branch makes an explicit `--url` target fail before probing whenever the local `gateway.auth.*` SecretRef is unresolved. That contradicts the documented contract that explicit URL calls do not fall back to stored credentials and breaks override workflows where the target is unrelated to the local gateway config; explicit targets should use only CLI-provided auth or no auth.
  - confidence: 0.9
- **[P1] Stop forwarding daemon auth to URL overrides:** `src/cli/daemon-cli/status.gather.ts:605-607`
  - body: When `--url` overrides the daemon probe target, the code skips `rpcFailureReason` but still forwards `daemonProbeAuth` resolved from local config. A copied status command can therefore send the operator's stored gateway token/password to an arbitrary URL; override probes should only send credentials that were explicitly supplied on the command line.
  - confidence: 0.86
