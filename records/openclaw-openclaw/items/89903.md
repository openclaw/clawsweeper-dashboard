## Summary

Keep open. The opt-in Slack pre-router idea is understandable, but the current patch is not merge-ready because a hook hit returns before the existing Slack inbound-turn pipeline handles session state, bot-loop protection, status/ack cleanup, and delivery metadata; it also lacks real Slack behavior proof for a new raw-message HTTP egress path.

## What This Changes

Adds `OPENCLAW_PRE_ROUTER_URL` and timeout env handling for a Slack HTTP pre-router hook, with unit and local HTTP fixture tests, so matched responses can be posted before LLM dispatch.

## Best Possible Solution

If maintainers want this feature, keep it opt-in but implement matcher hits through the existing Slack inbound-turn and delivery pipeline, document the trust boundary and env/config behavior, and add real Slack proof plus dispatch-level hit/miss/error coverage.
