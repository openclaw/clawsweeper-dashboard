## What This Changes

Adds a Slack monitor pre-router that reads `OPENCLAW_PRE_ROUTER_URL` and `OPENCLAW_PRE_ROUTER_TIMEOUT_MS`, POSTs Slack prompt context to that URL, and posts matched responses directly to Slack before normal LLM dispatch, with unit and local HTTP fixture tests.

## Best Possible Solution

If maintainers approve this feature, keep it opt-in but route hits through existing Slack turn and delivery invariants, document the trust/env behavior, and require real Slack hit/miss proof before merge.

## Reproduction Assessment

Yes, from source inspection: at PR head, a pre-router hit returns from `dispatchPreparedSlackMessage` immediately after `sendMessageSlack`, before current main's shared turn/session/delivery path runs.

## Solution Assessment

No. The current implementation is a plausible shortcut, but the best maintainable path is to integrate the hook with existing Slack turn/delivery helpers or first settle an explicit plugin/API design.
