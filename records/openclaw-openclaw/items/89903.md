## Summary

Keep this PR open, but it is not merge-ready: the Slack pre-router hit path is a new opt-in feature that currently bypasses established Slack turn/delivery invariants, adds env-controlled prompt egress and response injection, lacks real Slack proof, and has a failing typed test check.

## What This Changes

The PR adds `OPENCLAW_PRE_ROUTER_URL` and `OPENCLAW_PRE_ROUTER_TIMEOUT_MS` to let Slack monitor messages call an external HTTP matcher and post matched replies before running the LLM.

## Best Possible Solution

If maintainers want this capability, implement it through an approved plugin/generic hook seam or route matched replies through the existing Slack inbound-turn and delivery pipeline, then document the trust boundary and prove real Slack hit/miss behavior.

## Reproduction Assessment

## Solution Assessment

No. The direct Slack send-and-return path is the wrong layer for this feature; a safer design would use the existing plugin hook surface or route matched replies through normal Slack dispatch invariants.
