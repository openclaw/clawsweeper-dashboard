The PR adds a Slack monitor HTTP pre-router controlled by `OPENCLAW_PRE_ROUTER_URL` and `OPENCLAW_PRE_ROUTER_TIMEOUT_MS`, posting matched responses directly to Slack and skipping normal LLM dispatch.

## Best Possible Solution

Publish the matcher as a ClawHub/plugin package using `before_agent_reply` or `before_model_resolve`; if those hooks prove insufficient, continue the generic plugin API discussion rather than adding a Slack-only env hook.
