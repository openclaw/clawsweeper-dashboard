## Summary

Keep open: current main and the latest release still have both reported source paths, and the open implementation PR that closes this issue has not merged.

## What This Changes

The issue reports false unscheduled-reminder notes when isolated cron jobs lack a session key and `successfulCronAdds` is reset during compaction.

## Best Possible Solution

Land an updated https://github.com/openclaw/openclaw/pull/43404, or a narrow replacement, with regression tests for isolated cron coverage and compaction-preserved cron evidence.

## Reproduction Assessment

Yes, at source level. Current main resets `successfulCronAdds` during compaction and then checks existing cron coverage by `sessionKey` only, so an enabled isolated cron without `sessionKey` cannot suppress the note after the counter is lost.

## Solution Assessment

Yes, the maintainable direction is to preserve same-turn cron-add evidence across compaction and add carefully scoped same-agent isolated cron coverage. The linked PR should be updated or replaced rather than closing this issue before a merge.
