## Summary

Keep open: current main and the latest release still retain both reported failure paths, and the linked closing PR is open, unmerged, and merge-conflicting. The reporter's follow-up pointing to early PR attempts was useful, but none of those attempts has landed in OpenClaw main.

## What This Changes

The issue reports false unscheduled-reminder notes after isolated-session cron scheduling because existing-cron detection misses isolated jobs and the same-turn cron-add counter is reset across compaction.

## Best Possible Solution

A landed fix should preserve same-turn cron-add evidence through compaction and let the reminder guard recognize enabled same-agent isolated cron jobs without broadening reminder detection.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source: a successful cron add can be reset to zero during compaction, then an enabled isolated same-agent cron without `sessionKey` cannot satisfy the current session-key-only lookup.

## Solution Assessment

Is this the best way to solve the issue? Yes; the narrow maintainable fix is to preserve the same-turn counter through compaction and extend the existing guard lookup to same-agent isolated jobs without changing reminder detection broadly.
