## Summary

Keep open: current main still has both reported source gaps, and open PR #43404 explicitly closes this issue, so this should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests suppressing false unscheduled-reminder notes when isolated cron jobs or same-turn cron additions already cover a reminder commitment across compaction.

## Best Possible Solution

Land PR #43404 or an equivalent targeted change that preserves same-turn cron-add evidence across compaction and treats enabled isolated same-agent cron jobs as reminder coverage, with regression tests for both paths.

## Reproduction Assessment

Yes. Source inspection on current main shows the exact fall-through path: compaction can zero `successfulCronAdds`, and the existing-cron check only matches `sessionKey`, so an enabled isolated job without that key is not considered coverage.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable fix: preserve the counter across compaction and expand the existing-cron guard to same-agent isolated jobs without changing unrelated reminder detection policy.
