## What This Changes

The issue reports that the reminder guard can append a contradictory unscheduled-reminder note after isolated-session cron creation because cron-add evidence is lost during compaction and existing isolated crons are not matched.

## Best Possible Solution

Land an updated https://github.com/openclaw/openclaw/pull/43404 or a narrow replacement that preserves same-turn cron-add evidence across compaction and safely matches same-agent isolated cron jobs, with regression coverage.

## Reproduction Assessment

Yes, at source level. Current main resets `successfulCronAdds` during compaction and checks existing cron coverage by `sessionKey` only, so an enabled isolated cron without `sessionKey` cannot suppress the note after the counter is lost.

## Solution Assessment

Yes, the maintainable fix shape is to preserve same-turn cron-add evidence across compaction and add carefully scoped same-agent isolated-cron coverage; the linked PR should be updated or replaced rather than closing this issue before merge.
