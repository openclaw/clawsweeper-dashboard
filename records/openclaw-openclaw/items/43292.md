## Summary

Keep open. Current main still lets compaction clear the same-turn cron-add counter and the fallback guard only matches enabled cron jobs by `sessionKey`, so isolated same-agent crons remain invisible; #43404 is the open closing PR to review or replace.

## What This Changes

Fix the reminder guard so same-turn `cron.add` state survives compaction and enabled same-agent isolated cron jobs suppress false unscheduled-reminder notes.

## Best Possible Solution

Land #43404 or an equivalent focused fix that passes agent identity into the guard, treats enabled same-agent isolated cron jobs as coverage, preserves `successfulCronAdds` across compaction retry cleanup, and adds regressions for both paths.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: after compaction resets `successfulCronAdds`, a reminder-commitment reply falls through to a cron lookup that cannot match an enabled isolated same-agent cron because it checks only `sessionKey`.

## Solution Assessment

Yes. The narrow maintainable solution is to repair the existing guard and compaction-reset paths with regression coverage, not add a broad config opt-out or downstream delivery sanitizer.
