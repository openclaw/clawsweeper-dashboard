## Summary

Keep open: current main still has the reported source-level bug, and the linked implementation PR is open, conflicting, and not a landed fix.

## What This Changes

The issue asks OpenClaw to stop appending a false unscheduled-reminder note when isolated cron coverage exists and to preserve same-turn cron-add evidence across compaction.

## Best Possible Solution

Land an updated https://github.com/openclaw/openclaw/pull/43404, or a narrow replacement, with regression tests for isolated cron coverage and compaction-preserved cron evidence.

## Reproduction Assessment

Yes, at source level. Current main resets `successfulCronAdds` during compaction and then checks existing cron coverage by `sessionKey` only, so an enabled isolated cron without `sessionKey` cannot suppress the note after the counter is lost.

## Solution Assessment

Yes, the narrow maintainable direction is to preserve same-turn cron-add evidence across compaction and add carefully scoped same-agent isolated cron coverage. The linked PR should be updated or replaced rather than closing this issue before a merge.
