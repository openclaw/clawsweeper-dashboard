## What This Changes

The issue requests that the unscheduled-reminder guard stop appending a false disclaimer when isolated cron coverage or same-turn cron-add evidence exists across compaction.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/43404 with a narrow fix that preserves cron-add evidence across compaction and matches enabled isolated crons only from trusted same-agent/session evidence.

## Reproduction Assessment

Yes. Source inspection on current main shows compaction resets `successfulCronAdds` to zero, then an enabled isolated cron without `sessionKey` cannot satisfy the session-key-only existing-cron check.

## Solution Assessment

Yes, the reported direction is the right narrow fix. The implementation should preserve same-turn cron evidence across compaction and add isolated same-agent cron coverage without broadening suppression to unrelated jobs.
