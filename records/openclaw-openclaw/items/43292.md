## What This Changes

Fix false unscheduled-reminder notes for isolated-session cron reminders by preserving `successfulCronAdds` across compaction and recognizing enabled same-agent isolated cron jobs.

## Best Possible Solution

Land an updated https://github.com/openclaw/openclaw/pull/43404 or equivalent narrow fix that preserves same-turn cron evidence through compaction and suppresses the note for trusted same-agent isolated cron coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: after compaction resets `successfulCronAdds` to zero, an enabled isolated cron without `sessionKey` cannot satisfy the current session-key-only existing-cron check.

## Solution Assessment

Yes. The narrow maintainable solution is to preserve cron-add evidence across compaction and expand existing-cron coverage to trusted same-agent isolated jobs, with regression tests and without changing reminder-pattern policy.
