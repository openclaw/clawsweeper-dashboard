## What This Changes

The issue reports that the unscheduled-reminder guard can append a contradictory note after an isolated-session cron is created and compaction resets same-turn cron-add evidence.

## Best Possible Solution

Review and land an updated https://github.com/openclaw/openclaw/pull/43404, or a narrow replacement, with regression coverage for isolated-cron suppression and compaction-preserved cron-add evidence.

## Reproduction Assessment

Yes, at source level. Current main resets `successfulCronAdds` during compaction and checks existing cron coverage by `sessionKey` only, so an enabled isolated cron without `sessionKey` cannot suppress the note after the counter is lost.

## Solution Assessment

Yes, the maintainable direction is to preserve same-turn cron-add evidence across compaction and add carefully scoped same-agent isolated-cron coverage; the linked PR should be updated or replaced rather than closing this issue before merge.
