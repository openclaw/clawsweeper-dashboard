## What This Changes

Fix the reminder guard so isolated-session cron jobs and same-turn cron adds that survive compaction suppress the false unscheduled-reminder note.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/43404 or an equivalent targeted fix with regression coverage for isolated same-agent cron coverage and compaction counter preservation.

## Reproduction Assessment

Yes. Source inspection on current main shows the same fall-through path: compaction can zero `successfulCronAdds`, and the existing-cron check only matches `sessionKey`, so an enabled isolated job without that key is not considered coverage.

## Solution Assessment

Yes. The narrow maintainable direction is to preserve same-turn cron-add evidence across compaction and expand the existing-cron guard to trusted same-agent isolated jobs without changing unrelated reminder-pattern policy.
