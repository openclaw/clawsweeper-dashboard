## Summary

Keep open: current main and latest release still contain both reported reminder-guard gaps, and no merged same-repo PR implements the isolated-cron plus compaction-counter fix.

## What This Changes

The issue reports false unscheduled-reminder notes when isolated cron jobs are created and when the successful cron-add counter is reset during compaction.

## Best Possible Solution

Keep this issue open for a focused core fix that preserves same-turn cron-add evidence through compaction and treats enabled same-agent isolated crons as reminder coverage in the existing reply guard.

## Reproduction Assessment

Yes, at source level. Current main still resets the cron-add counter during compaction and only matches existing crons by `sessionKey`, so an enabled isolated cron without `sessionKey` cannot suppress the note after the counter is lost.

## Solution Assessment

Yes on direction. The narrowest maintainable solution is to fix the existing reply guard and compaction state path, not to add channel-specific suppression or a broad reminder-policy rewrite.
