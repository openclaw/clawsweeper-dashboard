## What This Changes

Fix false unscheduled-reminder notes when isolated-session cron reminders are created and compaction resets the successful cron-add counter.

## Best Possible Solution

Land a refreshed equivalent of the linked fix that preserves successfulCronAdds across compaction and treats enabled same-agent isolated jobs as reminder coverage, with focused regression tests.

Yes, source-level: a successful cron add can be reset to zero during compaction, then an enabled same-agent isolated cron without the current sessionKey cannot satisfy the existing lookup. I did not run a live Telegram or cron scenario in this read-only review.

## Solution Assessment

Yes on direction: the narrow maintainable fix is to preserve same-turn cron-add evidence through compaction and extend existing cron coverage to enabled same-agent isolated jobs. The open linked PR targets that shape but needs maintainer refresh or replacement because it is currently conflicting.
