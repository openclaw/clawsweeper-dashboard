Keep open: current main and latest release still show both source gaps, and the open implementation candidate that closes this issue is unmerged, conflicting, and still needs maintainer follow-up rather than being proof of a fix.

## What This Changes

The issue asks OpenClaw to stop appending the unscheduled-reminder note when a cron add succeeded before compaction or when an enabled isolated same-agent cron already covers the commitment.

## Best Possible Solution

Land a corrected version of https://github.com/openclaw/openclaw/pull/43404, or a narrow replacement, with regression tests for isolated cron coverage and compaction-preserved cron evidence, then close this issue from the merge.

## Reproduction Assessment

Yes, at source level. Current main resets successfulCronAdds during compaction and then checks existing cron coverage by sessionKey only, so an enabled isolated cron without sessionKey cannot suppress the note after the counter is lost.

## Solution Assessment

Yes, the requested fix boundary is the narrow maintainable direction: preserve same-turn cron-add evidence across compaction and add carefully scoped same-agent isolated cron coverage. The linked PR should be updated or replaced rather than closing this issue before a merge.
