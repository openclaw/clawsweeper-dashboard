## Summary

Keep open: current main still has both reported source gaps, and an open linked implementation PR has closing syntax but has not landed.

## What This Changes

The issue reports that the unscheduled-reminder guard can append a contradictory note after compaction when the successfully created cron is an isolated same-agent job without a session key.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/43404 with a narrow fix that preserves cron-add evidence across compaction and matches enabled isolated crons only for trusted same-agent coverage, with regression tests and real runtime proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: compaction resets `successfulCronAdds` to zero, then an enabled isolated cron without `sessionKey` cannot satisfy the current session-key-only existing-cron check.

## Solution Assessment

Yes, the reported fix direction is the narrow maintainable shape: preserve same-turn cron evidence across compaction and expand existing-cron coverage to trusted same-agent isolated jobs. The exact PR should still avoid over-suppression by using canonical agent/session evidence.
