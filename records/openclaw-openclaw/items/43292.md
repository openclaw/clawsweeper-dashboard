## What This Changes

The issue reports that the unscheduled-reminder guard can append a contradictory disclaimer for isolated-session cron reminders because compaction resets same-turn cron evidence and existing-cron coverage only matches by session key.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/43404 with a narrow regression-tested fix that preserves same-turn cron-add evidence across compaction and recognizes enabled isolated cron coverage without broad unrelated-job suppression.

## Reproduction Assessment

Yes. Source inspection on current main shows compaction resets successfulCronAdds to zero, then an enabled isolated cron without sessionKey cannot satisfy the session-key-only existing-cron check.

## Solution Assessment

Yes. The requested fix direction is the narrow maintainable path: preserve same-turn cron evidence across compaction and add trusted isolated same-agent cron coverage, with regression tests for both paths.
