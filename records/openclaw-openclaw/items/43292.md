## Summary

Keep open: current main and the latest release still have both reported source-level failure paths, and the prior target-repo PR for this issue was closed unmerged rather than landing a fix.

## What This Changes

The issue reports false unscheduled-reminder notes when isolated-session cron coverage is missed and same-turn cron-add evidence is lost across compaction.

## Best Possible Solution

Preserve same-turn cron-add evidence across compaction and add a scoped isolated-cron coverage check that uses canonical agent identity without broadly suppressing unrelated reminder warnings.

## Reproduction Assessment

Yes, at source level. Current main resets `successfulCronAdds` during compaction and checks existing cron coverage by `sessionKey` only, so an enabled isolated cron without `sessionKey` cannot suppress the note after the counter is lost.

## Solution Assessment

Yes for the issue direction, but not for closing now. The narrow maintainable fix is counter preservation plus scoped isolated-cron coverage with regression tests for unrelated cron jobs.
