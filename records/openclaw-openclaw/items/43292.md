## What This Changes

The issue reports false unscheduled-reminder notes when isolated-session cron coverage exists and same-turn cron-add evidence is lost during compaction.

## Best Possible Solution

Land an updated version of https://github.com/openclaw/openclaw/pull/43404, or a narrow replacement, with regression tests for isolated cron coverage and compaction-preserved cron evidence.

## Reproduction Assessment

Yes, at source level. Current main resets successfulCronAdds during compaction and then checks existing cron coverage by sessionKey only, so an enabled isolated cron without sessionKey cannot suppress the note after the counter is lost.

## Solution Assessment

Yes, the narrow maintainable direction is to preserve same-turn cron-add evidence across compaction and add carefully scoped same-agent isolated cron coverage. The linked PR should be updated or replaced rather than closing this issue before a merge.
