## What This Changes

The issue asks OpenClaw to stop appending the unscheduled-reminder note when an isolated cron was created successfully or when same-turn cron-add evidence was lost during compaction.

## Best Possible Solution

Review, update or replace https://github.com/openclaw/openclaw/pull/43404 with a narrow fix that preserves same-turn cron-add evidence across compaction and safely matches same-agent isolated cron jobs with regression coverage.

## Reproduction Assessment

Yes, at source level. Current main resets `successfulCronAdds` during compaction and only checks existing cron coverage by `sessionKey`, so an enabled isolated cron without a session key cannot suppress the note after the counter is lost.

## Solution Assessment

Yes in direction, with care around ownership. The maintainable fix is to preserve same-turn cron-add evidence across compaction and add scoped isolated-cron coverage without broadening suppression to unrelated jobs.
