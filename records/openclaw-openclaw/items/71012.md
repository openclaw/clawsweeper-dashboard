Overall confidence: 0.94

Full review comments:

- **[P1] Move `Parameters` outside the import type:** `extensions/memory-core/src/dreaming-shared-types.ts:35-37`
  - body: `Parameters` is a TypeScript global utility type, not a member of `./dreaming-narrative.js`. This type expression will not compile; wrap `Parameters<...>` around the imported function type instead.
  - confidence: 0.97
- **[P1] Import workspace resolution from its exporter:** `extensions/memory-core/src/dreaming-session-corpus.ts:46`
  - body: `dreaming-shared-types.ts` does not export `resolveMemoryDreamingWorkspaces`, so this named import fails. Import it from `openclaw/plugin-sdk/memory-core-host-status` or explicitly re-export it.
  - confidence: 0.97
- **[P2] Restore the current dreaming exports:** `extensions/memory-core/src/dreaming-phases.ts:979-983`
  - body: The refactor drops `filterRecallEntriesWithinLookback` and `__testing`, while current tests, `rem-harness`, and the memory-core API facade still import from `dreaming-phases.ts`. Preserve these exports or move all consumers in the same change.
  - confidence: 0.94
- **[P2] Preserve live recall filtering:** `extensions/memory-core/src/dreaming-phases.ts:668-674`
  - body: Current main filters lookback-matched recall entries through `filterLiveShortTermRecallEntries` before light and REM dreaming. This branch reads recalls directly and only applies the older lookback predicate, reintroducing stale-source candidates.
  - confidence: 0.9
- **[P2] Forward primary workspace into session ingestion:** `extensions/memory-core/src/dreaming-phases.ts:661-667`
  - body: Current main passes `primaryWorkspaceDir` into session transcript ingestion so multi-workspace dreaming keeps the initiating workspace context. The extracted call drops that parameter, which can change session corpus ingestion behavior.
  - confidence: 0.86
- **[P2] Pass the configured model into dream narratives:** `extensions/memory-core/src/dreaming-phases.ts:721-740`
  - body: Current main includes `execution.model` in the dreaming config and forwards it to narrative generation. The refactor omits that field, so custom dreaming model selection stops working.
  - confidence: 0.87
- **[P2] Use the capped detached narrative helper:** `extensions/memory-core/src/dreaming-phases.ts:721-730`
  - body: Current main routes detached dreaming narratives through `runDetachedDreamNarrative`, which caps concurrent detached narratives. Direct `queueMicrotask` calls bypass that cap during sweeps.
  - confidence: 0.88
- **[P2] Preserve slugged daily memory discovery:** `extensions/memory-core/src/dreaming-shared-types.ts:54`
  - body: Current main accepts daily memory files such as `YYYY-MM-DD-slug.md` and prioritizes canonical date-only notes. The extracted shared regex only matches `YYYY-MM-DD.md`, so slugged daily notes would stop flowing through dreaming ingestion.
  - confidence: 0.86
