  - confidence: 0.97

  - confidence: 0.97
- **[P2] Restore the dreaming public and test exports:** `extensions/memory-core/src/dreaming-phases.ts:979-981`
  - body: The refactor drops `__testing` and `filterRecallEntriesWithinLookback`, while current tests and the memory-core bundled facade still import them from `dreaming-phases.ts`. Preserve those exports or move all consumers in the same change.
  - confidence: 0.92
- **[P2] Preserve live recall filtering:** `extensions/memory-core/src/dreaming-phases.ts:668-674`
  - body: Current main filters short-term recall entries through `filterLiveShortTermRecallEntries` before light and REM dreaming. This branch reads recalls directly and only applies the older lookback predicate, which would reintroduce stale-source candidates.
  - confidence: 0.88
- **[P2] Carry forward dreaming model overrides:** `extensions/memory-core/src/dreaming-shared-types.ts:13-16`
  - body: Current main's resolved light and REM dreaming configs include `execution.model` and pass it to narrative generation. The extracted config type and narrative calls omit that field, so custom dreaming model selection would stop working.
  - confidence: 0.84
- **[P2] Use the capped detached narrative helper:** `extensions/memory-core/src/dreaming-phases.ts:721-730`
  - body: Current main routes detached dreaming narratives through `runDetachedDreamNarrative`, which caps concurrent detached narratives. This PR restores direct `queueMicrotask` calls to `generateAndAppendDreamNarrative`, bypassing that cap during cron sweeps.
  - confidence: 0.86
- **[P2] Keep the registration shim signature:** `extensions/memory-core/src/dreaming-phases.ts:979-981`
  - body: `registerMemoryDreamingPhases` is a compatibility shim on current main. Changing it to an async `Record<string, unknown>` function changes the exported type contract for no runtime benefit in a refactor-only PR.
