## Summary

Keep this PR open, but the proposed fix is not merge-ready: the current-main bug is real, yet the branch uses a pre-compaction array length as a cutoff after compaction has rebuilt the session message array, so it can still zero valid post-compaction usage. The PR also has only unit/format proof for a user-visible TUI counter problem.

## What This Changes

The PR adds `preCompactionMessageCount` state and changes stale assistant-usage clearing to stop at the captured pre-compaction message count.

## Best Possible Solution

Use the rewritten history's compaction boundary, such as latest compaction summary index/timestamp or an equivalent durable marker, to clear only assistant usage before the latest compaction; add a focused regression and real TUI/gateway counter proof.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. The current-main source path is source-reproducible: non-retry compaction calls a helper that zeros every assistant message usage snapshot.

## Solution Assessment

No, this is not the best fix yet. The durable boundary should be derived from the post-compaction history, not from the pre-compaction message-array length that compaction later invalidates.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use a post-compaction boundary for usage clearing:** `src/agents/embedded-agent-subscribe.handlers.compaction.ts:54`
  - body: The cutoff captured here is based on the message array before compaction starts, but `runCompactionWork` appends a compaction entry and replaces `agent.state.messages` with `buildSessionContext()` before `compaction_end` is emitted. After that rewrite, a valid post-compaction assistant can be at a low index while `preCount` is still the old history length, so the new `i >= preCount` break never fires and its usage is still zeroed. Please use the actual compaction boundary in the rewritten messages, like the replay-history helper does with the latest compaction summary index/timestamp.
