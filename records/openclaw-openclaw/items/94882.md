## Summary

Close as superseded: the same Telegram/Ollama thinking-menu bug is already owned by https://github.com/openclaw/openclaw/pull/94067, which fixes the menu layer with runtime catalog metadata across Telegram, Slack, and Discord. This branch changes the Ollama fallback instead, which can advertise thinking levels for uncataloged non-reasoning Ollama models.

## What This Changes

This PR changes the Ollama provider policy resolver so missing or undefined `reasoning` returns the full thinking profile and adds a unit test for that case.

## Best Possible Solution

Close this branch and review or land the runtime-catalog based fix in https://github.com/openclaw/openclaw/pull/94067 or an equivalent change that preserves actual Ollama `reasoning:true/false` metadata.

## Reproduction Assessment

Yes for source reproduction: the linked report gives concrete OpenClaw 2026.6.8 Telegram steps, and current main shows native menu resolution falling back to configured-only metadata while the runtime catalog can carry Ollama `reasoning` capability. I did not run a live Telegram/Ollama endpoint in this read-only review.

## Solution Assessment

No. The best fix is to pass runtime catalog metadata into native `/think` menu resolution so the provider policy sees actual `reasoning:true/false`, rather than treating missing metadata as reasoning-capable.
