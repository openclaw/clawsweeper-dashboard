## Summary

Keep open: this PR targets a real source-reproducible bootstrap gap, but the latest patch is not merge-ready because it gives default per-agent state directories prompt authority, can keep stale agentDir BOOTSTRAP.md active after setup, lacks real startup/session proof, and still needs a maintainer/security decision on workspace-only warnings versus agentDir precedence.

## What This Changes

The PR changes `src/agents/bootstrap-files.ts` so the shared bootstrap resolver loads present files from `resolveAgentDir()` in addition to workspace files and lets same-named agentDir files override workspace files.

## Best Possible Solution

Land a maintainer-approved policy: either warning-only diagnostics, or explicit configured-agentDir bootstrap precedence with docs, focused regression tests, and redacted runtime proof.

## Reproduction Assessment

Yes by source inspection: current main reads bootstrap files from workspaceDir only, while callers pass agentId into the shared resolver so a resolver change would affect normal run paths.

## Solution Assessment

No. The shared resolver is the right owner boundary, but using `resolveAgentDir()` directly is too broad because it includes default state directories and leaves bootstrap precedence policy unresolved.
