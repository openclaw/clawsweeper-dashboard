## Summary

## What This Changes

## Best Possible Solution

Keep workspace as the canonical bootstrap source unless maintainers explicitly approve agentDir precedence; if approved, gate loading to explicit agentDir configuration with upgrade/security tests and real startup proof, otherwise land a warning-only diagnostic/docs path.

No. Loading agentDir files as overriding prompt context is only a plausible policy, and this implementation also includes the default state-dir fallback; warning-only or explicit, maintainer-approved agentDir semantics would be safer.
