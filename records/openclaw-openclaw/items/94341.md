## Summary

Keep open, but this PR is not merge-ready: it targets the real agentDir bootstrap issue, yet it makes the default per-agent state directory prompt-authoritative without maintainer-approved policy, upgrade/security proof, or real behavior proof.

## What This Changes

The branch changes `src/agents/bootstrap-files.ts` so `resolveBootstrapFilesForRun` loads non-missing standard bootstrap files from the resolved per-agent `agentDir` and lets them override same-named workspace files.

## Best Possible Solution

Keep workspace as the canonical bootstrap source unless maintainers explicitly approve agentDir precedence; if approved, constrain it to a documented explicit policy with diagnostics, tests, and live upgrade proof.

## Reproduction Assessment

Yes, by source inspection: current main loads bootstrap files from `workspaceDir` only, while the linked report’s agentDir files are outside that source. I did not run a live multi-agent startup in this read-only review.

## Solution Assessment

No. Loading every resolved agentDir, including default state directories, is broader than the safest maintainable fix; warning-only diagnostics or a maintainer-approved explicit opt-in would better preserve the workspace/state boundary.
