## What This Changes

## Best Possible Solution

Reproduce the two-message Telegram DM flow against current main or latest stable, then add a focused regression that proves one stable session-store key gains or reuses a sessionId and transcript across consecutive DMs.

## Reproduction Assessment

## Solution Assessment

Unclear. The best fix depends on proving where the write is skipped or lost; likely candidates are Telegram inbound recording, session-key resolution, or the auto-reply session-store write, not a new product mode.
