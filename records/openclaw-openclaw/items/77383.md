## Summary

Keep open: current main does not implement `respondToMentionAll`, and the bundled Feishu plugin deliberately treats broadcast-only `@all` as not being a bot mention, so this is a product/plugin decision rather than an already-fixed current-main bug.

## What This Changes

The issue requests that Feishu/Lark `@所有人` group messages be recognized so an opt-in `respondToMentionAll` setting can allow replies.

## Best Possible Solution

If maintainers want this behavior, add a plugin-owned, documented opt-in in `@openclaw/feishu` that detects Feishu broadcast mentions safely without reclassifying them as direct bot mentions, and cover the #37706 false-positive cases.

## Reproduction Assessment

Yes for the current behavior: source inspection shows broadcast-only `@all` does not set `mentionedBot`, and `requireMention` gating returns before dispatch. No for a broken current-main contract, because `respondToMentionAll` is not a documented or schema-supported option in this repo.

## Solution Assessment

No for the proposed raw-content fallback as written; it risks recreating the prior `@_all` false-positive path. A safer direction is an explicit opt-in config plus structured parser coverage for text/post payload variants.
