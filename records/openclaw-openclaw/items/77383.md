## Summary

Keep open: current main intentionally excludes Feishu broadcast-only `@all` from bot-mention detection, and `respondToMentionAll` is not a documented or schema-supported OpenClaw option. Resolving this would require a Feishu plugin product/config decision, not conservative cleanup.

## What This Changes

The issue asks for Feishu group `@所有人`/`@all` messages to trigger bot replies when `respondToMentionAll` is enabled.

## Best Possible Solution

If maintainers want this behavior, add a Feishu-plugin-owned, documented opt-in that detects structured broadcast mentions safely without reclassifying `@all` as a direct bot mention, and cover the prior false-positive cases.

## Reproduction Assessment

Yes for the current behavior: source and regression tests show broadcast-only `@all` does not set `mentionedBot`, so `requireMention` gates it out before dispatch. No high-confidence broken current-main contract exists because `respondToMentionAll` is not documented or schema-supported here.

## Solution Assessment

No for the proposed raw-content fallback as written; it risks reopening the prior `@all` false-positive behavior. A safer solution would be an explicit opt-in plus parser coverage for Feishu text, mention metadata, and post payload variants.
