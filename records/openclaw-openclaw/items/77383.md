## Summary

Keep open: current main intentionally treats Feishu broadcast-only `@all` as not being a bot mention, and `respondToMentionAll` is not a supported Feishu config key, so this needs a maintainer product/plugin decision rather than cleanup closure.

## What This Changes

The issue requests an opt-in Feishu behavior where group `@所有人` messages can trigger replies when mention gating is enabled.

## Best Possible Solution

If maintainers want this behavior, add a Feishu-owned documented opt-in that tracks broadcast `@all` separately from direct bot mentions and covers text, metadata, and post payload variants.

## Reproduction Assessment

Yes for the current non-response behavior: source inspection shows broadcast-only `@all` does not set `mentionedBot`, and the group mention gate returns before dispatch. No for a broken existing contract, because current schema/docs do not support `respondToMentionAll`.

## Solution Assessment

No for the raw-content fallback as proposed; the safer solution is a documented opt-in that detects Feishu broadcast mentions without reclassifying them as direct bot mentions.
