## Summary

Keep open: current main intentionally excludes broadcast-only Feishu `@all` from bot-mention detection and exposes no `respondToMentionAll` config, so this is a Feishu plugin feature/policy request rather than implemented cleanup.

## What This Changes

Request to make Feishu group `@所有人` / `@all` trigger replies when `respondToMentionAll` is enabled even with mention gating.

## Best Possible Solution

If maintainers want this behavior, add a Feishu-owned documented opt-in that detects broadcast mentions separately from direct bot mentions, preserves the current safe default, and covers the known false-positive cases.

## Reproduction Assessment

Yes for current behavior: source inspection shows broadcast-only `@all` does not set `mentionedBot`, and `requireMention` gating returns before dispatch. No broken current-main contract is established because `respondToMentionAll` is not a documented or schema-supported option here.

## Solution Assessment

No for the proposed raw-content fallback as written; it overlaps with the prior `@all` false-positive class. The safer path is an explicit Feishu config option plus structured parser coverage for text, mention metadata, and post payload variants.
