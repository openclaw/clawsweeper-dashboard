## Summary

Keep open: current main intentionally treats Feishu/Lark broadcast-only `@all` as not being a bot mention, and `respondToMentionAll` is not a supported Feishu config key. This is a Feishu plugin product/config decision, not an already-fixed bug or a safe cleanup close.

## What This Changes

The issue requests that Feishu/Lark group `@all` / `@所有人` messages pass mention gating when `respondToMentionAll` is configured.

## Best Possible Solution

If maintainers want this behavior, add a documented Feishu-owned opt-in that detects broadcast mentions safely without reclassifying them as direct bot mentions, and preserve the existing safe default.

## Reproduction Assessment

Yes for the current behavior: source inspection shows broadcast-only `@all` does not set `mentionedBot`, and mention-gated groups return before dispatch. No for a broken documented contract, because `respondToMentionAll` is not supported by current schema or docs.

## Solution Assessment

No for the suggested raw-content fallback by itself; it overlaps with the prior false-positive behavior. The maintainable path is an explicit Feishu config option plus structured parser coverage and docs.
