## What This Changes

The PR resolves Codex app-server transcript tool-output caps from default/per-agent `agents.contextLimits.toolResultMaxChars`, applies them to streamed and snapshot tool-result messages, and adds regression tests.

## Best Possible Solution

Land this PR if maintainers accept using the existing agent context-limit contract for Codex native transcript projection, while leaving broader WebChat-specific knobs to https://github.com/openclaw/openclaw/issues/82246.

## Solution Assessment

Yes. Resolving the app-server transcript cap from the existing agent context-limit contract is the narrow maintainable fix for this bug, and the updated marker check avoids the earlier raw-text bypass concern.
