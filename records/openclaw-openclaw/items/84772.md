## Summary

Keep open: the PR fixes the source-reproducible configured-cap path and includes useful terminal proof, but the patch still leaves Codex app-server transcripts on a separate 12K fallback instead of the documented 16K tool-result default, and streamed fallback metadata can undercount later chunks.

## What This Changes

The PR updates the Codex app-server event projector and tests so native transcript tool-result projection uses configured default/per-agent `toolResultMaxChars` and emits a detailed truncation notice.

## Best Possible Solution

Align Codex app-server transcript projection with the existing `toolResultMaxChars` contract: explicit per-agent/default values win, unset config falls back to the documented 16K live tool-result default, and truncation notices stay accurate for snapshot and streamed fallback paths.

## Reproduction Assessment

Yes, source-reproducible: current main hardcodes the Codex app-server/native transcript cap at 12K while adjacent runtime/docs expose `toolResultMaxChars` as the live tool-result cap. The PR body also supplies terminal proof for the after-fix configured-cap path.

## Solution Assessment

No, not quite. Resolving the existing context-limit setting inside the projector is the right owner-boundary fix, but the branch should also align the unset default with the documented 16K cap and keep streamed truncation metadata accurate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Use the tool-result default when config is absent:** `extensions/codex/src/app-server/event-projector.ts:97`
  - body: The new resolver falls back to a projector-local `12_000` default when no default/per-agent value is configured, but the current context-limit contract documents and implements `toolResultMaxChars` with a 16,000-character live tool-result default. A no-config Codex app-server transcript can still truncate a 15K result below the documented persisted-result cap, so use the same 16K fallback or a shared helper and add a no-config regression case.
  - confidence: 0.8
- **[P3] Track full streamed output size before reporting it:** `extensions/codex/src/app-server/event-projector.ts:1959`
  - body: When streamed output crosses the cap, `originalChars` only includes the buffered text plus the delta that crossed the limit. Later `outputDelta` chunks return at the `current.length >= maxChars` guard, so a fallback transcript with no final `aggregatedOutput` can store a misleading exact original-size notice; track total seen characters or make this streamed notice non-exact.
  - confidence: 0.84
