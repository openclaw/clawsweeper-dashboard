## Summary

Keep open: the PR targets a real restart-stuck session bug, but this patch deletes the durable recovery marker after recovery has failed and does so without the eligibility filters that protect active or non-routable rows.

## What This Changes

## Best Possible Solution

Preserve `abortedLastRun` until resume succeeds or the session is explicitly failed/tombstoned, fix the stale run or lane owner path, and prove the mid-turn restart with a real follow-up message.

- **[P1] Preserve the recovery marker after failed retries:** `src/agents/main-session-restart-recovery.ts:893-896`
  - body: This branch calls `clearStaleAbortedRunFlags()` when recovery still reports failures.   - confidence: 0.9

If one failed row exhausts retries, this can mutate skipped current-process or non-routable rows that recovery deliberately preserved.
