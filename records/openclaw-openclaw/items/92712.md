## Summary

Keep this PR open, but the current patch is not merge-ready: it clears the durable restart-recovery marker after recovery has already failed, which can make a still-running stuck session invisible to later recovery instead of proving it is dispatchable.

## What This Changes

The PR adds a helper that clears `abortedLastRun` from running session rows after startup recovery retries are exhausted.

## Best Possible Solution

Preserve the restart recovery marker until resume succeeds or the session is explicitly failed, and fix the stale run or lane owner that keeps the session non-dispatchable after restart.
