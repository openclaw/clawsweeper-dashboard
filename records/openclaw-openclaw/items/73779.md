## Summary

Keep this PR open. Current main still has the eager reload behavior, but the proposed patch is not merge-ready because it defers the reload without updating the existing unit test, leaves a pending debounce timer alive across Control UI teardown, and misses the required changelog entry for a user-facing performance fix.

## What This Changes

The PR adds Control UI debounce state and changes `sessions.changed` handling so the incremental session update remains immediate while the full `sessions.list` reconciliation runs 5 seconds after the last push event.

## Best Possible Solution

Request changes on this PR: keep the narrow UI-only debounce direction, add lifecycle cleanup for the pending timer, update fake-timer coverage for delayed and coalesced reloads, initialize the test fixture state, and add the active changelog entry before relying on normal CI and changed-gate validation.

## Reproduction Assessment

Yes. Code inspection gives a high-confidence reproduction path: on the PR head, `sessions.changed` schedules `loadSessions` with `setTimeout`, while the existing unit test still asserts the call immediately after dispatch.

## Solution Assessment

No, not as submitted. Debouncing the full reconciliation is a maintainable fix for the reported fan-out pattern, but the implementation needs timer cleanup, updated tests, and changelog handling before it is the best solution.
