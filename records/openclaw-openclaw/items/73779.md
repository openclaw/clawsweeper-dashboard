## Summary

Keep this PR open. Current main still performs the eager Control UI session reload the PR targets, but the proposed patch has merge-blocking test and lifecycle gaps before it is safe to land.

## What This Changes

The PR adds Control UI timer state and replaces the immediate `sessions.changed` full `loadSessions` reconciliation with a 5-second debounce while keeping the incremental update path immediate.

## Best Possible Solution

Request changes on this PR: keep the incremental session-row update immediate, debounce only the full `sessions.list` reconciliation, clear the pending timer during Control UI teardown, add fake-timer coverage for delayed and coalesced reloads, initialize the test fixture state, and add the active-version changelog entry if maintainers classify this as a user-facing Control UI performance fix.

## Reproduction Assessment

Yes. Current main can be reproduced from source and tests: `handleGatewayEventUnsafe` immediately calls `loadSessions` on `sessions.changed`, and the existing unit test asserts that synchronous behavior.

## Solution Assessment

No, not as submitted. The debounce direction is a narrow maintainable fix for the reported load pattern, but the patch needs lifecycle cleanup, updated tests, and changelog handling before it is the best solution.
