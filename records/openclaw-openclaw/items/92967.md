## What This Changes

The issue requests warming model context-window metadata during Gateway startup so the first `/status` after a cold start reports model-specific limits instead of the 200k fallback.

## Best Possible Solution

Land one narrowed, proof-backed fix that makes the first cold `/status` output use correct model context metadata while preserving status/Gateway startup boundaries, then close this issue from the merged PR.

Unclear. Startup warming may be part of the fix, but the maintainable solution must make the first rendered status deterministic without violating the existing read-only/lazy-loading status boundary.
