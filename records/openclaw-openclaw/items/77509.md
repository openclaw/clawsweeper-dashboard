## Summary

Close as cannot reproduce: the official WhatsApp npm packages expose runtime entries and shipped dist files, and current main resolves those before the compiled-runtime warning path.

## What This Changes

The issue asks OpenClaw to stop warning that @openclaw/whatsapp is missing compiled runtime output when the installed package has valid dist runtime files.

## Best Possible Solution

Keep the documented source-entry plus runtime-entry contract; reopen only with the local installed package.json, package path, and full warning source if a stale shadow still warns.

## Reproduction Assessment

No. Current main resolves explicit runtime entries and inferred dist peers before the warning path, and the live WhatsApp 2026.5.3 package contains the expected runtime metadata and files.

## Solution Assessment

Yes. The existing contract-aligned behavior is the narrowest solution: honor runtimeExtensions/runtimeSetupEntry and warn only when no compiled runtime file can be resolved.
