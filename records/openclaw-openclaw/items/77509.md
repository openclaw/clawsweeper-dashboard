## Summary

Current main and the live WhatsApp 2026.5.3 npm artifact do not match the reported warning path: OpenClaw resolves explicit runtime entries and shipped dist files before it can emit the compiled-output warning.

## What This Changes

The issue reports a recurring compiled-runtime-output warning for `@openclaw/whatsapp@2026.5.3` even though the package appears to ship valid `dist/` output.

## Best Possible Solution

Keep the documented source-entry plus runtime-entry package contract; if the warning persists, a new report should include the exact warning path/source and installed package manifest so stale local shadow state can be separated from a real resolver bug.

## Reproduction Assessment

No. Current main resolves explicit `runtimeExtensions` and inferred dist peers before the warning path, and the live `@openclaw/whatsapp@2026.5.3` artifact contains the expected runtime metadata and files.

## Solution Assessment

Yes. The best current behavior is to honor `runtimeExtensions`/`runtimeSetupEntry` and repair or remove stale invalid managed plugin shadows through install/update/doctor paths, not strip valid source entry metadata.
