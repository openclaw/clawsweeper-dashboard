## What This Changes

The PR replaces the static heartbeat active-hours seek iteration cap with a 30-second minimum phase-aligned batch step plus backward walking, and adds scheduler regression tests for sub-minute and sub-second intervals.

## Best Possible Solution

Land the bounded phase-aligned seek with its regression tests, while treating the small-interval scheduling change as an intentional compatibility and availability-sensitive bug fix.

## Reproduction Assessment

Yes. Source inspection of current main shows the helper advances by `intervalMs` but stops after 10,080 iterations, so accepted sub-minute intervals can stop before the seven-day active-hours horizon.

## Solution Assessment

Yes. The v6 shape is the best current fix because it preserves phase alignment, uses a batch step below the minute-granular active-hours window, and keeps the production scan bounded without adding a config knob.
