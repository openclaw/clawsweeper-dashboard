## Summary

Keep this issue open. Current main has meaningful building blocks, including update dry-run planning, structured doctor lint findings, plugin-focused post-upgrade checks, post-update gateway health recovery, and rollback instructions, but it still lacks the requested unified pre/post-update assessment with risk scoring and smoke recommendations. The remaining work requires a maintainer-defined product contract rather than a narrow autonomous fix.

## What This Changes

Add a native pre/post-update compatibility scanner that produces structured risk findings, smoke-test recommendations, and rollback guidance.

## Best Possible Solution

Define one stable upgrade-assessment envelope that composes update dry-run facts, doctor lint findings, plugin post-upgrade probes, and post-update health results; use stable finding codes and advisory smoke/rollback guidance, while leaving backup and automatic restoration behind the separate maintainer decision in https://github.com/openclaw/openclaw/issues/14526.

## Solution Assessment

Yes for the requested outcome, but not as a separate validation framework. The narrowest maintainable design is an orchestration and reporting layer over existing canonical checks, with maintainers defining stable severity, smoke, and rollback semantics first.
