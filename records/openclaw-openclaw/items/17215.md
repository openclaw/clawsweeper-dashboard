The issue requests a native pre/post-update compatibility scanner, proposed as `doctor --upgrade-scan`, with schema/deprecation checks, risk-scored JSON and human reports, smoke recommendations, and rollback hints.

Define one maintainer-approved upgrade assessment contract that composes existing doctor, update, plugin compatibility, config migration, health/status, backup, and rollback guidance surfaces with explicit redaction, severity, JSON, and smoke-check semantics.
