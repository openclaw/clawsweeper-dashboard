## Summary

Keep open. Current main has adjacent update dry-run, post-update plugin convergence, static plugin payload smoke checks, doctor repair, plugin integrity reporting, and rollback docs, but it still does not provide the requested native pre/post-update compatibility scanner with risk-scored JSON/human output, smoke recommendations, or rollback hints.

## What This Changes

The issue requests a built-in pre/post-update compatibility scanner, likely under doctor or update, with schema/deprecation checks, smoke recommendations, risk scoring, JSON output, and rollback guidance.

## Best Possible Solution

Define a stable core upgrade-scan contract that composes config validation, doctor health checks, update dry-run planning, plugin diagnostics, gateway/preflight probes, backup/rollback guidance, redacted JSON output, and public docs.

## Reproduction Assessment

Not applicable; this is a feature request rather than a current-main bug. Source, docs, live issue state, and related-item inspection show adjacent primitives exist, but the requested scanner/report surface is absent.

## Solution Assessment

Unclear until maintainers choose the CLI/API surface, JSON schema, risk model, smoke criteria, rollback boundary, and redaction rules. A native scanner fits the operational-safety goal, but it should be designed as a stable core contract rather than a one-off flag.
