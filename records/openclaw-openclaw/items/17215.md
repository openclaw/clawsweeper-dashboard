## Summary

Keep open. Current main has adjacent update dry-run, update JSON, post-update doctor/plugin sync, compatibility diagnostics, and rollback documentation, but it still lacks the requested native pre/post-update compatibility scanner with a stable risk-scored JSON/human report, smoke recommendations, and rollback hints.

## What This Changes

Request a native `doctor --upgrade-scan` or update compatibility scan that checks pre/post-update drift and emits risk-scored JSON/human reports with smoke and rollback recommendations.

## Best Possible Solution

Define a stable core upgrade-scan contract that composes existing config validation, doctor checks, update dry-run planning, plugin diagnostics, health/status probes, backup/rollback guidance, and documented JSON output, then add focused tests and docs.

## Solution Assessment

Unclear. A native upgrade-safety scanner fits the problem, but maintainers still need to choose the CLI surface, report schema, risk scoring, smoke-check model, and rollback boundary before implementation.
