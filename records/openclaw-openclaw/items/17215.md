## Summary

Keep open: current main and the latest release have adjacent doctor lint, update dry-run/finalize, plugin convergence, integrity-drift reporting, and upgrade docs, but they still do not provide the requested native pre/post-update compatibility scanner with risk scoring, smoke recommendations, rollback hints, and one stable JSON/human upgrade report contract. ## What This Changes

The issue requests a native `doctor --upgrade-scan` style pre/post-update compatibility scanner with schema checks, smoke recommendations, risk reporting, JSON output, and rollback hints.

## Best Possible Solution

Define one stable core upgrade-scan contract that composes doctor lint/config validation, update dry-run planning, post-core plugin convergence, gateway or preflight checks, smoke recommendations, rollback guidance, redacted JSON, human output, tests, and docs.
