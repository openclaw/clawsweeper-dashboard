## Summary

Keep this issue open. Current main and the latest release include useful building blocks, especially doctor lint, plugin-only post-upgrade probes, update dry-run planning, plugin convergence checks, and gateway restart recovery, but they still do not provide the requested unified pre/post-update compatibility scanner with risk scoring, smoke recommendations, and rollback recommendation semantics.

## What This Changes

Request a native `doctor --upgrade-scan` or equivalent update scanner that compares pre/post-update state, emits risk reports, recommends smoke checks, and suggests rollback guidance.

## Best Possible Solution

Define one stable upgrade-assessment report that composes update dry-run facts, doctor lint findings, plugin post-upgrade probes, post-core plugin convergence, and gateway health results, while coordinating backup/rollback policy with https://github.com/openclaw/openclaw/issues/14526 and config-diff semantics with https://github.com/openclaw/openclaw/issues/38249.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new tooling/report contract request. Source and docs inspection give high-confidence confirmation that current main lacks the requested unified upgrade-scan capability.

## Solution Assessment

Yes in direction, but not as a separate validation framework. The maintainable solution is an orchestration/reporting layer over existing doctor, update, plugin, and health-check contracts, with maintainer-defined severity, smoke, and rollback semantics first.
