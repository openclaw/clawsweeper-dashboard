## Summary

Keep open. Current main still has adjacent update dry-run, doctor repair, plugin sync/integrity reporting, manual rollback docs, and update status surfaces, but not the requested native pre/post-update compatibility scanner with risk-scored JSON/human output, smoke recommendations, or rollback hints.

## What This Changes

Add a native upgrade compatibility scanner, proposed as `doctor --upgrade-scan`, that checks schema and behavior drift before and after updates and reports risk, smoke recommendations, and rollback hints.

## Best Possible Solution

Define a stable core upgrade-scan contract that composes config validation, doctor health contributions, update dry-run planning, plugin diagnostics, health/status probes, backup/rollback guidance, redacted JSON output, and documentation.

## Reproduction Assessment

Not applicable; this is a feature request rather than a current-main bug. Source, docs, live issue state, and related-item inspection give high confidence that the requested scanner/report contract is absent while adjacent primitives exist.

## Solution Assessment

Unclear until maintainers choose the CLI/API surface, JSON schema, risk model, smoke criteria, rollback boundary, and redaction rules. The requested native scanner fits the problem, but it should be designed as a stable core contract rather than a one-off doctor flag.
