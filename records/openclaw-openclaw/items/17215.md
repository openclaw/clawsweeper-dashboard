## Summary

Keep open. Current main has adjacent update dry-run, post-update doctor/plugin sync, backup, rollback docs, and plugin compatibility diagnostics, but it still does not implement the requested native pre/post-update compatibility scanner with a risk-scored JSON/human report, smoke recommendations, or rollback hints.

## What This Changes

Add a built-in upgrade compatibility scanner, likely through `doctor --upgrade-scan`, with pre/post-update checks, risk reporting, smoke recommendations, and rollback guidance.

## Best Possible Solution

Define a stable core `openclaw update` and/or `openclaw doctor --upgrade-scan --json` contract that composes existing config validation, doctor checks, plugin diagnostics, update dry-run planning, backup, health, and rollback guidance, then document and test it.
