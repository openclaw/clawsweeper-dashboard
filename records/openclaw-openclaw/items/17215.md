## Summary

Keep open. Current main has adjacent update dry-run, post-update doctor repair, plugin sync and integrity reporting, plugin compatibility notices, backup docs, and manual rollback guidance, but it still lacks the requested native pre/post-update compatibility scanner with risk-scored JSON and human output, smoke recommendations, and rollback hints.

## What This Changes

The issue requests a built-in upgrade scanner, likely under doctor or update tooling, that checks compatibility before and after updates and reports risks, smoke recommendations, and rollback hints.

## Best Possible Solution

Define a stable core upgrade-scan contract that composes config validation, doctor health checks, update dry-run planning, plugin diagnostics, health/status probes, backup and rollback guidance, redacted JSON output, and docs.

## Reproduction Assessment

Not applicable; this is a feature request rather than a current-main bug. Source and issue-context checks show adjacent update and doctor pieces exist, but the requested scanner/report surface is absent.

## Solution Assessment

Unclear until maintainers choose the CLI/API surface, JSON schema, risk model, smoke criteria, rollback boundary, and redaction rules. A native scanner fits the operational safety goal, but it should be designed as a stable core contract rather than a one-off flag.
