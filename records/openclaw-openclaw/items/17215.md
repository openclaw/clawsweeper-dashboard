## Summary

Keep open. Current main has adjacent update dry-run, post-update doctor/repair, gateway health/version restart checks, plugin sync and integrity reporting, backup docs, and manual rollback guidance, but it still lacks the requested native pre/post-update compatibility scanner with risk-scored JSON/human output, smoke recommendations, and rollback hints.

## What This Changes

Add a native pre/post-update compatibility scanner, likely through `doctor --upgrade-scan`, with schema/deprecation checks, smoke recommendations, risk-scored reports, and rollback hints.

## Best Possible Solution

Define a stable core upgrade-scan contract that composes config validation, doctor health contributions, update dry-run planning, plugin diagnostics, health/status probes, backup/rollback guidance, redacted JSON output, and documentation.

## Reproduction Assessment

Not applicable; this is a feature request rather than a current-main bug. Source, docs, live issue state, and related-item inspection give high confidence that the requested scanner/report contract is absent while adjacent primitives exist.

## Solution Assessment

Unclear until maintainers choose the CLI/API surface, JSON schema, risk model, smoke criteria, rollback boundary, and redaction rules. A native scanner fits the problem, but it should be a stable core contract rather than a one-off doctor flag.
