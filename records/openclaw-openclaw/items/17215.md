## Summary

Keep open. Current main has adjacent update dry-run, post-core plugin convergence, static plugin payload smoke checks, doctor repairs, update JSON output, and rollback guidance, but it still lacks the requested native pre/post-update compatibility scanner with risk-scored JSON/human reporting, smoke recommendations, and rollback recommendation hints.

## What This Changes

The issue requests a built-in `doctor --upgrade-scan` style pre/post-update compatibility scanner with schema/deprecation checks, smoke checks, risk reporting, and rollback guidance.

## Best Possible Solution

Define a stable core upgrade-scan contract that composes config validation, doctor health checks, update dry-run planning, plugin diagnostics, gateway/preflight probes, backup/rollback guidance, redacted JSON output, human output, tests, and docs.
