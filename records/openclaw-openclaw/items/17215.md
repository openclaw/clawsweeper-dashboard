## What This Changes

Add a native `doctor --upgrade-scan` style pre/post-update compatibility scanner with schema/deprecation checks, smoke recommendations, risk-scored JSON/human reports, and rollback hints.

## Best Possible Solution

Define one stable core upgrade-scan contract that composes doctor lint/config validation, update dry-run planning, post-core plugin convergence, gateway or preflight checks, smoke recommendations, rollback guidance, redacted JSON, human output, tests, and docs.
