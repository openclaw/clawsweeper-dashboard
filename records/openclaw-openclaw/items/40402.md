## Summary

Keep open: current main still lacks `--profile-id` on `models auth login`. The adjacent OpenAI Codex automatic profile-id bug is fixed, but the explicit operator-named OAuth profile feature remains pending and is already covered by open implementation PR https://github.com/openclaw/openclaw/pull/49315.

## What This Changes

The issue requests a `--profile-id` option for `openclaw models auth login` so OAuth auth profiles can be explicitly named.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/49315 or an equivalent narrow change that wires explicit named-profile login through CLI parsing, auth result persistence, docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for a new CLI flag. Source inspection on current main confirms `models auth login` lacks `--profile-id` while `paste-token` has it.

## Solution Assessment

Yes: a narrow `models auth login --profile-id` option with single-profile remapping, validation, docs, and tests is the maintainable direction. The current open PR is the right vehicle, but it needs maintainer review and rebase over recent auth-command changes.
