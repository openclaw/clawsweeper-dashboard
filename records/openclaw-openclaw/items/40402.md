## What This Changes

Add a `--profile-id` option to `openclaw models auth login` so OAuth login can persist a caller-chosen auth profile ID.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/49315 with a provider-auth contract that passes a typed requested profile target before provider auth writes credentials/config, rejects ambiguous methods, updates docs/tests, and includes real behavior proof.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for a new CLI flag. Source inspection confirms current main lacks `--profile-id` on `models auth login`, while `paste-token` has that flag.

## Solution Assessment

Yes in direction, but the feature is not implemented on main. The maintainable solution should pass an explicit target through the provider-auth contract before persistence, or reject auth methods that cannot safely honor it.
