## Summary

Keep open: current `main` still does not expose `--profile-id` on `models auth login`. The adjacent OpenAI Codex automatic profile naming path is improved, but the explicit operator-named OAuth profile feature remains pending and is already covered by open implementation PR https://github.com/openclaw/openclaw/pull/49315.

## What This Changes

The issue requests a `--profile-id` flag for `openclaw models auth login` so OAuth profile names can be chosen explicitly.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/49315 or an equivalent narrow change that wires explicit named-profile login through CLI parsing, auth result persistence, docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for a new CLI flag. Source inspection on current `main` does confirm the requested flag is absent from `models auth login` while present on `paste-token`.

## Solution Assessment

Yes, a narrow `models auth login --profile-id` option with single-profile remapping, validation, docs, and tests is the right implementation shape. The current open PR is the best vehicle, but it needs maintainer review and a careful rebase before merge.
