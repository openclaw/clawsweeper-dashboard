## What This Changes

Request adds a `--profile-id` CLI flag to `models auth login` so OAuth logins can persist under operator-chosen auth profile IDs.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/49315 with a provider-auth contract change that threads a typed target before provider credential/config writes, rejects ambiguous methods, and updates docs/tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a CLI feature request. Source inspection confirms current main lacks `--profile-id` on `models auth login`, while `paste-token` has that flag.

## Solution Assessment

Yes in direction, but the feature is not implemented on main. The maintainable solution should pass an explicit target through the provider-auth contract before persistence or reject auth methods that cannot safely honor it.
