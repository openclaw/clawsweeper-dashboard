## Summary

Keep open: current `main` still lacks an explicit `--profile-id` path for `models auth login`. The adjacent OpenAI Codex profile-collapse bug appears improved by email/stable-id derivation, but the requested operator-controlled naming feature remains pending and is already represented by open PR #49315.

## What This Changes

Add a `--profile-id <id>` option to `openclaw models auth login` so OAuth login can persist a user-chosen auth profile id.

## Best Possible Solution

Land #49315 or an equivalent narrow change that wires an explicit named-profile option through CLI parsing, provider auth execution, persistence, docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for a new CLI flag. Source inspection does confirm the requested flag is absent from current `models auth login` while present for `paste-token`.

## Solution Assessment

Yes, a narrow login profile-id option with validation and docs is the right shape, but the maintainer should settle the exact CLI semantics and use #49315 as the current implementation vehicle.
