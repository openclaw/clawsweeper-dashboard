## Summary

Keep open: current main still lacks a `--profile-id` or equivalent login profile override, and open PR #49315 is the active implementation path rather than a reason to close this issue.

## What This Changes

Add an operator-specified `--profile-id` flag to `openclaw models auth login` so OAuth logins can be stored under a chosen auth profile id.

## Best Possible Solution

Review and land a narrow implementation like #49315 that adds named-profile login support for unambiguous single-profile auth results, with tests and Codex OAuth docs, while preserving the existing identity-derived default behavior.

## Solution Assessment
