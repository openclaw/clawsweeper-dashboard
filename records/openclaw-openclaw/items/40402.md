## Summary

Keep open: current main still lacks the requested `models auth login --profile-id` flag. The adjacent OpenAI Codex automatic profile naming bug has been improved separately, but the explicit operator-named OAuth profile feature remains pending and is already covered by the open implementation PR https://github.com/openclaw/openclaw/pull/49315.

## What This Changes

Request to add a `--profile-id` flag to `openclaw models auth login` so OAuth logins can persist under operator-chosen auth profile IDs.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/49315 or an equivalent narrow change that adds the login flag, remaps only unambiguous single-profile auth results, and updates docs, changelog, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for a new CLI flag. Source inspection on current main confirms the flag is absent from `models auth login` and present only on `paste-token`.

## Solution Assessment

Yes, a command-layer `models auth login --profile-id` option with single-profile remapping is the narrow maintainable shape. The open PR is the right vehicle, but it should be reviewed and rebased before merge.
