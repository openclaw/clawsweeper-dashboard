## Summary

Keep open: current main and the latest release still lack `--profile-id` on `models auth login`. The discussion correctly separates the earlier Codex identity/profile bug from this remaining operator-controlled naming feature, and the active implementation path is still the open PR at https://github.com/openclaw/openclaw/pull/49315.

## What This Changes

Request to add a `--profile-id` CLI flag to `openclaw models auth login` so OAuth login can persist a caller-selected auth profile id.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/49315 with a provider-auth contract change that adds `--profile-id`, threads the target before provider side effects where supported, rejects ambiguous methods, and updates docs/tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a CLI feature request. Current-main source inspection confirms `models auth login` lacks `--profile-id`, while `paste-token` still has that flag.

## Solution Assessment

Yes in direction, but it is not implemented on main. The safest implementation should pass a typed target through provider auth before credentials/config are written, or reject methods that cannot safely honor it.
