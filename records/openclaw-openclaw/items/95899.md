## Summary

Keep open. The PR is a useful partial fix for the restart-recovery no-retype issue, but it is not merge-ready because it turns a fail-closed assistant-tail branch into automatic agent execution using only prompt text for side-effect idempotency, embeds recovered user text inside a system-styled prompt without a robust untrusted-data wrapper, and still lacks successful configured-gateway proof.

## What This Changes

The PR changes main-session restart recovery to recover inspectable assistant-tail transcripts by finding the last user message, parsing content-block formats, expanding transcript lookup, and adding a prompt-level idempotency guard.

## Best Possible Solution

Keep assistant-tail recovery behind a maintainer-approved safety contract: wrap recovered text as untrusted data, default-gate or confirm side-effecting tails, and require configured-gateway proof before merge.
