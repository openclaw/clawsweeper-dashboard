## Summary

Keep open: the PR is a useful partial implementation for the linked restart-recovery issue, but it is not merge-ready because it changes a fail-closed recovery branch into automatic agent execution, still embeds recovered user text as prompt instructions, and lacks successful configured-gateway proof.

## What This Changes

The PR changes main-session restart recovery to auto-resume inspectable assistant-tail transcripts by extracting the last user message, parsing structured content blocks, expanding the transcript lookup window, and adding a prompt-level idempotency guard.

## Best Possible Solution

Keep assistant-tail recovery behind an explicit maintainer-approved safety contract: use an untrusted-data prompt wrapper, add confirm-to-continue or default fail-closed behavior for pending tool-call tails, and require configured-gateway proof before merge.
