## Summary

Keep open. Current main still scopes Control UI gateway tokens by exact gateway URL, so the PR is not obsolete and it usefully carries forward the focused #43254 loopback-auth work while keeping #69647 separate. The proposed patch is not merge-ready because its central test and changelog claim model cross-host dashboard navigation with one shared `sessionStorage`, but browser `sessionStorage` is origin- and tab-partitioned; it also expands bearer-token reuse across all numeric `127/8` loopback addresses and the branch is currently dirty and not maintainer-modifiable.

## What This Changes

The PR adds Control UI token-scope normalization that maps `localhost`, IPv4 `127/8`, and bracketed IPv6 loopback gateway hostnames to one `sessionStorage` key, adds loopback sharing/isolation tests, and adds a changelog entry.

## Best Possible Solution

Keep the focused loopback-token work alive, but choose one maintainable direction: either narrow the implementation, tests, and changelog to same-origin `gatewayUrl` aliasing, or design a canonical-host redirect/server-mediated handoff for real cross-origin dashboard-link reuse. Preserve protocol, port, and path isolation, document the final token scope, and add tests that model browser storage partitioning accurately.

## Reproduction Assessment

Yes for the review finding: the PR test changes `location.host` while retaining one mocked `sessionStorage`, and the browser storage contract partitions `sessionStorage` by origin and tab. Current main's exact-host token scope is directly visible in `ui/src/ui/storage.ts`.

## Solution Assessment

No. The storage-key normalization is only a narrow fit for same-origin `gatewayUrl` aliases; it is not the best complete solution for cross-host dashboard links, and the all-`127/8` credential-scope expansion needs maintainer approval or narrowing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Do not model host changes with one sessionStorage:** `ui/src/ui/storage.node.test.ts:284-293`
  - body: This test switches `location.host` from `127.0.0.1` to `localhost` and `[::1]` while keeping the same mocked `sessionStorage`. Browsers partition `sessionStorage` by page origin and tab, so a token saved under the `127.0.0.1` page origin will not be available after navigating the page to `localhost` or `[::1]`. Please narrow the claim/test to same-origin `gatewayUrl` aliasing or add a real cross-origin handoff/canonical-host design before claiming dashboard-link reuse.
  - confidence: 0.86
