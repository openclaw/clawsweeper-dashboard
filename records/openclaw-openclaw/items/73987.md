## Summary

Keep open. Current `main` has not implemented loopback token-scope sharing, but this PR should not merge as-is because the positive test models cross-origin `sessionStorage` reuse and the bearer-token reuse boundary still needs explicit maintainer/security approval.

## What This Changes

The PR adds loopback host normalization to Control UI gateway token/session storage keys, expands storage tests, and adds a changelog fix entry.

## Best Possible Solution

Define the local Control UI credential boundary, then land focused storage-key normalization with browser-origin-accurate tests and docs/changelog wording that describes only the supported flow.

## Reproduction Assessment

Yes for source-level exact-key fragmentation: current `main` keys tokens by parsed protocol, host, and path, so same-origin gateway URL alias edits create separate keys. No for cross-origin dashboard-link reuse, because real `sessionStorage` is partitioned by origin and tab. ([developer.mozilla.org](https://developer.mozilla.org/en-US/docs/Web/API/Window/sessionStorage?lang=en-US&utm_source=openai))

## Solution Assessment

No. The PR touches the right module, but the accepted solution should either prove same-origin gateway URL alias reuse or implement an explicit handoff design, and maintainer/security approval is needed before broadening token reuse.

## Review Findings

- **[P2] Keep loopback token tests within one browser origin:** `ui/src/ui/storage.node.test.ts:281-287`
  - body: This test changes `location.host` from `127.0.0.1` to `localhost`, another 127/8 address, and `[::1]` while keeping the same mocked `sessionStorage`.
