## Summary

Keep this PR open because current main still scopes Control UI gateway tokens by exact parsed host, so the loopback-alias auth fragmentation remains. The branch is not merge-ready: GitHub reports it as conflicting, it still edits release-owned CHANGELOG.md, and it lacks real behavior proof for the browser/session-storage path.

## What This Changes

The PR changes Control UI storage so localhost, 127/8, and bracketed IPv6 loopback gateway aliases share the same scoped token while preserving protocol, port, and path isolation.

## Best Possible Solution

Rebase or replace this branch on current main, keep the precise loopback-only storage normalization with focused tests, remove the CHANGELOG.md edit, and merge only after redacted real browser or terminal proof shows the intended token reuse and isolation boundaries.

## Reproduction Assessment

Yes. Current main source shows session token keys are derived from the exact parsed gateway host, so saving a token under `ws://127.0.0.1:18789` will not satisfy `ws://localhost:18789` unless this helper changes.

## Solution Assessment

Mostly yes. The storage normalizer is the right boundary because both token lookup and URL-edit token reuse already centralize there, but this exact PR needs a current-main refresh, changelog removal, and real behavior proof before it is a mergeable fix.
