## Summary

Keep open. Current main still keys Control UI gateway tokens by the exact parsed host, so this PR is not obsolete, but the patch is not merge-ready: the added test overclaims cross-host dashboard reuse while retaining one mocked sessionStorage, and the auth-token scope expansion across loopback aliases needs maintainer approval. The linked #43254 context is useful source work, while #69647 tracks a different reverse-proxy path issue.

## What This Changes

The PR adds Control UI loopback host normalization for gateway token/storage scopes, adds storage tests for reuse and isolation, and adds a changelog entry crediting the earlier #43254 work.

## Best Possible Solution

Keep the focused loopback-token work alive, but define the intended scope first: if the target is same-origin `gatewayUrl` alias edits, narrow the tests/changelog to that behavior and approve or reduce the 127/8 mapping; if the target is true cross-host dashboard navigation, use a canonical host redirect or explicit handoff instead of relying on sessionStorage continuity.

## Reproduction Assessment

Yes. The PR finding is reproducible from the diff: the added test changes `location.host` while keeping one mocked `sessionStorage`, whereas current code/docs keep tokens in sessionStorage for the selected gateway URL and browser tab.

## Solution Assessment

No. Storage-key normalization is plausible for same-origin gateway URL edits, but it is not a complete solution for cross-host dashboard link reuse; that broader behavior needs a canonical-origin or handoff design, and 127/8 sharing needs explicit approval or narrowing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Do not claim host alias reuse with one sessionStorage:** `ui/src/ui/storage.node.test.ts:284-287`
  - body: This test switches `location.host` from `127.0.0.1` to `localhost` and `[::1]` while keeping the same mocked `sessionStorage`. Real browser sessionStorage is origin-partitioned, so a token saved before navigating to a different loopback host is not available after the host change. Please narrow this to same-origin `gatewayUrl` alias edits or add a real cross-origin handoff/canonical-host design before claiming dashboard link reuse.
  - confidence: 0.86
