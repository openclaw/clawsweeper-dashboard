## Summary

Keep this PR open, but not merge-ready. Current main and v2026.6.9 still scope Control UI tokens by exact gateway host, so the central loopback-fragmentation problem remains, but this branch broadens auth/session scope across all numeric 127/8 hosts, has browser-storage proof gaps, edits release-owned CHANGELOG.md, and is currently conflicting with maintainer edits disabled.

## What This Changes

The PR changes Control UI storage to normalize localhost, numeric 127/8, and bracketed IPv6 loopback gateway URLs to a shared session token/settings scope, adds storage tests, and edits CHANGELOG.md.

## Best Possible Solution

Land a refreshed or replacement Control UI storage fix that shares only maintainer-approved loopback aliases, preserves isolation for distinct local gateways, removes the changelog edit, and includes browser-aware proof for reuse and isolation.

## Reproduction Assessment

Yes, by source inspection. Current main and v2026.6.9 key Control UI tokens and sessions by exact parsed gateway host, so switching the selected gateway URL between localhost, 127.0.0.1, and [::1] changes the storage key.

## Solution Assessment

No as submitted. The storage normalizer is the right layer, but collapsing all numeric 127/8 hosts into one auth/session scope needs maintainer approval and browser-aware proof; the safer fix is a narrower, explicitly approved alias boundary.
