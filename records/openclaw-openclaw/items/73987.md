## Summary

Keep open. Current main still scopes Control UI session tokens by exact gateway host, so the PR is not obsolete and the #43254 context is useful. The patch is not merge-ready because its positive test and changelog claim cross-host loopback dashboard reuse while keeping one mocked sessionStorage object, and the bearer-token boundary change for 127/8/IPv6 loopback aliases needs maintainer product/security approval first.

## What This Changes

The PR adds Control UI gateway token-scope normalization for localhost, IPv4 127/8 literals, and bracketed IPv6 loopback aliases, plus storage tests and a changelog entry.

## Best Possible Solution

Define the intended product boundary first. If this is only meant to cover same-origin `gatewayUrl` alias edits, narrow the tests and changelog to that behavior and approve or reduce the 127/8 mapping; if it is meant to support true cross-host dashboard navigation, add an explicit canonical-origin redirect or token handoff design with docs.

## Reproduction Assessment

Yes. The review finding is reproducible from the diff and current docs/code: the test changes `location.host` while retaining a single mocked sessionStorage, but the documented token store is browser sessionStorage for the current tab and selected gateway URL.

## Solution Assessment

No. The normalization may be a maintainable fix for same-origin gateway URL alias edits, but it is not the best complete solution for cross-host dashboard link reuse; that needs an explicit browser-origin-aware design or narrower claims.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Do not model cross-host token reuse with one storage object:** `ui/src/ui/storage.node.test.ts:284-287`
  - body: This test switches `location.host` between `127.0.0.1`, `localhost`, `127.255.255.255`, and `[::1]` while keeping the same mocked `sessionStorage`. Real dashboard navigation across those hosts changes the page origin, so a token saved before the host change is not available from this storage object. Please narrow the test/changelog to same-origin `gatewayUrl` alias edits or add an explicit cross-origin handoff/canonical-host design before claiming dashboard link reuse.
  - confidence: 0.86
