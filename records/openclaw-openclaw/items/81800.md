## Summary

Keep open. This is a fresh beta.8 regression report with screenshots, and current `main` still has a plausible Control UI reconciliation path where the same external-channel assistant reply can arrive through both `session.message` history reload and `chat` final handling.

## What This Changes

The issue reports duplicate assistant replies in OpenClaw Control UI after sending messages through Telegram, while Telegram itself shows only one reply.

## Best Possible Solution

Keep this open and fix Control UI event reconciliation so an external Telegram reply shown through history and a matching chat final cannot render as two assistant bubbles, without changing Telegram delivery semantics.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. Source inspection gives a credible reproduction path: with Control UI idle on a Telegram session, a `session.message` reload and a `chat` final for the same persisted assistant reply can both update `chatMessages`.

## Solution Assessment

Unclear as an implementation because no patch is proposed. The safest fix direction is narrow Control UI dedupe/reconciliation around `session.message` and `chat` final ordering, not Telegram outbound send changes.
