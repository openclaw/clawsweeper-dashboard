## Summary

Keep open: current main has manual QQBot storage cleanup and an existing global `media.ttlHours` cleanup path, but it does not provide the requested QQBot-specific retention/default cleanup behavior and needs a maintainer scope decision.

## What This Changes

Request automatic QQBot download retention cleanup, optionally via `channels.qqbot.downloadRetentionDays`, so stale downloaded media does not accumulate.

## Best Possible Solution

Decide whether the supported answer is to document `media.ttlHours` in the QQBot docs, or add a plugin-owned retention/default cleanup path that also handles legacy QQBot download locations intentionally.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms the current paths and cleanup behavior: QQBot downloads media into the media tree, manual cleanup exists, and automatic cleanup only runs when the global media TTL is configured.

## Solution Assessment
