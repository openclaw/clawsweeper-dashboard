## Summary

Keep open: current main already has opt-in global media cleanup and a QQBot manual cleanup command, but QQBot docs still do not make the cleanup path discoverable enough to resolve the report.

## What This Changes

The issue asks for QQBot media downloads to be cleaned automatically or retained for a configurable period so stale files do not accumulate.

## Best Possible Solution

Document the existing QQBot cleanup path: `media.ttlHours` for opt-in automatic retention, `/bot-clear-storage` for manual deletion, and the current media root, while deferring per-channel or default deletion until maintainers choose that policy.

## Reproduction Assessment

Not applicable as a feature/docs request. Source inspection confirms the current behavior: QQBot stores current downloads in the shared media tree, manual cleanup exists, and automatic cleanup runs only when global `media.ttlHours` is configured.

## Solution Assessment

Yes, the docs-first direction is the best next step. It uses the existing global retention contract and manual QQBot command without introducing a new QQBot config key or a surprising default deletion policy.
