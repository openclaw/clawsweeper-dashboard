## Summary

Keep open: current main has opt-in global media cleanup and manual QQBot cleanup, but the QQBot docs still do not make that path discoverable and no linked PR owns the docs-first fix.

## What This Changes

The issue requests automatic retention cleanup for QQBot downloaded media so stale files do not accumulate.

## Best Possible Solution

Add a QQBot docs section that points operators to `media.ttlHours` for opt-in retention and `/bot-clear-storage` for manual cleanup, while deferring per-channel/default deletion until there is stronger user need.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms the relevant current behavior: QQBot writes to the shared media tree, manual cleanup exists, and automatic cleanup runs only when the global TTL is configured.

## Solution Assessment

Yes for the next change: a docs-first update is the narrowest maintainable solution because `media.ttlHours` already provides opt-in recursive cleanup, and adding QQBot-specific retention would create a new config surface before proving per-channel need.
