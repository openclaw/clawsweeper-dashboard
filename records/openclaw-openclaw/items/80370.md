## Summary

Keep open: current main does not have skill usage telemetry, but this PR is not merge-ready because the contributor proof is test-only and the latest head still has correctness blockers in the trusted telemetry emission path.

## What This Changes

The PR adds a `skill.used` diagnostic event for skill-file reads and skill command tool dispatch, exports it through Prometheus and OTEL, and updates docs, changelog, and tests.

## Best Possible Solution

Thread trusted skill-read and skill-command metadata through the existing hook context and command/full skill catalog, keep labels bounded, and land only with maintainer approval plus redacted real telemetry proof.

## Reproduction Assessment

Yes for the review findings: source inspection of PR head shows the second inline wrapper, params-derived command matching, and prompt-snapshot lookup. The requested telemetry itself is a feature, not a bug reproduction.

## Solution Assessment

No. The exporter shape is plausible, but the emission path should use trusted dispatch/read context and the command/full skill catalog instead of a second wrapper and model-controlled tool params.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Pass skill telemetry through the existing hook wrapper:** `src/auto-reply/reply/get-reply-inline-actions.ts:314-323`
  - body: `resolveSkillDispatchTools()` already returns tools from `createOpenClawTools()`, which wraps them with the before-tool-call hook by default. Wrapping the selected inline command tool again reruns policy and diagnostics for the same command; pass the skill telemetry context into the construction-time hook context instead.
  - confidence: 0.93
- **[P2] Require trusted dispatch context for command telemetry:** `src/agents/pi-tools.before-tool-call.ts:200-207`
  - body: This treats any successful wrapped tool call with `commandName` and `skillName` params as trusted skill usage. Those fields are ordinary tool arguments, so non-command tools can spoof `skill.used`; carry a dispatcher-owned marker or catalog entry instead.
  - confidence: 0.9
- **[P2] Use the command catalog for command skill telemetry:** `src/agents/pi-tools.before-tool-call.ts:227-231`
  - body: This resolves command telemetry only from `ctx.skillsSnapshot.resolvedSkills`, but snapshots contain prompt-visible skills while command specs include user-invocable skills. A hidden user-invocable command can dispatch successfully and never emit `skill.used`; use the dispatch catalog or full skill entries.
  - confidence: 0.9
