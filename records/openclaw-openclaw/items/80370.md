## Summary

Keep open: the latest PR head still has blocking telemetry correctness issues, and the external real-behavior proof is only targeted Vitest output rather than live or runtime telemetry evidence.

## What This Changes

The branch adds a `skill.used` diagnostic event, records skill reads and command-dispatched skill tools, exports it through diagnostics OTEL/Prometheus, and updates docs, changelog, and tests.

## Best Possible Solution

Route skill-used emission through explicit non-model-controlled read and command dispatch context, use a command-visible skill catalog for command telemetry, keep telemetry labels bounded, and land only after redacted live telemetry proof.

## Reproduction Assessment

Yes. Source inspection of the current PR head shows the duplicate wrapper, param-derived command telemetry, and prompt-visible-only snapshot paths without needing a live run.

## Solution Assessment

No. The exporter shape is reasonable, but the emission source should use explicit runtime command/read context rather than model/tool-controlled params and prompt-only snapshots.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Pass skill context through the existing hook wrapper:** `src/auto-reply/reply/get-reply-inline-actions.ts:323-332`
  - body: `createOpenClawTools()` already returns before-tool-call wrapped tools, so wrapping the selected inline tool again runs hook policy and tool execution diagnostics twice for command-dispatched skills. Pass the skill telemetry context into the existing wrapper instead of nesting another wrapper here.
  - confidence: 0.93
- **[P2] Trust command dispatch context instead of tool args:** `src/agents/pi-tools.before-tool-call.ts:217-225`
  - body: This treats any successful wrapped tool call carrying `commandName` and `skillName` params as command-dispatched skill usage. Those fields are model/tool-controlled on ordinary calls, so command telemetry should be gated on non-model-controlled inline dispatch context.
  - confidence: 0.9
- **[P2] Use command-visible skills for inline telemetry:** `src/auto-reply/reply/get-reply-inline-actions.ts:323-332`
  - body: The inline wrapper receives `targetSessionEntry.skillsSnapshot`, but that snapshot resolves only prompt-visible skills. Skills with `disable-model-invocation: true` can still run as slash commands, so command telemetry needs the command catalog or full runtime skill entry.
  - confidence: 0.9
