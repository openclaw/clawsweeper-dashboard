## Summary

Keep open: the latest head still has source-proven P2 telemetry correctness issues, and the external PR only supplies targeted test output instead of real after-fix telemetry proof.

## What This Changes

The PR adds a `skill.used` diagnostic event for skill file reads and command-dispatched skill tools, exports it via OTEL/Prometheus, and updates diagnostics docs, changelog, and tests.

## Best Possible Solution

Route skill-used emission through explicit non-model-controlled runtime read/command context, keep telemetry labels bounded, and land after redacted live telemetry proof.

## Reproduction Assessment

Yes. Source inspection of the PR head shows duplicate inline hook wrapping, command matching from tool params, and prompt-only snapshots that exclude command-only skills.

## Solution Assessment

No. The exporter shape is reasonable, but the emission source should use explicit runtime command/read context passed through the existing hook wrapper and a command-visible skill catalog.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Pass skill context through the existing hook wrapper:** `src/auto-reply/reply/get-reply-inline-actions.ts:323-332`
  - body: `createOpenClawTools()` already returns before-tool-call wrapped tools, so wrapping that tool again here makes inline skill commands run hook policy and emit tool execution diagnostics twice. Pass the skill telemetry context into the existing wrapper instead of nesting another one.
  - confidence: 0.93
- **[P2] Trust command dispatch context instead of tool args:** `src/agents/pi-tools.before-tool-call.ts:217-226`
  - body: This treats any successful tool call carrying `commandName` and `skillName` params as command-dispatched skill usage. Those fields are model/tool-controlled on ordinary calls, so command telemetry should be gated on non-model-controlled inline dispatch context.
  - confidence: 0.9
- **[P2] Use command-visible skills for inline telemetry:** `src/auto-reply/reply/get-reply-inline-actions.ts:323-332`
  - body: The wrapper receives `targetSessionEntry.skillsSnapshot`, but that snapshot is built from prompt-visible skills. Skills with `disable-model-invocation: true` can still run as slash commands, so use the command catalog or full runtime skill entry for command telemetry.
  - confidence: 0.9
