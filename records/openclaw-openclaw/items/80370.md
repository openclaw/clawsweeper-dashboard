The PR adds `skill.used` diagnostics for skill file reads and command-dispatched skill tools, exports them through Prometheus and OTEL, and updates docs, tests, and the changelog.

## Best Possible Solution

Rebase onto current main, thread trusted skill-read and command metadata through the existing dispatch/hook contexts and command/full skill catalog, keep labels bounded, then land only with maintainer approval plus redacted live telemetry proof.

Yes for the review findings: source inspection of PR head and current main shows the stale inline dispatch path, second hook wrapper, params-derived command matching, and command catalog versus prompt snapshot mismatch. This is a feature PR, not a current-main user bug reproduction.

## Solution Assessment

No. The exporter shape is plausible, but the implementation should preserve current inline dispatch policy and use trusted dispatch/read context plus the command/full skill catalog instead of a second wrapper and model-controlled params.

  - body: Current main routes skill `command-dispatch: tool` through `resolveSkillDispatchTools`, which applies profile/provider/global/agent/group/sender/sandbox/subagent/inherited policies before owner filtering.   - confidence: 0.92

  - body: `createOpenClawTools()` already returns tools wrapped with `wrapToolWithBeforeToolCallHook`. Wrapping the selected inline-command tool again reruns before-tool-call policy and duplicates tool diagnostics; pass the skill telemetry context into the construction-time hook context instead.
  - confidence: 0.93
- **[P2] Require trusted dispatch context for command telemetry:** `src/agents/pi-tools.before-tool-call.ts:196-203`
  - body: This recognizes command skill usage from generic `commandName` and `skillName` tool params after any successful wrapped tool call. Those fields can be ordinary tool arguments, so unrelated tools can spoof trusted `skill.used`; carry a trusted skill-command dispatch marker instead.
  - confidence: 0.9
- **[P2] Use the command catalog for command skill telemetry:** `src/agents/pi-tools.before-tool-call.ts:213-227`
  - body: Command specs include user-invocable skills, while `skillsSnapshot.resolvedSkills` only contains prompt-visible skills. A hidden but user-invocable command can dispatch successfully and never produce `skill.used`; look up the command's skill from the dispatch catalog or full skill entries.
  - confidence: 0.9
