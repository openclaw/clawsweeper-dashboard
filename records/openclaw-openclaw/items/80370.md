## Summary

Keep open: current main does not have the requested skill telemetry, but the PR still has blocking correctness issues in the emission path and the external proof is test-only.

## What This Changes

Adds a `skill.used` diagnostic event for skill reads and command-dispatched skill tools, exports it through Prometheus and OTEL, and updates diagnostics docs, changelog, and tests.

## Best Possible Solution

Thread trusted skill-read and command-dispatch metadata through the original hook context or a dedicated dispatch context, use command-catalog/full runtime skill metadata for command events, keep labels bounded, and land after redacted real telemetry proof.

## Reproduction Assessment

Yes for the review findings: source inspection of current main and PR head shows the existing construction-time wrapper, the PR’s second wrapper, the params-derived command match, and the prompt-visible snapshot lookup. I did not run live Gateway proof because this is a read-only review.

## Solution Assessment

No. The exporter shape is plausible, but the emission path should use trusted dispatch/read context and command-catalog metadata rather than a second wrapper and model/tool-controlled params.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Thread skill telemetry through the original hook context:** `src/auto-reply/reply/get-reply-inline-actions.ts:324-333`
  - body: `createOpenClawTools()` already returns tools wrapped with `wrapToolWithBeforeToolCallHook`, and this new wrapper wraps the selected inline command tool again. That reruns before-tool-call hooks and duplicates tool execution diagnostics; pass the skill context into the original creation-time hook instead.
  - confidence: 0.93
- **[P2] Require trusted command-dispatch context for telemetry:** `src/agents/pi-tools.before-tool-call.ts:196-203`
  - body: At this generic hook boundary, `commandName` and `skillName` are ordinary tool params. Any successful wrapped tool call with those keys can emit trusted `skill.used`; carry an internal dispatch marker or metadata instead of trusting params.
  - confidence: 0.9
- **[P2] Use command catalog metadata for command skill events:** `src/agents/pi-tools.before-tool-call.ts:223-227`
  - body: `resolvedSkills` comes from prompt-visible skills, while slash skill commands are built from eligible user-invocable entries. Prompt-hidden command skills can dispatch a tool but never emit `skill.used`; use the command spec or runtime entry as the telemetry source.
  - confidence: 0.9
