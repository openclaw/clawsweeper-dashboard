## Summary

Keep this PR open, but it is not merge-ready: the feature is still absent from current main, while the current patch still has correctness blockers in the skill telemetry emission path and only provides test-only proof.

## What This Changes

The PR adds a `skill.used` diagnostic event for skill file reads and command-dispatched skill tools, exports it through diagnostics-otel and diagnostics-prometheus, and updates docs, tests, and changelog.

## Best Possible Solution

Thread trusted skill-read and skill-command dispatch metadata through the existing hook context and command/full skill catalog, keep exporter labels bounded, and land only with maintainer approval plus redacted real telemetry proof.

## Reproduction Assessment

Yes for the review findings. Source inspection of current main and PR head shows the construction-time hook wrapper, the PR's second wrapper, params-derived command matching, and the prompt snapshot versus command catalog mismatch.

## Solution Assessment

No. The exporter shape is plausible, but the emission path should use trusted dispatch/read context and command-visible skill metadata instead of a second wrapper and model-controlled tool params.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Pass skill telemetry through the existing hook wrapper:** `src/auto-reply/reply/get-reply-inline-actions.ts:324-333`
  - body: `createOpenClawTools()` already returns tools wrapped with `wrapToolWithBeforeToolCallHook`. Wrapping the inline-command tool again reruns before-tool-call policy and duplicates tool diagnostics; pass the skill telemetry context into the construction-time hook context instead.
  - confidence: 0.93
- **[P2] Require trusted dispatch context for command telemetry:** `src/agents/pi-tools.before-tool-call.ts:196-203`
  - body: This identifies command skill usage from generic `commandName` and `skillName` tool params after any successful wrapped tool call. Those fields can be ordinary tool params, so non-command tools can spoof trusted `skill.used`; carry a trusted skill-command dispatch marker instead.
  - confidence: 0.9
- **[P2] Use the command catalog for command skill telemetry:** `src/agents/pi-tools.before-tool-call.ts:213-227`
  - body: Command specs include eligible `userInvocable` skills, while `skillsSnapshot.resolvedSkills` only contains prompt-visible skills. A hidden but user-invocable command can dispatch successfully and never produce `skill.used`; look up the command's skill from the dispatch catalog or full skill entries.
  - confidence: 0.9
