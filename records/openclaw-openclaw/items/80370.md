## Summary

Keep open. The telemetry feature is useful, but the current PR head still misses sandboxed skill reads and inline command-dispatched skills, and the external PR has only test-result proof instead of real runtime/exporter proof.

## What This Changes

Adds a `skill.used` diagnostic event for known skill reads and command-dispatched skills, Prometheus/OTEL export, docs/changelog entries, and targeted tests.

## Best Possible Solution

Thread the active per-run skill catalog into embedded and inline command tool hooks, keep exporter labels bounded, then land only after redacted live Prometheus, OTEL, terminal, or runtime-log proof.

## Reproduction Assessment

Yes. Source inspection of the PR head shows copied-workspace sandbox reads use a different skill catalog than the PR passes to tool hooks, and inline command dispatch lacks the snapshot required by the new command matcher.

## Solution Assessment

No. The exporter shape is reasonable, but the emission source needs to use the active production skill catalog for both read and command-dispatch paths before the PR is the narrow maintainable solution.
