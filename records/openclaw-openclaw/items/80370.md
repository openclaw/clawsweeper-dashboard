## Summary

Keep open: the feature is not implemented on current main, but the current branch still has source-proven correctness blockers and only test-based real behavior proof.

## What This Changes

The PR adds a `skill.used` diagnostic event for skill file reads and command-dispatched skill tools, then exports it through diagnostics-otel and diagnostics-prometheus with docs and tests.

## Best Possible Solution

Land a revised diagnostics contract that threads trusted skill-read and skill-command dispatch context through the existing hook path, uses bounded exporter labels, and includes maintainer approval plus redacted real telemetry proof.

## Reproduction Assessment

Yes for the review findings. Source inspection shows construction-time tool hook wrapping, the PR's second inline wrapper, params-derived command matching, and the prompt-snapshot versus command-catalog mismatch.

## Solution Assessment

No. The exporter direction is plausible, but the PR should use trusted dispatch/read context and the command/full skill catalog instead of a second wrapper and model-controlled tool params.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Pass skill telemetry through the existing hook wrapper:** `src/auto-reply/reply/get-reply-inline-actions.ts:324-333`
  - body: `createOpenClawTools()` already returns inline command tools wrapped with `wrapToolWithBeforeToolCallHook`. Wrapping the selected tool again reruns before-tool-call policy and duplicates tool diagnostics; pass the skill telemetry context into the construction-time hook context instead.
  - confidence: 0.93
- **[P2] Require trusted dispatch context for command telemetry:** `src/agents/pi-tools.before-tool-call.ts:196-203`
  - body: This classifies command skill usage from generic `commandName` and `skillName` tool params after any successful wrapped tool call. Those fields can be ordinary model-controlled tool params, so unrelated tools can spoof trusted `skill.used`; carry a trusted skill-command dispatch marker instead.
  - confidence: 0.9
- **[P2] Use the command catalog for command skill telemetry:** `src/agents/pi-tools.before-tool-call.ts:213-227`
  - body: Command specs include eligible user-invocable skills, while `skillsSnapshot.resolvedSkills` only contains prompt-visible skills. A hidden but user-invocable command can dispatch successfully and never produce `skill.used`; look up the command's skill from the dispatch catalog or full skill entries.
  - confidence: 0.9
