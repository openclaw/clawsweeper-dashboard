## Summary

Keep open: this is a useful diagnostics feature, but the latest diff still misses real production emission paths and the supplied proof is test-only.

## What This Changes

Adds `skill.used` diagnostics for known `SKILL.md` reads and command-dispatched skill tools, with Prometheus/OTEL exporters, docs, changelog, and tests.

## Best Possible Solution

Thread the active per-run skill catalogue into embedded tool hooks and inline command-dispatch hooks, keep labels bounded, then land with redacted live Prometheus or OTEL proof.

## Reproduction Assessment

Yes. Source inspection against the PR head shows the introduced gaps: sandboxed reads use the original snapshot while the prompt uses the effective workspace catalogue, and inline command dispatch lacks the snapshot required by the matcher.

## Solution Assessment

No. The exporter shape is reasonable, but the emission source needs to use the same production skill catalogue as the read and command-dispatch paths instead of only the direct test hook context.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use the sandbox skill catalogue for read telemetry:** `src/agents/pi-embedded-runner/run/attempt.ts:901`
  - body: In copied-workspace sandbox runs (`ro`/`none`), current main deliberately clears `skillsSnapshotForRun` and builds the prompt from `effectiveWorkspace` entries. Passing the original `params.skillsSnapshot` here means a model reading the sandbox `SKILL.md` path it was shown will not match the original snapshot path, so `skill.used` is missing for sandboxed reads. Thread a snapshot derived from the same run skill entries used for the prompt.
  - confidence: 0.88
- **[P2] Wire command telemetry into inline skill dispatch:** `src/agents/pi-tools.before-tool-call.ts:252-260`
  - body: The command matcher only emits when `ctx.skillsSnapshot` contains the skill, but production command-dispatched skills run through `createOpenClawTools(...)` in inline actions, whose hook context still has no skill snapshot. Even though that path passes `commandName` and `skillName` in tool args, the new event will not fire outside the direct hook test; pass the active skill snapshot into that hook context.
  - confidence: 0.9
