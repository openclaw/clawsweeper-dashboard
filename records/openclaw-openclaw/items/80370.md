Adds a `skill.used` diagnostics event for known `SKILL.md` reads and command-dispatched skill tools, with Prometheus/OTEL exports, docs, changelog, and focused tests.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

No. The exporter shape is reasonable, but the emission source needs to use the same active skill catalogue as the production read and command-dispatch paths before this is the right solution.
