## Summary

Keep open. The PR proposes a useful diagnostics feature, but the current patch still has blocking correctness issues in the emission path and the external proof remains test-only rather than a real telemetry run.

## What This Changes

Adds a `skill.used` diagnostic event for skill file reads and command-dispatched skill tools, exports it through Prometheus and OTEL, and updates docs, tests, and changelog.

## Best Possible Solution

Thread trusted skill-read and command-dispatch metadata through the existing hook context and command catalog/full skill entries, keep exporter labels bounded, and land only with maintainer approval plus redacted real telemetry proof.

## Reproduction Assessment

Yes for the review findings. Source inspection of current main and the PR head shows the construction-time hook wrapper, the PR’s params-derived command matching, and the prompt-visible snapshot versus command-catalog mismatch.

## Solution Assessment

No. The exporter shape is plausible, but the emission path should use trusted dispatch/read context and command-visible skill metadata rather than a second wrapper and model-controlled tool params.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Pass skill context through the existing hook wrapper:** `src/auto-reply/reply/get-reply-inline-actions.ts:324-333`
  - body: `createOpenClawTools()` already returns before-tool-call wrapped tools on current main. Wrapping the selected inline command tool again can rerun hook policy, loop tracking, and tool execution diagnostics; thread the skill telemetry context into the creation-time hook context instead.
  - confidence: 0.93
- **[P2] Require trusted dispatch context for command telemetry:** `src/agents/pi-tools.before-tool-call.ts:196-227`
  - body: This treats a successful wrapped tool call with `commandName` and `skillName` params as skill command usage. Those fields are ordinary tool params at this generic hook boundary, so normal tool calls can spoof `skill.used`; carry a trusted dispatch marker or skill metadata from inline command dispatch instead.
  - confidence: 0.9
- **[P2] Use command-visible skill metadata for command telemetry:** `src/agents/pi-tools.before-tool-call.ts:213-227`
  - body: `resolvedSkills` contains prompt-visible skills, but command specs are built from eligible user-invocable entries. A hidden user-invocable skill command can dispatch its tool and still miss `skill.used`; use the command catalog or full runtime skill entry.
  - confidence: 0.9
