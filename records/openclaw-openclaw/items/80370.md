## Summary

Keep open: current main does not have skill usage telemetry, and the latest PR head still has source-level correctness blockers around command telemetry trust, command catalog coverage, and double-wrapping inline command tools. External real behavior proof is still test-only.

## What This Changes

The PR adds a core `skill.used` diagnostic event for skill reads and command-dispatched skill tools, exports it through Prometheus and OTEL, and updates docs, changelog, and tests.

## Best Possible Solution

Thread trusted skill-read and command-dispatch metadata through the existing hook context, use command-visible skill metadata for command tools, keep exporter labels bounded, and land only after redacted real telemetry proof.

## Reproduction Assessment

Yes for the review findings. Source inspection of current main and the PR head shows the existing construction-time hook wrapper, the PR’s second inline-command wrapper, and the params-derived command telemetry path.

## Solution Assessment

No. The exporter shape is plausible, but the emission path should use trusted dispatch/read context and command-catalog metadata rather than a second wrapper and model/tool-controlled params.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Pass skill context through the existing hook wrapper:** `src/auto-reply/reply/get-reply-inline-actions.ts:324-333`
  - body: `createOpenClawTools()` already returns before-tool-call wrapped tools on current main. Wrapping the selected inline command tool again can rerun hook policy, loop tracking, and tool execution diagnostics; thread the skill telemetry context into the creation-time hook context instead.
  - confidence: 0.93
- **[P2] Require trusted dispatch context for command telemetry:** `src/agents/pi-tools.before-tool-call.ts:196-227`
  - body: This treats a successful wrapped tool call with `commandName` and `skillName` params as skill command usage. Those fields are ordinary tool params at this generic hook boundary, so normal tool calls can spoof `skill.used`; carry a trusted dispatch marker or skill metadata from inline command dispatch instead.
  - confidence: 0.9
- **[P2] Use command-visible skill metadata for command telemetry:** `src/agents/pi-tools.before-tool-call.ts:213-227`
  - body: `resolvedSkills` contains prompt-visible skills, but command specs are built from eligible user-invocable entries. A hidden user-invocable skill command can dispatch its tool and still miss `skill.used`; use the command catalog or full runtime skill entry.
  - confidence: 0.9
