  - body: The PR accepts app-url Add-on tokens after issuer/email validation without comparing the token principal to account.config.appPrincipal. Current main and tests require that binding, so this weakens the Google Chat Add-on auth boundary.

  - body: For an allowlisted Space with no per-space users list, the PR can turn sender access into open access instead of preserving the current deny-when-empty behavior covered by tests. That allows every sender in that Space to trigger the agent.
  - confidence: 0.92
- **[P1] Match only the configured bot mention:** `extensions/googlechat/src/monitor-access.ts:108-110`
  - body: Treating any mentioned Google Chat user with type BOT as an OpenClaw mention lets requireMention pass when a message mentions a different bot in the same Space, causing unsolicited group replies.
  - confidence: 0.9
- **[P1] Namespace dedupe keys by account:** `extensions/googlechat/src/monitor.ts:175-183`
  - body: The dedupe cache is module-global but keyed only by message.name, so multi-account deployments can drop one account's webhook event after another account processes the same Google Chat message id. Include an account-unique component before skipping messages.
  - confidence: 0.87
- **[P2] Stop logging raw webhook payloads:** `extensions/googlechat/src/monitor-webhook.ts:90-112`
  - body: The parse-error branches dump raw webhook JSON, which can include Add-on systemIdToken values and user message content. Log sanitized metadata instead of body content.
  - confidence: 0.9
- **[P2] Handle added-to-space events without messages:** `extensions/googlechat/src/monitor.ts:100-101`
  - body: The parser can create ADDED_TO_SPACE events without a message, but processing returns before lifecycle handling can run. The claimed event support is therefore dropped for common add-to-space payloads.
  - confidence: 0.83
- **[P2] Commit dedupe only after confirmed delivery:** `extensions/googlechat/src/monitor.ts:379-381`
  - body: The PR marks a message processed after dispatch resolves, but Google Chat delivery can catch and log send/upload failures internally. Duplicate webhook retries can then be skipped even when no reply was delivered.
  - confidence: 0.84
