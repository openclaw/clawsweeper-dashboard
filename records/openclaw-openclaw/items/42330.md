- **[P1] Keep Add-on tokens bound to appPrincipal:** `extensions/googlechat/src/auth.ts:293-318`
  - body: The PR removes `expectedAddOnPrincipal` from Google Chat request verification, so app-url Add-on tokens are accepted for the configured audience without being bound to `account.config.appPrincipal`.   - confidence: 0.93
- **[P1] Preserve deny-when-empty sender policy:** `extensions/googlechat/src/monitor-access.ts:240-245`
  - body: For an allowlisted Space with no per-space users list, `senderGroupPolicy` becomes `open`, allowing every sender in that Space instead of preserving the current deny-when-empty allowlist behavior.
  - confidence: 0.92
- **[P1] Match only the configured bot mention:** `extensions/googlechat/src/monitor-access.ts:108-110`
  - body: Treating any mentioned Google Chat user with `type === "BOT"` as an OpenClaw mention lets `requireMention` pass when a message mentions a different bot in the same Space, which can trigger unsolicited replies.
  - confidence: 0.9
- **[P1] Namespace dedupe keys by account:** `extensions/googlechat/src/monitor.ts:175-183`
  - body: The dedupe cache is module-global but keyed only by `message.name`, so multi-account deployments can drop one account's event after another account processes the same Google Chat message id.
  - confidence: 0.87
- **[P1] Refresh JWKS before rejecting unknown kids:** `extensions/googlechat/src/auth.ts:269-272`
  - body: `manualVerifyGoogleJwt` rejects a cached JWKS `kid` miss immediately; during normal Google key rotation, valid app-url webhook tokens can fail until the ten-minute cache expires.
  - confidence: 0.84
- **[P2] Stop logging raw webhook payloads:** `extensions/googlechat/src/monitor-webhook.ts:90-112`
  - body: The parse-error branches log `JSON.stringify(raw)`, which can include Add-on `systemIdToken` values and user message content in production logs.
  - confidence: 0.9
- **[P2] Handle added-to-space events without messages:** `extensions/googlechat/src/monitor.ts:100-101`
  - body: The parser can create `ADDED_TO_SPACE` events without a message, but `processGoogleChatEvent` returns before lifecycle handling can run, so the new event support is effectively dropped.
  - confidence: 0.83
- **[P2] Commit dedupe only after confirmed delivery:** `extensions/googlechat/src/monitor.ts:379-381`
  - body: The PR marks a message processed after dispatch returns, but delivery can catch send/upload failures internally, so duplicate webhook retries can be skipped even when no reply was delivered.
  - confidence: 0.84
