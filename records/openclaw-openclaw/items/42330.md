- **[P1] Keep Add-on tokens bound to appPrincipal:** `extensions/googlechat/src/auth.ts:307-320`
  - body: The app-url branch verifies Google Workspace Add-on issuers without checking that the token subject matches the configured `appPrincipal`.   - confidence: 0.96
- **[P1] Restore local media loading for Google Chat sendMedia:** `extensions/googlechat/src/channel.ts:446-449`
  - body: The new `sendMedia` path always calls remote media fetch and drops the `mediaLocalRoots` branch, so local file payloads from workspace/tool outputs fail instead of being loaded through the existing local-media contract.
  - confidence: 0.94
- **[P1] Match only this bot's mention identity:** `extensions/googlechat/src/monitor-access.ts:107-110`
  - body: Treating any mentioned `BOT` user as OpenClaw being mentioned makes `requireMention` over-permissive in Spaces with multiple bots. A mention of a different bot can satisfy the gate and trigger OpenClaw unexpectedly.
  - confidence: 0.93
- **[P1] Do not change reply threading defaults without migration:** `src/channels/dock.ts:454`
  - body: Changing the fallback from `off` to `all` affects every existing Google Chat account without an explicit `replyToMode`, so upgrades can start threading replies by default. This needs maintainer product approval and upgrade coverage, or the new behavior should stay opt-in.
  - confidence: 0.9
- **[P1] Namespace dedupe keys by account:** `extensions/googlechat/src/monitor.ts:177-179`
  - body: The dedupe cache is module-global but checks only `message.name`, so multi-account deployments that observe the same Space message can cause one account to drop another account's event as a duplicate for the TTL.
  - confidence: 0.85
- **[P2] Stop logging raw Google Chat webhook payloads:** `extensions/googlechat/src/monitor-webhook.ts:88-111`
  - body: These parse-error logs serialize raw Google Chat request bodies. Add-on payloads can include `authorizationEventObject.systemIdToken` and message content, so malformed-request paths can leak bearer material or private chat data into logs.
  - confidence: 0.91
- **[P2] Handle add-to-space events without requiring a message:** `extensions/googlechat/src/monitor.ts:98-101`
  - body: The parser can create `ADDED_TO_SPACE` events without a message, but `processGoogleChatEvent` returns immediately for that shape. The claimed added-to-space support therefore drops common lifecycle callbacks before the pipeline can observe them.
  - confidence: 0.86
