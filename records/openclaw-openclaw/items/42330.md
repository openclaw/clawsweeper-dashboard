## Summary

Keep open: this PR is a plausible Google Chat bug-fix attempt, but it is not safe to merge because current source comparison shows security-sensitive auth/access regressions, unsafe dedupe behavior, missing real behavior proof, and related active PRs now cover major parts of the Space/threading problem more narrowly.

## What This Changes

The PR changes Google Chat Space/thread handling, webhook parsing, auth/token handling, inbound access policy, outbound media delivery, webhook deduplication, and the Google Chat reply-to default.

## Best Possible Solution

Split the useful work into focused current-main fixes for Space classification, thread routing, Add-on lifecycle parsing, and replay-safe dedupe while preserving appPrincipal binding, sender allowlists, bot-specific mention gating, sanitized logging, and explicit maintainer review for any reply-to default change.

## Solution Assessment

  - confidence: 0.9

  - confidence: 0.84
