- `impact:message-loss`: The failure can prevent Telegram final media replies from being delivered when the configured media cap should allow them.

## Summary

Keep open. Current main still has the reported Telegram reply-media cap gap, and an open implementation PR already uses closing syntax for this issue, so the issue should stay open until that PR is reviewed and resolved.

## What This Changes

Telegram final and native media replies should honor configured `channels.telegram.mediaMaxMb` by passing the computed byte cap into outbound media loading.

## Best Possible Solution

Review and land the open Telegram parameter-threading PR if it preserves `maxBytes` through final and native reply delivery with regression coverage.

## Reproduction Assessment

Yes. Configure `channels.telegram.mediaMaxMb`, trigger a Telegram final or native reply with `mediaUrl`/`mediaUrls`, and current main reaches `loadWebMedia` through `buildOutboundMediaLoadOptions` without `maxBytes`.

## Solution Assessment

Yes. Threading the already-computed `mediaMaxBytes` through the existing Telegram reply delivery options is the narrowest maintainable fix, and the linked open PR is the right place to review it.
