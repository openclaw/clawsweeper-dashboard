## Summary

Keep open. Current main still documents and computes Telegram mediaMaxMb, but the reply-delivery and native-command media paths call the outbound media loader without passing maxBytes; the related fix PRs are closed unmerged.

## What This Changes

Make Telegram final and native-command media replies honor configured channels.telegram.mediaMaxMb by propagating mediaMaxBytes into outbound media loading.

## Best Possible Solution

Land a narrow Telegram plugin fix that carries the already-computed mediaMaxBytes through final-reply and native-command delivery into deliverMediaReply, then passes it as maxBytes to buildOutboundMediaLoadOptions with regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows that Telegram final/native media replies can reach deliverReplies and deliverMediaReply, where buildOutboundMediaLoadOptions is called without maxBytes despite a configured mediaMaxMb.

## Solution Assessment

Yes. Threading the existing mediaMaxBytes value through the reply delivery options is the narrowest maintainable fix and does not require a new setting, default, or policy change.
