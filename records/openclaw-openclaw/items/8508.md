## Summary

Keep open: current main still exposes ack reactions as one configured/resolved string, and the requested list or contextual selector remains a feature/API design item rather than cleanup that can be closed.

## What This Changes

Request to support configurable lists or contextual/dynamic selection for ack reaction emojis instead of one fixed ack reaction value.

## Best Possible Solution

Preserve the existing `ackReaction` string contract, then add a backward-compatible array or selector contract after maintainers choose selection semantics and per-channel fallback rules.

## Solution Assessment
