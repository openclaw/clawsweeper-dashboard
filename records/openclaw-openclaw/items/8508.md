## What This Changes

## Best Possible Solution

Choose a backward-compatible shared selector contract, preserve the existing `ackReaction` string behavior, then update config types, schemas, docs, SDK/channel seams if needed, and bundled-channel tests together.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence absence proof because current main and v2026.6.8 have only the singular ack reaction config/schema/resolver path.

## Solution Assessment

Unclear until maintainers choose the product/API shape. A configured array or deterministic selector is plausible, while model-driven pre-ack choice may be the wrong layer because it adds latency before the acknowledgement is sent.
