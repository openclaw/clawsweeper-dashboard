## Summary

Keep open: current main and v2026.5.12 still expose acknowledgement reactions as a single resolved value, while this request asks for a new configurable list or contextual selector contract that needs maintainer product/API direction.

## What This Changes

Request to let agents use configurable lists or dynamic/contextual selection for acknowledgement reaction emojis instead of one fixed `ackReaction` value.

## Best Possible Solution

Preserve the existing `ackReaction` string contract, then add a backward-compatible `ackReactions` array or selector contract only after maintainers choose selection semantics and per-channel fallback rules.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken existing behavior. Source inspection gives high confidence that current main lacks the requested array or contextual selector contract.

## Solution Assessment

Unclear until maintainers choose deterministic, random, rule-based, or model-driven selection. The maintainable path is a backward-compatible config/API contract with per-channel validation, not a narrow bug fix.
