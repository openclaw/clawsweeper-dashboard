## Best Possible Solution

Preserve the existing `ackReaction` string contract, then add a backward-compatible `messages.ackReactions` array or selector/strategy contract with schema, docs, SDK/channel seams, and bundled-channel tests after maintainer API review.

## Solution Assessment

Unclear until maintainers choose the behavior model. The safest direction is a backward-compatible config/API expansion rather than changing existing `ackReaction` semantics in place.
