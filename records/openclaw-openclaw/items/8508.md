Keep open: current main still supports only a single ack reaction value, while list-based or contextual selection would add a new config/API surface that needs maintainer direction.

## What This Changes

Request support for configurable ack reaction emoji lists or contextual per-message ack reaction selection instead of one fixed value.

## Best Possible Solution

Design an additive ack-selection contract that preserves ackReaction while adding a maintained list or strategy surface with schemas, docs, SDK/channel behavior, and bundled-channel fallback tests.
