## Summary

Keep open: current main still omits `humanDelay` from Telegram block reply dispatch, and the open closing PR has not merged yet.

## What This Changes

Telegram block streaming should apply existing `agents.defaults.humanDelay` and per-agent `humanDelay` between coalesced block replies.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/69022, or an equivalent Telegram-only fix with focused coverage and live proof, merges.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: Telegram never supplies `dispatcherOptions.humanDelay`, while the shared dispatcher only sleeps between later block replies when that option is present.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing resolver and pass it into Telegram's dispatcher options, matching peer channels without adding a new config surface.
