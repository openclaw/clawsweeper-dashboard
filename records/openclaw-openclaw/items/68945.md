The issue asks for Telegram block-stream replies to respect configured agent `humanDelay` between later streamed blocks.

## Reproduction Assessment

Yes. Source inspection against current main shows Telegram never supplies `dispatcherOptions.humanDelay`, while the shared dispatcher only delays later block replies when that option is present.

## Solution Assessment

Yes. The maintainable fix is to reuse the existing `resolveHumanDelayConfig` helper and pass it into Telegram's dispatcher options, matching peer channels without adding a new config surface.
