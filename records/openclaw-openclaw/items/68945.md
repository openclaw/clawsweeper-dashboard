## What This Changes

Telegram streamed block replies should honor `agents.defaults.humanDelay` and per-agent `humanDelay` overrides between coalesced block messages.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/69022 after conflict resolution and live Telegram proof, or merge an equivalent Telegram-only patch with focused regression coverage.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main never passes `dispatcherOptions.humanDelay` in the Telegram block-stream path, while the shared dispatcher only sleeps between later block replies when that option is present.

## Solution Assessment

Yes. The narrow maintainable fix is to expose `resolveHumanDelayConfig` through the Telegram runtime barrel and pass it into `dispatcherOptions.humanDelay`, reusing the existing dispatcher contract instead of adding Telegram-specific delay logic.
