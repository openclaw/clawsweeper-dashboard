## Summary

Keep open: this remains a valid Telegram parity bug on current main, and the canonical closing PR is still open rather than merged.

## What This Changes

Telegram streamed block replies should honor `agents.defaults.humanDelay` and per-agent `humanDelay` overrides by pausing between coalesced block replies.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/69022 or an equivalent Telegram-only patch with focused regression coverage and redacted live Telegram proof showing configured inter-block pauses.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main never passes `dispatcherOptions.humanDelay` in the Telegram block-streaming path, while the shared dispatcher only sleeps between later block replies when that option is present.

## Solution Assessment

Yes. The narrow maintainable fix is to export the existing resolver through the Telegram local runtime barrel and pass it into `dispatcherOptions.humanDelay`, matching peer channels without adding a new config surface.
