## What This Changes

The issue reports that Telegram streamed block replies ignore configured humanDelay pacing between coalesced block messages.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/69022 or an equivalent Telegram-only patch with focused regression coverage and redacted live Telegram proof showing the configured inter-block delay.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: Telegram never supplies dispatcherOptions.humanDelay on current main, while the shared dispatcher only sleeps between later block replies when that option is present.

## Solution Assessment

Yes. The narrow maintainable fix is to export resolveHumanDelayConfig through the Telegram local runtime barrel and pass it into dispatcherOptions.humanDelay, reusing the existing shared dispatcher contract.
