## Summary

Current main already implements the user-visible `/new`/`/reset` duplicate reset-hook protection through the shared `resetHookTriggered` marker and fallback guard, and the PR’s extra process-wide dedupe is stale and unsafe to merge.

## What This Changes

The PR adds a bounded process-wide dedupe set around `emitResetCommandHooks` and tests for repeated `/new` or `/reset` hook emission with the same previous session.

## Best Possible Solution

Close this stale PR and keep the shipped `resetHookTriggered` fallback implementation; a new narrow issue should be opened only if a distinct current-main live re-entry path is observed.

## Reproduction Assessment

No. I did not establish a current-main failure path; current source and regression tests show the fallback reads the shared `resetHookTriggered` marker, and the shipped changelog documents dedupe protection for native `/new` and `/reset` flows.

## Solution Assessment

No. The PR targets the right historical symptom, but current main already fixes the observable duplicate fallback path, while this patch adds an after-await claim and process-wide state that can still duplicate under interleaving and can suppress later valid hooks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Claim reset-hook duplicates before awaiting:** `src/auto-reply/reply/commands-core.ts:104-107`
  - body: The new guard checks `recentResetHookEmissions` before the hook fires, but the key is not added until after `await triggerInternalHook`. Two re-entrant callers with the same prior session can both pass the guard and both run `command:new` or `command:reset`, so the duplicate hook emission this PR is meant to prevent can still happen.
  - confidence: 0.9
- **[P2] Scope duplicate state to one reset attempt:** `src/auto-reply/reply/commands-core.ts:32-33`
  - body: The module-level set is keyed only by action, session key, and previous session id, so a later legitimate reset using the same stable prior session can return early and skip both internal command hooks and `before_reset`. The claim needs a reset-attempt boundary instead of process-wide memory.
  - confidence: 0.86
