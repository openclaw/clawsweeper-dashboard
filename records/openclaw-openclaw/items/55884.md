## Summary

Keep open. The PR addresses a real user-visible reset-hook duplication problem, but the proposed patch still permits duplicate emission under concurrent re-entry and can suppress valid later reset hooks. Current main has per-command fallback protection, not the scoped prior-session dedupe needed for this bug class.

## What This Changes

This PR adds a bounded module-level reset-hook dedupe set in `commands-core.ts` keyed by action, session key, and previous session id, plus tests for duplicate suppression and changed prior-session allow-through.

## Best Possible Solution

Implement reset-hook duplicate suppression in the current `commands-reset-hooks.ts` path with a claim made before the first await and scoped to one reset attempt or another short-lived boundary, while preserving valid later `/new`, `/reset`, and `/reset soft` emissions.

## Reproduction Assessment

Yes. A focused unit test can hold `triggerInternalHook` pending and call `emitResetCommandHooks` twice with the same prior session to show a post-await claim double-fires; another test can repeat a valid later reset with the same prior session id to catch process-wide over-suppression.

## Solution Assessment

No. The PR targets the right bug class, but the maintainable fix should be in the current reset-hook module, claim duplicates before awaiting hook execution, and avoid persistent process-wide suppression keyed only by prior session id.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Claim reset-hook duplicates before awaiting:** `src/auto-reply/reply/commands-core.ts:105-107`
  - body: The new guard checks `recentResetHookEmissions` before `await triggerInternalHook`, but the key is not added until after that await. Two concurrent callers with the same prior session can both pass the guard and both fire `command:new`/`command:reset`, so duplicate memory artifacts remain possible. Claim the key before the first await or use an equivalent pre-await per-attempt claim.
  - confidence: 0.9
- **[P2] Scope reset-hook duplicate state to one attempt:** `src/auto-reply/reply/commands-core.ts:33`
  - body: The module-level `recentResetHookEmissions` set persists across unrelated commands and is keyed only by action, session key, and previous session id. A later valid reset with the same stable prior session id can return early and skip both internal command hooks and `before_reset`. Keep the duplicate claim scoped to the command/request or another short-lived reset attempt.
  - confidence: 0.86
