## Summary

Keep open. The PR addresses a real duplicate reset-hook problem, but the current patch is stale against main’s reset-hook split and still has two merge-blocking correctness defects in the dedupe mechanism.

## What This Changes

The PR adds a bounded module-level reset-hook dedupe set in `commands-core.ts` plus focused tests in `commands-core.test.ts` for repeated `/new` and `/reset` hook emissions.

## Best Possible Solution

Implement duplicate suppression in `commands-reset-hooks.ts` with a pre-await claim scoped to the current reset attempt, preserving valid later `/new`, `/reset`, and `/reset soft` hook emissions.

## Reproduction Assessment

Yes. A focused unit test can hold `triggerInternalHook` pending, call `emitResetCommandHooks` twice with the same action/session key/previous session, and assert current main emits twice while the fix emits once.

## Solution Assessment

No. The PR targets the right bug, but its module-level dedupe is registered after the first await and can suppress valid later resets; the safer fix belongs in the current reset-hook module with tighter lifetime.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Claim reset-hook duplicates before awaiting:** `src/auto-reply/reply/commands-core.ts:105-107`
  - body: The new guard checks the dedupe set before `await triggerInternalHook`, but the key is not added until after that await. Two callers with the same prior session can both pass the guard and both fire `command:new` or `command:reset`, so the duplicate hook emission remains possible.
  - confidence: 0.9
- **[P2] Scope duplicate state to one reset attempt:** `src/auto-reply/reply/commands-core.ts:32-33`
  - body: The module-level `recentResetHookEmissions` set persists across unrelated commands and is keyed only by action, normalized session key, and previous session id. A later valid reset using the same stable prior session id can return early and skip both internal command hooks and `before_reset`.
  - confidence: 0.86
