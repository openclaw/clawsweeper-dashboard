## Summary

Keep open: the PR is a focused fix for a valid WebChat feature request, but the new token color rules do not match the generated code-block DOM, so the visible syntax highlighting would not apply.

## What This Changes

The PR adds `highlight.js` as a direct Control UI dependency and updates the markdown renderer, tests, and component styles to emit syntax-highlighted WebChat code blocks.

## Best Possible Solution

Align the highlight CSS scope with the generated `.code-block-wrapper pre code.hljs` DOM or add a matching renderer class, then include redacted real WebChat proof that colors and copy behavior both work.

## Reproduction Assessment

Yes for the review finding: source inspection shows the PR emits `hljs` on `<code>` inside `.code-block-wrapper`, while the added selectors require a `.code-block` ancestor. I did not run a browser because this review is read-only.

## Solution Assessment
