## Best Possible Solution

## Solution Assessment

Yes, the proposed code path is the narrow maintainable fix: explicit Telegram env/config still wins, a process default that already prefers IPv4 is preserved, and Node 22 otherwise gets the Telegram `ipv4first` default. It should be paired with live transport proof before merge.
