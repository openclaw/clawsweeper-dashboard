## Summary

Keep open. Current main still does not implement `openclaw models auth clean`, so the PR is not obsolete, but the latest patch should not merge as-is because it regresses current `models auth --agent <id> login-github-copilot` behavior and still needs user-facing docs/changelog follow-through.

## What This Changes

The PR adds a `models auth clean` CLI command with dry-run/JSON/agent support, auth-profile sanitizer/store helpers, and regression tests for stale auth cleanup and profile display safety.

## Best Possible Solution

Rebase the PR on current main, keep the cleanup command if maintainers still want the explicit prune UX, preserve current per-agent GitHub Copilot auth semantics by resolving and passing `agentDir` or using the generic provider-auth path, then add docs/changelog coverage and rerun the targeted auth/model CLI tests plus the changed gate.

## Reproduction Assessment

Yes. Code inspection gives a high-confidence reproduction: after the PR, `openclaw models auth --agent worker login-github-copilot` reaches `githubCopilotLoginCommand` without an agent directory even though that helper writes to `opts.agentDir`, so the scoped login no longer targets the requested agent store.

## Solution Assessment

No. The cleanup command approach is plausible, and several earlier review threads appear addressed, but the latest patch is not the best mergeable solution until it preserves existing `--agent` behavior and documents the new command.
