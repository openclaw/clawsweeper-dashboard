## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Bypass runtime snapshots for main-store migration:** `src/commands/models/auth-clean.ts:267`
  - body: The main-agent pre-lock migration trigger still calls `ensureAuthProfileStore(agentDir, { readOnly: false })`. That helper checks runtime auth snapshots before loading from disk, so with a running gateway and only legacy `auth.json` on disk it can return the snapshot without writing `auth-profiles.json`; `updateAuthProfileStoreWithLock` then creates an empty placeholder and the cleanup removes nothing from the legacy store.
  - confidence: 0.91
- **[P2] Emit one JSON payload for successful cleanup:** `src/commands/models/auth-clean.ts:197-213`
  - body: In `--json` mode the command logs the plan object before writing and then logs `{ ok, removed }` after writing, so a successful cleanup prints two top-level JSON documents on stdout. CLI JSON mode should be one parseable object containing both the plan/result fields, otherwise scripts cannot `JSON.parse` the command output.
  - confidence: 0.95
- **[P3] Remove the unsupported force hint:** `src/commands/models/auth-clean.ts:173-175`
  - body: The empty-config dry-run JSON warning still tells users to pass `--force`, but `models auth clean` only registers `--agent`, `--dry-run`, and `--json`. Following the diagnostic therefore fails in the CLI parser instead of providing the advertised escape hatch.
  - confidence: 0.97
