## What This Changes

This PR adds `openclaw models auth clean` with dry-run, JSON, and agent flags, plus auth-store cleanup helpers, profile-id sanitization, tests, and a changelog entry.

## Best Possible Solution

Merge only after fixing the CLI contract/docs gaps and adding redacted real-run proof against a disposable auth store, while preserving the current locked-store reload semantics.

## Reproduction Assessment

Yes for the PR defects: PR-head source emits a plan JSON before the write and a result JSON after the write, and the test asserts two log entries. The cleanup command itself is a new feature, so there is no current-main bug reproduction path for the command.

## Solution Assessment

No. The `models auth` CLI may be the right surface, but this implementation should emit one stable JSON result, remove the impossible force hint, document the new command, and include real behavior proof before merge.
