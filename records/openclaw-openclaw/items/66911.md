## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Emit one JSON document for --json:** `src/commands/models/auth-clean.ts:197-213`
  - body: `--json` logs the cleanup plan before mutating and later logs a separate `{ ok, removed }` result after the write. Successful commands therefore write two top-level JSON values to stdout, which breaks scripts that parse the result as one JSON document; build and emit one stable payload after the write instead.
  - confidence: 0.92
- **[P3] Remove the nonexistent force hint:** `src/commands/models/auth-clean.ts:173-174`
  - body: The JSON dry-run warning tells users to pass `--force`, but the command does not define a `--force` option and the non-dry path has no force escape. Remove that remediation text or add a deliberate force flag so scripts do not surface an impossible next step.
  - confidence: 0.84
- **[P3] Document the clean subcommand:** `src/cli/models-cli.ts:413-419`
  - body: This registers a user-facing auth cleanup command, but the models CLI docs still list the existing auth commands without `clean` or its `--dry-run`, `--json`, and `--agent` behavior. Add the docs entry so the published CLI reference matches the new surface.
  - confidence: 0.87
