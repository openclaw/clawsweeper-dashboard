## Summary

Keep open. Current main still lacks the command, and the latest PR head remains useful but is not mergeable because successful `--json` cleanup writes two JSON documents and the public models CLI docs omit the new command.

## What This Changes

Adds `openclaw models auth clean` with stale auth-profile pruning, dry-run/JSON/agent flags, auth-store scoping and locking changes, sanitizer coverage, and a changelog entry.

## Best Possible Solution

Keep the cleanup command, but revise this PR so JSON mode emits one stable result payload and the user-facing models docs cover the new command before merge.

## Reproduction Assessment

Yes. Source inspection of PR head gives a high-confidence reproduction: `openclaw models auth clean --json` on a run that removes profiles logs the plan and then a separate result JSON object.

## Solution Assessment

No. The feature direction is reasonable, but the current patch should preserve the single-document CLI JSON contract and align docs with the new user-facing surface.
