## Summary

Keep open: current main does not implement the requested `models auth clean` command, and this recent external feature PR still has unique work. It is not merge-ready because `--json` can emit two top-level JSON documents, the dry-run warning mentions a nonexistent `--force` option, the public CLI docs are not updated, and real behavior proof is still missing.

## What This Changes

Adds an `openclaw models auth clean` subcommand with dry-run, JSON, and agent flags, plus auth-store loading changes, profile-id sanitization, tests, and a changelog entry.

## Best Possible Solution

If maintainers still want this standalone cleanup command after the shipped fixes for https://github.com/openclaw/openclaw/issues/41634, merge a version that preserves current auth-store contracts, emits one stable JSON result, removes impossible guidance, updates the public auth docs, and includes redacted real command-run proof.

## Reproduction Assessment

Yes for the PR defects: PR-head source shows `--json` logs a plan and then a separate result, and the PR test expects two log entries. The cleanup command itself is a new feature, so there is no current-main bug reproduction path for the command.

## Solution Assessment

No. The models auth CLI may be the right surface, but this implementation needs the JSON contract, impossible force hint, docs, and real behavior proof fixed before it is a maintainable merge candidate.

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Emit one JSON document for --json:** `src/commands/models/auth-clean.ts:197-213`
  - body: `--json` writes the cleanup plan before mutating and then writes a separate `{ ok, removed }` result after the locked update. A successful cleanup therefore sends two top-level JSON values to stdout, which breaks scripts that parse a single JSON document; build and emit one stable result payload instead.
  - confidence: 0.92

  - confidence: 0.84
- **[P3] Document the clean subcommand:** `src/cli/models-cli.ts:413-419`
  - body: This registers a user-facing auth cleanup command, but the public authentication docs still describe the auth subcommands without `clean` or its `--dry-run`, `--json`, and `--agent` behavior. Add the docs entry so the published CLI surface matches the new command.
  - confidence: 0.87
