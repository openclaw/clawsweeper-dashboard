## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection on the latest PR head gives a high-confidence path: a main-agent legacy `auth.json` store plus a runtime auth snapshot lets `ensureAuthProfileStore` skip disk migration before `updateAuthProfileStoreWithLock` creates an empty `auth-profiles.json`.

## Solution Assessment

Full review comments:

- **[P1] Load disk before creating the auth store placeholder:** `src/commands/models/auth-clean.ts:267`
  - body: For the main-agent write path, this call can return a runtime auth snapshot and skip loading `auth.json`.   - body: In `--json` mode a successful cleanup logs the plan here and then logs a second `{ ok, removed }` object after the write. That makes stdout two top-level JSON documents, which breaks callers that parse the command as a single JSON result; include the final status in one payload instead.
