## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Bypass runtime snapshots for main-store migration:** `src/commands/models/auth-clean.ts:267`
  - body: `models auth clean` intends this call to migrate legacy `auth.json` before the locked writer can create an empty `auth-profiles.json`, but `ensureAuthProfileStore` can return an existing runtime snapshot without loading disk. With a live snapshot and only legacy auth on disk, cleanup skips migration and the subsequent locked update sees an empty placeholder, so stale profiles remain while the command reports a no-op.
  - confidence: 0.91
- **[P2] Emit one JSON payload for successful cleanup:** `src/commands/models/auth-clean.ts:197-213`
  - body: In `--json` mode the command logs a plan object here, then after a successful write logs a second `{ ok, removed }` object. Consumers reading stdout as JSON will get two pretty-printed documents and fail to parse the result; emit one final object that includes both the plan and write result.
  - confidence: 0.88
- **[P3] Remove the unsupported force hint:** `src/commands/models/auth-clean.ts:174`
  - body: This warning tells users to pass `--force`, but `models auth clean` does not register a `--force` option, so following the diagnostic fails as an unknown option. Either implement the flag with tests or change the warning to the supported recovery path.
  - confidence: 0.94
