## Summary

Keep open. Current main still lacks the requested auth cleanup command, and the latest PR keeps useful fixes for earlier runtime-overlay and profile-id validation concerns, but it still has a blocking JSON output contract bug plus missing user docs.

## What This Changes

The PR adds an `openclaw models auth clean` CLI command with auth-store local loading, profile-id sanitization/validation, command registration, tests, and a changelog entry for pruning stale auth profile entries.

## Best Possible Solution

Keep the feature, but make JSON mode return one stable result payload, document the new command, fix the impossible `--force` hint, and rerun the targeted auth/model/docs checks before normal maintainer review.

## Reproduction Assessment

Yes. Static inspection of the latest PR head shows `modelsAuthCleanCommand({ json: true })` logs a plan JSON object and then a second result JSON object, and the PR test expects exactly two log entries.

## Solution Assessment

No. The feature direction and latest auth-store scoping look reasonable, but the current patch should preserve the one-document `--json` CLI contract and update public docs before merging.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Emit one JSON document for --json:** `src/commands/models/auth-clean.ts:197-213`
  - body: `--json` logs the cleanup plan before mutating and later logs a separate `{ ok, removed }` object after the write. A successful command therefore writes two top-level JSON values to stdout, which breaks scripts that parse the command result as one JSON document. Build and emit a single payload after the write, with dry-run and no-op paths using the same shape.
  - confidence: 0.92
- **[P3] Document the clean subcommand:** `src/cli/models-cli.ts:413-419`
  - body: This registers a user-facing auth cleanup command, but the models CLI docs still list only `add`, `login`, `setup-token`, and `paste-token`. Add `clean` plus its `--dry-run`, `--json`, and `--agent` behavior so the published CLI reference matches the new surface.
  - confidence: 0.87
- **[P3] Remove the nonexistent force hint:** `src/commands/models/auth-clean.ts:170-175`
  - body: The JSON dry-run warning tells users to pass `--force`, but the command does not define a `--force` option and the non-dry path has no force escape. Either add a deliberate force flag or remove that remediation text so scripts do not surface an impossible next step.
  - confidence: 0.84
