## Summary

Keep open. The latest patch is useful and appears to have fixed the earlier Copilot agent-scoping concern, but the new `--json` path emits two top-level JSON documents on successful mutation, which is not a mergeable CLI contract for scripting.

## What This Changes

The PR adds `openclaw models auth clean` with stale auth-profile pruning, dry-run/JSON/agent flags, auth-store locking/local-store helpers, sanitization, changelog coverage, and tests.

## Best Possible Solution

Keep the cleanup command, but make the JSON mode a single stable payload and add models CLI documentation before normal maintainer review or merge.

## Reproduction Assessment

Yes. Code inspection gives a high-confidence reproduction: `openclaw models auth clean --json` on a run that removes profiles logs the plan JSON and then a separate result JSON object.

## Solution Assessment

No. The feature direction is reasonable, but the current patch should be narrowed to preserve the single-document `--json` CLI contract and document the new command.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Emit one JSON document for --json:** `src/commands/models/auth-clean.ts:203-220`
  - body: `--json` logs the cleanup plan before mutating and then logs a second `{ ok, removed }` object after the write. A successful command therefore writes two top-level JSON documents to stdout, which breaks scripts expecting one parseable command result. Return a single object that includes the plan and result fields after the write, with the dry-run/no-op paths doing the same.
  - confidence: 0.9

  - body: This registers a user-facing auth cleanup command, but the models CLI docs still list only `add`, `login`, `setup-token`, and `paste-token`. Add `clean` plus its `--dry-run`, `--json`, and `--agent` behavior so the docs match the CLI.
  - confidence: 0.86
