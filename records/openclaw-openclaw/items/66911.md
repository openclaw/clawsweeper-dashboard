## Summary

Keep open. Current main still lacks the cleanup command and this PR is useful, but latest head still has merge-blocking CLI contract issues: non-dry `--json` emits two top-level JSON documents, the public models CLI docs omit the new command, and one warning suggests a nonexistent `--force` flag.

## What This Changes

Adds `openclaw models auth clean` to prune stale auth profiles with dry-run, JSON, agent scoping, auth-store locking, display sanitization, tests, and a changelog entry.

## Best Possible Solution

Keep the feature direction, but repair the JSON output contract, document the new CLI surface, and remove or deliberately implement the force flow before maintainer review.

## Reproduction Assessment

Yes. Static inspection of latest head shows `modelsAuthCleanCommand({ json: true })` logs a plan at `src/commands/models/auth-clean.ts:197` and a separate result at `src/commands/models/auth-clean.ts:359`, and the PR test expects two log entries.

## Solution Assessment

No. The command belongs in the models/auth CLI surface, but the current patch should preserve one JSON document per invocation and keep docs aligned with the user-facing command.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Emit one JSON document for --json:** `src/commands/models/auth-clean.ts:197-213`
  - body: `--json` logs the cleanup plan before mutating and later logs a separate `{ ok, removed }` result after the write. Successful commands therefore write two top-level JSON values to stdout, which breaks scripts that parse the result as one JSON document. Build and emit one stable payload after the write, with dry-run and no-op paths using the same shape.
  - confidence: 0.92
- **[P3] Document the clean subcommand:** `src/cli/models-cli.ts:413-419`
  - body: This registers a user-facing auth cleanup command, but the models CLI docs still list only `add`, `login`, `setup-token`, and `paste-token`. Add `clean` plus its `--dry-run`, `--json`, and `--agent` behavior so the published CLI reference matches the new surface.
  - confidence: 0.87
- **[P3] Remove the nonexistent force hint:** `src/commands/models/auth-clean.ts:173-174`
  - body: The JSON dry-run warning tells users to pass `--force`, but the command does not define a `--force` option and the non-dry path has no force escape. Either add a deliberate force flag or remove that remediation text so scripts do not surface an impossible next step.
  - confidence: 0.84
