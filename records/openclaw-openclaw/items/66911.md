## Summary

Keep open. The PR is still useful because current main does not have `models auth clean`, but the latest head is not mergeable: it drops the current documented `--agent` scoping for `login-github-copilot` and still lacks the required user-facing docs/changelog coverage for a new CLI command.

## What This Changes

The PR adds `openclaw models auth clean` with stale auth-profile pruning, dry-run/JSON/agent support, sanitizer and store helpers, and related tests.

## Best Possible Solution

Keep the cleanup command work, but preserve the current per-agent auth contract by resolving the requested agent and passing `agentDir` to the Copilot helper or by keeping the generic provider-auth path, then add docs/changelog coverage for `models auth clean` and rerun the targeted auth/model CLI tests plus the changed gate.

## Reproduction Assessment

Yes. Code inspection gives a high-confidence reproduction: `openclaw models auth --agent poe login-github-copilot --yes` reaches the PR’s direct `githubCopilotLoginCommand` call without `agentDir`, while that helper writes credentials using `opts.agentDir`.

## Solution Assessment

No. The cleanup command direction is plausible, but the current patch is not the best mergeable solution because it bypasses the established scoped provider-auth path and leaves the new user-facing command undocumented.
