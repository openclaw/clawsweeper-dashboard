Full review comments:

- **[P2] Emit one JSON document for --json:** `src/commands/models/auth-clean.ts:197-213`
  - body: When `--json` performs a real cleanup, this logs the plan object and later logs a second `{ ok, removed }` object, so stdout is not a single JSON value and cannot be piped directly into `jq` or other JSON parsers. Return one final payload that includes the plan and result instead of emitting two top-level documents.
  - confidence: 0.94
- **[P2] Bypass runtime snapshots before locked cleanup:** `src/commands/models/auth-clean.ts:267`
  - body: The main-agent migration trigger still calls `ensureAuthProfileStore`, which can return a runtime snapshot before reading disk. With a live gateway snapshot and only legacy `auth.json` needing migration, this can skip the intended write-enabled migration before the locked update creates/loads an empty `auth-profiles.json`, leaving the cleanup unable to remove the legacy stale profiles.
  - confidence: 0.86
- **[P3] Remove the unsupported --force guidance:** `src/commands/models/auth-clean.ts:173-174`
  - body: The dry-run JSON warning tells users to pass `--force`, but the command registration only exposes `--agent`, `--dry-run`, and `--json`. Either implement and test `--force` or remove that guidance so users are not sent to a nonexistent flag.
  - confidence: 0.96
- **[P3] Document the new auth cleanup command:** `src/cli/models-cli.ts:432-438`
  - body: This adds a user-facing auth command with deletion semantics, but the public models CLI docs still list the existing auth commands without `models auth clean` or its safety guard behavior. Add the command, flags, and cleanup warning to the models CLI docs before exposing it.
  - confidence: 0.82
