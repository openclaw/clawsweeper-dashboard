## What This Changes

This PR adds `openclaw models auth clean` with dry-run, JSON, and per-agent cleanup support, plus auth-profile display sanitization, store-loader changes, tests, and a changelog entry.

## Best Possible Solution

Land the command after the migration trigger uses a disk-backed write-enabled load that bypasses runtime snapshots, JSON mode emits one stable result object, and the empty-config guard text matches supported CLI flags.

## Reproduction Assessment

Yes. Source inspection on the PR head gives a high-confidence path: a main-agent legacy `auth.json` store plus a runtime auth snapshot lets `ensureAuthProfileStore` skip disk migration before the locked writer creates `auth-profiles.json`.

## Solution Assessment

No. The command direction is reasonable, but the branch should use a disk-backed migration trigger, emit one JSON object, and remove the unsupported force hint before merge.

Full review comments:

- **[P1] Load disk before creating the auth store placeholder:** `src/commands/models/auth-clean.ts:267`
  - body: For the main-agent write path, this call can return a runtime auth snapshot and skip loading `auth.json`.   - confidence: 0.93
- **[P2] Emit one JSON payload for successful cleanup:** `src/commands/models/auth-clean.ts:197-213`
  - body: In `--json` mode a successful cleanup logs the plan here and then logs a second `{ ok, removed }` object after the write. That makes stdout two top-level JSON documents, which breaks callers that parse the command as a single JSON result.
  - confidence: 0.91

  - body: This warning tells users to pass `--force`, but the CLI command does not define that option. Users following the machine-readable hint will get an unknown-option failure, so the text should be removed or the flag should be implemented with tests.
  - confidence: 0.9
