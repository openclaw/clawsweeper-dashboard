## Summary

## What This Changes

This PR adds `openclaw models auth clean` with dry-run/JSON/agent support, auth-profile display sanitization and validation, agent-local auth-store loading, file-locked cleanup, a changelog entry, and tests.

## Best Possible Solution

Merge only after the command emits one stable JSON result, uses supported safety guidance, documents the public CLI behavior, preserves disk-backed locked writes, and includes redacted real-run proof of an actual stale-profile prune.

## Reproduction Assessment

Yes for the PR defects: PR-head source and tests show successful `--json` cleanup emits two logged JSON values, and CLI registration lacks the `--force` flag named by the warning. The cleanup command itself is a new feature, so there is no current-main bug reproduction path for the command.

## Solution Assessment

No as-is. The CLI surface is plausible, but the implementation needs a single parseable JSON contract, accurate operator guidance, public docs, and stronger prune proof before it is the maintainable fix.
