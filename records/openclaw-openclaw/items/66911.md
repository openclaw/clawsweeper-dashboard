## What This Changes

The PR adds `openclaw models auth clean` for pruning stale auth profiles, plus scoped auth-store loading, profile-id sanitization/validation, tests, and a changelog entry.

## Best Possible Solution

Merge only after the command emits a single stable JSON result, removes the unsupported `--force` guidance, documents the public auth-clean workflow, and includes redacted real-run proof of an actual stale-profile prune.

## Reproduction Assessment

Yes for the PR defects: PR-head source shows successful `--json` cleanup emits two top-level JSON values, and the warning names a `--force` flag that is not registered. The cleanup command itself is a new feature, so there is no current-main bug reproduction path for that command.

## Solution Assessment

No as-is. The command shape is plausible, but the implementation needs a single parseable JSON contract, accurate operator guidance, public docs, and stronger prune proof before it is the maintainable solution.
