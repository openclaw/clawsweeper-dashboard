## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Emit one JSON document for --json:** `src/commands/models/auth-clean.ts:197-213`
  - body: `--json` writes the cleanup plan before mutating and then writes a separate `{ ok, removed }` result after the locked update. A successful cleanup therefore sends two top-level JSON values to stdout, which breaks scripts that parse a single JSON document; build and emit one stable result payload instead.
  - confidence: 0.93
- **[P3] Remove the nonexistent force hint:** `src/commands/models/auth-clean.ts:173-174`
  - body: The dry-run JSON warning tells users to pass `--force`, but the command registers only `--agent`, `--dry-run`, and `--json`, and the non-dry path still throws. Remove that remediation text or add an intentional force option so clients do not surface an impossible next step.
  - confidence: 0.88
- **[P3] Document the clean subcommand:** `src/cli/models-cli.ts:432-438`
  - body: This registers a user-facing auth cleanup command, but the public models/auth CLI docs and command index still omit `clean` and its `--dry-run`, `--json`, and `--agent` behavior. Add the docs entry so the published CLI surface matches the new command.
  - confidence: 0.9
- **[P2] Keep custom main-agent cleanup scoped to its store:** `src/commands/models/auth-clean.ts:133-135`
  - body: When the literal `main` agent has a configured custom `agentDir`, this path still calls `loadAuthProfileStoreWithoutExternalProfiles(agentDir)`. That helper treats an explicit path outside the process-default main path as secondary and merges the default main store, so the cleanup plan can include profiles from the wrong auth store; use a local persisted load for this explicit target or make the helper config-aware.
  - confidence: 0.83
