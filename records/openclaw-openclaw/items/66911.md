## Summary

Keep open. The feature is still useful and current main does not implement it, but the latest PR head still breaks the `--json` CLI contract and omits the new command from the published models docs.

## What This Changes

The branch adds `openclaw models auth clean`, auth-profile sanitizing/store helpers, CLI registration, changelog coverage, and tests for pruning stale auth profiles across agent stores.

## Best Possible Solution

Keep the feature direction, but make `--json` emit one stable result payload and add user-facing docs for `clean` before normal maintainer review or merge.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence reproduction: run `modelsAuthCleanCommand({ json: true })` with at least one removable profile and it logs the plan JSON, then logs a separate result JSON object after the write.

## Solution Assessment

No. The cleanup command is a reasonable direction, but the current patch should preserve the one-document `--json` contract and update the models CLI docs before merging.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Emit one JSON document for --json:** `src/commands/models/auth-clean.ts:197-213`
  - body: `--json` logs the cleanup plan before mutating and later logs a second `{ ok, removed }` object after the write. A successful command therefore writes two top-level JSON values to stdout, which breaks scripts that parse the command result as one JSON document. Build and emit a single payload after the write, with dry-run and no-op paths using the same shape.
  - confidence: 0.92
- **[P3] Document the clean subcommand:** `src/cli/models-cli.ts:413-419`
  - body: This registers a user-facing auth cleanup command, but the models CLI docs still list only `add`, `login`, `setup-token`, and `paste-token`. Add `clean` plus its `--dry-run`, `--json`, and `--agent` behavior so the published CLI reference matches the new surface.
  - confidence: 0.87
