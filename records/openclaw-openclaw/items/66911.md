## Summary

Keep open: the latest head still adds a useful missing command, but it should not merge while `--json` emits two top-level documents, docs omit the new command, and a warning suggests a nonexistent `--force` flag.

## What This Changes

The PR adds a `models auth clean` command, auth profile ID sanitization/validation helpers, disk-local auth-store loading for cleanup, tests, and a changelog entry.

## Best Possible Solution

Keep the feature, but make JSON mode return one stable result payload, document the new command in CLI docs, and remove or deliberately implement the force flow before maintainer review.

## Reproduction Assessment

Yes. Static inspection of latest head shows `modelsAuthCleanCommand({ json: true })` logs a plan object and then a separate result object, and the PR test expects exactly two log entries.

## Solution Assessment

No. The command direction fits the CLI/auth surface, but the current patch should preserve the one-document JSON CLI contract and update the public docs before merging.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Emit one JSON document for --json:** `src/commands/models/auth-clean.ts:197-213`
  - body: `--json` logs the cleanup plan before mutating and later logs a separate `{ ok, removed }` object after the write. A successful command therefore writes two top-level JSON values to stdout, which breaks scripts that parse the command result as one JSON document. Build and emit a single payload after the write, with dry-run and no-op paths using the same shape.
  - confidence: 0.92
- **[P3] Document the clean subcommand:** `src/cli/models-cli.ts:413-419`
  - body: This registers a user-facing auth cleanup command, but the models CLI docs still list only `add`, `login`, `setup-token`, and `paste-token`. Add `clean` plus its `--dry-run`, `--json`, and `--agent` behavior so the published CLI reference matches the new surface.
  - confidence: 0.87
- **[P3] Remove the nonexistent force hint:** `src/commands/models/auth-clean.ts:173-174`
  - body: The JSON dry-run warning tells users to pass `--force`, but the command does not define a `--force` option and the non-dry path has no force escape. Either add a deliberate force flag or remove that remediation text so scripts do not surface an impossible next step.
  - confidence: 0.84
