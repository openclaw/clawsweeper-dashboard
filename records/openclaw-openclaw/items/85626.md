## Summary

Keep open: current `main` still has the reported forced-runtime asymmetry, and an open fix PR now references this issue with closing syntax, so the right cleanup path is to review and land or reject that PR rather than close the issue early.

## What This Changes

The issue requests that resumed sessions with forced CLI runtimes such as `claude-cli` avoid `MissingAgentHarnessError` when session/auth bindings are re-resolved.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/85627 or an equivalent focused fix that keeps CLI runtime policy out of the missing-plugin-harness failure path without changing the public plugin API.

## Reproduction Assessment

Yes, at source level: current `main` can still route a configured `claude-cli` runtime into `selectAgentHarnessDecision`, where it reaches the missing-harness throw instead of the CLI-runtime bypass used by model fallback. I did not execute the live auth-rotation session flow in this read-only review.

## Solution Assessment

Yes, the narrow fix direction is to make the harness selection boundary honor the existing CLI-runtime contract or keep CLI runtimes out of that boundary. The open PR is the immediate candidate to review against that shape.
