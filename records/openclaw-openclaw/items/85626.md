## Summary

Keep open: current main and the latest release still show the reported resolver asymmetry, and the linked fixing PR is open with closing syntax, so this issue should stay open until that PR lands or maintainers choose a different fix.

## What This Changes

The issue reports resumed CLI-backed sessions throwing `MissingAgentHarnessError` because `selectAgentHarnessDecision` treats valid CLI runtime ids such as `claude-cli` as missing embedded harnesses.

## Best Possible Solution

Review and land or intentionally close https://github.com/openclaw/openclaw/pull/85627; the end state should reuse the CLI-runtime classification in harness selection while preserving fail-closed behavior for real missing plugin harness ids.

## Reproduction Assessment

Yes by source inspection: a forced `claude-cli` runtime entering `selectAgentHarnessDecision` reaches `MissingAgentHarnessError` because only embedded/plugin harness ids are accepted. I did not execute the full live resumed-session auth-rotation flow.

## Solution Assessment

Yes, the narrow fix direction is to share or mirror CLI-runtime detection in the selection path while keeping non-CLI missing harness ids fail-closed. The issue should remain open until the linked PR merges or maintainers pick another implementation.
