## Summary

Keep open: current main still has the reported CLI-runtime versus harness-selection asymmetry, and the linked fix PR is open and unmerged, so this issue should remain the tracking item until that PR lands or is rejected.

## What This Changes

The issue reports that resumed `claude-cli` sessions can throw `MissingAgentHarnessError` because `selectAgentHarnessDecision` treats forced CLI runtime ids as missing plugin harnesses.

## Best Possible Solution

Review and land the linked narrow resolver fix, with focused regression coverage and redacted real resumed-session proof, while preserving fail-closed behavior for genuinely missing non-CLI harness ids.

## Reproduction Assessment

Yes for the core failure path by source inspection: a forced `claude-cli` runtime entering `selectAgentHarnessDecision` reaches `MissingAgentHarnessError` because only embedded/plugin harness ids are accepted. I did not execute the full resumed-session auth-rotation flow in this read-only review.

## Solution Assessment

Yes, the maintainable fix is to reuse the existing CLI-runtime classification in harness selection so valid CLI backend ids bypass embedded harness lookup consistently without weakening missing plugin-harness errors.
