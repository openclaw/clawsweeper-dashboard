## Summary

Keep open. Current main still lacks Feishu/Lark mixed-proxy docs and a Feishu-owned direct/ambient proxy policy, and same-author PR #75050 is still the active docs-only mitigation while runtime egress behavior needs maintainer review.

## What This Changes

Allow Feishu/Lark channel token and WebSocket traffic to bypass ambient proxy when needed while provider traffic keeps using proxy environment variables.

## Best Possible Solution

Land #75050 if docs-only NO_PROXY guidance is accepted; otherwise implement a Feishu-owned proxy policy covering SDK HTTP and WebSocket paths with tests and docs.

## Reproduction Assessment

Yes. Source inspection shows no Feishu proxy config key, no Feishu-owned SDK HTTP proxy override, and WebSocket handling that follows ambient proxy env; the specific Windows proxy error was not live-reproduced.

## Solution Assessment

Unclear. Docs-only guidance is the lowest-risk mitigation and already has PR #75050, while a proxyMode-style runtime option is more complete but changes channel egress policy and needs maintainer approval.
