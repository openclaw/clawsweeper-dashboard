Keep open: current main and v2026.6.10 still lack a Feishu/Lark mixed-proxy policy or docs mitigation, and the related open PRs cover partial, conflicting direct/ambient/docs directions that need maintainer and security review.

## What This Changes

Request a Feishu/Lark channel-specific proxy policy so provider traffic can keep using env proxy while Feishu/Lark token/API and WebSocket traffic can go direct or follow an explicit channel policy.

## Best Possible Solution

Settle one Feishu-owned egress policy covering SDK REST/token HTTP, WebSocket bootstrap/socket traffic, guarded-fetch call sites, docs, and focused tests before merging either docs-only or runtime proxy changes.

## Solution Assessment

Unclear until maintainers choose the egress policy. Docs-only NO_PROXY guidance is the lowest-risk mitigation, while a runtime proxyMode or default change needs product and security approval.
