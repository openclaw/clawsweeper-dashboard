## What This Changes

Request a Feishu/Lark channel proxy policy or troubleshooting guidance so Feishu traffic can bypass ambient proxy settings while provider traffic keeps using them.

## Best Possible Solution

Review and land #75050 if NO_PROXY documentation is the accepted mitigation; otherwise design a Feishu-owned proxy policy that covers SDK HTTP, guarded fetch token requests, and WebSocket setup with tests and docs.
