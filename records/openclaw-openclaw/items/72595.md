## Summary

Keep open. Current main still lacks Feishu/Lark mixed-proxy docs and a Feishu-owned per-channel direct/ambient proxy policy, and the same-author docs mitigation at https://github.com/openclaw/openclaw/pull/75050 remains open while runtime egress behavior needs maintainer judgment.

## What This Changes

Request Feishu/Lark channel guidance or support for routing Feishu HTTP and WebSocket traffic independently from provider traffic that uses ambient proxy environment variables.

## Best Possible Solution

Land the docs-only NO_PROXY mitigation if maintainers accept that path, and handle any runtime change as an explicit Feishu-owned proxy policy covering SDK HTTP, guarded fetch, and WebSocket paths with tests and documentation.

## Reproduction Assessment

Unclear. The docs PR is the narrowest low-risk mitigation, while a runtime proxyMode-style option is more complete but needs maintainer approval because it changes channel egress behavior.
