## Summary

Keep open. Current main still lacks a Feishu/Lark per-channel proxy policy and mixed-proxy docs, while the paired docs mitigation at https://github.com/openclaw/openclaw/pull/75050 and runtime proxy work around https://github.com/openclaw/openclaw/issues/65799 are still open and need maintainer policy review.

## What This Changes

Request a Feishu/Lark channel proxy policy or NO_PROXY guidance so provider traffic can keep ambient proxy env while Feishu token and WebSocket traffic can bypass it when required.

## Best Possible Solution

Resolve the docs mitigation if maintainers accept it, then land one Feishu-owned proxy policy covering Lark SDK HTTP, guarded-fetch HTTP, WebSocket, managed proxy behavior, tests, and docs.

## Solution Assessment

Unclear. NO_PROXY documentation is the narrowest mitigation, while a runtime proxyMode/default-direct policy is more complete but needs maintainer approval because it changes channel egress and compatibility behavior.
