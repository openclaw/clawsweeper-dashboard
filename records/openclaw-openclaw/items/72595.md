## Summary

Keep open. Current main still lacks Feishu/Lark mixed-proxy troubleshooting docs and a Feishu-owned direct/ambient proxy policy, while same-author PR #75050 is an active docs mitigation rather than a merged fix.

## What This Changes

Request Feishu/Lark channel traffic to bypass or explicitly control ambient proxy use while model-provider traffic keeps using env proxy settings.

## Best Possible Solution

Review and land #75050 if docs-only NO_PROXY guidance is accepted; otherwise implement a Feishu-owned proxy policy covering SDK HTTP, streaming-card fetches, and WSClient with focused tests and docs.

## Reproduction Assessment

Yes. The report gives a concrete Windows mixed-proxy setup, proxy env vars, failing tenant_access_token endpoint, and error; source inspection confirms separate Feishu HTTP, guarded-fetch, and WebSocket paths without one shared policy.

## Solution Assessment

Unclear. Docs-only NO_PROXY guidance is the lowest-risk mitigation and already has PR #75050, while a proxyMode-style runtime option is more complete but needs maintainer approval because it changes egress behavior.
