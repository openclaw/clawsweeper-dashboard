## Summary

Keep open. Current main still lacks Feishu/Lark mixed-proxy docs on the shipped page and lacks a Feishu-owned direct/ambient policy spanning SDK HTTP, guarded fetch, and WebSocket paths; the paired same-author docs mitigation remains open while runtime egress behavior needs maintainer judgment.

The issue requests Feishu/Lark channel proxy guidance or a per-channel proxy policy so provider traffic can keep using ambient proxy while Feishu HTTP and WebSocket traffic can bypass it when needed.

## Best Possible Solution

Land the docs-only NO_PROXY mitigation if maintainers accept it, then handle runtime behavior as an explicit Feishu-owned proxy policy covering SDK HTTP, guarded fetch, and WebSocket paths with tests and documentation.
