## Summary

## What This Changes

The PR changes isolated cron finalization to skip the fatal structured-error pre-dispatch branch when later payload text appears after the last error payload.

## Best Possible Solution

Move the recovery decision into `resolveCronPayloadOutcome` so the helper clears fatal state and selects the successful delivery payload before finalization decides whether dispatch is allowed.

## Solution Assessment

No. The proposed check is in the wrong layer: it runs after the helper has already chosen fatal state and fatal delivery payloads, while the best fix is to make the helper classify the recovered output correctly before finalization.
