This PR adds a recovery check in `src/cron/isolated-agent/run.ts` so cron delivery can continue when a later non-error payload follows an earlier error payload.

## Best Possible Solution

Move recovery into `resolveCronPayloadOutcome` before fatal state and delivery payloads are selected, preserve true fatal structured-error suppression, add focused regression coverage, and require redacted scheduled-run or delivery-attempt proof before merge.

## Reproduction Assessment

## Solution Assessment

No. The bug should be solved in `resolveCronPayloadOutcome` before fatal state and delivery payloads are selected; this PR bypasses the later guard after the helper has already chosen the fatal error payload.
