## Summary

Keep open: the PR targets a real doctor migration regression and includes useful live before/after proof, but the new guard can suppress canonical runtime synthesis in agent scopes where the legacy model-map entry is not actually renamed, so one routing edge should be fixed before merge.

## What This Changes

The PR adds a doctor regression test and a guard that skips Codex runtime pin synthesis when a legacy `openai-codex/*` model entry already has an explicit non-default `agentRuntime.id`.

## Best Possible Solution

Carry explicit non-default legacy runtime pins into the canonical model entry wherever a rewritten model can resolve to them, while keeping `auto` and `default` pins overwritable for Codex route preservation.

## Reproduction Assessment

Yes. The current source ordering and merge behavior reproduce the reported overwrite path with high confidence, and the PR body includes before/after live output for the exact default-agent config; I did not run tests because this review is read-only.

## Solution Assessment

No, not quite. The approach is right for the reported default-agent case, but it should also preserve or copy the explicit pin when the legacy map entry is used to make the decision but will not be renamed afterward.
