## What This Changes

The PR skips requester-session wake/steering for Discord completion delivery, suppresses ACP session identity notices when `session_info_update` is explicitly disabled, and adds focused tests.

## Best Possible Solution

Keep the narrow Discord delivery fix, but gate the ACP identity notice through `isAcpTagVisible(resolveAcpProjectionSettings(cfg), "session_info_update")`, strengthen the regression test, and require redacted real Discord/ACP proof before merge.

## Reproduction Assessment

## Solution Assessment
