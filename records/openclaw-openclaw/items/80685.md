This PR should stay open: the Discord ACP delivery fix is plausible, but the ACP session notice suppression is incomplete against the existing default visibility policy, and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR changes Discord subagent completion delivery to bypass requester-session wake/steering for Discord targets and adds ACP dispatch handling/tests for suppressing resolved session identity notices when `session_info_update` visibility is explicitly disabled.

## Best Possible Solution

Keep a narrow fix that preserves Discord ACP thread delivery while applying the same resolved ACP tag-visibility policy used by the projector, with focused regression tests and redacted live proof from a real Discord/ACP run.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. The source path is clear enough to review: current main unconditionally emits the identity notice and tries requester-session wake before direct completion delivery, but the PR provides tests rather than real Discord/ACP proof.

## Solution Assessment

No, not yet. The Discord direct-delivery direction looks plausible, but the `session_info_update` suppression should use resolved ACP tag-visibility defaults rather than checking only an explicit `false`, and the PR needs after-fix real behavior proof before merge.
