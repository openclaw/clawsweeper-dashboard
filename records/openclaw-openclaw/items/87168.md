## Summary

Keep open: current main still leaves the Codex image media-understanding route out of the automatic image provider order, while direct OpenAI remains prioritized and can later fail on OAuth-only installs that lack direct API-key auth. The protected `maintainer` label also blocks cleanup closure, and the linked fix attempt is closed unmerged rather than a viable owner for the remaining work.

## What This Changes

The issue requests auth-aware image media-understanding auto-selection so Codex/ChatGPT OAuth-only installs prefer the bounded Codex route instead of direct OpenAI when no direct `OPENAI_API_KEY` is configured.

## Best Possible Solution

Keep this issue open until maintainers approve and land an auth-aware media-understanding default that prefers bounded Codex image routing for Codex OAuth-only installs while preserving explicit direct OpenAI choices.

## Reproduction Assessment

Yes from source and reporter logs, but not from a live Codex OAuth run in this read-only pass. Current main still builds image auto candidates from the manifest auto-priority list that omits Codex while OpenAI remains prioritized, and direct OpenAI image execution later requires API-key auth.

## Solution Assessment

Unclear as an implementation choice: auth-aware media routing is the right direction, but the exact default and compatibility behavior needs maintainer approval. A narrow solution should prefer Codex only when Codex auth is available and direct OpenAI API-key auth is absent, while keeping provider-qualified direct OpenAI choices explicit.
