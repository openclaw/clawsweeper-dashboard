## What This Changes

The issue reports that the built-in `image` media-understanding tool can route to direct OpenAI instead of the configured Codex image route when overrides or auto-selection are used on Codex-OAuth-only installs.

## Best Possible Solution

Keep the issue open for a maintainer-owned fix that makes `image` override and auto-selection routing auth-aware, preserves configured `agents.defaults.imageModel`, and returns actionable errors when a requested direct provider lacks compatible auth.

## Reproduction Assessment

No live high-confidence reproduction was run here. The current source path is reproducible from inspection: the `image` tool accepts `model`, passes it into image fallback routing, and auto provider metadata still prioritizes direct OpenAI before the Codex-capable metadata path.

## Solution Assessment

No, the local prompt workaround is not the best fix. The maintainable solution should live in runtime routing/error handling, with tests for configured Codex image routes, provider-prefixed overrides, bare overrides, and unauthenticated direct OpenAI selection.
