## Best Possible Solution

Review and either land or replace https://github.com/openclaw/openclaw/pull/92824 with an auth-aware implicit media routing fix that preserves explicit direct OpenAI choices.

## Reproduction Assessment

Yes from source inspection and reporter logs: current main can admit direct OpenAI media candidates through broad OpenAI provider auth, while direct OpenAI image execution calls `requireApiKey`. I did not run a fresh live OAuth-only install in this read-only review.

## Solution Assessment

Unclear as a final implementation: auth-aware implicit routing appears to be the right boundary, but the linked PR is still draft and needs maintainer acceptance for the exact image/PDF fallback policy.
