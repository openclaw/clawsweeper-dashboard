## Summary

Keep open, but the patch should not merge as-is because it removes the documented command-sensitive node approval gate for exec and non-exec node command surfaces instead of solving the automation credential path.

## What This Changes

The branch changes node pairing authorization and tests so every node approval requires only `operator.pairing`, regardless of declared commands.

## Best Possible Solution

Preserve command-sensitive node approval scopes and choose a maintainer-approved automation path for trusted admin-capable approvals, with docs, regression coverage, and real setup proof.

## Reproduction Assessment

Yes for the linked failure mode by source inspection: the CLI path uses the method-level pairing scope, while current `main` requires admin for `system.run` node approvals. No live Multipass or packaged run was executed in this read-only PR review.

## Solution Assessment

No. The PR fixes the automation blockage by weakening the approval gate for every caller; the safer path is to preserve derived scopes and add or document a trusted admin approval route.
