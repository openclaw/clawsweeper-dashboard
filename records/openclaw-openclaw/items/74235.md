## Summary

Keep open: the Google Chat threading bug is real and the PR fixes the central path, but the latest head still lets explicit current-message reply targets inherit the inbound thread without confirming the outbound target is the same Google Chat space.

## What This Changes

The PR preserves Google Chat inbound thread resource names through reply delivery and message-tool sends, validates outbound thread names, updates Google auth transport selection, and adds regression coverage plus a changelog entry.

## Best Possible Solution

Land this PR after adding same-target guards and focused regressions so Google Chat resource conversion stays plugin-local and never crosses spaces.

## Reproduction Assessment

Yes. Source inspection of the latest PR head gives a concrete path: a message-tool or direct Google Chat send to a different space with replyTo equal to the current inbound message still resolves to the inbound thread.

## Solution Assessment

No, not yet. The PR's direction is the right narrow fix, but both explicit current-message remap paths need a target match check and regression coverage before merge.

  - confidence: 0.89
