## Summary

Keep open. This fixes a real Google Chat threading bug, but the current PR head still has two same-target guard gaps that can carry an inbound thread from one Google Chat space into a send targeting another space.

## What This Changes

The PR preserves Google Chat thread resources through webhook context, reply delivery, message-tool sends, outbound adapters, API validation, auth transport handling, tests, and a changelog entry.

## Best Possible Solution

Keep the plugin-boundary resource normalization, add same-target guards plus focused regression tests, then land this PR.

## Reproduction Assessment

Yes. Current-main source shows message-shaped reply/thread values reaching Google Chat thread.name, and the inspected before/after screenshots show the live thread placement failure and improvement.

## Solution Assessment

No. The overall plugin-boundary approach is the right shape, but the current patch must gate current-message-to-inbound-thread substitution to the same Google Chat target in both action paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Gate current-message action replies by target:** `extensions/googlechat/src/actions.ts:85-89`
  - body: When `replyTo` equals the current Google Chat message resource, this returns the inbound thread without checking that `to` still matches the inbound `currentChannelId`. A tool call targeting another space can pass a `spaces/AAA/threads/...` thread into a `spaces/BBB` create request, causing a failed send or wrong-thread attempt instead of an unthreaded send.
  - confidence: 0.89
- **[P2] Check target before adapter thread fallback:** `extensions/googlechat/src/channel.adapters.ts:252-264`
  - body: `resolveAutoThreadId` only checks the outbound target when `replyToId` is absent. If the shared runner passes an explicit current-message reply target while sending to another Google Chat space, this still returns the inbound thread and bypasses the intended different-target protection.
  - confidence: 0.88
