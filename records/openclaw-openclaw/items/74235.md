## Summary

Keep open: current main still has the Google Chat thread-routing gap, and this PR is the right general direction, but the current head still has two source-reproducible cross-target remap bugs before it is mergeable.

## What This Changes

The PR carries Google Chat thread resource names through inbound context, reply delivery, message-tool routing, API validation, auth transport selection, regression tests, and a changelog entry.

## Best Possible Solution

Keep the resource-name conversion and validation inside the Google Chat plugin boundary, add same-target guards for explicit current-message remaps, and cover both resolver paths with focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives a concrete current-head path: a Google Chat send to `spaces/BBB` with `replyTo` equal to the inbound `spaces/AAA/messages/...` can still return `spaces/AAA/threads/...`.

## Solution Assessment

No, not yet. The PR's architecture is maintainable, but both explicit current-message remap paths need the same normalized-target guard before this is the safest fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Require same-target checks for action reply remaps:** `extensions/googlechat/src/actions.ts:85-88`
  - body: When `replyTo` is the current inbound Google Chat message, this branch returns `inboundThreadId` without checking that `to` still matches `currentChannelId`. A send to another space can therefore pass the source space's thread to Google Chat instead of sending unthreaded or rejecting the unsupported target.
  - confidence: 0.91
- **[P2] Require same-target checks in auto thread remaps:** `extensions/googlechat/src/channel.adapters.ts:258-264`
  - body: The shared message-tool resolver also returns the inbound thread when `replyToId` equals `currentMessageId` without validating the outbound target. A different-space tool send can inherit the source thread, so this remap needs the same target guard and regression coverage.
  - confidence: 0.9
