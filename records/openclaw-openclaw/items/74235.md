## Summary

Keep this PR open: it addresses a real Google Chat thread-routing gap on current main, but the current head still has a source-reproducible cross-target remap bug in both the shared message-tool resolver and the direct action path.

## What This Changes

The PR preserves Google Chat inbound thread resource names through reply delivery, message-tool sends, and current-reply paths while adding validation and regression coverage for Google Chat thread resources.

## Best Possible Solution

Keep the Google Chat resource-name handling at the plugin boundary, but require the outbound target to match the inbound Google Chat space before converting a current-message resource into the inbound thread.

## Reproduction Assessment

Yes. Source inspection follows a concrete path where a send to spaces/BBB with replyTo set to the current spaces/AAA message can receive spaces/AAA's inbound thread before delivery.

## Solution Assessment

No, not yet. The main normalization approach is maintainable, but the PR needs the same-target guard added to the current-message remap in both resolver paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Gate current-message remaps by target:** `extensions/googlechat/src/channel.adapters.ts:258-264`
  - body: When `replyToId` equals the current inbound Google Chat message, this returns `currentThreadTs` without confirming that `to` is still the current space. A tool send to another space can inherit the inbound thread from the original space and pass the wrong `thread.name` to Google Chat.
  - confidence: 0.9
- **[P2] Gate direct action remaps by target:** `extensions/googlechat/src/actions.ts:85-89`
  - body: The direct Google Chat action path has the same gap: an explicit message-shaped `replyTo` equal to `currentMessageId` maps to the inbound thread before checking `to`. Add the same normalized target check here so direct dispatch cannot inject an inbound thread into a different space.
  - confidence: 0.88
