Keep open. The PR addresses a real current-main Google Chat thread-routing gap and has sufficient live proof, but two P2 cross-target remap bugs remain before merge.

The PR preserves Google Chat inbound thread resources through reply and message-tool delivery, validates thread resource names, adjusts Google auth transport, adds regression coverage, and updates the changelog.

## Best Possible Solution

Add same-target guards to both explicit current-message remap paths, keep Google Chat resource normalization inside the plugin boundary, then land with focused cross-target regression coverage.

## Reproduction Assessment

Yes, source-level. Current main still lacks a distinct MessageThreadId carrier and forwards replyToId into Google Chat thread.name, while the PR body provides inspected before/after live screenshots for the intended fixed behavior.

## Solution Assessment

No, not yet. The plugin-boundary normalization direction is maintainable, but the proposed patch must verify same-target delivery before translating an explicit current message id into the inbound thread.
