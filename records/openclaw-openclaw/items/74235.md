## Summary

Keep open: this is a real Google Chat threading bug and the PR has solid proof plus passing CI, but the latest patch still has two same-target guard gaps that can inject the inbound space's thread into a different outbound Google Chat target.

## What This Changes

The PR preserves Google Chat inbound thread resource names and normalizes outbound thread targets across natural replies, streamed/block replies, and message-tool sends.

## Best Possible Solution

Add same-target guards and focused regression tests for explicit current-message remaps, then land this Google Chat-owned thread-resource normalization path and reconcile the linked Google Chat threading issues.

## Reproduction Assessment

Yes. Current main has a source-reproducible Google Chat threading path where message-shaped or missing thread metadata can reach delivery, and the PR body supplies live before/after Google Chat screenshots for the user-visible thread placement.

## Solution Assessment

No, not yet. The ownership boundary and resource-shape approach are right, but the current patch still needs same-target checks before converting an explicit current-message reply target to the inbound thread.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate current-message remaps by target:** `extensions/googlechat/src/actions.ts:85-88`
  - body: When `replyTo` is the current Google Chat message resource, this returns the inbound thread without confirming `to` is still the inbound space. A tool send to another space in the same turn can pass the source space's thread into the other request instead of sending unthreaded or rejecting the unsupported cross-target reply.
  - confidence: 0.9
- **[P2] Check the target before auto-thread remaps:** `extensions/googlechat/src/channel.adapters.ts:258-264`
  - body: `resolveAutoThreadId` only checks the target when `replyToId` is absent. If the shared message tool supplies the current inbound message as an explicit `replyToId` while targeting a different space, this returns the source space's inbound thread and can misroute or fail the Google Chat send.
  - confidence: 0.9
