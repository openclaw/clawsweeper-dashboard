## Summary

Keep open: this PR addresses a real Google Chat threading bug on current main, but the latest head still has a same-target guard gap when a send explicitly replies to the current inbound message resource while targeting another space.

## What This Changes

The PR preserves Google Chat inbound thread resource names through reply delivery and message-tool sends, validates outbound thread resources, adjusts Google auth transport selection, adds regression coverage, and adds a changelog entry.

## Best Possible Solution

Keep the plugin-boundary resource conversion and validation, but add normalized same-target checks for explicit current-message remaps in both action and auto-thread paths with focused regression tests.

## Reproduction Assessment

Yes. Source inspection on PR head gives a concrete failing path: a Google Chat send to another space with replyTo equal to the current inbound message can still inherit the source space's inbound thread.

## Solution Assessment

No, not yet. The architecture is the right boundary, but the current patch needs same-target guards before the thread remap is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Require same-target checks for action reply remaps:** `extensions/googlechat/src/actions.ts:88`
  - body: When `replyTo` equals the current inbound Google Chat message, this branch returns the inbound thread without checking that `to` still matches `currentChannelId`. A send to another space can therefore pass the source space's thread to Google Chat instead of sending unthreaded or rejecting the unsupported target.
  - confidence: 0.91
- **[P2] Require same-target checks in auto thread remaps:** `extensions/googlechat/src/channel.adapters.ts:264`
  - body: The shared message-tool resolver only checks the target when `replyToId` is absent. With an explicit current-message `replyToId`, a different-space tool send can inherit the source inbound thread, so this remap needs the same normalized target guard and regression coverage.
  - confidence: 0.9
