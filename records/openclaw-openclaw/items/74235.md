## Summary

Keep open: this is an active, useful fix for a real Google Chat threading gap on current main, but the current PR head still has two P2 same-target guard bugs in explicit current-message remapping paths.

## What This Changes

The PR adds Google Chat thread-resource preservation and validation across inbound context, reply delivery, message-tool sends, auth transport tests, regression tests, and the changelog.

## Best Possible Solution

Land this PR after adding normalized same-target guards for explicit current-message remaps and regression tests for different-target Google Chat tool sends.

## Reproduction Assessment

Yes. Source inspection on PR head shows a Google Chat send to a different space with `replyTo` equal to the current inbound message can still inherit the source space's inbound thread.

## Solution Assessment

No, not yet. The plugin-boundary resource conversion is the right shape, but explicit current-message remaps need the same normalized target guard already used for omitted reply targets.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate current-message remaps by target:** `extensions/googlechat/src/actions.ts:86-88`
  - body: When `replyTo` equals the current inbound Google Chat message, this branch returns the inbound thread before checking that `to` is still the current space. A send to `spaces/BBB` during an inbound `spaces/AAA` turn can pass the `spaces/AAA` thread into the BBB request instead of sending unthreaded or rejecting the unsupported target.
  - confidence: 0.91
- **[P2] Guard auto-thread remaps for explicit replies:** `extensions/googlechat/src/channel.adapters.ts:258-264`
  - body: `resolveAutoThreadId` only checks the current target when `replyToId` is absent. The shared message-action path can supply the current inbound message as an explicit `replyToId`, so a different-space send can still receive the source space's inbound thread; add the same normalized target guard before returning `inboundThread`.
  - confidence: 0.9
