## Summary

Keep open: this PR fixes a real Google Chat threading gap on current main, but the latest head still has a narrow cross-target remap bug where an explicit current-message reply target can carry the source space's inbound thread into a different Google Chat space send.

## What This Changes

The PR preserves Google Chat thread resource names through inbound context, reply delivery, message-tool threading, API validation, auth transport handling, regression tests, and the changelog.

## Best Possible Solution

Add same-target guards before mapping current Google Chat message resources to the inbound thread in both the message-tool auto-thread resolver and direct action resolver, with regressions for explicit current-message replyTo sent to another space.

## Reproduction Assessment

Yes. Source inspection on PR head gives a high-confidence path: a Google Chat message-tool send to spaces/BBB with replyTo equal to the current inbound spaces/AAA message still resolves threadId to spaces/AAA/threads/inbound.

## Solution Assessment

No, not yet. Keeping Google Chat resource conversion inside the plugin boundary is the right design, but the current patch needs same-target checks for explicit current-message remaps before landing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate auto thread remaps by target:** `extensions/googlechat/src/channel.adapters.ts:258-264`
  - body: When `replyToId` is the current inbound message, this returns `inboundThread` without verifying that `to` still matches `currentChannelId`. A message-tool send to `spaces/BBB` during a `spaces/AAA` turn can therefore set `threadId` to `spaces/AAA/threads/...` and pass the source thread into the other-space send.
  - confidence: 0.92
- **[P2] Gate direct action remaps by target:** `extensions/googlechat/src/actions.ts:85-88`
  - body: `resolveActionThreadId` maps an explicit current-message `replyTo` to the inbound thread before checking the outbound target. Direct Google Chat action sends to a different space can inherit the source space's thread instead of sending unthreaded or rejecting the unsupported cross-target reply.
  - confidence: 0.88
