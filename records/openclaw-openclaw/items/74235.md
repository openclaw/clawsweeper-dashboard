## Summary

Keep open: this PR addresses a real current-main Google Chat thread-routing bug and includes live behavior proof, but the current head still lets explicit current-message reply targets map the source thread into a different outbound Google Chat space.

## What This Changes

The branch preserves Google Chat thread resource names through inbound context, reply delivery, message-tool routing, API validation, auth transport selection, regression tests, and the changelog.

## Best Possible Solution

Keep the Google Chat resource conversion at the plugin boundary, but add normalized same-target guards and regression tests for explicit current-message remaps before landing.

## Reproduction Assessment

Yes. Source inspection on PR head gives a high-confidence path: a Google Chat tool send to another space with replyTo equal to the current inbound message can still inherit the source space's inbound thread.

## Solution Assessment

No, not yet. The plugin-boundary design is the right direction, but the current patch needs same-target checks before remapping explicit current-message reply targets to the inbound thread.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate action reply remaps by target:** `extensions/googlechat/src/actions.ts:86-88`
  - body: When replyTo equals the current inbound Google Chat message, this branch returns the inbound thread without checking that to still matches currentChannelId. A direct send to another space can therefore carry the source space's thread into the other space request instead of sending unthreaded or rejecting the unsupported target.
  - confidence: 0.91
- **[P2] Gate auto thread remaps by target:** `extensions/googlechat/src/channel.adapters.ts:252-264`
  - body: resolveAutoThreadId only checks the target when replyToId is absent. With an explicit current-message replyToId, a different-space tool send can still receive the inbound thread from the source space, so this remap needs the same normalized target guard and regression coverage.
  - confidence: 0.9
