## Summary

Keep this PR open. Current main still has the Google Chat thread-routing gap, and the PR fixes the main path with supplied live screenshot proof, but the latest head still has two source-reproducible cross-target remap bugs that should block merge.

## What This Changes

The PR adds Google Chat thread resource validation, carries inbound thread IDs through reply and message-tool delivery, updates Google auth transport selection, and expands Google Chat/core regression coverage.

## Best Possible Solution

Keep the Google Chat resource conversion inside the plugin boundary, but require the normalized outbound target to match the inbound space before converting a current-message resource into the inbound thread.

## Reproduction Assessment

Yes. Source inspection on PR head shows explicit message-resource reply targets equal to currentMessageId return the inbound thread before a target match check in both the direct action and adapter resolver paths.

## Solution Assessment

No, not yet. The overall direction is maintainable, but the PR should add same-target guards and focused regressions before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Gate direct current-message remaps by target:** `extensions/googlechat/src/actions.ts:85-89`
  - body: When `replyTo` or `threadId` is the current inbound Google Chat message resource, this branch returns `inboundThreadId` before checking that `to` matches the current space. A send to `spaces/BBB` during a `spaces/AAA` turn can therefore pass the `spaces/AAA/threads/...` thread to the Chat API; require the normalized target match before converting.
  - confidence: 0.9
- **[P2] Gate plugin auto-thread remaps by target:** `extensions/googlechat/src/channel.adapters.ts:258-264`
  - body: `resolveAutoThreadId` checks the target only when `replyToId` is absent, so an explicit current-message `replyToId` can still return the inbound thread for a different outgoing space. Add the same normalized target guard for this explicit remap path so cross-space sends do not inherit the source thread.
  - confidence: 0.91
