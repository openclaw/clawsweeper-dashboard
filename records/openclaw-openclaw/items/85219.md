## Summary

Keep open. The PR is a focused Telegram delivery fix with supplied live proof, but current head still needs one alias-normalization repair because top-level compatibility aliases can fan out into multiple Telegram media sends.

## What This Changes

The branch teaches Telegram `sendMessage` actions to collect local path/filePath and structured attachment media, send multiple media items, add regression tests, and note the fix in the changelog.

## Best Possible Solution

Merge after Telegram `sendMessage` media collection treats top-level aliases as one canonical source while preserving `mediaUrls[]` and per-attachment multi-send behavior behind the existing media loader controls.

## Reproduction Assessment

Yes by source inspection: current main exposes `path`/`filePath` and structured attachment aliases through the message tool, while Telegram action runtime only reads `mediaUrl`/`media`. I did not run tests under the read-only review rules.

## Solution Assessment

No, not quite yet. The PR is the right narrow surface, but it should collapse top-level compatibility aliases to one media source before sequencing true multi-attachment inputs.
