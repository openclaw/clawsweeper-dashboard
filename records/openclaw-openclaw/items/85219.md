## Summary

Keep this PR open: the current Telegram action path does miss structured attachment media by source inspection, but this patch still needs live Telegram proof and should cover the full message-tool media shape before merge.

## What This Changes

The branch maps the first structured Telegram sendMessage attachment alias into mediaUrl and adds a mock action-runtime regression test for that path.

## Best Possible Solution

Normalize Telegram action sendMessage against the existing message-tool media contract, deliver all supported media aliases through the channel's media sequencing path, then add redacted live Telegram proof.

## Reproduction Assessment

Yes by source inspection: current main's Telegram action runtime reads only mediaUrl/media, while the message tool exposes path/filePath and structured attachment fields. I did not run tests because this review was read-only.

## Solution Assessment

No: the proposed direction is right, but mapping only the first structured attachment into a single mediaUrl is narrower than the existing message-tool contract and outbound media sequencing path.
