## Summary

Keep open: current main and v2026.6.9 still lack model-facing cursor, offset, page-token, or clean readable export support for `sessions_history`. The adjacent REST cursor, `/export-session` command, and open archive/family-history PRs are useful context, but they do not satisfy this request without maintainer product and transcript-security direction.

## What This Changes

Request to add pagination/offset/page-token and/or clean export support for `sessions_history` so long conversation history can be retrieved beyond the current bounded tool output.

## Best Possible Solution

Define one canonical visibility-checked `sessions_history` pagination and readable-export contract, likely reusing the existing REST cursor semantics while preserving redaction and avoiding a new global cap knob unless maintainers explicitly choose it.

## Reproduction Assessment

Yes, at source level: current main exposes no `sessions_history` cursor/export input and still caps sanitized tool output at 80 KiB. I did not run a live long-session repro because this was a read-only review.

## Solution Assessment

Unclear until maintainers choose the product shape. Reusing the existing HTTP cursor semantics for model-facing pagination looks safer than raising a global cap, while any readable export must preserve visibility, redaction, and reasoning/tool filtering contracts.
