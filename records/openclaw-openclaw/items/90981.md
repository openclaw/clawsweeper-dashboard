## Summary

Keep open: current main and v2026.6.10 still lack model-facing `sessions_history` offset/cursor support and a clean readable full-history export surface. The open pagination PR at https://github.com/openclaw/openclaw/pull/97101 is a candidate for only the pagination portion and explicitly leaves the export/product API decision unresolved.

## What This Changes

The issue requests pagination or offset support plus a cleaner export path for complete `sessions_history` conversation logs.

## Best Possible Solution

Keep this as the canonical maintainer-owned tracker until offset pagination lands or is rejected and maintainers decide whether readable export belongs in `sessions_history`, a separate export tool, an existing command/API path, or ClawHub/plugin work.

## Reproduction Assessment

Yes, at source level: current main exposes no `sessions_history` offset/cursor/export input, `chat.history` has no offset parameter, and the model-facing tool still caps output at 80 KiB. I did not run a live long-transcript repro because this review is read-only.

## Solution Assessment

No single proposed option is clearly best yet: additive pagination is being attempted by an open PR, but clean full-history export needs a maintainer decision on API shape and transcript-security boundaries. Reusing existing visibility/redaction paths is safer than adding a global cap knob.
