## Summary

Keep open: current main and the latest release still leave the agent-visible `sessions_history` tool as a bounded recall surface without offset, cursor, page token, or readable export support. Adjacent HTTP cursor pagination and trajectory export exist, but they do not fully satisfy the requested model-facing pagination/export workflow.

## What This Changes

Request pagination/offset or export support for `sessions_history` so long conversations can be retrieved beyond the current bounded response.

## Best Possible Solution

Define one canonical, visibility-checked, redacted session-history pagination/export contract, preferably reusing Gateway cursor semantics and avoiding a global size-limit config knob.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection confirms the current tool has no offset, cursor, page token, or export mode and still applies the reported 80 KiB cap.

## Solution Assessment

Unclear until maintainers choose the product/API shape. Pagination or export should reuse existing Gateway history contracts where possible, while a global configurable cap is the least attractive option because it expands config surface without solving clean export semantics.
