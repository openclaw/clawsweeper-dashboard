## Summary

Keep open: current main still exposes `sessions_history` as a bounded tool without cursor/export parameters. Existing HTTP cursor pagination and slash-command export cover adjacent surfaces, but they do not fully solve the requested agent-tool/readable-export workflow.

## What This Changes

Request adds pagination/offset or export support to `sessions_history` so long session history can be retrieved as a complete readable log.

## Best Possible Solution

Define one canonical, visibility-checked, redacted session-history pagination/export path, preferably reusing the existing HTTP cursor semantics and avoiding a global size-limit config knob.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection confirms the current tool only supports `sessionKey`, `limit`, and `includeTools`, then applies the 80 KiB cap reported by the issue.

## Solution Assessment

Unclear until maintainers choose the product shape. Pagination looks safer than raising a global cap, while export support should reuse existing session-history/export contracts rather than duplicate them.
