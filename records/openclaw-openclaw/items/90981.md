## Summary

Keep open: current main and v2026.6.9 still expose `sessions_history` as a bounded, sanitized recall tool with no model-facing cursor, offset, page token, or clean readable export contract. The open family-history PR at https://github.com/openclaw/openclaw/pull/90239 is useful adjacent work, but it does not satisfy this pagination/export request.

## What This Changes

Request to add pagination/offset and clean export support for `sessions_history` so long session histories can be retrieved beyond the current bounded output.

## Best Possible Solution

Define one canonical visibility-checked, bounded pagination and readable-export contract for session history, likely reusing existing REST cursor semantics while preserving redaction and avoiding a new global cap knob unless maintainers explicitly choose it.

## Reproduction Assessment

Yes, at source level: current main exposes no `sessions_history` cursor/export input and still caps sanitized tool output at 80 KiB. I did not run a live long-session repro because this was a read-only review.

## Solution Assessment

Unclear until maintainers choose the product shape. Reusing the existing HTTP cursor semantics for model-facing pagination looks safer than raising a global cap, while any readable export must preserve visibility, redaction, and reasoning/tool filtering contracts.
