## Summary

Keep open: current main and the latest checked release still expose `sessions_history` as a bounded tool without cursor, offset, page-token, or export parameters. The existing HTTP cursor endpoint, slash-command HTML export, and related family/archive PRs are useful adjacent work, but none currently resolves the requested agent-tool pagination/export workflow.

## What This Changes

The issue requests pagination, offset/page-token, or export support for `sessions_history` so long conversations can be retrieved beyond the current bounded response.

## Best Possible Solution

Define one canonical, visibility-checked, redacted session-history pagination/export contract, preferably reusing existing cursor semantics and avoiding a global size-limit config knob.

## Reproduction Assessment

Yes for the source-level limitation: current main still exposes a bounded `sessions_history` tool without cursor/export parameters. This is not a broken documented contract reproduction, and no live long-session run was performed in this read-only review.

## Solution Assessment

Unclear until maintainers choose the product shape. Pagination through the existing cursor model looks safer than raising a global cap, while export support should reuse existing visibility/redaction/export contracts instead of creating another transcript reader.
