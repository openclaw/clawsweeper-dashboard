## Summary

Keep open. Current main still resolves model-provider auth and requires an API key before plugin media audio execution reaches `transcribeAudio`, and open PR #75005 already references this issue with closing syntax but is not merged.

## What This Changes

Allow plugin-owned no-auth or local media understanding providers, especially audio/STT providers, to execute without forced model-provider API-key resolution.

## Best Possible Solution

Review and land PR #75005 or an equivalent explicit opt-in synthetic/no-auth media-provider hook with focused audio/video regression coverage and plugin docs.

## Reproduction Assessment

Yes. Configure a plugin-only audio media provider without credentials and trigger transcription; current main reaches `resolveProviderExecutionContext` and `resolveApiKeyForProvider` before `transcribeAudio`, producing the reported missing-key path.

## Solution Assessment

Yes in direction, but it is not merged yet. An explicit opt-in synthetic/no-auth hook is the narrow maintainable fix because it preserves auth failures for ordinary unmarked providers.
