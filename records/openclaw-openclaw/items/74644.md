## Summary

Keep open. Current main still resolves media-provider execution through model auth and `requireApiKey` before plugin audio transcription runs, and the linked closing PR #75005 is still the active unmerged implementation path.

## What This Changes

Allow local or no-auth media-understanding audio/STT providers to opt into running without model-provider API-key credentials.

## Best Possible Solution

Land PR #75005, or an equivalent explicit synthetic/non-secret media-provider auth opt-in with tests and docs, while preserving the existing credential requirement for unmarked providers.

## Reproduction Assessment

Yes. Configure a plugin-only audio `mediaUnderstandingProviders` entry without credentials and trigger transcription; current main resolves provider auth and calls `requireApiKey` before `transcribeAudio`.

## Solution Assessment

Yes in direction, but it is not merged yet. The narrow maintainable fix is an explicit media-provider no-auth/synthetic-auth opt-in, not a broad bypass of media auth.
