## Summary

Keep open. Current main still resolves provider auth and calls `requireApiKey` before invoking `transcribeAudio`, and the linked closing PR #75005 remains open and unmerged in the provided GitHub context.

## What This Changes

Allow media-understanding audio providers to explicitly opt into no-auth or synthetic non-secret auth for local/self-hosted STT.

## Best Possible Solution

Land PR #75005 or an equivalent explicit media-provider synthetic/no-auth opt-in with regression coverage and docs while preserving fail-closed behavior for unmarked providers.

## Reproduction Assessment

Yes, by source inspection. Configure a plugin media audio provider without resolvable credentials and trigger transcription; current main resolves provider auth with `requireApiKey` before calling `transcribeAudio`, so the provider cannot handle a local no-auth request.

## Solution Assessment

Yes in direction, but it is not merged yet. An explicit provider opt-in for synthetic non-secret media auth is narrower and safer than bypassing `requireApiKey` for every media provider.
