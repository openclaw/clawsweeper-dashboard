## Summary

Keep open. Current main still resolves model-provider auth and requires an API key before plugin media audio execution, and open PR #75005 already references this issue with a targeted fix that has not merged.

## What This Changes

Allow plugin-owned local or no-auth media understanding providers to execute audio transcription without requiring auth-store API credentials.

## Best Possible Solution

Media-provider no-auth should be an explicit plugin/provider opt-in that supplies a non-secret execution token for audio and video while preserving fail-closed auth for unmarked providers, with regression tests and docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure a plugin-only audio media provider without credentials and trigger transcription; current main calls `resolveProviderExecutionContext` and `requireApiKey` before `transcribeAudio`.

## Solution Assessment

Yes in direction, but it is not merged yet. An explicit opt-in synthetic/no-auth hook is the narrow maintainable fix because it avoids globally bypassing auth for ordinary providers.
