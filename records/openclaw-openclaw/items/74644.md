## Summary

Keep open. Current main still resolves media audio provider execution through model-provider auth and `requireApiKey` before invoking plugin `transcribeAudio`, and PR #75005 is an open, unmerged implementation path that uses closing syntax for this issue.

## What This Changes

Allow local or no-auth media-understanding audio/STT providers to opt into execution without a stored API key.

## Best Possible Solution

Land PR #75005 or an equivalent explicit media-provider synthetic/no-auth opt-in with regression coverage and docs, while preserving the current auth requirement for unmarked providers.

## Reproduction Assessment

Yes, by source inspection. Configure a plugin-only audio media provider without credentials and trigger transcription; current main resolves model-provider auth with `requireApiKey` before calling `transcribeAudio`.

## Solution Assessment

Yes in direction, but it is not merged yet. An explicit opt-in for synthetic non-secret media auth is the narrow maintainable fix; a global no-auth fallback would be unsafe.
