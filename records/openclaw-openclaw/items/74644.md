## Summary

Keep open. Current main still requires model-provider auth for media audio/video execution before plugin `transcribeAudio` or `describeVideo` runs, and the open closing PR #75005 is the active implementation path rather than merged behavior.

## What This Changes

Allow plugin-owned local/no-auth media understanding providers to opt into audio transcription without auth-store API-key credentials.

## Best Possible Solution

Review and land PR #75005, or an equivalent explicit synthetic/no-auth media-provider opt-in that preserves fail-closed behavior for ordinary remote providers.

## Reproduction Assessment

Yes. Configure a plugin-only `mediaUnderstandingProviders` audio provider without credentials and trigger transcription; current main resolves provider auth and calls `requireApiKey` before `transcribeAudio`.

## Solution Assessment

Yes in direction. The narrow maintainable fix is an explicit plugin/provider opt-in for synthetic non-secret media auth, not a broad bypass of media auth resolution.
