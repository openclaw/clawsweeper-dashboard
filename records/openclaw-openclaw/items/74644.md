## Summary

Keep open. Current main still resolves audio media providers through model auth and `requireApiKey` before `transcribeAudio`, and open PR #75005 is the active linked fix path.

## What This Changes

Allow `mediaUnderstandingProviders` audio/STT providers to opt into local or no-auth execution without being rejected by model API-key resolution.

## Best Possible Solution

Land PR #75005 or an equivalent explicit media-provider synthetic/no-secret auth hook with regression tests and docs.

## Reproduction Assessment

Yes. Configure a plugin-only media/STT provider without credentials and trigger audio transcription; current main calls `resolveProviderExecutionContext`, `resolveProviderExecutionAuth`, and `requireApiKey` before `transcribeAudio`.

## Solution Assessment

Yes in direction, but it is not on main yet. An explicit opt-in synthetic/non-secret auth path is the narrow maintainable solution, while ordinary unmarked remote providers should continue to fail closed.
