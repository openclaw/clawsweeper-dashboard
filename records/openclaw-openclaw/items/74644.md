## Summary

Keep open. Current main still requires media provider auth before `transcribeAudio` runs, and the open closing PR #75005 is the active implementation path rather than a reason to close this issue early.

## What This Changes

Allow plugin-owned local or no-auth media understanding providers to run audio transcription without a required API key.

## Best Possible Solution

Land PR #75005 or an equivalent explicit synthetic/non-secret media-provider auth opt-in, with regression coverage and fail-closed behavior preserved for unmarked remote providers.

## Reproduction Assessment

Yes. Configure a plugin-only media audio provider without credentials and trigger transcription; current main resolves provider auth and calls `requireApiKey` before invoking the plugin's `transcribeAudio`.

## Solution Assessment

Yes in direction, but it is not merged yet. An explicit opt-in synthetic auth hook is narrower and safer than bypassing media auth globally.
