## Summary

Keep open. Current main still resolves media provider execution through model auth and `requireApiKey` before invoking plugin transcription, and the linked closing PR #75005 remains open and unmerged.

## What This Changes

Allow plugin-owned local/no-auth media understanding providers to run audio/STT without auth-store API keys when explicitly opted in.

## Best Possible Solution

Land PR #75005 or an equivalent explicit media-provider synthetic/no-auth opt-in with regression tests and docs, while keeping ordinary remote providers fail-closed by default.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to configure a plugin-only audio media provider without credentials and trigger transcription; current main calls `resolveProviderExecutionContext`, `resolveProviderExecutionAuth`, and `requireApiKey` before `transcribeAudio`.

## Solution Assessment

Yes in direction, but it is not merged yet. An explicit opt-in synthetic/non-secret auth hook is the narrow maintainable solution; a broad bypass of media auth would be too risky.
