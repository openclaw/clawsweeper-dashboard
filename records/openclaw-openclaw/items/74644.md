## Summary

Keep open. Current main still resolves provider auth before invoking audio media-understanding providers, and PR #75005 is the existing open implementation path for this issue.

## What This Changes

Allow local or no-auth media-understanding audio/STT providers to opt in to execution without requiring auth-store API key credentials.

## Best Possible Solution

Review and land PR #75005, or an equivalent explicit provider-owned synthetic/no-auth media execution hook, with focused audio/video regression tests and docs that distinguish opted-in no-auth providers from missing credentials.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure a plugin-only audio media provider without provider credentials and trigger transcription; current main reaches resolveProviderExecutionAuth before transcribeAudio.

## Solution Assessment

Yes in direction, but it is not merged yet. An explicit opt-in synthetic/no-auth hook is the narrow maintainable fix because it avoids bypassing auth globally for unmarked remote providers.
