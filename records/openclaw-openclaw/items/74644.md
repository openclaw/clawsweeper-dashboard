## Summary

Keep open. Current main still requires model-provider API-key resolution before plugin media audio execution, and open PR #75005 already references this issue with closing syntax, so the issue should wait for that PR or an equivalent fix to merge.

## What This Changes

Allow explicitly local/no-auth media-understanding audio providers to transcribe without stored provider API-key credentials.

## Best Possible Solution

Land PR #75005 or an equivalent explicit synthetic/no-auth media-provider hook with focused audio/video regression coverage and docs that distinguish opted-in local providers from ordinary missing credentials.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure a plugin-only audio media provider without credentials and trigger transcription; current main reaches resolveProviderExecutionAuth before transcribeAudio.

## Solution Assessment

Yes in direction, but it is not merged yet. An explicit opt-in synthetic/no-auth hook is the narrow maintainable fix because it avoids globally bypassing auth for unmarked providers.
