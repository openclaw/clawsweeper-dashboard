## Summary

Keep open. Current main still builds the embedded-run tool set on every non-raw run and `resolvePluginTools` still invokes each registered plugin tool factory at request time; the recent tool-discovery and scoped-registry reuse work is present, but it does not provide per-plugin factory timing, a documented agent/run-level plugin-tool opt-out, or safe factory-result amortization.

## What This Changes

The issue reports repeated `core-plugin-tools` embedded-run latency and asks for per-run factory-cost mitigation, a plugin-tool opt-out, and per-plugin timing.

## Best Possible Solution

Keep this as the canonical tracker for the remaining plugin-tool factory-cost work. The next maintainable end state is per-plugin factory timing plus a documented agent/run-level plugin-tool opt-out, followed by narrowly keyed memoization only where traces prove the factory output is context-stable.
