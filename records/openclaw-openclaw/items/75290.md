## Summary

Keep open. Current main has meaningful partial fixes, including scoped tool discovery and per-plugin factory timing, but normal embedded runs still build the tool set per run and still invoke plugin tool factories at request time; a cheap documented plugin-tool-only opt-out and any proven-safe amortization remain unresolved.

## What This Changes

The issue requests reducing repeated `core-plugin-tools` prep latency by avoiding or amortizing per-run plugin tool factory work and adding diagnostics or opt-out controls.

## Best Possible Solution

Keep this as the canonical tracker for the remaining plugin-tool prep work: use the new per-plugin timings to identify hot factories, add a documented cheap opt-out for agents/runs that do not need plugin tools, and only memoize factory outputs proven context-stable.
