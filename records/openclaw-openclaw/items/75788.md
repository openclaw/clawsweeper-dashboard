## Summary

Close as ClawHub/plugin-track. The RFC describes an optional multi-agent memory governance plugin, not a current core bug or a demonstrated missing SDK seam. Current docs and source already support external code plugins with tools, hooks, services, gateway methods, CLI commands, memory supplements, and memory-public-artifact bridges; the closest in-tree precedent is memory-wiki, while no current main surface named memory-graph, graph.sqlite, or fleet-bridge exists to patch.

## What This Changes

Request to add or incubate an `@openclaw/plugin-fleet-bridge` plugin with typed prior-capsule/posterior-delta records, dissent-preserving compaction invariants, and migration guidance for multi-agent OpenClaw.

## Best Possible Solution

Keep OpenClaw core focused on reusable SDK seams and let the fleet bridge evolve as an external ClawHub code plugin. If that prototype proves a concrete missing SDK hook, open a narrower core issue for that hook instead of tracking the whole plugin in this repository.
