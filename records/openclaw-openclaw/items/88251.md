## Label Justifications

- `P1`: The underlying issue can stall Discord inbound processing for a session, which is a broken channel workflow affecting real users.
- `merge-risk: 🚨 compatibility`: The PR introduces a fixed 30-minute production cap that can undercut existing agent timeout configuration and the current 48-hour default.
- `merge-risk: 🚨 message-delivery`: On watchdog timeout the patch commits replay keys, so a legitimate long-running Discord message can be treated as consumed without a completed reply.
- `merge-risk: 🚨 session-state`: The PR explicitly allows a timed-out run to keep completing while later same-session work starts, creating session ordering/state risk.

## Summary

Keep open. The Discord queue-stall bug is source-reproducible, but this PR should not merge as-is because the new fixed 30-minute watchdog can preempt configured/default agent run timeouts and consume replay keys for legitimate long Discord turns; real Discord/gateway proof is also still missing.

## What This Changes

Adds a Discord inbound queue watchdog that aborts queued message processing after a timeout, commits replay keys on timeout, and adds focused queue regression tests.

## Best Possible Solution

Keep the Discord-owned queue-tail breaker idea, but make its deadline respect the effective agent/run timeout or an explicitly approved cap, preserve replay/session ordering semantics, and prove it with focused tests plus real Discord/gateway evidence.

## Reproduction Assessment

Yes from source inspection: current main serializes Discord inbound work per queue key and awaits processDiscordMessage without a per-task timeout, so a never-settling run leaves later same-key messages behind the pending tail. I did not run a live Discord repro in this read-only review.

## Solution Assessment

No as currently proposed: a Discord tail breaker is a plausible fix, but the hard 30-minute default ignores the existing agent timeout contract. The safer path is to tie the watchdog to the effective run budget or get explicit maintainer approval for a shorter cap.
