## Label Justifications

- `P1`: The PR targets a broken Discord inbound workflow where stuck runs can block later user messages, but its current fix can also interrupt real existing runs.
- `merge-risk: 🚨 compatibility`: The hard 30-minute watchdog can override existing configured agent timeout behavior during upgrades.
- `merge-risk: 🚨 message-delivery`: Timed-out inbound messages are committed as consumed, so a premature timeout can make user-visible Discord work disappear instead of retrying.
- `merge-risk: 🚨 session-state`: The queue can advance after a watchdog timeout while the earlier same-session run is still completing cooperatively.

## Summary

Keep open: the patch targets a real Discord queue stall, but it introduces a fixed Discord-owned timeout that can override existing agent timeout behavior and can let same-session work overlap after a cooperative abort. It also still lacks real Discord/Gateway behavior proof; the supplied proof is a focused synthetic Vitest path.

## What This Changes

Adds a Discord inbound run watchdog that aborts queued message processing after a 30-minute default, commits replay keys on timeout, and extends queue regression coverage.

## Best Possible Solution

Keep the queue-tail breaker, but make the timeout compatible with the existing agent timeout contract and get maintainer approval or a guard for same-session overlap before merge.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source: createChannelRunQueue awaits each keyed task with no per-task timeout, so a non-settling Discord processing promise can hold later jobs for the same key; I did not execute the new synthetic regression test.

## Solution Assessment

Is this the best way to solve the issue? No: a Discord queue-tail breaker is reasonable, but an unconditional 30-minute consumed-delivery timeout is not the narrowest compatibility-safe fix while OpenClaw already has configured agent run timeouts.

- **[P1] Honor configured agent timeouts before aborting Discord runs:** `extensions/discord/src/monitor/message-run-queue.ts:67`
