  - body: This test sets `maxChars` to 25 but then expects the 61-character table to be delivered as one `onBlockReply`.   - confidence: 0.91

  - body: The new guard checks `paragraphBreak.index - start <= paragraphLimit`, but the emitted slice extends through `paragraphBreak.index + paragraphBreak.length`. A blank-line separator containing spaces or extra newlines can push the emitted chunk over `maxChars`, violating the documented cap and the existing clamp behavior.
  - confidence: 0.9
- **[P2] Accept short table delimiter cells:** `src/markdown/table-spans.ts:52`
  - body: GFM delimiter cells are hyphen-only with optional alignment colons, but this regex requires at least three hyphens. Tables such as `| A | B |` / `| - | - |` will not be protected and can still split mid-table.
  - confidence: 0.82
- **[P3] Recognize one-column pipe tables:** `src/markdown/table-spans.ts:47-52`
  - body: After trimming outer pipes, `| Name |` has one cell, so the `cells.length >= 2` checks reject valid one-column pipe tables. Those tables remain unprotected by the new table-safe chunking path.
  - confidence: 0.78
