## Summary

Keep open: this PR targets real current-main block-streaming Markdown bugs and correctly carries forward the table-span direction from #66568, but the current head is not merge-ready because preserved separators still get trimmed in delivery and several table/cap edge cases remain broken.

## What This Changes

This PR updates `EmbeddedBlockChunker` to preserve newline separators and avoid splitting detected pipe tables, adds table-span tests, and updates streaming docs plus the changelog.

## Best Possible Solution

Land one shared block-streaming fix that preserves Markdown boundary semantics through `subscribeEmbeddedPiSession`, enforces channel caps, protects valid GFM pipe-table spans, and keeps docs, changelog, and regression tests aligned.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main drops paragraph separators in `EmbeddedBlockChunker` and then trims block delivery text, and it has no table-span protection. I did not run tests because this review was read-only.

## Solution Assessment

No. The shared chunker is the right surface, but this branch still needs a narrow repair so separators survive delivery, separator-inclusive chunks respect caps, and table detection matches valid GFM cases.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Carry separators through block delivery:** `src/agents/pi-embedded-block-chunker.test.ts:142-144`
  - body: The new regression only proves raw `EmbeddedBlockChunker` output. In production, `emitBlockChunk` still calls `.trimEnd()` before saving `assistantTexts` and invoking `onBlockReply`, so preserved `\n` or `\n\n` separators are removed before visible delivery.
  - confidence: 0.91
- **[P2] Count preserved separators against the cap:** `src/agents/pi-embedded-block-chunker.ts:217-221`
  - body: This guard compares only `paragraphBreak.index - start` to the cap, but the emitted slice extends through `paragraphBreak.index + paragraphBreak.length`. A blank-line separator with spaces or extra newlines can therefore produce a chunk longer than `maxChars`.
  - confidence: 0.9
- **[P2] Make the table fixture respect maxChars:** `src/agents/pi-embedded-subscribe.subscribe-embedded-pi-session.streams-soft-chunks-paragraph-preference.test.ts:53`
  - body: This test sets `maxChars` to 25 but expects the 61-character table to be delivered as one block reply. Raise the cap above the table length or assert the row-safe chunk sequence that the chunker actually guarantees under the cap.
  - confidence: 0.91
- **[P2] Accept short table delimiter cells:** `src/markdown/table-spans.ts:48`
  - body: The delimiter regex requires at least three hyphens per cell, so valid pipe tables using shorter delimiter cells are not recognized and can still split mid-table. Relax the delimiter check and add coverage for the short form.
  - confidence: 0.82
- **[P3] Recognize one-column pipe tables:** `src/markdown/table-spans.ts:43-48`
  - body: After trimming edge pipes, a row like `| Name |` has one cell, so the `cells.length >= 2` checks reject valid one-column pipe tables. Those tables remain unprotected unless the helpers allow the one-cell case with non-empty content.
  - confidence: 0.78
