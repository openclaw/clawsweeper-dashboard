## Summary

Keep this PR open for repair. Current main still has the reported streaming boundary behavior, and the PR is aimed at the right shared chunker surface, but the proposed patch is not merge-ready because it can exceed configured chunk caps and its table-span parser misses markdown-it/GFM-valid tables.

## What This Changes

The PR updates `EmbeddedBlockChunker` to preserve consumed newline and paragraph separators, adds table-span protection for block-streamed Markdown tables, adds unit and integration coverage, and updates streaming docs plus the changelog.

## Best Possible Solution

Land one canonical shared `EmbeddedBlockChunker` fix that preserves reconstructed Markdown boundaries without violating `maxChars`, aligns table detection with the markdown-it/GFM table contract, and keeps the focused unit, integration, docs, and changelog coverage tied to the linked streaming reports.

## Reproduction Assessment

Yes. Current main has a source-level reproduction for the original bug via `flushOnParagraph` consuming separators, and the PR-specific bugs reproduce from the diff with a long preserved separator crossing `maxChars` plus valid one-hyphen or single-column pipe tables.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Cap preserved paragraph separators before emitting:** `src/agents/pi-embedded-block-chunker.ts:217-221`
  - body: The new check only measures up to the start of the paragraph separator, then emits through `paragraphBreak.length`. A separator such as `\n \n` can push the emitted block past `maxChars`, contradicting the documented channel cap and the existing clamp behavior.
  - confidence: 0.9
- **[P2] Accept shorter GFM table delimiters:** `src/markdown/table-spans.ts:49`
  - body: The delimiter regex requires three or more dashes per cell, but markdown-it/GFM accepts one or more dashes with optional alignment colons, such as `:-:`. Those valid tables remain unprotected from mid-table splitting.
  - confidence: 0.86
- **[P3] Recognize single-column pipe tables:** `src/markdown/table-spans.ts:44-49`
  - body: After trimming outer pipes, a row like `| Name |` has one cell, so the `cells.length >= 2` checks reject valid one-column tables. These tables can still split across streamed block chunks.
  - confidence: 0.8
