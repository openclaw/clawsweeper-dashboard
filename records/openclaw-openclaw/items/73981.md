Full review comments:

- **[P2] Preserve separators through subscribe delivery:** `src/agents/pi-embedded-block-chunker.test.ts:142-143`
  - body: The new regression proves raw EmbeddedBlockChunker output, but production delivery still trims trailing whitespace before storing and invoking onBlockReply. A preserved trailing newline can still disappear before visible block delivery, so add delivery-level coverage and carry the boundary through that path.
  - confidence: 0.91
- **[P2] Count preserved separators against maxChars:** `src/agents/pi-embedded-block-chunker.ts:219-223`
  - body: This guard checks only the text before the paragraph separator, but the emitted slice includes the separator length. Spaced or multi-newline separators can therefore emit a chunk longer than maxChars, violating the documented channel cap behavior.
  - confidence: 0.9
- **[P2] Align the table fixture with the configured cap:** `src/agents/pi-embedded-subscribe.subscribe-embedded-pi-session.streams-soft-chunks-paragraph-preference.test.ts:53`
  - body: The test sets maxChars to 25 while expecting the full four-row table to be delivered as one block reply. Either raise the cap above the table length or assert the row-safe chunk sequence produced by the PR's max fallback behavior.
  - confidence: 0.91
- **[P2] Accept short GFM table delimiters:** `src/markdown/table-spans.ts:52`
  - body: The delimiter regex requires at least three hyphens per cell, but markdown-it's table rule accepts one or more. Tables using one- or two-dash separator cells will not be recognized as spans and can still split mid-table.
  - confidence: 0.84
- **[P3] Recognize single-column pipe tables:** `src/markdown/table-spans.ts:47-52`
  - body: After stripping edge pipes, a one-column row has one cell, so the cells.length >= 2 checks reject valid one-column tables. Those tables remain unprotected unless the helpers allow one non-empty cell and matching delimiter cells.
  - confidence: 0.78
