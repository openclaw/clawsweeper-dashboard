## Summary

Keep this PR open, but do not merge it as-is: the table-span work is on the right shared chunker surface, while the paragraph-boundary fix does not survive the production subscribe path and the table parser still treats partial streamed tables as complete.

## What This Changes

This PR adds table-span parsing, wires table-safe break checks into `EmbeddedBlockChunker`, preserves raw newline separators in chunker output, and adds chunker, subscribe integration, docs, and changelog coverage.

## Best Possible Solution

Carry boundary semantics through the shared streaming path into delivered block replies, keep table-safe break detection in the chunker, and add subscribe-level regression coverage for both reconstructed multi-paragraph Markdown and partial streamed tables.

## Reproduction Assessment

Yes, at source level. The current and proposed paths can be traced from `blockChunker.append()` through `emitBlockChunk()`, where delivered block text is trimmed before `onBlockReply`, and partial header-only table spans are treated as complete by the new parser.

## Solution Assessment

No. A shared chunker/table-span fix is the right surface, but this patch must also repair the subscriber delivery boundary and keep incomplete streamed tables buffered instead of only changing raw chunker output.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve separators past the subscriber trim:** `src/agents/pi-embedded-block-chunker.ts:220-223`
  - body: Keeping the paragraph separator in the raw chunk here does not fix the runtime path because `emitBlockChunk()` immediately calls `.trimEnd()` before pushing assistant text or delivering `onBlockReply`. A streamed `# Title\n\nFirst paragraph.` split at this boundary will still be delivered as trimmed chunks, so clients that reconstruct visible text by concatenating deliveries still lose the blank-line Markdown boundary. The fix needs to carry the separator through subscriber delivery or compare/dedupe on a normalized form while preserving the delivered text.
  - confidence: 0.88
- **[P2] Keep partial streamed tables buffered:** `src/markdown/table-spans.ts:69-80`
  - body: This parser marks a table span complete as soon as it sees the header and delimiter row, then only extends through rows already present in the current buffer. During incremental streaming, a header plus delimiter can arrive before the data rows; once the buffer reaches `maxChars`, the chunker can emit that partial table before later rows arrive, recreating the split-table bug the source PR called out. Extend spans to the buffer end until a real non-table line follows, or otherwise avoid treating header-only streamed tables as safe to flush.
  - confidence: 0.82
