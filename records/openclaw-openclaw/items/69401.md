## Summary

Keep open. Current main still gives inbound metadata structural treatment, but the live channel body, some thread context, and Dreaming transcript ingestion lack an end-to-end untrusted-channel boundary; related PR #64720 is relevant but remains open and does not settle memory provenance.

## What This Changes

Request to wrap inbound channel message bodies as untrusted content and preserve channel-origin provenance through memory/Dreaming.

## Best Possible Solution

Adopt the existing nonce-backed external-content wrapper or an equivalent escaped boundary for all channel-originated live body/thread text, then persist explicit untrusted-channel provenance through transcript storage, Dreaming promotion, and retrieval policy.

## Reproduction Assessment

Yes. A source trace on current main reproduces the gap: channel BodyForAgent reaches prompt assembly as baseBodyFinal, while Dreaming later flattens user transcript text without channel-origin provenance.

## Solution Assessment

Unclear as originally proposed. The trust-boundary direction is right, but the maintainable fix should reuse the existing nonce-backed wrapper and add memory provenance rather than only introducing a fixed XML-like tag.
