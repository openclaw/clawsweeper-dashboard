## Summary

Keep open. Current main still sanitizes inbound channel text but does not structurally wrap the live body or thread context before prompt assembly, and session-memory ingestion still flattens channel-origin user text without durable trust provenance; the related open PR is only a partial, unmerged prompt-slice candidate and the memory-provenance half remains separately open. kinthaiofficial's comment adds useful design evidence about nonce/tag-breakout handling and carrying untrusted-channel provenance through retrieval.

## What This Changes

Request to structurally mark inbound channel message bodies as untrusted content and preserve source trust/provenance so channel-origin prompt injection cannot blend into instructions or durable memory.

## Best Possible Solution

Use the existing nonce-backed external-content wrapper or an equivalent framework-owned ingress-trust boundary for live body, thread, and history text, then align durable channel-origin provenance with https://github.com/openclaw/openclaw/issues/7707 while revising https://github.com/openclaw/openclaw/pull/64720 for the prompt-body slice.

## Reproduction Assessment

Yes. Source inspection on current main shows channel BodyForAgent is sanitized but not wrapped, baseBodyFinal is sent into prompt assembly as ordinary body text, thread context is prepended as plain notes, and session-memory ingestion flattens user text without channel trust provenance.

## Solution Assessment

Unclear as originally proposed. The trust-boundary direction is sound, but fixed XML-like tags are weaker than the existing nonce-backed wrapper, and the memory side needs framework-assigned provenance rather than self-reported text tags.
