## Summary

Keep open. Current main and the latest release still sanitize inbound channel text and mark metadata as untrusted, but they do not structurally wrap generic channel body/thread text or carry durable channel-origin trust provenance through memory; the related hardening PR is still open and previously found to need trust-boundary fixes.

## What This Changes

The issue requests untrusted-content framing for inbound channel message bodies and source/trust provenance that survives memory ingestion and Dreaming.

## Best Possible Solution

Land a framework-owned ingress trust boundary that wraps external channel body, thread, and history text with nonce-backed external-content semantics, while coordinating durable memory source/trust provenance through https://github.com/openclaw/openclaw/issues/7707 and https://github.com/openclaw/openclaw/issues/42647.

## Reproduction Assessment

Yes, as a source-level repro. Current main shows channel BodyForAgent and thread text are sanitized but passed into prompt assembly without wrapExternalContent, and session-memory extraction flattens transcript text without durable channel trust provenance.

## Solution Assessment

Unclear as originally proposed. Structural wrapping is the right direction, but fixed XML-like tags should be replaced by the existing nonce-backed helper, and memory trust/provenance needs a framework-owned design rather than prompt text alone.
