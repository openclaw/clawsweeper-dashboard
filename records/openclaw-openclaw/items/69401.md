## Summary

Keep open. Current main still gives inbound channel metadata structural treatment, but the live channel body and some channel context are still plain user text, and Dreaming transcript ingestion still does not retain channel-origin trust provenance; open PR #64720 is relevant but remains unmerged and does not settle the memory-provenance part.

## What This Changes

Add structural untrusted-content boundaries and provenance for inbound channel message bodies before they reach model prompts and memory/Dreaming pipelines.

## Best Possible Solution

Adopt a nonce-backed or escaped wrapper for all channel-originated live body/context text and carry explicit untrusted-channel provenance through transcript storage, Dreaming promotion, and retrieval policy.

## Reproduction Assessment

Yes. A source trace on current main is enough: channel BodyForAgent reaches prompt assembly as baseBodyFinal, metadata is separated into untrusted blocks, and Dreaming session ingestion later flattens transcript user text without channel provenance.

## Solution Assessment

Unclear as originally proposed. The trust-boundary direction is right, but the maintainable fix should reuse or match the existing nonce-backed external-content wrapper and add memory provenance rather than only adding a fixed XML-like tag.
