- `P1`: The issue covers a cross-channel prompt-injection and persistent-memory poisoning boundary that can affect real deployments and needs prompt security review.
- `impact:security`: The reported problem is an untrusted content boundary that could let channel messages influence agent instructions or tool behavior.

## Summary

Keep open. Current main still sanitizes inbound channel text and wraps some channel metadata, but generic channel body and thread text are not structurally framed as untrusted content, and session-memory indexing still flattens user text without durable channel-origin trust provenance.

## What This Changes

The issue requests structural untrusted-content framing for inbound channel message bodies and provenance-aware memory handling for channel-origin content.

## Best Possible Solution

Define a framework-owned ingress trust boundary that frames channel body, thread, and history text with nonce-backed external-content semantics and persists channel-origin trust provenance through memory ingestion, Dreaming, and retrieval policy.

## Reproduction Assessment

Yes, by source inspection. Current main shows channel BodyForAgent and thread text are sanitized but passed into prompt assembly without structural external-content framing, and session-memory indexing flattens user text after stripping inbound metadata.

## Solution Assessment

Unclear as proposed. Structural wrapping is the right direction, but fixed XML-like tags are weaker than the existing nonce-backed helper and the memory side needs framework-assigned provenance/trust rather than only prompt text tags.
