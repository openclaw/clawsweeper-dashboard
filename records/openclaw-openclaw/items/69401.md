## Label Justifications

- `P1`: The issue covers a cross-channel prompt-injection and persistent-memory poisoning boundary that can affect real tool-enabled deployments.
- `impact:security`: The reported problem is an untrusted content boundary that could let channel messages influence agent instructions or tool behavior.

## Summary

Keep open. Current main still sanitizes inbound channel text and frames some metadata, but channel body/thread text is not wrapped with the existing nonce-backed external-content boundary and session-memory ingestion still flattens user text without durable channel-origin trust provenance.

## What This Changes

Requests structural untrusted-content framing for inbound channel message bodies and durable provenance through memory.

## Best Possible Solution

Define a framework-owned ingress trust boundary that wraps channel body, thread, and history text with nonce-backed external-content semantics and persists source trust provenance through memory ingestion and retrieval policy.

## Reproduction Assessment

Yes, by source inspection. Current main sanitizes channel BodyForAgent and thread text but passes them into prompt assembly without structural external-content framing, and session-memory indexing flattens user text after stripping inbound metadata.

## Solution Assessment

Unclear as proposed. Structural wrapping is the right direction, but fixed XML-like tags are weaker than the existing nonce-backed helper and the memory side needs framework-assigned provenance/trust rather than only prompt text tags.
