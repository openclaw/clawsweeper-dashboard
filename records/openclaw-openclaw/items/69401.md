## Summary

Keep open. Current main still has source-level evidence of the reported security gap: inbound channel body text is sanitized but not nonce-framed as untrusted external content, and durable memory ingestion flattens user turns without channel-origin trust provenance.

## What This Changes

The issue requests structural untrusted-content framing for inbound channel message bodies and durable source/trust attribution through memory.

## Best Possible Solution

Define a framework-owned ingress trust boundary that uses the existing nonce-backed external-content semantics for external channel body/thread/history text and persists channel-origin trust provenance through memory ingestion, Dreaming, and retrieval policy.

## Reproduction Assessment

Yes, by source inspection. Current main still sanitizes channel BodyForAgent and thread text but sends them into prompt assembly without external-content framing, and session-memory export flattens user text without durable channel-origin trust metadata.

## Solution Assessment

Unclear as proposed. Structural wrapping is the right direction, but fixed XML-like tags are weaker than the existing nonce-backed helper, and the memory side needs framework-assigned provenance/trust rather than prompt text tags alone.
