## Summary

Keep open. Current main still has source-level evidence of the reported trust-boundary gap: channel body/thread text is sanitized but not structurally framed as external content, and session-memory export still lacks durable channel-origin trust provenance; the related body-wrapping PR and memory trust issue are both still open.

## What This Changes

The issue requests untrusted framing for inbound channel message bodies and source-trust provenance for content that can later enter memory.

## Best Possible Solution

Land a maintainer-approved security-boundary design that frames genuinely external channel body and thread text with nonce-backed external-content semantics and carries framework-owned channel-origin trust provenance through memory ingestion, Dreaming, and retrieval policy.

## Reproduction Assessment

Yes, by source inspection. Current main passes channel BodyForAgent and thread text through sanitization into prompt assembly without wrapExternalContent, and session-memory export flattens transcript text without channel trust provenance.

## Solution Assessment

Unclear as proposed. Structural wrapping is the right direction, but fixed XML-like tags are weaker than the existing nonce-backed helper, and the memory side needs framework-owned provenance/trust rather than prompt text tags alone.
