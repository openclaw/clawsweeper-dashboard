## Summary

Keep open. Current main still leaves the live inbound channel body as ordinary prompt text, while Dreaming/session ingestion still flattens user transcript text without channel-origin trust provenance; the discussion's nonce/escaping and memory-provenance concerns remain valid, and the open implementation candidate is partial and unmerged.

## What This Changes

Request to wrap inbound channel message bodies as untrusted content and preserve channel-origin provenance through Dreaming and memory handling.

## Best Possible Solution

Adopt the existing nonce-backed external-content wrapper or an equivalent explicit ingress-trust wrapper for channel body and thread/history text, then coordinate durable untrusted-channel provenance and retrieval policy with https://github.com/openclaw/openclaw/issues/7707; a corrected https://github.com/openclaw/openclaw/pull/64720 can cover the prompt-body slice.

## Reproduction Assessment

Yes. A source trace on current main shows channel BodyForAgent reaches prompt assembly as baseBodyFinal without wrapExternalContent, and buildSessionEntry later strips/normalizes transcript user text into User lines without channel trust provenance.

## Solution Assessment

Unclear as originally proposed. The trust-boundary direction is right, but the maintainable fix should reuse nonce-backed external-content boundaries and add framework-assigned memory provenance rather than only adding a literal XML-like tag.
