## Summary

Keep open: current main still sanitizes inbound channel text and wraps some metadata, but the live message body, thread context, and durable memory provenance do not yet have the requested trust boundary.

## What This Changes

Request to wrap inbound channel message bodies as untrusted content and carry channel-origin trust/provenance through memory.

## Best Possible Solution

Design a framework-owned ingress trust boundary that uses nonce-backed wrapping for channel body and thread/history text, then preserve channel-origin provenance through memory retrieval policy in coordination with the open memory trust issues.

## Reproduction Assessment

Yes. Source inspection on current main shows channel BodyForAgent is sanitized but not wrapped, baseBodyFinal and threadContextNote enter prompt assembly as ordinary prompt text, and session-memory ingestion renders flattened user text without channel trust provenance.

## Solution Assessment

Unclear as originally proposed. The trust-boundary direction is sound, but a fixed tag wrapper is not the safest shape, and the complete solution needs explicit ingress trust classification plus durable memory provenance rather than body-only text decoration.
