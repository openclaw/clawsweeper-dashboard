## Summary

Keep open. Current main still has source-level gaps for structurally framing inbound channel body/thread text as untrusted content and for carrying channel-origin trust provenance into memory, while the linked body-wrapping PR and related memory-trust issue remain open.

## What This Changes

Request structural untrusted-content framing for inbound channel messages plus provenance-aware memory handling to reduce prompt-injection and memory-poisoning risk.

## Best Possible Solution

Define a framework-owned ingress trust boundary that uses the existing nonce-backed external-content semantics for channel body, thread, and history text, then persist channel-origin trust provenance through memory ingestion, Dreaming, and retrieval policy.

## Reproduction Assessment

Yes, by source inspection. Current main sanitizes inbound channel text but does not structurally frame channel body/thread text as external untrusted content, and session-memory indexing flattens user text without durable channel trust provenance.

## Solution Assessment

Unclear as proposed. Structural wrapping is the right direction, but the maintainable fix should reuse the existing nonce-backed external-content helper and include a provenance/trust contract for memory rather than only adding fixed XML-like prompt tags.
