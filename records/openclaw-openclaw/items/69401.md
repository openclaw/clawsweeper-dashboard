## Summary

Keep open. Current main still passes the central inbound channel body and thread context into prompt assembly as ordinary user text, while session-memory ingestion strips channel metadata and preserves only flattened User/Assistant text; the open prompt-body PR is relevant but unmerged and the memory-provenance half is still tracked separately.

## What This Changes

Request to wrap inbound channel message bodies and preserve untrusted-source attribution through memory/Dreaming so channel-originated content is not treated as trusted instruction text.

## Best Possible Solution

Use the existing nonce-backed external-content wrapper or an equivalent framework-owned ingress-trust wrapper for live body, thread, and history text, then align durable channel-origin provenance with https://github.com/openclaw/openclaw/issues/7707 while revising https://github.com/openclaw/openclaw/pull/64720 for the prompt-body slice.

## Reproduction Assessment

Yes. Source inspection on current main shows buildChannelTurnContext/finalizeInboundContext sanitize but do not wrap BodyForAgent, runPreparedReply sends baseBodyFinal into prompt assembly, and buildSessionEntry later renders flattened User lines without channel trust provenance.

## Solution Assessment

Unclear as originally proposed. The trust-boundary direction is right, but a fixed XML-like tag is weaker than the existing nonce-backed wrapper and the durable memory side needs framework-assigned provenance rather than agent- or text-self-reported trust.
