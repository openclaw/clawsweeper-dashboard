## Summary

Keep open. Current main still normalizes and sanitizes inbound channel body and thread text without structural untrusted-content wrapping, and session-memory ingestion still flattens user text without channel trust provenance; the related prompt-body PR is open and partial, while memory/provenance design remains tracked separately.

## What This Changes

Add structural untrusted-content boundaries for inbound channel messages and preserve untrusted channel provenance into prompt and memory paths.

## Best Possible Solution

Use nonce-backed external-content wrapping or an equivalent ingress-trust contract for live body, thread, and history text, then coordinate durable channel-origin provenance with the open memory/provenance design work.

## Reproduction Assessment

Yes. Source inspection on current main shows channel BodyForAgent is normalized and sanitized but not structurally wrapped, runPreparedReply passes body and thread context into prompt assembly as ordinary text, and buildSessionEntry flattens user text without channel trust provenance.

## Solution Assessment

Unclear as originally proposed. The trust-boundary direction is sound, but the fixed XML-like tag should be replaced with nonce-backed external-content boundaries or structured ingress trust metadata, and the memory side needs framework-assigned provenance rather than text-only tagging.
