## Summary

Keep open. Current main still sanitizes inbound channel text and wraps metadata, but it does not structurally frame generic channel body/thread text or carry channel-origin trust provenance through memory; the related PR is open, conflicting, and still needs trust-model fixes.

## What This Changes

Request untrusted-content framing for inbound channel message bodies and durable source trust/provenance through memory.

## Best Possible Solution

Define a framework-owned ingress trust boundary that frames channel body, thread, and history text with nonce-backed external-content semantics and persists channel-origin trust provenance through memory ingestion, Dreaming, and retrieval policy.

## Reproduction Assessment

Yes. Source inspection on current main shows BodyForAgent and thread text are sanitized but passed to prompt assembly without wrapExternalContent, while session-memory indexing flattens user text after stripping inbound metadata.

## Solution Assessment

Unclear as proposed. Structural wrapping is the right direction, but fixed XML-like tags are weaker than the existing nonce-backed helper and the memory side needs framework-assigned provenance/trust rather than only prompt text tags.
