## Summary

Keep open: current main and the latest release still exclude enabled-only legacy external hook plugins from Gateway startup unless they declare hook capability metadata or explicit hook policy config, and the open closing PR has not landed.

## What This Changes

The issue reports that external plugins from `plugins.load.paths` load in CLI commands but are silently skipped by the long-running Gateway daemon, so registered hooks such as security audit hooks never fire.

## Best Possible Solution
