## Summary

Keep open: current main and the latest release still gate Gateway hook startup on manifest hook capability or explicit hook policy, so enabled-only legacy external hook plugins can still be omitted from the daemon. The open linked fix path at https://github.com/openclaw/openclaw/pull/78229 is unmerged and still needs review/proof, so this issue remains necessary.

## What This Changes

Reports a regression where the long-running Gateway silently skips enabled external hook plugins from configured plugin load paths even though CLI plugin inspection loads them.

## Best Possible Solution

Land a narrow startup-planner fix that starts explicitly enabled legacy external hook plugins without bypassing existing activation and safety gates, with regression tests and redacted real Gateway proof.

## Reproduction Assessment

Yes, source-level. Current main excludes enabled-only legacy external hook plugins before Gateway onlyPluginIds, and the issue body supplies concrete Docker logs plus hook side-effect checks for the runtime symptom.

## Solution Assessment

Yes, the startup-planner layer is the right place to restore daemon loading. The open PR direction is plausible, but the final fix must preserve activation opt-outs and include real Gateway proof before the linked implementation can close this issue.
