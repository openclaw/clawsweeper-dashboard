## Summary

## What This Changes

The PR adds a control-diagnostic token helper and uses it to suppress leading NO_SUBPROCESS_STARTED text only for Matrix isolated cron delivery while keeping non-Matrix delivery unchanged.

## Best Possible Solution

Land only after maintainers explicitly accept the Matrix-only suppression contract, or adjust the branch to a broader documented control-diagnostic policy before merge.

## Reproduction Assessment

Yes, source-reproducible: current main's cron delivery normalization only strips NO_REPLY-style silent replies, so a leading NO_SUBPROCESS_STARTED text remains deliverable. I did not run tests in this read-only review, but the PR's focused tests demonstrate the intended Matrix suppression and non-Matrix preservation.

## Solution Assessment

Acceptable mitigation, but not a fully settled best fix until maintainers accept the Matrix-only policy. The implementation is placed at the cron delivery boundary after target resolution and keeps token matching centralized, which is the right shape if that policy is intended.
