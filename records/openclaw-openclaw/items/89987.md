## Summary

## What This Changes

The PR adds Matrix-only suppression for leading cron `NO_SUBPROCESS_STARTED` diagnostics before cron delivery and adds token/dispatch regression tests.

## Best Possible Solution

Keep the Matrix-only suppression at the cron delivery boundary, but restrict the control diagnostic match to token-only or colon-prefixed text and prove the real Matrix delivery behavior.

## Reproduction Assessment

No live current-main reproduction was established here. Source inspection shows current main only suppresses `NO_REPLY` and would pass nonempty `NO_SUBPROCESS_STARTED` text into cron outbound delivery.

## Solution Assessment

No. Suppressing before outbound delivery is the right boundary, but the delimiter is too broad and the external proof should exercise a real Matrix delivery path.
