## Summary

Keep this PR open: the patch is close, but the new classifier is still broad enough to suppress legitimate Matrix cron summaries that start with `NO_SUBPROCESS_STARTED`, and the supplied proof does not show live Matrix adapter or room behavior after the fix.

## What This Changes

The PR adds a control-diagnostic classifier and uses it to suppress leading `NO_SUBPROCESS_STARTED` cron text from Matrix delivery while keeping non-Matrix delivery unchanged.

## Best Possible Solution

Keep the Matrix-only suppression, but narrow the classifier to the proven control-diagnostic shape and add negative Matrix coverage for token-led explanatory summaries before merge.

## Reproduction Assessment

Source-reproducible: the PR regex accepts whitespace after the diagnostic token, and the Matrix delivery path turns a match into non-delivery. I did not run tests because this review is read-only.

## Solution Assessment

No; the current patch is close but still too broad because it can hide token-led explanatory summaries without a source-backed whitespace diagnostic contract. A narrower exact or colon-prefixed match plus negative delivery coverage is the safer fix.
