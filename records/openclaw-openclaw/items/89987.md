## What This Changes

The branch adds a control-diagnostic reply matcher and applies it to isolated cron delivery so Matrix targets suppress leading `NO_SUBPROCESS_STARTED` text while non-Matrix delivery and ordinary token mentions remain deliverable.

## Best Possible Solution

Land the scoped Matrix-only suppression if maintainers accept the diagnostic as control output, while keeping the non-Matrix and substantive-text safeguards covered by tests.

## Reproduction Assessment

Yes. Current main only filters `NO_REPLY` in cron delivery normalization, so a leading `NO_SUBPROCESS_STARTED` text payload remains nonempty and reaches direct delivery by source inspection.

## Solution Assessment

Yes, if maintainers accept the behavior change. Cron delivery dispatch has both resolved channel and fallback-summary context, while suppressing in the Matrix adapter would lose cron semantics and suppressing earlier would affect non-Matrix delivery.
