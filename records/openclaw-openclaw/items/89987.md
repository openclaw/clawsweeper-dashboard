Full review comments:

- **[P2] Narrow the control diagnostic delimiter:** `src/auto-reply/tokens.ts:196`
  - body: `isControlDiagnosticReplyText` treats `NO_SUBPROCESS_STARTED` followed by any whitespace as a control diagnostic. Because Matrix cron delivery passes this into `finishSilentReplyDelivery`, a substantive reply like `NO_SUBPROCESS_STARTED means the prior run lacked exec` would be dropped and marked attempted, so the main-summary fallback will not recover it. Please match only the exact generated diagnostic form, such as the colon-delimited control line, and add that regression.
