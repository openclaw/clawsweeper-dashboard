## Summary

Keep this PR open for maintainer review: the patch is a small, well-covered cron delivery fix, but it deliberately changes Matrix cron message delivery semantics and needs maintainer acceptance before merge.

## What This Changes

The PR adds a `NO_SUBPROCESS_STARTED` control-diagnostic matcher, uses it to suppress leading diagnostics only for Matrix isolated cron delivery, and adds focused regression coverage.

## Best Possible Solution

Land the narrow suppression only if maintainers accept that `NO_SUBPROCESS_STARTED` is a Matrix cron control diagnostic rather than a deliverable Matrix message, while preserving non-Matrix delivery as covered by the tests.

## Reproduction Assessment

Yes from source inspection: current `main` only normalizes `NO_REPLY`, so a synthesized leading `NO_SUBPROCESS_STARTED: ...` text would pass through `dispatchCronDelivery` to outbound delivery when a Matrix cron target is resolved. I did not run the test suite because this review is read-only and test commands can write caches or artifacts.

## Solution Assessment

Yes, conditionally: the implementation is the narrowest maintainable fix if maintainers accept Matrix-only suppression. If they do not, the better solution is a channel-neutral or explicitly documented diagnostic delivery policy.
