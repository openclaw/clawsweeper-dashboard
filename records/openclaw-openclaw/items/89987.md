Adds a classifier for leading `NO_SUBPROCESS_STARTED` replies and suppresses them before Matrix cron delivery while leaving non-Matrix cron delivery and `NO_REQUIRED_TOOL_AVAILABLE` deliverable.

## Best Possible Solution

Land this only after a redacted real Matrix cron run proves the no-send/no-fallback behavior and maintainers accept Matrix-only diagnostic suppression; otherwise move the policy to an approved channel or generic delivery contract.

## Reproduction Assessment
