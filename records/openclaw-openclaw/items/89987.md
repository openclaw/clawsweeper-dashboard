## Summary

## What This Changes

## Best Possible Solution

Keep the Matrix-only suppression, but narrow it to the exact or colon-prefixed control diagnostic shape, add leading-explanatory Matrix negative coverage, and require redacted live Matrix proof before merge.

## Reproduction Assessment

Yes for the PR-introduced defect: source inspection shows the new regex matches `NO_SUBPROCESS_STARTED` followed by whitespace, and the Matrix delivery path turns matched text into an empty payload. I did not run a live Matrix repro in this read-only review.

## Solution Assessment

No: suppressing Matrix cron control diagnostics is the right layer, but this implementation should distinguish the colon-form diagnostic from leading explanatory text and be backed by live Matrix proof.
