## Best Possible Solution

Land the dispatch-level suppression after redacted live Matrix cron proof confirms the diagnostic is not delivered and no fallback summary is queued, or after a maintainer explicitly accepts source-and-test proof for this narrow suppression.

## Reproduction Assessment

Yes, source-reproducible: current main and `v2026.6.1` only suppress `NO_REPLY`, so a Matrix cron synthesized text beginning with `NO_SUBPROCESS_STARTED` would pass into outbound delivery. I did not run a live Matrix reproduction in this read-only review.

## Solution Assessment

Yes, with a proof gap: cron dispatch is the right owner boundary because it already normalizes silent cron replies and controls fallback-summary prevention. Suppressing in the Matrix adapter would be broader and could affect non-cron Matrix text.
