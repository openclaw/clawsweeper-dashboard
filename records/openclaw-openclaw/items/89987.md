## Summary

Keep this PR open: the change addresses a real Matrix cron delivery leak, but the matcher can still suppress substantive Matrix replies that start with the diagnostic token, and the contributor proof does not show an actual after-fix Matrix cron delivery or redacted room/run log.

## What This Changes

The PR adds a NO_SUBPROCESS_STARTED control-diagnostic detector and suppresses matching Matrix cron delivery text while leaving non-Matrix delivery unchanged.

## Best Possible Solution

Land a narrow control-diagnostic contract that suppresses only the exact non-user diagnostic shape, keeps substantive leading explanations deliverable, and includes maintainer-accepted Matrix scope plus real after-fix Matrix cron proof.

## Reproduction Assessment

Yes for the patch defect: source inspection shows `isControlDiagnosticReplyText` matches `NO_SUBPROCESS_STARTED: <any text>` and Matrix delivery converts that to no outbound payload with deliveryAttempted=true. The original user-facing leak is also source-reproducible on current main because no control-diagnostic suppression exists there.

## Solution Assessment

No: the PR is a plausible mitigation, but not yet the best fix because the suppression predicate is broader than the stated control diagnostic and the Matrix-only policy lacks real Matrix run proof. A safer fix is an exact diagnostic contract plus a regression for leading substantive text.
