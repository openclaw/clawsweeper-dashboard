## What This Changes

Adds an auto-reply control diagnostic classifier and uses it to suppress leading `NO_SUBPROCESS_STARTED` text from Matrix cron delivery, with token and cron dispatch tests.

## Best Possible Solution

Keep the Matrix-only suppression, but narrow it to the exact or colon-prefixed diagnostic shape, add leading-explanatory Matrix negative coverage, and require redacted live Matrix proof before merge.

## Reproduction Assessment

Yes for the PR-introduced defect by source inspection: the new regex matches `NO_SUBPROCESS_STARTED` followed by whitespace, and Matrix delivery filters the matched text out before outbound send. I did not run a live Matrix reproduction in this read-only review.

## Solution Assessment

No: suppressing Matrix cron control diagnostics is the right layer, but this implementation should distinguish the colon-form diagnostic from leading explanatory text and be backed by live Matrix proof.
