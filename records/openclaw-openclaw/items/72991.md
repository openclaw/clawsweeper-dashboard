## Summary

Keep open. Current main still lacks an admin-owned policy hook source, machine-level discovery roots, policy-first dispatch/deny semantics, and disable immunity; the request is concrete but security-sensitive and needs maintainer design before implementation.

## What This Changes

The issue requests a machine-wide, admin-managed hook discovery tier that always loads before user-controlled hooks and cannot be disabled by user configuration or the hooks CLI.

## Best Possible Solution

Define a maintainer-approved core policy-hook architecture, then implement hardened POSIX and Windows discovery, deterministic policy-first dispatch with honored block decisions, fail-open crash handling, disable immunity, policy-aware CLI/status output, docs, changelog, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing capability is high-confidence from current source and docs because all existing hook tiers are bundled, plugin, managed user-state, or workspace scoped and remain controlled by user configuration or plugin enablement.

## Solution Assessment

Unclear until maintainer security and product design are settled. If OpenClaw accepts mandatory fleet governance hooks, the core hook loader, dispatcher, config schema, and hooks CLI are the right layer because plugin/hook-pack surfaces cannot enforce OS-owned trust roots or immunity from user disable settings.
