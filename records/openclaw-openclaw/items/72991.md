## Summary

Keep this issue open. Current main and the latest release still expose only user/plugin/workspace-managed hook tiers, while the requested admin-owned, non-disableable machine policy tier is a security-sensitive product/API design rather than stale cleanup.

## What This Changes

The issue requests a machine-wide, admin-managed hook discovery tier that loads before user-controlled hooks and cannot be disabled by ordinary user config or `openclaw hooks disable`.

## Best Possible Solution

Design a versioned, machine-owned hook/policy contract in core that proves OS-level ownership, exposes inventory/CLI status, runs before user-controlled hooks, and composes with typed plugin enforcement paths while keeping domain policies in plugins or ClawHub packages.

## Reproduction Assessment

Yes, at source level rather than as a bug reproduction. Current main and v2026.6.8 show only bundled, plugin, managed, and workspace hook tiers, with no admin-owned non-disableable policy source.

## Solution Assessment

Unclear for the exact proposed shape. The right layer is a maintainer-approved core policy seam plus plugin-owned domain policy packs; adding a one-off `/etc` loader without typed hook/API and security design would be too narrow.
