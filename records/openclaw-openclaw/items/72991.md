## Summary

Keep open. Current main still lacks an admin-owned, disable-immune hook discovery tier, and the requested machine-wide policy behavior is security-sensitive core design rather than a stale or ClawHub-only request.

## What This Changes

Request to add machine-wide policy hooks that load from admin-owned OS trust roots, run before user hooks, cannot be disabled by user config or CLI toggles, and expose policy-aware CLI/status output.

## Best Possible Solution

Define a maintainer-approved core policy-hook architecture, then implement hardened POSIX and Windows discovery, deterministic policy-first dispatch, disable immunity, policy-aware CLI/status output, docs, changelog, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing capability is high-confidence from current source and docs because all existing hook tiers are bundled, plugin, managed user-state, or workspace scoped and remain controlled by user configuration or plugin enablement.

## Solution Assessment

Unclear until maintainer security and product design are settled. If OpenClaw accepts mandatory fleet governance hooks, the core hook loader, dispatcher, config schema, and hooks CLI are the right layer because ordinary plugins cannot enforce OS-owned trust roots or immunity from user disable settings.

## Security Review

Status: needs_attention

Summary: The request is security-sensitive because it would introduce mandatory admin-controlled hook execution and blocking semantics ahead of user-controlled hooks.
