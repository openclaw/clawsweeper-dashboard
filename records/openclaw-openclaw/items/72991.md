## Summary

Keep open. Current main still has only bundled, plugin, managed, and workspace hook sources, and the requested machine-wide policy tier, disable immunity, policy-first dispatch, block semantics, and policy inventory CLI are not implemented.

## What This Changes

Add an admin-managed machine-wide hook policy tier that loads before user-controlled hooks, cannot be disabled by user config, and exposes policy-specific CLI inventory.

## Best Possible Solution

Define a maintainer-approved policy-hook architecture, then implement hardened discovery, source precedence, dispatch result semantics, CLI/status visibility, docs, changelog, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing capability is source-verifiable because current hook types, discovery, loader, dispatcher, CLI, and docs do not define a machine-wide policy tier.

## Solution Assessment

Unclear until maintainers approve the security and product design. If OpenClaw accepts mandatory fleet-governance hooks, the core hook loader, dispatcher, config/schema, and CLI are the right layer because ordinary plugins and user config cannot enforce machine-owned trust roots or disable immunity.
