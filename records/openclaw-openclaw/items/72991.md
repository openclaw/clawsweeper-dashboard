## Summary

Keep open. Current main still lacks an admin-owned machine-wide hook policy tier: hook sources, discovery, config gating, dispatch, CLI inventory, docs, and tests all model user/plugin/workspace-managed hooks, while the requested OS-owned, non-disableable policy layer is a core security/product design change rather than a ClawHub-only plugin.

## What This Changes

Add an admin-managed machine-wide hook discovery tier whose hooks always load first, cannot be disabled by user config or `openclaw hooks disable`, and are visible in policy-specific CLI inventory.

## Best Possible Solution

Define a maintainer-approved core policy-hook design, then implement hardened POSIX and Windows discovery, deterministic policy-first dispatch, explicit fail-open crash handling with honored block decisions, user-disable immunity, policy-aware CLI output, public docs, and regression tests.

## Reproduction Assessment

Not applicable as a feature request. The missing capability is high-confidence from current main because source, tests, and docs expose only bundled, plugin, managed/extraDir, and workspace hook tiers gated by user-controlled configuration.

## Solution Assessment

Unclear until maintainer security design is settled. If OpenClaw accepts mandatory enterprise hook governance, core hook loading/dispatch/CLI is the right layer because the existing plugin and hook-pack surfaces cannot enforce OS-owned roots, trusted ordering, or user-disable immunity.
