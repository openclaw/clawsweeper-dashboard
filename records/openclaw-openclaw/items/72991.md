## Summary

Keep open. Current main still has only bundled, plugin, managed, and workspace hook sources, and internal hook loading/CLI disablement remain controlled by user config rather than an admin-owned policy tier. The request is security-sensitive core design work because it needs OS trust roots, policy-first ordering, disable immunity, and audited inventory semantics that cannot be supplied by a normal ClawHub plugin alone.

## What This Changes

Add an admin-managed machine-wide policy hook discovery tier that always loads before user-controlled hooks, cannot be disabled by user hook controls, and is visible in hook inventory.

## Best Possible Solution

Define a maintainer-approved core policy-hook architecture, then implement hardened POSIX and Windows discovery, deterministic policy-first dispatch, fail-open crash handling with honored block decisions, disable immunity, policy-aware CLI output, docs, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing capability is high-confidence from current source and docs because all existing hook tiers are bundled, plugin, managed user-state, or workspace scoped and remain gated by user-controlled configuration.

## Solution Assessment

Unclear until maintainer security/product design is settled. If OpenClaw accepts this enterprise governance capability, the core hook loader, dispatcher, and hooks CLI are the right layer because current plugin/hook-pack surfaces cannot enforce OS-owned roots, trusted ordering, or user-disable immunity.
