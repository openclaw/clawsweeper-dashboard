## Summary

Keep open. Current main still lacks an admin-owned, disable-immune policy hook tier; discovery, precedence, startup gating, CLI toggles, and dispatch all remain limited to bundled, plugin, managed, workspace, or legacy user-configured hooks.

## What This Changes

Request to add a mandatory machine-wide hook policy tier that loads before user-controlled hooks, cannot be disabled by users, and exposes policy-specific inventory and CLI status.

## Best Possible Solution

Define a maintainer-approved core policy-hook architecture, then implement hardened cross-platform discovery, policy-first dispatch, disable immunity, CLI/status visibility, docs, changelog, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing capability is high-confidence from current source and docs because all implemented hook tiers are bundled, plugin, managed, workspace, or legacy user-configured paths.

## Solution Assessment

Unclear until maintainer security and product design are settled. If OpenClaw accepts mandatory fleet governance hooks, the core hook loader, dispatcher, config schema, and hooks CLI are the right layer because ordinary plugins cannot enforce OS-owned trust roots or immunity from user disable settings.
