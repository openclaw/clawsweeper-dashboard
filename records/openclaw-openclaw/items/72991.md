## Summary

Keep open. Current main still has only bundled, plugin, user-managed, and workspace hook tiers; there is no disable-immune admin policy source or policy-first blocking dispatch, so this remains a valid security-sensitive core design request.

## What This Changes

The issue requests an admin-managed, machine-wide hook policy tier whose hooks always load before user-controlled hooks and cannot be disabled by user configuration.

## Best Possible Solution

Define a maintainer-approved core policy-hook architecture, then implement hardened cross-platform discovery, policy-first dispatch, disable immunity, CLI/status visibility, docs, changelog, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing capability is high-confidence from current source and docs because all existing hook tiers are bundled, plugin, user-managed, or workspace scoped and remain controlled by user configuration or plugin enablement.

## Solution Assessment

Unclear until maintainer security and product design are settled. If OpenClaw accepts mandatory fleet governance hooks, the core hook loader, dispatcher, config schema, and hooks CLI are the right layer because ordinary plugins cannot enforce OS-owned trust roots or immunity from user disable settings.
