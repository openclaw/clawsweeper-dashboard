## Summary

Keep open. Current main still has only user-scoped or plugin-scoped hook discovery and user-controlled disable paths; the requested admin-owned, disable-immune policy tier is security-sensitive core behavior that needs maintainer design.

## What This Changes

Expose a machine-wide admin policy hook tier that loads before user hooks, resists user disables, and reports policy inventory in the hooks CLI.

## Best Possible Solution

Define a maintainer-approved core policy-hook architecture, then implement hardened POSIX and Windows discovery, policy-first dispatch, disable immunity, CLI/status reporting, docs, changelog, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing capability is high-confidence from current source and docs because all existing hook discovery and toggles are bundled, plugin, managed user-state, workspace, or user config based.

## Solution Assessment

Unclear until maintainer security and product direction is decided. If accepted, this belongs in core hook discovery, dispatch, config, and CLI rather than ClawHub because external plugins cannot enforce OS-owned trust roots or immunity from user-controlled settings.
