## Summary

Keep open. The prior ClawSweeper review remains correct on current main: OpenClaw still has only user/plugin/workspace-managed hook discovery, user-config gating, and observation-style internal hook dispatch, so there is no admin-owned, disable-immune policy tier.

## What This Changes

Request a machine-wide, admin-controlled hook policy tier that loads before user-controlled hooks and cannot be disabled by user config or hook CLI commands.

## Best Possible Solution

Define a maintainer-approved core policy-hook architecture, then implement hardened POSIX and Windows discovery, deterministic policy-first dispatch, disable immunity, policy-aware CLI/status output, docs, changelog, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing capability is high-confidence from current source and docs because no current hook source, loader, CLI, config schema, or dispatch path implements machine-wide policy hooks.

## Solution Assessment

Unclear until maintainer security and product design are settled. If OpenClaw accepts mandatory fleet governance hooks, the core hook loader, dispatcher, config schema, and hooks CLI are the right layer because ordinary plugins cannot enforce OS-owned trust roots or immunity from user disable settings.
