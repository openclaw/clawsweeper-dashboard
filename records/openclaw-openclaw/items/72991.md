## Summary

Keep this issue open. Current main and v2026.6.10 still lack an admin-owned, non-disableable machine policy hook tier, and the open implementation candidate at https://github.com/openclaw/openclaw/pull/76909 remains draft/conflicting and security-sensitive rather than a cleanup-close reason.

## What This Changes

Request an admin-managed machine-wide hook or policy discovery tier that always loads before user hooks and cannot be disabled by user config or `openclaw hooks disable`.

## Best Possible Solution

Approve a versioned machine-policy design with explicit admin ownership checks, precedence, audit output, and failure semantics, then land it through a reviewed core/plugin boundary with docs and upgrade proof.

## Reproduction Assessment

Yes, at source level rather than as a bug reproduction: current main enumerates only bundled, managed, workspace, plugin hook sources and four plugin origins, and v2026.6.10 shows the same missing machine-policy tier.

## Solution Assessment

Unclear for the exact proposed fifth-tier shape. The related system-plugin PR is a plausible candidate, but a machine-wide policy surface needs maintainer security/product approval before the implementation path can be called the best fix.
