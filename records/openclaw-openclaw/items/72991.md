## Summary

Keep this issue open. Current main and v2026.6.10 still lack an admin-owned, non-disableable machine policy hook/plugin tier, and the open candidate PR is draft/conflicting with unresolved product and security-boundary questions.

## What This Changes

Request an admin-managed machine-wide hook policy tier that loads before user hooks and cannot be disabled through user-controlled hook or plugin settings.

## Best Possible Solution

Approve a versioned machine-policy design with explicit admin ownership checks, precedence, audit output, and failure semantics, then land it through a reviewed core/plugin boundary with docs and upgrade proof.

## Reproduction Assessment

Yes, at source level rather than as a bug reproduction: current main enumerates only bundled, managed, workspace, and plugin hook sources plus four plugin origins, and v2026.6.10 shows the same missing machine-policy tier.

## Solution Assessment

Unclear for the exact proposed fifth-tier shape. The related system-plugin PR is a plausible candidate, but a machine-wide policy surface needs maintainer security/product approval before the implementation path can be called the best fix.
