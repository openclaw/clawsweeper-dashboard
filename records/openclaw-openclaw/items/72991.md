## Summary

Keep open. Current main still lacks a machine-wide, admin-owned policy hook tier, and this is not a ClawHub-only plugin request because the requested behavior depends on core-controlled OS trust roots, policy-first dispatch, disable immunity, and audit-visible CLI inventory.

## What This Changes

Request to add an admin-managed machine-wide hook policy tier with hardened OS-owned discovery roots, policy-first execution, disable immunity, CLI inventory, and defined failure semantics.

## Best Possible Solution

Define a maintainer-approved core policy-hook architecture, then implement hardened POSIX and Windows discovery, deterministic policy-first dispatch, fail-open crash handling with honored deny decisions, disable immunity, policy-aware CLI/status output, public docs, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing capability is high-confidence from current source and docs because all existing hook tiers are bundled, plugin, managed user-state, or workspace scoped and remain gated by user-controlled configuration.

## Solution Assessment

Unclear until maintainer security/product design is settled. If OpenClaw accepts this enterprise governance capability, the core hook loader, dispatcher, config/schema, and hooks CLI are the right layer because current plugin and hook-pack surfaces cannot enforce OS-owned roots, trusted ordering, or user-disable immunity.
