## Summary

Keep open. Current main still lacks the requested machine-wide policy hook tier, and the request is concrete but security-sensitive enough to need maintainer design rather than cleanup closure or an autonomous one-PR repair.

## What This Changes

Request to add an admin-managed machine-wide hook discovery tier that runs before user-controlled hooks and cannot be disabled by user-scoped OpenClaw hook settings.

## Best Possible Solution

Define a maintainer-approved core policy-hook architecture first, then implement hardened POSIX and Windows discovery, deterministic policy-first dispatch with honored block decisions, policy-aware CLI/status/docs, fail-open crash handling, disable immunity, changelog coverage, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing capability is high-confidence from current main because source, CLI, tests, and docs expose only bundled, plugin, managed, and workspace hook tiers controlled by user config or plugin enablement.

## Solution Assessment

Unclear until maintainer security and product design are settled. If OpenClaw accepts mandatory fleet governance hooks, the core hook loader, dispatcher, config/schema, and hooks CLI are the right layer because current plugin and hook-pack surfaces cannot enforce OS-owned trust roots or immunity from user disable settings.
