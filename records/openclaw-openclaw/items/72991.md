## Summary

Keep open. Current main still lacks an admin-owned, disable-immune machine-wide hook tier, and the request changes a security-sensitive trust boundary that needs maintainer product and security design before implementation.

## What This Changes

The issue requests a mandatory machine-wide hook policy tier that loads before user-controlled hooks, cannot be disabled by users, and has policy-specific CLI inventory.

## Best Possible Solution

Define a maintainer-approved core policy-hook architecture, then implement hardened cross-platform discovery, policy-first dispatch, disable immunity, CLI/status visibility, docs, changelog, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing capability is high-confidence from source and docs because current hook tiers are bundled, plugin, managed, workspace, or legacy user-configured paths.

## Solution Assessment

Unclear until maintainer security and product design are settled. If OpenClaw accepts mandatory fleet governance hooks, the core hook loader, dispatcher, config schema, and hooks CLI are the maintainable layer because ordinary plugins cannot enforce OS-owned roots or immunity from user disable settings.
