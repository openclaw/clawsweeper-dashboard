## Summary

Keep open. Current main still lacks the requested admin-owned hook policy tier, and the same author has an open related implementation PR that needs maintainer security/product review rather than issue cleanup closure.

## What This Changes

The issue requests an admin-managed machine-wide hook policy tier that loads before user hooks and cannot be disabled by user configuration.

## Best Possible Solution

Define the admin trust model first, then implement hardened core policy discovery, policy-first execution, disable immunity, CLI/status visibility, docs, changelog, and focused regression coverage, either by correcting the related PR or replacing it with a narrower design.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing capability is high-confidence from current source and docs because hook discovery still has only bundled, plugin, managed, and workspace tiers, and normal hooks remain gated by user-controlled config.

## Solution Assessment

Unclear until maintainers settle the product and security model. Core must be involved if OpenClaw accepts mandatory OS-owned policy hooks, but the exact hook-tier versus system-plugin design needs trust-boundary approval first.
