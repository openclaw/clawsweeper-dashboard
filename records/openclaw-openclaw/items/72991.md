## Summary

Keep open. Current main still lacks an admin-owned machine-wide hook or plugin policy tier, and the related same-author draft PR is an implementation candidate that needs maintainer product/security review rather than cleanup closure.

## What This Changes

Request to add an admin-managed, machine-wide hook discovery tier that runs before user-controlled hooks and cannot be disabled by user/workspace configuration.

## Best Possible Solution

Define the admin policy trust model first, then implement either hardened core policy hooks or a system-plugin design with OS ownership checks, disable immunity, ordering guarantees, CLI/status visibility, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks the requested admin-owned hook/plugin policy tier because current hook and plugin origins remain user/config/workspace/global/bundled scoped.

## Solution Assessment

Unclear until maintainers settle the product and security model. Core involvement is likely required if OpenClaw accepts non-removable machine-wide policy hooks, but the existing draft PR needs design and security review before it is the best fix path.
