## Summary

Keep open. Current main still has only user-scoped or plugin-scoped hook discovery and toggles, while the requested admin-owned, disable-immune policy tier would be security-sensitive core behavior that needs maintainer design before implementation.

## What This Changes

Add an admin-managed machine-wide hook tier that loads first, cannot be disabled by user config, and exposes policy-aware hook inventory and CLI behavior.

## Best Possible Solution

Define a maintainer-approved core policy-hook architecture, then implement hardened POSIX and Windows discovery, deterministic policy-first dispatch, disable immunity, policy-aware CLI/status output, docs, changelog, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing capability is high-confidence from current source and docs because all existing hook tiers are bundled, plugin, managed user-state, or workspace scoped and remain controlled by user configuration or plugin enablement.

## Solution Assessment

Unclear until maintainer security and product design are settled. If OpenClaw accepts mandatory fleet governance hooks, the core hook loader, dispatcher, config schema, and hooks CLI are the right layer because plugin surfaces cannot enforce OS-owned trust roots or immunity from user disable settings.
