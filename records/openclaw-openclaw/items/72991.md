## Summary

Keep open. Current main still lacks a machine-wide, admin-owned hook policy tier, disable immunity, policy-first dispatch, policy-only inventory, and deny/block semantics, so this is a valid security-sensitive core design request rather than stale cleanup or ClawHub-only work.

## What This Changes

The issue requests an admin-managed, machine-wide hook policy tier that always loads before user hooks and cannot be disabled by user-controlled configuration or CLI commands.

## Best Possible Solution

Define a maintainer-approved core policy-hook architecture, then implement hardened discovery, policy-first dispatch, disable immunity, CLI/status visibility, docs, changelog, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing capability is high-confidence from current source and docs because hook discovery, toggling, dispatch, and CLI status only cover bundled, plugin, managed, and workspace hooks.

## Solution Assessment

Unclear until maintainers approve the product and security design. If OpenClaw accepts mandatory fleet governance hooks, the core hook loader, dispatcher, config/schema, and hooks CLI are the right layer because ordinary plugins cannot enforce OS-owned trust roots or immunity from user disable settings.
