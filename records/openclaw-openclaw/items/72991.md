## Summary

Keep open. Current main still lacks an admin-owned machine-wide hook policy tier, and the request is not a ClawHub-only extension because it needs core-controlled OS trust roots, policy-first dispatch, disable immunity, and audit-visible CLI inventory. The related open PR adds user/agent-scoped hook config overrides, but it does not solve the mandatory fleet policy boundary described here.

## What This Changes

Add an admin-managed, machine-wide hook discovery tier that always loads before user-controlled hooks, cannot be disabled by user config or `openclaw hooks disable`, and is visible in hook inventory.

## Best Possible Solution

Define a maintainer-approved core policy-hook architecture, then implement hardened POSIX and Windows discovery, deterministic policy-first dispatch, fail-open crash handling with honored block decisions, disable immunity, policy-aware CLI output, docs, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing capability is high-confidence from current source, docs, and tests because all existing hook tiers are bundled, plugin, managed user-state, or workspace scoped and remain gated by user-controlled configuration.

## Solution Assessment

Unclear until maintainer security/product design is settled. If OpenClaw accepts this enterprise governance capability, the core hook loader, dispatcher, and hooks CLI are the right layer because current plugin/hook-pack surfaces cannot enforce OS-owned roots, trusted ordering, or user-disable immunity.
