## Summary

Keep open. Current main still lacks a machine-wide, admin-owned hook tier, disable immunity, policy-first dispatch, and policy inventory CLI surface, so this remains a valid security-sensitive feature request rather than a cleanup close.

## What This Changes

Request to add an admin-managed machine-wide policy hook tier that loads before user-controlled hooks, cannot be disabled by user config or `openclaw hooks disable`, and exposes policy-specific CLI inventory.

## Best Possible Solution

Define a maintainer-approved core policy-hook architecture, then implement hardened discovery, policy-first dispatch, disable immunity, CLI/status visibility, docs, changelog, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing capability is high-confidence from current source, docs, and tests because all hook sources remain bundled, plugin, managed, or workspace scoped and are still controlled by user config or plugin enablement.

## Solution Assessment

Unclear until maintainers settle the security and product design. If OpenClaw accepts mandatory fleet-governance hooks, the core hook loader, dispatcher, config/schema surface, and hooks CLI are the right layer because ordinary plugins cannot enforce OS-owned trust roots or immunity from user disable settings.
