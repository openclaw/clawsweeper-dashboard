## Summary

Keep open. Current main still has only bundled, plugin, managed, and workspace hook discovery, and normal user config/CLI paths can suppress hook loading; the requested machine-wide immutable policy tier is a new security-sensitive core feature that needs maintainer design.

## What This Changes

Request adds a machine-wide, admin-owned hook discovery tier that runs before user hooks, cannot be disabled by user config or CLI, and is visible through policy-specific hooks inventory commands.

## Best Possible Solution

Define a maintainer-approved core policy-hook architecture, then implement hardened discovery, policy-first dispatch, disable immunity, CLI/status visibility, docs, changelog, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing capability is high-confidence from source and docs because hook sources and hook CLI/config surfaces are limited to bundled, plugin, managed, and workspace tiers.

## Solution Assessment

Unclear until maintainers approve the security and product model. If OpenClaw accepts mandatory fleet governance hooks, the core hook loader, dispatcher, config schema, and hooks CLI are the right layer because ordinary plugins cannot enforce OS-owned roots or immunity from user disable settings.
