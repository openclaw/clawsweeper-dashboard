## Summary

Keep open. Current main still lacks an admin-owned, machine-wide hook policy tier with policy-first dispatch, disable immunity, hardened OS trust roots, and policy-specific CLI inventory, so this remains a security-sensitive feature request rather than cleanup-close material.

## What This Changes

The issue requests an admin-owned, machine-wide hook policy tier that always loads before user hooks and cannot be disabled through user config or the hooks CLI.

## Best Possible Solution

Define a maintainer-approved core policy-hook architecture, then implement hardened discovery, policy-first dispatch, disable immunity, CLI/status visibility, docs, changelog, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing capability is high-confidence from current source and docs because hook discovery remains bundled, plugin, managed, or workspace scoped and user config can suppress or disable normal hooks.

## Solution Assessment

Unclear until maintainers settle the product and security model. If OpenClaw accepts mandatory fleet-governance hooks, the core hook loader, dispatcher, config/schema surface, and hooks CLI are the right layer because ordinary plugins cannot enforce OS-owned trust roots or user-disable immunity.
