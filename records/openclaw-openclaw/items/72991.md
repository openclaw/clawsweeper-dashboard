## Summary

Keep open. Current main still lacks the machine-wide, admin-owned hook policy tier, and the existing discussion correctly distinguishes this from ordinary plugin or user-scoped hook configuration. The related open agent-scoped hook-policy PR is adjacent, but it does not provide OS trust roots, policy-first dispatch, user-disable immunity, or policy inventory for fleet administration.

## What This Changes

Request an admin-managed machine-wide policy hook discovery tier that always loads before user-controlled hooks, cannot be disabled by user hook controls, and is visible in hook inventory.

## Best Possible Solution

Define a maintainer-approved core policy-hook architecture, then implement hardened POSIX and Windows discovery, deterministic policy-first dispatch, fail-open crash handling with honored block decisions, disable immunity, policy-aware CLI output, docs, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing capability is high-confidence from current source, tests, and docs because all existing hook tiers are bundled, plugin, managed user-state, or workspace scoped and remain gated by user-controlled configuration.

## Solution Assessment

Unclear until maintainer security/product design is settled. If OpenClaw accepts this enterprise governance capability, the core hook loader, dispatcher, and hooks CLI are the right layer because current plugin and hook-pack surfaces cannot enforce OS-owned roots, trusted ordering, or user-disable immunity.
