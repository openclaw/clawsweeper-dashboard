## Summary

Keep this issue open. Current main and v2026.6.8 still lack an admin-owned, non-disableable machine policy hook tier, and the open same-author implementation candidate at https://github.com/openclaw/openclaw/pull/76909 is draft, unmerged, and security-sensitive rather than a cleanup reason to close this issue.

## What This Changes

Request to add an admin-owned, machine-wide hook policy tier that loads before user-controlled hooks and cannot be disabled by user config or `openclaw hooks disable`.

## Best Possible Solution

Review the open system-tier plugin PR and either land a maintainer-approved machine policy design or explicitly choose a narrower core policy seam plus plugin-owned domain policy packs.

## Reproduction Assessment

Yes, at source level rather than as a bug reproduction: current main and v2026.6.8 expose only bundled, plugin, managed, and workspace hook sources, with no admin-owned non-disableable policy source.

## Solution Assessment

Unclear for the exact proposed shape. The open system-plugin PR is a plausible candidate, but it changes security-sensitive plugin discovery, permissions, and priority semantics and still needs maintainer product/security review.
