## Summary

Keep open. Current main still lacks a machine-wide, admin-owned hook tier, disable immunity, policy-first dispatch, and policy inventory CLI surface, so this remains a valid security-sensitive core design request.

## What This Changes

Request an admin-managed machine-wide hook discovery tier that always loads before user hooks, cannot be disabled by users, and is visible in hook inventory commands.

## Best Possible Solution

Define a maintainer-approved core policy-hook architecture, then implement hardened discovery, policy-first dispatch, disable immunity, CLI/status visibility, docs, changelog, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing capability is high-confidence from current source and docs because all hook sources are bundled, plugin, managed, or workspace scoped and remain controlled by user configuration or plugin enablement.

## Solution Assessment

Unclear until maintainers settle the security and product design. If OpenClaw accepts mandatory fleet governance hooks, the core hook loader, dispatcher, config schema, and hooks CLI are the right layer because ordinary plugins cannot enforce OS-owned trust roots or immunity from user disable settings.
