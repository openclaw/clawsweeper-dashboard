Keep open: this is a recent, plausible WhatsApp delivery bug on the latest release, and current main still routes this flow through the same WhatsApp buffered reply and shared auto-reply dispatch boundary without an obvious duplicate or fix. ## What This Changes

The issue reports that WhatsApp direct-chat reply text can be silently dropped when the same agent turn also performs tool calls.

## Best Possible Solution

Add a focused regression around a direct WhatsApp turn with assistant text plus tool calls, then fix the narrow handoff that drops that visible text without changing group message-tool-only policy or intentional tool-progress suppression.

## Reproduction Assessment

No. The issue has credible live logs and a clear source path, but I do not have a high-confidence current-main reproduction because the failure is intermittent and source inspection did not isolate an unconditional drop.

## Solution Assessment

Unclear. The right direction is likely a narrow WhatsApp/shared-dispatch regression and fix, but the root cause should be pinned before changing block streaming, tool-result suppression, or source visible-reply policy.
