## Summary

Keep open: the PR targets a real claude-cli transcript gap, but the current patch has correctness and path-safety blockers before it should merge.

## What This Changes

The PR adds a read-through path for claude-cli-bound session history/listing surfaces to prefer Claude CLI project transcripts over OpenClaw delivery-mirror files.

## Best Possible Solution

Route Claude CLI reads through one validated Claude transcript resolver/parser, keep OpenClaw-store maintenance paths unchanged, and document the user-visible fix in the changelog.

## Reproduction Assessment

Yes from source inspection, but not from a live run: current main’s SSE/session-tool path reads only the OpenClaw `entry.sessionFile`, while Claude CLI continuity is stored under `.claude/projects/`. The PR’s private repro also matches that source path.

## Solution Assessment

No. Read-through is a reasonable fix direction, but this implementation should reuse the existing Claude CLI JSONL parser/path validation rather than feeding Claude-owned transcripts into OpenClaw transcript readers, and it needs the required changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use the Claude parser for CLI hook history:** `src/agents/cli-runner/session-history.ts:163`
  - body: This line can switch `loadCliSessionHistoryMessages`/reseed to the Claude-owned JSONL file, but the rest of this helper still parses OpenClaw session entries and later keeps only `type === "message"` or compaction records. Real Claude project transcripts are `type: "user"`/`"assistant"` records, as covered by `cli-session-history.claude.ts`, so the sub-agent/context-load path this PR claims to fix will still come back empty or malformed for actual claude-cli files. Reuse the existing Claude CLI parser for this branch and add a fixture with real Claude JSONL lines.
  - confidence: 0.87

  - body: `cliSessionId` is only trimmed before being joined into the path, so a stored value containing `../` can escape the computed Claude project directory before the returned candidate is handed to history/listing readers. Please validate the id as the expected Claude session filename (or at least a basename/UUID-shaped value) and confirm the realpath stays inside the project directory before returning it.
  - confidence: 0.8

  - body: This changes user-visible session history and `sessions_list` transcript paths for claude-cli sessions, but the PR does not touch `CHANGELOG.md`. Repo policy requires user-facing fixes to add a single-line entry under the active release fixes section.
  - confidence: 0.92

## Security Review

Status: needs_attention
