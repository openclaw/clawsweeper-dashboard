## Summary

Keep open: the reported behavior is source-reproducible and current main does not fully cover the PR’s sessions_list/SSE history target, but this branch still has blocking parser and filesystem-safety issues plus missing real behavior proof.

## What This Changes

The PR adds a Claude CLI transcript read-through path for CLI-bound sessions and threads it through hook history, `/sessions/:key/history`, and `sessions_list` transcript projection.

## Best Possible Solution

Route CLI-bound history and listing reads through one validated Claude transcript resolver/parser, keep maintenance/archive paths on the OpenClaw store, and preserve the existing Claude JSONL import semantics.

## Reproduction Assessment

Yes, source-reproducible: current main’s sessions_list and HTTP history paths still resolve OpenClaw transcript files only, while Claude CLI transcripts are parsed separately under the Claude project store. I did not run a live claude-cli install in this read-only review.

## Solution Assessment

No. Read-through is the right direction, but this branch should reuse the existing Claude JSONL parser and add fail-closed session-id/realpath validation before it is mergeable.
