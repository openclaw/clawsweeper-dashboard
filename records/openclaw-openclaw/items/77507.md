## Summary

Keep open: source inspection supports the reported `claude-cli` transcript gap, but this PR still routes at least one claimed fix path through the wrong parser, adds an unvalidated filesystem resolver, and lacks the required user-facing changelog entry.

## What This Changes

The PR adds a Claude CLI transcript read-through helper and threads it into CLI hook history, gateway session history reads, and `sessions_list` transcript path projection.

## Best Possible Solution

Route Claude CLI transcript reads through a single validated Claude JSONL resolver/parser, keep archive and maintenance paths on the OpenClaw store, and add regression coverage plus a one-line changelog fix entry.

## Reproduction Assessment

Yes, from source inspection rather than a live run: current main’s HTTP history and session listing paths resolve the OpenClaw `entry.sessionFile`, while the existing Claude CLI parser reads `.claude/projects` by CLI session id. The PR body’s private install report matches that source path.

## Solution Assessment

No. Read-through is a reasonable direction, but this implementation should reuse or extend the existing Claude CLI parser/resolver with strict path validation instead of feeding Claude-owned JSONL into OpenClaw transcript readers, and it needs changelog coverage.
