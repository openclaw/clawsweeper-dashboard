## Summary

Keep open. The PR addresses a credible claude-cli transcript-read bug, but the proposed branch still has blocking parser/path-safety issues and lacks after-fix proof from a real claude-cli session.

## What This Changes

The PR adds a Claude CLI transcript read-through resolver and threads it through CLI hook history, gateway session history, and the sessions_list transcript path projection.

## Best Possible Solution

Route CLI-bound reads through one validated Claude transcript resolver/parser, keep maintenance/archive paths on the OpenClaw store, and leave the separate writer-stall investigation tracked by the related open issue.

## Reproduction Assessment

Yes, source-reproducible: current main reads sessions_list and gateway history from OpenClaw transcript candidates only, while Claude CLI transcripts are parsed separately under the Claude project store. I did not run a live claude-cli install in this read-only review.

## Solution Assessment

No. Read-through is the right direction, but this branch should reuse the existing Claude JSONL parser and add fail-closed session-id and realpath validation before it is mergeable.
