## Summary

Keep this PR open, but do not merge it as-is: the read-through direction is useful, yet the branch overlaps newer current-main Claude CLI history import code and bypasses that safer projection for some surfaces.

## What This Changes

The PR adds Claude CLI transcript read-through plumbing for CLI-bound session history, hook history, and sessions list transcript paths while leaving `entry.sessionFile` in the OpenClaw session store.

## Best Possible Solution

Use the existing Claude CLI history resolver/projection as the shared source for CLI-owned transcripts, then add the missing sessions-list and HTTP history coverage with spawned-workspace handling and regression tests.

## Reproduction Assessment

Yes, source-reproducible for the remaining surfaces: current sessions-list and `/sessions/:key/history` paths still read from `entry.sessionFile`/OpenClaw transcript candidates rather than the Claude CLI binding. I did not run a live Claude CLI install in this read-only review.

## Solution Assessment

No. Reading the CLI store is the right direction, but this patch should reuse current main’s Claude CLI parser/filtering and must resolve spawned session workspaces before it is the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Prefer the spawned workspace for CLI history:** `src/gateway/sessions-history-http.ts:154`
  - body: For spawned sessions, `entry.spawnedWorkspaceDir` is the workspace the CLI run used, but this endpoint always resolves the agent's default workspace. A CLI-bound subagent or thread with its own workspace will still miss the live Claude transcript and fall back to the OpenClaw mirror path, so use the entry workspace when present.
  - confidence: 0.87
- **[P2] Route Claude CLI JSONL through the Claude parser:** `src/gateway/session-transcript-files.fs.ts:114`
  - body: Adding the Claude project file as a generic transcript candidate makes `/sessions/:key/history` parse raw Claude CLI JSONL through `readSessionTranscriptIndex`, which accepts any record with `message`. Claude files can contain sidechain/tool bookkeeping entries that current main's Claude importer filters and normalizes, so this can surface records the history UI should not show; reuse or extend the existing Claude CLI projection instead.
  - confidence: 0.84
