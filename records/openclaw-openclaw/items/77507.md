## Summary

Keep open. The reported gateway/session-history gap is still source-reproducible on current main, but this PR still has blocking parser and path-validation issues and does not include after-fix real behavior proof from a real claude-cli setup.

## What This Changes

The PR adds a Claude CLI transcript read-through resolver and threads it into CLI hook history, session history reads, and the sessions_list transcript path projection for claude-cli-bound sessions.

## Best Possible Solution

Route CLI-bound session history and listing reads through one validated Claude transcript resolver/parser, keep OpenClaw-store maintenance/archive paths unchanged, and preserve the existing Claude JSONL import semantics.

## Reproduction Assessment

Yes, from source inspection: current main’s session history and sessions_list path resolution still read OpenClaw transcript candidates only, while Claude CLI continuity is stored and parsed separately under the Claude project store. I did not run a live claude-cli install in this read-only review.

## Solution Assessment

No. Read-through is a reasonable direction, but this branch should reuse the existing Claude JSONL parser and add fail-closed session-id/path containment validation before it is safe to merge.
