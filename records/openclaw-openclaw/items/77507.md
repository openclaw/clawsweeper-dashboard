## Summary

Keep open. The bug is credible and current main does not route the MCP/SSE transcript surfaces through the Claude CLI project store, but this PR is not mergeable because it still routes Claude project JSONL through the OpenClaw hook-history parser, lacks fail-closed session-id/path validation, and has no after-fix real behavior proof.

## What This Changes

This PR adds a Claude CLI transcript read-through helper and threads it into CLI hook history, gateway session transcript candidates/history reads, and the `sessions_list` transcriptPath projection while leaving `sessionFile` writes unchanged.

## Best Possible Solution

Route CLI-bound history and listing reads through one validated Claude transcript resolver/parser, keep maintenance/archive paths on the OpenClaw store, and preserve the existing Claude JSONL import semantics.

## Reproduction Assessment

Yes, source-reproducible: current main’s `sessions_list` and `/sessions/:key/history` paths still resolve OpenClaw transcript files only, while Claude CLI transcripts are parsed separately under the Claude project store. I did not run a live claude-cli install in this read-only review.

## Solution Assessment

No. Read-through is the right direction, but this branch should reuse the existing Claude JSONL parser and add fail-closed session-id/realpath validation before it is mergeable.
