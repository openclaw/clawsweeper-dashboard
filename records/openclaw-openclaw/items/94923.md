## Summary

Keep open: the linked Windows visible-console bug is valid, but this patch is not merge-ready because it writes the hidden-launcher flag into the generated gateway script environment while the Scheduled Task action is resolved earlier from the separate installer env. As a result, normal gateway installs still choose `gateway.cmd` by default instead of `gateway.vbs`.

## What This Changes

The PR adds `OPENCLAW_WINDOWS_TASK_HIDDEN_LAUNCHER=1` to the generated gateway service environment and asserts that default in the daemon environment test.

## Best Possible Solution

Set the hidden-launcher default where `schtasks.ts` resolves `taskLaunchPath`, or pass the flag into the installer env before that decision, then cover fresh/update task actions and provide redacted live Windows proof.

## Reproduction Assessment

Yes at source level: current main uses `gateway.cmd` unless the scheduler installer env has a truthy hidden-launcher flag, and the linked issue supplies Windows 11 reproduction steps for the visible console.

## Solution Assessment

No: the PR sets the flag too late in the generated command-script environment, while the Scheduled Task launch path is chosen from the separate installer env.
