## Summary

## What This Changes

## Best Possible Solution

Set the hidden-launcher default at the scheduler/install env boundary or make the launcher resolver consume the merged install environment, then add fresh/update task-action coverage and redacted Windows proof.

## Reproduction Assessment

Yes at source level: current scheduler code resolves `.cmd` versus `.vbs` from the install env, while this PR places the flag only in the generated service environment. I did not run a native Windows desktop repro in this read-only review.

## Solution Assessment

No. Reusing the existing VBS hidden launcher is the right direction, but this patch applies the default too late; the maintainable fix should set the default before the task action is resolved and prove fresh/update behavior.
