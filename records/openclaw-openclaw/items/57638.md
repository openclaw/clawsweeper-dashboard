## Summary

Keep open. Current main still has per-job cron model, fallback, delivery, and context-budget behavior, but it does not expose or apply the requested fleet-wide `cron.defaults` surface; the related #9155 remains only a narrower delivery-default tracker.

## What This Changes

Request a fleet-wide `cron.defaults` config for `agentTurn` cron jobs to provide default model/fallbacks, delivery, contextTokens, and possibly sessionTarget when individual jobs omit them.

## Best Possible Solution

If maintainers accept the feature, add an explicit `cron.defaults` contract in core config types, zod/generated schema metadata, docs, and cron tests. Apply it only to `agentTurn` jobs when the corresponding per-job payload or delivery field is absent, preserve explicit job fields as highest precedence, leave `systemEvent` jobs unaffected, and coordinate the delivery-only portion with #9155.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is high confidence: static source/docs inspection shows `cron.defaults` is absent from the config contract and from model, delivery, and context-token resolution.

## Solution Assessment

Unclear until maintainers choose the product semantics. The narrow maintainable direction would be an opt-in core config surface rather than changing existing default delivery or model behavior globally.
