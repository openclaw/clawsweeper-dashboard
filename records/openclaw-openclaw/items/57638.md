## Summary

Keep open. Current main supports per-job cron model, fallback, delivery, and context-budget behavior, but it still has no fleet-wide `cron.defaults` config surface; related #9155 only covers the narrower delivery-default subset.

## What This Changes

The issue requests a fleet-wide `cron.defaults` config for agent-turn cron model, delivery, and context token defaults.

## Best Possible Solution

Add an opt-in first-class `cron.defaults` config surface in core, update types/zod/generated schema/docs/tests, and apply defaults only to missing `agentTurn` fields while preserving explicit job settings and leaving `systemEvent` jobs unchanged.

## Reproduction Assessment

Yes for the missing surface: source inspection against current main shows strict cron config rejects `cron.defaults` and the model, delivery, and context-token runtime paths never read it. No live cron execution is needed to verify this absence.

## Solution Assessment

Unclear until maintainers approve the public config contract. If approved, the requested precedence is a maintainable direction, with the safer implementation being opt-in defaults plus focused schema, runtime, and docs coverage.
