## Summary

Keep open. Current main still supports only per-job cron model/fallback/delivery behavior and agent/model defaults; it does not expose the requested fleet-wide `cron.defaults` surface for model, delivery, or context-token defaults.

## What This Changes

Request a `cron.defaults` config surface for missing `agentTurn` cron model, delivery, contextTokens, and possibly sessionTarget values.

## Best Possible Solution

Approve a typed `cron.defaults` contract, then implement it in config types/schema/help/docs and isolated `agentTurn` normalization/runtime with precedence tests that preserve explicit per-job settings and leave `systemEvent` jobs unchanged.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main gives high-confidence proof that the requested config surface and precedence behavior are absent.

## Solution Assessment

Unclear until maintainers approve the public config contract. The requested direction is plausible, but the maintainable path should reconcile model, delivery, contextTokens, and the narrower delivery-only tracker in one design.
