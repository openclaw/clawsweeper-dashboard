## Summary

Keep open: the reported WhatsApp filename fallback bug is still present in the active plugin code, but this branch edits old `src/web` files that current main no longer uses, so it would not fix runtime behavior.

## What This Changes

The PR branch appends MIME-derived extensions to WhatsApp document fallback filenames in old `src/web` send and auto-reply paths and updates the old send API tests.

## Best Possible Solution

Port the filename resolver to the active WhatsApp plugin send API and outbound media normalization, preserve explicit filenames with extensions, add focused plugin tests, and attach redacted real WhatsApp delivery proof.

## Reproduction Assessment

Yes, by source inspection rather than a live run: current main still sets the active WhatsApp document fallback filename to `"file"` for document sends without `sendOptions.fileName`, and the active test asserts that value.

## Solution Assessment

No. The MIME-extension fallback is narrow and reasonable, but this diff modifies old `src/web` files; the maintainable fix belongs in `extensions/whatsapp` with plugin tests and real delivery proof.
