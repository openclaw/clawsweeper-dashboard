## Summary

Keep open: the reported WhatsApp filename fallback bug is still valid, but this PR edits the old `src/web` WhatsApp paths while current main runs the plugin implementation under `extensions/whatsapp`. The branch also lacks after-fix real WhatsApp behavior proof, so it is not ready to merge as-is.

## What This Changes

The PR appends MIME-derived extensions to WhatsApp document fallback filenames in old `src/web` send and auto-reply code and updates the old send API tests.

## Best Possible Solution

Port the MIME-extension filename resolver into `extensions/whatsapp`, apply it to the active send API and outbound media normalization paths, preserve explicit filenames that already have extensions, update focused tests, and add the release changelog entry during landing.

## Solution Assessment
