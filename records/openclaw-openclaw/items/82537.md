## Summary

Keep open. Current main has session-key-scoped history reload and live transcript event plumbing, but it does not provide the requested opt-in token/operator-linked cross-device WebChat view; this remains a product and privacy feature request rather than a closeable stale, duplicate, or implemented item.

## What This Changes

The issue requests an opt-in mode where multiple paired browsers sharing a gateway token or operator context see unified WebChat history and real-time bidirectional updates.

## Best Possible Solution

Define a gateway-owned WebChat session sharing model that explicitly links devices or operator identities to a chosen session, reuses the existing session.message/history plumbing, and preserves secure per-device auth boundaries.

## Reproduction Assessment

Not applicable as a strict bug reproduction: the requested behavior is a new opt-in sharing mode. Source inspection confirms current main is scoped to the active session key and browser-local session selection, with same-session event plumbing already present.

## Solution Assessment

Unclear until maintainers choose the product and privacy model. The narrow maintainable direction is not a sidecar or DOM injection workaround, but a first-class gateway/session contract for shared WebChat views.
