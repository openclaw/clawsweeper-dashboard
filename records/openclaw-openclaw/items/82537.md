## Summary

Keep open. This is a new opt-in WebChat/Gateway session-linking feature, not a cleanup-closeable bug: current main already supports history and live refresh for an explicit `sessionKey`, but it does not define token/operator-based cross-device session discovery or merging.

## What This Changes

Request an opt-in mode where WebChat browsers sharing a gateway token or operator can see unified history and live updates for the same agent conversation.

## Best Possible Solution

Define an explicit shared-session or device-linking design for WebChat that keeps `sessionKey` as the canonical routing target and adds cross-device discovery/sync only after opt-in, visibility, and trust rules are agreed.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this requests new automatic/shared WebChat session selection across devices. Source inspection shows current same-session history and live events are keyed by `sessionKey`, with browser-local session selection.

## Solution Assessment

Unclear. Token/operator-based automatic merging may not be the safest default; the maintainable path is an explicit opt-in shared-session or device-linking design with clear session selection and visibility rules.
