## Summary

Keep open. Current main still treats WebChat/Control UI continuity as explicit session-key routing plus browser-local device/session state, while this issue asks for new opt-in cross-device shared-session behavior that needs product, UX, and trust-model design.

## What This Changes

Request an opt-in WebChat mode where multiple paired browsers sharing the same gateway/operator context see unified history and real-time updates for the same agent session.

## Best Possible Solution

Define a first-class shared WebChat session or device-linking contract that preserves explicit session identity while adding opt-in discovery, history hydration, live updates, UI visibility, and revocation/trust rules.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a feature request for automatic or opt-in shared WebChat session selection across devices. Source inspection confirms current main is session-keyed with browser-local session and device state, but no failing behavior against an existing contract was established.

## Solution Assessment

Unclear. The requested token/operator-based sharing may be useful, but the safer maintainable path is an explicit shared-session or device-linking design rather than automatic merging of browser identities.
