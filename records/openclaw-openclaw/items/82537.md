## Summary

Keep open. Current main and the latest v2026.6.8 release still expose WebChat through browser-local device identity plus selected session-key history and live subscriptions; the requested opt-in token/operator-linked cross-device session mode remains a product and security design item.

## What This Changes

The issue requests an opt-in WebChat mode where multiple devices linked by the same gateway token or operator share message history and real-time updates.

## Best Possible Solution

Define an explicit WebChat shared-session/device-linking contract with opt-in UX, transcript visibility rules, auth scope requirements, and session-key subscription semantics before implementation.

## Reproduction Assessment

No high-confidence live two-device reproduction was run. Source inspection does verify the current browser-local device identity plus selected-session routing, which is enough to confirm the requested opt-in shared-session mode is still missing.

## Solution Assessment

Unclear as proposed. Token/operator-based automatic merging is not yet the best-defined solution; the safer path is a maintainer-approved shared-session or device-linking design with explicit UI, visibility, and trust boundaries.
