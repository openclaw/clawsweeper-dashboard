## Summary

Keep open. Current main still routes WebChat history and live refresh through explicit sessionKey selection, while this issue asks for a new opt-in cross-device shared-session mode that needs product, UX, and trust-boundary decisions before implementation.

## What This Changes

Add opt-in cross-device WebChat session synchronization so browsers sharing a gateway token or operator can see unified history and live updates.

## Best Possible Solution

Define an explicit shared-session or device-linking contract for WebChat that keeps sessionKey as the canonical route and adds cross-device discovery only after opt-in, visibility, and authorization rules are agreed.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this asks for a new opt-in cross-device WebChat sync mode. Source inspection confirms current main is explicit sessionKey-based, but no existing contract promises token/operator-wide device merging.

## Solution Assessment

Unclear. Token/operator-based automatic merging is not yet the best defined fix; the maintainable path is an explicit shared-session or device-linking design with clear visibility, authorization, and UX semantics.
