## Summary

Keep open. Current main and v2026.6.6 still lack a supported skill priority, pinned-skill, or ordering contract; related open work overlaps but does not supersede the requested config and CLI surface, so this remains a maintainer product/API decision.

## What This Changes

Add user-configurable skill priority and fallback ordering so overlapping skills can be selected predictably via config and CLI.

## Best Possible Solution

Define one first-class skills ordering contract in the canonical config/schema/docs, apply it after eligibility and agent filters but before prompt limits, preserve deterministic fallback ordering, and validate it through snapshots, CLI inspection, docs, and truncation behavior.

## Reproduction Assessment

Yes, as a source-level feature gap: current main has alphabetical prompt ordering, strict skill config without priority/pinned fields, and no priority CLI subcommand. This is not a failing bug reproduction because the issue asks for new supported behavior.

## Solution Assessment

Unclear. User priority may be valuable, but the proposed sidecar priority.json is not clearly the best OpenClaw config/API shape; maintainers should choose between config ordering, router/plugin APIs, and truncation/pinning semantics before implementation.
