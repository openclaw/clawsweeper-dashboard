## Summary

Keep open. Current main still provides deterministic source precedence, eligibility filtering, alphabetical prompt ordering, prompt limits, and existing allowlist/always-include controls, but it does not provide a user-configurable skill priority or ordering contract; the closed unmerged priority PR and related open truncation/preferences items do not supersede this request.

## What This Changes

Request a user-configurable skill priority system and CLI so overlapping skills can be ordered or preferred consistently.

## Best Possible Solution

Define one first-class skills ordering contract in the existing config schema/docs, apply it after eligibility and per-agent filters but before prompt limits, and preserve deterministic fallback ordering for prompt-cache stability.

## Reproduction Assessment

Not applicable as a feature request. The absence is source-verifiable on current main through the loader, prompt builder, strict config schema, CLI registration, docs, and alphabetical ordering test.

## Solution Assessment

Unclear until maintainer product/API review. The proposed separate priority file is directionally useful, but the safer maintainable path likely belongs in the existing skills config contract so it composes with allowlists, snapshots, truncation, and per-agent filtering.
