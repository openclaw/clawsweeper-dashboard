## Summary

Keep open. Current main and v2026.6.8 still use deterministic alphabetical skill ordering with existing filters and prompt limits, but no supported skill priority, pinning config, or `openclaw skills priority` CLI surface; related work overlaps but does not supersede this product/API decision.

## What This Changes

Request to add user-configurable skill priority or ordering, including priority config and CLI management, so overlapping skills are selected predictably.

## Best Possible Solution

Define one first-class skills ordering contract in the existing config/schema/docs, apply it after eligibility and per-agent filters but before prompt limits, keep non-priority ordering deterministic, and cover snapshots, CLI inspection, docs, and truncation behavior together.

## Reproduction Assessment

Yes, as a source-level feature gap: current main has alphabetical prompt ordering, strict skill config without priority or pinned fields, and no priority CLI subcommand. This is not a failing bug reproduction because the issue asks for new supported behavior.

## Solution Assessment

Unclear. User priority is a plausible capability, but the proposed sidecar `priority.json` is not clearly the best OpenClaw API shape compared with a canonical config/schema/docs contract plus explicit truncation and snapshot semantics.
