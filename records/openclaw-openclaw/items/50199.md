The issue requests user-configurable skill priority so overlapping skills can be selected predictably.

Define one first-class skills ordering contract in the existing config/schema/docs, apply it after eligibility and per-agent filters but before prompt limits, and keep snapshots, CLI inspection, docs, and non-priority ordering deterministic.
