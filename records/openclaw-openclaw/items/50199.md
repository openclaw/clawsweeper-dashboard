## Summary

Keep open. Current main still has deterministic source precedence, eligibility filtering, stable alphabetical prompt ordering, and truncation handling, but it does not expose a user-configurable skill priority or ordering contract for choosing between overlapping eligible skills.

## What This Changes

The issue requests a configurable skill priority system, including config storage, priority-aware skill discovery and selection, and `openclaw skills priority` CLI commands.

## Best Possible Solution

Define one first-class skill ordering contract in the existing skills config/API surface, apply it after eligibility and per-agent filters but before prompt count/character limits, preserve deterministic fallback ordering for prompt-cache stability, and expose the effective order through CLI inspection, snapshots/runtime materialization, docs, and tests.

## Reproduction Assessment

Not applicable as a feature request. The absence is source-verifiable on current main through the loader, prompt builder, strict config schema, CLI registration, docs, and alphabetical ordering test.

## Solution Assessment

Unclear until maintainer product/API review. The proposed separate `~/.openclaw/skills-config/priority.json` and CLI are directionally useful, but a safer maintainable path likely belongs in the existing skills config contract so it composes with entries, allowlists, snapshots, truncation, and per-agent filtering.
