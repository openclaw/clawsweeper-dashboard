## Summary

Keep open. Current main and the latest release still have deterministic source precedence plus alphabetical prompt ordering, strict skills config fields, and no `openclaw skills priority` surface; the related truncation PR and closed unmerged priority PR do not supersede this broader ordering-contract request.

## What This Changes

The issue requests user-configurable skill priority so overlapping eligible skills can be ordered deterministically by user preference.

## Best Possible Solution

Define one first-class skills ordering contract in the existing config schema/docs, apply it after eligibility and per-agent filters but before prompt limits, and keep fallback ordering deterministic for prompt-cache stability.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The gap is source-verifiable because current main and v2026.5.22 have alphabetical ordering, strict skill config without priority, and no `openclaw skills priority` command.

## Solution Assessment

Unclear until maintainer product/API review. User-controlled ordering is useful, but the proposed separate priority file is probably less maintainable than extending the existing skills config contract so it composes with allowlists, prompt limits, and snapshots.
