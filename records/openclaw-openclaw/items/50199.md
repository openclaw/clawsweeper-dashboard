## Summary

Keep open. Current main still lacks a user-configurable skill priority or ordering contract; it has deterministic source precedence, eligibility filtering, alphabetical prompt ordering, prompt limits, and truncation warnings, but none of those satisfy the requested ability to prefer one eligible skill over another.

## What This Changes

Request a configurable priority system and CLI surface so users can prefer one eligible skill over another when skills overlap.

## Best Possible Solution

Define one first-class skills ordering contract in the existing config schema and docs, apply it after eligibility and per-agent filters but before prompt count/character limits, preserve deterministic fallback ordering, and expose the effective order through snapshots, runtime materialization, CLI inspection, and user-facing docs.

## Reproduction Assessment

Not applicable as a feature request. The absence is source-verifiable on current main through the skills loader, prompt builder, strict config schema, CLI registration, docs, and alphabetical ordering test.

## Solution Assessment

Unclear until maintainer product/API review. The proposed separate priority file and CLI are directionally useful, but the maintainable solution likely belongs in the existing skills config/API contract rather than a parallel priority store that bypasses entries, allowlists, and snapshot semantics.
