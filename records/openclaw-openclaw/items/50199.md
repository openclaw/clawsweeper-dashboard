## Summary

Keep open. Current main and the latest release still lack a user-configurable skill priority or ordering contract, and the prior priority implementation was closed unmerged, so this remains a product/API decision rather than cleanup-close material.

## What This Changes

Request to add user-configurable skill priority so overlapping skills can be ordered or preferred predictably.

## Best Possible Solution

Define one first-class skills ordering contract in the canonical config/schema/docs, apply it after eligibility and allowlists but before prompt limits, keep non-priority ordering deterministic, and wire the same semantics through snapshots, CLI inspection, tests, and truncation recovery.

## Reproduction Assessment

Yes for a source-level feature gap: current main has alphabetical skill prompt ordering, allowlists, and prompt limits, but no priority/pinned setting or priority CLI. This is not a strict bug reproduction because the issue asks for a new supported behavior.

## Solution Assessment

Unclear. User priority may be useful, but the proposed sidecar config file is not clearly the best fit for OpenClaw's canonical config model; maintainers should choose the durable config/API shape before implementation.
