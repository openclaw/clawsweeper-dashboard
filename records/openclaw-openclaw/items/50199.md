## Summary

Keep open. Current main still exposes deterministic skill source precedence, eligibility filtering, alphabetical prompt ordering, and prompt limits, but it does not provide a user-configurable skill priority or ordering contract; the related open truncation and preferences items overlap without superseding this narrower request.

## What This Changes

The issue requests a user-configurable priority/ordering system and CLI commands for choosing among overlapping eligible skills.

## Best Possible Solution

Define one first-class skills ordering contract in the existing config/docs surface, apply it after eligibility and per-agent filters but before prompt limits, and keep fallback ordering deterministic for prompt-cache stability.

## Reproduction Assessment

Not applicable as a feature request. The absence is source-verifiable on current main through the loader, prompt builder, strict config schema, CLI registration, docs, and alphabetical ordering test.

## Solution Assessment

Unclear until maintainer product/API review. The proposed separate priority file is directionally useful, but the safer maintainable path likely belongs in the existing skills config contract so it composes with allowlists, snapshots, truncation, and per-agent filtering.
