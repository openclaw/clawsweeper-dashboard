## Summary

Keep open. Current main makes skill loading and prompt exposure deterministic, but it still has no user-configurable skill priority or ordering contract for choosing among overlapping eligible skills.

## What This Changes

Add user-configurable skill priority/ordering, plus CLI support, so overlapping eligible skills can be preferred or treated as fallbacks.

## Best Possible Solution

Define one first-class skills ordering contract in the existing config/API surface, apply it after eligibility and per-agent filters but before prompt limits, preserve deterministic fallback ordering, and expose the effective order through CLI, docs, snapshots, and tests.

## Reproduction Assessment

Not applicable as a feature request. The absence is source-verifiable on current main through the loader, prompt builder, strict config schema, CLI registration, docs, and alphabetical-order regression test.

## Solution Assessment

Unclear until maintainer product/API review. The requested priority mechanism is valid, but the proposed separate priority.json may be less maintainable than extending the existing skills config contract so it composes with allowlists, snapshots, and truncation behavior.
