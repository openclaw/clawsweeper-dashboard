## Summary

Keep open. Current main still has deterministic source precedence and alphabetical prompt ordering, but it does not provide a user-configurable skill priority, pinning, or ordering contract, and the related truncation issue plus closed priority PRs do not supersede the broader feature request.

## What This Changes

The issue requests configurable skill priority or ordering so overlapping skills can be selected or surfaced predictably.

## Best Possible Solution

Define one first-class skills ordering contract in the existing config schema and public docs, apply it after eligibility and per-agent filters but before prompt count/character limits, preserve deterministic fallback ordering, and keep snapshots, runtime materialization, CLI inspection, and docs aligned.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request; the absence is source-verifiable on current main through the loader, prompt builder, strict config schema, CLI registration, docs, and alphabetical-order test.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainer product review; the requested direction is valid, but a separate priority file and CLI may not be the best long-term contract compared with extending the existing `skills` config, allowlists, and snapshot semantics.
