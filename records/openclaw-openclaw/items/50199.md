## Summary

Keep open. Current main and v2026.6.10 still provide deterministic skill precedence, alphabetical prompt ordering, prompt limits, and per-skill env/config entries, but no supported user-configurable skill priority or `openclaw skills priority` CLI surface; the related truncation and preference items overlap but do not supersede this product/API decision.

## What This Changes

The issue requests user-configurable priority ordering for skill selection when multiple installed skills overlap.

## Best Possible Solution

Define one first-class skills ordering contract in the existing config/schema/docs, apply it after eligibility and per-agent filters but before prompt limits, preserve deterministic default ordering, and cover snapshots, CLI inspection, docs, and truncation behavior together.

## Reproduction Assessment

Yes, as a source-level feature gap: current main and v2026.6.10 have alphabetical prompt ordering, strict skill config without priority or pinned fields, and no priority CLI subcommand. This is not a failing bug reproduction because the issue asks for new supported behavior.

## Solution Assessment

Unclear as proposed. User priority is a plausible capability, but the sidecar `skills-config/priority.json` design is not clearly the best OpenClaw API shape compared with a canonical config/schema/docs contract plus explicit truncation and snapshot semantics.
