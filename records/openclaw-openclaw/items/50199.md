## Summary

Keep open. Current main and the latest release still use fixed source precedence plus alphabetical prompt ordering, strict skill config fields, and the existing `openclaw skills` command set; related preference/truncation issues and the closed unmerged priority PR do not supersede this ordering-contract request.

## What This Changes

The issue requests user-configurable skill priority or ordering so overlapping eligible skills can be selected predictably.

## Best Possible Solution

Define one first-class skills ordering contract in the existing config schema and docs, apply it after eligibility and per-agent filters but before prompt limits, and keep fallback ordering deterministic.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The gap is source-verifiable because current main and v2026.5.20 have alphabetical ordering, strict skill config without priority, and no `openclaw skills priority` command.

## Solution Assessment

Unclear until maintainer product/API review. User-controlled ordering is useful, but the proposed separate `~/.openclaw/skills-config/priority.json` is probably less maintainable than extending the existing skills config contract so it composes with allowlists, prompt limits, and snapshots.
