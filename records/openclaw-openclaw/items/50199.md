## Summary

Keep open. Current main and latest release still have deterministic source precedence, alphabetical prompt ordering, prompt limits, per-skill enable/env/config entries, and no supported user-configurable skill priority contract or `openclaw skills priority` CLI surface; overlapping issues and the closed unmerged priority PR do not supersede the product/API decision.

## What This Changes

Request to add user-configurable skill priority ordering and CLI management so overlapping skills can be preferred or treated as fallbacks.

## Best Possible Solution

Define one maintainer-approved skills ordering contract in the canonical skills config/schema/docs, apply it after eligibility and per-agent filters but before prompt limits, and keep snapshots, CLI inspection, docs, and truncation behavior aligned.

## Reproduction Assessment

Yes, as a source-level feature gap: current main and v2026.6.10 have alphabetical prompt ordering, strict skill config without priority or pinned fields, and no priority CLI subcommand. This is not a failing bug reproduction because the issue asks for new supported behavior.

## Solution Assessment

Unclear as proposed. User-configurable priority is plausible, but the sidecar `skills-config/priority.json` design is not clearly the best OpenClaw API shape compared with a canonical config/schema/docs contract plus explicit snapshot and truncation semantics.
