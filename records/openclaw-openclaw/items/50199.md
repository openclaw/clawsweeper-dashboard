## Summary

Keep open. Current main still lacks a user-configurable skill priority or ordering contract; the existing behavior is deterministic source precedence, eligibility filtering, agent allowlists, alphabetical prompt ordering, and prompt truncation limits, which do not let a user prefer one eligible skill over another.

## What This Changes

Request to add user-configurable skill priority/ordering, including priority config, priority-aware skill discovery/selection, and `openclaw skills priority` CLI commands.

## Best Possible Solution

Define one first-class ordering contract in the existing skills config/schema and docs, apply it after eligibility and per-agent filters but before prompt count/character limits, keep deterministic fallback ordering, and expose the effective order through snapshots, runtime materialization, CLI inspection, and related truncation/pinning work.

## Reproduction Assessment

Not applicable as a feature request. The feature gap is source-verifiable on current main through the skills loader, prompt builder, strict config schema, CLI registration, docs, and alphabetical ordering test.

## Solution Assessment

Unclear until maintainer product/API review. The separate `~/.openclaw/skills-config/priority.json` proposal is directionally useful, but the maintainable solution likely belongs in the existing skills config and snapshot contract rather than a parallel priority store.
