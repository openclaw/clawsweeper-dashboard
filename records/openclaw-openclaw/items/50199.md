## Summary

Keep open. Current main still has deterministic source precedence, eligibility filtering, alphabetical prompt ordering, prompt limits, and skill enable/config controls, but no user-configurable skill priority or ordering contract; the related preferences and truncation items are adjacent and do not supersede this request.

## What This Changes

The issue requests a user-configurable skill priority system and CLI surface so overlapping skills can be ordered deterministically.

## Best Possible Solution

Define one first-class skills ordering contract in the existing config/schema/docs, apply it after eligibility and per-agent filtering but before prompt limits, and keep deterministic fallback ordering for cache stability.

## Reproduction Assessment

Not applicable as a feature request. The gap is source-verifiable on current main through the skills workspace ordering, strict config type/schema, CLI registration, docs, and tests that still assert alphabetical ordering.

## Solution Assessment

Unclear until maintainer product/API review. The proposed separate priority file is directionally useful, but the safer maintainable path likely belongs in the existing skills config contract so it composes with allowlists, prompt snapshots, truncation, and per-agent filtering.
