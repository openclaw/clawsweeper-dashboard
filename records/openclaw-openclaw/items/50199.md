## Summary

Keep open. Current main and latest release still expose deterministic source precedence, eligibility filtering, alphabetical prompt ordering, prompt limits, and truncation warnings, but no user-configurable skill priority/ranking contract or `openclaw skills priority` CLI surface; the related preference/truncation items do not supersede this broader product/API decision.

## What This Changes

The issue requests configurable skill priority or ordering so users can prefer one eligible skill over another when multiple installed skills can handle a task.

## Best Possible Solution

Define one canonical skills ordering contract, apply it after eligibility and per-agent filters but before prompt limits, keep deterministic fallback ordering, and align schema, CLI inspection, docs, snapshots, and truncation behavior if maintainers approve it.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a feature request for a new priority/ordering contract. Source inspection confirms current behavior is deterministic alphabetical ordering plus filters, not user-controlled priority selection.

## Solution Assessment

Unclear. A priority mechanism may be useful, but the proposed sidecar file and CLI surface are not clearly the best fit for OpenClaw's canonical config and high config-surface bar.
