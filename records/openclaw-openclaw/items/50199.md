## Summary

Keep open. Current main still has deterministic source precedence, eligibility filtering, alphabetical prompt ordering, prompt limits, and existing allowlist/entry controls, but no user-configurable skill priority or `openclaw skills priority` surface; the closed priority PR and adjacent open truncation/preferences issues do not supersede this request.

## What This Changes

Request to add user-configurable skill priority and CLI/config support so overlapping skills can be ordered or preferred.

## Best Possible Solution

Define one first-class skills ordering contract in the existing config schema/docs, apply it after eligibility and per-agent filters but before prompt limits, preserve deterministic fallback ordering, and explicitly invalidate snapshots when ordering-relevant config changes.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The absence is source-verifiable on current main through the loader, prompt builder, strict config schema, CLI registration, docs, and alphabetical ordering test.

## Solution Assessment

Unclear until maintainer product/API review. The proposed separate priority file is directionally useful, but the maintainable path likely belongs in the existing skills config contract so it composes with allowlists, snapshots, truncation, and per-agent filtering.
