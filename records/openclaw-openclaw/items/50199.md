## Summary

Keep open. Current main still has deterministic source precedence, eligibility filtering, alphabetical prompt ordering, and prompt truncation, but no user-configurable priority or ordering contract for overlapping eligible skills; the related open items overlap without fully superseding this request.

## What This Changes

Add a user-configurable skill priority or ordering mechanism so overlapping eligible skills can be selected predictably.

## Best Possible Solution

Define one first-class skills ordering contract in the existing config/docs surface, apply it after eligibility and per-agent filters but before prompt limits, and keep non-priority fallback ordering deterministic for prompt-cache stability.

## Reproduction Assessment

Not applicable as a feature request. The absence is source-verifiable on current main through the loader, prompt builder, strict config schema, CLI registration, docs, and alphabetical ordering test.

## Solution Assessment

Unclear until maintainer product/API review. The proposed separate `~/.openclaw/skills-config/priority.json` is directionally useful, but the safer maintainable path likely belongs in the existing skills config contract so it composes with allowlists, snapshots, truncation, and per-agent filtering.
