Keep open. The issue requests configurable priority/ranking for overlapping skills, including a new priority config surface and CLI commands.

## Best Possible Solution

Define one maintainer-approved skills ordering contract, preferably within the existing `skills.entries` and schema/docs surface, then apply it after eligibility/filtering and before prompt limits while preserving deterministic fallback order and snapshot invalidation semantics.
