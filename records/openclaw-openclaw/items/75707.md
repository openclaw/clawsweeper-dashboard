## Summary

Keep open. Current main has made the bundled runtime-dependency staging subclaim obsolete and added useful diagnostics/partial mitigations, but source and discussion still leave the broader Gateway idle CPU and per-message prep-latency problem unresolved.

## What This Changes

The issue reports Gateway idle CPU saturation and long per-message prep latency across session replay, compaction, plugin/model discovery, heartbeat, channel/plugin disabling, and provider fallback paths.

## Best Possible Solution

Keep this as the active Gateway performance tracker, preserve the linked profile evidence, and split confirmed current-main causes into focused fixes for heartbeat cooldown, model/plugin manifest scanning, prep-stage caching/prewarming, session/compaction cleanup, and channel/plugin disable invariants.

## Solution Assessment
