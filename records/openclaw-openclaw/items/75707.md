## Summary

Keep open. This is a fresh, multi-reporter Gateway CPU saturation report with concrete reproduction context, and current main only shows partial mitigations across the named hot paths rather than a proven end-to-end idle CPU fix. #75688 is closely related but not an exact duplicate because this report adds Linux/WhatsApp/Groq/session-compaction/plugin-disable claims that should remain visible for maintainer triage.

## What This Changes

The issue requests fixes for Gateway idle CPU saturation involving accumulated sessions, empty-session compaction behavior, model fallback tokenization, disabled Discord/plugin loading, runtime dependency staging, and session cleanup on v2026.4.29.

## Best Possible Solution

Keep this open as a Gateway performance/root-cause tracking item linked with #75688, then split validated causes into focused fixes: session restart/cleanup policy, empty-compaction behavior, fallback token accounting, runtime-deps hot-path profiling, and disabled channel/plugin invariants.

No. The issue gives credible steps and diagnostics, but a high-confidence reproduction still needs either a live Gateway profile with the same WhatsApp/Groq/fallback setup or a synthetic fixture that recreates accumulated sessions plus the compaction/fallback conditions on current main.
