## Summary

Keep open. Current main has several partial mitigations for the original subclaims, including disabled plugin/channel runtime-dependency handling, bounded Discord command deploy retries, empty-conversation compaction skips, and bounded session listing. It still builds embedded-run tools, resolves plugin tool discovery, builds the system prompt, and sets up streaming inside each embedded run, which matches the reporter's later per-message prep-stage trace and means the Gateway CPU/per-message latency problem is not proven fixed.

## What This Changes

The issue requests investigation and fixes for Gateway idle and per-message CPU saturation across session restart recovery, compaction, plugin/tool setup, disabled channel/plugin behavior, fallback tokenization, prompt building, and stream/model setup overhead.

## Best Possible Solution

Use this issue with #75688 as a Gateway performance cluster, pick a canonical tracker, collect a current-main prep-stage/profile trace, and split confirmed causes into targeted PRs for embedded-run tool setup caching, plugin tool discovery reuse, prompt/bootstrap cost reduction, stream/model-resolution reuse, and session startup cleanup invariants.
