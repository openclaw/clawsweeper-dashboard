## Summary

Keep open. Current main still keys Feishu receive, debounce, and final persistent dedupe on bare `message_id`, and this PR targets the right Feishu-owned bug, but the head cannot merge yet because it imports a private helper and still needs the required changelog entry.

## What This Changes

The PR changes Feishu inbound receive, debounce, and final dedupe to include media `file_key`/`image_key` values in dedupe keys and adds an audio regression test.

## Best Possible Solution

Keep the fix Feishu-owned with one shared media-aware dedupe-key helper used consistently by receive, debounce, suppression recording, persistent lookup, and final processing, while preserving bare `message_id` behavior for non-media and true retries.

## Reproduction Assessment

Yes. Current main has a high-confidence trace reproduction: two Feishu audio events for the same account with the same `message_id` and different `file_key` values collide in both the early processing claim and final persistent dedupe paths.

## Solution Assessment

No, not as currently patched. The implementation direction is narrow and appropriate, but the branch needs an exported or extracted Feishu-local helper plus the changelog entry before it is a maintainable merge candidate.
