## Summary

Keep open. The Feishu bug is valid and the implementation direction is narrow, but this PR head still imports a non-exported helper and is missing the required user-facing changelog entry, so it should be repaired before automerge.

## What This Changes

The PR changes Feishu inbound receive, debounce, suppression, and final dedupe to include media file/image keys for media messages and adds an audio regression test.

## Best Possible Solution

Keep the fix Feishu-owned with one shared media-aware dedupe-key helper used by receive, debounce, suppression recording, persistent lookup, and final processing, while preserving bare `message_id` behavior for non-media and true retries.

## Reproduction Assessment

Yes. Current main has a high-confidence trace reproduction: same account, same Feishu `message_id`, and different audio `file_key` values collide in both the early claim path and final persistent dedupe path.

## Solution Assessment

No, not as currently patched. The media-aware key is the right narrow direction, but the branch needs an exported or extracted helper plus a changelog entry before it is a maintainable merge candidate.
