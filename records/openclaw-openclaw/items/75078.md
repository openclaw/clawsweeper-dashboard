## Summary

Keep this PR open. The Feishu bug and fix direction are valid, but the branch currently imports a helper that current main does not export, so the patch would fail at build/typecheck before the regression fix can ship; it also still needs the required user-facing changelog entry.

## What This Changes

The PR changes Feishu inbound receive, debounce, and final dedupe to include media `file_key`/`image_key` values in the dedupe key for media messages and adds an audio regression test.

## Best Possible Solution

Keep the fix Feishu-owned, but repair the branch by using one exported or dedicated Feishu-local dedupe-key helper shared by receive/debounce and final processing, preserve bare `message_id` behavior for non-media and true retries, add the active-version changelog entry, then run the focused Feishu validation.

## Reproduction Assessment

Yes. Current main has a high-confidence trace reproduction: two Feishu audio events for the same account with the same `message_id` and different `file_key` values collide in both the early processing claim and final persistent dedupe paths.

## Solution Assessment

No, not as patched. The implementation direction is narrow and appropriate, but the unexported `parseMediaKeys` import blocks the build; exporting/extracting a shared helper plus adding the changelog entry is the safer mergeable solution.
