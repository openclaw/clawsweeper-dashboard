## Summary

Keep open. Current main still has the Feishu audio/media dedupe bug, and this PR targets the right Feishu-owned path, but the branch as shown is not mergeable because it imports a non-exported helper and still lacks the required changelog entry.

## What This Changes

The PR changes Feishu receive/debounce/final dedupe to include media `file_key` or `image_key` values for media messages and adds an audio regression test.

## Best Possible Solution

Keep the fix in the Feishu plugin, extract or export one shared media-aware dedupe-key helper used by receive/debounce and final processing, preserve bare `message_id` behavior for non-media and true retries, then add the active-version changelog entry and focused Feishu regression coverage.

## Reproduction Assessment

Yes. A source trace against current main reproduces the bug: two Feishu audio events for the same account with the same `message_id` and different `file_key` values collide in both the early processing claim and final persistent dedupe paths.

## Solution Assessment

No, not as currently patched. The direction is narrow and appropriate, but the branch must first use an exported/shared helper and add the required changelog entry before it is the best mergeable solution.
