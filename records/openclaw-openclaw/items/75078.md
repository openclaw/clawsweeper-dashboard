## Summary

Keep open. The Feishu media-aware dedupe direction matches a real current-main bug, but the PR is not merge-ready because it imports a private helper that current `bot-content.ts` does not export and it still lacks the required user-facing changelog entry.

## What This Changes

The PR changes Feishu inbound debounce and final persistent dedupe to include media `file_key`/`image_key` values and adds an audio reused-`message_id` regression test.

## Best Possible Solution

Centralize a Feishu-local media-aware dedupe-key helper and use it consistently across receive, debounce, suppression recording, persistent finalization, and regression coverage, with a single-line changelog entry.

## Reproduction Assessment

Yes. Current-main source inspection shows the early claim, debounce, suppression, and finalization paths key on bare `message_id`, so same-account Feishu audio events with the same `message_id` and different `file_key` values collide.

## Solution Assessment

No, not as currently patched. The media-aware key is the right narrow fix, but the branch needs a shared/exported helper and the required changelog entry before exact-head validation can make it mergeable.
