## Summary

Keep this PR open: the safe-restart direction is coherent, but the new preflight result underreports active task blockers when more than eight tasks are present.

## What This Changes

The PR adds a safe Gateway restart coordinator, restart preflight/request RPC handlers, `--safe` restart CLI handling for gateway/daemon commands, docs, changelog, and regression tests.

## Best Possible Solution

Keep the safe restart design, but fix the capped task-blocker reporting before merge and let maintainer review plus CI validate the new CLI/RPC surface.

## Reproduction Assessment

Yes. Source inspection gives a focused reproduction: call `createSafeGatewayRestartPreflight` with `activeTasks` equal to 10 and 10 task blockers, and the PR returns only eight task blockers with no omitted-count blocker.

## Solution Assessment

No, not yet. The overall approach is maintainable, but the task-blocker cap needs to report the omitted blockers accurately before this is the best implementation.

## Review Findings

Overall correctness: patch is incorrect
