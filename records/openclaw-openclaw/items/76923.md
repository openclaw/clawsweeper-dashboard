## Summary

Keep this PR open: the active-work restart coordinator direction is useful, but the new `--safe` path bypasses an existing restart safety guard and the user-facing change is missing the required changelog entry.

## What This Changes

The PR adds a safe Gateway restart coordinator, `gateway.restart.preflight`/`gateway.restart.request` RPCs, `gateway restart --safe` and `daemon restart --safe` CLI wiring, tests, and docs.

## Best Possible Solution

Preserve the explicit safe restart mode, but make it keep the existing restart config/future-version preflight behavior, cover that regression, and add the active changelog entry before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: `runDaemonRestart({ safe: true })` returns through the new RPC helper before `runServiceRestart` can execute the existing config preflight. I did not execute it because this review must keep the checkout read-only and `pnpm` is unavailable.

## Solution Assessment

No. The active-work coordinator is a reasonable direction, but the safe CLI path should reuse or expose the existing restart preflight guard instead of bypassing it.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Preserve restart config preflight for safe restarts:** `src/cli/daemon-cli/lifecycle.ts:264-266`
