## Summary

Keep open: the PR is useful but not merge-ready because the safe restart preflight and the actual scheduler deferral still disagree on queued task records, and the user-facing CLI/RPC change is missing the required changelog entry.

## What This Changes

The PR adds safe Gateway restart RPCs, `--safe` gateway/daemon restart CLI wiring, a restart coordinator, narrower task restart blockers, docs, and tests.

## Best Possible Solution

Make safe preflight, scheduler deferral, and shutdown/reload drain share one blocker contract, cover queued and stale-ended task records with regression tests, and add the required changelog entry before merge.

## Reproduction Assessment

Yes, by source inspection. A queued task-registry record with no command queue, pending replies, embedded runs, or running task blockers can make the new preflight report safe while the scheduled restart still waits on `getInspectableTaskRegistrySummary().active`.

## Solution Assessment

No. The coordinator shape is reasonable, but the patch needs to align the actual scheduled deferral with the new safe preflight semantics and include the required changelog entry.
