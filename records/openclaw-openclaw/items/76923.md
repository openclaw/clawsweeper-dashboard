## Summary

This PR should stay open for normal review because current main does not have the proposed safe restart RPC/CLI path, and the patch appears to have a source-level mismatch between the coordinator’s preflight result and the restart deferral code that actually emits the restart.

## What This Changes

The PR adds an explicit safe Gateway restart path with new `gateway.restart.preflight`/`gateway.restart.request` RPC methods, `openclaw gateway restart --safe`/`openclaw daemon restart --safe`, docs, and focused tests.

## Best Possible Solution

Land the explicit safe restart path only after the preflight counts, scheduled deferral predicate, docs, and changelog describe the same restart blockers.

## Reproduction Assessment

Yes for the PR regression: source inspection shows the coordinator bases its returned status on one blocker predicate while the scheduled restart still consults the existing global deferral predicate. I did not execute tests under the read-only review constraint.

## Solution Assessment

No. The direction is plausible, but the patch should first align the actual restart deferral check with the safe coordinator’s preflight semantics and add the required changelog entry.

- **[P2] Align safe preflight with the real restart deferral:** `src/infra/restart-coordinator.ts:157-163`
  - body: The safe coordinator now reports `scheduled` from `preflight.safe`, but the scheduled restart still goes through `scheduleGatewaySigusr1Restart`, which later runs the global `preRestartCheck`.
