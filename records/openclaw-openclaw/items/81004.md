## Summary

Keep open: current main still lacks content-safe output digest fields in the relevant CLI backend logs, and https://github.com/openclaw/openclaw/pull/81021 is an open implementation PR with closing syntax for this issue.

## What This Changes

The issue requests per-turn CLI backend log diagnostics with output byte counts and short content hashes for both Claude live-session turns and synthetic `before_agent_reply` turns.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81021 if maintainers accept the diagnostic log shape, then close this issue from the merged implementation.

## Reproduction Assessment

Yes. Source inspection on current main shows the successful Claude live-session turn log omits output byte/hash fields, and handled cron synthetic replies return before any comparable CLI backend turn log.

## Solution Assessment

Yes for the current implementation direction. A shared content-safe digest helper used at both the live-session completion and synthetic reply sites is the narrow maintainable fix, and this issue should remain open while the linked PR is still unmerged.
