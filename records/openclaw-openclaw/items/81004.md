## Summary

Keep open. Current main still lacks the requested `outBytes`/`outHash` diagnostics in the Claude live-session turn log and has no comparable CLI backend turn log for handled synthetic cron replies; the open linked implementation PR uses closing syntax for this issue, so the right path is to review and land or close that PR rather than closing this issue early.

## What This Changes

The issue requests bounded per-turn output byte count and short content hash fields in gateway CLI backend logs for both Claude live-session turns and synthetic reply paths.

## Best Possible Solution

Review and land the linked implementation PR at https://github.com/openclaw/openclaw/pull/81021 if it remains correct, so both real Claude turns and synthetic cron replies emit bounded digest diagnostics without raw assistant text.

## Reproduction Assessment

Yes. Source inspection on current main shows the Claude live-session success log omits `outBytes` and `outHash`, and the handled cron `before_agent_reply` path returns before any comparable CLI backend turn log.

## Solution Assessment

Yes. The narrow maintainable solution is a shared digest formatter used by both completion sites, with tests proving digest fields are present and raw response text is not logged; the open linked PR follows that shape.
