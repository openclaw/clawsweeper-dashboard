## Summary

Keep open. Current main still lacks the requested `outBytes`/`outHash` fields and the synthetic cron reply path still bypasses any comparable CLI backend turn log; the open implementation PR at https://github.com/openclaw/openclaw/pull/81021 uses closing syntax, so this issue should stay open until that PR lands or is explicitly closed.

## What This Changes

Add content-safe per-turn output byte count and short hash logging so repeated identical responses are detectable from `gateway.log` without reading transcripts.

## Best Possible Solution

Review and land the linked implementation PR if it remains correct, covering both live Claude turns and synthetic cron replies with bounded digest diagnostics and no raw assistant text in logs.

## Reproduction Assessment

Yes. Source inspection on current main shows the Claude live-session turn log omits `outBytes` and `outHash`, and the handled cron `before_agent_reply` path returns before any comparable CLI backend turn log.

## Solution Assessment

Yes. The narrow maintainable solution is the linked PR’s shared digest formatter used by both completion sites, with tests proving digest fields are present and raw response text is not logged.
