## Summary

Keep open. Current main still lacks the requested `outBytes`/`outHash` fields on Claude live-session turn logs, and the cron `before_agent_reply` handled path still returns synthetic output before any comparable CLI backend turn log; the open linked implementation at https://github.com/openclaw/openclaw/pull/81021 is the right path to review rather than closing this issue early.

## What This Changes

Request adding bounded response byte-count and short content-hash diagnostics to CLI backend gateway logs for both Claude live-session turns and synthetic cron replies.

## Best Possible Solution

Review, rebase if needed, and land https://github.com/openclaw/openclaw/pull/81021 if still correct, so both real Claude turns and synthetic cron replies emit bounded digest diagnostics without raw assistant text.

## Reproduction Assessment

Yes. Source inspection on current main shows the Claude live-session success log lacks `outBytes` and `outHash`, and the handled cron `before_agent_reply` path returns before any comparable CLI backend turn log.

## Solution Assessment

Yes. The narrow maintainable solution is a shared safe digest formatter used by both completion sites, with focused tests proving digest fields are present and raw assistant text is not logged; the linked open PR follows that shape.
