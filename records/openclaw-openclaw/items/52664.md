## Summary

Keep this PR open: the rawBody hook API is useful and current main does not already provide it, but the branch is still not merge-ready because some active-run queue injections can leave hook events reporting the previous direct user's rawBody.

## What This Changes

The PR adds optional rawBody to before_prompt_build, before_agent_start, and agent_end plugin hook events and threads it through channel, embedded, CLI, Codex, Copilot, queued, collect, and steer paths with docs and tests.

## Best Possible Solution

Land the API only after every active-run queue injection either forwards direct-user rawBody or explicitly clears it for internal/inter-session input, with focused tests and maintainer acceptance of the additive plugin contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: start a direct channel run to set currentRawBody, then inject via /steer, sessions_send, Talk active-run control, or subagent active wake; those callers omit rawBody, so the active run does not refresh or clear it.

## Solution Assessment

No. The API location is plausible and owner-aligned, but the implementation is incomplete until all queueMessage callers participate in the rawBody refresh/clear contract.
