## What This Changes

The branch adds optional `rawBody` to plugin hook event payloads, threads it through reply, queue, and embedded-runner paths, and updates hook docs, tests, SDK baseline hash, and the changelog.

## Best Possible Solution

Land the additive hook API only after the branch makes steered-input semantics unambiguous, rebases cleanly onto current main, and includes redacted real hook payload proof.

## Reproduction Assessment

Yes for the PR review finding: source inspection shows the steer path queues only prompt text while `agent_end` emits `params.rawBody`, and current docs say steered prompts are appended as user messages. I did not run a live channel/plugin reproduction in this read-only review.

## Solution Assessment

No. Exposing `rawBody` is a reasonable additive API direction, but this branch should not merge until steering semantics are fixed or explicitly represented and real hook proof is attached.
