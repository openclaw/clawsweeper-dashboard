## Best Possible Solution

Thread an explicit rawBody update-or-clear signal through the steer queue path, add a regression test for gated steered input after direct-user input, keep docs/types aligned, then have maintainers accept or reject the plugin API surface.

Yes. Source inspection gives a high-confidence reproduction path: send a direct-user steered input that sets `currentRawBody`, then a provenance-gated steered input whose `rawBody` is `undefined`; the active run has no clear signal and keeps the prior value.
