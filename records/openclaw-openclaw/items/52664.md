## Summary

Keep open: the feature is useful and the supplied live proof covers the embedded/Telegram path, but the public hook contract is still inconsistent because Codex and Copilot harness hook emitters do not carry the new `rawBody` field.

## What This Changes

The branch adds an optional `rawBody` field to `before_prompt_build`, `before_agent_start`, and `agent_end` plugin hook payloads, with embedded/CLI threading, queue handling, docs, and focused tests.

## Best Possible Solution

Carry the gated `rawBody` through the generic agent harness prompt and `agent_end` helper contract, add Codex/Copilot hook tests, and keep the docs aligned with the exact runtimes maintainers agree to support.

## Solution Assessment

No, not yet. The PR is a plausible implementation for embedded and CLI-runner paths, but the best fix is to put the field on the generic harness hook boundary so Codex and Copilot cannot drift from the documented plugin contract.
