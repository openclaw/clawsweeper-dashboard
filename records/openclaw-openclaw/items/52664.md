## Summary

Keep open: current main does not implement this hook field, and the PR is useful but should not merge yet because the diff can expose system-event prompt text as rawBody and lacks public docs/changelog coverage.

## What This Changes

The PR adds optional rawBody to before_prompt_build, before_agent_start, and agent_end hook events and threads it through reply, embedded-runner, deferred follow-up, and collect queue paths.

## Best Possible Solution

Land a rebased patch that exposes rawBody only for user-originated channel input, preserves immediate/follow-up/collect threading, leaves system and non-channel runs undefined, and documents/changelogs the API.

## Reproduction Assessment

Yes. The PR diff derives rawBodyForHooks from ctx.CommandBody ?? ctx.RawBody ?? ctx.Body for every prepared reply, and current heartbeat coverage shows heartbeat runs populate those fields with system prompt text.

## Solution Assessment

No. The additive hook field is the right direction, but the implementation should gate the value at the source so only user-originated channel runs carry rawBody, then add focused tests plus public docs and changelog coverage.
