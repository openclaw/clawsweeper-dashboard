## Summary

Keep this PR open: the rawBody hook API is useful and the main reply paths are mostly wired, but the latest patch still leaves active-run queue callers that can keep a stale prior rawBody on later hook events.

## What This Changes

Adds an optional rawBody field to plugin hook events and threads it through reply, embedded/CLI, Codex, Copilot, queued follow-up, collect, and steer paths with docs and tests.

## Best Possible Solution

Keep the additive hook field, but require every active-run queue injection to set rawBody explicitly to either direct-user text or undefined for internal/system handoffs, with focused tests for sessions_send, /steer, talk control, and subagent wakes before merge.

## Reproduction Assessment

Yes, from source inspection: call sessions_send or /steer into an active embedded run after a direct-user turn, and the queue options omit rawBody so currentRawBody is not refreshed or cleared. I did not run a live reproduction in this read-only review.

## Solution Assessment

No. Exposing rawBody at the hook boundary is a reasonable shape, but the implementation is not complete until the active-run queue API has explicit rawBody semantics for every caller.
