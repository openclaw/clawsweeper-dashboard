## Summary

Keep open: current main still lacks the requested hook `rawBody` API, but this PR needs real behavior proof and still has unresolved steering semantics where `agent_end` can pair final messages containing steered input with the original run-level `rawBody`.

## What This Changes

The PR adds optional `rawBody` to `before_prompt_build`, `before_agent_start`, and `agent_end` hook events, threads it through channel reply and embedded runner paths, and updates docs, tests, changelog, and the SDK baseline hash.

## Best Possible Solution

Land the additive hook API after rebasing and defining steering behavior so raw metadata is carried through steering or `rawBody` is omitted/marked when final hook messages include steered input.

## Reproduction Assessment

Yes, source-reproducible: PR head sets `agent_end.rawBody` from run params while current steering queues only prompt text and docs say Pi appends steered prompts as user messages before the next LLM call.

## Solution Assessment

No. Exposing `rawBody` is a reasonable additive API direction, but this branch should not merge until steering semantics are fixed or explicitly represented and real hook proof is attached.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid reporting stale rawBody after steered turns:** `src/agents/pi-embedded-runner/run/attempt.ts:3001`

Memory or analytics plugins can then pair the final conversation with stale raw input; carry raw metadata through steering or omit/mark `rawBody` once steered input is included.
