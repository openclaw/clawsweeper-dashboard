## Summary

Keep this PR open: the rawBody hook API is useful and not present on current main, but the branch is not merge-ready because `agent_end.rawBody` can stay tied to the original run while steered user messages are included in `messages`, and after-fix real behavior proof is still missing.

## What This Changes

The branch adds optional `rawBody` to `before_prompt_build`, `before_agent_start`, and `agent_end` hook events and threads it through reply, embedded-run, queued-run, docs, SDK baseline, tests, and changelog surfaces.

## Best Possible Solution

Land the additive hook API after rebasing and defining steering semantics, either by carrying raw metadata through steering or omitting/marking `rawBody` when steered input is included, with redacted live hook proof attached.

## Reproduction Assessment

Yes, source-reproducible: current main's steering API queues only prompt text, queue docs say steered messages are appended as user messages, and the PR emits `agent_end.rawBody` from the original run params.

## Solution Assessment

No. Exposing `rawBody` is a reasonable API direction, but this implementation should not merge until steered-run semantics and real behavior proof are resolved.
