## Summary

Keep this PR open: the additive hook API is useful and the happy path has credible proof, but the current steer-path implementation can leave `event.rawBody` stale when the latest steered input is provenance-gated away.

## What This Changes

The PR adds optional `rawBody` to plugin hook events and threads it through channel reply, queue, embedded-agent, and CLI-backed run paths.

## Best Possible Solution

Keep the additive hook field, but make the steer queue carry an explicit rawBody update/clear state so hook events cannot inherit stale direct-user text when the latest steered input is not direct external-user input.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: start a streaming run with direct external-user `rawBody`, then steer an inter-session or internal-system input whose `followupRun.rawBody` is `undefined`; the queue omits the option and `currentRawBody` is not cleared.

## Solution Assessment

No. The additive API shape is useful, but the best implementation needs an explicit rawBody update/clear signal across the steer queue so the absence contract cannot be confused with an omitted option.
