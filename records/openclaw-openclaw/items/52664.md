## Summary

Keep open: the hook API addition is still useful and is not on current main, but the branch still has unresolved steering semantics, no required real behavior proof, and a conflicting head.

## What This Changes

The PR adds optional `rawBody` to plugin hook events and threads it from channel reply handling through queued and embedded agent runs, with docs, SDK baseline, tests, and changelog updates.

## Best Possible Solution

Land the additive hook API after rebasing and defining steering behavior, either by carrying raw metadata through steering or omitting/marking `rawBody` when steered input is present.

## Reproduction Assessment

Yes, source-reproducible: current main steers only prompt text into active runs, docs say those messages are appended as user messages, and the PR reports `agent_end.rawBody` from the original run params.

## Solution Assessment

No. Exposing `rawBody` is a reasonable API direction, but this branch should not merge until steering semantics are fixed or explicitly represented and live hook proof is attached.
