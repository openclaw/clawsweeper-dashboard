## Summary

Keep this PR open for a focused repair: the main capability is useful and well demonstrated, but the steer path can associate hook events with text that the active run rejected. The additive plugin API also warrants compatibility review because plugins may rely on this field as the accepted user input.

## What This Changes

The PR adds an optional `rawBody` field to three plugin lifecycle events and propagates clean inbound text through immediate, deferred, collected, and steered agent runs.

## Best Possible Solution

Commit the active run’s `rawBody` state only after the matching steered message is successfully accepted, with focused failure-path coverage proving rejected steering retains the previously accepted input.

## Reproduction Assessment

Yes for the review finding: the source deterministically updates `currentRawBody` before an awaited steering operation that can reject, producing a clear failure path without requiring a live transport race.

## Solution Assessment

No, not yet. The additive hook field is a maintainable direction, but its mutable run state must be transactionally coupled to accepted steering input before it is a reliable plugin contract.
