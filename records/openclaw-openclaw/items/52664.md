## Summary

Keep open: the feature is useful and well-proven for direct channel messages, but the current PR head exposes the new public hook field before checking existing input provenance, which can turn inter-session or internal text into apparent direct user input for plugins.

The PR adds optional `rawBody` to `before_prompt_build`, legacy `before_agent_start`, and `agent_end` hook events, threads it through reply/followup/collect/steer embedded-run paths, and documents/tests the new hook payload.

## Best Possible Solution

Gate `rawBody` to direct external-user channel input, document provenance-marked absence semantics, and then let maintainers decide whether to accept the additive plugin hook API.

## Solution Assessment
