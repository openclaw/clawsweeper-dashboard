## Summary

This PR should stay open: current main does not already expose `rawBody` on these hook events, but the branch still has a blocking active-run steering semantics bug and lacks contributor-provided real behavior proof.

## What This Changes

The branch adds an optional `rawBody` field to `before_prompt_build`, `before_agent_start`, and `agent_end` hook events and threads it through reply and embedded-run queue paths, docs, SDK baseline hashes, tests, and changelog.

## Best Possible Solution

Land the additive hook API after it defines and implements active-run steering semantics, either by carrying raw metadata through steering or by omitting/marking `rawBody` when steered input enters `messages`, and after redacted live hook proof is posted.

## Reproduction Assessment

Yes, source-reproducible: the steer path passes only decorated prompt text, the embedded queue API accepts only text, queue docs say steering appends user messages, and the PR emits `agent_end.rawBody` from the original run params.

## Solution Assessment

No. Exposing `rawBody` is a reasonable API direction, but this diff should not merge until steered-turn semantics and real behavior proof are resolved.
