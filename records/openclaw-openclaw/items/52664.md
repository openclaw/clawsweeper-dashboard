## Summary

The PR is still necessary and its main channel/system-event behavior is well proven, but the steer path mutates the hook’s run-level input state before confirming that the new message was accepted, so it is not ready to merge at the current head.

## What This Changes

The PR adds an optional clean user-input field to `before_prompt_build`, legacy `before_agent_start`, and `agent_end`, threading it through immediate, deferred, collected, and steered agent runs.

## Best Possible Solution

Make `rawBody` represent the latest successfully accepted user input by updating it only after steering succeeds, with rejection-path coverage, then land the documented additive hook contract after owner review.

## Reproduction Assessment

Yes from source for the failed-steer edge: the branch updates `currentRawBody` before the awaited steering operation that can reject, although no live rejection reproduction was supplied.

## Solution Assessment

No at the current head: threading canonical inbound text through existing paths is the right design, but the run state must track accepted messages rather than attempted injections.
