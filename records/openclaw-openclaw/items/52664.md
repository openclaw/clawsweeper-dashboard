## Best Possible Solution

Land this only after the active-run queue contract consistently sets or clears rawBody for every injection path and maintainers accept the plugin API semantics as the stable hook contract.

No. The proposed API shape is plausible, but the implementation is not the best complete fix until one queue-level contract makes every active-run injection either set direct-user rawBody or clear it explicitly.
