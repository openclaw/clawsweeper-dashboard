The PR adds an optional clean user-input field to plugin lifecycle hooks and threads it through immediate, deferred, collected, and steered agent runs.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Mostly yes: adding one optional field at the existing plugin boundary is narrower and more stable than requiring plugins to parse channel envelopes, but collect aggregation must honor the documented contract.
