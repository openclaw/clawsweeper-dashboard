## Summary

Keep this PR open, but it still needs changes before merge: the additive hook API is useful, yet `agent_end.rawBody` can stay tied to the original run while active-run steering appends later user messages, and no after-fix real behavior proof has been posted.

## What This Changes

The PR adds an optional `rawBody` field to plugin hook events, threads it through reply and embedded-run paths, and updates hook docs, SDK baseline hashes, tests, and changelog.

## Best Possible Solution

Land the additive hook API once `agent_end` semantics stay correct for steered turns and redacted live hook output proves `rawBody` behavior.

## Reproduction Assessment

Yes, by source inspection: active-run steering queues only decorated prompt text, while the PR reports `agent_end.rawBody` from the original run params and queue-steering docs say steered input is appended as user messages before the next model call.

## Solution Assessment

No. Exposing `rawBody` is a reasonable API direction, but the PR should first avoid stale `agent_end.rawBody` after accepted steering or carry raw metadata through the steer queue.
