## Summary

Keep open: the `rawBody` hook field is useful and the supplied live proof covers the main channel and system-event paths, but the PR still has a source-proven steered-input clearing bug and adds a public plugin-hook API that needs maintainer acceptance.

## What This Changes

The PR adds an optional `rawBody` field to prompt-build, legacy before-agent-start, and agent-end plugin hook events and threads it through channel, queue, steer, embedded-agent, and CLI-backed run paths.

## Best Possible Solution

Define `rawBody` as an accepted plugin hook contract, pass an explicit update-or-clear signal through steer queue options, cover direct-to-gated steer regression, and keep docs/types aligned.

## Reproduction Assessment

Yes, source-reproducible. A direct channel turn can set `currentRawBody`, then a steered `inter_session` or `internal_system` input is gated to `undefined` but the PR omits any clear signal, so the active run keeps the old value.

## Solution Assessment

No. Threading clean user input to hooks is a useful shape, but the best fix must explicitly clear steered `rawBody` for gated inputs and then get maintainer acceptance of the plugin API contract.
