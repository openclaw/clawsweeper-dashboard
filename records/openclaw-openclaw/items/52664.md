## Summary

Keep open: current main still lacks this hook API, but the latest PR head still has unresolved steered-turn `agent_end.rawBody` semantics and no contributor-provided real behavior proof.

## What This Changes

The PR adds optional `rawBody` fields to plugin hook events, threads clean user input through reply and embedded-run paths, and updates hook docs, SDK baseline hashes, tests, and changelog.

## Best Possible Solution

Land the additive hook API after it either carries raw metadata through active-run steering or omits/marks `rawBody` when messages include steered input, and after redacted live hook output proves the behavior.

## Reproduction Assessment

Yes, source-reproducible: active-run steering queues only decorated prompt text, queue steering appends that text as user messages, and the PR emits `agent_end.rawBody` from the original run params.

## Solution Assessment

No. Exposing `rawBody` is a reasonable API direction, but the current diff should first resolve steered-turn semantics and provide real behavior proof.
