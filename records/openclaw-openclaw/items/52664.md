## Summary

Keep open: the API direction is plausible, but the branch still has a blocking steering semantics issue and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds optional `rawBody` to plugin hook events, threads it through reply and embedded-run paths, and updates hook docs, SDK baseline hashes, tests, and changelog.

## Best Possible Solution

Land the additive hook API only after steering semantics are unambiguous and redacted real plugin/channel proof shows `rawBody` in hook events.

## Reproduction Assessment

Yes, by source inspection: active-run steering queues only decorated prompt text, while the PR adds `agent_end.rawBody` from the original run params and steering docs say queued messages are appended as user messages before the next model call. I did not run a live steer session in this read-only review.

## Solution Assessment

No. Exposing `rawBody` is a reasonable API direction, but `agent_end` should omit or mark it after accepted steering, or carry raw input metadata through the steer queue before landing.
