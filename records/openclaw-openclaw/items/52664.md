## Summary

Keep open. Current main still lacks `rawBody` on the target plugin hook events, and this PR addresses a real plugin API gap, but the branch is not merge-ready because it exposes system-event text as `rawBody` despite the new contract and lacks the required docs/changelog/test coverage.

## What This Changes

The PR adds optional `rawBody` to `before_prompt_build`, legacy `before_agent_start`, and `agent_end` hook events and threads it through reply, embedded-runner, deferred follow-up, and collect queue paths.

## Best Possible Solution

Land a rebased branch that exposes `rawBody` only for user-originated channel input, carries it through immediate/follow-up/collect paths, leaves system and non-channel runs undefined, and updates plugin docs, changelog, and focused tests.

## Reproduction Assessment

Yes. The supplied diff derives `rawBody` unconditionally from `ctx.CommandBody ?? ctx.RawBody ?? ctx.Body`, and current heartbeat paths/tests provide heartbeat prompt text in those fields, so a focused heartbeat hook test can reproduce the contract violation.

## Solution Assessment

No. The additive hook field is the right product direction, but the implementation should gate the value before storing or forwarding it so system-event and non-channel runs cannot masquerade as raw user input.
