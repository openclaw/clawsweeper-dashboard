## Summary

Keep open. The feature is useful and current main does not already expose `rawBody` on these hook events, but the PR is not merge-ready because provenance-gated steered input can leave a previous direct-user `rawBody` visible on later hook events.

## What This Changes

Adds optional `rawBody` to plugin hook events and threads channel-origin clean user input through reply, queue, steer, embedded-agent, and CLI paths with docs and tests.

## Best Possible Solution

Accept the hook API only with explicit update-or-clear semantics on the steer queue path, regression coverage for direct-user then gated-steer input, aligned docs/types, and clean touched agent CI.

## Reproduction Assessment

Yes, source-reproducible. A direct channel turn can set `currentRawBody`, then a steered inter-session/internal-system input is gated to `undefined` but the PR sends no clear signal, so the active run keeps the old value.

## Solution Assessment

No, not yet. Threading clean user input to hooks is a useful shape, but the best fix must explicitly clear steered rawBody for gated inputs before maintainer API acceptance.
