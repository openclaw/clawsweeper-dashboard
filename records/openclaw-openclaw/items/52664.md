## Summary

Keep open: the core rawBody hook direction is useful and mostly well proven, but the PR is not merge-ready because active-run queue injections can still leave a previous direct user's rawBody attached to later internal or handoff messages.

## What This Changes

The PR adds an optional rawBody field to before_prompt_build, before_agent_start, and agent_end plugin hook events and threads it through channel, embedded, CLI, Codex, Copilot, queued, collect, and steer paths.

## Best Possible Solution

Before merge, make active-run queue handling clear rawBody by default for non-direct injections or require every queue caller to pass an explicit rawBody/rawBody: undefined value, with tests for direct steer refresh and internal handoff clearing.

## Reproduction Assessment

Yes, from source inspection. On PR head, the embedded runner only updates currentRawBody when the rawBody key is present, while sessions_send, Talk active-run control, and subagent active-wake callers still enqueue without that key.

## Solution Assessment

No. The feature shape is plausible, but the best fix is to make the active-run queue boundary enforce refresh-or-clear semantics for all injections instead of relying on one direct reply steer caller being updated.
