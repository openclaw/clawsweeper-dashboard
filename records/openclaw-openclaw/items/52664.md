## Summary

Keep open: current main still lacks the requested hook API, so this PR is not obsolete, but it remains blocked by missing real behavior proof and a source-visible stale `rawBody` problem in the steering path.

## What This Changes

The PR adds optional `rawBody` to plugin hook events, threads it through reply and embedded-runner paths, and updates docs, tests, SDK baseline hash, and changelog.

## Best Possible Solution

Rebase onto current main and land the additive hook API only after defining steering semantics, either by carrying raw metadata through steering or omitting/marking `rawBody` when final hook messages include steered user input.

## Reproduction Assessment

Yes, source-reproducible: PR head emits `agent_end.rawBody` from original run params while current steering queues only prompt text and current docs say Pi appends steered prompts as user messages before the next LLM call.

## Solution Assessment

No. Exposing clean user input is a reasonable additive API, but this branch should not merge until the steer-mode contract is made non-misleading and real hook output proof is attached.
