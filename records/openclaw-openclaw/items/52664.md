## Summary

Keep open: the plugin hook API is useful, but the current steer path can keep exposing an older direct-user `rawBody` after a later provenance-gated steered input is intentionally cleared to `undefined`.

## What This Changes

The PR adds optional `rawBody` to prompt-build, legacy before-agent-start, and agent-end plugin hook events and threads it through channel reply, queue, steer, embedded-agent, and CLI-backed paths.

## Best Possible Solution

Define `rawBody` as an accepted plugin hook contract, pass an explicit update-or-clear signal through steer queue options, cover direct-to-gated steer regression, and keep docs/types aligned.

## Reproduction Assessment

Yes, source-reproducible. A direct channel turn can set `currentRawBody`, then a steered inter-session/internal-system input is gated to `undefined` but the PR omits any clear signal, so the active run keeps the old value.

## Solution Assessment

No. Threading clean user input to hooks is a useful shape, but the best fix must explicitly clear steered rawBody for gated inputs and then get maintainer acceptance of the plugin API contract.
