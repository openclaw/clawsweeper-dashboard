## Summary

Keep open: current main still lacks the proposed hook field, and this PR remains useful but not merge-ready because the steered-turn semantics are still inconsistent with the proposed contract and no real behavior proof has been posted.

## What This Changes

The PR adds optional `rawBody` to plugin hook events and threads it through channel reply, queue, embedded runner, docs, SDK baseline hash, tests, and changelog surfaces.

## Best Possible Solution

Land the additive hook API after rebasing, attaching real hook proof, and defining steering behavior so `rawBody` never conflicts with `agent_end.messages`.

## Reproduction Assessment

Yes, source-reproducible for the review finding: PR head emits `agent_end.rawBody` from run params while current steering appends queued prompt text as user messages without raw metadata. I did not run a live channel/plugin reproduction in this read-only review.

## Solution Assessment

No. Exposing `rawBody` is a reasonable additive API direction, but this branch should not merge until steering semantics are fixed or explicitly represented and real hook proof is attached.
