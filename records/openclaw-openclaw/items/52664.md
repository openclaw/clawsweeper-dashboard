## Summary

Keep open: the feature is useful and the supplied live proof covers the happy path, but current PR head still publishes an unsafe and incomplete plugin hook contract for `rawBody`. The field is not gated by input provenance, and the CLI-backed hook path is left without the new value for channel-originated runs.

## What This Changes

Adds optional `rawBody` to plugin hook events and threads it through reply, followup, collect, steer, embedded-agent, docs, and focused tests.

## Best Possible Solution

Gate `rawBody` to direct external-user channel input, forward it through every hook runner path that handles channel-originated runs, update docs for provenance-marked absence, and then let maintainers decide whether to accept the additive plugin API.

## Reproduction Assessment

## Solution Assessment

No. Threading clean user text through the reply/run path is the right shape, but it is not the best solution until provenance and sibling hook surfaces share the same contract.
