## Summary

The capability remains useful and the normal-path proof is strong, but the current head mutates lifecycle-hook state before a steered message is confirmed accepted, so a rejected steer can expose input that never entered the active run.

## What This Changes

Adds an optional clean user-input field to plugin lifecycle hook events and threads it through immediate, queued, collected, and steered agent runs.

## Best Possible Solution

Update currentRawBody only after the corresponding steered message is successfully accepted, then add a focused rejection-path regression test proving failed steering leaves lifecycle-hook input unchanged.

## Reproduction Assessment

Yes. Source provides a high-confidence path: queueMessage can reject asynchronously, while the PR changes currentRawBody before awaiting that operation.

## Solution Assessment

No, not yet. Exposing clean input on lifecycle hooks is a reasonable solution, but the steer-path state change must occur only after the associated message is accepted.
