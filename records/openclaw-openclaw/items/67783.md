## Summary

Keep open: the PR has strong proof for the Feishu topic typing bug, but it still needs a blocking fix because card-action callbacks use synthetic message ids that would become typing-reaction targets.

## What This Changes

The PR adds a separate Feishu inbound message id for typing reactions while preserving the existing reply/thread target, plus focused Feishu tests.

## Best Possible Solution

Preserve topic/root reply placement, target typing reactions at real inbound Feishu message ids, and make synthetic card-action callbacks fall back to the real reply target or skip typing with regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main routes topic replies to the root while the dispatcher uses that same id for typing reactions; the PR body also provides live Feishu/Lark before/after screenshots for the visible behavior.

## Solution Assessment

No, not as written. Splitting reply placement from typing targeting is the right narrow fix, but the preferred typing target must be limited to real Feishu message events so synthetic card-action ids are not sent to the reaction API.
