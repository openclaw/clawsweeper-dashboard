## Summary

Keep open: current main still has a source-reproducible Feishu topic typing-target bug and this PR addresses it with real proof, but the patch still lets synthetic card-action ids become Feishu reaction targets, so it needs a narrow repair before merge.

## What This Changes

The branch threads a Feishu inbound message id into the reply dispatcher so typing reactions prefer the user's current message over the reply target, with focused tests.

## Best Possible Solution

Preserve topic/root reply placement, target typing reactions at real inbound Feishu message ids, and make synthetic card-action callbacks fall back to the real reply target or skip typing with regression coverage.

## Reproduction Assessment

Yes for source-level reproduction: current main routes topic or configured replyInThread replies to the root while the dispatcher uses that same id for typing reactions. The PR screenshots provide live before/after evidence of the visible Feishu/Lark behavior.

## Solution Assessment

No, not as written. Splitting reply placement from typing targeting is the right narrow fix, but the preferred typing target must be limited to real Feishu message events so synthetic card-action ids are not sent to the reaction API.
