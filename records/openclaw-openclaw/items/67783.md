## Summary

Keep open. Current main still mis-targets Feishu typing reactions in topic/replyInThread mode, and this PR is a narrow plausible fix, but the current diff can send typing reactions to suppressed synthetic card-action ids when no real reply target exists.

## What This Changes

This PR adds an optional Feishu inbound message id, passes `ctx.messageId` into the reply dispatcher, uses it for typing-reaction targeting with `replyToMessageId` fallback, and adds focused tests.

## Best Possible Solution

Preserve topic/root reply routing while targeting typing reactions at the actual inbound message only when a real reaction anchor exists, with regression tests for both topic messages and suppressed synthetic card actions plus the required user-facing changelog entry.

## Reproduction Assessment

Yes. For the reported bug, a topic/replyInThread event with different `ctx.rootId` and `ctx.messageId` reproduces from source because current main replies to the root and reacts to that same id. The PR regression is reproducible with `inboundMessageId` present while `replyToMessageId` is omitted, matching card-action `suppress_reply_target` synthetic events.

## Solution Assessment

No, not as written. The main direction is narrow and maintainable, but the dispatcher should keep the old no-target suppression before selecting `inboundMessageId` as the reaction target.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve no-target typing suppression:** `extensions/feishu/src/reply-dispatcher.ts:168-169`
  - body: This fallback makes `onReplyStart` add a reaction whenever `inboundMessageId` is present, even when `replyToMessageId` was intentionally omitted. Feishu card actions can synthesize events with `suppress_reply_target` and ids like `card-action-${token}` when there is no real open message id, so this would send reaction API calls to invalid synthetic ids. Keep the old no-target guard, then use the inbound id only as the target when a real reply/reaction anchor exists.
  - confidence: 0.86
