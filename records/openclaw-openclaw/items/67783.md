## What This Changes

The PR adds an optional Feishu inbound message id, passes `ctx.messageId` into both reply-dispatcher call sites, prefers it for typing reactions, and adds focused Feishu tests.

## Best Possible Solution

Gate the new inbound typing target to real Feishu message events, preserve topic/root reply placement, and make synthetic callbacks fall back to a real reply target or skip typing with regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main routes topic replies to the root while using that same id for typing reactions, and the PR body supplies live Feishu/Lark screenshots for the before/after behavior.

## Solution Assessment

No, not as written. Splitting reply placement from typing-reaction targeting is the right fix shape, but the preferred typing target must exclude synthetic card-action and bot-menu events.
