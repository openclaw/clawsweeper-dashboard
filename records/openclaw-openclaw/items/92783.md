## Summary

Keep open: current main has `cron.run` force-run and an immediate heartbeat wake path, but it still lacks the requested deterministic no-spend dry-run or scheduler dispatch preview. The adjacent pre-spend cancel hook at https://github.com/openclaw/openclaw/issues/92296 and non-agent cron command work at https://github.com/openclaw/openclaw/issues/91534 do not supersede this scheduler diagnostics request.

## What This Changes

Request a public force-run or dry-run scheduler diagnostics interface for cron and heartbeat paths so canaries can validate dispatch without waiting for ambient ticks or paying for live model turns.

## Best Possible Solution

Define a narrow Gateway and CLI scheduler diagnostics contract that keeps existing `cron.run` and manual wake behavior intact while adding explicit no-model/no-send planning semantics for cron and heartbeat dispatch.

## Reproduction Assessment

Yes, at source level. Current main exposes cron force-run and immediate heartbeat wake paths, but the inspected protocol, CLI, docs, and runner code do not expose a no-model/no-send dry-run or heartbeat-specific scheduler diagnostic command.

## Solution Assessment

Unclear. The requested capability is useful, but the best implementation needs maintainer product design around public RPC/CLI shape and side-effect guarantees before a fix PR is safe.
