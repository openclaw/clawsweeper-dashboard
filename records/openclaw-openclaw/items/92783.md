## What This Changes

Request a scheduler diagnostic interface to force-run or dry-run heartbeat/cron scheduled actions without waiting for ambient ticks or paying for a live agent turn.

## Best Possible Solution

Define an additive Gateway/CLI scheduler diagnostics contract that preserves current `cron.run` behavior while adding explicit heartbeat/cron no-model dry-run or run-planning semantics after maintainer API review.

## Reproduction Assessment

Yes, at source level. Current main exposes real cron force-run and manual heartbeat wake paths, but no heartbeat run command or no-model scheduled-action dry-run parameter exists in the CLI or Gateway protocol.

## Solution Assessment

Unclear as an implementation. The request is useful, but the best fix needs maintainer design around public RPC/CLI shape, no-side-effect guarantees, run logging, and delivery rerouting rather than a one-off flag.
