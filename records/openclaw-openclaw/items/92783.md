## Summary

Keep open: current main already has a cron force-run path, but the requested deterministic no-spend dry-run and heartbeat-specific exercise path remain missing. The related pre-spend hook issue addresses model-call cancellation, not this scheduler diagnostic interface.

## What This Changes

Request adds a deterministic scheduler diagnostic interface to force-run or dry-run heartbeat/cron jobs without waiting for ambient scheduling or paying for a live agent turn.

## Best Possible Solution

Define a narrow Gateway/CLI scheduler diagnostics contract that preserves existing `cron.run` behavior while adding explicit no-model dry-run or heartbeat-run planning semantics only after maintainer API review.

## Reproduction Assessment

Yes, at source level. Current main exposes cron manual run, but the heartbeat CLI has only last/enable/disable and the wake path schedules a real heartbeat runner rather than a dry-run/no-model dispatch preview.

## Solution Assessment

Unclear. A scheduler dry-run could be useful, but the best implementation needs maintainer design around public RPC/CLI shape, no-side-effect guarantees, run logging, and delivery rerouting.
