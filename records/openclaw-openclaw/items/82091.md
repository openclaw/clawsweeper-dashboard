## Summary

Keep this open: current main still uses the generic hot-reload stop/start path without account-preserving restart semantics, and the active fix is the unmerged PR at https://github.com/openclaw/openclaw/pull/82540.

## What This Changes

The issue reports that adding a second WeChat account through QR login hot-reloads the channel and leaves previously running WeChat account monitors stopped until a full gateway restart.

## Best Possible Solution

Land a generic gateway fix that treats hot-reload stops as manager-owned handoffs, restarts with the full known live account set, and keeps the regression covered without adding WeChat-specific policy to core.

## Reproduction Assessment

No, not with a live WeChat account in this review. The issue has concrete logs and the current gateway plus plugin source show a clear source-reproducible hot-reload path.

## Solution Assessment

Yes, the best direction is the generic gateway reconciliation path already being attempted in https://github.com/openclaw/openclaw/pull/82540; closing this issue should wait until that or an equivalent fix is merged and validated.
