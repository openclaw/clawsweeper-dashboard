## Summary

This should stay open: the report is concrete, current main does not contain a relevant fix beyond v2026.5.12, and the implicated generic gateway reload/login paths still need a focused multi-account lifecycle regression.

## What This Changes

The issue reports that adding a second WeChat account via QR login hot reload stops the previously running WeChat account monitor until a full gateway restart.

## Best Possible Solution

Keep core channel-agnostic and make the gateway reconcile or preserve the full configured account set after plugin-owned account-index changes, covered by a fake external multi-account channel regression.

## Reproduction Assessment

No, not with a live WeChat account in this read-only review. The reporter provided concrete logs, and current source plus the @tencent-weixin/openclaw-weixin@2.4.3 package show a source-reproducible reload/login lifecycle path that is strong enough to keep open.

## Solution Assessment

Unclear until a regression pins the interleaving. The likely best fix is in generic gateway channel reconciliation, not a WeChat-specific core policy branch or plugin-internal workaround.
