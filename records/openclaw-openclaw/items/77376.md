## What This Changes

The branch adds an explicit `emitCommandHooks` opt-in to `sessions.create`, passes it from the Control UI `/new` path, exports reused reset/session hook helpers, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Land the explicit `emitCommandHooks` gate with the regression tests after correcting the changelog so release notes reference the PR that actually merges.

## Reproduction Assessment

Yes, source-level. Current main routes typed Control UI `/new` into `sessions.create`, and that handler creates a child session without the documented `command:new`/`before_reset` hook path used by `session-memory`.

## Solution Assessment

Yes, with the changelog correction. The explicit opt-in is the narrow maintainable fix because only the Control UI command transition should restore webchat hook delivery; ordinary SDK parent-session creates should stay hook-free unless explicitly opted in.
