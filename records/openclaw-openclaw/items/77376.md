## Summary

Keep this PR open: the explicit `emitCommandHooks` boundary addresses the prior broad `parentSessionKey` problem, but the protocol schema change is not merge-ready because generated Swift Gateway protocol models are stale.

## What This Changes

The PR adds an explicit `emitCommandHooks` flag to `sessions.create`, sends it from the Control UI `/new` flow, emits reset-style hooks only when set, adds gateway tests, and records a changelog entry.

## Best Possible Solution

Land the explicit hook-emission flag with regenerated protocol artifacts and regression coverage, preserving no-hook behavior for ordinary SDK programmatic child-session creates.

## Reproduction Assessment

Yes, source-level. Current main routes Control UI typed `/new` through `new-session` to `sessions.create` with `parentSessionKey`, and that handler does not emit the documented command/reset/lifecycle hooks.

## Solution Assessment

No, not merge-ready as currently proposed. The explicit flag is the narrow maintainable direction, but the protocol schema addition must be kept in sync with generated Swift protocol models.
