## Summary

Keep open: current main has the merged fix for the `config patch` half, but `openclaw config set` still creates array parents for numeric-looking path segments outside patch mode even though `channels.discord.guilds` is a string-keyed record.

## What This Changes

The issue reports that `openclaw config patch` and `openclaw config set` treat all-digit object keys as array indexes, breaking Record-shaped config such as Discord guild IDs.

## Best Possible Solution

Make `config set` choose object containers for schema-backed record/object paths with numeric-looking keys while preserving array containers for actual array paths, then add focused regression coverage.

## Reproduction Assessment

Yes at source level: current `setAtPath` still creates array parents for all-digit next segments when `successMode` is `set`, while the Discord guilds schema requires an object record. I did not execute the CLI because this review is read-only.

## Solution Assessment

Yes: the narrow maintainable fix is a config CLI path-construction repair plus regression tests, not a new config option or a rollback of the already-merged patch-mode fix.
