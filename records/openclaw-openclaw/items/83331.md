## Summary

Keep open: current main has the merged fix for the `config patch` half, but `config set` still uses numeric-index container creation for numeric-looking path segments. The linked open PR is the implementation candidate, so this issue should stay open until that PR lands or is replaced.

## What This Changes

Preserve numeric-looking string object keys in `openclaw config patch` and `openclaw config set`, especially Discord guild IDs under `channels.discord.guilds`.

## Best Possible Solution

Land a focused config CLI fix that preserves numeric Record keys for `config set` while keeping bracket-index array paths working, with regression coverage for both cases.

## Reproduction Assessment

Yes, at source level: current `config set` passes `numericObjectKeys: false`, so `channels.discord.guilds.<snowflake>.requireMention` can create an array where the schema requires a record. I did not run the CLI because this review is read-only.

## Solution Assessment

Yes on direction, but the implementation should preserve bracket-index array creation rather than globally turning numeric path segments into object keys. The linked PR is the right candidate to review and repair before this issue closes.
