## What This Changes

The issue asks `config patch` and `config set` to preserve numeric-looking string keys for Record/object config paths such as Discord guild IDs instead of treating them as array indices.

## Best Possible Solution

Land a fix that preserves numeric string keys for schema-backed Record/object paths while keeping bracket-index array paths working, then close this issue after the linked PR or a replacement merges.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: `config set` still passes `numericObjectKeys: false`, so a numeric Discord snowflake segment can auto-create an array where the schema requires a record object.

## Solution Assessment

Is this the best way to solve the issue? Yes in direction: fix the config CLI path-construction logic and add focused regression coverage, while preserving documented bracket-index array behavior.
