## What This Changes

The issue asks for all-digit string keys to remain object keys for Record-typed config paths in both `config patch` and `config set`.

## Best Possible Solution

Land the schema-aware config writer fix in https://github.com/openclaw/openclaw/pull/83769 after normal maintainer review, then let the closing reference close this issue.

## Reproduction Assessment

Yes, at source level: current main still creates missing containers from all-digit next segments for `config set`, while `channels.discord.guilds` is a string-keyed record. I did not execute the mutating CLI path because this review is read-only.

## Solution Assessment

Yes: a schema-aware container choice for config path writes is the narrow maintainable fix, because it preserves numeric record keys without globally disabling numeric array-index behavior.
