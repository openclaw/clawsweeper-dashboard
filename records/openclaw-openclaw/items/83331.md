## Summary

Keep open: current main has the merged fix for the `config patch` half, but path-based `openclaw config set` still creates array parents for numeric-looking segments outside patch mode, while `channels.discord.guilds` is a string-keyed object record.

## What This Changes

The issue requests that config patch/set preserve numeric-looking string keys under object/Record config paths such as `channels.discord.guilds`.

## Best Possible Solution

Make `config set` choose object containers for schema-backed record/object paths with numeric-looking keys while preserving array containers for actual array paths, then add focused regression coverage.
