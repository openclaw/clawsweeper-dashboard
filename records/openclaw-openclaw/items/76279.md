## Summary

Keep open: current main still has the CLI context-engine ingestion gap, and this PR is aimed at the right path, but the branch still skips valid engines without `afterTurn` and its new `SessionManager` assistant fixtures fail PR-head type checks.

## What This Changes

The PR adds CLI post-turn context-engine ingestion before CLI compaction and expands `cli-compaction` tests for normal, no-budget, throwing, and missing-`afterTurn` cases.

## Best Possible Solution

Mirror the embedded and harness lifecycle for CLI turns after transcript persistence: call `afterTurn` when available, otherwise feed the new messages through `ingestBatch` or per-message `ingest`, while keeping no-budget ingestion separate from compaction.

## Reproduction Assessment

Yes. Source inspection shows current main persists CLI turns and then runs a CLI compaction lifecycle that never feeds the completed turn into `afterTurn`, `ingestBatch`, or `ingest`; I did not run a live gateway repro during this read-only review.

## Solution Assessment

No, not as written. Calling the context engine from the CLI path is the right direction, but the patch should preserve valid engines without `afterTurn` and fix the type-invalid fixtures before merge.

## Review Findings

Overall correctness: patch is incorrect
