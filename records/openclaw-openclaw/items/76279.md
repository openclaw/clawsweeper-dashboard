## Summary

Keep open: the PR fixes the right CLI context-engine gap, but the current patch still skips valid engines that omit optional `afterTurn()` and it is missing the required user-facing changelog entry.

## What This Changes

The PR adds CLI post-turn context-engine ingestion before CLI compaction and adds unit coverage for normal, no-budget, throwing, and no-`afterTurn` cases.

## Best Possible Solution

Mirror the embedded and harness post-turn contract in the CLI lifecycle: call `afterTurn` when available, otherwise ingest the just-appended turn via `ingestBatch` or per-message `ingest`, with focused tests and a single-line changelog entry.

## Reproduction Assessment

Yes. Current main can be verified by reading `runCliTurnCompactionLifecycle`, and the PR’s remaining bug is reproducible with a valid `ContextEngine` that implements required `ingest()` but has no `afterTurn()`.

## Solution Assessment

No. Calling `afterTurn` for CLI turns is the right direction, but skipping the documented ingest fallback is not compatible with the current context-engine contract.
