## Summary

Keep open: the PR addresses a real current-main CLI context-engine gap, but the proposed patch still skips valid context engines that implement `ingest`/`ingestBatch` without optional `afterTurn`, and it is missing the required changelog entry.

## What This Changes

The PR adds CLI post-turn context-engine `afterTurn` ingestion before existing CLI compaction and adds unit coverage for normal, no-budget, throwing, and no-`afterTurn` cases.

## Best Possible Solution

Update the CLI lifecycle to match the embedded and harness post-turn contract: call `afterTurn` when available, otherwise ingest the just-appended turn through `ingestBatch` or per-message `ingest`, with focused tests and a changelog entry.

## Reproduction Assessment

Yes. Current main can be verified by reading `runCliTurnCompactionLifecycle`, and the PR’s remaining bug is reproducible with a valid `ContextEngine` that has required `ingest()` but no `afterTurn`.

## Solution Assessment

No. Calling `afterTurn` for CLI turns is the right direction, but the patch should preserve the documented fallback path for all valid context engines and record the user-facing fix in `CHANGELOG.md`.
