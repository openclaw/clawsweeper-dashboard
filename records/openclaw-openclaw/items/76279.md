## Summary

Keep open: the PR targets a real CLI context-engine ingestion bug, but the current patch still skips valid engines that lack `afterTurn` and its added fixtures fail current test typing.

## What This Changes

The PR changes CLI compaction to call the context engine after persisted CLI turns and adds focused tests for normal, no-budget, throwing, and no-`afterTurn` cases.

## Best Possible Solution

Mirror the embedded and harness context-engine lifecycle in the CLI path: after transcript persistence, call `afterTurn` when available and otherwise fall back to `ingestBatch` or per-message `ingest`, while preserving current initialization and compaction behavior.

## Reproduction Assessment

Yes, by source inspection. Current main persists CLI turns and then enters a lifecycle that never feeds the completed turn to `afterTurn`, `ingestBatch`, or `ingest`.

## Solution Assessment

No, not as written. Calling the context engine from the CLI path is the right direction, but the patch must add the fallback ingestion path, fix the invalid assistant-message fixtures, and rebase over current CLI compaction initialization.
