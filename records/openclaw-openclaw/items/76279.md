## Summary

Keep open. The reported CLI context-engine ingestion gap is still present on current main and the PR has adequate after-fix live-output proof, but the patch is not mergeable as written because it skips valid engines without `afterTurn` and adds type-invalid `SessionManager` fixtures.

## What This Changes

The PR adds CLI post-turn context-engine ingestion before the existing CLI compaction path and adds tests for normal, no-budget, throwing, and missing-`afterTurn` cases.

## Best Possible Solution

Mirror the embedded and harness lifecycle for CLI turns: initialize and resolve the context engine after transcript persistence, call `afterTurn` when available, otherwise feed the new turn through `ingestBatch` or per-message `ingest`, and keep the no-budget ingest path separate from compaction.

## Reproduction Assessment

Yes, by source inspection. Current main persists CLI turns and then runs a CLI compaction lifecycle that never feeds the completed turn into `afterTurn`, `ingestBatch`, or `ingest`.

## Solution Assessment

No, not as written. Calling `afterTurn` from the CLI path is the right direction, but the patch must keep valid engines without `afterTurn` working and fix the type-invalid fixtures before it is the maintainable fix.
