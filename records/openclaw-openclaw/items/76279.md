## Summary

Keep open: the CLI context-engine ingestion gap is valid, but this PR is not merge-ready because it skips valid engines without `afterTurn` and adds invalid assistant-message fixtures.

## What This Changes

The PR adds post-transcript context-engine ingestion to the CLI compaction lifecycle and extends `cli-compaction` tests.

## Best Possible Solution

Mirror the harness lifecycle in the CLI path: after transcript persistence, feed the completed turn through `afterTurn` when available, otherwise `ingestBatch` or per-message `ingest`, while preserving current initialization and compaction behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows the CLI path persists a turn and then calls a lifecycle that never feeds the completed turn into `afterTurn`, `ingestBatch`, or `ingest`.

## Solution Assessment

No, not as written. Calling the context engine from the CLI path is the right direction, but the patch must preserve engines without `afterTurn`, use valid assistant-message fixtures, and rebase over current initialization/import changes.
