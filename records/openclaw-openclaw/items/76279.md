## Summary

Keep open. The underlying CLI context-engine ingestion bug is valid and the PR includes real after-fix live output, but the patch still skips valid engines without `afterTurn` and adds incomplete assistant-message fixtures, so it needs a focused repair before merge.

## What This Changes

The PR adds CLI post-turn context-engine ingestion before existing CLI compaction and adds tests for normal, no-budget, throwing, and no-`afterTurn` cases.

## Best Possible Solution

Mirror the embedded and harness lifecycle for CLI turns: call `afterTurn` when available, otherwise ingest the new messages through `ingestBatch` or per-message `ingest`, while preserving current initialization and type-correct fixtures.

## Reproduction Assessment

Yes, by source inspection. Current main persists CLI turns and then runs a compaction lifecycle that never feeds completed CLI turns into the context engine, and the PR-head fallback gap is visible from the `ContextEngine` contract.

## Solution Assessment

No, not as written. Calling `afterTurn` for CLI turns is the right direction, but the patch needs the established ingest fallback and current-main-compatible tests before it is the maintainable fix.
