## Summary

Keep open: the PR targets a real current-main CLI context-engine ingestion gap, but the submitted patch still skips valid engines without `afterTurn`, has invalid test fixtures, and is merge-conflicting with newer CLI compaction initialization changes.

## What This Changes

The PR adds post-transcript context-engine ingestion for CLI-backed agent turns and expands `cli-compaction` tests.

## Best Possible Solution

Mirror the harness lifecycle in the CLI path after transcript persistence: initialize and resolve the context engine, call `afterTurn` when available, otherwise fall back to `ingestBatch` or per-message `ingest`, and keep compaction gated separately by the token budget.

## Reproduction Assessment

Yes. Source inspection on current main shows the CLI path persists a turn and then calls a lifecycle that never feeds the completed turn into `afterTurn`, `ingestBatch`, or `ingest`.

## Solution Assessment

No, not as written. Calling the context engine from the CLI path is the right direction, but the patch must preserve engines without `afterTurn`, use valid assistant-message fixtures, and rebase over current CLI compaction initialization/import changes.
