## Summary

Keep open: current main still lacks CLI context-engine ingestion, and this PR targets the right bug, but the branch is not mergeable as written because it skips valid engines without `afterTurn` and adds type-invalid SessionManager fixtures.

## What This Changes

The PR adds post-turn context-engine ingestion to CLI compaction and expands `cli-compaction` tests for normal, no-budget, throwing, and missing-`afterTurn` cases.

## Best Possible Solution

Rebase onto current main and mirror the embedded/harness finalization path for CLI: call `afterTurn` when available, otherwise feed the new messages through `ingestBatch` or per-message `ingest`, with typed SessionManager fixtures and focused tests.

## Reproduction Assessment

Yes. Source inspection shows the CLI turn is persisted and then the current CLI compaction lifecycle never feeds the completed turn into the context engine; no live repro was run because this review is read-only.

## Solution Assessment

No, not as written. Calling the context engine from the CLI path is the right direction, but the patch must preserve valid engines without `afterTurn` and use valid assistant-message fixtures before merge.
