## Summary

Keep this PR open: current main still has the CLI context-engine ingestion gap, and the PR has useful source work, but the branch needs focused repairs before merge.

## What This Changes

The PR adds CLI post-turn context-engine ingestion before CLI compaction and extends CLI compaction tests for normal, no-budget, throwing, and missing-afterTurn cases.

## Best Possible Solution

Rebase and repair the PR so CLI turns use the same completed-turn context-engine lifecycle as harness/embedded runs while keeping compaction budget-gated.

## Reproduction Assessment

Yes, by source inspection rather than by running a live CLI turn in this read-only review. Current main persists CLI turns and then enters a lifecycle that never feeds the completed turn to afterTurn, ingestBatch, or ingest unless compaction happens.

## Solution Assessment

No, not as written. The insertion point is right, but the patch needs the ingest fallback, turn maintenance, valid SessionManager fixtures, and current-main initialization behavior before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Fall back to ingest when afterTurn is absent:** `src/agents/command/cli-compaction.ts:212`
  - body: `afterTurn` is optional, but `ingest()` is required by the context-engine contract. This branch silently skips engines that only implement `ingestBatch()` or `ingest()`, so CLI turns can still remain invisible to valid plugins; mirror the harness/embedded fallback path here.
  - confidence: 0.94
- **[P2] Run turn maintenance after successful ingestion:** `src/agents/command/cli-compaction.ts:223`
  - body: The completed-turn lifecycle runs context-engine maintenance after a successful afterTurn/ingest finalization. This new CLI path stops after afterTurn, so engines that rely on `maintain({ reason: "turn" })` still miss CLI turn maintenance.
  - confidence: 0.86
- **[P2] Preserve context-engine initialization on rebase:** `src/agents/command/cli-compaction.ts:199`
  - body: Current main initializes built-in context engines before resolving the CLI compaction engine. The PR head resolves directly from a stale base, so the rebase must keep `ensureContextEnginesInitialized()` before `resolveContextEngine()` or it can regress built-in engine resolution.
  - confidence: 0.84
- **[P2] Use valid assistant-message fixtures:** `src/agents/command/cli-compaction.test.ts:190-194`
  - body: The new `SessionManager.appendMessage()` fixtures append assistant objects that fail the current message type contract; CI reports TS2345 for these calls. Build the fixtures with a valid assistant-message helper including the required assistant metadata before merge.
  - confidence: 0.95
