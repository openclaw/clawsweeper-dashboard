## Summary

## What This Changes

The PR modifies CLI compaction to call `contextEngine.afterTurn` after persisted CLI turns and adds tests for afterTurn ingestion and failure handling.

## Best Possible Solution

Continue the shared CLI-runner finalization fix in https://github.com/openclaw/openclaw/pull/81869 against https://github.com/openclaw/openclaw/issues/81558, with fallback ingest and turn maintenance.

## Reproduction Assessment

Yes. Source inspection shows current main can persist CLI turns without a normal post-turn context-engine finalizer, and the canonical issue includes live claude-cli versus embedded Codex evidence for the same LCM gap.

## Solution Assessment

No, not this PR as written. The maintainable fix belongs in the shared CLI-runner lifecycle so it can reuse `afterTurn`, `ingestBatch`/`ingest` fallback, and turn maintenance instead of a `cli-compaction`-only hook.
