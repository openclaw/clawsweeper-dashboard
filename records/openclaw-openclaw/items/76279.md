## Summary

Keep this PR open: it targets a real current-main CLI context-engine ingestion gap, but the branch still has two concrete blockers before merge: it skips valid engines without `afterTurn`, and the new SessionManager fixtures fail test type checking.

## What This Changes

The PR adds CLI post-turn context-engine ingestion before CLI compaction and expands `cli-compaction` tests for normal, no-budget, throwing, and missing-`afterTurn` cases.

## Best Possible Solution

Mirror the embedded and harness lifecycle for CLI turns after transcript persistence: call `afterTurn` when available, otherwise feed the new messages through `ingestBatch` or per-message `ingest`, while keeping no-budget ingestion separate from compaction.

## Reproduction Assessment

Yes. Source inspection shows current main persists CLI turns and then runs a CLI compaction lifecycle that never feeds the completed turn into `afterTurn`, `ingestBatch`, or `ingest`.

## Solution Assessment

No, not as written. Calling the context engine from the CLI path is the right direction, but the patch should preserve valid engines without `afterTurn` and fix the type-invalid fixtures before merge.
