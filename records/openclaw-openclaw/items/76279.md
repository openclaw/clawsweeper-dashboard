## Summary

Keep open: current main still lacks CLI post-turn context-engine ingestion, and this PR is useful, but the patch is not mergeable because it skips valid engines without optional `afterTurn` and adds type-invalid SessionManager message fixtures.

## What This Changes

The PR adds CLI agent post-turn context-engine ingestion before existing CLI compaction and adds tests for normal, no-budget, throwing, and no-`afterTurn` cases.

## Best Possible Solution

Update the CLI lifecycle to mirror the embedded and harness post-turn contract: call `afterTurn` when available, otherwise ingest the just-appended CLI messages through `ingestBatch` or per-message `ingest`, with type-correct tests on current main.

## Reproduction Assessment

Yes, by source inspection. Current main’s CLI compaction lifecycle never feeds completed CLI turns into the context engine, and the PR’s remaining defect is visible from the `ContextEngine` type contract because valid engines can omit optional `afterTurn`.

## Solution Assessment

No, not as written. Calling `afterTurn` for CLI turns is the right direction, but the maintainable fix should keep the established ingest fallback path and use valid SessionManager message fixtures.
