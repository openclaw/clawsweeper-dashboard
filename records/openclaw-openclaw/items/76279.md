## Summary

Keep this PR open: it targets a real CLI context-engine gap and includes useful live Lossless-Claw proof, but the patch still skips valid engines that implement the required ingest path without optional afterTurn and its new test fixtures fail type checking.

## What This Changes

The PR adds CLI post-turn context-engine ingestion before existing CLI compaction and adds unit coverage for normal, no-budget, throwing, and no-afterTurn cases.

## Best Possible Solution

Update the CLI lifecycle to mirror the embedded and harness post-turn contract: call afterTurn when available, otherwise deliver the just-appended CLI messages through ingestBatch or per-message ingest, with type-correct tests and current-main initialization preserved.

## Reproduction Assessment

Yes. Source inspection on current main shows the CLI lifecycle never calls afterTurn, ingestBatch, or ingest, and the remaining PR defect is reproducible from the ContextEngine type contract with any valid engine that lacks optional afterTurn.

## Solution Assessment

No. Calling afterTurn for CLI turns is the right direction, but the patch should preserve the documented ingest fallback path and fix the new SessionManager test fixtures before it is mergeable.
