## Summary

Keep open. The bug is valid and the PR is pointed at the right CLI lifecycle, but the current patch still skips valid context engines without `afterTurn` and its new `SessionManager` assistant fixtures fail test typechecking.

## What This Changes

The PR adds post-turn context-engine ingestion to the CLI compaction lifecycle and expands `cli-compaction` tests for ingestion, no-budget, failure, and no-`afterTurn` cases.

## Best Possible Solution

Mirror the harness and embedded lifecycle after CLI transcript persistence: call `afterTurn` when available, otherwise use `ingestBatch` or per-message `ingest`, while keeping no-budget ingestion separate from compaction.

## Reproduction Assessment

Yes. Source inspection shows current main persists CLI turns and then runs a CLI compaction lifecycle that never feeds the completed turn into `afterTurn`, `ingestBatch`, or `ingest`.

## Solution Assessment

No, not as written. Calling the context engine from the CLI path is the right fix direction, but the patch needs the established ingest fallback and valid assistant-message fixtures before merge.
