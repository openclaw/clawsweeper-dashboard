## Summary

Keep this PR open: it addresses a real CLI context-engine ingestion gap, but the patch still skips valid engines that lack `afterTurn` and its new SessionManager fixtures do not satisfy the current assistant-message contract.

## What This Changes

The PR adds post-turn CLI context-engine ingestion from session files and tests ingestion, no-budget, failure, and missing-`afterTurn` cases.

## Best Possible Solution

Keep the PR open and repair it by mirroring the harness/embedded post-turn lifecycle for CLI transcripts: call `afterTurn` when available, otherwise feed new messages through `ingestBatch` or `ingest`, while preserving current initialization and valid test fixtures.

## Reproduction Assessment

Yes. Source inspection on current main shows the CLI path persists a turn and immediately calls a lifecycle that only compacts and never feeds the completed turn into `afterTurn`, `ingestBatch`, or `ingest`.

## Solution Assessment

No, not as written. The direction is correct, but the patch needs the existing fallback contract for engines without `afterTurn`, valid assistant fixtures, and a rebase over current CLI compaction initialization changes.
