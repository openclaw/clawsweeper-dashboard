## Summary

Keep this PR open, but it still needs changes before merge: current main lacks CLI post-turn context-engine ingestion, and the proposed patch skips valid engines without `afterTurn` plus misses the required changelog entry.

## What This Changes

The PR adds CLI post-turn context-engine `afterTurn` ingestion before CLI compaction and unit coverage for normal, no-budget, throwing, and no-`afterTurn` cases.

## Best Possible Solution

Update the CLI lifecycle to mirror the embedded and harness post-turn contract: call `afterTurn` when available, otherwise ingest the just-appended turn through `ingestBatch` or per-message `ingest`, with focused tests and a changelog entry.

## Reproduction Assessment

Yes. Current main can be verified by reading `runCliTurnCompactionLifecycle`, and the PR blocker is reproducible with a valid `ContextEngine` that has required `ingest()` but no optional `afterTurn`.

## Solution Assessment

No. Calling `afterTurn` for CLI turns is the right direction, but the patch should preserve the documented fallback path for all valid context engines and record the user-facing fix in `CHANGELOG.md`.
