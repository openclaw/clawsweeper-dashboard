## Summary

Keep open: the PR addresses a real CLI context-engine lifecycle gap, but the proposed implementation skips valid engines that rely on the required `ingest()` contract when `afterTurn` is absent, and it is missing the required user-facing changelog entry.

## What This Changes

The PR branch adds CLI post-turn context-engine `afterTurn` ingestion before existing CLI compaction and expands `cli-compaction` tests for normal, no-budget, throwing, and no-`afterTurn` cases.

## Best Possible Solution

Update the CLI lifecycle to mirror the embedded and harness contract: call `afterTurn` when available, otherwise ingest the just-appended turn through `ingestBatch` or per-message `ingest`, and add a single-line release note.

## Reproduction Assessment

Yes. A focused unit reproduction can use a valid `ContextEngine` with required `ingest()` but no `afterTurn`; the proposed no-`afterTurn` path currently asserts no ingestion, demonstrating the remaining contract gap without a live gateway.

## Solution Assessment

No. Calling `afterTurn` for Lossless-Claw is the right direction, but the maintainable fix should preserve the documented fallback path for all valid context engines and include the required changelog entry.
