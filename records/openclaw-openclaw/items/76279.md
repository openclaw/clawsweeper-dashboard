## Summary

Keep open: the PR addresses a real CLI context-engine lifecycle gap, but the patch still skips valid engines without `afterTurn` and its new `SessionManager` fixtures fail test type checking.

## What This Changes

Adds CLI post-turn context-engine ingestion in `runCliTurnCompactionLifecycle` and expands `cli-compaction` tests for afterTurn, no-budget, thrown, and missing-afterTurn cases.

## Best Possible Solution

Mirror the harness and embedded lifecycle for CLI turns after transcript persistence: call `afterTurn` when available, otherwise feed new messages through `ingestBatch` or per-message `ingest`, while keeping ingestion independent of the compaction budget.

## Reproduction Assessment

Yes. Source inspection shows current main persists CLI transcripts and immediately runs `runCliTurnCompactionLifecycle`, but that lifecycle returns unless a token budget exists and never feeds the completed turn into `afterTurn`, `ingestBatch`, or `ingest`.

## Solution Assessment

No, not as written. The hook point is right, but the patch should preserve valid engines without `afterTurn` and repair the type-invalid test fixtures before merge.
