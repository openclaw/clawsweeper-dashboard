## Summary

Keep open: the reported CLI context-engine gap is source-backed on current main and this PR is useful, but the patch still skips valid engines without `afterTurn`, omits normal turn maintenance, has invalid assistant-message fixtures, and is currently merge-conflicting with main.

## What This Changes

The PR adds CLI post-turn context-engine ingestion before CLI compaction and expands CLI compaction tests around normal ingestion, no budget, thrown errors, and missing `afterTurn` behavior.

## Best Possible Solution

Wire successful CLI turns through the shared context-engine finalization pattern after transcript persistence: `afterTurn` when present, `ingestBatch`/`ingest` fallback otherwise, followed by turn maintenance without coupling ingestion to compaction budget.

## Reproduction Assessment

Yes. Source inspection shows current main persists CLI turns and then runs a CLI compaction lifecycle that never feeds ordinary completed turns into `afterTurn`, `ingestBatch`, or `ingest`; the PR body and the companion issue also provide live Lossless-Claw evidence.

## Solution Assessment

No, not as written. Calling the context engine from the CLI path is the right direction, but the patch should preserve valid engines without `afterTurn`, schedule normal turn maintenance, use valid assistant-message fixtures, and rebase onto current main behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Fall back to ingest when afterTurn is absent:** `src/agents/command/cli-compaction.ts:212`
  - body: `afterTurn` is optional while `ingest` is required by the context-engine contract. This branch silently skips engines that implement only `ingestBatch` or `ingest`, so CLI turns can still remain invisible to valid plugins; use the same `afterTurn` / `ingestBatch` / `ingest` fallback as the harness and embedded paths.
  - confidence: 0.93
- **[P2] Schedule turn maintenance after successful ingest:** `src/agents/command/cli-compaction.ts:213-223`
  - body: The shared harness finalizer runs `runContextEngineMaintenance({ reason: "turn" })` after a successful `afterTurn`/ingest path, but this patch only runs maintenance during compaction. Engines with maintenance-phase indexing or transcript cleanup still miss normal CLI turns unless a compaction happens.
  - confidence: 0.84
- **[P2] Use valid assistant messages in SessionManager fixtures:** `src/agents/command/cli-compaction.test.ts:190-194`
  - body: `SessionManager.appendMessage()` accepts the current `Message` contract, and assistant messages require `api`, `provider`, `model`, `usage`, and `stopReason`. These fixtures omit those fields, so `check-test-types` fails; build them with the existing assistant-message fixture helper or include the required fields.
  - confidence: 0.91
