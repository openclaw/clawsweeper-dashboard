## Summary

Keep open: current main still lacks CLI turn ingestion into the context engine, and this PR addresses a real bug but is not merge-ready. The patch only calls optional `afterTurn`, the new `SessionManager.appendMessage()` assistant fixtures fail the current message contract, and the branch is conflicting with later CLI compaction initialization/import changes.

## What This Changes

The PR adds post-transcript CLI context-engine ingestion and focused tests in the CLI compaction lifecycle.

## Best Possible Solution

Mirror the harness lifecycle in the CLI path: after transcript persistence, feed the completed turn through `afterTurn` when available, otherwise `ingestBatch` or per-message `ingest`, while preserving current initialization and compaction behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows CLI transcripts are persisted and then passed to a lifecycle that never feeds the completed turn into `afterTurn`, `ingestBatch`, or `ingest`.

## Solution Assessment

No, not as written. Calling the context engine from the CLI path is the right direction, but the patch must preserve engines without `afterTurn`, use valid assistant-message fixtures, and rebase over current initialization/import changes.
