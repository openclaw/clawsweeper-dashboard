## Summary

Keep open. Current main now covers threaded Google Chat replies for messages that reach OpenClaw, but the second requested capability is still missing: there is no optional user-OAuth, Workspace Events, or chat.messages delivery mode for non-mention space messages. Related #68967 is open and scoped to thread/session behavior, while #9764 remains the adjacent user-OAuth foundation tracker.

## What This Changes

The issue requests Google Chat replies to stay in the triggering thread and optionally receive all non-mention messages in spaces.

## Best Possible Solution

Add a Google Chat plugin-owned optional delivery/auth mode for non-mention space messages, preserving the current service-account webhook defaults and documenting the new scopes, credential storage, setup, and fail-closed behavior.
