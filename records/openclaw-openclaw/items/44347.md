## Summary

Keep this open. Current main and v2026.6.6 already cover the low-level Google Chat thread send primitive, but the issue still has a distinct request to receive non-mention space messages, and current Google Chat config/auth remains service-account-only with no delegated user OAuth or approved message-visibility design.

## What This Changes

The issue requests Google Chat replies to stay in the originating thread and an optional way to receive all messages in spaces instead of only bot mentions.

## Best Possible Solution

Keep this issue open until maintainers either split the all-messages/OAuth work into a precise canonical issue or approve a plugin-owned Google Chat delegated-auth design that covers non-mention space delivery safely.

## Reproduction Assessment

Yes at source level, but not as a live Google Chat repro in this read-only review. Current main and v2026.6.6 still have only service-account Google Chat credentials and no delegated OAuth or exact all-space-message delivery mode.

## Solution Assessment

No, this is not ready for an automated fix as written because it combines a partly tracked threading fix with a security-sensitive OAuth/message-visibility feature. The best path is maintainer-approved delegated-auth design or a narrower canonical split before implementation.
