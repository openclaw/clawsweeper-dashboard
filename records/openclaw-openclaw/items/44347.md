## Summary

Keep open. Current main handles threaded replies for delivered Google Chat messages, but this issue’s all-space/non-mention receive mode still has no user OAuth, Workspace Events, config, schema, setup, or docs path; https://github.com/openclaw/openclaw/issues/9764 is related OAuth groundwork and https://github.com/openclaw/openclaw/pull/68967 is thread/session work, not the remaining delivery design.

## What This Changes

Request Google Chat support for replying in inbound threads and optionally receiving all non-mention messages in spaces.

## Best Possible Solution

Keep the remaining work as a Google Chat plugin-owned opt-in all-message delivery mode, coordinated with https://github.com/openclaw/openclaw/issues/9764 where useful, while preserving the existing service-account and mention-gated defaults.

## Reproduction Assessment

Yes at source level: current main shows delivered messages carry thread metadata into replies, while auth, config/schema, setup, and docs expose no user-OAuth, Workspace Events, or all-space receive mode. I did not run a live Google Workspace reproduction in this read-only review.

## Solution Assessment

No, current main is not the full solution for this issue. The maintainable path is a Google Chat plugin-owned auth and event-delivery design, not closing after the threaded-reply half or adding a core workaround.
