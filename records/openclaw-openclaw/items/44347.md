Keep open. Current main covers threaded delivery for Google Chat messages that are already delivered to OpenClaw, but the requested non-mention/all-space receive mode is still missing and needs auth, event-delivery, credential, and Workspace security decisions.

## What This Changes

Request Google Chat parity by replying in source threads and optionally receiving non-mention messages in spaces.

## Best Possible Solution

Keep the shipped threaded reply behavior and design a Google Chat plugin-owned, opt-in all-message space delivery mode with explicit OAuth or Workspace Events credentials, safe service-account defaults, and docs.

## Reproduction Assessment

Yes, at source level. Current main has delivered-message thread metadata and threaded sends, but there is no accepted OAuth/event-delivery config or credential path for non-mention space messages.

## Solution Assessment

Unclear as an implementation because no patch is proposed for the remaining receive-all behavior. The maintainable direction is a Google Chat plugin-owned OAuth or Workspace Events design, not a core workaround or cleanup close.
