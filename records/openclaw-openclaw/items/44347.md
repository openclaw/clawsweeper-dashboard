## Summary

Keep open. Current main covers the normal Google Chat threaded-reply path for messages that are delivered to OpenClaw, but the requested opt-in receive-all/non-mention space mode is still missing and overlaps credential, event-delivery, and Workspace security design that should not be closed as cleanup.

## What This Changes

Requests Google Chat replies to stay in the triggering thread and an optional way to receive non-mention messages in spaces.

## Best Possible Solution

Keep this open as Google Chat plugin-owned work for an opt-in all-message space delivery mode, coordinated with user OAuth where useful, while preserving service-account and mention-gated defaults.

## Reproduction Assessment

Yes at source level: current main carries delivered Google Chat thread metadata into replies, but auth, config/schema, setup, and docs expose no all-space/non-mention receive mode. I did not run a live Google Workspace reproduction in this read-only review.

## Solution Assessment

Unclear as an implementation because no patch is proposed for the remaining half. The maintainable direction is a plugin-owned auth and event-delivery design, not a core workaround or closure against only the completed thread-reply path.
