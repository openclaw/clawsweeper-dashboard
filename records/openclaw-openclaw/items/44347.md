## Summary

Keep this open: current main and v2026.6.6 already handle Google Chat outbound replies into the inbound thread, but the remaining request to receive non-mention space messages is still missing and needs a security-approved OAuth or Workspace Events design.

## What This Changes

Request threaded Google Chat replies and an optional way for the Google Chat channel to receive all space messages, not only messages that invoke or mention the bot.

## Best Possible Solution

Keep this as the broader Google Chat product/security tracking item: preserve shipped threaded replies, review the sessionThread PR separately, and design plugin-owned Workspace Events or delegated-OAuth ingestion with explicit scopes, consent, allowlists, retention, and docs.
