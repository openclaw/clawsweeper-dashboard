## Summary

Current main now implements same-thread Google Chat replies, but the second requested capability—receiving non-mention messages in spaces—remains unresolved and needs a deliberate Google authentication and event-delivery decision. The open threading PR and user-OAuth issue are relevant, but neither fully resolves this combined issue.

## What This Changes

Add same-thread Google Chat replies and an optional way to receive all messages in spaces without requiring an @mention.

## Best Possible Solution

Keep the current plugin-local threaded-send behavior, then define one canonical Google Chat design for user OAuth and non-mention event delivery with explicit scopes, consent, credential handling, privacy behavior, and live API proof.

## Reproduction Assessment

Yes at source level. Threaded sending is now present on current main, while the current service-account configuration and webhook surface still provide no implemented receive-all or non-mention mode.

## Solution Assessment

Partly. The current threaded-send implementation is the narrow adapter-level solution for the first request, but the second request needs a shared OAuth and event-subscription design rather than assuming a new token alone will change inbound delivery.
