## Summary

Keep open. Current main handles threaded replies for Google Chat messages that are delivered, but it still lacks an opt-in all-space/non-mention receive mode; the related #9764 is OAuth groundwork, and #68967 addresses thread/session behavior rather than the remaining auth and delivery design.

## What This Changes

Request Google Chat threaded replies and an optional mode to receive all messages in spaces without @mentioning the bot.

## Best Possible Solution

Keep the remaining work in the Google Chat plugin as an opt-in all-message delivery mode, coordinated with #9764, while preserving the current service-account and mention-gated defaults.

## Reproduction Assessment

Yes at source level: current main shows threaded reply routing is wired, while auth, config/schema, webhook setup, and docs expose no user-OAuth, Workspace Events, or all-space receive mode. A live Google Chat run was not established in this read-only review.

## Solution Assessment

No, current main is not the full solution for this issue. The narrow maintainable path is a Google Chat plugin-owned opt-in auth and event delivery design, not a core workaround or closing after the threaded-reply half.
