## Summary

Keep open. Current main still lacks a Google Chat receive-all space-message mode or user OAuth/Workspace Events auth path, and the thread-placement work is only partially present on main with related PRs still open.

## What This Changes

Google Chat should keep bot replies in the source thread and optionally ingest non-mention messages from spaces.

## Best Possible Solution

Review and land or replace the focused Google Chat thread-routing work, then add a plugin-owned opt-in receive-all mode using user OAuth or Workspace Events with service-account defaults preserved, credential-aware docs, tests, and live Workspace proof.

## Reproduction Assessment

Yes, at source level. Current main still has service-account-only Google Chat auth/config with no user OAuth or Workspace Events receive-all mode; the thread side is only partially covered and remains tied to open PR work.

## Solution Assessment

Unclear as one combined change. The maintainable path is to split thread placement from receive-all delivery and treat receive-all as an explicit opt-in auth/event-delivery feature.
