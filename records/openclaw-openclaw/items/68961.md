- **[P1] Forward messageProvider through the dispatch params:** `src/agents/command/attempt-execution.ts:365`
  - body: The new `embeddedParams` object drops `messageProvider`, while current main passes `params.opts.messageProvider ?? params.messageChannel` into the embedded runner. Existing embedded runs can lose provider-specific message/tool routing context as soon as they go through `runAgent`.

  - body: When `extraSystemPrompt` is absent, `systemPrompt` is omitted and `settingSources` is never set. The SDK treats that as an empty custom prompt with `settingSources: []`, so normal claude-sdk runs skip project settings and CLAUDE.md policy.
  - confidence: 0.93
- **[P1] Apply toolsAllow to OpenClaw MCP tools:** `src/agents/claude-sdk/run.ts:499-501`
  - body: `toolsAllow` is copied into the SDK `tools` option, which controls SDK built-ins, while the OpenClaw MCP inventory is built without `runtimeToolAllowlist`. Restricted runs can expose native tools outside the requested allow-list or fail to expose intended namespaced MCP tools.
  - confidence: 0.9
- **[P1] Pass memberRoleIds into native tool inventory:** `src/agents/claude-sdk/run.ts:253-257`
  - body: The native-tool bridge forwards group id/channel/space context but omits `memberRoleIds`. Group runs under claude-sdk can pick different requester-origin or role-scoped permissions than the embedded runtime.
  - confidence: 0.91
- **[P1] Normalize OpenClaw deny IDs for SDK built-ins:** `src/agents/claude-sdk/sdk-tool-policy.ts:56-61`
  - body: The deny collector trims entries but keeps raw OpenClaw names such as `exec`; the SDK built-in is named `Bash`. A policy like `tools.deny: ["exec"]` can still leave SDK Bash available under claude-sdk.
  - confidence: 0.92
- **[P1] Report failed native tool results as MCP errors:** `src/agents/claude-sdk/native-tools-adapter.ts:203`
  - body: `agentResultToMcp` always returns `isError: false`, even for OpenClaw tool results that encode failure in metadata. The SDK model will see failed native tool executions as successful and may continue from invalid state.
  - confidence: 0.9
- **[P1] Dispatch ACP runtime explicitly:** `src/agents/runtime-dispatch.ts:54`
  - body: `runAgent` maps every non-claude-sdk selection to `runEmbeddedPiAgent`, and the selector treats `runtime.type: "acp"` as that default bucket. Callers using the new public dispatch API with an ACP-configured agent will bypass ACP-specific session and policy behavior.
  - confidence: 0.86
- **[P2] Resolve workspace path before passing SDK cwd:** `src/agents/claude-sdk/run.ts:554`
  - body: The SDK query receives `params.workspaceDir` directly, but the embedded runner first resolves and falls back to the canonical workspace. Persisted sessions with moved or deleted workspaces can fail early instead of using the established fallback path.
  - confidence: 0.78
- **[P2] Skip partial assistant deltas in the mirror:** `src/agents/claude-sdk/session-mirror.ts:230`
  - body: `projectSdkMessage` treats `partial_assistant` like a final assistant message, so streaming deltas can be persisted as standalone assistant turns. That can feed incomplete duplicate content back into transcript/history consumers.
  - confidence: 0.82
- **[P2] Preserve the actual stop reason in mirrored records:** `src/agents/claude-sdk/session-mirror.ts:257`

  - confidence: 0.77
