## Summary

Keep this PR open: current main still does not have the claude-sdk runtime surface, but the PR is not mergeable as-is because the new runtime bridge has tool-policy/security regressions and no real after-fix SDK proof.

## What This Changes

The PR adds an opt-in Claude Agent SDK runtime, native MCP tool bridge, runtime dispatch/API/config/docs/tests, generated Claude sub-agent files, and dependency/lockfile updates while the current selector keeps embedded as the default.

## Best Possible Solution

Land a smaller opt-in claude-sdk runtime only after it preserves embedded tool-policy, ACP, session, and settings contracts and includes redacted live SDK plus native-tool proof; defer any default flip to a maintainer-approved soak.

## Reproduction Assessment

Yes for the blocking PR defects: source inspection of PR head shows omitted `toolsAllow`/`memberRoleIds`, verbatim deny aliases, omitted `settingSources`, and ACP routing collapsed to embedded. No live runtime proof is provided for the new claude-sdk behavior.

## Solution Assessment

No. The direction is plausible, but this implementation is not the narrowest maintainable solution until it preserves existing runtime policy contracts, ACP semantics, SDK settings behavior, and proves a real SDK/native-tool run.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Filter native MCP tools by toolsAllow:** `src/agents/claude-sdk/run.ts:233-267`
  - body: `toolsAllow` is the restricted-run allow-list used by cron and isolated runs, but this builds the native MCP server from the full OpenClaw tool inventory and only forwards `toolsAllow` to SDK built-ins. A restricted run can still expose native message, cron, sessions, or plugin tools.
  - confidence: 0.93
- **[P1] Pass memberRoleIds into native tool inventory:** `src/agents/claude-sdk/run.ts:233-267`
  - body: The embedded runtime passes `memberRoleIds` into `createOpenClawCodingTools()`, but the claude-sdk bridge omits it. Group-channel requests can lose role-scoped origin and policy information under the new runtime.
  - confidence: 0.88
- **[P1] Normalize deny aliases for SDK built-ins:** `src/agents/claude-sdk/sdk-tool-policy.ts:56-61`
  - body: This helper forwards deny entries verbatim to `disallowedTools`, but OpenClaw policy commonly uses canonical names such as `exec`, `read`, and `edit` while the SDK built-ins are `Bash`, `Read`, and `Edit`. With `tools.deny: ["exec"]`, SDK Bash can remain available.
  - confidence: 0.9
- **[P1] Load project settings when invoking the SDK:** `src/agents/claude-sdk/run.ts:550-563`
  - body: The SDK call omits `settingSources`; the SDK contract makes that isolation mode with no filesystem settings loaded, and it requires `project` to load CLAUDE.md. Normal claude-sdk turns can therefore ignore project instructions and permission settings.
  - confidence: 0.87
- **[P2] Return MCP errors for failed native tool results:** `src/agents/claude-sdk/native-tools-adapter.ts:203`
  - body: `agentResultToMcp` marks every non-throwing OpenClaw tool result as successful. Tools can report failure in structured result metadata, so Claude may continue as if a failed native tool call succeeded.
  - confidence: 0.86
- **[P2] Dispatch ACP runtime entries through ACP:** `src/agents/runtime-dispatch.ts:46-54`
  - body: `runAgent` is documented and exported as a runtime dispatch across claude-sdk, embedded, and ACP, but `selectAgentRuntime` collapses `runtime.type: "acp"` into the default branch and this wrapper calls the embedded runner. Public dispatch callers using ACP-configured agents bypass the ACP path.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: The PR introduces concrete tool-policy and requester-context bypass risks in the new SDK runtime bridge.

Concerns:

- **[high] Runtime tool allow-list bypass:** `src/agents/claude-sdk/run.ts:233`
  - body: The claude-sdk native MCP bridge builds and exposes the full OpenClaw tool inventory without applying `params.toolsAllow`, so restricted cron or isolated runs can receive tools outside the intended allow-list.
  - confidence: 0.93
- **[high] SDK built-in deny alias bypass:** `src/agents/claude-sdk/sdk-tool-policy.ts:56`
  - body: Deny entries are passed to `disallowedTools` without mapping OpenClaw canonical names like `exec` to SDK built-ins like `Bash`, leaving command execution available despite an OpenClaw deny rule.
  - confidence: 0.9
- **[medium] Role-scoped policy context is dropped:** `src/agents/claude-sdk/run.ts:233`
  - body: The native tool inventory omits `memberRoleIds`, so role-based routing or tool restrictions in group channels can be evaluated with weaker requester context than the embedded runtime.
  - confidence: 0.88
- **[medium] Project settings and CLAUDE.md are skipped:** `src/agents/claude-sdk/run.ts:550`
  - body: The SDK invocation omits `settingSources`, so project CLAUDE.md and `.claude` settings are not loaded; that can bypass repository-level instructions or permission settings the runtime depends on.
  - confidence: 0.86
