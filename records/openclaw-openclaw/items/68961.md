Full review comments:

- **[P1] Load project settings for SDK turns:** `src/agents/claude-sdk/run.ts:558`
  - body: When `extraSystemPrompt` is absent, this leaves `systemPrompt` undefined and never sets `settingSources`. The SDK contract says omitted settings load no filesystem settings, so normal OpenClaw SDK turns skip the Claude Code preset, CLAUDE.md, and project/user policy that workspaces can depend on.
  - confidence: 0.91
- **[P1] Apply toolsAllow before registering MCP tools:** `src/agents/claude-sdk/run.ts:233-267`
  - body: `toolsAllow` is only forwarded to the SDK built-in `tools` option, while the OpenClaw MCP inventory is built without `runtimeToolAllowlist` or the embedded construction plan. A constrained run can therefore still expose native OpenClaw tools that the caller intended to withhold.
  - confidence: 0.9
- **[P1] Normalize deny names for SDK built-ins:** `src/agents/claude-sdk/sdk-tool-policy.ts:56-61`
  - body: This copies deny entries verbatim into `disallowedTools`, but existing OpenClaw policy uses names like `exec` while the SDK built-in is `Bash`. A config such as `tools.deny: ["exec"]` still leaves command execution available under the SDK path.
  - confidence: 0.88
- **[P1] Preserve requester context for native tools:** `src/agents/claude-sdk/run.ts:246-267`
  - body: The native-tool builder uses `params.messageProvider` without falling back to `messageChannel` and does not pass `memberRoleIds` into `createOpenClawCodingTools`. That diverges from the embedded path and can break channel-specific tools, role-scoped routing, and group policy checks.
  - confidence: 0.87
- **[P1] Mark failed tool results as MCP errors:** `src/agents/claude-sdk/native-tools-adapter.ts:203`
  - body: `agentResultToMcp` always returns `isError: false`, so OpenClaw tools that report structured failures without throwing are presented to Claude as successful tool calls. The adapter needs to derive the MCP error flag from the tool result status as well as exceptions.
  - confidence: 0.86
- **[P2] Skip partial assistant messages in the mirror:** `src/agents/claude-sdk/session-mirror.ts:230-257`
  - body: `projectSdkMessage` persists `partial_assistant` frames as canonical assistant turns. Streaming deltas can then become duplicate incomplete transcript messages, which pollutes downstream history and context consumers.
  - confidence: 0.78
