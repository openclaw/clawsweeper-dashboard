Full review comments:

- **[P1] Load Claude Code preset and project settings by default:** `src/agents/claude-sdk/run.ts:558-563`
  - body: When `extraSystemPrompt` is absent, `systemPrompt` is omitted and `settingSources` is never set. The SDK 0.1.77 source turns that into an empty custom prompt and `settingSources: []`, so normal claude-sdk runs skip the Claude Code preset plus project `CLAUDE.md`/settings. That drops project permission and convention policy for opted-in agents.
  - confidence: 0.93
- **[P1] Apply toolsAllow to OpenClaw MCP tools:** `src/agents/claude-sdk/run.ts:499-501`
  - body: `toolsAllow` is copied into the SDK `tools` option, which the SDK defines as the built-in tool set, while the OpenClaw MCP inventory is built without `runtimeToolAllowlist`. Restricted cron or agent runs can therefore expose native `message`/`cron`/`sessions_*` tools outside the requested allow-list, or fail to expose the intended namespaced MCP tool names.
  - confidence: 0.9
- **[P1] Pass memberRoleIds into native tool inventory:** `src/agents/claude-sdk/run.ts:253-257`
  - body: The native-tool bridge forwards group id/channel/space context but omits `memberRoleIds`. Current OpenClaw tools use those role IDs for requester-origin and role-scoped routing, so group runs under claude-sdk can pick different accounts or permissions than the embedded runtime.
  - confidence: 0.91
- **[P1] Normalize OpenClaw deny IDs for SDK built-ins:** `src/agents/claude-sdk/sdk-tool-policy.ts:56-61`
  - body: The deny collector trims entries but keeps raw OpenClaw names such as `exec`; the SDK built-in is named `Bash`. A policy like `tools.deny: ["exec"]` can still leave SDK Bash available under claude-sdk, which is a command-execution policy bypass.
  - confidence: 0.92
- **[P1] Report failed native tool results as MCP errors:** `src/agents/claude-sdk/native-tools-adapter.ts:203`
  - body: `agentResultToMcp` always returns `isError: false`, even for OpenClaw tool results that encode failure in `details.status: "failed"` or similar metadata. The SDK model will see failed native tool executions as successful and may continue from invalid state instead of recovering.
  - confidence: 0.9
- **[P1] Dispatch ACP runtime explicitly:** `src/agents/runtime-dispatch.ts:50-54`
  - body: `runAgent` maps every non-claude-sdk selection to `runEmbeddedPiAgent`, and the selector treats `runtime.type: "acp"` as that default bucket. Callers using the new public dispatch API with an ACP-configured agent will bypass ACP-specific session and policy behavior.
  - confidence: 0.86
