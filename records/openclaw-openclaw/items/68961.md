- **[P1] Preserve messageProvider in runtime dispatch:** `src/agents/command/attempt-execution.ts:360-366`
  - body: The new embeddedParams object drops messageProvider, while current main passes params.opts.messageProvider ?? params.messageChannel into runEmbeddedPiAgent. Provider-specific channel runs can lose the correct provider after this rewrite.
  - confidence: 0.92
- **[P1] Filter native MCP tools by toolsAllow:** `src/agents/claude-sdk/run.ts:233-267`
  - body: toolsAllow is only forwarded to SDK built-ins; the native MCP server is built from the full OpenClaw tool inventory. Restricted cron or isolated runs can still expose message, cron, sessions, or plugin tools outside the runtime allow-list.
  - confidence: 0.94
- **[P1] Pass memberRoleIds into native tool inventory:** `src/agents/claude-sdk/run.ts:233-267`
  - body: The embedded runtime carries requester role IDs into createOpenClawCodingTools(), but the claude-sdk bridge omits them. Group-channel runs can lose role-scoped origin and policy decisions under the new runtime.
  - confidence: 0.88
- **[P1] Normalize SDK built-in deny aliases:** `src/agents/claude-sdk/sdk-tool-policy.ts:56-61`
  - body: This forwards deny entries verbatim to disallowedTools, but OpenClaw policies commonly use canonical names such as exec, read, and edit while the SDK built-ins are Bash, Read, and Edit. A config that denies exec can still leave SDK Bash available.
  - confidence: 0.9
- **[P1] Load project settings for SDK runs:** `src/agents/claude-sdk/run.ts:550-564`
  - body: The SDK call omits settingSources; upstream SDK documentation says filesystem settings and CLAUDE.md are not loaded by default. Normal claude-sdk turns can ignore project instructions and permission settings.
  - confidence: 0.86
- **[P2] Use the Claude Code preset for normal SDK turns:** `src/agents/claude-sdk/run.ts:490-496`
  - body: When extraSystemPrompt is absent, systemPrompt is undefined, so the SDK uses no Claude Code preset. That changes the expected instruction/tooling baseline for the runtime this PR is trying to introduce.
  - confidence: 0.78
- **[P2] Resolve the workspace before passing SDK cwd:** `src/agents/claude-sdk/run.ts:554`
  - body: The SDK path passes params.workspaceDir directly as cwd, while the embedded runtime resolves and falls back to the canonical workspace. Long-lived sessions with moved or missing workspace paths can fail instead of recovering.
  - confidence: 0.8
- **[P2] Return MCP errors for failed native tool results:** `src/agents/claude-sdk/native-tools-adapter.ts:203`
  - body: agentResultToMcp marks every non-throwing OpenClaw tool result as successful. Tools can report failure through structured result metadata, so Claude may continue as if a failed native call succeeded.
  - confidence: 0.84
- **[P2] Skip partial assistant deltas when mirroring:** `src/agents/claude-sdk/session-mirror.ts:230`
  - body: projectSdkMessage persists partial_assistant frames the same way as final assistant messages. Streaming deltas can become duplicate incomplete assistant turns in the canonical transcript.
  - confidence: 0.76
- **[P2] Preserve real stop reasons in mirrored messages:** `src/agents/claude-sdk/session-mirror.ts:250-257`

  - confidence: 0.74
