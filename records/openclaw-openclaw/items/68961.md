Adds a Claude Agent SDK runtime driver, OpenClaw native MCP tool bridge, runtime dispatch/API/config/schema/docs/tests, generated Claude sub-agent files, and dependency/lockfile updates.

## Best Possible Solution

Land this only as an explicit opt-in runtime after it preserves embedded allow/deny semantics, Claude Code settings/prompt behavior, transcript/tool error semantics, and has targeted plus live smoke proof; defer any default flip.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Filter native MCP tools by toolsAllow:** `src/agents/claude-sdk/run.ts:233-267`
  - body: `toolsAllow` is the runtime allow-list used by cron and isolated runs, but the PR builds the native MCP server from the full `createOpenClawCodingTools()` inventory and only passes `toolsAllow` to SDK built-ins. A run restricted to `read` can still expose OpenClaw tools such as message, cron, or sessions tools.
  - confidence: 0.93
- **[P1] Pass memberRoleIds into native tool inventory:** `src/agents/claude-sdk/run.ts:233-267`
  - body: The embedded runtime passes `memberRoleIds` into `createOpenClawCodingTools()`, but this bridge omits it. Group-channel requests can therefore lose role-scoped origin/policy information under claude-sdk and behave differently from embedded runs.
  - confidence: 0.88
- **[P1] Normalize OpenClaw deny aliases for SDK built-ins:** `src/agents/claude-sdk/sdk-tool-policy.ts:56-61`
  - body: The helper forwards deny entries verbatim to `disallowedTools`, but OpenClaw policy commonly uses IDs such as `exec`, `read`, and `edit` while Claude SDK built-ins are `Bash`, `Read`, and `Edit`. With `tools.deny: ["exec"]`, SDK Bash can remain available.
  - confidence: 0.9
- **[P1] Load project settings for SDK runs:** `src/agents/claude-sdk/run.ts:550-564`
  - body: The SDK options never set `settingSources`, and the SDK defaults that field to an empty list. That means claude-sdk runs ignore project `.claude` settings and CLAUDE.md instructions unless the adapter explicitly includes the project setting source.
  - confidence: 0.86
- **[P2] Always pass the Claude Code preset:** `src/agents/claude-sdk/run.ts:490-496`
  - body: When `extraSystemPrompt` is absent, `systemPromptOption` is `undefined`, so ordinary runs use the SDK's minimal prompt instead of the Claude Code preset. Pass the `claude_code` preset by default and add `append` only when extra prompt text exists.
  - confidence: 0.88
- **[P2] Return MCP errors for failed native tool results:** `src/agents/claude-sdk/native-tools-adapter.ts:183-203`
  - body: `agentResultToMcp` marks every non-throwing OpenClaw tool result as successful. OpenClaw tools can report failure via structured result metadata without throwing, so Claude may continue as if a failed native tool call succeeded.
  - confidence: 0.86
- **[P2] Dispatch ACP runtime entries through ACP:** `src/agents/runtime-dispatch.ts:46-55`
  - body: `runAgent` is documented as dispatching across claude-sdk, embedded, and ACP, but `selectAgentRuntime` collapses `runtime.type: "acp"` into the default branch and `runAgent` then calls the embedded runner. Callers using the new public dispatch with an ACP-configured agent would bypass the ACP path.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: Security review needs attention because the new SDK runtime bridge introduces concrete command/tool policy bypasses.

Concerns:

- **[high] Runtime tool allow-list bypass:** `src/agents/claude-sdk/run.ts:233`
  - body: The claude-sdk native MCP bridge builds and exposes the full OpenClaw tool inventory without applying `params.toolsAllow`, so restricted cron or isolated runs can receive tools outside the intended allow-list.
  - confidence: 0.93
- **[high] SDK built-in deny alias bypass:** `src/agents/claude-sdk/sdk-tool-policy.ts:56`
  - body: Deny entries are passed to `disallowedTools` without mapping OpenClaw canonical names like `exec` to SDK built-ins like `Bash`, leaving command execution available despite an OpenClaw deny rule.
  - confidence: 0.9
- **[medium] Project settings are skipped:** `src/agents/claude-sdk/run.ts:550`
  - body: The SDK invocation omits `settingSources`, so project CLAUDE.md and `.claude` settings are not loaded; that can bypass repository-level instructions or permission settings the runtime depends on.
  - confidence: 0.86
