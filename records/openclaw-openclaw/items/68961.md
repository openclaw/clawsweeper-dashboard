## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Apply toolsAllow before exposing native MCP tools:** `src/agents/claude-sdk/run.ts:233-267`
  - body: `toolsAllow` is used by cron and isolated runs to restrict the runtime tool surface, but this builds the OpenClaw MCP server from the full tool inventory and only passes `toolsAllow` to SDK built-ins. A constrained run can still expose native message, cron, sessions, or plugin tools outside its allow-list.
  - confidence: 0.93
- **[P1] Pass memberRoleIds into native tool inventory:** `src/agents/claude-sdk/run.ts:233-267`
  - body: The embedded runtime passes `memberRoleIds` into `createOpenClawCodingTools()`, but the claude-sdk bridge omits it. Group-channel requests can lose role-scoped origin and policy context under this runtime.
  - confidence: 0.88
- **[P1] Normalize OpenClaw deny aliases for SDK built-ins:** `src/agents/claude-sdk/sdk-tool-policy.ts:56-61`
  - body: This helper forwards deny entries verbatim to `disallowedTools`, but OpenClaw policy commonly uses IDs such as `exec`, `read`, and `edit` while the SDK built-ins are `Bash`, `Read`, and `Edit`. A deny rule like `tools.deny: ["exec"]` can leave SDK Bash available.
  - confidence: 0.9
- **[P1] Load the Claude Code preset and project settings:** `src/agents/claude-sdk/run.ts:550-564`
  - body: When `extraSystemPrompt` is absent, `systemPrompt` is undefined, and the query options never set `settingSources`. The SDK contract says omitted settings sources load no filesystem settings, so normal runs skip project settings and CLAUDE.md instructions.
  - confidence: 0.87
- **[P2] Return MCP errors for failed native tool results:** `src/agents/claude-sdk/native-tools-adapter.ts:203`
  - body: `agentResultToMcp()` returns `isError: false` for every non-throwing OpenClaw tool result. Tools that report structured failure metadata can be presented to Claude as successful tool calls, which breaks the model's error-recovery path.
  - confidence: 0.86
- **[P2] Dispatch ACP runtime entries through ACP:** `src/agents/runtime-dispatch.ts:54`
  - body: `runAgent` is presented as the stable runtime dispatch API, but `runtime.type: "acp"` falls into the `default` bucket and this wrapper calls `runEmbeddedPiAgent()`. Plugin callers using the new public dispatch with ACP-configured agents would bypass ACP behavior.
  - confidence: 0.8

  - confidence: 0.9
