## Summary

Keep this PR open: the claude-sdk runtime is not implemented on current main, but the proposed patch is not merge-ready because source review still finds tool-policy, requester-context, ACP dispatch, and transcript-mirroring regressions, and the external PR lacks real behavior proof.

## What This Changes

The PR adds a Claude Agent SDK runtime driver, runtime dispatch/config/plugin-SDK/docs/tests, generated `.claude/agents` files, and dependency/lockfile updates.

## Best Possible Solution

Land a smaller opt-in claude-sdk runtime only after it preserves embedded ACP/tool-policy/session contracts, adds targeted regression coverage, includes live SDK/native-tool proof, and defers any default flip to a maintainer-approved soak.

## Reproduction Assessment

Yes for the blocking PR defects by source inspection: the PR builds native tools without `toolsAllow`/`memberRoleIds`, passes SDK deny names verbatim, bypasses ACP dispatch, and mirrors partial/final state incorrectly. No live claude-sdk behavior proof is provided.

## Solution Assessment

No. The direction is plausible, but the current patch is not the narrowest maintainable solution until it preserves existing tool-policy, requester-context, ACP, and transcript contracts and proves a real SDK run.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Filter native MCP tools by toolsAllow:** `src/agents/claude-sdk/run.ts:233-267`
  - body: `toolsAllow` is the runtime allow-list used by cron and isolated runs, but this builds the native MCP server from the full OpenClaw tool inventory and only forwards the allow-list to SDK built-ins. A restricted run can still expose native message, cron, sessions, or plugin tools outside the intended policy.
  - confidence: 0.93
- **[P1] Pass memberRoleIds into native tool inventory:** `src/agents/claude-sdk/run.ts:233-267`
  - body: The embedded runtime passes `memberRoleIds` into `createOpenClawCodingTools()`, but this bridge omits it. Group-channel requests can lose role-scoped origin and policy information under claude-sdk, producing different behavior from embedded runs.
  - confidence: 0.88
- **[P1] Normalize OpenClaw deny aliases for SDK built-ins:** `src/agents/claude-sdk/sdk-tool-policy.ts:56-61`
  - body: This helper forwards deny entries verbatim to `disallowedTools`, but OpenClaw policy commonly uses names such as `exec`, `read`, and `edit` while the Claude SDK built-ins are `Bash`, `Read`, and `Edit`. With `tools.deny: ["exec"]`, SDK Bash can remain available.
  - confidence: 0.9
- **[P2] Return MCP errors for failed native tool results:** `src/agents/claude-sdk/native-tools-adapter.ts:203`
  - body: `agentResultToMcp` marks every non-throwing OpenClaw tool result as successful. Tools can report failure in structured result metadata, so Claude may continue as if a failed native tool call succeeded.
  - confidence: 0.86
- **[P2] Dispatch ACP runtime entries through ACP:** `src/agents/runtime-dispatch.ts:54`
  - body: `runAgent` is documented as dispatching across claude-sdk, embedded, and ACP, but `selectAgentRuntime` collapses `runtime.type: "acp"` into the default branch and this wrapper then calls the embedded runner. Callers using the new public dispatch with ACP-configured agents would bypass the ACP path.
  - confidence: 0.84
- **[P3] Stop documenting a default flip that is not implemented:** `src/extensionAPI.ts:32-36`
  - body: The public compatibility export says `runAgent` is preferred after a Phase 3 default flip and that claude-sdk is the default, but the selector keeps the legacy path as default. Plugin authors reading this surface can migrate expecting behavior the code does not provide.
  - confidence: 0.82
- **[P2] Do not mirror partial assistant frames as final turns:** `src/agents/claude-sdk/session-mirror.ts:230`
  - body: `projectSdkMessage()` treats `partial_assistant` the same as finalized `assistant` frames, so streaming deltas can be written as standalone canonical assistant messages. That can duplicate incomplete content into history and downstream context consumers.
  - confidence: 0.84
- **[P2] Preserve the actual stop reason in mirrored assistant records:** `src/agents/claude-sdk/session-mirror.ts:257`
  - body: Mirrored assistant records are written with `stopReason: "stop"` before the run's terminal result is known. If the SDK run later aborts, times out, or errors after emitting content, transcript consumers will record that assistant turn as successful.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: The patch introduces concrete tool-policy and requester-context bypasses in the new SDK runtime bridge.

Concerns:

- **[high] Runtime tool allow-list bypass:** `src/agents/claude-sdk/run.ts:233`
  - body: The claude-sdk native MCP bridge builds and exposes the full OpenClaw tool inventory without applying `params.toolsAllow`, so restricted cron or isolated runs can receive tools outside the intended allow-list.
  - confidence: 0.93
- **[high] SDK built-in deny alias bypass:** `src/agents/claude-sdk/sdk-tool-policy.ts:56`
  - body: Deny entries are passed to `disallowedTools` without mapping OpenClaw canonical names like `exec` to SDK built-ins like `Bash`, leaving command execution available despite an OpenClaw deny rule.
  - confidence: 0.9
- **[medium] Role-scoped policy context is dropped:** `src/agents/claude-sdk/run.ts:233`
  - body: The native tool inventory omits `memberRoleIds`, so role-based routing or restrictions in group channels can be evaluated with weaker requester context than the embedded runtime.
  - confidence: 0.88
- **[medium] Failed native tools are reported as success:** `src/agents/claude-sdk/native-tools-adapter.ts:203`
  - body: The MCP bridge maps every non-throwing native tool result to `isError: false`, which can hide blocked or failed tool outcomes from the SDK's error-recovery path.
  - confidence: 0.86
