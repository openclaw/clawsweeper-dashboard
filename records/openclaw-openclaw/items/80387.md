## Summary

Keep open: current main still selects an owning npm from the package prefix without excluding Homebrew Cellar or other per-Node prefixes, and the linked closing PR is still open rather than merged.

## What This Changes

The issue requests that self-update avoid reinstalling OpenClaw into Homebrew Cellar or other per-Node version-manager prefixes so the CLI survives Node upgrades.

## Best Possible Solution

Land or replace the linked PR with a focused guard in `src/infra/update-global.ts` plus regression tests for ephemeral Homebrew Cellar and Node version-manager layouts while preserving stable-prefix behavior.

## Reproduction Assessment

Yes: this is high-confidence source-reproducible because current main derives `<prefix>/bin/npm` from the package root and lacks an ephemeral-prefix guard, matching the reporter's Homebrew Cellar evidence. I did not live-run a destructive `brew upgrade` repro in this checkout.

## Solution Assessment

Yes: the narrow maintainable direction is to reject known per-Node ephemeral prefixes before choosing the owning npm, then keep stable global prefixes unchanged. The existing linked PR appears to target that boundary, so the issue should remain open for review and landing of that fix.
