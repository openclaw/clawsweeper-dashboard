## Summary

Keep open: current main still lacks the ephemeral per-Node install-prefix guard, and the linked implementation PR is open with closing syntax rather than merged, so this issue should remain available for that review/landing path.

## What This Changes

The issue reports that `openclaw update` can keep reinstalling into Homebrew Cellar or other per-Node global npm prefixes, leaving the `openclaw` command dangling after a Node version-manager upgrade deletes the old prefix.

## Best Possible Solution

Review and land or replace the linked PR with a focused guard that rejects known per-Node ephemeral prefixes before selecting an owning npm, with regression coverage for Homebrew Cellar and common Node version-manager layouts.

## Reproduction Assessment

Yes. This is high-confidence source-reproducible because current main derives `<prefix>/bin/npm` from the package root and has no guard for Homebrew Cellar or Node version-manager prefixes, matching the reporter's dangling-symlink evidence.

## Solution Assessment

Yes. The narrow maintainable fix is to reject ephemeral per-Node prefixes before choosing the owning npm while preserving stable-prefix behavior; the linked PR appears to target that exact boundary.
