## Summary

Keep open: current main still derives an owning npm from the installed package prefix without excluding Homebrew Cellar or other per-Node prefixes, and the linked closing PR remains open rather than merged.

## What This Changes

Prevent `openclaw update` from reinstalling npm global installs into Homebrew Cellar or other per-Node version-manager prefixes that disappear after Node upgrades.

## Best Possible Solution

Land or replace the linked PR with a focused `src/infra/update-global.ts` fix that rejects per-Node npm commands while preserving npm install detection, adds regression coverage for Homebrew Cellar and common Node managers, and closes this issue only after merge.

## Reproduction Assessment

Yes. This is high-confidence source-reproducible on current main because the resolver maps a Cellar-shaped package root to its adjacent npm and the reporter supplied concrete dangling-symlink evidence; I did not run a destructive Homebrew upgrade locally.

## Solution Assessment

Yes on direction, with one implementation caveat. The narrow maintainable solution is to reject known per-Node npm commands before update argv selection while keeping trapped npm installs detectable, so the linked PR or a replacement should handle that boundary before landing.
