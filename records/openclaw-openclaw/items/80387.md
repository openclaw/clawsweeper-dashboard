## Summary

Keep open: current main and v2026.5.12 still prefer the adjacent npm from the package prefix without excluding Homebrew Cellar or other per-Node prefixes, while the linked closing PR remains open, conflicting, and previously reviewed as needing correction.

## What This Changes

The issue reports that `openclaw update` can keep reinstalling into a per-Node Homebrew or version-manager directory, leaving the CLI symlink dangling after that Node version is removed.

## Best Possible Solution

Repair or replace the linked PR with a focused resolver fix that still detects trapped npm installs but uses a stable/PATH npm for known per-version prefixes, with regression coverage and real `openclaw update` relocation proof.

## Reproduction Assessment

Yes. The source path is high-confidence: a package root under `/opt/homebrew/Cellar/node/<version>/lib/node_modules/openclaw` still makes the resolver select that prefix's `bin/npm`, and the reporter supplied matching dangling-symlink evidence after `brew upgrade`.

## Solution Assessment

Yes, the fix direction is narrow and maintainable, but it has not landed. The implementation should separate npm ownership detection from reinstall command selection so per-version prefixes can be detected without being reused for the next install.
