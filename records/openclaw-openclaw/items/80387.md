## Summary

Keep open: current main still allows the self-update resolver to prefer an npm binary under the installed package prefix without excluding Homebrew Cellar or other per-Node version-manager prefixes, and the linked fix PR https://github.com/openclaw/openclaw/pull/80386 is still open rather than merged.

## What This Changes

The issue reports that `openclaw update` can keep reinstalling OpenClaw into a per-version Node prefix, leaving the `openclaw` command dangling after Homebrew or another Node manager deletes that prefix.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/80386 with a fix that preserves npm install detection while forcing update installs out of per-version prefixes, with regression tests for Homebrew Cellar and Node-manager layouts.

## Reproduction Assessment

Yes. The source path is high-confidence reproducible: current main derives `<prefix>/bin/npm` from the running package root and feeds that install target into global update, matching the reporter's dangling Homebrew Cellar symlink evidence.

## Solution Assessment

Yes for the issue direction: reject known per-Node ephemeral prefixes for the update install command while preserving stable-prefix behavior. The linked PR needs revision or replacement because prior review found it can also break detection of trapped npm installs.
