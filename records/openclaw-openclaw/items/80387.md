## Summary

Keep open: current main and the latest release still prefer the adjacent npm under the inferred package prefix, so a Homebrew Cellar or other per-Node global install can keep reinstalling into a prefix that disappears after a Node upgrade. The paired fixing PR at https://github.com/openclaw/openclaw/pull/80386 is still open and conflicting, so this issue should stay open until that PR is repaired, landed, or explicitly closed.

## What This Changes

Keep the `openclaw` CLI reachable across Homebrew and Node version-manager upgrades by avoiding self-updates into ephemeral per-Node global npm prefixes.

## Best Possible Solution

Repair or replace https://github.com/openclaw/openclaw/pull/80386 so npm ownership detection still recognizes trapped npm installs while the reinstall command falls back to PATH npm for known ephemeral per-Node prefixes, with regression coverage for Homebrew Cellar and common version-manager layouts.

## Reproduction Assessment

Yes. A high-confidence source reproduction is `pkgRoot` under `/opt/homebrew/Cellar/node/<version>/lib/node_modules/openclaw` with an adjacent `bin/npm`; current main will prefer that adjacent npm, matching the reporter's dangling symlink evidence after `brew upgrade`.

## Solution Assessment

Yes for keeping this issue open while the paired PR is resolved. The narrow fix should separate npm install ownership detection from reinstall command choice so OpenClaw can detect the trapped install but relocate it through a stable npm path.
