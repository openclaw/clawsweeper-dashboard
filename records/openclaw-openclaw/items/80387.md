## Summary

Keep open: current main and the latest release still allow self-update to prefer an adjacent npm under the running package prefix, and the linked closing PR remains open with a review blocker instead of being merged.

## What This Changes

The issue asks OpenClaw self-update to avoid reinstalling into per-Node global prefixes so the `openclaw` command survives Homebrew and version-manager Node upgrades.

## Best Possible Solution

Land a corrected version of the linked PR or a replacement that separates npm ownership detection from reinstall command selection, with regression coverage for Homebrew Cellar and common Node version-manager layouts plus real self-update relocation proof.

## Reproduction Assessment

Yes. With `pkgRoot` under `/opt/homebrew/Cellar/node/<version>/lib/node_modules/openclaw` and an adjacent `bin/npm` present, current source selects that adjacent npm for the update path, matching the reporter's dangling-symlink evidence.

## Solution Assessment

Yes, the maintainable direction is a focused updater resolver fix. The open PR targets the right surface, but the fix must preserve npm ownership detection while refusing per-version npm for the reinstall command.
