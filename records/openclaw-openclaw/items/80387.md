## Summary

Keep open: current main still selects an adjacent npm from the package prefix without excluding Homebrew Cellar or other per-Node version-manager prefixes, and the explicitly linked fix PR is open rather than merged.

## What This Changes

The issue asks for npm self-update command selection to keep `openclaw` reachable after Homebrew or Node version-manager upgrades delete per-version global npm prefixes.

## Best Possible Solution

Update the npm self-update resolver so trapped installs still detect as npm, but per-version Homebrew and Node version-manager prefixes reinstall through a stable PATH/global prefix, with regression coverage for those layouts.

## Reproduction Assessment

Yes. Current main is source-reproducible because a package root under `<prefix>/lib/node_modules/openclaw` leads to `<prefix>/bin/npm` with no ephemeral-prefix guard, matching the reporter's Homebrew Cellar symlink evidence.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix, but it is not landed yet. The safest implementation should separate npm ownership detection from reinstall command selection so stable prefixes keep existing behavior.
