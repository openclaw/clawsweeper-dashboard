## Summary

Keep open: current main still lacks an ephemeral Homebrew Cellar or Node-version-manager prefix guard in the global update resolver, and the linked closing PR remains open rather than merged.

## What This Changes

The issue reports that OpenClaw can disappear from PATH after a Node upgrade when self-update keeps reinstalling into a per-version Homebrew Cellar or version-manager npm prefix.

## Best Possible Solution

Land or replace the linked PR with a focused fix that rejects ephemeral per-Node npm commands for update execution while still detecting trapped npm installs, plus regression coverage for Homebrew Cellar and common version-manager layouts.

## Reproduction Assessment

Yes. This is high-confidence source-reproducible on current main because the resolver infers `<prefix>` from the reported package-root shape and chooses `<prefix>/bin/npm` when it exists; I did not run a destructive live Homebrew upgrade.

## Solution Assessment

Yes on the requested direction, but the open implementation still needs review follow-up. The narrow maintainable fix is to separate npm ownership detection from preferred install-command selection so trapped installs can be updated with PATH npm.
