## Summary

Keep open: current main and the latest release still select an adjacent npm from the package prefix without excluding Homebrew Cellar or other per-Node ephemeral prefixes, and the linked closing PR remains open and conflicting rather than merged.

## What This Changes

The issue requests that OpenClaw self-update avoid per-Node package-manager prefixes so the CLI stays on PATH across Homebrew and version-manager Node upgrades.

## Best Possible Solution

Correct or replace the linked PR with a focused resolver fix that separates npm ownership detection from reinstall command selection, then add regression coverage for Homebrew Cellar and common Node version-manager layouts while preserving stable-prefix behavior.

## Reproduction Assessment

Yes. This is high-confidence source-reproducible because current main derives `<prefix>/bin/npm` from the running package root and lacks an ephemeral-prefix guard, matching the reporter's Homebrew Cellar evidence.

## Solution Assessment

Yes, the narrow maintainable direction is to reject known per-Node ephemeral prefixes for the reinstall command while preserving npm ownership detection and stable global prefixes; the existing linked PR targets the right area but still needs review follow-up.
