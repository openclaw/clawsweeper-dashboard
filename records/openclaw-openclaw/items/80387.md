## Summary

Keep open: current main still lets a package root under a per-Node prefix resolve to that prefix's adjacent npm, and the linked fixing PR is still open rather than merged.

## What This Changes

The issue asks OpenClaw self-update to avoid reinstalling global npm installs into Homebrew Cellar or other per-Node version-manager prefixes that can disappear after Node upgrades.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/80386 with a narrow ephemeral-prefix guard plus regression tests while preserving stable global-prefix behavior.

## Reproduction Assessment

Yes: this is high-confidence source-reproducible because current main derives an owning npm from the package root and lacks an ephemeral Homebrew Cellar or version-manager prefix guard. I did not run a destructive macOS Homebrew upgrade in this checkout.

## Solution Assessment

Yes: the narrow maintainable fix is to reject known per-Node ephemeral prefixes before choosing the adjacent npm, then fall back to PATH npm for relocation to a stable prefix. The linked PR targets that boundary, so the issue should remain open for review and landing of that fix.
