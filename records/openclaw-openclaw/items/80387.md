## Summary

Keep open: current main still uses the adjacent npm inferred from the running package root, with no Homebrew Cellar or version-manager ephemeral-prefix guard, and the linked closing PR remains open and conflicting.

## What This Changes

The issue reports that `openclaw update` can keep reinstalling into a per-version Node prefix, leaving the `openclaw` shim dangling after a Homebrew or version-manager Node upgrade.

## Best Possible Solution

Correct or replace the linked PR with a focused updater fix that separates npm ownership detection from reinstall command selection and adds regression coverage for common ephemeral Node-prefix layouts.

## Reproduction Assessment

Yes. Source inspection shows current main still selects an adjacent `<prefix>/bin/npm` from the package root without excluding Homebrew Cellar or version-manager prefixes, matching the reporter's dangling symlink evidence.

## Solution Assessment

Yes. The best direction is to keep stable-prefix ownership behavior but refuse per-version npm commands for reinstall, while preserving manager detection so trapped installs can still update out of the bad prefix.
