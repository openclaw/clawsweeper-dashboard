## Summary

Keep open: current main and the latest release still lack an ephemeral Node-prefix guard, and the linked closing PR remains open and conflicting rather than merged.

## What This Changes

The issue asks for `openclaw update` to keep the CLI installed in a version-agnostic npm prefix when the current global install is trapped under a Homebrew Cellar or other per-Node version-manager directory.

## Best Possible Solution

Land a corrected resolver fix that still detects trapped npm installs as npm but chooses a stable PATH npm for reinstall when the current package root is under known per-Node ephemeral prefixes, with regression coverage for Homebrew Cellar and common Node managers.

## Reproduction Assessment

Yes. The issue is source-reproducible on current main: a package root under `/opt/homebrew/Cellar/node/<version>/lib/node_modules/openclaw` with an adjacent `bin/npm` will be treated as the preferred npm command and carried into self-update, matching the reporter's dangling-symlink evidence.

## Solution Assessment

Yes in direction, but not as already merged. The maintainable fix is to separate npm ownership detection from reinstall command selection so trapped installs can be recognized while the reinstall avoids known per-version prefixes.
