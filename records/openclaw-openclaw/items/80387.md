## What This Changes

Ensure `openclaw update` keeps the CLI reachable by avoiding per-Node npm prefixes that disappear after Homebrew or version-manager Node upgrades.

## Best Possible Solution

Land or replace the linked PR with a focused update resolver fix that keeps trapped npm installs detectable while using a stable PATH npm for relocation, plus regression coverage for ephemeral prefixes.

## Reproduction Assessment

Yes. The issue has concrete Homebrew dangling-symlink evidence, and current main source shows the resolver can select the adjacent Cellar npm; I did not run a destructive live `brew upgrade` locally.

## Solution Assessment

Yes for the requested direction: the narrow maintainable fix is to reject per-Node npm binaries for reinstall while preserving npm ownership detection for the trapped package root. The linked PR should be reviewed or revised to that boundary before closing this issue.
