## What This Changes

Keep the `openclaw` command reachable across Homebrew or Node version-manager upgrades by preventing self-update from reinstalling into per-version Node prefixes.

## Best Possible Solution

Keep this issue open until the linked PR is corrected or replaced with a resolver fix that separates npm ownership detection from reinstall command selection and adds regression coverage for common per-Node layouts.

Yes. This is high-confidence source-reproducible because current main derives `<prefix>/bin/npm` from the running package root and lacks an ephemeral-prefix guard, matching the reporter's Homebrew Cellar symlink evidence.

## Solution Assessment
