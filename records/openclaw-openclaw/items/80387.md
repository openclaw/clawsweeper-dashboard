## Summary

Keep open: current main still selects an adjacent npm from the inferred package prefix without excluding Homebrew Cellar or other per-Node version-manager prefixes, and the linked closing PR is open with review feedback rather than merged.

## What This Changes

Keep `openclaw update` reachable across Homebrew or Node-manager version bumps by avoiding per-version npm prefixes during self-update.

## Best Possible Solution

Land or replace the linked PR with a focused resolver fix that separates npm ownership detection from install-command selection, rejects ephemeral per-Node prefixes for the update argv, and adds regression coverage for the full self-update path.

## Reproduction Assessment

Yes. Current source shows a package root under `<prefix>/lib/node_modules/openclaw` can select `<prefix>/bin/npm` when present, matching the reporter's Homebrew Cellar dangling-symlink evidence; I did not run a destructive brew upgrade in this checkout.

## Solution Assessment

The direction is correct, but the linked implementation still needs review follow-up. The maintainable fix is to keep trapped npm installs detectable while choosing PATH npm for ephemeral per-Node prefixes during the reinstall.
