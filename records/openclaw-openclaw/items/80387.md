## Summary

Keep open: current main still selects an adjacent npm from the package-owned prefix without excluding Homebrew Cellar or other per-version Node-manager prefixes, and the linked fix PR is still open rather than merged.

## What This Changes

The issue asks OpenClaw self-update to avoid reinstalling into per-version Homebrew or Node-manager directories so the `openclaw` command remains on PATH after Node upgrades.

## Best Possible Solution

Land or replace the linked fix with a narrow updater change that refuses per-version npm commands while still detecting the trapped install as npm-owned, plus regression coverage for the full self-update path.

## Reproduction Assessment

Yes. The issue is high-confidence source-reproducible because current main infers `<prefix>/bin/npm` from the package root and lacks an ephemeral-prefix guard, matching the reporter's Homebrew dangling-symlink evidence.

## Solution Assessment

Yes on direction, but not yet landed. The maintainable fix is to reject known per-version npm command paths while preserving npm manager detection; the linked PR needs that adjustment before it can resolve the issue.
