## Summary

Keep open: current main still allows the updater to select an adjacent npm from an inferred package prefix without excluding Homebrew Cellar or other per-Node version-manager prefixes, and the linked fixing PR is still open with review feedback to resolve.

## What This Changes

OpenClaw should keep the CLI reachable after Homebrew or Node version-manager upgrades by avoiding self-update reinstalls into per-version Node prefixes.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/80386 with a fix that keeps npm install detection intact while choosing stable PATH npm for ephemeral per-Node prefixes, plus targeted regression coverage.

## Reproduction Assessment

Yes: this is high-confidence source-reproducible because current main can infer a Homebrew Cellar-style `<prefix>` from `pkgRoot` and select `<prefix>/bin/npm`, matching the reporter's dangling symlink evidence. I did not run a destructive macOS Homebrew upgrade locally.

## Solution Assessment

Yes on direction, but not yet implemented on main: the maintainable fix is to reject known per-Node npm prefixes for update command selection while preserving valid stable-prefix behavior and npm manager detection. The open linked PR is the right review path, but it needs the existing review feedback resolved before landing.
