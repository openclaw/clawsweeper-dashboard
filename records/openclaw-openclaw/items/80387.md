## Summary

Keep open: current `main` still picks an owning npm from the installed package prefix without excluding Homebrew Cellar or other per-Node prefixes, and the linked closing PR remains open rather than merged.

## What This Changes

The issue asks for `openclaw update` to avoid reinstalling into per-Node Homebrew or version-manager prefixes so the CLI stays on PATH after Node upgrades.

## Best Possible Solution

Land or replace the linked PR with a focused resolver fix that separates npm ownership detection from reinstall command selection, plus regression coverage for Homebrew Cellar and Node version-manager layouts.

## Reproduction Assessment

Yes. Current `main` source still maps `<prefix>/lib/node_modules/openclaw` to `<prefix>/bin/npm`, matching the reporter's Homebrew Cellar dangling-shim evidence; I did not run a destructive live `brew upgrade` repro.

## Solution Assessment

Yes. The narrow maintainable direction is to reject per-Node ephemeral npm commands for reinstall while preserving stable-prefix npm ownership detection; the open linked PR should be reviewed or revised on that boundary.
