## Summary

Keep open: the fix targets a real Discord message-delivery bug, but this PR is not merge-ready because the production block reply pipeline still flushes buffered text before media and the PR only changes/tests the lower-level coalescer path. It also needs real Discord behavior proof before merge.

## What This Changes

The PR changes `createBlockReplyCoalescer` to merge buffered text into a following media payload and adds coalescer unit tests for merged media, media-only, and reply-target conflict cases.

## Best Possible Solution

Move the merge to the production block reply pipeline boundary, preserve reply and visibility metadata, reconcile the existing tests, and prove a live Discord text-plus-attachment reply sends as one message.

## Reproduction Assessment

Yes. Source inspection shows the production pipeline still calls `coalescer?.flush({ force: true })` before sending media payloads, which reproduces the reported split without needing a live Discord run.

## Solution Assessment

No. The coalescer-only implementation is not the best fix because the production pipeline bypasses that media branch; the maintainable fix should cover `createBlockReplyPipeline` and preserve the coalescer’s reply/visibility boundaries.
