This PR adds a fallback from `fs.promises.link` to exclusive `writeFile(..., { flag: "wx" })` for session usage-cost lock creation and adds a regression test for `ENOTSUP`.

## Best Possible Solution

## Solution Assessment

Yes for the code shape: handling unsupported hard-link errors inside `writeUsageCostCacheLockAtomically` preserves the existing lock owner, release path, `EEXIST` behavior, and stale-lock handling. The remaining question is proof acceptance, not a better implementation layer.
