## Summary

Keep open: current main still has the hard-link-only lock path, and this PR is a focused, likely correct fix with regression coverage, but the external PR still needs real behavior proof from an actual unsupported-hard-link filesystem before merge.

## What This Changes

Adds a fallback in `writeUsageCostCacheLockAtomically` from `fs.promises.link` to exclusive `writeFile(..., { flag: "wx" })` for unsupported-link errors, plus an ENOTSUP regression test.

## Best Possible Solution

## Solution Assessment
