## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Require source context before preserving hash evidence:** `extensions/memory-wiki/src/markdown.ts:224`
  - body: Because `hash` now participates in the non-empty guard, `{ hash: "..." }` survives as a claim evidence entry. Current lint, claim-health, and provenance coverage use `claim.evidence.length` to decide whether a claim has structured evidence, so this suppresses missing-evidence warnings even when there is no sourceId, path, lines, or kind that a verifier could re-hash. Drop hash-only entries or require source-bearing evidence alongside `hash`.
  - confidence: 0.87
- **[P2] Define the digest format in the schema:** `extensions/memory-wiki/src/tool.ts:44`
  - body: The PR describes a SHA-256 source hash, but the exposed `wiki_apply` schema accepts any non-empty string. That persists invalid or algorithm-ambiguous digest values into frontmatter, so downstream verification cannot distinguish real source hashes from arbitrary text. Use a strict hex or `sha256:<hex>` contract, or document an intentionally opaque field before exposing it as verification data.
  - confidence: 0.78
