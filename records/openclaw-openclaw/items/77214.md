## Summary

Keep open: current main still has the misleading `plugins.allow` fallback, and two open PRs explicitly target this issue, so the issue should stay open until one fix merges.

## What This Changes

The issue requests CLI unknown-subcommand diagnostics that recognize manifest-declared agent tool names and avoid recommending `plugins.allow` for tool names.

## Best Possible Solution

Review and land one of #77220 or #77221, preserving the distinction that agent tools are model-use tools rather than CLI subcommands.

## Reproduction Assessment

Yes. Source inspection shows that with a nonempty `plugins.allow` and an unknown primary token like `lcm_recent`, current main has no tool-owner branch and returns the allowlist suggestion from `src/cli/run-main-policy.ts`.

## Solution Assessment

Yes. The narrow maintainable fix is to use manifest `contracts.tools` ownership before the generic allowlist fallback while preserving existing plugin-id and command-alias diagnostics.
