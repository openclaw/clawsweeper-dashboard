## Summary

Keep open: current main still has the misleading `plugins.allow` fallback for agent-tool-shaped root commands, and the attached closing PRs have not merged yet.

## What This Changes

Requests CLI unknown-subcommand diagnostics to recognize plugin-owned agent tool names and avoid suggesting `plugins.allow` for tool names.

## Best Possible Solution

Land one focused CLI fix that looks up manifest `contracts.tools` ownership before the `plugins.allow` fallback, with regression coverage for agent-tool names and preserved plugin-id/alias behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows the missing-command policy can reproduce the bad message with a restrictive `plugins.allow`, a non-reserved unknown token, and no command-alias owner; no branch checks manifest `contracts.tools` before the fallback.

## Solution Assessment

Yes. The narrow maintainable fix is a manifest-backed tool-owner diagnostic before the allowlist fallback, without making agent tools executable as CLI subcommands or changing plugin config semantics.
