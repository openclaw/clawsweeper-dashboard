## Summary

Keep open: current main still has the source-level `plugins.allow` fallback for unknown root tokens without consulting manifest `contracts.tools`, and two open closing PRs already target the fix.

## What This Changes

The issue asks the CLI to recognize agent tool names declared by plugin manifests and avoid suggesting `plugins.allow` for tool names.

## Best Possible Solution

Land one manifest-backed CLI diagnostic fix with regression coverage for `contracts.tools`, then close the competing PR as duplicate or superseded.

## Reproduction Assessment

Yes, at source level: the current policy only resolves command aliases and then emits the `plugins.allow` suggestion for any disallowed unknown token. Manifest `contracts.tools` is present elsewhere but not consulted in this path.

## Solution Assessment

Yes, the narrow maintainable fix is to look up manifest-declared agent tool ownership before the allowlist fallback while preserving existing plugin-id and command-alias behavior.
