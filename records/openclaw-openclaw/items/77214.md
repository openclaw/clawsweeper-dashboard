## Summary

Keep open: current main still has the misleading `plugins.allow` fallback for agent-tool-looking root commands, and two open closing PRs are already paired with this issue for the fix.

## What This Changes

Change the unknown CLI subcommand diagnostic so manifest-declared agent tool names are described as model tools instead of suggesting `plugins.allow` edits.

## Best Possible Solution

Review and land one small CLI implementation that adds manifest `contracts.tools` ownership diagnostics while preserving existing plugin-id, command-alias, and runtime slash-command behavior.

## Reproduction Assessment

Yes. Source inspection shows an unknown root token with restrictive `plugins.allow` can still reach the fallback allowlist message because current main checks command aliases but not manifest-declared `contracts.tools`.

## Solution Assessment

Yes for the requested direction. The narrow maintainable fix is to add a manifest-backed agent-tool ownership branch in the CLI diagnostic path, not to make agent tools CLI subcommands or change `plugins.allow` semantics.
