## Summary

Keep open: current `main` still lacks an agent-tool-aware diagnostic, and two open PRs already target this issue with closing references, so the issue should remain open until one fix lands or maintainers choose a different patch.

## What This Changes

The issue asks for the CLI unknown-subcommand handler to recognize manifest-declared agent tool names and avoid suggesting `plugins.allow` edits for them.

## Best Possible Solution

Land one focused fix that resolves manifest `contracts.tools` ownership before generic unknown-command or allowlist diagnostics, while preserving existing plugin-id allowlist behavior and adding regression coverage.

## Reproduction Assessment

Yes, at source level: current `main` has no `contracts.tools` lookup in the missing-command policy, and the fallback still tells users to add the unknown token to `plugins.allow`. I did not run the CLI because this review was read-only, but the control flow and contracts are clear from source.

## Solution Assessment

Yes, the narrow maintainable fix is to use existing manifest tool ownership metadata for diagnostics only. The fix should not make agent tools CLI subcommands or change the `plugins.allow` contract.
