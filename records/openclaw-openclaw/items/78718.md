## Summary

Keep open for maintainer review. The PR is a focused implementation for the linked heartbeat precedence report, and I found no blocking code-review findings, but it changes behavior that current docs and tests explicitly encode and overlaps an open empty-heartbeat semantics question.

## What This Changes

The PR makes agent heartbeat enablement merge per-agent heartbeat blocks over `agents.defaults.heartbeat` across summary, scheduler, system-prompt, ACP spawn, docs, tests, and changelog.

## Best Possible Solution

Have heartbeat owners decide the precedence and empty-object contract, then land the centralized implementation with docs/tests if that inheritance model is accepted.

## Reproduction Assessment

Yes for current-main behavior by source inspection: the summary and scheduler paths both use explicit-only gates, and the current test suite asserts that mixed defaults-plus-explicit disables the default-inheriting agent. I did not execute the runtime because this review is read-only.

## Solution Assessment

Yes, if maintainers accept the new semantics. Centralizing policy in `isHeartbeatEnabledForAgent` is the narrow maintainable shape, but the allowlist-to-inheritance change and empty-object behavior need owner approval.
