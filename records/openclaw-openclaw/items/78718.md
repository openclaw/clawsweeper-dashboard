## Summary

Keep this PR open, but the current patch is not mergeable: it changes documented/tested heartbeat scoping semantics, still mishandles disabled/empty per-agent heartbeat config, leaves other heartbeat policy gates on the old rule, and the supplied screenshot shows a module-resolution failure rather than successful after-fix behavior.

## What This Changes

The PR changes heartbeat summary, scheduler selection, and tests so agents without per-agent heartbeat blocks inherit `agents.defaults.heartbeat` even when another agent has explicit heartbeat config.

## Best Possible Solution

If maintainers accept the new precedence, centralize effective heartbeat resolution and reuse it across summary, scheduler, system-prompt guidance, ACP relay, docs, changelog, and regression tests.

## Reproduction Assessment

Yes, at source level: current main excludes a default-inheriting agent once another agent has a heartbeat block. However, current docs and tests also define that explicit-only scoping rule, so this is a reproducible behavior-change request rather than a strict documented-contract bug.

## Solution Assessment

No. The requested direction may be acceptable after maintainer agreement, but this patch is not the best implementation because it misparses disabled cadence, leaves other policy gates unchanged, and does not update docs/changelog or provide successful runtime proof.
