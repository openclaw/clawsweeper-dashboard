## Summary

Keep open: this is a focused, plausible fix for a real source-reproducible heartbeat precedence problem, and I found no blocking review findings. It still needs maintainer approval because the patch changes documented and tested config precedence from explicit-only agent heartbeats to standard defaults-plus-overrides semantics.

## What This Changes

This PR changes heartbeat enablement so scheduler, system-prompt, and ACP spawn paths use a shared per-agent-over-defaults heartbeat policy, with docs, tests, and changelog updated for the new precedence.

## Best Possible Solution

Adopt one centralized heartbeat precedence contract across runtime surfaces, docs, and tests, then close the linked issue and duplicate reports after the accepted implementation merges.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: the mixed defaults-plus-explicit config follows explicit-only gates on current main, and the existing test expects the default-inheriting agent to be disabled.

## Solution Assessment

Yes, if maintainers accept the precedence change. Centralizing enablement in `isHeartbeatEnabledForAgent` and reusing it from scheduler, system-prompt, and ACP spawn paths is the narrow maintainable fix shape.
