## Summary

Keep open: the PR addresses a real source-reproducible heartbeat precedence bug and has sufficient after-fix proof, but the patch is not merge-ready because current main now has a doctor heartbeat enumeration path that would remain on the old explicit-only policy after this change.

## What This Changes

The PR changes heartbeat enablement precedence so per-agent heartbeat config merges over `agents.defaults.heartbeat` across scheduler, summary, system prompt, ACP spawn, docs, tests, and changelog.

## Best Possible Solution

Adopt one explicit heartbeat precedence contract and use one shared policy across scheduler, summary/status, system prompt, ACP spawn, doctor diagnostics, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the mixed defaults-plus-explicit config follows explicit-only branches in summary and scheduler code, and the checked-in test currently expects the default-inheriting agent to be disabled.

## Solution Assessment

No as-is. Centralizing heartbeat enablement is the right direction, but the PR needs current-main doctor alignment and maintainer resolution of the empty-object heartbeat contract before it is the narrow maintainable fix.
