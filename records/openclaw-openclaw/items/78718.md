## Summary

Keep open: the reported heartbeat fallback problem is still present on current main and this PR is a plausible implementation candidate, but it now needs rebase/maintainer review because current main added heartbeat doctor diagnostics that the PR does not keep aligned.

## What This Changes

The PR changes heartbeat enablement precedence so per-agent heartbeat blocks merge over `agents.defaults.heartbeat` across scheduler, system prompt, ACP spawn, docs, tests, and changelog.

## Best Possible Solution

Adopt one centralized heartbeat precedence contract, update scheduler, summary, prompt, ACP spawn, doctor diagnostics, docs, and regression tests together, then close the linked canonical issue and duplicates after merge.

## Reproduction Assessment

Yes. Source inspection shows the mixed defaults-plus-explicit config still follows explicit-only gates in current `main`, and the checked-in test expects the default-inheriting `main` agent to be disabled.

## Solution Assessment

No as-is. Centralizing heartbeat enablement is the right direction, but the PR needs rebase and doctor-diagnostic alignment with current main before it is the narrow maintainable fix.
