## What This Changes

The PR expands gateway chat-history inline image redaction for `source.data`, data-url `url`, and `image_url.url` payload shapes and adds regression tests for redaction plus accumulated omitted-byte accounting.

## Best Possible Solution

I did not run a live Gateway/WebChat repro in this read-only review.

## Solution Assessment

No. The redaction logic is the right direction, but patching `server-methods/chat.ts` is no longer the maintainable fix because current main centralizes display projection in `chat-display-projection.ts`.
