## Summary

Keep open: this is a useful gateway display bug fix, but the current PR is not the right integration point after main moved chat-history display projection into the shared projector. The branch also still lacks real after-fix Gateway/WebChat behavior proof, so it should not merge as-is.

## What This Changes

The PR adds gateway chat.history sanitization for inline image payload shapes in `server-methods/chat.ts` and regression tests for redaction plus omitted-byte accounting.

## Best Possible Solution

Port the expanded inline-image redaction into `src/gateway/chat-display-projection.ts`, keep byte accounting accurate across stripped fields, cover `chat.history`, live `session.message`, and session-history snapshots, and leave durable transcript claim-checking to https://github.com/openclaw/openclaw/issues/1210.

## Reproduction Assessment

Yes, at source level: a message with `content: [{ type: "image", source: { type: "base64", data: "..." } }]` passed through the current projector keeps the nested image payload because current main only strips top-level `image.data` and audio `source.data`. I did not run a live Gateway/WebChat repro in this read-only review.

## Solution Assessment

No. The redaction logic is useful, but placing it only in `server-methods/chat.ts` is no longer the maintainable fix because current main centralizes display projection in `chat-display-projection.ts`.
