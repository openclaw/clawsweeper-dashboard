## Summary

Keep open: the PR fixes a real gateway display-history problem and the byte-accounting review note was addressed, but current main moved chat history, live session message, and session-history snapshot normalization into the shared display projector. The branch still patches the old `server-methods/chat.ts` sanitizer location and is dirty against main, so it needs a focused port before merge.

## What This Changes

The PR adds gateway `chat.history` sanitizer logic and tests to remove inline image data payloads from image content blocks and accumulate omitted byte counts.

## Best Possible Solution

Port the expanded inline-image redaction into `src/gateway/chat-display-projection.ts`, keep byte accounting accurate across stripped fields, cover all display paths, and leave durable media-reference storage to #1210.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to pass a message with `content: [{ type: "image", source: { type: "base64", data: "..." } }]` through `projectChatDisplayMessage` or `projectRecentChatDisplayMessages`; current main leaves that nested image payload intact.

## Solution Assessment

No. The PR's behavior is useful, but the implementation location is no longer the narrowest maintainable fix because current main centralizes UI display projection in `chat-display-projection.ts`.
