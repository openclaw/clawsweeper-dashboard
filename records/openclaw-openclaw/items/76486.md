## Summary

Close as `clawhub`: DeepSeek-TUI is a useful optional CLI backend, but current main already has the generic plugin and CLI-backend APIs needed for an external plugin, and OpenClaw policy keeps optional integrations out of core unless there is a clear product/security reason.

## What This Changes

The issue requests first-class DeepSeek-TUI support as an `agentRuntime.id: "deepseek-tui"` CLI backend with DeepSeek auth, model defaults, session persistence, and optional MCP integration.

## Best Possible Solution

Publish a DeepSeek-TUI integration as an external native OpenClaw plugin through ClawHub using the existing CLI backend registration seam, and revisit core only if that exposes a missing generic SDK capability.

## Reproduction Assessment

Not applicable; this is a feature request for a new optional CLI backend, not a reported failure. Source inspection confirms the relevant question is product scope and plugin API coverage rather than a current-main reproduction path.

## Solution Assessment

Yes; closing as ClawHub-scope is the narrowest maintainable path because current main already exposes the needed external plugin and CLI-backend contracts. Adding a bundled core backend would need a separate product/security reason beyond the issue’s optional integration request.
