## Summary

This PR is a plausible targeted fix for the approval-handler stop race, and the R2 follow-up addresses the Matrix pre-bind cleanup concern; the only remaining issue I found is a small docs gap for the newly exported SDK hook.

## What This Changes

Adds stopped-state checks to channel approval delivery, an optional `cancelDelivered` interaction hook, Matrix reaction-target cleanup, and regression coverage for stop races during delivery.

## Best Possible Solution

Keep the runtime cleanup approach, but document the new optional SDK hook so channel plugins with `deliverPending` side effects know how to release delivered-but-unbound state.

## Reproduction Assessment

Yes. Current-main source shows the awaited delivery/bind/map-write race, and the PR body supplies deterministic before/after stub-plugin logs that exercise the stop window.

## Solution Assessment

Mostly yes. The stopped flag plus binding-aware cleanup is the narrow fix for this race, and the Matrix `cancelDelivered` follow-up closes the pre-bind side-effect leak; the new SDK hook still needs a short docs update.
