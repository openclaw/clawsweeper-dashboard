## Summary

Keep open: the patch fixes the post-bind orphan path, but the current PR still drops a successfully delivered entry when stop fires before bindPending, leaving Matrix-style native state untracked.

## What This Changes

The PR adds stopped-state guards around shared native approval delivery, unbinds bindings after a stop during bindPending, and adds a unit test for that in-flight bind window.

## Best Possible Solution

Keep the stopped-state guard, but ensure every delivered entry is either bound and unbound or routed through an existing finalization cleanup path before it is dropped.

## Reproduction Assessment

Yes. Current main shows the deliverPending/bindPending/activeEntries race, and the linked issue plus PR body provide deterministic stub-plugin counts; the remaining PR finding is source-reproducible through Matrix registering reaction targets in deliverPending before bindPending.

## Solution Assessment

No, not as written. The post-bind stopped cleanup is the right narrow direction, but the pre-bind stopped branch must not drop a delivered entry without cleanup.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Clean up delivered entries before the stopped return:** `src/infra/approval-handler-runtime.ts:515-522`
