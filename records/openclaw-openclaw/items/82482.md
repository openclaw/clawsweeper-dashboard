## Summary

Keep this PR open: it targets a real current-main approval-handler stop race and has useful proof, but the current head still has a blocking cleanup gap when stop fires after deliverPending returns and before bindPending runs.

## What This Changes

The PR adds a stopped flag to channel approval delivery so in-flight native approval bindings are not inserted after stop, plus a regression test for the bindPending-to-activeEntries race.

## Best Possible Solution

Keep the stopped-state cleanup, but make delivered entries pass through a cleanup path even when stop fires before bindPending, and cover that window with a deterministic regression test.

## Reproduction Assessment

Yes. Current-main source shows the deliverPending/bindPending/activeEntries race, and the paired issue plus PR body provide deterministic stub-plugin before/after proof; the remaining review finding is source-reproducible through Matrix reaction state registered in deliverPending before bindPending.

## Solution Assessment

No, not as written. The stopped flag and post-bind unbind are the right narrow shape, but the pre-bind stopped branch must cleanup the delivered entry instead of returning null immediately.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86
