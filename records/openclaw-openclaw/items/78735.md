## Summary

Keep open: the gateway security hardening is useful and the supplied proof is sufficient, but the current branch still has blocking docs and changelog regressions that should be fixed before merge.

## What This Changes

The PR rejects `dangerouslyForceUnsafeInstall=true` for gateway `skills.install`, stops web and macOS clients sending it, adds schema/handler tests, and updates related docs and changelog text.

## Best Possible Solution

Land a narrow gateway protocol/runtime/client hardening that forces gateway installs through scans while preserving upload archive docs, security guidance, and existing changelog history.

## Reproduction Assessment

Yes by source inspection. Current main accepts the boolean gateway parameter, forwards it to `installSkill`, and the scanner skips critical blocking when the flag is true; I did not run a live gateway in this read-only pass.

## Solution Assessment

No for the current branch state. Rejecting the true override and forcing gateway installs through scans is the right direction, but the docs and changelog regressions must be repaired before merge.
