## Summary

Keep open: current main still exposes an operational gateway install-scan bypass, and the PR is a focused hardening path. Because it removes a documented gateway protocol/security override, it should get protocol/security owner review before merge rather than be swept closed.

## What This Changes

The branch rejects `dangerouslyForceUnsafeInstall=true` for gateway `skills.install`, forces gateway-backed skill installs to run scans, removes web/macOS client emission of the field, and updates docs/tests/changelog.

## Best Possible Solution

Land the hardening after protocol/security owner review, preserving the CLI plugin break-glass path while making gateway-backed skill dependency installs fail closed.

## Reproduction Assessment

Yes, from source inspection: current main validates the boolean, forwards it from `skills.install`, and the scanner skips critical blocking when it is true. I did not run a live gateway request in this read-only review.

## Solution Assessment

Yes in direction: rejecting `true` at the schema boundary, forcing `false` into the installer, and removing client emission is the narrow maintainable fix. Because this removes a documented protocol/security override, owner review is still the safer path before merge.
