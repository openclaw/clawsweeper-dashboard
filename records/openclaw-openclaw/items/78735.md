## Summary

Keep this PR open: the core hardening direction is valid, but the latest diff drops documentation for the still-supported uploaded skill archive install path while changing adjacent gateway install security docs.

## What This Changes

The PR removes the usable `dangerouslyForceUnsafeInstall` gateway override from `skills.install`, updates web/macOS clients and docs, and adds schema/handler tests for rejecting `true` while tolerating legacy `false`.

## Best Possible Solution

Land the gateway fail-closed hardening after restoring the uploaded-archive install docs and getting protocol/security owner review for the documented contract removal.

## Reproduction Assessment

Yes, from source inspection: current main validates the boolean, forwards it from `skills.install`, and the scanner skips critical blocking when it is true. I did not run a live gateway request in this read-only review.

## Solution Assessment

Yes in direction: rejecting `true` at the schema boundary, forcing `false` into the installer, and removing client emission is the narrow maintainable fix. The PR should first restore docs for the still-supported upload mode and then receive protocol/security owner review.
