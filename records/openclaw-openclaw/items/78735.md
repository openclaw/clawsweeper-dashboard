## Summary

Keep open: the patch is directionally correct and I found no blocking code-review findings, but it removes a documented gateway/security protocol override and needs gateway/security owner review before merge.

## What This Changes

The PR rejects `dangerouslyForceUnsafeInstall=true` for gateway `skills.install`, forces gateway skill dependency installs to run with scans enforced, removes client-sent fields, and updates related docs/tests/changelog.

## Best Possible Solution

Land the gateway boundary hardening after owner review, keeping the CLI plugin break-glass override separate from gateway-backed skill dependency installs.

## Reproduction Assessment

Yes from source inspection: current main accepts the optional boolean, forwards it through `skills.install`, and the scanner skips critical blocking when the value is true. I did not run a live gateway request in this read-only review.

## Solution Assessment

Yes in direction: rejecting `true` at the schema boundary, forcing `false` into the installer, and removing client emission is the narrow maintainable fix. Because it changes a documented protocol/security contract, owner review is still the safer merge path.
