## Summary

This is a plausible security hardening PR, not a cleanup-close candidate: current main still accepts and forwards the dangerous gateway override, while the PR narrows that path and includes enough after-fix gateway output to satisfy the real-behavior proof gate. Because it removes a documented gateway protocol override and affects plugin install security policy, it should stay open for maintainer/security review rather than an automated close or repair lane.

## What This Changes

The PR changes gateway skill installation to reject `dangerouslyForceUnsafeInstall=true`, stops web UI and macOS clients from sending the flag, and updates related tests, docs, and the changelog.

## Best Possible Solution

Land a maintainer/security-approved gateway boundary change that rejects the true override server-side, always invokes skill installs with scans enforced, keeps CLI plugin break-glass behavior separate, and preserves current ClawHub/upload/gateway install docs and tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the schema accepts the optional boolean, the gateway handler forwards it, and the scanner skips critical blocks when the value is true; I did not run a live gateway exploit in this read-only review.

## Solution Assessment

Yes in direction. Rejecting `true` at the gateway boundary and passing `false` to the installer is the narrow maintainable fix, but the protocol/security policy change should be approved by the relevant owners before merge.
