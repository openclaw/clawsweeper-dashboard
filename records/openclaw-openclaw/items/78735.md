## Summary

Keep open: the runtime direction is a useful gateway security hardening, and the supplied real behavior proof is sufficient, but the branch is not merge-ready because it still damages changelog/docs history and removes valid upload-mode security guidance.

## What This Changes

The PR rejects `dangerouslyForceUnsafeInstall=true` for gateway `skills.install`, stops web and macOS clients sending the field, adds gateway schema/handler tests, and updates docs plus changelog text.

## Best Possible Solution

Land the narrow gateway protocol/runtime/client hardening after preserving current changelog history, upload-mode protocol docs, private archive security guidance, and removing unrelated plugin-doc churn.

## Reproduction Assessment

Yes, by source inspection. Current main accepts the boolean gateway parameter, forwards it to `installSkill`, and the scanner skips critical blocking when the flag is true; I did not run a live gateway in this read-only review.

## Solution Assessment

No, not in the current branch state. Rejecting `true` and forcing gateway installs to scan is the right narrow direction, but the changelog and documentation regressions need repair before merge.
