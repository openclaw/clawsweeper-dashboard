## Summary

Keep open: the gateway hardening is well directed and the PR supplies real behavior proof, but the latest head still removes supported upload-mode/security documentation and adds unrelated duplicate plugin docs, so it should not merge as-is.

## What This Changes

The PR tightens gateway `skills.install` validation for `dangerouslyForceUnsafeInstall`, removes the field from web and macOS callers, and updates tests, docs, and changelog content.

## Best Possible Solution

Land a scoped gateway hardening that rejects the unsafe override while preserving upload-mode docs, the private archive security note, and the existing plugin docs structure.

## Reproduction Assessment

Yes. Current main accepts the optional boolean, forwards it from the gateway handler, and the scanner skips critical blocking when the flag is true; this was verified by source inspection rather than a live gateway run.

## Solution Assessment

Yes in direction, but not in the current branch state. Rejecting `true` at the schema/handler boundary is narrow and maintainable, but the docs regressions and unrelated plugin-docs expansion need repair before merge.
