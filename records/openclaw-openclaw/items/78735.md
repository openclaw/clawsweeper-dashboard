## Summary

Keep open for maintainer/security review. Current main still allows and forwards the gateway install override, and the PR is directionally correct with no blocking code findings, but it removes a documented gateway protocol escape hatch and needs explicit owner approval before merge.

## What This Changes

The branch rejects `dangerouslyForceUnsafeInstall=true` in gateway `skills.install`, forces gateway installs to scan, removes client-sent fields from web UI/macOS, and updates docs/tests/changelog.

## Best Possible Solution

Land a maintainer-approved hardening patch that rejects the gateway override server-side while preserving CLI plugin break-glass behavior and keeping protocol docs, clients, and tests aligned.

## Reproduction Assessment

Yes via source reproduction: current main accepts the optional boolean, forwards it to `installSkill`, and the scanner skips critical blocks when it is true. I did not run a live exploit in this read-only review.

## Solution Assessment

Yes in direction: rejecting true at the gateway boundary and always passing false is the narrow maintainable fix. Because this removes a documented protocol override, maintainer/security approval is still the right merge gate.
