## Summary

Keep this PR open: it addresses a real source-visible gateway install-scan bypass and now includes sufficient copied runtime proof, but the patch still leaves the protocol schema/tests and several public docs inconsistent with the new hardening behavior.

## What This Changes

The PR makes gateway `skills.install` reject `dangerouslyForceUnsafeInstall: true`, always forwards `false` to skill installation, stops the UI client from sending the flag, and updates tests, gateway protocol docs, and the changelog.

## Best Possible Solution

Enforce gateway install scans server-side, keep any legacy compatibility explicit in the schema, update every public doc that still advertises the gateway override, and merge only after maintainer approval of the protocol/security compatibility choice.

## Reproduction Assessment

Yes for source reproduction: current main validates and forwards `dangerouslyForceUnsafeInstall` through `skills.install`, and the scanner bypasses critical blocks when that flag is true. I did not run a live exploit, but the source path is direct.

## Solution Assessment

No, not yet. Rejecting `true` in the gateway handler is the right narrow runtime hardening, but the schema/test contract and remaining docs need to match the intended compatibility model before this is the maintainable fix.
