## Summary

Keep open: the patch addresses a real gateway scan-bypass path and I found no blocking correctness finding, but it removes a documented gateway protocol override and should get maintainer/security approval before merge.

## What This Changes

The PR changes gateway `skills.install` to reject `dangerouslyForceUnsafeInstall=true`, stops web UI and macOS clients from sending the field, and updates related docs, tests, and changelog.

## Best Possible Solution

Land a maintainer/security-approved gateway boundary change that rejects the true override server-side, always invokes skill installs with scans enforced, and keeps CLI plugin break-glass behavior separate.

## Reproduction Assessment

Yes. Source inspection on current main shows the schema accepts the optional boolean, the handler forwards it, and the scanner skips critical blocks when the value is true; I did not run a live gateway exploit in this read-only review.

## Solution Assessment

Yes in direction. Rejecting `true` at the gateway boundary and passing `false` to the installer is the narrow maintainable fix, but the protocol/security policy change should be approved by the relevant owners before merge.
