## Summary

Keep open: the gateway hardening direction is sound and source-reproducible, but the current branch drops unrelated current changelog content and removes still-valid private upload install documentation.

## What This Changes

The PR rejects `dangerouslyForceUnsafeInstall=true` for gateway `skills.install`, forces gateway skill installs to pass `false`, stops web/macOS clients from sending the field, and updates tests/docs/changelog.

## Best Possible Solution

Land a scoped gateway hardening that rejects the unsafe override while preserving existing upload-mode docs, current changelog entries, and the CLI plugin break-glass path.

## Reproduction Assessment

Yes. Current main accepts the optional boolean, forwards it from `skills.install`, and the scanner skips critical blocking when it is true; I verified that path by source inspection rather than a live gateway run.

## Solution Assessment

Yes in direction, but not in this branch state. Rejecting `true` at the schema/handler boundary is narrow and maintainable, but the unrelated docs and changelog regressions need to be repaired before owner review.
