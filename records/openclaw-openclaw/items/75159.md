## Summary

Keep this PR open. The Feishu-owned direction is appropriate for the linked topic-reply bug, but the proposed prompt hint is appended for all topic-scoped group sessions and can conflict with automatic source delivery; it also lacks the required user-facing changelog entry.

## What This Changes

The PR adds Feishu topic-scope prompt helpers and tests, then appends a `thread-reply` message-tool hint to `GroupSystemPrompt` for `group_topic` and `group_topic_sender` sessions.

## Best Possible Solution

Keep the fix inside the Feishu plugin, but make the topic-thread guidance compatible with delivery mode and add release-note coverage. The maintainable end state is either a message-tool-only prompt override, conditional wording for cases where the model chooses the message tool, or a Feishu delivery-boundary fix that preserves topic routing without instructing automatic-delivery turns to call the message tool.

## Reproduction Assessment

Yes. The linked bug has a source-level reproduction on current main: topic-scoped Feishu group turns that rely on the message tool are prompted toward `action=send`, while Feishu only uses thread delivery for `action=thread-reply`; the PR’s new regression path is visible by combining automatic group delivery with the unconditional message-tool hint.

## Solution Assessment

No, not as written. The extension-local approach is the right ownership boundary, but the prompt hint must be gated or conditional so it does not contradict automatic final-reply delivery, and the user-facing fix needs a changelog entry.
