## Summary

Keep open. The PR targets a real Feishu-owned prompt bug from #74903, and the extension-local direction is right, but the proposed prompt is unconditional and conflicts with supported automatic/fallback group delivery modes; it also lacks the required user-facing changelog entry.

## What This Changes

The PR adds Feishu topic-scope prompt helpers/tests and appends a thread-reply message-tool hint to group system prompts for `group_topic` and `group_topic_sender` sessions.

## Best Possible Solution

Keep the fix Feishu-owned, but make the topic-thread guidance compatible with delivery mode: either emit it only for message-tool-only turns, or phrase it so automatic/fallback turns still use normal final replies while explicit message-tool sends use `thread-reply` with the topic root. Add focused regression coverage and a changelog fix entry.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: configure a Feishu group topic session with automatic visible replies or a message-tool-unavailable fallback, and the PR’s unconditional hint conflicts with the current-main automatic prompt even though current-main Feishu dispatch already threads the final reply.

## Solution Assessment

No, not as written. The extension-local prompt addendum is the right narrow path for message-tool-only turns, but it needs delivery-mode gating or conditional wording plus changelog coverage before merge.
