## Summary

Keep open: the Feishu-owned prompt approach is the right boundary for the related bug, but this PR conflicts with automatic/fallback delivery modes and is missing required changelog coverage.

## What This Changes

The PR adds Feishu topic-scope prompt helpers/tests and appends a thread-reply message-tool hint for `group_topic` and `group_topic_sender` group sessions.

## Best Possible Solution

Keep the fix Feishu-owned, but make the topic hint delivery-mode aware or conditional on choosing the message tool, then add a focused Feishu changelog fix entry.

## Reproduction Assessment

Yes. A source-level reproduction exists with a Feishu `group_topic` or `group_topic_sender` group using automatic visible replies, or with the message tool unavailable: current main delivers the final reply through the threaded dispatcher, while the PR still tells the model to call the message tool.

## Solution Assessment

No. The extension-local prompt override is maintainable, but the current wording is too broad; conditional wording is safer than duplicating core delivery-mode resolution inside Feishu.
