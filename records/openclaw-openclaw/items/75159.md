## Summary

Keep open. The PR follows the right Feishu-owned direction from #74903 and @zhangkang-wy's analysis, but the current diff appends a message-tool `thread-reply` instruction even in automatic/fallback delivery modes where OpenClaw already sends the final reply to the topic root; it also lacks the required changelog entry.

## What This Changes

The branch adds Feishu helpers/tests that append a topic-thread message-tool hint to group system prompts for `group_topic` and `group_topic_sender` sessions.

## Best Possible Solution

Keep the fix extension-local, but make the topic hint conditional on tool-only delivery or phrase it only as a replacement for the tool-only `action=send` path, then add focused regression coverage and a changelog fix entry.

## Reproduction Assessment

Yes. For the original bug, a Feishu topic-scoped group using tool-only visible replies reaches the shared `action=send` prompt while Feishu only threads `thread-reply`; for the PR regression, automatic/fallback delivery is a supported path and current main already routes final replies to the topic root.

## Solution Assessment

No. The Feishu-owned prompt addendum is the right boundary, but the proposed unconditional wording conflicts with automatic delivery; a delivery-mode-aware hint is the narrower maintainable fix.
