## Summary

Keep open: the same-source transcript mirror is a useful fix for the Telegram/context divergence, but the current patch still has a blocking rich-payload projection bug that can throw after a message is delivered and leave transcript state out of sync.

## What This Changes

The PR adds gateway-side mirroring of successful same-source `message.action` `send` results into the assistant transcript, with direct, Telegram topic, rich-payload, external-target, and failed-send regression coverage.

## Best Possible Solution

Keep the same-source mirror boundary, but project mirror text from normalized presentation and interactive payloads and cover title-only/coerced rich payloads without changing external-target or failed-send behavior.

## Reproduction Assessment

Yes for the patch defect by source inspection: current normalization accepts title-only presentations, while the PR iterates raw `presentation.blocks`. The original source-reply gap has live Telegram after-fix proof in the PR body, but I did not run the flow in this read-only review.

## Solution Assessment

No as currently written. The same-source mirror boundary is the right direction, but rich payload text collection should reuse normalized presentation and interactive values before appending transcript content.
