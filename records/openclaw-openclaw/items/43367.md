## Summary

Keep this open: current main has partial hardening for concurrent agent config mutation and OAuth refresh locking, but session/yield/announce delivery still has source-supported best-effort and retry-limit paths, and related open issues and PRs show the remaining multi-agent reliability work is not complete.

## What This Changes

The issue asks OpenClaw to make parallel multi-agent creation and execution reliable by preventing config lost updates, session-lock cascades, OAuth refresh races, and detached child work.

## Best Possible Solution

Keep the config and OAuth hardening, then review, land, or replace the narrower delivery and lock-lifecycle fixes; keep heavy orchestration or monitoring layers out of core unless a missing core primitive is proven.

## Reproduction Assessment

Partly yes from source: config mutation and OAuth refresh races have current-main fixes, while remaining subagent/session delivery loss is source-supported and backed by production comments but still needs a fresh current-main live repro for the full four-agent workflow.

## Solution Assessment

No single current-main solution fully solves this issue; the best path is narrower maintained fixes for delivery durability and lock lifecycle, not closing the broad report because config/auth fixes landed.
