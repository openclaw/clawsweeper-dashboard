## Summary

Keep open: current main has fixed or reduced some pieces, especially non-interactive `agents add`, OAuth refresh serialization, stale lock handling, and session-lock failover classification, but the reported multi-agent reliability cluster is still not fully resolved because session-store contention and subagent completion-loss/detached-work paths remain active or separately tracked.

## What This Changes

The issue asks OpenClaw to make parallel multi-agent creation and execution reliable by fixing config write races, session-lock failures, OAuth refresh race noise, and detached child work visibility or cleanup.

## Best Possible Solution

Keep this as the umbrella until maintainers either split and close it against narrower canonical items or land focused fixes for session-store contention, subagent delivery failure visibility, and detached-work cleanup semantics.
