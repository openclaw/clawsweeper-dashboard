## Summary

Keep open: current main fixes the concurrent non-interactive `agents add` mutation path and avoids treating local session-lock errors as model fallback, but current/released source plus fresh production comments still show unresolved session write-lock contention, subagent delivery loss, and broader cleanup/orchestration decisions.

## What This Changes

The issue asks OpenClaw to make multi-agent orchestration reliable across concurrent agent creation, parallel agent runs, lock handling, provider fallback noise, and detached child work.

Keep this issue open as the umbrella, treat config mutation and local-lock failover as partial fixes, review or replace the narrow session-lock/subagent-delivery PRs, and split token-ring, cancellation, and operator-visibility policy into maintainer-owned follow-ups.
