## Summary

Keep open. Current main has partially fixed the concurrent config mutation and OAuth refresh-race pieces, but the core multi-agent reliability problem still has source-backed give-up, queue rejection, session-lock, and detached-delivery states, plus open linked repair work that is not a merged canonical fix.

## What This Changes

The issue asks OpenClaw to make multi-agent orchestration reliable across concurrent agent creation, parallel agent runs, provider failover, session locks, and detached child work.

## Best Possible Solution

Adopt a maintainer-approved reliability contract that keeps config mutations serialized, makes subagent completion delivery durable and exactly-once visible or explicitly recoverable, prevents session-lock self-contention, avoids unrelated auth failover noise, and exposes clear cleanup/cancel state for detached child work.

## Reproduction Assessment

Yes at source level, but not as one live current-main run in this review. The issue comments provide concrete commands/logs, and current source still contains bounded delivery give-up and requester wake-up rejection paths that match the remaining reports.

## Solution Assessment

No, closing or treating one partial PR as sufficient is not the best way to solve this issue. The best path is a maintainer-shaped durability, serialization, failover, and cleanup contract with live multi-agent proof across the main affected surfaces.
