## Summary

Keep open. Current main has mitigations for stale plugin dependency state, post-update plugin sync, restart health checks, and stable Homebrew path selection, but the core macOS/Homebrew runtime-ownership problem remains: the Gateway service still composes an external Node runtime with a global CLI entrypoint, and a maintainer explicitly confirmed that broader policy is unresolved.

## What This Changes

Harden macOS/Homebrew updates so the CLI, LaunchAgent runtime, host package, plugin SDK, and plugin dependency state cannot drift after brew or OpenClaw upgrades.

## Best Possible Solution

Choose one macOS Gateway runtime authority and align installer, update, doctor/status, LaunchAgent generation, plugin update cleanup, and user-facing diagnostics around that boundary.

## Reproduction Assessment

No high-confidence deterministic reproduction is available from this checkout. The report includes concrete macOS/Homebrew logs and current source confirms the external-runtime boundary, but reproducing the dylib ABI break requires an affected macOS Homebrew state.

## Solution Assessment

Unclear. Current main's plugin dependency cleanup and update health checks address parts of the drift story, but the remaining runtime ownership choice needs maintainer direction before a narrow implementation is safe.
