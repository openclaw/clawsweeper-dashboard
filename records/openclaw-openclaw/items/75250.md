## Summary

Keep open. Current main addresses legacy plugin dependency staging and Homebrew Cellar path stability, but macOS Gateway still depends on an external Node/global CLI runtime layout; a maintainer explicitly kept that broader runtime-ownership problem unresolved and #76092 remains an open implementation slice.

## What This Changes

The issue requests robust macOS update behavior so Homebrew Node, the global OpenClaw package, LaunchAgent runtime, plugin SDK, and plugin dependency state do not drift after Homebrew or OpenClaw updates.

## Best Possible Solution

Settle macOS Gateway runtime ownership, then make LaunchAgent generation, update, doctor, and diagnostics enforce one runtime/package authority while retaining legacy plugin cleanup as migration support.

## Reproduction Assessment

No. The issue includes concrete macOS/Homebrew logs and current source confirms the external runtime boundary, but the dyld ABI failure requires an affected Homebrew machine state outside this checkout.

## Solution Assessment

Unclear. Current main handles legacy plugin dependency cleanup and Homebrew Cellar path stability, while the remaining bundled-versus-external runtime decision should be resolved by maintainer review of #76092 or a follow-up policy change.
