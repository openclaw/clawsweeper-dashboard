## Summary

Keep open. Current main has handled the stale plugin-runtime-deps/cache portion, but the macOS Gateway still uses an external Node plus global CLI LaunchAgent layout, and a maintainer explicitly kept the broader Homebrew runtime-ownership problem open.

## What This Changes

The issue asks for macOS updates to stop breaking OpenClaw when Homebrew Node, global CLI packages, LaunchAgent runtime paths, and plugin dependency state drift out of sync.

## Best Possible Solution

Maintainers should choose one macOS Gateway runtime authority, then align CLI install/update, launchd ProgramArguments, doctor/status health checks, and plugin dependency cleanup around that contract.

No. The report includes concrete macOS/Homebrew logs and current source confirms the external-runtime boundary, but the missing libsimdjson failure requires an affected Homebrew state not available from this checkout.

## Solution Assessment
