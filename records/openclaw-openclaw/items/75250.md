## Summary

Keep open. Current main mitigates stale plugin dependency state, package/plugin refresh, and service restart health, but the macOS Gateway still relies on an external Node/global CLI LaunchAgent layout; a maintainer explicitly kept the broader Homebrew runtime-ownership problem open and #76092 is still an open unmerged slice.

## What This Changes

The issue asks OpenClaw to survive macOS Homebrew/OpenClaw updates by aligning the Gateway service runtime, CLI package, plugin SDK/dependencies, and stale dependency caches.

## Best Possible Solution

Adopt a single macOS Gateway runtime authority and make installer, update, doctor, LaunchAgent generation, plugin cleanup, and diagnostics enforce that boundary; review #76092 as the first slice if it matches the intended policy.

## Reproduction Assessment

No high-confidence deterministic reproduction is available from this checkout. The issue includes concrete macOS/Homebrew logs and current source confirms the external-runtime boundary, but the native dylib ABI break requires an affected Homebrew machine state.
