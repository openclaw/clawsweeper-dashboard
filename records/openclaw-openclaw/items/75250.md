## Summary

Keep open. Current main has addressed the stale `plugin-runtime-deps` cleanup slice and has Homebrew Cellar path stabilization, but the macOS Gateway still uses an external Node/global CLI service layout, and maintainer discussion explicitly keeps the broader runtime-ownership problem open.

## What This Changes

Prevent macOS Homebrew/OpenClaw updates from breaking Gateway or CLI availability by keeping Node, package entrypoint, plugin SDK, and plugin dependency state coherent.

## Best Possible Solution

Maintainers should choose a single macOS Gateway runtime authority, then align installer, update, doctor/status, LaunchAgent generation, and plugin dependency cleanup around that authority.

## Reproduction Assessment

No. The report has concrete logs and current-main source confirms the external runtime coupling, but reproducing Homebrew dylib ABI churn requires an affected macOS Homebrew/launchd state not available in this checkout.

## Solution Assessment

Unclear. Current-main plugin dependency cleanup is the right direction for stale cache debris, but the broader Homebrew Node/global package ownership problem needs a platform decision before the best implementation is knowable.
