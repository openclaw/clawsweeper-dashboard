## Summary

Keep open. Current main has addressed the legacy plugin-runtime-deps cache mismatch and has Homebrew Cellar path stabilization, but the maintainer discussion and current macOS Gateway docs/source still leave the broader Homebrew runtime-ownership problem unresolved, with #76092 open as an active implementation slice.

## What This Changes

The issue asks OpenClaw to keep macOS Homebrew/OpenClaw updates from breaking CLI and Gateway availability by aligning Node runtime, package, SDK, plugin dependency cache, LaunchAgent, and diagnostics behavior.

## Best Possible Solution

Settle one macOS Gateway runtime authority, then make installer, update, doctor, LaunchAgent generation, plugin cleanup, and diagnostics enforce that boundary while preserving the completed plugin dependency cleanup.
