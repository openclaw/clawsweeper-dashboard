Keep open: current main has partial mitigations for legacy plugin dependency cache cleanup and Homebrew path stability, but a maintainer explicitly kept the broader macOS/Homebrew runtime ownership problem open and current related PRs are still unmerged.

## What This Changes

The issue asks OpenClaw to survive macOS Homebrew/OpenClaw updates by keeping the CLI, gateway service runtime, plugin SDK/dependencies, and LaunchAgent paths from drifting out of sync.

## Best Possible Solution

Settle one macOS gateway runtime authority, then land or replace the focused open PR slices so installer, update, doctor, LaunchAgent generation, plugin cleanup, and diagnostics enforce that boundary consistently.
