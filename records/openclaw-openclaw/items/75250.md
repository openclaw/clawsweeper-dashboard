## Summary

Keep open. Current main has mitigations for stale plugin dependency state, service refresh, stable Homebrew Cellar paths, and restart health diagnostics, but the maintainer discussion and current macOS docs/source still leave the broader Homebrew/runtime ownership problem unresolved.

## What This Changes

The issue requests a robust macOS update/runtime strategy so Homebrew Node, the Gateway LaunchAgent, the global OpenClaw package, plugin SDK files, and plugin dependency state cannot drift after updates.

## Best Possible Solution

Choose one macOS Gateway runtime authority and align installer, update, doctor/status, LaunchAgent generation, plugin cleanup, and user-facing diagnostics around that boundary.

## Reproduction Assessment

No. The report includes concrete macOS/Homebrew logs and current source confirms the external runtime boundary, but reproducing the native library ABI break requires the affected Homebrew state.

## Solution Assessment

Unclear. Current main's cleanup and health checks solve important subparts, but the broader runtime ownership decision needs maintainer direction before a narrow implementation is safe.
