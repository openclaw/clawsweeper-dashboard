## Summary

Keep open. Current main has mitigations for stale plugin dependency state and post-update version/plugin-health checks, but the macOS Gateway still uses an external Node/global CLI LaunchAgent layout, and the maintainer discussion explicitly says the Homebrew runtime-ownership problem remains unresolved.

## What This Changes

The issue reports that macOS/Homebrew updates can leave OpenClaw's CLI, Gateway LaunchAgent runtime, host package, plugin SDK, and plugin dependency cache out of sync.

## Best Possible Solution

Choose one macOS Gateway runtime authority and align installer, update, doctor/status, LaunchAgent generation, plugin update cleanup, and user-facing diagnostics around that boundary.

## Reproduction Assessment

No high-confidence deterministic reproduction is available from this checkout. The report includes concrete macOS/Homebrew logs and current source confirms the external-runtime boundary, but reproducing the dylib ABI break requires an affected macOS Homebrew state.

## Solution Assessment

Unclear. Current main's plugin dependency cleanup and restart health checks address parts of the drift story, but the larger runtime ownership choice needs maintainer direction before a narrow fix is safe.
