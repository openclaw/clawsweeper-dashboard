## Summary

Keep open. Current main has superseded the stale `plugin-runtime-deps` cache part, but the macOS local Gateway still relies on an external Node/global CLI launchd layout, so the Homebrew Node ABI and runtime-ownership problem remains unresolved.

## What This Changes

The issue asks OpenClaw to prevent macOS Homebrew/OpenClaw updates from breaking the CLI, Gateway service runtime, plugin SDK, and plugin dependency state through mixed runtime drift.

## Best Possible Solution

Maintainers should decide the macOS Gateway runtime authority, then align installer, update, doctor/status, LaunchAgent command generation, and package/plugin dependency cleanup around that single authority.

## Reproduction Assessment

No high-confidence deterministic reproduction path is available from this checkout. The report includes concrete macOS/Homebrew logs, and current source/docs confirm the remaining external-runtime boundary, but reproducing the dylib ABI break requires an affected macOS Homebrew state.

## Solution Assessment

Unclear. Current main's plugin dependency rearchitecture is the right direction for the cache/SDK mismatch, but it does not solve the broader Homebrew Node/global package ownership problem.
