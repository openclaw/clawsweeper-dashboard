## Summary

Keep open. Current main resolves much of the legacy plugin-runtime-deps drift through the newer dependency model and cleanup, but macOS local mode still uses an external Node/global CLI launchd service, so the reported Homebrew ABI and runtime authority problem remains.

## What This Changes

The issue reports macOS update failures when Homebrew-managed Node, the global OpenClaw package, the Gateway LaunchAgent, plugin min-host requirements, and legacy plugin dependency caches drift after brew or OpenClaw updates.

## Best Possible Solution

Maintainers should choose the macOS Gateway runtime authority, then align installer, update, doctor/status, LaunchAgent generation, and plugin dependency cleanup around that contract so Node, package root, service command, host version, and plugin install roots are validated as one coherent state.

## Reproduction Assessment

No. The issue has concrete reporter logs that map to current code paths, but a high-confidence reproduction requires a live macOS/Homebrew state with broken Node dylibs plus mixed LaunchAgent/global package versions.

## Solution Assessment

Unclear. Current main's dependency cleanup is the right direction for the stale plugin-runtime-deps portion, but it does not settle the broader external-runtime ownership problem; a narrower code fix should wait for that platform decision.
