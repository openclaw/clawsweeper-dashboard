## Summary

Current main already fixes the reported ACPX timeout failure with a narrower capability-aware implementation, while this PR's universal timeout drop would regress adapters that explicitly advertise timeout support.

## What This Changes

The PR changes `AcpxRuntime.setConfigOption` to return early for `timeout` and `timeout_seconds` for every ACPX backend.

## Best Possible Solution

Keep the current-main capability-aware path: omit timeout for adapters that do not advertise it, preserve it for adapters that do, and retain narrow stale-record guards for known unsupported commands.

## Reproduction Assessment

Yes. Source inspection plus the PR's before/after logs give a high-confidence path: released ACPX forwarded `timeout` to Claude ACP, and Claude ACP rejects unknown config ids.

## Solution Assessment

No. The PR fixes the observed Claude failure, but current main is the better solution because it respects advertised backend capabilities instead of suppressing timeout universally.
