## Summary

## What This Changes

The PR hoists ACPX `timeout`/`timeout_seconds` filtering out of the Codex-only branch and updates runtime tests so non-Codex agents ignore those keys while still forwarding non-timeout options.

## Best Possible Solution

Land the narrow ACPX runtime guard with its regression tests once required checks finish, then track the separately noted configured-timeout plumbing issue outside this PR.

## Reproduction Assessment

Yes, source inspection plus the PR's before/after logs give a high-confidence path: current main forwards `timeout` for a non-Codex ACPX session, `acpx` sends it as `session/set_config_option`, and Claude ACP rejects unknown config ids. I did not run the live Claude setup myself, so this is source-reproducible rather than locally reproduced.

## Solution Assessment

Yes, this is a narrow maintainable fix for the reported breakage because OpenClaw already owns `runtimeOptions.timeoutSeconds` for turn timing and the ACPX wire key is unsupported by the exercised adapter. A broader capability-refinement cleanup can be separate, but it is not needed to stop the failing spawn path.
