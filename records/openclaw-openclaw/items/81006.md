## Summary

Current main already fixes the reported Claude ACP timeout failure with a safer capability-aware path, while this PR is now conflicting and its unconditional all-backend timeout suppression would break backends that explicitly advertise a timeout config alias.

## What This Changes

The PR hoists ACPX `timeout`/`timeout_seconds` suppression ahead of the Codex branch so all ACP agents ignore those keys, and updates focused runtime tests around non-Codex forwarding.

## Best Possible Solution

Keep current main's capability-aware ACP timeout handling and close this conflicting branch rather than merging the unconditional all-backend suppression.

## Reproduction Assessment

Yes. Source inspection plus the PR's before/after logs give a high-confidence path: pre-fix ACPX forwards `timeout` to Claude's `session/set_config_option`, and Claude ACP rejects that unadvertised config ID.

## Solution Assessment

No for this PR branch as it stands. Current main's narrower capability-aware fix is the better solution because it fixes the Claude failure while preserving `timeout` aliases for backends that explicitly advertise them.
