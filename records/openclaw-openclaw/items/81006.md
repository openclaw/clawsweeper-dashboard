## Summary

Close: current main already fixes the ACPX timeout failure with a narrower capability-aware implementation; this PR’s universal timeout drop is no longer the right patch because it would discard timeout controls from backends that explicitly advertise support.

## What This Changes

The PR changes ACPX `setConfigOption` to drop `timeout` and `timeout_seconds` for every ACP backend before delegating config options.

## Best Possible Solution

Keep main’s capability-aware timeout handling: omit unsupported timeout hints, tolerate optional unsupported timeout rejections, and preserve forwarding when a backend advertises timeout support.

## Reproduction Assessment

No high-confidence current-main failure path remains: current source prevents the Claude timeout wire call and tests cover that behavior. The shipped-version failure is well supported by the PR logs and Claude ACP source.

## Solution Assessment

No. The PR fixes the observed Claude failure but does it by suppressing timeout for every ACPX backend; current main’s advertised-capability path is narrower and preserves supported timeout controls.
