## Summary

Current main already fixes the reported ACPX Claude timeout failure with a narrower capability-aware implementation that shipped in v2026.5.12; this PR's remaining universal timeout drop would now remove advertised timeout support from backends that explicitly expose it.

## What This Changes

The PR changes ACPX runtime `setConfigOption` to drop `timeout` and `timeout_seconds` for every ACP backend before forwarding.

## Best Possible Solution

Keep the shipped capability-aware timeout filtering and the later best-effort timeout rejection handling, rather than merging a universal ACPX runtime suppression.

## Reproduction Assessment

No, not against current main: source and tests show the Claude timeout path is already suppressed. The original v2026.5.7 failure remains high-confidence from the PR's live logs and the Claude ACP dependency contract.

## Solution Assessment

No. The PR fixes the observed Claude failure, but the universal early return is broader than the shipped implementation that preserves timeout support for backends that advertise `timeout` or `timeout_seconds`.
