## What This Changes

The PR moves ACPX `timeout`/`timeout_seconds` filtering before the Codex-specific branch and updates runtime tests so non-Codex ACP agents ignore timeout while still forwarding `mode`.

## Best Possible Solution

Land the narrow ACPX runtime guard with its regression tests after normal maintainer validation, then track the separately noted configured-timeout plumbing issue outside this PR.

Yes. Source inspection shows current main forwards `timeout` for non-Codex ACPX sessions, and the PR body provides real before/after logs showing Claude ACP rejects that key before the patch and completes after it; I did not run the live Claude setup myself.

## Solution Assessment

Yes. Hoisting the timeout skip is a narrow maintainable fix because `timeoutSeconds` remains an OpenClaw runtime option for turn timing, while the exercised ACP adapter does not advertise `timeout` as a wire config option and non-timeout keys still forward.
