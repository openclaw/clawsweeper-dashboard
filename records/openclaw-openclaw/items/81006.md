## What This Changes

The PR changes ACPX `setConfigOption` to skip `timeout` and `timeout_seconds` for every ACP backend and updates ACPX runtime tests for timeout suppression plus non-timeout forwarding.

## Best Possible Solution

Keep the merged capability-aware implementation from https://github.com/openclaw/openclaw/pull/80812 and resolve the paired issue https://github.com/openclaw/openclaw/issues/81005 against that current-main fix.

## Reproduction Assessment

Yes. Source inspection plus the PR’s before/after logs and the Claude ACP package source give a high-confidence path: forwarding unadvertised `timeout` reaches `session/set_config_option` and Claude rejects it as an unknown config option.

## Solution Assessment

No. The branch’s blanket ACPX skip is no longer the best solution because current main landed a narrower advertised-key gate that fixes Claude while preserving timeout forwarding for backends that explicitly advertise support.
