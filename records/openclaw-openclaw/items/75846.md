Keep open. Current main still has an assistant-only Claude CLI stream-json parser, so `tool_use` records cannot reach `stream: "tool"` consumers; the matching fix PR is closed unmerged.

## What This Changes

## Best Possible Solution

Add a Claude stream-json tool-event bridge in the CLI parser and thread it through both live-session and one-shot runner paths while preserving the existing `stream: "tool"` contract.

## Solution Assessment

Yes. A parser-level bridge is the narrow maintainable fix because the missing signal originates in the Claude CLI JSONL parser and downstream consumers already handle `stream: "tool"` once emitted.
