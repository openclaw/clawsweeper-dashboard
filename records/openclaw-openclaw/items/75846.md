## Summary

Keep open. Current main still has an assistant-text-only Claude CLI JSONL streaming parser, so Claude `tool_use` stream records cannot reach `stream: "tool"` consumers; the directly related PR #76075 is closed without merge.

## What This Changes

Surface Claude CLI `tool_use` stream-json events through `onAgentEvent` so iMessage and webchat can display in-progress tool activity.

## Best Possible Solution

Add a Claude stream-json tool-event bridge in the CLI parser, wire it through live-session and one-shot runner paths, and preserve the existing `stream: "tool"` contract for downstream consumers.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: feed `createCliJsonlStreamingParser` a Claude `tool_use` `content_block_start` plus `input_json_delta`/`content_block_stop`; current main has no `onToolEvent` callback and returns before emitting anything.

## Solution Assessment

Yes. A parser-level bridge is the narrowest maintainable fix because the missing signal originates in Claude CLI JSONL parsing and downstream surfaces already consume `stream: "tool"` once emitted.
