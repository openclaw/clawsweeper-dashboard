## Summary

Keep open. Current main still has assistant-only Claude CLI JSONL parsing and runner plumbing, while PR #76075 is an open, unmerged closing fix for the same bug.

## What This Changes

The issue asks for Claude CLI stream-json `tool_use` records to be forwarded through `onAgentEvent` as `stream: "tool"` events so iMessage and webchat can show tool activity.

## Best Possible Solution

Review and land PR #76075, or an equivalent parser-level bridge that emits the existing `stream: "tool"` contract from both Claude CLI one-shot and live runner paths.

## Reproduction Assessment

Yes, from source inspection. Feed `createCliJsonlStreamingParser` a Claude stream-json `content_block_start` with `content_block.type: "tool_use"` and an `input_json_delta`; current main has no tool callback, so no `stream: "tool"` event can be produced.

## Solution Assessment

Yes. A parser-level bridge is the narrow maintainable fix because the missing signal originates in the Claude CLI JSONL parser and downstream consumers already understand `stream: "tool"` once emitted.
