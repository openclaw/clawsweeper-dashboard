## Summary

Keep open. Current main still parses Claude CLI stream-json records only into assistant text deltas, so Claude `tool_use` stream events cannot reach the existing OpenClaw `stream: "tool"` event path that webchat and other consumers already rely on.

## What This Changes

Add Claude CLI stream-json `tool_use` handling so downstream `onAgentEvent` subscribers receive `stream: "tool"` events during tool execution.

## Best Possible Solution

Normalize Claude CLI stream-json tool start/input events in the CLI parser and emit the existing generic OpenClaw tool-event contract from both one-shot and live runner paths, without iMessage- or webchat-specific special cases.

## Reproduction Assessment

Yes. Feed `createCliJsonlStreamingParser` a Claude `stream_event` `content_block_start` with `content_block.type: "tool_use"` plus an `input_json_delta`; current main has no tool callback, so no `stream: "tool"` event can be produced.

## Solution Assessment

Yes. A parser-level tool-event bridge is the narrow maintainable fix because the missing signal originates in the Claude CLI JSONL parser and the downstream surfaces already support `stream: "tool"` once emitted.
