## Summary

Keep open. Current main still drops Claude CLI tool-use stream-json records, and open PR #76075 already references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Surface Claude CLI `tool_use` stream-json events as existing `stream: "tool"` agent events for iMessage and webchat consumers.

## Best Possible Solution

Review and land PR #76075 or an equivalent parser-level bridge that normalizes Claude CLI tool-use stream records and emits the existing generic `stream: "tool"` contract from both one-shot and live runner paths.

## Reproduction Assessment

Yes. Feed `createCliJsonlStreamingParser` a Claude `stream_event` `content_block_start` with `content_block.type: "tool_use"` plus an `input_json_delta`; current main has no tool callback, so no `stream: "tool"` event can be produced.

## Solution Assessment

Yes. A parser-level `onToolEvent` bridge threaded through the one-shot and live CLI runner paths is the narrow maintainable fix because downstream consumers already understand `stream: "tool"` once emitted.
