## Summary

Keep open. Current main still drops Claude CLI `tool_use` stream-json records before they can become agent tool events, and open PR #76075 already references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests forwarding Claude CLI `tool_use` stream events as downstream `stream: "tool"` agent events for iMessage and webchat consumers.

## Best Possible Solution

Review and land PR #76075 or an equivalent parser-level bridge that normalizes Claude CLI tool-use stream records and emits the existing generic `stream: "tool"` contract from both one-shot and live runner paths.

## Reproduction Assessment

Yes. Feed `createCliJsonlStreamingParser` a Claude stream-json `content_block_start` with `content_block.type: "tool_use"` followed by an `input_json_delta`; current main has no tool callback, so no `stream: "tool"` event can be produced.

## Solution Assessment

Yes. A parser-level tool-event bridge is the narrow maintainable fix because the missing signal originates in the Claude CLI JSONL parser and downstream consumers already handle `stream: "tool"` once it is emitted.
