## Summary

Keep open. Current main still drops Claude CLI `tool_use` stream-json records before they can become `stream: "tool"` agent events, and the directly related PR #76075 is closed without merge.

## What This Changes

The issue asks Claude CLI stream-json `tool_use` events to be bridged into existing `stream: "tool"` agent events for webchat and iMessage consumers.

## Best Possible Solution

Add a parser-level Claude stream-json tool-event bridge and wire it through both Claude CLI live-session and one-shot runner paths while preserving the existing `stream: "tool"` contract.

## Reproduction Assessment

Yes, from source inspection. Feed `createCliJsonlStreamingParser` a Claude stream-json `content_block_start` tool-use record plus `input_json_delta`; current main has no tool callback, so no `stream: "tool"` event can be emitted.

## Solution Assessment

Yes. A parser-level bridge is the narrow maintainable fix because the missing signal originates in the Claude CLI JSONL parser and downstream consumers already handle `stream: "tool"` once emitted.
