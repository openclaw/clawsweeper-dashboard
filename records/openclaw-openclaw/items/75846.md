## Summary

Keep open. Current main still drops Claude CLI tool-use stream records because the parser and both CLI runner paths only expose assistant text deltas, and PR #76075 is an open unmerged closing fix for the same bug.

## What This Changes

Report asks the Claude CLI JSONL streaming path to surface `tool_use` records as `stream: "tool"` agent events for downstream consumers.

## Best Possible Solution

Review and land PR #76075 or an equivalent parser-level bridge, then let GitHub close this issue after merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: feed `createCliJsonlStreamingParser` a Claude stream-json `tool_use` `content_block_start` plus `input_json_delta`; current main rejects non-text deltas and has no tool callback, so no `stream: "tool"` event can be emitted.

## Solution Assessment

Yes. A parser-level bridge into the existing `stream: "tool"` contract is the narrow maintainable fix because downstream consumers already handle tool events once the Claude CLI runner emits them.
