## Summary

Keep open. Current main still has the Claude CLI stream-json parser wired for assistant text deltas only, so Claude `tool_use` stream records cannot become OpenClaw tool events for WebChat or channel monitors even though the downstream tool-event contract already exists.

## What This Changes

The issue requests surfacing Claude CLI `tool_use` stream-json events through `onAgentEvent` as live tool activity for iMessage and WebChat consumers.

## Best Possible Solution

Add a generic Claude stream-json tool-event normalization path in the CLI parser and wire both one-shot and live Claude CLI runner paths to emit the existing OpenClaw `stream: "tool"` contract, without iMessage- or WebChat-specific branching.

## Reproduction Assessment

Yes. Feed `createCliJsonlStreamingParser` a Claude `stream_event` for `content_block_start` with `content_block.type: "tool_use"` plus `input_json_delta`; current main has no tool callback and only emits assistant text deltas.

## Solution Assessment

Yes. A parser-level bridge is the narrowest maintainable fix because the missing signal originates in Claude CLI JSONL normalization, and downstream surfaces already handle `stream: "tool"` once it is emitted with `toolCallId`.
