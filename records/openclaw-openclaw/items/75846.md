## Summary

Keep open. Current main still has an assistant-text-only Claude CLI JSONL streaming path, so Claude stream-json tool-use records cannot become `stream: "tool"` agent events; the directly related implementation PR is closed without merge.

## What This Changes

The issue requests that Claude CLI stream-json `tool_use` activity be surfaced through `onAgentEvent` as tool-stream events for downstream consumers such as iMessage and webchat.

## Best Possible Solution

Add a parser-level Claude stream-json tool-event bridge and wire it through both live and one-shot CLI runner paths, emitting sanitized events compatible with the existing `stream: "tool"` consumer contract.

## Reproduction Assessment

Yes, from source inspection. Feeding current main's `createCliJsonlStreamingParser` a Claude stream-json `content_block_start` tool-use record plus `input_json_delta` has no current callback or runner forwarding path, so no `stream: "tool"` event can be emitted.

## Solution Assessment

Yes. An optional parser callback wired into both Claude CLI runner paths is the narrow maintainable fix because downstream consumers already handle the generic tool-stream contract once events are emitted.
