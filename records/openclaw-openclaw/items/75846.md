## Summary

Keep open. Current main still drops Claude CLI `tool_use` stream-json records because the parser and both CLI runner paths only surface assistant text deltas, and open PR #76075 is an unmerged closing fix for the same bug.

## What This Changes

The issue asks for Claude CLI `tool_use` stream-json records to be bridged through `onAgentEvent` as `stream: "tool"` so iMessage and webchat can show tool activity.

## Best Possible Solution

Review and land PR #76075 or an equivalent parser-level bridge that emits the existing `stream: "tool"` contract from both Claude CLI one-shot and live runner paths.

## Reproduction Assessment

Yes. Feed `createCliJsonlStreamingParser` a Claude stream-json `content_block_start` with `content_block.type: "tool_use"` and an `input_json_delta`; current main has no tool callback, so no `stream: "tool"` event can be produced.

## Solution Assessment

Yes. A parser-level bridge is the narrow maintainable fix because the missing signal originates in the Claude CLI JSONL parser and downstream consumers already understand `stream: "tool"` once emitted.
