## Summary

Keep open. Current main has a real gap: the Claude CLI JSONL streaming parser only projects text deltas into agent events, while the embedded Pi path already emits `stream: "tool"` events and the gateway has consumers for those events. This is narrow enough for a focused fix PR rather than cleanup closure.

## What This Changes

Requests that Claude CLI `tool_use` JSONL events be surfaced as `stream:tool` agent events so iMessage and WebChat can show tool activity.

## Best Possible Solution

Add a shared Claude stream-json tool-event projection in the CLI backend parser, use it from both one-shot and live stdio execution paths, emit the current OpenClaw `stream: "tool"` contract, and cover it with focused parser plus runner tests and a changelog fix entry.

## Reproduction Assessment

Yes. A high-confidence synthetic reproduction is to feed a Claude stream-json tool-use record through the existing `executePreparedCliRun` stdout harness while subscribed to `onAgentEvent`; current code will emit assistant text deltas but has no parser branch or callback for tool events.

## Solution Assessment

Yes, the suggested direction is the narrow maintainable fix, with one adjustment: match the current embedded Pi `stream: "tool"` event shape (`phase`, `name`, `args`, `toolCallId`) and add any `callId` alias only if consumers still need it.
