## Summary

Keep open. Current main still drops Claude CLI stream-json tool-use records because the parser only forwards assistant text deltas, and the related fix PR #76075 is closed without merge.

## What This Changes

Surface Claude CLI `tool_use` stream-json events through `onAgentEvent` so iMessage, webchat, and other `stream: "tool"` consumers see tool activity.

## Best Possible Solution

Add a Claude stream-json tool-event bridge in the parser and wire both one-shot and live-session CLI paths to emit the existing `stream: "tool"` contract, with focused regression tests and a changelog entry.

## Reproduction Assessment

Yes, from source inspection. Feed `createCliJsonlStreamingParser` a Claude stream-json `content_block_start` tool-use record followed by `input_json_delta`; current main has no tool callback, so no `stream: "tool"` event can be emitted.

## Solution Assessment

Yes. A parser-level bridge is the narrow maintainable fix because the missing signal originates in the Claude CLI JSONL parser and downstream consumers already handle `stream: "tool"` once emitted.
