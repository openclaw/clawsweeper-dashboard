## Summary

Keep open. Current main still drops Claude CLI tool_use JSONL records before they can become stream: "tool" agent events, and open PR #76075 references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests forwarding Claude CLI tool_use JSONL stream events through the existing onAgentEvent stream: "tool" contract for webchat and iMessage consumers.

## Best Possible Solution

Review and land PR #76075, or an equivalent narrow parser-level bridge that emits the existing stream: "tool" contract from both one-shot and live Claude CLI runner paths.

## Reproduction Assessment

Yes. Feed createCliJsonlStreamingParser a Claude stream_event content_block_start with content_block.type: "tool_use" and a content_block_delta input_json_delta; current main has no onToolEvent path and only forwards text_delta as assistant output.

## Solution Assessment

Yes. A parser-level tool-event bridge is the narrow maintainable fix because the missing signal originates in the Claude CLI JSONL parser, while downstream webchat/iMessage routing already handles stream: "tool" once emitted.
