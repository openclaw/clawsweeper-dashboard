## Summary

Keep open. Current main still has an assistant-only Claude CLI stream-json parser and both CLI runner paths only forward assistant deltas, while the existing agent stream contract and embedded path already support `stream: "tool"`; the directly related fix PR is closed without merge.

## What This Changes

The issue asks for Claude CLI `tool_use` JSONL events to be forwarded as tool stream agent events for downstream consumers.

## Best Possible Solution

Add a Claude stream-json tool-event bridge at the parser layer and wire it through both one-shot and live-session CLI runner paths while preserving the existing `stream: "tool"` contract.

## Reproduction Assessment

Yes, from source inspection. Feed `createCliJsonlStreamingParser` a Claude `content_block_start` tool-use record plus `input_json_delta`; current main has no tool callback, so no `stream: "tool"` event can be emitted.

## Solution Assessment

Yes. A parser-level bridge is the narrow maintainable fix because the missing signal originates in the Claude CLI JSONL parser and downstream consumers already handle `stream: "tool"` once emitted.
