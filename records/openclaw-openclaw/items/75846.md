## Summary

Keep open. Current main still ignores Claude CLI `tool_use` stream-json events, and the linked open PR #76075 references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Report asks for the Claude CLI JSONL backend to forward `tool_use` stream records through `onAgentEvent` as OpenClaw `stream: "tool"` events for iMessage and webchat.

## Best Possible Solution

Review and land PR #76075 or an equivalent parser-level bridge that normalizes Claude CLI tool-use stream records and emits the existing generic `stream: "tool"` contract from both one-shot and live runner paths.

## Reproduction Assessment

Yes. Feed `createCliJsonlStreamingParser` a Claude `stream_event` `content_block_start` with `content_block.type: "tool_use"` plus an `input_json_delta`; current main has no tool callback, so no `stream: "tool"` event can be produced.

## Solution Assessment

Yes. A parser-level tool-event bridge is the narrow maintainable fix because the missing signal originates in the Claude CLI JSONL parser and downstream consumers already handle `stream: "tool"` once emitted.
