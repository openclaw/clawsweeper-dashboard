## Summary

Keep open. Current main still has an assistant-text-only Claude CLI JSONL streaming path, so Claude stream-json `tool_use` records cannot become `stream: "tool"` agent events; the directly related implementation PR is closed without merge.

## What This Changes

Surface Claude CLI stream-json `tool_use` activity as existing `stream: "tool"` agent events for downstream webchat and iMessage consumers.

## Best Possible Solution

Add a Claude stream-json tool-event bridge in `createCliJsonlStreamingParser` and wire it through both live and one-shot CLI runner paths, emitting sanitized events compatible with the existing `stream: "tool"` consumer contract.

## Reproduction Assessment

Yes, from source inspection. Feeding `createCliJsonlStreamingParser` a Claude stream-json `content_block_start` tool-use record plus `input_json_delta` has no current-main tool callback or runner forwarding path, so no `stream: "tool"` event can be emitted.

## Solution Assessment

Yes. A parser-level optional tool-event callback wired into both Claude CLI runner paths is the narrow maintainable fix because downstream consumers already handle `stream: "tool"` once emitted.
