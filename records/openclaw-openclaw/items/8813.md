## Summary

Keep open: current main and the latest stable release still emit read-based SKILL.md loading guidance even when tool names are already available to prompt construction, and the remaining question is the product/plugin contract for a registered skill-loader tool.

## What This Changes

The issue requests changing the Skills system prompt so a registered `Skill` loader tool is preferred over `read` for loading matching SKILL.md instructions.

## Best Possible Solution

Define the supported skill-loader tool contract, then update `src/agents/system-prompt.ts`, focused prompt tests, and system-prompt docs while preserving the current read fallback.

## Reproduction Assessment

Yes. Source inspection shows current main and `v2026.6.10` unconditionally emit read-based skill-loading guidance even though registered tool names are available to prompt construction.

## Solution Assessment

Unclear as proposed. Preferring a registered loader tool is plausible, but maintainers should first choose whether the contract is name-based, namespaced, or an explicit skill-loader capability.
