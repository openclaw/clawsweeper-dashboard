## Summary

Keep open: current main still emits read-based skill-loading instructions and the Skills prompt cannot branch on a registered `Skill` loader tool; the related prompt PR remains open and adjacent but does not implement this contract.

## What This Changes

Request that the system prompt prefer a registered `Skill` loader tool over `read` when loading matching `SKILL.md` instructions.

## Best Possible Solution

Define the Skill-loader contract, then update `src/agents/system-prompt.ts`, the skills formatter text, focused prompt tests, and public system-prompt docs so registered loader tools are preferred while the existing read fallback remains intact.

## Reproduction Assessment

Yes for the current prompt behavior: source inspection on current main shows the Skills section always emits read-based loading and cannot see a registered `Skill` tool. No runtime test was run because this was a read-only review.

## Solution Assessment

Unclear until maintainers choose the contract. A normalized tool-name branch is narrow, but an explicit skill-loader capability may be safer and the formatter-level read instructions need to stay consistent.
