## Summary

Keep open. Current main still emits read-based Skills guidance even though runtime tool names can reach prompt construction, and the adjacent open #63940 does not resolve the requested Skill-tool preference.

## What This Changes

Request the Skills system prompt to invoke a registered `Skill` tool when present instead of always instructing the model to read `SKILL.md` files.

## Best Possible Solution

Add an approved Skill-tool signal, pass it through prompt building, emit non-contradictory Skill-tool loading guidance in both Skills section and skills catalog formats, preserve the read fallback, and update focused tests plus system-prompt docs.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: call `buildAgentSystemPrompt` with `skillsPrompt` and `toolNames` containing `Skill`; current main computes `availableTools` but the Skills section and catalog still say to use `read`. I did not run a live test in this read-only review.

## Solution Assessment

Unclear pending maintainer choice on the detection contract. The narrow implementation belongs around `src/agents/system-prompt.ts` and the skills formatters, with tests and docs updated after the approved signal is chosen.
