## Summary

Keep open: current main still emits read-based skill-loading instructions and does not give the Skills section access to a registered `Skill` loader tool; the related prompt PR at https://github.com/openclaw/openclaw/pull/63940 is still open and remains read-based.

## What This Changes

Request system-prompt skill loading to invoke a registered `Skill` loader tool when present instead of always instructing models to read SKILL.md.

## Best Possible Solution

Define the Skill-loader contract, then update `src/agents/system-prompt.ts`, skill formatter text, focused prompt tests, and public system-prompt docs so registered loader tools are preferred while the read fallback remains intact.

## Reproduction Assessment

Yes for the current prompt behavior: source inspection on current main shows the Skills section always emits read-based loading and cannot see a registered `Skill` tool. No runtime test was run because this was a read-only review.

## Solution Assessment

Unclear until maintainers choose the contract. A normalized tool-name branch is narrow, but an explicit skill-loader capability may be safer and the related prompt PR should be reconciled first.
