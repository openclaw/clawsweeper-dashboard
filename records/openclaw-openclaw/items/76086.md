## Summary

Keep open. The cooldown direction matches the linked heartbeat-flood reports, but the patch regresses the deprecated public plugin `runtime.system.requestHeartbeatNow()` alias by making old calls cooled-down event wakes rather than immediate wakes; the protected `maintainer` label also rules out cleanup closure.

## What This Changes

The PR replaces raw heartbeat wake reasons with typed `source`/`intent` wake requests, centralizes cooldown and flood checks, and updates cron, gateway, agent, plugin runtime, docs, changelog, and heartbeat tests.

## Best Possible Solution

Keep the typed wake/cooldown design, but preserve `runtime.system.requestHeartbeatNow()` as an immediate compatibility alias by default and use the new structured `requestHeartbeat({ intent: "event" })` path for cooled-down plugin events.

## Reproduction Assessment

Yes. A focused unit test can start a heartbeat runner, record one successful run so `nextDueMs` is in the future, call `createPluginRuntime().system.requestHeartbeatNow()` with no `intent`, and verify the PR currently routes it as an `event` wake that is skipped as `not-due` rather than an immediate legacy wake.

## Solution Assessment

No. Centralizing cooldown with typed wake intent is the right core direction, but making the deprecated no-opts alias default to `intent: "event"` is not the narrowest compatible fix; the safer alternative is to default that alias to immediate behavior and require new callers to opt into cooled event intent explicitly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve requestHeartbeatNow as an immediate alias:** `src/plugins/runtime/runtime-system.ts:20-21`
  - body: `runtime.system.requestHeartbeatNow()` is a documented/deprecated plugin API for waking after a plugin enqueues a system event, but this wrapper now defaults old calls to `intent: "event"`. After any recent heartbeat, the new cooldown path returns `not-due`, so existing plugins can stop delivering their queued event immediately and wait until the next interval. Keep the compatibility alias immediate by default and let new `requestHeartbeat({ intent: "event" })` callers opt into cooldown.
  - confidence: 0.89
