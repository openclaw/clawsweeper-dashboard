Keep open. Current main contains useful mitigations for restart deferral, event-loop diagnostics, local log fallback, plugin-registry/session-list hot paths, and Discord status visibility, but I do not see proof that the reported CPU-hot Gateway event-loop starvation is gone; `/readyz` is still served inside the same Gateway Node process.
