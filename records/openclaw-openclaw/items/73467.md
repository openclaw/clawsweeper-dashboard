Current main has meaningful mitigations for readiness diagnostics, restart deferral, task reconciliation, log-follow fallback, and plugin tool hot paths, but it does not prove the reported CPU-hot Gateway event-loop starvation is gone; `/readyz` still executes on the same Gateway event loop and the broader control-plane separation remains open in #42026.
