Keep open. ## What This Changes

The issue reports OpenClaw 2026.4.26 Gateway event-loop starvation under multi-agent orchestration, causing `/readyz` timeouts, delayed Discord/WebUI control traffic, `logs --follow` exits, and reload deferral behind active-run accounting.

## Best Possible Solution

Keep this as a Gateway reliability umbrella until maintainers capture current-main profiles under the reported multi-agent Discord/WebUI/memory load, then split confirmed hot paths or lifecycle-accounting defects into narrow follow-ups.

## Reproduction Assessment

No high-confidence exact current-main reproduction is established. The source path for `/readyz` starvation is clear because readiness is still served by the Gateway event loop, but the current-main workload that makes the main thread CPU-hot is not identified.

## Solution Assessment

Not applicable as a patch review. The current mitigations are useful but not terminal; the maintainable path is profile-driven removal, bounding, or offloading of confirmed hot paths plus validation of restart/task accounting.
