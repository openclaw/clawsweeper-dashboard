## What This Changes

The issue asks OpenClaw to make Slack message-send outcomes retry-safe by returning receipts for confirmed sends, hard errors for definite failures, and explicit unknown/reconciliation state for ambiguous sends.

## Best Possible Solution

Define one Slack message-tool outcome contract: confirmed sends return receipts, definite pre-accept failures return hard errors, and post-accept unknowns surface a retry-unsafe state backed by Slack reconciliation or idempotency.

## Solution Assessment
