## What This Changes

The PR changes update/doctor package handoff logic, ClawHub-to-npm fallback gating, managed package root resolution, tests, and changelog.

## Best Possible Solution

Fix the legacy metadata path so configs that previously had no `lastTouchedVersion` remain old-parent-readable, then attach current-head beta.4 package-upgrade proof before merge.

## Reproduction Assessment

Yes, source-level with strong external proof: the linked issue describes the beta4-to-beta5 package-upgrade path, and current source plus the shipped v2026.5.12 env shape match the reported old-parent handoff failure. I did not rerun the Docker Package Acceptance lane in this read-only review.

## Solution Assessment

No. The overall direction is right, but this head still lets a legacy writable parent path write candidate-version metadata when the source config had no metadata, so the narrow fix is to preserve the source metadata state instead of falling back to `VERSION`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

- **[P1] Preserve missing legacy config metadata:** `src/flows/doctor-health-contributions.ts:342-345`

Preserve the source metadata absence or use an explicitly old-parent-readable value instead.
