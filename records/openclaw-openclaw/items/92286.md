## Summary

The PR has strong live proof and uses the existing transcript rewrite owner boundary, but it rewrites all thinking-bearing messages on the durable active branch rather than the provider replay context that was actually rejected. Because this is persisted session-state behavior and the PR carries the protected `maintainer` label, it should remain open for an explicit boundary decision and a compacted-history check.

## What This Changes

The PR adds successful-retry notification and durable active-branch transcript repair so rejected Anthropic thinking replay does not poison subsequent turns.

## Best Possible Solution

Keep the existing successful-retry gate and transcript rewrite mechanism, but explicitly define and test whether durable cleanup should mirror only the rejected replay context or intentionally sanitize the full active branch, including a compacted-session case.

Yes. The PR body provides a live Anthropic reproduction that captures a valid signed-thinking message, corrupts its signature, observes provider rejection, runs the patched recovery, and verifies a successful subsequent replay.

## Solution Assessment

Unclear. Reusing the existing transcript rewrite and session refresh paths is maintainable, but maintainers must confirm whether full active-branch sanitation is the intended policy or narrow it to the rejected replay context.
