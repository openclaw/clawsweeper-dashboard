The PR should remain open for normal maintainer merge judgment because it deliberately rewrites persisted session history and carries the protected `maintainer` label, not because an actionable defect was found.

## What This Changes

The PR adds a successful-retry callback that strips thinking blocks from the persisted active session branch after Anthropic or Bedrock rejects signed-thinking replay, then refreshes lock and in-memory session state.

## Best Possible Solution

Land the narrowly gated durable repair through the existing transcript rewrite path so provider-rejected replay state is removed permanently while accepted signed-thinking history remains unchanged.
