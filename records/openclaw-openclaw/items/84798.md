Keep open for the normal PR merge path: the patch is a narrow, source-backed fix for a nested retry loop with no blocking code-review findings, but it intentionally changes runtime retry ownership so exact-head gates and maintainer acceptance should control merge.

## What This Changes

This PR disables pi-coding-agent agent-level auto-retry in prepared embedded Pi settings, updates the corresponding settings test, and adds an Unreleased changelog fix entry.

## Best Possible Solution

Keep the retry ownership singular: Pi SDK agent-level retry stays disabled for OpenClaw embedded runs, while OpenClaw's own bounded failover/retry layer remains responsible for recovery.

## Reproduction Assessment

Yes, source-reproducible: current main prepares Pi settings without disabling retry, while pi-coding-agent 0.75.3 defaults retry.enabled to true and retries by removing the assistant error before continuing. I did not run a live Feishu/Qwen loop in this read-only review.

## Solution Assessment

Yes. A runtime-scoped setRetryEnabled(false) at the prepared settings seam is a narrow maintainable fix because it preserves on-disk user settings and avoids duplicating toggles at each embedded run call site.
