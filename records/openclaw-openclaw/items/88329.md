Full review comments:

- **[P1] Expand quota fallbacks before cooldown skips:** `src/agents/model-fallback.ts:1513`
  - body: The new quota fallback chain is only added after a primary provider attempt fails. When the pinned provider is already in persisted cooldown, the loop takes the cooldown precheck path, records `Provider ... is in cooldown (suspending lanes)`, and can exhaust the single candidate before reaching this branch, so the repeated `All models failed (1)` case reported in the PR comments remains broken. Build the quota candidates before `resolveCooldownDecision()` or make that decision path aware of the quota fallback list.
  - confidence: 0.88
- **[P1] Match current Codex usage-limit messages:** `src/agents/embedded-agent-helpers/failover-matches.ts:10`
  - body: The new Codex matcher only accepts `you have hit your ... usage limit`; it misses OpenClaw's current `You've reached your Codex subscription usage limit.` wrapper and upstream Codex's `The usage limit has been reached` / `You've hit your usage limit` forms. Pinned Codex subscription-limit runs can still keep an empty chain and fail despite the docs promising subscription-limit recovery.
  - confidence: 0.9
