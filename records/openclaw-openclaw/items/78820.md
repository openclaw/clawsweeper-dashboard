## Summary

Keep open: current main still performs the package swap inside `update.run` before scheduling the live Gateway restart, so the reported transient broken `dist` tree remains plausible and source-backed. The related stale-module reports overlap but do not supersede this control-plane `update.run` race.

## What This Changes

The issue asks for `update.run` to avoid leaving npm global `dist` modules missing during an in-place package update.

## Best Possible Solution

Package-manager `update.run` should avoid mutating the live package tree from inside the Gateway, either by handing off to an updater outside the running process or by failing closed with clear `openclaw update` guidance.

## Reproduction Assessment

No, not as a high-confidence live reproduction. The issue provides concrete v2026.5.6 failure details, and source inspection shows the package swap happens before the `update.run` restart is scheduled, but no disposable install was mutated to catch the race live.

## Solution Assessment

Unclear. A pure temp-directory swap alone does not settle the live-process hazard; the maintainable direction is either an explicit outside-Gateway handoff or a fail-closed package-update path for control-plane `update.run`.
