## Summary

Keep open: current main still uses session attention age for embedded-run active-abort eligibility, while this PR switches that decision to embedded-run progress age and adds focused coverage. The patch looks correct for the narrow bug, but it changes a session recovery predicate that can either abort active embedded work or leave wedged sessions unrecovered, so it still needs maintainer reconciliation with the adjacent queued/terminal recovery items before merge.

## What This Changes

The PR changes diagnostic embedded-run active-abort eligibility to use the active run's last progress age instead of session activity age and adds regression tests for refreshed queued activity and recent embedded-run progress.

## Best Possible Solution

Land the progress-age predicate with its focused tests once maintainers confirm it is compatible with queued-work and terminal-progress recovery policy, then require exact-head checks to pass.
