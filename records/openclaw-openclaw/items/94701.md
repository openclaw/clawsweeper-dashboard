## Summary

Keep open: current main still gates embedded-run active-abort recovery on refreshed session activity age, while this PR changes the decision to use active run progress age and adds focused regression coverage. The patch looks correct, but the recovery predicate is session-state and availability sensitive, so maintainers should reconcile it with adjacent queued/terminal embedded-run recovery work before merge.

## What This Changes

The PR changes diagnostic embedded-run active-abort eligibility to use the active run's last progress age instead of session activity age and adds regression tests for refreshed queued activity and recent embedded-run progress.

## Best Possible Solution

Land the progress-age predicate with the focused tests once maintainers confirm it is compatible with the queued-work and terminal-progress recovery policy, then require exact-head checks to pass.
