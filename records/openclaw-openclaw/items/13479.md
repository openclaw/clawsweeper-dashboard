## Summary

Keep open. Current main still carries cron descriptions through the model, edit, submit, RPC, and search paths, but the cron dashboard renderer does not read or render `job.description`; searches did not find a canonical duplicate or open PR that owns this missing display.

## What This Changes

Show each cron job’s prompt/task and human-readable summary in the Web UI cron dashboard.

## Best Possible Solution

Render the stored `CronJob.description` as a visible summary or description in the Control UI cron job card or selected-job detail path, reusing the existing cron storage/RPC contract and adding focused view coverage.

## Reproduction Assessment

Yes, at source level. A cron job with a non-empty `description` is carried through types, edit/submit plumbing, and backend search, but `renderJob`/`renderJobPayload` in current main never read that field.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation update with tests; storage, RPC, and cron service behavior already support the field.
