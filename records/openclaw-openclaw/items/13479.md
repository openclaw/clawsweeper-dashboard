## Summary

Keep open. Current main still carries cron descriptions through the contract, storage, edit form, and search, but the Control UI cron dashboard card/detail renderer only shows the schedule, prompt/system payload, delivery, and actions; it never renders the saved description as the requested human-readable summary.

## What This Changes

The issue requests showing each cron job’s prompt/task and human-readable summary from the Web UI dashboard.

## Best Possible Solution

Render `CronJob.description` as a visible Summary or Description field in the Control UI cron job card or selected-job detail area for both `systemEvent` and `agentTurn` jobs, without changing the cron storage/RPC contract or prompt rendering.

## Reproduction Assessment

Yes, at source level. A cron job with a non-empty `description` is accepted, saved, edited, and searchable, but the current `renderJob`/`renderJobPayload` display path does not read that field.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation update with focused view coverage because the backend contract and edit plumbing already carry the field.
