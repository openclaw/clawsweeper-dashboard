## Summary

Keep open. Current main carries cron descriptions through storage, editing, and search, and it renders the prompt/task payload, but the cron job card/detail renderer still never displays the saved description as the requested human-readable summary.

## What This Changes

Show each Web UI cron job’s prompt/task and human-readable summary from the cron dashboard without requiring users to inspect config files.

## Best Possible Solution

Render stored CronJob.description as a visible Summary in the cron job card or selected-job detail area, while leaving cron storage, RPC, search, and prompt rendering contracts intact.

## Reproduction Assessment

Yes, at source level. A cron job with a non-empty description is supported by the type, edit form, submit payload, and search path, but renderJob/renderJobPayload never display that field.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation update with focused view coverage; storage, RPC, and payload rendering already provide the needed data.
