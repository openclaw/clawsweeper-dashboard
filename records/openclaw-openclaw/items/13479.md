## Summary

Keep open. Current `origin/main` still carries cron descriptions through storage, edit, submit, and search paths, but the Control UI cron dashboard renderer does not display the saved description as the requested human-readable summary.

## What This Changes

Show each scheduled cron job’s prompt/task and saved human-readable summary in the Web UI cron dashboard so users can inspect jobs without opening config files.

## Best Possible Solution

Render `CronJob.description` as a visible Summary or Description field in the Control UI cron job card or selected-job detail area for both `systemEvent` and `agentTurn` jobs, using the existing cron storage/RPC contract.

## Reproduction Assessment

Yes, at source level. A cron job with a non-empty description flows through the model, edit form, submit payload, and backend search, but the current `renderJob`/`renderJobPayload` dashboard path does not read `job.description`.

## Solution Assessment

Yes. The narrowest maintainable fix is a Control UI presentation update with focused view coverage, because the backend and edit plumbing already carry the field and the prompt/task display already exists.
