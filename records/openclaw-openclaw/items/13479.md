## Summary

Keep open. Current main already stores, edits, searches, and exposes cron descriptions, but the Control UI job card/detail renderer still never reads `job.description`, so the requested summary is still missing from the dashboard.

## What This Changes

Show the cron job prompt/task and saved human-readable description or summary in the Control UI cron dashboard.

## Best Possible Solution

Render stored `CronJob.description` as a visible summary in the Control UI cron job card or selected-job detail area, with focused view coverage and no cron storage/RPC contract changes.

## Reproduction Assessment

Yes, at source level. A cron job with a non-empty `description` is carried through the type, edit form, submit payload, backend search, and SDK mapping, but `renderJob`/`renderJobPayload` never render that field.

## Solution Assessment

Yes. The narrowest maintainable path is a Control UI presentation update with tests; storage, RPC, and cron service behavior already support the field.
