## Summary

Keep open. Current main carries cron descriptions through the shared type, edit/save path, and search, and it renders the prompt/system payload, but the Control UI job card/detail path still does not render the saved description as the requested summary.

## What This Changes

Request to let users view a cron job in the Control UI and see its prompt/task plus a human-readable summary or description.

## Best Possible Solution

Render CronJob.description as a visible Summary or Description field in the Control UI job card or selected-job detail area for both systemEvent and agentTurn jobs, leaving the existing cron storage/RPC contract and payload rendering intact.

## Reproduction Assessment

Yes, at source level. A cron job with a non-empty description is accepted, saved, edited, and searchable, but the current renderJob/renderJobPayload display path does not read that field.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation update with focused view coverage because backend and edit plumbing already carry the field.
