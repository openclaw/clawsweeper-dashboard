## Summary

Keep open. Current main still carries cron descriptions through the type, storage, edit form, and search metadata, but the Control UI job card/detail renderer shows only schedule, payload prompt/system text, delivery, and state metadata, so the requested human-readable summary is still missing.

## What This Changes

Show each cron job's prompt/task and human-readable description summary in the Web UI dashboard/detail path.

## Best Possible Solution

Render non-empty CronJob.description as a visible Summary in the Control UI cron job card or selected-job detail area for both system-event and agent-turn jobs without changing cron storage or RPC contracts.

## Reproduction Assessment

Yes, at source level. A cron job can carry a non-empty description through the contract, storage, edit/save path, and search metadata, but the current job card and payload renderer do not display that field.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation update because the backend cron contract and edit/save plumbing already carry the summary text.
