## Summary

Keep open. Current main still stores and edits cron job descriptions and renders the prompt/task payload in the Control UI job card, but it does not visibly render the saved description as the requested human-readable summary. The issue remains narrow, valid, and suitable for a focused UI repair.

## What This Changes

Expose each cron job's prompt/task and human-readable description or summary in the Web UI cron dashboard so users can inspect scheduled tasks without reading config files.

## Best Possible Solution

Add a visible Summary or Description field to the Control UI cron job card or selected-job detail area for both systemEvent and agentTurn jobs, using the existing CronJob.description value while leaving cron storage, RPC schemas, payload rendering, and run-history selection semantics intact.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: render a CronJob with a non-empty description; current renderJob calls renderJobPayload, and neither path reads or renders job.description.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation change with focused view tests; the current storage, Gateway protocol, and edit form plumbing already support the field.
