## Summary

Keep open. Current main already stores, edits, submits, and searches cron job descriptions, and it renders the prompt/task payload in the cron job card, but the Control UI still does not visibly render `job.description` as the requested human-readable summary.

## What This Changes

Show each cron job's prompt/task and human-readable summary in the Web UI cron dashboard.

## Best Possible Solution

Render stored `CronJob.description` as a visible Summary in the cron job card or selected-job detail area while leaving cron storage, RPC, search, and prompt rendering contracts intact.

## Reproduction Assessment

Yes, at source level. A cron job with a non-empty description is supported by the type, edit form, submit payload, and search path, but `renderJob` and `renderJobPayload` never display that field.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation update with focused view coverage; storage, RPC, and payload rendering already provide the needed data.
