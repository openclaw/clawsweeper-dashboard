## Summary

Keep open. Current main still stores, searches, and edits cron job descriptions and renders the prompt/task payload, but the main Control UI cron job card does not visibly render the saved description as the requested human-readable summary.

## What This Changes

Show each cron job's prompt/task and saved human-readable summary from the Web UI cron dashboard without inspecting config files.

## Best Possible Solution

Render saved CronJob.description as a visible Summary or Description field in the Control UI cron job card or selected-job detail area for both systemEvent and agentTurn jobs.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: render a CronJob with a non-empty description; current renderJob calls renderJobPayload, and neither path reads or displays job.description.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation repair with focused view tests because storage, search, edit form, and cron RPC plumbing already carry the field.
