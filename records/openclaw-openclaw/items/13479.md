## Summary

Keep open. Current main still supports cron job descriptions in the contract, edit/save flow, and filtering metadata, but the Web UI job card/detail renderer still does not display the saved description as the requested human-readable summary.

## What This Changes

Request to show a cron job's prompt/task and human-readable summary from the Web UI cron dashboard instead of requiring users to inspect configuration files.

## Best Possible Solution

Render non-empty CronJob.description as a visible Summary in the Control UI cron job card or selected-job detail area for both system-event and agent-turn jobs, without changing cron storage or RPC contracts.

## Reproduction Assessment

Yes, at source level. A cron job with a non-empty description is supported by the type, form submit path, and filter metadata, but the current job renderer and payload renderer do not display that field.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation update because the backend contract and edit/save plumbing already carry the summary text.
