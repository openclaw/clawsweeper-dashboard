## Summary

Keep open. Current main still supports cron descriptions in the data model, edit flow, and search, but the Control UI job card/selection path does not visibly render `job.description` as the requested human-readable summary.

## What This Changes

Show each cron job's prompt/task and human-readable summary in the Web UI cron dashboard when a job is inspected or selected.

## Best Possible Solution

Render `CronJob.description` as a visible Summary or Description field in the cron job card or selected-job detail for both `systemEvent` and `agentTurn` jobs, while preserving payload rendering and run-history selection behavior.

## Reproduction Assessment

Yes, at source level. A `CronJob` with non-empty `description` flows through types, form save, and search, but `renderJob`/`renderJobPayload` never read `job.description` and clicking a row only switches run history to job scope.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation update with focused view coverage because storage, RPC, and edit-form plumbing already carry the field.
