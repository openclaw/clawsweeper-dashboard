## Summary

Keep open. Current main and the latest release still store and edit cron descriptions and render prompt/system payloads, but the Control UI cron job card/detail path does not display the saved description as the requested human-readable summary.

## What This Changes

Show each cron job's prompt/task and saved human-readable summary in the Web UI when users inspect a job.

## Best Possible Solution

Render non-empty CronJob.description as a visible Summary in the cron job card or selected-job detail area for both system-event and agent-turn jobs while preserving payload rendering and run-history selection.

## Reproduction Assessment

Yes, at source level. Current main and v2026.5.22 carry and save CronJob.description, but renderJob/renderJobPayload never display it and the cron job detail labels have no Summary entry.

## Solution Assessment

Yes. The narrowest maintainable path is a Control UI presentation, English i18n, generated locale sync, and focused test update because the backend storage/RPC and edit/save plumbing already carry the summary text.
