## Summary

Keep open. Current main still carries and edits cron job descriptions, and the dashboard renders the prompt/task payload, but the Control UI job card/detail rendering still does not display the saved description as the requested human-readable summary.

## What This Changes

Show a cron job's prompt/task and human-readable summary or description in the Web UI cron dashboard.

## Best Possible Solution

Render saved CronJob.description as a visible Summary or Description field in the Control UI cron job card or selected-job detail area for both systemEvent and agentTurn jobs, without changing the cron storage/RPC contract.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: render a CronJob with a non-empty description; current renderJob and renderJobPayload never read job.description, so the prompt appears but the summary does not.

## Solution Assessment

Yes. The narrowest maintainable fix is a Control UI presentation change because shared types, persistence, search, and edit-form plumbing already carry the field.
