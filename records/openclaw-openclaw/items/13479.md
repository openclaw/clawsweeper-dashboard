## Summary

Keep open. Current main still carries cron descriptions through the data model and edit/submit path, and it renders prompt/system payloads on cron job cards, but the dashboard rendering path still does not display the saved description as the requested human-readable summary.

## What This Changes

The issue requests that clicking or inspecting a cron job in the Web UI reveal the job's prompt/task and human-readable summary without requiring manual config inspection.

## Best Possible Solution

Render CronJob.description as a visible Summary or Description field in the Control UI cron job card or selected-job detail area for both systemEvent and agentTurn jobs, using the existing cron storage/RPC contract.

## Reproduction Assessment

Yes, at source level. A job with a non-empty description flows through the type model, edit form, submit payload, and backend search, but current renderJob/renderJobPayload does not read job.description.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation update with focused view coverage because the backend and edit plumbing already carry the field and prompt/task display already exists.
