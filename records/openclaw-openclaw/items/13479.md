## Summary

Keep open. Current main and the latest release still carry and save cron descriptions, but the Control UI job card/detail renderer only shows schedule, payload, delivery, and state, so the requested human-readable summary remains missing.

## What This Changes

The issue asks for the Web UI cron dashboard to reveal each job's prompt/task and human-readable description when inspecting a cron job.

## Best Possible Solution

Render non-empty CronJob.description as a visible Summary in the cron job card or selected-job detail area for both system-event and agent-turn jobs while preserving existing payload rendering and run-history selection.

## Reproduction Assessment

Yes, at source level. Current main and v2026.5.22 carry and save CronJob.description, but renderJob/renderJobPayload never display it and the cron job detail labels have no Summary entry.

## Solution Assessment

Yes. The narrowest maintainable path is a Control UI presentation, English i18n, generated locale sync, and focused test update because the backend storage/RPC and edit/save plumbing already carry the summary text.
