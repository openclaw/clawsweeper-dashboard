## Summary

Keep open. Current main and v2026.5.20 still store and edit cron descriptions and render prompt/system payloads, but the Control UI cron job card/detail renderer does not display the saved description as the requested human-readable summary.

## What This Changes

The issue asks the Web UI cron dashboard to show each cron job's prompt/task and human-readable description/summary without requiring config-file inspection.

## Best Possible Solution

Render non-empty CronJob.description as a visible Summary in the Control UI cron job card or selected-job detail area for both system-event and agent-turn jobs, preserving the existing storage/RPC contract and run-history selection behavior.

## Reproduction Assessment

Yes, at source level. Current main and v2026.5.20 carry and save CronJob.description, but renderJob/renderJobPayload never read it and the cron job detail labels have no Summary entry.

## Solution Assessment

Yes. The narrowest maintainable fix is a Control UI presentation, i18n, and test update because backend cron storage and edit/save plumbing already carry the summary text.
