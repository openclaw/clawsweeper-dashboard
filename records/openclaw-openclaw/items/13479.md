## Summary

Keep open. Current main still renders cron prompt/task payloads and supports stored descriptions, but the main Control UI cron dashboard does not display the saved description as the requested human-readable summary when a job is inspected or selected.

## What This Changes

Show each Web UI cron job's prompt/task and saved human-readable summary/description from the cron dashboard.

## Best Possible Solution

Render saved CronJob.description as a visible Summary or Description field in the main Control UI cron job card or selected-job detail area for both systemEvent and agentTurn jobs, while preserving the existing cron storage/RPC contract and row-click run-history behavior.

## Reproduction Assessment

Yes, at source level. Render a CronJob with a non-empty description; current renderJob and renderJobPayload never read job.description, while row clicks only load job-scoped run history.

## Solution Assessment

Yes. The narrowest maintainable fix is a Control UI presentation change with focused view coverage because shared types, persistence, search, and edit-form plumbing already carry the field.
