## Summary

Keep open. Current main still carries cron descriptions through the contract, edit/save path, and list/search metadata, but the Control UI job renderer does not display `job.description` as the requested human-readable summary.

## What This Changes

The issue requests that the Web UI cron dashboard show a cron job's prompt/task and saved human-readable summary when users inspect a job.

## Best Possible Solution

Render non-empty `CronJob.description` as a visible Summary in the Control UI cron job card or selected-job detail area for both system-event and agent-turn jobs, without changing cron storage or RPC contracts.

## Reproduction Assessment

Yes, at source level. A cron job with a non-empty description is supported by the type, form, submit payload, and search path, but the current job renderer never displays that field.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation update because the backend contract and edit/save plumbing already carry the summary text.
