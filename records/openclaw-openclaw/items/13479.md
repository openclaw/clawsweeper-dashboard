## Summary

Current main still omits the saved cron description from the job display, and the open linked PR remains the canonical implementation candidate. The issue should stay open until that PR is updated for current payload branches and merged, or otherwise resolved.

## What This Changes

Show each cron job’s saved human-readable description alongside its existing prompt, command, or system-event details in the Control UI.

## Best Possible Solution

Update and land the linked PR so one shared summary renderer handles every current payload kind while preserving existing escaped rendering, delivery details, and job-scoped run-history selection.

## Reproduction Assessment

Yes, at source level: save a non-empty cron description and inspect its Control UI job card; the model and edit flow retain the value, but the display renderer never reads it. The linked PR also supplies live visual evidence of the same gap and proposed result.

## Solution Assessment

Yes, with a current-main update. Rendering the existing description through one shared helper in the established payload-detail area is the narrowest maintainable approach; it should cover system-event, command, and agent-turn jobs rather than adding storage, RPC, config, or another details path.
