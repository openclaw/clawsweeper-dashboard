## Summary

Keep open. Current main still plumbs cron descriptions through the type, edit/save, and search path, but the Control UI job renderer displays name, schedule, agent, state, payload, delivery, chips, and actions without showing the saved description as the requested summary.

## What This Changes

The issue requests that the Web UI cron dashboard reveal each cron job's prompt/task and human-readable summary when a job is selected.

## Best Possible Solution

Render non-empty CronJob.description as a visible Summary in the cron job card or selected-job detail area for both system-event and agent-turn jobs without changing cron storage or RPC contracts.

## Reproduction Assessment

Yes, at source level. A cron job with a non-empty description is supported by the type, edit form, submit payload, and search path, but the current job card/detail renderer never displays that field.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation update because cron storage, RPC, search, and prompt rendering already carry the data needed for the requested summary.
