## Summary

Keep open. Current main still carries cron descriptions through storage, editing, and search, and it renders prompt/system payload text, but the Control UI job card/detail renderer still does not read or display the saved description as the requested human-readable summary.

## What This Changes

Request to show each cron job’s prompt/task and saved human-readable summary in the Web UI cron dashboard.

## Best Possible Solution

Render stored CronJob.description as a visible summary in the Control UI job card or selected-job detail area for both systemEvent and agentTurn jobs, with focused view coverage and no cron storage/RPC contract changes.

## Reproduction Assessment

Yes, at source level. A cron job with a non-empty description flows through the type, edit form, submit payload, backend persistence, and search, but renderJob/renderJobPayload never read that field.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation update with tests; storage, RPC, and cron service behavior already support the field.
