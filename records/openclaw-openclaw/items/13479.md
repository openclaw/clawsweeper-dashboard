## Summary

Keep open. Current main still carries cron descriptions through the model, storage, edit form, and search metadata, but the Control UI job card/detail renderer only shows the payload prompt/system text and delivery metadata, not the saved description as the requested summary.

## What This Changes

Request to show each cron job's prompt/task and human-readable summary in the Control UI cron dashboard/detail view.

## Best Possible Solution

Render non-empty CronJob.description as a visible Summary in the Control UI cron job card or selected-job detail area for both system-event and agent-turn jobs, without changing cron storage or RPC contracts.

## Reproduction Assessment

Yes, at source level. A cron job with a non-empty description is supported by the type, storage, edit/save path, and query metadata, but the current job renderer and payload renderer do not display that field.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation update because the backend contract and edit/save plumbing already carry the summary text.
