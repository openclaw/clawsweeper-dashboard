## Summary

Keep open. Current main still carries cron descriptions through storage and edit/submit paths, and it renders prompt/system payloads in the cron dashboard, but it does not display the saved description as the requested human-readable summary.

## What This Changes

Show each cron job’s prompt/task and saved description summary in the Web UI so users can understand scheduled jobs without opening config files.

## Best Possible Solution

Render CronJob.description as a visible Summary or Description field in the Control UI cron job card or selected-job detail area for both systemEvent and agentTurn jobs, using the existing cron storage/RPC contract.

## Reproduction Assessment

Yes, at source level. A cron job with a non-empty description flows through the model, edit form, submit payload, and backend search, but the current renderJob/renderJobPayload dashboard path does not read job.description.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation update with focused view coverage, because the backend and edit plumbing already carry the field and the prompt/task display already exists.
