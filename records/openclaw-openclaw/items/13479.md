## Summary

Keep open. Current main still carries cron descriptions through the data model, edit form, submit path, and backend search, but the Control UI job card/detail renderer still does not display the saved description as the requested human-readable summary.

## What This Changes

The issue asks the Control UI cron dashboard to show each job's prompt/task and saved human-readable summary or description when inspecting a cron job.

## Best Possible Solution

Render CronJob.description as a visible Summary or Description field in the Control UI job card or selected-job detail area for both systemEvent and agentTurn jobs, using the existing cron storage/RPC contract.

## Reproduction Assessment

Yes, at source level. A cron job with a non-empty description is accepted, saved, edited, and searchable, but the current renderJob/renderJobPayload display path does not read that field.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation update with focused view coverage because backend and edit plumbing already carry the field.
