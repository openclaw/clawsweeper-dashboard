## Summary

Keep open. Current main still persists and edits cron descriptions and renders prompt/system payloads, but the Control UI job card/detail renderer does not display `job.description` as the requested human-readable summary.

## What This Changes

Request to show each cron job's prompt/task and human-readable summary in the Control UI cron dashboard.

## Best Possible Solution

Render non-empty `CronJob.description` as a visible Summary in the Control UI cron job card or selected-job detail area for both system-event and agent-turn jobs without changing cron storage or RPC contracts.

## Reproduction Assessment

Yes, at source level. Current main maps and submits `CronJob.description`, but `renderJob`/`renderJobPayload` never read that field and the English job detail labels have no Summary label.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation update because the backend cron contract and edit/save plumbing already carry the summary text.
