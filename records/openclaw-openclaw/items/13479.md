## What This Changes

Show each cron job’s saved description as a human-readable Summary alongside its existing prompt or system-event details in the Control UI.

## Best Possible Solution

Add a localized `cron.jobDetail.summary` label to the linked focused PR, sync generated locale outputs, preserve current payload and run-history selection behavior, and close the issue when that PR merges.

## Reproduction Assessment

Yes at source level: current main stores and edits `job.description`, but the cron job renderer omits it. The linked PR’s live recording also demonstrates the intended after-change UI behavior.

## Solution Assessment

Yes, after the requested localization repair: rendering the existing description in the existing job-detail group is narrower and more maintainable than adding another data contract or separate detail workflow.
