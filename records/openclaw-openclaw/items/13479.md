## Summary

Keep open. Current main and the latest shipped release still persist and edit cron descriptions and render prompt/system payloads, but the Control UI job card/detail renderer does not display `job.description` as the requested human-readable summary.

## What This Changes

Show each cron job's prompt/task and human-readable summary in the Web UI cron dashboard so users do not have to inspect config files.

## Best Possible Solution

Render non-empty `CronJob.description` as a visible Summary in the existing Control UI cron job card or selected-job detail area for both system-event and agent-turn jobs, without changing cron storage or RPC contracts.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.20` carry and save `CronJob.description`, but `renderJob` and `renderJobPayload` never read it and the UI labels have no Summary entry.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation update because the backend cron contract and edit/save plumbing already carry the summary text.
