## Summary

Keep open. Current main still stores, edits, searches, and renders cron prompt/delivery details, but the cron job card/detail renderer does not read `job.description`, so the requested human-readable summary remains missing.

## What This Changes

Request to show a cron job's prompt/task and saved human-readable summary in the Control UI cron dashboard.

## Best Possible Solution

Render stored `CronJob.description` as a visible Summary in the cron job card or selected-job detail area, with no cron storage/RPC contract changes.

## Reproduction Assessment

Yes, at source level. A cron job with a non-empty `description` is carried through the type, edit form, submit payload, backend search, and SDK mapping, but `renderJob`/`renderJobPayload` never render that field.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation update with focused view coverage; storage, RPC, and prompt rendering already support the needed data.
