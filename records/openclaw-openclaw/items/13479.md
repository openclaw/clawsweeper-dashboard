## Summary

Keep open. Current `main` still has the cron description data path, edit form, submit payload, and search support, but the Control UI job card/detail rendering only shows the prompt/system payload and delivery metadata, not the saved description as the requested human-readable summary.

## What This Changes

Request to show a cron job's prompt/task and human-readable summary when inspecting a cron job in the Web UI.

## Best Possible Solution

Render stored `CronJob.description` as a visible Summary in the cron job card or selected-job detail area for both systemEvent and agentTurn jobs, with focused Control UI coverage and no storage/RPC changes.

## Reproduction Assessment

Yes, at source level. A job with a non-empty `description` is supported by the type, edit form, submit payload, and search path, but the current job card/detail renderer never displays that field.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation update; cron storage, RPC, search, and payload rendering already carry the data needed for the requested summary.
