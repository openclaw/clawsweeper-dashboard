## Summary

Keep open. Current main still carries cron job descriptions through storage, editing, and search, and it renders the prompt/system payload, but the Control UI job card/detail renderer does not display `job.description` as the requested human-readable summary.

## What This Changes

Show each Web UI cron job's prompt/task and stored human-readable summary from the cron dashboard or job detail view.

## Best Possible Solution

Render stored `CronJob.description` as a visible Summary in the Control UI cron job card or selected-job detail area, without changing the cron storage or RPC contract.

## Reproduction Assessment

Yes, at source level. A cron job with a non-empty `description` is carried through the type, edit form, submit payload, and backend search, but `renderJob` and `renderJobPayload` never render it.

## Solution Assessment

Yes. The narrowest maintainable path is a Control UI presentation update with focused view coverage; storage, RPC, and prompt rendering already support the needed data.
