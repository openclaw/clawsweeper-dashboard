## Summary

Keep open. Current main still stores and edits cron descriptions and renders the prompt/system payload, but the Control UI cron job card/detail renderer does not display `job.description` as the requested human-readable summary.

## What This Changes

Show a cron job's prompt/task and human-readable summary in the Control UI so users can inspect scheduled work without opening config files.

## Best Possible Solution

Render non-empty `CronJob.description` as a visible Summary in the Control UI cron job card or selected-job detail area for both system-event and agent-turn jobs while preserving the existing cron storage/RPC contract and row-click behavior.

## Reproduction Assessment

Yes, at source level. Current main stores and edits `CronJob.description`, but `renderJob` and `renderJobPayload` never display it and the English job-detail labels have no Summary entry.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation update because storage, API, and edit/save plumbing already carry the summary text.
