## Summary

Current main and the latest release still do not render stored cron descriptions in the Control UI cron job detail path. This issue should stay open while the linked implementation PR is refreshed, reviewed, and either merged or closed.

## What This Changes

The issue requests showing a cron job’s prompt/task and human-readable summary in the Web UI so users can understand scheduled jobs without opening config files.

## Best Possible Solution

Land a narrow Control UI change that renders the saved cron description as a visible Summary for all current cron payload detail branches while keeping the existing storage/RPC contract and payload rendering intact.

## Reproduction Assessment

Yes, from source: create or edit a cron job with a non-empty description, then inspect the Control UI cron job renderer on current main; `job.description` is stored but not rendered in the job card/detail path.
