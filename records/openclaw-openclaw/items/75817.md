## Summary

Current main now fixes the reported secondary `undefined.error` gateway crash path by creating fallback channel loggers/runtimes before channel start, stop, and failure diagnostics, with regression coverage for a missing startup wiring case.

## What This Changes

Report a gateway crash where a missing channel logger turns a channel task rejection into a fatal `Cannot read properties of undefined (reading 'error')` under event-loop pressure.

## Best Possible Solution

Keep the current local channel-manager fallback logger/runtime implementation; track any remaining DNS latency or prompt-assembly pressure separately with a fresh stability bundle if it still reproduces.

## Reproduction Assessment

Yes for the proximate crash path. The current regression test covers a channel loaded after manager construction with missing startup wiring and a rejecting `startAccount`, which exercises the missing-logger scenario without reproducing the full DNS-pressure incident.

## Solution Assessment

Yes. The implemented local fallback is narrower and safer than changing the global unhandled-rejection policy, which still correctly exits on unrelated fatal promise failures.
