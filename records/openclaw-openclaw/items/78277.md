## Best Possible Solution

Finish cross-process serialization for every mutating bootstrap-token state path, replace the test with non-reentrant multi-process or stale-snapshot coverage, rebase, and require redacted after-fix setup-code proof before merge.

## Solution Assessment

No. Cross-process locking is the right narrow direction, but this patch leaves one mutating writer process-local and uses a same-process reentrant test that does not prove the reported race is closed.
