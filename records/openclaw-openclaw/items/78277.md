The PR wraps most device-bootstrap token JSON mutations in a cross-process file lock and adds a lock-contention regression test for token issuance.

Yes. Source inspection shows independent processes can load stale devices/bootstrap.json snapshots and overwrite bootstrap-token binding or revocation because current main uses only process-local serialization around whole-file writes.

No. Cross-process file locking is the right narrow direction, but this PR leaves a mutating device-bound revocation path on the old lock and does not prove the stale-snapshot resurrection case.
