## Summary

Keep open: the PR targets a real current/shipped bootstrap-token race, but it is not merge-ready because one mutating writer remains process-local, the regression test does not prove the stale cross-process race, the branch conflicts with main, and external real behavior proof is missing.

## What This Changes

The PR adds a `withBootstrapStateLock` file-lock wrapper around most `devices/bootstrap.json` token state mutations and adds a device-bootstrap lock-contention regression test.

## Best Possible Solution

Finish cross-process serialization for every mutating bootstrap-token writer, add non-reentrant multi-process or stale-snapshot regression coverage, rebase, and require redacted after-fix Gateway/setup-code proof before merge.

## Reproduction Assessment

Yes. ## Solution Assessment

No. Cross-process locking is the right narrow direction, but this patch leaves one mutating writer process-local, uses a same-process contention test for a cross-process stale-snapshot bug, and lacks real after-fix setup-code proof.
