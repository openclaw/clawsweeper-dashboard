## Summary

Keep open: this is a draft, maintainer-labeled PR with a concrete P2 regression in refresh completion semantics; the responsive-refresh direction is plausible but needs a narrow follow-up before merge.

## What This Changes

The branch changes Control UI chat, settings, render timing metadata, tests, and changelog to decouple chat/history, config/schema, and channel status refresh work and reduce the chat render window.

## Best Possible Solution

Keep the responsive refresh design, but preserve a post-history completion path for manual chat refresh before undrafting and merging.

## Reproduction Assessment

No live browser repro was established in this read-only review. Source inspection does show the current blocking await paths and a high-confidence introduced manual-refresh regression in the PR diff.

## Solution Assessment

No; the decoupling direction is maintainable, but resolving `refreshChat` before history completes breaks callers that intentionally disable internal scrolling and await the refresh. A safer solution keeps tab refresh nonblocking while giving manual refresh a post-history completion or callback path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

- **[P2] Preserve post-history completion for manual refresh:** `ui/src/ui/app-chat.ts:767-768`

Please keep the nonblocking tab refresh behavior but give manual refresh a post-history completion path.
