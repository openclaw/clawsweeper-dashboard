## Summary

Keep open. The PR addresses a plausible ACP/Claude auth collision, but it is not merge-ready because the default scrubs provider env vars from unrecognized/custom ACP harnesses, the live `sessions_spawn` after-fix path was not proven, and the branch is reported unmergeable against current main.

## What This Changes

The PR adds ACP harness provider-env scrubbing via an `env -u` command prefix, introduces `acp.scrubProviderEnv`, updates config metadata, and adds acpx regression tests.

## Best Possible Solution

Scrub only proven known harness credential collisions by default, preserve custom/unknown harness env inheritance unless explicitly configured otherwise, document the opt-out in ACP operator docs, rebase regenerated config metadata, and add live ACP spawn proof before merge.

## Reproduction Assessment

No high-confidence live reproduction was established here. Source inspection shows current main does not scrub provider credentials from ACP harness commands, and the PR body provides a plausible reporter repro, but the failing and fixed live `sessions_spawn` path was not run in this review.

## Solution Assessment

No. The known-harness scrub is a plausible narrow fix, but scrubbing the union of known provider credentials for every unknown/custom harness is too broad for a default upgrade path; preserve custom env inheritance or make broad isolation explicit.
