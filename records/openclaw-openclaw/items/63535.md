## Summary

Keep open: the cron session recovery problem is plausible and current main still lacks stale-running recovery, but this PR is not safe to merge because it can finalize active or no-timeout runs, misses the visible base cron row, skips retention-disabled configs, and lacks real behavior proof.

## What This Changes

This PR adds a two-hour stale-running cron run recovery pass to the session reaper, returns an `orphansRecovered` count, and adds a unit test for cron run rows.

## Best Possible Solution

Keep the PR open for a rebased recovery design that runs independently of pruning, uses an authoritative abandoned-run signal, updates stable base rows plus aliases, follows approved interrupted-run status semantics, and includes focused regression coverage plus real behavior proof.

## Solution Assessment

No. A reaper recovery pass is plausible, but this implementation depends on pruning being enabled, relies on `updatedAt` age alone, only updates run aliases, and chooses `done` without an approved interrupted-run contract.

Overall confidence: 0.94

Full review comments:

- **[P1] Prove abandonment before finalizing runs:** `src/cron/session-reaper.ts:106-109`
  - body: This marks any `status: "running"` cron run row older than two hours as `done`, but no-timeout cron agent turns are supported and `updatedAt` is not a heartbeat. An active run can be finalized and later pruned while it is still executing.
  - confidence: 0.93
- **[P1] Recover the stable cron session row:** `src/cron/session-reaper.ts:106`
  - body: The recovery branch only writes the `:run:` alias row. Current isolated cron persistence stores visible state under the stable `agent:...:cron:<job>` key and session listing hides run aliases, so the user-visible row can remain stuck as `running`.
  - confidence: 0.92
- **[P2] Run recovery when pruning is disabled:** `src/cron/session-reaper.ts:83-85`
  - body: The `retentionMs === null` return exits before the orphan scan, so users who set `cron.sessionRetention: false` still get stuck `running` rows after a crash even though they only disabled pruning.
  - confidence: 0.94
- **[P3] Avoid epoch endedAt values:** `src/cron/session-reaper.ts:106`
  - body: When `updatedAt` is missing or malformed, this branch persists `endedAt: 0`, which can display as the Unix epoch. Use the sweep time or another valid terminal timestamp fallback instead.
  - confidence: 0.78
