## Summary

Keep this PR open: current main still lacks cron orphan-session recovery, but this branch has blocking correctness gaps, needs a maintainer decision on interrupted-run terminal status, and still has no real after-fix behavior proof.

## What This Changes

The PR adds a two-hour stale-running recovery pass to `src/cron/session-reaper.ts`, returns `orphansRecovered`, logs recoveries, and adds one unit test.

## Best Possible Solution

Rework the recovery patch so it runs independently of pruning, proves abandonment before finalizing, repairs the stable base row plus any legacy run aliases, uses the approved interrupted-run status, and adds focused regressions.

## Reproduction Assessment

Yes, source-reproducible: current main has no orphan-recovery path, and synthetic stale session-store rows can exercise the retention-disabled, base-key, and long-running/no-timeout gaps. I did not run tests because this review is read-only.

## Solution Assessment

No. A reaper-based repair pass is plausible, but this implementation depends on pruning, only repairs hidden run aliases, relies on `updatedAt` alone as abandonment proof, and chooses terminal status without an approved semantic contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Prove a run is abandoned before finalizing it:** `src/cron/session-reaper.ts:106-109`
  - body: This marks any `status: "running"` cron run row older than two hours as `done`, but agent-turn jobs can disable timeouts and `updatedAt` is not a heartbeat. An active long run can be finalized and later pruned while it is still executing.
  - confidence: 0.93
- **[P1] Recover the stable cron session row:** `src/cron/session-reaper.ts:106`
  - body: The recovery loop updates only `:run:` alias rows. Current isolated cron persistence writes visible session state to the stable `agent:...:cron:<job>` row, and session listing hides run aliases, so the UI can still show the cron session as `running`.
  - confidence: 0.92
- **[P2] Run recovery when pruning is disabled:** `src/cron/session-reaper.ts:83-85`
  - body: The `retentionMs === null` return exits before orphan scanning. With `cron.sessionRetention: false`, stale running sessions remain stuck even though disabling pruning should not disable crash-status recovery.
  - confidence: 0.94
- **[P3] Avoid epoch endedAt values:** `src/cron/session-reaper.ts:106`
  - body: When `updatedAt` is missing, this branch uses the `0` fallback as `endedAt`, which can surface as the Unix epoch. Use the sweep time or another valid terminal timestamp instead.
  - confidence: 0.82
