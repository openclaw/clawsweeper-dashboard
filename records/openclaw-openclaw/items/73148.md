## Summary

Keep open. Current main fixes the fully opaque diagnostic, but the default image path still depends on Sharp being available and the provided latest-release comments show the packaged-install failure remains user-visible on 2026.5.2.

## What This Changes

The issue asks OpenClaw to stop failing image processing with an opaque optimizer error when the Sharp runtime dependency is missing from packaged installs.

## Best Possible Solution

Guarantee Sharp is available for packaged `media-understanding-core` image ops, with install/upgrade proof; add an under-cap original-image fallback only if maintainers accept that media policy.

## Reproduction Assessment

Yes. Source inspection gives a concrete path: image media reaches `optimizeImageToJpeg`, then `resizeToJpeg`, then `media-understanding-core` dynamically imports `sharp`; provided latest-release comments confirm the missing dependency on packaged installs.

## Solution Assessment

Unclear. The current diagnostic behavior is the right narrow fix for the opaque message, but the remaining failure needs an owner decision on dependency staging versus guarded fallback behavior.
