## Summary

Keep open. Current main fixes much of the original opacity and dependency-declaration problem, but #74243 is still an open, unmerged implementation candidate that explicitly closes this issue and covers the remaining optimizer-fallback policy.

## What This Changes

The issue asks OpenClaw to avoid opaque `Failed to optimize image` failures when `sharp` is missing by declaring or staging Sharp reliably and surfacing the underlying optimizer error.

## Best Possible Solution

Review and either land #74243 for the guarded original-image fallback, or explicitly supersede it after confirming the current dependency and diagnostic fixes fully cover the intended behavior.

## Reproduction Assessment

Yes. The report and follow-up comments provide concrete versions, install shapes, missing-`sharp` checks, and image-tool call paths; current source confirms the same Sharp-backed optimizer boundary.

## Solution Assessment

Unclear. Current main is a strong partial fix for diagnostics and dependency ownership, but the best final behavior depends on whether maintainers accept #74243's guarded fallback for already-sendable originals.
