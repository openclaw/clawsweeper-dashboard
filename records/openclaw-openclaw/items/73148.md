## Summary

Keep open. Current main has improved the opaque optimizer error message, but the normal image/media path still depends on the Sharp-backed media-understanding-core optimizer while packaged installs still do not make Sharp a root dependency or include extension node_modules.

## What This Changes

The issue asks OpenClaw to make image/media optimization work and report actionable errors when Sharp is absent from packaged installs.

## Best Possible Solution

Guarantee media-understanding-core's Sharp dependency is available in packaged installs, preserve the current actionable optimizer cause, and treat any original-image fallback as an explicit media policy decision.

## Reproduction Assessment

Yes. Source inspection shows normal images still enter the Sharp-backed optimizer, and the issue discussion includes concrete packaged-install repros where Sharp is absent; I did not run a live packaged install in this read-only sweep.

## Solution Assessment

Unclear. The current diagnostic improvement is useful but not sufficient; the narrow maintainable fix needs an explicit maintainer decision on native dependency packaging versus fallback behavior.
