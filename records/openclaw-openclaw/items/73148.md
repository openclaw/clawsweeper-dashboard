## Summary

Keep open. The fully opaque diagnostic is partially fixed on current main, but the missing-Sharp packaged-install failure remains source-reproducible: normal image/media paths still require the Sharp-backed optimizer, with no guarded fallback and no root package dependency on Sharp.

## What This Changes

The issue asks OpenClaw to avoid failing every image with an opaque optimizer error when Sharp is missing, either by reliably packaging Sharp or by surfacing/falling back from the underlying dependency failure.

## Best Possible Solution

Guarantee Sharp is available to `media-understanding-core` in packaged installs, and consider a guarded original-buffer fallback for already-under-cap safe images while preserving actionable error causes.

## Reproduction Assessment

Yes. Current source still routes ordinary image media through the Sharp-backed optimizer, and recent comments report the same missing-Sharp failure on OpenClaw 2026.5.2; I did not run a live install repro in this read-only review.

## Solution Assessment

Unclear. Surfacing the underlying cause is a good partial fix, but the best complete fix needs maintainer direction on whether the primary boundary is packaged runtime dependency staging, a hard dependency, or a conservative media fallback.
