## Summary

Close as implemented on current main. Missing-Sharp failures now carry the underlying cause, in-limit non-HEIC images can continue without the optional optimizer, and regression coverage locks in the unavailable-optimizer behavior; I could not prove the last fallback fix shipped in v2026.5.3, so this is current-main-only.

## What This Changes

The issue asks for the image/media pipeline to avoid opaque `Failed to optimize image` failures when optional `sharp` is missing.

## Best Possible Solution

Keep the current behavior: preserve optimizer causes, pass through safe in-limit originals when optional Sharp is unavailable, and keep HEIC or over-cap media fail-closed with an actionable Sharp error.

## Reproduction Assessment

No. The historical report and comments are high-confidence, but current main has source and test coverage showing missing Sharp no longer produces the opaque every-image failure: in-limit originals pass through and cap/HEIC failures include the Sharp cause.

## Solution Assessment

Yes. The current implementation is the narrowest maintainable fix for this report because it keeps the native optimizer optional, preserves security/size guards, and makes missing-Sharp failures actionable rather than moving the native dependency into root package policy.
