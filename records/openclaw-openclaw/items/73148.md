## Summary

Keep open. Current main improves the opaque optimizer error and adds an under-cap original-image fallback, but the packaged-install Sharp availability problem is not fully closed and this issue is already paired with open PR #77197 using closing syntax.

## What This Changes

The report asks OpenClaw to handle missing Sharp in image/media optimization by installing it reliably or surfacing the missing-dependency cause instead of only returning `Failed to optimize image`.

## Best Possible Solution

Guarantee `media-understanding-core` can resolve Sharp in packaged installs while keeping the current cause-preserving diagnostics and guarded under-cap fallback; review #77197 as a narrow diagnostic improvement if that scope is still desired.

## Reproduction Assessment

Yes. Source inspection shows the normal image path still depends on the Sharp-backed optimizer for over-cap/direct optimization, and the issue discussion includes fresh packaged-upgrade reproductions where Sharp is absent.

## Solution Assessment

Unclear. The current main behavior is a useful partial fix, but the remaining packaged-install failure should be resolved through an explicit native dependency/runtime-deps policy rather than by closing this issue early.
