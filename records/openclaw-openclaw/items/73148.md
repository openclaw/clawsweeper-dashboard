## Summary

Keep open. Current main improves the diagnostic and in-limit fallback paths, but the packaged Sharp availability problem is still not resolved, and an open PR with closing syntax is already the active candidate.

## What This Changes

The issue asks OpenClaw to avoid opaque image-tool failures when the Sharp optimizer dependency is missing and to make the dependency/failure path actionable.

## Best Possible Solution

Review the linked PR, then make the durable packaged-install path ensure `media-understanding-core` can load Sharp or fail with actionable guidance for images that require optimization.

## Reproduction Assessment

Yes. Source inspection shows missing Sharp still breaks images that require optimization, and the discussion includes fresh packaged-install reproductions on current stable releases.

## Solution Assessment

Unclear. The current implementation is a useful partial fix, but the remaining best fix needs a maintainer decision between plugin-local runtime dependency staging, root optional dependency ownership, and fallback/error-policy wording.
