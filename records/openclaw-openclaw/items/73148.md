## Summary

Close as implemented. The reported missing-sharp and opaque image-optimization failure is fixed in the shipped v2026.5.22 code path, and current main has moved image processing off the sharp-backed bundled extension path entirely.

## What This Changes

The issue asks OpenClaw to stop returning an opaque image-tool failure when the sharp optimizer dependency is missing by making the dependency available and/or surfacing or bypassing the optimizer failure.

## Best Possible Solution

Keep the shipped/current media loader behavior: dependencies needed by image processing resolve from the root package graph, optimizer causes stay visible, and safe in-limit originals can bypass unavailable optimization.

## Reproduction Assessment

No current-main failure reproduction remains. The original reproducer targeted old installs with missing sharp, while v2026.5.22 packages sharp at the root and current main no longer uses the sharp-backed image-op path.

## Solution Assessment

Yes. The shipped/current direction addresses both availability and diagnostics without relying on nested extension package dependencies, and current main further simplifies the dependency boundary by using a root rastermill dependency.
