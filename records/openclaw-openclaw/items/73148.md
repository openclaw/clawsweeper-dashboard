## Summary

Keep open. Current main improves the diagnostic path and can pass through in-limit originals, but the packaged-install Sharp availability gap remains unresolved and an open PR with closing syntax is still the active implementation candidate.

## What This Changes

The report asks OpenClaw to stop returning an opaque image-optimization error when Sharp is missing, either by installing/staging Sharp reliably or surfacing the missing dependency cause.

## Best Possible Solution

Review the linked PR, then make the durable packaging path guarantee that `media-understanding-core` can load Sharp in packaged installs without a manual post-upgrade reinstall.

## Reproduction Assessment

Yes. Source inspection shows the over-cap/direct optimizer path still requires Sharp, and the discussion has fresh stable-release packaged-install reproductions where Sharp is absent.

## Solution Assessment

Unclear. Current main has a useful diagnostic and under-cap fallback, but the remaining root problem is dependency packaging policy, while the open PR only covers a narrower install-hint path.
