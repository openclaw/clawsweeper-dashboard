## Summary

Keep open. Current main fixes the fully opaque diagnostic, but normal image media still depends on Sharp being available and the root package still does not install Sharp directly, matching the fresh 2026.5.2 reports.

## What This Changes

The issue asks OpenClaw to stop failing every image with an opaque optimizer error when the Sharp native dependency is missing.

## Best Possible Solution

Guarantee the Sharp-backed media-understanding image ops dependency is available in packaged installs, and only add an under-cap original-image fallback if maintainers accept that media policy with guard-preserving tests.

## Reproduction Assessment

Yes. Source inspection shows current main still routes normal images through the Sharp-backed optimizer while the root package does not install Sharp directly, and fresh issue comments reproduce the same path on OpenClaw 2026.5.2.

## Solution Assessment

Unclear. The current diagnostic behavior is a useful partial fix, but the remaining user-visible failure needs a maintainer decision on dependency staging versus a conservative media fallback.
