## Summary

Current main fixes the reported opaque missing-Sharp failure: optimizer errors now keep the underlying cause, and in-limit non-HEIC images are passed through when the optional optimizer is unavailable.

## What This Changes

The issue asks for image optimization to stop failing opaquely when Sharp is missing, either by making Sharp available or surfacing or bypassing the missing dependency.

## Best Possible Solution

Keep the current main behavior: preserve actionable optimizer causes, pass through safe in-limit originals when Sharp is unavailable, and keep HEIC or over-cap media fail-closed unless a separate packaging-policy change hard-requires Sharp.

## Reproduction Assessment

No, not against current main. Source and regression tests now show the missing-Sharp path either returns an in-limit original image or throws an error that includes the Sharp import cause, so the original opaque every-image failure no longer matches current behavior.

## Solution Assessment

Yes. The current fix is the narrow maintainable path for this bug: it improves diagnostics and avoids failing safe in-limit images without turning Sharp into a root native dependency or bypassing caps/HEIC conversion safety.
