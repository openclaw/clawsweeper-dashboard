## Summary

Keep open. Current main has improved the diagnostic so the missing Sharp cause is no longer fully opaque, but the normal image/media path still depends on Sharp and the packaged root still does not declare it, matching the fresh v2026.5.2 reports in the discussion.

## What This Changes

The issue requests that image media processing either have Sharp available in packaged installs or produce actionable/fallback behavior instead of a generic optimizer failure.

## Best Possible Solution

Choose and implement a supported packaging policy for media-understanding-core so Sharp is reliably available in packaged installs or the Sharp-dependent capability moves to an installable plugin, while preserving the current actionable error cause.

## Reproduction Assessment

Yes, source-reproducible rather than locally reproduced. Current main still routes normal images through the Sharp-backed optimizer, and the issue discussion includes fresh v2026.5.2 packaged-install failures where Sharp is absent.

## Solution Assessment

Unclear. The diagnostic improvement on main is a good partial fix, but the durable solution needs maintainer direction on whether Sharp belongs in the core package, a managed plugin dependency path, or an externalized plugin.
