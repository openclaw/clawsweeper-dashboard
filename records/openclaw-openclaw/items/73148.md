## Summary

Keep open. Current main improves the opaque optimizer error, but the packaged Sharp availability problem remains unresolved: image media still goes through the Sharp-backed optimizer by default, while the root npm package still does not directly install Sharp and runtime overlays explicitly avoid plugin `node_modules`.

## What This Changes

The issue asks OpenClaw to make image optimization usable or clearly diagnosable when the Sharp runtime dependency is missing from packaged installs.

## Best Possible Solution

Guarantee Sharp availability for the packaged `media-understanding-core` image ops path, or intentionally add a guarded original-image fallback for already-safe images while preserving actionable backend-missing errors.

## Reproduction Assessment

Yes, from source inspection plus fresh release comments. Current main still routes normal image media through the Sharp-backed optimizer, while the package metadata does not make Sharp a root-installed dependency and the runtime overlay skips plugin `node_modules`.

## Solution Assessment

Unclear. The diagnostic part is now handled on main, but the complete fix needs maintainer choice between native dependency packaging/staging and a conservative fallback policy.
