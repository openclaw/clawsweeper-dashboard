## Summary

Keep open. Current main has fixed the opaque diagnostic, but the reported packaged-install failure class is still not fully resolved because non-sips image optimization still depends on Sharp being available and current source/docs do not show a startup-time dependency repair or guarded original-image fallback.

## What This Changes

The issue asks for image optimization failures without installed Sharp to become diagnosable and/or stop breaking the image tool on packaged installs.

## Best Possible Solution

Choose the product boundary for packaged image optimization, then either guarantee Sharp availability before image ops load or add a guarded original-image fallback that preserves HEIC, pixel-limit, and MIME safety checks.

## Reproduction Assessment

Yes. The issue and comments give concrete installs where `sharp` is absent, and current source still routes non-sips image resizing through the Sharp-backed `media-understanding-core` public artifact.

## Solution Assessment

Unclear. The diagnostic fix is maintainable and covered, but the remaining functional behavior needs maintainer judgment between dependency packaging, explicit repair, and fallback semantics.
