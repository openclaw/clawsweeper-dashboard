## Summary

Keep open. Current main fixes the fully opaque optimizer message, but the normal image path still depends on Sharp and the package metadata/build packaging still do not show a reliable packaged-install path for that native dependency; fresh discussion also reproduces the same missing-Sharp failure on v2026.5.2.

## What This Changes

The issue asks OpenClaw to surface missing-Sharp optimizer failures clearly and make image/media handling work reliably when the Sharp backend is absent from packaged installs.

## Best Possible Solution

Guarantee media-understanding-core can resolve Sharp in packaged installs and upgrades while preserving actionable optimizer errors; any fallback should preserve MIME/resource guards and fail closed on unsafe inputs.

## Reproduction Assessment

Yes by source inspection and issue evidence. Current main still routes normal images through the Sharp-backed optimizer, and the package metadata excludes extension node_modules while only the plugin package declares sharp; comments add fresh v2026.5.2 packaged-install reproductions.

## Solution Assessment

Unclear. The current diagnostic change is useful but not sufficient; the durable fix needs a maintainer decision on native dependency packaging versus runtime-deps staging or a conservative media fallback.
