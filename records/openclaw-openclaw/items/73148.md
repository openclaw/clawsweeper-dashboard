## Summary

Keep open. Current main fixes the opaque optimizer diagnostic, but the remaining packaged-install behavior still needs maintainer policy on making Sharp reliably available versus adding a guarded fallback when optimization cannot run.

## What This Changes

The issue asks for image optimization failures without installed Sharp to become diagnosable and/or avoid breaking the image tool on packaged installs.

## Best Possible Solution

Choose the product boundary for packaged image optimization, then either guarantee Sharp materialization for `media-understanding-core` before image ops load or add a guarded original-image fallback that preserves HEIC, pixel-limit, and MIME safety checks.

## Reproduction Assessment

Yes. The issue and comments provide concrete package installs where `sharp` is absent, and current source still routes non-sips image resizing through the Sharp-backed `media-understanding-core` public artifact.

## Solution Assessment

Unclear. The diagnostic change is maintainable and covered, but the remaining functional behavior needs a maintainer decision between dependency packaging and fallback semantics.
