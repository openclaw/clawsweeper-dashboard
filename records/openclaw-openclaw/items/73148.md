## Label Justifications

- `P1`: The original report broke image/media processing for real installed users across channels.
- `impact:message-loss`: The failure affected inbound image analysis and outbound media delivery when image optimization could not run.

## Summary

Current main and shipped releases no longer use the Sharp-backed image optimizer that produced the opaque missing-`sharp` failure. The media path now uses a root-declared Rastermill dependency, and the first release containing that replacement was v2026.5.26.

## What This Changes

The issue reports that image optimization hides missing `sharp` failures behind a generic `Failed to optimize image` error and that packaged installs can omit `sharp`.

## Best Possible Solution

Keep the Rastermill-backed packaged implementation; new reports should show a current image-processing failure rather than relying on `sharp` being absent from the package manifest.

## Reproduction Assessment

Yes for the historical shipped path: the report and follow-up comments consistently identify a missing `sharp` runtime import hidden by the old optimization wrapper. Current main no longer has that Sharp import path, so the original reproduction no longer applies.

## Solution Assessment

Yes. Replacing the Sharp-backed implementation with a root-declared Rastermill dependency is a more maintainable fix than adding `sharp` to current packages for code that no longer imports it.
