## Summary

Keep open. Current main fixes the fully opaque optimizer error, but normal image/media handling still depends on the Sharp-backed optimizer while the root package does not install Sharp directly; fresh discussion reports the same packaged-install failure on v2026.5.2.

## What This Changes

The issue asks OpenClaw to make image optimization failures actionable when Sharp is missing and to ensure normal packaged installs can process images without manual Sharp installation.

## Best Possible Solution

Guarantee the media-understanding-core Sharp dependency is available in packaged installs, preserve actionable optimizer causes, and only add an under-cap original-image fallback if maintainers accept that media policy.

## Reproduction Assessment

Yes. Source inspection shows normal images still enter the Sharp-backed optimizer, and the issue discussion includes fresh v2026.5.2 packaged-install reproductions where Sharp is absent.

## Solution Assessment

Unclear. The current diagnostic fix is useful but incomplete; the durable fix needs a maintainer decision on native dependency packaging versus a conservative media fallback.
