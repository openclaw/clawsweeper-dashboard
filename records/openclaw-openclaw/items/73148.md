## Summary

Keep open. Current main fixes the fully opaque diagnostic, but normal image/media sends still require the Sharp-backed optimizer and the packaged root manifest still does not install or ship Sharp for that extension path.

## What This Changes

The report asks OpenClaw to make missing Sharp failures in image preprocessing actionable, either by packaging Sharp reliably or by surfacing and safely handling the dependency failure.

## Best Possible Solution

Guarantee the Sharp-backed media-understanding dependency is available in packaged installs, while keeping the current actionable error and only adding an under-cap original-image fallback if maintainers accept that media policy with guard-preserving tests.

## Reproduction Assessment

Yes. Source inspection shows current main still routes normal image tool and outbound image paths through the Sharp-backed optimizer while the root published package does not install or ship Sharp directly; recent comments reproduce the released failure on 2026.5.2.

## Solution Assessment

Unclear. The current diagnostic change is the right narrow fix for the opaque error, but the remaining user-visible failure needs a maintainer decision on dependency staging versus root dependency versus a conservative fallback.
