## Summary

Keep open. Current main preserves the underlying Sharp error in the optimizer message, but it still does not guarantee Sharp is available in packaged installs and the normal image path still fails instead of using a safe fallback when the backend is missing.

## What This Changes

The issue asks OpenClaw to make image optimization diagnosable and functional when the Sharp native dependency is absent from packaged installs.

## Best Possible Solution

Guarantee Sharp is available to the packaged `media-understanding-core` image ops path, or intentionally add a guarded fallback for already-safe originals while preserving actionable error causes.

## Reproduction Assessment

Yes. Current source still routes normal image media through the Sharp-backed optimizer, while the root package does not directly install Sharp and excludes extension `node_modules`; the latest issue comment confirms the same path failing on OpenClaw 2026.5.2.

## Solution Assessment

Unclear. The current diagnostic fix is useful but incomplete; the best complete fix needs a maintainer decision on the native dependency/package boundary versus a conservative media fallback.
