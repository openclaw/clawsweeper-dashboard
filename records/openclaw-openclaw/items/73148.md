## Summary

Keep open. Current main fixes the fully opaque error string, but the installed-package failure remains source-reproducible because image optimization still depends on `sharp` while the root package does not declare it and packaged extension `node_modules` are excluded.

## What This Changes

The issue asks for missing-`sharp` image optimization failures to be actionable and/or for `sharp` to be reliably available in installed OpenClaw packages.

## Best Possible Solution

Make the packaged image optimization backend reliable by shipping or resolving `sharp` for `media-understanding-core` in normal installs, or by adopting a maintainer-approved guarded fallback for already-safe originals while preserving the current actionable error detail.

## Reproduction Assessment

Yes, from source inspection. A current install without resolvable `sharp` still reaches `media-understanding-core` image ops during normal image optimization, and the latest issue comment reports the same failure on 2026.5.2 with the clearer error.

## Solution Assessment

Unclear. The diagnostic fix is narrow and already present, but the complete fix needs a maintainer packaging/runtime decision before changing native dependency placement or adding a fallback policy.
