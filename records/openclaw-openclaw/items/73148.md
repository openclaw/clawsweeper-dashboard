## Summary

Keep open. Current main improves the error detail, but missing Sharp still breaks normal image/media processing and the packaged install path still does not prove that Sharp is available.

## What This Changes

The issue asks OpenClaw to stop failing image/media processing with an opaque optimizer error when the Sharp runtime dependency is missing.

## Best Possible Solution

Guarantee the `media-understanding-core` Sharp dependency is available in packaged/global installs, or add a guarded fallback for already-safe originals while preserving actionable optimizer causes.

## Reproduction Assessment

Yes. Source inspection shows current main still routes normal image media through the Sharp-backed optimizer while the root package does not install Sharp directly, and a fresh 2026.5.2 report confirms the same failure in an installed package.

## Solution Assessment

Unclear. The current diagnostic improvement is useful but incomplete; the complete fix needs a maintainer decision between native dependency packaging/staging and a conservative media fallback.
