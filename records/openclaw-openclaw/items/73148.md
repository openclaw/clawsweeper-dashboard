## Summary

Keep open. Current main has made the missing-Sharp error more actionable, but the issue is still explicitly paired with open PR #74243, which remains unmerged and carries the remaining guarded original-image fallback policy.

## What This Changes

The issue asks OpenClaw to avoid an opaque `Failed to optimize image` failure when `sharp` is missing by declaring/staging the dependency reliably and surfacing the underlying optimizer error.

## Best Possible Solution

Review and either land PR #74243 for the guarded original-image fallback, or close/supersede that PR with an explicit decision that current dependency staging and diagnostic behavior are sufficient.

## Reproduction Assessment

Yes. The issue and follow-up comments provide concrete install shapes, missing-`sharp` checks, versions, and media paths; current source confirms the Sharp-backed optimizer boundary and the direct optimizer failure path, though the exact opaque diagnostic is now partially improved on main.

## Solution Assessment

Unclear. Current main is a valid partial fix for diagnostics and dependency ownership, while PR #74243 is the current proposed fix for sendable-image fallback behavior and should be reviewed against HEIC and pixel-limit safeguards before this issue closes.
