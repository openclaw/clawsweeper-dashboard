## Summary

Keep open. Current main fixes the opaque diagnostic by preserving resize/import failure details, but it still does not make Sharp available in packaged installs or provide a guarded original-image fallback when the optimizer backend is unavailable.

## What This Changes

Request that image optimization stop returning only `Failed to optimize image` when Sharp is missing, either by making Sharp available or by surfacing the underlying dependency failure.

## Best Possible Solution

Decide the product boundary for packaged image optimization, then either make Sharp reliably available for `media-understanding-core` in packaged installs or implement a guard-preserving fallback for already-safe originals while keeping the current actionable error detail.

## Reproduction Assessment

Yes. The issue and comments give concrete install shapes and missing-`sharp` checks, and current source still routes optimized image handling through the Sharp-backed `resizeToJpeg` boundary.

## Solution Assessment

Unclear. Current main is a good diagnostic fix, but it is not the full functional fix if OpenClaw is expected to process images without a manually installed Sharp backend.
