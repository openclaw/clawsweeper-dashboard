## Summary

Keep open. Current main fixes the opaque diagnostic path, but it still does not guarantee that the Sharp dependency used by normal image optimization is available in packaged/global installs, and fresh comments report the same missing-Sharp failure on v2026.5.2.

## What This Changes

The issue asks OpenClaw to make missing Sharp failures in image optimization actionable and to ensure the image tool works on installs where Sharp is not already present.

## Best Possible Solution

Keep the current diagnostic improvement, then make packaged installs reliably provide `media-understanding-core`'s Sharp runtime dependency with regression coverage and package/global install proof.

## Reproduction Assessment

Yes. Source inspection shows normal image paths still require the Sharp-backed backend when `OPENCLAW_IMAGE_BACKEND=sips` is not used, while the packaged root does not directly install Sharp; fresh v2026.5.2 comments confirm the same failure in real installs.

## Solution Assessment

Unclear. Current main is the right narrow fix for diagnostics, but it is only partial; the remaining user-visible break should be solved through a deliberate packaged runtime-dependency strategy rather than another diagnostic-only change.
