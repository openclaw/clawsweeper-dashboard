## Summary

Keep open. Current main only partially improves the missing-Sharp failure by preserving details and allowing in-limit originals, while packaged installs can still lack `sharp` and an open closing PR is already the active implementation path.

## What This Changes

The issue asks for the image/media pipeline to avoid opaque `Failed to optimize image` failures when `sharp` is missing and to make the missing dependency actionable or reliably available.

## Best Possible Solution

Review the linked PR, then make packaged installs consistently provide the media-understanding Sharp backend or fail with an actionable dependency repair path for images that cannot pass through unchanged.

## Reproduction Assessment

Yes. Source inspection shows current main still requires dynamic `import("sharp")` for resize/HEIC paths, the root package does not install `sharp`, and fresh release comments reproduce missing Sharp in packaged installs.

## Solution Assessment

Unclear. Current main's diagnostics and in-limit fallback are useful but not sufficient for images that require resizing, so the remaining fix should resolve the packaged dependency path rather than only rewording the terminal error.
