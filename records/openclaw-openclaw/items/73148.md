## Summary

Keep open. Current main fixes the fully opaque error text, but the normal image path still depends on the Sharp-backed optimizer and does not guarantee Sharp is available or preserve already-safe originals when the backend is missing.

## What This Changes

The issue requests clearer and more resilient handling when missing Sharp causes image optimization to fail in packaged installs.

## Best Possible Solution

Guarantee the media-understanding image backend is available in packaged installs, or add a conservative fallback that only passes through already-safe images while preserving clear dependency errors for unsafe cases.

## Reproduction Assessment

Yes. Source inspection shows the default image path still calls the Sharp-backed optimizer, and the issue discussion includes a fresh packaged 2026.5.2 reproduction when Sharp is absent.

## Solution Assessment

Unclear. The diagnostic-only fix is necessary but not sufficient; maintainers should choose between reliable Sharp packaging, a guarded original-buffer fallback, or both.
