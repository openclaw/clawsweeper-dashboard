## Summary

Current main now handles the missing-Sharp path with actionable diagnostics and a conservative in-limit original-image fallback, while preserving fail-closed behavior for oversized and HEIC media.

## What This Changes

The issue asks for missing Sharp image-optimizer failures to be diagnosed clearly and for packaged installs to keep image handling working without manual dependency repair.

## Best Possible Solution

Keep the current main behavior: surface the Sharp import cause, pass through safe in-limit originals, and leave oversized or HEIC media fail-closed unless maintainers later choose a broader native-dependency packaging policy.

## Reproduction Assessment

Yes. Source inspection and regression tests show the missing optimizer path and verify current main now falls back for in-limit non-HEIC images while preserving a clear Sharp error for cases that still require optimization.

## Solution Assessment

Yes. The current implementation is a narrow maintainable fix: it avoids a new root native dependency, keeps resource guards intact, and makes the remaining failure mode actionable.
