## Summary

Keep open. Current main no longer drops every optimizer cause and has an under-cap original-image fallback, but the issue is still tied to an open direct fix PR and fresh packaged-install reports where missing Sharp still blocks normal image delivery.

## What This Changes

The issue asks OpenClaw to make missing Sharp failures actionable and keep image/media handling working in packaged installs where Sharp is absent.

## Best Possible Solution

Review and land or close #77197, then keep the plugin-local Sharp dependency path documented and actionable for packaged installs.

## Reproduction Assessment

Yes, at source level: with Sharp unavailable, normal resize work still flows through the media-understanding-core Sharp import and over-cap or HEIC paths fail rather than using the under-cap fallback. The issue discussion adds fresh packaged-install reproductions on v2026.5.2.

## Solution Assessment

Unclear. The current main diagnostic and under-cap fallback are good partial fixes, but the remaining best path depends on maintainers accepting #77197's install-hint approach and the existing plugin-local runtime-deps policy instead of making Sharp a root hard dependency.
