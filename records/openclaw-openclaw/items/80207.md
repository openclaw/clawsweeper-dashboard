## What This Changes

The PR adds an optional `inventoryFingerprint` to plugin metadata snapshots, computes it during snapshot loading, reuses it in current-snapshot fingerprint checks, and adds a stat-spy regression test plus changelog entry.

## Best Possible Solution

Land the narrow snapshot-fingerprint handoff after maintainer validation, preserving explicit snapshot lifecycle invalidation instead of adding a broad discovery cache.

## Reproduction Assessment

Yes, source-reproducible: current main passes `snapshot.index` into current-snapshot fingerprint checks, which falls through to `resolveInstalledManifestRegistryIndexFingerprint` and `fs.statSync`. The linked report also provides concrete compat-mode idle Gateway profiling output.

## Solution Assessment

Yes. Carrying the already-computed inventory fingerprint on `PluginMetadataSnapshot` is the narrowest maintainable fix and matches the plugin boundary guidance for explicit snapshot-owned metadata.
