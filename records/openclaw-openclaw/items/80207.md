## What This Changes

The PR adds an optional `inventoryFingerprint` to plugin metadata snapshots, stores it during snapshot loading, reuses it during current-snapshot fingerprint checks, adds a stat-spy regression test, and updates the changelog.

## Best Possible Solution

Land the narrow snapshot-owned inventory fingerprint handoff after maintainer validation, keeping explicit snapshot lifecycle invalidation instead of adding a process-wide discovery cache.

## Reproduction Assessment

## Solution Assessment

Yes. Carrying the already-computed inventory fingerprint on `PluginMetadataSnapshot` is the narrowest maintainable fix I see and matches the plugin boundary guidance for explicit snapshot-owned metadata.
