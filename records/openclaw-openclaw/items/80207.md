## What This Changes

The branch adds an optional `inventoryFingerprint` to plugin metadata snapshots, reuses it for current-snapshot control-plane fingerprints, adds a no-restat regression test, and updates the changelog.

## Best Possible Solution

Land the narrow snapshot-owned inventory fingerprint handoff after normal maintainer validation, without adding a broader persistent plugin discovery cache.

## Reproduction Assessment

Yes, source-reproducible: current main still passes `snapshot.index` through current-snapshot fingerprint checks, which falls back to `fs.statSync`-backed manifest/package signatures when an inventory fingerprint is not supplied. The linked issue and PR body also provide concrete live profiling and after-fix idle Gateway output.

## Solution Assessment

Yes. Carrying the already-computed inventory fingerprint on `PluginMetadataSnapshot` is the narrowest maintainable fix for this remaining current-snapshot restat path, and it matches the plugins guide's preference for explicit snapshot-owned metadata over broad persistent discovery caches.
