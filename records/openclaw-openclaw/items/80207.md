## Summary

Keep open: this is a focused performance bug-fix PR that is not already on current main, has sufficient real behavior proof, green relevant CI, and no blocking correctness finding from this review.

## What This Changes

The PR stores a plugin inventory fingerprint on metadata snapshots and reuses it during current-snapshot compatibility checks to avoid repeated bundled manifest stat walks in compat discovery mode.

## Best Possible Solution

Land the narrow snapshot-fingerprint handoff after normal maintainer validation, preserving explicit snapshot lifecycle invalidation instead of adding a broader discovery cache.

## Reproduction Assessment

Yes, source-reproducible: current main passes snapshot.index into current-snapshot fingerprint checks, which falls through to resolveInstalledManifestRegistryIndexFingerprint and fs.statSync. The linked issue also provides concrete compat-mode idle Gateway profiling output.

## Solution Assessment

Yes. Carrying the already-computed inventory fingerprint on PluginMetadataSnapshot is the narrowest maintainable fix I found, and it aligns with the explicit snapshot-owned metadata boundary.
