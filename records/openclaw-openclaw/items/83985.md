## Summary

Keep open: current main still uses the built-in 12 KiB reseed-history cap at the production CLI reseed call site, and the backend config schema has no per-backend cap field. The linked implementation PR https://github.com/openclaw/openclaw/pull/83986 is still open, so this issue should remain open until that PR lands or maintainers choose a different config contract.

## What This Changes

Expose a per-backend `maxReseedHistoryChars` config value for CLI session reseed prompt history instead of always using the built-in 12 KiB cap.

## Best Possible Solution

Review and either land or revise the linked per-backend cap PR, keeping the existing 12 KiB default for unset configs and adding focused schema, docs, and prepare-path tests.

## Reproduction Assessment

Yes for the source-level behavior: current main has a renderer override parameter but the production prepare call never passes a backend-configured value. The live Telegram/Claude JSONL scenario was not rerun in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public config contract. The proposed per-backend knob is a maintainable direction because it preserves the default and follows nearby CLI backend tuning patterns, but it is still a new user-facing config surface.
