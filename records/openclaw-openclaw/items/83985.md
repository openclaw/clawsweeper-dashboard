## Summary

Keep open: current main still has the hardcoded CLI reseed-history cap in production because the helper override is not connected to backend config, and an open linked PR is already the implementation path for this issue.

## What This Changes

The issue asks for a per-backend configuration value to control the CLI session reseed history cap instead of always using the built-in 12 KiB default.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/83986 after the public config contract is approved, or replace it with an equivalent maintainer-approved patch that preserves the current default and adds regression coverage.

## Reproduction Assessment

Yes for the source-level behavior: current main has a hardcoded fallback cap and the production call site does not pass a configured value. The live Telegram and Claude JSONL scenario was not rerun during this read-only review.

## Solution Assessment

Unclear until maintainers choose the public config contract. The proposed optional per-backend knob is a maintainable direction because it preserves the default and follows nearby CLI backend tuning patterns, but it is still new user-facing config surface.
