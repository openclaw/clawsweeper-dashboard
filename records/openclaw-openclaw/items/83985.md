Keep open: current main still has the 12 KiB CLI reseed-history cap with no per-backend config path, and the open linked implementation PR has closing syntax but has not merged. The slice-direction issue at https://github.com/openclaw/openclaw/issues/83157 is related but orthogonal, so it does not supersede this config request.

## What This Changes

Expose a per-backend CLI reseed history cap so long-context CLI backends can raise the `buildCliSessionHistoryPrompt` limit while preserving the current default for unset configs.

## Best Possible Solution

Review or rebase https://github.com/openclaw/openclaw/pull/83986 so it preserves current raw transcript reseed safety, adds the optional cap through types/schema/docs/tests, and coordinates with https://github.com/openclaw/openclaw/issues/83157 without mixing the two fixes.

## Reproduction Assessment

Yes for the source-level behavior: current main hardcodes the fallback cap and the production call site does not pass a configured value. The reporter's live Telegram/Claude JSONL scenario was not independently rerun in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public config contract. A per-backend optional cap is a maintainable direction because it preserves the default, but the linked PR must be rebased around current reseed behavior before it is the best concrete fix.
