## What This Changes

Add opt-in per-sender inbound DM rate limits for pairing and allowlist channel policies before messages consume the global queue.

## Best Possible Solution

Define a shared channel-ingress rate-limit contract, then add an opt-in Telegram implementation with docs and focused tests for pairing and allowlist DMs before queue admission while keeping defaults unchanged.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a feature/config request. Source inspection confirms the requested per-sender inbound-rate config is absent, while current pairing traffic is already blocked before Telegram dispatch and bounded by hard-coded pairing-store behavior.

## Solution Assessment

Unclear as proposed: a Telegram-only knob would help the reported deployment, but the issue frames a cross-channel ingress policy. The safer maintainable direction is a shared channel-ingress contract with plugin implementations and explicit default-off compatibility.
