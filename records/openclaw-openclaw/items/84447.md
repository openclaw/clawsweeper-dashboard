## Summary

Keep open: current main already blocks unauthorized Telegram DMs before agent dispatch and has non-configurable pairing-request TTL/cap behavior, but it does not provide the requested opt-in per-sender inbound DM rate-limit config or shared channel-ingress contract.

## What This Changes

Request to add opt-in per-sender inbound DM throttling for pairing or allowlist channel policies before messages enter the global queue.

## Best Possible Solution

Define a shared channel-ingress rate-limit contract, then add an opt-in Telegram implementation and docs/tests that preserve current default behavior while covering pairing and allowlist DMs before agent dispatch.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a feature/config request. Source inspection confirms the requested per-sender inbound-rate config is absent, while current pairing traffic is already blocked before Telegram dispatch and bounded by non-configurable pairing-store behavior.

## Solution Assessment

Unclear as proposed: a channel-local Telegram-only knob would not cover the issue's cross-channel framing, so the better solution is a shared ingress policy contract with plugin implementations and explicit defaults.
