## Summary

Keep open. Current main documents and implements useful reset/compact controls, but the requested wall-clock pruning of normal conversation history and an explicit prune-context/session-boundary mutation surface are still absent and require API/product judgment.

## What This Changes

The issue requests explicit session-boundary documentation and controls, immediate TTL-based context pruning during an active session, and gateway/API methods for pruning or resetting session context.

## Best Possible Solution

Make a maintainer API/product decision, then either document the current TTL semantics more explicitly or add an additive authenticated prune-context/session-boundary surface with Pi/context-engine behavior and focused tests.

## Reproduction Assessment

Yes, at source level: configure contextPruning.mode as cache-ttl and continue after TTL; current main only prunes eligible toolResult messages during context assembly, not normal conversation history on a wall clock.

## Solution Assessment

No, current main is not the full solution: /new, /reset, sessions.reset, and /compact cover useful controls but not TTL-based conversation-history expiry or an explicit prune-context API.
