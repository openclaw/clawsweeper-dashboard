## Summary

Keep open. Current main partially defines session lifecycle and exposes reset/compact controls, but the requested wall-clock TTL pruning of normal conversation history and explicit prune-context mutation surface are still absent and require maintainer API/product direction.

## What This Changes

Request to define session boundaries and add controllable mid-session TTL pruning/reset behavior, including an explicit prune-context API.

## Best Possible Solution

Make a maintainer product/API decision, then either document current TTL semantics more explicitly or add an additive authenticated prune-context/session-boundary surface with Pi/context-engine implementation and focused tests.

## Reproduction Assessment

Yes, at source level: configure contextPruning.mode as cache-ttl and continue after the TTL; current main only prunes eligible toolResult messages during context assembly and does not expire normal conversation history on a wall clock.

## Solution Assessment

No, current main is not the full solution: /new, /reset, sessions.reset, and /compact cover useful controls but not TTL-based conversation-history expiry or an explicit prune-context API.
