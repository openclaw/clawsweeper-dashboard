## Summary

Keep open: current main and v2026.6.8 define session lifecycle and expose reset/compact controls, but the requested immediate TTL conversation-history pruning and manual prune-context control surface remain missing and need product/security direction.

## What This Changes

The issue requests explicit session-boundary documentation, immediate TTL pruning of old context, and user/API controls to prune or reset sessions without restarting OpenClaw.

## Best Possible Solution

Define one canonical contract distinguishing tool-result pruning, conversation compaction, manual reset, and any new retention/prune policy, then expose approved controls through typed gateway/session and context-engine seams with docs and tests.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.8 have reset and compaction, but no manual prune-context method and no wall-clock conversation-history TTL expiry; this is not a strict bug under the documented contract because cache-TTL pruning is currently tool-result-only.

## Solution Assessment

No, not as standalone POST endpoints yet. The maintainable path is a maintainer-approved typed session/context contract with authorization, persistence, summarization, and recovery semantics before transport adapters are added.
