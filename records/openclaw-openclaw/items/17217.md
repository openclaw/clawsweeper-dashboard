## Summary

Keep open. Current main covers the documented session lifecycle and reset/compact controls, but cache-ttl pruning is still scoped to old tool-result trimming during context assembly and there is still no explicit prune-context/session-boundary mutation API.

## What This Changes

Request to define session boundaries, make TTL pruning expire context regardless of session length, and add explicit mid-session pruning/reset controls.

## Best Possible Solution

Choose the intended contract, then either document TTL as tool-result-only with `/compact`, `/new`, `/reset`, and `sessions.reset` as supported controls, or add an additive authenticated prune-context/session-boundary API with Pi/context implementation and regression coverage.

## Reproduction Assessment

Yes, at source level. Configure `contextPruning.mode: "cache-ttl"` and continue after the TTL; current main only considers pruning during context assembly and only rewrites eligible `toolResult` messages, not normal user/assistant history.

## Solution Assessment

No, current main is only a partial solution. Reset and compact controls exist, but wall-clock expiry of normal conversation history and a manual prune-context API are absent, so the next step is a maintainer product/API decision.
