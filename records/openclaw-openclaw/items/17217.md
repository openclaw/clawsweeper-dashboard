## Summary

Keep open. Current main covers some session-boundary controls, but the requested wall-clock conversation-history TTL pruning and explicit prune-context mutation surface are still not implemented and need maintainer API/product judgment.

## What This Changes

Request to define session boundaries, make TTL pruning apply during active sessions, and expose manual prune/reset controls without restarting OpenClaw.

## Best Possible Solution

Make a maintainer product/API decision, then either document current TTL semantics more explicitly or add an additive authenticated prune-context/session-boundary surface with Pi/context-engine implementation and focused tests.

## Reproduction Assessment

Yes, at source level: configure contextPruning.mode as cache-ttl and continue after TTL; current main only prunes eligible toolResult messages during context assembly, not normal conversation history on a wall clock.

## Solution Assessment

No, current main is not the full solution: /new, /reset, sessions.reset, and /compact cover useful controls but not TTL-based conversation-history expiry or an explicit prune-context API.
