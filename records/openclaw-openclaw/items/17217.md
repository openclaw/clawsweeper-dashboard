## Best Possible Solution

Define one canonical contract distinguishing reset, compaction, tool-result pruning, and any new retention/prune mutation, then expose approved controls through typed gateway/session protocol and docs.

## Solution Assessment

No, not as standalone POST endpoints. The maintainable solution should be one typed session/context mutation contract with explicit retention, persistence, summarization, and authorization semantics before adapters are added.
