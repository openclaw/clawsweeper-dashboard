## Best Possible Solution

Define one canonical contract that separates tool-result pruning, conversation compaction, manual reset, session TTL/rolling pruning, and authorized API controls, then implement or document that contract consistently.

## Reproduction Assessment

## Solution Assessment

No, not as standalone POST endpoints yet. The maintainable path is a maintainer-approved typed session/context contract with authorization, persistence, summarization, and recovery semantics before transport adapters are added.
