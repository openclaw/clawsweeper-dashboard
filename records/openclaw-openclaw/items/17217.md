## Summary

Keep open. Current main documents session lifecycle and reset/compact controls, but cache-ttl pruning is still defined and implemented as in-memory old tool-result trimming; it does not expire normal conversation history on wall-clock TTL or expose a prune-context mutation API.

## What This Changes

Request to define session boundaries, make contextPruning TTL expire old conversation context mid-session, and add manual prune/reset controls without restarting OpenClaw.

## Best Possible Solution

Choose the intended contract, then either document TTL as tool-result-only with /compact, /new, /reset, and sessions.reset as supported controls, or add an additive authenticated prune-context/session-boundary API with Pi/context implementation and regression coverage.

## Reproduction Assessment

Yes, at source level. With contextPruning.mode set to cache-ttl, current main only considers pruning during context assembly and pruneContextMessages only rewrites eligible toolResult messages, so normal user/assistant history older than TTL remains in context.

## Solution Assessment

No, current main is only a partial solution for the requested direction. The maintainable next step is a product/API decision on whether TTL should remain a tool-result cache optimization or become a conversation-history/session-boundary mutation surface.
