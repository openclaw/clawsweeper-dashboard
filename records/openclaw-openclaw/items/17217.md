## Summary

Keep open: current main and v2026.6.8 define session lifecycle and expose reset/compact controls, but the requested immediate TTL conversation-history pruning and manual prune-context surface are still missing and require maintainer product/security direction.

## What This Changes

The issue requests explicit session boundaries, immediate TTL-based pruning during active sessions, and user/API controls to prune or reset session context without restarting OpenClaw.

## Best Possible Solution

Define one canonical contract distinguishing ephemeral tool-result pruning, persisted conversation compaction, manual reset, and any new retention/prune policy, then implement approved mutations through typed gateway/session and context-engine lifecycle seams with docs and tests.

## Solution Assessment
