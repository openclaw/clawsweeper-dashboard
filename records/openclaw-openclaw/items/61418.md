## Summary

Keep open. Current main still has the reported mismatch: the Gateway/shared model-list path is auth-aware, but the embedded local TUI path used by `openclaw chat`, `openclaw terminal`, and `openclaw tui --local` still maps the raw allowed/full catalog.

## What This Changes

The issue requests that the TUI `/models` command show only models/providers with configured authentication, matching `openclaw models list`.

## Best Possible Solution

Reuse `resolveVisibleModelCatalog` in `EmbeddedTuiBackend.listModels()` with the correct runtime config, workspace, and auth context, then add focused embedded-backend regression coverage.

## Reproduction Assessment

Yes, at source level. The local TUI aliases route to `EmbeddedTuiBackend`, whose `listModels()` still returns the raw allowed/full catalog when no explicit allowlist narrows it.

## Solution Assessment

Yes. The narrow maintainable solution is to align the embedded backend with the existing shared visibility helper instead of adding a TUI-only filter or changing Gateway semantics.
