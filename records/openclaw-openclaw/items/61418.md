## Summary

Keep open. Current main and the latest release still have the reported local TUI mismatch: the embedded backend lists the raw allowed/full catalog, while the Gateway/shared path and docs define configured/auth-available model visibility.

## What This Changes

The issue asks for the TUI `/models` picker to show only configured or auth-backed models, matching `openclaw models list` and Gateway model-list behavior.

## Best Possible Solution

Reuse `resolveVisibleModelCatalog` in `EmbeddedTuiBackend.listModels()` with the correct runtime config, workspace, and auth context, then add focused embedded-backend regression coverage.

## Reproduction Assessment

Yes, at source level. The local TUI aliases route to `EmbeddedTuiBackend`, whose `listModels()` still returns the raw allowed/full catalog when no explicit allowlist narrows it.

## Solution Assessment

Yes. The narrow maintainable fix is to align the embedded backend with the existing shared visibility helper instead of adding a TUI-only auth filter or changing Gateway semantics.
