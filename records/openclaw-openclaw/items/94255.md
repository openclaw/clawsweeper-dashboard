## What This Changes

The issue reports corrupted persisted transcript media metadata when padded MediaTypes or MediaUrls holes are compacted and later attachment metadata shifts to an earlier attachment.

## Best Possible Solution

Keep this issue as the canonical tracker and land https://github.com/openclaw/openclaw/pull/94257, then close the issue once the fix reaches main.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: MediaTypes such as ["", "image/png"] are compacted to ["image/png"] before positional lookup, so the first attachment can receive the later type.

## Solution Assessment

Yes. Preserving array cardinality in the persisted transcript reader is the narrow owner-boundary fix because the writer and sibling media consumers already treat Media* arrays as positional.
