## Summary

Current main and shipped release v2026.4.5 solve the central outage workflow without creating a duplicate log stream: `openclaw logs` reads the configured local JSONL when the implicit local Gateway is unavailable and renders it in the existing human-readable format.

## What This Changes

Add a separately persisted plain-text log file beside the JSONL log for readable diagnostics when Gateway-backed log viewing is unavailable.

## Best Possible Solution

Keep the single bounded JSONL log canonical and direct operators to `openclaw logs`, which provides readable local fallback output when the Gateway is unavailable.

## Reproduction Assessment

No. On current main, an implicit local Gateway failure causes `openclaw logs` to read the configured file directly and render readable output, so the reported blocked diagnostic workflow no longer reproduces.

## Solution Assessment

No. The shipped local fallback is narrower and safer than duplicating every event into a second persistent file with separate rotation, retention, and redaction obligations.
