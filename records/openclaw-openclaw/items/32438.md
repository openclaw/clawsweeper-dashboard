## Summary

Keep open. Current main and the latest tagged release still persist Gateway file logs as a single redacted JSONL stream, while CLI and Control UI formatting happens only when reading that JSONL; the duplicate report and same-author implementation PRs are closed without a merged formatted sidecar.

## What This Changes

The issue requests a derived human-readable formatted log file beside the existing JSONL Gateway log so operators can inspect logs when the Control UI is unavailable.

## Best Possible Solution

Add the sidecar in the logger layer from the resolved active `logging.file`, keep the existing JSONL contract untouched, and document/test redaction plus date/rotation behavior.

## Reproduction Assessment

Yes, at source level: current main writes one redacted JSONL payload to one configured active file, and CLI/Control UI paths parse that JSONL at read time. I did not run a live Gateway because this review is read-only.

## Solution Assessment
