## Repair Work Prompt

Implement a focused OpenClaw logger enhancement for https://github.com/openclaw/openclaw/issues/32438. Current main writes only JSONL file logs; add a derived human-readable companion file next to the active `logging.file` path, inserting `-formatted` before the extension, with lines like `YYYY-MM-DD HH:mm:ss [level] [subsystem] message`. Keep the existing JSONL file, `logs.tail`, CLI, Control UI, scripts, and public config behavior unchanged; do not add a config key unless maintainers explicitly request it. Apply the same redaction boundary before writing formatted lines. Define and test behavior for default rolling date files, custom stable file paths, `logging.maxFileBytes` rotation, and pruning/disk growth. Update operator docs and `CHANGELOG.md` because this is user-facing. Stop and ask for maintainer review if the implementation would require changing the JSONL contract or adding a new config/policy knob.
