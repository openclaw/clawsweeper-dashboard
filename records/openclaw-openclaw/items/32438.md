## Summary

Keep open. Current main still writes a single redacted JSONL Gateway file and only formats log output at read time in the CLI or Control UI; the related formatted-sidecar implementation PRs are closed unmerged, and no open replacement appears to own the remaining work.

## What This Changes

Request a derived human-readable formatted log sidecar next to the existing JSONL Gateway file log for operator inspection when Control UI log tailing is unavailable.

## Best Possible Solution

Land a focused logger-layer enhancement that derives a redacted human-readable companion path from the active `logging.file`, keeps JSONL consumers unchanged, and documents the sidecar path plus retention behavior.

## Reproduction Assessment

Yes for the current behavior. Source inspection shows enabling file logging and emitting any log event writes only the configured JSONL file, with no formatted companion path in logger settings, config schema, docs, CLI, or UI.

## Solution Assessment

Yes. A derived logger-layer sidecar is the narrowest maintainable direction, provided it preserves the JSONL contract and deliberately handles redaction plus rotation/pruning.
