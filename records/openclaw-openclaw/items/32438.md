## What This Changes

The issue requests a derived human-readable formatted log file beside the existing JSONL Gateway log file.

## Best Possible Solution

Land a focused logger-layer enhancement that derives a readable companion path from `logging.file`, writes redacted plain-text lines beside unchanged JSONL, and documents/tests rotation, pruning, and consumer compatibility.

## Reproduction Assessment

Yes. Source inspection shows `buildLogger` appends one redacted JSONL payload to one active file and targeted searches found no formatted sidecar symbols, settings, or docs; I did not run a live Gateway in this read-only review.

## Solution Assessment

Yes for the requested direction. A logger-layer companion file is the narrow maintainable solution if it preserves the JSONL contract, shares redaction, and defines rotation/pruning deliberately; the existing CLI local fallback is useful but does not provide a persistent readable file.
