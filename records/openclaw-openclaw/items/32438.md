## Summary

Keep open. Current `main` and latest release still persist Gateway file logs as a single redacted JSONL stream; the CLI fallback and Control UI parse that JSONL at display time, but no durable formatted companion file exists. The earlier duplicate is closed, the same-author implementation PR is closed unmerged, and the merged date-boundary log discovery work is adjacent rather than a replacement.

## What This Changes

Add a plain-text formatted log file beside the existing JSONL Gateway file log so operators can inspect logs when the Gateway or Control UI is unavailable.
