## Summary

Keep open. Current main and v2026.5.12 still persist Gateway file logs as a single JSONL stream; the CLI and Control UI only format that JSONL at read time, and the direct implementation PRs for this request were closed unmerged.

## What This Changes

The issue requests a derived plain-text formatted log file next to the existing JSONL Gateway log so operators can inspect logs when the Control UI is unavailable.

## Best Possible Solution

Land a focused logger-layer enhancement that derives a human-readable companion path from the active `logging.file`, writes redacted plain-text lines beside the unchanged JSONL file, and documents rotation/pruning behavior.

## Reproduction Assessment

Yes, at source level: current main and v2026.5.12 write only one JSONL file and format logs only when CLI/UI read that file. I did not run a live Gateway because this was a read-only review.

## Solution Assessment

Yes for the direction: a logger-layer sidecar is the narrow maintainable solution if it preserves the JSONL contract, shares redaction, and handles rolling/rotation deliberately. Changing the existing file format or adding a competing configured path would be more disruptive.
