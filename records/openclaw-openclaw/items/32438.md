## Summary

Keep open. Current main and the latest release still persist Gateway file logs as one JSONL file, while CLI and Control UI formatting happens only when those consumers read the JSONL; the earlier duplicate and same-author implementation PRs are closed without a merged fix.

## What This Changes

Request a derived plain-text companion log file beside the existing JSONL Gateway file log so operators can tail or grep readable logs when Control UI/RPC access is unavailable.

## Best Possible Solution

Land a focused logger-layer enhancement that derives the companion path from the active `logging.file`, writes redacted plain-text lines beside unchanged JSONL, and documents/test-covers rotation, pruning, and consumer compatibility.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.5.12` shows only one JSONL file is written and formatting happens at read time through CLI/UI consumers; I did not run a live Gateway in this read-only review.

## Solution Assessment

Yes for the requested direction. A logger-layer companion file is the narrow maintainable path if it preserves the JSONL contract, shares redaction, and defines rotation/pruning behavior deliberately.
