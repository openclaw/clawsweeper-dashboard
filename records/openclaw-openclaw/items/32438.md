## Summary

Keep open. Current main and v2026.5.12 still persist Gateway file logs as a single JSONL stream; CLI and Control UI readability is produced at read time, and the direct formatted-sidecar PRs were closed unmerged.

## What This Changes

Request adding a derived plain-text formatted companion log file beside `logging.file` while keeping the existing JSONL log and consumers unchanged.

## Best Possible Solution

Land a focused logger-layer enhancement that derives a human-readable companion path from the active `logging.file`, writes redacted plain-text lines beside the unchanged JSONL file, and documents rotation/pruning behavior.

## Solution Assessment

Yes, the logger-layer sidecar is the narrow maintainable direction if it preserves the JSONL contract, shares redaction, and defines rolling/rotation behavior. The closed PRs are useful context but should not be cherry-picked blindly.
