## Summary

Keep open. Current main and the latest release still persist Gateway file logs as a single JSONL stream, while CLI and Control UI formatting happens only when reading that JSONL file; the duplicate report and two same-author implementation PRs are closed without a merged sidecar fix.

## What This Changes

The issue requests a derived human-readable formatted log file alongside the existing JSONL Gateway log so operators can inspect logs when the Control UI is unavailable.

## Best Possible Solution

Add the sidecar at the logger layer, derive it from the resolved active file path, preserve the JSONL file and all existing consumers, and cover redaction plus rolling-log lifecycle in docs and tests.

## Reproduction Assessment

Yes, at source level: current main writes one redacted JSONL payload to one resolved log file, and both CLI and Control UI read/format that JSONL at display time. I did not run a live Gateway because this review is read-only.

## Solution Assessment

Yes for the requested direction: a logger-layer companion file is the narrowest maintainable path because it preserves the existing JSONL contract and works when Gateway RPC/UI paths are unavailable. The implementation should decide rotation/pruning deliberately and reuse the existing redaction sink.
