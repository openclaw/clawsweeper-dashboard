## Summary

Keep open. Current main and v2026.5.12 still persist Gateway file logs as a single JSONL stream, while CLI and Control UI formatting happens only at read time; the direct implementation attempts at https://github.com/openclaw/openclaw/pull/27446 and https://github.com/openclaw/openclaw/pull/32442 were both closed unmerged.

## What This Changes

Add a derived plain-text formatted companion log file beside the existing JSONL file without changing the JSONL path, format, CLI, Control UI, or scripts.

## Best Possible Solution

Land a focused logger-layer enhancement that derives the formatted companion path from `logging.file`, writes redacted plain-text lines beside unchanged JSONL, and documents/test-covers rotation, pruning, and read-time consumer compatibility.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.5.12 write only one JSONL log file and format that JSONL only when CLI or Control UI reads it; I did not run a live Gateway in this read-only review.

## Solution Assessment

Yes for the requested direction. A logger-layer sidecar is the narrow maintainable solution if it preserves the JSONL contract, shares redaction, and handles rolling/rotation deliberately; changing the existing JSONL file format would be more disruptive.
