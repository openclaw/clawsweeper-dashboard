## Summary

Keep open. Current main and latest release still persist Gateway file logs as JSONL only; CLI, channel logs, and the Control UI format those JSONL lines at read/display time but do not create the requested durable formatted companion file.

## What This Changes

Add a derived human-readable formatted log file beside the existing JSONL Gateway file log, without changing the existing JSONL path or consumers.

## Best Possible Solution

Land a focused logger-layer enhancement that derives a readable companion path from the resolved active JSONL file, writes redacted plain-text lines beside the unchanged JSONL stream, and documents the exact rotation/retention behavior.

## Reproduction Assessment

Yes. Enabling file logging and emitting a log event reaches `buildLogger`, which currently appends one redacted JSON payload to one active file; no current source, docs, or tests derive or write a formatted companion file.

## Solution Assessment

Yes. A logger-layer sidecar is the narrowest durable solution for `tail`/`grep` use when the Gateway or Control UI is unavailable; CLI/UI display formatting helps live reads but does not solve direct disk inspection.
