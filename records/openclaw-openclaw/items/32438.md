## Summary

Keep open. Current main still persists Gateway file logs as one redacted JSONL stream; CLI and Control UI formatting happen while reading that stream, the earlier implementation PR was closed unmerged, and the merged date-boundary log PR does not add the requested durable formatted companion file.

## What This Changes

The issue requests a derived human-readable formatted log file next to the existing JSONL Gateway log when file logging is enabled.

## Best Possible Solution

Land a focused logger-layer enhancement that writes redacted plain-text companion lines beside unchanged JSONL logs, with docs and regression coverage for redaction, rolling dates, rotation/pruning, and unchanged CLI/Control UI consumers.

## Reproduction Assessment

## Solution Assessment

Yes. A logger-layer derived companion file is the narrowest durable solution for disk inspection when Gateway or Control UI access is unavailable; CLI/UI pretty printing still depends on reading the JSONL stream.
