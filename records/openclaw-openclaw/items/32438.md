## Summary

Keep open. Current main and v2026.5.12 still persist Gateway file logs as one JSONL stream, while CLI and Control UI readability is produced by parsing that JSONL at read time; the related formatted-sidecar PRs are closed unmerged, and the merged date-boundary log discovery PR is adjacent rather than a replacement.

## What This Changes

The issue asks for an automatically derived plain-text formatted companion log file beside the existing JSONL Gateway file log, without changing JSONL consumers or adding config.

## Best Possible Solution

Land a focused logger-layer enhancement that derives a human-readable companion path from the active `logging.file`, writes redacted plain-text lines beside the unchanged JSONL file, handles rolling date changes and rotation/pruning deliberately, and updates operator docs plus regression coverage.

## Reproduction Assessment

Yes for the current behavior: source inspection of current main and v2026.5.12 shows the logger writes only JSONL file logs and CLI/UI format them later for display. No live Gateway run was performed because this was a read-only review.

## Solution Assessment

Yes, the requested logger-layer sidecar is the narrowest maintainable direction if it preserves the JSONL contract, shares redaction, and defines rolling/rotation behavior. The closed PRs are useful context but should not be cherry-picked blindly.
