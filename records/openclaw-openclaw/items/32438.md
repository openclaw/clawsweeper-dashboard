Request a derived human-readable formatted companion log file beside the existing JSONL Gateway file log without changing existing JSONL consumers.

## Best Possible Solution

Implement the sidecar in the logger layer from the active `logging.file`, reuse existing redaction and timestamp/subsystem parsing, keep JSONL consumers unchanged, and document the exact path plus retention behavior.
