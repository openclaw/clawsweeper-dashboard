## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Escape validator property names as JSON strings:** `packages/gateway-protocol/src/index.ts:940`
  - body: The new formatter wraps raw property names in double quotes. If a client or model sends an unexpected key containing a double quote, the model-facing message is not a valid JSON string token, for example `foo"bar` becomes `unexpected property "foo"bar"`, which recreates the delimiter-copy ambiguity this PR is meant to remove. Use a shared JSON string quoting helper for the gateway and plugin validator messages and add a quote-containing property-name regression.
  - confidence: 0.87
- **[P2] Use the post-limit count for no-op after hooks:** `src/agents/embedded-agent-runner/compact.ts:1411`
  - body: In the direct path, transcript sanitizing/history limiting runs before this no-real-conversation branch, and the before-hook metrics use the current limited `session.messages`. Passing `messageCountOriginal` here can emit an after event whose `messageCount` is larger than the paired before event on a successful no-op, which breaks observers that rely on the lifecycle counts. Pass the current `session.messages.length` or before count and cover a limited heartbeat-only transcript.
  - confidence: 0.82
