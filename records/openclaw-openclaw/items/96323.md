## Summary

This PR is useful but not merge-ready: current main still has unbounded OpenAI success-body reads, but this patch adds an unproven Batch output ceiling and bypasses shared provider response helpers, while the submitted proof only exercises the bounded-reader primitive rather than the changed Batch, OAuth, and Realtime paths.

## What This Changes

The PR changes OpenAI embedding Batch status/file content, ChatGPT OAuth token exchange, and Realtime client-secret success responses to use byte-capped reads via `readResponseWithLimit`.

## Best Possible Solution

Land a bounded-read fix that reuses shared provider JSON/text helpers and either streams/splits Batch output or has maintainer-approved cap proof plus changed-path real behavior proof.

## Solution Assessment

No, not yet. The direction is right, but the best fix should reuse shared provider readers where applicable and settle the Batch output cap or streaming strategy before changing success-path behavior.
