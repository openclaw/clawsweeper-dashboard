## Summary

Close as implemented on current main: the denied-PDF hot path is now skipped before the PDF factory is built, automatic PDF model/auth resolution is deferred until execution, and the direct fix landed after v2026.5.2 in #76773.

## What This Changes

The issue asks OpenClaw to stop initializing the PDF tool/pdfjs path on every agent turn, especially when `tools.deny` blocks `pdf`.

## Best Possible Solution

Keep the current lazy, denylist-aware implementation and include it in the next release so v2026.5.2 users can upgrade off the hot-path stall.

## Reproduction Assessment

No current-main failing reproduction remains: source and tests now show denied PDF factories are skipped and PDF model resolution is deferred. The reporter's v2026.5.2 trace matches the previously fixed hot path covered by #76728 and #76773.

## Solution Assessment

Yes. The current fix is the narrowest maintainable path because it reuses existing tool policy semantics and avoids adding a PDF async mode or new config knob.
