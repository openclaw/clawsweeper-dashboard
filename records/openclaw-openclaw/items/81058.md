## Summary

Keep open. Current main still strips historical reasoning for every strict `openai-completions` replay path, and the bundled SGLang provider currently has no provider-owned replay policy to opt Kimi-style preserved reasoning out; because that default was intentionally added to fix a separate Qwen/oMLX/vLLM regression, this needs a narrow maintainer decision rather than cleanup closure.

## What This Changes

The issue reports empty user-visible responses from SGLang-hosted Kimi 2.6/OpenAI-compatible reasoning models after OpenClaw began stripping prior assistant reasoning from all Chat Completions replay history.

## Best Possible Solution

Add a narrowly scoped provider/model replay policy for SGLang-hosted Kimi-style preserved-reasoning models, or a documented generic compat seam, while keeping the Qwen/oMLX/VLLM historical-reasoning strip behavior intact.

## Reproduction Assessment

No high-confidence live reproduction of the empty-response symptom is available. The source-level policy path is reproducible: SGLang uses `openai-completions`, lacks a replay hook, and therefore inherits `dropReasoningFromHistory: true` on current main.

## Solution Assessment

Unclear. Reverting the broad default is not the safest fix because it intentionally fixed Qwen/oMLX/VLLM replay failures; the maintainable path is likely a targeted SGLang/Kimi or model-compat replay exception.
