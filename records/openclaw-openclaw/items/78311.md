Keep open: current main still derives the Control UI quick Settings Exec policy badge from `agents.defaults.exec.security`, while the documented/runtime config contract uses `tools.exec.security`. The linked open PRs are implementation candidates, not a reason to close before one merges.

## What This Changes

The issue asks the Control UI Settings Exec Policy display to read the canonical `tools.exec.security` config key instead of the stale `agents.defaults.exec.security` path.

## Best Possible Solution
