## Summary

Keep open: current main still has the reported memory-flush gap, and the issue is paired with an open same-author fix PR that references it with closing syntax. Cleanup should wait for https://github.com/openclaw/openclaw/pull/83178 to land or be closed with an alternative.

## What This Changes

The issue requests adding the existing transcript-token estimator fallback to the memory-flush path so providers without usage data, such as MiniMax, do not leave session totalTokens undefined.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/83178 and land a guarded estimator fallback with focused regression coverage, or close that PR with a concrete alternative before closing this issue.

## Reproduction Assessment

Yes, by source inspection: create a non-CLI session with missing or stale totalTokens and a transcript with no usage records, then runMemoryFlushIfNeeded leaves tokenCountForFlush undefined because it never calls the transcript estimator. I did not execute a live MiniMax run.

## Solution Assessment

Yes on direction, with review caveat: reusing estimatePromptTokensFromSessionTranscript in runMemoryFlushIfNeeded is the narrow maintainable fix, but it should be guarded so byte-size-only transcript reads do not accidentally become usage-estimation paths.
