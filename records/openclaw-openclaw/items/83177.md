## What This Changes

The issue requests a memory-flush fallback so MiniMax or other providers without usage data do not leave session `totalTokens` undefined and repeatedly compact.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/83178 or a replacement that adds the estimator fallback only on the intended transcript-usage path, with regression coverage for missing usage and byte-size-only snapshots, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection shows a non-CLI session with stale or missing `totalTokens` and no transcript usage records leaves `tokenCountForFlush` undefined because memory flush never calls the existing transcript estimator.

## Solution Assessment

Unclear as submitted. Reusing `estimatePromptTokensFromSessionTranscript` is the right narrow direction, but the linked PR still needs the fallback gated to transcript-usage reads and covered against the byte-size-only snapshot path.
