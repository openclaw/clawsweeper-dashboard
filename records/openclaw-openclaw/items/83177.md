## What This Changes

The issue asks for MiniMax/no-usage sessions to derive `totalTokens` in the memory-flush path using the existing transcript estimator fallback.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83178, or a narrow replacement that applies the estimator only on the intended transcript-usage path with regression coverage, then close this issue after merge.

## Reproduction Assessment

Yes, source-reproducible: current main still leaves `tokenCountForFlush` undefined when the session has stale or missing `totalTokens` and the transcript usage snapshot has no prompt tokens. I did not run a live MiniMax/WebChat reproduction in this read-only pass.

## Solution Assessment

Yes in direction: reusing `estimatePromptTokensFromSessionTranscript` in the memory-flush transcript-usage path is the narrow maintainable fix. The current linked PR still needs the fallback guard narrowed so byte-size-only snapshots do not change semantics.
