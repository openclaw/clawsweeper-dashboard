Overall confidence: 0.88

Full review comments:

- **[P1] Keep route-backed main sessions mirrorable:** `src/gateway/server-methods/webchat-mirror.ts:203-210`
  - body: `resolveWebchatMirrorTarget` returns before consulting the stored external route whenever the session head is channel-agnostic. Current routing can store a valid external `deliveryContext` on main/direct sessions, so those WebChat replies would never register a mirror target.
  - confidence: 0.91
- **[P2] Preserve provider-specific group targets:** `src/gateway/server-methods/webchat-mirror.ts:122-123`
  - body: This rebuilds group/channel sessions as generic `group:<peer>` or `channel:<peer>`. Providers such as Telegram and Discord require provider-specific sendable forms, so mirrored group replies can fail or route to the wrong destination.
  - confidence: 0.88
- **[P1] Normalize WhatsApp DM echo keys:** `src/gateway/server-methods/webchat-mirror.ts:128-129`
  - body: Direct WhatsApp sessions are rewritten to `whatsapp:<peer>`, and the mirror dedupe marker uses that value as the peer id. WhatsApp inbound contexts use the bare `OriginatingTo`, so echoed mirrored DMs can miss dedupe and be processed again.
  - confidence: 0.85
- **[P2] Trim command text before mirroring:** `src/gateway/server-methods/chat.ts:1378`
  - body: This guard only skips messages that literally start with `/`, while OpenClaw command detection trims first. Inputs such as ` /help` can execute locally and also be mirrored to the external chat.
  - confidence: 0.93
- **[P1] Preserve Matrix clean-shutdown persistence:** `extensions/matrix/src/matrix/monitor/index.ts:142`
  - body: Replacing `releaseSharedClientInstance(client, "persist")` with `stopSharedClientInstance(client)` skips `stopAndPersist`, so a normal restart can lose the restart-safe sync marker and skip backlog.
  - confidence: 0.86
