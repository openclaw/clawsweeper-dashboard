## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve Matrix sync persistence on shutdown:** `extensions/matrix/src/matrix/monitor/index.ts:143`
  - body: This replaces the monitor cleanup path with `stopSharedClientInstance(client)`, which only calls `client.stop()`. Current main uses `releaseSharedClientInstance(client, "persist")` so `stopAndPersist()` completes; without that persisted sync state, Matrix restarts can fall back to dropping pre-startup backlog messages.
  - confidence: 0.9
- **[P1] Stop reading Discord message type as channel type:** `extensions/discord/src/monitor/message-utils.ts:134-136`
  - body: `rawData.type` on Discord messages is the message type, not the channel type. Treating it as `ChannelType` means normal messages can override missing REST metadata and route DMs/group DMs through the wrong policy path.
  - confidence: 0.88
- **[P1] Avoid defaulting unknown no-guild events to DMs:** `extensions/discord/src/monitor/message-handler.preflight.ts:309-310`
  - body: Group DMs also omit `guild_id`. When channel lookup fails and the payload has no reliable channel type, this fallback coerces the event to a DM, bypassing group-DM handling and potentially collapsing separate group chats into direct-session routing.
  - confidence: 0.82
- **[P2] Parse mirror peers without truncating colon IDs:** `src/gateway/server-methods/webchat-mirror.ts:186-188`
  - body: The resolver splits session rest with `split(":", 3)`, so peers that legitimately contain colons, such as Matrix room IDs, are truncated before the mirror target is rebuilt. Webchat replies for those sessions can be delivered to the wrong destination.
  - confidence: 0.86
- **[P2] Suppress mirrored commands after trimming input:** `src/gateway/server-methods/chat.ts:1375`
  - body: This guard checks only `rawMessage.startsWith("/")`, while command detection trims first. Inputs like `" /help"` still execute as local commands but would be mirrored to the external conversation.
  - confidence: 0.9
- **[P2] Keep mirrored group targets sendable:** `src/gateway/server-methods/webchat-mirror.ts:113-114`
  - body: The generic `group:${peer}` target built here is not accepted by every channel's send parser; for example Telegram only strips `group:` after a `telegram:` prefix and Discord does not accept `group:` at all. Prefer the stored delivery context or channel-specific target normalization.
  - confidence: 0.82
- **[P2] Normalize WhatsApp dedupe peers to inbound form:** `src/gateway/server-methods/webchat-mirror.ts:161-165`
  - body: Mirrored WhatsApp DMs are marked seen with `peerId: params.target.to`, which this resolver can prefix as `whatsapp:<jid>`. WhatsApp inbound contexts use the bare `msg.from` as `OriginatingTo`, so echo dedupe will miss and can reprocess mirrored messages.
  - confidence: 0.8
- **[P2] Keep Tlon media uploads functional:** `extensions/tlon/src/urbit/upload.ts:35`
  - body: The Tlon upload change stops uploading fetched media to Tlon storage and returns the fetched URL instead. That regresses `sendMedia` for images that need to be hosted by Tlon before sending.
  - confidence: 0.84
