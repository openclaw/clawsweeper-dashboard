## Summary

Keep this PR open. The WebChat/Control UI routing problem is real on current main, but this branch is not merge-ready because the useful Gateway fix is mixed with Discord, WhatsApp, Matrix, SDK, Tlon, docs, and lockfile changes, and the current PR head still has concrete routing and security regressions.

## What This Changes

This PR adds WebChat/Control UI mirroring for channel-backed sessions, Discord DM/proxy handling, WhatsApp proxy configuration/docs, and broad channel/SDK/test cleanup across Discord, Matrix, Tlon, WhatsApp, Gateway, and lockfile surfaces.

## Best Possible Solution

Split or replace this with a narrow Gateway/session fix for Control UI replies to externally backed sessions, then handle Discord proxy/DM hardening, WhatsApp proxy config, and Matrix/SDK cleanup in separately owned PRs.

## Reproduction Assessment

Yes. Current main has a high-confidence static reproduction: WebChat-originated `chat.send` still resolves as the internal WebChat channel, and the WebChat docs do not describe external-channel mirroring for Control UI replies.

## Solution Assessment

No. The WebChat mirror direction is useful, but this PR is not the best way to solve it because it combines multiple ownership domains and introduces concrete routing and security regressions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Do not default metadata misses to direct DMs:** `extensions/discord/src/monitor/message-handler.preflight.ts:309-312`
  - body: When Discord channel lookup fails and `guild_id` is absent, this forces DM semantics. Group DMs also omit `guild_id`, so this can bypass group-DM policy and collapse group conversations into direct-user routing.
  - confidence: 0.86
- **[P1] Ignore message types when deriving channel type:** `extensions/discord/src/monitor/message-utils.ts:132-139`
  - body: `message.rawData.type` is a Discord message type, not a channel type. Using it as `ChannelType` can make real DMs look non-DM or apply arbitrary channel semantics when REST metadata is unavailable.
  - confidence: 0.9
- **[P2] Skip mirror registration for delivered agent runs:** `src/gateway/server-methods/agent.ts:512-520`
  - body: For agent RPC calls with `deliver: true`, normal outbound delivery already sends the final reply. Registering `webchatMirrorTarget` here makes the lifecycle handler mirror the same text again.
  - confidence: 0.84
- **[P2] Preserve full peer IDs when resolving mirror targets:** `src/gateway/server-methods/webchat-mirror.ts:92-94`
  - body: Splitting session keys on every colon truncates peer IDs such as Matrix room IDs, so mirrored replies can target `channel:!room` instead of the full room ID.
  - confidence: 0.82
- **[P2] Trim command text before mirroring externally:** `src/gateway/server-methods/chat.ts:1378`
  - body: This guard checks the raw untrimmed message for `/`, while command parsing trims first. Inputs like ` /help` can execute as local commands and still be mirrored to the external chat.
  - confidence: 0.88
- **[P1] Preserve Matrix sync persistence on shutdown:** `extensions/matrix/src/matrix/monitor/index.ts:142`
  - body: Replacing the persist release path with a plain stop drops the awaited `stopAndPersist()` path. Graceful Matrix restarts can then look like cold starts and skip backlog replay.
  - confidence: 0.86
- **[P1] Keep Matrix lifecycle setters on the live registry:** `extensions/matrix/src/matrix/thread-bindings.ts:65-66`
  - body: The new local manager registry diverges from the exported lifecycle setters, so session lifecycle commands can update an empty registry instead of the active Matrix bindings.
  - confidence: 0.84
