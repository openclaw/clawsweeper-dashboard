## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve Matrix sync persistence on shutdown:** `extensions/matrix/src/matrix/monitor/index.ts:142`
  - body: This replaces the Matrix monitor's persistent release path with `stopSharedClientInstance`, so a normal monitor shutdown no longer calls `stopAndPersist()`. Current Matrix tests assert the persist release mode because otherwise restarts can lose saved sync state and skip backlog delivery.
  - confidence: 0.93
- **[P1] Keep Matrix binding setters on the exported registry:** `extensions/matrix/src/matrix/thread-bindings.ts:66`
  - body: The new manager registry lives in `thread-bindings.ts`, but the public Matrix runtime/API barrels still export the lifecycle setters from `thread-bindings-shared.ts`. `/session idle` and `/session ttl` would update the stale shared registry instead of the managers created by this file.
  - confidence: 0.88
- **[P2] Trim webchat input before suppressing slash-command mirrors:** `src/gateway/server-methods/chat.ts:1375`
  - body: The mirror guard checks `rawMessage.startsWith("/")`, while command parsing trims first. Inputs like ` /help` or `\n/new` still execute as local commands but would be mirrored to the external chat, leaking operator-only commands to users.
  - confidence: 0.9
- **[P2] Preserve full session peer ids when deriving mirror targets:** `src/gateway/server-methods/webchat-mirror.ts:176`
  - body: `split(":", 3)` truncates channel peer ids that legitimately contain colons, such as Matrix room ids (`!room:example.org`). The derived mirror target can then point at `channel:!room` instead of the real room id, so webchat replies route to the wrong destination.
  - confidence: 0.86
