## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep route-backed main sessions mirrorable:** `src/gateway/server-methods/webchat-mirror.ts:206-207`
  - body: `canMirrorToExternalRoute` rejects channel-agnostic scopes before falling back to the stored delivery route. Current routing still uses `agent:<id>:main` for default DMs and `agent:<id>:direct:<peer>` for per-peer DMs, so those sessions never mirror even when their session entry has a valid external `deliveryContext`.
  - confidence: 0.91
- **[P2] Preserve channel prefixes for group mirror targets:** `src/gateway/server-methods/webchat-mirror.ts:122-123`
  - body: For group and channel-shaped sessions this rebuilds the outbound target as `group:<peer>` or `channel:<peer>` for every provider. That breaks providers whose send parsers require provider-scoped forms, such as Telegram topics/groups needing `telegram:group:...` and LINE expecting `line:group:...`, so mirrored replies can be sent to the wrong destination or fail.
  - confidence: 0.88
- **[P2] Trim command text before mirroring:** `src/gateway/server-methods/chat.ts:1378`
  - body: This check only skips raw messages that literally start with `/`, but OpenClaw command detection normalizes and trims first. Inputs like ` /help` or a newline before `/new` will still execute as local commands and also be mirrored to the external chat.
  - confidence: 0.93
- **[P2] Await mirrored input before dispatching the turn:** `src/gateway/server-methods/chat.ts:1379-1387`
  - body: The operator input mirror is fired and forgotten, then the agent dispatch starts immediately. On a slow or rate-limited external channel, the assistant's mirrored reply can arrive before the operator message that triggered it, breaking the bidirectional transcript order this feature is trying to restore.
  - confidence: 0.82
- **[P1] Await Matrix sync persistence during cleanup:** `extensions/matrix/src/matrix/monitor/index.ts:142`
  - body: Replacing the awaited `releaseSharedClientInstance(client, "persist")` path with `stopSharedClientInstance(client)` makes monitor shutdown resolve without waiting for the clean sync-store flush. A quick restart can therefore see no trusted persisted sync state and drop Matrix backlog that arrived during the restart window.
  - confidence: 0.86
