## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep route-backed main sessions mirrorable:** `src/gateway/server-methods/webchat-mirror.ts:206-207`
  - body: `canMirrorToExternalRoute` rejects channel-agnostic scopes before consulting the stored delivery route. Current routing can still use `agent:<id>:main` and `agent:<id>:direct:<peer>` for routed DMs, so those webchat sessions never mirror even when their session entry has a valid external `deliveryContext`.
  - confidence: 0.91
- **[P2] Preserve provider prefixes for group mirror targets:** `src/gateway/server-methods/webchat-mirror.ts:122-123`
  - body: For group/channel-shaped sessions this rebuilds every outbound target as `group:<peer>` or `channel:<peer>`. Providers such as Telegram and LINE require provider-scoped forms like `telegram:group:...` or `line:group:...`, so mirrored replies can fail or go to the wrong target.
  - confidence: 0.88
- **[P2] Trim command text before mirroring:** `src/gateway/server-methods/chat.ts:1378`
  - body: This guard only skips raw messages that literally start with `/`, but OpenClaw command detection trims before checking. Inputs like ` /help` or a newline before `/new` can be handled as local commands and also mirrored to the external chat.
  - confidence: 0.93
- **[P2] Await mirrored input before dispatching the turn:** `src/gateway/server-methods/chat.ts:1379-1387`
  - body: The operator-input mirror is fired and forgotten before `dispatchInboundMessage` starts. On a slow or rate-limited external channel, the assistant's mirrored reply can arrive before the operator message that triggered it, breaking transcript order.
  - confidence: 0.82
- **[P1] Await Matrix sync persistence during cleanup:** `extensions/matrix/src/matrix/monitor/index.ts:142`
  - body: Replacing the awaited `releaseSharedClientInstance(client, "persist")` path with `stopSharedClientInstance(client)` lets monitor shutdown resolve without waiting for the clean sync-store flush. A quick restart can therefore lose the trusted persisted sync marker and drop Matrix backlog that arrived during the restart window.
  - confidence: 0.86
