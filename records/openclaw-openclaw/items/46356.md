## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks the central WebChat-to-originating-channel mirror behavior, so the underlying user problem is real, but this branch mixes that fix with Discord, WhatsApp, Matrix, SDK, dependency, and docs churn while introducing multiple blocking routing, lifecycle, and security regressions.

## What This Changes

This PR adds Control UI/WebChat mirroring for channel-backed sessions, Discord DM/proxy handling, WhatsApp proxy configuration/docs, and broad Matrix/Tlon/SDK/test cleanup across 96 files.

## Best Possible Solution

Split this into owner-reviewed pieces: a narrow Gateway/session change for Control UI replies to externally backed sessions that preserves command and route-safety guards, then separate Discord proxy/DM hardening, WhatsApp proxy config, and Matrix/Tlon/SDK cleanup PRs with focused tests and live or near-live channel proof where feasible.

## Reproduction Assessment

Yes. The underlying issue has a high-confidence static reproduction from current main: WebChat is treated as an internal surface, routing-policy coverage prevents internal WebChat turns from routing externally without explicit delivery, and the docs still describe WebChat replies as returning to WebChat.

## Solution Assessment

No. The WebChat mirror direction is useful, but this PR is not the best way to solve it because it combines unrelated channel/plugin/dependency work and introduces concrete Discord, Matrix, and command-mirroring regressions; a smaller Gateway-first fix is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Do not default metadata misses to direct DMs:** `extensions/discord/src/monitor/message-handler.preflight.ts:313-314`
  - body: When Discord channel lookup fails and guild_id is absent, this forces DM semantics. Group DMs also omit guild_id, so this can bypass group-DM policy and collapse group conversations into direct-user routing.
  - confidence: 0.86
- **[P1] Ignore message types when deriving channel type:** `extensions/discord/src/monitor/message-utils.ts:133-139`
  - body: message.rawData.type is a Discord message type, not a channel type. Using it as ChannelType can make real DMs look non-DM or apply arbitrary channel semantics when REST metadata is unavailable.
  - confidence: 0.9
- **[P2] Skip mirror registration for delivered agent runs:** `src/gateway/server-methods/agent.ts:509-516`
  - body: For agent RPC calls with deliver: true, normal outbound delivery already sends the final reply. Registering webchatMirrorTarget here makes the lifecycle handler mirror the same text again.
  - confidence: 0.84
- **[P2] Preserve full peer IDs when resolving mirror targets:** `src/gateway/server-methods/webchat-mirror.ts:85-97`
  - body: Splitting the session key on every colon truncates peer IDs such as Matrix room IDs. A key like agent:main:matrix:channel:!room:example.org can resolve to channel:!room instead of the full room ID.
  - confidence: 0.82
- **[P2] Trim command text before mirroring externally:** `src/gateway/server-methods/chat.ts:1375`
  - body: This guard checks the untrimmed message for '/', while command parsing trims first. Inputs like ' /help' can execute as local commands and still be mirrored to the external chat.
  - confidence: 0.88
- **[P1] Preserve Matrix sync persistence on shutdown:** `extensions/matrix/src/matrix/monitor/index.ts:143`
  - body: Replacing releaseSharedClientInstance(client, "persist") with stopSharedClientInstance(client) drops stopAndPersist(). Graceful Matrix restarts can then look like cold starts and skip backlog replay.
  - confidence: 0.86
- **[P1] Keep Matrix lifecycle setters on the live registry:** `extensions/matrix/src/matrix/thread-bindings.ts:65-66`
  - body: This file creates a new local manager registry while exported lifecycle setters still use thread-bindings-shared.ts. Session lifecycle commands can update an empty registry instead of the active Matrix bindings.
  - confidence: 0.84
