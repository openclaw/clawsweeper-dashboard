## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep route-backed main sessions mirrorable:** `src/gateway/server-methods/webchat-mirror.ts:203-210`
  - body: `resolveWebchatMirrorTarget` rejects channel-agnostic scopes before consulting the stored delivery route. Current DM routing still defaults to `agent:<id>:main`, so sessions with a valid external `deliveryContext` never register a mirror target.
  - confidence: 0.91
- **[P2] Preserve provider prefixes for group mirror targets:** `src/gateway/server-methods/webchat-mirror.ts:122-123`
  - body: For group/channel-shaped sessions this rebuilds targets as `group:<peer>` or `channel:<peer>`. Providers such as Telegram require provider-scoped forms like `telegram:group:...`, so mirrored replies can fail or route to the wrong destination.
  - confidence: 0.88
- **[P2] Trim command text before mirroring:** `src/gateway/server-methods/chat.ts:1378`
  - body: This guard only skips raw messages that literally start with `/`, while OpenClaw command detection trims first. Inputs like ` /help` can execute locally and also be mirrored to the external chat.
  - confidence: 0.93
- **[P1] Await Matrix sync persistence during cleanup:** `extensions/matrix/src/matrix/monitor/index.ts:142`
  - body: Replacing the awaited `releaseSharedClientInstance(client, "persist")` path with `stopSharedClientInstance(client)` lets monitor shutdown complete without the clean sync-store persist, so a restart can lose the restart-safe sync marker and skip backlog.
  - confidence: 0.86
