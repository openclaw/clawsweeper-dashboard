## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Ignore message type in Discord channel fallback:** `extensions/discord/src/monitor/message-utils.ts:136-139`
  - body: `rawData.type` on a Discord message is the message type, not the channel type. When REST channel lookup is unavailable, a normal DM with `type: 0` can be treated as a guild text channel, skipping DM/group-DM policy handling and undermining the proxy fallback this PR adds.
  - confidence: 0.9
- **[P2] Trim webchat text before suppressing command mirrors:** `src/gateway/server-methods/chat.ts:1378`
  - body: This guard checks the untrimmed raw text, but command detection trims before recognizing slash commands. Inputs like ` /help` or `\n/new` will execute as local commands and still be mirrored into the external chat, leaking operator-only command text.
  - confidence: 0.94
- **[P2] Preserve full peer IDs when parsing mirror sessions:** `src/gateway/server-methods/webchat-mirror.ts:94-106`
  - body: Splitting the session key on every colon truncates valid peer IDs such as Matrix room IDs (`!room:example.org`) before building the mirror target. Because the derived target is returned before the stored route fallback, webchat replies can be sent to `channel:!room` instead of the real room.
  - confidence: 0.88
- **[P2] Skip mirror registration for delivered agent runs:** `src/gateway/server-methods/agent.ts:512-520`
  - body: The agent RPC can already deliver final output through the normal outbound path when `deliver: true`; registering a mirror target unconditionally means the lifecycle finalizer can send the same text again to the external conversation. Gate mirror registration on the resolved delivery plan or avoid mirroring already-delivered runs.
  - confidence: 0.83
- **[P2] Register mirror context before non-agent replies finish:** `src/gateway/server-methods/chat.ts:1398-1404`
  - body: The chat path only records `webchatMirrorTarget` inside `onAgentRunStart`, so command/policy replies that finish without starting an agent run only broadcast back to webchat. Those synchronous final replies need to mirror to the external route too, otherwise the external conversation sees the operator input but misses the immediate response.
  - confidence: 0.81
- **[P1] Preserve Matrix sync state on monitor shutdown:** `extensions/matrix/src/matrix/monitor/index.ts:134-144`
  - body: Replacing the shared-client release path with `stopSharedClientInstance` drops the `stopAndPersist()` shutdown path. A graceful Matrix monitor restart can then lose its persisted sync cursor and treat backlog messages as pre-startup history instead of replaying them.
  - confidence: 0.86
