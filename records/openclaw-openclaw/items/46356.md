## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep route-backed main sessions mirrorable:** `src/gateway/server-methods/webchat-mirror.ts:203-210`
  - body: `resolveWebchatMirrorTarget` rejects channel-agnostic scopes before consulting the stored delivery route. Current routing can store a valid external `deliveryContext` on main-scoped sessions, so those sessions never register a mirror target.
  - confidence: 0.91
- **[P2] Preserve provider-specific group targets:** `src/gateway/server-methods/webchat-mirror.ts:122-123`
  - body: This rebuilds every group/channel session as `group:<peer>` or `channel:<peer>`. Providers such as Telegram require provider-scoped forms, so mirrored group replies can fail or route to the wrong destination.
  - confidence: 0.88
- **[P2] Trim command text before mirroring:** `src/gateway/server-methods/chat.ts:1378`
  - body: This guard only skips messages that literally start with `/`, while OpenClaw command detection trims first. Inputs like ` /help` can execute locally and also be mirrored to the external chat.
  - confidence: 0.93
- **[P1] Preserve Matrix clean-shutdown persistence:** `extensions/matrix/src/matrix/monitor/index.ts:142`
  - body: Replacing the awaited `releaseSharedClientInstance(client, "persist")` path with `stopSharedClientInstance(client)` skips `stopAndPersist`, so a normal restart can lose the restart-safe sync marker and skip backlog.
  - confidence: 0.86
