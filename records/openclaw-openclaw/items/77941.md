## Summary

Keep open: current main partially overlaps by pruning aged unreferenced session artifacts and by offering an interactive doctor archive prompt, but it still lacks the requested audited headless orphan/unindexed transcript archive mode with exact per-file metadata, sha256, quarantine manifest, and explicit archive/prune semantics.

## What This Changes

Add a native sessions cleanup mode that enumerates orphan or unindexed transcript artifacts across agent stores, reports per-file metadata, and safely archives or prunes them without editing transcript contents.

## Best Possible Solution

Define the session lifecycle policy first, then add a bounded core-owned cleanup mode that reports exact candidate files and supports a safe archive-with-manifest path before any destructive prune path.

## Reproduction Assessment

Yes for the missing capability: current source shows `sessions cleanup` has no orphan archive/prune flags or per-file metadata output, while doctor’s orphan archive path is interactive-only. I did not run a live filesystem repro because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the policy: the requested native command is plausible and aligned with the core session-store boundary, but archive-vs-prune semantics and historical transcript safety need an explicit decision first.
