## Summary

Keep open: current main still lacks the requested audited `sessions cleanup` orphan-transcript archive/prune mode, and the related doctor `--yes` thread was explicitly closed/withdrawn in favor of this canonical design work.

## What This Changes

Request native `openclaw sessions cleanup` support to enumerate unindexed transcript files and archive or prune them with dry-run audit metadata.

## Best Possible Solution

Define an explicit orphan transcript maintenance submode under `sessions cleanup` with store discovery/Gateway routing, per-file audit metadata, and archive/prune behavior gated by deliberate operator confirmation.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs give a high-confidence verification path showing the requested flags and file-level audit/archive workflow are absent on current main.

## Solution Assessment

Unclear until maintainers choose the UX boundary. Extending `sessions cleanup` is plausible, but it should be reconciled with doctor’s interactive safety policy and the current retention-based cleanup contract.
