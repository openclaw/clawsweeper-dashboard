## Summary

Keep open: current main has partial session cleanup and doctor paths, but it still lacks the requested audited, headless `sessions cleanup` archive/prune mode with per-file metadata and safe transcript-lifecycle policy.

## What This Changes

The issue requests a native `openclaw sessions cleanup` mode to identify, dry-run, archive, or prune orphan/unindexed transcript files across agent session stores.

## Best Possible Solution

Define the transcript lifecycle policy first, then implement a bounded core-owned cleanup mode with exact candidate reporting and safe archive-with-manifest semantics before any destructive prune path.

## Reproduction Assessment

Yes. Current source shows no `sessions cleanup` archive-orphans/prune-unindexed option, no Gateway schema field for that mode, and only aggregate unreferenced-artifact reporting.

## Solution Assessment

Unclear. A native cleanup mode is likely the right core boundary, but maintainers need to choose archive versus prune semantics and how to protect legitimate historical transcripts.
