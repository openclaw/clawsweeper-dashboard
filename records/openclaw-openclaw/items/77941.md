## Summary

Keep open: current main has adjacent cleanup and doctor support for unreferenced transcripts, but it does not expose the requested audited sessions cleanup mode with per-file paths, hashes, archive manifest, or orphan-specific archive/prune controls.

## What This Changes

Request adds a native sessions cleanup mode to identify, dry-run, archive, or prune transcript files not indexed by `sessions.json`.

## Best Possible Solution

Define an explicit orphan transcript maintenance submode under `sessions cleanup` that shares store discovery/Gateway routing, reports per-file audit metadata, and archives or prunes only under intentional operator confirmation.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms the current CLI lacks orphan-specific archive/prune flags while existing cleanup only exposes retention-based unreferenced artifact pruning and aggregate reporting.

## Solution Assessment

Unclear until maintainers choose the UX boundary. Extending `sessions cleanup` is plausible, but it should be reconciled with the existing `doctor` interactive archive flow and the current retention-based cleanup policy.
