## Summary

Keep open: current main has partial cleanup paths, but it still lacks the requested audited, headless `sessions cleanup` mode for orphan/unindexed transcript archive or prune with per-file metadata, hashes, and archive manifest semantics. The remaining decision is policy-sensitive because adjacent reports show unindexed historical transcripts can also be user history.

## What This Changes

Request to add a native `openclaw sessions cleanup` mode that identifies transcript files not indexed by `sessions.json` and safely archives or prunes them with dry-run audit metadata.

## Best Possible Solution

Define the transcript lifecycle policy first, then add a bounded core-owned cleanup mode that reports exact candidate files and supports safe archive-with-manifest semantics before any destructive prune path.

## Reproduction Assessment

Yes. Current source shows no `sessions cleanup` archive-orphans/prune-unindexed option, no Gateway schema field for that mode, and no per-file audit output beyond aggregate unreferenced-artifact counts.

## Solution Assessment

Unclear until maintainers choose the policy. The requested native command is plausible and fits the core session-store boundary, but archive versus prune semantics and historical transcript safety need an explicit decision first.
