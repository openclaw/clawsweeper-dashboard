## Summary

Keep open: current main still lacks the requested audited orphan-transcript archive/prune mode, and related discussion treats this as the canonical design thread rather than a resolved doctor bug.

## What This Changes

Request adds a native `openclaw sessions cleanup` mode to audit, archive, or prune transcript files that are no longer indexed by `sessions.json`.

## Best Possible Solution

Define an explicit `sessions cleanup` orphan-transcript submode with store discovery, per-file audit metadata, manifest-backed archive/quarantine, and deliberate confirmation for destructive prune.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, tests, and live related-thread context give a high-confidence verification path that the requested audited archive/prune mode is absent on current main.

## Solution Assessment

Unclear until maintainers choose the UX and safety boundary. Extending `sessions cleanup` is plausible, but the current aggregate prune path and doctor’s interactive archive path are not the requested audited workflow.
