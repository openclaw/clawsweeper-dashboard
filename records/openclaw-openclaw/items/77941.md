## Summary

Keep open: current main and v2026.6.6 have a partial aged-prune path for old unreferenced session artifacts, but they still lack the requested headless audited archive/prune mode with per-file metadata, sha256, quarantine/manifest semantics, and the doctor archive path remains interactive-only.

## What This Changes

Request a native `openclaw sessions cleanup` mode to enumerate, archive, or prune orphan/unindexed session transcript files and sidecars safely.

## Best Possible Solution

Keep this open for a maintainer decision: either document the existing aged deletion path as sufficient or add one explicit audited orphan-transcript mode with per-file dry-run metadata, safe archive/prune apply semantics, and manifest evidence.

## Reproduction Assessment

Yes, for the missing requested mode: source inspection shows no `sessions cleanup` archive/prune-unindexed flag, only aggregate aged artifact pruning, and doctor orphan archiving remains interactive-only. I did not run a live production-style orphan cleanup scenario in this read-only review.

## Solution Assessment

Unclear: the existing generic prune path is a useful partial solution, but it is not the audited archive/manifest workflow requested here. The best maintainable solution needs maintainer choice on explicit archive semantics versus documenting or tightening current prune behavior.
