## Summary

Keep open: current main partially overlaps by pruning old unreferenced session artifacts, but it does not provide the requested explicit orphan-transcript archive/quarantine mode, per-file dry-run metadata, sha256 reporting, or manifest. The doctor path still keeps archive remediation interactive-only, and recent production comments show the headless maintenance gap remains meaningful.

## What This Changes

Request a native `openclaw sessions cleanup` mode that enumerates orphan/unindexed transcript files and safely archives or prunes them with audited dry-run/apply output.

## Best Possible Solution

Make a maintainer decision between documenting the existing aged prune path as sufficient or adding one explicit audited orphan-transcript mode with per-file dry-run metadata and safe archive/prune apply semantics.

## Reproduction Assessment

Yes, from source inspection rather than a live run: current main lacks a dedicated sessions cleanup orphan-transcript archive/prune option and only reports aggregate unreferenced-artifact counts. The doctor archive path is still intentionally interactive-only for headless repair.

## Solution Assessment
