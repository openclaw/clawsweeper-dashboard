## Summary

Keep open: current main has adjacent retention-based unreferenced artifact pruning and doctor’s interactive orphan archive prompt, but it still does not provide the requested audited sessions cleanup mode with per-file paths, hashes, archive manifest, or orphan-specific archive/prune controls.

## What This Changes

The issue requests a native sessions cleanup mode that can identify, audit, archive, or prune transcript files that are present on disk but not indexed by sessions.json.

## Best Possible Solution

Define an explicit sessions cleanup orphan-transcript mode with store discovery, per-file audit metadata including sha256, quarantine-manifest archive support, explicit prune confirmation, docs, and tests around true-orphan classification.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence negative verification that current main lacks the requested orphan-specific cleanup flags and only has aggregate retention-based pruning plus doctor’s interactive archive path.

## Solution Assessment

Unclear until maintainers choose the UX and data-safety boundary. A dedicated sessions cleanup submode is plausible, but it should be reconciled with doctor’s intentionally interactive behavior, session retention semantics, and valid historical transcript preservation.
