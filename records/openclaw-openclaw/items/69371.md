## Summary

Keep open. Current main has adjacent Pi, status, compaction, cron, and delivery work, but the requested ARM64 edge-device stability bundle remains unresolved and is too broad for conservative cleanup.

## What This Changes

Request to improve OpenClaw stability on ARM64 edge devices by reducing sessions/status overhead, adding compaction high-water triggers, retrying cron/message delivery, adding sessions_send receipts, and adding edge-device diagnostics.

## Best Possible Solution

Split this into measured, owner-scoped work: first profile and optimize large-session sessions/status paths, then separately define compaction high-water behavior, cron retry policy, message retry/queueing, sessions_send receipt semantics, and edge-device diagnostics.

## Reproduction Assessment

No high-confidence live reproduction was established because this review did not exercise a Pi/Rock5 host with the reported 137-session store. Source inspection does confirm that several requested flags, config keys, and delivery/receipt contracts remain absent on current main.

## Solution Assessment

Not applicable as a patch. The maintainable direction is to split the broad enhancement into profiled subsystem changes rather than one bundled repair PR.
