## Summary

Keep open: the goal is valid and the linked issue still needs a fix, but this patch can call channel startup before the full channel registry is pinned and then prevents the later sidecar path from starting channels normally.

## What This Changes

This PR moves gateway channel startup to the top of `startGatewayPostAttachRuntime` and changes the later sidecar phase to await that early startup promise.

## Best Possible Solution

Replace this with a dependency-aware startup design that pins the channel registry before channel start, preserves ACPX readiness semantics, and proves a real configured channel connects before `plugins.runtime-post-bind` completes.
