## Summary

Keep open: the goal is valid, but the branch is not merge-ready because the early channel-start promise can run before channel registry pinning and then suppress the normal sidecar start path.

## What This Changes

This PR starts channels at the beginning of `startGatewayPostAttachRuntime()` and passes a wrapper to `startGatewaySidecars()` so the later sidecar phase waits for that early promise.

## Best Possible Solution

Revise the gateway/plugin startup seam so channel registry state is available before any early channel start, keep a normal fallback until configured channels are proven to start exactly once, and verify live channel connection.
