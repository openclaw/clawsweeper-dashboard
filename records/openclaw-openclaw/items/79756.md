## Summary

Keep open: the goal is valid and current main still has the startup coupling, but this PR is not merge-ready because it starts channels before channel registry pinning and can turn channel startup into a no-op.

## What This Changes

The branch starts `params.startChannels()` early in `startGatewayPostAttachRuntime()` and later has `startGatewaySidecars()` await that promise instead of invoking the normal channel-start path.

## Best Possible Solution

Replace this reorder with dependency-aware startup that guarantees a populated channel registry before channel startup and proves a real configured channel connects before slow ACPX/runtime plugin work completes.
