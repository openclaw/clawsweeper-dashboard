## Summary

Keep open: the PR targets a real gateway startup problem, but the patch is not ready because it adds an awaited ACP backend gate on the sidecar completion path and lacks after-fix real behavior proof.

## What This Changes

Moves channel startup in `startGatewaySidecars()` ahead of Gmail/internal-hook startup and introduces an ACP backend readiness promise before memory/post-ready sidecars and ACP identity reconciliation.

## Best Possible Solution

Keep channel startup independent of Gmail/hooks while keeping ACP readiness waits scoped to ACP-only background consumers; handle the broader heartbeat/cron decoupling in a dependency-driven startup design for https://github.com/openclaw/openclaw/issues/79625.

## Reproduction Assessment

Yes for the review finding: source inspection shows the new ACP readiness promise is awaited before `startGatewaySidecars()` can resolve, and gateway readiness plus scheduled services depend on that resolution. I did not run a live ACPX gateway because this review is read-only.

## Solution Assessment

No: creating `channelsPromise` earlier is the right direction, but awaiting `acpBackendPromise` before sidecar completion reintroduces an ACP gate for unrelated readiness and scheduled services. ACP readiness should stay inside identity reconciliation or another ACP-only background task.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep ACP readiness off the sidecar completion path:** `src/gateway/server-startup-post-attach.ts:524`
  - body: This await makes `startGatewaySidecars()` wait for `waitForAcpRuntimeBackendReady()` whenever ACP is enabled. `startGatewayPostAttachRuntime()` only marks sidecars ready and logs `gateway ready` after `startGatewaySidecars()` resolves, and heartbeat/cron activation is gated on `startupSidecarsReady`, so a slow or unhealthy ACP backend now delays ACP-independent sidecars again. Leave the ACP wait inside the identity-reconcile task or another ACP-only background task.
  - confidence: 0.9
