## Summary

Keep this PR open: the user-installed channel metadata gap is source-reproducible, but the patch is not mergeable because it lacks real behavior proof and currently breaks scoped setup discovery plus existing tests.

## What This Changes

The PR adds `listAllChatChannels()` and switches configure removal/setup discovery to include metadata for installed channel plugins from non-bundled origins.

## Best Possible Solution

Implement installed channel metadata through the existing scoped/resolved channel catalog helpers, then add focused tests and real `openclaw configure` proof with private details redacted.

## Reproduction Assessment

Yes, source-reproducible: current main labels configure removal choices with `listChatChannels()` only, so a configured non-bundled channel falls back to its raw id. I did not run the interactive command because this review is read-only.

## Solution Assessment

No, not as written: the direction is reasonable, but setup discovery should reuse the scoped/resolved catalog entries instead of a parameterless global scan, and the test mocks need to be updated.
