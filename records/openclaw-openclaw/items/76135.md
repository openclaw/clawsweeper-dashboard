## Summary

Keep open: current main still suppresses verbose/tool progress for ordinary group sessions, and the requested trusted private-room behavior needs a new, explicit, privacy-sensitive config policy rather than cleanup closure.

## What This Changes

Request an opt-in policy that lets private messaging group sessions surface verbose/tool progress while preserving safe default suppression for ordinary groups.

## Best Possible Solution

Add a provider-agnostic, fail-closed group verbose-progress policy with typed global/per-channel/per-room config, docs privacy warnings, and regression coverage for ordinary groups, trusted private rooms, forum topics, and Slack non-direct suppression.

## Reproduction Assessment

Yes, at source level. Signal groups become ChatType group, dispatch gates verbose/tool progress off for ordinary groups, and existing tests assert text tool summaries stay suppressed in group sessions.

## Solution Assessment

Unclear pending maintainer product review. Explicit opt-in configuration is the right direction, but maintainers need to choose naming, precedence, per-room semantics, and how it relates to https://github.com/openclaw/openclaw/pull/44027 and https://github.com/openclaw/openclaw/pull/64182.
