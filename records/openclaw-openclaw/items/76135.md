## Summary

Keep open. Current main still suppresses verbose/tool progress for ordinary group chats even when group final replies are automatic and verbose is enabled; the requested opt-in is valid, but it changes privacy-sensitive group-visible output and overlaps open PRs that need maintainer policy direction.

## What This Changes

Allow private group/channel messaging sessions to opt into visible verbose/tool progress while preserving safe default suppression for ordinary groups.

## Best Possible Solution

Land a maintainer-approved provider-agnostic verbose-progress policy resolver with strict schema, tests, docs, and changelog coverage, preserving default suppression for ordinary groups and coordinating or superseding #44027/#64182.

## Reproduction Assessment

Yes. A focused dispatch check with `ChatType: "group"`, `messages.groupChat.visibleReplies: "automatic"`, verbose enabled, and a text `onToolResult` payload exercises the current suppression path; existing tests already assert the ordinary-group, forum, and direct-chat split.

## Solution Assessment

Unclear. The requested explicit opt-in is safer than enabling verbose progress for every group, but maintainers should choose the final policy shape before code changes because this deliberately broadens sensitive visible output.
