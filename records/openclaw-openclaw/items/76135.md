## Summary

Keep open. Current main still suppresses verbose/tool progress for ordinary group sessions even when group final replies can be automatic and verbose mode is active; the remaining change is privacy-sensitive and overlaps open PRs #44027 and #64182.

## What This Changes

Add an explicit opt-in policy/config path so private messaging group or channel sessions can surface verbose/tool progress while preserving safe default suppression for ordinary groups.

## Best Possible Solution

Add a provider-agnostic verbose-progress policy resolver at the shared dispatch gate, with strict config schema/types/docs/tests/changelog coverage, default-off ordinary groups, explicit trusted-room opt-in, and preserved Slack non-direct safety unless maintainers deliberately change it.

## Reproduction Assessment

Yes. A focused dispatch path with `ChatType: "group"`, `messages.groupChat.visibleReplies: "automatic"`, verbose enabled, and a text `onToolResult` payload reproduces the drop; existing tests already assert ordinary group suppression while forum topics deliver.

## Solution Assessment

Unclear. An explicit opt-in config policy is safer than making `/verbose` alone deliver in all groups, but maintainers should choose the supported semantics and reconcile the two open PRs first.
