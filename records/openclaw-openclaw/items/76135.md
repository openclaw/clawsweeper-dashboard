## Summary

Keep open: current main still suppresses verbose/tool progress for ordinary group sessions, and adding trusted-room delivery requires a privacy-sensitive policy/config decision rather than a narrow bug cleanup.

## What This Changes

The issue requests an opt-in policy that lets private messaging group sessions surface verbose/tool progress while keeping ordinary group defaults conservative.

## Best Possible Solution

Add a provider-agnostic verbose-progress policy resolver at the shared dispatch gate, backed by strict global/per-channel/per-group config, docs warnings, regression tests, and default-off behavior for ordinary groups.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Signal group turns set `ChatType: "group"`, verbose mode can be active, but dispatch still requires non-group or forum context before sending verbose/tool progress; existing tests assert ordinary group suppression.

## Solution Assessment

Unclear. An explicit opt-in policy is the right direction, but maintainers need to choose the supported config semantics and reconcile the narrower open PRs before implementation.
