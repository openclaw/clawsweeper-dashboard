## Summary

Keep open. Current main still hard-suppresses verbose/tool progress for ordinary group chats, even when final group replies are configured as automatic and `/verbose` is enabled; the requested opt-in policy is real but needs maintainer review because it deliberately changes privacy-sensitive group-visible output.

## What This Changes

Request an explicit opt-in config policy that lets private messaging group/channel sessions receive verbose tool progress while preserving the current safe default for ordinary shared rooms.

## Best Possible Solution

Add a provider-agnostic verbose-progress policy resolver at the shared dispatch gate, with strict schema/docs/changelog coverage, preserving default group suppression and Slack non-direct safety unless an explicit trusted-room opt-in is approved.

## Reproduction Assessment

Yes. A focused dispatch check with `ChatType: "group"`, `messages.groupChat.visibleReplies: "automatic"`, verbose enabled, and a text `onToolResult` payload exercises the current gate; existing tests already assert ordinary group tool summaries are suppressed while direct/forum paths deliver them.

## Solution Assessment

The requested explicit opt-in direction is safer than enabling all verbose group sessions, but the final shape is not settled; maintainer review should decide whether the knob is global, channel-level, per-group, `whenVerbose`, or command-only.
