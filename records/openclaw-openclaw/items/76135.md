## Summary

Keep open: current main still suppresses verbose/tool progress for ordinary group sessions, and enabling private-room delivery needs an explicit privacy-safe policy/config decision rather than conservative cleanup.

## What This Changes

Request an opt-in way for private messaging group sessions to receive verbose/tool progress while keeping ordinary group defaults safe.

## Best Possible Solution

Add a provider-agnostic, default-off verbose-progress policy resolver with strict global/per-channel/per-group config, docs privacy warnings, and regression coverage for direct, forum, ordinary group, and Signal private-room cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Signal group turns set ChatType to group, /verbose can be active for the session, but dispatch still requires non-group or forum context before sending verbose/tool progress.

## Solution Assessment

Unclear. An explicit opt-in policy is the right direction, but maintainers need to choose the supported config semantics and reconcile the narrower open PRs before implementation.
