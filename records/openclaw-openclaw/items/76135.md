## Summary

Keep open. Current main still suppresses verbose/tool progress for ordinary group sessions even when group final replies are automatic and verbose mode is enabled; enabling this safely requires maintainer policy review because it changes privacy-sensitive group-visible output and overlaps open PRs #44027 and #64182.

## What This Changes

Allow private messaging group sessions to opt in to visible verbose/tool progress while preserving safe defaults for ordinary group rooms.

## Best Possible Solution

Add a provider-agnostic verbose-progress policy resolver at the shared dispatch gate, with strict schema/types/docs/tests/changelog coverage, default-off group behavior, and explicit trusted-room opt-in while preserving Slack non-direct safety unless maintainers choose otherwise.

## Reproduction Assessment

Yes. A focused dispatch path with ChatType "group", messages.groupChat.visibleReplies "automatic", verbose enabled, and a text onToolResult payload reproduces the drop; existing tests already assert ordinary group suppression while direct and forum paths deliver.

## Solution Assessment

Unclear. An explicit opt-in policy is safer than making /verbose alone deliver in all groups, but maintainers should choose the supported config shape and reconcile it with the existing open PRs before implementation.
