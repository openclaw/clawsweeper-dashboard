## Summary

Keep open: current main still suppresses verbose/tool progress in ordinary group sessions, and the requested trusted-room behavior needs an explicit privacy-sensitive config policy rather than conservative cleanup.

## What This Changes

Requests an opt-in policy/config path so private group or channel sessions can surface verbose/tool progress while preserving safe defaults for ordinary rooms.

## Best Possible Solution

Add a provider-agnostic group verbose-progress policy in shared dispatch, backed by strict global/per-channel/per-room config, docs warnings, and regression coverage while keeping ordinary groups default-off.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Signal groups set ChatType to group, dispatch only allows verbose/tool progress for non-groups or forum groups, and the existing dispatch test asserts ordinary group text tool summaries are suppressed.

## Solution Assessment

Unclear. An explicit opt-in policy is the right direction, but maintainers need to choose the supported global, per-channel, and per-room semantics and reconcile #44027 and #64182 before implementation.
