## Summary

Keep open. Current main still suppresses verbose/tool progress for ordinary group sessions, and the remaining fix needs an explicit privacy-safe policy decision rather than a cleanup close.

## What This Changes

The issue requests an opt-in way for private messaging group sessions to surface verbose/tool progress while preserving the safe default suppression for ordinary groups.

## Best Possible Solution

Add a provider-agnostic verbose-progress policy resolver with default-off ordinary group behavior, explicit trusted-room opt-in, strict schema/types/docs/tests/changelog coverage, and preserved Slack non-direct safety unless maintainers choose otherwise.

## Reproduction Assessment

Yes. A focused dispatch path with `ChatType: "group"`, `messages.groupChat.visibleReplies: "automatic"`, verbose enabled, and a text `onToolResult` payload reproduces the drop because the shared gate makes `shouldSendToolSummaries` false; existing tests already assert that ordinary group text summaries are suppressed.

## Solution Assessment

Unclear. An explicit opt-in policy is the right direction, but maintainers should choose the supported global/per-channel/per-room config shape and reconcile #44027 and #64182 before implementation.
