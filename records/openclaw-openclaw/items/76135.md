## Summary

Keep open: current main still suppresses verbose/tool progress in ordinary group sessions, and the requested trusted-room behavior needs an explicit privacy-sensitive policy/config decision rather than cleanup closure.

## What This Changes

Request an opt-in policy so trusted messaging group sessions can show verbose/tool progress while ordinary groups remain default-suppressed.

## Best Possible Solution

Add a provider-agnostic, explicit opt-in group verbose-progress policy in shared dispatch, backed by strict global/per-channel/per-room config, docs privacy warnings, and regression coverage while keeping ordinary groups default-off.

## Reproduction Assessment

Yes. Current source gives a high-confidence path: Signal groups set ChatType to group, shared dispatch only permits verbose/tool progress for non-groups or forum groups, and the existing dispatch test asserts group tool text is suppressed.

## Solution Assessment

Unclear. An explicit opt-in policy is the right direction, but maintainers need to settle the privacy/config semantics and reconcile #44027 and #64182 before implementation.
