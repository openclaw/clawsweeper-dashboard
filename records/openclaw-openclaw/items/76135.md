## Summary

Keep open: current main still suppresses ordinary group verbose/tool progress, and solving the private-room use case requires a new privacy-sensitive config/policy rather than conservative cleanup.

## What This Changes

The issue requests an opt-in configuration path for ordinary messaging group sessions to receive verbose/tool progress while preserving safe default suppression.

## Best Possible Solution

Add a provider-agnostic verbose-progress policy resolver at the shared dispatch gate, backed by strict global/per-channel/per-group config, docs warnings, and regression coverage for direct, forum, ordinary group, and Signal group sessions.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Signal group turns set `ChatType: "group"`, `/verbose` state can be active, but dispatch still requires non-group or forum context before sending verbose/tool progress.

## Solution Assessment

Unclear. An explicit opt-in policy is the right direction, but maintainers need to choose the supported config semantics and reconcile the narrower open PRs before implementation.
