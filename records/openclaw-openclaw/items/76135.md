## Summary

Keep open: current main still suppresses verbose/tool progress for ordinary group sessions, and the requested trusted-room behavior needs an explicit privacy-aware config policy rather than a narrow bug cleanup.

## What This Changes

Request to add opt-in verbose/tool progress delivery for private project-room group/channel messaging sessions while preserving safe default suppression.

## Best Possible Solution

Add a provider-agnostic verbose-progress policy resolver at the shared dispatch gate, backed by strict global/per-channel/per-group config, default-off ordinary group behavior, docs warnings, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Signal group turns set `ChatType: "group"`, verbose mode can be active, but dispatch still requires non-group or forum context before sending verbose/tool progress.

## Solution Assessment

Unclear. An explicit opt-in policy is the right direction, but maintainers need to choose the supported config semantics and reconcile the related open PRs before implementation.
