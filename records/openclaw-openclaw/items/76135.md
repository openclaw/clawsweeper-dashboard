## Summary

Keep open. Current main still suppresses verbose/tool progress for ordinary group chats even when final replies are configured as automatic, and the requested opt-in is real but privacy-sensitive and overlaps existing open PRs that need maintainer policy direction.

## What This Changes

Request an explicit opt-in policy so private messaging group/channel sessions can surface verbose/tool progress while keeping ordinary group defaults safe.

## Best Possible Solution

Add a provider-agnostic verbose-progress policy resolver at the shared dispatch gate, with strict schema/docs/changelog coverage, preserving default group suppression and Slack non-direct safety unless an explicit trusted-room opt-in is approved.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: use a `ChatType: "group"` turn with `messages.groupChat.visibleReplies: "automatic"`, verbose enabled, and a text `onToolResult` payload; current main's shared gate drops the text while existing tests prove direct/forum exceptions still deliver.

## Solution Assessment

Unclear. The requested opt-in direction is safer than enabling all verbose group sessions, but the exact maintainable shape needs maintainer product/privacy review and should be reconciled with the existing narrower PRs.
