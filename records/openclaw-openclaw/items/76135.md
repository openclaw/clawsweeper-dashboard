## Summary

Current main and the latest stable release implement the central group-room verbose/progress behavior through the existing session `/verbose` state, with focused dispatch tests and user-facing docs showing that group sessions can opt in while staying quiet by default.

## What This Changes

Requested support for private group/channel sessions to show verbose tool/progress output when the operator opts in while preserving safe defaults for shared rooms.

## Best Possible Solution

Keep the shipped session-scoped `/verbose` behavior and docs/security warning rather than adding a second group verbose-progress config surface unless a new narrower need appears.

## Reproduction Assessment

No current-main failure reproduction remains: source inspection and regression tests show group sessions now deliver tool summaries when verbose is enabled and suppress them when verbose is off. Historical source and merged PR evidence explain the old suppression path.

## Solution Assessment

Yes. Reusing the existing per-session `/verbose` state is the narrower maintainable fix for durable private group sessions and avoids expanding OpenClaw's already large config surface.
