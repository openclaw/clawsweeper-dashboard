## Summary

Keep open. Current main still suppresses verbose/tool progress for ordinary group sessions even when group final replies are automatic and verbose mode is enabled, and the missing opt-in policy is privacy-sensitive enough to need maintainer product review rather than cleanup closure.

## What This Changes

Allow private messaging group/project-room sessions to opt in to visible verbose/tool progress while preserving safe default suppression in ordinary group rooms.

## Best Possible Solution

Add a provider-agnostic verbose-progress policy resolver at the shared dispatch gate, with strict schema/types/docs/tests/changelog coverage, default-off group behavior, explicit trusted-room opt-in, and preserved Slack non-direct safety.

## Reproduction Assessment

Yes. A focused dispatch path with ChatType "group", messages.groupChat.visibleReplies "automatic", verbose enabled, and a text onToolResult payload reproduces the drop; existing tests already assert ordinary group suppression while direct/forum paths deliver.

## Solution Assessment

Unclear. An explicit opt-in policy is safer than making /verbose alone deliver in all groups, but maintainers should choose the supported config shape and reconcile it with the existing open PRs before implementation.
