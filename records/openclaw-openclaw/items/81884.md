## What This Changes

The PR constrains manifest `providerCatalogEntry` and `providerDiscoveryEntry` resolution so only relative entries and `.ts` to `.js` fallbacks that stay inside the plugin root populate `providerDiscoverySource`, with registry regression tests.

## Best Possible Solution

Land a narrow registry-boundary fix that rejects out-of-root provider catalog/discovery metadata before runtime loading, after maintainer review and real behavior proof from a plugin registry/startup path.
