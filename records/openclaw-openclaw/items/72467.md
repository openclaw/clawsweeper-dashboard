## Summary

Keep open: the underlying Discord security-audit bug is valid, but this PR still reads the merged Discord config directly instead of the effective account allowFrom resolver, and the external contributor proof gate is still missing.

## What This Changes

The branch adds top-level Discord allowFrom to the security-audit owner allowlist check, a regression test, and a changelog entry.

## Best Possible Solution

Compute the audit owner allowlist with resolveDiscordAccountAllowFrom({ cfg, accountId }) plus credential-store entries, with regression coverage for top-level allowFrom and named-account precedence.

## Reproduction Assessment

Yes, at source level. On current main, native Discord commands with a configured guild, no guild/channel users allowlist, no credential-store or legacy allowlist, and only top-level channels.discord.allowFrom leave ownerAllowFromConfigured false and reach the no_allowlists warning.

## Solution Assessment

No. The PR is aimed at the right bug, but the narrow maintainable fix should reuse the existing effective Discord allowFrom resolver instead of adding another direct merged-config read.

## Review Findings

Overall correctness: patch is incorrect
