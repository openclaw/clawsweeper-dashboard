## Summary

Keep open for maintainer architecture review: current main and v2026.6.8 are still intentionally SQLite-first, and this request is a broad product direction change rather than a narrow bug or already-owned implementation task.

## What This Changes

The issue requests a future backend-neutral storage architecture so OpenClaw can support both embedded SQLite and a Postgres-class managed-service database.

## Best Possible Solution

Keep this as a maintainer architecture/design item only if OpenClaw wants a managed-service database direction; first produce an RFC that reconciles AGENTS.md storage policy with the SQLite migration stack, external session-storage tracker, backend selection, migrations, SDK transition, and backup/restore semantics.

## Reproduction Assessment

Not applicable; this is an architecture feature request rather than a bug. Source and release inspection show current main and v2026.6.8 still use SQLite-specific state paths, so there is no failing current-main repro path to run.

## Solution Assessment

No, not as an immediate implementation task. The maintainable path is a maintainer-approved design/RFC that decides whether OpenClaw should change its SQLite-only storage policy before any code PR tries to add a Postgres-class backend.
