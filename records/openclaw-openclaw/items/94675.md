Define a backend-neutral database abstraction so OpenClaw can keep SQLite first-class while supporting an async Postgres-class managed-service backend.

## Best Possible Solution

Keep this as a maintainer-owned design/RFC tracker, then split implementation into staged backend seams, async transaction/storage contracts, migration/doctor policy, and backend-specific proof once product direction is approved.
