## What This Changes

The issue reports that Control UI startup RPCs such as `commands.list`, `models.list`, `sessions.list`, `chat.history`, and `models.authStatus` take 5-10 seconds after upgrade and can delay webchat sends into timeout behavior.

## Best Possible Solution

Profile a current-main gateway restart plus Control UI reload with RPC timing and startup diagnostics, then make the proven slow gateway work bounded, cached, coalesced, or deferred so chat history and sends remain responsive during metadata refreshes.
