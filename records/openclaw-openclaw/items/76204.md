## Summary

Close as implemented on current main. The embedded-runner boundary now converts runtime-only empty transcript prompts into a non-empty marker, skips truly blank visible prompt submissions before provider calls, and repairs persisted blank user messages before replay; no release tag contains the proof commit yet.

## What This Changes

The issue asks OpenClaw to prevent empty gateway/system-wake user content from reaching model providers and poisoning auth-profile cooldowns.

## Best Possible Solution

Keep the current gateway and embedded-runner boundary guards and ship them in the next release; reopen only if a new caller can still submit an empty provider message array.

## Reproduction Assessment

Yes. The source-level reproduction is an empty or blank visible prompt reaching the embedded run; current tests cover both the runtime-only empty transcript case and the blank prompt skip before provider submission.

## Solution Assessment

Yes. The current approach is narrower than provider-specific cooldown handling because it prevents malformed empty turns before provider submission while still preserving runtime context and repairing persisted bad session history.
