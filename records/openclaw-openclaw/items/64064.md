## Summary

Keep open. Current main still lacks Anthropic advisor support, and this PR is the active implementation path, but the branch still has blocking Anthropic-contract issues around model pairing and raw server-side block replay.

## What This Changes

The PR adds Anthropic advisor beta tool config/injection, server-side Anthropic tool-result display handling, stop-reason hardening, docs, live validation scripts, and tests.

## Best Possible Solution

Implement advisor support with pair-aware config/defaults, raw server-side content preservation through replay and pruning, and docs/tests aligned to Anthropic's beta contract.

## Reproduction Assessment

Yes. Source inspection reproduces the blockers: params.advisor: true selects an incompatible Sonnet advisor for the documented Sonnet executor, and the replay path drops or downgrades raw server-side advisor blocks instead of preserving them.

## Solution Assessment

No. The feature direction is useful, but the current PR is not the best mergeable shape because fixed default pairing and display-only server-tool state do not match the Anthropic advisor contract; a pair-aware config plus raw replay/pruning path is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Use a compatible advisor default:** `extensions/anthropic/stream-wrappers.ts:241`
  - body: `params.advisor: true` returns `claude-sonnet-4-6` while the docs advertise that shorthand under a `claude-sonnet-4-6` executor. Anthropic's advisor docs require that executor to pair with `claude-opus-4-7`, and invalid pairs return 400, so the documented common config and validation script can fail before stream handling runs.
  - confidence: 0.93
- **[P2] Preserve pending server tool uses:** `src/agents/anthropic-transport-stream.ts:822-824`
  - body: Anthropic's docs say a dangling advisor call can end with `pause_turn` and a `server_tool_use` block, with the advisor executing on resumption. This branch drops every `server_tool_use`, so a paused advisor call has no raw block to replay on the next request.
  - confidence: 0.86
- **[P2] Round-trip advisor result blocks:** `src/agents/anthropic-transport-stream.ts:283-290`
  - body: The replay path turns `advisor_tool_result` and redacted `encrypted_content` into `serverToolDisplay` text, then skips or falls back to plain text on later Anthropic requests. The SDK exposes these blocks as message content params and marks redacted advisor content as verbatim round-trip state, so text-only replay loses server-side tool context.
  - confidence: 0.9
