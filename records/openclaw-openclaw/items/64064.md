## Summary

Keep open. Current main still lacks Anthropic advisor support, and this PR is the active implementation path for the open tracker, but the patch still has API-contract issues around default model pairing and raw advisor result replay that should block merge until a maintainer design/rebase pass happens.

## What This Changes

The PR adds Anthropic advisor beta header/tool injection, server-side tool-result display handling, stop-reason hardening, docs, live validation scripts, and focused Anthropic transport/wrapper/chat-content tests.

## Best Possible Solution

Implement a maintainer-reviewed Anthropic advisor feature with pair-aware advisor config, raw server-side result preservation through replay and pruning, and docs that match the supported config contract.

## Reproduction Assessment

Yes. Source inspection reproduces the two blockers: params.advisor: true injects claude-sonnet-4-6 as the advisor for the documented Sonnet 4.6 executor, and convertAnthropicMessages drops raw advisor_tool_result blocks instead of replaying them.

## Solution Assessment

No. The feature direction is useful, but display-only server-tool text and a fixed Sonnet default are not the narrow maintainable solution for Anthropic's current advisor contract; raw block replay plus pair-aware config is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Use a compatible default advisor model:** `extensions/anthropic/stream-wrappers.ts:225`
  - body: `params.advisor: true` injects `claude-sonnet-4-6` as the advisor model while the docs advertise that shorthand for a `claude-sonnet-4-6` executor. Anthropic's advisor docs require that executor to pair with `claude-opus-4-7`, and invalid pairs return 400, so the documented common config and validation script can fail before stream handling runs.
  - confidence: 0.93
- **[P2] Round-trip advisor result blocks:** `src/agents/anthropic-transport-stream.ts:283-290`
  - body: The replay path converts `advisor_tool_result` and redacted `encrypted_content` into `serverToolDisplay` text, then skips or falls back to plain text on later Anthropic requests. Anthropic expects full assistant content, including advisor result blocks, on subsequent turns; store and replay the raw server-side blocks instead of replacing them with display-only text.
  - confidence: 0.9
