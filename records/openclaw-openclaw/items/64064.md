## Summary

Keep open. The feature is useful and has live Anthropic proof, but the current branch is not mergeable as-is because it drops current-main Anthropic thinking-prefill protection, documents/implements an invalid default advisor pairing, and does not preserve raw server-side tool blocks for multi-turn replay.

## What This Changes

The branch adds Anthropic advisor-tool opt-in config, beta header/tool injection, server-side tool result display handling, docs, validation scripts, and transport/wrapper/chat-content tests.

## Best Possible Solution

Land a maintainer-reviewed Anthropic provider implementation rebased on current main that preserves the thinking-prefill wrapper, derives or requires valid executor/advisor pairs, round-trips raw server-side tool blocks through replay and pruning, and keeps docs/tests aligned with Anthropic's beta contract.

## Reproduction Assessment

Yes. Source inspection gives high-confidence reproduction paths: current main lacks advisor injection/server-side block handling, while PR head shows model-pair, replay, and stale-wrapper regressions against current main and the Anthropic docs contract.

## Solution Assessment

No. The direction is useful, but the patch should preserve current-main wrapper behavior and implement pair-aware config plus raw server-side block replay instead of using a Sonnet default and display-only history.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Preserve the thinking prefill wrapper:** `extensions/anthropic/stream-wrappers.ts:319`
  - body: Current main always composes `createAnthropicThinkingPrefillWrapper` so extended-thinking Anthropic requests do not end with an assistant prefill. This branch's wrapper chain ends with the advisor wrapper and omits that current-main guard, so rebasing or merging it would reintroduce rejected thinking-prefill payloads.
  - confidence: 0.94
- **[P2] Use a compatible advisor default:** `extensions/anthropic/stream-wrappers.ts:225-241`
  - body: `advisor: true` resolves to `claude-sonnet-4-6`, and the new docs show that shorthand on a `claude-sonnet-4-6` executor. Anthropic's advisor contract requires compatible executor/advisor pairs with the advisor at least as capable, so the documented common setup can fail before streaming runs.
  - confidence: 0.93
- **[P2] Round-trip raw server-side tool blocks:** `src/agents/anthropic-transport-stream.ts:833-837`
  - body: The stream handler stores advisor/server-tool results as synthetic display text and later skips or downgrades that data on Anthropic replay. Anthropic's contract expects full assistant content, including `advisor_tool_result` and encrypted server-tool result blocks, to be passed back on later turns, so multi-turn advisor/server-tool state is lost.
  - confidence: 0.92
