## Summary

Keep open. Current main still lacks Anthropic advisor support, but the PR head still has API-contract blockers around advisor model compatibility and raw server-side block replay, so this needs maintainer review rather than cleanup closure.

## What This Changes

The PR adds Anthropic advisor beta opt-in/config, injected advisor tool payloads, server-side tool display handling, stop-reason hardening, docs, validation scripts, and tests.

## Best Possible Solution

Land a maintainer-reviewed implementation that derives or requires valid executor/advisor pairs, preserves raw server-side tool blocks through replay and pruning, and keeps docs and tests aligned with Anthropic's beta contract.

## Reproduction Assessment

Yes. Source inspection shows current main lacks advisor config/tool injection and server-side result handling, and the PR-head blockers are visible in the advisor resolver, docs, replay conversion, and Anthropic SDK/docs contract.

## Solution Assessment

No. The direction is useful, but the implementation should be pair-aware and preserve raw server-side blocks instead of replacing them with display-only text history.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Choose a compatible advisor default:** `extensions/anthropic/stream-wrappers.ts:225-241`
  - body: `advisor: true` resolves to `claude-sonnet-4-6`, and the new docs show that shorthand on a `claude-sonnet-4-6` executor. Anthropic documents that executor as pairing with `claude-opus-4-7` and returns a 400 for invalid pairs, so the documented common setup can fail before stream handling runs.
  - confidence: 0.93
- **[P2] Preserve raw server-side tool blocks:** `src/agents/anthropic-transport-stream.ts:833-837`
  - body: The stream handler stores `advisor_tool_result` and encrypted/redacted server-tool results as synthetic display text and later skips or downgrades that data on Anthropic replay. The SDK exposes these blocks as assistant content params, with encrypted advisor content required to be round-tripped verbatim, so text-only history breaks multi-turn advisor/server-tool context.
  - confidence: 0.91
