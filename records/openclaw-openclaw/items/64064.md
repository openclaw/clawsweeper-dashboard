## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Validate advisor model pairs before injecting the tool:** `extensions/anthropic/stream-wrappers.ts:319`
  - body: `resolveAnthropicAdvisor` only checks that the configured advisor name looks like a Claude model, so configs such as the PR's Sonnet executor with a Haiku advisor can still be sent and rejected by Anthropic. The advisor docs require a valid pair where the advisor is Sonnet 4.6 or stronger and at least as capable as the executor, so this needs pair-aware validation and tests before the new config surface is safe.
  - confidence: 0.86
- **[P1] Preserve server-tool blocks for Anthropic replay:** `src/agents/anthropic-transport-stream.ts:419-447`
  - body: The new transport path renders advisor/server-tool results as display text and drops `server_tool_use` markers instead of preserving the raw blocks. Anthropic requires advisor result content to be round-tripped verbatim, and pause-turn resumes require the unchanged assistant message with the pending `server_tool_use`; otherwise follow-up turns can lose encrypted advisor context or fail with a 400.
  - confidence: 0.9
- **[P1] Handle pause_turn as resumable advisor state:** `src/agents/anthropic-transport-stream.ts:450-475`
  - body: Mapping `pause_turn` to a normal stop makes OpenClaw treat a pending server-side advisor call as a completed assistant turn. For advisor flows Anthropic documents `pause_turn` as a resumable state that must resend the unchanged assistant message with the same tool and beta header, so this can silently drop the continuation path.
  - confidence: 0.84
