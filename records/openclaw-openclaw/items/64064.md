## Summary

Keep open. Current main still lacks the requested Anthropic advisor support, and this PR has credible live proof, but it remains merge-conflicting and has blocking API-contract issues around advisor model compatibility and multi-turn server-side block replay.

## What This Changes

The branch adds Anthropic advisor beta config/header/tool injection, server-side tool result display handling, stop-reason hardening, docs, validation scripts, and focused tests.

## Best Possible Solution

Land a maintainer-reviewed advisor implementation that derives or requires valid executor/advisor pairs, preserves raw server-side result blocks through replay and pruning, and keeps docs aligned with the beta contract.

## Reproduction Assessment

Yes. Source inspection shows current main lacks advisor config/tool injection and server-side result handling, and PR-head blockers are visible in the advisor resolver and replay conversion paths.

## Solution Assessment

No. The direction is useful, but the implementation should be pair-aware and preserve raw server-side blocks instead of replacing them with display-only text history.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Choose a valid advisor default:** `extensions/anthropic/stream-wrappers.ts:225`
  - body: `advisor: true` resolves to `claude-sonnet-4-6`, and the new docs show that shorthand on a `claude-sonnet-4-6` executor. Anthropic documents that Sonnet 4.6 must pair with Opus 4.7 as advisor and that invalid pairs return `400 invalid_request_error`, so the documented common setup can fail before stream handling runs. ([platform.claude.com](https://platform.claude.com/docs/it/agents-and-tools/tool-use/advisor-tool))
  - confidence: 0.93
- **[P2] Round-trip server-side tool results:** `src/agents/anthropic-transport-stream.ts:833-837`
  - body: The stream handler stores `advisor_tool_result` and encrypted/redacted server-tool results as synthetic display text and later skips or downgrades that data on Anthropic replay. Anthropic documents these as assistant content that should be included on subsequent turns, including encrypted content decrypted by the server next turn, so replacing them with text-only history breaks multi-turn advisor context. ([platform.claude.com](https://platform.claude.com/docs/it/agents-and-tools/tool-use/advisor-tool))
  - confidence: 0.91
