## Summary

Keep open. Current main still lacks the requested Anthropic advisor support, but this PR has blocking API-contract issues around compatible advisor model selection and raw server-side block replay that should be resolved before merge.

## What This Changes

The PR adds Anthropic advisor config/header/tool injection, display handling for server-side tool results, stop-reason hardening, docs, validation scripts, and focused tests.

## Best Possible Solution

Land a maintainer-reviewed advisor implementation that derives or requires compatible executor/advisor pairs, preserves raw server-side result blocks through replay and pruning, and keeps https://github.com/openclaw/openclaw/issues/63930 as the canonical feature thread until merge.

## Reproduction Assessment

Yes, source-reproducible. On PR head, advisor: true resolves to claude-sonnet-4-6 and advisor_tool_result becomes serverToolDisplay text rather than a raw replayable block; current main lacks both paths.

## Solution Assessment

No. The direction is useful, but the current patch should use a pair-aware advisor model contract and preserve raw server-side blocks instead of relying on display-only replay.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Choose a compatible advisor default:** `extensions/anthropic/stream-wrappers.ts:225-241`
  - body: `advisor: true` resolves to `claude-sonnet-4-6`, and the docs show that shorthand under a `claude-sonnet-4-6` executor. Anthropic requires executor/advisor models to form a valid pair, with Sonnet 4.6 using Opus 4.7, so the documented common config can fail with a 400 before stream handling runs. ([platform.claude.com](https://platform.claude.com/docs/en/agents-and-tools/tool-use/advisor-tool?utm_source=openai))
  - confidence: 0.93
- **[P2] Preserve raw server-side tool blocks:** `src/agents/anthropic-transport-stream.ts:833-837`
  - body: This stores `advisor_tool_result` and encrypted/redacted server-tool results as synthetic display text, then skips or downgrades them on later Anthropic requests. The API exposes these as raw server-side blocks and marks encrypted content as verbatim round-trip data, so preserve and prune the raw blocks instead of replacing them with text-only history. ([platform.claude.com](https://platform.claude.com/docs/en/api/beta/messages?utm_source=openai))
  - confidence: 0.91
