## Summary

Keep open. Current main still lacks the requested Anthropic advisor support, and this active implementation is not merge-ready because it drops current Anthropic wrapper behavior and conflicts with the provider contract for model pairing and multi-turn server-tool replay.

## What This Changes

The branch adds Anthropic advisor config/header/tool injection, server-side tool-result display handling, stop-reason hardening, docs, validation scripts, and focused tests.

## Best Possible Solution

Land a rebased Anthropic provider implementation that keeps existing wrapper behavior, uses pair-aware advisor config, and preserves server-side blocks through replay/pruning with accurate docs and tests.

## Reproduction Assessment

Yes. Source inspection shows current main lacks advisor config/tool injection and generic server-side block handling, and PR-head inspection shows the remaining merge blockers in wrapper composition, model pairing, and replay behavior.

## Solution Assessment

No. The direction is useful, but this patch should be rebased and changed to preserve the existing thinking-prefill guard, validate or choose compatible advisor pairs, and round-trip raw server-side blocks instead of relying on display-only text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Preserve the thinking prefill wrapper:** `extensions/anthropic/stream-wrappers.ts:319`
  - body: This wrapper chain now stops after the optional advisor wrapper and drops the current-main `createAnthropicThinkingPrefillWrapper`. Extended-thinking requests can again be sent with a trailing assistant prefill, which Anthropic rejects; keep the existing thinking-prefill wrapper in the composed chain when adding advisor.
  - confidence: 0.94
- **[P2] Use a compatible advisor default:** `extensions/anthropic/stream-wrappers.ts:225-241`
  - body: `advisor: true` resolves to `claude-sonnet-4-6`, and the new docs demonstrate that shorthand under a Sonnet 4.6 executor. Anthropic documents that Sonnet 4.6 must pair with Opus 4.7 as advisor and invalid pairs return `400 invalid_request_error`, so the documented default can fail for the common case. citeturn2view0.
  - confidence: 0.93
- **[P2] Round-trip raw server-side tool blocks:** `src/agents/anthropic-transport-stream.ts:833-837`
  - body: The stream stores advisor/server-tool results as synthetic display text and skips or downgrades them during replay. Anthropic's multi-turn example appends the full response content, including `advisor_tool_result` blocks, before continuing the conversation; this patch loses those raw blocks and encrypted content, so later turns lose advisor state. citeturn2view0.
  - confidence: 0.92
