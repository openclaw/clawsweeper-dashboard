Current main still lacks the requested Anthropic advisor support, and this PR has useful live proof, but the branch is currently conflicting and still has merge-blocking issues around preserved Anthropic thinking-prefill behavior, advisor model compatibility, and provider-required server-side block replay.

The branch adds Anthropic advisor params/header/tool injection, display handling for Anthropic server-side tool result blocks, stop-reason hardening, docs, live validation scripts, and focused tests.

## Best Possible Solution

Land a maintainer-reviewed Anthropic provider implementation that preserves existing wrapper behavior, validates compatible advisor pairs, round-trips raw server-side advisor blocks through replay/pruning, exposes accurate docs, and keeps the canonical tracker centralized.

## Solution Assessment

No. The feature direction is useful, but this PR is not the best mergeable solution until it preserves the thinking-prefill wrapper, uses or validates compatible advisor pairs, and round-trips raw server-side blocks instead of replacing them with display-only text.
