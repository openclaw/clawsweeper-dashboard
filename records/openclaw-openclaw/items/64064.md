## Summary

Keep open. Current main still lacks Anthropic advisor support, and this PR contains useful implementation work and live validation, but the patch still has blocking model-compatibility and transcript-replay issues that need maintainer design review before merge.

## What This Changes

The PR adds Anthropic advisor beta config/header/tool injection, server-side tool-result display handling, stop-reason hardening, docs, validation scripts, and focused tests.

## Best Possible Solution

Land a maintainer-reviewed advisor implementation that derives or requires compatible advisor models, preserves raw server-side result blocks through replay and pruning, and keeps https://github.com/openclaw/openclaw/issues/63930 as the canonical feature thread until merge.

## Reproduction Assessment

Yes. Source inspection of PR head shows advisor: true resolving to a Sonnet advisor under a Sonnet executor, and shows advisor_tool_result content becoming serverToolDisplay text instead of raw replayable blocks.

## Solution Assessment

No. The feature direction is useful, but the current patch needs a pair-aware advisor model contract and raw server-side block preservation instead of a fixed Sonnet default and display-only replay.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Choose a compatible advisor default:** `extensions/anthropic/stream-wrappers.ts:225-241`
  - body: `advisor: true` defaults the advisor model to `claude-sonnet-4-6`, and the added docs show that shorthand under a `claude-sonnet-4-6` executor. Anthropic requires the executor/advisor models to form a valid pair, with Sonnet 4.6 using Opus 4.7 as advisor, so the documented common config can fail with a 400 before stream handling runs.
  - confidence: 0.93
- **[P2] Round-trip advisor result blocks:** `src/agents/anthropic-transport-stream.ts:833-837`
  - body: This stores `advisor_tool_result` and encrypted/redacted server-tool results as synthetic display text, then skips or downgrades them on later Anthropic requests. The SDK exposes raw advisor result block params and marks redacted encrypted content as verbatim round-trip data, so preserve and prune the raw server-side blocks instead of replacing them with text-only history.
  - confidence: 0.91
