## Summary

Keep open: current main still lacks Anthropic advisor support, and this PR remains useful but not merge-ready because the advertised shorthand injects an invalid advisor pairing and the transport drops raw advisor/server-side result blocks instead of preserving the API replay contract.

## What This Changes

The PR adds Anthropic advisor beta header/tool injection, display handling for server-side tool result blocks, stop-reason hardening, docs, live validation scripts, and focused transport/wrapper/chat-content tests.

## Best Possible Solution

Rework the implementation so advisor config is pair-aware, raw server-side result blocks are stored and replayed through the pruning pipeline, and the docs describe the supported config and replay semantics.

## Reproduction Assessment

Yes. The diff shows params.advisor: true on a Sonnet 4.6 executor injects Sonnet 4.6 as the advisor, which Anthropic documents as an invalid pair, and a synthetic advisor_tool_result streamed by the PR is converted to serverToolDisplay text instead of raw replay content.

## Solution Assessment

No. The feature direction is valid, but display-only server-tool text plus a fixed Sonnet default is not the narrowest maintainable implementation for Anthropic's advisor API; the safer path is pair-aware config plus raw/opaque server-block replay with pruning support.
