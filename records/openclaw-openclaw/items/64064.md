## Summary

Keep open. Current main still lacks this capability, but this PR has blocking advisor compatibility and replay issues: the shorthand config can choose an invalid common model pair, and advisor/server-side result blocks are converted to display-only text instead of preserving the raw blocks Anthropic expects on later turns.

## What This Changes

The PR adds Anthropic advisor beta header/tool injection, generic server-side tool-result display handling, stop-reason hardening, advisor docs, live validation scripts, and focused Anthropic transport/wrapper/chat-content tests.

## Best Possible Solution

Implement a maintainer-reviewed advisor feature that derives or requires compatible advisor models, preserves raw server-side result blocks through replay and pruning, and documents the supported beta behavior accurately.

## Reproduction Assessment

Yes. Source inspection of PR head reproduces the two blockers: params.advisor: true resolves to claude-sonnet-4-6 for the documented Sonnet 4.6 executor, and the stream/replay path stores advisor_tool_result content as serverToolDisplay text instead of raw blocks.

## Solution Assessment

No. The feature direction is useful, but this implementation is not the best current solution because it needs pair-aware advisor config and raw server-side block preservation instead of a fixed Sonnet default and display-only replay.

  - confidence: 0.91
