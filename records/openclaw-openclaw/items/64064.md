## Summary

Keep open. Current main still lacks the requested Anthropic advisor support, but this PR should not merge as-is because the documented/default advisor pair can be rejected by Anthropic and the patch does not preserve raw server-side tool result blocks for multi-turn replay.

## What This Changes

The branch adds Anthropic advisor beta config/header/tool injection, display handling for server-side tool results, stop-reason hardening, docs, validation scripts, and focused tests.

## Best Possible Solution

Land a maintainer-reviewed advisor implementation that derives or requires valid executor/advisor pairs, preserves raw server-side tool result blocks through replay and pruning, and keeps the user-facing docs aligned with Anthropic's beta contract.

## Reproduction Assessment

Yes, source-reproducible. Current main lacks advisor config/tool injection and server-side result handling, and the PR-head blockers are visible in the advisor resolver, replay conversion, and Anthropic's documented beta contract.

## Solution Assessment

No. The direction is useful, but the implementation should use a pair-aware advisor model contract and preserve raw server-side blocks instead of replacing them with display-only text history.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Choose a compatible advisor default:** `extensions/anthropic/stream-wrappers.ts:225-241`
  - body: `advisor: true` resolves to `claude-sonnet-4-6`, and the new docs show that shorthand on a `claude-sonnet-4-6` executor. Anthropic requires that executor to pair with `claude-opus-4-7` as advisor and returns a 400 for invalid pairs, so the documented common setup can fail before the stream handling runs. ([platform.claude.com](https://platform.claude.com/docs/en/agents-and-tools/tool-use/advisor-tool))
  - confidence: 0.93
- **[P2] Preserve raw server-side tool blocks:** `src/agents/anthropic-transport-stream.ts:833-837`
  - body: The stream handler stores `advisor_tool_result` and encrypted/redacted server-tool results as synthetic display text and later skips or downgrades that data on Anthropic replay. Anthropic documents these as raw assistant content that should be round-tripped, including encrypted result content that is decrypted on the next turn, so replacing them with text-only history breaks multi-turn advisor context. ([platform.claude.com](https://platform.claude.com/docs/en/agents-and-tools/tool-use/advisor-tool))
  - confidence: 0.91
