## Summary

Keep open. Current main does not implement advisor support, and this PR still has merge-blocking Anthropic compatibility issues around the default advisor pair and raw server-side block replay.

## What This Changes

The PR adds Anthropic advisor beta header/tool injection, server-side tool-result display handling, stop-reason hardening, docs, validation scripts, and focused tests.

## Best Possible Solution

Land a maintainer-reviewed implementation that uses compatible pair-aware advisor config, stores and replays raw server-side blocks through pruning, and documents the supported contract.

## Reproduction Assessment

Yes. Enable the PR's documented `params.advisor: true` on `anthropic/claude-sonnet-4-6` and inspect that it injects Sonnet as the advisor; also replay an assistant turn containing `advisor_tool_result` and observe that the raw block is not sent back.

## Solution Assessment

No. The feature direction is useful, but display-only server-tool text and a fixed Sonnet advisor default do not satisfy Anthropic's current API contract; raw block preservation with compatible model-pair validation is the safer design.
