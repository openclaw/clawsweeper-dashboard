## Summary

Keep open: the feature is useful and current main still lacks it, but this PR violates current Anthropic advisor model-pair and multi-turn replay contracts, so it should not merge as-is.

## What This Changes

This PR adds Anthropic advisor beta opt-in, server-side tool-result display handling, unknown stop-reason hardening, advisor docs, validation scripts, and focused tests.

## Best Possible Solution

Land a maintainer-reviewed advisor implementation that uses compatible pair-aware config or explicit advisor selection, preserves and prunes raw server-side result blocks for Anthropic replay, and keeps docs aligned with those contracts.

## Reproduction Assessment

Yes. Source inspection of PR head plus Anthropic docs and SDK types shows `advisor: true` resolves to a Sonnet advisor for a Sonnet executor, and `advisor_tool_result` or redacted results become display text rather than replayable raw blocks.

## Solution Assessment

No. The direction is useful, but the current patch needs a pair-aware advisor model contract and raw server-side block preservation through replay/pruning instead of a fixed Sonnet default and display-only history.
