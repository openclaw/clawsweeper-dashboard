## Summary

Keep open. The PR targets a real missing Anthropic advisor/server-side-tool feature, and the discussion includes useful live Anthropic validation, but the current patch is not merge-ready: current main still lacks advisor support, the PR head is unmergeable, and the patch conflicts with Anthropic's current advisor contract for model pairing and multi-turn result replay. ([platform.claude.com](https://platform.claude.com/docs/en/agents-and-tools/tool-use/advisor-tool))

## What This Changes

The PR adds Anthropic advisor-tool beta header/tool injection, server-side tool-result display handling, stop-reason hardening, user docs, live validation scripts, and focused transport/wrapper/chat-content tests.

## Best Possible Solution

Rebase the PR or replace it with a focused maintainer-reviewed implementation that uses a valid advisor model contract, preserves server-side advisor result blocks through transcript storage/replay and pruning, documents the supported config shape, and then uses that implementation to resolve the open advisor tracker while leaving already-closed satellite requests closed.

## Reproduction Assessment

Yes. For the blocking findings, the reproduction path is contract-level and code-level: enable params.advisor: true on a Sonnet 4.6 executor and inspect the outgoing PR payload/default model, then run a multi-turn advisor result through convertAnthropicMessages and verify the next request lacks the required advisor_tool_result block.

## Solution Assessment

No. The direction is useful, but the patch's display-only server-tool representation is not the best fix because Anthropic requires verbatim advisor result replay; a raw/opaque server-block representation integrated with pruning is the safer maintainable path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Use a compatible default advisor model:** `extensions/anthropic/stream-wrappers.ts:225`
  - body: `params.advisor: true` injects `claude-sonnet-4-6` as the advisor model while the docs advertise the shorthand for a Sonnet 4.6 executor. Anthropic's current compatibility table requires that executor to use Opus 4.7 as the advisor, and invalid pairs return 400, so the documented common config will fail until the default is changed or users must choose a compatible model explicitly.
  - confidence: 0.92
- **[P2] Round-trip advisor result blocks:** `src/agents/anthropic-transport-stream.ts:283-290`
  - body: The replay path skips `serverToolDisplay` blocks or falls back to plain text, so `advisor_tool_result` and redacted `encrypted_content` are not sent back as Anthropic expects on later turns. This loses required advisor state and can make follow-up requests invalid or lower quality; store and replay the raw server-side blocks instead of display-only text.
  - confidence: 0.89
