## Summary

Keep this PR open for maintainer follow-up. The feature is still missing from current main and has useful live validation, but the branch is conflicting and the patch still has blocking provider-contract issues around current Anthropic wrapper behavior, advisor model compatibility, and raw server-side tool replay.

## What This Changes

The PR adds Anthropic advisor-tool opt-in config, beta/header and tool injection, server-side tool result display handling, stop-reason hardening, docs, validation scripts, and tests.

## Best Possible Solution

Land a maintainer-reviewed Anthropic provider implementation rebased on current main that preserves the thinking-prefill wrapper, derives or requires valid executor/advisor pairs, round-trips raw server-side tool blocks through replay and pruning, and keeps docs/tests aligned with Anthropic's beta contract.

## Reproduction Assessment

Yes. Source inspection gives high-confidence reproduction paths: current main lacks advisor injection and server-side block handling, while PR head has visible model-pair, replay, and stale-wrapper regressions against current main and the Anthropic docs contract.

## Solution Assessment

No. The direction is useful, but the patch should preserve current-main wrapper behavior and implement pair-aware config plus raw server-side block replay instead of using a Sonnet default and display-only history.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Re-add the thinking prefill wrapper:** `extensions/anthropic/stream-wrappers.ts:319`
  - body: Current main always composes `createAnthropicThinkingPrefillWrapper` so extended-thinking Anthropic requests do not end with an assistant prefill. This PR's wrapper composition ends with the advisor wrapper and drops that current-main guard, so rebasing/merging it would reintroduce rejected thinking-prefill payloads.
  - confidence: 0.94
- **[P2] Choose a compatible advisor default:** `extensions/anthropic/stream-wrappers.ts:225-241`
  - body: `advisor: true` resolves to `claude-sonnet-4-6`, and the new docs show that shorthand on a `claude-sonnet-4-6` executor. Anthropic documents that executor/advisor models must form a valid pair with the advisor at least as capable as the executor and returns 400 for invalid pairs, so the documented common setup can fail before streaming runs.
  - confidence: 0.93
- **[P2] Preserve raw server-side tool blocks:** `src/agents/anthropic-transport-stream.ts:833-837`
  - body: The stream handler stores `advisor_tool_result` and encrypted/redacted server-tool results as synthetic display text and later skips or downgrades that data on Anthropic replay. Anthropic's contract says to pass the full assistant content, including `advisor_tool_result` blocks, back on subsequent turns, so text-only history loses required multi-turn advisor/server-tool context.
  - confidence: 0.92
