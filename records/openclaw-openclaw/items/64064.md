## Summary

Keep open. Current main still lacks Anthropic advisor support, and this PR is the active implementation candidate, but the patch should not merge yet because its documented shorthand emits an invalid Sonnet-to-Sonnet advisor pair and its replay path discards the raw advisor/server-tool blocks that Anthropic expects on later turns.

## What This Changes

The PR adds Anthropic advisor beta header/tool injection, server-side tool-result display handling, stop-reason hardening, docs, live validation scripts, and focused transport/wrapper/chat-content tests.

## Best Possible Solution

Rework the implementation around a pair-aware advisor config default and a raw server-side block representation that survives transcript storage, provider replay, pruning, and docs without polluting cross-provider fallback text.

## Reproduction Assessment

Yes. Enable `params.advisor: true` for an Anthropic Sonnet 4.6 model and inspect the outgoing advisor payload for the invalid default model; separately replay an assistant turn containing `advisor_tool_result` through `convertAnthropicMessages` and verify the raw result block is not sent back.

## Solution Assessment

No. The feature direction is useful, but replacing raw server-tool blocks with display text is not the narrowest maintainable solution for Anthropic's replay contract; the safer fix stores opaque/raw blocks and makes stripping explicit only when the advisor tool is removed.
