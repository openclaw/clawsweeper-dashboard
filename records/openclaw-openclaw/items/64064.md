## Summary

Keep open. The feature is still useful and current main still lacks advisor support, but this PR needs changes before merge because the branch drops current Anthropic wrapper behavior and does not satisfy Anthropic's advisor pairing and replay contracts.

## What This Changes

The PR adds Anthropic advisor-tool configuration, beta header/tool injection, server-side tool result display handling, docs, validation scripts, and tests.

## Best Possible Solution

Land a maintainer-reviewed Anthropic provider implementation rebased on current main that preserves the thinking-prefill wrapper, validates or requires compatible executor/advisor pairs, round-trips raw server-side tool blocks through replay/pruning, and keeps docs/tests aligned with Anthropic's beta contract.

## Reproduction Assessment

Yes. Source inspection gives high-confidence reproduction paths for the review findings: PR head omits the current-main wrapper, defaults Sonnet-as-advisor for a Sonnet executor, and converts server-side result blocks to display-only text instead of preserving raw replay content.

## Solution Assessment

No. The direction is useful, but the patch should preserve current-main wrapper behavior and implement pair-aware config plus raw server-side block replay instead of using a Sonnet default and display-only history.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Preserve the thinking prefill wrapper:** `extensions/anthropic/stream-wrappers.ts:319`
  - body: Current main always composes `createAnthropicThinkingPrefillWrapper` so extended-thinking Anthropic requests do not end with an assistant prefill. This branch's wrapper chain ends with the advisor wrapper and omits that guard, so rebasing or merging it would reintroduce rejected thinking-prefill payloads.
  - confidence: 0.94
- **[P2] Use a compatible advisor default:** `extensions/anthropic/stream-wrappers.ts:225-241`
  - body: `advisor: true` resolves to `claude-sonnet-4-6`, and the docs show that shorthand on a `claude-sonnet-4-6` executor. Anthropic's advisor contract requires the advisor to be at least as capable as the executor, with Sonnet 4.6 paired to Opus 4.7, so the documented common setup can fail with an invalid model pair.
  - confidence: 0.93
- **[P2] Round-trip raw server-side tool blocks:** `src/agents/anthropic-transport-stream.ts:833-837`
  - body: The stream handler stores advisor/server-tool results as synthetic display text and later skips or downgrades that data on Anthropic replay. Anthropic's SDK contract expects assistant content such as `advisor_tool_result` and encrypted result payloads to be passed back verbatim on later turns, so multi-turn advisor/server-tool state is lost.
  - confidence: 0.92
