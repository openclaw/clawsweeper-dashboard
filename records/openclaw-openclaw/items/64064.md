## Summary

Keep open. Current main still lacks Anthropic advisor support, but the PR has blocking API-contract issues around valid advisor model pairing and replay of server-side result blocks, so it should not be merged as-is.

## What This Changes

Adds Anthropic advisor beta support by injecting advisor tools and beta headers from model params, translating server-side tool results into display text, hardening stop-reason mapping, and adding docs/tests/live validation scripts.

## Best Possible Solution

Land a maintainer-reviewed advisor implementation that derives or requires valid executor/advisor pairs, preserves raw server-side result blocks through replay/pruning, and keeps docs aligned with Anthropic's beta contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main drops server_tool_use/advisor_tool_result blocks, cannot inject advisor_20260301 tools, and throws on unknown Anthropic stop reasons; the PR-head blockers are visible in the resolver and replay conversion.

## Solution Assessment

No. The direction is useful, but this implementation should use a pair-aware advisor model contract and preserve raw server-side blocks instead of replacing the API replay data with display-only text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Choose a compatible advisor default:** `extensions/anthropic/stream-wrappers.ts:225-241`
  - body: `advisor: true` resolves to `claude-sonnet-4-6`, and the new docs show that shorthand on a `claude-sonnet-4-6` executor. Anthropic requires that executor to pair with `claude-opus-4-7` as advisor and returns a 400 for invalid pairs, so the documented common setup can fail before the stream handling runs. ([platform.claude.com](https://platform.claude.com/docs/en/agents-and-tools/tool-use/advisor-tool?utm_source=openai))
  - confidence: 0.93
- **[P2] Preserve raw server-side tool blocks:** `src/agents/anthropic-transport-stream.ts:833-837`
  - body: The stream handler stores `advisor_tool_result` and encrypted/redacted server-tool results as synthetic display text and later skips or downgrades that data on Anthropic replay. Anthropic documents these as raw assistant content that should be round-tripped, including encrypted result content, so replacing them with text-only history breaks multi-turn advisor context. ([platform.claude.com](https://platform.claude.com/docs/en/agents-and-tools/tool-use/advisor-tool?utm_source=openai))
  - confidence: 0.91
