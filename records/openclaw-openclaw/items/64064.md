## Summary

This PR should stay open because the feature is still useful and not implemented on current main, but the current branch has blocking compatibility and transcript-preservation concerns before merge.

## What This Changes

The PR adds Anthropic advisor beta header and tool injection, server-side tool result display handling, stop-reason hardening, docs, validation scripts, and focused tests.

## Best Possible Solution

Land a maintainer-reviewed advisor implementation that derives or requires valid executor/advisor pairs, preserves raw server-side blocks through replay and pruning, and keeps https://github.com/openclaw/openclaw/issues/63930 as the canonical feature thread until the design is complete.

## Reproduction Assessment

Yes. The issue is source-reproducible: current main lacks advisor injection and server-side result handling, while PR head adds both but converts result blocks into display text rather than raw replayable Anthropic blocks.

## Solution Assessment

No. The direction is useful, but the current patch should use a pair-aware advisor model contract and preserve raw server-side blocks instead of relying on display-only replay.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Choose a compatible advisor default:** `extensions/anthropic/stream-wrappers.ts:225-241`
  - body: `advisor: true` resolves to `claude-sonnet-4-6`, and the new docs show that shorthand under a `claude-sonnet-4-6` executor. Anthropic's advisor beta is model-pair constrained, so the documented common config can fail before the stream-handling code runs; make the shorthand pair-aware or require an explicit compatible advisor model.
  - confidence: 0.91
- **[P2] Preserve raw server-side tool blocks:** `src/agents/anthropic-transport-stream.ts:833-837`
  - body: The new stream path stores `advisor_tool_result` and encrypted/redacted server-tool results as synthetic display text, then skips or downgrades them on later Anthropic requests. The API exposes these as server-side content blocks and encrypted content needs verbatim preservation for full multi-turn behavior, so retain and prune the raw blocks instead of replacing them with text-only history.
  - confidence: 0.9
