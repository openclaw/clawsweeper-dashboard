## Summary

Keep open. Current main still lacks Anthropic advisor support, but this PR is not ready because the documented `advisor: true` path injects an invalid Sonnet/Sonnet advisor pair and the transport drops raw advisor result blocks that Anthropic requires for multi-turn replay.

## What This Changes

The PR adds Anthropic advisor beta tool injection, server-side tool-result display handling, stop-reason hardening, docs, live validation scripts, and focused tests.

## Best Possible Solution

Rebase or replace this PR with a maintainer-reviewed implementation that uses pair-aware advisor config, preserves raw advisor/server-side result blocks through transcript replay and pruning, and documents the supported contract.

## Reproduction Assessment

Yes. The diff gives deterministic checks: `advisor: true` for `anthropic/claude-sonnet-4-6` injects `claude-sonnet-4-6` as advisor, and replaying an assistant turn containing `advisor_tool_result` through `convertAnthropicMessages` omits the raw result block.

## Solution Assessment

No. The feature direction is useful, but a static default advisor model and display-only server-tool text are not the narrow maintainable fix for Anthropic's current advisor replay contract.
