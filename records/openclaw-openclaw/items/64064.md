## Best Possible Solution

Land a maintainer-reviewed Anthropic provider implementation that preserves current wrapper composition, validates compatible executor/advisor pairs, round-trips raw server-side blocks through replay/pruning, and keeps docs/tests aligned with the beta contract.

## Reproduction Assessment

## Solution Assessment

No. The feature direction is useful, but the current patch should not merge until it preserves thinking-prefill behavior, uses or validates compatible advisor pairs, and round-trips raw server-side blocks instead of synthetic display-only text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Preserve the thinking prefill wrapper:** `extensions/anthropic/stream-wrappers.ts:319`
  - body: Current main always composes the Anthropic thinking-prefill guard so extended-thinking requests do not end with an assistant prefill. This wrapper chain ends after the advisor wrapper, so merging it after rebase would reintroduce rejected thinking-prefill payloads.
  - confidence: 0.94
- **[P2] Use a compatible advisor default:** `extensions/anthropic/stream-wrappers.ts:225-241`
  - body: `advisor: true` resolves to `claude-sonnet-4-6`, and the docs show that shorthand on a Sonnet 4.6 executor. Anthropic requires the advisor to be at least as capable as the executor and lists Opus 4.7 for Sonnet 4.6, so the documented common setup can fail with a 400.
  - confidence: 0.93
- **[P2] Round-trip raw server-side tool blocks:** `src/agents/anthropic-transport-stream.ts:833-837`
  - body: The stream handler stores advisor/server-tool results as synthetic display text and skips or downgrades them during Anthropic replay. Anthropic expects full assistant content, including `advisor_tool_result` and encrypted content, on later turns, so multi-turn advisor state is lost.
  - confidence: 0.92
