## Summary

Keep open. Current main still omits `sessionKey` from the plugin `message_sent` canonical payload even though the emitter already has the key and uses it for the internal `message:sent` hook.

## What This Changes

The issue requests that `deliverOutboundPayloads` forward the available session key into the plugin `message_sent` event and context.

## Best Possible Solution

Patch `createMessageSentEmitter` to pass `params.sessionKeyForInternalHooks` into `buildCanonicalSentMessageHookContext`, then add focused delivery-path regression coverage for both the plugin event and context.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register a `message_sent` hook, call `deliverOutboundPayloads` with `session.key` or `mirror.sessionKey`, and the current canonical payload will omit `sessionKey` before both plugin mappers run.

## Solution Assessment

Yes. Forwarding the already-computed `sessionKeyForInternalHooks` into the canonical builder is the narrowest maintainable fix because the public types, docs, and mapper tests already support the field.
