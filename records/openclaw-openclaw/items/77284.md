## Summary

Current main still omits `sessionKey` from the plugin `message_sent` canonical payload in `createMessageSentEmitter`, even though the same outbound path computes that key and uses it for the internal `message:sent` hook. The linked fix PR remains open and unmerged, so this issue should stay open until that or an equivalent patch lands.

## What This Changes

The issue requests that `createMessageSentEmitter` forward the already-computed outbound session key into the plugin `message_sent` event and context.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/73706 or an equivalent narrow patch that forwards `sessionKeyForInternalHooks` into the canonical sent-message context and adds delivery-path regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: send through `deliverOutboundPayloads` with `session.key` or `mirror.sessionKey` and a plugin `message_sent` hook will not receive `sessionKey` because the emitter omits it before the mappers run.

## Solution Assessment

Yes. Forwarding the already-computed `sessionKeyForInternalHooks` into `buildCanonicalSentMessageHookContext` is the narrowest maintainable fix because the public hook types and mappers already support the field.
