## Summary

Keep open. Current main and the latest release still drop the already-computed outbound session key before the plugin `message_sent` event/context are built, while the open fix PR remains the active path.

## What This Changes

The issue requests forwarding the outbound delivery session key into the plugin `message_sent` canonical event and context.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73706, or an equivalent narrow patch that passes `sessionKeyForInternalHooks` into `buildCanonicalSentMessageHookContext` with regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `deliverOutboundPayloads` computes `sessionKeyForInternalHooks`, but `createMessageSentEmitter` does not put it into the canonical sent-message context before the plugin mappers run.

## Solution Assessment

Yes. Forwarding the already-computed `sessionKeyForInternalHooks` into the canonical builder is the narrowest maintainable fix because the public types and mappers already support optional `sessionKey`.
