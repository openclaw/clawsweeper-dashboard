## Summary

Keep open. Current main still drops the available session key before emitting the plugin `message_sent` hook even though the public hook types, docs, and mappers already support that correlation field.

## What This Changes

The issue asks for outbound `message_sent` plugin events and contexts emitted by `deliverOutboundPayloads` to include the available session key.

## Best Possible Solution

Forward `params.sessionKeyForInternalHooks` into `buildCanonicalSentMessageHookContext` in `createMessageSentEmitter`, then add focused regression coverage that asserts both plugin event and context include it.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `deliverOutboundPayloads` with `mirror.sessionKey` or `session.key` and `message_sent` hooks enabled, and current main drops the key before the plugin mappers run.

## Solution Assessment

Yes. Forwarding the already-computed key into the canonical builder is the narrowest maintainable fix because the mapper, public event type, and docs already support the field.
