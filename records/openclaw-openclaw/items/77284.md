## Summary

## What This Changes

The issue requests forwarding the already-computed outbound `sessionKey` into the plugin `message_sent` canonical event/context emitted by `createMessageSentEmitter`.

## Best Possible Solution

Keep the shipped canonical `sessionKey` forwarding for `deliverOutboundPayloads` and leave remaining outbound hook coverage work to https://github.com/openclaw/openclaw/issues/50126.

## Reproduction Assessment

Yes. The original source-level failure path is clear: without forwarding `sessionKey` into the canonical sent-message context, both plugin mappers omitted it; current main no longer has that failure path.

## Solution Assessment

Yes. The implemented fix is the narrow owner-boundary change: reuse the already-computed `sessionKeyForInternalHooks` in `createMessageSentEmitter` and keep the existing canonical mappers as the single public hook mapping path.
