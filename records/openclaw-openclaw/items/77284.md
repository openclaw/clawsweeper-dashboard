## Summary

Keep open. Current main still builds the plugin `message_sent` canonical payload without `sessionKey`, while the same delivery path computes and passes that key to the internal `message:sent` hook; an open PR appears to cover the fix, but it has not landed on main yet.

## What This Changes

Request to thread the existing outbound session key into the plugin `message_sent` event and context emitted by `createMessageSentEmitter`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/73706 or an equivalent narrow patch that forwards `sessionKeyForInternalHooks` into the canonical sent-message context and pins the plugin event/context behavior with delivery-path regression tests.

## Reproduction Assessment

Yes, as a high-confidence source reproduction rather than a live run: register a `message_sent` plugin hook and send through `deliverOutboundPayloads` with `session.key` or `mirror.sessionKey`; current main omits that key before the plugin mappers run.

## Solution Assessment

Yes. Forwarding the already-computed `sessionKeyForInternalHooks` into `buildCanonicalSentMessageHookContext` is the narrowest maintainable fix because the public types, docs, and mappers already support the field.
