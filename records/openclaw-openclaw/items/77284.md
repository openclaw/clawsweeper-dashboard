## Summary

Keep open. Current `main` still drops the available session key before building the canonical plugin `message_sent` payload, while the internal `message:sent` hook on the same path receives it and the public plugin hook contract already supports it.

## What This Changes

Forward the available outbound session key into the plugin `message_sent` event and context emitted from `deliverOutboundPayloads`.

## Best Possible Solution

Patch the outbound emitter to pass the already-computed session key into the canonical sent-message context and lock it with a delivery-path regression test.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `deliverOutboundPayloads` with `session.key` or `mirror.sessionKey` and a registered `message_sent` hook, and current `main` builds the plugin event/context from a canonical object without `sessionKey`.

## Solution Assessment

Yes. Forwarding `params.sessionKeyForInternalHooks` into `buildCanonicalSentMessageHookContext` is the narrowest maintainable fix because the docs, types, builder, and plugin mappers already support the field.
