## Summary

Keep open. Current main still drops the available session key from the plugin `message_sent` canonical payload while the same emitter forwards it to the internal `message:sent` hook, and the public plugin hook contract already supports the field.

## What This Changes

Forward the outbound delivery session key into the plugin `message_sent` event and context emitted by `createMessageSentEmitter`.

## Best Possible Solution

Populate the canonical sent-message context with `params.sessionKeyForInternalHooks`, then add focused regression coverage for both plugin event and context on the shared delivery path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: invoke `deliverOutboundPayloads` with a `session.key` or `mirror.sessionKey` and a registered `message_sent` hook, and the plugin event/context will be built from a canonical object that lacks `sessionKey`.

## Solution Assessment

Yes. Forwarding the already-computed `sessionKeyForInternalHooks` into `buildCanonicalSentMessageHookContext` is the narrowest maintainable fix because the builder, mappers, docs, and internal hook path already support the field.
