## Summary

Keep open. Current main still builds the plugin `message_sent` payload without the available session key, while the same delivery path passes that key to the internal hook and the public plugin mappers already support it.

## What This Changes

The issue requests that outbound `message_sent` plugin events and contexts include `sessionKey` when `deliverOutboundPayloads` has a session or mirror session key.

## Best Possible Solution

Forward `params.sessionKeyForInternalHooks` into `buildCanonicalSentMessageHookContext` and add a focused outbound delivery regression test for both plugin event and context session-key propagation.

## Reproduction Assessment

Yes. Registering a `message_sent` hook and calling `deliverOutboundPayloads` with `session.key` or `mirror.sessionKey` will route through the current source path that omits `sessionKey` before the plugin mappers run.

## Solution Assessment

Yes. Passing the already-computed `sessionKeyForInternalHooks` into the canonical builder is the narrowest maintainable fix because the public types, docs, and mappers already support the field.
