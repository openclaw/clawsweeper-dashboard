## Summary

Keep open: current main still drops the already-computed session key from the plugin `message_sent` canonical payload, and the linked fix remains open and unmerged.

## What This Changes

Forward the outbound delivery session key into the plugin `message_sent` canonical context so plugins receive it in both the event and hook context.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/73706 or an equivalent narrow patch that forwards `sessionKeyForInternalHooks` into the sent-message canonical context and adds delivery-path regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `deliverOutboundPayloadsCore` computes `sessionKeyForInternalHooks`, `createMessageSentEmitter` omits it from the canonical plugin payload, and the mappers only emit `sessionKey` when canonical contains it.

## Solution Assessment

Yes. Forwarding the already-computed `sessionKeyForInternalHooks` into `buildCanonicalSentMessageHookContext` is the narrowest maintainable fix because the public hook types and mappers already support the field.
