## Summary

Keep open. Current `main` still builds the plugin `message_sent` canonical payload without the session key even though the same emitter has it available for the internal `message:sent` hook, and the plugin hook docs/types already support that correlation field.

## What This Changes

Ensure `deliverOutboundPayloads` passes the available session key through `createMessageSentEmitter` so plugin `message_sent` event and context include `sessionKey`.

## Best Possible Solution

Patch `createMessageSentEmitter` to populate canonical `sessionKey` from `params.sessionKeyForInternalHooks`, then add a focused regression assertion for the plugin event and context on the delivery path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: register a `message_sent` hook, call `deliverOutboundPayloads` with `session.key` or `mirror.sessionKey`, and the current canonical payload will omit `sessionKey` before both plugin mappers run.

## Solution Assessment

Yes. Forwarding the already-computed `sessionKeyForInternalHooks` into the canonical builder is the narrowest maintainable fix because the public types, docs, and mapper tests already support the field.
