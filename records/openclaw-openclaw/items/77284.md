## Summary

Keep open. Current main still drops the available session key before the plugin `message_sent` mapper runs, while the public hook contract and internal `message:sent` path already support that field.

## What This Changes

The issue asks OpenClaw to include the available session key in plugin `message_sent` event and context payloads emitted from `deliverOutboundPayloads`.

## Best Possible Solution

Forward `params.sessionKeyForInternalHooks` into the canonical sent-message context in the shared outbound emitter and cover both plugin event and context payloads with a focused regression test.

## Reproduction Assessment

Yes. Register a `message_sent` plugin hook and call `deliverOutboundPayloads` with `session.key` or `mirror.sessionKey`; current source builds the canonical plugin payload without `sessionKey` before both plugin mappers run.

## Solution Assessment

Yes. Passing the already-computed `sessionKeyForInternalHooks` into `buildCanonicalSentMessageHookContext` is the narrowest maintainable fix because the public types, docs, and mappers already support the field.
