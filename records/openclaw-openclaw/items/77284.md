## Summary

Keep open. Current main still drops the available outbound `sessionKey` before the plugin `message_sent` mappers run, and the linked fix at https://github.com/openclaw/openclaw/pull/73706 remains open and unmerged.

## What This Changes

Forward the already-computed outbound session key into the plugin `message_sent` canonical context.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/73706 or an equivalent narrow patch that threads `sessionKeyForInternalHooks` into the sent-message canonical context and adds regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send through `deliverOutboundPayloads` with `session.key` or `mirror.sessionKey`, and a plugin `message_sent` hook will receive no `sessionKey` because the emitter omits it before the mappers run.

## Solution Assessment

Yes. Forwarding the already-computed `sessionKeyForInternalHooks` into `buildCanonicalSentMessageHookContext` is the narrowest maintainable fix because the public hook types and mappers already support the field.
