## Summary

Keep open: current main still deletes the Twitch client-manager registry entry only after `disconnectAll()` completes, and the open linked fix PR has not merged yet.

## What This Changes

The issue asks for Twitch `removeClientManager` to evict its registry entry even when manager disconnect cleanup throws.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/84375, then close this issue only after the fix reaches main.

## Reproduction Assessment

Yes. Source inspection on current main shows `removeClientManager` deletes only after `disconnectAll()` completes, so mocking `disconnectAll` to reject would leave `getClientManager(accountId)` populated.

## Solution Assessment

Yes. Moving `registry.delete(accountId)` into a `finally` block while preserving the original disconnect error is the narrow maintainable fix, and the linked PR is the right implementation path to review.
