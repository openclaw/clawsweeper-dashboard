## Summary

Keep open. Current main still has only global session visibility and agent-to-agent policy, so the PR is not obsolete, but the proposed patch is incomplete for an access-control feature: it accepts per-agent config without enforcing it across all session tools, and it misses required generated config artifacts and changelog coverage.

## What This Changes

The PR adds per-agent `agents.list[].tools.sessions` and `agentToAgent` config schema/types/help, threads those overrides into session list/send policy resolution, and adds session-access coverage plus an unrelated session path regression test.

## Best Possible Solution

Settle the canonical per-agent session/A2A policy shape, then implement it in the shared session-visibility helper and every session tool surface (`sessions_list`, `sessions_history`, `sessions_send`, and `session_status`) with generated config schema, docs/changelog, and focused deny/allow tests for each tool.

## Reproduction Assessment

Yes. A source-level reproduction is clear: configure global `tools.sessions.visibility: "all"` with a per-agent `sessions.visibility: "self"`, then call `sessions_history` or `session_status` against another agent; as submitted, those callers still resolve visibility and A2A policy from global config only.

## Solution Assessment

No, not as submitted. Per-agent overrides are a plausible direction, but the patch must enforce them uniformly and update generated config/changelog artifacts; a safer implementation would centralize requester-agent tool resolution so all session tools share the same policy path.
