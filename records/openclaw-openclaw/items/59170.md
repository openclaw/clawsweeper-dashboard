- **[P1] Wire per-agent policy through every session tool:** `src/config/zod-schema.agent-runtime.ts:582`
  - body: The schema accepts `agents.list[].tools.sessions` and `agentToAgent`, but this branch only passes requester tools into `sessions_list` and `sessions_send`. `sessions_history` and `session_status` still build policy from the global config, so a restrictive per-agent setting would not protect transcript reads or status lookups. Please wire the same requester-agent resolution through all four session tools before exposing the config.
  - confidence: 0.93
- **[P2] Keep global A2A policy as an upper bound:** `src/agents/tools/sessions-access.ts:112-114`
  - body: When a per-agent `allow` is present, this replaces the global allowlist instead of combining with it. That lets a per-agent entry bypass the existing global participation gate, which is a security-sensitive behavior change for cross-agent sends/list/history/status. Please preserve the global gate as an upper bound, or get explicit maintainer approval and tests for the new override semantics.
  - confidence: 0.78
