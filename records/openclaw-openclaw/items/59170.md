## Summary

Keep open: the requested per-agent access-control capability is still useful and not implemented on current main, but this PR is not merge-ready because it lacks real behavior proof and only partially enforces the new policy surface.

## What This Changes

The PR adds per-agent `agents.list[].tools.sessions` and `agents.list[].tools.agentToAgent` config with schema/help updates, policy-helper changes, list/send wiring, and focused tests.

## Best Possible Solution

Define one fail-closed per-agent access-control contract in `src/plugin-sdk/session-visibility.ts`, cover schema/types/docs/generated metadata and every session tool, and align it with the open A2A and sandbox visibility work before merge.

No. Per-agent scoping is a reasonable direction, but this patch is not the best merge path until enforcement is centralized, history/status are covered, generated metadata is refreshed, real behavior proof is supplied, and maintainers choose the final access-control semantics.
