## Summary

Keep open. Current main still lacks this per-agent `agents.list[].tools` session/A2A policy surface, but the PR is not correct as submitted because it exposes new access-control config without enforcing it across all session tools and without generated schema/changelog follow-through.

## What This Changes

The PR adds per-agent `agents.list[].tools.sessions` and `agentToAgent` schema/types/help, threads those overrides into session list/send policy, and adds session-access plus unrelated session-path tests.

## Best Possible Solution

Centralize requester-agent tool resolution, enforce the selected per-agent policy through `sessions_list`, `sessions_history`, `sessions_send`, `session_status`, and plugin-sdk session visibility consumers, then regenerate config artifacts and add docs/changelog coverage.

## Reproduction Assessment

Yes. A source-level reproduction is clear: configure global `tools.sessions.visibility: "all"` with a per-agent `sessions.visibility: "self"`, then call `sessions_history` or `session_status` for another agent; the submitted diff leaves those tools on global policy.

## Solution Assessment

No. Per-agent overrides are a plausible direction, but this patch only threads them into part of the session-tool surface and does not handle generated schema or changelog requirements.
