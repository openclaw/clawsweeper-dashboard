Adds per-agent `agents.list[].tools.sessions` and `agents.list[].tools.agentToAgent` config, policy-helper overrides, list/send wiring, schema/help labels, focused tests, and an unrelated session-path regression test.

## Best Possible Solution

Define one fail-closed per-agent access-control contract in `src/plugin-sdk/session-visibility.ts`, covering schema/types/help/docs/baseline and all session tools, or intentionally split the work through the narrower active A2A and sandbox-visibility PRs.

## Solution Assessment

No. Per-agent scoping is a reasonable direction, but this patch is not the best merge path until enforcement is centralized on the current main helper, all session surfaces are covered, generated metadata is refreshed, proof is supplied, and maintainers choose the final semantics.
