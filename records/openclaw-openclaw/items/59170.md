## Summary

Keep open. Current main still has global-only session/A2A policy, and this PR is a plausible feature direction, but the branch is not merge-ready because it accepts new per-agent access-control config before enforcing it consistently across all session tools, omits the generated config baseline update, and lacks real behavior proof.

## What This Changes

The PR adds per-agent `agents.list[].tools.sessions` and `agentToAgent` config schema/types/help plus session access helper and list/send wiring for requester-specific visibility and A2A policy.

## Best Possible Solution

Settle one shared requester-aware access-control contract in the Plugin SDK helper, enforce it through `sessions_list`, `sessions_history`, `sessions_send`, and `session_status`, then update schema/types/docs/generated metadata and focused tests in one coherent PR.

## Reproduction Assessment

Yes for the PR findings. Source inspection shows the patch makes per-agent access-control config valid while leaving history/status on global-only policy, and the missing generated config baseline is visible from the live PR file list and repo scripts.

## Solution Assessment

No. Per-agent scoping is a reasonable direction, but this patch is not the best merge path until it centralizes requester-agent policy resolution, covers every session surface, refreshes generated config metadata, and aligns with adjacent policy PRs.
