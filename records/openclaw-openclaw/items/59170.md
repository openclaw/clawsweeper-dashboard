- **[P1] Wire the override through every session tool:** `src/config/zod-schema.agent-runtime.ts:582-592`
  - body: The schema now accepts per-agent `sessions` and `agentToAgent` settings, but the PR head only passes `agentEntry?.tools` into `sessions_list` and `sessions_send`; `sessions_history` and `session_status` still build the global A2A policy and global effective visibility. That means an operator can configure a specialist as `visibility: "self"` or `agentToAgent.enabled: false` and still have transcript/status access governed by the broad global setting, which breaks the least-privilege boundary this PR is adding.
  - confidence: 0.9

  - body: When `agents.list[].tools.agentToAgent.allow` is explicitly `[]`, this branch selects the per-agent array and then calls `matchesAllow(targetAgentId)`, but `matchesAllow()` returns true for an empty list. A configured empty allowlist therefore permits every target while A2A is enabled, instead of isolating the agent as an allowlist override should.
  - confidence: 0.88
