## Summary

Keep open. The feature is still useful, but the live PR is incomplete for an access-control change: it accepts per-agent policy config while only wiring part of the session-tool surface, omits the generated config baseline, conflicts with current main, and lacks real behavior proof.

## What This Changes

The PR adds per-agent `agents.list[].tools.sessions` and `agents.list[].tools.agentToAgent` config metadata plus requester-aware list/send policy wiring, focused tests, and an unrelated session-path regression test.

## Best Possible Solution

Review or replace this with one requester-aware contract in `src/plugin-sdk/session-visibility.ts` that covers schema/types/docs, generated baselines, all four session tools, and aligned A2A/per-agent semantics.

## Reproduction Assessment

Yes for the PR findings. Source inspection of the live diff shows per-agent config becomes valid while history/status remain on global policy, and the repo tracks a generated config baseline that is absent from the PR files.

## Solution Assessment

No. Per-agent scoping is a reasonable direction, but this patch is not the best merge path until policy resolution is centralized, every session surface applies it, generated metadata is refreshed, and maintainers resolve the overlap with https://github.com/openclaw/openclaw/pull/39102, https://github.com/openclaw/openclaw/pull/53821, and https://github.com/openclaw/openclaw/issues/48785.
