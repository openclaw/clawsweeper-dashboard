## Summary

Keep open. The requested per-agent access-control capability is still not on current main, and this external PR is a plausible implementation attempt, but the branch is incomplete for a security-sensitive config surface and remains paired with the same author's open tracking issue.

This PR adds per-agent `agents.list[].tools.sessions` and `agents.list[].tools.agentToAgent` config, helper overrides, partial session-tool wiring, config metadata, and focused tests.

## Best Possible Solution

Define one fail-closed per-agent access-control contract in `src/plugin-sdk/session-visibility.ts`, covering schema/types/help/docs/baseline, requester identity, all four session tools, and the chosen outbound/inbound semantics.

## Solution Assessment

No. Per-agent scoping is a reasonable direction, but this patch is not the best merge path until enforcement is centralized on the current main helper, every session surface is covered, generated metadata is refreshed, proof is supplied, and maintainers choose final semantics.
