## Summary

Keep open. Per-agent session and A2A scoping is a useful security-sensitive feature, but this diff is not merge-ready because it accepts new per-agent access-control config while only wiring part of the runtime surface, misses the generated config baseline, lacks real behavior proof, and overlaps active narrower policy work.

## What This Changes

The PR adds per-agent `agents.list[].tools.sessions` and `agentToAgent` schema/types/help, updates requester-specific access logic for session list/send, and adds focused tests plus an unrelated session-path repro test.

## Best Possible Solution

Land or replace this with one requester-aware access-control contract in `src/plugin-sdk/session-visibility.ts` that covers schema/types/docs, all four session tools, generated baselines, tests, and aligned A2A semantics.

## Reproduction Assessment

Yes for the PR findings. Source inspection of the live diff shows per-agent config becomes valid while history/status remain on global policy, and the repo tracks a generated config baseline that the PR does not update.

## Solution Assessment

No. Per-agent scoping is a reasonable direction, but this patch is not the best merge path until it centralizes policy resolution, covers every session surface, refreshes generated metadata, and aligns with overlapping policy PRs.
