## Summary

Keep open. Current main and v2026.6.9 have session picker accessibility basics, timestamps, New Session, and Sessions-page label editing, but the chat picker/sidebar still do not request or render derived titles and do not expose rename controls in the requested VoiceOver-friendly history surface.

## What This Changes

The issue requests a VoiceOver-friendly, keyboard-reachable chat history selector with readable session titles, timestamps, active-session state, reopen/switch behavior, and accessible rename controls.

## Best Possible Solution

Keep this as the accessibility-focused session-history tracker; land or replace the derived-title UI PR, then add one canonical picker/sidebar rename and announcement path with focused keyboard and VoiceOver proof.

## Reproduction Assessment

Yes, at source level: current main exposes picker/search/switch/timestamp basics, but the picker does not request derived titles and does not expose rename controls. I did not run a live macOS VoiceOver pass.
