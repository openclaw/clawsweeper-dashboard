## Summary

Keep open. Current main and v2026.6.9 have keyboard-accessible session picker basics, timestamps, New Session, and Sessions-page label editing, but the VoiceOver-friendly history request is still incomplete because the chat picker/sidebar do not consume derived titles and do not expose a chat-history rename affordance.

## What This Changes

Request a keyboard- and VoiceOver-friendly conversation history selector with readable session titles, timestamps, active-state announcements, New Session, switching/reopening, and renaming controls.

## Best Possible Solution

Keep this as the accessibility-focused session-history tracker, land one canonical title/rename implementation path for the Control UI picker/sidebar, then require focused keyboard and VoiceOver proof for title, timestamp, active session, New Session, switch/reopen, and rename announcements.

## Reproduction Assessment

Yes at source level: current main has picker/search/switch/timestamp basics, but it does not request or render derived titles and does not expose rename controls in the chat picker/sidebar. I did not run a live VoiceOver pass.

## Solution Assessment

Partially. The existing sessions.create, sessions.patch, includeDerivedTitles, picker, and sidebar paths are the right implementation boundary, but the best fix needs one canonical accessible UI path rather than another parallel session API or isolated slice.
