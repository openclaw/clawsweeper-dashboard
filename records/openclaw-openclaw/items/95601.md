## Summary

Keep open. Current main has a keyboard-openable session picker, timestamps, active-state markup, New Session, and an existing Sessions-page label editor, but the requested VoiceOver-friendly conversation history remains incomplete because readable derived titles and picker/sidebar rename controls are still split across open PRs and lack combined VoiceOver proof.

## What This Changes

Request a VoiceOver-friendly Control UI conversation-history selector with keyboard navigation, readable session titles and timestamps, active-state announcements, New Session, switch/reopen, and rename controls.

## Best Possible Solution

Keep this as the accessibility-focused Control UI tracker, land or replace the derived-title and picker/sidebar rename candidates, then require keyboard and VoiceOver proof for the combined history workflow.

## Reproduction Assessment

Yes, at source level. Current main shows picker/search/switch/timestamp basics, but it does not request or render derived titles and does not expose rename controls in the chat picker/sidebar; no live VoiceOver run was performed.

## Solution Assessment

Partially. Reusing existing sessions.list includeDerivedTitles and sessions.patch label support is the right boundary, but maintainers still need to choose the final accessible picker/sidebar UX and require VoiceOver proof.
