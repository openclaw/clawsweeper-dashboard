## Summary

Keep open. Current main already has a keyboard-reachable session picker, timestamps, New Session, switch feedback, and Sessions-page label editing, but the VoiceOver-friendly history selector request is not complete because readable derived titles and in-chat/sidebar rename controls remain in open related work.

## What This Changes

Request an accessible Control UI conversation-history selector that lets VoiceOver users identify, reopen, switch, rename, and create sessions with announced titles and timestamps.

## Best Possible Solution

Keep this open as the accessibility-focused session-history UX tracker, land or replace the derived-title and picker/sidebar rename PRs, then require focused VoiceOver and keyboard proof for active session, title, timestamp, New Session, switch, rename, and reopen announcements.

## Reproduction Assessment

Yes at source level for the missing parts: current main has picker/search/switch/timestamp basics, but does not request or render derived titles and does not expose rename controls in the chat picker/sidebar. No live VoiceOver reproduction pass was run.

## Solution Assessment

Partially. The existing sessions.create, sessions.patch, includeDerivedTitles, picker, and sidebar paths are the right implementation boundary, but the best fix is one canonical accessible UI path rather than another parallel session API or duplicate PR.
