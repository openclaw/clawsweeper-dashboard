## Summary

Keep open. Current main and v2026.6.9 have good session-picker accessibility basics, timestamps, New Session, and Sessions-page label editing, but they still do not consume derived titles or expose rename controls in the chat picker/sidebar, so the VoiceOver-friendly history request is not complete.

## What This Changes

The issue requests a VoiceOver-friendly, keyboard-reachable conversation-history selector with readable session titles, timestamps, active-session announcement, and switch/rename/reopen controls.

## Best Possible Solution

Keep this as the accessibility-focused session-history tracker, land or replace the derived-title and picker/sidebar rename work, then require focused keyboard and VoiceOver proof for active session, title, timestamp, New Session, switch, rename, and reopen announcements.

## Reproduction Assessment

Yes at source level: current main has picker/search/switch/timestamp basics, but it does not request or render derived titles and does not expose rename controls in the chat picker/sidebar. I did not run a live VoiceOver pass.

## Solution Assessment

Partially. The existing sessions.create, sessions.patch, includeDerivedTitles, picker, and sidebar paths are the right implementation boundary, but the best fix is one canonical accessible UI path rather than another parallel session API or duplicate PR.
