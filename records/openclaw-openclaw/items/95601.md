## Summary

Keep open. Current main and v2026.6.9 still have only partial session-history accessibility support: the picker/sidebar expose switching, timestamps, and New Session basics, but the Control UI does not consume Gateway-derived titles and does not expose rename controls where users switch or reopen chat sessions.

## What This Changes

The issue requests a VoiceOver-friendly conversation-history selector with keyboard navigation, readable session titles and timestamps, active-session announcements, and controls for reopening, switching, and renaming sessions.

## Best Possible Solution

Keep this as the accessibility-focused session-history tracker, land or replace the derived-title and picker/sidebar rename slices in one coherent Control UI path, then require keyboard and VoiceOver proof for active session, title, timestamp, New Session, switch, rename, and reopen announcements.

## Reproduction Assessment

Yes at source level, but not with live VoiceOver proof. Current main exposes picker/listbox and timestamps, while derived readable titles and picker/sidebar rename controls remain incomplete.

## Solution Assessment

No, the issue is not solved yet. The right implementation boundary appears to be the existing sessions.list, includeDerivedTitles, sessions.patch, picker, and sidebar paths, but maintainers need to choose one canonical UI shape across overlapping title/rename/sidebar work.
