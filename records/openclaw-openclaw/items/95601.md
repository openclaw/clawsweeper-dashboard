## Summary

Keep open. Current main and v2026.6.9 provide a base Control UI session picker, New Session control, timestamps, active-state markup, and Sessions-page label editing, but the requested VoiceOver-friendly history experience remains incomplete because readable derived titles and picker/sidebar rename controls are still split across open work and lack combined VoiceOver proof.

## What This Changes

The issue requests a VoiceOver-friendly conversation-history selector with keyboard access, readable session titles, timestamps, active-session announcements, switching/reopening, and rename controls.

## Best Possible Solution

Keep this as the accessibility-focused Control UI tracker, land or replace the derived-title and picker/sidebar rename candidates, then require keyboard and VoiceOver proof for title, timestamp, active state, New Session, switch/reopen, and rename announcements.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.9 show picker/search/switch/timestamp basics, but they do not request or render derived titles and do not expose rename controls in the chat picker/sidebar; no live VoiceOver run was performed.

## Solution Assessment

Partially. The existing sessions.list, sessions.patch, picker, and Sessions-page label paths are the right implementation boundary, but maintainers still need to choose the final accessible picker/sidebar UX and require VoiceOver proof.
