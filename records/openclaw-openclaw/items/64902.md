## Summary

Keep open. Current main still exposes fallback failures through structured logs, lifecycle events, and trajectory records, but it does not persist the requested ordered session JSONL custom event before the `model-snapshot` marker; the related open harness PR is orthogonal to this transcript audit gap.

## What This Changes

The issue asks OpenClaw to persist a sanitized session JSONL custom event for a failed primary model attempt before a successful fallback's `model-snapshot` entry.

## Best Possible Solution

Add one bounded, sanitized fallback audit custom transcript event at the fallback/session boundary before `model-snapshot`, reusing existing fallback-step metadata and leaving routing, retry, cooldown, and user-visible notice policy unchanged.

## Reproduction Assessment

Yes, at source level: mock a primary candidate failure followed by fallback success, then inspect `runWithModelFallback` callbacks and `sessionManager.appendCustomEntry` ordering before `model-snapshot`. I did not run tests in this read-only review, but current source and existing fallback probe coverage make the gap clear.

## Solution Assessment

Yes, the narrow maintainable direction is to persist a sanitized transcript audit record from the existing fallback-step or failed-attempt metadata. A broader change to fallback selection, cooldown policy, or visible channel notices would solve a different problem.
