## Summary

Keep open. Current main has fallback attempts, structured subsystem logs, trajectory sidecars, lifecycle notices, and docs, but it still does not persist the requested ordered session JSONL audit event for the failed primary attempt before the fallback model snapshot.

## What This Changes

Request to persist a structured, sanitized session JSONL record for the failed primary model attempt that triggers a successful model fallback.

## Best Possible Solution

Persist one bounded, sanitized custom transcript event at the fallback/reply-runner boundary before the model snapshot or retry while keeping existing routing, logs, notices, and trajectory behavior intact.

## Reproduction Assessment

Yes. A focused reproduction can mock a primary candidate failure followed by fallback success, then inspect the session manager entries for an ordered sanitized custom event before `model-snapshot`; current source inspection shows that event is missing.

## Solution Assessment

Yes. The narrowest maintainable fix is to reuse existing fallback-step/attempt metadata to write a sanitized transcript audit record, not to change fallback eligibility, model routing, or user-visible notice policy.

When the selected/primary candidate fails and a later fallback candidate is attempted or succeeds, persist a custom JSONL entry before the fallback `model-snapshot`/model-selection transcript marker. Keep the change narrow: do not change fallback eligibility, model routing, auth cooldown policy, trajectory recording, lifecycle notice behavior, or provider-specific logic.
