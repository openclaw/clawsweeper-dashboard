## Summary

Keep open. Current main and the latest release emit structured fallback logs, lifecycle events, and trajectory records, but they still do not persist the requested ordered session JSONL custom audit event before the `model-snapshot` marker.

## What This Changes

The issue asks OpenClaw to persist the failed primary model attempt that triggered a successful fallback in the session JSONL audit trail.

## Best Possible Solution

Persist one sanitized, ordered fallback audit custom transcript event before `model-snapshot`, reusing existing fallback-step metadata while leaving routing, retry, cooldown, and user-visible notice policy unchanged.

## Reproduction Assessment

Yes, at source level. A primary failure followed by fallback success flows through `recordFailedCandidateAttempt`, `logModelFallbackDecision`, and `onFallbackStep`, while `model-snapshot` is appended later with no fallback custom-entry path.

## Solution Assessment

Yes. The maintainable fix is a narrow persisted audit record from existing sanitized fallback-step metadata, not a change to fallback selection, cooldown policy, or user-visible notices.
