Fix the fallback transcript audit gap from https://github.com/openclaw/openclaw/issues/64902. When a primary model attempt fails and a later fallback succeeds, persist one bounded sanitized session JSONL custom event before the existing `model-snapshot` entry. Reuse existing `fallbackStep*`/attempt metadata where possible; do not change fallback candidate selection, cooldowns, routing policy, user-visible notices, or trajectory behavior. Do not store raw provider bodies, credentials, request IDs, or unsanitized stack/error text in the transcript. Add focused regression coverage that mocks primary failure plus fallback success and asserts ordered transcript persistence before `model-snapshot`.
