## Summary

Keep open. Current main has structured fallback logs, lifecycle events, and trajectory records, but the requested ordered session JSONL custom event before the `model-snapshot` marker is still missing; the only open related PR is an orthogonal harness-routing fix.

## What This Changes

Request a persisted transcript audit record when a primary model attempt fails and a later fallback succeeds.

## Best Possible Solution

Add one bounded, sanitized fallback audit custom transcript event at the fallback/session boundary before `model-snapshot`, reusing existing `fallbackStep*` metadata and leaving routing, retry, cooldown, and user-visible notice policy unchanged.

## Reproduction Assessment

Yes, at source level: a primary failure followed by fallback success flows through `recordFailedCandidateAttempt`, `logModelFallbackDecision`, and `onFallbackStep`, while `model-snapshot` is appended later in replay-history with no fallback custom entry path. I did not run tests in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is to persist a sanitized transcript audit record from existing fallback-step or failed-attempt metadata; changing fallback selection, cooldown policy, or visible channel notices would solve different problems.
