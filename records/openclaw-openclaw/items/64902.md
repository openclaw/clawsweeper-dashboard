## Summary

Keep open. Current main and the latest release expose fallback failures through subsystem logs, lifecycle events, and trajectory sidecars, but they still do not persist the requested ordered session JSONL custom event before the `model-snapshot` marker.

## What This Changes

The issue asks OpenClaw to persist a structured session JSONL event for the primary-model failure that triggers a successful fallback.

## Best Possible Solution

Persist one bounded, sanitized custom transcript event at the fallback/session boundary before `model-snapshot`, reusing existing fallback-step metadata while leaving routing, retry, cooldown, lifecycle, and notice policy unchanged.

## Reproduction Assessment

## Solution Assessment
