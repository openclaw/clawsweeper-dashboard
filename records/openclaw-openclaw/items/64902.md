## Summary

Keep open. Current main has structured fallback logs, lifecycle events, trajectory sidecars, and fallback notice state, but it still does not persist the requested ordered session JSONL custom event for the failed primary attempt before the `model-snapshot` record.

## What This Changes

The issue asks OpenClaw to persist a structured, sanitized session transcript audit event when a primary model failure triggers a successful fallback.

## Best Possible Solution

Persist one bounded, sanitized custom transcript event at the fallback/session boundary before `model-snapshot`, while leaving fallback eligibility, model routing, lifecycle events, trajectory capture, and user-visible notice policy unchanged.

## Reproduction Assessment

Yes. A focused source-level reproduction can mock a primary candidate failure followed by fallback success, then inspect session manager entries for an ordered sanitized custom event before `model-snapshot`; current main emits logs/lifecycle/trajectory but no transcript custom event.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing fallback-step or failed-attempt metadata to write a sanitized transcript audit record, not to change fallback selection, retry policy, or user-visible fallback notices.
