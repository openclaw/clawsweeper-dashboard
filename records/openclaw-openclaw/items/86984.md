## Summary

Keep open: the report is recent, externally reported, and current main still has paths that pass inline or replayed image `data` straight into Anthropic/OpenAI image payloads without canonical base64 validation.

## What This Changes

The issue requests that inbound and replayed image attachments be guaranteed ASCII base64 before Anthropic/OpenAI provider payload assembly.

## Best Possible Solution

Add one canonical image-data normalization/validation boundary before provider payload assembly, then cover ACP inline, Anthropic replay, and OpenAI Responses replay with malformed-image regression tests.
