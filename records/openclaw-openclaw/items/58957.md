## Label Justifications

- `P1`: The report describes a broken long-running agent/channel workflow where model switches can silently strand or fail sessions for real users.
- `impact:session-state`: The failure depends on carried-over transcript/context size, compaction state, and whether an existing session can continue safely.

## Summary

Keep open. Current main has partial improvements around context metadata, status display, and downstream overflow text, but the central model-switch paths still apply or acknowledge a new model without comparing carried-over session size to the target model budget, and the fallback-on-overflow behavior remains an open product decision.

## What This Changes

The issue asks OpenClaw to clearly warn, recover, or compact when model switches or long-session continuations carry context that is too large for the selected model.

## Best Possible Solution

Keep this as the broader canonical thread for a shared model-switch context-risk warning across WebChat, messaging directives, and gateway session patching, while letting the open larger-fallback PR handle only that narrower subcase if maintainers accept it.

## Reproduction Assessment

## Solution Assessment

No. Current main is only a partial solution: context metadata and some downstream overflow handling improved, but there is no shared switch-time risk warning/recovery path and fallback-on-overflow remains an open product decision.
