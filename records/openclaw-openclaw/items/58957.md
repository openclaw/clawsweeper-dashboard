## Summary

Keep open. Current main has better overflow recovery than the original report, but the central long-session model-switch problem is still source-backed: model switch paths still persist and acknowledge the new model without a target-budget check, and token-budget preflight compaction still has early exits that can leave heartbeat or fresh-token sessions dependent on later overflow handling.

## What This Changes

The issue requests clearer handling and recovery when carried-over session context is too large during model switching or continuation.

## Best Possible Solution

Extend the existing context-budget and compaction paths so model switches and continuations warn or recover from oversized carried-over context, fresh token totals participate in token-budget preflight, and fallback routing preserves context-overflow cause information.

## Reproduction Assessment

Yes. Source inspection on current main shows deterministic missing paths: model switch entry points patch and acknowledge without a target-budget check, while preflight token compaction still returns early for heartbeat and fresh-token sessions.

## Solution Assessment

Yes for a narrow repair. The best path is to extend existing sessions.patch/directive/UI acknowledgements, preflight compaction, and overflow/fallback cause plumbing rather than introduce a broad new compaction mode or config knob.
