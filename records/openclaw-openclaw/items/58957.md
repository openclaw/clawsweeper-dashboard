## Summary

Keep open. Current main has useful downstream mitigations for context-overflow classification and non-heartbeat preflight compaction, but the central carried-context model-switch problem remains because `/model`, Control UI, and `sessions.patch` still persist and acknowledge model changes without a target-context budget decision or recovery policy.

## What This Changes

Request clearer, actionable recovery when a long session is switched or continued on a model whose effective context budget cannot safely carry the existing history.

## Best Possible Solution

Define one shared model-switch and continuation policy that estimates carried context against the selected model’s effective budget and consistently compacts, warns, or fails across chat directives, Control UI, sessions.patch, heartbeat, and runtime paths.

## Reproduction Assessment

Yes at source level: `/model`, Control UI, and `sessions.patch` persist and acknowledge the selected model without a carried-context budget decision; I did not run a live Telegram/UI repro.

## Solution Assessment

Unclear as an automated one-off fix: better downstream error copy exists, but the durable fix needs maintainer policy for compact versus warn versus fail versus fallback across runtime boundaries.
