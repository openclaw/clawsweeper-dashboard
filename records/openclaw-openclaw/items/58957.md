## Summary

Keep open. Current main still has source-backed gaps for the reported long-session model-switch failure: model switch entry points persist and acknowledge the new model without a target-budget warning, while preflight compaction can still skip heartbeat and fresh-token sessions before token threshold evaluation.

## What This Changes

The issue asks OpenClaw to clearly warn or recover when a long-running session carries too much context into a model switch or continuation.

## Best Possible Solution

Extend the existing model-switch, preflight compaction, and fallback-cause paths so oversized carried-over context is warned, compacted, or reported with a specific recovery hint, while coordinating the reserve-token formula with the open focused PR.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: switch-model entry points acknowledge without a target-budget check, while preflight compaction returns early for heartbeat and fresh-token sessions before token threshold evaluation.

## Solution Assessment

Yes for a narrow first pass, but automatic compaction/downgrade policy still needs maintainer judgment. The maintainable direction is to reuse sessions.patch/directive/UI acknowledgements, preflight compaction, and fallback-cause plumbing rather than add a separate mode or config knob first.
