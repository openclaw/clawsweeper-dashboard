## Summary

Keep this open: current main has better context-overflow copy and preflight machinery than the original report described, but the central failure class is still not fully resolved because fallback can still absorb a prompt-size failure, heartbeat/Codex-runtime preflight still skips the local proactive path, and related open PRs cover only sub-parts of the issue.

## What This Changes

The issue asks OpenClaw to make long-session model switches fail visibly and recoverably when carried-over context is too large for the selected model.

## Best Possible Solution

Keep this as the canonical model-switch/context-overflow bug until OpenClaw has a coordinated fix that preserves visible overflow recovery, avoids silent fallback stranding, and covers heartbeat, Codex-runtime, and normal embedded-runtime sessions with focused regression tests and docs updates.

## Reproduction Assessment

Yes for a source-level reproduction path, but not a live-provider reproduction in this review. Current main still has the heartbeat/Codex preflight skips and prompt-stage fallback decision that match the reported silent-fallback mechanism.

## Solution Assessment

No single small fix is clearly the best solution yet. The best path needs a maintainer decision across fallback signaling, proactive compaction thresholds, heartbeat behavior, and Codex-runtime ownership; open PRs currently cover only parts of that surface.
