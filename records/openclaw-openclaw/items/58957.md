## Summary

Keep open. Current main has partial mitigations for context metadata and post-error context-overflow copy, but it still does not guard a model switch or continuation by estimating carried session context against the selected model budget, and heartbeat/Codex paths still have policy-specific compaction skips that need maintainer direction.

## What This Changes

Requests clearer user-visible context-limit failures and safer recovery when switching or continuing long-running sessions whose carried context is too large for the selected model.

## Best Possible Solution

Define one model-switch and continuation policy that estimates carried session context against the selected model's effective budget, gives a specific recovery message, and consistently chooses compact, fail, or fallback across heartbeat, Codex, and provider-runtime paths.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current `/model` persistence does not perform a carried-context budget guard, and heartbeat preflight still returns before threshold checks. I did not run a live channel repro in this read-only review.

## Solution Assessment

Unclear as a single automatic fix: better error copy is necessary but not sufficient, because the durable solution needs a maintainer-owned policy for compaction, fallback, and warning behavior across model/provider/runtime boundaries.
