## Summary

Keep open. Current main has partial mitigations for context-overflow visibility and stale model metadata, but it still accepts a session `/model` switch without estimating carried session context against the target model or offering a compact/new-session recovery choice before the next turn.

## What This Changes

The issue asks OpenClaw to surface and recover clearly when a model switch or long-session continuation carries too much session context for the selected model.

## Best Possible Solution

Keep this issue as the canonical thread and implement a shared switch-time context-risk assessment plus maintainer-approved recovery policy across chat directives, Control UI/WebChat session patching, and embedded runtime overflow handling.

## Reproduction Assessment

Yes, source-level. Current main applies and acknowledges `/model` without a carried-context budget check, while recovery and compaction decisions happen later in runtime paths with explicit skips for some reported lanes.

## Solution Assessment

No, current main is only a partial solution. The best fix needs a shared context-risk warning/recovery design plus a maintainer decision on automatic compaction and fallback-on-overflow policy.
