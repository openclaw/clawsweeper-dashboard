## Label Justifications

- `P1`: The issue describes real long-running agent/channel sessions becoming unresponsive or silently moving to fallback behavior during model switches.
- `impact:session-state`: The failure depends on accumulated session context and can strand long-running session state without clear recovery guidance.
- `impact:auth-provider`: The affected path includes model selection, provider routing, and fallback behavior during live model switches.

## Summary

Keep open. Current main still has source-backed gaps for the reported oversized-context model-switch failure: model switch paths accept or acknowledge the new model without a target-budget warning, and preflight compaction can still return before threshold evaluation for heartbeat or fresh-token sessions.

## What This Changes

The issue asks OpenClaw to clearly surface and recover from oversized carried-over session context when switching models or continuing long-running sessions.

## Best Possible Solution

Add one shared model-switch context-risk path for Control UI, WebChat slash commands, and messaging directives, then coordinate it with compaction/fallback fixes so oversized carried-over context is compacted, refused with a recovery hint, or reported with the real cause.

## Reproduction Assessment

Yes, at source level. Switching models in a long session can still be accepted and acknowledged without a target-budget check, and current preflight compaction code still has heartbeat and fresh-token early returns before threshold evaluation.

## Solution Assessment

Unclear for the full requested automatic behavior because warning thresholds, auto-compaction, and fallback-cause surfacing need maintainer policy. The narrow maintainable direction is a shared model-switch risk check plus coordinated compaction/fallback fixes, not separate one-off messages.
