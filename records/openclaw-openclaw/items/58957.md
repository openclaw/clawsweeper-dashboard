## Summary

Keep open. Current main has useful partial mitigation for model context metadata and post-error context-overflow handling, but the central carried-context model-switch problem remains: switch paths still apply and acknowledge model changes without a target-budget recovery policy, while heartbeat and Codex preflight paths still skip OpenClaw threshold compaction before the later runtime guard can help.

## What This Changes

The issue requests clearer recovery and proactive handling when switching or continuing a long session whose carried context exceeds the selected model's effective budget.

## Best Possible Solution

Define one shared model-switch and continuation policy that estimates carried session context against the selected model's effective budget, gives a specific recovery message, and consistently chooses compact, fail, or fallback across messaging, UI, heartbeat, Codex, and provider-runtime paths.

## Reproduction Assessment

Yes. Source inspection shows current model-switch paths apply and acknowledge overrides without a carried-context budget guard, and preflight still skips heartbeat and Codex runtime paths before threshold checks; I did not run a live channel repro.

## Solution Assessment

Unclear as a single automatic fix. Better error copy is necessary but not sufficient, because the durable solution needs a maintainer-owned policy for compaction, fallback, warning, and recovery behavior across model/provider/runtime boundaries.
