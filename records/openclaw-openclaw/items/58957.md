## Summary

Keep open. Current main still has source-backed gaps for the reported long-session model-switch failure: model switch entry points persist or acknowledge the new model without a carried-context budget warning, while related compaction/fallback work remains split across open follow-up PRs and policy choices.

## What This Changes

The issue requests clearer handling when model switching carries too much session context, including specific overflow errors, compaction guidance or automation, and better session-size observability.

## Best Possible Solution

Add one shared model-switch context-risk path for Control UI, WebChat slash commands, and messaging directives, then coordinate it with the compaction/fallback fixes so oversized carried-over context is either compacted, refused with a clear recovery hint, or reported with the real cause.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: switch models in a long session whose carried context is above or near the target budget, and current main persists/acknowledges the switch without a target-budget warning while some preflight paths can return before token threshold evaluation.

## Solution Assessment

Unclear for the full requested automatic behavior because compaction-on-switch, warning thresholds, and fallback-cause surfacing need maintainer policy. The narrow maintainable direction is to reuse existing sessions.patch/directive/UI model-switch paths plus the compaction/fallback contracts instead of adding a separate mode first.
