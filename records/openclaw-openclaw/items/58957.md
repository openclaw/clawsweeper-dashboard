## Summary

Keep open. Current main has partial safeguards for context overflow, but the reported long-session model-switch/fallback failure is still credible: model-switch paths do not pre-check session size against the target model, and preflight compaction still has source-level early exits that can skip token-budget compaction for heartbeat or fresh-token sessions.

## What This Changes

The issue asks OpenClaw to make oversized carried-over session context during model switches/continuations visible and recoverable through clearer errors, compaction or warnings, and better context-size observability.

## Best Possible Solution

Implement a narrow runtime/UX fix that pre-checks or warns on model switches using current session token snapshots and target model context limits, evaluates fresh persisted totals in preflight compaction instead of skipping them, and preserves a context-overflow/compaction recovery cause when fallback routing is involved.

## Reproduction Assessment

Yes. The source-level reproduction is deterministic: a session with fresh persisted token counts or a heartbeat turn can bypass token-budget preflight compaction, while current model-switch entry points patch the model without checking session size; the discussion also provides multiple live confirmations.

## Solution Assessment

Unclear as a complete design, but the best fix is not a blanket auto-compact on every switch. The maintainable path is to extend existing context-budget, compaction, and fallback-cause plumbing with focused guards and user-visible recovery text.
