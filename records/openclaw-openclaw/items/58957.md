## Summary

Keep open. Current main has partial protections after a run starts, but the carried-context model-switch failure remains: model switches are still accepted and acknowledged before a target-budget recovery decision, and several runtime paths still depend on later overflow handling rather than a shared switch-time policy.

## What This Changes

The issue asks OpenClaw to handle oversized carried session context when a conversation is switched or continued on a smaller or different model, instead of failing silently or surfacing ambiguous overload/fallback messages.

## Best Possible Solution

Define one shared switch/continuation policy that estimates carried context against the selected model budget, then consistently warns, compacts, starts fresh, or blocks with a specific recovery message across chat directives, Control UI, sessions.patch, heartbeat, Codex runtime, and fallback paths.

## Reproduction Assessment

Yes, source-level: the switch paths persist and acknowledge model overrides without a target-budget check, while preflight and fallback handling only partially cover later overflow paths; no live Telegram/UI repro was run.

## Solution Assessment

No automatic one-file fix is clearly best yet; maintainers need to choose the shared compact/warn/fallback/start-fresh policy before implementing it across the switch and continuation boundaries.
