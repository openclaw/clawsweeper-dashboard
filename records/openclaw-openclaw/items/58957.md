## Label Justifications

- `impact:session-state`: The issue centers on carried session history, token accounting, compaction state, and continuity across model switches.

## Summary

Keep open. Current main has useful context metadata and context-overflow handling, but the core carried-context model-switch problem remains: switch paths still apply and acknowledge model changes without a target-budget recovery policy, and heartbeat/Codex/native paths leave compaction and fallback behavior split across runtimes.

## What This Changes

The issue asks OpenClaw to make long-session model switches or continuations clearly surface or recover from carried context that exceeds the selected model's effective context budget.

## Best Possible Solution

Define one shared model-switch and continuation policy that compares carried context with the selected model's effective budget and consistently warns, compacts, fails, or falls back across messaging, Control UI, heartbeat, and Codex/native paths.

## Reproduction Assessment

Yes at source level, not live-reproduced: current switch paths persist and acknowledge overrides without a carried-context budget comparison, and related v2026.6.10/user reports show overflow or fallback symptoms after model switches.

## Solution Assessment

Unclear as a single automatic repair; durable resolution needs maintainer-owned policy for warning, compaction, fallback, and recovery behavior across model/provider/runtime boundaries.
