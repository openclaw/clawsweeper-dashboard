## Summary

Keep open. Current main still matches the reported delivery-default behavior, the SQLite migration is doctor-owned rather than automatic at Gateway runtime, and the open migration-only report does not cover the delivery default or delivery-failure precedence raised here.

## What This Changes

The issue reports an upgrade regression where legacy cron jobs are not imported into the SQLite-backed store and newly added isolated cron jobs default to announce delivery, causing channel-required run failures.

## Best Possible Solution

Keep this issue open as the combined upgrade and delivery-default report, route the JSON-to-SQLite import work through the existing doctor/migration design, and make an explicit maintainer decision on announce-by-default versus no-delivery-by-default for channel-less installs.

## Reproduction Assessment

Yes from source inspection: current main reads cron jobs from SQLite, leaves legacy JSON import to doctor, defaults isolated cron add jobs to announce, and returns a run error for unresolved non-best-effort delivery. The exact npm upgrade was not executed in this read-only review.

## Solution Assessment

No, the current documented doctor path and announce default do not fully solve the reported upgrade experience. The maintainable fix should preserve the database-first runtime boundary while making legacy import and channel-less delivery behavior explicit and safe.
