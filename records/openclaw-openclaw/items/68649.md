## Summary

Keep open. Current main still has the PDF-specific stalled remote-fetch gap: remote PDF loads reach the shared web-media loader without an idle-read timeout, while the lower fetch layer only aborts stalled body reads when callers supply `readIdleTimeoutMs`; the focused fix PR is closed unmerged, and the broader stuck-session recovery work remains separately tracked.

## What This Changes

The issue asks for remote `pdf` tool calls to time out instead of wedging the agent session, with broader recovery for `/new`, `/restart`, and stuck-session watchdog behavior.

## Best Possible Solution

Add optional web-media idle-read timeout plumbing, set a conservative default for remote PDF loads, and cover the PDF call seam with regression tests while leaving generic stuck-session recovery to https://github.com/openclaw/openclaw/issues/71127.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a remote PDF URL reaches `loadWebMediaRaw` without an idle-read timeout, and the lower fetch/reader stack only aborts stalled body reads when `readIdleTimeoutMs` is supplied.

## Solution Assessment

Yes for the PDF-specific defect. The narrow maintainable fix is to plumb an idle-read timeout through `WebMediaOptions` and the PDF tool; scheduler interruption and watchdog recovery should stay in the canonical stuck-session issue.

## Repair Work Prompt

Fix the PDF-specific stalled remote-fetch bug from https://github.com/openclaw/openclaw/issues/68649. Add an optional `readIdleTimeoutMs` field to the shared web-media options, forward it from `src/media/web-media.ts` into `fetchRemoteMedia`, and make `src/agents/tools/pdf-tool.ts` pass a conservative positive idle timeout for remote PDF loads so stalled HTTP body reads abort and return a tool error instead of wedging the session. Add focused regression coverage proving remote PDF execution forwards the timeout and web-media forwards it to `fetchRemoteMedia`; keep SSRF policy, maxBytes, sandbox/local path handling, provider calls, and tool schema unchanged. Do not implement `/new`, `/restart`, generic watchdog, model-call, or session-recovery policy in this PR; those belong to https://github.com/openclaw/openclaw/issues/71127. Add a user-facing changelog fix entry if repo policy still requires it for this behavior change.
