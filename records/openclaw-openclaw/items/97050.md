## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Remove npmrc from the runtime COPY:** `Dockerfile:190`
  - body: This new COPY mixes `/app/...` sources with a bare `.npmrc`. Docker resolves `COPY --from` sources from the source stage filesystem, so this can fail by looking for the wrong path; changing it to `/app/.npmrc` would publish an auth-only config file. Keep `.npmrc` out of the runtime image.
  - confidence: 0.94
- **[P1] Restore the final runtime stage:** `Dockerfile:163-167`
  - body: Dropping `FROM base-runtime` makes the named base stage become the full runtime image. Existing tests still use that marker to bound final-image assertions, and CI is failing on those assertions, so preserve the separate final stage unless maintainers approve a contract change with matching tests/docs.
  - confidence: 0.92
- **[P2] Align the browser install contract test:** `Dockerfile:238-250`
  - body: The consolidated install path no longer contains the literal `apt-get install -y --no-install-recommends xvfb` string that the Dockerfile test asserts, and CI is failing at that assertion. Either restore the tested install command shape or update the test to prove the consolidated install path still installs `xvfb` when browser support is enabled.
  - confidence: 0.86
- **[P3] Keep provider owner tuples typed:** `src/plugins/providers.test.ts:204-222`
  - body: Replacing the owner tuple construction with `as any` erases the shape that these provider ownership tests protect. Use const tuples, `satisfies`, or a typed helper instead of weakening the fixture contract.
  - confidence: 0.78
