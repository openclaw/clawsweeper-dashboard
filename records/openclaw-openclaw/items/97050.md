Full review comments:

- **[P1] Remove npmrc from the runtime COPY:** `Dockerfile:190`
  - body: This new COPY mixes `/app/...` sources with a bare `.npmrc`. Docker resolves `COPY --from` sources from the source stage root, while runtime-assets has `/app/.npmrc`; the build can fail here, and changing it to `/app/.npmrc` would publish an auth-only config file. Keep `.npmrc` out of the runtime image.
  - confidence: 0.94
- **[P1] Restore the final runtime stage:** `Dockerfile:164-167`
  - body: Dropping `FROM base-runtime` makes the named base stage become the full runtime image. The branch's own Dockerfile test still expects `FROM base-runtime`, and users or automation targeting `base-runtime` would get different contents, so preserve the stage unless maintainers approve a contract change with tests/docs.
  - confidence: 0.9

  - body: Replacing the owner tuple construction with `as any` erases the shape that these provider ownership tests protect. Use const tuples, `satisfies`, or a typed helper instead of weakening the fixture contract.
  - confidence: 0.78
