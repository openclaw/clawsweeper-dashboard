Full review comments:

- **[P1] Do not copy npmrc into the runtime image:** `Dockerfile:190`
  - body: This COPY adds `.npmrc` to the final runtime image. The repo keeps npmrc for registry/auth-only entries, and `.dockerignore` does not exclude it, so credentialed builds can bake npm configuration or tokens into `/app/.npmrc`; copy only the package metadata needed at runtime.
  - confidence: 0.96
- **[P2] Restore the final runtime stage boundary:** `Dockerfile:164-167`
  - body: The Stage 3 section no longer starts with `FROM base-runtime`, so runtime image instructions are folded into the `base-runtime` stage itself. Current Dockerfile tests and downstream release assumptions depend on the separate final runtime boundary; keep the layer consolidation inside that final stage instead.
  - confidence: 0.95
- **[P2] Reject stale snapshots before rotating backups:** `src/config/io.ts:2597-2603`
  - body: The pre-backup snapshot guard was removed, so a config changed during preflight reaches `maintainConfigBackups` before the write rejects. Current tests require no backup rotation in that stale-snapshot path; keep a guard before backup rotation as well as after it.
  - confidence: 0.94
- **[P2] Test through a public config I/O surface:** `src/config/io.async-race-guard.test.ts:5`
  - body: The new test imports `assertBaseSnapshotStillCurrent`, but that helper remains file-local in `io.ts`, so the test cannot typecheck or prove supported behavior as written. Exercise the public write path or intentionally expose a supported test seam instead.
  - confidence: 0.97
