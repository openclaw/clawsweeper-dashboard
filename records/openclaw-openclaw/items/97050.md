Full review comments:

- **[P1] Remove npmrc from the runtime COPY:** `Dockerfile:190`
  - body: Docker resolves `COPY --from` sources from the source-stage filesystem root, so the bare `.npmrc` source points outside `/app` and can fail the image build.   - confidence: 0.94
- **[P1] Restore the final runtime stage:** `Dockerfile:163`
  - body: Dropping the separate `FROM base-runtime` instruction turns the named base stage into the final runtime image and changes a tested target boundary. Preserve the final runtime stage unless maintainers explicitly approve the compatibility change with matching tests, docs, and proof.
  - confidence: 0.91
- **[P2] Align the browser install assertion:** `src/dockerfile.test.ts:97`
  - body: The test still expects a literal `apt-get install -y --no-install-recommends xvfb`, but the Dockerfile now appends `xvfb` to `install_list` and installs `$install_list`. Update the assertion to prove the consolidated path still installs browser dependencies, or restore the command shape.
  - confidence: 0.87

  - body: Replacing the owner tuple construction with `as any` erases the shape this provider ownership fixture is supposed to protect. Use const tuples, `satisfies`, or a typed helper instead of weakening the contract test.
