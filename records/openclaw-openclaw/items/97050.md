## Summary

Keep open: the branch is not mergeable because it breaks Docker runtime build ordering, breaks the browser-image build path, removes a protected config write race guard, and lacks inspectable real behavior proof.

## What This Changes

The PR consolidates optional Docker runtime install steps, moves runtime asset copies after optional install/Corepack setup, and makes config base-snapshot checking asynchronous while removing the post-backup recheck.

## Best Possible Solution

Rework the Docker stage so file-dependent commands run after their required COPY steps, preserve the config post-backup snapshot check, and add focused proof for default Docker, browser Docker, and the config backup-rotation race.

## Reproduction Assessment

Yes. Source inspection shows the PR uses package.json and node_modules before copying them, and current tests show the removed config recheck is required to reject a backup-rotation race.

## Solution Assessment

No. The performance direction is plausible, but the current implementation is not the best fix because it changes required Docker build ordering and removes a tested concurrency guard instead of making only the read path asynchronous.

- **[P1] Keep Corepack after package.json is copied:** `Dockerfile:296`
  - body: This RUN now executes before the runtime stage copies package.json, but it still evaluates `require('./package.json').packageManager`.   - confidence: 0.98
- **[P1] Install Playwright only after node_modules is copied:** `Dockerfile:285`
  - body: With `OPENCLAW_INSTALL_BROWSER=1`, this line calls the Playwright CLI before runtime node_modules is copied. The current Dockerfile explicitly keeps this after node_modules is present, and release builds use the browser arg, so browser images will fail to build.
  - confidence: 0.97
- **[P1] Preserve the post-backup snapshot check:** `src/config/io.ts:2601-2603`
  - body: The removed second snapshot assertion after `maintainConfigBackups` is what rejects a concurrent edit made during backup rotation. With only the pre-backup check, that edit can be overwritten by the following atomic rename; keep a post-backup recheck, async or not.
  - confidence: 0.96
