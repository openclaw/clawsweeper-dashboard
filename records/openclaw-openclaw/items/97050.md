## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Copy package.json before preparing pnpm:** `Dockerfile:296`
  - body: This `RUN` still evaluates `require('./package.json').packageManager`, but the PR moved the runtime-asset `COPY` lines, including `package.json`, below it. A normal runtime image build reaches this command before `/app/package.json` exists, so the build fails instead of producing the optimized image.
  - confidence: 0.97
- **[P1] Keep browser install after node_modules is copied:** `Dockerfile:285`
  - body: The browser install still invokes `/app/node_modules/playwright-core/cli.js`, but the PR now runs it before `COPY --from=runtime-assets /app/node_modules ./node_modules`. Builds with `OPENCLAW_INSTALL_BROWSER=1` will fail because the Playwright CLI is not present yet.
  - confidence: 0.96
- **[P1] Preserve the post-backup snapshot guard:** `src/config/io.ts:2601`
  - body: The removed second snapshot check is not redundant: `maintainConfigBackups` runs between the two checks, and the existing test mutates the config during that hook to prove the write must reject. Without the post-backup check, the atomic replace can overwrite a concurrent config edit.
  - confidence: 0.95
- **[P2] Compute Docker apt architecture inside RUN:** `Dockerfile:167`
  - body: `ARG OPENCLAW_APT_ARCH="$(dpkg --print-architecture ...)"` gives Docker a literal default string, not a shell-evaluated architecture. When the Docker CLI option writes that value into the apt source line, builds can get an invalid architecture instead of `amd64`, `arm64`, or the platform value.
  - confidence: 0.86
