- **[P1] Remove npmrc from the runtime COPY:** `Dockerfile:190`
  - body: This new runtime COPY adds a bare `.npmrc` source. `COPY --from` resolves sources from the source-stage root, so this can fail the image build; changing it to `/app/.npmrc` would instead publish registry/auth configuration in the runtime image. Keep `.npmrc` out of the final runtime stage.
  - confidence: 0.94
- **[P1] Restore the final runtime stage:** `Dockerfile:163-167`
  - body: Dropping `FROM base-runtime` makes the named base-runtime target accumulate the full runtime image. Current tests and downstream users that build the named target expect the base boundary, so preserve the separate final stage unless maintainers approve a contract change with docs and tests.
  - confidence: 0.9
- **[P2] Keep the browser install test aligned:** `src/dockerfile.test.ts:97`
  - body: The branch no longer contains an explicit `apt-get install ... xvfb` command; it adds `xvfb` to `install_list` and installs `$install_list`. This changed assertion cannot pass against the rewritten Dockerfile, so either preserve the explicit install or assert the consolidated install-list path.
  - confidence: 0.86
- **[P3] Keep provider owner tuples typed:** `src/plugins/providers.test.ts:204-222`
  - body: Replacing the owner tuple construction with `as any` erases the shape these provider ownership tests are meant to protect. Use const tuples, `satisfies`, or a typed helper instead of weakening the fixture contract.
  - confidence: 0.78
