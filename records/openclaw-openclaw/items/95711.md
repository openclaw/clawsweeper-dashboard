## Summary

## What This Changes

The PR changes Feishu streaming-card final handling to keep accumulated stream text when the final payload is contained in it, and adds a regression test for a final payload that repeats only the last streamed block.

## Best Possible Solution

Land a Feishu-local final-retention fix that preserves accumulated text only for an actual last-block/suffix or source-tracked block case, with contained-but-not-suffix regression coverage and redacted live Feishu proof or a maintainer proof override.

## Reproduction Assessment

Yes, at source level. Current main appends block deliveries into streamText but then replaces the streaming-card buffer with the final payload, so a final carrying only the last block can collapse the visible card; I did not run a live Feishu tenant session.

## Solution Assessment

No as submitted. The Feishu dispatcher is the right owner boundary, but the broad includes check should be narrowed to a suffix or tracked-last-block case and still needs real Feishu proof or an explicit maintainer proof override.
