## What This Changes

The PR changes Feishu streaming-card final delivery to keep accumulated streamed text when the final payload is contained in it, and adds a regression test for a final payload that repeats only the last streamed block.

## Best Possible Solution

Land a Feishu-local final-retention fix that preserves accumulated text only for a safe last-block or source-tracked suffix case, with contained-but-not-suffix coverage and redacted Feishu/Lark proof or an explicit maintainer proof override.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.10 append block deliveries into streamText but then replace the streaming-card buffer with the final payload, so a final carrying only the last block can collapse the visible card; I did not run a live Feishu tenant session.

## Solution Assessment

No as submitted. The Feishu dispatcher is the right owner boundary, but the broad includes check should be narrowed to a suffix or source-tracked last-block case, and the PR still needs real Feishu proof or an explicit maintainer proof override.
