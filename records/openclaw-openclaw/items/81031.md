## Summary

Keep open. Current main still has a narrow Feishu interactive-card text parser that can fall back to `[Interactive Card]`; the mention path already propagates `WasMentioned`, but the Feishu body helper does not make the direct bot mention explicit in the agent-facing body.

## What This Changes

The issue reports two Feishu channel bugs: interactive cards read as a placeholder and direct bot mentions in mention-required group chats are not made explicit enough to the agent.

## Best Possible Solution

Extend Feishu card text extraction with representative regression coverage, and add a trusted explicit-bot-mention signal or hint for Feishu agent context without weakening mention gating or reintroducing stripped mention text.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows the parser fallback path and body-helper omission, but the exact cron card payload and observed no-reply behavior still need targeted verification.

## Solution Assessment

Unclear until a concrete card payload is covered, but a narrow parser-plus-context fix is the maintainable direction. Exposing raw card JSON should be a fallback because it expands the public Feishu message info surface.
