## Best Possible Solution

Land a refreshed Feishu-local cleanup fix that returns the shutdown Promise through `stopFeishuMonitor()`, adds focused HTTP close regression coverage, and includes redacted real Feishu restart or port-release proof; keep the broader resource lifecycle work in https://github.com/openclaw/openclaw/issues/48793 separate.
