## Summary

Keep open: this PR targets a real source-reproducible Feishu HTTP cleanup race that is still present on current main, but the branch still drops the new async shutdown promise at the public stop wrapper and lacks after-fix real behavior proof.

## What This Changes

The PR adds an awaited Feishu HTTP server close helper and uses it in monitor state and webhook abort cleanup before clearing Feishu server identity maps.

## Best Possible Solution

Land a refreshed Feishu-local async shutdown fix that returns the stop promise through the public wrapper, updates affected tests/hooks to await cleanup, and includes redacted real Feishu restart or port-release proof; keep the broader ResourceManager proposal separate.

## Reproduction Assessment

Yes, at source level: current main still closes Feishu HTTP servers and clears state before the close callback in both stop cleanup and webhook abort cleanup. A live restart loop would measure the external EADDRINUSE or memory-growth symptom.

## Solution Assessment

Yes, the narrow Feishu-owned awaitable HTTP close helper is the right immediate direction, but this branch is incomplete until the public stop wrapper returns the Promise and real after-fix behavior is shown.
