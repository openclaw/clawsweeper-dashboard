## Summary

Keep open: the PR is aimed at a real Feishu HTTP shutdown race and covers the right state and webhook cleanup paths, but the public stop wrapper still drops the new Promise and the external PR has no after-fix real behavior proof.

## What This Changes

The branch adds an awaited Feishu HTTP server close helper with a timeout and reuses it during monitor state and webhook abort cleanup.

## Best Possible Solution

Keep this PR open and require the narrow async propagation fix plus real shutdown/restart proof before landing or replacing it.

## Reproduction Assessment

Yes, at source level: current main closes Feishu HTTP servers and clears their state before the close callback in both stop and webhook-abort cleanup paths. I did not run a live restart loop in this read-only review.

## Solution Assessment

No, not yet. Awaiting server close in the state and transport paths is the right direction, but the public stop wrapper must return the Promise so callers and tests can actually wait for port release.
