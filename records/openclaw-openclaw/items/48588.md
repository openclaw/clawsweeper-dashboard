## Summary

Keep open: this PR targets a real Feishu cleanup bug that is still present on current main and the latest release, but the branch is incomplete because the public stop wrapper still drops the new async shutdown Promise and the PR lacks after-fix real behavior proof.

## What This Changes

The PR adds a Feishu HTTP server close helper and awaits it from monitor state and webhook abort cleanup before clearing Feishu server maps.

## Best Possible Solution

Land a refreshed Feishu-local async shutdown fix that returns the stop Promise through the public wrapper, updates affected tests/hooks to await cleanup, keeps the Node-recommended close ordering with a bounded timeout, and leaves the broader ResourceManager design to https://github.com/openclaw/openclaw/issues/48793.

## Reproduction Assessment

Yes, at source level: current main still closes Feishu HTTP servers and clears state before the close callback in both stop cleanup and webhook abort cleanup. A live restart loop would be needed to measure the external EADDRINUSE or memory-growth symptom.

## Solution Assessment

No, not as-is: the awaited close helper is the right narrow direction, but the public stop wrapper still must return the Promise and real after-fix behavior proof is missing.
