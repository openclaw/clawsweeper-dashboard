## Summary

Keep open: current main and the latest release still clear Feishu HTTP server state before HTTP close completion, and this PR targets that bug, but it still needs real behavior proof plus a small branch update so callers can await shutdown through the public stop wrapper.

## What This Changes

The PR adds a Feishu HTTP server close helper and awaits it from monitor-state and webhook abort cleanup before clearing state references.

## Best Possible Solution

Land a refreshed Feishu-owned async shutdown fix that returns the stop promise through the public wrapper, awaits affected cleanup hooks/tests, and keeps the broader ResourceManager design as separate follow-up work.

## Reproduction Assessment

Yes, at source level: current main still calls `server.close()` and immediately clears Feishu HTTP server state in both stop and webhook-abort cleanup paths. A live repro would start a Feishu webhook monitor, abort or stop it, and immediately restart on the same port while observing port-release behavior.

## Solution Assessment

Yes, the Feishu-local awaitable close helper is the narrow maintainable direction, but the current branch is not complete because the public stop wrapper still drops the shutdown promise. Real after-fix restart proof is also required before merge.
