## Best Possible Solution

Land a refreshed Feishu-local cleanup fix that returns the shutdown Promise through stopFeishuMonitor(), adds focused HTTP close sequencing coverage, includes redacted real restart or port-release proof, and keeps broader resource-lifecycle architecture in the separate design issue.

## Reproduction Assessment

## Solution Assessment

No, not as-is: awaiting bounded HTTP shutdown is the right narrow fix shape, but the public stop wrapper still drops the async cleanup Promise and the PR lacks after-fix real behavior proof.
