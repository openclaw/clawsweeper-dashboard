## Summary

Keep open: this is a useful Feishu cleanup fix for a source-reproducible shutdown race, but it is not merge-ready because the public stop helper still drops the new async shutdown promise and the PR has no after-fix real behavior proof.

## What This Changes

The PR adds a Feishu HTTP server close helper and awaits server shutdown before clearing state in monitor state and webhook abort cleanup.

## Best Possible Solution

Land or replace this with a narrow Feishu cleanup fix that awaits and bounds HTTP server shutdown in both state and webhook abort paths, exposes an awaitable stop helper, and keeps the broader ResourceManager design in its separate follow-up.

## Reproduction Assessment

Yes, at source level: current main calls server.close() and clears Feishu HTTP server state before the close callback in both stop and webhook-abort cleanup paths. A live restart loop would only be needed to measure the external memory or EADDRINUSE symptom.

## Solution Assessment

No, not yet. The awaited close helper is the right narrow direction, but the public stop wrapper and cleanup tests need to be able to await the new async shutdown, and the external PR still needs real behavior proof.
