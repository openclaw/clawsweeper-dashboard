## Summary

## What This Changes

The PR adds a Feishu HTTP server close helper and uses it in monitor state and webhook abort cleanup so server shutdown is awaited before clearing per-account state.

## Best Possible Solution

Land a refreshed Feishu-local cleanup fix that returns and awaits shutdown through `stopFeishuMonitor()`, adds focused HTTP close regression coverage, and includes redacted real Feishu restart or port-release proof; keep the broader resource lifecycle proposal in https://github.com/openclaw/openclaw/issues/48793 separate.

## Solution Assessment
