- **[P2] Build one service-config note instead of moving it:** `src/commands/doctor-gateway-services.ts:470-479`
  - body: The moved `entrypointSourceCheckout` note still fires whenever the layout flag is true, so a run that also appends `gatewayEntrypointMismatch` still emits this standalone panel and then the audit panel. Because the note now sits after the empty-audit return, the source-checkout-only warning path is also dropped; build one message from the layout/audit facts before emitting.
  - confidence: 0.95
- **[P2] Keep injected messages transcript-only:** `src/gateway/server-methods/chat-transcript-inject.ts:90`
  - body: Changing the stored `provider` to `originAgent` breaks consumers that suppress transcript-only rows by checking `provider === "openclaw"` with `model === "gateway-injected"`. A `chat.inject` call with `originAgent` now writes a gateway-injected row that can be replayed or handled as live model output; keep the sentinel metadata stable or update all filters.
  - confidence: 0.9
- **[P2] Accept the verbosity values OpenClaw emits:** `src/gateway/open-responses.schema.ts:230`
  - body: This schema admits only `auto` and `verbose`, but OpenClaw's Responses payload builder and tests use `text.verbosity` values `low`, `medium`, and `high`. Requests containing the supported local payload shape would still be rejected by the strict parser.
  - confidence: 0.88
