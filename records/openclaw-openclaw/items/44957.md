## What This Changes

The PR adds an opt-in cron `outputHistory` payload flag that records recent isolated-agent outputs in job state and injects them into later isolated cron prompts.

## Best Possible Solution

Land a rebased, maintainer-approved cron memory implementation that filters non-substantive replies, normalizes the new flag consistently, documents the opt-in stateful contract, and includes redacted real cron-run proof.

## Reproduction Assessment

Yes for the PR review blockers: the control-token persistence path and malformed `outputHistory` normalization are source-reproducible from PR-head code plus current NO_REPLY suppression behavior. The underlying request is a new opt-in capability rather than a current-main bug reproduction.

## Solution Assessment

No. The direction is plausible, but the current patch is not the best mergeable solution until it filters control replies, normalizes malformed values, gets maintainer approval for the opt-in memory contract, rebases, and provides real behavior proof.
