## Summary

Keep open: current main still does not have the proposed cron output-history capability, and the PR head has source-reproducible correctness blockers plus missing real behavior proof before it can be considered for merge.

## What This Changes

Adds a cron `outputHistory` agent-turn flag, persists bounded `recentOutputs`, injects prior outputs into isolated-agent prompts, and updates cron tool, normalization, gateway schema, and tests.

## Best Possible Solution

Land a rebased version only if it stores substantive prior outputs, normalizes the new flag consistently with adjacent fields, preserves current cron contracts, and includes redacted real cron-run proof.

## Reproduction Assessment

Yes for the review findings by source inspection: PR head records any successful `outputText`, and PR head can return ok with silent/control output such as `NO_REPLY`. I did not run tests because this review is required to keep the checkout read-only.

## Solution Assessment

No. The feature direction is plausible, but this patch should filter non-substantive control output, normalize malformed `outputHistory` inputs, rebase onto current cron contracts, and show real behavior proof before merge.
