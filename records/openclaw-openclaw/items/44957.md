## Summary

Keep open. Current main does not implement this cron output-history contract, and the PR still has source-reproducible correctness blockers plus missing real cron-run proof before it can be considered for merge.

## What This Changes

The PR adds an opt-in cron `outputHistory` payload field that records recent isolated-agent outputs in cron state and injects them into later isolated-agent prompts, with runtime, tool, protocol, normalization, and test updates.

## Best Possible Solution

If maintainers accept bounded prior-output memory for isolated cron, land a rebased implementation that filters non-substantive control replies, normalizes the new flag consistently, updates public docs/protocol/tests, and includes real cron-run proof.

## Reproduction Assessment

Yes for the PR review blockers: the timer recording gate, silent-token delivery path, and normalization behavior are source-reproducible from PR head. Reproduction is not applicable to the underlying feature request because current docs define isolated cron as fresh-session behavior.

## Solution Assessment

No. The direction is plausible, but this patch is not yet the best maintainable solution because it needs product approval, control-token filtering, malformed-value cleanup, rebase/CI work, docs alignment, and real behavior proof.
