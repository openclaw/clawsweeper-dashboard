## Summary

Keep open. This is still a live feature PR: current main does not implement output-history cron prompts, but the PR still has source-backed correctness issues and no real behavior proof from a real cron run.

## What This Changes

The PR adds an `outputHistory` cron agent-turn flag, records recent outputs in job state, injects them into isolated-agent prompts, and updates cron schemas, tools, and tests.

## Best Possible Solution

Land a rebased version that stores only substantive prior outputs, keeps malformed cron payloads recoverable, preserves the protocol/tool contract, and includes redacted real setup proof.

## Reproduction Assessment

Yes for the review findings from source inspection: the PR records any successful `outputText`, while cron can return ok with `NO_REPLY`/heartbeat-style control output. I did not run tests because this review must keep the checkout read-only.

## Solution Assessment

No. The feature direction is plausible, but this patch should filter non-substantive control output, normalize malformed `outputHistory` values, and then show real cron behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Filter control-token replies before recording history:** `src/cron/service/timer.ts:347-351`
  - body: This appends any successful `outputText` to `recentOutputs`, but cron can return `status: "ok"` with silent/control replies like `NO_REPLY` or heartbeat acknowledgements still in `outputText`. Those tokens would be persisted and injected into the next prompt as if they were substantive previous output.
  - confidence: 0.88
- **[P3] Drop malformed outputHistory values during normalization:** `src/cron/normalize.ts:105`
  - body: Adding `outputHistory` as an agent-turn hint leaves malformed values like `outputHistory: "true"` in the payload, unlike the adjacent `allowUnsafeExternalContent` cleanup. The normalized request then fails schema validation instead of recovering from a common stringified-boolean shape.
  - confidence: 0.74
