## Summary

Keep open: current main has not implemented this feature, and the PR still has source-reproducible correctness issues plus missing real behavior proof, so it is useful but not ready to merge or stale-close.

## What This Changes

Adds a cron `outputHistory` agent-turn flag that stores bounded recent outputs in cron job state and injects them into later isolated cron prompts, with tool, normalization, gateway schema, and test updates.

## Best Possible Solution

Rebase the feature onto current cron contracts, record only substantive prior outputs, normalize the new flag consistently with adjacent payload fields, and add redacted real-run proof before maintainer review/merge.

## Reproduction Assessment

Yes. The review findings are source-reproducible: PR head records every successful `outputText` at `src/cron/service/timer.ts:347-351`, while delivery can return `ok` with `NO_REPLY` still preserved at `src/cron/isolated-agent/delivery-dispatch.ts:540-547`; no tests were run because this review must keep the checkout read-only.

## Solution Assessment

No. The feature direction is plausible, but this patch should filter non-substantive control output, normalize malformed `outputHistory` inputs, rebase onto current cron contracts, and show real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Filter control-token replies before recording history:** `src/cron/service/timer.ts:347-351`
  - body: This appends any successful `outputText` to history, but the delivery path can return `status: "ok"` while preserving silent/control text such as `NO_REPLY` or `HEARTBEAT_OK`. Those tokens would be persisted and injected into later prompts as if they were substantive prior output.
  - confidence: 0.88

  - body: The new hint recognizes boolean `outputHistory`, but malformed values such as `"true"` are not dropped or coerced before schema validation, unlike `allowUnsafeExternalContent`. Flattened or model-generated requests can fail instead of recovering from a common stringified boolean shape.
  - confidence: 0.74
