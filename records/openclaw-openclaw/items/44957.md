## Summary

Keep this PR open, but it is not merge-ready: current main does not implement cron output history, while the PR is conflicting, lacks real behavior proof, and still has source-backed blockers around recording control-only replies and malformed `outputHistory` normalization.

## What This Changes

This PR adds an opt-in cron `outputHistory` flag that stores recent isolated-agent outputs in cron job state and injects them into later isolated cron prompts, with gateway/tool schema updates and tests.

## Best Possible Solution

Land a rebased, maintainer-approved opt-in cron memory mode that filters non-substantive outputs, normalizes the new flag consistently, and includes redacted real cron-run proof for record-and-inject behavior.

## Reproduction Assessment

Yes for the PR review blockers: PR-head source inspection shows successful `outputText` is persisted without filtering control-only replies, and non-boolean `outputHistory` is not normalized before strict schema validation. The underlying request is a new opt-in capability, not a current-main bug reproduction.

## Solution Assessment

Unclear as proposed: opt-in bounded history is a plausible solution, but the branch needs current-main rebase work, filtering/normalization fixes, and explicit maintainer approval for stateful isolated cron prompts before it is the safest implementation path.
