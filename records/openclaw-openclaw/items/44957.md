## Summary

Keep open: current main does not implement cron output history, so the PR is not obsolete, but the current head still has a correctness blocker and lacks required real behavior proof. The author addressed several early review comments, including delivery-mode recording and external-hook wrapping, but the branch still records non-substantive control output as future prompt history.

## What This Changes

The PR adds an `outputHistory` cron agent-turn option, stores bounded `recentOutputs` in cron job state, injects those outputs into isolated-agent prompts, and updates cron tool, normalization, gateway schema, and tests.

## Best Possible Solution

Land a rebased version only if it stores substantive prior outputs, handles malformed `outputHistory` inputs consistently with adjacent fields, preserves current cron/protocol contracts, and includes redacted real cron-run proof.

## Reproduction Assessment

Yes for the review findings from source inspection: the PR records any successful `outputText`, while cron can return ok with silent/control output such as `NO_REPLY`. I did not run tests because this review is required to keep the checkout read-only.

## Solution Assessment

No. The feature direction is plausible, but this patch should filter non-substantive control output, normalize malformed `outputHistory` values, rebase onto current cron contracts, and show real behavior proof before merge.
