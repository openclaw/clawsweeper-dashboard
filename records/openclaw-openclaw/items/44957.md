## Summary

Keep open: current main does not implement cron output history, and the PR still has source-backed correctness issues plus no real behavior proof from a real cron run.

## What This Changes

The PR adds an outputHistory cron agent-turn flag that stores recent isolated-agent outputs in job state and injects them into later isolated-agent prompts, with schema, tool, and test updates.

## Best Possible Solution

Land a rebased version that stores only substantive prior outputs, preserves robust cron payload normalization, keeps the protocol/tool contract aligned, and includes redacted real setup proof.

## Reproduction Assessment

Yes for the review findings from source inspection: current cron paths can return ok with HEARTBEAT_OK or NO_REPLY in outputText, and the PR writes any ok outputText into recentOutputs. I did not run tests because the review must keep the checkout read-only.

## Solution Assessment

No. The feature direction is plausible, but this patch should filter non-substantive control output, normalize malformed outputHistory values, and then show real cron behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Filter control-token replies before recording history:** `src/cron/service/timer.ts:347-351`
  - body: This records every successful outputText, but cron already treats HEARTBEAT_OK and NO_REPLY as control-only or silent replies. Those runs can return status ok with the token still in outputText, so outputHistory would persist the token and inject it into the next prompt as if it were a substantive previous result.
  - confidence: 0.88
- **[P3] Drop malformed outputHistory values during normalization:** `src/cron/normalize.ts:105`
  - body: Adding outputHistory as an agent-turn hint leaves malformed values like outputHistory: "true" in the payload, which later fails schema validation. The adjacent allowUnsafeExternalContent path deletes non-boolean values before validation; outputHistory should get the same cleanup or a deliberate coercion.
  - confidence: 0.74
