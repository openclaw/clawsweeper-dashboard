## Summary

This PR should stay open: current main still lacks the proposed cron output-history contract, and the PR remains a new feature with unresolved source-review findings plus missing real behavior proof.

## What This Changes

The PR adds an opt-in cron `outputHistory` flag that stores recent isolated-agent outputs in job state and injects them into later isolated cron prompts, with tool, normalization, protocol, and test updates.

## Best Possible Solution

Decide whether isolated cron should get bounded prior-output memory; if accepted, land a rebased implementation that filters non-substantive outputs, normalizes the new flag consistently, updates public docs/protocol/tests, and includes real cron-run proof.

## Reproduction Assessment

Yes for the PR review blockers: they are source-reproducible from the PR diff and PR-head delivery path. Reproduction is not applicable to the underlying feature request because current docs define isolated cron as fresh-session behavior.

## Solution Assessment

No. The feature direction is plausible, but this patch needs product approval, control-token filtering, normalization cleanup, rebase/mergeability work, docs alignment, and real behavior proof before it is the narrow maintainable solution.
