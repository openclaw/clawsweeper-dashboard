## What This Changes

This PR adds an opt-in cron `outputHistory` flag, `recentOutputs` job state, prompt injection, gateway/tool schema support, and tests for isolated cron agent runs.

## Best Possible Solution

Land a rebased, maintainer-approved opt-in cron memory mode that filters non-substantive output before storing, normalizes the new flag consistently, includes focused regression coverage, and has redacted real cron-run proof.

## Reproduction Assessment

Yes for the PR review blockers: PR-head source inspection shows successful `outputText` is persisted without filtering control-only replies, and non-boolean `outputHistory` is not normalized before strict schema validation. The underlying request is a new opt-in capability, not a current-main bug reproduction.

## Solution Assessment

Unclear as proposed: a per-job opt-in history flag is plausible, but the branch must be rebased and the persisted-memory contract needs maintainer approval. The safer implementation filters non-substantive output before persistence and treats the new flag like adjacent boolean fields in normalization.

  - confidence: 0.89

  - confidence: 0.82
