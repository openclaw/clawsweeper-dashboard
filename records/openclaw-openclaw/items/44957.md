## What This Changes

The branch adds an opt-in cron `agentTurn` `outputHistory` flag that persists recent run outputs in job state and injects them into later isolated-agent prompts, with schema, tool, and test updates.

## Best Possible Solution

Land a rebased, maintainer-approved opt-in cron memory mode that filters non-substantive output, normalizes the new flag consistently, documents the stateful contract, and includes redacted real cron-run proof.

## Reproduction Assessment

Yes for the PR review blockers: source inspection shows the PR-head recording gate can persist current-main silent/control output, and malformed nested `outputHistory` values can survive normalization until schema validation. The underlying request is a new opt-in capability, not a current-main bug reproduction.

## Solution Assessment

No. The direction is plausible, but this patch is not the best mergeable solution until it filters control replies, normalizes malformed values, documents the opt-in memory contract, rebases, and provides real behavior proof.
