## Summary

Keep this open. Current main still applies the generic cross-agent session-history gate before honoring spawned child lineage, and the original `visibility="all"` allowlist repro is still blocked because spawned lineage is only loaded for `tree` mode. The prior fix PRs were closed unmerged, and there is no open PR that specifically references this issue.

## What This Changes

The issue asks for parent agents to read their own spawned ACP child sessions via `sessions_history` without adding the ACP harness agent id to `tools.agentToAgent.allow`.

## Best Possible Solution

Add a narrow lineage-aware session visibility rule in the shared guard: parent-owned spawned targets should be allowed before generic cross-agent rejection for the default `tree` case and before the `all`-mode allowlist denial when the target is a proven child. Non-spawned cross-agent access should continue to require the existing `tools.agentToAgent` policy, with regression coverage at both guard and `sessions_history` tool level.

## Repair Work Prompt

Fix the ACP child session visibility bug for this issue. Current main lets a parent spawn an ACP child such as `agent:codex:acp:<uuid>` with `spawnedBy` pointing at the parent, but `sessions_history` denies access because `createSessionVisibilityChecker` handles cross-agent targets before checking spawned lineage, and `visibility="all"` does not load lineage before applying `tools.agentToAgent.allow`. Implement a narrow fix in `src/plugin-sdk/session-visibility.ts`: allow proven parent-owned spawned targets before generic cross-agent denial for `tree`, and before the `all`-mode allowlist denial when the target is in the requester’s spawned set. Do not add hard-coded ACP/Codex/OpenCode ids, do not broaden access for non-spawned cross-agent sessions, and keep the existing A2A policy for unrelated targets. Add a RED/GREEN guard test for `agent:main:main` reading `agent:opencode:acp:<uuid>` or `agent:codex:acp:<uuid>` under default `tree`, plus a tool-level `sessions_history` regression. Also audit `session_status` because it has an extra pre-guard cross-agent check. Add a single-line changelog fix entry only if the repo’s active changelog policy requires it, crediting the human reporter/commenter rather than Codex.
