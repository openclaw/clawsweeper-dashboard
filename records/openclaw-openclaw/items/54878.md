## What This Changes

Parent agents should be able to read their own spawned ACP child session history without adding the child harness agent id to tools.agentToAgent.allow.

## Best Possible Solution

Keep the current narrow lineage-aware guard behavior, which uses trusted spawnedBy/parent ownership metadata and preserves existing cross-agent denial for unrelated sessions.

## Reproduction Assessment

No for current main. The reported requester agent:main:main to target agent:codex:acp:* path is now allowed when the child is in the spawned-key set, and regression tests cover that source path.

## Solution Assessment

Yes. The implemented fix is the narrow maintainable approach: allow only verified requester-owned child sessions before generic cross-agent checks, instead of broadening tools.agentToAgent.allow.
