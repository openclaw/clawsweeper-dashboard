## Label Justifications

- `P1`: This is a reported regression in a real Feishu channel workflow where image/media delivery fails for users who migrated appSecret credentials to SecretRef storage.
- `impact:message-loss`: The observable failure is that local images sent through Feishu MEDIA are not received by the chat recipient.
- `impact:auth-provider`: The implicated surface is Feishu appSecret SecretRef resolution for exec/Keychain-backed auth material.

## Summary

Keep open: current main and v2026.6.6 still have the Feishu media path resolving app credentials through the strict runtime account resolver, while all closing PRs for this regression remain open.

## What This Changes

The issue reports that Feishu MEDIA/local image delivery fails when `channels.feishu.*.appSecret` is stored as an exec/Keychain SecretRef instead of plaintext.

## Best Possible Solution

Land one canonical Feishu fix that resolves SecretRef-backed app credentials from the active runtime snapshot before media/client creation across outbound and reply-dispatcher paths, then close this issue after merge.

## Reproduction Assessment

Yes, source-reproducible with high confidence: a file/exec SecretRef appSecret reaches `resolveFeishuRuntimeAccount` strict mode and throws before the Feishu media upload can build a client. I did not run a live Feishu tenant send in this read-only review.

## Solution Assessment

Yes, the best fix is to feed Feishu media and reply delivery resolved app credentials from the active runtime snapshot before client creation. Rethrowing the typed SecretRef error would improve visibility but would not restore image delivery.
