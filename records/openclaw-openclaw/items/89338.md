## Summary

## What This Changes

The issue reports that Feishu MEDIA/image upload fails when `channels.feishu.*.appSecret` is stored as a Keychain/exec SecretRef instead of plaintext.

## Best Possible Solution

Review and land one canonical fix that resolves Feishu app credentials from the active async SecretRef runtime before all outbound media/client-build paths, with regression coverage for outbound, message-tool, and reply-dispatcher delivery.

## Reproduction Assessment

Yes for source reproduction: current `resolveFeishuRuntimeAccount` throws on non-env SecretRefs, and current media/outbound callers still feed unresolved config into Feishu media upload paths. I did not run a live Feishu tenant send during this read-only review.

## Solution Assessment

Yes on the requested direction, but current main is not fixed; the best implementation should resolve credentials before outbound media/client creation instead of only masking or surfacing fallback errors. The open broad candidate https://github.com/openclaw/openclaw/pull/89454 appears to cover more relevant paths than the narrower rethrow-only approach, but it still needs maintainer review and live-proof judgment before merge.
