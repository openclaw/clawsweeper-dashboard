## Label Justifications

- `P1`: This is a user-facing Feishu regression where image/media delivery fails for users who migrated `appSecret` to SecretRef storage.
- `impact:message-loss`: The observable failure is that local Feishu media messages are not delivered to the recipient.
- `impact:auth-provider`: The root path is Feishu `appSecret` SecretRef resolution for exec/Keychain-backed auth material.

## Summary

Keep open: current main and v2026.6.8 still have the Feishu media path resolving app credentials through the strict SecretRef resolver before upload, while three open PRs still claim this issue with closing syntax.

## What This Changes

The issue asks for Feishu MEDIA/local image upload to work when `appSecret` is stored as an exec/Keychain SecretRef.

## Best Possible Solution

Land one canonical Feishu fix that feeds media, message-tool, and reply-dispatcher sends resolved app credentials before Feishu client creation, then close this issue after the merge.

## Reproduction Assessment

Yes, at source level. Current main still sends Feishu media through strict `resolveFeishuRuntimeAccount`, which throws on unresolved exec/file SecretRefs, and the outbound catch can degrade the image send to text or URL fallback.

## Solution Assessment

Yes for keeping this open, but no merged fix exists yet. The maintainable fix is to use the active resolved runtime credentials before Feishu media/client creation across the affected delivery paths rather than closing the issue while linked PRs are still open.
