## Label Justifications

- `P1`: This is a user-facing Feishu regression where image/media delivery fails for users who migrated `appSecret` to SecretRef storage.
- `impact:message-loss`: The observable failure is that local Feishu media messages are not delivered to the recipient.
- `impact:auth-provider`: The root path is Feishu `appSecret` SecretRef resolution for exec/Keychain-backed auth material.

## Summary

Keep open: current main and the latest release still have Feishu media/reply paths that can hit strict `SecretRef` resolution for `appSecret`, while the three PRs that explicitly target this regression remain open.

## What This Changes

The issue reports that Feishu MEDIA/image uploads fail when `appSecret` is stored as an exec/Keychain `SecretRef` because media delivery does not use the resolved secret string.

## Best Possible Solution

Land one canonical Feishu fix that feeds media, message-tool, and source-channel reply delivery resolved app credentials from the active runtime snapshot before client creation, then close this issue after merge.

## Reproduction Assessment

Yes. The source path is high-confidence: a file/exec/keychain `appSecret` SecretRef reaches `resolveFeishuRuntimeAccount` strict mode through `sendMediaFeishu`, throws `FeishuSecretRefUnavailableError`, and the outbound media path can degrade to a non-image fallback.

## Solution Assessment

Yes for the direction, but not yet landed. The best fix is to use the active resolved runtime credential snapshot before Feishu media/client creation across outbound and reply-dispatcher paths; only rethrowing the typed error improves visibility but does not restore image delivery.
