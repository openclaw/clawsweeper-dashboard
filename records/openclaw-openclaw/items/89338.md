## Label Justifications

- `P1`: This is a reported regression in a real channel workflow where Feishu media/image delivery fails for users who migrated credentials to SecretRef storage.
- `impact:message-loss`: The reported failure is that local images sent through Feishu MEDIA are not received by the chat recipient.
- `impact:auth-provider`: The root surface is SecretRef credential resolution for Feishu appSecret, including exec/Keychain-backed auth material.

## Summary

Keep open: current main and the latest release still have Feishu media paths that call strict runtime credential resolution with unresolved SecretRefs, and the linked fix PRs are all still open rather than merged.

## What This Changes

Fix Feishu MEDIA/local image upload when `channels.feishu.*.appSecret` is stored as an exec/Keychain SecretRef instead of plaintext.

## Best Possible Solution

Land one canonical Feishu fix that resolves SecretRef-backed app credentials from the active runtime snapshot before media/client creation, covers outbound and reply-dispatcher paths, and then close this issue after the PR merges.

## Reproduction Assessment

Yes, source-reproducible: an exec/file SecretRef appSecret reaches `resolveFeishuRuntimeAccount` strict mode and throws, while Feishu media/outbound catch blocks still convert upload failure into fallback text. I did not run a live Feishu tenant send in this read-only review.

## Solution Assessment

Yes, the maintainable fix is to feed Feishu media/outbound delivery a resolved runtime credential snapshot or move resolution to the client-build boundary. Merely rethrowing the SecretRef error improves visibility but does not restore image upload delivery.
