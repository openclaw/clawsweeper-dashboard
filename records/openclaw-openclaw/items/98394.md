## Best Possible Solution

## Reproduction Assessment

Yes at source level. Current main stores approval errors globally, inserts newer approvals at the front of the queue, and only checks whether the old ID remains anywhere in the queue before showing a failed resolve error.

## Solution Assessment

Yes. Capturing the request client and requiring the same active queue item before setting the error is narrower than per-approval error state or gateway changes and preserves the existing stale-resolution refresh path.
