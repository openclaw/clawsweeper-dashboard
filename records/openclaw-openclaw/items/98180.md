## Summary

Keep open: current main and the latest release still contain generic-looking `timeoutSeconds: 600` agent examples, and `agent-via-gateway` still falls back to 600 seconds when config is unset even though the shared agent runtime default is 48 hours.

## What This Changes

The issue asks to remove or qualify generic `agents.defaults.timeoutSeconds: 600` examples and align the unset `agent-via-gateway` timeout fallback with the 48-hour runtime default.

## Best Possible Solution

Update the docs to omit `agents.defaults.timeoutSeconds` unless a short cap is intentional, and make `agent-via-gateway` inherit the shared agent timeout resolver for unset config.

## Reproduction Assessment

Yes. Source inspection reproduces the mismatch: current docs and `agent-via-gateway` still use 600-second examples/fallbacks while the shared runtime resolver defaults unset agent timeouts to 48 hours.

## Solution Assessment

Yes. The narrow maintainable path is to clean the misleading docs examples and reuse the existing shared timeout semantics instead of adding another config knob.
