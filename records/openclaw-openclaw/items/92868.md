## Summary

Keep open: the linked gateway restart bug is real, but this patch should not merge as-is because it turns `browser.profiles.*` into a no-op reload path and can skip the runtime config refresh that running browser profile resolution depends on.

## What This Changes

Adds a no-op reload-plan rule for `browser.profiles` paths and a unit test asserting those paths do not trigger gateway restart.

## Best Possible Solution

Classify `browser.profiles.*` as hot/no-restart, preferably through browser-owned reload metadata, so profile config writes avoid SIGUSR1 while still refreshing active runtime config and browser profile state.

## Reproduction Assessment

Yes. Source inspection shows the PR makes `browser.profiles.*` a no-op reload plan, and current no-op plans return before runtime config activation while the browser profile resolver depends on the runtime config refresh source.

## Solution Assessment

No. Avoiding SIGUSR1 for browser profile patches is the right goal, but `kind: "none"` is too broad because it also skips hot reload; the safer fix is hot/no-restart semantics for profile paths.
