## Summary

Keep open. Current main still rejects provider entries that omit `models`, and this PR remains the canonical open fix path, but it needs contributor real-behavior proof plus deterministic test and private-network policy changes before merge.

## What This Changes

This PR changes `ModelProviderSchema.models` to default to an empty array and adds schema plus memory embedding tests for provider entries that omit `models` while configuring `baseUrl`.

## Best Possible Solution

Land a rebased narrow fix that accepts omitted provider `models` at the validation boundary, uses current schema/runtime paths, adds deterministic regression coverage, and preserves explicit private-network opt-in for embedding baseUrls.

## Reproduction Assessment

Yes, by source inspection. Current main validates raw config with `OpenClawSchema.safeParse`, and `ModelProviderSchema` still requires `models`, so the PR-body config is rejected before memory embeddings can use the configured `baseUrl`.

## Solution Assessment

No, not as submitted. Defaulting `models` is the right narrow direction, but this patch still needs deterministic current-seam tests, real behavior proof, and maintainer-approved private-network behavior for embedding baseUrls.

  - body: The new test relies on real DNS for `example.com` during the SSRF preflight before mocked `fetch` is reached. In restricted CI this can fail before it tests baseUrl precedence, so use the current remote-client seam or mock the pinned hostname/lookup path.

Preserve the existing explicit `allowPrivateNetwork` policy for localhost, RFC1918, link-local, and metadata targets before widening this input shape.
  - confidence: 0.82
