## Summary

Keep open: current main still requires `models.providers.<id>.models`, so the PR targets a real config bug, but the submitted patch is not ready because it expands a security-sensitive embedding `baseUrl` path without resolving the private-network policy issue and adds a DNS-dependent regression test.

## What This Changes

The PR defaults `ModelProviderSchema.models` to `[]` and adds schema plus memory embedding tests for provider entries that set credentials or `baseUrl` without listing models.

## Best Possible Solution

Land a rebased replacement that accepts omitted provider `models`, keeps parsed `models` as `[]`, routes memory embedding private endpoints through explicit policy, and uses deterministic regression coverage.

## Reproduction Assessment

Yes, at source level. Current main requires `models` during `OpenClawSchema.safeParse`, so an omitted-model provider block is rejected before the embedding client can use its configured `baseUrl`.

## Solution Assessment

No, not as submitted. Defaulting `models` is the right config-boundary direction, but the patch should also align memory embedding `baseUrl` with explicit private-network policy and avoid DNS-dependent tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate private embedding endpoints behind explicit policy:** `src/config/zod-schema.core.ts:258-264`
  - body: Defaulting `models` lets omitted-model provider blocks pass validation and reach the memory embedding HTTP path. That path turns the configured host into `allowedHostnames`, which skips private/internal address checks, so pair this schema change with the existing explicit private-network policy before accepting the new config shape.
  - confidence: 0.83
- **[P2] Stub the SSRF lookup in the embedding regression:** `packages/memory-host-sdk/src/host/embeddings.test.ts:191-199`
  - body: The new regression relies on `example.com` resolving during `fetchWithSsrFGuard` preflight before the mocked `fetch` is reached. In restricted CI or offline local runs this can fail for DNS reasons unrelated to the behavior under test, so use the pinned-hostname/lookup seam or test the remote client resolver directly.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff expands an accepted config shape into a memory embedding `baseUrl` path whose current SSRF policy can skip private-network checks.

Concerns:

- **[medium] Configured embedding baseUrls can bypass private-network checks:** `packages/memory-host-sdk/src/host/remote-http.ts:10`
  - body: `buildRemoteBaseUrlPolicy` delegates to `ssrfPolicyFromHttpBaseUrlAllowedHostname`; the SSRF guard skips private/internal checks for configured `allowedHostnames`, so omitted-model provider entries should not expose that path without an explicit operator policy.
  - confidence: 0.83
