## Summary

Keep open: the omitted-model provider config bug is still present on current main, but this PR needs real behavior proof and a safer memory-embedding private-network policy before merge.

## What This Changes

The PR defaults `ModelProviderSchema.models` to `[]` and adds schema plus memory embedding regression tests for provider entries that omit `models` while setting credentials or `baseUrl`.

## Best Possible Solution

Land a rebased replacement that accepts omitted provider `models` at the config boundary, keeps parsed `models` as `[]`, routes memory embedding private endpoints through the existing `request.allowPrivateNetwork` policy, and uses deterministic regression coverage.

## Reproduction Assessment

Yes, at source level. `validateConfigObjectRaw` calls `OpenClawSchema.safeParse`, current `ModelProviderSchema` still requires `models`, and the embedding runtime can only use `providerConfig.baseUrl` after validation succeeds.

## Solution Assessment

No, not as submitted. Defaulting `models` is the right direction, but the patch should align memory embedding `baseUrl` with explicit private-network policy and avoid DNS-dependent tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate private embedding endpoints behind explicit policy:** `src/config/zod-schema.core.ts:258-264`
  - body: Defaulting `models` lets baseUrl-only provider blocks pass validation and reach the memory embedding HTTP path. That path turns the configured host into `allowedHostnames`, which skips private/internal address checks, so localhost, RFC1918, or metadata endpoints can be reached without the documented `request.allowPrivateNetwork` opt-in.
  - confidence: 0.86
- **[P2] Stub the SSRF lookup in the embedding regression:** `packages/memory-host-sdk/src/host/embeddings.test.ts:191-199`
  - body: The new regression relies on `example.com` resolving during `fetchWithSsrFGuard` preflight before the mocked `fetch` is reached. In restricted CI or offline local runs this can fail for DNS reasons, so use the pinned-hostname/lookup seam or test `resolveRemoteEmbeddingBearerClient` directly.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff enables omitted-model provider `baseUrl` entries to reach a memory embedding HTTP policy that currently allowlists the configured host and skips private-network checks.

Concerns:

- **[medium] Configured embedding baseUrls can bypass private-network checks:** `packages/memory-host-sdk/src/host/remote-http.ts:10`
  - body: `buildRemoteBaseUrlPolicy` delegates to `ssrfPolicyFromHttpBaseUrlAllowedHostname`; the SSRF guard skips private/internal checks when the requested host is in `allowedHostnames`, so accepting omitted-model provider entries should not expose that path without explicit operator policy.
  - confidence: 0.84
