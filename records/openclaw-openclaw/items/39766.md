## Summary

Keep open. The PR addresses a valid source-level config and memory-embedding problem that is not implemented on current main, but it is not merge-ready because generated config schema artifacts remain stale, the new embedding regression can depend on real DNS, and embedding baseUrl private-network semantics need maintainer/security judgment.

## What This Changes

The PR defaults `ModelProviderSchema.models` to `[]` and adds schema plus memory-embedding tests for provider entries that configure a `baseUrl` without model declarations.

## Best Possible Solution

Land a narrow update or replacement that makes provider-only config valid across Zod, generated schema, docs, and changelog, with deterministic regression coverage and maintainer-approved private-network handling for memory embedding base URLs.

## Reproduction Assessment

Yes, source-level. Current main validates raw config through `OpenClawSchema.safeParse`, and `ModelProviderSchema` still requires `models`, so a provider block with `baseUrl` but no model list is rejected before embedding routing can use it.

## Solution Assessment

No, not as submitted. Defaulting `models` is the right narrow direction, but the mergeable solution also needs generated schema alignment, deterministic tests, changelog coverage, and a clear private-network policy for embedding base URLs.

## Security Review

Status: needs_attention

Summary: The diff has no dependency or workflow supply-chain changes, but accepting omitted-model provider entries reaches embedding baseUrl handling whose current host policy can skip private-network checks.

Concerns:

- **[medium] Configured embedding base URLs can bypass private-network checks:** `packages/memory-host-sdk/src/host/remote-http.ts:10`
  - body: `buildRemoteBaseUrlPolicy` returns a policy containing the configured host as `allowedHostnames`, and `resolvePinnedHostnameWithPolicy` skips private-network checks for allowed hostnames. Since this PR accepts more provider/baseUrl-only config shapes, maintainers should either keep normal SSRF checks for those base URLs or require an explicit private-network opt-in before allowing localhost, metadata, or RFC1918 targets.
  - confidence: 0.78
