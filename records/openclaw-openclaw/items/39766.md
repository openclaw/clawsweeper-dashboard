## Summary

Keep open. The PR still targets a real current-main config validation gap, but it is not merge-ready because it lacks after-fix real behavior proof and leaves contract alignment, private-network policy, and deterministic test issues unresolved.

## What This Changes

The branch changes `ModelProviderSchema.models` to default to an empty array and adds schema plus memory embedding regression tests for provider entries that omit `models`.

## Best Possible Solution

Land a rebased focused fix or replacement that accepts omitted provider `models` at the config input boundary, keeps public config schema and TypeScript types aligned, uses deterministic regression coverage, and applies maintainer-approved private-network policy for embedding baseUrls.

## Reproduction Assessment

Yes, by source inspection on current main: raw config validation still runs `OpenClawSchema.safeParse`, and `ModelProviderSchema` still requires `models`, so the omitted-model provider config is rejected before embedding routing can use its custom `baseUrl`.

## Solution Assessment

No, not as submitted. Defaulting or otherwise accepting omitted `models` is the right direction, but this patch also needs public contract alignment, deterministic tests, real behavior proof, and a private-network policy decision for embedding baseUrls.

  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff expands access to the memory embedding baseUrl path whose current configured-host policy can bypass private-network checks.

Concerns:

- **[medium] Configured embedding baseUrls can bypass private-network checks:** `packages/memory-host-sdk/src/host/remote-http.ts:10`
  - body: `buildRemoteBaseUrlPolicy` delegates to `ssrfPolicyFromHttpBaseUrlAllowedHostname`, and the SSRF guard skips private/internal checks for hosts in `allowedHostnames`. Since this PR allows more baseUrl-only provider configs to reach that path, memory embeddings should use explicit `allowPrivateNetwork` semantics or block private/internal targets by default.
