## Summary

Keep open: current main still has the reported validation bug and this PR is a plausible fix, but the submitted diff is not mergeable because generated schema/docs, embedding baseUrl security semantics, deterministic test setup, and changelog coverage are still unresolved.

## What This Changes

The PR defaults `ModelProviderSchema.models` to an empty array and adds schema plus memory embedding regression coverage for provider entries that only configure credentials or `baseUrl`.

## Best Possible Solution

Land a narrow equivalent fix that keeps provider-only config valid across Zod, generated schema, and docs, while preserving explicit SSRF opt-in semantics for private endpoints and testing the embedding route without live DNS.

## Reproduction Assessment

Yes, source-level. `OpenClawSchema.safeParse` validates raw config and current `ModelProviderSchema` still requires `models`, so a provider block with `baseUrl` but no model list is rejected before embedding routing can use it.

## Solution Assessment

No, not as submitted. Defaulting `models` is the right narrow direction, but the mergeable solution also needs generated schema/docs alignment, deterministic remote embedding tests, changelog coverage, and maintainer-approved private-network behavior.

## Security Review

Status: needs_attention

Summary: Needs attention: there are no dependency or workflow supply-chain changes, but the config acceptance change reaches embedding baseUrl handling whose current host policy can skip private-network checks.

Concerns:

- **[medium] Configured embedding base URLs can bypass private-network checks:** `packages/memory-host-sdk/src/host/remote-http.ts:10`
  - body: `buildRemoteBaseUrlPolicy` returns a policy containing the configured host as `allowedHostnames`, and `resolvePinnedHostnameWithPolicy` skips private-network checks for allowed hostnames.
