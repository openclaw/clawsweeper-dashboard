## Summary

Keep open: the underlying config-validation bug is still present on current main, but this PR is not mergeable as submitted because it leaves generated schema/changelog drift and needs maintainer security judgment for embedding `baseUrl` private-network policy.

## What This Changes

The PR defaults `ModelProviderSchema.models` to an empty array and adds schema plus embedding regression coverage for provider entries that omit a model list.

## Best Possible Solution

Land a narrow updated fix that keeps provider-only config valid across Zod, generated schema, docs, and runtime expectations while preserving explicit private-endpoint policy and deterministic regression coverage.

## Reproduction Assessment

Yes, source-level. Current `OpenClawSchema.safeParse` still validates raw config against a `ModelProviderSchema` where `models` is required, so the provider block from the PR body is rejected before embedding routing can use its `baseUrl`.

## Solution Assessment

No, not as submitted. Defaulting `models` is the right narrow direction, but the mergeable solution also needs generated schema/docs alignment, changelog coverage, deterministic embedding tests, and maintainer-approved private-network behavior.
