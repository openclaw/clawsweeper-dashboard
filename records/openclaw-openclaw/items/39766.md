## Summary

Keep this PR open. Current main still rejects `models.providers.<id>` entries that omit `models`, so the config-boundary bug remains even though https://github.com/openclaw/openclaw/issues/39589 was closed for the lower-level embedding `baseUrl` routing fix; the submitted patch still needs security/product handling, deterministic test cleanup, and real behavior proof before merge.

## What This Changes

The PR changes `ModelProviderSchema.models` to default to `[]` and adds schema plus memory embedding regression tests for provider entries that configure `baseUrl` without listing models.

## Best Possible Solution

Land a rebased replacement that accepts omitted provider `models` at the config boundary, keeps public config behavior aligned, adds deterministic regression coverage, and routes private/internal embedding endpoints through explicit policy.

## Reproduction Assessment

Yes, at source level. Current main validates raw config with `OpenClawSchema.safeParse`, and `ModelProviderSchema` still requires `models`, so the omitted-model provider config in the PR body is rejected before embedding routing can use its custom `baseUrl`.

## Solution Assessment

No, not as submitted. Defaulting or otherwise accepting omitted `models` is the right direction, but the patch needs deterministic tests and maintainer-approved private-network handling for memory embedding baseUrls.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep private embedding endpoints behind explicit policy:** `src/config/zod-schema.core.ts:258-264`
  - body: Defaulting `models` lets baseUrl-only provider blocks reach the memory embedding HTTP path. That path currently converts the configured host into `allowedHostnames`, which skips private/internal address checks, so the fix should thread the existing explicit private-network policy or keep localhost/metadata/RFC1918 targets blocked by default.
  - confidence: 0.85
- **[P2] Avoid live DNS in the embedding regression:** `packages/memory-host-sdk/src/host/embeddings.test.ts:196-197`
  - body: The added regression calls `embedQuery` against `https://example.com/v1`, so SSRF preflight DNS can fail before the mocked fetch is reached. Use the current pinned-hostname/lookup seam or test `resolveRemoteEmbeddingBearerClient` directly to keep this coverage deterministic in restricted CI.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: Accepting omitted-model provider entries expands the memory embedding `baseUrl` path that currently bypasses private-network checks via configured-host allowlisting.

Concerns:

- **[medium] Configured embedding baseUrls can bypass private-network checks:** `packages/memory-host-sdk/src/host/remote-http.ts:10`
  - body: `buildRemoteBaseUrlPolicy` delegates to `ssrfPolicyFromHttpBaseUrlAllowedHostname`, and the SSRF guard skips private/internal checks when the requested host is in `allowedHostnames`; this PR should not make the omitted-model config shape reach that path without explicit operator policy.
