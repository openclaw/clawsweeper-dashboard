## Summary

Keep open. The branch targets a real current-main config bug, but it is not merge-ready because the generated public schema would remain stale, the added embedding test can depend on live DNS, and configured embedding baseUrl private-network semantics need maintainer/security judgment.

## What This Changes

The PR defaults `ModelProviderSchema.models` to an empty array and adds schema plus remote embedding regression tests for provider entries without model lists.

## Best Possible Solution

Land a narrow equivalent fix after regenerating the config schema/docs baseline, adding current memory-host tests with mocked SSRF lookup, adding the changelog entry, and resolving the private-network opt-in behavior.

## Reproduction Assessment

Yes, source-level. `OpenClawSchema.safeParse` validates raw config and current `ModelProviderSchema` requires `models`, so a provider with `baseUrl` and no model list is rejected before embeddings can use the configured endpoint.

## Solution Assessment

No, not as submitted. Defaulting `models` is the right narrow direction, but the mergeable solution also needs the generated/public schema, deterministic tests, changelog, and the embedding baseUrl SSRF policy aligned.

## Security Review

Status: needs_attention

Summary: The diff has no dependency or workflow supply-chain change, but it expands successful configuration of embedding baseUrls whose current policy can skip private/internal address checks.

Concerns:

- **[medium] Configured embedding baseUrls can skip private-network checks:** `packages/memory-host-sdk/src/host/remote-http.ts:10`
  - body: `buildRemoteBaseUrlPolicy` returns a policy containing the configured host as `allowedHostnames`, and `resolvePinnedHostnameWithPolicy` skips private-network checks for allowed hostnames.
