## Summary

Keep open. The PR still targets a real config-validation gap because current main requires `models` under `models.providers.<id>`, but the submitted patch remains incomplete: it needs public config contract alignment, deterministic current-seam regression coverage, real behavior proof, and maintainer/security direction for private-network embedding base URLs.

## What This Changes

The PR defaults `ModelProviderSchema.models` to `[]` and adds schema plus memory embedding regression tests for provider entries that configure `baseUrl` without a `models` array.

## Best Possible Solution

Land a rebased focused fix that accepts omitted provider `models`, keeps the public config/type contract explicit, uses deterministic regression tests, and applies maintainer-approved private-network policy to embedding baseUrls.

## Reproduction Assessment

Yes, by source inspection. Current main still validates raw config through `OpenClawSchema.safeParse`, and `ModelProviderSchema` still requires `models`, so the PR body's provider-only config is rejected before the embedding runtime can use its custom `baseUrl`.

## Solution Assessment

No, not as submitted. Defaulting or otherwise accepting omitted `models` is the right direction, but the patch also needs public contract alignment, deterministic current-seam tests, and a maintainer-approved private-network policy for embedding baseUrls.

## Security Review

Status: needs_attention

Summary: Accepting omitted-model provider entries expands access to embedding `baseUrl` handling whose current configured-host policy can bypass private-network checks.

Concerns:

- **[medium] Configured embedding baseUrls can bypass private-network checks:** `packages/memory-host-sdk/src/host/remote-http.ts:10`
  - body: `buildRemoteBaseUrlPolicy` delegates to `ssrfPolicyFromHttpBaseUrlAllowedHostname`, and the SSRF guard skips private/internal checks for hosts in `allowedHostnames`. Since this PR allows more provider baseUrl-only configs to reach that path, memory embeddings should use explicit private-network opt-in semantics or block private/internal targets by default.
  - confidence: 0.84
