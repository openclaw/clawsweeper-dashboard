## Summary

Keep open: this PR targets a real config validation bug linked to #39589, and the author already narrowed the branch by dropping the unrelated UI work, but the current patch is not merge-ready because the public generated schema, embedding SSRF policy, deterministic test setup, and changelog are still unresolved.

## What This Changes

The PR defaults `ModelProviderSchema.models` to an empty array and adds schema plus memory embedding regression coverage for provider entries that only configure credentials or `baseUrl`.

## Best Possible Solution

Land a narrow equivalent fix that keeps provider-only config valid across Zod, generated schema, docs, and changelog, while preserving explicit private-network semantics and testing embedding baseUrl routing without live DNS.

## Reproduction Assessment

Yes, source-level. Current `OpenClawSchema.safeParse` validation still requires `models.providers.*.models`, so the reported provider block with `baseUrl` and no `models` is rejected before memory embedding routing can use it.

## Solution Assessment

No, not as submitted. Defaulting `models` is the right narrow direction, but the mergeable solution also needs generated schema/docs alignment, deterministic remote embedding tests, changelog coverage, and maintainer-approved private-network behavior.

## Security Review

Status: needs_attention

Summary: No dependency or workflow supply-chain issue was found, but the config acceptance change reaches embedding baseUrl handling whose current host policy can skip private-network checks.

Concerns:

- **[medium] Configured embedding base URLs can bypass private-network checks:** `packages/memory-host-sdk/src/host/remote-http.ts:10`
  - body: `buildRemoteBaseUrlPolicy` returns a policy containing the configured host as `allowedHostnames`, and `resolvePinnedHostnameWithPolicy` skips private-network checks for allowed hostnames. Since this PR accepts credential/baseUrl-only provider entries, maintainers should either keep normal SSRF checks for those baseUrls or require an explicit private-network opt-in before allowing localhost, metadata, or RFC1918 targets.
  - confidence: 0.8
