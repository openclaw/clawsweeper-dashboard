## Summary

Keep open. Current main still has the reported config validation problem, so this PR is not obsolete, but the proposed patch should not merge as-is because it leaves the generated public config schema stale, lacks the required changelog entry, needs a deterministic embedding test, and expands a configured embedding baseUrl path whose SSRF policy needs maintainer/security review.

## What This Changes

The PR makes `ModelProviderSchema.models` default to an empty array and adds config plus embedding regression tests for provider entries that only set credentials or `baseUrl`.

## Best Possible Solution

Land an equivalent narrow fix after aligning the Zod default, generated JSON schema/docs baseline, changelog, and current memory-host regression tests, with an explicit decision on whether configured embedding baseUrls should keep normal SSRF checks or require a documented private-network opt-in.

## Reproduction Assessment

Yes. A source-level reproduction is a config with `models.providers.openai.baseUrl` and credentials but no `models`; current main requires `models` during `OpenClawSchema.safeParse`, so validation rejects the config before the embedding runtime can use the configured baseUrl.

## Solution Assessment

No, not as submitted. Defaulting `models` to `[]` is a reasonable core direction, but the mergeable fix also needs the generated/public config contract refreshed, deterministic SSRF-guarded tests, a changelog entry, and a settled remote embedding baseUrl security policy.
