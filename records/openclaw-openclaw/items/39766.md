## Summary

Keep open. Current main still rejects provider entries that omit `models`, so the config gap remains, but this PR still needs real behavior proof plus provider-request policy and deterministic-test fixes before it is mergeable.

## What This Changes

The PR defaults `ModelProviderSchema.models` to `[]` and adds schema plus memory embedding tests for provider `baseUrl` entries that omit `models`.

## Best Possible Solution

Land a complete config-contract fix that accepts omitted provider models with safe array output, aligns memory embedding HTTP with provider request policy, and keeps deterministic regression coverage on the current test surface.

## Reproduction Assessment

Yes, at source level. Current main requires `models` in `ModelProviderSchema`, validation rejects the omitted field, and the memory embedding client would otherwise read `models.providers.<id>.baseUrl`.

## Solution Assessment

No, not as submitted. Defaulting `models` is the right config-boundary direction, but the patch still needs provider request policy alignment, deterministic tests, a rebase, and real behavior proof before merge.
