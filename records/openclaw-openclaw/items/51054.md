Keep open. Current main and the latest release still lack the specific Media Understanding credential-source guidance, while runtime/schema evidence shows inline media-model `apiKey` is not a supported config field and two focused docs PRs for this issue were closed unmerged.

Clarify Media Understanding documentation so users know provider credentials come from auth profiles, provider env vars, or `models.providers.<provider>.apiKey`, not inline `tools.media.*.models[].apiKey`.
