## Summary

The PR is still useful, but the fence exception is too broad: it trusts any well-formed appended transcript entry with nested `message.role === "user"`, which can be the same shape as an external transcript append, so it weakens takeover detection rather than only accepting OpenClaw-owned queued writes.

## What This Changes

The PR broadens embedded session-fence benign-write handling for queued transcript appends and same-length transcript rewrites, with focused session-lock tests.

## Best Possible Solution

Accept OpenClaw-owned queued user transcript writes through owned-write provenance or an equivalent narrow marker, while preserving takeover detection for valid external user-message appends.

## Reproduction Assessment

Yes for the review finding by source inspection: under the PR head, appending a normal nested user transcript entry while the fence is active reaches the new benign predicate and is trusted.

## Solution Assessment

No; the target false positive is real, but trusting `message.role === "user"` is too broad compared with the existing owned-write publication boundary.
