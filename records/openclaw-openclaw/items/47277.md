  - body: This gives each user a separate QMD state directory, but QMD still indexes the default workspace `MEMORY.md` and `memory/**/*.md` collections. Isolated QMD users can therefore build separate indexes from the same shared corpus instead of only their scoped memory files.
  - confidence: 0.9

  - body: When `userId` is present, the scanner still adds root-level `memory/*.md` files for every user. The existing compaction flush path writes durable notes to `memory/YYYY-MM-DD.md`, so one user's flush can be searched by every other isolated user.
