Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Honor default-on isolation when config is absent:** `extensions/memory-core/src/isolation-identity.ts:27-28`
  - body: readIsolation() returns null when memory.isolation is omitted, so resolveIsolationIdentity() returns no userId even though resolveMemorySearchConfig() defaults isolation.enabled to true. Fresh configs therefore run memory tools unscoped instead of enforcing the advertised default-on isolation.
  - confidence: 0.9
- **[P1] Scope flush-written daily memory before sharing root files:** `packages/memory-host-sdk/src/host/internal.ts:263`
  - body: In isolation mode this still adds root-level memory files for every user. The default memory flush path writes durable session notes to memory/YYYY-MM-DD.md, so one user's compaction notes can be indexed and returned by memory_search for another isolated user.
  - confidence: 0.91
- **[P1] Scope QMD collections, not just QMD state:** `extensions/memory-core/src/memory/qmd-manager.ts:347-351`
  - body: This scopes the QMD state directory by userId, but the resolved default QMD collections still point at the workspace root and memory/**/*.md. Each per-user QMD index can therefore ingest all users' memory files unless the collection paths or query filters are also user-scoped.
  - confidence: 0.86
- **[P2] Wire validated isolation options into runtime behavior:** `src/agents/memory-search.ts:421-426`
  - body: The schema accepts scope, fallbackPolicy, and pathEncoding, but the resolved search config only carries enabled. Users can set validated options such as pathEncoding: "whitelist" or scope: "agent" without the runtime consistently honoring them.
  - confidence: 0.84
