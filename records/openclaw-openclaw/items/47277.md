Full review comments:

- **[P1] Honor default-on isolation when config is absent:** `extensions/memory-core/src/isolation-identity.ts:27-29`
  - body: `resolveMemorySearchConfig` defaults `isolation.enabled` to true, but this helper treats a missing `memory.isolation` block as no policy and returns an unscoped identity. Existing configs without the new block can therefore still build unscoped memory managers while the resolved settings claim isolation is enabled.
  - confidence: 0.9
- **[P1] Stop sharing flush-written root memory across users:** `packages/memory-host-sdk/src/host/internal.ts:213`
  - body: In scoped mode this still adds root-level `memory/*.md` files for every user. The current memory flush plan writes durable notes to `memory/YYYY-MM-DD.md`, so one user's flushed session notes can be indexed and returned for another isolated user unless those writes are scoped or explicitly global.
  - confidence: 0.88
- **[P1] Scope QMD collections by user, not only state:** `extensions/memory-core/src/memory/search-manager.ts:101-105`
  - body: Passing `userId` into `QmdMemoryManager` only changes manager cache/state directories; `resolveMemoryBackendConfig` still builds QMD collections from the shared workspace `MEMORY.md` and `memory/**/*.md` roots. QMD users can still index/search the shared memory tree despite the advertised per-user isolation.
  - confidence: 0.86
- **[P2] Apply the new isolation options at runtime:** `src/agents/memory-search.ts:424-426`
  - body: The schema accepts `scope`, `fallbackPolicy`, and `pathEncoding`, but the resolved runtime settings only retain `enabled`, and the file/path code defaults to hash encoding. Users can set supported-looking options that are ignored or only partially honored, which is unsafe for a compatibility-sensitive config surface.
  - confidence: 0.78
