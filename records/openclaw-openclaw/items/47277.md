  - body: `resolveMemorySearchConfig` defaults isolation to enabled, but this resolver treats a missing `memory.isolation` block as disabled and returns no `userId`.   - confidence: 0.91
- **[P1] Route flush memory into user directories:** `packages/memory-host-sdk/src/host/internal.ts:262-263`
  - body: With a `userId`, `listMemoryFiles` still includes root-level `memory/*.md` for every isolated user, while the flush plan writes durable notes to `memory/YYYY-MM-DD.md`. A flush from one session can become searchable by other users.
  - confidence: 0.89
- **[P2] Honor declared isolation settings:** `src/config/zod-schema.ts:122-126`
  - body: The schema accepts `scope` and `pathEncoding`, but runtime resolution only reliably carries `enabled`; identity resolution ignores `scope`, and path guards/listing use the default encoding. Valid config can parse while silently not changing the isolation boundary operators requested.
