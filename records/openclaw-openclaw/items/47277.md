## Summary

Keep open. Current main does not already provide this per-user memory isolation, but the PR is not merge-ready because the latest head still has a source-level default-on isolation mismatch, unresolved config/security semantics, a conflicting merge state, and no real two-user proof.

## What This Changes

The PR adds a `memory.isolation` config surface and threads optional user identity through memory-core search/get tools, builtin/QMD memory managers, host memory path filtering, SDK exports, and tests.

## Best Possible Solution

Land this or a replacement only after maintainers approve the memory isolation contract, the branch is cleanly rebased, default/config semantics are fixed, legacy/shared-memory behavior is documented or migrated, and redacted real two-user proof shows search and get cannot cross-read users.

## Solution Assessment

Unclear as the best fix. Per-user memory isolation likely belongs in memory-core or a shared memory-host contract, but the current PR still needs maintainer-approved defaults, shared/root memory policy, upgrade behavior, and real proof before it can be considered the best implementation.
