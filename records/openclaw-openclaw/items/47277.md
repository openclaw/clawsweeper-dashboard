## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Honor default-on isolation when config is absent:** `extensions/memory-core/src/isolation-identity.ts:27-29`
  - body: `resolveMemorySearchConfig()` defaults `settings.isolation.enabled` to true when `memory.isolation` is unset, but this resolver returns `null` for the same absent config and then `resolveIsolationIdentity()` returns `undefined`. Existing configs without the new block still run memory tools without a user scope, so the promised default-on isolation is not actually enforced.
  - confidence: 0.93
- **[P1] Scope QMD collections, not only state directories:** `extensions/memory-core/src/memory/qmd-manager.ts:347-351`
  - body: The PR gives QMD a per-user state directory, but default QMD collections still come from the workspace-wide `MEMORY.md` and `memory/**/*.md` sources. Each user's QMD manager can therefore index the same full memory tree, so QMD deployments can still return another user's files despite separate cache/state directories.
  - confidence: 0.9
- **[P1] Keep flush-written memory out of shared root scans:** `packages/memory-host-sdk/src/host/internal.ts:262-263`
  - body: Isolated scans still add root-level `memory/*.md` for every user, while the default memory flush plan writes durable session memories to `memory/YYYY-MM-DD.md`. A DM compaction can therefore write user-specific content into a file that every other isolated user indexes and can retrieve.
  - confidence: 0.88
