## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Scope flush-written daily memory before sharing root files:** `packages/memory-host-sdk/src/host/internal.ts:263`
  - body: In isolation mode this still adds root-level memory files for every user. The memory flush plan writes durable session notes to `memory/YYYY-MM-DD.md`, so a flush from one user can be indexed and returned by `memory_search` for another isolated user. Route flushes into the encoded user directory or stop treating daily root files as shared in isolated mode.
  - confidence: 0.91
- **[P1] Honor default-on isolation when config is absent:** `extensions/memory-core/src/isolation-identity.ts:27-28`
  - body: `readIsolation()` returns `null` when `memory.isolation` is omitted, so `resolveIsolationIdentity()` returns no userId even though `resolveMemorySearchConfig()` resolves `isolation.enabled` to true. Fresh configs therefore run unscoped despite the advertised default-on isolation, and QMD's fail-closed guard is skipped too.
  - confidence: 0.9
- **[P2] Wire validated isolation options into runtime behavior:** `src/agents/memory-search.ts:421-426`
  - body: The schema accepts `scope`, `fallbackPolicy`, and `pathEncoding`, but the memory search config only carries `enabled`, and the path helpers default to hash mode. Users can set validated options such as `pathEncoding: "whitelist"` or `scope: "agent"` without the runtime consistently honoring them.
  - confidence: 0.84
