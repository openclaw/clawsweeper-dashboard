## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Honor default isolation when the config block is absent:** `extensions/memory-core/src/isolation-identity.ts:27-28`
  - body: The PR documents `memory.isolation.enabled` as defaulting to true, and `resolveMemorySearchConfig` resolves it that way, but `readIsolation()` returns `null` when the block is absent. That makes `resolveIsolationIdentity()` return `undefined`, so existing configs without the new block still create unscoped memory managers instead of the new default-on isolation or a fail-closed identity error.
  - confidence: 0.94
- **[P1] Stop sharing flush-written root memory across isolated users:** `src/memory-host-sdk/host/internal.ts:213`
  - body: In isolated mode `listMemoryFiles()` still adds root-level `memory/*.md` for every user, while the default memory flush plan writes durable notes to `memory/YYYY-MM-DD.md`. A DM compaction flush can therefore write one user's notes into a file that every isolated user later indexes and searches.
  - confidence: 0.91
- **[P1] Scope QMD collections, not only QMD state directories:** `extensions/memory-core/src/memory/qmd-manager.ts:347-350`
  - body: This change passes `userId` into the QMD state directory, but the resolved QMD collections still come from the workspace root and `memory/**/*.md`. A per-user QMD index can still ingest shared or other-user paths unless collection resolution is scoped through the same encoded user memory root or an explicit shared-memory contract.
  - confidence: 0.88
