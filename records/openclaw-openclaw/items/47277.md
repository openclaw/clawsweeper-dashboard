Full review comments:

- **[P1] Enforce the default isolation path:** `extensions/memory-core/src/isolation-identity.ts:64-66`
  - body: `resolveMemorySearchConfig` now defaults `memory.isolation.enabled` to true when the config block is absent, but this resolver treats an absent block as disabled and returns no `userId`. In the normal default config, both builtin and QMD managers are therefore created unscoped, so the advertised default-on isolation boundary is bypassed unless the user explicitly adds `memory.isolation`.
  - confidence: 0.92
- **[P1] Stop indexing shared flush files for every user:** `src/memory-host-sdk/host/internal.ts:213`
  - body: In isolation mode this still indexes root-level `memory/*.md` for every user. The existing memory-flush plan writes durable notes to `memory/YYYY-MM-DD.md`, so one user's flushed memory can be returned to another isolated user unless writes are scoped too or root files are explicitly separated from per-user memory.
  - confidence: 0.88
- **[P2] Wire or remove the new isolation config fields:** `src/config/zod-schema.ts:122-126`
  - body: The schema accepts `scope` and `pathEncoding`, but the runtime ignores `scope` and always uses the default hash encoding when listing, reading, and creating QMD paths. Users can set valid config that has no effect, so either thread the resolved settings through the isolation paths or drop the unsupported fields before exposing them.
  - confidence: 0.8
