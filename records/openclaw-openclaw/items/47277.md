## Summary

Keep open: the PR is a substantial default-on privacy feature, but the latest head still leaves concrete isolation gaps, is marked unmergeable in the provided context, and has no after-fix real behavior proof from a multi-user setup.

## What This Changes

The PR adds memory.isolation config/schema, identity/path-encoding helpers, and userId plumbing through memory-core search/get and QMD manager paths.

## Best Possible Solution

Rework this behind an owner-reviewed privacy design that scopes identity, read/search/write paths, builtin and QMD backends, migration/defaults, docs, and live validation together.

## Reproduction Assessment

Yes for the PR defects: source inspection of the latest head shows default configs can remain unscoped, QMD collections still cover workspace-wide memory, and compaction flushes still write shared root files. I did not run tests because this is a read-only review.

## Solution Assessment

No. The current patch is not the best way to solve the issue because it layers a default-on privacy boundary over only part of the memory system; the safer path is an owner-reviewed design with backend parity and write-path enforcement before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Enable isolation when the config block is omitted:** `extensions/memory-core/src/isolation-identity.ts:64-66`
  - body: The resolver returns `undefined` whenever `memory.isolation` is absent, but the PR documents isolation as default-on and `resolveMemorySearchConfig` defaults it to enabled. Existing configs with memory search but no new block therefore create unscoped managers and bypass the new privacy boundary.
  - confidence: 0.92
- **[P1] Scope QMD collections by user:** `extensions/memory-core/src/memory/qmd-manager.ts:347-351`
  - body: This only moves QMD state and index files under `users/<encoded>`. The resolved collections are still the workspace-level `MEMORY.md` and `memory/**/*.md` collections, so every user's separate QMD index can ingest all users' memory files.
  - confidence: 0.9
- **[P1] Route flush writes into the isolated user directory:** `src/memory-host-sdk/host/internal.ts:213`
  - body: In isolated mode the indexer still adds root `memory/*.md` files for every user. Since the existing compaction flush writes durable notes to `memory/YYYY-MM-DD.md`, a DM flush can become shared search context for other isolated users.
  - confidence: 0.86
- **[P2] Honor the accepted isolation options:** `src/agents/memory-search.ts:420-426`
  - body: The schema accepts `scope` and `pathEncoding`, but the resolved search settings keep only `enabled` and downstream calls use hard-coded user/hash behavior. Users can set valid options that parse successfully but do not affect isolation.
  - confidence: 0.82
