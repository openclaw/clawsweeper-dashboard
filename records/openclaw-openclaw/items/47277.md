## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Apply the default isolation policy consistently:** `extensions/memory-core/src/isolation-identity.ts:27-29`
  - body: `readIsolation` returns `null` when `memory.isolation` is absent, so the identity resolver treats omitted config as isolation off even though the schema/defaults and PR text present isolation as default-enabled. In the common omitted-config case, memory tools can still run without a user scope and index shared memory, so the privacy boundary is not actually default-on.
  - confidence: 0.9
- **[P1] Scope sender identities by route context:** `extensions/memory-core/src/isolation-identity.ts:68-70`
  - body: Returning `senderId` unchanged makes the raw sender string the isolation key. Channel/plugin sender IDs are not guaranteed globally unique, so two accounts or transports that both report `123` can land in the same hashed memory directory and QMD state bucket; the isolation key needs the trusted channel/account/session route context, not just the sender token.
  - confidence: 0.86
- **[P1] Route flush writes into isolated memory buckets:** `packages/memory-host-sdk/src/host/internal.ts:262-263`
  - body: In isolation mode this still indexes root-level `memory/*.md` for every user. The current flush plan writes compaction memories to `memory/YYYY-MM-DD.md`, so a private note flushed from one user's session can be indexed and returned for every other isolated user unless flush writes move under the same user-scoped bucket or root daily files stop being treated as private memory.
  - confidence: 0.9
- **[P2] Honor or remove the extra isolation config fields:** `src/agents/memory-search.ts:420-426`
  - body: The schema accepts `scope`, `fallbackPolicy`, and `pathEncoding`, but the resolved memory-search settings only carry `enabled`, and the builtin/QMD path helpers keep using their default hash behavior. Users can set accepted config that does not actually change runtime behavior, which is especially risky for an isolation boundary.
  - confidence: 0.82
