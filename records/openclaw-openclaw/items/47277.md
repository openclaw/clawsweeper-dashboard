- **[P1] Enable isolation when the config block is absent:** `extensions/memory-core/src/isolation-identity.ts:25-29`
  - body: The PR describes isolation as default-on, but `readIsolation()` returns `null` whenever an existing config omits `memory.isolation`; `resolveIsolationIdentity()` then supplies no user scope and the installation remains shared. Resolve a missing block to the documented default policy, or make the feature explicitly opt-in and align the schema, PR body, and upgrade behavior.
  - confidence: 0.99
- **[P1] Route memory flush writes into the isolated scope:** `packages/memory-host-sdk/src/host/internal.ts:125-130`
  - body: The normal pre-compaction flush still appends private session-derived notes to root `memory/YYYY-MM-DD.md`, and this changed branch explicitly allows every isolated user to read root memory files. Derive the flush destination from the same isolation identity, or stop treating root memory as part of a user-private isolation guarantee.
  - confidence: 0.98
- **[P2] Remove or implement the advertised isolation modes:** `src/config/zod-schema.ts:121-126`
  - body: The new schema accepts `scope` and `pathEncoding`, but runtime identity selection ignores `scope` and the read, index, and QMD paths use the default hash encoding rather than the configured value. Thread these settings through one canonical runtime path or remove them until they are supported so operator choices are not silently ignored.
  - confidence: 0.97
