## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Honor default isolation when the config block is absent:** `extensions/memory-core/src/isolation-identity.ts:27-29`
  - body: The schema and PR body say isolation defaults on, but `readIsolation()` returns `null` when `memory.isolation` is missing. Existing configs without the new block therefore return `userId: undefined`, so builtin managers run unscoped and the QMD fail-closed guard also does not trigger. Treat absence as the default isolation policy or make the rollout explicitly opt-in.
  - confidence: 0.94
- **[P1] Stop sharing flush-written root memory across isolated users:** `src/memory-host-sdk/host/internal.ts:213`
  - body: When `userId` is present, this still indexes root-level `memory/*.md` for every user. Current compaction/memory-flush behavior writes private durable notes to `memory/YYYY-MM-DD.md`, so after a flush those notes can appear in every user's isolated search results unless writes move under the same user scope or root memory is no longer treated as shared by default.
  - confidence: 0.91
- **[P1] Scope QMD collections, not only QMD state directories:** `extensions/memory-core/src/memory/qmd-manager.ts:347-351`
  - body: This gives each user a separate QMD config/cache directory, but the resolved QMD collections are still built before any userId is available and still point at the agent-wide `MEMORY.md` and `memory/**/*.md` tree. Each user's QMD index can therefore ingest the same shared memory corpus, so QMD does not get the end-to-end isolation promised by the PR.
  - confidence: 0.89
- **[P2] Apply the new isolation options at runtime:** `src/config/zod-schema.ts:121-126`
  - body: The config schema accepts `scope`, `fallbackPolicy`, and `pathEncoding`, but runtime paths only consistently consume `enabled`; identity resolution hard-codes user-first behavior and host/QMD path code defaults to hash encoding. Either wire these options through every read/list/QMD path with tests or remove the unused surface before it becomes a misleading public config contract.
  - confidence: 0.84
