## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Make absent isolation config honor the default-on contract:** `extensions/memory-core/src/isolation-identity.ts:27-29`
  - body: The PR sets resolved memory search settings to `enabled: true` when `cfg.memory?.isolation` is absent, but `readIsolation()` returns `null` for the same absent block and `resolveIsolationIdentity()` then returns no `userId`. Existing configs without the new block therefore run unscoped instead of fail-closed/default-isolated, so the advertised default-on isolation can silently fail open.
  - confidence: 0.94
- **[P1] Scope flush-written root memory before sharing it:** `src/memory-host-sdk/host/internal.ts:213`
  - body: In user-isolated mode this still adds root-level `memory/*.md` files to every user's corpus. Current memory flush writes durable notes to `memory/YYYY-MM-DD.md` by default, so one user's compacted session notes can be searched by another isolated user unless flush output is moved into the user scope or root daily files stop being shared.
  - confidence: 0.92
- **[P2] Wire validated isolation options into runtime behavior:** `src/agents/memory-search.ts:420-426`
  - body: The schema accepts `scope`, `fallbackPolicy`, and `pathEncoding`, but resolved memory settings only carry `enabled`, and path/list/QMD helpers keep using the default hash encoding and user scope. Users can set accepted config that is silently ignored or applied inconsistently, which is unsafe for a new isolation contract.
  - confidence: 0.86
