Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Honor default-on isolation when config is absent:** `extensions/memory-core/src/isolation-identity.ts:26`
  - body: `resolveMemorySearchConfig` defaults isolation to enabled when `memory.isolation` is missing, but this resolver returns no policy for that same missing config. Existing upgraded configs therefore get unscoped memory tools/QMD managers instead of the PR's advertised fail-closed default.
  - confidence: 0.94
- **[P1] Scope QMD collections, not only QMD state:** `extensions/memory-core/src/memory/qmd-manager.ts:347-351`
  - body: Moving QMD state under a per-user directory does not change which files QMD indexes. The resolved default collections still point at workspace `MEMORY.md` and `memory/**/*.md`, so each user's QMD index can ingest every user's files and return cross-user memory results.
  - confidence: 0.93
- **[P1] Keep private flush output out of shared root:** `src/memory-host-sdk/host/internal.ts:212`
  - body: The isolated scan still includes root-level `memory/*.md` for every user, but current memory flushes write private compaction notes to `memory/YYYY-MM-DD.md`. Until flushes are user-scoped or root files are not treated as shared by default, private notes can be searched by other users.
  - confidence: 0.91
- **[P2] Implement or remove accepted isolation options:** `src/config/types.memory.ts:22-29`
  - body: The PR accepts `scope`, `fallbackPolicy`, and `pathEncoding` as config, but the resolved memory search config only carries `enabled`, and the storage/read/QMD paths use their own default hash behavior. Security-sensitive config fields that do not actually control the runtime boundary will mislead operators.
  - confidence: 0.88
