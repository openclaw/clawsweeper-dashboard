Overall confidence: 0.88

Full review comments:

- **[P1] Apply default isolation when config is omitted:** `extensions/memory-core/src/isolation-identity.ts:27-29`
  - body: `resolveMemorySearchConfig` reports isolation enabled by default, but `resolveIsolationIdentity` returns `undefined` whenever `memory.isolation` is absent. That means users who rely on the advertised default-on behavior still get an unscoped manager that can search all memory files; either preserve legacy sharing by default or enforce identity for the omitted-config path too.
  - confidence: 0.91
- **[P1] Scope QMD collections by the resolved user:** `extensions/memory-core/src/memory/search-manager.ts:60`
  - body: Passing `userId` only into the QMD cache and manager does not scope what QMD indexes: `resolveMemoryBackendConfig` still builds the default collection from the workspace-wide `memory/**/*.md`. Each per-user QMD index can ingest every user's files unless the resolved collections or query filter are partition-aware.
  - confidence: 0.88
- **[P1] Keep memory flush output out of the shared root:** `src/memory-host-sdk/host/internal.ts:213`
  - body: In isolated mode this line indexes root-level `memory/*.md` for every user, but the existing flush plan writes compaction memories to `memory/YYYY-MM-DD.md`. A private DM compaction note can therefore land in the shared root and become searchable by other users.
  - confidence: 0.9
- **[P1] Bound per-user memory manager lifetimes:** `extensions/memory-core/src/memory/manager.ts:169-171`
  - body: Adding raw `userId` to the cache key creates one default `MemoryIndexManager` per sender, and each non-status manager starts file watchers, session listeners, interval sync, and database state. Busy channels can accumulate these long-lived managers indefinitely unless the design uses bounded eviction, per-request scoping, or a shared partition-aware manager.
  - confidence: 0.82
- **[P2] Honor the advertised isolation config fields:** `src/agents/memory-search.ts:420-426`
  - body: The public config types expose `scope`, `fallbackPolicy`, and `pathEncoding`, but the search settings only propagate `enabled`, while host/QMD path helpers use their hard-coded default encoding and `scope` is never applied. Users who set these new fields will get different runtime behavior than the schema promises.
  - confidence: 0.78
