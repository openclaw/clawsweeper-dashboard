## Review Findings

Overall confidence: 0.86

Full review comments:

- **[P1] Make default isolation actually scope memory:** `extensions/memory-core/src/isolation-identity.ts:27-29`
  - body: `resolveMemorySearchConfig` resolves isolation as enabled when the config block is absent, but this helper returns `null` for the same absent block and then `resolveIsolationIdentity` returns `undefined`. That means the advertised default-on mode creates unscoped builtin/QMD managers and can still share memory across users unless operators explicitly add `memory.isolation`, defeating the privacy default this PR introduces.
  - confidence: 0.92
- **[P2] Wire pathEncoding through the runtime:** `src/config/zod-schema.ts:126`
  - body: This schema accepts `memory.isolation.pathEncoding`, but the runtime helpers still call the encoding-sensitive path functions with their default `hash` mode and `readIsolation` never returns `pathEncoding`. Users who configure `whitelist` get silently hashed directories instead, so the new config surface is misleading and untested end to end.
  - confidence: 0.83
