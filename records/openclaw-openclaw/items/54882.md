## What This Changes

The issue asks for verbose tool display to show file paths when Read, Write, or Edit tool calls use the `file` argument alias.

## Best Possible Solution

Settle the alias contract, then either land a refreshed https://github.com/openclaw/openclaw/pull/77312 that aligns display, diagnostics, validation, and tests, or document canonical `path` expectations and close the PR path deliberately.
