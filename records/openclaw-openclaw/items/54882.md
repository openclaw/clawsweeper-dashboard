## What This Changes

The issue reports that verbose Read/Write/Edit tool summaries omit the displayed file path when tool arguments use the `file` alias instead of `path`.

## Best Possible Solution

Keep this issue open while https://github.com/openclaw/openclaw/pull/77312 is reviewed; then land a narrow display/diagnostic fix with regression coverage or explicitly standardize user-facing behavior on canonical `path`.

## Reproduction Assessment

## Solution Assessment

Unclear. Adding `record.file` is narrow for display, but diagnostics should either stay aligned with canonical `path` validation or be paired with an intentional alias contract change.
