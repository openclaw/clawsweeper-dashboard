## Summary

Keep open: current main and the latest release still expose skill names and descriptions as raw skill metadata, while adjacent localization work covers UI chrome rather than the requested CLI/UI/ClawHub/prompt metadata contract.

## What This Changes

The issue requests localized skill names and descriptions for `openclaw skills list`, the Control UI Skills page, ClawHub-installed skills, and possibly agent-facing skill lists.

## Best Possible Solution

Define a locale-aware skill metadata contract with stable keys and English fallback, then resolve display fields in CLI, Control UI, ClawHub metadata, and any approved prompt-visible path through an explicit locale policy.

## Reproduction Assessment

Yes. Source inspection on current main shows `skills.status` exposes raw skill `name` and `description`, the CLI and Control UI render those fields directly, and the gateway request has no locale parameter.

## Solution Assessment

Unclear until maintainers choose the public metadata shape. A Chinese-only bundled-skill patch would be too narrow; the maintainable path is a contract that works for bundled, workspace, plugin, and ClawHub skills without changing stable keys.
