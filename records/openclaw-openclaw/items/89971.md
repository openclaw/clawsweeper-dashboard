## Summary

Keep open: current main and v2026.6.8 still expose skill names and descriptions as raw SKILL.md metadata across CLI, Control UI, gateway status, and prompt-facing skill lists. The remaining work is a product/API contract for localized display metadata, not conservative cleanup.

## What This Changes

Request locale-aware skill names and descriptions in Control UI, `openclaw skills list`, ClawHub-provided skill metadata, and possibly agent-visible tool/skill listings.

## Best Possible Solution

Define a locale-aware skill metadata contract with stable canonical keys, English fallback, and explicit UI/CLI/prompt locale policy before adding translations or publisher-provided fields.

## Reproduction Assessment

Yes. Source inspection shows current main exposes raw SKILL.md `name` and `description`, the CLI and Control UI render those fields directly, and `skills.status` has no locale parameter.

## Solution Assessment

Unclear until maintainers choose the public metadata shape. A Chinese-only bundled-skill patch would be too narrow; the maintainable fix is a contract that works for bundled, workspace, plugin, and ClawHub skills without changing stable keys.
