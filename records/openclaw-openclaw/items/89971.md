## Summary

Keep open: current main and v2026.6.8 still expose skill names and descriptions as raw skill metadata, and the related open Skills-page i18n PR explicitly leaves localized skill names/descriptions out of scope. The remaining work is a locale-aware skill metadata and prompt/UI/CLI policy decision, not conservative cleanup.

## What This Changes

Request localized skill names and descriptions in `openclaw skills list`, the Control UI Skills page, ClawHub-published skill metadata, and possibly agent-visible skill/tool listings.

## Best Possible Solution

Define a locale-aware skill display metadata contract with stable canonical IDs, English fallback, ClawHub publisher fields, and explicit UI/CLI/prompt locale policy, then update gateway, CLI, UI, docs, and focused tests together.

## Reproduction Assessment

Yes, as a source-level feature gap. Current main exposes only raw `name` and `description` strings, renders them directly in CLI/UI/prompt paths, and has no locale parameter or localized skill metadata contract.

## Solution Assessment

Unclear until maintainers choose the public metadata shape. A Chinese-only bundled-skill translation would be too narrow; the maintainable solution is a locale-aware contract that preserves stable keys and works for bundled, workspace, plugin, and ClawHub skills.
