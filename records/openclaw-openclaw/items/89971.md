## Summary

Keep open: current main and v2026.6.8 still expose skill names and descriptions as raw SKILL.md metadata across CLI, Control UI, and prompt-facing skill lists. The related Skills-page i18n PR only localizes surrounding UI chrome and explicitly leaves manifest names/descriptions out of scope, so the remaining work is a product/API contract rather than conservative cleanup.

## What This Changes

The issue requests localized skill names and descriptions in `openclaw skills list`, the Control UI Skills surfaces, ClawHub-provided skill metadata, and possibly agent-visible tool/skill lists.

## Best Possible Solution

Define a locale-aware skill display metadata contract with stable canonical keys, English fallback, publisher/bundled metadata rules, UI/CLI rendering behavior, and an explicit prompt-locale policy before adding translations.

## Reproduction Assessment

Yes. Source inspection shows current main exposes raw SKILL.md `name` and `description` values through `skills.status`, CLI formatting, Control UI rendering, and the prompt-facing skills list.

## Solution Assessment

Unclear until maintainers choose the public metadata shape. A Chinese-only edit to bundled SKILL.md files would be too narrow; the maintainable path is a localized display contract that preserves canonical identity and defines whether model-visible metadata localizes.
