## Summary

Keep open: current main and the reported latest release still expose skill `name` and `description` as unlocalized strings, and both the CLI and Control UI render those fields directly. This is not ClawHub-only cleanup because ClawHub publishers would still need a core skill metadata/API contract for localized display fields.

## What This Changes

Requested localized skill names and descriptions for `openclaw skills list`, the Control UI Skills page, ClawHub-installed skills, and possibly agent-visible skill/tool lists.

## Best Possible Solution

Define one localized skill metadata contract that preserves stable skill keys, lets bundled and ClawHub skills provide translated display fields, and has CLI/UI resolve the active locale through the existing locale machinery where possible.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the feature gap: `skills.status` returns raw skill names/descriptions, and both the CLI and Skills UI render those strings without locale resolution.

## Solution Assessment

Unclear until maintainers choose the public metadata shape. A narrow Chinese-only bundled-skill patch would be too small; the maintainable direction is a contract that works for bundled, workspace, plugin, and ClawHub skills without changing stable keys.
