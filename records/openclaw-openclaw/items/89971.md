## Summary

Keep open: current main and v2026.6.9 still expose skill names and descriptions as raw skill metadata, and the remaining work is a locale-aware skill metadata contract that needs maintainer product/API direction.

## What This Changes

The issue asks for skill names and descriptions to be localized in the Control UI Skills page, `openclaw skills list`, ClawHub-provided skills, and possibly agent tool lists.

## Best Possible Solution

Define a locale-aware skill display metadata contract with stable canonical slugs, English fallback, publisher-provided fields, and explicit UI/CLI/prompt locale policy before updating gateway, CLI, UI, docs, and tests.

## Reproduction Assessment

Not applicable as a failing bug, but yes at source level: current main and v2026.6.9 expose only raw `name` and `description` strings through skill status, CLI, UI, and prompt paths.

## Solution Assessment

Unclear until maintainers choose the public metadata shape. A Chinese-only bundled-skill translation would be too narrow; the maintainable path is a locale-aware contract that preserves stable keys across bundled, workspace, plugin, and ClawHub skills.
