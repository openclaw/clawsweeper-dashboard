## Summary

Keep open: current main and v2026.6.8 still expose skill names and descriptions as raw skill metadata, and the remaining work is a locale-aware product/API contract rather than conservative cleanup.

## What This Changes

The issue requests localized display metadata for skill names and descriptions across `openclaw skills list`, the Control UI Skills page, ClawHub-published skills, and possibly agent-facing skill lists.

## Best Possible Solution

Define a locale-aware skill display metadata contract with stable canonical slugs, English fallback, ClawHub publisher fields, and explicit UI/CLI/prompt locale policy before updating gateway, CLI, UI, docs, and tests.

## Reproduction Assessment

Yes, as a source-level feature gap rather than a broken existing contract: current main exposes only raw `name` and `description` strings, renders them directly in CLI/UI/prompt paths, and has no `skills.status` locale parameter.

## Solution Assessment

Unclear until maintainers choose the public metadata shape. A Chinese-only bundled-skill translation would be too narrow; the maintainable path is a locale-aware contract that preserves stable keys and works for bundled, workspace, plugin, and ClawHub skills.
