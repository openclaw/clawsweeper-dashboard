## Summary

Keep open: current main and the latest release still expose skill `name` and `description` as raw `SKILL.md` metadata, and the CLI, Control UI, and prompt-facing skill list consume those raw fields. This needs a core skill metadata/API contract, not only ClawHub publisher action.

## What This Changes

Add localized display names and descriptions for skills across `openclaw skills list`, Control UI Skills views, and publisher-provided ClawHub skill metadata.

## Best Possible Solution

Define a locale-aware skill metadata contract with stable keys and English fallback, then resolve display fields in CLI/UI and any approved prompt surfaces through an explicit locale policy.

## Reproduction Assessment

Yes. Source inspection on current main shows `skills.status` exposes raw `name` and `description`, the CLI and Control UI render those fields directly, and the gateway request has no locale parameter.

## Solution Assessment

Unclear until maintainers choose the public metadata shape. A Chinese-only bundled-skill patch would be too narrow; the maintainable path is a contract that works for bundled, workspace, plugin, and ClawHub skills without changing stable keys.
