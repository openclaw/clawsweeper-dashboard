The PR adds a WhatsApp `listGroupsLive` directory method that reads groups from the active Baileys socket, registers it on the WhatsApp channel adapter, and adds mocked directory tests plus a voice-call test type adjustment.

## Best Possible Solution

Implement the WhatsApp live group directory so the reported standalone CLI command can fetch groups from the linked account, keep a safe config fallback for unavailable live access, and require redacted live output before merge.

## Reproduction Assessment

Yes, source-level. Current main only exposes config-backed WhatsApp group listing, and the PR's live method still falls back to that config path whenever the standalone CLI process has no in-process WhatsApp controller.

## Solution Assessment
