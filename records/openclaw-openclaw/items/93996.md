The PR registers WhatsApp `listGroupsLive`, implements it by reading `groupFetchAllParticipating()` from a registered Baileys socket with config fallback, and adds mocked tests plus a voice-call test type adjustment.

## Best Possible Solution

Implement the WhatsApp live group directory through a plugin-owned live lookup that works from the standalone CLI and target resolver, preserves the config fallback when live access is unavailable, and is proven with redacted linked-account output.

## Reproduction Assessment

## Solution Assessment

No. Calling Baileys from the WhatsApp plugin is the right owner boundary, but using only the process-local gateway controller is not the narrowest maintainable fix for the reported standalone CLI command.
