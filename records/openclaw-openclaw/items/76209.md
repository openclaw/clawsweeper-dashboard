## Repair Work Prompt

Fix the plugin scan cold-start performance issue reported in https://github.com/openclaw/openclaw/issues/76209. Current main still uses direct `fs.existsSync` in `src/plugins/bundle-manifest.ts`, `src/plugins/bundled-dir.ts`, and `src/plugins/bundled-plugin-scan.ts`. Add a small cached existence helper owned by the plugin scan path, preferably scan-scoped or explicitly invalidatable, and wire only the hot discovery probes through it. Preserve plugin metadata freshness rules from `src/plugins/AGENTS.md`: do not add an unbounded process-wide metadata cache that can hide install/doctor/registry refresh changes. Add focused Vitest coverage for cache hits and stale-state/invalidation boundaries, and include a single-line `CHANGELOG.md` entry if the repair is user-facing performance work. Do not change unrelated plugin discovery behavior, dependency resolution, or broad startup architecture; #67000 remains the separate warm-session issue.
