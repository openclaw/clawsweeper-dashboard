The issue requests reducing cold-start plugin scan latency by caching repeated synchronous `fs.existsSync` probes in plugin manifest discovery paths.

## Best Possible Solution

## Solution Assessment

Yes on the direction: a narrow lifecycle-owned cache around repeated plugin manifest scan probes is the maintainable fix. A broad process-global filesystem cache is riskier unless maintainers explicitly accept that cache lifetime and invalidation boundary.
