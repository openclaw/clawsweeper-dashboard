## Summary

Keep open. Current main still only partially addresses session-artifact cleanup: it has deleted/reset archive retention and optional disk-budget pressure cleanup, but no recycle-based garbage-collection config, and the related open preservation request in #56609 needs maintainer policy before changing default deletion behavior.

## What This Changes

The issue requests configurable session maintenance garbage collection for orphaned `.reset`, `.deleted`, `.bak`, and legacy saved-session artifacts, with a recycle directory and retention window.

## Best Possible Solution

Define one maintainer-approved session-artifact retention policy, then encode it consistently in `session.maintenance` types, strict schema, cleanup/runtime behavior, CLI docs, and regression tests.

## Reproduction Assessment

Yes. A high-confidence source reproduction path exists: create old `.deleted`, `.reset`, and `.bak` archives under enforced maintenance; current code and tests show deleted/reset retention behavior while `.bak` remains unless disk-budget pressure is configured, and the schema lacks the proposed keys.

## Solution Assessment

Unclear as proposed. The recycle-and-retention model may be reasonable, but it is not yet the best maintainable fix until maintainers reconcile it with #56609 and choose the archive deletion policy.
