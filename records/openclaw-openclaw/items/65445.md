## Summary

Keep open. Current main partially cleans session artifacts, but it still does not provide the requested recycle-based garbage-collection settings, normal maintenance still lacks age cleanup for `.bak` archives outside disk-budget pressure, and the related archive-retention issues show this needs a maintainer policy decision before implementation.

## What This Changes

The issue requests configurable recycle-based garbage collection for orphaned `.jsonl.reset.*`, `.jsonl.deleted.*`, `.jsonl.bak.*`, and legacy session backup artifacts during session maintenance.

## Best Possible Solution

Define one maintainer-approved session-artifact lifecycle policy, then implement the chosen retention/recycle behavior consistently across config types, strict schema, config help, CLI cleanup, runtime maintenance, docs, and focused tests.

## Reproduction Assessment

Yes, from source inspection. Configure enforced maintenance with old `.deleted`, `.reset`, and `.bak` archives: current main cleans deleted/reset archives by retention, preserves `.bak` outside disk-budget pressure, and lacks the proposed garbage-collection config keys.

## Solution Assessment

Unclear as proposed. A recycle-based GC model may be reasonable, but maintainers first need to reconcile hard deletion, archive preservation, default retention, and naming semantics with the related open issues.
