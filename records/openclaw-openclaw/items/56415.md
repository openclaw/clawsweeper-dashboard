## Summary

Keep open: the patch is not safe to merge because it would reject normal external-channel `/phone arm` and `/phone disarm` calls that current main deliberately preserves while already blocking gateway-scoped non-admin callers.

## What This Changes

The PR removes the WebChat-only condition from phone-control arm/disarm admin checks and rewrites tests to expect Telegram callers without `operator.admin` to be rejected.

## Best Possible Solution

Keep the current scoped-gateway hardening; if stricter external-channel control is desired, define it as an owner/channel authorization policy with WebChat, gateway-client, and native-channel tests.

## Reproduction Assessment

Yes, by source inspection. Invoke the phone-control command handler with `channel: "telegram"` and no `gatewayClientScopes`: current main's tests expect arm/disarm behavior to work, while the PR guard would return `requires operator.admin`.

## Solution Assessment

No. The PR conflates internal gateway client scopes with external sender authorization; the narrower maintainable fix is the current helper, or a new explicit owner/channel policy if maintainers choose stricter behavior.
