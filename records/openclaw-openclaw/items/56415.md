## Summary

Keep open; the patch is not mergeable as-is because it would reject normal Telegram native `/phone arm` and `/phone disarm` calls that current main intentionally permits without gateway scopes, and it lacks required real Telegram proof for an auth-sensitive visible command change.

## What This Changes

The PR changes phone-control arm/disarm to require `operator.admin` whenever `gatewayClientScopes` is missing or lacks admin, and updates tests to expect Telegram no-scope callers to be blocked.

## Best Possible Solution

Keep the current scoped-gateway hardening unless maintainers deliberately choose a stricter external-channel policy; if they do, model it through explicit owner/channel authorization with WebChat, scoped gateway, and Telegram coverage.

## Reproduction Assessment

Yes for the PR-introduced regression by source inspection: Telegram native `/phone arm` and `/phone disarm` execute without `gatewayClientScopes`, so this patch would reject authorized external callers. No high-confidence current-main bypass was established because main tests intentionally permit no-scope external callers while blocking scoped non-admin gateway callers.

## Solution Assessment

No. The patch conflates internal control-plane gateway scopes with external chat authorization; a stricter policy should be an explicit owner/channel authorization change if maintainers choose that direction.
