## Summary

Keep open: this PR is not merge-ready because it changes the current intentionally tested split between internal gateway clients and authorized external channel callers, causing normal Telegram/native plugin command contexts without `gatewayClientScopes` to be rejected.

## What This Changes

The PR removes the webchat-only condition from phone-control arm/disarm `operator.admin` checks and updates tests to expect Telegram callers without gateway scopes to be rejected.

## Best Possible Solution

Keep the current scoped-gateway hardening unless maintainers choose a stricter phone-control policy; if so, implement it through an explicit owner/channel authorization model with WebChat, scoped gateway, and native channel coverage.

## Reproduction Assessment

Yes, source-reproducible: invoke phone-control with `channel: "telegram"` and no `gatewayClientScopes`; the PR returns `requires operator.admin` while current main tests expect authorized external arm/disarm to remain available.

## Solution Assessment

No. The PR uses internal gateway scopes as a universal authorization signal; the narrower maintainable path is the current helper, or a new explicit owner/channel policy if maintainers decide to tighten external phone-control commands.

## Review Findings

Overall correctness: patch is incorrect
