## Summary

Do not merge as-is: current main already blocks non-admin gateway-scoped phone-control callers, while the PR would also reject ordinary authorized Telegram/external-channel commands that intentionally have no `gatewayClientScopes`. The PR also still lacks real Telegram or external-channel proof for this visible auth behavior.

## What This Changes

The PR changes phone-control arm/disarm guards and tests so `operator.admin` gateway scopes are required on every channel.

## Best Possible Solution

Keep the current scoped-gateway hardening unless maintainers deliberately move phone-control to owner-only external policy, in which case implement explicit owner/channel authorization with WebChat, gateway-scope, and Telegram coverage.

## Reproduction Assessment

Yes for the PR regression by source inspection: a Telegram `/phone arm` or `/phone disarm` context has no `gatewayClientScopes`, so the PR returns `requires operator.admin`. No for the claimed current-main bypass, because current tests and the Telegram command path define authorized external no-scope callers as intentional behavior.

## Solution Assessment

No. The PR conflates internal gateway client scopes with external sender/owner authorization; the safer solution is the current helper or a new explicit owner/channel policy if maintainers choose stricter control.
