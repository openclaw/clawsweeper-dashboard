## Summary

Keep open: the PR still rejects authorized Telegram/native phone-control callers that do not carry internal gateway scopes, while current main intentionally permits external no-scope callers and separately blocks scoped non-admin gateway callers.

## What This Changes

The PR removes the webchat-only condition from phone-control arm/disarm admin gates and changes tests to expect Telegram callers without `operator.admin` to be blocked.

## Best Possible Solution

Keep the current scoped-gateway hardening unless maintainers deliberately choose stricter external-channel policy; if they do, implement it through explicit owner/channel authorization with WebChat, scoped gateway, and Telegram coverage.

## Reproduction Assessment

Yes for the PR-introduced regression by source inspection: Telegram native `/phone arm` and `/phone disarm` execute without `gatewayClientScopes`, so this patch rejects authorized external callers. No high-confidence current-main bypass was established because current tests intentionally permit no-scope external callers while blocking scoped non-admin gateway callers.

## Solution Assessment

No. The patch conflates internal control-plane gateway scopes with external chat authorization; a stricter policy should be an explicit owner/channel authorization decision if maintainers choose that direction.
