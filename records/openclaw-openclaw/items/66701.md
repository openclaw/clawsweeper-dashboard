## Summary

Keep this open: current main still has the reported silent-reply leak path, and the open closing PRs are implementation candidates rather than grounds to close the tracker before merge.

## What This Changes

The issue asks OpenClaw to suppress `NO_REPLY` outputs that include model reasoning or `think` preambles instead of delivering the preamble to chat.

## Best Possible Solution

Land one canonical shared silent-reply classifier with regression coverage for reasoning wrappers, glued and trailing final tokens, and substantive-message false positives; then close this issue after the fix merges.

## Reproduction Assessment

Yes. Source inspection on current main shows a non-empty preamble before `NO_REPLY` normalizes to deliverable text, and the existing tests still assert strip-and-deliver behavior for trailing-token content.

## Solution Assessment

Unclear. Current main is not the solution, and the open PRs choose different suppression policies; the maintainable fix should centralize the classifier after maintainer selection of the intended semantics.
