## Summary

Keep this open: current main still preserves non-empty preamble text before a trailing `NO_REPLY`, and the linked open PRs are implementation candidates rather than grounds to close the tracker before merge.

## What This Changes

The issue asks OpenClaw to suppress replies where reasoning or think preamble text precedes the `NO_REPLY` silent token instead of posting that preamble to chat.

## Best Possible Solution

Land one shared silent-reply classifier in the normalization/delivery path with regression coverage for reasoning tags, bare think preambles, whitespace-delimited and glued final tokens, and existing substantive-message cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a final payload with non-empty preamble plus trailing `NO_REPLY` normalizes to the preamble instead of `null`, and a glued `here.NO_REPLY` suffix is not stripped.

## Solution Assessment

Unclear. Current main is not the solution, and the open PRs choose different suppression policies; the maintainable fix should centralize classification with an explicit maintainer decision on trailing-token semantics.
