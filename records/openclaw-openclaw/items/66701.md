Keep this issue open. Current main still preserves non-empty reasoning or preamble text before a trailing whitespace-delimited NO_REPLY token, and the linked open PRs are implementation candidates rather than grounds to close the tracker before merge.

## What This Changes

The issue requests tolerant silent-reply suppression when models prepend reasoning or think text before a final NO_REPLY token.

## Best Possible Solution

Land one central silent-reply classifier used by normalization and delivery paths, with regression coverage for bare think, <think> blocks, whitespace-delimited trailing tokens, glued final-token examples, and existing substantive-message behavior.
