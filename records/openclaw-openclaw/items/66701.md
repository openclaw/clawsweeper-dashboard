## Summary

Keep this issue open. Current main still preserves reasoning or preamble text after stripping a trailing whitespace-delimited silent token, and glued suffixes like the reported final-token example remain non-matches. The linked open PRs are implementation candidates, so the issue should remain the canonical tracker until maintainers choose and land one suppression policy.

## What This Changes

The issue requests tolerant NO_REPLY suppression when models prepend reasoning or think blocks before the silent token, including final-token and glued-suffix forms that currently reach channel delivery.

## Best Possible Solution

Keep this issue as the canonical tracker and land one central silent-reply classifier used by normalization, final delivery, and any streaming/block incomplete-turn paths, with regression coverage for bare think, <think> blocks, whitespace-delimited trailing tokens, glued final-token examples, and existing intentional substantive-message behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the normalizer test expects a preamble before trailing NO_REPLY to remain deliverable, and the token test expects a glued suffix like interject.NO_REPLY to remain unchanged; both feed the shared reply dispatcher path.

## Solution Assessment

Unclear. The current implementation is not the solution, and the open PRs choose different policies; the maintainable fix should centralize the classifier rather than adding per-surface ad hoc checks, while preserving whichever non-silent trailing-token behavior maintainers approve.
