## Summary

Keep this open: current main still keeps non-empty preamble text before a trailing or glued `NO_REPLY`, and the open closing PRs are implementation candidates rather than grounds to close the tracker before merge.

## What This Changes

The issue asks for shared `NO_REPLY` suppression to tolerate model-emitted reasoning or think preambles and final-token silent replies without posting the preamble to group channels.

## Best Possible Solution

Land one central silent-reply classifier in the shared normalization/delivery path, with regression coverage for reasoning tags, bare think preambles, whitespace-delimited and glued final tokens, and existing substantive-message cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a final payload with non-empty preamble plus trailing `NO_REPLY` normalizes to the preamble instead of `null`, and a glued `here.NO_REPLY` suffix is not stripped.

## Solution Assessment

Unclear. Current main is not the solution, and the open PRs choose different suppression policies; the maintainable fix should be a shared classifier with an explicit maintainer decision on trailing-token semantics.
