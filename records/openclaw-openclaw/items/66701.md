## Summary

Keep open: current main still strips a trailing silent token and can deliver the non-empty reasoning preamble, while live GitHub shows open closing PRs rather than a merged fix.

## What This Changes

The issue requests suppressing model outputs where reasoning or think preamble text precedes a final `NO_REPLY` token so internal reasoning is not posted to chat.

## Best Possible Solution

Land one canonical shared silent-reply classifier with regression coverage for reasoning wrappers, glued/final tokens, and substantive-message false positives, then close this issue after the fix merges.

## Reproduction Assessment

Yes. Source inspection on current main shows a non-empty preamble before `NO_REPLY` normalizes to deliverable text, and current tests still assert delivery for text before a trailing silent token.

## Solution Assessment

Unclear. Current main is not the solution, and the open PRs choose different suppression policies; the maintainable fix should centralize the classifier after maintainer selection of the intended semantics.
