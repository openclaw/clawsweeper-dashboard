## Summary

Keep open: current main and v2026.5.18 still classify only exact silent-token payloads as silent, so a reasoning-prefixed payload can be normalized into deliverable preamble text; https://github.com/openclaw/openclaw/pull/80930 is an open closing PR with supplied dispatcher proof but it has not merged.

## What This Changes

The issue requests suppressing `think`/reasoning-prefixed `NO_REPLY` outputs so raw model reasoning is not posted to group channels.

## Best Possible Solution

Review and land one canonical fix, preferably the focused shared classifier path in https://github.com/openclaw/openclaw/pull/80930 if maintainers accept that boundary, then close this issue after merge and shipped proof.

## Reproduction Assessment

Yes, at source level. Current main still exact-matches only bare silent-token payloads and then delivers non-empty preamble text after stripping a trailing `NO_REPLY`; I did not run tests because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the suppression boundary. A narrow reasoning-prefixed classifier looks maintainable, but broad final-token suppression would conflict with the existing substantive-reply tests and older false-positive fix.
