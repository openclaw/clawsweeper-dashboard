## What This Changes

The issue requests a WebUI option to buffer streamed assistant deltas and display the full response at once instead of rendering a typewriter-style stream.

## Best Possible Solution

Add a browser-local WebChat display preference that defaults to current streaming, buffers assistant text deltas until the terminal event, keeps clear reading/final/error behavior, and includes focused UI tests and docs; avoid gateway config unless maintainers explicitly want server-side policy.
