## Summary

Keep open: current main and the latest v2026.6.8 release still render WebChat assistant deltas as live stream content, and the requested instant-display behavior is a new WebChat UX preference that needs maintainer product/settings-surface approval rather than conservative cleanup closure.

## What This Changes

The issue requests an opt-in WebChat setting to buffer assistant stream deltas and render the final response at once instead of live typewriter-style text.

## Best Possible Solution

Add a browser-local WebChat display preference, leave live streaming as the default, buffer assistant text deltas until the terminal event in instant mode, and cover reading indicators, final/error/abort fallback, markdown finalization, i18n, and focused UI tests.

## Reproduction Assessment

Not applicable as a bug repro; this is a feature request. Source inspection still confirms the premise: current WebChat appends deltas into `chatStream` and renders the active stream during the run.

## Solution Assessment

Unclear until maintainers choose the UX surface. The narrowest maintainable path appears to be a browser-local Control UI setting rather than a new gateway config key, but that product choice should precede implementation.
