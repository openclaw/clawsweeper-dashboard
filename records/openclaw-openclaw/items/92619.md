## Summary

Keep open: current main and v2026.6.6 still render WebChat assistant deltas as live streaming text, and there is no WebChat rendering-mode setting. The request fits Web frontend ergonomics, but the exact preference surface and terminal/abort behavior need maintainer product direction first.

## What This Changes

Add an opt-in WebUI/WebChat setting that buffers streaming assistant deltas and renders the response only when the turn completes.

## Best Possible Solution

Add a browser-local WebChat rendering preference, likely in Appearance, that preserves streaming as the default and buffers only the client-side display path unless maintainers explicitly choose a broader config contract.

## Reproduction Assessment

Not applicable as a bug reproduction, but the current behavior is source-reproducible: WebChat appends each delta into chatStream and renders it as an active streaming item. I did not run a live browser repro because this read-only review only needed current source and existing tests to confirm the behavior.

## Solution Assessment

Unclear until maintainers choose the product surface. The narrowest maintainable direction appears to be a browser-local Appearance preference, not a new Gateway config key, because WebChat currently has no persisted config section.
