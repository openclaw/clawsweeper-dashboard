## Summary

Keep open: current main and the latest release still render WebChat assistant deltas live, and the related auto-scroll selector does not provide an instant-display mode. This is a WebChat UX preference that needs maintainer product direction before implementation.

## What This Changes

Requests an opt-in WebChat setting or mode that buffers assistant streaming deltas and renders the completed response at once instead of token-by-token.

## Best Possible Solution

If maintainers accept the UX, add a browser-local WebChat display preference that keeps live streaming as the default, buffers assistant text deltas until final completion, and preserves indicators plus final/error reconciliation.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection confirms the current premise: WebChat applies delta events to live stream state and renders that stream during the assistant turn.

## Solution Assessment

Unclear until maintainers choose the UX surface. The narrowest maintainable path appears to be a browser-local Control UI setting rather than a new gateway config key, but the buffering semantics still need product direction.
