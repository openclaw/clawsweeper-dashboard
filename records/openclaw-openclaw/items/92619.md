## Summary

Keep open: current main and v2026.6.8 still update WebChat from live assistant deltas with no instant-display preference, and the requested toggle is a new Web UI UX/settings decision rather than conservative cleanup.

## What This Changes

The issue requests an opt-in Web UI setting or mode that buffers assistant streaming deltas and renders the full response at turn completion.

## Best Possible Solution

Accept the UX only after maintainer direction, then add a browser-local WebChat display preference that keeps streaming as default, buffers assistant text deltas until terminal events, preserves progress indicators/finalization paths, and covers UI tests plus i18n/docs.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection does confirm the premise: current main processes WebChat deltas into `chatStream` and renders active stream bubbles during the turn.

## Solution Assessment

Unclear until maintainers choose the UX surface. A browser-local Control UI preference appears narrower than a new gateway config key because WebChat currently has no persisted config section.
