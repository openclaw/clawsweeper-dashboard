Request adds an opt-in WebChat/WebUI setting or config path to buffer streamed assistant text and render it only when the turn finishes.

## Best Possible Solution

Add a browser-local WebChat display preference that leaves streaming as the default, buffers only assistant text deltas until the terminal event, and covers reading indicators, final/error fallback, markdown finalization, i18n, and focused UI tests.

## Reproduction Assessment

Not applicable as a bug repro. Source inspection confirms the current behavior premise: current main processes chat delta events into `chatStream` and renders active stream bubbles during the turn.

## Solution Assessment

Unclear until maintainers choose the UX surface. The narrowest maintainable path appears to be a browser-local Control UI setting rather than a new gateway config key, but that product choice should be made before implementation.
