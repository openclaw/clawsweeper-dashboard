## Summary

Keep open: this fixes a real usage-cost blind spot and the latest push addresses cache invalidation plus operator-configured zero pricing, but the patch still changes generated all-zero catalog pricing into missing-cost entries for providers where zero is the intended free/local cost.

## What This Changes

The PR updates usage-cost transcript scanning and tests so unconfigured all-zero model pricing with token usage is reported as missingCostEntries, adds config-only pricing detection, and bumps the durable cache version.

## Best Possible Solution

Keep the cache bump and Codex missing-cost behavior, but make zero-cost classification source-aware so generated local/free provider catalogs remain complete $0 while Codex unknown pricing surfaces as missing.

## Reproduction Assessment

Yes, source-reproducible with high confidence: current main preserves transcript cost.total: 0 unless pricing is tiered or missing, and the PR body supplies before/after live CLI output from a 2026.5.20 host. I did not run commands that write artifacts because this review is read-only.

## Solution Assessment

No, not yet: the direction is right for Codex unknown pricing, but treating every unconfigured generated all-zero price as unknown is not the safest maintainable boundary. The fix should carry an explicit pricing source/status or otherwise preserve generated free/local provider zeros.
