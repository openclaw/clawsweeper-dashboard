## Summary

Keep this PR open: the bug fix direction is useful and the real-host proof is credible, but source inspection and CI show the patch clears positive transcript-recorded costs whenever local pricing is absent, which would underreport known spend.

## What This Changes

The PR adds a usage-cost pricing-known helper and regression test so tokenized turns with all-zero resolved pricing are counted as missing-cost entries instead of confident zero spend.

## Best Possible Solution

Keep the missing-cost signal for generated all-zero unknown pricing, preserve positive transcript-recorded costs, and either source-scope the zero-pricing override or define an explicit unknown-pricing marker.

## Reproduction Assessment

Yes. Source inspection and the PR CI reproduce the patch defect: tokenized transcript entries with positive recorded `usage.cost.total` and no resolved local pricing now lose that cost, with 14 session-cost tests failing.

## Solution Assessment

No. The direction is right, but the implementation should only drop fabricated zero costs tied to a resolved all-zero pricing row, not every absent-pricing case or positive transport-recorded cost.
