## Summary

Keep open: the bug report and live proof are useful, but the patch currently turns provider-reported positive transcript costs into missing entries whenever no local pricing row resolves, which would regress usage-cost aggregation before merge.

## What This Changes

The PR adds zero-priced model detection in usage-cost transcript scanning and a regression test so tokenized turns with all-zero resolved pricing are surfaced as missing-cost entries instead of confident zero spend.

## Best Possible Solution

Keep the missing-cost signal for generated all-zero unknown pricing, but preserve positive transcript-reported costs and either source-scope the zero-pricing override or define an explicit unknown-pricing marker so intentional free models remain distinguishable.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction for the patch defect: a transcript entry with tokens, a positive `usage.cost.total`, and no resolved local pricing will have its cost cleared by the new branch instead of preserved.

## Solution Assessment

No. The direction is right, but the implementation should only drop fabricated zero costs tied to a resolved all-zero pricing row, not every absent-pricing case or positive transport-reported cost.
