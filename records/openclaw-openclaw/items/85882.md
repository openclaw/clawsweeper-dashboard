## Summary

Keep this PR open: the reported spend-blind spot is real and the supplied proof is useful, but the latest patch still has merge-blocking compatibility issues around durable cache invalidation and generated zero-cost provider catalogs.

## What This Changes

The PR changes usage-cost transcript scanning so tokenized turns with all-zero unconfigured pricing are counted as missing-cost entries, while explicitly configured zero-cost models remain complete $0 entries.

## Best Possible Solution

Define a source-aware pricing contract that distinguishes unknown pricing from intentionally free generated zeros, invalidate existing usage-cost caches, and land regression coverage for Codex unknown pricing, configured free pricing, generated free/local pricing, and cache upgrade behavior.

## Reproduction Assessment

Yes. Current main and v2026.5.22 source show recorded cost.total: 0 is treated as complete cost data, and the PR body supplies before/after live terminal output for the same logged usage; I did not run tests because this review is read-only.

## Solution Assessment

No. The direction is right for Codex-like unknown pricing, but the patch needs cache invalidation and a source-aware zero/free pricing contract before it is the narrow maintainable fix.
