## Summary

Keep open: the updated head fixes the earlier positive-recorded-cost regression, but it still changes the meaning of every all-zero/tierless model cost row. Current docs and defaults use all-zero cost for local/custom providers, so this needs a pricing-contract decision or source-aware fix before merge.

## What This Changes

This PR changes usage-cost transcript scanning so tokenized turns with all-zero resolved pricing are counted as missing-cost entries, and adds a regression test for the codex/gpt-5.5 zero-price case.

## Best Possible Solution

Preserve documented zero-cost configs as complete cost data unless a source-aware or explicit unknown-pricing marker identifies the catalog row as unknown, then add tests for both codex unknown pricing and intentional local/free zero pricing.

## Reproduction Assessment

Yes, by source inspection: current main parses transcript cost.total: 0 as a cost breakdown and never increments missingCostEntries, while the PR body supplies live before/after output for the codex/gpt-5.5 case. I did not run a local repro because this review is read-only.

## Solution Assessment

No, not yet: the patch addresses the codex zero-price symptom, but the current implementation also reclassifies documented intentional zero-cost configs as unknown. The safer fix needs an explicit or source-aware unknown-pricing distinction.
