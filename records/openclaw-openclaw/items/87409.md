## Summary

Keep open. The write-lock ownership move matches current agent-core listener behavior, but the PR needs maintainer review before merge because it exposes an internal lock runner through the public session SDK and the supplied proof is test-only.

## What This Changes

The PR moves embedded session write-lock fencing into createAgentSession/AgentSession, passes the embedded attempt lock into session creation, and deletes the obsolete private session hook wrappers.

## Best Possible Solution

Keep the owned-session lock seam, but hide or explicitly approve the public SDK surface before merge and add real embedded-run proof for the changed concurrency path.

## Reproduction Assessment

Yes, source inspection shows the relevant path: embedded attempts release the session lock around provider streaming, then session writes and write-capable hooks need to reacquire/fence before transcript mutation or tool execution. I did not run the PR tests in this read-only review.

## Solution Assessment

Mostly, but not as submitted. Moving the lock seam into AgentSession is the right ownership direction; exposing that lock runner through the public session SDK needs maintainer approval or an internal-only shape.
