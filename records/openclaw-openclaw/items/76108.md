## Summary

Keep open: current main still implements and documents sticky auto-fallback after cooldown, so the report remains valid and needs a maintainer policy decision rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to retry the configured primary model after its cooldown expires instead of keeping the last successful fallback across sessions.

## Best Possible Solution

Define the intended recovery policy for auto-sourced model overrides after auth cooldown expiry, then align candidate ordering, session inheritance/reset behavior, diagnostics, docs, and focused tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to seed a session or parent session with an auto fallback override and an expired primary auth cooldown; current selection keeps the fallback and the fallback runner tries it before the configured primary.

## Solution Assessment

Unclear. Retrying the primary first after cooldown would address the report, but it conflicts with current documented behavior; a cooldown-aware or configurable recovery policy is likely safer.
