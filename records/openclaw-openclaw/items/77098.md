## Summary

This PR should stay open because the helper-only change does not fix the loader restore path described by the linked report and changes same-plugin corpus registration semantics to first-wins without a contract update.

## What This Changes

The PR changes `registerMemoryCorpusSupplement` from replacing an existing same-plugin corpus supplement to keeping the first registration and ignoring later ones.

## Best Possible Solution

Fix the loader state restoration or service registration lifecycle so lazy non-activating loads preserve active corpus supplements, then cover it with a regression test and changelog entry.

## Reproduction Assessment

No high-confidence runtime reproduction was run here. Source inspection does show the proposed helper change cannot reliably affect the loader snapshot/restore path that the linked live trace points to.

## Solution Assessment

No. The narrow maintainable fix should target the loader restore or service/runtime registration boundary that drops active supplement state, not make all same-plugin corpus supplement registrations first-wins.
