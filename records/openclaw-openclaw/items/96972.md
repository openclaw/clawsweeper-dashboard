## Summary

Keep this PR open, but it still needs changes before merge: the memory cap is useful, yet the guard is placed before complete SSE events are drained, so a valid coalesced transport chunk over 64 KiB can fail as a protocol error.

## What This Changes

The PR adds a 64 KiB OpenAI/Codex SSE parser buffer cap and a regression test for a delimiter-free oversized SSE response.

## Best Possible Solution

Keep the overflow guard, but drain complete SSE events first and enforce the cap only on the remaining unterminated buffer tail, with regression coverage for malformed and coalesced-valid chunks.

## Reproduction Assessment

Yes by source inspection. Current main retains an unterminated SSE buffer until a \n\n boundary appears, and the PR-introduced coalesced-chunk failure is source-reproducible from the new guard running before the drain loop.

## Solution Assessment

No. The cap is the right mitigation, but as written it is only a plausible fix; the best narrow fix is to process complete events first and cap only the leftover unterminated buffer.
