- `merge-risk: 🚨 availability`: The changed parser can fail a valid streaming response before processing complete SSE events, making OpenAI/Codex streaming unavailable for that request.

## Summary

Keep this PR open: the OOM hardening is useful, but the current guard can reject valid SSE streams when a transport chunk contains more than 64 KiB of already-delimited events before the parser drains them.

## What This Changes

The PR adds a 64 KiB guard to the OpenAI/Codex SSE parser buffer and a focused test for an oversized delimiter-free SSE response.

## Best Possible Solution

Keep the buffer bound, but drain complete SSE events first and enforce the cap only on the remaining unterminated buffer tail, with regression coverage for both malformed and coalesced-valid chunks.

## Reproduction Assessment

Yes. Source inspection shows current main keeps appending to `buffer` until a `\n\n` boundary appears, and the PR-introduced regression is also source-reproducible with one large chunk containing many complete small SSE events.

## Solution Assessment

No. The guard is the right kind of fix, but it is in the wrong spot; the narrow maintainable solution is to process complete events first and cap only the leftover unterminated buffer.
