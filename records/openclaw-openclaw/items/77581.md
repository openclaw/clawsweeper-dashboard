## Summary

Keep open: current main still renders the empty thinking override as `Default (...)` or `inherit`, and lightweight `sessions.list` rows do not carry the resolved inherited row default, so the Control UI cannot reliably show the effective inherited thinking level.

## What This Changes

Request to make Control UI thinking dropdowns distinguish a blank per-session override from the effective inherited/default thinking level.

## Best Possible Solution

Keep the runtime behavior unchanged, expose or compute the effective thinking level for Control UI rows, and render blank overrides as inherited/effective labels while preserving `value=""` as clear override.

## Reproduction Assessment

Yes, source-reproducible: the async `sessions.list` lightweight path omits the resolved row default, and the two UI dropdown renderers label the empty override as `Default (...)` or `inherit`. I did not live-run the UI because `pnpm` is unavailable in this checkout.

## Solution Assessment

Yes, the narrow maintainable fix is UI/API-row labeling only: compute the effective inherited thinking level and display Inherited/Override/Off semantics without changing `sessions.patch` or reply execution.
