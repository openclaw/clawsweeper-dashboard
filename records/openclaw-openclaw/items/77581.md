## Summary

Keep open. Current main still treats a blank per-session thinking override as a generic default/inherit option in the Control UI, while the resolved runtime default is computed elsewhere; the async sessions list also omits resolved row defaults for lightweight rows, so the UI does not have enough row-level metadata to consistently show the effective inherited level.

## What This Changes

The issue requests Control UI thinking selects to display inherited/effective thinking levels instead of labeling blank per-session overrides as `Default/off`.

## Best Possible Solution

Expose or compute the resolved effective thinking default for session rows and use a shared Control UI label helper so blank, explicit non-off, and off states are displayed consistently without changing runtime execution or `sessions.patch` semantics.

## Reproduction Assessment

Yes, from source inspection: the blank override option is rendered as `Default (...)` or `inherit`, while lightweight session rows can omit the resolved effective default needed by the UI. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

Yes. A narrow UI plus sessions-list metadata fix is the maintainable path; the runtime resolver already exists, so the fix should not change reply execution, routing, provider policy, or patch semantics.
