## Summary

Current main and the latest release already fix the reported false `Default (off)` label by resolving session thinking defaults through the same config-aware path used by runtime, then passing that effective value to the Control UI.

## What This Changes

The issue asks the Control UI thinking dropdowns to show the effective inherited thinking level instead of treating a blank per-session override as `Default (off)`.

## Best Possible Solution

Keep the shipped config-aware default-resolution path; any future preference for `Inherited:`/`Override:` copy should be handled as a separate narrow UX wording change.

## Reproduction Assessment

No. Against current main, the source path and regression tests show configured inherited defaults reach the UI as `Default (high)` or the resolved configured value, so the reported `Default (off)` state no longer matches current behavior.

## Solution Assessment

Yes. The shipped fix addresses the misleading effective-state bug at the gateway/session-row boundary while preserving `value=""` as clear per-session override and avoiding runtime behavior changes.
