## Summary

Current main already fixes the reported stale `Default (off)` display by resolving and rendering the configured thinking default in both chat and session thinking controls; the remaining `Inherited:`/`Override:` wording would be a separate UX naming change, not the original off-state bug.

## What This Changes

The issue requests clearer Control UI thinking dropdown labels for sessions that inherit an effective thinking default instead of storing a per-session override.

## Best Possible Solution

Keep the current main behavior that labels no session override as `Default (<resolved level>)` using gateway-provided thinking metadata, and treat any `Inherited:`/`Override:` copy change as a separate UX decision.

## Reproduction Assessment

No. Current-main source and UI tests show inherited thinking defaults render as `Default (high)` or `Default (adaptive)`, not `Default (off)`, when the gateway exposes the resolved default.

## Solution Assessment

Yes. The implemented path is narrow and maintainable: gateway rows/defaults expose `thinkingDefault`, and the Control UI uses that display metadata without changing `sessions.patch` or runtime thinking execution.
