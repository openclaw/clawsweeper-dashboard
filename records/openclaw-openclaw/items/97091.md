## Summary

Keep open: current main and the latest release still allow tool-call blocks from a length-truncated assistant message to reach execution, and the open linked fix PR should be reviewed under the protected maintainer label.

## What This Changes

The issue reports that a max-token-truncated streamed assistant turn can commit and execute an incomplete `sessions_spawn` tool call, launching an unintended child session.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/97092 if clean, or replace it with an equivalent focused agent-core fix that only executes finalized `toolUse` turns and strips non-executable tool-call blocks from committed and replayed assistant messages.

## Reproduction Assessment

Yes, source-reproducible: construct a final assistant message with `stopReason: "length"` and a `toolCall` block, and current main passes it through the existing stop-reason check into `executeToolCalls`. I did not run tests because this review is read-only.

## Solution Assessment

Yes. The best fix is the narrow agent-core boundary already represented by the linked PR: only finalized `toolUse` turns should dispatch tools, and non-executable tool-call blocks should not be committed to `message_end` or replay context.
