## Summary

Keep open: current `main` still lets a `length` assistant message with a `toolCall` reach execution, and the issue has both a protected `maintainer` label and an open linked fix PR that should resolve it before closure.

## What This Changes

The issue reports that a length-truncated streamed assistant turn can commit and execute a partial `sessions_spawn` tool call, creating an unintended child session.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/97092 if the PR review stays clean, or replace it with an equivalent focused agent-core fix that only executes finalized `toolUse` turns and strips non-executable tool-call blocks from committed/replayed assistant messages.

## Reproduction Assessment

Yes, source-reproducible: construct an assistant message with `stopReason: "length"` and a `toolCall` block, and current `main` passes it through the existing stop-reason check into `executeToolCalls`. I did not run tests because this review is read-only.

## Solution Assessment

Yes. The best fix is the narrow agent-core boundary shown by the linked PR: only finalized `toolUse` turns should dispatch tools, and non-executable tool-call blocks should not be committed to `message_end` or replay context.
