Keep open: current main and the latest release still allow a length-truncated assistant message carrying tool-call blocks to reach agent-core tool execution, and the open linked PR is the active fix path under a protected maintainer label.

## What This Changes

The issue reports that a max-token-truncated streamed assistant turn can commit and execute an unintended partial `sessions_spawn` tool call.

## Best Possible Solution

Merge a focused agent-core guard that executes only finalized `toolUse` turns and removes non-executable tool-call blocks from committed and replayed assistant messages.
