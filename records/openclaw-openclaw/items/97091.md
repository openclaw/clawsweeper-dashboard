## Summary

Keep open: current main and latest release still allow a length-truncated assistant message carrying tool-call blocks to reach agent-core tool execution, and the open linked PR is the active fix path under a protected maintainer label.

## What This Changes

The issue reports that a streamed assistant turn ending at the output limit can commit and execute an incomplete `sessions_spawn` tool call.

## Best Possible Solution

Merge a focused agent-core fix that executes only finalized `toolUse` assistant turns and strips non-executable tool-call blocks before commit/replay.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: return a final assistant message with `stopReason: "length"` and a `toolCall` block, and current main routes it into `executeToolCalls`.

## Solution Assessment

Yes for the linked direction. Agent-core owns tool execution and replay context, so gating execution there is the narrow maintainable fix; provider-specific cleanup would miss other adapters producing the same terminal shape.
