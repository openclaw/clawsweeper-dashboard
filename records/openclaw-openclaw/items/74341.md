## Summary

## What This Changes

The PR adds support checks for forced/session-pinned agent harness selection, new unsupported-provider tests, broader Codex harness provider IDs, a status-test helper update, and a changelog entry.

## Best Possible Solution

Land the selector support check after rebasing, with explicit/session-pinned plugin runtimes throwing on unsupported candidates and Codex OpenAI-family support gated to explicit Codex runtime requests.

## Reproduction Assessment

Yes. Current main source shows the forced-plugin branch returns the pinned harness without calling `supports()`, and the PR regression is source-reproducible against the existing auto-mode docs/test boundary.

## Solution Assessment

No as written. The selector-side support check is the right narrow fix, but the Codex provider widening needs runtime gating and the unsupported explicit-runtime path should follow current main's fail-closed contract.
