## Summary

Keep open. ## What This Changes

The branch validates forced plugin-harness support before selecting Codex, broadens the Codex harness default provider IDs, adds harness selection/Codex support tests, updates a status-test helper, and adds a changelog entry.

## Best Possible Solution

Keep the forced `supports()` guard, make `openai`/`openai-codex` support conditional on explicit/requested Codex runtime contexts, preserve auto-mode proxy behavior, update both existing and new contract tests, and add redacted patched-gateway proof.

## Solution Assessment

No. The forced `supports()` guard is the right narrow fix, but unconditionally broadening the Codex default provider IDs is not the safest implementation because it breaks the existing auto-mode provider boundary.
