## Summary

Keep open: current main still builds Feishu group session identity from the chat-scoped group peer, so multiple Feishu app accounts in the same group can share the same persisted session despite account-aware routing inputs.

## What This Changes

The issue requests Feishu group sessions to distinguish multiple bot apps in the same group by app/account identity, not only by group chat ID.

## Best Possible Solution

Implement Feishu-owned account-aware group session identity with focused regression coverage while preserving legacy single-account behavior where feasible.

## Reproduction Assessment

Yes, source-backed rather than live: the reporter's same-group multi-app steps map directly to current main because Feishu group peers are chat-scoped and non-direct session-key construction ignores `accountId`. A real Feishu multi-app group was not run during this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to make Feishu group session/routing identity account-aware, not to rewrite every channel's group session-key contract globally.
