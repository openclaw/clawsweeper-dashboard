## Summary

Keep open. Current main still carries the Feishu account into routing, but default Feishu group sessions and core non-direct session keys remain group-only, so multiple Feishu bot accounts that resolve to the same agent in one group can still share session state and last-route identity.

## What This Changes

The issue requests account/app-aware Feishu group session identity for multiple Feishu bot applications in the same group chat.

## Best Possible Solution

Design an account-aware Feishu group-session boundary that preserves existing single-account behavior, chooses a stable account/app key and migration story, and adds regression coverage for same-group multi-account bots.

## Reproduction Assessment

Yes, at source level. Two Feishu accounts that resolve to the same agent and default group peer still build the same group session key because `accountId` reaches routing but is omitted from non-direct session keys.

## Solution Assessment

Unclear. The requested app/account suffix is directionally right, but the maintainable fix needs an explicit account-aware key shape and compatibility plan before changing persisted group sessions.
