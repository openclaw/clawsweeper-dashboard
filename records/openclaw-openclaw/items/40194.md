## Summary

Keep open. Current main still keys Feishu group sessions by agent, channel, peer kind, and group peer only; existing multi-account and mention-target fixes do not isolate multiple Feishu bot accounts sharing the same group and agent.

## What This Changes

The issue requests Feishu group session isolation by bot application/account so multiple Feishu robots in one group do not share one OpenClaw session.

## Best Possible Solution

Design an account-aware Feishu group session boundary that preserves single-account behavior, chooses the accountId/appId key and migration story, and adds regression coverage for multiple Feishu accounts in one group.

## Reproduction Assessment

Yes, at source level. Two Feishu accounts that resolve to the same agent and default group peer still build the same group session key because accountId reaches route resolution but is not included in non-direct session keys.

## Solution Assessment

Unclear. The reported appId/account suffix is directionally right, but the maintainable fix needs an explicit OpenClaw account-aware key shape and compatibility plan before changing persisted group sessions.
