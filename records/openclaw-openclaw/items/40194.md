## Summary

Keep open: current main and v2026.6.10 still build Feishu group session identity from chat/topic/sender without a bot app or account component, so multiple Feishu bot apps in one group can still collide on the same durable session boundary.

## What This Changes

The issue reports Feishu group sessions colliding across multiple bot apps because group session identity is not scoped by bot app/account.

## Best Possible Solution

Add a maintainer-approved Feishu group session identity that includes a stable bot/account boundary, with upgrade-safe migration or doctor handling, regression tests, and live multi-bot Feishu proof.

No implemented fix is present. The best fix must choose a stable Feishu bot discriminator and an upgrade-safe migration or doctor plan before changing durable group session keys.
