## Summary

Keep open: current main still routes Feishu group sessions with a group-only peer key, so multiple Feishu apps in the same chat can converge on one session and overwrite account-specific delivery context.

## What This Changes

Request account-aware Feishu group session routing so different Feishu apps in the same group keep separate bot identities and sessions.

## Best Possible Solution

Implement a Feishu-owned account-aware group routing path with focused regression coverage, docs, and changelog while preserving legacy single-account sessions where feasible.

## Reproduction Assessment

Yes. The reporter's same-group three-app steps map directly to current source: Feishu group peer IDs are chat-scoped and the non-direct session-key builder ignores accountId.

## Solution Assessment

Yes. The narrow maintainable fix is to make Feishu group session routing account-aware, rather than rewriting all channel group key shapes.
