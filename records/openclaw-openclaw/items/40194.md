## Summary

Keep open: current main still lets Feishu group routing collapse multiple bot accounts in the same group onto the same group session key unless they route to different agents or a narrower group-session scope is configured.

## What This Changes

The issue asks Feishu group sessions to distinguish multiple bot apps in the same group so each mention uses the matching account identity.

## Best Possible Solution

Define an account-aware Feishu group-session contract, preserve legacy group transcript lookup where needed, and add a regression test with two Feishu accounts in one group.

## Reproduction Assessment

Yes, from source inspection: two Feishu accounts in the same group with the same agent/default group scope both resolve to the same group session key. I did not run a live Feishu setup.

## Solution Assessment

Unclear. The reported appId suffix captures the right isolation need, but the maintainable fix should use OpenClaw's account/session contract and handle legacy group session keys deliberately.
