## Summary

Current main still has a source-level Feishu group-session collision path for multiple accounts that route to the same agent and group peer, so this should not be swept closed.

## What This Changes

The issue reports that multiple Feishu bots in the same group can share one group session and reply with the wrong bot identity.

## Best Possible Solution

Keep the issue open and design an account-aware Feishu group session boundary that preserves existing single-account routing and includes regression coverage for multiple Feishu accounts in one group.

## Reproduction Assessment

Yes at source level, but not via a live Feishu run. Two Feishu accounts that resolve to the same agent and group peer still build the same group session key because the accountId is not included for non-direct peers.

## Solution Assessment

Unclear. The reported appId suffix captures the right isolation requirement, but the maintainable fix should choose an account/app-aware key shape that fits OpenClaw routing, docs, and migration behavior.
