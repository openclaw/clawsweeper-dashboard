## Summary

Keep open. Current main has the merged service-environment guard, but it still does not compare the active Gateway runtime PID with the caller's self/ancestor PID set, and open PR #75819 is the linked closing implementation candidate.

## What This Changes

Request `openclaw update` to detect when it is invoked from inside the active Gateway process tree and fail clearly before the update/restart path can silently leave old code running.

## Best Possible Solution

Land PR #75819 or an equivalent conservative updater preflight that reads the active managed Gateway runtime PID, walks the caller PID ancestry, and aborts before service stop or package mutation when the Gateway is an ancestor.

## Reproduction Assessment

Yes. The report gives concrete systemd plus agent-exec steps, and current main still lacks the active runtime PID versus caller-ancestry comparison that would block that context without service marker env.

## Solution Assessment

Yes. A defensive ancestry preflight before update side effects is the narrowest maintainable fix; reusing or exporting the existing bounded walker is safer than adding a detached restart helper first.
