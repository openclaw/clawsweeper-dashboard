## Summary

Keep open. Current main still lacks the requested active-gateway-PID ancestry guard, and the issue is already tied to open fix work rather than something ClawSweeper should close early.

## What This Changes

Request that `openclaw update` detect when it is invoked from a process whose ancestor chain includes the active gateway PID and abort or defer the restart instead of silently leaving the old gateway running.

## Best Possible Solution

Add a conservative updater preflight that reads the active managed gateway runtime PID, walks the caller ancestry, and aborts before package replacement with clear external-shell and `--no-restart` guidance.

## Reproduction Assessment

Yes. The report gives concrete systemd and agent-exec steps, and current source confirms there is no runtime-PID ancestry comparison in the package-update path.

## Solution Assessment

Yes. The defensive refusal approach is the narrowest maintainable fix; current main is incomplete, so the best route is to review and land or close the active linked PRs.
