## Summary

Keep open. Current main still lacks the requested active-Gateway-PID ancestry guard, and this issue is paired with open implementation PR #75819 plus related updater-safety PR #75729.

## What This Changes

The issue asks `openclaw update` to detect when a package update is launched from inside the active Gateway process ancestry and fail clearly before the restart/package-mutation path can silently leave old code running.

## Best Possible Solution

Land or resolve #75729 first, then rebase and review #75819 while keeping a narrow fail-closed ancestry preflight before service stop or package replacement, with the required changelog and targeted updater/stale-PID tests.

## Reproduction Assessment

Yes. The issue gives concrete systemd plus Gateway-owned exec steps, PR #75819 comments include no-marker live proof, and current main source shows no active runtime-PID ancestry comparison.

## Solution Assessment

Yes. The defensive refusal before service stop/package mutation is the narrowest maintainable fix for this failure mode; a detached restart helper is a larger separate design.
