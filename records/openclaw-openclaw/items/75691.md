## Summary

Keep open. Current main still only blocks package updates when gateway service marker environment variables are present; it does not compare the active Gateway runtime PID with the caller ancestry, and open PR #75819 already targets that missing guard.

## What This Changes

Request a package-update preflight that detects when `openclaw update` is invoked from inside the active Gateway process tree and refuses or defers restart to avoid a silent stale Gateway.

## Best Possible Solution

Land one conservative updater preflight that reads the active managed Gateway runtime PID, walks caller ancestry, and aborts before package replacement with clear external-shell and `--no-restart` guidance.

## Reproduction Assessment

Yes. The issue gives concrete systemd and agent-exec steps, and current source confirms the package-update path lacks a runtime-PID ancestry comparison.

## Solution Assessment

Yes. The defensive refusal approach is the narrowest maintainable fix; the best route is to review and land or close the active linked PRs rather than create another branch.
