## Summary

Keep open. Current main still lacks the requested active-gateway-PID ancestry guard; it only blocks package updates when gateway service marker environment is inherited, and an open closing PR already targets this exact issue.

## What This Changes

Request `openclaw update` to detect when it is invoked from inside the active gateway process tree and fail clearly before package update/restart work.

## Best Possible Solution

Review and land a narrow preflight that compares the active managed gateway runtime PID with the caller ancestor chain before package replacement, with clear external-shell and `--no-restart` guidance.

## Reproduction Assessment

Yes. The issue gives concrete systemd/in-lane update steps, and current main confirms there is no active runtime-PID ancestry comparison in the package-update path.

## Solution Assessment

Yes. The defensive refusal approach is the narrowest maintainable fix for the silent restart failure; the open closing PR is the right path to review rather than creating a duplicate repair lane.
