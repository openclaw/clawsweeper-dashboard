## Summary

Keep open. Current main still does not implement the requested PPID-ancestry guard for `openclaw update`; it only blocks package updates when gateway service marker environment variables are inherited. The issue also has an open closing PR, so the right cleanup path is to review and land or reject that PR rather than close the issue early or queue duplicate automation.

## What This Changes

Add an `openclaw update` preflight that detects when the caller is inside the active gateway process tree and refuses or defers restartful package updates from that context.

## Best Possible Solution

Land a narrow updater preflight that reads the active managed gateway runtime PID, walks the caller's bounded ancestor chain, and fails before service stop or package replacement when the live gateway is an ancestor. Reuse or extract the existing ancestor-walk helper, keep the operator message explicit, and leave system-level restart-helper behavior to the separate systemd restart work.

## Reproduction Assessment

Yes. The issue gives concrete systemd reproduction steps, and current-main source inspection shows no updater comparison between active gateway runtime PID and caller PPID ancestry; this read-only review did not run a live systemd service repro.

## Solution Assessment

Yes for the defensive refusal direction. The narrow maintainable solution is a pre-package-update ancestry guard, and open PR #75819 is the natural implementation candidate to review against that exact requirement before closing the issue.
