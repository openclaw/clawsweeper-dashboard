## Summary

Keep open. Current main only blocks package updates when gateway service environment markers are inherited; it still does not compare the active gateway PID with the caller PPID ancestry, and open PR #75819 is the direct closing implementation candidate.

## What This Changes

Request `openclaw update` detect when it is invoked from inside the active gateway process tree and fail before a supervised restart can silently no-op.

## Best Possible Solution

Land a narrow updater preflight that reads the managed gateway runtime PID, walks the caller ancestry, and aborts before package replacement with external-shell and `--no-restart` guidance, preferably by reviewing and reconciling PR #75819 with the adjacent PR #75729 work.

## Reproduction Assessment

Yes. The report gives concrete systemd and agent-exec steps, and current source confirms the missing runtime-PID ancestry comparison in the package-update path.

## Solution Assessment

Yes. The defensive refusal approach is the narrowest maintainable fix because it prevents package mutation before the supervised restart can silently fail; auto-deferred systemctl restart would add more privilege and supervisor-policy surface.
