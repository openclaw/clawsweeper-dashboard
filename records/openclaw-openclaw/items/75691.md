## Summary

Keep open. Current main still lacks the requested active-gateway-PID ancestry guard, and this issue is already paired with open fix PRs rather than something ClawSweeper should close or duplicate.

## What This Changes

Request that `openclaw update` detect gateway-owned caller ancestry and fail before package update/restart can silently leave the old gateway process running.

## Best Possible Solution

Add a conservative updater preflight that reads the active managed gateway runtime PID, walks caller ancestry, and aborts before package replacement with clear external-shell and `--no-restart` guidance.

## Reproduction Assessment

Yes. The report gives concrete systemd and agent-exec steps, and current source confirms there is no runtime-PID ancestry comparison in the package-update path.

## Solution Assessment

Yes. The defensive refusal approach is the narrowest maintainable fix; current main is incomplete, so the best route is to review and land or close the active linked PRs.
