## Summary

Keep open. Current main has the merged service-environment guard from #75729, but it still does not compare the active Gateway runtime PID with the caller's self/ancestor PID set, and open PR #75819 is the linked implementation candidate for that missing guard.

## What This Changes

`openclaw update` should detect package updates launched from inside the active Gateway process tree and fail or defer before the restart path can silently leave old code running.

## Best Possible Solution

Add a conservative updater preflight that reads the active managed Gateway runtime PID, walks the caller self/ancestor PID set, and aborts with external-shell or `--no-restart` guidance before stopping the service or replacing the package tree.

## Reproduction Assessment

Yes. The report gives concrete systemd and agent-exec steps, and current main still lacks the active runtime PID versus caller-ancestry comparison that would block a gateway-owned child without service marker env.

## Solution Assessment

Yes. A defensive ancestry preflight is the narrowest maintainable solution; the best route is to review and land #75819 or an equivalent reconciled patch rather than close this issue early.
