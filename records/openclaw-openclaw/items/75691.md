## Summary

Keep open. Current main still lacks the requested active-gateway-PID ancestry guard: package updates are blocked only when the updater inherits gateway service marker environment variables, while the reported systemd child-process case can miss those markers. The issue is also paired with open closing PR #75819, so the right cleanup path is review and land or close that PR rather than closing this issue now.

## What This Changes

Request that `openclaw update` detect when it is invoked from inside the active gateway process tree and refuse or defer restartful package updates so supervised restarts cannot silently fail.

## Best Possible Solution

Review one canonical fix that reads the active managed gateway runtime PID, walks the caller's bounded ancestor chain, and refuses restartful package updates from inside that tree before package replacement or service stop. The implementation should reuse or extract the existing ancestor-walk helper and leave broader system-level restart-helper design to the separate systemd work.

## Reproduction Assessment

Yes. The issue gives concrete systemd and in-lane `exec` reproduction steps, and current-main source inspection shows the package-update path does not block the case where the active gateway PID is an ancestor but service marker env vars are absent.

## Solution Assessment

Yes for the defensive refusal direction. The narrow maintainable solution is a preflight before package mutation; PR #75819 appears closest to that requirement, while the env-marker-only path from #75729 does not fully cover this report by itself.
