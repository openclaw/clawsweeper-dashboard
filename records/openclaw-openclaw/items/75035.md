## Summary

Keep open: current main does not already implement `before_agent_run`, and the PR remains useful, but the current head still has a blocking privacy/security issue because the embedded runner logs raw hook block reasons verbatim.

## What This Changes

The PR adds a public `before_agent_run` pass/block hook before model submission, redacted blocked-turn persistence, diagnostics/docs/config updates, and focused hook/runtime tests.

## Best Possible Solution

Land the hook after keeping raw block reasons out of logs and docs, documenting only safe persisted metadata, and rerunning the focused hook/runtime plus generated contract checks.

## Reproduction Assessment

Yes for the PR finding: source review of the PR diff shows the embedded blocked path logging `beforeRunDecision.reason` verbatim. Not applicable for the feature request itself because current main has no `before_agent_run` behavior to reproduce.

## Solution Assessment

No as-is: placing the gate before image loading, `llm_input`, and model submission is the right runtime point, but raw internal reasons must not be logged or described as persisted metadata.
