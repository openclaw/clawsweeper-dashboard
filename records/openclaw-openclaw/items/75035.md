## Summary

Keep open: current main does not already contain this hook, and review found a privacy-sensitive blocker because embedded blocked runs log raw internal reasons that can contain prompt text.

## What This Changes

The PR adds a public `before_agent_run` pass/block hook, wires it into embedded and CLI runs before model submission, redacts blocked-turn persistence/live delivery, and updates diagnostics, docs, generated baselines, changelog, and tests.

## Best Possible Solution

Land a reason-safe public hook contract after maintainer API review, with raw reasons kept out of user-visible surfaces, transcripts, diagnostics, and general logs.

## Reproduction Assessment

Not applicable for the feature itself: current main lacks `before_agent_run`, so there is no existing broken behavior to reproduce. Source review gives a high-confidence path for the PR finding: a blocker can return a reason containing prompt text and the embedded runner logs it verbatim.

## Solution Assessment

No for merge as-is: placing the gate before image loading, `llm_input`, and model submission is the right runtime point, but the logging path must redact internal block reasons before this privacy-sensitive hook lands.

## Review Findings

Full review comments:

- **[P2] Stop logging raw block reasons:** `src/agents/pi-embedded-runner/run/attempt.ts:2879`
  - body: This line writes `beforeRunDecision.reason` to the Gateway log. The new hook receives raw prompts, and the added tests show a blocker reason can contain `secret prompt`; in that case the PR keeps the prompt out of transcripts and model hooks but still persists it in OpenClaw logs. Log the plugin id/outcome or a sanitized category instead.
  - confidence: 0.86

  - confidence: 0.75
