## Summary

Keep open. Current main does not have `before_agent_run`, and this active maintainer-labeled feature PR is still useful, but latest head still has a web approval modal cancel path that can submit `deny` even when that decision is not allowed.

## What This Changes

The PR adds a pass/block-only `before_agent_run` lifecycle hook before model submission, redacted blocked-turn transcript projection across clients, approval allowed-decision propagation, docs, changelog, and tests.

## Best Possible Solution

Land the lifecycle gate only after the web approval modal preserves restricted decision sets on cancel, while keeping the current raw-content and transcript authorization boundaries intact.

## Reproduction Assessment

Yes for the review finding from source inspection: render a web approval with `allowedDecisions: ["allow-once"]`, then press Escape or cancel the modal; the PR head calls `handleExecApprovalDecision("deny")` even though the gateway validates decisions against the allowed set.

## Solution Assessment

No. The feature direction is coherent, but the PR should first make modal dismissal submit `deny` only when it is allowed, or otherwise leave the approval pending without resolving an unsupported decision.
