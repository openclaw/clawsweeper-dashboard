## Summary

Keep open: current main does not have this plugin lifecycle gate, and the protected maintainer PR still logs raw block reasons from a raw-conversation hook path before merge.

## What This Changes

The PR adds a plugin-facing `before_agent_run` pass/block hook before provider submission, redacted blocked-turn persistence, diagnostics/docs/config updates, and focused runtime tests.

## Best Possible Solution

Keep the pre-model gate design, but merge only after blocker reasons are kept out of logs/docs and the redacted blocked-turn behavior remains covered by focused runtime and contract checks.

## Reproduction Assessment

Yes for the blocking review finding: source inspection of PR head shows `beforeRunDecision.reason` logged verbatim in the embedded-runner block path. The feature itself is new on the PR branch and absent from current main.

## Solution Assessment

No as-is: placing the gate before image loading, `llm_input`, and provider submission is the right runtime point, but raw internal reasons should not be logged or documented as diagnostics-visible.
