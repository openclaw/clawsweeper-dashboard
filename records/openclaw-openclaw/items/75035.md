## What This Changes

This PR adds a `before_agent_run` plugin lifecycle gate, blocked-input transcript projection, approval decision propagation, client display updates, docs, changelog, and tests.

## Best Possible Solution

Land the lifecycle gate after the web approval modal preserves restricted decision sets on dismissal and the full changed gate is green.

## Reproduction Assessment

Yes for the review finding from source inspection: render an approval with `allowedDecisions: ["allow-once"]`, then press Escape or cancel the modal; PR head submits `deny` even though the gateway validates decisions against the allowed set.

## Solution Assessment

No. The feature direction is coherent, but the PR should first make modal dismissal submit `deny` only when it is allowed, or otherwise keep the approval pending without resolving an unsupported decision.
