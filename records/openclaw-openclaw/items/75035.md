## Summary

Keep open. This is a maintainer-authored PR with the protected `maintainer` label, and the latest head still has blocking security/correctness gaps in the new lifecycle gate: raw prompt access is not covered by the existing conversation-access policy, blocked prompts can still reach downstream `llm_input` hooks, and blocked transcript writes bypass the established session DAG append path.

## What This Changes

The PR adds a `before_agent_run` plugin lifecycle gate with pass/block/ask decisions, plugin approval routing, blocked/redacted prompt transcript sidecars, allowed-decision approval UI handling, Swift protocol updates, and targeted tests.

## Best Possible Solution

Land a revised focused lifecycle gate that treats `before_agent_run` as a raw-conversation hook requiring explicit conversation access for non-bundled plugins, terminates blocked/denied/timed-out runs before any downstream raw-content hook or model path, and persists redacted owner-visible blocked turns through a SessionManager-compatible session DAG contract.

## Reproduction Assessment

Yes. A focused test can register a blocking `before_agent_run` hook plus an `llm_input` hook and assert that `llm_input` is not called; the current diff would still call it. A loader-policy test can also register `before_agent_run` from a non-bundled plugin without `allowConversationAccess` and show it is admitted despite receiving raw prompt/history.

## Solution Assessment

No. The feature direction is reasonable, but this patch is not the safest implementation until raw prompt access is gated consistently, blocked decisions fail closed before all downstream raw-content hooks, and transcript writes preserve the existing branch semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate before_agent_run behind conversation access:** `src/plugins/hook-types.ts:103-104`
  - body: `before_agent_run` receives the raw `prompt` and full `messages`, but the new hook name is not added to the conversation hook set enforced by the loader. That lets non-bundled plugins register this raw-content hook without `plugins.entries.<id>.hooks.allowConversationAccess=true`; add it to that policy and cover the loader behavior.
  - confidence: 0.91
- **[P1] Stop blocked prompts before llm_input hooks:** `src/agents/pi-embedded-runner/run/attempt.ts:2603`
  - body: When `before_agent_run` blocks, denies, or times out, this new block only sets `skipPromptSubmission`. The existing `llm_input` hook below is outside that guard, so the blocked `effectivePrompt` and history are still delivered to raw input hook handlers before the run returns blocked.
  - confidence: 0.9
- **[P2] Persist blocked prompts through SessionManager:** `src/config/sessions/transcript.ts:421`
  - body: The new helper writes a Pi `type: "message"` entry with `fs.promises.appendFile` and reconstructs `parentId` by scanning the last physical message. That bypasses the SessionManager append/branch contract used by current readers, so rewrites or concurrent branch updates can attach the blocked turn to the wrong leaf. Add a manager-backed append path or a supported sidecar entry format.
  - confidence: 0.78
