## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Recheck admin scope for original-content SSE streams:** `src/gateway/sessions-history-http.ts:306-307`
  - body: When a stream is opened with `includeBlockedOriginalContent`, `isStreamStillAuthorized` only verifies that the caller can still use `chat.history`. If that client's admin scope is revoked while read access remains, later refreshes keep using the captured `includeBlockedOriginalContent` flag and can continue returning blocked originals; require `ADMIN_SCOPE` in this reauth path whenever the stream requested originals.
  - confidence: 0.84
- **[P2] Preserve blocked originals for authorized live SSE updates:** `src/gateway/session-history-state.ts:238-240`
  - body: For unpaginated SSE streams, transcript updates take the inline append path, but `appendBlockedUserMessageToSessionTranscript` emits only the redacted message and `appendInlineMessage` strips any `originalBlockedContent` sidecar unconditionally. An admin caller that requested blocked originals therefore sees the redacted stub for live blocked turns until a refresh/reconnect; use a refresh path or attach the sidecar only for authorized original-content streams.
  - confidence: 0.86
