## Summary

Keep open. Current main does not yet resolve the reported Discord outage: the gateway still treats a plain SIGTERM as shutdown, and the related open SIGHUP hardening PR explicitly says the observed SSH-disconnect/SIGTERM lifecycle path remains separate.

## What This Changes

The issue asks for Discord and the gateway to remain online and reconnect after an SSH session running `openclaw chat` disconnects on a VPS.

## Best Possible Solution

Keep this issue open and investigate the SIGTERM/TUI/systemd lifecycle with a real Ubuntu headless service repro; if the root cause is missing headless systemd setup, fix docs/doctor, otherwise patch the daemon lifecycle without changing normal SIGTERM shutdown semantics.

## Solution Assessment

Unclear. The SIGHUP handler in https://github.com/openclaw/openclaw/pull/84422 is not sufficient for the reported SIGTERM path; the maintainable fix depends on proving whether the cause is service/session setup or OpenClaw lifecycle code.
