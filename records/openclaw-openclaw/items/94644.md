Full review comments:

- **[P2] Gate the warning on real multi-root accumulation:** `extensions/matrix/src/matrix/client/storage.ts:286`
  - body: This condition warns whenever there is one compatible populated sibling. In the existing token-rotation path, an empty current-token root plus one old populated root is expected and is intentionally reused for continuity, so this now logs “Multiple populated token-hash dirs” and tells operators to archive stale dirs even when the selected old root is the only populated store. Count the canonical root only when it has state, or require multiple populated candidates, and add a no-warning assertion for the single-old-root rotation case.
  - confidence: 0.91
- **[P3] Put warning fields in logger metadata:** `extensions/matrix/src/matrix/client/storage.ts:290-294`
  - body: The PR calls this a structured warning, but parentDir, canonicalTokenHash, and sibling names are interpolated into the message string. RuntimeLogger.warn supports a metadata object and adjacent Matrix warnings pass one, so log processors cannot reliably key on these fields. Use a stable message and pass the directory/hash fields as metadata.
  - confidence: 0.86
