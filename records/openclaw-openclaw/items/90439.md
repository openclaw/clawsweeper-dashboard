The PR adds a storage-neutral embedded/CLI run session target resolver, makes `sessionFile` optional on run params, and migrates several helper callers to config-backed session stores.

## Best Possible Solution

Keep the storage-neutral seam direction, but carry the resolved agent/session target through delegated compaction before `sessionFile` becomes optional and add focused coverage for non-agent helper keys.

## Reproduction Assessment

Yes. The source path is clear: a helper run with a non-agent `temp:*` session key and explicit non-main `agentId` can enter delegated compaction without a `sessionFile` or `agentId` in runtime context.

## Solution Assessment

No. The seam is a plausible migration direction, but the best fix must preserve the resolved run target through delegated compaction before dropping the explicit file target.
