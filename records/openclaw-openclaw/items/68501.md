## Summary

Keep open. Current main and the latest release still preserve explicit inline dreaming mode as `memory/YYYY-MM-DD.md`, and the runtime/schema/docs do not expose a configurable inline target; this remains a memory-core config-contract decision rather than completed cleanup.

## What This Changes

Request to make memory-core inline dreaming output configurable so Obsidian-style vaults can write managed dreaming blocks under `memory/daily-memory/YYYY-MM-DD.md` instead of only `memory/YYYY-MM-DD.md`.

## Best Possible Solution

Choose a narrow memory-core inline target contract, then update schema, normalization, markdown writing, docs, and focused tests while preserving the current default and managed markers.

## Reproduction Assessment

Yes. Source inspection of current main shows explicit inline light/REM dreaming still resolves to `memory/YYYY-MM-DD.md`, and the manifest/runtime config expose no inline target-path field.

## Solution Assessment

Unclear. A path or preset knob would solve the reported workflow, but maintainers should choose the permanent config shape because OpenClaw has a high bar for new config surface.
