Request adding a memory-core dreaming inline-storage target/path config so managed Light/REM blocks can write under layouts such as `memory/daily-memory/{date}.md`.

## Best Possible Solution

Add a memory-core-owned inline storage target option that defaults to the existing path, validates constrained workspace-relative destinations, preserves managed markers, and updates schema, docs, and focused tests for Obsidian daily-note layouts.
