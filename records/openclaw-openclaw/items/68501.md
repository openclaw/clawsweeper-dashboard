## Summary

Keep open. Current main and v2026.6.1 still hardcode inline memory-core dreaming output to `memory/YYYY-MM-DD.md` and expose only storage mode fields, so the requested `daily-memory` target remains a new plugin config/API decision rather than a cleanup-close candidate.

## What This Changes

Add a configurable memory-core dreaming inline output target so managed light/REM blocks can be written under layouts like `memory/daily-memory/YYYY-MM-DD.md` while preserving managed markers and docs.

## Best Possible Solution

Decide the memory-core inline target contract, then implement a narrowly validated plugin config, writer resolution, docs, and regression tests that preserve the current default and managed markers.

## Reproduction Assessment

Yes for the current limitation: source and tests show inline dreaming writes only to `memory/YYYY-MM-DD.md` and no config field accepts a different target. A live sweep is not necessary to prove the missing capability.

## Solution Assessment

Unclear. A path knob is plausible, but the best permanent shape needs maintainer judgment because OpenClaw’s config surface is high-bar and memory-core owns the daily-note layout contract.
