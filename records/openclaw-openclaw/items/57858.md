## Summary

Keep open. Current main still teaches the bundled coding-agent skill to launch Claude Code with a raw `claude --permission-mode bypassPermissions --print` command, and the host exec path does not generically scrub the reported Claude Code nesting marker before spawning child processes.

## What This Changes

Make the bundled coding-agent skill avoid leaking Claude Code nesting environment variables when it dispatches Claude Code subprocesses.

## Best Possible Solution

Scrub Claude Code nesting markers at the Claude Code spawn boundary and document/test the safe command template for the bundled coding-agent skill.

## Reproduction Assessment

No. There is a clear source-level env inheritance path and upstream evidence for filtering `CLAUDECODE`, but this runner cannot live-run Claude Code with the reported parent env markers.

## Solution Assessment

Yes. The narrowest maintainable direction is to clear the Claude Code nesting variables in the bundled skill command examples and, if the same leak applies to the Anthropic CLI backend, add them to the plugin-owned `CLAUDE_CLI_CLEAR_ENV` with focused coverage.
