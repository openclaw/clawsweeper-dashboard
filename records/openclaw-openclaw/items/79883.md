## Summary

Keep open: current main has the later ACP timeout-tolerance work for invalid-params/unsupported text, but it still does not classify the Pi-style ACP `-32601 Method not found` `session/set_config_option` rejection described here. The branch remains useful, but it needs to be rebased onto the current classifier and tests rather than closed as already implemented.

## What This Changes

The PR changes ACP manager runtime-control handling to skip only optional `timeout` config writes when the backend reports `session/set_config_option` as Method-not-found, with regression tests for timeout and non-timeout behavior.

## Best Possible Solution

Update current main’s existing ACP timeout-rejection classifier to also tolerate Pi-style `session/set_config_option` `-32601 Method not found` for optional timeout keys, and add a focused regression test without changing required config failure behavior.

## Reproduction Assessment

Yes, from source. Current main still sends timeout config hints when advertised keys are unknown, and the landed classifier does not match the Pi-style `-32601 Method not found` rejection shown in this PR’s real output.

## Solution Assessment

Yes in direction, but the branch is not the best merge artifact now. The maintainable fix is to fold this PR’s Method-not-found case into the current post-81603 classifier and tests, preserving the stricter failures for model/thinking/permission and runtime-unavailable errors.
