## Summary

Keep open: current main still applies the persisted ACP `timeout` option through `session/set_config_option` before the turn, and this PR is a small targeted fix with focused tests, contributor-provided live Pi ACP proof, and green checks. A broader open PR covers the same timeout-control class differently, so maintainers should choose the desired direction rather than auto-close this PR.

## What This Changes

The PR skips only `timeout` config-control rejections that report `session/set_config_option` as `Method not found`, while preserving fail-fast behavior for other ACP runtime config errors and adding manager tests.

## Best Possible Solution

Keep ACP turn timeouts enforced in the manager and make optional unsupported timeout controls non-fatal, either by landing this narrow Pi-compatible fix or by selecting the broader timeout-control cleanup already under review.

## Reproduction Assessment

Yes. Current main clearly sends `timeout` through `runtime.setConfigOption` before `runTurn`, so a backend that rejects `session/set_config_option` with `Method not found` fails before the turn; I did not run a live Pi ACP session in this read-only review.

## Solution Assessment

Partly. The patch is a valid narrow fix for the reported Pi `-32601` failure, but the broader maintainable direction may be to stop sending manager-owned timeout as a backend config option, as explored in https://github.com/openclaw/openclaw/pull/73693.
