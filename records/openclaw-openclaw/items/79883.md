## Summary

Keep open: this is a focused fix for a real current ACP/Pi timeout-control compatibility failure, and I found no blocking correctness or security regression. The only remaining wrinkle is process-level: the automated proof-template check is failing even though the PR body includes usable live-output proof.

## What This Changes

The PR changes ACP runtime-control application so a backend `Method not found` error for only the optional `timeout` config option is skipped before continuing the turn, with regression coverage.

## Best Possible Solution

Land this narrow compatibility fix after normal PR review and CI, keeping OpenClaw's local timeout enforcement while preserving fail-fast behavior for required ACP config controls.

## Reproduction Assessment

Yes, source-reproducible with high confidence: current main sends `timeoutSeconds` as ACP config key `timeout`, and `pi-acp@0.0.26` lacks `session/set_config_option`, which the ACP SDK reports as `-32601 Method not found`. I did not run a live Pi ACP backend locally in this read-only review.

## Solution Assessment

Yes, this is the narrowest maintainable fix I found: skip only the optional timeout control when the backend reports `session/set_config_option` is missing, while continuing to fail for model/thinking/approval controls, advertised-key mismatches, and other timeout errors.
