## Summary

Current main already implements the useful ACP timeout-control tolerance through the merged https://github.com/openclaw/openclaw/pull/81603 path, with broader regression coverage than this PR and release provenance in the beta channel. This PR is now redundant with that implementation and is also merge-conflicting against main.

## What This Changes

This PR skips optional ACP `timeout` config writes when a backend rejects `session/set_config_option` as unsupported and adds manager regression tests for timeout versus non-timeout failures.

## Best Possible Solution

Keep the current main implementation from https://github.com/openclaw/openclaw/pull/81603 and close this conflicting PR as already implemented.

## Reproduction Assessment

No high-confidence failing reproduction remains on current main because the timeout rejection path is now implemented away. The original bug was well-supported by this PR's Pi ACP live output and the linked diagnostics, but current main now catches the unsupported optional timeout rejection and continues the turn.

## Solution Assessment

Yes. The current-main solution is the better maintainable fix because it treats unsupported `timeout`/`timeout_seconds` config writes as best-effort hints while keeping required config and ordinary backend failures strict.
