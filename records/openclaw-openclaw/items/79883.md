## Summary

Keep open: this PR is a focused, plausible fix for Pi-style ACP backends that reject only the optional `timeout` config RPC with `Method not found`, and the supplied live output plus tests cover that path. It overlaps with broader open ACP timeout-control work and intentionally does not handle the advertised-key timeout rejection path discussed in https://github.com/openclaw/openclaw/issues/78546, so maintainer selection is still needed before merge.

## What This Changes

The PR catches ACP `session/set_config_option` Method-not-found errors for the `timeout` config key during pre-turn runtime-control application and adds manager tests for timeout-only skip and non-timeout failure behavior.

## Best Possible Solution

Keep the manager-owned turn timeout local and make unsupported optional timeout controls non-fatal without swallowing real user/config errors, choosing either this focused Pi fix or a broader canonical ACP timeout-control fix that also covers advertised-key and unknown-config cases.

## Reproduction Assessment

Yes, source/reporter reproducible: current main maps `timeoutSeconds` to the ACP `timeout` config option and applies it before `runTurn`, while the PR body supplies live Pi ACP output showing the Method-not-found failure no longer blocks the turn after the patch. I did not run a live ACP backend in this read-only review.

## Solution Assessment

Partly. The patch is a correct narrow compatibility fix for the reported Pi `-32601 Method not found` shape, but the broader ACP timeout-control cleanup in related open PRs may be more maintainable because it also covers advertised-key and unknown-config rejection paths.
