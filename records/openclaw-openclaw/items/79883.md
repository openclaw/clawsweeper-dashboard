## Summary

Keep this PR open for maintainer review. The patch is a plausible narrow fix for the Pi ACP timeout-control failure, but it overlaps an open broader ACP timeout/control cleanup that may be the cleaner landing path.

## What This Changes

The PR skips only `timeout` `session/set_config_option` errors whose message reports `Method not found`, and adds manager tests for the timeout skip and non-timeout failure cases.

## Best Possible Solution

Keep ACP turn timeouts enforced by the manager and make unsupported optional timeout controls non-fatal, either by landing this narrow compatibility fix or by choosing the broader ACP timeout-control cleanup.

## Reproduction Assessment

Yes. Current main sends `timeoutSeconds` as `session/set_config_option(timeout)` before `runTurn`, so a backend that returns `Method not found` for that method fails before the turn; the PR body also supplies copied live Pi ACP before/after output.

## Solution Assessment

Partly. This is a correct narrow fix for the reported Pi failure, but the broader open cleanup in https://github.com/openclaw/openclaw/pull/73693 may be the more maintainable long-term solution because it removes the redundant timeout mirror and handles more unsupported-control forms.
