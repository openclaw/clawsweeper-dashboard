## Summary

Keep open. Current main has the broader ACP timeout-hint work from https://github.com/openclaw/openclaw/pull/81603, but it still does not classify the Pi `ACP -32601 Method not found` shape that this PR proves, so the remaining compatibility case is not implemented yet.

## What This Changes

The PR skips optional ACP `timeout` config writes that fail with `session/set_config_option` Method-not-found and adds manager regression tests.

## Best Possible Solution

Extend the current main timeout-rejection classifier to cover `ACP -32601`/`Method not found` for optional timeout keys, preserve the existing strict failures, and add the Pi-shaped regression without undoing https://github.com/openclaw/openclaw/pull/81603.

Yes with integration changes. The right fix is to fold this PR's Method-not-found case into the existing current-main helper rather than merging the older branch in a way that drops the broader https://github.com/openclaw/openclaw/pull/81603 behavior.
