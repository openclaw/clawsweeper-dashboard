## Summary

Keep open. Current main fixed the advertised-key and Claude/Codex ACP timeout paths via the merged Claude-specific fix, but it still emits and applies `timeout` when capability keys are unknown, so the Pi `session/set_config_option` Method-not-found failure remains a useful, narrowly scoped compatibility fix that now needs rebase/maintainer review.

## What This Changes

The PR skips only ACP `timeout` config writes that fail with `session/set_config_option` Method-not-found and adds manager regression tests for timeout-only skip versus non-timeout failure.

## Best Possible Solution

Rebase this PR on current main, preserve the merged advertised-key and Claude/Codex ACP timeout handling, and land the generic `timeout` Method-not-found compatibility guard with focused manager tests.

## Reproduction Assessment

Yes from source and supplied live output. Current main still emits `timeout` for unknown capability-key sets and applies it before the turn, while the PR body shows a real Pi ACP run succeeding after this exact fallback.

## Solution Assessment

Yes, after rebasing. The patch is the narrowest remaining fix for backends that expose `setConfigOption` but reject the optional timeout control with Method-not-found, and it preserves fail-fast behavior for non-timeout controls.
