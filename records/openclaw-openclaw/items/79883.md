The PR adds a manager-side fallback that skips only optional ACP timeout `session/set_config_option` Method-not-found errors and tests timeout-only skip versus non-timeout failure behavior.

## Best Possible Solution

Extend current main's existing optional-timeout rejection helper to include the reported Method-not-found case, keep advertised-key gating and required-config failures intact, and retain focused manager regression coverage.

## Reproduction Assessment

## Solution Assessment

Yes in direction, after rebase. The maintainable path is to add the Method-not-found case to the existing current-main optional-timeout classifier rather than reintroducing the older standalone helper.
