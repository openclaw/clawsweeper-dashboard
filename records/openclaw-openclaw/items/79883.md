## Summary

Keep open: this is a valid current-main ACP compatibility fix with sufficient live proof and no blocking code/security findings, but maintainer review should choose between this narrow Pi-focused patch and the broader ACP timeout-control cleanup.

## What This Changes

This PR catches ACP `session/set_config_option` Method-not-found errors for the `timeout` config key, skips only that optional timeout control before a turn, and adds manager tests for timeout and non-timeout behavior.

## Best Possible Solution

Keep ACP turn timeouts enforced by the manager and make unsupported optional timeout controls non-fatal, either by landing this focused compatibility fix or by choosing the broader ACP cleanup.

## Reproduction Assessment

Yes from source and supplied live output. Current main maps `timeoutSeconds` to ACP config key `timeout` and applies it before `runTurn`, so a backend returning Method-not-found for `session/set_config_option` blocks the turn.

## Solution Assessment

Partly. The diff is a correct narrow fix for the reported Pi failure, but https://github.com/openclaw/openclaw/pull/73693 may be the more maintainable long-term fix because it removes the redundant timeout mirror and handles more unsupported-control shapes.
