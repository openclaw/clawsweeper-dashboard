## What This Changes

This PR skips optional ACP `timeout` config writes when `session/set_config_option` is rejected as Method-not-found and adds manager regression tests for timeout-only tolerance versus non-timeout failure.

## Best Possible Solution

Keep the merged manager-side timeout tolerance on main and close this conflicting branch as superseded by the shipped implementation.

## Solution Assessment

Yes. The merged main solution is the better maintained version because it covers `timeout` and `timeout_seconds` plus unsupported/invalid-param rejection shapes while preserving fail-fast behavior for required config and backend runtime failures.
