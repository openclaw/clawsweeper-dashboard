## Summary

Keep open. The cache-growth bug is real on current main, but this PR should not merge as-is because it bundles unrelated web_search provider/credential rewrites and tool-schema behavior changes with the cache fix, plus the cache eviction logic needs tightening.

## What This Changes

The PR adds FIFO caps to four agent runtime cache Maps and also changes agent tool schemas plus a large web_search provider and credential implementation.

## Best Possible Solution

Land a narrow cache-only replacement on current main that caps the four intended Maps, evicts only on new-key inserts, preserves current provider/SecretRef/schema behavior, and adds focused regression coverage.

## Reproduction Assessment

Yes. Current main has module-level Maps in the reported paths with distinct-key writes and no maximum size; repeated distinct session keys, workspace paths, trace files, or model IDs reproduce unbounded growth at source level.

## Solution Assessment

No. The cache-cap direction is the right fix, but this PR is not the best implementation because it mixes unrelated web_search, credential, and schema changes into the memory patch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Route web_search through the runtime provider registry:** `src/agents/tools/web-search.ts:25`
  - body: The PR replaces the registry-backed web_search path with hard-coded provider handling. Current main supports bundled and runtime providers through `runWebSearch`/runtime provider resolution, so custom, Firecrawl, Tavily, DuckDuckGo, or disabled-plugin configurations can stop working or fall back incorrectly.
  - confidence: 0.93
- **[P1] Read web_search keys from plugin-owned SecretRefs:** `src/agents/tools/web-search.ts:553-563`
  - body: These resolvers use legacy tool config or env vars instead of the plugin-owned webSearch config and resolved runtime SecretRefs. Existing migrated users can get missing-key failures even when their selected provider is configured.
  - confidence: 0.9
- **[P2] Evict workspace cache entries only for new keys:** `src/agents/workspace.ts:92`
  - body: The eviction helper runs before every `workspaceFileCache.set`, including refreshes of a path already in the map. At capacity, refreshing an existing file evicts an unrelated oldest key and then overwrites the same key, reducing effective capacity.
  - confidence: 0.88
- **[P2] Preserve sub-second node clip durations:** `src/agents/tools/nodes-tool.ts:165-170`
  - body: Adding `minimum: 1000` to `durationMs` rejects 0-999 ms values that current main accepts under the existing 300000 ms maximum. That is an unrelated behavior change for node media operations.
  - confidence: 0.82
- **[P3] Apply the model cache cap consistently:** `src/agents/context.ts:52-58`
  - body: Only the discovered-model path enforces the new model-cache cap, while configured context windows write to the same cache without a guard. A large configured model list can exceed the cap and later have configured entries evicted.
  - confidence: 0.76
