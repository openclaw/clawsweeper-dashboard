## Summary

Keep open. The underlying memory-growth concern is real on current main, but this PR is not mergeable as-is because it bundles the cache caps with unrelated web_search provider/credential rewrites and tool-schema validation changes that introduce concrete regressions.

## What This Changes

The PR adds FIFO caps to four agent runtime Map caches and also changes multiple agent tool schemas plus web_search provider and credential handling.

## Best Possible Solution

Land a narrow cache-only replacement on current main that caps the intended Maps with deterministic oldest-entry eviction only when inserting new keys, preserves existing provider/SecretRef and tool-schema behavior, and adds focused regression coverage for cache size and refresh semantics.

## Reproduction Assessment

Yes. Current main has module-level Maps in the reported paths with distinct-key writes and no maximum size; repeated distinct session keys, workspace paths, trace files, or model IDs reproduce unbounded growth at the source level.

## Solution Assessment

No. The cache-cap direction is appropriate, but this PR is not the best way to solve it because it mixes the fix with unrelated provider, credential, and schema behavior changes; a cache-only replacement is the safer path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Route web_search through the provider registry:** `src/agents/tools/web-search.ts:25`
  - body: The PR replaces the registry-backed web_search path with a hard-coded provider list. Bundled or runtime providers outside that list, including plugin-provided search integrations, can stop working or fall back incorrectly.
  - confidence: 0.93
- **[P1] Use plugin-owned SecretRefs for web_search keys:** `src/agents/tools/web-search.ts:553-563`
  - body: These resolvers read legacy tool config or env vars instead of plugin-owned webSearch config and resolved SecretRefs. Existing users with valid plugin config can see missing-key failures after this change.
  - confidence: 0.9
- **[P2] Evict workspace cache only for new keys:** `src/agents/workspace.ts:92`
  - body: The eviction call runs before every workspaceFileCache.set, including refreshes for a path already in the map. At capacity, refreshing an existing file evicts an unrelated oldest entry and reduces effective capacity.
  - confidence: 0.88
- **[P2] Preserve sub-second node clip durations:** `src/agents/tools/nodes-tool.ts:165-170`
  - body: Adding minimum: 1000 to durationMs rejects 0-999 ms values that current main accepts up to the existing 300000 ms maximum. That is an unrelated behavior change for camera clips.
  - confidence: 0.82
- **[P3] Apply the model cache cap consistently:** `src/agents/context.ts:52-58`
  - body: Only the discovered-model path enforces the new model-cache cap, while configured context windows write to the same cache without a guard. A large configured model list can exceed the cap and later have configured entries evicted.
  - confidence: 0.76
