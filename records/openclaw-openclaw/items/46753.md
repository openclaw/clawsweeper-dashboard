## Summary

Keep this PR open, but it is not mergeable as-is: the cache-cap idea is plausible, while the branch still mixes unrelated provider/schema rewrites with blocking regressions and lacks after-fix real daemon/cache proof.

## What This Changes

The PR adds oldest-first caps to several agent-runtime Map caches and also changes many agent tool schemas plus replaces the existing web_search runtime-provider path with a local provider implementation.

## Best Possible Solution

Land a narrow cache-only patch that preserves bootstrap refresh and registry-backed web_search behavior, keeps unrelated tool-schema changes out of scope, adds focused eviction coverage, and includes redacted real daemon/cache proof.

## Reproduction Assessment

No. Source inspection shows several process-global Maps are uncapped on current main, but neither the PR body nor this review establishes a failing current-main daemon path with heap-growth proof.

## Solution Assessment

No. A narrow cache-only patch is the maintainable solution; this branch also changes provider routing, credential lookup, and tool schemas, which introduces unrelated regressions and review risk.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve bootstrap refresh behavior:** `src/agents/bootstrap-cache.ts:23-24`
  - body: Current main refreshes workspace bootstrap files before reusing a cached snapshot so long-running sessions pick up workspace edits. This branch returns cached files before reloading, so later bootstrap changes can be missed.
  - confidence: 0.92
- **[P1] Route web_search through the runtime provider registry:** `src/agents/tools/web-search.ts:1903-1906`
  - body: The branch replaces the registry-backed web_search path with hard-coded local providers. Configured bundled or runtime providers outside that set can stop resolving or fall back to the wrong provider.
  - confidence: 0.93
- **[P1] Read search keys from plugin-owned config:** `src/agents/tools/web-search.ts:905-911`
  - body: The new resolvers only read local scoped config or environment variables, but current provider setup stores configured credentials under plugin-owned config paths. Existing configured users can hit missing-key errors even when their provider is configured.
  - confidence: 0.9
- **[P2] Preserve sub-second recording durations:** `src/agents/tools/nodes-tool.ts:165-170`
  - body: Adding a 1000 ms minimum rejects durationMs values below one second, while current main only caps the maximum. That validation tightening is unrelated to cache caps and can break previously accepted node recording inputs.
  - confidence: 0.86
- **[P2] Evict workspace cache only for new keys:** `src/agents/workspace.ts:92-93`
  - body: Evicting before every set includes same-key refreshes. At capacity, refreshing a changed file can delete an unrelated entry and then overwrite the existing key, causing avoidable cache churn and disk reads.
  - confidence: 0.8
- **[P3] Apply model-cache limits consistently:** `src/agents/context.ts:82`
  - body: Configured context-window entries bypass the new cap, so a large configured model set can overfill the map and later have configured entries evicted by the discovered-model path.
  - confidence: 0.73
