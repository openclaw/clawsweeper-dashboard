## Summary

Keep open: source inspection supports the unbounded-cache concern, but this branch is not mergeable because it regresses bootstrap refresh, bypasses web_search plugin/runtime provider and credential contracts, tightens unrelated schemas, and lacks after-fix real behavior proof.

## What This Changes

This PR adds oldest-first caps to several agent-runtime cache maps while also tightening many tool schemas and replacing registry-backed web_search routing with local provider code.

## Best Possible Solution

Land a narrow cache-only fix that preserves bootstrap refresh and registry-backed web_search behavior, adds focused eviction coverage, and includes redacted real daemon/cache proof.

## Reproduction Assessment

No high-confidence failing daemon reproduction is available. Source inspection shows several process-global Maps are uncapped on current main, but the PR body and comments do not establish heap-growth proof for a long-running gateway process.

## Solution Assessment

No. A narrow cache-only patch is the maintainable fix; this branch mixes cache caps with provider routing, credential lookup, and schema changes that are unrelated and regression-prone.

Full review comments:

- **[P1] Preserve bootstrap refresh behavior:** `src/agents/bootstrap-cache.ts:23-24`
  - body: Current main refreshes workspace bootstrap files before reusing a cached snapshot so long-running sessions pick up workspace edits. This branch returns cached files before reloading, so later bootstrap changes can be missed.
  - confidence: 0.93
- **[P1] Route web_search through the runtime provider registry:** `src/agents/tools/web-search.ts:1903-1906`
  - body: The branch replaces registry-backed web_search resolution with hard-coded local provider selection. Configured bundled, external, or runtime providers outside that list can stop resolving or fall back to the wrong provider.
  - confidence: 0.94
- **[P1] Read search keys from plugin-owned config:** `src/agents/tools/web-search.ts:905-911`
  - body: The new provider key readers only check local scoped config or environment variables, but current provider setup stores configured credentials under plugin-owned config paths and SecretRefs. Existing configured users can hit missing-key errors even when their provider is set up.
  - confidence: 0.91
- **[P2] Preserve sub-second recording durations:** `src/agents/tools/nodes-tool.ts:165-170`
  - body: Adding a 1000 ms minimum rejects durationMs values below one second, while current main only caps the maximum. That validation tightening is unrelated to cache caps and can break previously accepted node recording inputs.
  - confidence: 0.86
- **[P2] Evict workspace cache only for new keys:** `src/agents/workspace.ts:92-93`
  - body: This eviction runs before every set, including same-key refreshes. At capacity, refreshing a changed file can delete an unrelated entry and then overwrite the existing key, causing avoidable cache churn and extra disk reads.
  - confidence: 0.8
- **[P3] Apply model-cache limits consistently:** `src/agents/context.ts:82`
  - body: Configured context-window entries bypass the new cap, so a large configured model set can overfill the map and later have configured entries evicted by the discovered-model path.
  - confidence: 0.73
