## Summary

Keep open: the cache leak concern is still source-visible on current main, but this PR is not safe to merge as-is because it combines useful cache caps with unrelated web-search provider rewrites, credential-path regressions, schema tightening, and no after-fix daemon proof.

## What This Changes

The PR adds bounded eviction to bootstrap, workspace file, cache-trace, and model-context caches while also adding broad tool-schema bounds and replacing `web_search` with local provider implementations.

## Best Possible Solution

Land a narrow replacement that only caps the relevant caches, preserves provider-registry and plugin-owned credential contracts, and adds focused eviction regression coverage.

## Reproduction Assessment

No high-confidence live reproduction: current main has source-visible unbounded maps in the targeted cache paths, but this review did not reproduce heap growth in a long-running daemon and the PR body says that was not verified.

## Solution Assessment

No. Cache caps are a reasonable direction, but this PR is not the best solution because it combines them with unrelated provider rewrites and schema tightening that violate current contracts.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Route web_search through the provider registry:** `src/agents/tools/web-search.ts:25`
  - body: The replacement implementation uses a hard-coded provider set and local routing instead of `runWebSearch`/the runtime provider registry, so bundled and third-party providers outside this list stop working even though current tests require active registry providers to execute through `runWebSearch`.
  - confidence: 0.92
- **[P1] Read search keys from plugin-owned config:** `src/agents/tools/web-search.ts:905-910`
  - body: The new key resolvers read provider-specific legacy config or environment variables directly, but current onboarding/docs use `plugins.entries.<provider>.config.webSearch.*` and SecretRefs for bundled providers; configured users can now get missing-key results even when the plugin is configured.
  - confidence: 0.9
- **[P2] Preserve sub-second recording durations:** `src/agents/tools/nodes-tool.ts:165-170`
  - body: Adding `minimum: 1000` to `durationMs` rejects sub-second camera or screen recordings that current main accepts, and this validation tightening is unrelated to the cache-cap change.
  - confidence: 0.84
- **[P2] Evict only when inserting a new workspace cache key:** `src/agents/workspace.ts:92`
  - body: The workspace cache eviction runs before every write, including same-key refreshes. At capacity, refreshing an existing file can evict an unrelated entry and reduce cache usefulness without lowering memory usage.
  - confidence: 0.78
