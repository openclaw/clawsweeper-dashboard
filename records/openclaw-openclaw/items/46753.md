## Summary

Keep this PR open, but do not merge it as-is. The cache-cap goal is valid, yet the branch also replaces the registry-backed `web_search` path with hard-coded provider logic and tightens unrelated tool schemas, which would break current provider and config contracts.

## What This Changes

The PR adds max-size eviction to several agent-runtime caches while also changing many agent tool schemas and replacing the registry-backed `web_search` wrapper with local provider implementations.

## Best Possible Solution

Land a cache-only fix with insertion-only bounded eviction and focused regression coverage, while keeping `web_search` on the registry-backed provider path and moving schema hardening into separate compatibility-reviewed work.

## Reproduction Assessment

Yes, at source level. Current main has unbounded module-level cache Maps, and the PR-introduced regressions are visible from the diff against current tests/docs for provider registry and plugin-owned config behavior.

## Solution Assessment

No. Oldest-first caps are a reasonable direction for the cache leak, but this PR is not the best fix because it mixes that narrow bug fix with unrelated `web_search` and schema changes that break established contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep web_search on the runtime provider registry:** `src/agents/tools/web-search.ts:1903-1906`
  - body: This replaces the current registry-backed `web_search` wrapper with a local provider choice. Current main supports runtime/custom providers and bundled providers such as Firecrawl, Exa, Tavily, DuckDuckGo, MiniMax, and SearXNG through `runWebSearch`; after this change those providers either fall through the local list or hit `Unsupported web search provider`, breaking existing configured searches.
  - confidence: 0.94
- **[P1] Preserve plugin-owned web-search credentials:** `src/agents/tools/web-search.ts:1946-1953`
  - body: The new execution path resolves keys through provider-specific helpers that read legacy `tools.web.search.*` shapes or env vars. Current docs and tests require `plugins.entries.<plugin>.config.webSearch.*` and SecretRef-backed credentials to work, so users with canonical plugin-owned config would get missing-key errors even though the provider is configured.
  - confidence: 0.92
- **[P2] Avoid evicting on workspace cache refreshes:** `src/agents/workspace.ts:92`
  - body: This eviction runs before every `workspaceFileCache.set`, including when the key is already present and only the file identity changed. At capacity, refreshing an existing cached file deletes an unrelated oldest entry and then overwrites the same key, causing avoidable cache churn; only evict when inserting a new key.
  - confidence: 0.88
- **[P2] Preserve sub-second camera clip durations:** `src/agents/tools/nodes-tool.ts:167-170`
  - body: Adding `minimum: 1000` to `durationMs` tightens the existing nodes tool schema, which currently only has a 300000 ms maximum. Calls that used short sub-second clips were valid before and would now be rejected, despite the PR being described as behavior-neutral cache work.
  - confidence: 0.81
