## Summary

This PR should stay open: the cache-growth concern is valid from source inspection, but the branch still combines that narrow fix with unrelated provider rewrites and schema tightening that break current contracts, and it lacks real after-fix daemon/cache behavior proof.

## What This Changes

The branch adds oldest-first caps to several agent-runtime global `Map` caches while also changing many tool schemas and replacing the current `web_search` runtime-provider path with a large local provider implementation.

## Best Possible Solution

Land a narrow replacement that only caps the relevant caches, preserves current provider-registry and credential contracts, and adds focused eviction regression coverage.

## Reproduction Assessment

No for a high-confidence live reproduction: current main has source-visible unbounded maps in the targeted cache paths, but this review did not reproduce heap growth in a long-running daemon and the PR body says that was not verified.

## Solution Assessment

No. Cache caps are the right direction, but this PR is not the best solution because it combines them with unrelated provider rewrites and schema tightening that violate current contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Route web_search through the provider registry:** `src/agents/tools/web-search.ts:25`
  - body: The replacement implementation no longer calls the runtime/plugin provider registry, so bundled and third-party providers outside the hard-coded list stop working even though current tests require active registry providers to execute through `runWebSearch`.
  - confidence: 0.92
- **[P1] Read search keys from plugin-owned config:** `src/agents/tools/web-search.ts:553-563`
  - body: The new key resolvers read legacy `tools.web.search.*` fields or environment variables directly, but current onboarding and providers use plugin-owned config paths. Users with configured plugin credentials can now receive missing-key errors.
  - confidence: 0.9
- **[P2] Preserve sub-second recording durations:** `src/agents/tools/nodes-tool.ts:165-170`
  - body: Adding `minimum: 1000` to `durationMs` rejects sub-second camera or screen recordings that current main accepts, and this validation tightening is unrelated to the cache-cap change.
  - confidence: 0.84
- **[P2] Evict only when inserting a new workspace cache key:** `src/agents/workspace.ts:92`
  - body: The workspace cache eviction runs before every write, including same-key refreshes. At capacity, refreshing an existing file can evict an unrelated entry and reduce cache usefulness without lowering memory usage.
  - confidence: 0.78
