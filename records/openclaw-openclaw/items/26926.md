## Summary

Keep open. Current main still rejects Synology Chat inbound webhook payloads without text, and neither the outbound URL-based media path nor the merged JSON/alias webhook compatibility work implements inbound image, CSV, or PDF attachment handoff.

## What This Changes

The issue requests inbound image, CSV, and PDF attachment support for the Synology Chat plugin.

## Best Possible Solution

Implement Synology-owned inbound attachment parsing and safe staging for accepted images, CSVs, and PDFs, then pass media facts into the existing channel turn media context with docs and focused tests.

## Reproduction Assessment

Yes, by source inspection: an attachment-only Synology webhook with token and user but no text is rejected by parsePayload, and the downstream Synology message/context types have no media handoff. This proves the requested capability is absent on current main, though it is a feature gap rather than a broken documented contract.

## Solution Assessment

Yes on the plugin-local direction, but not as an automatic cleanup repair. The maintainable fix should stay in extensions/synology-chat after confirming Synology's attachment API contract and file-safety rules.
