## Summary

Keep open. Current main still treats Synology Chat inbound webhooks as text-only while media support is outbound URL delivery, so requested inbound image/CSV/PDF attachment support remains unimplemented and no canonical duplicate or open fix PR was found.

## What This Changes

Add inbound Synology Chat support for images, CSV files, and PDFs so local/private chats can send attachments to agents.

## Best Possible Solution

Implement plugin-owned inbound attachment parsing and safe staging in Synology Chat, then feed supported files into the shared channel media context with docs and focused regression tests.

## Reproduction Assessment

Yes, by source inspection: an attachment-only webhook with token and user but no text is rejected by parsePayload, and the inbound turn never passes media into buildContext. This is a missing feature rather than a broken documented contract.

## Solution Assessment

Yes on direction, but not as an automated repair yet. The maintainable path is to keep Synology-specific parsing/download behavior inside the plugin and reuse the shared inbound media context after confirming Synology's payload and media-safety rules.
