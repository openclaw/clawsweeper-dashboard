## Summary

Keep open. Current main still treats Synology Chat inbound webhooks as text-only, the downstream Synology inbound turn path has no media handoff, and prior review discussion explicitly kept this feature gap open with no merged duplicate fix found.

## What This Changes

The issue requests image, CSV, and PDF attachment support for inbound Synology Chat messages.

## Best Possible Solution

Implement plugin-owned inbound attachment parsing and safe staging for accepted images, CSVs, and PDFs, then pass media facts into the shared channel turn context with Synology docs and focused webhook/context tests.

## Reproduction Assessment

Yes, by source inspection: an attachment-only Synology webhook with token and user but no text is rejected by parsePayload, and the downstream Synology turn path has no media fields. This proves the requested capability is absent on current main, though it is a feature gap rather than a broken documented contract.

## Solution Assessment

Yes on direction, but not as an automatic cleanup close or narrow repair yet. The maintainable path is to keep Synology-specific parsing/download/staging inside the plugin, reuse the shared inbound media context, and first confirm Synology's attachment contract plus file-safety rules.
