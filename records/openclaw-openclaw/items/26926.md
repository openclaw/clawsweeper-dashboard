## Summary

Keep open. Current main still treats Synology Chat inbound webhooks as text-only and only supports media on the outbound URL-delivery path, so inbound image/CSV/PDF attachment support remains unimplemented and no canonical duplicate or merged fix was found.

## What This Changes

Request support for inbound images, CSV files, and PDFs from Synology Chat into OpenClaw agents.

## Best Possible Solution

Implement plugin-owned inbound attachment parsing and safe staging in Synology Chat, then pass supported media facts into the existing shared channel media context with docs and focused regression tests.

## Reproduction Assessment

Yes, by source inspection: an attachment-only webhook with token and user but no text is rejected as missing required fields, and the inbound delivery/buildContext path has no media field. This proves the requested capability is absent on current main, though it is a feature gap rather than a broken documented contract.

## Solution Assessment

Yes on direction, but not as an automated repair yet. The maintainable path is to keep Synology-specific parsing/download behavior inside the plugin and reuse the shared inbound media context after confirming Synology's payload and file-safety rules.
