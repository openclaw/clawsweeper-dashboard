## Summary

Keep open: current main and the latest release still treat Synology Chat inbound webhooks as required-text messages and do not ingest images, CSVs, or PDFs into inbound media context. The remaining work is real but needs Synology attachment contract and safe-download policy decisions before implementation.

## What This Changes

Request to add inbound image, CSV, and PDF attachment support to the bundled Synology Chat plugin.

## Best Possible Solution

Confirm Synology's attachment webhook/retrieval contract, then implement plugin-local bounded inbound media parsing/download/staging that feeds existing generic inbound media context with docs and focused tests.

## Reproduction Assessment

Yes, at source level: an attachment-only Synology webhook cannot pass the required-text parser, and the delivery/context path has no media fields. A real NAS attachment webhook payload is still missing, so the external API shape is not fully reproduced.

## Solution Assessment

Unclear until Synology's attachment event and retrieval contract are confirmed. If that contract supports the feature, plugin-local bounded download/staging plus generic media-context population is the narrowest maintainable direction.
