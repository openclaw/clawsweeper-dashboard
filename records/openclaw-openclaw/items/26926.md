## Summary

Keep this open. Current main and v2026.6.5 still handle inbound Synology Chat webhooks as text-only, and the remaining work needs maintainer product/security approval because it would ingest untrusted files from a chat service.

## What This Changes

The issue requests Synology Chat inbound attachment support for images, CSV files, and PDFs.

## Best Possible Solution

Confirm Synology's real attachment event and retrieval contract, then implement plugin-local inbound attachment handling using shared safe media storage and canonical inbound media context fields.

## Reproduction Assessment

Yes, at source level: current parser and typed payload require non-empty text and no inbound attachment fields are carried into context. A live Synology attachment payload is still missing for exact field names.

## Solution Assessment

Unclear for the exact implementation until the Synology attachment contract is verified, but the plugin-local boundary is the narrowest maintainable place to solve it.
