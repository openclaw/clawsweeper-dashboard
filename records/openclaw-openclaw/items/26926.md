## Summary

Keep open. Current `main` still handles Synology Chat inbound webhooks as text-only, while the existing outbound URL-based media path and webhook compatibility work do not cover inbound image, CSV, or PDF attachments.

## What This Changes

Request support for inbound Synology Chat images, CSV files, and PDFs instead of text-only webhook messages.

## Best Possible Solution

Implement Synology-owned inbound attachment parsing and safe staging for accepted images, CSVs, and PDFs, then pass media facts into the existing turn media context with docs and focused tests.

## Reproduction Assessment

Yes, by source inspection: an attachment-only Synology webhook with token and user but no text is rejected by `parsePayload`, and the downstream turn path has no media handoff. This proves the requested capability is still absent on current `main`, though it is a feature gap rather than a broken documented contract.

## Solution Assessment

Yes on the plugin-local direction, but not as an automatic cleanup close or repair. The maintainable fix should stay in `extensions/synology-chat` after confirming Synology's attachment contract and file-safety rules.
