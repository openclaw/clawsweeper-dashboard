## Summary

Keep this open. Current main still implements Synology Chat inbound webhooks as text-only, while the existing media support is outbound URL delivery; the prior maintainer/Codex comment is still accurate on the current checkout at 542821cd1e60e34caec7df84f8583007b5b98f63.

## What This Changes

Add Synology Chat inbound attachment support for images, CSV files, and PDFs so private-network Synology Chat users can send files to OpenClaw agents.

## Best Possible Solution

Implement inbound attachment handling inside the Synology Chat plugin: parse attachment-bearing webhook payloads, allow attachment-only messages, safely stage accepted images/CSVs/PDFs into the existing media context fields, document the behavior, and keep Synology-specific policy out of core unless a reusable SDK seam is actually missing.
