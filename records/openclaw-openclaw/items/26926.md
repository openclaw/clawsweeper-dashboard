## Summary

Keep open. Current main still treats Synology Chat inbound webhooks as text-only while media support is outbound URL delivery, so inbound image/CSV/PDF attachment support remains unimplemented and needs maintainer review before implementation.

## What This Changes

Request inbound Synology Chat support for images, CSV files, and PDFs instead of text-only webhook messages.

## Best Possible Solution

Add plugin-owned inbound attachment parsing and safe staging in Synology Chat, then feed supported files into the existing channel media context with docs and focused tests.

## Reproduction Assessment

Yes, by source inspection: an attachment-only webhook with token and user but no text is rejected before delivery, and the delivered inbound turn has no media field. This is a feature gap, not a broken documented contract.

## Solution Assessment

Yes on direction, but not as an automated repair yet. The maintainable path is Synology-plugin parsing/download behavior that reuses the shared inbound media context after the Synology contract and safety policy are settled.
