## Summary

Keep open. Current main still treats Synology Chat inbound webhooks as text-only: payload parsing requires text, delivery carries no media facts, and docs/tests only cover text input plus outbound URL-based file delivery.

## What This Changes

Request inbound Synology Chat support for images, CSV files, and PDFs so private-network users can send attachments to agents.

## Best Possible Solution

Implement plugin-owned inbound attachment parsing and safe staging in `extensions/synology-chat`, then feed supported media into the existing channel turn media facts with focused docs and tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an attachment-only payload with token and user fields but no text is rejected by parsePayload, and no later Synology path passes media into the turn context.

## Solution Assessment

Yes. Plugin-owned inbound attachment handling is the narrowest maintainable direction because shared turn media fields already exist; add a core seam only if the confirmed Synology contract exposes a reusable gap.
