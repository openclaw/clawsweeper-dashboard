## Summary

Keep open. Current main still has outbound URL-based Synology Chat media delivery, but inbound webhook handling remains text-only and cannot pass images, CSVs, or PDFs into the shared channel media context.

## What This Changes

Add inbound Synology Chat attachment support for images, CSV files, and PDFs.

## Best Possible Solution

Implement plugin-owned inbound attachment parsing and safe staging in extensions/synology-chat, then feed supported files into the existing channel turn media facts with docs and regression tests.

## Reproduction Assessment

Yes, by source inspection: an attachment-only Synology webhook with token and user but no text returns null in parsePayload, and no later Synology path carries media into buildContext. This is a feature gap rather than a broken documented contract.

## Solution Assessment

Yes on direction, but not as an automated fix yet: the maintainable path is to keep the work inside the Synology Chat plugin and use the existing media context fields. Confirm Synology's attachment payload contract and media safety limits before coding.
