## Summary

Keep open. Current main still treats Synology Chat inbound webhooks as text-only; outbound file-by-URL support and webhook compatibility fixes do not implement inbound image, CSV, or PDF attachment handling.

## What This Changes

Request support for inbound images, CSV files, and PDFs from Synology Chat messages.

## Best Possible Solution

Implement plugin-owned inbound attachment parsing and safe staging for accepted images, CSVs, and PDFs, then pass media facts into the existing channel turn media context with Synology docs and focused tests.

## Reproduction Assessment

Yes, by source inspection: an attachment-only Synology webhook with token and user but no text is rejected by parsePayload, and the downstream Synology turn path has no media fields. This proves the requested capability is absent on current main, though it is a feature gap rather than a broken documented contract.

## Solution Assessment

Yes on direction, but not as an automatic cleanup close or repair. The maintainable path is to keep Synology-specific parsing, download, and staging inside the plugin while reusing the shared inbound media context after confirming Synology's contract and file-safety rules.
