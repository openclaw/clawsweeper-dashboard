## Summary

Keep open: current main and the latest release still only support Synology Chat text inbound webhooks plus outbound URL-based media sends; inbound images, CSVs, and PDFs remain missing and need maintainer/security review before implementation.

## What This Changes

Request inbound support for images, CSV files, and PDFs in the Synology Chat plugin.

## Best Possible Solution

Add plugin-local inbound attachment parsing and safe staging for allowed images, CSVs, and PDFs, then pass media facts into the existing inbound media context with focused docs and tests.

## Reproduction Assessment

Yes, by source inspection: an attachment-only Synology webhook without text is rejected before delivery, and the downstream Synology message/context types have no media fields. I did not run a live Synology NAS attachment webhook.

## Solution Assessment

Partly. The plugin-local direction is right because shared inbound media context already exists, but the requested minor extension is not enough without confirming Synology's attachment contract and file-ingestion safety policy.
