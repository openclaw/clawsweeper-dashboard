## Summary

Keep open. Current main still implements Synology Chat inbound webhooks as text-only, and the existing outbound URL-based media support plus webhook compatibility fixes do not handle inbound image, CSV, or PDF attachments.

## What This Changes

The issue requests inbound Synology Chat support for image, CSV, and PDF attachments.

## Best Possible Solution

Implement Synology-owned inbound attachment parsing and safe staging for accepted images, CSVs, and PDFs, then pass media facts into the existing channel turn media context with docs and focused tests.

## Reproduction Assessment

Yes, by source inspection: an attachment-only Synology webhook with token and user but no text is rejected by parsePayload, and the downstream message/context path has no media handoff. This proves the requested capability is absent on current main, though it is a feature gap rather than a broken documented contract.

## Solution Assessment

Yes on the plugin-local direction, but not as an automatic cleanup close or repair. The maintainable fix should stay in extensions/synology-chat after confirming Synology's attachment API contract and file-safety rules.
