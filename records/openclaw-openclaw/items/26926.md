## Summary

Keep open. Current main and the latest release still support Synology Chat media only on the outbound URL-delivery path, while inbound webhook payloads, parsing, delivery, context, docs, and tests remain text-only; prior review comments also kept this open and no attachment-specific duplicate or fix PR was found.

## What This Changes

Request support for inbound Synology Chat images, CSV files, and PDFs so local/private-network users can upload attachments to agents.

## Best Possible Solution

Implement plugin-owned inbound attachment parsing/download/staging for approved image/CSV/PDF payloads and pass media facts into the shared channel turn context, with Synology docs and focused regression tests.

## Reproduction Assessment

Yes, by source inspection: an attachment-only webhook with token and user but no text is rejected as missing required fields, and the delivery/buildContext path has no media field. This proves the requested capability is absent on current main, though it is a feature gap rather than a broken documented contract.

## Solution Assessment

Yes on direction, but not as an automated repair yet. The maintainable path is to keep Synology-specific parsing and staging inside the plugin, reuse the existing shared inbound media context, and first confirm Synology's attachment contract plus file-safety rules.
