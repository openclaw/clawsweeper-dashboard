## Summary

Keep open: current main still rejects attachment-only Synology Chat inbound webhooks before delivery, and adding file ingestion needs maintainer decisions around Synology's payload contract and download safety.

## What This Changes

Request inbound Synology Chat support for images, CSV files, and PDFs so attachment-bearing messages can reach agents.

## Best Possible Solution

Implement plugin-local inbound attachment parsing and safe staging for allowed image, CSV, and PDF files, pass media facts through the existing channel inbound context, and document/test the behavior after confirming Synology's contract.

## Reproduction Assessment

Yes, by source inspection: an attachment-only Synology webhook without text is rejected before delivery, and the downstream Synology message/context path has no media fields. I did not run a live Synology attachment webhook.

## Solution Assessment

Partly. The plugin-local direction is right and can use the existing channel media context seam, but the requested minor extension is not sufficient without confirming Synology's attachment contract and file-ingestion safety policy.
