## Summary

Current main still drops inbound Synology Chat attachments: webhook parsing requires text, the inbound event model has no media metadata, and neither tests nor docs define inbound image, CSV, or PDF handling. The issue remains necessary, but implementation should wait for maintainer approval of the Synology retrieval contract and safe-download policy.

## What This Changes

Add inbound image, CSV, and PDF attachment support to the Synology Chat plugin.

## Best Possible Solution

Confirm the official Synology attachment event and authenticated retrieval contract, then implement bounded media staging entirely within the Synology plugin and populate the existing generic inbound media context fields, with focused tests and documentation.

## Reproduction Assessment

Yes at source level: attachment-only events cannot pass the required-text parser, and the delivery/context path has no inbound media fields. A real NAS webhook payload was not provided, so the exact external attachment shape remains unverified.

## Solution Assessment

Unclear until Synology's attachment event and retrieval API are confirmed. If that contract supports the feature, plugin-local bounded download and generic media-context population is the narrowest maintainable solution.
