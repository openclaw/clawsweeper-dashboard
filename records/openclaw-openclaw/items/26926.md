## Summary

Keep open. Current main still supports Synology Chat media only on the outbound URL-delivery path; inbound webhook payloads, parsing, delivery, turn context, docs, and tests remain text-only, and no merged duplicate/fix for inbound attachments was found.

## What This Changes

Request inbound attachment support for images, CSV files, and PDFs in the Synology Chat plugin.

## Best Possible Solution

Implement plugin-owned inbound attachment parsing and safe staging after verifying Synology's contract, then feed media facts into the existing channel turn context with docs and focused tests.

## Reproduction Assessment

Yes, by source inspection: an attachment-only Synology webhook with a valid token and user but no text reaches parsePayload's text-required rejection, and no media facts are passed to the turn context. I did not run a live Synology webhook.

## Solution Assessment

Unclear on exact implementation until Synology's attachment contract and file-safety policy are verified. The maintainable direction is to keep Synology-specific parsing/download/staging inside the plugin and reuse the shared inbound media context.
