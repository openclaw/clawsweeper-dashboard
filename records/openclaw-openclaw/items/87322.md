## Summary

Keep open. Current main and v2026.6.9 still source-reproduce the Mattermost overwrite path, and the open closing PR is the active candidate fix, so this issue should remain open until that PR lands or is deliberately replaced.

## What This Changes

Mattermost should preserve block-streamed text segments as separate visible posts or accumulated preview content instead of overwriting prior text blocks in one draft post.

## Best Possible Solution

Ship a focused Mattermost plugin fix that preserves text/tool block boundaries under the existing streaming configuration, with separate follow-up tracking for error-response draft deletion if it stays out of scope.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: the normal Mattermost inbound path disables block delivery and the draft stream keeps editing `streamPostId`; I did not run a live Mattermost server in this read-only review.

## Solution Assessment

Yes. The maintainable path is Mattermost-local: honor existing blockStreaming when preview is off and rotate or detach draft previews at text/tool boundaries; the linked PR is the active implementation candidate.
