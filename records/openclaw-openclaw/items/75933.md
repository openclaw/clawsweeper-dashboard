## Summary

Keep open. Current main still has the Codex file-URL normalization bug, and the PR is a narrow, boundary-respecting fix with regression coverage and changelog coverage; remaining work is maintainer review and CI, not cleanup closure.

## What This Changes

The PR changes the Codex conversation turn input builder to decode safe local `file://` image paths, fall back to remote image URLs when local conversion fails, and add regression tests plus a changelog entry.

## Best Possible Solution

Land this PR after normal maintainer review and CI, keeping file URL normalization in the Codex adapter through the public plugin SDK file-access helper.

## Reproduction Assessment

Yes, from current-main source inspection plus a Node URL semantics probe. The adapter forwards raw `.pathname` values for `file://` paths, which leaves encoded spaces and can treat remote-host file URLs as local-looking paths before Codex receives them.

## Solution Assessment

Yes. Reusing `safeFileURLToPath` via the plugin SDK subpath is the narrowest maintainable fix, and falling back to `mediaUrl` only when local conversion fails preserves existing local-first behavior.
