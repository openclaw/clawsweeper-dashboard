## Summary

Keep open. Current main and the latest stable release still gate Telegram DM-topic auto-renaming on `!entry?.systemSent`, so a topic whose session entry is already marked bootstrapped can skip `generateTopicLabel` and `editForumTopic` and keep the fallback title.

## What This Changes

Telegram DM topics should receive the generated `autoTopicLabel` title after the first real message instead of keeping the provisional fallback title.

## Best Possible Solution

Keep the repair inside the Telegram plugin by recording an idempotent per-DM-topic label attempt independently of core `systemSent`, while preserving default-on config, direct-over-account precedence, `RawBody` input, and fire-and-forget delivery.

## Reproduction Assessment

Yes, at source level. A dispatch-level test can preload the existing DM-topic harness with `SessionKey`, `RawBody`, a visible final response, and a session-store entry where `systemSent: true`; current main will skip `generateTopicLabel` and `editForumTopic`.

## Solution Assessment

Not applicable to a proposed patch because this is an issue. The maintainable fix is to decouple Telegram topic-label attempts from core prompt/session bootstrap state instead of changing global first-turn or `systemSent` semantics.
