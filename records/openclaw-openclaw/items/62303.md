## Summary

Keep open. Current main still gates Telegram DM topic auto-labeling on the session-store `systemSent` flag, and the generated-label plus `editForumTopic` rename path remains unreachable when that flag is already true before the first meaningful topic label attempt.

## What This Changes

The issue asks to restore Telegram DM topic auto-renaming so new DM topics receive the generated `autoTopicLabel` title instead of keeping the provisional first-message title.

## Best Possible Solution

Keep the fix in the Telegram plugin by tracking an idempotent per-topic label attempt/state that is separate from core `systemSent`, while preserving the existing default-on config behavior, direct-vs-account config precedence, `RawBody` input, and fire-and-forget rename delivery.

## Reproduction Assessment

Yes. A focused unit test can dispatch a DM-topic message with `SessionKey`, `RawBody`, a visible final response, and a session-store entry where `systemSent: true`; current main will skip `generateTopicLabel` and `editForumTopic` because `isFirstTurnInSession` is false.

## Solution Assessment

No patch is proposed here. The narrow maintainable fix is to decouple Telegram topic-label attempts from core session prompt state rather than changing core first-turn/session semantics for all channels.
