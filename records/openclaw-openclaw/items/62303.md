## Summary

Keep open. Current main still gates Telegram DM topic auto-labeling on the core session `systemSent` flag, so a DM topic whose session already has setup state can skip `generateTopicLabel` and never reach the `editForumTopic` rename path; no merged fix or superseding tracker is evident from the provided issue context or current code.

## What This Changes

The issue reports that Telegram DM forum topics keep their provisional title because `autoTopicLabel` no longer reliably renames new topics after the first user message.

## Best Possible Solution

Keep the repair inside the Telegram plugin by recording an idempotent per-DM-topic label attempt or equivalent plugin-owned state, independent of core `systemSent`, while preserving config precedence and fire-and-forget rename behavior.

## Reproduction Assessment

Yes. A focused dispatch test can load a DM-topic session entry with `systemSent: true`, send the first meaningful user message with `RawBody`, and assert that current main skips `generateTopicLabel` and `editForumTopic`.

## Solution Assessment

No patch is proposed here. The best repair is a narrow Telegram plugin change that decouples topic-label attempts from core prompt/session setup semantics rather than changing global first-turn behavior.
