## Summary

Keep open: current main still has the documented pairing default, and changing it after owner bootstrap is a cross-channel security-default decision rather than a broken-contract bug fix.

## What This Changes

Request that unset channel DM policy auto-harden from pairing to an owner-derived allowlist after first-owner pairing bootstrap.

## Best Possible Solution

Make an explicit maintainer decision on the post-bootstrap DM default, then implement the approved policy centrally with per-channel/account tests and updated pairing/security docs.

## Reproduction Assessment

Yes for current source behavior: unset DM policy resolves to pairing, and an unallowlisted direct sender with mayPair reaches the pairing-required branch. I did not run a live channel because the shared source path and docs are direct enough for triage.

## Solution Assessment

No as an automated bug-fix lane: the proposed hardening is plausible, but it changes documented cross-channel defaults and needs maintainer agreement on owner-to-channel mapping and group-policy semantics.
