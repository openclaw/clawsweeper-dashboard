## Label Justifications

- `P2`: This is a normal memory/session artifact correctness improvement with limited blast radius and an active implementation path, not an urgent runtime outage.
- `impact:session-state`: The issue concerns session-memory artifact timestamps and filenames used to correlate prior conversation state across sessions.

## Summary

Keep open. Current main and the latest release still format bundled `session-memory` filenames, fallback timestamp slugs, and `# Session` headers from the host/process timezone rather than `agents.defaults.userTimezone`, and the active implementation path remains open in https://github.com/openclaw/openclaw/pull/49968.

## What This Changes

Respect `agents.defaults.userTimezone`, or a deliberate memory-specific timezone contract, when naming and timestamping session-memory files.

## Best Possible Solution

Adopt one configured-timezone contract for session-memory filenames, fallback slugs, and headers, preferably through the reviewed human-day path or a focused replacement that preserves background writes, opt-in LLM slugging, collision-safe filenames, and updated docs/tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `agents.defaults.userTimezone` to `Asia/Jakarta`, run with a different host `TZ`, trigger `session-memory`, and current main still formats the filename/header from the host/process timezone path.

## Solution Assessment

Unclear as a landing vehicle. Respecting the resolved user timezone for this surface is the right end state, but maintainers need to choose between the broad human-day PR, a focused session-memory fix, or a dedicated memory-specific timezone contract.
