## Summary

Keep open: current main and the latest release fix the older UTC-vs-host-local session-memory problem, and auto-flush daily paths already use `agents.defaults.userTimezone`, but the bundled `session-memory` hook still formats filenames and headers from host/env timezone rather than the configured user timezone. The remaining work needs maintainer confirmation of the memory artifact timezone contract before automation changes shipped behavior.

## What This Changes

Request memory artifact filenames and headers, especially bundled `session-memory` files, to use `agents.defaults.userTimezone` or a dedicated memory timezone setting instead of UTC or host time.

## Best Possible Solution

Define one memory artifact timezone contract, preferably reusing `agents.defaults.userTimezone` without adding a new config key, then update the session-memory handler, focused tests, and docs to match that contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `agents.defaults.userTimezone` differently from host `TZ`, trigger bundled `session-memory`, and the handler still formats the filename/header from host/env timezone rather than config.

## Solution Assessment

Unclear until maintainers confirm the product contract. The narrow technical fix is likely to reuse the existing configured-timezone helper in `session-memory`, but that changes shipped host-local artifact semantics.
