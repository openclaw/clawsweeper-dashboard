## Summary

Keep open. Current main moved `session-memory` from UTC to host-local timestamps, but the hook still does not read `agents.defaults.userTimezone`, so configured timezone support for generated memory filenames and `# Session` headers remains missing.

## What This Changes

Make generated `session-memory` files use `agents.defaults.userTimezone` for date prefixes, fallback time slugs, and `# Session` headers instead of UTC or host-local time.

## Best Possible Solution

Update the bundled `session-memory` hook to resolve `agents.defaults.userTimezone`, use that resolved zone for the filename date, fallback HHMM slug, and `# Session` header, preserve host-timezone fallback semantics, and align docs/tests/changelog.

## Reproduction Assessment

Yes. Exercise the hook with timestamp `2026-04-09T00:44:38.000Z`, `cfg.agents.defaults.userTimezone: "Asia/Jakarta"`, and a different host `TZ`; source inspection shows the output follows `process.env.TZ` instead of the configured timezone.

## Solution Assessment

Yes. Reusing the existing `resolveUserTimezone` semantics in `session-memory` is the narrowest maintainable fix; adding a new `memoryFlushTimezone` key would be unnecessary for this report.
