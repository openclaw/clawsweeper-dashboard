## Summary

Keep open. Current main shipped the related UTC-to-host-local fix, but the bundled `session-memory` hook still does not consult `agents.defaults.userTimezone`; its filename date, fallback HHMM slug, and `# Session` header all derive from the process/host timezone path.

## What This Changes

Respect `agents.defaults.userTimezone` when naming and heading memory files written by session-memory and related memory flush flows.

## Best Possible Solution

Update the bundled `session-memory` hook to resolve `agents.defaults.userTimezone` through the existing timezone helper, use that resolved zone for the filename date, fallback HHMM slug, and `# Session` header, preserve host-timezone fallback semantics, and document the hook behavior.

## Reproduction Assessment

Yes. A focused reproduction can trigger the hook with timestamp `2026-04-09T00:44:38.000Z`, `cfg.agents.defaults.userTimezone: "Asia/Jakarta"`, and a different host `TZ`; current source shows the formatter never reads that config, so the output follows the host/process timezone instead.

## Solution Assessment

Yes. The narrowest maintainable solution is to reuse the existing `resolveUserTimezone` semantics in `session-memory` rather than adding a new config key or changing unrelated memory-core flush behavior, which already uses `userTimezone`.
