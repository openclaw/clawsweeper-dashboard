## Summary

Keep open: current main and v2026.6.10 still format bundled `session-memory` filenames and `# Session` headers from host/env local time, not `agents.defaults.userTimezone`; memory-core flush paths already use `userTimezone`, so the remaining gap is a product/compatibility decision for session-memory artifacts.

## What This Changes

Request that memory artifact filenames and session headers respect `agents.defaults.userTimezone` instead of UTC or host-local time.

## Best Possible Solution

Define one memory artifact timezone contract, preferably reusing `agents.defaults.userTimezone` without adding a new config key, then update the session-memory handler, focused tests, and docs while keeping reset archive filename work separate.

## Reproduction Assessment

Yes at source level: set `agents.defaults.userTimezone` differently from host `TZ`, trigger bundled `session-memory`, and the handler still formats the filename/header from host/env local time. I did not run a live end-to-end reset flow in this read-only review.

## Solution Assessment

Unclear until maintainers confirm the product contract. Reusing existing `userTimezone` is likely the narrowest implementation if accepted, while adding a dedicated memory timezone config would expand an already-large config surface.
