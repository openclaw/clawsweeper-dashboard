## Summary

Keep open: current main and v2026.6.10 still make bundled session-memory filenames and headers follow host/env local time, not `agents.defaults.userTimezone`. The auto-flush path now uses the configured timezone, so there is a real consistency gap, but changing shipped memory artifact naming needs maintainer product and compatibility approval.

## What This Changes

The issue requests that session-memory filenames and markdown headers use the configured user timezone instead of UTC or host-local time.

## Best Possible Solution

Define one memory artifact timezone contract; if accepted, reuse `agents.defaults.userTimezone` in session-memory, keep auto-flush behavior consistent, and update focused tests plus docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `agents.defaults.userTimezone` differently from host `TZ`, trigger bundled `session-memory`, and the handler still formats the filename/header from host/env timezone rather than config.

## Solution Assessment

Unclear until maintainers confirm the product contract. The narrow technical fix is likely to reuse the existing configured-timezone helper in `session-memory`, while a separate `memoryFlushTimezone` knob would add config surface and needs stronger justification.
