## Summary

Keep open. Current main still ignores `agents.defaults.userTimezone` for the bundled `session-memory` filename date, fallback HHMM slug, and `# Session` header, while memory-core compaction flush paths already use the configured timezone; overlapping open PRs need maintainer coordination rather than issue cleanup.

## What This Changes

Request that generated memory file filenames and headers respect `agents.defaults.userTimezone` instead of UTC or host-local time.

## Best Possible Solution

Make `session-memory` use the existing configured-timezone helpers for filenames, fallback slugs, and headers while preserving backgrounded writes and opt-in model slugging, then update focused docs and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `agents.defaults.userTimezone` to `Asia/Jakarta`, run with a different host `TZ`, trigger `session-memory`, and current main will still format from the host/process timezone.

## Solution Assessment

Yes. Reusing `resolveUserTimezone`-style semantics in `session-memory` is the narrow maintainable direction; the main open question is whether maintainers want that as a focused fix or via the broader human-day PR.
