## Summary

Keep open. Current main still lacks a manifest-level install-time config patch contract, the only direct implementation attempt is closed unmerged, and the remaining work is a product/security design decision rather than a narrow repair.

## What This Changes

The issue requests a manifest-level `configPatch` field so plugin install can apply required OpenClaw config changes automatically.

## Best Possible Solution

Define a documented, auditable plugin install config-intent contract with allowlisted surfaces, validation before write, visible diff and skip path, conflict and array rules, uninstall guidance, and tests across every install route.

## Reproduction Assessment

No runtime bug reproduction is required for this feature request. Source inspection gives a high-confidence reproduction of the missing capability: manifests are not parsed for `configPatch`, install results cannot carry it, and install persistence does not apply it.

## Solution Assessment

Yes, the best path is a constrained config-intent design rather than a raw deep-merge manifest field. The earlier direct PR showed the wiring is plausible, but it left product and security choices unresolved.
