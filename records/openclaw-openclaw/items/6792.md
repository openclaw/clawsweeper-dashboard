## Summary

Keep open. Current main still lacks a manifest-level install-time config patch or safer config-intent contract, and the issue discussion already identifies the remaining third-party config mutation design as product/security-sensitive.

## What This Changes

Request allowing plugins to declare install-time config changes in `openclaw.plugin.json` so `openclaw plugins install` can apply required user config automatically.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract with constrained targets, validation before commit, exact diff display, a skip path, conflict and array semantics, and uninstall guidance provenance.

## Reproduction Assessment

Not applicable as a feature request rather than a bug reproduction. There is a high-confidence current-main verification path by inspecting the manifest type/loader, install result, install persistence, install callers, provider-auth prior art, and docs.

## Solution Assessment

No. The raw automatic deep-merge proposal is not yet the best maintainable solution; a constrained config-intent contract is safer for third-party install-time config mutation.

## Security Review

Status: needs_attention

Summary: The requested feature would let third-party plugin metadata persistently mutate user config during install, so it needs explicit trust, audit, and rollback guardrails.

Concerns:

- **[medium] Define trust rules for manifest-driven config writes:**
  - body: A manifest-level install patch could redirect tools, providers, hooks, channels, or model behavior in user config. Maintainers should define allowed paths, validation, confirmation or skip behavior, exact diff display, conflict handling, array semantics, and provenance before accepting an implementation.
