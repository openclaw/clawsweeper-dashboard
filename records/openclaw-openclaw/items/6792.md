## Summary

Keep open. Current main still lacks a manifest-level install-time config patch contract, and the related implementation PR was closed unmerged, so this remains a feature/API design item rather than conservative cleanup.

## What This Changes

Add a plugin manifest field that lets plugin installs automatically apply declared OpenClaw config changes.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract that validates before commit, shows a diff with a skip path, defines conflict and array behavior, records provenance for uninstall guidance, and covers all install routes with regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main shows the manifest field and install-time application path still do not exist.

## Solution Assessment

Unclear as proposed: automatic deep merge from arbitrary third-party manifests is too broad by itself. A safer solution is a narrower config-intent flow with validation, visible diffs, skip behavior, conflict and array rules, and uninstall provenance.
