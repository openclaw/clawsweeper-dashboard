## Summary

Keep open. Current main still lacks a manifest-level install-time config patch or safer config-intent path, and the remaining work needs product/security design before implementation.

## What This Changes

Request to let native plugin manifests declare an install-time `configPatch` so `openclaw plugins install` can update user config automatically.

## Best Possible Solution

Define a documented install-time plugin config-intent contract with allowed surfaces, pre-commit validation, visible diffs, skip behavior, conflict and array semantics, uninstall guidance provenance, and coverage for every install route.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main confirms manifests, install results, install callers, and install persistence still have no install-time manifest `configPatch` path.

## Solution Assessment

No as proposed: an unrestricted automatic deep-merge from third-party manifests is too broad. A narrower config-intent flow with explicit policy, validation, visible diff, skip behavior, conflict rules, and provenance is the safer direction.
