## Summary

Keep open. Current main still lacks a manifest-level install-time config patch or safer config-intent contract, and the related implementation PR is closed unmerged, so this remains a product/security design item rather than conservative cleanup.

## What This Changes

Add a plugin manifest field that lets `openclaw plugins install` apply declared config changes automatically during plugin installation.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract with allowed surfaces, validation before commit, visible diff and skip behavior, conflict and array rules, uninstall provenance, and coverage for every install route.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main shows plugin manifests, install results, and install persistence still have no install-time `configPatch` path.

## Solution Assessment

Unclear as proposed: raw automatic deep-merge from third-party manifests is too broad by itself. A safer solution is a narrower config-intent flow with validation, visible diffs, skip behavior, conflict and array rules, and uninstall provenance.
