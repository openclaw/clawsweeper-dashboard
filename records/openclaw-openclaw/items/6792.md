## Summary

Keep open. Current main still lacks a manifest-level install-time config patch or safer config-intent contract, and the related implementation PR was closed unmerged, so this remains product/security design work rather than conservative cleanup.

## What This Changes

The issue requests a manifest-level `configPatch` contract so plugin installs can apply required OpenClaw config changes automatically.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract with allowed surfaces, validation before commit, visible diff and skip behavior, conflict and array semantics, uninstall guidance provenance, and tests across every install route.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main shows manifests, install results, and install persistence still have no install-time manifest `configPatch` path.

## Solution Assessment

Unclear as proposed: raw automatic deep-merge from third-party manifests is too broad by itself. The safer path is a narrower config-intent flow with explicit policy, validation, visible diffs, skip behavior, conflict rules, and provenance.
