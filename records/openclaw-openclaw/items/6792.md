## What This Changes

Requests a plugin-manifest contract that can apply validated configuration changes automatically during plugin installation.

## Best Possible Solution

Add a typed, owner-scoped install setup-intent contract that previews a validated diff and requires an explicit apply/skip decision, defines per-field conflict semantics, rejects secrets or unsafe global ownership, records provenance for upgrade/uninstall guidance, and works consistently across every install source.

## Risks / Open Questions

- Allowing third-party manifests to deep-merge arbitrary root configuration can overwrite user-selected providers, model defaults, tool policy, channel or hook settings, and plugin enablement without a trustworthy ownership boundary.
- Array behavior is inherently field-specific: replacement can silently discard existing entries, while union can preserve conflicting or security-sensitive policy, so a generic deep-merge rule is unsafe.
- Automatic application without an explicit preview/skip or consent step would hide a critical security decision during installation, contrary to the repository's setup direction.
- Without durable provenance and ownership constraints, uninstall and upgrade cannot distinguish plugin-applied values from later user edits, making safe rollback or guidance unreliable.
