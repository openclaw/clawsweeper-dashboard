## Summary

Keep open. Current main and the latest release still lack a manifest-level install-time config patch contract, and the remaining work is a product/security design decision rather than conservative cleanup.

## What This Changes

Request to let plugin manifests declare an install-time `configPatch` that `openclaw plugins install` can apply to user config automatically.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract with allowed surfaces, validation before write, visible diff/skip behavior, conflict and array rules, install provenance, uninstall guidance, and coverage for all install routes.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main and v2026.5.12 lack the requested manifest/install contract.

## Solution Assessment

No. The useful direction is clear, but the raw deep-merge manifest field proposed here is not the best final shape until OpenClaw defines the security, consent, conflict, array, and uninstall semantics.

Status: needs_attention

Summary: The requested contract is security-sensitive because third-party manifests would be able to influence persisted OpenClaw config during install.
