## Summary

Keep open. Current main still lacks a manifest-level install-time config mutation contract, and the remaining work is a product/security design decision rather than conservative cleanup.

## What This Changes

The issue requests a manifest-declared `configPatch` field that `openclaw plugins install` can apply to user config automatically.

## Best Possible Solution

Define a constrained, auditable install-time plugin config intent contract with allowed surfaces, pre-write validation, visible diff or skip behavior, conflict and array rules, install provenance, uninstall guidance, and coverage for every install route.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection on current main shows the requested manifest/install contract is absent, while provider-auth config patches and manual `openclaw config patch` do not cover plugin install manifests.

## Solution Assessment

No, not as an arbitrary deep merge from third-party manifests. The maintainable path is a constrained config intent contract with explicit security, consent, conflict, array, validation, and uninstall semantics.
