## Summary

Keep open: current main and v2026.6.5 still lack a manifest-level install-time config mutation contract, and the only direct implementation PR was closed unmerged. The remaining work is a product/security design decision, not conservative cleanup.

## What This Changes

The issue requests a plugin manifest `configPatch` or equivalent install-time config intent so `openclaw plugins install` can apply required user configuration automatically.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract rather than a raw silent patch: validate before commit, show the exact diff with a skip path, define conflict and array behavior, record provenance for uninstall guidance, and cover all install routes with contract tests.
