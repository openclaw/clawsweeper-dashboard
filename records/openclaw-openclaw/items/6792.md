## Summary

Current main still lacks an install-time plugin configuration proposal contract, and the only directly linked implementation closed unmerged after concrete concerns about unrestricted writes and destructive array semantics; this remains necessary but needs maintainer-owned product and security design.

## What This Changes

Request a plugin-manifest contract that can automatically apply validated configuration changes during plugin installation.

## Best Possible Solution

Define a typed, auditable plugin setup-intent contract that validates proposed changes, previews a diff, requires explicit consent or deliberate noninteractive opt-in, uses field-specific conflict semantics, records provenance, and provides uninstall guidance without granting arbitrary manifest-wide config authority.

## Reproduction Assessment

Not applicable; this issue requests a new install-time plugin capability rather than reporting behavior that violates a current documented or tested contract.

## Solution Assessment

No—the proposed unconditional arbitrary deep merge is not the best solution because it creates competing ownership of user configuration; a constrained, consented setup-intent contract better fits the existing manifest-first control plane.
