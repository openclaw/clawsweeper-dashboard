## Summary

Keep open. Current main still lacks a manifest-level install-time config patch contract, and the remaining work needs product and security design rather than conservative cleanup.

## What This Changes

Request to let plugins declare an install-time `configPatch` or equivalent config intent in `openclaw.plugin.json` so `openclaw plugins install` can update user config automatically.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract with allowed surfaces, validation before write, visible diff and skip behavior, conflict and array rules, install provenance, uninstall guidance, and coverage for all install routes.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks the requested manifest/install contract.

## Solution Assessment

No. The useful direction is clear, but a raw deep-merge manifest field is not the best final shape until OpenClaw defines security, consent, conflict, array, and uninstall semantics.

## Security Review

Status: needs_attention

Summary: The requested contract is security-sensitive because third-party manifests would be able to influence persisted OpenClaw config during install.

  - confidence: 0.9
