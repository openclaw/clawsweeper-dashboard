## Summary

Keep open. Current main still lacks a manifest-level install-time config mutation contract; source and docs only show provider-auth/manual config patch surfaces, and the related implementation at https://github.com/openclaw/openclaw/pull/43421 was closed unmerged with security and array-merge concerns still unresolved.

## What This Changes

The issue requests a new plugin manifest `configPatch` field that `openclaw plugins install` would merge into the user's OpenClaw config automatically.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract with allowed surfaces, validation before write, visible diff or skip behavior, conflict and array rules, install provenance, uninstall guidance, and coverage for every install route.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection show current main lacks the requested manifest/install contract; the existing provider-auth `configPatch` and manual `openclaw config patch` surfaces do not cover plugin install.

## Solution Assessment

No, not as a raw arbitrary deep merge from third-party manifests. The maintainable path needs a constrained, auditable config intent contract with security, consent, conflict, array, and uninstall semantics defined first.
