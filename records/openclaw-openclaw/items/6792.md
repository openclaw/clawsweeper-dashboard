## Summary

The usability gap still exists, but unrestricted automatic deep merging is not safe to implement without a maintainer product and security decision. Current main has richer setup metadata, setup-runtime wizards, config contracts, and scoped install persistence, yet no generic install-time configuration intent covering this service/tool use case; the closed implementation attempt also confirmed unresolved arbitrary-key and array-overwrite risks.

## What This Changes

Requests a manifest-declared install-time configuration intent that automatically updates validated OpenClaw configuration when a plugin is installed.

## Best Possible Solution

Define a typed, auditable plugin setup-intent contract integrated with the existing manifest/setup-runtime boundary: restrict writable domains, require preview and explicit confirmation or a clear skip path, validate before commit, define per-field conflict semantics, and record provenance for later review or uninstall guidance.

## Reproduction Assessment

Not applicable: this issue requests a new plugin installation capability, and current source confirms that no manifest-level install-time configuration mutation contract exists.

## Solution Assessment

No, not as an unrestricted automatic deep merge. The best direction is a constrained setup-intent API using current manifest/setup seams, with operator consent, field-aware conflict handling, validation, and provenance.
