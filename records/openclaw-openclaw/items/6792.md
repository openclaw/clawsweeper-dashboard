## Summary

Keep open. Current main still lacks a manifest-level install-time config patch or config-intent contract, and the linked implementation PR was closed unmerged, so this remains a feature/API design item rather than conservative cleanup.

## What This Changes

Request a plugin manifest `configPatch` or equivalent config-intent field so `openclaw plugins install` can apply required user config changes automatically.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract that validates before commit, shows a diff with a skip path, defines conflict and array behavior, records uninstall guidance provenance, and covers all install routes with regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main shows plugin manifests and install persistence still have no install-time `configPatch` path.

## Solution Assessment

Unclear as proposed: raw automatic deep-merge from third-party manifests is too broad by itself. A safer solution is a narrower config-intent flow with validation, visible diffs, skip behavior, conflict and array rules, and uninstall provenance.
