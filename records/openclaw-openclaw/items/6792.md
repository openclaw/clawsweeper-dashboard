## Summary

Keep open: current main and the latest release still lack a manifest-level install-time config patch or safer equivalent, and the remaining work is a product/security contract for third-party config mutation rather than stale cleanup.

## What This Changes

The issue requests a plugin manifest `configPatch` field that `openclaw plugins install` would apply to user config automatically.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract with allowed surfaces, explicit diff/skip behavior, validation before write, conflict and array semantics, provenance for uninstall guidance, and coverage across all install routes.
