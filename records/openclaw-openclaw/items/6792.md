## Summary

Keep open: current main and the latest release still lack a manifest-level install-time config intent contract, and the issue remains a product/security decision about third-party plugins mutating persisted user config.

## What This Changes

The issue requests a plugin manifest `configPatch` field that `openclaw plugins install` would apply to `openclaw.json` automatically with validation and visible install output.

## Best Possible Solution

Define a constrained install-time configuration intent contract that previews and validates changes, scopes plugin-owned surfaces, records provenance, and works across every plugin install route without silent arbitrary config writes.

## Reproduction Assessment

Yes for the missing capability: current-main source inspection shows manifests and install results do not parse or carry a manifest-level `configPatch`, and install persistence has no merge path for it. This is a source-reproducible feature gap rather than a broken existing behavior contract.

## Solution Assessment

Unclear as proposed. Manifest-declared config intent is a plausible plugin API direction, but automatic deep-merge writes need maintainer-approved security, ownership, diff, conflict, array, and uninstall semantics first.
