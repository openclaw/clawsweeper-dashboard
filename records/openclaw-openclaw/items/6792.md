## Summary

Keep open. Current main still lacks a manifest-level install-time config patch or safer config-intent contract, and the remaining design is product/security-sensitive rather than a narrow missing code path.

## What This Changes

Request a manifest-level install-time `configPatch` or equivalent config-intent field so plugins can apply validated config changes during `openclaw plugins install`.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract with validation before commit, exact diff display, a skip path, constrained targets, conflict and array semantics, and provenance for uninstall guidance across all install routes.

## Reproduction Assessment

Not applicable as a feature request rather than a bug reproduction. There is a high-confidence current-main verification path by inspecting the manifest type/loader, install result, install persistence, install callers, provider-auth prior art, and manifest docs.

## Solution Assessment

No. The raw automatic deep-merge proposal is not yet the best maintainable solution without guardrails; a constrained config-intent contract is safer for third-party install-time config mutation.
