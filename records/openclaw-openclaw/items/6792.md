## Summary

Keep open. Current main still lacks a manifest-level install-time config patch contract, and the related implementation PR #43421 was closed unmerged; the remaining work is product/security design for third-party config mutation, not conservative cleanup.

## What This Changes

The issue requests a manifest-level install-time config patch mechanism so plugins can apply required OpenClaw config changes during installation.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract that validates before commit, shows the proposed diff with a skip path, records provenance for uninstall guidance, and specifies conflict and array semantics across all install routes.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main still lacks the requested manifest and install persistence contract.

## Solution Assessment

No, a raw arbitrary deep-merge `configPatch` is not yet clearly the best maintainable solution. The safer direction is a narrower documented config intent contract with validation, diff/skip UX, provenance, and tests across install routes.
