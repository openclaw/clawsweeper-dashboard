## Summary

Keep open: current main and v2026.6.10 still lack a manifest-level install-time config intent path, and the closed implementation PR did not resolve the remaining product/security contract.

## What This Changes

The issue requests a plugin manifest `configPatch` field that `openclaw plugins install` can validate, merge into user config, and report during installation.

## Best Possible Solution

Define a documented, constrained plugin install config-intent contract with allowed surfaces, validation before write, visible diff and skip behavior, conflict/list rules, uninstall guidance, and tests across every install route.

## Reproduction Assessment

Yes, at source level. A manifest containing `configPatch` would currently be ignored because manifests do not parse it, install results cannot carry it, and install persistence only records/enables plugins.

## Solution Assessment

No for the raw deep-merge proposal as written. The maintainable solution is a constrained config-intent contract that keeps setup explicit, validates before commit, and records enough provenance for later review or uninstall guidance.
