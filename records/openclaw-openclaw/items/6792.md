## Summary

Keep open. Current main still lacks a manifest-level install-time `configPatch` or safer config-intent contract, and the remaining work needs product/security design rather than conservative cleanup.

## What This Changes

The issue requests a plugin manifest `configPatch` field that `openclaw plugins install` would apply to user config at install time.

## Best Possible Solution

Define a documented, auditable install-time plugin config-intent contract with allowed surfaces, pre-commit validation, visible diffs, skip behavior, conflict and array semantics, uninstall guidance provenance, and tests across every install route.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main gives high confidence that manifests, install results, install callers, and install persistence still have no install-time manifest `configPatch` path.

## Solution Assessment

No as proposed: raw automatic deep-merge from third-party manifests is too broad by itself. The safer path is a narrower config-intent flow with explicit policy, validation, visible diffs, skip behavior, conflict rules, and provenance.
