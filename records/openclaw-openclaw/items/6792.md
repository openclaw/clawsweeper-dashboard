## Summary

Keep open: current main and v2026.6.5 still lack a manifest-level install-time config mutation contract, and the prior implementation PR was closed unmerged. The remaining work is product and security design, not conservative cleanup.

## What This Changes

Request to let plugin manifests declare an install-time config patch that plugin install would validate, merge into OpenClaw config, and show to the operator.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract that validates before commit, shows the exact diff with a skip path, scopes allowed writes, records provenance, and covers every install route with contract tests.

Not applicable as a bug reproduction: this is a requested plugin manifest/install capability. Current-main source inspection confirms the requested manifest field and install-time mutation path are absent.

## Solution Assessment
