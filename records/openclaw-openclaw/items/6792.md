## Summary

Keep open. Current main still does not define a manifest-level install-time config patch contract, and the remaining work is a product/security design decision for third-party config mutation rather than conservative cleanup.

## What This Changes

Request a plugin manifest `configPatch` field that `openclaw plugins install` can apply to user config automatically with validation and visible merge behavior.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract with allowed surfaces, validation before write, visible diff/skip behavior, conflict and array rules, install provenance, uninstall guidance, and coverage for all install routes.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main and v2026.5.12 lack the requested manifest/install contract.

## Solution Assessment

No. The useful direction is clear, but a raw deep-merge manifest field is not the best final shape until OpenClaw defines security, consent, conflict, array, and uninstall semantics.
