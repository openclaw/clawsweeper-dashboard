## Summary

Keep open. Current main and v2026.5.12 still lack a manifest-level install-time config patch contract, and the remaining work is a product/security decision for third-party config mutation rather than conservative cleanup.

## What This Changes

The issue requests a manifest-level `configPatch` field that `openclaw plugins install` can apply to user config during plugin installation.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract with allowed surfaces, validation before write, visible diff or skip behavior, conflict and array rules, install provenance, uninstall guidance, and coverage for all install routes.

## Reproduction Assessment

Not applicable as a feature request. Source, docs, and latest-release inspection show that current main and v2026.5.12 lack the requested manifest/install contract.

## Solution Assessment

No. The direction is useful, but a raw arbitrary `configPatch` deep merge is not the best final shape until OpenClaw defines security, consent, conflict, array, and uninstall semantics.
