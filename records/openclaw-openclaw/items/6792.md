## Summary

Keep open. Current main still lacks a manifest-level install-time config patch or safer config-intent contract, and the discussion already identifies unresolved product/security semantics for third-party config mutation.

## What This Changes

Add a plugin-manifest `configPatch` field that `openclaw plugins install` would automatically merge into the user's config.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract with constrained targets, validation before commit, exact diff display, a skip path, conflict and array semantics, and uninstall guidance provenance.

## Reproduction Assessment

Not applicable as a feature request rather than a bug reproduction. There is a high-confidence current-main verification path by inspecting the manifest type/loader, install result, install persistence, install callers, provider-auth prior art, docs, and the closed related PR.

## Solution Assessment

No. The raw automatic deep-merge proposal is not yet the best maintainable solution; a constrained config-intent contract is safer for third-party install-time config mutation.

## Security Review

Status: needs_attention

Summary: The request would let third-party plugin metadata persistently mutate user config during install, so it needs explicit trust, audit, and rollback/skip guardrails.
