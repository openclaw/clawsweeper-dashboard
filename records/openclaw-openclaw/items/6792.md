## Summary

Keep open. Current main still lacks a manifest-level install-time config patch or safer config-intent contract, and maintainer discussion already flagged the remaining third-party config mutation design as product/security-sensitive.

## What This Changes

The issue requests a manifest-level install-time config patch mechanism so plugins can apply required OpenClaw config during installation.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract with constrained targets, validation before commit, exact diff display, a skip path, conflict and array semantics, and uninstall guidance provenance.

## Reproduction Assessment

Not applicable as a feature request rather than a bug reproduction. There is a high-confidence current-main verification path by inspecting the manifest type/loader, install result, install persistence, install callers, provider-auth prior art, and manifest docs.

## Solution Assessment

No. The raw automatic deep-merge proposal is not yet the best maintainable solution; a constrained config-intent contract is safer for third-party install-time config mutation.
