## Summary

Keep open. Current main still lacks a manifest-level install-time config mutation contract, and the already-linked implementation attempt was closed unmerged while consent, conflict handling, array semantics, provenance, and security boundaries remain unresolved.

## What This Changes

Requests a `configPatch` field in plugin manifests so `openclaw plugins install` can apply plugin-declared config changes automatically.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract with allowed surfaces, validation before write, visible diff or skip behavior, conflict and array rules, install provenance, uninstall guidance, and coverage for every install route.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection show current main lacks the requested manifest/install contract; provider-auth `configPatch` and manual `openclaw config patch` do not cover plugin install manifests.

## Solution Assessment

No, not as an arbitrary deep merge from third-party manifests. The maintainable path is a constrained, auditable config intent contract with explicit security, consent, conflict, array, and uninstall semantics.
