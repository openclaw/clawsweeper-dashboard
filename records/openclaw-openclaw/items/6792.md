## Summary

Keep open. Current main and v2026.6.5 still do not have a manifest-level install-time config mutation contract, and the remaining question is a product/security design choice rather than conservative cleanup.

## What This Changes

Request to let plugin manifests declare an install-time config patch so plugin installation can update required OpenClaw config automatically.

## Best Possible Solution

Define a constrained, auditable install-time plugin config-intent contract with allowed surfaces, pre-write validation, visible diff or skip behavior, conflict and array rules, install provenance, uninstall guidance, and coverage for every install route.

## Reproduction Assessment

Not applicable as a feature request. Current-main source and docs inspection confirms the requested manifest/install contract is absent, while provider-auth config patches and manual config patch flows do not cover plugin install manifests.

## Solution Assessment

No, not as an arbitrary automatic deep merge from third-party manifests. The maintainable path is a constrained config-intent contract with explicit security, consent, conflict, array, validation, and uninstall semantics.
