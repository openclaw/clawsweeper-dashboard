## Summary

Keep open. Current main and the latest release still lack a manifest-level install-time config patch contract, the direct implementation PR is closed unmerged, and the remaining blocker is maintainer product/security direction for third-party config mutation.

## What This Changes

Request adding a manifest-declared install-time `configPatch` so plugin installs can apply validated user config changes automatically.

## Best Possible Solution

Define a constrained, documented install-time plugin config intent contract that validates before commit, shows the exact diff with a skip path, limits or classifies allowed surfaces, defines conflict and array semantics, records provenance for uninstall guidance, and covers every install route with regression tests.

## Reproduction Assessment

Yes for the missing capability: current source has no manifest `configPatch` field, install result payload, or persistence path to apply one. This is a source-reproducible feature gap rather than a broken existing behavior contract.

## Solution Assessment

No for the raw deep-merge proposal as written: arbitrary manifest-driven writes are too broad without a trust, consent, allowed-surface, audit, and uninstall contract. A constrained config-intent API is the maintainable direction.
