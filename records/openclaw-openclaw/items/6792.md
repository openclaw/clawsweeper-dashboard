## Summary

Keep open. Current main still lacks a manifest-level install-time config patch contract, and the prior linked implementation PR was closed unmerged, so this remains a product/security design item rather than conservative cleanup.

## What This Changes

The issue requests an `openclaw.plugin.json` `configPatch` field that `openclaw plugins install` would validate, merge into user config, and report to the user.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract that validates before commit, shows the user what will change, supports a skip path, records provenance for uninstall guidance, and covers all install routes with regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection gives high-confidence evidence that current main still lacks the requested manifest field and install-time application path.

## Solution Assessment

Unclear as proposed: a raw automatic deep merge is not enough by itself for a safe third-party install contract. The maintainable direction is a narrower, documented config-intent flow with validation, visible diffs, conflict rules, and install/uninstall provenance.
