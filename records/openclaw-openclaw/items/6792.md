## Summary

Current main still has no manifest-level install-time configuration mutation contract, and the existing declarative setup/config metadata only describes discovery, migrations, and plugin-owned configuration rather than applying arbitrary root configuration. The request remains useful and vision-aligned, but unrestricted automatic deep merge is not safe to implement without a maintainer product/security decision because it can overwrite user settings, arrays, provider routing, and credential-adjacent configuration; the prior implementation PR was closed unmerged after reviewers raised those same scope and overwrite concerns.

## What This Changes

Add a manifest-declared install-time configuration intent so plugin installation can configure required OpenClaw settings automatically.

## Best Possible Solution

Add a declarative, auditable plugin setup-intent contract with constrained operations, explicit preview/confirmation or opt-in application, schema validation, deterministic conflict and array rules, recorded provenance, and safe uninstall guidance instead of accepting arbitrary silent root-config deep merges.

## Risks / Open Questions

- A third-party manifest-driven patch could silently overwrite persisted settings, arrays, provider/model choices, auth-related configuration, plugin policy, or other plugins unless allowed operations and conflicts are tightly constrained.
- Applying configuration automatically during every install path would create upgrade and operator-surprise risk unless the CLI previews the exact changes, supports refusal, validates atomically, and records provenance.
- Uninstall or reinstall cannot safely infer which values remain plugin-owned after user edits, so automatic rollback would risk deleting or reverting legitimate user configuration.
