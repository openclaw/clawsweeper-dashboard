## Summary

Keep open. Current main still lacks a manifest-level install-time config patch or safer config-intent contract, and the related implementation PR at https://github.com/openclaw/openclaw/pull/43421 was closed unmerged, so the remaining work is product/security design rather than conservative cleanup.

## What This Changes

Request to let plugin manifests declare install-time config changes that `openclaw plugins install` validates, applies, and reports automatically.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract with allowed surfaces, validation before commit, a visible diff and skip path, conflict and array semantics, provenance for uninstall guidance, and tests across all install routes.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main shows plugin manifests, install results, and install persistence still have no install-time `configPatch` path.

## Solution Assessment

Unclear as proposed: raw automatic deep-merge from third-party manifests is too broad by itself. A safer solution is a narrower config-intent flow with validation, visible diffs, skip behavior, conflict and array rules, and uninstall provenance.
