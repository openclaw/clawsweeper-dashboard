## Summary

Keep open. Current main still lacks a manifest-level install-time `configPatch` or safer config-intent path, and the requested behavior needs product/security design for third-party install-time config mutation rather than conservative cleanup.

## What This Changes

Request to let plugin manifests declare an install-time `configPatch` so `openclaw plugins install` can apply needed config automatically.

## Best Possible Solution

Define a documented, auditable install-time plugin config-intent contract with allowed surfaces, pre-commit validation, visible diffs, skip behavior, conflict and array semantics, uninstall guidance provenance, and tests across every install route.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main gives high confidence that manifest parsing, install results, install callers, and install persistence still do not carry manifest-level `configPatch` data.

## Solution Assessment

No for the raw automatic deep-merge proposal; it would let third-party packages mutate arbitrary config without enough guardrails. A constrained config-intent contract with allowlisted surfaces, diff/skip behavior, validation, conflict and array rules, and uninstall provenance is the safer solution.
