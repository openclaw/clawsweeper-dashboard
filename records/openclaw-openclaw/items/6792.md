## Summary

Keep open. Current main still lacks a manifest-level install-time config patch contract, and the closed related implementation PR was not merged, so this remains a feature/API design item rather than conservative cleanup.

## What This Changes

The issue requests a new `configPatch` field in plugin manifests that `openclaw plugins install` would apply automatically to the user's config.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract that validates before commit, shows a diff with a skip path, defines conflict and array behavior, records provenance for uninstall guidance, and covers all install routes with regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main gives high-confidence evidence that the manifest field and install-time application path still do not exist.

## Solution Assessment

Unclear as proposed: automatic deep merge from arbitrary plugin manifests is not enough by itself for safe third-party install-time config mutation. The maintainable direction is a narrower documented config-intent flow with validation, visible diffs, skip behavior, conflict rules, and provenance.
