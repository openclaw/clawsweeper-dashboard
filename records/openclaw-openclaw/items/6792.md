## Summary

Keep open. Current main still lacks a manifest-level install-time config mutation contract, and the remaining work is a product/security decision rather than a narrow missing code path.

## What This Changes

The issue requests an `openclaw.plugin.json` `configPatch` field that `openclaw plugins install` can validate, display, and merge into user config during plugin installation.

## Best Possible Solution

Define a constrained, auditable install-time plugin config intent contract with allowed surfaces, validation before write, visible diff or skip behavior, conflict and array rules, install provenance, uninstall guidance, and coverage for every install route.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection show current main lacks the requested manifest/install contract; provider-auth `configPatch` and manual config patching do not cover plugin install manifests.

## Solution Assessment

No, not as an arbitrary deep merge from third-party manifests. The maintainable path is a constrained config intent contract with explicit security, consent, conflict, array, and uninstall semantics.
