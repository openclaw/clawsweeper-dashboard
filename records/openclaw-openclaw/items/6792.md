## Summary

Keep open. Current main and the latest release still lack a manifest-level install-time config intent/patch contract, and the remaining work is a product/security decision about third-party config mutation rather than a narrow cleanup close.

## What This Changes

The issue requests a manifest-level install-time `configPatch` contract so plugins can apply required OpenClaw config changes during `openclaw plugins install`.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract with validation before commit, visible diff/consent and skip behavior, conflict and array semantics, provenance for uninstall guidance, and coverage across every install route.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/API request. Source inspection does give high-confidence proof that current main lacks the requested manifest-level install-time config patch path.

## Solution Assessment

No: unrestricted automatic deep merge from third-party manifests is not the best final shape. A typed, auditable config intent contract with explicit validation, conflict rules, consent/skip behavior, and provenance is the safer direction.
