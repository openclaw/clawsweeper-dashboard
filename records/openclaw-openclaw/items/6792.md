## Summary

Keep open. Current main and v2026.5.12 still lack a manifest-level install-time config patch contract, and the remaining work is a product/security decision for third-party config mutation rather than conservative cleanup.

## What This Changes

Request to let plugin manifests declare install-time config changes that `openclaw plugins install` can validate, merge, display, and persist automatically.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract with allowed surfaces, validation before write, visible diff/skip behavior, conflict and array rules, install provenance, uninstall guidance, and coverage for all install routes.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection shows current main and v2026.5.12 lack the requested manifest/install contract.

## Solution Assessment

No. The useful direction is clear, but a raw arbitrary `configPatch` deep merge is not the best final shape until OpenClaw defines security, consent, conflict, array, and uninstall semantics.
