## Summary

Keep this open: current main still exposes only the fixed 30-minute Discord component callback TTL, and the open closing implementation candidate is https://github.com/openclaw/openclaw/pull/84189.

## What This Changes

The issue requests a bounded Discord `agentComponents.ttlMs` config field that preserves the 30-minute default and is forwarded into component registration for sends and edits.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/84189; the desired end state is a bounded documented TTL override that preserves the 30-minute default and proves send/edit propagation.

## Reproduction Assessment

Not applicable as a feature/config request rather than a broken existing contract. Source inspection confirms the current limitation: the registry accepts `ttlMs`, but Discord config and normal send/edit paths do not expose or pass it.

## Solution Assessment

Yes, the requested config-to-registration plumbing is the right boundary for this behavior. Maintainers should still decide whether the linked PR's TTL maximum is the right public contract.
