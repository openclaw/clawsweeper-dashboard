## What This Changes

Add a bounded Discord `agentComponents.ttlMs` config option while preserving the 30-minute default and forwarding the resolved value into component send/edit registration.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/84189; the end state should preserve the default TTL, bound the override, document it, and prove send/edit propagation.

## Reproduction Assessment

Not applicable as a feature/config request rather than a broken existing contract. Source inspection confirms the current limitation: the registry accepts `ttlMs`, but Discord config and normal send/edit paths do not expose or pass it.

## Solution Assessment

Yes, the requested config-to-registration plumbing is the right boundary for this behavior. Maintainers should still review the chosen maximum TTL and interaction-window tradeoff in the linked PR.
