## Summary

Keep open. The relaxed QMD retry and scope-denied debug work are useful and the live Discord proof is credible, but the PR widens default QMD memory recall into channel sessions, which is a compatibility and security-boundary change that needs explicit maintainer policy or an opt-in repair before merge.

## What This Changes

The PR adds a relaxed lexical retry for empty QMD searches, emits debug metadata for scope-denied QMD searches, broadens the default QMD scope to channel sessions, updates focused tests, and adds a changelog entry.

## Best Possible Solution

Keep the relaxed retry and scope-denied debug behavior, but leave channel QMD recall opt-in through memory.qmd.scope unless maintainers explicitly approve and document the broader default.

## Reproduction Assessment

Yes, source-level: current main allows only direct sessions in DEFAULT_QMD_SCOPE and QmdMemoryManager returns empty results before searching when a channel session is denied. I did not run a live QMD/Discord repro in this read-only review.

## Solution Assessment

No, not as submitted. The retry and debug changes are a good narrow repair, but changing default channel memory scope is a product/security boundary that should remain opt-in or be explicitly approved and documented.
