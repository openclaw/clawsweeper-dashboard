## What This Changes

The PR updates QMD memory recall to allow channel sessions in the default scope, retry zero-hit lexical QMD searches with a shorter priority-keyword query, add debug fallback reporting, and cover the behavior with focused tests plus a changelog entry.

## Best Possible Solution

Land the PR after normal CI and maintainer review if channel-session QMD recall is the intended product default; keep the retry bounded to lexical zero-hit QMD searches and preserve the existing session transcript visibility filter.

Yes. Current main reproduces the scope-denial path by resolving the default QMD scope to direct-only and returning `[]` before QMD execution for a channel session key; the PR adds tests for default channel allowance, scope-denied debug output, and the noisy lexical retry path.

## Solution Assessment

Yes, for the stated Discord channel QMD regression. The patch uses the existing QMD scope policy and search/debug seams, adds a conservative one-shot retry only for empty lexical searches, and avoids changing the separate active-memory timeout work tracked in #75761.
