The issue reports that before_tool_call plugin approvals cannot route back to the originating channel because turn-source metadata is not forwarded into plugin.approval.request.

## Best Possible Solution

## Reproduction Assessment

Yes. Current main source shows the before_tool_call plugin approval request omits turn-source fields, and the gateway no-route path returns decision:null when no approval client, delivery, or turn-source route exists.

## Solution Assessment

Yes. Forwarding the existing HookContext turn-source data into the existing plugin.approval.request protocol is the narrow maintainable fix; the open linked PR follows that shape instead of adding a new routing API.
