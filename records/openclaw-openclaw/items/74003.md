## Summary

Keep open: current main and the latest release still omit turn-source routing from before_tool_call plugin approval requests, while the direct fix is tracked by open PRs rather than merged code.

## What This Changes

The issue asks OpenClaw to forward originating channel routing metadata into `before_tool_call` plugin approval requests so approval prompts can be delivered back to chat channels.

## Best Possible Solution

Review and land or reject the focused route-forwarding PR so trusted HookContext turn-source metadata reaches `plugin.approval.request`, then close this issue from the merged fix.

## Reproduction Assessment

Yes, at source level: current main sends a plugin approval request without turn-source fields, and the gateway no-route branch returns `decision: null`, which maps to the reported error. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to carry trusted runtime route metadata through `HookContext` into `plugin.approval.request`; the issue comment correctly rejects reading it from model-generated tool params.
