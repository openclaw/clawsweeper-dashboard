## What This Changes

The issue requests fixing before_tool_call plugin approvals so the approval request carries the originating channel route instead of failing with no approval route.

## Best Possible Solution

Review and land or replace the focused route-forwarding PR so trusted HookContext turn-source metadata reaches plugin.approval.request with regression coverage, then close this issue from the merged fix.

## Reproduction Assessment

Yes at source level. A before_tool_call requireApproval request reaches plugin.approval.request without turn-source fields, and the gateway no-route branch returns decision:null; no live Telegram run was executed in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to carry trusted runtime route metadata through HookContext into plugin.approval.request; reading args.params would be wrong because those are model-generated tool parameters.
