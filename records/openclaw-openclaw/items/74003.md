## Summary

Keep open: current main still omits turn-source routing from before_tool_call plugin approval requests, and the direct fix is tracked by the open, unmerged pull request https://github.com/openclaw/openclaw/pull/90918 rather than already implemented on main.

## What This Changes

This issue requests forwarding the originating channel route from before_tool_call plugin approvals so plugin.approval.request can deliver the prompt back to the user’s channel.

## Best Possible Solution

Land or replace the open route-forwarding PR with a focused fix that carries trusted runtime turn-source metadata through HookContext into plugin.approval.request and covers the no-route failure path with regression tests.

## Reproduction Assessment

Yes. Source inspection shows before_tool_call requireApproval reaches plugin.approval.request without turn-source route fields, and the shared gateway path returns decision:null when no approval route exists.

## Solution Assessment

Yes. The narrow fix is to forward trusted runtime route fields from HookContext into plugin.approval.request; reading args.params would not solve it because those are model-generated tool parameters, not framework routing metadata.
