## Summary

Current main still matches the reported routing bug: channel route options reach the agent tool factory, but the `before_tool_call` approval context and `plugin.approval.request` payload still omit `turnSource*` fields, so the gateway can return `decision: null` when no other approval route is available.

## What This Changes

The issue asks for `before_tool_call` plugin approvals to carry originating channel route metadata into `plugin.approval.request`.

## Best Possible Solution

Add trusted route fields to the hook context from `createOpenClawCodingTools`, forward defined values to `plugin.approval.request`, and cover both trusted-policy and hook-return approval paths with regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: channel route options enter the tool factory, are dropped before the hook approval helper, and the gateway returns `decision: null` when it cannot deliver through any fallback route.

## Solution Assessment

Yes. The narrow maintainable fix is to use trusted runtime route context from the agent tool factory rather than model-generated tool params, then pass those fields through the existing gateway approval contract.
