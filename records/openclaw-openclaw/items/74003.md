## Summary

Keep this open. Current main still sends `plugin.approval.request` without trusted turn-source routing fields even though the gateway uses those fields to keep channel-origin approval prompts routable; the linked repair PR #74150 is closed unmerged.

## What This Changes

This issue asks OpenClaw to forward originating channel route metadata into plugin approval requests created by `before_tool_call` `requireApproval` hooks.

## Best Possible Solution

Carry the trusted originating channel fields through the agent hook context into `plugin.approval.request`, with regression tests for ordinary hook approvals and trusted-policy approvals while preserving fail-closed gateway behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: route options enter `createOpenClawCodingTools`, are dropped from the hook context, and the gateway already returns `decision: null` when no other approval route is available.

## Solution Assessment

Yes. The narrow maintainable fix is to forward trusted runtime context from `createOpenClawCodingTools`, not model-generated tool params, into `plugin.approval.request`.

## Repair Work Prompt

Fix the plugin approval routing bug tracked by https://github.com/openclaw/openclaw/issues/74003. Extend the trusted hook context in `src/agents/pi-tools.before-tool-call.ts` to carry `turnSourceChannel`, `turnSourceTo`, `turnSourceAccountId`, and `turnSourceThreadId`; populate those fields from `createOpenClawCodingTools` options in `src/agents/pi-tools.ts` using `messageProvider`, `messageTo`, `agentAccountId`, and `messageThreadId`; and forward present values to `plugin.approval.request` from `requestPluginToolApproval`. Cover both ordinary `before_tool_call` `requireApproval` and trusted-policy approval paths. Do not read route metadata from model-generated tool params, do not weaken gateway validation or no-route fail-closed behavior, and do not make Telegram-specific core changes. Add a user-facing `CHANGELOG.md` fix entry. Use #74150 as prior context but re-check current main before applying any code.
