## Summary

Keep open. Current `main` still has the before_tool_call plugin approval routing gap, and the only linked repair PR was closed unmerged after the params-based fix was identified as unsafe.

## What This Changes

The issue requests forwarding trusted turn-source routing metadata from `before_tool_call` plugin approvals to `plugin.approval.request` so same-chat approval routing works.

## Best Possible Solution

Thread trusted originating route metadata from the run/tool-construction context into `HookContext` or an equivalent trusted structure, then forward only that trusted metadata to `plugin.approval.request` with routed and route-less regression coverage.

## Reproduction Assessment

Yes, at source level. The agent has trusted route options before tool wrapping, the hook approval request drops them, and the gateway returns `decision: null` when no other approval route exists.

## Solution Assessment

Yes, if the fix uses trusted runtime context. The original issue sketch and closed PRâ€™s params/baseParams approach are not the best solution because tool params are model-controlled.
