## Summary

Keep this PR open, but the current patch should not merge: it broadens the trusted-plugin state boundary to community/private ClawHub records and still does not cover the reported Microsoft Teams npm-only official catalog path.

## What This Changes

The PR changes plugin manifest trust validation so ClawHub installs from official, community, and private channels can be marked trusted official installs when package metadata matches the official catalog, and adds registry tests plus a repro script.

## Best Possible Solution

Replace the channel-label expansion with a provenance-safe official install fix that handles npm-only official catalog entries or an approved portable install-record flow, while keeping arbitrary community/private ClawHub records untrusted.
