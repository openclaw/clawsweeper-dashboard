## Summary

Keep open: the PR is a useful start, but it is not merge-ready because the new per-agent storage still leaves visible render/bootstrap paths reading the default main-agent override, and the supplied screenshots manually write a `v2` localStorage key that this PR does not use.

## What This Changes

This PR changes Control UI assistant avatar persistence from one flat localStorage value to an `agents` map keyed by agent id, threads that id through the assistant-identity controller, and adds controller tests.

## Best Possible Solution

Land one canonical fix that scopes every local assistant avatar read and write by the active agent, preserves intentional upgrade behavior, and proves set, switch, clear, and reload in the real Control UI.

## Reproduction Assessment

Yes. Current main and the latest release both store assistant avatar overrides under one global localStorage key, and UI callers read that helper without an agent id, matching the linked report's cross-agent overwrite path.

## Solution Assessment

No. The storage/controller direction is plausible, but the best fix must update every visible reader, including render and bootstrap, and prove the browser behavior rather than only controller helpers.
