## Summary

Keep open. Current main and the latest release still lack Discord message update/delete listener wiring, edit-aware replay, and a source-message-to-active-run cancellation map; https://github.com/openclaw/openclaw/issues/20173, https://github.com/openclaw/openclaw/issues/55691, and https://github.com/openclaw/openclaw/pull/79855 cover overlapping slices but do not fully replace this Discord edit-plus-delete request.

## What This Changes

Request Discord support for edited messages to re-trigger processing and deleted messages to cancel the associated active agent session.

## Best Possible Solution

Define the Discord edit/delete policy, then implement it inside the bundled Discord plugin with typed update/delete listeners, hydration/filtering, edit-aware replay, precise source-message cancellation, config/docs alignment, and focused tests.

## Reproduction Assessment

Yes, at source level. Current main has no registered Discord MESSAGE_UPDATE or MESSAGE_DELETE listener path, and same-message edits would collide with replay dedupe keyed by account, channel, and message id.

## Solution Assessment

Unclear as an automatic repair. The Discord plugin is the right boundary, but maintainers should settle defaults, opt-in/config shape, debounce, partial update hydration, replay supersede behavior, and cancellation authorization first.
