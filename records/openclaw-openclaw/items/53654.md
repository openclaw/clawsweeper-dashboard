## Summary

Keep open. Current main still has no Discord inbound path for message edits or deletes, and the related Discord trackers cover only edit reprocessing, not delete-to-cancel.

## What This Changes

Request support for Discord message edits to reprocess bot mentions and message deletes to cancel the active run tied to the deleted Discord message.

## Best Possible Solution

Implement this in the bundled Discord plugin with typed update/delete dispatch support, edit-aware replay semantics, and a source-message active-run cancellation map.

## Reproduction Assessment

Yes. Source inspection on current main is high-confidence: MESSAGE_UPDATE or MESSAGE_DELETE would not match any registered Discord monitor listener, and same-id edit replay needs explicit handling beyond the current replay key.

## Solution Assessment

Yes. The narrow maintainable solution belongs in the Discord plugin, reusing the existing preflight, authorization, debouncer, and run queue paths with only minimal generic seams if source-message cancellation cannot stay local.
