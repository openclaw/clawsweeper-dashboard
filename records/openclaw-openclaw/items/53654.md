## Summary

Keep open. Current main and v2026.5.22 still lack Discord MESSAGE_UPDATE/MESSAGE_DELETE listener wiring, edit-aware replay semantics, and source-message-id-to-active-run cancellation; related open items cover overlapping pieces but do not fully replace this Discord-specific combined request.

## What This Changes

Add Discord message edit reprocessing and message delete cancellation so corrected or deleted source messages can update or abort the agent run they triggered.

## Best Possible Solution

Define the Discord edit/delete policy, then implement it in the bundled Discord plugin with typed update/delete listeners, hydration/filtering, edit-aware replay, precise source-message cancellation, config/docs alignment, and focused tests.

## Reproduction Assessment

Yes, at source level. Current main has Discord gateway event types available through the pinned dependency but no registered MESSAGE_UPDATE or MESSAGE_DELETE listener path, and same-message edits would collide with replay dedupe keyed by account, channel, and message id.

## Solution Assessment

Unclear as an automatic implementation. The bundled Discord plugin is the right boundary, but maintainers should settle defaults, config shape, debounce and partial-update hydration, replay supersede behavior, and cancellation authorization first.
