- `impact:message-loss`: Discord edit/delete gateway events are currently not delivered into the inbound agent workflow.

## Summary

Keep open. Current main still has no Discord MESSAGE_UPDATE or MESSAGE_DELETE inbound listener, no update/delete gateway hydration path, and no source Discord message-id cancellation map; the open edit-only and cross-channel recall trackers overlap but do not supersede this combined Discord request.

## What This Changes

Add Discord message edit and delete handling so edited messages can be reprocessed and deleted source messages can cancel active runs.

## Best Possible Solution

Define the Discord edit/delete semantics, then implement them inside the bundled Discord plugin with typed update/delete listeners, monitor registration, hydration/filtering, edit-aware replay, precise source-message cancellation, config/docs alignment, and focused tests.

## Reproduction Assessment

Yes, at source level. Current main has no registered Discord MESSAGE_UPDATE or MESSAGE_DELETE listener path, and same-message edits would collide with replay dedupe keyed by account/channel/message id.

## Solution Assessment

Unclear as an automatic repair. The Discord plugin is the right boundary, but maintainers should settle opt-in/defaults, debounce, partial update hydration, and cancellation authorization first.
