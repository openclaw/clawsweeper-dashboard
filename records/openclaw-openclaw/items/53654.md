## Summary

Keep open. Current main still has no Discord inbound path for messageUpdate or messageDelete, and the related edit-only trackers do not cover the broader delete-to-cancel behavior.

## What This Changes

Add Discord messageUpdate and messageDelete handling so eligible edits reprocess prompts and deletes cancel the active run started by that source message.

## Best Possible Solution

Implement and ship the Discord plugin behavior with regression tests for typed gateway dispatch, startup registration, edit replay, debounce/dedupe, and delete-to-cancel mapping.

## Reproduction Assessment

Yes. Source inspection on current main is high-confidence: Discord dispatches are delivered only to registered listeners, and no MessageUpdate or MessageDelete listener or startup registration exists.

## Solution Assessment

Yes. The narrow maintainable solution belongs in the bundled Discord plugin, reusing the existing preflight, authorization, debouncer, replay guard, and run queue paths with generic core seams only if source-message cancellation cannot stay local.

## Repair Work Prompt

Implement Discord edit-to-reprocess and delete-to-cancel for the bundled Discord plugin. Add typed MESSAGE_UPDATE and MESSAGE_DELETE listener support in the internal gateway/listener layer, register those listeners during Discord monitor startup, route eligible edited messages through the existing Discord preflight/mention/authorization/debounce path with explicit edit-aware replay semantics, and add a precise source Discord message-id to active-run cancellation map for delete-to-cancel. Keep the fix inside extensions/discord unless a generic core seam is strictly necessary; do not add bundled ids or Discord-specific policy to core. Account for partial MESSAGE_UPDATE payloads with hydration before mention/content decisions. Add focused regression tests and update user-facing docs/changelog for the new Discord behavior. Coordinate with #20173 and closed #38505, but keep scope to Discord messageUpdate/messageDelete handling for this cluster.
