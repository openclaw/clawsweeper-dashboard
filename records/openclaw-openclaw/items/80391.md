## Summary

Keep open: the wrapper hang is a source-reproducible bug and the timeout change is useful, but this PR is not ready because it still mixes in release-verification timeout changes and lacks contributor-provided real behavior proof.

## What This Changes

The PR adds a 5-second SIGKILL timeout to Crabbox wrapper sanity probes and also changes release/package verification limits, optional import allowances, and the workspace bootstrap smoke timeout.

## Best Possible Solution

Land a narrow Crabbox wrapper timeout with redacted real timeout proof, and handle any release-verifier unblock as separate release-owner-reviewed work.

## Reproduction Assessment

Yes, from source inspection: current main runs the Crabbox wrapper `--version` and `run --help` probes with unbounded `spawnSync`, so a selected binary that never exits will block before the requested command starts. I did not run a fake-binary repro because this was a read-only review.

## Solution Assessment

No, not as submitted: the wrapper timeout itself is the narrow maintainable fix, but the shared release smoke timeout should be split or explicitly reviewed as release-path work. The PR also needs real after-fix behavior proof before merge.
