## Summary

Keep open for maintainer review: the patch is small, clean, proof-supported, and current checks are green; the remaining question is whether maintainers accept the 5-second fail-fast bound in this automation wrapper.

## What This Changes

The PR adds a 5-second `SIGKILL` timeout and timeout-status mapping to the Crabbox wrapper's shared `checkedOutput` helper, affecting sanity probes and broker config inspection.

## Best Possible Solution

Land the bounded preflight if maintainers want stale or hung Crabbox binaries to fail fast; otherwise adjust the timeout constant or narrow the helper scope before merge.

## Reproduction Assessment

Yes. Current main has unbounded `spawnSync` sanity probes, so a selected Crabbox binary that hangs in `--version` or `run --help` can block the wrapper before it reaches the sanity-check failure path.

## Solution Assessment

Yes, with maintainer acceptance of the timeout value. A bounded timeout in the shared helper is a narrow fix for the hang, but the 5-second budget also applies to broker config inspection and should be intentional.
