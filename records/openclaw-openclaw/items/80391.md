## Summary

## What This Changes

The branch adds a 5-second SIGKILL timeout to Crabbox wrapper sanity probes and also changes release/postpublish verification timeout and optional-import allowlist behavior.

## Best Possible Solution

Keep the Crabbox timeout patch narrow, split or explicitly justify the release-verifier changes, and land only after redacted real wrapper output proves the timeout path.

## Reproduction Assessment

Yes, from source inspection: current main uses unbounded `spawnSync` for the wrapper’s `--version` and `run --help` probes, so a selected Crabbox binary that never exits will block before the wrapper reaches the requested command. I did not run a full fake-binary repro in this read-only review.

## Solution Assessment

No, not as submitted: the Crabbox timeout itself is a narrow maintainable fix, but the branch also changes shared release verification behavior. The safer path is to isolate the wrapper fix or get release-path review and proof for the extra changes.
