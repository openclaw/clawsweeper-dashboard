## What This Changes

The PR adds a 5-second `SIGKILL` timeout to `scripts/crabbox-wrapper.mjs`'s `checkedOutput` helper and maps timed-out Crabbox probes to status 124.

## Best Possible Solution

Merge the scoped timeout once maintainers are comfortable with the 5-second automation bound and the wrapper preflights remain green on the exact head.

## Reproduction Assessment

Yes. Current main source uses unbounded `spawnSync` for the selected Crabbox `--version` and `run --help` checks, so a non-returning selected binary can block the wrapper; the PR body supplies after-fix Windows proof for that path.

## Solution Assessment

Yes. Bounding the existing wrapper helper is a narrow maintainable fix for hung preflights; the only open solution-fit question is whether the same 5-second bound should also apply to broker config detection.
