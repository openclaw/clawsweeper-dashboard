## What This Changes

The PR adds a 5-second SIGKILL timeout to Crabbox wrapper sanity probes and also changes release/postpublish verification timeouts and optional import allowlists.

## Best Possible Solution

Keep the Crabbox timeout patch narrow, rebase it onto current main, drop or split the release-verifier changes, and land only after redacted real wrapper output proves the timeout path.

## Reproduction Assessment

Yes, from source inspection: replacing the selected Crabbox binary with one that never exits would hang current main inside checkedOutput before the wrapper reaches the requested command. I did not run that fake-binary repro in this read-only review.

## Solution Assessment

No, not as submitted: the wrapper timeout is a narrow maintainable fix, but the branch also changes shared release verification behavior. The safer solution is a focused wrapper patch plus separate release-verifier review if those changes are still needed.
