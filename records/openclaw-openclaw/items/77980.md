## Summary

Keep open: current main still has source-reproducible pre-auth bootstrap-token and device-signature DoS gaps, and the umbrella is paired with open discovery/fix work in #76322, #77527, and #77492 rather than a merged implementation.

## What This Changes

Request hardening for the gateway bootstrap-token DoS family by adding rate limiting, lockout, and alerting around failed bootstrap-token and device-signature verifies.

## Best Possible Solution

Land or replace narrow security fixes for bootstrap-token and device-signature rate limiting, then define a shared failed-auth storm alerting policy before closing the umbrella.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a WS connect with a device identity and bogus bootstrap token reaches `verifyDeviceBootstrapToken` before any bootstrap-token limiter, and the linked PRs document real PoC runs. I did not execute the live flood tests in this read-only review.

## Solution Assessment

Unclear as a complete solution: #77527 and #77492 appear to be narrow per-surface rate-limit fixes, but this umbrella also asks for lockout and failed-verify storm alerting that need maintainer security policy before closure.
