## Summary

Keep open: current main still leaves the bootstrap-token mutex path and device-signature verification path without the requested pre-auth limiter scopes, and this umbrella is already paired with open canonical fix PRs plus an unresolved alerting/lockout policy decision.

## What This Changes

Track gateway bootstrap-token and device-signature pre-auth DoS protections, repeated-bad-token lockout, and failed-verify storm alerting.

## Best Possible Solution

Land or replace the narrow bootstrap-token and device-signature rate-limit fixes, then define the shared failed-auth lockout and storm-alerting policy before closing the umbrella.

## Reproduction Assessment

Yes. Source inspection on current main shows a signed device handshake with a bogus bootstrap token reaches the mutex-backed bootstrap verifier before any bootstrap-token limiter, and device-signature verification still runs before the auth decision path.

## Solution Assessment

Unclear as a complete solution. The open per-surface PRs are the right narrow direction, but the umbrella also asks for lockout and failed-verify storm alerting policy that should be settled by the gateway/security owners.
