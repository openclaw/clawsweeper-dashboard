## Summary

Keep open: current main still lacks dedicated bootstrap-token and device-signature limiter scopes before expensive pre-auth work, the issue is paired with open fix PRs, and lockout/failed-verify alerting policy remains unresolved.

## What This Changes

Request to harden gateway bootstrap auth with pre-auth rate limiting, lockout, and alerting for failed bootstrap-token and device-signature verifies.

## Best Possible Solution

Land or replace the narrow bootstrap-token and device-signature rate-limit fixes, then settle the shared lockout and failed-verify storm-alerting policy before closing the umbrella.

## Reproduction Assessment

Yes. Source inspection on current main shows a signed device handshake with a bogus bootstrap token reaches the mutex-backed bootstrap verifier before any bootstrap-token limiter, and device-signature verification runs before the auth decision path.

## Solution Assessment

Unclear as a complete solution. The open per-surface limiter PRs are the right narrow direction, but the umbrella also asks for lockout and failed-verify storm alerting policy that should be settled by gateway/security owners.
