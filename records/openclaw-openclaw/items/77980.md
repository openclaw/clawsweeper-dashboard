## Summary

## What This Changes

The issue asks OpenClaw to harden the gateway bootstrap-token DoS family with pre-auth rate limiting, lockout, and alerting for repeated failed verifies.

## Best Possible Solution

Land or replace the device-signature limiter, then either implement a maintainer-approved failed-verify storm alerting policy or split that policy into a canonical follow-up before closing this umbrella.

## Reproduction Assessment

Yes for the remaining device-signature limiter: current main reaches device-id derivation and v3/v2 Ed25519 signature verification without a device-signature-specific limiter. The alerting portion is not fully reproducible until maintainers define the storm threshold and delivery policy.

## Solution Assessment

Unclear for the full umbrella. The shipped bootstrap-token limiter is the accepted path for that sub-surface, but the remaining solution depends on review of the open device-signature PR and a deliberate failed-verify storm alerting decision.
