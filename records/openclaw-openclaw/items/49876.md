## Summary

Keep open. Current main and the latest release have warning and diagnostic safeguards, but they still do not provide the requested fail-closed or ledger-backed delivery contract for cron output after ordinary tool failures; later discussion also shows the failure class is broader than the original Gemini cron examples.

## What This Changes

The issue requests fail-closed delivery for isolated cron output when tools fail or task completion cannot be verified, including suppression of placeholder or fabricated model text.

## Best Possible Solution

Define a maintainer-approved unattended-output contract for cron that uses structured run/tool metadata for delivery decisions, keeps generic claim-verification extensible through plugins where appropriate, documents default versus strict behavior, and proves both compatibility and strict-mode failure handling.

## Reproduction Assessment

No high-confidence live current-main cron reproduction was run. Source-level reproduction is clear: current helpers and tests preserve or recover final assistant text after tool warnings, matching the delivery gap reported here.

## Solution Assessment

Unclear. The requested phrase filter is not enough by itself; the maintainable path is a structured run/tool status policy with explicit cron defaults, optional strictness, and a decision about what remains plugin-owned claim verification.
