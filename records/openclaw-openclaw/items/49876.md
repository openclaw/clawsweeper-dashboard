## Summary

Keep open: current main has partial cron and embedded-runner hardening, but it still lacks the requested fail-closed or ledger-backed delivery policy for unsupported final assistant output after tool failures.

## What This Changes

The issue asks for cron or agent delivery to fail closed instead of sending fabricated assistant output when required tool work fails or cannot be verified.

## Best Possible Solution

Define a maintainer-approved core finalization policy that uses structured tool-result state to block, replace, or mark unsupported cron output without relying on model self-suppression or brittle text markers.

## Reproduction Assessment

Yes, source-reproducible: current main still has paths where final assistant text can win after selected non-fatal tool warnings or recoverable/non-mutating tool errors. The exact model/provider incidents were not live-reproduced in this read-only review.

## Solution Assessment

No complete solution exists on current main. The best fix needs maintainer policy for cron finalization, unsupported success claims, compatibility, and native tool-failure recovery rather than only adding marker-string suppression.
