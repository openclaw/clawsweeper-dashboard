## Summary

Keep open. Current main has warning and fatal-denial safeguards, but it still preserves or recovers final assistant text after ordinary tool warnings, so the requested unattended fail-closed or ledger-backed delivery contract is not implemented.

## What This Changes

The issue asks for isolated cron output to fail closed or suppress delivery when tool calls fail, required data is missing, or the model produces placeholder or unsupported success text.

## Best Possible Solution

Define a maintainer-approved unattended-output contract that uses structured tool/run metadata for cron delivery, documents default versus strict behavior, and handles broader claim verification without relying on phrase filters alone.

## Reproduction Assessment

No high-confidence live current-main repro was run. Source inspection and current tests do show a high-confidence source path where final assistant text remains deliverable or recoverable after ordinary tool-warning payloads when the current fatal checks do not match.

## Solution Assessment

Unclear. The requested hard-coded phrase filter is not sufficient by itself; the maintainable direction is a structured run/tool status policy with explicit cron defaults, opt-in strictness, and a decision about which generic claim-verification policies belong in core versus plugins.
