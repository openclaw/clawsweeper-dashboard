## Summary

Keep open. Current main and the latest release still allow success-looking assistant text to remain deliverable after some tool failures, and the discussion adds broader non-cron evidence that this needs a maintainer-approved security and delivery policy rather than cleanup closure.

## What This Changes

The issue asks for cron and related agent outputs to fail closed or stay silent instead of delivering unsupported assistant text after failed or missing tool calls.

## Best Possible Solution

Define a maintainer-approved unattended-output contract that uses structured tool and run metadata for cron delivery, handles broader success-claim verification without phrase filters alone, and documents default versus strict behavior.

## Reproduction Assessment

Yes at source level, but no live current-main repro was run. Current source and tests show success-looking final assistant text can remain deliverable after tool-error payloads when current fatal checks do not match.

## Solution Assessment

Unclear. The requested hard-coded phrase filter is not sufficient by itself; the safer direction is a structured run/tool status policy with explicit cron defaults, opt-in strictness, and a maintainer decision on what belongs in core versus plugins.
