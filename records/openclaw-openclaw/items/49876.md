## Summary

Keep open: current main has partial cron/tool-warning hardening and failure alerts, but no general fail-closed or ledger-backed finalization policy for unsupported cron/session output after failed or missing tool work.

## What This Changes

The issue requests fail-closed or clean-error behavior for cron/session outputs when tool work fails, is missing, or cannot substantiate the final assistant text.

## Best Possible Solution

Define a maintainer-approved core finalization policy for unattended and tool-backed turns that uses structured tool-result/run state to suppress, replace, or mark unsupported output without relying only on model self-suppression or brittle text markers.

## Reproduction Assessment

Yes at source level, but not from a live run in this read-only review. Current main still has paths where final assistant text can be delivered after selected non-fatal tool warnings or recoverable failures.

## Solution Assessment

No complete solution exists on current main. The issue's marker-filter idea is too narrow by itself; the safer direction is a structured finalization or ledger-backed policy that covers failed tools, missing tool attempts, and unsupported success claims.
