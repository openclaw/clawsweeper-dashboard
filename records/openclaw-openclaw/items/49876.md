## Summary

Keep open: current main has targeted cron/tool-error hardening, but it still lacks a general fail-closed or ledger-backed finalization policy for cron/session output that is unsupported by failed or missing tool work.

## What This Changes

The issue asks OpenClaw to prevent isolated cron sessions from delivering fabricated or draft-looking output after tool failures, missing data, or incomplete work.

## Best Possible Solution

Define a maintainer-approved core finalization policy for unattended and tool-backed turns that uses structured run/tool state to suppress, replace, or clearly mark unsupported output without relying only on model self-suppression or brittle marker text.

## Reproduction Assessment

Yes at source level, but not from a live run in this read-only review. Current main still has paths where selected failed-tool warnings can be treated as recovered by final assistant text and then delivered.

## Solution Assessment

No complete solution exists on current main. The issue's marker-filter proposal is too narrow by itself; the maintainable direction is a structured finalization or ledger-backed policy that covers failed tools, missing tool attempts, and unsupported success claims.
