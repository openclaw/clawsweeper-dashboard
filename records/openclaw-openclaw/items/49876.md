## Summary

Keep open: current main has partial cron and embedded-runner mitigations for structured errors, but it still does not provide the requested fail-closed or ledger-backed policy for unsupported final assistant output after tool failures.

## What This Changes

The issue asks OpenClaw to prevent cron or unattended agent sessions from delivering fabricated or unsupported final output after tool failures.

## Best Possible Solution

Define a maintainer-approved core finalization policy that uses structured tool result state to block, replace, or mark unsupported cron output without relying on model self-suppression or brittle text markers.

## Reproduction Assessment

Yes, source-reproducible: current main still has code paths where final assistant text can win after non-fatal tool warnings or non-mutating tool errors with a visible reply. The exact model/provider incidents were not live-reproduced here.

## Solution Assessment

No complete solution exists on current main. The best fix needs maintainer policy for cron finalization, unsupported success claims, and compatibility rather than only adding marker-string suppression.
