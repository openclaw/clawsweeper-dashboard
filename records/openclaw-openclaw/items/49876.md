## Summary

Keep open: current main has partial cron hardening for fatal structured errors, denial signals, and no-reply cases, but unattended cron output can still be delivered from later assistant text without a structural claim-vs-tool-ledger or fail-closed verifier, and even the partial hardening is newer than the latest release checked.

## What This Changes

Request fail-closed cron delivery so unattended runs with failed or unverifiable tool work do not deliver fabricated, placeholder, or unsupported success output.

## Best Possible Solution

Define a core-owned unattended finalization policy that suppresses or cleanly fails unverified cron output after tool errors, using structured tool/failure ledger evidence and an upgrade-safe plan across unattended and interactive paths.

## Reproduction Assessment

No live current-main repro was run in this read-only pass; source review gives a high-confidence plausible path because cron finalization still treats later final assistant text as recovery proof and tests expect some recovered warning output to remain ok.

## Solution Assessment

Current main is not a complete solution: structured-denial, fatal-error, and NO_REPLY handling are useful but too narrow, and regex-only suppression would still miss clean unsupported success claims without a structural ledger policy.
