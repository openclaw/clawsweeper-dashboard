## Summary

Keep open: current main has partial cron and embedded-runner hardening for fatal denial signals, structured error payloads, and tool-error warnings, but it still does not provide the requested fail-closed or claim-vs-tool-ledger boundary before cron output is delivered.

## What This Changes

The issue requests fail-closed delivery for unattended cron sessions so fabricated, placeholder, or unsupported completion output is not delivered after tool failures or incomplete work.

## Best Possible Solution

Define a core-owned finalization policy for unattended runs that tracks unresolved tool failures and unsupported success claims before delivery, with explicit maintainer-approved defaults for warn, fail-closed, and recovery behavior.

## Reproduction Assessment

Unclear as a live repro: the issue has multiple concrete production and deterministic reports, and source inspection shows current main still lacks a general fail-closed or claim-ledger gate. I did not run a live cron/model reproduction in this read-only pass.

## Solution Assessment

Unclear until maintainers choose the product contract. Fail-closed or claim-ledger gating is directionally the right layer, but the default strictness and config surface are compatibility-sensitive.
