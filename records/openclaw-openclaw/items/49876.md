## Summary

Issue #49876 should stay open because current main still allows cron runs to deliver final assistant text after some tool-error payloads and the broader fail-cleanly/tool-ledger enforcement behavior remains an unresolved core agent-loop product and safety decision, not a stale or plugin-only request.

## What This Changes

The issue requests a core fail-cleanly behavior for cron and chat-surface agent sessions so unsupported success claims or hallucinated outputs are not delivered after tool failures or missing tool calls.

## Best Possible Solution

Keep the issue open for a maintainer-owned core policy design that distinguishes recoverable tool warnings from unsupported success claims and proves cron, chat, and channel delivery behavior with upgrade-safe tests.
