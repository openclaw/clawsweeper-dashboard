## Summary

Keep open: current main and the latest release have partial cron/tool-warning hardening, but not a universal fail-closed or ledger-backed policy that prevents final assistant text from being delivered after failed or unsupported tool work.

## What This Changes

The issue requests fail-closed or ledger-backed delivery behavior so cron and related agent sessions do not send fabricated, placeholder, or unsupported success output after tool-call failures.

## Best Possible Solution

Define a maintainer-approved core finalization policy that uses structured tool-result/run state to block, replace, or mark unsupported cron output without relying only on model self-suppression or brittle text markers.

## Reproduction Assessment

Yes, at source level: current main still has paths where final assistant text can win after selected non-fatal tool warnings or recoverable/non-mutating tool errors. I did not live-run the reporter's Gemini or local-model setups in this read-only review.

## Solution Assessment

No complete solution exists on current main. The best fix needs maintainer policy for cron finalization, unsupported success claims, compatibility, and native tool-failure recovery rather than only adding marker-string suppression.
