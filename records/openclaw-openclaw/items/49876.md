## Summary

Keep open: current main has partial warning and narrow fatal-denial handling, but no general fail-closed or claim-vs-tool-ledger gate before cron or final assistant output delivery.

## What This Changes

The issue asks OpenClaw to prevent cron and unattended agent runs from delivering fabricated, placeholder, or unsupported success output after tool failures.

## Best Possible Solution

Define a core-owned finalization and delivery policy that fails closed or marks failure for unresolved tool-error and unsupported-success states before unattended output is delivered.

## Solution Assessment

No: current main's warning and narrow fatal-denial paths are useful hardening, but they are not the requested fail-closed or claim-ledger boundary.
