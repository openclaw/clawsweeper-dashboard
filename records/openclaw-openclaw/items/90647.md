## Summary

Keep open for maintainer review: the patch addresses the macOS `SKILL.md` file-descriptor leak with real macOS `lsof` proof and I did not find a blocking code defect, but the new Windows native watcher path remains unproven outside mocks.

## What This Changes

The PR replaces per-file chokidar watching for skills roots on macOS/Windows with native recursive `fs.watch` plus parent reattach/fallback logic, and adds watcher lifecycle regression tests.

## Best Possible Solution

Land the native recursive watcher after maintainers accept the macOS proof and either accept the Windows risk, request a Windows smoke, or narrow the native path to macOS until Windows is proven.

## Reproduction Assessment

Yes, source inspection plus the PR proof gives a high-confidence reproduction path: current main watches `SKILL.md` through chokidar, and the PR body supplies real macOS `lsof` output showing per-skill descriptors before and none after. I did not independently run macOS or Windows locally.

## Solution Assessment

Yes, this appears to be the best fix shape for the reported macOS leak: directory-only chokidar would lose in-place edit refresh, polling adds steady stat cost, and the native recursive approach matches the shipped memory watcher pattern. The remaining solution-fit question is whether Windows needs live proof before merge.
