## Summary

Keep open for maintainer review: the patch is a plausible and likely correct fix for the skills watcher FD leak, but the external real-behavior proof does not yet exercise the macOS/Windows native watcher path that this PR changes.

## What This Changes

The PR changes the skills runtime watcher to use native recursive `fs.watch` for existing skill roots on macOS/Windows, keep chokidar elsewhere and on fallback, and adds focused watcher lifecycle tests.

## Best Possible Solution

Land this native-recursive watcher approach after redacted macOS descriptor proof confirms `SKILL.md` FDs stay flat and skill edits still refresh; keep Linux on chokidar and avoid new config knobs.

## Reproduction Assessment

Yes, source-backed: https://github.com/openclaw/openclaw/issues/90578 gives deterministic macOS `lsof` steps, and current main still admits `SKILL.md` into the chokidar watcher path. I did not run the macOS reproduction in this read-only review.

## Solution Assessment

Yes, likely: the native-recursive macOS/Windows approach matches the sibling memory watcher and preserves `SKILL.md` edit detection better than the directory-only alternative at https://github.com/openclaw/openclaw/pull/90614. The missing piece is target-platform proof, not a different code layer.
