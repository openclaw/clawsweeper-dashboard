Keep open. Current main still exposes the exact reported pruned-image marker as replay text and lets WebChat send that literal as ordinary text into the agent, so the marker-collision path has not been fixed.

## What This Changes

Treat the pruned image marker literal as ordinary user text and prevent OpenClaw's internal replay marker from causing recursive Kimi output.

## Best Possible Solution

Replace or encode the internal pruned-image replay marker so it cannot collide with user-authored text, while preserving stale-media reference scrubbing and current-turn image hydration.

## Reproduction Assessment

No, not end-to-end. Source inspection gives a high-confidence path showing the literal marker is accepted by WebChat and still emitted into provider replay, but the Kimi loop needs a live Kimi run or a focused mocked regression.

## Solution Assessment

Yes. The narrowest maintainable solution is in the shared replay marker representation or sanitization boundary; a Kimi-only output filter would leave the ambiguous model-visible marker in shared replay context.
