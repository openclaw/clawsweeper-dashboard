The issue asks OpenClaw to prevent Kimi Coding from recursively responding with the internal pruned-image marker when a user sends that marker as plain text.

## Best Possible Solution

Keep the issue open and fix the replay/sanitization boundary so OpenClaw does not send an internal pruned-image marker as an ambiguous ordinary text token. Add a focused regression for user-authored literal marker text while preserving stale-media pruning and prompt-cache behavior.
