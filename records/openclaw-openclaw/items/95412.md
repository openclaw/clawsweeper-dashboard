## What This Changes

## Best Possible Solution

Land the focused RequestClient hardening after maintainer acceptance or tuning of the 8 MiB ceiling, and track any broader Discord success-body policy separately.

## Solution Assessment

Yes for the scoped RequestClient path. The PR uses the existing plugin SDK response-limit helper and preserves parsing/status behavior; the maintainer choice is the cap value and whether broader Discord API hardening should be handled separately.
