## Summary

Keep open: the PR is a focused attempt at the stale-gateway update problem, but it is not merge-ready because the warning is added to generic update output before current main's managed restart and verification path runs, and the contributor has not supplied real update-flow proof.

## What This Changes

The PR adds a terminal warning in `printResult()` after successful version-changing updates telling users to run `openclaw gateway restart`.

## Best Possible Solution

Keep restart guidance tied to the actual update outcome: warn only when restart was skipped, failed, unmanaged, or still unhealthy after verification, while leaving the current managed restart and verification path authoritative.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path for the PR defect: a successful version-changing update calls `printResult()` before `maybeRestartService()`, so the new warning can appear before the command restarts and verifies the Gateway. I did not run a live package-update flow in this read-only review.

## Solution Assessment

No, this is only a plausible mitigation, not the best fix. The warning should be emitted from restart outcome handling, or maintainers should choose a broader canonical fix for the stale-gateway cluster before closing the linked issue.
