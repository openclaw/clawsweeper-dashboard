## Summary

Keep open. Current main and the latest release still send bare `/new` and `/reset` startup prompts without the requested anti-fabrication/tool-verification guardrail, and open PR #75127 already targets the issue with closing syntax, so this should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to prevent bare session reset startup prompts from causing models to present invented calendar, email, or JSON data as user-provided facts.

## Best Possible Solution

Land PR #75127, or an equivalent narrow reset-prompt and test change, so every bare reset prompt variant explicitly forbids presenting invented structured data as real and requires tool verification before acting on calendar, email, task, memory, or JSON claims.

## Reproduction Assessment

Unclear. There is a high-confidence static path showing bare `/new` and `/reset` send the current reset prompt, but the reported fabricated first response depends on the Xiaomi model/provider and lacks transcript evidence.

## Solution Assessment

Yes. A narrow reset-prompt contract guardrail with direct prompt regression tests is the smallest maintainable repo-level fix supported by the evidence; broader runtime permission or routing changes are not justified by this report alone.
