## Summary

Keep open. Current main still lacks an explicit anti-fabrication and tool-verification guardrail in the bare reset startup prompts, and the linked open PR #75127 uses closing syntax for this issue, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Add a reset-startup guardrail so bare `/new` and `/reset` prompts do not let models present invented calendar, email, or JSON data as user-provided context.

## Best Possible Solution

Review and land PR #75127, or an equivalent narrow prompt-and-test change, so every bare reset prompt variant forbids presenting invented data as real and requires verification before action.

## Reproduction Assessment

Unclear. Static inspection gives a high-confidence path showing the reset prompts sent by both chat and Gateway lack the guardrail, but the reported model output itself is provider/model-dependent and no live transcript was supplied.

## Solution Assessment

Yes. A shared reset prompt guardrail with direct tests for normal, full-bootstrap, and limited-bootstrap variants is the narrowest maintainable repository-level fix supported by the evidence.
