## Summary

Keep open. Current main still sends bare `/new` and `/reset` startup prompts without the requested anti-fabrication or tool-verification guardrail, and the issue is paired with open PR #75127 using closing syntax, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to prevent bare session reset prompts from letting models fabricate calendar/email/JSON-style data and act on it as real user-provided context.

## Best Possible Solution

Review and land PR #75127, or an equivalent narrow prompt-and-test change, so every bare reset prompt variant explicitly forbids presenting invented data as real and requires tool verification before acting on external-looking data claims.

## Reproduction Assessment

Unclear. Static inspection gives a high-confidence way to reproduce the missing prompt guardrail, but the exact hallucinated first response depends on the reported model/provider and no live transcript was provided.

## Solution Assessment

Yes. A narrow reset prompt contract plus direct prompt regression tests is the smallest maintainable repository-level fix supported by the evidence; broader runtime permission or routing changes are not justified by this report alone.
