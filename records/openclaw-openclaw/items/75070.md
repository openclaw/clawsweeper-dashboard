## Summary

Keep open. Current main still sends bare `/new` and `/reset` startup prompts without the requested anti-fabrication/tool-verification guardrail, and the issue is already paired with open PR #75127 using closing syntax, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

Request that session reset prompts prevent models from fabricating calendar, email, JSON, or other user data and require tool verification before acting on reset-time claims.

## Best Possible Solution

Review and land PR #75127, or an equivalent narrow prompt-and-test change, so all bare reset prompt variants explicitly forbid presenting invented data as real and require tool verification before acting on calendar, email, task, memory, file, or JSON claims.

## Reproduction Assessment
