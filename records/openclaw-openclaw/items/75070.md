## Summary

Keep open. Current main still sends bare `/new` and `/reset` startup prompts without the requested anti-fabrication/tool-verification guardrail, and PR #75127 is open with closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Prevent session reset startup prompts from letting models present invented calendar, email, JSON, memory, or other structured data as user-provided context, and require tool verification before acting on real data.

## Best Possible Solution

Review and land PR #75127, or an equivalent narrow prompt-and-test change, so all bare reset prompt variants explicitly forbid presenting invented data as real and require tool verification before acting on calendar, email, task, memory, file, or JSON claims.

## Reproduction Assessment

Unclear. Static inspection gives a high-confidence path showing the reset prompts lack the requested guardrail, but the exact fabricated first response depends on the Xiaomi model/provider and no transcript evidence was provided.

## Solution Assessment

Yes. A narrow reset prompt contract guardrail with direct prompt regression tests is the smallest maintainable repository-level fix supported by the evidence; broader runtime permission or routing changes are not justified by this report alone.
