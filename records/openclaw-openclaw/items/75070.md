## Summary

Keep open. Current main still lacks an anti-fabrication guardrail in the shared reset prompt, and open PR #75127 already targets this issue with closing syntax, so the right path is to review and land or replace that PR rather than close this issue early.

## What This Changes

The issue reports that bare session resets can lead the model to present invented calendar/email/JSON-style data as if it were user-provided input.

## Best Possible Solution

Review and land PR #75127, or an equivalent narrow prompt-and-test change, so all bare reset prompt variants forbid presenting invented data as real and require verification before action.

## Reproduction Assessment

Unclear for the exact live model output: the fabricated response depends on the provider/model and no transcript was supplied. Source inspection does show that current reset prompts reach chat and Gateway reset paths without the requested anti-fabrication guardrail.

## Solution Assessment

Yes. A shared reset prompt guardrail with direct assertions for normal, full-bootstrap, and limited-bootstrap variants is the narrowest maintainable fix; runtime permission changes or model-specific routing changes would be broader than the evidence supports.
