## Summary

Keep open. Current main still has reset/startup prompt paths that send a model-facing greeting prompt without the requested anti-fabrication and tool-verification guardrail, and the issue is already paired with open PR #75127 using closing syntax, so cleanup should wait for that PR or an equivalent fix to land.

## What This Changes

The issue requests a reset/startup guardrail so empty or bootstrap reset turns do not cause models to present invented calendar, email, or structured data as real user-provided input.

## Best Possible Solution

Land PR #75127, or an equivalent narrow prompt/test change, so every reset prompt variant clearly tells the model not to present invented data as real and to verify real calendar, email, task, or structured-data claims with tools before acting.

## Reproduction Assessment

Unclear. There is a high-confidence static reproduction for the missing guardrail in the reset prompt paths, but the live fabricated JSON/calendar response depends on the reported model/provider and lacks transcript evidence.

## Solution Assessment

Yes. A narrow reset-prompt contract with direct prompt regression tests is the best repo-level fix supported by the evidence; broader runtime permission, provider, or routing changes are not justified by this report alone.
