## Summary

Keep open. Current main still lacks the requested anti-fabrication/tool-verification wording in the bare session reset prompts, and the issue is already paired with open PR #75127 using closing syntax, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests a reset-session guardrail so empty or bootstrap reset prompts do not lead the model to present invented calendar, email, or JSON data as real.

## Best Possible Solution

Review and land PR #75127, or an equivalent narrow prompt-and-test change, so all bare reset prompt variants explicitly forbid presenting invented data as real and require tool verification before acting on calendar, email, task, memory, file, or JSON claims.

## Reproduction Assessment

## Solution Assessment
