## Summary

Keep open. Current main still sends bare `/new` and `/reset` through reset startup prompts without the requested anti-fabrication or tool-verification guardrail, and open PR #75127 is already linked as the intended fix.

## What This Changes

The issue asks OpenClaw to guard session-reset startup prompts against models presenting invented calendar, email, or JSON data as real user context.

## Best Possible Solution

Review and land PR #75127, or an equivalent narrow prompt-and-test change covering every bare reset prompt variant.

## Reproduction Assessment

Unclear. Static inspection gives a high-confidence path showing reset prompts lack the requested guardrail, but the fabricated first response depends on a specific model/provider and no transcript evidence was provided.

## Solution Assessment

Yes. A shared reset prompt contract with direct prompt regression tests is the narrowest maintainable repository-level fix supported by the evidence.
