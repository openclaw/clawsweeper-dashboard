## Summary

Keep open. Current main still lacks an explicit anti-fabrication/tool-verification guardrail in the shared bare reset prompt variants, and open PR #75127 references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that session reset startup can prompt a model to fabricate calendar/email/JSON-like data as if it were user-provided context.

## Best Possible Solution

Land PR #75127, or an equivalent narrow shared prompt-and-test change, so every bare reset prompt variant forbids presenting invented data as real and requires verification before acting on external data claims.

## Reproduction Assessment

Unclear. Static inspection gives a high-confidence path showing the guardrail is missing on both reset dispatch surfaces, but the reported fabricated output depends on a specific provider/model and no live transcript was included.

## Solution Assessment

Yes. A focused shared reset prompt contract with direct prompt regression tests is the narrowest maintainable repository-level fix supported by the evidence; broader runtime or tool-permission changes are not justified by this report alone.
