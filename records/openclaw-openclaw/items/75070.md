## Summary

Keep open. Current main still sends bare reset startup prompts without an explicit no-fabrication or tool-verification guardrail, and #75127 remains open as the focused implementation candidate.

## What This Changes

The issue asks OpenClaw to prevent bare session resets from causing models to present invented calendar, email, or JSON data as real user-provided context.

## Best Possible Solution

Land #75127, or an equivalent narrow reset prompt and regression-test change, across both auto-reply and Gateway reset dispatch before closing this as implemented.

Is this the best way to solve the issue? Yes: a focused reset prompt contract with direct prompt regression tests is the narrowest maintainable repository-level fix supported by the evidence.
