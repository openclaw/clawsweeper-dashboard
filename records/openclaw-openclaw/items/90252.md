## Summary

Keep open: the repeated-warning bug is real, but this PR's archive-all conflict policy is too broad for persisted upgrade state and still lacks real behavior proof from a patched upgraded install.

## What This Changes

The PR changes the legacy plugin install-index migration to archive `plugins/installs.json` after any conflicting shared SQLite install metadata and updates doctor migration tests to expect a one-time warning.

## Best Possible Solution

Land a narrow state-migration rule that archives only legacy install records proven superseded by canonical SQLite metadata, keeps true conflicts visible, and includes real two-run upgrade proof.

## Reproduction Assessment

Yes, at source level: leave a conflicting legacy `plugins/installs.json` beside a shared SQLite installed-plugin index, then run legacy state migration twice. Current main returns before archiving, so the same legacy file can be read again.

## Solution Assessment

No. The right layer is state migration, but the proposed archive-all branch is broader than the maintainable fix because it retires unresolved conflicts instead of proving SQLite is authoritative first.
