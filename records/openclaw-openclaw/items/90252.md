Keep open: the PR targets a real repeated-warning bug, but the current patch archives every installed-plugin conflict, including unresolved package drift and malformed legacy metadata, which is too broad for an upgrade migration path. The linked issue has also grown beyond plugin install-index warnings, so this PR should not close the canonical issue without a maintainer-approved narrower policy.

## What This Changes

The PR archives the legacy `plugins/installs.json` file on installed-plugin metadata conflicts, changes the warning text, and updates doctor state-migration tests to expect a quiet second run.

## Best Possible Solution

Use a maintainer-approved conflict predicate that archives only when shared SQLite proves it supersedes the legacy install record, keep unresolved conflicts warning-visible, and keep or split the broader linked issue until non-plugin legacy-source warnings are handled.

## Reproduction Assessment

Yes at source level: seed a legacy `plugins/installs.json` beside a conflicting shared SQLite installed-plugin index and run legacy state migrations twice. Current main returns before archiving, so the file remains and the warning repeats; this read-only review did not execute the flow.

## Solution Assessment

No. The PR fixes the repeated warning symptom, but archiving every conflict is broader than the safest migration policy; the best fix preserves unresolved conflicts and archives only when SQLite proves it supersedes the legacy npm artifact.
