## Summary

Keep this PR open, but it is not merge-ready: current main already has narrower delivery-mirror/gateway-injected history handling, and this branch would hide rows that current tests and docs still treat as visible or raw/audit data.

## What This Changes

The PR adds a transcript-artifact predicate for OpenClaw `delivery-mirror` and `gateway-injected` assistant rows, filters those rows from `chat.history` and `sessions.get`, over-reads transcript tails before limiting, and updates gateway tests/docs plus `CHANGELOG.md`.

## Best Possible Solution

Rebase onto current main, narrow the filter to only proven duplicate/internal artifacts, preserve or add an explicit raw/audit history path for stored rows, and keep release-note context in the PR body or squash message instead of `CHANGELOG.md`.

## Reproduction Assessment

Yes. Source and current-main tests show the PR would hide every OpenClaw `delivery-mirror` and `gateway-injected` assistant row, while current main intentionally preserves some of those rows in history surfaces.

## Solution Assessment

No. Consumer-boundary filtering is a plausible layer, but this implementation is too broad unless maintainers explicitly approve a new raw/audit contract for `sessions.get` and injected rows.
