## Summary

Keep this PR open: the parser guard is a focused OpenRouter music-stream bugfix, but it still needs contributor-owned real behavior proof before merge, and maintainers should notice that it changes malformed `data:` frames from failure to skip.

## What This Changes

The PR wraps OpenRouter music-generation SSE `JSON.parse` in a try/catch and adds a regression test where a malformed `data:` frame is skipped before later audio is processed.

## Best Possible Solution

Keep the provider-local parse guard only after redacted runtime proof shows the OpenRouter music malformed-frame path is safe; otherwise return a stable malformed-stream error instead of a raw `SyntaxError`.

## Reproduction Assessment

Yes, source-reproducible: current main calls `JSON.parse(data)` directly on non-`[DONE]` OpenRouter SSE data, so a malformed `data:` line throws. I did not run tests because this review is read-only.

## Solution Assessment

Unclear until proof: catching parse failure at the OpenRouter provider boundary is the right shape and matches generic OpenRouter streaming examples, but this PR has not proven the audio-specific malformed-frame path in a real setup.
