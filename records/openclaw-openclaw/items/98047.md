## Summary

Keep open: the PR targets a real current-main parser crash boundary, but the proposed catch silently skips any malformed OpenRouter `data:` frame and can still return a successful partial track; contributor-owned real behavior proof is also still missing.

## What This Changes

The PR wraps OpenRouter music SSE `JSON.parse` in try/catch and adds a regression test that skips one malformed frame before processing later valid audio.

## Best Possible Solution

Catch malformed OpenRouter music SSE JSON at this boundary, but fail with a stable malformed-stream error unless provider-specific evidence proves the exact frame is ignorable.

## Reproduction Assessment

Yes, source-reproducible: current main calls `JSON.parse(data)` directly on non-`[DONE]` OpenRouter SSE data, so a malformed `data:` frame throws. I did not run tests because this review is read-only.

## Solution Assessment

No. Catching the parser failure locally is the right boundary, but returning `false` is too broad because it can convert malformed audio stream data into a successful partial output.
