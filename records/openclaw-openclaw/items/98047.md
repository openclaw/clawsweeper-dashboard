## Summary

Keep open: the PR fixes the right local parser boundary, but the current skip-on-parse-failure behavior can turn malformed OpenRouter audio stream data into a successful partial track, and the contributor still has not provided real behavior proof beyond unit output.

## What This Changes

This PR wraps OpenRouter music SSE `JSON.parse` in try/catch to skip malformed `data:` frames and adds a unit test that later valid audio still succeeds.

## Best Possible Solution

Catch malformed OpenRouter music SSE JSON at this boundary, but fail with a stable provider-stream error unless OpenRouter-specific evidence proves the exact frame is ignorable; then update the regression test and add real behavior proof.

## Reproduction Assessment

Yes, at source level: feed a malformed `data:` payload followed by valid audio and `[DONE]`; the PR branch returns a non-empty track because the catch returns `false` and `generateMusic` only rejects an empty final buffer. I did not run tests because this review is read-only.

## Solution Assessment

No. Catching the parser failure locally is the right boundary, but swallowing every unparsable `data:` payload is too broad for audio output; fail closed unless a documented ignorable frame is proven.
