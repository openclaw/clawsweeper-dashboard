## Summary

Keep open for human revision: current main still has the Control UI `/model status` and `/model list` bug, but this PR collapses documented `list` and `status` behavior into the bare `/model` summary path, adds an undocumented `info` alias, and lacks after-fix real behavior proof.

## What This Changes

The PR adds a Control UI reserved-subcommand set so `/model status`, `/model list`, and `/model info` show the existing bare model summary instead of calling `sessions.patch`, plus executor tests for those tokens and a normal model-selection case.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` to the documented picker/list behavior, routes `/model status` to detailed model/auth/endpoint output, preserves normal model selection, and leaves `info` unchanged unless the shared contract and docs are updated.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `/model status` parses to `args: "status"`, Control UI dispatch passes it unchanged, and current main calls `sessions.patch` for every non-empty `/model` argument.

## Solution Assessment

No. Avoiding the bad `sessions.patch` call is necessary, but this patch should preserve documented `list` and `status` handlers and should not add `info` unless the shared contract and docs are updated.
