## Summary

Keep open: current main still has the source-reproducible Control UI `/model status` and `/model list` bug, but this PR is not merge-ready because it returns the wrong documented responses, adds an undocumented `info` alias, lacks real behavior proof, and is currently conflicting.

## What This Changes

The PR reserves `status`, `list`, and `info` in the Control UI `/model` executor so they show the bare model-info response, and adds executor tests including a normal model-selection regression case.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` and `/model status` to the documented informational behavior, preserves normal model selection, and adds focused regression coverage plus redacted real Control UI proof.

## Reproduction Assessment

Yes, source-backed with high confidence: `/model status` parses to `args: "status"`, Control UI passes that arg unchanged, and current main sends every non-empty `/model` arg to `sessions.patch`. I did not run a live Control UI session in this read-only review.

## Solution Assessment

No. Avoiding the bad `sessions.patch` call is necessary, but this PR collapses documented `list` and `status` responses into the bare summary branch and adds an undocumented `info` mode.
