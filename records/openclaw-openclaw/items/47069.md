## Summary

Keep open: the current-main bug path is still present, but this PR is not merge-ready because it collapses documented `/model list` and `/model status` behavior into the bare `/model` summary, adds an undocumented `info` alias, and still lacks after-fix real Control UI proof.

## What This Changes

The PR adds a Control UI `/model` reserved-subcommand set for `status`, `list`, and `info`, routes those tokens through the bare model-info branch, and adds slash-command executor tests including a normal model-name regression.

## Best Possible Solution

Align the Control UI local `/model` handling with the documented shared contract: keep normal model selection, route `list` to the model list/picker path, route `status` to detailed provider/auth/endpoint output, and leave `info` unreserved unless the contract and docs are deliberately extended.

## Reproduction Assessment

Yes. Source inspection on current main shows `/model status` parses to `args: "status"`, dispatch passes it unchanged, and `executeModel` calls `sessions.patch` for every non-empty argument.

## Solution Assessment

No. The PR prevents the bad patch call, but it returns the wrong documented responses for `list` and `status` and adds an undocumented `info` alias without a shared contract/docs decision.
