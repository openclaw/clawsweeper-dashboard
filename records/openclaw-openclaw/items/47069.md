## Summary

Keep open. The underlying Control UI bug is still present on current main, but this PR should not merge as-is because it maps documented `list` and `status` modes to the bare model summary and also reserves undocumented `info`; related #46894 and #46895 remain useful coordination points.

## What This Changes

The PR adds a Control UI `/model` reserved-subcommand guard for `status`, `list`, and `info`, plus tests for those subcommands and normal model selection.

## Best Possible Solution

Implement explicit Control UI handling that matches the shared command contract: bare `/model` keeps the current summary/picker behavior, `/model list` follows the model browser/list path, `/model status` shows detailed model/auth/endpoint diagnostics, and `info` remains a normal model selection unless the shared contract and docs reserve it.

## Reproduction Assessment

Yes. Source inspection on current main shows `/model status` and `/model list` parse as local command `model` with non-empty args, and `executeModel` sends those args to `sessions.patch`.

## Solution Assessment

No. The PR avoids the bad `sessions.patch` call, but it drops documented `list` and `status` semantics and reserves `info` without a shared command contract.
