## Best Possible Solution

Land one rebased canonical Control UI fix that routes `/model list` and `/model status` to documented list/status flows, preserves normal model selection, removes or documents `info`, and includes focused tests plus redacted UI proof.

## Reproduction Assessment

Yes. Source inspection shows `/model status` or `/model list` is parsed and dispatched as a non-empty arg, and current `executeModel` sends any non-empty arg to `sessions.patch`.

## Solution Assessment

No. Avoiding the bad `sessions.patch` call is necessary, but the maintainable fix is to route `list` and `status` to the documented shared semantics and remove or explicitly document `info`.
