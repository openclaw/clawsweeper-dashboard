## Summary

Keep open: current main still has the Control UI `/model status` and `/model list` routing bug, but this branch is not ready because it collapses documented list/status behavior into the bare summary and reserves undocumented `info`.

## What This Changes

The PR changes the Control UI slash-command executor to treat `/model status`, `/model list`, and `/model info` as model-info queries and adds regression tests for those aliases plus normal model selection.

## Best Possible Solution

Align the Control UI local `/model` handling with the shared directive/docs contract: keep normal model selection, route `/model list` to the model list/picker behavior, route `/model status` to detailed status/auth/endpoint output, and leave `info` unreserved unless the command contract is deliberately extended.

## Reproduction Assessment

Yes, by source inspection. On current main, `/model status` and `/model list` parse as non-empty args and `executeModel` sends those args to `sessions.patch`; I did not run UI/tests because this review is read-only.

## Solution Assessment

No. The PR prevents the bad patch call, but it does so by dropping documented `list` and `status` semantics and by adding an undocumented `info` reservation that the shared command contract does not define.
