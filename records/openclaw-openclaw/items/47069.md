## Summary

Keep open: the underlying Control UI bug is still present on current main, but this PR should not merge as-is because it collapses documented `/model list` and `/model status` behavior into the bare summary path, reserves undocumented `info`, and omits the required changelog entry.

## What This Changes

The PR adds a Control UI reserved-subcommand set for `/model status`, `/model list`, and `/model info`, plus tests that those arguments avoid `sessions.patch` while a normal model name still patches.

## Best Possible Solution

Align Control UI local `/model` handling with the shared directive/docs contract: preserve normal model selection, make `list` use the model-list/picker path, make `status` show detailed provider/auth/endpoint status, and leave `info` on the model-selection path unless the public command contract is deliberately extended.

## Reproduction Assessment

Yes by source inspection. Current main parses `/model status` and `/model list` as non-empty args, and `executeModel` sends those args to `sessions.patch` instead of a special command path.

## Solution Assessment

No. The PR prevents the bad patch call, but it does so by losing the documented `list` and `status` semantics and by adding an undocumented `info` reservation.
