## Summary

Keep open: current main still routes non-empty `/model` arguments to `sessions.patch`, and this PR avoids that call by collapsing documented `/model list` and `/model status` behavior into the bare summary path while also reserving undocumented `info`.

## What This Changes

The PR adds a Control UI `/model` reserved-subcommand set for `status`, `list`, and `info`, plus tests so those tokens show the existing bare model summary instead of calling `sessions.patch`.

## Best Possible Solution

Align the Control UI local `/model` handling with the shared directive/docs contract: preserve normal model selection, make `/model list` use the model-list/picker path, make `/model status` show detailed auth/endpoint status, and leave `info` unreserved unless the command contract is deliberately extended.

## Reproduction Assessment

Yes, by source inspection. Current main parses `status` and `list` as `/model` args and `executeModel` sends any non-empty arg to `sessions.patch`, matching the linked issue's gateway error logs.

## Solution Assessment

No. The PR stops the bad patch call, but it drops documented `list` and `status` semantics and adds `info` outside the shared command contract; the safer repair is to match the existing directive behavior.
