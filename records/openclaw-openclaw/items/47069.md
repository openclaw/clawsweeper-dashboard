## Summary

Keep open. The linked bug is still source-reproducible on current main, but this PR does not preserve the documented `/model list` and `/model status` behaviors and still lacks after-fix real Control UI proof.

## What This Changes

This PR changes the Control UI `/model` slash-command executor so `status`, `list`, and `info` use the bare model-info branch, with tests for those tokens and normal model selection.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` and `/model status` through the documented responses, preserves normal model selection, and only adds `info` if the shared contract and docs are extended.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `/model status` parses to `args: "status"`, Control UI dispatch passes it unchanged, and current main calls `sessions.patch` for every non-empty `/model` argument.

## Solution Assessment

No. Avoiding the bad `sessions.patch` call is necessary, but this PR collapses documented `list` and `status` modes into the bare summary branch and adds an undocumented `info` alias.
