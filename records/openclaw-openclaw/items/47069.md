## Summary

Keep open: the PR targets a real Control UI regression, but the current diff collapses documented `/model list` and `/model status` behavior into the bare summary path, reserves an undocumented `info` alias, lacks after-fix real behavior proof, and overlaps another open fix attempt.

## What This Changes

The PR adds a reserved `status`/`list`/`info` set in Control UI `/model` handling and tests those tokens plus a normal model name against `sessions.patch` behavior.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` to the documented model-list/picker behavior, routes `/model status` to detailed status/auth/endpoint output, preserves normal model selection, and leaves `info` unreserved unless the shared contract and docs are deliberately extended.

## Reproduction Assessment

Yes. Source inspection on current main shows `/model status` parses to `args: "status"`, is dispatched unchanged, and then reaches the `sessions.patch` branch because `executeModel` only treats empty args as informational.

## Solution Assessment

No. The PR prevents the bad patch call, but it does so by flattening documented `list` and `status` outputs into the bare `/model` summary and by adding an undocumented `info` alias without contract/docs agreement.
