## Summary

Keep open. The underlying Control UI bug is still present on current main, but this PR is not the right merge as-is because it collapses documented `/model list` and `/model status` modes and reserves undocumented `/model info`; it also lacks external real-behavior proof.

## What This Changes

The PR changes the Control UI `/model` slash-command executor to treat `status`, `list`, and `info` as informational tokens and adds executor tests for those paths plus normal model setting.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` to the compact picker/list behavior, routes `/model status` to detailed status/auth/endpoint output, preserves normal model selection, and leaves `info` unchanged unless the shared contract and docs are updated.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `/model status` parses to `args: "status"`, Control UI dispatch passes it unchanged, and current main sends every non-empty `/model` argument to `sessions.patch`.

## Solution Assessment

No. Avoiding the bad `sessions.patch` call is necessary, but this patch should preserve the documented `list` and `status` handlers and should not add `info` without shared contract/docs support.
