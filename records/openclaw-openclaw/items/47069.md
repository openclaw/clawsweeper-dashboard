## Summary

Keep open. The underlying Control UI bug is still present on current main, but this PR should not merge as-is because it routes documented `list` and `status` modes to the bare model summary and reserves undocumented `info`; related #46894 and #46895 remain open coordination points.

## What This Changes

The PR adds a Control UI guard that routes `/model status`, `/model list`, and `/model info` to the existing bare model-info branch, plus node tests covering those subcommands and normal model selection.

## Best Possible Solution

Implement Control UI `/model` subcommand handling that matches the shared command contract: bare `/model` keeps the current summary/picker behavior, `/model list` follows the model browser/list path, `/model status` shows detailed model/auth/endpoint diagnostics, and `info` remains a normal model selection unless the shared contract and docs reserve it. Add focused Control UI tests plus a changelog entry for the user-facing fix.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: `/model status` parses as local command `model` with args `status`, and `executeModel` sends any non-empty args to `sessions.patch`. The linked bug’s logs match that path.

## Solution Assessment

No. The PR prevents the immediate bad `sessions.patch` call, but it does so by collapsing documented `list` and `status` behavior into the bare summary and reserving undocumented `info`; explicit subcommand handling aligned with the shared directive is the safer fix.
