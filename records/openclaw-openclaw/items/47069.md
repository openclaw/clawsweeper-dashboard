## Summary

Keep open: the reported Control UI bug is still source-reproducible on current main, but this PR is not correct as-is because it collapses documented `/model list` and `/model status` behaviors into the bare summary path and adds an undocumented `info` alias.

## What This Changes

The PR changes the Control UI `/model` slash-command executor so `status`, `list`, and `info` are treated as informational requests rather than model identifiers, with executor tests for those cases and model selection.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` and `/model status` through the documented informational behavior, preserves normal model selection, and leaves `info` unchanged unless the shared contract is extended.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `/model status` parses to `args: "status"`, Control UI dispatch passes it unchanged, and current main calls `sessions.patch` for every non-empty `/model` argument.

## Solution Assessment

No. Avoiding the bad `sessions.patch` call is necessary, but this PR should preserve the documented `list` and `status` responses and should not add `info` without a shared contract/docs decision.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Route list and status to their documented handlers:** `ui/src/ui/chat/slash-command-executor.ts:125`
  - body: The new guard sends `/model list` and `/model status` into the same bare `/model` summary branch. Docs and the shared directive handler distinguish those modes: `list` should use the model browser/picker path and `status` should show provider/auth/endpoint details, so this avoids the patch call but still returns the wrong Control UI response.
  - confidence: 0.9
- **[P3] Avoid adding the undocumented info alias:** `ui/src/ui/chat/slash-command-executor.ts:118`
  - body: Adding `info` to the reserved set creates a Control UI-only alias that the docs and shared directive contract do not recognize. It also prevents selecting a real model or alias named `info`; either define the alias across the shared contract and docs or leave it on the model-selection path.
  - confidence: 0.82
