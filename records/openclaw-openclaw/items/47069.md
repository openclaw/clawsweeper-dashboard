## Summary

Keep open. Current main still source-reproduces the reported Control UI `/model status` and `/model list` failure, but this PR is not merge-ready because it returns the bare summary instead of the documented list/status responses and still lacks after-fix real behavior proof.

## What This Changes

The PR changes the Control UI slash-command executor so `/model status`, `/model list`, and `/model info` use the bare model-info branch, and adds executor tests plus a normal model-selection regression test.

## Best Possible Solution

Land or replace one narrow Control UI fix that handles `/model list` and `/model status` according to the docs/shared directive behavior, preserves normal model selection, and only adds `info` if the shared contract is updated.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: `/model status` parses to `args: "status"`, Control UI dispatch passes it unchanged, and `executeModel` calls `sessions.patch` for every non-empty `/model` argument.

## Solution Assessment

No. Avoiding the bad `sessions.patch` call is necessary, but this PR collapses documented `list` and `status` modes into the bare summary branch and adds an undocumented `info` alias.
