## Summary

Keep open: this PR targets a real Control UI model-command bug, but it still lacks after-fix UI proof, is conflicting, and routes documented `/model list` and `/model status` modes to the bare summary path while adding an undocumented `info` keyword.

## What This Changes

The PR adds a Control UI guard that treats `/model status`, `/model list`, and `/model info` as existing model-info queries, plus executor tests for those arguments and normal model selection.

## Best Possible Solution

Land one rebased canonical Control UI fix that routes `/model list` and `/model status` to documented flows, preserves normal model selection, removes or documents `info`, and includes focused tests plus redacted UI proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `/model status` and `/model list` parse as non-empty args, dispatch preserves those values, and `executeModel` sends every non-empty `/model` argument to `sessions.patch`.

## Solution Assessment

No. Avoiding the bad `sessions.patch` call is necessary, but this PR collapses documented `list` and `status` modes into the bare summary branch and adds an undocumented `info` keyword.
