## Summary

Keep open. Current main still has the reported Control UI `/model status` and `/model list` failure path, but this PR should not merge as written because it collapses documented `list` and `status` behavior into the bare summary branch and lacks contributor-run real behavior proof.

## What This Changes

The PR reserves `status`, `list`, and `info` in the Control UI `/model` executor and adds tests for those tokens plus a normal model-setting regression.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` to the compact picker/list behavior, routes `/model status` to detailed status/auth/endpoint output, preserves normal model selection, and leaves `info` unchanged unless the shared contract and docs are updated.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `/model status` parses to `args: "status"`, Control UI dispatch passes it unchanged, and current `executeModel` calls `sessions.patch` for every non-empty argument.

## Solution Assessment

No. Avoiding the bad patch call is the right direction, but this patch returns the wrong documented responses for `list` and `status` and adds `info` as a new reserved alias without shared contract/docs support.

## Review Findings
