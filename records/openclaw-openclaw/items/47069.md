## Summary

Keep open. ## Best Possible Solution

Revise or replace one narrow Control UI fix that routes `/model list` and `/model status` to the documented informational flows, preserves normal model selection, removes or explicitly documents `info`, and adds focused tests plus redacted real Control UI proof.

## Solution Assessment

No. Avoiding the bad `sessions.patch` call is necessary, but this PR collapses documented `list` and `status` modes into the bare summary branch and adds an undocumented `info` mode instead of aligning with the shared directive contract.
