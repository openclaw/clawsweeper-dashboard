## Summary

Keep open: the linked Control UI bug is still present on current main, but this PR should not merge as-is because it routes documented `/model list` and `/model status` modes into the bare summary branch and also reserves undocumented `info`. The repair is narrow, but it needs to align with the shared command contract, add the required changelog entry, and coordinate the overlapping open fix PR for the same bug.

## What This Changes

The PR adds a Control UI `/model` reserved-subcommand set for `status`, `list`, and `info`, plus tests that those tokens show the existing bare model summary instead of calling `sessions.patch` and a regression test for a normal model name.

## Best Possible Solution

Align the Control UI local `/model` handling with the shared directive/docs contract: preserve normal model selection, make `/model list` use the model-list/picker path, make `/model status` show detailed status/auth/endpoint information, and leave `info` unreserved unless the shared contract is deliberately extended.

## Reproduction Assessment

Yes by source inspection. On current main, `/model status` and `/model list` parse as non-empty args and `executeModel` sends those args to `sessions.patch`; I did not run tests because this review is read-only and `pnpm` is unavailable in the checkout.

## Solution Assessment

No. The PR prevents the bad patch call, but it does so by dropping documented `list` and `status` semantics and by adding an undocumented `info` reservation that the shared command contract does not define.
