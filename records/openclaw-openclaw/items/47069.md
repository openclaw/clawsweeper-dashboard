## Summary

Keep open. The underlying Control UI bug is still source-reproducible on current main, but this PR is not merge-ready because it collapses documented `/model list` and `/model status` behavior into the bare summary path, adds an undocumented `info` reservation, and still lacks real behavior proof from a live Control UI run.

## What This Changes

The PR changes Control UI `/model` slash-command handling so `status`, `list`, and `info` route to the model-info branch instead of `sessions.patch`, with regression tests for those arguments and normal model selection.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` to the documented model-list/picker behavior, routes `/model status` to detailed status/auth/endpoint output, preserves normal model selection, and leaves `info` unreserved unless the shared contract and docs are deliberately extended.

## Reproduction Assessment

Yes, by source inspection. Current main parses `/model status` as `args: "status"`, then `executeModel` treats every non-empty argument as a model patch and calls `sessions.patch`; I did not run UI/tests because this review is read-only.

## Solution Assessment

No. The PR prevents the bad patch call, but it does so by dropping the documented `list` and `status` semantics and by adding an undocumented `info` alias that needs contract/docs agreement first.
