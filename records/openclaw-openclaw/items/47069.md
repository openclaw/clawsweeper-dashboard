## Summary

Keep open: the current-main bug is still source-reproducible, but this PR is not merge-ready because it collapses documented `/model list` and `/model status` behavior into the bare summary path, adds an undocumented `info` alias, and lacks contributor-run real behavior proof.

## What This Changes

The PR reserves `status`, `list`, and `info` in the Control UI `/model` executor so they show the existing model-info summary instead of calling `sessions.patch`, and adds slash-command executor tests including a normal model-setting regression.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` to the documented model-list/picker behavior, routes `/model status` to detailed status/auth/endpoint output, preserves normal model selection, and treats `info` only after the shared contract and docs agree.

## Reproduction Assessment

Yes, from source inspection rather than a live UI run: current main parses `/model status` as `args: "status"`, dispatches it unchanged, and then calls `sessions.patch` for every non-empty model argument.

## Solution Assessment

No. Avoiding the bad patch call is the right direction, but this patch returns the wrong documented responses for `list` and `status` and adds `info` as a new reserved alias without a shared contract/docs update.
