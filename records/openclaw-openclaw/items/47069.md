## Summary

Keep open. The underlying Control UI bug is still present on current main, but this PR should not merge as-is because it collapses documented `/model list` and `/model status` behavior into the bare summary path and also reserves undocumented `info`.

## What This Changes

The PR changes the Control UI slash-command executor so `/model status`, `/model list`, and `/model info` use the existing model-information branch instead of `sessions.patch`, and adds node tests including a normal model-selection regression.

## Best Possible Solution

Align the Control UI local `/model` implementation with the shared command contract: bare `/model` can keep the current summary or picker entry point, `/model list` should use the model browser/list behavior, `/model status` should show detailed model/auth/endpoint diagnostics, ordinary model selections should still call `sessions.patch`, and `info` should only become reserved if the shared parser and docs add that mode consistently.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: `/model status` is parsed as local command `model` with args `status`, and `executeModel` sends any non-empty args to `sessions.patch`.

## Solution Assessment

No. The PR prevents the immediate bad patch call, but it is not the best fix because it maps documented `list` and `status` modes to the bare summary and reserves undocumented `info`; the safer fix is explicit mode handling that matches the shared directive and docs.
