## Summary

Keep open: the PR targets a valid source-reproducible Control UI bug, but the current patch still returns the wrong documented behavior for `/model list` and `/model status`, adds an undocumented `info` alias, and lacks required after-fix real Control UI proof.

## What This Changes

The PR changes the Control UI `/model` slash-command executor so `status`, `list`, and `info` use the model-info branch instead of `sessions.patch`, and adds executor regression tests.

## Best Possible Solution

Revise or replace one narrow Control UI fix that routes `/model list` and `/model status` to the documented list/status flows, preserves normal model selection, removes or documents `info`, and adds focused tests plus redacted real Control UI proof.

## Reproduction Assessment

Yes, source-backed with high confidence: `/model status` parses to `args: "status"`, Control UI dispatch passes that string unchanged, and current main sends every non-empty `/model` argument to `sessions.patch`. I did not run a live Control UI session in this read-only review.

## Solution Assessment

No. Avoiding the bad `sessions.patch` call is necessary, but this PR collapses `/model list` and `/model status` into the bare summary branch and adds an undocumented `info` mode instead of following the documented handlers.
