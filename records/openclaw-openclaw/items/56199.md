## Summary

Keep open. Current main and the latest release still expose DeepSeek V3.2 only as the general Volcengine online-inference model ID, while the Coding Plan catalog omits the vendor-documented `deepseek-v3.2` plan model that matches the reporter's Lite/Coding Plan setup.

## What This Changes

The issue reports a Volcengine Lite/Coding Plan API key passing OpenClaw setup checks but failing at runtime with `401 The API key format is incorrect`.

## Best Possible Solution

Align the bundled Volcengine Coding Plan catalog and docs with the vendor-supported `/api/coding/v3` model names while keeping online-inference IDs under the general `volcengine` provider.
