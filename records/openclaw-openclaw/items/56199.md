## Summary

Keep open. Current `main` and the latest release still show the Volcengine general DeepSeek V3.2 row under `/api/v3`, while the Coding Plan provider uses `/api/coding/v3` and still lacks the vendor-documented `deepseek-v3.2` plan model that matches the reporter's Lite/Coding Plan setup.

## What This Changes

The issue reports a Volcengine Lite/Coding Plan API key passing setup but failing at runtime with `401 The API key format is incorrect` when using DeepSeek.

## Best Possible Solution

Align the Volcengine plugin-owned Coding Plan catalog and docs with the vendor-supported `/api/coding/v3` DeepSeek model ID, preserving the general online-inference catalog.
