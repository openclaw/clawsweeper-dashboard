## Summary

Keep open. Current main and the latest release still advertise the Volcengine online-inference DeepSeek V3.2 row under the general provider, while the Coding Plan provider lacks the vendor-documented `deepseek-v3.2` model that matches the reporter's Lite/Coding Plan setup.

## What This Changes

The issue reports a Volcengine Lite/Coding Plan key that passes local setup but fails at runtime with `401 The API key format is incorrect` when using a DeepSeek Volcengine model.

## Best Possible Solution

Align the Volcengine plugin-owned Coding Plan catalog and docs with the vendor-supported `/api/coding/v3` model IDs while preserving the general online-inference catalog.
