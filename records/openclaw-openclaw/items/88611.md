## What This Changes

The issue requests an agent-facing TaskFlow managed-mode authoring surface for creating, linking, waiting, resuming, finishing, and failing flows.

## Best Possible Solution

Keep core TaskFlow CLI/tooling focused on inspection and cancellation, and publish any agent-facing taskflow tool as a ClawHub plugin built on `api.runtime.tasks.managedFlows`.

Not applicable; this is a new agent/tool surface request, not a broken existing TaskFlow behavior.

Yes for closing as ClawHub scope. The plugin runtime already provides the needed managed-flow API for a plugin-owned agent tool, while a default core tool would add a new orchestration surface contrary to the lean-core direction.
