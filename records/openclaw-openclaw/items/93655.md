## Summary

Keep open: this PR addresses a real downstream embedded-run recovery gap and now implements the maintainer-requested phase-safe split between model stalls, side-effecting tool stalls, and unknown/pre-model recovery. I found no blocking patch defect, but the session-state/availability risk and overlap with the provider-boundary opencode-go fixes still need maintainer review before merge.

## What This Changes

The PR preserves diagnostic `stuck_recovery` through embedded-run aborts, classifies stuck recovery by active attempt phase, tracks active potential-side-effect tools, and adds focused regression tests.

## Best Possible Solution

Land this only as the downstream replay-safety guard for generic stuck-session recovery, with the opencode-go stream termination fix reviewed separately as the provider-boundary solution.
