## Summary

Keep open: the PR targets a real Feishu mention-format bug that current main still has, but the current branch is not mergeable as-is because it predates reply fallback plumbing, expands a trimmed Feishu entrypoint surface, uses an unproven native-mention post layout, and lacks real Feishu behavior proof.

## What This Changes

The PR changes Feishu plain post sends to prepend structured mention targets as native post `at` elements, adds Feishu send tests, adds a changelog entry, and re-exports mention helpers from the Feishu plugin entrypoint.

## Best Possible Solution

Land a rebased Feishu send-path fix that preserves reply fallback behavior, emits native `at` elements only from explicit structured mention metadata using a Feishu-valid post layout, keeps body text literal, avoids entrypoint export expansion unless deliberately approved, and includes redacted real Feishu proof.

## Reproduction Assessment

Yes for the source-level payload path: current main still turns structured mention targets into `<at user_id="...">` strings inside one post `md` element. No live Feishu tenant proof currently shows the visible blue-rendering failure or the after-fix result.

## Solution Assessment

No, not as-is. Preserving structured mention metadata until post serialization is the right direction, but this branch must be rebased to preserve reply fallback behavior, avoid accidental public export expansion, and use a Feishu-valid native-mention layout before merge.
