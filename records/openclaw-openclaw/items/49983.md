## Summary

Keep open: current main still serializes structured Feishu mention targets as pseudo-XML inside one post `md` element, and this PR is the right general direction, but the branch is stale against current Feishu reply and mention-routing behavior, lacks live Feishu proof, and needs API-layout cleanup before merge.

The PR changes Feishu plain post sends to emit structured mention targets as native post `at` elements, adds Feishu send tests, expands Feishu entrypoint exports, and adds a changelog entry.

## Best Possible Solution

Land a rebased Feishu fix that preserves reply fallback and explicit-only mention policy, emits Feishu-valid native `at` elements from structured targets, keeps private helpers out of the entrypoint unless deliberately promoted, and includes redacted live Feishu proof.

## Solution Assessment

No, not as-is. Preserving structured mention metadata until post serialization is the right direction, but this branch needs a current-main rebase, Feishu-valid payload layout proof or adjustment, and real behavior proof before merge.
