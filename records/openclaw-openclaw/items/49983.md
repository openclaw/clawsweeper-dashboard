## Summary

Keep open: this is the canonical active fix for the Feishu mention-color bug and current main still serializes structured mentions into one `md` payload, but the head is not merge-ready because it lacks live Feishu proof and has concrete compatibility regressions against current main and the upstream post-message layout contract.

## What This Changes

This PR updates Feishu post-message serialization to prepend native `at` elements from structured mention targets, adds Feishu send tests, updates `CHANGELOG.md`, and expands the Feishu plugin entrypoint exports.

## Best Possible Solution

Land a rebased Feishu send-path fix that preserves current reply fallback behavior, emits native `at` elements only from explicit structured mention metadata using a Feishu-valid post layout, keeps body text literal, avoids entrypoint export expansion unless deliberately approved, and includes redacted real Feishu proof.

## Reproduction Assessment

Yes for the source-level payload path: current main can be inspected to show structured mention metadata is converted into pseudo-XML text inside one `md` post element. No live Feishu tenant proof currently demonstrates the visible blue-rendering failure or the after-fix result.

## Solution Assessment

No, not as-is. Preserving structured mention metadata until post serialization is the right direction, but this branch must preserve current reply fallback plumbing and use a Feishu-valid native-mention plus Markdown layout before it is the narrow maintainable fix.

Feishu's post contract says `md` nodes occupy their own paragraph and cannot share a line with other tags, so mentioned messages can be rejected or misrendered; keep the mention row text-only or move Markdown into a separate valid paragraph.
  - confidence: 0.82
