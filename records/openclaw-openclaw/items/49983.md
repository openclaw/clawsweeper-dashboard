Keep open: current main still flattens explicit Feishu `mentions` into one post `md` element, so the central send-path fix remains useful, but this branch is not merge-ready because it carries a stale Feishu entrypoint export hunk and lacks real Feishu runtime proof.

This PR changes Feishu post-message serialization to emit native `at` elements from structured mention targets, adds focused send tests, updates the changelog, and adds Feishu entrypoint helper exports.

## Best Possible Solution

Land a narrow Feishu send-path fix that emits native post `at` elements only from explicit structured mention metadata, keeps body `<at ...>` text literal, avoids entrypoint export expansion, and includes redacted real Feishu proof.

## Reproduction Assessment

Yes for the code-level bug: current main can be source-checked by calling `sendMessageFeishu` with a non-empty `mentions` array and inspecting that the outgoing post payload receives pseudo-XML text inside one `md` element. I did not run a live Feishu tenant reproduction in this read-only review.
