## Summary

Keep open: current main still has the Feishu post-mention serialization gap, and this PR is the active canonical fix path, but the branch is not mergeable as-is because it lacks real Feishu proof and conflicts with newer Feishu fallback and mention-routing behavior.

## What This Changes

The PR changes Feishu post serialization, tests, and changelog text to emit native `at` elements for structured mentions while preserving literal body text, and it exports mention helpers from the Feishu plugin entrypoint.

## Best Possible Solution

Rebase a narrow Feishu fix that serializes only explicit structured mention targets as Feishu-valid post `at`/`text` rows, preserves reply fallback, keeps helpers private, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes for the source-level path: current main still turns structured Feishu `mentions` into `<at user_id="...">` text inside one post `md` element. No high-confidence live Feishu tenant reproduction of the color difference is present.

## Solution Assessment

No, not as-is. Native `at` elements are the right direction, but the PR needs a current-main rebase, fallback preservation, Feishu-valid post text elements or live API proof, explicit mention routing, and real behavior proof.
