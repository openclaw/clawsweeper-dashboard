## Summary

Keep open: the Feishu mention-rendering fix is still relevant, but this branch is not merge-ready against current main because it lacks real Feishu proof and conflicts with newer reply fallback, explicit-only mention routing, and plugin boundary changes.

## What This Changes

The PR changes Feishu post sends to emit structured mentions as native post `at` elements, adds Feishu send tests, expands Feishu entrypoint exports, and adds a changelog entry.

## Best Possible Solution

Land a rebased Feishu fix that emits native `at` elements only from explicit structured mention targets, preserves reply fallback and private plugin boundaries, uses or proves a Feishu-valid post layout, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes for the source-level payload path: current main still turns structured Feishu mentions into `<at user_id="...">` text inside one post `md` element. I did not establish live Feishu tenant proof of the visible white-versus-blue rendering.

## Solution Assessment

No, not as-is. Preserving structured mention metadata until post serialization is the right direction, but this branch needs a current-main rebase, explicit-only mention routing, payload-layout proof or adjustment, and real behavior proof before merge.
