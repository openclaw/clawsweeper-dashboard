## Summary

Keep this PR open: current main still has the source-level Feishu post mention problem, but this branch is not merge-ready because it lacks real Feishu proof and still has concrete correctness blockers from the latest review.

## What This Changes

The PR changes Feishu plain post sends to serialize structured mention targets as native post `at` elements, adds Feishu send tests, expands Feishu entrypoint exports, and adds a changelog entry.

## Best Possible Solution

Land a rebased Feishu send-path fix that preserves current reply fallback behavior, keeps entrypoint exports intentional, emits Feishu-valid native mention elements from structured metadata, and includes redacted real Feishu proof.

## Reproduction Assessment

Yes for the source-level payload path: current main still converts structured Feishu mention targets into `<at user_id="...">` text inside one post `md` element. I do not have live Feishu tenant proof showing the visible before/after rendering.

## Solution Assessment

No, not as-is. Preserving structured mention metadata until post serialization is the right direction, but this branch needs rebase/repair for reply fallback behavior, entrypoint exports, and a Feishu-valid post element layout before merge.
