## Summary

Keep this PR open, but do not merge it as-is: the Feishu mention bug is still source-reproducible on current main, while the branch lacks real Feishu proof and has current-main regressions around reply fallback, explicit mention routing, post element shape, and plugin entrypoint surface.

## What This Changes

The PR changes Feishu post serialization to accept structured mentions, emit native `at` elements with name fallbacks, add send tests, export mention helpers from the plugin entrypoint, and add a changelog entry.

## Best Possible Solution

Rebase a narrow Feishu fix that preserves reply fallback, keeps mention helpers private, converts only explicit structured mention targets into Feishu-valid post rows, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes for the source-level serialization path: current main turns structured Feishu `mentions` into `<at user_id="...">` text inside a single post `md` element. No live Feishu tenant proof for the white-versus-blue rendering is present.

## Solution Assessment

No, not as-is. Native `at` elements are a plausible fix direction, but this branch needs a current-main rebase, explicit-only mention routing, Feishu-valid post text elements or live API proof, and real behavior proof before merge.
