The PR changes Feishu post message serialization so structured mentions are emitted as native `at` elements instead of inline `<at ...>` text inside one post body.

## Best Possible Solution

Rebase a narrow Feishu fix that preserves reply fallback, keeps mention helpers private, converts only explicit structured mention targets into Feishu-valid post rows, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes for the source-level serialization path: current main turns structured Feishu `mentions` into `<at user_id="...">` text inside a single post `md` element. No live Feishu tenant reproduction/proof is present for the white-versus-blue rendering itself.

## Solution Assessment

No, not as-is. Native `at` elements are the right direction, but the PR needs a current-main rebase, fallback preservation, documented post `text` elements or live API proof, explicit mention routing, and real behavior proof.
