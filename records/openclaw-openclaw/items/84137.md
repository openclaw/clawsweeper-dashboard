## What This Changes

The issue asks whether Codex app-server should keep post-tool raw assistant completion on the current fail-fast default or move it to a distinct longer post-tool synthesis budget by default.

## Best Possible Solution

Keep the current compatible default unless maintainers approve a longer post-tool default backed by real Codex app-server workload evidence, then update the runtime, docs, and focused regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a semantic/defaults decision. Source and tests give high-confidence proof of the current behavior: post-tool raw assistant progress uses the short assistant idle budget, while pre-tool raw assistant output can release final text.

## Solution Assessment

Unclear until maintainers choose the desired default. The linked config-knob PR is the conservative short-term path, while changing the default should wait for workload proof and explicit agreement on stuck-turn latency.
