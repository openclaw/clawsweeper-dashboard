## Summary

Keep open: current main has the conservative configurable timeout from the related merged PR, but it intentionally preserves the default fail-fast semantics and does not settle the post-tool terminal-state/subagent handoff decision called out by the issue discussion.

## What This Changes

The issue asks whether Codex post-tool raw assistant completion should keep the current short fail-fast default or become a distinct longer post-tool synthesis/terminal-wait state.

## Best Possible Solution

Keep the compatible default until maintainers approve a semantic change backed by real Codex workload evidence, then implement a distinct post-tool terminal-wait contract with focused regression coverage and split any proven independent subagent handoff bug.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this issue asks for a default semantics decision. Current-main source and tests prove the present behavior, while the subagent handoff logs are useful evidence but not a reproduced current-main failure in this review.

## Solution Assessment

Unclear until maintainers choose the intended default. The merged config knob is the conservative short-term solution; changing the default or terminal-state contract needs workload proof and explicit acceptance of the stuck-turn latency tradeoff.
