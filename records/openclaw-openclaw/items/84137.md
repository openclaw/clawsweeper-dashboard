## What This Changes

This issue asks whether Codex app-server post-tool raw assistant completion should keep the current short fail-fast default or move to a longer/clearer post-tool synthesis and terminal-state contract.

## Best Possible Solution

Keep the opt-in timeout knob as the compatibility path, then make an explicit maintainer decision on the default post-tool terminal contract with workload proof and regression coverage for pre-tool release, post-tool budget, stuck-turn latency, and queued subagent handoff behavior.
