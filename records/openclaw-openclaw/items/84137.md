## Summary

Keep open: current main has the conservative operator timeout knob, but the default post-tool semantics and the stale subagent embedded-run handoff question remain unresolved, and a maintainer explicitly asked to keep this as the semantic hub.

## What This Changes

The issue asks whether Codex app-server post-tool raw assistant completion should keep the current fail-fast default or become a longer/default terminal-state contract, including subagent handoff interactions.

## Best Possible Solution

Keep the compatible default for now, then change the default or terminal-state contract only after maintainers approve the tradeoff and a follow-up PR adds focused pre-tool, post-tool, stuck-turn, and subagent handoff proof.

## Reproduction Assessment

There is no high-confidence bug reproduction for the remaining semantic decision. Source and tests do reproduce the current default/fallback behavior, while the subagent handoff report supplies realistic logs but not a focused current-main repro.

## Solution Assessment

Unclear until maintainers choose the desired default. The merged config knob is the right conservative short-term path, but a default semantic change needs workload proof and explicit acceptance of slower stuck-turn detection.
