## Summary

Keep open. Current main still resolves the context engine in the native subagent-spawn preparation path before any built-in context-engine initialization, and open PR #73904 already references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that native subagent spawn preparation resolves the configured context engine before built-in engines are initialized, causing cliBackend-only cold starts to fail with an unregistered legacy engine.

## Best Possible Solution

Land PR #73904, or an equivalent narrow patch, so native subagent spawn preparation initializes built-in context engines before resolving the configured engine. Keep #75129 as the separate companion for the CLI compaction path unless maintainers choose to combine the two fixes.

## Reproduction Assessment

Yes. A cold process that reaches native `sessions_spawn` before any embedded PI path initializes context engines can call `resolveContextEngine()` against an empty registry and hit the exact reported `legacy` error.

## Solution Assessment

Yes. Initializing built-in context engines at the spawn-side resolution boundary is the narrowest maintainable fix for this path; making registration a module import side effect would have broader runtime and bundling implications.
