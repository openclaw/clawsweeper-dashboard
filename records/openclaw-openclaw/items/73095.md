## Summary

Keep open. Current main still has the reported initialization-order bug in the native subagent spawn preparation path, and open PR #73904 references this issue with closing syntax, so this should remain open until that PR or an equivalent fix lands.

## What This Changes

Initialize built-in context engines before native `sessions_spawn` subagent preparation resolves the configured context engine on cliBackend-only cold starts.

## Best Possible Solution

Land PR #73904, or an equivalent narrow patch, so `prepareContextEngineSubagentSpawn()` initializes built-in context engines before resolving. Keep the CLI compaction pattern tracked separately through #75129 unless maintainers choose to combine the fixes.

## Reproduction Assessment

Yes. A cold cliBackend-only process can enter `sessions_spawn` through `spawnSubagentDirect()` and reach `prepareContextEngineSubagentSpawn()` before any embedded PI path initializes the registry, causing the exact missing-legacy-engine error.

## Solution Assessment

Yes. Initializing built-in engines at the spawn preparation resolution boundary is the narrowest maintainable fix for this path; making registration an import side effect would be broader and riskier.
