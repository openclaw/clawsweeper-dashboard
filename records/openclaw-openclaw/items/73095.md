## Summary

Keep open. Current main still reaches the native subagent spawn context-engine resolution path without initializing built-in engines first, and open PR #73904 already references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Fix `sessions_spawn` on cliBackend-only cold starts by initializing built-in context engines before `prepareContextEngineSubagentSpawn()` resolves the configured context engine.

## Best Possible Solution

Land PR #73904, or an equivalent focused patch, so native subagent spawn preparation initializes built-in context engines before resolving the configured engine. Keep PR #75129 as the separate companion for the CLI compaction path unless maintainers choose to combine both init-order fixes.

## Reproduction Assessment

Yes. A cold process that reaches native `sessions_spawn` before any embedded PI path initializes context engines will call `resolveContextEngine()` against an empty registry and can hit the exact reported `legacy` error.

## Solution Assessment

Yes. Initializing built-in context engines at the spawn-prep resolution boundary is the narrowest maintainable fix for this path; making registration a top-level import side effect would have broader runtime and bundling implications.
