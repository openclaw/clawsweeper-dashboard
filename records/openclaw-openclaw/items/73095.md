## Summary

Keep open. Current main still has the reported cold-registry ordering bug in the native subagent spawn path, and open PR #73904 already references this issue with closing syntax, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests initializing the built-in context-engine registry before native `sessions_spawn` subagent preparation resolves the configured engine, especially for cliBackend-only cold starts.

## Best Possible Solution

Land PR #73904, or an equivalent narrow patch, so native subagent spawn preparation initializes built-in context engines before resolving the configured engine. Keep PR #75129 as the separate companion for the CLI compaction path unless maintainers choose to combine the two fixes.

## Reproduction Assessment

Yes. A cold process that reaches native `sessions_spawn` before any embedded PI path initializes context engines can call `resolveContextEngine()` against an empty registry and hit the exact reported `legacy` error.

## Solution Assessment

Yes. Initializing built-in context engines at the spawn-side resolution boundary is the narrowest maintainable fix for this path; making registration a module import side effect would have broader runtime and bundling implications.
