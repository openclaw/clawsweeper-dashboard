## Summary

Keep open. Current main still has the reported resolve-before-initialize path in native subagent spawn preparation, and open PR #73904 references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that cliBackend-only cold starts can call subagent spawn preparation before built-in context engines are initialized, causing `sessions_spawn` to fail with a missing `legacy` context engine.

## Best Possible Solution

Land PR #73904, or an equivalent narrow patch, so `prepareContextEngineSubagentSpawn()` initializes built-in context engines before resolving. Keep the CLI compaction mirror path tracked through #75129 unless maintainers intentionally combine the fixes.

## Reproduction Assessment

Yes. A cold cliBackend-only process can enter `sessions_spawn` through `spawnSubagentDirect()` and reach `prepareContextEngineSubagentSpawn()` before any embedded PI path initializes the registry, producing the exact missing-legacy-engine error.

## Solution Assessment

Yes. Initializing built-in engines at the spawn preparation resolution boundary is the narrowest maintainable fix for this path; making registration an import side effect would be broader and riskier.
