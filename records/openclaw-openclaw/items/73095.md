## Summary

Keep open. Current main still resolves the context engine in the native subagent spawn preparation path without first initializing built-in context engines, and open PR #73904 is already linked as the focused implementation candidate for this exact issue.

## What This Changes

The issue reports that cliBackend-only cold starts can fail native `sessions_spawn` because `prepareContextEngineSubagentSpawn()` resolves the default `legacy` context engine before the lazy registry initializer runs.

## Best Possible Solution

Land PR #73904, or an equivalent narrow patch, so native subagent spawn preparation initializes built-in context engines before resolving; keep the cli-compaction follow-up in #75129 separate unless maintainers intentionally combine the two call sites.

## Reproduction Assessment

Yes. A cold cliBackend-only process can enter `sessions_spawn` through `spawnSubagentDirect()` and reach `prepareContextEngineSubagentSpawn()` before any embedded PI path initializes the context-engine registry, producing the reported missing-`legacy` error.

## Solution Assessment

Yes. Initializing built-in engines at the spawn preparation boundary is the narrowest maintainable fix for this path; changing registration to import-time side effects would be broader and riskier.
