## Summary

Keep open. Current main still has the reported cold-registry ordering bug in the native `sessions_spawn` path, and open PR #73904 references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests initializing the context-engine registry before native subagent spawn preparation resolves the configured engine, preventing cliBackend-only cold starts from failing on the default `legacy` engine.

## Best Possible Solution

Land PR #73904, or an equivalent narrow patch, so native subagent spawn preparation initializes built-in context engines before resolving the configured engine. Keep #75129 as the companion CLI compaction fix unless maintainers intentionally combine the two paths.

## Reproduction Assessment

Yes. A cold process that reaches native `sessions_spawn` before any embedded PI path initializes context engines can call `resolveContextEngine()` against an empty registry and hit the exact reported `legacy` error.

## Solution Assessment

Yes. Initializing built-in context engines at the spawn-side resolution boundary is the narrowest maintainable fix for this path; moving registration to a module import side effect would have broader runtime and bundling implications.
