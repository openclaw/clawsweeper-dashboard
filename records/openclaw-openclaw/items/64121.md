## Summary

Keep open. Current main still renders one combined chat session selector on desktop and mobile, with agent grouping only inside that selector rather than separate Agent and Agent Session controls. The related #68959 issue has already been closed as a duplicate of this tracker, #70851 only addresses internal-session picker noise, and #68926 remains the open implementation candidate rather than a merged fix.

## What This Changes

Split the Control UI chat picker into separate Agent and Agent Session controls while keeping Model and Thinking bound to the active session.

## Best Possible Solution

Land a revised #68926 or equivalent Control UI change that adds an Agent selector from agentsList, filters Agent Session choices to the selected agent, switches agents to the most recent session or agent main fallback, preserves current Model/Thinking active-session patch behavior, and covers both desktop and mobile controls with focused tests.

## Reproduction Assessment

Yes. A source-level reproduction is enough: current main renders one session select from resolveSessionOptionGroups on desktop and mobile, and configured agents without session rows are not directly selectable as separate Agent choices.

## Solution Assessment

Yes. The requested split remains the right product direction, but it should build on the shared chat session-control helpers and preserve current session metadata contracts rather than duplicating older selector logic.
