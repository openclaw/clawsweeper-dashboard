## Summary

Keep open. Current main still renders one combined session selector on desktop and mobile, with agent grouping only inside that selector rather than separate Agent and Agent Session controls. The newer #68959 discussion correctly closed as a duplicate of this tracker, #70851 is only an internal-session noise mitigation, and open PR #68926 is the current implementation candidate but is unmerged and has blocking review feedback.

## What This Changes

Split the Control UI chat picker into separate Agent and Agent Session controls while keeping Model and Thinking bound to the active session.

## Best Possible Solution

Land a revised #68926 or equivalent Control UI change that adds an Agent selector from agentsList, filters the Agent Session selector to the selected agent, switches agents to the most recent session or agent main fallback, preserves the current Model/Thinking active-session patch behavior, and covers both desktop and mobile controls with focused tests.

## Reproduction Assessment

Yes. A source-level reproduction is enough: current main renders one session select from resolveSessionOptionGroups on desktop and mobile, and configured agents without session rows are not directly selectable as separate Agent choices.

## Solution Assessment

Yes, the requested split is the right product direction for this issue. The safest implementation should build on the current shared chat session-control helpers and preserve the current thinking/model metadata contracts rather than duplicating stale selector logic.
