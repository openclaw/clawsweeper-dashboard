## Summary

Keep open. Current main still has a single combined chat session picker on desktop and mobile, with agent grouping inside that picker rather than separate Agent and Agent Session controls; related #68926 is the open implementation candidate, not a merged fix.

## What This Changes

Split the Control UI chat session picker into separate Agent and Agent Session controls while keeping Model and Thinking scoped to the active session.

## Best Possible Solution

Land a revised #68926 or equivalent shared Control UI change that adds an Agent selector from agentsList, filters Agent Session choices to the selected agent, preserves sessions.patch behavior for Model and Thinking, and covers desktop and mobile controls with focused tests.

## Reproduction Assessment

Yes. Source inspection on current main shows one combined session select on desktop and mobile, with agent names only used as optgroup labels rather than a separate Agent control.

## Solution Assessment

Yes. The requested split is the maintainable direction, but it should build on the shared chat session-control helpers and active-session patching contracts rather than duplicating picker logic.
