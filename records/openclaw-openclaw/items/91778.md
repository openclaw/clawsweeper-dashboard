## Summary

This remains a valid released memory-core regression report: v2026.6.5 can pause vector search when identity metadata is absent, and current main still fails closed on that state. Current main contains additional atomic-reindex metadata-cache work, but neither that change nor the open related PRs conclusively fix the reported successful OpenAI-compatible rebuild followed by missing metadata.

## What This Changes

Fix `memory_search` remaining paused with missing index metadata after an explicit rebuild when using an OpenAI-compatible embedding provider.

## Best Possible Solution

Preserve fail-closed protection for genuine provider or model changes, but make an explicit CLI rebuild atomically persist and reopen matching `memory_index_meta_v1`; add a real OpenAI-compatible rebuild, status, and runtime regression covering the reporter's configuration.

## Reproduction Assessment

Yes at source level: released and current code clearly produce the reported paused state when identity metadata is missing, and the issue supplies repeatable CLI/runtime evidence. The precise reason metadata becomes missing after the successful OpenAI-compatible rebuild still needs a focused live current-main reproduction.

## Solution Assessment

No, the suggested unconditional forced sync inside normal search is not yet the best fix because it can cause unexpected paid re-embedding and weaken intentional mismatch protection. The narrower solution is to repair the explicit rebuild's metadata publication and reopen invariant, then prove it with the reported provider setup.
