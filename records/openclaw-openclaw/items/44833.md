## Summary

Keep open. Current main still routes captionless Telegram group photos through a media placeholder but evaluates mention patterns against empty caption text, so configured catch-all group activation can still be skipped before image understanding.

## What This Changes

The issue requests consistent Telegram image understanding for captionless photo-only group messages when configured group mention policy would allow the turn.

## Best Possible Solution

Land a narrow Telegram/shared mention-gating fix with regression coverage for captionless group photos and configured catch-all mention patterns, while preserving default mention-required group behavior.

## Reproduction Assessment

Yes, at source level. Current main deterministically maps a captionless Telegram photo to `<media:image>`, evaluates mention patterns against empty text, and returns `null` from the group no-mention branch before image understanding.

## Solution Assessment

Yes. The narrow maintainable direction is to let configured mention rules authorize media-only Telegram group turns, likely by covering the empty-text catch-all path in `matchesMentionWithExplicit` and Telegram group-body tests rather than changing media download or provider vision code.
