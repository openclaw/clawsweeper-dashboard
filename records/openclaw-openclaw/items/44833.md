## Summary

Keep open. Current main still has the source-level Telegram path where captionless group photos become `<media:image>` but mention matching is evaluated against empty Telegram text, so a configured catch-all pattern does not authorize the turn and the group no-mention branch returns before image understanding.

## What This Changes

Make Telegram group captionless photo messages trigger image understanding consistently with DMs and captioned group photos under the configured group mention policy.

## Best Possible Solution

Land a narrow Telegram/media mention-gating fix with regression coverage for captionless group photos using configured catch-all mention patterns, while preserving default mention-required group behavior.

## Reproduction Assessment

Yes, at source level. Current main deterministically maps a captionless Telegram photo to `<media:image>`, evaluates mention patterns against empty text, and returns `null` from the group no-mention branch before image understanding.

## Solution Assessment

Yes. The narrow maintainable fix is in Telegram/shared mention gating for media-only group turns, not in media download or provider image-understanding logic.
