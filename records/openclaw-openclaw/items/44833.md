## Summary

Keep open. Current main still evaluates Telegram group mention patterns against empty caption text for captionless photo messages, so a configured catch-all pattern can be skipped before the image reaches downstream understanding; the closed related PR was never merged.

## What This Changes

The issue requests that Telegram group photo-only messages trigger image understanding the same way captioned group photos and DM photos do, subject to group mention policy.

## Best Possible Solution

Land a narrow Telegram/shared mention-gating fix with regression coverage for captionless group photos and configured catch-all mention patterns, while preserving default mention-required group behavior.

## Reproduction Assessment

Yes, at source level. Current main maps a captionless Telegram photo to `<media:image>`, evaluates mention matching against empty caption text, and can return `null` from the group no-mention branch before image understanding.

## Solution Assessment

Yes. The narrow maintainable fix is to let configured mention regexes authorize the empty-text media-only case, with Telegram group regression tests, rather than changing media download or provider vision behavior.
