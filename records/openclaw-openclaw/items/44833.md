## Summary

Keep open. Current main still recognizes captionless Telegram photos as media, but group mention matching uses empty caption text and can skip the turn before the image reaches downstream understanding.

## What This Changes

Telegram group photo-only messages should trigger image understanding when the group is configured to allow that activation, instead of requiring a caption workaround.

## Best Possible Solution

Land a narrow Telegram mention-gating fix with regression coverage for captionless group photos and configured catch-all mention patterns, while preserving default mention-required group behavior.

## Reproduction Assessment

Yes, at source level. Current main maps a captionless Telegram photo to `<media:image>`, checks mention patterns against empty caption text, and can return `null` from the group no-mention branch before image understanding runs.

## Solution Assessment

Yes. The narrow maintainable fix is to make the media-only Telegram group activation path honor configured mention rules such as catch-all patterns, with focused tests, rather than changing provider vision or media download behavior.
