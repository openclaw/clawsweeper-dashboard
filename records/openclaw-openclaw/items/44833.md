## Summary

Keep open. Current main still routes captionless Telegram photos to `<media:image>` but evaluates group mentions against empty Telegram text, so configured catch-all patterns such as `.*` do not authorize the turn and the no-mention branch returns before image understanding; the directly related fix PR is closed unmerged.

## What This Changes

Make Telegram group photo-only messages trigger image understanding consistently with DMs when group mention or activation policy allows replies.

## Best Possible Solution

Implement a narrow Telegram/shared mention-gating fix so configured catch-all mention patterns can authorize captionless media-only group turns while preserving normal group safety policy and media routing.

## Reproduction Assessment

Yes, at source level. The issue gives concrete steps, and current main has a deterministic path where captionless Telegram photos have empty mention text, bypass configured regex matching, and hit the group no-mention return before image understanding.

## Solution Assessment

Yes. The narrow maintainable fix is to adjust mention matching or Telegram inbound activation for configured catch-all media-only group turns with focused regression tests, not to change media download or provider image understanding.
