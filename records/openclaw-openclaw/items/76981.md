## Summary

Current main already implements the requested core-plus-external-reference pattern through the native skills system: OpenClaw injects a bounded metadata-only skills catalog, instructs the model to read the selected SKILL.md only when relevant, supports skill-local references, and exposes config limits for prompt overhead. The same behavior is present in the v2026.5.2 release.

## What This Changes

The issue requests native support for splitting large agent instruction files into a small core plus lazily loaded external references to reduce context bloat.

## Best Possible Solution

Keep using the native skills and context-budget surfaces for large playbooks and publish reusable workflows as workspace, plugin-bundled, or ClawHub skills instead of adding a duplicate manifest import layer.

## Reproduction Assessment

Not applicable. This is a feature request, and the current-main plus v2026.5.2 source/docs inspection verifies the requested pattern rather than reproducing a bug.

## Solution Assessment

Yes. The existing skills surface is the narrow maintainable solution: core supplies a metadata-only catalog and on-demand SKILL.md reads with skill-local references, while config budgets cap prompt overhead.
