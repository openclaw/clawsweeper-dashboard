## Summary

Current main already has the requested core-plus-external-reference pattern through skills: OpenClaw injects only a compact skills catalog into the prompt and instructs the agent to read the matching SKILL.md on demand, with per-agent allowlists, skill roots, plugin-shipped skills, and prompt-size limits.

## What This Changes

The issue requests native support for a core prompt plus lazily loaded external references architecture to reduce agent context bloat.

## Best Possible Solution

Keep the existing skills-based progressive disclosure path as the native solution and direct users to move large task-specific procedures into workspace, agent, plugin, or ClawHub skills instead of adding a parallel lazy_load manifest surface.

## Reproduction Assessment

Not applicable. This is a feature request, and source plus docs inspection shows the requested context-bloat mitigation is already available through the current skills system on main.

## Solution Assessment

Yes. The existing skills catalog plus on-demand SKILL.md reads is the narrowest maintainable solution because it reuses the documented skill/plugin/ClawHub surfaces instead of adding another AGENTS.md import mechanism.
