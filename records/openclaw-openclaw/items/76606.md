## Summary

Keep open. Current main still preserves cron `toolsAllow: []`, but the cron executor forwards it without `disableTools` and the embedded runner treats an empty allowlist as unrestricted, so the reported no-tools path is still not honored.

## What This Changes

The issue asks for `agentTurn` runs with `toolsAllow: []` to avoid expensive core/plugin tool construction or otherwise provide a supported explicit no-tools mode.

## Best Possible Solution

Honor the existing cron `toolsAllow: []` disable-all contract by short-circuiting tool construction for explicit empty allowlists, then handle non-empty allowlist profiling as a separate performance follow-up if needed.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: cron preserves `toolsAllow: []`, forwards it to the embedded runner without `disableTools`, and the embedded runner treats the empty list as unrestricted before constructing tools.

## Solution Assessment

Yes for the narrow first fix. Translating explicit cron `toolsAllow: []` into the existing no-tools runtime path is the smallest maintainable repair while preserving undefined allowlists as unrestricted.
