## Summary

Keep open. Current main still preserves empty cron `toolsAllow` arrays as a disable-all payload, but the embedded runner treats the same empty array as unrestricted and builds tools before filtering; related PR #76773 only reduced denied optional media factory work.

## What This Changes

Request a supported no-tools `agentTurn` path and earlier allowlist application so core/plugin tool construction is skipped when no tools are intended.

## Best Possible Solution

Honor explicit empty cron `agentTurn` tool allowlists as disable-all before tool construction, while keeping omitted `toolsAllow` as the default unrestricted mode and leaving broader factory caching/lazy-loading as separate work.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: cron preserves `toolsAllow: []`, forwards it to the embedded runner, and the runner treats it as unrestricted after constructing tools. I did not run a live timing probe.

## Solution Assessment

Yes for the narrow repair: translate an explicitly empty cron/runtime allowlist into the existing `disableTools` short-circuit, or make the embedded runner distinguish omitted from empty before calling `createOpenClawCodingTools`. Avoid bundling broad lazy-loading or cache policy changes into this fix.
