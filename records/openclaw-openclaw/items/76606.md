## Summary

Keep open: current main still preserves explicit empty cron tool allowlists while the embedded runner treats an empty allowlist as unrestricted and builds tools first; merged #76773 only addresses a subset of optional factory cost.

## What This Changes

The issue requests a supported no-tools or pre-construction allowlist path for embedded agent turns because `core-plugin-tools` remains slow even with `toolsAllow: []`.

## Best Possible Solution

Honor explicit empty cron/runtime tool allowlists before tool construction, using the existing `disableTools` short-circuit or equivalent runner semantics while leaving broader caching and lazy-loading work separate.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: cron preserves `toolsAllow: []`, forwards it to the embedded runner, and the runner treats the empty list as unrestricted after constructing tools. I did not run a live timing probe.

## Solution Assessment

Yes for a narrow first fix: distinguish omitted allowlists from explicit empty arrays before `createOpenClawCodingTools(...)` and short-circuit to no tools. Broader plugin factory caching should stay a follow-up.
