## Summary

Keep open. Current main has partial allowlist performance work, but it still treats `toolsAllow: []` as unrestricted and the public cron `agentTurn` schema does not expose a supported no-tools mode.

## What This Changes

Request a supported no-tools or early allowlist path for `agentTurn` so expensive core/plugin tool construction can be skipped when not needed.

## Best Possible Solution

Define the public `agentTurn` contract for no-tools runs, then thread that intent and narrow allowlists into tool construction before expensive core/plugin factories run.

## Reproduction Assessment

Yes for the `toolsAllow: []` semantics from source: current main treats an empty allowlist as unrestricted and still builds tools. No for the exact 30s timing split because I did not run a live timing probe.

## Solution Assessment

Unclear until maintainers choose the API contract. Reinterpreting `toolsAllow: []` is risky; adding an explicit public `disableTools` or `toolsMode: "none"` and pushing allowlist filtering earlier looks safer.
