## Summary

Keep open. Current main still preserves explicit empty cron `toolsAllow` lists but the embedded runner treats an empty allowlist as unrestricted and builds tools before filtering, so the no-tools `agentTurn` path is not implemented.

## What This Changes

The issue asks for `agentTurn` to support an explicit no-tools or early allowlist path so expensive core/plugin tool construction is skipped.

## Best Possible Solution

Honor the existing cron `toolsAllow: []` disable-all contract before `createOpenClawCodingTools(...)` runs, while leaving broader plugin factory caching or lazy-loading as separate performance follow-ups.

## Reproduction Assessment

Yes. Source inspection on current main shows cron preserves and forwards `toolsAllow: []`, while the embedded runner treats the empty list as unrestricted and constructs tools before filtering; I did not run a live timing probe.

## Solution Assessment

Yes for a narrow first fix. Map explicit empty cron/runtime allowlists to the existing `disableTools` short-circuit, or distinguish `undefined` from `[]` before construction; do not bundle broad tool caching into the same repair.
