Keep open. Current main still treats an explicit empty `toolsAllow` as unrestricted and constructs core/plugin tools before filtering, while merged #76773 only reduces a subset of optional media/PDF factory cost.

## What This Changes

Apply `agentTurn` allowlist or no-tools intent before expensive core/plugin tool construction, especially for explicit empty allowlists.

## Best Possible Solution

Honor explicit empty runtime/cron allowlists through the existing no-tools short-circuit before tool construction, and keep broader caching or lazy-loading improvements as separate follow-ups.

## Solution Assessment

Yes for the narrow repair: distinguish `undefined` from explicit `[]` or translate the empty list to the existing `disableTools` path before construction. #76773's optional-factory optimization is complementary, not a complete fix.
