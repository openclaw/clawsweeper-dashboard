## Summary

Keep open. Current main still builds and deploys each Discord account's native command set without deriving Discord picker visibility from per-channel bindings, while channel policy is enforced only after a slash interaction is submitted; the related open issues cover narrower deploy/cache/stale-command problems, not this full picker-visibility feature.

## What This Changes

The issue requests per-channel Discord slash-command picker visibility for multi-bot setups so each bot's commands are only visible in its assigned channels.

## Best Possible Solution

Keep this as an open Discord plugin feature until maintainers choose and document a supported per-account, per-channel native-command visibility model with targeted tests and live Discord proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure multiple Discord accounts in one guild with separate channel bindings and native commands enabled; current main registers each account's command set and only rejects disallowed channels after slash selection.

## Solution Assessment

Unclear. The requested UX is valid, but the best solution needs maintainer choice between Discord permission reconciliation, guild/channel-scoped registration, per-account command include/exclude, text-only controls, and documented platform limits.
