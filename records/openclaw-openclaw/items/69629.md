## Summary

Keep open. Current main and the latest release still register each Discord account's native command set at application/guild scope and enforce per-channel policy only after a slash interaction is submitted; the user request is valid but needs a Discord visibility/config design decision before implementation.

## What This Changes

Requests Discord slash commands for multiple bot accounts in the same guild to be hidden from channels where that account is not allowed.

## Best Possible Solution

Keep this as an open Discord plugin feature until maintainers choose and document a supported per-account, per-channel native-command visibility model with targeted tests and live Discord proof.

## Reproduction Assessment

Yes, by source inspection rather than live Discord: configure multiple Discord accounts in one guild with separate allowed channels and native commands enabled; current main publishes each account's native command set and rejects disallowed channels only after selection. Two users report the same real-world picker behavior.

## Solution Assessment

Unclear. The requested UX is valid, but the best implementation needs a maintainer choice between Discord permission reconciliation with the required OAuth contract, derived channel scoping from bindings, explicit native-command visibility config, or a documented text-only workaround.
