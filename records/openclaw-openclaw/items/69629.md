## Summary

Keep open. Current main still protects Discord slash-command execution after selection, but it does not provide per-channel command picker visibility for same-guild multi-bot setups; the newest user follow-up reinforces that Discord-side command permissions still leave the picker cluttered.

## What This Changes

Request per-channel Discord slash-command visibility for multi-account guild setups so each bot account's commands are hidden outside its configured channels.

## Best Possible Solution

Keep this as a Discord plugin feature request until maintainers choose the supported visibility model and operator setup for per-account, per-guild-channel native commands.

## Reproduction Assessment

Yes. Configure multiple Discord bot accounts in one guild with separate configured channels and native commands enabled; current source shows commands are registered for the account and only rejected after a wrong-channel interaction is submitted.

## Solution Assessment

Unclear. The requested UX is valid, but hiding Discord’s command picker likely needs a maintained Discord permission or registration strategy rather than the existing OpenClaw runtime authorization filter.
