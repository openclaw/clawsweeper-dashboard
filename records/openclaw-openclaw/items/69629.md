## Summary

Keep open. Current main still leaves Discord picker visibility independent of per-channel bindings, and the remaining work is a Discord plugin feature/design decision rather than a conservative cleanup close.

## What This Changes

The issue requests per-channel Discord slash-command visibility so each bot account's commands only appear in its assigned channels.

## Best Possible Solution

Keep this as an open Discord plugin feature until maintainers choose and document a supported per-account, per-channel native-command visibility model, with targeted tests and live Discord proof.

## Reproduction Assessment

Yes. Configure multiple Discord accounts in one guild with separate channel bindings and native commands enabled; source inspection shows current main registers each account's command set and only rejects disallowed channels after slash selection.

## Solution Assessment

Unclear. The requested UX is valid, but the narrow maintainable solution needs maintainer choice because Discord channel command permissions require a Bearer-token/user authorization model rather than the current bot-token deploy path.
