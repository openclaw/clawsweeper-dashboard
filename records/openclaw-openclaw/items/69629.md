## Summary

Keep open. Current main still publishes each Discord account's slash commands at application/guild scope and only enforces per-channel policy after a command is selected; two users corroborate the picker clutter, and the remaining work needs a Discord visibility/config design decision.

## What This Changes

Request per-channel slash-command visibility so only the Discord bot/account assigned to the current channel appears in multi-bot guild command pickers.

## Best Possible Solution

Design and document a supported Discord per-account, per-channel native-command visibility model, then add targeted tests and live Discord proof for the chosen behavior.

## Reproduction Assessment

Yes, source-reproducible rather than live-verified: configure multiple Discord accounts in one guild with separate allowed channels and native commands enabled; current main deploys each account's command set and rejects wrong-channel use only after selection.

## Solution Assessment

Unclear. The requested UX is valid, but the best implementation needs a maintainer choice between Discord permission reconciliation, deriving visibility from bindings/channel config, adding explicit native-command visibility config, or documenting a text-only workaround.
