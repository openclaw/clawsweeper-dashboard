## Summary

Keep open. Current main still enforces Discord native-command authorization after an interaction is submitted, but it does not hide same-guild multi-bot slash commands from the Discord picker by channel; solving that needs a maintained Discord permission/registration policy choice.

## What This Changes

Request per-channel Discord slash-command visibility so each bot account's commands only appear in the channels assigned to that account.

## Best Possible Solution

Keep this as an open Discord plugin feature request until maintainers choose a supported model for per-account, per-guild-channel native command registration or Discord permission reconciliation, with docs and tests covering multi-bot setups.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: configure multiple Discord accounts in one guild with separate channel bindings and native commands enabled; current main registers the account command set and only rejects disallowed channels after the slash interaction is submitted.

## Solution Assessment

Unclear. The requested UX is valid, but the best solution needs maintainer choice between Discord permission reconciliation, guild/channel-scoped registration, per-account command inclusion, text-only controls, or documented platform limits.
