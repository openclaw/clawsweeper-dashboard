## Summary

Keep open. Current main still exposes each enabled Discord account's native command set at application or guild scope and only rejects the wrong channel after slash-command selection; the second user report confirms this is still a real multi-bot UX problem that needs maintainer Discord visibility policy, not cleanup closure.

## What This Changes

Request per-channel Discord slash-command visibility so multi-account bots in one guild only show commands in their configured channels.

## Best Possible Solution

Adopt and document a supported Discord per-account, per-channel native-command visibility policy, with focused deploy/auth tests and live multi-account Discord proof for the chosen behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure multiple Discord accounts in one guild with separate allowed channels and native commands enabled; current main deploys account command sets at application/guild scope and rejects wrong-channel selections afterward.

## Solution Assessment

Unclear. The requested UX is valid, but the best implementation needs a maintainer choice between Discord permission reconciliation, deriving visibility from existing channel/binding config, adding explicit visibility config, or documenting a workaround.
