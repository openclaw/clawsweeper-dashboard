## Summary

Keep open. Current main still registers each Discord account's native command set without deriving Discord picker visibility from per-channel bindings, and it only enforces channel policy after a slash interaction is submitted; the corroborating comment reports the same multi-account UX problem.

## What This Changes

Request per-account, per-channel Discord slash-command picker visibility so multiple bot accounts in one guild only show commands in their assigned channels.

## Best Possible Solution

Keep this as an open Discord plugin feature until maintainers choose and document a supported per-account, per-channel native-command visibility model with targeted tests and live Discord proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure multiple Discord accounts in one guild with separate channel bindings and native commands enabled; current main registers each account's command set and only rejects disallowed channels after slash selection.

## Solution Assessment

Unclear. The Discord plugin is the right layer, but maintainers still need to choose between Discord permission reconciliation, binding-derived scopes, per-account native-command controls, text-only fallback, or documenting a platform limit.
