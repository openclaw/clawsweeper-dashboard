## Summary

Keep open. Current main still publishes each Discord account's native command set without deriving Discord picker visibility from per-channel bindings, while channel authorization is enforced only after the interaction is submitted; this remains a Discord plugin feature and policy decision.

## What This Changes

Request per-channel Discord slash-command picker visibility in same-guild multi-bot deployments so each bot's commands appear only in its configured channels.

## Best Possible Solution

Keep this as an open Discord plugin feature until maintainers choose and document a supported per-account, per-channel native-command visibility model with targeted tests and live Discord proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure multiple Discord accounts in one guild with separate channel bindings and native commands enabled; current main registers each account's command set and only rejects disallowed channels after slash selection.

## Solution Assessment

Unclear. The requested UX is valid, but the best solution needs maintainer choice between Discord permission reconciliation, guild/channel-scoped registration, per-account command inclusion or exclusion, text-only controls, and documented platform limits.
