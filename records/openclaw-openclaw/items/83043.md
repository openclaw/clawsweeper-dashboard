## Summary

Keep open for normal maintainer review: this is an active, focused performance PR with sufficient copied runtime proof and no blocking correctness or security finding from this read-only pass.

## What This Changes

The PR adds a private Map-backed cache for regexes used to extract id, parentId, type, and role from oversized transcript JSONL prefixes.

## Best Possible Solution

Land the private cache if maintainers accept the micro-performance win, keeping it local to session-utils and validating with focused session-utils checks.

## Reproduction Assessment

Yes for the performance target by source inspection: current main compiles fresh regexes in the private helpers and the oversized record path calls them for the four metadata fields. I did not rerun the benchmark in this read-only pass, but the PR body includes copied Node 22 benchmark output.

## Solution Assessment

Yes; caching private, non-global regexes behind the existing helpers is a narrow maintainable way to avoid repeated construction while preserving the current parsing contract. The cache should remain private and be gated by the existing session-utils behavior tests.
