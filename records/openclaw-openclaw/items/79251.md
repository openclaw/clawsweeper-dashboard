## Summary

Keep open: the focused display-name fix is well supported and the contributor has now supplied realistic gateway output, but the patch misses the server-side `sessions.list` search path, so newly visible account names would not be searchable.

## What This Changes

The PR adds a direct-session account-name fallback in `buildGatewaySessionRow` plus regression tests for Feishu-like direct keys and label/origin precedence.

## Best Possible Solution

Share the new direct-session account-name resolution with the `sessions.list` search filter, add focused search coverage, add the required changelog entry, and then review the linked issue at https://github.com/openclaw/openclaw/issues/79183 through this PR.

## Reproduction Assessment

Yes. Source inspection shows current main can persist Feishu direct origin labels as open ids with account metadata, and current row construction falls through to `origin.label`; the PR body also supplies realistic `sessions.list` output after the patch.

## Solution Assessment

No, not quite yet. The display fallback is the narrow maintainable direction, but it should also feed the pre-row-enrichment search helper so server-side search remains consistent with visible display names.
