## Summary

## What This Changes

The PR changes memory search sync resolution so unset `memorySearch.sync.watch` is disabled for default-purpose gateway managers while preserving explicit values and updating memory docs/tests.

## Best Possible Solution

Land a memory-only default change after preserving existing QMD status-borrow behavior and making the `sync.watch: true` upgrade path an explicit maintainer-owned compatibility decision.

## Reproduction Assessment

Yes. Source inspection shows current main resolves unset `sync.watch` to true and both builtin and QMD long-lived managers gate watcher allocation on that resolved setting; I did not run the watcher tests in this read-only review.

## Solution Assessment

No. The resolver layer is the right place for the default policy, but the PR should preserve QMD status reuse in gateway mode before maintainers decide whether to accept the compatibility change.
