## Summary

Keep open. Current main has manual `/new`, context-usage UI warnings, preflight compaction, and overflow recovery, but it does not detect topic changes or prompt/auto-switch sessions based on topic plus context pressure; this needs product and runtime-policy design rather than conservative cleanup.

## What This Changes

Request adds a smart reminder or automatic session switch when context use is high and the user appears to change topics.

## Best Possible Solution

Define an opt-in generic context-pressure reminder policy that reuses existing token data, asks before fresh-session creation on likely topic changes, and composes with `/new`, `/compact`, compaction, and channel session routing.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection shows the requested topic-change reminder/auto-switch behavior is not present on current main.

## Solution Assessment

Unclear. The user problem is valid, but the exact thresholds, classifier, opt-in surface, and auto-switch behavior need maintainer UX/runtime-policy decisions before implementation.
