## Summary

Keep open. Current main still has manual `/new`/`/reset`, daily/idle rollover, context usage notices, and compaction, but it does not implement topic-change detection, a high-context new-session reminder, or automatic session switching; the remaining behavior needs maintainer UX policy because automatic rollover can affect transcript continuity across UI and channel surfaces.

## What This Changes

Add a smart session-switch reminder when context usage is high and the user changes topics, with optional automatic new-session switching near exhaustion.

## Best Possible Solution

Define an opt-in session hygiene feature that uses fresh context snapshots, detects explicit or high-confidence topic changes, shows a non-destructive reminder first, and gates automatic rollover behind documented config.

## Reproduction Assessment

Not applicable as a bug reproduction. The absence of the requested feature is high-confidence from source and docs inspection: current high-context behavior warns/compacts, and session rollover is explicit, daily, or idle-policy based.

## Solution Assessment

Unclear. The reminder portion may be useful, but automatic rollover at 95% is not clearly the safest default; a maintainer-reviewed, prompt-first opt-in design is the narrower maintainable direction.
