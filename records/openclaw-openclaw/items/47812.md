## Summary

Keep open. Current main still provides manual reset/New Chat, daily/idle session rollover, and high-context compaction notices, but it does not implement topic-change detection, a high-context new-session reminder, or automatic session switching at the requested threshold. The remaining work needs maintainer UX/product policy because automatic rollover would affect transcript continuity across Control UI and message-channel surfaces.

## What This Changes

Add an intelligent session-switch reminder when context usage is high and the user changes topics, with automatic new-session switching near context exhaustion.

## Best Possible Solution

Define an opt-in session hygiene feature: use fresh context snapshots, detect explicit or high-confidence topic changes, show a non-destructive reminder first, and allow automatic rollover only behind documented config with safeguards for transcript continuity and channel/UI parity.

## Reproduction Assessment

Not applicable as a bug reproduction. The absence of the requested feature is high-confidence from source and docs inspection: current high-context behavior warns/compacts, and session rollover is explicit, daily, or idle-policy based.

## Solution Assessment

Unclear. The reminder part may be useful, but automatic rollover at 95% is not clearly the safest default; a maintainer-reviewed, opt-in prompt-first design is the narrower maintainable direction.
