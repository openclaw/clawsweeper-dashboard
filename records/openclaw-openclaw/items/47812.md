## Summary

Keep open. Current main has manual `/new`, a Control UI context-pressure notice with a Compact action, preflight/overflow compaction, and adjacent open trackers, but it does not detect topic changes or prompt/auto-switch sessions based on topic plus context usage; this is new cross-channel UX/runtime policy rather than cleanup-safe stale work.

## What This Changes

The issue requests smart reminders or automatic session switching when context usage is high and the user changes topics.

## Best Possible Solution

Define a generic, opt-in context-pressure reminder policy that can reuse existing token data, ask before fresh-session creation on likely topic changes, and compose with `/new`, `/compact`, compaction, and channel session routing without silent data loss.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection shows the requested topic-change reminder/auto-switch behavior is not currently present on main.

## Solution Assessment

Unclear. The user problem is valid, but the exact requested auto-switch thresholds and topic classifier need maintainer UX and runtime-policy decisions; a generic opt-in reminder is safer than channel-specific or unconditional auto-switching.
