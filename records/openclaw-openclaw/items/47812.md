## Summary

Keep open. Current main still has manual `/new`/`/reset`, daily/idle session rollover, and high-context compaction notices, but it does not implement topic-change detection, a context-driven new-session reminder, or automatic session switching; the remaining behavior needs maintainer UX/product policy.

## What This Changes

Request a smart session hygiene feature that prompts or switches sessions based on high context usage plus a detected topic change.

## Best Possible Solution

Define an opt-in session hygiene design: use current context snapshots, detect explicit or high-confidence topic changes, show a non-destructive reminder first, and gate any automatic rollover behind documented config with channel/UI parity.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection gives high confidence that the requested feature is absent: current behavior warns/compacts at high context and rolls sessions only via explicit, daily, or idle reset paths.

## Solution Assessment

Unclear. A reminder may be a good direction, but the requested automatic switch at 95% is not clearly the safest default; a maintainer-reviewed prompt-first, opt-in design is the narrower maintainable path.
