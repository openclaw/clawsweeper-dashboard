## Summary

Keep open. Current main has manual fresh-session controls, context usage warnings, Compact/preflight compaction, and Telegram DM auto-topic labeling, but it still lacks a generic topic-change detector or context-pressure reminder/auto-switch policy. The pure threshold rollover part overlaps the still-open https://github.com/openclaw/openclaw/issues/53321, while the topic-change reminder remains a separate product decision.

## What This Changes

The issue requests smart reminders or automatic session switching when high context usage coincides with a detected topic change.

## Best Possible Solution

Define an opt-in generic context-pressure session-transition policy that reuses existing token data, composes with https://github.com/openclaw/openclaw/issues/53321 for rollover, and separately specifies topic-change classification, prompt wording, thresholds, and continuity handling.
