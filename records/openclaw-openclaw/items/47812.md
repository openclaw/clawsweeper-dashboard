## Summary

Keep open. Current main has context-usage display, manual compact/reset paths, and documented auto-compaction, but it does not implement semantic topic-change detection or an automatic/prompted session switch policy; the request is a new product feature that needs maintainer UX and policy direction.

## What This Changes

Add smart session switch reminders and optional automatic new-session behavior based on high context usage plus topic changes.

## Best Possible Solution

Design an opt-in session freshness policy that reuses existing context-usage telemetry, adds clear user consent for prompts, and avoids silent session resets until maintainers settle the UX and configuration contract.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source review shows current main has related context usage and compaction primitives, but no topic-change-triggered session switch behavior to reproduce.

## Solution Assessment

Unclear. The requested direction is useful, but fixed thresholds and automatic switching are not yet clearly the best maintainable solution without maintainer decisions on opt-in policy, UX, and whether semantic detection belongs in UI, core session policy, or an extension surface.
