## Summary

Keep open. Current main still lacks the requested topic-change detection, high-context new-session reminder, and automatic session rollover; the existing behavior covers manual reset, daily/idle rollover, and compaction. The related topic-labeling report is adjacent, not a replacement for this session-switch UX request, and the remaining work needs maintainer product policy before an automated fix is safe.

## What This Changes

Request to add topic-aware session hygiene that prompts for a new session at high context usage on topic changes and auto-starts a new session at a higher context threshold.

## Best Possible Solution

Define an opt-in session hygiene feature across UI and message-channel surfaces: use fresh context snapshots, detect explicit or high-confidence topic changes, prompt before rollover, make thresholds configurable, and gate any automatic new-session behavior behind durable reset safeguards.

## Reproduction Assessment

Not applicable as a bug reproduction. The absence of the requested feature is high-confidence from source and docs inspection: high-context paths warn or compact, while session rollover is explicit, daily, idle, or configured freshness policy only.

## Solution Assessment

Unclear. The reminder direction may be useful, but the proposed 95% automatic switch is not clearly the safest default; a maintainer-reviewed, opt-in, prompt-first design is the narrower maintainable path.
