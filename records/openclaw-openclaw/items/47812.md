## Summary

Current main monitors context pressure, flushes memory before compaction, and offers manual fresh-session recovery, but it does not provide topic-aware reminders or automatic session rotation. The overlapping auto-split report covers the context-limit half but does not preserve this issue's distinct topic-change UX, so this needs an explicit maintainer product decision rather than cleanup closure.

## What This Changes

Add a prompt to start a fresh session when high context usage coincides with a topic change, and automatically switch sessions near context exhaustion.

## Best Possible Solution

Define one canonical session-pressure design around the existing context-monitoring boundary: preserve explicit user control by default, specify continuity and active-run behavior, and incorporate the topic-change prompt into https://github.com/openclaw/openclaw/issues/53321 only if maintainers decide the two UX paths should share ownership.

## Solution Assessment

No, not as an isolated keyword/LLM detector plus unconditional threshold switch.
