- `impact:message-loss`: The reported and source-backed failure mode suppresses the final assistant reply that should be delivered to the channel.

## Summary

Keep open: current main and the v2026.6.1 release still contain the reported Codex projector condition, and source inspection shows that a synthesized missing tool result becomes a prompt error before the final assistant reply can be delivered.

## What This Changes

The issue reports a regression where completed embedded Codex turns with orphan native tool calls synthesize failed tool results and then suppress the final assistant reply from channel delivery.

## Best Possible Solution

Keep the synthetic failed `tool.result` records for transcript and trajectory consistency, but treat a completed Codex turn with final assistant text as deliverable/degraded instead of prompt-failed.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: start a native tool item, complete the Codex turn with an `agentMessage`, and `buildResult()` currently sets a missing-tool-result prompt error. I did not run a live Discord/Codex scenario because this review was read-only.

## Solution Assessment

Yes, the best fix is to keep synthesizing failed tool results but stop using completed final assistant text as a reason to fail the whole turn. Existing payload behavior already prefers recovered assistant text after tool activity, so the fix belongs at the projector/run-result boundary rather than in channel-specific delivery.
