## What This Changes

Fix subagent runtime thinking validation so explicit non-off thinking on supported OpenAI/Codex GPT-5 models is accepted instead of collapsed to off-only catalog metadata.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84902 or a successor with shared resolver coverage plus live or runtime-probe proof, then close this issue with the merged commit and release provenance.

## Reproduction Assessment

Yes at source level, but I did not run a live subagent. A `reasoning: false` catalog entry reaches `resolveThinkingProfile`, returns off-only before provider policy, and `get-reply-run` emits the reported unsupported-thinking message while `subagent-spawn` forwards the explicit thinking override.

## Solution Assessment

Yes, the best direction is a shared resolver fix that lets explicitly authoritative provider thinking profiles override stale catalog false for OpenAI/Codex while preserving off-only behavior for non-reasoning models/providers.
