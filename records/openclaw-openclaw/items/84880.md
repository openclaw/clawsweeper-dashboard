## What This Changes

The issue reports that explicit non-off `thinking` passed to `sessions_spawn` for OpenAI/Codex GPT-5 subagents is rejected as off-only in v2026.5.19 and v2026.5.20.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84902 or a successor that makes provider thinking profiles authoritative where appropriate, then close this issue with merged-commit and release provenance.

## Reproduction Assessment

Yes at source level. A stale `reasoning: false` catalog row reaches `resolveThinkingProfile`, returns off-only before provider policy, and the subagent path forwards the explicit thinking override into the rejecting reply/run validation path.

## Solution Assessment

Yes, the best direction is a shared resolver fix that lets explicitly authoritative provider thinking profiles override stale catalog false for OpenAI/Codex while preserving off-only behavior for genuinely non-reasoning models/providers.
