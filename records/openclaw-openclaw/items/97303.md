## Summary

Keep open: this PR is not merge-ready because the new projection can hide standalone delivered-message mirrors that current main intentionally preserves, and the external contributor proof is still only test output rather than real or user-like session-history behavior.

## What This Changes

The PR routes sessions_history, sessions_list message hydration, and last-message preview extraction through a new visible-chat transcript projection that filters tool artifacts, delivery mirrors, and terminal send confirmations while preserving raw includeTools output.

## Best Possible Solution

Keep the shared visible projection, but preserve standalone delivery mirrors unless a synthetic message-tool mirror replaces them, cover bounded history/list/preview cases, and add redacted real session-history proof.

## Reproduction Assessment

Yes, source-reproducible. Current main keeps standalone delivery-mirror rows visible, while the PR helper filters them after projection, so a bounded history/list/preview window can lose delivered text without a live checkout mutation.

## Solution Assessment

No. The shared projection layer is a reasonable owner boundary, but this implementation filters raw delivery mirrors unconditionally; the safer fix is to preserve unmatched mirrors and only hide artifacts that have a synthetic visible replacement.
