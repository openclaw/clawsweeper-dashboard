## Summary

Keep open: this PR moves visible session transcript projection into a shared helper, but current head still drops any raw delivery-mirror row whose text matches any synthetic mirror in the same projected window, so separate repeated delivered replies can disappear from visible history/list/preview surfaces; contributor real-behavior proof is also still insufficient.

## What This Changes

The PR adds a shared visible-chat transcript projection for sessions_history, sessions_list hydration, and last-message previews, with tests for hiding tool artifacts while preserving raw includeTools output.

## Best Possible Solution

Keep the shared visible projection, but make delivery-mirror suppression occurrence-aware, cover repeated identical sends across history/list/preview, and add redacted real or user-like before/after output.

## Reproduction Assessment

Yes. Source inspection of PR head shows a constructed window with a standalone delivery-mirror row and a separate synthetic mirror using the same text will drop the standalone row from all new visible projection callers.

## Solution Assessment

No. The shared projection layer is the right boundary, but the current Set-based text match is too broad; the fix should pair the exact raw mirror occurrence or honor display-hidden rows from projectChatDisplayMessages only.
