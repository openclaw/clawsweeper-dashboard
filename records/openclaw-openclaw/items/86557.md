## What This Changes

Adds a security audit finding and regression coverage for restrictive Claude CLI permission modes being overridden by OpenClaw YOLO exec policy, plus a changelog entry.

## Best Possible Solution

Make the audit check reuse or exactly mirror the Claude live effective exec policy, including sandbox-host defaults and approval overrides, then keep the focused audit warning, tests, and changelog.

## Reproduction Assessment

Yes, source-level: on the PR head, tools.exec.host="sandbox" with no explicit security/ask still falls through to the new full/off defaults in the audit collector. Current main's effective exec resolver instead defaults sandbox effective hosts to security=deny, so the warning would be false.

## Solution Assessment

No: the direction is good, but the audit implementation should share or exactly mirror the existing effective exec policy resolver instead of approximating YOLO from raw config fields.
