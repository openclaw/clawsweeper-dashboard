## Summary

Keep open. Current main still constructs the ACP bridge as a CLI-mode `GatewayClient` with persisted device identity and default admin scope, while the existing local shared-auth identity bypass is limited to backend calls and native approval clients; #74472 and the open #75820 do not cover the ACP bridge path.

## What This Changes

The issue asks for ACP runtime and direct `acpx openclaw` launches to stop looping on scope-upgrade approval after 2026.4.26 and initialize after one approval.

## Best Possible Solution

Add a constrained ACP bridge helper that omits the persisted device identity only for trusted loopback token/password ACP connections, with regression tests preserving normal remote and no-shared-auth pairing behavior.

## Reproduction Assessment

Yes. A high-confidence source-backed reproduction is to seed a lower-scope paired CLI identity, run `openclaw acp` or `acpx openclaw` against a loopback token/password Gateway, and observe the ACP bridge sending the persisted CLI identity with default admin scope before initialize can complete.

## Solution Assessment

Yes. Mirroring the native approval/local backend pattern for the ACP bridge is the narrow maintainable fix; broad auto-approval or global pairing-policy changes would be riskier and would not match the existing dependency boundaries.

## Repair Work Prompt

Fix the ACP scope-approval loop reported at https://github.com/openclaw/openclaw/issues/74650. Current main constructs the ACP bridge GatewayClient in `src/acp/server.ts` as a CLI client without `deviceIdentity: null`, so local loopback token/password ACP connections can reuse a stale lower-scope CLI identity and request `operator.admin` before initialize. Add a constrained helper for ACP that omits persisted device identity only when the resolved Gateway URL is loopback and the bootstrap auth has token or password, matching the native approvals safety boundary. Do not change remote ACP, unauthenticated ACP, backend pairing policy, global scope defaults, or broad auto-approval behavior. Add regression coverage in `src/acp/server.startup.test.ts` for local shared-auth omission plus remote/no-shared-auth preservation, and update adjacent tests only as needed. Add a single-line user-facing `CHANGELOG.md` fix entry with contributor credit if a human reporter is credited. Related context: #74484 is the broader scope-deadlock report, #74472 fixed only native approval clients, and #75820 is an open CLI fallback PR that does not fix ACP bridge identity selection.
