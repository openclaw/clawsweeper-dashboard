## Summary

Close as implemented on current main: DeepSeek's lightweight provider-policy surface now exposes the same V4 thinking profile as the runtime hook, session rows feed those levels to Control UI, and regression coverage plus the Unreleased changelog entry cover the path.

## What This Changes

The issue asks for the Control UI `/think` argument picker for native DeepSeek V4 models to show `xhigh` and `max` instead of only the base five thinking levels.

## Best Possible Solution

Keep the provider-owned runtime and lightweight policy surfaces sharing the same DeepSeek V4 thinking helper and ship the Unreleased fix in the next release.

## Reproduction Assessment

Yes. The original gap is source-reproducible from the fallback path to the lightweight provider-policy artifact; current main no longer reproduces it because DeepSeek now exports that policy hook and tests assert `xhigh|max`.

## Solution Assessment

Yes. The current implementation fixes the provider-owned policy surface instead of hard-coding DeepSeek behavior in the Control UI, matching the existing manifest/runtime-provider design.
