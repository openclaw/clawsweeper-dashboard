## Summary

Current main already exposes `xhigh` and `max` in the Control UI `/think` argument picker and the native DeepSeek V4 runtime profile, so the reported 2026.5.2 behavior is obsolete on main.

## What This Changes

The issue asks for the Control UI `/think` argument picker to include `xhigh` and `max` for native DeepSeek V4 models.

## Best Possible Solution

Keep the current main implementation and let the next release carry the already-present `/think` picker and DeepSeek V4 profile behavior to users on 2026.5.2.

## Reproduction Assessment

No for current main: source inspection shows the Control UI `/think` argument menu now receives `xhigh` and `max` from browser-local command choices. The original 2026.5.2 report is credible, but the current-main picker path no longer matches the reported five-option state.

## Solution Assessment

Yes: current main fixes the observable picker omission without adding a new product mode, and the DeepSeek provider runtime profile already accepts the same levels. The remaining action is release propagation rather than another source change for this issue.
