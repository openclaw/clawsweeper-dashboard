Expose `xhigh` and `max` for native DeepSeek V4 `/think` picker options by wiring DeepSeek's lightweight provider-policy surface to its V4 thinking profile.

## Best Possible Solution

Keep the shared DeepSeek V4 thinking helper and lightweight policy export on main, then reconcile the now-superseded open fix PRs against this landed implementation.

## Solution Assessment

Yes: sharing the DeepSeek V4 profile helper between the runtime provider entry and lightweight policy artifact is the narrow maintainable fix, and it avoids adding DeepSeek-specific logic to the Control UI picker.
