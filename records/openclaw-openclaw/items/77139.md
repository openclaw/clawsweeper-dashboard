## Summary

Current main now implements the missing DeepSeek lightweight provider-policy thinking hook, so the Control UI path can surface `xhigh` and `max` for native DeepSeek V4 models even when the runtime plugin registry is not active.

## What This Changes

The issue asks for the Control UI `/think` picker to expose `xhigh` and `max` for native `deepseek/deepseek-v4-pro` and `deepseek/deepseek-v4-flash` sessions.

## Best Possible Solution

Keep the current shared DeepSeek V4 profile wired through both runtime and lightweight policy surfaces, and handle the duplicate open fix PRs as superseded by main.

## Reproduction Assessment

No for current main: source inspection shows the affected fallback path now reaches DeepSeek's lightweight `resolveThinkingProfile`, and regression tests cover the V4 `xhigh`/`max` levels. The original 2026.5.2 behavior was source-reproducible from the missing policy export described in the report.

## Solution Assessment

Yes: the current fix is the narrow owner-module solution, sharing the DeepSeek V4 profile between the runtime plugin and lightweight provider-policy artifact instead of adding DeepSeek-specific UI logic.
