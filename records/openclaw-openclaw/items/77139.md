## Summary

Close as implemented on current main: DeepSeek now exposes the V4 `xhigh` and `max` profile through the lightweight provider-policy surface used by the fallback thinking resolver, with focused regression coverage. The fix is on main in c36f8f1e390fa149dcf5dfddf334de054cd364b1 but is not present in v2026.5.3.

## What This Changes

The issue requests that the Control UI `/think` picker expose `xhigh` and `max` for native DeepSeek V4 models when it resolves thinking options through the bundled provider policy surface.

## Best Possible Solution

Keep the shared DeepSeek V4 thinking helper wired through both runtime and lightweight policy surfaces, and supersede any stale open PRs that duplicate c36f8f1e390fa149dcf5dfddf334de054cd364b1.

## Reproduction Assessment

Yes for the reported released/source path: the fallback resolver depends on the bundled policy surface, and older DeepSeek policy code lacked `resolveThinkingProfile`. Current main no longer reproduces because the policy surface now returns the seven-level V4 profile.

## Solution Assessment

Yes: the main-branch fix is the narrow maintainable solution because it mirrors the existing DeepSeek runtime profile into the lightweight provider-policy artifact without adding UI-specific DeepSeek logic.
