## Summary

Keep open: current main still resolves the Discord route/default owner before any cross-bot target selection, and mention/reply detection is scoped to the running handler's bot ID. The related PRs #47393 and #67460 are still open in the provided context, so neither has shipped the requested precedence or default-owner suppression.

## What This Changes

The issue requests deterministic Discord multi-bot routing where explicit bot mentions win over reply targets, which win over the per-channel default owner, with the default owner suppressed for targeted messages.

## Best Possible Solution

Land one consolidated Discord-owned routing fix that maps configured bot user IDs to accounts/agents before default routing and suppresses the default owner whenever a mention or reply target selects another configured bot.

## Reproduction Assessment

Yes, at source level. GTZhou provided concrete guild-channel mention/reply/reply-plus-mention steps, and current main still derives route/default ownership before target signals while comparing mentions and replies only to the running bot ID.

## Solution Assessment

Yes. The requested precedence is the narrow maintainable direction, but it should be reconciled with #47393 and #67460 so route ownership, cross-bot mention selection, reply-target selection, and default-owner suppression land coherently.
