## Summary

Keep open: current main still resolves the Discord route/default owner before cross-bot target selection, and explicit mention/reply handling is scoped to the running handler's bot ID. Related PRs #47393 and #67460 are still open, so they do not yet supersede or implement the requested precedence.

## What This Changes

Request deterministic Discord routing where explicit bot mentions win first, replied-to bot messages win second, and the channel default owner only responds when no targeted signal exists.

## Best Possible Solution

Implement Discord-owned target selection before shared default routing, mapping configured Discord bot user IDs to OpenClaw accounts/agents and suppressing the default owner whenever an explicit mention or reply target selects another configured bot.

## Reproduction Assessment

Yes. GTZhou provided concrete guild-channel steps for mention, reply, and reply-plus-mention cases, and current-main source inspection shows reply and explicit mention checks remain scoped to the running bot ID.

## Solution Assessment

Yes. The requested precedence is the narrow maintainable fix direction, but it should be reconciled with #47393 and #67460 so owner routing, reply-target selection, and default-owner suppression land coherently.
