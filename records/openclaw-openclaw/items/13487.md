## Summary

Keep open: current main still lacks cross-bot target selection before default routing, and the Discord mention/reply checks remain scoped to only the running handler's bot user ID.

## What This Changes

Request deterministic Discord multi-bot routing where explicit bot mentions win over reply targets, reply targets win over the channel default owner, and targeted messages suppress the default owner.

## Best Possible Solution

Implement Discord-owned target selection before shared default routing, with a small generic suppression seam only if needed to carry targeted-other-bot state downstream.

## Reproduction Assessment

Yes. GTZhou provided concrete guild-channel steps for mention, reply, and reply-plus-mention cases, and current-main source inspection shows the same failure path without needing to mutate the checkout.

## Solution Assessment

Yes. The requested precedence is the narrow maintainable direction, but it should be reconciled with the overlapping open PRs so the Discord plugin and shared mention-gating pieces land coherently.
