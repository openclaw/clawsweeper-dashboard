## Summary

Keep open: current main still resolves the Discord route/default owner before cross-bot target selection, and mention/reply detection is scoped to the running handler's bot ID. The related open PRs are partial/overlapping work, not shipped implementation of this issue's requested precedence and suppression.

## What This Changes

Request deterministic Discord routing precedence: explicit bot mention, then reply target, then per-channel default owner, suppressing the default owner whenever a targeted signal exists.

## Best Possible Solution

Land one consolidated Discord-owned routing fix that maps configured Discord bot user IDs to accounts/agents before default routing, selects mention before reply target before default owner, and suppresses the default owner whenever a targeted Discord signal exists.

## Reproduction Assessment

Yes, at source level. GTZhou provided concrete guild-channel mention/reply/reply-plus-mention steps, and current main still resolves route/default ownership before target signals while comparing mentions and replies only to the running bot ID.

## Solution Assessment

Yes, the requested precedence is the narrow maintainable direction. It should be reconciled with #47393 and #67460 so route ownership, cross-bot mention selection, reply-target selection, and default-owner suppression land coherently.
