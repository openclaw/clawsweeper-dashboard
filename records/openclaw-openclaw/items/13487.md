## What This Changes

The issue requests deterministic Discord multi-bot routing where explicit native mentions outrank reply targets, and both suppress the channel default owner.

## Best Possible Solution

Land one Discord-owned routing change that maps native mention/reference bot IDs to configured OpenClaw accounts before default-owner routing, suppresses non-target default responders, and reconciles the open partial PRs.

## Reproduction Assessment

Yes, at source level. Current main resolves Discord routing before cross-bot mention/reference target state and only compares reply metadata to the running bot ID, so targeted alternate bots cannot suppress the default owner.

## Solution Assessment

Yes for the requested end state, but the implementation path needs maintainer coordination. The narrow direction is a Discord-owned target-selection step before shared default routing, reconciled with the open related PRs.
