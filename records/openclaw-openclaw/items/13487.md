## Summary

Keep open: current main still lacks Discord cross-bot target selection and default-owner suppression for explicit bot mentions or reply targets, and the related PRs are open partial work rather than a shipped resolution.

## What This Changes

The issue requests deterministic Discord multi-bot routing where explicit bot mentions win over reply targets, and both suppress the per-channel default owner.

## Best Possible Solution

Land one consolidated Discord-owned routing fix that maps configured Discord bot user IDs to accounts/agents before default routing, selects explicit mentions before reply targets before default owner, and suppresses the default owner whenever a targeted Discord signal exists.

## Reproduction Assessment

Yes, at source level. GTZhou provided concrete guild-channel mention/reply/reply-plus-mention steps, and current main still resolves route/default ownership before target signals while comparing mentions and replies only to the running bot ID.

## Solution Assessment

Yes, the requested precedence is the narrow maintainable direction. It should be reconciled with #47393 and #67460 so route ownership, cross-bot mention selection, reply-target selection, and default-owner suppression land coherently.
