## Summary

Keep open: current main still lacks the requested Discord cross-bot target precedence and default-owner suppression, and the remaining behavior overlaps open partial work plus explicit product-review labels.

## What This Changes

Requested deterministic Discord multi-bot routing precedence: explicit mentions first, reply target second, and default owner only when no targeted signal applies.

## Best Possible Solution

Add one Discord-owned target-selection layer that maps native mentioned bot IDs and referenced-message bot author IDs to configured OpenClaw accounts before default-owner routing, then prove default-owner suppression with focused preflight and routing tests.
