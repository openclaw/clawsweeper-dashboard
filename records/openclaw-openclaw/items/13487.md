## Label Justifications

- `P1`: The missing precedence breaks active multi-bot Discord workflows by producing no intended reply, duplicate replies, or wrong-agent handling.
- `impact:message-loss`: A reply or explicit cross-bot mention can fail to reach the intended responder or can produce an unwanted default-owner response.
- `impact:session-state`: Fallback routing can associate targeted input with the default agent's conversation instead of the intended bot's session.

## Summary

Current main does not implement the requested cross-bot precedence: Discord resolves the ordinary route before deriving mention/reply facts, compares both signals only with the running bot ID, and shared gating has no targeted-other-bot suppression result. The open related PR is partial and conflicting, so this issue remains the broader canonical product decision.

## What This Changes

The issue requests Discord routing precedence of explicit bot mention, then replied-to bot, then channel default owner, with the default owner suppressed for targeted messages.

## Best Possible Solution

Create one Discord-owned target-selection path that maps configured bot user IDs to accounts or agents, applies explicit-mention then referenced-author precedence, and invokes canonical routing/default ownership only when no target exists.

## Reproduction Assessment

Yes at source level. Current preflight routes before target analysis, exposes only the running bot ID, and cannot map a mention or referenced author to another configured bot; no fresh live Discord run was performed in this read-only review.

## Solution Assessment

Yes for the architectural direction: resolve Discord-native target identity inside the Discord plugin before shared routing. The exact precedence over explicit bindings and existing owner behavior still requires maintainer approval.
