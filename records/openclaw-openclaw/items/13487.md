## Summary

Keep open: current main still lacks Discord cross-bot target selection and default-owner suppression for explicit bot mentions or reply targets, while the related pull requests remain open partial work rather than a shipped fix.

## What This Changes

The issue requests deterministic Discord multi-bot routing where explicit bot mentions win first, reply targets win second, and the channel default owner only handles untargeted messages.

## Best Possible Solution

Land one Discord-owned routing fix that maps configured Discord bot user IDs to accounts or agents before default routing, applies mention > reply-target > default-owner precedence, and proves default-owner suppression with focused tests.

## Reproduction Assessment

Yes, at source level rather than by a new live run. The issue discussion gives concrete multi-bot Discord guild steps, and current main still resolves default routing before cross-bot target metadata while only comparing mentions and replies to the running bot ID.

## Solution Assessment

Yes, the requested precedence is the narrow maintainable direction for the documented multi-bot Discord surface. It should be reconciled with https://github.com/openclaw/openclaw/pull/47393 and https://github.com/openclaw/openclaw/pull/67460 so route ownership, reply targeting, and default-owner suppression land coherently.
