## Label Justifications

- `P1`: The issue affects live multi-bot Discord routing and can cause missing, duplicate, or wrong-agent responses in channel workflows.
- `impact:message-loss`: Targeted Discord messages can be consumed by the default owner or fail to reach the intended bot response path.
- `impact:session-state`: Routing a targeted Discord message to the default owner can attach the turn to the wrong agent session and conversation state.

## Summary

Keep open: current main and the latest release still choose Discord routing before cross-bot mention or reply-target selection, compare those signals only to the running bot ID, and do not have a default-owner suppression result for another targeted bot.

## What This Changes

The issue asks for deterministic Discord multi-bot routing where explicit mentions win first, reply targets win second, and the channel default owner responds only when no target signal exists.

## Best Possible Solution

Confirm the Discord active-target contract, then implement target selection inside the Discord plugin before default routing with explicit mentioned bot accounts first, referenced bot author second, and the default owner only when no target signal exists.

## Reproduction Assessment

Yes, at source level: current main resolves Discord routing before mention/reply target selection and compares mention/reply metadata only to the current handler botUserId. I did not run a live Discord multi-bot replay in this read-only review.

## Solution Assessment

Unclear until maintainers confirm the product contract; the requested precedence is plausible, but generic mention suppression alone is not the best fix because Discord needs a configured bot/account/agent target map before default routing.
