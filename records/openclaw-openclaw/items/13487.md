## Summary

Keep open: current main and the latest release still lack the requested Discord cross-bot mention/reply target selection, and the related implementation PRs are closed unmerged or only cover adjacent slices.

## What This Changes

The issue requests Discord single-target routing precedence: explicit bot mention first, replied-to bot second, and default owner only when no targeted signal exists.

## Best Possible Solution

Define Discord-owned target identity selection before ordinary routing: configured bot mentions first, replied-to bot author second, then existing default/binding routing, with tests proving non-target/default-owner suppression.

## Reproduction Assessment

Yes, at source level with high confidence: current main routes before target selection and compares mention/reply metadata only against the running bot ID. I did not run a live Discord multi-bot reproduction in this read-only review.

## Solution Assessment

Yes for the requested end state, but not as a generic suppression flag alone. The maintainable path is a Discord-owned target-selection step before shared binding/default routing, with maintainer-approved compatibility semantics.
