## Summary

Keep open: current main still lacks a Discord target-selection step that maps explicit mentions or reply targets across configured bot accounts before default-owner routing, and related partial PRs remain open rather than merged.

## What This Changes

The issue requests strict Discord multi-bot routing precedence: explicit bot mention, then replied-to bot, then per-channel default owner, with default-owner suppression for targeted messages.

## Best Possible Solution

Land one Discord-owned routing change that computes eligible bot targets from native mention/reference metadata before default routing, applies the requested precedence, and adds focused tests plus live Discord proof for default-owner suppression.

## Reproduction Assessment

Yes, at source level. Current main resolves Discord routing before cross-bot mention/reference target state and only compares reply metadata to the running bot ID, so a targeted alternate bot cannot suppress the default owner.

## Solution Assessment

Yes for the end state, but implementation needs maintainer coordination. The narrow maintainable direction is a Discord-owned target-selection step that precedes shared default routing and is reconciled with the open partial PRs.
