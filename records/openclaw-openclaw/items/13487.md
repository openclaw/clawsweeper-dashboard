## Label Justifications

- `P1`: The issue affects live multi-bot Discord channel routing and can make the intended bot fail to answer or the default owner answer instead.
- `impact:message-loss`: Targeted Discord messages can be dropped, consumed by the wrong default owner, or produce duplicate responses.
- `impact:session-state`: Wrong-agent Discord routing can attach the turn to the wrong agent session and conversation state.

## Summary

Keep open: current main and v2026.6.8 still resolve Discord routing before cross-bot mention or reply-target selection, compare targeting metadata only with the running bot user ID, and have no default-owner suppression result for targeted multi-bot messages.

## What This Changes

The issue asks Discord multi-bot routing to prefer explicit bot mentions, then replied-to bot authors, and only fall back to the per-channel default owner when no targeted bot signal exists.

## Best Possible Solution

Confirm the Discord multi-bot target policy, then implement target selection in the Discord plugin before default routing with a configured bot-user-id to account/agent map, mention-first and reply-target-second precedence, and suppression tests for default-owner cases.

## Reproduction Assessment

Yes. Source inspection shows current main resolves the Discord route before cross-bot target selection and compares mention/reply metadata only to the running botUserId; I did not run a live Discord multi-bot scenario in this read-only pass.

## Solution Assessment

Unclear until maintainer product direction is confirmed. The requested precedence is plausible, but the durable fix needs an owner decision on whether the target map and suppression state live in Discord preflight, shared routing, or a channel contract.
