## Summary

Keep open: current main and the latest release still resolve Discord routing before cross-bot mention or reply-target selection, and the remaining fix needs maintainer product review because it changes multi-bot responder precedence and overlaps partial open work.

## What This Changes

The issue requests deterministic Discord multi-bot routing precedence: explicit bot mention first, replied-to bot second, and per-channel default owner only when no targeted signal exists.

## Best Possible Solution

Add one Discord-owned target-selection layer that maps native mentioned bot IDs and referenced-message bot author IDs to configured OpenClaw accounts before default routing, then prove default-owner suppression with focused preflight and routing tests.

## Reproduction Assessment

Yes, at source level. Current main resolves the Discord route before cross-bot mention/reference targeting and only compares reply metadata to the running botUserId; the discussion also includes a relevant live multi-bot Discord reproduction.

## Solution Assessment

Yes for the requested end state, but not as another independent partial repair. The maintainable path is a consolidated Discord target-selection step reconciled with related open work and product-review labels.
