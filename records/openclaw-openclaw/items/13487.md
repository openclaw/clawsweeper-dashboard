## Summary

Keep open: current main still lacks Discord cross-bot target selection and default-owner suppression for explicit bot mentions or reply targets, and the linked pull requests remain open partial work rather than a shipped fix.

## What This Changes

The issue requests deterministic Discord routing where explicit bot mentions outrank reply targets, reply targets outrank the channel default owner, and the default owner is suppressed whenever a targeted signal exists.

## Best Possible Solution

Land one Discord-owned routing fix that maps configured Discord bot user IDs to accounts or agents before default routing, applies mention then reply-target then default-owner precedence, and proves default-owner suppression with focused tests.

## Reproduction Assessment

Yes, at source level. The issue discussion gives concrete multi-bot Discord guild-channel steps, and current main still resolves route/default ownership before targeted cross-bot metadata while only comparing mentions and replies to the running bot ID.

## Solution Assessment

Yes, the requested precedence is the narrow maintainable direction. It should be reconciled with https://github.com/openclaw/openclaw/pull/47393 and https://github.com/openclaw/openclaw/pull/67460 so route ownership, cross-bot target selection, reply targeting, and default-owner suppression land coherently.
