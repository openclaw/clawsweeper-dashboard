## Label Justifications

- `P1`: The issue affects real multi-bot Discord routing and can cause targeted messages to be handled by the wrong responder or receive no visible reply.
- `impact:message-loss`: The reported Discord reply and mention cases can produce no visible reply, duplicate responses, or wrong-agent responses in the channel.
- `impact:session-state`: Routing a targeted Discord message to the default owner can attach the turn to the wrong agent or session state.

## Summary

Keep open: current main and v2026.5.22 still resolve Discord routing before cross-bot mention or reply-target selection, and the remaining fix needs maintainer product review because it overlaps an open partial PR and changes multi-bot routing semantics.

## What This Changes

The issue requests deterministic Discord multi-bot routing where explicit native mentions win over reply-target bot ownership, and both suppress the per-channel default owner.

## Best Possible Solution

Add one Discord-owned target-selection layer that maps native mentioned bot IDs and referenced-message bot author IDs to configured OpenClaw accounts before default-owner routing, then prove default-owner suppression with focused preflight and routing tests.

## Reproduction Assessment

Yes, at source level. Current main resolves the Discord route before cross-bot mention/reference targeting and only compares reply metadata to the running botUserId; the discussion also includes a relevant live multi-bot Discord reproduction.

## Solution Assessment

Yes for the requested end state, but not as another independent partial repair. The maintainable path is a consolidated Discord target-selection step reconciled with the open related PR and product-review labels.
