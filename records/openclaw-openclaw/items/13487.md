## Summary

Keep open: current main still lacks a complete Discord target-selection step that maps explicit mentions or reply targets across configured bot accounts before default-owner routing, and related partial PRs remain open rather than merged.

## What This Changes

Request deterministic Discord multi-bot routing where explicit bot mentions outrank reply-target routing, reply targets outrank the channel default owner, and targeted cases suppress default-owner responses.

## Best Possible Solution

Land one Discord-owned routing change that computes eligible bot targets from Discord mention/reference metadata before default routing, applies the requested precedence, and adds focused tests for default-owner suppression.

## Reproduction Assessment

Yes, at source level. The issue and comments describe a concrete multi-bot Discord guild scenario, and current main still compares mention and reply metadata only to the running bot ID after route resolution.

## Solution Assessment

Yes, the requested precedence is the narrow maintainable direction for this multi-bot Discord surface, but it needs maintainer coordination with the linked open PRs before implementation lands.
