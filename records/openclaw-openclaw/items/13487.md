## Label Justifications

- `P1`: The source-confirmed gap breaks a real multi-bot Discord workflow and can produce missing, wrong, or duplicate responses.
- `impact:message-loss`: The intended mentioned or replied-to bot can be skipped or never selected, while the default owner may also respond.
- `impact:session-state`: Targeted messages can remain attached to the route and session of the preselected default agent.

## Summary

Current main still lacks cross-bot target selection for the requested precedence, while the related open PR handles only explicit-mention suppression and does not cover reply targeting. This issue should remain open for a maintainer decision on the complete Discord routing contract.

## What This Changes

Route Discord guild messages to explicitly mentioned configured bots first, otherwise the replied-to configured bot, otherwise only the channel default owner, suppressing all non-target responders.

## Best Possible Solution

Add a Discord-plugin-owned closed target-selection result that maps configured bot user IDs to accounts and agents, applies explicit-mention then reply-author precedence, and invokes existing binding/default routing only for untargeted messages.

## Reproduction Assessment

Yes at source level: current main selects the route before Discord target facts and only recognizes replies to the current handler bot. A live Discord reproduction was not run during this read-only review.

## Solution Assessment

Yes, with a narrower ownership shape: native Discord metadata should produce one plugin-owned target before shared fallback routing. The open generic suppression PR is incomplete because it omits reply-target and cross-account selection.
