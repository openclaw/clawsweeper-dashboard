## Summary

Keep open: current main still lacks Discord cross-bot target selection and default-owner suppression for explicit bot mentions or reply targets, and the related fixes remain open partial PRs rather than merged code.

## What This Changes

Request Discord guild routing precedence that targets explicitly mentioned bot accounts first, replied-to bot authors second, and only falls back to the per-channel default owner when no target signal exists.

## Best Possible Solution

Land one Discord-owned routing change that computes targeted bot eligibility before default routing, applies mention > reply-target > default-owner precedence, and proves default-owner suppression with focused tests.

## Reproduction Assessment

Yes, at source level. The issue discussion gives a concrete multi-bot Discord guild scenario, and current main still resolves routing before cross-bot target metadata while comparing mentions and replies only to the running bot ID.

## Solution Assessment

Yes, the requested precedence is the narrow maintainable direction for this multi-bot Discord surface. It should be reconciled with https://github.com/openclaw/openclaw/pull/47393 and https://github.com/openclaw/openclaw/pull/67460 before implementation lands.
