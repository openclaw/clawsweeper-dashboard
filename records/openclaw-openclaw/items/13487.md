## Summary

Keep open: current main still resolves the Discord route before cross-bot mention or reply-target selection, compares mention/reference metadata only to the running bot ID, and lacks a default-owner suppression decision. Related implementation PRs are still open, so this should remain the coordinating issue.

## What This Changes

Request deterministic Discord multi-bot routing where explicit bot mentions win first, replies to a bot target that bot second, and the channel default owner responds only when no target signal exists.

## Best Possible Solution

Land one Discord-owned routing change that computes target bot eligibility from Discord mention/reference metadata before default routing, applies the requested precedence, and proves default-owner suppression with focused tests.

## Reproduction Assessment

Yes, at source level. The issue includes a concrete multi-bot Discord guild scenario, and current main still compares mentions and replies only to the running bot ID after resolving the route.

## Solution Assessment

Yes, the requested precedence is the narrow maintainable direction for this Discord multi-bot surface. A generic mention-gating flag alone would not map Discord bot IDs to OpenClaw agents, so it needs to be reconciled with the overlapping open PRs before landing.
