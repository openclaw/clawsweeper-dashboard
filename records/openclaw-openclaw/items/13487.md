## Summary

Keep open: current main still lacks Discord cross-bot target selection and default-owner suppression for explicit bot mentions or reply targets. The issue discussion includes a concrete multi-bot Discord reproduction, and the related fixes remain open partial PRs rather than merged behavior.

## What This Changes

Request deterministic Discord multi-bot routing with precedence of explicit bot mention, then reply target, then per-channel default owner, with no default-owner double response.

## Best Possible Solution

Land one Discord-owned routing change that computes target bot eligibility from Discord mention/reference metadata before default routing, applies mention > reply-target > default-owner precedence, and proves default-owner suppression with focused tests.

## Reproduction Assessment

Yes, at source level. The discussion gives a concrete multi-bot Discord guild scenario, and current main still routes before cross-bot target metadata while comparing mentions and replies only to the running bot ID.

## Solution Assessment

Yes, the requested precedence is the maintainable direction for this Discord multi-bot surface. The implementation should use native Discord metadata, not plain-text names, and be reconciled with the overlapping open PRs before landing.
