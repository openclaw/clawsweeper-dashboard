## Summary

Keep open: current main still lacks Discord cross-bot target selection and default-owner suppression for explicit bot mentions or reply targets. The discussion includes a concrete multi-bot Discord guild reproduction, and the related implementation attempts remain open PRs rather than merged behavior.

## What This Changes

The issue requests deterministic Discord multi-bot routing precedence: explicit bot mention first, reply target second, and per-channel default owner only when no target signal exists.

## Best Possible Solution

Land one coordinated Discord-owned routing change that computes target bot eligibility from Discord mention/reference metadata before default routing, applies mention > reply-target > default-owner precedence, and proves default-owner suppression with focused tests.

## Reproduction Assessment

Yes, at source level. The discussion gives a concrete multi-bot Discord guild scenario, and current main still resolves routing before cross-bot target metadata while comparing mentions and replies only to the running bot ID.

## Solution Assessment

Yes, the requested precedence is the narrow maintainable direction for this Discord multi-bot surface. The implementation should be reconciled with https://github.com/openclaw/openclaw/pull/47393 and https://github.com/openclaw/openclaw/pull/67460 before landing.
