## Summary

Keep open: current main still resolves the Discord route before selecting cross-bot mention or reply targets, and the possible fixes remain open, partial PRs or maintainer product decisions rather than merged behavior.

## What This Changes

Request deterministic Discord multi-bot routing that prefers explicit bot mentions, then replied-to bot authors, then the per-channel default owner while suppressing non-target default responders.

## Best Possible Solution

Land one Discord-owned target-selection change that maps native mention/reference bot IDs to configured OpenClaw accounts before default-owner routing, suppresses non-target default responders, and reconciles the overlapping open PRs.

## Reproduction Assessment

Yes, at source level. Current main resolves Discord routing before cross-bot mention/reference target state and only compares metadata to the running botUserId, while the issue discussion includes a relevant live multi-bot reproduction.

## Solution Assessment

Yes for the requested end state, but not as another independent repair branch. The maintainable path is a consolidated Discord-owned target-selection step reconciled with the open related PRs and maintainer product labels.
