## Label Justifications

- `P1`: The issue affects real multi-bot Discord routing and can cause targeted messages to be handled by the wrong responder or not visibly answered.
- `impact:message-loss`: The reported Discord reply and mention cases can produce no visible reply, duplicate responses, or wrong-agent responses in the channel.
- `impact:session-state`: Routing a targeted Discord message to the default owner can associate the turn with the wrong agent or session state.

## Summary

Keep open: current main still makes Discord route/default decisions before any cross-bot target selection, and related fixes remain open PRs rather than merged behavior.

## What This Changes

Discord multi-bot routing should prefer explicit bot mentions, then replied-to bot targets, then the per-channel default owner, with the default owner suppressed when a specific bot is targeted.

## Best Possible Solution

Land one Discord-owned target-selection change that maps native mention/reference bot IDs to configured OpenClaw accounts before default-owner routing, suppresses non-target default responders, and reconciles the overlapping open PRs.

## Reproduction Assessment

Yes, at source level. Current main resolves Discord routing before cross-bot mention/reference target state and only compares metadata to the running botUserId; the issue discussion also includes a relevant live multi-bot reproduction.

## Solution Assessment

Yes for the requested end state, but not as another independent partial repair. The maintainable path is a consolidated Discord target-selection step reconciled with the open related PRs and product-review labels.
