## Label Justifications

- `P1`: Multi-bot Discord messages can receive no intended reply, a duplicate reply, or a reply from the wrong agent in an active channel workflow.
- `impact:message-loss`: The live report shows reply-shaped messages can be processed without producing the intended bot's visible response.
- `impact:session-state`: Ordinary default or binding routing can associate targeted input with a different agent and conversation session.

## Summary

Current main and the latest release still lack cross-bot target selection: Discord chooses the ordinary binding/default route first, then evaluates mention and reply metadata only against the currently running bot. The open related PR adds partial shared suppression but does not implement reply-target routing or the complete precedence, so this issue remains necessary as the canonical product decision.

## What This Changes

Add deterministic Discord multi-bot routing where explicit bot mentions take precedence over replied-to bot identity, which takes precedence over the channel default owner, while suppressing non-target responders.

## Best Possible Solution

Keep target selection in the Discord plugin: build a configured bot-user-ID to account/agent mapping, select explicit mentioned bots first and the referenced-message bot second, and call ordinary binding/default routing only when neither targeted signal resolves; add focused tests for selection and non-target suppression.

## Reproduction Assessment

Yes at source level with high confidence: current main demonstrably resolves routing before target signals and recognizes reply metadata only for the running bot ID. This read-only review did not run a live Discord setup, but the commenter also supplied a concrete live reproduction pattern.

## Solution Assessment

Yes for the requested product precedence, but not as only a shared mention-gating flag. The narrow maintainable implementation is Discord-owned target identity selection before existing binding/default routing, with the generic gating layer consuming only transport-neutral decisions.
