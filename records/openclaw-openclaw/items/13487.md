## Label Justifications

- `P1`: Multi-bot Discord routing can currently send no intended reply, duplicate replies, or a reply from the wrong agent in active channel workflows.
- `impact:message-loss`: The issue and live comment describe reply-shaped or targeted Discord messages failing to produce the intended bot response.
- `impact:session-state`: Default or binding routing can associate targeted Discord input with the wrong agent and conversation session.

## Summary

Keep open: current main and the latest release still lack Discord cross-bot target selection, and the related implementation PRs are closed unmerged or only partial, so no merged fix owns the requested precedence.

## What This Changes

The issue requests Discord multi-bot routing precedence where explicit bot mentions beat reply-target metadata, and both targeted cases suppress the channel default owner.

## Best Possible Solution

Define and implement Discord-owned target identity selection before ordinary routing: explicit configured bot mentions first, replied-to bot author second, then existing default/binding routing, with tests proving non-target/default-owner suppression.

## Reproduction Assessment

Yes, at source level with high confidence: current main routes before target selection and compares mention/reply metadata only against the running bot ID. I did not run a live Discord setup in this read-only review.

## Solution Assessment

Yes for the requested end state, but not as a generic suppression flag alone. The maintainable fix is a Discord-owned target-selection step before shared binding/default routing, with maintainer-approved compatibility semantics.
