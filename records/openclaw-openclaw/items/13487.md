## Label Justifications

- `P1`: The issue affects live multi-bot Discord routing and can cause missing, duplicate, or wrong-agent responses in channel workflows.
- `impact:message-loss`: Targeted Discord messages can be consumed by the default owner or fail to reach the intended bot response path.

- `maturity:stable`: Discord is scored M4 Stable and this issue targets its Conversation Routing and Delivery category.

## Summary

Keep open: current main and v2026.6.10 still resolve Discord routing before cross-bot mention or reply-target selection, compare those target signals only to the running handler bot ID, and lack a default-owner suppression state for another targeted bot.

## What This Changes

Request deterministic Discord multi-bot routing where explicit bot mentions win over reply targets, and reply targets win over the per-channel default owner.

## Best Possible Solution

Confirm the Discord active-target contract, then implement target selection inside the Discord plugin before shared default routing with explicit mentioned bot accounts first, referenced bot author second, and default owner only when no target signal exists.

## Reproduction Assessment

Yes, at source level: current main resolves Discord routing before mention/reply target selection and compares mention/reply metadata only to the current handler botUserId. I did not run a live Discord multi-bot replay in this read-only review.

## Solution Assessment

Unclear until maintainers confirm the product contract. Generic mention suppression alone is not the best fix because Discord needs a configured bot/account/agent target map before default routing.
