## Summary

Keep open: current main still lacks Discord cross-bot target selection and default-owner suppression for explicit bot mentions or reply targets, and the related fixes are still open partial PRs rather than shipped code.

## What This Changes

Request deterministic Discord multi-bot routing where explicit bot mentions outrank reply targets and both suppress per-channel default-owner routing.

## Best Possible Solution

Land one Discord-owned routing fix that maps configured Discord bot user IDs to account or agent targets before default routing, applies mention > reply-target > default-owner precedence, and proves default-owner suppression with focused tests.

## Reproduction Assessment

Yes, at source level. The issue discussion gives a concrete multi-bot Discord guild scenario, and current main still resolves the route before cross-bot target metadata while only comparing mentions and replies to the running bot ID.

## Solution Assessment

Yes, the requested precedence is the narrow maintainable direction for the documented multi-bot Discord surface. It should be reconciled with https://github.com/openclaw/openclaw/pull/47393 and https://github.com/openclaw/openclaw/pull/67460 so route ownership, reply targeting, and default-owner suppression land coherently.
