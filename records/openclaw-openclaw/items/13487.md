## Label Justifications

- `P1`: The issue affects live Discord multi-bot routing and can cause missing, duplicate, or wrong-agent responses in channel workflows.
- `impact:message-loss`: Targeted Discord messages can be consumed by the default owner or fail to reach the intended bot response path.
- `impact:session-state`: Routing a targeted Discord message to the default owner can attach the turn to the wrong agent session and conversation state.

## Summary

Keep open: current main and the latest release still choose the Discord route before cross-bot mention or reply-target selection, compare mention/reply metadata only against the running bot id, and lack a default-owner suppression state for targeted Discord messages.

## What This Changes

Requested deterministic Discord multi-bot routing precedence where explicit mentions win over reply targets, reply targets win over the default owner, and the default owner is suppressed whenever a targeted signal exists.

## Best Possible Solution

Confirm the Discord active-target contract, then implement target selection inside the Discord plugin before default routing with explicit mentioned bot accounts first, referenced bot author second, and default owner only when no targeted signal exists.

## Reproduction Assessment

Yes, at source level: current main resolves Discord routing before mention/reply target selection and compares mention/reply metadata only to the current handler botUserId. I did not run a live Discord multi-bot replay in this read-only review.

## Solution Assessment

Unclear until maintainers confirm the product contract; the requested precedence is plausible, but generic mention suppression alone is not the best fix because Discord needs a configured bot/account/agent target map before default routing.
