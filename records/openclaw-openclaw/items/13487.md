## Label Justifications

- `P1`: The issue affects live multi-bot Discord routing and can cause missing, duplicate, or wrong-agent responses in channel workflows.
- `impact:message-loss`: The reported behavior can suppress the intended bot response or allow the default owner to consume targeted Discord traffic.
- `impact:session-state`: Routing a targeted Discord message to the default owner can attach the turn to the wrong agent session and conversation state.

## Summary

Keep open: current main and the latest release still resolve the Discord route before cross-bot target selection, compare explicit mention and reply metadata only to the current handler's botUserId, and lack the requested default-owner suppression state. Related work is either closed unmerged or only partially overlaps this exact precedence request.

## What This Changes

Request deterministic Discord multi-bot routing where explicit bot mentions beat reply targets, reply targets beat the per-channel default owner, and the default owner is suppressed for targeted messages.

## Best Possible Solution

Confirm the Discord active-target contract, then implement Discord-owned target selection before default routing with a configured bot-user-id to account/agent map, explicit mentions first, reply author second, and focused suppression tests.

## Reproduction Assessment

Yes at source level: current main resolves Discord routing before mention/reply target selection and compares mention/reply metadata only to the current handler's botUserId. I did not run a live Discord multi-bot replay in this read-only review.

## Solution Assessment

Unclear until maintainer product direction is confirmed; the requested precedence is plausible, but the durable fix needs an approved owner boundary for target maps and suppression semantics.
