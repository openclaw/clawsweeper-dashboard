## Summary

Keep open: current main still chooses the Discord route before cross-bot mention/reference target selection, and the related fixes are open PRs rather than merged or shipped behavior.

## What This Changes

Request deterministic Discord multi-bot routing precedence where explicit bot mentions win over reply targets, reply targets win over the channel default owner, and targeted messages suppress the default responder.

## Best Possible Solution

Land one Discord-owned routing change that maps native mention/reference bot IDs to configured OpenClaw accounts before default-owner routing, suppresses non-target default responders, and reconciles the open partial PRs.

## Reproduction Assessment

Yes, at source level. Current main resolves Discord routing before cross-bot mention/reference target state and only compares metadata to the running botUserId, while the issue discussion includes a relevant live multi-bot reproduction.

## Solution Assessment

Yes for the requested end state, but the implementation path needs maintainer coordination. The narrow maintainable direction is a Discord-owned target-selection step before shared default routing, reconciled with the open related PRs.
