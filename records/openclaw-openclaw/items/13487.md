## Summary

Keep open: current main still lacks Discord cross-bot target selection and default-owner suppression for explicit bot mentions or reply targets. The discussion includes a concrete multi-bot Discord guild reproduction, and the related partial fixes are still open PRs rather than merged behavior.

## What This Changes

Request deterministic Discord multi-bot routing where explicit bot mentions outrank reply targets, reply targets outrank the per-channel default owner, and targeted messages suppress default-owner double replies.

## Best Possible Solution

Land one Discord-owned routing change that computes targeted bot eligibility from Discord mention/reference metadata before default routing, applies mention > reply-target > default-owner precedence, and proves default-owner suppression with focused tests.

## Reproduction Assessment

Yes, at source level. The discussion gives a concrete multi-bot Discord guild scenario, and current main still resolves routing before cross-bot target metadata while comparing mentions and replies only to the running bot ID.

## Solution Assessment

Yes, the requested precedence is the narrow maintainable direction for this Discord multi-bot surface. The implementation should be reconciled with https://github.com/openclaw/openclaw/pull/47393 and https://github.com/openclaw/openclaw/pull/67460 before landing.
