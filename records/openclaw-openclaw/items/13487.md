## Summary

Keep open: current main still computes Discord routing/default selection before cross-bot target selection, and mention/reply metadata is checked only against the running handler's bot ID. The related PRs in the provided context are still open and partial, so they do not yet implement or supersede the requested precedence.

## What This Changes

Request deterministic Discord multi-bot routing where explicit mentions win over reply targets, reply targets win over the channel default owner, and the default owner is suppressed whenever a targeted bot is selected.

## Best Possible Solution

Implement Discord-owned target selection before shared default routing, mapping configured Discord bot user IDs to OpenClaw accounts/agents and suppressing the default owner whenever explicit mention or reply metadata selects another configured bot.

## Reproduction Assessment

Yes. GTZhou provided concrete guild-channel steps for mention, reply, and reply-plus-mention cases, and current-main source inspection shows the mention/reply checks remain scoped to the running bot ID.

## Solution Assessment

Yes. The requested precedence is the narrow maintainable direction, but it should be reconciled with #47393 and #67460 so owner routing, reply-target selection, and default-owner suppression land coherently.
