## Label Justifications

- `P1`: The issue affects live multi-bot Discord routing and can cause missing, duplicate, or wrong-agent responses in channel workflows.
- `impact:message-loss`: The reported behavior can suppress the intended bot response or allow the default owner to consume targeted Discord traffic.
- `impact:session-state`: Routing a targeted Discord message to the default owner can attach the turn to the wrong agent session and conversation state.

## Summary

Keep open: current main and the latest release still resolve Discord routing before cross-bot target selection, compare mention and reply metadata only to the running handler bot ID, and do not expose a default-owner suppression decision. Related PRs that attempted parts of this behavior are closed unmerged, and the remaining Discord precedence contract needs maintainer product review.

## What This Changes

The issue requests deterministic Discord multi-bot routing precedence: explicit bot mentions first, reply-target bot second, and default owner only when no targeted signal applies.

## Best Possible Solution

Confirm the Discord active-target contract, then implement Discord-owned target selection before default routing with a configured bot-user-id to account/agent map, explicit mentions first, reply author second, default owner only as fallback, and focused suppression tests.

## Reproduction Assessment

Yes at source level: current main resolves Discord routing before cross-bot target selection and compares mention/reply metadata only to the running botUserId. I did not run a live Discord multi-bot replay in this read-only review.

## Solution Assessment

Unclear until maintainer policy is confirmed. The requested precedence is plausible and well-scoped, but the durable fix needs an approved owner boundary for the configured bot map and suppression semantics.
