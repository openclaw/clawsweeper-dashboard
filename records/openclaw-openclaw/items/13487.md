## Summary

Keep open: current main still lacks Discord cross-bot target selection and default-owner suppression for explicit bot mentions or reply targets, and the related fixes remain open partial PRs rather than merged code.

## What This Changes

The issue requests deterministic Discord multi-bot routing where explicit bot mentions outrank reply targets, reply targets outrank the channel default owner, and the default owner is suppressed whenever a targeted bot signal exists.

## Best Possible Solution

Land one Discord-owned routing change that computes targeted bot eligibility before default routing, applies mention > reply-target > default-owner precedence, and proves default-owner suppression with focused tests.

## Reproduction Assessment

Yes, at source level. The issue discussion gives a concrete multi-bot Discord guild scenario, and current main still resolves the route before cross-bot target metadata while only comparing mentions and replies to the running bot ID.

## Solution Assessment

Yes, the requested precedence is the narrow maintainable direction for the multi-bot Discord surface, but it should be reconciled with https://github.com/openclaw/openclaw/pull/47393 and https://github.com/openclaw/openclaw/pull/67460 before implementation lands.
