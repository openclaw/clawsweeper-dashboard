## Summary

Keep open: current main still resolves the Discord route before selecting cross-bot mention or reply targets, and the possible fixes are still open PRs or maintainer/product decisions rather than merged behavior.

## What This Changes

The issue requests deterministic Discord multi-bot routing precedence: explicit bot mention first, reply target second, and default owner only when no target signal exists.

## Best Possible Solution

Land one Discord-owned routing change that maps native mention/reference bot IDs to configured OpenClaw accounts before default-owner routing, suppresses non-target default responders, and reconciles the overlapping open PRs.

Yes, at source level. Current main resolves Discord routing before cross-bot mention/reference target state and only compares metadata to the running botUserId, while the issue discussion includes a relevant live multi-bot reproduction.

## Solution Assessment

Yes for the requested end state, but not as another independent repair branch. The maintainable path is a consolidated Discord-owned target-selection step reconciled with the open related PRs and maintainer product labels.
