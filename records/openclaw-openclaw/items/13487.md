## Summary

Keep open: current main still resolves Discord routing before cross-bot target selection and only compares mention/reply metadata to the running bot identity, so the requested default-owner suppression is not implemented. The related partial fixes remain open rather than merged.

## What This Changes

Request deterministic Discord multi-bot routing: explicit bot mentions first, reply-target bot second, and per-channel default owner only when no targeted signal exists.

## Best Possible Solution

Land one Discord-owned routing change that computes targeted bot eligibility from Discord mention/reference metadata before default routing, applies mention > reply-target > default-owner precedence, and proves suppression with focused tests.

## Reproduction Assessment

Yes, at source level. The issue and comment provide concrete multi-bot Discord scenarios, and current main still compares mentions/replies only against the running bot after route resolution.

## Solution Assessment

Yes. The requested precedence is the narrow maintainable direction, but it should be reconciled with https://github.com/openclaw/openclaw/pull/47393 and https://github.com/openclaw/openclaw/pull/67460 before landing.
