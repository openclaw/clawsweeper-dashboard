## Label Justifications

- `P1`: The issue affects real multi-bot Discord routing and can cause targeted messages to be handled by the wrong responder or not visibly answered.
- `impact:message-loss`: The reported Discord reply and mention cases can produce no visible reply, duplicate responses, or wrong-agent responses in the channel.
- `impact:session-state`: Routing a targeted Discord message to the default owner can associate the turn with the wrong agent or session state.

## Summary

Keep open: current main still derives the Discord route before cross-bot mention/reply target selection, and the related fixes are still open PRs rather than merged behavior.

## What This Changes

The issue requests deterministic Discord multi-bot routing precedence: explicit mention first, reply target second, default owner only when no target signal exists.

## Best Possible Solution

Add one Discord-owned target-selection layer that maps native mention and referenced-message bot IDs to configured OpenClaw accounts before default-owner routing, then cover default-owner suppression with focused preflight/routing tests.

## Reproduction Assessment

Yes, at source level. Current main resolves the Discord route before cross-bot mention/reference targeting and only compares reply metadata to the running botUserId; the issue discussion also includes a relevant live multi-bot Discord reproduction.

## Solution Assessment

Yes for the requested end state, but not as another independent partial repair. The maintainable path is a consolidated Discord target-selection step reconciled with the open related PRs and product-review labels.
