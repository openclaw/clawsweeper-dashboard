## Summary

Keep open: current main still passes only the explicit `sessions_spawn.model` into ACP spawn initialization, so the linked bug is real, but this PR is not merge-ready because it broadens alias model fallback semantics and still lacks live ACP/acpx behavior proof.

## What This Changes

The PR makes ACP alias spawns resolve a configured OpenClaw ACP alias model into `runtimeOptions.model` and adds focused tests for alias and raw harness behavior.

## Best Possible Solution

Land a narrow ACP spawn fix that forwards explicit `sessions_spawn.model` or an explicit per-alias `model.primary`, preserves raw harness and unmodeled-alias defaults unless maintainers approve otherwise, and includes live ACP/acpx proof plus regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `spawnAcpDirect` resolves an ACP alias to its harness id but passes only `params.model` into ACP initialization, so an alias `model.primary` is dropped when the tool call omits `model`.

## Solution Assessment

No, not as submitted. The fix belongs in ACP spawn, but using the effective model helper broadens behavior to global defaults; the safer path is explicit alias model forwarding or maintainer-approved effective-default semantics with upgrade and live proof.
