## Best Possible Solution

Keep this as the canonical filesystem ACL tracker until an owner-reviewed `tools.fs` or successor policy lands with deny-wins path rules, explicit upgrade/default semantics, coverage for filesystem-adjacent tools, and a documented exec decision.

## Solution Assessment

No for a quick automated fix. The maintainable solution is a security/product-reviewed filesystem policy contract with explicit deny precedence, canonicalization, upgrade behavior, filesystem-adjacent coverage, and an exec scope decision.
