## What This Changes

The issue requests a configurable filesystem sandbox policy with mode, allowed paths, and deny paths for OpenClaw tool access.

## Best Possible Solution

Keep this as the canonical filesystem ACL tracker until an owner-reviewed `tools.fs` or successor policy lands with deny-wins path rules, explicit upgrade/default semantics, coverage for filesystem-adjacent tools, and a documented exec decision.

## Reproduction Assessment

Yes. Source inspection shows current main and `v2026.5.20` accept only `tools.fs.workspaceOnly`, while tests preserve unrestricted host file access when that boolean is false or unset.

## Solution Assessment

No for a quick automated fix. The maintainable solution is a security/product-reviewed filesystem policy contract with explicit deny precedence, canonicalization, upgrade behavior, subagent/media/apply_patch coverage, and an exec scope decision.
