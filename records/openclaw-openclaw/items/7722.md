## What This Changes

Requests a filesystem access policy config with sandbox/full mode and allow/deny path lists enforced across read, write, and exec tools.

## Best Possible Solution

Keep this as the canonical filesystem ACL tracker until an owner-reviewed policy surface lands with deny-wins path rules, canonical path semantics, upgrade behavior, subagent/media/apply_patch coverage, and an explicit exec decision.

## Reproduction Assessment

Yes, source-reproducible. Current main and v2026.5.20 only accept `tools.fs.workspaceOnly`, and tests preserve unrestricted host access when that boolean is false or unset.

## Solution Assessment

No for a quick automated fix. The maintainable solution is a security/product-reviewed filesystem policy contract, likely under `tools.fs`, with explicit semantics for deny precedence, canonicalization, upgrades, subagents, media/apply_patch coverage, and exec scope.
