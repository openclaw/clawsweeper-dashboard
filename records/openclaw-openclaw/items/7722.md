## Summary

Keep open. Current main and the latest release still expose only the narrower `tools.fs.workspaceOnly` boolean, while the requested configurable allow/deny filesystem policy and exec strategy remain unmerged product/security work.

## What This Changes

Request a filesystem access configuration surface for sandbox/full mode, allowed paths, and deny paths enforced across file and exec-style tools.

## Best Possible Solution

Keep this as the canonical tracker until maintainers choose a `tools.fs` or successor policy contract, land reviewed filesystem/media/apply_patch coverage, and explicitly decide whether exec is in scope or a documented separate follow-up.
