## Summary

Keep open. Current main still lacks the requested allow/deny filesystem policy and exec coverage; the existing `workspaceOnly` guard is narrower, and the related PathGuard PR is open and explicitly partial for shell execution.

## What This Changes

The issue requests a configurable filesystem sandbox policy with allowed paths, deny paths, sandbox/full mode, and enforcement across file and exec tools.

## Best Possible Solution

Keep this tracked until OpenClaw has a versioned filesystem policy contract with deny-wins path rules for filesystem tools and local media, plus an explicit maintainer-approved strategy for exec.

## Reproduction Assessment

Yes. Source inspection is a high-confidence reproduction path: the strict schema rejects the requested keys, the runtime policy only models `workspaceOnly`, and exec construction receives no filesystem allow/deny policy.

## Solution Assessment

No. The current `workspaceOnly` boolean is a narrower guardrail, and the open PathGuard work is only a partial direction until merged and the exec gap is resolved or deliberately split into a separate tracked decision.
