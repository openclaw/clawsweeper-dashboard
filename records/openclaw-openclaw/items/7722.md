## Summary

Keep open: current main and v2026.6.6 still expose only the narrower `tools.fs.workspaceOnly` boolean plus exec approval/tool policy controls, not the requested allow/deny filesystem policy with an explicit exec strategy. ## What This Changes

The issue requests a configurable filesystem access policy such as `tools.fileAccess` or an equivalent allow/deny sandbox surface enforced across file and exec tools.

## Best Possible Solution

Keep this issue open as the canonical filesystem ACL tracker until maintainers choose and land a versioned `tools.fs` or successor policy, with deny-wins path rules across filesystem surfaces and either a safe exec strategy or an explicit linked exec follow-up.
