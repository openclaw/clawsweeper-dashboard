## What This Changes

Request to add configurable filesystem access restrictions with allow and deny paths, enforced across file operations and exec.

## Best Possible Solution

Keep this as the canonical filesystem access policy issue until maintainers choose one versioned `tools.fs` or successor contract, enforce it consistently across local file/media/apply_patch paths, and either design exec coverage or split it into a tracked follow-up.
