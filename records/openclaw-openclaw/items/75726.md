## Summary

Keep open. Current main and the latest release only strip the modern runtime-context header and notice, while the generated async/metadata wrapper body that this issue asks to remove can still remain in user-facing sanitized text; the active implementation path is the open closing PR at https://github.com/openclaw/openclaw/pull/75755.

## What This Changes

The issue asks OpenClaw to strip echoed modern runtime-context scaffolding, including async-completion and metadata wrapper paragraphs, before user-facing channel delivery.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75755, or an equivalent shared sanitizer fix, with generator-aligned coverage for async completion wrappers, metadata wrappers, false positives, and legacy/delimited behavior.

## Reproduction Assessment

Yes, at source level. A modern header and notice followed by the current async-completion body will pass through `sanitizeUserFacingText` with the header removed but the generated command-detail and relay-instruction paragraphs still present.

## Solution Assessment

Yes. A shared fix in `src/agents/internal-runtime-context.ts` is the narrow maintainable solution because multiple user-facing sanitization and delivery surfaces flow through that helper; channel-specific stripping would leave other surfaces exposed.
