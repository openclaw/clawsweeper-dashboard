## Summary

Keep open. Current main still removes only the modern runtime-context header and privacy notice; the async-completion wrapper body described in the report can remain on the user-facing sanitizer path, and open PR #75755 is the active closing implementation candidate.

## What This Changes

Ensure user-facing sanitization removes echoed modern OpenClaw runtime-context wrapper bodies, including async-completion and metadata paragraphs, without dropping the real reply.

## Best Possible Solution

Review and land #75755, or an equivalent narrow shared-sanitizer fix, with regression coverage for async completion wrappers, conversation/sender metadata wrappers, false positives, and existing legacy/delimited behavior.

Yes. A high-confidence source-level reproduction is to pass the modern header and notice followed by the async-completion prompt body and a visible reply through `sanitizeUserFacingText`; current main strips the preface but leaves the unmarked async body.

## Solution Assessment

Yes. Fixing `stripInternalRuntimeContext` is the narrow maintainable path because several display and delivery surfaces already share that helper; a Discord- or Slack-only patch would leave other surfaces exposed.
