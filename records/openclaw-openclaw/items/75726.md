## Summary

Keep open. Current main partially addresses the modern runtime-context preface, but source inspection shows it still leaves the unmarked async-completion wrapper body described in the report, and #75755 is the open closing PR for that remaining sanitizer work.

## What This Changes

The issue requests that shared user-facing sanitization strip modern OpenClaw runtime-context wrappers, including async-completion and metadata wrapper bodies, before channel delivery.

## Best Possible Solution

Review and land #75755, or an equivalent narrow shared-sanitizer change, with regression coverage for async, metadata, bootstrap-warning, false-positive, and legacy/delimited cases.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass the modern header, privacy notice, unmarked async-completion body from `buildExecEventPrompt`, and a visible reply through `sanitizeUserFacingText`; current main removes only the preface and leaves the wrapper body.

## Solution Assessment

Yes. A shared `stripInternalRuntimeContext`/sanitizer fix is the narrow maintainable solution because Discord, Slack, Telegram, and other surfaces share the user-facing sanitizer; a channel-specific patch would leave other outputs exposed.
