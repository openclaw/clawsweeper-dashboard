## Summary

Keep open. Current main and the latest release still only strip the modern runtime-context header/notice, while generated async-completion wrapper text can remain in sanitized user-facing output; the open closing PR is the right active implementation path.

## What This Changes

The issue asks OpenClaw to strip the modern runtime-context wrapper and generated metadata/async paragraphs from user-facing channel output when a model echoes internal scaffolding.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75755, or an equivalent generator-aligned shared sanitizer fix, with coverage for async completion wrappers, metadata wrappers, false positives, and legacy/delimited behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a modern runtime-context header and notice followed by the current async-completion body from `buildExecEventPrompt`; current `sanitizeUserFacingText` strips only the preface and can leave the generated wrapper body visible.

## Solution Assessment

Yes. A shared fix in `src/agents/internal-runtime-context.ts` is the narrow maintainable solution because multiple user-facing sanitizer and delivery surfaces flow through that helper; channel-specific stripping would leave other surfaces exposed.
