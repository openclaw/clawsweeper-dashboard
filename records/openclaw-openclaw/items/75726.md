## Summary

Keep open. Current main still only strips the modern two-line runtime-context preface, while the async-completion wrapper body described in the report can remain in sanitized user-facing text; an open linked PR (#75755) uses closing syntax for this issue, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that OpenClaw strip the modern runtime-context wrapper body, not only the two-line preface, when echoed model output is sanitized before Discord, Slack, or other channel delivery.

## Best Possible Solution

Review and land #75755, or an equivalent narrow fix, in the shared internal-runtime-context sanitizer with regression coverage for async completion wrappers, conversation/sender metadata wrappers, false positives, and existing legacy/delimited behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass the modern header and notice followed by the async-completion prompt body and a visible reply through `sanitizeUserFacingText`; current main strips the preface but leaves the unmarked async body.

## Solution Assessment

Yes. The maintainable solution belongs in the shared runtime-context sanitizer because multiple display and delivery surfaces already depend on `stripInternalRuntimeContext`; a channel-specific Discord or Slack fix would leave other user-facing surfaces exposed.
