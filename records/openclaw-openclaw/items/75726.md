## Summary

Keep open. Current main has partial modern-header handling, but it only removes the two-line preface and does not consume the unmarked async-completion/runtime-context body that the report shows leaking before the visible reply.

## What This Changes

Extend user-facing runtime-context sanitization so echoed modern OpenClaw runtime-context wrappers are removed while preserving the real assistant response.

## Best Possible Solution

Add a bounded parser in `src/agents/internal-runtime-context.ts` for the modern next-turn/runtime-event wrapper body, update `hasInternalRuntimeContext`, and add focused sanitizer regressions for async completion, conversation/sender metadata, false positives, and existing legacy/delimited behavior.

## Reproduction Assessment

Yes. A source-level reproduction is clear: pass the two modern preface lines followed by an unmarked async-completion wrapper paragraph and a visible reply through `sanitizeUserFacingText`; current main removes only the preface lines and leaves the wrapper body.

## Solution Assessment

Yes, the narrow maintainable fix belongs in the shared internal-runtime-context sanitizer because both assistant text cleanup and display surfaces already route through it; avoid channel-specific fixes unless a final-delivery contract test shows an uncovered payload field.
