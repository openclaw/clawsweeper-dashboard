## Summary

Keep open. Current main has partial modern preface stripping, but it does not consume the unmarked async-completion/runtime-context body described in the report, and #75755 is still the open closing implementation candidate.

## What This Changes

The issue requests extending the user-facing internal runtime-context sanitizer to strip modern runtime-context wrapper bodies, not only the legacy or delimited forms.

## Best Possible Solution

Land #75755 or an equivalent shared sanitizer fix with regression coverage for async completion wrappers, conversation/sender metadata wrappers, false positives, and existing legacy/delimited behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass the modern header and notice followed by the async-completion prompt body and a visible reply through `sanitizeUserFacingText`; current main strips the preface but leaves the unmarked async body.

## Solution Assessment

Yes. The shared `stripInternalRuntimeContext` direction is the narrow maintainable fix because multiple user-facing display and delivery surfaces depend on that helper; a channel-specific Discord or Slack fix would leave other surfaces exposed.
