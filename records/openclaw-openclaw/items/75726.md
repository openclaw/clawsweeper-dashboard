## Summary

Keep open. Current main only strips the modern runtime-context preface, while unmarked async-completion and bootstrap wrapper text can still remain on the shared user-facing sanitizer path; #75755 is the active open fix candidate with closing syntax.

## What This Changes

The issue asks OpenClaw to strip modern runtime-context wrapper text before assistant replies are delivered to Discord, Slack, Telegram, and other user-facing channels.

## Best Possible Solution

Review and land #75755, or an equivalent shared sanitizer change, with regression coverage for async completion, metadata/replied-message blocks, bootstrap warnings, nested wrappers, false positives, and existing legacy/delimited behavior.

## Reproduction Assessment

Yes. A high-confidence source-level repro is to pass the modern header and notice followed by the async-completion prompt body and a visible reply through `sanitizeUserFacingText`; current source removes only the preface and leaves the async body text.

## Solution Assessment

Yes. The best fix belongs in the shared runtime-context sanitizer or immediately adjacent shared user-facing sanitizer, not in Discord/Slack/Telegram-specific delivery code; the open PR is the right review target before closing this issue.
