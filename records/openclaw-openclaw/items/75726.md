## Summary

Keep open. Current main and the latest shipped release strip the modern runtime-context header and privacy notice, but the source still leaves the unmarked async-completion wrapper body named in the report, and the open #75755 PR still references this issue with closing syntax.

## What This Changes

The issue asks for the shared user-facing sanitizer to remove modern OpenClaw runtime-context wrapper text before it reaches Discord, Slack, and similar channel outputs.

## Best Possible Solution

The shared sanitizer should strip the modern runtime-context preface plus known generated wrapper paragraphs while preserving the actual reply, with regression coverage for async completions, metadata wrappers, false positives, and existing legacy/delimited blocks.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass the modern header, notice, async-completion prompt body, and a visible reply through `sanitizeUserFacingText`; current source removes the preface but leaves the unmarked async body.

## Solution Assessment

Yes. A narrow shared-sanitizer fix is the maintainable solution because multiple UI and delivery surfaces consume `stripInternalRuntimeContext`; channel-specific cleanup would leave other paths exposed.
