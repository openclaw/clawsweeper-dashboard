## Summary

Keep open. Current main and the latest release recognize and strip the modern two-line runtime-context preface, but source inspection still shows the async-completion wrapper body can remain after that preface, and #75755 is the open closing PR for the broader sanitizer fix.

## What This Changes

The issue requests expanding the shared user-facing sanitizer so echoed modern OpenClaw runtime-context wrappers, including async-completion and metadata prompt bodies, are removed before channel delivery.

## Best Possible Solution

Land #75755 or an equivalent narrow shared-sanitizer change with regression coverage for async-completion wrappers, metadata wrappers, repeated wrappers, and false-positive quoted text.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a string containing the modern header, the privacy notice, an async-completion prompt body, and a visible reply passed through `sanitizeUserFacingText`; current source removes the preface but leaves the async body.

## Solution Assessment

Yes. A shared `stripInternalRuntimeContext` or adjacent shared-sanitizer fix is the narrow maintainable solution because the affected text can reach multiple display and delivery surfaces; a channel-only patch would leave other call sites exposed.
