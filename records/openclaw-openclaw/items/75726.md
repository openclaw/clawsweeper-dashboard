## Summary

Keep open for maintainer follow-up. Current main and the latest release now strip the well-formed modern runtime-context preface, and the latest issue comment reports that this resolves the dominant real-world leak, but an open closing PR still targets the broader wrapper-body handling and current main does not consume the async-completion prompt body described in the original report.

## What This Changes

Request to strip modern OpenClaw runtime-context wrappers, including async and metadata prompt bodies, from user-facing sanitizer output.

## Best Possible Solution

Review #75755 against the shipped preface-only implementation, then either close the PR and this issue as satisfied or land a bounded structure-aware body stripper with false-positive coverage for JSON, fenced replies, async completions, and metadata blocks.

## Reproduction Assessment

Yes for the broader wrapper-body portion: source inspection shows current main strips the modern header/notice but would leave an echoed async-completion body from `buildExecEventPrompt`. The exact header preface leak no longer reproduces on current main because the shared sanitizer now removes that preface.

## Solution Assessment

Unclear as a final close. The shipped line-based preface strip is a safer, validated fix for the dominant visible leak, but the broader body-stripping behavior in the open closing PR still needs normal maintainer review or an explicit decision that it is superseded.
