## Summary

Close as implemented. Current main and shipped v2026.4.27+ recognize and strip the modern runtime-context preface from shared user-facing sanitization paths, and the latest reporter validation confirms the real-world corpus is functionally cleaned in 2026.5.2.

## What This Changes

Strip the modern OpenClaw runtime-context wrapper from user-facing channel output when a model echoes it.

## Best Possible Solution

Keep the shipped shared sanitizer behavior, and handle typo-tolerant double-wrapper hardening or replay-history cleanup only as separate focused follow-ups if maintainers want broader coverage.

## Reproduction Assessment

No for current main: the modern header plus privacy notice is stripped by `stripInternalRuntimeContext`, and current tests cover the sanitizer path. The original 2026.4.26 report remains source-reproducible because that release lacked the modern-header stripping helper.

## Solution Assessment

Yes. The shipped shared helper is the narrow maintainable fix because it covers user-facing sanitizer callers without adding channel-specific Discord, Slack, or Telegram behavior.
