## Summary

Close as implemented. Current main strips the modern runtime-context header and privacy notice through the shared user-facing sanitizer, has regression coverage on that path, and the fix is included in shipped releases; later corpus validation in the discussion confirmed the reported leak is functionally resolved.

## What This Changes

The issue reports modern OpenClaw runtime-context wrapper text leaking into user-facing channel output when a model echoes internal scaffolding.

## Best Possible Solution

Keep the shipped line-based preface sanitizer, and track broader replay/transcript hygiene separately in #76629 if that remaining class needs work.

## Reproduction Assessment

No current-main failing reproduction remains. Source inspection and regression tests show the exact modern header/notice is stripped through sanitizeUserFacingText, while the historical failure is documented against 2026.4.26-era installs.

## Solution Assessment

Yes. The shared sanitizer is the narrowest maintainable fix for the user-facing leak across channel surfaces; the stricter paragraph-walker proposal in #75755 is superseded by the shipped line-based implementation and carries more false-positive risk.
