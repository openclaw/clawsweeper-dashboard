## Summary

Keep open. Current main still strips only the modern runtime-context header and notice, while the unmarked async-completion wrapper body can remain on the user-facing sanitizer path; open PR #75755 is the active closing implementation candidate.

## What This Changes

The issue requests that OpenClaw strip echoed modern runtime-context wrapper bodies from user-facing channel output, not only the two-line runtime-context preface.

## Best Possible Solution

Review and land #75755, or an equivalent narrow shared-sanitizer fix, with regression coverage for async completion wrappers, conversation/sender metadata wrappers, false positives, and existing legacy/delimited behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass the modern header and notice followed by the async-completion prompt body and a visible reply through `sanitizeUserFacingText`; current main removes the preface but leaves the unmarked async body.

## Solution Assessment

Yes. The issue’s proposed shared-sanitizer direction is the narrow maintainable fix because multiple display and delivery surfaces depend on `stripInternalRuntimeContext`; a Discord- or Slack-only sanitizer would leave other surfaces exposed.
