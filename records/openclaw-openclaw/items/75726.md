## Summary

Keep open. Current main still removes only the modern runtime-context header and privacy notice, while the async-completion wrapper body described in the report can remain on the user-facing sanitizer path; open PR #75755 is the active closing implementation candidate.

## What This Changes

The issue asks OpenClaw to strip modern runtime-context wrappers and their known async/metadata body paragraphs from user-facing model output.

## Best Possible Solution

Review and land #75755, or an equivalent narrow shared-sanitizer fix, with regressions for async completion wrappers, conversation/sender metadata wrappers, false positives, and legacy/delimited behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass the modern header and notice followed by the async-completion prompt body and a visible reply through `sanitizeUserFacingText`; current main removes the preface but leaves the unmarked async body.

## Solution Assessment

Yes. The shared sanitizer direction is the narrow maintainable fix because several display and delivery paths depend on `stripInternalRuntimeContext`; a Discord- or Slack-only cleanup would leave other surfaces exposed.
