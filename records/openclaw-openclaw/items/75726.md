## Summary

Keep open. Current main still only strips the modern runtime-context header and privacy notice, while the async-completion body described in the report can remain in user-facing sanitized text; the issue is also already paired with open PR #75755 using closing syntax, so it should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests extending the shared user-facing sanitizer so echoed modern OpenClaw runtime-context wrappers and their async/metadata body paragraphs are removed before Discord/Slack-style delivery while preserving the real reply.

## Best Possible Solution

Review and land #75755, or an equivalent narrow shared-sanitizer fix, with regression coverage for async completion wrappers, conversation/sender metadata wrappers, false positives, and existing legacy/delimited runtime-context behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass the modern header and notice followed by the async-completion prompt body and visible reply through `sanitizeUserFacingText`; current main strips the preface but leaves the unmarked async body path intact.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared runtime-context/user-facing sanitizer used by delivery and display surfaces; a Discord- or Slack-specific patch would leave the same echoed wrapper exposed elsewhere.
