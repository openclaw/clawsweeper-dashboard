## Summary

Keep open. Current main still only strips the modern runtime-context header/notice preface, while the unmarked async-completion or metadata body described in the report can remain in user-facing sanitized text; the linked open PR #75755 is the right place to review or land the fix before this issue closes.

## What This Changes

The issue requests extending the shared user-facing sanitizer so echoed modern OpenClaw runtime-context wrappers, including async-completion and metadata body paragraphs after the modern header, are removed while preserving the real reply.

## Best Possible Solution

Review and land #75755, or an equivalent narrow shared-sanitizer fix, with regression coverage for async-completion wrappers, conversation/sender metadata wrappers, false positives, and existing legacy/delimited behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass the modern header and notice followed by the async-completion prompt body and a visible reply through `sanitizeUserFacingText`; current main strips the preface but leaves the unmarked async body.

## Solution Assessment

Yes. The maintainable fix belongs in the shared runtime-context sanitizer because multiple user-facing paths call it; a Discord-only or Slack-only filter would leave other delivery surfaces exposed.
