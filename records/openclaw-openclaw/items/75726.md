## Summary

Keep open. Current main and v2026.5.3 strip the modern runtime-context header and privacy notice, and the latest reporter validation says that materially reduces the real leak corpus, but the source still leaves the unmarked async/context body described in the report and #75755 remains the open unmerged fix candidate with closing syntax.

## What This Changes

The issue asks OpenClaw to strip echoed modern runtime-context wrappers from user-facing channel output.

## Best Possible Solution

Land #75755 or an equivalent narrow shared-sanitizer fix with regression coverage for async completion wrappers, metadata wrappers, false positives, and existing legacy/delimited behavior.

## Reproduction Assessment

Yes. A high-confidence source-level repro is to pass the modern header, privacy notice, async-completion body from `buildExecEventPrompt`, and a visible reply through `sanitizeUserFacingText`; current source removes the preface and continues copying the unmarked async body.

## Solution Assessment

Yes. The best fix belongs in `stripInternalRuntimeContext` or the shared user-facing sanitizer because multiple display and delivery surfaces use that helper; a Discord- or Slack-only sanitizer would leave other outputs exposed.
