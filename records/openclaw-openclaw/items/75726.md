## Summary

Keep open. Current main still strips the modern runtime-context header and privacy notice but not the async-completion wrapper body described in the report, and #75755 is the active open fix candidate.

## What This Changes

Extend the user-facing sanitizer so echoed modern OpenClaw runtime-context wrappers, including async command completion bodies, are stripped before Discord/Slack delivery while preserving visible replies.

## Best Possible Solution

Review and land #75755, or an equivalent narrow shared-sanitizer fix, with regression coverage for async completion wrappers, metadata wrappers, false positives, and existing legacy/delimited behavior.

## Reproduction Assessment

Yes. A high-confidence source-level repro is to pass the modern header and notice followed by the async-completion prompt body and a visible reply through `sanitizeUserFacingText`; current source removes the preface but leaves the unmarked async body.

## Solution Assessment

Yes. Fixing `stripInternalRuntimeContext` or the shared sanitizer is the narrow maintainable solution because multiple display and delivery surfaces use that helper; a Discord- or Slack-only patch would leave other surfaces exposed.
