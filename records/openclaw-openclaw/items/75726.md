## Summary

Keep open. Current main still strips only the modern runtime-context header and privacy notice, leaving the async-completion wrapper body on the shared user-facing sanitizer path; open PR #75755 is the active unmerged closing implementation candidate.

## What This Changes

The issue requests extending the shared user-facing sanitizer to strip modern OpenClaw runtime-context wrapper bodies, including async-completion and metadata paragraphs, while preserving the real reply.

## Best Possible Solution

Review and land #75755, or an equivalent narrow shared-sanitizer fix, with regression coverage for async-completion wrappers, metadata wrappers, false positives, and legacy/delimited behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass the modern header and privacy notice followed by the async-completion prompt body and a visible reply through `sanitizeUserFacingText`; current main would strip the preface but leave the async body.

## Solution Assessment

Yes. Fixing `stripInternalRuntimeContext` is the narrow maintainable path because multiple display and delivery surfaces share it; a Discord- or Slack-only patch would leave other user-facing paths exposed.
