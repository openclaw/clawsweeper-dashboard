## Summary

Keep open. Current main still strips only the modern runtime-context preface, while the unmarked async-completion wrapper body can remain on the user-facing sanitizer path; open PR #75755 is the active closing implementation candidate.

## What This Changes

Request to extend the shared user-facing sanitizer so echoed modern OpenClaw runtime-context wrappers and async-completion metadata bodies are stripped while preserving the real assistant reply.

## Best Possible Solution

Review and land #75755, or an equivalent narrow shared-sanitizer fix, with regression coverage for async completion wrappers, conversation/sender metadata wrappers, false positives, and existing legacy/delimited behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass the modern header and notice followed by the async-completion prompt body and a visible reply through sanitizeUserFacingText; current main removes the preface but leaves the unmarked async body.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared stripInternalRuntimeContext/sanitizeUserFacingText path because live chat, gateway sanitization, and channel-facing helpers reuse it; a Discord- or Slack-only filter would leave other surfaces exposed.
