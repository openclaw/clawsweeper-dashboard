## Summary

Keep open. Current main still strips only the modern runtime-context header and privacy notice, while the async-completion and metadata wrapper bodies described in the report can remain in user-facing output; the open linked PR is the active fix path.

## What This Changes

The issue requests a shared sanitizer fix so echoed modern OpenClaw runtime-context wrappers are removed before Discord, Slack, Telegram, and other channel delivery.

## Best Possible Solution

Review and land #75755, or an equivalent narrow shared-sanitizer fix, with coverage for async completion wrappers, metadata wrappers, bootstrap warnings, nested wrappers, false positives, and existing legacy/delimited behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: pass the modern header, privacy notice, async-completion wrapper body, and a visible reply through `sanitizeUserFacingText`; current main strips the preface but leaves the unmarked body.

## Solution Assessment

Yes. Fixing `stripInternalRuntimeContext` or the shared user-facing sanitizer is the narrow maintainable solution because multiple display and delivery surfaces use that helper; a channel-only filter would leave other outputs exposed.
