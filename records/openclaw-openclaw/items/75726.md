## Summary

Keep open. Current main still only removes the modern runtime-context preface and notice; the generated async-completion body can remain in sanitized user-facing text, and open PR #75755 is the active closing candidate.

## What This Changes

Extend the shared sanitizer to strip modern OpenClaw runtime-context wrapper bodies from user-facing channel output while preserving the real reply.

## Best Possible Solution

Review and land #75755, or an equivalent narrow shared-sanitizer fix, with regression coverage for async-completion wrappers, metadata wrappers, false positives, and legacy/delimited behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass the modern header and privacy notice followed by the async-completion prompt body and a visible reply through `sanitizeUserFacingText`; current main strips the preface but leaves the async body.

## Solution Assessment

Yes. Fixing `stripInternalRuntimeContext` is the narrow maintainable boundary because multiple display and delivery surfaces share that helper; a Discord- or Slack-only patch would leave other paths exposed.
