## Summary

Keep open. Current main still strips only the modern runtime-context header and privacy notice, while the async-completion wrapper body described in the report can remain in sanitized user-facing text; open PR #75755 is the active fix candidate.

## What This Changes

The issue asks OpenClaw to strip modern runtime-context wrapper text, including async-completion and metadata paragraphs, before Discord/Slack-style channel delivery.

## Best Possible Solution

Review and land #75755, or an equivalent narrow shared-sanitizer fix, with regression coverage for async-completion wrappers, metadata wrappers, false positives, and existing legacy/delimited behavior.

## Reproduction Assessment

Yes, at source level. Passing the modern header and notice followed by the async-completion prompt body and a visible reply through `sanitizeUserFacingText` will remove only the preface and leave the unmarked async body.

## Solution Assessment

Yes. Fixing `stripInternalRuntimeContext` or the shared user-facing sanitizer is the narrow maintainable solution because multiple display and delivery surfaces use that helper; a Discord- or Slack-only patch would leave other surfaces exposed.
