## Summary

Keep open. Current main still strips only the modern runtime-context header/notice preface, while the async-completion wrapper body described in the report can remain on the shared user-facing sanitizer path; open PR #75755 is the active closing implementation candidate.

## What This Changes

The issue requests shared user-facing sanitization for modern OpenClaw runtime-context wrappers, including async-completion and metadata body paragraphs when models echo them.

## Best Possible Solution

Land #75755 or an equivalent narrow shared-sanitizer fix with regression coverage for async completion wrappers, conversation/sender metadata wrappers, false positives, and existing legacy/delimited behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass the modern header and notice followed by the async-completion prompt body and a visible reply through `sanitizeUserFacingText`; current main strips the preface but leaves the unmarked body text.

## Solution Assessment

Yes. The shared `stripInternalRuntimeContext`/`sanitizeUserFacingText` path is the narrow maintainable surface because multiple channel and display outputs depend on it; a Discord- or Slack-only cleanup would leave other surfaces exposed.
