## Summary

Keep open. Current main still only strips the modern runtime-context preface, not the async/metadata wrapper body described here, and PR #75755 is the active unmerged closing implementation candidate.

## What This Changes

The issue asks OpenClaw to strip modern runtime-context wrappers and related async/metadata prompt bodies from user-facing channel output when a model echoes internal scaffolding.

## Best Possible Solution

Land PR #75755 or an equivalent shared sanitizer fix with regression coverage for async completion wrappers, conversation/sender metadata wrappers, false positives, and legacy/delimited behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass the modern header/privacy notice followed by the async-completion prompt body and a visible reply through `sanitizeUserFacingText`; current main strips the preface but leaves the unmarked async body text.

## Solution Assessment

Yes. Fixing `stripInternalRuntimeContext` is the narrow maintainable path because several display and delivery surfaces share that helper; a Discord- or Slack-only patch would leave other outputs exposed.
