## Summary

Keep open. Current main has a shipped guard for the modern runtime-context preface, but it still does not consume the async-completion and metadata wrapper paragraphs described here; the open #75755 PR still references this issue with closing syntax and is the active fuller fix candidate.

## What This Changes

The issue asks for the shared user-facing sanitizer to strip the modern runtime-context wrapper, including async completion and metadata paragraphs, before channel delivery.

## Best Possible Solution

Land #75755 or an equivalent shared sanitizer fix with regression coverage for async completion, metadata/replied-message/bootstrap wrappers, false positives, and existing legacy/delimited behavior.

## Reproduction Assessment

Yes, at source level. Passing the modern header and notice followed by the `An async command you ran earlier...` body through `sanitizeUserFacingText` will remove the preface but leave the unmarked async wrapper paragraphs in current main.

## Solution Assessment

Yes. The #75755 direction of fixing the shared `stripInternalRuntimeContext` path is the narrow maintainable solution; a Discord-, Slack-, or Telegram-only filter would leave other user-facing surfaces exposed.
