## Summary

Keep open. Current main still only strips the modern runtime-context preface, while the async/runtime wrapper body described in the report remains able to pass through the shared user-facing sanitizer and final outbound delivery.

## What This Changes

The issue asks OpenClaw to strip the full modern runtime-context wrapper body, not only the two-line preface, when echoed into assistant text sent to chat channels.

## Best Possible Solution

Add a bounded modern-wrapper parser in src/agents/internal-runtime-context.ts, keep the shared sanitizer as the single owner, preserve existing legacy/delimited behavior, and add focused regressions for async completion, conversation/sender metadata, false positives, and outbound delivery.

## Reproduction Assessment

Yes. A source-level reproduction is clear: pass the modern two-line preface followed by the async-completion wrapper body and a visible reply through sanitizeUserFacingText; current main removes only the preface and leaves the unmarked wrapper body.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared internal-runtime-context sanitizer because reply normalization, live projection, transcript cleanup, and channel delivery already reuse that helper; channel-specific fixes would leave other surfaces inconsistent.
