## Summary

Keep open. Current main still strips only the modern runtime-context preface and leaves the async-completion wrapper body described in the report; #75755 is the active open fix candidate with closing syntax, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Modern OpenClaw runtime-context wrappers should be stripped from user-facing channel output when a model echoes them.

## Best Possible Solution

Review and land #75755, or an equivalent narrow shared-sanitizer fix, with regression coverage for async completion wrappers, metadata wrappers, false positives, and existing legacy/delimited behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass the modern header and notice followed by the async-completion prompt body and a visible reply through `sanitizeUserFacingText`; current main removes only the preface and leaves the unmarked async body.

## Solution Assessment

Yes. Fixing `stripInternalRuntimeContext` or the shared sanitizer is the narrow maintainable solution because multiple display and delivery surfaces use that helper; a Discord- or Slack-only patch would leave other surfaces exposed.
