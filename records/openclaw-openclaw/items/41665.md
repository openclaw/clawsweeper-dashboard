## Summary

Keep open. The OS-filter metadata direction is correct, but this PR is stale against current main, misses newer unscoped Homebrew install specs that would still appear on Windows, and mixes in an xurl installer-kind change that needs explicit maintainer review.

## What This Changes

The branch adds per-installer OS filters to bundled skill Homebrew install entries and changes xurl's fallback installer metadata from unsupported `npm` to supported `node`.

## Best Possible Solution

Refresh or replace the branch with a current-main inventory audit of all bundled Homebrew install specs, add OS metadata where appropriate, split or explicitly approve the xurl node installer activation, and add a small regression test that catches new unscoped Homebrew specs.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: on `win32`, unscoped install specs survive `normalizeInstallOptions`, and onboarding lists missing skills with remaining install options; current main still has representative unscoped Homebrew specs.

## Solution Assessment

No. The metadata approach is the right direction, but this PR is not the best mergeable fix because it misses current-main skill metadata and includes an unrelated xurl package-installer activation.
