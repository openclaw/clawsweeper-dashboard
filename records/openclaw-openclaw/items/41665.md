## Summary

Keep open. The branch uses the right metadata seam, but it is stale against current main: newer bundled skills still have unscoped Homebrew installers, removed skill paths are still touched, and xurl's `npm` to `node` change expands installer behavior beyond the stated OS-filter scope.

## What This Changes

The PR adds per-installer OS metadata to bundled skill Homebrew install specs across 21 `SKILL.md` files and changes `skills/xurl`'s fallback installer from `npm` to `node`.

## Best Possible Solution

Refresh or replace the branch with a current-main inventory audit of bundled Homebrew install specs, add OS metadata where appropriate, split or explicitly approve the xurl node installer activation, and add a small regression guard for unscoped Homebrew installers.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: on `win32`, unscoped install specs survive `normalizeInstallOptions`, and onboarding lists missing skills whose `install.length > 0`; current main still has representative unscoped Homebrew specs.

## Solution Assessment

No. The metadata approach is the right direction, but this PR is not the best mergeable fix because it misses current-main skill metadata and includes an unrelated xurl package-installer activation.
