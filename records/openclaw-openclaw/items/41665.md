## Summary

Keep open: the metadata direction is valid, but this PR is stale against current main and would still leave Windows onboarding with unscoped Homebrew installers; it also changes xurl's installer kind outside the stated OS-filter scope.

## What This Changes

The branch adds per-installer `os` filters to Homebrew entries in bundled skill manifests and changes `skills/xurl` fallback installer metadata from `npm` to `node`.

## Best Possible Solution

Refresh or replace the branch against current main with a complete audit of bundled Homebrew install specs, keep xurl installer activation split or explicitly approved, and add focused regression coverage for platform-filtered install options.

## Reproduction Assessment

Yes. Source inspection shows unscoped install specs survive platform filtering and onboarding lists missing skills with install options; current main still has representative unscoped Homebrew specs. I did not run the Windows onboarding flow in this read-only review.

## Solution Assessment

No. Per-installer `os` metadata is the right narrow mechanism, but this PR is not the best mergeable fix because it misses current-main skill metadata and mixes in xurl installer activation.
