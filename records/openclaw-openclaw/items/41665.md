## Summary

Keep open: the OS-filter mechanism is the right direction, but this stale PR is incomplete against current main, still needs real Windows proof, and mixes in an xurl installer behavior change outside the stated metadata-only scope.

## What This Changes

The PR adds installer-level OS metadata to Homebrew-backed skill definitions and changes xurl's fallback installer metadata from `npm` to `node`.

## Best Possible Solution

Keep installer-level OS metadata as the mechanism, update all current brew-only or platform-limited install specs with filters or native Windows installers, and leave xurl's npm-to-node activation to a separate reviewed change.

## Reproduction Assessment

Yes, source-reproducible: `normalizeInstallOptions` keeps install specs with no `os` on every platform, and onboarding lists missing skills that still have install options. I did not run a live Windows Git Bash onboarding flow in this read-only review.

## Solution Assessment

No. Installer-level `os` metadata is the narrow maintainable mechanism, but this branch needs to be refreshed for current main and should split or explicitly review the xurl package-installer activation.
