## Summary

Keep this PR open. The requested Windows onboarding fix is valid and uses the documented OS-filter mechanism, but the branch is stale against current main: unscoped Homebrew installer specs still exist in bundled skills that this PR does not cover, and the xurl installer-kind change needs explicit maintainer review because it activates a package-manager install path.

## What This Changes

The PR adds per-installer OS metadata to bundled skill Homebrew install entries and changes the xurl fallback installer kind from `npm` to `node`.

## Best Possible Solution

Refresh the branch against current main, audit every bundled `kind: "brew"` installer, add appropriate top-level or per-installer OS metadata, preserve or add Windows-capable installers only when they are intentionally supported, and add a small inventory test for unscoped bundled Homebrew specs.

## Reproduction Assessment

Yes. I did not run a live Windows onboarding session during this read-only review.

## Solution Assessment

No. The proposed direction is correct, but the current PR is not the best mergeable form because it is stale against the bundled skill inventory and mixes an xurl package-installer activation into an OS-filter cleanup. A refreshed metadata audit plus an inventory regression test would be safer.
