## Summary

Keep open: the OS-metadata approach matches the existing install-filter contract, but the branch is stale against current main and would still leave newer unscoped Homebrew installers visible during Windows onboarding; the xurl installer-kind change also needs explicit review.

## What This Changes

The branch adds per-installer OS metadata to Homebrew install specs in bundled skill `SKILL.md` files and changes `skills/xurl` fallback installer metadata from `npm` to `node`.

## Best Possible Solution

Refresh or replace the branch with a current-main inventory audit of bundled Homebrew install specs, add OS metadata where appropriate, split or explicitly approve the xurl node installer activation, and add a small regression guard for unscoped Homebrew installers.

Yes. A high-confidence source reproduction exists: on `win32`, unscoped install specs survive `normalizeInstallOptions`, and onboarding lists missing skills whose `install.length > 0`; current main still has representative unscoped Homebrew specs.

## Solution Assessment

No. The metadata direction is maintainable, but this branch is not the best mergeable fix because it misses current-main skill metadata and mixes in an unrelated xurl installer activation.
