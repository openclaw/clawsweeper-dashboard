## Summary

Keep open: current main does not have this WhatsApp listener API, and the active PR still needs correctness fixes, real lookup proof, and maintainer API/privacy review before merge.

## What This Changes

The PR adds a WhatsApp `ActiveWebListener.lookupPnLidEntry` type and implementation for resolving LID or phone inputs to phone-number results, with listener mock updates and a changelog entry.

## Best Possible Solution

If maintainers want this API, implement it by reusing or mirroring the existing LID resolver semantics, preserving authDir-first behavior, making hosted-LID support explicit, and documenting or gating the privacy-sensitive surface with redacted live proof.

## Reproduction Assessment

Yes. Source inspection of PR head shows `lookupPnLidEntry` calls Baileys before authDir fallback, while current resolver code and tests require local mapping to win; Baileys source also shows hosted LIDs are not accepted by `getPNForLID`.

## Solution Assessment

No. The API may be useful, but this patch should mirror existing resolver ordering, make hosted-LID behavior explicit, and receive privacy/API review plus real lookup proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Prefer authDir mappings before Baileys lookup:** `extensions/whatsapp/src/inbound/monitor.ts:934`
  - body: The new method calls `lidLookup.getPNForLID` before `jidToE164(..., { authDir })`. Current inbound resolution first uses local reverse mappings and the existing monitor test asserts that path avoids Baileys, so this API can resolve differently or hit Baileys unnecessarily when a local mapping exists.
  - confidence: 0.86
- **[P2] Make hosted-LID lookup explicit:** `extensions/whatsapp/src/inbound/monitor.ts:934`
  - body: This method classifies `@hosted.lid` as a LID but sends it to Baileys `getPNForLID`; Baileys v7.0.0-rc10 skips inputs that are not `@lid`. Hosted LID inputs therefore only work if the local reverse file is present, so the API needs an explicit hosted path or a narrower contract.
  - confidence: 0.84
