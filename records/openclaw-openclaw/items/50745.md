## Best Possible Solution

Keep this as a WhatsApp-owned API, but implement it through the existing Baileys/authDir bidirectional PN/LID mapping contract, expose the new surface additively through supported barrels, and require real resolved lookup proof before merge.

## Solution Assessment

No, not yet. The maintainable shape is to reuse the existing WhatsApp mapping helpers and Baileys methods for both directions, then make the public listener change additive or explicitly approved as a breaking API change.
