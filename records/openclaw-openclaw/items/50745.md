- **[P2] Prefer authDir mappings before Baileys lookup:** `extensions/whatsapp/src/inbound/monitor.ts:934-938`
  - body: The new method calls `lidLookup.getPNForLID` before checking the local authDir reverse mapping. Current WhatsApp resolution first uses `jidToE164(..., { authDir })`, and tests assert local mappings avoid Baileys, so this can resolve differently or consult Baileys when local state already has the answer.
  - confidence: 0.86

  - body: `@hosted.lid` inputs are classified as LIDs but still go through `getPNForLID`; Baileys v7 filters that API to regular `@lid` users, with hosted LIDs represented by a separate predicate. Hosted support should use the local reverse mapping path first or the public contract should be narrowed.
  - confidence: 0.78
