## What This Changes

The PR adds `CronService.stopGraceful()`, tracks in-flight cron timer/manual runs, awaits the drain during gateway hot reload, and adds cron regression tests.

## Best Possible Solution

Land a rebased graceful cron restart that updates the cron contract and all implementations, bounds drain waiting with fallback logging, keeps focused regression coverage, and includes redacted `config.patch` proof.

## Reproduction Assessment

Yes. Source inspection shows current main stops and rebuilds cron synchronously while timer and manual cron execution can later re-enter persistence paths; I did not run a live `config.patch` reproduction in this read-only review.

## Solution Assessment

No. The graceful-drain direction is appropriate, but this branch needs a current-main rebase through the cron contract/lazy wrapper and a bounded wait policy before it is the best mergeable fix.
