## Summary

Keep open. The PR targets a real cron hot-reload race, but current main does not already contain the proposed graceful-stop behavior, the branch is currently dirty against main, and the patch still introduces an unbounded hot-reload wait that should be fixed before merge.

## What This Changes

The PR adds a `CronService.stopGraceful()` path, tracks in-flight timer and manual cron run promises, switches cron hot reload to await graceful shutdown, and adds regression tests for locked, manual-run, and timer-tick drains.

## Best Possible Solution

Land a rebased cron-owned fix that prevents old cron instances from writing stale state after hot reload while preserving current jobs.json/jobs-state.json semantics, with a bounded drain or fallback tied to the existing reload deferral policy and regression coverage for locked operations, manual runs, timer ticks, and the timeout fallback.

## Reproduction Assessment

Yes. The target race can be exercised by triggering a cron hot reload while a timer or manual cron run is suspended in its worker phase, and the PR's added tests model those deferred worker and post-worker persist windows; the new unbounded-wait finding can be reproduced with an active cron agent turn that has `timeoutSeconds <= 0` and a `cron.*` config reload.

## Solution Assessment

No, not as-is. Draining tracked cron work before replacement is the right direction, but the hot-reload path needs a bounded wait or fallback so one long-lived cron run cannot prevent config changes from applying forever.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Bound the graceful cron drain before reload:** `src/gateway/server-reload-handlers.ts:137`
  - body: The new reload path awaits `stopGraceful()`, and `stopGraceful()` waits on `Promise.allSettled(pendingRuns)` with no deadline. Since cron jobs can explicitly disable their timeout with `timeoutSeconds <= 0`, one hung or long-lived run can keep a `cron.*` config reload from ever applying. Please bound this wait or fall back after the configured reload deferral window.
  - confidence: 0.9
