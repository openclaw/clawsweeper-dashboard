## Summary

Keep open. Current main has adjacent plugin hooks for `agent_end`, `after_tool_call`, and `subagent_ended`, but the requested workspace/internal hook events and `toolCallThreshold` configuration are still absent, and related open work shows this is an active API-design area rather than stale cleanup.

## What This Changes

The issue requests workspace/internal gateway lifecycle hooks for subagent completion, agent turn completion, and configurable tool-call thresholds.

## Best Possible Solution

Choose one documented lifecycle contract: either bridge selected plugin lifecycle events into workspace/internal hooks with typed, minimized payloads and tests, or explicitly route these use cases to plugin hooks and hook packs while preserving security boundaries.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source and docs inspection on current main and v2026.5.7 give high-confidence evidence that the requested internal workspace events and threshold config are absent.

## Solution Assessment

Unclear as proposed. The `subagent:ended` bridge is mechanically small, but the exact internal-vs-plugin surface, payload limits, threshold config, and hook ordering are API and security decisions best settled by maintainers first.
