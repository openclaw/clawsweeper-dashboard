## Summary

Keep open. Current main and v2026.6.6 still lack the requested workspace/internal lifecycle hook contract; existing plugin and context-engine hooks are adjacent but not a substitute, and related open issues only cover slices of the broader API design.

## What This Changes

The issue requests workspace/internal lifecycle hooks for subagent completion, turn completion, and tool-call threshold events.

## Best Possible Solution

Approve one bounded lifecycle API: start with a minimized subagent completion contract if maintainers want it, then separately decide whether turn/tool-count data belongs in internal hooks, typed plugin hooks, context-engine callbacks, or diagnostics.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 expose adjacent plugin/context-engine lifecycle seams but do not expose the requested HOOK.md workspace/internal subagent, turn-complete, or tool-call-threshold events. This is a missing capability rather than a broken documented behavior.

## Solution Assessment

Unclear pending maintainer API design. Bridging subagent completion into internal hooks is plausible, but the broader turn-complete and threshold parts need owner-boundary, payload, timing, config, and security decisions before a fix PR should be queued.
