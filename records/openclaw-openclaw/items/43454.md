## Summary

Keep open. Current main and v2026.5.12 still lack the requested workspace/internal lifecycle events; related comments add useful memory/workaround context, but this remains a product and security design request rather than a cleanup-close candidate.

## What This Changes

Request typed gateway/workspace lifecycle hooks for subagent completion, completed agent turns, and tool-call thresholds so automations can react without prompt-only or polling workarounds.

## Best Possible Solution

Define one canonical lifecycle-hook contract with minimal typed payloads, explicit opt-in and security boundaries, ordering and failure semantics, focused tests, and docs before wiring subagent, turn-complete, or threshold events.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source, docs, tests, and the latest release show the requested workspace lifecycle events are not present while adjacent plugin hooks exist.

## Solution Assessment

Unclear as proposed. Bridging subagent_ended into internal hooks is mechanically plausible, but the full request needs maintainer decisions on surface ownership, payload limits, threshold config, ordering, timeout, and failure behavior.
