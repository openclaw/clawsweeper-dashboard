## Summary

Keep open. Current main still has the workspace/internal hook gap for `subagent:ended`, `agent:turnComplete`, and `agent:toolCallThreshold`; Plugin SDK lifecycle hooks and context-engine callbacks exist nearby, but they do not give `HOOK.md` workspace hooks the requested lifecycle events.

## What This Changes

Expose workspace/internal lifecycle hooks for subagent completion, agent turn completion, and tool-call threshold events.

## Best Possible Solution

Define a maintainer-approved lifecycle hook contract, starting with a narrow documented `subagent:ended` workspace/internal bridge and tests, then split turn-complete and tool-threshold semantics into explicit follow-up API design.

## Reproduction Assessment

Yes for the feature gap: static inspection on current main shows workspace hooks can register event strings, but there is no internal emitter or documented event for the requested lifecycle points. The adjacent `subagent_ended`, `agent_end`, and tool hooks are Plugin SDK surfaces, not `HOOK.md` workspace events.

## Solution Assessment

Unclear. The Phase 1 bridge is plausible, but the broader turn-complete and mid-turn threshold behavior should be designed around payload privacy, hook ordering, prompt mutation, and configuration semantics before implementation is queued.
