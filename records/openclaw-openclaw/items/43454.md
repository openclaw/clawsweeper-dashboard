## Summary

Keep open. Current main still lacks the requested workspace/internal lifecycle events; adjacent plugin hooks exist, but they are not the standard HOOK.md workspace surface and do not define the proposed turn-complete or tool-call-threshold semantics.

## What This Changes

The issue requests new workspace/internal gateway lifecycle hooks for subagent completion, agent turn completion, and tool-call threshold events.

## Best Possible Solution

Define one documented lifecycle contract with minimized typed payloads, explicit opt-in, ordering, timeout/failure behavior, regression tests, and docs; if plugin hooks are the intended surface, document that instead of adding parallel workspace semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main gives high-confidence evidence that the requested workspace/internal events and threshold semantics are absent while adjacent plugin hooks exist.

## Solution Assessment

Unclear as proposed. Bridging subagent completion into internal hooks is mechanically plausible, but internal-versus-plugin ownership, payload limits, threshold configuration, ordering, timeout/failure behavior, and privacy boundaries need maintainer design first.
