## Summary

Keep open. Current main and v2026.5.12 still lack the requested workspace/internal lifecycle events; adjacent plugin hooks exist, but they do not provide the proposed HOOK.md subagent event, turn-complete event, tool-call threshold event, or threshold config, and the API needs maintainer product and security design.

## What This Changes

Request adding workspace/internal lifecycle hooks for subagent completion, agent turn completion, and tool-call threshold events so hook authors can react to agent lifecycle points.

## Best Possible Solution

Define a documented lifecycle hook contract with minimized typed payloads, explicit opt-in and security boundaries, ordering, timeout/failure semantics, tests, and docs; decide whether plugin hooks or internal HOOK.md hooks are the canonical surface before implementation.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source, docs, tests, and the latest release show that current OpenClaw lacks the requested internal lifecycle events while exposing only adjacent plugin hooks.

## Solution Assessment

Unclear as proposed. Bridging subagent_ended into internal hooks is mechanically plausible, but the full request needs maintainer decisions on surface ownership, payload limits, threshold config, ordering, timeout, and failure behavior.
