## Summary

Keep open. Current main and the latest release still lack the requested workspace/internal lifecycle events; adjacent plugin hooks exist, but they are not the same HOOK.md workspace surface and the request needs API, product, and security design.

## What This Changes

Expose workspace/internal lifecycle hook events for subagent completion, agent turn completion, and tool-call thresholds so operator hooks can react without polling or prompt rules.

## Best Possible Solution

Define a documented lifecycle hook contract with minimized typed payloads, explicit opt-in and security boundaries, timeout/failure behavior, ordering semantics, tests, and docs; if plugin hooks are the intended surface, document that route instead of adding parallel workspace events.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source and release inspection give high confidence that current main lacks the requested workspace/internal lifecycle events while adjacent plugin hooks exist.

## Solution Assessment

Unclear as proposed. Bridging subagent_ended into internal hooks is mechanically plausible, but maintainers need to decide workspace-versus-plugin ownership, payload limits, threshold configuration, ordering, timeout, and failure behavior first.
