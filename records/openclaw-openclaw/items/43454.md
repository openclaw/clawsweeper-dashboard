## Summary

Keep open for maintainer API and security design. Current main and v2026.6.6 still expose adjacent plugin and context-engine seams, but they do not provide the requested workspace/internal `subagent:ended`, turn-complete, or tool-call-threshold hook contract; the active turn-hook PRs only cover part of the broader request.

## What This Changes

The issue requests standard workspace/internal lifecycle hook events for subagent completion, agent turn completion, and tool-call thresholds.

## Best Possible Solution

Approve one deliberate lifecycle contract that separates operator workspace hooks from typed plugin/runtime hooks, defines bounded payloads for subagent completion and turn completion, and treats mid-turn thresholds as opt-in only if maintainers accept the timing and security tradeoff.

## Reproduction Assessment

Yes at source level: current main exposes adjacent plugin and context-engine lifecycle seams, but the standard workspace/internal hook event family lacks `subagent` and there are no turn-complete or tool-call-threshold internal events. This is a missing capability rather than a broken documented behavior.

## Solution Assessment

Unclear pending maintainer API design. Bridging subagent completion into workspace hooks is plausible, but the broader request also needs decisions on whether turn completion belongs in internal hooks, plugin hooks, context engines, or diagnostic events, plus how to bound sensitive payloads.
