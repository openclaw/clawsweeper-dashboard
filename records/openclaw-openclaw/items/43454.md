## Summary

Keep open. Current main still lacks the requested workspace/internal lifecycle hooks and tool-call threshold semantics; adjacent plugin hooks exist, but they are not equivalent, and the active related hook discussions make this an API/security design item.

## What This Changes

Request gateway/workspace lifecycle hooks for subagent completion, agent turn completion, and tool-call thresholds so workspace hooks can react without polling or prompt rules.

## Best Possible Solution

Define one documented lifecycle contract with minimized typed payloads, explicit opt-in, timeout/failure behavior, tests, and docs; if workspace hooks are not the intended surface, document plugin hooks as the supported route.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main gives high-confidence evidence that the requested workspace/internal events and threshold semantics are absent while adjacent plugin hooks exist.

## Solution Assessment

Unclear as proposed. Bridging subagent_ended into internal hooks is mechanically plausible, but internal-versus-plugin ownership, payload limits, ordering, timeout/failure behavior, and threshold semantics need maintainer API and security judgment first.
