## Summary

Keep open. Current main still lacks the requested workspace/internal lifecycle hook events; adjacent Plugin SDK hooks cover similar runtime moments but do not make `subagent:ended`, `agent:turnComplete`, or `agent:toolCallThreshold` available to `HOOK.md` workspace hooks.

## What This Changes

Request exposing subagent completion, turn-complete, and tool-call-threshold lifecycle events through workspace/internal hooks.

## Best Possible Solution

Define a maintainer-approved workspace lifecycle hook contract, start with a documented and tested `subagent:ended` internal-hook bridge, and split turn-complete plus threshold behavior into explicit follow-up design.

## Reproduction Assessment

Not applicable as a feature request. Static inspection on current main gives high-confidence evidence that workspace hooks can register event strings, but no internal emitter or documented event exists for the requested lifecycle points.

## Solution Assessment

Unclear. The narrow `subagent:ended` bridge is plausible, but the broader turn-complete and mid-turn threshold surfaces should be designed around payload privacy, hook ordering, prompt/message mutation, and configuration semantics before coding.
