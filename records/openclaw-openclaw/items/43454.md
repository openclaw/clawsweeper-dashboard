## Summary

Keep open: current main and v2026.6.10 still expose subagent completion only through the typed plugin `subagent_ended` hook, while workspace internal hooks have no `subagent` event type, documented `subagent:ended` event, turn-complete event, or tool-call threshold event. The request fits OpenClaw's extension-point direction, but it needs maintainer product and security judgment because it adds lifecycle API/config surface at agent and session-state boundaries.

## What This Changes

Request to add workspace/gateway lifecycle hook events for subagent completion, agent turn completion, and tool-call thresholds, including typed context and optional threshold config.

## Best Possible Solution

Keep this open until maintainers choose the lifecycle API split, then land a narrow typed contract with security/ordering docs and focused tests for subagent completion plus turn/tool boundaries.

## Reproduction Assessment

Yes. Source inspection on current main shows the internal hook event union and docs lack the requested `subagent:ended`, turn-complete, and tool-call threshold events, while subagent completion only emits through the plugin hook runner.
