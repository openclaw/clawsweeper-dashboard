## Summary

Keep open. Current main still lacks the requested internal/workspace lifecycle events and tool-call threshold semantics; adjacent plugin hooks exist, but they are not equivalent, and the active related discussion makes this an API/security design item rather than cleanup-ready stale work.

## What This Changes

Requests gateway/internal lifecycle hooks for workspace automations, including subagent completion, turn completion, and tool-call threshold events.

## Best Possible Solution

Define one documented lifecycle contract with minimized typed payloads, explicit opt-in and timeout/failure behavior, tests, and docs; if workspace hooks are not the intended surface, document plugin hooks as the supported route for these use cases.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main gives high-confidence evidence that the requested internal/workspace events and threshold semantics are absent while adjacent plugin hooks exist.

## Solution Assessment

Unclear as proposed. Bridging subagent_ended into internal hooks is mechanically plausible, but internal-versus-plugin ownership, payload limits, ordering, timeout/failure behavior, and threshold semantics need maintainer API and security judgment first.
