## Summary

Keep open. Current main still lacks the requested workspace/internal lifecycle events and tool-call threshold semantics; adjacent plugin hooks exist, but they are not equivalent to `subagent:ended`, `agent:turnComplete`, or `agent:toolCallThreshold` for workspace hooks, and the recent comments plus related open work show this still needs maintainer API and security design.

## What This Changes

Add workspace/internal lifecycle hook events for subagent completion, agent turn completion, and tool-call threshold handling.

## Best Possible Solution

Define a documented lifecycle hook contract with minimized typed payloads, explicit opt-in and security boundaries, timeout and failure behavior, ordering semantics, tests, and docs; if workspace hooks are not the intended surface, document plugin hooks as the supported route.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main gives high-confidence evidence that the requested workspace/internal events and threshold semantics are absent while adjacent plugin hooks exist.

## Solution Assessment

Unclear as proposed. Bridging `subagent_ended` into internal hooks is mechanically plausible, but maintainers need to decide internal-versus-plugin ownership, payload limits, threshold configuration, ordering, timeout, and failure behavior first.
