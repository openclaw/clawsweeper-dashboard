## Summary

Keep open. Current main has adjacent plugin lifecycle hooks for `agent_end`, `after_tool_call`, and `subagent_ended`, but the requested workspace/internal hook events and `toolCallThreshold` configuration are still absent, and the related open discussions show this remains an active API-design question rather than cleanup-ready stale work.

## What This Changes

The issue requests workspace/internal lifecycle hooks for subagent completion, agent turn completion, and tool-call threshold events.

## Best Possible Solution

Define one documented lifecycle contract with minimized typed payloads, explicit opt-in and timeout behavior, tests, and docs; if workspace hooks are not the intended surface, document that plugin hooks are the supported route for these use cases.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source and docs inspection on current main gives high-confidence evidence that the requested internal/workspace events and threshold config are absent.

## Solution Assessment

Unclear as proposed. Bridging `subagent_ended` into internal hooks is mechanically plausible, but payload limits, hook ordering, threshold semantics, and internal-versus-plugin ownership need maintainer API and security judgment first.
