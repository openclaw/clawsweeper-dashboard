## Best Possible Solution

Define one canonical lifecycle-hook contract with minimal typed payloads, explicit opt-in/security boundaries, ordering and failure semantics, focused tests, and docs before wiring subagent, turn-complete, and threshold events.

Unclear as proposed. Bridging subagent_ended into internal hooks is mechanically plausible, but the full request needs maintainer decisions on surface ownership, payload limits, threshold config, ordering, timeout, and failure behavior.
