## Summary

Keep open for maintainer API and security design. Current main has adjacent plugin, context-engine, and workspace/internal hook infrastructure, but it still does not expose the requested workspace/internal `subagent:ended`, turn-complete, or tool-call-threshold lifecycle events; a lower-detail turn-event issue was already closed in favor of this issue as the canonical broader thread.

## What This Changes

The issue requests new Gateway lifecycle hook events for workspace automation: subagent completion, agent turn completion, and optional tool-call threshold triggers.

## Best Possible Solution

Approve one deliberate lifecycle contract that separates operator workspace hooks from typed plugin/runtime hooks, defines bounded payloads for subagent completion and turn completion, and treats mid-turn thresholds as opt-in only if maintainers accept the timing and security tradeoff.

## Reproduction Assessment

Yes at source level: current main exposes adjacent plugin `subagent_ended` and context-engine `onSubagentEnded`/`afterTurn` seams, but the standard workspace/internal hook event family lacks `subagent` and there are no turn-complete or tool-call-threshold internal events. This is a missing capability rather than a broken documented behavior.

## Solution Assessment

Unclear pending maintainer API design. Bridging subagent completion into workspace hooks is plausible, but the broader request also needs decisions on whether turn completion belongs in internal hooks, plugin hooks, context engines, or diagnostic events, plus how to bound sensitive payloads.
