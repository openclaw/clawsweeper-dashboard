## Summary

Keep open. Current main still lacks the requested workspace/internal lifecycle events; adjacent Plugin SDK hooks and related open issues cover pieces of the lifecycle space, but not this workspace hook API or the turn-complete/threshold contract.

## What This Changes

The issue requests workspace/internal lifecycle hooks for subagent completion, agent turn completion, and tool-call thresholds.

## Best Possible Solution

Define a maintainer-approved, documented workspace lifecycle hook contract, likely splitting a `subagent:ended` bridge from separate turn-complete and threshold design work.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main gives high-confidence evidence that workspace hooks can register event strings but the requested lifecycle event types and emitters are absent.

## Solution Assessment

Unclear as a final API. The narrow subagent bridge is plausible, but turn-complete and threshold hooks need maintainer decisions on payload privacy, ordering, mutation rights, and configuration semantics.

## Security Review

Status: needs_attention

Summary: The requested hook surfaces could expose run content and add mid-turn automation, so the trust boundary needs maintainer review.

Concerns:

- **[medium] Define workspace lifecycle data exposure boundaries:**
  - body: A `subagent:ended` or turn-complete workspace hook could expose subagent output, turn metadata, or tool-call details to local hook code, so the contract needs explicit data minimization, opt-in, timeout, and logging rules.
  - confidence: 0.86
- **[medium] Set mid-turn mutation and threshold semantics:**
  - body: A tool-call threshold hook could fire during an active turn and influence later prompt/session behavior, so ordering, one-shot behavior, and handler failure semantics need product and security review before implementation.
  - confidence: 0.8
