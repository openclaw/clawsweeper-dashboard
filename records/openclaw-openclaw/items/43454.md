## Summary

Keep open. Current main still lacks the requested workspace/internal lifecycle events and tool-call threshold semantics; adjacent plugin hooks exist, but they are not equivalent to `subagent:ended`, `agent:turnComplete`, or `agent:toolCallThreshold` for workspace hooks, and the request needs API/product/security design.

## What This Changes

Request to add workspace/internal Gateway lifecycle hooks for subagent completion, turn completion, and configurable tool-call threshold events.

## Best Possible Solution

Define one documented lifecycle hook contract with minimized typed payloads, explicit opt-in/security boundaries, timeout and failure behavior, tests, and docs; if workspace hooks are not the intended surface, document plugin hooks as the supported route.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection on current main gives high-confidence evidence that the requested workspace/internal events and threshold semantics are absent while adjacent plugin hooks exist.

## Solution Assessment

Unclear as proposed. Bridging subagent_ended into internal hooks is mechanically plausible, but maintainers need to decide internal-versus-plugin ownership, payload limits, threshold configuration, ordering, timeout, and failure behavior first.
