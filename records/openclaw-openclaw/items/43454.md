## Summary

Keep open. Current main still does not expose the requested workspace/internal lifecycle events; the nearby Plugin SDK hooks cover subagent completion, agent completion, and tool calls, but not as `HOOK.md` workspace events with a settled public payload/config contract.

## What This Changes

Requests workspace hook lifecycle events for subagent completion, agent turn completion, and tool-call threshold handling.

## Best Possible Solution

Define a maintainer-approved workspace lifecycle hook contract, start with a documented and tested `subagent:ended` bridge if approved, and split turn-complete plus threshold semantics into explicit follow-up design.

## Solution Assessment

Unclear. The narrow `subagent:ended` bridge is plausible, but the broader turn-complete and threshold surfaces need maintainer decisions on payload privacy, ordering, prompt/message mutation, and configuration semantics before implementation.
