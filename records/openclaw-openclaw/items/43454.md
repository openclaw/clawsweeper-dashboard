## Summary

Keep open. Current main and the latest release still lack the requested workspace/internal subagent, turn-complete, and tool-call-threshold lifecycle events; adjacent plugin hooks exist but are not equivalent, and the recent discussion/workarounds reinforce that this needs API, product, and security design rather than cleanup closure.

## What This Changes

Request adds workspace/internal Gateway lifecycle hooks for subagent completion, agent turn completion, and configurable tool-call threshold events.

## Best Possible Solution

Define a documented lifecycle hook contract with minimized typed payloads, explicit opt-in and security boundaries, timeout/failure behavior, ordering semantics, tests, and docs; if workspace hooks are not the intended surface, document plugin hooks as the supported route.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection gives high-confidence evidence that current main lacks the requested internal lifecycle events while adjacent plugin hooks exist.

## Solution Assessment

Unclear as proposed. Bridging subagent_ended into internal hooks is mechanically plausible, but maintainers need to decide internal-versus-plugin ownership, payload limits, threshold configuration, ordering, timeout, and failure behavior first.
