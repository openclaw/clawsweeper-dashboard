## Summary

Keep open: current `main` and v2026.6.6 still lack the requested workspace/internal lifecycle hook contract, and the only live implementation PR covers a partial agent/tool boundary set while remaining dirty and maintainer-owned.

## What This Changes

Request to expose workspace/internal lifecycle hooks for subagent completion, agent turn completion, and tool-call threshold events.

## Best Possible Solution

Approve one minimized lifecycle API boundary, starting with a narrow subagent completion event only if maintainers accept the payload and security model, then separately decide turn-complete and tool-threshold semantics.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 expose adjacent plugin/context-engine lifecycle seams but do not expose the requested HOOK.md workspace/internal subagent, turn-complete, or tool-call-threshold events. This is a missing capability rather than a broken documented behavior.

## Solution Assessment

Unclear pending maintainer API design. Bridging subagent completion into internal hooks is plausible, but the broader turn-complete and threshold pieces need owner-boundary, payload, timing, config, and security decisions first.

## Security Review

Status: needs_attention

Summary: The issue is security-sensitive because lifecycle hooks run trusted local code at more runtime points and may expose sensitive session, child-run, or tool metadata.

- **[medium] Minimize trusted hook payloads:** `src/hooks/loader.ts:46`

  - confidence: 0.9
