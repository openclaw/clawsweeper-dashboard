## Summary

Current v2026.6.6 and `main` already route QMD first and skip embedding readiness work for BM25 `search` mode. The issue should remain open because a QMD startup/open failure can trigger builtin fallback whose missing-provider error masks the original failure, but the report lacks the configuration and logs needed to confirm that path.

## What This Changes

Investigate and fix `memory_search` being disabled for QMD search when no embedding-provider credentials are configured.

## Best Possible Solution

Preserve embedding-independent QMD BM25 search and, if reproduction confirms fallback masking, return the primary QMD failure alongside any builtin embedding failure so users can diagnose the actual backend problem.

## Reproduction Assessment

No. Current v2026.6.6 and main use a QMD-first BM25 path, while this report omits the exact configuration, version, disabled payload, and preceding QMD warning needed to reproduce the claimed embedding gate.

## Solution Assessment

Unclear. A direct embedding-gate fix would duplicate behavior current code already has; the likely maintainable repair is preserving the primary QMD failure if diagnostics confirm fallback is masking it.
