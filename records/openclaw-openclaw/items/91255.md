## Summary

Keep open: the fix targets the right memory-core path, but the patch assumes `qmd collection show --json` returns JSON; the inspected upstream qmd v2.5.3 command prints text, so the enrichment can silently no-op and leave the linked rebind bug unfixed.

## What This Changes

The PR enriches missing QMD collection paths by running `qmd collection show <name> --json` from `listCollectionsBestEffort()` and copying a parsed `path` field into the listed collection metadata.

## Best Possible Solution

Parse the actual `collection show` text output or use a qmd surface that is proven to return machine-readable path metadata, then add a regression for stale same-name/same-pattern roots and provide redacted live qmd proof before merge.

## Reproduction Assessment

Yes. From current-main source, a listed collection without `path` and with a matching pattern makes `shouldRebindCollection()` return false, so a root-only workspace move cannot trigger the remove/add rebind.

## Solution Assessment

No. Enriching metadata before the rebind decision is the right layer, but this implementation relies on JSON output that upstream qmd v2.5.3 does not produce for `collection show`.
