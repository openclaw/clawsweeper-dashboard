## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Route ADC token exchange through guarded fetch:** `extensions/google/vertex-adc.ts:65`
  - body: The ADC helper mints authorized_user tokens with global `fetch`, and the service-account path below does the same, while the model request uses OpenClaw's guarded model fetch. Proxy-aware or restricted-network setups can route model traffic through the guard while OAuth exchange bypasses those controls, so thread the guarded or injected fetch into token minting before enabling this provider path.
  - confidence: 0.86
- **[P2] Key cached ADC tokens by credential source:** `extensions/google/vertex-adc.ts:151-152`
  - body: `resolveGoogleVertexAdcToken` returns any unexpired cached bearer token before reading the active ADC file, env, credential identity, refresh token, service account, or project. A long-running gateway that rotates credentials can keep sending Vertex requests with a previous account's token until expiry.
  - confidence: 0.88
