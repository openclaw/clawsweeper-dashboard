## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Pass guarded fetch into ADC token exchange:** `extensions/google/vertex-adc.ts:65`
  - body: The ADC helper mints authorized_user and service-account tokens through direct `fetch`, while the model request uses OpenClaw's guarded model fetch. Proxy-aware or restricted-network setups can route model traffic through the guard while token exchange bypasses it, so thread the guarded or injected fetch into token minting before using this provider path.
  - confidence: 0.86
- **[P2] Key ADC token cache by credential source:** `extensions/google/vertex-adc.ts:151-152`
  - body: `resolveGoogleVertexAdcToken` returns any unexpired cached token before reading the active ADC file, env, credential identity, refresh token, service account, or project. A long-running gateway that rotates credentials can keep sending Vertex requests with a previous account's bearer token until expiry.
  - confidence: 0.88
