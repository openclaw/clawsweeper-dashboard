## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Pass guarded fetch into ADC token exchange:** `extensions/google/vertex-adc.ts:65`
  - body: resolveGoogleVertexAdcToken mints authorized_user and service-account tokens through global fetch, while model traffic uses OpenClaw's guarded transport. Proxy-aware, restricted-network, and test callers can route model requests through the guard while credential exchange bypasses it, so thread the guarded or injected fetch into the token helper before minting.
  - confidence: 0.88
- **[P2] Key the ADC token cache by credential source:** `extensions/google/vertex-adc.ts:151-152`
  - body: The resolver returns any unexpired cached token before reading the active ADC file, credential type, refresh token, service-account identity, project, or env source. A long-running gateway that rotates credentials can keep sending Vertex requests with the previous account's bearer token until expiry.
  - confidence: 0.87
- **[P2] Resolve templated Vertex hosts before composing URLs:** `extensions/google/transport-stream.ts:232`
  - body: new URL() accepts https://{location}-aiplatform.googleapis.com, so this branch can preserve {location} as the literal request host. Resolve the template with the selected region or global endpoint before returning the origin and composing the Vertex path.
  - confidence: 0.82
