## What This Changes

The PR adds a standalone Google Vertex provider path in the Google plugin with ADC auth, Vertex endpoint routing, model catalog and region helpers, transport tests, and unrelated oc-path formatting churn.

## Best Possible Solution

Land one Google-owned Vertex path with guarded ADC token exchange, credential-scoped caching, focused tests, config/setup alignment, and redacted live-run proof after maintainer alignment on the provider direction.

## Reproduction Assessment

Yes for the blocking review findings: PR-head source shows token exchange uses global fetch and returns a cached token before reading the active ADC source. The end-to-end Google Vertex behavior itself still lacks inspectable real-run proof.

## Solution Assessment

No, not as-is. The Google-plugin-owned Vertex direction is plausible, but this branch needs guarded credential exchange, credential-scoped caching, real behavior proof, and maintainer alignment with the overlapping Google provider work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Pass injected fetch into ADC token minting:** `extensions/google/vertex-adc.ts:65`
  - body: The ADC helper mints authorized_user tokens through global `fetch`, and service-account minting below uses the same path, while model requests go through `buildGuardedModelFetch`. Proxy-aware or restricted-network setups can route model traffic through OpenClaw's guard while OAuth token exchange bypasses it, so thread the guarded or injected fetch into token minting before using this provider path.
  - confidence: 0.88
- **[P2] Key the ADC token cache by credential source:** `extensions/google/vertex-adc.ts:151-152`
  - body: `resolveGoogleVertexAdcToken` returns any unexpired cached token before reading the active ADC file, env, credential identity, refresh token, service account, or project. A long-running gateway that rotates credentials can keep sending Vertex requests with a previous account's bearer token until expiry, so read the active source first and key the cache by stable credential identity.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: Needs attention because the PR adds ADC token minting that bypasses guarded transport and can reuse an old account token after credential rotation.

  - body: Token minting uses global `fetch` rather than the guarded or injected fetch path used for model requests, so proxy and network-policy controls can apply to model traffic but not OAuth token exchange.
