## Summary

Keep open. This PR targets real remaining Google Vertex gaps, but the latest head is not mergeable as-is because ADC token exchange still bypasses guarded transport, token caching is not credential-scoped, templated Vertex hosts can survive into requests, and contributor real behavior proof is still missing.

## What This Changes

The branch adds a standalone Google Vertex provider in the Google plugin with ADC auth, Vertex endpoint/project/location routing, a model catalog, thought-signature replay handling, manifest metadata, and focused tests.

## Best Possible Solution

Consolidate on one Google Vertex path that keeps provider ownership in the Google plugin, uses guarded token exchange, scopes ADC token caching to the active credentials, normalizes endpoints before request composition, supports the intended ADC types, and includes focused tests plus redacted live-run proof.

## Reproduction Assessment

Yes. Source inspection at PR head gives high-confidence reproduction paths for the blockers: token minting uses global fetch, the ADC cache is returned before checking the active credential source, and templated Vertex hosts can be preserved literally.

## Solution Assessment

No, not as-is. A dedicated Google-plugin-owned Vertex implementation is plausible, but this branch needs guarded token exchange, credential-scoped caching, endpoint-template normalization, real behavior proof, and maintainer direction against the open Google GenAI path before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Pass guarded fetch into ADC token exchange:** `extensions/google/vertex-adc.ts:65`
  - body: resolveGoogleVertexAdcToken mints both authorized_user and service-account tokens through global fetch, while model requests use OpenClaw's guarded/injected transport. Proxy-aware, restricted-network, and test callers can route model traffic through the guard while credential exchange bypasses it, so thread the guarded or injected fetch into the token helper before minting.
  - confidence: 0.88
- **[P2] Key the ADC token cache by credential source:** `extensions/google/vertex-adc.ts:151-152`
  - body: The resolver returns any unexpired cachedToken before reading the active env, ADC path, refresh token, service-account identity, or project metadata. A long-running gateway that rotates credentials can keep sending Vertex requests with the previous account's bearer token until expiry.
  - confidence: 0.87
- **[P2] Resolve templated Vertex hosts before composing URLs:** `extensions/google/transport-stream.ts:228-229`
  - body: new URL() accepts https://{location}-aiplatform.googleapis.com, so this branch can preserve {location} as the literal request host. Resolve the template with the selected region or global endpoint before returning the origin and composing the Vertex path.
  - confidence: 0.82
