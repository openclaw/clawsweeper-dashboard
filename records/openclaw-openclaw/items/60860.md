## Summary

Keep open. This PR is a plausible implementation path for the open Google Vertex work, but current main only partially covers that area and this branch still has blocking ADC transport/cache bugs plus missing real behavior proof.

## What This Changes

The PR adds a dedicated Google Vertex AI provider path in the Google plugin with ADC auth, Vertex endpoint routing, provider catalog/registration changes, and focused tests.

## Best Possible Solution

Land or supersede one Google-plugin-owned Vertex implementation that preserves guarded token exchange, scopes ADC token caching to credential identity, normalizes regional/global/template endpoints, covers both authorized_user and service_account ADC, and includes focused tests plus redacted live-run proof.

## Reproduction Assessment

Yes. Source inspection gives high-confidence reproduction paths: use an injected or guarded fetch, rotate ADC credentials while the token cache is warm, or configure a templated Vertex base URL and PR head bypasses the guard, reuses the prior token, or emits a literal templated host.

## Solution Assessment

No, not as-is. A dedicated Google Vertex provider is a maintainable direction, but this branch needs guarded-fetch token minting, credential-scoped caching, endpoint-template normalization, and real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Pass guarded fetch into ADC token minting:** `extensions/google/vertex-adc.ts:65`
  - body: `resolveGoogleVertexAdcToken()` mints ADC tokens with global `fetch` before the model request constructs OpenClaw's guarded transport. Proxy-aware, restricted-network, and test callers can route the Vertex request through the guard while the credential exchange bypasses it; pass an injected or guarded fetch into the token helper before minting.
  - confidence: 0.88
- **[P2] Key the ADC token cache by credential source:** `extensions/google/vertex-adc.ts:151-152`
  - body: The resolver returns any unexpired `cachedToken` before reading the active env, ADC path, refresh token, service-account identity, or project metadata. A long-running gateway that rotates credentials can keep sending Vertex requests with the previous account's bearer token until expiry.
  - confidence: 0.87
- **[P2] Normalize templated Vertex hosts before composing URLs:** `extensions/google/transport-stream.ts:228-229`
  - body: `new URL()` accepts `https://{location}-aiplatform.googleapis.com`, so this branch can preserve `{location}` as the literal request host and location. Resolve the template with the selected region before returning the origin or composing the Vertex path.
  - confidence: 0.82
