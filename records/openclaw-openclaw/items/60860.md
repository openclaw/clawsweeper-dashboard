## Summary

Keep open. The PR still contains useful Google Vertex work that is not fully present on current main, but it is not mergeable as-is because the latest head still has guarded-transport, token-cache, endpoint-template, and real behavior proof blockers.

## What This Changes

The branch adds a dedicated Google Vertex provider in the Google plugin with ADC auth, Vertex endpoint routing, model catalog/registration changes, thought-signature handling, and focused tests.

## Best Possible Solution

Choose one Google Vertex implementation path and land a guarded Google-plugin-owned provider that supports project/location resolution, authorized_user and service_account ADC, endpoint normalization, focused regression tests, and redacted live-run proof.

## Reproduction Assessment

Yes. Source inspection of the PR head gives concrete reproduction paths for the blockers: token exchange bypasses guarded fetch, the cache can reuse a prior ADC token after credential changes, and a templated Vertex base URL can survive as a literal request host.

## Solution Assessment

No, not as-is. A dedicated Google-plugin-owned Vertex path is plausible, but this branch needs guarded token exchange, credential-scoped caching, endpoint-template normalization, and real behavior proof before merge; maintainers may also consolidate with the open Google GenAI direction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Pass guarded fetch into ADC token exchange:** `extensions/google/vertex-adc.ts:65`
  - body: `resolveGoogleVertexAdcToken` mints both authorized_user and service-account tokens through global `fetch`, while the model request uses OpenClaw's guarded/injected transport. Proxy-aware, restricted-network, and test callers can route model traffic through the guard while credential exchange bypasses it, so thread the guarded or injected fetch into the token helper before minting.
  - confidence: 0.88
- **[P2] Key the ADC token cache by credential source:** `extensions/google/vertex-adc.ts:151-152`
  - body: The resolver returns any unexpired `cachedToken` before reading the active env, ADC path, refresh token, service-account identity, or project metadata. A long-running gateway that rotates credentials can keep sending Vertex requests with the previous account's bearer token until expiry.
  - confidence: 0.87
- **[P2] Resolve templated Vertex hosts before composing URLs:** `extensions/google/transport-stream.ts:228-229`
  - body: `new URL()` accepts `https://{location}-aiplatform.googleapis.com`, so this branch can preserve `{location}` as the literal request host. Resolve the template with the selected region or global endpoint before returning the origin and composing the Vertex path.
  - confidence: 0.82
