## Summary

Keep open. Current main is not equivalent to this PR, and the PR still has concrete auth/routing blockers before it is safe to merge.

## What This Changes

The PR adds a dedicated `google-vertex` provider in the Google plugin with ADC token minting, Vertex endpoint routing, region/project helpers, a model catalog, transport changes, and tests.

## Best Possible Solution

Land a Google-plugin-owned Vertex implementation that keeps provider ownership separate, uses guarded fetch for ADC exchange, scopes token caching to credential identity, normalizes global/regional/template endpoints, and includes tests plus changelog coverage.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: inject/proxy the transport fetch, rotate ADC credentials while the cache is warm, or configure a templated Vertex base URL and the PR code bypasses the intended fetch path, reuses the old token, or emits a literal templated host.

## Solution Assessment

No, not as-is. A dedicated `google-vertex` provider is the right direction, but this branch needs the auth-cache, fetch-path, templated-host, and changelog fixes before it is a maintainable merge candidate.
