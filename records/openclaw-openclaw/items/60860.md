## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Pass guarded fetch into ADC token exchange:** `extensions/google/vertex-adc.ts:65`
  - body: The ADC helper mints authorized_user tokens through global `fetch`, and service-account minting below uses the same path, while model requests go through `buildGuardedModelFetch`. Proxy-aware or restricted-network setups can route model traffic through OpenClaw's guard while OAuth token exchange bypasses it, so thread an injected or guarded fetch into token minting before using this provider path.
  - confidence: 0.88
- **[P2] Key ADC token cache by credential source:** `extensions/google/vertex-adc.ts:151-152`
  - body: `resolveGoogleVertexAdcToken` returns any unexpired cached token before reading the active ADC path, credential type, refresh token, service account, or project. A long-running gateway that rotates credentials can keep sending Vertex requests with a previous account's bearer token until expiry, so read the active source first and key the cache by stable credential identity.
  - confidence: 0.9
