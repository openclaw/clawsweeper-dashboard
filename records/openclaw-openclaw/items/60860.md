Full review comments:

- **[P1] Handle native Vertex API models in the exact provider:** `extensions/google/vertex-provider-registration.ts:56-59`
  - body: This PR removes `google-vertex` from the Google provider hook aliases, so the new exact `google-vertex` provider owns existing configs. Its stream factory only returns a stream for `api === "google-generative-ai"`, so explicit `api: "google-vertex"` configs that current main routes through the Vertex transport can lose the native stream path; handle that API here or preserve the alias until migration is complete.
  - confidence: 0.88
- **[P1] Route ADC token minting through guarded fetch:** `extensions/google/vertex-adc.ts:65`
  - body: The new ADC helper refreshes `authorized_user` tokens with global `fetch`, and service-account minting below uses the same pattern, while model requests use `buildGuardedModelFetch`. Proxy or restricted-network deployments can route model traffic through OpenClaw's guard while OAuth token exchange bypasses it, so pass an injected or guarded fetch into token minting before enabling this provider path.
  - confidence: 0.9
- **[P2] Key the ADC token cache by credential source:** `extensions/google/vertex-adc.ts:151-152`
  - body: `resolveGoogleVertexAdcToken` returns any unexpired cached token before reading the active ADC path, credential type, refresh token, service account, or project. A long-running gateway that rotates credentials can keep sending Vertex requests with a previous account's bearer token until expiry, so read the active source first and key the cache by stable credential identity.
  - confidence: 0.92
