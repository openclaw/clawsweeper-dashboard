## What This Changes

The PR adds a dedicated `google-vertex` provider in the Google plugin with ADC auth, Vertex endpoint routing, a model catalog, service-account token exchange, and Gemini thinking-signature replay support.

## Best Possible Solution

Land a Google-plugin-owned Vertex implementation that aligns manifest auth discovery, ADC token minting, credential-scoped caching, global/regional endpoint routing, tests, docs, and changelog coverage.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: use an injected/proxy fetch, rotate ADC credentials while the cache is warm, or configure a templated Vertex base URL and the PR code bypasses the intended fetch path, reuses an old token, or emits a literal templated host.

## Solution Assessment

No, not as-is. A dedicated `google-vertex` provider is the right direction, but this branch needs the auth-cache, fetch-path, templated-host, and changelog fixes before it is a maintainable merge candidate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Pass the transport fetch into ADC token exchange:** `extensions/google/vertex-adc.ts:65`
  - body: `resolveGoogleVertexAdcToken()` mints ADC tokens with global `fetch`, while the model request uses OpenClaw's guarded transport fetch. Proxy-aware, restricted-network, or test callers can route the Vertex request correctly while token minting bypasses that configured path.
  - confidence: 0.88
- **[P2] Key the ADC token cache by credential source:** `extensions/google/vertex-adc.ts:151-152`
  - body: The resolver returns any unexpired `cachedToken` before reading the active ADC file, refresh token, service-account identity, or project env. A long-running gateway that rotates credentials can keep sending Vertex requests with the previous account's bearer token until expiry.
  - confidence: 0.86
- **[P2] Substitute templated Vertex hosts before composing URLs:** `extensions/google/transport-stream.ts:223-224`
  - body: `new URL()` accepts `https://{location}-aiplatform.googleapis.com`, so template-style base URLs are preserved as a literal `{location}` host. Replace the template with the resolved region before returning the origin.
  - confidence: 0.8
- **[P3] Add the required changelog entry:** `extensions/google/index.ts:281`
  - body: Registering a user-facing `google-vertex` provider with new ADC behavior is a `feat` change, but this PR does not add an active-version `CHANGELOG.md` entry.
  - confidence: 0.92

## Security Review

Status: needs_attention

Concerns:

  - body: The new cache returns an unexpired token before checking the active ADC file path, refresh token, service-account identity, or env, so credential rotation can keep using the previous account's bearer token until expiry.
