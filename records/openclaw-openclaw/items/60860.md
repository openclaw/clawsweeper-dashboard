## Summary

Keep this PR open. Current main does not fully implement the branch's intended Google Vertex provider split or service-account ADC support, but the latest PR head still has blocking auth-path and credential-isolation defects that should be fixed before merge.

## What This Changes

The branch adds a dedicated `google-vertex` provider in the Google plugin with ADC-backed catalog/config resolution, Vertex request URL routing, authorized_user and service-account token exchange, manifest auth metadata, and focused Google transport tests.

## Best Possible Solution

Land or replace this with a maintainer-approved Google plugin implementation that gives `google-vertex` exact provider ownership, keeps selection-time and runtime ADC semantics aligned, supports authorized_user and service_account credentials, preserves injected fetch/proxy behavior, keys token caching by credential identity, handles global/regional/template Vertex endpoints, and includes focused auth/routing tests plus a user-facing changelog entry.

## Reproduction Assessment

Yes. The review has a high-confidence source-level reproduction path: the PR head calls ADC token resolution without the stream fetch option, returns cached ADC tokens before reading the active credential source, and preserves a literal `{location}` host through `new URL()`.

## Solution Assessment

No, not as-is. A dedicated Google Vertex provider is a maintainable direction for #60736 and #58775, but this patch should first fix ADC fetch injection, credential-keyed caching, templated endpoint substitution, and changelog coverage.

  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: Needs attention because the PR adds Google ADC token minting with concrete credential-isolation and network-path concerns.

Concerns:

- **[medium] Unkeyed ADC token cache can reuse prior credentials:** `extensions/google/vertex-adc.ts:151`
  - body: The new cache returns an unexpired token before checking the active ADC file path, refresh token, service-account identity, or env. Credential rotation can keep using the previous account's bearer token until expiry.
  - confidence: 0.86

  - body: The transport does not pass the caller's injected fetch into ADC token refresh, so credential minting can bypass configured proxy, test, or restricted network paths even when the model request is guarded.
