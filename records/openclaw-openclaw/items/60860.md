## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Pass guarded fetch into ADC token minting:** `extensions/google/vertex-adc.ts:65`
  - body: `resolveGoogleVertexAdcToken()` mints ADC tokens with global `fetch` before the model request constructs OpenClaw's guarded transport. Proxy-aware, restricted-network, and test callers can route the Vertex request through the guard while the credential exchange bypasses it; pass an injected or guarded fetch into the token helper before minting.
  - confidence: 0.88
- **[P2] Key the ADC token cache by credential source:** `extensions/google/vertex-adc.ts:151-152`
  - body: The resolver returns any unexpired `cachedToken` before reading the active env, ADC path, refresh token, service-account identity, or project metadata. A long-running gateway that rotates credentials can keep sending Vertex requests with the previous account's bearer token until expiry.
  - confidence: 0.87
- **[P2] Substitute templated Vertex hosts before composing URLs:** `extensions/google/transport-stream.ts:228-229`
  - body: `new URL()` accepts `https://{location}-aiplatform.googleapis.com`, so this branch can preserve `{location}` as the literal request host. Replace the template with the resolved region before returning the origin.
  - confidence: 0.82
