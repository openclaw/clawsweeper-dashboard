## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Pass guarded fetch into ADC token exchange:** `extensions/google/vertex-adc.ts:65`
  - body: The ADC helper mints authorized_user and service-account tokens through direct `fetch`, while the model request is sent through OpenClaw's guarded model fetch. Proxy-aware or restricted-network setups can route model traffic through the guard while token exchange bypasses it, so thread the guarded or injected fetch into token minting before using this provider in production paths.
  - confidence: 0.86
- **[P2] Key ADC token cache by credential source:** `extensions/google/vertex-adc.ts:151-152`
  - body: `resolveGoogleVertexAdcToken` returns any unexpired cached token before reading the active ADC file, env, credential identity, refresh token, service account, or project. A long-running gateway that rotates credentials can keep sending Vertex requests with a previous account's bearer token until expiry.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff adds Google ADC credential exchange, and the current implementation has guarded-transport and credential-cache concerns.

Concerns:

- **[medium] ADC token exchange bypasses guarded transport:** `extensions/google/vertex-adc.ts:65`
  - body: Token minting uses direct `fetch` rather than the guarded/injected fetch path used for model requests, so proxy and network-policy controls can apply to model traffic but not OAuth token exchange.
  - confidence: 0.86
- **[medium] Unkeyed ADC cache can reuse prior credentials:** `extensions/google/vertex-adc.ts:151`
  - body: The global cache returns an unexpired bearer token before checking the active credential source, so credential rotation can continue using the old account token until expiry.
  - confidence: 0.88
