## Summary

Keep open: current main and v2026.6.8 still default Telegram DMs to pairing, still send pairing challenges to unknown DM senders, and still use manual numeric allowFrom setup rather than token-verified owner ID capture. The related bootstrap issue overlaps, but this remains the narrower canonical Telegram access-default/security-default thread.

## What This Changes

Request to make Telegram setup default to dmPolicy="allowlist" by capturing a verified owner Telegram ID during onboarding.

## Best Possible Solution

Implement Telegram-owned verified owner enrollment for fresh setup: show a short one-time token, capture only the matching Telegram sender ID, write dmPolicy="allowlist" plus allowFrom, update docs/tests, and preserve explicit pairing configs or define a clear migration boundary.

## Reproduction Assessment

Yes. Source inspection is enough: an omitted Telegram dmPolicy still parses and runs as pairing, and unauthorized DMs under pairing still receive pairing challenges.

## Solution Assessment

No, not as a blanket default flip or first-sender capture. The best fix is verified fresh-setup owner enrollment plus an explicit compatibility decision for existing omitted-policy configurations.
