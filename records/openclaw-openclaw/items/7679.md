## Summary

Keep open: current main and v2026.5.22 still default Telegram DMs to pairing, still issue pairing challenges to unknown DM senders, and still require manual numeric allowFrom entry instead of token-verified owner capture.

## What This Changes

Request fresh Telegram setup to default to allowlist mode and capture the owner Telegram ID into allowFrom instead of leaving DMs in pairing mode.

## Best Possible Solution

Implement Telegram-owned onboarding that generates a short verification token, captures only the matching numeric sender ID, writes dmPolicy="allowlist" plus allowFrom for fresh owner setup, preserves explicit pairing configs, and updates schema/runtime/docs/tests.

## Reproduction Assessment

Yes, at source level. Current main still defaults omitted Telegram dmPolicy to pairing, sends pairing challenges to unknown DMs, and documents/tests that behavior.

## Solution Assessment

No for a default-only change. The maintainable fix needs token-verified owner capture during Telegram setup plus compatibility handling, not merely flipping the schema default or trusting the first inbound sender.
