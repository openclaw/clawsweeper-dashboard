## Summary

Keep open: current main and v2026.6.6 still default omitted Telegram DM policy to pairing and still use manual numeric allowFrom setup, so verified owner-ID allowlist onboarding remains unimplemented and needs maintainer security/product direction.

## What This Changes

The issue asks Telegram setup to default new direct-message access to an owner allowlist by token-verifying and capturing the owner Telegram ID during onboarding.

## Best Possible Solution

Implement Telegram-owned verified owner enrollment for fresh setup: show a short one-time token, capture only the matching sender ID, write dmPolicy="allowlist" plus allowFrom, and preserve or explicitly migrate existing pairing-mode configurations.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 still default omitted Telegram DM policy to pairing, document pairing as default, and use manual numeric allowFrom entry instead of verified owner capture.

## Solution Assessment

No, not as a blanket default flip or first-sender capture; the maintainable solution is verified fresh-setup owner enrollment plus a deliberate compatibility decision for existing omitted-policy configs.
