## Summary

Keep open: current main and v2026.5.12 still default Telegram DMs to pairing, still issue pairing challenges to unknown DM senders, and still use manual numeric allowFrom setup. The requested owner capture would change Telegram authorization defaults, so it needs maintainer security/product review before implementation.

## What This Changes

Fresh Telegram setup should default to dmPolicy="allowlist" and populate allowFrom with the owner Telegram ID captured during onboarding.

## Best Possible Solution

Implement this as Telegram-owned onboarding hardening: generate a short verification token, require the intended owner to send it to the bot, capture that matching numeric Telegram user ID, write dmPolicy="allowlist" plus allowFrom for fresh setup, and preserve existing explicit pairing configs.

## Reproduction Assessment

Yes for the current behavior, though this is a defaults/onboarding feature request rather than a failing-test bug. Source inspection shows omitted Telegram dmPolicy still defaults and falls back to pairing, unknown DMs still receive pairing challenges, and setup/docs still use manual numeric allowFrom entry.

## Solution Assessment

Yes directionally, but only with token-verified owner capture. A schema/runtime default flip alone is not the best solution because allowlist mode requires a sender ID and first-message capture would introduce an ownership race.
