## Summary

Keep open: current main and v2026.6.10 still default Telegram DMs to pairing, still use manual numeric allowFrom entry, and still lack token-verified owner ID capture during onboarding. The related open issues overlap but do not supersede this narrower Telegram setup/security-default request.

## What This Changes

Request fresh Telegram setup to default to dmPolicy="allowlist" with the intended owner Telegram ID captured into allowFrom instead of leaving DMs in pairing mode.

## Best Possible Solution

Implement Telegram-owned verified owner enrollment for fresh setup, with explicit upgrade behavior for existing omitted-policy configs, plus schema/runtime/docs/tests and a doctor migration only if a shipped config shape is invalidated.

## Reproduction Assessment

Yes, at source level: current main still defaults omitted Telegram DM policy to pairing, sends pairing challenges to unknown DMs, documents pairing as default, and uses manual numeric allowFrom entry instead of verified owner capture. I did not run tests because this review is read-only.

## Solution Assessment

No, not as a blanket default flip or first-message capture. The maintainable solution is verified fresh-setup owner enrollment plus a deliberate compatibility decision for existing omitted-policy installs.
