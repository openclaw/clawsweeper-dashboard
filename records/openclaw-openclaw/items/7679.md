## Summary

Keep open: current main and the latest release still default Telegram DMs to pairing, still issue pairing challenges to unknown DM senders, and still use manual numeric allowFrom setup instead of token-verified owner capture.

## What This Changes

The issue asks for fresh Telegram setup to default to allowlist mode by capturing the owner Telegram ID during onboarding.

## Best Possible Solution

Implement this as Telegram-owned onboarding: generate a short verification token, capture only the matching numeric sender ID, write dmPolicy="allowlist" plus allowFrom for fresh owner setup, and preserve explicit pairing configs with docs and tests.

## Reproduction Assessment

Yes, at source level. Current main still resolves omitted Telegram dmPolicy to pairing, sends pairing challenges to unknown DMs, and documents/tests that behavior; I did not run live Telegram setup because the source path directly shows the current state.

## Solution Assessment

No for a default-only change. The maintainable fix needs token-verified owner capture during Telegram setup plus schema/runtime/docs/tests, not merely flipping the default.
