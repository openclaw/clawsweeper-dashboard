## Summary

Keep open: current main and v2026.6.8 still default Telegram DMs to pairing and still rely on manual numeric allowFrom setup, so verified owner-ID allowlist onboarding remains unimplemented and needs maintainer security/product direction.

## What This Changes

The issue asks for Telegram setup to default fresh DMs to allowlist mode and auto-capture the owner Telegram ID into allowFrom.

## Best Possible Solution

Implement Telegram-owned verified owner enrollment for fresh setup: show a short one-time token, capture only the matching sender ID, write dmPolicy="allowlist" plus allowFrom, and preserve or deliberately migrate existing pairing-mode configurations.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.8 still default omitted Telegram dmPolicy to pairing, send pairing challenges to unknown DMs, document pairing as default, and use manual numeric allowFrom entry instead of verified owner capture.

## Solution Assessment

No, not as a blanket default flip or first-sender capture. The maintainable solution is verified fresh-setup owner enrollment plus an explicit compatibility decision for existing omitted-policy configs.
