## Summary

Keep open: current main and v2026.6.10 still default Telegram DMs to pairing, still document pairing as the default, and still require manual numeric allowFrom entry instead of token-verified owner capture during onboarding.

## What This Changes

Request fresh Telegram setup to default to dmPolicy="allowlist" with a verified owner Telegram ID captured during onboarding instead of default pairing.

## Best Possible Solution

Implement a maintainer-approved Telegram setup flow that generates a short verification token, captures only the matching sender ID, writes dmPolicy="allowlist" plus allowFrom, and preserves explicit pairing compatibility for existing installs with docs and regression tests.

## Reproduction Assessment

Yes, at source level: current main still defaults omitted Telegram DM policy to pairing, sends pairing challenges to unknown DMs, documents pairing as default, and uses manual numeric allowFrom entry instead of verified owner capture. I did not run tests or a live Telegram bot because this review is read-only.

## Solution Assessment

No, not as a blanket default flip or first-message capture. The maintainable solution is verified fresh-setup owner enrollment plus a deliberate compatibility decision for existing omitted-policy installs.
