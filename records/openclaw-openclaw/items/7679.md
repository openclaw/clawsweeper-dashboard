## Summary

Keep open: current main and the latest release still default Telegram DMs to pairing, still use manual numeric allowFrom setup, and do not implement token-verified owner ID capture during onboarding.

## What This Changes

The issue requests changing fresh Telegram setup from pairing-mode DMs to token-verified owner capture that writes allowlist mode and the owner Telegram ID.

## Best Possible Solution

Implement Telegram-owned verified owner enrollment for fresh setup: show a short one-time token, capture only the matching sender ID, write dmPolicy="allowlist" plus allowFrom, update docs/tests, and preserve explicit pairing configs or define a migration boundary.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.8 still default omitted Telegram DM policy to pairing, document pairing as default, and use manual numeric allowFrom entry instead of verified owner capture.

## Solution Assessment

No, not as a blanket default flip or first-sender capture. The maintainable solution is verified fresh-setup owner enrollment plus a deliberate compatibility decision for existing omitted-policy configs.
