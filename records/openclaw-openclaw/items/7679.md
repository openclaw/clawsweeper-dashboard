## Summary

Keep open. Current main still defaults Telegram DMs to pairing and still models setup as manual numeric allowFrom entry rather than token-verified owner ID capture; the discussion’s warning about first-message capture remains relevant.

## What This Changes

Request to make fresh Telegram setup default to allowlist mode by capturing the owner’s Telegram user ID during onboarding.

## Best Possible Solution

Implement Telegram-owned onboarding that displays a short verification token, captures only the sender who sends that token, writes explicit dmPolicy="allowlist" plus allowFrom, and updates schema/runtime/docs/tests while preserving existing pairing configs.

## Reproduction Assessment

Yes, for the reported current behavior. Source inspection on current main shows omitted Telegram dmPolicy still resolves to pairing, and existing tests assert pairing-code responses for unauthorized DMs under pairing policy.

## Solution Assessment

Unclear as a default-only change. The maintainable solution needs token-verified owner capture plus explicit allowFrom; flipping the default alone is not safe or complete.
