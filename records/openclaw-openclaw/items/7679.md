## Summary

Keep open: current main and v2026.6.6 still use Telegram pairing as the omitted/default DM policy and still rely on manual numeric allowFrom setup, so verified owner-ID allowlist onboarding is not implemented.

## What This Changes

The issue requests changing Telegram fresh setup so it defaults to allowlist mode with a verified captured owner Telegram ID instead of pairing mode.

## Best Possible Solution

Implement verified Telegram owner enrollment for fresh setup while preserving or explicitly migrating existing pairing-mode configurations.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 still default omitted Telegram DM policy to pairing, still document pairing as default, and setup still uses manual numeric allowFrom entry rather than verified owner capture.

## Solution Assessment

No, a blanket default flip or first-sender capture is not the best solution; the safer path is verified fresh-setup owner enrollment plus an explicit upgrade/compatibility decision for existing omitted-policy configs.
