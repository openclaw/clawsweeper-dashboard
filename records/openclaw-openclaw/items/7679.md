## Summary

Keep open: current main and v2026.6.6 still default Telegram DMs to pairing and still rely on manual numeric allowlist entry, while related bootstrap/default issues only overlap and do not supersede this narrower Telegram owner-enrollment request.

## What This Changes

Request fresh Telegram setup to default to an allowlist by enrolling the intended owner's numeric Telegram ID instead of leaving DMs in pairing mode.

## Best Possible Solution

Implement Telegram-owned fresh-setup enrollment that verifies a short-lived token from the intended owner, stores that sender's numeric ID, and writes explicit allowlist config while preserving existing pairing-mode installs unless maintainers approve a migration.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 default missing Telegram DM policy to pairing, send pairing challenges on that path, and only support manual numeric allowFrom setup. This is a security-default feature request, not a broken documented contract.

## Solution Assessment

No, not as a blanket default flip or first-message capture. The maintainable solution is verified fresh-owner enrollment with explicit allowlist config and a maintainer-approved upgrade boundary for existing omitted-policy setups.
