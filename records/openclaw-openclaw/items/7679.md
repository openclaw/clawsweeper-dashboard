## Summary

Keep open: current main and v2026.6.6 still default Telegram DMs to pairing and still use manual numeric allowFrom setup, so the requested verified owner-enrollment default is not implemented and needs maintainer security/product direction.

## What This Changes

The issue asks fresh Telegram setup to default to allowlist mode by auto-capturing the owner Telegram ID during onboarding.

## Best Possible Solution

Implement Telegram-owned fresh-setup enrollment that verifies a short-lived token from the intended owner, stores that numeric ID in `allowFrom`, and writes explicit `dmPolicy: "allowlist"` without changing existing explicit pairing configs.

## Reproduction Assessment

Yes at source level: current main and v2026.6.6 default omitted Telegram DM policy to pairing, still issue pairing challenges, and only support manual numeric allowFrom setup. This is a requested secure-default/onboarding behavior change rather than a broken documented contract.

## Solution Assessment

No, a blanket default flip or first-sender capture is not the best solution; verified fresh-setup owner enrollment is safer, and omitted-policy upgrade behavior needs maintainer approval.
