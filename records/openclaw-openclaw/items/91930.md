## Label Justifications

- `impact:auth-provider`: The issue concerns provider-owned model discovery and prevents upstream models from being selected through OpenClaw.

## Summary

Keep open: the reported catalog mismatch remains on current main and v2026.6.5. OpenClaw documents Zen as a multi-model catalog, the live Zen endpoint currently returns 48 models including the reported DeepSeek model, but the bundled OpenCode plugin owns no static or live model-catalog contribution; no viable open or merged PR currently resolves the Zen gap.

## What This Changes

Make the OpenCode Zen models shown by the live provider catalog available through OpenClaw model listing and selection.

## Best Possible Solution

Register an OpenCode-owned live Zen catalog using the existing guarded catalog helper, normalize each supported row into accurate model metadata, preserve a bounded fallback catalog for endpoint failures, and document any intentional exclusions.

## Reproduction Assessment

Yes, from source and the live contract with high confidence: the endpoint returns 48 models, but current main and v2026.6.5 have no OpenCode-owned catalog declaration or registration. An authenticated end-to-end model request was not run.

## Solution Assessment

Yes. Provider-owned live discovery through the existing plugin SDK helper is the narrowest maintainable solution; hardcoding a changing Zen catalog in core would violate the established ownership boundary.
