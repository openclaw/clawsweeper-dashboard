## Summary

Keep open. Current main still runs `before_tool_call` from the supplied Browser tool params before Browser execution, while the policy-checked current tab URL is only available on later Browser route responses; the reporter's fail-secure credential-vault workaround confirms the remaining plugin API gap.

## What This Changes

The issue asks for Browser `before_tool_call` params for `act`/`type` actions to include the current page URL or provide an equivalent hook-time query path for domain-pinning checks.

## Best Possible Solution

Add a narrow, additive browser/plugin contract that gives pre-action policy hooks a policy-safe current tab URL for `act`/`type`, or an equivalent supported query seam, with docs and tests covering blocked/private URLs and supported browser profiles.

## Reproduction Assessment

Yes, at source level. Current main passes the original Browser `act`/`type` params into `before_tool_call` before Browser execution, and the only policy-safe current URL helper is used later for route responses.

## Solution Assessment

Unclear as an exact implementation. The requested capability is valid, but maintainers should choose whether the permanent seam is an additive hook param, a typed hook context/query API, or another Browser-owned contract that preserves URL policy and plugin compatibility.
