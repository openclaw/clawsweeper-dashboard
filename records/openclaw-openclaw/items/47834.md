## Summary

Keep open. Current main still does not provide browser `act`/`type` `before_tool_call` hooks with hook-time current-page URL context, and the reporter's `openclaw-credential-vault` workaround confirms a remaining plugin API gap rather than a resolved core behavior. The broader credential-firewall discussion at https://github.com/openclaw/openclaw/issues/18245 is related, but it does not supersede this narrower hook contract and URL policy question.

## What This Changes

The issue requests current browser page URL context for browser `act`/`type` `before_tool_call` hooks so domain-pinned credential plugins can make pre-injection security decisions.

## Best Possible Solution

Add a narrow, additive browser/plugin hook contract that provides a policy-safe active-tab URL for browser `act`/`type`, or an equivalent supported query seam, with docs and regression coverage for blocked and private URL handling.

## Reproduction Assessment

Yes, at source level. The adapter runs `before_tool_call` before the browser tool executes, the hook runner forwards only existing params, and the browser route resolves a safe current URL only in the response path.

## Solution Assessment

No for copying the raw `_currentUrl` sketch into params as-is. The maintainable solution is a typed, additive, policy-safe hook-time URL contract or query seam that preserves existing hook ordering and URL filtering semantics.
