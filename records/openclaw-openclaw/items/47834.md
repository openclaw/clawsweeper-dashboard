## Summary

Keep open. Current main still does not expose a policy-safe current browser tab URL to `before_tool_call` hooks before `browser` `act`/`type` runs, and the reporter’s credential-vault workaround confirms a real plugin API gap rather than a resolved core behavior.

## What This Changes

Request that browser `before_tool_call` hook events for `act`/`type` include the current page URL or provide an equivalent hook-time query seam for domain-pinned credential checks.

## Best Possible Solution

Add a narrow, additive browser/plugin contract that gives pre-action browser policy hooks a policy-safe current tab URL for `act`/`type`, or an equivalent supported query seam, with docs and regression tests for blocked/private URLs and supported browser profiles.

## Reproduction Assessment

Yes, at source level. Current main calls `before_tool_call` with only supplied or policy-adjusted params before the browser tool executes, while the current tab URL is resolved later inside browser route response code.

## Solution Assessment

No for copying the raw `_currentUrl` sketch into params as-is. The maintainable solution is a typed, additive, policy-safe hook-time URL contract or query seam that preserves existing hook ordering and URL filtering semantics.
