## Summary

Keep open. Current main still does not provide the current browser tab URL at the `before_tool_call` decision point for browser `act`/`type`, and the reporter's fail-secure credential-vault workaround confirms a real remaining plugin API gap rather than an obsolete report.

## What This Changes

Expose the browser tab's current URL to `before_tool_call` hooks for browser `act`/`type` actions, or provide an equivalent hook-time query seam.

## Best Possible Solution

Add a narrow, additive browser/plugin contract that gives pre-action browser policy hooks a policy-safe current tab URL for `act`/`type`, or an equivalent supported query seam, with docs and regression coverage for blocked/private URLs and existing-session profiles.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: browser `act` params are passed to `before_tool_call` before execution, while active-tab URL resolution only happens later in the browser route response path.

## Solution Assessment

No for copying the raw `_currentUrl` sketch into params as-is. The maintainable solution should expose a typed, policy-safe current URL or query seam that reuses the existing safe route URL policy.
