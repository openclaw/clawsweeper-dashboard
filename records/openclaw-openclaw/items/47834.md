## Summary

Keep open. Current main still does not provide hook-time current browser URL context for browser `act`/`type`, and the reporter's fail-secure credential-vault workaround confirms a remaining plugin API gap rather than resolving the core contract.

## What This Changes

Expose the current browser tab URL to `before_tool_call` hooks for browser `act`/`type`, or provide an equivalent query seam, so credential plugins can enforce domain pins before injecting secrets.

## Best Possible Solution

Add a narrow, additive browser/plugin contract that provides a policy-safe current tab URL before `act`/`type` execution, with docs and tests for managed, node-proxied, and existing-session profiles.

## Reproduction Assessment

Yes. Source inspection on current main shows the hook event receives only the original params before browser execution, while policy-checked URL resolution happens later inside browser route responses.

## Solution Assessment

No for copying a raw `_currentUrl` sketch into params as-is. The maintainable path is a typed, policy-safe hook-time URL contract or query seam that preserves existing hook ordering and browser URL filtering semantics.
