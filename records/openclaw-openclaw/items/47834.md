## Summary

Keep open. Current main still does not provide hook-time current browser URL context for browser `act`/`type`, and the reporter's fail-secure credential-vault workaround confirms the plugin API gap remains rather than resolving it.

## What This Changes

Add hook-time current browser page URL context for browser `act`/`type` `before_tool_call` hooks so credential plugins can enforce domain pins.

## Best Possible Solution

Add a narrow, additive browser/plugin contract that gives pre-action browser policy hooks a policy-safe current tab URL for `act`/`type`, or an equivalent supported query seam, with docs and tests for blocked/private URLs and supported browser profiles.

## Reproduction Assessment

Yes. Source inspection on current main shows `before_tool_call` receives only the original/policy-adjusted params before browser execution, while policy-checked URL resolution happens later in browser route responses.

## Solution Assessment

No for copying a raw `_currentUrl` sketch into params as-is. The maintainable path is a typed, policy-safe hook-time URL contract or query seam that preserves existing hook ordering and browser URL filtering semantics.
