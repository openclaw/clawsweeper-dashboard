## Summary

Keep open. Current main still does not provide hook-time current browser URL context for browser `act`/`type`; the reporter's credential-vault workaround is useful and fail-secure, but it confirms a remaining plugin API gap.

## What This Changes

Expose the current browser page URL, or an equivalent query seam, to `before_tool_call` hooks for browser `act`/`type` actions.

## Best Possible Solution

Add a narrow, typed, policy-safe hook-time current URL contract, or an equivalent supported query seam, with docs and tests for managed and existing-session browser profiles.

## Reproduction Assessment

Yes, source-level reproduction is high confidence. The generic adapter calls `before_tool_call` with the original params before browser execution, while the browser route resolves safe URLs only in the post-action response path.

## Solution Assessment

No for copying a raw `_currentUrl` sketch into params as-is. The maintainable fix should expose a typed, policy-safe current URL or query seam that preserves existing hook ordering and browser URL filtering semantics.
