## Summary

Keep open. Current main and the latest tagged release still lack hook-time current browser URL context for browser `act`/`type`, and the reporter's fail-secure credential-vault workaround confirms a real plugin API gap rather than resolving it.

## What This Changes

Expose the current browser page URL, or an equivalent supported query seam, to `before_tool_call` hooks for browser `act`/`type` actions.

## Best Possible Solution

Add a narrow, additive browser/plugin contract that gives pre-action browser policy hooks a policy-safe current tab URL for `act`/`type`, or an equivalent supported query seam, with docs and regression tests for blocked/private URLs and supported browser profiles.

## Reproduction Assessment

Yes, at source level. Current main passes only supplied or policy-adjusted params into `before_tool_call` before browser execution, while browser current URL resolution happens later in route response helpers.

## Solution Assessment

No for copying a raw `_currentUrl` sketch into params as-is. The maintainable fix is a typed, policy-safe hook-time URL contract or query seam that preserves existing hook ordering and browser URL filtering semantics.
