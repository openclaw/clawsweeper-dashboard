## Summary

Keep open. Current main still does not provide a policy-safe current browser tab URL to `before_tool_call` hooks before browser `act`/`type` runs, and the reporter’s `openclaw-credential-vault` workaround confirms a real but security-sensitive plugin API gap rather than a resolved core behavior.

## What This Changes

Issue requests browser `before_tool_call` hook params for `act`/`type` actions to include the current page URL, or an equivalent hook-time query seam.

## Best Possible Solution

Add a narrow, additive browser/plugin contract that gives pre-action browser policy hooks a policy-safe current tab URL for `act`/`type`, or an equivalent supported query seam, with docs and regression tests for blocked/private URLs and supported browser profiles.

## Reproduction Assessment

Yes, at source level. The generic adapter calls `runBeforeToolCallHook` before `tool.execute`, and the hook runner forwards only existing params while the browser route resolves safe tab URLs later in the route response path.

## Solution Assessment

No for copying the raw `_currentUrl` sketch into params as-is. The maintainable solution is a typed, additive, policy-safe hook-time URL contract or query seam that preserves existing hook ordering and URL filtering semantics.
