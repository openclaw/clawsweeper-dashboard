## What This Changes

The PR threads agent-scoped `mediaLocalRoots` from Feishu bot reply dispatcher construction into `sendMediaFeishu` for reply-path media uploads.

## Best Possible Solution

Keep the shipped shared reply-media staging implementation as the canonical fix and close this duplicate Feishu-specific plumbing PR; separate Feishu docs or hint improvements can stay in their narrower PRs.

## Reproduction Assessment

No failing current-main path remains. The original missing-root path is source-explained, but current main stages local reply media through shared managed-media normalization before Feishu delivery, and v2026.5.2 contains that path.

## Solution Assessment

Yes. The current shipped solution is narrower and more maintainable than adding Feishu-only dispatcher roots because it centralizes reply-media staging and keeps local reads constrained by the existing managed-media access policy.
