The PR bounds Z.AI endpoint-probe non-2xx JSON error-body reads with the shared plugin SDK response-size helper and adds focused stream-limit regression tests.

## Best Possible Solution

Land the plugin-local bounded-read hardening after contributor-supplied real behavior proof confirms oversized Z.AI probe responses fail closed without buffering unbounded data.

## Reproduction Assessment

Yes, with the proof caveat: the fix stays inside the Z.AI plugin, uses the public plugin SDK bounded-response helper, and preserves the existing fallback semantics. A core/internal provider HTTP helper would violate the plugin boundary, so the chosen SDK helper is the right layer.
