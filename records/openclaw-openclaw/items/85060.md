## What This Changes

The PR adds `publicArtifacts.listArtifacts` support to `memory-lancedb`, factors memory artifact listing into `memory-host-core`, and updates memory-core/LanceDB tests.

## Best Possible Solution

Keep the shared public-artifact helper and LanceDB capability registration, but refactor the registration to preserve lazy/narrow SDK loading and require real bridge proof before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows `memory-wiki` consumes `listActiveMemoryPublicArtifacts`, while `memory-lancedb` does not register a `publicArtifacts` provider, so the bridge has no artifacts from the LanceDB memory slot.

## Solution Assessment

No, not as written. The shared helper and LanceDB provider are the right direction, but the PR should preserve the existing lazy SDK import boundary and include real bridge proof.
