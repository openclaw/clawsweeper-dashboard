## Summary

Keep open. Current main has meaningful Android chat support, image attachments, assistant-launch routing into Chat, and Tailscale-aware connection guidance, but the official Android app is still documented and implemented as a Connect-first companion node; the remaining request is Android product direction plus selective upstreaming, not a completed core cleanup item.

## What This Changes

The issue asks maintainers to discuss a chat-first Android app surface or selectively upstream focused Android UX, media, and connectivity improvements from an external fork.

## Best Possible Solution

Android/mobile maintainers should choose the official shape first: default Chat, an explicit chat-first mode, or continued Connect-first behavior, then review #70993 or smaller focused PR slices for chat UX, media playback, and Tailscale/connectivity improvements.
