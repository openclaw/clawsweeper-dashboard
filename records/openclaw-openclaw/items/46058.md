## Summary

Keep open. Current main has meaningful Android chat support, assistant-launch routing into Chat, image attachment handling, and Tailscale-aware connection guidance, but it still documents and implements Android as a companion-node app with an Overview/Connect-first shell rather than deciding the requested chat-first Android product direction.

## What This Changes

The issue asks whether OpenClaw should adopt a chat-first Android surface or accept focused Android UX, media, and connectivity improvements from an external fork.

## Best Possible Solution

Android maintainers should choose the official shape first: default Chat, an explicit chat-first mode, or Overview/Connect-first with focused PR slices for chat UX, media playback, and Tailscale/connectivity improvements.
