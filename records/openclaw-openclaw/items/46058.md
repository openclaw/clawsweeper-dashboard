## Summary

Keep open. Current main has meaningful Android chat, image attachments, assistant-launch routing, voice/Talk support, and Tailscale-aware gateway guidance, but the official app still documents Android as a companion node and opens into a Home/Overview shell rather than a chat-first default; the linked `clawchat2-public`, AndroidForClaw, and open Android chat-parity PR are useful product references, not a settled core decision.

## What This Changes

Request to discuss a chat-first official Android app surface and possibly upstream focused Android UX, media, and connectivity improvements from an external fork.

## Best Possible Solution
