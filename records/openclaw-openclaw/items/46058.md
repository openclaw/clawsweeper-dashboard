## Summary

Keep open: current main and v2026.6.10 have a stronger Android chat path, a prominent Chat/Talk entry point, assistant-to-chat launch routing, and Tailscale-aware pairing docs, but the official app is still documented as a Gateway-required companion node and still opens into the post-onboarding Overview rather than a chat-first default. The linked `clawchat2-public` fork and the AndroidForClaw/HermesApp reference are useful external product evidence, not a resolved OpenClaw core decision.

## What This Changes

This issue asks maintainers to consider an official chat-first Android app mode/surface or narrow upstreamable Android UX, media, and connectivity slices from an external fork.

## Best Possible Solution

Android owners should choose the official launch/navigation and media/connectivity scope first, then invite narrow proof-backed PRs rather than import a broad fork.
