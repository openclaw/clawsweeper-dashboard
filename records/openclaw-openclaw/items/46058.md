## Summary

Keep open. Current main has meaningful Android chat support, image attachments, assistant-launch routing into Chat, and Tailscale-aware connection guidance, but the official Android app still defaults and documents a Connect-first companion-node flow; the remaining question is Android product direction and focused upstreaming scope, not a completed core fix.

## What This Changes

The issue asks whether OpenClaw should add an official chat-first Android mode/surface or accept focused Android UX, media, and connectivity improvements from an external fork.

## Best Possible Solution

Android/mobile owners should choose the official shape first: default Chat, an explicit chat-first mode, or continued Connect-first behavior, then review focused PR slices for chat UX, media playback, and Tailscale/connectivity improvements.
