## Summary

Keep open. Current main still carries session and turn-source metadata through the exec approval path, but the native macOS prompt neither renders `sessionKey` nor decodes/displays the existing requester route fields.

## What This Changes

Request to show session and requester/channel context in the macOS native exec approval dialog.

## Best Possible Solution

Add native macOS prompt rows for non-empty `sessionKey` and existing `turnSource*` metadata, while deferring new sender or tool-reason fields until the shared approval contract defines them.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: create an `ExecApprovalPromptRequest` with a non-empty `sessionKey` and current `buildAccessoryView` never reads or displays it; gateway and node approvals both route through that presenter.

## Solution Assessment

Yes for the narrow direction. Rendering existing `sessionKey` and decoding/displaying existing `turnSource*` fields in Swift is the smallest maintainable fix; brand-new sender or tool-reason fields should wait for a shared payload contract.
