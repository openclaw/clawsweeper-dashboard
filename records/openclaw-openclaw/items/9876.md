## Summary

Keep open. Current main still does not render `sessionKey` in the macOS native exec approval prompt, and the Swift prompt model still lacks decoded requester/channel fields even though the shared TypeScript payload carries turn-source metadata.

## What This Changes

Request to show the macOS native exec approval session key and, where safe, channel/sender/tool context in the approval popup.

## Best Possible Solution

Add a trimmed Session row in the native prompt first, then display only safe existing turn-source context after maintainer privacy and UX signoff.

## Reproduction Assessment

Yes, by source inspection. The macOS prompt request carries or decodes `sessionKey`, the node and gateway paths invoke the same presenter, and `buildAccessoryView` never renders that field while the Swift model lacks requester metadata fields.

## Solution Assessment

Unclear for the full request. Rendering the existing Session row is the narrow maintainable first step, but adding Channel/Sender/Thread or tool-reason display changes metadata exposure and should reuse existing turn-source fields only after maintainer signoff.
