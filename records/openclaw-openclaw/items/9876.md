## Summary

Keep this open: current main and latest release still do not show the existing session key in the native macOS exec approval prompt, and the richer requester-context display still needs a privacy-aware maintainer decision.

## What This Changes

Request to show session and requester context, such as channel and sender, in the macOS exec approval popup.

## Best Possible Solution

Add a trimmed Session row to the native macOS prompt first, then define a reviewed requester-context display contract from existing turn-source metadata.

## Reproduction Assessment

Yes, at source level: current main passes sessionKey into the native macOS prompt path and the web UI renders it, while buildAccessoryView lacks a Session row and requester fields. I did not live-run the macOS alert in this read-only review.

## Solution Assessment

Partly. The Session row is the narrow maintainable first fix, but channel, sender, and thread context should be a reviewed product/security display contract rather than raw ad hoc fields.
