## Summary

Keep open. Current main still carries session and turn-source metadata through the exec approval path, but the macOS native prompt does not render the existing session key and the Swift prompt model does not decode requester/channel fields.

## What This Changes

Request to show the session key and requester/channel context in the macOS exec approval dialog.

## Best Possible Solution

Add a trimmed Session row in the native prompt first, then display only safe existing turnSource channel/thread/account context after maintainer privacy and UX signoff.

## Reproduction Assessment

Yes, by source inspection. The macOS prompt request carries or decodes sessionKey, the node and gateway paths invoke the same presenter, and buildAccessoryView never renders that field while the Swift model lacks requester metadata fields.

## Solution Assessment

Unclear for the full request. Rendering the existing Session row is the narrow maintainable first step, but adding Channel/Sender/Thread or tool-reason display changes metadata exposure and should reuse existing turnSource fields only after maintainer signoff.
