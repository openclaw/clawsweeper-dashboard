## Summary

Keep this open: current main and the latest release still carry `sessionKey` and turn-source metadata through the exec approval path, but the macOS native prompt does not display the session and has no native requester-context fields. The narrow Session row is clear, while Channel/Sender/Thread display needs maintainer product/privacy judgment.

## What This Changes

Request to show session identity and requester/channel context in the macOS exec approval popup.

## Best Possible Solution

Add a trimmed Session row for the existing `sessionKey`, then extend the native prompt request model to display approved safe Channel/Sender/Thread context from existing turn-source metadata with focused macOS prompt coverage.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction: current main passes `sessionKey` into the native prompt request, but `buildAccessoryView` has no Session row and the Swift request model has no requester fields.

## Solution Assessment

Partially. The proposed Session row is the narrowest parity fix, but the wider Channel/Sender/Thread display should wait for maintainer product/privacy decisions on safe metadata and prompt wording.
