## Summary

Keep this open: current main and the latest release still carry session and turn-source metadata through exec approval paths, but the native macOS popup still does not render the existing Session value or decode/display requester context.

## What This Changes

The issue requests that the macOS exec approval popup display the existing session key and richer requester/channel context for approvals.

## Best Possible Solution

Add a trimmed Session row to the native prompt first, then define a privacy-aware requester-context display contract from existing turn-source metadata with focused Swift, gateway, and prompt coverage.

## Reproduction Assessment

Yes, at source level: current main passes `sessionKey` into the native macOS prompt path and the web UI renders it, while `buildAccessoryView` lacks a Session row and requester fields. I did not live-run the macOS alert in this read-only review.

## Solution Assessment

Partly. The Session row is the narrow maintainable first fix, but channel, sender, and thread context should be a reviewed product/security display contract rather than raw ad hoc fields.
