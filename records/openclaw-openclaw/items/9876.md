## Summary

Keep open. Current main still passes `sessionKey` through the node and gateway approval paths and carries turn-source metadata in TypeScript, but the macOS native prompt neither renders the session nor decodes/displays requester context; no duplicate or merged fix was found.

## What This Changes

Requests that the macOS exec approval popup show the existing session key and higher-level requester context such as channel, sender, and tool reason.

## Best Possible Solution

Add a trimmed Session row in the Swift prompt first, then extend the native request model to display only approved safe turn-source context with focused macOS prompt coverage.

## Reproduction Assessment

Yes, by source inspection on current main. The node and gateway paths pass `sessionKey` into `ExecApprovalsPromptPresenter`, but `buildAccessoryView` omits a Session row and the Swift model lacks turn-source fields.

## Solution Assessment

Unclear for the full request. Rendering the existing Session row is the narrow maintainable first step, but Channel/Sender/Thread/tool-reason display needs maintainer UX and privacy signoff before implementation.
