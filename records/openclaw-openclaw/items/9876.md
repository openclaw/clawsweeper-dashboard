## Summary

## What This Changes

Request to show the triggering session and requester/channel context in the macOS native exec approval popup.

## Best Possible Solution

Add the trimmed Session row first, then decode and display only maintainer-approved, sanitized requester context from existing turn-source metadata with focused native prompt coverage.

## Reproduction Assessment

Yes, at source level: current main passes sessionKey into the native prompt path and the web UI renders it, while macOS buildAccessoryView lacks a Session row and requester fields. I did not live-run the macOS alert in this read-only review.

## Solution Assessment

Partly. The Session row is the narrow maintainable first fix, but channel, sender, and thread context should be a reviewed product/security display contract rather than raw ad hoc fields.
