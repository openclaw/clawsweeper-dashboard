## Summary

Keep open. Current main still passes `sessionKey` through the macOS node/gateway approval paths and carries turn-source metadata in TypeScript, but the native macOS prompt neither renders Session nor decodes requester context; no duplicate or merged fix was found.

## What This Changes

The issue requests showing `sessionKey` plus safe requester/channel context in the macOS exec approval prompt.

## Best Possible Solution

Add a trimmed Session row in the Swift prompt first, then expose only maintainer-approved safe turn-source context through the native request model with focused macOS prompt coverage.

## Reproduction Assessment

Yes, by source inspection on current main. The node and gateway paths pass `sessionKey` into `ExecApprovalsPromptPresenter`, but `buildAccessoryView` omits a Session row and the Swift model lacks turn-source requester fields.

## Solution Assessment

Unclear for the full request. Rendering the existing Session row is the narrow maintainable first step, but Channel/Sender/Thread/tool-reason display needs maintainer UX and privacy signoff before implementation.
