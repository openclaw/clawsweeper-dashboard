## Summary

Keep open. Current main still carries session and turn-source metadata through exec approval requests, but the macOS native prompt does not render the existing session key and the Swift prompt model does not decode higher-level requester context.

## What This Changes

The issue requests macOS exec approval popups to show the session key and richer requester context such as channel and sender.

## Best Possible Solution

Render a trimmed non-empty Session row in the macOS prompt first, then extend the native prompt request model to display safe, normalized turn-source context such as channel, destination, account, and thread where available.

## Reproduction Assessment

Yes, by source inspection. `ExecApprovalPromptRequest` has `sessionKey` and both node and gateway paths can pass it to `ExecApprovalsPromptPresenter`, but `buildAccessoryView` never renders a Session row.

## Solution Assessment

Partly. Rendering the existing session key is the narrowest maintainable parity fix, while channel/sender display should wait for a maintainer-approved, privacy-aware requester-context contract rather than ad hoc raw field rendering.
