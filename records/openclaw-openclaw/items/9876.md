## Summary

Keep this open: current main still carries `sessionKey` and turn-source metadata through the exec approval path, but the macOS native prompt does not render the session and the Swift prompt model has no requester-context fields. The Session row is a clear parity improvement; Channel/Sender/Thread display needs maintainer product and privacy judgment.

## What This Changes

Request to show session and requester context in the macOS native exec approval popup.

## Best Possible Solution

Add a trimmed Session row for the existing `sessionKey`, then display approved safe Channel/Sender/Thread context from existing turn-source metadata with focused macOS prompt coverage.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction: current main passes `sessionKey` into the native prompt request, but `buildAccessoryView` has no Session row and the Swift request model has no requester-context fields.

## Solution Assessment

Partially. The proposed Session row is the narrowest parity fix, but richer requester-context display should wait for maintainers to decide which metadata is safe and useful to show.
