## Summary

Keep this open: current main and the latest release still receive `sessionKey` in the macOS exec approval path but do not render it, and the richer requester fields exist in the TypeScript approval contract without a native prompt display decision yet.

## What This Changes

Show `sessionKey` and higher-level requester/channel context in the macOS exec approval dialog so users can identify what triggered a command approval.

## Best Possible Solution

Add a trimmed Session row for the existing `sessionKey`, then extend the native gateway request model to display approved safe Channel/Sender/Thread context from existing turn-source metadata with focused macOS prompt coverage.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction: current main passes `sessionKey` into the native prompt request, but `buildAccessoryView` has no Session row and the Swift request model has no requester fields. I did not run the macOS alert live in this read-only review.

## Solution Assessment

Partially. The proposed Session row is the narrowest parity fix, but the wider Channel/Sender/Thread display should wait for a maintainer decision on safe requester metadata and prompt wording.
