## Summary

Keep open. Current main still carries `sessionKey` and shared turn-source metadata through the exec approval path, but the native macOS approval prompt does not render the session or decode/display requester context; the prior maintainer-facing comments reached the same conclusion and the source still matches that gap.

## What This Changes

Show session and requester/channel context in the macOS native exec approval popup.

## Best Possible Solution

Add native macOS prompt support for non-empty Session plus safe rows from existing turn-source metadata, and defer sender names or tool reasoning until the shared approval payload defines those fields consistently.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: construct an `ExecApprovalPromptRequest` with a non-empty `sessionKey` and the current `buildAccessoryView` path never reads or displays that field, while gateway and node prompt paths pass through the same presenter.

## Solution Assessment

Yes for the narrow direction. Rendering existing `sessionKey` and decoding/displaying existing turn-source fields in Swift is the smallest maintainable fix; adding new sender or tool-reason fields should wait for a shared payload contract.
