## Summary

Keep this open: current main and the latest release still omit the existing Session value from the native macOS exec approval prompt, and the richer requester/channel context needs maintainer/security review before display.

## What This Changes

The issue requests adding session and requester context to the macOS exec approval popup so operators can identify which session or channel triggered a command approval.

## Best Possible Solution

Add the trimmed Session row as the narrow parity fix, then expose only maintainer-approved, sanitized requester context from existing turn-source metadata with focused macOS and gateway coverage.

## Reproduction Assessment

Yes, at source level: current main passes sessionKey into the native macOS prompt path and the Control UI renders Session, while the macOS buildAccessoryView lacks a Session row and requester fields. I did not live-run the macOS alert.

## Solution Assessment

Partly. The Session row is the narrow maintainable first fix, but channel, sender, and thread context should be a privacy-aware display contract rather than raw ad hoc fields.
