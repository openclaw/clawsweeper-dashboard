## Summary

Keep open. Current main still carries session and turn-source metadata through the exec approval path, but the macOS native prompt does not render `sessionKey` and does not decode/display requester context such as channel, sender, or thread.

## What This Changes

Request to add session and requester/channel/sender context to the macOS native exec approval popup.

## Best Possible Solution

Add a trimmed Session row in the Swift prompt first, then display only approved safe turn-source context with focused macOS layout coverage.

## Reproduction Assessment

Yes, by source inspection. The node and gateway paths pass `sessionKey` into the presenter, but `buildAccessoryView` never renders a Session row and the Swift model lacks requester-context fields.

## Solution Assessment

Unclear for the full request. Rendering the existing Session row is the narrow maintainable first step, but Channel/Sender/Thread or tool-reason display should reuse existing turn-source metadata only after maintainer UX/privacy signoff.
