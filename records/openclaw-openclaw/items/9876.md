## Summary

Keep this open: current main and the latest shipped release still pass session metadata through the exec approval path, but the macOS native prompt does not render the existing session and does not decode requester context fields. The Session row is a narrow parity fix; Channel/Sender/Thread display needs maintainer product and privacy review.

## What This Changes

Request to show sessionKey and safe requester/channel/sender context in macOS exec approval popups.

## Best Possible Solution

Add a trimmed Session row for the existing sessionKey, then display only maintainer-approved, sanitized requester metadata from the existing turn-source fields.

## Reproduction Assessment

Yes. Source inspection on current main shows sessionKey reaches the native approval request, but buildAccessoryView has no Session row and the Swift request model has no requester-context fields.

## Solution Assessment

Partially. The proposed Session row is the narrowest maintainable parity fix, but richer Channel/Sender/Thread display should wait for maintainers to decide which metadata is safe and useful in a native exec approval prompt.
