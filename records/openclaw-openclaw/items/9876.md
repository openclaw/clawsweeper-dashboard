## Summary

Keep open. Current main still passes or decodes `sessionKey` for the macOS exec approval prompt and carries `turnSource*` metadata through the TypeScript approval path, but the native Swift prompt does not render Session/Channel/Sender context.

## What This Changes

Request to show the triggering session and readable requester or channel context in the macOS exec approval popup.

## Best Possible Solution

Add a trimmed Session detail row in the native prompt, then design a minimal safe mapping from existing `turnSource*` metadata to readable Channel/Thread context before considering new sender or tool-reason fields.

## Reproduction Assessment

Yes. Source inspection on current main shows `sessionKey` reaches or is decoded by the macOS prompt path, while `buildAccessoryView` never renders it and the Swift model lacks requester metadata fields.

## Solution Assessment

Unclear for the full request. The Session row is the narrow first step, but richer requester context should reuse safe existing `turnSource*` fields and needs maintainer privacy/UX agreement before adding sender or tool-reason data.
