## Summary

Keep open. Current main still carries session and requester metadata through the exec approval path, but the macOS native prompt does not render the existing session key and the Swift prompt model lacks higher-level requester fields.

## What This Changes

Show session key plus safe requester/channel context in the macOS exec approval popup.

## Best Possible Solution

Add a trimmed Session row in the native prompt first, then display safe existing turnSource channel/thread/account context only after maintainer privacy and UX signoff.

## Reproduction Assessment

Yes. Source inspection on current main shows sessionKey reaches or is decoded by the macOS prompt path, while buildAccessoryView never renders it and the Swift model lacks requester metadata fields.

## Solution Assessment

Unclear for the full request. The Session row is the narrow maintainable first step, while richer requester context should reuse existing turnSource fields and needs maintainer agreement before displaying sender or tool-reason details.
