## Summary

Keep open: current main and the latest shipped tag still omit the Session row and native requester metadata from the macOS approval alert, while the TypeScript approval payload and Control UI already carry or render that context.

## What This Changes

The issue requests that the macOS exec approval popup show the existing session key and richer requester context such as channel or sender.

## Best Possible Solution

Add a trimmed Session row first, then decode and display only privacy-reviewed requester context from existing turn-source metadata with focused macOS prompt coverage.

## Reproduction Assessment

Yes, at source level: current main passes sessionKey into the native prompt path and the web UI renders it, while buildAccessoryView never renders a Session row. I did not live-run the macOS alert in this read-only review.

## Solution Assessment

Partly. The Session row is a narrow maintainable first fix, but adding channel, sender, and thread context should be a reviewed product/security decision rather than ad hoc new fields.
