## Summary

Keep open. Current main still decodes or passes `sessionKey` into the macOS exec approval prompt path, but the native Swift presenter does not render a Session row or higher-level requester context, and prior ClawSweeper comments reached the same source-backed conclusion.

## What This Changes

Request to show the session key plus requester/channel context in the macOS native exec approval dialog.

## Best Possible Solution

Add a trimmed Session row in the native prompt first, then map existing `turnSource*` metadata into safe Channel/Thread display fields after maintainer privacy and UX signoff.

## Reproduction Assessment

Yes. Source inspection on current main shows `sessionKey` reaches or is decoded by the macOS prompt path, while `buildAccessoryView` never renders it and the Swift model lacks requester metadata fields.

## Solution Assessment

Unclear for the full request. The Session row is the narrow maintainable first step, but richer requester context should reuse existing `turnSource*` fields and needs maintainer agreement before adding sender or tool-reason display.
