## Summary

Keep open. Current main still passes session data through exec approval paths and carries turn-source metadata in TypeScript, but the macOS native prompt does not render the existing session key or decode/display higher-level requester context; prior review comments reached the same conclusion and no duplicate fix was found.

## What This Changes

Request to show session key plus safe requester/channel context in the macOS exec approval popup.

## Best Possible Solution

Render a trimmed Session row first, then add explicitly approved, non-sensitive requester context to the gateway/native request contract and macOS layout with focused Swift and gateway tests.

## Reproduction Assessment

Yes, by source inspection on current main. The node and gateway paths pass sessionKey into ExecApprovalsPromptPresenter, while buildAccessoryView omits a Session row and the Swift model lacks requester metadata fields.

## Solution Assessment

Unclear for the full request. Rendering the existing Session row is the narrow maintainable first step, but Channel/Sender/Thread/tool-reason display needs maintainer UX and privacy signoff before wiring it into the native prompt.
