## Summary

Keep open. Current main still passes or decodes `sessionKey` in the macOS exec approval path, but the native prompt does not render a Session row and the Swift request model still lacks higher-level requester metadata fields.

## What This Changes

Request to show the macOS exec approval session key and richer requester context such as channel, sender, thread, or tool context.

## Best Possible Solution

Add a trimmed Session row in the native prompt first, then map existing `turnSource*` metadata into safe Channel/Thread display fields after privacy and UX review.

## Reproduction Assessment

Yes. Source inspection on current main shows `sessionKey` reaches or is decoded by the macOS prompt path, while `buildAccessoryView` never renders it and the Swift model lacks requester metadata fields.

## Solution Assessment

Unclear for the full request. Rendering the existing session key is the narrow maintainable first step, but richer requester context should reuse existing `turnSource*` metadata and needs maintainer agreement before exposing sender or tool-reason details.
