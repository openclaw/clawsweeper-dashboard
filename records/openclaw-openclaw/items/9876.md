## Summary

Keep open. Current main still carries session and turn-source metadata through the exec approval path, but the native macOS prompt does not render `sessionKey` and does not decode or display channel/thread requester context.

## What This Changes

Request macOS exec approval dialogs to show the triggering session plus requester/channel context so users can identify which conversation caused an approval request.

## Best Possible Solution

Add native macOS prompt rows for non-empty `sessionKey` and existing `turnSource*` metadata, while deferring sender names or tool reasoning until the shared approval contract defines them.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: build an `ExecApprovalPromptRequest` with a non-empty `sessionKey` and the current `buildAccessoryView` path never reads or displays it; gateway and node approvals both route through that presenter.

## Solution Assessment

Yes for the narrow direction. Rendering existing `sessionKey` and decoding/displaying existing `turnSource*` fields in Swift is the smallest maintainable fix; new sender or tool-reason fields should wait for a shared payload contract.
