## Summary

Keep this open: current main and the latest release still carry session and turn-source metadata through exec approval paths, but the native macOS prompt does not render the existing Session value or decode/display requester context.

## What This Changes

Request to show sessionKey plus safer requester context such as channel, sender, and thread in the native macOS exec approval popup.

## Best Possible Solution

Add a trimmed Session row to the native prompt first, then define a privacy-aware requester-context display contract from existing turn-source metadata with focused Swift and gateway coverage.

## Reproduction Assessment

Yes, at source level: current main passes sessionKey into the native macOS prompt path and the web UI renders it, while buildAccessoryView lacks a Session row and requester fields. I did not live-run the macOS alert in this read-only review.

## Solution Assessment

Partly. The Session row is the narrow maintainable first fix, but channel, sender, and thread context should be a reviewed product/security display contract rather than raw ad hoc fields.
