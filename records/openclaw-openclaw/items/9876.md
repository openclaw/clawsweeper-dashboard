## Summary

Keep open. Current main still decodes or passes `sessionKey` into the macOS exec approval prompt path, but the native prompt does not render it and the Swift request model does not expose the existing TypeScript `turnSource*` requester metadata.

## What This Changes

Request to show session and requester/channel context in the macOS exec approval popup.

## Best Possible Solution

Add a trimmed Session row to the native prompt first, then reuse safe existing `turnSource*` metadata for readable channel/thread context before adding any new sender or tool-reason contract.

## Reproduction Assessment

Yes. Source inspection on current main shows `sessionKey` reaches or is decoded by the macOS prompt path, while `buildAccessoryView` never renders it and the Swift model lacks requester metadata fields.

## Solution Assessment

Unclear for the full request. The Session row is the narrow immediate fix, but richer requester context should use existing `turnSource*` fields first and needs maintainer agreement before adding sender or tool-reason data.
