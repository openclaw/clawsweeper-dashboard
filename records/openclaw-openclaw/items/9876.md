## Summary

Keep this open: current main still carries session and turn-source metadata through the exec approval path, but the macOS native prompt does not render the existing session and does not decode requester context fields. The narrow Session row is clear; Channel/Sender/Thread display needs maintainer product and privacy judgment.

## What This Changes

The issue requests that the macOS exec approval popup show the session key and richer requester context such as channel and sender.

## Best Possible Solution

Add a trimmed Session row for the existing sessionKey, then extend the Swift request model and presenter to display only approved, sanitized requester metadata from the existing turn-source fields.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction: current main passes sessionKey into the native prompt request, but buildAccessoryView has no Session row and the Swift request model has no requester-context fields.

## Solution Assessment

Partially. The proposed Session row is the narrowest maintainable parity fix, but richer requester-context display should wait for maintainers to decide which metadata is safe and useful in a native exec approval prompt.
