## Summary

Keep this open: current main and v2026.5.12 still omit session/requester context from the native macOS exec approval prompt while the data is carried elsewhere, and richer requester display needs maintainer product/privacy/security review.

## What This Changes

The issue requests that the macOS exec approval dialog show the existing session key and potentially richer requester/channel context.

## Best Possible Solution

Add a trimmed Session row for the existing sessionKey, then expose only maintainer-approved, sanitized turn-source metadata for Channel/Sender/Thread context.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.5.12 shows sessionKey reaches the native approval request and the web UI renders it, but the native buildAccessoryView has no Session row and the Swift request model has no requester-context fields.

## Solution Assessment

Partially. The proposed Session row is the narrowest maintainable parity fix, but richer Channel/Sender/Thread display should wait for maintainers to decide which metadata is safe and useful in a native exec approval security prompt.
