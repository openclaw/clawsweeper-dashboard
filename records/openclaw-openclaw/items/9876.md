## Summary

Keep open. Current main still omits `sessionKey` from the macOS native exec approval popup, and the broader Channel/Sender/Thread display needs maintainer UX/privacy direction before implementation.

## What This Changes

The issue requests that the macOS exec approval popup show session identity and requester/channel context.

## Best Possible Solution

Add a trimmed Session row in the native prompt first, then display only safe existing turn-source context after maintainer UX/privacy signoff.

## Reproduction Assessment

Yes, by source inspection. The macOS prompt request carries or decodes `sessionKey`, both node and gateway paths invoke the same presenter, and `buildAccessoryView` never renders it.

## Solution Assessment

Unclear for the full request. Rendering the existing Session row is the narrow maintainable first step, but Channel/Sender/Thread or tool-reason display should reuse existing turn-source metadata only after maintainer privacy and UX signoff.
