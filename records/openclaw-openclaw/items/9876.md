## Summary

Keep open: current main and v2026.6.10 still carry session and turn-source metadata for exec approvals, but the macOS native prompt renders neither the existing session key nor safe requester/channel context.

## What This Changes

Request to add session and requester context to the macOS exec approval popup, starting with the existing `sessionKey` and extending to safe channel/sender metadata.

## Best Possible Solution

The macOS approval prompt should display the existing session key and, after maintainer security review, a redacted human-readable origin context derived from the gateway approval metadata.

## Reproduction Assessment

Yes. Source inspection on current main shows `sessionKey` reaches the macOS prompt request, but `buildAccessoryView` never renders it and the Swift request model lacks the richer turn-source fields.

## Solution Assessment

Partly. Adding a trimmed Session row is the narrow parity fix, but displaying channel/sender/thread context should be designed with privacy-safe labels rather than dumping raw identifiers.
