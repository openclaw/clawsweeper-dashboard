## Summary

Current main and v2026.6.5 still omit the originating session from the macOS exec approval popup, while no canonical replacement item exists. The session parity change is well-defined, but richer requester identity needs an explicit privacy and security product decision.

## What This Changes

Show the originating session in the macOS exec approval popup and consider adding safe, human-readable channel and sender context.

## Best Possible Solution

Render a trimmed non-empty Session row in the shared native presenter, then separately define and review a privacy-minimized requester-context schema with stable human-readable Channel, Sender, and Thread fields.

## Reproduction Assessment

Yes, at source level. On current main, an approval request with a non-empty `sessionKey` reaches `ExecApprovalsPromptPresenter`, whose Context rows omit Session while the Control UI renders it.

## Solution Assessment

Partly. The proposed Session row is the narrowest maintainable parity fix, but adding channel or sender identity is not yet the best-defined solution until maintainers choose privacy, redaction, naming, and cross-channel semantics.
