## Summary

Keep this open: current main and the latest release still pass `sessionKey` into macOS exec approval paths but the native prompt does not render it, and the Swift request model still lacks the richer requester fields already present in the TypeScript approval contract. Prior discussion also kept this open for maintainer follow-up rather than closing it as stale or implemented.

## What This Changes

Request that the macOS exec approval popup show the triggering session and, where safe, requester context such as channel, sender, or thread.

## Best Possible Solution

Add a trimmed Session row for the existing `sessionKey`, then add maintainer-approved Channel/Sender/Thread rows from safe turn-source metadata with focused macOS prompt coverage.

## Reproduction Assessment

Yes, source-level: current main passes `sessionKey` into native exec approval requests, but `buildAccessoryView` has no Session row and the Swift model has no requester fields. I did not run the macOS alert live in this read-only review.

## Solution Assessment

Partially. The proposed Session row is the narrowest parity fix, while the wider Channel/Sender/Thread display should wait for a maintainer decision on privacy-safe requester metadata and wording.
