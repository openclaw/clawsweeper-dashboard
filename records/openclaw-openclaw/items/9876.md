## Summary

Keep open. Current main still carries `sessionKey` and `turnSource*` metadata through the exec approval path, but the macOS native prompt does not render a Session row and the Swift prompt model does not decode requester context for display.

## What This Changes

Request to show the exec approval session and requester/channel context in the macOS approval popup.

## Best Possible Solution

Add a trimmed Session row in the native prompt first, then map safe existing `turnSource*` metadata into Channel/Thread/Sender display fields after maintainer privacy and UX signoff.

## Reproduction Assessment

Yes. Source inspection on current main shows `sessionKey` reaches or is decoded by the macOS prompt path, while `buildAccessoryView` never renders it and the Swift model lacks requester metadata fields.

## Solution Assessment

Unclear for the full request. The Session row is the narrow maintainable first step, but richer requester context should reuse existing `turnSource*` fields and needs maintainer agreement before adding sender or tool-reason display.
