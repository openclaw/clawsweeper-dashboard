## Summary

Keep open. Current main still passes and decodes `sessionKey` for macOS exec approvals, but the native presenter only renders working directory, agent, executable, host, security, and ask mode; it also does not decode the existing TypeScript `turnSource*` requester metadata, so the requested requester context remains missing.

## What This Changes

Request adds session and requester/channel context to the native macOS exec approval popup.

## Best Possible Solution

Add a trimmed Session row to the native presenter, then extend the native request model to display safe existing `turnSource*` channel/thread context while deferring sender or tool-reason fields until their contract is defined.

## Reproduction Assessment

Yes. Source inspection on current main shows the prompt receives or decodes `sessionKey` but never renders it, and the Swift model lacks the requester metadata fields already present in the TypeScript approval payload.

## Solution Assessment

Unclear for the full request. The Session row is the narrow immediate change, but richer requester context should reuse existing `turnSource*` fields first and needs maintainer agreement before adding sender or tool-reason metadata to the native UI.
