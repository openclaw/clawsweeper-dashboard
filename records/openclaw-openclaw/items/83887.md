## Summary

Keep open: current main and the latest release both still have a private Twitch client-manager registry with no whole-registry test reset helper, while repo policy requires module state to be safe under `--isolate=false`. Likely related people are Tak Hoffman for the current-main registry snapshot and jaydenfyi for the original Twitch plugin registry history.

## What This Changes

The issue requests a test-only clearable export for the Twitch client-manager registry to prevent module state leaking across Vitest runs with `--isolate=false`.

## Best Possible Solution

Add a narrow test-only registry reset helper with focused coverage for `getOrCreateClientManager` reuse, and coordinate any adjacent runtime cleanup with https://github.com/openclaw/openclaw/issues/83886.

## Reproduction Assessment

Yes from source inspection: two calls to `getOrCreateClientManager` with the same account ID reuse the private module registry, and current main has no exported helper to clear that registry between tests. I did not run Vitest because this review is explicitly read-only and test runs can create cache artifacts in this checkout.

## Solution Assessment

Yes: a narrow test-only registry reset helper is the maintainable fix, matching existing test-reset patterns without changing Twitch runtime behavior. If maintainers choose to combine it with https://github.com/openclaw/openclaw/issues/83886, the runtime change should stay scoped to `removeClientManager`.
