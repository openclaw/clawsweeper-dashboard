## Summary

Keep open: current main and the latest release still have the reported progress-draft gate state bug, and https://github.com/openclaw/openclaw/pull/83128 is an open closing PR that should be reviewed or closed before this issue is resolved.

## What This Changes

The issue reports that the shared channel progress draft gate marks streaming as started before async startup finishes, leaving stale active state if startup rejects.

## Best Possible Solution

Review and land the narrow reset-on-rejection fix in https://github.com/openclaw/openclaw/pull/83128, or close that PR and keep this issue as the canonical bug until an equivalent fix merges.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: first work schedules the timer, the timer calls a rejecting `onStart` through a swallowed catch, and later `noteWork()` returns true because `started` stayed true.

## Solution Assessment

Yes. The narrow maintainable fix is to clear or defer the started state on `onStart` rejection while preserving single-flight startup; the active linked PR follows that shape.
