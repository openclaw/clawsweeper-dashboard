## Summary

Keep open: current main still has the reported progress-draft gate state bug, and the linked closing PR is open but unmerged. The right cleanup path is to review and land or replace that PR, then close this issue after the fix is on main.

## What This Changes

The issue requests fixing `createChannelProgressDraftGate` so a rejected async progress-draft startup cannot leave the gate marked as started.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/83128 with regression coverage for timer rejection, direct rejection, and retry behavior, then close this issue as fixed.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: first `noteWork()` schedules the timer, the timer calls `start().catch(() => {})`, `start()` sets `started` before `onStart` resolves, and a rejected `onStart` leaves later `noteWork()` returning true from stale state.

## Solution Assessment

Yes. Resetting `started` and `startPromise` on `onStart` rejection while preserving the synchronous single-flight guard is the narrow maintainable fix; the issue should remain open until the linked implementation lands.
