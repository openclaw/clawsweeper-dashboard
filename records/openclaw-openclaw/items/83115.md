## Summary

Keep open: the bug is still present on current main and in the latest release, and an open closing PR is already linked for the narrow fix. The issue should stay open until that PR is reviewed and merged or explicitly rejected.

## What This Changes

The issue reports that `createChannelProgressDraftGate` marks progress drafts as started before async `onStart` completes, leaving stale active state after rejection.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/83128; if it lands, close this issue with the merged commit proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: first work schedules the timer, the timer invokes a rejecting `onStart` through a swallowed catch, and later `noteWork()` returns true because `started` stayed true.

## Solution Assessment

Yes. Resetting/clearing the gate when `onStart` rejects is the narrow maintainable fix, as long as the implementation preserves the existing single-flight startup behavior.
