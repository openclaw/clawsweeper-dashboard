## Summary

Keep this PR open. Current main still treats Linux env-configured nvm, fnm, Volta, asdf, and Bun roots as generated minimal service PATH entries, and the PR remains a narrow fix for that behavior; related #75440 is complementary because it handles stale preserved PATH entries during reinstall planning rather than fresh minimal PATH generation.

## What This Changes

The PR excludes Linux version-manager roots from generated minimal daemon service PATH expectations while preserving stable user, npm/pnpm, Nix, explicit runtime-bin, and macOS behavior, with daemon PATH tests, a changelog entry, and minor merge-format touch-ups.

## Best Possible Solution

Land a current-main patch that removes Linux version-manager env/default roots from minimal service PATH generation while preserving stable user bins, PNPM_HOME/NPM_CONFIG_PREFIX, Nix profiles, explicit selected Node bin dirs, macOS behavior, service audit alignment, and focused tests/changelog; coordinate with #75440 for stale preserved PATH cleanup.

## Reproduction Assessment

Yes. On current main, calling getMinimalServicePathPartsFromEnv on Linux with HOME plus NVM_DIR, FNM_DIR, VOLTA_HOME, ASDF_DATA_DIR, or BUN_INSTALL reproduces the issue by returning those roots; service audit then treats the helper output as required PATH entries.

## Solution Assessment

Yes. Updating src/daemon/service-env.ts is the narrowest maintainable fix because gateway service generation and audit both share that helper; #75440 is a safer complement for preserved old PATH entries, not a replacement for this fresh-generation policy change.
