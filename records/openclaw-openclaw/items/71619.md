## Summary

Keep open: current main still emits Linux env/default version-manager roots as minimal service PATH entries, and this PR removes that behavior with matching test and changelog coverage. The work is not superseded by #75440, which covers stale preserved PATH entries rather than fresh minimal PATH generation.

## What This Changes

The PR splits stable and version-manager service PATH roots so Linux minimal PATH excludes nvm/fnm/Volta/asdf/Bun roots, keeps stable user/npm/pnpm/Nix roots, updates daemon PATH tests, adds a changelog entry, and carries minor merge-format updates.

## Best Possible Solution

Land a narrow current-main patch that excludes Linux version-manager env/default roots from generated minimal service PATH expectations while preserving explicit runtime bin dirs, stable user/npm/pnpm/Nix roots, macOS behavior, workspace/proc filtering, service audit alignment, and changelog/test coverage.

## Reproduction Assessment

Yes. A source-level reproduction is clear on current main: call getMinimalServicePathPartsFromEnv on Linux with HOME plus NVM_DIR, FNM_DIR, VOLTA_HOME, ASDF_DATA_DIR, or BUN_INSTALL and the helper emits those roots; service audit then uses that helper as the expected PATH set.

## Solution Assessment

Yes. Updating service-env is the narrowest maintainable fix because gateway service audit and service environment generation share that helper; the PR's direction matches the bug while leaving explicit runtime-bin PATH support in place.
