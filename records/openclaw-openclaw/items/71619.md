## Summary

Keep open: current main still includes Linux version-manager roots in the daemon minimal service PATH, so the PR addresses a real remaining bug; the current blocker is that GitHub reports the branch as dirty against main.

## What This Changes

The PR refactors daemon service PATH assembly so Linux excludes env/default nvm/fnm/Volta/asdf/Bun roots while keeping stable user/npm/pnpm/Nix roots, updates tests, and adds a changelog entry.

## Best Possible Solution

Resolve the current-main conflict and land the narrow service-env policy change that keeps Linux expected PATH limited to stable user, npm/pnpm, and Nix roots while preserving current audit/install filtering.

## Solution Assessment

Yes after rebase. Updating service-env is the narrowest maintainable fix because service audit consumes the same helper; the patch needs reconciliation with current-main service-path-policy and includeMissingUserBinDefaults changes.
