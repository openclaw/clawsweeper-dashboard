## Summary

## Best Possible Solution

Normalize Windows argv before building the respawn plan, keep the existing skip gates intact, and land only after packaged native Windows smoke shows the affected commands start without manual `NODE_OPTIONS`.

## Reproduction Assessment

Yes, source-reproducible: the linked issue provides Windows crash steps and current main still has the Windows respawn bypass. The PR-introduced argv-ordering problem is also source-reproducible because `entry.ts` normalizes Windows argv only after `buildCliRespawnPlan()` runs.

## Solution Assessment

No: the stack-size mitigation is directionally narrow, but this diff builds the new Windows child argv before existing Windows normalization and still lacks packaged after-fix proof. The safer path is to normalize first, test that compatibility case, and then prove the packaged Windows CLI flows.
