## Summary

Keep open. Current main still lacks the cron JSON/quiet-helper fix, and this PR is not merge-ready because its CLI-level retry can replay mutating gateway RPCs while the branch also carries broad unrelated and security-sensitive changes. Reviewed PR: https://github.com/openclaw/openclaw/pull/49109

## What This Changes

This PR changes cron CLI helper status checks and CLI gateway transport retry behavior, adds cron/gateway tests, and currently also carries broad status, plugin registry, SSRF, build-entrypoint, generated-baseline, and channel/plugin test changes.

## Best Possible Solution

Split the work: land #54475 or an equivalent structured gateway-layer pre-hello fix, then apply a small current-main cron patch that skips the helper in JSON mode and uses the existing probe-mode gateway options with `expectFinal: false`.

## Reproduction Assessment

Yes. Static current-main inspection reproduces the cron path: `cron add --json` prints JSON and then still invokes the helper, which performs a plain `cron.status` call. The transient gateway close path has a focused unit-level reproduction in related #54475.

## Solution Assessment

No. The cron helper direction is right, but the PR’s blanket CLI retry is not the narrow maintainable fix because it is message-text based and method-agnostic; the safer alternative is a structured gateway-layer pre-hello tolerance plus a separate cron helper patch rebased onto the current runtime split.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Avoid retrying mutating gateway RPCs:** `src/cli/gateway-rpc.ts:50-68`
  - body: The retry loop retries any method when `callGateway` rejects with matching close/timeout text. Because `callGateway` sends the RPC after `hello-ok`, a close before the response can still happen after the gateway processed a mutating command like `cron.add`, so this can duplicate jobs or repeat side effects. Keep retry scoped to a structured pre-request/pre-hello signal, or use the focused gateway-layer fix instead.
  - confidence: 0.8
