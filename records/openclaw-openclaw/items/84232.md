## Summary

Keep this PR open, but do not merge as-is: the intended Slack source-reply fix is well supported, yet the new best-effort override is broader than the current-source route it means to protect.

## What This Changes

The PR forces `message_tool_only` source replies that omit a target or target the current source conversation to best-effort delivery before target normalization, with Slack-shaped regression coverage.

## Best Possible Solution

Scope the best-effort downgrade to omitted-target or verified current-source routes, preserve required durability for any explicit non-current route, and land with the existing Slack proof plus a regression for the channel-only edge case.

## Reproduction Assessment

Yes, source inspection and linked issue evidence give a high-confidence path: `bestEffort:false` makes message sends required, required durable delivery demands `reconcileUnknownSend`, and Slack lacks that capability in the reported path.

## Solution Assessment

No, not yet. The chosen source-reply best-effort path is narrow and plausible, but the implementation needs an explicit-channel guard before it is the safest solution.
