## What This Changes

The PR changes `runMessageAction` so implicit and explicit-current-source `message_tool_only` sends use best-effort delivery, with Slack-shaped regression tests preserving required delivery for explicit external targets.

## Best Possible Solution

Land this after the source-reply policy only downgrades true current-source replies and tests prove explicit external or channel-only routes keep required durable semantics.

## Reproduction Assessment

Yes. The current-main source path still requires `reconcileUnknownSend` for required sends, and the PR body includes real Slack Web API proof that the patched source-reply cases deliver while explicit external targets still fail closed.

## Solution Assessment

No, not quite. The direction is right, but the current implementation only protects explicit target fields; the safer fix is to classify explicit channel/provider routes before mutating `bestEffort`.

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve required delivery for explicit channel routes:** `src/infra/outbound/message-action-runner.ts:609-612`
  - body: `applyImplicitSourceReplySendPolicy` only treats `target`/`to`/`channelId`/`targets` as explicit routes, so a `message_tool_only` send with `channel` set and no target still gets `bestEffort = true` before normalization infers a target from the current context. If cross-route sends are allowed, that downgrades an explicit routed send with `bestEffort:false` and bypasses the required durable `reconcileUnknownSend` guard this PR says external sends should keep. Treat explicit channel routes as explicit unless they are proven to be the current source conversation, and add a regression for that shape.
  - confidence: 0.87
