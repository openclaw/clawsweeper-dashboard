## Summary

Keep this PR open: the direction is useful, but it is not merge-ready because it leaves the explicit current-source target shape from the linked Slack evidence on the required durable path, and the PR body has no after-fix real Slack proof.

## What This Changes

The PR changes `runMessageAction` to force best-effort delivery for `message_tool_only` sends without explicit target fields and adds Slack-like core-send regression tests.

## Best Possible Solution

Treat targets that resolve to the current source conversation as source replies for this policy, keep required durable semantics for true external explicit sends, and prove the Slack `message_tool_only` path with redacted real output.

## Reproduction Assessment

Yes at source level: current main maps `bestEffort:false` to required delivery, required delivery requires `reconcileUnknownSend`, and Slack does not declare that capability. I did not run live Slack proof in this read-only review.

## Solution Assessment

No. The PR covers omitted-target source replies, but the linked evidence shows the failing Slack path can include an explicit current-channel target; the safer solution is to recognize current-source targets or prevent source replies from carrying required durable args.
