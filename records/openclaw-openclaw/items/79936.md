## Summary

Keep open. Current main still has English-only CLI setup/onboarding prompt copy, but this is a new localization feature request that needs maintainer direction on CLI locale policy and translation architecture rather than conservative cleanup.

## What This Changes

Add CLI localization for setup/onboarding wizard prompts, including a translation helper and zh-CN/zh-TW locale coverage across wizard, onboard, and channel setup files.

## Best Possible Solution

Design a CLI localization layer for wizard/onboarding copy with explicit locale resolution, English fallback behavior, coverage for prompts/notes/progress/validators, focused regression tests, and user-facing docs.

## Solution Assessment

No final implementation is proposed yet. The issue's t() wrapper sketch is useful inventory, but the maintainable solution needs a locale policy, fallback contract, and broader copy coverage before a narrow code change is safe.
