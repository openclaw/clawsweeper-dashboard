## Summary

Keep open. Current main still has English-only setup/onboarding prompts, and this is a valid core CLI first-run UX feature that needs localization design rather than stale cleanup or ClawHub routing.

## What This Changes

Request CLI localization support for setup and onboarding wizard prompts that are currently hardcoded in English.

## Best Possible Solution

Design a CLI localization layer for wizard and onboarding copy with explicit locale selection, fallback behavior, regression tests, and docs before wrapping the prompt inventory.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection does confirm the current-main state: setup and onboarding prompt paths still render raw English strings without a CLI i18n layer.

## Solution Assessment

No final implementation is proposed here. The issue's t() wrapper sketch is a useful inventory, but the maintainable solution also needs locale policy, fallback rules, and a decision on sharing versus separating the existing UI i18n catalog.
