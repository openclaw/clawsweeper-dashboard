## Summary

Keep open. Current main still lacks CLI i18n and still renders setup/onboarding prompts as raw English strings; the related open localization PR is useful context, but it is not merged and does not fully settle the requested CLI locale policy or full prompt coverage.

## What This Changes

The issue requests a CLI localization layer and Chinese locale maps for setup/onboarding wizard prompts across `src/wizard`, `src/commands`, and `src/flows`.

## Best Possible Solution

Design and land a CLI localization policy that covers locale selection, persistence or environment override, fallback behavior, security/dynamic copy, focused regression tests, docs, and coordinated review of https://github.com/openclaw/openclaw/pull/44695 or a narrower replacement.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection does confirm the reported current-main state: the setup and onboarding paths still render raw English strings without a CLI i18n module.

## Solution Assessment

No final implementation is proposed here. The issue's `t()` wrapper sketch is useful inventory, but the maintainable solution also needs locale policy, fallback rules, dynamic/security-copy coverage, docs, tests, and coordination with the existing open PR.
