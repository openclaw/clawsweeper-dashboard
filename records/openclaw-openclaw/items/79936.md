Keep open. Current main still lacks a CLI setup/onboarding i18n layer, and this is a valid core first-run UX feature that needs maintainer design rather than cleanup closure.

Add localization support for CLI setup/onboarding wizard prompts so non-English users do not see hardcoded English copy.

## Best Possible Solution

Establish a CLI locale contract, catalog shape, fallback behavior, tests, and docs, then land a focused implementation that covers setup/onboarding prompts and security-facing copy consistently.

## Solution Assessment
