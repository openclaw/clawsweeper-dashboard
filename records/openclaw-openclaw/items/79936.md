## Summary

Keep open. Current main still has English-only CLI setup/onboarding prompts, and this is a valid core first-run UX feature request that needs localization design rather than stale cleanup or ClawHub routing.

## What This Changes

The issue requests CLI setup/onboarding localization by adding a wizard i18n layer and wrapping hardcoded English prompt text across setup, onboard, channel, skill, hook, plugin, and remote-gateway flows.

## Best Possible Solution

Design a CLI-localization layer for wizard/onboarding copy with an explicit locale-selection policy, fallback behavior, regression tests, and docs before wrapping the prompt inventory.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection does confirm the reported current-main state: the setup/onboarding prompt paths still render raw English strings without a CLI i18n layer.

## Solution Assessment

No final implementation is proposed here. The issue's t() wrapper sketch is a useful inventory, but the best maintainable solution also needs locale selection, fallback rules, coverage for non-prompt wizard text, and a decision on sharing versus separating the existing UI i18n catalog.
