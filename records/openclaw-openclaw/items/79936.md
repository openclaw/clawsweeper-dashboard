## Summary

Keep open. Current main still has English-only CLI setup/onboarding prompts, and the related open localization PR is a partial implementation vehicle rather than proof that the broader 11-file zh-CN/zh-TW request is complete.

## What This Changes

Add CLI localization for setup/onboarding wizard prompts across `src/wizard`, `src/commands`, and `src/flows`, including locale maps and `t()` wrappers for zh-CN and zh-TW.

## Best Possible Solution

Design and land a CLI localization layer with explicit locale selection, fallback/persistence rules, generated or maintainable locale catalogs, regression coverage, docs, and a clear relationship to the existing UI i18n pipeline.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main confirms the reported state: setup/onboarding prompt paths still render raw English strings without a production CLI i18n layer.

## Solution Assessment

Unclear as a final implementation. The issue's `t()` wrapper inventory is useful, but the maintainable path also needs locale source, fallback behavior, persistence or override policy, tests, docs, and coordination with https://github.com/openclaw/openclaw/pull/44695.
