## Summary

Keep open. Current main still has path-backed session persistence: `sessions.json` metadata plus per-session JSONL transcript files, with no external storage backend contract or config surface. The prior ClawSweeper review and the later maintainer comment both align with the current source state.

## What This Changes

Request support for a pluggable external session storage backend so multiple OpenClaw instances can share session metadata and transcripts without load-balancer stickiness.

## Best Possible Solution

Design a core, versioned session persistence contract that covers metadata and transcript semantics together, keeps filesystem storage as the default, adds config/schema/docs and migration/doctor support, proves one multi-instance-safe backend, and only then exposes a narrow plugin-facing backend registration surface for additional stores.

## Reproduction Assessment

Not applicable as a bug reproduction. The feature gap is high-confidence by source and docs inspection: current main still resolves, reads, writes, and documents sessions as filesystem paths.

## Solution Assessment

Unclear for the exact proposed shape. The external-backend direction is valid, but a Redis/MongoDB-style key-value interface alone is not enough unless it also handles transcript JSONL semantics, concurrency, pruning, migration, and credentials.
