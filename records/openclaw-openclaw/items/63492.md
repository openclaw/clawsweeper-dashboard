## Summary

Keep open. Current main and the latest release still use a path-only filesystem session store with local JSON metadata and JSONL transcripts, and related PostgreSQL/pluggable-store reports were closed in favor of this issue as the canonical tracker rather than implementing the feature.

## What This Changes

Request adds a pluggable external session-storage backend and configuration for Redis/MongoDB-style stores so multiple OpenClaw instances can share session state.

## Best Possible Solution

Define a versioned gateway-owned session persistence contract that covers metadata and transcripts, keep filesystem storage as the compatibility default, prove one multi-instance-safe backend, and update config/schema/docs, migration or doctor behavior, and SDK/plugin boundaries together.

## Reproduction Assessment

Not applicable: this is a feature and architecture request, not a reproducible bug. Source, docs, and release inspection give high confidence that current main and the latest release still lack an external session-storage backend selector or adapter API.

## Solution Assessment

Unclear for the exact proposed `get`/`set`/`delete` wrapper. The external-backend direction is reasonable, but the maintainable solution must include metadata, transcripts, ordering, consistency, pruning, migration, credentials, and SDK or plugin ownership.
