## Summary

Keep open. Current main still persists session metadata through path-backed `sessions.json` files and per-session JSONL transcripts, while the requested multi-instance external storage contract remains unimplemented and needs maintainer design.

## What This Changes

Add a pluggable external session storage backend so multiple OpenClaw gateway instances can share session state without sticky routing.

## Best Possible Solution

Define a versioned core session persistence contract that covers both metadata and transcript semantics, keeps filesystem storage as the default, proves one multi-instance-safe backend, and updates config/schema/docs plus migration and doctor support.

## Reproduction Assessment

Not applicable as a bug reproduction. The feature gap is high-confidence from source and docs inspection: current main still resolves, reads, writes, tests, and documents sessions as local paths and JSONL files.

## Solution Assessment

Unclear for the exact proposed `get`/`set`/`delete` shape. The external-backend direction is valid, but the maintainable solution must include transcript ordering, locking, pruning, migration, and credential handling rather than only a key-value metadata adapter.
