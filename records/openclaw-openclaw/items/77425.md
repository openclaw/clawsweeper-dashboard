## Summary

Keep open: current main still has the ClawDock dashboard host-port bug and this PR fixes the right boundary, but the latest PR head still uses Bash-only regex captures in a helper documented for zsh sourcing.

## What This Changes

The PR adds ClawDock shell logic to read `OPENCLAW_GATEWAY_PORT` from `.env`, rewrite `clawdock-dashboard` URLs to `localhost:<host_port>`, and add an Unreleased changelog entry.

## Best Possible Solution

Keep the ClawDock URL rewrite, but parse the host-published port with bash/zsh-portable shell code and add a focused helper smoke for numeric, IPv4 publish, bracketed IPv6 publish, missing, and malformed values.

## Reproduction Assessment

Yes. Current main’s bug is source-reproducible from the Compose port mapping and direct URL open path, and the PR blocker is source-reproducible from the documented zsh install path plus zsh’s `BASH_REMATCH` behavior.

## Solution Assessment

No. Rewriting the URL in ClawDock is the narrow maintainable fix, but the parser should avoid Bash-only capture arrays in a helper that users source from zsh.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use a shell-portable port parser:** `scripts/clawdock/clawdock-helpers.sh:184-186`
  - body: ClawDock’s public install sources this file from `.zshrc`, but these new branches read Bash’s `BASH_REMATCH`. In zsh, `=~` populates `match` unless the `BASH_REMATCH` option is enabled, so `OPENCLAW_GATEWAY_PORT=127.0.0.1:3200` or `[::1]:3200` falls back to `18789` and leaves the remapped dashboard URL broken.
  - confidence: 0.91
