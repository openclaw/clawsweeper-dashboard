## Summary

Keep open: the PR fixes the main ClawDock host-port direction, but the latest diff adds Bash-only `BASH_REMATCH` capture for Compose-style port values while ClawDock is documented as sourced from `.zshrc`, leaving that advertised path broken for zsh users.

## What This Changes

This PR changes `scripts/clawdock/clawdock-helpers.sh` to read `OPENCLAW_GATEWAY_PORT` from `.env`, rewrite `clawdock-dashboard` URLs to localhost on the host-published port, and add a changelog entry.

## Best Possible Solution

Keep the ClawDock URL rewrite, but make the `.env` gateway-port parser shell-portable for the documented zsh sourcing path while preserving numeric, `host:port`, and `[::1]:port` behavior.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main opens the URL printed from inside the Docker CLI container while Docker may publish a different host port. The PR-specific blocker is also source-reproducible from the new `BASH_REMATCH` use under the documented `.zshrc` sourcing path, though no live smoke was run.

## Solution Assessment

No. Rewriting at the ClawDock helper boundary is the right approach, but the proposed parser should avoid Bash-only capture variables or explicitly run under Bash so the documented zsh install path handles Compose-style ports.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Avoid Bash-only captures in the helper:** `scripts/clawdock/clawdock-helpers.sh:184-186`
  - body: ClawDock is installed by sourcing this file from `.zshrc`, and earlier helper history specifically preserved zsh compatibility. In native zsh, these `BASH_REMATCH` captures are not populated by default, so `OPENCLAW_GATEWAY_PORT=127.0.0.1:3200` or `[::1]:3200` falls through to the default `18789` and `clawdock-dashboard` still opens the wrong host port for the Compose-style values this PR claims to support. Please parse the suffix port with shell-portable parameter expansion or another zsh-compatible helper.
