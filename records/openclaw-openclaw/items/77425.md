## Summary

Keep this PR open: it targets a valid ClawDock bug, but the latest head still uses Bash-only regex capture state in a helper documented for zsh sourcing, and the proof does not show the actual dashboard URL rewrite with a remapped gateway port.

## What This Changes

The PR adds a ClawDock helper that reads `OPENCLAW_GATEWAY_PORT`, rewrites the dashboard URL to `localhost:<host_port>`, and adds a changelog entry for the Docker remapped-port fix.

## Best Possible Solution

Keep the ClawDock-level URL rewrite, replace the port parser with shell-portable logic aligned with the existing core contract, and require after-fix dashboard output for a remapped port before merge.

## Reproduction Assessment

Yes. Source inspection shows current main publishes a configurable host gateway port but opens the URL reported from inside the container; the PR-head portability regression is also source-reproducible from the documented zsh install path and zsh regex contract.

## Solution Assessment

No. Rewriting at the ClawDock helper boundary is the right direction, but the parser should not depend on bash-only capture variables when the documented install path sources the helper from zsh.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Use portable regex captures in ClawDock:** `scripts/clawdock/clawdock-helpers.sh:183-186`
  - body: ClawDock's documented install path sources this helper from `.zshrc`, so the shebang does not force bash.   - confidence: 0.91
