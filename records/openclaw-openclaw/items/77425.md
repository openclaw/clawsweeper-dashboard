## Summary

Keep open: the PR targets a real ClawDock dashboard bug, but the latest head still has a blocking zsh portability defect and the external contribution lacks required after-fix real behavior proof.

## What This Changes

The PR adds a ClawDock helper to read `OPENCLAW_GATEWAY_PORT` from `.env`, rewrites `clawdock-dashboard` container URLs to `localhost:<port>`, and adds an Unreleased changelog entry.

## Best Possible Solution

Keep the ClawDock-level URL rewrite, but use a shell-portable parser aligned with the existing core port contract and require real ClawDock output proving the remapped dashboard URL works before merge.

## Reproduction Assessment

Yes, by source inspection: current main publishes a configurable host gateway port but opens the URL produced inside the container network, and the PR head still has a zsh-specific failure path. I did not run a full Docker ClawDock live scenario.

## Solution Assessment

No. Rewriting the URL in ClawDock is the right boundary, but using Bash-only regex capture state in a helper documented for `.zshrc` sourcing is not the narrowest maintainable implementation.

- **[P2] Use portable regex captures in ClawDock:** `scripts/clawdock/clawdock-helpers.sh:183-186`
  - body: ClawDock's documented install path sources this helper from `.zshrc`, so the shebang does not force bash.
