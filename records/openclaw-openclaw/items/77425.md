## Summary

Keep this PR open: it fixes the right ClawDock boundary for #77344, but the latest head still uses Bash-only regex capture in a helper documented for zsh sourcing, so the Compose-style port cases can still fall back to 18789 for normal zsh users.

## What This Changes

The PR adds ClawDock `.env` gateway-port parsing, rewrites `clawdock-dashboard` container URLs to `localhost:<host_port>`, and adds a changelog entry.

## Best Possible Solution

Keep the ClawDock host-port rewrite, but make the port parser portable across bash and zsh while matching the core `resolveGatewayPort` Compose-style contract.

## Reproduction Assessment

Yes. Source inspection shows the original ClawDock bug on current main, and the remaining PR regression is source-reproducible from the documented zsh install path plus zsh's published `=~` capture semantics.

## Solution Assessment

No. Rewriting the URL in ClawDock is the right boundary, but the current PR implementation should avoid `BASH_REMATCH` or explicitly handle zsh's default `$match` behavior before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use shell-portable regex captures:** `scripts/clawdock/clawdock-helpers.sh:183-186`
  - body: ClawDock is documented as sourced from `.zshrc`, but these branches read Bash's `BASH_REMATCH`.
