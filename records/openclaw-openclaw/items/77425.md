## Summary

Keep open: the PR targets the right ClawDock bug, but the latest helper uses Bash-only `BASH_REMATCH` for the Compose-style cases while ClawDock is documented as sourced from zsh, so the default install path can still fall back to port 18789.

## What This Changes

The PR adds a ClawDock helper to read `OPENCLAW_GATEWAY_PORT`, rewrites `clawdock-dashboard` URLs to `localhost:<host_port>`, and adds a changelog entry.

## Best Possible Solution

Keep the ClawDock boundary fix, but make the port parser shell-portable for bash/zsh and add a focused shell parser check before merge.

## Reproduction Assessment

Yes. Source inspection plus the zsh manual gives a high-confidence path: source the helper from zsh as documented, set `.env` to `OPENCLAW_GATEWAY_PORT=127.0.0.1:3200`, and the PR's `BASH_REMATCH[1]` capture is not populated unless zsh's compatibility option is set.

## Solution Assessment

No. Rewriting the URL in ClawDock is the right boundary, but the current parser should avoid Bash-only capture arrays or explicitly handle zsh's `$match` behavior.
