## Summary

Keep open: the underlying ClawDock host-port bug is valid, but the latest PR head still uses Bash-only `BASH_REMATCH` in a helper documented for zsh sourcing, so Compose-style `host:port` values can still fall back to `18789` on the normal install path.

## What This Changes

The PR adds ClawDock logic to read `OPENCLAW_GATEWAY_PORT` from the project `.env`, rewrite `clawdock-dashboard`'s container URL to `localhost:<host_port>`, and add an Unreleased changelog entry.

## Best Possible Solution

Keep the ClawDock-level URL rewrite, but make `_clawdock_read_env_gateway_port` shell-portable for bash and zsh and cover the parser shapes before merge.

## Reproduction Assessment

Yes. Current main and the PR diff give a high-confidence source path: Compose can publish a different host port, `clawdock-dashboard` gets a container-side dashboard URL, and the PR's new parser still uses Bash-only capture variables on the documented zsh install path.

## Solution Assessment

No. Rewriting the URL in ClawDock is the right boundary, but the parser should use shell-portable extraction rather than depending on `BASH_REMATCH` in a zsh-sourced helper.

- **[P2] Use a shell-portable port parser:** `scripts/clawdock/clawdock-helpers.sh:184-187`
  - body: ClawDock is installed by sourcing this helper from `.zshrc`, but these branches read Bash's `BASH_REMATCH`.
