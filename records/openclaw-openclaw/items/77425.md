## Summary

Keep open: the PR targets a valid ClawDock dashboard bug on current main, but the latest head still uses Bash-only capture state in a helper documented for zsh sourcing, so Compose-style host port values can still fall back to the default port.

## What This Changes

The PR adds a ClawDock `.env` gateway-port reader, rewrites `clawdock-dashboard` output to `localhost:<host port>`, and adds a changelog entry for #77344.

## Best Possible Solution

Keep the ClawDock URL rewrite, but parse `OPENCLAW_GATEWAY_PORT` with shell-portable logic and validate bare, IPv4 bind, IPv6 bind, missing, and malformed values under both bash and zsh.

## Reproduction Assessment

Yes, by source inspection: Compose can publish a host port different from container port 18789 while current `clawdock-dashboard` opens the URL printed inside the CLI container. The PR-specific blocker is also source-reproducible from zsh docs because zsh does not populate `BASH_REMATCH` by default.

## Solution Assessment

No. Rewriting the dashboard URL at the ClawDock boundary is the right fix, but the parser should avoid Bash-only captures because the supported install path sources the helper from `.zshrc`.
