## Summary

Keep open: the PR targets the right ClawDock boundary, but latest head still has a shell-portability bug on the documented zsh install path because the new parser reads Bash-only regex capture state for Compose-style host-port values.

## What This Changes

The PR adds a ClawDock helper to read `OPENCLAW_GATEWAY_PORT` from `.env`, rewrites `clawdock-dashboard` output to `localhost:<host_port>`, and adds an Unreleased changelog entry.

## Best Possible Solution

Keep the ClawDock URL rewrite and changelog, but replace the parser with shell-portable extraction that matches the existing Compose-style port contract before merge.

## Reproduction Assessment

Yes by source inspection. Current main can emit and open the container-side `18789` dashboard URL while Compose publishes a different host port, and the PR head's remaining parser bug follows from the documented zsh capture behavior.

## Solution Assessment

No. Rewriting the URL in ClawDock is the right fix boundary, but depending on `BASH_REMATCH` is not the narrowest maintainable implementation for a helper that is documented to be sourced from zsh.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use a shell-portable port parser:** `scripts/clawdock/clawdock-helpers.sh:184-186`
  - body: ClawDock is installed by sourcing this helper from `.zshrc`, but these branches read Bash's `BASH_REMATCH`. In zsh, regex captures go to `match` unless the `BASH_REMATCH` option is set, so `127.0.0.1:3200` or `[::1]:3200` can parse as empty and fall back to `18789`, leaving remapped dashboard URLs broken on the documented install path.
  - confidence: 0.91
