## Summary

Keep this PR open: the core ClawDock direction is useful and matches the linked Docker dashboard bug, but the proposed helper would break an already-supported `OPENCLAW_GATEWAY_PORT` form and the changelog points at the wrong PR number.

## What This Changes

The PR adds a ClawDock helper to read `OPENCLAW_GATEWAY_PORT`, rewrites the container-reported dashboard URL to `localhost:<port>`, adds an empty-URL fallback, and adds a changelog note.

## Best Possible Solution

Normalize the `.env` value with the same numeric and Compose-style host-publish semantics as `resolveGatewayPort`, then rewrite the dashboard URL and fix the changelog reference before landing.

## Reproduction Assessment

Yes for the PR defect from source inspection: current tests accept `OPENCLAW_GATEWAY_PORT=127.0.0.1:18789` and `[::1]:28789`, while the PR helper would return those raw strings into the URL replacement. I did not run a live Docker reproduction.

## Solution Assessment

No. Rewriting the container URL at the ClawDock helper boundary is the right narrow approach, but the helper must parse the supported host-publish forms before substituting the port.
