## Summary

Keep open: current main still has the ClawDock dashboard host-port gap, and this PR fixes the right boundary but is not merge-ready because the new helper substitutes raw `.env` values that can already be valid Compose host-publish strings.

## What This Changes

The PR adds a ClawDock helper to read `OPENCLAW_GATEWAY_PORT`, rewrites `clawdock-dashboard` URLs to `localhost:<port>`, adds an empty-output fallback, and adds a changelog entry.

## Best Possible Solution

Normalize `.env` gateway port parsing to the numeric host port, rewrite the dashboard URL at the ClawDock helper boundary, correct the changelog reference, and then land with targeted shell smoke plus the changed gate.

## Reproduction Assessment

Yes, from source and a focused shell substitution check: supported `OPENCLAW_GATEWAY_PORT` values like `127.0.0.1:3200` and `[::1]:3200` become invalid `http://localhost:...` URLs under the PR rewrite. I did not run a live Docker reproduction.

## Solution Assessment

No. Rewriting the container-reported URL in ClawDock is the right narrow fix, but the helper must preserve the current gateway port contract by parsing Compose-style host-publish values before substitution.
