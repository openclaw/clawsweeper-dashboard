## Summary

Keep open. Current main and the latest release still lack a structural per-channel or per-spawn completion-announce suppression policy, and the remaining work is a public completion-audience/config decision rather than an already-implemented cleanup.

## What This Changes

The issue requests a public per-channel or generalized policy to suppress or reroute sub-agent completion announcements away from chat channels.

## Best Possible Solution

Define one backward-compatible completion-audience contract with current channel delivery as the default, explicit internal/parent/none behavior, optional per-channel policy, and coordinated schema/types/docs/tests across the existing announce delivery paths.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction, but yes for a source-level feature gap: current main's config schema, spawn schema, docs, and delivery code show no deterministic per-channel suppression without relying on model output tokens.

## Solution Assessment

Is this the best way to solve the issue? The requested direction is valid, but the exact sketched config key is not yet proven to be the best permanent API; a unified completion-audience contract should cover suppression while coordinating with routing and deterministic skip work.
