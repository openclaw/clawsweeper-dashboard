## Summary

Keep open. Current main still lacks supported per-channel or public per-spawn suppression for sub-agent completion announcements, and the live discussion plus related open trackers show this still needs maintainer API/product direction rather than cleanup closure.

## What This Changes

Request adds configurable per-channel or general deny-list policy to suppress or reroute sub-agent completion announcements from channel-bound requester sessions.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior, supports channel-aware suppression and coordinated per-spawn semantics, applies before user-facing delivery decisions, and updates config schema/types, docs, and focused delivery tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high-confidence evidence that current main has no supported per-channel or public per-spawn suppression policy and still external-delivers top-level completion announces by default.

## Solution Assessment

Unclear until maintainers choose the public primitive. The delivery layer plus `agents.defaults.subagents` and `sessions_spawn` are the right implementation surfaces, but the design should coordinate with https://github.com/openclaw/openclaw/issues/8299, https://github.com/openclaw/openclaw/issues/27445, and https://github.com/openclaw/openclaw/issues/69492.
