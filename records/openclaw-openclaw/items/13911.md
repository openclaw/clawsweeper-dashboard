## Summary

Keep open: current main and the latest release still lack a public per-channel or per-spawn policy to suppress sub-agent completion announcements, and the related open issues cover overlapping global suppression, parent routing, and hook ideas rather than replacing this channel-specific request.

## What This Changes

The issue requests a public way to suppress or reroute sub-agent completion announcements per channel, especially for Telegram-like messaging channels.

## Best Possible Solution

Keep this open as the per-channel design item and add a backward-compatible completion delivery policy that defaults to current behavior, can suppress or make completions internal-only for selected channels or spawn calls, and applies before both agent delivery and direct fallback sends.

## Reproduction Assessment

Yes, at source level: current main exposes no per-channel sub-agent announce suppression config or public sessions_spawn parameter, while top-level completion delivery still resolves a user-facing requester route. I did not run a live Telegram/iMessage scenario because the missing surface is directly visible in schema, docs, tests, and delivery code.

## Solution Assessment

Unclear as proposed: a channels.telegram.subagentAnnounce key is plausible, but the best fix should be a maintainer-approved completion delivery policy that preserves current defaults and coordinates with global suppression and parent-routing proposals.
