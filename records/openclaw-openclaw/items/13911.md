## Summary

Keep open. Current main and the latest v2026.6.6 release still lack a public per-channel or equivalent policy for suppressing sub-agent completion announces, and the overlapping open issues do not fully replace this channel-level product/API decision.

## What This Changes

Request to add per-channel or equivalent config/policy to suppress or reroute sub-agent completion announces away from user-facing channels such as Telegram.

## Best Possible Solution

Define one backward-compatible completion delivery policy that defaults to current behavior but can mark selected channel or spawn completions as suppressed or internal-only, then apply it before both requester-agent handoff and direct fallback sends with schema, docs, and focused tests updated together.

## Reproduction Assessment

Yes, at source level: current main exposes no public `sessions_spawn` announce/silent parameter and no `agents.defaults.subagents` or per-channel suppression policy, while docs still describe only token-driven suppression. I did not run a live Telegram or iMessage scenario because the missing public surface is directly visible in source, docs, and tests.

## Solution Assessment

Unclear as a one-off Telegram key. The maintainable solution is a generic, backward-compatible delivery policy coordinated with the open suppression, routing, and hook proposals so channel defaults, per-spawn overrides, and fallback delivery do not diverge.
