## Summary

Keep open. Current main still does not expose a per-channel or per-spawn sub-agent completion announce suppression policy, and the active discussion points to an unsettled product/API decision rather than an already-implemented or safely duplicate cleanup.

## What This Changes

Request a supported config or policy to suppress sub-agent completion announcements for selected channels such as Telegram while preserving normal interactive replies.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current requester-chat delivery, supports channel and/or spawn-level suppressed or internal-only handling, applies before agent delivery and fallback sends, and updates config schema/types, generated docs, public docs, and focused delivery tests.

## Reproduction Assessment

Yes, for the current behavior and missing capability. Source, docs, and tests on current main show no supported suppression key or public `sessions_spawn` announce policy, while completion delivery still resolves external requester routes and direct fallback sends.

## Solution Assessment

Unclear pending maintainer API direction. The requested delivery-layer control is pointed at the right area, but the primitive should be chosen alongside https://github.com/openclaw/openclaw/issues/8299, https://github.com/openclaw/openclaw/issues/64658, https://github.com/openclaw/openclaw/issues/27445, and https://github.com/openclaw/openclaw/issues/69492.
