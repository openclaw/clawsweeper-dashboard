## Summary

Keep open. Current main still lacks a public per-channel or per-spawn policy for suppressing sub-agent completion announces, and related open issues cover general suppression or parent routing rather than replacing this channel-specific request.

## What This Changes

Add per-channel control to suppress or reroute sub-agent completion announcements while preserving normal interactive replies in that channel.

## Best Possible Solution

Define one backward-compatible completion-delivery policy that defaults to current behavior, can express channel-specific suppression or internal-only delivery, and applies before every announce delivery and fallback path.

## Reproduction Assessment

Yes, at source level. Current main exposes no per-channel sub-agent announce suppression in config or `sessions_spawn`, while docs and code still rely on exact-token suppression and existing delivery paths.

## Solution Assessment

Unclear until maintainers choose the public contract. A per-channel knob is plausible, but it should be coordinated with the broader suppression and parent-routing issues instead of adding a competing policy surface.
