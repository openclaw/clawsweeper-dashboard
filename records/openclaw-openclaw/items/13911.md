## Summary

Keep open: current main and the latest shipped tag still lack a supported per-channel or public per-spawn policy for suppressing or rerouting sub-agent completion announcements, and the related open suppression, routing, and audience-classification threads have not replaced this request.

## What This Changes

Request a config or public spawn policy to suppress sub-agent completion announcements per channel while preserving normal interactive channel replies.

## Best Possible Solution

Add one backward-compatible completion delivery policy that defaults to current behavior, supports channel-aware suppression plus coordinated per-spawn semantics, and aligns with the open suppression, parent-routing, and audience-classification work before updating config, docs, and focused tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source and latest-tag inspection give high confidence that no supported per-channel or public per-spawn suppression policy exists.

## Solution Assessment

Unclear until maintainers choose the public primitive. The maintainable direction is a single backward-compatible delivery policy threaded through config schema/types, `sessions_spawn`, announce delivery, docs, and tests rather than relying on model output tokens.
