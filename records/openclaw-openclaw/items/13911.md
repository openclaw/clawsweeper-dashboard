## Summary

Keep open. Current main still lacks a structural per-channel or per-spawn policy for suppressing sub-agent completion announces, and the existing behavior is documented as requester-chat delivery rather than a broken contract.

## What This Changes

The issue requests a framework-level way to suppress or reroute sub-agent completion announcements per channel or spawn so channel-bound sessions are not cluttered with raw completion output.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior, can mark selected sub-agent completions as internal-only or suppressed, and is applied before both announce-agent delivery and direct fallback sends.

## Reproduction Assessment

Yes. Source inspection of current main shows the missing capability: strict config has no suppression key, `sessions_spawn` exposes no announce policy, and the delivery path still resolves external requester targets with direct fallback sends.

## Solution Assessment

Unclear. The request targets the right delivery layer, but the exact public config/API should be chosen with maintainer direction and coordinated with #69492 so all completion delivery paths honor the same policy.
