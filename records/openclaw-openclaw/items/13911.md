## Label Justifications

- `P2`: Multiple users report disruptive raw completion delivery across channels, but this remains a bounded product/API improvement rather than an emergency.
- `impact:message-loss`: The requested policy directly determines whether sub-agent completion messages reach, bypass, reroute from, or are suppressed on user-facing channels.

## Summary

Current main still automatically routes sub-agent completions toward the requester and has no structural per-channel suppression policy. The request remains useful and distinct from the older global/per-spawn suppression issue, but the public policy shape and precedence need maintainer product direction before implementation.

## What This Changes

Add a structural policy to suppress or reroute sub-agent completion announcements for selected channels while preserving ordinary interactive channel replies.

## Best Possible Solution

Define one backward-compatible completion-audience model with current user delivery as the default, explicit internal-only and suppressed modes, and clear precedence across global, per-channel, and per-spawn policy; apply the decision before all primary and fallback delivery paths.

## Reproduction Assessment

Yes at source level: current main resolves user-facing completion delivery and its strict configuration surfaces contain no per-channel suppression policy, matching independent Telegram and iMessage reports. A live current-main transport reproduction was not run during this read-only review.

## Solution Assessment

Yes in direction, but not as a Telegram-specific boolean alone. A generic completion-audience policy coordinated with global and per-spawn controls is the narrower maintainable solution and avoids competing sources of truth.
