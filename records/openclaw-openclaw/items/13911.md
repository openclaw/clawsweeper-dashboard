## Summary

Keep open. Current main still lacks a supported per-channel or public per-spawn suppression/reroute policy for sub-agent completion announcements, and the active related work shows this needs maintainer API direction rather than cleanup closure.

## What This Changes

The issue asks for framework-level channel-aware suppression or rerouting of sub-agent completion announcements so parent agents can decide what reaches chat.

## Best Possible Solution

Add one backward-compatible completion delivery policy that defaults to current behavior, supports channel-aware suppression plus coordinated per-spawn semantics, and aligns with the open audience-classification work before updating config, docs, and focused tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection on current main gives high confidence that no supported per-channel or public per-spawn suppression policy exists.

## Solution Assessment

Unclear until maintainers choose the public primitive. The maintainable direction is a single delivery policy threaded through config schema/types, `sessions_spawn`, announce delivery, docs, and tests rather than relying on model output tokens.
