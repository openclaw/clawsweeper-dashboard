## Summary

Keep open. Current main still lacks a structural per-channel or per-spawn suppression/reroute policy for sub-agent completion announces, and the discussion includes current Telegram/iMessage confirmations plus maintainer direction to align this with #69492.

## What This Changes

The issue requests a config or spawn-time control to suppress or reroute sub-agent completion announcements per channel.

## Best Possible Solution

Ship a backward-compatible completion delivery policy that defaults to current behavior, can classify or suppress sub-agent completion announces by channel and/or spawn call, and is implemented with the #69492 audience-classification direction.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the config/tool/docs surfaces expose no suppression or reroute option, while delivery code and tests still preserve external requester delivery and direct fallback sends.

## Solution Assessment

Unclear. The requested direction is valid, but the best fix should likely use #69492's broader audience/delivery classification rather than committing first to a narrow `channels.telegram.subagentAnnounce` flag.
