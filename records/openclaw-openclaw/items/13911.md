## Summary

Keep open: current main still lacks a per-channel or per-spawn policy to suppress sub-agent completion announcements, and related open items cover broader global suppression, parent routing, or plugin hooks rather than replacing this channel-specific product/API decision.

## What This Changes

The issue requests a structural per-channel or per-spawn policy to suppress sub-agent completion announcements from user-facing chat channels.

## Best Possible Solution

Keep this as the per-channel suppression design item, then add a backward-compatible completion delivery policy that defaults to current behavior and applies before requester handoff and direct fallback paths after maintainers choose the public API shape.

## Reproduction Assessment

Yes. Source inspection is enough for the missing capability: current main exposes no per-channel sub-agent announce suppression config or public sessions_spawn parameter, while completion delivery still resolves an external requester target.

## Solution Assessment

Unclear as proposed. A channel key such as channels.telegram.subagentAnnounce is plausible, but the best solution should be a maintainer-approved completion delivery policy that preserves current defaults and covers both agent handoff and direct fallback paths.
