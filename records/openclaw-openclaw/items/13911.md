## Summary

Keep open. Current main and the latest stable release still lack a per-channel sub-agent completion suppression policy, while source and docs still preserve automatic requester/channel delivery paths when no model-level skip token is produced.

## What This Changes

Request per-channel config or policy to suppress or reroute sub-agent completion announcements without disabling normal interactive channel replies.

## Best Possible Solution

Define one backward-compatible completion-audience model with current delivery as the default, explicit per-channel suppressed or internal-only states, clear precedence with global/per-spawn controls, and coverage before all primary and fallback delivery paths.

## Reproduction Assessment

Yes, at source level: current main resolves top-level completion delivery toward external requester routes and exposes no per-channel suppression policy in config, tool schema, docs, or tests. I did not run a live Telegram/iMessage transport reproduction in this read-only review.

## Solution Assessment

Yes in direction, but not as a standalone Telegram-specific boolean. The maintainable solution is a generic completion-audience policy coordinated with global suppression and parent-routing work so OpenClaw does not grow competing delivery controls.
