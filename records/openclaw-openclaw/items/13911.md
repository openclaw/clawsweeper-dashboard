## Summary

Keep open. Current main and the latest release still lack a public per-channel sub-agent announce suppression policy, and the overlapping global suppression, parent-routing, and hook requests do not fully replace this per-channel channel-policy request.

## What This Changes

The issue requests a structural per-channel or equivalent policy to suppress sub-agent completion announcements from channel sessions.

## Best Possible Solution

Define one backward-compatible completion-delivery policy that defaults to current behavior, can suppress or internalize completions for selected channels, and applies before both requester-agent handoff and direct fallback delivery with schema, docs, and focused delivery tests updated together.

## Reproduction Assessment

Yes, at source level: current main has no public per-channel sub-agent announce policy or `sessions_spawn` suppression/routing parameter, while delivery still resolves user-facing targets and direct fallback sends. I did not run a live Telegram/iMessage reproduction because the missing public surface is directly visible in source, docs, and latest-release code.

## Solution Assessment

Unclear. A per-channel suppress knob matches the reported Telegram/iMessage pain, but the best maintainable fix is likely one completion-delivery policy coordinated with the open global suppression and parent-routing requests.
