## Summary

Keep open. Current main and v2026.6.8 have improved the default completion handoff, but they still do not expose a documented per-channel sub-agent announce suppression policy; the remaining work is a public config/API choice that overlaps, but is not replaced by, the global suppression and announce-routing issues.

## What This Changes

Request a structural per-channel policy to suppress or reroute sub-agent completion announcements so channel chats like Telegram or iMessage are not cluttered by automated completion output.

## Best Possible Solution

Maintainers should choose one backward-compatible completion delivery policy that can express channel-specific suppression without competing with the global skip and parent-routing contracts, then wire it through schema, docs, delivery tests, and all announce fallback paths.

## Reproduction Assessment

Yes. Current main and v2026.6.8 expose no documented per-channel sub-agent announce suppression key or `sessions_spawn` parameter, and the strict config schema would reject the proposed shape; I did not need a live Telegram run to prove the missing public contract.

## Solution Assessment

Unclear. Per-channel suppression is a plausible user-facing control, but it overlaps enough with global suppression and announceTarget routing that maintainers should settle the API before a fix PR.
