## Summary

Keep open. Current main and v2026.5.12 still lack a supported per-channel or public per-spawn suppression/reroute policy, while the live discussion and related open issues show this needs maintainer API direction rather than cleanup closure.

## What This Changes

Request a per-channel config or spawn-time policy to suppress or reroute sub-agent completion announcements from chat channels.

## Best Possible Solution

Add one backward-compatible completion delivery policy that defaults to current behavior, supports channel-aware suppression plus coordinated per-spawn semantics, applies before user-facing delivery decisions, and aligns with the open suppression/routing/audience-classification issues.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection on current main and v2026.5.12 gives high confidence that no supported per-channel or public per-spawn suppression policy exists.

## Solution Assessment

Unclear until maintainers choose the public primitive. The maintainable direction is one delivery policy threaded through config schema/types, `sessions_spawn`, announce delivery, docs, and tests rather than relying on model output tokens.
