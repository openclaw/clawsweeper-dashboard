## Summary

Keep open: current main and v2026.6.8 still lack a public per-channel or per-spawn policy to suppress sub-agent completion announcements, while current delivery code and tests preserve top-level external requester delivery.

## What This Changes

Request a framework-level way to suppress or reroute sub-agent completion announcements per channel, especially for Telegram sessions.

## Best Possible Solution

Keep this as the per-channel suppression design item and add a backward-compatible completion delivery policy that defaults to current behavior, can suppress or make completions internal-only for selected channels or spawn calls, and is coordinated with the broader suppression and parent-routing issues.

## Reproduction Assessment

Yes, at source level: current main exposes no per-channel sub-agent announce suppression config or sessions_spawn parameter, and top-level completion delivery still resolves an external requester target. I did not run a live Telegram scenario because the missing surface is explicit in source, docs, and tests.

## Solution Assessment

Unclear as proposed: channels.telegram.subagentAnnounce is plausible, but the best fix should be a maintainer-approved completion delivery policy that preserves current defaults and applies before both requester-agent delivery and direct fallback sends.
