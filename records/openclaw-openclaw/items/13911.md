## Summary

Keep open: current main and the latest release still lack a supported per-channel or per-spawn suppression/reroute policy for sub-agent completion announcements; existing code only has model-token suppression and delivery-routing hardening, so this remains a product/API decision rather than a cleanup close.

## What This Changes

Request a configurable policy to suppress or reroute sub-agent completion announcements per channel, globally, or per spawn.

## Best Possible Solution

Add one backward-compatible completion delivery policy that defaults to current behavior, supports channel-aware suppression plus coordinated per-spawn internal-only/reroute semantics, and updates config schema/types, docs, generated help, and focused delivery tests.

## Reproduction Assessment

No, not as a bug reproduction; this is a feature/config request. Source and latest-release inspection give high confidence that no supported per-channel or public per-spawn suppression policy exists.

## Solution Assessment

Unclear until maintainers choose the public primitive. The maintainable direction is a single backward-compatible delivery policy threaded through config, sessions_spawn, announce delivery, docs, and tests rather than relying on model-output tokens.
