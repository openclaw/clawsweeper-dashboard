## Summary

Keep open. Current main still lacks a supported per-channel or public per-spawn suppression policy for sub-agent completion announcements, and the live discussion plus product-decision labels show this needs maintainer API direction rather than cleanup closure.

## What This Changes

The issue requests a public policy or config surface to suppress sub-agent completion announcements per channel while keeping normal interactive channel replies working.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior, supports channel-aware suppression and coordinated per-spawn semantics, applies before user-facing delivery decisions, and updates config schema/types, docs, and focused delivery tests.

## Reproduction Assessment

No, not as a strict bug reproduction because this is a feature/config request. Source inspection gives high confidence that current main has no supported per-channel or public per-spawn suppression policy.

## Solution Assessment

Unclear until maintainers choose the public primitive. The delivery layer plus agents.defaults.subagents and sessions_spawn are the right implementation surfaces, but the design should coordinate with https://github.com/openclaw/openclaw/issues/8299, https://github.com/openclaw/openclaw/issues/27445, and https://github.com/openclaw/openclaw/issues/69492.
