## Summary

Keep open. Current main has improved the old raw-send fallback behavior, but it still has no structural per-channel or per-spawn policy to suppress sub-agent completion announces, and the live discussion points to unresolved public config/API work.

## What This Changes

Request a structural config or spawn option to suppress sub-agent completion announcements for selected channels while keeping normal interactive channel replies enabled.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior, can suppress or keep completions internal for selected channels and/or spawn calls, applies before external delivery decisions, and updates config schema/types, docs, and focused delivery tests in coordination with https://github.com/openclaw/openclaw/issues/69492.

## Reproduction Assessment

Yes, at source level. Current config/types and `sessions_spawn` expose no suppression or reroute policy, while delivery code and docs still allow top-level completion handoffs to deliver back to the requester chat unless the model emits a suppression token.

## Solution Assessment

Unclear pending maintainer API direction. The delivery layer is the right implementation area, but channel-vs-spawn precedence and the relationship to the broader audience-classification work need a product decision first.
