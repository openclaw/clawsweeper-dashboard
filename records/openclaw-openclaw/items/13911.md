## Summary

Keep open: current main and v2026.6.8 still lack a public per-channel or per-spawn policy to suppress sub-agent completion announcements, and the related open issues overlap rather than replacing this channel-policy request.

## What This Changes

The issue requests a structural per-channel or per-spawn option to suppress sub-agent completion announcements from user-facing channel chats.

## Best Possible Solution

Keep this as the per-channel suppression design item and add a backward-compatible completion delivery policy that defaults to current behavior, can suppress or make completions internal-only for selected channels or spawn calls, and is coordinated with the broader suppression and parent-routing issues.

## Reproduction Assessment

Yes, at source level: current main has no public per-channel sub-agent announce suppression config or sessions_spawn parameter, and top-level completion delivery still resolves an external requester target. I did not run a live Telegram/iMessage scenario because the missing surface is directly visible in schema, docs, tests, and delivery code.

## Solution Assessment

Unclear as proposed: a channels.telegram.subagentAnnounce key is plausible, but the best fix should be a maintainer-approved completion delivery policy that preserves current defaults and applies before both requester-agent delivery and direct fallback sends.
