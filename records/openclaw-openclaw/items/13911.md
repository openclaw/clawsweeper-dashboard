## Summary

Keep open. Current main still lacks a public per-channel or per-spawn suppression/reroute policy for sub-agent completion announces, and the issue discussion includes current Telegram/iMessage confirmations plus a maintainer comment explicitly keeping the work open pending #69492-aligned API direction.

## What This Changes

The issue requests a configurable way to suppress or reroute sub-agent completion announcements per channel, such as Telegram or iMessage, while keeping normal interactive channel replies working.

## Best Possible Solution

Ship a backward-compatible completion delivery policy that defaults to current behavior but can classify sub-agent completion announces as external, internal-only, suppressed, or rerouted by channel and/or spawn call. Apply it before both announce-agent delivery and direct fallback sends, then update config types, generated schema/help, public docs, and focused delivery tests in coordination with #69492.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: config/tool/docs expose no suppression or reroute option, while delivery code and tests still preserve external requester delivery and direct fallback sends.

## Solution Assessment

Unclear. The requested direction is valid, but the best fix should likely use #69492's broader audience/delivery classification rather than committing first to a narrow channels.telegram.subagentAnnounce flag.
