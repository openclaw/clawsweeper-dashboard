## Summary

Keep open. Current main still has no supported per-channel or public per-spawn policy for suppressing sub-agent completion announcements, and live related discussion points to an unresolved public API/config decision rather than a cleanup closure.

## What This Changes

Add a supported per-channel policy to suppress or reroute sub-agent completion announcements away from channel chats such as Telegram.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior, supports per-channel suppression and coordinates with per-spawn parent/none semantics, applies before external delivery decisions, and updates config schema/types, docs, and focused delivery tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source, docs, and regression tests on current main give high-confidence evidence that no supported per-channel or public `sessions_spawn` suppression policy exists while requester-chat delivery remains the default for top-level dormant completions.

## Solution Assessment

Unclear until maintainers choose the API shape. The subagent defaults/tool schema and announce-delivery layer are the right implementation surfaces, but the design should be coordinated with https://github.com/openclaw/openclaw/issues/8299, https://github.com/openclaw/openclaw/issues/64658, and https://github.com/openclaw/openclaw/issues/69492.
