Keep open. Current main and the latest shipped tag still lack a public per-channel or per-spawn policy for suppressing sub-agent completion announces, and the remaining decision is the API/product shape rather than a stale or implemented request.

## What This Changes

The issue requests a deterministic per-channel or per-spawn way to suppress or reroute sub-agent completion announces away from user chat channels.

## Best Possible Solution

Add a backward-compatible subagent completion delivery policy that defaults to current behavior, supports deterministic internal-only or suppressed delivery for selected channels and/or spawn calls, and threads that decision through config types/schema/help, sessions_spawn, registry/lifecycle delivery, docs, and focused tests while coordinating with the related open routing and audience-classification work.
