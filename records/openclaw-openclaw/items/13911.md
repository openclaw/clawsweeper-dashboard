## Summary

Keep open. Current main still has no supported per-channel or public per-spawn policy for suppressing sub-agent completion announcements, and the live discussion keeps this active for maintainer API/product direction rather than cleanup closure.

## What This Changes

Add a channel-aware configuration or broader announce policy that can suppress sub-agent completion announcements for selected channels such as Telegram.

## Best Possible Solution

Add a backward-compatible completion-announcement policy that defaults to current behavior, supports channel-aware suppression and coordinated per-spawn semantics, applies before external delivery decisions, and updates config schema/types, docs, and focused delivery tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source, docs, and tests on current main give high-confidence evidence that no supported per-channel or public per-spawn suppression policy exists.

## Solution Assessment

Unclear until maintainers choose the public primitive. The delivery layer plus agents.defaults.subagents and sessions_spawn are the right implementation surfaces, but the design should be coordinated with https://github.com/openclaw/openclaw/issues/8299, https://github.com/openclaw/openclaw/issues/27445, and https://github.com/openclaw/openclaw/issues/69492.
