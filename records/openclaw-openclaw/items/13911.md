## Summary

Keep open: current main and v2026.6.10 still lack a public per-channel or per-spawn policy to suppress sub-agent completion announcements, and the related open items cover global suppression, parent routing, or a direct-message leak rather than replacing this channel-specific product/API decision.

## What This Changes

The issue requests a per-channel configuration or policy that can suppress or reroute sub-agent completion announcements so channel replies can remain interactive without automated completion noise.

## Best Possible Solution

Add a backward-compatible completion-delivery policy that defaults to current behavior and can suppress or make internal-only selected channel/spawn completions before active wake, requester handoff, and direct fallback paths, after maintainers choose the public API shape.

## Reproduction Assessment

Yes. Source inspection is enough for the missing capability: current main exposes no per-channel sub-agent announce suppression config or public sessions_spawn parameter, while top-level completion delivery still resolves an external requester target.

## Solution Assessment

Unclear as proposed. A channel key such as channels.telegram.subagentAnnounce is plausible, but the best solution should be a maintainer-approved completion delivery policy that composes with global suppression, parent routing, and fallback delivery semantics.
