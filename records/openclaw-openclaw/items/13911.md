## Summary

Keep open: current main and the latest v2026.6.8 release still lack a public per-channel or per-spawn policy to suppress sub-agent completion announcements, and the issue has active maintainer/product-decision labels rather than an implemented or stale cleanup shape.

## What This Changes

Request a framework-level way to suppress or reroute sub-agent completion announcements for selected channels, especially Telegram/iMessage-style chat surfaces.

## Best Possible Solution

Keep this as the per-channel suppression design item and add a backward-compatible completion delivery policy that defaults to current behavior, can suppress or make completions internal-only for selected channels or spawn calls, and is coordinated with the broader global suppression and parent-routing issues.

## Reproduction Assessment

Yes. Source inspection is enough: current main exposes no per-channel sub-agent announce suppression config or public sessions_spawn parameter, and top-level completion delivery still resolves an external requester target.

## Solution Assessment

Unclear as proposed. A channel key such as channels.telegram.subagentAnnounce is plausible, but the best fix should be a maintainer-approved completion delivery policy that preserves defaults and applies before both requester-agent delivery and direct fallback sends.
