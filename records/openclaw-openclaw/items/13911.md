## Summary

Keep open. Current main and the latest v2026.6.6 release still lack a structural per-channel or per-spawn policy for suppressing sub-agent completion announces, and the related open trackers do not fully replace this channel-level product/API decision.

## What This Changes

Request a configurable policy to suppress or reroute sub-agent completion announcements for selected channels so raw child results do not clutter user chats.

## Best Possible Solution

Define one backward-compatible completion delivery policy that defaults to current behavior but can mark selected channel or spawn completions as internal-only or suppressed, then apply it before both requester-agent handoff and direct fallback sends with schema, docs, and focused tests updated together.

## Reproduction Assessment

Yes, at source level. Current main exposes no public `sessions_spawn` announce/silent parameter and no `agents.defaults.subagents` or per-channel suppression policy, while docs still describe requester-chat delivery and exact-token suppression only.

## Solution Assessment

No, not as a one-off Telegram-only key. The maintainable solution is a generic, backward-compatible delivery policy coordinated with the related suppression, parent-routing, and event-audience work so channel defaults, per-spawn overrides, and fallback delivery do not diverge.
