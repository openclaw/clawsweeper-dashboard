## Summary

Keep open. Current main and the latest stable release still lack a documented per-channel sub-agent announce suppression policy, and the remaining work is a public config/API decision rather than cleanup-safe stale work.

## What This Changes

The issue requests a framework-level way to suppress or reroute sub-agent completion announcements per channel, especially for Telegram-style chat sessions.

## Best Possible Solution

Define one backward-compatible completion audience/suppression contract that defaults to current delivery, supports selected channel suppression or internal-only delivery, and has clear precedence with global and per-spawn controls.

## Reproduction Assessment

Yes, at source level: current main exposes no per-channel sub-agent announce suppression in config, tool schema, docs, or delivery gates. I did not run a live Telegram or iMessage transport reproduction in this read-only review.

## Solution Assessment

Unclear pending maintainer direction. The requested direction is valid, but the maintainable fix should be a generic completion-audience policy rather than a standalone Telegram-specific boolean.
