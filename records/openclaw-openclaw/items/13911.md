## Summary

Keep open: current main and v2026.6.8 still lack a public per-channel or per-spawn policy to suppress sub-agent completion announces, while related open issues cover broader global suppression, parent routing, and hook designs rather than replacing this channel-policy request.

## What This Changes

The issue requests a channel-scoped way to suppress sub-agent completion announcements while leaving normal interactive channel replies intact.

## Best Possible Solution

Keep this as the canonical per-channel suppression design item and add a backward-compatible completion delivery policy that defaults to current behavior, can suppress or make completions internal-only for selected channels or spawn calls, and is coordinated with the broader suppression and parent-routing issues.

## Reproduction Assessment

Yes, at source level: current main has no public per-channel sub-agent announce suppression config or `sessions_spawn` parameter, and top-level completion delivery still resolves an external requester target. I did not run a live Telegram/iMessage scenario because the missing surface is directly visible in schema, docs, tests, and delivery code.

## Solution Assessment

Unclear as proposed: a `channels.telegram.subagentAnnounce` key is plausible, but the best fix should be a maintainer-approved completion delivery policy that preserves current defaults and applies before both parent-handoff delivery and direct fallback sends.
