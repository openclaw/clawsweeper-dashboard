## Summary

Keep open: current main and v2026.6.8 still lack a public per-channel or per-spawn policy to suppress sub-agent completion announces, and the remaining work overlaps broader announce-routing/product decisions rather than a finished implementation.

## What This Changes

The issue requests a configurable per-channel policy that suppresses or reroutes sub-agent completion announcements while preserving ordinary interactive channel replies.

## Best Possible Solution

Choose one backward-compatible completion-delivery policy that can express channel suppression and compose with parent-routing work, then apply it before both requester-agent handoff and fallback/direct delivery paths.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the missing capability: current main and v2026.6.8 expose no sessions_spawn parameter or agents.defaults.subagents key for per-channel sub-agent announce suppression.

## Solution Assessment

Unclear until maintainers choose the public contract. A channel-level suppress policy is plausible, but it should be coordinated with the open global suppression and announceTarget routing requests so OpenClaw does not grow parallel delivery controls.
