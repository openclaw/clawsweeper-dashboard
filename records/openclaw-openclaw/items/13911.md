## Summary

Keep open: current main and v2026.6.8 still lack a structural per-channel suppression/routing policy for sub-agent completion announces, and the remaining work needs maintainer API/config direction rather than cleanup closure.

## What This Changes

Requested per-channel or channel-deny control for sub-agent completion announcements so chat channels like Telegram or iMessage can keep normal replies while suppressing raw sub-agent completion noise.

## Best Possible Solution

Choose one backward-compatible completion-delivery policy that covers per-channel suppression and coordinates with the global skip and parent-routing proposals, then apply it before both announce-agent delivery and fallback/direct sends.

## Reproduction Assessment

Yes, at source level: current main has no per-channel sub-agent announce policy in config, no native sessions_spawn announce control, and docs still describe exact-token suppression plus top-level external delivery.

## Solution Assessment

Unclear as a final implementation. Per-channel suppression is a plausible core delivery policy, but maintainers should settle the public contract and its relationship to global skip and parent routing before code changes.
