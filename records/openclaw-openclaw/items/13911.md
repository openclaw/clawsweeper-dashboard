## Summary

Keep open. Current main and v2026.6.6 still lack a public per-channel sub-agent announce suppression policy; related open issues cover global suppression and parent routing, but neither fully replaces this channel-specific request.

## What This Changes

The issue requests a public per-channel policy to suppress or reroute sub-agent completion announcements away from chat channels such as Telegram while keeping normal interactive replies working.

## Best Possible Solution

Define one backward-compatible sub-agent completion delivery policy that covers per-channel suppression, general skip, and parent routing without competing knobs, then document and test the chosen precedence across spawn, config, delivery, and docs surfaces.

## Reproduction Assessment

Yes, at source level. Current main exposes no public per-channel sub-agent announce policy in sessions_spawn, agents.defaults.subagents, or the sub-agent docs, and the delivery path still has no policy gate before channel-capable completion delivery.

## Solution Assessment

Unclear pending maintainer product/API direction. Per-channel suppression would solve the reported Telegram/iMessage noise, but the maintainable fix should be a unified completion-delivery policy that also coordinates with global suppression and parent-routing proposals.
