## Summary

Keep open. Current main still lacks a structural per-channel or per-spawn way to suppress sub-agent completion announcements, and the remaining decision is the public completion-audience/config API rather than a stale or already-implemented code path.

## What This Changes

Request a config or delivery-policy control that can suppress or reroute sub-agent completion announcements for selected channels such as Telegram while leaving ordinary channel replies enabled.

## Best Possible Solution

Define one backward-compatible completion-audience contract with current channel delivery as the default, explicit parent/internal/none behavior, optional per-channel policy, and coordinated schema/types/docs/tests across the existing announce delivery paths.

## Reproduction Assessment

Not applicable as a bug reproduction, but yes for a source-level feature gap. Current main's public spawn schema, strict config surface, docs, and completion-delivery code show there is no deterministic per-channel suppression without relying on model output tokens.

## Solution Assessment

The requested direction is valid, but the exact sketched config key is not yet proven to be the best permanent API. A unified completion-audience contract should cover per-channel suppression while coordinating with deterministic skip and parent-routing work.
