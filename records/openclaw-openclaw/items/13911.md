## Summary

Keep open. Current main still lacks a structural per-channel or per-spawn policy for suppressing sub-agent completion announcements, and the related live work is either broader event-audience design or adjacent routing work rather than an implementation of this request.

## What This Changes

Request a public config/tool policy to suppress or reroute sub-agent completion announcements per channel instead of always delivering them to the requester channel.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior, supports suppressed or internal-only handling for selected channels and/or spawn calls, applies before external delivery decisions, and updates config schema/types, docs, and focused delivery tests.

## Reproduction Assessment

Yes, at source level. Current config/tool schemas and docs expose no supported suppression policy, while the delivery code still resolves external requester delivery for top-level requesters.

## Solution Assessment

Unclear pending maintainer API direction. A delivery-layer policy is the right area, but naming and precedence should coordinate with https://github.com/openclaw/openclaw/issues/8299, https://github.com/openclaw/openclaw/issues/27445, https://github.com/openclaw/openclaw/issues/64658, and https://github.com/openclaw/openclaw/issues/69492.
