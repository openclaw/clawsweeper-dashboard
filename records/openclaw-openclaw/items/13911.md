## Summary

Close as duplicate/superseded: the same remaining framework-level sub-agent announce suppression work is already tracked by the older open canonical issue at https://github.com/openclaw/openclaw/issues/8299, while the broader parent/channel routing shape remains covered by https://github.com/openclaw/openclaw/issues/27445.

## What This Changes

The issue requests a per-channel config policy to suppress or reroute sub-agent completion announcements away from noisy user-facing channels such as Telegram.

## Best Possible Solution

Keep the product/API decision on https://github.com/openclaw/openclaw/issues/8299 for deterministic suppression, coordinate any parent/channel routing semantics with https://github.com/openclaw/openclaw/issues/27445, and avoid a separate per-channel-only implementation thread here.

## Reproduction Assessment

Yes, at source level: current main exposes no public sessions_spawn announce/silent parameter and no agents.defaults.subagents per-channel announce policy, while docs only describe exact sentinel-token suppression. This is a missing feature/config surface rather than a broken documented behavior.

## Solution Assessment

Yes for cleanup: consolidating this narrower per-channel suppression request into the older open suppression tracker is the best repository-maintenance path. The implementation itself still needs maintainer API direction so suppression, parent routing, and channel defaults do not become competing knobs.
