## Summary

Keep open. Current main and v2026.6.6 still lack a structural per-channel sub-agent announce suppression policy, and the open related suppression/routing issues overlap but do not fully replace this per-channel request.

## What This Changes

The issue asks for a config or policy that can suppress or reroute sub-agent completion announces per channel, especially for Telegram-style chat noise.

## Best Possible Solution

Define one backward-compatible completion audience policy that covers channel-level suppression, parent/requester handoff, and direct fallback behavior without adding competing knobs.

## Reproduction Assessment

Yes, at source level: current main and the latest release expose no per-channel sub-agent announce suppression in config, sessions_spawn, or the delivery policy. I did not run a live Telegram/iMessage transport reproduction in this read-only review.

## Solution Assessment

Unclear pending maintainer product direction. A per-channel switch is plausible, but it should be folded into a single completion audience/routing contract with the broader suppression and parent-routing trackers.
