## Summary

Keep open. Current main still lacks a structural per-channel or per-spawn suppression/reroute policy for sub-agent completion announces, and recent maintainer context keeps this tied to #69492 rather than closing it as implemented or stale.

## What This Changes

The issue requests a config or spawn-level policy to suppress or reroute sub-agent completion announcements for selected channels such as Telegram or iMessage.

## Best Possible Solution

Ship a backward-compatible completion delivery policy that defaults to current behavior, can classify or suppress sub-agent completion announces by channel and/or spawn call, and is applied before both announce-agent delivery and direct fallback sends.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: config/tool/docs surfaces expose no suppression or reroute option, while delivery code and tests still preserve external requester delivery and direct fallback sends.

## Solution Assessment

Unclear. The requested direction is valid, but the best maintainable fix should likely align with #69492's broader audience/delivery classification instead of committing first to a narrow Telegram-specific flag.
