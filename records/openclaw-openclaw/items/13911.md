## Summary

Keep open: current main and the latest shipped release still lack a supported per-channel or public per-spawn policy for suppressing or rerouting sub-agent completion announcements, and the related open suppression, routing, and audience-classification threads are adjacent rather than replacements.

## What This Changes

Request a config or tool-level policy to suppress or reroute sub-agent completion announcements per channel, especially for Telegram, so the main agent controls user-facing output.

## Best Possible Solution

Add one backward-compatible completion delivery policy that defaults to current behavior, supports channel-aware suppression plus coordinated per-spawn semantics, and updates config schema/types, docs, generated help, and focused delivery tests.

## Reproduction Assessment

No, not as a bug reproduction; this is a feature/config request. Source and latest-tag inspection give high confidence that no supported per-channel or public per-spawn suppression policy exists.

## Solution Assessment

Unclear until maintainers choose the public primitive. The maintainable direction is a single backward-compatible delivery policy threaded through config, `sessions_spawn`, announce delivery, docs, and tests rather than relying on model output tokens.
