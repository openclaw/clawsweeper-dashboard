## Summary

Keep open: current main and v2026.6.10 still lack a public per-channel or per-spawn policy to suppress sub-agent completion announcements, and the related open reports overlap without replacing this channel-specific product/API decision.

## What This Changes

Request a public per-channel or equivalent policy to suppress or reroute sub-agent completion announcements from user-facing channels such as Telegram.

## Best Possible Solution

Define a backward-compatible completion-delivery policy that defaults to current behavior and can make selected channel or spawn completions suppressed/internal before active wake, requester handoff, and direct text/media fallback paths.

## Reproduction Assessment

Yes for the missing capability. Source inspection shows current main exposes no per-channel sub-agent announce suppression config or public sessions_spawn parameter while top-level completion delivery still resolves an external requester target.

## Solution Assessment

Unclear as proposed. A channel key such as channels.telegram.subagentAnnounce is plausible, but maintainers should choose one completion-delivery policy that composes with global suppression, parent routing, extension hooks, and fallback reliability.
