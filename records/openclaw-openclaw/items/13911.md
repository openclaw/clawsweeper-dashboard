## Summary

Keep open. Current main still has no supported per-channel or public per-spawn suppression/reroute policy for sub-agent completion announcements, and the related open work does not yet implement this delivery-routing primitive.

## What This Changes

Request adds structural per-channel or per-spawn suppression/reroute policy for sub-agent completion announcements instead of relying on model-produced silent tokens.

## Best Possible Solution

Add one backward-compatible completion delivery policy that defaults to current behavior, supports channel-aware suppression plus coordinated per-spawn semantics, and aligns with the open audience-classification work before updating config, docs, and focused tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection on current main gives high confidence that no supported per-channel or public per-spawn suppression policy exists.

## Solution Assessment

Unclear until maintainers choose the public primitive. The maintainable direction is a single backward-compatible delivery policy threaded through config schema/types, `sessions_spawn`, announce delivery, docs, and tests rather than relying on model output tokens.
