## Summary

Keep open. Current main and the latest shipped release still lack a supported per-channel or public per-spawn suppression policy, and the live discussion/labels indicate this needs maintainer API direction rather than cleanup closure.

## What This Changes

The issue requests a supported way to suppress or reroute sub-agent completion announcements per channel so channels like Telegram do not receive raw sub-agent output.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior, supports channel-aware suppression and coordinated per-spawn semantics, applies before user-facing delivery decisions, and aligns with https://github.com/openclaw/openclaw/issues/69492.

## Reproduction Assessment

No, not as a strict bug reproduction because this is a feature/config request. Source inspection on current main and v2026.5.12 gives high confidence that no supported per-channel or public per-spawn suppression policy exists.

## Solution Assessment

Unclear until maintainers choose the public primitive. The delivery layer plus agents.defaults.subagents and sessions_spawn are the likely implementation surfaces, but the design should coordinate with https://github.com/openclaw/openclaw/issues/8299, https://github.com/openclaw/openclaw/issues/27445, and https://github.com/openclaw/openclaw/issues/69492.
