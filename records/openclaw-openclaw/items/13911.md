Keep open. Current main still has no structural per-channel or per-spawn suppression policy for sub-agent completion announcements, and the live discussion plus related open items show this still needs maintainer product/API direction rather than conservative cleanup.

## What This Changes

The issue requests a per-channel or per-spawn config policy to suppress or reroute sub-agent completion announcements away from user chat channels.

## Best Possible Solution

Add a backward-compatible completion delivery/audience policy that defaults to current behavior but can mark selected sub-agent completions as internal-only or suppressed before any user-facing channel delivery.

## Reproduction Assessment

Yes, source-level. Current main's strict sub-agent config schema and sessions_spawn schema lack any suppression/reroute knob, while delivery code and tests still preserve external completion delivery for top-level channel requesters.

## Solution Assessment

Unclear. The request is valid, but the final maintainable solution should be a backward-compatible policy aligned with the broader audience-classification work rather than a one-off Telegram-only switch.
