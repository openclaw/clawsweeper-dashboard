## Summary

The requested structural suppression and parent-only routing behavior is still missing on current main, but the older open canonical issue already tracks the same framework-level config/per-spawn policy and explicitly includes per-channel granularity, so maintaining both issues would split one product/API decision.

## What This Changes

Add a structural completion-delivery policy that can suppress or internalize sub-agent announcements for selected channels or spawn calls.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/8299 as the single canonical tracker and resolve there a generic, backward-compatible internal-only/suppressed completion mode that composes with both primary and fallback delivery paths.

## Reproduction Assessment

Yes, at source level: current main has no suppress/internal-only field in the subagent config or sessions_spawn schema, and documented silence still depends on model output tokens.

## Solution Assessment

No, a separate per-channel tracker is not the best path because the older canonical issue already covers global, per-spawn, per-channel, and parent-only variants of the same product decision.
