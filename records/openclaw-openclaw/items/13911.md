## Summary

Keep open. Current main still has no supported per-channel or per-spawn policy for suppressing sub-agent completion announcements, and the live discussion contains recent maintainer-associated guidance to keep the feature open for API/product direction rather than cleanup closure.

## What This Changes

Request a supported config or spawn-time policy to suppress or reroute sub-agent completion announcements for selected channels such as Telegram.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior, supports channel-aware suppression and coordinated per-spawn semantics, applies before external delivery decisions, and updates config schema/types, docs, and focused delivery tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source, docs, and regression-test inspection on current main give high-confidence evidence that no supported per-channel or public `sessions_spawn` suppression policy exists.

## Solution Assessment

Unclear until maintainers choose the API shape. The delivery layer and subagent config/tool schema are the right implementation surfaces, but the design should be coordinated with https://github.com/openclaw/openclaw/issues/8299, https://github.com/openclaw/openclaw/issues/64658, and https://github.com/openclaw/openclaw/issues/69492.
