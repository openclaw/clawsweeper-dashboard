## Summary

Keep open: current main and the latest v2026.6.8 release still lack a structural per-channel or per-spawn policy to suppress sub-agent completion announcements, and the overlapping open issues do not replace this narrower channel-policy request.

## What This Changes

The issue requests a configurable way to suppress or reroute sub-agent completion announcements per channel so Telegram, iMessage, and similar chats are not cluttered by raw sub-agent output.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior, can mark selected channels or spawn calls as suppressed/internal-only, applies before primary and fallback delivery paths, and updates schema, help, docs, and focused tests.

## Reproduction Assessment

Yes for the missing capability: source inspection on current main shows no supported config key or sessions_spawn parameter, and completion delivery still resolves an external channel target. A live Telegram run is not required to prove the absent public control.

## Solution Assessment

Unclear as a final product contract. A structural delivery policy is better than behavioral NO_REPLY rules, but maintainers should choose whether the permanent shape is per-channel suppression, per-spawn routing, or a broader audience policy before implementation.
