## Summary

Keep open. Current main still has no structural per-channel or per-spawn sub-agent announce suppression policy, and the live thread plus related open items show this remains unresolved public API/config work rather than stale or implemented cleanup.

## What This Changes

The issue requests a per-channel or per-spawn policy to suppress sub-agent completion announcements so channel-bound chats are not cluttered by raw sub-agent output.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior, can suppress or keep completions internal for selected channels and/or spawn calls, applies before external delivery decisions, and updates config schema/types, docs, and focused delivery tests in coordination with the related audience-classification work.

## Reproduction Assessment

Yes, at source level. Current config/types and sessions_spawn schemas expose no suppression or reroute policy, while docs and delivery code still route top-level completion handoffs back to the requester chat unless the model emits ANNOUNCE_SKIP or NO_REPLY.

## Solution Assessment

Unclear pending maintainer API direction. The delivery layer is the right area, but the naming, channel-vs-spawn precedence, and relationship to https://github.com/openclaw/openclaw/issues/8299, https://github.com/openclaw/openclaw/issues/27445, https://github.com/openclaw/openclaw/issues/64658, and https://github.com/openclaw/openclaw/issues/69492 need a product decision first.
