## Summary

Keep open: current main and v2026.5.22 still do not expose a supported per-channel or per-spawn policy for suppressing sub-agent completion announcements, and the latest discussion ties the remaining work to an unresolved delivery/audience API decision rather than a cleanup close.

## What This Changes

The issue requests a config or per-spawn policy to suppress or reroute sub-agent completion announcements for selected chat channels.

## Best Possible Solution

Add one backward-compatible completion delivery policy that defaults to current behavior, supports channel-aware suppression and coordinated internal-only or reroute semantics, and updates config schema/types, generated docs/help, and focused announce-delivery tests.

## Reproduction Assessment

No, not as a bug reproduction; this is a feature/config request. Source and latest-release inspection give high confidence that no supported per-channel or public per-spawn suppression policy exists today.

## Solution Assessment

Unclear until maintainers choose the public primitive. The maintainable direction is a single backward-compatible delivery policy threaded through config, sessions_spawn, announce delivery, docs, and tests rather than relying on model-output tokens.
