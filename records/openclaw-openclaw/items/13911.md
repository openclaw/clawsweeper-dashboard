## Summary

Keep open. Current main still has no supported per-channel or public per-spawn policy for suppressing sub-agent completion announcements, and recent discussion already keeps this feature active for maintainer API/product direction rather than cleanup closure.

## What This Changes

The issue requests a supported per-channel or per-spawn way to suppress sub-agent completion announcements from being posted back into channel chats.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior, supports channel-aware suppression and coordinated per-spawn semantics, applies before external delivery decisions, and updates config schema/types, docs, and focused delivery tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection on current main gives high-confidence evidence that no supported per-channel or public per-spawn suppression policy exists.

## Solution Assessment

Unclear until maintainers choose the public primitive. The delivery layer plus `agents.defaults.subagents` and `sessions_spawn` are the right implementation surfaces, but the design should be coordinated with https://github.com/openclaw/openclaw/issues/8299, https://github.com/openclaw/openclaw/issues/27445, and https://github.com/openclaw/openclaw/issues/69492.
