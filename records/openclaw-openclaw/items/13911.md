## Summary

Keep open. Current main and v2026.6.6 still lack a public per-channel or per-spawn policy for suppressing sub-agent completion announces, and the related open issues cover global suppression, parent routing, or extension hooks rather than replacing this channel-specific request.

## What This Changes

Request per-channel configuration or policy to suppress or reroute sub-agent completion announcements from chat channels such as Telegram and iMessage.

## Best Possible Solution

Define one backward-compatible completion-delivery policy that defaults to current behavior, can express channel-specific suppression or internal-only delivery, and applies before every announce delivery and fallback path.

## Reproduction Assessment

Yes, at source level. Current main exposes no per-channel sub-agent announce suppression in config or `sessions_spawn`, while docs still rely on exact-token suppression and top-level external delivery.

## Solution Assessment

Unclear until maintainers choose the public contract. A per-channel knob is plausible, but it should be coordinated with global suppression and parent-routing work instead of adding a competing policy surface.
