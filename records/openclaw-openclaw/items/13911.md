## Summary

Keep open. Current main and the latest v2026.6.6 release still do not expose a public per-channel sub-agent announce suppression policy, and the closely related open suppression/routing issues overlap but do not fully replace this per-channel request.

## What This Changes

Request a public per-channel/config policy to suppress or reroute sub-agent completion announcements away from chat channels such as Telegram.

## Best Possible Solution

Define one backward-compatible completion-delivery policy that covers per-channel suppression alongside the broader skip/routing work, defaults to current behavior, gates all user-facing completion delivery paths, and updates config/schema/help/docs plus focused announce-delivery tests.

## Reproduction Assessment

Yes, at source level. Current main exposes no public per-channel sub-agent announce policy or `sessions_spawn` suppression/routing parameter, while delivery still resolves user-facing targets and fallback paths.

## Solution Assessment

Unclear. A per-channel suppress knob would address the reported Telegram/iMessage noise, but the maintainable fix is likely one completion-delivery policy coordinated with the open global suppression and parent-routing requests.
