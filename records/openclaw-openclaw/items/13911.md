## Summary

Keep open. Current main still has no supported per-channel or per-spawn policy for suppressing sub-agent completion announcements, and recent discussion plus related open work show the remaining decision is public API/config design rather than cleanup closure.

## What This Changes

Request a config or tool policy to suppress or reroute sub-agent completion announcements per channel without relying on model-emitted silence tokens.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior, supports per-channel suppression and coordinates with per-spawn parent/none semantics, applies before external delivery decisions, and updates config schema/types, docs, and focused delivery tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source and docs inspection on current main gives high-confidence evidence that no supported per-channel or public `sessions_spawn` suppression policy exists while current tests preserve external requester delivery.

## Solution Assessment

Unclear until maintainers choose the API shape. The delivery layer and subagent config/tool schema are the right implementation surfaces, but the final design should be coordinated with https://github.com/openclaw/openclaw/issues/8299, https://github.com/openclaw/openclaw/issues/64658, and https://github.com/openclaw/openclaw/issues/69492.
