## Summary

Keep open. Current main still lacks a structural per-channel or per-spawn policy for suppressing native sub-agent completion announcements, and the related open issues need maintainer API direction rather than cleanup closure.

## What This Changes

Requests a configurable way to suppress or reroute sub-agent completion announcements per channel while keeping normal interactive channel replies working.

## Best Possible Solution

Add a backward-compatible native completion delivery policy that defaults to current requester-chat behavior, supports channel and/or spawn-level suppressed or internal-only handling, and updates config schema/types, generated docs, public docs, and focused delivery tests.

## Reproduction Assessment

Yes, at source level for the missing capability. Current schema/types/docs and the `sessions_spawn` tool schema expose no supported suppression policy, while top-level completion delivery still resolves external requester delivery.

## Solution Assessment

Unclear pending maintainer API direction. The delivery-layer control is the right area, but the final shape should coordinate with https://github.com/openclaw/openclaw/issues/8299, https://github.com/openclaw/openclaw/issues/64658, https://github.com/openclaw/openclaw/issues/27445, and https://github.com/openclaw/openclaw/issues/69492.
