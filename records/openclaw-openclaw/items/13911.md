## Summary

Keep open. Current main still lacks a supported per-channel or per-spawn policy for suppressing sub-agent completion announcements, and the live related work is complementary API/design discussion rather than an implementation of this request.

## What This Changes

Add config or spawn-time control to suppress or reroute sub-agent completion announcements for selected channels such as Telegram or iMessage.

## Best Possible Solution

Define a backward-compatible completion delivery policy that defaults to current behavior, supports selected-channel suppression and/or internal-only delivery, and updates config schema/types, tool schema, docs, and focused announce-delivery tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request, but the absence is source-reproducible: current config/tool schemas expose no supported suppression policy while delivery still resolves external requester routing for top-level sessions.

## Solution Assessment

Unclear pending maintainer API direction. A delivery-layer policy is the right area, but maintainers should choose naming and precedence across per-channel suppression, per-spawn routing, and the related event-audience design before implementation.
