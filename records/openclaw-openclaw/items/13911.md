## Summary

Keep open. Current main still lacks a structural per-channel or per-spawn policy for suppressing sub-agent completion announcements, and the live related issues show this needs maintainer API direction rather than cleanup closure.

## What This Changes

The issue requests a native per-channel or per-spawn policy to suppress sub-agent completion announcements from channel-bound sessions while keeping normal interactive replies working.

## Best Possible Solution

Add a backward-compatible native completion delivery policy that defaults to current behavior, supports suppressed or internal-only handling for selected channels and/or spawn calls, applies before external delivery decisions, and updates config schema/types, docs, and focused delivery tests.

## Reproduction Assessment

Yes, at source level for the missing capability. Current schema/types/docs and the `sessions_spawn` tool schema expose no supported suppression policy, while top-level completion delivery still resolves external requester delivery.

## Solution Assessment

Unclear pending maintainer API direction. A delivery-layer policy is the right area, but the final shape should coordinate with https://github.com/openclaw/openclaw/issues/8299, https://github.com/openclaw/openclaw/issues/64658, https://github.com/openclaw/openclaw/issues/27445, and https://github.com/openclaw/openclaw/issues/69492.
