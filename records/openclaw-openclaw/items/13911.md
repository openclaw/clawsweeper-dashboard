## Summary

Keep open. Current main still lacks a supported per-channel or per-spawn sub-agent completion announce suppression policy; source, docs, tests, and the maintainer discussion all show requester-chat delivery remains the default and the public API shape is unsettled.

## What This Changes

Request a supported config/API policy to suppress or reroute sub-agent completion announces per channel while preserving normal interactive replies.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current requester-chat delivery, supports selected channel and/or spawn-level internal-only or suppressed completion handling, and updates config schema/types/help, docs, and focused announce-delivery tests in coordination with the related routing/audience trackers.

## Reproduction Assessment

Yes, for the current behavior and missing capability. Source/docs/tests on current main show no supported suppression key or public `sessions_spawn` announce policy, while top-level completion delivery still resolves an external requester target with `deliver: true`.

## Solution Assessment

Unclear pending maintainer API direction. The request targets the right delivery layer, but the exact primitive should be chosen alongside https://github.com/openclaw/openclaw/issues/8299, https://github.com/openclaw/openclaw/issues/64658, https://github.com/openclaw/openclaw/issues/27445, and https://github.com/openclaw/openclaw/issues/69492.
