## Summary

Keep open. Current main and v2026.5.12 still lack a supported per-channel or public per-spawn suppression/reroute policy, and the live discussion/labels show this needs maintainer API direction rather than cleanup closure.

## What This Changes

The issue requests a supported config or spawn-level way to suppress or reroute sub-agent completion announcements per channel.

## Best Possible Solution

Add a backward-compatible completion delivery policy that defaults to current behavior, supports channel-aware suppression plus coordinated per-spawn semantics, applies before user-facing delivery decisions, and aligns with https://github.com/openclaw/openclaw/issues/8299, https://github.com/openclaw/openclaw/issues/27445, and https://github.com/openclaw/openclaw/issues/69492.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection on current main and v2026.5.12 gives high confidence that no supported per-channel or public per-spawn suppression policy exists.

## Solution Assessment

Unclear until maintainers choose the public primitive. The maintainable direction is one delivery policy threaded through config schema/types, `sessions_spawn`, announce delivery, docs, and tests rather than relying on model output tokens.
