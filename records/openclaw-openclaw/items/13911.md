## Summary

Keep open. Current main and v2026.6.6 still lack a structural per-channel or per-spawn policy for suppressing sub-agent completion announces, and the overlapping open suppression/routing trackers do not fully replace this channel-specific product/API decision.

## What This Changes

Request to add deterministic per-channel suppression or routing policy for sub-agent completion announcements so channels such as Telegram can avoid raw automated completion posts.

## Best Possible Solution

Define one backward-compatible completion delivery policy that can mark selected channel or spawn completions as internal-only or suppressed, then apply it before both requester-agent handoff and direct fallback sends with schema, docs, and focused delivery tests updated.

## Reproduction Assessment

Yes. Source inspection is a high-confidence reproduction path: current main exposes no public `sessions_spawn` announce/silent parameter and no `agents.defaults.subagents` or per-channel suppression policy, while docs still document only exact-token suppression.

## Solution Assessment

No, not as a one-off Telegram-only key. The maintainable solution is a generic backward-compatible delivery policy coordinated with the open suppression and parent-routing trackers so channel defaults, per-spawn overrides, and fallback delivery do not diverge.
