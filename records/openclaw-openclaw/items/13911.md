## Summary

Keep open. Current main still lacks a structural per-channel or per-spawn suppression/reroute policy for sub-agent completion announces, and the discussion points to #69492 before choosing the public API shape.

## What This Changes

Request a config or sessions_spawn policy to suppress or reroute sub-agent completion announcements for selected channels while keeping normal interactive channel replies working.

## Best Possible Solution

Ship a backward-compatible completion delivery policy that defaults to current behavior, supports internal-only or suppressed sub-agent completion delivery by channel and/or spawn call, and applies before both announce-agent delivery and direct fallback sends.

## Reproduction Assessment

Yes. Source inspection on current main shows the config schema, public type, generated schema, sessions_spawn tool surface, delivery code, tests, and docs still have no structural suppression or reroute policy while preserving external completion delivery.

## Solution Assessment

Unclear. The requested direction is valid, but the maintainable fix should likely align with #69492's audience/delivery classification instead of committing first to a Telegram-only or channel-only flag.
