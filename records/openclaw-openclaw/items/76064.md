## Summary

Keep this PR open: the opt-in RRF mode is useful, but the current patch feeds raw RRF scores into existing memory-search thresholds, so enabling it can filter out otherwise valid hybrid results; public config docs and generated artifacts also need completion.

## What This Changes

Adds an opt-in `weighted`/`rrf` fusion setting for hybrid memory search, wiring memory-core merge logic, agent config/schema surfaces, docs, and unit tests.

## Best Possible Solution

Keep the opt-in config shape, but normalize RRF scores to the existing 0-1 score contract or make thresholding fusion-aware, then add manager-level coverage and update the memory config docs, generated baseline, and changelog.

## Reproduction Assessment

Yes. A static reproduction is enough: with `fusion: "rrf"`, default weights, and `k=60`, a top hit in both ranked lists scores `1/61` (~0.016), below the manager's default 0.35 strict threshold and 0.3 keyword fallback floor.

## Solution Assessment

No. Rank-based fusion is a reasonable feature direction, but using raw RRF as the existing filtered score is not maintainable unless the score is normalized or the threshold path is made fusion-aware.
