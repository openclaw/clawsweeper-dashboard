## Summary

Keep open: the feature shape is useful, but the patch makes opt-in RRF scores too small for the existing `minScore` gate, so normal hybrid searches can return no results; it also leaves required user-facing config docs/changelog artifacts incomplete.

## What This Changes

Adds an opt-in `weighted`/`rrf` hybrid memory-search fusion mode, wiring config/schema/docs and unit coverage for bundled memory-core merging.

## Best Possible Solution

Keep the opt-in config shape, but make RRF scores compatible with current threshold semantics or make thresholding fusion-aware, add manager-level regression coverage, then update canonical docs, generated config baseline, and changelog.

## Reproduction Assessment

Yes. A static reproduction follows from the formula and current defaults: enable `fusion: "rrf"` with normal `minScore`, return one top vector+keyword hit, and the manager-level search path filters it out because ~0.016 is below 0.35 and 0.3.

## Solution Assessment

No. Raw RRF ranking is reasonable for ordering, but using that raw value as the existing filtered `score` is not the narrowest maintainable solution unless the score is normalized or the threshold path is made fusion-aware.
