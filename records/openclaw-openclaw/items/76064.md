## Summary

Keep open. The feature direction is useful, but the PR's raw RRF scores are incompatible with the existing memory-manager score filters, and the new user-facing config key is not fully represented in the canonical docs/generated surfaces.

## What This Changes

Adds an opt-in `weighted`/`rrf` fusion mode for hybrid memory search across memory-core merging, agent config/schema surfaces, docs, and helper tests.

## Best Possible Solution

Keep the opt-in config shape, but normalize RRF scores to the existing score contract or make thresholding fusion-aware, then add manager-level coverage and complete docs/changelog/generated config updates.

## Reproduction Assessment

Yes. A static reproduction is enough: with `fusion: "rrf"`, default weights, and `k=60`, the best possible shared top hit scores about 0.016, below the current 0.35 strict threshold and 0.3 relaxed keyword floor.

## Solution Assessment

No. Rank-based fusion is reasonable, but raw reciprocal-rank scores should not feed the existing filtered `score` contract unless they are normalized or the filter path understands the fusion mode.
