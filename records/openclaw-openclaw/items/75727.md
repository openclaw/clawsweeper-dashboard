Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve captions when replacing media replies:** `src/gateway/server-methods/chat.ts:184-185`
  - body: `stripVisibleTextFromMediaSupplement` clears `payload.text` before replacement content is built. A reply like `Here is the image.\nMEDIA:...` can be rewritten as `Image reply` or image-only instead of preserving the directive-stripped caption, so users lose the assistant's visible text when the raw MEDIA turn is replaced.
  - confidence: 0.92
- **[P2] Limit replacement to the current raw MEDIA turn:** `src/gateway/server-methods/chat.ts:1567-1569`
  - body: The replacement helper searches backward for any assistant MEDIA message matching the payload URL. If an older active-branch message has the same media URL and the current raw turn is absent or already normalized, this rewrites history and skips appending the current media reply, misrouting the response.
  - confidence: 0.86
- **[P2] Avoid double transcript updates after rewrite:** `src/gateway/server-methods/chat.ts:2785-2791`
  - body: `rewriteTranscriptEntriesInSessionFile` already emits a file-level transcript update when it rewrites the branch. Emitting this extra message update afterward makes SSE clients refresh history and then append the same replacement message, so live WebChat can still see a duplicate assistant reply.
  - confidence: 0.9
