## Summary

Keep open: the PR targets a real current-main WebChat/Codex media gap, but current head still has three source-level blockers in the transcript replacement path. The related merged duplicate-media work at https://github.com/openclaw/openclaw/pull/75481 and Codex media work at https://github.com/openclaw/openclaw/pull/77889 and https://github.com/openclaw/openclaw/pull/80166 are useful context, but they do not make this PR obsolete.

## What This Changes

The PR changes gateway and agent media handling so Codex local `MEDIA:` replies and structured local-media tool results normalize into one WebChat media message.

## Best Possible Solution

Land a narrowed gateway fix that replaces only the current raw Codex `MEDIA:` transcript turn, preserves directive-stripped visible captions, and emits one coherent transcript update without duplicating history streams.

## Reproduction Assessment

Yes. Source inspection on current main shows the agent-run WebChat media supplement append path but no raw Codex `MEDIA:` transcript replacement seam; the PR also adds a focused regression that constructs that raw assistant transcript plus media payload path.

## Solution Assessment

No as written. The replacement seam is the right place to solve this, but the current implementation can drop visible captions, rewrite an older matching assistant message, and duplicate unpaginated `chat.history` SSE output after a rewrite refresh.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve media captions during transcript replacement:** `src/gateway/server-methods/chat.ts:170-171`
  - body: `stripVisibleTextFromMediaSupplement` clears `payload.text` before both assistant display content and `mediaMessage` are built. A raw reply such as `Here is the image.\nMEDIA:...` can therefore be replaced with `Image reply` or image-only content instead of the directive-stripped caption; keep visible text for non-TTS media replacements while removing only the `MEDIA:` directive.
  - confidence: 0.9
- **[P2] Match only the latest raw MEDIA assistant turn:** `src/gateway/server-methods/chat.ts:1497-1499`
  - body: The helper scans the entire transcript branch in reverse and rewrites the first assistant message with an equivalent `MEDIA:` URL. If an older turn reused the same media URL while the current reply did not persist a raw directive, this rewrites history and suppresses the new media append; restrict matching to the current/latest assistant entry or pass the expected entry id.
  - confidence: 0.88
- **[P2] Avoid appending after the rewrite refresh:** `src/gateway/server-methods/chat.ts:2644-2650`
  - body: A successful rewrite already emits a transcript refresh, and unpaginated `chat.history` SSE refreshes its state from disk on that update. Emitting an inline `message` update immediately afterward appends the same replacement to the refreshed history, so use a replace/deduped event or rely on the refresh for this path.
  - confidence: 0.87
