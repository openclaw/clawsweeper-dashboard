## Summary

Keep this PR open, but do not merge it as-is. Current main still has the media transcript append path this PR is trying to improve, but the proposed replacement logic can rewrite an unrelated latest assistant MEDIA turn and drops the existing assistant-media transcript idempotency guard; the user-facing fix also needs the required changelog entry.

## What This Changes

The PR changes agent tool media filtering to trust non-external structured local media markers and changes WebChat transcript persistence to replace a raw assistant MEDIA directive turn with normalized media content.

## Best Possible Solution

Land a corrected version that only replaces the raw transcript turn when normalized media references match the current reply, carries the same assistant-media idempotency marker as the append path, preserves the external-tool and local-root media trust boundaries, and adds the required changelog plus focused regression coverage.

## Reproduction Assessment

Yes. The PR body gives a concrete Codex/WebChat reproduction, and the added regression test models the failing current-main shape: a raw assistant `MEDIA:<path>` transcript turn plus a normalized media payload should result in one image-bearing assistant message, not duplicate text-plus-image turns.

## Solution Assessment

No, not as currently written. The direction is appropriate, but the implementation should normalize and compare media references instead of replacing any latest assistant MEDIA turn, and it should preserve the existing transcript idempotency key on the replacement message.
