## Summary

Keep open: current main still lacks the Codex raw `MEDIA:` transcript replacement, but this PR cannot merge as-is because its structured `trustedLocalMedia` path bypasses the current exact raw-tool-name trust boundary.

## What This Changes

The PR adds WebChat transcript replacement for raw Codex `MEDIA:` replies, broadens Pi tool-result local media trust for structured non-external markers, updates tests, adjusts slash-command browser-import typing, and adds a changelog entry.

## Best Possible Solution

Land a rebased version that keeps the current raw-tool exact-name hardening while adding an explicit owned/provenance-safe path for legitimate workspace media markers and preserving the Codex transcript replacement test.

## Reproduction Assessment

Yes. A source-level reproduction is a Codex/AppServer assistant transcript whose latest assistant message contains `MEDIA:<local path>` followed by an equivalent normalized media-bearing payload; current main appends a separate assistant-media message. The trust regression is also reproducible by passing `Web_Search` plus a structured `trustedLocalMedia` local path through `filterToolResultMediaUrls`.

## Solution Assessment

No. The transcript replacement seam is narrow, but the structured local-media trust change skips the exact-name gate that current main added to prevent normalized/case-variant trusted-tool collisions; a safer fix needs explicit owned-tool provenance before local paths pass through.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep structured media behind the exact-name gate:** `src/agents/pi-embedded-subscribe.tools.ts:327-339`
  - body: This condition lets any non-external result carrying `details.media.trustedLocalMedia === true` skip the `builtinToolNames` exact-name check. Current main has a guard for the case where `Web_Search` plus that marker must still drop local paths; with this branch it would return the local path instead. Please keep structured local-media trust tied to an exact registered/owned tool or another provenance signal that untrusted client/tool results cannot supply.
  - confidence: 0.86
