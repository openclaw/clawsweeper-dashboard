## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve visible text when replacing MEDIA replies:** `src/gateway/server-methods/chat.ts:170-171`
  - body: `stripVisibleTextFromMediaSupplement` clears `payload.text` for every media payload before replacement content is built. A reply like `Here is the image.\nMEDIA:...` can therefore become image-only or synthetic-label content instead of preserving the assistant caption; keep non-TTS text or merge the cleaned directive text into the replacement.
  - confidence: 0.9
- **[P2] Only replace the current MEDIA reply:** `src/gateway/server-methods/chat.ts:1497-1499`
  - body: This scans the whole transcript branch for any assistant `MEDIA:` directive matching the payload. If an older turn reused the same local path while the current turn did not persist a raw directive, this rewrites history and suppresses the new media append; constrain replacement to the current/latest assistant turn being normalized.
  - confidence: 0.88
- **[P2] Emit one transcript update for replacements:** `src/gateway/server-methods/chat.ts:2644-2650`
  - body: `rewriteTranscriptEntriesInSessionFile` already emits a refresh update after persisting the rewritten branch. Emitting an inline `message` update immediately afterward makes unbounded history streams refresh to the replacement and then append the same replacement again.
  - confidence: 0.91
