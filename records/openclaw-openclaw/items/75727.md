## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve captions when replacing media replies:** `src/gateway/server-methods/chat.ts:170-171`
  - body: `stripVisibleTextFromMediaSupplement` clears `payload.text` for every media-bearing payload before `assistantContent` and `mediaMessage` are built. A reply such as `Here is the image.\nMEDIA:...` can be rewritten as synthetic `Image reply` text instead of preserving the assistant caption; keep directive-stripped visible text for non-TTS media replacements.
  - confidence: 0.91
- **[P2] Replace only the current raw MEDIA turn:** `src/gateway/server-methods/chat.ts:1497-1499`
  - body: The helper searches the whole active transcript branch for any assistant `MEDIA:` directive matching the payload. If an older turn reused the same media URL while the current reply did not persist a raw directive, this rewrites history and suppresses the new media append; restrict replacement to the latest/current assistant turn or pass the expected entry id.
  - confidence: 0.88
- **[P2] Avoid emitting a second transcript update after rewrite:** `src/gateway/server-methods/chat.ts:2644-2650`
  - body: `rewriteTranscriptEntriesInSessionFile` already emits a transcript refresh after persisting a changed rewrite. Emitting an inline `message` update immediately afterward makes no-cursor SSE clients append the same replacement as a new message after refreshing history, so live clients can see the duplicate this PR is trying to remove.
  - confidence: 0.9
