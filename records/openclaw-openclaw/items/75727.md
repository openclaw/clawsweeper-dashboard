## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve captions when replacing media replies:** `src/gateway/server-methods/chat.ts:170-171`
  - body: `stripVisibleTextFromMediaSupplement` clears `payload.text` for every media-bearing payload before replacement content is built. A reply such as `Here is the image.\nMEDIA:...` can be rewritten as `Image reply` or image-only instead of preserving the assistant caption, so keep the directive-stripped visible text for non-TTS image replacements.
  - confidence: 0.91
- **[P2] Replace only the current raw MEDIA turn:** `src/gateway/server-methods/chat.ts:1497-1499`
  - body: The replacement helper searches the entire transcript branch for any matching assistant `MEDIA:` directive. If an older turn reused the same media URL while the current reply did not persist a raw directive, this rewrites history and suppresses the new media append; restrict replacement to the current/latest assistant turn or pass the expected entry id.
  - confidence: 0.88
- **[P2] Avoid emitting a second update after rewrite:** `src/gateway/server-methods/chat.ts:2644-2650`
  - body: `rewriteTranscriptEntriesInSessionFile` already emits a transcript refresh after persisting a changed rewrite. Emitting an inline `message` update immediately afterward can make live history clients refresh to the rewritten branch and then append the same replacement again.
  - confidence: 0.9
