## Summary

Keep open: the PR is aimed at a real Gateway/WebChat media transcript bug, but the latest head still introduces blocking transcript behavior regressions and the supplied real behavior proof is stale for the current head.

## What This Changes

The PR changes Gateway/WebChat media transcript handling so Codex local `MEDIA:` replies become a single normalized inline media message, with related tool media trust and replay sanitation tests.

## Best Possible Solution

Keep the raw-directive replacement approach, but preserve the model-authored visible text, replace only the current raw assistant turn, and send a single transcript update path with refreshed current-head proof.

## Reproduction Assessment

Yes, source-level. Current main can append an assistant-media supplement without replacing a raw Codex `MEDIA:` transcript turn, and the PR adds focused tests for that seam.

## Solution Assessment

No as written. Replacing the raw directive turn is the right direction, but this patch still needs to preserve text, bound replacement to the current turn, and avoid duplicate transcript update events.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve text when replacing MEDIA replies:** `src/gateway/server-methods/chat.ts:170-171`
  - body: `stripVisibleTextFromMediaSupplement` clears `payload.text` for every media reply before the replacement content is built. When replacing a raw `Here is the image.\nMEDIA:<path>` assistant turn, that drops the model-authored text and replaces it with the synthetic `Image reply`, so WebChat loses visible assistant content while removing the directive.
  - confidence: 0.91
- **[P2] Avoid double-emitting replacement updates:** `src/gateway/server-methods/chat.ts:2642-2648`
  - body: `rewriteTranscriptEntriesInSessionFile` already emits a transcript update after persisting the rewritten branch. Emitting `replaced.message` again here makes history streams refresh to the rewritten transcript and then append the same assistant message inline, which can duplicate the replacement in live WebChat views.
  - confidence: 0.88
- **[P2] Only replace the current MEDIA reply:** `src/gateway/server-methods/chat.ts:1497-1499`
  - body: This searches the whole active transcript branch for any matching assistant `MEDIA:` directive. If an older assistant turn reused the same output path and the current turn did not persist a raw directive, the helper rewrites the old message and then suppresses the current media append, altering history and dropping the new inline image.
  - confidence: 0.86
