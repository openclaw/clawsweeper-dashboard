## Summary

Keep this PR open: the bug fix is plausible and the structured media guard mostly fits the existing boundary, but the latest patch still has two blocking transcript-update correctness issues and the real behavior proof is stale for the current head.

## What This Changes

The PR changes Gateway/WebChat media transcript handling and tool-result media trust so Codex `MEDIA:` image replies are normalized into a single inline assistant image with tests and a changelog entry.

## Best Possible Solution

Constrain replacement to the current raw assistant entry, keep transcript updates on a single live emit path, and refresh proof for the current head before merge.

## Reproduction Assessment

Yes, source-level. Current main can append a media-bearing assistant supplement after Pi has already persisted a raw Codex `MEDIA:` assistant turn, and the PR adds a direct regression for that transcript/media seam.

## Solution Assessment

No as written. Replacing equivalent raw image directive turns is the right direction, but the patch needs bounded replacement and a single transcript-update path before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid double-emitting appended media updates:** `src/gateway/server-methods/chat.ts:2677-2684`
  - body: The append fallback already emits a transcript update through `appendInjectedAssistantMessageToTranscript`. Emitting again here sends two `session.message` events for the same assistant-media message whenever the transcript file resolves to the session key, so WebChat can show the duplicate this PR is trying to prevent. Carry `sessionKey` through the single append emit path, or only emit separately for rewrites.
  - confidence: 0.86
- **[P2] Bound media replacement to the current assistant turn:** `src/gateway/server-methods/chat.ts:1507-1509`
  - body: The helper still scans the whole active transcript branch for any assistant `MEDIA:` entry matching the payload URL, then returns without appending a supplement. If a generated tool reuses a stable output path and the current turn does not persist a raw directive, this can rewrite an older reply and drop the current inline image message. Restrict replacement to the current/latest assistant entry, or append when the match is not the current reply.
  - confidence: 0.83
