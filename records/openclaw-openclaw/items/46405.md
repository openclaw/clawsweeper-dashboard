## Summary

Keep open: current main appears to cover the gateway restart problem through a newer config-write flow, but it does not contain the PR's MiniMax XML-to-toolUse promotion, and the proposed parser has correctness bugs that would still drop or corrupt valid XML tool calls.

## What This Changes

The PR adds MiniMax XML tool-call parsing/promotion before assistant text extraction and changes `config.patch` restart scheduling and guidance so hot-reloadable config writes avoid direct SIGUSR1 restarts.

## Best Possible Solution

Rebase or replace this with a smaller current-main PR that adds robust MiniMax XML-to-`toolUse` promotion with entity decoding and attribute-order-safe parsing, while leaving the newer gateway restart flow unchanged unless a focused current-main regression test proves it still fails.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction for the PR findings: valid XML such as `<invoke id="x" name="Bash">` will not match the PR regex, and XML entities in parameters are assigned without decoding. I did not execute tests because this was a read-only review.

## Solution Assessment

No. The overall direction is useful, but the parser needs to handle valid XML attribute ordering and entity decoding, and the gateway portion needs to be reconciled with current main's newer config-write restart path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Parse XML attributes in any order:** `src/agents/pi-embedded-utils.ts:43-49`
  - body: `parseMinimaxToolCallXml` only recognizes `<invoke>` and `<parameter>` when `name` is the first attribute. Valid XML like `<invoke id="x" name="Bash">...` is skipped, so no `toolUse` block is produced and the existing sanitization path can remove the XML text, silently losing the tool call.
  - confidence: 0.91
- **[P2] Decode XML entities before passing tool inputs:** `src/agents/pi-embedded-utils.ts:49-51`
  - body: Parameter bodies are copied directly into `input`, so valid XML encodings such as `&amp;`, `&lt;`, `&quot;`, and `&apos;` are delivered to tools literally. That corrupts commands or file contents whenever MiniMax emits well-formed XML with escaped characters.
  - confidence: 0.88
