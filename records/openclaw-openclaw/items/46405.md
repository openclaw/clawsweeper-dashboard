## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Gate XML promotion to MiniMax outputs:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:272`
  - body: This calls the MiniMax XML promoter for every assistant message. Any provider that emits or quotes `<minimax:tool_call><invoke ...>` text could have plain text converted into an executable `toolUse` block, expanding tool execution beyond the MiniMax compatibility case. Please gate promotion on MiniMax provider/model/API or move it behind a provider-owned hook.
  - confidence: 0.87
- **[P2] Parse invoke attributes in any order:** `src/agents/pi-embedded-utils.ts:39`
  - body: The parser only matches `<invoke>` when `name` is the first attribute, so valid XML such as `<invoke id="x" name="Bash">` is ignored and the intended tool call is dropped. The same ordering problem applies to `<parameter>` attributes; parse attributes independently of order before promoting the tool call.
  - confidence: 0.9
- **[P2] Decode XML entities in parameter values:** `src/agents/pi-embedded-utils.ts:48-51`
  - body: Parameter text is copied directly into the tool input, so valid XML like `&amp;`, `&lt;`, or `&quot;` reaches tools encoded instead of as the intended characters. Decode standard XML entities after extraction and update the test that currently preserves `&amp;`.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `src/agents/pi-embedded-utils.ts:33`
  - body: This PR changes user-visible agent/tool execution and gateway restart behavior, but `CHANGELOG.md` is not touched. Add a concise fix entry for the MiniMax behavior, and omit the gateway entry if the replacement drops the now-superseded gateway changes.
  - confidence: 0.8
