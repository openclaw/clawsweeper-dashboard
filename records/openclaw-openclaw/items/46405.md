## Summary

Keep open. The PR targets real MiniMax and gateway issues, but current main still strips MiniMax XML instead of promoting toolUse blocks, while the proposed parser has correctness bugs and the gateway half is stale against the newer config write-flow plus active #43803 follow-up.

## What This Changes

The PR adds MiniMax XML tool-call parsing/promotion before assistant text extraction, changes config.patch restart scheduling and gateway wording, and adds agent/gateway regression tests.

## Best Possible Solution

Split/rebase against current main: land a focused MiniMax XML promotion with order-independent attribute parsing and standard XML entity decoding, and resolve config.patch reload behavior through the current config write-flow and #43803.

## Reproduction Assessment

Yes. The parser defects reproduce with direct valid XML inputs, and current-main source/tests show MiniMax XML is still sanitized rather than promoted; this read-only review did not execute tests.

## Solution Assessment

No. The PR identifies real bugs, but the combined stale branch is not the narrowest maintainable fix; a current-main split with robust XML parsing and the existing config write-flow is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Accept XML attributes in any order:** `src/agents/pi-embedded-utils.ts:41`
  - body: `parseMinimaxToolCallXml` only recognizes `<invoke>` when `name` is the first attribute. Valid XML such as `<invoke id="x" name="Bash">` is skipped, after which no `toolUse` block is produced and the sanitizer strips the intended call.
  - confidence: 0.9
- **[P2] Decode XML entities before building tool input:** `src/agents/pi-embedded-utils.ts:48-51`
  - body: Parameter text is copied directly from the XML capture, so encoded values like `&amp;`, `&lt;`, or `&quot;` reach tools unchanged. Decode standard XML entities before assigning `toolUse.input` values so commands and file contents match what the model encoded.
  - confidence: 0.88
