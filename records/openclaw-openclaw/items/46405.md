## Summary

Keep this PR open, but do not merge it as-is. Current main has already moved the gateway restart behavior into a newer config write-flow, while the MiniMax XML promotion is still absent and the proposed parser has two blocking correctness bugs.

## What This Changes

The PR adds MiniMax XML tool-call parsing/promotion before assistant text extraction, changes config.patch restart scheduling and gateway tool/system prompt wording, and adds agent/gateway regression tests.

## Best Possible Solution

Split the remaining work against current main: keep the newer config write-flow as the gateway boundary, resolve any remaining config.patch reload issue through that flow and #43803, and implement MiniMax XML promotion only if still needed with order-independent XML attribute parsing, standard entity decoding, and focused agent tests.

## Reproduction Assessment

Yes. The MiniMax parser findings reproduce from direct parser inputs using valid XML attribute ordering and encoded parameter values, and current-main source/tests show MiniMax XML is still stripped rather than promoted; this read-only review did not run tests.

## Solution Assessment

No. The PR points at real bugs, but the combined branch is not the best merge shape because the gateway half is stale/superseded and the MiniMax parser would still drop valid XML or pass encoded tool arguments through unchanged.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Parse XML attributes in any order:** `src/agents/pi-embedded-utils.ts:41`
  - body: `parseMinimaxToolCallXml` only recognizes `<invoke>` and `<parameter>` when `name` is the first attribute. Valid XML such as `<invoke id="x" name="Bash">` is skipped, after which no `toolUse` block is produced and the sanitizer strips the intended call.
  - confidence: 0.9
- **[P2] Decode XML entities before building tool input:** `src/agents/pi-embedded-utils.ts:48-51`
  - body: Parameter text is copied directly from the XML capture, so encoded values like `&amp;`, `&lt;`, or `&quot;` reach tools unchanged. Decode standard XML entities before assigning `toolUse.input` values so commands and file contents match what the model encoded.
  - confidence: 0.88
