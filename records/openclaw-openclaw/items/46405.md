## Summary

Keep open: current main still strips MiniMax XML tool calls instead of promoting them, but this PR is not merge-ready because the MiniMax parser has concrete correctness bugs and the gateway half overlaps a newer current-main config-write flow.

## What This Changes

The PR adds MiniMax XML tool-call parsing/promotion in agent message handling and changes gateway `config.patch` restart scheduling, tests, and agent prompt/tool wording for hot-reloadable config writes.

## Best Possible Solution

Replace or rebase this into a smaller current-main PR that adds robust MiniMax XML-to-`toolUse` promotion with attribute-order-safe parsing and XML entity decoding, while preserving the newer gateway config-write flow unless a focused current-main regression proves more gateway work is still needed.

## Reproduction Assessment

Yes. Source inspection reproduces the PR findings: the PR regex skips valid XML like `<invoke id="x" name="Bash">`, and parameter bodies are copied without XML entity decoding; current main also still has strip-only MiniMax behavior.

## Solution Assessment

No. The direction is useful, but the parser needs valid XML attribute handling and entity decoding, and the gateway changes need to be reconciled with current main's newer config-write restart path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Parse XML attributes in any order:** `src/agents/pi-embedded-utils.ts:43-49`
  - body: `parseMinimaxToolCallXml` only recognizes `<invoke>` and `<parameter>` when `name` is the first attribute. Valid XML like `<invoke id="x" name="Bash">...` is skipped, so no `toolUse` block is produced and the later text sanitization path can remove the XML entirely.
  - confidence: 0.91
- **[P2] Decode XML entities before passing tool inputs:** `src/agents/pi-embedded-utils.ts:49-51`
  - body: Parameter bodies are copied directly into `input`, so XML encodings such as `&amp;`, `&lt;`, `&quot;`, and `&apos;` are delivered to tools literally. That corrupts commands or file contents whenever MiniMax emits well-formed XML with escaped characters.
  - confidence: 0.88
