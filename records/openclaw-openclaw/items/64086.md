## Summary

Keep this issue open. Current main still lacks a plugin footer status contribution API and the TUI footer consumption path, while open PR #72557 explicitly resolves this issue and should be reviewed, landed, revised, or closed before the issue is closed.

## What This Changes

Request a public plugin API, tentatively `registerStatusProvider`, so context engine and other plugins can contribute compact status text to the TUI footer.

## Best Possible Solution

Keep this issue open until a focused implementation lands that exposes a stable status-provider registration/storage contract and either wires it into the TUI footer or explicitly tracks that rendering step with #53774.

## Reproduction Assessment

## Solution Assessment

Unclear. A narrow plugin registration API matches existing plugin patterns, but the final return shape and footer rendering behavior should be settled by maintainer review of PR #72557 and coordination with #53774.
