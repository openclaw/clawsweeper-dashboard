## Summary

Keep open. Current main still lacks a per-binding or ordinary per-channel Discord visible identity override; the existing webhook persona path is limited to bound thread sessions, and the related webhook identity report is already closed back to this issue as the canonical request.

## What This Changes

Add a Discord binding-level identity override so one bot token can display different agent names per routed channel.

## Best Possible Solution

Define a Discord-plugin-owned visible identity contract for ordinary routed Discord replies, likely reusing OutboundIdentity plus webhook username/avatar plumbing with schema, docs, tests, and safe fallback behavior.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken defined behavior. Source inspection shows strict route bindings lack the requested nickname field and ordinary Discord replies only get webhook identity through bound thread records.

## Solution Assessment

Unclear. The use case is concrete, but the proposed binding-level nickname is not clearly the best contract; maintainers should choose the Discord identity and webhook credential model before implementation.
