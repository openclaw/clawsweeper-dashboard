## Summary

Keep open. Current main still has no per-binding Discord identity override for ordinary routed channel replies; the only webhook persona path is tied to Discord thread bindings, so this remains a new Discord plugin config and delivery decision.

## What This Changes

Request an optional Discord binding identity or nickname override so one bot token can present different agent names per routed Discord channel.

## Best Possible Solution

Have maintainers choose a Discord-plugin-owned identity contract, then reuse the existing outbound identity and webhook sender for ordinary routed replies without putting Discord policy into generic core bindings.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior report. Source inspection shows route bindings validate only routing/session fields, and ordinary Discord sends do not use per-binding identity unless a bound thread has webhook credentials.

## Solution Assessment

Unclear. The requested nickname field captures the user need, but the maintainable solution should be selected by maintainers because Discord nickname mutation and webhook persona delivery have different config, permission, and secret-handling tradeoffs.
