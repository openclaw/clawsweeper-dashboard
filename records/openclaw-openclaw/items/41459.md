## Summary

Keep open: current main has Discord webhook persona delivery only for bound thread sessions, while this canonical issue still tracks ordinary Discord binding/channel identity override for one bot token. The linked webhook-identity issue was already closed as a duplicate of this issue, and the remaining work needs maintainer product/security direction rather than cleanup closure.

## What This Changes

The issue requests an optional per-binding Discord nickname or identity override so one Discord bot token can present different agent identities per channel.

## Best Possible Solution

If accepted, implement this inside the Discord plugin by reusing the existing OutboundIdentity and webhook sender path, adding a typed config contract, docs, focused tests, and avoiding core channel-specific schema unless a real plugin API gap appears.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection gives a high-confidence absence check for ordinary Discord binding/channel identity override on current main.

## Solution Assessment

Not as proposed without design review: Discord nickname mutation is guild-member scoped, while webhook username/avatar_url is the documented per-message identity mechanism. The maintainable path is likely Discord-plugin-local webhook identity plumbing rather than a literal per-channel nickname field. ([docs.discord.com](https://docs.discord.com/developers/resources/guild?utm_source=openai))
