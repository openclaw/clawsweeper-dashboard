## Summary

Keep open: current main and v2026.6.10 still do not provide a per-binding Discord nickname or ordinary routed-reply identity override, and the closest open related work is adjacent self-profile/profile-mutation design rather than the one-token, multi-agent visible-identity decision this issue asks for.

## What This Changes

Request an optional Discord binding-level nickname/identity override so multiple agents using one Discord bot token can appear with different visible names per channel.

## Best Possible Solution

Have maintainers choose one Discord plugin-owned visible-identity model, likely reusing webhook persona delivery if approved, then implement it with explicit schema, credential handling, docs, tests, and Discord proof.
