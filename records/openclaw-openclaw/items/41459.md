## Summary

Keep open: current main has the Discord webhook sender and thread-bound persona delivery pieces, but it still does not expose a per-binding or ordinary routed-reply Discord identity override for one bot token across multiple agents.

## What This Changes

Request adds a Discord binding-level nickname or identity override so agents sharing one Discord bot token can appear with distinct names per channel.

## Best Possible Solution

Add a Discord-plugin-owned visible identity path that reuses `OutboundIdentity` and `sendWebhookMessageDiscord`, with secret-safe config/docs and no core channel-specific binding field unless maintainers identify a real plugin API gap.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection confirms current main lacks a binding nickname/webhook schema and only uses webhook identity for bound thread replies.

## Solution Assessment

Unclear until maintainers choose the config and secret ownership model. The maintainable direction is likely to reuse the Discord plugin webhook sender and existing outbound identity plumbing rather than add a raw channel-specific field to the core binding contract by default.
