## Summary

Keep open. Current main still lacks a per-binding or ordinary per-channel Discord visible identity override; the existing webhook persona path is limited to bound thread replies, and the related Discord webhook identity report is already closed back to this issue as the canonical remaining request.

## What This Changes

Request optional Discord binding identity or nickname configuration so multiple agents using one bot token can display distinct names per Discord channel.

## Best Possible Solution

Define a Discord-plugin-owned visible identity contract for ordinary routed Discord replies, likely reusing existing OutboundIdentity and webhook username/avatar plumbing with schema, docs, tests, and safe fallback behavior.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken defined behavior. Source inspection shows the requested nickname field is not accepted by the strict binding schema, and current webhook persona delivery is limited to bound threads.

## Solution Assessment

Unclear. The user need is concrete, but the proposed binding-level nickname is not clearly the right contract; maintainers should choose the Discord identity and webhook credential model before implementation.
