## Summary

Keep open. Current main and the latest shipped release still lack ordinary per-channel or per-binding Discord visible identity overrides; the existing webhook persona path only applies to thread-bound replies, and the related webhook request is already closed back to this issue as the canonical remaining request.

## What This Changes

Request adding an optional Discord binding-level nickname or identity override so one bot token can show different agent names per routed channel.

## Best Possible Solution

Define a Discord-plugin-owned visible identity contract for ordinary routed Discord replies, reusing OutboundIdentity and webhook username/avatar plumbing with schema, docs, tests, and safe fallback behavior.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken defined behavior. Source inspection shows the proposed binding-level nickname is not accepted today, and current webhook persona delivery is limited to thread-bound replies.

## Solution Assessment

Unclear. The user need is concrete, but the proposed binding-level nickname is not clearly the best contract; maintainers should choose the Discord identity and webhook credential model before implementation.
