## Summary

Keep open. Current main still lacks a per-binding or per-agent visible Discord identity override for ordinary routed replies; the existing webhook persona path is limited to bound thread sessions, and the closed related report points here as the canonical remaining request.

## What This Changes

Add an optional Discord binding identity or nickname override so one bot token can display different agent names per bound channel.

## Best Possible Solution

Define a Discord-plugin-owned visible identity contract and delivery path, likely reusing OutboundIdentity plus webhook username/avatar support, with docs and focused tests before exposing new config.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken defined behavior. Source inspection shows the proposed nickname field would be rejected by the strict binding schema, and ordinary Discord replies still fall back to the bot account unless thread-binding webhook credentials exist.

## Solution Assessment

Unclear. The user need is concrete, but Discord nickname mutation and webhook persona delivery have different permission, credential, fallback, and rate-limit semantics, so maintainers should choose the config and delivery contract first.
