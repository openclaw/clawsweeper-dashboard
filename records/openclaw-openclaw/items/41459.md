## Summary

Keep open: current main still lacks a per-binding Discord visible identity override for ordinary route-bound replies. The existing webhook persona path is limited to thread-bound delivery, and the remaining work adds public config/API behavior that needs maintainer direction.

## What This Changes

The issue requests an optional per-binding Discord nickname or visible identity override so one Discord bot token can appear as different agents in different bound channels.

## Best Possible Solution

Add a documented Discord binding identity config in the bundled Discord channel surface, using webhook persona sends for normal route-bound replies with safe fallback to bot-account delivery.

## Reproduction Assessment

Yes, as a source-level gap. Adding the requested nickname field to a route binding would be rejected by the strict schema, and normal Discord route delivery does not carry binding identity into the outbound path.

## Solution Assessment

Unclear as proposed because this issue has no patch and Discord bot nicknames are not per-channel. A webhook persona path is likely the narrower maintainable solution, but the public config/API shape needs maintainer decision.
