## Summary

Keep open: current main still lacks a per-binding Discord visible identity override for ordinary route-bound replies. The existing webhook persona path is limited to thread-bound session delivery, and solving this adds public config/API surface that needs maintainer direction.

## What This Changes

Request an optional per-binding Discord nickname or visible identity override so one Discord bot token can show different agent names in different channel bindings.

## Best Possible Solution

Add a documented Discord route-binding identity path in the bundled Discord plugin, preferably using webhook persona sends for normal channel replies when configured and falling back cleanly to bot-account sends.

## Reproduction Assessment

Yes. A source-level reproduction is to configure a Discord route binding with the requested nickname field: the strict binding schema has no such field, and the normal route/reply path does not carry binding identity into Discord delivery.

## Solution Assessment

Unclear as an implementation because this issue has no patch. The user problem is valid, but changing Discord nicknames is likely the wrong primitive; a documented webhook persona path with explicit fallback is the narrower maintainable direction.
