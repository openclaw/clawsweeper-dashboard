## Summary

Keep open. Current main still has Discord webhook username/avatar delivery only for credentialed thread bindings, while ordinary route bindings have no supported per-channel or per-binding visible identity override; the remaining work is a product/config decision, not conservative cleanup.

## What This Changes

Request to add a Discord-specific per-binding identity override so multiple agents using one Discord bot token can display distinct names per routed channel.

## Best Possible Solution

Keep this as the canonical Discord identity design item and expose an opt-in Discord-owned outbound persona contract that reuses the existing webhook sender without mutating the bot's shared nickname per reply.

## Reproduction Assessment

Not applicable as a bug reproduction: this requests a new Discord presentation capability. Source and docs confirm ordinary route bindings currently cannot configure per-binding visible Discord identity.

## Solution Assessment

No. Changing a nickname per response is not the best shape because Discord nicknames are guild-member state; a Discord-owned webhook persona path is safer, but needs maintainer decisions on config ownership, secrets, precedence, and fallback behavior.
