## Summary

Keep open: current main has Discord webhook persona plumbing, but it is limited to thread-bound sessions and does not expose a binding-level or ordinary routed-channel identity override. This remains a clear feature/config design request, not a stale cleanup close.

## What This Changes

Add a Discord binding-level display identity override so one bot token can show different agent names per bound channel.

## Best Possible Solution

Design this as a Discord-plugin-owned visible identity feature that reuses the existing webhook username/avatar sender where appropriate, documents required Discord permissions and fallback behavior, and avoids adding generic core fields unless a real SDK/config gap is identified.

## Reproduction Assessment

Not applicable. This is a new Discord presentation capability request; source inspection is enough to show current main does not expose the requested override for ordinary routed bindings.

## Solution Assessment

No, not as literally proposed. Mutating a bot nickname per response is not the best fit because Discord nickname changes are guild-member scoped, while webhook execution supports per-message username/avatar; maintainers should choose a plugin-local webhook identity design. ([docs.discord.com](https://docs.discord.com/developers/resources/guild?utm_source=openai))
