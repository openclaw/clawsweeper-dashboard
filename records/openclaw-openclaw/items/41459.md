## Summary

Keep this issue open. Current main still does not provide a per-channel or per-binding Discord visible identity override for ordinary route-bound replies; the only webhook persona path is scoped to bound Discord threads, and the related per-agent webhook request was already closed as a duplicate of this issue rather than implemented.

## What This Changes

The issue requests an optional per-channel Discord identity override so one Discord bot token can present different agent names for different route bindings.

## Best Possible Solution

Implement a documented Discord route-binding identity path in the bundled Discord plugin, preferably using webhook persona sends for ordinary channel replies where available and falling back to bot-account sends when permissions or webhook setup are unavailable. The change should align config types, strict schema, generated help/labels, Discord docs, and focused tests for normal channel bindings and fallback behavior.

## Reproduction Assessment

Yes. A source-level reproduction is to configure two Discord route bindings with different desired nicknames: the strict binding schema rejects a nickname field, and the normal Discord reply path has no route-binding identity to pass to webhook delivery.

## Solution Assessment

Not applicable as a patch review because this issue has no proposed diff. The requested goal is valid, but mutating Discord nicknames is likely not the best primitive; per-message webhook persona delivery with explicit fallback is the safer maintainable direction.
