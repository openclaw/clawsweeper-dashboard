## Summary

Keep open. Current main and v2026.5.22 still do not expose a Discord route-binding nickname or visible identity override for ordinary routed channel replies; the existing webhook persona path only applies to bound thread replies, so the remaining work is a product/API decision rather than cleanup.

## What This Changes

Request adding a Discord route-binding identity override so one Discord bot token can display different agent names in different bound channels.

## Best Possible Solution

Define a Discord-plugin-owned visible identity contract for routed Discord replies, reusing existing outbound identity and webhook username/avatar delivery with documented credential and fallback rules.

## Reproduction Assessment

Not applicable; this is a feature request rather than a bug report. Source inspection confirms current route bindings do not expose a nickname or outbound identity override for ordinary Discord replies.

## Solution Assessment

Unclear. The user need is valid, but a raw binding nickname alone is probably too narrow because Discord visible identity also needs webhook credentials, avatar handling, fallback behavior, and ownership rules.
