## Summary

Keep open. Current main still lacks a Discord route-binding or ordinary channel reply identity override; existing persona delivery is only available when a Discord thread binding has webhook credentials.

## What This Changes

Request adding a Discord binding-level identity or nickname override so one bot token can show different agent names per routed channel.

## Best Possible Solution

Define a Discord-plugin-owned visible identity contract for ordinary routed replies, likely reusing OutboundIdentity plus webhook username/avatar delivery, with schema, docs, tests, and explicit credential/fallback rules.

## Reproduction Assessment

Not applicable; this is a feature request rather than a bug. Source inspection confirms the requested binding-level identity field is absent and ordinary Discord route replies do not use a per-channel visible identity override.

## Solution Assessment

Unclear. The user need is concrete and current webhook username/avatar primitives exist, but maintainers should choose the public config, ownership, and credential/fallback contract before implementation.
