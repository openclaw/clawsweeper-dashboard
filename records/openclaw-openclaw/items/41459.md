## Summary

Keep open. Current main still lacks a per-binding or per-agent visible Discord identity override for ordinary routed channel replies, while the related duplicate at https://github.com/openclaw/openclaw/issues/46351 is already closed and points here as the canonical remaining feature request.

## What This Changes

Add an optional Discord binding identity override so one Discord bot token can show different agent names per bound channel.

## Best Possible Solution

Have maintainers choose a Discord-plugin-owned identity contract, likely reusing the existing webhook username/avatar sender and OutboundIdentity plumbing without putting Discord-specific policy into generic core bindings.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken existing behavior. Source inspection shows ordinary Discord route bindings have no accepted identity field and normal channel replies do not use webhook persona delivery unless a bound thread supplies webhook credentials.

## Solution Assessment

Unclear. The requested nickname field captures the user need, but Discord nickname mutation and webhook persona delivery have different permission, credential, and fallback semantics, so the final shape needs maintainer product direction.
