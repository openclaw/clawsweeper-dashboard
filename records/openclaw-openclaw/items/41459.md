## Summary

Keep open. Current main only applies Discord webhook persona delivery to thread-bound sessions, while ordinary binding-routed Discord channel replies still fall back to the bot account and the binding schema has no identity override; this remains a new Discord plugin product/config decision.

## What This Changes

Add a Discord binding-level identity override so one bot token can show different agent names per bound channel.

## Best Possible Solution

Add a Discord-plugin-owned identity delivery path for ordinary routed messages, likely reusing existing OutboundIdentity and webhook username/avatar support after maintainers choose the config shape.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior report. Source inspection shows current bindings choose the agent only, and ordinary Discord sends do not use a per-binding identity override.

## Solution Assessment

No final solution has been chosen. The proposed nickname field is useful directionally, but the maintainable fix likely belongs in the Discord plugin delivery/config surface and should reuse the existing webhook identity plumbing rather than adding Discord policy to core bindings blindly.
