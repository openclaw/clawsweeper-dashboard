## Summary

Keep open. Current main still has no per-binding or ordinary per-channel Discord visible identity override; the only webhook persona path is for thread-bound replies, and the related webhook identity report is already closed back to this issue as the canonical remaining request.

## What This Changes

Add an optional Discord binding identity override so one bot token can present different visible agent names per channel.

## Best Possible Solution

Define a Discord-plugin-owned visible identity contract for ordinary routed Discord replies, likely reusing OutboundIdentity and webhook username/avatar plumbing with schema, docs, tests, and safe fallback behavior.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken defined behavior. Source inspection shows the requested nickname field is not accepted by the strict binding schema, and current webhook persona delivery is limited to bound threads.

## Solution Assessment

Unclear. The user need is concrete, but the proposed binding-level nickname is not clearly the best contract; maintainers should choose the Discord identity and webhook credential model before implementation.
