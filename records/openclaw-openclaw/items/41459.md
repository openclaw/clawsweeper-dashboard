## Summary

Keep open: current main has Discord thread-bound webhook persona delivery and outbound identity plumbing, but it still does not expose a per-binding nickname or ordinary routed Discord reply identity override. The related closed report at https://github.com/openclaw/openclaw/issues/46351 was explicitly consolidated into this issue, so this remains the canonical product/config decision.

## What This Changes

Request adds an optional Discord binding-level nickname or identity override so one Discord bot token can show different agent names in different bound channels.

## Best Possible Solution

Have maintainers choose the Discord identity model first, likely reusing agents.list[].identity plus the existing webhook sender in the Discord plugin for ordinary routed replies, then add schema, docs, and focused tests for that chosen surface.

## Reproduction Assessment

Not applicable as a feature request. Source inspection does show the requested config surface and ordinary routed Discord reply identity override are absent, while thread-bound webhook persona delivery exists.

## Solution Assessment

Unclear. A binding-level nickname field is a plausible user-facing shape, but the narrower maintainable direction should first compare per-binding identity against existing agent identity and Discord plugin-owned webhook config.
