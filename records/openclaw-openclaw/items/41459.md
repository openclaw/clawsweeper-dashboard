## Summary

Keep open. Current main still does not provide a per-binding or per-agent visible Discord identity override for ordinary routed Discord channel replies; the only related duplicate is already closed and points back here as the canonical remaining feature request.

## What This Changes

Add an optional Discord binding identity override so one bot token can present different visible agent names per routed Discord channel.

## Best Possible Solution

Have maintainers choose a Discord-plugin-owned visible identity contract, likely reusing OutboundIdentity and the existing webhook username/avatar sender without putting Discord-specific policy into generic core bindings unless a real SDK seam is missing.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source inspection gives high confidence that current main lacks the requested route-binding identity surface and only applies Discord webhook persona delivery to bound thread replies.

## Solution Assessment

Unclear. The requested nickname field captures the user need, but Discord nickname mutation and webhook persona delivery have different permission, credential, and fallback semantics, so maintainers should choose the product/API shape before implementation.
