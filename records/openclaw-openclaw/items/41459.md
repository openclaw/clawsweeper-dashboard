## Summary

Keep open: current main and v2026.6.8 still lack a per-binding Discord nickname or ordinary routed-reply identity override. Existing Discord webhook persona delivery is limited to thread-bound sessions, and this issue remains the canonical open thread for the one-token, multi-agent Discord visible-identity decision.

## What This Changes

The issue requests an optional per-binding Discord nickname so one bot token can present different visible agent identities per channel.

## Best Possible Solution

Have maintainers choose the Discord visible-identity model first, then implement it inside the Discord plugin with schema, docs, tests, and explicit security handling for webhook credentials or bot-profile mutation behavior.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection gives high-confidence absence of the requested binding-level nickname and ordinary routed-reply identity override on current main and v2026.6.8.

## Solution Assessment

Unclear as proposed. A binding-level nickname is plausible, but the best fix should first compare that shape against existing agent identity, Discord thread-binding webhook delivery, and the adjacent self-profile proposal.
