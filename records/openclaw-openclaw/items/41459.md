## Summary

Keep open. Current main still does not expose a per-binding or ordinary per-channel Discord visible identity override; the only webhook persona path is tied to Discord thread bindings, and the related duplicate points back here as the canonical remaining request.

## What This Changes

Add an optional Discord identity override so different agents bound to different Discord channels can appear with distinct visible names while sharing one bot token.

## Best Possible Solution

Define a Discord-plugin-owned visible identity contract that reuses outbound identity and webhook username/avatar support for ordinary routed replies, with schema, docs, tests, and safe fallback behavior.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken defined behavior. Source inspection shows the proposed nickname field is not accepted by the strict binding schema, and ordinary Discord text sends only use webhook persona when a thread binding supplies webhook credentials.

## Solution Assessment

Unclear. The user need is concrete, but maintainers should choose the Discord identity and credential model before adding a new binding/config surface.
