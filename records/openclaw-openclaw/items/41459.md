## Summary

Keep open. Current main still lacks a normal Discord route-binding or per-channel visible identity override; the only webhook persona path is tied to thread bindings, and the related webhook proposal was already closed back to this as the canonical remaining feature.

## What This Changes

The issue requests an optional Discord binding-level nickname/identity override so one bot token can present different agent names in different Discord channels.

## Best Possible Solution

Define a Discord-plugin-owned visible identity contract for routed Discord replies, likely reusing OutboundIdentity plus webhook username/avatar delivery, with schema, docs, tests, and explicit credential/fallback rules.

## Reproduction Assessment

Not applicable; this is a feature request rather than a bug. Source inspection confirms the requested binding-level identity field is absent and ordinary Discord route sends still use the bot-account path unless a thread-bound webhook binding is present.

## Solution Assessment

Unclear. The user need is concrete and current webhook username/avatar primitives exist, but maintainers should choose the public config, ownership, and credential/fallback contract before implementation.
