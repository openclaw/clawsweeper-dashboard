## Summary

Keep open: current main still lacks a per-binding Discord visible identity override for ordinary route-bound replies. The existing webhook persona path is limited to thread-bound delivery, and the remaining work adds public config/API behavior that needs maintainer direction.

## What This Changes

Add an optional per-binding Discord nickname/persona override so one bot token can display different agent identities in different bound channels.

## Best Possible Solution

Define a documented Discord route-binding persona config and use webhook persona delivery for configured normal channel replies, with secure credential handling and fallback to bot-account sends.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: add the requested nickname to a Discord route binding and current main rejects or drops it because the binding schema is strict and normal Discord replies receive no route-binding identity.

## Solution Assessment

Unclear because this issue has no patch. The user problem is valid, but mutating Discord nicknames is likely the wrong primitive; an explicit webhook persona path for configured route-bound replies is the narrower maintainable direction.
