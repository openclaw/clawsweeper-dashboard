## Summary

Keep open: current main and the latest release still only use Discord webhook persona delivery for thread-bound sessions, not for ordinary per-channel route bindings. The related webhook-override issue was closed as a duplicate of this issue, and an earlier PR was explicitly closed for a different future implementation, so this remains the canonical product/API decision.

## What This Changes

The request is to let Discord-bound agents present different visible Discord identities per binding or channel while sharing one bot token.

## Best Possible Solution

Keep this as the canonical Discord visible-identity issue until maintainers choose a plugin-owned design that reuses existing outbound identity and webhook sender plumbing without adding Discord-specific schema to core unless a real SDK gap requires it.

## Reproduction Assessment

Not applicable. This is a feature/configuration request, and source inspection confirms the requested per-binding or per-channel visible Discord identity is not an existing documented behavior to reproduce as broken.

## Solution Assessment

Unclear. The requested `bindings[].nickname` shape is probably not the best final API because current code already has agent identity and webhook persona primitives, but the user-visible outcome remains unsolved for normal Discord route bindings.
