## Best Possible Solution

Keep this PR as the canonical positional-completion fix, reuse or align with the expansion-safe zsh escaping direction from https://github.com/openclaw/openclaw/pull/64490, then land after redacted live zsh proof and fresh validation.

## Reproduction Assessment

Yes, source-level: current main and the latest release emit only option specs for leaf zsh functions while `openclaw wiki ingest` declares a required local-file positional. I did not establish an interactive zsh Tab-completion session in this read-only review.

## Solution Assessment

No, not as submitted. Reading Commander `registeredArguments` in the leaf zsh generator is the right narrow design, but the branch needs `$`/backtick escaping, a rebase, and real zsh proof before it is the best mergeable solution.
