## Summary

Keep open: current main and the latest release still omit positional arguments from leaf zsh completion functions, and this PR remains the canonical active fix path. The branch is not merge-ready because the new positional-label escaping still misses zsh `$`/backtick interpolation, real zsh behavior proof is absent, and GitHub currently reports the branch as conflicting.

## What This Changes

The PR adds zsh positional argument specs for leaf Commander commands, focused completion tests for file, directory, generic, optional, and variadic positionals, and a changelog entry.

## Best Possible Solution

Keep this PR as the canonical fix path, rebase it, apply expansion-safe escaping for the new positional label path, and land after redacted live zsh proof demonstrates file completion for the reported command.

## Reproduction Assessment

Yes, source-level: current main and the latest release emit only option specs for leaf zsh functions while `openclaw wiki ingest` declares a required local-file positional. I did not establish an interactive Tab-completion session in this read-only review.

## Solution Assessment

No as submitted. Reading Commander `registeredArguments` in the leaf zsh generator is the right narrow design, but the patch needs `$`/backtick escaping, a rebase, and real zsh proof before it is the best mergeable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Escape zsh interpolation in positional labels:** `src/cli/completion-cli.ts:230-235`
  - body: `escapeZshLabel` feeds descriptions into double-quoted `_arguments` specs but does not escape `$` or backticks. Zsh performs parameter and command substitution inside double quotes, so a positional description from core or plugin metadata can be evaluated when users source generated completions instead of displayed literally.
