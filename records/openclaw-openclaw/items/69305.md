## What This Changes

The PR teaches the zsh completion generator to emit Commander positional argument specs for leaf commands, chooses filesystem completion actions for explicit file/directory arguments, adds regression tests, and adds a changelog entry.

## Best Possible Solution

## Reproduction Assessment

Yes. Generate zsh completion for a Commander leaf command with `.argument("<path>", ...)` and inspect the emitted function; current main’s leaf template emits option specs only while the memory-wiki `wiki ingest` command declares the affected positional file argument.

## Solution Assessment

Yes, once rebased. Using Commander’s `registeredArguments` in the zsh leaf renderer is the narrowest maintainable fix, and the latest branch also avoids the earlier false positive for generic config key paths.
