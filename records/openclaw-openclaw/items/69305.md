## Summary

Keep open. Current main and the latest release still omit zsh positional specs for leaf commands, and this PR is the active canonical fix path, but the patch is not mergeable as submitted because the new positional-label escaping still allows zsh interpolation and the PR lacks live after-fix zsh proof.

## What This Changes

The PR adds zsh positional argument specs for leaf Commander commands, tests file/directory/generic/optional/variadic cases, and adds a changelog entry.

## Best Possible Solution

Keep this PR as the canonical positional-argument fix, align its zsh label escaping with the expansion-safe direction in https://github.com/openclaw/openclaw/pull/64490, rebase it, and land after redacted live zsh proof.

## Reproduction Assessment

Yes, source-level: current main and `v2026.5.12` emit only option specs for leaf zsh functions while `openclaw wiki ingest` declares a required local-file positional. I did not establish an interactive Tab-completion session in this read-only review.

## Solution Assessment

No as submitted. Reading Commander `registeredArguments` in the leaf zsh generator is the right narrow design, but the patch needs `$`/backtick escaping, a rebase, and real zsh proof before it is the best mergeable solution.
