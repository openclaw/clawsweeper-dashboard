## What This Changes

The PR adds zsh positional argument specs for leaf Commander commands, focused completion tests for file/directory/optional/variadic arguments, and a changelog entry.

## Best Possible Solution

Keep this branch as the canonical fix path, add expansion-safe zsh label escaping, rebase it, and land after redacted live zsh proof demonstrates file completion for `openclaw wiki ingest`.

## Reproduction Assessment

Yes, source-level: current main emits only option specs for leaf zsh functions while `openclaw wiki ingest` declares a `<path>` positional. I did not establish an interactive Tab-completion session in this read-only review.

## Solution Assessment

No as submitted: using Commander `registeredArguments` in the leaf zsh generator is the right narrow design, but the patch needs `$`/backtick escaping, a rebase, and live zsh proof before it is merge-ready.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Escape zsh interpolation in positional labels:** `src/cli/completion-cli.ts:230-235`
  - body: The new label helper feeds descriptions into double-quoted `_arguments` specs but only escapes backslash, double quote, colon, and brackets. A positional description containing `$` or backticks can be expanded by zsh when the generated script is sourced, so the new positional path needs shell-interpolation escaping too.
  - confidence: 0.86
