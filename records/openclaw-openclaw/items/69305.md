## Summary

Keep open: current main still has the zsh positional-completion bug and this is the only open canonical fix, but the PR should not merge until positional labels use complete double-quoted zsh escaping and the contributor adds real zsh Tab-completion proof.

## What This Changes

The PR adds zsh positional argument specs for leaf Commander commands, focused completion tests, and a changelog entry.

## Best Possible Solution

Keep the fix in the zsh leaf generator, share or extend the zsh escaping helper for positional labels, and land after live proof shows file completion for the reported wiki ingest flow.

## Reproduction Assessment

Yes for source-level reproduction: current main’s leaf zsh branch calls `generateZshArgs(cmd)` only, and `openclaw wiki ingest` declares `<path>` as a file positional. I did not establish a live zsh Tab-completion session in this read-only review.

## Solution Assessment

No as submitted: using Commander `registeredArguments` in the leaf zsh generator is the right narrow design, but the patch needs the `$`/backtick escaping gap fixed and live proof before it is merge-ready.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Escape zsh interpolation in positional labels:** `src/cli/completion-cli.ts:230-235`
  - body: The new helper emits positional labels inside double-quoted `_arguments` specs but does not escape `$` or backticks. If a positional description contains either character, zsh can expand it when completion runs instead of displaying it literally, and the existing option-description escape follow-up would not cover this new helper.
  - confidence: 0.86
