## Summary

Keep open: this is a valid, narrow fix path for a real zsh completion bug, but the current PR still lacks required live zsh behavior proof, is conflicting, and leaves shell interpolation unescaped in the new positional label path.

## What This Changes

The PR adds zsh positional argument specs for leaf Commander commands, heuristic file/directory actions, focused completion tests, and a changelog entry.

## Best Possible Solution

Keep this branch as the canonical fix path, escape positional labels for zsh interpolation, rebase the conflict, and land only after redacted live zsh proof shows file completion for `openclaw wiki ingest`.

## Reproduction Assessment

Yes, source-level: current main emits only option specs for leaf zsh functions while `openclaw wiki ingest` declares a `<path>` positional. I did not establish an interactive Tab-completion session in this read-only review.

## Solution Assessment

No as submitted: using Commander `registeredArguments` in the leaf zsh generator is the right narrow design, but the patch needs the interpolation escaping fix, a rebase, and live zsh proof before it is merge-ready.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Escape zsh interpolation in positional labels:** `src/cli/completion-cli.ts:230-235`
  - body: The new label helper feeds descriptions into double-quoted `_arguments` specs but only escapes backslash, double quote, colon, and brackets. A positional description containing `$` or backticks can be expanded by zsh when the generated script is sourced, so the new positional path needs shell-interpolation escaping too.
  - confidence: 0.86
