## Summary

Keep open. Current main still lacks a `channelTimeoutSeconds` config/schema/docs surface and the runtime timeout helper has no channel-aware fallback; cron only has explicit per-job timeout override behavior.

## What This Changes

The issue requests per-channel agent timeout overrides for Telegram, Discord, webchat, and cron while preserving existing global timeout fallback semantics.

## Best Possible Solution

Add a backward-compatible typed channel timeout resolver shared by interactive and cron paths, document the fallback order, and keep existing global timeout behavior unchanged unless explicit channel overrides are configured.

## Reproduction Assessment

Yes for the repository-level gap: `channelTimeoutSeconds` is absent from source/docs, and the timeout resolver has no channel input. The production OAuth soft-throttle hang is not needed to reproduce the missing configuration feature.

## Solution Assessment

Yes, a generic channel-aware timeout resolver is the narrow maintainable direction. Built-in default values should be handled as maintainer policy, separate from accepting and applying explicit config overrides.

## Repair Work Prompt

Implement explicit per-channel agent timeout configuration for this issue without changing the existing global default unless maintainers later approve built-in per-channel defaults. Add a typed `channelTimeoutSeconds` map to `agents.defaults` and `agents.list[]`, with schema validation, generated config metadata/help/docs, and runtime resolution order: cron job `payload.timeoutSeconds`; per-agent channel override; global channel override; existing per-agent/global timeout behavior where supported; current default. Thread normalized channel identity into interactive Telegram/Discord/webchat reply and Gateway chat/agent paths, and use a conservative `cron` or delivery-channel fallback for isolated cron runs only when no job timeout is set. Preserve `timeoutSeconds: 0` no-timeout semantics for explicit overrides. Do not introduce bundled plugin IDs into core beyond generic channel keys, and do not change provider HTTP/model timeout semantics. Add focused tests for resolver precedence, strict schema acceptance/rejection, auto-reply/Gateway chat channel selection, and cron override priority. Add a user-facing CHANGELOG entry and update agent config docs.
