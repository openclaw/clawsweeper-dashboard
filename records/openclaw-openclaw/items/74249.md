## Summary

Keep open. Current main still lacks the requested `channelTimeoutSeconds` config/schema/docs surface, and runtime timeout resolution remains global plus explicit overrides rather than channel-aware.

## What This Changes

Add per-channel agent runtime timeout overrides for Telegram, Discord, webchat, and cron while preserving existing global and per-job timeout semantics.

## Best Possible Solution

Add a backward-compatible typed `channelTimeoutSeconds` map to agent defaults and entries, pass normalized channel identity into timeout resolution for interactive paths, preserve cron job `payload.timeoutSeconds` precedence, and document/test the fallback order.

## Reproduction Assessment

Yes for the repository-level gap: current main has no `channelTimeoutSeconds` key and no channel argument in timeout resolution. The live OAuth soft-throttle hang was not reproduced, but it is not needed to verify the missing configuration capability.

## Solution Assessment

Yes, a generic channel-aware resolver is the narrow maintainable direction. The only unclear part is whether OpenClaw should ship built-in channel-specific default values or keep the new behavior opt-in.

## Repair Work Prompt

Implement per-channel agent timeout overrides requested in the linked issue. Add a backward-compatible `channelTimeoutSeconds` map for `agents.defaults` and `agents.list[]` with strict schema/types/generated docs support, thread normalized channel identity into the shared agent timeout resolver, and apply the requested precedence: cron job `payload.timeoutSeconds`, per-agent channel override, global channel override, per-agent timeout when the config model supports it, global timeout, then built-in default. Preserve existing behavior when the map is unset, keep cron explicit timeout as highest priority, and do not hardcode policy defaults such as Telegram=120s unless maintainers explicitly approve. Add focused unit/integration coverage for schema validation, resolver fallback order, auto-reply channel calls, webchat/chat RPC behavior, agent RPC origin channel behavior, and cron preservation. Add a user-facing `CHANGELOG.md` fix/feature entry and update the agent config reference/docs. Do not change unrelated channel transport timeouts like `channels.telegram.timeoutSeconds`.
