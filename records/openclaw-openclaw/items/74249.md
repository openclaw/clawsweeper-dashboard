## Summary

Keep open. Current main still has only a global agent runtime timeout plus explicit request/job overrides; there is no typed `channelTimeoutSeconds` config, schema/docs surface, or channel-aware timeout resolution for Telegram, Discord, webchat, or cron fallback behavior.

## What This Changes

Request per-channel agent timeout overrides under `agents.defaults` and agent entries so interactive channels can fail fast without shortening webchat or cron runs.

## Best Possible Solution

Add a generic, typed `channelTimeoutSeconds` resolver for defaults and agent entries, thread normalized channel identity through the agent timeout call sites, keep cron `payload.timeoutSeconds` as top priority, and document/test the fallback order without changing existing defaults unless maintainers approve.

## Reproduction Assessment

Yes for the repository-level gap: `channelTimeoutSeconds` is absent and current timeout resolution has no channel parameter. The provider soft-throttle hang was not live-reproduced, but it is not needed to verify the missing configuration capability.

## Solution Assessment

Yes for the override mechanism if implemented as a generic resolver extension. The suggested built-in default values are the part that needs maintainer product judgment rather than automatic repair.

## Repair Work Prompt

Implement per-channel agent timeout overrides for https://github.com/openclaw/openclaw/issues/74249. Add a generic `channelTimeoutSeconds` config surface for `agents.defaults` and `agents.list[]` entries, with normalized string channel keys such as `telegram`, `discord`, and `webchat`; do not hard-code only bundled channels or exclude third-party channel plugins. Extend the agent timeout resolver so the fallback order is: explicit cron/request timeout override, per-agent channel timeout, global channel timeout, per-agent/global existing timeout behavior as currently supported, then the existing default. Thread channel identity through the interactive reply path and webchat/gateway agent paths without changing existing behavior when `channelTimeoutSeconds` is absent. Keep cron `payload.timeoutSeconds` as the top-priority explicit override. Add schema/types/generated config help/docs coverage and focused unit tests for fallback order, strict schema acceptance/rejection, interactive channel resolution, webchat resolution, and cron explicit override precedence. Add a user-facing changelog entry. Do not change built-in default values for Telegram, Discord, webchat, or cron unless a maintainer explicitly approves that policy change.
