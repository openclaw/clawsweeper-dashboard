## Summary

Keep this issue open. Current main still has global/default agent-run timeout resolution plus explicit request or cron payload overrides, but no channel-aware timeout config for Telegram, Discord, WebChat, or cron defaults. The related Telegram stall and active-memory reports are useful context, but they do not supersede or implement the requested public config surface.

## What This Changes

Request to add channel-aware agent-run timeout configuration for Telegram, Discord, WebChat, and cron defaults while preserving explicit cron job timeout precedence.

## Best Possible Solution

Add one shared channel-aware timeout resolver used by Gateway agent runs, WebChat/chat.send, and cron fallback paths; preserve existing global behavior when no override is configured, keep cron `payload.timeoutSeconds` first, and update config schemas, generated metadata/help, docs, and focused regression tests together.

## Reproduction Assessment

Yes for the config/runtime gap: current main can be verified by searching for `channelTimeoutSeconds`, inspecting the strict schemas, and following Gateway/WebChat/cron timeout calls. The OpenAI soft-throttle hang itself remains environment-dependent.

## Solution Assessment

Unclear until maintainers approve the public API shape and default values. The maintainable implementation direction is a shared resolver plus schema/docs coverage, not separate transport-level timeout tweaks in each channel.
