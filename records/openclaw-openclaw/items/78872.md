## Summary

Keep open: this PR targets a credible Feishu topic-session split and includes useful live log proof, but the current head still breaks existing normal-group root-session behavior and adds authenticated Feishu message reads before group admission gates.

## What This Changes

The PR adds Feishu topic-thread hydration from `rootId`, suppresses `rootId` for `omt_*` topic routing, adds two Feishu regression tests, and adds a changelog entry.

## Best Possible Solution

Keep the Feishu-local direction, but add a reliable discriminator for Feishu topic-message-format topics, preserve existing normal group root-key flows, and move the extra hydration behind admission checks.

## Reproduction Assessment

Yes, from source inspection: current main can route a normal group topic first turn by `omt_*` when no root is present and a follow-up by `om_*` when a root is present; the contributor also posted redacted 2026.5.6 before/after Feishu logs. I did not execute tests in this read-only review.

## Solution Assessment

No as-is: the Feishu plugin is the right surface and the live proof is useful, but this patch treats all normal-group `om_*`/`omt_*` pairs as native topics and performs the new message lookup before group admission gates.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

- **[P2] Preserve normal group root-topic sessions:** `extensions/feishu/src/bot.ts:545-547`
  - body: This condition treats every normal-group `root_id` starting `om_` plus an `omt_*` thread as a native topic. Existing tests cover the same shape and require normal OpenClaw-created group threads to keep the `om_*` root key, so this branch would reroute those follow-ups by `omt_*` instead.
  - confidence: 0.94

  - confidence: 0.86
