Full review comments:

- **[P2] Flush buffered Chat text before tool-call fallback:** `src/gateway/openai-http.ts:1262`
  - body: With buffering enabled, assistant deltas only set `sawAssistantDelta` when the chunker flushes. A short buffered response that later resolves to tool calls can enter the tool-call branch with `sawAssistantDelta` still false, write fallback commentary and tool-call chunks, then flush the buffered text during finalization, producing duplicate or out-of-order content.
  - confidence: 0.88
- **[P2] Finalize Responses after flushing buffered text:** `src/gateway/openresponses-http.ts:952`
  - body: `requestFinalize()` captures `accumulatedText` before `finalUsage` allows `maybeFinalize()` to run, but `maybeFinalize()` flushes the chunker only afterward. For short buffered streams, the final `response.output_text.done`, `response.content_part.done`, and `response.completed` payloads can omit the tail that was just emitted as a delta.
  - confidence: 0.9
- **[P2] Complete the public config contract:** `src/config/types.gateway.ts:329-330`
  - body: Adding `streaming` under both endpoint configs makes `minChars`, `maxChars`, and `idleMs` public operator-facing keys, but the config labels/help and Gateway docs still only describe the existing endpoint fields. Please add the metadata/docs and focused schema/runtime coverage so operators and generated config surfaces see the same contract the runtime accepts.
  - confidence: 0.82
