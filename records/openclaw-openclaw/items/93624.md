## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Honor configured chunk thresholds:** `src/gateway/gateway-streaming-chunker.ts:67-70`
  - body: With buffering enabled, `minChars` never triggers a flush and `maxChars` does not cap emitted chunks: `push()` appends the whole delta and `maybeFlush()` flushes the entire buffer only after it is already at or past `maxChars`. A config like `{ minChars: 200, maxChars: 800, idleMs: 0 }` can therefore hold text until 800/finalization rather than 200, and a crossing delta can emit more than 800 chars, so the new knobs do not provide the chunk-size contract they advertise.
  - confidence: 0.88
- **[P2] Flush Chat deltas before tool calls:** `src/gateway/openai-http.ts:1262`
  - body: `streamChunker.push(content)` keeps `sawAssistantDelta` false until a later flush. If a streamed response ends with tool calls while a short assistant delta is buffered, the tool-call branch can write fallback commentary and tool-call chunks before finalization flushes the buffered text, producing duplicate or out-of-order Chat Completions SSE content.
  - confidence: 0.88
- **[P2] Refresh Responses final text after flushing:** `src/gateway/openresponses-http.ts:952`
  - body: `maybeFinalize()` flushes the buffered delta after `finalizeRequested.text` has already been captured. The flush updates `accumulatedText`, but the following `response.output_text.done`, `response.content_part.done`, and `response.completed` events still use the stale `finalizeRequested.text`, so configured short streams can send a tail delta that is missing from final payloads.
  - confidence: 0.9
- **[P2] Complete the public config contract:** `src/config/types.gateway.ts:330`
  - body: This adds a strict exported Gateway config field, but the config labels/help and user-facing Gateway docs still do not describe `streaming.minChars`, `streaming.maxChars`, or `streaming.idleMs`. Before merge, those knobs need metadata/docs and focused schema/runtime coverage so Control UI, config lookup, and operators see the same contract that runtime accepts.
  - confidence: 0.84
