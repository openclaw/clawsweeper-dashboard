## Summary

Keep open. The PR targets the right narrow doctor-auth bug, but the current head disables the existing default-store OAuth refresh whenever a non-default direct agent store makes the default pass labeled, and the external PR still lacks real `openclaw doctor` behavior proof.

## What This Changes

The branch enumerates configured agent auth stores for `openclaw doctor`, loads them read-only, labels multi-agent auth notes with agent ids, exports direct auth-source detection, and adds regression tests plus a changelog entry.

## Best Possible Solution

Keep read-only labeled diagnostics for non-default direct agent stores, keep the default/runtime target eligible for the existing interactive OAuth refresh path, and add redacted real CLI proof before merge.

## Reproduction Assessment

Yes. Source inspection at PR head is enough: a direct non-default auth store causes the default/runtime target to be labeled, and the `params.agentId` gate then skips `confirmAutoFix` for that default pass.

## Solution Assessment

No. The labeling direction is right, but refresh eligibility should be based on target kind or `agentDir === undefined`, not on whether the diagnostic title has an agent-id suffix.
