## Summary

Keep open. The PR addresses the right user problem from #71867, but the current head regresses the existing interactive OAuth refresh path for the default/runtime auth store when a non-default direct agent store also exists.

## What This Changes

The PR makes `openclaw doctor` enumerate direct per-agent auth stores, load them read-only, label auth diagnostics with agent IDs, add source-check exports/tests, and add a changelog entry for #71867.

## Best Possible Solution

Keep read-only labeled diagnostics for non-default direct agent stores, but make refresh eligibility target-based so the default/runtime store preserves its existing interactive OAuth refresh behavior.

## Reproduction Assessment

Yes. A static PR-head reproduction is enough: with `main` and `coder` agents and any direct non-default auth store, the PR includes a labeled default target with `agentDir: undefined`, then the `params.agentId` gate skips the refresh prompt.

## Solution Assessment

No. The direction is maintainable, but refresh should be gated on the target kind or `agentDir === undefined`, not on whether the note title has an agent label.
