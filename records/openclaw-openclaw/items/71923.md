## Summary

Keep open: the PR remains a focused fix for the still-open doctor-auth issue, but the current head introduces a default/runtime OAuth refresh regression and lacks required real behavior proof.

## What This Changes

The PR makes `openclaw doctor` enumerate direct per-agent auth stores, label auth diagnostics by agent, add a direct auth-source check, regression tests, and a changelog entry.

## Best Possible Solution

Keep labeled read-only diagnostics for non-default direct stores, preserve default/runtime interactive OAuth refresh behavior, and include redacted real `openclaw doctor` output or logs before merge.

## Reproduction Assessment

Yes. Source inspection of PR head shows a multi-agent direct-store path where the default/runtime target is labeled and therefore skips `confirmAutoFix`; current main and doctor docs show the default interactive OAuth refresh prompt should still be preserved.

## Solution Assessment

No. Agent-labeled diagnostics are the right direction, but refresh eligibility should distinguish the default/runtime target from non-default direct agent targets instead of using the presence of an agent label.
