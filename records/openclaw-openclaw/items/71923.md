## Summary

Keep open: the PR addresses the right doctor-auth diagnostic gap for #71867, but the latest branch regresses the existing default-store OAuth refresh path when the default target is labeled in a multi-agent run.

## What This Changes

The PR enumerates direct per-agent auth stores in `openclaw doctor`, loads them read-only, suffixes auth diagnostics with agent IDs, exports a direct auth-store source helper, adds regression tests, and updates the changelog for #71867.

## Best Possible Solution

Keep labeled read-only diagnostics for non-default direct agent stores, but preserve the existing interactive refresh path for the default/runtime auth store and cover that behavior with a regression test.

## Reproduction Assessment

Yes. A static reproduction is enough: a multi-agent config with a direct non-default store makes the PR include `{ agentId: main, agentDir: undefined }`, and the new `params.agentId` gate disables refresh before `confirmAutoFix` can run.

## Solution Assessment

No. The direction is right, but refresh eligibility should be based on the target kind or `agentDir === undefined`, not on whether the output title has an agent label.
