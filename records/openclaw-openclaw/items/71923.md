## Summary

Keep open: this is a focused implementation for the open doctor-auth labeling issue, but the current PR head still has blocking default-agent/default-refresh regressions and lacks required real behavior proof.

## What This Changes

The branch changes `openclaw doctor` auth health to enumerate configured agent auth stores, label auth diagnostics with agent ids, add a direct auth-source helper, extend regression tests, and add a changelog entry.

## Best Possible Solution

Update the PR to distinguish runtime-default and configured per-agent targets, preserve interactive refresh for the true default runtime store, check the configured default agent store, and add redacted real doctor output or logs before merge.

## Reproduction Assessment

Yes. Source inspection of PR head shows the default/runtime target is labeled and then blocked from refresh; docs and code also show non-main default agents can have their own `agentDir` that the PR target filtering skips.

## Solution Assessment

No. Agent-labeled diagnostics are the right direction, but refresh eligibility and store selection should be based on explicit target kind and resolved agent directory, not on whether a display suffix exists.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use the configured default agent auth dir:** `src/commands/doctor-auth.ts:217-225`
  - body: When a multi-agent config uses a non-main default with its own `agentDir`, this filters that default id out of `directTargets` and substitutes `{ agentId: defaultAgentId, agentDir: undefined }`. The undefined auth path resolves through the implicit main store, so doctor can label main credentials as the configured default and miss the default agent's real auth store.
  - confidence: 0.82
- **[P2] Preserve refresh for the default auth store:** `src/commands/doctor-auth.ts:307-312`
  - body: With any non-default direct auth store, the default/runtime target is added with an agent label, then `params.agentId` makes `shouldRefresh` false. That removes the existing interactive OAuth refresh prompt for the default store in the multi-agent path this PR enables; gate diagnostics-only behavior on target kind instead of label presence.
  - confidence: 0.9
