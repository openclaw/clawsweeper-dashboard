## Summary

Keep open. The branch addresses the original diagnostic-labeling problem, but it still regresses the existing default-store OAuth refresh path because the default runtime target is labeled with an agent id and then refresh is disabled for every labeled pass.

## What This Changes

The PR enumerates direct per-agent auth stores for `openclaw doctor`, loads them read-only, adds agent-id suffixes to auth diagnostics, exports a direct auth-store source helper, extends regression tests, and adds a changelog entry for #71867.

## Best Possible Solution

Keep the per-agent read-only diagnostics and direct-store filtering, but make refresh eligibility depend on the default/runtime target (`agentDir: undefined`) or an explicit `canRefresh` flag instead of the display label; update the regression test so the labeled default store still prompts and refreshes while non-default direct stores remain diagnostics-only.

## Reproduction Assessment

Yes. A static reproduction is enough: with `main` and `coder` direct auth stores, the PR includes the default store as `{ agentId: main, agentDir: undefined }`, then `params.agentId ? false` prevents the default refresh prompt.

## Solution Assessment

No. The requested diagnostic direction is sound, but the current patch uses the note label as the refresh gate; the safer solution is to gate refresh on the target's runtime/default semantics.
