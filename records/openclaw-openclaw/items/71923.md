## Summary

Keep open: this is a relevant focused fix for the linked doctor auth bug, but the latest PR head still regresses the default/runtime OAuth refresh path and the external PR has only test/check output, not real behavior proof.

## What This Changes

The PR makes `openclaw doctor` auth health enumerate direct per-agent auth stores, label auth notes with agent ids, add direct auth-source detection, extend regression tests, and update the changelog.

## Best Possible Solution

Keep the read-only labeled diagnostics for non-default direct agent stores, preserve default/runtime interactive OAuth refresh behavior, and require redacted real `openclaw doctor` proof before merge.

## Reproduction Assessment

Yes. Source inspection of PR head shows a multi-agent direct-store path where the default/runtime target is labeled and therefore skips `confirmAutoFix`; current main and doctor docs show the default interactive OAuth refresh prompt should still be preserved.

## Solution Assessment

No. Agent-labeled diagnostics are the right direction, but refresh eligibility should be based on the target kind, such as non-default direct agent stores, rather than whether the display title has an agent-id suffix.
