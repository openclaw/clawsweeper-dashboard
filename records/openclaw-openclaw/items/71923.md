## Summary

Keep open; the branch still regresses existing interactive doctor behavior by disabling OAuth refresh for the default/runtime auth store whenever a non-default direct agent store causes the default pass to be labeled.

## What This Changes

The PR updates `openclaw doctor` auth health to enumerate direct per-agent auth stores, load them read-only, suffix auth diagnostic notes with agent IDs, export a direct auth-store source helper, add regression tests, and add a changelog entry for #71867.

## Best Possible Solution

Keep labeled read-only diagnostics for non-default direct agent stores while preserving the existing interactive refresh path for the default/runtime auth store and covering that case with regression tests.

## Reproduction Assessment

Yes. A static reproduction is enough: a multi-agent config with a direct non-default auth store makes the PR include the default/runtime store as a labeled target, and the new `params.agentId` gate skips `confirmAutoFix` for that target.

## Solution Assessment

No. The PR direction is right, but refresh eligibility should be based on the target kind or `agentDir === undefined`, not on whether the diagnostic title has an agent label.
