## Summary

Keep open. The PR targets a real doctor/auth diagnostics gap for #71867, but the latest head still regresses the existing default-store OAuth refresh prompt when multi-agent auth targets are labeled.

## What This Changes

The branch enumerates direct per-agent auth stores in `openclaw doctor`, loads them read-only, suffixes auth diagnostics with agent IDs, exports a direct auth-store source helper, adds regression tests, and adds a changelog entry for #71867.

## Best Possible Solution

Keep the per-agent read-only diagnostics and direct-store filtering, but gate refresh eligibility on the default/runtime target (`agentDir: undefined`) instead of the display label; non-default direct agent-store passes should remain diagnostics-only.

## Reproduction Assessment

Yes. A static reproduction is enough: with `main` and `coder` direct auth stores, the PR includes the default store as a labeled `{ agentId: main, agentDir: undefined }` target and then disables refresh because `params.agentId` is set.

## Solution Assessment

No. The requested direction is right, but the current patch is not the narrowest correct solution because it uses the note title label as the refresh gate; refresh should be controlled by whether the pass is the default/runtime target.
