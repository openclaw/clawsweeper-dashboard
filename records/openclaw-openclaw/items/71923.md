## What This Changes

The branch enumerates configured agent auth stores for `openclaw doctor`, labels multi-agent auth diagnostics, uses read-only runtime auth loading, exports a direct auth-source helper, and adds regression tests plus a changelog entry.

## Best Possible Solution

Keep the read-only labeled diagnostics for non-default direct agent stores, keep the default/runtime target eligible for the existing interactive OAuth refresh path, and add redacted real CLI proof before merge.

## Reproduction Assessment

Yes. Source inspection of PR head gives a high-confidence path: with a default store plus a non-default direct auth store, the default/runtime pass is labeled with an agent id and then skips `confirmAutoFix`.

## Solution Assessment

No. Labeling per-agent diagnostics is the right direction, but refresh eligibility should be based on target kind or `agentDir === undefined`, not on whether the display title has an agent-id suffix.
