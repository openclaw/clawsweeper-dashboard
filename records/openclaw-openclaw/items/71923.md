## Summary

Keep open: the PR addresses a valid doctor auth diagnostic gap, but the current head still disables the default/runtime OAuth refresh prompt when another direct agent auth store exists, and the external PR only provides test/check proof rather than real doctor output.

## What This Changes

The PR makes `openclaw doctor` enumerate configured agent auth stores, label auth diagnostics with agent ids, add a direct auth-source helper, add regression tests, and update the changelog.

## Best Possible Solution

Keep labeled read-only diagnostics for non-default direct agent stores, preserve default/runtime interactive OAuth refresh behavior, and include redacted real `openclaw doctor` output or logs before merge.

## Reproduction Assessment

Yes. Source inspection of PR head shows a multi-agent direct-store path where the default/runtime target is labeled and therefore skips `confirmAutoFix`; current main and doctor docs show the default interactive OAuth refresh prompt should still be preserved.

## Solution Assessment

No. Agent-labeled diagnostics are the right direction, but refresh eligibility should be based on the target kind, such as non-default direct agent stores, rather than whether the display title has an agent-id suffix.
