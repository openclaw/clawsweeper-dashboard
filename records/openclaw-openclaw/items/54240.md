## Summary

Keep open. Current main still has the reported Control UI ordering gap: the model picker optimistically patches session state, but the composer/send path and run controls do not expose or await a pending model-change operation before dispatching `chat.send`.

## What This Changes

The issue requests that Control UI model dropdown changes be committed or awaited before the next `chat.send` so an immediate message uses the selected model.

## Best Possible Solution

Add a Control UI-side per-session model-switch barrier: store the pending `sessions.patch`, disable the model picker and send/queue controls while it is pending, await it before dispatching sends, roll back optimistic selection on patch failure, add focused regression coverage, and keep the Gateway `chat.send` protocol unchanged.

## Reproduction Assessment

Yes. A high-confidence regression test can defer the model picker’s `sessions.patch`, trigger a send before resolving it, and assert that `chat.send` is not called until the patch succeeds; current main lacks that barrier in source.

## Solution Assessment

Yes. The UI-side barrier is the narrowest maintainable fix because the picker already owns session persistence and `chat.send` intentionally has no model override field; a protocol change would be broader than needed for this race.

## Repair Work Prompt

Fix the Control UI model dropdown race reported at https://github.com/openclaw/openclaw/issues/54240. When a user changes the chat model and immediately sends, `chat.send` must wait until the model `sessions.patch` for that session has either succeeded or failed. Keep the fix in the Control UI unless a maintainer asks for a broader protocol change: add per-session pending model-patch state/promise, disable the model picker and send/queue controls while pending, await the pending operation before normal sends, and roll back optimistic selection plus avoid sending when the patch fails. Preserve existing provider-qualified model behavior from prior fixes; do not add a `model` field to `chat.send`. Add focused tests that defer `sessions.patch`, attempt a send, prove `chat.send` waits, and cover failure rollback/no-send. Add a user-facing `CHANGELOG.md` fix entry thanking `@betamod`.
