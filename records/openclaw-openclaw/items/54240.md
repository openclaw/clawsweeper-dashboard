## Summary

Keep open. Current main at 068b33de871718ad13217eb9c10826de8b4b2c2a still has the Control UI ordering gap: model selection patches the session optimistically, but the pending patch is not exposed to shared state, send controls, or the send path before `chat.send` can start.

## What This Changes

The issue asks for the Control UI chat model dropdown change to persist or fail before the next chat send can run with the selected model.

## Best Possible Solution

Add a Control UI-side per-session model-switch barrier that records the pending `sessions.patch`, disables model selection and send/queue controls while pending, awaits the patch before any affected send starts, rolls back optimistic selection and preserves the draft if the patch fails, and leaves the Gateway `chat.send` protocol unchanged.

## Reproduction Assessment

Yes. A focused regression test can defer the model picker’s `sessions.patch`, trigger `handleSendChat` before resolving it, and assert that `chat.send` is not called until the patch succeeds; current main has no state or send-path hook that could enforce that ordering.

## Solution Assessment

Yes. The proposed UI-side barrier is the narrowest maintainable fix because the dropdown already owns session persistence and the current protocol has no `chat.send` model override field; changing the wire contract would be broader than this race requires.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/54240. The observable bug is that changing the Control UI chat model dropdown and immediately sending can start `chat.send` before the dropdown's `sessions.patch` persists, so the run can use the prior/default session model. Implement this in the Control UI only: track a per-session pending model-patch operation/status, disable the model picker and send/queue controls while it is pending, make `handleSendChat` and queued sends await the pending patch before any affected `chat.send`, and if the patch fails, roll back the optimistic selection, surface the existing error style, preserve/restore draft and attachments, and do not send. Add focused regression tests that defer `sessions.patch` and verify `chat.send` waits, plus a failure-path test. Add a user-facing `CHANGELOG.md` fix entry with `Thanks @betamod`.
