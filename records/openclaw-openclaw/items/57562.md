## Summary

Keep this open. Current main still rejects unknown Feishu account-level keys through the strict Feishu account schema, and the doctor unknown-key repair path still does not cover plugin channel-schema additional-property failures; related reports now identify likely legacy Feishu config keys, but this issue should be consolidated rather than closed as fixed.

## What This Changes

Fix an upgrade config migration failure where `channels.feishu.accounts.main` is rejected for additional properties and `openclaw doctor --fix` does not repair the config.

## Best Possible Solution

Keep this issue open while consolidating it with #57567 and #65177. The desired end state is a targeted Feishu-owned doctor/config migration for confirmed legacy account fields such as `botName` to `name`, with focused regression tests, while keeping strict validation for truly unknown keys.

## Reproduction Assessment

Partial. This issue alone lacks the offending property, but related #57567 and #65177 provide a high-confidence reproduction using `channels.feishu.accounts.main.botName`, which current main still rejects through the strict Feishu account schema.

## Solution Assessment

No current-main fix exists yet. The narrow maintainable solution is a targeted doctor/Feishu compatibility migration for confirmed legacy keys, not a broad schema relaxation that would hide misspelled account config.
