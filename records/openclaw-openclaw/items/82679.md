## Summary

Keep open: the PR targets four source-proven bugs and mostly changes the right owner-local paths, but the new integer parser still accepts fractional numeric values by truncating them. The PR also needs after-fix real behavior proof beyond the supplied Node probe and Vitest output before merge.

## What This Changes

Adds limit guards for diagnostic/session usage helpers, adds voice-call CLI numeric option validation, and adds regression tests for those paths.

## Best Possible Solution

Reject non-integer numeric CLI values before truncation, keep the focused regression coverage, add real after-fix runtime proof, and land once required checks are green.

## Reproduction Assessment

Yes. Current main has direct source paths for the reported zero-limit and NaN cases, and the PR finding is source-reproducible with a fractional value such as `--port 3334.9`.

## Solution Assessment

No. The validation direction is right, but the helper should reject non-integer values instead of silently truncating them, and the proof should show real after-fix behavior outside tests.
