## Summary

Keep open. Current `main` still has the reported Telegram helper gap: default markdown-mode sends and media follow-up text can pass one oversized `sendMessage` payload, while the existing chunk planner is only reached through the HTML-mode branch.

## What This Changes

Ensure Telegram default markdown-mode outbound text over Telegram's Bot API limit is chunked instead of sent as a single message.

## Best Possible Solution

Fix the Telegram plugin helper so all text sends, including default markdown text and media follow-up text, plan safe chunks before Bot API delivery while preserving markdown rendering, plain-text fallback, threading, link preview, and reply-markup placement.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to call `sendMessageTelegram` with default options and >4096 characters using a mocked `api.sendMessage`; current `main` reaches the markdown branch and constructs one oversized send.

## Solution Assessment

Yes, the Telegram plugin helper is the right boundary because direct helper callers and registered payload delivery can bypass core pre-chunking. The safest repair is to reuse the existing planner for markdown/default text after rendering to Telegram HTML, not to move Telegram-specific behavior into core.
