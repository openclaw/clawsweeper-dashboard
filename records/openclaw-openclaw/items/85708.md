## Summary

This PR is a focused gateway restart fix with source-backed reproduction, strong terminal proof, and no discrete blocking code finding. It should stay open for maintainer confirmation of the intentional unbounded embedded-run wait behavior before merge.

## What This Changes

The PR removes the hidden 30s active embedded-run restart cap, passes the configured restart drain budget to embedded runs, and carries forced SIGUSR1 deferral-timeout intent through restart handling with focused tests.

## Best Possible Solution

Land the focused fix if maintainers accept the documented restart budget as the single source of truth for embedded runs; otherwise revise toward an explicit, documented embedded-run watchdog with tests.

## Reproduction Assessment

Yes. Source inspection on current main shows embedded runs are capped at 30s even when the documented restart drain budget is longer or unbounded; I did not run local tests because this review is read-only.

## Solution Assessment

Yes, if maintainers accept the documented drain-budget contract. A separate embedded-run watchdog is a valid alternative policy choice, but it should be explicit and documented rather than hidden behind the current cap.
