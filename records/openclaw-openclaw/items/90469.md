## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Wire the startup plugin before enabling it:** `extensions/openclaw-governance/src/index.ts:65-70`
  - body: The manifest loads this plugin on Gateway startup and the UI hints say it records AIBOM, scans DLP, and persists cost entries, but `register()` only discards `api` and registers no hooks, subscriptions, services, or middleware. That means an operator can enable the bundled plugin and still get no governance rows from real turns; either wire the supported plugin hooks now, or make this startup-lazy/internal and remove the user-facing runtime claims until that follow-up lands.
  - confidence: 0.9
- **[P1] Stop persisting raw DLP matches by default:** `extensions/openclaw-governance/src/store/sqlite.ts:107`
  - body: The DLP findings table stores `matched_snippet` as raw text, so SSNs, card numbers, emails, IPs, and similar matches can be retained in the governance database. For a DLP plugin this should default to redacted or hash-only evidence, with any raw retention behind an explicit operator setting and matching tests.
  - confidence: 0.92
- **[P2] Preserve cache prices in config overrides:** `extensions/openclaw-governance/src/config.ts:22`
  - body: `CostLedger` now supports `cacheReadUsd` and `cacheWriteUsd`, but `GovernancePluginConfig.pricesPerMillion`, the normalizer, and the manifest schema only accept `inputUsd` and `outputUsd`. Since custom prices replace the default model row, overriding a model's input/output price silently makes cache reads/writes cost `$0`; include cache fields in the config contract or merge overrides with defaults.
  - confidence: 0.86
