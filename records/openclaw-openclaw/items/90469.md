## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Wire the startup plugin before enabling it:** `extensions/openclaw-governance/src/index.ts:65-70`
  - body: `openclaw.plugin.json` marks this plugin for startup activation, but `register(api)` only voids the API and explicitly defers all INFERENCE_END/OUTBOUND_PAYLOAD/SKILL_INVOKE wiring. An operator could enable the plugin and get no AIBOM, DLP, or cost-ledger behavior from real turns, so this should either register the real runtime hooks or not ship as an enabled governance plugin yet.
  - confidence: 0.95
- **[P1] Stop storing raw DLP matches by default:** `extensions/openclaw-governance/src/store/sqlite.ts:36`
  - body: The DLP row shape includes `matchedSnippet`, and the SQLite insert persists that text verbatim. With the default `log` action, future runtime wiring would turn the governance database into an at-rest collection of SSNs, card numbers, emails, and similar sensitive data; store redacted/hash-only evidence by default or require an explicit raw-retention opt-in with tests.
  - confidence: 0.92
- **[P2] Expose cache price overrides in plugin config:** `extensions/openclaw-governance/openclaw.plugin.json:81-84`
  - body: `CostLedger` now prices cache-read and cache-write tokens, but the manifest schema only allows `inputUsd` and `outputUsd` under `pricesPerMillion` and rejects other keys. Operators cannot override cache pricing despite the ledger using those fields, so the schema and normalizer need to accept the same price shape as the ledger.
  - confidence: 0.88
