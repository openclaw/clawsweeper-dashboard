## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Wire the startup plugin before enabling it:** `extensions/openclaw-governance/src/index.ts:65-70`
  - body: The manifest loads this plugin on startup and the UI hints say it records AIBOM, scans DLP, and persists costs, but `register()` only discards `api` and registers no hooks, services, subscriptions, or middleware. Operators can enable the bundled plugin and still get no governance rows from real turns; either wire the supported hooks now or remove the startup/user-facing runtime claims until that follow-up lands.
  - confidence: 0.91
- **[P1] Stop persisting raw DLP matches by default:** `extensions/openclaw-governance/src/store/sqlite.ts:107`
  - body: The DLP findings table stores `matched_snippet` as raw text, so SSNs, card numbers, emails, IPs, and similar matches can be retained in the governance database. A DLP plugin should default to redacted or hash-only evidence, with any raw retention behind an explicit operator setting and focused tests.
  - confidence: 0.92
- **[P2] Preserve cache-price overrides in config:** `extensions/openclaw-governance/src/config.ts:22`
  - body: `CostLedger` now prices `cacheReadUsd` and `cacheWriteUsd`, but the plugin config type, normalizer, and manifest schema only accept `inputUsd` and `outputUsd`. Since custom prices replace the default model row, overriding input/output pricing silently makes cache reads and writes cost zero; include cache fields in the config contract or merge overrides with defaults.
  - confidence: 0.86
