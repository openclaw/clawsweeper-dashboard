Full review comments:

- **[P1] Wire runtime hooks before startup activation:** `extensions/openclaw-governance/src/index.ts:65-70`
  - body: The manifest and UI hints advertise per-turn AIBOM, DLP, and cost-ledger behavior, but `register()` is a no-op and explicitly defers the actual event wiring. An enabled startup plugin would load successfully while doing none of the promised governance work from real turns.
  - confidence: 0.95
- **[P1] Stop storing raw DLP matches by default:** `extensions/openclaw-governance/src/store/sqlite.ts:107`
  - body: The DLP schema stores `matched_snippet` as plaintext, and the insert path writes the scanner's raw match text. Since findings include SSNs, credit cards, emails, IPs, and similar data, this turns the governance database into an at-rest sensitive-data collection unless snippets are redacted, hashed, or explicitly opted into.
  - confidence: 0.92
- **[P2] Accept cache pricing override fields:** `extensions/openclaw-governance/src/config.ts:22`
  - body: The ledger now prices `cacheReadUsd` and `cacheWriteUsd`, but the plugin config type, normalizer, and schema still accept only input/output prices. Operators cannot override cache pricing for models with non-default cache rates, so custom pricing silently falls back to zero cache rates.
  - confidence: 0.86
