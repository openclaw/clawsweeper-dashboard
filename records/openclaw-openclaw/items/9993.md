## Summary

Keep open: current main and v2026.6.5 include adjacent config hardening, but they still do not expose the requested synchronous plugin/API-level pre-apply veto before config writes persist.

## What This Changes

The issue requests a synchronous `config:pre-apply` hook that can inspect current and pending config and block `config.patch` or `config.apply` before disk writes.

## Best Possible Solution

Design one documented pre-persistence config mutation contract shared by `config.set`, `config.patch`, `config.apply`, and config mutation helpers, with typed rejection and tests proving rejected writes leave disk and runtime config unchanged.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.5 lack a config pre-apply hook in the plugin hook catalog, and config write paths commit through shared helpers without a plugin veto. I did not run a live crash reproduction because this read-only review can verify the missing API directly from source and docs.

## Solution Assessment

Partly: a config pre-apply hook is the right class of seam, but the best implementation is a shared documented config mutation contract, not an RPC-only wrapper. Maintainers still need to settle ordering, failure, redaction, actor metadata, and recovery semantics before implementation.
