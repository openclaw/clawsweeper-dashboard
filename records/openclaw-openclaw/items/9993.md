## Summary

Keep open: current main has adjacent config validation, SecretRef preflight, base-hash guarded writes, and an agent-tool allowlist, but it still does not expose the requested synchronous plugin/API veto before config mutations persist.

## What This Changes

Request a synchronous plugin/API pre-apply hook that can inspect and reject config.set, config.patch, and config.apply changes before they are written.

## Best Possible Solution

Define one documented shared pre-persistence config mutation contract that covers every config writer, exposes typed current/pending/actor/change metadata, and guarantees rejected writes leave disk and runtime config unchanged.

## Reproduction Assessment

Yes, at source level: current main lacks a config pre-apply hook in the plugin hook catalog, and config.set/config.patch/config.apply commit through shared helpers without a plugin veto.

## Solution Assessment

Unclear until maintainers settle the API and security semantics. A config pre-apply hook is the right class of seam, but the maintainable shape is a shared pre-persistence config mutation contract rather than wrapping only Gateway RPC methods.

## Security Review

Status: needs_attention

Summary: The issue is security-sensitive because the proposed hook would run at the config mutation boundary and expose or decide sensitive config changes.
