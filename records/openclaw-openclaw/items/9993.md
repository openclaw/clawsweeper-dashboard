## Summary

Keep open: current main still has validation, SecretRef preflight, base-hash checks, and agent-facing config write guardrails, but it does not expose the requested mandatory synchronous plugin/API veto before config persistence.

## What This Changes

Requested a synchronous config pre-apply hook that can validate and reject gateway config mutations before they are written.

## Best Possible Solution

Add a documented additive config-write interception contract in the shared mutation flow after pending config computation and built-in validation/preflight but before persistence, with typed reject semantics and regression coverage proving rejected writes leave disk and active runtime config unchanged.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/API gap. Source inspection gives high confidence that current main still lacks a mandatory plugin-level config pre-apply veto despite existing validation and model-facing guardrails.

## Solution Assessment

Unclear until the config validation/mutation security direction is settled. The maintainable path is a shared config mutation seam covering RPC and runtime/plugin write helpers, not a hook attached only to one gateway method.
