## Summary

Keep open: the external config-guard workaround is useful, but current main still lacks a mandatory synchronous plugin/API veto point before config mutations are persisted.

## What This Changes

Request a synchronous config pre-apply hook that lets validators inspect and reject pending config changes before disk writes.

## Best Possible Solution

Define one documented shared pre-persistence config mutation contract that covers all config writers with typed rejection and tests proving rejected writes leave disk and runtime config unchanged.

## Reproduction Assessment

Not applicable as a runtime bug: this is a new plugin/API interception request. Source inspection gives high confidence that current main has no mandatory pre-persistence veto; config.patch and config.apply validate and preflight, then write through commitGatewayConfigWrite.

## Solution Assessment

Unclear until maintainers settle API and security semantics. A shared pre-persistence config mutation contract is the maintainable shape; wrapping only config.patch and config.apply would be too narrow.
