## Summary

Keep open: current main still lacks the requested mandatory synchronous plugin/API veto before config persistence, and the issue discussion plus related open preflight work treat this as a security-sensitive API design gap rather than completed cleanup.

## What This Changes

Request a config pre-apply hook that lets external validators inspect and reject pending config changes before config writes are persisted.

## Best Possible Solution

Define one documented shared pre-persistence config mutation contract that covers config.set, config.patch, config.apply, and shared mutation helpers with typed rejection and regression tests proving rejected writes leave disk and runtime config unchanged.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new plugin/API interception request. Source inspection gives high confidence that current main has no mandatory pre-persistence veto; config.patch/config.apply validate and preflight, then write through commitGatewayConfigWrite.

## Solution Assessment

Unclear until maintainers settle API and security semantics. A shared pre-persistence config mutation contract is the maintainable shape; wrapping only config.patch and config.apply would be too narrow.
