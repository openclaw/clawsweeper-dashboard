## Summary

Keep this open. Current main has adjacent config validation, SecretRef preflight, base-hash guarded writes, runtime preflight, and agent-tool allowlisting, but it still lacks the requested synchronous plugin/API-level config pre-apply veto before config mutations are persisted.

## What This Changes

Requested addition of a synchronous config pre-apply hook that can validate and reject gateway config mutations before persistence.

## Best Possible Solution

Keep this open for maintainer design of a narrow, documented config-write interception seam that runs after built-in validation/preflight and before persistence across config.set/config.patch/config.apply and shared mutation helpers.

## Reproduction Assessment

Yes for the source-level API gap: current hook names and config write paths can be inspected on main, and they do not expose a config pre-apply hook. I did not run a crash repro because the request is for a new interception seam rather than a single deterministic crash bug.

## Solution Assessment

Unclear until maintainer design review. A synchronous config-write hook is the right class of solution, but the durable design needs explicit decisions on hook ordering, failure semantics, recursion avoidance, and shared-helper coverage.
