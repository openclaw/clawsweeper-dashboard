## Summary

Keep open: current main has config schema validation, SecretRef write preflight, base-hash guards, and agent-tool allowlists, but it still lacks the requested mandatory synchronous plugin/API veto before config changes persist.

## What This Changes

Request a `config:pre-apply` hook that can synchronously inspect and reject pending config writes before `config.patch` or `config.apply` persists them.

## Best Possible Solution

Define one documented shared pre-persistence config mutation contract that covers all config writers with typed rejection and regression tests proving rejected writes leave disk and runtime config unchanged.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source inspection as an API gap rather than a runtime bug: current hook catalogs and config write paths show no mandatory pre-persistence plugin veto before config writes are committed.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers settle API and security semantics. A shared pre-persistence config mutation contract is the maintainable shape; wrapping only `config.patch` and `config.apply` would be too narrow.
