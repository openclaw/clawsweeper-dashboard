## Summary

Keep open: current main has adjacent config validation, SecretRef preflight, base-hash guarded writes, and agent-tool guardrails, but it still lacks the requested mandatory synchronous plugin/API veto before config persistence. Prior review comments and related open preflight/API discussions support treating this as a real design gap, not as completed core cleanup.

## What This Changes

Request a synchronous `config:pre-apply` hook that can inspect current and pending config for Gateway config writes and reject unsafe changes before disk persistence.

## Best Possible Solution

Define a documented shared pre-persistence config mutation contract after pending config computation and built-in validation/preflight, covering config.set, config.patch, config.apply, and shared mutation helpers with typed rejection and regression tests proving rejected writes leave disk and runtime config unchanged.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new plugin/API interception request. Source inspection gives high confidence that the requested mandatory pre-persistence veto is absent on current main.

## Solution Assessment

Unclear until maintainers settle API and security semantics. A shared pre-persistence config mutation contract is the maintainable shape; wrapping only config.patch and config.apply would be too narrow.

Status: needs_attention
