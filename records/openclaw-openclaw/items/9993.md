## Summary

Keep this open. Current main and v2026.6.10 have adjacent config validation, SecretRef preflight, base-hash guarded writes, agent-tool write guards, and runtime snapshot preflight, but they still do not expose the requested synchronous plugin/API-level config pre-apply veto before config mutations are persisted.

## What This Changes

The issue requests a synchronous config pre-apply hook that can validate and reject pending config writes before config.set, config.patch, config.apply, or shared config mutation helpers persist changes.

## Best Possible Solution

Keep this open for maintainer design of a narrow, documented config-write interception seam that runs after built-in validation and SecretRef/runtime preflight, before persistence, and covers config.set/config.patch/config.apply plus shared mutation helpers.

## Reproduction Assessment

Yes at source level: current main has validation and SecretRef preflight before config writes, but no plugin hook with current config, pending config, changed paths, actor metadata, and a typed reject result before persistence. I did not run a live crashing config repro because the missing API is directly visible in the source path.

## Solution Assessment

Unclear until maintainer design review. A synchronous config-write hook is the right class of solution, but the durable design needs explicit decisions on hook ordering, failure semantics, recursion avoidance, and shared-helper coverage.
