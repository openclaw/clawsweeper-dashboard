## Summary

Keep open: current main has built-in validation, SecretRef preflight, base-hash checks, and agent-facing config write guardrails, but it still lacks the requested mandatory synchronous plugin/API veto before config persistence.

## What This Changes

Request to add a synchronous config:pre-apply hook that can inspect and reject pending config.set, config.patch, or config.apply changes before they are written.

## Best Possible Solution

Add a documented additive config-write interception contract in the shared mutation flow after pending config computation and built-in validation/preflight but before persistence, with typed reject semantics and regression coverage proving rejected writes leave disk and active runtime config unchanged.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/API gap. Source inspection gives high confidence that current main still lacks a mandatory plugin-level config pre-apply veto despite existing validation and guardrails.

## Solution Assessment

Unclear until maintainer API and security semantics are settled. A pre-persistence interception point is the right shape, but it should be a shared config mutation contract covering RPC and runtime/plugin write helpers, not a hook attached only to one gateway method.
