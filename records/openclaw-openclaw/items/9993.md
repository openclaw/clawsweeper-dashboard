## Summary

Keep open. Current main has adjacent config validation, SecretRef write preflight, base-hash guards, destructive-array guards, and agent config-write allowlists, but it still lacks the requested mandatory synchronous plugin/API-level pre-apply veto before config persistence.

## What This Changes

The issue requests a synchronous config pre-apply hook that can validate and reject gateway config writes before persistence.

## Best Possible Solution

Define one maintainer-approved synchronous config-write interception contract at the shared config mutation boundary, with typed reject results, security semantics, docs, and tests proving rejected writes leave disk and runtime state unchanged.

## Reproduction Assessment

Yes at source level: current main exposes config.set/config.patch/config.apply and shared config mutation helpers with built-in validation/preflight, but no mandatory plugin/API-level pre-persistence veto. I did not rerun the historical provider-model incident because the remaining issue is the missing API contract.

## Solution Assessment

Yes in direction, but the exact API needs maintainer design. A shared write-boundary hook is more maintainable than wrapping only agent tools or Gateway RPC handlers, while existing SecretRef preflight and agent allowlists solve only adjacent cases.
