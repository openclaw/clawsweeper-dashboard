## Summary

Keep open: current main and the latest shipped release have adjacent validation, SecretRef preflight, base-hash checks, and config-tool controls, but still lack the requested mandatory synchronous plugin/API veto before config mutations persist.

## What This Changes

The issue requests a synchronous config pre-apply hook that lets validators inspect and reject pending config mutations before persistence.

## Best Possible Solution

Define one documented shared pre-persistence config mutation contract that covers all config writers, exposes typed current/pending/actor/change metadata, and guarantees rejected writes leave disk and runtime config unchanged.

## Reproduction Assessment

Yes at source level, not as a runtime bug: current main lacks a config pre-apply hook in PluginHookName, and config.set/config.patch/config.apply commit through shared helpers without a plugin veto.

## Solution Assessment

Unclear until maintainers settle the API and security semantics. A config pre-apply hook is the right class of seam, but the maintainable shape is a shared pre-persistence config mutation contract rather than wrapping only Gateway RPC methods.

## Security Review

Status: needs_attention

Summary: The requested hook is security-sensitive because it would run at the config mutation boundary and expose current and pending config data to plugins.

Concerns:

- **[medium] Define fail-closed hook semantics:** `src/gateway/server-methods/config-write-flow.ts:209`
  - body: A rejectable pre-persistence hook must specify handler failure, timeout, plugin disablement, recursion, reload/restart follow-up, and emergency recovery behavior so config writes cannot unexpectedly fail open or deadlock the Gateway.
  - confidence: 0.9
- **[medium] Cover all config writers consistently:** `src/config/mutate.ts:363`
  - body: Adding the hook only to RPC handlers would leave shared helpers such as replaceConfigFile and mutateConfigFile as bypass paths, making the validation boundary inconsistent.
  - confidence: 0.9
- **[medium] Define config redaction and actor metadata:** `src/gateway/server-methods/config.ts:465`
  - body: The hook would expose current and pending config near SecretRef and auth surfaces, so the contract needs explicit redaction, actor metadata, and audit semantics before plugins can safely inspect or reject writes.
  - confidence: 0.84
