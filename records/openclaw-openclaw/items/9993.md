## Summary

Keep open: current main has adjacent validation, SecretRef preflight, base-hash guarded writes, no-op handling, and agent-tool allowlist guardrails, but it still lacks the requested mandatory synchronous plugin/API veto before config persistence.

## What This Changes

Request a synchronous config pre-apply hook that lets validators inspect and reject pending gateway config.patch/config.apply writes before persistence.

## Best Possible Solution

Define a documented shared pre-persistence config mutation contract after pending config computation and built-in validation/preflight, covering config.set, config.patch, config.apply, and shared mutation helpers with typed rejection and regression tests proving rejected writes leave disk and runtime config unchanged.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new plugin/API interception request. Source inspection gives high confidence that the requested mandatory pre-persistence veto is absent on current main.

## Solution Assessment

Unclear until maintainers settle the API and security semantics. A shared pre-persistence config mutation contract is the right shape; wrapping only config.patch and config.apply would be too narrow.

Status: needs_attention
