## Summary

Keep open: current main has config schema validation, SecretRef preflight, base-hash guarded writes, and agent-facing config write guardrails, but it still lacks the requested mandatory synchronous plugin/API veto before config persistence.

## What This Changes

Add a synchronous config pre-apply hook/API that can inspect and reject pending config writes before `config.set`, `config.patch`, and `config.apply` persist them.

## Best Possible Solution

Add a documented shared config-write interception contract after pending config computation and built-in validation/preflight but before persistence, with typed rejection and regression tests proving rejected writes leave disk and active runtime config unchanged.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/API gap. Source inspection gives high confidence that current main still lacks a mandatory plugin-level config pre-apply veto.

## Solution Assessment

Unclear until maintainer API and security semantics are settled. A pre-persistence interception point is the right shape, but it should be a shared config mutation contract rather than a hook attached only to one gateway method.
