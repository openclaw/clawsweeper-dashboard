## Summary

Keep open. Current main and v2026.6.10 have adjacent validation, SecretRef preflight, base-hash guarded config writes, and agent-tool allowlists, but they still do not provide a mandatory synchronous plugin/API-level pre-persistence veto for config changes.

## What This Changes

The issue requests a synchronous config pre-apply hook that lets validators inspect and reject pending gateway config changes before they are persisted.

## Best Possible Solution

Define one maintainer-approved synchronous config-write interception contract at the shared config mutation boundary, with typed reject results, security semantics, docs, and tests proving rejected writes leave disk and runtime state unchanged.

## Reproduction Assessment

Yes, at source level: current config.set/config.patch/config.apply compute and validate pending config, then commit without a plugin-visible synchronous veto point. I did not rerun the historical bad-model incident, but the missing hook surface is directly inspectable on current main.

## Solution Assessment

Unclear on exact API shape, but the direction is valid: wrapping only the gateway tool would be too narrow, while a shared write-boundary interception contract is the maintainable solution if maintainers approve the public plugin/security semantics.
