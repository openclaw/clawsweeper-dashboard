## Summary

Keep open: current main has stronger built-in config validation and agent-tool guardrails, but it still lacks the requested synchronous plugin/API-level pre-persistence veto for config writes.

## What This Changes

Request a synchronous `config:pre-apply` hook that receives current and pending config and can reject config changes before they are written.

## Best Possible Solution

Add a documented additive config-write interception contract in the shared mutation flow, after pending config computation and built-in validation/preflight but before persistence, with typed reject semantics and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/API gap. Source inspection gives high confidence that current main still has no mandatory plugin-level config pre-apply veto despite existing validation and guardrails.

## Solution Assessment

Unclear until the RFC direction is settled. The maintainable path is a shared config mutation seam covering RPC and runtime/plugin write helpers, not a narrow hook attached only to one gateway method.
