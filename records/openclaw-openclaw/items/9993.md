## Summary

Keep open. Current main has built-in config validation, SecretRef preflight, base-hash guarded writes, and an agent-facing config write allowlist, but it still lacks the requested synchronous plugin/API-level config pre-apply veto before config mutations are persisted; the existing issue discussion also links this to the broader config validation/mutation design work.

## What This Changes

The issue requests a synchronous config pre-apply hook that can inspect current and pending config and reject gateway config mutations before persistence.

## Best Possible Solution

Design a documented synchronous config-write interception API invoked by config.set, config.patch, config.apply, and shared mutation helpers after pending config computation plus built-in validation/preflight but before persistence, with typed rejection, actor/baseHash/changed-path metadata, sensitive-data policy, and regression tests proving rejection leaves disk and active runtime config unchanged.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new plugin/API feature request. Source inspection gives a high-confidence absence check because the hook catalog lacks a config mutation hook and config write paths persist after validation/preflight without a plugin veto.

## Solution Assessment

Unclear as an implementation until maintainers settle the API contract. The requested direction is plausible, but the best solution needs product and security design around hook ordering, failure policy, recursion, actor identity, and sensitive config exposure.
