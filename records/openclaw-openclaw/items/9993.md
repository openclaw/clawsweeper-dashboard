## Summary

Keep open. Current main and v2026.6.8 have adjacent safeguards for config writes, but they still do not provide the requested synchronous plugin/API-level config pre-apply veto before persistence.

## What This Changes

The issue requests a synchronous config pre-apply hook that can validate and reject pending gateway config writes before they are persisted.

## Best Possible Solution

Define one maintainer-approved synchronous config-write interception contract at the shared config mutation boundary, with typed reject results, security semantics, docs, and tests proving rejected writes leave disk and runtime state unchanged.

## Reproduction Assessment

Yes, at source level: current main exposes config mutation RPCs and shared mutation helpers with validation/preflight, but no mandatory plugin/API-level pre-persistence veto. I did not replay the historical model incident because the remaining issue is the missing API contract.

## Solution Assessment

Yes in direction, but the exact API needs maintainer and security design. A shared write-boundary hook is more maintainable than wrapping only agent tools or Gateway RPC handlers, while existing SecretRef preflight and agent allowlists solve only adjacent cases.
