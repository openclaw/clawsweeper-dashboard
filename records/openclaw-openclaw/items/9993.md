## Summary

Keep open: current main has adjacent config validation, SecretRef write preflight, base-hash guarded writes, and agent-tool allowlisting, but it still lacks the requested mandatory synchronous plugin/API veto before config changes are persisted.

## What This Changes

The issue requests a synchronous `config:pre-apply` hook that can inspect and reject pending config writes before `config.set`, `config.patch`, `config.apply`, or shared config mutation helpers persist them.

## Best Possible Solution

Define one documented shared pre-persistence config mutation contract that covers all config writers, exposes typed current/pending/actor/change metadata, and guarantees rejected writes leave disk and runtime config unchanged.

## Reproduction Assessment

Not applicable as a runtime bug: this is a new plugin/API interception request. Source inspection gives high confidence that current main has no mandatory pre-persistence veto; config writes validate and preflight, then write through shared config mutation helpers.

## Solution Assessment

Unclear until maintainers settle API and security semantics. A shared pre-persistence config mutation contract is the maintainable shape; wrapping only the gateway RPC methods would be too narrow.

## Security Review

Status: needs_attention

Summary: The requested hook is security-sensitive because it would run on the config mutation trust boundary and expose current/pending config data to plugins.

Concerns:

- **[medium] Define fail-closed hook semantics:** `src/gateway/server-methods/config-write-flow.ts:209`
  - body: A rejectable pre-persistence hook must specify handler failure, timeout, plugin disablement, recursion, reload/restart follow-up, and emergency recovery behavior so config writes cannot unexpectedly fail open or deadlock the Gateway.
  - confidence: 0.9
- **[medium] Cover all config writers consistently:** `src/config/mutate.ts:363`
  - body: Adding the hook only to RPC handlers would leave shared helpers such as `replaceConfigFile` and `mutateConfigFile` as bypass paths, making the validation boundary inconsistent.
  - confidence: 0.9
- **[medium] Define config redaction and actor metadata:** `src/gateway/server-methods/config.ts:437`
  - body: The hook would expose current and pending config near SecretRef and auth surfaces, so the contract needs explicit redaction, actor metadata, and audit semantics before plugins can safely inspect or reject writes.
  - confidence: 0.84
