## Summary

Current main still lacks the requested public, mandatory plugin veto for configuration writes. It now has stronger built-in validation and an internal runtime pre-commit check, but that internal callback is not in the Plugin SDK hook catalog, receives only the pending source config, and does not provide the requested current config, action, patch, actor, or typed rejection contract; the issue should remain open for the maintainer-approved config mutation API design.

## What This Changes

Requests a synchronous plugin-level pre-apply hook that can veto configuration mutations before persistence.

## Best Possible Solution

Define one documented Plugin SDK interception contract at the canonical validated pre-commit boundary, covering all supported config writers and providing redacted current/pending config, mutation kind, changed paths, base hash, actor metadata, and a typed reject result with tested no-write/no-runtime-change guarantees.

## Reproduction Assessment

Yes, at source level: current main's public hook registry has no configuration mutation event, while config writes flow through shared persistence paths that are not all tool calls. This read-only review did not intentionally write a bad config or crash a gateway.

## Solution Assessment

Yes in direction, but not as a gateway-tool-only event. The maintainable solution is a single public Plugin SDK contract layered on the canonical validated pre-commit boundary so every supported writer observes the same policy.

## Risks / Open Questions

- A synchronous veto lets plugin code block all configuration changes, so maintainers must define which plugins may register it, timeout and exception behavior, fail-open versus fail-closed policy, ordering with schema and SecretRef preflight, recursion prevention, and recovery when a validator itself is broken.
- Adding the hook only to gateway config.patch/config.apply would leave shared config writers outside the policy and create a misleading security boundary.
- Exposing current configuration and raw mutation payloads to plugins may reveal resolved or sensitive values unless the contract preserves SecretRef and redaction boundaries.
