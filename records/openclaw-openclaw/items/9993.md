## Summary

Keep open. Current main and v2026.6.6 have built-in config validation, active SecretRef preflight, base-hash guarded writes, and an agent-facing config mutation allowlist, but they still do not expose the requested mandatory synchronous plugin/API veto before config mutations persist.

## What This Changes

Requests a synchronous config pre-apply plugin/API hook that can inspect and reject pending config writes before config.set, config.patch, or config.apply persist changes.

## Best Possible Solution

Add one documented typed interception contract at the shared config-write owner boundary after pending-config computation and built-in validation/preflight, but before persistence, with tests proving rejection leaves disk and active runtime config unchanged.

## Reproduction Assessment

Yes at source level. Current main exposes config mutation RPCs and shared SDK mutation helpers with validation/preflight but no mandatory plugin-level pre-persistence veto, so an external validator remains bypassable.

## Solution Assessment

Yes in direction, but the exact API requires maintainer design. A shared config-write owner-boundary hook is more maintainable than wrapping only the agent gateway tool or individual RPC methods.
