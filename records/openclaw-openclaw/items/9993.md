## Summary

Keep open: current main has config validation, SecretRef preflight, base-hash guarded writes, and an agent-tool allowlist, but it still lacks the requested mandatory synchronous plugin/API veto before config changes persist.

## What This Changes

Request a synchronous config pre-apply hook that lets validators inspect current and pending config and reject config.patch/config.apply before persistence.

## Best Possible Solution

Define one documented shared pre-persistence config mutation contract that covers all config writers with typed rejection and tests proving rejected writes leave disk and runtime config unchanged.

## Reproduction Assessment

Not applicable as a runtime bug: this is a new plugin/API interception request. Source inspection gives high confidence that current main has no mandatory pre-persistence veto; config.patch and config.apply validate and preflight, then write through shared config mutation helpers.

## Solution Assessment

Unclear until maintainers settle API and security semantics. A shared pre-persistence config mutation contract is the maintainable shape; wrapping only config.patch and config.apply would be too narrow.

## Security Review

Status: needs_attention

Summary: The requested hook sits on the config mutation trust boundary and needs explicit failure, timeout, recursion, redaction, and all-writer semantics before implementation.

Concerns:

- **[medium] Define fail-closed hook semantics:** `src/gateway/server-methods/config-write-flow.ts:209`
  - body: A rejectable pre-persistence hook must specify handler failure, timeout, plugin disablement, recursion, reload/restart follow-up, and emergency recovery behavior so config writes cannot unexpectedly fail open or deadlock the Gateway.
  - confidence: 0.9
- **[medium] Cover all config writers consistently:** `src/config/mutate.ts:363`
  - body: Adding the hook only to RPC handlers would leave shared helpers such as replaceConfigFile and mutateConfigFile as bypass paths, making the validation boundary inconsistent.
  - confidence: 0.9

  - body: The hook would expose current and pending config near SecretRef and auth surfaces, so the contract needs explicit redaction, actor metadata, and audit semantics before plugins can safely inspect or reject writes.
  - confidence: 0.84
