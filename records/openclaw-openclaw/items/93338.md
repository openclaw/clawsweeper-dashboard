## What This Changes

## Best Possible Solution

Preserve the shared default-account contract and fix the Matrix named-account case at the Matrix routing or account-selection boundary with focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main passes Matrix's effective account id into `resolveAgentRoute()`, while omitted `match.accountId` bindings are indexed under `DEFAULT_ACCOUNT_ID`, so the linked named-account Matrix room-binding shape can miss the binding.

## Solution Assessment

No. The PR fixes the symptom by changing the shared account-binding contract; the narrower maintainable fix is Matrix-scoped handling that keeps explicit `accountId: "*"` as the wildcard path.
