## What This Changes

The branch adds a Matrix inbound-routing fallback that re-resolves default-account bindings for named Matrix accounts, expands Matrix route tests, keeps the shared omitted-account routing test default-scoped, and adds two ad hoc Telegram schema-check scripts.

## Best Possible Solution

Keep the shared omitted-account contract, restrict Matrix fallback to the effective default or only Matrix account, add a negative multi-account regression test, remove the unrelated scripts, and update the PR body with current-head proof.

## Reproduction Assessment

Yes, source-level. The PR fallback checks only that params.accountId is not the literal default account, so any named Matrix account that falls through to the default agent can re-resolve and match omitted-account bindings.

## Solution Assessment

No. A Matrix-scoped fallback is plausible, but the current condition is too broad; the maintainable fix should respect resolveMatrixDefaultOrOnlyAccountId and keep non-default accounts from matching default-account bindings.
