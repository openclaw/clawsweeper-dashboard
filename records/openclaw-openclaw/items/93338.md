## Summary

Keep open. The PR targets a real Matrix named-account routing gap, but the submitted fix is not merge-ready because it changes omitted `match.accountId` from default-account scope to any-account scope in shared routing, contradicting current source, tests, and docs and risking cross-account message misrouting.

## What This Changes

The PR changes shared route binding normalization so omitted or blank `bindings[].match.accountId` becomes an any-account wildcard and adds Matrix room-routing regression tests.

## Best Possible Solution

Preserve the shared default-account binding contract and fix the Matrix named-account room-binding case at the Matrix routing or account-selection boundary.

## Reproduction Assessment

Yes, source-level. Current main indexes omitted `match.accountId` under the default account, while Matrix can pass a named default-or-only account such as `ops` into shared routing, so the reported binding shape can miss the named-account route.

## Solution Assessment

No. The PR fixes the symptom by changing the shared account-binding contract; the narrower maintainable fix is Matrix-scoped handling that keeps explicit `accountId: "*"` as the any-account path.
