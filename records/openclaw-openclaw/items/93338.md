## Summary

Keep open. ## What This Changes

The PR adds a Matrix inbound-route fallback that retries default-account routing for named accounts, expands Matrix route tests, and adds two Telegram schema-check scripts.

## Best Possible Solution

Keep the shared omitted-account contract, restrict the Matrix fallback to the effective default-or-only Matrix account, add positive and negative multi-account tests, remove the unrelated scripts, and refresh current-head proof.

## Reproduction Assessment

Yes, source-level. Current main scopes omitted match.accountId to the default account, and PR head retries that default-account lookup for every non-default Matrix account when the first route falls through.

## Solution Assessment

No. A Matrix-scoped fallback is plausible, but this condition is too broad; the maintainable fix should respect resolveMatrixDefaultOrOnlyAccountId and keep explicit accountId:"*" as the any-account contract.
