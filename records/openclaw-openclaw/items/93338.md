## Best Possible Solution

## Reproduction Assessment

Yes, source-level. Current main indexes omitted `match.accountId` under the default account, while Matrix can pass a named default-or-only account such as `ops` into shared routing, so the reported binding shape can miss the named-account route.

## Solution Assessment

No. The PR fixes the symptom by changing a shared account-binding contract; the narrower maintainable fix is Matrix-scoped handling that keeps explicit `accountId: "*"` as the any-account path.
