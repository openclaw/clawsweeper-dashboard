## Summary

Keep open. The PR targets a real Matrix per-room routing bug, but the branch fixes it by redefining shared `bindings[].match.accountId` omission from default-account scope to any-account scope, which conflicts with current tests/docs and can misroute existing multi-account channel traffic.

## What This Changes

The PR changes shared route binding normalization so omitted or blank `match.accountId` becomes `*`, and adds core and Matrix routing tests for Matrix room bindings.

## Best Possible Solution

Preserve shared default-account binding semantics and fix the Matrix named default/only account case at the Matrix routing or account-selection boundary, with focused Matrix coverage.

## Reproduction Assessment

Yes, source-level. Current main passes Matrix's effective account id into `resolveAgentRoute()`, while omitted `match.accountId` bindings are indexed under the default account, so a named Matrix account can miss the reported room binding shape.

## Solution Assessment

No. The PR fixes the symptom by changing the shared account-binding contract; the narrower maintainable fix is Matrix-scoped handling that keeps explicit `accountId: "*"` as the any-account path.
