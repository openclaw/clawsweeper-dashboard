## Summary

Keep open. The linked Matrix bug is real and this PR is focused, but the current patch changes the shared route-binding contract from “omitted accountId means default account” to “omitted accountId means any account,” which conflicts with current docs and tests and can misroute other multi-account channels.

## What This Changes

This PR changes shared route binding normalization so an omitted binding match accountId becomes `*`, updates the core routing expectation, and adds Matrix room-message binding tests.

## Best Possible Solution

Preserve the shared routing contract and fix the Matrix named-account case at the Matrix routing/account-selection boundary with focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main passes Matrix's effective accountId into `resolveAgentRoute`, while missing `match.accountId` is indexed under `DEFAULT_ACCOUNT_ID`, so the linked named-account Matrix room-binding shape can miss the binding.

## Solution Assessment

No. The PR fixes the symptom by changing the shared account-binding contract; the better fix is Matrix-scoped handling that keeps `accountId: "*"` as the explicit wildcard.
