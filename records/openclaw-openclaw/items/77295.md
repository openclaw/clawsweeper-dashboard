## Summary

Keep open: current main still applies the Flux edit route and request schema to every Fal image edit, while Fal GPT Image 2 documents a different `/edit` endpoint with plural `image_urls`. This is a narrow Fal plugin bug with clear source and upstream-contract evidence.

## What This Changes

Route Fal GPT Image 2 reference-image edits to the correct edit endpoint and request schema while preserving existing Flux image-to-image behavior.

## Best Possible Solution

Add model-aware Fal edit routing and request-body construction in the Fal plugin, with regression coverage for GPT Image 2 and Flux plus user-facing docs/changelog updates if behavior text changes.

## Solution Assessment

Yes: the narrowest maintainable fix is provider-local model-family handling in `extensions/fal/image-generation-provider.ts`, with tests that preserve Flux while adding GPT Image 2 edit coverage.
