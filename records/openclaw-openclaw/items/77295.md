## Summary

Keep open: current main still routes Fal GPT Image 2 edits through the Flux-style endpoint/body, and the issue is paired with open PR #77388 using closing syntax, so it should stay open until that PR is reviewed and merged or replaced.

## What This Changes

Fal GPT Image 2 reference-image edits should use the Fal `/edit` endpoint, plural `image_urls`, supported edit sizing, and the correct reference-image limits instead of the Flux-only edit contract.

## Best Possible Solution

Land or replace #77388 with a provider-local Fal fix that branches endpoint, request body, sizing, and input limits by model family, with focused regression tests and user-facing docs/changelog updates.

## Reproduction Assessment

Yes. Source inspection on current main plus Fal's published endpoint schemas gives a high-confidence path: `fal/openai/gpt-image-2` is parsed into the Fal provider, then current code appends `/image-to-image`, sends `image_url`, and blocks edit `aspectRatio`.

## Solution Assessment

Yes. A provider-local, model-family-specific Fal fix is the narrowest maintainable solution because Fal's GPT Image 2 edit contract differs from the existing Flux contract, and #77388 is the active implementation candidate.
