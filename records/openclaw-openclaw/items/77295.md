## Summary

Keep open: current main still uses the Flux-style Fal edit route and body for every Fal image edit, while Fal's GPT Image 2 edit docs require `/edit` with `image_urls`.

## What This Changes

The issue requests a Fal provider fix so `fal/openai/gpt-image-2` reference-image edits use the GPT Image 2 edit endpoint and request schema.

## Best Possible Solution

Patch the Fal plugin locally to select the correct edit subpath and image input field per Fal model family, then update regression tests, docs, and changelog.

## Reproduction Assessment

Yes from source inspection: `fal/openai/gpt-image-2` routes to provider `fal` with model `openai/gpt-image-2`, and current main would POST edits to `/openai/gpt-image-2/image-to-image` with `image_url`. I did not run a live Fal generation request.

## Solution Assessment

Yes: a provider-local, model-family-specific fix is the narrowest maintainable boundary because Fal's own endpoints differ between Flux and GPT Image 2, and no core API change is required.
