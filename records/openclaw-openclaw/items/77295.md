## Summary

Keep open: current main still has the reported Fal image-edit routing/body behavior, and open PR #77388 explicitly uses closing syntax for this issue. The issue should remain open until that PR is reviewed and either merged or closed.

## What This Changes

Fal image edit handling is requested to route GPT Image 2 and Nano Banana edit-capable models to Fal’s `/edit` schema while preserving Flux `/image-to-image` behavior.

## Best Possible Solution

Review and land a provider-local fix that selects the correct Fal edit endpoint/body per model family, preserves Flux behavior, updates Fal capabilities/docs, and adds regression tests for GPT Image 2 and Nano Banana edit paths.

## Reproduction Assessment

Yes from source inspection and official Fal docs: current main routes `fal/openai/gpt-image-2` edits to `/openai/gpt-image-2/image-to-image` with `image_url`, while Fal documents `/openai/gpt-image-2/edit` with `image_urls`. I did not perform a live Fal request.

## Solution Assessment

Yes: a Fal plugin-local, model-family-specific fix is the narrowest maintainable path because the endpoint and request schema differ between Flux and GPT Image/Nano Banana families, and no core API change is required.
