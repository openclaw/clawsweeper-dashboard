## Summary

Keep open: current main still applies the Flux Fal edit contract to every Fal image edit, while Fal GPT Image 2 edit requires `/edit` and plural `image_urls`; the reporter's hotfix comments also confirm that this provider-local direction works live.

## What This Changes

The issue requests fixing Fal GPT Image 2 reference-image edits so `image_generate` uses Fal's `/edit` endpoint and plural `image_urls` payload instead of the Flux edit contract.

## Best Possible Solution

Update the Fal plugin to choose `/edit` plus `image_urls` for `openai/gpt-image-*` edit models while preserving Flux `/image-to-image` plus `image_url`, then align focused tests, docs, and changelog.

## Reproduction Assessment

Yes, source-reproducible: current main routes `fal/openai/gpt-image-2` edits to `openai/gpt-image-2/image-to-image` and sends `image_url`, while Fal documents `openai/gpt-image-2/edit` with required `image_urls`. I did not run a live Fal request because no Fal key was present in the environment.

## Solution Assessment

Yes, the narrowest maintainable fix is provider-local, model-family-specific Fal edit handling. A core API change is not needed because the existing `image_generate` model override and Fal plugin boundary already carry the necessary information.
