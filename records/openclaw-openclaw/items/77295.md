## Summary

Keep open: current main still applies the Flux Fal edit contract to every Fal image edit, so `fal/openai/gpt-image-2` with a reference image routes to `/openai/gpt-image-2/image-to-image` with `image_url` even though Fal's GPT Image 2 edit contract is `/edit` with `image_urls`.

## What This Changes

Fix Fal GPT Image 2 reference-image editing so it uses the model's edit endpoint and request schema while preserving existing Flux image-to-image behavior.

## Best Possible Solution

Patch the Fal plugin locally with model-specific edit endpoint, body, and size handling, then add regression coverage plus user-facing docs and changelog updates.

## Reproduction Assessment

Yes from source inspection: `fal/openai/gpt-image-2` resolves to the Fal provider with model `openai/gpt-image-2`, and current main would POST edits to `/openai/gpt-image-2/image-to-image` with `image_url`. I did not run a live current-main Fal request, but the reporter verified the corrected contract against Fal live.

## Solution Assessment

Yes: a provider-local, model-family-specific fix is the narrowest maintainable solution because Fal documents different edit endpoints and payload fields for GPT Image 2 versus Flux, and no core API change is required.
