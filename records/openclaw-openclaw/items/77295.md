## Summary

Keep open: current main still routes Fal GPT Image 2 reference-image edits through the Flux-style endpoint/body, and #77388 is an open linked implementation PR with closing syntax that should be reviewed or landed before this issue closes.

## What This Changes

The issue asks the Fal image provider to route GPT Image 2/Nano Banana edit requests to Fal's `/edit` endpoints, use `image_urls`, and align edit capabilities without breaking Flux.

## Best Possible Solution

Land a provider-local Fal fix that branches endpoint, payload shape, limits, sizing, docs, changelog, and regression coverage by model family; #77388 is the active implementation candidate.

## Reproduction Assessment

Yes. Source inspection on current main shows the `fal/openai/gpt-image-2` override reaches the Fal provider as `openai/gpt-image-2`, then appends `/image-to-image` and sends `image_url` even though Fal's GPT Image 2 edit docs require `/edit` with `image_urls`.

## Solution Assessment

Yes. A provider-local, model-family-specific Fal fix is the narrowest maintainable direction; the linked open PR is the right place to review that implementation before closing this issue.
