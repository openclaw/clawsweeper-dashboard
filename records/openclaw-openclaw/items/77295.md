## Summary

Keep open: current main still constructs Fal GPT Image 2 edits with the Flux-style endpoint and payload, and the open linked implementation PR should be reviewed or landed before this issue closes.

## What This Changes

The issue requests a Fal provider fix so GPT Image 2 image edits use Fal's `/edit` contract instead of `/image-to-image`, including the correct reference-image payload and edit sizing behavior.

## Best Possible Solution

Land a provider-local Fal fix that routes GPT Image 2/Nano Banana edit models to `/edit`, preserves Flux `/image-to-image` behavior, serializes request bodies by model family, and updates regression coverage plus user-facing release notes.

## Reproduction Assessment

Yes, from source. Current main parses `fal/openai/gpt-image-2` into the Fal provider, appends `/image-to-image`, sends `image_url`, and rejects edit aspect ratios while Fal documents GPT Image 2 edit at `/edit` with `image_urls`.

## Solution Assessment

Yes. A provider-local, model-family-specific Fal fix is the narrowest maintainable direction; the open PR is the right place to review that implementation before closing this issue.
