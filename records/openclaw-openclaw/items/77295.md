## Summary

Keep open: current main still routes every Fal image edit through the Flux-style `/image-to-image` path and singular `image_url` body, while the linked open PR #77388 is the active closing implementation candidate for this issue.

## What This Changes

The issue requests model-family-specific Fal image edit routing and request bodies so GPT Image 2/Nano Banana edit requests use `/edit` with `image_urls` while Flux behavior stays compatible.

## Best Possible Solution

Align the Fal plugin with each supported Fal image-edit model family's documented endpoint and request schema, with regression coverage that preserves the existing Flux contract.

## Reproduction Assessment

Yes. Source inspection on current main shows `fal/openai/gpt-image-2` is parsed to provider `fal` and model `openai/gpt-image-2`, then the Fal provider appends `/image-to-image` and sends `image_url`, which conflicts with Fal's documented GPT Image 2 `/edit` schema.

## Solution Assessment

Yes for the central bug: a provider-local, model-family-specific fix is the narrowest maintainable path. Because open PR #77388 already references this issue with closing syntax, the issue should remain open until that PR is reviewed and either landed or replaced.
