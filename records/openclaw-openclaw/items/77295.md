## Summary

Keep open: current main still has the Fal provider behavior described here, and #77388 is an open implementation PR with closing syntax that should be reviewed or landed before this issue closes.

## What This Changes

Fal GPT Image 2 image edits should use Fal's `/edit` endpoint and `image_urls` payload instead of the Flux-style `/image-to-image` path and singular `image_url` field.

## Best Possible Solution

Land #77388 or an equivalent provider-local Fal fix with regression coverage, docs, and changelog updates, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Source inspection on current main traces `fal/openai/gpt-image-2` to the Fal provider, where edit requests append `/image-to-image`, send `image_url`, and retain one-image/edit-aspect restrictions; I did not run a live Fal request.

## Solution Assessment

Yes. A provider-local, model-family-specific Fal fix is the narrowest maintainable direction because Fal's GPT Image 2/Nano Banana edit contracts differ from Flux while the shared image runtime can stay generic.
