## Summary

Keep open: current main still has the Fal edit routing and payload behavior described here, and #77388 is an open closing PR that should be reviewed or landed before this issue closes.

## What This Changes

Requests Fal image edits for GPT Image 2 and Nano Banana models to use Fal edit endpoints and edit payloads instead of the Flux-only image-to-image route.

## Best Possible Solution

Review and land #77388 or an equivalent provider-local Fal fix with regression coverage and docs/changelog updates while preserving existing Flux image-to-image behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows `fal/openai/gpt-image-2` reaches the Fal provider as `openai/gpt-image-2`, then edit mode appends `/image-to-image` and sends singular `image_url`; I did not run a live Fal request.

## Solution Assessment

Yes. A provider-local model-family fix is the narrowest maintainable path: route GPT Image/Nano Banana edit models to `/edit` with their documented payloads while leaving Flux on `/image-to-image`.
