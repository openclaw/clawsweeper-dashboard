## Summary

Keep open. Current main has the WebChat upload transport for non-video files and can extract text-like files plus PDFs, but the requested Word document extraction is still missing: DOC/DOCX MIME types are not in the default input-file extraction allowlist, vendor Office attachments are skipped before media-understanding file extraction, and the bundled document-extract plugin only advertises PDF.

## What This Changes

Add WebChat document uploads for TXT, PDF, DOC, and DOCX with file picker support, extraction into agent-readable text, validation, metadata/progress UI, and retained downloadable references.

## Best Possible Solution

Keep the shipped non-video upload transport, then make a maintainer-reviewed document extraction decision: adopt or adapt the existing document-extractor plugin seam for DOCX, route DOCX through WebChat/media-understanding extraction when approved, and explicitly document or reject legacy .doc behavior with tests and docs aligned.
