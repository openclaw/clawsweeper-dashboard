## Summary

Keep open. Current main covers WebChat upload selection, non-video file transport, and text/PDF extraction, but Word document extraction is still skipped and the open DOCX work does not settle legacy .doc or parsing policy.

## What This Changes

Add WebChat file uploads for PDF, TXT, DOC, and DOCX documents with extracted text sent to the assistant.

## Best Possible Solution

Keep the current managed-media upload transport, add policy-backed DOCX extraction through the existing media/document-extractor pipeline if approved, and explicitly reject or defer legacy .doc unless maintainers approve a safe extractor.

## Reproduction Assessment

Yes. Source and tests show a Word-shaped WebChat upload can be accepted and transported, but Office and legacy Word MIME types are classified as binary and skipped before text extraction.

## Solution Assessment

No. The issue’s proposed multer-style endpoint is not the best fit for current architecture; the maintainable path is extending the existing chat.send managed-media and document-extractor/media-understanding pipeline.
