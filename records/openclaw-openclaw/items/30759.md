## Summary

Current main already implements the central WebChat document-upload request: the Control UI accepts non-video files including PDF, TXT, DOC, and DOCX, and `chat.send` persists non-image attachments as agent-readable media paths. The implementation is also present in the latest release tag, so this can close as implemented.

## What This Changes

Add WebChat support for uploading document files such as PDF, Word, and TXT from the chat interface.

## Best Possible Solution

Keep the shipped generic WebChat attachment pipeline and leave broader UI/model/skill menu work to #30389 rather than reopening this narrower document-upload request.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source and regression tests provide a high-confidence verification path for PDF and generic document upload handling on current main.

## Solution Assessment

Yes. Reusing the generic Control UI plus Gateway attachment pipeline is more maintainable than adding a WebChat-only multer endpoint, and it covers the central ability to upload document files from WebChat.
