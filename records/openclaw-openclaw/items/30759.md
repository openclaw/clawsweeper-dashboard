## Summary

Current main already supports WebChat document/file uploads through the Control UI picker, chat.send attachment serialization, and Gateway non-image media staging, so the central document-upload request is implemented.

## What This Changes

Add WebChat document upload support for PDF, Word, TXT, and similar files from the chat input.

## Best Possible Solution

Keep the merged WebChat/Gateway attachment implementation; broader model and skill picker ideas belong with the still-open broader WebChat UX item #30389.

## Reproduction Assessment

Not applicable. This is a feature request, and source plus regression tests show the requested WebChat document upload path now exists for PDF, TXT/text, and Office document formats.

## Solution Assessment

Yes. The implemented path reuses the existing chat.send and media-staging contracts instead of adding a separate /api/webchat/upload endpoint, which keeps WebChat aligned with Gateway media and PDF-tool behavior.
