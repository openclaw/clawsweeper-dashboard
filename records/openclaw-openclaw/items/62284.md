## Summary

Keep open. Current main still lacks timed exec approval support, the same-author implementation PR was closed unmerged, and the active related trust-window PRs still need maintainer security/product decisions before one canonical implementation can land.

## What This Changes

Request a time-bounded exec approval mode such as `/approve 30m` or `/approve 1h`, with UI buttons, CLI visibility, expiry handling, and optional default duration config.

## Best Possible Solution

Define one maintainer-approved timed exec approval/trust model with clear authority, scope, expiry, audit, hard-deny, restart, UI, CLI, protocol, docs, and tests, then land a single implementation that supersedes competing PRs.
