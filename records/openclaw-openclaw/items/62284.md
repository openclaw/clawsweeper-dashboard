## Summary

Keep open. Current main still lacks timed exec approvals, and the related trust-window implementations are open or superseded with unresolved product/security proof blockers rather than a shipped canonical fix.

## What This Changes

Add a time-bounded exec approval mode so an operator can approve matching host commands for a limited duration instead of once or permanently.

## Best Possible Solution

Define one canonical timed exec approval or trust-window model with explicit authority, expiry, hard-deny behavior, audit/status visibility, restart semantics, protocol/UI/CLI coverage, and real behavior proof before landing it.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives a high-confidence existence check: current main has only once, always, and deny approval decisions with no timed approval storage, protocol payload, parser, UI action, or CLI display.

## Solution Assessment

Unclear as proposed. Timed approvals are plausible, but maintainers still need to choose between per-command timed approvals and broader trust windows and settle authority, persistence, hard-deny, audit, and status semantics.
