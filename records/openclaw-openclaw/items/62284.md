## Summary

Keep open. Current main still lacks timed exec approvals, and the related trust-window PRs are still open with security/product and real-proof blockers, so this is not a conservative cleanup close.

## What This Changes

Add time-bounded exec approvals so a user can temporarily approve command execution for a duration instead of approving once or permanently allowing it.

## Best Possible Solution

Define one maintainer-approved timed exec approval or trust model with explicit authority, scope, expiry, audit, hard-deny semantics, restart behavior, UI/CLI/protocol/docs coverage, and tests, then land one canonical implementation.

## Reproduction Assessment

Not applicable as a bug reproduction, but yes for source-level verification: current main lacks duration parsing, duration schema, timed storage, timed UI buttons, and timed CLI display across the central approval paths.

## Solution Assessment

Unclear as proposed. The timed-approval direction is useful, but maintainers still need to choose between per-command timed approval and broader trust windows, plus settle storage, authority, audit, and hard-deny behavior.

## Security Review

Status: needs_attention
