## Summary

Keep open; the branch is useful but not ready because it labels every no-key `sessions.create` caller as `WebUI chat` and still does not make the local `openclaw sessions` output show that friendly label.

## What This Changes

The PR adds a default `WebUI chat` label for no-key `sessions.create` calls, gateway regression tests, and a changelog entry.

## Best Possible Solution

Update the PR so the default label semantics match the intended creator surface and the sessions CLI/JSON can surface the friendly name, with the changelog aligned to the actual behavior.

## Reproduction Assessment

Yes, source-reproducible: current main's Control UI `/new` path calls `sessions.create` without `label`, and the gateway writes only the provided label. The remaining PR defect is also source-visible because the local sessions CLI drops labels before rendering or JSON output.

## Solution Assessment

No; the proposed gateway fallback is a partial solution, but it is overbroad for non-WebUI no-key callers and does not fix the `openclaw sessions` readability path cited by the PR and linked issue.

## Review Findings

Overall correctness: patch is incorrect

Even after this gateway write, both text output and `--json` still omit the friendly label, so the cited user-visible problem remains unless the CLI projection/rendering is updated or the changelog/closing scope is narrowed.
