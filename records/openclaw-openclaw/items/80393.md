## Summary

Keep open: the PR is a plausible scoped fix for the Copilot image auth bypass, and the screenshots support auth-layer progress, but current head fails TypeScript because the baseUrl follow-up reassigns a `const` model in `resolveImageRuntime`.

## What This Changes

The PR updates Copilot IDE header constants and changes GitHub Copilot image runtime setup to exchange the raw OAuth token for a short-lived Copilot API token and baseUrl.

## Best Possible Solution

Keep the scoped token/baseUrl approach, repair the const reassignment with a mutable resolved-model pattern, then rerun typecheck, lint, and focused Copilot/media validation before maintainer review.

## Reproduction Assessment

Yes for source-level reproduction: current main's image runtime uses the raw resolved key while the existing Copilot helper and chat path exchange tokens and carry baseUrl. The supplied before/after screenshots also show the reported integrator error changing after the PR.

## Solution Assessment

No, not as currently written, because the PR fails TypeScript by assigning to a `const` model. After that mechanical fix, reusing the existing Copilot token exchange and baseUrl contract is the narrowest maintainable solution for the scoped auth issue.
