## Summary

Keep open for normal maintainer review and landing: current main still routes GitHub Copilot image requests with the raw resolved credential, while this PR adds the focused token exchange/baseUrl alignment and I found no blocking correctness finding.

## What This Changes

The PR updates GitHub Copilot image-tool authentication to exchange OAuth credentials for a short-lived Copilot API token, applies the returned Copilot baseUrl, and refreshes Copilot IDE header constants.

## Best Possible Solution

Land this scoped auth/baseUrl fix after maintainer review, sequence it with the companion payload-shape fix, and consider a follow-up that routes media understanding through the generic prepareRuntimeAuth seam for all providers.

## Reproduction Assessment

Yes, source-reproducible: current main passes the raw resolved GitHub credential through the image runtime, while adjacent Copilot completion paths exchange it for a runtime token and baseUrl. The contributor screenshots provide live before/after evidence that the integrator authorization failure moves past the auth layer after the patch.

## Solution Assessment

Yes for the scoped auth fix: reusing resolveCopilotApiToken and applying the returned baseUrl matches the existing simple-completion behavior. A generic media prepareRuntimeAuth path would be cleaner long-term but is broader than the immediate bug fix.
