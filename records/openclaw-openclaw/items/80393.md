## Summary

Keep open: the PR targets a valid GitHub Copilot image-tool auth bug, but it drops the endpoint returned by the Copilot token exchange, so proxy or enterprise-routed Copilot image requests can still use the wrong base URL.

## What This Changes

The PR updates Copilot IDE header constants and adds a Copilot token exchange before media-understanding image requests for `github-copilot` models.

## Best Possible Solution

Keep the image runtime aligned with existing Copilot runtime auth: exchange the token, propagate both token and baseUrl into the model used for completion, and add focused media-understanding regression coverage.

## Reproduction Assessment

Yes for source-level reproduction: current main image requests use the raw resolved key, while Copilot token exchange returns a runtime token plus baseUrl and the normal chat path applies both. I did not run a live Copilot request in this read-only review, but the PR screenshots provide before/after real-output evidence for the auth-layer symptom.

## Solution Assessment

No: the token exchange is the right repair direction, but the PR is not complete until the returned Copilot baseUrl is carried into the image completion model like the existing chat path.
