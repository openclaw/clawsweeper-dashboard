## Summary

Keep open. Current main still has the reported Anthropic-compatible payload gap: replay validation only merges consecutive user turns, a focused test documents that adjacent assistant turns are preserved, and the Anthropic transport returns converted payload entries without a final same-role coalescing pass.

## What This Changes

The issue requests Anthropic-compatible message normalization for custom `anthropic-messages` providers so adjacent same-role messages are coalesced and invalid trailing assistant messages are not sent to strict proxies.

## Best Possible Solution

Add focused Anthropic Messages normalization at the replay or outbound transport boundary that coalesces adjacent user and assistant payload entries and strips invalid trailing assistant payloads, while leaving unrelated providers and bundled dependency code alone.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: feed validateAnthropicTurns a user, assistant, assistant history and current main preserves all three turns; the Anthropic transport can also append adjacent user payload entries and returns params without a final coalescing pass.

## Solution Assessment

Yes. A narrow Anthropic-compatible replay/transport normalization is the maintainable path; patching @mariozechner/pi-ai directly or applying a global all-provider merge would be broader and risk changing providers with different replay contracts.
