## Summary

Keep open. Current main still has a source-proven Anthropic Messages path where adjacent same-role payload entries can survive: replay validation merges consecutive user turns only, a focused test preserves adjacent assistant turns, and the Anthropic transport returns converted payloads without a final coalescing pass. The related MiniMax and trailing-assistant-prefill fixes cover neighboring strict-turn-shape failures but do not supersede this custom `anthropic-messages` proxy case.

## What This Changes

The issue requests Anthropic Messages normalization for custom providers/proxies so consecutive same-role payload messages are coalesced and invalid trailing assistant payloads are removed before requests.

## Best Possible Solution

Add a narrow Anthropic-compatible normalization at the outbound Anthropic transport or replay boundary that coalesces adjacent user and assistant payload entries and strips invalid trailing assistant payloads without changing unrelated provider policies. Cover the exact custom `anthropic-messages` proxy shape in focused tests, and update transcript hygiene docs plus changelog if behavior changes.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: `validateAnthropicTurns` preserves adjacent assistant turns, and `convertAnthropicMessages` can emit adjacent user payload entries when tool-result-as-user payloads are followed by a user turn because it returns `params` without final coalescing.

## Solution Assessment

Yes. The best repair is a narrow Anthropic-compatible replay/transport normalization; patching the bundled dependency directly or globally merging all providers would be broader and could change provider contracts that do not need this behavior.
