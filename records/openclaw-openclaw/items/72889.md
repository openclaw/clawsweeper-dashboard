## Summary

Keep open. Current main still has a source-proven Anthropic Messages path where adjacent same-role payload entries can survive, especially tool-result-as-user followed by a user turn and non-trailing assistant+assistant replay. The related trailing-prefill fixes address neighboring ending-shape failures, but they do not fully cover this custom `anthropic-messages` proxy case.

## What This Changes

The issue requests Anthropic Messages replay/payload normalization so consecutive same-role messages are merged before strict custom proxies reject them with 400 errors.

## Best Possible Solution

Add a narrow Anthropic Messages normalization at the replay or outbound transport boundary that coalesces adjacent user and assistant payload entries, keeps valid tool-use structure intact, and preserves the existing trailing-assistant protections. Cover the custom `anthropic-messages` proxy shape with focused tests, and update transcript-hygiene docs plus changelog if behavior changes.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: `validateAnthropicTurns` preserves adjacent assistant turns, and `convertAnthropicMessages` can emit adjacent `user` payload entries when an Anthropic tool result is followed by a user turn because it returns the payload array without final coalescing.

## Solution Assessment

Yes. The narrow maintainable solution is OpenClaw-owned Anthropic replay/payload normalization; patching `node_modules` or globally changing every provider’s turn policy would be broader and risk changing unrelated provider contracts.
