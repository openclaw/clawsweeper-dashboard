## Summary

Keep open. Current main still has a source-proven Anthropic Messages payload path where adjacent same-role entries can survive, especially assistant/assistant replay and user/tool-result-as-user/user conversion; existing fixes cover consecutive user replay and trailing assistant prefill, but not the general same-role coalescing requested here.

## What This Changes

The issue asks OpenClaw to normalize Anthropic Messages payloads for custom `anthropic-messages` proxies by merging consecutive same-role entries before provider requests.

## Best Possible Solution

Add a narrow Anthropic Messages outbound normalization that coalesces adjacent `user` and `assistant` payload entries after replay/tool-result conversion, preserves block ordering and thinking/tool-use semantics, and keeps the existing trailing assistant prefill stripping intact. Cover the custom `anthropic-messages` proxy shape with focused transport/replay tests and add a user-facing changelog entry if behavior changes.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: `validateAnthropicTurns` preserves `user, assistant, assistant`, and `convertAnthropicMessages` can emit adjacent `user` payloads when tool results are converted to Anthropic user messages beside ordinary user turns.

## Solution Assessment

Yes. A narrow Anthropic-compatible outbound normalization is the maintainable fix; patching the bundled dependency directly or globally changing all provider transcript behavior would be broader and risk provider-specific regressions.
