## Summary

Keep open. Current main still leaves a source-proven Anthropic Messages path where adjacent same-role payload entries can survive: replay validation only coalesces consecutive `user` turns, the focused test still locks in non-merged adjacent `assistant` turns, and the Anthropic transport returns converted payloads without a final same-role merge pass. The related MiniMax and trailing-prefill issues address neighboring strict-turn-shape failures but do not supersede this custom `anthropic-messages` proxy case.

## What This Changes

The issue requests Anthropic-compatible payload normalization for custom `anthropic-messages` providers so consecutive same-role messages are coalesced and invalid trailing assistant payloads are stripped before the API call.

## Best Possible Solution

Add a narrow Anthropic Messages normalization at the replay or outbound transport boundary that coalesces adjacent user and assistant payload entries and strips invalid trailing assistant payloads without changing unrelated provider policies. Cover the exact custom `anthropic-messages` proxy shape in focused tests and update transcript hygiene docs plus changelog if behavior changes.

## Reproduction Assessment

Yes. A source-level reproduction is clear: `validateAnthropicTurns` preserves a `user, assistant, assistant` sequence, and `convertAnthropicMessages` can emit adjacent `user` payload entries when an Anthropic tool-result-as-user payload is followed by a user turn because it returns `params` without final coalescing.

## Solution Assessment

Yes. The best repair is a narrow Anthropic-compatible replay/transport normalization; patching the bundled dependency directly or globally merging all providers would be broader and risk changing provider contracts that do not need this behavior.
