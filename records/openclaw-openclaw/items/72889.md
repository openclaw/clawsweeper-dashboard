## Summary

Keep open. Current main still has an Anthropic Messages outbound path where transport repair can synthesize a `toolResult` before the next real user turn, and the converter maps that `toolResult` to another Anthropic `user` payload without a final coalescing pass.

## What This Changes

The issue requests Anthropic-compatible outbound message normalization so consecutive same-role payload messages do not trigger HTTP 400 errors in stricter custom proxies.

## Best Possible Solution

Add a final Anthropic Messages outbound normalizer after replay repair and conversion, scoped to Anthropic-compatible transports, with regression tests, transcript-hygiene docs, and a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: an assistant tool call missing a result followed by a real user turn becomes `assistant`, `toolResult`, `user`, which the Anthropic converter sends as `assistant`, `user`, `user`.

## Solution Assessment

Yes. Fixing the final Anthropic transport payload is the narrowest maintainable path because earlier turn validation cannot see provider-specific `toolResult` to `user` conversion.
