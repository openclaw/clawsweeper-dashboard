## Summary

Keep open. Current main still has an Anthropic Messages outbound path where replay repair can create adjacent `user` payload entries after conversion, matching the reported custom-proxy 400 failure mode.

## What This Changes

Normalize Anthropic-compatible outbound payloads so consecutive same-role messages are merged before they are sent to Anthropic Messages APIs or compatible proxies.

## Best Possible Solution

Add a final Anthropic-compatible outbound payload normalization pass after transport conversion, with focused regression coverage and transcript-hygiene/changelog updates.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a repaired assistant tool call followed by a real user turn becomes `assistant`, `toolResult`, `user`, then converts to adjacent Anthropic payload roles `assistant`, `user`, `user`.

## Solution Assessment

Yes. Fixing the final Anthropic transport payload boundary is the narrowest maintainable solution because earlier replay validation cannot see the later `toolResult` to `user` payload conversion, and patching only `pi-ai` would miss OpenClaw's boundary-aware transport path.
