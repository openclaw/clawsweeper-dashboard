## Summary

Keep open. Current main still has an Anthropic Messages outbound path where repaired `toolResult` entries are converted into `user` payload entries and can sit next to a following real user payload entry without a final coalescing pass, matching the reporter's proxy 400 failure mode.

## What This Changes

The issue requests Anthropic-compatible outbound message normalization so consecutive same-role messages are merged before proxy/API submission.

## Best Possible Solution

Add final Anthropic Messages payload normalization at the outbound transport boundary after replay repair and conversion, scoped to Anthropic-compatible APIs, with focused regression coverage plus transcript-hygiene docs and changelog updates.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: an assistant tool call with a synthesized or repaired `toolResult` followed by a real user turn becomes adjacent Anthropic `user` payload entries because conversion happens after replay validation.

## Solution Assessment

Yes. Fixing the final Anthropic transport payload is the narrowest maintainable solution because replay validation cannot see the later `toolResult` to Anthropic `user` conversion, and patching only `pi-ai` would miss OpenClaw's boundary-aware transport path.
