## Summary

Keep open. Current main still has an Anthropic Messages outbound path where repaired `toolResult` entries become `user` payload entries next to a following real `user` entry, with no final same-role coalescing pass before the request body is sent.

## What This Changes

The issue asks OpenClaw to normalize Anthropic Messages payloads by merging consecutive same-role outbound messages for custom Anthropic-compatible providers and proxies.

## Best Possible Solution

Add final Anthropic-compatible payload coalescing at the transport boundary after replay repair and conversion, with focused request-payload regression coverage plus docs and changelog updates.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: an assistant tool call with a synthesized or repaired `toolResult` followed by a real user turn becomes adjacent Anthropic `user` payload entries in the final request body.

## Solution Assessment

Yes. Fixing the final Anthropic transport payload is the narrowest maintainable solution because earlier replay validation cannot see the later `toolResult` to Anthropic `user` conversion, and patching only `pi-ai` would miss OpenClaw's boundary-aware transport path.
