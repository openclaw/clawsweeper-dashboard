Current main still has the reported Anthropic-compatible payload gap: `toolResult` conversion can create adjacent outbound `user` messages and the transport sends that array unchanged, while the reporter confirmed the local coalescing patch was still required after upgrading to OpenClaw 2026.5.2.

## What This Changes

The issue requests final outbound normalization for Anthropic-compatible provider messages so consecutive same-role turns are merged before strict proxies receive the request.

## Best Possible Solution

## Solution Assessment

Yes. A final outbound coalescing step at the Anthropic transport/shared wrapper boundary is the narrowest maintainable fix; patching `node_modules` or rewriting stored transcript history would be the wrong layer.
