## Summary

Keep open. Current main still maps repaired `toolResult` turns into Anthropic `user` payload entries after earlier turn validation, so those entries can sit next to a following real user message and still match the reported strict-proxy 400 failure mode.

## What This Changes

Report that Anthropic-compatible `anthropic-messages` payloads can contain consecutive same-role messages after tool-result repair, causing strict custom proxies to reject requests.

## Best Possible Solution

Add final Anthropic Messages payload normalization at the outbound transport boundary after replay repair and conversion, with focused regression coverage, transcript-hygiene docs, and a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: an assistant tool call with a synthesized or repaired `toolResult` followed by a real user turn becomes adjacent Anthropic `user` payload entries after conversion.

## Solution Assessment

Yes. A final outbound Anthropic payload coalescing pass is the narrowest maintainable fix because transcript validation runs before `toolResult` becomes an Anthropic `user` payload, and patching only pi-ai would miss OpenClaw's transport path.
