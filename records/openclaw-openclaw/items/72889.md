## Summary

Keep open. Current main still has a source-proven Anthropic Messages path where replay repair can yield adjacent same-role outbound payload entries: replay validation merges only consecutive `user` turns, the focused test preserves adjacent `assistant` turns, and the Anthropic transport converts tool results into `user` payload entries before returning the payload without a final coalescing pass.

## What This Changes

Normalize custom `anthropic-messages` payloads by merging consecutive same-role messages and stripping invalid trailing assistant turns before sending to Anthropic-compatible endpoints.

## Best Possible Solution

Add a final Anthropic Messages payload normalization at the outbound transport boundary so adjacent `user` and `assistant` payload entries are coalesced after all replay repair and transport conversion have run, while keeping the change scoped to Anthropic-compatible APIs and documenting the resulting transcript rule.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: an assistant tool call followed by a repaired/synthesized `toolResult` and then a user turn converts to adjacent Anthropic `user` payload entries, and the current converter returns those entries without coalescing.

## Solution Assessment

Yes. A final Anthropic Messages payload normalization is the narrowest maintainable fix because replay-level validation alone cannot see the later `toolResult`-as-`user` conversion, and patching the bundled dependency would miss OpenClaw's current boundary-aware transport path.
