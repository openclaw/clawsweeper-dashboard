## Summary

Keep open. Current main still has a real source-level gap around native subagent sandbox inheritance: workspace lineage is carried, but resolved Docker env/setup state is not, and the recreate/workspace behavior needs an explicit security-aware policy before a repair PR is safe.

## What This Changes

The issue reports that sandboxed subagents do not reliably receive configured Docker env/setup state or refreshed workspace/bootstrap state after sandbox recreation.

## Best Possible Solution

Define the native sandbox inheritance contract, carry resolved Docker env/setup only for authorized same-agent sandboxed children, preserve cross-agent isolation, and add regression coverage for recreate/bootstrap freshness.

## Reproduction Assessment

Yes, at source level: current main has no carrier for resolved parent Docker env/setup into native sessions_spawn, and the child resolves sandbox config independently. I did not establish a live Docker failing run in this read-only review.

## Solution Assessment

Unclear until maintainers choose the inheritance/security policy. A narrow same-agent resolved-context handoff with negative cross-agent coverage is safer than sharing parent containers or forwarding Docker env indiscriminately.
