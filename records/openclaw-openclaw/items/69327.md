## Summary

Keep open. Current main still carries spawned workspace lineage but does not carry the parent’s resolved Docker env/setup sandbox context into native subagent spawns, so the report remains valid and needs a sandbox inheritance policy decision.

## What This Changes

The issue asks for sandboxed subagent runs to receive configured Docker env variables, current workspace state, and recreated setupCommand effects.

## Best Possible Solution

Define native sandbox inheritance explicitly: same-agent sandboxed children should observe the effective Docker env/setup and intended workspace after recreate, while cross-agent children should not receive parent Docker state without an explicit reviewed policy.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main has no parent resolved Docker env/setup context handoff from native sessions_spawn into the child run. I did not run a live Docker repro in this read-only review.

## Solution Assessment

Unclear until maintainers choose the inheritance policy. A narrow same-agent resolved-context handoff with negative cross-agent coverage looks safer than sharing the parent container wholesale.
