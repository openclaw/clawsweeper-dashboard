## Summary

Keep open. Current main has fixed adjacent pieces, including explicit Docker env filtering and native subagent cwd/workspace plumbing, but the remaining native subagent sandbox inheritance/config-refresh contract is still not proven or fully represented in the spawn context.

## What This Changes

The issue reports that sandboxed native subagents can miss configured Docker env, current workspace/bootstrap effects, and recreated sandbox state.

## Best Possible Solution

Define the native sandbox inheritance and runtime-config refresh contract, preserve cross-agent isolation, then add Docker-backed proof for same-agent env/setup/recreate plus a negative cross-agent leakage case.

## Reproduction Assessment

Yes at source level, but not as a live Docker repro here. Current main still has no resolved Docker env/setup carrier in `SpawnedToolContext`, and native subagent spawning reloads runtime config instead of using the parent run's resolved sandbox context.

## Solution Assessment

Unclear until maintainers choose the security boundary. A narrow same-agent resolved-context or explicit runtime-config refresh path is safer than sharing parent containers or copying Docker env/setup across any allowed target indiscriminately.
