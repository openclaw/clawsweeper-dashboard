## Summary

Keep open: current main has the explicit Docker env filtering fix, but the reported subagent inheritance/recreate behavior is still unresolved and now has a fresh maintainer-side comment asking for a sandbox="inherit" design decision.

## What This Changes

The issue asks for sandboxed subagent runs to receive configured Docker env/setup/workspace state and to reflect `openclaw sandbox recreate --agent ...` changes reliably.

## Best Possible Solution

Define the same-agent versus cross-agent sandbox inheritance contract, then add Docker-backed regression coverage for env, setupCommand, workspace freshness, recreate behavior, and negative cross-agent leakage cases.

## Reproduction Assessment

No high-confidence live current-main reproduction was run. Source inspection supports the missing parent runtime-resolved Docker env/setup carrier and stale workspace concerns, but the exact same-agent static-config symptom still needs Docker-backed confirmation.

## Solution Assessment

Unclear. Copying parent Docker env/setup blindly is not the safest fix; maintainers need to decide whether sandbox="inherit" means runtime parent env inheritance or target-agent static config inheritance before code changes.
