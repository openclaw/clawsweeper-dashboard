## Summary

Keep open. Current main still preserves spawned workspace lineage only and resolves the child sandbox Docker config independently, so there is no resolved parent Docker env/setup handoff for native subagent spawns; the remaining fix needs an explicit sandbox inheritance security policy.

## What This Changes

Ensure sandboxed native subagents reliably see configured Docker env, current workspace state, and recreated setupCommand/bootstrap state.

## Best Possible Solution

Define native sandbox inheritance explicitly, then carry effective Docker env/setup for authorized same-agent sandboxed children while adding negative cross-agent coverage and recreate/bootstrap regression proof.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main has no parent resolved Docker env/setup context carrier from the sandboxed parent run into native sessions_spawn, and the child resolves sandbox context fresh.

## Solution Assessment

Unclear until maintainers choose the inheritance policy. A narrow same-agent resolved-context handoff with explicit negative cross-agent tests looks safer than sharing a parent container or forwarding Docker env indiscriminately.
