## Summary

Keep open: current main mitigates the original WebChat full-replace UI race, but it still acknowledges `chat.send` before durable user-turn persistence or dispatch progress is guaranteed, and the active server-side fix candidate is open with merge blockers rather than merged.

## What This Changes

The issue reports WebChat optimistic user messages disappearing when history reconciliation runs before the accepted send becomes durable or starts an agent turn.

## Best Possible Solution

Keep this issue as the canonical WebChat durable-send boundary thread until PR https://github.com/openclaw/openclaw/pull/80670 or a narrower replacement lands with recorder-based user-turn persistence, idempotency, rollover, hook, retry, and visible failure coverage.

No high-confidence live current-main reproduction was established here. Source inspection still shows `chat.send` ACKs before durable user-turn persistence and asynchronous dispatch progress, while current tests cover UI preservation rather than that server-side contract.

## Solution Assessment

No, current main is only a partial solution: preserving optimistic UI messages is necessary, but the remaining best fix is a maintainer-approved server-side durable-progress or visible-failure boundary using the current recorder/session-accessor path.
