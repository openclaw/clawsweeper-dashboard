## Summary

Keep open: the proof supports the intended gateway-to-diagnostic attribution path, but the patch still has a blocking run-boundary bug where a later same-session run with absent or invalid `clientContext` can inherit the previous run's opaque upstream context. The new gateway/protocol and plugin-observable diagnostic surface also needs compatibility cleanup and explicit maintainer acceptance before merge.

## What This Changes

The PR adds an optional gateway `agent` `clientContext` request field, normalizes it, stores it in diagnostic session state, emits it on `message.queued` and `session.state`, and regenerates Swift protocol models.

## Best Possible Solution

Clear or replace per-run diagnostic context on every gateway agent run, preserve additive client compatibility, and land the generic core diagnostic context only after maintainers explicitly accept the API boundary.

## Reproduction Assessment

Yes for the blocking patch bug: source inspection shows that a same-session context-bearing run followed by a no-context or invalid-context run leaves the old value in diagnostic session state. I did not run tests because this review was constrained to read-only inspection.

## Solution Assessment

No. The direction is plausible, but the current implementation is not the best mergeable shape until per-run clearing is fixed, Swift/client compatibility is preserved, and maintainers accept the new core diagnostic API boundary.

Full review comments:

- **[P1] Clear clientContext when the run has none:** `src/logging/diagnostic.ts:915-918`
  - body: `setDiagnosticSessionClientContext` returns when normalization yields `undefined`, so a later run in the same diagnostic session that omits `clientContext` or sends an oversized/invalid bag leaves the previous run's `state.clientContext` in place. Because `message.queued` and `session.state` now emit that state field, downstream diagnostic plugins can misattribute the later run to stale upstream context; make the seeder actively clear the field and cover same-session context-to-none/invalid transitions.
  - confidence: 0.93
- **[P2] Preserve Swift AgentParams source compatibility:** `apps/shared/OpenClawKit/Sources/OpenClawProtocol/GatewayModels.swift:821`
  - body: The wire schema change is optional, but the regenerated Swift `AgentParams` initializer now requires every existing caller to add `clientcontext: nil`. That makes an additive protocol field source-breaking for OpenClawKit consumers; adjust the generator/output to give the new optional argument a default or provide a compatibility initializer.
  - confidence: 0.82
