## Summary

Keep open. Current main still carries the reported WhatsApp plugin dependency path from `@openclaw/whatsapp` to Baileys and then to a pinned `@whiskeysockets/libsignal-node` package whose upstream metadata declares GPL-3.0; this is a concrete license/supply-chain concern, but the mitigation requires maintainer/legal distribution judgment rather than conservative automated cleanup.

## Best Possible Solution

Keep this as the canonical open license-compliance issue until maintainers choose an explicit policy for the WhatsApp plugin, then apply the selected mitigation narrowly across package metadata, lockfile/runtime dependency staging, docs, and changelog.
