## Summary

Keep open. Current main still contains the reported WhatsApp dependency path from `@openclaw/whatsapp` to Baileys and then to the pinned GPL-3.0 `@whiskeysockets/libsignal-node` tarball. The report is concrete and recent, but the mitigation requires maintainer/legal distribution judgment rather than conservative cleanup.

## What This Changes

The issue reports that the WhatsApp plugin pulls in a GPL-3.0 libsignal component and asks OpenClaw to address the license-infection risk.

## Best Possible Solution

Keep this as the canonical issue until maintainers choose an explicit license posture for the WhatsApp plugin, then apply the chosen mitigation narrowly: replace the dependency, isolate or disable the affected distribution path, or document an approved distribution model with matching package, lockfile, release, and docs updates.
