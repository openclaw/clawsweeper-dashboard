## Summary

Keep open: current main and the latest release still omit configured agent identity names from `sessions_send` A2A prompt context, and the direct closing PR remains open/draft rather than a merged fix.

## What This Changes

The issue requests adding `agents.list[].identity.name` to `sessions_send` message, reply, and announce A2A prompt contexts so receivers do not infer the wrong sender.

## Best Possible Solution

Land the linked fix PR, or a narrow replacement, that threads existing config into all A2A context builders, parses agent session keys without falling back to `main`, and covers message/reply/announce contexts with focused proof.

## Reproduction Assessment

Yes, as a source-level reproduction. Current main builds A2A context from session/channel fields only, while configured `agents.list[].identity.name` exists and is never passed to the context builders.

## Solution Assessment

Yes. The narrow maintainable fix is to thread existing config into the sessions_send A2A builders and resolve names from explicitly parsed agent session keys; the open closing PR is the current candidate but has not merged.
