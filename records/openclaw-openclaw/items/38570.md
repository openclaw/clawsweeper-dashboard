## Summary

Keep open. Current main still omits the requester's configured identity name from the `sessions_send` agent-to-agent message, reply, and announce prompt contexts, while the related implementation PR #38579 is closed unmerged and #60911 addressed adjacent UI sender display rather than receiver prompt attribution.

## What This Changes

Request that inter-agent `sessions_send` prompt contexts include the sender agent's configured `agents.list[].identity.name` so the receiving agent can identify the requester without inferring from the session key.

## Best Possible Solution

Add a small owner-module fix that resolves the requester agent id from the effective requester session key, looks up the configured identity name, and threads that optional name through all three A2A prompt builders without changing routing, delivery, visibility policy, or config schema.
