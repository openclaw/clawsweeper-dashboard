## Summary

Current main still omits the requester agent identity from the sessions_send agent-to-agent prompt contexts. The related fix PR was closed unmerged, so the issue remains a narrow source-backed bug rather than a cleanup close.

## What This Changes

Include the requester agent's configured `agents.list[].identity.name` in the initial, reply, and announce sessions_send A2A prompt contexts.

## Best Possible Solution

Thread the optional requester identity name through the existing sessions_send A2A helper boundary, add regression coverage for all three prompt contexts, and leave routing, delivery, visibility, and config shape unchanged.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current main has no parameter or call-site plumbing capable of emitting `Agent 1 (requester) name`, so a configured requester identity cannot appear in the A2A prompt.

## Solution Assessment

Yes. The narrow fix is to resolve the requester agent id from the effective requester session, read the existing configured identity name, and pass it into the current prompt builders rather than changing session routing or requiring workspace workaround tables.

## Repair Work Prompt

Fix the sessions_send A2A requester attribution bug tracked by https://github.com/openclaw/openclaw/issues/38570. Current main builds the initial, reply, and announce A2A prompts without the requester agent's configured `agents.list[].identity.name`, causing receiver agents to infer the sender from the session key. Implement a narrow fix in the sessions_send tool path: resolve the requester agent id from the effective requester session key, read the configured identity name with existing agent config/identity helpers, pass an optional requester name into `buildAgentToAgentMessageContext`, `buildAgentToAgentReplyContext`, and `buildAgentToAgentAnnounceContext`, and emit `Agent 1 (requester) name: <name>.` only when a non-empty configured name exists. Preserve existing fallback behavior when no name is configured. Do not change routing, delivery, visibility policy, session key semantics, UI sender rendering, or config schema. Add/update focused regression tests proving the name appears in the initial target prompt, ping-pong reply prompt, and final announce prompt, and that no name line is emitted when absent. Include a single-line CHANGELOG.md fix entry because this is user-facing.
