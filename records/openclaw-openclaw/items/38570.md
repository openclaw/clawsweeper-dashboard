## Summary

Keep open. Current main still omits configured agent identity names from the `sessions_send` A2A message, reply, and announce contexts, and the linked closing PR is open/draft with an unresolved review finding and missing real behavior proof.

## What This Changes

The issue asks for inter-agent `sessions_send` prompts to include the requester’s configured `identity.name` so receiving agents do not attribute the sender to the wrong agent.

## Best Possible Solution

Land the linked fix PR, or a narrow replacement, that threads existing config into all A2A context builders and emits identity-name lines only when the agent id is parsed from the session key with regression coverage.

## Reproduction Assessment

Yes. Source inspection on current `main` shows every A2A context builder only receives and emits session/channel fields, so a configured requester `identity.name` cannot appear in the generated prompt.

## Solution Assessment

Yes. The best fix is a narrow config-threading change in the existing sessions_send A2A prompt helpers, with explicit parsed-session-key behavior and tests for message, reply, and announce contexts.
