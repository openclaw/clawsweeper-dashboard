## Summary

Keep open: current main still omits the requester agent's configured identity name from `sessions_send` A2A message, reply, and announce prompts, and the directly related fix PRs were closed unmerged. The broader UI sender-label request remains separately tracked, but it does not cover this model-facing prompt-context bug.

## What This Changes

Add the requester agent's configured `identity.name` to `sessions_send` A2A message, reply, and announce context so receiving agents can identify the sender.

## Best Possible Solution

Resolve the requester identity from the existing session/config context and add an optional requester-name line to A2A message, reply, and announce prompts, with fallback behavior unchanged when no trustworthy name is available.
