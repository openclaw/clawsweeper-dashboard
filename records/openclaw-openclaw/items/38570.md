## Summary

Keep open. Current main still emits only session/channel lines in the A2A contexts, and the direct closing PR is open/draft with review/proof blockers, so this issue remains the canonical tracker.

## What This Changes

Add the configured requester identity name to sessions_send agent-to-agent message, reply, and announce context so receiving agents do not infer the sender from the session key.

## Best Possible Solution

Land the linked fix PR, or a narrow replacement, that threads existing config into all A2A context builders and emits identity-name lines only when the agent id is parsed from the session key, with regression coverage and real A2A proof.

## Reproduction Assessment

Yes. Source inspection on current main shows every A2A context builder only receives and emits session/channel fields, so a configured requester `identity.name` cannot appear in the generated prompt.

## Solution Assessment

Yes. The narrow maintainable fix is to thread existing config into the sessions_send A2A prompt helpers, parse agent session keys explicitly, and skip name emission when parsing fails.
