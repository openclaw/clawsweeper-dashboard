The issue requests adding the sender agent's configured `identity.name` to `sessions_send` A2A message, reply, and announce prompt contexts so receivers do not infer the wrong persona.

## Best Possible Solution

Land a narrow fix through https://github.com/openclaw/openclaw/pull/91271 or a replacement that resolves requester identity once from config, preserves prompt-cache session placeholders, and covers initial, reply, and announce contexts.

## Solution Assessment

Yes, the linked fix direction is narrow if rebased. Resolving the sender name from existing agent config and threading it through the existing A2A prompt builders is more maintainable than routing changes, new config, or agent-memory workarounds.
