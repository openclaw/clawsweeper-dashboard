## Summary

Keep open. Current main still omits the configured requester identity name from the `sessions_send` A2A message, reply, and announce prompt contexts; the matching implementation PR is closed unmerged, and the closed related UI-attribution issue covered a different surface.

## What This Changes

Include the sending agent's configured `identity.name` in `sessions_send` inter-agent prompt context so the receiver can identify the requester reliably.

## Best Possible Solution

Thread an optional requester identity name through the existing A2A prompt builders, resolve it from the effective requester session key and current config, and cover the initial, reply, and announce prompts with regression tests.

## Solution Assessment
