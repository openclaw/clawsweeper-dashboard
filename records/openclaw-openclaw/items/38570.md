## Summary

Keep open: current main still cannot include configured identity names in A2A prompt context, and the linked closing PR remains open/draft rather than a merged fix.

## What This Changes

The issue asks for `sessions_send` A2A prompt contexts to include the requester’s configured agent identity name so receivers do not infer the wrong sender from the session key.

## Best Possible Solution

Land the linked fix PR, or a narrow replacement, that threads existing OpenClaw config into all A2A context builders, parses agent session keys without defaulting to `main`, and covers message/reply/announce contexts plus real A2A proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `buildAgentSessionLines` emits only session/channel fields and its callers do not pass config, so a configured `agents.list[].identity.name` cannot appear in the generated prompt.

## Solution Assessment

Yes. The maintainable fix is a narrow sessions_send helper/caller change that passes config and emits names only after explicit agent-session-key parsing; the linked PR is the current candidate but still needs review repair and proof.
