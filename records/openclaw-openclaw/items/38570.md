## Repair Work Prompt

Fix the `sessions_send` requester attribution bug from https://github.com/openclaw/openclaw/issues/38570. Current main builds A2A message/reply/announce prompt contexts without the requester agent's configured `identity.name`, so receiving agents only see the requester session key/channel and can misidentify the sender. Implement a narrow core fix: derive the requester agent id from the existing requester session key, look up `cfg.agents.list[].identity.name`, and pass an optional requester name into `buildAgentToAgentMessageContext`, `buildAgentToAgentReplyContext`, and `buildAgentToAgentAnnounceContext`. Emit `Agent 1 (requester) name: <name>.` only when a non-empty configured identity name exists; preserve old output when it does not. Do not change routing, delivery, session visibility, input provenance schema, UI sender display, or agent selection behavior. Use PR #38579 as useful prior art but produce a current-main patch. Add regression coverage for initial send, reply, and announce prompts, plus fallback/no-name behavior. Add a user-facing changelog fix entry with appropriate human credit if known.
