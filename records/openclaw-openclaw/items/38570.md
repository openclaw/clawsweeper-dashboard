## Summary

## What This Changes

The issue asks for `sessions_send` A2A prompts to include the requester agent's configured `identity.name` so receiving agents do not infer the wrong sender.

## Best Possible Solution

Review, rebase, prove, and land or replace https://github.com/openclaw/openclaw/pull/91271 with a focused helper-boundary fix that preserves prompt-cache placeholders and covers the initial, reply, and announce A2A prompt phases.

## Solution Assessment

Yes: resolving the requester name from existing agent config and threading it through the shared prompt builders is narrower than changing routing, adding config, or relying on workspace-memory workarounds.
