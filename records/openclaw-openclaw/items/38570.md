## Summary

Keep open: current main still only supplies requester session/channel in the sessions_send A2A prompt context, while the linked fix PR #38579 was closed unmerged. The reported change is narrow and still fits the current config model for per-agent identity names.

## What This Changes

Request that sessions_send include the sender agent's configured identity.name in the initial message, reply, and announce A2A contexts.

## Best Possible Solution

Resolve the requester agent id from the requester session key, look up the configured identity.name from the resolved OpenClaw config, and add an optional requester name line to all A2A prompt contexts with focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes, from source inspection: configure a requester agent with identity.name, call sessions_send to another agent, and inspect extraSystemPrompt; current main has no path that can emit Agent 1 requester name. I did not run a live gateway flow, but the builder/call-site gap is direct and deterministic.

## Solution Assessment

Yes. The narrow maintainable fix is to thread an optional requester identity name through the existing sessions_send A2A prompt builders, preserving the current fallback when no configured name exists.
