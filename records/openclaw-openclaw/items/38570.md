## Summary

Current main still omits the requester identity name from all three A2A prompt contexts, and the closed direct fix PR remains unmerged. The issue is valid, narrow, and source-reproducible, so it should stay open as a ClawSweeper repair candidate.

## What This Changes

The issue asks for `sessions_send` A2A prompt context to include the requester agent's configured `identity.name` so receiving agents do not infer the wrong sender.

## Best Possible Solution

Thread an optional requester identity name from existing agent config into the three A2A prompt builders, with a safe fallback that omits the line when the requester session cannot be parsed or has no configured name.

## Reproduction Assessment

Yes, source-reproducible. Configure two agents with a requester `identity.name`, call `sessions_send`, and inspect the generated `extraSystemPrompt`; current main has no code path that can include that name in the A2A prompt context.

## Solution Assessment

Yes. The narrow maintainable fix is prompt-context plumbing plus targeted regression tests; broader durable UI sender identity and sender-authentication work should remain with the separate related issues.
