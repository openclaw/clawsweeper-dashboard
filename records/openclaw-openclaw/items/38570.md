## Summary

Keep open. Current main still omits configured agent identity names from `sessions_send` A2A message, reply, and announce contexts, and the linked closing PR is open/draft with review blockers, so this issue should remain open until that fix or a replacement lands.

## What This Changes

The issue asks for `sessions_send` inter-agent prompt context to include the configured requester identity name so receiving agents do not infer the wrong sender from the session key.

## Best Possible Solution

Review and land the linked fix PR, or a narrow replacement, that threads existing config into all A2A context builders and emits identity-name lines only for parsed agent session keys with regression coverage.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: the current A2A context helpers and their callers have no config or identity-name path, so a configured requester identity cannot appear in the generated prompt.

## Solution Assessment

Yes. The narrow maintainable solution is to thread existing configured identities into the shared A2A prompt context for message, reply, and announce paths without changing routing or adding config.
