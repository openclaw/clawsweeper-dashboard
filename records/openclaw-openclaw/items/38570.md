Keep this issue open: current main and the latest release still omit the requester agent's configured identity name from the shared sessions_send A2A prompt context, and https://github.com/openclaw/openclaw/pull/91271 is the active but unmerged fix path.

## What This Changes

The issue asks for sessions_send agent-to-agent prompt context to include the requester agent's configured identity name so receivers do not infer the wrong sender.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/91271 with a focused fix that resolves the requester identity from the requester session key, threads optional identity.name through all three A2A prompt builders, preserves session-key placeholders, and adds regression plus real multi-agent proof.

## Reproduction Assessment

Yes at source level: current main's initial, reply, and announce A2A prompt builders include requester session/channel metadata but no configured requester identity name. I did not run a live multi-agent sessions_send exchange in this read-only review.

## Solution Assessment

Yes, the helper-boundary fix is the best direction after rebase: resolve the requester name from existing agent config once and thread it through the shared prompt builders instead of changing routing, config, or relying on agent memory workarounds.
