## Summary

Current main and v2026.6.5 still omit the requester’s configured identity name from the canonical agent-to-agent prompt contexts, so the attribution bug remains. The viable closing implementation is the open pull request at https://github.com/openclaw/openclaw/pull/91271, which should remain paired with this issue until its real-behavior proof gate is satisfied and it merges.

## What This Changes

Include the sending agent’s configured identity name in the initial, reply, and announce contexts produced by sessions_send.

## Best Possible Solution

Resolve the requester agent from the existing session/config data, conditionally add its configured identity name through the shared session-line helper for all three A2A prompt shapes, preserve current output when no name is configured, and land that focused change after a redacted live multi-agent run confirms the receiving context or behavior.
