## Summary

Keep open: current main and the latest release still omit configured agent identity names from sessions_send A2A prompt context, and the direct closing PR is open/draft rather than merged.

## What This Changes

The issue reports that agent-to-agent sessions_send prompt context omits configured requester identity names, causing receivers to attribute messages to the wrong agent.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/82949, or a narrow replacement, after fixing the session-key fallback issue and proving real A2A sessions_send behavior.

## Reproduction Assessment

Yes, as a source-level reproduction. Current main builds A2A context from session/channel fields only, while configured agents.list[].identity.name exists and is not passed to the context builders.

## Solution Assessment

Yes for the issue direction. The maintainable fix is to thread existing config into all A2A context builders and resolve names from explicitly parsed agent session keys without defaulting unresolved senders to main.
