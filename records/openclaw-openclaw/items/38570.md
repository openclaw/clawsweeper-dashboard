## Summary

Keep open: current main and the latest shipped release still omit configured agent identity names from `sessions_send` A2A prompt context, and the direct closing PR remains open, draft, and unmerged.

## What This Changes

The issue asks for `sessions_send` A2A message, reply, and announce context to include the configured requester identity name instead of requiring receivers to infer it from a session key.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/82949, or a narrow replacement, after it threads existing config into all A2A context builders, parses agent session keys without defaulting unresolved keys to `main`, and includes focused tests plus real A2A proof.

## Reproduction Assessment

Yes. Source inspection on current main shows the shared A2A helper emits only session/channel fields and its callers do not pass config, so configured `agents.list[].identity.name` cannot appear in generated message, reply, or announce context.

## Solution Assessment

Yes in direction. The narrow maintainable fix is to pass existing config into the sessions_send A2A context builders and emit identity names only when the agent id is explicitly resolved from the session key; the linked PR is the active candidate but is not merged yet.
