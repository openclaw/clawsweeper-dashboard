## Summary

Keep open: current main and the latest release still omit the configured requester identity name from the `sessions_send` agent-to-agent prompt context, and the open closing PR is the right implementation path to review or replace rather than closing this issue early.

## What This Changes

The issue requests adding the requester's configured `agents.list[].identity.name` to the initial, reply, and announce `sessions_send` agent-to-agent contexts so receiving agents can identify the sender without inferring from the session key.

## Best Possible Solution

Review, rebase, and land or narrowly replace https://github.com/openclaw/openclaw/pull/91271 so the requester agent id is resolved from the session key and the configured identity name is threaded through all three A2A prompt contexts without changing routing, visibility, or delivery behavior.

## Reproduction Assessment

Yes, at source level: current main builds the A2A prompt context from session/channel fields only, and the callers do not pass config or the requester identity name into the context builders. I did not run a live multi-agent exchange.

## Solution Assessment

Yes, the best current path is a narrow fix or replacement for the linked PR that resolves the requester from the existing session key and reads the existing configured identity name. Closing the issue before that PR lands would be premature.
