## Summary

Keep open: current main still limits light dreaming to daily memory files, redacted session transcripts, and existing recall entries, while workspace file-creation events remain outside the promotion path. This is a useful memory feature request, but it needs a maintainer product/API decision because the clean fix likely adds an opt-in workspace-event source with safety policy rather than repairing a documented broken behavior.

## What This Changes

Request an opt-in dreaming light-phase source that records workspace file-creation events so new tools, config files, or data files can become memory promotion candidates.

## Best Possible Solution

Design an opt-in workspace-event memory source/API with allowlisted paths, redaction, scoring, and docs/tests before wiring it into light dreaming and deep promotion.

## Reproduction Assessment

Yes. Source inspection on current main shows light dreaming only feeds from daily memory, session transcripts, and existing recall entries; no live workspace run was performed in this read-only review.

## Solution Assessment

Unclear. A workspace.watchPaths-style source is plausible, but the best fix needs maintainer agreement on memory-source ownership, safety filters, and whether to extend memory/plugin APIs instead of adding a broad core watcher.
