## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Do not reuse a settled one-shot GatewayClient:** `src/gateway/call.ts:58-59`
  - body: `GatewayClient` stores `onHelloOk` and `onClose` in constructor options, so returning an existing live client reuses callback closures from the first `callGateway` promise. A later same-key call can time out or drive the wrong settled closure instead of sending its own method.
  - confidence: 0.94
- **[P1] Include the full gateway context in the pool key:** `src/gateway/call.ts:53-54`
  - body: The key only includes method/token/password, but the client is constructed with URL, TLS fingerprint, scopes, device identity, approval runtime token, mode/name, instance id, and protocol bounds. Reuse across any of those differences can route a request through the wrong target or authorization context.
  - confidence: 0.94
- **[P1] Preserve abort and accepted-run forwarding:** `src/gateway/call.ts:711-714`
  - body: Current main forwards `signal` and `onAccepted` into `GatewayClient.request`, and `agent-via-gateway` uses the exported `GatewayRequestFunction`/abort hook to stop accepted runs on process signals. The PR drops that contract, so interrupted gateway agent runs can keep running.
  - confidence: 0.92
- **[P1] Do not leave per-call CLI clients open indefinitely:** `src/gateway/call.ts:74-79`
  - body: `releasePooledClient` is a no-op, so a one-shot CLI call can leave WebSockets, tick timers, and reconnect loops alive after its promise resolves. Current main explicitly stops the active client before settling.
  - confidence: 0.9
- **[P1] Retain cleanup-root realpath guards:** `src/commands/doctor/shared/plugin-dependency-cleanup.ts:63-68`
  - body: The branch walks symlinked plugin entries and later removes collected targets recursively. Current main rejects extension/plugin roots that realpath outside OpenClaw cleanup roots; dropping that guard can delete external plugin directories through a symlink.
  - confidence: 0.91
- **[P2] Actually enforce the pool size cap:** `src/gateway/call.ts:62-71`
  - body: When the pool is at capacity and all entries are still live, this loop deletes nothing and then adds the new client anyway. Diverse method/auth combinations can therefore grow past the advertised cap.
  - confidence: 0.88
