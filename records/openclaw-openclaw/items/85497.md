- **[P1] Restore cleanup target validation before recursive removal:** `src/commands/doctor/shared/plugin-dependency-cleanup.ts:151-156`
  - body: Current main validates cleanup targets before `fs.rm`, including parent-segment checks, expected names, realpath containment, and symlink rejection. This branch removes that guard path and deletes every collected target, so `OPENCLAW_PLUGIN_STAGE_DIR` or symlinked extension roots can point `doctor --fix` at arbitrary directories.
  - confidence: 0.94
- **[P1] Include the full connection identity in the pool key:** `src/gateway/call.ts:53-54`
  - body: The pool key only includes method plus token/password, but the `GatewayClient` is created with URL, TLS fingerprint, scopes, mode, device identity, approval token, and protocol bounds. A later same-method call can reuse a client connected to the wrong gateway or with the wrong scopes, which breaks auth and remote routing boundaries.
  - confidence: 0.91
- **[P1] Do not keep one-shot gateway clients alive indefinitely:** `src/gateway/call.ts:74-79`
  - body: `releasePooledClient` is a no-op, while `GatewayClient` owns a WebSocket, reconnect timer, and tick interval. Existing `callGateway` stops the client after each wrapper request; this change can keep CLI processes alive after a successful one-shot call and the size cap does not evict live entries.
  - confidence: 0.88
- **[P1] Preserve abort and accepted callback forwarding:** `src/gateway/call.ts:711-714`
  - body: Current main forwards caller `signal`, `onAccepted`, and `onSignalAbort` so `openclaw agent` can learn the accepted run id and abort the active gateway run on SIGINT/SIGTERM. The PR branch no longer exposes or forwards those options, so interrupted agent runs can keep running server-side or fail to compile when rebased.
  - confidence: 0.9
