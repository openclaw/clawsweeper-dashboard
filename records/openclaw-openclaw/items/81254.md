## Summary

Keep open. Current `main` and the latest release still register the Windows node service through a direct `node.cmd` Scheduled Task target with interactive logon flags, and open linked PRs explicitly target this bug, so the issue should close only after a fix lands.

## What This Changes

The issue reports that `openclaw node install` on Windows creates an `OpenClaw Node` Scheduled Task that leaves a persistent visible `openclaw-node` command window at login instead of running as a background user service.

## Best Possible Solution

Land or replace the linked Windows node launcher fix so the Scheduled Task and `openclaw node start` path stay hidden while preserving status, stop, restart, parser, and fallback behavior, then close this issue after Windows proof.

## Reproduction Assessment

## Solution Assessment
