## Summary

Keep open. Current main and the latest release still route the Windows node service through the shared scheduled-task installer that runs `node.cmd` directly, and the linked fixing PRs are still open rather than merged.

## What This Changes

The issue reports that Windows `openclaw node install` registers a scheduled task that leaves a persistent visible command window instead of running the node host in the background.

## Best Possible Solution

Review and land a scoped node-specific hidden launcher fix, preserving gateway scheduled-task behavior and existing node status/start/stop semantics.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: the node service injects `OpenClaw Node` plus `node.cmd`, the generated `.cmd` runs the long-lived process directly, and the scheduled task registers that script with the interactive task path; no live Windows login was run in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is in the Windows scheduled-task launcher path for node installs, and because open closing PRs already exist, the issue should stay open until one of those fixes lands.
