## Summary

Keep open. Current main still routes Windows node service installs through `node.cmd` and the shared `schtasks /TR <script> ... /IT` path while the docs promise a background node service; the linked draft pull request https://github.com/openclaw/openclaw/pull/81267 is the right resolution path but is unmerged.

## What This Changes

Make Windows `openclaw node install` and `openclaw node start` run the node service without leaving a visible persistent `openclaw-node` command window.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81267 or an equivalent narrow node-only hidden-launcher fix after real Windows Scheduled Task proof, while preserving `node.cmd` for status/restart/script inspection.

## Reproduction Assessment

Yes. The reporter supplied concrete Windows 11 steps, and source inspection shows node installs still target `node.cmd` through the interactive scheduled-task path; I did not run a live Windows sign-in in this read-only review.

## Solution Assessment
