## Summary

Keep open. This is a fresh latest-release Windows report with a clear source-backed path: current main still installs the node host through the shared Windows Scheduled Task path that directly runs `node.cmd` with an interactive task option, while the docs describe `openclaw node install` as a background service. A prior exact report was closed stale, so the new 2026.5.7 confirmation should become the active tracking item rather than be swept away.

## What This Changes

The issue asks for `openclaw node install` on Windows to run the `OpenClaw Node` Scheduled Task invisibly as a background user service after login.

## Best Possible Solution

Fix the Windows node service install path so the managed node host starts invisibly at login while preserving service status, start, stop, restart, and Startup-folder fallback behavior.

## Reproduction Assessment

Yes, we have a high-confidence source-backed reproduction path, though not a local Windows live repro in this read-only review. Current main maps `openclaw node install` to `OpenClaw Node`/`node.cmd`, creates an ONLOGON task with `/IT`, and the reporter reproduced that exact task as a persistent visible cmd window on Windows 11 with 2026.5.7.

## Solution Assessment

Yes, the best fix belongs in the Windows daemon service path, not in docs or user workaround guidance. The likely repair is to keep the canonical service contract but launch the node task through a hidden/non-console wrapper or otherwise avoid an interactive visible `.cmd` lifetime for the scheduled task.
