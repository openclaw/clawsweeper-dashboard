- `impact:session-state`: The issue is about preserving or reusing subagent session context instead of losing expert conversation state between spawns.

## Summary

Keep open. Current main and the latest shipped release still lack native `sessions_spawn` reuse for expert subagents: native run mode creates a fresh UUID-backed child session, native resume remains ACP-only, and Microsoft Teams advertises thread capability but does not register the subagent thread-binding hook. The KeepMyClaw comment is a useful operational backup workaround, but it does not implement the requested OpenClaw native reuse contract.

## What This Changes

The issue requests a native or Teams-backed way for expert subagents to reuse an existing session instead of creating a new session on each `sessions_spawn(mode="run")`.

## Best Possible Solution

Define an explicit native subagent reuse contract, or implement a Teams thread-binding hook path, with scoped isolation, TTL/invalidation, concurrency locking, cleanup behavior, tool-schema/docs updates, and regression coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is an enhancement request. Source inspection gives a high-confidence missing-feature path: native run-mode spawns still create fresh UUID sessions, native resume is ACP-only, and Teams has no subagent thread-binding hook registration.

## Solution Assessment

Unclear. Registry-based reuse is plausible, but maintainers should first choose the boundary between core TTL reuse, a Teams `subagent_spawning` hook, and the existing explicit `sessions_send(sessionKey=...)` follow-up model.
