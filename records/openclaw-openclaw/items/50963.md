## Summary

Keep open. Current main still lacks native subagent session reuse for `sessions_spawn`, and Teams still has no `subagent_spawning`/conversation-binding path that would make persistent expert sessions work; the KeepMyClaw comment is useful operational backup context, but it does not replace an OpenClaw session-lifecycle feature.

## What This Changes

Request support for reusing an existing expert subagent session instead of creating a fresh native subagent session on each `sessions_spawn` call, especially for Teams.

## Best Possible Solution

Keep this open until maintainers choose and document an explicit persistent or reusable native subagent path, either through a Teams plugin hook using existing seams or a core `sessions_spawn` reuse contract with lifecycle, cleanup, and routing tests.

## Reproduction Assessment

Yes. Source inspection on current main shows native `sessions_spawn` creates a fresh random subagent session, and persistent `mode="session"` is gated on thread-binding hooks that Teams does not register.

## Solution Assessment

Unclear. The requested behavior is valid, but the best fix depends on maintainer choice between a Teams-specific `subagent_spawning` hook and a broader core reuse/TTL contract.
