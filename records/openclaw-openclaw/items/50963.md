## Summary

Keep open: current main still lacks native subagent session reuse, while the existing resume path is ACP-only and Discord/Feishu thread hooks do not cover Teams. The remaining work is a product/API decision around isolation, TTL, locking, cleanup, and channel behavior, not a strict bug-fix cleanup item.

## What This Changes

The issue requests reusable subagent sessions so repeated expert-agent spawns can preserve context or avoid repeated cold starts.

## Best Possible Solution

Define an explicit native subagent reuse contract, or a Teams-compatible thread/session hook path, with clear isolation, liveness/TTL, locking, cleanup, docs, and tests before implementation.

## Reproduction Assessment

Yes for the source-level gap: native `sessions_spawn` still selects a fresh UUID child key and does not pass any native reuse or resume field. I did not run a live Teams/provider cost reproduction in this read-only review.

## Solution Assessment

No as a ready-to-implement direction: the request is valid, but adding a reuse parameter or automatic pooling before deciding ownership, isolation, liveness, and locking would be premature. The safer path is maintainer design first, then a focused implementation with tests.
