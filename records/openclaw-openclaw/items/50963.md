## Summary

Keep open. Current main and the latest release still use fresh UUID-backed native subagent sessions, keep `resumeSessionId` ACP-only, and have no native reuse/pooling contract; the remaining work needs a maintainer product decision before implementation.

## What This Changes

The issue requests reusable expert subagent sessions so repeated `sessions_spawn(mode="run")` calls can preserve expert-agent context instead of always starting a fresh child session.

## Best Possible Solution

Define an explicit native subagent reuse contract, then implement it in the core spawn path with documented isolation, TTL/liveness, locking, cleanup, and channel/thread behavior.

## Reproduction Assessment

Yes at the source level: native `sessions_spawn` still chooses a fresh UUID child session key and the schema/docs/tests keep native resume or reuse absent. I did not run a live Teams or provider-cost reproduction in this read-only review.

## Solution Assessment

Unclear. Core registry-based reuse is plausible and likely better than reviving per-channel hooks for `mode="run"`, but maintainers still need to choose isolation, TTL, locking, cleanup, and API shape before implementation.
