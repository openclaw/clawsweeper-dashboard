## Summary

Keep open. Current main still lacks native subagent session reuse: run mode creates a fresh UUID-backed child session, native resume remains ACP-only, persistent native sessions require thread binding, and Microsoft Teams has thread capability but no matching thread-binding/subagent hook surface. This is feature/API design work, not conservative cleanup.

## What This Changes

Request native expert subagents, especially on Microsoft Teams, to reuse an existing session instead of creating a fresh `sessions_spawn(mode="run")` child session each time.

## Best Possible Solution

Define an explicit native subagent reuse contract, or a Teams thread-binding implementation, with scoped isolation, TTL/invalidation, concurrency locking, cleanup, tool-schema/docs updates, and regression coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is an enhancement request. Source inspection gives a high-confidence missing-feature path: native run-mode spawns still create fresh UUID sessions and native resume remains ACP-only.

## Solution Assessment

Unclear. Registry-based reuse is plausible, but maintainers should first choose the boundary between core TTL reuse, a Teams `subagent_spawning` hook, and the existing explicit `sessions_send(sessionKey=...)` follow-up model.
