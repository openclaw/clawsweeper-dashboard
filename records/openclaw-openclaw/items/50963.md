## Summary

Keep open: current main and the latest release still create native run-mode subagent sessions with fresh UUID child keys, keep native resume/reuse absent, and document persistent subagent sessions as thread-binding-only. The KeepMyClaw comment is useful operational backup context, but it does not provide live Teams/native subagent session reuse.

## What This Changes

Request to let repeated expert-agent `sessions_spawn` calls reuse a native subagent session instead of always starting a fresh child session.

## Best Possible Solution

Define an explicit native subagent reuse or Teams-compatible session-binding contract, then implement the chosen seam with isolation, liveness/TTL, locking, cleanup, docs, and focused tests.

## Reproduction Assessment

Yes for a source-level reproduction: current main still chooses a fresh native child session key with `crypto.randomUUID()` and has no native reuse/resume field. I did not run a live Teams/provider-cost scenario in this read-only review.

## Solution Assessment

No immediate implementation path is clearly best yet: adding a Teams hook, adding a reuse key, or automatic pooling each changes isolation and lifecycle semantics. The best next step is maintainer agreement on the core/session-binding contract before a fix PR.
