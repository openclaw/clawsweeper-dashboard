## Summary

Keep open. Current main still lacks the requested automatic reuse path for native subagents: run mode creates a fresh child session key, native session mode depends on thread-binding hooks, and Microsoft Teams has thread capability but no matching hook/placement surface. This is feature/API design work, not conservative cleanup.

## What This Changes

Requests native subagent session reuse so repeated `sessions_spawn(mode="run")` calls, especially from Microsoft Teams, can continue an existing expert-agent session instead of minting a new one.

## Best Possible Solution

Design an explicit native subagent reuse contract, or a Teams thread-binding implementation, with scoped ownership, TTL/invalidation, auth/model/config isolation, tool schema/docs updates, and regression coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is an enhancement request. Source inspection confirms current main still creates a fresh native subagent session for each run-mode spawn and does not expose a native subagent reuse target.

## Solution Assessment

Unclear. Registry-based reuse is plausible, but maintainers should first choose the boundary between core TTL reuse, a Teams `subagent_spawning` hook, and the existing explicit `sessions_send(sessionKey=...)` follow-up model.
