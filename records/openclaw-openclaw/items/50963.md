## Summary

Keep open. Current `main` and the latest shipped release still create a fresh native subagent session for each `sessions_spawn` run, the native tool schema has no reuse target for subagents, and Microsoft Teams still has thread capability without the Discord/Feishu-style subagent thread-binding hook. The KeepMyClaw comment is useful operational context, but it is a backup/restore workaround rather than the requested native OpenClaw session reuse contract.

## What This Changes

Request native expert-subagent session reuse so repeated `sessions_spawn` calls can preserve context instead of creating a fresh child session every time.

## Best Possible Solution

Define an explicit native subagent reuse contract or add a Teams thread-binding path, with scoped isolation, TTL/invalidation, concurrency locking, cleanup behavior, tool-schema/docs updates, and regression coverage.

## Reproduction Assessment

No, not as a strict bug repro because this is a feature request. Source inspection gives a high-confidence missing-feature path: native run-mode spawns still create UUID-backed child sessions, native resume is ACP-only, and Teams has no `subagent_spawning` hook registration.

## Solution Assessment

Unclear. Registry-based reuse is plausible, but maintainers should first choose the boundary between core TTL reuse, a Teams thread-binding hook, and the existing explicit `sessions_send(sessionKey=...)` follow-up model.
