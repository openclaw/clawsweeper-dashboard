## Summary

Keep this open: current main still prunes expired sandbox runtimes and registry entries without deleting the corresponding Docker sandbox workspace directory. The target-repo fix attempts are closed unmerged, and the only open closing reference is a conflicting fork PR with the same risky cleanup shape.

## What This Changes

The issue asks for sandbox pruning to delete expired Docker sandbox workspace directories as well as containers and registry entries.

## Best Possible Solution

Add backend-aware workspace cleanup that removes only an expired, OpenClaw-managed, non-shared and unused workspace after confirmed main runtime removal, with regression coverage for deletion and preservation cases.

## Reproduction Assessment

## Solution Assessment

No for the direct recursive deletion shape from the closed PRs. The maintainable fix is a guarded backend-aware cleanup path with active-use, browser, agent-scope, and shared-workspace protections.
