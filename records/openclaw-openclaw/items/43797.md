## Summary

Keep open. Current main still prunes expired sandbox runtimes and registry entries without deleting the local non-shared workspace directory, and the linked open PR #43835 is already an implementation candidate for this issue.

## What This Changes

The issue requests that sandbox pruning clean up expired Docker sandbox workspace directories under the sandbox workspace root, not only the runtime container and registry entry.

## Best Possible Solution

Land or replace the existing guarded cleanup so prune and recreate remove only OpenClaw-owned, non-shared sandbox workspace directories after runtime removal, with focused regression coverage.

## Reproduction Assessment

Yes, at source level. An expired registry entry reaches the prune loop, which removes the runtime and registry entry while the separately created non-shared workspace directory remains untouched.

## Solution Assessment

No for current main; it does not yet solve the leak. The narrow maintainable fix is PR #43835 or an equivalent guarded cleanup in the sandbox lifecycle, not a new force-fresh mode or config option.
