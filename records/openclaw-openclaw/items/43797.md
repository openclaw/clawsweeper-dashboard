## Summary

Keep this open: current main still prunes expired sandbox runtimes and registry entries without deleting the corresponding Docker sandbox workspace directory, and the related target-repo fix PRs are closed unmerged while the only open closing reference is a conflicting fork PR.

## What This Changes

The issue requests that sandbox pruning remove expired sandbox workspace directories in addition to Docker runtimes.

## Best Possible Solution

Add backend-aware workspace cleanup that removes only an expired, OpenClaw-managed, non-shared and unused workspace after confirmed main runtime removal, with regression coverage for deletion and preservation cases.

## Reproduction Assessment

Yes, at source level. An expired Docker sandbox registry entry reaches `pruneSandboxContainers`, which removes the runtime and registry entry while the workspace created by `ensureSandboxWorkspaceLayout` has no deletion path.

## Solution Assessment

No for the direct recursive deletion shape from the closed PRs. The maintainable fix is a guarded backend-aware cleanup path with active-use and shared-workspace protections.
