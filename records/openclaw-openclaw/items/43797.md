## Summary

Keep open. Current main still prunes expired sandbox runtimes and registry entries without deleting the corresponding non-shared workspace directory, and PR #43835 is already an open implementation candidate linked with closing syntax.

## What This Changes

The issue asks for sandbox prune to remove expired OpenClaw-owned workspace directories as well as the sandbox runtime/container.

## Best Possible Solution

Land or replace PR #43835 with guarded cleanup that deletes only OpenClaw-owned, non-shared sandbox workspace directories after runtime removal, plus regression coverage.

## Reproduction Assessment

Yes, at source level. An expired sandbox registry entry reaches `pruneSandboxContainers`, which removes the runtime and registry entry while the workspace directory created by `ensureSandboxWorkspaceLayout` remains untouched.

## Solution Assessment

No for current main. The narrow maintainable solution is PR #43835 or an equivalent guarded cleanup in the sandbox lifecycle, not a new force-fresh mode or config option.
