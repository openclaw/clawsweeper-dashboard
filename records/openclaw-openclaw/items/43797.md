## Summary

Keep this open: current main and the latest release still remove expired sandbox runtimes and registry entries without deleting the non-shared sandbox workspace directory, and the open closing PR is not merged and currently conflicts.

## What This Changes

Reported that sandbox prune leaves per-scope workspace directories under `~/.openclaw/sandboxes`, causing disk growth and stale sandbox workspace state after Docker containers are removed.

## Best Possible Solution

Land a current-main fix that removes non-shared sandbox workspace directories after their owning sandbox runtime is pruned, preserves shared-scope workspaces, and adds focused regression coverage.

## Reproduction Assessment

Yes, from source inspection: an expired sandbox registry entry reaches backend runtime removal and registry deletion, but the Docker-backed sandbox workspace directory is never removed. I did not run a live Docker prune in this read-only review.

## Solution Assessment

No, current main is not the fix; it still leaves non-shared sandbox workspace directories behind. The narrow maintainable path is to repair or replace the linked closing PR with shared-scope safety and tests.
