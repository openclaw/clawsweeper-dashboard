## Summary

Keep open: current main still removes expired sandbox runtimes and registry entries without deleting the matching workspace directory, and open PR #43835 already targets this issue with closing syntax.

## What This Changes

Sandbox pruning should delete expired per-session or per-agent workspace directories under the sandbox workspace root after removing the associated runtime.

## Best Possible Solution

Review and land PR #43835 after rebasing it onto current main, or replace it with the same narrow sandbox-owned fix plus regression coverage if the branch cannot be updated.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: expired entries flow through `maybePruneSandboxes` to `pruneSandboxContainers`, which removes the runtime and registry entry but never deletes the resolved workspace directory.

## Solution Assessment

Yes in direction. The narrow fix belongs in the sandbox prune owner path, using existing scope/workspace helpers and preserving shared workspaces; the open PR should be validated against current main before landing.
