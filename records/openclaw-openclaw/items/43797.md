## Summary

Keep open. Current main still removes expired sandbox runtimes and registry entries without deleting the corresponding non-shared workspace directory, and PR #43835 is already an open implementation candidate for this issue.

## What This Changes

Sandbox prune should remove OpenClaw-owned non-shared workspace directories under `sandbox.workspaceRoot` when expired runtimes are removed.

## Best Possible Solution

Land or replace the existing guarded cleanup so prune and recreate remove only OpenClaw-owned, non-shared sandbox workspace directories after runtime removal, with focused regression coverage.

## Reproduction Assessment

Yes, at source level. An expired non-shared sandbox registry entry reaches `pruneSandboxContainers`, which removes the runtime and registry entry while the workspace directory created by `ensureSandboxWorkspaceLayout` remains untouched.

## Solution Assessment

No for current main; it does not yet solve the leak. The narrow maintainable path is PR #43835 or an equivalent guarded cleanup in the sandbox lifecycle, not a new force-fresh mode or config option.
