## Summary

Keep open: current main and v2026.5.12 still prune expired sandbox runtimes and registry entries without removing Docker workspace directories. The linked fork PR is a candidate, but prior cleanup attempts showed deletion-safety issues that need owner/product review before this should land.

## What This Changes

The issue requests that sandbox prune remove expired OpenClaw-managed workspace directories as well as their Docker runtimes.

## Best Possible Solution

Add guarded backend-aware prune cleanup that removes only expired OpenClaw-managed unused workspace state after confirmed runtime removal, with regression coverage for delete and preserve cases.

## Reproduction Assessment

Yes, at source level. An expired Docker sandbox registry entry reaches pruneSandboxContainers, which removes the runtime and registry entry while the workspace created by ensureSandboxWorkspaceLayout has no deletion path.

## Solution Assessment

No, current main does not solve the issue and the naive direct-deletion shape is not the safest fix. The maintainable direction is guarded backend-aware cleanup tied to stored workspace identity and active-use checks.
