The PR changes memory-core `memory_search` zero-hit handling so healthy QMD empty results skip the blocking forced sync while non-QMD dirty or unhealthy states still sync and retry.

## Best Possible Solution

Land one canonical memory-core fix that keeps QMD zero-hit update work out of the interactive `memory_search` wait, with this branch viable after green CI if maintainers accept the QMD freshness tradeoff.

## Reproduction Assessment

Yes. Current main and the latest release still await a forced sync after zero `memory_search` results, and QMD sync delegates to update maintenance that can exceed the interactive search budget.

## Solution Assessment

Yes as a narrow mitigation. The updated PR is in the right memory-core layer and preserves non-QMD dirty retry; bounded/background QMD refresh is the safer alternative only if maintainers require immediate freshness repair.
