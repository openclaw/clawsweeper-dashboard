## Summary

Keep open. Current main has fixed the unresolved SecretRef crash and the `agents list`/bare `agents` plugin-preload paths, including the landed #74220 work, but the reported `agents add --non-interactive` hang class is still exposed because `agents add` has no exact startup-policy override and inherits the parent `agents` plugin-load policy.

## What This Changes

The issue requests that `openclaw agents list` degrade gracefully when Telegram bot tokens are unresolved SecretRefs and that `openclaw agents` subcommands exit after completion instead of hanging when plugin loading leaves handles open.

## Best Possible Solution

Keep the shipped SecretRef and `agents list`/bare `agents` fixes, then add a focused follow-up so non-interactive `agents add` does not preload plugin runtimes before completing and exiting, while retaining any plugin-aware behavior needed by the interactive wizard.

## Reproduction Assessment

Yes for the remaining source-level exposure: current main resolves `agents add` through the parent `agents` policy, which still loads plugins. The exact hang still needs the reporter's plugin/runtime setup or an equivalent handle-leaking plugin to live-reproduce.

## Solution Assessment

No. The current partial solution is not complete because it leaves `agents add --non-interactive` on the broad plugin-preload path; the safer fix is a targeted startup-policy/lifecycle change with tests around non-interactive add and existing interactive behavior.
