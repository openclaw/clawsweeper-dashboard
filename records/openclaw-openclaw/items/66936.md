Keep open. Current main fixes the unresolved SecretRef crash and the `agents`/`agents list` plugin-preload path, but `agents add` still reaches startup as `['agents','add']` and inherits the broad parent plugin-load policy, so the reported hang class is not fully resolved.

## What This Changes

The issue reports that `openclaw agents list` crashes on unresolved Telegram SecretRef values and that `openclaw agents` subcommands can hang after completing when plugin runtimes leave handles open.

## Best Possible Solution

Keep the shipped SecretRef and list/bare-agents fixes, then make non-interactive config-only `agents add` avoid plugin runtime preload while preserving or explicitly testing interactive setup behavior.

Yes for the remaining source-level exposure path: current main has no exact no-plugin `agents add` policy, and startup keeps `agents add` as the evaluated command path. The full process hang still needs the reporter's plugin/runtime setup for live reproduction.

## Solution Assessment
