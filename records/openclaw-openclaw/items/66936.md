## Summary

Keep open. Current main fixes the unresolved SecretRef crash and the `agents`/`agents list` plugin-preload hang path, but `openclaw agents add ... --workspace ... --non-interactive` still reaches the broad parent `agents` startup policy that preloads plugins before the config-only add command runs.

## What This Changes

The issue reports that `openclaw agents list` crashes on unresolved Telegram SecretRefs and that `openclaw agents` subcommands can hang after completing.

## Best Possible Solution

Keep the shipped SecretRef and list fixes, then add a narrow argv-sensitive startup policy so config-only `agents add` invocations avoid plugin runtime preload while interactive setup keeps any plugin surfaces it needs.
