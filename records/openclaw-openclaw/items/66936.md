## Summary

Keep open. Current main has fixed the unresolved SecretRef crash and the `agents`/`agents list` plugin-preload paths, but the reported non-interactive `openclaw agents add ... --workspace ...` path still inherits the broad `agents` startup policy that preloads plugins before the config-only add flow runs.

## What This Changes

The issue asks for `openclaw agents list` to tolerate unresolved Telegram SecretRefs and for `openclaw agents` subcommands to exit after completion instead of hanging from plugin preload side effects.

## Best Possible Solution

Keep the shipped SecretRef and `agents list` fixes, then add a narrow argv-sensitive startup policy so config-only `agents add` invocations avoid plugin runtime preload while preserving any plugin-backed interactive setup surfaces.
