## Summary

Keep open. Current main has fixed the unresolved SecretRef crash and the `agents`/`agents list` plugin-preload hang paths, but `agents add` still falls through the broad parent startup policy before the non-interactive config-only add flow runs.

## What This Changes

The issue asks for `openclaw agents list` to tolerate unresolved Telegram SecretRefs and for `openclaw agents` subcommands to exit instead of hanging after completion.

## Best Possible Solution

Keep the shipped list/SecretRef fixes and add a narrow argv-sensitive no-plugin startup path for config-only `agents add` invocations while preserving interactive add setup behavior.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a live current-main hang in this read-only review; at source level, `agents add` still selects the broad `agents` plugin-preload policy before action-level flags choose the config-only flow.

## Solution Assessment

Is this the best way to solve the issue? Yes: the narrow maintainable path is to extend the existing CLI startup-policy tests and catalog so non-interactive/config-only `agents add` avoids plugin preload without changing interactive setup.
