## Best Possible Solution

Keep the shipped SecretRef and list/bare-agents fixes, then add a targeted startup-policy regression fix so config-only or non-interactive `agents add` avoids plugin runtime preload while interactive setup behavior remains intact.

## Repair Work Prompt

Fix the remaining `openclaw agents add` CLI hang exposure from https://github.com/openclaw/openclaw/issues/66936. Current main already handles unresolved SecretRef failures in `agents list` and routes bare `agents`/`agents list` without plugin preload; do not disturb those fixes. Add a narrow startup policy for the config-only/non-interactive `agents add` path so commands with explicit add flags such as `--workspace`, `--model`, `--agent-dir`, `--bind`, or `--non-interactive` do not preload plugin runtimes before the command completes. Preserve interactive `agents add` wizard/channel setup behavior; if avoiding preload there would change setup semantics, keep interactive loading and document the distinction in tests. Add/update regression coverage in the CLI startup/path/preaction tests and command tests, add a user-facing CHANGELOG.md fix entry, and stop if the implementation requires a broad plugin lifecycle rewrite or a new config option.
