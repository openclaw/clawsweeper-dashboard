## Summary

Keep open. Current `main` fixes the unresolved SecretRef crash and the `agents`/`agents list` plugin-preload hang path, but the reported non-interactive `agents add` path can still inherit the broad parent `agents` plugin preload policy.

## What This Changes

The issue reports that `openclaw agents list` crashes on unresolved Telegram SecretRef values and that `openclaw agents` subcommands can complete while leaving the CLI process alive.

## Best Possible Solution

Keep the shipped SecretRef/list fixes and add a narrow argv-sensitive startup policy so non-interactive `agents add` avoids plugin runtime preload while interactive setup keeps the plugin surfaces it needs.

## Reproduction Assessment

Yes for source-level exposure: current `main` resolves `openclaw agents add ...` through `['agents','add']`, which only matches the parent `agents` `loadPlugins: "always"` policy. I did not establish a live hanging process with the reported third-party plugin in this read-only review.

## Solution Assessment

No. The current implementation is partial: it handles SecretRef/list and bare-agent paths, but the narrow maintainable completion is an `agents add` startup-policy regression for non-interactive/config-only use plus a built CLI smoke with a handle-holding plugin.
