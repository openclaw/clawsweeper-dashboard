## Summary

Keep open. Current main fixes the unresolved SecretRef crash and the `agents`/`agents list` plugin-preload path, but `agents add` still has no exact no-plugin startup policy and can still inherit the broad parent `agents` plugin-load behavior that matches the remaining hang class.

## What This Changes

The issue reports that `openclaw agents list` crashes on unresolved Telegram SecretRef values and that `openclaw agents` subcommands can leave the CLI process running after completion.

## Best Possible Solution

Keep the shipped SecretRef and list/bare-agents fixes, then add a narrow startup-policy and regression-test fix so config-only/non-interactive `agents add` avoids plugin runtime preload while preserving interactive setup behavior.
