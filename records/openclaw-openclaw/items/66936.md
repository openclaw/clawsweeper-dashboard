## Summary

Keep open. Current main fixes the unresolved SecretRef crash and the `agents`/`agents list` plugin-preload hang path, but non-interactive `openclaw agents add ... --workspace ...` still reaches the broad parent `agents` startup policy that preloads plugins before the config-only add flow runs.

## What This Changes

The issue asks for `openclaw agents list` to tolerate unresolved SecretRefs and for `openclaw agents` subcommands to exit instead of hanging after completion.

## Best Possible Solution

Keep the shipped SecretRef and list fixes, then add a narrow argv-sensitive startup policy so config-only `agents add` invocations avoid plugin runtime preload while interactive setup keeps any plugin surfaces it needs.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for source-level exposure: current main derives `agents add` before action handling and has no exact no-plugin policy for that path. I did not establish a fresh live hanging process with the third-party plugin in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? No, the current implementation is still partial. The narrow maintainable fix is an `agents add` startup-policy regression fix with focused coverage and a built CLI smoke against a handle-holding plugin.
