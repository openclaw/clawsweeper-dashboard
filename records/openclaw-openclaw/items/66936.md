The issue asks for `openclaw agents list` to tolerate unresolved Telegram SecretRefs and for `openclaw agents` subcommands to exit cleanly after completing.

## Best Possible Solution

Keep the shipped SecretRef and list fixes, then add a narrow startup-policy regression fix so documented non-interactive `agents add` avoids plugin runtime preload and exits cleanly while interactive setup keeps the plugin behavior it needs.
