## Summary

Keep open. Current main fixes the unresolved SecretRef crash and the `agents`/`agents list` plugin-preload hang paths, but non-interactive `openclaw agents add ... --workspace ...` still inherits the broad parent `agents` policy that preloads plugins before the config-only add flow runs.

## What This Changes

The issue asks for `agents list` to tolerate unresolved Telegram SecretRefs and for `openclaw agents` subcommands to exit cleanly after completion.

## Best Possible Solution

Keep the shipped SecretRef and list fixes, then add a narrow argv-sensitive startup policy so config-only `agents add` invocations avoid plugin runtime preload while interactive setup preserves any plugin-backed surfaces it needs.
