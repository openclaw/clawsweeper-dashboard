## Summary

Keep open: current main still drops the gateway freshness signal before the TUI footer state merge, while this PR is a focused, proof-backed fix for that remaining TUI surface. The merged `/status` fix covers the sibling status path but does not supersede this PR.

## What This Changes

The PR adds `totalTokensFresh` to TUI session info and maps fresh-but-absent totals to `0` so the TUI footer shows `0/limit` instead of `?/limit` for fresh sessions.

## Best Possible Solution

Land the narrow TUI state propagation, preserving `?` only for stale or genuinely unknown totals and keeping the gateway as the freshness source of truth.

## Reproduction Assessment

Yes at source level: current main emits `totalTokensFresh` from gateway session rows, but the TUI state model does not carry it and clears missing `totalTokens` to `null`. I did not run a live TUI repro in this read-only review, but the PR body supplies after-fix terminal proof for fresh and stale TUI payloads.

## Solution Assessment

Yes: the PR fixes the TUI-owned session-info merge boundary and uses the existing gateway freshness contract. A shared formatter fallback would be worse because it could hide stale or genuinely unknown totals.
