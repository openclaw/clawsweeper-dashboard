## Summary

Keep open: current main and the latest release still contain the exact WhatsApp credential-clear path and log line described by the report, and targeted searches did not find a canonical issue that owns this specific plugin-update session wipe.

## What This Changes

The issue reports that updating the WhatsApp plugin from 2026.6.1 to 2026.6.5 deletes Baileys credential files and forces a QR re-pair.

## Best Possible Solution

Preserve linked WhatsApp auth across plugin updates, and only clear credentials after an explicit logout or a verified WhatsApp-owned migration/terminal-state decision with backup and tests.

## Reproduction Assessment

Yes for the destructive credential-clear path from source: current main emits the exact reported log line after removing the WhatsApp auth directory. No for the full user flow: this read-only review did not run a real `openclaw plugins update whatsapp` Docker/ClawHub repro.

## Solution Assessment

Unclear. Preserving auth across plugin updates is the right end state, but changing when a WhatsApp `loggedOut` or conflict event may delete credentials needs maintainer product/security judgment.
