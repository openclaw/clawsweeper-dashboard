## Label Justifications

- `impact:data-loss`: The report is about deletion of persisted Baileys credential files without backup or warning.

- `impact:crash-loop`: The reporter observed the gateway entering an auto-restart crash loop after the credentials were cleared.

## Summary

Keep open: current main and the latest release still delete managed WhatsApp Baileys auth state through `logoutWeb()` with no pre-logout backup or restore path, and the targeted open fix PR is not merged yet.

## What This Changes

The issue reports that updating the WhatsApp plugin can trigger terminal WhatsApp Web cleanup that silently removes Baileys session credentials and forces a full QR re-pair.

## Best Possible Solution

Land a WhatsApp-owned fix that preserves or safely backs up Baileys credentials before terminal cleanup during plugin-update-triggered restarts, with failure-visible logging, focused tests, and real update-path proof.

## Reproduction Assessment

Yes at source level: current main reaches `logoutWeb()` from terminal WhatsApp reconnect/setup paths and deletes the managed auth directory. I did not run a live WhatsApp plugin-update repro in this read-only review.

## Solution Assessment

Yes, the best narrow direction is a WhatsApp-plugin fix that prevents unrecoverable credential loss during terminal cleanup; the open PR is the right work lane but still needs repair and proof before this issue can close.
