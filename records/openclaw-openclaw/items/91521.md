## Summary

Keep open: current main and v2026.6.9 still make tokenless Tailscale Serve auth a Control UI WebSocket-only path, while assistant-media metadata is an HTTP read route that needs bearer/query/device auth before it can mint a media ticket.

## What This Changes

The issue reports that Control UI over Tailscale Serve with default token auth can load the UI but cannot load assistant-generated images from `/__openclaw__/assistant-media`.

## Best Possible Solution

Give verified Tailscale Control UI sessions a narrow scoped way to obtain assistant-media metadata or media tickets without making Tailscale identity a general HTTP API credential.

## Reproduction Assessment

Yes at source level, but not live: tokenless Tailscale WebSocket auth can open Control UI, while assistant-media metadata is an HTTP route and the UI sends bearer auth only when a token-like credential exists.

## Solution Assessment

No automatic broad HTTP-auth change is safe here; the reporter's trusted-proxy workaround is valid, but the product fix should be a narrower maintainer-approved media credential or ticket path.
