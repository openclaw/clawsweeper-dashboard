## Summary

Keep open: current main and v2026.6.8 still route tokenless Tailscale Control UI assistant-media metadata through HTTP auth, so the reported image-rendering failure remains unresolved and auth-boundary-sensitive.

## What This Changes

The issue reports that Control UI assistant-media image URLs fail under token auth when accessed through Tailscale Serve.

## Best Possible Solution

Give verified Tailscale Control UI sessions a narrow scoped credential or media-ticket bootstrap for protected Control UI media routes without making Tailscale identity satisfy general HTTP APIs.

## Reproduction Assessment

Yes at source level, but not live: tokenless Tailscale WebSocket auth can open Control UI, while assistant-media metadata is a token-authenticated HTTP route and the UI only sends a bearer token when one is available.

## Solution Assessment

No automatic broad HTTP-auth change is safe here. The reporter's trusted-proxy workaround is valid, but the product fix should be a narrower maintainer-approved media credential or ticket path.
