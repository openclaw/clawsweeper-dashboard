## What This Changes

This PR changes exec-approvals directory validation to follow a terminal state-directory symlink with `fs.statSync` and adds a POSIX regression test for `OPENCLAW_HOME` and `OPENCLAW_STATE_DIR` pointing at the same symlink.

## Best Possible Solution

Support the documented deployment topology through a narrowly validated terminal-symlink policy that preserves parent, destination-file, hardlink, and migration safeguards, then add redacted real exec proof.

## Reproduction Assessment

Yes, from source inspection. Current main resolves `OPENCLAW_STATE_DIR` into the exec approvals path and then rejects a terminal symlink via `lstatSync`, matching the failure the PR’s new regression test describes.

## Solution Assessment

No, not as submitted. Switching to `statSync` fixes the reported topology but also accepts unvalidated terminal symlinks outside the trusted root; the better fix is a narrower validated symlink allowance with explicit rejected-case coverage.
