## Summary

Keep open: current main does not provide a per-call private realtime voice-call objective separate from the spoken opener, and the linked outbound task-call tracker is a broader open feature rather than a merged or superseding implementation.

## What This Changes

The issue requests a per-call private realtime voice-call objective or brief for outbound realtime calls, separate from the first spoken `message`.

## Best Possible Solution

Design a per-call objective/brief contract for the voice-call plugin, store it as call metadata, and inject it into realtime session instructions plus realtime consult context without changing `message` as the spoken opener.

## Reproduction Assessment

Yes from source inspection: the current CLI, tool schema, Gateway RPC, outbound call options, and realtime config do not accept a private per-call objective separate from `message`. This is a confirmed missing feature surface rather than a failing bug reproduction.

## Solution Assessment

No patch is proposed yet. The requested direction is plausible, but the best implementation needs maintainer API/product approval before adding the new public flag/field and private-context handling.
