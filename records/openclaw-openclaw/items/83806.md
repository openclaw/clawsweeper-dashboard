## Summary

Close as duplicate/superseded: the requested per-call private realtime objective is a narrow slice of the already-open outbound task-call tracker, and current main confirms Voice Call still exposes call-control primitives rather than a separate hidden task objective.

## What This Changes

Request adds an `objective` or `brief` field to outbound realtime Voice Call CLI/tool/RPC calls so the first spoken `message` can stay a natural opener while the hidden task goal is available to the realtime session.

## Best Possible Solution

Fold the `objective`/`brief` API detail into https://github.com/openclaw/openclaw/issues/59245, and coordinate with https://github.com/openclaw/openclaw/issues/79055 only if maintainers choose a generic per-call preload seam instead of a Voice Call task-call-specific field.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection gives high-confidence current-main evidence that the requested private objective field is absent from the CLI/tool/RPC, call options, and realtime instruction path.

## Solution Assessment

Yes for cleanup: the best next step is to carry this detailed API shape into the existing outbound task-call tracker rather than splitting the same task-call objective design across parallel issues.
