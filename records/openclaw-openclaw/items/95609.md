Keep open. Current main and v2026.6.10 still send the resend notice and fail non-resumable restart-recovery tails, while the open linked PR is only a partial candidate that still needs maintainer scope and proof review.

## What This Changes

This issue requests replacing the non-resumable main-session restart resend/fail branch with no-retype recover-and-continue or recover-and-confirm handling.

## Best Possible Solution

Choose and land a bounded recovery contract that re-presents the recovered user request without retyping, preserves approval gates and side-effect safety, and keeps the existing user/tool/toolResult auto-resume path intact.
