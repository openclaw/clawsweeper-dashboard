## Summary

Keep open. Current main and v2026.6.10 still send the non-resumable restart-recovery resend notice and then fail the session, while the related implementation candidate at https://github.com/openclaw/openclaw/pull/95899 is still open and unmerged.

## What This Changes

The issue requests a no-retype restart-recovery flow for non-resumable transcript tails that reuses the preserved transcript request while preserving side-effect and approval safety gates.

## Best Possible Solution

Keep this issue open as the canonical recovery request, review https://github.com/openclaw/openclaw/pull/95899 as the current candidate, and land only a design that removes the retype requirement without silently replaying uncertain side effects or approvals.

## Reproduction Assessment

Yes. At source level, a marked running main session whose recent transcript tail ends in an assistant message or stale approval-pending tool result takes the resume-block branch, sends the resend notice, and marks the session failed.

## Solution Assessment

Unclear as an exact implementation. The no-retype outcome is the right user-facing target, but maintainers still need to choose the safe policy for confirm-to-continue, approval reconstruction, or idempotency before this should be automated.
