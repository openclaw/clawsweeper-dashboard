## Summary

Keep open. Current main and v2026.6.10 still send the non-resumable restart notice asking for a resend and then fail the session; the open linked PR is a candidate implementation, not merged proof, and the remaining behavior needs maintainer product judgment around side-effect safety.

## What This Changes

The issue requests replacing the non-resumable restart-recovery resend notice with a recover-and-continue or recover-and-confirm flow that reuses the existing transcript request.

## Best Possible Solution

Review or narrow https://github.com/openclaw/openclaw/pull/95899, then land a core restart-recovery design that re-presents the user request without bypassing approvals or double-executing side effects.

## Reproduction Assessment

Yes, at source level. A marked running main session whose recent transcript tail ends in an assistant message or stale approval-pending tool result takes the resume-block branch, sends the unresumable resend notice, and marks the session failed.

## Solution Assessment

Unclear as a final implementation path. The no-retype direction is sound, but maintainers still need to choose the safe behavior for confirmation, stale approvals, and side-effect idempotency before this can be closed.
