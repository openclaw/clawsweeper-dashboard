## Summary

Keep open: current main still has a stateless OpenAI Codex Responses image-generation path, and the requested continuation IDs plus follow-up inputs would be a new additive tool/API contract rather than an already-fixed bug.

## What This Changes

Expose OpenAI Responses image-generation continuation metadata and opt-in follow-up inputs through `image_generate` for iterative image edits.

## Best Possible Solution

Design an additive opt-in continuation contract that captures safe Responses/image-generation IDs from the OpenAI Codex path, exposes them in tool details, accepts explicit continuation inputs, preserves stateless defaults, and updates focused tests plus docs.

## Reproduction Assessment

Yes, at source level: current main’s schema/provider path has no continuation inputs and strips response/call IDs from the Codex image-generation result. This is a feature request rather than a broken documented behavior with a live failure repro.

## Solution Assessment

Unclear as an implementation because there is no patch yet. The requested direction is plausible, but maintainers need to choose the shared/API contract for continuation metadata, input names, and storage/privacy defaults before a focused implementation.
