## Summary

Keep open: current main still has a stateless `image_generate` contract for the OpenAI Codex Responses image path, so the requested continuation IDs and follow-up inputs are not already implemented and require an additive API/product decision.

## What This Changes

Request adding opt-in Responses continuation metadata and inputs to `image_generate` for OpenAI iterative image edits.

## Best Possible Solution

Design an additive opt-in continuation contract that captures safe Responses/image-generation IDs from the OpenAI Codex path, exposes them in tool details, accepts explicit continuation inputs, preserves stateless defaults, and updates tests/docs.

Yes for source-level reproduction: current main’s schema/provider path has no continuation inputs and strips response/call IDs from the Codex image-generation result. I did not establish a live failing bug repro, and the request is a new feature rather than broken documented behavior.

## Solution Assessment

Unclear as an implementation because there is no patch yet. The requested direction is plausible, but the best fix needs maintainer API design around generic versus OpenAI-specific metadata and `store`/privacy behavior.
