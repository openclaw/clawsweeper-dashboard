## Summary

Keep open. Current main still lacks the requested stateful OpenAI Responses image-generation continuation surface, and this is a new provider/tool API feature with privacy and contract choices rather than an already-defined bug fix.

## What This Changes

Expose continuation metadata and input options for OpenAI Responses-backed `image_generate` so generated images can be refined with response/image-generation IDs instead of only re-uploaded as reference images.

## Best Possible Solution

Add an opt-in OpenAI image continuation contract through the shared image-generation types, `image_generate` schema, bundled OpenAI provider, docs, changelog, and regression tests, while keeping the default stateless behavior unchanged.

## Reproduction Assessment

Yes, at source level. Current main sends Codex OAuth `image_generate` through `/responses` with `store: false`, parses image results/usage only, and the shared tool/request schema lacks continuation inputs or reusable IDs.

## Solution Assessment

Unclear until maintainers choose the API shape. The requested opt-in stateful path is plausible, but exposing IDs and store/continuation controls needs a deliberate provider contract and privacy decision.
