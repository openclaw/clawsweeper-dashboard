## Summary

Current main still has the reported ClawDock behavior, and two open PRs already target the same fix, including a closing PR. This should stay open until one of those PRs lands or maintainers choose a replacement.

## What This Changes

The issue asks for `clawdock-dashboard` to open the Docker-published host gateway port when `OPENCLAW_GATEWAY_PORT` differs from the container-internal `18789` port.

## Best Possible Solution

Land #77425, #75754, or an equivalent narrow ClawDock helper fix that remaps the browser URL to the Compose-published host gateway port while leaving the gateway’s internal port and core dashboard command unchanged.

## Reproduction Assessment

Yes. A high-confidence source reproduction is `OPENCLAW_GATEWAY_PORT=3200`, start Docker Compose, then run `clawdock-dashboard`: the helper opens the URL reported from the CLI container instead of the host-published port.

## Solution Assessment

Yes. The narrow maintainable solution is in `scripts/clawdock/clawdock-helpers.sh`, translating only the host-browser URL while preserving Docker’s internal gateway port and the generic dashboard command.
