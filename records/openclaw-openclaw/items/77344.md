## Summary

Keep open: current `main` still opens the dashboard URL reported from inside the Docker CLI container, and open linked PRs #77425 and #75754 already target the helper fix. The issue should remain open until one canonical fix lands.

## What This Changes

Make `clawdock-dashboard` open the Docker host-published gateway dashboard port instead of the container-internal gateway port.

## Best Possible Solution

Land #77425, #75754, or an equivalent narrow ClawDock helper fix that rewrites loopback dashboard URLs to the Compose-published host gateway port while preserving the existing internal gateway port.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `OPENCLAW_GATEWAY_PORT=3200`, start Docker Compose, source the helper, then run `clawdock-dashboard`; the helper opens the CLI-container URL instead of the host-published URL.

## Solution Assessment

Yes. The narrow maintainable solution is in `scripts/clawdock/clawdock-helpers.sh`, translating only the host-browser URL while leaving Docker's internal gateway port and the core dashboard command unchanged.
