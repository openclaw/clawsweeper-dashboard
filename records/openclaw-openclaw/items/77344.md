## Summary

Current main still has the reported ClawDock behavior, and the same reporter has an open paired fix PR, #75754, that targets the right helper boundary. This should stay open until that PR or an equivalent replacement lands.

## What This Changes

The issue asks for `clawdock-dashboard` to open the Docker Compose published host gateway port rather than the CLI-container internal `18789` URL.

## Best Possible Solution

Land #75754 or an equivalent narrow helper fix that maps loopback dashboard URLs to the Compose-published host gateway port while preserving scheme, path, and token redaction.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with `OPENCLAW_GATEWAY_PORT=3200`, Compose publishes host port `3200` to internal `18789` while `clawdock-dashboard` opens the `openclaw-cli dashboard --no-open` URL unchanged.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `scripts/clawdock/clawdock-helpers.sh`, translating only the host-browser URL while leaving the core dashboard command and Docker internal gateway port unchanged.
