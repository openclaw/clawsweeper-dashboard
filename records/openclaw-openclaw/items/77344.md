## Summary

Keep open: current main and the latest release still open the dashboard URL reported by the Docker CLI container unchanged, while a live open PR from the same reporter is already linked as the fix path.

## What This Changes

`clawdock-dashboard` should open the host-published Docker gateway URL when `OPENCLAW_GATEWAY_PORT` differs from the container-internal gateway port.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/75754 after fixing its token argv exposure, or land an equivalent narrow helper fix, while preserving the related `--no-deps` dashboard change from https://github.com/openclaw/openclaw/pull/80494.

## Reproduction Assessment

Yes, source-level. Configure Compose with a host-published gateway port different from `18789`, start the gateway, then run `clawdock-dashboard`; current main opens the CLI-reported loopback URL unchanged while Compose publishes a different host port.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `scripts/clawdock/clawdock-helpers.sh`, translating the host-browser URL from Docker Compose's published port without changing gateway runtime or dashboard command behavior; the linked PR direction is right once its token-handling issue is fixed.
