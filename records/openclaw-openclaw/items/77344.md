## Summary

Keep open: current main still opens the dashboard URL emitted inside the Docker CLI container unchanged, while Docker Compose can publish that internal gateway port on a different host port; the same reporter also has open paired PR #75754 targeting this exact helper behavior.

## What This Changes

Make `clawdock-dashboard` open the host-published Docker Compose gateway port when `OPENCLAW_GATEWAY_PORT` differs from the container-internal `18789` port.

## Best Possible Solution

Land #75754 or an equivalent narrow ClawDock helper fix that maps loopback dashboard URLs to the Compose-published host gateway port without changing gateway runtime behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: set `OPENCLAW_GATEWAY_PORT=3200`, start Docker Compose, and run `clawdock-dashboard`; the helper opens the container-reported `18789` URL instead of the host-published port.

## Solution Assessment

Yes. The narrow maintainable solution belongs in `scripts/clawdock/clawdock-helpers.sh`, translating the host-browser URL while leaving the Docker internal port and core dashboard command unchanged.
