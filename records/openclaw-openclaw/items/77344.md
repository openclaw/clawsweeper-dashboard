## What This Changes

Make `clawdock-dashboard` open the Docker host-published gateway port when `OPENCLAW_GATEWAY_PORT` differs from the container's internal `18789` port.

## Best Possible Solution

Land #75754 or an equivalent narrow update in `scripts/clawdock/clawdock-helpers.sh` that remaps loopback dashboard URLs to the Compose-published host port while preserving token redaction.

## Reproduction Assessment

Yes. Source inspection plus an isolated shell check on current main shows `clawdock-dashboard` opens the CLI-reported `18789` URL unchanged even when `OPENCLAW_GATEWAY_PORT=3200`.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the ClawDock helper, translating only the host-browser URL while leaving Docker's internal gateway port and the core dashboard command unchanged.
