## Summary

Current main still has the reported ClawDock dashboard behavior, and open PRs already target the fix, so the issue should remain open until one canonical PR lands or is replaced.

## What This Changes

The issue asks `clawdock-dashboard` to open the host-published Docker Compose gateway port instead of the container-internal loopback port reported by the CLI container.

## Best Possible Solution

Land #75754, #77425, or an equivalent narrow ClawDock helper fix that rewrites only the host-browser dashboard URL to the Compose-published gateway port while preserving token redaction and the existing gateway runtime contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `OPENCLAW_GATEWAY_PORT=3200`, start Docker Compose, source `scripts/clawdock/clawdock-helpers.sh`, and run `clawdock-dashboard`; the helper opens the CLI-container URL rather than the host-published port.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `scripts/clawdock/clawdock-helpers.sh`, translating the helper's browser URL while leaving Docker's internal gateway port and core dashboard command unchanged.
