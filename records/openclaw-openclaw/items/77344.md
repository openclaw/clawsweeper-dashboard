## Summary

Current `main` still has the reported ClawDock behavior: the helper opens the first URL printed by the CLI container without translating it to Docker's published host port. This should stay open because the same reporter has an open paired fix PR, #75754, that targets the right helper boundary.

## What This Changes

Make `clawdock-dashboard` open the Docker-published host gateway port when `OPENCLAW_GATEWAY_PORT` differs from the internal `18789` port.

## Best Possible Solution

Land #75754 or an equivalent narrow ClawDock helper fix that remaps loopback dashboard URLs to the Compose-published host gateway port while preserving token redaction and existing gateway behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction is `OPENCLAW_GATEWAY_PORT=3200`, start Docker Compose, then run `clawdock-dashboard`: the helper opens the CLI-container URL it parsed instead of resolving the published host port.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `scripts/clawdock/clawdock-helpers.sh`, translating the helper's host-browser URL while leaving Docker's internal gateway port and core dashboard command unchanged.
