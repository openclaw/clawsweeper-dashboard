## Summary

Current `main` still matches the reported ClawDock behavior, and an open paired PR from the same reporter already targets the right helper boundary. This should stay open until #75754 or an equivalent fix lands.

## What This Changes

The issue asks for `clawdock-dashboard` to open the Docker Compose host-published gateway port instead of the CLI container's internal gateway URL when `OPENCLAW_GATEWAY_PORT` differs from `18789`.

## Best Possible Solution

Land #75754 or an equivalent narrow ClawDock helper fix that remaps loopback dashboard URLs to the Compose-published host gateway port while preserving token redaction and leaving core gateway behavior unchanged.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with `OPENCLAW_GATEWAY_PORT=3200`, Compose publishes host port `3200` to container port `18789`, while `clawdock-dashboard` opens the CLI-container URL unchanged.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `scripts/clawdock/clawdock-helpers.sh` by translating the host-browser URL; changing the core dashboard command or Docker's internal gateway port would broaden the fix unnecessarily.
