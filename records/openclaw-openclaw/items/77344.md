## Summary

Keep open: current main still opens the CLI-reported dashboard URL unchanged while Docker Compose can publish a different host port, and #75754 is an open paired fix PR from the same reporter.

## What This Changes

The issue asks for `clawdock-dashboard` to open the Docker-published host gateway port instead of the gateway URL printed from inside the Compose CLI container.

## Best Possible Solution

Land #75754 or an equivalent narrow ClawDock helper fix that remaps loopback dashboard URLs to the Compose-published host gateway port while preserving token redaction and existing gateway behavior.

## Reproduction Assessment

Yes, source-backed. With a host-published gateway port that differs from the internal `18789`, current `clawdock-dashboard` opens the unmodified URL from `openclaw-cli dashboard --no-open` instead of resolving Docker's published host port.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `scripts/clawdock/clawdock-helpers.sh`, translating the host-browser URL while leaving Docker's internal gateway port and core dashboard command unchanged.
