## Summary

Current main still has the reported ClawDock behavior, and the issue should stay open because two open PRs, #75754 and #77425, explicitly target the fix.

## What This Changes

The report asks for `clawdock-dashboard` to open the Docker host-published gateway port when `OPENCLAW_GATEWAY_PORT` differs from the internal container port.

## Best Possible Solution

Land #75754, #77425, or an equivalent narrow ClawDock helper fix that rewrites loopback dashboard URLs to the Compose-published host gateway port while preserving existing token handling.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with `OPENCLAW_GATEWAY_PORT=3200`, Compose publishes host port 3200 to container port 18789 while `clawdock-dashboard` opens the CLI-container URL unchanged.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `scripts/clawdock/clawdock-helpers.sh`, translating only the host-browser URL while leaving Docker's internal gateway port and the core dashboard command alone.
