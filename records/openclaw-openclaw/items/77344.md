## Summary

Current main still has the reported ClawDock behavior, and two open PRs already target the same bug. This issue should stay open until one of those fixes, or an equivalent helper-only replacement, lands.

## What This Changes

Request that `clawdock-dashboard` open the Docker host-published gateway port instead of the container-internal dashboard URL.

## Best Possible Solution

Land #75754, #77425, or an equivalent narrow `scripts/clawdock/clawdock-helpers.sh` fix that remaps the dashboard URL to Docker's host-published gateway port while preserving token handling and existing gateway behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `OPENCLAW_GATEWAY_PORT=3200`, start Compose, then run `clawdock-dashboard`; the helper opens the URL emitted from inside the CLI container instead of translating it to the host-published port.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `scripts/clawdock/clawdock-helpers.sh`, because the core dashboard command is correct for its runtime and the host-browser translation is ClawDock-specific.
