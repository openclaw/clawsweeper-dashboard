## Summary

Keep open. Current `main` still opens the URL printed by the Docker CLI container without translating it to Docker Compose's host-published gateway port, and the same reporter has an open closing PR that targets the right helper boundary.

## What This Changes

The issue reports that `clawdock-dashboard` opens the Docker CLI container's internal dashboard URL instead of the Docker Compose host-published gateway port when `OPENCLAW_GATEWAY_PORT` is remapped.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/75754, or an equivalent narrow helper fix, so `clawdock-dashboard` remaps loopback dashboard URLs to the Compose-published host gateway port while preserving token redaction and existing gateway behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `OPENCLAW_GATEWAY_PORT=3200`, start the Docker Compose gateway, then run `clawdock-dashboard`; current main opens the CLI-container URL it parsed instead of resolving the published host port.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `scripts/clawdock/clawdock-helpers.sh`, translating the host-browser URL while leaving Docker's internal gateway port, dashboard command, and gateway runtime unchanged.
