## Summary

Keep open. The PR still targets a real gap on current main, but the branch is stale/dirty against the current update handler, has concrete correctness issues, and lacks external-contributor real behavior proof for the failed update/restart path.

## What This Changes

The PR adds a bounded `openclaw doctor --non-interactive` summary helper, persists/formats `doctorSummary` on restart sentinels, and wires it into failed `update.run` responses with adjacent tests.

## Best Possible Solution

Refresh the branch onto current main, preserve the current update/status and restart semantics, keep the doctor summary fixed-argv, bounded, and output-limited, and either narrow or redesign the config-reload diagnostic scope with maintainer approval.

## Reproduction Assessment

Yes for the PR defects by source inspection: PR head is stale against current main, early update failures can still skip the new summary, and the runtime mock has a duplicate export key. I did not run tests because this review is required to keep the checkout read-only.

## Solution Assessment

No, not as submitted. The diagnostic idea is plausible, but the patch needs a current-main rebase, the early-failure and mock fixes, clarified config-reload scope, maintainer product judgment, and real behavior proof before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Rebase onto the current update handler:** `src/gateway/server-methods/update.ts:20`
  - body: PR head's handler is based on an older `update.ts` and would drop current main's `update.status`, runtime config access, install-surface and restart-disabled handling, latest sentinel recording, restart continuations, package-swap restart flags, and `ok=false` semantics for skipped updates. Rebase and reapply the doctor-summary addition on top of the current handler.
  - confidence: 0.93
- **[P2] Use a fallback entry for early update failures:** `src/gateway/server-methods/update.ts:72-78`
  - body: If config loading or package-root resolution throws before `root` is assigned, this new call passes `entry: undefined`, so `runDoctorNonInteractiveSummary` returns `null` and the intended diagnostic is skipped for early `update.run` failures. Initialize a safe fallback root/entry before the try or otherwise pass a known entry to the summary helper.
  - confidence: 0.86
- **[P2] Remove the duplicate runtime mock export:** `src/infra/outbound/target-resolver.test.ts:27-30`
  - body: The mocked runtime module defines `getActivePluginChannelRegistry` twice, so the first property is shadowed and type/lint checks may reject the object literal. Keep only the configurable mock export.
  - confidence: 0.92
