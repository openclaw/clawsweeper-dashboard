## Summary

Keep this PR open for maintainer review. Current main still lacks the requested doctor-summary behavior, but the PR head is stale against the current update handler and still has concrete blocking issues before it can be merged safely.

## What This Changes

The branch adds a bounded non-interactive doctor-summary subprocess for failed `update.run` results, adds a `doctorSummary` restart-sentinel field/formatter, and updates adjacent tests/mocks.

## Best Possible Solution

Refresh the branch onto current main, preserve the existing update/status and restart-scheduling behavior, add bounded doctor summaries only where they are immediately useful, and either implement a non-stale config-reload diagnostic path or defer that part to #55347.
