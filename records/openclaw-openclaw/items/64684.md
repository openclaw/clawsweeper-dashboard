## Summary

Keep this PR open. Current main still only records a doctor hint for update restart sentinels and only logs config-reload restart-check failures, so the branch is not obsolete, but the proposed patch still has concrete correctness blockers before merge.

## What This Changes

The PR adds a bounded doctor-summary subprocess for failed `update.run` results, adds `doctorSummary` to restart sentinel payload formatting/tests, and adjusts related gateway and test harness mocks.

## Best Possible Solution

Land a focused diagnostic slice only after the branch is refreshed: failed `update.run` calls should attach a bounded, output-limited doctor summary even for early failures, restart sentinel formatting should display it, and config-reload diagnostics should either use a non-stale immediate path or be explicitly deferred to the broader self-healing work in #55347.

## Reproduction Assessment

Yes. Static current-main inspection shows the requested behavior is absent, and PR-head inspection gives direct reproduction paths for the two blocking findings: an early config/root failure before `result.root` is set and a duplicate key in the runtime mock object.

## Solution Assessment

No, not as submitted. The proposed direction is narrow and plausible for update failures, but the branch needs the fallback doctor entry, duplicate mock cleanup, changelog coverage, and a clear config-reload scope before it is the maintainable fix.
