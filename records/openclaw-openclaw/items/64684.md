## Summary

Keep open. Current main does not implement the doctor-summary behavior, but this PR branch is stale/conflicting, has correctness issues, and lacks after-fix real behavior proof, so it needs contributor/maintainer refresh rather than cleanup close.

## What This Changes

This PR adds a bounded `openclaw doctor --non-interactive` summary helper for failed `update.run` sentinels, formats a new `doctorSummary` field, and updates related tests/test mocks.

## Best Possible Solution

Refresh the branch onto current main, add doctor summaries through the current update sentinel builder without dropping status/handoff semantics, clarify or remove the config-reload scope, and provide redacted real failed-update/restart proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the PR head omits current update handler behavior, still skips doctor summary when early failures have no root, and contains a duplicate mock export. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No, not as submitted. The diagnostic direction is plausible, but it must be layered onto the current update handler and sentinel builder with clarified config-reload scope and real behavior proof.

- **[P1] Rebase onto the current update handler:** `src/gateway/server-methods/update.ts:20`
  - body: PR head is based on an older `update.run` implementation and would drop current main's `update.status`, runtime config access, managed-service handoff, latest sentinel recording, restart continuations, package-swap restart behavior, and current ok semantics.   - confidence: 0.94

  - body: If config loading or package-root resolution throws before `root` is assigned, this new call passes `entry` as undefined, so `runDoctorNonInteractiveSummary` returns null and the intended diagnostic is skipped for early update failures. Initialize a safe fallback root/entry before the try or otherwise pass a known entry to the helper.
  - confidence: 0.87

  - body: The mocked runtime module defines `getActivePluginChannelRegistry` twice, shadowing the first object property and risking duplicate-key lint/type failures. Keep only the configurable mock export.
  - confidence: 0.91
