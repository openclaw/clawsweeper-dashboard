## Summary

Keep open. Current main and the latest stable release still leave clean `session-start`/`watch` syncs dependent on in-process session dirty state, so a restart or missed transcript update can leave session files unindexed until a forced, targeted, or full-reindex path runs.

## What This Changes

Add a memory-core catch-up/reconciliation path so session transcript indexing catches up after gateway restarts or missed transcript update events without requiring `--force`.

## Best Possible Solution

Add a bounded memory-core reconciliation path that makes on-disk session transcript state authoritative for startup or another bootstrap point, then surface drift clearly enough that `Dirty: no` is not mistaken for verified freshness.

## Reproduction Assessment

Yes. The source-level path is high confidence: after restart or a missed transcript event, `sessionsDirty=false` and `sessionsDirtyFiles` is empty; `warmSession()` uses `reason: "session-start"`, the gate returns false, and `syncSessionFiles()` is skipped.

## Solution Assessment

Yes. A bounded reconciliation scan in the memory-core owner module is the narrowest maintainable direction; removing the `session-start`/`watch` exclusion outright would risk expensive hot-path rescans.
