## Summary

Keep open. Current main still skips session transcript sync after restart or missed events when no in-process session dirty state exists, while the docs define session transcripts as searchable memory content; the related observability work does not close the indexing gap.

## What This Changes

Add startup or catch-up reconciliation so memory-core session transcript indexing does not silently fall behind after gateway restarts or missed transcript events.

## Best Possible Solution

Add a bounded memory-core reconciliation path that compares session transcript files with durable DB file state, marks only missing or changed paths dirty, and keeps hot-path scans bounded.

## Reproduction Assessment

Yes. Source inspection on current main shows the restart or missed-event state: empty sessionsDirtyFiles, session-start/watch or normal cli sync, a false session gate, and no syncSessionFiles call; the issue and follow-up comment provide live operator repros.

## Solution Assessment

Yes. A bounded reconciliation scan in memory-core is the narrowest maintainable fix; making session-start/watch unconditional full scans would risk expensive hot-path work, and the open observability PR does not repair indexing.
