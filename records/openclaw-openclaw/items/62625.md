## Summary

Keep open. Current main still lacks a startup/session catch-up reconciliation path, and clean `session-start` or `cli` syncs still skip the `sessions` source when no current-process dirty files are recorded; the open observability PR at https://github.com/openclaw/openclaw/pull/42014 does not fix the indexing gap.

## What This Changes

Request a memory-core startup catch-up scan that reconciles on-disk session transcripts with indexed `sessions` rows after gateway restarts or missed transcript events.

## Best Possible Solution

Add a bounded memory-core reconciliation path during startup/bootstrap that compares session files with durable DB state, marks only missing or changed session files dirty, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the restart or missed-event state: `sessionsDirty=false`, `sessionsDirtyFiles` empty, `warmSession()` uses `reason: "session-start"`, the gate returns false, and `syncSessionFiles()` is skipped; I did not run a live gateway in this read-only review.

## Solution Assessment

Yes. A bounded reconciliation scan in the memory-core owner module is the narrowest maintainable fix; turning `session-start` or `watch` into unconditional full scans would risk expensive hot-path work.
