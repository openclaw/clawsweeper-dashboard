## Solution Assessment

Yes, this is the narrowest maintainable fix I found: it reuses the existing safe fallback writer instead of adding a second prose path, and it leaves the existing subagent, detached cron, and fallback-on-runtime-error behavior intact. Expanding the same behavior to light/REM without a concrete stale-diary promotion case would be broader than this bug fix.
