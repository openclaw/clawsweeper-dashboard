## Summary

Keep open: current main has related pieces for mention gating, ambient group context, and opt-in memory consolidation, but it does not implement a governed durable natural-language rule layer or conflict semantics for promoted rules. The concrete Feishu multi-bot message-loss pieces are already tracked by narrower open work, so this broader product request still needs maintainer direction rather than cleanup closure.

## What This Changes

The issue requests durable conversational rule learning plus explicit multi-bot group semantics where every explicitly mentioned bot replies once and unmentioned bots stay silent.

## Best Possible Solution

Keep this broad issue open for maintainer product design, while handling the concrete Feishu multi-bot delivery bugs in the narrower open Feishu trackers and deciding whether durable rules belong in memory-core, a plugin/API seam, or core configuration.

## Reproduction Assessment

Partly yes at source level: current code and docs show mention gating, passive room context, and memory promotion exist, while the requested governed durable rule layer does not. The exact live multi-agent production behavior still needs live Feishu/Lark proof for the channel-specific pieces.

## Solution Assessment

Unclear as a single implementation: the Feishu message-loss bugs should be fixed in their narrower trackers, while durable rule promotion needs a maintainer product decision about rule precedence, storage, review, and plugin/core ownership.
