## Summary

Keep open for maintainer product review: current main has durable memory/dreaming and configured group broadcast behavior, but it does not provide a governed natural-language rule layer with conflict/supersession semantics or an explicit contract that every explicitly mentioned bot replies once in multi-bot group messages.

## What This Changes

Request durable conversational rule promotion with conflict/supersession semantics, plus deterministic reply behavior when a group message explicitly mentions multiple bots.

## Best Possible Solution

Split this into a product design for governed rule promotion in the memory/plugin layer and a channel contract for explicit multi-bot mention dispatch, then implement narrow follow-ups with live Feishu/Lark proof.

## Reproduction Assessment

Source-level yes for the missing governed rule layer and partial mention semantics, but the reported production Feishu/Lark multi-agent behavior was not live-reproduced in this review.

## Solution Assessment

The current requested direction is plausible but too broad as one change; existing memory, dreaming, and broadcast mechanisms are partial and do not define rule authority, supersession, or per-mentioned-bot dispatch.
