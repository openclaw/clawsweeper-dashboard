## Summary

Keep open: current main has adjacent memory promotion, plugin hook, and Feishu broadcast machinery, but it still does not define the requested durable rule-governance layer or visible reply semantics for every explicitly mentioned bot.

## What This Changes

Request durable promotion of conversational rules into a governed rule layer and explicit group-chat semantics where every explicitly mentioned bot replies while unmentioned bots stay silent.

## Best Possible Solution

Split this into maintainer-approved contracts for durable rule precedence and explicit multi-mention routing, then implement focused channel and memory/plugin-surface changes against those contracts.

## Reproduction Assessment

Yes, at source level: current Feishu broadcast dispatch fans out to all configured agents but only the routed active agent gets a real reply dispatcher, and memory docs show promotion without a governed rule-conflict layer. I did not establish a live Feishu/Lark reproduction.

## Solution Assessment

No, not as one broad direct implementation. The maintainable path is to first define rule precedence and multi-mention channel semantics, while treating the linked listen-only plugin and `before_dispatch` hook as partial mitigations rather than the final product contract.
