## Summary

Keep open: current main has adjacent memory promotion, Skill Workshop, and a `before_dispatch` hook that supports listen-only mitigations, but it still does not define the requested governed durable rule-precedence layer or an explicit all-mentioned-bots-visible-reply contract. ## What This Changes

Request to add durable natural-language rule promotion and precedence plus group-chat semantics where every explicitly mentioned bot replies once and unmentioned bots stay silent.

## Best Possible Solution

Define maintainer-approved contracts for durable rule precedence and explicit multi-mention routing, then split the implementation across memory/plugin surfaces and the affected channel behavior.

## Solution Assessment

No, not as a broad direct fix from this issue text. The maintainable path is a product/API decision first, with the linked listen-only plugin and `before_dispatch` treated as partial mitigations rather than the full rule-learning and multi-mention contract.
