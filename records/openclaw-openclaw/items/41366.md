## Summary

Keep open: current main still does not implement the requested durable natural-language rule layer or the explicit Feishu/Lark semantics where every mentioned bot visibly replies once. Source inspection shows Feishu broadcast still gives only one routed agent a real reply dispatcher while observer agents use no-op senders, and memory promotion still appends scored candidates into MEMORY.md rather than governing durable rule precedence or conflict supersession.

## What This Changes

The issue requests durable natural-language rule promotion with conflict precedence plus explicit multi-mention group-chat semantics where every mentioned bot responds and unmentioned bots stay silent.

## Best Possible Solution

Define a maintainer-approved durable rule-precedence contract and Feishu all-mentioned-bots routing policy, preferably through memory/plugin-owned seams where possible, then implement focused tests, docs, and live multi-bot proof.
