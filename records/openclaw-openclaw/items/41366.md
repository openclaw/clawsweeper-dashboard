## Summary

Keep open. Current main has related mention-gating, Feishu multi-mention context, broadcast, and memory-promotion pieces, but the requested durable rule layer with conflict precedence remains a product/architecture feature rather than an implemented behavior or narrow bug fix.

## What This Changes

The issue asks for durable promotion of conversational natural-language rules plus group-chat semantics where every explicitly mentioned bot replies once and unmentioned bots stay silent.

## Best Possible Solution

Define a bounded rule-promotion architecture: memory plugins can capture and review candidate rules, while core/channel contracts document precedence and explicit mention semantics with regression coverage for Feishu and other group-capable channels.

## Reproduction Assessment

No. This is primarily a feature request; source inspection confirms related current-main behavior, but this review did not establish a failing live multi-bot reproduction path.

## Solution Assessment

Unclear. The requested direction is plausible, but the best fix needs maintainer design on where durable conversational rules live and how they override workspace files without weakening safety boundaries.
