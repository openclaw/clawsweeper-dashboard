## Summary

Keep open: current main still has source-confirmed gaps for both halves of the request, and the linked listen-only plugin/before_dispatch path is only a partial mitigation rather than a durable rule-precedence or all-mentioned-bots reply contract.

## What This Changes

The issue asks for durable natural-language rule promotion with conflict precedence plus explicit semantics that every directly mentioned bot replies once while unmentioned bots stay silent.

## Best Possible Solution

Define a maintainer-approved durable rule-precedence layer and explicit multi-mention routing contract, then implement it through memory promotion, channel dispatch, tests, and docs while treating the listen-only plugin path as an adjacent mitigation.

## Reproduction Assessment

Yes, at source level: Feishu broadcast currently records turns for multiple agent sessions but creates a visible dispatcher for only one active agent, and short-term promotion writes to MEMORY.md rather than a governed rule layer. I did not run a live Feishu/Lark group.

## Solution Assessment

No, not as a direct autonomous patch from the issue text. The maintainable path is a maintainer product/API decision first, with before_dispatch and the linked listen-only plugin treated as partial mitigations only.
