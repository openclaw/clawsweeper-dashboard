## Summary

Keep open: current main still has source-confirmed gaps for both halves of the request, and the linked listen-only plugin/before_dispatch path is only a partial mitigation rather than a durable rule-precedence or all-mentioned-bots reply contract.

## What This Changes

The issue requests an opt-in durable natural-language rule promotion layer with conflict precedence, plus explicit multi-mention group semantics where every mentioned bot replies and unmentioned bots stay silent.

## Best Possible Solution

Define a maintainer-approved durable rule-precedence layer and explicit multi-mention routing contract, then implement it through memory promotion, channel dispatch, tests, and docs while treating the listen-only plugin path as an adjacent mitigation.

## Reproduction Assessment

Yes, at source level: Feishu broadcast currently records multiple agent sessions but creates a visible dispatcher for only one active agent, and memory promotion writes durable content to MEMORY.md rather than a governed rule layer. I did not run a live Feishu/Lark multi-bot group.

## Solution Assessment

No, not as an autonomous patch from the issue text. The maintainable path is a maintainer product/API decision first, with before_dispatch and the linked listen-only plugin treated as partial mitigations only.
