## Summary

Keep open: current main has adjacent memory promotion, Skill Workshop, and hook/plugin mitigation paths, but it still lacks the requested governed durable rule-precedence layer and an explicit contract that every explicitly mentioned bot replies visibly in multi-agent group chats.

## What This Changes

The issue requests durable natural-language rule promotion with conflict precedence plus group-chat semantics where every explicitly mentioned bot replies once and unmentioned bots stay silent.

## Best Possible Solution

Define maintainer-approved contracts for durable rule precedence and explicit multi-mention routing, then split implementation across memory/plugin surfaces and the affected channel behavior.

No, not as a direct broad implementation from this issue text. The maintainable path is a product/API decision first, with the linked listen-only plugin and hooks treated as partial mitigations rather than the full rule-learning and multi-mention contract.
