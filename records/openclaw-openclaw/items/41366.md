## Summary

Keep open: current main has adjacent memory promotion, broadcast, ambient-room, and `before_dispatch` hook surfaces, but it still lacks a governed durable rule-precedence layer and a channel contract that all explicitly mentioned bots reply once while unmentioned bots stay silent.

## What This Changes

Request a durable natural-language rule learning path with conflict precedence plus explicit multi-mention reply semantics for group chats.

## Best Possible Solution

Define maintainer-approved contracts for durable rule precedence and explicit multi-mention routing, then split implementation across memory/plugin surfaces and affected channel behavior.

No, not as a direct broad implementation from the issue text. The maintainable path is a product/API decision first, with the linked listen-only plugin and `before_dispatch` hook treated as partial mitigations rather than the full durable-rule and multi-mention contract.
