## Summary

Keep open: current main still lacks the requested durable rule-precedence contract and explicit all-mentioned-bots reply semantics. The linked listen-only plugin and narrower Feishu/group-context follow-ups are useful adjacent work, but they do not replace the product/API decision this issue asks for.

## What This Changes

The issue requests durable natural-language rule learning with conflict precedence plus group-chat semantics where every explicitly mentioned bot replies once and unmentioned bots stay silent.

## Best Possible Solution

Define maintainer-approved durable rule precedence and multi-mention routing contracts, then implement them through the memory, plugin, channel, test, and docs boundaries.

## Reproduction Assessment

Yes, at source level: Feishu broadcast reaches multiple agent sessions but creates a real reply dispatcher only for one active agent, while memory promotion appends to MEMORY.md instead of a governed rule layer. I did not run a live Feishu/Lark group.

## Solution Assessment

No, not as a direct broad patch from the issue text. The maintainable path is a product/API decision first, with before_dispatch and the linked listen-only plugin treated as partial mitigations only.
