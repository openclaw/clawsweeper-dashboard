## Summary

Keep open: current main has adjacent Feishu broadcast, mention-gating, observe-only, and durable-memory machinery, but it still lacks a settled durable rule-precedence layer and an explicit contract that every explicitly mentioned bot replies visibly once. The linked `openclaw-listen-only` mitigation and related PR #42400 are useful context, but they cover listen-only/background handling rather than the requested rule learning and multi-mention semantics.

## What This Changes

The issue requests durable promotion of conversational rules into a governed rule layer plus group-chat semantics where every explicitly mentioned bot replies once and unmentioned bots stay silent.

## Best Possible Solution

Split the work into a maintainer-approved design for durable rule precedence/promotion and a separate channel contract for explicit multi-bot mentions, then implement Feishu and shared channel tests against that contract.

## Reproduction Assessment

Yes, at source level: Feishu broadcast runs all configured agents but only the routed active agent gets a real visible reply dispatcher, and memory has durable promotion surfaces without a governed rule-conflict layer. I did not perform a live Feishu/Lark reproduction.

## Solution Assessment

No, not as a single immediate implementation item. The adjacent broadcast/listen-only/memory pieces are not a complete solution; the maintainable path is to define the product contract first and then land focused implementation PRs.
