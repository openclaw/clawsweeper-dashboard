## Summary

Keep open: current main still has adjacent memory promotion, Skill Workshop, and plugin hooks, but it does not define the requested governed durable rule-precedence layer or visible reply semantics for every explicitly mentioned bot. The linked https://github.com/flowolforg/openclaw-listen-only plugin plus `before_dispatch` are useful mitigations for listen-only/background-context behavior, not a full implementation of this issue.

## What This Changes

The issue requests durable natural-language rule promotion with conflict handling and group-chat semantics where every explicitly mentioned bot replies once while unmentioned bots stay silent.

## Best Possible Solution

Define maintainer-approved contracts for durable rule precedence and explicit multi-mention routing, then implement focused memory/plugin-surface and channel changes against those contracts.

## Reproduction Assessment

Yes, at source level: current Feishu broadcast dispatch can run multiple agent sessions but only one agent gets a real Feishu reply dispatcher, and memory docs show promotion to `MEMORY.md` without a governed rule-conflict layer. I did not establish a live Feishu/Lark reproduction.

## Solution Assessment

No, not as one broad direct change. The maintainable path is to first decide rule precedence and multi-mention semantics, while treating `before_dispatch` and the linked listen-only plugin as partial mitigation paths.
