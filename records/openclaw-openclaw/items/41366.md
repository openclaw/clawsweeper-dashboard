## Summary

Current main now provides opt-in durable memory promotion, but the issue’s explicit conflict-precedence and every-mentioned-bot reply contracts are not fully implemented or canonically tracked elsewhere.

## What This Changes

Requests an optional durable conversational-rule promotion layer plus deterministic reply behavior for messages that explicitly mention multiple bots.

## Best Possible Solution

Keep memory-core dreaming as the durable promotion path, add an explicit reviewed contract for behavioral-rule conflicts and supersession there, and define Feishu multi-account mention arbitration separately with tests for single mention, multiple mentions, unmentioned accounts, and configured broadcast behavior.

## Reproduction Assessment

Yes at the source level: current main exposes durable promotion and per-account mention admission, but no explicit conflicting-rule precedence or every-mentioned-account reply guarantee. The exact live Feishu multi-account scenario was not executed during this read-only review.

## Solution Assessment

No, not as one first-class core rule layer. The maintainable path is to extend memory-core's existing promotion governance and separately define channel-owned Feishu arbitration, avoiding parallel durable state and a universal transport policy.
