## Summary

Keep open: current main has memory files, dreaming/promotion, mention gating, and some broadcast machinery, but it does not define a first-class durable rule layer or a cross-channel contract that every explicitly mentioned bot must reply once. The linked openclaw-listen-only workaround and related #42400 cover context/suppression use cases only, not the full rule-learning and multi-mention semantics requested here.

## What This Changes

The issue requests durable natural-language rule learning plus group-chat semantics where every explicitly mentioned bot replies once and unmentioned bots stay silent by default.

## Best Possible Solution

Keep this as a product-design issue and split the eventual work into a durable rule-precedence contract plus narrower channel tests or API changes for explicit multi-mention behavior.

## Reproduction Assessment

Not applicable as a strict bug reproduction: the expected durable rule layer and multi-bot reply contract are not current documented behavior. Source inspection does confirm current main only has memory/dreaming, workspace rules, and mention-gating/broadcast primitives.

## Solution Assessment

No. The requested direction is useful but too broad for one narrow fix; the maintainable path is to first define rule storage, precedence, and channel reply semantics, then implement scoped pieces with regression coverage.
