## Summary

Keep open: current main still lacks both requested contracts: durable rule precedence for natural-language training and explicit all-mentioned-bots reply semantics. The existing Feishu broadcast path and the linked listen-only plugin are useful adjacent work, but they do not replace the product/API decision this issue asks for.

## What This Changes

Requests durable natural-language rule promotion with conflict precedence plus group multi-mention behavior where every explicitly mentioned bot replies and unmentioned bots stay silent.

## Best Possible Solution

Define maintainer-approved durable rule precedence and multi-mention routing contracts, then implement them through memory, plugin, channel, tests, and docs boundaries.

## Reproduction Assessment

Yes, at source level: Feishu broadcast reaches multiple agent sessions but creates a real reply dispatcher only for one active agent, while memory promotion appends to MEMORY.md rather than a governed rule layer. I did not run a live Feishu/Lark group.

## Solution Assessment

No, not as a direct broad patch from the issue text. The maintainable path is a product/API decision first, with before_dispatch and the linked listen-only plugin treated as partial mitigations only.
