## Summary

Keep open: current main has durable memory, dreaming promotion, Active Memory recall, and mention gating, but not the requested governed rule layer or channel-neutral explicit multi-mention target-set semantics. The open listen-only work in #42400 and the linked openclaw-listen-only plugin are useful adjacent context, not a canonical fix for this broader product gap.

## What This Changes

Request a governed durable conversational rule layer that reconciles with workspace rules, plus explicit multi-mention semantics where every mentioned bot replies once and unmentioned bots stay silent.

## Best Possible Solution

Split this into two maintainer-owned design tracks: a governed durable rule or typed memory layer with provenance, opt-in promotion, precedence, supersession, rollback, and documented interaction with AGENTS.md/SOUL.md/MEMORY.md/Dreaming/Active Memory; and a channel/plugin SDK contract for explicit mention target sets so channels can deterministically route one visible reply from each explicitly mentioned bot or agent while keeping unmentioned agents silent by default.

## Reproduction Assessment

No high-confidence live reproduction path is available from the issue alone because the report depends on production Feishu/Lark multi-bot sessions and session history. Static verification does reproduce the capability gap: current main exposes boolean mention facts and Markdown memory promotion, not a durable governed rule layer or explicit mention target set.

## Solution Assessment

Not applicable as a patch because this issue has no proposed code change. The requested direction is plausible, but the maintainable path is a designed core/plugin contract split rather than a single narrow bug fix or a ClawHub-only plugin.
