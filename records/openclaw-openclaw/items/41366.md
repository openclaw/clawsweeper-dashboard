## Summary

Keep open: current main has partial mention-gating, Feishu broadcast-session, and durable-memory machinery, but the requested durable rule-governance layer and explicit multi-mentioned-bots-reply-visible semantics are not implemented as a settled product contract.

## What This Changes

The issue requests durable conversational rule promotion plus explicit group-chat behavior where every directly mentioned bot replies and unmentioned bots stay silent.

## Best Possible Solution

Split or specify a maintainer-approved design that defines a durable rule layer with conflict precedence and a channel contract for explicit multi-bot mentions, then implement Feishu and shared channel hooks against that contract.

## Reproduction Assessment

Yes, at source level for the current gap: Feishu broadcast runs all configured agents but only gives the routed active agent a visible reply dispatcher, and durable memory lacks a governed rule-conflict layer. I did not establish a live Feishu/Lark reproduction path.

## Solution Assessment

No. The current broadcast/listen-only pieces are adjacent but not the best complete solution; the maintainable path is to first define product semantics for rule precedence and explicit multi-mention routing, then make focused implementation PRs.
