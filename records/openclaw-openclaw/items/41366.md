## What This Changes

The issue requests durable natural-language rule learning plus explicit semantics where every explicitly mentioned bot replies once in multi-agent group chat.

## Best Possible Solution

Define a maintainer-approved durable rule-precedence contract and Feishu all-mentioned-bots routing contract, preferably through memory/plugin-owned seams where possible, then implement focused tests, docs, and live multi-bot proof.

## Reproduction Assessment

Yes, at source level: current Feishu broadcast creates one visible dispatcher while observer agents cannot send visible replies, and current memory promotion/flush paths do not provide a governed durable rule layer. I did not run a live Feishu/Lark multi-bot group.

## Solution Assessment

No, not as an autonomous fix from the issue text. The maintainable path is a product/API decision first, then a scoped implementation that preserves compatibility while adding explicit rule precedence and routing semantics.
