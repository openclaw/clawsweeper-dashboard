## Summary

Keep open: current main and v2026.6.8 still lack the requested terminal CLI to list approved pairings or revoke paired senders. The lower-level pairing store and in-chat `/allowlist` command cover pieces of the behavior, but this needs maintainer product/security direction before adding an admin-facing pairing-management surface.

## What This Changes

The issue requests `openclaw pairing` commands to list approved senders, revoke approved senders, and optionally show merged pairing-store approvals in channel status.

## Best Possible Solution

Define one account-scoped admin pairing-management surface that coordinates terminal CLI behavior with `/allowlist`, channel status, and the broader contact-inventory design.

## Reproduction Assessment

Yes, at source level: current `openclaw pairing list` lists pending requests and `pairing approve` writes approved senders, but no terminal command lists or revokes approved pairings. This is a missing feature path rather than a broken existing CLI contract.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. The proposed commands are plausible, but the maintainable shape should avoid creating a second partial permission-management view beside `/allowlist` and https://github.com/openclaw/openclaw/issues/48620.
