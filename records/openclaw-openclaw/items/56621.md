## Summary

Keep open: current main and the latest release still expose only pending pairing request listing and approval in the shell CLI, with no `list-approved` or `revoke` command and no merged approved-sender status output. The adjacent `/allowlist` chat command and pairing-store helpers are useful implementation context, but they do not satisfy the requested admin CLI surface.

## What This Changes

The issue requests `openclaw pairing list-approved` and `openclaw pairing revoke` commands, plus merged config and pairing-store approved-sender visibility in `channels status --probe`.

## Best Possible Solution

Add account-aware shell commands for approved pairing-store sender listing and revocation, define safe config/store merge semantics for status output, and keep the broader contact inventory work coordinated with https://github.com/openclaw/openclaw/issues/48620.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, release-tag inspection, and live issue context show the requested shell CLI/status surface is not present on current main or v2026.5.19.

## Solution Assessment

Unclear until maintainers settle the security and status-output semantics. The narrow `openclaw pairing` commands fit the existing store helpers, but full merged status output should be designed deliberately because it prints authorized sender identifiers.
