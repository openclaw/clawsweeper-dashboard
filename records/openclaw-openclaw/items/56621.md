## Repair Work Prompt

Implement issue 56621 as a focused OpenClaw CLI/status feature. Add `openclaw pairing list-approved <channel> [--account <accountId>] [--json]` in `src/cli/pairing-cli.ts` that merges config `allowFrom` with `readChannelAllowFromStore`, labels each entry source (`config`, `pairing`, or both), preserves deterministic ordering, and supports extension channel ids like existing pairing commands. Add `openclaw pairing revoke <channel> <sender> [--account <accountId>] [--json]` that removes only pairing-store approvals via `removeChannelAllowFromStoreEntry`, preserves default-account legacy unscoped removal semantics, and clearly reports config-owned entries as not revocable by this command. Update `channels status --probe` to expose/display an effective merged allowlist or safe paired-store summary without breaking schema-light status payload compatibility. Update CLI/channel pairing docs and add a user-facing changelog entry. Add focused tests for default account, non-default account, config-only entries, pairing-store entries, duplicate source merging, JSON output, and status formatting. Do not add config options, do not change DM authorization behavior, do not change group authorization, and do not migrate credential files at startup.
