## Summary

Keep open: current main and v2026.6.8 still lack terminal commands to list approved channel pairings or revoke a paired sender. The existing pairing store and in-chat `/allowlist` flow cover pieces of the behavior, but this admin-facing authorization surface still needs maintainer product and security direction.

## What This Changes

The issue requests `openclaw pairing list-approved` and `openclaw pairing revoke` commands, plus merged approved-sender visibility in channel status.

## Best Possible Solution

Define one account-scoped admin pairing/contact management surface that coordinates terminal CLI behavior with `/allowlist`, channel status, and the broader contact-inventory design.

## Reproduction Assessment

Yes. Source inspection shows `openclaw pairing list` lists pending requests and `pairing approve` writes approved senders, while no terminal command lists or revokes approved pairings.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. The proposed commands are plausible, but the best shape should avoid a second partial permission-management surface beside `/allowlist` and https://github.com/openclaw/openclaw/issues/48620.
