## Summary

Keep open: current main and v2026.6.6 still lack the requested CLI to list approved DM pairings or revoke paired senders, and channel status does not merge pairing-store approvals into its `allow:` output. The request is valid, but it needs maintainer product and security design because it exposes and mutates sender authorization state and overlaps the broader contact-inventory work at https://github.com/openclaw/openclaw/issues/48620.

## What This Changes

Request adds account-aware admin CLI surfaces to list approved DM pairings, revoke a paired sender, and show config plus pairing-store allowFrom entries in channel status.

## Best Possible Solution

Define one account-scoped admin pairing-management surface that reuses existing pairing-store primitives, clearly distinguishes config vs pairing-store entries, and coordinates with the broader contact inventory design.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection verifies the gap: current main exposes pending `pairing list` and `pairing approve`, but no approved-sender CLI inventory or revoke command.

## Solution Assessment

Unclear until maintainers choose the product/security boundary. The proposed commands are plausible, but the best path should be one account-aware admin surface coordinated with `/allowlist` and https://github.com/openclaw/openclaw/issues/48620 rather than a competing permission view.
