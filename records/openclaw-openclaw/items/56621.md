## Summary

Keep open: current main and v2026.5.20 still expose shell pairing commands for pending request listing and approval only; store read/remove primitives and an in-chat `/allowlist` flow exist, but the requested shell `list-approved`/`revoke` commands and merged status output are not implemented.

## What This Changes

The issue requests shell CLI commands to list approved paired senders and revoke pairing-store approvals, plus merged config/store allowlist display in channel status.

## Best Possible Solution

Add account-aware shell admin commands and status output using the existing pairing-store and allowlist semantics, with source labels, privacy/redaction policy, docs, and focused tests aligned with the broader contacts-management direction.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, live issue context, and release-tag inspection show the requested shell CLI/status surface is not present on current main or v2026.5.20.

## Solution Assessment

Unclear until maintainers settle the security and UX semantics. The pairing CLI is a plausible home, but it should stay aligned with existing `/allowlist` behavior and the broader contacts-management issue.
