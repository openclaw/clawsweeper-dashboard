## Summary

Keep open: current main and v2026.6.8 still lack terminal commands to list approved channel pairings or revoke a paired sender. Existing lower-level store primitives and the in-chat `/allowlist` flow cover pieces of the behavior, but the requested admin-facing authorization surface still needs maintainer product and security direction.

## What This Changes

The issue requests CLI/admin support to list approved channel pairings, revoke paired senders, and optionally merge pairing-store state into channel status output.

## Best Possible Solution

Define one account-scoped admin pairing/contact management surface that coordinates terminal CLI behavior with `/allowlist`, channel status, and the broader contact-inventory design instead of adding a competing partial permission-management path.

## Reproduction Assessment

Yes. Source inspection shows `openclaw pairing list` lists pending requests and `pairing approve` writes approved senders, while no terminal command lists or revokes approved pairings.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. The proposed commands are plausible, but the best shape should avoid a second partial permission-management surface beside `/allowlist` and https://github.com/openclaw/openclaw/issues/48620.
