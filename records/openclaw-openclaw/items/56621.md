## Summary

Keep open: current main and v2026.6.10 still do not expose terminal `pairing list-approved` or `pairing revoke`, and the existing store helpers plus in-chat `/allowlist` only partially cover the requested admin workflow. The remaining work is security-sensitive product design, not conservative cleanup.

## What This Changes

The issue requests CLI commands to list approved paired senders and revoke approved senders, plus merged pairing/config allowlist visibility in channel status.

## Best Possible Solution

Define one account-scoped admin pairing/contact management surface that coordinates terminal CLI behavior with `/allowlist`, channel status, and https://github.com/openclaw/openclaw/issues/48620, with explicit redaction and authorization semantics.

## Reproduction Assessment

Yes. Source inspection shows current `openclaw pairing list` lists pending requests and `pairing approve` writes approved senders, while no terminal command lists or revokes approved pairings.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. The proposed commands are plausible, but the best shape should avoid a second partial permission-management surface beside `/allowlist` and the broader contact inventory issue.
