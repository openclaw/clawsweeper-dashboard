## Summary

Keep open: current main still lacks a WhatsApp forward action, and the request is a new WhatsApp plugin capability with privacy, gating, and shared action-contract choices rather than an obsolete cleanup item.

## What This Changes

Add a WhatsApp message action that forwards an existing message or media item by message ID to a destination chat.

## Best Possible Solution

Design a gated WhatsApp-owned forward action, add shared action typing only if maintainers want it public across channels, and cover target authorization, message lookup, media handling, and docs.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken current contract. Source inspection shows current main has no WhatsApp `forward` action in discovery, dispatch, runtime handling, config gates, docs, or the active listener surface.

## Solution Assessment

Unclear: the proposed shape is plausible, but the best solution depends on maintainer decisions around privacy gating and whether `forward` belongs in the shared message action contract or only inside the WhatsApp plugin.
