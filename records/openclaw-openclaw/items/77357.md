## Summary

Keep open: current main does not implement WhatsApp forwarding, and cross-chat forwarding needs maintainer decisions around privacy, authorization, and whether the action is shared or WhatsApp-specific.

## What This Changes

Request to add `action=forward` to the WhatsApp channel so an agent can forward an existing message to another chat by `message_id` and `target`.

## Best Possible Solution

Design a gated `@openclaw/whatsapp` forward action, keeping it plugin-owned unless maintainers intentionally promote forwarding into the shared channel action contract.

## Reproduction Assessment

Not applicable: this is a feature request, not broken current behavior. Source inspection confirms current main advertises and handles WhatsApp reactions and polls, but no forward operation.

## Solution Assessment

Unclear: the proposed parameter shape is plausible, but the maintainable solution should first settle authorization, message retention, media behavior, and shared-versus-plugin action typing.

## Risks / Open Questions

- Forwarding can move private text and media across chats, so source/target authorization and an operator-controlled gate need product review.
- The shared message action registry currently has no `forward`, so maintainers need to decide whether this is a generic channel action or a WhatsApp-specific plugin path.
- A message-id-only API may be insufficient unless the WhatsApp runtime stores or can retrieve the original forwardable message payload.
