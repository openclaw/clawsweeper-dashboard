## Summary

Keep open: current main still lacks a WhatsApp forward action, and adding one is a new WhatsApp plugin capability that needs action-contract, target-authorization, and privacy/media-retention decisions before implementation.

## What This Changes

Add a WhatsApp message-tool action to forward an existing audio, image, or document message by message ID to a destination chat.

## Best Possible Solution

Design a WhatsApp-owned forward action with explicit target authorization, source-message retention rules, a config gate, docs, and shared action typing only if multiple channels need the same contract.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source inspection of current main shows forwarding is absent from WhatsApp discovery, dispatch, runtime, active listener, config schema, and docs.

## Solution Assessment

Unclear: the requested JSON shape is plausible, but the maintainable fix needs a settled safety model for cross-chat private-content forwarding before a narrow implementation can be reviewed.
