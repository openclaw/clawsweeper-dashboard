## Summary

Keep open: current main still does not expose WhatsApp message forwarding, and adding cross-chat forwarding is a new WhatsApp plugin capability with privacy, authorization, and public action-contract decisions.

## What This Changes

The issue requests a WhatsApp channel `forward` action that forwards an existing message to another chat using `message_id` and `target`.

## Best Possible Solution

Design a gated `@openclaw/whatsapp` forward action, extend the shared action contract only if maintainers want it public across channels, and add docs plus coverage for target authorization, message lookup, and media forwarding.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken current contract. Source inspection shows no WhatsApp `forward` action in discovery, dispatch, config, shared action names, or listener APIs.

## Solution Assessment

Unclear: the proposed JSON shape is plausible, but the maintainable solution first needs decisions on privacy gating and whether `forward` belongs in the shared message-tool contract or only in the WhatsApp plugin.
