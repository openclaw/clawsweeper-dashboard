## Summary

Keep open: current main still lacks WhatsApp message forwarding, and adding it is a new WhatsApp plugin capability that needs action-contract, authorization, privacy, and likely config-gating decisions.

## What This Changes

The request asks for a WhatsApp `forward` message-tool action that forwards an existing message ID to a destination chat.

## Best Possible Solution

Design a WhatsApp-owned forward action with explicit target authorization, privacy rules, config gating, docs, and tests; update the shared action-name contract only if maintainers want forwarding to become a generic channel action.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source inspection at current main shows no advertised, typed, configured, or dispatched WhatsApp `forward` action.

## Solution Assessment

Unclear: the requested parameter shape is plausible, but maintainers should first decide whether forwarding is WhatsApp-specific or shared and how message lookup, target authorization, and media privacy should work.
