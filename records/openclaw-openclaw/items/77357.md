## Summary

Keep open: current main still lacks a WhatsApp forwarding action, and adding one is a new WhatsApp plugin capability with privacy, authorization, config-gating, and dependency-contract decisions.

## What This Changes

The issue requests a WhatsApp channel `forward` action that forwards an existing message to another chat by `message_id` and destination `target`.

## Best Possible Solution

Design a WhatsApp-owned forward action with explicit target resolution, authorization, privacy/audit behavior, and a documented config gate; promote it to the shared message-action contract only if maintainers want forwarding across multiple channels.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source inspection of current main verifies forwarding is absent from shared action names, WhatsApp discovery/dispatch/runtime handling, config schema, docs, and the active listener surface.

## Solution Assessment

Unclear: the proposed `action=forward` shape is plausible, but maintainers should first decide the WhatsApp safety model and whether forwarding is a WhatsApp-specific action or a shared message-tool action.
