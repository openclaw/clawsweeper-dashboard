## Summary

Keep open: current main still does not expose WhatsApp message forwarding, and adding it would be a new WhatsApp plugin capability with privacy, authorization, target-resolution, and action-contract decisions rather than conservative cleanup.

## What This Changes

Add a WhatsApp message-tool `forward` action that forwards an existing message by id to another WhatsApp chat.

## Best Possible Solution

Design a WhatsApp-owned, gated forward action with documented privacy rules, target resolution, and regression coverage, extending the shared action contract only if maintainers want forwarding to be a generic message-tool action.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source inspection on current main verifies that forwarding is absent from discovery, dispatch, config, docs, and the active listener surface.

## Solution Assessment

Unclear: the requested `action=forward` shape is plausible, but maintainers should decide whether forwarding is WhatsApp-specific or shared, how it is gated, and how private media/message IDs are authorized before implementation.
