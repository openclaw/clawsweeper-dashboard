## Summary

Keep open: current main still lacks WhatsApp message forwarding, and this is a new WhatsApp plugin capability that needs maintainer decisions around privacy, authorization, config gating, and shared action-contract scope.

## What This Changes

The issue requests a WhatsApp `forward` message action that forwards an existing message to another destination chat.

## Best Possible Solution

Design a gated WhatsApp-plugin forward action with docs and tests for target authorization, message lookup/retention, and media forwarding; widen shared action typing only if maintainers want `forward` to be a cross-channel contract.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken current contract. Source inspection of discovery, handler, config schema, shared action names, docs, and listener types shows no current WhatsApp `forward` action.

## Solution Assessment

Unclear: the requested shape is plausible, but the best solution should first settle privacy gating and whether `forward` belongs in the shared message action contract or only in the WhatsApp plugin.
