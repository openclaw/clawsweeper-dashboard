## Summary

Keep open: current main still lacks WhatsApp message forwarding, and the request is a new WhatsApp plugin capability that needs maintainer decisions around privacy, target authorization, config gating, and the action contract.

## What This Changes

The issue requests a WhatsApp `forward` action that can forward an existing message, including audio, images, and documents, to another chat.

## Best Possible Solution

Design a WhatsApp-owned forwarding action with explicit target authorization, message lookup/retention rules, config gating, docs/tests, and shared action typing only if maintainers want forwarding exposed generically.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection of current main verifies that forwarding is absent from shared action names, WhatsApp discovery, dispatch, runtime handling, config schema, and relevant docs.

## Solution Assessment

Unclear: the requested `action=forward` shape is plausible, but it is not enough by itself. The maintainable solution needs a safety model, target-resolution rules, and a decision on whether the action is WhatsApp-specific or shared across channels.
