## Summary

Keep open: current main and the latest release still show the Discord native `/voice` command can be routed through the Talk Voice plugin before TTS handling, and the linked repair PR is closed unmerged with no newer canonical fix found.

## What This Changes

The issue requests that Discord `/voice list` provide usable voice selection or a clear fallback and `/voice chat <text>` send outbound audio instead of status-like output.

## Best Possible Solution

Fix Discord native command routing so provider-resolved `/voice` builds and dispatches the core TTS command, while plugin commands on Discord are matched through their Discord native names such as `/talkvoice`.

## Reproduction Assessment

Yes, source-level. Current main maps Discord TTS to `/voice`, Talk Voice also registers canonical `/voice` with Discord native `/talkvoice`, and the dispatcher can match the plugin command before TTS dispatch.

## Solution Assessment

Yes for the narrow repair: preserving the documented `/voice` TTS and `/talkvoice` Talk Voice split is the maintainable fix. Adding `/vc list` or `/vc chat` is separate product work and was already rejected as the wrong fix path in the closed linked PR.
