## Summary

Keep open: current main still shows a source-level Discord command collision where Discord `/voice` can be matched as the Talk voice plugin command instead of the documented Discord TTS command, and the only linked implementation PR is closed unmerged.

## What This Changes

Reporter asks for Discord `/voice list` and `/voice chat` to provide usable voice selection and outbound audio instead of status-only text.

## Best Possible Solution

Make Discord native dispatch resolve plugin commands with provider-native naming so Discord `/voice` stays on TTS and `/talkvoice` owns Talk voice selection.

## Reproduction Assessment

Yes, source-level: current main maps Discord TTS to `/voice`, Talk voice registers canonical `/voice` with Discord alias `/talkvoice`, and Discord native dispatch calls plugin matching without provider-native context. I did not run a live Discord session.

## Solution Assessment

Yes for the narrow repair: provider-native plugin command matching in Discord dispatch is the maintainable fix. Adding `/vc chat` or repurposing `/voice chat <text>` would be a separate feature/UX decision, not the regression fix.
