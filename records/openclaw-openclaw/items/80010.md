Keep open: current main still routes Discord native `/voice` through the Talk voice plugin path before the TTS command path, and the only linked implementation PR was closed unmerged.

## What This Changes

Discord `/voice list` and `/voice chat` report Talk voice status/list behavior instead of providing a usable Discord TTS/audio flow.

## Best Possible Solution

Make Discord native-command dispatch respect provider-native plugin aliases so `/voice` remains TTS and `/talkvoice` owns Talk voice config, with focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes, source-level: current main shows Discord `/voice` can be matched as the Talk voice plugin command before the TTS path. I did not run a live Discord session, so this is not a full end-to-end reproduction.

## Solution Assessment

Yes for the narrow fix: restore the documented `/voice` TTS and `/talkvoice` Talk voice split. Adding `/vc chat` or redefining `/voice chat` is not the best fix without a separate product decision.
