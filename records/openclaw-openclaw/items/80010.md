Keep open: current main still has the source-level Discord `/voice` routing problem, and the only linked implementation PR was closed unmerged after review found it did not fix the documented command split safely.

## What This Changes

The issue reports that Discord `/voice list` and `/voice chat` do not provide usable voice selection or outbound audio on OpenClaw 2026.5.7.

## Best Possible Solution

Restore the documented split so Discord `/voice` reaches TTS, Discord `/talkvoice` manages Talk voice selection, and one-shot audio guidance points to `/voice audio <text>`.

## Reproduction Assessment

Yes, source-level: current main can still build a Discord native `/voice` prompt that generic plugin dispatch can match as Talk voice before TTS handling. I did not live-run Discord, so this is not a live UI reproduction.

## Solution Assessment

Yes for the narrow repair: fix Discord native command routing so provider-specific native aliases do not let canonical plugin names preempt built-in TTS. Adding `/vc list` or `/vc chat` is not the best fix without separate UX and authorization work.
