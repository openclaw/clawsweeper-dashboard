## Summary

Keep open: current main still has a source-level Discord native `/voice` routing collision where the Talk voice plugin can intercept the documented Discord TTS command, and the only linked fix PR is closed unmerged.

## What This Changes

The issue reports that Discord `/voice list` is not usable for voice selection and `/voice chat ...` does not produce outbound audio.

## Best Possible Solution

Preserve the documented split: Discord `/voice` should execute TTS, Discord `/talkvoice` should manage Talk voice selection, and one-shot audio guidance should use `/voice audio <text>`.

## Reproduction Assessment

Yes, source-level: current main shows Discord native `/voice` can be matched as the Talk voice plugin command before TTS handling. I did not run a live Discord session, so this is not a live UI reproduction.

## Solution Assessment

Yes for the narrow fix: pass the Discord channel/native-name context through native plugin matching so `/voice` stays TTS and `/talkvoice` stays Talk voice. Adding `/vc chat` or changing `/voice chat <text>` semantics is not the best fix without separate UX approval.
