## Best Possible Solution

Restore the documented split so Discord `/voice` runs the TTS command, Discord `/talkvoice` manages Talk voice selection, and one-shot audio guidance remains `/voice audio <text>` unless maintainers choose a new alias.

Yes, the narrow maintainable solution is to make Discord native plugin matching respect the provider-native command surface so `/voice` reaches TTS while `/talkvoice` reaches Talk voice.
