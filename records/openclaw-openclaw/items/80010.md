## Summary

Keep open: current main still has the source-level Discord `/voice` routing problem, and the only linked implementation PR is closed unmerged. The narrow remaining fix is to keep Discord native `/voice` on the TTS path while routing Talk voice through `/talkvoice`.

## What This Changes

The issue reports Discord `/voice list` returning no usable voice selection and `/voice chat <text>` returning status-like output instead of outbound audio.

## Best Possible Solution

Use channel-aware plugin command matching in Discord native dispatch so Discord `/voice` stays on TTS, `/talkvoice` owns Talk voice selection, and focused tests lock that split down.

## Reproduction Assessment

Yes, source-level: current main registers Discord TTS as `/voice`, registers Talk voice as canonical `/voice` with Discord alias `/talkvoice`, and then calls `matchPluginCommand` without Discord channel context. I did not run a live Discord session, so the exact UI output is not independently observed.

## Solution Assessment

Yes for the narrow fix: Discord dispatch should pass channel context into plugin matching and preserve the documented `/voice` TTS versus `/talkvoice` Talk voice split. Adding `/vc chat` or making `/voice chat <text>` a new audio alias is not the best fix without separate maintainer UX approval.

## Repair Work Prompt

Fix the Discord native `/voice` command collision reported in https://github.com/openclaw/openclaw/issues/80010. On current main, `extensions/discord/src/monitor/native-command.ts` calls `nativeCommandRuntime.matchPluginCommand` without `{ channel: "discord" }` both while resolving the native command definition and while dispatching the runtime prompt; this lets Talk voice canonical `/voice` intercept Discord's native `/voice` TTS command even though Talk voice declares `nativeNames.discord = "talkvoice"`. Update Discord native dispatch to use channel-aware plugin matching so `/voice` falls through to the built-in TTS command and `/talkvoice` still executes Talk voice status/list/set. Do not add `/vc list`, `/vc chat`, or new `/voice chat <text>` semantics; one-shot audio should remain the documented `/voice audio <text>` Discord TTS path unless maintainers separately choose a new alias. Add focused regression coverage in `extensions/discord/src/monitor/native-command.plugin-dispatch.test.ts` and keep or extend `src/plugins/commands.test.ts` as needed. Add a changelog entry if the final patch is user-visible. Validate with `node scripts/run-vitest.mjs extensions/discord/src/monitor/native-command.plugin-dispatch.test.ts src/plugins/commands.test.ts` and `git diff --check`.
