## Summary

Keep open: current main shows a real Discord command-surface mismatch, but the requested `/voice chat <text>` behavior is not the documented TTS contract and changing it would conflict with existing `/tts chat` semantics.

## What This Changes

The issue asks for Discord `/voice list` to provide usable voice selection and `/voice chat <text>` to send outbound audio instead of status-like text.

## Best Possible Solution

Reconcile the Discord voice-command UX so users can discover Talk voice selection via `/talkvoice list` and one-shot TTS via `/voice audio <text>`, or intentionally add a new alias without breaking `/tts chat`.

## Reproduction Assessment

No high-confidence live reproduction is available because the screenshots/logs were not attached. Source inspection does reproduce the key mismatch: Discord `/voice` routes to TTS, while Talk voice listing is exposed as `/talkvoice list`.

## Solution Assessment

No clear code fix should be applied automatically yet. The safest path is a maintainer UX decision on whether to clarify the existing commands, expand native choices/help, or add a new alias for one-shot audio.
