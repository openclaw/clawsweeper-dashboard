## What This Changes

The issue asks Discord `/voice list` to expose usable voices and `/voice chat <text>` to send outbound audio instead of status-only replies.

## Best Possible Solution

Fix the documented command split: Discord `/voice` should reach TTS, `/talkvoice` should manage Talk voice selection, and one-shot audio users should be guided to `/voice audio <text>` unless maintainers intentionally add a new alias.
