## Summary

Keep open: current main still shows a Discord voice-command UX mismatch, but the requested `/voice chat <text>` behavior is not the existing TTS contract and should not be changed automatically without a maintainer decision.

## What This Changes

The issue asks Discord `/voice list` to expose usable voice choices and `/voice chat <text>` to send outbound audio in-channel.

## Best Possible Solution

Reconcile the Discord voice-command UX by explicitly surfacing `/talkvoice list` for Talk voice selection and `/voice audio <text>` for one-shot TTS, or intentionally add a new alias after deciding the command semantics.

## Reproduction Assessment

No high-confidence live reproduction is available because the screenshots/logs were not attached. Source inspection does reproduce the key mismatch: Discord native `/voice` maps to TTS, while Talk voice listing is exposed as `/talkvoice list`.

## Solution Assessment

No automatic code fix is clearly the best solution yet. The narrow maintainable path is a maintainer UX decision on whether to clarify existing commands, expand Discord native choices/help, or add a deliberate alias without breaking `/tts chat` semantics.
