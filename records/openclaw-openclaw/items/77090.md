## Summary

Keep open: current main does not implement the requested auto-revert setting or policy, and open PR #77209 explicitly targets this issue, so the issue should remain open until that PR or a successor lands or is declined.

## What This Changes

Request automatic return to the configured primary/default model after image analysis instead of leaving the session on a vision-capable model.

## Best Possible Solution

Review #77209 or a successor that defines a clear, backwards-compatible image-analysis reversion policy while preserving explicit user-driven model switches.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current docs define `/model` and Control UI model changes as persistent session overrides. Source inspection shows image analysis has a separate image-model path but no turn-completion revert contract on main.

## Solution Assessment

Unclear: the linked opt-in config approach is plausible, but changing when a session model reverts is a model-selection UX decision. The narrowest maintainable path is to settle that policy in the open PR before implementation lands.
