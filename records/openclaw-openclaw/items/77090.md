## What This Changes

The issue requests automatic reversion from an image-capable model back to the configured primary model after image analysis completes.

## Best Possible Solution

Keep the issue open while PR #77209 is reviewed; any accepted fix should preserve the user-vs-auto override contract, add focused tests, and update config docs/changelog if a new option ships.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current docs define `/model`, model pickers, and `sessions.patch` as persistent selections, while image analysis already has separate image-model routing. The source review confirms the requested behavior is not an existing documented contract on current main.

## Solution Assessment

Unclear: the linked PR's opt-in config approach is plausible, but maintainers should first decide whether image-triggered fallback should be temporary, whether manual vision-model switches should remain persistent, and how this interacts with existing auto fallback state.
