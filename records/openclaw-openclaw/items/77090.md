## Summary

Keep open: current main treats explicit model selection as a persistent session override, while image analysis already has separate `agents.defaults.imageModel` routing that does not change the session model. Auto-reverting after vision use would be a new model-selection policy, not cleanup for behavior already implemented on main.

## What This Changes

The issue requests automatic reversion from a vision/image model to the configured primary chat model after image analysis completes.

## Best Possible Solution

Define the model-selection policy first: keep automatic image analysis on `agents.defaults.imageModel` without mutating session selection, and add an opt-in temporary/revert mode only if maintainers want manual vision-model switches to be non-persistent.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current docs and code define `/model`, picker, `session_status(model=...)`, and `sessions.patch` as persistent user selections, while the image tool has a separate image-model path. The issue requests a new convenience policy rather than showing current main violating an existing contract.

## Solution Assessment

Unclear: automatic reversion could conflict with documented persistent session-model semantics. The narrow maintainable path is a maintainer UX decision about whether this should be an opt-in setting, a different command, or guidance to use `agents.defaults.imageModel` for image analysis.
