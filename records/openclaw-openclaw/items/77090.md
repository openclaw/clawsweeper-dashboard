## Summary

Keep open: current main documents and tests persistent model-selection behavior, and an open linked PR already proposes this feature with closing syntax, so this issue should stay open until that PR is reviewed, merged, or closed.

## What This Changes

Request automatic reversion from a vision/image-analysis model back to the configured primary model, optionally behind a new `agents.defaults.revertAfterImageModel` setting.

## Best Possible Solution

Decide the model-selection policy in the linked PR, then either land an opt-in auto-revert implementation for fallback-owned image/PDF turns with docs/tests or close the PR and keep users on explicit `agents.defaults.imageModel` plus `/model default`.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current docs and tests define persistent `/model` and auto fallback behavior. The requested change is reproducible from source as missing feature behavior, not as a violation of an existing contract.

## Solution Assessment

Unclear: the proposed opt-in config is a plausible narrow implementation, but maintainers should first choose whether fallback-owned vision/PDF model selections should expire automatically or remain durable recovery state.
