## Summary

Keep open: current main has separate image-model/media-understanding paths, but manual model selection is documented and implemented as persistent session state, so auto-reverting after image analysis is a new model-selection policy rather than an implemented fix.

## What This Changes

The issue requests automatic return to the configured primary model after image analysis, possibly via a new `revertAfterImageModel` setting.

## Best Possible Solution

Define a clear session model policy that keeps automatic image analysis on `agents.defaults.imageModel`/media-understanding without persisting `/model`, and only add an opt-in revert setting if maintainers want explicit manual vision switches to be temporary.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current docs and tests define `/model` as persistent session state, while image analysis already has separate image-model paths. The report describes a requested UX/policy change rather than a violation of an existing contract.

## Solution Assessment

Unclear: automatic reversion could be useful, but it changes documented session model semantics. Maintainers should first decide whether manual vision-model switches should be temporary or whether users should be steered to `agents.defaults.imageModel` for image turns.
