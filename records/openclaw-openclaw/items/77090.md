## Summary

Keep open: current main intentionally treats `/model` and auto fallback model overrides as persistent session state, and open PR #77209 is the active closing implementation path. Closing this issue before that PR, or a replacement, lands would be premature.

## What This Changes

Request automatic return to the configured primary/default chat model after image or PDF/image-model analysis, optionally via a new `agents.defaults.revertAfterImageModel` setting.

## Best Possible Solution

Review and either land PR #77209 or replace it with a media-specific implementation that records why an auto override was selected, reverts only image/PDF analysis overrides if the product policy accepts it, and keeps ordinary text failover sticky.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current docs and tests define `/model` and auto-fallback overrides as persistent session state. Source inspection confirms the reported behavior is current policy, not a violation of an existing contract.

## Solution Assessment

Unclear: the active PR is the right place to evaluate the change, but its current approach clears all auto fallbacks rather than media-specific ones. The safer solution needs an owner decision and a narrower media-source signal.
