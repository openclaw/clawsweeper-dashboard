## Summary

Keep open: current main and v2026.6.6 still lack a post-image-analysis revert policy or `revertAfterImageModel` config, and the open implementation at https://github.com/openclaw/openclaw/pull/78486 is not merged and still has proof, correctness, and mergeability concerns.

## What This Changes

The issue requests automatic return from a vision/image-analysis model to the configured primary model after image analysis completes.

## Best Possible Solution

Decide the product policy first, then land a corrected opt-in implementation or fold it into the broader auto-revert policy work at https://github.com/openclaw/openclaw/issues/39022 with media/PDF-scoped completion detection, full persisted state cleanup, docs, and regression tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a feature request. Source inspection confirms current main has no `revertAfterImageModel` config or post-image revert policy, while docs describe sticky auto-fallback behavior until recovery or reset.

## Solution Assessment

Unclear until maintainer product review. An opt-in flag is safer than unconditional reversion, but the open PR's current shape is not the best implementation because prior review found it was not tied to an explicit media/PDF completion signal and did not persist all cleared fallback state.
