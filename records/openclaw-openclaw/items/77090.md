## Summary

Keep open: current main already has separate image-model/media-understanding paths, but session model selection is intentionally persistent, so auto-reverting after image analysis is a new UX/model-selection policy rather than an already-defined bug fix.

## What This Changes

The issue requests an automatic switch back to the configured primary/default model after a vision/image-analysis turn, optionally controlled by a new `revertAfterImageModel` setting.

## Best Possible Solution

Define a clear session model policy: keep automatic image analysis on the existing `agents.defaults.imageModel` routing path, and only add an opt-in revert behavior if maintainers decide manual vision-model switches should be temporary.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current docs and source define `/model` and `sessions.patch` as persistent session selections. The current image-analysis code already has a separate `agents.defaults.imageModel` path, so the request is for new behavior rather than a broken existing contract.

## Solution Assessment

Unclear: the proposed config option is plausible, but automatic reversion changes model-selection semantics. A safer maintainer path is to first decide whether manual vision-model switches should be temporary or whether users should configure image analysis through `agents.defaults.imageModel`.
