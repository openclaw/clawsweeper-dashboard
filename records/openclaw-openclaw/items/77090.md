## Summary

Keep open: current main still has no post-image model reversion policy, and the existing code, docs, and tests intentionally preserve session model and auto-fallback selections across later turns. This remains a feature/config/product decision, with an open implementation attempt still needing review and proof.

## What This Changes

Request automatic return to the configured primary/default model after image or vision analysis, either through a new config option or image-turn detection.

## Best Possible Solution

Decide the session model policy first; if accepted, implement a media/PDF-specific auto-revert that triggers only after real media analysis and clears/persists the full session override state with docs and tests.

Not applicable as a strict bug reproduction: current source, docs, and tests define persistent `/model` and auto-fallback behavior. This issue asks for a new post-image policy rather than a broken documented behavior.

## Solution Assessment

Unclear: the requested behavior may be useful, but the maintainable fix needs an explicit image/PDF completion signal and full session-state cleanup. A broad post-run reset would conflict with current fallback persistence semantics.
