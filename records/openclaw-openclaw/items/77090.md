## Summary

Keep open: current main has separate image-model/media-understanding paths, but `/model` is documented and implemented as a persistent session selection, so automatic reversion would be a new model-selection policy rather than already-implemented behavior.

## What This Changes

Request to automatically return a session to the configured primary/default model after image analysis instead of leaving a vision model selected.

## Best Possible Solution

Define a clear session model policy that keeps automatic image analysis on `agents.defaults.imageModel`/media-understanding without persisting `/model`, and only add an opt-in revert setting if maintainers want explicit manual vision switches to be temporary.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current docs and code define `/model` as persistent, while image analysis already has separate image-model paths. The report does not include a concrete current-main flow that violates an existing contract.

## Solution Assessment

Unclear: the proposed config is plausible, but automatic reversion changes model-selection semantics. Maintainers should first decide whether manual vision-model switches should be temporary or whether users should be steered to `agents.defaults.imageModel`.
