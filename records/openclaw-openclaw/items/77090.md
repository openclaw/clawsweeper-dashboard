## What This Changes

The issue requests an opt-in or automatic policy that returns sessions from a vision/image model to the configured primary model after image or PDF analysis.

## Best Possible Solution

Review #77209 as the implementation candidate and settle the model-selection policy around `modelOverrideSource: "auto"` before adding any schema/docs/tests/changelog changes.

## Reproduction Assessment

Not applicable as a strict bug reproduction: the request changes model-selection behavior, while current docs define `/model` and picker changes as persistent session overrides. Source inspection verifies image/PDF tooling has separate image-model fallback paths but no completed-turn auto-revert policy.

## Solution Assessment

Unclear: an opt-in `revertAfterImageModel` setting is plausible, but the narrow maintainable solution depends on whether maintainers want automatic fallback overrides, manual vision-model selections, or both to become temporary.
