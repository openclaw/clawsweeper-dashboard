Keep open: current main does not implement an auto-revert policy, and open PR #77209 explicitly tracks this issue with a candidate implementation, so the issue should remain open until that PR is reviewed, merged, or closed.

## What This Changes

Request automatic reversion from an image/vision model back to the configured primary model after image analysis, possibly through an opt-in agent default.

## Best Possible Solution

Review #77209 against the model-selection contract and, if accepted, implement an opt-in policy that only clears automatic image/fallback state while preserving explicit user model selections.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current docs and code define persistent user model selections and separate image-model/fallback behavior. The remaining question is a new UX/config policy, not a current-main violation of an existing contract.

## Solution Assessment

Unclear until maintainer review: an opt-in setting is a plausible narrow fix, but automatic reversion changes session model semantics. The safer direction is to settle the policy in #77209 and keep explicit user selections persistent.
