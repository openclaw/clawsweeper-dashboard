## Summary

Keep open: current main separates image-model routing from persistent session model selection, and automatic reversion after image analysis would be a new model-selection policy rather than already-implemented behavior.

## What This Changes

Request automatic return to the configured primary/default chat model after a vision/image analysis turn, optionally via a new `revertAfterImageModel` setting.

## Best Possible Solution

Define the intended session model policy first: keep explicit user model selections persistent while routing image analysis through `agents.defaults.imageModel`, or add a deliberate opt-in temporary image-mode setting with docs and regression coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current docs and code define `/model` and session model overrides as persistent user selections. The requested behavior is a new convenience policy for mixed image/text sessions, not a violation of an existing contract.

## Solution Assessment

Unclear: automatic reversion may be useful, but it changes model-selection semantics. Maintainers should decide whether only automatic media routing should use `agents.defaults.imageModel` or whether explicit vision-model switches should become temporary behind an opt-in setting.
