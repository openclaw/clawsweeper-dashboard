## Summary

Keep open: current main documents and implements `/model` as a persistent session selection, while image analysis already uses separate image-model fallback paths, so auto-reverting would be a new UX policy/config choice rather than a cleanup-close item.

## What This Changes

The issue requests automatic reversion from a vision/image model back to the configured primary model after image analysis.

## Best Possible Solution

Define the model-selection policy first: keep automatic image analysis on `agents.defaults.imageModel` without altering session overrides, and add opt-in revert behavior only if maintainers want manual vision-model selections to be temporary.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this requests new model-selection behavior, while current docs and source define manual model switches as persistent session overrides. Source inspection shows image analysis already has separate image-model fallback paths rather than mutating the session model.

## Solution Assessment

Unclear: automatic reversion or a `revertAfterImageModel` setting may be useful, but maintainers need to decide whether persistent `/model` and picker selections should become temporary around image analysis.
