## Summary

Keep open: current main and v2026.6.9 still reject package versions with SemVer build metadata, but this PR strips after `+` before validating, so malformed versions can pass the release packaging guard; it also still lacks inspectable real `pnpm pack` proof.

## What This Changes

The PR drops SemVer build metadata for package changelog heading lookup and adds stable/beta metadata unit expectations.

## Best Possible Solution

Validate the complete OpenClaw package version with optional valid SemVer build metadata, strip only a validated suffix for changelog lookup, cover malformed suffix cases, and attach redacted pack output or an artifact.

## Reproduction Assessment

Yes. A read-only current-main node probe of `resolvePackageChangelogVersions("2026.6.6+jm")` reproduces the unsupported-version failure, and both prepack package paths call the same changelog helper.

## Solution Assessment

No, not as written. Normalizing valid build metadata to the base changelog identity is the right fix boundary, but the full version must be validated before `baseVersion` is derived.
