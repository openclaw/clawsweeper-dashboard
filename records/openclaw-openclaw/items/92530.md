## Summary

Keep this PR open because it targets a real release-packaging bug, but the current patch is not merge-ready: it strips build metadata before validating the full package version and still lacks inspectable real `pnpm pack` proof.

## What This Changes

The PR changes changelog packaging to resolve package versions with SemVer build metadata to the base changelog section and adds positive regression expectations for stable and prerelease metadata versions.

## Best Possible Solution

Accept valid SemVer build metadata only after validating the complete package version, derive changelog lookup keys from the validated base release/prerelease, keep malformed metadata fail-closed, and attach inspectable `pnpm pack` proof.

## Solution Assessment
