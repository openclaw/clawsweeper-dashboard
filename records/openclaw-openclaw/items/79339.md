## What This Changes

The PR updates embedded Pi session media directive handling and tests so final `MEDIA:` directives and structured tool media are preserved for outbound replies.

## Best Possible Solution

Land after media-only `text_end` delivery is recorded or deduped before `message_end`, regression coverage proves one delivery, and `CHANGELOG.md` records the user-facing fix.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction: a final `MEDIA:` directive can remain buffered until an empty final flush, and the existing `text_end` path returns before consuming it.

## Solution Assessment

No. The approach is mostly right, but the patch must also prevent a media-only `text_end` flush from being replayed at `message_end`; adding the missing changelog entry is the other required cleanup.
