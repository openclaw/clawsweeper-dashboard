Keep open. Current main still leaves the parent Docker cache directory root-owned in the browser-enabled build path, and the PR's one-line ownership widening is the right narrow fix; it still needs the repo-required changelog entry and focused validation before merge. ## What This Changes

The PR changes the optional browser-enabled Docker build block to chown `/home/node/.cache` instead of only `/home/node/.cache/ms-playwright`.

## Best Possible Solution

Merge the focused ownership fix with a changelog entry, a small ownership assertion, and Docker validation, or explicitly replace it with #42771 if maintainers choose the broader `/opt/playwright` migration.

## Solution Assessment

Yes. For the currently documented `/home/node/.cache/ms-playwright` path, chowning the parent cache directory is the narrowest maintainable fix; #42771 is a broader alternative, not a prerequisite for this defect.
