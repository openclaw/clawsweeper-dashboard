## What This Changes

The PR changes the optional Docker browser-install block to chown `/home/node/.cache` instead of only `/home/node/.cache/ms-playwright`.

## Best Possible Solution

Merge a focused ownership fix after adding release-note credit and targeted Docker/browser validation; if maintainers prefer the `/opt/playwright` design, make #42771 or a refreshed replacement the canonical path and close this PR only after that direction is explicit or merged.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence reproduction path: in browser-enabled Docker builds, root creates `/home/node/.cache/ms-playwright`, the Dockerfile chowns only that child directory, and the runtime image later runs as `node`, so writes directly under `/home/node/.cache` can fail.

## Solution Assessment

Yes. For the current documented `/home/node/.cache/ms-playwright` path, widening the ownership target to `/home/node/.cache` is the narrowest maintainable fix; the remaining required addition is changelog coverage and Docker validation, not a broad rewrite in this PR.
