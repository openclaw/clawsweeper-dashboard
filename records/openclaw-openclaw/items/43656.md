  - body: The remote `sessions_spawn` branch sends a normal `agent` request to `agent:<id>:main`; it does not create an isolated child session, register lifecycle state, or preserve the current spawn runtime/sandbox/model/context contract. Add a real remote spawn RPC or leave this tool local-only until that contract exists.
  - confidence: 0.99

  - body: The remote spawn `agent` request omits `idempotencyKey`, but the Gateway `AgentParamsSchema` requires it. Cross-Gateway spawn calls will be rejected as invalid params before the remote run starts.
  - confidence: 0.99
- **[P2] Require credentials for token-auth peers:** `src/agents/tools/gateway-peer.ts:61-64`
  - body: The helper returns a URL override even when a peer has no token, but `callGateway` requires explicit credentials for URL overrides in normal token/password auth modes. Either require peer credentials here or route through an explicit paired-device/federation auth path so the documented tokenless peer shape is not misleading.
  - confidence: 0.87
- **[P2] Preserve sessionKey and label exclusivity:** `src/agents/tools/sessions-send-tool.ts:73-80`
  - body: The remote `sessions_send` path only checks that at least one target is present, so callers can pass both `sessionKey` and `label` and silently prefer `sessionKey`. Match the local path by rejecting ambiguous targeting before routing remotely.
