  - body: The new remote path takes `peerOpts.gatewayUrl` and passes it directly to `callGateway`, after resolving either a configured peer or raw `gatewayUrl`. Cross-gateway spawns will be rejected as invalid params before the remote run starts; generate and forward an idempotency key like the local agent paths do.
  - confidence: 0.98

  - body: `resolveGatewayPeerOptions` returns a URL override even when the peer has no token, and the PR documents/tests tokenless peers as valid. `callGateway` rejects explicit URL overrides without explicit credentials, so those peers fail before connecting; either require peer credentials in schema/help/tests or add a reviewed auth-none peer path.
  - confidence: 0.9
