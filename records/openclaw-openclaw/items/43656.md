## Summary

Keep open. The PR is a plausible implementation candidate for https://github.com/openclaw/openclaw/issues/43605, but the current head still has blocking protocol, auth, and security issues, and it lacks real two-Gateway behavior proof.

## What This Changes

The PR adds a `gateway.peers` config map, a peer-resolution helper, tests, and cross-Gateway `gateway`/`gatewayUrl`/`gatewayToken` branches for `sessions_send` and `sessions_spawn`.

## Best Possible Solution

Rework the patch around a documented peer auth/addressing contract, preserve local session-tool validation semantics, satisfy the Gateway `agent` protocol, and prove send plus spawn across two real Gateways before closing the linked tracker.

## Reproduction Assessment

Yes for the blocking PR defects from source inspection: remote `sessions_spawn` omits the required `idempotencyKey`, tokenless peers hit current `callGateway` explicit-auth requirements, and remote `sessions_send` diverges from the local `sessionKey`/`label` contract. I did not run the PR branch because this review is read-only.

## Solution Assessment

No. The `gateway.peers` direction is plausible, but this patch bypasses existing URL policy for ad-hoc URL overrides and fails the current agent protocol; the safer path is a reviewed peer-auth contract with consistent validation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Include idempotencyKey in remote spawn requests:** `src/agents/tools/sessions-spawn-tool.ts:167-179`
  - body: The remote `sessions_spawn` path calls the Gateway `agent` method without `idempotencyKey`, but `AgentParamsSchema` requires it. Cross-Gateway spawns will be rejected as invalid params before the remote run starts.
  - confidence: 0.95
- **[P1] Keep gatewayUrl behind the agent-tool URL policy:** `src/agents/tools/gateway-peer.ts:66-67`
  - body: The helper returns any caller-supplied `gatewayUrl`, and the session tools pass it directly to `callGateway`, so an agent tool call can target arbitrary ws(s) endpoints instead of the existing loopback/configured-remote allowlist.
  - confidence: 0.86
- **[P2] Require or explicitly support tokens for peer overrides:** `src/agents/tools/gateway-peer.ts:59-62`
  - body: Named peers may resolve to an explicit URL with `gatewayToken: undefined`, but `callGateway` rejects explicit URL overrides without explicit credentials. Tokenless peers therefore fail before reaching the remote Gateway despite the config type making `token` optional.
  - confidence: 0.91
- **[P2] Reject ambiguous remote send targets:** `src/agents/tools/sessions-send-tool.ts:70-80`
  - body: The remote branch allows `sessionKey` and `label` together and silently uses `sessionKey`. The local path rejects that ambiguous combination, and the remote path should keep the same contract to avoid routing to the wrong session.
  - confidence: 0.88
