The issue requests caching the builtin memory manager's indexed-content check so repeated memory searches do not query SQLite every time.

## Best Possible Solution

The end state should be a memory-manager-owned cached indexed-content state with complete invalidation coverage and focused regression tests for warm searches, writes, deletes, reset, and in-place reindex DB swaps.
