## Summary

Keep open. Current `main` still has the reported repeated SQLite preflight probes, and #62597 is an open, unmerged PR with closing syntax for this issue, so this issue should remain open until that PR lands or is replaced.

## What This Changes

Request caching `MemoryIndexManager.hasIndexedContent()` so repeated memory searches do not re-query SQLite on every preflight.

## Best Possible Solution

Land or replace #62597 with a memory-core-local indexed-content cache that updates after successful chunk writes and invalidates on deletes, stale cleanup, reset, and database replacement paths.

## Reproduction Assessment

Yes. On current `main`, a nonblank `MemoryIndexManager.search()` reaches `hasIndexedContent()`, and that method directly prepares SQLite `SELECT 1` probes; a focused regression test can count `db.prepare` calls across repeated searches.

## Solution Assessment

Yes. Caching inside `memory-core` with exhaustive invalidation around index mutations is the narrowest maintainable fix; the existing open PR is the right implementation lane if it is refreshed and reviewed.
