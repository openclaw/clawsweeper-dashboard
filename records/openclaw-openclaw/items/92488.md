## Summary

This PR identifies a real OpenResponses guard bug, but the proposed fix is incomplete: it forwards an empty message into shared ingress validation, which throws before the image can reach the agent. The mocked endpoint test hides that downstream failure, and real running-gateway proof is still required.

## What This Changes

The PR allows image-only `/v1/responses` input past the missing-message guard and adds regression tests for image-only and empty-content requests.

## Best Possible Solution

Normalize the active OpenResponses image-only user turn into the same non-empty prompt representation used by Chat Completions, preserve the 400 response for input with neither text nor images, and cover the shared ingress boundary.

## Reproduction Assessment

Yes. Current main rejects the image-only request at the OpenResponses HTTP guard, while source tracing of this patch shows it would instead fail the shared agent command's blank-message validation.

## Solution Assessment

No. Relaxing the HTTP guard is necessary but not sufficient because the sibling endpoint also normalizes its active image-only turn into a non-empty prompt before shared ingress.
