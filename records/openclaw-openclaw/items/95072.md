## Summary

Keep open: current main and the latest release still take the in-process fallback for env-less Windows restarts, and https://github.com/openclaw/openclaw/pull/95095 is the open closing PR for the safer Scheduled Task detection path.

## What This Changes

Request to make Windows `/restart` detect scheduled-task supervision when service environment markers are missing, so restart can use `schtasks` instead of the in-process fallback.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/95095 with real Windows PID/uptime proof and task-name coverage, or explicitly reaffirm the current in-process fallback contract.

## Reproduction Assessment

Yes, source-level: current main returns disabled on `win32` without Scheduled Task markers, and the run loop then takes the in-process restart fallback. I did not run a live Windows gateway in this read-only review.

## Solution Assessment

Unclear: the original detached-respawn proposal is not the best fix because it reopens the orphan-child risk. The linked `schtasks` detection path is the safer candidate, but it needs maintainer agreement and live Windows proof.
