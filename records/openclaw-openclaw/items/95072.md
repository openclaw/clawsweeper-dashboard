## Summary

Keep open: current main and the latest release still take the env-marker/in-process fallback path for env-less Windows restarts, and https://github.com/openclaw/openclaw/pull/95095 is an open closing candidate rather than a merged fix.

## What This Changes

This issue requests a Windows `/restart` fix so env-less manually started gateways with an installed Scheduled Task can perform a real process restart instead of preserving PID and uptime.

## Best Possible Solution

Review and land a Windows-safe schtasks-aware restart fix, likely through https://github.com/openclaw/openclaw/pull/95095 after exact task detection and Windows proof, or explicitly keep/document the in-process fallback if maintainers choose that contract.

## Reproduction Assessment

Yes, source-level: current main returns no Windows supervisor for missing env markers, then disables full-process restart and the run loop falls back to in-process restart. I did not run a live Windows gateway in this read-only review.

## Solution Assessment

Unclear: detached respawn is not the best fix because it reopens the orphaned-child risk, while the schtasks-probe direction is plausible but still needs exact task-name handling, maintainer contract review, and Windows proof.
