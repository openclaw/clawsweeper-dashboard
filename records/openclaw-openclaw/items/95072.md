## Summary

Keep open: current main and v2026.6.9 still show the env-only Windows supervisor detection and in-process fallback, and https://github.com/openclaw/openclaw/pull/95095 is the open closing candidate that should be reviewed before this issue closes.

## What This Changes

Windows /restart should produce a real gateway process restart instead of an in-process restart when Scheduled Task markers are missing.

## Best Possible Solution

Review and land a narrow Windows-safe schtasks detection fix, or explicitly keep and document the in-process fallback if maintainers reject fresh-PID semantics for this setup.

## Reproduction Assessment

Yes, at source level: current main returns disabled for win32 without Scheduled Task markers, and the run loop then takes the in-process restart fallback. I did not run a live Windows gateway in this read-only review.

## Solution Assessment

Unclear: the original detached-respawn direction is not the best path because it reopens the orphaned-child risk, while the linked schtasks-probe PR is safer but still needs maintainer review and live Windows proof.
