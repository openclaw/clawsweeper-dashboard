## What This Changes

The issue requests that Windows `/restart` perform a real gateway process restart when Scheduled Task supervision exists but supervisor environment markers are missing.

## Best Possible Solution

Review, repair, and either land or reject the linked schtasks-probe PR; do not create a duplicate fix while that candidate remains open.

## Reproduction Assessment

Yes, at source level: on win32 with no Scheduled Task/env markers, current main returns no supervisor and disables detached respawn, which drives the run loop into the in-process fallback. I did not run a live Windows Scheduled Task scenario in this read-only review.

## Solution Assessment

Unclear until the linked PR is reviewed; direct Scheduled Task probing is the safer direction than unsafe detached respawn, but exact task-name and upgrade behavior need maintainer review.
