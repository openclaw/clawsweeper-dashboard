## Summary

Keep open: current main and v2026.6.10 still detect Windows Scheduled Task supervision only from environment markers, then fall back to an in-process restart when those markers are absent; the linked open PR is the active candidate fix.

## What This Changes

Fix Windows `/restart` so an env-less Scheduled Task install can perform a real gateway process restart instead of reporting success while keeping the same PID.

## Best Possible Solution

Review and land or narrow the linked schtasks-probe PR while preserving the safe Windows restart contract and avoiding unmanaged detached respawn.

## Reproduction Assessment

Yes, at source level: on win32 with no Scheduled Task env markers, current main returns no supervisor and disables detached respawn, which sends the run loop into the in-process fallback. I did not run a live Windows Scheduled Task scenario in this read-only review.

## Solution Assessment

No to the original detached-respawn proposal; the safer direction is Scheduled Task detection and handoff, but the linked PR still needs maintainer review and Windows proof before it is the accepted fix.
