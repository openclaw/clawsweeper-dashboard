## Summary

Keep open: current main and the latest release still use the in-process fallback for env-less Windows restart, and an open linked PR now owns the safer schtasks-detection repair path for this issue.

## What This Changes

The issue asks OpenClaw to make Windows `/restart` produce a real gateway process restart when a manually started gateway lacks Scheduled Task environment markers.

## Best Possible Solution

Review and land a corrected version of https://github.com/openclaw/openclaw/pull/95095, or close that PR and choose a maintainer-approved Windows restart contract before closing this issue.

## Reproduction Assessment

Yes, at source level: env-less win32 supervisor detection returns null, `restartGatewayProcessWithFreshPid()` returns disabled, and the run loop falls back to in-process restart. I did not run live Windows proof in this read-only review.

## Solution Assessment

No for the original detached-respawn proposal; it would reverse a shipped no-orphan safety decision. The safer direction is a schtasks-based detector/handoff fix, with exact task-name handling and live Windows proof.
