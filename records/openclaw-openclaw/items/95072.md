## What This Changes

The issue asks OpenClaw to make Windows channel `/restart` hand off to a real process restart when an installed Scheduled Task exists but supervisor environment markers are missing.

## Best Possible Solution

Review and either land the linked Windows-safe Scheduled Task detection PR with real PID/uptime proof, or explicitly keep and document the shipped in-process fallback contract.

## Reproduction Assessment

Yes, at source level: current main returns no `schtasks` supervisor for env-less Windows and then keeps ordinary restart on the in-process fallback. I did not run a live Windows gateway in this read-only review.

## Solution Assessment
