## Summary

Keep open. Current main has plugin hook timeouts and some plugin/channel-local breakers, but it still lacks the requested framework-level plugin circuit breaker with plugin/hook lock attribution, persisted breaker state, operator status/reset controls, half-open/forced-open behavior, and per-plugin degradation policy; the production incident comment and open related report make this a maintainer runtime/product design item, not stale cleanup.

## What This Changes

Request a framework-level plugin circuit breaker that can degrade faulty plugin hooks or tools without taking down the gateway.

## Best Possible Solution

Design a core plugin-runtime resilience feature that starts with passive telemetry and plugin/hook attribution, then adds persisted breaker state, operator status/reset controls, explicit fail modes, and active open/half-open behavior once maintainer policy is settled.

Not applicable as a bug reproduction path: this is a framework feature/design request. Source inspection gives a high-confidence current-main gap check for the missing generic breaker and operator surfaces.

## Solution Assessment
