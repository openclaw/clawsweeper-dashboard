## Summary

Keep open. Current main has hook timeouts, fail-open/fail-closed hook policy, startup isolation for plugin registration failures, and some plugin-specific cooldowns, but it does not provide the requested framework-level plugin circuit breaker with lock attribution, persisted breaker state, status/reset controls, or per-plugin degradation policy. The discussion also has recent production context, and the overlapping https://github.com/openclaw/openclaw/issues/55253 remains open rather than superseding this broader design item.

## What This Changes

Request a framework-level plugin circuit breaker that isolates failing plugin hooks, tracks errors and lock overruns, persists breaker state, exposes operator visibility/reset controls, and supports graceful degradation policies.

## Best Possible Solution

Design a core plugin-runtime resilience feature that adds plugin/hook attribution, passive telemetry, persisted breaker state, operator status/reset controls, and explicit per-plugin fail modes while preserving existing plugin-local degradation where it is already safer.
