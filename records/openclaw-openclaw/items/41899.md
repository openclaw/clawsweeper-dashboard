## Summary

Keep open. Current main has per-hook timeouts, session-lock cleanup, and plugin-local breakers, but still lacks the requested framework-level plugin circuit breaker with plugin/hook lock attribution, persisted state, operator status/reset controls, half-open/forced-open behavior, and per-plugin degradation policy; the production incident comment makes this an active runtime design item rather than cleanup.

## What This Changes

Request a framework-level plugin circuit breaker that can isolate misbehaving plugin hooks and degrade only the affected plugin while keeping the gateway responsive.

## Best Possible Solution

Design a staged core plugin-runtime resilience feature: passive telemetry and plugin/hook lock attribution first, then persisted breaker state and status/reset controls, then active skip/half-open/forced-open behavior with explicit per-plugin fail modes.
