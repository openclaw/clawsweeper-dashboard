## Summary

Keep open: this is a plausible focused UI bug fix with sufficient screenshot proof, but the branch is stale against current main. Current main renders the active Chat session picker from the shared `ui/src/ui/chat/session-controls.ts` module, while this PR still patches the old inline helper body, so the proposed change would not reliably fix the live desktop/mobile controls after rebase.

## What This Changes

The PR changes Chat UI session select rendering and tests so fallback session options stay selected after rerender, with separate desktop/mobile selector markers in the old helper implementation.

## Best Possible Solution

Port the selected-option property binding and fallback-session rerender coverage to `ui/src/ui/chat/session-controls.ts`, preserving the merged session-preservation behavior from https://github.com/openclaw/openclaw/pull/59611.

## Reproduction Assessment

## Solution Assessment

No, not as currently proposed. The `live()` option-property approach is the right narrow mechanism, but it needs to move to the current shared session-control module instead of the old helper implementation.
