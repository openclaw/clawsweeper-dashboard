## Summary

Keep open: the reported selector desync is valid and the PR has usable before/after proof, but the submitted branch no longer patches the active current-main render path after the session picker was moved into the shared chat session-control module.

The PR updates Control UI Chat session select rendering, mobile selector targeting, and view tests so desktop and mobile session pickers reselect the active fallback option after rerender.

## Best Possible Solution

Keep the shared session-control module as the single picker owner, add the live selected-option binding there, and cover the fallback rerender case in the shared renderer tests.

## Solution Assessment
