## Summary

Keep open: the reported UI bug and proof are credible, but the submitted patch no longer lands on the active runtime path after current `main` refactored both desktop and mobile Chat session pickers into the shared session-control renderer.

## What This Changes

The PR changes Control UI Chat session picker rendering, mobile selector targeting/CSS, and view tests so fallback session options are reselected after rerender.

## Best Possible Solution

Move the live selected-option binding into `ui/src/ui/chat/session-controls.ts`, add rerender fallback coverage against the shared renderer, and leave the broader session-preservation behavior from https://github.com/openclaw/openclaw/pull/59611 unchanged.

## Reproduction Assessment

Yes, at source level. Current `main` still uses attribute-selected session options in the shared picker, and the inspected screenshots show the visible selector/thread mismatch before and after the branch fix.

## Solution Assessment

No, not as currently proposed. The live selected-property binding is the right narrow repair, but it must be applied to the current shared session-control renderer instead of the old helper path.
