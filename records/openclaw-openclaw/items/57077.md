## Summary

Keep open: the PR contains a useful selector-state fix, but current main now renders the Chat session dropdown through the shared `ui/src/ui/chat/session-controls.ts` path, while this branch patches the older helper-local renderer. As-is, the branch is conflicting and would not reliably fix the active runtime dropdown after rebase.

## What This Changes

The PR changes Control UI Chat session picker rendering and tests so desktop/mobile dropdown options keep the active session selected after rerenders, plus mobile selector class/CSS targeting.

## Best Possible Solution

Port the live selected-option binding into `ui/src/ui/chat/session-controls.ts`, keep the shared desktop/mobile renderer, and add focused rerender fallback-session coverage in `ui/src/ui/views/chat.test.ts`.

## Reproduction Assessment

Yes, at source level. Current main still uses attribute-based `?selected` in the active shared renderer, and the inspected PR screenshots show the before selector/content mismatch and after aligned behavior.

## Solution Assessment

No, not in this branch shape. The `live()` selected-property approach is the narrow maintainable fix, but it needs to be applied to the current shared session-control renderer rather than the old helper path.
