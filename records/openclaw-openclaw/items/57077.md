## Summary

Keep open: the bug and fix direction are valid, but this branch is stale against current `main` and applies the selected-option repair to the old inline helper instead of the shared selector now used by both desktop and mobile Chat controls.

## What This Changes

The PR applies live selected-option binding to Chat session picker options, adjusts mobile session-control CSS targeting, and adds fallback-rerender tests for desktop and mobile pickers.

## Best Possible Solution

Move the live selected-state repair and fallback-rerender coverage into `ui/src/ui/chat/session-controls.ts`, preserving the merged session-preservation behavior from https://github.com/openclaw/openclaw/pull/59611.

## Reproduction Assessment

Yes, at source level. Render the shared Chat session select, simulate stale DOM selection, change `state.sessionKey` to a fallback session, rerender, and assert both `value` and `selectedOptions[0].value` match the fallback session.

## Solution Assessment

No, not as currently proposed. The `live()` property-binding approach is the right narrow repair, but it needs to be applied to current main's shared session selector instead of the older inline helper implementation.
