## Summary

Keep open: the reported UI desync is plausible and the PR has useful proof, but the branch now patches an older inline helper path while current main renders Chat session pickers through the shared session-control module. The useful fix should be ported before merge.

## What This Changes

The PR attempts to keep the Chat session picker visually synced with the active session by binding option selection during rerender and separating desktop/mobile session-select targeting.

## Best Possible Solution

Keep `ui/src/ui/chat/session-controls.ts` as the single picker owner, add the live selected-option binding there, and cover the fallback rerender case in current shared-renderer tests.

## Reproduction Assessment

Yes, at source level. Current main still uses attribute-selected options in the shared Chat session picker, and the PR screenshots show the visible selector/thread mismatch before and after the branch fix.

## Solution Assessment

No, not as currently proposed. The live selected-option binding is the right narrow repair, but it needs to move to `ui/src/ui/chat/session-controls.ts` and current shared-renderer coverage instead of the old inline helper path.
