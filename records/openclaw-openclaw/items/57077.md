## Summary

This PR is useful but should not merge as-is: current main moved the active desktop and mobile Chat session picker rendering into the shared `session-controls.ts` module, while the branch applies the live selection binding to the old helper path.

## What This Changes

The PR updates Control UI Chat session picker rendering and tests to force the selected option state during fallback-session rerenders and adjust mobile selector targeting.

## Best Possible Solution

Keep `ui/src/ui/chat/session-controls.ts` as the single session-picker owner, bind the active option against live DOM state there, and add focused fallback-rerender coverage for desktop and mobile callers.

Yes, at source level. Current main still uses `?selected` in the shared session picker, and the missing case is a rerender where the live select state has drifted while the active session is injected as a fallback option.

## Solution Assessment

No, not as currently proposed. The live selected-option binding is the right narrow repair, but it needs to move from the obsolete helper renderer into the current shared `session-controls.ts` path.
