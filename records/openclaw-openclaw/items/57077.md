## Summary

## What This Changes

The branch changes Chat session picker option selection binding and mobile selector styling/tests so fallback session options are reselected after rerender.

## Best Possible Solution

Move the live selected-option binding into `ui/src/ui/chat/session-controls.ts`, add rerender fallback coverage against that shared renderer, and preserve the merged session-preservation behavior from https://github.com/openclaw/openclaw/pull/59611.

## Solution Assessment

No, not as currently proposed. The live selected-property binding is the right narrow repair, but it must be applied to the current shared session-control renderer instead of the old helper path.
