## Summary

Keep open: current main still suppresses the early accepted-message Discord typing cue for guild/channel messages, while this report is specifically about Discord channel/thread conversations. The request is narrow, source-reproducible, and aligned with the current messaging-channel stability priorities.

## What This Changes

The issue asks Discord to emit an immediate typing cue for accepted `typingMode: "instant"` channel/thread messages before session, memory, queue, or reply-run startup work can delay user-visible feedback.

## Best Possible Solution

Extend the Discord plugin's accepted-message early typing cue to actionable guild channel/thread messages when `typingMode` resolves to `instant`, while leaving the existing reply-run typing loop as keepalive.

## Reproduction Assessment

Yes from source: current main accepts the early cue only for direct messages and explicitly tests that guild messages do not send it. I did not run a live Discord gateway timing repro.

## Solution Assessment

Yes, the best fix is a narrow Discord plugin change at the existing post-preflight acceptance point, with regression tests for accepted guild/thread messages and unchanged suppression paths.
