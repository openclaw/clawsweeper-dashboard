## Summary

Keep open. Current main still runs bootstrap maintenance through the foreground default while deferred compaction execution is background-only and automatic background scheduling remains turn-only, so the reported OpenClaw host scheduling gap is still source-reproducible.

## What This Changes

The issue asks OpenClaw to prevent context-engine bootstrap/reconcile and hot-cache deferred compaction debt from remaining pending without a valid compaction execution path.

## Best Possible Solution

Add a generic bootstrap/reconcile background follow-up for engines already opted into background maintenance, and leave Lossless-Claw cache/dedup policy inside the plugin unless it exposes a concrete OpenClaw API gap.

## Reproduction Assessment

Yes, at source level: bootstrap maintenance still runs as foreground while deferred debt execution is background-only and background scheduling is still gated to turn maintenance. I did not run a live minimized LCM session in this read-only sweep.

## Solution Assessment

Yes for the narrow OpenClaw half: extend the existing deferred-maintenance lifecycle so bootstrap/reconcile can schedule a valid background debt consumer. The hot-cache and duplicate-ledger behavior should remain plugin work unless the repair proves a missing generic host API.
