## Summary

Keep open. Current main and v2026.4.29 still have the host scheduling gap: bootstrap maintenance runs as foreground work, while debt-consuming maintenance is only enabled for background runs that are currently scheduled from turn maintenance.

## What This Changes

The issue asks OpenClaw to prevent context-engine bootstrap/reconcile and hot-cache maintenance from leaving deferred compaction debt pending without a valid follow-up compaction path.

## Best Possible Solution

Add a generic host-level bootstrap/reconcile follow-up path for background-capable context engines so deferred debt can run with allowDeferredCompactionExecution enabled, while keeping lossless-claw-specific hot-cache policy outside core unless a generic API signal is needed.

## Reproduction Assessment

Partly yes: the host scheduling gap is reproducible from current source with a fake context engine by calling bootstrapHarnessContextEngine and observing foreground maintenance without allowDeferredCompactionExecution or a queued background follow-up. The hot-cache sticky-debt case has concrete live traces but still needs a synthetic plugin-side repro.

## Solution Assessment

Yes for the host half: a narrow scheduling or additive contract fix in context-engine maintenance is the maintainable path. The hot-cache half should be solved in the plugin or through a generic policy-superseded signal rather than embedding lossless-claw policy in OpenClaw core.
