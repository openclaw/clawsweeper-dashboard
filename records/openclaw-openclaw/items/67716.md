## Summary

Keep open. Current main still matches the reported host-side scheduling gap: bootstrap maintenance runs as foreground work, while deferred compaction debt execution is enabled only for background maintenance and only turn maintenance is automatically deferred.

## What This Changes

The issue asks OpenClaw to prevent context-engine deferred compaction debt from remaining pending after bootstrap/reconcile imports or hot-cache-triggered maintenance.

## Best Possible Solution

Add a generic host-level background maintenance follow-up for bootstrap/reconcile debt, and keep hot-cache policy plus row-level import idempotency in the context-engine/plugin owner surface unless a small generic SDK seam is needed.

## Reproduction Assessment

Yes for the host scheduling gap: source inspection shows bootstrap maintenance runs in foreground while only turn maintenance schedules background execution with allowDeferredCompactionExecution. The hot-cache and duplicate-import symptoms have credible live evidence but no minimized current-main core repro.

## Solution Assessment

Yes, a narrow core scheduling fix is the best path for the host half because it preserves plugin-owned compaction policy and avoids embedding lossless-claw-specific rules in OpenClaw core.

## Repair Work Prompt

Fix openclaw/openclaw#67716 for the host-side bootstrap/reconcile scheduling gap. Current main calls bootstrap maintenance with reason "bootstrap" and no executionMode, runContextEngineMaintenance only defers reason "turn" for engines with turnMaintenanceMode "background", and allowDeferredCompactionExecution is only true in background execution. Add a generic background follow-up path for bootstrap/reconcile maintenance that can consume deferred compaction debt after the session lane is idle. Do not add lossless-claw-specific policy, plugin ids, new config, startup/load-time migrations, or changes to external plugin hot-cache policy. Add focused regression coverage in context-engine maintenance plus the PI/Codex harness lifecycle surfaces. If the implementation appears to require lossless-claw transcript import idempotency or hot-cache policy changes, stop and report that as separate owner/plugin work. Add a CHANGELOG.md fix entry because this is user-visible runtime behavior.
