## Summary

Keep open. Current main still has a source-proven OpenClaw scheduling gap: bootstrap maintenance reaches the foreground path, deferred compaction debt execution is background-only, and automatic background scheduling remains gated to turn maintenance; related comments add live Lossless Claw corroboration and a maintainer request for OpenClaw plus plugin follow-up.

## What This Changes

The issue asks OpenClaw to prevent bootstrap/reconcile and hot-cache context-engine maintenance from leaving deferred compaction debt sticky or unpaid.

## Best Possible Solution

Add a generic OpenClaw bootstrap/reconcile background follow-up that can consume deferred debt for engines already opted into background maintenance, while leaving Lossless Claw cache and deduplication policy in the plugin unless a concrete host API gap is proven.

## Reproduction Assessment

Yes at source level: bootstrap maintenance is invoked as foreground, deferred debt execution is only enabled for background runs, and automatic background scheduling is currently turn-only. I did not run a live LCM session in this read-only review.

## Solution Assessment

Yes for the narrow OpenClaw half: extend the existing deferred-maintenance lifecycle so bootstrap/reconcile can schedule a valid background debt consumer. The hot-cache and duplicate-ledger portions should remain Lossless Claw work unless they expose a missing generic OpenClaw API.
