## What This Changes

The issue asks OpenClaw to prevent bootstrap/reconcile and hot-cache deferred compaction debt from remaining pending without a valid follow-up compaction path.

## Best Possible Solution

Add a generic OpenClaw background follow-up for bootstrap/reconcile maintenance that can consume deferred debt, while leaving Lossless Claw hot-cache and row-idempotent import policy in the plugin unless a missing core seam is proven.

## Solution Assessment

Yes for the OpenClaw host half: a narrow scheduler/lifecycle fix is the maintainable path. The issue should not be solved by embedding Lossless Claw hot-cache or deduplication policy into core.
