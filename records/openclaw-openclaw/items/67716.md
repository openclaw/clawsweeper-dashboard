## Summary

Keep open. Current main and v2026.5.2 still match the reported host-side gap: bootstrap maintenance runs in foreground mode, while the only automatic background follow-up that enables deferred debt execution is turn-triggered maintenance; the later macOS/lossless-claw comment adds independent operator evidence of sticky deferred maintenance debt.

## What This Changes

The issue reports that context-engine bootstrap/reconcile and hot-cache maintenance can record deferred compaction debt without a later eligible background run to consume it.

## Best Possible Solution

Add a generic host-level background follow-up for bootstrap/reconcile maintenance that is allowed to consume deferred debt, and keep lossless-claw-specific hot-cache policy in the plugin unless a small SDK signal is needed.

## Reproduction Assessment

Yes for the host scheduling gap: source inspection shows bootstrap maintenance can run only in foreground while deferred debt execution is enabled only in background runs. The hot-cache half has credible live traces but no minimized synthetic repro in core.

## Solution Assessment

Yes for the host half: a narrow context-engine maintenance scheduling fix is the maintainable path. The hot-cache policy half should not be solved by embedding lossless-claw-specific rules in OpenClaw core.

## Repair Work Prompt

Fix openclaw/openclaw#67716's OpenClaw host scheduling gap. Current main runs bootstrapHarnessContextEngine maintenance with reason "bootstrap" and no executionMode; runContextEngineMaintenance defaults foreground and only queues the background worker for reason "turn"; allowDeferredCompactionExecution is only set for executionMode "background". Add a generic host-level follow-up so background-capable context engines can consume deferred compaction debt after bootstrap/reconcile imports messages. Do not add lossless-claw-specific strings, thresholds, or hot-cache policy to OpenClaw core; if the hot-cache half needs a new generic SDK signal, stop and document that as follow-up. Add regression coverage with fake context engines for bootstrap foreground maintenance followed by eligible background debt execution, keep existing turn-maintenance behavior intact, and add a changelog entry because runtime behavior changes.
