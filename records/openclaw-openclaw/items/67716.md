## Summary

Keep open. Current main still has the OpenClaw host-side scheduling gap: bootstrap maintenance runs through the foreground default, deferred debt execution is only enabled for background maintenance, and automatic background scheduling remains gated to turn maintenance; later issue comments add live macOS and Telegram-backed Lossless-Claw evidence plus a request for OpenClaw follow-up.

## What This Changes

The issue asks OpenClaw to prevent bootstrap/reconcile and hot-cache maintenance paths from leaving context-engine deferred compaction debt pending indefinitely.

## Best Possible Solution

Add a generic OpenClaw bootstrap/reconcile background follow-up that can consume deferred debt for engines already opted into background maintenance, while leaving Lossless-Claw cache and deduplication policy in the plugin unless a missing host API is proven.

## Reproduction Assessment

Yes at source level: bootstrap maintenance is invoked without background execution, deferred debt execution is background-only, and automatic background scheduling is currently turn-only. I did not run a live LCM session in this read-only review.

## Solution Assessment

Yes for the narrow OpenClaw half: extend the existing deferred-maintenance lifecycle so bootstrap/reconcile can schedule a valid background debt consumer. The hot-cache and duplicate-ledger portions should remain Lossless-Claw work unless they expose a concrete generic OpenClaw API gap.
