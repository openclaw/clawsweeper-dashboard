## Summary

Keep open. Current main and v2026.5.7 still show the host-side scheduling gap: bootstrap maintenance runs as foreground work, while the only automatic background follow-up that enables deferred compaction debt execution is turn-triggered; later comments add independent macOS and Telegram/lossless-claw evidence rather than proving this is obsolete.

## What This Changes

The issue asks OpenClaw to prevent context-engine bootstrap/reconcile and hot-cache maintenance paths from leaving deferred compaction debt stranded.

## Best Possible Solution

Add a generic OpenClaw background follow-up for bootstrap/reconcile maintenance that can consume deferred debt, while keeping Lossless Claw hot-cache and row-idempotent import policy in the plugin unless a missing core seam is proven.

## Reproduction Assessment

Yes at source level for the host scheduling gap: bootstrap maintenance is foreground by default, the background scheduler only runs for turn maintenance, and deferred debt execution is enabled only in background mode. The hot-cache and duplicate-ledger halves have credible live logs but no minimized current-main repro in this review.

## Solution Assessment

Yes for the OpenClaw host half: a narrow scheduler/lifecycle fix is the maintainable path. The whole issue should not be solved by embedding Lossless Claw hot-cache or deduplication policy into core.
