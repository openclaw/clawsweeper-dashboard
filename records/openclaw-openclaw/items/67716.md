## Summary

Keep open. Current main still has the host-side scheduling gap: bootstrap maintenance runs as foreground maintenance, deferred debt execution is only enabled for background maintenance, and automatic background scheduling remains limited to turn maintenance; the later live comments add credible macOS and Telegram-backed Lossless-Claw evidence rather than making this stale.

## What This Changes

The issue reports context-engine deferred compaction debt being created during bootstrap/reconcile or hot-cache maintenance and then remaining unpaid.

## Best Possible Solution

Add a generic OpenClaw bootstrap/reconcile background follow-up that can consume deferred debt for engines already opted into background maintenance, while leaving Lossless-Claw cache and deduplication policy in the plugin unless a concrete host API gap is identified.

## Reproduction Assessment

Yes at source level: bootstrap maintenance is invoked without background execution, deferred debt execution is background-only, and automatic background scheduling is turn-only. I did not run a live LCM session in this read-only review.

## Solution Assessment

Yes for the narrow OpenClaw half: extending the existing deferred-maintenance lifecycle so bootstrap/reconcile can schedule a valid background debt consumer is the smallest maintainable host fix. Plugin-internal hot-cache and duplicate-ledger policy should remain separate unless the fix proves a missing generic OpenClaw API.
