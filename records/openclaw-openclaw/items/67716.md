## Summary

Keep open. Current main still has a source-proven OpenClaw host scheduling gap: bootstrap maintenance runs in foreground, deferred compaction execution is background-only, and automatic background scheduling is still turn-only; recent comments add live corroboration and a maintainer request for OpenClaw plus Lossless Claw follow-up.

## What This Changes

The issue asks OpenClaw to prevent bootstrap/reconcile and hot-cache maintenance paths from leaving context-engine deferred compaction debt pending indefinitely.

## Best Possible Solution

Add a generic OpenClaw bootstrap/reconcile background follow-up that can consume deferred debt for engines already opted into background maintenance, while keeping Lossless Claw cache and deduplication policy in the plugin unless a narrow host API gap is demonstrated.

## Reproduction Assessment

Yes, at source level: bootstrapHarnessContextEngine calls maintenance as reason "bootstrap" with no executionMode, runContextEngineMaintenance defaults to foreground, and deferred debt execution is only enabled for background runs. I did not run a live LCM session in this read-only review.

## Solution Assessment

Yes for the narrow OpenClaw half: extend the existing deferred-maintenance lifecycle so bootstrap/reconcile can schedule a valid background debt consumer. The hot-cache and duplicate-ledger portions should stay in Lossless Claw or a narrowly proven SDK seam, not hard-coded in OpenClaw core.
