## Summary

Keep open. Current main still has the source-level host scheduling gap: bootstrap maintenance runs through the foreground path, while the only automatic background follow-up that enables deferred compaction debt execution is still gated to turn maintenance.

## What This Changes

The issue requests that context-engine bootstrap/reconcile and hot-cache maintenance not leave deferred compaction debt pending indefinitely.

## Best Possible Solution

Add a generic OpenClaw background follow-up for bootstrap/reconcile maintenance that can consume deferred debt, while keeping Lossless Claw hot-cache and row-idempotent import policy in the plugin unless a missing core seam is proven.

## Reproduction Assessment

Yes at source level for the host scheduling gap: bootstrap maintenance defaults to foreground, background scheduling is turn-only, and debt execution is enabled only in background mode. The hot-cache and duplicate-ledger portions have credible live logs but no minimized current-main reproduction in this review.

## Solution Assessment

Yes for the OpenClaw host half: a narrow scheduler/lifecycle fix is the maintainable path. The whole issue should not be solved by embedding Lossless Claw hot-cache or deduplication policy into core.
