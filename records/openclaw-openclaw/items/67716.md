## Summary

Keep open. Current main and v2026.5.7 still show the OpenClaw host scheduling gap: bootstrap maintenance runs foreground by default, deferred execution is only enabled for background runs, and the automatic background scheduler remains turn-only; newer comments add corroborating macOS and Telegram/lossless-claw evidence plus a request to pursue OpenClaw and Lossless Claw fixes.

## What This Changes

The issue reports that context-engine bootstrap/reconcile and hot-cache maintenance can record deferred compaction debt that remains unpaid because debt execution is only allowed in background maintenance.

## Best Possible Solution

Add a generic OpenClaw background follow-up for bootstrap/reconcile maintenance that can consume deferred debt, while keeping Lossless Claw hot-cache and row-idempotent import policy in the plugin unless a missing core seam is proven.

## Reproduction Assessment

Yes at source level for the OpenClaw host scheduling gap: bootstrap maintenance is foreground by default, the automatic background scheduler is turn-only, and deferred debt execution is enabled only for background runs. The hot-cache and duplicate-ledger portions have credible live evidence but no minimized current-main repro in this review.

## Solution Assessment

Yes for the narrow OpenClaw host half: a scheduler/lifecycle fix is the maintainable path. The broader Lossless Claw hot-cache and deduplication policy should not be embedded into OpenClaw core without a proven missing API seam.
