## Summary

Keep open. Current main still has a source-proven OpenClaw host scheduling gap: bootstrap maintenance runs through the foreground path, deferred compaction execution is background-only, and automatic background scheduling is still limited to turn maintenance; recent live comments and a maintainer follow-up request make this active, not cleanup-closeable.

## What This Changes

Ensure bootstrap/reconcile and hot-cache deferred compaction debt cannot remain stranded after context-engine maintenance records it.

## Best Possible Solution

Add a generic OpenClaw bootstrap/reconcile background follow-up that can consume deferred debt for engines already opted into background maintenance, while leaving Lossless Claw cache and ledger policy in the plugin unless a concrete host API gap is proven.

## Reproduction Assessment

Yes, at source level: bootstrapHarnessContextEngine calls maintenance as reason "bootstrap" with no executionMode, runContextEngineMaintenance defaults to foreground, and deferred debt execution is only enabled for background runs. I did not run a live LCM session in this read-only review.

## Solution Assessment

Yes for the narrow OpenClaw half: extend the existing deferred-maintenance lifecycle so bootstrap/reconcile can schedule a valid background debt consumer. The hot-cache and duplicate-ledger portions should stay in Lossless Claw or behind a narrowly proven SDK seam, not hard-coded in OpenClaw core.
