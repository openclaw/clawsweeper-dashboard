## Summary

Keep open: current main still has the source-level OpenClaw scheduling gap where bootstrap maintenance runs in foreground while deferred compaction debt execution is background-only and automatic background scheduling is turn-only; live macOS and Telegram-backed comments add credible session-state evidence, and no canonical merged fix owns this exact bootstrap/reconcile debt path.

## What This Changes

The issue asks OpenClaw to prevent context-engine bootstrap/reconcile and hot-cache maintenance paths from leaving deferred compaction debt pending indefinitely.

## Best Possible Solution

Add a generic OpenClaw bootstrap/reconcile background follow-up that can consume deferred debt for engines already opted into background maintenance, while keeping Lossless-Claw cache and deduplication policy in the plugin unless a missing host API is proven.

## Reproduction Assessment

Yes at source level: bootstrap maintenance runs without background execution, debt execution is background-only, and automatic background scheduling is currently turn-only. I did not establish a live minimized LCM reproduction in this read-only review.

## Solution Assessment

Yes for the narrow OpenClaw half: extend the existing deferred-maintenance lifecycle so bootstrap/reconcile can schedule a valid background debt consumer. Plugin-specific hot-cache and duplicate-ledger policy should remain outside the OpenClaw fix unless it exposes a concrete host contract gap.
