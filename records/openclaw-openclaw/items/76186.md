## Summary

Current main and the shipped v2026.5.2 release now cover the central performance request with key-based stable system-prompt reuse and cached prompt-report tool schema stats; the discussion also includes independent confirmations that the severe prep delay improved after upgrading.

## What This Changes

Request to cache repeated embedded-run prep work within a session so system-prompt, tool, and stream setup do not add roughly 48 seconds before every model turn.

## Best Possible Solution

Keep the shipped key-based cache for stable prompt/report artifacts, and only add narrower caches later if profiling proves a dynamic run-specific stage can be safely keyed.

## Reproduction Assessment

Yes for the original structural behavior: the old trace path measured repeated embedded-run prep stages, and current source shows the targeted stable-cache fix. I did not reproduce a failing current-main run, and the latest discussion says the severe delay no longer reproduces after upgrading.

## Solution Assessment

Yes. The shipped fix addresses the expensive stable prompt/report work while keeping channel, auth, sandbox, and tool-policy setup dynamic, which is a safer boundary than blanket session caching.
