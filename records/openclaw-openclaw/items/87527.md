## Summary

Keep open: current main still defaults omitted child stdin to `inherit`, while this PR makes a plausible narrow fix for service-managed child spawns. The patch looks correct, but the service-mode default is compatibility-sensitive and should get focused regression coverage before maintainers treat it as fully ready.

The PR changes `createChildAdapter` so service-managed runtimes default omitted stdin to a closed pipe and reuse a local service marker for detach handling.

## Best Possible Solution

Land the adapter-level service-mode default with a focused regression test that protects both service-managed no-input EOF and explicit `stdinMode` overrides.
