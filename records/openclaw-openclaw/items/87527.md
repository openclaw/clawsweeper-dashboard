## What This Changes

The PR changes `createChildAdapter` so service-managed runtimes default omitted `stdinMode` to `pipe-closed` while reusing one service-managed runtime check for detached spawning.

## Best Possible Solution

Land this default change only after focused adapter coverage and redacted launchd/Node proof for the omitted-stdin child-spawn path, or fold the same behavior into the canonical hardening tracked by https://github.com/openclaw/openclaw/issues/77750.

## Solution Assessment

Likely yes as a narrow adapter mitigation, but not merge-ready yet. The adapter owns the omitted-stdin default, while https://github.com/openclaw/openclaw/issues/77750 still owns broader fallback and sibling-spawn hardening decisions.
