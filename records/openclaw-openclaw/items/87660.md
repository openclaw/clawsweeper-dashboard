## Summary

Keep open. Current main and v2026.6.6 still implement deterministic short-term Dreaming promotion into `MEMORY.md`, while this issue asks for a broader lifecycle-aware curator with locked/stable/evolving/stale policy, approval safeguards, configurable evidence budget, and reviewable diffs. The merged mitigation at https://github.com/openclaw/openclaw/pull/87654 compacted promoted snippets, and the later duplicate at https://github.com/openclaw/openclaw/issues/92797 was closed back to this issue rather than replacing it.

## What This Changes

Request to add a lifecycle-aware, likely LLM-assisted long-term memory curator for `MEMORY.md` that classifies entries, preserves provenance, respects locks, and emits reviewable diffs.

## Best Possible Solution

Design a memory-core curator or memory-plugin seam that classifies root-memory entries with schema-validated LLM output, respects locked anchors, uses a configurable evidence budget, and emits a reviewable patch before any write.
