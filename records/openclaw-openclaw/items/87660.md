## Summary

Keep open. Current main and the latest release still provide deterministic Dreaming promotion, snippet caps, visible provenance metadata, report-only review signals, and size-based compaction, but not lifecycle-aware LLM curation of existing MEMORY.md entries.

## What This Changes

Add an LLM-assisted long-term memory curator that classifies and updates existing MEMORY.md entries by lifecycle policy with provenance, a configurable evidence budget, reviewable diffs, locked-anchor approval, and rollback-safe writes.

## Best Possible Solution

Design this as a memory-core lifecycle curator or approved memory plugin/API seam with structured LLM validation, bounded evidence input, visible provenance, locked-anchor approval, rollback-safe writes, and a human-reviewable diff before durable memory changes.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/design request. Source, docs, and the latest release show deterministic promotion and compaction behavior while lifecycle-aware curation is absent.

## Solution Assessment

Unclear until maintainers choose the product boundary. The requested direction is plausible, but the maintainable solution needs explicit policy for locked sections, deletion or archival approval, evidence budgets, schema validation, provenance, and rollback behavior.
