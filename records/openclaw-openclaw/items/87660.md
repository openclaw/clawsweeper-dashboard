## Summary

Keep open. Current main and the latest release still implement deterministic Dreaming promotion with snippet caps, visible provenance, and size-based compaction; the requested lifecycle-aware LLM curator for existing MEMORY.md entries remains unimplemented and needs product/security design.

## What This Changes

The issue requests a lifecycle-aware LLM curation stage for root MEMORY.md that classifies locked, stable, evolving, and stale entries, preserves provenance, supports a configurable evidence budget, and produces reviewable diffs.

## Best Possible Solution

Design this as a memory-core lifecycle curator or memory-plugin/API seam with structured LLM output validation, bounded evidence input, visible provenance, locked-anchor approval, rollback-safe writes, and a human-reviewable diff before durable memory changes.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/design request. Source, docs, tests, and the latest release show the current deterministic promotion and compaction behavior, while the lifecycle-aware curator capability is absent.

## Solution Assessment

Unclear until maintainers choose the product boundary. The requested direction is plausible, but the safest solution needs explicit policy for locked sections, deletion or archival approval, evidence budgets, schema validation, provenance, and rollback behavior.
