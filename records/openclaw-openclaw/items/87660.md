## What This Changes

The issue requests an explicit lifecycle-aware, likely LLM-assisted curator for root MEMORY.md entries with locked/stable/evolving/stale policies, provenance, reviewable diffs, and safeguards.

## Best Possible Solution

Design this as a memory-core lifecycle curator or plugin/API seam with structured LLM validation, bounded evidence input, visible provenance, locked-anchor approval, rollback-safe writes, and a human-reviewable diff before durable memory changes.
