## Summary

## What This Changes

The issue requests lifecycle-aware, likely LLM-assisted curation of root MEMORY.md entries with locked/stable/evolving/stale policies, provenance, bounded evidence input, and reviewable diffs.

## Best Possible Solution

Design this as a memory-core lifecycle curator or approved plugin/API seam with structured LLM validation, bounded evidence input, visible provenance, locked-anchor approval, rollback-safe writes, and a human-reviewable diff before durable memory changes.

Not applicable as a bug reproduction: this is a feature/design request. Current source, docs, and v2026.6.9 show deterministic promotion and compaction behavior while the lifecycle-aware curator capability is absent.
