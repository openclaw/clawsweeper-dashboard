  - body: This import adds `readTailscaleStatusJson`, but the patched `src/infra/tailscale.ts` does not export that helper. As-is, the branch cannot typecheck or run once this file is compiled, so either implement/export the helper or remove the daemon probe.
  - confidence: 0.96
- **[P2] Allow official control URLs through runtime validation:** `src/gateway/server-runtime-config.ts:155`
  - body: Config validation now accepts `https://controlplane.tailscale.com` with `serve`/`funnel`, but runtime still throws for any non-empty `controlUrl`. That makes an explicitly configured official control plane fail at startup even though it passed config validation and remains supported Tailscale infrastructure.
  - confidence: 0.92
- **[P2] Use a supported source for the daemon control URL:** `src/gateway/server-runtime-config.ts:42-45`
  - body: `readDaemonControlServerUrl()` looks for `Self.ControlURL`, `CurrentTailnet.ControlURL`, `Prefs.ControlURL`, and top-level `ControlURL` in `tailscale status --json`, but upstream `ipnstate.Status` does not expose those fields. Real Headscale daemons can therefore skip the intended fast-fail and still hit the old late serve/funnel failure path.
  - confidence: 0.86
- **[P2] Redact daemon control URLs in startup errors:** `src/gateway/server-runtime-config.ts:167`
  - body: The new startup error interpolates `daemonControlUrl` verbatim. If the daemon login-server URL includes userinfo or query parameters, `runGatewayCommand()` will print those details into ordinary terminal/log output, so sanitize the URL before including it.
  - confidence: 0.9
