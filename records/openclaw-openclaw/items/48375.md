## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use the official-control predicate in runtime validation:** `src/gateway/server-runtime-config.ts:155-159`
  - body: Config validation accepts official Tailscale control URLs with `serve`/`funnel`, but this runtime guard rejects any non-empty `controlUrl`. A config that pins `https://controlplane.tailscale.com` can pass validation and then fail startup, so the runtime path should use the same official/custom predicate.
  - confidence: 0.9
- **[P2] Read daemon control state from a supported field:** `src/gateway/server-runtime-config.ts:41-45`
  - body: This helper reads `ControlURL` from `tailscale status --json`, but upstream status JSON is `ipnstate.Status` and does not expose those `ControlURL` paths. Existing Headscale daemons without `gateway.tailscale.controlUrl` will miss the intended fast-fail and fall through to the old Serve/Funnel failure path.
  - confidence: 0.89
- **[P2] Verify controlUrl in the advertised tailnet mode:** `src/gateway/server-runtime-config.ts:161-164`
  - body: When `controlUrl` is set, this branch skips daemon inspection and never compares the local daemon's actual login server. A typo or stale URL is accepted while the gateway binds to whichever tailnet the daemon already uses; either enforce it here or make the docs/field explicitly record-only.
  - confidence: 0.82
- **[P2] Redact daemon control URLs before reporting errors:** `src/gateway/server-runtime-config.ts:167`
  - body: The startup error interpolates `daemonControlUrl` verbatim and the CLI prints startup exceptions directly. Strip userinfo and query data before including a daemon login-server URL in terminal or gateway logs.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff adds Gateway networking configuration and still has concrete URL-disclosure and wrong-tailnet exposure risks.

Concerns:

- **[medium] Daemon control URL can leak credentials:** `src/gateway/server-runtime-config.ts:167`
  - body: `daemonControlUrl` is included verbatim in a startup error that the CLI prints; userinfo or query tokens in a login-server URL would be written to ordinary logs.
  - confidence: 0.82
- **[medium] Configured control URL is not enforced:** `src/gateway/server-runtime-config.ts:161`
  - body: The documented `mode=off` plus `bind=tailnet` path accepts `controlUrl` without checking the local daemon's actual control server, so a stale value can leave the gateway exposed on an unintended tailnet while appearing configured for another.
  - confidence: 0.78
