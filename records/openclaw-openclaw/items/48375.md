## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Enforce or downgrade the direct-tailnet controlUrl contract:** `src/gateway/server-runtime-config.ts:161-164`
  - body: The supported Headscale path is `mode=off` with `bind=tailnet`, but this branch only probes the daemon when `controlUrl` is absent and mode is serve/funnel. A configured `controlUrl` is therefore stored without applying or verifying the daemon's actual login server, so a typo or stale daemon login can expose the Gateway on the wrong tailnet while the config suggests otherwise. Either verify the daemon target for this path or make the field explicitly record-only in code, docs, and tests.
  - confidence: 0.91
- **[P2] Use a real daemon source for the login server:** `src/gateway/server-runtime-config.ts:41-46`
  - body: `readDaemonControlServerUrl()` reads `ControlURL` from several `tailscale status --json` paths, but the upstream Tailscale `ipnstate.Status`/`TailnetStatus` shape inspected does not expose those fields. That means real Headscale daemons can still pass this fast-fail and hit the older opaque serve/funnel failure path; use a documented command/API or remove this runtime claim.
  - confidence: 0.86
- **[P2] Honor official control URLs at runtime:** `src/gateway/server-runtime-config.ts:155-159`
  - body: Config validation accepts `https://controlplane.tailscale.com` with serve/funnel, but runtime throws as soon as any `controlUrl` is present. Users who explicitly pin the official control plane through config or `--tailscale-control-url` would pass validation and then fail startup.
  - confidence: 0.93
- **[P2] Redact daemon control URLs before throwing:** `src/gateway/server-runtime-config.ts:165-168`
  - body: The daemon URL is interpolated directly into the startup exception, and the CLI prints startup exceptions to terminal/service logs. Strip userinfo, query, and fragment before including this value so a login-server URL cannot leak credentials or tokens in ordinary Gateway logs.
  - confidence: 0.88
