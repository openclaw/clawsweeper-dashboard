## Summary

Keep open. Current main still lacks custom Tailscale control-server support, and this PR is the active implementation for the already-closed superseded feature issue, but the current patch has blocking Gateway runtime/security findings before merge.

## What This Changes

This PR adds `gateway.tailscale.controlUrl`, `--tailscale-control-url`, config/schema UI metadata, validation/runtime tests, runtime daemon checks, and Headscale-oriented Tailscale Gateway documentation.

## Best Possible Solution

Finish this as a rebased Gateway feature with one shared official-vs-custom predicate, sanitized URL output, a clear enforced or documented tailnet/off contract, focused tests, and the required changelog entry.

## Reproduction Assessment

Yes. Source inspection of PR head gives high-confidence reproduction paths for the review findings: official `controlUrl` plus serve/funnel is rejected at runtime, daemon URLs are embedded in thrown startup errors, and mode off never checks the configured control URL. I did not run live Headscale E2E because this review was read-only.

## Solution Assessment

No. The feature direction is plausible, but this patch is not the best implementation until runtime/config validation agree, URLs are redacted before errors, and the supported tailnet/off path is either enforced or explicitly narrowed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Allow official control URLs in serve/funnel:** `src/gateway/server-runtime-config.ts:155-159`
  - body: Config validation now permits official Tailscale control URLs, but runtime still throws for any non-empty `controlUrl` before checking the host. A deployment that explicitly pins `https://controlplane.tailscale.com` will fail startup even though it is still using supported Tailscale infrastructure.
  - confidence: 0.91
- **[P2] Redact daemon control URLs in startup errors:** `src/gateway/server-runtime-config.ts:167`
  - body: `daemonControlUrl` comes from the local Tailscale daemon and may include userinfo or query tokens from `tailscale up --login-server`. Gateway startup prints this thrown error, so sanitize to an origin/host before including it in the message.
  - confidence: 0.9
- **[P2] Enforce controlUrl in the supported tailnet path:** `src/gateway/server-runtime-config.ts:161-164`
  - body: The documented Headscale path is `mode=off` with `bind=tailnet`, but the new daemon check only runs for serve/funnel and skips checks when `controlUrl` is set. A typo or stale daemon login is silently ignored, so the gateway can bind whichever tailnet the daemon is already joined to.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/cli/gateway-cli/run.ts:502-503`
  - body: This adds a user-facing config key and CLI flag, but the PR file list does not include `CHANGELOG.md`. Add a single-line entry under the active `### Changes` section with contributor credit before merge.
  - confidence: 0.96
