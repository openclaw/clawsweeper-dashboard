Full review comments:

- **[P1] Make the control URL enforce the tailnet target:** `src/gateway/server-runtime-config.ts:161-164`
  - body: In the advertised Headscale path, `controlUrl` is copied into the runtime config but this branch only reads the daemon URL when `controlUrl` is absent and mode is `serve`/`funnel`. With `gateway.bind="tailnet"` and `mode="off"`, a typo or stale daemon login is silently ignored and the Gateway still binds on whatever tailnet the local daemon already joined, so the new option is not a reliable safety control.
  - confidence: 0.88
- **[P2] Allow official control URLs at runtime:** `src/gateway/server-runtime-config.ts:155-159`
  - body: Config validation and tests accept `https://controlplane.tailscale.com` with `serve`/`funnel`, but runtime still throws for any non-empty `controlUrl`. Use the same `isOfficialTailscaleControlServer()` distinction here so explicitly spelling the official control plane does not fail at startup.
  - confidence: 0.93
- **[P2] Use a supported daemon control-server source:** `src/gateway/server-runtime-config.ts:41-45`
  - body: `readDaemonControlServerUrl()` probes `Self.ControlURL`, `CurrentTailnet.ControlURL`, `Prefs.ControlURL`, and `ControlURL` from `tailscale status --json`, but the upstream `ipnstate.Status` shape documents status fields like `AuthURL`, `Self`, and `CurrentTailnet` rather than those `ControlURL` paths. The new fast-fail therefore looks covered only because the test mocks fields the real CLI status payload is not documented to expose. ([pkg.go.dev](https://pkg.go.dev/tailscale.com/ipn/ipnstate?utm_source=openai))
  - confidence: 0.82
- **[P2] Redact daemon control URLs before throwing:** `src/gateway/server-runtime-config.ts:165-168`
  - body: This error embeds `daemonControlUrl` verbatim, and `runGatewayCommand()` prints startup exceptions directly. If a daemon or compatible implementation returns a URL with userinfo or query tokens, this leaks sensitive control-plane details into normal terminal or service logs; strip credentials and query data before formatting the message.
  - confidence: 0.86
