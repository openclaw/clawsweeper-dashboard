## Summary

Keep open: the PR has useful bug-fix intent, but the Telegram change does not reach the actual spooled-update path because the inbound debouncer swallows flush errors before they can make `bot.handleUpdate()` reject. The supplied proof and PR scope are also mismatched: the body proves only unit-tested OpenAI embedding behavior while the title and second commit change Telegram delivery.

## What This Changes

The branch changes OpenAI embedding model normalization for non-native base URLs and makes Telegram message processing rethrow dispatch errors after sending the fallback message.

## Best Possible Solution

Fix failure propagation at the Telegram handler/debounce/spool boundary, add a regression through `bot.handleUpdate()` and spooled drain, supply real Telegram proof, and split the OpenAI embedding change into its own reviewed PR.

## Reproduction Assessment

## Solution Assessment
