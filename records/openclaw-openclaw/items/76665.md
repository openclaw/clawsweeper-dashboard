## Summary

Keep open: current main contains the shipped z.ai-style Pi auto-compaction guard from #76056, but this report provides fresh 2026.5.2 session-backup evidence where the stored transcript itself loses prior visible turns. Source inspection does not show an intentional reset, maintenance, repair, or cache-TTL pruning path that should drop those turns within 78 seconds, so closing as implemented would be unsafe.

## What This Changes

The issue reports that consecutive Feishu DM turns using a z.ai/GLM provider rewrite the session JSONL so the first user/assistant exchange disappears before the second turn runs.

## Best Possible Solution

Add a deterministic regression around consecutive Feishu/Lark text-plus-DOCX turns on the same session with a z.ai-style provider, then fix the proven transcript rewrite boundary without widening pruning or reset behavior for unrelated providers.
