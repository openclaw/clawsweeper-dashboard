## Summary

Keep open: the report is recent, postdates the shipped z.ai consecutive-turn guard, and includes concrete backup excerpts showing prior visible turns missing from the session file after a Feishu DOCX/media follow-up. Current main contains the known z.ai guard and several paths that should not intentionally drop a 78-second-old user/assistant exchange, but the remaining trigger is not isolated enough for a safe speculative fix.

## What This Changes

The issue requests that Feishu sessions using `zai/glm-5.1` retain prior conversation history across consecutive turns, including when the second turn carries a DOCX/media attachment.

## Best Possible Solution

Add a deterministic regression for two consecutive Feishu DM turns where the second turn carries DOCX/media under a z.ai-style GLM route, then fix the proven transcript rewrite, routing, repair, or compaction boundary only.
