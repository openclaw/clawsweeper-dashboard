## Summary

Keep open: current main contains the z.ai-style Pi auto-compaction guard from #76056, but the report provides fresh 2026.5.2 session-backup evidence showing prior user/assistant turns disappearing within 78 seconds after that release. Source inspection does not show an intentional reset, pruning, or repair path that would remove those valid visible turns, and the provider metadata mismatch keeps the root cause unresolved.

## What This Changes

The issue asks OpenClaw to preserve conversation history across consecutive Feishu DM turns when using a z.ai/GLM gateway, especially a text turn followed by a DOCX/media message.

## Best Possible Solution

Add a deterministic regression for a consecutive Feishu DM text turn followed by a DOCX/media turn on a z.ai-style GLM provider, then fix the proven transcript rewrite, session-key, or compaction boundary without broadening unrelated pruning behavior.
