## Summary

Keep open: the report is recent, post-2026.5.2, and describes visible session transcript loss that current main should not intentionally perform. ## What This Changes

The issue reports that z.ai/GLM consecutive-turn session history is dropped between Feishu messages, especially when the second turn includes a DOCX/media attachment.

## Best Possible Solution

Add a deterministic regression for a Feishu DM two-turn z.ai/GLM conversation where the second turn carries DOCX/media, then fix the proven boundary in session routing, transcript persistence/repair, or compaction without broadening unrelated pruning behavior.
