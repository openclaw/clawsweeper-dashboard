## Summary

Keep open for maintainer review. Current main does not implement the requested validator stack, and the open draft PR still has blocking correctness/security findings, missing real behavior proof, dirty stacked dependencies, and recovery-policy questions.

## What This Changes

The PR adds a stacked loop-aware compaction guard pipeline ending in a pure post-compaction validator, plus config/schema/docs wiring and focused tests.

## Best Possible Solution

Rebase or consolidate the stack onto current pi-hooks and checkpoint surfaces, reuse Unicode-aware retention logic, separate trusted guard policy from bounded untrusted transcript data, align docs/help, and add real guarded-compaction proof before merge.

## Reproduction Assessment

Yes for the PR defects: source inspection of PR head and current main shows the old hook path, ASCII-only validator normalization, prompt-boundary issue, and docs/runtime contradiction. This is not a current-main bug reproduction because the PR is a new reliability feature.

## Solution Assessment

No, not as currently proposed. The direction is plausible, but the patch needs current-main rebase, Unicode retention fixes, prompt-boundary hardening, docs alignment, real behavior proof, and maintainer recovery-policy review before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Move guard logic onto pi-hooks:** `src/agents/pi-extensions/compaction-safeguard.ts:9`
  - body: Current main loads the active compaction safeguard from `src/agents/pi-hooks`, but this branch adds the guard imports and logic under the old `pi-extensions` path. Rebase the guard stack onto the active hook tree so the runtime path that actually runs compaction receives the change.
  - confidence: 0.9
- **[P2] Preserve Unicode goals in validation:** `src/agents/post-compaction-validator.ts:180-184`
  - body: `normalizeText` strips every non-ASCII letter and `isReflectedInSummary` treats an empty normalized source as already reflected. A latest goal written in Chinese, Japanese, or another non-Latin script can be omitted from the summary while validation still passes; reuse or match the existing Unicode-aware overlap machinery.
  - confidence: 0.91
- **[P2] Wrap failure signatures as untrusted prompt data:** `src/agents/pi-extensions/compaction-instructions.ts:96`
  - body: `failure.signature` is derived from transcript/tool error text and is inserted directly into compaction guidance. Tool errors can contain attacker-controlled text, so wrap or escape these signatures before placing them next to instructions.
  - confidence: 0.84
- **[P2] Keep guard policy out of custom instructions:** `src/agents/pi-extensions/compaction-safeguard.ts:872-893`
  - body: The guard text built here is fixed compaction policy, but it is passed as `customInstructions` into a path whose current contract wraps custom text as untrusted prompt data. Add a trusted guard-instruction seam and keep transcript signal details bounded as data.
  - confidence: 0.78
- **[P2] Align guard docs with runtime behavior:** `docs/gateway/configuration-reference.md:1036-1039`
  - body: The config reference says `guard` is reserved and has no runtime effect, while the stacked branch wires `guard.enabled` into safeguard compaction behavior. Users who set the flag would get the wrong contract if these docs ship unchanged.
  - confidence: 0.9
