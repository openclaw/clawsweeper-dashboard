## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve non-Latin goals in the validator:** `src/agents/post-compaction-validator.ts:183-184`
  - body: `normalizeText` drops every non-ASCII letter, and `isReflectedInSummary` treats an empty normalized source as already reflected. A latest goal like `请提供状态更新` will therefore pass even when the summary omits it, so a failed compaction can be marked `ok` and skip the reset recommendation. Please use the existing Unicode-aware tokenization/overlap approach from the current safeguard quality audit and add a non-Latin regression test.
  - confidence: 0.91
- **[P2] Align guard docs with the runtime hook:** `docs/gateway/configuration-reference.md:1036`
  - body: This docs text says `agents.defaults.compaction.guard` has no runtime effect, but the same branch wires `guard.enabled` into the safeguard runtime and uses it to augment compaction instructions for high-risk sessions. Operators reading the config reference or help text would be told the opposite of what the merged branch does, so update the docs/help/tests to describe the actual effect or split the config-only docs from the runtime PR.
  - confidence: 0.87
