## Summary

Keep open. Current main does not implement the proposed validator stack, and this draft stacked branch still has active correctness/security blockers plus missing real behavior proof; the checkpoint-backed recovery comment is useful design context rather than a close reason.

## What This Changes

The branch adds a stacked loop-aware compaction guard pipeline with config scaffolding, transcript-tail detection and scoring, safeguard instruction augmentation, a pure post-compaction validator, docs, and tests.

## Best Possible Solution

Rebase and narrow the guard stack onto current pi-hooks, reuse Unicode-aware retention helpers, separate trusted policy from untrusted transcript signals, align docs with runtime behavior, decide checkpoint-backed recovery policy, and add real guarded-compaction proof.

## Reproduction Assessment

Yes for the PR review findings: source inspection of PR head and current main shows the stale hook path, ASCII-only validator normalization, prompt-boundary issue, and docs/runtime contradiction. This is not a current-main bug reproduction because the PR is a new feature.

## Solution Assessment

No, not as currently proposed. The direction is plausible, but the branch needs a current-main rebase, Unicode retention fixes, prompt-boundary hardening, docs alignment, real behavior proof, and maintainer recovery-policy review before merge.
