## Summary

Keep open. Current main still does not implement this PR's intended post-compaction validator, and the adjacent summary quality audit/checkpoint work is not equivalent. The PR is also draft, stacked on still-open prerequisite compaction-guard PRs, stale against the current `pi-hooks` layout, and the proposed diff has concrete review blockers around generated config schema drift and untrusted guard-signal text in prompt instructions.

## What This Changes

This draft PR adds stacked loop-aware compaction guard work: config scaffolding, transcript-tail detection, guard scoring, guarded compaction instruction augmentation, and a pure post-compaction validator with focused tests.

## Best Possible Solution

Keep this PR open as part of the loop-aware compaction stack, but require a rebase onto current main, generated config schema/docs updates, untrusted-data wrapping for guard evidence inserted into prompts, and a maintainer decision on whether validation remains diagnostic-only or integrates with the existing checkpoint snapshot recovery path.

Not applicable as a user-facing bug reproduction; this is a feature PR. The current-main gap is reproducible with source searches and by inspecting the existing compaction quality, side-effect, and checkpoint paths, while the proposed validator behavior is represented by deterministic unit tests in the PR diff/body but was not executed in this read-only review.

## Solution Assessment

No, not in its current form. The validator direction is plausible, but the branch is draft/stacked/stale, misses generated public schema output, and should harden untrusted transcript-derived guard data before it becomes part of compaction instructions.
