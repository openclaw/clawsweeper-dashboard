## Summary

Keep open. This is draft stacked feature work, current main still lacks the proposed validator/guard stack, and the PR has merge-blocking current-main rebase, Unicode retention, prompt-boundary, docs, and real-behavior-proof gaps.

## What This Changes

The PR adds a stacked loop-aware compaction guard pipeline ending in a pure post-compaction validator, plus config/schema/docs and focused tests.

## Best Possible Solution

Rebase or consolidate the stack onto current pi-hooks, reuse Unicode-aware retention checks, separate trusted guard policy from bounded untrusted transcript data, align docs/help, and connect failed validation to checkpoint-backed recovery semantics before merge.

## Reproduction Assessment

Yes for the PR defects: source inspection of PR head and current main shows the old hook path, ASCII-only validator normalization, prompt-boundary issue, and docs/runtime contradiction. This is not a current-main bug reproduction because the PR is a new reliability feature.

## Solution Assessment

No, not as currently proposed. The direction is plausible, but the patch needs current-main rebase, Unicode retention fixes, prompt-boundary hardening, docs alignment, real behavior proof, and maintainer recovery-policy review before it is mergeable.
