## What This Changes

The PR branch adds loop-aware compaction guard config/docs, transcript-tail detection, guard scoring, guarded compaction instruction augmentation, a pure post-compaction validator, and focused tests.

## Best Possible Solution

Land or rework the lower loop-aware compaction stack in order, regenerate config schema artifacts, harden prompt-data handling, and make an explicit maintainer call on validator-driven restore versus diagnostic reset recommendation.

Not applicable as a user-facing bug reproduction; this is a feature PR. The current-main gap is source-verifiable because the validator/guard symbols are absent and the existing post-compaction hook only runs transcript update and memory-sync side effects.

## Solution Assessment

No, not in the current form. The validator direction is plausible, but the branch needs schema regeneration, prompt-data hardening, current-main stack/rebase handling, and a maintainer decision on recovery semantics before it is the narrowest maintainable solution.
