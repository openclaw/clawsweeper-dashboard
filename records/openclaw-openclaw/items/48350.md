The branch adds a stacked loop-aware compaction guard pipeline ending in a pure post-compaction validator, plus config/schema/docs wiring and focused tests.

## Best Possible Solution

Consolidate the guard stack onto current `pi-hooks` and checkpoint-backed recovery surfaces, reuse Unicode-aware retention logic, separate trusted guard policy from bounded untrusted signal data, align docs/help, and add real guarded-compaction proof before merge.

## Solution Assessment

No, not as currently proposed. The direction is plausible, but the patch needs a current-main rebase, Unicode retention fixes, prompt-boundary hardening, docs alignment, real behavior proof, and maintainer recovery-policy review before merge.
