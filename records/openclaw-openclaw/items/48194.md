## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Dispatch every additional target:** `src/cron/isolated-agent/run.ts:198`
  - body: `resolvedAdditionalTargets` is returned from delivery-context resolution, but the caller still dispatches only `prepared.resolvedDelivery`. Jobs that configure extra targets therefore deliver only to the primary route.
  - confidence: 0.95
- **[P2] Preserve additionalTargets during delivery patches:** `src/gateway/protocol/schema/cron.ts:191`
  - body: Adding `additionalTargets` to the shared delivery schema admits it in update payloads, but `mergeCronDelivery` never copies or applies the field. Delivery updates can drop an existing fanout list or ignore a requested replacement.
  - confidence: 0.91
- **[P2] Isolate optional target resolution failures:** `src/cron/isolated-agent/run.ts:181-185`
  - body: Resolving all secondary targets with `Promise.all` lets one bad optional target reject the whole cron delivery context before the primary route can run. Secondary target failures need per-target isolation.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `src/cron/types.ts:35-36`
  - body: This is a user-facing feature that adds a public cron delivery field, but the PR does not update `CHANGELOG.md`. OpenClaw policy requires a changelog entry before merge.
  - confidence: 0.86
