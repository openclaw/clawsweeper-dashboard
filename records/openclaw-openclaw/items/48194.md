## Summary

Keep open. Current main still has single-target cron delivery, while this PR is not complete enough to merge because it adds the additional-target contract and resolves secondary routes without dispatching them or preserving them through delivery updates. The overlapping #44859 is a broader open implementation candidate, so maintainer judgment is needed before choosing the canonical path.

## What This Changes

The PR adds a cron `delivery.additionalTargets` contract across types, protocol schema, normalization, and delivery-plan resolution, then resolves those targets during isolated cron setup.

## Best Possible Solution

Land one canonical end-to-end cron fanout implementation: define the contract in current `delivery-plan` and protocol surfaces, normalize and preserve target arrays through add/update, dispatch each secondary target after primary delivery with per-target failure isolation, record trace/status for every route, and update CLI/docs/tests for the visible user-facing behavior.

## Reproduction Assessment

Yes. A static reproduction path is enough: configure `delivery.additionalTargets` on the PR branch and trace the run path; the PR resolves those targets but the caller and dispatcher still use only the primary `resolvedDelivery`.

## Solution Assessment

No. The current PR shape is not the best solution because it adds schema/type plumbing without end-to-end delivery, update semantics, trace visibility, or tests; the safer approach is a current-main implementation that treats fanout as a complete delivery path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Dispatch every additional target:** `src/cron/isolated-agent/run.ts:198`
  - body: `resolveCronDeliveryContext` returns `resolvedAdditionalTargets`, but the caller still destructures only the primary delivery context and `dispatchCronDelivery` receives only `resolvedDelivery`. Jobs with `delivery.additionalTargets` will therefore deliver to the primary route only.
  - confidence: 0.95
- **[P2] Preserve additionalTargets during delivery patches:** `src/gateway/protocol/schema/cron.ts:191`
  - body: The schema now accepts `delivery.additionalTargets`, including patch inputs via the shared delivery properties, but `mergeCronDelivery` never copies or applies that field. Any delivery update can drop an existing fanout list or ignore a requested replacement.
  - confidence: 0.91
- **[P2] Isolate optional target resolution failures:** `src/cron/isolated-agent/run.ts:181-185`
  - body: The new `Promise.all` resolves all secondary targets before the run continues, so one invalid optional route rejects `resolveCronDeliveryContext` and can block the otherwise valid primary path. Secondary routes need per-target failure isolation.
  - confidence: 0.88
