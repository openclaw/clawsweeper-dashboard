## Summary

Keep open. Current main still emits `llm_output` with identifiers, assistant output, and usage counters only; the model context-window, effective context-budget, and max-output-token metadata exist elsewhere in the runtime but are not exposed to plugin hook consumers.

## What This Changes

Requested adding model context-window and max-output-token metadata to the `llm_output` plugin hook event so plugins can calculate context utilization.

## Best Possible Solution

Add optional, backwards-compatible native context-window, effective context-token budget, and model max-output-token metadata to `llm_output`, document it, and leave utilization calculation to plugins unless core defines an exact numerator.

## Reproduction Assessment

Yes. Source inspection of `PluginHookLlmOutputEvent` plus the Pi, CLI, and Codex emitters shows the requested fields are absent while model/context-budget metadata exists before emission.

## Solution Assessment

Yes. An additive plugin-hook contract update is the narrowest maintainable path because plugins already observe `llm_output`, and plugin-local model lookup tables would drift.
