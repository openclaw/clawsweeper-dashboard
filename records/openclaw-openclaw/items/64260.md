## Summary

Keep open. Current main still exposes identifiers, assistant output, and usage counters in `llm_output`, while model context-window, effective context-budget, and max-output-token metadata exist internally, so the requested additive plugin hook feature remains unimplemented.

## What This Changes

The issue requests adding context-window and model max-token metadata to the `llm_output` plugin event so plugins can calculate context utilization.

## Best Possible Solution

Add optional, backwards-compatible native and effective token-limit metadata to `llm_output`, document it, and leave utilization calculation to plugins unless core defines an exact numerator.

## Reproduction Assessment

Yes. A source-level current-main check of `PluginHookLlmOutputEvent` plus the Pi, CLI, and Codex app-server emitters shows the requested fields are absent while model/context-budget metadata exists before emission.

## Solution Assessment

Yes. An additive plugin-hook contract update is the narrowest maintainable path because plugins already observe `llm_output`, and plugin-local model lookup tables would drift.
