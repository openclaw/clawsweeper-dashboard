## Summary

Keep open. The prior ClawSweeper assessment still matches current `main`: `llm_output` carries identifiers, assistant output, and usage counters, but not native context-window, effective context-budget, or model max-output-token metadata even though those values exist nearby in resolved model/attempt state.

## What This Changes

Request to add context-window and model maximum-output-token metadata to `llm_output` hook events so plugins can calculate context utilization.

## Best Possible Solution

Add optional, backwards-compatible numeric fields for native context window, effective context-token budget, and model max output tokens to `llm_output`; leave utilization plugin-derived unless core defines an exact numerator.

## Reproduction Assessment

Yes. A static current-main check of `PluginHookLlmOutputEvent` plus the Pi, CLI, and Codex app-server emitters shows the requested fields are absent while model/context-budget metadata exists before emission.

## Solution Assessment

Yes. An additive hook-contract update is the narrowest maintainable path because plugins already observe `llm_output`, and requiring plugin-local model lookup tables would drift.
