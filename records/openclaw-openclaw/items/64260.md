## What This Changes

Expose context-window size or effective context-limit metadata in `llm_output` so plugin cost and context-health alerts can compute utilization.

## Best Possible Solution

Keep the additive hook contract now on main: expose resolved budget/source/reference metadata and let plugins compute utilization from existing usage counters.

## Solution Assessment

Yes. An additive plugin-hook contract fed by `ContextWindowInfo` is the narrow maintainable solution, and leaving the percentage calculation to plugins avoids defining a second derived field over existing usage counters.
