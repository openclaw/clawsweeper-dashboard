## Summary

Keep open. Current main still exposes `llm_output` as identifiers, assistant text, and usage counters only; the requested model context-window, effective context budget, and max-output-token metadata is available elsewhere but not threaded through the plugin hook contract. Related #64327 is useful context for naming the effective budget field, but it does not supersede this request.

## What This Changes

The issue asks for `llm_output` hook events to include model context-window and maximum-output-token metadata so plugins can calculate context utilization.

## Best Possible Solution

Add optional, backwards-compatible numeric fields to `llm_output` for native context window, effective context-token budget, and model max output tokens across Pi, CLI, and Codex emitters; leave utilization plugin-derived unless core defines an exact numerator.

## Reproduction Assessment

Yes. A static current-main check of `PluginHookLlmOutputEvent` and the Pi, CLI, and Codex app-server emitters reproduces the missing fields without a live LLM call.

## Solution Assessment

Yes. An additive hook-contract update is the narrowest maintainable solution because plugins already observe `llm_output` and core already resolves the relevant model metadata.
