## Summary

Keep open. Current main and the latest release still omit native context-window, effective context-token, max-output-token, and utilization metadata from `llm_output`, while the runtime already resolves those model limits before hook emission.

## What This Changes

The issue requests adding model context-window and max-token metadata, plus optional utilization, to the plugin `llm_output` event.

## Best Possible Solution

Add optional, backward-compatible native and effective token-limit metadata to `llm_output`, document it, and leave utilization calculation to plugins unless core defines an exact numerator.

## Reproduction Assessment

Yes for the current gap: static inspection of the hook type plus Pi, CLI, and Codex emitters shows the requested fields are absent while model-limit metadata exists before emission.

## Solution Assessment

Yes. An additive plugin-hook contract update is the narrowest maintainable solution because plugins already observe `llm_output`, and plugin-local model limit lookup tables would drift.
