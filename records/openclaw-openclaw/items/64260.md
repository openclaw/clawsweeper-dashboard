## Summary

Keep open. Current main still exposes only identifiers, assistant output, and usage counters in `llm_output` while model context-window, effective context-budget, and max-output-token metadata exists before emission, so this remains a narrow additive plugin-hook feature rather than a closeable implemented item.

## What This Changes

Request to add model context-window and max-output-token metadata, optionally utilization, to the `llm_output` plugin hook event.

## Best Possible Solution

Add optional, backwards-compatible native and effective token-limit metadata to `llm_output`, document it, and leave utilization calculation to plugins unless core defines an exact numerator.

## Reproduction Assessment

Yes. A static current-main check of `PluginHookLlmOutputEvent` plus the Pi, CLI, and Codex app-server emitters shows the requested fields are absent while model/context-budget metadata exists before emission.

## Solution Assessment

Yes. An additive plugin-hook contract update is the narrowest maintainable path because plugins already observe `llm_output`, and plugin-local model lookup tables would drift.
