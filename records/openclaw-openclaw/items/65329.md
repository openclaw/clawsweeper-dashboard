## Summary

Keep open. Current main still leaves the reported MS Teams personal-DM media gap: after direct download returns no media, both Bot Framework and Graph fallback paths are still gated on HTML `<attachment>` IDs, while the DM handler passes a Graph-style conversation ID into the resolver.

## What This Changes

The issue asks for MS Teams personal-DM inline images and file attachments to be downloaded and passed to the agent instead of leaving unresolved `msteams://media/...` placeholders.

## Best Possible Solution

Fix the bundled MS Teams plugin’s inbound personal-DM media resolver to preserve the original Bot Framework conversation ID, attempt a narrowly scoped DM fallback for real non-HTML media/file attachments after direct download fails, and keep mention-only HTML messages out of fallback logging and network calls.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `resolveMSTeamsInboundMedia` for a personal DM where `downloadMSTeamsAttachments()` returns `[]` and `extractMSTeamsHtmlAttachmentIds()` returns `[]`; current main then skips both Bot Framework and Graph fallback paths.

## Solution Assessment

No. Current main is not the best solution because the remaining fallback decision is tied to HTML `<attachment>` IDs and the caller can hide the original Bot Framework DM ID; the safer repair is a narrow MS Teams plugin change that preserves both IDs and only falls back for real media/file candidates.
