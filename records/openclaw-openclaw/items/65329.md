## Summary

Keep open. Current main still leaves the MS Teams personal-DM media gap: non-HTML DM media has no Bot Framework or Graph fallback after direct download returns no media, and the caller passes a Graph-style conversation ID into the resolver.

## What This Changes

The issue requests that MS Teams personal-DM inline images and file attachments be downloaded and delivered to the agent instead of leaving unresolved media placeholders.

## Best Possible Solution

Fix the bundled MS Teams plugin’s inbound personal-DM media resolver to preserve the original Bot Framework conversation id, add a narrowly scoped fallback for real non-HTML media/file candidates, and keep mention-only HTML messages out of fallback network calls.

## Reproduction Assessment

Yes, source-level. A high-confidence reproduction is a personal-DM call where `downloadMSTeamsAttachments()` returns `[]` and `extractMSTeamsHtmlAttachmentIds()` returns `[]`; current main then skips both fallback paths, and the real caller can pass a Graph-style id that prevents Bot Framework DM detection.

## Solution Assessment

Yes, a narrow MS Teams plugin repair is the right path. The implementation should not remove the HTML guard wholesale; it should preserve the original DM id and only fall back for real non-HTML media/file candidates so the earlier mention-only 404 fix stays intact.
