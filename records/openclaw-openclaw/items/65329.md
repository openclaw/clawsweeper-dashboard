## Summary

Keep open. Current main still leaves the MS Teams personal-DM non-HTML media path without a Bot Framework fallback after direct download returns no media, and related open work explicitly leaves clipboard-paste DM images as a separate follow-up.

## What This Changes

The issue asks OpenClaw to download inline images and file attachments sent to the MS Teams bundled plugin in personal DMs instead of leaving unresolved media placeholders.

## Best Possible Solution

Keep this open and land a narrow bundled MS Teams plugin fix that preserves the original Bot Framework DM id, falls back only for real non-HTML media/file candidates, and adds regression coverage without broadening mention-only fallback behavior.

## Reproduction Assessment

Yes, source-level. A personal-DM call where direct attachment download returns `[]` and `extractMSTeamsHtmlAttachmentIds()` returns `[]` still skips both Bot Framework and Graph fallback paths on current main; a live Teams tenant was not run in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is in the MS Teams plugin inbound media path: carry both the original Bot Framework conversation id and the Graph chat id, then attempt Bot Framework fallback for real non-HTML DM media without removing the mention-only HTML guard.
