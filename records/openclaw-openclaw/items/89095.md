## Label Justifications

- `P1`: The issue can leave yielded parent sessions without a terminal completion event and park real agent workflows indefinitely.
- `impact:session-state`: The child run can reach terminal state while the parent/requester session lacks the corresponding completion state.
- `impact:message-loss`: The subagent completion notification can be retried, failed, and dropped before reaching the parent-visible context.

## Summary

Keep open: current main and v2026.6.9 still have the asymmetric `waitForAgentJob` timeout fallback and the no-output subagent completion delivery path still fails with `completion agent did not produce a visible reply`. The latest comments broaden the report beyond hard timeouts to clean completions with no deliverable payload, so the open wait-layer PR is useful but not sufficient by itself.

## What This Changes

The issue asks OpenClaw to notify the parent/requester session when a subagent timeout or no-output terminal completion would otherwise be silently dropped.

## Best Possible Solution

Land or replace the wait-layer timeout snapshot fix, then pair it with a maintainer-approved no-deliverable-payload terminal notification path so yielded parents receive exactly one bounded completion/failure event.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.9 still have the pending-timeout wait fallback gap and the `(no output)` delivery failure branches. I did not run a fresh live timeout or clean-stop subagent reproduction in this read-only review.

## Solution Assessment

No, the currently open wait-layer PR is a necessary bounded slice but not the complete fix. The best solution also needs to handle terminal completions that produce no deliverable payload, including the clean-stop case raised in the latest comment.
