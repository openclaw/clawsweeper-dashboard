## Summary

Keep open: current main still has the reported asymmetric `waitForAgentJob` outer-timer fallback, and the issue is already paired with open fix PRs that have not merged. ## What This Changes

The issue requests that forced sub-agent run timeouts notify the parent session instead of leaving the parent waiting or polling for terminal state.

## Best Possible Solution

Land one canonical fix that preserves pending timeout snapshots and proves that an `expectsCompletionMessage` timeout with no visible child output produces a parent-visible terminal notification, then close this issue after merge.

## Reproduction Assessment

Yes. The source path is clear: create a pending timeout snapshot, let the outer `waitForAgentJob` timer fire before timeout grace publishes it, and current main resolves without that timeout snapshot; the reporter also supplied live no-output timeout evidence.

## Solution Assessment

Partly. The symmetric wait-layer fallback is necessary and narrow, but the final fix should also prove the no-output delivery path so a timeout does not become a failed invisible completion.
