- `impact:session-state`: The child run can reach terminal timeout state while the parent session lacks the corresponding terminal completion state.

## Summary

Keep open: current main still has the asymmetric `waitForAgentJob` outer timeout fallback, and the reporter-confirmed zero-output timeout path still maps to a failed visible-reply delivery instead of a parent-visible terminal notification. Open related PRs cover the wait-layer slice, but none has merged or fully owns the no-output delivery behavior.

## What This Changes

The issue asks for forced sub-agent run timeouts to notify the parent/requester session instead of silently dropping the completion event.

## Best Possible Solution

Land or replace one canonical fix that preserves pending timeout snapshots and ensures zero-output forced timeouts produce exactly one parent-visible terminal notification, with live proof for the parent/requester path.

## Reproduction Assessment

Yes, source-level: current main still has the asymmetric wait fallback and the failed `(no output)` completion delivery branch. I did not run a live parent/subagent timeout in this read-only review.

## Solution Assessment

No, current main is only partial. The wait-layer repair is a good bounded slice, but the best solution also needs to repair or explicitly split the zero-output timeout delivery path so the parent receives a terminal notification.
