- `P1`: The issue now has latest-release evidence that a yielded parent can park indefinitely when child completion delivery has no visible payload.
- `impact:session-state`: The child run can reach terminal state while the parent/requester session lacks the corresponding completion state.

## Summary

Keep open: current main and v2026.6.9 still have the pending-timeout wait fallback gap and the failed `(no output)` completion delivery path; the open PRs are partial candidates, not merged fixes, and the newest issue evidence broadens the remaining failure beyond timeout-only runs.

## What This Changes

The issue asks OpenClaw to notify the parent/requester session when a subagent reaches a terminal state but produces no deliverable completion payload, instead of silently dropping the completion event.

## Best Possible Solution

Land or replace one canonical wait-layer fix for hard pending timeout snapshots, then ensure terminal subagent completions with no deliverable payload produce one parent-visible failure/completion event; keep broader retry/expiry configurability separate.

## Reproduction Assessment

Yes. Source-level reproduction is clear: current main still ignores pending timeout snapshots in the outer wait fallback and still returns `visible_reply_missing` for failed `(no output)` subagent completions; I did not run a live timeout or clean-stop parent session in this read-only review.

## Solution Assessment

No, current main is not the best solution. The wait-layer PR is a good bounded slice, but the best fix also needs to cover the no-deliverable-payload terminal notification path now verified on v2026.6.9.
