## What This Changes

The PR clears auto-reply runtime `modelProvider`/`model` cache fields when `/new` or `/reset` creates a fresh channel session, adds persisted-store regression coverage, and updates the changelog.

## Best Possible Solution

Merge the focused reset-state fix after CI/review, keeping broader hot-reload-without-reset behavior as a separate product decision if maintainers want it.

## Reproduction Assessment

Yes, from source inspection: seed a persisted channel session row with plain `model`/`modelProvider` and no override fields, then process authorized `/new` or `/reset`; current main merges the fresh entry over the old row without clearing those runtime fields. I did not execute the regression test because this review is read-only.

## Solution Assessment

Yes. Clearing only the runtime cache fields during fresh auto-reply session initialization is the narrow maintainable fix, while the existing preserved-selection helper keeps explicit user overrides intact and JSON persistence drops the cleared fields.
