## Summary

Keep open for merge: current main still has the source-reproducible stale auto-reply runtime model cache path, and this PR cleanly targets that reset bug with regression coverage and a changelog entry. Related discussion in #77322 and the closed alternative PR #77326 supports the fix direction.

## What This Changes

The PR clears auto-reply `modelProvider` and `model` runtime cache fields during fresh `/new` or `/reset` session initialization, adds persisted-store regression coverage, and updates `CHANGELOG.md`.

## Best Possible Solution

Merge this focused reset-state fix after normal maintainer review, leaving any broader hot-reload-without-reset behavior as a separate product decision.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed a persisted channel session with plain `model`/`modelProvider` and no override fields, then process authorized `/new` or `/reset`; current main merges the fresh entry over the old row and keeps those runtime fields.

## Solution Assessment

Yes. Clearing only the runtime cache fields during fresh auto-reply session initialization is the narrow maintainable fix, while existing preserved-selection logic keeps explicit user overrides intact and JSON persistence removes the cleared fields.
