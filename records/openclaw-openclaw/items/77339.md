## What This Changes

The PR clears cached auto-reply runtime `modelProvider` and `model` fields during fresh `/new` or `/reset` session initialization, adds a persisted-store regression test, and updates the changelog.

## Best Possible Solution

Review and land this PR, or an equivalent reset-state fix, so fresh auto-reply channel sessions discard runtime cache fields while preserving explicit user overrides.

## Reproduction Assessment

Yes. From source inspection, seed a persisted channel session row with plain `model`/`modelProvider` and no override fields, then process authorized `/new` or `/reset`; current main merges the new entry over the old row without clearing those fields, and active model resolution reads `sessionEntry.model`.

## Solution Assessment

Yes. Clearing only the runtime cache fields during fresh session initialization is the narrow maintainable fix; the existing preserved-selection path continues to keep explicit user overrides, and JSON persistence removes the cleared fields.
