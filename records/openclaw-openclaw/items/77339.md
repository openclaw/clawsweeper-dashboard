## Best Possible Solution

Land the focused auto-reply reset fix after normal review and CI, preserving explicit user overrides while keeping broader hot-reload/session model policy changes separate.

## Solution Assessment

Yes. Clearing only the runtime cache fields in the existing fresh-session reset block is the narrow maintainable fix, while the preserved-selection helper keeps explicit user overrides and JSON persistence drops the undefined fields.
