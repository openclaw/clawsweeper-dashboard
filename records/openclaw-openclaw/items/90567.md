## Summary

Close: the reporter confirmed the PDF 401 was caused by using a Token Plan key under the pay-as-you-go Xiaomi provider, and current source/docs already define the supported `xiaomi-token-plan` provider path.

## What This Changes

The issue reports a 401 from the PDF tool when a Xiaomi Token Plan key was configured under the Xiaomi pay-as-you-go provider, then adds that switching to the Token Plan provider/env/model resolved it.

## Best Possible Solution

Keep the shipped `xiaomi` versus `xiaomi-token-plan` provider split and treat broader validation of hand-edited mismatched config as a separate ergonomics proposal if maintainers want it.

## Reproduction Assessment

No. There is no current-main failing path for the documented `xiaomi-token-plan` configuration; the reporter confirmed that path works, and live Xiaomi credentials were not available for an independent end-to-end PDF call.

## Solution Assessment

Yes. For the reported failure, selecting `xiaomi-token-plan` with `XIAOMI_TOKEN_PLAN_API_KEY` and a Token Plan model is the narrow supported solution; warning on arbitrary hand-written provider/key mismatches would be separate product polish.
