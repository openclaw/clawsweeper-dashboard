## Summary

The reported 401 was caused by using a Token Plan key with the pay-as-you-go provider id; the reporter confirms that the documented Token Plan provider, credential variable, and model work with the PDF tool in the same environment.

## What This Changes

Investigate a PDF-tool 401 occurring when a Xiaomi Token Plan endpoint and key were configured under the pay-as-you-go Xiaomi provider.

## Best Possible Solution

Keep the shipped provider split and direct Token Plan users to `xiaomi-token-plan` with its matching credential variable; consider broader validation of manually authored mismatched configuration only as a separate ergonomics proposal.

## Reproduction Assessment

No. The original 401 depends on configuring a Token Plan key under the pay-as-you-go provider, while the reporter confirms that the documented Token Plan provider and credential pairing succeeds for PDF analysis.

## Solution Assessment

Yes for the reported failure. Selecting `xiaomi-token-plan` with `XIAOMI_TOKEN_PLAN_API_KEY` and a Token Plan model is the canonical narrow solution; validating arbitrary hand-written mismatches would be separate product polish.
