## Summary

Close as a resolved configuration/support issue: the reporter confirmed PDF works after switching from the pay-as-you-go `xiaomi` provider to the documented `xiaomi-token-plan` provider, and current source/docs keep those auth namespaces separate.

## What This Changes

The issue reports a PDF-tool 401 when a Xiaomi Token Plan key is configured under the pay-as-you-go Xiaomi provider path.

## Best Possible Solution

Keep the shipped provider split and direct Token Plan users to `xiaomi-token-plan`; treat broader warnings for hand-written mismatched config as a separate ergonomics proposal if maintainers want it.

## Reproduction Assessment

No. The original 401 depends on a mismatched provider/key configuration, while the reporter confirmed that the documented Token Plan provider and credential pairing works for PDF analysis.

## Solution Assessment

Yes for the reported failure. Using `xiaomi-token-plan` with `XIAOMI_TOKEN_PLAN_API_KEY` is the narrow supported solution; warning on every manually authored mismatch would be separate product polish.
