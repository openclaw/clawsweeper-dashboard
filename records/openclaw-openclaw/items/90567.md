## Summary

Current main and the latest stable release already cover the reported Xiaomi Token Plan PDF path: the reporter confirmed the failure was a provider/env mismatch, and the shipped Xiaomi plugin now has first-class Token Plan provider setup plus key-prefix validation.

## What This Changes

The issue reports a PDF-tool 401 when a Token Plan Xiaomi key is configured under the pay-as-you-go Xiaomi provider.

## Best Possible Solution

Keep the shipped Xiaomi Token Plan provider path and direct Token Plan users to `xiaomi-token-plan` with its matching env var and PDF model ref.

## Reproduction Assessment

No. The original 401 depends on using a Token Plan key under the pay-as-you-go provider, while the reporter confirmed the documented Token Plan provider/env/model pairing works for PDF analysis.

## Solution Assessment

Yes for the reported failure. The narrow maintainable solution is the shipped `xiaomi-token-plan` provider with `XIAOMI_TOKEN_PLAN_API_KEY`; optional broader linting for hand-written mismatches can be handled separately if needed.
