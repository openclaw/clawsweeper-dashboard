## Summary

Keep this PR open for maintainer-directed revision. Current main still lacks the missing-session recovery path, but this branch has blocking correctness findings, a policy-sensitive session-start surface, missing real behavior proof, and unrelated cron changes.

## What This Changes

The PR adds structured session-miss errors, session auto-bootstrap, a new `sessions.start` gateway method and `sessions_start` agent tool, send delivery metadata/read receipts, generated Swift/tool-display updates, and cron scheduler changes.

## Best Possible Solution

Revise or replace this with a focused session recovery change that preserves existing visibility/A2A/sandbox gates, defines the lifecycle contract, updates protocol schemas/generated clients/tool catalogs/docs/tests together, and splits cron scheduling into a separate review path.

## Reproduction Assessment

Yes for the current source-level limitation: main still returns `session not found` before any bootstrap path and has no `sessions.start`/`sessions_start` surface. I did not live-reproduce the original Ubuntu/MiniMax upgrade scenario.

## Solution Assessment

No. The direction is plausible, but this PR is not the best shape because it misroutes missing named sessions, omits policy gates, has lifecycle/rollback bugs, and bundles unrelated cron scheduler changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use the repo TypeBox package:** `src/agents/tools/sessions-start-tool.ts:1`
  - body: The new tool imports `@sinclair/typebox`, but this repo declares and consistently imports `typebox`; loading the file can fail before `sessions_start` is usable.
  - confidence: 0.94
- **[P1] Do not redirect missing named sessions to main:** `src/gateway/server-methods/sessions.ts:408-410`
  - body: When any agent-scoped key is missing, this bootstraps the agent's main session and continues there. A missing key like `agent:ops:incident-123` would be delivered into `agent:ops:main`, merging unrelated conversations instead of returning not-found.
  - confidence: 0.91
- **[P1] Allocate a fresh id before bootstrapping dead sessions:** `src/gateway/sessions-bootstrap.ts:117-122`
  - body: `applySessionsPatchToStore()` preserves an existing store entry, so bootstrapping a dead main entry without `sessionId` can still leave no fresh id for transcript creation and fail recovery.
  - confidence: 0.89
- **[P2] Enforce visibility policy before sessions_start:** `src/agents/tools/sessions-start-tool.ts:32-47`
  - body: `sessions_start` forwards any `agentId` directly to the gateway and returns session metadata without the visibility, sandbox, and A2A checks used by `sessions_send`. Restricted agents can probe or start sessions outside their allowed surface.
  - confidence: 0.92
- **[P2] Classify sessions.start under write scope:** `src/gateway/server-methods-list.ts:72`
  - body: Adding `sessions.start` to the public method list without adding it to `method-scopes.ts` leaves it unclassified. Non-admin clients with normal write scopes will hit the default admin requirement.
  - confidence: 0.9
- **[P2] Register sessions_start in profile catalogs:** `src/agents/openclaw-tools.ts:266-268`
  - body: `createOpenClawTools` exposes the new tool, but profile filtering is driven by `CORE_TOOL_DEFINITIONS`. Without a catalog entry, coding or messaging profiles can filter the recovery tool out before execution.
  - confidence: 0.86
- **[P1] Keep the cron fallback near the next occurrence:** `src/cron/schedule.ts:126-128`
  - body: Changing the fallback anchor to `now + 48h` can skip the next daily occurrence whenever this recovery path is used. Existing regression coverage documents this 48-hour jump as the failure mode to avoid.
  - confidence: 0.87
- **[P1] Catch schedule failures in forced cron preflight:** `src/cron/service/ops.ts:542-544`
  - body: Calling `computeJobNextRunAtMs()` directly during force preflight can throw for malformed schedules before the manual run is enqueued. The maintenance path records schedule errors instead of aborting the operator action.
  - confidence: 0.84
- **[P2] Restore the prior session entry on rollback:** `src/gateway/sessions-bootstrap.ts:139-140`
  - body: When transcript creation fails, rollback unconditionally deletes the store entry. In force or replacement flows, a transient filesystem error can erase a previously valid session record.
  - confidence: 0.82
- **[P2] Send read receipts before terminal completion:** `src/agents/tools/sessions-send-tool.a2a.ts:52-58`
  - body: The receipt path only runs after `agent.wait` returns `ok`, which means a terminal snapshot. For long-running fire-and-forget sends, no receipt is emitted even though the target agent began processing.
  - confidence: 0.86
