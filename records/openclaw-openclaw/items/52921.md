## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use the repo TypeBox package:** `src/agents/tools/sessions-start-tool.ts:1`
  - body: The new tool imports `@sinclair/typebox`, while this repo declares and consistently imports `typebox`; loading this file can fail before `sessions_start` is usable.
  - confidence: 0.94
- **[P1] Don't redirect missing named sessions to main:** `src/gateway/server-methods/sessions.ts:408-428`
  - body: When `sessions.send` misses any agent-scoped key, this bootstraps the agent's main session and continues there. A missing key like `agent:ops:incident-123` would be silently delivered into `agent:ops:main`, merging unrelated conversations instead of returning `SESSION_NOT_FOUND`.
  - confidence: 0.91
- **[P1] Allocate a fresh id before bootstrapping dead sessions:** `src/gateway/sessions-bootstrap.ts:117-122`
  - body: `applySessionsPatchToStore()` preserves an existing store entry. If the dead main entry exists without `sessionId`, this patch does not allocate a new id before transcript creation, so bootstrap can fail and roll back the saved entry instead of recovering the session.
  - confidence: 0.88
- **[P2] Enforce visibility policy before sessions_start:** `src/agents/tools/sessions-start-tool.ts:32-45`
  - body: `sessions_start` forwards any `agentId` directly to the gateway and returns session metadata without the visibility, sandbox, and A2A checks used by `sessions_send`. In restricted setups, a model could start or probe sessions it should not be able to access.
  - confidence: 0.92
- **[P2] Classify sessions.start under write scope:** `src/gateway/server-methods-list.ts:70`
  - body: Adding `sessions.start` to the public method list without adding it to `method-scopes.ts` leaves it unclassified. Non-admin clients with normal `operator.write` scopes will hit the default `operator.admin` requirement when calling the new API.
  - confidence: 0.9
- **[P2] Register sessions_start in profile catalogs:** `src/agents/openclaw-tools.ts:263-265`
  - body: `createOpenClawTools` exposes the new tool, but profile filtering is driven by `CORE_TOOL_DEFINITIONS`. Without a catalog entry, agents using normal coding or messaging profiles will have the recovery tool filtered out before execution.
  - confidence: 0.86
- **[P1] Keep the cron fallback near the next occurrence:** `src/cron/schedule.ts:126-128`
  - body: Changing the fallback anchor to `now + 48h` can skip the next daily occurrence whenever this recovery path is used. The existing regression coverage documents this 48-hour jump failure mode, so the fallback should stay anchored near the next day instead of two days out.
  - confidence: 0.87
- **[P1] Catch schedule failures in forced cron preflight:** `src/cron/service/ops.ts:542-543`
  - body: Calling `computeJobNextRunAtMs()` directly during force preflight can throw for malformed schedules before the manual run is enqueued. The maintenance path catches those failures and records schedule errors, so the forced run path should use the same recoverable behavior.
  - confidence: 0.84
- **[P2] Restore the prior session entry on bootstrap failure:** `src/gateway/sessions-bootstrap.ts:138-140`
  - body: When transcript creation fails, rollback unconditionally deletes the store entry. In force or replacement flows, a transient filesystem error can erase a previously valid session record instead of preserving it.
  - confidence: 0.82
- **[P2] Emit receipts when processing starts:** `src/agents/tools/sessions-send-tool.a2a.ts:54-58`
  - body: The receipt path only runs after `agent.wait` returns `ok`, which is a terminal snapshot. For long-running fire-and-forget sends, no receipt is emitted even though the target agent began processing.
  - confidence: 0.86
- **[P2] Preserve bootstrap storage errors:** `src/gateway/server-methods/sessions.ts:418-424`
  - body: If transcript creation fails, `bootstrapAgentSession()` returns `UNAVAILABLE`, but this branch rewrites every non-`AGENT_NOT_FOUND` bootstrap failure to `SESSION_NOT_FOUND`, hiding storage or permission failures from callers.
  - confidence: 0.84
- **[P2] Emit session changes when sessions.start creates:** `src/gateway/server-methods/sessions.ts:887-895`
  - body: `sessions.start` can create a new session, but the handler returns success without emitting the session list change event used by `sessions.subscribe`, so other clients may not refresh until a later mutation.
  - confidence: 0.8
