## Summary

Keep open. Current main still lacks the requested session recovery surface, and this PR remains an implementation candidate, but it has missing real behavior proof plus blocking correctness and security findings that need maintainer-directed revision.

## What This Changes

The PR adds structured session-miss errors, `sessions.send` auto-bootstrap, a new `sessions.start` gateway method and `sessions_start` agent tool, send `messageId`/read-receipt behavior, generated Swift/tool display updates, and cron scheduler changes.

## Best Possible Solution

Land a focused session recovery change that preserves existing visibility/A2A/sandbox gates, defines the lifecycle/protocol contract, updates schemas/generated clients/docs/tests together, and splits cron scheduling into a separate review path.

## Reproduction Assessment

Yes for the current source-level limitation: main still returns `session not found` before any bootstrap path and has no `sessions.start`/`sessions_start` surface. I did not live-reproduce the original Ubuntu/MiniMax upgrade scenario.

## Solution Assessment

No. The direction is plausible, but this PR is not the best shape because it misroutes missing named sessions, omits access-policy gates, has lifecycle/rollback bugs, and bundles unrelated cron scheduler changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use the repo TypeBox package:** `src/agents/tools/sessions-start-tool.ts:1`
  - body: The new tool imports `@sinclair/typebox`, while this repo declares and consistently imports `typebox`; loading this file can fail before `sessions_start` is usable.
  - confidence: 0.94
- **[P1] Keep missing named sessions from falling back to main:** `src/gateway/server-methods/sessions.ts:408-428`
  - body: When an agent-scoped key is missing, this bootstraps the agent's main session and continues there. A missing key like `agent:ops:incident-123` would be delivered into `agent:ops:main`, merging unrelated conversations instead of returning not-found.
  - confidence: 0.91
- **[P1] Allocate a fresh session id before bootstrap:** `src/gateway/sessions-bootstrap.ts:117-122`
  - body: `applySessionsPatchToStore()` preserves an existing store entry, so bootstrapping a dead main entry without `sessionId` can still leave no fresh id for transcript creation and fail recovery.
  - confidence: 0.89
- **[P2] Apply session visibility gates to sessions_start:** `src/agents/tools/sessions-start-tool.ts:32-47`
  - body: `sessions_start` forwards any `agentId` directly to the gateway and returns session metadata without the visibility, sandbox, and A2A checks used by `sessions_send`, allowing restricted agents to probe or start sessions outside their allowed surface.
  - confidence: 0.92
- **[P2] Classify sessions.start under write scope:** `src/gateway/server-methods-list.ts:72`
  - body: Adding `sessions.start` to the public method list without adding it to `method-scopes.ts` leaves it unclassified, so non-admin clients with normal write scopes will hit the default admin requirement.
  - confidence: 0.9
- **[P2] Add sessions_start to profile catalogs:** `src/agents/openclaw-tools.ts:266-268`
  - body: `createOpenClawTools` exposes the new tool, but profile filtering is driven by `CORE_TOOL_DEFINITIONS`; without a catalog entry, coding or messaging profiles can filter the recovery tool out before execution.
  - confidence: 0.86
- **[P1] Keep cron fallback anchored near the next run:** `src/cron/schedule.ts:126-128`
  - body: Changing the fallback anchor to `now + 48h` can skip the next daily occurrence whenever this recovery path is used, reintroducing the 48-hour jump the existing fallback avoided.
  - confidence: 0.87
- **[P1] Catch schedule errors in force preflight:** `src/cron/service/ops.ts:542-544`
  - body: Calling `computeJobNextRunAtMs()` directly during force preflight can throw for malformed schedules before the manual run is enqueued; the maintenance path records schedule errors instead of aborting the operator action.
  - confidence: 0.84
- **[P2] Restore the previous session entry on rollback:** `src/gateway/sessions-bootstrap.ts:138-140`
  - body: When transcript creation fails, rollback unconditionally deletes the store entry. In force or replacement flows, a transient filesystem error can erase a previously valid session record.
  - confidence: 0.82
- **[P2] Emit receipts when processing starts:** `src/agents/tools/sessions-send-tool.a2a.ts:52-58`
  - body: The receipt path only runs after `agent.wait` returns `ok`, which is a terminal snapshot. For long-running fire-and-forget sends, no receipt is emitted even though the target agent began processing.
  - confidence: 0.86
