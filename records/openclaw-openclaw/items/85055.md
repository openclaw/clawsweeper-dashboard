## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep fallback claim IDs independent of rank:** `extensions/memory-wiki/src/compile.ts:1258`
  - body: `buildStableClaimId` hashes the sorted claim index, but `sortClaims(page)` ranks by freshness and confidence. An unchanged id-less claim can receive a different `claim_id` when mtimes age or another claim's freshness changes, breaking supersession links and downstream lookups; derive the fallback from stable claim/page fields instead.
  - confidence: 0.92
- **[P2] Preserve duplicate explicit claim IDs:** `extensions/memory-wiki/src/compile.ts:1373-1378`
  - body: Reconciled claims are keyed in a `Map` by `claim_id` only, so copied frontmatter or duplicate explicit IDs overwrite earlier entries. The later reconciled row can then be reused for multiple raw claims, corrupting statement/status/supersession output in `claims.jsonl`; keep reconciliation paired to each raw claim or reject/disambiguate duplicates.
  - confidence: 0.88
- **[P2] Keep compile manifest source-import provenance truthful:** `extensions/memory-wiki/src/compile.ts:1540`
  - body: The new manifest always records `source_import`, but `wiki.compile` still syncs imported sources and calls `compileMemoryWikiVault` without passing that sync result. A compile that imported or skipped sources can therefore emit zeros and `not-run`, making the cache manifest untrustworthy; pass the sync result through the compile paths or avoid claiming source-import metrics there.
  - confidence: 0.84
