Full review comments:

- **[P1] Make the agent status path read-only:** `extensions/memory-wiki/docs/AGENT_MEMORY_CONTRACT.md:7`
  - body: The new contract tells agents that `wiki.status` is a pure read preflight, but the registered model-facing `wiki_status` tool still calls `syncMemoryWikiImportedSources` before resolving status. Agents following this contract through the exposed tool will still import/compile and mutate cache state, so the read/write boundary remains inconsistent.
  - confidence: 0.88
- **[P2] Carry sync provenance into auto-refresh manifests:** `extensions/memory-wiki/src/compile.ts:1756`
  - body: When imported sources change through `syncMemoryWikiImportedSources`, `refreshMemoryWikiIndexesAfterImport` can compile immediately, but that path still calls `compileMemoryWikiVault(params.config)` without the sync counts. Tool/search/doctor paths that only perform this sync can therefore leave a manifest saying the source import was default zero/not-run even though imports were applied.
  - confidence: 0.86
- **[P2] Keep fallback IDs independent of claim order:** `extensions/memory-wiki/src/compile.ts:1262`
  - body: The fallback `claim_id` hash includes `originalIndex`, so inserting or moving an id-less claim earlier in the same page changes IDs for otherwise unchanged claims. Those IDs are now used by receipts and supersession metadata, so derive fallback IDs from stable claim/page content and only disambiguate true duplicates.
  - confidence: 0.82
