## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep generated claim IDs stable:** `extensions/memory-wiki/src/compile.ts:1258`
  - body: `buildStableClaimId` includes the sorted claim index in the hash, and `buildClaimsDigestRecords` calls it after `sortClaims`. Adding a higher-ranked claim or changing confidence/freshness can therefore change an existing no-id claim's `claim_id`, which breaks `supersedes` / `superseded_by` links and downstream cache consumers. Hash an order-independent basis such as page path plus statement/source/evidence, and add a reorder/insert regression test.
  - confidence: 0.86
- **[P2] Record sync results in compile manifests:** `extensions/memory-wiki/src/cli.ts:534`
  - body: The synced compile path captures `sync`, but `wiki.compile` discards it and calls `compileMemoryWikiVault(params.config)`, while the gateway compile path does the same after syncing. When imported sources are added or updated, the emitted strict manifest reports zero source-import counts and `indexes_refreshed: false`. Pass `{ operation: "compile", ...sync }` into both compile paths and cover it in tests.
  - confidence: 0.9
