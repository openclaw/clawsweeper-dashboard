## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Keep fallback claim IDs independent of sorted rank:** `extensions/memory-wiki/src/compile.ts:1258`
  - body: `buildStableClaimId` hashes the index from `sortClaims(page)`, and that sort ranks claims by freshness. An unchanged id-less claim can therefore get a different `claim_id` as freshness changes, breaking stable claim references and supersession links.
  - confidence: 0.91
- **[P2] Do not collapse duplicate explicit claim IDs:** `extensions/memory-wiki/src/compile.ts:1373-1378`
  - body: Re-indexing reconciled claims in a `Map` by `claim_id` overwrites earlier records when copied frontmatter or duplicate IDs appear. The later lookup then reuses the wrong reconciled object for multiple raw claims and can corrupt `claims.jsonl` status or supersession output.
  - confidence: 0.9
- **[P2] Pass source-import provenance into compile manifests:** `extensions/memory-wiki/src/compile.ts:1540`
  - body: `wiki.compile` and gateway compile still sync imported sources before compiling, but the manifest only sees `sourceImport` when callers pass the new option. Those paths can emit `source_import` as zero or `not-run` immediately after doing import work, which makes the manifest unreliable.
  - confidence: 0.86
- **[P1] Provide an upgrade path for read-only wiki.status:** `extensions/memory-wiki/src/cli.ts:468-470`
  - body: Current main uses `wiki.status` as a refresh trigger for imported sources, but this branch makes status read-only. Existing automation will only discover the change when cache freshness stops updating, so this needs compatibility handling or explicit public migration docs and proof for callers moving to `wiki.refresh`.
  - confidence: 0.82
