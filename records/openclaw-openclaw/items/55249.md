## Summary

Keep open. Current main has backend/session-store support for labels, but the CLI workflow requested here is still incomplete: there is no `openclaw sessions label` subcommand, local CLI rows do not carry labels into text or JSON output, and the CLI docs do not document label management.

## What This Changes

Request to add human-readable session labels or nicknames, including a CLI label command and visible label fields in session listings and JSON output.

## Best Possible Solution

Reuse the existing session-label validation and store semantics, then add a focused CLI/docs layer: a label management command for stored sessions, label/displayName propagation through local session display rows, label rendering in the table and JSON output, tests for the new command/output, and a short user-facing changelog entry.
