## Summary

Keep open. Current main has the backend label contract and Control UI label editing, but the requested local `openclaw sessions` label display and convenient CLI/chat label workflow are still missing; the related open PR and auto-label issue cover narrower adjacent pieces, not the whole request.

## What This Changes

Request to add user-settable session labels or nicknames that persist and appear in CLI, chat, and session listing surfaces.

## Best Possible Solution

Add a focused manual label workflow over the existing `sessions.patch` label contract, show labels in local CLI text and JSON while preserving raw keys, and leave auto-labeling or richer typed metadata to narrower follow-ups.

## Reproduction Assessment

Not applicable as a bug reproduction, but the feature gap is source-reproducible: current main lacks a sessions label subcommand, `/label` handler, and local label projection/rendering.

## Solution Assessment

Yes for the focused manual-label part: reusing the existing `label` field and `sessions.patch` path is the narrow maintainable direction, while auto-title generation and richer typed metadata should remain separate scope.
