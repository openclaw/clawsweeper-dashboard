## Summary

Keep this PR open. Current main still lacks a corporate proxy plus SSL-inspection troubleshooting guide, but the branch is not mergeable as-is because it edits localized docs that this repo no longer maintains, uses stale systemd unit guidance, recommends inline proxy environment values that current service audit treats as repair-worthy, and includes raw diagnostics that can expose proxy credentials.

## What This Changes

The PR adds a docs-only VPS guide for corporate proxy and SSL inspection setup under systemd, with proxy environment variables, `NODE_EXTRA_CA_CERTS`, and troubleshooting steps in English and Chinese docs.

## Best Possible Solution

Rework the contribution as an English-only docs update, centered on `docs/security/network-proxy.md` with a short `docs/vps.md` cross-link. The guide should use current `openclaw-gateway.service` user-systemd commands, `proxy.enabled` plus `proxy.proxyUrl` or durable `OPENCLAW_PROXY_URL`, startup-safe `NODE_EXTRA_CA_CERTS` guidance, and redaction-safe diagnostics; remove the localized docs edit from this source repo.

Not applicable as a user bug, but the review findings are reproducible by reading the PR composite diff against current main and checking the current docs/service code. The current-main gap is confirmed by `rg` and the PR-specific blockers are visible in the changed docs lines.

## Solution Assessment

No. The topic is useful, but this branch is not the best implementation because it mixes localized source edits with English docs, teaches stale/inline service override patterns, and prints raw proxy diagnostics instead of aligning with the current managed proxy and service environment contract.
