## Summary

Keep this PR open. The corporate proxy and SSL-inspection topic is still useful and not fully covered on current main, but the branch is not mergeable as-is because it edits localized docs that this repo no longer maintains, uses stale systemd service names, teaches inline proxy environment values that current service repair logic removes, and includes raw proxy diagnostics that can expose credentials.

## What This Changes

This PR adds a docs-only corporate proxy and SSL inspection section to `docs/vps.md` and `docs/zh-CN/vps.md`, covering systemd environment variables, `NODE_EXTRA_CA_CERTS`, and curl/journalctl troubleshooting.

## Best Possible Solution

Rework the contribution as an English-source docs update centered on `docs/security/network-proxy.md`, with a short `docs/vps.md` cross-link. It should use `openclaw-gateway.service` user-systemd commands, prefer `proxy.enabled` plus `proxy.proxyUrl` or durable `OPENCLAW_PROXY_URL`, include startup-safe corporate CA guidance, and make diagnostics redaction-safe without editing localized docs in this repo.

## Reproduction Assessment

Not applicable as a runtime bug. The review findings are reproducible by comparing the PR patch against current main docs and service/proxy contract files, especially `docs/AGENTS.md`, `docs/vps.md`, `docs/security/network-proxy.md`, and `src/daemon/service-env.ts`.

## Solution Assessment

No. The requested topic is valid, but this branch is not the best solution because it targets stale service commands, conflicts with managed proxy configuration and i18n policy, and asks users to print raw proxy diagnostics; a narrower English-source update aligned with current proxy docs is safer.
