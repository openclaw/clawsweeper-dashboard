## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Remove localized docs source edits:** `docs/zh-CN/vps.md:8-11`
  - body: `docs/AGENTS.md` says foreign-language docs are not maintained in this repo and `docs/<locale>/**` should not be added or edited here. This PR updates `docs/zh-CN/vps.md`, so the localized copy should be removed and left to the publish/i18n pipeline.
  - confidence: 0.96
- **[P2] Use the current gateway service unit:** `docs/vps.md:127`
  - body: Current Linux daemon docs and constants use the user unit `openclaw-gateway.service`, but this section tells users to edit and restart `openclaw` with `sudo systemctl`. Standard installs will target the wrong unit or fail, so the example should use `systemctl --user ... openclaw-gateway.service` and mention system-unit installs separately.
  - confidence: 0.92
- **[P2] Align proxy setup with managed proxy config:** `docs/vps.md:135-145`
  - body: Current proxy docs and service env code prefer `proxy.enabled` plus `proxy.proxyUrl`, or durable `OPENCLAW_PROXY_URL` for installed services. Recommending inline `HTTP_PROXY`/`HTTPS_PROXY` and `no_proxy` service values conflicts with the managed service contract and the audit path that flags embedded proxy env values.
  - confidence: 0.9
- **[P2] Redact proxy diagnostics:** `docs/vps.md:168-174`
  - body: The `/proc/<pid>/environ` probe and proxy log grep can reveal proxy URLs with credentials or internal endpoints when users paste troubleshooting output. The docs should avoid raw env/log dumps or show an explicit redaction-safe command.
  - confidence: 0.88
