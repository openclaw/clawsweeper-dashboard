## Summary

Keep this PR open, but not merge-ready: it fixes the right plugin-registry failure mode, yet the new recoverability guard is broader than the discovery contract and can force source rebuilds for durable but non-discoverable install records.

## What This Changes

This PR makes `policy-changed` plugin registry refreshes fall back to a source rebuild when persisted install records are missing from the persisted `plugins[]` view, sharing that check with snapshot loading and adding regression tests.

## Best Possible Solution

Keep the recovery behavior, but gate the missing-install fallback on the same discoverability rules as plugin discovery so durable broken install records remain stable.

## Reproduction Assessment

Yes from source: current main's `policy-changed` fast path updates enablement without rebuilding `plugins[]`, so a persisted index with install records but an empty plugin view can stay broken. I did not run the GPU sandbox reproducer.

## Solution Assessment

No. The right fix is to recover missing install-record-backed plugins, but this implementation should use discovery-backed recoverability rather than raw path existence.
