The branch adds a shared installed-plugin recovery helper, discovery probes, policy-refresh and snapshot integration, focused registry tests, and a small channel test-helper regex cleanup.

## Best Possible Solution

Land this shared recovery implementation only after maintainers accept the persisted-registry rebuild semantics, keeping stale or unrecoverable install records on the fast path.

## Solution Assessment

Yes, if maintainers accept the upgrade behavior. Sharing the recovery predicate across policy refresh and snapshot loading is narrower than a validator-only fallback and keeps `installRecords` durable while treating `plugins[]` as the rebuildable view.
