## Summary

Keep this PR open for maintainer review: the patch appears to fix the source-reproducible policy-refresh bug, but it intentionally changes persisted registry rebuild behavior and overlaps with another open PR for the same linked issue.

## What This Changes

The branch makes `policy-changed` plugin registry refreshes fall back to a source rebuild when recoverable install records are missing from the persisted `plugins[]` view, sharing that check with snapshot loading.

## Best Possible Solution

Land one canonical discovery-backed recovery implementation that preserves durable install records, rebuilds only when missing plugin entries can be recovered by normal discovery, and keeps stale or unrecoverable records on the fast path.

## Reproduction Assessment

Yes. Current main can take the `policy-changed` fast path while durable install records are absent from `plugins[]`, and the linked issue provides a concrete GPU sandbox reproduction where path/npm plugins become invisible.

## Solution Assessment

Yes, if this PR is chosen as the canonical implementation. Fixing the persisted registry fast-path guard is narrower and more maintainable than adding validator-only fallback behavior because docs treat `installRecords` as durable and `plugins[]` as rebuildable.
