## Summary

Keep open: current `main` and `v2026.6.8` still have `AgentSession` post-turn paths reading the in-memory model snapshot, and the open closing PR at https://github.com/openclaw/openclaw/pull/93056 is not merged yet.

## What This Changes

The issue requests fixing stale `AgentSession.this.model` state after session-level `/model` switches so compaction, retry, thinking, branch summary, and context usage use the selected model.

## Best Possible Solution

Land a reviewed fix that synchronizes the running session from the canonical selected provider/model identity, or otherwise makes all affected post-turn reads use that canonical identity, with cross-provider regression coverage.

## Reproduction Assessment

Yes, source-level. Current `main` can persist a different provider/model override while a running `AgentSession` keeps the old `agent.state.model`, and the reported post-turn paths still read that snapshot.

## Solution Assessment

Unclear for the open PR until it receives a full PR review. The best fix direction is to synchronize at the shared session model-switch boundary or use the selected provider/model identity for every affected post-turn read, not to close the issue before the candidate merges.
