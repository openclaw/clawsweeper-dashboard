## Summary

Keep open: current main and the latest release still have the stale `AgentSession.this.model` post-turn reads, while the broad linked fix was closed unmerged and the remaining open related PR is a narrower, previously reviewed incomplete fix.

## What This Changes

The issue reports that session-level model state used by `AgentSession` post-turn reads remains stale after a runtime `/model` switch.

## Best Possible Solution

Synchronize the running session from the canonical selected provider/model at the shared model-switch boundary, or make all post-turn consumers resolve from that canonical identity, with cross-provider regression coverage and live switch proof.

## Reproduction Assessment

Yes, source-level. Persisting a new session-entry provider/model while a running `AgentSession` keeps its old `agent.state.model` leaves current post-turn consumers reading the stale snapshot for compaction, retry, branch summary, thinking, and context usage.

## Solution Assessment

No, the known helper-only PRs are not the best fix because their registry lookup starts from stale provider/model identity. The safer fix is to synchronize the live session from the canonical selected identity or resolve all post-turn reads from that selected identity.
