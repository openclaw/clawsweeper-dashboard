## Summary

Keep open: current main and the latest release still have the stale `AgentSession.this.model` post-turn reads, and the open linked fix at https://github.com/openclaw/openclaw/pull/93056 is not merged yet and still has review/proof blockers.

## What This Changes

The issue reports that `/model` changes can update the persisted selected model without refreshing the running `AgentSession` model snapshot, leaving compaction, retry, thinking, context usage, and branch-summary logic on stale model metadata.

## Best Possible Solution

Review, repair, and land the linked open fix PR or replace it with a canonical model-switch-boundary fix, then close this issue after the merge reaches main.

## Reproduction Assessment

Yes, source-level. Persisting a different provider/model override while a running `AgentSession` keeps its old `agent.state.model` leaves current post-turn methods reading stale `this.model` metadata.

## Solution Assessment

No current-main solution exists. The best fix should synchronize the running session from the canonical selected provider/model at the shared model-switch boundary or make post-turn consumers resolve from that canonical identity, with cross-provider coverage.
