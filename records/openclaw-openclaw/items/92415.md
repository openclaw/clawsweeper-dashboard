## What This Changes

Ensure all AgentSession post-turn model-dependent behavior follows the newly selected model after a mid-session model switch.

## Best Possible Solution

Carry the canonical newly selected provider/model identity—or the already resolved active Model object—into AgentSession once at the lifecycle-owned switch/run boundary, then use that same model for request and post-turn decisions with cross-model and cross-provider regression coverage.
