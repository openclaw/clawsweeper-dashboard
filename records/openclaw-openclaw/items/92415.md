## What This Changes

The issue reports that `/model` switches can update persisted selection while a running `AgentSession` keeps using the previous in-memory model snapshot for post-turn context, thinking, retry, and branch-summary decisions.

## Best Possible Solution

Review and repair https://github.com/openclaw/openclaw/pull/93056, or replace it with a narrower fix that synchronizes the running session from the canonical selected provider/model and proves cross-provider context and branch-summary behavior.

## Reproduction Assessment

Yes, source-level. Persisting a different provider/model override while a running `AgentSession` keeps its old `agent.state.model` leads current post-turn consumers to read stale `this.model`; I did not run the long-context live path in this read-only pass.

## Solution Assessment

Yes for keeping the issue open, but no for the earlier helper-only approach. The fix should start from the newly selected provider/model or synchronize the session snapshot at the shared switch boundary, then add regression and live proof.
