## Summary

Keep this open: current main still has the stale `AgentSession.model` snapshot reads, the latest release has the same shape, and the only linked fix PR was closed unmerged after review found it still keyed from stale model identity.

## What This Changes

The issue asks OpenClaw to refresh or re-resolve session-level model state after `/model` switches so post-turn context, thinking, retry, and branch-summary logic use the active model.

## Best Possible Solution

Land a focused sessions fix that resolves post-turn model consumers from the canonical selected provider/model identity, or refreshes the `AgentSession` snapshot at every model-switch boundary, with production AgentSession regression coverage for cross-model context and cross-provider summary routing.

## Reproduction Assessment

Yes, at source level: current main still stores model override selection separately while `AgentSession` post-turn consumers read `this.agent.state.model` directly. I did not run a live long-context Telegram/CLI repro in this read-only pass.

## Solution Assessment

Yes, the best fix is a narrow session-state repair, but the closed helper PR was not the best implementation because it resolved the registry key from the stale snapshot. A maintainable fix should start from canonical selected model state or update the snapshot at the switch boundary.
