## Summary

Keep open: current `main` and the latest release still have the stale `AgentSession.this.model` post-turn reads, while the broader linked fix was closed unmerged and the remaining open related PR is narrower and previously reviewed as insufficient.

## What This Changes

This issue reports that `AgentSession` keeps a stale model snapshot after `/model` switches, causing compaction, retry, thinking, context usage, and branch-summary decisions to use the previous model/provider.

## Best Possible Solution

Synchronize the running session from the canonical selected provider/model identity at the shared model-switch boundary, or make all post-turn consumers resolve from that canonical identity, with cross-provider regression coverage and live model-switch proof.

## Reproduction Assessment

Yes, source-level. Persist a different provider/model selection while a running `AgentSession` keeps its old `agent.state.model`; current post-turn code then reads `this.model` for compaction, thinking, retry, branch summary, and context usage.

## Solution Assessment

No, the closed helper-only PR is not the best fix because it looked up the registry from the stale identity. The maintainable fix should start from the newly selected provider/model or update the session snapshot at the shared switch boundary.
