## Summary

This is a current core session-state regression, and current main still leaves AgentSession post-turn behavior tied to an older in-memory model after persisted model selection changes. The issue should remain open because https://github.com/openclaw/openclaw/pull/92424 explicitly owns the fix but is still open, and its proposed lookup appears to retain the old provider/model identity rather than adopting the newly selected one.

## What This Changes

Synchronize AgentSession’s model identity and model-dependent post-turn decisions after a session model switch.

## Best Possible Solution

When the persisted live model switch is adopted, resolve the selected provider/model once and update AgentSession’s canonical model object, then add lifecycle regression coverage proving context limits, thinking capabilities, retries, compaction, usage, and branch summaries all follow cross-provider and cross-context-window switches.

## Reproduction Assessment

Yes. Current source establishes separate persisted-selection and AgentSession snapshot update paths plus direct post-turn snapshot consumers, and the reporter supplied a concrete live 1M-versus-200k context observation.

## Solution Assessment

No, the linked implementation is not yet the best fix: resolving the registry with the old snapshot identity cannot adopt a newly selected model. Synchronizing the canonical AgentSession model once at the live-switch boundary is narrower and avoids repeated hot-path discovery.
