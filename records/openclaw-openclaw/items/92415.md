## Summary

Current main still separates the selected session override from AgentSession’s in-memory model snapshot, and multiple post-turn decisions read that snapshot. The report is actionable and already has an open closing PR, so this issue should remain open until that PR is corrected as needed, proven against representative consumers, and merged.

## What This Changes

Refresh the model metadata used by AgentSession post-turn logic after a session changes models or providers.

## Best Possible Solution

Land a focused fix that makes every AgentSession post-turn consumer observe the selected provider/model after all override paths, with regression tests covering a context-window change and a provider or reasoning-capability change.

## Reproduction Assessment

Yes, at high-confidence source level. Current main updates the persisted session override separately from the live AgentSession snapshot, and the reporter supplied a concrete 1M-versus-200k context-usage observation that follows this path.

## Solution Assessment

Unclear.
