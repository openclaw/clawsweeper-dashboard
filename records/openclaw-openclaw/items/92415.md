## Summary

Keep open: current main and the latest release still contain the stale `AgentSession.this.model` reads after session-scoped `/model` changes, and the open closing PR is a fix candidate rather than an implemented fix.

## What This Changes

The issue reports that session-scoped model switches update stored provider/model overrides but leave the running `AgentSession` model snapshot stale for post-turn context, thinking, retry, compaction, and branch-summary reads.

## Best Possible Solution

Land the linked root-cause fix or an equivalent replacement that synchronizes the running session from the canonical selected provider/model before post-turn reads use `this.model`.

## Reproduction Assessment

Yes, source-level: the model picker can persist a new provider/model selection while `AgentSession` post-turn code still reads the old in-memory `this.model` snapshot. I did not run the live long-context scenario in this read-only pass.

## Solution Assessment

Yes, the issue points at the right root boundary. The best fix should synchronize the running session from the stored selected provider/model, not re-resolve from the already-stale model identity at each consumer.
