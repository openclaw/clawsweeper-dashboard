## Summary

Keep open: current main and the latest release still have direct `AgentSession.this.model` reads after session-scoped model override writes, and the linked closing PR is open rather than merged. The issue is still necessary as the canonical tracker for this session-state bug.

## What This Changes

The issue reports stale session-level model state after a session-scoped `/model` switch, causing post-turn session reads to use the previous model for context, thinking, retry, and branch-summary decisions.

## Best Possible Solution

Land or replace the linked fix PR with a canonical session-model sync that refreshes `AgentSession` from the selected session override before model-dependent prompt and post-turn decisions, with focused regression coverage and real behavior proof.

## Reproduction Assessment

Yes, source-level: current main writes session overrides from the model picker while `AgentSession` still uses the in-memory `this.model` snapshot for post-turn context, retry, thinking, and summary paths. I did not run the live long-context Telegram or CLI scenario in this read-only review.

## Solution Assessment

No final fix is on main yet. The narrow helper idea is not enough for cross-provider switches; the best direction is to sync from canonical session override state before model-dependent session decisions, then prove that through the linked PR or a replacement.
