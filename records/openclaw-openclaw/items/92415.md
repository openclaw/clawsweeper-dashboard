## Summary

Keep open: current main and the latest release still have the stale `AgentSession.this.model` reads, and a mergeable open PR is the active closing fix candidate rather than proof that the issue is resolved.

## What This Changes

The issue reports that session-level model state is not refreshed after a `/model` switch, so post-turn AgentSession behavior can use the previous model's context, reasoning, thinking-level, and provider data.

## Best Possible Solution

Land the linked root-cause fix or an equivalent maintainer-approved replacement that synchronizes the running session from the canonical selected provider/model before post-turn reads use it.

## Reproduction Assessment

Yes, source-level: the model picker can persist a new provider/model selection while AgentSession post-turn code still reads the old in-memory `this.model` snapshot. I did not run the live long-context scenario in this read-only review.

## Solution Assessment

Yes, the issue points at the right root boundary. The best fix should synchronize the running session from the stored selected provider/model, not re-resolve from the already-stale model identity at each consumer.
