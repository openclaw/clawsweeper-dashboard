## Summary

Keep open: current main and the latest release still have the stale AgentSession model snapshot reads, while the linked closing PR remains open and has blocking review findings rather than a merged fix.

## What This Changes

The issue requests fixing stale AgentSession model state after model switches so context, thinking, retry, usage, and branch-summary logic use the active selected model.

## Best Possible Solution

Synchronize the running session from the canonical selected provider/model identity at the shared model-switch boundary, or make post-turn consumers resolve from that canonical identity, with cross-provider regression coverage and live switch proof.

## Reproduction Assessment

Yes, source-level. Persisting a different provider/model through the session override path can leave AgentSession.agent.state.model untouched while current post-turn methods read this.model for context, thinking, retry, branch-summary, and usage decisions.

## Solution Assessment

The helper-only direction from the closed PR is not the best fix because it can look up registry state from the stale provider/model identity. The stronger solution is to synchronize from the newly selected provider/model or update the session snapshot at the shared switch boundary.
