## Summary

Current main still has a source-backed split between the persisted session model override and `AgentSession.agent.state.model`, and the linked closing PR remains open. Keep this issue as the canonical tracker while maintainers verify that the proposed repair follows the newly selected model identity rather than only refreshing metadata for the previous identity.

## What This Changes

Make AgentSession post-turn model-dependent behavior use the active model after a session model switch.

## Best Possible Solution

Use one canonical active-model resolver that consumes the current session override identity and registry, then apply that resolved model to both agent attempts and every post-turn consumer with cross-provider regression coverage.

## Reproduction Assessment

Yes at source level. Current main stores the active session override separately from `AgentSession.agent.state.model` and directly reads that snapshot in the identified post-turn methods; the reporter also supplied a live 1M-versus-200k context mismatch.

## Solution Assessment

No, the linked approach is not yet proven as the best fix because it looks up registry metadata using the potentially stale snapshot identity. The narrow durable fix is a shared resolver for the active override identity and model object.
