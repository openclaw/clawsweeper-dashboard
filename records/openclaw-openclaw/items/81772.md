## Summary

Keep open: current main and v2026.6.6 can carry explicit Telegram rich links, including a `tg://user?id=...` link, but the remaining request is a new outbound mention contract or resolver rather than an already-implemented plain `@username` rewrite.

## What This Changes

The issue requests Telegram-channel support for highlighted user mentions using `tg://user?id=...`, either by rewriting mention intent or exposing a helper for agents to construct proper mentions.

## Best Possible Solution

Define a narrow outbound mention contract that accepts known numeric Telegram user IDs and renders Telegram user links through the existing rich-message path, leaving plain `@username` text unchanged unless maintainers approve a resolver.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partly: source inspection shows OpenClaw does not rewrite plain `@username` text or expose an outbound mention helper, but live Telegram notification/highlight behavior still needs a real Telegram repro with a known target user ID.

## Solution Assessment

Is this the best way to solve the issue? Unclear as proposed: explicit `tg://user?id=...` links already fit the rich-message path, so a typed numeric-ID mention helper is safer than a blanket Markdown parse-mode change or automatic username rewrite.
