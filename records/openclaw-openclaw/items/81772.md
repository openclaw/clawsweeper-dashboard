## Summary

Keep open: current main already supports explicit Telegram tg://user links when the sender writes them, but it still lacks the requested automatic @username-to-user-ID rewrite or explicit mention helper/config contract.

## What This Changes

The issue requests Telegram support for highlighted, clickable user mentions using tg://user?id= links instead of plain @username text.

## Best Possible Solution

Add a Telegram-owned explicit mention helper or mentionAliases-style config that maps known handles to numeric Telegram user IDs, document it, and avoid best-effort arbitrary username lookup in the send path.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken documented behavior. Source inspection shows explicit Markdown/HTML links can already be sent, while the requested @username-to-ID helper or alias path is absent.

## Solution Assessment

No: blindly rewriting every @username is not the best fix without deterministic numeric IDs. A configured alias map or explicit message-tool helper would be narrower and safer.
