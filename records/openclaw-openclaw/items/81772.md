## Summary

Keep open: current main and the latest release support explicit `tg://user?id=...` inline links, but the requested plain `@username` rewrite or typed Telegram mention helper is still a product/API decision rather than implemented behavior.

## What This Changes

The issue requests Telegram outbound support for highlighted user mentions via `tg://user?id=...`, including either automatic mention formatting or a helper for known user IDs.

## Best Possible Solution

Maintainers should either document explicit `tg://` inline links as the supported path or define a narrow Telegram mention contract for known numeric user IDs before implementing alias or resolver behavior.

## Reproduction Assessment

Partly: source inspection shows OpenClaw preserves explicit `tg://user?id=...` links but does not rewrite plain `@username` text or expose a typed Telegram mention helper. I did not establish the live Telegram notification behavior with a known target user ID.

## Solution Assessment

Unclear as proposed: a blanket `@username` rewrite is not the best solution because Telegram needs numeric user IDs and has privacy constraints. A typed known-ID mention helper, explicit docs, or a maintainer-approved alias policy would be safer.
