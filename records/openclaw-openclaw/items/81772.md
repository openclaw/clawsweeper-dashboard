## Summary

Keep open: current main and the latest release already support explicit Telegram inline/rich links, but the requested automatic `@username` rewrite or outbound numeric-ID mention helper is still missing and needs a Telegram product/API decision.

## What This Changes

The issue asks OpenClaw's Telegram channel to support highlighted clickable user mentions using `tg://user?id=...`, either by rewriting `@username`/mention intent or by exposing a helper for agents to construct numeric-ID mentions.

## Best Possible Solution

Keep the explicit rich-link send path and let maintainers decide whether Telegram should add a plugin-owned numeric-ID mention helper, a Discord-style alias map, or documentation that explicit `tg://user?id=...` links are the supported path.

## Reproduction Assessment

Not applicable as a strict bug reproduction. Source inspection shows explicit link formatting and the absence of an outbound helper/rewrite, but this read-only review did not run a live Telegram bot send to verify client highlighting or notifications.

## Solution Assessment

Unclear as proposed. Automatic `@username` rewriting is not the best default without a reliable username-to-ID source; a numeric-ID helper, alias map, or docs-only decision would be safer and should be chosen by maintainers.
