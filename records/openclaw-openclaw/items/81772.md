## Summary

Keep open: current main already renders explicit Telegram Markdown links through HTML parse mode, so a caller that knows the numeric user ID can likely emit a tg://user?id= mention, but OpenClaw does not currently provide the requested automatic @username-to-user-ID rewrite or a Telegram mention helper/alias surface.

## What This Changes

The issue requests Telegram channel support for highlighted user mentions by converting mention intent or @username text into tg://user?id= links.

## Best Possible Solution

Add a Telegram-owned mention alias/helper contract that uses explicit numeric user IDs to render safe tg:// mentions, document the supported syntax, and avoid best-effort arbitrary username resolution in the send path.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken documented behavior. Source inspection shows explicit Markdown links are already formatted for Telegram, while the requested alias/helper behavior is absent.

## Solution Assessment

No: automatically rewriting every @username is not the safest default without a deterministic ID mapping. A configured alias map or explicit message-tool helper would be the narrower maintainable solution.
