## Summary

Keep open: current main can already send explicit Telegram inline links through the Markdown-to-HTML path, but the requested automatic `@username` to numeric-ID rewrite or Telegram mention helper/alias contract is still missing and needs a maintainer product decision.

## What This Changes

Add Telegram outbound mention support so agents can produce highlighted, clickable user mentions via `tg://user?id=` links, either by rewriting `@username`/mention intent or by exposing an explicit helper.

## Best Possible Solution

Add a Telegram-owned explicit mention helper or deterministic alias/user-ID mapping on the message-tool/config surface, document it, and avoid rewriting raw `@username` text unless the mapping is explicit.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken documented behavior. Source and upstream docs show explicit inline `tg://user?id=` links can already be formatted today, while the requested `@username`-to-ID/helper path is absent.

## Solution Assessment

No: automatically rewriting every `@username` is not the best default because Telegram mention links require numeric IDs and have privacy/member constraints. A helper or explicit alias mapping would be the narrower maintainable direction.
