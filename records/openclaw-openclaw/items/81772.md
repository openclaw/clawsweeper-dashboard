## Summary

Keep open: current main can send explicit Telegram tg://user links through Markdown-to-HTML rendering, but the requested automatic @username-to-user-ID rewrite or Telegram mention helper/config surface is still missing and needs product judgment.

## What This Changes

Request support for Telegram outbound highlighted mentions so plain @username or explicit mention intent can become clickable tg://user?id= mentions.

## Best Possible Solution

Add a Telegram-owned explicit mention helper or mentionAliases-style config that maps known handles to numeric Telegram user IDs, document the syntax, and avoid best-effort arbitrary username lookup in the send path.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken documented behavior. Source inspection shows explicit Markdown/HTML links can be sent today, while the requested @username-to-ID alias/helper path is absent.

## Solution Assessment

No: blindly rewriting every @username in Telegram sends is not the safest solution without a deterministic ID mapping. A configured alias map or explicit message-tool helper using numeric user IDs would be the narrower maintainable path.
