## Summary

Keep open: current main already preserves explicit Telegram `tg://user?id=...` links, but it does not define a safe outbound mention contract or resolve plain `@username` text to numeric Telegram user IDs. That remaining behavior is a product/API decision, not a strict bug fix.

## What This Changes

Requested Telegram channel support for highlighted, clickable `tg://user?id=...` mentions instead of plain `@username` text.

## Best Possible Solution

Define a narrow Telegram outbound mention contract for known numeric user IDs through the plugin or SDK message surface, leaving plain `@username` unchanged unless maintainers approve a resolver or alias policy.

## Reproduction Assessment

Partly: source inspection shows explicit `tg://user?id` links render, but plain `@username` text is not rewritten or resolved. I did not establish a live Telegram notification repro with a real target user.

## Solution Assessment

Unclear as proposed: preserving explicit `tg://user?id` links is already supported, while automatic `@username` rewriting needs a numeric-ID or alias policy before it is safe. A typed known-ID mention helper or documented explicit-link workflow is the safer direction.
