## Summary

## What This Changes

The issue requests Telegram channel support for highlighted, clickable, notifying mentions using `tg://user?id=` links.

## Best Possible Solution

Define a narrow Telegram outbound mention contract for known numeric user IDs, likely mirroring Discord `mentionAliases` or adding a typed message helper, and leave unknown `@username` text unchanged.

## Reproduction Assessment

Partly: source inspection shows OpenClaw does not rewrite plain `@username` text or expose a typed Telegram outbound mention helper. I did not establish the live Telegram notification behavior with a known target user ID.

## Solution Assessment

Unclear as proposed: a blanket `@username` rewrite is not the best path because Telegram needs numeric IDs and privacy constraints apply. A typed known-ID helper or explicit alias policy is safer once maintainers choose the product shape.
