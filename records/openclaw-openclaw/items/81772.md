## Summary

Keep open: current main can send explicit Telegram `tg://user?id=` links through its Markdown-to-HTML path, but the requested automatic `@username` rewrite or mention helper is still a new Telegram-owned product/config surface. This needs a deterministic numeric-ID mapping decision before implementation.

## What This Changes

Add Telegram support for highlighted, clickable user mentions by converting known user references to `tg://user?id=` links or exposing a mention helper.

## Best Possible Solution

Add a Telegram-owned mention helper or alias map that requires explicit numeric user IDs, document the supported syntax, and avoid blind best-effort `@username` rewrites in the send path.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken documented behavior. Source inspection shows explicit `tg://user?id=` links are already renderable, while automatic `@username` mapping or helper support is absent.

## Solution Assessment

No: blindly rewriting every `@username` is not the best default because Telegram usernames are mutable and numeric IDs are the durable primitive. A configured alias map or explicit helper would be the narrower maintainable solution.
