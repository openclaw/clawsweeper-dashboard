## Summary

Keep open: current main can already send explicit Telegram inline links through its Markdown-to-HTML path, but the requested automatic `@username` rewrite or first-class Telegram mention helper/alias contract is still missing and needs a product/API decision.

## What This Changes

Request support for Telegram outbound mentions that render as highlighted clickable `tg://user?id=` user mentions instead of plain `@username` text.

## Best Possible Solution

Add a Telegram-owned explicit mention helper or deterministic alias/user-ID mapping on the plugin message/config surface, document it, and leave raw `@username` text unchanged unless a numeric-ID mapping is explicit.

## Reproduction Assessment

Not applicable: this is a feature request, not broken documented behavior. Source inspection and Telegram docs show explicit inline `tg://user?id=` links can already be formatted, while the requested automatic/helper path is absent.

## Solution Assessment

No: automatically rewriting every `@username` is not the best default because Telegram mentions require numeric user IDs and have privacy/member constraints. A helper or explicit alias mapping is the narrower maintainable direction.
