## Summary

Keep open: current main can send explicit Telegram inline links, but the requested automatic @username-to-user-ID rewrite or Telegram mention helper/alias contract is still missing and needs maintainer product judgment.

## What This Changes

The issue requests Telegram outbound highlighted mentions via tg://user?id links, either by rewriting @username text or by exposing a mention helper.

## Best Possible Solution

Add a Telegram-owned explicit mention helper or mentionAliases-style config that maps known handles to numeric Telegram user IDs, document the syntax, and avoid arbitrary username lookup in the send path.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken documented behavior. Source inspection shows explicit Markdown/HTML links can be sent today, while the requested @username-to-ID alias/helper path is absent.

## Solution Assessment

No: blindly rewriting every @username is not the safest solution without deterministic ID mapping and Telegram privacy constraints. A configured alias map or explicit message-tool helper using numeric user IDs would be the narrower maintainable path.
