## What This Changes

The issue requests Telegram-channel support for highlighted outbound mentions using `tg://user?id=...` links instead of plain `@username` text.

## Best Possible Solution

Define a narrow Telegram outbound mention contract for known numeric user IDs through the plugin or SDK message surface, leaving plain `@username` unchanged unless maintainers approve a resolver or alias policy.

## Reproduction Assessment

Partly: source inspection shows OpenClaw does not rewrite plain `@username` text or expose a typed outbound Telegram mention helper. I did not establish live Telegram notification behavior with a known target user ID.

## Solution Assessment

Unclear as proposed: a blanket `@username` rewrite is not the best path because Telegram needs numeric user IDs and has privacy constraints. A typed known-ID mention helper or explicit documented link workflow is safer once maintainers choose the product shape.
