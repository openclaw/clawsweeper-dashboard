## Summary

Keep open: current main can already deliver explicit Telegram inline links through the Markdown-to-HTML send path, but it does not provide automatic outbound `@username` rewriting, a Telegram mention helper, or a Telegram alias-to-user-ID contract. That remaining behavior is a Telegram UX/API decision, not a conservative cleanup close.

## What This Changes

Request support for highlighted, clickable, notifying Telegram mentions by formatting outbound messages as `tg://user?id=` links instead of plain `@username` text.

## Best Possible Solution

Add a Telegram-owned explicit mention helper or deterministic alias/user-ID mapping on the message-tool/config surface, document it, and avoid rewriting raw `@username` text unless the mapping is explicit.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken documented OpenClaw behavior. Source inspection shows explicit inline links can already be formatted, while the requested automatic rewrite/helper path is absent.

## Solution Assessment

No: automatically rewriting every `@username` is not the best default because Telegram mentions require numeric IDs and have privacy/member constraints. A helper or explicit alias mapping would be the narrower maintainable direction.
