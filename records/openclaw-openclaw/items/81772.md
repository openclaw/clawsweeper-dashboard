## Summary

Keep open: current main can already send explicit Telegram `tg://user?id=` inline links through Markdown-to-HTML rendering, but it still does not provide the requested automatic `@username` to numeric-ID rewrite or a Telegram mention helper/alias contract. The remaining work needs a Telegram UX/API decision because Telegram usernames are mutable and Bot API mentions require numeric user IDs plus eligibility constraints.

## What This Changes

The issue requests Telegram support for highlighted, clickable user mentions using `tg://user?id=` links or an equivalent helper.

## Best Possible Solution

Add a Telegram-owned explicit mention helper or deterministic alias/user-ID mapping, document it, and avoid rewriting raw `@username` text unless the numeric user ID mapping is explicit.

## Reproduction Assessment

Not applicable: this is a feature/API request rather than a broken documented behavior. Source review confirms explicit `tg://user?id=` links can already flow through Markdown/HTML, while the requested automatic username mapping or helper is absent.

## Solution Assessment

No: automatic raw `@username` rewriting is not the best default because Telegram mention links require numeric user IDs and have privacy/member constraints. A helper or explicit alias mapping would be the narrower maintainable direction.
