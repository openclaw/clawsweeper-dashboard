## Summary

Keep open: current main preserves explicit Telegram `tg://user?id=...` links, but it still has no safe outbound contract for converting plain `@username` text or mention intent into numeric Telegram user mentions. The remaining work needs a Telegram API/product decision, not cleanup closure.

## What This Changes

Request support in the Telegram channel for highlighted, clickable, notifying user mentions via `tg://user?id=...` instead of plain `@username` text.

## Best Possible Solution

Define a narrow Telegram outbound mention contract for known numeric user IDs, keep explicit `tg://user` links working, and avoid automatic username lookup unless maintainers approve an alias policy.

## Reproduction Assessment

No high-confidence live reproduction yet; source inspection shows OpenClaw does not rewrite plain `@username` text, but Telegram notification/highlight behavior still needs a real target-user proof.

## Solution Assessment

Unclear as proposed: explicit known-ID links already work, while automatic `@username` rewriting is unsafe without a numeric-ID or alias policy. A typed known-user-id mention helper or alias-backed contract is the safer direction.
