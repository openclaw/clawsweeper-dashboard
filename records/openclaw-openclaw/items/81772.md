## Summary

Keep open: current main and v2026.6.10 already preserve explicit Telegram `[name](tg://user?id=...)` links, but OpenClaw still has no safe outbound contract for turning plain `@username` text into a numeric-ID mention or exposing known-user mention metadata. That remaining behavior needs a Telegram product/API decision before implementation.

## What This Changes

Request Telegram outbound support for clickable, notifying user mentions, either by formatting known numeric-ID `tg://user` links or exposing a mention helper.

## Best Possible Solution

Define a narrow Telegram outbound mention contract for known numeric user IDs through the plugin or gateway message surface, keep explicit `tg://user` links working, and avoid automatic username resolution unless maintainers approve a safe alias policy.

## Reproduction Assessment

Partly: source inspection shows explicit `tg://user?id` links render, while plain `@username` text is not resolved or rewritten. I did not establish live Telegram notification proof with a real target user.

## Solution Assessment

Unclear as proposed: explicit known-ID links already work, but automatic `@username` rewriting is not the best safe fix without a numeric-ID or alias policy. A typed known-user-id mention helper or documented explicit-link workflow is the safer direction.
