## Summary

Keep open: current main can already send explicit Markdown links as Telegram HTML anchors, including tg:// user links, but it still lacks the requested automatic @username-to-user-ID rewrite or a documented Telegram mention helper/alias surface.

## What This Changes

Add Telegram outbound mention support so agents can send highlighted, clickable tg://user?id= mentions instead of plain @username text.

## Best Possible Solution

Add a Telegram-owned mention helper or alias contract using explicit numeric user IDs, document the supported syntax, and avoid blind @username rewriting.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken documented behavior. Source inspection shows explicit Markdown/HTML tg links can be sent today, while automatic @username rewriting or a helper surface is absent.

## Solution Assessment

No: automatically rewriting arbitrary @username text is not the best default without deterministic ID mapping. A Telegram-owned alias map or explicit helper that requires numeric user IDs is the narrower maintainable direction.
