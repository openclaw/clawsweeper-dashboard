## Summary

Current main already solves the reported npm-install Discord crash by publishing package-local compiled runtime artifacts for external plugins, including `@openclaw/discord`, so the runtime no longer has to load `src/monitor/native-command.ts` through jiti in the normal package path.

## What This Changes

The issue asks OpenClaw to stop Discord npm installs from crash-looping on the `./native-command.args.js` source import, either by disabling native jiti loading for the loader helper or by shipping compiled Discord runtime files.

## Best Possible Solution

Keep the package-local runtime publishing path and verify the next Discord plugin release includes its compiled `dist` entries; a broader loader-policy cleanup can be separate from this release crash.

## Reproduction Assessment

No, I did not establish a failing current-main runtime path. The release report and source inspection make the old source-only package path clear, while current main routes publishable plugin packages through built `dist` runtime entries.

## Solution Assessment

Yes. For the reported npm-install crash, shipping and selecting compiled Discord runtime files is the narrowest maintainable fix because it removes source jiti loading from the published package path without changing global loader policy.
