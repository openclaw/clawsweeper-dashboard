## Summary

Close as implemented on main. The reporter's follow-up correctly retracts the hardcoded `tryNative` hypothesis, and current main now solves the observed source-only npm package problem by building and publishing package-local runtime `dist` entries for publishable plugins, including `@openclaw/discord`; the fix is on main but not yet in a shipped tag.

## What This Changes

Investigate a Discord npm package crash loop where `native-command.ts` cannot resolve its `./native-command.args.js` sibling under Node 25/OpenClaw 2026.5.2.

## Best Possible Solution

Keep the package-local runtime build/publish path and ship the next `@openclaw/discord` package with built runtime entries; do not flip the intentional native-first `buildPluginLoaderJitiOptions` default.

## Reproduction Assessment

No live Discord gateway reproduction was run. The reported 2026.5.2 source-only package failure is source-backed by the `.js` sibling import shape and by v2026.5.2 lacking the package-local runtime build script, while current main routes future npm packages to built runtime entries.

## Solution Assessment

Yes. Building and declaring package-local runtime JavaScript is the narrowest maintainable fix for installed npm plugins, preserving source-checkout behavior and the intentional native-first loader path for built dist modules.
