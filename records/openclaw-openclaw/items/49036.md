## Summary

Keep open: current main still lacks `openclaw doctor` coverage for gateway lock files, and the open implementation candidate that closes this issue remains unmerged. ## What This Changes

Add `openclaw doctor` detection and optional `--fix` cleanup for stale or malformed gateway lock files under `$TMPDIR/openclaw-$UID/gateway.*.lock`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/49112 or an equivalent focused doctor contribution that reports gateway lock state and repairs only stale, dead, empty, or malformed locks under `--fix`.
