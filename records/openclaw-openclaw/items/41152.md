## Summary

Keep open: current main and the latest release still route ordinary async gateway exec approval completions through the compact background notification formatter, which tails output to 400 characters and normalizes whitespace before the agent resumes. The only focused fix linked from the discussion was closed unmerged, and exact-title search found no merged replacement that owns this output-formatting bug.

## What This Changes

The issue requests preserving full multi-line async gateway exec approval output when the completion is delivered back to the agent session.

## Best Possible Solution

Add a dedicated gateway approval-completion formatter that preserves captured output and newlines, explicitly marks real capture-cap truncation, and leaves compact background `notifyOnExit` summaries unchanged.

## Repair Work Prompt

Fix the async gateway exec approval completion output bug tracked at https://github.com/openclaw/openclaw/issues/41152. Current `src/agents/bash-tools.exec-host-gateway.ts` builds non-diagnostics approval follow-up text with `normalizeNotifyOutput(tail(outcome.aggregated, DEFAULT_NOTIFY_TAIL_CHARS))`, which drops the front of long output and strips newlines before resuming the agent session. Add a dedicated approval-completion formatter that preserves captured `outcome.aggregated` text and line structure, adds an explicit note only when the existing exec capture cap actually truncates output, and does not change ordinary background `notifyOnExit` summaries in `src/agents/bash-tools.exec-runtime.ts`. Add focused regression coverage in `src/agents/bash-tools.exec.approval-id.test.ts` for >400-character multi-line Discord/gateway approval completion and for capture-cap truncation signaling. Do not add config options, change exec capture limits, broaden background notification behavior, or edit CHANGELOG.md; put release-note context in the PR body if needed.
