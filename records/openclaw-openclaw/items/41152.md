## Summary

Keep open. Current main still routes async gateway and node exec approval completion through the compact notification formatter, so multiline output over 400 characters is tailed and whitespace-normalized before the follow-up agent turn. The linked fix PR #41170 is closed and unmerged in the provided GitHub context, and the latest shipped tag still contains the same compacting path.

## What This Changes

The issue requests that async exec approval completion deliver captured multiline command output with newlines preserved, instead of silently sending a 400-character normalized tail to the agent.

## Best Possible Solution

Implement a narrow approval-completion formatter split: gateway and node async approval follow-ups should deliver captured output with line breaks preserved up to the existing exec capture cap, explicitly mark capture-cap truncation, and leave ordinary background `notifyOnExit` summaries compact.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: approve an async gateway or node exec whose captured output is multiline and over 400 characters, and the completion text flows through `tail(..., DEFAULT_NOTIFY_TAIL_CHARS)` plus `normalizeNotifyOutput` before the agent follow-up prompt.

## Solution Assessment

Yes. The best fix is the scoped approach described by the closed #41170 branch: change only approval-completion result formatting and add regression coverage for multiline output and capture-cap truncation, rather than broadening the shared compact notification formatter.
