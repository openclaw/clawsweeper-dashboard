## What This Changes

The issue reports that async exec approval completion output delivered as a System message is truncated to a 400-character tail and has newlines collapsed.

## Best Possible Solution

Split approval-completion formatting from background notification summaries: preserve captured multiline output up to the existing exec capture cap, mark cap truncation, and leave notifyOnExit compact.

Yes. Source inspection on current main gives a high-confidence reproduction path: approved gateway or node async completions with multiline output over 400 characters are tailed and normalized before System follow-up delivery.

## Solution Assessment

Yes. The narrow maintainable fix is a formatter split for approval-completion results with gateway and node regression coverage, not a change to the generic background notifyOnExit formatter.
