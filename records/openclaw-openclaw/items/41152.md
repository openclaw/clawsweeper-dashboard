## Summary

Keep open: current main and the latest release still route approved gateway exec follow-up output through the compact 400-character whitespace-normalizing formatter, while the linked closing fix PR is open and unmerged.

## What This Changes

The issue reports that Discord async gateway exec approval follow-up results are flattened and tailed to about 400 characters before being injected back into the agent conversation.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92185 if it remains clean, then close this issue from that merged fix rather than opening another repair PR.

## Reproduction Assessment

Yes for source-level reproduction: current main builds approved gateway exec follow-ups with a 400-character tail and whitespace normalizer, so multiline output over that size is delivered as a single-line tail. I did not run a live Discord approval round trip in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve captured output only for approval-completion follow-ups while keeping ordinary background `notifyOnExit` summaries compact, which is the boundary used by the open linked PR.
