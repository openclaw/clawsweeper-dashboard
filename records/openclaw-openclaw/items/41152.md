## Summary

Keep open. Current main still sends async exec approval completion output through the compact notification formatter, so multiline output is tailed to 400 characters and whitespace-collapsed before the follow-up `System:` prompt is built; the linked focused fix PR is closed unmerged.

## What This Changes

The issue requests preserving full multiline async exec approval output in Discord/gateway follow-up delivery instead of silently tailing and flattening it.

## Best Possible Solution

Split approval-completion result formatting from compact background notification summaries, preserving captured multiline output up to the existing exec capture cap and explicitly marking real capture-cap truncation.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: async gateway or node exec approval completions with multiline output over 400 characters are tailed and whitespace-normalized before System follow-up delivery.

## Solution Assessment

Yes. The narrow maintainable fix is to separate approval-completion formatting from `notifyOnExit` summary formatting and add gateway/node regression coverage, rather than changing the generic compact notification formatter.
