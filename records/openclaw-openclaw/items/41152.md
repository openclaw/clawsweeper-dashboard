## Summary

Keep open. Current main and v2026.5.7 still route non-diagnostics async exec approval completions through a 400-character tail plus whitespace normalization before the follow-up agent turn, and the focused fix PR at https://github.com/openclaw/openclaw/pull/41170 is closed and unmerged.

## What This Changes

Async exec approval completions should deliver full multiline command output to the follow-up agent turn instead of a 400-character whitespace-collapsed tail.

## Best Possible Solution

Preserve captured multiline async approval completion output up to the existing exec capture cap, mark real capture-cap truncation, and leave compact background notifications unchanged.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: an approved gateway or node async exec completion with multiline output over 400 characters reaches tail plus normalizeNotifyOutput before the follow-up agent turn.

## Solution Assessment

Yes. The narrow maintainable fix is to split approval-completion formatting from compact background notification formatting, preserve captured newlines, and add focused regression coverage for gateway and node approval completions.
