## Summary

Keep open: current main moved native Discord approvals to the inline path, but the async approval follow-up formatter still tails ordinary non-native/internal follow-up output to 400 characters and collapses whitespace; the open closing PR is still the right active fix path.

## What This Changes

The issue reports that async gateway exec approval follow-up messages tail long output to roughly 400 characters and collapse newline boundaries before resuming the agent.

## Best Possible Solution

Review, refresh if needed, and land https://github.com/openclaw/openclaw/pull/92185 or an equivalent narrow fix that preserves captured output only for async approval-completion follow-ups while keeping ordinary background notifications compact.

## Reproduction Assessment

Yes for source-level reproduction of the remaining async route: current main still formats approved async gateway exec follow-ups through the 400-character tail and whitespace normalizer. The exact Discord DM path now appears to wait inline on native approvals, so I did not mark this as a live reproduced Discord failure.

## Solution Assessment

Yes. The best fix is the narrow formatter split proposed by the open closing PR: preserve captured approval-completion output and real capture-cap truncation while leaving ordinary `notifyOnExit` summaries compact.
