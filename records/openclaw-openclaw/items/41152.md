## Summary

Keep open: current main and v2026.5.20 still compact async exec approval completions with DEFAULT_NOTIFY_TAIL_CHARS and normalizeNotifyOutput before follow-up injection, and the only direct fix PR is closed unmerged.

## What This Changes

The issue reports that async exec approval completion follow-ups deliver only a whitespace-normalized tail of command output instead of preserving the captured multiline result.

## Best Possible Solution

Split approval-completion result formatting from compact notifyOnExit summaries for gateway and node, preserving multiline captured output up to the existing exec capture cap and marking real cap truncation explicitly.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: async gateway or node exec approval completions with multiline output over 400 characters are tailed to DEFAULT_NOTIFY_TAIL_CHARS and whitespace-normalized before follow-up injection.

## Solution Assessment

Yes. The maintainable fix is to split approval-completion result formatting from the compact background notification formatter and cover both gateway and node approval paths with focused regressions.
