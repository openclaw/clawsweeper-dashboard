## Summary

Keep open. Current main and the latest shipped release still route non-diagnostics async exec approval completions through the compact notification formatter, so multiline output over 400 characters is tailed and whitespace-normalized before the agent receives the follow-up; the linked focused fix PR is closed unmerged.

## What This Changes

The issue asks OpenClaw to preserve full multiline async exec approval output in the follow-up System message and mark real capture-cap truncation explicitly.

## Best Possible Solution

Add a dedicated approval-completion result formatter that preserves captured multiline output up to the existing exec capture cap, marks cap truncation explicitly, and leaves generic background notifyOnExit summaries compact.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: async gateway or node exec approval completions with multiline output over 400 characters are tailed and whitespace-normalized before the follow-up is injected into the agent session.

## Solution Assessment

Yes. The best fix is the narrow formatter split described by the closed focused PR: preserve approval-completion output and add a truncation marker without changing the existing compact background notification path.
