## Summary

Keep open: current main and the latest release still tail async exec approval completion output through the 400-character notify formatter and normalize whitespace before the follow-up reaches the agent.

## What This Changes

The issue reports that async exec approval results delivered as System follow-ups silently drop the front of long output and strip newlines.

## Best Possible Solution

Add a dedicated async exec approval result formatter for gateway and node completions that preserves captured multiline output up to the existing exec capture cap, marks real cap truncation explicitly, and keeps generic background notifications compact.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: async gateway or node exec approval completions with multiline output over 400 characters are tailed to DEFAULT_NOTIFY_TAIL_CHARS and whitespace-normalized before follow-up injection.

## Solution Assessment

Yes. The narrow maintainable fix is to split approval-completion result formatting from the compact notifyOnExit summary formatter and cover both gateway and node approval paths with regressions.
