## Summary

Keep open. Current main and the latest release still route gateway and node async exec approval completions through the 400-character compact notification formatter before follow-up delivery, while the linked fix PR #41170 is closed and unmerged.

## What This Changes

Async exec approval completion should preserve captured multiline command output in follow-up agent turns instead of delivering only a whitespace-normalized 400-character tail.

## Best Possible Solution

Split approval-completion formatting from compact notification formatting: preserve captured newlines up to the existing exec capture cap, mark capture-cap truncation, and keep ordinary background notifications compact.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main reproduction path: any approved gateway or node async exec with multiline output over 400 characters reaches `tail` plus `normalizeNotifyOutput` before the follow-up agent turn.

## Solution Assessment

Yes. The narrow maintainable solution is to reimplement the useful direction from closed PR #41170 on current main, with formatter-specific tests, rather than changing the shared compact notification formatter globally.
