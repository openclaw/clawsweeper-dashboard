## Summary

Keep open. Current main and the latest shipped release still format non-diagnostics async exec approval completions through the compact notify path, so output over 400 characters is tailed and whitespace-normalized before the agent receives the follow-up; the linked focused fix PR is closed unmerged.

## What This Changes

The issue asks async exec approval follow-up System messages to preserve complete multiline command output instead of a 400-character whitespace-normalized tail.

## Best Possible Solution

Split approval-completion result formatting from compact background notifications, preserve captured multiline output up to the existing exec capture cap, and mark real capture-cap truncation explicitly.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: async gateway or node exec approval completions with multiline output over 400 characters are tailed and whitespace-normalized before follow-up delivery.

## Solution Assessment

Yes. The narrow maintainable fix is to use a dedicated approval-completion formatter with regression coverage, instead of changing generic background `notifyOnExit` summaries.
