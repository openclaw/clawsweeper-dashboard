## Summary

Keep open: current main still has the async approval follow-up formatter that tails output to 400 characters and normalizes whitespace, while the open closing PR remains the active fix path. The exact native Discord path is narrower after the merged inline-approval work, but the remaining async follow-up route is still source-reproducible.

## What This Changes

The issue reports that async gateway exec approval follow-up output is delivered back to the agent as a flattened roughly 400-character tail instead of preserving full multiline output.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92185, or an equivalent narrow formatter fix, so approval-completion follow-ups preserve captured multiline output while ordinary background notifications stay compact.

## Reproduction Assessment

Yes for source-level reproduction of the remaining async route: current main still formats approved async gateway exec follow-ups through the 400-character tail and whitespace normalizer. I did not establish a live Discord DM reproduction because native Discord approvals now appear to wait inline on current main.

## Solution Assessment

Yes, the best fix shape is the narrow formatter split owned by the open closing PR: preserve captured approval-completion output and real capture-cap signaling while leaving ordinary notifyOnExit summaries compact.
