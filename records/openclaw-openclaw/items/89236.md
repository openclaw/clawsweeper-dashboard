## Best Possible Solution

Land the Slack dispatcher fallback after redacted live Slack proof shows omitted `userId` resolves to the inbound sender and explicit `userId` still wins; track Discord parity separately if maintainers want it.

## Reproduction Assessment

## Solution Assessment

Yes as a code shape, pending live proof. The dispatch layer is the narrow owner boundary for translating generic message action params into Slack-specific runtime calls, while the Slack runtime can keep requiring an explicit `memberInfo` `userId`.
