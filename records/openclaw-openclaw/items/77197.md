## Summary

Keep open. The PR targets the right missing-sharp resize-loop path, but the new remediation text is still not actionable for the reported global/package install failures because it tells operators to run local install commands that may not affect OpenClaw's runtime import path.

## What This Changes

The PR adds a missing-sharp-specific `optimizeImageToJpeg` error branch, a regression test, and an unreleased changelog entry for the diagnostic improvement.

## Best Possible Solution

Land the diagnostic improvement after the error text and regression test give global/package-install-aware remediation, while keeping #73148 open for the broader bundled Sharp runtime-dependency policy.

## Reproduction Assessment

Yes, from source inspection rather than executing tests. Current main routes image loading through `loadWebMedia` and `optimizeImageToJpeg`, while the Sharp runtime import can fail and the PR's new test exercises that missing-optimizer path directly.

## Solution Assessment

No, not as written. Reusing the existing missing-optimizer detector at the exhausted resize-loop throw is maintainable, but the suggested install commands need to be global/package-path aware to solve the reported operator experience.
