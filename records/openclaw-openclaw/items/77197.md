## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection shows missing `sharp` makes every `resizeToJpeg` attempt throw, after which current main emits the generic resize-loop error; the PR’s mocked regression test exercises that path, though I did not run a live global install.

## Solution Assessment

No, not as written. The detector branch is the narrow maintainable fix, but the install guidance should use a command that actually places `sharp` on the global OpenClaw runtime import path or deliberately change package dependency policy.
