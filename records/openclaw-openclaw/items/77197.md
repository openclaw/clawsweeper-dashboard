## Summary

Keep open: the branch adds the right missing-sharp detector hook, but the revised error and changelog still present `npm install -g openclaw@latest --include=optional` as the global repair even though `sharp` is not a root optional dependency and bundled plugin deps are not installed by postinstall.

## What This Changes

The PR adds a missing-sharp-specific error message in `optimizeImageToJpeg`, a regression test, and a changelog entry for the image optimizer diagnostic.

## Best Possible Solution

Land the diagnostic branch after making the global-install remediation name a command that actually places `sharp` on the global runtime resolution path, with the regression test and changelog asserting that wording.

## Reproduction Assessment

Yes, by source inspection: current main can make every `resizeToJpeg` attempt throw through the existing missing-`sharp` path, then `optimizeImageToJpeg` falls through to the generic optimizer error. ## Solution Assessment

No, not as written. The detector placement is the narrow maintainable fix, but the global-install hint should assert a direct global or package-prefix `sharp` install unless package metadata is changed to make the OpenClaw reinstall command true.
