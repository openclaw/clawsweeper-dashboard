Keep open: the diagnostic hook is in the right place, but the PR still presents `npm install -g openclaw@latest --include=optional` as the global repair even though `sharp` is not declared in the root package dependencies or optionalDependencies and bundled plugin dependencies are not installed by package postinstall.

## What This Changes

The PR adds a missing-`sharp` branch in `optimizeImageToJpeg`, a direct regression test, and a changelog entry for #73148.

## Best Possible Solution

Land the diagnostic-only fix after the global-install text and regression assertion point to a command that installs `sharp` into a resolver-visible global or OpenClaw package-prefix location.

## Reproduction Assessment

Yes. Source inspection shows current main can exhaust every `resizeToJpeg` attempt when `sharp` import fails and then throw the generic resize-loop error; the PR's mocked regression test models that path.

## Solution Assessment

No, not as written. The missing-optimizer detector branch is the narrow maintainable fix, but the global repair copy should not rely on a root optional dependency that the package does not declare.
