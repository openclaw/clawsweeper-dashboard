## Summary

Keep open: the PR targets the right missing-`sharp` resize-loop failure, but the latest diff still recommends local `pnpm add sharp` / `npm install sharp` / `bun add sharp` commands for a failure reported in a global packaged install, so the guidance can leave operators on the same broken runtime path.

## What This Changes

The PR adds a missing-`sharp`-specific `optimizeImageToJpeg` error branch, regression coverage, and an unreleased changelog entry.

## Best Possible Solution

Land the diagnostic improvement after the error text and regression test point global/package installs to the documented reinstall/update or package-prefix remediation, while leaving broader Sharp packaging policy in #73148.

## Reproduction Assessment

Yes. Current main routes image-tool media through `loadWebMedia` and `optimizeImageToJpeg`; when `resizeToJpeg` cannot import `sharp`, the resize loop exhausts and currently throws the generic optimizer failure.

## Solution Assessment

No, not as written. Reusing the existing missing-optimizer detector at the exhausted resize-loop throw is maintainable, but the remediation text must be global/package-install-aware to solve the reported operator experience.
