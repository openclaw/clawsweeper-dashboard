## Summary

Keep open: current main still lacks an alias-scoped route identity, and one Anthropic CLI auth migration path can still collapse colliding `claude-cli/*` and `anthropic/*` allowlist entries. The remaining work is a config/runtime product decision rather than conservative cleanup.

## What This Changes

The issue requests support for using the same Anthropic model through separate CLI and API routes in one agent config, preferably via per-alias `agentRuntime` and auth-profile selection.

## Best Possible Solution

Define one route-identity contract for model aliases or sibling keys, then make doctor/auth migrations preserve colliding CLI/API entries with clear warnings and regression coverage.

## Reproduction Assessment

No, not for the exact `doctor --fix` command without running it. Source inspection does reproduce the destructive Anthropic CLI provider-auth migration path, while current doctor migration tests show partial preservation for legacy keys.

## Solution Assessment

Unclear. Per-alias runtime overrides are plausible, but maintainers need to choose whether route identity belongs in alias metadata, sibling model keys, or a narrower migration collision guard.
