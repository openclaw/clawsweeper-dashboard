## Summary

Keep open: current main still has the destructive migration shape for colliding `claude-cli/*` and `anthropic/*` model entries, and the requested dual-route identity needs a maintainer config/runtime decision rather than cleanup closure.

## What This Changes

Requests a way to configure two aliases for the same Anthropic model with different runtime/auth routes, plus a non-destructive doctor migration when legacy `claude-cli/*` keys collide with canonical `anthropic/*` keys.

## Best Possible Solution

Define one explicit route-identity contract for same-provider/same-model runtime selection and make doctor preserve colliding Claude CLI/API entries with a clear warning or migration path.

## Reproduction Assessment

Yes. Source inspection shows a config with both `claude-cli/<model>` and `anthropic/<model>` entries reaches `rewriteModelEntryMap`, skips overwriting the existing canonical key, deletes the legacy key, and then replaces the default model map.

## Solution Assessment

Unclear. Per-alias runtime overrides are plausible, but maintainers need to choose whether route identity belongs in an aliases array, a sibling-key syntax, or a narrower non-destructive doctor collision guard first.
