## Summary

Keep open: current main and v2026.6.8 support one model-scoped `agentRuntime.id` for a canonical provider/model entry, but they still do not provide a first-class way to expose two selectable aliases for the same Anthropic model with different CLI/API route identities.

## What This Changes

Request a per-alias or equivalent route-specific model configuration so one Anthropic model can be selected through both Claude CLI and direct API routes.

## Best Possible Solution

Define a maintainer-approved route identity model in config, runtime selection, doctor migration, and docs so Anthropic API and `claude-cli` routes can coexist without reviving legacy provider namespaces as steady-state runtime config.

## Reproduction Assessment

Not applicable as a feature request; source inspection confirms current main and v2026.6.8 lack the requested per-alias runtime/auth route shape.

## Solution Assessment

Unclear until maintainers choose the public config contract; the existing model-scoped `agentRuntime.id` handles one route, but dual routes require design across config, migration, picker/status, and auth-profile semantics.
