## Summary

Current main still lacks a per-alias or per-route runtime identity, and the Anthropic CLI migration source confirms the reported collision path can drop the legacy `claude-cli/*` entry. This should stay open for an explicit config/migration design decision rather than conservative cleanup.

## What This Changes

Request per-alias or per-route `agentRuntime` selection so the same Anthropic model can be configured for both Claude CLI and direct API routes without `doctor --fix` collapsing one route.

## Best Possible Solution

Choose a single maintainer-approved route identity contract that preserves colliding legacy Claude CLI/API configurations and makes runtime plus auth selection explicit without overloading existing auth-profile syntax.

## Reproduction Assessment

Yes, from source inspection: a config with both `claude-cli/<model>` and `anthropic/<model>` entries will hit `rewriteModelEntryMap`, skip copying over the existing canonical key, and delete the legacy key. I did not execute doctor because the review is read-only.

## Solution Assessment

Unclear. The requested direction is plausible, but the best fix needs a maintainer decision on whether route identity belongs in a `runtimes` array, a new key syntax, or a narrower doctor collision guard.
