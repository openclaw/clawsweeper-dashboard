## Summary

This should stay open: current main still has the reported destructive migration shape, and the requested dual-route model identity needs an explicit config/runtime product decision rather than cleanup closure.

## What This Changes

The issue requests a way to configure separate API and Claude CLI routes for the same Anthropic model alias, plus a non-destructive doctor migration for colliding legacy `claude-cli/*` and canonical `anthropic/*` entries.

## Best Possible Solution

Choose one route-identity contract that preserves colliding Claude CLI/API configs and makes runtime plus auth selection explicit without overloading the existing single model key.

## Reproduction Assessment

Yes, from source inspection: a config with both `claude-cli/<model>` and `anthropic/<model>` entries reaches `rewriteModelEntryMap`, which skips copying over the existing canonical key and then deletes the legacy key. I did not execute doctor because the review is read-only.

## Solution Assessment

Unclear. Per-alias runtime overrides are plausible, but the best solution needs maintainer agreement on whether route identity belongs in an aliases array, a sibling-key syntax, or a narrower doctor collision guard.
