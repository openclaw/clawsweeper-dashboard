## Summary

Keep open: current main supports a single model-scoped runtime policy for a canonical provider/model entry, but it still does not provide first-class dual selectable aliases or route identities for the same Anthropic model across API and Claude CLI paths. Narrow related fixes solved migration and `/btw` runtime-alias bugs, not this broader config/product request.

## What This Changes

Request per-alias `agentRuntime` or equivalent route identity so one Anthropic model can be selected through both direct API and Claude CLI routes in one agent config.

## Best Possible Solution

Define a maintainer-approved route identity model across config schema, runtime selection, model picker/status, doctor migration, and docs, without reviving legacy provider namespaces as steady-state runtime config.

## Solution Assessment
