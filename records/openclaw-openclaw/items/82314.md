## Summary

Current main still has a source-proven collision path that can remove the legacy Claude CLI route, and the remaining fix needs an explicit model-route/config design rather than conservative cleanup.

## What This Changes

Requests a way to configure the same Anthropic model through separate Claude CLI and direct API routes, preferably via per-alias agentRuntime overrides.

## Best Possible Solution

Define one route identity contract for model selection, runtime, and auth profile selection, then make the Claude CLI migration preserve colliding legacy/API routes with focused docs and regression coverage.

Yes. Source inspection shows a config containing both claude-cli/<model> and anthropic/<model> entries can hit the migration path that skips copying over the existing canonical key and deletes the legacy key; I did not execute doctor because this review is read-only.

## Solution Assessment

Unclear. Per-alias runtime overrides are plausible, but a narrower collision-preserving migration may be the safer immediate fix while maintainers decide the long-term route identity syntax.
