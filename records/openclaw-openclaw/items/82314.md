## Summary

Keep open: current main has model-scoped `agentRuntime` and auth-profile selection improvements, but it still does not provide a first-class alias-scoped route identity for two aliases of the same canonical Anthropic model with different runtimes. The related `/btw` runtime-alias bug was merged separately and does not supersede this config/product request.

## What This Changes

The issue requests per-alias runtime routing so one canonical Anthropic model can expose both a Claude CLI subscription-route alias and a direct API-route alias in one agent config.

## Best Possible Solution

Keep one canonical design discussion open and decide the narrow public config/migration shape that preserves dual Anthropic routes without adding duplicate runtime sources of truth.

## Reproduction Assessment

Yes at source level: current config supports one alias and one runtime policy per configured model key, while the migration/docs keep runtime policy model/provider-scoped. I did not run `doctor --fix` because this review is read-only, but the relevant source and tests show the remaining config-shape gap.

## Solution Assessment

Unclear: per-alias `agentRuntime` is plausible, but not proven as the best shape versus profile-suffixed selection, sibling keys, or a narrower migration preservation rule. Maintainers should choose the public contract before implementation.
