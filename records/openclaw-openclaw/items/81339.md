## Summary

Keep this issue open because current main still trims the WebChat message before optimistic display and the `chat.send` payload, and an open linked fix PR already targets the exact problem with real browser proof.

## What This Changes

The issue requests that Control UI/WebChat preserve leading whitespace in sent code blocks and ASCII diagrams instead of trimming it before `chat.send`.

## Best Possible Solution

Review and land the linked WebChat fix, or a narrow replacement, so blank and command checks use trimmed text while normal user-authored chat text, queued text, optimistic display, and `chat.send` preserve the original message bytes with regression coverage.

## Solution Assessment
