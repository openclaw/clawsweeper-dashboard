## Summary

Keep open: current main still trims WebChat submitted text before optimistic rendering and `chat.send`, and the issue is already paired with an open focused fix PR that uses closing syntax.

## What This Changes

The issue requests that WebChat preserve leading whitespace in submitted code blocks so ASCII and box-drawing diagrams keep their alignment.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81344 if the diff remains correct, then close this issue from the merged fix.

## Reproduction Assessment

Yes. Current main trims the WebChat draft in `handleSendChat()` and trims again in `sendChatMessage()`, so the issue's indented diagram would lose leading spaces before optimistic rendering and the `chat.send` payload.

## Solution Assessment

Yes. The narrow maintainable fix is to keep the original submitted message for normal WebChat sends while using trimmed text only for blank-message guards, command parsing, and command history; the open linked PR appears to follow that boundary.
