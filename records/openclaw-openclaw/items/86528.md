## Summary

Keep this PR open: the central fix is useful and the previous loopback-only concern was narrowed, but the patch still has a blocking state-directory mismatch that can skip the stale-node restart for supported custom state locations.

## What This Changes

The PR adds a `CLIENT_VERSION_MISMATCH` gateway handshake detail, rejects same-install local node-host version mismatches, and makes the node host exit so launchd/systemd can restart it.

## Best Possible Solution

Use the same state-dir resolver as the node-host config path, then cover at least one overridden state-dir case so same-install local-node detection matches real supported installs before merge.

## Reproduction Assessment

Yes, at source level: current main has no version-mismatch restart guard, while the node host reports a version and reconnects through the gateway client path. I did not reproduce the full package-update plus launchd restart cycle locally.

## Solution Assessment

No, not yet: the same-install identity check is the right direction, but it must resolve `node.json` through the same state-dir contract as the node host. After that narrow repair, the approach looks maintainable for the local-node case while leaving the remote skew issue to https://github.com/openclaw/openclaw/issues/83736.
