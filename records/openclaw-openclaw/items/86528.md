## Summary

Keep open for merge review. I found no blocking correctness or security finding in the latest patch, but the change remains compatibility- and availability-sensitive because it deliberately makes same-install local node hosts exit on release-version drift.

## What This Changes

The PR adds a gateway handshake guard and node-host reconnect handling so same-install local node hosts exit for supervisor restart when their release version differs from the gateway, with focused gateway and runner tests.

## Best Possible Solution

Land the narrow local-node restart guard only after current-head CI and maintainer review accept the upgrade behavior, while leaving broader subordinate-node skew tolerance tracked in https://github.com/openclaw/openclaw/issues/83736.

## Solution Assessment

Yes, with maintainer acceptance of the upgrade risk. The PR uses existing version, instanceId, loopback, and reconnect-pause surfaces instead of adding a new config path, while preserving the remote-node skew work as a separate issue.
