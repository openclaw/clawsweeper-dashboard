## Summary

Keep open: this is a focused bug fix with credible log proof, but the current diff still hand-resolves `node.json` instead of using the shared state-dir contract, so supported non-default state directories can miss the stale-node restart guard.

## What This Changes

The PR adds a `CLIENT_VERSION_MISMATCH` gateway handshake detail, rejects same-install local node-host version mismatches, pauses reconnects, exits the node host for supervisor restart, and adds focused gateway/node-host tests.

## Best Possible Solution

Land this after the local node-id lookup uses the same `resolveStateDir`-based path as node-host config and coverage proves a non-default state dir while preserving the remote-node exemption.

## Reproduction Assessment

Yes. The PR body gives a concrete mismatch simulation with before/after logs, and the source path is clear: a long-lived node host reconnects to a newer gateway and should exit on `CLIENT_VERSION_MISMATCH`; I did not execute it during this read-only review.

## Solution Assessment

No, not yet. The recovery design is narrow, but the implementation should reuse the existing node-host state-dir resolver contract before it is the best maintainable fix.
