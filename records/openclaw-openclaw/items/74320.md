## Summary

Keep open: current main still has the trimmed-string indentation guards that can end a `tasks:` block on an unknown indented field, and this PR is the focused fix with targeted regression coverage and sufficient real parser proof.

## What This Changes

This PR changes `parseHeartbeatTasks` to use raw-line indentation checks for task-block termination and adds parser regression tests for unknown indented fields and block boundaries.

## Best Possible Solution

Land the existing narrow raw-line indentation fix with its regression tests after normal exact-head merge gates pass.

## Reproduction Assessment

Yes. Source inspection on current main shows the two guards check already-trimmed strings for leading whitespace, and the PR includes before/after terminal output from the real parser showing later tasks dropped before the fix and retained after it.

## Solution Assessment

Yes. Replacing only the two dead `trimmed` / `nextTrimmed` indentation guards with raw-line checks is the narrowest maintainable fix and matches the adjacent parser behavior from the merged sibling fix.
