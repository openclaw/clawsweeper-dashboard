## Summary

No blocking findings. Current main and the latest release still have the dead trimmed-string indentation guards, while this PR narrowly switches those guards to raw-line checks and adds focused parser regressions, so the item remains useful and should stay open for normal maintainer landing.

## What This Changes

This PR changes `parseHeartbeatTasks` to use raw source-line indentation checks for unknown fields and adds regression tests for indented unknown fields plus task-block boundary cases.

## Best Possible Solution

Land the narrow raw-line indentation fix with its regression tests so unknown indented HEARTBEAT.md task metadata no longer prevents later scheduled tasks from being parsed.

## Reproduction Assessment

Yes from source inspection and supplied live parser output: a `HEARTBEAT.md` `tasks:` block whose first task has an unknown indented field makes current main stop parsing later tasks. I did not execute a local repro in this read-only review.

## Solution Assessment

Yes. Replacing only the two dead trimmed-string indentation guards with raw-line checks is the narrowest maintainable fix, matches the sibling parser invariant, and preserves top-level block termination.
