## Summary

Keep open, but this PR is not mergeable as-is: current main and the public gateway docs keep control-plane writes actor-scoped by `deviceId+clientIp`, while the PR makes that privileged write budget renewable per connection. The existing ClawSweeper review already raised the same proof/security concern, and this read-only pass agrees.

## What This Changes

The PR changes gateway control-plane write rate-limit keys to include `connId` and adds a test expecting separate write budgets for concurrent gateway sessions.

## Best Possible Solution

Keep the actor-scoped `deviceId+clientIp` throttle as the baseline; add per-connection fairness only as a documented additional layer after explicit security/product approval.

## Reproduction Assessment

Yes for the PR-introduced regression by source inspection: after exhausting a write budget under one `connId`, changing `connId` creates a distinct key and a fresh bucket. No current-main bug is reproduced because current main intentionally keeps the actor-scoped key when identity is present.

## Solution Assessment

No. Replacing the actor key with a connection key is not the narrowest maintainable solution; preserve the existing actor-level guard and layer connection fairness separately only if maintainers choose that policy.
