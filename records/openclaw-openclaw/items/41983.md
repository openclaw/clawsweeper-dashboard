## Summary

Keep open for maintainer handling: the PR makes a security-sensitive gateway rate-limit policy change, lacks real behavior proof, and has a blocking correctness/security finding against the shipped actor-scoped throttle contract.

## What This Changes

The PR changes gateway control-plane write rate-limit keys to include `connId` and adds a test that gives concurrent sessions from the same device/IP separate write budgets.

## Best Possible Solution

Keep the actor-scoped `deviceId+clientIp` write throttle unless maintainers approve a separate connection-fairness mechanism with docs, tests, security review, and real gateway proof.

## Reproduction Assessment

Yes for the PR-introduced behavior by source inspection: the diff appends `connId` to every non-empty key and the added test expects `conn-b` to bypass `conn-a`'s exhausted budget. No current-main bug was reproduced because current main and shipped docs keep the `deviceId+clientIp` contract.

## Solution Assessment

No. Changing the primary limiter key to per-connection budgets weakens privileged write throttling; connection fairness should be a separately designed and documented layer if maintainers want it.
