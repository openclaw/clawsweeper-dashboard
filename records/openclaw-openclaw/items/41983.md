## Summary

Keep open for maintainer review: the PR is a focused gateway change, but it changes a documented security-relevant control-plane throttle from actor-scoped to per-connection without a maintainer-approved policy or real behavior proof.

## What This Changes

The PR changes gateway control-plane write rate-limit keys to include `connId` for every connection and adds tests expecting separate budgets for same-device/IP sessions.

## Best Possible Solution

Keep the actor-scoped limiter as the default; if higher throughput is needed, handle it through an explicitly reviewed and documented policy, likely via the configurable path in https://github.com/openclaw/openclaw/pull/58735, with focused tests and real gateway proof.

## Reproduction Assessment

## Solution Assessment

No. The patch changes a security-relevant policy boundary rather than fixing a defined bug; the safer direction is to preserve the actor-scoped default and route any tuning through explicit maintainer-approved policy and proof.
