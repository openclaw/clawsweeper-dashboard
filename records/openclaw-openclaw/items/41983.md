## Summary

Keep this PR open but do not merge it as-is: the diff changes a shipped actor-scoped gateway write throttle into a connection-scoped budget, which weakens the privileged write guard and lacks real behavior proof. Current main and the latest release still document and implement the `deviceId+clientIp` boundary, so this needs maintainer product/security review rather than cleanup closure.

## What This Changes

The PR changes gateway control-plane write rate-limit keys to include `connId` whenever present and adds a test expecting separate budgets for concurrent gateway sessions.

## Best Possible Solution

Keep `deviceId+clientIp` as the baseline privileged-write throttle; if multi-session fairness is desired, design it as a documented second layer or configuration policy after gateway/security review with real proof.

## Reproduction Assessment

Yes for the PR-introduced behavior by source inspection: changing `connId` changes the rate-limit key for the same device/IP, and the added test encodes a fresh budget for the second connection. No current-main bug was reproduced because current main and release v2026.5.12 match the documented `deviceId+clientIp` contract.

## Solution Assessment

No. Replacing the actor-scoped key with a connection-scoped key weakens a privileged write throttle; the safer path is to keep the existing guard and add any connection fairness only as a reviewed, documented policy layer.
