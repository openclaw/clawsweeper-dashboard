## Summary

Keep open: the PR changes a documented actor-scoped gateway control-plane write throttle into a per-connection throttle, which is a security/product policy change and still lacks contributor real behavior proof.

## What This Changes

The PR changes gateway control-plane write rate-limit keys to include `connId` whenever present and adds a test showing a second concurrent session gets a fresh write budget.

## Best Possible Solution

Keep the actor-scoped `deviceId+clientIp` throttle as the baseline, and add per-connection fairness only as an additional documented layer after maintainer security approval.

## Reproduction Assessment

Yes for the PR-introduced behavior by source inspection: changing `connId` changes the key and creates a fresh bucket for the same device/IP. No current-main bug is reproduced because current main and docs intentionally keep the actor-scoped key when identity exists.

## Solution Assessment

No. Replacing the actor key with a connection key is not the narrowest maintainable fix; per-connection fairness should be additive and explicitly approved as a security/product policy change.

## Review Findings

Overall confidence: 0.9

Full review comments:
