## Summary

Keep open: current main does not implement the PR's per-connection budget isolation, but the patch conflicts with the documented actor-scoped gateway write throttle and needs maintainer policy/security judgment before merge.

## What This Changes

The PR adds `connId` to gateway control-plane rate-limit keys and tests that two concurrent sessions from the same client get independent `config.patch` budgets.

## Best Possible Solution

Keep the actor-scoped `deviceId+clientIp` write budget, and add per-connection fairness only as an additional limiter if maintainers explicitly choose that gateway policy.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: under the PR key, repeated `config.patch` calls from the same device/IP can get a fresh budget by reconnecting with a new `connId`.

## Solution Assessment

No. Replacing the actor key with a connection key is not the narrowest maintainable fix; preserving the `deviceId+clientIp` guard and layering any per-connection fairness beneath it would be safer.

## Review Findings

## Security Review

Status: needs_attention
