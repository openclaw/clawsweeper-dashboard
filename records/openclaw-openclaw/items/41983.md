## Summary

This PR should stay open because it proposes a real gateway behavior change, but the current patch weakens the documented actor-scoped control-plane write throttle and still lacks contributor-provided real behavior proof.

## What This Changes

The PR changes gateway control-plane write rate-limit keys to include `connId` whenever present and adds a test expecting concurrent sessions from the same device/IP to have separate write budgets.

## Best Possible Solution

Keep the existing `deviceId+clientIp` control-plane write throttle unless maintainers explicitly design a second, documented connection-fairness layer with security review and real gateway proof.

## Reproduction Assessment

Yes for the PR-introduced behavior by source inspection: the changed key gives the same device/IP a separate write bucket for each `connId`. I did not reproduce a current-main bug because current main and the latest release match the documented `deviceId+clientIp` contract.

## Solution Assessment

No. Replacing the actor-scoped key with a connection-scoped key is not the narrowest maintainable fix because it weakens a privileged write throttle; connection fairness should be layered separately only after maintainer approval.
