## Summary

Keep open for human review: the branch changes a documented actor-scoped gateway write throttle into a per-connection throttle, which creates a reconnect bypass for privileged control-plane writes and lacks real behavior proof or maintainer product/security approval.

## What This Changes

The PR changes gateway control-plane write rate-limit keys to include `connId` and adds a test expecting separate write budgets for concurrent sessions from the same client identity.

## Best Possible Solution

Keep the `deviceId+clientIp` write throttle as the baseline guard; add any per-connection fairness as a separate documented layer only after explicit product/security approval and real gateway proof.

## Reproduction Assessment

Yes for the PR-introduced regression by source inspection: changing `connId` changes the rate-limit key and gives a fresh bucket while the actor identity is unchanged. No current-main bug was reproduced because current main intentionally keeps actor-scoped keys when identity is present.

## Solution Assessment

No. Replacing the actor key with a connection key is not the narrowest maintainable solution; preserve the existing actor-level guard and layer connection fairness separately only if maintainers choose that policy.
