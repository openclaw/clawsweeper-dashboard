## Summary

Keep open for maintainer review, but the current patch should not merge: it changes the documented actor-scoped control-plane write throttle into per-connection budgets, weakening a gateway security guardrail and still lacking real behavior proof.

## What This Changes

The PR changes the gateway control-plane write rate-limit key to include `connId` whenever present and adds coverage expecting same-device/IP sessions to have separate write budgets.

## Best Possible Solution

Keep the actor-scoped `deviceId+clientIp` throttle; if maintainers want per-connection fairness, design a documented policy with security review, bounded cleanup, tests, and real gateway proof.

## Reproduction Assessment

Yes for the PR-introduced behavior by source inspection: the branch appends `connId` to non-empty keys and adds a same-device/IP, different-connection budget test. No current-main bug is reproduced because current main follows the documented actor-scoped contract.

## Solution Assessment

No. Changing the primary limiter key to per-connection budgets weakens a shipped control-plane guardrail; a separate documented fairness policy would be safer if maintainers want this behavior.
