## Summary

Keep open for explicit maintainer handling: the patch is not mergeable as-is because it changes a documented actor-scoped privileged write throttle into per-connection budgets, weakens the guardrail, and still lacks contributor real behavior proof.

## What This Changes

The PR changes gateway control-plane write rate-limit keys to include `connId` and adds a same-device/IP, different-connection budget test.

## Best Possible Solution

Preserve the actor-scoped throttle; if connection fairness is desired, design it as a separately documented gateway policy with security review, tests, and real gateway proof.

## Reproduction Assessment

Yes for the PR-introduced behavior by source inspection: the diff appends `connId` to non-empty keys and adds a same-device/IP, different-connection budget test. No current-main bug is reproduced because current main follows the documented actor-scoped contract.

## Solution Assessment

No. Changing the primary limiter key to per-connection budgets changes a shipped security policy; a separate fairness layer would be the safer maintainable direction if maintainers want this behavior.
