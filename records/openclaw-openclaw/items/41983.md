## Summary

Keep open for maintainer judgment: the PR conflicts with the documented and shipped actor-scoped control-plane write throttle, weakens a privileged write limiter, and still has no real behavior proof.

## What This Changes

The PR changes gateway control-plane rate-limit keys to include `connId` whenever present and adds a test expecting concurrent sessions from the same device/IP to receive separate write budgets.

## Best Possible Solution

Keep the actor-scoped `deviceId+clientIp` write throttle unless maintainers approve a separate connection-fairness mechanism with docs, tests, security review, and real gateway proof.

## Reproduction Assessment

Yes for the PR-introduced behavior by source inspection: the diff appends `connId` to every non-empty key and the added test expects `conn-b` to bypass `conn-a`'s exhausted budget. I did not establish a current-main bug because current main matches the documented `deviceId+clientIp` contract.

## Solution Assessment

No. Changing the primary limiter key to per-connection budgets weakens privileged write throttling; connection fairness should be a separately designed and documented layer if maintainers want it.
