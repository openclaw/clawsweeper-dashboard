## Summary

Keep open: this PR still proposes a gateway throttling policy change that is not implemented on current main, conflicts with the documented actor-scoped control-plane write budget, lacks real behavior proof, and needs maintainer product/security judgment before it can be mergeable.

## What This Changes

The PR changes gateway control-plane write rate-limit keys to include `connId` whenever present and adds a test expecting separate write budgets for concurrent sessions.

## Best Possible Solution

Keep `deviceId+clientIp` as the baseline privileged-write throttle; add any per-connection fairness as a separate documented layer only after maintainer product/security approval and real gateway proof.

## Reproduction Assessment

Yes for the PR-introduced behavior by source inspection: adding `connId` to the key gives an unchanged device/IP a fresh bucket per connection. No current-main bug was reproduced because current main matches the documented `deviceId+clientIp` contract.

## Solution Assessment

No. Replacing the actor-scoped key with a connection-scoped key is not the narrowest maintainable solution; keep the existing guard and layer connection fairness separately only if maintainers choose that policy.
