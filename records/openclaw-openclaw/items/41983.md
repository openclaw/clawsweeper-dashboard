## Summary

Keep open: this PR changes the documented actor-scoped gateway control-plane write throttle into a per-connection budget, which needs maintainer product/security approval and real gateway proof before it can be considered mergeable.

## What This Changes

The PR changes gateway control-plane write rate-limit keys to include `connId` whenever present and adds a test expecting separate write budgets for concurrent sessions.

## Best Possible Solution

Keep `deviceId+clientIp` as the baseline privileged-write throttle; if multi-session fairness is desired, design it as a documented second layer after gateway/security review with real proof.

## Reproduction Assessment

Yes for the PR-introduced behavior by source inspection: adding `connId` to the normal key gives the same device/IP a fresh bucket per connection. No current-main bug was reproduced because main and release v2026.5.12 match the documented `deviceId+clientIp` contract.

## Solution Assessment

No. Replacing the actor-scoped key with a connection-scoped key weakens the privileged write throttle; keep the existing guard and layer connection fairness separately only if maintainers choose that policy.
