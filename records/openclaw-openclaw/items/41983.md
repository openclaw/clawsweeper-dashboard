## Summary

Keep open for maintainer handling: the branch changes a shipped gateway write-throttle policy from actor-scoped to connection-scoped, which weakens the current guardrail and lacks real behavior proof or product/security approval.

## What This Changes

The branch appends `connId` to the gateway control-plane write rate-limit key when present and adds a test showing same-device/IP sessions get separate budgets.

## Best Possible Solution

Preserve the actor-scoped throttle; if connection fairness is desired, design it as a separately documented gateway policy or config surface with security review, tests, and real gateway proof.

## Reproduction Assessment

Yes for the PR-introduced behavior by source inspection: the diff appends `connId` to non-empty keys and adds a same-device/IP, different-connection budget test. No current-main bug is reproduced because current main follows the documented actor-scoped contract.

## Solution Assessment

No. Changing the primary limiter key to per-connection budgets changes a shipped security policy; a separate fairness layer or explicit configurable policy would be safer if maintainers want this behavior.
