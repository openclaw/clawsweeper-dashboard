## Summary

Keep open, but do not merge as-is: current main and public docs define actor-scoped `deviceId+clientIp` throttling, while this PR changes privileged gateway writes to per-connection budgets without real behavior proof and needs maintainer product/security approval.

## What This Changes

The PR changes gateway control-plane write rate-limit keys to include `connId` and adds a test expecting separate budgets for concurrent sessions from the same client identity.

## Best Possible Solution

Keep `deviceId+clientIp` as the baseline write throttle; add per-connection fairness only as a separate documented layer after explicit product/security approval.

## Reproduction Assessment

Yes for the PR-introduced regression by source inspection: changing `connId` changes the rate-limit key and gives a fresh bucket while the actor identity is unchanged. No current-main bug was reproduced because current main intentionally keeps actor-scoped keys when identity is present.

## Solution Assessment

No. Replacing the actor key with a connection key is not the narrowest maintainable solution; preserve the existing actor-level guard and layer connection fairness separately only if maintainers choose that policy.
