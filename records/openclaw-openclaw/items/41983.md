## Summary

Keep this PR open for maintainer/security review: it changes a documented control-plane hardening contract from a shared `deviceId+clientIp` budget to per-connection buckets, which would let reconnecting or parallel sockets bypass the aggregate write limit. https://github.com/openclaw/openclaw/pull/41983

## What This Changes

The PR changes gateway control-plane write rate-limit keys to include `connId` for every connection and adds a regression test showing two concurrent sessions get separate write budgets.

## Best Possible Solution

Keep the documented aggregate `deviceId+clientIp` control-plane write cap unless maintainers approve a connection-aware design that also preserves an aggregate guard and updates docs/tests.

## Reproduction Assessment

Yes for the PR regression. Source inspection shows current docs and implementation define a shared per-`deviceId+clientIp` bucket, while the PR diff gives each `connId` its own bucket.

## Solution Assessment

No. The proposed implementation is not the narrowest maintainable solution because it weakens a documented security limiter; a safer direction is a maintainer-approved hierarchical limit or keeping `connId` as the unknown-identity fallback.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve the aggregate device/IP budget:** `src/gateway/control-plane-rate-limit.ts:29-31`
  - body: `config.apply`, `config.patch`, and `update.run` are documented as 3 requests per 60s per `deviceId+clientIp`. Adding `connId` to every non-empty key lets the same device/IP open or reconnect sockets to get fresh buckets, bypassing the shared control-plane write limiter and restart hardening. Keep `connId` as the unknown-identity fallback or add an aggregate cap before allowing per-connection isolation.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff weakens the documented control-plane write rate limit by making the bucket per reconnectable WebSocket connection.

Concerns:

- **[medium] Connection-scoped buckets bypass aggregate write limits:** `src/gateway/control-plane-rate-limit.ts:29`
  - body: The current limiter is a security hardening measure for config/update write RPCs. With `connId` in every key, the same authenticated device/IP can create multiple sockets and multiply the allowed write/restart attempts per minute.
