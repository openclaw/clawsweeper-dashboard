## Summary

Keep open: the patch changes a documented actor-scoped gateway write throttle into a per-connection budget, lacks real behavior proof, and needs maintainer product/security judgment rather than cleanup closure.

## What This Changes

The branch appends `connId` to the gateway control-plane write rate-limit key and adds coverage expecting same-device/IP gateway sessions to receive separate write budgets.

## Best Possible Solution

Preserve the actor-scoped throttle unless maintainers explicitly approve a separate documented fairness policy with security review, tests, and real gateway proof.

## Reproduction Assessment

Yes for the PR-introduced behavior by source inspection: the branch appends `connId` to non-empty keys and adds a same-device/IP, different-connection budget test. No current-main bug is reproduced because current main follows the documented actor-scoped contract.

## Solution Assessment

No. Changing the primary limiter key to per-connection budgets weakens a shipped security guardrail; a separate fairness layer or explicitly documented policy would be safer if maintainers want this behavior.
