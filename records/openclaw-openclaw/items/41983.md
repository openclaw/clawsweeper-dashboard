## Summary

Keep open for human review, but do not merge as-is: current main and docs define an actor-scoped privileged write budget, while this PR grants a fresh budget per connection and still lacks real behavior proof.

## What This Changes

The PR changes the gateway control-plane rate-limit key to include `connId` whenever present and adds a test expecting same-device/IP sessions to have independent write budgets.

## Best Possible Solution

Keep the shipped actor-scoped `deviceId+clientIp` throttle; if per-connection fairness is desired, design it as an explicit documented policy with security review, bounded cleanup, tests, and real gateway proof.

## Reproduction Assessment

Yes for the PR-introduced behavior by source inspection: the diff appends `connId` to non-empty limiter keys and adds a same-device/IP separate-budget test. No current-main bug is reproduced because current main follows the documented actor-scoped contract.

## Solution Assessment

No. Changing the primary limiter key to per-connection budgets weakens a shipped control-plane guardrail; a separate documented fairness policy would be safer if maintainers want this behavior.
