## Summary

Keep open: current main still documents and enforces the privileged control-plane write budget per actor identity, while this PR changes that policy to a renewable per-connection budget and has no contributor proof.

## What This Changes

The PR changes gateway control-plane write rate-limit keys to include `connId` whenever present and adds a test showing separate budgets for two concurrent sessions.

## Best Possible Solution

Keep the actor-scoped `deviceId+clientIp` throttle as the baseline, and add per-connection fairness only as an additional documented layer after maintainer security approval.

## Reproduction Assessment

Yes for the PR-introduced regression by source inspection: after exhausting a write budget under one `connId`, changing `connId` creates a distinct key and a fresh bucket. No current-main bug is reproduced because current main intentionally keeps the device/IP key when identity is present.

## Solution Assessment

No. Replacing the actor key with a connection key is not the narrowest maintainable solution; preserve the existing actor-level guard and layer connection fairness separately only if maintainers choose that policy.
