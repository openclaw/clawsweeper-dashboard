## Summary

Keep open: the branch weakens a documented actor-scoped gateway write throttle by making privileged write budgets renewable per connection, and the PR has neither maintainer policy approval nor after-fix real behavior proof.

## What This Changes

The PR changes the gateway control-plane write rate-limit key to include `connId` whenever available and adds a test for separate write budgets across concurrent sessions.

## Best Possible Solution

Keep the actor-scoped `deviceId+clientIp` throttle as the baseline, and add per-connection fairness only as an additional documented layer after maintainer approval.

## Reproduction Assessment

Yes for the PR-introduced regression by source inspection: after exhausting `config.patch` under one `connId`, changing `connId` produces a distinct key and a fresh bucket. No current-main bug was reproduced because current main intentionally keeps the device/IP key when identity is present.

## Solution Assessment

No. Replacing the actor key with a connection key is not the narrowest maintainable solution; preserve the existing actor-level guard and layer a second connection-level limiter only if maintainers choose that policy.
