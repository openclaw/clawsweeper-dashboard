## Summary

Keep open: current main still uses the documented device/IP actor scope, while this PR changes the privileged gateway write throttle to a per-connection budget and needs maintainer security/product agreement before merge. The prior memory-growth concern is mitigated on current main by bucket pruning and a hard cap, but the identity-boundary concern remains.

## What This Changes

The PR appends `connId` to gateway control-plane write rate-limit keys whenever it is present and adds a test showing same-device/IP sessions get separate budgets.

## Best Possible Solution

Keep the actor-scoped `deviceId+clientIp` budget, and add per-connection fairness only as an additional documented layer if maintainers explicitly approve that policy.

## Reproduction Assessment

Yes. Source inspection of the PR diff gives a high-confidence path: same device/IP with a different `connId` resolves to a different key, so reconnecting refreshes the 3-per-60s write budget.

## Solution Assessment

No. Replacing the actor budget with a connection budget is not the narrowest maintainable solution; preserving the actor cap and layering any connection fairness separately is safer.
