## Summary

Keep this PR open for maintainer review: current main and the shipped changelog define the control-plane write throttle as device/IP scoped, while this patch makes the budget connection-scoped and weakens that actor-level cap.

## What This Changes

The PR changes gateway control-plane write rate-limit keys to include `connId` and adds a test showing two same-device/IP sessions get separate write budgets.

## Best Possible Solution

Keep the actor-scoped write budget, and add per-connection fairness only as a layered design if maintainers explicitly want that policy.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: under the PR key, `config.patch` calls from the same device/IP but different `connId` values use different buckets, so reconnecting refreshes the 3-per-60s write budget.

## Solution Assessment

No. Replacing the actor key with a connection key is not the narrowest maintainable fix; a safer direction would preserve the device/IP cap and add any per-connection fairness as a second layer after maintainer approval.
