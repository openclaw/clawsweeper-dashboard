## Summary

Keep open for maintainer review: the PR is not merge-ready because it turns a documented actor-scoped gateway write throttle into per-connection budgets without proof or a maintainer-approved security rationale.

## What This Changes

The branch changes gateway control-plane write rate-limit keys to include `connId` whenever present and adds a test expecting separate budgets for same-device/IP sessions.

## Best Possible Solution

Keep the actor-scoped limiter as the default; if per-connection fairness or higher throughput is needed, handle it through an explicitly reviewed, documented policy with focused tests and real gateway proof.

## Reproduction Assessment

Yes for the PR-introduced behavior by source inspection: the diff appends `connId` to every non-empty limiter key and adds a same-device/IP separate-budget test. No current-main bug is reproduced because main and docs define the actor-scoped key.

## Solution Assessment

No. The patch changes a security-relevant policy boundary rather than fixing a defined bug; a safer direction is to preserve the actor-scoped default and route any tuning through an explicit policy/config discussion such as https://github.com/openclaw/openclaw/pull/58735.
