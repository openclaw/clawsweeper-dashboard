## Summary

Keep open: current `main` still lacks the requested Discord opt-in policy, and source inspection shows unmentioned audio in a `requireMention` guild channel can be transcribed only for mention detection, then recorded as placeholder history when the mention gate drops it.

## What This Changes

Request a per-Discord guild or channel `preflightTranscribe` option that silently transcribes unmentioned audio so later mentioned turns can use transcript-backed history.

## Best Possible Solution

Add a Discord-owned opt-in policy, defaulting off, that runs after sender authorization, writes transcript-backed pending history without dispatching the agent, and documents privacy and cost tradeoffs.

## Reproduction Assessment

Yes, by source inspection. An authorized attachment-only Discord guild message in a `requireMention` channel can transcribe for mention detection, but if no mention is detected the handler records the prebuilt placeholder history entry and returns before transcript-backed context exists.

## Solution Assessment

Unclear for immediate implementation. The requested option is plausible, but the maintainable fix needs an approved Discord config contract and silent transcript storage policy first.
