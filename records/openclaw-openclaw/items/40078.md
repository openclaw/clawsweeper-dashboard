## Summary

Keep open: current main and latest release still only use Discord audio preflight to detect spoken mentions, then store unmentioned guild audio as placeholder-backed history instead of transcript-backed history. The requested Discord feature gap remains, but it needs maintainer approval for privacy, STT cost, config scope, and transcript persistence semantics before implementation.

## What This Changes

The issue requests a default-off Discord option to silently transcribe audio in mention-gated channels and store transcript-backed history without triggering an agent reply.

## Best Possible Solution

Add a Discord-owned, default-off opt-in that runs only after sender authorization, writes transcript-backed skipped history without agent dispatch, and documents privacy, provider-cost, history, and echo behavior.

## Reproduction Assessment

Yes, at source level. An authorized audio-only Discord guild message in a requireMention channel can be transcribed for mention detection, but when no mention is found the handler records placeholder-backed pending history and returns before transcript-backed context is persisted.

## Solution Assessment

Unclear for immediate implementation. A default-off option is plausible, but maintainers need to approve the Discord config contract and silent transcript storage policy first.
