## Summary

Keep open: current main and the latest release still lack the requested Discord opt-in, and the existing path only uses preflight transcription to detect spoken mentions before dropping unmentioned requireMention audio into placeholder-backed history. This is a valid feature request, but silent provider transcription and persistence need maintainer approval on privacy, cost, and config semantics before implementation.

## What This Changes

The issue requests a default-off Discord guild/channel option to transcribe unmentioned audio in requireMention channels, store the transcript in history, and suppress any agent reply until a later mention.

## Best Possible Solution

Add a Discord-owned, default-off opt-in that runs only after sender authorization, stores transcript-backed pending history without dispatching the agent, and documents the privacy/cost tradeoffs; if maintainers prefer the plugin route, document the supported Listen-Only setup as the alternative instead of adding a core key.

## Reproduction Assessment

Yes, by source inspection. An authorized audio-only Discord guild message in a requireMention channel can be preflight-transcribed for mention detection, but if no mention is found the handler records placeholder-style dropped history and returns before transcript-backed context exists.

## Solution Assessment

Unclear for immediate implementation. The requested default-off option is plausible, but the maintainable fix needs an approved Discord config contract and silent transcript storage policy first; a documented Listen-Only plugin workflow may be the product alternative.
