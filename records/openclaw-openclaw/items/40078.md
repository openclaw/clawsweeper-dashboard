## Summary

Keep open: current main and v2026.5.19 still lack the requested Discord opt-in, and the existing Discord preflight path only uses transcription for DM or mention-detection gating before recording unmentioned guild audio as placeholder-backed dropped history. The request is valid source-reproducible feature work, but silent STT provider calls and transcript persistence need maintainer approval for privacy, cost, and config semantics.

## What This Changes

Request adding a Discord guild/channel `preflightTranscribe` option that silently transcribes unmentioned audio in `requireMention` channels and stores transcripts for later mentioned turns without triggering an agent reply.

## Best Possible Solution

Add a Discord-owned, default-off opt-in that runs only after sender authorization, stores transcript-backed pending history without dispatching the agent, and documents privacy/cost tradeoffs and `echoTranscript` composition.

## Reproduction Assessment

Yes, by source inspection. An authorized audio-only Discord guild message in a `requireMention` channel can be preflight-transcribed for mention detection, but if no mention is found the handler records placeholder-style dropped history and returns before transcript-backed context exists.

## Solution Assessment

Unclear for immediate implementation. The requested default-off option is plausible, but the maintainable fix needs an approved Discord config contract and silent transcript storage policy first.
