## Summary

Keep open: current main and v2026.6.10 still lack the requested default-off silent Discord transcription mode for unmentioned audio in requireMention channels, and adding it needs maintainer product/security approval.

## What This Changes

The issue requests a Discord guild/channel option to silently transcribe unmentioned audio in requireMention channels so later mentioned turns can use the transcript from history.

## Best Possible Solution

Define a Discord-owned, default-off silent transcription policy with privacy, retention, cost, inheritance, schema, docs, and focused requireMention history tests before implementation.

## Reproduction Assessment

Yes at source level: current Discord preflight can transcribe audio for mention detection, but if no mention is found it records pending history from placeholder text and returns before transcript-backed context is built. I did not run a live Discord plus STT reproduction in this read-only review.

## Solution Assessment

No, not as a bare boolean alone. The maintainable solution needs a default-off Discord policy with consent, retention, cost, inheritance, schema, docs, and tests rather than only wiring transcription before the skip return.
