## Summary

Keep open: current main and v2026.5.22 still only use Discord audio preflight for mention detection and persist skipped no-mention guild audio as placeholder history, so the requested silent transcript-backed history behavior remains missing and needs maintainer product approval.

## What This Changes

Request a Discord guild/channel `preflightTranscribe` opt-in that transcribes unmentioned audio in `requireMention` channels, stores transcript-backed history, and suppresses replies.

## Best Possible Solution

Add a Discord-owned, default-off opt-in that runs only after sender authorization, writes transcript-backed pending history without agent dispatch, and documents privacy, provider-cost, and `echoTranscript` behavior.

## Reproduction Assessment

Yes, at source level. An authorized audio-only Discord guild message in a `requireMention` channel can be transcribed for mention detection, but when no mention is found the handler records placeholder-backed dropped history and returns before transcript-backed context is persisted.

## Solution Assessment

Unclear for immediate implementation. The default-off option is plausible, but maintainers need to approve the Discord config contract and silent transcript storage policy first.
