## Summary

Keep open: current main still lacks a Discord opt-in policy that stores transcript-backed history for unmentioned audio in `requireMention` guild channels. The request is well-scoped and source-reproducible, but it is a feature/config decision because it changes when OpenClaw spends transcription tokens and stores private audio-derived text.

## What This Changes

Add a per-guild or per-channel Discord `preflightTranscribe` option that silently transcribes unmentioned audio in mention-gated channels for later history context.

## Best Possible Solution

Add a Discord-owned opt-in policy, defaulting off, that runs after sender authorization, writes transcript-backed pending history without dispatching the agent, and documents the privacy/cost tradeoff.

## Reproduction Assessment

Yes, by source inspection. An authorized audio-only Discord guild message in a `requireMention` channel can be transcribed for mention detection, but if no mention is found the handler records placeholder-based pending history and returns before transcript-backed context is built.

## Solution Assessment

Unclear for immediate implementation. The requested per-channel/guild option is plausible, but the maintainable fix needs an approved Discord config contract and silent transcript storage policy first.
