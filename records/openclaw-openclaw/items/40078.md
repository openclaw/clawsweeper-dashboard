## Summary

Keep open: current main still lacks the requested Discord opt-in policy, and source inspection shows unmentioned audio in a requireMention guild channel can be transcribed for mention detection but then stored as placeholder-style pending history when the mention gate drops it.

## What This Changes

The issue requests a per-guild or per-channel Discord option to silently transcribe unmentioned audio in requireMention channels and preserve the transcript for later mentioned turns.

## Best Possible Solution

Add a Discord-owned opt-in policy, defaulting off, that runs after sender authorization, stores transcript-backed pending history without dispatching the agent, and documents privacy and cost tradeoffs.

## Reproduction Assessment

Yes, by source inspection. An authorized Discord guild audio message can reach preflight transcription for mention detection, but if no mention is found the handler records a prebuilt placeholder-style history entry and returns before transcript-backed context exists.

## Solution Assessment

Unclear for immediate implementation. The requested option is plausible, but the maintainable fix needs an approved Discord config contract and silent transcript storage policy first.
