## Summary

Keep open: current main and the latest release still lack a Discord opt-in policy that persists transcript-backed history for unmentioned audio in mention-gated guild channels, so this remains a real feature/product decision rather than cleanup.

## What This Changes

Add a default-off Discord guild/channel `preflightTranscribe` option that silently transcribes unmentioned audio in mention-gated channels and stores transcripts in history for later mentions.

## Best Possible Solution

Add a Discord-owned opt-in policy, defaulting off, that runs after sender authorization, writes transcript-backed pending history without dispatching the agent, and documents the privacy and cost tradeoff.

## Reproduction Assessment

Yes, by source inspection. An authorized attachment-only Discord guild message in a `requireMention` channel can be transcribed for mention detection, but if no mention is found the handler records placeholder-based pending history and returns before transcript-backed context is built.

## Solution Assessment

Unclear for immediate implementation. The requested option is plausible, but the maintainable fix needs an approved Discord config contract and silent transcript storage policy first.
