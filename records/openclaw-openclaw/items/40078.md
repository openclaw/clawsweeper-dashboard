## Summary

Keep open. Current main still lacks a Discord guild/channel silent preflight transcription setting, and the existing preflight path uses audio transcripts only for mention detection before recording the original `<media:audio>` history entry when mention gating skips the message; related #30034 does not supersede this history-ingestion feature.

## What This Changes

Request adding a default-off Discord guild/channel `preflightTranscribe` option that silently transcribes unmentioned audio in `requireMention` channels and stores the transcript in history without dispatching an agent reply.

## Best Possible Solution

Add a Discord-owned, backward-compatible default-off guild setting with channel override that transcribes authorized audio-only messages into pending history while keeping agent replies mention-gated, with config schema/types, docs, changelog, and regression coverage.

## Reproduction Assessment

Yes, at source level. In a `requireMention` Discord guild channel, an unmentioned audio-only message can be transcribed only for mention detection, then the skip path records the original placeholder history entry and returns without preserving the transcript.

## Solution Assessment

Unclear until maintainers accept the new behavior. If approved, the requested default-off Discord config is the narrowest maintainable direction because it separates transcript ingestion from reply dispatch without changing existing defaults.
