## Summary

Keep open: current main still uses Discord audio preflight only to detect spoken mentions, then records unmentioned audio through transcript-free history. The feature fits channel priorities, but silent processing and retention need an explicit maintainer product and security decision.

## What This Changes

Add opt-in silent transcription of unmentioned Discord audio so later mention-gated turns can use transcript-backed history.

## Best Possible Solution

Add a Discord-plugin-owned, default-off policy only after defining authorization order, transcript retention and expiry, later-history exposure, provider-cost behavior, and operator visibility; keep member and channel access checks ahead of all transcription.

## Reproduction Assessment

Yes at source level: current main transcribes audio for spoken-mention detection but records a skipped message from its original body without the transcript. No live Discord environment was exercised during this read-only review.

## Solution Assessment

No, not as fully specified: the opt-in behavior is plausible, but a boolean alone does not define the required authorization, retention, cost, and observability contract.
