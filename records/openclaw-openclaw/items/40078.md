## Summary

Keep open: current main and v2026.5.20 still lack the requested Discord opt-in, and the source path still records unmentioned `requireMention` guild audio as dropped placeholder history before transcript-backed context can be persisted. This is valid feature work, but silent STT calls, transcript storage, privacy, cost, and config semantics need maintainer product review first.

## What This Changes

Add a default-off Discord guild/channel option that silently transcribes unmentioned audio in `requireMention` channels and saves the transcript for later mentioned turns without triggering a bot reply.

## Best Possible Solution

Add a Discord-owned, default-off opt-in that runs only after sender authorization, stores transcript-backed pending history without dispatching the agent, and documents privacy/cost tradeoffs plus `echoTranscript` composition.

## Reproduction Assessment

Yes, at source level. An authorized audio-only Discord guild message in a `requireMention` channel can be transcribed for mention detection, but when no mention is found the handler records placeholder-style dropped history and returns before transcript-backed context exists.

## Solution Assessment

Unclear for immediate implementation. The requested default-off option is plausible, but the maintainable fix needs an approved Discord config contract and silent transcript storage policy first.
