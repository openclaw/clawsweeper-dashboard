## Summary

Keep open. Current main still does not provide a Discord guild/channel silent preflight transcription option, and the existing Discord audio preflight path is scoped to DM or spoken-mention detection rather than storing transcripts for later unmentioned history context.

## What This Changes

The issue requests an opt-in Discord guild/channel `preflightTranscribe` setting that transcribes incoming audio in mention-gated channels, stores the transcript in pending history, and suppresses agent dispatch until a later mention.

## Best Possible Solution

Add a Discord-owned, backward-compatible opt-in setting with guild-level default and channel-level override, then use it in preflight after authorization to transcribe eligible audio-only no-mention messages, store the transcript in pending guild history, and return without dispatching an agent turn.

## Reproduction Assessment

Yes. The code-level reproduction is high confidence: configure a Discord guild channel with `requireMention: true`, send audio-only messages without a bot mention, and the no-mention path stores the normalized history entry rather than a transcript.

## Solution Assessment

Yes. An explicit opt-in Discord config flag is the narrowest maintainable solution because it preserves current mention gating while separating transcript ingestion from response dispatch.
