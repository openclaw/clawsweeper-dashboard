## Summary

Keep open. Current main still lacks a Discord guild/channel silent preflight transcription setting, and the existing Discord audio preflight path only uses transcripts for mention detection or later dispatch, not for storing unmentioned audio transcripts in history.

## What This Changes

Request a default-off Discord guild/channel option that pre-transcribes unmentioned audio in requireMention channels, stores transcripts in history, and suppresses replies.

## Best Possible Solution

Add a Discord plugin-owned, backward-compatible default-off guild setting with channel override that transcribes authorized audio into pending history without dispatching replies, plus schema/types, docs, changelog, and regression coverage.

## Reproduction Assessment

Yes, at source level. In a `requireMention` Discord guild channel, unmentioned audio can be transcribed only for mention detection, then the skip path records the original media-placeholder history entry and returns.

## Solution Assessment

Unclear until maintainers approve the new behavior. If approved, a default-off Discord guild/channel setting is the narrowest maintainable direction because it decouples transcript ingestion from reply dispatch without changing existing defaults.
