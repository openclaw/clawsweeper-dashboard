## Label Justifications

- `P2`: The issue blocks a useful Discord voice memo workflow, but the fix is an opt-in feature/config decision rather than an urgent regression.
- `impact:message-loss`: Unmentioned Discord voice messages are not available as transcript text when a later mentioned turn needs them.
- `impact:session-state`: The requested change controls whether derived speech content is retained and replayed into later channel context.
- `impact:security`: Silent transcription would send and retain uninvoked speech content, changing privacy and authorization expectations.

## Summary

Keep open: current main and the latest release still use Discord audio preflight for mention detection only, then record unmentioned audio as transcript-free history. The requested workflow is source-reproducible, but silent audio processing and retention need maintainer product and security decisions before implementation.

## What This Changes

Add a default-off Discord guild/channel option to silently transcribe unmentioned audio while `requireMention` still suppresses agent replies.

## Best Possible Solution

Define and implement a Discord-owned, default-off silent transcription policy only after maintainers approve authorization order, transcript retention, provider-cost behavior, and operator visibility.

## Reproduction Assessment

Yes at source level: current main can transcribe audio for mention detection, but the no-mention skip path records history without that transcript. I did not run a live Discord environment in this read-only review.

## Solution Assessment

No, not as a bare boolean: the requested behavior is plausible, but the maintainable solution needs explicit privacy, retention, cost, config inheritance, and observability semantics.
