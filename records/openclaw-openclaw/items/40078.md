## Summary

Keep open. Current main still lacks a Discord guild/channel silent preflight transcription option, and the existing Discord audio preflight only feeds mention detection or an immediately-dispatched turn, not stored history for later mentioned summaries.

## What This Changes

The issue requests an opt-in Discord guild/channel setting that silently transcribes unmentioned audio in requireMention channels so later mentioned turns can use the transcripts from history.

## Best Possible Solution

Add a default-off Discord-owned guild setting with channel override that transcribes authorized unmentioned audio into pending channel history without dispatching an agent response, with schema, docs, changelog, and regression coverage.

## Reproduction Assessment

Yes, source-level. In a Discord guild channel with `requireMention: true`, unmentioned audio reaches the mention-gate skip branch and records the prebuilt history entry whose body remains `<media:audio>`.

## Solution Assessment

Yes. A default-off Discord config option is the narrow maintainable direction because it preserves mention-gated responses while separating transcript ingestion from reply dispatch.
