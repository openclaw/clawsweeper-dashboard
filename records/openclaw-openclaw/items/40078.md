## Summary

Keep open: current main and the latest release still do not provide silent transcript capture for unmentioned Discord audio in requireMention channels, and the remaining work is a product/security decision rather than a safe cleanup close.

## What This Changes

Add an opt-in Discord channel or guild setting that silently preflight-transcribes unmentioned audio in requireMention channels and stores the transcript for later mentioned turns.

## Best Possible Solution

Define a Discord-owned, default-off silent transcription policy with explicit privacy, retention, cost, schema, docs, and requireMention-history tests before implementing it.

## Reproduction Assessment

Yes, source-level: an unmentioned Discord guild audio message in a requireMention channel can be recorded into pending history and skipped before transcript-backed admitted-turn context is built. I did not run live Discord/STT in this read-only review.

## Solution Assessment

No, not as a bare boolean alone; the requested direction is plausible, but the best fix needs maintainer-approved privacy, retention, cost, inheritance, observability, schema, docs, and tests before adding the config surface.
