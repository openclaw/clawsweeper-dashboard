## Label Justifications

- `P2`: This is a real Discord voice-memo workflow gap with limited blast radius, but it is an opt-in enhancement requiring product/security review rather than an urgent outage.
- `impact:message-loss`: Speech content from unmentioned Discord voice messages is unavailable when a later mentioned prompt asks about it.
- `impact:session-state`: The request changes what derived content is retained in channel history and replayed into later context.
- `impact:security`: The feature would silently send unmentioned audio to STT providers and retain transcripts, which is privacy-sensitive.

## Summary

Keep open: current main and the latest release still record unmentioned Discord audio through transcript-free pending history, so the requested silent history transcription is not implemented. The remaining work adds a default-off config surface and changes silent processing/retention behavior, so it needs maintainer product and security review before a fix PR.

## What This Changes

The issue requests opt-in silent Discord audio transcription for mention-gated channels so later mentioned prompts can use transcripts from earlier voice messages.

## Best Possible Solution

Keep the current default, then add a Discord-plugin-owned, default-off policy only after maintainers define authorization order, transcript retention, provider-cost behavior, and operator visibility.

## Reproduction Assessment

Yes, at source level: Discord preflight can transcribe audio for mention detection, but when `requireMention` skips the message it records dropped history from the original message text and returns before transcript-backed context is built. I did not run a live Discord/STT reproduction in this read-only review.

## Solution Assessment

No, not as fully specified: a default-off option is plausible, but the proposed boolean does not yet define consent, retention, cost, operator visibility, or exact inheritance semantics. The maintainable path is a Discord-plugin policy after those decisions, not an automated strict bug fix.
