## Summary

Keep open. Current main still lacks a Discord guild/channel silent preflight transcription config, and mention-skipped Discord audio is still recorded from the original media-placeholder history entry; this is a new user-facing Discord policy/config decision, not conservative cleanup.

## What This Changes

Request a default-off Discord guild/channel option that silently transcribes unmentioned audio into history while keeping `requireMention` reply gating.

## Best Possible Solution

Add a Discord-owned, backward-compatible default-off guild setting with channel override that transcribes authorized audio into pending history without dispatching replies, with schema/types, docs, changelog, and regression coverage.

## Reproduction Assessment

Yes, at source level. In a `requireMention` Discord guild channel, an unmentioned audio-only message can be transcribed only for mention detection, then the skip path records the original placeholder history entry and returns without preserving the transcript.

## Solution Assessment

Unclear until maintainers approve the new behavior. If approved, a default-off Discord config is the narrowest maintainable path because it decouples transcript ingestion from reply dispatch without changing existing defaults.
