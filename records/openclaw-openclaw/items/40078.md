## Summary

Keep open. Current main still lacks a Discord guild/channel `preflightTranscribe` setting, and the existing Discord preflight path only uses audio transcription for DM or spoken-mention detection, not for silent history ingestion after an unmentioned guild voice message is skipped.

## What This Changes

Request to add a default-off Discord guild/channel `preflightTranscribe` option that transcribes authorized unmentioned audio into channel history without triggering an agent reply.

## Best Possible Solution

Add a Discord-owned, backward-compatible opt-in flag with guild-level default and channel-level override, then store authorized silent audio transcripts in pending channel history while preserving mention-gated replies.

## Reproduction Assessment

Yes, from source inspection rather than a live run. In a Discord guild channel with `requireMention: true`, unmentioned audio reaches the no-mention skip path and records the normalized `<media:audio>` history entry instead of a transcript.

## Solution Assessment

Yes. A default-off Discord config flag is the narrow maintainable direction because it separates transcript ingestion from reply dispatch without changing existing mention-gated behavior.

## Repair Work Prompt

Fix the Discord silent preflight transcription gap for https://github.com/openclaw/openclaw/issues/40078. Add a backward-compatible default-off `preflightTranscribe` boolean under `channels.discord.guilds.<guild>` and `channels.discord.guilds.<guild>.channels.<channel>`, with channel override over guild default. After guild/channel access and sender authorization succeed, if an inbound guild message is audio-only, would otherwise be skipped only because `requireMention: true` and no mention was detected, and `preflightTranscribe` resolves true, run the existing Discord audio transcription path using `tools.media.audio`, store the transcript in pending guild history for that channel with the original sender/timestamp/message id, and return without triggering the agent reply pipeline. Do not transcribe unauthorized senders, do not bypass mention-gated replies, and preserve current DM plus spoken-mention detection behavior. Update config types, strict schema, generated config metadata/docs as required by repo policy, Discord docs, changelog, and focused regression tests.
