## Summary

Keep this open: current main has Discord audio preflight for mention detection, but it still lacks the requested opt-in silent transcription path that records transcripts for later history context without dispatching an agent reply.

## What This Changes

Add a Discord guild/channel-level option to silently transcribe audio in requireMention channels and make later mentioned turns see those transcripts in history.

## Best Possible Solution

Add an explicit Discord-owned guild/channel policy that defaults off, runs after sender authorization, records transcript-backed pending history on mention-gated drops, suppresses agent dispatch, and documents the privacy/cost tradeoff.

## Reproduction Assessment

Yes by source inspection. Current main can transcribe audio for mention detection, but when the transcript does not mention the bot the no-mention path records the placeholder-based history entry and returns without storing the transcript.

## Solution Assessment

Unclear for immediate implementation. The proposed per-channel option is plausible, but maintainers should first approve the config contract and silent transcription policy because it changes when OpenClaw spends provider calls and stores transcript text.
