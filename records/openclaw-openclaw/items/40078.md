## Summary

Keep open: current main has Discord audio preflight for mention detection, but not an opt-in silent history-only transcription mode for unmentioned requireMention messages. Adding it would be a new Discord config surface with cost/privacy policy implications around processing unmentioned audio.

## What This Changes

Requested a per-guild or per-channel Discord option to silently transcribe audio messages in requireMention channels so later mentioned turns can use transcript history without triggering an immediate bot reply.

## Best Possible Solution

Add an explicit opt-in Discord guild/channel setting, or a clearly documented alternative, that transcribes authorized audio-only messages into pending channel history without dispatching a reply, with tests and docs covering mention-gated history behavior.

## Reproduction Assessment

Yes. Source inspection shows a high-confidence path: unmentioned requireMention guild messages are dropped after pending-history recording, and that history entry is built from the original message text/media placeholder rather than any transcript.

## Solution Assessment

Unclear. The requested boolean is plausible, but the safest maintainable solution needs maintainer agreement on opt-in scope, authorization, cost/privacy behavior, and whether transcripts should live only in pending channel history or durable session history.
