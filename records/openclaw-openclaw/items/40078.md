## Summary

Keep open: current main and the latest release still lack the requested Discord opt-in, and the current preflight path records unmentioned `requireMention` guild audio as dropped history before any transcript-backed context can be persisted. This is valid feature work, but it needs maintainer direction on privacy, provider cost, persistence, and config semantics before implementation.

## What This Changes

Request a default-off Discord guild/channel option that silently transcribes audio-only messages in `requireMention` channels without replying, so later mentioned turns can use transcript-backed history.

## Best Possible Solution

Add a Discord-owned, default-off opt-in that runs only after sender authorization, stores transcript-backed dropped history without dispatching the agent, and documents privacy/cost tradeoffs plus `echoTranscript` composition.

## Reproduction Assessment

Yes, at source level. An authorized audio-only Discord guild message in a `requireMention` channel can be transcribed for mention detection, but when no mention is found the handler records placeholder-style dropped history and returns before transcript-backed context exists.

## Solution Assessment

Unclear for immediate implementation. The requested default-off option is plausible, but the maintainable fix needs an approved Discord config contract and silent transcript storage policy first.
