## Summary

Keep open: current main still only uses Discord audio preflight for mention detection, then records placeholder-based history on mention-gated drops. The requested silent transcript capture remains a new opt-in Discord policy with privacy, cost, and config-contract implications.

## What This Changes

The issue requests an opt-in Discord guild or channel option to transcribe unmentioned audio messages into history while suppressing agent replies until the bot is later mentioned.

## Best Possible Solution

Add a Discord-owned opt-in policy, defaulting off, that runs after sender authorization, transcribes selected unmentioned audio into pending history without dispatching the agent, and documents the privacy/cost tradeoff.

## Reproduction Assessment

Yes by source inspection. Current main can transcribe Discord audio for mention detection, but the unmentioned requireMention drop path records the placeholder history entry and returns before any transcript-backed context is stored.

## Solution Assessment

Unclear for immediate implementation. The proposed per-channel option is plausible, but the maintainable path needs maintainer approval for the config contract and silent transcript storage policy first.
