## Summary

Keep open: current main and the latest release still lack the requested Discord opt-in policy, and the source path still records unmentioned requireMention audio as placeholder-backed dropped history instead of transcript-backed context. The related `requireMention:false` request was closed as a different case because those messages continue into the normal media pipeline.

## What This Changes

The issue requests an opt-in Discord guild or channel setting that silently transcribes unmentioned audio without dispatching an agent reply so later mentioned turns can use transcript-backed history.

## Best Possible Solution

Add a Discord-owned, default-off opt-in policy that runs only after sender authorization, stores transcript-backed pending history without dispatching the agent, and documents the privacy and cost tradeoffs.

## Reproduction Assessment

Yes, by source inspection. An authorized audio-only Discord guild message can be preflight-transcribed for mention detection, but if no mention is found the handler records placeholder-style dropped history and returns before transcript-backed context exists.

## Solution Assessment

Unclear for immediate implementation. The requested option is plausible, but the maintainable fix needs an approved Discord config contract and silent transcript storage policy first.
