## Summary

Keep open. Current main still lacks the requested Discord guild/channel silent preflight transcription setting, and the existing Discord audio preflight path only uses transcripts for mention detection or dispatch, not for storing unmentioned audio transcripts in history.

## What This Changes

Request a default-off Discord guild/channel option that silently transcribes unmentioned audio in `requireMention` channels and stores transcripts in history without triggering a reply.

## Best Possible Solution

Add a Discord plugin-owned, backward-compatible default-off guild setting with channel override that transcribes authorized audio into pending history without dispatching replies, with schema/types, docs, changelog, and focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main builds pending history before audio preflight, records that original placeholder entry on the `requireMention` skip path, and only applies `preflightAudioTranscript` when a message reaches dispatch.

## Solution Assessment

Unclear until maintainers approve the new behavior. If approved, a default-off Discord guild/channel setting is the narrowest maintainable direction because it decouples transcript ingestion from reply dispatch without changing existing defaults.
