## Summary

Keep open. Current main and the latest local release snapshot still lack a Discord guild/channel `preflightTranscribe` option, and the existing Discord preflight path either skips unmentioned guild audio or uses its transcript only for spoken-mention detection before recording the placeholder history entry.

## What This Changes

Add an opt-in Discord guild/channel setting that silently transcribes unmentioned audio messages in `requireMention` channels and stores those transcripts for later history context without dispatching an agent reply.

## Best Possible Solution

Add a Discord-owned, backward-compatible opt-in setting with guild-level default and channel-level override, then store authorized silent audio transcripts in pending channel history without triggering an agent turn.

## Reproduction Assessment

Yes. The code-level path is clear: in a Discord guild channel with `requireMention: true`, unmentioned audio-only messages either do not enter the transcription path or have any mention-detection transcript discarded while the recorded history entry remains `<media:audio>`.

## Solution Assessment

Yes. A default-off Discord config flag is the narrow maintainable direction because it preserves mention-gated responses while separating transcript ingestion from reply dispatch.
