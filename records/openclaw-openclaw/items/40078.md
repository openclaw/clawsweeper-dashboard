## Summary

Keep open: current main still does not implement silent Discord audio transcription for unmentioned `requireMention` messages. The report is a plausible feature request with source-level reproduction, but it needs maintainer product approval because it adds a new opt-in config and changes privacy/cost behavior for authorized but unmentioned audio.

## What This Changes

Add a Discord guild/channel `preflightTranscribe` option that transcribes inbound audio without triggering a bot reply so later mentioned turns can use transcript-backed history.

## Best Possible Solution

Add a Discord-owned opt-in policy, defaulting off, that runs after sender authorization, transcribes selected unmentioned audio into pending history without dispatching the agent, and documents the privacy/cost tradeoff.

## Reproduction Assessment

Yes by source inspection. An audio-only authorized guild message in a `requireMention` channel can be transcribed for mention detection, but if no mention is found the handler records placeholder-based pending history and returns before transcript-backed context is built.

## Solution Assessment

Unclear for immediate implementation. The proposed per-channel option is plausible, but the maintainable fix needs an approved Discord config contract and silent transcript storage policy first.
