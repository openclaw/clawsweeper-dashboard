## Summary

Keep open: current main and the latest release still lack a Discord opt-in policy that silently transcribes unmentioned audio and persists transcript-backed pending history in mention-gated guild channels. This is a real feature/product decision, not conservative cleanup.

## What This Changes

Add a per-guild or per-channel Discord option to transcribe audio-only messages without dispatching the agent so later mentioned turns can use transcript-backed history.

## Best Possible Solution

Add a Discord-owned opt-in, default-off silent preflight transcription policy after sender authorization that records transcript-backed pending history without dispatching the agent, with docs for privacy and provider-cost tradeoffs.

## Reproduction Assessment

Yes, by source inspection. An authorized audio-only Discord guild message in a `requireMention` channel can be transcribed for mention detection, but when no mention is found the handler records placeholder-based pending history and returns before transcript-backed context is built.

## Solution Assessment

Unclear for immediate implementation. The requested option is plausible, but the maintainable fix needs an approved Discord config contract and silent transcript storage policy first.
