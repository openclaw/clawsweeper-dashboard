## Summary

Keep open: current main still source-reproduces the misleading QMD embedding status, and an open fixing PR already references this issue with closing syntax, so the issue should stay open until that PR or a replacement lands.

## What This Changes

Request that `openclaw memory status --deep` stop reporting QMD embeddings as unavailable when `memory.qmd.searchMode` is intentionally set to lexical `search` mode.

## Best Possible Solution

Land a narrow memory-core fix, or a replacement for the linked PR, that represents lexical QMD embedding readiness as skipped or not applicable in status output while preserving real unavailable errors for vector modes.

## Reproduction Assessment

Yes. Current main source and tests show `searchMode: "search"` disables QMD vectors, then the embedding probe maps that intentional false result to the generic unavailable error printed by `memory status --deep`.

## Solution Assessment

Yes. The narrow maintainable fix is in the QMD embedding-probe/status-rendering path: suppress or explicitly mark the embedding probe as skipped for lexical QMD mode without changing QMD search semantics.
