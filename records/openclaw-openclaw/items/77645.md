## Summary

Keep open: current main still matches the reported confusion. The documented QMD lexical mode intentionally disables semantic vectors, but deep memory status still probes embeddings and the QMD manager turns that disabled state into the reported embeddings error.

## What This Changes

Make `openclaw memory status --deep` treat QMD lexical `searchMode: "search"` as an intentional embeddings skip instead of an embeddings failure.

## Best Possible Solution

Preserve disabled vector status for lexical QMD mode, but make the deep status surface report embeddings as skipped or omit the embeddings error for that intentional mode while keeping real vector failures visible for `vsearch` and `query`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure QMD with `searchMode: "search"` and run deep memory status; the current QMD manager test already asserts the exact unavailable embedding error for that mode.

## Solution Assessment

Yes. The narrow maintainable fix is to distinguish intentional lexical-mode skips from unavailable semantic-vector setups in the memory-core QMD status/probe path, with regression coverage for the user-facing status output.
