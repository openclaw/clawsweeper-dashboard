## Summary

Keep open: current main still makes `memory status --deep` report QMD embeddings unavailable for lexical `searchMode: "search"`, even though docs and prior changelog state that this mode skips semantic probes and embedding maintenance.

## What This Changes

The issue requests clearer `openclaw memory status --deep` output for QMD BM25-only `searchMode: "search"` so intentionally disabled vectors are not reported as an embedding failure.

## Best Possible Solution

Teach the QMD/status path to omit or explicitly mark embedding readiness as skipped for lexical `searchMode: "search"`, while preserving real vector and embedding errors for `vsearch` and `query`.

## Reproduction Assessment

Yes, source-reproducible: current main calls the embedding probe during `memory status --deep`, and QMD lexical mode maps intentionally disabled vectors into the default unavailable embedding error. I did not run a live CLI command because this review was read-only.

## Solution Assessment

Yes, the narrow maintainable fix is in `memory-core` status/probe presentation: skip or mark the embedding probe as skipped when QMD vectors are disabled by `searchMode: "search"`, without changing config semantics.
