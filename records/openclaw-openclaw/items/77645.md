## Summary

Keep open: current main still source-reproduces the misleading QMD embedding status, and the linked fixing PR is open but unmerged, so this issue should remain open until that PR or a replacement lands.

## What This Changes

Report that `openclaw memory status --deep` should not show QMD embeddings as unavailable when `memory.qmd.searchMode` intentionally uses lexical `search` mode.

## Best Possible Solution

Land a narrow memory-core fix, or a replacement for the linked PR, that represents lexical QMD embedding readiness as skipped or not applicable while preserving real unavailable errors for vector modes.

## Reproduction Assessment

Yes. Current main source and the focused QMD test show `searchMode: "search"` disables vectors, then the embedding probe maps that intentional false result to the generic unavailable error printed by deep memory status.

## Solution Assessment

Yes. The narrow maintainable fix is to distinguish skipped lexical QMD mode in the embedding-probe/status-rendering path without changing QMD search semantics.
