## Summary

Keep open. Current main and v2026.6.8 still lack channel-origin metadata and channel-bound approval cards for device scope upgrades, while related issues cover narrower Telegram, Feishu, or browser-approval slices rather than replacing this broader product/security design item.

## What This Changes

Request to attach triggering channel context to device scope-upgrade approval requests and send an approval card back to the originating chat channel.

## Best Possible Solution

Define a generic, replay-safe approval origin/callback contract for device scope upgrades, keep pending device pairing as the source of truth, and render channel-specific approval UI behind typed channel/plugin seams.
