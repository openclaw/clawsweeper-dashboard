## Summary

Keep open. Current main has partial mitigations, but provider billing/rate-limit and runtime diagnostics can still be built as user-facing channel payloads without the requested suppress, custom-message, or owner-only disclosure policy across WhatsApp and Telegram.

## What This Changes

The issue asks for configurable suppression, custom copy, or owner-only routing for raw provider/runtime error messages before they reach WhatsApp, Telegram, or similar chat users.

## Best Possible Solution

Define and document one outbound provider/runtime error-disclosure contract before channel delivery, with per-channel or per-peer silent, custom-copy, and owner-only modes while retaining raw diagnostics in logs and approved operator surfaces.

## Reproduction Assessment

Yes, source-level. Current main still builds billing/rate-limit provider failure text as channel-deliverable payloads without a complete suppress/custom/owner-only policy; no live channel replay was run.
