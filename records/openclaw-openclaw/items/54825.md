## Summary

Keep open. Current main and v2026.5.18 still have only partial mitigations: billing/rate-limit failure text can still be built as user-facing channel copy, WhatsApp has no suppress/custom/owner-only config surface, and Telegram's existing policy is narrower than the requested disclosure contract.

## What This Changes

The issue requests suppressible, customizable, or owner-only handling for raw provider/runtime error messages sent to WhatsApp and Telegram users.

## Best Possible Solution

Define and document one outbound provider/runtime error disclosure contract before channel delivery, with per-channel or per-peer silent, custom-copy, and owner-only modes while keeping raw diagnostics in logs and approved operator surfaces.

## Reproduction Assessment

Yes, source-level. Current main still has code paths that build billing/rate-limit failure reply text without a complete suppress/custom/owner-only channel disclosure policy.

## Solution Assessment

Unclear as an implementation because the issue proposes several valid policies. The maintainable path is a deliberate shared or channel-owned disclosure contract, not another one-channel filter or default flip.
