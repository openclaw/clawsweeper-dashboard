## Summary

Keep open. Current main and the latest release still have source-level paths where billing/rate-limit provider failures become channel-deliverable text, WhatsApp still lacks the requested suppress/custom/owner-only config surface, and the related open PR/issues are not merged replacements for the remaining product/security decision.

## What This Changes

The issue requests per-channel or per-peer control to suppress, customize, or owner-route provider/runtime error replies on WhatsApp and Telegram.

## Best Possible Solution

Define and document one outbound provider/runtime error-disclosure contract before channel delivery, with per-channel or per-peer suppress, custom-copy, and owner-only modes while preserving raw diagnostics in logs and approved operator surfaces.

## Reproduction Assessment

Yes, at source level. Current main still builds billing/rate-limit provider failures as deliverable text and lacks the requested WhatsApp policy/config surface; no live WhatsApp or Telegram replay was run in this read-only review.

## Solution Assessment

Unclear as an implementation because the issue proposes several valid policy options. The maintainable solution is a maintainer-chosen shared disclosure contract, not another isolated channel filter with conflicting defaults.
