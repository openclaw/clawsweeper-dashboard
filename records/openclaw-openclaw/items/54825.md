## Summary

Keep open. Current main has partial mitigations and several narrower related items, but it still lacks the requested cross-channel suppress/custom/owner-only disclosure policy for provider and gateway errors.

## What This Changes

Request a channel-level policy to suppress, customize, or owner-route raw provider/gateway error messages instead of sending them to WhatsApp or Telegram users.

## Best Possible Solution

Define and document an outbound provider/runtime error disclosure policy before channel delivery, with per-channel or per-peer silent, custom-copy, and owner-only modes while preserving raw diagnostics in logs and approved operator surfaces.

## Reproduction Assessment

Yes, source-level. Current main still builds billing/rate-limit failure payloads before any complete channel disclosure policy, WhatsApp only filters payload.isError, and Telegram only has partial always/once/silent controls.

## Solution Assessment

Unclear as an implementation because the issue proposes several possible policies. The maintainable path is a deliberate shared or channel-owned disclosure contract, not scattered filters or a one-channel default change.
