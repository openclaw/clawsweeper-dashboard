## Summary

Keep open. Current main and the latest release have partial mitigations, but they still do not provide the requested cross-channel suppress/custom/owner-only disclosure policy for provider and gateway errors, and the related open items cover narrower slices.

## What This Changes

The issue requests a policy to suppress, customize, or owner-route raw provider/runtime error messages before they are sent to WhatsApp, Telegram, and similar channels.

## Best Possible Solution

Define and document one outbound provider/runtime error disclosure contract before channel delivery, with per-channel or per-peer silent, custom-copy, and owner-only modes while keeping raw diagnostics in logs and approved operator surfaces.

## Reproduction Assessment

Yes, source-level. Current main still has paths that build billing/rate-limit failure reply text without a complete channel disclosure policy, while WhatsApp and Telegram only provide partial controls.

## Solution Assessment

Unclear as an implementation because the issue proposes several valid policies. The maintainable path is a deliberate shared or channel-owned disclosure contract, not another one-channel filter or default flip.
