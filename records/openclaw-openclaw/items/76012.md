## What This Changes

The PR expands startup-gated Gateway RPCs, returns the shared `startup-sidecars` retry details from method-level startup gates, adds regression coverage, and records the fix in the changelog.

## Best Possible Solution

Keep the runtime, test, and changelog changes, add a matching Gateway protocol note for startup-gated RPC retries, then let CI and maintainer review decide merge readiness.

## Reproduction Assessment

Yes. Static current-main inspection reproduces the mismatch: startup-gated methods return retryable `UNAVAILABLE` with only `{ method }`, while the shared retry helper requires `details.reason === "startup-sidecars"`.

## Solution Assessment

Mostly yes. Reusing the existing startup retry helper and retry-after constant is the narrowest runtime fix; the safer completion is to document that the same retry shape now applies to startup-gated RPC methods.
