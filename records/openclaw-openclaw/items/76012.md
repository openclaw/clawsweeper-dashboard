## Summary

Keep open. The runtime patch is the right narrow direction, but it still changes the public Gateway RPC retry contract without the matching protocol docs and active changelog entry, and the protected maintainer label means it should stay in review rather than be closed.

## What This Changes

The PR expands startup-gated Gateway RPC methods, returns the shared `startup-sidecars` retry shape from method-level startup gates, and adds regression coverage for those gated methods.

## Best Possible Solution

Keep the runtime change, add the public Gateway protocol note and an active `CHANGELOG.md` Fixes entry, then let CI and maintainer review decide merge readiness.

## Reproduction Assessment

Yes. Static current-main inspection reproduces the mismatch: startup-gated methods return retryable `UNAVAILABLE` with only `{ method }`, while the shared retry helper recognizes only `details.reason === "startup-sidecars"`.

## Solution Assessment

Mostly yes. Reusing the existing startup retry helper and retry-after constant is the narrowest maintainable runtime fix; the safer completion is to document the method-level retry contract and add the required changelog entry.
