The runtime approach matches the existing startup retry helper and WebSocket handshake contract, but the PR changes the Gateway wire contract without the required protocol docs and changelog coverage, and the protected maintainer label means merge still needs explicit maintainer handling.

## What This Changes

This PR expands the gateway startup-gated RPC list, changes method-level startup rejections to use the shared `startup-sidecars` retry shape, and adds regression coverage across gated methods.

## Best Possible Solution

Keep the runtime change, add the public Gateway protocol note and active changelog entry, then let CI and maintainer review decide merge readiness.

## Reproduction Assessment

Yes. Static current-main inspection reproduces the mismatch: startup-gated methods return retryable `UNAVAILABLE` with only `{ method }`, while the shared retry helper recognizes only errors with `details.reason: "startup-sidecars"`.

## Solution Assessment

Mostly yes. Reusing the existing startup retry helper and retry-after constant is the narrowest maintainable runtime fix; the safer completion is to document the method-level retry contract and add the required changelog entry.
