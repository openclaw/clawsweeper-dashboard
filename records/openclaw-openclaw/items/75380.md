## Summary

Keep open. Current main still leaves the cache-trace and Anthropic provider-payload diagnostic JSONL sinks effectively unbounded: both create append-only queued writers without any size/archive options, and the shared writer only supports cap-and-drop behavior rather than rotation. The only linked fix PR is closed unmerged, so there is no implemented or canonical open fix protecting this report.

## What This Changes

Request a bounded rotation or max-size policy for cache-trace and provider payload JSONL diagnostic files.

## Best Possible Solution

Add a small shared bounded diagnostic JSONL policy for these sinks: rotate the active file into a bounded archive ring with sane defaults, expose config/env controls where users expect them, preserve queued-writer filesystem safety and payload redaction, and document the behavior with focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: enable OPENCLAW_CACHE_TRACE or OPENCLAW_ANTHROPIC_PAYLOAD_LOG, run repeated embedded model turns, and the active JSONL files append through getQueuedFileWriter without any wired max-size/archive policy. I did not live-run traffic because this review is read-only and must not create artifacts.

## Solution Assessment

Yes. A focused fix in the queued diagnostic writer plus cache-trace/provider-payload config wiring is the narrow maintainable direction; manual truncation or cap-only dropping would leave operators without bounded retained diagnostics.
