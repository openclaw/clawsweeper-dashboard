## Summary

Keep open. Current main still has the reported diagnostic JSONL growth gap: cache-trace and Anthropic provider payload diagnostics resolve only enablement/path/content toggles and then append through the queued writer without a wired size/archive policy. The related fix PR was closed without merging, so there is no implemented or canonical open fix protecting this report.

## What This Changes

The issue requests bounded rotation and configurable max-size/archive retention for cache-trace and provider payload diagnostic JSONL files.

## Best Possible Solution

Add a small shared bounded diagnostic JSONL policy for these sinks: rotate to a bounded archive ring with sane defaults, expose cache-trace and provider-payload config/env controls, keep redaction and filesystem safety intact, and document the behavior with focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: enable OPENCLAW_CACHE_TRACE or OPENCLAW_ANTHROPIC_PAYLOAD_LOG, run repeated embedded model turns, and the active JSONL files append through getQueuedFileWriter without any max-size or archive policy wired into those sinks. I did not live-run traffic because this review is read-only and must not create artifacts.

## Solution Assessment

Yes. A focused change in the queued diagnostic writer plus cache-trace/provider-payload config wiring is the narrow maintainable fix; manual truncation or cap-only suppression would still leave operators without bounded retained diagnostics.
