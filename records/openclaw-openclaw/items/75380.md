## Summary

Keep open. Current main still leaves the cache-trace and provider-payload diagnostic JSONL paths without a configured size cap or archive rotation, and open PR #75389 explicitly references this issue as its fix, so the issue should stay open until that PR or an equivalent fix merges.

## What This Changes

Add configurable max-size and rotation/retention controls for cache-trace and provider payload diagnostic JSONL files so they do not grow without bound.

## Best Possible Solution

Review and land PR #75389 or an equivalent focused change that adds bounded rotation/retention for these diagnostic JSONL writers, wires the config/schema/docs/tests, and preserves the existing diagnostic redaction and filesystem safety checks.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: enable cache tracing or the anthropic payload logger, generate repeated agent turns, and the active JSONL file appends indefinitely because neither call site passes a cap or rotation policy.

## Solution Assessment

Yes, keeping this open is the right issue state because a closing PR is already open but unmerged. The maintainable code solution is to extend the queued diagnostic JSONL writer/config surface rather than relying on manual truncation or cron cleanup.
