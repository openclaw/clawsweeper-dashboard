## Summary

Keep open: current main and the latest release still write the cache-trace and Anthropic payload diagnostic JSONL sinks without rotation/archive retention, and the open linked PR has not merged.

## What This Changes

The issue requests a bounded rotation or max-size policy for `cache-trace.jsonl` and provider payload diagnostic JSONL output.

## Best Possible Solution

Land or replace the canonical diagnostic-retention fix with in-process bounded retention for cache-trace and Anthropic payload diagnostics, aligned schema/docs/tests, and real OpenClaw proof; maintainers should decide whether this is short-term JSONL rotation or a SQLite diagnostic-events migration.

## Reproduction Assessment

Yes by source inspection and reporter evidence: sustained cache-trace or Anthropic payload logging still appends diagnostic JSONL on current main and `v2026.6.6` without max-size or archive retention on those sinks. I did not run a long-duration live gateway repro in this read-only review.

## Solution Assessment

Unclear for the exact open PR until it receives full review, rebase, and real proof, but the fix belongs at the diagnostic writer/sink boundary rather than external logrotate. Maintainers still need to choose the permanent retention/config shape and whether these diagnostics should move to SQLite instead of extending JSONL runtime knobs.
