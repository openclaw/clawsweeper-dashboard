## Summary

Keep open. Current main still has uncapped diagnostic JSONL sinks for cache trace and Anthropic provider payload logging, and the related fix PR is closed without merging, so the reported storage growth remains actionable.

## What This Changes

Request a configurable max-size and rotation policy for cache-trace and provider payload diagnostic JSONL logs.

## Best Possible Solution

Add a shared bounded diagnostic JSONL rotation policy for queued diagnostic writers, wire it into cache-trace and provider payload logging with documented defaults and env/config controls, and preserve existing redaction plus symlink/hardlink protections.
