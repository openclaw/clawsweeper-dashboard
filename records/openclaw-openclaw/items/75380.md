## Summary

Keep open. Current main and the latest release still leave the cache-trace and Anthropic payload diagnostic JSONL sinks without wired rotation/default caps, and an open closing PR already targets this issue.

## What This Changes

Request bounded rotation and max-size configuration for diagnostic JSONL payload and cache-trace logs.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/76407, then close this issue as implemented while keeping gateway.err.log and journald rotation reports separate.

Yes, at source level. Enabling `diagnostics.cacheTrace` or `OPENCLAW_ANTHROPIC_PAYLOAD_LOG` and generating turns reaches writers that append JSONL without rotation or diagnostic-specific caps on current main.

## Solution Assessment

Yes for the current direction. Extending the shared queued writer with bounded archive rotation and wiring the two diagnostic sinks through schema/docs/tests is the narrow maintainable fix, but the open PR must be reviewed and made mergeable first.
