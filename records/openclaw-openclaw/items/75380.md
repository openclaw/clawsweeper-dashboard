## Summary

Keep open. Current main still wires cache-trace and the current Anthropic/provider payload diagnostic sink to queued JSONL writers without a size/archive policy, while the related attempted fix PR is closed unmerged and the latest comment adds credible severity evidence.

## What This Changes

Add configurable max-size and archive rotation for cache-trace and provider/Anthropic payload diagnostic JSONL files so they cannot grow without bound.

## Best Possible Solution

Add a shared bounded diagnostic JSONL policy for cacheTrace and payload diagnostics with sane defaults, archive retention, config/env/schema/docs coverage, and focused tests while preserving existing writer hardening.
