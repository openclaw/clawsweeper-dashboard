## Summary

Keep open. Current main still leaves the cache-trace and Anthropic provider-payload diagnostic JSONL writers without a bounded rotation/archive policy, and the only linked closing PR was closed unmerged, so the reported disk-growth gap remains valid.

## What This Changes

The issue requests bounded max-size and archive rotation controls for provider-payload and cache-trace JSONL diagnostic files under OpenClaw logs.

## Best Possible Solution

Add a shared bounded retention policy for queued diagnostic JSONL files, wire it through cache-trace and provider-payload settings/env/docs/tests, and preserve existing redaction, path-safety checks, and append ordering.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: enable cache trace or `OPENCLAW_ANTHROPIC_PAYLOAD_LOG`, run repeated model turns, and the active JSONL files append through queued writers with no active size/archive policy. I did not live-run traffic because this review is read-only and must not create artifacts.

## Solution Assessment

Yes. A focused diagnostic-writer cap/rotation change is the narrow maintainable direction; true archive rotation is the safer match for the expected behavior than silently dropping later diagnostics at a cap.
