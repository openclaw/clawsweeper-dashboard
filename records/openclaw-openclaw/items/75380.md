## Summary

Keep open. Current main still appends cache-trace and Anthropic payload diagnostic JSONL without wired max-size/archive rotation, and the issue discussion includes a closed unmerged fix PR plus a later 46 GB cache-trace incident.

## What This Changes

Add bounded max-size and archive rotation policy for diagnostic JSONL sinks used by cache trace and provider payload logging.

## Best Possible Solution

Add a shared rotate-and-retain path for diagnostic JSONL writers, wire cache trace and Anthropic payload logging through it with documented defaults, and leave gateway.err.log, journald, and session-transcript maintenance to their separate reports.

## Reproduction Assessment

Yes. Source inspection shows enabling `diagnostics.cacheTrace` or `OPENCLAW_ANTHROPIC_PAYLOAD_LOG` creates append-only JSONL writers with no wired cap or archive policy, and the comments provide real large-file incidents.

## Solution Assessment

Yes for the narrow direction. Extending the queued diagnostic writer to rotate and retain bounded archives, then wiring both sinks through that path, is the maintainable fix; broader doctor warnings and cache-trace content defaults can be follow-ups.
