## Summary

Keep open. Current main still routes cache-trace and Anthropic provider-payload diagnostics through append-only queued JSONL writers with no wired size/archive policy, and the linked fix PR is closed without merging, so there is no implemented or canonical open fix protecting the report.

## What This Changes

Request bounded rotation or max-size controls for diagnostic JSONL files used by provider payload logging and cache-trace logging.

## Best Possible Solution

Add a small shared bounded diagnostic JSONL policy for these sinks: rotate to a bounded archive ring with sane defaults, expose config/env controls where users expect them, preserve queued-writer filesystem safety and payload redaction, and document the behavior with focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: enable OPENCLAW_CACHE_TRACE or OPENCLAW_ANTHROPIC_PAYLOAD_LOG, run repeated embedded model turns, and the active JSONL files append through getQueuedFileWriter without any max-size or archive policy wired in. I did not live-run traffic because this review is read-only and must not create artifacts.

## Solution Assessment

Yes. A focused fix in the queued diagnostic writer plus cache-trace/provider-payload config wiring is the narrow maintainable direction; manual truncation or cap-only dropping would leave operators without bounded retained diagnostics.
