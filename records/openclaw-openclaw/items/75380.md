## Summary

Keep open. Current main still writes cache-trace and Anthropic payload diagnostic JSONL through uncapped queued writers, and #76407 is an open closing PR for the same work, so this should wait for that PR instead of being closed or queued separately.

## What This Changes

Request bounded rotation/max-size controls for `cache-trace.jsonl` and provider payload diagnostic JSONL files under the OpenClaw logs directory.

## Best Possible Solution

Review and land or close #76407; after it merges, close this issue as implemented while keeping gateway stderr, journald, and session-transcript storage reports separate.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: enable `diagnostics.cacheTrace` or `OPENCLAW_ANTHROPIC_PAYLOAD_LOG`, generate repeated turns, and current main appends each event through `getQueuedFileWriter` without max-size/archive options.

## Solution Assessment

Yes for the current direction. A shared queued-writer rotation/retention mode wired into both diagnostic sinks with schema, docs, and tests is the narrow maintainable fix, with #76407 as the active implementation candidate.
