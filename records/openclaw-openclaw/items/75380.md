## Summary

Keep open. Current main still leaves the cache-trace and Anthropic payload diagnostic JSONL sinks without wired max-size/archive rotation, and the closed prior PR did not land a replacement fix while the issue discussion now includes a 46 GB cache-trace incident.

## What This Changes

Request a bounded rotation and max-size policy for cache-trace and provider payload diagnostic JSONL files under OpenClaw logs.

## Best Possible Solution

Add shared bounded diagnostic JSONL rotation for cache trace and Anthropic payload logging with sane defaults, config/env schema, docs, focused tests, and a changelog entry, while leaving gateway.err.log, journald, and transcript-maintenance work to their separate reports.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: enable `diagnostics.cacheTrace` or `OPENCLAW_ANTHROPIC_PAYLOAD_LOG` during repeated runs and the active JSONL files append through `getQueuedFileWriter` without rotation or wired caps; the issue discussion also includes real large-file incidents.

## Solution Assessment

Yes for the narrow fix direction. Extending the shared diagnostic writer to rotate and retain bounded archives, then wiring both sinks through it, is the maintainable path; broader cache-trace content defaults and doctor checks can be follow-ups if maintainers want them in scope.
