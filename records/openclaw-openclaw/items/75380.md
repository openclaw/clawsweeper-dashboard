## Summary

Keep open. Current main still routes both cache-trace and Anthropic provider payload diagnostics to append-only queued JSONL writers without passing a max-size or archive policy, and the linked fix PR #75389 is still open and unmerged in the provided GitHub context.

## What This Changes

The issue asks OpenClaw to add configurable max-size and rotation defaults for cache-trace and provider payload diagnostic JSONL files under the logs directory.

## Best Possible Solution

Review and land #75389 or a successor that adds a bounded diagnostic JSONL policy for cache trace and provider payload logging, wires config/env/docs/tests, and preserves current redaction plus queued-writer filesystem protections.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: enable `OPENCLAW_CACHE_TRACE=1` or `OPENCLAW_ANTHROPIC_PAYLOAD_LOG=1`, run repeated embedded/Anthropic turns, and the active JSONL files append through queued writers without a size/archive policy. I did not live-run traffic because this read-only review must not create artifacts.

## Solution Assessment

Yes for the issue direction: a focused bounded-policy change at the diagnostic writer/caller boundary is the narrow maintainable fix. The current main implementation is not sufficient, and cap-only behavior would be less complete than the requested rotation/archive model unless deliberately documented.
