## Summary

Keep open. Current main still appends the cache-trace and Anthropic payload diagnostic JSONL files through uncapped writer paths, and an open closing PR already targets this issue, so the right cleanup is to review that PR rather than close or queue duplicate work.

## What This Changes

Request rotation, retention, and configurable size bounds for diagnostic JSONL outputs that currently grow without an archive policy.

## Best Possible Solution

Review and land or close #76407; after it merges, close this issue as implemented while tracking broader diagnostic-file hygiene separately.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: enable `diagnostics.cacheTrace` or `OPENCLAW_ANTHROPIC_PAYLOAD_LOG`, generate repeated turns, and the current writers append JSONL through uncapped `getQueuedFileWriter` paths.

## Solution Assessment

Yes for the current direction, but it is not implemented on main yet. A shared queued-writer rotation/retention mode wired into both diagnostic sinks with schema, docs, and focused tests is the narrow maintainable fix.
