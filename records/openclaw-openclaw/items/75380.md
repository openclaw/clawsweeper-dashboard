## Summary

Keep open. Current main still has uncapped cache-trace and Anthropic provider-payload diagnostic JSONL writers, and the linked fix PR is closed without merging, so there is no implemented or canonical open fix protecting the report.

## What This Changes

The issue requests a bounded max-size and archive/rotation policy for provider payload and cache trace diagnostic JSONL files.

## Best Possible Solution

Add a small shared bounded diagnostic JSONL policy that preserves queued-writer filesystem safety and redaction, rotates to a bounded archive ring for these sinks, exposes config/env controls where users expect them, and documents the defaults alongside focused regression tests.

Yes. ## Solution Assessment
