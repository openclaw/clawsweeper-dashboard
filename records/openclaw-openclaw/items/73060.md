## Summary

Keep open. Current main still drops `threadId` on the direct Mattermost `handleAction` send path, and this PR is the focused fix; it still needs the required changelog entry and a stale fallback comment update before merge.

## What This Changes

The PR updates Mattermost message-tool send handling to use `threadId` as the fallback reply root between `replyToId` and legacy `replyTo`, with regression tests.

## Best Possible Solution

Land this focused Mattermost plugin fix after adding the Unreleased Fixes changelog entry and correcting the fallback comment, while leaving #67784 and #58439 to track the remaining media-path/local-roots work.

## Reproduction Assessment

Yes. Call `mattermostPlugin.actions.handleAction` with `action: "send"`, a Mattermost target, `message`, and `threadId` but no `replyToId`; current main forwards no reply root to `sendMessageMattermost`.

## Solution Assessment

Yes, with cleanup. The `replyToId -> threadId -> replyTo` order is the narrow maintainable fix because it preserves explicit reply precedence and matches the existing Mattermost sendText/sendMedia transport behavior.
