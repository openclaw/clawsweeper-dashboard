## Summary

Keep open. The patch addresses a real current-main Mattermost bug and remains a plausible focused fix, but it still needs real behavior proof before merge and has a stale fallback comment.

## What This Changes

The PR adds `params.threadId` as a normalized fallback in Mattermost `handleAction` sends and adds direct-send regression tests for trimming and `replyToId` precedence.

## Best Possible Solution

Land this focused Mattermost handler fix after correcting the comment and adding redacted real Mattermost proof, while keeping broader media-path work tracked in https://github.com/openclaw/openclaw/issues/67784 and https://github.com/openclaw/openclaw/pull/58439.

## Reproduction Assessment

Yes, at source level. On current main, call Mattermost `handleAction` with `action: "send"`, a Mattermost target, and `threadId` but no `replyToId`; the handler drops `threadId`, and the sender only threads when `replyToId` becomes Mattermost `rootId`.

## Solution Assessment

Yes, with small cleanup. Falling back `replyToId -> threadId -> replyTo` is the narrow maintainable fix for explicit thread targets and matches existing Mattermost `sendText`/`sendMedia` behavior.
