The PR updates Mattermost message-tool send handling to choose `replyToId`, then `threadId`, then legacy `replyTo` for the Mattermost reply root, with regression tests for trimmed `threadId` and `replyToId` precedence.

## Best Possible Solution

## Solution Assessment

Yes, with small maintenance fixes. The `replyToId ?? threadId ?? replyTo` order is the narrow maintainable behavior change because it matches existing Mattermost outbound helpers while preserving explicit `replyToId` precedence.
