## Summary

Keep open. Current main still has no supported Telegram parse-mode configuration at channel or account level, and the code, docs, and tests still treat outbound Telegram delivery as HTML with plain-text fallback; the related custom-emoji and spacing issues do not supersede this broader config request.

## What This Changes

Request to add Telegram parse-mode configuration so users can choose plain text or formatted Telegram output instead of the current fixed HTML behavior.

## Best Possible Solution

Add a Telegram-owned typed parse-mode setting with documented precedence, preserve current HTML behavior by default unless maintainers decide otherwise, and wire the resolved mode through sends, captions, edits, previews, delivery helpers, schema/types, metadata, docs, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection on current main gives high confidence that no supported parseMode config exists and that Telegram delivery still uses HTML parse mode in the main paths.

## Solution Assessment

Yes in direction, but not as an automatic repair. The maintainable path is a Telegram plugin config addition after maintainers choose supported modes, defaults, account precedence, and escaping/trust policy.
