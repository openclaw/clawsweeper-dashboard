## Summary

Keep open. Current main and the latest shipped release still lack a supported Telegram parse-mode config, while the Telegram delivery, preview, edit, caption, docs, and tests continue to assume HTML formatting with plain-text fallback.

## What This Changes

The issue requests a configurable Telegram parse mode for channel-level and optional per-account Telegram settings.

## Best Possible Solution

Add a Telegram-owned typed parse-mode setting with documented precedence, preserve current HTML behavior unless maintainers deliberately change the default, and wire the resolved mode through all outbound Telegram rendering paths with focused tests and docs.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection on current main and v2026.5.12 gives high confidence that no supported parseMode config exists and Telegram delivery still uses HTML parse mode in the main paths.

## Solution Assessment

Yes directionally, but not as an automatic repair. The maintainable path is a Telegram plugin config addition after maintainers choose supported modes, defaults, account precedence, and escaping/trust policy.
