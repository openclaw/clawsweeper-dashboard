## Summary

Keep open. Current main still lacks a supported Telegram parse-mode config at either the channel or account level, and the main send, caption, edit, draft-preview, and delivery-helper paths still resolve rendered Telegram output to HTML; the related custom-emoji and spacing issues are narrower follow-ups, not replacements for this config request.

## What This Changes

Request adds optional top-level and per-account Telegram parse-mode configuration so users can choose formatting behavior instead of relying on hardcoded Telegram HTML output.

## Best Possible Solution

Add a Telegram-owned typed parse-mode setting with documented channel/account precedence, wire the resolved mode through text sends, captions, edits, draft previews, and delivery helpers, then update schema, exported types, generated metadata/help, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection on current main gives high confidence that Telegram config has no parseMode field and delivery paths still send rendered output as HTML.

## Solution Assessment

Yes in direction, but not as an automatic repair. The maintainable solution is a Telegram plugin config addition after maintainers choose supported modes, defaults/backcompat behavior, account precedence, and escaping policy.
