## Summary

Keep open. Current main still has no supported Telegram parse-mode configuration at channel or account level, and the outbound Telegram paths still hardcode HTML with plain-text fallback; the related custom-emoji and spacing issues are adjacent but do not supersede this broader config request.

## What This Changes

Add optional Telegram parse-mode configuration at channel/account level instead of relying on one hardcoded outbound mode.

## Best Possible Solution

Add a Telegram-owned typed parse-mode setting with documented precedence, preserve current HTML behavior by default unless maintainers choose otherwise, and wire the resolved mode through sends, captions, edits, previews, delivery helpers, schema/types, metadata, docs, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection on current main gives high confidence that no supported parseMode config exists and that Telegram delivery still uses HTML parse mode in the main paths.

## Solution Assessment

Yes in direction, but not as an automatic repair. The maintainable path is a Telegram plugin config addition after maintainers choose supported modes, defaults, account precedence, and escaping/trust policy.
