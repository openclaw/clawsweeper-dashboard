## Summary

Keep open. Current main still has no supported Telegram parse-mode config at channel or account level, and the remaining work needs maintainer scoping around supported modes, defaults, precedence, and escaping behavior.

## What This Changes

Add an optional Telegram parse-mode configuration so Telegram output can be sent as plain text, HTML, Markdown, or another approved mode instead of always using the current HTML path.

## Best Possible Solution

Add a Telegram-owned typed parse-mode setting with explicit channel/account precedence, wire the resolved mode through all Telegram delivery surfaces, and document intentionally unsupported modes.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection gives high confidence that current main rejects or omits parseMode in the Telegram config contract and still sends rendered Telegram output as HTML.

## Solution Assessment

Yes in direction, but not as an automatic repair. The maintainable path is a Telegram plugin config addition after maintainers choose supported modes, defaults, account precedence, and escaping behavior.
