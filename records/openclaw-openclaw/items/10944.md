## Summary

Keep open. Current main still does not expose a supported Telegram parse-mode setting, while docs, tests, and delivery paths continue to hardcode HTML formatting with plain-text fallback; the related custom-emoji and spacing reports are adjacent but do not supersede this broader config request.

## What This Changes

Request to add configurable Telegram parse mode at channel/account level, including plain text and formatted modes with per-account customization.

## Best Possible Solution

Add a Telegram-owned typed parse-mode setting with documented precedence, preserve current HTML behavior by default unless maintainers decide otherwise, and wire the resolved mode consistently through config, docs, delivery, previews, captions, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection on current main gives high confidence that no supported parseMode config exists and that Telegram delivery still uses HTML parse mode in the main paths.

## Solution Assessment

Yes directionally, but not as an automatic repair. The maintainable path is a Telegram plugin config addition after maintainers choose supported modes, defaults, account precedence, and escaping/trust policy.
