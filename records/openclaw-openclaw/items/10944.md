## Summary

Keep open. Current main still has no supported Telegram parse-mode config at channel or account level, and the Telegram send, caption, edit, and preview paths still force HTML parse mode with plain-text fallback only after formatting failures.

## What This Changes

Request adds an optional Telegram parseMode config so users can choose plain text, Markdown, or HTML instead of hardcoded Telegram formatting.

## Best Possible Solution

Add a Telegram-owned typed parse-mode setting with clear channel/account precedence, wire the resolved mode through all Telegram delivery surfaces, and document intentionally unsupported modes.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection of current main gives high confidence that Telegram config cannot accept parseMode and delivery still defaults to HTML parse mode.

## Solution Assessment

Yes in direction, but not ready for an automatic repair. The maintainable path is a Telegram plugin config addition after maintainers choose supported modes, precedence, defaults, and escaping behavior.
