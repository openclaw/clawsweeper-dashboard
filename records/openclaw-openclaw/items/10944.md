## Summary

Keep open. Current main still has no supported Telegram parse-mode config while Telegram text, captions, draft previews, and lower-level delivery helpers still force HTML formatting; related custom-emoji and spacing issues are narrower or separately tracked.

## What This Changes

Add a configurable Telegram parse mode, optionally per account, so Telegram output is not limited to the current hardcoded HTML rendering path.

## Best Possible Solution

Add a Telegram-owned typed parse-mode setting with documented channel/account precedence, wire the resolved mode through every Telegram delivery surface, and update schema, types, generated metadata, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection gives high confidence that current main omits parseMode from the strict Telegram config contract and still sends rendered Telegram output as HTML.

## Solution Assessment

Yes in direction, but not as an automatic repair. The maintainable path is a Telegram plugin config addition after maintainers choose supported modes, default/backcompat behavior, account precedence, and escaping policy.
