## Summary

Keep open. Current main still lacks a supported Telegram parse-mode config, and the existing code/docs continue to hardcode Telegram HTML formatting with plain-text fallback only after HTML handling fails.

## What This Changes

The issue requests a configurable Telegram `parseMode` setting at channel and per-account scope so users can choose plain text, HTML, or Markdown-style Telegram formatting.

## Best Possible Solution

Add a typed Telegram plugin `parseMode` config that preserves current HTML behavior by default, define any per-account inheritance deliberately, and wire the setting through text sends, captions, edits/previews, delivery helpers, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection give high confidence that current main has no supported Telegram parse-mode config and still uses HTML formatting by default.

## Solution Assessment

Yes directionally, but not as an automatic narrow bug fix. The maintainable solution is a deliberate config/API addition that preserves existing HTML behavior unless maintainers choose a migration or opt-in alternative.
