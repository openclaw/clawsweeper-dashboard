## Summary

Keep open. Current `main` and the latest shipped tag still lack a supported Telegram parse-mode config, while Telegram send, caption, edit, preview, docs, and tests remain built around HTML parse mode with fallback behavior.

## What This Changes

The issue requests a Telegram `parseMode` configuration option, including optional per-account overrides, so users can choose plain text, Markdown-family formatting, or HTML behavior instead of the hardcoded Telegram mode.

## Best Possible Solution

Add a Telegram-owned typed parse-mode setting with documented precedence, preserve current HTML behavior unless maintainers deliberately choose a migration, and wire the resolved mode through all outbound Telegram rendering paths with tests and docs.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source, docs, and release inspection give high confidence that no supported Telegram `parseMode` config exists and the main paths still use HTML parse mode.

## Solution Assessment

Yes directionally, but not as an automatic repair. The maintainable path is a Telegram plugin config addition after maintainers choose supported modes, defaults, account precedence, and escaping/trust policy.
