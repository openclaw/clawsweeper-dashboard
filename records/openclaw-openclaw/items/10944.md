## Summary

Keep open. Current main has moved Telegram text delivery toward rich messages and preserves more formatting than the original report described, but it still does not provide a supported top-level or per-account Telegram parse-mode config, and the remaining behavior is a product/config decision rather than conservative cleanup.

## What This Changes

Request adds a configurable Telegram formatting mode at channel and per-account scope so users can choose plain, Markdown-style, or HTML-style behavior instead of the built-in Telegram formatting path.

## Best Possible Solution

Decide the supported Telegram formatting contract first, then implement only the chosen config/API/docs path across schema, typed config, account merging, sends, captions, edits, previews, and focused regression tests while preserving current rich-message defaults.

## Reproduction Assessment

Yes, by source inspection. The strict Telegram config schema and typed config lack parseMode, while current delivery paths only expose limited per-send HTML formatting and still force HTML for captions/edits; no live Telegram run was performed.

## Solution Assessment

Unclear. A parseMode config may still be a valid operator knob, but current main's rich-message path means maintainers should first choose whether the remaining fix is config, documentation, caption/edit behavior, or a narrower formatting API.
