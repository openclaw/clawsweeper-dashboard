## Summary

Keep open. Current main and the latest shipped release still lack a supported Telegram parse-mode config, while text sends, media captions, draft previews, and tests continue to assume Telegram HTML formatting with plain-text fallback only after HTML handling fails.

## What This Changes

The issue requests a configurable Telegram parse mode for Telegram channels, including a top-level setting and optional per-account override.

## Best Possible Solution

Add a typed Telegram plugin `parseMode` config that preserves current HTML behavior by default, define any per-account inheritance deliberately, and wire it consistently through text sends, captions, edits/previews, delivery helpers, docs, generated metadata, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, tests, and release inspection give high confidence that current main has no supported Telegram parse-mode config and still uses HTML formatting by default.

## Solution Assessment

Yes directionally, but not as an automatic narrow bug fix. The maintainable solution is a deliberate config/API addition that preserves existing HTML behavior unless maintainers choose a migration or opt-in alternative.
