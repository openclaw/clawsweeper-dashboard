## Summary

Keep open. Current main and the latest shipped tag still lack a supported Telegram parse-mode config, while the central Telegram send, caption, preview, and lower-level delivery paths continue to force HTML formatting with plain-text fallback behavior.

## What This Changes

Request to add a configurable Telegram parse mode at the channel and optional account level, covering plain text, Markdown, and HTML-style delivery.

## Best Possible Solution

Add a Telegram-owned typed parse-mode setting with documented precedence, preserve current HTML behavior unless maintainers deliberately choose a migration, and wire the resolved mode through text sends, captions, edits, previews, delivery helpers, docs, metadata, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source, docs, tests, and release inspection give high confidence that no supported Telegram parseMode config exists and the main paths still use HTML parse mode.

## Solution Assessment

Yes directionally, but not as an automatic repair. The maintainable path is a Telegram plugin config addition after maintainers choose supported modes, defaults, account precedence, and escaping/trust policy.
