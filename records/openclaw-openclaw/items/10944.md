## Summary

Keep open. Current main still has no supported Telegram parse-mode config at channel or account level, while docs, source, and tests continue to treat Telegram outbound delivery as HTML with fallback behavior; related custom-emoji and spacing items do not replace this broader config request.

## What This Changes

Request adding optional Telegram channel/account `parseMode` configuration so deployments can choose plain text, HTML, or Markdown-style formatting instead of the current fixed delivery mode.

## Best Possible Solution

Add a Telegram-owned typed parse-mode setting with documented channel/account precedence, preserve current HTML behavior by default unless maintainers choose otherwise, and wire the resolved mode through sends, captions, edits, draft previews, delivery helpers, schema/types, metadata, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection on current main gives high confidence that no supported `parseMode` config exists and that Telegram delivery still uses HTML parse mode in the main paths.

## Solution Assessment

Yes in direction, but not as an automatic repair. The maintainable path is a Telegram plugin config addition after maintainers choose supported modes, defaults, account precedence, and escaping/trust policy.
