## Summary

Keep open. Current main still lacks a supported Telegram parse-mode config, while Telegram send, caption, edit, preview, docs, and tests continue to assume HTML formatting with plain-text fallback only after parse failures.

## What This Changes

Request an optional Telegram parse-mode configuration, including possible top-level and per-account overrides, instead of hardcoded Telegram formatting.

## Best Possible Solution

Add a Telegram-owned typed parse-mode setting with documented precedence, preserve current HTML behavior unless maintainers deliberately choose a migration, and wire the resolved mode through all outbound Telegram rendering paths with tests and docs.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection on current main and v2026.5.12 gives high confidence that no supported Telegram parseMode config exists and the main paths still use HTML parse mode.

## Solution Assessment

Yes directionally, but not as an automatic repair. The maintainable path is a Telegram plugin config addition after maintainers choose supported modes, defaults, account precedence, and escaping/trust policy.
