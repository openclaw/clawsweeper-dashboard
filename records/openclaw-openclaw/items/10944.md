## Summary

Keep open. Current main and the latest shipped release still lack a supported Telegram parse-mode config, while the central Telegram send, caption, edit, preview, and delivery paths continue to force HTML; related custom-emoji and spacing threads are narrower, not canonical replacements.

## What This Changes

Request to add a supported Telegram `parseMode` config, optionally per account, for Telegram outbound rendering modes such as plain text, Markdown, or HTML.

## Best Possible Solution

Add a Telegram-owned typed parse-mode setting with documented precedence, preserve current HTML behavior unless maintainers deliberately choose a migration, and wire the resolved mode through sends, captions, edits, previews, delivery helpers, docs, metadata, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source, docs, related issue state, and release inspection give high confidence that no supported Telegram `parseMode` config exists today.

## Solution Assessment

Yes directionally, but not as an automatic repair. The maintainable path is a Telegram plugin config addition after maintainers choose supported modes, defaults, account precedence, migration behavior, and escaping/trust policy.
