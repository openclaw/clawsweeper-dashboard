## Summary

Keep open. Current main still lacks a supported Telegram parse-mode config at the channel or account level, and the send, caption, edit, and preview paths still force HTML formatting with only parse-failure fallback to plain text.

## What This Changes

The issue requests an optional Telegram channel/account parseMode config so operators can choose plain text, Markdown/MarkdownV2, or HTML instead of relying on forced HTML formatting.

## Best Possible Solution

Add a Telegram-owned typed parse-mode setting with channel and account resolution, then wire the resolved mode consistently through sends, captions, edits, and previews while documenting intentionally unsupported modes.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection gives high confidence that current main cannot accept parseMode in Telegram config and still sends rendered output as HTML.

## Solution Assessment

Yes in direction, but not ready for automatic repair. The maintainable path is a Telegram-owned config addition with consistent delivery wiring; supported modes and escaping semantics need maintainer scoping first.
