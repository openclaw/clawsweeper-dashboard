Keep open. Current main and the latest release still expose `richMessages` and internal HTML/rich formatting paths, but they do not provide the requested Telegram `parseMode` config at channel or account scope; supported modes, defaults, and interaction with rich messages still need maintainer direction.

## What This Changes

Request to add configurable Telegram message parse mode, including a possible top-level and per-account `parseMode` setting for plain text, Markdown, or HTML formatting.

## Best Possible Solution

Define one maintainer-approved Telegram formatting contract, then add typed config, docs, metadata, and tests that preserve current defaults while wiring the chosen behavior through sends, captions, edits, previews, and delivery helpers.

## Reproduction Assessment

## Solution Assessment

Unclear as proposed: a parse-mode knob is plausible, but supported values, defaults, per-account scope, and interaction with `richMessages` need maintainer direction. The maintainable solution is one documented formatting contract wired across all Telegram delivery paths.
