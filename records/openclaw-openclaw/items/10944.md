## Summary

Keep open. Current main still lacks a supported Telegram parse-mode config at either channel or account level, while the Telegram send, caption, edit, and preview paths continue to render and send HTML by default with only failure fallback to plain text.

## What This Changes

Request to add optional Telegram parse-mode configuration, including possible channel-level and per-account overrides, so Telegram output is not limited to one hardcoded parse mode.

## Best Possible Solution

Add a Telegram-owned typed parse-mode setting with clear channel/account precedence, wire the resolved mode through text sends, captions, edits, and previews, and document intentionally unsupported modes.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection on current main gives high confidence that Telegram config cannot accept parseMode and delivery still uses HTML parse mode by default.

## Solution Assessment

Yes in direction, but not ready for an automatic repair. The maintainable path is a Telegram plugin config addition with consistent delivery wiring, after maintainers choose the supported modes and safety semantics.
