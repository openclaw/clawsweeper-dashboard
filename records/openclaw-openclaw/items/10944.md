## Summary

Keep open. Current main still has no supported Telegram parse-mode config and the Telegram delivery paths still force HTML formatting; related custom-emoji and spacing items are narrower, so this remains a valid enhancement needing maintainer scoping.

## What This Changes

Request a Telegram channel/account parse-mode config so users can choose plain text, Markdown/MarkdownV2, or HTML behavior instead of a hardcoded mode.

## Best Possible Solution

Add a Telegram-owned typed parse-mode setting with documented channel/account precedence, wire the resolved mode through every Telegram delivery surface, and update schema, types, metadata, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection gives high confidence that current main omits parseMode from the strict Telegram config contract and still sends rendered Telegram output as HTML.

## Solution Assessment

Yes in direction, but not as an automatic repair. The maintainable path is a Telegram plugin config addition after maintainers choose supported modes, defaults, account precedence, and escaping behavior.
