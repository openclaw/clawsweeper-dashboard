## Label Justifications

- `P2`: This is a valid configured-path bug that can install skills into the wrong directory, but it is bounded to path resolution and not a core outage.
- `impact:data-loss`: The reported install writes skill files into an unexpected literal directory, leaving the intended workspace without the installed skill data.

## Summary

Keep open: current main and the latest release still preserve `${XDG_CONFIG_HOME}` as a literal path segment in the shared resolver used by skill installs, and an open linked implementation PR already targets the exact bug.

## What This Changes

The issue asks skill installation paths to expand `${XDG_CONFIG_HOME}` from the process environment instead of creating a literal `${XDG_CONFIG_HOME}` directory.

## Best Possible Solution

Land or replace the linked resolver fix so supported environment placeholders are expanded before workspace and skill install directories are resolved, with focused tests for the Docker/XDG install case.

## Reproduction Assessment

Yes, at source level. Current main sends `${XDG_CONFIG_HOME}/workspace` through `resolveHomeRelativePath` to `path.resolve(trimmed)` without substitution, and ClawHub appends `skills/<slug>` under that unresolved workspace path.

## Solution Assessment

Yes. Fixing the shared resolver is the narrow maintainable solution because workspace and ClawHub install paths already use it; a ClawHub-only patch would leave other configured path callers inconsistent.
