## Label Justifications

- `P2`: This is a valid configured-path bug that can install skills into the wrong directory, but it has limited blast radius and is not an outage or security incident.
- `impact:data-loss`: The issue can write installed skill files into an unexpected literal directory, making the skill unavailable from the configured workspace.

## Summary

Keep open: current main and the latest release still preserve `${XDG_CONFIG_HOME}` as a literal path segment in the shared resolver path used by ClawHub skill installs, and the exact fix path is already represented by an open implementation PR.

## What This Changes

The issue requests that ClawHub skill installation expand `${XDG_CONFIG_HOME}` instead of installing under a literal placeholder directory.

## Best Possible Solution

Land one focused shared resolver fix that expands supported env placeholders before filesystem resolution, preserves unresolved-placeholder passthrough, proves the ClawHub install path, and then close this issue.

## Reproduction Assessment

Yes, at source level. Current main sends `${XDG_CONFIG_HOME}/workspace` through `resolveHomeRelativePath` to `path.resolve(trimmed)` without substitution, and ClawHub appends `skills/<slug>` under that unresolved workspace path.

## Solution Assessment

Yes. Fixing the shared resolver is the narrow maintainable solution because workspace and ClawHub install paths already use it; a ClawHub-only patch would leave other configured path callers inconsistent.
