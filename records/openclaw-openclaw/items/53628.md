## Summary

Keep open. Current main still leaves environment placeholders literal in the shared path resolver used by configured workspaces and ClawHub skill installs, and open closing PRs #53718 and #58281 already target the same bug.

## What This Changes

The issue requests that ClawHub skill installs expand `${XDG_CONFIG_HOME}` in configured workspace paths instead of creating a literal `${XDG_CONFIG_HOME}` directory.

## Best Possible Solution

Land one canonical shared resolver fix that expands supported `$VAR` and `${VAR}` placeholders before path resolution while preserving existing tilde and unknown-placeholder behavior, with regression coverage for the resolver and the active-workspace ClawHub skill-install path.

## Reproduction Assessment

Yes. Current main gives a high-confidence static reproduction: a configured workspace like `${XDG_CONFIG_HOME}/workspace` flows through `resolveUserPath` unchanged and then becomes the base for `skills/<slug>`. I did not run a filesystem install because this was a read-only review.

## Solution Assessment

Yes. Fixing the shared path resolver is the narrowest maintainable direction because workspace resolution, config-derived paths, and skill install targets already use that helper; the issue should remain open until one canonical PR lands.
