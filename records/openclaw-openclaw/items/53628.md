## Summary

Keep open: current main still leaves environment placeholders such as `${XDG_CONFIG_HOME}` literal in shared user-path resolution, and the ClawHub skill install path still flows through that resolver before writing into `skills/`. An open PR also uses closing syntax for this issue, so the issue should remain open until maintainers land or close the canonical fix.

## What This Changes

The issue asks OpenClaw/ClawHub skill installation to expand `${XDG_CONFIG_HOME}` from the process environment instead of creating a literal `${XDG_CONFIG_HOME}` directory.

## Best Possible Solution

Land one canonical resolver fix that expands supported environment placeholders before path resolution, with regression coverage for `resolveHomeRelativePath`/`resolveUserPath` and the active-workspace ClawHub skill-install path; then confirm whether the separate ClawHub CLI needs a coordinated release before closing this as implemented.
