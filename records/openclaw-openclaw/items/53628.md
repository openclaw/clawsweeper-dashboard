## Summary

Keep open. Current main still routes configured workspace paths through a resolver that expands only `~`, so `${XDG_CONFIG_HOME}` remains literal before ClawHub skill installation; the issue is already paired with open closing PRs #53718 and #58281, so it should stay open until one canonical fix lands.

## What This Changes

Expand `${XDG_CONFIG_HOME}` and related environment placeholders in configured workspace paths before installing ClawHub skills.

## Best Possible Solution

Land one shared resolver fix that expands supported `$VAR` and `${VAR}` placeholders before path resolution while preserving existing tilde expansion and unknown-placeholder behavior, with regression coverage for the resolver and the active-workspace ClawHub skill-install path.

## Reproduction Assessment

Yes. A configured workspace such as `${XDG_CONFIG_HOME}/workspace` still flows through `resolveUserPath` to `path.resolve(trimmed)` without environment substitution, and a Node `path.resolve` check preserved the literal placeholder.

## Solution Assessment

Yes. Fixing the shared path resolver is the narrowest maintainable direction because workspace resolution, config-derived paths, and ClawHub skill install targets already share that helper; a one-off display or install-path patch would leave other path callers inconsistent.
