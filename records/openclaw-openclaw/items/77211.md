## Summary

Keep open: the PR reverses Telegram's documented final-only off-mode behavior, conflicts with the later open report asking for stronger suppression of internal Telegram tool/status cards, and still lacks live proof of the changed Telegram behavior.

## What This Changes

The PR changes Telegram dispatch to allow default tool-progress messages when preview streaming is off, adds preview-ordering guards after visible non-preview delivery, and adds an exact raw-target reply-dedupe shortcut.

## Best Possible Solution

Decide the precedence between Telegram off-mode/toolProgress settings and session `/verbose`, then align shared dispatch, Telegram wiring, docs, tests, and live Telegram proof around one policy.

## Reproduction Assessment

Yes. Source inspection shows current main sets suppression for Telegram off mode and the docs define that mode as final-only; the PR diff unsets the suppression flag, making the contract mismatch source-reproducible without a live run.

## Solution Assessment

No. The proposed suppression change is not the best fix as-is because it contradicts current docs and the later open Telegram leak report; maintainers first need to settle the off-mode versus verbose-progress policy.
