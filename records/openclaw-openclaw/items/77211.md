## Summary

Keep open. Current main documents and tests Telegram off mode as final-only, so this PR needs a maintainer policy decision before it can be treated as a mechanical bug fix; as written, it also leaves docs/changelog out of sync and has one weak regression assertion.

## What This Changes

The PR lets Telegram `streamMode: "off"` deliver default tool-progress messages, adds preview/finalization ordering state after visible payloads, adjusts Telegram dispatch tests, and adds an exact raw-target shortcut to messaging reply dedupe.

## Best Possible Solution

Choose the Telegram off-mode contract explicitly; if standalone progress is desired, align dispatch, regression tests, docs, and changelog, otherwise keep the final-only suppression contract and rework or close this PR.

## Reproduction Assessment

Yes. Source inspection shows current main suppresses default tool progress in Telegram `streamMode: "off"`, and the colocated regression test asserts that path; this is reproducible as a proposed policy change against the current contract, not as a failing bug under existing docs.

## Solution Assessment

No, not as-is. The runtime change is narrow, but it reverses a documented final-only policy without docs/changelog alignment and leaves the updated test able to pass without proving dispatch occurred.
