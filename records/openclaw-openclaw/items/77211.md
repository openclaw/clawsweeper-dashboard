## Summary

Keep open: the PR’s central behavior change would make Telegram `streaming.mode: "off"` send standalone default tool-progress messages, which conflicts with current docs and with the related open leak report asking for stronger suppression. The branch also lacks live proof of the changed Telegram off-mode tool-progress delivery and needs maintainer policy judgment before merge.

## What This Changes

The PR changes Telegram dispatch so `streaming.mode: "off"` no longer sets default tool-progress suppression, adds stale-preview ordering guards after visible non-preview delivery, and short-circuits exact follow-up reply target dedupe.

## Best Possible Solution

Decide the precedence between Telegram off-mode/tool-progress settings and session `/verbose`, then align dispatch, docs, tests, and live Telegram proof around that policy.

## Reproduction Assessment

Yes. Source inspection shows current main suppresses default tool progress for Telegram off mode and the docs define that mode as final-only; the PR would deliberately change that behavior rather than fix a clearly broken documented contract.

## Solution Assessment

No. The proposed suppression change is not the best fix as-is because it contradicts the current public contract and the related open leak report; maintainers should settle the policy before code repair or merge.
