## Summary

Keep this PR open for maintainer handling, but it is not merge-ready: the central Telegram change reverses the documented final-only behavior for `streaming.mode: "off"` and would risk exposing internal tool/status messages in the same configuration as the related privacy bug.

## What This Changes

The PR changes Telegram dispatch to stop setting `suppressDefaultToolProgressMessages` when preview streaming is off, adds preview-finalization ordering guards, short-circuits exact follow-up reply target dedupe matches, and updates Telegram tests.

## Best Possible Solution

Preserve final-only Telegram streaming-off suppression; if maintainers want visible progress with previews off, add an explicit documented opt-in with regression coverage and live Telegram proof rather than changing the default.

## Reproduction Assessment

Yes, at source level: the patch unsets Telegram default-progress suppression for `streamMode: "off"`, while current main and docs keep that mode final-only. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No. The safer solution is to keep streaming-off suppression and use the existing `progress` or preview tool-progress configuration for visible progress, or add a separately reviewed opt-in if maintainers want a new behavior.
