## Summary

This PR should close because current main and the shipped docs already chose the opposite Telegram behavior: `streaming.mode: "off"` is final-only and suppresses generic tool-progress chatter.

## What This Changes

The PR changes Telegram dispatch so `streamMode: "off"` no longer sets `suppressDefaultToolProgressMessages`, and updates the regression test to expect that flag to be absent.

## Best Possible Solution

Keep the shipped final-only Telegram implementation; any request to show non-preview progress while preview streaming is off should be a new product/config discussion, not a revert of the #72363 fix.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with `streamMode: "off"`, the PR leaves `suppressDefaultToolProgressMessages` undefined, and core delivery only suppresses generic progress when that flag is true.

## Solution Assessment

No. The patch is not the best solution because it reopens the Telegram non-streaming progress leak that current docs, tests, and the #72363 fix define as final-only behavior.
