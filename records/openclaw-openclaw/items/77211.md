## Summary

Keep open. The PR changes Telegram off-mode behavior against the current documented final-only contract, leaves docs/changelog out of sync, and still has a weak regression assertion noted in review.

## What This Changes

The PR changes Telegram dispatch so preview-off mode leaves default tool progress unsuppressed, adds preview-order tracking for visible payload finalization, updates Telegram tests, and adds an exact raw-target shortcut to messaging reply dedupe.

## Best Possible Solution

Keep the Telegram off-mode contract internally consistent; if maintainers choose to allow standalone progress in off mode, update code, tests, Telegram/shared streaming docs, and CHANGELOG together.

## Reproduction Assessment

Yes. From source inspection, current main suppresses default tool progress for Telegram `streamMode: "off"`, and the colocated regression test asserts that behavior; I did not run tests because this review is read-only.

## Solution Assessment

No, not as-is. The production change is narrow, but it reverses the documented final-only policy without docs/changelog updates and leaves a regression assertion that can pass without proving dispatch happened.
