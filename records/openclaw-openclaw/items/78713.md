## Summary

Keep open: current main and the latest release still use the explicit-only heartbeat policy, so the requested mixed defaults-plus-opt-out behavior remains unresolved. Because current docs and tests now codify the existing policy, changing it needs maintainer compatibility/product judgment rather than an automatic narrow bug fix.

## What This Changes

The issue requests per-agent heartbeat override semantics so agents without `agents.list[].heartbeat` inherit `agents.defaults.heartbeat` even when a sibling has explicit heartbeat config.

## Best Possible Solution

Maintainers should choose the intended heartbeat scope contract; if mixed defaults should work, centralize one enabled-agent policy across summary, scheduler, system prompt, ACP spawn, doctor diagnostics, docs, and regression tests.
