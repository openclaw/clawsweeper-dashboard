## Summary

Keep open: current main and the latest release still use explicit-only heartbeat scoping in mixed defaults-plus-explicit configs, so the reported behavior remains source-reproducible and needs a maintainer contract decision before a fix PR is safe.

## What This Changes

The issue requests per-agent heartbeat overrides to fall back to `agents.defaults.heartbeat` for agents without explicit heartbeat config instead of disabling those agents when any sibling has a heartbeat block.

## Best Possible Solution

Maintainers should choose the intended heartbeat scope contract; if mixed defaults should work, centralize one enabled-agent policy across summary, scheduler, system prompt, ACP spawn, doctor diagnostics, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection of current main and `v2026.6.10` shows the mixed defaults-plus-explicit config still disables the agent without its own heartbeat block, and the adjacent test encodes that behavior.

## Solution Assessment

Unclear. Per-agent override semantics would solve the reported workflow, but it changes the currently documented/tested heartbeat policy and needs maintainer compatibility approval before implementation.
