## Summary

Keep open: current main still contains the explicit-only heartbeat gates, and an open linked PR already uses closing syntax for this issue, so this should not be closed or duplicated before that PR is reviewed and either merged or closed.

## What This Changes

The issue asks for agents without per-agent heartbeat config to keep inheriting `agents.defaults.heartbeat` even when another agent has an explicit heartbeat override.

## Best Possible Solution

Review the linked heartbeat precedence PR, accept or reject the per-agent-over-default contract explicitly, then close this issue after the accepted implementation state is settled.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: the mixed defaults-plus-explicit config follows explicit-only branches in `heartbeat-summary.ts` and `heartbeat-runner.ts`, and the current test expects `main` to be disabled.

## Solution Assessment

Yes, if maintainers accept the config-contract change. Centralizing heartbeat enablement and applying per-agent-over-default semantics across scheduler, prompt, and ACP paths is the narrow maintainable solution; until the linked PR merges, this issue should remain open.
