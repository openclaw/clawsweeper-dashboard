## Summary

Keep open: current main and `v2026.6.8` still enforce explicit-only heartbeat scoping, so the mixed defaults-plus-opt-out behavior remains unresolved. Because docs, tests, prompt guidance, scheduler selection, and doctor diagnostics now share that contract, changing it needs maintainer compatibility/product judgment rather than an automated cleanup close.

## What This Changes

The issue requests standard per-agent heartbeat override semantics so `agents.defaults.heartbeat` still applies to agents without explicit heartbeat config when another agent has an override.

## Best Possible Solution

Maintainers should choose the intended heartbeat scope contract; if mixed defaults should work, centralize one enabled-agent policy across summary, scheduler, system prompt, ACP spawn, doctor diagnostics, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection of current main and `v2026.6.8` shows the mixed defaults-plus-explicit config still disables the agent without its own heartbeat block, and the adjacent test encodes that behavior.

## Solution Assessment

Unclear. Per-agent override semantics would solve the reported workflow, but it changes the currently documented/tested heartbeat policy and needs maintainer compatibility approval before implementation.
