## Summary

Keep open: current main still has the mixed defaults-plus-explicit heartbeat behavior, and https://github.com/openclaw/openclaw/pull/78718 is an open implementation candidate with closing syntax for this issue. The remaining decision is whether maintainers want to change the documented/tested heartbeat precedence contract or keep current semantics and handle diagnostics separately.

## What This Changes

Change heartbeat precedence so agents without per-agent heartbeat inherit `agents.defaults.heartbeat` even when another agent has an explicit heartbeat block.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/78718 if maintainers approve standard override semantics; otherwise keep explicit-only runtime behavior and use https://github.com/openclaw/openclaw/issues/46101 for diagnostic warnings.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the mixed defaults-plus-explicit config follows explicit-only gates in the summary, scheduler, system-prompt, and ACP paths, and the checked-in test expects the default-inheriting agent to be disabled.

## Solution Assessment

Unclear until maintainers decide the config contract. If standard override semantics are accepted, centralizing enablement and aligning the scheduler, prompt, ACP, docs, and tests is the narrow maintainable fix; if not, the best path is a doctor warning instead.
