## Summary

Keep open: current main still uses explicit-only heartbeat gates, and an open implementation PR with closing syntax is already paired with this issue, so it should remain open until that PR is merged or closed.

## What This Changes

Request to let agents without per-agent heartbeat config inherit agents.defaults.heartbeat even when another agent has an explicit heartbeat override.

## Best Possible Solution

Adopt one centralized heartbeat precedence contract across summary, scheduler, system-prompt, ACP spawn, docs, and tests, then close the canonical and duplicate reports after the accepted implementation merges.

## Reproduction Assessment

Yes. Source inspection shows the mixed defaults-plus-explicit config follows explicit-only branches in the summary and scheduler paths, and the checked-in test expects the default-inheriting agent to be disabled.

## Solution Assessment

Yes if maintainers accept the changed precedence contract. Centralizing enablement and using per-agent-over-default semantics across all heartbeat surfaces is the narrow maintainable direction; until the linked PR merges, this issue should stay open.
