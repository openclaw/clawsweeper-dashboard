## Summary

Keep open: current main and latest release still preserve stale restart-aborted subagent runs for recovery, and four open unmerged PRs already target this issue. The issue should stay open until maintainers land or reject a canonical fix path.

## What This Changes

The issue reports that restart orphan recovery can resume stale subagent runs after long downtime instead of aging them out under the existing stale-run liveness window.

## Best Possible Solution

Review and land one narrow linked PR that applies the existing stale-run liveness policy to restart-aborted recovery while preserving fresh restart recovery.

## Reproduction Assessment

Yes. Current main has source and test evidence: a three-hour-old unended run with `abortedLastRun: true` is expected to recover instead of being retired.

## Solution Assessment

Yes in direction. The maintainable fix is to reuse the existing stale-unended subagent liveness policy at restore/orphan recovery boundaries, with regression coverage for stale and fresh aborted runs.
