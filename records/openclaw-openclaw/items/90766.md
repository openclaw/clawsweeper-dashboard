## Summary

Keep open: current main and the latest release still preserve stale `abortedLastRun` subagent runs for restart recovery, and the linked open PRs are implementation candidates rather than a reason to close before merge.

## What This Changes

The issue asks for restart-time orphan recovery to retire stale `abortedLastRun` subagent runs instead of automatically resuming them after long downtime.

## Best Possible Solution

Land one focused linked PR that reuses the existing stale-unended subagent liveness cutoff in both restore reconciliation and orphan recovery, then close this issue after merge.

## Reproduction Assessment

Yes, source-reproducible: current main has a test that keeps a three-hour-old `abortedLastRun` subagent run recoverable and calls `agent.wait`, and the runtime path has no stale-age gate before recovery. I did not run a live multi-hour gateway downtime repro in this read-only sweep.

## Solution Assessment

Yes, the best direction is to reuse the existing stale-unended liveness policy at the restore/orphan-recovery boundary without adding config or changing fresh restart recovery. Current main has not implemented that yet, but the linked open PRs are aimed at that repair.
