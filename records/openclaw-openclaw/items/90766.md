## What This Changes

The issue reports that restart-time orphan recovery can resume subagent runs older than the existing stale-run window instead of retiring them as stale.

## Best Possible Solution

## Reproduction Assessment

Yes at source level. Current main has both implementation and test coverage showing a three-hour-old `abortedLastRun` subagent run remains eligible for restart recovery instead of stale pruning.

## Solution Assessment

Yes in direction. The narrow maintainable fix is to apply the existing stale-run liveness policy to restart-aborted restore and orphan-recovery paths while preserving fresh restart recovery.
