## Summary

Keep open: current main and v2026.6.8 still allow stale restart-aborted subagent runs to bypass stale pruning and proceed toward orphan recovery, while four open linked PRs already target the fix.

## What This Changes

The issue reports that stale restart-aborted subagent runs can be treated as recoverable after downtime longer than the stale-run window.

## Best Possible Solution

Review and land one narrow linked PR that applies the existing subagent stale-liveness policy at the restore and orphan-recovery boundaries, with regression coverage and docs aligned to the new behavior.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: a 3-hour-old restored run with `abortedLastRun: true` is expected by the test suite to call `agent.wait` and remain recoverable.

## Solution Assessment

Yes in direction. The maintainable fix is to reuse the existing stale-run liveness policy for restart-aborted restore and orphan recovery, while explicitly preserving fresh restart recovery.
