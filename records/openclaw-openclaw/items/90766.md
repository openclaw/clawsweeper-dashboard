## Summary

Keep open: current main and v2026.6.8 still preserve stale restart-aborted subagent runs as recoverable, and four open linked PRs already target the fix.

## What This Changes

This issue requests aging out stale `abortedLastRun` subagent runs during restart orphan recovery instead of automatically resuming them after long downtime.

## Best Possible Solution

Land one narrow fix that reuses the existing subagent liveness policy at both restore reconciliation and orphan recovery, preserves fresh aborted recovery, and adds regression coverage for stale versus fresh cases.

## Reproduction Assessment

Yes, at source level: current main has a test that keeps a 3-hour-old `abortedLastRun` restored run recoverable, and the recovery code has no stale age gate before resume. I did not run tests because this review is read-only.

## Solution Assessment

No for current main: it still implements the stale-aborted exemption. The best fix is the narrow reuse of existing liveness policy proposed by the linked PRs, with coverage proving stale runs retire and fresh aborted runs still recover.
