## Summary

Keep open: current main and v2026.6.8 still exempt stale `abortedLastRun` subagent runs from stale pruning and can resume them, while four unmerged linked PRs already target the fix.

## What This Changes

The issue requests that restart-time subagent orphan recovery age out stale aborted child runs instead of automatically replaying or resuming them after long downtime.

## Best Possible Solution

Review and land one narrow linked PR that reuses the existing stale-run liveness policy at restore and orphan-recovery boundaries while preserving fresh restart recovery.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: a three-hour-old restored run with `abortedLastRun: true` is expected by the test suite to call `agent.wait` and remain recoverable.

## Solution Assessment

Yes in direction. The maintainable fix is to apply the existing subagent stale-liveness policy to restart-aborted restore/orphan recovery, with coverage for stale and fresh aborted runs.
