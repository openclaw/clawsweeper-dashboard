## What This Changes

The issue asks for orphan recovery to age out stale restart-aborted subagent runs instead of replaying them after downtime beyond the stale-run window.

## Best Possible Solution

Review and land one narrow linked PR that reuses the existing stale-run liveness policy for restart-aborted recovery while preserving fresh recovery and adding regression coverage.

## Reproduction Assessment

Yes, at source level. Current main has a test that creates a three-hour-old `abortedLastRun` restored subagent run and expects it to call `agent.wait` and remain listed, matching the reported stale resurrection path.

## Solution Assessment

Yes in direction. The maintainable fix is to reuse the existing stale-unended subagent liveness policy at restore/orphan-recovery boundaries, with tests proving stale aborted runs retire and fresh aborted runs still recover.
