## Summary

Keep open: current main and the latest release still lack the requested staged model test/promote/rollback workflow; the existing model CLI provides immediate setting, manual fallbacks, auth probes, OpenRouter scan, and runtime failover primitives, but not the safer migration flow.

## What This Changes

Request a staged model migration UX that can add a candidate as fallback, verify it, promote it to primary, preserve the old primary as fallback, and roll back if needed.

## Best Possible Solution

Design a provider-generic `openclaw models` migration flow that stages a candidate, verifies real agent and sub-agent execution before promotion, preserves the old primary as fallback, stores rollback metadata, documents config transitions, and coordinates the fallback-chain test scope with https://github.com/openclaw/openclaw/issues/6599.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, live issue state, and latest-release checks show no staged migrate/test/promote/rollback surface on current main or v2026.5.22.

## Solution Assessment

Unclear until maintainers choose the command UX and verification depth. The maintainable path is to build on existing model config, fallback, auth-probe, and scan primitives rather than adding a provider-specific partial workflow.
