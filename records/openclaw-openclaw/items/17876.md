## Summary

Keep open. Current main still lacks the requested staged model migration workflow; existing model commands support immediate primary changes, manual fallback management, OpenRouter scan/probe, and status auth probes, but not generic candidate staging, verification before promotion, automatic old-primary fallback preservation, or rollback state.

## What This Changes

Request a staged `openclaw models` migration workflow that tests a new model before promotion, keeps the previous primary as fallback, and supports rollback.

## Best Possible Solution

Add a maintainer-designed `openclaw models` migration flow that stages an arbitrary candidate, verifies it before promotion, preserves the old primary as fallback, persists rollback state, and documents/tests the config transitions.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, and issue-context inspection on current main confirm the requested migrate/test/promote/rollback workflow is not present.

## Solution Assessment

Unclear until maintainers define the product contract. The maintainable direction is to reuse existing model config, fallback, and auth-probe primitives instead of stretching the OpenRouter-specific `models scan` path into a generic migration workflow.
