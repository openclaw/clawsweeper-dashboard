## Summary

Keep open: current main and v2026.5.22 still lack the requested staged model test/promote/rollback workflow; existing model commands provide immediate setting, fallback list management, auth probes, OpenRouter scan, and runtime failover primitives, but not a provider-generic migration flow.

## What This Changes

Request a safer model-switching workflow that stages a candidate model as a fallback, verifies it before promotion, preserves the old primary as fallback, and supports rollback.

## Best Possible Solution

Design a provider-generic `openclaw models` migration flow that stages a candidate, verifies real agent and sub-agent execution before promotion, preserves the old primary as fallback, stores rollback metadata, and documents the config transitions.
