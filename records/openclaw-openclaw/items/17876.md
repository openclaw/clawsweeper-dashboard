## Summary

Keep open. Current main still lacks the requested staged model migration workflow, and maintainer discussion already says this needs follow-up rather than cleanup closure.

## What This Changes

Request a provider-generic `openclaw models` workflow to stage a new model, verify it before promotion, preserve the old primary as fallback, and support rollback.

## Best Possible Solution

Add a maintainer-designed `openclaw models` migration flow that stages any provider/model candidate, verifies intended agent surfaces before promotion, preserves the previous primary in the fallback chain, stores rollback state, and updates docs/tests for those config transitions.

## Solution Assessment

Unclear until maintainers define the product contract. The maintainable direction is a provider-generic workflow built around existing set, fallback, status/probe, scan, and runtime failover primitives, not an OpenRouter-only scan extension or rollback-only tweak to `models set`.
