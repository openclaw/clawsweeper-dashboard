## Summary

Keep open: current main and the latest release still provide immediate model setting, manual fallback management, auth-profile probes, and OpenRouter scan support, but not the requested staged model migration workflow with verification-before-promotion and rollback.

## What This Changes

Request a staged model-switching workflow that can test a candidate model as fallback, promote it only after verification, preserve the old primary, and roll back if needed.

## Best Possible Solution

Design one coherent `openclaw models` migration flow that stages arbitrary candidates, verifies them before promotion, preserves the old primary as fallback, persists rollback state in the appropriate canonical store, and documents/tests the transitions.

## Reproduction Assessment

Yes, at source level. Current main exposes immediate model setting, manual fallback management, OpenRouter scan/probe support, and auth-profile probes, but no staged migrate/test/promote/rollback workflow.

## Solution Assessment

Unclear until maintainers choose the product contract. A staged migration workflow is plausible, but it should be reconciled with the fallback-chain test and all-configured-model probe issues before implementation is queued.
