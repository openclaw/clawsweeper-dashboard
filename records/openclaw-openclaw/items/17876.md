## Summary

Keep open. Current main still lacks the requested staged model migration workflow, and the linked fallback-test and per-model health-check items cover narrower probe surfaces rather than candidate staging, verification-before-promotion, old-primary preservation, and rollback.

## What This Changes

Request a staged model migration UX that tests a candidate model before promotion, preserves the old primary as fallback, and supports rollback.

## Best Possible Solution

Add a maintainer-designed `openclaw models` migration workflow that stages an arbitrary candidate, verifies it before promotion, preserves the old primary as fallback, persists rollback state, and documents/tests the config transitions.

## Solution Assessment
