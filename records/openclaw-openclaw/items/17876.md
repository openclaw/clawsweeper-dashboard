## Summary

Keep open. Current main still lacks the requested staged, provider-generic model migration workflow; the existing surface covers status/probe, immediate primary setting, manual fallback edits, OpenRouter scanning, and runtime failover, and the discussion includes explicit maintainer guidance to keep this open.

## What This Changes

Request a staged model migration workflow that adds a candidate as fallback, verifies it, promotes it to primary, preserves the old primary as fallback, and supports rollback.

## Best Possible Solution

Add a maintainer-designed `openclaw models` workflow that stages any provider/model candidate, verifies intended agent surfaces before promotion, preserves the previous primary in the fallback chain, persists rollback state, and documents/tests the config transitions.

## Reproduction Assessment

Not applicable as a feature request rather than a bug report. Current-main source and docs inspection gives a high-confidence verification path: no staged test/migrate/promote/rollback workflow exists.

## Solution Assessment

Unclear until maintainers define the product contract. The maintainable direction is a provider-generic workflow built around existing model status/probe, set, fallback, scan, and runtime failover primitives, not an OpenRouter-only scan extension or a rollback-only tweak.
