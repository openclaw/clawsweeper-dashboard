## Summary

Keep open. Current main still has immediate default-model setting, manual fallback management, OpenRouter-specific scan, auth-profile probes, and runtime failover, but it does not provide the requested staged test/promote/rollback model migration workflow; the live discussion also contains explicit keep-open guidance.

## What This Changes

Request a staged model-switching workflow that can test a candidate model, promote it after verification, preserve the old primary as fallback, and roll back if the new model fails.

## Best Possible Solution

Design a provider-generic `openclaw models` migration flow that stages a candidate, verifies real agent and sub-agent execution before promotion, preserves the old primary as fallback, stores rollback metadata, and coordinates with the narrower fallback-chain probe tracked at https://github.com/openclaw/openclaw/issues/6599.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs make the absence of staged migrate/test/promote/rollback behavior high-confidence.

## Solution Assessment

Unclear until maintainers define the command UX and verification depth. The maintainable direction is to build on existing model config, fallback, auth-probe, and scan primitives rather than adding a provider-specific partial workflow.
