## Summary

Keep open. Current main still lacks the requested staged test/promote/rollback model migration workflow; existing model set, fallback, status-probe, OpenRouter scan, and runtime failover paths are useful building blocks but do not provide the requested provider-generic migration UX, and the live issue discussion already contains explicit keep-open product-design guidance.

## What This Changes

The issue requests a staged model migration workflow that can test a candidate model before promotion, preserve the old primary as fallback, and roll back safely.

## Best Possible Solution

Design and ship a documented provider-generic `openclaw models` migration flow that stages arbitrary candidates, verifies real execution before promotion, preserves the previous primary as fallback, persists rollback state, and coordinates the fallback-chain probe surface with https://github.com/openclaw/openclaw/issues/6599.

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection on current main gives high-confidence proof that the requested staged test/promote/rollback workflow is not present.

## Solution Assessment
