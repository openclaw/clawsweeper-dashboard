## Summary

Keep open. Current main still has immediate default-model setting, manual fallback management, OpenRouter scan, auth-profile probes, and runtime failover, but it does not provide the requested staged test/promote/rollback model migration workflow; the live discussion also contains explicit keep-open guidance.

## What This Changes

The issue requests a staged model migration workflow that tests a candidate model before promotion, keeps fallback safety, and supports rollback after a bad model change.

## Best Possible Solution

Design a provider-generic `openclaw models` migration flow that stages a candidate, verifies real agent and sub-agent execution before promotion, preserves the old primary as fallback, stores rollback metadata, and coordinates with the narrower fallback-chain probe tracked at https://github.com/openclaw/openclaw/issues/6599.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Current source and docs make the absence of staged migrate/test/promote/rollback behavior high-confidence.

## Solution Assessment

Unclear until maintainers define the command UX and verification depth. The maintainable direction is to build on existing model config, fallback, auth-probe, and scan primitives rather than adding a provider-specific partial workflow.
