## Summary

Keep open. Current main still lacks the requested staged model migration workflow, and the issue has explicit keep-open guidance; existing model set, fallback, scan, and probe surfaces are useful pieces but do not provide candidate staging, verify-before-promotion, old-primary preservation, or rollback.

## What This Changes

The issue requests a staged model migration workflow that can test a candidate model before promotion, keep the old primary as fallback, and support rollback.

## Best Possible Solution

Add a maintainer-designed `openclaw models` migration workflow that stages an arbitrary candidate, verifies it before promotion, preserves the old primary as fallback, persists rollback state, and documents/tests the config transitions.

## Solution Assessment

Unclear until maintainers settle the command shape and rollback-state contract. The maintainable path should reuse generic model/auth/fallback resolution rather than building an OpenRouter-specific migration flow.
