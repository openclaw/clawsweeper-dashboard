## Summary

Keep open. Current main still lacks the requested staged, provider-generic model migration workflow, and the linked fallback-test issue is narrower than this issue’s promotion and rollback scope.

## What This Changes

Request a staged model migration workflow that tests a candidate model before promotion, preserves the old primary as fallback, and supports rollback.

## Best Possible Solution

Define and implement a maintainer-designed `openclaw models` migration workflow that stages any provider/model candidate, verifies intended agent surfaces before promotion, preserves the old primary in the fallback chain, persists rollback state, and documents/tests the config transitions.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, live issue discussion, and related-thread inspection give a high-confidence verification path that the staged test/migrate/promote/rollback workflow is absent on current main.

## Solution Assessment

Unclear until maintainers define the product contract. The maintainable direction is a provider-generic workflow built on existing set, fallback, status/probe, scan, auth, and runtime failover primitives rather than an OpenRouter-only scan extension or rollback-only tweak.
