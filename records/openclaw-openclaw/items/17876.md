## Summary

Keep open. Current main still provides immediate primary model setting, manual fallback management, OpenRouter scan/probe, and auth-profile probes, but not staged candidate testing, promotion that preserves the old primary as fallback, or rollback; the April 26 maintainer review also explicitly kept this feature open for product follow-up.

## What This Changes

The issue requests a staged model migration workflow that tests a new model as fallback before promoting it to primary, with rollback support.

## Best Possible Solution

Add a maintainer-designed `openclaw models` migration workflow that stages an arbitrary candidate, verifies it before promotion, preserves the old primary as fallback, persists rollback state, and documents/tests the config transitions.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection on current main confirm the staged migrate/test/promote/rollback workflow is not present.

## Solution Assessment

Unclear until maintainers settle the product contract. The maintainable direction is to reuse existing model config, fallback, and auth-probe primitives rather than building a provider-specific workflow around `models scan`.
