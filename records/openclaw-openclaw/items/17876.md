## Summary

Current main and the latest release still provide immediate primary-model updates, manual fallback management, and live auth-profile probes, but not the requested staged verification, promotion, previous-primary preservation, and rollback lifecycle. The narrower related issues remain open and do not supersede this broader product request; explicit prior maintainer guidance also supports keeping it open for design.

## What This Changes

Request a staged model migration workflow that tests a candidate model before promotion, preserves the old primary as fallback, and supports rollback.

## Best Possible Solution

Define one provider-generic `openclaw models` migration lifecycle that uses canonical model resolution and config mutation, exposes explicit verification levels, atomically promotes the candidate while preserving intentional fallback order, and stores any rollback record in the appropriate SQLite state owner.
