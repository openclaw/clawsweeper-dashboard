## Summary

Keep open. Current main still has immediate model setting, manual fallback management, auth/status probes, and OpenRouter scan helpers, but it does not provide the requested staged test/promote/rollback model migration workflow; the discussion also includes explicit keep-open guidance and the related fallback-test issue is only a narrower piece of the requested flow.

## What This Changes

Add a staged model migration workflow that can test a candidate model before promotion, preserve the old primary as fallback, and roll back a failed migration.

## Best Possible Solution

Design and ship a documented models migration flow that stages arbitrary candidates, verifies real execution paths before promotion, preserves the previous primary as fallback, persists rollback state, and explains how it relates to fallback-chain probes.

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection on current main confirm the requested staged test/promote/rollback workflow is not present.

## Solution Assessment

Unclear as an implementation until maintainers define the command UX and verification depth. The maintainable direction is a provider-generic workflow that reuses existing model config, auth, and fallback primitives rather than adding a provider-specific partial migration path.
