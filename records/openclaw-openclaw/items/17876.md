## Summary

Keep open. Current main still lacks the requested staged model migration workflow: it has immediate default-model setting, manual fallback management, OpenRouter-specific scan/probe support, auth-profile probes, and runtime failover, but not generic candidate staging, verification-before-promotion, old-primary preservation, or rollback.

## What This Changes

Requests a staged model migration workflow that tests a candidate model as a fallback before promotion, preserves the previous primary as fallback, and supports rollback.

## Best Possible Solution

Define and implement a provider-generic openclaw models migration flow that stages a candidate, verifies the intended agent surfaces, promotes only after success, preserves the old primary as fallback, persists rollback state, and documents/tests the config transitions.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, tests search, and live issue context give a high-confidence verification path that staged test/migrate/promote/rollback behavior is absent on current main.

## Solution Assessment

Unclear until maintainers define the product contract. The maintainable direction is a provider-generic migration workflow built on the existing set, fallback, status/probe, auth, scan, and runtime failover primitives rather than an OpenRouter-only or probe-only patch.
