## Summary

Keep open: current main and v2026.5.20 still provide immediate model setting, auth probes, manual fallback management, OpenRouter scan, and runtime failover, but not the requested staged test/promote/rollback migration workflow.

## What This Changes

The issue requests a staged model migration workflow with candidate testing, promotion, fallback preservation, and rollback instead of immediate primary-model replacement.

## Best Possible Solution

Design a provider-generic openclaw models migration flow that stages a candidate, verifies real agent and sub-agent execution before promotion, preserves the old primary as fallback, stores rollback metadata, and coordinates with https://github.com/openclaw/openclaw/issues/6599.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, related issue, and release checks show no staged migrate/test/promote/rollback surface on current main or v2026.5.20.

## Solution Assessment

Unclear until maintainers choose the command UX and verification depth. The maintainable path is to build on existing model config, fallback, auth-probe, and scan primitives rather than adding a provider-specific partial workflow.
