## Summary

Keep open: current main and v2026.6.10 still expose immediate model setting, manual fallback management, auth/profile probing, and OpenRouter scan support, but not the requested staged migrate/test/promote/rollback workflow. The related open issues cover narrower fallback-chain and per-model probe pieces, so they should inform the design rather than replace this broader workflow request.

## What This Changes

Add a staged model migration UX that tests a candidate model before promotion, preserves the old primary as fallback, and supports rollback.

## Best Possible Solution

Design one coherent `openclaw models` migration workflow that stages arbitrary candidate models, verifies them before promotion, preserves the old primary as fallback, persists rollback state in the appropriate canonical store, and documents/tests the config transitions.
