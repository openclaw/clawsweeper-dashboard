## Summary

Keep open. Current main still lacks the requested staged model test/promote/rollback workflow; it only has immediate primary setting, manual fallback management, auth/profile probes, OpenRouter scan, and runtime failover. The related fallback-test issue at https://github.com/openclaw/openclaw/issues/6599 and per-model health-check issue/PR at https://github.com/openclaw/openclaw/issues/63145 / https://github.com/openclaw/openclaw/pull/63259 cover narrower pieces, not this full migration workflow.

## What This Changes

Request a staged model migration workflow that tests a candidate model as a fallback, promotes it after verification, preserves the old primary, and supports rollback.

## Best Possible Solution

Define and ship a provider-generic `openclaw models` migration flow that stages a candidate, verifies it before promotion, preserves the old primary as fallback, stores rollback state, and documents/tests config transitions.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection of current main confirms no staged migrate/test/promote/rollback workflow or rollback state exists.

## Solution Assessment

Unclear until maintainers settle the product contract. The maintainable direction is to build on existing model config, fallback, auth-probe, and runtime failover primitives rather than making the workflow OpenRouter-specific.
