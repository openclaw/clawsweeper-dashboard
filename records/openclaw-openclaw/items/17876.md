## What This Changes

The issue requests a staged model migration workflow that tests a candidate model before promotion, preserves the old primary as fallback, and can roll back a failed model change.

## Best Possible Solution

Design one coherent `openclaw models` migration flow that stages arbitrary candidate models, verifies them before promotion, preserves the old primary as fallback, persists rollback state in the appropriate canonical store, and documents/tests the config transitions.
