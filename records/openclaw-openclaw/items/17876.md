Requests a staged model migration workflow so users can add a new model as fallback, verify it, promote it to primary, and roll back safely.

## Best Possible Solution

Design a provider-generic `openclaw models` migration flow that reuses existing config, fallback, scan, and auth-probe primitives while coordinating the fallback-chain probe subset with https://github.com/openclaw/openclaw/issues/6599.
