## Summary

Keep open. Current main still lacks the staged, provider-generic model migration workflow, and the live issue discussion includes explicit maintainer keep-open guidance; related fallback-test and per-model probe work covers only pieces of the request.

## What This Changes

The issue requests a staged model migration workflow that can test a candidate model before promotion, preserve the old primary as fallback, and roll back after promotion.

## Best Possible Solution

Define and implement a maintainer-designed `openclaw models` migration flow that stages any provider/model candidate, verifies intended agent surfaces before promotion, preserves the old primary in the fallback chain, persists rollback state, and documents/tests the config transitions.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, related-thread, and live discussion inspection give a high-confidence verification path that staged test/migrate/promote/rollback behavior is absent on current main.

## Solution Assessment

Unclear until maintainers define the product contract. The maintainable direction is a provider-generic migration workflow built on existing set, fallback, status/probe, scan, auth, and runtime failover primitives rather than an OpenRouter-only scan extension or fallback-test-only tweak.
