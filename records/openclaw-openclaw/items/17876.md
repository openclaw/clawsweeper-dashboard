## What This Changes

Request a safer model-switching workflow that stages a candidate model, verifies it before promotion, preserves the old primary as fallback, and supports rollback.

## Best Possible Solution

Design a provider-generic `openclaw models` migration flow that stages a candidate, verifies real agent and sub-agent execution before promotion, preserves the old primary as fallback, stores rollback metadata, and documents the config transitions.
