## Summary

Keep open: the branch is a plausible slice of the reset-archive recovery work, but it does not fix the advertised `chat.history` or `sessions_history` production path because those call the async transcript readers, not the synchronous readers changed here. It also still lacks external real behavior proof.

## What This Changes

The PR adds reset-archive discovery and uses it as a fallback for synchronous session message and title-field reads when the active transcript file is missing.

## Best Possible Solution

Route reset-archive fallback through a shared active-or-latest-reset resolver used by sync and async history, stats, and title paths, coordinated with https://github.com/openclaw/openclaw/pull/76134 and backed by redacted real Gateway/session-history proof.

## Reproduction Assessment

Yes, at source level. With only a `<sessionId>.jsonl.reset.<timestamp>` archive and no active `<sessionId>.jsonl`, current main's async `chat.history` path returns `[]` because `readRecentSessionMessagesAsync` checks only active transcript candidates.

## Solution Assessment

No. The helper is reasonable, but wiring it only into synchronous readers is not the narrow maintainable fix for the advertised behavior because the production history surfaces use async readers.
