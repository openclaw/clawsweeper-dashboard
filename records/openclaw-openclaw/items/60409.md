## Summary

Keep open: this is a plausible reset-archive recovery fix, but the patch still misses the production async paths used by `chat.history`, `sessions_history`, and async session-list title hydration, and the external PR currently has only test/CI proof rather than real behavior proof.

## What This Changes

The PR adds a reset-archive lookup helper, wires it into synchronous session-message and title-field reads, and adds focused gateway session-utils tests for archive fallback behavior.

## Best Possible Solution

Route reset-archive fallback through the shared transcript-path resolution used by sync and async history readers, count/stats, and title hydration, then coordinate with the narrower async fix in https://github.com/openclaw/openclaw/pull/76134.

## Reproduction Assessment

Yes, from source inspection. On current main, an archive-only `<sessionId>.jsonl.reset.<timestamp>` with no active `<sessionId>.jsonl` makes `chat.history` reach `readRecentSessionMessagesAsync`, whose active-only path returns `[]`.

## Solution Assessment

No. The helper is reasonable, but this PR wires it only into synchronous readers while the production history and title-hydration surfaces use async readers; a shared active-or-reset resolver is the narrower maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

- **[P2] Use the reset fallback in the async history readers:** `src/gateway/session-utils.fs.ts:101-113`
