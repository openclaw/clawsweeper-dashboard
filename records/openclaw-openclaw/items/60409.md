## Summary

Keep open: this PR is a focused archive-recovery attempt, but it still misses the default user-facing `chat.history` and `sessions_history` paths because current main routes those through async readers that the branch does not change; it also lacks external real behavior proof before merge.

## What This Changes

The branch adds a reset-archive lookup helper and wires it into synchronous session message and title-field reads with focused gateway session-utils tests.

## Best Possible Solution

Route reset-archive fallback through a shared active-or-latest-reset resolver used by sync and async history, stats, and title paths, coordinating with https://github.com/openclaw/openclaw/pull/76134 and requiring redacted real Gateway/session-history proof before merge.

## Reproduction Assessment

Yes, at source level. An archive-only `<sessionId>.jsonl.reset.<timestamp>` with no active `<sessionId>.jsonl` still makes current main's async `chat.history` path return `[]` because `readRecentSessionMessagesAsync` only checks active transcript candidates.

## Solution Assessment

No. The helper is reasonable, but this PR wires it into synchronous readers while the production history and async title/list surfaces use async readers; a shared active-or-reset resolver is the narrower maintainable fix.
