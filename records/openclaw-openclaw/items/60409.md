## Summary

Keep open: the PR is a plausible reset-archive fix, but its fallback is still wired through sync readers while the advertised `chat.history` and `sessions_history` surfaces use async readers on current main, so archive-only sessions would still return empty history through the production path.

## What This Changes

The PR adds reset-archive lookup helpers and tests, wiring sync `readSessionMessages` and sync title derivation to read the newest `.reset.*` archive when the active transcript is absent.

## Best Possible Solution

Use one shared active-or-latest-reset archive resolver for the sync and async history/title/count paths, or coordinate this branch with https://github.com/openclaw/openclaw/pull/76134, then require redacted real Gateway proof before merge.

## Reproduction Assessment

Yes, at source level: with only a `<sessionId>.jsonl.reset.<timestamp>` archive and no active `<sessionId>.jsonl`, current main's `chat.history` calls `readRecentSessionMessagesAsync`, which returns `[]` when `findExistingTranscriptPath` finds no active candidate.

## Solution Assessment

No. The helper is useful, but wiring it only into sync readers does not fix the advertised `chat.history` and `sessions_history` surfaces; the maintainable fix needs the async readers too, or coordination with https://github.com/openclaw/openclaw/pull/76134.
