## Summary

Keep open. Current main still follows the reported Discord DM path: the auto-thread plan creates a channel-shaped reply target, DMs have no auto-thread context, and `ctx.To` is finalized from that channel-shaped target before the user-shaped DM conversation target is computed. The focused related fix PR is closed unmerged, so this remains narrow, actionable Discord plugin work.

## What This Changes

The issue asks for Discord inbound DMs to set semantic `ctx.To` to `user:<userId>` while preserving channel-shaped Discord API delivery targets.

## Best Possible Solution

Make an extension-local Discord message-handler change that computes `dmConversationTarget` before `effectiveTo`, uses it as the DM fallback for semantic inbound `ctx.To`, leaves Discord API delivery targets channel-shaped, and adds a focused regression test asserting `To`, `From`, and `ChatType` for inbound DMs.

## Reproduction Assessment

Yes. The current source path deterministically reproduces the issue for direct messages: `replyTarget` is `channel:<dmChannelId>`, `autoThreadContext` is null, and `ctx.To` is assigned from that value before the user-shaped DM target is available.

## Solution Assessment

Yes. The narrowest maintainable fix is to change only the semantic inbound context fallback for DMs and add regression coverage; changing delivery routing or component-interaction behavior would be broader than this report requires.
