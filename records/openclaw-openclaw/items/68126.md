## Summary

Keep open. Current main still assigns text-DM inbound `ctx.To` from the channel-shaped reply target, while the existing user-shaped DM target is only used for `OriginatingTo` and `lastRoute`; the related focused fix PR is closed unmerged and no open replacement was found.

## What This Changes

The issue requests that Discord text-DM inbound contexts use `ctx.To = user:<userId>` for semantic downstream routing while preserving channel-ID delivery to Discord.

## Best Possible Solution

Discord text-message inbound handling should resolve the DM conversation target before choosing the semantic context target, use the `user:<id>` target for `ctx.To` on DMs, leave Discord API delivery targets channel-shaped, and lock the behavior with a focused regression test.

## Reproduction Assessment

Yes. The current source path deterministically reproduces the issue for direct messages: `replyTarget` is `channel:<dmChannelId>`, `autoThreadContext` is null, and `ctxPayload.To` is assigned from that channel-shaped value.

## Solution Assessment

Yes. The narrowest maintainable fix is extension-local: change the fallback for semantic inbound `ctx.To` to prefer the existing DM conversation identity, keep delivery routing unchanged, and add assertions for `To`, `From`, and `ChatType` in the Discord process test.
