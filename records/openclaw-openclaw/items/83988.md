## Summary

Keep this PR open but not merge-ready: the latest head still has a live Telegram delivery defect, the test type check is failing, and no real Telegram proof shows the after-fix behavior.

## What This Changes

This PR changes ACP final-mode TTS dispatch to defer visible-text settlement for Telegram, send accumulated text as a voice caption on successful TTS, preserve audio-only supplements for non-caption channels, and add dispatcher tests.

## Best Possible Solution

Make caption-capable final-mode TTS a channel-owned capability, hold or suppress live block text until the TTS outcome is known, and prove one-captioned-voice success plus text fallback in live Telegram.

## Reproduction Assessment

Yes, at source level: current main settles visible text before final TTS, and the patched live path still flushes block replies before finalization. I did not run a live Telegram session in this read-only review.

## Solution Assessment

No. Deferring settleVisibleText inside finalization is not enough for live mode; the maintainable fix should move caption support behind a channel-owned capability and prevent the earlier block send on the captioned-voice success path.
