## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve text when queued voice delivery fails:** `src/auto-reply/reply/dispatch-acp.ts:281`
  - body: When `shouldDeferTextForTts` is true, the fallback only runs if `deliver()` returns false. On the normal direct path, `sendFinalReply` returns true once the voice reply is queued, while actual Telegram delivery failures are recorded later by `waitForIdle`/`failedCounts`; a non-`VOICE_MESSAGES_FORBIDDEN` `sendVoice` error will therefore skip both the old pre-settled text and this fallback. Please keep a text fallback tied to the queued delivery outcome before deferring settlement.
  - confidence: 0.88
- **[P2] Move caption policy out of ACP core:** `src/auto-reply/reply/dispatch-acp.ts:241-244`
  - body: The new `CAPTIONED_VOICE_CHANNELS` allowlist makes ACP dispatch know a bundled channel id and decide transport caption semantics. Channel plugins already own TTS voice delivery capabilities and Telegram transport behavior, so this should be exposed as a channel/plugin capability or helper instead of a core allowlist that can drift.
  - confidence: 0.82
