## What This Changes

This PR defers ACP visible-text settlement for final-mode TTS, sends deferred text as a captioned voice payload on Telegram, tracks final delivery failures, and adds dispatch tests.

## Best Possible Solution

Land only after the patch prevents live block text from being queued before captioned final TTS, preserves fallback behavior on TTS or send failure, and has redacted live Telegram proof; maintainers should explicitly accept the temporary Telegram gate or move caption support behind a channel-owned capability.

## Reproduction Assessment

## Solution Assessment

No. Deferring settleVisibleText() alone is not the best fix because live block delivery and dispatcher serialization can still put text before the captioned final voice note; the repair needs to gate that path or prove it cannot fire.
