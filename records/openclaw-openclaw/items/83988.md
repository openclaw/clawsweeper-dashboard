## Summary

Keep this PR open, but it is not merge-ready: the linked bug is source-reproducible, yet the patch still loses deferred caption text in the Telegram live media path and uses a core Telegram allowlist instead of channel-owned capability data.

## What This Changes

The PR defers visible-text settlement for final-mode ACP TTS, adds a Telegram-only captioned-voice gate, includes text on deferred TTS media, and adds dispatch tests.

## Best Possible Solution

Keep the final-mode TTS deferral, but preserve caption text through the Telegram lane, move captioned-voice support behind a channel-owned capability/helper, and require redacted live Telegram proof before merge.

## Reproduction Assessment

Yes, at source level: current main settles visible ACP text before final TTS synthesis, and the Telegram lane can then strip caption text from TTS supplement media. I did not run a live Telegram/TTS session in this read-only review.

## Solution Assessment

No. The deferral is the right bug boundary, but this implementation still lets Telegram strip deferred TTS text from supplement media and hardcodes caption policy in core instead of a channel-owned capability.

Use a channel-owned captioned-voice capability or helper so new caption-capable channels do not require editing ACP dispatch and non-caption channels keep their own policy.
  - confidence: 0.84
