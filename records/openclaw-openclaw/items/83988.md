## Best Possible Solution

Land a fallback-safe final-mode TTS fix that settles and verifies deferred direct deliveries, sends skipTts text fallback on queued voice failure, keeps caption support channel-owned, and includes redacted live Telegram proof.

## Solution Assessment

No: the direction is right, but the current patch is not the best merge shape because queued voice delivery failure can still lose deferred text and caption support is core-owned instead of channel-owned.

- **[P1] Preserve text after queued voice failures:** `src/auto-reply/reply/dispatch-acp.ts:285`
  - body: When `deliver()` queues the captioned voice reply, `finalMediaDelivered` becomes true, so this block never calls `settleVisibleText()`.
