## Summary

Keep open: the latest push appears to fix the earlier non-deferred duplicate-text concern, but the Telegram live-delivery path still appears to strip caption text from TTS media after a preview exists, and the PR still lacks real Telegram proof.

## What This Changes

This PR changes ACP final-mode TTS dispatch to defer visible-text settlement for Telegram, attach accumulated visible text to the final TTS payload, preserve audio-only supplements for non-deferred channels, and add dispatch-acp tests.

## Best Possible Solution

Make the Telegram final-media path preserve caption text for deferred final-mode TTS while suppressing or replacing the preview text, then prove the linked issue with a real Telegram run before landing.

## Reproduction Assessment

## Solution Assessment

No. The PR adds caption text in ACP core, but the real Telegram live-preview final-media path strips TTS supplement text before media delivery, so the implementation is not yet the narrow maintainable fix.
