## Summary

Keep open for maintainer review: the patch looks technically coherent and no discrete blocking code defect was found, but it changes Telegram final-mode TTS delivery semantics and adds a plugin-facing capability that needs explicit compatibility/product acceptance before merge.

## What This Changes

The PR adds a `captionedFinalText` TTS voice capability, enables it for Telegram, defers or suppresses final-mode TTS text delivery across ACP and generic dispatch, and updates tests, docs, and the plugin SDK API baseline.

## Best Possible Solution

Land this path only after maintainers explicitly accept the captioned-final default and plugin capability contract, while leaving configurable separate delivery tracked in https://github.com/openclaw/openclaw/issues/42539.

## Solution Assessment

Yes as a technical fix shape: a typed channel capability and core dispatch deferral are cleaner than Telegram-layer delete-and-resend behavior. It still needs maintainer product acceptance because the default Telegram delivery layout and plugin contract change.
