The PR adds a `captionedFinalText` channel capability, enables it for Telegram, and changes ACP/non-ACP final-mode TTS dispatch, docs, and tests so deferred text can be sent as voice-note captions with text fallback behavior.

## Best Possible Solution

Fix suppressed-route accounting so only actual delivered media or visible text blocks the fallback, add full ACP and non-ACP `{ ok: true, suppressed: true }` regression coverage, then refresh current-head Telegram proof for always, tagged, and media cases.

## Reproduction Assessment

Yes for the root bug from source inspection: current main settles ACP visible text before final TTS synthesis, and the PR supplies a live Telegram before screenshot. The remaining suppressed-route bug is source-reproducible from `routeReply` returning `{ ok: true, suppressed: true }`, but I did not run it live.

## Solution Assessment

No, not yet: the capability-based shape is plausible, but the current implementation still conflates handled/suppressed routed sends with actual delivery in fallback gates. The best fix is to separate handled, queued, and delivered-media state before accepting the new capability contract.

Full review comments:

- **[P2] Don't treat suppressed final TTS routes as queued:** `src/auto-reply/reply/dispatch-from-config.ts:2088`
  - body: In the routed final path, `{ ok: true, suppressed: true }` still returns `queuedFinal: true`. The accumulated block fallback later requires `!queuedFinal`, so a reply-payload hook that suppresses the captioned voice note skips the text fallback even though no user-visible message was sent. Return handled/queued separately from delivered, or gate this fallback on `isRoutedReplyDelivered(result)`.
  - confidence: 0.88
- **[P2] Keep suppressed ACP final media out of visible-delivery flags:** `src/auto-reply/reply/dispatch-acp-delivery.ts:491-496`
  - body: The suppressed-route branch leaves `deliveredFinalTtsMedia` false, but it still marks the final reply and visible text as delivered. `finalizeAcpTurnOutput` then skips the deferred text fallback for captioned final TTS, so hook-suppressed voice sends can drop the accumulated reply text. Track suppression as handled separately, or make fallback ignore these flags when no media was delivered.
  - confidence: 0.9
