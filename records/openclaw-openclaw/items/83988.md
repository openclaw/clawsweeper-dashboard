## Summary

Keep this PR open, but do not merge it as-is: it targets a real Telegram final-mode TTS churn bug, yet the current patch still misses the Telegram live-preview path that strips caption text before `sendVoice`, and it has no after-fix real behavior proof.

## What This Changes

The branch defers ACP visible-text settlement for caption-capable final-mode TTS, includes text on deferred TTS media payloads, preserves non-deferred audio-only supplements, and adds dispatch-acp unit coverage.

## Best Possible Solution

Preserve deferred text through the Telegram final-media/TTS supplement path, or a channel capability seam, so final-mode TTS produces one captioned voice note with text fallback on synthesis or `sendVoice` failure, then prove it with focused Telegram coverage and live proof.

## Reproduction Assessment

Yes at source level: current main settles visible ACP text before final TTS, and the Telegram final-media path strips text when late media follows a streamed preview. I did not run live Telegram proof in this read-only review.

## Solution Assessment

No. Deferring settlement in ACP is plausible, but attaching text there is incomplete while Telegram's final-media path strips it; the fix needs to preserve caption text in the Telegram lane or through a channel capability contract.
