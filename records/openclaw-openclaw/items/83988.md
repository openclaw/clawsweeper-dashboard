## Summary

Keep open: this is a focused fix branch for a real Telegram final-mode TTS bug, but the current patch still stops in ACP dispatch and does not update the Telegram preview/final-media path that strips text from TTS supplement media before `sendVoice`, so it can leave the linked churn unresolved; real Telegram proof is also still absent.

## What This Changes

The branch defers ACP visible-text settlement for Telegram final-mode TTS, conditionally attaches accumulated text to the final TTS media payload, preserves audio-only supplements for other channels, and adds dispatch-acp unit coverage.

## Best Possible Solution

Fix this at the Telegram delivery or channel-capability boundary so deferred final-mode TTS sends one voice message with caption text, avoids a separate finalized preview bubble, preserves fallback behavior, and is covered by dispatch plus Telegram delivery tests and redacted live proof.

## Reproduction Assessment

Yes, at source level: current main flushes or queues visible ACP text before final TTS, and the Telegram final-media path finalizes preview text before sending TTS media. I did not run live Telegram proof in this read-only review.

## Solution Assessment

No. Deferring settlement in ACP dispatch and adding `text` to the mocked final payload is not enough because Telegram's real TTS supplement media path strips that text and finalizes the preview separately.
