## Summary

## What This Changes

The PR adds a `captionedFinalText` channel capability and changes ACP and generic final-mode TTS dispatch so caption-capable channels can defer visible text into the final voice-note caption with text fallback handling.

## Best Possible Solution

Land this PR or a maintainer-approved equivalent only after maintainers accept the captioned-final capability/default behavior, current-head checks finish, and the separate-delivery policy remains tracked separately in https://github.com/openclaw/openclaw/issues/42539.

## Reproduction Assessment

Yes. Current-main and latest-release source show visible text is settled or block-delivered before final accumulated TTS audio on the ACP and generic dispatch paths; I did not run a fresh live reproduction in this read-only review.

## Solution Assessment

Yes, as an implementation shape: a generic capability-gated dispatch-layer fix is cleaner than the older Telegram-only delete/recombine approach. It is still compatibility-sensitive because it introduces a plugin-facing capability and changes Telegram final-mode delivery policy.
