## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Constrain captionedFinalText to voice-note delivery:** `src/channels/plugins/types.core.ts:305`
  - body: `captionedFinalText` is added as an independent boolean, so a plugin can declare `{ synthesisTarget: "audio-file", captionedFinalText: true }`. Both dispatch paths use that boolean to suppress visible text before final TTS, but audio-file channels have no voice-note caption contract, so an invalid opt-in can silently hide or misdeliver the reply text. Make this a discriminated type or have the resolver ignore `captionedFinalText` unless `synthesisTarget === "voice-note"`, and add a regression for the invalid combination.
  - confidence: 0.9
