## What This Changes

This PR adds a `captionedFinalText` voice TTS capability, enables it for Telegram, and updates ACP and non-ACP reply dispatch to defer or suppress final-mode text until captioned voice delivery or text fallback.

## Best Possible Solution

Land this only after maintainers explicitly accept `captionedFinalText` as the supported channel capability and Telegram default for this bug, while leaving separate text-plus-voice delivery tracked in https://github.com/openclaw/openclaw/issues/42539.
