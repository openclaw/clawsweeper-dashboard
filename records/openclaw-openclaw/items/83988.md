## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve deferred text when a final payload exists:** `src/auto-reply/reply/dispatch-from-config.ts:2895`
  - body: When captioned final TTS is active, text-only block replies are converted to an empty block delivery payload and held in `accumulatedVisibleBlockText`. The only later TTS/fallback path is guarded by `replies.length === 0`, so a resolver that also returns a final media-only payload, or any final payload that cannot produce TTS media, skips this recovery path and the text that current main would have sent as a block is never delivered. Please flush the deferred text unless a combined captioned voice payload actually succeeds.
  - confidence: 0.88
- **[P2] Document the captioned final TTS capability:** `src/channels/plugins/types.core.ts:322`
  - body: `captionedFinalText` is a new plugin-facing channel capability that changes core delivery behavior by suppressing live block text and relying on caption/fallback semantics. The patch only adds type JSDoc and the API baseline hash; public plugin docs still do not describe when to opt in, what core suppresses, or what fallback guarantees channel authors get.
  - confidence: 0.86
