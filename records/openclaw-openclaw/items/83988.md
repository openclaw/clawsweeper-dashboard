## Summary

Keep this PR open: it targets a real Telegram final-mode TTS bug, but the patch defers visible-text settlement for every final-mode TTS channel instead of only caption-capable voice delivery, which can change message ordering for existing non-Telegram users. It also lacks external real Telegram proof, and the PR’s Real behavior proof check is failing.

## What This Changes

The PR defers `settleVisibleText()` for final-mode ACP TTS, sends accumulated visible text with the final TTS media payload, and adds focused dispatch tests for ordering, success, and fallback.

## Best Possible Solution

Keep the defer-then-fallback shape, but gate it on an explicit captioned-voice channel capability or equivalent proven contract, preserve existing eager settlement for other channels, and add real Telegram proof plus non-Telegram regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main settles visible ACP text before final-mode TTS synthesis, so Telegram cannot use the later voice send as the first visible captioned reply.

## Solution Assessment

No. The PR uses the right high-level defer-and-fallback idea, but it applies the behavior to all final-mode TTS channels instead of only channels that can preserve visible text inline with the voice media.
