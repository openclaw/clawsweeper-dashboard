## Summary

Keep this PR open, but it is not merge-ready: the patch still leaves the live ACP block-delivery path able to show Telegram text before the captioned final voice note, and caption eligibility is hardcoded in core instead of coming from channel capabilities.

## What This Changes

The PR changes ACP final-mode TTS finalization to defer visible-text settlement for Telegram, include accumulated text as a voice caption, track final-delivery failures, and add dispatcher regression tests.

## Best Possible Solution

Suppress or buffer visible live block text for caption-capable final-mode TTS through a channel-owned capability, preserve text fallback on TTS or send failure, and refresh exact-head Telegram proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: ACP live text deltas flush through `deliver("block")` and `sendBlockReply` before `finalizeAcpTurnOutput()` can synthesize the captioned final voice note.

## Solution Assessment

No. Deferring `settleVisibleText()` is only part of the fix; the branch also needs to suppress live block text and read caption eligibility from channel capabilities rather than a core Telegram check.
