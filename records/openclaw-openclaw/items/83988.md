## Summary

Keep open: the latest branch still has a queued Telegram voice-send failure path that can skip the standalone text fallback, and the supplied proof remains mocked/unit-only for visible Telegram behavior.

## What This Changes

The PR defers final-mode ACP TTS text settlement for caption-capable voice channels, includes accumulated visible text on the TTS media payload, and adds dispatch-acp tests for ordering and fallback behavior.

## Best Possible Solution

Land a fallback-safe final-mode TTS fix that treats failed queued voice delivery as not visibly final, routes caption support through a channel/plugin-owned capability or helper, and is proven with live Telegram evidence plus focused tests.

## Reproduction Assessment

Yes at source level: current main settles visible text before final-mode TTS, and the PR's mocked ordering test captures that path. I did not establish a live Telegram reproduction in this read-only review.

## Solution Assessment

No. The latest patch addresses the main ordering shape, but the queued-failure fallback still does not fire because hasDeliveredFinalReply remains true, and caption support still needs a channel-owned contract rather than ACP-core Telegram policy.
