## Summary

## What This Changes

The PR adds a `captionedFinalText` channel TTS capability, opts Telegram into it, defers/suppresses final-mode TTS text on ACP and generic dispatch paths, and updates tests, docs, and the SDK API baseline.

## Best Possible Solution

Merge this only after maintainers explicitly accept the `captionedFinalText` SDK contract and Telegram captioned-final default, while keeping separate text-plus-voice delivery tracked by https://github.com/openclaw/openclaw/issues/42539.
