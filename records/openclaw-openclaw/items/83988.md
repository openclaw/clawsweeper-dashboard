## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use media-delivery state for ACP fallback:** `src/auto-reply/reply/dispatch-acp.ts:337`
  - body: `delivery.deliver()` returns true for a hook-suppressed routed final, but no media reached the user and the coordinator deliberately leaves `hasDeliveredFinalTtsMedia()` false. Assigning that boolean to `finalMediaDelivered` still makes the fallback gate skip the accumulated text, so a suppressed captioned voice send can drop the reply. Recompute this from the coordinator's media-delivered state after the send, or return a distinct media-delivered result, and cover the full ACP finalize path.
  - confidence: 0.9
- **[P2] Do not count suppressed routed final media as queued:** `src/auto-reply/reply/dispatch-from-config.ts:2080`
  - body: For routed final sends, `{ ok: true, suppressed: true }` means the hook cancelled provider delivery, but this return still sets `queuedFinal` true. The accumulated-block fallback later requires `!queuedFinal`, so a suppressed captioned voice route skips the text fallback even though `deliveredMedia` is false. Gate the fallback on actual delivery for media payloads, and add a suppressed-route regression rather than only `{ ok: false }`.
  - confidence: 0.84
