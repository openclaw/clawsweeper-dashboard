## Summary

Keep open. The branch addresses part of the final-settlement ordering, but it still needs a live-mode Telegram fix/proof because ACP live text can be queued before final TTS, and caption eligibility is still encoded as a core Telegram check.

## What This Changes

The PR changes ACP final-mode TTS dispatch to defer visible-text settlement for caption-capable voice delivery and adds delivery-state/test coverage for captioned voice fallback behavior.

## Best Possible Solution

Land a stream-aware fix that prevents or reconciles the live Telegram preview for captioned final TTS, moves caption eligibility to a channel/plugin capability or explicitly approved temporary seam, and includes redacted Telegram proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source inspection confirms current main settles visible text before final TTS, but the live Telegram churn path has not been independently reproduced in a real Telegram session.

## Solution Assessment

Is this the best way to solve the issue? No; deferring final settlement alone does not gate live block delivery, and the caption decision should be channel-owned or explicitly accepted as a temporary core exception.
