## Summary

Keep open: the PR targets a real Telegram final-mode TTS churn bug, but it is not merge-ready because it defers text settlement for all final-mode TTS channels and only falls back when synthesis/no-media fails, not when queued media delivery later fails. Real Telegram behavior proof is also missing.

## What This Changes

The PR moves final-mode ACP TTS ahead of visible-text settlement, sends accumulated text with the TTS media payload, and adds dispatch-acp tests for ordering and fallback.

## Best Possible Solution

Gate deferral on an explicit captioned-voice capability, preserve a text fallback for actual media delivery failure, keep eager settlement elsewhere, and require redacted live Telegram proof.

## Reproduction Assessment

Yes. Current main calls settleVisibleText() before final TTS synthesis, and Telegram delivery can attach reply.text as a sendVoice caption; I did not run a live Telegram scenario in this read-only review.

## Solution Assessment

No. The defer-and-caption idea is right for Telegram, but this PR applies it to all final-mode TTS channels and only falls back on synthesis/no-media outcomes rather than actual delivery failure.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate TTS deferral to caption-capable voice channels:** `src/auto-reply/reply/dispatch-acp.ts:238`
  - body: `willAttemptFinalTts` now skips upfront text settlement for every final-mode TTS channel. Telegram can put `text` on `sendVoice`, but WhatsApp sends audio captions as a text follow-up after the voice note and Discord voice messages cannot include text, so existing non-Telegram users can get delayed or reordered visible text. Keep eager settlement unless the channel advertises or otherwise proves captioned voice delivery.
  - confidence: 0.9
- **[P1] Preserve fallback when queued media delivery fails:** `src/auto-reply/reply/dispatch-acp.ts:279-284`
  - body: The new fallback runs only when `finalMediaDelivered` is false, but direct delivery returns true as soon as the final media payload is queued; channel send errors are caught later by the dispatcher. With text settlement skipped, a queued-but-failed voice/media send can leave the user with no visible final text, where current main had already settled the text before attempting TTS media.
  - confidence: 0.84
