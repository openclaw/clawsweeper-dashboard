## Summary

Keep this PR open for maintainer review: the patch appears to address the reported Telegram final-mode TTS churn, but it adds a public channel capability and changes Telegram delivery policy, so the remaining decision is product/API acceptance rather than a narrow code repair.

## What This Changes

The PR adds a `captionedFinalText` channel TTS capability, opts Telegram into it, and changes ACP and generic dispatch to defer or suppress final-mode TTS text until it can be sent as a voice-note caption with fallback handling.

## Best Possible Solution

Land this only after a maintainer explicitly accepts `captionedFinalText` as the channel capability and Telegram captioned voice as the final-mode churn fix, while keeping configurable separate delivery tracked separately.
