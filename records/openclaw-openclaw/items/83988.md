## Summary

## What This Changes

The PR adds a captioned final TTS channel capability, enables it for Telegram, and changes ACP plus generic reply dispatch to suppress live text before sending final-mode TTS as one captioned voice note.

## Best Possible Solution

Merge only after public plugin/TTS docs describe the `captionedFinalText` contract and maintainers explicitly accept Telegram final-mode TTS delaying visible text to deliver one captioned voice note.

## Reproduction Assessment

Yes, by source inspection: current main calls `settleVisibleText()` before final TTS in the ACP path and only sends the generic accumulated block TTS supplement after visible block delivery. The PR also includes live before/after Telegram screenshots for the observable churn.

## Solution Assessment

Partly: the channel capability is the right owner boundary and avoids Telegram hardcoding in core, but the new public contract needs docs and the Telegram timing change needs maintainer acceptance before it is the best merge-ready solution.
