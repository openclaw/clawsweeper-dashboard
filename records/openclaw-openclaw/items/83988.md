## Summary

## What This Changes

The PR adds a channel `captionedFinalText` TTS capability, enables it for Telegram, and updates ACP/non-ACP final-mode TTS delivery, tests, docs, and the SDK API baseline so caption-capable channels defer or suppress text until final voice-caption delivery.

## Best Possible Solution

Land this PR or a maintainer-approved narrow variant after explicitly accepting the `captionedFinalText` channel capability, while keeping configurable separate text-plus-voice delivery tracked in https://github.com/openclaw/openclaw/issues/42539.

## Reproduction Assessment

Yes. Current main and `v2026.6.8` still settle ACP visible text before final TTS, and the PR includes live before-fix Telegram screenshot evidence for the user-visible text-plus-voice churn.

## Solution Assessment

Yes for the implementation shape, pending maintainer API acceptance. A capability-gated core fix across ACP and non-ACP paths is cleaner than the older Telegram-only delete/re-send approach, but the new plugin-facing contract must be approved or narrowed.
