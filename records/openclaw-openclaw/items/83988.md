## Summary

Keep this PR open, but it is not merge-ready: the fix is aimed at a real Telegram final-mode TTS churn bug, yet the current branch can drop deferred block text on abort and still needs live Telegram proof plus a clearer public channel-capability contract.

## What This Changes

The branch defers ACP final-mode TTS text settlement for caption-capable channels, adds a `captionedFinalText` channel capability used by Telegram, suppresses live block delivery in that path, and adds focused unit tests.

## Best Possible Solution

Land a narrow fix that defers text only when accumulated text is flushed on success, failure, and abort, with a documented channel capability contract and redacted live Telegram proof.

## Reproduction Assessment

Yes, source inspection reproduces the original ordering problem on current main, and the PR's own Docker tests encode the mocked failure path. The new abort-path regression is also source-reproducible because deferred block delivery returns before finalization can flush text.

## Solution Assessment

No, not yet. Deferring settlement behind a channel capability is the right direction, but this branch needs an abort fallback and a documented capability contract before it is the safest maintainable fix.
