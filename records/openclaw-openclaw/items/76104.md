## Summary

Keep open: current main still has a concrete source-level gap for Feishu direct-session `sessions_send` announce delivery when the stored session channel is `webchat` but the explicit `deliveryContext` points at Feishu. Earlier generic route-preservation work covers adjacent cases, but the reported direct Feishu shape is not covered by the current parser or regression tests.

## What This Changes

The issue asks for `sessions_send` replies triggered from Feishu direct messages to route back to the Feishu origin instead of the dashboard webchat channel.

## Best Possible Solution

Make persisted external `deliveryContext` authoritative for direct/announce delivery when the stored session channel is internal or stale, with a generic regression test for the reported Feishu DM shape.

## Reproduction Assessment

Yes. A focused unit test can seed a session row for `agent:main:feishu:direct:ou_x` with `channel: "webchat"` and `deliveryContext: { channel: "feishu", to: "user:ou_x" }`, then assert `sessions_send` announce delivery resolves Feishu.

## Solution Assessment

Yes for the requested direction, but the safest fix is generic delivery resolution for explicit external `deliveryContext`, not a Feishu-specific core branch or a new config flag.

## Repair Work Prompt

Fix the Feishu direct-session `sessions_send` announce route reported at https://github.com/openclaw/openclaw/issues/76104. In current main, a stored session row like `agent:main:feishu:direct:ou_x` with `channel: "webchat"` and `deliveryContext: { channel: "feishu", to: "user:ou_x" }` can resolve announce delivery to the internal webchat channel because `normalizeSessionDeliveryFields` treats session channel fields as primary. Make the repair generic: explicit persisted external `deliveryContext` should win over internal/stale webchat fields for direct/announce delivery, while preserving existing safeguards that avoid mixing route fields across unrelated external channels. Do not add a Feishu-only branch or a config flag. Add focused regression coverage in the delivery-context and sessions_send announce-target path, and add a single-line user-facing changelog fix entry crediting the human reporter if policy allows.
