## Summary

Keep open: current main still has a source-level route-resolution gap for the reported Feishu DM shape. A direct Feishu session key cannot produce an announce target from the key alone, and the sessions-list fallback can drop or misroute the explicit Feishu delivery context when the stored session channel is stale/internal.

## What This Changes

The issue reports that `sessions_send` replies from Feishu DMs should return to the originating Feishu conversation instead of webchat.

## Best Possible Solution

Make explicit persisted external `deliveryContext` authoritative for announce/direct delivery when the stored session channel is internal or stale, and cover the Feishu direct session shape with regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed a sessions-list row for `agent:main:feishu:direct:ou_x` with `channel: "webchat"` and `deliveryContext: { channel: "feishu", to: "user:ou_x" }`, then assert announce delivery resolves Feishu.

## Solution Assessment

Yes. The narrow maintainable fix is generic delivery-context precedence for explicit external persisted routes, not a Feishu-specific branch or a new config flag.
