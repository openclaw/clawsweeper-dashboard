## Summary

Keep open: current main still has a plausible source-level gap where `sessions_send` announce delivery for a Feishu direct-session key can lose the explicit Feishu `deliveryContext` when the stored session channel is `webchat`. Earlier generic route-preservation work shipped, but this later Feishu direct-message shape is not covered by the current parser or regression tests.

## What This Changes

Fix Feishu direct-message `sessions_send` replies so announce delivery routes to the origin Feishu DM instead of the stored webchat session channel.

## Best Possible Solution

Make explicit persisted external `deliveryContext` authoritative for direct/announce delivery when the stored session channel is an internal or stale webchat value, with a regression test for the reported Feishu DM session shape.

## Reproduction Assessment

Yes. A focused unit test can seed a session row or stored entry for `agent:main:feishu:direct:ou_x` with `channel: "webchat"` and `deliveryContext: { channel: "feishu", to: "user:ou_x" }`, then assert `sessions_send` announce delivery resolves Feishu.

## Solution Assessment

Yes for the direction, but the safest repair is generic delivery resolution for explicit external `deliveryContext`, not a Feishu-specific core branch or a new config flag.
