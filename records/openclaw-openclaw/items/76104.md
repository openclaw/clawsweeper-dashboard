## Summary

Keep open: current main still resolves the reported Feishu direct-session shape through session-list delivery normalization, where a persisted internal `webchat` channel can outrank the explicit Feishu `deliveryContext`.

## What This Changes

The issue asks for `sessions_send` replies from Feishu DMs to route back to Feishu instead of the dashboard webchat route.

## Best Possible Solution

Make explicit persisted external `deliveryContext` authoritative for announce/direct delivery when the stored session channel is internal or stale, and cover the Feishu DM row shape with regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: seed a session row for `agent:main:feishu:direct:ou_x` with `channel: "webchat"` and `deliveryContext: { channel: "feishu", to: "user:ou_x" }`, then assert `resolveAnnounceTarget` or `sessions_send` announce delivery resolves Feishu.

## Solution Assessment

Yes. The narrow maintainable fix is generic delivery resolution for explicit external `deliveryContext`, not a Feishu-specific branch or a new config flag.
