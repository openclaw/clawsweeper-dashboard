## Summary

Keep open: current main still has a deterministic route-precedence gap for the reported Feishu direct-session shape, where `channel: "webchat"` can dominate an explicit Feishu `deliveryContext` before `sessions_send` announce delivery.

## What This Changes

The issue requests that `sessions_send` replies from Feishu direct messages route back to Feishu instead of falling back to webchat.

## Best Possible Solution

Make explicit persisted `deliveryContext` authoritative over internal/webchat session-channel metadata for direct external sessions, using a generic normalization or announce-target rule with regression coverage for the reported Feishu DM shape.

## Reproduction Assessment

Yes. A focused unit or gateway-tool test can seed `agent:main:feishu:direct:ou_x` with `channel: "webchat"`, Feishu `deliveryContext`, and no Feishu `lastChannel`, then assert the announce target resolves to Feishu.

## Solution Assessment

Unclear whether the issue’s exact suggested priority order is the best patch, but the safest maintainable direction is a generic delivery-context precedence fix rather than Feishu-specific routing in core.

## Repair Work Prompt

Keep the fix generic: do not add Feishu-specific core branching or import Feishu internals into core. Treat authoritative persisted `deliveryContext` as the announce route when the primary channel is internal/webchat and the context has a deliverable external channel/to, or adjust `resolveAnnounceTarget` to recover that raw context before normalized session fields drop it. Add regression coverage for `agent:main:feishu:direct:ou_x` with `channel: "webchat"`, `origin.provider: "feishu"`, `deliveryContext: { channel: "feishu", to: "user:ou_x" }`, and no Feishu `lastChannel`/`lastTo`; assert the final announce send targets Feishu. Preserve existing behavior for sessions with no external delivery context and for cross-channel conflict tests that intentionally avoid mixing route fields. Add a user-facing changelog fix entry under the active version, without bot or maintainer attribution.
