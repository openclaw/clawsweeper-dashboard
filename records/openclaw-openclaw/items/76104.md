## Summary

Keep open: current `main` still has a source-level routing gap for the reported Feishu direct-message session shape. The announce path can lose the explicit Feishu `deliveryContext` when the persisted session channel is stale/internal, so this remains a narrow bug-fix candidate.

## What This Changes

Fix `sessions_send` replies from Feishu DMs so announce delivery routes back to the originating Feishu channel instead of webchat.

## Best Possible Solution

Make explicit persisted external `deliveryContext` authoritative for announce/direct delivery when the stored session channel is internal or stale, while keeping the fix generic and covered by a Feishu DM regression test.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed a session row for `agent:main:feishu:direct:ou_x` with `channel: webchat` and `deliveryContext: { channel: feishu, to: user:ou_x }`, then assert `resolveAnnounceTarget` or the `sessions_send` announce flow targets Feishu.

## Solution Assessment

Yes. The narrow maintainable fix is generic delivery resolution for explicit external `deliveryContext`, not a Feishu-specific branch or a new config flag.
