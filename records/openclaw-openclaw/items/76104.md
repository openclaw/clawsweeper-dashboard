## Summary

Keep open: current main still has a plausible generic delivery-context precedence gap for the reported Feishu direct-session shape, and the earlier route-preservation fixes do not cover `channel: webchat` conflicting with an explicit Feishu `deliveryContext`.

## What This Changes

The issue requests that Feishu DM-originated `sessions_send` replies return through Feishu instead of being routed to the Dashboard/webchat channel.

## Best Possible Solution

Make explicit persisted external `deliveryContext` authoritative for announce delivery when it conflicts with an internal/webchat session channel, while preserving existing `lastChannel` behavior for genuinely updated external routes.

## Reproduction Assessment

Yes. A focused unit test can seed `agent:main:feishu:direct:ou_x` with `channel: "webchat"` and `deliveryContext: { channel: "feishu", to: "user:ou_x" }`, then assert `sessions_send` announce delivery resolves Feishu.

## Solution Assessment

Unclear until patched, but the narrow maintainable direction is a generic delivery-resolution precedence fix with regression coverage, not a Feishu-specific core branch or config flag.
