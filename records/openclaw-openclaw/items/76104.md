## Summary

Keep open: current main still has a source-level routing gap for the reported Feishu DM shape. The announce path can fall back to `sessions.list`, but `deliveryContextFromSession` still normalizes the stored session channel ahead of the explicit Feishu delivery context.

## What This Changes

Feishu `sessions_send` replies should return to the originating Feishu DM instead of dashboard WebChat when the persisted session channel is stale.

## Best Possible Solution

Make an explicit persisted external `deliveryContext` authoritative for announce/direct delivery when the stored session channel is internal or stale, and cover the Feishu DM session shape with a regression test.

## Reproduction Assessment

Yes, as a source-level reproduction: seed a session row for `agent:main:feishu:direct:ou_x` with `channel: "webchat"` and `deliveryContext: { channel: "feishu", to: "user:ou_x" }`, then resolve the `sessions_send` announce target. Current normalization keeps the stale session channel primary.

## Solution Assessment

Yes. The narrow maintainable fix is generic delivery resolution for explicit external `deliveryContext`, not a Feishu-specific branch or a new config flag.
