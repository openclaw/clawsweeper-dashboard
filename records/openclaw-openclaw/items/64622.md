## Summary

Keep open. The branch is still useful in scope, but it is conflicting, lacks real A2A/gateway behavior proof, and has P1 blockers that can change existing timeout behavior and persist cross-channel route metadata under the wrong session.

## What This Changes

The PR adds a 45-second `sessions_send` A2A timeout floor, removes selected inter-session provenance fields, sanitizes webchat-backed main-session metadata, rewrites outbound mirror routes to `currentSessionKey`, adds focused tests, and adds heartbeat anti-loop template docs.

## Best Possible Solution

Rework this into smaller policy-aligned fixes: handle timeout semantics through the canonical A2A timeout work, preserve resolved outbound route identity for persistence, and keep metadata cleanup covered by focused tests plus real gateway proof.

Yes for the PR blockers: source inspection of the diff and current resolver/persistence contracts shows the timeout-floor and route-key rewrite failure modes. No high-confidence live reproduction is present for the original multi-agent runtime symptoms.

## Solution Assessment

No. The current patch broadens timeout semantics and conflates active run continuity with persisted outbound route identity; a narrower routing fix plus the canonical timeout-policy work is safer.
