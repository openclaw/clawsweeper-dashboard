## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Scope timeout floor to true cross-agent sends:** `src/agents/tools/sessions-send-tool.ts:70-71`
  - body: `resolveAgentIdFromSessionKey()` falls back to the default agent id, and `effectiveRequesterKey` also defaults to main, so this condition treats ordinary requester/target keys as A2A and coerces short same-agent or local timeouts to 45 seconds. Callers that explicitly asked for fast non-zero timeouts would now wait much longer outside the case this PR is trying to protect.
  - confidence: 0.93
- **[P1] Preserve resolved route keys for cross-target sends:** `src/infra/outbound/message-action-threading.ts:89-95`
  - body: This rewrites the resolved external route to `currentSessionKey` before `ensureOutboundSessionEntry`. Cross-target sends can then persist external `from`/`to`/`chatType` metadata under the active main/webchat session, corrupting later routing and policy decisions.
  - confidence: 0.9
