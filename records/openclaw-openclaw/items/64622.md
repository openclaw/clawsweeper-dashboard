This PR adds a 45-second `sessions_send` timeout floor, removes some A2A provenance fields, sanitizes webchat-backed main-session metadata, rewrites outbound mirror routes to `currentSessionKey`, and adds focused tests plus docs-template anti-loop guidance.

## Best Possible Solution

Rework the runtime changes into smaller, policy-aligned fixes: preserve resolved outbound route identity for persistence, keep any webchat metadata cleanup compatible with current route metadata, and handle timeout semantics through the canonical A2A timeout work.
