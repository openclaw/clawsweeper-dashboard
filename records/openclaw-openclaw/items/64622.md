## Summary

Keep this PR open but do not merge as-is: the useful routing-cleanup direction is mixed with two blocking runtime regressions, missing real behavior proof, and an unrelated docs-template change.

## What This Changes

The PR adds A2A `sessions_send` timeout/provenance changes, webchat main-session metadata cleanup, outbound mirror `currentSessionKey` rewriting, regression tests, and docs template guidance.

## Best Possible Solution

Rework this into narrower runtime fixes: keep compatible webchat metadata cleanup, scope timeout handling to true cross-agent waits or coordinate it with the dedicated timeout PR, preserve resolved outbound route identity unless same-conversation equivalence is proven, and move docs-template policy separately.

## Reproduction Assessment

Yes for the PR blockers: source inspection shows the timeout floor applies beyond true cross-agent sends and the outbound route identity is rewritten before persistence. The original live multi-agent timeout symptom still lacks a high-confidence real setup reproduction in this PR.

## Solution Assessment

No. The direction is plausible, but this patch is not the best fix because it broadens timeout semantics, rewrites cross-target route identity, mixes unrelated docs policy, and lacks real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Scope the timeout floor to true cross-agent sends:** `src/agents/tools/sessions-send-tool.ts:70-71`
  - body: `resolveAgentIdFromSessionKey()` normalizes or falls back to an agent ID, so this condition is true for same-agent and local session sends whenever both keys are present. A caller that requested `timeoutSeconds: 5` would now wait up to 45 seconds outside the A2A case this PR is trying to protect.
  - confidence: 0.93
- **[P1] Preserve resolved route keys for cross-target sends:** `src/infra/outbound/message-action-threading.ts:89-95`
  - body: When `currentSessionKey` is present, this branch rewrites the resolved external route before `ensureOutboundSessionEntry`. Cross-target sends can then persist external `from`/`to`/`chatType` metadata under the current main/webchat session, corrupting later routing and policy decisions.
  - confidence: 0.9
