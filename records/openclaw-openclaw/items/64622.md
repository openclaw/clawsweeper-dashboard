## Summary

Keep open. The branch is a plausible fix direction, but current review still finds blocking timeout and outbound routing defects, plus missing after-fix real behavior proof for the reported A2A/gateway flow.

## What This Changes

The branch adds a 45-second A2A `sessions_send` timeout floor, changes A2A provenance, sanitizes webchat-backed main-session metadata, rewrites outbound mirror routes to `currentSessionKey`, adds tests, and adds docs template guidance.

## Best Possible Solution

Revise this as a narrower runtime fix: scope timeout changes to true cross-agent sends, keep resolved outbound route identity unless the route is equivalent, retain the useful webchat metadata cleanup, and split unrelated docs-template policy into separate work.

## Reproduction Assessment

Yes for the PR review blockers: source inspection shows the timeout floor applies too broadly and the outbound route identity is unconditionally rewritten before persistence. The original live multi-agent timeout report remains under-specified.

## Solution Assessment

No. The direction is plausible, but this patch is not the best solution because it broadens timeout policy, rewrites route identity across targets, mixes unrelated docs policy, and lacks real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Scope the timeout floor to true cross-agent sends:** `src/agents/tools/sessions-send-tool.ts:70-71`
  - body: `resolveAgentIdFromSessionKey()` normalizes or falls back to an agent ID, so this condition is true for same-agent and local session sends whenever both keys are present. A caller that requested `timeoutSeconds: 5` would now wait up to 45 seconds outside the A2A case this PR is trying to protect.
  - confidence: 0.93
- **[P1] Preserve resolved route keys for cross-target sends:** `src/infra/outbound/message-action-threading.ts:89-95`
  - body: When `currentSessionKey` is present, this branch rewrites the resolved external route before `ensureOutboundSessionEntry`. Cross-target sends can then persist external `from`/`to`/`chatType` metadata under the current main/webchat session, corrupting later routing and policy decisions.
  - confidence: 0.9

## Security Review
