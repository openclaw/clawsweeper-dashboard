## Summary

Keep open. The PR targets a plausible A2A/gateway bug cluster, but the current patch is not mergeable because it over-applies the timeout floor, rewrites outbound route identity in a routing/privacy-sensitive way, and only provides test-based proof.

## What This Changes

The PR changes A2A `sessions_send` timeout and provenance handling, gateway main-session route metadata cleanup, outbound mirror session-key handling, focused tests, and two docs templates.

## Best Possible Solution

Revise this into a narrower runtime fix that scopes timeout flooring to true cross-agent sends, sanitizes webchat-backed main-session metadata, preserves outbound continuity only for route-equivalent targets, and splits the docs-template policy into a separate change.

## Reproduction Assessment

Yes for the PR review blockers: source inspection shows the timeout floor applies whenever both normalized ids are truthy, and the PR diff rewrites a resolved external route to the current session before persistence. The original production timeout remains under-specified in the linked issue.

## Solution Assessment

No. The requested direction is plausible, but this patch is not the best implementation because it over-broadens timeout policy, unconditionally rewrites route identity, and lacks real behavior proof from a live setup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Scope the timeout floor to true cross-agent sends:** `src/agents/tools/sessions-send-tool.ts:70-71`
  - body: `resolveAgentIdFromSessionKey()` normalizes/falls back to an agent id, so this condition is true for same-agent and local session sends whenever both keys are present. A caller that requested `timeoutSeconds: 5` for a local send would now wait up to 45 seconds, regressing fast-fail behavior outside the A2A case this PR is trying to protect.
  - confidence: 0.93
- **[P1] Preserve resolved route keys for cross-target sends:** `src/infra/outbound/message-action-threading.ts:89-95`
  - body: When `currentSessionKey` is present, this branch rewrites the resolved QQ/Discord/etc. route to the active session before `ensureOutboundSessionEntry`. Cross-target sends can then persist external `from`/`to`/`chatType` metadata under the current main/webchat session, corrupting later routing and policy decisions.
  - confidence: 0.9
