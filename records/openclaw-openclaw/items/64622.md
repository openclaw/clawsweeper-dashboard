## Summary

Keep open: the problem area is real, but this PR is not merge-ready because the patch broadens short `sessions_send` timeouts, can write cross-target route metadata under the current session key, removes source provenance from inter-session handoffs, lacks real behavior proof, and is currently conflicting with base.

## What This Changes

The branch adds a `sessions_send` timeout floor, changes A2A provenance propagation, sanitizes webchat-backed main-session route metadata, rewrites outbound mirror session keys to `currentSessionKey`, adds focused tests, and adds unrelated pending-review heartbeat template guidance.

## Best Possible Solution

Land a narrower routing fix that preserves existing timeout and provenance contracts, only reuses current session continuity when the resolved route belongs to the same base conversation, rebases cleanly, removes unrelated docs churn, and includes redacted real A2A behavior proof.

Yes for the PR-introduced blockers: source inspection shows the timeout predicate is always true for resolved keys and the outbound mirror branch overwrites resolved routes with `currentSessionKey`. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No. The PR is a plausible mitigation for real A2A/session-routing pain, but the best fix must preserve existing timeout semantics, source provenance, and route ownership instead of applying broad rewrites.
