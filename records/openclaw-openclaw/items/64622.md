## Summary

Keep open. This external PR targets real agent/gateway routing pain, but it is not merge-ready because the timeout floor is over-broad, the outbound route rewrite can corrupt session route metadata, and the PR still lacks after-fix proof from a real A2A/gateway setup.

## What This Changes

The PR adds a 45-second `sessions_send` timeout floor, removes A2A provenance fields, sanitizes webchat-backed main-session metadata, rewrites outbound mirror routes to `currentSessionKey`, adds focused tests, and adds docs-template anti-loop guidance.

## Best Possible Solution

Rework the runtime changes into smaller policy-aligned fixes: scope timeout changes to true cross-agent sends or the canonical timeout tracker, preserve resolved outbound route identity for persistence, and prove the result in a real A2A/gateway run.

## Reproduction Assessment

Yes for the PR blockers: source inspection of the diff and current resolver/persistence contracts shows the timeout-floor and route-key rewrite failure modes. No live reproduction is present for the original multi-agent runtime symptoms.

## Solution Assessment

No. The patch addresses plausible symptoms, but the timeout floor is over-broad and the route-key rewrite conflates active run continuity with persisted outbound route identity.
