## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Limit the timeout floor to true cross-agent sends:** `src/agents/tools/sessions-send-tool.ts:68-69`
  - body: `resolveAgentIdFromSessionKey` returns a non-empty id even for non-agent keys by defaulting to `main`, and both keys are present in ordinary sends. This condition therefore raises same-agent/session-local nonzero timeouts to at least 45s, breaking callers that intentionally set a short timeout to fail fast.
  - confidence: 0.91
- **[P1] Preserve the resolved mirror key for external targets:** `src/infra/outbound/message-action-threading.ts:87-92`
  - body: For a main/webchat run sending to an explicit QQ/Discord/Slack target, `resolvedRoute.sessionKey` is the destination conversation key. Replacing it with `currentSessionKey` before `ensureOutboundSessionEntry` writes the external channel metadata onto the current session and sets `__sessionKey` to the wrong conversation.
  - confidence: 0.9
- **[P1] Keep source provenance on inter-session messages:** `src/agents/tools/sessions-send-tool.ts:587-588`
  - body: Source session/channel are part of the inter-session safety envelope and existing sessions_send tests assert they are carried into `inputProvenance`. Deleting them leaves prompts and transcripts with only `sourceTool`, weakening attribution and policy context for A2A handoffs.
  - confidence: 0.86
