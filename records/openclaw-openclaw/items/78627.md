## Summary

Keep open for maintainer triage: current main does not prove that newly onboarded non-Azure custom NVIDIA providers automatically get `thinking: "medium"`, but the report may still expose a real thinking-precedence gap when explicit/session/global defaults are active.

## What This Changes

The issue asks for unsupported thinking parameters to be disabled for a custom NVIDIA OpenAI-compatible provider, ideally at provider level.

## Best Possible Solution

Decide the thinking precedence contract, then either document the existing per-model `params.thinking: "off"` workaround or add a capability-based clamp for unsupported custom providers without hardcoding NVIDIA in core.

## Reproduction Assessment

No. Source inspection shows plausible explicit/session/global paths to a non-off thinking level, but it does not prove that this newly added non-Azure custom NVIDIA provider auto-sends `medium` on current main.

## Solution Assessment

Unclear. A provider-level disable knob may be useful, but the narrower maintainable solution may be existing per-model `params.thinking: "off"` or capability metadata that suppresses unsupported thinking payloads.
