## Summary

Keep open. Current main still has the reported Telegram isolated-ingress path, and the active fix PR at https://github.com/openclaw/openclaw/pull/81975 references this issue with closing syntax, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that Telegram isolated polling ingress drains spooled updates through `handleUpdate` before the grammY bot is initialized, causing repeated `Bot not initialized` failures.

## Best Possible Solution

Review and land the focused init-before-drain PR, or an equivalent fix with regression coverage, then let the merge close this issue.

## Reproduction Assessment

Yes from source and dependency inspection, though not from a fresh live gateway run. Current main's default isolated path drains through `handleUpdate` without runner initialization, and grammY 1.42.0 throws until `init()` or constructor `botInfo` supplies bot info.

## Solution Assessment

Yes. Initializing the grammY bot before isolated spool drain starts is the narrow maintainable fix; avoid the issue-body `if (!bot.botInfo)` guard because grammY's getter throws before initialization.
