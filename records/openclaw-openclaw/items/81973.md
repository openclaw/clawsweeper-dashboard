## Summary

Keep open. Current main still has the isolated Telegram polling path that drains spooled updates with `handleUpdate` before any explicit grammY initialization, and the issue is already paired with an open focused fix PR that uses closing syntax.

## What This Changes

The issue reports that Telegram isolated polling ingress can stall because spooled updates are dispatched through grammY before the polling bot is initialized.

## Best Possible Solution

Land the narrow initialization fix in https://github.com/openclaw/openclaw/pull/81975 after its merge gates pass, then close this issue through that merged PR.

## Reproduction Assessment

Yes. Current main's isolated ingress path can be followed from source to a direct `handleUpdate()` call without `bot.init()`, and grammY 1.42.0 documents that this throws until bot info is initialized; the issue also has live macOS/Homebrew log confirmation.

## Solution Assessment

Yes. Initializing the existing grammY bot before isolated worker startup and spool drain is the narrow maintainable fix, while keeping transient init failures on the existing recoverable setup retry path.
