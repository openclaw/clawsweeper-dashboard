## Summary

Keep open. Current main still contains the source-proven Telegram isolated ingress initialization bug, and the open focused fix PR should be reviewed or landed before this issue closes.

## What This Changes

The issue reports that Telegram isolated polling ingress drains spooled updates through `handleUpdate()` before the grammY bot is initialized, causing repeated `Bot not initialized` failures.

## Best Possible Solution

Land the linked narrow fix PR or an equivalent change that initializes the bot before isolated spool drain starts, with focused regression coverage and live Telegram proof before merge.

## Reproduction Assessment

Yes from source and dependency-contract inspection, though not from a fresh live gateway run. Current main routes default isolated ingress to direct `handleUpdate()` without initialization, and grammY 1.42.0 throws until `init()` or constructor `botInfo` sets bot info.

## Solution Assessment

Yes. Calling grammY's idempotent `bot.init()` before isolated drain startup, with the existing recoverable setup retry handling, is the narrow maintainable fix; this issue should stay open until the linked PR or equivalent lands.
