Keep open. ## What This Changes

Fix Telegram isolated polling ingress so the grammY bot is initialized before spooled updates are dispatched through `handleUpdate`.

## Best Possible Solution

Land the narrow Telegram initialization fix in https://github.com/openclaw/openclaw/pull/81975 after the normal proof and CI gates, then close this issue through the merged PR.

## Reproduction Assessment

Yes. The source path is clear on current main and v2026.5.12, and the issue discussion includes real deployment logs showing the repeated `Bot not initialized` failure; I did not run a fresh live gateway locally.

## Solution Assessment

Yes. Initializing the grammY bot before isolated spool drain starts is the narrow maintainable fix; avoid checking `bot.botInfo` directly before init because grammY's getter throws when bot info is unavailable.
