## Summary

Keep open: current main and the latest release still pin the official Weixin channel catalog to 2.4.3 while npm now resolves the package to 2.4.6, and post-core update code can still prefer the official catalog target during sync. A straight catalog bump is not obviously safe because the related Weixin 2.4.4 startup report is still open, and the broader exact-pin/latest policy is already tracked separately.

## What This Changes

The issue requests updating the official external Weixin plugin catalog entry from `@tencent-weixin/openclaw-weixin@2.4.3` to 2.4.6 and clarifying whether user pins should survive core-upgrade catalog reconciliation.

## Best Possible Solution

Verify or coordinate a safe Weixin release, then bump the official catalog with matching integrity and explicitly decide/document whether post-core official sync may override a user-pinned official external plugin.

## Reproduction Assessment

Yes, at source level: current main and `v2026.6.10` still pin Weixin to 2.4.3 while npm metadata resolves 2.4.6, and the post-core update path enables official catalog sync. I did not run a full packaged core-upgrade scenario in this read-only review.

## Solution Assessment

No, not as a simple bump: updating to 2.4.6 without resolving the open Weixin 2.4.4 startup report and exact-pin override policy could trade one upgrade problem for another. The safer path is maintainer-approved version validation plus explicit pin/sync semantics.
