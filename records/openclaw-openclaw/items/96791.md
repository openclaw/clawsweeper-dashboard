## What This Changes

The issue requests updating the official external Weixin plugin catalog pin from 2.4.3 to 2.4.6 and preventing official catalog reconciliation from undoing explicit user pins.

## Best Possible Solution

Keep the catalog on a maintainer-approved known-good Weixin release, and make official-plugin sync either preserve explicit user pins or clearly document the intentional override behavior.

## Reproduction Assessment

## Solution Assessment

No, the suggested short-term catalog bump is only a partial fix because the linked Weixin startup regression may make 2.4.6 unsafe. The better path is a maintainer-approved known-good catalog version plus explicit user-pin preservation or documentation.
