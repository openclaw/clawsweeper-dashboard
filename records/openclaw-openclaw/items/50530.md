## Summary

Keep open. Current main still preserves single mid-paragraph newlines through the Telegram Markdown-to-HTML and reply/send delivery paths, and Telegram config/docs do not expose a collapse-newlines option; the related newline-preservation reports make a conservative Telegram-owned option the right boundary.

## What This Changes

The issue requests collapsing soft single newlines in Telegram outbound prose while preserving intentional paragraph, list, quote, and code formatting, possibly via a per-channel option.

## Best Possible Solution

Add a Telegram-owned, Markdown-aware soft-line-break normalization path behind an explicit config option that preserves blank lines, fenced code, quote/list boundaries, escaping, chunk limits, and plain-text fallback behavior.

## Reproduction Assessment

Yes. A static reproduction is markdownToTelegramHtml("Line one\nLine two"): the shared Markdown IR emits a literal newline for the softbreak and the Telegram delivery paths forward the rendered text without normalization.

## Solution Assessment

Yes, if solved as a conservative Telegram-specific option. A shared Markdown IR or generic chunking change would be broader than the request and could regress newline-preservation behavior covered by related reports.
