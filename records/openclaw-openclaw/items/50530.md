## Summary

Keep open. Current main still preserves Telegram single newlines through the Markdown-to-HTML delivery path and exposes no collapse setting, so this remains a product/config decision rather than conservative cleanup.

## What This Changes

Request to add Telegram outbound formatting that collapses soft single newlines inside paragraphs and list items while preserving intentional blank lines, possibly via a Telegram channel option.

## Best Possible Solution

Add a Telegram-owned, Markdown-aware soft-line-break normalization path after maintainers choose the default and config semantics, preserving blank lines, fences, blockquotes, list boundaries, escaping, chunk limits, and plain fallback behavior.

## Reproduction Assessment

Yes at source level: current main maps Markdown softbreaks to literal newlines and forwards Telegram chunks without a collapse pass. I did not run a live Telegram send in this read-only review.

## Solution Assessment

Unclear until maintainers choose product semantics. The maintainable direction is a Telegram-specific, Markdown-aware normalization path, likely behind an explicit config/default decision, not unconditional global newline flattening.
