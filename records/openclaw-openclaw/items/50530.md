Keep open. Current main still has no Telegram soft-newline collapse policy or config surface, and the related closed combined report explicitly leaves the newline/paragraph decision here.

## What This Changes

The issue requests Telegram-owned normalization that collapses soft mid-paragraph single newlines while preserving intentional breaks before sending messages.

## Best Possible Solution

Define a Telegram-plugin formatting policy, likely behind an explicit config/default decision, that collapses prose hard wraps while preserving blank lines, lists, blockquotes, fenced code, captions, rich messages, and plain-text fallbacks.

## Reproduction Assessment

Yes, at source level: current main routes Telegram Markdown/rich text without a soft-newline collapse pass, and existing chunking tests preserve single newlines inside a paragraph. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

Unclear. Telegram plugin ownership is the right layer, but the best solution depends on a maintainer product choice for default versus opt-in behavior and on preserving existing newline-sensitive cases.
