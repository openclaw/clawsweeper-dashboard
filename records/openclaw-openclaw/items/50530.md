## Summary

Keep open. Current main and the latest release still preserve single newlines through Telegram Markdown rendering, there is no collapse-newlines config or formatter policy, and the related combined formatting report explicitly routed the newline/paragraph decision back to this issue.

## What This Changes

Request to collapse soft single newlines in Telegram prose/list output while preserving intentional paragraph breaks, possibly behind a channel option.

## Best Possible Solution

Define a Telegram-owned, Markdown-aware soft-line normalization policy, likely opt-in or explicitly approved as a default, with focused tests and docs for standard HTML, rich messages, captions, code fences, blockquotes, lists, and plain fallback.

## Reproduction Assessment

Yes, at source level: current Telegram send paths pass single-newline Markdown through chunking and HTML/rich rendering without a collapse pass. I did not run a live Telegram delivery proof in this read-only review.

## Solution Assessment

Unclear as a product choice: the Telegram plugin is the right owner, but maintainers need to decide default behavior versus an explicit option because newline preservation is already relied on by related reports.
