## What This Changes

## Best Possible Solution

Add a Telegram-owned, Markdown-aware soft-line-break normalization path after maintainers choose the default and config semantics, preserving blank lines, fences, blockquotes, list boundaries, escaping, chunk limits, and plain fallback behavior.

## Solution Assessment

Unclear until maintainers choose product semantics. The maintainable direction is a Telegram-specific, Markdown-aware normalization path, likely behind an explicit config/default decision, not unconditional global newline flattening.
