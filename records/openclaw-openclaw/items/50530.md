## What This Changes

The issue requests Telegram-specific handling that collapses soft single newlines inside prose before sending while preserving intentional breaks.

## Best Possible Solution

Add a Telegram-owned, Markdown-aware soft-line-break normalization path after maintainers choose default and config semantics, preserving blank lines, fences, blockquotes, list boundaries, escaping, chunk limits, and plain fallback behavior.

## Reproduction Assessment

Yes at source level: Markdown softbreak and hardbreak tokens render as literal newlines, and Telegram delivery passes original or paragraph chunks to the formatter with no collapse option. I did not run a live Telegram send in this read-only review.

## Solution Assessment

Unclear until maintainers choose product semantics. The maintainable direction is a Telegram-specific, Markdown-aware normalization path, likely behind an explicit config/default decision, not unconditional global newline flattening.
