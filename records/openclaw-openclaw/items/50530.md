## Summary

Keep open. Current main still sends Telegram Markdown through chunking and HTML/rich rendering without a soft-newline-collapse policy, the latest release lacks a matching config surface, and the related combined formatting report routed the paragraph/newline policy back to this issue.

## What This Changes

The issue requests Telegram-specific normalization that collapses soft single newlines within paragraphs/list items before delivery while preserving intentional paragraph breaks.

## Best Possible Solution

Define a Telegram-owned, Markdown-aware soft-line policy with a conservative default/config choice that preserves blank lines, code fences, blockquotes, list boundaries, captions, rich messages, and HTML/plain fallback behavior.

## Reproduction Assessment

Yes at source level: current Telegram send paths pass single-newline Markdown through chunking and rendering without a collapse pass. I did not run live Telegram proof in this read-only review.

## Solution Assessment

Unclear as a product choice: Telegram plugin ownership is the right layer, but maintainers need to choose default behavior versus an explicit option because newline preservation is already a shipped expectation.
