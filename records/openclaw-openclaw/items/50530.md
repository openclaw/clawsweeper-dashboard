## Summary

Keep open. Current main and v2026.6.8 still preserve single newlines through Telegram Markdown/rich rendering, and the closed combined formatting report explicitly routes the paragraph/newline policy back here.

## What This Changes

The issue asks Telegram delivery to collapse soft single newlines inside prose while preserving intentional paragraph breaks.

## Best Possible Solution

Define a Telegram-owned, Markdown-aware normalization policy and default/config choice, then implement it with focused tests for prose, lists, blockquotes, fenced code, rich messages, captions, and plain fallback.

## Reproduction Assessment

Yes, at source level: the Markdown renderer maps softbreak and hardbreak tokens to literal newlines, and Telegram send paths pass Markdown through without a collapse policy. I did not run live Telegram proof in this read-only review.

## Solution Assessment

Unclear as a product choice: the Telegram plugin is the right owner, but maintainers need to decide default behavior versus an explicit option because related reports depend on newline preservation.
