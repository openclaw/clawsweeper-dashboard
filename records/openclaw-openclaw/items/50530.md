## Summary

Keep open. Current main and the latest release still preserve Telegram soft line breaks through the Markdown-to-HTML delivery path and expose no collapse option; related newline-preservation reports make this a maintainer product/config decision rather than cleanup.

## What This Changes

Request to collapse single mid-paragraph or list newlines in Telegram outbound messages while preserving intentional double-newline paragraph breaks, possibly via a per-channel option.

## Best Possible Solution

Add a Telegram-owned, Markdown-aware soft-line-break normalization path after maintainers choose default/config semantics, preserving blank lines, fences, blockquotes, lists, chunk limits, escaping, and plain-text fallback behavior.

## Reproduction Assessment

Yes, from source inspection: current main preserves softbreaks as literal newlines and sends Telegram-rendered chunks without a collapse step. I did not run a live Telegram send in this read-only review.

## Solution Assessment

Unclear until maintainers choose product semantics. A Telegram-specific optional normalization path is the narrow maintainable direction; default-on flattening or shared Markdown IR changes would be riskier.
