## Summary

Keep open. Current main still does not provide Telegram soft-newline collapse behavior; recent rich-message work explicitly preserves line breaks, so this remains a maintainer product/config decision rather than a cleanup close.

## What This Changes

Request to add Telegram soft-line-break normalization that collapses single mid-paragraph newlines while preserving intentional breaks, potentially through channel config.

## Best Possible Solution

Define a Telegram-owned, Markdown-aware soft-line-break normalization policy with maintainer-approved default or opt-in config, preserving blank lines, code/pre blocks, blockquotes, lists, rich messages, and plain fallback behavior.

## Reproduction Assessment

Yes, at source level: current Telegram delivery preserves Markdown softbreaks, and the rich path now turns them into <br> rather than collapsing them. I did not run a live Telegram client reproduction in this read-only review.

## Solution Assessment

Unclear as a product choice: the Telegram plugin is the right owner, but unconditional collapse conflicts with existing newline-preservation behavior while an option adds config surface.
