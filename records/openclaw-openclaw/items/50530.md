## Summary

Keep open. Current main still preserves Telegram Markdown soft line breaks through rendering and delivery, and there is no Telegram config/docs/UI option for collapsing mid-paragraph newlines. The request is valid and narrow, with related newline-preservation reports making a Telegram-owned opt-in repair safer than a shared formatter change.

## What This Changes

Request a Telegram-specific normalization option that collapses single soft newlines within paragraphs or list items before delivery while preserving intentional blank-line breaks.

## Best Possible Solution

Add a Telegram-owned, Markdown-aware soft-line-break normalization path, preferably behind an explicit `channels.telegram` config option, and update the Telegram config schema, UI hints, docs, and tests while leaving shared Markdown IR and shared chunking semantics unchanged.

## Reproduction Assessment

Yes. A focused static reproduction is `markdownToTelegramHtml("Line one\nLine two")`: current source maps the Markdown softbreak to a literal newline and Telegram delivery paths send the rendered result without a collapse step.

## Solution Assessment

Yes, if implemented as a conservative Telegram-specific option. A shared Markdown IR or global chunking change would be broader than the report and could regress the newline-preservation behavior that other reports asked to restore.
