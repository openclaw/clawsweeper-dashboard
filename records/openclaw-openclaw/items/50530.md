## Summary

Keep open. Current main still preserves Telegram soft line breaks through Markdown rendering and delivery, and there is no collapse option; related newline-preservation reports make the default/config choice a maintainer product decision rather than a cleanup close.

## What This Changes

Request to collapse soft single newlines inside Telegram prose and list output before delivery while preserving intentional paragraph breaks, possibly via a Telegram config option.

## Best Possible Solution

Add a Telegram-owned, Markdown-aware soft-line-break normalization path behind a maintainer-approved config/default, preserving blank lines, fenced code, blockquotes, list boundaries, escaping, chunk limits, and plain-text fallbacks.

## Reproduction Assessment

Yes, from source inspection. Softbreaks append `\n`, and the Telegram formatter plus reply/direct send paths forward the rendered HTML and fallback text without a normalization step; I did not run a live Telegram send in this read-only review.

## Solution Assessment

Unclear until maintainers choose the product semantics. A Telegram-specific optional normalization path is the narrow maintainable direction; changing shared Markdown IR/chunking or default-on flattening would be broader and riskier.
