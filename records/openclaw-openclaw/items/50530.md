## What This Changes

Requests Telegram outbound formatting that collapses soft single newlines inside prose while preserving intentional paragraph breaks.

## Best Possible Solution

Add a Telegram-owned, Markdown-aware soft-line-break normalization path after maintainers choose default and config semantics, preserving blank lines, fences, blockquotes, list boundaries, tables, escaping, chunk limits, and plain fallback behavior.

## Solution Assessment

Unclear until maintainers choose product semantics. The narrow maintainable path is Telegram-specific, Markdown-aware normalization, likely behind an explicit config/default decision, rather than unconditional global newline flattening.
