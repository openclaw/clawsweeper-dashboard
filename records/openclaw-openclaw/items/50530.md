## Summary

Keep open. Current main still preserves Telegram single newlines through the Markdown-to-HTML path and exposes no collapse option, so this remains a valid Telegram formatting improvement that needs maintainer product/config judgment before implementation.

## What This Changes

Request collapse of single mid-paragraph or list-item newlines in Telegram outbound messages while preserving intentional double-newline breaks.

## Best Possible Solution

Add a Telegram-owned, Markdown-aware soft-line-break normalization path after maintainers choose default and config semantics, preserving blank lines, fences, blockquotes, list boundaries, escaping, chunk limits, and plain fallback behavior.

## Reproduction Assessment

Yes at source level: Markdown softbreak and hardbreak tokens render as literal newlines, and Telegram delivery passes original or paragraph chunks to the formatter with no collapse option. I did not run a live Telegram send in this read-only review.

## Solution Assessment

Unclear until maintainers choose the product semantics. The maintainable direction is a Telegram-specific, Markdown-aware normalization path, likely behind an explicit config/default decision, not unconditional global newline flattening.
