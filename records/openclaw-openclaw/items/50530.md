## Summary

Keep open. Current main still preserves single mid-paragraph newlines through the Telegram Markdown-to-HTML and plain-fallback delivery paths, and Telegram config/docs do not expose a collapse-newlines option. The issue is valid, narrow, and suitable for a conservative Telegram-owned repair, but the fix should avoid a global newline change because related closed reports depended on preserving paragraph breaks.

## What This Changes

Request to collapse single soft line breaks inside Telegram prose or list output before sending while preserving intentional paragraph breaks, optionally through a Telegram config setting.

## Best Possible Solution

Add a Telegram-owned, Markdown-aware soft-line-break normalization path behind an explicit config option, preserving blank lines, fenced code, quote/list boundaries, escaping, chunk limits, and plain-text fallback behavior.

## Reproduction Assessment

Yes. A static reproduction is `markdownToTelegramHtml("Line one\nLine two")`: shared Markdown IR emits a literal newline for the softbreak and both reply/direct Telegram send paths forward the rendered text without normalization.

## Solution Assessment

Yes, if solved as a conservative Telegram-specific option. A shared Markdown IR or generic chunking change would be broader than the request and could regress existing newline-preservation behavior.
