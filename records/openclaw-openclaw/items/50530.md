## Summary

Keep open. Current main still preserves Telegram soft line breaks through Markdown rendering and delivery, and there is no Telegram config/docs option for collapse behavior; this is valid but needs a maintainer product/config decision rather than cleanup closure.

## What This Changes

The issue requests Telegram-specific handling to collapse soft mid-paragraph line breaks while preserving intentional paragraph breaks.

## Best Possible Solution

Add a Telegram-owned, Markdown-aware soft-line-break normalization path behind an explicit config surface, preserving blank lines, fenced code, blockquotes, list boundaries, chunk limits, escaping, and plain-text fallback behavior.

## Reproduction Assessment

Yes. Source inspection shows `markdownToTelegramHtml("Line one\nLine two")` follows Markdown IR softbreak handling that emits a literal newline, and Telegram reply/direct send paths forward the rendered HTML and plain fallback without normalization.

## Solution Assessment

Yes, if scoped as a conservative Telegram-specific option. A shared Markdown IR or global chunking change would be broader than the issue and could regress the newline-preservation behavior covered by related reports.
