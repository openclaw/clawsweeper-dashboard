## Summary

Keep open. Current main still preserves single Markdown soft line breaks through Telegram rendering and delivery, and Telegram config/docs do not expose a collapse-newlines option; the request remains valid but should be implemented conservatively inside the Telegram plugin.

## What This Changes

Collapse single mid-paragraph newlines in Telegram outbound prose while preserving intentional paragraph breaks and structured Markdown.

## Best Possible Solution

Add a Telegram-owned, Markdown-aware soft-line-break normalization option that preserves blank lines, fenced code, blockquotes, list boundaries, escaping, chunk limits, and plain-text fallback behavior.

## Reproduction Assessment

Yes. A static reproduction is the Telegram renderer path for `Line one\nLine two`: Markdown IR emits a literal softbreak newline and the Telegram send paths forward it without normalization.

## Solution Assessment

Yes, if the repair is Telegram-specific and guarded by an explicit config option. A shared Markdown IR or generic chunking change would be broader than the reported Telegram display problem and risks regressing newline preservation.
