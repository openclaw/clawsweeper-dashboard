## Summary

Keep open. Current main and the latest shipped release still preserve Telegram single newlines through the outbound Markdown-to-HTML path and expose no collapse option, so the remaining work is a Telegram formatting/config product decision rather than conservative cleanup.

## What This Changes

Request to add Telegram-specific normalization that collapses single soft line breaks within paragraphs or list items before sending while preserving intentional paragraph breaks.

## Best Possible Solution

Add a Telegram-owned, Markdown-aware soft-line-break normalization path after maintainers choose the default and config semantics, preserving blank lines, fences, blockquotes, list boundaries, escaping, chunk limits, and plain fallback behavior.

## Reproduction Assessment

Yes at source level: Markdown softbreak and hardbreak tokens render as literal newlines, and Telegram delivery passes original or paragraph chunks to the formatter with no collapse option. I did not run a live Telegram send in this read-only review.

## Solution Assessment

Unclear until maintainers choose the product semantics. The maintainable direction is a Telegram-specific, Markdown-aware normalization path, likely behind an explicit config/default decision, not unconditional global newline flattening.
