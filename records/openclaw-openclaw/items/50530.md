## Summary

Keep open. Current main still preserves single newlines through Telegram Markdown/HTML delivery, the rich-message path now explicitly preserves them as `<br>`, and neither current main nor the latest release exposes a soft-newline collapse policy or config surface.

## What This Changes

The issue requests Telegram outbound formatting that collapses soft single newlines within prose while preserving intentional paragraph breaks.

## Best Possible Solution

Define a Telegram-owned, Markdown-aware soft-line-break normalization policy with a maintainer-selected default or explicit option, preserving blank lines, code/pre blocks, blockquotes, lists, rich messages, and plain fallback behavior.

## Reproduction Assessment

Yes, at source level: current main maps Markdown softbreaks to literal newlines and Telegram delivery passes Markdown through without a collapse policy. I did not run a live Telegram client reproduction in this read-only review.

## Solution Assessment

Unclear as a product choice: the Telegram plugin is the right owner, but an unconditional collapse default could conflict with existing newline-preservation behavior, while an option adds config surface.
