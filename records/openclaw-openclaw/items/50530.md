## Summary

Keep open. Current main still preserves single soft line breaks through the Telegram Markdown-to-HTML and delivery paths, and Telegram config/docs do not expose a collapse-newlines option; the repair should stay Telegram-specific and conservative because related newline-preservation reports depended on keeping intentional breaks.

## What This Changes

Request to collapse single mid-paragraph newlines in Telegram delivery while preserving intentional paragraph breaks.

## Best Possible Solution

Add a Telegram-owned, Markdown-aware soft-line-break normalization path behind an explicit config option, preserving blank lines, fenced code, quote/list boundaries, escaping, chunk limits, and plain-text fallback behavior.

## Reproduction Assessment

Yes. A static reproduction is tracing `markdownToTelegramHtml("Line one\nLine two")`: shared Markdown IR emits a literal newline for the softbreak, and both reply and direct Telegram send paths forward the rendered text without normalization.

## Solution Assessment

Yes, if implemented as a conservative Telegram-specific option. A shared Markdown IR or generic chunking change would be broader than this issue and could regress existing newline-preservation behavior.
