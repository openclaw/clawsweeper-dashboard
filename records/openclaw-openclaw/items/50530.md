## Summary

Keep open. Current main and the latest release still preserve Telegram soft line breaks through Markdown rendering and delivery, and there is no collapse option; inverse newline-preservation reports make this a maintainer product/default decision rather than a cleanup close.

## What This Changes

Request a Telegram outbound formatting behavior or option that collapses single soft line breaks inside prose/list items while preserving intentional blank lines.

## Best Possible Solution

Add a Telegram-owned, Markdown-aware soft-line-break normalization path behind maintainer-approved config/default semantics, preserving blank lines, fences, blockquotes, list boundaries, escaping, chunk limits, and plain-text fallbacks.

## Reproduction Assessment

Yes, from source inspection: current main preserves softbreaks as literal newlines and forwards Telegram-rendered chunks without a normalization step. I did not run a live Telegram send in this read-only review.

## Solution Assessment

Unclear until maintainers choose product semantics. A Telegram-specific optional normalization path is the narrow maintainable direction; changing shared Markdown IR/chunking or default-on flattening would be broader and riskier.
