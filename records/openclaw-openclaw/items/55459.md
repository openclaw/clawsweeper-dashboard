## Summary

Keep open: current main still ships the 1Password exec SecretRef recipe without cache disablement, and the generic exec resolver only applies configured args/env. The broader batching issue #72278 does not supersede this backend-specific documentation gap.

## What This Changes

The issue requests that 1Password `op` exec SecretRef usage avoid the macOS Tahoe daemon/cache hang, either by code behavior, config shape, or documentation.

## Best Possible Solution

Update the 1Password exec SecretRef docs and examples to disable `op` caching and pass the needed service-account env, while leaving generic provider batching or `op` auto-detection to separate tracked work.

No, not as a live high-confidence reproduction: this review did not run macOS Tahoe with 1Password CLI. Source inspection does show current main would invoke the documented `op read` example without the cache-disabling flag/env that upstream documents.

## Solution Assessment
