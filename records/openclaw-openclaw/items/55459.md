Keep open: current main still documents and references 1Password `op read` without `--cache=false` or `OP_CACHE=false`, while the exec resolver only passes explicitly configured args/env, so the reporter's workaround remains unimplemented.

## What This Changes

Request documents or changes the exec SecretRef provider so 1Password `op read` runs with caching disabled in macOS LaunchAgent/service-account setups.

## Best Possible Solution

Update the gateway SecretRef docs and bundled 1Password skill references to disable `op` caching for LaunchAgent/service-account usage, while leaving core autodetection or broader batching changes to separate tracked work.
