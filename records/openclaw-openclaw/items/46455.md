## Summary

Keep this PR open, but it should not merge as-is. Current main already shipped the zero-valued poll default tolerance, while this branch still weakens explicit poll-intent validation and handles empty Discord component skeletons too high in the shared send path.

## What This Changes

The PR strips poll and empty component parameters from message sends, narrows poll intent detection, and updates poll/send regression tests.

## Best Possible Solution

Keep the shipped zero-default poll tolerance, make one explicit maintainer decision for hallucinated non-zero poll params, and handle empty Discord component skeletons in the Discord parser/runtime so all consumers are covered.

## Reproduction Assessment

Yes. Current-main tests and source reproduce the poll behavior, and the Discord parser path is source-reproducible with a `components.modal.fields: []` payload.

## Solution Assessment

No. The branch contains useful context, but the safer solution is narrower and lower-level: preserve or intentionally revise poll-intent policy in one canonical PR, and normalize empty Discord specs at the parser boundary.
