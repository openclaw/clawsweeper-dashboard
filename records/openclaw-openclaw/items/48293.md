## Summary

Keep open. Current main still lacks this transcript-tail detector, but the PR is draft, stacked on an open config-scaffold dependency, and has a concrete detector false negative around timestamp-bearing reminder text.

## What This Changes

The branch adds a pure transcript-tail detector and focused unit tests for loop-risk signals, while carrying the stacked guard-config scaffold from its dependency.

## Best Possible Solution

Keep this as the staged detector candidate, narrow the timestamp exemption, add regression coverage, rebase with the config scaffold, and land only after focused validation plus contributor proof.

## Reproduction Assessment

Yes for the review finding: source inspection shows a repeated directive/reminder entry with an appended `Current time:` line is exempted before stale recurrence counting. Not applicable as a standalone bug report because the PR is a new feature module.

## Solution Assessment

No, not as-is. A pure detector is the right boundary for this stage, but the timestamp exemption should be limited to standalone refresh entries or known post-compaction prefixes before runtime consumers depend on it.
