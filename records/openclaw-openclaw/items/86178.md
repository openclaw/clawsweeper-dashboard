## Summary

## What This Changes

Report that the managed Memory Dreaming Promotion cron can run silently without producing fresh Dream Diary, session-ingestion, or durable-memory promotion output.

## Best Possible Solution

Keep this issue as the tracker for managed Dreaming cron status and visibility until maintainers decide and prove how failed or no-op sweeps should surface after the SQLite migration.

## Solution Assessment

Unclear. The SQLite migration is the right fix for the legacy JSON and WSL2 sidecar failures, but the remaining best fix depends on maintainer direction for how failed or no-op Dreaming sweeps should be reflected in cron status, doctor, logs, or diagnostics.
