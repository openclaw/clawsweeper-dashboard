Full review comments:

- **[P1] Verify the macOS asset names, not the total count:** `.github/workflows/verify-release-assets.yml:56-58`
  - body: The workflow succeeds as soon as any three assets are present. Current stable publishing can add non-macOS assets such as dependency evidence and Windows Companion artifacts, so a release with no `OpenClaw-${VERSION}` macOS zip/dmg/dSYM assets can still satisfy this check. Please poll the release asset names and require the exact macOS contract.
  - confidence: 0.93
- **[P2] Do not mask release API failures as missing assets:** `.github/workflows/verify-release-assets.yml:53-54`
  - body: The `|| echo "0"` path converts every `gh api` failure into a zero-asset poll, including auth, rate-limit, and GitHub outage errors. That makes an infrastructure/API failure look like a delayed private publish and waits for the full timeout instead of surfacing the real cause.
  - confidence: 0.88
- **[P2] Check once more before timing out:** `.github/workflows/verify-release-assets.yml:60`
  - body: The final loop iteration sleeps for another minute and then reports the previous asset count without a final API read. Assets that appear during that final minute are reported missing even though they arrived within the intended 45-minute window.
  - confidence: 0.84
