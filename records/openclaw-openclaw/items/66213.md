## Summary

## What This Changes

The PR adds a GitHub Actions workflow that polls stable GitHub releases for macOS assets and warns on appcast drift, plus a CODEOWNERS rule for root appcast.xml.

## Best Possible Solution

Land or replace a release-owner-approved verifier that preserves real API failures, polls through the full intended window, and enforces the documented macOS asset/appcast invariant while keeping broader updater atomicity under https://github.com/openclaw/openclaw/issues/48306.
