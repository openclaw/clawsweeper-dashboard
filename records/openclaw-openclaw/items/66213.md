## Summary

Keep open. Current main still lacks the proposed post-publish macOS asset verifier and root appcast CODEOWNERS rule, and the release docs support the invariant, but the PR is not merge-ready because the polling loop can misreport API failures as missing assets and can miss assets uploaded during the final wait window.

## What This Changes

Adds a GitHub Actions workflow that polls stable releases for three macOS assets and warns on missing appcast entries, plus a CODEOWNERS rule for the root appcast feed.

## Best Possible Solution

Land a release-owner-approved post-publish guard after fixing the polling loop so real API failures stay visible and the full wait window is checked, while leaving broader updater atomicity to #48306.

## Reproduction Assessment

Yes. Current main lacks the verifier and appcast CODEOWNERS entry, the public v2026.4.12 release still has zero assets, and the two PR defects are reproducible by static inspection of the shell loop.

## Solution Assessment

No, not yet. The direction is narrow and maintainable, but the workflow should not collapse gh api failures into zero assets and should recheck before timing out.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Fail real API errors instead of counting them as zero assets:** `.github/workflows/verify-release-assets.yml:53-54`
  - body: The `|| echo "0"` fallback makes every `gh api` failure look like a release with no assets. A 403, rate limit, outage, or missing release will burn the full polling loop and report the wrong incident cause instead of surfacing the API failure.
  - confidence: 0.91
- **[P2] Recheck assets before declaring the timeout:** `.github/workflows/verify-release-assets.yml:60-62`
  - body: The loop sleeps after the 45th poll and then fails without another API call. If the private publish uploads assets during that final minute, the workflow reports a missing-asset failure even though assets arrived within the intended waiting window.
  - confidence: 0.88
