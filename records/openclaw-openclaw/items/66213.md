## Summary

Keep open. Current main still lacks the proposed post-publish macOS asset verifier and `/appcast.xml` CODEOWNERS rule, and the reported stable-release asset gap is substantiated; however, the PR’s polling loop has two concrete reliability bugs that should be fixed before merge.

## What This Changes

The PR adds a GitHub Actions workflow that polls stable release assets after publication and adds a CODEOWNERS rule for the root Sparkle appcast feed.

## Best Possible Solution

Land a release-owner-approved guard that checks stable macOS release assets and appcast consistency without masking GitHub API failures or falsely failing the final polling window, while keeping the broader updater reliability work tracked by #48306.

## Reproduction Assessment

Yes. Current main lacks the verifier and appcast CODEOWNERS entry, and the public release API reproduces the motivating case: `v2026.4.12` has zero assets while comparable beta/current stable releases have three macOS assets. The two PR defects are reproducible by static inspection of the shell loop.

## Solution Assessment

No, not yet. The direction is narrow and maintainable, but the loop should preserve real `gh api` errors and perform a final check before declaring timeout; after that, release-owner review is the right merge gate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Fail real API errors instead of counting them as zero assets:** `.github/workflows/verify-release-assets.yml:53-54`
  - body: The `|| echo "0"` fallback makes every `gh api` failure look like a release with no assets. A 403, rate limit, GitHub outage, or missing release will burn the full 45-minute loop and report the wrong incident cause instead of failing with the real API error.
  - confidence: 0.91
- **[P2] Recheck assets before declaring the timeout:** `.github/workflows/verify-release-assets.yml:60-62`
  - body: The loop sleeps after the 45th poll and then fails without another API call. If the private publish uploads assets during that final minute, the workflow reports a missing-asset failure even though the assets arrived within the intended waiting window.
  - confidence: 0.88
