## Summary

Keep open: current main still lacks the focused public verifier and appcast ownership guard, and this PR remains useful but has two workflow correctness defects plus missing real behavior proof.

## What This Changes

The PR adds a GitHub Actions workflow that polls stable macOS release assets after publish and adds a root appcast.xml CODEOWNERS rule.

## Best Possible Solution

Land or replace a release-owner-approved verifier that checks stable macOS assets and appcast ownership without masking API failures or missing the final polling window; keep broader updater atomicity and version-skew work under https://github.com/openclaw/openclaw/issues/48306.
