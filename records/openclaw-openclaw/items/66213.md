## Summary

Keep open: current main still lacks this public post-publish verifier and root appcast ownership guard, but the PR still has two polling correctness bugs and missing real dispatch proof before merge.

## What This Changes

The PR adds a GitHub Actions workflow that polls stable releases for macOS assets and warns on appcast drift, plus a root CODEOWNERS rule for appcast.xml.

## Best Possible Solution

Land or replace a release-owner-approved verifier that preserves real API failures, polls through the full intended window, and keeps appcast ownership aligned with documented release readiness.

## Reproduction Assessment

Yes for the release-artifact failure class and the PR defects: historical releases v2026.4.12 and v2026.3.13-1 still have zero assets, current main has no verifier, and the polling issues are visible in the head workflow. I did not reproduce a current latest-release outage because v2026.5.7 has the expected assets.

## Solution Assessment

No, not yet. The direction is narrow, but the workflow should fail real API errors, avoid the final-window false negative, and include real dispatch proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Fail API errors instead of reporting zero assets:** `.github/workflows/verify-release-assets.yml:53-54`
  - body: The polling loop turns any `gh api` failure into asset count `0`, so auth, rate-limit, missing-release, or GitHub outage errors wait the full polling window and then report the wrong missing-asset diagnosis. This verifier should surface real API failures separately instead of hiding them as a not-ready release.
  - confidence: 0.91
- **[P2] Check again before declaring the timeout:** `.github/workflows/verify-release-assets.yml:60-62`
  - body: The loop sleeps after the 45th poll and then exits without another API call, so assets uploaded during the last minute of the intended 45-minute window are still reported missing. Sleep only when another check will run, or perform a final check before failing.
  - confidence: 0.88
