The PR adds a GitHub Actions workflow that verifies stable macOS release assets after publication and adds a root CODEOWNERS rule for appcast.xml.

## Best Possible Solution

Land or replace a release-owner-approved post-publish verifier that preserves real API failures, polls through the intended window, and keeps appcast ownership aligned with stable macOS release readiness.

## Reproduction Assessment

## Solution Assessment

No, not yet. The direction is narrow and useful, but the workflow should fail real API errors, avoid the final-window false negative, and include real dispatch proof before merge.
