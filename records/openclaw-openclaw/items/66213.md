## Summary

Keep open: current main still lacks the post-publish verifier and root appcast ownership guard, and the release-asset failure class is real, but this PR still has two workflow correctness defects and no after-fix real behavior proof.

## What This Changes

The PR adds a GitHub Actions workflow that polls stable macOS release assets after publication and adds CODEOWNERS ownership for appcast.xml.

## Best Possible Solution

Land or replace a release-owner-approved verifier that preserves API failures, polls through the intended window, and keeps appcast ownership aligned with the release docs.

## Reproduction Assessment

Yes for the release-artifact failure class and the PR defects: GitHub release API still shows v2026.4.12 with zero assets, current main has no verifier workflow, and the two polling bugs are visible in the PR head source. I did not run the new workflow because this was a read-only review and the PR lacks posted dispatch logs.

## Solution Assessment

No, not yet. The direction is narrow and useful, but the workflow should fail real API errors, avoid the final-window false negative, and include real dispatch or log proof before merge.
