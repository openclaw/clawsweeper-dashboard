## Summary

Keep open: this PR targets a real source-reproducible memory-wiki lint bug, but the current patch still has blocking correctness issues and lacks real behavior proof for the added bridge self-heal path.

## What This Changes

The PR expands memory-wiki lint target normalization for titles, aliases, fragments, and relative markdown links, then adds an imported-source self-heal path with tests.

## Best Possible Solution

Merge a narrow memory-wiki fix that preserves case-sensitive path validation, case-folds title and alias matches separately, verifies group-specific imported-source provenance before skipping repair, and includes real proof for the self-heal path.

## Reproduction Assessment

Yes. Source inspection shows current main only indexes path-shaped link targets, and the PR body plus related reports provide live-vault examples; I did not execute the lint command in this read-only pass, so this is source-reproducible rather than locally reproduced.

## Solution Assessment

No. The direction is right, but the implementation should separate path-sensitive matching from normalized title and alias matching, and the repair guard should require generated group provenance before skipping.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve path casing during lint normalization:** `extensions/memory-wiki/src/lint.ts:55`
  - body: Lowercasing every normalized target makes path-based links case-insensitive. In a native markdown vault on a case-sensitive filesystem or Git checkout, `[Alpha](sources/Alpha.md)` will now pass when the actual file is `sources/alpha.md`, suppressing a real broken-link warning. Keep path keys case-preserving and case-fold only titles and aliases.
  - confidence: 0.9
- **[P2] Require provenance fields before skipping repair:** `extensions/memory-wiki/src/source-page-shared.ts:18`
  - body: This completeness check can return true for an imported source page that still lacks bridge or unsafe-local provenance fields. When sync state is current, such a page is skipped here and existing `missing-import-provenance` lint warnings remain until the source changes. Include the group-specific generated fields in this completeness test before returning unchanged.
  - confidence: 0.88
