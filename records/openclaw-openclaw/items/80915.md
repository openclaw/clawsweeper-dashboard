## Summary

Keep open: this is a plausible focused bug fix for current memory-wiki lint behavior, but the current PR head has two blocking correctness issues and incomplete real behavior proof for the added bridge self-heal path.

## What This Changes

The PR updates memory-wiki lint target normalization for title and alias wikilinks, then adds bridge imported-source self-repair logic and regression tests.

## Best Possible Solution

Merge a narrow memory-wiki fix that preserves case-sensitive path validation, case-folds only title and alias lookup keys, requires group-specific imported-source provenance before skipping repair, and keeps broader markdown-aware extraction work in the existing related issues.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main only accepts path-shaped lint targets while title-style Obsidian targets are extracted into linkTargets, and the PR body provides terminal after-fix proof for that title-link false positive.

## Solution Assessment

No. The direction is right, but this implementation over-normalizes path targets and under-validates imported-source provenance before skipping repair; the safer fix separates path-sensitive keys from case-folded title and alias keys and checks the generated provenance contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve path casing during lint normalization:** `extensions/memory-wiki/src/lint.ts:55`
  - body: Lowercasing every normalized target makes relative path links case-insensitive. On a case-sensitive filesystem or Git checkout, `[Alpha](sources/Alpha.md)` would now pass when the actual file is `sources/alpha.md`, hiding a real broken-link warning. Preserve case for path and basename keys, and case-fold only title and alias lookup.
  - confidence: 0.9
- **[P2] Require group provenance before skipping repair:** `extensions/memory-wiki/src/source-page-shared.ts:18`
  - body: This helper can return true for a bridge or unsafe-local source page that has `sourcePath` and `## Content` but is still missing the group-specific provenance fields. When sync state is current, `writeImportedSourcePage` will return unchanged and the existing `missing-import-provenance` lint warning remains. Require the generated bridge or unsafe-local fields before skipping.
  - confidence: 0.88
