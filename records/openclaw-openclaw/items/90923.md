- `P1`: The PR addresses a broken macOS gateway install and restart workflow that prevents external-APFS-home users from running the managed gateway.

## Summary

Keep open for maintainer merge review: current main still has the external-APFS LaunchAgent failure, and this PR is a focused, proof-backed daemon fix, but the managed plist location change is upgrade-sensitive.

## What This Changes

The branch adds a boot-volume-aware LaunchAgent home resolver and routes macOS daemon install, restart, uninstall, doctor scan, supervisor restart, update restart helper, and regression tests through it.

## Best Possible Solution

Land the single canonical boot-volume-aware LaunchAgent resolver after maintainer acceptance of the external-home plist policy, with macOS app plist introspection either explicitly accepted as follow-up scope or aligned before merge.

## Reproduction Assessment

Yes. Current main builds and bootstraps the plist path from HOME, and the linked report plus PR live output show launchd error 5 when that plist is on an external APFS volume.

## Solution Assessment

Yes for the daemon path: a single launchd-specific resolver reused by install, repair, restart, uninstall, and doctor is narrower and safer than an install-only copy. The only solution-fit question is whether the macOS app plist snapshot reader should be aligned in this PR or handled as follow-up.
