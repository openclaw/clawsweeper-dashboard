## Summary

Keep this PR open: current main and the latest release still use HOME-derived macOS LaunchAgent plist paths, and this PR is the active viable fix candidate. The patch is broadly correct, with one low-severity stale manual-hint issue and compatibility/availability risk that maintainers should explicitly accept before merge.

## What This Changes

The PR adds a shared boot-volume-aware LaunchAgent home resolver and routes macOS daemon, restart, update-recovery, inspection, app lookup, and regression tests through it for external-APFS HOME setups.

## Best Possible Solution

Land the shared boot-volume LaunchAgent resolver after maintainer acceptance of the external-home plist-location change, with manual hint cleanup either before merge or tracked as a narrow follow-up.

## Reproduction Assessment

Yes. Current main and v2026.6.8 build the LaunchAgent plist under HOME, while the linked issue and PR terminal proof show launchctl error 5 for an external-APFS plist and success from the boot-volume plist; I did not run a fresh macOS external-volume test in this checkout.

## Solution Assessment

Yes, with a small polish gap. A shared LaunchAgent-home resolver reused by every lifecycle path is the maintainable fix; the earlier install-only copy split state, while this PR routes the real bootstrap/restart/update/app lookup paths through one canonical path.
