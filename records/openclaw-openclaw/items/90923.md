## Summary

Keep open: this is the active, proof-positive fix candidate for the external-APFS-home LaunchAgent bootstrap failure. ## What This Changes

The branch adds a boot-volume-aware macOS LaunchAgent home resolver and routes daemon lifecycle, update recovery/restart helpers, macOS app lookup, and regression tests through it for external APFS homes.

## Best Possible Solution

Land the shared boot-volume LaunchAgent resolver after maintainer acceptance of the external-home plist-location change, ideally fixing or explicitly deferring the cleanup-hint mismatch.

## Reproduction Assessment

Yes. Current main and v2026.6.8 derive the LaunchAgent plist from HOME, and the linked issue plus PR terminal proof show launchctl error 5 for an external-APFS plist and success from the boot-volume path.

## Solution Assessment

Yes, with one polish gap. A single LaunchAgent-home resolver is the maintainable fix because it avoids split install-only state and now covers lifecycle, recovery, and app lookup paths; the manual cleanup hint should use the same path or be tracked as follow-up.
