## Summary

Keep open. Latest head resolved the earlier `lastHash` length and `config.patch` write concerns, but agent-scoped session lists still key the cache on every templated store, and the public Gateway protocol docs still omit the new hash/unchanged contract.

## What This Changes

The branch adds a gateway-side `sessions.list` result cache with `hash`/`lastHash` unchanged responses, `config.patch` no-op/source-shape fixes, invalidation signals, generated Swift protocol updates, tests, and a changelog entry.

## Best Possible Solution

Keep the additive backend cache, but scope store fingerprinting to the same target set as `loadCombinedSessionStoreForGateway` and document `hash`, `lastHash`, and `{ unchanged: true }` in the public Gateway protocol docs before landing.

## Reproduction Assessment

Yes. With templated per-agent session stores, call `sessions.list` with `agentId`, then touch an unrelated agent store; the PR cache key changes even though the miss path is agent-scoped. The docs gap is reproducible by comparing the schema addition with `docs/gateway/protocol.md`.

## Solution Assessment

No, not yet. The backend approach is backward-compatible, but the mergeable solution needs agent-scoped store fingerprints plus public protocol documentation.
