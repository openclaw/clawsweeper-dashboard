The PR adds a Gateway `sessions.list` result cache with `lastHash`/`hash` unchanged responses, config/session/transcript/subagent invalidation inputs, config.patch no-op fixes, generated Swift protocol updates, tests, and a changelog entry.

## Best Possible Solution

Document the additive `sessions.list` cache contract in the public Gateway protocol docs, then land the backend after protocol generation and targeted gateway/config/subagent tests pass on the exact PR head.

Yes.
