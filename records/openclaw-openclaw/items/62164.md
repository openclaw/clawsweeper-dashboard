## Summary

Keep open. The latest head still over-scopes the cache key for agent-scoped session lists and lacks public protocol docs; the earlier `lastHash` max-length note is resolved, but these blockers remain before merge.

## What This Changes

The PR adds a gateway `sessions.list` result cache with `hash`/`lastHash` unchanged responses, config patch no-op fixes, subagent/transcript invalidation signals, Swift protocol baselines, tests, and a changelog entry.

## Best Possible Solution

Keep the additive backend cache, but align store fingerprinting with the same requested-agent target set as the cache-miss loader and document the public hash contract before merge.

## Reproduction Assessment

Yes. With templated per-agent session stores, call `sessions.list` with `agentId` and then mutate an unrelated agent store; the PR cache key changes because it fingerprints all stores while the miss path loads only the requested agent. The docs gap is reproducible by comparing the PR schema diff with `docs/gateway/protocol.md`.

## Solution Assessment

No, not yet. The overall backend approach is maintainable and backward-compatible, but it needs agent-scoped store fingerprinting plus public protocol documentation to be the narrow mergeable solution.
