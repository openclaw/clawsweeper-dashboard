## Summary

Keep open. The latest PR head is still not mergeable: the new cache key fingerprints all templated agent stores for agent-scoped `sessions.list` calls, the public Gateway protocol docs do not describe the new `lastHash`/`hash`/`unchanged` contract, and the changelog includes an unrelated alpha-release bullet.

## What This Changes

The branch adds a gateway-side `sessions.list` result cache with `hash`/`lastHash` unchanged responses, config snapshot/no-op fixes, subagent/transcript invalidation signals, generated Swift models, tests, and changelog entries.

## Best Possible Solution

Keep the additive backend cache, but scope store fingerprinting to the same target set as `loadCombinedSessionStoreForGateway`, document the public Gateway protocol contract, and remove the unrelated changelog bullet before landing.

## Reproduction Assessment

Yes. With templated per-agent session stores, call `sessions.list` with `agentId`, then touch an unrelated agent store; the PR cache key changes even though the miss path is agent-scoped. The docs and changelog gaps are directly reproducible by comparing the PR diff with `docs/gateway/protocol.md` and `CHANGELOG.md`.

## Solution Assessment

No, not yet. The backend approach and split UI/Apple follow-ups are reasonable, but the mergeable solution needs agent-scoped store fingerprints, public protocol docs, and an accurate changelog.
