## Summary

Keep open. The latest head still needs changes before merge: it adds the cache/unchanged response path, but the cached and hashed rows do not preserve current main's `hasActiveRun` session-list contract, and the public Gateway protocol docs are not updated for `lastHash`, returned `hash`, or the unchanged response shape.

## What This Changes

This PR adds a Gateway `sessions.list` result cache with `lastHash`/`hash` unchanged responses, config/session/transcript/subagent invalidation inputs, config patch no-op fixes, generated Swift models, tests, a benchmark, and a changelog entry.

## Best Possible Solution

Update the branch onto current main, make the cache hash and payload reflect the exact visible `sessions.list` rows including active-run state, document the additive protocol contract, and rerun the focused Gateway/config/subagent/protocol checks.

## Reproduction Assessment

Yes. Current main has a focused `sessions.list` test for `hasActiveRun`, and the latest PR head can be inspected to show the cache hash/write/response path uses raw `result.sessions` before that enrichment.

## Solution Assessment

No, not yet. A backend result cache with additive `lastHash` is a reasonable direction, but it is not the best mergeable solution until it preserves current active-run row semantics and updates the public protocol docs.
