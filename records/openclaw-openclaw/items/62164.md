## Summary

Keep open. The latest PR head still has two merge-blocking review issues: agent-scoped `sessions.list` cache keys fingerprint all agent stores, and the additive `lastHash` / `hash` / unchanged response protocol remains undocumented.

## What This Changes

Adds Gateway `sessions.list` result caching with `lastHash` unchanged responses, config.patch source/no-op fixes, subagent/transcript invalidation signals, generated Swift protocol updates, tests, a benchmark, and a changelog entry.

## Best Possible Solution

Keep the additive backend cache, but scope store fingerprints to the same agent store set as the miss path and document the public hash contract before merge.

## Reproduction Assessment

Yes. Configure templated per-agent session stores and call `sessions.list` with `agentId`; the PR cache key fingerprints every agent store while the load path is scoped to the requested agent. The docs gap is reproduced by comparing the PR schema to `docs/gateway/protocol.md`.

## Solution Assessment

No, not yet. The cache direction is maintainable, but the mergeable solution must preserve current agent-scoped performance semantics and update the public protocol docs.
