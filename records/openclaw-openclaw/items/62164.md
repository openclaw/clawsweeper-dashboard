## Summary

Keep open. The latest head addresses the earlier `lastHash` length nit, but still has two merge blockers: agent-scoped `sessions.list` requests use an all-store cache fingerprint, and the public Gateway protocol docs do not describe the new hash/unchanged contract.

## What This Changes

The PR adds a Gateway `sessions.list` result cache with `lastHash`/`hash`/`unchanged` protocol fields, config.patch no-op/source-shape fixes, subagent/transcript invalidation signals, Swift protocol baselines, tests, a benchmark, and a changelog entry.

## Best Possible Solution

Keep the additive backend cache, but make store fingerprinting use the same requested-agent target set as the cache-miss loader and document the public hash contract before merge.

## Reproduction Assessment

Yes. On the PR head, configure templated per-agent session stores and call `sessions.list` with `agentId`; an unrelated agent store write changes the cache key because it fingerprints all stores. The docs gap is reproducible by comparing the PR schema to `docs/gateway/protocol.md`.

## Solution Assessment

No, not yet. The cache direction is maintainable and additive, but the mergeable solution needs agent-scoped fingerprinting plus public protocol documentation for the new request and response behavior.
