## Summary

Keep this PR open, but it still needs changes before merge. The backend cache direction is useful and current main does not already implement it, but the PR cache key still does global store fingerprinting for agent-scoped lists and the public Gateway protocol docs do not describe the new hash/unchanged contract.

## What This Changes

The PR adds a Gateway `sessions.list` result cache with `lastHash`/`unchanged` responses, config.patch source no-op fixes, subagent/transcript invalidation signals, generated Swift protocol updates, tests, a benchmark, and a changelog entry.

## Best Possible Solution

Keep the additive backend cache, but scope store fingerprinting to the same agent target set as the cache-miss load path and document the public hash contract before merge.

## Reproduction Assessment

Yes. Configure templated per-agent session stores and call `sessions.list` with `agentId`; the PR cache key fingerprints all agent stores while the miss path is scoped to the requested agent. The docs gap is reproduced by comparing the PR schema to `docs/gateway/protocol.md`.

## Solution Assessment

No, not yet. The cache direction is maintainable, but the mergeable solution needs agent-scoped fingerprints and public protocol documentation for the new request/response behavior.
