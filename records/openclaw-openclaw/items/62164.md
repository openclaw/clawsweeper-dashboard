## Summary

Keep open. The latest head fixes the earlier active-run cache concern and the Greptile hardening comments, but it still regresses the current-main agent-scoped `sessions.list` store path and leaves the new public `lastHash`/`hash`/`unchanged` protocol undocumented.

## What This Changes

This PR adds a Gateway `sessions.list` result cache with `lastHash` unchanged responses, config/session/transcript/subagent invalidation inputs, config patch no-op fixes, generated Swift models, tests, a benchmark, and a changelog entry.

## Best Possible Solution

Land the backend cache only after the cache miss path and cache fingerprint use the same agent-scoped store set as current main, and document the additive protocol contract while keeping client adoption separate.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure templated per-agent session stores, call `sessions.list` with `agentId`, then observe the PR cache miss and cache key still read/fingerprint unrelated agent stores; the docs gap is reproduced by comparing the PR schema to `docs/gateway/protocol.md`.

## Solution Assessment

No, not yet. The additive backend cache is a reasonable direction, but it is not the best mergeable solution until it preserves current agent-scoped `sessions.list` performance semantics and updates the public protocol docs.
