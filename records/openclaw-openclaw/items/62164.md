## Summary

Keep open. The latest PR head fixed the earlier cache-miss store load to pass `agentId`, but the new cache key still fingerprints every templated agent store and the public Gateway protocol docs still do not describe the new `lastHash`/`hash`/`unchanged` contract.

## What This Changes

The PR adds a Gateway `sessions.list` result cache with `lastHash` unchanged responses, config patch source/no-op fixes, subagent and transcript invalidation signals, generated Swift protocol params, tests, a benchmark, and a changelog entry.

## Best Possible Solution

Keep the backend cache additive, but make cache fingerprinting use the same agent-scoped store target set as `sessions.list` and document the public protocol contract before merging.

## Reproduction Assessment

Yes. Configure templated per-agent session stores, call `sessions.list` with `agentId`, and the latest PR still builds the cache key from every agent store even though the miss path loads only the requested agent; the docs gap is reproduced by comparing the PR schema to `docs/gateway/protocol.md`.

## Solution Assessment

No, not yet. The cache direction is reasonable, but the mergeable solution should preserve current agent-scoped performance semantics and update the public Gateway protocol docs.
