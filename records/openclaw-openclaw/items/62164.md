## Summary

Keep open. The latest PR head still has narrow merge blockers: scoped `sessions.list` requests are keyed by all templated stores, the public Gateway protocol docs omit the new hash contract, and the changelog includes an unrelated alpha-release note. The linked UI and Apple follow-up PRs are useful context, but this backend slice is not merge-ready yet.

## What This Changes

This PR adds a Gateway `sessions.list` result cache with `hash`/`lastHash` unchanged responses, config.patch source/no-op handling, subagent/transcript invalidation signals, generated Swift protocol model updates, tests, and changelog notes.

## Best Possible Solution

Keep the additive backend cache, but make store fingerprinting match the agent-scoped loader, document the public protocol contract, and limit the changelog to behavior implemented by this PR before landing.

## Reproduction Assessment

Yes. With templated per-agent session stores, call `sessions.list` with `agentId`, then touch an unrelated agent store; the PR cache key changes even though the cache-miss path is agent-scoped. The docs and changelog gaps are directly reproducible by comparing the PR head with `docs/gateway/protocol.md` and `CHANGELOG.md`.

## Solution Assessment

No, not yet. The backend cache direction and split UI/Apple follow-ups are reasonable, but the mergeable solution needs scoped store fingerprints, public protocol docs, and an accurate changelog.
