## Summary

Keep open: this targets a real TUI editing problem, but the patch is not safe because it canonicalizes matched Backspace input to a byte that the pinned key matcher treats as Ctrl+Backspace in Windows Terminal sessions. Real WSL2 post-response proof is also missing.

## What This Changes

The PR changes the TUI Backspace deduper to return ASCII BS for every recognized Backspace representation and updates focused deduper tests accordingly.

## Best Possible Solution

Keep the canonical bug open and replace this normalization with a repro-backed fix that preserves pi-tui's key contract and proves Backspace after an agent response.

## Reproduction Assessment

Yes for the patch regression by source inspection: raw DEL is converted to `\x08`, and pi-tui treats `\x08` as Ctrl+Backspace under Windows Terminal. No live WSL2 post-response reproduction was supplied for the linked issue.

## Solution Assessment

No. The best fix should preserve the dependency key contract and target the proven post-response input lifecycle path, not canonicalize all recognized Backspace inputs to ASCII BS.
