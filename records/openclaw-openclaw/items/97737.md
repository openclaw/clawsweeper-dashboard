## Label Justifications

- `P1`: The linked bug affects a fundamental interactive TUI editing key during normal agent use on WSL2/Ubuntu.
- `merge-risk: 🚨 compatibility`: The PR changes the normalized key byte in a way that conflicts with pi-tui's Windows Terminal raw-BS compatibility heuristic.

## Summary

Keep open: the PR targets the right TUI input path, but the byte normalization is unsafe because pi-tui treats raw BS differently under Windows Terminal/WT_SESSION, and the PR only provides unit/fake-PTY proof rather than real WSL2 terminal evidence.

## What This Changes

The branch changes the TUI backspace deduper to return ASCII BS for any recognized backspace and updates focused deduper tests to expect that normalization.

## Best Possible Solution

Keep the TUI input fix narrow, but preserve or emit a byte pi-tui handles as plain Backspace across WT_SESSION and add both a focused regression test and real WSL2 post-response proof before merge.

## Reproduction Assessment

Yes at source level for the PR defect: with WT_SESSION set, pi-tui treats raw BS as Ctrl+Backspace while raw DEL remains plain Backspace, and this PR converts DEL into BS. No live WSL2 terminal repro was run in this read-only review.

## Solution Assessment

No. The TUI input listener is the right layer, but ASCII BS is not pi-tui's portable plain-Backspace representation; the safer fix is to preserve or normalize to a representation the dependency still matches as Backspace under WT_SESSION.
