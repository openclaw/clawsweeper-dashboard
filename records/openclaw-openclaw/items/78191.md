## Summary

## What This Changes

The issue asks migration apply to avoid writing unresolved plugin entries that can make Gateway startup reject the config.

## Best Possible Solution

Keep the shipped warning/install-hint behavior and MCP import mapping; any future automatic plugin installation during migration should be handled as a separate product/security decision.

## Reproduction Assessment

No. The current-main source and shipped regression tests show the reported crash path no longer reproduces: missing `plugins.entries` are warnings, and Claude/Hermes MCP imports target `mcp.servers`.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix: preserve user config, avoid Gateway startup failure, and provide install guidance without running package installs from migration or startup.
