## Summary

Current main still performs the synchronous installed-plugin-index read before the plugin loader cache lookup, so this remains a valid narrow performance bug. There is also an open closing PR (#77379), so the issue should stay open until that PR is reviewed, revised, or merged.

## What This Changes

Avoid synchronous installed-plugin-index record loading on warm `resolvePluginTools()` and plugin-loader cache paths.

## Best Possible Solution

Review and land or revise #77379 so cache-hit paths avoid the sync installed-index read while preserving installed-index freshness through existing snapshot or loader invalidation semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows the installed-index sync read happens inside `resolvePluginLoadCacheContext()` before the loader cache lookup, and `resolvePluginTools()` can reach that loader path for runtime plugin tools; I did not run a benchmark in this read-only review.

## Solution Assessment

Unclear for the exact process-lifetime memoization proposed in the issue. The narrow maintainable fix is to skip the hot-path read only with invalidation or snapshot semantics that keep installed-index changes observable.
