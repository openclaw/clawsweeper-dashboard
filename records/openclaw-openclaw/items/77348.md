## Summary

Keep open: current main still performs the synchronous installed-plugin index read while building the plugin load cache context before checking the reusable loader cache, and an open linked fix PR still declares this issue as its closing issue.

## What This Changes

Avoid re-reading installed plugin index records synchronously on every plugin tool resolution/cache-context build when loader cache entries are already warm.

## Best Possible Solution

Rebase, review, and land https://github.com/openclaw/openclaw/pull/77379 or an equivalent invalidation-aware memoization fix, then close this issue after it reaches main.

## Reproduction Assessment

Yes, at source level. Current main still calls the synchronous installed-index loader before the plugin loader cache lookup, and the issue discussion includes real OpenClaw 2026.5.7 instrumentation matching that path.

## Solution Assessment

Yes, conditionally. The linked PR’s memoization plus cache-clear invalidation is the narrowest maintainable fix direction, but it needs rebase and current validation before merge.
