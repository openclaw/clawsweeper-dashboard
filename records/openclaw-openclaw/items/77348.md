## Summary

Current main still performs the synchronous installed-plugin-index read before the plugin loader cache lookup, so this remains a valid narrow performance bug. The issue should stay open while the linked unmerged fix PR is reviewed instead of being closed early.

## What This Changes

Avoid running `loadInstalledPluginIndexInstallRecordsSync()` on every `resolvePluginTools()`/plugin-loader cache hit by moving or reusing install-record state off the hot path.

## Best Possible Solution

Land a narrow loader-cache fix, likely through #77379 after review, that avoids the sync installed-index read on cache hits while preserving installed-index freshness on writes or snapshot replacement.

## Reproduction Assessment

Yes. Source inspection shows `resolvePluginTools()` can reach `loadOpenClawPlugins()`, and current main computes `resolvePluginLoadCacheContext()` with the synchronous installed-index read before checking the reusable loader cache.

## Solution Assessment

Unclear for the exact linked PR until it is reviewed. The narrow maintainable direction is to skip the sync read on proven cache hits while tying any reused install-record state to existing installed-index or metadata invalidation semantics.
