## Summary

Current main still performs the synchronous installed-index read before the plugin loader registry cache lookup, and an open closing PR now targets the fix, so this issue should remain open until that PR or an equivalent change lands.

## What This Changes

Avoid repeated synchronous installed-plugin-index reads on the plugin tool resolution hot path while preserving installed-index freshness.

## Best Possible Solution

Review and land #77379 or an equivalent narrow loader change that avoids the hot-path sync read and invalidates cached install-record state whenever installed-index data can change.

## Reproduction Assessment

Yes. Source inspection shows the relevant `resolvePluginTools()` fallback path can call `loadOpenClawPlugins()`, which builds `resolvePluginLoadCacheContext()` and reads install records synchronously before checking the reusable plugin registry cache; no benchmark was run in this read-only review.

## Solution Assessment

Yes for keeping this open. The narrow maintainable solution is the linked PR's direction or equivalent: reuse install records on the hot path while clearing that state through existing loader and registry invalidation paths.
