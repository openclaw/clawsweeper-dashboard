## Summary

Keep open: current main still performs the synchronous installed-index read before the plugin loader cache lookup, and open unmerged PR #77379 is the linked implementation path, so the issue should remain open until that PR lands or is replaced.

## What This Changes

The issue asks to remove redundant synchronous installed-plugin-index reads from the `resolvePluginTools()` hot path before plugin loader cache hits.

## Best Possible Solution

Land PR #77379 or an equivalent narrow loader fix that skips the sync installed-index read on cache-hit paths while preserving install/update/uninstall invalidation semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows `resolvePluginTools()` reaches runtime plugin loading, `loadOpenClawPlugins()` builds the load context first, and that context calls the synchronous installed-index reader before checking the reusable registry cache.

## Solution Assessment

Yes, but not by closing this issue now. The linked PR direction is narrow and maintainable if validation confirms the memo is cleared on loader-cache and installed-index mutation paths.
