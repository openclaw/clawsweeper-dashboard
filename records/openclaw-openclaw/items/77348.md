## Summary

Keep this open: current main still performs the synchronous installed-index read before the plugin-loader cache lookup, and an open linked fix PR already declares that it closes this issue. The right cleanup is to review and land or reject that PR, not to close the issue first.

## What This Changes

Avoid synchronous installed plugin index reads on every `resolvePluginTools()` hot-path call, including plugin-loader cache hits.

## Best Possible Solution

Review and land the linked loader memoization PR if its invalidation behavior stays correct for install, update, uninstall, and registry-refresh paths.

## Reproduction Assessment

Yes, from source inspection: `resolvePluginTools()` can reach `loadOpenClawPlugins()`, which builds the load-cache context and calls the sync installed-index reader before checking the reusable registry cache. The comments add real 2026.5.7 operational timing evidence, though I did not run a benchmark in this read-only review.

## Solution Assessment

Yes, the narrow maintainable direction is to avoid the hot-path sync read while tying cached install-record state to existing loader/index invalidation. The linked PR appears to be the active implementation candidate, so closing this issue before that PR lands would be premature.
