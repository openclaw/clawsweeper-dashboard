## Summary

Current main still performs the synchronous installed-plugin index read before the plugin loader cache lookup, and an open closing PR already targets this issue, so it should stay open until that PR is reviewed, merged, or replaced.

## What This Changes

Avoid synchronous installed-plugin index reads on warm plugin tool/runtime loader cache paths.

## Best Possible Solution

Review and land or revise PR #77379 so runtime loader cache hits avoid the sync installed-index read while installed-index changes remain observable.

## Reproduction Assessment

Yes for the runtime-load branch. Source inspection shows `resolvePluginTools()` can reach `loadOpenClawPlugins()`, which computes `resolvePluginLoadCacheContext()` and performs the sync installed-index read before checking the reusable loader cache; I did not run a benchmark in this read-only review.

## Solution Assessment

Partly. Removing the hot-path sync read is the right direction, but the safest solution should reuse or invalidate install-record state through the existing loader/current-metadata lifecycle rather than adding an unconditional hidden metadata cache.
