## Summary

Keep open: current main still performs the installed-index sync read before the plugin-loader cache lookup, and open PR #77379 is the implementation candidate rather than a reason to close this issue before merge.

## What This Changes

Avoid calling `loadInstalledPluginIndexInstallRecordsSync` on warm plugin-loader cache paths reached by `resolvePluginTools`.

## Best Possible Solution

Review and land #77379, or an equivalent narrow fix, so installed records are reused on loader-cache hit paths while index writes and loader-cache clears still invalidate the reused state.

## Reproduction Assessment

Yes. Source inspection of current main shows `resolvePluginTools()` can reach `loadOpenClawPlugins()`, which computes `resolvePluginLoadCacheContext()` and reads install records synchronously before the reusable registry cache lookup.

## Solution Assessment

No for closing this issue yet: current main is not fixed. The maintainable path is an invalidated cache/fingerprint or equivalent PR that avoids the sync read on warm loader-cache paths without making installed-index changes invisible.
