Keep open. Current main still performs the synchronous installed-index install-record read before the reusable plugin-loader cache lookup, and the issue is already paired with an open closing PR rather than fixed on main.

## What This Changes

Avoid synchronous installed plugin index install-record reads on `resolvePluginTools()` / `loadOpenClawPlugins()` cache-hit hot paths.

## Best Possible Solution

Review and land a narrow fix like https://github.com/openclaw/openclaw/pull/77379 that avoids the hot-path sync read while invalidating cached install records on plugin inventory/index mutations.

## Solution Assessment

Yes, the linked PR direction is the narrowest maintainable path if it clears the memo from both full loader-cache resets and registry-only source-change invalidation. The issue should stay open until that PR or an equivalent fix is merged.
