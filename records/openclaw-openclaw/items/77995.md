## Summary

Keep open. The report has a detailed release-to-release regression matrix and current main still routes the `status` RPC through the same summary/task-inspection path; I did not find a merged fix or canonical duplicate, and the logs point to broader event-loop starvation rather than a resolved status-only issue.

## What This Changes

Report that after upgrading to 2026.5.4 or 2026.5.3-1, loopback gateway `status` stalls around 50s and Discord heartbeat/fetch timers time out while 2026.5.2 stays responsive.

## Best Possible Solution

## Reproduction Assessment
