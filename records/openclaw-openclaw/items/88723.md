Full review comments:

- **[P2] Require Codex to be disabled before suppressing warnings:** `src/config/validation.ts:1737-1739`
  - body: The new early return is based only on PI/OpenClaw runtime policy. A config can still have `plugins.entries.codex` enabled, or contain Codex plugin config, while pinning OpenAI models to PI; this patch would now hide the missing `@openclaw/codex` install warning even though the user explicitly configured that plugin. Gate this suppression on `plugins.entries.codex.enabled === false` before treating Codex as intentionally unavailable.
  - confidence: 0.89
- **[P2] Honor model-scoped PI runtime policy too:** `src/config/validation.ts:211-215`
  - body: `collectConfiguredAgentHarnessRuntimes()` cannot make `runtimes.includes(OPENCLAW_AGENT_RUNTIME_ID)` true for model-scoped PI/OpenClaw policies because that helper intentionally filters OpenClaw out. Configs such as `agents.defaults.models["openai/gpt-5.5"].agentRuntime.id = "pi"` still fall through to the missing Codex plugin warning, even though current runtime policy resolution supports those model scopes. Use the actual provider/model policy resolver or add a helper that records explicit OpenClaw selections.
  - confidence: 0.86
