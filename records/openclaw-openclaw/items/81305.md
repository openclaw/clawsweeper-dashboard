## Summary

Keep open: the reported regression is valid and the PR is close, but the branch still leaves the native slash fast path flattening skill-command newlines and needs real after-fix behavior proof before merge.

## What This Changes

The PR preserves newline tails for `/skill` and direct skill command normalization/session setup, updates structural prefix handling, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Share one newline-preservation decision across the normalizer, full session setup, and native fast bootstrap while preserving user payload lines verbatim for skill args and keeping unrelated commands single-line.

## Reproduction Assessment

Yes. Source inspection shows current main drops newline tails in `normalizeCommandBody()`, while the skill invocation parser can accept multiline args if the tail reaches it.

## Solution Assessment

No, not yet. The normalizer direction is right, but the PR needs the same newline handling in native fast bootstrap and should avoid structural-prefix stripping inside preserved user payload lines.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Reuse line-break preservation in fast bootstrap:** `src/auto-reply/reply/session.ts:108-114`
  - body: The new preserve decision is only used by full `initSessionState()`. Native slash commands are handled first by `maybeResolveNativeSlashCommandFastReply()`, and `initFastReplySessionState()` still strips structural prefixes without preserving line breaks, so native `/skill` or direct skill commands can still reach dispatch with multiline args flattened to spaces. Please share this helper with the fast bootstrap path and cover that route.
  - confidence: 0.86
- **[P2] Keep preserved payload lines out of prefix stripping:** `src/auto-reply/reply/mentions.ts:194-196`
  - body: In preserve mode this still removes any line-leading `Label:` text across the whole multiline command. That means a payload like `/skill demo\nTitle: fix this` is retained but changed to `fix this` before dispatch, which breaks the goal of preserving full multiline skill input. Limit structural-prefix cleanup to wrapper text or the command head, not the user args tail.
  - confidence: 0.78
