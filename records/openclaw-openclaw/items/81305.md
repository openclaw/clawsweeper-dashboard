## Summary

Keep open: the PR targets a valid regression, but the current diff still misses the real session/fast-path path that flattens multiline command text, and it broadens multiline preservation to other arg-accepting aliases without updating affected parsers.

## What This Changes

The PR reattaches text after the first newline in slash-command normalization and adds regression tests plus a changelog entry for multiline `/skill` and direct skill command args.

## Best Possible Solution

Land a narrow fix that preserves multiline args through the channel turn, session, and fast-path command paths for `/skill` and direct skill commands, while scoping or updating other arg-accepting aliases and adding real Telegram or gateway proof.

## Reproduction Assessment

Yes. Source inspection shows current main drops the newline tail in `normalizeCommandBody()`, and the proposed branch still leaves the shared session and fast-path setup collapsing real newlines before command dispatch.

## Solution Assessment

No. Reattaching the tail in the normalizer is necessary, but this patch bypasses the real session setup in its command-context test and does not handle non-skill arg parsers that become multiline-exposed.
