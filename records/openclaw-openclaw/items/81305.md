## Summary

Keep open: the fix is aimed at a valid regression, but this PR still loses newline structure through the real session command path and broadens multiline preservation to aliases whose handlers do not parse multiline args. External real behavior proof is also still mock/test-only.

## What This Changes

The branch reattaches text after the first newline during slash-command normalization, adds `/skill` and direct-skill regression tests, and updates the changelog.

## Best Possible Solution

Land a narrow fix that preserves multiline payloads through the actual session and fast-path command context for `/skill` and direct skill commands, while either limiting newline preservation to skill dispatch or updating affected builtin command parsers.

## Reproduction Assessment

Yes, source-reproducible: current main truncates at the normalizer and the PR still flattens newlines through `stripStructuralPrefixes()` before dispatch. I did not run tests because this is a read-only review.

## Solution Assessment

No. Reattaching the tail in the normalizer is part of the right direction, but the implementation needs to preserve newlines through session setup and avoid breaking builtin arg aliases such as `/side`.
