## Summary

Keep open: current main still delegates real Pub/Sub push handling to `gog gmail watch serve`, and the available evidence does not prove the Docker plus Tailscale Funnel watcher failure is fixed.

## What This Changes

The issue reports that Gmail Pub/Sub pushes reach Google Pub/Sub but are not processed by OpenClaw’s Gmail webhook path in a Docker plus Tailscale Funnel setup.

## Best Possible Solution

Keep the issue open and narrow follow-up to the Gmail watcher/setup boundary: capture watcher logs, confirm the `gog` failure mode, then patch OpenClaw’s watcher config, diagnostics, or Docker/Tailscale docs as needed.

## Reproduction Assessment

No; there is not a high-confidence live reproduction against current main. The source path is clear, but reproducing the reported 500 requires live Gmail, Google Pub/Sub, Tailscale Funnel, and the watcher log line that explains `handlePush` failure.

## Solution Assessment

Unclear; broad Gateway hook changes are probably not the right fix because `/hooks/gmail` already works and is tested. The narrow solution should target `gog gmail watch serve` integration or deployment diagnostics after the watcher-side error is known.
