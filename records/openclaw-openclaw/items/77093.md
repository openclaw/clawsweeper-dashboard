## Summary

Keep open: the supported Gmail Pub/Sub path still routes through `gog gmail watch serve`, the reporter's added evidence narrows the failure to that watcher boundary, and current main does not show an implemented fix.

## What This Changes

Real Gmail Pub/Sub pushes delivered through Docker plus Tailscale Funnel should be accepted by the Gmail watcher and forwarded into `/hooks/gmail`.

## Best Possible Solution

Make the supported Gmail Pub/Sub watcher path process real pushes successfully, or fail with a targeted diagnostic that identifies OpenClaw config, Docker/Tailscale exposure, or upstream `gog` watch state.

## Reproduction Assessment

No; there is not a high-confidence current-main live reproduction. The reporter's 2026.4.15 evidence and source/dependency inspection make the watcher boundary credible, but a live Gmail, Pub/Sub, Docker, and Tailscale Funnel run is still needed to capture the actual `gog` failure line.

## Solution Assessment

Unclear; broad Gateway hook changes are not the best first fix because `/hooks/gmail` already works and is tested. The narrow path is to capture the watcher-side error, then patch OpenClaw watcher setup/diagnostics or upstream `gogcli` depending on that failure.
