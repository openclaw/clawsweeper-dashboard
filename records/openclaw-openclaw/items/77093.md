Keep open: current main still delegates the public Gmail Pub/Sub receiver to `gog gmail watch serve`, and the new reporter evidence narrows the failure to that watcher boundary rather than proving a fix on main.

The issue asks OpenClaw to process real Gmail Pub/Sub pushes through the Gmail hook path in a Docker plus Tailscale Funnel deployment.

## Best Possible Solution

Keep the issue open and resolve the watcher boundary: capture the `gog` handlePush error, then patch OpenClaw integration, diagnostics, docs, or upstream `gogcli` according to that failure mode.
