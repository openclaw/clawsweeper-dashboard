## Summary

Keep open: current main still supports Gmail Pub/Sub as a documented feature, but the reported Docker plus Tailscale Funnel failure remains narrowed to the `gog gmail watch serve` boundary and is not proven fixed on main.

## What This Changes

The issue requests that real Gmail Pub/Sub pushes in a Docker plus Tailscale Funnel setup be accepted by the Gmail watcher and forwarded into OpenClaw's `/hooks/gmail` path.

## Best Possible Solution

Keep the issue open until a maintainer captures the watcher-side `gog` failure, then patch the narrow watcher integration, diagnostics, docs, or upstream dependency with focused regression coverage.

## Reproduction Assessment

No; there is no high-confidence live reproduction against current main because it requires Gmail, Google Pub/Sub, `gog`, and Tailscale Funnel in this Docker setup. The source path and reporter's 500-vs-200 split strongly narrow the failing boundary to `gog gmail watch serve`.

## Solution Assessment

Unclear; the report's watcher-boundary diagnosis is the right direction, but the best fix cannot be chosen until the `gog` `handle push failed` error is known. A broad Gateway hook change would be premature because `/hooks/gmail` already accepts and dispatches Gmail-shaped payloads.
