## Summary

Keep open: current main still uses `gog gmail watch serve` as the real Pub/Sub push receiver, and the reporter's follow-up narrows the failure to that watcher boundary while the internal `/hooks/gmail` Gateway path works.

## What This Changes

The issue reports that Gmail Pub/Sub pushes reach Google Pub/Sub but are not processed by OpenClaw's Gmail webhook watcher in a Docker plus Tailscale Funnel deployment.

## Best Possible Solution

The end state should be a verified Gmail watcher path that accepts real Pub/Sub pushes through Docker/Tailscale, with the fix scoped to OpenClaw watcher args/config/diagnostics or an upstream `gogcli` issue based on the captured watcher error.

## Reproduction Assessment

No; there is not a high-confidence current-main live reproduction here. The source path and reporter evidence are strong, but reproducing the failure requires Gmail, Google Pub/Sub, Tailscale Funnel, Docker, and the missing watcher-side `gog` error line.

## Solution Assessment

Unclear; changing the Gateway hook handler is not the narrowest fix because the internal `/hooks/gmail` path is already covered and works. The next repair should target the `gog gmail watch serve` integration or deployment diagnostics once the watcher error is known.
