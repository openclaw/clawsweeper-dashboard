## Summary

Keep open: current main still depends on the `gog gmail watch serve` boundary for public Gmail Pub/Sub pushes, and the reporter's added evidence isolates the failure before OpenClaw's internal `/hooks/gmail` handler rather than showing that the issue is fixed or superseded.

## What This Changes

The issue asks OpenClaw to process real Gmail Pub/Sub push deliveries in a Docker plus Tailscale Funnel setup where Pub/Sub delivery works but the Gmail watcher endpoint returns 500 and no hook is processed.

## Best Possible Solution

Keep this open and investigate the Gmail watcher boundary: capture the gog error in the reported Docker/Tailscale setup, then patch OpenClaw setup/config/diagnostics or coordinate an upstream gogcli fix if needed.
