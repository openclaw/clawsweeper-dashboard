- `impact:message-loss`: The requested feature is about scheduled fixed channel messages being delivered directly instead of relying on agent-mediated delivery.

## Summary

Keep open: current main and the latest release still only support `systemEvent` and `agentTurn` cron payloads, so token-free fixed text delivery to a channel remains unimplemented. The related direct-exec/direct-invoke issues are useful context, but they are broader open design tracks rather than shipped replacements for this narrower static-message request.

## What This Changes

Add a token-free cron payload that sends fixed text directly to a channel target without invoking an agent or model turn.

## Best Possible Solution

Define an additive no-token fixed-text cron delivery contract, or deliberately fold fixed text into the broader direct cron design, then wire protocol, validation, runtime dispatch, CLI, Control UI, agent-tool guidance, docs, and focused tests together.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and latest-release inspection give high confidence that current OpenClaw cannot express or dispatch `payload.kind="staticMessage"`.

## Solution Assessment

Unclear until maintainers choose the public cron API. A dedicated static payload is plausible, but it should be reconciled with broader direct cron/direct-invoke tracks and existing outbound delivery semantics before implementation.
