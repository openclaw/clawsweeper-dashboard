## Summary

Keep open: current main and the latest release still lack a token-free fixed-text cron payload, and the related no-LLM cron/direct-invoke work is broader open product work rather than a shipped static message contract.

## What This Changes

The issue requests a new `staticMessage` cron payload that sends fixed text directly to a channel without an agent/model turn.

## Best Possible Solution

Define an additive no-token fixed-text cron delivery contract, or explicitly fold fixed-text delivery into the broader no-LLM cron design, then update protocol, runtime, CLI, Control UI, agent-tool guidance, docs, and focused tests together.

## Reproduction Assessment

Not applicable for a feature request. Source inspection gives high confidence that current main and `v2026.5.19` cannot express or dispatch `payload.kind="staticMessage"`.

## Solution Assessment

Unclear until maintainers choose the public cron API. A dedicated static payload is plausible for fixed messages, but it should be reconciled with https://github.com/openclaw/openclaw/issues/18160, https://github.com/openclaw/openclaw/issues/37131, and existing outbound delivery semantics before implementation.
