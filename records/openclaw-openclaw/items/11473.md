## Summary

Current main solves the central request through command cron jobs: a fixed command can emit literal stdout, the Gateway delivers that text through normal announce routing, and no model-backed agent turn runs.

## What This Changes

Request a cron payload that sends fixed channel text without invoking an AI model.

## Best Possible Solution

Keep command cron jobs as the canonical deterministic no-model delivery path instead of adding a parallel static-message payload; schedule a command that prints the fixed text and announce its stdout to the target channel.

## Reproduction Assessment

Not applicable as a feature request; current-main source, tests, CLI, and docs provide a high-confidence equivalent workflow using command stdout plus announce delivery.

## Solution Assessment

Yes. Reusing the command payload and existing delivery machinery is narrower and more maintainable than adding a competing payload type solely for literal text.
