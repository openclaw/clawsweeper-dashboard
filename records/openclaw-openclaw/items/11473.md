## Summary

Keep open: current main and v2026.5.19 still only allow `systemEvent` and `agentTurn` cron payloads, and the broader direct-invoke/direct-exec work is open or partial rather than a shipped fixed-text cron delivery contract.

## What This Changes

The issue asks for a token-free `staticMessage` cron payload that sends fixed text directly to a channel without an agent/model turn.

## Best Possible Solution

Define an additive no-token fixed-text cron delivery contract, or fold fixed-text delivery into the broader no-LLM cron design, then update protocol, runtime, CLI, Control UI, agent-tool guidance, docs, and focused tests together.

## Reproduction Assessment

Not applicable for a feature request. Source inspection gives high confidence that current main and v2026.5.19 cannot express or dispatch `payload.kind="staticMessage"`.

## Solution Assessment

Unclear until maintainers choose the public cron API. A dedicated static payload is plausible for fixed messages, but it should be reconciled with broader direct cron/direct-invoke work and existing outbound delivery semantics before implementation.
