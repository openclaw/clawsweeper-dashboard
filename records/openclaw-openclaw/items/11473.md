## Summary

Current main and the latest stable release solve the central request through command cron jobs: they execute without starting an agent/model, turn stdout into the run summary, and use normal cron delivery to announce that output to a channel. A separate `staticMessage` payload would now duplicate the broader shipped command contract.

## What This Changes

Add a cron payload that sends fixed text directly to a channel without an AI call.

## Best Possible Solution

Keep command cron jobs as the single canonical no-model execution and delivery path; use a simple output command for fixed messages rather than adding a parallel payload kind.
