## Summary

Keep open: current main still has only `systemEvent` and `agentTurn` cron payloads, and visible isolated delivery still goes through the agent runner. The related open trackers at https://github.com/openclaw/openclaw/issues/18160 and https://github.com/openclaw/openclaw/issues/37131 are broader design work, not a shipped or exact replacement for fixed text-to-channel cron delivery.

## What This Changes

Request adding a `staticMessage` cron payload that sends fixed text to a channel target without an AI/model turn.

## Best Possible Solution

Define an additive token-free cron message-delivery contract and wire it consistently through cron types, Gateway schema, validation, scheduler dispatch, delivery tracing, CLI/UI, agent tool guidance, and docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Static source checks on current main show no `staticMessage` payload and no token-free direct channel-delivery branch.

## Solution Assessment

Unclear until maintainers choose the public cron API. A `staticMessage` payload is a plausible smaller feature, but it should align with the related direct-exec, direct-tool, and output-routing designs rather than adding a conflicting cron-only contract.
