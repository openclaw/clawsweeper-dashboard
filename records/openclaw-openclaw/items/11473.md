## Summary

The central request is now implemented and shipped: command cron jobs run without an agent/model call, and their deterministic stdout can be delivered through the existing announce channel routing. The implementation uses the general `command` payload rather than adding the proposed `staticMessage` payload, avoiding another overlapping cron execution branch.

## What This Changes

Add a cron payload that sends fixed text directly to a channel without invoking a model.

## Best Possible Solution

Keep command cron as the single supported deterministic no-model execution path; improve documentation with a fixed-text exact-argv example if users still have trouble discovering this use case.

## Reproduction Assessment

Not applicable as a bug reproduction: this was a feature request. Source and shipped documentation now provide a high-confidence verification path for token-free deterministic output delivery.

## Solution Assessment

Yes. The general command payload is the narrower maintainable product architecture because it handles deterministic model-free output without adding a second overlapping direct-output payload branch.
