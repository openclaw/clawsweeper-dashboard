## Summary

Keep open. Current main still lacks a token-free static cron message payload, and the related direct-exec cron tracker is broader than fixed text-to-channel delivery.

## What This Changes

The issue requests a new `staticMessage` cron payload that sends fixed text directly to a channel without model execution or token cost.

## Best Possible Solution

Define an additive no-token fixed-text cron delivery contract, or explicitly fold fixed text into the broader no-LLM cron design, then update protocol, runtime, CLI, Control UI, agent-tool guidance, docs, and focused tests together.

## Reproduction Assessment

Not applicable for a feature request. Source inspection gives high confidence that current main cannot express the proposed payload because cron type/schema/runtime/CLI/UI surfaces accept only `systemEvent` or `agentTurn`.

## Solution Assessment

Unclear until maintainers choose the public cron API. A direct static payload is plausible for fixed messages, but it should be reconciled with https://github.com/openclaw/openclaw/issues/18160 and existing outbound delivery semantics before implementation.
