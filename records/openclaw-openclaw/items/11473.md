## Summary

Keep open. Current main still lacks a token-free static cron message payload: the cron contract, Gateway schema, runtime validation, CLI, Control UI, agent tool schema, and docs still model `systemEvent` and `agentTurn`, while execution routes visible fixed text through heartbeat or an isolated agent run. Related #18160 is broader direct-exec work and remains open, so it is not a shipped replacement for fixed text-to-channel delivery.

## What This Changes

Request adds a `staticMessage` cron payload that sends fixed text directly to a channel target without invoking an agent/model.

## Best Possible Solution

Define an additive non-agent cron delivery contract for fixed text, then wire it through cron types, Gateway validation, job validation, runtime dispatch/run logs, CLI, Control UI, agent cron tool guidance, and public docs while reusing the existing cron outbound delivery resolver.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main cannot express the proposed payload because the cron contract and user-facing surfaces accept only `systemEvent` or `agentTurn`.

## Solution Assessment

Unclear until maintainers choose the public cron API. A direct static payload is plausible and existing outbound delivery plumbing can be reused, but the design should be reconciled with the broader open #18160 direct-exec proposal before implementation.
