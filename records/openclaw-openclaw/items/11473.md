## Summary

Keep open. Current main still cannot express or execute a token-free fixed-text cron payload; prior discussion correctly treats this as unresolved product/API work rather than an implemented or duplicate cleanup item.

## What This Changes

Request adding a `staticMessage` cron payload that sends fixed text to a channel target without invoking an AI agent.

## Best Possible Solution

Define an additive non-agent fixed-text cron delivery contract, or explicitly fold that use case into the canonical no-LLM cron design, then update protocol, runtime, CLI, Control UI, agent-tool guidance, docs, and focused tests together.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main cannot express the proposed payload because the cron contract and user-facing surfaces accept only `systemEvent` or `agentTurn`.

## Solution Assessment

Unclear until maintainers choose the public cron API. A direct static payload is plausible and narrower than shell exec for fixed text, but it should be reconciled with https://github.com/openclaw/openclaw/issues/18160 and existing outbound delivery semantics before implementation.
