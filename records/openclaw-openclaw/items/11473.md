## Summary

Keep open. Current main still has no token-free static cron message payload; cron contracts, Gateway validation, runtime dispatch, CLI, UI, and docs still model only `systemEvent` and `agentTurn`.

## What This Changes

The issue requests a new static cron payload that sends fixed text directly to a channel without starting an AI turn.

## Best Possible Solution

Define an additive non-agent cron delivery contract for fixed text-to-channel delivery, reuse the existing outbound delivery plumbing, and wire it through cron types, Gateway validation, runtime, CLI, Control UI, agent-tool guidance, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main cannot express the proposed payload because cron contracts and user-facing surfaces accept only `systemEvent` or `agentTurn`.

## Solution Assessment

Unclear until maintainers choose the public cron API. A direct static payload is plausible, but it should be reconciled with https://github.com/openclaw/openclaw/issues/18160 and the existing outbound delivery helpers before implementation.
