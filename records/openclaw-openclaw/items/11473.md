## Summary

Keep open. Current main still has no token-free static cron message payload: the cron contract, Gateway schema, runtime dispatch, CLI, Control UI, agent cron tool, and docs still expose scheduled visible output through `systemEvent` or `agentTurn`, and related #18160 is broader open direct-exec work rather than a shipped fixed-text channel delivery path.

## What This Changes

Add a `staticMessage` cron payload that sends fixed text directly to a channel target without running an AI turn.

## Best Possible Solution

Define an additive non-agent cron delivery contract for fixed text-to-channel messages, or explicitly fold fixed text into the broader non-agent cron design, then wire the chosen contract consistently through types, schema, runtime dispatch, CLI, Control UI, agent tool guidance, tests, and public docs.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The missing behavior is high-confidence from source and docs inspection: current main exposes only `systemEvent` and `agentTurn`, and direct fixed text cannot be scheduled for channel delivery without an agent-mediated path.

## Solution Assessment

Unclear. `staticMessage` is a plausible narrow solution for fixed reminders, but maintainers should decide the public payload shape and its relationship to #18160 before implementation.
