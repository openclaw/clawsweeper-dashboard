## Summary

Keep open. Current main still only supports `systemEvent` and `agentTurn` cron payloads, so token-free fixed text delivery would be new public cron/API work rather than conservative cleanup.

## What This Changes

The issue requests a new `staticMessage` cron payload that sends fixed text directly to a channel without invoking an AI agent.

## Best Possible Solution

Define an additive non-agent cron delivery contract for fixed text-to-channel delivery, reuse existing outbound delivery plumbing, and wire it through cron types, Gateway validation, runtime, CLI, Control UI, agent-tool guidance, and docs.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main cannot express the proposed payload because cron contracts and user-facing surfaces accept only `systemEvent` or `agentTurn`.

## Solution Assessment

Unclear until maintainers choose the public cron API. A direct static payload is plausible, but it should be reconciled with #18160 and the existing outbound delivery helpers before implementation.
