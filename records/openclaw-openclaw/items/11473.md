## Summary

Keep open: current main still cannot define or dispatch a token-free static cron message, and the related no-LLM cron work is open or adjacent rather than a shipped fixed-text delivery contract.

## What This Changes

The issue requests a new `staticMessage` cron payload that sends fixed text directly to a channel without running an agent/model turn.

## Best Possible Solution

Define an additive no-token fixed-text cron delivery contract, or fold fixed-text delivery into the broader no-LLM cron design, then update protocol, runtime, CLI, Control UI, agent-tool guidance, docs, and focused tests together.

## Reproduction Assessment

Not applicable for a feature request. Source inspection gives high confidence that current main cannot express or dispatch `payload.kind="staticMessage"`.

## Solution Assessment

Unclear until maintainers choose the public cron API. A dedicated static payload is plausible for fixed messages, but it should be reconciled with https://github.com/openclaw/openclaw/issues/18160 and existing outbound delivery semantics before implementation.
