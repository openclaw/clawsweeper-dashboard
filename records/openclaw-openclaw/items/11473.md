## Summary

Keep open. Current main still cannot express or run a token-free fixed-text cron payload; the cron contract, protocol schema, runtime, CLI, Control UI, agent tool guidance, and docs still model only `systemEvent` and `agentTurn`.

## What This Changes

The issue requests a `staticMessage` cron payload that sends fixed text directly to a channel without invoking an AI turn.

## Best Possible Solution

Define an additive non-agent cron delivery contract for fixed text-to-channel messages, or explicitly fold that use case into the canonical no-LLM cron design, then update protocol, runtime, CLI, Control UI, agent-tool guidance, docs, and focused tests together.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main cannot express the proposed payload because cron contracts and user-facing surfaces accept only `systemEvent` or `agentTurn`.

## Solution Assessment

Unclear until maintainers choose the public cron API. A direct static payload is plausible and lower risk than shell exec, but it should be reconciled with https://github.com/openclaw/openclaw/issues/18160 and existing outbound delivery semantics before implementation.
