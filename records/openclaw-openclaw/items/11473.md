## Summary

Keep open. Current main still does not provide a token-free static cron message payload; cron creation, validation, runtime dispatch, CLI, UI, agent tooling, and docs still route cron payloads through `systemEvent` or `agentTurn`, and related #18160 remains broader open design work rather than a shipped replacement.

## What This Changes

Request adds a `staticMessage` cron payload that sends fixed text directly to a channel without invoking an agent or model.

## Best Possible Solution

Define an additive non-agent cron delivery contract for fixed text, then wire it through cron types, Gateway validation, job validation, runtime dispatch/run logs, CLI add/edit, Control UI, agent cron-tool guidance, and public docs while reusing existing outbound delivery plumbing.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main cannot express the proposed payload because the cron contract and user-facing surfaces accept only `systemEvent` or `agentTurn`.

## Solution Assessment

Unclear until maintainers choose the public cron API. A direct static payload is plausible, but it should be reconciled with #18160 and the existing outbound delivery helpers before implementation.
