## Summary

Keep open. Current main still cannot express or execute a token-free fixed-text cron payload, and the related direct-exec cron issue is broader product work rather than a shipped or exact replacement for fixed text-to-channel delivery.

## What This Changes

The issue requests a new `staticMessage` cron payload that sends fixed text directly to a channel without invoking an AI turn.

## Best Possible Solution

Define an additive non-agent fixed-text cron delivery contract, or explicitly fold that use case into the canonical no-LLM cron design, then update protocol, runtime, CLI, Control UI, agent-tool guidance, docs, and focused tests together.

## Reproduction Assessment

Not applicable for a feature request. Source inspection gives high confidence current main cannot express the proposed payload because the cron contract and user-facing surfaces accept only `systemEvent` or `agentTurn`.

## Solution Assessment

Unclear until maintainers choose the public cron API. A direct static payload is plausible and narrower than shell exec for fixed text, but it should be reconciled with https://github.com/openclaw/openclaw/issues/18160 and existing outbound delivery semantics before implementation.
