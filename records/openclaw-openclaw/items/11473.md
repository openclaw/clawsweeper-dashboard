## Summary

Keep open. Current main still only supports agent-mediated cron payloads (`systemEvent` and `agentTurn`) across the cron contract, Gateway schema, runtime, CLI, Control UI, agent tool guidance, and docs; the related direct-exec tracker at https://github.com/openclaw/openclaw/issues/18160 is broader open design work, and https://github.com/openclaw/openclaw/pull/51276 closed unmerged.

## What This Changes

Add a static fixed-text cron payload that sends directly to a channel target without invoking an agent or model.

## Best Possible Solution

Define an additive non-agent fixed-text cron delivery contract, or explicitly fold fixed text into the canonical no-LLM cron design, then update protocol, runtime, CLI, Control UI, agent-tool guidance, docs, and focused tests together.

## Reproduction Assessment

Not applicable for a feature request. Source inspection gives high confidence current main cannot express the proposed payload because the cron contract and user-facing surfaces accept only `systemEvent` or `agentTurn`.

## Solution Assessment

Unclear until maintainers choose the public cron API. A direct static payload is plausible for fixed text, but it should be reconciled with https://github.com/openclaw/openclaw/issues/18160 and existing outbound delivery semantics before implementation.
