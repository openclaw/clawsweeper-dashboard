## Summary

Keep open. Current main still has no token-free static cron message payload: cron payload types, Gateway validation, runtime dispatch, CLI, Control UI, agent tool schema, and docs still model visible scheduled output through `systemEvent` or `agentTurn`, while related #18160 remains broader open design work rather than a shipped fixed-text channel delivery path.

## What This Changes

The issue requests a `staticMessage` cron payload that sends fixed text directly to a channel without AI processing or token cost.

## Best Possible Solution

Define an additive non-agent cron delivery contract for fixed text-to-channel messages, or explicitly fold fixed text into the broader non-agent cron design, then wire the chosen contract through types, schema, runtime dispatch, CLI, Control UI, tests, and public docs using existing outbound delivery plumbing where appropriate.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The missing behavior is high-confidence from source and docs inspection: current main exposes only `systemEvent` and `agentTurn`, and direct fixed text cannot be scheduled for channel delivery without an agent-mediated path.

## Solution Assessment

Unclear. `staticMessage` is a plausible narrow solution for fixed reminders, but maintainers should decide the public payload shape and its relationship to #18160 before implementation.
