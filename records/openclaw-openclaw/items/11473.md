## Summary

Keep open: current main and v2026.5.22 still only support agent-mediated cron payloads, and the broader direct-exec/direct-invoke tracks remain open rather than shipped fixed-text channel delivery.

## What This Changes

Add a static cron payload that sends configured text directly to a channel target without model execution or token cost.

## Best Possible Solution

Define an additive no-token fixed-text cron delivery contract, or explicitly fold fixed text into the broader direct cron design, then wire protocol, validation, runtime dispatch, CLI, Control UI, agent-tool guidance, docs, and focused tests together.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection against origin/main and v2026.5.22 gives high confidence that OpenClaw cannot express or dispatch payload.kind="staticMessage" today.

## Solution Assessment

Unclear until maintainers choose the public cron API. A dedicated static payload is plausible, but it should be reconciled with broader direct cron/direct-invoke tracks and existing outbound delivery semantics before implementation.
