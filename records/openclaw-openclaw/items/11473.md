## Summary

Current main and v2026.6.5 solve the central user problem through command cron jobs: deterministic output runs without a model-backed agent turn and can be announced directly to a channel, so a second `staticMessage` payload would duplicate the supported no-token path.

## What This Changes

Request a cron payload that sends fixed text directly to a channel without invoking an AI model.

## Best Possible Solution

Keep command cron as the single deterministic no-model path and use a fixed-output command with normal announce delivery for scheduled literal messages.

## Reproduction Assessment

Yes at the source-contract level: the prior model-only payload limitation and the shipped command payload's no-model delivery path are directly verifiable in types, runtime integration, CLI, tests, and docs.

## Solution Assessment

Yes; the shipped command payload is the narrower maintainable solution because it reuses the existing cron delivery pipeline and avoids a parallel static-output payload contract.
