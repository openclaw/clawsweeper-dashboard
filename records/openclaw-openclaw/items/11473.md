## Summary

Keep open: current main and v2026.5.22 still lack a token-free fixed-text cron payload, and the related direct-exec/direct-invoke issues are broader open design tracks rather than a shipped replacement for static channel delivery.

## What This Changes

The issue requests a new cron payload that sends fixed text directly to a channel without invoking an AI turn.

## Best Possible Solution

Define an additive no-token fixed-text cron delivery contract, or explicitly fold fixed text into the broader direct cron design, then wire protocol, validation, runtime dispatch, CLI, Control UI, agent-tool guidance, docs, and focused tests together.

## Reproduction Assessment

Yes, at source level rather than as a bug repro: current main cannot express or dispatch payload.kind="staticMessage" across the cron type, Gateway schema, runtime validation, CLI, UI, and docs surfaces. The same limitation is visible in v2026.5.22.

## Solution Assessment

Unclear until maintainers choose the public cron API. A dedicated static payload is plausible, but it should be reconciled with broader direct cron/direct-invoke tracks and existing outbound delivery semantics before implementation.
