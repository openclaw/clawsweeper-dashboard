## Summary

Keep open. Current main still has no token-free fixed cron message payload: the cron contract, Gateway validation, runtime dispatch, CLI, Control UI, agent tool schema, docs, and focused tests remain limited to `systemEvent` and `agentTurn`; the related direct-exec issue and closed PR do not ship or settle this narrower fixed-text delivery contract.

## What This Changes

Request a `staticMessage` cron payload that sends fixed text to a channel without running an AI turn.

## Best Possible Solution

Define an additive no-LLM fixed-text cron delivery contract, or explicitly fold fixed text into the broader no-LLM cron design, then update protocol, runtime, CLI, Control UI, agent-tool guidance, docs, and focused tests together.

## Reproduction Assessment

Not applicable for a feature request. Source inspection on current main gives high confidence that OpenClaw cannot express the proposed payload because the cron contract and user-facing surfaces accept only `systemEvent` or `agentTurn`.

## Solution Assessment

Unclear until maintainers choose the public cron API. A direct static payload is plausible for fixed messages, but it should be reconciled with https://github.com/openclaw/openclaw/issues/18160 and existing outbound delivery semantics before implementation.
