## Summary

Keep open. Current main and v2026.5.12 still have no token-free static cron message payload: the cron payload contract, Gateway schema, runtime validation/execution, CLI, Control UI, agent cron tool, and docs still model only `systemEvent` and `agentTurn`. The related direct-exec tracker and open `/tools/invoke` PRs are adjacent, not shipped fixed text-to-channel cron delivery.

## What This Changes

The issue requests a `staticMessage` cron payload that sends fixed text directly to a channel without agent processing or model tokens.

## Best Possible Solution

Define an additive no-token fixed-text cron delivery contract, or explicitly fold fixed text into the broader no-LLM cron design, then update protocol, runtime, CLI, Control UI, agent-tool guidance, docs, and focused tests together.

## Reproduction Assessment

Not applicable for a feature request. Source inspection of cron type/schema/runtime/CLI/UI/docs gives high confidence that current main and v2026.5.12 cannot express `payload.kind="staticMessage"`.

## Solution Assessment

Unclear until maintainers choose the public cron API. A dedicated static payload is plausible for fixed messages, but it should be reconciled with https://github.com/openclaw/openclaw/issues/18160 and the existing outbound delivery semantics before implementation.
