## Summary

Keep open. Current main still has no token-free static cron message path: cron payloads, validation, runtime dispatch, CLI, Control UI, agent tool guidance, and docs all model scheduled visible output through `systemEvent` or `agentTurn`, and related #18160 is broader direct-exec work rather than a shipped fixed-text delivery contract.

## What This Changes

Request adds a `staticMessage` cron payload that sends fixed text directly to a channel target without invoking an agent or model.

## Best Possible Solution

Define a maintainer-approved additive non-agent cron delivery contract, then wire it consistently through cron types, Gateway schema, runtime dispatch, CLI, Control UI, agent tool guidance, docs, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection on current main is high-confidence: cron payloads and UI/CLI/docs expose only `systemEvent` and `agentTurn`, and runtime dispatch never sends fixed text directly to a channel.

## Solution Assessment

Unclear. `staticMessage` is a plausible narrow answer for fixed reminders, but maintainers should decide whether it should be distinct or folded into the broader #18160 non-agent cron design before implementation.
