## Summary

Keep open: current main and v2026.5.20 still only model `systemEvent` and `agentTurn` cron payloads, so no-token fixed-text channel delivery remains missing product/API work rather than an obsolete report.

## What This Changes

The issue requests a new `staticMessage` cron payload that sends fixed text directly to a channel target without invoking an AI turn.

## Best Possible Solution

Define an additive no-token fixed-text cron delivery contract, or explicitly fold fixed-text delivery into the broader no-LLM cron design, then wire protocol, runtime dispatch, CLI, Control UI, agent-tool guidance, docs, and focused tests together.

## Reproduction Assessment

Not applicable for a feature request. Source inspection gives high confidence that current main and v2026.5.20 cannot express or dispatch `payload.kind="staticMessage"`.

## Solution Assessment

Unclear until maintainers choose the public cron API. A dedicated static payload is plausible for fixed messages, but it should be reconciled with the open no-LLM cron/direct-invoke designs and existing outbound delivery semantics before implementation.
