## Summary

Keep open. Current main and the latest release still do not provide a token-free static cron message payload; the cron contract, Gateway schema, runtime validation/execution, CLI, Control UI, agent tool guidance, and docs remain limited to `systemEvent` and `agentTurn`, while related direct-invoke/direct-exec work is adjacent rather than a shipped fixed-text cron delivery contract.

## What This Changes

The issue requests a `staticMessage` cron payload that sends fixed text directly to a channel target without starting an AI/agent turn.

## Best Possible Solution

Define an additive no-token fixed-text cron delivery contract, or explicitly fold fixed text into the broader no-LLM cron design, then update protocol, runtime dispatch, delivery semantics, CLI, Control UI, agent-tool guidance, docs, and focused tests together.

## Reproduction Assessment

Not applicable for a feature request. Source inspection gives high confidence that current main and `v2026.5.12` cannot express the proposed static payload because cron type/schema/runtime/CLI/UI/docs surfaces accept only `systemEvent` or `agentTurn`.

## Solution Assessment

Unclear until maintainers choose the public cron API. A direct static payload is plausible for fixed messages, but it should be reconciled with https://github.com/openclaw/openclaw/issues/18160 and the current `/tools/invoke` operator work before implementation.
