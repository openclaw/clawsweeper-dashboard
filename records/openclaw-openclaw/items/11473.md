## Summary

Keep open. Current main and the latest release still lack a token-free static cron message payload: cron types, Gateway schema, runtime validation/execution, CLI, Control UI, agent tool schema, and docs still model only `systemEvent` and `agentTurn`, while the related direct-exec work remains broader and unshipped.

## What This Changes

Add a cron payload kind that sends fixed text directly to a configured channel target without invoking an agent or model.

## Best Possible Solution

Define an additive no-token fixed-text cron delivery contract, or explicitly fold fixed text into the broader no-LLM cron design, then update protocol, runtime, CLI, Control UI, agent-tool guidance, docs, and focused tests together.

## Reproduction Assessment

Not applicable for a feature request. Source inspection gives high confidence that current main and `v2026.5.12` cannot express the proposed static payload because cron type/schema/runtime/CLI/UI/docs surfaces accept only `systemEvent` or `agentTurn`.

## Solution Assessment

Unclear until maintainers choose the public cron API. A direct static payload is plausible for fixed messages, but it should be reconciled with https://github.com/openclaw/openclaw/issues/18160, the operator `/tools/invoke` work, and existing outbound delivery semantics before implementation.
