## Summary

Close as implemented: current main and v2026.6.5 now ship command cron jobs that run without an agent/model call and can announce captured output to channels, covering the requested no-token fixed-message use case even though the payload kind is named `command` rather than `staticMessage`.

## What This Changes

Request a no-AI cron payload that sends fixed text directly to a channel so scheduled reminders do not spend model tokens.

## Best Possible Solution

Keep the shipped command cron path as the token-free core solution; a pure non-shell static-message shortcut should be a separate UX/product follow-up if maintainers still want that narrower affordance.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, merged-PR, and release inspection confirm current main now has a no-agent command cron path with announce delivery.

## Solution Assessment

Yes. The shipped solution is not named `staticMessage`, but command cron jobs provide the requested no-token scheduled output path through the existing cron delivery system without adding a second static-only payload contract.
