## Summary

Keep open: current main and the latest release still only support `systemEvent` and `agentTurn` cron payloads, so the token-free fixed-text channel delivery requested here remains real product work. The related direct-exec/direct-invoke issues are broader open design tracks, not shipped replacements for static channel messages.

## What This Changes

The issue requests a new static cron payload that sends fixed text directly to a channel without running an AI turn.

## Best Possible Solution

Define an additive no-token fixed-text cron delivery contract, or explicitly fold fixed text into the broader direct cron design, then update protocol, validation, runtime dispatch, CLI, Control UI, agent-tool guidance, docs, and focused tests together.

## Reproduction Assessment

Yes. At source level, current main cannot express or dispatch `payload.kind="staticMessage"`: the cron type, Gateway schema, runtime validation, CLI, UI, and docs all expose only `systemEvent` and `agentTurn`.

## Solution Assessment

Unclear until maintainers choose the public cron API. A dedicated static payload is plausible, but it should be reconciled with the broader direct cron/direct-invoke tracks and existing delivery semantics before implementation.
