## Summary

Keep open: current main and the latest release still only support agent-mediated cron payloads, and the related direct-exec/direct-invoke issues are broader open design tracks rather than shipped fixed-text delivery.

## What This Changes

The issue requests a cron payload that sends fixed text directly to a channel target without starting an AI turn.

## Best Possible Solution

Define an additive no-token fixed-text cron delivery contract, or deliberately fold fixed text into the broader direct cron design, then wire protocol, validation, runtime dispatch, CLI, Control UI, agent-tool guidance, docs, and focused tests together.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and latest-release inspection give high confidence that current OpenClaw cannot express or dispatch payload.kind="staticMessage".

## Solution Assessment

Unclear until maintainers choose the public cron API. A dedicated static payload is plausible, but it should be reconciled with broader direct cron/direct-invoke tracks and existing outbound delivery semantics before implementation.
