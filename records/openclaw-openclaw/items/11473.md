## Summary

Keep open. Current main still cannot express a token-free fixed cron message: the cron payload contract, Gateway validator, runtime, CLI, Control UI, agent tool guidance, and docs only support `systemEvent` or `agentTurn`, while the related no-LLM cron work is still unresolved.

## What This Changes

Request adds a `staticMessage` cron payload that sends fixed text directly to a channel without an agent or LLM turn.

## Best Possible Solution

Define an additive no-token fixed-text cron delivery contract, or explicitly fold fixed text into the broader no-LLM cron design, then update protocol, runtime, CLI, Control UI, agent-tool guidance, docs, and focused tests together.

## Reproduction Assessment

Not applicable for a feature request. Source inspection on current main gives high confidence that OpenClaw cannot express the proposed payload because the cron contract and user-facing surfaces accept only `systemEvent` or `agentTurn`.

## Solution Assessment

Unclear until maintainers choose the public cron API. A direct static payload is plausible for fixed messages, but it should be reconciled with https://github.com/openclaw/openclaw/issues/18160 and existing outbound delivery semantics before implementation.
