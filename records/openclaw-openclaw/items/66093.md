## What This Changes

Fix repeated CLI session resets and lost conversation context when `google-gemini-cli` is used as a fallback provider with OAuth auth profiles.

## Best Possible Solution

Keep the current auth-epoch-based CLI reuse behavior, and open a new targeted report if repeated Gemini fallback resets reproduce on a build containing 25f16f8fe6c901f81542b70e45d37c6669588ad7 or later.

## Reproduction Assessment

Yes. The issue provides concrete logs and setup, and the source-level reproduction is the former stored/current `authProfileId` mismatch on a CLI binding; current main now gates that mismatch on a stable versioned auth epoch.

## Solution Assessment

Yes. The current fix solves this at the CLI session identity boundary, preserving cross-provider auth isolation while allowing same-OAuth-identity session reuse instead of forwarding unrelated primary-provider auth profiles.
