## Label Justifications

- `P2`: This is a bounded memory-retrieval reliability bug affecting users of slow QMD query mode.
- `merge-risk: 🚨 compatibility`: The patch changes the runtime effect of an existing timeout setting and can lengthen waits for upgraded configurations that set a value above 15 seconds.

## Summary

Keep this PR open for revision: it addresses the real timeout mismatch, but the proposed outer deadline can still expire before QMD finishes its own timeout handling, and the supplied Vitest output is not real after-fix QMD behavior proof.

## What This Changes

The PR makes the `memory_search` tool deadline use the larger of its existing 15-second floor and the configured QMD search timeout.

## Best Possible Solution

Keep the 15-second floor but derive the outer tool deadline from the configured QMD timeout plus a small named grace interval, with focused default/override tests and redacted live proof of a search exceeding 15 seconds.

## Reproduction Assessment

Yes. Current source starts a 15-second whole-tool timer before QMD setup while QMD may be configured to run longer, providing a high-confidence source reproduction of the premature timeout.

## Solution Assessment

No. Reading the existing setting is the right direction, but making the outer and inner limits identical leaves a deadline race; the outer wrapper should include a small explicit grace interval.
