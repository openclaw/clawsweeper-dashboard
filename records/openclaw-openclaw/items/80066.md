## Summary

Keep open. Current main still has a source-backed path that builds the reported edit/write warning as an `isError` final reply and Feishu delivery has no matching stale-warning guard; the only related fix PR is closed unmerged and targeted `kind: "tool"` while this path reaches Feishu as `kind: "final"`.

## What This Changes

The issue asks OpenClaw to stop repeatedly showing a stale Feishu `Edit: ... AGENTS.md failed` warning when workspace edits actually succeed, or make any real failure actionable.

## Best Possible Solution

Feishu should suppress recovered stale edit/write warning payloads at a boundary that can identify this false-positive pattern while preserving genuine final errors and existing non-Feishu warning semantics.
