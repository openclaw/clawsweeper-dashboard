## Summary

Keep open. Current `origin/main` and the latest shipped release still have a source-backed path that builds recovered edit/write failures as `isError` reply payloads and sends them through Feishu as final replies without a stale-warning guard; the only linked fix PR is closed unmerged and targeted tool-kind delivery, not the final path.

## What This Changes

The issue asks Feishu to stop showing repeated false edit/write failure warnings for workspace `AGENTS.md` edits when the file actually updates.

## Best Possible Solution

Add a narrow suppression for recovered stale edit/write warning payloads at a boundary that can identify the final-reply warning, while preserving genuine final errors and adding regression coverage for Feishu delivery.

## Reproduction Assessment

Yes, at source level: current main can build the mutating edit/write warning as an `isError` reply item and send it through `sendFinalReply`, where Feishu has no matching guard. I did not establish a live Feishu workspace reproduction.

## Solution Assessment

Yes for a focused replacement fix. The closed related PR is not sufficient because it filters tool-kind delivery while the source-proven warning reaches Feishu as a final reply.
