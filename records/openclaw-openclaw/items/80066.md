## Summary

Keep open. Current main still has the source-backed Feishu path that can deliver a stale mutating-tool failure warning as a visible Feishu message after a successful final reply, and the referenced fix commits have not landed on current main.

## What This Changes

The issue reports that Feishu repeatedly shows a failed workspace AGENTS.md edit warning even though the file update succeeds.

## Best Possible Solution

Land a Feishu-owned dispatcher fix that suppresses stale tool-kind error payloads while preserving final actionable error replies, with focused Feishu regression coverage and a user-facing changelog entry.

Yes for a source reproduction path, but not for a live Feishu session. Current main creates `isError` warning payloads for mutating edit failures, shared dispatch preserves error tool payloads, and Feishu currently sends them as channel messages.

## Solution Assessment

Yes, a Feishu-owned filter for tool-kind error payloads that preserves final errors is the narrowest maintainable fix. The referenced hclsys commit demonstrates that shape, but it still needs to land on current main with review and validation.
