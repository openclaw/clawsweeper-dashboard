## Summary

Keep open. Current main still has a source-backed path that can emit the reported Feishu warning, and the linked fix PR is open with review feedback that its tool-kind guard may miss the final-delivery path.

## What This Changes

The issue reports that Feishu repeatedly shows a workspace AGENTS.md edit-failed warning even when the file update succeeds.

## Best Possible Solution

Fix the stale Feishu warning at the embedded-run warning policy or Feishu final-delivery boundary, preserving genuine final failures and adding regression plus redacted Feishu runtime proof.

## Reproduction Assessment

Yes, source-level: current main builds a mutating-tool warning with isError: true, returns it as a reply item, and sends it through final reply delivery to the Feishu dispatcher. I did not live-test a Feishu workspace.

## Solution Assessment

Yes for the issue direction, but the linked PR’s current tool-kind-only guard is not enough if the warning reaches Feishu as a final reply. The narrow maintainable fix should suppress recovered mutating-tool warnings before or at the final-delivery boundary while preserving genuine final errors.
