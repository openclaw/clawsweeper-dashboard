## Summary

Keep open. Current main still has a source-backed Feishu path that can send the reported raw `⚠️ Edit ... failed` tool-error payload after a successful turn, and the timeline references unmerged fix commits rather than a shipped/current-main fix.

## What This Changes

The issue asks OpenClaw to stop showing repeated Feishu warnings for successful workspace file edits, or make genuine failures actionable and non-repetitive.

## Best Possible Solution

Fix the Feishu reply dispatcher so stale tool-kind error payloads from recovered/successful turns are not delivered as chat messages, while final actionable error replies still reach the user.

## Reproduction Assessment

Yes for a source reproduction path, but not from a live Feishu session. Current main still converts failed edit tool state into an `isError` warning payload, and Feishu does not filter tool-kind error payloads before delivery.

## Solution Assessment

Yes, a Feishu-owned dispatcher fix with regression coverage is the narrow maintainable path. A broad shared-dispatch policy change would risk changing Telegram, approvals, media, and genuine failure reporting behavior.
