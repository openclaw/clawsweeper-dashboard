## Summary

Keep open. Current main still builds the reported recovered mutating-tool warning as an `isError` final reply and Feishu delivery has no guard for that path; the only related fix PR is closed unmerged and targeted `kind: "tool"`, which does not match the source-proven final-delivery route.

## What This Changes

The issue asks Feishu to stop repeatedly delivering a misleading `⚠️ Edit ... failed` warning for successful workspace `AGENTS.md` edits, or to make genuine failures actionable.

## Best Possible Solution

Add a narrow regression-tested suppression for recovered stale edit/write warning payloads at the shared warning policy or Feishu final-delivery boundary, while preserving genuine final errors.

## Reproduction Assessment

Yes at source level: current main can build the mutating edit/write warning as an `isError` reply and send it through Feishu final delivery. I did not establish a live Feishu/macOS reproduction.

## Solution Assessment

Yes for a focused replacement fix: target the actual final-warning path, not the closed PR's tool-kind-only guard. The safer implementation should preserve real final failures and add coverage for the final-dispatch route.
