## Summary

Keep open. Current main still has a source-backed Feishu path that can surface the reported raw mutating-tool failure warning even when the turn later succeeds, and I found no current-main fix to close against.

## What This Changes

The issue asks OpenClaw to stop sending repeated Feishu notifications that claim workspace AGENTS.md edits failed when the file update actually succeeds.

## Best Possible Solution

Fix the Feishu reply dispatcher so successful turns do not deliver stale raw mutating-tool failure warnings as user-visible Feishu messages, while preserving genuine actionable errors.

## Reproduction Assessment

Yes for a source reproduction path, but not for a live Feishu session. Current main converts failed edit tool state into a `failed` warning payload, and Feishu currently leaves those error tool payloads visible by default.

## Solution Assessment

Yes, a Feishu-owned dispatcher fix with regression coverage is the narrowest maintainable path. A global tool-error policy change would risk other channels, approvals, media, and genuine failure reporting.
