## Summary

Keep open: current main still has a source-backed path that can turn a failed `edit` tool state into a user-visible `⚠️ ... Edit ... failed` payload, and the Feishu dispatcher does not currently suppress or reinterpret that stale error payload after an otherwise successful turn.

## What This Changes

The issue asks OpenClaw to stop repeatedly showing `⚠️ 📝 Edit: in ~/.openclaw/workspace/AGENTS.md failed` in Feishu when the workspace `AGENTS.md` edit appears to have succeeded.

## Best Possible Solution

Fix the Feishu-visible path so successful Feishu turns do not deliver stale raw edit-failure payloads, while coordinating with the generic edit warning/noop work and preserving genuine actionable failures.

## Reproduction Assessment

Yes for a source reproduction path, but not for a live Feishu session. Current main can produce an `isError` edit-failure warning after assistant success text, and Feishu currently does not filter that error payload.

## Solution Assessment

Yes, a focused Feishu delivery fix or a narrowly shared prepared option is the best path if the warning is stale; if the root is an edit false-positive, reuse the existing edit recovery/noop work instead of globally suppressing tool errors.
