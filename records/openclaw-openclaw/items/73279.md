## Summary

Keep open. The PR targets a real Discord duplicate-reply bug, but the proposed wording makes final-response delivery unconditional even though current main still has message-tool-only delivery for default guild/channel turns, and the PR still lacks after-fix real Discord proof.

## What This Changes

The PR changes Codex runtime and bundled Discord guidance to steer normal current-conversation Discord replies to final response delivery, with a Codex unit assertion for the new wording.

## Best Possible Solution

Make the Codex and Discord guidance delivery-mode-aware: use final response delivery only when source replies are automatic, preserve message(action=send) guidance for message-tool-only turns, and cover both paths with focused tests plus redacted real Discord proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Discord guild turns can be message_tool_only, automatic final delivery is suppressed in that mode, and the PR wording tells agents to rely on final delivery anyway.

## Solution Assessment

No. Prompt and skill guidance are the right fix boundary, but this PR ignores sourceReplyDeliveryMode; the safer solution is conditional guidance plus tests for automatic and message-tool-only turns.
