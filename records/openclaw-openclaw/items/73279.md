## Summary

Keep open. The PR addresses a real duplicate-reply prompt problem for automatic Discord thread delivery, but the proposed wording is too broad for current main: default Discord guild/channel turns are message-tool-only, so unconditional final-response guidance can produce no visible reply. The PR also needs the required user-facing changelog entry before merge.

## What This Changes

The PR changes Codex developer instructions, the bundled Discord skill, and a Codex thread-lifecycle unit test so normal current-conversation replies are steered toward final assistant delivery instead of `message` tool sends.

## Best Possible Solution

Make the Codex and Discord guidance mode-aware: use final-response guidance only when the resolved source reply delivery mode is automatic, keep `message(action=send)` guidance for message-tool-only Discord guild/channel turns, add regression coverage for both modes, and record the fix in the changelog.

## Reproduction Assessment

Yes. The original duplicate path is reproduced by the linked Discord `autoThread: true` steps in #73278, and the PR-introduced regression is source-reproducible by following Discord guild turns through `ChatType: "channel"` to `sourceReplyDeliveryMode: "message_tool_only"` and suppressed automatic final delivery.

## Solution Assessment

No. The requested direction fixes the automatic-delivery half of the bug, but unconditional final-response guidance is not the narrowest maintainable solution because current Discord guild/channel defaults require the message tool for visible output.
