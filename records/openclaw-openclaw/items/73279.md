## Summary

Keep open: the PR targets a real Discord autoThread duplicate-reply bug, but the patch is still unsafe because it makes final-response delivery unconditional even though current default Discord guild/channel replies are message-tool-only and suppress automatic final delivery.

## What This Changes

The branch changes Codex developer instructions, the bundled Discord skill, and a unit assertion to steer current-conversation Discord replies to final assistant output instead of `message.send`.

## Best Possible Solution

Make the Codex and Discord guidance delivery-mode-aware: use final assistant delivery only when automatic source delivery is active, keep message-tool guidance for message-tool-only turns, and cover both paths with focused tests plus redacted Discord proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Discord guild/channel turns can be `message_tool_only`, automatic source delivery is suppressed in that mode, and the PR tells agents to rely on final delivery anyway.

## Solution Assessment

No. Prompt and skill guidance are the right fix boundary, but this patch ignores `sourceReplyDeliveryMode`; the safer solution is conditional wording that preserves default group/channel message-tool-only behavior.
