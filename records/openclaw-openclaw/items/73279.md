## Summary

This PR should stay open because the proposed guidance fixes the duplicate-autoThread symptom too broadly: current main still has delivery modes where Discord channel replies are message-tool-only, so unconditional final-response guidance can make default channel replies non-visible. It also still lacks after-fix real Discord behavior proof.

## What This Changes

The PR changes Codex runtime developer instructions, the bundled Discord skill, and a Codex unit test to steer normal current-conversation Discord replies away from `message.send` and toward final assistant delivery.

## Best Possible Solution

Make the Codex and Discord guidance delivery-mode-aware: discourage same-conversation message sends only when automatic source delivery is active, preserve `message(action=send)` guidance for message-tool-only turns, and cover both paths with focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path for the original duplicate symptom and the PR-introduced regression: Discord channel turns can resolve to either automatic final delivery or `message_tool_only`, and the PR wording ignores that distinction.

## Solution Assessment

No. Prompt and skill guidance are the right fix boundary, but this PR makes final-response delivery unconditional instead of respecting the resolved source reply delivery mode.
