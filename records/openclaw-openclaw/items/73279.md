## Summary

Keep open. The reported duplicate-reply bug is real, but this patch would make reply guidance unconditional even though current main still has message-tool-only visible delivery modes where final responses are intentionally private.

## What This Changes

The PR changes Codex app-server developer instructions, the bundled Discord skill, and a Codex thread-lifecycle test so normal current-conversation replies are steered toward final assistant delivery instead of the message tool.

## Best Possible Solution

Make the Codex runtime and Discord skill guidance delivery-mode-aware: use final-response wording only for automatic source delivery, keep message-tool send guidance for tool-only visible delivery, cover both paths with focused regression tests, and add the required changelog fix entry.

## Reproduction Assessment

Yes. The linked bug report has concrete Discord autoThread steps, and current main provides a source-level reproduction path: channel turns resolve to message_tool_only and automatic final delivery is suppressed.

## Solution Assessment

No. The proposed prompt direction fixes the duplicate-thread case, but the maintainable fix must branch on the resolved delivery mode so default Discord guild/channel and Codex harness tool-only turns still produce visible replies when intended.
