## Summary

Keep open. This PR targets a real Discord auto-thread duplicate-reply bug, but the diff is not mergeable as written because it unconditionally tells Codex and the Discord skill to rely on final-response delivery while current Discord guild/channel turns default to message-tool-only visible replies.

## What This Changes

The PR changes Codex app-server developer instructions, the bundled Discord skill text, and a Codex thread-lifecycle unit test so normal current-conversation replies are steered away from `message` tool sends and toward final assistant delivery.

## Best Possible Solution

Make the Codex runtime and Discord skill guidance delivery-mode-aware: use final-response wording only when automatic source delivery is active, keep `message(action=send)` guidance for `message_tool_only`, cover both paths with regression tests, and add the required changelog fix entry.

## Reproduction Assessment

Yes. The original bug has concrete Discord `autoThread: true` steps in #73278, and the PR-introduced regression is reproducible from current main by tracing Discord guild turns through `sourceReplyDeliveryMode: "message_tool_only"` to suppressed automatic final delivery.

## Solution Assessment

No. The PR addresses a real prompt cause for automatic thread delivery, but unconditional final-response guidance is not the narrowest maintainable fix because current Discord guild/channel defaults require the message tool for visible output.
