## Summary

Keep open. The PR targets a real Discord autoThread duplicate-reply bug, but the proposed unconditional final-response guidance can break default Discord guild/channel replies where current main deliberately uses `message_tool_only` delivery, and the PR still lacks after-fix real behavior proof.

## What This Changes

The PR changes Codex developer instructions, the bundled Discord skill, and a Codex unit test to steer normal current-conversation replies away from the `message` tool and toward final assistant delivery.

## Best Possible Solution

Make the Codex and Discord guidance delivery-mode-aware: discourage duplicate same-conversation message sends only when automatic source delivery is active, preserve `message(action=send)` for `message_tool_only`, and cover both paths with focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path for both the original duplicate autoThread bug and the PR-introduced regression: default Discord channel turns resolve to `message_tool_only`, while automatic final delivery is suppressed in that mode.

## Solution Assessment

No. Prompt and skill guidance are the right fix boundary, but this PR makes final-response delivery unconditional instead of respecting the resolved delivery mode.
