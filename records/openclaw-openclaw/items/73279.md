## Summary

Keep open. The PR targets a real duplicate-reply cause, but the proposed unconditional final-response guidance conflicts with current main's default Discord guild/channel delivery mode, where final replies are private and visible output must use the message tool.

## What This Changes

The PR changes Codex developer instructions, the bundled Discord skill, and a Codex thread-lifecycle test so normal current-conversation replies are steered to final assistant delivery instead of message-tool sends.

## Best Possible Solution

Make the Codex developer instructions and Discord skill delivery-mode aware: final-response guidance only when the source conversation is in automatic delivery mode, and explicit message-tool guidance when final replies are private/tool-only. Keep the linked issue open until the corrected PR merges.

## Reproduction Assessment

Yes. The original duplicate path is described with concrete Discord `autoThread: true` steps in the PR and linked issue, and the PR-introduced regression is reproducible by tracing current main's default Discord guild/channel `message_tool_only` path where final replies are suppressed.

## Solution Assessment

No. The prompt-only direction is reasonable, but the current implementation is too broad because it ignores `sourceReplyDeliveryMode`; the safer fix is conditional wording plus tests for both automatic autoThread delivery and default private guild/channel delivery.
