## Summary

Keep open. The PR targets a real duplicate-reply prompt problem, but its replacement guidance is too broad for current main because Discord guild/channel turns default to private/message-tool-only final replies; merged as-is, it can steer agents away from the only visible reply path for default Discord channels.

## What This Changes

The PR changes Codex developer instructions, the bundled Discord skill, and a Codex thread-lifecycle unit test so normal current-conversation replies are steered to final assistant delivery instead of `message` tool sends.

## Best Possible Solution

Revise the PR so Codex and Discord guidance follows the resolved source reply delivery mode: final-response guidance only for automatic source delivery, message-tool guidance for private/message-tool-only turns, with focused regression coverage for both automatic auto-thread delivery and default Discord guild/channel delivery.

## Reproduction Assessment

Yes. The original duplicate path is documented by #73278 with concrete Discord `autoThread: true` steps, and the PR-introduced regression can be reproduced from current main by tracing Discord guild turns to `sourceReplyDeliveryMode: "message_tool_only"` and suppressed final delivery.

## Solution Assessment

No. The proposed direction fixes a real prompt cause for automatic delivery, but unconditional final-response wording is not the narrowest maintainable solution because current Discord channel defaults require `message(action=send)` for visible output.
