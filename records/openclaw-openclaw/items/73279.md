## Summary

Keep open. The underlying duplicate-reply report is real, but this patch should not merge as-is because it unconditionally tells Codex that final responses will be delivered even when current main marks Discord guild/channel turns as `message_tool_only` and suppresses automatic final delivery; it also lacks the required changelog entry.

## What This Changes

The PR changes Codex developer-instruction text, the bundled Discord skill, and a Codex thread-lifecycle test to steer normal current-conversation replies away from Discord `message` tool sends.

## Best Possible Solution

Revise the Codex and Discord guidance to follow the resolved source reply delivery mode: final-response wording only for automatic delivery, `message(action=send)` wording for message-tool-only turns, explicit messaging actions still allowed, focused tests for both automatic auto-thread delivery and default Discord guild/channel delivery, plus one active changelog fix entry.

## Reproduction Assessment

Yes. The original duplicate path has concrete Discord `autoThread: true` steps in #73278, and the PR-introduced regression is reproducible from current main by tracing Discord guild turns to `sourceReplyDeliveryMode: "message_tool_only"` and suppressed automatic final delivery.

## Solution Assessment

No. The PR addresses a real prompt cause for automatic thread delivery, but unconditional final-response guidance is not the narrowest maintainable fix because current Discord guild/channel defaults require `message(action=send)` for visible output.
