## Summary

Keep open: the Discord `/codex bind` bug is source-reproducible, but this PR is not merge-ready because it adds a new direct deprecated session-store/file-path path in Discord dispatch instead of using the existing transcript resolver, and contributor real Discord proof is still missing.

## What This Changes

The PR updates Discord native plugin command dispatch and tests so active sessions pass `sessionId` and `sessionFile` into plugin command handlers.

## Best Possible Solution

Pass the Discord plugin command session metadata through `resolveSessionTranscriptLegacyFileTarget`, matching Telegram, and require redacted live Discord `/codex bind` proof before merge.

## Reproduction Assessment

Yes from source: current Discord plugin dispatch passes `sessionKey` and auth profile but not `sessionFile`, while the Codex bind handler returns the reported error when `ctx.sessionFile` is missing. I did not establish a live Discord slash-command reproduction.

## Solution Assessment

No: passing a session file is the right repair, but this implementation should use the existing transcript runtime helper rather than adding a new direct sessions.json/file-path path.
