## Summary

Keep this PR open: the Ollama loopback proxy bypass is useful and narrowly tested, but the latest head introduces a production resolution blocker by importing a private local-only SDK subpath from the bundled Ollama provider.

## What This Changes

The branch adds a guarded managed-proxy direct path for exact configured loopback Ollama memory embedding origins, switches the provider SSRF policy to exact-origin trust, updates docs, and adds focused SSRF/Ollama tests plus private SDK subpath metadata.

## Best Possible Solution

Keep the loopback-only transport behavior, but make the internal helper reachable through a production-safe bundled/internal seam with contract coverage before maintainers accept the proxy policy exception.
