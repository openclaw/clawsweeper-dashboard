## Summary

Keep this PR open: the loopback-only Ollama embedding bypass is a plausible fix for the linked proxy failure, but it still needs maintainer review for the proxy/security and public SDK boundary, plus current-head real behavior proof.

The PR adds a guarded managed-proxy bypass for exact loopback Ollama memory embedding fetches, switches Ollama embedding SSRF trust from hostname to exact origin, updates tests, exports new SDK types, and documents proxy behavior.

## Best Possible Solution

Land a loopback-only Ollama embedding fix only after maintainers accept the proxy exception and SDK boundary, then either keep the bypass marker internal or document and test it as public API with narrowed docs and current-head proof.

## Reproduction Assessment

Yes, by source inspection: current main sends strict guarded fetches through EnvHttpProxyAgent when the managed proxy is active, and the Ollama embedding provider passes no bypass option. I did not run a live Ollama/proxy repro in this read-only review.

## Solution Assessment

Unclear until maintainer policy review: the loopback-only bypass is plausibly narrow, but the public SDK marker and broad docs are not yet the best maintainable shape. A safer path is to keep the seam internal or explicitly document and test it as public API.
