## Summary

Keep open: this is a useful targeted fix for Ollama memory embeddings under managed proxy mode, but it is not merge-ready until maintainers accept the proxy/security contract, the SDK surface is handled deliberately, docs match the loopback-only implementation, and current-head proof is refreshed.

## What This Changes

The PR adds a managed-proxy direct-bypass marker for Ollama memory embedding fetches, switches Ollama SSRF trust to exact origin, adds fetch-guard/Ollama coverage, refreshes the plugin SDK baseline, and documents the proxy behavior.

## Best Possible Solution

Land only after maintainers choose the proxy and SDK contract, then either keep the bypass marker internal to bundled Ollama or document and test it as public API, with docs and proof updated for the loopback-only behavior.

## Reproduction Assessment

Yes, by source inspection: current main sends strict Ollama embedding fetches through EnvHttpProxyAgent when the managed proxy is active, and the Ollama provider passes no bypass option. I did not run a live Ollama/proxy repro in this read-only review.

## Solution Assessment

Unclear until maintainers decide the proxy/API boundary. Exact loopback direct routing is a plausible narrow fix, but publishing the bypass marker through plugin SDK and documenting it as local/self-hosted is not yet the safest maintainable shape.
