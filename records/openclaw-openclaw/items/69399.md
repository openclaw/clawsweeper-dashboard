The PR adds an `agents.defaults.notifyOnModelFallback` config flag, uses it in `runReplyAgent` to show fallback and fallback-cleared notices outside verbose mode, updates config schema/help/docs artifacts, and adds fallback-notice e2e coverage.

## Best Possible Solution
