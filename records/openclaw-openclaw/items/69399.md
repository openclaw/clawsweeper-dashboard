## Summary

Keep open. The feature is still absent from current main, and this PR needs fixes for the queued-config test path, config help accuracy, and external real behavior proof before merge.

## What This Changes

The PR adds a default-false `agents.defaults.notifyOnModelFallback` config option, docs/schema/help entries, reply-runner notice logic, and e2e coverage for fallback and fallback-cleared notices outside verbose mode.

## Best Possible Solution

Land one canonical default-false fallback-notice implementation that exercises the production queued-config path, documents both fallback and cleared notices, and includes redacted real behavior proof from a normal-mode fallback reply.

## Reproduction Assessment

Yes, source-reproducible. Current main gates both user-visible fallback notices on verbose mode, and current tests assert verbose-off suppression while lifecycle events still emit.

## Solution Assessment

No, not as currently patched. The opt-in direction is narrow, but the tests need to pass the fallback config through the queued run and the config help/generated schema should describe both fallback and cleared notices.
