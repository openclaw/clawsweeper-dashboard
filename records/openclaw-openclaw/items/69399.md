## Summary

Keep open. Current main still lacks the requested default-false fallback-notice config, and this PR remains useful, but the current patch has a source-reproducible test defect, incomplete generated help text, no real behavior proof, and a public config-surface decision still needs maintainer agreement.

## What This Changes

The PR adds `agents.defaults.notifyOnModelFallback`, widens fallback and fallback-cleared reply notices outside verbose mode when that option is true, and updates docs, config schema/help, generated artifacts, and e2e tests.

## Best Possible Solution

Land one canonical default-false fallback-notice implementation that exercises the production queued-config path, documents both fallback and cleared notices, and includes redacted real behavior proof from normal-mode fallback output.

## Reproduction Assessment

Yes. Source inspection shows current main gates reply-visible fallback and fallback-cleared notices on `verboseEnabled`, lacks the requested config key, and still tests verbose-off suppression while lifecycle events emit.

## Solution Assessment

No, not as currently patched. The opt-in direction is narrow, but the tests need to pass the fallback config through the queued run path and the config help/generated schema should describe both fallback and fallback-cleared notices.
