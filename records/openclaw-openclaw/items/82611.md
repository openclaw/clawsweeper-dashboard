## Summary

Keep open. Current main and the reported v2026.5.12 release source-reproduce the Feishu observer path, but the live-event suppression is an intentional external-channel visibility policy, so this needs a maintainer decision on session-scoped Control UI observation rather than cleanup closure.

## What This Changes

The issue asks for Control UI WebChat to stream live assistant/tool updates while observing a selected Feishu direct-chat session whose run was initiated from Feishu.

## Best Possible Solution

Implement a selected-session-only observer stream for external-channel runs, with tests proving Feishu/external DM assistant/tool progress reaches the selected session and does not leak to unrelated Control UI clients.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: Feishu runs are registered as non-Control-UI-visible, and the gateway suppresses assistant/tool live broadcasts for that flag; I did not run the live Feishu OAuth setup.

## Solution Assessment

Is this the best way to solve the issue? Not as a global visibility flip; the maintainable fix is a session-scoped observer contract that preserves hidden external-run behavior for unrelated clients.
