## Summary

Keep open: current main still buffers Codex app-server assistant deltas and emits assistant text only at terminal completion, while WebChat live updates are driven by assistant agent events. The related fixes addressed final-event/raw-completion hangs, not this live-delta regression, and no duplicate or implemented-on-main close reason applies.

## What This Changes

Investigate a regression where Control UI WebChat stops receiving live streaming updates for openai/gpt-5.5 when authenticated through Codex OAuth after v2026.5.12.

## Best Possible Solution

Restore live WebChat feedback for Codex OAuth runs by projecting safe Codex app-server assistant/progress events through the existing agent-event chat stream without exposing commentary or tool progress as final assistant text.

Partially: the report gives concrete steps, and source inspection shows Codex assistant deltas are buffered until completion while WebChat needs assistant agent events for live deltas. I did not run a live Codex OAuth WebUI session, so this is source-reproducible rather than fully reproduced.
