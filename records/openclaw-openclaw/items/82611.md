## Summary

Keep open. Current main still makes Feishu-originated turns non-Control-UI-visible before the gateway live chat/agent broadcast paths run, so the report is source-reproducible; the remaining fix needs an explicit selected-session observer policy rather than a broad visibility flip.

## What This Changes

The issue asks for Control UI WebChat to live-update while observing the selected session during Feishu-originated runs.

## Best Possible Solution

Define a session-scoped observer stream for external-channel runs that only reaches clients subscribed to the exact selected session, while preserving hidden external-run behavior for unrelated clients.

## Reproduction Assessment

Yes, at source level: Feishu runs enter as channel "feishu", current main marks non-WebChat channel runs isControlUiVisible=false, and the gateway live broadcast paths are gated on that flag. I did not run a live Feishu OAuth flow during this read-only review.

## Solution Assessment

No, not as a global visibility flip. The safer fix is a selected-session observer contract that streams only to exact-session subscribers and keeps external-channel traffic hidden from unrelated Control UI clients.
