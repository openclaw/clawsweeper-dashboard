## Summary

Keep open. Current main still makes Feishu-originated runs non-Control-UI-visible and then gates live chat, agent, and session-tool fanout on that flag; fixing the selected-session observer path needs an explicit privacy/product decision rather than a broad visibility flip.

## What This Changes

The issue requests live Control UI updates while observing a selected session whose run was initiated from Feishu DM.

## Best Possible Solution

Define a session-scoped observer path for external-channel runs that reaches only clients subscribed to the exact selected session, with regression coverage and live Feishu proof, while preserving the hidden-run boundary for unrelated clients.

## Reproduction Assessment

Yes at source level: Feishu dispatches through an external channel session, current main marks that run not Control-UI-visible, and the gateway gates the live chat/agent/session-tool broadcasts on that flag. I did not run the live Feishu OAuth workflow.

## Solution Assessment

No, not as a broad isControlUiVisible flip. The safer fix is a selected-session observer contract that streams only to authorized/subscribed clients for that exact session and keeps hidden external-channel traffic out of global Control UI fanout.
