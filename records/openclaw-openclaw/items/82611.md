## Summary

Keep open. Current main still suppresses live Control UI chat/agent updates for Feishu-originated runs, and fixing it safely needs a maintainer decision on session-scoped external-channel visibility rather than a broad hidden-run visibility flip.

## What This Changes

The issue requests live Control UI updates while observing a Feishu-originated run in the currently selected session.

## Best Possible Solution

Define a selected-session observer contract for external-channel runs that sends assistant/tool progress only to clients attached to that exact session, with regression coverage proving no unrelated live-event leakage.

## Reproduction Assessment

Yes at source level: Feishu dispatch registers an external-channel run, and current gateway live broadcasting checks isControlUiVisible before sending assistant/tool updates. I did not execute the live Feishu OAuth workflow.

## Solution Assessment

No broad visibility flip is not the best fix; the safer solution is a session-scoped observer stream that preserves the existing hidden-run policy for unrelated Control UI clients.
