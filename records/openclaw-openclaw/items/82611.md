## Best Possible Solution

Emit or project safe final-answer Codex assistant deltas into the existing gateway `assistant`/chat-delta contract while preserving the current final-event and tool-progress behavior.

Do we have a high-confidence way to reproduce the issue? ## Solution Assessment

Is this the best way to solve the issue? Yes in direction: the narrow maintainable fix is to reuse the existing `assistant` event and `chat` delta pipeline, adding Codex-side filtering for final-answer deltas rather than adding a new UI-only transport or config flag.
