## Summary

Keep open. The report is fresh and source-reproducible on current main, but the remaining decision is not a narrow bug fix: current code intentionally suppresses live Control UI chat/agent events for non-internal channel runs, so Feishu observer streaming needs a deliberate session-scoped policy rather than a global visibility flip.

## What This Changes

Request Control UI WebChat to live-update the currently selected session while a Feishu DM, rather than WebChat, initiates the agent run.

## Best Possible Solution

Define and implement a session-scoped observer stream for external-channel runs that reaches only clients subscribed to the exact selected session, with regression coverage for Feishu/external DM streaming and no global live-event leakage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: a Feishu-originated turn is an external channel run, and current main marks external runs not Control-UI-visible before the gateway chat/agent live broadcast paths run.

## Solution Assessment

Is this the best way to solve the issue? Not as a broad visibility flip; the safer solution is a selected-session observer contract that preserves the current hidden external-run policy for unrelated Control UI clients.
