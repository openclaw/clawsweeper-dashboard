## Summary

Keep open. Current main still suppresses live Control UI broadcasts for non-internal channel runs, so the Feishu observer gap is source-reproducible and needs a session-scoped observer policy before a fix PR is safe.

## What This Changes

Control UI should live-update the selected session while observing a Feishu-originated direct-message run.

## Best Possible Solution

Define a selected-session observer stream for external-channel runs that reaches only clients observing that exact session while preserving hidden external-run behavior for unrelated Control UI clients.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: Feishu runs are external channel runs, current main marks them not Control-UI-visible, and the gateway live event paths are gated on that flag.

## Solution Assessment

Is this the best way to solve the issue? Not by globally flipping visibility; the maintainable fix is a selected-session observer contract with regression coverage and leakage checks.
