## Best Possible Solution

Keep the probe-after-bootstrap-failure shape, but cover the observed code-5/I/O-error race in the daemon restart path only when `launchctl print` confirms the intended service is loaded, with tests for both recoverable race and genuine failure cases plus macOS proof of the race path.
