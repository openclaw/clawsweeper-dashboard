## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use the shipped overload cooldown knob:** `src/agents/pi-embedded-runner/run.ts:313-315`
  - body: The branch reads `agents.defaults.embeddedPi.overloadBackoffMaxMs`, but current main schema/docs/tests expose `auth.cooldowns.overloadedBackoffMs`. A rebase that keeps this path would add a second obsolete knob and ignore the documented setting users already have.
  - confidence: 0.82
- **[P1] Gate profile sourcing to live setup:** `test/test-env.ts:338`
  - body: The patch calls profile loading from ordinary `installTestEnv()` setup. A developer profile that exports live flags or provider credentials can turn non-live test workers into live/network runs before isolated live setup has run.
  - confidence: 0.9
- **[P2] Refresh the live env snapshot after profile loading:** `test/test-env.ts:336`
  - body: The live env snapshot is captured before profile loading, but staging uses that snapshot to choose `OPENCLAW_CONFIG_PATH` and `OPENCLAW_STATE_DIR`. Live runs relying on `~/.profile` for those paths can stage the wrong config or state.
  - confidence: 0.86
- **[P2] Preserve config include files when staging live HOME:** `test/test-env.ts:315-316`
  - body: The staging code copies only the active config file into the temp home. Config include files resolved relative to the original config root can disappear, so valid live configs with sibling includes fail unless the user opts into the real HOME.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete live-test secret exposure risk by sourcing profile environment into ordinary test setup.

Concerns:

- **[medium] Profile sourcing can leak live credentials into unit tests:** `test/test-env.ts:338`
  - body: Because profile env loading runs outside the live-only path in the PR, non-live test setup can import provider keys, GitHub tokens, and live flags from a developer profile before isolation. That broadens secret exposure and can trigger expensive or flaky live network suites unexpectedly.
