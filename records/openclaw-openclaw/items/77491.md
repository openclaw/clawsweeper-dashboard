## What This Changes

The PR flips `tools.loopDetection.enabled` to default on, updates docs/schema/help/generated baseline/test/changelog surfaces, and adds an explicit opt-out test while preserving `generic_repeat` as warning-only.

## Best Possible Solution

Land the narrow default-on change after maintainer approval, with `unknown_tool_repeat` accurately reflected across docs/help/changelog/generated schema text and the existing warning-only `generic_repeat` behavior preserved.

## Reproduction Assessment

Yes. Source inspection on current main shows undefined config resolves to `enabled: false`, detection short-circuits when disabled, and the current unit test asserts repeated default calls are not flagged.

## Solution Assessment

Partly. The runtime change is the narrowest shape if maintainers accept default-on behavior, but the proposed user-visible text should include `unknown_tool_repeat` or avoid saying only stable no-progress detectors can hard-block.
