## Summary

Keep open for maintainer review and normal merge handling. The patch appears correct and still necessary; current main and v2026.6.10 still invoke/quarantine selected discovery-time context-engine factories, while this PR cleanly separates read-only discovery metadata from runtime factory activation.

## What This Changes

The PR adds internal lifecycle metadata to context-engine registrations so read-only plugin discovery cannot invoke or quarantine discovery-only factories before runtime activation.

## Best Possible Solution

Land the narrow lifecycle-marker fix once maintainers accept read-only context-engine registrations as metadata-only while keeping quarantine for real runtime factory failures.

## Reproduction Assessment

Yes, at source level: current main exposes context-engine registration during read-only discovery/tool-discovery and then invokes/quarantines selected factories during resolution. The exact-head proof artifact exercises that lifecycle with a real generated plugin and shows discovery fallback without quarantine followed by runtime activation.

## Solution Assessment

Yes. The internal lifecycle marker is the narrowest maintainable fix I found because it preserves the existing plugin API and runtime quarantine behavior while separating read-only discovery metadata from runtime factory construction.
