## Summary

Keep open as the active fix PR for the linked shipped regression. The patch looks correct and the new exact-head artifact satisfies the real-behavior proof gate, but the fallback/quarantine semantic change is still compatibility-sensitive maintainer territory before merge.

## What This Changes

The branch adds internal lifecycle metadata to context-engine registrations, tags plugin registrations by full versus read-only discovery mode, skips quarantine for discovery-only factories, and adds regression coverage.

## Best Possible Solution

Land the narrow lifecycle-marker fix once maintainers accept discovery-only context-engine registrations as metadata-only while keeping quarantine for actual runtime factory failures.

## Reproduction Assessment

Yes, at source level: current main exposes context-engine registration during read-only discovery/tool-discovery and then invokes/quarantines selected factories during resolution. The lossless-claw v0.13.1 source throws the reported error in those read-only modes; the PR artifact exercises the same lifecycle with a real generated plugin.

## Solution Assessment

Yes. The internal lifecycle marker is the narrowest maintainable fix I found because it preserves the existing plugin API and runtime quarantine behavior while separating read-only discovery metadata from runtime factory construction.
