## Summary

Keep open for maintainer follow-up: the feature is still not implemented on current main as an explicit compaction fallback config, but the PR branch is not mergeable as-is because it rewrites stale compaction internals, drops several current-main runtime safeguards, and lacks contributor real-behavior proof.

## What This Changes

The PR adds `agents.defaults.compaction.modelFallbacks` to config types/schema and rewrites direct Pi compaction to retry configured fallback models during resolution, auth, and runtime failures.

## Best Possible Solution

Settle the explicit compaction fallback API in https://github.com/openclaw/openclaw/issues/56781, then rework this PR or replace it with a narrow current-main patch that preserves the existing fallback pipeline, auth scope, stream wrappers, context budget, checkpoint persistence, prompt policy, and retry behavior.

## Reproduction Assessment

Yes for the PR regressions: source comparison of PR head against current main shows the missing workspace auth scope, stream wrapping, context-budget setting, checkpoint persistence, prompt policy, and retry behavior. No live runtime repro was run because this was a read-only review.

## Solution Assessment

No. The feature direction may be useful, but this branch adds a new public config contract while rewriting stale compaction internals; a maintainer-approved contract plus a narrow current-main patch is safer.
