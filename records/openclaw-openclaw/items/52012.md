## Summary

Keep open: current main only covers implicit compaction fallback via the session model chain, while this PR proposes a new explicit `modelFallbacks` config surface; the branch is not mergeable as-is because its large stale compaction rewrite drops current-main runtime/auth/checkpoint behavior.

## What This Changes

This PR adds an `agents.defaults.compaction.modelFallbacks` config/schema field and rewrites direct Pi compaction to retry configured fallback models during resolution, auth, and runtime failures.

## Best Possible Solution

If maintainers want explicit compaction fallback configuration, implement it as a narrow additive patch on current main that reuses the existing fallback/session-preparation pipeline and preserves the explicit-model semantics introduced by #74470.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists by comparing PR head `1c6b2f91d6089e2ba91cb7bc9fc2d7ac5ca5e34d` with current main: the changed lines directly omit required auth scope, stream wrapping, settings caps, checkpoint persistence, and fallback-session metrics behavior.

## Solution Assessment

No. The feature direction is useful, but this PR is not the best implementation because it rewrites stale compaction internals; the safer solution is a small current-main change that layers explicit fallback config onto the existing pipeline.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Wrap provider streams before assigning compaction streams:** `src/agents/pi-embedded-runner/compact.ts:958-960`
  - body: The PR assigns `providerStreamFn` directly to `session.agent.streamFn`. Current compaction goes through `resolveEmbeddedAgentStreamFn`, provider text transforms, and runtime extra-param wrappers; bypassing that stack can make provider-owned or OAuth-backed compaction calls fail even after auth prep succeeds.
  - confidence: 0.93
- **[P1] Persist checkpoints after successful compaction:** `src/agents/pi-embedded-runner/compact.ts:1238-1242`
  - body: The success path now runs post-compaction side effects and continues without calling `persistSessionCompactionCheckpoint`. The default legacy engine delegates into this direct path, so successful compactions would stop producing checkpoints for compaction list/restore workflows.
  - confidence: 0.91
- **[P1] Pass the context window into prepared compaction settings:** `src/agents/pi-embedded-runner/compact.ts:906-910`
  - body: `createPreparedEmbeddedPiSettingsManager` is now called without `contextTokenBudget`. The reserve-floor safety cap only runs when that value is present, so small-context compaction models can inherit an oversized reserve floor and repeatedly overflow instead of compacting.
  - confidence: 0.9
- **[P2] Use the compacted fallback session for metrics and hooks:** `src/agents/pi-embedded-runner/compact.ts:1244-1251`
  - body: When primary `session.compact()` fails and fallback `fbSession.compact()` succeeds, the code disposes the fallback session and then computes `tokensAfter`, `messageCountAfter`, diagnostics, and hooks from the original `session.messages`. That reports pre-compaction state and can trigger incorrect follow-up compaction decisions.
  - confidence: 0.89
- **[P2] Keep workspace auth scope in compaction auth resolution:** `src/agents/pi-embedded-runner/compact.ts:431-436`
  - body: The new auth helper omits `workspaceDir` when calling `getApiKeyForModel`, although current main passes the resolved workspace. Workspace-scoped provider auth evidence and env/secret resolution are therefore invisible for primary and fallback compaction models.
  - confidence: 0.86
