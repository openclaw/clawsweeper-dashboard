## Summary

Keep open. The feature direction is still relevant because current main has only implicit compaction fallback behavior, but this PR is not ready to merge: the branch rewrites stale direct-compaction internals, has blocking runtime regressions, lacks contributor real behavior proof, and the explicit public fallback contract is still tracked in https://github.com/openclaw/openclaw/issues/56781.

## What This Changes

The branch adds `agents.defaults.compaction.modelFallbacks` to config types/schema and rewrites direct Pi compaction to retry fallback models across resolution, auth, and runtime compaction failures.

## Best Possible Solution

Settle the explicit compaction fallback API in https://github.com/openclaw/openclaw/issues/56781, then replace or rework this branch as a narrow current-main patch that preserves the existing fallback pipeline, auth scope, stream wrappers, context budget, checkpoint persistence, and fallback-session metrics.

## Reproduction Assessment

Yes for the PR regressions: source comparison of PR head `1c6b2f91d6089e2ba91cb7bc9fc2d7ac5ca5e34d` against current main shows the missing workspace auth scope, stream wrapping, context-budget setting, checkpoint persistence, and fallback-session state handling. No live runtime repro was run because this was a read-only review.

## Solution Assessment

No. The feature direction may be useful, but this branch is not the best implementation as-is because it adds a new public config contract while rewriting stale compaction internals; a maintainer-approved contract plus a narrow current-main patch is safer.

  - body: The PR assigns `providerStreamFn` directly to `session.agent.streamFn`. Current compaction goes through the prepared embedded stream wrapper path, so provider-owned or OAuth-backed compaction calls can fail or miss runtime transport behavior even after auth prep succeeds.

  - confidence: 0.91

  - body: `createPreparedEmbeddedPiSettingsManager` is now called without `contextTokenBudget`. The small-context reserve cap only runs when that value is present, so compacting with smaller context models can inherit an oversized reserve floor and overflow again.
  - confidence: 0.9

  - body: When primary `session.compact()` fails and fallback `fbSession.compact()` succeeds, the code assigns `result = fbResult` but still computes `tokensAfter`, message counts, diagnostics, and hooks from the original session, reporting pre-compaction state.
  - confidence: 0.89
