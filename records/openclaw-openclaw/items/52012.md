## Summary

Keep open. Current main only has implicit compaction fallback behavior, the explicit public config contract is still unresolved in the related issue, and this PR still has blocking compaction regressions plus missing real behavior proof.

## What This Changes

The PR adds `agents.defaults.compaction.modelFallbacks` to config types/schema and rewrites direct Pi compaction to retry fallback models across resolution, auth, and runtime compaction failures.

## Best Possible Solution

Settle the public compaction fallback API in https://github.com/openclaw/openclaw/issues/56781, then implement a narrow current-main patch or replacement PR that reuses the existing fallback pipeline while preserving auth scope, stream wrappers, context-budget settings, checkpoints, and fallback-session metrics.

## Reproduction Assessment

Yes for the PR regressions: source comparison of PR head `1c6b2f91d6089e2ba91cb7bc9fc2d7ac5ca5e34d` against current main shows the missing workspace auth scope, stream wrapping, context-budget settings, checkpoint persistence, and fallback-session state handling. No live runtime repro was run because this was a read-only review.

## Solution Assessment

No. The feature direction may be useful, but this branch is not the best implementation because it rewrites stale compaction internals; a maintainer-approved contract plus a narrow current-main patch or replacement would be safer.

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Route compaction streams through runtime wrappers:** `src/agents/pi-embedded-runner/compact.ts:958-960`
  - body: The PR assigns `providerStreamFn` directly to `session.agent.streamFn`. Current compaction goes through `resolveEmbeddedAgentStreamFn` plus provider runtime/text wrappers, so provider-owned or OAuth-backed compaction calls can fail even after auth prep succeeds.
  - confidence: 0.93
- **[P1] Persist compaction checkpoints on success:** `src/agents/pi-embedded-runner/compact.ts:1238-1242`
  - body: The success path runs post-compaction side effects without calling `persistSessionCompactionCheckpoint`. Standard direct compaction would stop producing checkpoints for list/restore workflows after a successful compaction.
  - confidence: 0.91
- **[P1] Pass context token budget into prepared settings:** `src/agents/pi-embedded-runner/compact.ts:906-910`
  - body: `createPreparedEmbeddedPiSettingsManager` is now called without `contextTokenBudget`. The small-context reserve cap only runs when that value is present, so compacting with smaller context models can inherit an oversized reserve floor and overflow again.
  - confidence: 0.9
- **[P2] Compute metrics from the successful fallback session:** `src/agents/pi-embedded-runner/compact.ts:1236-1252`
  - body: When primary `session.compact()` fails and fallback `fbSession.compact()` succeeds, the code assigns `result = fbResult` but still computes `tokensAfter`, message counts, diagnostics, and hooks from the original session, reporting pre-compaction state.
  - confidence: 0.89
- **[P2] Pass workspaceDir into compaction auth resolution:** `src/agents/pi-embedded-runner/compact.ts:431-436`
  - body: `tryAuthForModel` calls `getApiKeyForModel` without `workspaceDir`, while current main passes the resolved workspace. Workspace-scoped provider auth evidence and env/secret resolution are therefore invisible for primary and fallback compaction models.
  - confidence: 0.86
