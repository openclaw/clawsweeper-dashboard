## Summary

Keep open. The feature is not already implemented on current main, but this PR is not merge-ready because the branch introduces a new public compaction fallback config contract while regressing several current compaction runtime paths and still lacks after-fix real behavior proof.

## What This Changes

The branch adds `AgentCompactionConfig.modelFallbacks` plus schema support and rewrites direct Pi compaction to retry configured fallback models during resolution, auth, and runtime failures.

## Best Possible Solution

Settle the explicit compaction fallback API in `https://github.com/openclaw/openclaw/issues/56781`, then rework this PR or replace it with a narrow current-main patch that preserves the existing compaction pipeline, auth scope, stream wrappers, context budget, checkpoint persistence, prompt policy, and retry behavior.

## Reproduction Assessment

Yes for the PR regressions: source comparison of PR head against current main shows the missing workspace auth scope, stream wrapping, context-budget setting, checkpoint persistence, prompt policy, and retry behavior. No live runtime repro was run in this read-only review.

## Solution Assessment

No. The feature direction may be useful, but this branch adds a public config contract while rewriting stale compaction internals; a maintainer-approved contract plus a narrower current-main patch is safer.

  - confidence: 0.84

  - confidence: 0.9

  - confidence: 0.93

  - confidence: 0.91

  - confidence: 0.89
