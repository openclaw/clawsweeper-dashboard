## Summary

Keep this PR open, but it should not merge as-is. Current main has adjacent compaction fallback support for the session model path, while this PR adds a new explicit `agents.defaults.compaction.modelFallbacks` config and rewrites a now much more evolved compaction pipeline, reintroducing several current-main regressions around checkpoints, runtime stream wrapping, settings, prompts, and fallback metrics.

## What This Changes

The PR adds an `agents.defaults.compaction.modelFallbacks` config field and changes direct embedded compaction to try configured fallback models during resolution, auth, and runtime compaction failures.

## Best Possible Solution

Rebase on current main and implement explicit compaction-model fallbacks as a thin extension of the existing `runWithModelFallback` and compaction runtime context paths, preserving checkpoint, runtime-plan, prompt, settings, hook, and metrics behavior with focused regression tests plus config docs/changelog coverage.

## Reproduction Assessment

Yes for the PR regressions by source inspection: the PR head shows the stale code paths that omit checkpoint persistence, runtime stream wrapping, and context budget threading while current main contains those contracts. I did not run live compaction tests because this review was read-only.

## Solution Assessment

No. The current PR is not the best way to solve the request because it replaces a stale copy of `compact.ts` instead of composing with the current fallback runner and runtime-plan compaction pipeline.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve compaction checkpoint persistence:** `src/agents/pi-embedded-runner/compact.ts:1238-1242`
  - body: The PR's success path runs post-compaction side effects but never captures or persists a compaction checkpoint snapshot. Current main captures before compaction and persists after success so `sessions.compaction.*` restore/list data survives; merging this would drop that feature for direct compactions.
  - confidence: 0.91
- **[P1] Route compaction streams through runtime wrappers:** `src/agents/pi-embedded-runner/compact.ts:958-960`
  - body: Assigning `providerStreamFn` directly bypasses the current compaction stream preparation stack, including `resolveEmbeddedAgentStreamFn`, runtime auth handling, websocket transport choice, provider text transforms, and extra params. Providers that rely on those wrappers can fail auth or send the wrong payload shape during compaction.
  - confidence: 0.9
- **[P1] Keep the context budget in prepared Pi settings:** `src/agents/pi-embedded-runner/compact.ts:906-910`
  - body: This call drops `contextTokenBudget`, so `createPreparedEmbeddedPiSettingsManager` cannot apply the small-context compaction reserve cap. For 16k-class models, the uncapped default reserve floor can exceed the model window and push compaction back into overflow loops.
  - confidence: 0.88
- **[P2] Use fallback session state after fallback compaction:** `src/agents/pi-embedded-runner/compact.ts:1236-1245`
  - body: When fallback compaction succeeds, the PR stores the `fbSession.compact()` result but continues computing `tokensAfter`, `messageCountAfter`, diagnostics, and hooks from the original `session.messages`. That original session did not run the successful compact, so callers and hooks can receive stale post-compaction metrics.
  - confidence: 0.88
- **[P2] Preserve configured prompt overrides and provider transforms:** `src/agents/pi-embedded-runner/compact.ts:843-872`
  - body: The PR builds one `appendPrompt` and wraps it directly, but current main first honors `resolveSystemPromptOverride` and then applies provider system-prompt transforms. Configured agent prompt overrides and provider prompt hooks would no longer apply to compaction requests.
  - confidence: 0.86
- **[P2] Pass owner context into action discovery:** `src/agents/pi-embedded-runner/compact.ts:783-797`
  - body: The action discovery input no longer includes `senderIsOwner`. Owner-scoped channel actions can therefore disappear from the compaction prompt even when the caller is an owner, diverging from normal run-time action discovery.
  - confidence: 0.82
