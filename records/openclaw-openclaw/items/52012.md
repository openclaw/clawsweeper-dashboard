## Summary

Keep open. Current main preserves exact explicit compaction-model semantics and this PR still represents a new config surface, but the branch is not merge-ready because its broad compaction rewrite introduces blocking regressions and lacks real behavior proof.

## What This Changes

The PR adds `agents.defaults.compaction.modelFallbacks` to config types/schema and rewrites direct Pi compaction to retry configured fallback models across resolution, auth, and runtime failures.

## Best Possible Solution

Decide the explicit-compaction fallback semantics first; if accepted, replace this with a narrow current-main patch that adds config/schema/docs/tests while reusing the existing fallback and compaction-session preparation pipeline.

## Reproduction Assessment

Yes for the PR regressions: source comparison of head `1c6b2f91d6089e2ba91cb7bc9fc2d7ac5ca5e34d` against current main shows missing workspace auth scope, stream wrapping, context-budget settings, checkpoint persistence, and fallback-session state handling. No live runtime repro was run because this was a read-only review.

## Solution Assessment

No. The feature direction may be useful, but this branch is not the best implementation because it rewrites stale compaction internals; a narrow current-main implementation is safer after maintainers decide the config semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Wrap provider streams before assigning compaction streams:** `src/agents/pi-embedded-runner/compact.ts:958-960`
  - body: The PR assigns `providerStreamFn` directly to `session.agent.streamFn`. Current compaction goes through `resolveEmbeddedAgentStreamFn` plus provider transform/runtime wrappers; bypassing that stack can make provider-owned or OAuth-backed compaction calls fail even after auth prep succeeds.
  - confidence: 0.93
- **[P1] Persist checkpoints after successful compaction:** `src/agents/pi-embedded-runner/compact.ts:1238-1242`
  - body: The success path runs post-compaction side effects and metrics without calling `persistSessionCompactionCheckpoint`. Standard direct compaction flows would stop producing checkpoints for list/restore workflows after a successful compaction.
  - confidence: 0.91
- **[P1] Pass context token budget into prepared settings:** `src/agents/pi-embedded-runner/compact.ts:906-910`
  - body: `createPreparedEmbeddedPiSettingsManager` is now called without `contextTokenBudget`. The small-context reserve cap only runs when that value is present, so compacting with smaller context models can inherit an oversized reserve floor and repeatedly overflow.
  - confidence: 0.9
- **[P2] Use fallback session state for post-compaction metrics:** `src/agents/pi-embedded-runner/compact.ts:1236-1252`
  - body: When primary `session.compact()` fails and fallback `fbSession.compact()` succeeds, the code assigns `result = fbResult` but still computes `tokensAfter`, message counts, diagnostics, and hooks from the original session. That reports pre-compaction state and can trigger incorrect follow-up compaction decisions.
  - confidence: 0.89
- **[P2] Pass workspaceDir into compaction auth resolution:** `src/agents/pi-embedded-runner/compact.ts:431-436`
  - body: The new auth helper calls `getApiKeyForModel` without `workspaceDir`, while current main passes the resolved workspace. Workspace-scoped provider auth evidence and env/secret resolution are therefore invisible for primary and fallback compaction models.
  - confidence: 0.86
