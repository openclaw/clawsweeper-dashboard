## Summary

Keep open. The feature is still useful, but this PR is not merge-ready: current main only supports session fallback for unset compaction models, the PR adds a new public config contract, and the current head still has blocking compaction/auth/runtime regressions plus no real behavior proof.

## What This Changes

The branch adds `AgentCompactionConfig.modelFallbacks` to config type/schema and rewrites direct Pi compaction to retry configured fallback models during resolution, auth, and runtime failures.

## Best Possible Solution

Approve the explicit compaction fallback API first, then land a narrow current-main implementation that preserves the existing compaction pipeline, auth scope, prompt policy, stream wrappers, context budgeting, checkpoint persistence, and retry behavior.

## Reproduction Assessment

Yes for the PR regressions by source comparison: PR head omits current-main auth scope, context budgeting, prompt policy, stream wrapping, checkpoint persistence, metrics, and retry behavior. No live compaction outage was run in this read-only review.

## Solution Assessment

No. The direction may be useful, but the branch adds a public config contract while rewriting stale compaction internals; a maintainer-approved contract plus a narrower patch against current main is safer.
