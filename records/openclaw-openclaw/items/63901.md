## Summary

Keep open. Current main still does not provide session-aware provider request metadata for OpenRouter-style `user`/`trace` attribution: it has static `extra_body` pass-through and a native Responses metadata hook, but the OpenAI-compatible completions/OpenRouter path still lacks dynamic session metadata and the related open PRs are not merged.

## What This Changes

Request session-aware API-provider request metadata, such as OpenRouter `trace`/`user` fields or `user = sessionKey`, for cost attribution across agents, graphs, and tasks.

## Best Possible Solution

Keep this open until maintainers define a privacy-conscious opt-in provider metadata contract and land the chosen `user`/`trace` behavior with cache-safe plumbing, payload-shape tests, and docs.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request, not broken documented behavior. Source inspection gives high-confidence proof that current main still lacks the requested session-aware OpenRouter/OpenAI-compatible metadata path.

## Solution Assessment

No. Static `extra_body` is only a partial workaround, and raw automatic session-key forwarding needs an explicit privacy-safe contract, provider/API guards, cache isolation, tests, and documentation before it is the right solution.
