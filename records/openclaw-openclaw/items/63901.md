## Summary

Keep open. Current main still lacks a session-aware path for provider request-body attribution: provider extra-param and stream-wrapper contexts do not receive session identity, OpenAI-compatible completions only support static `extra_body`, and the OpenRouter wrapper only handles routing/thinking behavior. The two related open PRs cover overlapping pieces, so this should remain a maintainer design/review item rather than a cleanup close.

## What This Changes

Request provider request metadata support so OpenClaw can pass `user` or `trace` fields, ideally derived from session/graph/task identity, to API providers for cost attribution.

## Best Possible Solution

Keep this open until maintainers choose a privacy-safe attribution design: an additive provider SDK/runtime seam for sanitized session attribution, OpenRouter/OpenAI-compatible `user` and `trace` handling, explicit precedence with static `extra_body`, focused request-payload tests, and docs that make third-party metadata forwarding opt-in.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence reproduction path: configure an OpenAI-compatible provider without static `extra_body` and there is no current-main path from `sessionKey`/`sessionId` into provider extra params, the OpenRouter wrapper, or the request body.

## Solution Assessment

Unclear for the proposed options. Static `extra_body` is already available and automatic `user = sessionKey` would be narrow, but the safer maintainable solution is an opt-in sanitized attribution seam with documented precedence rather than forwarding raw session keys by default.
