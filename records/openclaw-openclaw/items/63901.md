## Summary

Keep open. Current main supports static `extra_body` pass-through and a native transport-turn metadata seam, but it still does not provide session-aware OpenRouter/OpenAI-compatible request-body `user` or `trace` attribution; related PRs #54725 and #64813 remain open for design review.

## What This Changes

Request to add session-aware provider request metadata, such as OpenRouter/OpenAI-compatible `trace` or `user` fields, for per-graph, per-task, and per-agent cost attribution.

## Best Possible Solution

Add an opt-in, sanitized attribution design through a documented provider SDK/runtime seam, define precedence with static `extra_body`, support provider-specific `user`/`trace` payload shapes where appropriate, and add focused tests plus docs.

## Reproduction Assessment

Yes. Source inspection of current main shows no path that moves `sessionKey` or `sessionId` into provider extra-param contexts, OpenRouter stream wrapping, or an OpenAI-compatible `user`/`trace` request body.

## Solution Assessment

Unclear. The requested direction is useful, but automatically setting raw `user = sessionKey` is not clearly the safest default; an opt-in attribution seam with sanitization and documented precedence is the maintainable path.
