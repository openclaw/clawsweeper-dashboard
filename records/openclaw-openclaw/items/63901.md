## Summary

Keep open. Current main has static `extra_body` pass-through and an adjacent native OpenAI transport metadata seam, but it still does not implement dynamic session-aware OpenRouter/OpenAI-compatible `user` or `trace` request-body attribution; the open related PRs need maintainer design review first.

## What This Changes

Request to pass per-session graph/task/agent attribution metadata, such as OpenRouter `trace` or OpenAI-compatible `user`, into provider API requests.

## Best Possible Solution

Keep the issue open until maintainers choose an opt-in attribution design that threads sanitized session identity or an attribution object through provider hooks/OpenRouter request-body wrappers, with precedence tests and docs.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms the missing capability: provider extra-param and OpenRouter wrapper contexts do not receive sessionKey/sessionId, and only static `extra_body` is documented and tested.

## Solution Assessment

Unclear. Static `extra_body` is already a narrow hard-coded workaround, but dynamic `user`/`trace` needs an opt-in sanitized SDK/config design rather than automatically sending raw session keys.
