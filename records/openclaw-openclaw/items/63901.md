## Summary

Keep open. Current main supports static OpenAI-compatible `extra_body` pass-through and a native transport-turn metadata seam, but it still does not provide session-aware provider request-body `user` or `trace` attribution; the related open PRs #54725 and #64813 need maintainer design review before this can close.

## What This Changes

Request to forward per-session graph/task/agent metadata, or at least the session key as `user`, into provider API request bodies for cost attribution.

## Best Possible Solution

Add an opt-in, sanitized provider attribution design through a documented provider SDK/runtime seam, with OpenRouter/OpenAI-compatible `user`/`trace` handling, precedence tests, and docs.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main lacks the requested dynamic session-aware `user`/`trace` request-body path.

## Solution Assessment

Unclear. Static `extra_body` pass-through is not enough, and automatically sending raw `user = sessionKey` is not clearly the safest default; an opt-in sanitized attribution seam is the more maintainable direction.
