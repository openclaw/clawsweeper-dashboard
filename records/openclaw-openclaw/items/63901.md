## Summary

Keep open. Current main still lacks a session-aware provider request metadata contract for OpenRouter/LiteLLM-style `user` or `trace` attribution; the existing code only provides static `extra_body` pass-through and native Responses metadata, while related implementation PRs remain open with proof, cache, privacy, and API-shape blockers.

## What This Changes

The issue requests provider request metadata support so API providers such as OpenRouter can attribute costs by session, graph, task, or agent.

## Best Possible Solution

Keep this open until maintainers define one privacy-safe provider metadata contract, then land the chosen session attribution path with provider/API guards, cache-safe plumbing, focused payload tests, and docs.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request, not broken documented behavior. Source inspection gives high-confidence proof that current main lacks dynamic/session-aware provider request metadata for the OpenAI-compatible/OpenRouter path.

## Solution Assessment

No settled best implementation yet. Static `extra_body` is only a workaround, and automatic session-key forwarding needs a maintainer-approved opt-in privacy contract, provider/API guards, cache isolation, tests, and documentation.
