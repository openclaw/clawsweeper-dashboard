## Summary

Keep open. Current main has adjacent tracing and static request-body pass-through, but it still lacks a settled session-aware provider metadata contract for OpenRouter/LiteLLM-style `user` or `trace` attribution, and the related open PRs still need maintainer API/privacy review.

## What This Changes

The issue requests a provider metadata contract so OpenClaw can pass trace or user attribution fields to upstream API providers for per-session/task cost tracking.

## Best Possible Solution

Keep this issue open until maintainers choose one privacy-safe, opt-in provider request metadata contract, then land the chosen attribution path with provider/API guards, cache-safe plumbing, focused payload tests, and docs.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main lacks dynamic/session-aware `user` or `trace` provider request metadata for the OpenAI-compatible/OpenRouter path.

## Solution Assessment

No settled best implementation yet. Static `extra_body`, generic `traceparent`, and transport-turn hooks are adjacent, but automatic session-key forwarding needs a maintainer-approved opt-in privacy contract, provider/API guards, cache isolation, tests, and documentation.
