Full review comments:

- **[P1] Count dropped summaries before declaring followup drain complete:** `src/auto-reply/reply/queue/drain-all.ts:21`
  - body: `scheduleFollowupDrain` keeps draining while `queue.droppedCount > 0`, but this waiter only counts `items.length` plus the active-drain sentinel. A queue that has summarized overflow work pending but no queued items will return `{ drained: true }`, so the SIGUSR1 path can restart before those summarized messages are delivered.
  - confidence: 0.91
- **[P1] Keep the restart bypass out of the public queue API:** `src/plugin-sdk/channel-lifecycle.core.ts:33-36`
  - body: `ChannelRunQueue` is exported through `openclaw/plugin-sdk/channel-lifecycle`, so adding `enqueueInternal` promotes an internal gateway-drain bypass to plugin callers. That gives third-party code a second enqueue path with shutdown-admission semantics that maintainers have not explicitly accepted; keep this helper private to core or document and test it as a public SDK contract.
  - confidence: 0.83
