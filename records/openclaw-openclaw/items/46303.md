## Summary

Keep open: the PR targets a real source-reproducible message-loss path and has useful terminal proof, but the latest head still has blocking restart-drain correctness gaps.

## What This Changes

The PR adds global inbound debouncer flushing, followup/channel run queue drain waiters, gateway-drain internal enqueue bypasses, SIGUSR1 restart integration, and related tests.

## Best Possible Solution

Keep the targeted in-memory restart-drain fix, but keep live debouncers registered until ingress is closed and make drain waiters mirror the actual queue pending predicates; leave durable post-restart replay to https://github.com/openclaw/openclaw/issues/49692.

## Reproduction Assessment

Yes. Current main keeps debounce buffers in closure-local timers and the SIGUSR1 shutdown path has no global debounce/followup drain, so the reported loss path is source-reproducible without a live Signal account.

## Solution Assessment

No. The PR is close and follows the right boundary, but pre-close stale debouncer eviction and incomplete followup pending accounting can still let restart-time work be lost.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep idle live debouncers registered until ingress closes:** `src/auto-reply/inbound-debounce.ts:99-100`
  - body: This stale-entry branch unregisters any debouncer idle for five minutes during the restart flush, but `server.close()` has not run yet and channel handlers still hold the debouncer object. A late inbound request can buffer on that now-unregistered handle, so the global restart sweep cannot find the timer-backed buffer before shutdown; defer this cleanup to owner teardown or a post-ingress-close path.
  - confidence: 0.86
- **[P2] Count dropped summaries as pending followup work:** `src/auto-reply/reply/queue/drain-all.ts:21-22`
  - body: `waitForFollowupQueueDrain()` can return drained for a queue with `droppedCount > 0` and no items, but `scheduleFollowupDrain()` still treats that summary state as work to process. During SIGUSR1 shutdown this can close the gateway before overflow summaries are emitted; include `droppedCount` in the pending count and cover the summary-only case.
  - confidence: 0.84
