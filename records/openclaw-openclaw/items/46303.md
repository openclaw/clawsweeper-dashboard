## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Count dropped summaries before declaring queues drained:** `src/auto-reply/reply/queue/drain-all.ts:21-22`
  - body: `waitForFollowupQueueDrain()` can return `drained: true` while a queue still has `droppedCount > 0`. The real drain loop continues while `queue.items.length > 0 || queue.droppedCount > 0`, so a restart can proceed before overflow-summary followups are delivered. Mirror the drain loop's pending-work predicate and add a regression for a summary-only queue.
  - confidence: 0.93
- **[P1] Route overflow summaries through the drain bypass:** `src/auto-reply/reply/queue/drain.ts:748-749`
  - body: After this PR marks the gateway draining before followup drain, normal queued items use `runTrustedFollowup()` to set `allowDuringGatewayDrain`, but this overflow-summary branch still passes `effectiveRunFollowup` directly. Synthetic summary runs then enter the embedded runner without the bypass and can be rejected by `GatewayDrainingError`, losing the summarized messages during restart.
  - confidence: 0.89
- **[P2] Make global debouncer registration lifecycle-owned:** `src/auto-reply/inbound-debounce.ts:478-480`
  - body: The new global registry keeps a strong reference to every `createInboundDebouncer()` instance, but sibling call sites such as Telegram, Signal, Mattermost, and WhatsApp still do not unregister their debouncers on shutdown. In in-process reload cycles, stale closures can accumulate and future SIGUSR1 sweeps can run old handler state; either make registration opt-in/lifecycle-owned or update all debouncer owners.
  - confidence: 0.81
- **[P2] Complete the queue-state test fixture:** `src/auto-reply/reply/queue/drain-all.test.ts:7-20`
  - body: `createMockQueue()` is declared to return `FollowupQueueState`, but the added fixture omits current required fields such as `summaryElisions` and `evictedSummaryCount`. That leaves the new test file out of sync with `state.ts` and can fail strict test type checking before the drain assertions run.
  - confidence: 0.86
