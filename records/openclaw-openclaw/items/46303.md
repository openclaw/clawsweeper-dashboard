## What This Changes

The PR adds restart-time flushing of inbound debouncers, channel run queue and followup queue drain waiters, gateway-drain bypass plumbing, and related tests across auto-reply, gateway, Discord, Feishu, plugin SDK, and command queue code.

## Best Possible Solution

Land a focused restart-drain fix that keeps the early gateway-draining gate, drains accepted debounced and queued work with the same pending-work predicate as the real drain loop, and includes redacted production-path proof.

## Reproduction Assessment

Yes for the PR-introduced issue by source inspection: a followup queue with droppedCount > 0 but no items or draining sentinel is reported drained by the new waiter. I did not establish a live Signal/config.patch/SIGUSR1 reproduction in this read-only review.

## Solution Assessment
