## Summary

Keep open: current main still has the reported message-loss path, but the latest PR branch has blocking restart-lifecycle, followup-drain, SDK-compatibility, and availability concerns that should be resolved before merge.

## What This Changes

The PR adds a global inbound debouncer flush registry, an all-followup queue drain waiter, SIGUSR1 restart integration, and bundled channel teardown/test updates.

## Best Possible Solution

Revise the branch around a restart-drain design that gates external ingress immediately, allows trusted internally flushed work to drain, mirrors followup queue pending semantics, and keeps the debouncer SDK contract backward-compatible.

## Reproduction Assessment

Yes. Source inspection on current main shows debounced messages live in closure-local timer buffers, and the SIGUSR1 restart path does not flush those buffers or wait for all followup queues before teardown.

## Solution Assessment

No. The PR targets the right failure mode, but the current approach delays the external enqueue gate, misses summary-only followup work, and changes plugin-facing debounce contracts instead of adding a compatible lifecycle seam.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate external enqueues before restart drain awaits:** `src/cli/gateway-cli/run-loop.ts:424`
  - body: The branch flushes debouncers and waits for active/followup drains before `markGatewayDraining()` runs. During those awaited steps, remote channel traffic can still enqueue new LLM/tool work, prolonging restart and recreating a late-work loss window; add an external-ingress gate or internal-only bypass so trusted flushed work can drain without accepting new external commands.
  - confidence: 0.84
- **[P2] Include dropped summaries in followup drain waiting:** `src/auto-reply/reply/queue/drain-all.ts:21-22`
  - body: `scheduleFollowupDrain` keeps draining while `queue.items.length > 0 || queue.droppedCount > 0`, but this new waiter only counts queued items plus the active-drain sentinel. A summary-only overflow queue can report drained before the summary followup is sent.
  - confidence: 0.91
- **[P2] Keep live idle debouncers registered during restart flush:** `src/auto-reply/inbound-debounce.ts:83-84`
  - body: The stale-entry branch can unregister a live but quiet debouncer during the restart flush while the server is still accepting ingress. A late message for that channel can then buffer on a handle no later global sweep can see before teardown.
  - confidence: 0.86
- **[P2] Preserve the public flushKey return contract:** `src/auto-reply/inbound-debounce.ts:235-240`
  - body: `createInboundDebouncer` is exported through plugin SDK subpaths, but the PR changes `flushKey` from the old `Promise<void>` behavior to `Promise<boolean>` and expands the returned object. Keep existing plugin-facing calls compatible and expose delivery/count state through an additive API.
  - confidence: 0.78
