## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Do not proceed after incomplete debounce flushes:** `src/auto-reply/inbound-debounce.ts:64-74`
  - body: `flushAllInboundDebouncers()` races each debouncer against the timeout, but when the timeout wins it resolves `{ drained: false }` and then discards that signal by returning only `flushedCount`. The underlying flush can still enqueue work after the restart waiters have already observed the queues, or remaining buffers can be left for shutdown to drop. Please propagate incomplete flush state and keep the restart path from treating it as drained/no-op.
  - confidence: 0.9
- **[P1] Count overflow summaries while waiting for followup drain:** `src/auto-reply/reply/queue/drain-all.ts:19-22`
  - body: This waiter reports drained when a queue has no `items` and is not currently `draining`, but the production drain loop also keeps running while `droppedCount > 0` so it can emit the overflow summary. A restart can therefore proceed before that user-visible summary work is processed. Include `droppedCount` in the pending predicate and cover the summary-only case.
  - confidence: 0.92
