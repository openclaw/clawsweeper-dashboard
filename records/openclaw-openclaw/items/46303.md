## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Count dropped summaries as pending followup work:** `src/auto-reply/reply/queue/drain-all.ts:20-21`
  - body: The production drain loop keeps running while `queue.items.length > 0 || queue.droppedCount > 0`, because summarize-on-overflow state is still queued work. This waiter only counts `items.length` and `draining`, so a queue holding only overflow summary state can return `drained: true` and let SIGUSR1 continue before the summary followup is emitted. Mirror the production predicate or include `droppedCount`, and add a droppedCount-only regression test.
  - confidence: 0.91
- **[P1] Stop evicting live debouncers during restart flush:** `src/auto-reply/inbound-debounce.ts:88-90`
  - body: This auto-eviction runs during the restart sweep before `server.close()` has proven ingress is closed. A live but quiet channel older than five minutes can be removed from the global registry, then a late message can buffer on that unregistered handle and miss the final global flush before teardown. Do not unregister live handles from `flushAllInboundDebouncers`; stale cleanup needs an explicit lifecycle signal or a path that runs after ingress is detached.
  - confidence: 0.84
