## Review Findings

Full review comments:

- **[P1] Count dropped summaries as pending followups:** `src/auto-reply/reply/queue/drain-all.ts:21-22`
  - body: `waitForFollowupQueueDrain` treats a queue as pending only when it has items or an active drain. The real drain loop also runs while `droppedCount > 0`, so a summary-only overflow queue can be reported as drained and SIGUSR1 shutdown can continue before that followup summary is delivered. Include `queue.droppedCount` in this count and add a summary-only regression case.
  - confidence: 0.94
- **[P1] Preserve the public debouncer return contract:** `src/auto-reply/inbound-debounce.ts:480`
  - body: `createInboundDebouncer` is re-exported through the channel plugin SDK, so adding required `ReturnType` members here can break external plugins or tests that provide typed debouncer mocks. Keep the public return shape compatible, or make the restart-only controls internal/explicitly optional with the usual SDK contract discussion and coverage.
  - confidence: 0.86
- **[P2] Stop timed-out flushes from enqueueing late work:** `src/auto-reply/inbound-debounce.ts:65-75`
  - body: The timeout branch races `handle.flushAll(...)` against a timer but does not cancel or await the original flush. If the timer wins, that flush can continue in the background and enqueue channel/followup work after the restart path has already measured and waited on queues, recreating a shutdown-time loss window.
  - confidence: 0.82
