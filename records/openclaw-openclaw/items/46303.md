## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Count overflow summaries before declaring drain complete:** `src/auto-reply/reply/queue/drain-all.ts:21-22`
  - body: `scheduleFollowupDrain` keeps draining while `queue.items.length > 0 || queue.droppedCount > 0`, but this new waiter only counts `items.length` plus the in-flight sentinel. A restart can therefore see `{ drained: true }` while a queue only has overflow-summary state left, skip the remaining summary followup, and still lose messages during teardown. Mirror the drain predicate here, including `droppedCount`, before returning drained.
  - confidence: 0.92
- **[P2] Keep internal drain helpers out of the plugin SDK:** `src/plugin-sdk/channel-lifecycle.core.ts:60-62`
  - body: `waitForChannelRunQueueDrain` is added to `channel-lifecycle.core.ts`, which is re-exported by the public `openclaw/plugin-sdk/channel-lifecycle` facade, while `ChannelRunQueue` also gains an internal enqueue bypass. That publishes restart-drain internals as plugin API without the docs/baseline/compat decision required for SDK changes; move the registry/wait helper to an internal module or explicitly document and baseline the new public contract.
  - confidence: 0.84
