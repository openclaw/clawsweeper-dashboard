## Summary

Keep this PR open: the current-main bug is real, but the branch still has merge-blocking lifecycle, queue-drain, and SDK-compatibility issues that can preserve or reintroduce restart-time message loss.

## What This Changes

The PR adds a global inbound debouncer flush registry, a followup queue drain waiter, SIGUSR1 restart integration, and bundled channel teardown/test updates.

## Best Possible Solution

Revise the branch around a restart-drain design that gates external ingress immediately, permits trusted internally flushed followup work, mirrors existing queue pending semantics, and preserves public SDK compatibility.

## Reproduction Assessment

Yes. Source inspection on current main shows debounced messages live in closure-local timer buffers and the SIGUSR1 restart path does not flush those buffers or wait for all followup queues before teardown.

## Solution Assessment

No. The PR targets the right failure mode, but the current diff delays the external enqueue gate, misses summary-only followup work, evicts live idle debouncers during restart flush, and changes a public SDK-returned method contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate external enqueues before restart drain awaits:** `src/cli/gateway-cli/run-loop.ts:424`
  - body: The branch awaits debounce flushing, active/runtime drain, and followup drain before `markGatewayDraining()` runs. During that window remote channel traffic can still enqueue new LLM/tool work, prolonging restart and recreating a late-work loss window; add an external-ingress gate or internal-only bypass so trusted flushed work can drain without accepting new external commands.
  - confidence: 0.84
- **[P2] Include dropped summaries in followup drain waiting:** `src/auto-reply/reply/queue/drain-all.ts:21-22`
  - body: `scheduleFollowupDrain` keeps draining while `queue.items.length > 0 || queue.droppedCount > 0`, but this new waiter only counts queued items plus the active-drain sentinel. A summary-only overflow queue can report drained before the summary followup is sent.
  - confidence: 0.91
- **[P2] Keep live idle debouncers registered during restart flush:** `src/auto-reply/inbound-debounce.ts:83-84`
  - body: The stale-entry branch can unregister a live but quiet debouncer during the restart flush while the server is still accepting ingress. A late message for that channel can then buffer on a handle no later global sweep can see before teardown.
  - confidence: 0.86
- **[P2] Preserve the public flushKey return contract:** `src/auto-reply/inbound-debounce.ts:235-240`
  - body: `createInboundDebouncer` is exported through plugin SDK subpaths, but the PR changes `flushKey` from the old `Promise<void>` behavior to `Promise<boolean>` and expands the returned object. Keep existing plugin-facing calls compatible and expose delivery/count state through an additive API.
  - confidence: 0.78
