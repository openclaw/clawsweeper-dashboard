- **[P1] Do not let timed-out flushes continue detached:** `src/auto-reply/inbound-debounce.ts:64-74`
  - body: The timeout branch races `handle.flushAll()` against a timer and then returns only `flushedCount`, while the original flush can keep running in the background. If that slow `onFlush` later enqueues channel or followup work after the restart drain waiters have already observed empty queues, shutdown can still drop the buffered message; surface the not-drained state or bound the flush cooperatively without detaching it.
  - confidence: 0.86
- **[P1] Count overflow summaries before reporting drained:** `src/auto-reply/reply/queue/drain-all.ts:21-22`
  - body: `scheduleFollowupDrain()` treats `queue.droppedCount > 0` as pending work, but this waiter only counts `items.length` and the active-drain sentinel. A queue retaining summarize-on-overflow work can be reported drained before its summary prompt is emitted, so include `droppedCount` and add regression coverage for that state.
  - confidence: 0.84
- **[P2] Keep the drain bypass out of the public SDK surface:** `src/plugin-sdk/channel-lifecycle.core.ts:28-31`
  - body: `plugin-sdk/channel-lifecycle` is a documented public subpath, but this patch adds `enqueueInternal` as a parallel queue call for restart drain bypasses without API discussion or docs. Keep that bypass private to core/bundled restart plumbing, or get explicit maintainer approval and document the new plugin contract.
  - confidence: 0.78
