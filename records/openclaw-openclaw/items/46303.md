## Summary

Keep open: the PR is a plausible fix for restart-time message loss, but it is not merge-ready because the new followup-drain waiter does not mirror the existing queue drain predicate, the real-behavior proof stops at helper-level terminal drivers, and one required CI lane is still red.

## What This Changes

The PR adds a global inbound debouncer registry, restart-time debounce/channel/followup drain waits, and internal gateway-drain enqueue bypasses across the command, followup, and Discord channel queues.

## Best Possible Solution

Land a focused restart-drain fix that preserves the early gateway-draining gate, flushes accepted debounced work under an internal drain context, waits using the same pending-work predicate as the real drain loops, and includes redacted real restart proof.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. The source path is credible because current main stores debounced inbound messages in closure-local maps/timers and the SIGUSR1 run loop has no global debouncer or followup-queue flush before teardown.

## Solution Assessment

No, not quite yet. The implementation direction is maintainable, but waitForFollowupQueueDrain should mirror scheduleFollowupDrain’s droppedCount-aware predicate and the PR needs production-path proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Count dropped summaries while waiting for queue drain:** `src/auto-reply/reply/queue/drain-all.ts:21`
  - body: `waitForFollowupQueueDrain` can report drained when a queue has no `items` and is not currently marked `draining`, even though `droppedCount` still represents pending overflow-summary work. The real drain loop keeps running while `queue.items.length > 0 || queue.droppedCount > 0`, so the restart path can proceed to server close before that summary followup is processed. Include `queue.droppedCount` in the pending count and cover the summary-only case.
  - confidence: 0.82
