## Summary

Keep open. The PR targets a real SIGUSR1 message-loss path, but the proposed branch still needs restart lifecycle and SDK contract changes before it is safe to merge.

## What This Changes

The PR adds a global inbound debouncer flush path, a followup queue drain waiter, SIGUSR1 restart integration, and bundled channel teardown/test updates.

## Best Possible Solution

Revise the branch around a backward-compatible restart-drain design: gate or pause external ingress as restart begins, allow only trusted internally flushed work to drain, mirror followup queue pending semantics including dropped summaries, and expose any new drain metadata through additive SDK APIs.

## Reproduction Assessment

Yes. Source inspection on current main shows debounced messages live in closure-local timer buffers and the SIGUSR1 restart path does not flush them or wait for followup queues before teardown.

## Solution Assessment

No. The PR addresses the right failure mode, but the current approach leaves an external enqueue window during restart and changes plugin-facing debounce contracts instead of adding a compatible lifecycle seam.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate external command enqueues when restart begins:** `src/cli/gateway-cli/run-loop.ts:424`
  - body: The PR waits through debounce flushing and followup draining before setting `gatewayDraining`. During that awaited window, inbound channel traffic can still enqueue new command work, so restarts can be prolonged and late work can still be lost or create avoidable LLM/tool cost.
  - confidence: 0.84
- **[P2] Count dropped-summary followup work:** `src/auto-reply/reply/queue/drain-all.ts:21-22`
  - body: `scheduleFollowupDrain` keeps draining while `queue.items.length > 0 || queue.droppedCount > 0`, but the new all-queue waiter only counts queued items plus the active-drain sentinel. A summary-only overflow queue can report drained before the summary followup is sent.
  - confidence: 0.91
- **[P2] Keep live debouncers registered until teardown:** `src/auto-reply/inbound-debounce.ts:83-84`
  - body: The stale-entry eviction path removes debouncers during restart flush based only on age while channel ingress can still deliver events. A live but quiet channel can then buffer a late message on a handle that no later global sweep can see before teardown.
  - confidence: 0.86
- **[P2] Preserve the public flushKey return contract:** `src/auto-reply/inbound-debounce.ts:235-240`
  - body: `createInboundDebouncer` is exported through plugin SDK subpaths, but the PR changes `flushKey` from `Promise<void>` to `Promise<boolean>`. Existing plugins typed to the old contract can fail to type-check; expose delivery/count state through an additive API instead.
  - confidence: 0.78
