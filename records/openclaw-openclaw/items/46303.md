## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Count dropped summaries before declaring followups drained:** `src/auto-reply/reply/queue/drain-all.ts:21-22`
  - body: The production drain loop keeps working while `queue.items.length > 0 || queue.droppedCount > 0`, but this waiter ignores `droppedCount`. A summarize-on-overflow queue can therefore be reported as drained during restart while summary followup work is still pending.
  - confidence: 0.9
- **[P1] Preserve the plugin-facing debouncer return contract:** `src/auto-reply/inbound-debounce.ts:478`
  - body: `createInboundDebouncer` is re-exported through the plugin SDK, and the inferred return object now requires `flushKeyWithCount`, `flushAll`, and `unregister`. That can break typed third-party runtime stubs for an internal restart-drain feature unless the new methods are kept out of the public contract or shipped with an explicit compat/deprecation path.
  - confidence: 0.82
- **[P2] Report timed-out debounce flushes before restart continues:** `src/auto-reply/inbound-debounce.ts:89`
  - body: When a debouncer hits the timeout, the race can return `{ drained: false }`, but `flushAllInboundDebouncers` drops that state and returns only `flushedCount`. The restart path can treat a timed-out or incomplete sweep like idle work and continue while buffered messages remain or finish flushing too late.
  - confidence: 0.8
