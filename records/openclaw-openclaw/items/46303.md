Full review comments:

- **[P1] Count overflow summaries as pending followups:** `src/auto-reply/reply/queue/drain-all.ts:21`
  - body: `scheduleFollowupDrain` keeps draining while `queue.items.length > 0 || queue.droppedCount > 0`, because overflow summaries are followup work even when the item array is empty. This new waiter only counts `items.length` and `draining`, so a summary-only queue can report drained and let SIGUSR1 shutdown continue before the summary followup runs.
  - confidence: 0.98
- **[P1] Keep the drain bypass out of the public queue contract:** `src/plugin-sdk/channel-lifecycle.core.ts:30-33`
  - body: `ChannelRunQueue` is exported through `openclaw/plugin-sdk/channel-outbound`, so adding `enqueueInternal` makes the restart-drain bypass a public plugin API that any channel can call. Keep this capability internal or add explicit SDK approval/docs/tests before exposing a permanent parallel enqueue path.
  - confidence: 0.9
