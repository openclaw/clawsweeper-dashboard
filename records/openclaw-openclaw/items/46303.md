## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Surface incomplete debounce flushes:** `src/auto-reply/inbound-debounce.ts:89`
  - body: `handle.flushAll()` can return `drained:false` after the deadline with buffers still present, but this aggregation drops that state and returns only the count. The SIGUSR1 path can then continue to close/restart without knowing buffered inbound messages were left behind, recreating the message-loss case this PR is meant to prevent.
  - confidence: 0.84
- **[P2] Include overflow summaries in drain pending checks:** `src/auto-reply/reply/queue/drain-all.ts:21`
  - body: The production drain loop runs while `queue.items.length > 0 || queue.droppedCount > 0`, but this waiter only counts `items` and `draining`. A summarize-overflow queue with only `droppedCount` left will be reported drained and can be discarded on restart before its summary followup runs.
  - confidence: 0.9
- **[P1] Preserve the debouncer SDK return contract:** `src/auto-reply/inbound-debounce.ts:478`
  - body: `createInboundDebouncer` is re-exported through the plugin SDK, so adding new required return members changes the inferred `ReturnType` that third-party plugin runtime stubs and tests implement. Keep the old public shape and put restart-only handles behind an internal registry, or add an explicit maintainer-approved compatibility/versioning path with focused SDK coverage.
  - confidence: 0.75
