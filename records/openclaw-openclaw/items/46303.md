Full review comments:

- **[P1] Count overflow summaries as pending follow-up work:** `src/auto-reply/reply/queue/drain-all.ts:21-22`
  - body: `waitForFollowupQueueDrain()` currently ignores `queue.droppedCount`, so a queue that has overflow-summary work but no `items` and no active drain is reported as fully drained. The real drain loop keeps running while `items.length > 0 || droppedCount > 0`, so SIGUSR1 shutdown can proceed before those summarized messages are delivered.
  - confidence: 0.94
- **[P2] Contain timed-out debounce flush callbacks:** `src/auto-reply/inbound-debounce.ts:66-75`
  - body: The timeout path races `handle.flushAll()` and then returns while the original flush can still be executing. Because `flushBuffer()` removes the buffer before awaiting the callback, a slow `onFlush` can enqueue work after the restart path has already waited on channel/follow-up queues, recreating the late-message loss window this PR is trying to close.
  - confidence: 0.86
