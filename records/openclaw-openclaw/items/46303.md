Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Count overflow summaries as pending follow-up work:** `src/auto-reply/reply/queue/drain-all.ts:21`
  - body: `waitForFollowupQueueDrain()` reports a queue as drained when `items.length === 0` and `draining` is false, but the real drain loop keeps running while `droppedCount > 0` to deliver overflow summaries. During restart this can allow shutdown to proceed while summarized follow-up work is still pending, recreating message loss for capped queues.
  - confidence: 0.93
- **[P2] Complete the mock queue state shape:** `src/auto-reply/reply/queue/drain-all.test.ts:7-20`
  - body: `createMockQueue()` is declared to return `FollowupQueueState`, but the returned object omits required `summaryElisions` and `evictedSummaryCount` fields from the current queue state contract. Add those fields here so the new test file type-checks against the queue shape it is exercising.
  - confidence: 0.9
- **[P2] Contain timed-out debounce flush work:** `src/auto-reply/inbound-debounce.ts:66-75`
  - body: When the timeout branch of this `Promise.race` wins, the underlying `handle.flushAll(...)` keeps running and can still call `onFlush` later. The restart path may then observe empty channel/follow-up queues and continue toward close before that late flush enqueues its work, so the timeout needs to cancel, await, or otherwise account for the in-flight flush task.
  - confidence: 0.82
