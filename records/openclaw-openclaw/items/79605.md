The issue reports that Telegram `streaming.mode: "partial"` does not create preview/edit updates and only sends the final message.

## Best Possible Solution

Restore Telegram partial preview edits for long text-only direct-model replies while preserving the 30-character first-preview debounce, then add focused regression coverage and redacted live Telegram DM proof or equivalent runtime logs.

## Reproduction Assessment

No high-confidence live reproduction was run here. The issue has concrete reporter steps, and current source shows a final-only path when no usable partial text reaches Telegram before finalization or before the 30-character preview gate.

## Solution Assessment

No patch is proposed here. The rejected related PR shows that removing the partial-mode debounce is not the best fix; the safer path is to diagnose where direct-model text deltas stop reaching the existing Telegram draft stream and preserve the debounce behavior.
