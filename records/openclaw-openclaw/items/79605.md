## Summary

Keep open: current main still documents and implements Telegram partial preview streaming, but the direct-model failure point is not isolated; related debounce-removal PRs were closed unmerged and a maintainer explicitly asked for delta/send/edit logs before changing the debounce behavior.

## What This Changes

The issue reports that Telegram `streaming.mode: "partial"` on OpenClaw 2026.5.7 sends only the final Telegram message instead of creating and editing a live preview.

## Best Possible Solution

Restore Telegram partial preview edits for long text-only direct-model replies while preserving the 30-character first-preview debounce, then add focused regression coverage and redacted live Telegram DM proof or equivalent runtime logs.

## Reproduction Assessment

No, there is not a high-confidence live reproduction from this review. The issue has concrete steps and current source shows a plausible final-only path when usable text deltas do not reach Telegram before finalization or before crossing the first-preview debounce.

## Solution Assessment

No patch is proposed here. The rejected related PR shows that removing the partial-mode debounce is not the best fix; the safer path is to identify where direct-model text deltas stop reaching the existing Telegram draft stream while preserving the debounce.
