The issue reports that Telegram `streaming.mode: "partial"` sends only a final Telegram message instead of creating and editing a live preview message.

## Best Possible Solution

Restore observable Telegram partial preview edits for long text-only direct-model replies while preserving the 30-character first-preview debounce, with focused regression coverage and redacted runtime proof showing text deltas, preview `sendMessage`, `editMessageText`, and finalization.

## Solution Assessment

No patch is proposed on this issue. The previously proposed debounce-removal direction is not the best fix because maintainer review identified that debounce as intentional Telegram push-notification UX; the safer fix needs to preserve it and isolate where usable text deltas stop.
