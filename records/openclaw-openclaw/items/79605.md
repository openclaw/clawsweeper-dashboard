## Summary

Keep open: the Telegram partial-preview path is documented and still present on current main, but the exact break point is not isolated and maintainer discussion says the issue needs a fix that preserves the intentional 30-character first-preview debounce.

## What This Changes

The issue reports that Telegram `streaming.mode: "partial"` sends only the final message in OpenClaw 2026.5.7 instead of creating and editing a live preview message.

## Best Possible Solution

Restore Telegram partial preview edits for long text-only direct-model replies while preserving the 30-character first-preview debounce, then add focused regression coverage and redacted live Telegram DM proof or equivalent runtime logs.

## Reproduction Assessment

No high-confidence live reproduction was run here. The issue has concrete steps and current source shows a plausible final-only path when no usable partial text reaches Telegram before finalization or before the 30-character preview gate.

## Solution Assessment

No patch is proposed here. The related closed PRs show that removing the partial-mode debounce is not the best fix; the safer path is to confirm where text deltas stop and preserve the existing Telegram draft-stream contract.
