## Summary

Keep open: current main still matches the reported Telegram partial-preview failure surface, and the most recent maintainer follow-up asks for live gateway logs before choosing a fix because the issue may be upstream text-delta delivery rather than Telegram config.

## What This Changes

The issue requests restoring Telegram `streaming.mode: "partial"` so DMs create a preview message and update it with `editMessageText` instead of sending only the final response.

## Best Possible Solution

Preserve Telegram's first-preview debounce, isolate whether cumulative partial text over the threshold reaches the draft stream, then add a focused regression plus redacted live/log proof showing preview `sendMessage` followed by `editMessageText` before final delivery.

## Reproduction Assessment

No, we do not have a high-confidence live reproduction in this review. Source inspection shows how final-only delivery can happen when no usable partial text, or only below-threshold partial text, reaches Telegram before final, but the requested gateway logs are still needed to confirm the reported direct-model path.

## Solution Assessment

No patch is proposed, and the closed debounce-removal PR is not the best fix because maintainers want to preserve that UX. The best path is a narrower fix after confirming whether text deltas are missing upstream or being dropped in Telegram draft streaming.
