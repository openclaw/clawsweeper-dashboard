## Summary

Keep open: current main still has the relevant Telegram partial-preview path, the reported final-only behavior is not fixed, and a maintainer has already requested targeted live logs to isolate whether text deltas arrive before final delivery.

## What This Changes

The issue reports that Telegram `streaming.mode: "partial"` in OpenClaw 2026.5.7 sends only a final Telegram message instead of creating and editing a live preview.

## Best Possible Solution

Keep this issue open and restore Telegram partial preview streaming in the Telegram plugin or upstream partial-callback path while preserving the intentional first-preview debounce and adding focused regression coverage plus live-log proof.

## Reproduction Assessment

No live high-confidence reproduction was run in this review. Source inspection gives a medium-confidence reproduction path for first partials below Telegram's 30-character preview threshold, but the exact production report still needs the requested logs to confirm whether usable text deltas arrive before final delivery.

## Solution Assessment

No proposed threshold-only fix is not the best solution; maintainers closed that direction because it removes an intentional debounce. The better solution is a focused fix that makes partial previews appear when deltas are available without changing the push-notification debounce contract.
