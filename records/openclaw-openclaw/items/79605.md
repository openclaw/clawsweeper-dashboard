Keep open: current main still has the documented Telegram partial-preview path, no debounce-preserving fix has landed, and maintainer follow-up is explicitly waiting on runtime delta/send/edit logs to isolate the break point.

## What This Changes

The issue reports that Telegram `streaming.mode: "partial"` sends only one final Telegram message instead of creating a preview message and updating it with `editMessageText`.

## Best Possible Solution

Restore Telegram partial preview edits for long text-only direct-model replies while preserving the 30-character first-preview debounce, with focused regression coverage plus a redacted live Telegram DM smoke or equivalent runtime logs.

## Reproduction Assessment

No high-confidence live reproduction was run here. The issue has concrete 2026.5.7 steps, and current source shows the final-only path when no usable cumulative partial crosses the Telegram draft stream before finalization or before the 30-character preview gate.

## Solution Assessment

No patch is proposed here. Removing the partial-mode debounce is not the right fix because maintainer review rejected that direction; the narrower solution is to trace where usable direct-model text deltas stop reaching the Telegram draft stream while preserving the debounce.
