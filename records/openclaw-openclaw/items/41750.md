## Summary

Keep open: current main still refuses or fails when an untracked unhealthy loopback CDP listener occupies the managed browser port, and the related fix PR is closed unmerged. This remains a narrow browser-plugin recovery bug rather than stale cleanup.

## What This Changes

Request browser tool recovery for an unhealthy OpenClaw-managed Chrome process that keeps holding the managed CDP port.

## Best Possible Solution

Add browser-plugin-owned recovery that probes CDP health, verifies the listener is the expected OpenClaw-managed Chromium profile, terminates it gracefully with forced escalation if needed, waits for the port to clear, and preserves healthy reuse plus attachOnly/remote refusal behavior.
