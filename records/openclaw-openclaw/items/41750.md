## Repair Work Prompt

Fix the managed browser stale-CDP-port recovery for https://github.com/openclaw/openclaw/issues/41750. Current main still fails when the local managed Chrome CDP port is occupied by an unresponsive/untracked process: launchOpenClawChrome checks ensurePortAvailable before recovery, and ensureBrowserAvailable refuses an unhealthy untracked loopback CDP listener. Implement the fix inside the browser plugin boundary. Add a short CDP health probe and recover only when the process can be identified as OpenClaw-managed for that profile/user-data-dir or otherwise safely owned by OpenClaw. Do not broadly kill arbitrary port owners, do not use shell-interpolated process commands, and keep remote and attachOnly behavior unchanged. Add regression coverage for an occupied/unresponsive managed CDP port and for the non-OpenClaw/external-port refusal path.
