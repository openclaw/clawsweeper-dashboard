Fix the browser plugin bug where an unresponsive OpenClaw-managed Chrome process can keep the CDP port occupied and cause persistent browser tool timeouts. Work only in the browser plugin unless a tiny shared helper is clearly required. Add guarded stale managed Chrome recovery that probes CDP health, identifies the listener/process as Chromium owned by OpenClaw's expected user-data-dir/profile and CDP port, terminates gracefully with escalation, waits for the port to clear, then launches fresh. Preserve remote CDP, attachOnly, Browserless, Chrome MCP, and unrelated local port owner behavior. Do not use shell-interpolated process commands, do not kill generic port owners, and do not add a config option. Add focused regression tests for the untracked occupied/unresponsive CDP-port path and for refusing unrelated/external CDP listeners. Add a user-facing changelog fix entry if behavior changes ship.
