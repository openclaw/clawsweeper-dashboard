## Summary

Keep open: current main still exits on null or undefined unhandled-rejection reasons, and this PR is a focused plausible fix, but it still needs the required changelog entry plus maintainer acceptance of the global suppression policy versus a Slack-scoped handler.

## What This Changes

The PR adds a null/undefined unhandled-rejection guard in the global infra handler and adds fatal-detection regression tests for both values.

## Best Possible Solution

Land a policy-approved version with focused regression tests and a changelog entry: keep the core nullish guard only if maintainers accept the global policy, otherwise move the suppression into Slack-owned runtime startup through the existing handler seam.

## Reproduction Assessment

Yes. Source inspection on current main shows process unhandledRejection with undefined or null falls through to exitWithTerminalRestore("unhandled rejection"), and Slack socket-mode can emit the matching undefined disconnect reason when OpenClaw owns reconnects.

## Solution Assessment

Unclear. The proposed code is narrow and technically addresses the observed crash, but it changes global process policy; a Slack-scoped registered handler is the safer boundary if maintainers want owner-specific suppression.
