## Summary

Keep open: current main still has no Matrix pre-dispatch policy that accepts normal room traffic while skipping messages explicitly aimed at another configured Matrix account. The related participation PR remains open and broad, so it is not a merged resolution for this focused request.

## What This Changes

The issue requests an opt-in Matrix behavior that skips LLM dispatch when a room message explicitly mentions a different configured OpenClaw Matrix account.

## Best Possible Solution

Add a Matrix-plugin-owned, explicit opt-in sibling-mention skip before typing and LLM dispatch, with documented roster semantics and focused handler, mention resolver, config, docs, and regression coverage.

## Reproduction Assessment

Yes, at source level. A Matrix room message from a human with requireMention=false passes the current gate even when it explicitly mentions another configured Matrix account, because current main only computes whether this account was mentioned.

## Solution Assessment

Unclear. The requested early skip is plausible, but maintainers need to choose the Matrix config shape, sibling roster source, alias matching, and coordination with https://github.com/openclaw/openclaw/pull/73455 and https://github.com/openclaw/openclaw/issues/80432.
