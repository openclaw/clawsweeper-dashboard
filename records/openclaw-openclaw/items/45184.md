## Summary

Keep open: current main still lacks a Matrix sibling-mention skip before typing or LLM dispatch. The related Matrix participation PR is open and blocked, so it is useful context but not a merged or canonical resolution.

## What This Changes

Request an opt-in Matrix dispatch policy that skips non-mentioned sibling bot accounts when a room message explicitly mentions a different configured bot.

## Best Possible Solution

Add a focused Matrix-plugin-owned, explicit opt-in policy that detects mentions of configured sibling Matrix accounts and skips non-target agents before typing or LLM dispatch, with docs and handler tests; coordinate but do not depend on the broader participation PR unless that PR is narrowed and fixed.

## Reproduction Assessment

Yes, at source level. Current main only determines whether the current Matrix account was mentioned, and with `requireMention: false` there is no pre-dispatch branch that skips a message because it explicitly targets another configured bot.

## Solution Assessment

Unclear. The requested direction is plausible and has a Discord precedent, but maintainers should choose the Matrix config shape and sibling-mention detection contract before implementation.
