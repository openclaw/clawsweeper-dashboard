## Summary

Keep open: current main and v2026.6.10 still only decide whether the current Matrix account was mentioned; they do not provide an opt-in Matrix gate that skips ambient dispatch when a human room message explicitly mentions a different configured bot account.

## What This Changes

The issue requests an opt-in Matrix policy that skips non-mentioned accounts before dispatch when a room message explicitly mentions another configured Matrix bot account.

## Best Possible Solution

Add a Matrix-owned opt-in equivalent to ignoreOtherMentions for configured sibling accounts, with schema/docs/tests and conservative defaults, or explicitly document that Matrix ambient rooms should continue dispatching on other-target mentions.

## Reproduction Assessment

Yes, source-level: current Matrix code dispatches unconfigured human room messages when requireMention is false and only tracks whether this bot was mentioned. I did not run a live Matrix room, but the handler, resolver, schema, docs, and tests make the reported gap clear.

## Solution Assessment

Unclear: an opt-in Matrix-owned gate is plausible and has Discord precedent, but maintainers still need to choose the exact config name, target scope, and default behavior before implementation.
