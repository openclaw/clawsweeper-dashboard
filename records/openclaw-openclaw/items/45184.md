## Summary

Keep open: current main still only decides whether the receiving Matrix account was mentioned, so a room with `requireMention: false` can dispatch a human message that explicitly mentions a different configured Matrix account. The remaining work is a Matrix policy/config decision, not a stale cleanup candidate.

## What This Changes

Request an opt-in Matrix behavior that skips typing and LLM dispatch when a room message explicitly targets another configured bot account.

## Best Possible Solution

Add a Matrix-plugin-owned, explicit opt-in sibling-target skip before typing and LLM dispatch, with documented roster semantics and focused handler, resolver, config, docs, and regression coverage.

## Reproduction Assessment

Yes, at source level. A human Matrix room message with `requireMention: false` reaches dispatch even if `m.mentions` or visible text targets a different configured Matrix account, because current main only asks whether this account was mentioned.

## Solution Assessment

Unclear. The early skip is plausible, but maintainers still need to choose the config shape, sibling roster source, alias matching rules, and coordination with https://github.com/openclaw/openclaw/pull/73455 and https://github.com/openclaw/openclaw/issues/80432.
