## Summary

Keep open: current main still has no Matrix pre-dispatch policy that skips a room message because it explicitly targets a different configured Matrix account. The related participation PR remains open and broad, so it is useful context but not a merged or narrow resolution.

## What This Changes

Requested an opt-in Matrix policy to skip LLM dispatch for non-mentioned accounts when a room message explicitly mentions another configured bot account.

## Best Possible Solution

Add a focused Matrix-plugin-owned, explicit opt-in sibling-mention skip before typing/LLM dispatch, with documented sibling roster/alias semantics, handler tests, and coordination with the outbound sibling-mention issue.

## Reproduction Assessment

Yes, at source level. With a Matrix room configured requireMention:false, a human message that mentions another configured Matrix account but not the current account passes the current no-mention gate because the handler only computes whether this account was mentioned.

## Solution Assessment

Unclear. The requested early skip is plausible and has a Discord ignoreOtherMentions precedent, but maintainers should choose the Matrix config shape, sibling roster source, and collision behavior before implementation.
