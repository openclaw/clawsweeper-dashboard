## Summary

Keep open: current main and v2026.6.6 still lack a Matrix opt-in gate that skips pre-dispatch model work when a human message mentions a different configured Matrix account. The related broad participation PR at https://github.com/openclaw/openclaw/pull/73455 is open, conflicting, broad, and not a safe canonical replacement for this narrower request.

## What This Changes

The issue requests an opt-in Matrix dispatch policy that lets non-mentioned accounts ignore room messages explicitly mentioning a different configured Matrix account before typing or LLM dispatch.

## Best Possible Solution

Define a default-off Matrix account/room policy that identifies configured sibling-account mentions before typing/model dispatch, keeps existing ambient room behavior unchanged unless enabled, and ships with schema, docs, and focused handler tests.

## Reproduction Assessment

Yes, source-level. Current main resolves mentions only for the active Matrix account and skips unmentioned room traffic only when requireMention is true, so a requireMention:false room can still dispatch a human message that explicitly mentions a configured sibling account.

## Solution Assessment

Unclear until maintainers choose the product contract. A narrow Matrix pre-dispatch sibling-mention gate appears better scoped than the broad AI participation PR, but the exact opt-in flag or mode needs approval.
