## Summary

Keep open: current main and v2026.6.10 still lack a Matrix opt-in gate that skips pre-dispatch work when a human room message explicitly mentions a different configured Matrix account. The related broad participation PR is closed unmerged, while the open sibling-mention rendering and roster issues are complementary rather than replacements.

## What This Changes

The issue requests an opt-in Matrix dispatch gate that skips LLM dispatch when a room message explicitly mentions a different configured bot account.

## Best Possible Solution

Define a Matrix-owned, default-off account/room policy that detects configured sibling-account mentions before typing/model dispatch, with schema, docs, and focused handler tests.

## Reproduction Assessment

Yes, source-level. Current Matrix ingress resolves mention state for the active account and only drops no-mention room traffic when mention gating is required, so a requireMention=false room can still dispatch a human message that mentions a sibling account.

## Solution Assessment

Unclear until maintainers choose the product contract. A default-off pre-dispatch sibling-mention gate appears narrower than the closed broad AI participation PR, but the config name/mode and roster semantics need owner approval.
