## Summary

Keep open: current main and v2026.6.6 still lack the requested Matrix opt-in gate for skipping dispatch when a human message explicitly mentions a different configured Matrix account. The related broad participation PR is open, conflicting, and missing real behavior proof, so it is not a safe canonical replacement for this narrower behavior.

## What This Changes

The issue requests an opt-in Matrix dispatch rule that lets non-mentioned accounts ignore room messages explicitly targeting a different configured bot account before typing or model dispatch starts.

## Best Possible Solution

Have maintainers approve the Matrix opt-in semantics, then add a Matrix-owned pre-dispatch sibling-mention skip before typing/model dispatch with schema/types/docs and focused handler tests.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.6 shows a human room message with requireMention: false is not dropped merely because it explicitly mentions a configured sibling account; the existing mention check only asks whether the active account was mentioned.

## Solution Assessment

Unclear until maintainers choose the product contract. A narrow Matrix pre-dispatch sibling-mention skip appears better scoped than the broad AI participation PR, but the exact opt-in flag or mode needs approval.
