## Summary

Keep open: current main still makes the Matrix mention decision only for the receiving account, and Matrix has no opt-in policy to skip dispatch when a human message explicitly targets a different configured account. The related participation PR is still open/conflicting with review blockers, so it is not a landed implementation path.

## What This Changes

Request an opt-in Matrix policy that skips LLM dispatch for a non-mentioned Matrix account when a room message explicitly mentions a different configured bot account.

## Best Possible Solution

Add a Matrix-plugin-owned, explicit opt-in sibling-target skip before typing and LLM dispatch, with documented roster semantics and focused handler, mention resolver, config, docs, and regression coverage.

## Reproduction Assessment

Yes, at source level. With requireMention false, a human Matrix room message whose m.mentions or matrix.to pill targets another configured account is not dropped because current main only resolves mention state for selfUserId before dispatch.

## Solution Assessment

Unclear. An early opt-in skip is plausible, but maintainers still need to choose the config shape, sibling roster source, alias rules, and coordination with the related Matrix participation and outbound mention work.
