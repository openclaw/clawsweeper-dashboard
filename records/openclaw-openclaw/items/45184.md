## Summary

Keep open: current `main` still lacks a Matrix pre-dispatch option that accepts normal room traffic while skipping a message explicitly aimed at another configured Matrix bot account. The related participation PR is still open and broad, so it is not a merged resolution for this focused request.

## What This Changes

Request an opt-in Matrix policy such as exclusive mentions or `skipIfOtherBotMentioned` to avoid dispatching non-target bots in multi-account rooms.

## Best Possible Solution

Add a Matrix-plugin-owned, explicit opt-in sibling-mention skip before typing and LLM dispatch, with documented roster semantics and focused handler, mention resolver, config, and docs coverage.

## Reproduction Assessment

Yes, at source level. A Matrix room message from a human with `requireMention:false` passes the current gate even when it mentions another configured Matrix account, because the handler only computes whether the current account was mentioned.

## Solution Assessment

Unclear. The requested early skip is plausible, but maintainers need to choose the Matrix config shape, sibling roster source, alias matching, and interaction with the broader participation work before implementation.
