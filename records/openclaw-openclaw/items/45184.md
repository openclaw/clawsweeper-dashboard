## Summary

Keep open: current main still has no Matrix pre-dispatch policy that skips a human room message because it explicitly targets another configured Matrix bot account, and the only related implementation PR is still open, broad, and not a merged resolution.

## What This Changes

Request an opt-in Matrix policy that skips LLM dispatch for a bot when a room message explicitly mentions a different configured bot account.

## Best Possible Solution

Add a Matrix-plugin-owned, explicit opt-in sibling-mention skip before typing and LLM dispatch, with documented roster/alias semantics and focused handler, mention resolver, config, and docs coverage.

## Reproduction Assessment

Yes, at source level. With a Matrix room configured requireMention:false, a human message that mentions another configured Matrix account but not the current account passes current gating because the handler only computes whether this account was mentioned.

## Solution Assessment

Unclear. The requested early skip is plausible and has a Discord ignoreOtherMentions precedent, but maintainers should choose the Matrix config shape, sibling roster source, alias matching, and collision behavior before implementation.
