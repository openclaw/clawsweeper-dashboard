- `impact:message-loss`: The issue concerns Matrix messages being dispatched to the wrong account path or suppressed after unnecessary dispatch in multi-bot rooms.

## Summary

Keep open: current main still only checks whether the receiving Matrix account was mentioned, and there is no opt-in Matrix policy to skip dispatch when a human message explicitly targets a different configured account.

## What This Changes

Add an opt-in Matrix policy that skips typing and LLM dispatch for non-mentioned accounts when a room message explicitly mentions a different configured bot account.

## Best Possible Solution

Add a Matrix-plugin-owned, explicit opt-in sibling-target skip before typing and LLM dispatch, with documented roster semantics and focused handler, mention resolver, config, docs, and regression coverage.

## Reproduction Assessment

Yes, at source level. With requireMention false, a human Matrix room message whose m.mentions or matrix.to pill targets another configured account is not dropped because current main only resolves mention state for selfUserId before dispatch.

## Solution Assessment

Unclear. An early opt-in skip is plausible, but maintainers still need to choose the config shape, sibling roster source, alias rules, and coordination with the open Matrix participation and outbound mention work.
