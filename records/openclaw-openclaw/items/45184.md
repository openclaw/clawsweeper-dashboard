## Summary

Keep open. Current main still has Matrix account inventory and sender-scoped `allowBots` filtering, but no opt-in pre-dispatch gate for human room messages that visibly mention a different configured Matrix bot while the current account is unmentioned; the related broad participation PR is still open and not merged.

## What This Changes

Request an opt-in Matrix behavior that skips dispatch for non-mentioned accounts when a room message explicitly mentions a different configured bot account.

## Best Possible Solution

Add a backward-compatible Matrix account and per-room opt-in that runs before typing, media/history work, session recording, and LLM dispatch, skipping only when a visible explicit mention targets another configured Matrix bot and the current account is not mentioned.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a human Matrix room event with `requireMention: false`, `m.mentions.user_ids` naming another configured bot, and no mention of the current account bypasses the sender-only bot gate and continues to dispatch.

## Solution Assessment

Yes. A narrow Matrix plugin pre-dispatch opt-in is the maintainable fix for the typing/token problem; prompt-only suppression still burns tokens, and the related broad participation PR is not merged on current main.
