## Summary

Keep open: current `main` and `v2026.6.6` still lack a Matrix opt-in gate that skips pre-dispatch typing/model work when a human room message explicitly mentions a different configured Matrix account. The broad related participation PR was closed unmerged as an unmergeable landing base, so it does not supersede this narrower request.

## What This Changes

The issue asks for an opt-in Matrix dispatch policy that skips non-target bot accounts before typing/model work when a room message explicitly mentions another configured bot account.

## Best Possible Solution

Define a default-off Matrix account/room policy that identifies configured sibling-account mentions before typing/model dispatch, preserves current ambient room behavior unless enabled, and ships with schema, docs, and focused handler tests.

## Reproduction Assessment

Yes, source-level. Current Matrix handler resolves mentions for the active account only and only drops unmentioned room traffic when `shouldRequireMention` is true, so a `requireMention: false` room can dispatch a human message that mentions a sibling account.

## Solution Assessment

Unclear until maintainers choose the product contract. A narrow default-off pre-dispatch sibling-mention gate appears better scoped than the closed broad AI participation PR, but the flag/mode and interactions with outbound sibling mention work need owner approval.
