## Summary

Keep open. Current main still lacks an opt-in Matrix pre-dispatch gate for human room messages that explicitly mention a different configured Matrix bot; the existing Matrix `allowBots` policy only gates messages whose sender is another configured bot, and the related broad participation PR is still open rather than merged.

## What This Changes

The issue requests an opt-in Matrix mode that skips LLM dispatch for non-mentioned accounts when a room message explicitly mentions a different configured bot account.

## Best Possible Solution

Add a backward-compatible Matrix account and room opt-in that runs before typing, media/history work, session recording, and LLM dispatch, skipping only when a visible explicit mention targets another configured Matrix bot and the current account is not mentioned.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a human room event visibly mentioning another configured Matrix bot, with the current account unmentioned and `requireMention: false`, bypasses the existing sender-only bot gate and proceeds to dispatch.

## Solution Assessment

Yes. A narrow Matrix plugin pre-dispatch opt-in is the maintainable fix for the typing/token problem; prompt-only suppression or the broad open participation PR would still leave this issue unresolved on current main until a focused gate lands.
