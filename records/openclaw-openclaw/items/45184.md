## Summary

Keep open. Current main still has only sender-scoped Matrix bot filtering and self-only mention detection, so a human room message that explicitly mentions a sibling bot can still reach dispatch when this account has `requireMention: false`; the related broad participation PR is open and unmerged.

## What This Changes

The issue requests an opt-in Matrix pre-dispatch gate that skips unmentioned bot accounts when a room message explicitly mentions a different configured Matrix bot account.

## Best Possible Solution

Add a backward-compatible Matrix plugin opt-in, at account and per-room scope, that runs before typing/media/history/session dispatch and skips only when a visible explicit mention targets another configured Matrix bot while the current account is not mentioned.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a human room event with `requireMention: false`, `m.mentions.user_ids` naming another configured bot, and no mention of the current account bypasses the sender-only bot gate and continues to dispatch.

## Solution Assessment

Yes, a narrow opt-in Matrix pre-dispatch gate is the maintainable direction. Prompt-only suppression still burns tokens and shows typing, while the broader AI participation PR is not ready to serve as the fix.
