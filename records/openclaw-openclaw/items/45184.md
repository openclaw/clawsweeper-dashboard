## Summary

Keep open: current main still has receiver-scoped Matrix mention handling and no opt-in policy to skip a non-target account before typing/model dispatch when a human message explicitly mentions a different configured Matrix account. The related participation PR is open and broader than this request, so it is not a landed implementation.

## What This Changes

The issue requests an opt-in Matrix dispatch policy that skips non-mentioned bot accounts when a message explicitly mentions a different configured bot account.

## Best Possible Solution

Add a Matrix-plugin-owned opt-in sibling-target suppression gate before typing and reply dispatch, with documented roster semantics, config/schema support, and regression coverage coordinated with the broader Matrix participation and outbound mention work.

## Reproduction Assessment

Yes, at source level. With requireMention false, a room event whose visible Matrix mention targets another configured account is not dropped because current main only checks whether selfUserId was mentioned before continuing to reply dispatch.

## Solution Assessment

Unclear. An early opt-in skip is plausible, but maintainers still need to choose the config shape, sibling roster source, alias behavior, and how it should relate to the broader Matrix participation and outbound mention work.
