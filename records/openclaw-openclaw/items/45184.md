## Summary

Keep open. Current main and the latest shipped release still only gate Matrix room dispatch on whether the current bot was mentioned; they do not detect that a human message explicitly targeted a different configured Matrix account before typing/LLM dispatch.

## What This Changes

Request an opt-in Matrix dispatch policy that skips non-target bot accounts when a room message explicitly mentions a different configured account.

## Best Possible Solution

Define a Matrix-plugin-owned, explicit pre-dispatch policy for sibling-account mentions, with tests and docs that settle roster source, alias/ID matching, and collision behavior before any LLM/typing side effects occur.

## Reproduction Assessment

Yes, at source level. A human sender is not `isConfiguredBotSender`, `requireMention: false` bypasses the no-mention skip, and the handler then proceeds to typing setup and `dispatchReplyFromConfig`.

## Solution Assessment

No final fix shape is approved yet. A narrow explicit pre-dispatch gate may be the right implementation, but maintainers need to choose the Matrix participation/roster semantics and decide whether the broader related PR should be replaced or narrowed.
