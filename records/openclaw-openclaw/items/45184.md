## Summary

Keep open: current main and latest release still lack a Matrix opt-in gate that skips pre-dispatch typing/model work when a human room message explicitly mentions a different configured Matrix account. The related broad participation PR is closed unmerged, and the open outbound sibling-mention issue is complementary rather than a replacement.

## What This Changes

The issue requests an opt-in Matrix pre-dispatch gate that skips non-mentioned bot accounts when a message explicitly mentions a different configured bot account.

## Best Possible Solution

Define a Matrix-owned, default-off account/room policy that identifies configured sibling-account mentions before typing/model dispatch, preserves current ambient room behavior unless enabled, and ships with schema, docs, and focused handler tests.

## Reproduction Assessment

Yes, source-level. Current Matrix handler resolves mentions for the active account only and only drops unmentioned room traffic when requireMention is true, so a requireMention=false room can dispatch a human message that mentions a sibling account.

## Solution Assessment

Unclear until maintainers choose the product contract. A narrow default-off pre-dispatch sibling-mention gate appears better scoped than the closed broad AI participation PR, but the flag or mode and sibling roster semantics need owner approval.
