Keep open: current main and latest v2026.6.6 still lack the requested Matrix opt-in sibling-mention pre-dispatch skip, and the broader related PR at https://github.com/openclaw/openclaw/pull/73455 is open, conflicting, and missing real behavior proof rather than a safe canonical replacement.

## What This Changes

Request an opt-in Matrix policy to skip dispatch for non-mentioned accounts when a room message explicitly mentions a different configured Matrix bot.

## Best Possible Solution

Choose the Matrix-owned opt-in semantics, then implement a default-compatible pre-dispatch gate before typing/model dispatch with focused schema, docs, and handler tests.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: with requireMention=false, a human room message mentioning a sibling account avoids the configured-bot sender filters and the mention check only evaluates the current bot.

## Solution Assessment

Unclear. A narrow pre-dispatch skip is plausible, but maintainers first need to choose between an exclusive mention mode, a dedicated skip flag, or the broader participation policy in https://github.com/openclaw/openclaw/pull/73455.
