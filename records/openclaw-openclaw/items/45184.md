Keep open: current main and the latest release still lack the requested Matrix opt-in sibling-mention pre-dispatch skip. The linked broad Matrix participation PR is open, conflicting, and not a merged canonical fix.

## What This Changes

The issue requests an opt-in Matrix dispatch guard that skips LLM dispatch for non-mentioned sibling bot accounts when a room message explicitly mentions another configured OpenClaw account.

## Best Possible Solution

Add a Matrix-owned, default-compatible opt-in gate before typing or model dispatch that detects explicit mentions of configured sibling accounts, with schema/types/docs and focused handler tests for multi-account rooms.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: a human room message in a Matrix room with `requireMention: false` is not stopped when it explicitly targets another configured bot.

## Solution Assessment

Unclear until maintainers choose the product contract. A narrow pre-dispatch sibling-mention skip looks more maintainable for this issue than the broad AI participation PR, but the exact config semantics need approval.
