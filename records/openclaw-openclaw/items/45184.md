## Summary

Keep open: current main and v2026.6.6 still lack the requested Matrix opt-in gate that skips dispatch when a human message explicitly mentions a different configured Matrix account. The related broad Matrix participation PR is open, unmerged, and not a safe canonical fix for this narrower behavior.

## What This Changes

The issue requests an opt-in Matrix dispatch gate that skips LLM dispatch for non-mentioned bot accounts when a message explicitly mentions a different OpenClaw Matrix account.

## Best Possible Solution

Add a Matrix-owned, default-compatible opt-in gate before typing/model dispatch that detects explicit mentions of configured sibling accounts, with schema/types/docs and focused handler tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the handler only checks whether the active Matrix account was mentioned and skips only when `requireMention` is true, so the reported `requireMention: false` sibling-mention dispatch path remains reachable.

## Solution Assessment

Unclear until maintainers choose the product contract. A narrow pre-dispatch sibling-mention skip is a better fit for this issue than the broad AI participation PR, but the exact opt-in flag or mode needs approval.
