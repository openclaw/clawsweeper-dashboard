## Summary

Keep open. Current main still lacks an opt-in Matrix pre-dispatch gate that skips a non-mentioned account when a human room message explicitly mentions another configured Matrix bot account.

## What This Changes

Request a Matrix multi-account option that responds to general room traffic but skips LLM dispatch when a message explicitly targets a different configured bot account.

## Best Possible Solution

Add a backward-compatible Matrix account and room opt-in that runs before typing, media/history work, session recording, and LLM dispatch, skipping only when a visible explicit mention targets another configured Matrix bot and the current account is not mentioned.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a room event from a human sender whose visible Matrix mention or m.mentions target is another configured bot, while the current account has requireMention false and no self mention; current main has no pre-dispatch drop for that path.

## Solution Assessment

Yes. A narrow Matrix plugin pre-dispatch policy is the maintainable fix for the typing/token problem; prompt-only suppression or a late AI participation gate still spends model work before deciding not to answer.
