## Summary

Keep open: current main and the latest release still only check whether the receiving Matrix account was mentioned, so a room with `requireMention: false` can still dispatch a human message aimed at a different configured bot. This remains a valid source-level feature request, but it needs a Matrix policy/config decision rather than cleanup closure.

## What This Changes

Requested Matrix behavior: skip LLM dispatch for non-mentioned accounts when a human room message explicitly mentions another configured Matrix bot account.

## Best Possible Solution

Add a Matrix-plugin-owned, explicit opt-in sibling-target skip before typing and LLM dispatch, with documented roster semantics and focused handler, resolver, config, docs, and regression coverage.

## Reproduction Assessment

Yes, at source level. With `requireMention: false`, a human Matrix room message whose `m.mentions.user_ids` or visible pill targets another configured account is not dropped because current main only resolves mention state for `selfUserId` before dispatch.

## Solution Assessment

Unclear. The early skip is plausible, but maintainers still need to choose the config shape, sibling roster source, alias matching rules, and relationship to the open participation and outbound sibling-mention work.
