## Summary

Keep open: current main still lacks a Matrix policy that suppresses non-target accounts before typing/LLM dispatch when a message explicitly mentions another configured Matrix account. The source path remains receiver-scoped, and the related participation PR is open rather than a landed implementation.

## What This Changes

Request an opt-in Matrix multi-account dispatch policy that skips non-mentioned bot accounts when a room message explicitly targets a different configured account.

## Best Possible Solution

Add a Matrix-plugin-owned opt-in sibling-target suppression gate before typing and reply dispatch, with documented roster semantics and focused config, handler, mention resolver, docs, and regression coverage.

## Reproduction Assessment

Yes, at source level. With requireMention false, a human Matrix room event whose m.mentions or matrix.to pill targets another configured account is not dropped because current main only checks whether selfUserId was mentioned before dispatch.

## Solution Assessment

Unclear. An early opt-in skip is plausible, but maintainers still need to choose the config shape, sibling roster source, alias behavior, and how it should relate to the broader Matrix participation and outbound mention work.
