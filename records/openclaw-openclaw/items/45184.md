## Summary

Keep open: current main and latest release still lack the requested Matrix opt-in sibling-mention pre-dispatch skip, and the broad related Matrix participation PR is open, dirty, and not a merged canonical fix.

## What This Changes

The issue requests an opt-in Matrix guard so a bot that normally handles general room messages skips dispatch when the message explicitly mentions a different configured Matrix bot account.

## Best Possible Solution

Add a Matrix-owned, default-compatible opt-in gate before typing or model dispatch that detects explicit mentions of configured sibling accounts, with schema/types/docs and focused handler tests for multi-account rooms.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: a human room message in a `requireMention: false` Matrix room is not stopped by the current self-mention gate even when it explicitly targets another configured bot. I did not run a live Matrix room, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

Unclear until maintainers choose the product contract. A narrow pre-dispatch skip looks more maintainable for this issue than the broad AI participation PR, but the exact opt-in config shape needs maintainer approval.
