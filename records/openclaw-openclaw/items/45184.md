## Summary

Keep open: current main and v2026.6.6 still lack the requested Matrix opt-in gate for skipping dispatch when a human message explicitly mentions a different configured Matrix account. The related broad Matrix participation PR is open, unmerged, and too broad to supersede this narrower behavior.

## What This Changes

Requested behavior is an opt-in Matrix dispatch gate that lets non-mentioned bot accounts skip a message when the message explicitly mentions a different configured bot account.

## Best Possible Solution

Add a Matrix-owned, default-compatible opt-in gate before typing/model dispatch that detects explicit mentions of configured sibling accounts, with schema/types/docs and focused handler tests.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.6 shows a human room message with `requireMention: false` is not dropped merely because it explicitly mentions a configured sibling account; the existing mention check only asks whether the active account was mentioned.

## Solution Assessment

Unclear until maintainers choose the product contract. A narrow Matrix pre-dispatch sibling-mention skip appears better scoped than the broad AI participation PR, but the exact opt-in flag or mode needs approval.
