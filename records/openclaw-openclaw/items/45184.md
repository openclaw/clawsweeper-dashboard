## Summary

Keep open: current main still lacks the requested Matrix opt-in sibling-mention pre-dispatch skip; existing Matrix logic only checks mentions of the current account and filters configured bot senders, while the broader related Matrix participation PR is open and unmerged.

## What This Changes

Request an opt-in Matrix dispatch policy that lets non-mentioned accounts skip LLM dispatch when a room message explicitly mentions another configured Matrix bot.

## Best Possible Solution

Have maintainers choose the Matrix-owned opt-in policy shape, then land a default-compatible pre-dispatch gate before typing/LLM dispatch with schema, docs, and focused handler tests.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current Matrix handler checks only selfUserId for mentions, and requireMention=false prevents the existing no-mention skip from firing before dispatch.

## Solution Assessment

Unclear; the narrow pre-dispatch skip is plausible, but maintainers need to choose between a new skip knob, an exclusive mention mode, or the broader participation policy already proposed in #73455.
