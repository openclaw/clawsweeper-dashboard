## Summary

Keep open: current main and v2026.6.10 still lack an opt-in Matrix policy that skips dispatch when a human room message explicitly mentions another configured Matrix account, and the remaining choice is a Matrix product/config decision rather than cleanup closure.

## What This Changes

The issue requests an opt-in Matrix dispatch gate that skips pre-LLM handling when an incoming room message explicitly mentions another configured bot account.

## Best Possible Solution

Define a Matrix-owned, default-off account/room policy that detects configured sibling-account mentions before typing/model dispatch, with schema, docs, and focused handler tests.

## Reproduction Assessment

Yes, source-level. Current Matrix ingress resolves mention state for the active account and only drops no-mention room traffic when mention gating is required, so a requireMention=false room can still dispatch a human message that explicitly mentions a sibling account.

## Solution Assessment

Unclear until maintainers choose the product contract. A default-off pre-dispatch sibling-mention gate appears narrower than the closed broad AI participation PR, but the config name, roster semantics, and outbound coordination need owner approval.
