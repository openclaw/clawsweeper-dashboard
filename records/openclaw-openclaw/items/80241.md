## Summary

Keep open: current main still has the source-backed memory indexing bug, but this PR is not mergeable as written because the new batch-size matcher can route quota/provider failures into skip-file handling and the real endpoint proof gate is still unmet.

## What This Changes

This PR adds adaptive splitting for inline memory embedding batches, skip-file handling for irreducible embedding size failures, SDK export coverage, tests, docs, and a changelog entry.

## Best Possible Solution

Land a narrowed memory-core fix that adaptively splits only concrete input or batch-size failures, preserves quota/provider failures as command failures, and includes real low-limit endpoint proof.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main can batch unknown OpenAI-compatible chunks up to the generic 8000 budget while the reported endpoint rejects combined batches around 2000. I did not run the live Qianfan endpoint in this read-only pass.

## Solution Assessment

No, not as currently written: adaptive splitting belongs in memory-core, but the matcher must avoid treating quota/rate/provider failures as irreducible input-size failures. The PR also needs real after-fix endpoint proof before merge.
