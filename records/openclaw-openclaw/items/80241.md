## Summary

Keep open: this PR is the canonical open implementation path for the linked memory indexing bug, but the current patch still has a blocking classifier bug and lacks after-fix real behavior proof from a low-limit embedding endpoint.

## What This Changes

This PR adds adaptive inline memory embedding batch splitting, per-file skip handling for irreducible embedding-size failures, tests, docs, SDK export coverage, and a changelog entry for low-limit OpenAI-compatible embedding providers.

## Best Possible Solution

Land a memory-core fix that adaptively splits concrete per-input or per-batch size failures, preserves quota/rate/provider failures as command failures, and includes real low-limit endpoint proof.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main can batch unknown OpenAI-compatible chunks to the generic 8000/8192 budget while the reported endpoint rejects combined batches around 2000; I did not run the live Qianfan endpoint.

## Solution Assessment

No, not as currently written. Adaptive splitting belongs in memory-core, but the matcher must not route quota/provider failures into skip-file handling, and the PR still needs real after-fix endpoint proof.
