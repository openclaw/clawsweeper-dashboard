## Summary

Keep open: the linked compaction bug is valid enough to review, but this PR is not safe to merge because it promotes plugin `llm.allowModelOverride` into host authority without carrying `llm.allowedModels`, which bypasses the current plugin allowlist path.

## What This Changes

The PR reads context-engine plugin `llm.allowModelOverride` and `allowAgentIdOverride` from config and passes them as runtime LLM authority flags instead of hardcoded false.

## Best Possible Solution

Keep plugin LLM policy and `allowedModels` as the source of truth, then fix the startup/config-lifecycle path with regression coverage and real LCM or gateway proof before merge.

## Reproduction Assessment

Yes for the PR-introduced defect by source inspection: the patch sets authority `allowModelOverride` from plugin config but omits `allowedModels`, while current runtime code picks authority policy before plugin policy. I did not reproduce the full LCM compaction failure in a live gateway.

## Solution Assessment

No. The current implementation already has a normalized plugin-policy path with allowlist coverage, so the safer fix is to repair the reported startup/config freshness issue without bypassing that policy.
