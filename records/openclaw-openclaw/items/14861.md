## Summary

Current main and v2026.6.6 still lack the requested redacted Gateway boot outcome summary; the only direct implementation PR was closed unmerged as dirty and too broad, so this issue remains the canonical maintainer decision point.

## What This Changes

Request a Gateway startup summary that reports which sidecars/hooks started, skipped, or failed, especially Gmail watcher missing-account and missing-gog cases.

## Best Possible Solution

Land a fresh, narrow Gateway-only redacted startup outcome collector after sidecars settle, preserving current readiness semantics and closing this issue only after the maintainer-approved implementation ships or reaches main.

## Reproduction Assessment

Yes. Current-main source shows that with hooks enabled but hooks.gmail.account absent, Gateway does not schedule the Gmail watcher sidecar, and the lifecycle wrapper would suppress that skip reason even if called.

## Solution Assessment

Yes as a direction, but it needs maintainer scope approval. The maintainable path is a narrow fixed-vocabulary, redacted Gateway outcome collector, not reviving the closed broad/conflicting branch.
