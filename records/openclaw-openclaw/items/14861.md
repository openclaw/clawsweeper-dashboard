## Summary

Keep open. Current main still lacks a structured Gateway boot outcome summary, and the concrete Gmail watcher missing-account case remains silent during normal startup.

## What This Changes

Request a Gateway boot summary that reports started, skipped, and failed outcomes for sidecars and hooks such as Gmail watcher, channels, and internal hooks.

## Best Possible Solution

Add a redacted startup outcome collector around the existing Gateway sidecar flow and emit one concise boot summary after sidecars settle.

## Reproduction Assessment

Yes. A source-level reproduction is clear: with hooks.enabled=true and no hooks.gmail.account, normal startup bypasses the Gmail lifecycle wrapper, and even the wrapper suppresses that reason when called.

## Solution Assessment

Yes. The narrow maintainable fix is outcome collection and logging around the existing startup paths, not changing hook/channel startup behavior or making optional skips fatal.
