## Summary

Keep open. Current main and the latest release still lack a structured Gateway boot outcome summary, the Gmail watcher missing-account path remains silent during normal startup, and the only linked implementation PR was closed unmerged with a note that the underlying issue is still valid.

## What This Changes

The issue requests a Gateway startup summary that reports started, skipped, and failed subsystem outcomes, with Gmail watcher skip reasons surfaced instead of silently discarded.

## Best Possible Solution

Land a fresh, narrow Gateway-only startup outcome collector with fixed redacted vocabulary after maintainer approval of scope and wording, while preserving current readiness and optional-sidecar failure semantics.

## Reproduction Assessment

Yes. Source inspection shows that with hooks enabled but hooks.gmail.account absent, Gateway never schedules the Gmail watcher and the lifecycle wrapper suppresses that exact reason; I did not run a live gateway in this read-only pass.

## Solution Assessment

Unclear until maintainers approve the operator-facing scope. The best technical shape appears to be a narrow redacted Gateway startup outcome collector, but the prior PR was closed unmerged because branch scope and maintainer pickup were not clean.
