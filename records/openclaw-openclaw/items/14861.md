## Summary

Keep open. Current main and the latest release still do not emit the requested normal Gateway boot outcome summary; the Gmail watcher missing-account case is still silent in normal startup, and the only PR that attempted the fix was closed unmerged and conflicting.

## What This Changes

The issue requests a Gateway startup summary that reports started, skipped, and failed outcomes for subsystems/hooks, especially Gmail watcher startup skips.

## Best Possible Solution

Keep this issue open until maintainers approve and land a fresh narrow Gateway startup outcome collector that reports redacted subsystem status without changing readiness or sidecar failure semantics.

## Reproduction Assessment

Yes. Source inspection shows that with hooks enabled but hooks.gmail.account absent, Gateway never schedules the Gmail watcher and the lifecycle wrapper suppresses that exact reason; I did not run a live gateway in this read-only review.

## Solution Assessment

Yes directionally, but not via the closed unmerged branch. The maintainable path is a fresh, narrow Gateway-only implementation after maintainers confirm the summary scope, fixed vocabulary, and redaction contract.
