## Summary

Keep open. Current main still lacks the requested structured Gateway boot outcome summary: it has compact startup logs, point warnings, startup timing traces, and plugin-init failure warnings, but no started/skipped/failed collector for Gateway sidecars or hooks; the Gmail watcher missing-account path is still silent during normal startup.

## What This Changes

The issue requests Gateway startup observability that reports which subsystems and hooks started, skipped, or failed, with Gmail watcher startup as the concrete silent-skip example.

## Best Possible Solution

Add a redacted startup outcome collector around the existing Gateway sidecar flow and emit one concise post-sidecar summary while preserving current readiness and failure behavior.

## Reproduction Assessment

Yes, at source level. With hooks enabled but no hooks.gmail.account, startGatewaySidecars never calls the Gmail watcher lifecycle wrapper; even when the wrapper runs, it suppresses the no-account skipped reason.

## Solution Assessment

Yes. The narrow maintainable path is to collect outcomes where each sidecar already starts and log them once after sidecars settle; making optional skips fatal or only restoring formatted startup output would not solve this observability gap.
