## Summary

Keep open: current main still lacks an operator-facing started/skipped/failed Gateway boot outcome summary. The Gmail watcher example remains source-reproducible: the normal startup path skips the watcher when no account is configured, and the lifecycle wrapper still suppresses the no-account reason when it is called.

## What This Changes

The issue requests Gateway startup observability that reports which subsystems, hooks, and sidecars started, skipped, or failed, with Gmail watcher as the concrete silent-skip example.

## Best Possible Solution

Add a redacted Gateway startup outcome collector around existing sidecar and hook start points, then emit one concise post-sidecar summary while preserving current readiness and failure behavior.

## Reproduction Assessment

Yes, at source level. With hooks enabled but no hooks.gmail.account, current startup skips the Gmail watcher path, and the lifecycle wrapper still suppresses the no-account reason when it is called.

## Solution Assessment

Yes. The narrow maintainable fix is to collect redacted outcomes where subsystems already start or skip and log one summary after sidecars settle; making optional skips fatal or relying on trace-only diagnostics would not solve the operator visibility gap.
