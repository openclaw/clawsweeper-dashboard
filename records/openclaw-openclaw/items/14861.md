## Summary

Keep open. Current main still has point logs and a plugin-failure summary, not a structured Gateway boot outcome summary; the concrete Gmail watcher missing-account case remains silent, and #73187 is an overlapping formatting request rather than a superseding outcome collector.

## What This Changes

Request a Gateway startup observability summary that reports started, skipped, and failed subsystem or hook outcomes, including Gmail watcher non-start reasons.

## Best Possible Solution

Add a redacted startup outcome collector around the existing sidecar flow and emit one concise post-sidecar boot summary covering hooks, Gmail watcher, channels, plugin services, memory, recovery sidecars, and gateway_start hooks.

## Reproduction Assessment

Yes. Source inspection on current main shows normal startup bypasses Gmail watcher lifecycle unless hooks.enabled and hooks.gmail.account are both present; even when the wrapper runs, it suppresses the missing-account reason.

## Solution Assessment

Yes. The narrow maintainable fix is to collect outcomes where each sidecar already starts and log them once after sidecars settle; making optional skips fatal or only restoring formatted startup output would not solve this observability gap.
