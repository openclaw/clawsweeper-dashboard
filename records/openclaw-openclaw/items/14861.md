## Summary

Keep open. Current main still lacks a structured Gateway boot outcome summary: startup has point logs and plugin failure warnings, but no started/skipped/failed collector for sidecars or hooks, and the Gmail watcher missing-account path is still unreported during normal startup.

## What This Changes

The issue asks for Gateway startup observability that reports which subsystems and hooks started, skipped, or failed during boot.

## Best Possible Solution

Add a redacted startup outcome collector around the existing Gateway sidecar flow and emit one concise post-sidecar summary covering Gmail watcher, internal hooks, channels, plugin services, memory scheduling, recovery sidecars, and gateway_start hooks.

## Reproduction Assessment

Yes, at source level. Current main bypasses the Gmail watcher lifecycle unless hooks.enabled and hooks.gmail.account are both present, and the wrapper suppresses the missing-account skipped reason even when it runs.

## Solution Assessment

Yes. The narrow maintainable fix is to collect outcomes where each sidecar already starts and log them once after sidecars settle; making optional skips fatal or only restoring formatted startup output would not solve this observability gap.
