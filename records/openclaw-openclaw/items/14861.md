## Summary

Keep open: current main and the latest shipped release still lack a Gateway boot outcome summary, and the concrete Gmail watcher no-account startup path remains silent.

## What This Changes

The issue requests a Gateway boot summary that reports started, skipped, and failed subsystem outcomes, especially for Gmail watcher and hook startup failures.

## Best Possible Solution

Add a redacted Gateway startup outcome collector around existing sidecar and hook start points, then emit one concise post-sidecar summary while preserving readiness and failure behavior.

## Reproduction Assessment

Yes, at source level. With hooks enabled but no hooks.gmail.account, current startup skips the Gmail watcher sidecar path, and the lifecycle wrapper suppresses the same no-account reason if called directly.

## Solution Assessment

Yes for the requested direction. The maintainable solution is to collect redacted started/skipped/failed outcomes where subsystems already start or skip, rather than making optional skips fatal or relying on trace-only diagnostics.
