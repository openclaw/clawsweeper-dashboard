## Summary

Keep open. Current main still lacks a structured Gateway boot outcome summary, and the Gmail missing-account path remains silent during normal startup; #73187 overlaps startup log presentation but does not supersede this outcome-reporting request.

## What This Changes

Request adds Gateway startup observability that reports which sidecars/hooks initialized, skipped, or failed, including Gmail watcher non-start reasons.

## Best Possible Solution

Add a redacted startup outcome collector around the existing Gateway sidecar flow and emit one concise boot summary after sidecars settle while preserving readiness and failure behavior.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: with hooks.enabled=true and no hooks.gmail.account, normal startup bypasses the Gmail lifecycle wrapper, and the wrapper still suppresses that reason if called.

## Solution Assessment

Yes. The narrow maintainable fix is outcome collection and logging around existing startup paths, not making optional skips fatal or folding this into the broader formatted-log request.

## Repair Work Prompt

Fix the Gateway startup observability gap tracked by https://github.com/openclaw/openclaw/issues/14861. Add a redacted startup outcome collector around the existing post-attach sidecar flow so the Gateway emits one concise boot summary after sidecars settle, including started/skipped/failed outcomes for Gmail watcher, internal hooks, channels, plugin services, memory startup, recovery sidecars, and gateway_start hook dispatch where practical. Preserve existing non-fatal startup semantics and do not make optional skips fatal. Do not attempt the broader #73187 formatted-log restoration or add a new config option unless maintainers request it. Add/update focused tests for Gmail missing-account/no-gog reporting and boot-summary output, and add a CHANGELOG.md entry because this changes user-facing logs.
