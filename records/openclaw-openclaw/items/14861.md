## Summary

Keep open: current main still lacks an operator-facing Gateway boot outcome summary, and the concrete Gmail watcher startup skip remains source-reproducible. The related formatted-startup-output issue is adjacent, but it does not supersede this request for per-subsystem started/skipped/failed observability.

## What This Changes

The issue requests a redacted Gateway startup summary that reports subsystem and hook startup outcomes, especially Gmail watcher skipped or failed reasons.

## Best Possible Solution

Add a redacted Gateway startup outcome collector around existing sidecar and hook start points, then emit one concise post-sidecar summary while preserving readiness and failure behavior.

## Reproduction Assessment

Yes, at source level. With hooks enabled but no hooks.gmail.account, current startup skips the Gmail watcher path, and the lifecycle wrapper still suppresses the no-account reason when it is called directly.

## Solution Assessment

Yes. The maintainable fix is to collect redacted started/skipped/failed outcomes where subsystems already start or skip and log one summary after sidecars settle, rather than making optional skips fatal or relying on trace-only diagnostics.
