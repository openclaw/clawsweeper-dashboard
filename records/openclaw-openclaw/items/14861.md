## Label Justifications

- `P2`: The observability gap makes production startup failures difficult to diagnose but does not itself make the core Gateway unusable.
- `impact:message-loss`: A silently unstarted Gmail ingestion watcher can leave inbound messages unprocessed without an operator-visible startup signal.

## Summary

Current main and v2026.6.5 still lack a Gateway subsystem outcome summary, and the open closing implementation at https://github.com/openclaw/openclaw/pull/86710 has not merged and currently conflicts with main.

## What This Changes

Add a concise Gateway boot summary that reports started, skipped, and failed subsystem outcomes, including Gmail watcher non-start reasons.

## Best Possible Solution

Land a current-main-compatible version of the linked PR with a concise redacted outcome collector after sidecars settle, preserving readiness and asynchronous failure semantics.

## Reproduction Assessment

Yes, at source level: current Gateway startup bypasses the Gmail watcher lifecycle when the account is absent and emits no aggregate outcome for that skipped state.

## Solution Assessment

Yes in direction: a redacted collector at the existing sidecars-ready boundary is the narrowest maintainable solution, although maintainers must approve the exact reported subsystem set.
