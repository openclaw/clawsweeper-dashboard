## Summary

Keep open: current main still lacks the requested structured Gateway boot outcome summary, and the concrete Gmail watcher no-account path remains silent during normal startup.

## What This Changes

The issue requests a redacted Gateway startup summary that reports started, skipped, and failed outcomes for subsystems such as Gmail watcher, hooks, channels, memory, cron, and plugin services.

## Best Possible Solution

Add a redacted Gateway startup outcome collector around existing sidecar and hook start points, then emit one concise post-sidecar summary while preserving readiness and failure behavior.

## Reproduction Assessment

Yes, at source level. With hooks enabled but no hooks.gmail.account, current startup skips the Gmail watcher sidecar path, and the lifecycle wrapper still suppresses the no-account reason when called directly.

## Solution Assessment

Yes for the direction. The maintainable fix is to collect redacted started/skipped/failed outcomes where subsystems already start or skip and log one summary after sidecars settle, rather than making optional skips fatal or relying on trace-only diagnostics.
