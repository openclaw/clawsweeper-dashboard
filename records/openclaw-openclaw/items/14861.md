## Summary

Keep open: current main still lacks a normal Gateway boot outcome summary, and the Gmail watcher skip path remains source-reproducible. Existing startup tracing, compact startup logs, and plugin-init warnings are useful but do not provide per-subsystem started/skipped/failed observability.

## What This Changes

Request adding a redacted Gateway boot outcome summary that reports subsystem and hook startup success, skip, and failure states, especially Gmail watcher non-start reasons.

## Best Possible Solution

Add a redacted Gateway startup outcome collector around the existing sidecar and hook start points, then emit one concise post-sidecar summary while preserving readiness and failure behavior.

## Reproduction Assessment

Yes, at source level. With hooks enabled but no hooks.gmail.account, current startup skips the Gmail watcher path, and the lifecycle wrapper still suppresses the no-account reason when called directly.

## Solution Assessment

Yes for the direction. The maintainable fix is to collect redacted started/skipped/failed outcomes where subsystems already start or skip and log one summary after sidecars settle, rather than making optional skips fatal or relying on trace-only diagnostics.
