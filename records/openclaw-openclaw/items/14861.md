## Summary

Keep open. Current main still lacks the requested structured Gateway boot outcome summary: startup has point logs, readiness gating, startup trace timings, and plugin failure warnings, but no started/skipped/failed collector for sidecars or hooks; the Gmail watcher missing-account path is still silent during normal startup.

## What This Changes

Add Gateway startup observability that reports started, skipped, and failed outcomes for sidecars and hooks such as the Gmail watcher.

## Best Possible Solution

Add a redacted startup outcome collector around the existing Gateway sidecar flow and emit one concise post-sidecar summary while preserving current readiness and failure behavior.

## Reproduction Assessment

Yes, at source level. With hooks enabled but no hooks.gmail.account, startGatewaySidecars never calls the Gmail watcher lifecycle wrapper; even when the wrapper runs, it suppresses the no-account skipped reason.

## Solution Assessment

Yes. The narrow maintainable solution is to collect outcomes where each sidecar already starts and log them once after sidecars settle; making optional skips fatal or only restoring formatted startup output would not solve this observability gap.
