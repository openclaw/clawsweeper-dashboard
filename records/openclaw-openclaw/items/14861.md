## Summary

Keep open: current main and v2026.5.20 still lack a Gateway boot outcome summary, and the concrete Gmail watcher no-account startup path remains silent during normal startup. The related open Gmail watcher PR only changes warning wording and does not implement the requested subsystem outcome collector.

## What This Changes

Request a Gateway startup summary that reports which subsystems and hooks started, skipped, or failed, especially Gmail watcher prerequisites.

## Best Possible Solution

Add a redacted Gateway startup outcome collector around existing sidecar and hook start points, then emit one concise post-sidecar summary while preserving current readiness and failure behavior.

## Reproduction Assessment

Yes, at source level. With hooks enabled but no hooks.gmail.account, current startup skips the Gmail watcher sidecar path, and the lifecycle wrapper suppresses the same no-account reason if called directly.

## Solution Assessment

Yes for the requested direction. The maintainable solution is to collect redacted started/skipped/failed outcomes where subsystems already start or skip, rather than making optional skips fatal or relying on trace-only diagnostics.
