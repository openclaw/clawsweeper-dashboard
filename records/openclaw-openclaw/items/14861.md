## Summary

Keep open. Current main still lacks a structured Gateway boot outcome summary: startup has compact point logs and plugin/channel warnings, but no started/skipped/failed collector, and the Gmail watcher missing-account path remains unreported during normal startup.

## What This Changes

The issue requests a Gateway boot summary that reports subsystem, hook, and sidecar startup outcomes instead of silently dropping skipped or failed startup reasons.

## Best Possible Solution

Add a redacted startup outcome collector around the existing Gateway sidecar flow and emit one concise post-sidecar summary covering Gmail watcher, internal hooks, channels, plugin services, memory scheduling, recovery sidecars, and gateway_start hooks.

## Reproduction Assessment

Yes. Source inspection on current main shows normal startup bypasses Gmail watcher lifecycle unless hooks.enabled and hooks.gmail.account are both present; even when the wrapper runs, it suppresses the missing-account skipped reason.

## Solution Assessment

Yes. The narrow maintainable fix is to collect outcomes where each sidecar already starts and log them once after sidecars settle; making optional skips fatal or only restoring formatted startup output would not solve this observability gap.
