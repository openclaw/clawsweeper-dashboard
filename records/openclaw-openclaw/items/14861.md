## Summary

Keep open. Current main still lacks the requested structured Gateway boot outcome summary: startup has point logs and plugin-init warnings, but no started/skipped/failed collector for sidecars/hooks, and the Gmail watcher missing-account path remains unreported.

## What This Changes

Request structured Gateway startup logging that reports subsystem started/skipped/failed outcomes, especially Gmail watcher hook skip reasons.

## Best Possible Solution

Add a redacted startup outcome collector around the existing sidecar flow and emit one concise post-sidecar boot summary covering Gmail watcher, internal hooks, channels, plugin services, memory, recovery sidecars, and gateway_start hooks.

## Reproduction Assessment

Yes. Source inspection on current main shows normal startup bypasses Gmail watcher lifecycle unless hooks.enabled and hooks.gmail.account are both present; even when the wrapper runs, it suppresses the missing-account reason.

## Solution Assessment

Yes. The narrow maintainable fix is to collect outcomes where each sidecar already starts and log them once after sidecars settle; making optional skips fatal or only restoring formatted startup output would not solve this observability gap.
