## Summary

Keep open. Current main still lacks the requested structured Gateway boot outcome summary: startup has compact logs, timing traces, point warnings, and plugin-init failure warnings, but no started/skipped/failed collector for sidecars or hooks; the Gmail watcher missing-account path remains silent during normal startup.

## What This Changes

Request a redacted Gateway boot summary that reports started, skipped, and failed outcomes for subsystems and hooks, especially Gmail watcher non-start reasons.

## Best Possible Solution

Add a redacted startup outcome collector around existing Gateway sidecar start points and emit one concise post-sidecar summary while preserving current readiness and failure behavior.

## Reproduction Assessment

Yes, at source level. With hooks enabled but no hooks.gmail.account, startGatewaySidecars never calls the Gmail watcher lifecycle wrapper; even when the wrapper runs, it suppresses the no-account skipped reason.

## Solution Assessment

Yes. The narrow maintainable path is to collect outcomes where each sidecar already starts and log them once after sidecars settle; making optional skips fatal or only restoring formatted startup output would not solve this observability gap.
