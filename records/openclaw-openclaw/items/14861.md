## Summary

Keep open. Current main still lacks the requested structured Gateway boot outcome summary, and the concrete Gmail watcher missing-account path remains silent during normal startup; related #73187 overlaps startup log UX but does not supersede this specific outcome-reporting request.

## What This Changes

The issue requests a Gateway boot summary that reports subsystem startup outcomes, especially Gmail watcher skipped or failed startup reasons.

## Best Possible Solution

Add a redacted startup outcome collector around the existing Gateway sidecar flow and emit one concise boot summary after sidecars settle while preserving non-fatal startup semantics.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: with hooks.enabled=true and no hooks.gmail.account, normal startup bypasses the Gmail lifecycle wrapper, and the wrapper still suppresses that reason when called.

## Solution Assessment

Yes. The narrow maintainable fix is outcome collection and logging around existing startup paths, not changing hook/channel startup behavior or making optional skips fatal.
