## Summary

Keep open. Current main at f3145f6 still lacks a structured Gateway boot outcome summary, and the concrete Gmail watcher case remains silent when hooks are enabled but no Gmail account is configured because normal startup never calls the lifecycle wrapper in that state.

## What This Changes

Request a Gateway startup summary that reports initialized, skipped, and failed subsystems or hooks, including gmail-watcher and other sidecars, after Gateway boot or restart.

## Best Possible Solution

Add a structured, redacted startup outcome collector around the existing sidecar flow and emit one concise normal boot summary after sidecars settle, without changing optional subsystem startup semantics or making skipped hooks fatal.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: with hooks.enabled=true and no hooks.gmail.account, startGatewaySidecars bypasses the Gmail lifecycle wrapper, and the wrapper currently suppresses the missing-account reason when invoked.

## Solution Assessment

Yes. The narrow maintainable direction is startup outcome collection and logging around the existing Gateway sidecar paths, not changing hook/channel startup behavior or treating optional skips as failures.
