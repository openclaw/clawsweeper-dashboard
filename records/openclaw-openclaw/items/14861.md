## Summary

Keep open: current main still lacks a normal Gateway boot outcome summary, and the concrete Gmail watcher no-account path is still silent because startup bypasses the watcher lifecycle before it can report a skip reason. Related open work overlaps diagnostics, but it does not close or fully cover this issue.

## What This Changes

The issue asks for Gateway boot-time observability that reports started, skipped, and failed subsystem outcomes such as the Gmail watcher and hooks.

## Best Possible Solution

Add a redacted Gateway startup outcome collector around the existing sidecar and hook start points, then emit one concise post-sidecar summary while preserving readiness and failure behavior.

## Reproduction Assessment

Yes, at source level. With hooks enabled but no hooks.gmail.account, current startup skips the Gmail watcher sidecar path, and the lifecycle wrapper suppresses the no-account reason when called directly.

## Solution Assessment

Yes for the direction. The maintainable fix is to collect redacted started/skipped/failed outcomes where subsystems already start or skip and log one summary after sidecars settle, rather than making optional skips fatal or relying on trace-only diagnostics.
