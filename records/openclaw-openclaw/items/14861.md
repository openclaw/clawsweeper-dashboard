## Summary

Keep open: current main and v2026.5.20 still lack a normal Gateway boot outcome summary, and the Gmail watcher missing-account path remains silent because startup only schedules the watcher when an account is already configured.

## What This Changes

Add Gateway startup observability that reports started, skipped, and failed subsystem or hook outcomes, including Gmail watcher skip reasons.

## Best Possible Solution

Add a redacted Gateway startup outcome collector around existing sidecar and hook start points, then emit one concise post-sidecar summary while preserving readiness and failure behavior.

## Reproduction Assessment

Yes, at source level. With hooks enabled but no hooks.gmail.account, current startup skips the Gmail watcher sidecar path, and the lifecycle wrapper suppresses the no-account reason when called directly.

## Solution Assessment

Yes for the direction. The maintainable fix is to collect redacted started/skipped/failed outcomes where subsystems already start or skip and log one summary after sidecars settle, rather than making optional skips fatal or relying on trace-only diagnostics.
