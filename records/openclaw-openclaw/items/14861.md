## Summary

Current main and the latest release still lack the requested Gateway boot outcome summary, and the Gmail watcher missing-account path remains silent during normal startup. The only direct implementation PR was closed unmerged, so this issue is still necessary.

## What This Changes

Request a Gateway startup summary that reports which subsystems and hooks started, skipped, or failed, including Gmail watcher skip reasons.

## Best Possible Solution

Add a bounded, redacted Gateway startup outcome collector that reports started/skipped/failed outcomes for sidecars and hooks, including Gmail watcher non-start reasons, without changing readiness semantics.

## Reproduction Assessment

Yes. Source inspection shows that with hooks enabled but hooks.gmail.account absent, Gateway never schedules the Gmail watcher and the lifecycle wrapper also suppresses that exact reason. I did not run the Gateway because this review is read-only.

## Solution Assessment

Yes, a redacted Gateway startup outcome collector is the cleanest maintainable direction; a Gmail-only warning would leave the broader subsystem observability problem unsolved. The open question is the product scope of the summary.
