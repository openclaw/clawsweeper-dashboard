## Summary

Keep open: current main and v2026.5.22 still lack a redacted Gateway boot outcome summary, and the Gmail watcher missing-account path remains silent during normal startup. The related open Gmail watcher PR only changes warning wording and unrelated docs, so it does not own this subsystem outcome collector work.

## What This Changes

The issue requests Gateway startup observability that reports which subsystems started, skipped, or failed, especially Gmail watcher prerequisites like missing account or missing gog.

## Best Possible Solution

Add a redacted Gateway startup outcome collector around existing sidecar and hook start points, then emit one concise post-sidecar summary while preserving current readiness and failure behavior.

## Reproduction Assessment

Yes, at source level. With hooks enabled but no hooks.gmail.account, current startup skips the Gmail watcher sidecar path, while the watcher/lifecycle code still contains and can suppress the reported non-start reason.

## Solution Assessment

Yes for the requested direction. The narrow maintainable fix is to collect redacted started/skipped/failed outcomes where subsystems already start or skip, rather than making optional skips fatal or relying on trace-only diagnostics.
