## Summary

Keep open. Current main and v2026.6.10 still lack the requested structured Gateway boot outcome summary, and the Gmail watcher missing-account path remains silent during normal startup; the direct implementation PR is closed unmerged.

## What This Changes

The issue requests a Gateway startup summary that reports which subsystems, hooks, and sidecars started, skipped, or failed, especially the Gmail watcher missing-account case.

## Best Possible Solution

Land a fresh narrow Gateway-only fixed-vocabulary, redacted startup outcome collector after sidecars settle, with maintainer-approved subsystem scope and no readiness behavior change.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with hooks enabled but hooks.gmail.account absent, startup does not schedule the Gmail watcher, and the lifecycle helper would suppress that missing-account reason even if called.

## Solution Assessment

Yes in direction, but the exact subsystem scope and wording need maintainer agreement. The maintainable solution is a fixed-vocabulary redacted Gateway startup outcome collector that preserves readiness behavior.
