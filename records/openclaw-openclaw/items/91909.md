## What This Changes

The issue requests fixing an inbound follow-up dispatch race where messages queued after run completion remain undrained until watchdog recovery.

## Best Possible Solution

Land a narrow queue-owned fix that makes active-run follow-up enqueues kick an idle drain, with regression coverage and redacted live or scripted proof that immediate same-lane follow-ups no longer wait for watchdog recovery.

Yes for the central race: using the existing enqueue-owned idle restart path is the narrowest maintainable fix. The repair should still check the stale `active_reply_work` recovery subcase from the WhatsApp follow-up so the fix does not leave a second wedge path uncovered.
