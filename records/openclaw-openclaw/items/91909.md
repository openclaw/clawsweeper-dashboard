## Summary

Keep open. Current main and latest release still contain the reported follow-up enqueue shape: the active-run branch enqueues with idle restart disabled and only schedules a drain when one immediate liveness recheck says the previous run is no longer active, so the reported terminal-window race remains plausible and unowned by a narrower merged fix.

## What This Changes

The issue reports a timing-sensitive inbound follow-up dispatch race where a same-thread message can be enqueued after run completion without starting a new drain, leaving the agent silent until stuck-session recovery.

## Best Possible Solution

Make the generic follow-up queue and active-run lifecycle guarantee that queued work cannot remain idle after active-run teardown, with a regression test for the terminal-window case rather than a Slack-specific workaround.
