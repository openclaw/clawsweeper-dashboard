## Summary

The issue requests a cross-cutting security model for persistent provenance/taint tracking, content-derived execution denial, exec risk classification, and provenance-aware secret and outbound-action guardrails.

## Best Possible Solution

Keep this as the security architecture umbrella until maintainers either land a coordinated provenance/taint contract or explicitly split the work into canonical implementation issues covering propagation, risk classification, and enforcement end to end.
