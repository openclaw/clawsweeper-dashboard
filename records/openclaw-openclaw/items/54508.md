## Summary

Keep open. Current main has important prompt-injection mitigations, including external-content wrapping, immediate web_fetch metadata, inter-session message provenance, plugin hook blocking, exec approvals, and outbound hooks, but it still does not provide the requested persistent taint propagation or provenance/risk-aware enforcement across execution, secrets, persistence, and outbound delivery. The related open issues are useful slices, not a merged or canonical superseding fix for this umbrella.

## What This Changes

The issue requests a core security model for persistent external-content tainting, default denial of content-derived execution, exec risk classes, and provenance-aware guardrails for secrets and outbound actions.

## Best Possible Solution

Keep this as the core security architecture umbrella unless maintainers explicitly split it into canonical implementation issues. The end state should be structured provenance/taint metadata for external inputs and tool results, propagation through summaries and persistence, stable action/risk/provenance fields in hook and approval payloads, and core enforcement that prevents untrusted content from authorizing execution, secret access, or outbound actions without guarded review.
