## Summary

Keep open: this is useful ACPX auth-boundary hardening, but the PR is not merge-ready because Gemini auth selector env vars still inherit into the child harness and configured `gemini --acp ...` commands are forwarded with a duplicate `--acp`.

## What This Changes

The PR adds generated ACPX wrappers for built-in Claude and Gemini commands that strip selected provider API-key env vars from child process environments, updates wrapper discovery/reaping, and adds focused tests.

## Best Possible Solution

Merge the narrow wrapper hardening after covering the linked Gemini strip surface, preserving configured-command argv correctly, and leaving doctor/config opt-in policy to the linked follow-up issue.

## Reproduction Assessment

No live runtime reproduction was run in this read-only review. Source inspection gives a high-confidence path: launch the PR-generated Gemini wrapper with `GOOGLE_GENAI_USE_GCA` or `GOOGLE_GENAI_USE_VERTEXAI` set and observe those vars still pass through; configured `gemini --acp --model ...` also source-reproduces duplicate `--acp`.

## Solution Assessment

No, not yet. The generated-wrapper approach is the right narrow owner boundary, but the Gemini metadata must match the upstream auth selector surface and the wrapper must avoid altering configured Gemini argv.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Include Gemini auth selectors in the child env scrub:** `extensions/acpx/src/codex-auth-bridge.ts:42`
  - body: The new Gemini metadata deletes only `GEMINI_API_KEY` and `GOOGLE_API_KEY`, but Gemini CLI also selects auth from `GOOGLE_GENAI_USE_GCA` and `GOOGLE_GENAI_USE_VERTEXAI` before API-key auth. A mixed-credential gateway can still steer the ACP child into the wrong auth lane, so extend the strip list and tests to cover the linked Gemini auth surface.
  - confidence: 0.88
- **[P3] Avoid duplicating --acp for configured Gemini commands:** `extensions/acpx/src/codex-auth-bridge.ts:543`
  - body: When config already contains `gemini --acp --model ...`, this wraps `parts.slice(1)` in a Gemini wrapper whose default args already include `--acp`, so the child sees `gemini --acp --acp --model ...`. Drop the default ACP arg from forwarded configured args or make the wrapper avoid prepending it in that path.
  - confidence: 0.78
