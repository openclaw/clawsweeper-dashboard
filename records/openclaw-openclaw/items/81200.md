## Summary

Keep open. The supplied terminal proof is sufficient for the env-stripping behavior, but the PR introduces a blocking Gemini compatibility regression: acpx currently rewrites `gemini --acp` to `--experimental-acp` for Gemini CLI versions below 0.33.0, and wrapping Gemini as `node gemini-acp-wrapper.mjs` bypasses that existing normalization while the wrapper always launches `gemini --acp`.

## What This Changes

The PR adds ACPX wrapper metadata and docs so built-in Claude and Gemini child harnesses strip selected provider auth environment variables while preserving the parent process env.

## Best Possible Solution

Keep the wrapper-level credential hardening, but preserve acpx's existing Gemini version/flag normalization before landing; broader auth-contract and doctor/preflight work can remain with https://github.com/openclaw/openclaw/issues/69260.

## Reproduction Assessment

Yes, source-level: acpx 0.7.0 only performs the old-Gemini `--experimental-acp` rewrite when the spawned command basename is `gemini`, while this PR makes acpx spawn a Node wrapper that always invokes `gemini --acp`. A focused test with a fake pre-0.33.0 Gemini binary would reproduce the argument regression without live credentials.

## Solution Assessment

No, not as written. The env-stripping wrapper is the right narrow boundary, but it must carry forward the existing acpx Gemini flag compatibility or use an acpx seam that still applies it.
