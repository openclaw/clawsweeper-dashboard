## Summary

Keep open: the ACPX boundary hardening is useful and belongs in the plugin, but the PR currently duplicates the ACP flag for explicitly configured Gemini commands. Maintainers also need to accept the compatibility tradeoff from stripping provider auth env by default.

## Best Possible Solution

Keep the harness-local env hardening, fix Gemini argument normalization before merge, and have maintainers explicitly accept the auth-env compatibility change as the new default.
