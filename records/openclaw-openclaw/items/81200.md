## Summary

Keep open: the credential-stripping direction is useful and the updated proof is sufficient, but the PR currently regresses acpx's existing Gemini launch compatibility by hiding the real `gemini` command behind a Node wrapper.

## What This Changes

The PR adds ACPX-generated Claude and Gemini wrappers that strip selected provider auth environment variables from child harness processes and updates related cleanup/runtime tests.

## Best Possible Solution

Keep the child-only env scrubbing, but make the Gemini wrapper preserve acpx's version-aware `--acp`/`--experimental-acp` behavior before merge.

## Reproduction Assessment

Yes for the PR regression by source inspection: acpx 0.7.0 only applies the Gemini legacy flag fallback when it sees the spawned command basename as `gemini`, and this PR changes the visible command to the generated Node wrapper.

## Solution Assessment

No for the current diff: the env-stripping wrapper is the right narrow hardening direction, but it must preserve or reimplement acpx's Gemini launch normalization to avoid breaking older Gemini installs.
