## Summary

Keep open: this PR is a focused implementation for the linked ACPX ambient provider-env issue, and I did not find a blocking correctness or supply-chain finding in the proposed patch. It still needs maintainer review/landing rather than cleanup closure because it changes auth-boundary policy and introduces an explicit Gemini preserve flag.

## What This Changes

The PR adds generated ACPX wrappers for built-in Claude and Gemini harnesses that strip selected provider auth env vars from child processes, updates wrapper cleanup/runtime detection, docs, live Docker script env handling, and regression tests.

## Best Possible Solution

Land the focused child-env stripping change if maintainers accept the auth-boundary policy, and keep broader auth-contract and doctor/preflight work tracked under https://github.com/openclaw/openclaw/issues/69260.

## Reproduction Assessment

Yes for the narrow child-env exposure: current main source shows ACPX wrappers passing process.env into child harnesses and no Gemini wrapper, while upstream Gemini uses the reported env vars for auth selection. The durable Gemini auth drift and real Claude billing behavior were not live-reproduced here.

## Solution Assessment

Yes for the focused first slice: wrapper-local child-env scrubbing is narrow, keeps the parent environment unchanged, and avoids broad gateway env stripping. Broader auth-contract, preflight, and doctor UX should remain a separate maintainer decision.
