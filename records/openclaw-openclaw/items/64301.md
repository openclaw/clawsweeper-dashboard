## Summary

Keep this PR open, but it is not merge-ready: the reported current-main miss is source-reproducible, while the proposed reverse lookup still allows tail-only exact remapping that can execute the wrong allowlisted tool and the PR has no after-fix real provider proof.

## What This Changes

The PR adds reverse lookup to embedded-runner tool-call normalization for underscore-stripped, dot-prefixed, dropped-leading-character, and hex-suffixed malformed tool names.

## Best Possible Solution

Land a narrower embedded-runner normalization fix that recovers strict prefixed or hash-suffixed allowlisted names, fails closed for tail-only and ambiguous matches, and includes focused tests plus redacted provider proof.

## Reproduction Assessment

Yes, source-level: current main leaves functionsread5f3787893b0d4c49a as read5f3787893b0d4c49a after prefix stripping and cannot map it to allowlisted read. I did not run a live affected-provider reproduction.

## Solution Assessment

No. Prefix/hash recovery is a reasonable direction, but the current PR should remove or structurally gate dropped-leading-character exact recovery and add regression tests before merge.
