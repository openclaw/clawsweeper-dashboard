## Summary

Keep open: current main still has the malformed hex-name gap, but this PR is not merge-ready because dropped-leading-character exact recovery can rewrite an unrelated unknown tool to an allowlisted tool, and the author has not supplied real-run proof.

## What This Changes

The PR modifies `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts` to reverse-match malformed streamed tool names against allowed names using underscore-stripped, dot-prefixed, dropped-leading-character, and digit-bearing hex-suffix variants.

## Best Possible Solution

Land a narrower embedded-runner normalizer that recovers strict prefixed/hash malformed names, fails closed for tail-only or ambiguous cases, and includes focused regression tests plus redacted real-provider proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `functionsread5f3787893b0d4c49a` with `read` allowlisted strips to `read5f3787893b0d4c49a` and remains unresolved on current main.

## Solution Assessment

No. Strict prefix plus hash-suffix recovery is a reasonable direction, but dropped-leading-character exact remapping should be removed or gated on independent structural evidence before this dispatch-sensitive path lands.
