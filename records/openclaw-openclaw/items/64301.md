## Summary

Keep open: current main still misses the reported hex-suffixed malformed tool-name recovery, but this PR should not merge as-is because tail-only exact recovery can dispatch the wrong allowlisted tool and the PR still lacks real-provider proof.

## What This Changes

The PR adds a reverse-lookup fallback in embedded-runner tool-call normalization for underscore-stripped, dot-prefixed, dropped-leading-character, and digit-bearing hex-suffixed malformed tool names.

## Best Possible Solution

Land a narrow embedded-runner normalizer that recovers strict prefixed/hash malformed names, fails closed for tail-only or ambiguous cases, and includes focused regression tests plus redacted real-provider proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: functionsread5f3787893b0d4c49a with read allowlisted strips to read5f3787893b0d4c49a and remains unresolved on current main.

## Solution Assessment

No. Strict prefix plus hash-suffix recovery is a reasonable direction, but tail-only exact remapping should be removed or gated on independent structural evidence before this dispatch-sensitive path lands.
