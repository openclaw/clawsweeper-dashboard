## Summary

Keep open: current main still lacks the useful reverse lookup for the reported malformed tool names, but this PR remains blocked by missing real-provider proof and by a tail-only recovery path that can coerce an unknown tool name into an allowlisted tool.

## What This Changes

The PR adds reverse matching in embedded-runner tool-call normalization for underscore-stripped, dot-prefixed, dropped-leading-character, and digit-bearing hex-suffixed malformed tool names.

## Best Possible Solution

Land a narrower normalizer that recovers strict prefixed/hash malformed names, fails closed for ambiguous or tail-only cases, and includes focused regression tests plus redacted real-provider proof.

## Reproduction Assessment

Yes, via source inspection rather than a live provider run. On current main, a `functionsread5f3787893b0d4c49a` tool name with `read` allowlisted strips only to `read5f3787893b0d4c49a` and remains unresolved.

## Solution Assessment

No. Strict prefix plus hash-suffix recovery is a reasonable direction, but tail-only exact recovery is too broad for dispatch-sensitive code and should be removed or gated on independent structural evidence.
