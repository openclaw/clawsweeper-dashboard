## Summary

Keep open. Current main and the latest release still treat `streamTo` as ACP-only: native subagent calls ignore the field, tests and docs lock that behavior, and implementing native parent progress requires maintainer product and security decisions.

## What This Changes

The issue requests support for `streamTo="parent"` when `sessions_spawn` uses `runtime="subagent"`.

## Best Possible Solution

Keep this as the native subagent parent-stream design tracker and implement it only after maintainers choose event semantics, verbosity inheritance, delivery, truncation, redaction, and trust policy.

## Reproduction Assessment

Not applicable as a strict bug reproduction, but yes for current behavior by source inspection: native subagent calls ignore `streamTo`, tests assert that contract, and docs mark the field ACP-only.

## Solution Assessment

No, removing ACP-only gating or adding an unconditional heartbeat is not the best maintainable fix. The safer path is an explicit native progress contract with redaction, truncation, verbosity, delivery, and event semantics decided before implementation.
