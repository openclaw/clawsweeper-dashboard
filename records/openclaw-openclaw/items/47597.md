## Summary

Keep open. Current main fixes the related hard-error path by ignoring ACP-only fields on native subagent spawns, but the requested native `runtime="subagent"` parent streaming/progress relay is still absent and documented as ACP-only.

## What This Changes

Add `streamTo="parent"` support for `runtime="subagent"` so native subagent runs can stream progress back to the parent session.

## Best Possible Solution

Define a native subagent progress contract, then add a typed parent-stream option to the subagent spawn path with owned relay cleanup, timeout handling, docs, changelog, and focused coverage for `mode="run"` and `mode="session"`.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main accepts `runtime:"subagent"` with `streamTo:"parent"`, strips it before native dispatch, and has no native parent relay path.

## Solution Assessment

Unclear. The current silent-drop behavior is the right compatibility fix for provider-injected ACP-only fields, but the requested feature needs a maintainer-chosen event source and visibility model before a heartbeat or full relay can be called the best fix.
