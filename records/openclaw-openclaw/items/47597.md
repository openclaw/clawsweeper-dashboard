## Summary

Keep open. Current main and v2026.5.20 still keep `streamTo` ACP-only for `sessions_spawn`; the earlier hard-error path is now ignored for native subagents, but native parent-visible progress streaming remains unimplemented and needs product/security decisions.

## What This Changes

Request to let `sessions_spawn({ runtime: "subagent", streamTo: "parent" })` send real-time native subagent progress to the parent session.

## Best Possible Solution

Keep this as the native subagent parent-stream design tracker and implement only after maintainers choose event shape, redaction/truncation, verbosity inheritance, delivery semantics, and how it relates to yielded-wait UI state.

## Reproduction Assessment

Not applicable as a strict bug reproduction, but source inspection is high-confidence: current main ignores `streamTo` for native subagent calls, tests assert that contract, and docs mark the field ACP-only.

## Solution Assessment

No, the suggested remove-the-guard or unconditional heartbeat approach is not enough by itself. The safer maintainable path is an explicit native progress contract with redaction, truncation, verbosity, delivery, and event semantics decided before implementation.
