## Summary

Keep open. Current main and the latest shipped tag still treat `streamTo` as ACP-only for `sessions_spawn`; native subagent calls ignore it, and adding parent-visible native progress needs product and security decisions around event shape, redaction, verbosity, and delivery.

## What This Changes

Request support for `sessions_spawn({ runtime: "subagent", streamTo: "parent" })` so native subagent progress can stream to the parent session.

## Best Possible Solution

Keep this as the native subagent parent-stream design tracker and implement only after maintainers choose event semantics, delivery, verbosity inheritance, truncation, redaction, and trust policy.

## Reproduction Assessment

Not applicable as a strict bug reproduction, but source inspection is high-confidence: current main ignores `streamTo` for native subagent calls, tests assert that contract, and docs mark the field ACP-only.

## Solution Assessment

No, the suggested remove-the-guard or unconditional heartbeat approach is not the best maintainable fix. The safer path is an explicit native progress contract with redaction, truncation, verbosity, delivery, and event semantics decided before implementation.
