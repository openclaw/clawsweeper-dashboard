## Summary

Keep open. Current main intentionally treats `streamTo` as ACP-only for `sessions_spawn`: native subagent calls accept the field for compatibility but ignore it, tests and docs lock that behavior, and the remaining native parent-progress relay needs maintainer API, verbosity, and redaction decisions.

## What This Changes

The issue requests `streamTo="parent"` support for `runtime="subagent"` so parent sessions can receive live native subagent progress instead of waiting silently.

## Best Possible Solution

Keep this as the native subagent streaming/API design tracker, handle the narrower yielded-parent wait indicator separately, and implement native `streamTo` only after maintainers choose event semantics, verbosity, truncation, and redaction policy.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence in current behavior: native subagent calls ignore `streamTo`, tests assert that contract, and docs describe the field as ACP-only.

## Solution Assessment

No, simply lifting the ACP-only gate or adding an unbounded heartbeat is not the best fix. The maintainable path needs an explicit native progress contract with redaction, truncation, verbosity, and event semantics before implementation.
