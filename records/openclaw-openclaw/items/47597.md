## Summary

Keep open. Current main tolerates provider-injected `streamTo` by treating it as ACP-only, but native subagent spawns still have no parent progress relay or streaming contract.

## What This Changes

Request adding `streamTo:"parent"` support to native `runtime:"subagent"` `sessions_spawn` calls so parent sessions receive live progress before completion.

## Best Possible Solution

Define a native subagent parent-progress contract, then implement one owned relay path with docs, changelog, and coverage for run/session modes, cleanup, timeout, no-output, and nested delivery behavior.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a broken existing bug, but yes as a static feature gap: current source and tests show `streamTo` is ignored for native subagents and no relay path exists.

## Solution Assessment

Is this the best way to solve the issue? Unclear. The current silent-ignore behavior is the right compatibility fix for provider-injected ACP-only fields, but adding support needs a maintainer-chosen event source and visibility model rather than a simple guard removal.
