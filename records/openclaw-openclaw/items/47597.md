## Summary

Keep open. Current main and latest release v2026.5.22 still keep `streamTo` ACP-only for `sessions_spawn`; the earlier hard-error path is fixed by ignoring ACP-only fields for native subagents, but native parent-visible progress streaming remains unimplemented and needs product/security decisions.

## What This Changes

Request to add parent-visible live progress streaming for `sessions_spawn` with `runtime="subagent"` via `streamTo="parent"`.

## Best Possible Solution

Keep this as the native subagent parent-stream design tracker and implement only after maintainers choose event shape, redaction/truncation, verbosity inheritance, delivery semantics, and its relationship to yielded-wait UI state.

## Reproduction Assessment

Yes, at source level. Current main describes `streamTo` as ACP-only, drops it for native subagent execution, and tests assert that `spawnSubagentDirect` receives no `streamTo` property.

## Solution Assessment

No, the suggested remove-the-guard or unconditional heartbeat approach is not enough by itself. The safer maintainable path is an explicit native progress contract with redaction, truncation, verbosity, delivery, and event semantics decided before implementation.
