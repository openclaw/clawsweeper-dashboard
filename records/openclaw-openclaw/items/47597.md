## Summary

Keep open. Current main covers the provider-injected `streamTo` crash by treating the field as ACP-only for subagent spawns, but native subagent spawns still have no parent progress relay or streaming contract.

## What This Changes

The issue requests allowing `sessions_spawn` with `runtime:"subagent"` and `streamTo:"parent"` to send real-time progress updates to the parent session.

## Best Possible Solution

Define a native subagent parent-progress contract first, then implement one owned relay path with docs, changelog, and coverage for run/session modes, cleanup, timeout, and nested delivery behavior.

## Reproduction Assessment

Yes, as a source-level feature gap rather than a broken existing contract. Current main ignores `streamTo` for subagent spawns, tests that it is not forwarded, and documents it as ACP-only.

## Solution Assessment

Unclear. The desired UX is valid, but a simple guard removal or heartbeat needs maintainer approval for event source, visibility level, cleanup, timeout, and isolation semantics.
