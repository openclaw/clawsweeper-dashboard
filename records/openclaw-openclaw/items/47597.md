## Summary

Keep open. Current `main` tolerates `streamTo="parent"` on native subagent spawns by ignoring it, but it still does not implement the requested live parent-session progress stream for `runtime="subagent"`; the closed related progress-callback issue also points back here as the canonical tracker.

## What This Changes

Request support for `streamTo="parent"` on native `runtime="subagent"` `sessions_spawn` calls so parent sessions can see child progress before completion.

## Best Possible Solution

Keep this as the canonical core feature tracker and design a bounded, redacted native-subagent parent progress surface that reuses prepared Gateway/run facts and has focused tests/docs for the chosen visibility contract.

## Reproduction Assessment

Yes for the current behavior boundary: source and tests show `streamTo` is ACP-only and ignored for native subagent spawns. I did not run a live Gateway flow because this is a read-only review and the remaining request is a feature rather than broken documented behavior.

## Solution Assessment

Unclear until maintainers choose the progress and redaction contract. The maintainable path is not to blindly forward child output, but to define bounded parent-visible summaries or events first, then wire that into the native subagent path.
