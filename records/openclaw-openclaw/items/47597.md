## Summary

Keep open. Current `main` and the latest release tolerate `streamTo="parent"` on native subagent spawns by ignoring it, so the old error/crash symptom is improved, but the requested live parent-session progress relay for `runtime="subagent"` is still not implemented; the closed related report points back to this issue as the canonical tracker.

## What This Changes

The issue requests allowing `sessions_spawn` with `runtime="subagent"` to use `streamTo="parent"` for live parent-session progress updates.

## Best Possible Solution

Keep this as the canonical core feature tracker and design a bounded, redacted native-subagent parent progress surface that reuses prepared Gateway/run facts instead of treating ACP and subagent progress as unrelated ad hoc paths.

## Reproduction Assessment

Not a bug reproduction, but yes for the current behavior boundary: source and tests show `streamTo` is ACP-only and ignored for subagent spawns. I did not run a live Gateway flow because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the progress contract. A maintainable fix should define redaction, visibility, and event semantics before wiring native subagent events or heartbeats into parent-session system events.
