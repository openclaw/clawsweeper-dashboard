## Summary

Keep open. Current main fixed the related hard-error tolerance path, but it still explicitly treats `streamTo` as ACP-only and does not provide native subagent parent streaming. The remaining work needs a maintainer runtime/product decision rather than conservative cleanup.

## What This Changes

Requested support for `streamTo="parent"` on `runtime="subagent"` so native subagents can report progress to the parent session in real time.

## Best Possible Solution

Keep the shipped ACP-field tolerance, then design and implement a native subagent progress surface in core with explicit event source and visibility rules, update the `sessions_spawn` schema/docs, and cover both `mode="run"` and `mode="session"` with focused tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: current tests accept `runtime="subagent"` plus `streamTo:"parent"` but assert the field is stripped, and source shows native subagents start through gateway `agent` without a parent relay.

## Solution Assessment

Unclear. The current silent-drop behavior is the right compatibility fix for provider-injected ACP-only fields, but it is not the requested feature; the heartbeat idea may be viable, yet maintainers should choose the event source and visibility model first.
