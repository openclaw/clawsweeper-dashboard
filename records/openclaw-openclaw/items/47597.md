## Summary

Keep open: current main and v2026.6.10 still treat `streamTo` as ACP-only and omit it from native `spawnSubagentDirect`, so the requested native parent progress stream remains real work that needs maintainer product and security design.

## What This Changes

The issue requests `streamTo="parent"` support for `sessions_spawn` with `runtime="subagent"` so parent sessions can see live child progress instead of only final completion.

## Best Possible Solution

Keep this as the canonical native subagent parent-stream design tracker, then implement a bounded progress contract with conservative redaction, parent routing, cleanup, and tests after maintainer approval.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.10 only materialize `streamTo` for ACP, tests assert native subagent calls ignore it, and docs tell users to omit it for native spawns. I did not run a live long-running subagent session in this read-only review.

## Solution Assessment

No implementation path is approved yet. The best fix is not simply passing `streamTo` into `spawnSubagentDirect`; it needs a native progress event contract with conservative visibility, routing, cleanup, and security boundaries.
