## Summary

Keep open. Current main intentionally treats `streamTo` as ACP-only: native subagent calls accept the field for compatibility but ignore it, and the remaining native parent-progress relay needs an API, visibility, and redaction decision.

## What This Changes

The issue requests adding `streamTo="parent"` support for native `runtime="subagent"` spawns so parent sessions can receive live child progress instead of only completion results.

## Best Possible Solution

Keep this as the native subagent streaming/API design tracker, land the narrower yielded-parent wait indicator separately, and implement native `streamTo` only after maintainers choose redaction, truncation, verbosity, and event semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence in current behavior: native subagent calls ignore `streamTo`, tests assert that contract, and docs describe the field as ACP-only.

## Solution Assessment

No, simply lifting the ACP-only gate or adding an unbounded heartbeat is not the best fix. The maintainable path needs an explicit native progress contract with redaction, truncation, and visibility policy before implementation.
