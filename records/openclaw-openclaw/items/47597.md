## Summary

Keep open: current main and v2026.6.10 still treat `streamTo` as ACP-only, and the native `runtime="subagent"` path has no parent progress relay. This remains the canonical tracker for native subagent parent streaming, with security/product design needed before implementation.

## What This Changes

Request to allow `sessions_spawn` with `runtime: "subagent"` to use `streamTo: "parent"` for parent-session progress visibility.

## Best Possible Solution

Keep this as the canonical native subagent parent-stream design tracker and implement only after maintainers approve the visibility, redaction, routing, cleanup, and test contract.

## Reproduction Assessment

Yes, at source level: current main only materializes `streamTo` for ACP, tests assert native/default subagent calls drop it, and docs tell users to omit it for native spawns. I did not run a live long-running subagent session in this read-only review.

## Solution Assessment

No implementation path is approved yet. The best fix is not simply passing `streamTo` into `spawnSubagentDirect`; it needs a native progress event contract with conservative visibility, routing, cleanup, and security boundaries.
