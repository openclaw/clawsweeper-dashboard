## Summary

Keep open: current main mitigates the hard failure by ignoring ACP-only `streamTo` for native subagents, but it still does not implement the requested native subagent parent progress stream. Source, docs, tests, and related duplicate cleanup all point to this issue as the canonical design tracker.

## What This Changes

Request native `sessions_spawn` subagents to support `streamTo: "parent"` so parent sessions receive progress before final completion.

## Best Possible Solution

Design a maintainer-approved native subagent progress relay or status-event path that derives parent routing from trusted session/run metadata, redacts and bounds child output, preserves current completion delivery, and documents/tests both ACP-only and native behavior.

## Reproduction Assessment

Yes, at source level. Current main only passes `streamTo` to ACP, tests assert native subagent calls drop it, and docs describe native subagents as completion-only with `sessions_yield`.

## Solution Assessment

No final implementation shape is approved yet. Simply removing the runtime gate or adding heartbeat text would be too narrow without a maintainer-approved security/redaction/routing contract for native subagent progress.
