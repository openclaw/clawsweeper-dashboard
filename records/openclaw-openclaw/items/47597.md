## Summary

Keep open. Current main intentionally treats `streamTo` as ACP-only for `sessions_spawn`; native subagent calls accept the field for compatibility but ignore it, and the remaining native parent-progress stream needs maintainer API, verbosity, truncation, and redaction decisions.

## What This Changes

Request to support `streamTo="parent"` for native `runtime="subagent"` spawns so parent sessions can see real-time child progress.

## Best Possible Solution

Keep this as the native subagent streaming/API design tracker and implement only after maintainers choose event semantics, verbosity inheritance, truncation, and redaction policy.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence in current behavior: native subagent calls ignore `streamTo`, tests assert that contract, and docs describe the field as ACP-only.

## Solution Assessment

No, simply lifting the ACP-only gate or adding a heartbeat is not the best maintainable fix. The safer path is an explicit native progress contract with redaction, truncation, verbosity, delivery, and event semantics decided before implementation.
