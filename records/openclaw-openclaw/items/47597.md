## Summary

Keep open. Current main fixed the hard failure from ACP-only `streamTo` being present on native subagent calls, but it still intentionally treats native subagent `streamTo` as ignored and has no parent progress relay for `runtime="subagent"`; the remaining request is a product/API feature that needs visibility and redaction policy.

## What This Changes

Request native `sessions_spawn` subagent runs to support `streamTo: "parent"` progress updates instead of ACP-only streaming.

## Best Possible Solution

Keep this as the canonical native subagent streaming/API design item, land the narrower parent wait indicator through https://github.com/openclaw/openclaw/issues/58067 separately, and only add native `streamTo` after maintainers choose the visibility and redaction semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence in the current behavior: native subagent calls ignore `streamTo`, tests assert that contract, and docs describe the field as ACP-only.

## Solution Assessment

No, simply lifting the ACP-only gate is not yet the best fix. The maintainable path needs an explicit native progress contract, redacted summaries, and a decision about whether parent visibility follows verbose/reasoning settings or a new API policy.
