## Summary

Keep open. Current main only tolerates accidental `streamTo` on native subagent spawns; it does not implement parent streaming, and adding that would be a new security-sensitive runtime/API behavior.

## What This Changes

Request `sessions_spawn({ runtime: "subagent", streamTo: "parent" })` to stream native subagent progress to the parent session.

## Best Possible Solution

Design a native subagent progress path that derives parent context server-side, emits bounded/redacted progress summaries, and documents how it interacts with `sessions_yield` and completion announce behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main ignores `streamTo` for native subagent spawns and docs still describe it as ACP-only.

## Solution Assessment

Unclear until maintainers choose the native progress and visibility policy. The hard-failure workaround is already on main, but full `streamTo` support should be designed separately from the narrower UI wait indicator in https://github.com/openclaw/openclaw/issues/58067.
