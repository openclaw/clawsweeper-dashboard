## Summary

Keep open: current main and the latest release still make `streamTo` ACP-only and intentionally ignore it for native subagent spawns, so the requested native parent progress stream remains missing and needs maintainer product/security design.

## What This Changes

The issue requests allowing `sessions_spawn` with `runtime="subagent"` to stream progress updates back to the parent session via `streamTo="parent"`.

## Best Possible Solution

Keep this open as the canonical native subagent parent-stream design tracker and require a maintainer-approved streaming, redaction, and session-isolation contract before implementation.

## Reproduction Assessment

Yes, at source level: current main and `v2026.6.8` only materialize `streamTo` for ACP and tests assert native subagent calls ignore it. I did not run a live long-running subagent session in this read-only review.

## Solution Assessment

No implementation path is approved yet. The best fix is not merely passing `streamTo` into `spawnSubagentDirect`; it needs a native progress event contract with conservative visibility, routing, cleanup, and security boundaries.
