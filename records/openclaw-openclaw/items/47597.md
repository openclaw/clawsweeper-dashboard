## Summary

Keep open. Current main fixed the related hard-error tolerance path by ignoring ACP-only fields on native subagent spawns, but it still documents and implements `streamTo:"parent"` as ACP-only and has no native subagent parent-progress relay.

## What This Changes

The issue requests native `streamTo:"parent"` support for `runtime:"subagent"` in `sessions_spawn`, so child-agent progress can be surfaced to the parent session in real time.

## Best Possible Solution

Keep the shipped ACP-field tolerance, then design and implement a native subagent parent-progress surface in core: extend `SpawnSubagentParams` and `sessions_spawn` docs/schema, add a scoped event/progress relay with cleanup and visibility rules, and cover both `mode:"run"` and `mode:"session"` with focused tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current tests accept `runtime:"subagent"` plus `streamTo:"parent"` but assert the field is stripped, and source shows native subagents launch through gateway `agent` without a parent relay.

## Solution Assessment

Unclear. The current silent-drop behavior is the narrow compatibility fix for provider-injected ACP-only fields, but the requested feature still needs maintainer design before choosing heartbeat progress versus a fuller event relay.
