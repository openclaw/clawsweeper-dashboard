## Summary

Keep open. Current main and v2026.5.20 still treat `streamTo` as ACP-only and ignore it for native subagent spawns; the hard-error compatibility bug was fixed, but real native parent progress streaming remains a product and security design decision.

## What This Changes

The issue requests parent-visible streaming/progress updates when `sessions_spawn` uses `runtime="subagent"`.

## Best Possible Solution

Keep this as the native subagent parent-stream design tracker and implement only after maintainers choose event shape, redaction/truncation, verbosity inheritance, delivery semantics, and alignment with yielded-wait UI state.

## Reproduction Assessment

Not applicable as a strict bug reproduction, but source inspection is high-confidence: current main ignores `streamTo` for native subagent calls, tests assert that contract, and docs mark the field ACP-only.

## Solution Assessment

No, the suggested remove-the-guard or heartbeat-only approach is not enough by itself. The safer maintainable path is an explicit native progress contract with redaction, truncation, verbosity, delivery, and event semantics decided before implementation.
