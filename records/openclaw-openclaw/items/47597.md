## Summary

Keep open. Current main still intentionally treats `streamTo` as ACP-only for `sessions_spawn`; the native subagent path ignores it, docs and tests lock that contract, and a native parent-progress stream still needs product and security decisions.

## What This Changes

Add parent-visible real-time progress streaming for `sessions_spawn` with `runtime="subagent"` using `streamTo="parent"`.

## Best Possible Solution

Keep this as the native subagent parent-stream design tracker and implement only after maintainers choose event semantics, verbosity inheritance, delivery, truncation, and redaction policy.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection is high-confidence for current behavior: native subagent calls ignore `streamTo`, tests assert that contract, and docs mark the field ACP-only.

## Solution Assessment

No, simply lifting the ACP-only behavior or adding an unconditional heartbeat is not the best maintainable fix. The safer path is an explicit native progress contract with redaction, truncation, verbosity, delivery, and event semantics decided before implementation.
