## Summary

Keep open: current main now tolerates stray ACP-only `streamTo` on native subagent calls by ignoring it, but native subagent parent progress streaming is still not implemented and this issue remains the canonical tracker for that product/security decision.

## What This Changes

The issue requests `streamTo="parent"` or equivalent parent-visible progress support for native `runtime="subagent"` sessions.

## Best Possible Solution

Keep this issue open as the canonical native subagent progress-stream design tracker, and implement only after maintainers choose a bounded, redacted native progress/event contract that preserves the current completion-delivery path.

## Reproduction Assessment

Yes, at source level: `sessions_spawn` strips `streamTo` unless `runtime` is ACP, tests assert native calls drop it, and native subagent runs are registered for terminal completion rather than live parent streaming. I did not run a live session repro because this was a read-only review.

## Solution Assessment

No final implementation shape is approved yet. Simply removing a guard or adding heartbeat text is not the best fix without a maintainer-approved native event, routing, and redaction contract.
