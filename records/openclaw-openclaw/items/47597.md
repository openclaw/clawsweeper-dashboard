## Summary

Keep open. Current main has shipped the compatibility fix that accepts and ignores ACP-only `streamTo` on native subagent spawns, but it still documents and implements parent streaming as ACP-only. The related merged PR fixed the hard-error failure mode; it did not add realtime parent progress streaming for `runtime="subagent"`.

## What This Changes

Request adding `streamTo="parent"` realtime progress streaming for native `runtime="subagent"` sessions, similar to the existing ACP parent relay.

## Best Possible Solution

Keep the shipped ACP-field tolerance, then design a native subagent parent progress surface with explicit event source, filtering, delivery, and cleanup rules. Update the `sessions_spawn` schema/docs and cover both `mode="run"` and `mode="session"` with focused tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current tests accept `runtime="subagent"` with `streamTo="parent"` but assert the field is stripped, and the source shows native subagent dispatch plus lifecycle-only registry tracking rather than realtime parent streaming.

## Solution Assessment

Unclear. The current compatibility fix is the right narrow answer for provider-injected ACP-only fields, but it is not the requested feature; the best implementation needs maintainer product/runtime choices before a safe patch can be written.
