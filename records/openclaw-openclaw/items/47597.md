## Summary

Keep open. Current main handles the related hard-error case by treating `streamTo` as ACP-only and ignoring it for native subagent spawns, but native `runtime="subagent"` still has no parent streaming/progress relay, so the requested feature remains unimplemented.

## What This Changes

The issue requests `streamTo:"parent"` support for native `runtime:"subagent"` so long-running child agents can stream real-time progress back to the parent session.

## Best Possible Solution

Define a native subagent progress contract, then add a typed parent-stream option to the subagent spawn path, route permitted progress through one owned relay mechanism, update schema/docs/changelog, and cover `mode="run"`, `mode="session"`, timeout, cleanup, and no-output behavior with focused tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main accepts `runtime:"subagent"` with `streamTo:"parent"` but tests assert the field is stripped, and the native spawn/registry code has no parent relay path.

## Solution Assessment

Unclear. The current silent-drop behavior is the right compatibility fix for provider-injected ACP-only fields, but the requested feature needs a maintainer-chosen event source and visibility model before a heartbeat or full relay implementation can be called the best fix.
