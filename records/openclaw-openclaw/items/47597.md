## Summary

Keep open. Current main and the latest release still keep `streamTo` ACP-only for `sessions_spawn`; the hard-error compatibility problem is fixed, but native subagent parent-visible progress streaming remains unimplemented and needs a product/security decision.

## What This Changes

Request to support `streamTo: "parent"` for `runtime: "subagent"` so native subagent progress can stream back to the parent session.

## Best Possible Solution

Define a native subagent progress contract that reuses existing completion/task surfaces where possible, with explicit redaction, verbosity, delivery routing, tests, and docs.

## Reproduction Assessment

Yes, as a source-level feature gap: current main gates `streamTo` to ACP and tests assert native/default subagent calls strip it before `spawnSubagentDirect`. No live failing run was attempted because this is a requested new capability, not a broken documented native subagent contract.

## Solution Assessment

No, the suggested guard removal or heartbeat alone is not the best solution. The maintainable path needs an explicit native progress event contract with redaction, truncation, verbosity, delivery, and yielded-wait semantics decided first.
