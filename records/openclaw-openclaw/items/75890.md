## Summary

Keep open. Current main still drops linked-device Note-to-Self Signal sync messages, and this PR targets the right Signal plugin path, but the current head still makes an automatic privacy-sensitive default change while docs, changelog, and regression coverage remain unchanged.

## What This Changes

The PR changes the Signal event handler to promote self-addressed `syncMessage.sentMessage` envelopes into normal inbound `dataMessage` handling for linked-device Note-to-Self chats.

## Best Possible Solution

Land a Signal-plugin-owned implementation after maintainer approval of the default behavior, with the number-model docs, changelog, and focused loop-protection tests updated together.

## Reproduction Assessment

Yes. A source-level reproduction is clear: feed `createSignalEventHandler` a receive event whose own-account `syncMessage.sentMessage` has matching destination/source; current main returns before dispatch, while the PR routes it onward.

## Solution Assessment

Unclear as currently proposed. The destination/source carve-out is the narrow technical guard, but automatic enablement needs maintainer approval or a documented config gate before it is the safest product/security solution.
