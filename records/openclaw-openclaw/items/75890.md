## Summary

Keep this PR open, but it is not merge-ready: the requested Signal Note-to-Self support is useful, yet the current branch can re-admit bot-authored self sync traffic and silently changes the meaning of an existing `allowFrom` entry without a maintainer-approved upgrade contract.

## What This Changes

The PR promotes self-addressed Signal linked-device `syncMessage.sentMessage` envelopes to inbound `dataMessage` when the operator number appears in `channels.signal.allowFrom`, and updates Signal docs plus `CHANGELOG.md`.

## Best Possible Solution

Land this only after the Signal self-chat mode has an explicit maintainer-approved opt-in contract, a tested discriminator that rejects OpenClaw-authored self sync replays, inspectable redacted real Signal proof, and a refreshed branch without release-owned changelog edits.
