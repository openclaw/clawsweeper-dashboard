## Summary

Keep open. Current main still drops Signal Note-to-Self sync envelopes, and this PR is a plausible implementation path for the linked-device use case, but the current patch still makes a privacy-sensitive personal-account routing change without a documented opt-in/default decision, docs, changelog, or regression coverage.

## What This Changes

The PR promotes self-addressed Signal `syncMessage.sentMessage` envelopes to `dataMessage` in the Signal event handler so linked-device Note-to-Self messages enter normal inbound processing.

## Best Possible Solution

Land a Signal-plugin-owned fix that validates the sync sent-message shape, preserves sentTranscript loop protection, uses a maintainer-approved opt-in/default, and updates docs, changelog, config metadata, and targeted tests together.

## Reproduction Assessment

Yes. A source-level reproduction is clear: feed `createSignalEventHandler` a receive event whose own-account `syncMessage.sentMessage` has matching destination/source; current main returns before dispatch, while the PR routes that envelope.

## Solution Assessment

No. The destination/source guard matches signal-cli’s payload shape, but automatic ingestion without a documented opt-in/default decision is not the safest maintainable solution for personal-account private notes.
