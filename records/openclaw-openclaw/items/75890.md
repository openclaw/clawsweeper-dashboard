## Summary

Keep open. The user problem is valid and current main still drops Signal Note-to-Self sync envelopes, but the latest PR head still weakens the Signal loop-protection boundary by promoting allowlisted self-addressed `sentMessage` sync traffic before the own-account/sync guard can reject echoes or replays.

## What This Changes

The PR adds Signal linked-device Note-to-Self routing behind an implicit `allowFrom` self-chat opt-in, plus matching Signal docs and a changelog entry.

## Best Possible Solution

Add a tested Signal self-chat implementation that accepts only user-authored Note-to-Self sync messages under explicit opt-in while continuing to drop outbound and replayed sentTranscript sync echoes.

## Reproduction Assessment

Yes. Source inspection shows current main returns for own-account messages and for any `syncMessage`, so a self-addressed `syncMessage.sentMessage` envelope cannot reach inbound dispatch; I did not run a live Signal linked-device test.

## Solution Assessment

No. The PR is in the right Signal module and now includes docs/changelog, but destination/source equality plus `allowFrom` opt-in is not yet the safest maintainable guard because it can match sentTranscript-shaped self-sync traffic.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

- **[P2] Preserve the sync echo guard for self-chat:** `extensions/signal/src/monitor/event-handler.ts:573-585`

Add an origin/echo discriminator and regression coverage for both user-authored Note-to-Self and outbound/replayed self-sync traffic before landing.
