## Summary

Keep open: current main still lacks the requested Signal Note-to-Self path, but this PR should not merge as-is because the new self-chat branch can also route self-addressed sync echoes across the existing loop-protection boundary.

## What This Changes

The PR changes Signal event handling to promote self-addressed `syncMessage.sentMessage` traffic as inbound when the operator's E.164 is in `allowFrom`, with matching Signal docs and a changelog entry.

## Best Possible Solution

Add Note-to-Self support with a tested discriminator that accepts user-authored self-chat while continuing to drop OpenClaw outbound echoes and other sync traffic, then land the docs/changelog with current-main validation.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main drops own-account messages and any `syncMessage`, while signal-cli documents Note-to-Self-shaped receive payloads as `syncMessage.sentMessage` with matching source/destination identifiers. I did not run a live linked-device Signal test in this read-only review.

## Solution Assessment

No. The `allowFrom` opt-in and docs direction are plausible, but destination/source equality alone is not the narrowest safe guard; the patch needs an origin or echo discriminator plus regression tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep self-sync echoes out of self-chat:** `extensions/signal/src/monitor/event-handler.ts:573-585`
  - body: This branch promotes every self-addressed `syncMessage.sentMessage` in self-chat mode, and because the later own-account guard is in the same `else if` chain it is skipped for Note-to-Self sync traffic. signal-cli receive docs show `syncMessage.sentMessage` with matching source and destination identifiers, which is also the sentTranscript shape OpenClaw was already dropping to prevent replay loops. A replayed or echoed OpenClaw Note-to-Self reply can therefore re-enter the agent; add an origin/echo guard and coverage before merge.
