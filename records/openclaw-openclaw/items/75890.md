## Summary

Keep this PR open, but it should not merge as-is: the requested Signal Note-to-Self behavior is plausible, yet the current patch still crosses the existing sync-message loop-protection boundary without a proven way to distinguish user-authored self notes from echoed or replayed bot sentTranscripts.

## What This Changes

The PR changes the Signal monitor to promote self-addressed linked-device `syncMessage.sentMessage` envelopes into inbound `dataMessage`, updates Signal docs, and adds a changelog entry.

## Best Possible Solution

Add Signal Note-to-Self support only with an explicit self-chat opt-in plus a tested echo/replay discriminator, preserving the existing sentTranscript loop guard for bot-authored outbound traffic.

## Reproduction Assessment

Yes for the current limitation: source inspection shows current main returns for own-account senders and for any `syncMessage` envelope, so a linked-device Note-to-Self `syncMessage.sentMessage` cannot reach inbound dispatch. I did not run a live Signal linked-device reproduction in this read-only review.

## Solution Assessment

No. The direction is maintainable if accepted, but destination/source equality plus allowFrom self-listing is not enough to preserve loop protection without an origin, device, timestamp, or outbound-echo guard and focused tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep self sync echoes out of self-chat:** `extensions/signal/src/monitor/event-handler.ts:573-585`
  - body: This branch promotes every self-addressed `syncMessage.sentMessage` in self-chat mode before the own-account guard can run. signal-cli represents sent-message sync traffic with source and destination account identifiers, and the existing code drops sync envelopes to prevent sentTranscript replay loops; a replayed or echoed OpenClaw Note-to-Self reply can therefore be routed back into the agent. Add an origin/echo discriminator and coverage for both user-authored Note-to-Self and outbound echoes before merge.
  - confidence: 0.84
