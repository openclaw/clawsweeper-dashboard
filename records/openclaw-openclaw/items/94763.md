  - body: After the retry omits `dimensions`, this returns raw provider-default vectors while `provider.dimensions` and `cacheKeyData` still advertise the configured size. Memory-core creates sqlite-vec tables and cache rows from the returned vector length, so truncate/validate or fail before returning mismatched vectors.
  - confidence: 0.92
- **[P1] Narrow the dimensions rejection latch:** `src/plugins/openai-compatible-embedding-provider.ts:389`
  - body: The matcher accepts generic `bad request` text and any connection reset, then the catch path permanently adds the client to the no-dimensions WeakSet. A transient reset or unrelated 400 on the first call would silently omit dimensions for later calls on that client.
  - confidence: 0.88
- **[P2] Remove or use the retry cap constant:** `src/plugins/openai-compatible-embedding-provider.ts:377`
  - body: `DIMENSIONS_RETRY_MAX` is declared but never read. Core TypeScript validation enables `noUnusedLocals`, so wire it into the retry logic or remove it before this can pass the normal type gate.
  - confidence: 0.95

  - body: The committed proof script imports from `/media/vdc/openclaw`, so a normal clone either fails or executes code outside the reviewed checkout. Make the script repo-relative or keep this proof in the PR body instead of committed code.
