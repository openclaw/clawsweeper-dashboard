## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve dimensions for compatible endpoints:** `src/plugins/openai-compatible-embedding-provider.ts:314-319`
  - body: This changes configured `dimensions` into an `api.openai.com` hostname allowlist. Current memory config maps `outputDimensionality` into `dimensions`, the provider/cache identity still records that value, and existing docs/tests support OpenAI-compatible endpoints that need custom vector sizes. Any non-OpenAI endpoint or proxy that accepts `dimensions` will now silently receive the wrong request shape. Preserve the configured field by default and only retry or omit it after a provider actually rejects or resets on that field.
  - confidence: 0.9
- **[P2] Do not stop splitting before single-item recovery:** `extensions/memory-core/src/memory/manager-embedding-policy.ts:175-180`
  - body: The new depth guard throws once depth reaches 4 even when the batch still has multiple items and the error is splittable. For a 32-item batch that only succeeds at single-item requests, current main keeps splitting to one item, while this PR fails at two-item leaves. Distinguish whole-endpoint outages from oversized batches, or keep splitting until a safe minimum with focused coverage.
  - confidence: 0.84
