## Summary

- **[P1] Keep fallback vectors consistent with configured dimensions:** `src/plugins/openai-compatible-embedding-provider.ts:466-468`
  - body: After the retry omits `dimensions`, this returns raw provider-default vectors while provider identity still records the configured size. Memory-core uses the returned vector length for sqlite-vec readiness and cache entries, so truncate or fail before returning mismatched vectors.
  - confidence: 0.92
- **[P1] Latch only on dimensions-specific rejection:** `src/plugins/openai-compatible-embedding-provider.ts:389`
  - body: The matcher accepts generic `bad request` text and any connection reset, then the catch permanently adds the client to the no-dimensions WeakSet. A transient reset or unrelated 400 on the first call would silently omit dimensions for all later calls on that client.
  - confidence: 0.88
- **[P1] Remove the checkout-specific proof import:** `scripts/proof-embedding-retry.mjs:3`
  - body: The committed proof script imports from `/media/vdc/openclaw`, so a normal clone either fails or executes code outside the reviewed checkout. Make the script repo-relative or keep this proof in the PR body instead of committed code.
  - confidence: 0.95

  - body: `DIMENSIONS_RETRY_MAX` is declared but never read. Core TypeScript validation has `noUnusedLocals` enabled, so either wire it into the retry logic or remove it.
