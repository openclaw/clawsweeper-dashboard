## What This Changes

The PR changes `getNoProgressStreak` to count only the contiguous no-progress tool-call tail, adds two regression tests, and adds a changelog entry.

## Best Possible Solution

Land this scoped tail-semantics fix, or an equivalent maintainer patch, with regression coverage for known polling and no-progress breaker behavior.

## Solution Assessment

Yes. Switching the nonmatching and missing-result branches from `continue` to `break` is the narrowest maintainable fix and preserves the existing thresholds, detectors, and config surface.
