## Summary

Keep open: the PR addresses a real stale-history bug direction, but it is not merge-ready because the shared helper change also narrows the documented global circuit breaker and the contributor has not supplied after-fix real behavior proof.

## What This Changes

The PR changes agent no-progress loop detection to count only the contiguous current tail for identical tool/result history and adds unit tests plus a changelog entry.

## Best Possible Solution

Split tail-scoped per-pattern no-progress detection from a session/run-scoped global breaker, keep the stale-history regression for known polling, and reconcile this branch with https://github.com/openclaw/openclaw/pull/79267.

## Reproduction Assessment

Yes. Source inspection on current main shows getNoProgressStreak skips intervening nonmatching records, so repeated process poll no-progress calls, another tool call, then a process retry can still count stale matches.

## Solution Assessment

No as written. Tail semantics are appropriate for the per-pattern stale retry bug, but applying them through the helper used by globalCircuitBreakerThreshold is not the narrowest maintainable fix.
