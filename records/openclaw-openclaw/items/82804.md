## Summary

Keep open: this PR has the protected `maintainer` label, and the patch appears to reintroduce raw child-result delivery that current `main` recently removed from the subagent completion path. It needs maintainer judgment and real channel proof before merge.

## What This Changes

The PR adds a `sendMessage` fallback for single subagent completion announcements when the requester completion-agent returns no visible payload, plus focused regression-test updates.

## Best Possible Solution

Keep subagent completion delivery mediated; if empty requester-agent payloads are valid failures, fix the requester handoff, retry path, or an explicitly approved mediated fallback rather than raw-sending child output.

## Reproduction Assessment

Yes, at source level: current main returns `completion agent did not produce a visible reply` when a completion announcement expects a message and the gateway-agent result has no visible payload. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No: the proposed direct `sendMessage` fallback is not the safest fix because it bypasses the requester-agent delivery contract that current main and recent history preserve. A maintainer should decide whether a narrow mediated fallback is acceptable or whether the handoff/retry path should be repaired instead.
