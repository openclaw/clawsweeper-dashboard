## Summary

Keep this PR open for maintainer review: the patch is a narrow, correct OpenAI provider hardening with no blocking findings, but it deliberately changes oversized success JSON from unbounded buffering to a capped error and should be accepted as that compatibility/availability tradeoff.

## What This Changes

The PR replaces two OpenAI success-path JSON reads with `readProviderJsonResponse` and adds focused regression tests for Batch status and Realtime client-secret oversized responses.

## Best Possible Solution
