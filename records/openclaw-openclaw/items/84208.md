## Best Possible Solution

Keep the CLI-boundary validation for click-coords, leave resize validation in shared runBrowserResizeWithOutput or move any message change there, and back it with actual CLI proof or focused regression coverage.

Yes for the click-coords half: current main converts `abc` with Number, places NaN in the request body, and JSON serialization turns it into null. The resize half does not reproduce as stated on current main because the shared resize helper already exits before posting non-finite dimensions.
