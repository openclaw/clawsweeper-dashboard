## What This Changes

The PR adds finite-number guards to the browser `click-coords` and `resize` CLI handlers before dispatching browser requests.

## Best Possible Solution

Keep the CLI-boundary validation for `click-coords`, remove or centralize the resize validation in `runBrowserResizeWithOutput`, and back the result with focused CLI proof or regression coverage.

## Reproduction Assessment

Yes for the `click-coords` half: current main sends `Number("abc")` into the `/act` body path, and JSON serialization turns it into `null`. The resize half does not reproduce as stated because current main already rejects non-finite dimensions in `runBrowserResizeWithOutput`.

## Solution Assessment

No, not as written. The `click-coords` guard is the right boundary, but the resize guard should be removed or centralized in the existing shared helper to avoid duplicate validation paths.
