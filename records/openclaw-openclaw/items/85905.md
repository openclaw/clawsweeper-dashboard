## Best Possible Solution

Normalize HEIC/HEIF inside OpenClaw-owned WebChat/media image handling before any provider call, then cover both native prompt-image delivery and text-only `tools.media.image` descriptions with focused regressions.

Yes. The narrow maintainable fix is to reuse or centralize HEIC/HEIF normalization before WebChat prompt-image and media-understanding provider calls, rather than relying on external filesystem conversion races.
