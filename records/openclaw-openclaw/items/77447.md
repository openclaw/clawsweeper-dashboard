## Summary

Keep open: current main has provider replay hygiene, narrow session repair, memory readiness checks, and security/secrets audits, but no configurable dry-run scan/sanitize workflow for unsafe persisted text across transcripts, summaries, active memory, and memory stores; the related PR is narrower and does not supersede this request.

## What This Changes

Add a configurable memory/transcript hygiene doctor or command surface that scans durable session and memory stores for unsafe persisted artifacts, reports JSON findings by rule, and later applies conservative span-level sanitization with allowlisted exceptions.

## Best Possible Solution

Design a core-owned read-only JSON scan/report contract first, using existing session and memory path resolution, then add conservative sanitizer apply behavior after rule scope, allowlists, locking, backups, and docs are reviewed.
