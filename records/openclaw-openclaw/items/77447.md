## Summary

Keep open: current main has adjacent transcript hygiene, session repair, memory readiness, and redacted recall paths, but it does not implement the requested configurable dry-run scan/sanitize workflow across durable transcripts, summaries, active memory, and memory stores; the related PR is narrower and still open.

## What This Changes

Request a configurable doctor/plugin-style hygiene surface to dry-run scan and optionally sanitize unsafe persisted transcript and memory artifacts with JSON findings and allowlisted exceptions.

## Best Possible Solution

Design a read-only JSON scan/report contract first, then add conservative apply-mode sanitization after maintainers settle rule semantics, allowlists, backups, locking, and docs.
