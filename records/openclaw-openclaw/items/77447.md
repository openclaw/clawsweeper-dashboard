## Summary

Keep open: current main has adjacent provider replay hygiene, narrow transcript repair, memory readiness checks, and memory artifact audits, but it does not provide the requested configurable dry-run scan/sanitize workflow across durable transcripts, summaries, active memory, and memory stores. The linked PR is narrower and remains open, so it does not supersede this feature request.

## What This Changes

Add a configurable memory/transcript hygiene doctor or plugin surface that scans and optionally sanitizes unsafe persisted text with dry-run JSON reporting.

## Best Possible Solution

Design a core-owned read-only JSON scan/report contract first, using existing session and memory path resolution, then add conservative sanitizer apply behavior after rule scope, allowlists, locking, backups, and docs are reviewed.
