## Summary

Keep open: current main has provider replay hygiene, targeted transcript repair, memory readiness/status checks, and narrower memory-ingestion filters, but it does not provide the requested configurable dry-run scan/report/sanitize workflow across durable transcripts, summaries, active memory, and memory stores. ## What This Changes

Add a configurable memory/transcript hygiene doctor or plugin surface that dry-runs scans for unsafe persisted text and can conservatively sanitize unsafe spans with JSON evidence.

## Best Possible Solution

Design a scanner-first core/operator contract with JSON findings, redacted snippets, configured storage roots, and reviewed rule semantics before adding conservative apply-mode sanitation.
