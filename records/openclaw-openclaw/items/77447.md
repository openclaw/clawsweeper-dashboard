## Summary

Keep open: current main has provider replay hygiene, narrow transcript/session repairs, and memory readiness/artifact audits, but no configurable dry-run scan/sanitize workflow across persisted transcripts, summaries, active memory, and memory stores.

## What This Changes

Add a configurable doctor or plugin surface to dry-run scan and optionally sanitize unsafe persisted memory/transcript artifacts such as raw paths, file URLs, MEDIA directives, and tool-output leakage.

## Best Possible Solution

Design a core-owned read-only JSON scan/report contract first, reusing existing session and memory path resolution, then add opt-in sanitizer apply behavior after rule, allowlist, locking, backup, and docs semantics are reviewed.

## Reproduction Assessment

Not applicable: this is a feature request rather than broken existing behavior. Source and docs inspection shows adjacent repair/readiness tools, but no existing command that can be reproduced as a failing implementation of this workflow.

## Solution Assessment

Unclear until design review: a read-only JSON scanner appears to be the safest first step, but sanitizer/apply behavior and configurable rule semantics need maintainer decisions before a fix PR is safe.
