## Summary

Keep open: current main and v2026.6.8 still lack the requested operator-facing memory/transcript hygiene scanner or dry-run sanitizer; existing code covers narrow transcript repairs, memory artifact repairs, provider replay hygiene, and ingestion filtering rather than configurable durable-store hygiene.

## What This Changes

The issue requests a configurable memory/transcript doctor command or plugin surface that can scan and conservatively sanitize unsafe persisted artifacts such as local paths, file URLs, delivery directives, and internal tool-output leakage.

## Best Possible Solution

Design a read-only hygiene lint first with stable rule IDs, scoped store ownership, JSON findings, and redacted snippets; add apply/sanitizer behavior only after maintainers approve repair semantics, backups, allowlists, and plugin/core boundaries.
