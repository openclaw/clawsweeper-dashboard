## Summary

Keep open: current main has preventive/display sanitizers and narrow legacy transcript repair, but it does not provide the requested operator-facing dry-run/JSON hygiene scanner or sanitizer for durable transcripts and memory stores.

## What This Changes

The issue requests a configurable memory/transcript hygiene doctor or plugin surface that scans and sanitizes unsafe persisted artifacts such as local paths, file URLs, delivery directives, and internal tool-output leakage.

## Best Possible Solution

Design a read-only doctor/lint hygiene check with stable rule IDs and JSON findings first, then add repair/sanitizer behavior only after maintainer approval of ownership, backups, allowlists, and redaction semantics.
