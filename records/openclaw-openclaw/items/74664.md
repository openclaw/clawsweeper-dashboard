## Summary

Keep open: current `main` still sends OpenAI Completions tool payloads without checking `compat.supportsTools`, and open PR #74738 is already paired to this issue with closing syntax, so the issue should remain open until that PR or an equivalent patch lands.

## What This Changes

The issue requests that OpenAI-compatible chat-completions payloads omit populated `tools`, `tool_choice`, and the empty `tools: []` fallback when a model sets `compat.supportsTools: false`.

## Best Possible Solution

Land PR #74738 or an equivalent narrow patch that gates populated tools, `tool_choice`, and the tool-history fallback with `supportsModelTools(model)`, with regression tests and a changelog entry.

## Reproduction Assessment

Yes. Passing an OpenAI Completions model with `compat.supportsTools: false` and non-empty `context.tools` to `buildOpenAICompletionsParams()` on current `main` deterministically sets `params.tools`; tool-call history without current tools also sets `tools: []`.

## Solution Assessment

Yes. Enforcing the existing `supportsModelTools()` helper inside `buildOpenAICompletionsParams()` is the narrow maintainable fix and avoids provider-specific workarounds.
