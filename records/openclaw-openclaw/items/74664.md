## Summary

Keep open: current main still omits the `supportsModelTools(model)` guard in the OpenAI Completions payload builder, and open PR #74738 is already paired to this issue with closing syntax.

## What This Changes

Honor `compat.supportsTools: false` for OpenAI Completions payloads by omitting populated `tools`, default `tool_choice`, and the `tools: []` history fallback.

## Best Possible Solution

Land #74738 or an equivalent narrow patch that gates OpenAI Completions tool emission and the history fallback with `supportsModelTools(model)`, with focused regression tests and the required changelog entry.

## Reproduction Assessment

Yes. On current main, a model with `compat.supportsTools: false` and non-empty `context.tools` still deterministically reaches the unguarded `params.tools` assignment; tool-call history without current tools reaches the unguarded `tools: []` fallback.

## Solution Assessment

Yes. Applying the existing `supportsModelTools()` helper inside `buildOpenAICompletionsParams()` is the narrow maintainable fix; the upstream dependency bump alone does not cover this local payload builder.
