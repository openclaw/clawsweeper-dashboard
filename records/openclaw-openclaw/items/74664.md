## Summary

Keep open: current main still has the OpenAI Completions payload builder emitting populated tools, tool_choice, and the tool-history tools: [] fallback without checking the existing supportsTools compat contract, and #74738 remains the paired open fix PR.

## What This Changes

OpenAI-compatible chat-completions payloads should omit tools, tool_choice, and the tools: [] history fallback when a model sets compat.supportsTools to false.

## Best Possible Solution

Review and land #74738 or an equivalent narrow patch that gates OpenAI Completions tool emission and the history fallback with supportsModelTools(model), with regression coverage and a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a deterministic current-main reproduction path: call buildOpenAICompletionsParams() with compat.supportsTools false and non-empty context.tools, or with tool-call history and no current tools, and the builder reaches unguarded params.tools assignments.

## Solution Assessment

Yes. Applying the existing supportsModelTools() helper inside buildOpenAICompletionsParams() is the narrow maintainable fix; provider-specific workarounds would leave this lower-level payload path open.
