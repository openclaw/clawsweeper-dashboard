## Summary

## What This Changes

The PR adds Google Chat media-upload 403 fallback-to-text-link handling across outbound adapter, monitor reply delivery, and message actions, with regression tests for fallback, local-path rejection, and non-403 propagation.

## Best Possible Solution

Land a Google Chat plugin-owned fallback for remote http(s) media only when upload fails with Google's insufficient-authentication-scope denial, covered across adapter/reply/action paths and shown in a redacted live Google Chat run.

## Reproduction Assessment

Yes. Source inspection and Google's API contract give a high-confidence path: Google Chat app auth uses chat.bot, current main uploads media via attachments:upload, and that upload endpoint requires user-auth scopes; I did not run a live Google Chat workspace repro.

## Solution Assessment

No. The text-link fallback is the right owner-boundary fix, but this patch should restrict the fallback to Google's insufficient-authentication-scope upload denial and still needs live Google Chat proof.
