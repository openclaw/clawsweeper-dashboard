## Summary

Keep open: this PR addresses a real Google Chat message-loss bug, but the current patch is not merge-ready because its upload-403 detector is broader than the documented app-auth scope failure and the supplied proof is mocked/local only.

## What This Changes

The branch adds Google Chat text-link fallback handling for attachment upload 403s in message actions, outbound sendMedia, and monitor reply delivery, plus regression tests.

## Best Possible Solution

Narrow the fallback to the known Google Chat upload insufficient-scope/PERMISSION_DENIED contract for remote http(s) media across all affected paths, then add redacted live app-auth Google Chat proof or explicit maintainer acceptance.

## Reproduction Assessment

Yes, at source level. Current main uses the `chat.bot` auth scope and routes remote Google Chat media through attachments:upload in the affected paths without a text-link fallback; I did not run live Google Chat credentials.

## Solution Assessment

No, not as-is. The plugin-owned fallback is the right layer, but it should use a narrow upload auth/scope detector rather than any `403`, and it still needs live or explicitly accepted proof.
