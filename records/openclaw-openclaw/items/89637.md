## What This Changes

## Best Possible Solution

Narrow the fallback to the known Google Chat upload insufficient-scope/PERMISSION_DENIED contract for remote URLs across all three paths, then merge one canonical branch with live app-auth Google Chat proof or explicit maintainer acceptance.

## Reproduction Assessment

## Solution Assessment

No, not as-is. The plugin-owned fallback is the right layer, but it should use a narrow upload auth/scope detector rather than any 403 and needs live or explicitly accepted proof.
