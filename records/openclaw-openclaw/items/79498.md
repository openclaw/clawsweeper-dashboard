## Summary

Keep open. The PR addresses a real security-audit false positive for routed sandboxed WhatsApp groups and includes terminal proof, but the latest diff can also suppress the multi-user warning for default direct-DM main sessions under `sandbox.mode="non-main"`; this security-labeled heuristic needs a fix and maintainer/security review before merge.

## What This Changes

The PR makes the security audit compute route-aware shared-ingress agents, treats external ingress as guarded by non-main sandboxing, and updates trust-model tests, security docs, and changelog.

## Best Possible Solution

Land one reconciled route-aware heuristic that exempts sandboxed group/channel routes, preserves direct-DM main-session warnings unless secure DM scope creates non-main keys, and keeps tests/docs/changelog aligned after security maintainer review.

## Reproduction Assessment

Yes, at source level. Current main's docs and session-key builder show default direct DMs use the main session, while the PR diff treats direct-DM signals as externally sandboxed under `non-main`; I did not run tests because this was a read-only review.

## Solution Assessment

No. The route-aware group-agent narrowing is the right direction, but the implementation needs to distinguish direct DM ingress from group/channel ingress before treating `non-main` as a sandbox guard.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep direct DMs on the main-session risk path:** `src/security/audit-extra.sync.ts:1380`
  - body: This path adds `dmPolicy="open"` and wildcard `allowFrom` to shared ingress, but then checks every shared-ingress agent as `sessionKind: "external"`.
