## Summary

Keep open: current main still appears to miss the documented Slack `message_sent` delivery correlation, and the `maintainer` label blocks conservative cleanup closure even with a source-backed reproduction.

## What This Changes

The issue requests that Slack assistant reply delivery emit `message_sent` with the delivered Slack timestamp and that outbound sent hooks carry available session/run correlation fields.

## Best Possible Solution

Implement the fix in the Slack-owned reply delivery path and the generic outbound sent-hook emitter, then add focused regression coverage for Slack delivered IDs, session/run correlation, and failure payload shape.

## Reproduction Assessment

Yes. Source inspection shows the normal Slack inbound assistant reply path reaches `deliverReplies` and `sendMessageSlack` without sent-hook emission, while generic outbound computes a session key but does not put it in the canonical sent context.

## Solution Assessment

Yes for the main fix boundary. The narrow maintainable path is to emit `message_sent` where Slack replies are actually delivered and to forward already-supported correlation fields through generic outbound, with any new typed thread fields kept additive and deliberate.
