## Summary

Keep open: current main has partial guardrails, but the central repeated-overflow problem is not fully solved. Source still allows successful-stop overflow detection from `usage.input + cacheRead`, normal overflow recovery still compacts before fallback tool-result truncation, and two related fix PRs remain open/unmerged.

## What This Changes

The issue asks OpenClaw to prevent repeated auto-compaction loops after successful compaction in tool-heavy live sessions.

## Best Possible Solution

Unify overflow recovery so tool-heavy sessions try deterministic truncation before repeated compaction and successful-stop overflow checks ignore unreliable accumulated usage, with trigger metadata and regression coverage.
