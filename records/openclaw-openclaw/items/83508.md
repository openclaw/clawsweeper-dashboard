## Summary

Keep open: current main still lacks a supported OpenClaw config, SecretRef, or env-file path that turns vaulted Claude OAuth material into the environment seen by a gateway-spawned Claude ACP harness.

## What This Changes

The issue requests supported Claude ACP harness authentication for `sessions_spawn({ runtime: "acp", agentId: "claude" })` and also reports a timeout-control mismatch for that path.

## Best Possible Solution

Define an explicit ACPX plugin-owned per-agent harness auth contract, preferably SecretRef-aware, and pass resolved credentials only to the selected harness while documenting `ACPX_AUTH_*` as the low-level env convention.

## Reproduction Assessment

Yes. The reporter provided live before/after evidence, and source inspection shows `sessions_spawn` reaches ACP session creation without a configured harness credential env path while the Claude wrapper only inherits the gateway process environment.

## Solution Assessment

No for silent auto-sourcing, because that would implicitly widen credential exposure. The maintainable path is an explicit per-agent ACPX harness auth/env contract plus docs for the existing `ACPX_AUTH_*` convention; the timeout sub-report is already addressed on current main.
