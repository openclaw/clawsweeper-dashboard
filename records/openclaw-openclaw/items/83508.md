## Best Possible Solution

Add or document an explicit ACPX plugin-owned, SecretRef-aware per-agent harness auth/env contract that maps configured credentials to `ACPX_AUTH_*` or child env, and avoid silent secrets-file auto-source unless maintainers approve that security posture.
