## Summary

Keep open: current main and the latest release still lack a supported ACPX/Claude harness env or SecretRef forwarding path, so the vaulted OAuth auth request remains unresolved. ## What This Changes

Request a supported ACP Claude auth path that forwards vaulted OAuth credentials into gateway-spawned ACP harnesses, plus cleanup for timeoutSeconds control forwarding.

## Best Possible Solution

Add an explicit ACPX plugin-owned per-harness auth/env contract, preferably SecretRef-aware, document the ACPX_AUTH_* gateway-env workaround, and avoid silently auto-sourcing arbitrary secret files.

## Reproduction Assessment

Yes, at source level: sessions_spawn reaches ACP session creation without an env payload, and the Claude wrapper only inherits the gateway process environment. I did not run a live Claude Max/acpx OAuth repro in this read-only review.

## Solution Assessment

No for the auto-source proposal: silently loading ~/.openclaw/secrets/anthropic.env would be a credential-boundary decision. The safer path is an explicit per-harness env or SecretRef contract, while the timeoutSeconds part is already handled on current main.
