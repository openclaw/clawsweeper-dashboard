## Best Possible Solution

Add an explicit ACPX plugin-owned harness environment contract, preferably SecretRef-aware and generic per agent, then pass the resolved env only to the selected harness without silently auto-sourcing arbitrary secret files.

## Reproduction Assessment

Yes, from source inspection: `sessions_spawn` reaches ACP session creation without forwarding env, and the Claude wrapper only inherits the gateway process environment. I did not run a live Claude/acpx OAuth repro in this read-only review.

## Solution Assessment

No for the issue’s auto-source suggestion: silently loading `~/.openclaw/secrets/anthropic.env` would be a credential-boundary decision. The safer direction is an explicit ACPX per-agent env or SecretRef contract, while the timeout sub-issue is already handled on current main.
