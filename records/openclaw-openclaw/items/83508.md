## What This Changes

The issue requests a supported way for ACPX/Claude `sessions_spawn` to forward vaulted OAuth or SecretRef/env credentials into spawned `claude-agent-acp` sessions.

## Best Possible Solution

Add an explicit ACPX plugin-owned, SecretRef-aware per-agent harness auth/env contract that maps configured credentials to ACPX or harness env, document the supported `ACPX_AUTH_*` workaround, and avoid silent auto-source behavior.

## Reproduction Assessment

Yes. The source path is clear: `sessions_spawn` reaches ACP session creation without env forwarding, the Claude wrapper only inherits gateway `process.env`, and acpx requires configured auth or `ACPX_AUTH_*` for ACP auth-method selection.

## Solution Assessment

No for silent auto-source behavior. The maintainable fix is an explicit SecretRef-aware ACPX harness auth/env contract; the timeout sub-issue already has current-main coverage.
