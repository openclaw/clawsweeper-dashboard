## Best Possible Solution

Add an explicit ACPX plugin-owned, SecretRef-aware per-agent harness auth/env contract that maps configured credentials to ACPX or harness env, with docs and tests, while avoiding silent auto-source behavior.

Yes, source-reproducible: sessions_spawn reaches ACP ensureSession without env forwarding, the Claude wrapper only inherits gateway process.env, and upstream acpx requires ACPX_AUTH_* or auth config for this auth path.

## Solution Assessment

No for the auto-source suggestion: silently loading ~/.openclaw/secrets/anthropic.env would hide a credential boundary; the better fix is an explicit SecretRef-aware ACPX harness auth/env contract, while the timeout sub-issue is already handled on main.
