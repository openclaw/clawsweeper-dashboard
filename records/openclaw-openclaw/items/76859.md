## Summary

Keep this PR open for maintainer review: it adds a new official community-skills catalog, but the install guidance conflicts with the current ClawHub-only `openclaw skills install <slug>` contract and the catalog policy needs an explicit docs/product decision.

## What This Changes

The PR adds a new `docs/tools/community-skills.md` page listing four third-party skill repositories and inserts it into the Skills docs navigation.

## Best Possible Solution

Align community skill discovery with ClawHub or a maintainer-approved curated docs policy, and make every install command match a real supported install path before merging.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a docs PR. The reviewable failure is source-reproducible from current main: `openclaw skills install` expects a ClawHub slug and rejects slashes, while the new page documents `nerua1/skill-name`.

## Solution Assessment

No. The proposed docs page is not ready as the best solution until the invalid install guidance is fixed and maintainers decide whether official docs should host a curated third-party skills list instead of routing discovery through ClawHub.
