## Summary

Keep open. Current main and v2026.5.7 still do not accept or forward `memorySearch.local.maxThreads`, while node-llama-cpp exposes `maxThreads` on the `getLlama` options boundary; the related open PR overlaps the local llama tuning surface but does not implement this thread-limit option.

## What This Changes

Request to add `memorySearch.local.maxThreads` for local node-llama-cpp embeddings and forward it to `getLlama`.

## Best Possible Solution

Add a narrow, validated `memorySearch.local.maxThreads` option for the built-in local embedding provider, preserve existing defaults when unset, forward it only to `getLlama`, and update config schema/help/docs/tests/changelog together.

## Reproduction Assessment

Not applicable as a feature request. The current gap is high-confidence from source inspection: schema, resolved config, provider options, node-llama shim, docs, tests, and runtime call all omit `maxThreads`.

## Solution Assessment

Yes. The narrowest maintainable solution is a config/type/resolution/runtime passthrough modeled on existing `contextSize`, with explicit validation semantics and no unrelated GPU or flash-attention changes.
