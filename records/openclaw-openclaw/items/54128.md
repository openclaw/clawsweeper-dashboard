## What This Changes

The issue requests adding a `memorySearch.local.maxThreads` option for local node-llama-cpp embeddings so users can tune CPU thread usage during memory indexing.

## Best Possible Solution

Add a validated `memorySearch.local.maxThreads` field for the built-in local embedding provider, preserve default behavior when unset, carry it through resolved config and provider option types, pass it to `getLlama` only when configured, and align docs, generated config metadata, tests, and changelog.
