## Summary

Keep open. Current main and the latest shipped release still do not accept or forward `memorySearch.local.maxThreads`; the overlapping open PR changes other local llama knobs but does not cover this requested thread-limit option.

## What This Changes

Request to add a `memorySearch.local.maxThreads` setting for local node-llama-cpp embeddings and pass it through to `getLlama` for CPU thread tuning.

## Best Possible Solution

Add a narrow, validated local embedding `maxThreads` option, preserve existing defaults when unset, forward it only to the built-in local node-llama-cpp provider, and update schema, generated config surfaces, docs, tests, and changelog together.

## Reproduction Assessment

## Solution Assessment

Yes. The maintainable path is a narrow config/type/resolution/runtime passthrough modeled on `contextSize`, with explicit validation semantics and no unrelated local llama tuning changes.
