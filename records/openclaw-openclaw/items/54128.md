## Summary

Keep open. The requested thread-limit knob is still absent from current main: there is no config/schema/docs carrier and the local provider still initializes node-llama-cpp with only `logLevel`, while the upstream `LlamaOptions` type includes `maxThreads`. The related PR #60683 is open but, from the provided context, covers other local llama settings and does not resolve this request.

## What This Changes

Add a configurable `memorySearch.local.maxThreads` setting for local node-llama-cpp embeddings so users can tune CPU thread usage during memory indexing.

## Best Possible Solution

Add a narrow, validated `memorySearch.local.maxThreads` field for the built-in local embedding provider, carry it through default/per-agent resolution and provider option types, pass it to `getLlama` only when configured, and update focused tests plus public/generated config documentation.
