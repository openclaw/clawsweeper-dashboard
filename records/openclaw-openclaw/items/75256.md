## Summary

Keep open: current main still has the Mattermost preview overwrite problem, and this branch is useful but not merge-ready because its block split can race the draft lifecycle and its public config contract is incomplete.

## What This Changes

This PR adds Mattermost `streaming.mode` block preview splitting, opt-in `streaming.toolPreview` args rendering, tool-args propagation to tool-start callbacks, and focused Mattermost schema/account/draft-stream tests.

## Best Possible Solution

Land a Mattermost-local block preview mode that freezes only materialized preview posts, preserves nested streaming defaults across account overrides, and keeps schema, generated metadata, docs, tests, and changelog aligned.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: a Mattermost turn with reasoning, tool starts, and partial replies repeatedly calls `draftStream.update(...)` on one retained preview post, with no boundary call to `forceNewMessage()`.

## Solution Assessment

No, not as currently implemented. The `block` mode direction is narrow and maintainable, but the split operation must be lifecycle-aware and the new public config surface needs merge, docs, generated metadata, and changelog follow-through.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Wait for the preview post before splitting:** `extensions/mattermost/src/mattermost/draft-stream.ts:209`
  - body: `signalBoundary()` treats content as split-ready as soon as a caller queued `draftStream.update()`. The draft loop is throttled and can still have pending text or an in-flight create/update, so `forceNewMessage()` can clear unsent text or race the old create before the previous preview post is materialized. Flush or await the draft lifecycle before resetting the post id.
  - confidence: 0.9
- **[P2] Deep-merge streaming account overrides:** `extensions/mattermost/src/mattermost/accounts.ts:118`
  - body: Mattermost account merging still deep-merges only `commands`. With this PR, an account override like `streaming: { toolPreview: "args" }` replaces a channel-wide `streaming: { mode: "block" }`, silently falling back to partial mode. Add `streaming` to the nested merge keys and cover mixed root/account overrides.
  - confidence: 0.87
- **[P2] Align Mattermost streaming docs and metadata:** `extensions/mattermost/src/config-schema-core.ts:127`
  - body: The new schema accepts object-form `streaming.mode` values `partial` and `block`, plus `streaming.toolPreview`, but the checked-in docs and generated config metadata still describe the old scalar mode surface or omit the new keys. Users following docs or config UI metadata can still get rejected or stripped config unless those public surfaces are updated or compatibility is added.
  - confidence: 0.85
- **[P3] Add the required changelog entry:** `extensions/mattermost/src/types.ts:39-49`
  - body: This PR changes user-facing Mattermost behavior and adds public config options, but the PR head does not update `CHANGELOG.md`. Add a single-line entry under the active section, leaving attribution blank if there is no known non-bot human to credit.
