## Summary

Keep open. The PR addresses a real Mattermost draft-preview overwrite bug, but the latest head is not mergeable as-is because it narrows the current shared streaming config contract, documents an incorrect preview-off path, exposes a misleading raw-args security footgun, and still lacks real Mattermost after-fix proof.

## What This Changes

The PR adds Mattermost draft-preview boundary splitting for block mode, Mattermost streaming/tool-preview config handling, docs/changelog/generated metadata, and focused tests.

## Best Possible Solution

Rebase onto current main, implement Mattermost boundary splitting through the existing shared streaming.mode contract, preserve off/progress/preview/block settings, fix or remove the raw-args preview wording, and add real Mattermost proof.

## Reproduction Assessment

Yes, by source inspection. Current main keeps one Mattermost draft stream post id and updates it across partial, reasoning, and tool callbacks without rotating at block-mode boundaries; I did not run a live Mattermost UI repro in this read-only review.

## Solution Assessment

No. Boundary splitting is the right Mattermost-local fix direction, but this PR should preserve the current shared streaming contract, remove the false preview-off workaround, and either redact raw tool args or document them as a sensitive opt-in before merge.

  - confidence: 0.9

  - confidence: 0.88
