## Summary

Keep open. The Mattermost overwrite bug remains source-reproducible on current main, and this PR is a plausible fix path, but the current head narrows the shared streaming config contract, documents an unusable preview-off workaround, describes raw tool args as sanitized, and lacks real Mattermost proof.

## What This Changes

The PR adds Mattermost draft-preview boundary splitting controlled by streaming mode, a tool-preview verbosity option, docs/changelog/generated metadata, and focused tests.

## Best Possible Solution

Rebase onto current main and implement Mattermost boundary splitting through the existing shared streaming.mode contract while preserving off/progress/preview/block settings, fixing the tool-args wording or redaction, and adding real Mattermost proof.

## Reproduction Assessment

Yes, source-level. Configure Mattermost with active preview streaming and trigger reasoning/tool transitions; current main keeps updating one draft stream without rotating the preview at boundaries, though I did not run a live Mattermost UI repro.

## Solution Assessment

No. Boundary splitting is the right Mattermost-local direction, but this PR must preserve the current shared streaming config shape, remove the false preview-off guidance, and fix the raw-args security wording before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve the shared streaming config shape:** `extensions/mattermost/src/config-schema-core.ts:98-104`
  - body: Current main already accepts Mattermost `streaming` as scalar/boolean/object form with `off`, `partial`, `block`, `progress`, and nested `preview`/`progress`/`block` settings. This schema replaces that with only `{ mode?: "partial" | "block", toolPreview? }`, so supported configs such as `streaming: "off"` or `{ mode: "progress" }` would be rejected after merge.
  - confidence: 0.89
- **[P2] Remove the false preview-off guidance:** `docs/channels/mattermost.md:287`
  - body: The docs say users can disable Mattermost preview streaming by using only block streaming or omitting the streaming object, but this PR head still creates the draft preview stream unconditionally for inbound turns. That guidance would leave users without a working preview-off path and conflicts with current main’s `streaming: "off"` behavior.
  - confidence: 0.9
- **[P2] Do not describe raw tool args as sanitized:** `docs/channels/mattermost.md:291`
  - body: `toolPreview: "args"` is documented as a sanitized summary, but the implementation renders raw command/path/input/text/JSON values and only trims or truncates them. Operators could enable it expecting redaction, so either redact in code or document it as a sensitive opt-in.
  - confidence: 0.88
