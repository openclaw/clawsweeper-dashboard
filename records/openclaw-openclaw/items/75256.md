## Summary

Keep open: the block-mode lifecycle approach addresses the source-reproducible Mattermost preview overwrite problem, but the current PR head still has blocking docs/security-copy issues around preview-off guidance and raw tool-argument previews.

## What This Changes

The PR adds Mattermost `streaming.mode` and `streaming.toolPreview` config, boundary-splitting draft-preview logic, tool-argument preview formatting, docs/changelog updates, and focused Mattermost tests.

## Best Possible Solution

Land the Mattermost-local block preview mode after correcting preview-off guidance and either adding real tool-argument redaction or documenting `toolPreview: "args"` as a raw/truncated sensitive opt-in.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction: Mattermost updates the same draft post id for partial, reasoning, and tool phases, while the boundary reset API is never called by the monitor path.

## Solution Assessment

No, not as currently documented. The lifecycle-aware split is a narrow maintainable direction, but the docs should not advertise an off workaround that code does not implement or describe raw tool args as sanitized.
