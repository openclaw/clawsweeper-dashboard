## Summary

Keep open: current main still has the Mattermost overwrite behavior, but this PR is not merge-ready because its split path can race pending preview delivery and the new config surface is not fully aligned.

## What This Changes

The PR adds Mattermost `streaming.mode` and `streaming.toolPreview` config, passes sanitized tool args through tool-start events, and wires draft preview boundary splitting with focused tests.

## Best Possible Solution

Land a Mattermost-local block preview mode that freezes only delivered preview posts, preserves nested streaming defaults across account overrides, and keeps schema, docs, generated metadata, and changelog aligned.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: a Mattermost turn with reasoning, tool, and partial updates repeatedly calls `draftStream.update(...)` on the same preview post. The PR regression can be reproduced by queuing a draft update and signaling a boundary before the prior preview create or flush resolves.

## Solution Assessment

No, not as currently implemented. The `block` mode direction is narrow and maintainable, but the split operation must be lifecycle-aware and the public config surface needs merge, docs, generated metadata, and changelog follow-through.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Flush the draft before rotating preview posts:** `extensions/mattermost/src/mattermost/draft-stream.ts:209`
  - body: `signalBoundary()` rotates as soon as content was queued with `draftStream.update()`, but the draft loop can still have pending text or an in-flight create/update. `forceNewMessage()` then clears pending state while the old create can still set `streamPostId`, so the next phase can overwrite the old post or lose the queued preview. Make the boundary split wait for or flush the prior draft lifecycle before resetting the post id.
  - confidence: 0.9
- **[P2] Deep-merge streaming account overrides:** `extensions/mattermost/src/mattermost/accounts.ts:118`
  - body: `mergeMattermostAccountConfig()` still lists only `commands` as a nested merge key. With this PR, an account override like `streaming: { toolPreview: "args" }` replaces a channel-wide `streaming: { mode: "block" }`, silently falling back to partial mode. Add `streaming` to the nested merge keys and cover mixed root/account overrides.
  - confidence: 0.88
- **[P2] Align Mattermost streaming docs with the schema:** `extensions/mattermost/src/config-schema-core.ts:127`
  - body: The new schema accepts object-form `streaming.mode` values `partial` and `block`, plus `streaming.toolPreview`, but the checked-in Mattermost and streaming docs still advertise scalar `streaming: "partial"` and unsupported `off`/`progress` modes for Mattermost. Users following those docs will get rejected or stripped config unless the docs are updated or compatibility is added.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `extensions/mattermost/src/types.ts:39-49`
  - body: This PR changes user-facing Mattermost behavior and adds public config options, but the PR head still does not update `CHANGELOG.md`. Add a single-line entry under the active Unreleased section, leaving attribution blank if there is no known non-bot human to credit.
