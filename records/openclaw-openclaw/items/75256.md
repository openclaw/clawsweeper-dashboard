## Summary

Keep this PR open, but it is not merge-ready: the Mattermost block-mode direction addresses a real current-main overwrite problem, while the patch still has a preview lifecycle race and leaves the new config surface inconsistent.

## What This Changes

The PR adds Mattermost `streaming.mode` and `streaming.toolPreview` config, block-boundary draft preview splitting, sanitized tool-argument previews, generated config metadata, and focused tests.

## Best Possible Solution

Land a Mattermost-local block preview mode that freezes only materialized preview posts at phase boundaries, preserves nested streaming defaults across account overrides, and keeps schema, generated metadata, public docs, and changelog aligned.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: a Mattermost turn with reasoning, tool, and partial updates repeatedly edits one draft preview post; the PR race is reproduced by queuing a preview update and signaling a boundary before that update flushes.

## Solution Assessment

No, not as currently implemented. The owner-local `block` mode is the right direction, but the split operation must wait for or generation-guard pending draft lifecycle work, and the public config surface needs merge, docs, and changelog fixes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Wait for the preview post before splitting:** `extensions/mattermost/src/mattermost/draft-stream.ts:209`
  - body: `signalBoundary()` treats content as split-ready as soon as it was queued with `draftStream.update()`. Because the draft loop is throttled and can have an in-flight create/update, `forceNewMessage()` can clear pending text before it is posted or race with a pending create so the next phase still edits the old post. Flush, materialize, or generation-guard the prior preview before rotating.
  - confidence: 0.89
- **[P2] Deep-merge streaming account overrides:** `extensions/mattermost/src/mattermost/accounts.ts:118`
  - body: Mattermost account merging still deep-merges only `commands`. With this PR, an account override such as `streaming: { toolPreview: "args" }` replaces a channel-wide `streaming: { mode: "block" }`, silently falling back to partial mode. Add `streaming` to the nested merge keys and cover mixed root/account overrides.
  - confidence: 0.86
- **[P2] Align Mattermost streaming docs with the schema:** `extensions/mattermost/src/config-schema-core.ts:127`
  - body: The new schema accepts object-form `streaming.mode` values `partial` and `block` plus `toolPreview`, but the checked-in Mattermost/streaming docs still advertise scalar `streaming: "partial"` with `off` and `progress` modes. Users following those docs will get rejected or stripped config unless the docs are updated or compatibility is added.
  - confidence: 0.83
- **[P3] Add the required changelog entry:** `extensions/mattermost/src/types.ts:39-49`
  - body: This PR changes user-facing Mattermost behavior and adds public config. Add a single-line entry under the active version; do not invent contributor attribution if no real human credit is known.
