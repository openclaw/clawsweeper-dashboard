## Summary

Keep open: the PR addresses the source-level Mattermost overwrite path, but its latest head still has two merge-blocking docs/security-copy defects. The docs describe an unusable preview-off workaround and call raw tool argument rendering sanitized.

## What This Changes

The PR adds Mattermost `streaming.mode` block preview splitting, `streaming.toolPreview`, schema/types/account resolution, docs/changelog updates, and focused Mattermost tests.

## Best Possible Solution

Land the Mattermost-local block preview mode with accurate preview-off guidance and either real tool-arg redaction or explicit raw sensitive opt-in documentation.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction: Mattermost sends reasoning, partial replies, and tool status through one draft stream while `draft-stream.ts` edits the existing post id until `forceNewMessage()` clears it, and the monitor never calls that reset.

## Solution Assessment

No, not as currently documented. The block-mode lifecycle approach is narrow and maintainable, but the branch should remove the false preview-off workaround and stop describing raw tool args as sanitized before merge.

- **[P2] Remove the unusable preview-off workaround:** `docs/channels/mattermost.md:287`
  - body: The docs now tell users to disable Mattermost preview streaming by using block streaming, but this PR head still creates a draft preview stream for every inbound turn and passes `disableBlockStreaming: true` into dispatch. That setting disables block replies here, not the draft preview path, so either wire a real preview-off behavior or remove the workaround claim before documenting it.
  - confidence: 0.9
- **[P2] Do not call raw tool args sanitized:** `docs/channels/mattermost.md:291`
  - body: `toolPreview: "args"` is documented as a sanitized one-line summary, but `summarizeMattermostToolArgs()` renders raw command/path/input/text/JSON values, preserves multiline content, and only truncates length. Operators could enable it expecting redaction, so either redact in code or document it as raw sensitive opt-in.
  - confidence: 0.88
