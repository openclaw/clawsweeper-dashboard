Keep open: the PR targets a real current-main Mattermost preview overwrite bug, but the head is stale against the current streaming contract and still has incorrect/security-sensitive docs for preview disabling and raw tool-argument display.

## What This Changes

The PR adds Mattermost draft-preview boundary splitting for block mode, a tool-argument preview option, schema/account resolution/docs/changelog updates, and focused tests.

## Best Possible Solution

Rebase the PR onto current main, wire Mattermost block splitting through the existing `account.streamingMode` contract, preserve `off` and `progress`, and either redact tool args or document `toolPreview: "args"` as raw sensitive opt-in.

## Reproduction Assessment

Yes. Source inspection on current main shows Mattermost `block` preview phases still update one draft stream/post without calling `forceNewMessage()` at assistant/reasoning/tool boundaries.

## Solution Assessment

No, not as currently proposed. The boundary-controller approach is a good Mattermost-local fix, but the branch should build on the current shared streaming contract and correct the preview-off and raw-args documentation before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve the existing streaming contract:** `extensions/mattermost/src/config-schema-core.ts:89-101`
  - body: Current main already accepts Mattermost `streaming: "off"`, `mode: "progress"`, boolean/scalar forms, and nested `preview`/`progress`/`block` settings through the shared resolver. This branch narrows the schema to a Mattermost-only `partial | block` object and adds a separate `previewStreamMode`, so rebasing it as-is would reject documented configs and bypass the current `off`/`progress` behavior. Please wire the split to `account.streamingMode === "block"` and keep the existing schema variants.
  - confidence: 0.9
- **[P2] Remove the unusable preview-off workaround:** `docs/channels/mattermost.md:287`
  - body: The docs say Mattermost preview streaming can be disabled by using only block streaming or omitting the streaming object, but the monitor still creates the draft preview for every non-`off` streaming mode and passes `disableBlockStreaming: true` into dispatch. The documented disable path should be `streaming.mode: "off"`, and this branch needs to preserve that schema support.
  - confidence: 0.92
- **[P2] Do not call raw tool args sanitized:** `docs/channels/mattermost.md:291`
  - body: `toolPreview: "args"` is documented as a sanitized one-line summary, but `summarizeMattermostToolArgs()` renders raw command/path/input/text/JSON values and only trims/truncates length. Operators could enable it expecting redaction, so either redact in code or document it as raw sensitive opt-in.
  - confidence: 0.9
