## Summary

Keep open: the state/report split is directionally right, but this branch currently fails type checking and does not actually carry the new warning-history state through run result metadata. It also lacks required after-fix real behavior proof from a real setup.

## What This Changes

The PR moves bootstrap truncation warning dedupe history out of `systemPromptReport.bootstrapTruncation.warningSignaturesSeen` into a new `SessionEntry.bootstrapPromptWarningState` field across agent, auto-reply, cron, gateway reset, and tests.

## Best Possible Solution

Keep the state/report separation, but first expose the warning history on the produced run metadata, update the session slot-key contract, and add focused regression coverage for CLI and embedded once-mode persistence.

## Reproduction Assessment

Yes, by source inspection and CI. The branch reads a new `EmbeddedPiRunMeta` property that current run results neither type nor populate, and the PR merge check reports the resulting type errors.

## Solution Assessment

No. Separating runtime dedupe state from current-run report metadata is the maintainable direction, but this implementation must carry that state through the actual CLI and embedded result contracts before removing the report-backed history.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

  - body: Adding `bootstrapPromptWarningState` to `SessionEntry` trips the reserved-slot exhaustiveness check because `SESSION_ENTRY_RESERVED_SLOT_KEY_LIST` is unchanged. CI reports TS2344 at `src/plugins/session-entry-slot-keys.ts:129`, so the branch cannot typecheck until the new key is classified there.
  - confidence: 0.98
- **[P1] Emit warning history before reading it from run results:** `src/auto-reply/reply/agent-runner-execution.ts:1595`
  - body: These new calls read `result.meta.bootstrapPromptWarningSignaturesSeen`, but neither CLI nor embedded final run metadata declares or populates that field on current main. CI reports TS2339 on each read, and after the report field is removed the once-mode history would resolve to empty instead of deduping warnings. Carry the state through `PreparedCliRunContext`, `EmbeddedPiRunMeta`, and final result metadata before switching callers.
  - confidence: 0.96
