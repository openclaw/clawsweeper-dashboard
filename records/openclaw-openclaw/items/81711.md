## Summary

Keep open: source inspection shows the branch drops the only CLI-visible once-mode warning history before adding the replacement field to CLI run results. The updated terminal proof covers the intended store/report separation, but not the CLI handoff that would regress.

## What This Changes

The PR moves bootstrap prompt warning dedupe history from `systemPromptReport` into `SessionEntry.bootstrapPromptWarningState`, updates persistence/reset paths, and adjusts tests.

## Best Possible Solution

Keep the state/report separation, but carry the new warning dedupe state through CLI prepared context and CLI result metadata, then add a focused CLI once-mode regression before merge.

## Reproduction Assessment

Yes, by source inspection: the PR reads post-run dedupe state from `result.meta.bootstrapPromptWarningSignaturesSeen`, while CLI results currently expose only `systemPromptReport`. Once the PR removes the report history field, the CLI once-mode state becomes empty after a CLI run.

## Solution Assessment

No. Separating runtime dedupe state from current-run report metadata is the right direction, but this implementation must first pass that state through CLI prepare/result metadata instead of removing the only CLI state source.
