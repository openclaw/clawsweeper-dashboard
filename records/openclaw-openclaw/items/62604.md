## Summary

Keep open. Current main still only handles clipboard image file items; `data:image/...;base64,...` payloads delivered as text/plain or text/html fall through to the browser textarea paste behavior, and the related current image-paste/send report remains open.

## What This Changes

The issue requests hardening the Control UI image paste flow so image data URLs do not become composer text, while keeping send-failure draft restoration centralized and clarifying installed dist hotpatch expectations.

## Best Possible Solution

Add a focused Control UI paste hardening path that converts or cleanly rejects `data:image/...;base64,...` text/html/plain clipboard payloads before they enter the draft, preserve the existing file-backed paste path, keep draft/attachment restoration in the send pipeline, and split any package source-map policy change into a separate maintainer decision.

## Reproduction Assessment

Yes. ## Solution Assessment

Yes. The narrowest maintainable fix is in `ui/src/ui/views/chat.ts` plus regression tests around attachment side-store serialization and send restoration; changing hashed dist/source-map packaging is not needed to fix the observable paste bug.
