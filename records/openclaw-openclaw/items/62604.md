## Summary

Keep open. Current main still only intercepts clipboard file items whose MIME starts with `image/` and returns without `preventDefault()` for text/plain or text/html data-image payloads, while no merged fix or test covers the raw composer base64 failure.

## What This Changes

Prevent Control UI image paste from inserting raw `data:image/...;base64,...` text into the composer and keep send/draft restoration centralized in the send pipeline.

## Best Possible Solution

Add focused Control UI paste hardening for data-image text/html/plain clipboard payloads, preserve existing file-backed attachment and send-restoration behavior, and leave hotpatch/source-map policy to a separate support decision.

## Reproduction Assessment

Yes, at source level. A focused DOM test can dispatch a paste event with only text/plain or text/html containing `data:image/...;base64,...` against the composer textarea and assert current main does not prevent default insertion or convert it to an attachment.

## Solution Assessment

Yes for the central bug. The maintainable fix belongs in `handlePaste` with regression coverage, while draft restoration should remain in the existing send pipeline.
