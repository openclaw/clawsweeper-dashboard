## Summary

Keep open: current main still has the childless Codex-native task maintenance/cancel gap, and the patch is narrowly aimed, but this draft external PR is missing required after-fix real behavior proof and has a small docs mismatch.

## What This Changes

The PR adds Codex-native childless task detection so maintenance marks stale registry-only mirrors lost after 30 minutes and cancellation marks them cancelled without child-session teardown.

## Best Possible Solution

Land a version that keeps the Codex-native registry-only detection narrowly scoped, documents the special stale/cancel semantics, and includes real behavior proof for the cancel and stale-loss paths.

## Reproduction Assessment

Yes, source-reproducible: current main creates childless `codex-native` subagent rows, maintenance treats no `childSessionKey` as backed, and cancellation returns the no-cancellable-child-session reason. I did not run a live stale Codex-native task in this read-only review.

## Solution Assessment

Mostly yes: the helper scopes the exception to `runtime=subagent`, `taskKind=codex-native`, no child session, and Codex-thread identity, which is narrower than changing all childless subagent rows. The patch should also update task docs and add real behavior proof before merge.
