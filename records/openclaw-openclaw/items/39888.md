## Summary

Keep open. The observability goal is useful, but the branch is no longer correct against current main: it patches the old command file, emits an untrusted diagnostic event, reports context usage with provider totals, and has no real after-fix CLI diagnostics proof.

## What This Changes

The PR adds `model.usage` diagnostic emission to the CLI agent command path after session-store updates.

## Best Possible Solution

Add a focused shared usage-diagnostic helper or active-path CLI implementation that matches reply-runner trusted emission, resolved context limits, prompt-context accounting, regression coverage, and redacted live diagnostics proof.

## Reproduction Assessment

Yes. Source inspection on current main shows the active CLI path persists usage through `updateSessionStoreAfterAgentRun()` but has no `model.usage` diagnostic emission, while the reply-runner emits the trusted event.

## Solution Assessment

No. The requested direction is useful, but this patch targets a stale wrapper and diverges from the trusted emitter, context-limit, and prompt-context semantics already used by the diagnostics path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Port the emission to the active CLI path:** `src/commands/agent.ts:1101-1148`
  - body: This block is added to the old in-file command implementation, but current main runs `openclaw agent` through `src/agents/agent-command.ts` and leaves `src/commands/agent.ts` as a re-export. After rebase, this either conflicts or fails to instrument the active CLI path.
  - confidence: 0.96
- **[P2] Use the trusted diagnostic emitter:** `src/commands/agent.ts:1129`
  - body: The PR calls `emitDiagnosticEvent`, which marks the event untrusted. Protected consumers such as `diagnostics-prometheus` drop untrusted events, so core-owned `model.usage` emission should match reply-runner and call `emitTrustedDiagnosticEvent`.
  - confidence: 0.91
- **[P2] Report prompt-context usage:** `src/commands/agent.ts:1144-1147`
  - body: The PR sets `context.used` to provider `totalTokens`, which can include output, cache, and accumulated tool-loop calls. Current docs and tests define this field as the live prompt/context snapshot, so derive it from `promptTokens` or prompt-side `lastCallUsage` instead.
  - confidence: 0.9
- **[P2] Reuse the resolved context limit:** `src/commands/agent.ts:1115-1121`
  - body: The new fallback chain still bypasses runtime `agentMeta.contextTokens` and `resolveContextTokensForModel`, so provider-qualified model config and runtime context metadata can emit stale limits. Reuse the same resolution semantics as session persistence and reply-runner.
  - confidence: 0.85
