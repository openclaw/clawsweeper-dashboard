  - body: This block is added to `src/commands/agent.ts`, but current main leaves that file as a one-line re-export while the active `openclaw agent` body lives in `src/agents/agent-command.ts`. After rebase, this either conflicts or fails to instrument the active CLI path.

  - body: The patch reports `context.used` as provider total tokens, which can include output, cache, and accumulated tool-loop calls. Current docs and reply-runner define this field as the live prompt/context snapshot, so derive it the same way reply-runner does.
  - confidence: 0.9
- **[P2] Reuse the resolved context limit:** `src/commands/agent.ts:1115-1119`
  - body: The fallback chain bypasses runtime `agentMeta.contextTokens` and `resolveContextTokensForModel`, so provider-qualified config and runtime context metadata can emit stale limits. Reuse the same resolution semantics as session persistence and reply-runner.
  - confidence: 0.86
