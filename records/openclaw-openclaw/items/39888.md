## Summary

Keep open. The requested diagnostics coverage is still valid on current main, but this PR is not mergeable as-is because it targets the obsolete command wrapper, emits an untrusted event, reports the wrong context usage semantics, and lacks real after-fix behavior proof.

## What This Changes

The PR adds `model.usage` diagnostic emission for CLI agent runs by modifying `src/commands/agent.ts`.

## Best Possible Solution

Port the instrumentation into `src/agents/agent-command.ts` or a small shared helper, matching the reply-runner's trusted emission, prompt-context accounting, current context-limit resolution, tests, changelog, and redacted real OTEL proof.

## Reproduction Assessment

Yes for source reproduction, though I did not run a live OTEL setup. Current main persists CLI agent usage in the active command path but has no `model.usage` diagnostic emission there, while the reply-runner emits the trusted event.

## Solution Assessment

No. The goal is right, but this patch targets the obsolete wrapper file and does not match current trusted emission, context-limit, or prompt-context semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Port the emission to the active CLI path:** `src/commands/agent.ts:1101-1148`
  - body: This hunk adds the event to the old command implementation, but current main runs `openclaw agent` from `src/agents/agent-command.ts` and leaves `src/commands/agent.ts` as a re-export. After rebase, this either conflicts or does not instrument the active path.
  - confidence: 0.96
- **[P2] Use the trusted diagnostic emitter:** `src/commands/agent.ts:1129`
  - body: The PR calls `emitDiagnosticEvent`, which marks the event untrusted. Core-owned `model.usage` emission should match the reply-runner and call `emitTrustedDiagnosticEvent`; otherwise protected consumers such as diagnostics-prometheus drop it.
  - confidence: 0.9
- **[P2] Report prompt-context usage:** `src/commands/agent.ts:1144-1147`
  - body: The PR sets `context.used` to provider `totalTokens`, which can include output, cache, and multi-call tool-loop totals. Current docs and helpers define this field as the live prompt/context snapshot, so derive it from `promptTokens` or `lastCallUsage` instead.
  - confidence: 0.89
- **[P2] Reuse the resolved context limit:** `src/commands/agent.ts:1115-1121`
  - body: The context-limit fallback bypasses runtime `agentMeta.contextTokens` and `resolveContextTokensForModel`, so configured long-context and provider-qualified model values can emit stale limits. Reuse the same resolution semantics as session persistence and the reply-runner.
  - confidence: 0.84
