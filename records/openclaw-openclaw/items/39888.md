## Summary

Keep open. Current main still has the CLI usage-diagnostics gap, but this PR is stale against the active command path and would not satisfy the current trusted diagnostics/context accounting contract without a port.

## What This Changes

This PR adds a CLI-side `model.usage` diagnostic event to the old `src/commands/agent.ts` flow after session-store updates.

## Best Possible Solution

Add the CLI usage event in `src/agents/agent-command.ts` or a shared usage-diagnostics helper, matching reply-runner/session-store semantics for trust, agent id, context limit, context used, cost, and trace metadata.

## Reproduction Assessment

Yes. Source inspection on current main shows the active CLI command path persists usage but does not emit `model.usage`, while the reply-runner path does emit it through trusted diagnostics.

## Solution Assessment

No. The requested direction is right, but this PR is not the best current fix because it patches the obsolete wrapper and misses current trusted diagnostic, agent-label, and context-accounting behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Port the event to the active command path:** `src/commands/agent.ts:1101-1150`
  - body: Current main moved the real `agentCommandInternal` implementation to `src/agents/agent-command.ts`, while `src/commands/agent.ts` is now only a re-export. This hunk will not cover the active CLI path after rebase, so CLI runs would still miss `model.usage`.
  - confidence: 0.95
- **[P2] Emit the usage event as trusted:** `src/commands/agent.ts:1125-1128`
  - body: This core-owned event calls `emitDiagnosticEvent`, which marks it untrusted. Protected consumers such as diagnostics-prometheus skip untrusted events, and OTEL loses trusted parentage; match reply-runner with `emitTrustedDiagnosticEvent`.
  - confidence: 0.91
- **[P2] Report prompt context usage instead of total tokens:** `src/commands/agent.ts:1144-1146`
  - body: `context.used` is set from `totalTokens`, which can include output, cache, and multi-call/tool-loop totals. Use the prompt-context derivation used by reply-runner so context utilization is not inflated.
  - confidence: 0.88
- **[P2] Reuse resolved context-limit accounting:** `src/commands/agent.ts:1114-1118`
  - body: The fallback chain still bypasses runtime `agentMeta.contextTokens` and `resolveContextTokensForModel`, including configured long-context behavior such as `context1m`. Reuse the session-store/reply-runner accounting before emitting `context.limit`.
  - confidence: 0.86
- **[P3] Include the agent id on CLI usage events:** `src/commands/agent.ts:1127-1131`
  - body: Current `model.usage` consumers expose an agent label, and reply-runner now emits `agentId`. Omitting it here collapses CLI usage metrics to an unknown agent even when the command has a resolved session agent id.
  - confidence: 0.8
