## Summary

Keep open: the CLI telemetry gap still exists on current main, but this PR cannot merge as-is because it targets the pre-refactor command file, emits an untrusted diagnostic event, misreports context usage semantics, and lacks after-fix real behavior proof.

## What This Changes

The branch adds `model.usage` diagnostic emission in `src/commands/agent.ts` after CLI agent session-store updates.

## Best Possible Solution

Port the instrumentation into `src/agents/agent-command.ts` or a small shared helper, matching reply-runner trusted emission, context-limit resolution, prompt-context accounting, focused regression tests, and redacted real diagnostics proof.

## Reproduction Assessment

Yes. Source inspection on current main shows the active CLI command path persists usage through `updateSessionStoreAfterAgentRun()` but has no `model.usage` diagnostic emission, while the reply-runner emits the trusted event.

## Solution Assessment

No. The goal is correct, but the current patch targets an obsolete wrapper path and does not match the trusted emission, context-limit, or prompt-context semantics used by current diagnostics code.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Port the emission to the active CLI path:** `src/commands/agent.ts:1101-1148`
  - body: This hunk adds the event to the old in-file command implementation, but current main runs `openclaw agent` from `src/agents/agent-command.ts` and leaves `src/commands/agent.ts` as a re-export. After rebase, this either conflicts or does not instrument the active path.
  - confidence: 0.96
- **[P2] Use the trusted diagnostic emitter:** `src/commands/agent.ts:1129`
  - body: The PR calls `emitDiagnosticEvent`, which marks the event untrusted. Core-owned `model.usage` emission should match reply-runner and call `emitTrustedDiagnosticEvent`; otherwise protected consumers such as diagnostics-prometheus drop it.
  - confidence: 0.9
- **[P2] Report prompt-context usage:** `src/commands/agent.ts:1144-1147`
  - body: The PR sets `context.used` to provider `totalTokens`, which can include output, cache, and multi-call tool-loop totals. Current docs and helpers define this field as the live prompt/context snapshot, so derive it from `promptTokens` or `lastCallUsage` instead.
  - confidence: 0.89
- **[P2] Reuse the resolved context limit:** `src/commands/agent.ts:1115-1121`
  - body: The context-limit fallback bypasses runtime `agentMeta.contextTokens` and `resolveContextTokensForModel`, so configured long-context and provider-qualified model values can emit stale limits. Reuse the same resolution semantics as session persistence and reply-runner.
  - confidence: 0.84
