## Summary

Keep open: the CLI usage-telemetry gap still exists on current main, but this PR is not mergeable as-is because it targets the pre-refactor command file and emits an untrusted, semantically inaccurate usage event without real behavior proof.

## What This Changes

The PR adds `model.usage` diagnostic event emission to the CLI agent command after session-store updates in `src/commands/agent.ts`.

## Best Possible Solution

Port the telemetry into `src/agents/agent-command.ts` or a small shared helper, using trusted event emission, resolved context-limit semantics, prompt-context accounting, focused tests, and redacted real OTEL proof.

## Reproduction Assessment

Yes. Source inspection shows current main persists CLI agent usage in the active command path but has no `model.usage` diagnostic emission there, while the reply-runner emits the trusted event.

## Solution Assessment

No. The requested direction is right, but this patch targets an obsolete wrapper and does not match current trusted emission, context-limit, or prompt-context semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Port the emission to the active CLI path:** `src/commands/agent.ts:1101-1148`
  - body: This hunk adds the event to the old in-file command implementation, but current main runs `openclaw agent` from `src/agents/agent-command.ts` and leaves `src/commands/agent.ts` as a re-export. After rebase, this either conflicts or does not instrument the active path.
  - confidence: 0.96
- **[P2] Use the trusted diagnostic emitter:** `src/commands/agent.ts:1129`
  - body: The PR calls `emitDiagnosticEvent`, which marks the event untrusted. Core-owned `model.usage` emission should match reply-runner and call `emitTrustedDiagnosticEvent`; otherwise protected consumers such as diagnostics-prometheus drop it.
  - confidence: 0.9
- **[P2] Report prompt-context usage:** `src/commands/agent.ts:1144-1147`
  - body: The PR sets `context.used` to provider `totalTokens`, which can include output, cache, and multi-call tool-loop totals. Current docs and helpers define this field as the live prompt/context snapshot, so derive it from `promptTokens` or `lastCallUsage` instead.
  - confidence: 0.89
- **[P2] Reuse the resolved context limit:** `src/commands/agent.ts:1115-1121`
  - body: The context limit fallback bypasses runtime `agentMeta.contextTokens` and current `resolveContextTokensForModel` semantics, so configured long-context and provider-qualified model values can emit stale limits. Reuse the same resolution semantics as session persistence and reply-runner.
  - confidence: 0.84
