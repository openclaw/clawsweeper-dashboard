## Summary

Keep open: the diagnostics gap is valid, but this PR needs changes because it targets the obsolete command wrapper and diverges from current trusted emission and context-accounting semantics.

## What This Changes

The PR adds CLI-path `model.usage` diagnostic emission to `src/commands/agent.ts` after session-store updates, including usage, cost, context, and CLI channel metadata.

## Best Possible Solution

Port the instrumentation to `src/agents/agent-command.ts` or a shared usage-diagnostics helper with reply-runner parity, regression coverage, and a changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows CLI agent runs persist usage in `src/agents/command/session-store.ts` but do not emit `model.usage`, while the reply-runner path does emit that event.

## Solution Assessment

No. The requested direction is right, but this PR is not the best current patch because it targets the stale wrapper file and does not match trusted emission or prompt-context accounting semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Port the emission to the active command path:** `src/commands/agent.ts:1101-1148`
  - body: Current main has `src/commands/agent.ts` as a re-export to `src/agents/agent-command.ts`, so this hunk targets the old implementation shape. After rebase it will not affect the active `openclaw agent` path, leaving CLI runs without `model.usage`.
  - confidence: 0.96
- **[P2] Use the trusted diagnostic emitter:** `src/commands/agent.ts:1129`
  - body: This is a core-owned usage event, but the PR calls `emitDiagnosticEvent`, which marks it untrusted. Consumers such as diagnostics-prometheus drop untrusted events, so match reply-runner and use `emitTrustedDiagnosticEvent`.
  - confidence: 0.9
- **[P2] Report prompt-context usage instead of provider totals:** `src/commands/agent.ts:1144-1147`
  - body: The PR reports `context.used` from provider total usage, which can include output, cache, and multi-call tool-loop totals. Docs and reply-runner coverage define this field as the live prompt/context snapshot, so derive it from `promptTokens` or `lastCallUsage`.
  - confidence: 0.89
- **[P2] Reuse current context-limit resolution:** `src/commands/agent.ts:1115-1121`
  - body: The fallback chain bypasses runtime `agentMeta.contextTokens` and `resolveContextTokensForModel`, so configured context windows and provider-qualified discovery values can emit a stale `context.limit`.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/commands/agent.ts:1101`
  - body: This is a user-facing diagnostics fix, but the PR only changes code. Add a single-line `CHANGELOG.md` fix entry when porting the implementation.
  - confidence: 0.82
