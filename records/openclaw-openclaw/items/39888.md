## Summary

Keep open: current main still has the CLI usage-diagnostics gap, but this PR needs to be rebased and ported to the active agent command path with trusted/context-compatible diagnostics plus real behavior proof before merge.

## What This Changes

The PR adds a `model.usage` diagnostic emission block to `src/commands/agent.ts` after CLI agent session-store persistence.

## Best Possible Solution

Port the emitter to `src/agents/agent-command.ts` or a shared helper with reply-runner-compatible trust, trace, context-limit, and prompt-context semantics, then add focused regression coverage and redacted real CLI diagnostics proof.

## Reproduction Assessment

Yes. Source inspection shows `openclaw agent` now runs through `src/agents/agent-command.ts`, persists usage after the run, and does not emit `model.usage`, while reply-runner does.

## Solution Assessment

No. The requested behavior is the right direction, but this patch edits the retired wrapper and diverges from current trusted diagnostics, trace, and prompt-context semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Port the emission to the active CLI command:** `src/commands/agent.ts:1101-1148`
  - body: This block is added to the old `src/commands/agent.ts` implementation, but current main runs `openclaw agent` through `src/agents/agent-command.ts` and leaves this file as a re-export. After rebase, this either conflicts or fails to instrument the active CLI path.
  - confidence: 0.96
- **[P2] Use the trusted diagnostic emitter:** `src/commands/agent.ts:1125`
  - body: Core-owned `model.usage` events need to match reply-runner and use `emitTrustedDiagnosticEvent`; this call uses `emitDiagnosticEvent`, which marks the event untrusted and causes trusted-only exporters such as Prometheus to drop it.
  - confidence: 0.91
- **[P2] Report prompt-context usage:** `src/commands/agent.ts:1141-1144`
  - body: The patch reports `context.used` as provider total tokens, which can include output, cache, and accumulated tool-loop calls. Current docs define this field as the live prompt/context snapshot, so it should be derived the same way reply-runner does.
  - confidence: 0.9
- **[P2] Reuse the resolved context limit:** `src/commands/agent.ts:1114-1118`
  - body: The fallback chain bypasses runtime `agentMeta.contextTokens` and `resolveContextTokensForModel`, so provider-qualified config and runtime context metadata can emit stale limits. Reuse the same resolution semantics as session persistence and reply-runner.
  - confidence: 0.86
