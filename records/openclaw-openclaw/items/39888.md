  - body: This is a core-owned usage event, but the PR calls `emitDiagnosticEvent`, which marks it untrusted. Protected consumers such as diagnostics-prometheus drop untrusted events, and OTEL will not use the trace as a trusted parent; match reply-runner and use `emitTrustedDiagnosticEvent`.
  - confidence: 0.9

  - body: The PR sets `context.used` to provider `totalTokens`, which can include output, cache, and multi-call tool-loop usage. Docs and reply-runner tests define this field as the live prompt/context snapshot, so derive it from `promptTokens` or `lastCallUsage` instead.
  - confidence: 0.89

  - body: The fallback chain still bypasses `resolveContextTokensForModel` and runtime `agentMeta.contextTokens` used by session persistence. Configured context windows, provider-qualified discovery values, and long-context settings can therefore emit a stale `context.limit`.
  - confidence: 0.84

  - body: This is a user-facing diagnostics/telemetry fix, but the PR only changes code. Add a single-line `CHANGELOG.md` fix entry when porting the implementation so release notes cover CLI usage visibility.
