## Summary

Keep open. The telemetry gap still exists on current main, but this PR is not merge-ready because it patches the old command file, uses an untrusted diagnostic emitter, and does not match current context accounting.

## What This Changes

The PR adds `model.usage` diagnostic event emission for CLI agent runs by importing usage, context, cost, and diagnostic helpers into `src/commands/agent.ts` and emitting after session-store persistence.

## Best Possible Solution

Port the useful change into the current active CLI path in `src/agents/agent-command.ts` or a shared usage-diagnostics helper. Emit a trusted `model.usage` event after session persistence, reuse the resolved model/provider/context/cost accounting already used by session storage and reply-runner, and add focused CLI regression coverage.

## Reproduction Assessment

Yes. Static current-main inspection is enough: the CLI path updates stored usage without emitting `model.usage`, while the reply-runner path emits it through the diagnostics system.

## Solution Assessment

No. The requested direction is right, but this PR is not the best current fix because it targets an obsolete wrapper and diverges from trusted emission plus context accounting used by the established reply-runner path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Port the emission to the current command implementation:** `src/commands/agent.ts:1104-1151`
  - body: Current main moved the real CLI agent implementation to `src/agents/agent-command.ts`, while `src/commands/agent.ts` is now only a re-export. This hunk will not exercise the active CLI path until it is rebased and ported to the current file or a shared helper.
  - confidence: 0.94
- **[P2] Use the trusted diagnostic emitter:** `src/commands/agent.ts:1128`
  - body: This emits `model.usage` through `emitDiagnosticEvent`, which marks the event untrusted. Protected consumers such as diagnostics-prometheus skip untrusted events, so CLI usage would still be absent from that exporter; match reply-runner and use `emitTrustedDiagnosticEvent`.
  - confidence: 0.9
- **[P2] Report live context usage instead of total provider usage:** `src/commands/agent.ts:1144-1146`
  - body: `context.used` is set to `totalTokens`, but provider totals can include cached input, output, and multi-call/tool-loop usage. The CLI event should derive live context from `promptTokens` or `lastCallUsage`, like reply-runner, so utilization is not inflated.
  - confidence: 0.88
- **[P2] Reuse resolved context limit accounting:** `src/commands/agent.ts:1117-1121`
  - body: The new fallback chain bypasses the current `resolveContextTokensForModel` and runtime `agentMeta.contextTokens` path used by session persistence. Config-driven long-context settings such as `context1m` can therefore emit the wrong `context.limit`.
  - confidence: 0.84
