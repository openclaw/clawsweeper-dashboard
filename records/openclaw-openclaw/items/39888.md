## Summary

Keep open. The underlying CLI diagnostic gap is still source-reproducible on current main, but this PR cannot merge as-is because it patches the old command implementation, lacks required real behavior proof, and diverges from the established trusted `model.usage` semantics.

## What This Changes

The PR adds `model.usage` diagnostic event emission to the old CLI agent command implementation in `src/commands/agent.ts`.

## Best Possible Solution

Port the usage-diagnostic emission into `src/agents/agent-command.ts` or a shared helper that matches reply-runner trust, context-limit resolution, prompt-context accounting, trace handling where appropriate, focused regression coverage, and redacted real CLI/OTEL proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main persists CLI usage through `updateSessionStoreAfterAgentRun()` but has no `model.usage` diagnostic emitter in the active CLI path, while reply-runner emits the trusted event.

## Solution Assessment

No. The requested direction is right, but this patch is not the best current solution because it targets the old command file and diverges from trusted emitter, context-limit, and prompt-context semantics used elsewhere.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Port the emission to the active CLI path:** `src/commands/agent.ts:1101-1150`
  - body: This block is added to the old in-file command implementation, but current main runs `openclaw agent` through `src/agents/agent-command.ts` and leaves this file as a re-export. After rebase, this either conflicts or fails to instrument the active CLI path.
  - confidence: 0.96
- **[P2] Use the trusted diagnostic emitter:** `src/commands/agent.ts:1125`
  - body: The PR calls `emitDiagnosticEvent`, which marks the event untrusted. Prometheus drops untrusted diagnostics, so core-owned `model.usage` emission should match reply-runner and call `emitTrustedDiagnosticEvent`.
  - confidence: 0.91
- **[P2] Report prompt-context usage:** `src/commands/agent.ts:1144-1147`
  - body: The PR sets `context.used` to provider `totalTokens`, which can include output, cache, and accumulated tool-loop calls. Current docs and tests define this field as the live prompt/context snapshot, so derive it from `promptTokens` or prompt-side `lastCallUsage` instead.
  - confidence: 0.9
- **[P2] Reuse the resolved context limit:** `src/commands/agent.ts:1114-1118`
  - body: The fallback chain bypasses runtime `agentMeta.contextTokens` and `resolveContextTokensForModel`, so provider-qualified config and runtime context metadata can emit stale limits. Reuse the same resolution semantics as session persistence and reply-runner.
  - confidence: 0.85
