## Summary

Keep this PR open, but do not merge the current patch. The reported CLI diagnostics gap is still source-reproducible on current main, while the branch targets the retired command body, uses the untrusted emitter, reports the wrong context-used value, and lacks after-fix real CLI/OTEL proof.

## What This Changes

The PR adds `model.usage` diagnostic event emission for CLI agent runs by computing usage, cost, and context fields in `src/commands/agent.ts`.

## Best Possible Solution

Port the emission to `src/agents/agent-command.ts` or a shared helper that matches reply-runner trust, context-limit resolution, prompt-context accounting, trace handling where applicable, focused regression coverage, and redacted real CLI diagnostics proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main persists CLI usage through `updateSessionStoreAfterAgentRun()` but has no active CLI `model.usage` emitter, while reply-runner emits the trusted event. I did not execute a live CLI/OTEL run in this read-only review.

## Solution Assessment

No. The requested direction is right, but this patch is not the best current solution because it edits the retired command body and diverges from trusted emitter, context-limit, and prompt-context semantics.
