Yes, source-level. Current main persists CLI agent usage in the active command path but has no `model.usage` emission there, while the reply-runner path emits the trusted event.

Full review comments:

- **[P1] Port the emission to the active command path:** `src/commands/agent.ts:1101-1148`
  - body: This hunk adds the diagnostic event to the old in-file command implementation, but current main makes `src/commands/agent.ts` a one-line re-export and runs the CLI command from `src/agents/agent-command.ts`. After rebase this either conflicts or leaves the active `openclaw agent` path without `model.usage` emission.
  - confidence: 0.96
- **[P2] Use the trusted diagnostic emitter:** `src/commands/agent.ts:1129`
  - body: This is a core-owned usage event, but the PR calls `emitDiagnosticEvent`, which marks it untrusted. Protected consumers such as diagnostics-prometheus ignore untrusted events, so the CLI path should match reply-runner and call `emitTrustedDiagnosticEvent`.
  - confidence: 0.9
- **[P2] Report prompt-context usage, not provider totals:** `src/commands/agent.ts:1144-1147`
  - body: The PR sets `context.used` to provider `totalTokens`, which can include output, cache, and multi-call tool-loop totals. Docs and reply-runner code define this field as the live prompt/context snapshot, so derive it from `promptTokens` or `lastCallUsage` instead.
  - confidence: 0.89
- **[P2] Resolve the current context limit:** `src/commands/agent.ts:1115-1121`
  - body: The fallback chain bypasses runtime `agentMeta.contextTokens` and `resolveContextTokensForModel`, so configured context windows, provider-qualified discovery values, and long-context settings can emit a stale `context.limit`. Reuse the same resolution semantics as session persistence/reply-runner.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/commands/agent.ts:1101`
  - body: This is a user-facing diagnostics/telemetry fix, but the PR only changes code. Add a single-line `CHANGELOG.md` fix entry when porting the implementation so release notes cover CLI usage visibility.
  - confidence: 0.82
