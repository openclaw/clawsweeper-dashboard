## Summary

Keep open: the CLI usage telemetry gap still exists on current main, but this patch is stale against the active command path and would misreport or hide some diagnostics if applied as-is.

## What This Changes

The PR adds CLI `model.usage` diagnostic emission to the old `src/commands/agent.ts` agent command implementation after session-store updates.

## Best Possible Solution

Port the instrumentation to `src/agents/agent-command.ts` or a shared helper, matching reply-runner trusted emission, context accounting, tests, and changelog coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows CLI agent runs persist usage in `src/agents/agent-command.ts` but do not emit `model.usage`, while the reply-runner path does emit that event.

## Solution Assessment

No. The requested direction is right, but this PR is not the best current fix because it targets the obsolete wrapper file and does not match current trusted emission or prompt-context accounting semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Port the emission to the active command path:** `src/commands/agent.ts:1101-1148`
  - body: Current main moved the active implementation to `src/agents/agent-command.ts`, while `src/commands/agent.ts` is now only a re-export. This hunk targets the old implementation shape, so after rebase it will not instrument `openclaw agent` runs.
  - confidence: 0.96
- **[P2] Use the trusted diagnostic emitter:** `src/commands/agent.ts:1129`
  - body: This core-owned usage event calls `emitDiagnosticEvent`, which marks it untrusted. Protected consumers such as diagnostics-prometheus ignore untrusted events, so CLI usage would still be absent there; match reply-runner and use `emitTrustedDiagnosticEvent`.
  - confidence: 0.9
- **[P2] Report prompt-context usage, not provider totals:** `src/commands/agent.ts:1144-1147`
  - body: The patch sets `context.used` to provider `totalTokens`, but docs and reply-runner behavior define this field as the live prompt/context snapshot. Derive it from prompt tokens or `lastCallUsage` so cached input, output, and tool-loop calls do not inflate context utilization.
  - confidence: 0.89
- **[P2] Reuse current context-limit resolution:** `src/commands/agent.ts:1115-1121`
  - body: The context limit fallback bypasses the current `resolveContextTokensForModel` path and runtime `agentMeta.contextTokens` handling used by session persistence. Configured context windows and provider-qualified discovery values can therefore emit a stale `context.limit`.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/commands/agent.ts:1101`
  - body: This is a user-facing diagnostics fix, but the PR changes only code. Add a single-line `CHANGELOG.md` fix entry when porting the implementation so release notes cover CLI usage visibility.
  - confidence: 0.82
