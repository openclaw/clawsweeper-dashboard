## Summary

Keep open. The underlying CLI diagnostics gap is still source-reproducible on current main, but this branch is not mergeable as-is because it targets a retired command body, misses current trusted/context semantics, and lacks real behavior proof.

## What This Changes

The PR adds a `model.usage` diagnostic event emission block to the old `src/commands/agent.ts` CLI path after session-store updates.

## Best Possible Solution

Port a reply-runner-compatible trusted usage emission to `src/agents/agent-command.ts` or a shared helper, add focused regression coverage, and require redacted real CLI diagnostics proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `openclaw agent` uses `src/agents/agent-command.ts`, persists usage through `updateSessionStoreAfterAgentRun()`, and has no active CLI `model.usage` emitter while reply-runner already emits one.

## Solution Assessment

No. The requested direction is right, but this patch edits the retired `src/commands/agent.ts` command body and should instead reuse the active command path or a shared reply-runner-compatible diagnostic helper.

## Review Findings

  - body: Current main leaves this file as a re-export to `src/agents/agent-command.ts`, and that active command path still only persists usage. After rebase, this added block either conflicts or does not instrument `openclaw agent`.

  - confidence: 0.9

  - body: This fallback bypasses `resolveContextTokensForModel` and runtime `agentMeta.contextTokens`, so provider-qualified config and runtime context metadata can emit stale limits.
  - confidence: 0.86
