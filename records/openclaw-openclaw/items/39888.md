## Summary

## What This Changes

The branch adds a `model.usage` diagnostic emission block to `src/commands/agent.ts` for `openclaw agent` runs, including usage, cost, context, duration, and `channel: "cli"` fields.

## Best Possible Solution

Port the emitter to `src/agents/agent-command.ts` or a shared helper that matches reply-runner trust, trace, context-limit, and prompt-context semantics, then add focused regression coverage and redacted real CLI diagnostics proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `openclaw agent` runs through `src/agents/agent-command.ts`, persists usage after the run, and has no CLI `model.usage` emitter while reply-runner already emits one.

## Solution Assessment

No. The requested behavior is the right direction, but this patch edits the retired command body and should instead instrument the active command path with reply-runner-compatible diagnostics semantics.

  - confidence: 0.9

  - body: The fallback chain bypasses runtime `agentMeta.contextTokens` and `resolveContextTokensForModel`, so provider-qualified config and runtime context metadata can emit stale limits. Reuse the same resolution semantics as session persistence and reply-runner.
  - confidence: 0.86
