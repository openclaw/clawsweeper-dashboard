## Summary

Keep open: the reported installer bug is source-reproducible and the proposed seam is plausible, but the patch leaves the service audit/doctor path out of sync for Volta/pnpm-style installs and still needs post-patch real behavior proof before merge.

## What This Changes

The PR adds helpers that find the install-time `openclaw` binary directory on PATH, include it with the Node bin directory in daemon service `extraPathDirs`, updates gateway and node-host install planning, and adds regression coverage plus a changelog entry.

## Best Possible Solution

Land a version that adds only the resolved OpenClaw CLI bin directory, teaches service audit/doctor to accept that exact generated entry, and includes redacted post-patch macOS proof of the regenerated service PATH and child-process CLI resolution.

## Reproduction Assessment

Yes. Current main excludes darwin user bin directories and passes only the Node bin as an extra service PATH dir, and the PR body supplies real macOS 2026.5.18 env output showing the missing OpenClaw bin directory.

## Solution Assessment

No, not yet. The install-planning seam is the right place to fix this, but the patch should also align service audit/doctor behavior and prove the patched service in a real macOS install.
