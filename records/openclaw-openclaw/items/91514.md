## Summary

Keep open: current main and v2026.6.9 still only tolerate rejected ACP timeout config writes, while persisted/default `thinking` can still map to Claude ACP `effort`; the direct effort fix PR is closed unmerged and the broader Claude ACP PRs still have active compatibility blockers.

## What This Changes

The issue reports that Claude Haiku ACP sessions can fail when OpenClaw sends a mapped `effort` config option unsupported by that model.

## Best Possible Solution

Make ACP pre-turn thinking controls model/capability-aware or explicitly best-effort for inherited/default thinking, while preserving strict failures for required model and permission controls.

## Reproduction Assessment

Yes, from source inspection rather than a live run: persisted/default ACP `thinking` can map to `effort`, and current main only catches unsupported timeout config writes before `runTurn`. The pinned Claude ACP dependency throws when a config id is absent from the active session's model-specific options.

## Solution Assessment

No, current main is not the best end state because it still treats pre-turn thinking/effort rejection as fatal. The safest fix shape needs maintainer confirmation on whether unsupported thinking is optional only for inherited/default controls or for all pre-turn thinking controls.
