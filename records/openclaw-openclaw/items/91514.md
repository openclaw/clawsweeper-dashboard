## Summary

Keep open: current main and v2026.6.8 still only tolerate rejected ACP timeout config writes, while persisted/default `thinking` can still map to Claude ACP `effort`; the direct fix PR is closed unmerged and the broader Claude ACP PRs are still not safe canonical fixes.

## What This Changes

The issue asks OpenClaw to avoid failing Claude ACP Haiku sessions when a default `thinking` level is mapped and sent as the `effort` config option.

## Best Possible Solution

Make ACP pre-turn thinking controls model/capability-aware, or explicitly best-effort for inherited/default thinking, while preserving strict failures for controls maintainers still consider required.

## Reproduction Assessment

Yes, from source inspection rather than a live run: persisted/default ACP `thinking` can map to `effort`, and current main only catches unsupported timeout config writes. The pinned Claude ACP dependency throws when a config id is absent from the active session's model-specific options.

## Solution Assessment

No, current main is not the best end state because it still treats pre-turn thinking/effort rejection as fatal. The safest fix shape needs maintainer confirmation on whether unsupported thinking is optional only for inherited/default controls or for all pre-turn thinking controls.
