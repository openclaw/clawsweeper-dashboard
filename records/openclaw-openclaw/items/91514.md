## Summary

Keep open: current main and the latest release still only tolerate rejected ACP timeout config writes, while persisted/default `thinking` can still map to Claude ACP `effort`; the direct fix PR is closed unmerged and the broader Claude ACP PRs are not safe canonical fixes yet.

## What This Changes

Report asks OpenClaw to avoid failing Claude Haiku ACP sessions when a default thinking setting is sent as unsupported `effort` config.

## Best Possible Solution

Make ACP pre-turn thinking controls model/capability-aware, or explicitly best-effort for inherited/default thinking, while preserving strict failures for controls maintainers still consider required.

## Reproduction Assessment

Yes, source-reproducible: current main maps persisted/default ACP `thinking` to advertised `effort`, and `applyManagerRuntimeControls` only treats timeout config rejection as optional. The published Claude ACP adapter rejects absent config ids and only advertises `effort` for models with supported effort levels.

## Solution Assessment

Unclear. The narrow symptom is clear, but the best permanent fix needs maintainer direction on whether unsupported thinking should be optional only for inherited/default controls, handled in ACPX, or resolved through model-aware config capability checks.
