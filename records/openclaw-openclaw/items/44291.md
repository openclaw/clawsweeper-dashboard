## Summary

Keep open because current main still has no native PowerShell CI smoke for contributor commands. The Windows CI lane remains a bash-dispatched test-only job, while `pnpm check:docs` is covered by Ubuntu docs jobs; the related `check:docs` shell-agnostic script work helps the command itself but does not add the requested PowerShell coverage.

## What This Changes

Request a lightweight Windows native PowerShell CI smoke for high-value contributor commands such as `pnpm check:docs` so shell-specific regressions are caught before Windows contributors hit them locally.

## Best Possible Solution

Add a narrowly scoped native PowerShell smoke in the existing Windows CI surface for `pnpm check:docs`, keep the current Windows test task intact, document the guarantee in the CI docs, and cover the workflow routing so future edits cannot silently drop the smoke.
