## Summary

Keep open: current main still lacks the requested native PowerShell contributor-command CI smoke. The docs formatter itself is now shell-agnostic, but the CI graph still runs `pnpm check:docs` only in the Ubuntu docs job while the Windows lane remains a bash-dispatched `test` task.

## What This Changes

Add a lightweight Windows PowerShell CI smoke for contributor commands such as `pnpm check:docs`.

## Best Possible Solution

Add a bounded native PowerShell smoke path to `checks-windows` for `pnpm check:docs`, keep the existing Windows test task intact, and document the CI guarantee in `docs/ci.md`.

## Solution Assessment

Yes, the requested direction is a narrow maintainable fix if it stays small. The safer path is a native PowerShell `pnpm check:docs` smoke in the existing Windows lane, not duplicating the full Windows matrix or adding heavier build smoke by default.
