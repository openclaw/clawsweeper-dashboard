## Summary

Keep open: current main still does not have a native PowerShell CI smoke lane for contributor commands; existing Windows CI is a bash-dispatched focused test lane, while the related docs-format issue and PR are narrower than this coverage request.

## What This Changes

Request to add a lightweight Windows native PowerShell CI smoke lane for high-value contributor commands such as `pnpm check:docs` and `pnpm build:strict-smoke`.

## Best Possible Solution

Add a deliberately small native `pwsh` Windows CI task for maintainer-selected commands, keep it separate from `pnpm test:windows:ci`, and prove it on a real Windows runner.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a CI coverage request. Source inspection gives high confidence that current main does not run selected contributor commands under native PowerShell in CI.

## Solution Assessment

Yes directionally, but not as an automated patch until maintainers choose the command set and runner budget. The maintainable direction is a tiny native `pwsh` smoke lane with Windows-runner proof.
