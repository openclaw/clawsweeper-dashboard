## Summary

Keep open. Current main still lacks the requested native PowerShell contributor-command CI smoke: Windows CI only runs the existing Windows test task, the job defaults to bash for normal commands, and `pnpm check:docs` remains covered only by the Ubuntu docs job.

## What This Changes

The issue requests a small CI lane that runs selected contributor commands under native Windows PowerShell, starting with `pnpm check:docs`.

## Best Possible Solution

Add a bounded native PowerShell smoke path to the existing `checks-windows` job for `pnpm check:docs`, keep the current Windows test task intact, and document the guarantee in `docs/ci.md` with a small workflow/routing guard so the smoke cannot be dropped silently.

Not applicable as a product bug reproduction; this is a CI coverage gap. The gap is high-confidence from static workflow inspection: current main has no `pnpm check:docs` step running with `shell: pwsh`.

## Solution Assessment

Yes, the requested direction is the narrowest maintainable fix if it stays small. The safer shape is an existing Windows matrix/task addition for `pnpm check:docs`, not duplicating the full Windows matrix or adding heavier build smoke by default.
