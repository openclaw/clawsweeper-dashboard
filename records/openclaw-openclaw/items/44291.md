## What This Changes

The issue requests a small native Windows PowerShell CI smoke lane for selected contributor-facing commands such as docs checks or strict build smoke commands.

## Best Possible Solution

Add a deliberately small native `pwsh` Windows CI task for maintainer-selected contributor commands, keep it separate from `pnpm test:windows:ci`, and prove it on a real Windows runner.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a CI coverage request. Source inspection gives high confidence that current main still does not run selected contributor commands under native PowerShell in CI.

## Solution Assessment

Yes directionally, but not as an automated patch until maintainers choose the command set and runner budget. The maintainable direction is a tiny native `pwsh` smoke lane with Windows-runner proof.
