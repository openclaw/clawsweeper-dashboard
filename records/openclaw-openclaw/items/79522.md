## What This Changes

## Best Possible Solution

Capture a redacted real Codex/acpx event stream for the progress-only completion, then fix the acpx/Codex ACP adapter or runtime result contract so premature stop becomes `failed` or `timed_out` while legitimate text-only completions still succeed.
