## Summary

Keep open: current main still treats an ACP stream that emits progress text followed by `done` as a successful background task, while the only attempted PR for this issue was closed after reviewer and contributor agreed the core progress-text heuristic was unsafe.

## What This Changes

The issue asks OpenClaw to stop reporting ACP/Codex spawned runs as successful when the child only emits initial progress text and does not perform or report the requested work.

## Best Possible Solution

Capture the real Codex/acpx event stream, then fix the adapter or runtime contract so premature close or failure becomes `ACP_TURN_FAILED` while legitimate text-only `done` completions still succeed.

## Reproduction Assessment

No high-confidence live reproduction is available yet. Source inspection gives a credible current-main path for `text_delta` followed by `done` to become `succeeded`, but the real Codex/acpx event stream from the report is still missing.

## Solution Assessment

Unclear until the adapter stream is captured. The closed PR's core progress-text heuristic is not the best fix; the safer direction is adapter/runtime-level failure signaling for premature termination.
