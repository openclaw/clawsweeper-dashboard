## Summary

Keep open: current main and v2026.6.10 still lack the requested structured `sessions_send` action/routing parameters, while A2A suppression still depends on exact sentinel text; the narrower announce threadId fallback has been fixed, but the central handoff contract remains unresolved.

## What This Changes

The issue requests structured `sessions_send` handoff actions plus channel/thread context inheritance and routing overrides for A2A announce delivery.

## Best Possible Solution

Define one backward-compatible A2A handoff contract that carries structured control intent and routing context, keeps shipped sentinel compatibility where needed, and lands with focused sessions_send, announce, docs, and regression coverage.

## Reproduction Assessment

Yes, at source level: current main lacks structured `sessions_send` action/routing parameters, and the skip helpers still require exact sentinel text. I did not run a live Telegram handoff because the missing public surface is directly visible in source, docs, and the latest release tag.

## Solution Assessment

Unclear as a final API shape. A structured handoff contract is the right maintainable direction, but maintainers should settle how it relates to announceTarget, dispatch-only mode, private delivery, and handoff acknowledgements before implementation.
