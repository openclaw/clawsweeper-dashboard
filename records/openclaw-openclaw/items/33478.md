## Summary

Keep open. Current main still lacks the requested structured `sessions_send` action/announce override surface, and a recent maintainer comment explicitly keeps this as acceptance criteria for the A2A handoff protocol work; only the narrower threadId preservation slice has landed.

## What This Changes

Request structured callback actions plus inherited and overrideable thread-aware announce routing for agent-to-agent `sessions_send` handoffs.

## Best Possible Solution

Fold this into the first-party A2A handoff contract with backward-compatible structured skip/no-reply/dispatch outcomes, inherited and overrideable announce routing, docs, and regression tests.

## Reproduction Assessment

Yes, by source inspection for the reported fragility: mixed text such as `Task complete. REPLY_SKIP` fails the exact-token helpers, and the `sessions_send` schema has no structured action or announce override to bypass model text. The threadId loss slice is partially covered by current tests and code.

## Solution Assessment

Yes at the direction level. Structured control outcomes are more maintainable than loosening magic-string parsing, but the implementation should land through the coordinated A2A handoff protocol rather than as an isolated `sessions_send` tweak.
