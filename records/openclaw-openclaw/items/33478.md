## Summary

Keep open. Current main still lacks the requested structured `sessions_send` action and announce override API, the A2A path still relies on exact control-token text, and maintainer discussion keeps this as acceptance criteria for the open handoff protocol work.

## What This Changes

The issue requests structured `sessions_send` callback actions plus inherited and overrideable thread/channel context for A2A handoff announce turns.

## Best Possible Solution

Fold this into the first-party A2A handoff contract with backward-compatible structured skip/no-reply/dispatch outcomes, inherited and overrideable announce routing, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: mixed text such as `Task complete. REPLY_SKIP` fails the exact-token helpers, and the current `sessions_send` schema has no structured action or announce override field to bypass model text.

## Solution Assessment

Yes at the direction level. Structured control outcomes are more maintainable than loosening magic-string parsing, but the implementation should land through the coordinated A2A handoff protocol rather than as an isolated parser tweak.
