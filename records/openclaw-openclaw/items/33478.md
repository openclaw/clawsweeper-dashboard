## Summary

Keep open: current main and v2026.5.22 still lack the requested structured `sessions_send` skip/action and announce override contract. The threadId announce-routing slice is partly covered, but maintainer discussion and related open handoff PRs show the structured outcome and protocol/API decision remain unresolved.

## What This Changes

The issue requests a structured `sessions_send` skip/action contract plus inherited or explicit thread-aware announce routing for agent handoffs.

## Best Possible Solution

Define one backward-compatible A2A handoff outcome contract with structured skip/no-reply controls, legacy token compatibility, inherited thread-aware announce routing, optional override fields, docs, and regression coverage for chatty control-token output.

## Solution Assessment

Yes at the direction level, but not as an isolated regex or schema tweak. The maintainable path is coordinated A2A handoff protocol work that preserves legacy tokens while adding structured outcomes and thread-aware announce routing.
