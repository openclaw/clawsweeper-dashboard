## Summary

Keep open. Current main still lacks the requested structured `sessions_send` action and announce routing override fields, and the A2A lifecycle still depends on exact magic-token text; the thread-routing slice has partial fixes, but maintainer discussion explicitly keeps this as acceptance criteria for the broader handoff protocol work.

## What This Changes

Request adds structured `sessions_send` skip/announce controls plus routing context inheritance and overrides for agent handoff announce delivery.

## Best Possible Solution

Fold this into the first-party A2A handoff contract with backward-compatible structured skip/no-reply/dispatch outcomes, inherited and overrideable announce routing, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: mixed text such as `Task complete. REPLY_SKIP` fails the exact-token helpers, and the current `sessions_send` schema has no structured action or announce override field to bypass model text.

## Solution Assessment

Yes at the direction level: structured control is more maintainable than loosening magic-string parsing. The implementation should land through the coordinated A2A handoff protocol rather than as an isolated parser tweak.
