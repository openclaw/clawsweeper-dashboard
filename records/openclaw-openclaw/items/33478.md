## Summary

Keep this open. Current main partly covers announce threadId preservation, but the requested structured sessions_send action/outcome contract and explicit announce routing overrides are still missing; the issue discussion also has explicit maintainer guidance to keep this as A2A handoff acceptance criteria.

## What This Changes

This issue requests a backward-compatible structured sessions_send action/outcome API with skip/announce routing controls and thread-context inheritance for agent-to-agent handoffs.

## Best Possible Solution

Keep this as acceptance criteria for first-party A2A handoff work: add a backward-compatible structured sessions_send outcome/action contract for skip/no-reply/announce suppression, explicit announce target/thread overrides, legacy token compatibility, and regression coverage for token chatter plus threaded announce routing.
