## Summary

Keep open: current main still lacks the requested structured `sessions_send` skip/action and announce override contract, and a maintainer comment explicitly keeps this as A2A handoff acceptance criteria. The thread-routing portion is partially fixed, but the structured outcome surface remains unresolved.

## What This Changes

The issue requests structured `sessions_send` skip/action callbacks plus inherited or overridden thread context for A2A announce delivery.

## Best Possible Solution

Define a backward-compatible A2A handoff outcome contract with structured skip/no-reply controls, legacy token compatibility, thread-aware announce routing, override fields, and regression coverage for chatty control-token output plus inherited or overridden targets.

## Reproduction Assessment

Yes for source-level behavior: current main lacks the requested structured action and announce override fields, while the A2A loop still consumes exact text tokens. I did not run a live Telegram/A2A reproduction in this read-only review.

## Solution Assessment

Yes at the direction level, but not as an isolated regex or schema tweak. The maintainable path is coordinated A2A handoff protocol work that preserves legacy tokens while adding structured outcomes and thread-aware announce routing.
