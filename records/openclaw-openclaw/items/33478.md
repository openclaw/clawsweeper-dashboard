## Summary

Keep this open. Current main still has only partial thread-context fixes for the announce path; it does not implement the requested structured `sessions_send` action/outcome contract or explicit announce routing overrides, and a maintainer comment explicitly says this should remain acceptance criteria for the broader A2A handoff work.

## What This Changes

Request a backward-compatible structured `sessions_send` callback/action contract plus channel/thread inheritance and explicit announce routing overrides for agent handoffs.

## Best Possible Solution

Keep this as acceptance criteria for first-party A2A handoff work: add a backward-compatible structured `sessions_send` outcome/action contract for skip/no-reply/dispatch/announce suppression, explicit announce target/thread overrides, inherited channel/thread context, legacy token compatibility, docs, and regression tests for token chatter plus threaded announce routing.

## Reproduction Assessment

Yes. The source-level reproduction is deterministic: a reply such as `Task complete. REPLY_SKIP` does not satisfy the exact-token helper, and `sessions_send` currently has no structured action or routing override fields to bypass that model-text contract.

## Solution Assessment

Yes, the requested direction is the maintainable solution, but it should land as part of the A2A handoff contract rather than a loose token substring parser. A structured control result keeps backward compatibility while removing model-chatter fragility and making routing explicit.
