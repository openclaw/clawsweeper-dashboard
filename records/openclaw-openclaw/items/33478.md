## Summary

Keep open: current main and v2026.6.5 preserve thread context for `sessions_send` announcements, but the central typed terminal-outcome request remains missing because A2A still depends on exact model-emitted `REPLY_SKIP` and `ANNOUNCE_SKIP` text. Maintainer discussion also points this issue at the open handoff-protocol work instead of cleanup closure.

## What This Changes

Request to add structured `sessions_send` terminal actions plus inherited or explicit thread routing for agent handoff announcements.

## Best Possible Solution

Keep automatic thread preservation as the routing source of truth, then define one typed terminal-outcome contract inside the approved A2A handoff design; add explicit routing overrides only if maintainers make them the single supported escape hatch.

## Reproduction Assessment

Yes, at source level. Current main still has no typed `sessions_send` terminal action and still requires exact `REPLY_SKIP`/`ANNOUNCE_SKIP` model text, while source and tests show the separate thread-loss path has been repaired.

## Solution Assessment

No, not exactly as proposed. Typed terminal outcomes are the maintainable direction, but the parameter names, compatibility policy, and routing overrides should be decided in the canonical handoff protocol rather than added as a parallel API.
