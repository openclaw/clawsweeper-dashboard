## Summary

Keep open. Current main still lacks structured `sessions_send` action and announce override fields, exact control-token handling still makes chatty `REPLY_SKIP` text a non-structured outcome, and the linked A2A handoff discussion explicitly keeps this as acceptance criteria; only part of the threadId preservation slice is already covered.

## What This Changes

Request structured `sessions_send` skip/announce actions plus inherited or overridable channel/thread context for agent-to-agent handoff announce delivery.

## Best Possible Solution

Define a backward-compatible A2A handoff outcome contract with structured skip/no-reply controls, keep legacy tokens, preserve thread-aware announce routing, and add regression tests for chatter around control tokens plus thread inheritance.

## Reproduction Assessment

Yes for source-level current behavior: the tool schema lacks the requested structured action and announce override fields, while the A2A loop still consumes exact text tokens. I did not run a live Telegram/A2A reproduction in this read-only review.

## Solution Assessment

Yes at the direction level, but not as an isolated regex or schema tweak. The maintainable path is coordinated A2A handoff protocol work that preserves legacy tokens while adding structured outcomes and thread-aware announce routing.
