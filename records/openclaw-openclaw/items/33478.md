## Summary

Keep open. Current main and the latest release still expose token-driven `sessions_send` A2A behavior without structured callback actions or announce override parameters, and the live discussion keeps this as acceptance criteria for the A2A handoff protocol work at https://github.com/openclaw/openclaw/pull/63789.

## What This Changes

Add structured `sessions_send` callback controls and preserve or override thread-aware announce routing for agent handoffs.

## Best Possible Solution

Fold structured skip/no-reply outcomes and inherited or overrideable announce routing into the first-party A2A handoff contract, preserving legacy tokens while adding docs and focused regression tests.

## Reproduction Assessment

Yes for source-level current behavior: `sessions_send` lacks structured action and announce override fields, and the A2A control path still uses exact-token helpers. This is a feature/API gap rather than a broken existing documented contract.

## Solution Assessment

Yes at the direction level. Structured control outcomes are more maintainable than broadening magic-string parsing, but the implementation should land through coordinated A2A handoff protocol work rather than an isolated schema tweak.
