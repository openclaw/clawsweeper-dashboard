## Summary

Keep open. Current main still lacks the requested structured sessions_send control action and announce override API, while the A2A path remains driven by exact control-token text; maintainer discussion also keeps this as acceptance criteria for the open handoff protocol work.

## What This Changes

Request structured sessions_send handoff controls plus inherited or overrideable announce routing so agent handoffs do not depend on exact REPLY_SKIP text or lose thread context.

## Best Possible Solution

Fold this into the first-party A2A handoff contract with backward-compatible structured skip/no-reply/dispatch outcomes, inherited and overrideable announce routing, docs, and regression tests.

## Reproduction Assessment

Yes for the current behavior gap by source inspection: mixed text such as `Task complete. REPLY_SKIP` fails the exact-token helpers, and the sessions_send schema has no structured action or announce override field to bypass model text.

## Solution Assessment

Yes at the direction level. Structured control outcomes are more maintainable than loosening magic-string parsing, but the implementation should land through the coordinated A2A handoff protocol rather than as an isolated parser tweak.
