## Summary

Keep open: current main and the latest release still expose sentinel-text A2A skip semantics for `sessions_send`, while only the narrower threadId announce fallback has been fixed. ## What This Changes

The issue requests structured A2A handoff controls for skip/no-reply behavior and preserved channel/thread context during announce delivery.

## Best Possible Solution

Define one default-compatible model-facing A2A handoff contract that carries structured control intent and routing context, preserves existing exact-token compatibility where needed, and keeps it aligned with the related announceTarget and dispatch-mode issues.

## Reproduction Assessment

Yes, at source level: current main lacks structured sessions_send action/routing parameters, and the skip helpers still require exact sentinel text. I did not run a live Telegram handoff because the missing public surface is directly visible in source and docs.

## Solution Assessment

Unclear as an exact API shape. A structured action/routing contract is a plausible maintainable direction, but maintainers should settle how it relates to announceTarget, dispatch-only mode, and skip/suppress policy before implementation.
