## Summary

Keep open. Current main still exposes token-driven `sessions_send` A2A behavior without structured callback actions or announce override parameters, and recent project review context keeps this as acceptance criteria for the A2A handoff protocol work; only the narrower threadId preservation slice has landed.

## What This Changes

Requests structured `sessions_send` callback actions plus inherited or overrideable announce routing/thread context for agent handoffs.

## Best Possible Solution

Fold structured skip/no-reply outcomes and inherited or overrideable announce routing into the first-party A2A handoff contract, preserving legacy tokens while adding docs and focused regression tests.

## Reproduction Assessment

Yes for the current behavior by source inspection: `sessions_send` lacks structured action and announce override fields, and control tokens are still exact full-text checks. This is a feature/API gap rather than a broken existing documented contract.

## Solution Assessment

Yes at the direction level. Structured control outcomes are more maintainable than broadening magic-string parsing, but the implementation should land through the coordinated A2A handoff protocol instead of an isolated schema tweak.
