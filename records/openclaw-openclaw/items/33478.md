## Summary

Keep open. Current main still relies on exact magic-token text for the A2A reply/announce flow and does not expose the requested structured `sessions_send` action or announce routing override; the issue also has explicit maintainer direction to remain acceptance criteria for the broader A2A handoff work.

## What This Changes

Request structured `sessions_send` skip/announce controls plus inherited or explicit channel/thread routing for agent handoff announce turns.

## Best Possible Solution

Fold this into the first-party A2A handoff contract with backward-compatible structured skip/no-reply/dispatch outcomes, explicit announce target/thread overrides, inherited routing context, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: `Task complete. REPLY_SKIP` fails the exact-token helper, and the current `sessions_send` schema has no structured action or announce override fields to bypass model text.

## Solution Assessment

Yes, the requested structured-control direction is the maintainable solution, but it should land as part of the A2A handoff protocol instead of as a looser magic-token parser.
