## Summary

Keep open. Current main and v2026.6.6 still rely on exact `REPLY_SKIP`/`ANNOUNCE_SKIP`/`NO_REPLY` sentinel text and do not expose a structured `sessions_send` action or routing override; a narrow threadId fallback was fixed, but the central handoff contract remains unresolved and related open PRs are not safe canonical replacements.

## What This Changes

Request structured `sessions_send` callback actions plus inherited or overrideable thread context for A2A handoff announce delivery.

## Best Possible Solution

Define one narrow A2A handoff contract that makes non-delivery intent structured, carries trusted channel/thread context through final delivery, and avoids growing into a broad orchestration framework.

## Reproduction Assessment

Yes, source-level: current main lacks the requested structured `sessions_send` action/routing parameters and still recognizes non-delivery through exact sentinel text. I did not run a live Telegram handoff in this read-only review.

## Solution Assessment

Unclear. Structured intent is the right direction compared with prompt-only magic strings, but maintainers still need to choose the public A2A API shape and how it relates to the open handoff-acknowledgement work.
