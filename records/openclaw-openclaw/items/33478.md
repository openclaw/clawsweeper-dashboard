## Summary

Keep open. Current main still lacks the requested structured `sessions_send` action/outcome and announce override contract, exact magic-token handling remains in the A2A flow, and a maintainer comment explicitly says this should remain acceptance criteria for the broader A2A handoff work.

## What This Changes

Request to replace fragile magic-token handoff controls with structured `sessions_send` actions while preserving and optionally overriding announce channel/thread routing.

## Best Possible Solution

Fold this into the first-party A2A handoff contract with backward-compatible structured skip/no-reply/dispatch outcomes, explicit announce target/thread overrides, inherited context, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: a reply like `Task complete. REPLY_SKIP` fails the exact-token helper, and current `sessions_send` has no structured action or announce override fields to bypass model text.

## Solution Assessment

Yes, the requested structured-control direction is the maintainable solution, but it should land as part of the A2A handoff protocol rather than as a looser token parser.
