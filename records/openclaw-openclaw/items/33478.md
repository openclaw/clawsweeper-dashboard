## Summary

Keep open. Current main and the latest release still lack a structured `sessions_send` skip/action surface, while only part of the thread-routing problem has been addressed; the issue remains useful as A2A handoff acceptance criteria with explicit maintainer/product-direction signals.

## What This Changes

Request structured `sessions_send` skip/announce actions plus inherited or explicit route/thread context for A2A handoff announce turns.

## Best Possible Solution

Keep this open as an A2A handoff acceptance-criteria issue until maintainers settle the structured outcome API and tests cover exact-token, structured no-reply, announce-skip, and thread-inheritance behavior together.

## Reproduction Assessment

Yes at source level: current main still exposes only message-based `sessions_send` parameters and exact sentinel suppression, while thread-context inheritance is only partially covered by existing tests. No live Telegram reproduction was run in this read-only review.

## Solution Assessment

Unclear as an implementation choice: structured outcome/action support is likely the right direction, but maintainers need to decide whether it belongs directly on `sessions_send` or in a broader A2A handoff contract.
