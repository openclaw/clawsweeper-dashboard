## Summary

Keep open. The threadId preservation slice is present on current main, but current main and v2026.5.12 still expose only the legacy `sessions_send` schema, use exact `REPLY_SKIP`/`ANNOUNCE_SKIP` text helpers, and related A2A/private-delivery work remains open, so the structured handoff contract still needs maintainer API direction.

## What This Changes

Request structured `sessions_send` callback actions plus inherited or overrideable announce routing for agent-to-agent handoffs.

## Best Possible Solution

Define a backward-compatible structured handoff outcome in the A2A contract, keep legacy token compatibility, preserve or override announce routing explicitly, and cover token chatter plus thread inheritance with focused regression tests.

## Reproduction Assessment

Yes for source-level current behavior: the tool schema lacks the requested structured action and announce override fields, and the A2A loop still consumes exact text tokens. I did not run a live Telegram/A2A reproduction in this read-only review.

## Solution Assessment

Yes at the direction level, but not as an isolated regex or schema tweak. The maintainable path is coordinated A2A handoff protocol work that preserves legacy tokens while adding structured outcomes and thread-aware announce routing.
