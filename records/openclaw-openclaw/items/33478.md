## Summary

Keep open. Current main still lacks structured `sessions_send` callback actions and announce override fields, while the threadId preservation slice is only partially addressed and the live discussion explicitly keeps this as A2A handoff acceptance criteria.

## What This Changes

Request structured callback actions for `sessions_send` and preserved or explicit thread context routing during agent handoff announce turns.

## Best Possible Solution

Define a backward-compatible structured handoff outcome in the A2A contract, keep legacy token compatibility, add thread-aware announce routing or overrides, and cover token chatter plus thread inheritance with focused regression tests.

## Reproduction Assessment

Yes for source-level current behavior: the tool schema lacks the requested structured action and announce override fields, and the A2A loop still consumes exact text tokens. I did not run a live Telegram/A2A reproduction in this read-only review.

## Solution Assessment

Yes at the direction level, but not as an isolated regex or schema tweak. The maintainable path is coordinated A2A handoff protocol work that preserves legacy tokens while adding structured outcomes and thread-aware announce routing.
