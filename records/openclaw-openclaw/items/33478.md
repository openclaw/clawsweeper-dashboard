## Summary

Keep open. Current main still has token-driven `sessions_send` A2A behavior with no structured callback action or announce override fields, while recent discussion keeps this as acceptance criteria for the broader handoff protocol work; only a narrower threadId preservation slice has landed.

## What This Changes

The issue requests structured `sessions_send` callback controls and inherited or overrideable thread-aware announce routing for agent handoffs.

## Best Possible Solution

Fold structured skip/no-reply outcomes and inherited or overrideable announce routing into the first-party A2A handoff contract, preserving legacy tokens while adding docs and focused regression tests.

## Reproduction Assessment

Yes, by source inspection: current main lacks the requested structured action and announce override fields, and the control-token helpers still require exact full-text matches. This is a feature/API gap rather than a broken existing documented contract.

## Solution Assessment

Yes at the direction level: structured control outcomes are more maintainable than broadening magic-string parsing. The implementation should land through the coordinated A2A handoff protocol rather than as an isolated schema tweak.
