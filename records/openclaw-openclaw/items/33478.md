## Summary

Keep open. Current main still lacks the requested structured `sessions_send` action and announce override surface, exact control-token handling remains part of the A2A contract, and the thread-routing fixes only cover part of the reported context-inheritance problem.

## What This Changes

Requests a structured `sessions_send` handoff outcome and announce-routing API that avoids exact `REPLY_SKIP` text and preserves channel/thread context during A2A announces.

## Best Possible Solution

Define a backward-compatible A2A handoff outcome contract with structured skip/no-reply controls, preserve legacy tokens, keep thread-aware announce routing, and add regression tests for chatty control-token output plus inherited or overridden thread targets.

## Reproduction Assessment

Yes for source-level current behavior: the tool schema lacks the requested structured action and announce override fields, while the A2A loop still consumes exact text tokens. I did not run a live Telegram/A2A reproduction in this read-only review.

## Solution Assessment

Yes at the direction level, but not as an isolated regex or schema tweak. The maintainable path is coordinated A2A handoff protocol work that preserves legacy tokens while adding structured outcomes and thread-aware announce routing.
