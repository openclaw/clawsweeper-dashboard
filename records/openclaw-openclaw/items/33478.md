## Summary

Keep open: current main and v2026.6.6 still lack the structured `sessions_send` action/routing parameters, while current thread-preservation fixes only cover part of the reported handoff-context problem. ## What This Changes

The issue requests structured `sessions_send` skip/announce actions plus inherited or explicitly overridden thread routing context for agent handoffs.

## Best Possible Solution

Keep this as a maintainer-owned A2A handoff acceptance requirement and settle one structured non-delivery/routing contract across the related open handoff issues.
