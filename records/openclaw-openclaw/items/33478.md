## Summary

Keep this issue open. Current main and shipped releases have repaired the concrete thread-loss paths identified by related reports, but agent-to-agent reply suppression still relies on exact model text, and the structured handoff contract remains undecided. The related draft proposal is not an implementation or safe superseding item, while repository guidance in the discussion explicitly retains this issue as focused acceptance criteria.

## What This Changes

Add structured handoff control outcomes and preserve or explicitly override the originating channel and thread context for agent announcements.

## Best Possible Solution

Define one typed handoff outcome that represents reply, skip, dispatch, and announce behavior, carry the captured requester delivery context through the canonical announce pipeline, and retain text tokens only as an explicitly deprecated compatibility boundary if a shipped contract requires it.

## Reproduction Assessment

Yes at source level for the remaining protocol weakness: current documentation and tests require exact `REPLY_SKIP`, `ANNOUNCE_SKIP`, and `NO_REPLY` text. The historical thread-loss variants have shipped fixes and regression coverage, so that part is not established as a current-main failure.

## Solution Assessment

The requirement is valid, but the proposed collection of independent parameters is not yet proven to be the best permanent API. A single structured outcome integrated with the existing delivery-context owner is narrower and less likely to create conflicting routing semantics.
