## Summary

Keep open: current main has fixed part of the thread-routing loss, but it still lacks the structured `sessions_send` handoff action/override contract, and maintainer discussion explicitly treats this as a remaining A2A reliability requirement rather than a stale duplicate.

## What This Changes

The issue requests structured `sessions_send` callback actions plus inherited or explicit channel/thread context for agent handoff announce delivery.

## Best Possible Solution

Keep this open for a maintainer-owned A2A handoff contract that preserves existing conversation mode while adding structured non-delivery intent and tested channel/thread inheritance without prompt-only magic strings.

## Reproduction Assessment

Yes, source-level: current main still lacks a structured `sessions_send` action/route override and still uses exact control-token text in the A2A flow. I did not run a live Telegram handoff, and the thread-routing portion is partly covered by current tests.

## Solution Assessment

Unclear as an implementation choice. Structured handoff intent is a better direction than prompt-only `REPLY_SKIP`, but maintainers should settle the public A2A API shape across the related routing, dispatch, and suppression reports before implementation.
