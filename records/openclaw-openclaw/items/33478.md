## Summary

Keep open. Current main still lacks the requested structured `sessions_send` control action and announce override parameters, while the A2A flow still depends on exact control-token text; prior triage also keeps this as acceptance criteria for the open handoff protocol work.

## What This Changes

Request structured `sessions_send` callback actions plus inherited and overrideable channel/thread context for agent-to-agent handoff announce delivery.

## Best Possible Solution

Fold this into the first-party A2A handoff contract with backward-compatible structured skip/no-reply/dispatch outcomes, inherited and overrideable announce routing, docs, and regression tests.

## Reproduction Assessment

Yes, by source inspection: mixed text such as `Task complete. REPLY_SKIP` fails the exact-token helpers, and the `sessions_send` schema has no structured action or announce override field to bypass model text.

## Solution Assessment

Yes at the direction level. Structured control outcomes are more maintainable than loosening magic-string parsing, but the implementation should land through the coordinated A2A handoff protocol rather than as an isolated sessions_send tweak.
