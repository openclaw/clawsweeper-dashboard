## Summary

Keep open: current main and shipped releases now preserve thread context for `sessions_send` announcements, but the central reliability requirement remains because handoff termination still depends on exact model-emitted `REPLY_SKIP` and `ANNOUNCE_SKIP` text. The existing maintainer direction explicitly retains this issue as acceptance criteria for the open handoff-protocol work, so the remaining public contract needs product review rather than cleanup closure.

## What This Changes

Add structured no-reply and announce controls for agent handoffs, inherit the originating delivery context, and optionally support explicit announce routing overrides.

## Best Possible Solution

Keep automatic thread preservation as the canonical routing path, then define one typed terminal-outcome contract within the approved A2A handoff design; add explicit routing overrides only if maintainers establish them as the single supported owner-controlled escape hatch.

## Reproduction Assessment

Yes, at source level. Current main still requires exact `REPLY_SKIP` and `ANNOUNCE_SKIP` model text, while source and shipped history show that the separate thread-loss path has already been repaired.

## Solution Assessment

No, not exactly as proposed. Typed terminal outcomes are the maintainable direction, but the parameter names, backward-compatibility policy, and explicit routing overrides should be decided inside the canonical handoff protocol rather than added as an independent parallel API.
