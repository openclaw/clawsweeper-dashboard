## Summary

Keep open. Current main still lacks the requested structured `sessions_send` action and announce override surface, while a recent maintainer comment keeps this as acceptance criteria for the A2A handoff work; only the narrower threadId preservation slice has landed.

## What This Changes

The issue requests structured `sessions_send` callback controls plus inherited or overrideable thread context for A2A announce delivery.

## Best Possible Solution

Fold structured skip/no-reply outcomes and inherited or overrideable announce routing into the first-party A2A handoff contract, preserving legacy tokens while adding docs and focused regression tests.

## Reproduction Assessment

Yes. Source inspection shows `sessions_send` has no structured action or announce override parameters, and the current helpers only treat exact full-text control tokens as skip/silent outcomes.

## Solution Assessment

Yes at the direction level. Structured control outcomes are more maintainable than broadening magic-string parsing, but the implementation should land through the coordinated A2A handoff protocol rather than an isolated schema tweak.
