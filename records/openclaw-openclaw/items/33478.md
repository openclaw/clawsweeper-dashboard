## Summary

Keep open. Current main still lacks the requested structured `sessions_send` action/announce override surface, exact control-token handling still leaves chatty `REPLY_SKIP` outside the contract, and maintainer discussion plus open follow-up PRs show this remains active A2A handoff product work rather than cleanup.

## What This Changes

Request adds structured `sessions_send` skip/announce routing controls and thread-context inheritance for A2A handoff announce turns.

## Best Possible Solution

Define a backward-compatible A2A handoff outcome contract with structured skip/no-reply controls, preserve legacy tokens, keep thread-aware announce routing, and add regression tests for chatty control-token output plus thread inheritance.

## Reproduction Assessment

Yes for source-level current behavior: the tool schema lacks the requested structured action and announce override fields, while the A2A loop still consumes exact text tokens. I did not run a live Telegram/A2A reproduction in this read-only review.

## Solution Assessment

Yes at the direction level, but not as an isolated regex or schema tweak. The maintainable path is coordinated A2A handoff protocol work that preserves legacy tokens while adding structured outcomes and thread-aware announce routing.
