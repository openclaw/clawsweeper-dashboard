## Summary

Keep open. Current main still lacks the requested structured `sessions_send` action and announce routing override API, while A2A control still depends on exact magic-token text; the thread inheritance slice has partial fixes, and maintainer discussion explicitly keeps this as acceptance criteria for the broader handoff protocol work.

## What This Changes

Add structured `sessions_send` skip/announce controls and preserve or override channel/thread context during agent handoff announce turns.

## Best Possible Solution

Fold this into the first-party A2A handoff contract with backward-compatible structured skip/no-reply/dispatch outcomes, inherited and overrideable announce routing, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: mixed text such as `Task complete. REPLY_SKIP` fails the exact-token helpers, and the current `sessions_send` schema has no structured action or announce override field to bypass model text.

## Solution Assessment

Yes at the direction level: structured control outcomes are more maintainable than loosening magic-string parsing. The implementation should land through the coordinated A2A handoff protocol rather than as an isolated parser tweak.
