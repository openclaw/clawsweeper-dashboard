## Summary

Keep open: current main and the latest release still lack the requested structured `sessions_send` action and announce override surface, while A2A control handling still depends on exact token outcomes. The threadId routing slice is partly fixed, but the maintainer discussion explicitly keeps this as acceptance criteria for the broader handoff contract.

## What This Changes

Request structured `sessions_send` callback controls plus inherited or overridden thread context for A2A announce and final-delivery turns.

## Best Possible Solution

Define a backward-compatible A2A handoff outcome contract with structured skip/no-reply controls, legacy token compatibility, thread-aware announce routing, override fields, and regression coverage for chatty control-token output plus inherited or overridden targets.

## Reproduction Assessment

Yes for source-level current behavior: the tool schema lacks the requested structured action and announce override fields, while the A2A loop still consumes exact text tokens. I did not run a live Telegram/A2A reproduction in this read-only review.

## Solution Assessment

Yes at the direction level, but not as an isolated regex or schema tweak. The maintainable path is coordinated A2A handoff protocol work that preserves legacy tokens while adding structured outcomes and thread-aware announce routing.
