## Summary

Keep open: current main and v2026.6.6 still expose only sentinel-text A2A skip semantics for `sessions_send`, while only part of the thread-routing problem has been fixed.

## What This Changes

Request structured `sessions_send` handoff controls and inherited channel/thread routing so A2A announce/no-reply behavior does not depend on exact model text.

## Best Possible Solution

Define one narrow, default-compatible A2A handoff contract that gives `sessions_send` structured non-delivery/routing intent, preserves channel/thread context in tests, and avoids becoming a broad orchestration framework.

## Reproduction Assessment

Yes, at source level. The current `sessions_send` schema has no structured action parameter, and the skip helpers only recognize token-only sentinel text, so mixed prose plus `REPLY_SKIP` remains outside the supported path.

## Solution Assessment

Unclear as an API shape, but the current sentinel-only path is not the best final state. A structured action/routing contract is plausible, but maintainers should settle the exact model-facing parameters and compatibility behavior first.
