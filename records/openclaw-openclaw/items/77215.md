## Summary

Keep open. The docs direction is useful and the supplied proof is sufficient for a docs-only change, but the current PR head still points Control UI Talk at the non-realtime Talk keys and overstates Realtime auth as API-key-only despite current main supporting the `openai-codex` OAuth fallback.

## What This Changes

The PR updates `docs/providers/openai.md` and `CHANGELOG.md` to add a Platform-billing warning for OpenAI Realtime voice/Talk.

## Best Possible Solution

Keep the billing warning, but edit it to use `talk.realtime.*`, keep the Voice Call realtime key path, and describe Platform billing without removing the supported `openai-codex` OAuth fallback.

Yes for the review findings: PR-head docs plus current main source/docs show the exact config-key and OAuth-fallback mismatches. A live OpenAI billing session is not needed to prove these docs regressions.

## Solution Assessment
