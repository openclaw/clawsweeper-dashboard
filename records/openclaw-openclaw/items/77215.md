## Summary

Keep open; this PR should not merge as-is because current main now supports and documents OpenAI Realtime using `openai-codex` OAuth as an auth fallback, so the new Platform-credits-required wording is too broad. The external PR also still lacks acceptable after-fix real behavior proof or a maintainer `proof: override`.

## What This Changes

The PR adds an OpenAI provider documentation note and changelog entry saying Realtime voice/Talk require OpenAI Platform credits rather than Codex/ChatGPT subscription quota.

## Best Possible Solution

Revise the docs to distinguish direct `OPENAI_API_KEY` Platform billing failures from the current Codex OAuth fallback, while keeping clear that `openai-codex` is an auth/profile route and not a realtime provider id.

## Reproduction Assessment

Yes, by source inspection: current main falls back to `openai-codex` OAuth for OpenAI Realtime when no API key is configured, and tests cover backend and browser client-secret minting. I did not run a live Realtime session.

## Solution Assessment

No; as written, the PR is too broad because it says Realtime requires Platform credits even though current main supports and documents Codex OAuth for native OpenAI Realtime. The safer fix is a narrower troubleshooting note for unfunded or wrong `OPENAI_API_KEY` cases.
