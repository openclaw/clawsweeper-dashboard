## Summary

Keep open: the PR addresses a real OpenAI Realtime billing documentation gap, but the current head still documents stale Control UI Talk config keys and overstates API-key-only Realtime auth despite current source and tests supporting an `openai-codex` OAuth fallback.

## What This Changes

The PR adds an OpenAI provider docs note and changelog entry explaining that Realtime voice uses OpenAI Platform billing rather than Codex/ChatGPT subscription quota.

## Best Possible Solution

Revise the note to keep the Platform billing warning while using `talk.realtime.*` keys and preserving the supported `openai-codex` OAuth fallback.

## Reproduction Assessment

Yes for the review findings: the PR head contains stale `talk.provider` / `talk.providers` wording and an API-key-only implication, while current main source, docs, and tests show the `talk.realtime.*` and OAuth-fallback contract. A live billing repro is not needed to prove these docs mismatches.

## Solution Assessment

No in the current form. A narrow docs note is the right solution, but it should describe the Platform quota failure mode without telling users to configure the speech/TTS Talk key or implying API-key auth is the only supported Realtime path.
