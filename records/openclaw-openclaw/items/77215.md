## What This Changes

The PR adds an OpenAI provider docs note and changelog entry explaining that OpenAI Realtime voice uses OpenAI Platform billing rather than Codex/ChatGPT subscription quota.

## Best Possible Solution

Revise the note to keep the Platform billing warning while using `talk.realtime.*` keys and preserving the supported `openai-codex` OAuth fallback.

## Reproduction Assessment

Yes for the review findings: the PR head contains the stale docs wording, while current main source, docs, and tests show the `talk.realtime.*` and OAuth-fallback contract. A live billing repro is not needed to prove these docs mismatches.

## Solution Assessment

No in the current form. A narrow docs note is the right solution, but it should describe the Platform quota failure mode without telling users to configure the speech/TTS Talk key or implying API-key auth is the only supported Realtime path.
