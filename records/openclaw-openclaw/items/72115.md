The PR hardens Skill Workshop workspace skill writes with realpath containment, write-boundary revalidation, prompt-budget parity, SDK exports, docs, tests, and related plugin loader, proxy, QA, and CI support updates.

## Best Possible Solution

Land the Skill Workshop hardening after the bundled override comparison uses the same user-path and realpath normalization as bundled-dir resolution, proxy activation fails closed when required HTTP hooks cannot load, and exact-head checks are green; maintainers should also decide whether the non-Workshop CI/QA/proxy support changes stay in this branch or split out.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Normalize bundled override paths before comparing:** `src/plugin-sdk/facade-loader.ts:149-160`
  - body: `resolveBundledPluginsDir` expands `OPENCLAW_BUNDLED_PLUGINS_DIR` with `resolveUserPath`, but this comparison uses `path.resolve` on the raw env value. A valid override like `~/bundled` can be treated as outside the override and pass `installRuntimeDeps: false`, skipping runtime dependency staging for the bundled public surface.
  - confidence: 0.88
- **[P2] Fail closed when proxy HTTP hooks cannot load:** `src/infra/net/proxy/proxy-lifecycle.ts:255-258`
  - body: When `global-agent` cannot be loaded, `bootstrapNodeHttpStack` now logs and returns, but `startProxy` still registers the proxy as active. Node `http`/`https` callers that rely on those hooks can bypass the operator-managed proxy while `OPENCLAW_PROXY_ACTIVE` is set, so activation should throw or the fallback should be explicitly test-only.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff is security-sensitive and needs attention because proxy activation can proceed after the Node HTTP proxy hook dependency is unavailable.

Concerns:

- **[medium] Proxy activation can lose Node HTTP hooks:** `src/infra/net/proxy/proxy-lifecycle.ts:255`
  - body: The PR changes `global-agent` from a required static import to an optional resolver. If it is unavailable, `bootstrapNodeHttpStack` returns after a warning and `startProxy` still registers an active proxy, which can leave Node `http`/`https` traffic outside the configured filtering proxy.
  - confidence: 0.82
