## Summary

Keep open. Current main has not implemented the Skill Workshop hardening, and the latest PR head still has blocking review findings in proxy fail-closed behavior and bundled override path handling before it is merge-ready.

## What This Changes

The PR hardens Skill Workshop writes with realpath containment, write-boundary revalidation, prompt-budget enforcement, SDK exports, docs/tests, and related QA/plugin/proxy stabilization changes.

## Best Possible Solution

Land the Skill Workshop hardening after proxy activation fails closed when required Node HTTP hooks cannot load, bundled override comparisons use the same user-path normalization as bundled-dir resolution, and the user-facing change has a changelog entry. Keep the memory provenance fix in #71898 or #72886 rather than expanding this PR.

## Reproduction Assessment

Yes for the PR findings. A focused proxy unit test can simulate unavailable global-agent resolution and assert startProxy rejects/restores env, and a facade-loader test can use a home-relative OPENCLAW_BUNDLED_PLUGINS_DIR override to verify runtime dependency staging still occurs.

## Solution Assessment

No, not yet. The plugin-owned Skill Workshop hardening direction fits the architecture, but the current patch should restore proxy fail-closed behavior, normalize bundled override paths consistently, and add the required changelog before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Fail closed when proxy hooks are unavailable:** `src/infra/net/proxy/proxy-lifecycle.ts:255-258`
  - body: When global-agent cannot be resolved, bootstrapNodeHttpStack only logs and returns, but startProxy continues to push an active registration. That can leave Node http/https callers bypassing the operator-managed proxy while the process reports proxy mode as active.
  - confidence: 0.86
- **[P2] Normalize bundled override paths before comparing:** `src/plugin-sdk/facade-loader.ts:149-156`
  - body: resolveBundledPluginsDir expands OPENCLAW_BUNDLED_PLUGINS_DIR through resolveUserPath, but this comparison uses path.resolve on the raw env value. A valid override such as ~/bundled can be treated as outside the override and skip runtime dependency staging for bundled facade loads.
  - confidence: 0.84
- **[P3] Add a required changelog entry:** `docs/plugins/skill-workshop.md:114-120`
  - body: This PR changes user-visible Skill Workshop behavior and plugin SDK/runtime behavior, but CHANGELOG.md is not updated. Repo policy requires a changelog entry for user-facing fix/feat/perf changes before merge.
  - confidence: 0.78
