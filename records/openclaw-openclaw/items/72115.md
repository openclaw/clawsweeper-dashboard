## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Project the full skill catalog in budget previews:** `src/agents/skills/workspace.ts:930-943`
  - body: `previewSkillsPromptImpact` computes the baseline from the full skill catalog, but the projected state is built from `loadWorkspaceSkillEntries()` only. Because passing `entries` bypasses normal `loadSkillEntries()` merging, managed, bundled, extra, and `.agents` skills disappear from the projected catalog, so adding a workspace skill that would evict one of those entries can still pass the prompt-budget guard.
  - confidence: 0.89
- **[P2] Normalize bundled override paths before comparing:** `src/plugin-sdk/facade-loader.ts:149-156`
  - body: `resolveBundledPluginsDir()` expands `OPENCLAW_BUNDLED_PLUGINS_DIR` with `resolveUserPath`, but this comparison uses `path.resolve` on the raw env value. A valid override like `~/bundled` can be treated as outside the override and skip runtime dependency staging for the bundled facade.
  - confidence: 0.86
- **[P2] Fail closed when proxy hooks are unavailable:** `src/infra/net/proxy/proxy-lifecycle.ts:255-258`
  - body: When `global-agent` cannot be loaded, `bootstrapNodeHttpStack` logs and returns, but `startProxy` still registers the proxy as active. In that runtime, Node `http`/`https` traffic can bypass the operator-managed proxy while `OPENCLAW_PROXY_ACTIVE` is set, so activation should fail or be explicitly limited to a test-only fallback.
  - confidence: 0.8
