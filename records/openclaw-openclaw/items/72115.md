## Summary

Keep open. Current main does not implement the Skill Workshop hardening, and the PR is useful, but the current head is not merge-ready because it introduces proxy fail-open behavior, has a bundled override normalization bug, and lacks the required changelog entry.

## What This Changes

The PR hardens Skill Workshop write paths with realpath containment, TOCTOU revalidation, prompt-budget enforcement, SDK exports, docs, tests, and related CI/parity stabilization changes.

## Best Possible Solution

Land the Skill Workshop hardening only after proxy activation fails closed when required Node HTTP hooks cannot load, bundled override comparisons use the same user-path normalization as bundled-dir resolution, and the user-facing change has a changelog entry.

## Reproduction Assessment

Yes for the PR findings. A focused proxy unit test can force global-agent resolution to be unavailable and assert startProxy rejects/restores env, and a facade-loader test can use a home-relative OPENCLAW_BUNDLED_PLUGINS_DIR override to verify staged runtime deps are still used.

## Solution Assessment

No, not yet. The plugin-owned Skill Workshop hardening direction fits the architecture, but the current patch should restore proxy fail-closed behavior, normalize bundled override paths consistently, and add the required changelog before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Fail closed when proxy hooks are unavailable:** `src/infra/net/proxy/proxy-lifecycle.ts:255-258`
  - body: When global-agent cannot be resolved, `bootstrapNodeHttpStack` only logs and returns, but `startProxy` still registers the proxy as active. That can leave Node `http`/`https` callers bypassing the configured operator proxy while the process reports proxy mode as active.
  - confidence: 0.86
- **[P2] Normalize bundled override paths before comparing:** `src/plugin-sdk/facade-loader.ts:149-156`
  - body: `resolveBundledPluginsDir` expands `OPENCLAW_BUNDLED_PLUGINS_DIR` through `resolveUserPath`, but this comparison uses `path.resolve` on the raw env value. A valid override such as `~/bundled` can be treated as outside the override and skip runtime dependency staging for bundled facade loads.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `docs/plugins/skill-workshop.md:114-120`
  - body: This PR changes user-visible Skill Workshop behavior and plugin SDK/runtime behavior, but `CHANGELOG.md` is not updated. Repo policy requires a changelog entry for user-facing fix/feat/perf changes before merge.
  - confidence: 0.78
