## Summary

Keep open. Current main still lacks the Skill Workshop hardening this PR is trying to add, but the PR head is not ready because it still has a bundled-plugin runtime-dependency path normalization bug and a security-sensitive proxy fail-open regression.

## What This Changes

This PR hardens Skill Workshop writes with realpath containment, write-boundary revalidation, and core prompt-budget preview, exports supporting SDK seams, updates docs/tests, and carries related QA, plugin-loader, proxy, and CI stabilization changes.

## Best Possible Solution

Keep the plugin-owned Skill Workshop hardening direction, but land it only after bundled override comparison uses the same user-path normalization as bundled-dir resolution, proxy activation fails closed when Node HTTP hooks cannot be installed, the required changelog is added, and the branch is refreshed against current main with targeted plus changed-lane validation.

## Reproduction Assessment

Yes. The facade issue can be reproduced with a home-relative OPENCLAW_BUNDLED_PLUGINS_DIR pointing at a built bundled plugin that needs runtime dependency staging, and the proxy issue can be reproduced by making global-agent resolution unavailable while proxy.enabled is true.

## Solution Assessment

No, not yet. The Skill Workshop write-path approach matches the repo boundary model, but the current PR head is not the narrowest maintainable solution until the loader normalization and proxy fail-closed behavior are corrected.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Normalize bundled override paths before comparing:** `src/plugin-sdk/facade-loader.ts:149-156`
  - body: resolveBundledPluginsDir expands OPENCLAW_BUNDLED_PLUGINS_DIR with resolveUserPath, but this comparison uses path.resolve on the raw value. A valid override such as ~/bundled can be treated as outside the override and pass installRuntimeDeps: false, skipping runtime dependency staging for the bundled public surface.
  - confidence: 0.88
- **[P2] Fail closed when proxy HTTP hooks cannot load:** `src/infra/net/proxy/proxy-lifecycle.ts:255-258`
  - body: When global-agent cannot be loaded, bootstrapNodeHttpStack logs and returns, but startProxy still registers the proxy as active. In that runtime, node:http/node:https callers that rely on those hooks can bypass the operator-managed proxy while OPENCLAW_PROXY_ACTIVE is set.
  - confidence: 0.8
- **[P3] Add a changelog entry for this user-facing hardening:** `docs/plugins/skill-workshop.md:114-120`
  - body: This PR changes user-visible Skill Workshop write behavior and plugin docs, but CHANGELOG.md is not touched. Repo policy requires user-facing fix/feat/perf changes to add an active-version changelog entry with appropriate contributor credit.
  - confidence: 0.82

## Security Review
