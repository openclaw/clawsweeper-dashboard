Request bounded or rate-limited gateway log handling so repeated WebSocket heartbeat timeout output cannot grow macOS gateway logs without limit.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/79892 after resolving current-main stdout/stderr scope and retention defaults; keep backup amplification and diagnostic JSONL rotation in their narrower linked issues.

## Solution Assessment

No, current main is only a partial mitigation. The maintainable path is daemon-owned retention for launchd logs, with heartbeat dedupe, diagnostic JSONL rotation, and backup cleanup handled separately.
