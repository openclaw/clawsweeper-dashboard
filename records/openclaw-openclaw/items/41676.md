## Summary

Keep open. Current main partly mitigates the original macOS stderr sink, but daemon-owned `gateway.log` still has no retention path and the focused daemon log-retention PR is still open.

## What This Changes

The issue reports unbounded gateway log growth from repeated WebSocket heartbeat timeout output and asks for rate limiting plus a bounded rotation/max-size policy.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/79892 with current-main daemon log retention for launchd-owned logs, and leave backup amplification, diagnostic JSONL rotation, and heartbeat dedupe to their narrower tracks.

## Reproduction Assessment

The original heartbeat spam into active `gateway.err.log` is only partially applicable now because macOS launchd stderr is suppressed to `/dev/null`.

## Solution Assessment

No, current main is not a complete solution. The narrow maintainable path is a daemon-owned retention change for launchd logs, with heartbeat dedupe and backup cleanup handled separately.
