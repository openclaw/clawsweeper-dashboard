## Summary

Keep open. Current main still maps the macOS LaunchAgent stdout/stderr streams to fixed `gateway.log` and `gateway.err.log` files without daemon-owned retention, and the active open PR that targets this surface has not merged yet.

## What This Changes

The issue requests bounded gateway service logs so repeated heartbeat/error output cannot grow `gateway.err.log` indefinitely.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/79892 or an equivalent daemon log-retention patch, while tracking heartbeat dedupe, CPU-loop, and backup amplification as separate follow-ups if they remain after retention lands.

## Reproduction Assessment

No high-confidence live reproduction was established for the original heartbeat timeout loop. Source inspection does give a high-confidence reproduction path for the active defect: launchd appends stdout/stderr to fixed `gateway.log` and `gateway.err.log` files with no current-main retention hook.

## Solution Assessment

Yes for the retention half: bounding the launchd-owned service sinks at daemon lifecycle boundaries is the narrow maintainable direction. The exact max-size/archive defaults and any extra heartbeat dedupe policy still need maintainer approval or the active PR review path.
