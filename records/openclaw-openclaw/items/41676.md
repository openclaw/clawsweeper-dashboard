## Repair Work Prompt

Fix the unbounded daemon-managed gateway service logs tracked by https://github.com/openclaw/openclaw/issues/41676. Add a bounded retention policy for macOS launchd `gateway.log` / `gateway.err.log` service output, using the existing structured logger defaults as precedent where practical: about 100 MB active file and a small archive count. Account for launchd keeping stdout/stderr file descriptors open; do not implement a rename-only rotation that leaves active writes going to an unlinked or archived file. Prefer a small reusable helper in the daemon/logging boundary with tests. Do not change provider-payload/cache-trace JSONL rotation handled by #75380/#76407, and do not solve backup pruning from #76502 in this PR. Add or update docs for the service-log retention behavior and include a user-facing CHANGELOG entry. Validate with focused daemon/logging tests and the changed gate if available.
