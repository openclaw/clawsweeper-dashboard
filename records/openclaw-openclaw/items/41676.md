## Summary

Keep open. Current main suppresses macOS launchd stderr, so the exact active `gateway.err.log` sink has changed, but daemon-owned `gateway.log` still writes to a fixed file with no retention hook and the focused retention PR is still open.

## What This Changes

Requests bounded or rate-limited Gateway service logging so repeated heartbeat/WebSocket failures cannot grow `gateway.err.log` or `gateway.log` indefinitely.

## Best Possible Solution

Rebase or replace https://github.com/openclaw/openclaw/pull/79892 so current main bounds the launchd-owned `gateway.log` path, documents suppressed stderr behavior, and leaves heartbeat dedupe and backup amplification to their narrower tracks.

## Reproduction Assessment

Yes, for the remaining retention gap by source inspection: current main still writes launchd stdout to a fixed `gateway.log` without a retention hook. No live current-main reproduction was established for the original heartbeat spam into active `gateway.err.log`, and Darwin stderr is now suppressed.

## Solution Assessment

No, the issue is not fully solved on current main. The best path is a current-main retention change for launchd-owned logs, with any heartbeat dedupe or stale legacy stderr cleanup treated as explicit follow-up scope.
