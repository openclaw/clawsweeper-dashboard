## Summary

Keep open. Current main has partially mitigated the original macOS `gateway.err.log` spam by suppressing launchd stderr, but the daemon-owned stdout log is still a fixed append-only file with no retention hook, and the focused daemon log-retention PR remains open and conflicting.

## What This Changes

The issue requests bounded gateway log growth for repeated heartbeat timeout noise, especially rotation or max-size control for launchd-owned gateway logs.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/79892 with current-main daemon retention for launchd-owned logs, then leave backup amplification and diagnostic JSONL rotation to https://github.com/openclaw/openclaw/issues/76502 and https://github.com/openclaw/openclaw/issues/75380.

## Reproduction Assessment

Yes, for the remaining daemon retention gap by source inspection: current main still writes launchd stdout to a fixed `gateway.log` without a max-size or rotation hook. The original active `gateway.err.log` heartbeat spam path is partly mitigated because macOS launchd stderr is now suppressed to `/dev/null`.

## Solution Assessment

No, current main is not a complete solution. The narrow maintainable path is a daemon-owned retention change for launchd logs, with heartbeat dedupe, diagnostic JSONL rotation, and backup cleanup handled separately.
