## Summary

Keep open: current main still has no retention cap for the launchd-owned `gateway.log`/`gateway.err.log` service files, and recent user comments added stronger live disk-usage evidence. An open PR now targets this exact launchd service-log retention path, so the right cleanup is to review/land or close that PR rather than close this issue or queue duplicate work.

## What This Changes

Request bounded rotation or max-size control for Gateway service stdout/stderr logs so repeated heartbeat or error spam cannot grow `gateway.err.log` without bound.

## Best Possible Solution

Land an owner-reviewed launchd service-log retention fix for `gateway.log` and `gateway.err.log`, while keeping diagnostic JSONL rotation, backup amplification, and CPU hot-loop reports tracked separately.
