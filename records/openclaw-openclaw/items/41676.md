## Summary

Keep open. Current main and the latest release suppress macOS LaunchAgent stderr, but the active launchd stdout sink is still a fixed file with no daemon-owned max-size or rotation, and the focused retention work remains open rather than merged.

## What This Changes

The issue requests bounded gateway log output so repeated heartbeat timeout spam cannot grow gateway supervisor logs to multi-GB files.

## Best Possible Solution

Land or replace the focused daemon retention work so macOS gateway supervisor logs are bounded by default, while leaving backup cleanup and diagnostic JSONL rotation to their separate canonical items.

## Reproduction Assessment

Yes, at source level for the remaining retention gap: current main still writes macOS LaunchAgent stdout to a fixed file and has no daemon max-size or rotation helper. I did not live-reproduce the original heartbeat timeout spam loop.

## Solution Assessment

Yes, a daemon-owned retention pass at launchd install/restart is the right boundary, but current main is only a partial solution. The maintainable path is maintainer review of https://github.com/openclaw/openclaw/pull/79892 or an equivalent replacement with macOS upgrade proof.
