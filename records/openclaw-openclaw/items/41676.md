## Summary

Keep open. Current main still sends macOS launchd stderr directly to `gateway.err.log` with no retention hook, while `logging.maxFileBytes` only rotates the structured Gateway JSONL logger; the newer May 3 live evidence shows the same disk-fill class is still active in the field.

## What This Changes

Request bounded retention or rate limiting for Gateway supervisor logs, especially `gateway.err.log`, so repeated WebSocket heartbeat timeout lines cannot grow without limit.

## Best Possible Solution

Add bounded retention for macOS launchd supervisor stdout/stderr logs in the daemon layer, keep it distinct from structured JSONL rotation, and document the boundary while leaving backup amplification to #76502.

## Reproduction Assessment

Yes for the unbounded supervisor-log class: current source shows launchd stderr goes straight to `gateway.err.log` with no OpenClaw rotation hook. The exact heartbeat flood was not live-reproduced, and the exact message is not present in first-party source.

## Solution Assessment

Yes, bounded supervisor-log retention is the narrowest maintainable first fix because it protects users from any noisy stderr producer. Producer deduplication can follow once the source of the heartbeat line is identified.
