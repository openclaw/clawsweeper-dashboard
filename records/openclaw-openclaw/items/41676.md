## Summary

Keep open. Current main still leaves macOS launchd supervisor stdout/stderr logs unbounded: `gateway.err.log` is wired directly through `StandardErrorPath`, while `logging.maxFileBytes` only rotates the structured JSONL gateway logger. The earlier ClawSweeper review still applies because the relevant daemon/logging files are unchanged since that review.

## What This Changes

The issue requests rate limiting or deduplication for repeated WebSocket heartbeat timeout lines and bounded rotation/max-size control for macOS `gateway.err.log`.

## Best Possible Solution

Add bounded retention for macOS launchd supervisor logs in the daemon layer, separate from the structured JSONL logger, so `gateway.log` and `gateway.err.log` cannot grow indefinitely during long-running service operation. Cover install/restart paths, preserve service parsing and env-wrapper behavior, and document supervisor-log retention separately from `logging.maxFileBytes`.

## Reproduction Assessment

Yes for the unbounded supervisor-log class: current source shows launchd stderr goes straight to `gateway.err.log` with no OpenClaw rotation hook. The exact heartbeat flood was not live-reproduced, and the exact message is not present in first-party source.

## Solution Assessment

Yes, bounded supervisor-log retention is the narrowest maintainable first fix because it protects users from any noisy stderr producer. Producer deduplication can follow once the upstream source of the exact heartbeat line is identified.
