## Summary

Keep open. Current main still routes macOS launchd stdout/stderr directly to `gateway.log` and `gateway.err.log`, while the existing `logging.maxFileBytes` rotation only covers the structured JSONL gateway logger.

## What This Changes

The issue requests bounded or rate-limited gateway supervisor logging so repeated heartbeat timeout stderr cannot grow `gateway.err.log` indefinitely.

## Best Possible Solution

Add bounded retention for `gateway.log` and `gateway.err.log` in the daemon/supervisor log surface, keep it distinct from structured JSONL logging, and cover install/restart behavior plus docs.

## Reproduction Assessment

Yes for the unbounded supervisor-log class: current source shows launchd stderr goes straight to `gateway.err.log` with no OpenClaw rotation hook. The exact heartbeat flood was not live-reproduced, and the exact message is absent from first-party source.

## Solution Assessment

Yes, bounded supervisor-log retention is the best first fix because it protects users from any noisy stderr producer. Producer deduplication can follow once the source of the exact heartbeat line is identified.
