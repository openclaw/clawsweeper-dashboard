## Summary

Keep open: the instrumentation boundary and supplied proof look strong, but the patch currently has a merge-blocking diagnostics privacy issue because MCP failures can be written to the timeline before the existing URL redaction path runs.

## What This Changes

The PR adds gated diagnostics timeline spans for bundle MCP server startup, connection, and tools/list discovery, plus focused runtime coverage using a stdio MCP probe server.

## Best Possible Solution

Land the focused timeline instrumentation after MCP span error events use redacted messages or omit sensitive details, while preserving aggregate `bundle-tools` timing and existing MCP behavior.

## Reproduction Assessment

Yes for the observability gap. The linked issue and current main source show only aggregate `bundle-tools` timing while per-server connect/list work runs in `src/agents/pi-bundle-mcp-runtime.ts` without timeline spans; I did not run a live multi-server gateway in this read-only review.

## Solution Assessment

No, not yet. Reusing the timeline span helper at the MCP catalog boundary is the right shape, but the patch needs a redacted-error path before it is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Redact MCP errors before timeline emission:** `src/agents/pi-bundle-mcp-runtime.ts:265-269`
