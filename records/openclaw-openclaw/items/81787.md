## Summary

Keep open: the span placement is the right bundle MCP catalog path, but the patch wraps MCP connect/list failures with generic timeline error events that bypass the URL redaction already used by the surrounding warning.

## What This Changes

This PR wraps bundle MCP server startup, connection, and `tools/list` catalog discovery in diagnostics timeline spans and adds a Vitest coverage path using a stdio MCP probe server.

## Best Possible Solution

Keep the timeline-only implementation at the bundle MCP catalog boundary, sanitize or suppress error text for these spans, and add a failure/redaction assertion plus redacted timeline proof.

## Reproduction Assessment

Yes. Source inspection and the linked issue reproduction show current main records only aggregate `bundle-tools` prep timing while per-server `connect` and `tools/list` work has no timeline span.

## Solution Assessment

Partly. The PR uses the narrow maintainable boundary, but it should not merge until timeline error emission is redacted and the PR includes after-fix diagnostics proof beyond test output.
