## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve automatic fallback when tool delivery downgrades:** `src/infra/heartbeat-runner.ts:1692-1693`
  - body: `getReplyFromConfig` can downgrade requested `message_tool_only` delivery to `automatic` when message-tool delivery is unavailable, but this gate still drops every non-tool payload whenever `usesHeartbeatResponseTool` is true. That makes those heartbeat runs silently report `ok-empty` instead of delivering the ordinary fallback reply selected by the delivery contract.
  - confidence: 0.86
- **[P2] Reject non-git dirs before honoring no-update:** `scripts/install.sh:2361`
  - body: When `GIT_UPDATE=0`, an existing directory without `.git` takes the no-update branch and then reaches `run_pnpm -C "$repo_dir" install/build`. Add the same non-git-directory rejection used by `install-cli.sh` before skipping git updates.
  - confidence: 0.9
- **[P2] Identify the preferred plugin by its install path:** `src/infra/npm-managed-root.ts:461`
  - body: Using only `packageName === preferredPackageName` means a nested copy of the same package can overwrite a peer pin already collected from the top-level plugin being installed. Track the preferred package instance by path/realpath so nested duplicates cannot refresh managed peer specs with stale ranges.
  - confidence: 0.82
