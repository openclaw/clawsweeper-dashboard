## Summary

## What This Changes

The PR adds a separate `mcporterEnv` that strips `XDG_CONFIG_HOME` and `XDG_CACHE_HOME` from mcporter spawns and updates the qmd-manager test expectation.

## Best Possible Solution

Keep mcporter's own config discovery out of the per-agent XDG config path while explicitly preserving per-agent QMD env for the qmd MCP server, with regression coverage and redacted live proof.

## Reproduction Assessment

Yes, source-reproducible. The diff removes `XDG_CACHE_HOME` from the mcporter process, while mcporter stdio servers inherit `process.env` and qmd uses `XDG_CACHE_HOME` to choose its SQLite index path.

## Solution Assessment

No. Stripping XDG from mcporter is only half of the fix; the qmd MCP server still needs per-agent QMD env while mcporter's own config discovery must avoid the qmd XDG tree.
