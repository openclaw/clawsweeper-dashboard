The PR adds six curated OpenRouter image model IDs, routes those current IDs through OpenRouter's dedicated `/images` API, parses `data[].b64_json`, and updates provider tests/docs.

## Best Possible Solution

Keep the curated update only after the `/images` path maps or clamps request fields per model capability and the PR includes redacted real OpenClaw proof; leave broader dynamic catalog policy to the linked issue.
