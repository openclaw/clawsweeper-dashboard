## Summary

Keep open. Current main still lacks success-side Feishu delivery-result logs, and this PR’s useful observability direction needs a narrow repair because it would misreport non-thread fallback sends as replies.

## What This Changes

The PR adds Feishu reply-dispatcher success logs for outbound text, card, and media sends and updates dispatcher tests for text-path log output.

## Best Possible Solution

Land a narrow Feishu change that logs actual delivery outcomes from the dispatcher for text, card, and media sends, including direct-send, reply/thread, fallback, and messageId coverage, plus the required changelog entry.

## Reproduction Assessment

Yes. Source inspection and existing fallback tests show non-thread replies to withdrawn or missing Feishu targets can fall back to direct create while the PR logs mode from requested reply metadata.

## Solution Assessment

No for the PR as-is. The maintainable fix should return or derive the actual delivery mode from the send helpers, then have dispatcher logging use that structured outcome instead of sendReplyToMessageId alone.
