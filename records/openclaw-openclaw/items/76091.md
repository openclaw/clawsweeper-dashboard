## Summary

Keep this PR open, but it needs one correctness fix before merge: the carried typing-owner approach is the right direction, yet the current branch still skips preflight typing for accepted default Discord guild/channel turns without a mention, which is the monitored-channel flow this change is meant to improve.

## What This Changes

The PR carries Discord accepted-message typing feedback through queueing and reply dispatch, adds skipped-queue cleanup, updates shared typing callback behavior, and includes related tests/docs plus unrelated oc-path formatting churn.

## Best Possible Solution

Land this PR or a cleaned equivalent after making accepted preflight typing include default message-tool-only guild/channel turns that do not require a mention, while preserving explicit `typingMode` opt-outs and delayed modes.

## Reproduction Assessment

Yes: source inspection on current main shows a DM-only accepted typing cue followed by a separate dispatcher typing owner, and the supplied recording covers the real Discord before/after behavior. I did not perform an independent live Discord run.

## Solution Assessment

No: carrying one typing feedback owner through queueing and dispatch is the right fix shape, but the current gating leaves default no-mention monitored guild/channel turns without the intended immediate typing feedback.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Honor default message-tool typing in guild channels:** `extensions/discord/src/monitor/message-handler.ts:82`
  - body: `shouldStartAcceptedTypingFeedback` now skips accepted guild/group messages without a mention whenever `typingMode` is not explicitly set. Accepted Discord channels with `requireMention: false` default to message-tool-only source delivery, and `resolveTypingMode` treats that as immediate typing so users can see the agent is working even when automatic final replies are suppressed. This means the monitored-channel no-mention flow can still wait until later dispatcher/model output, so please carry the preflight typing owner for accepted no-mention channel turns that do not require a mention while still respecting explicit `message`, `thinking`, and `never` modes.
  - confidence: 0.87
