## Summary

The branch is close, but it still skips accepted-preflight typing for default Discord guild/channel messages without a mention. That conflicts with the documented tool-only group reply policy, so the delayed Discord channel typing case remains partially unfixed.

## What This Changes

The PR carries Discord reply typing feedback from accepted preflight through queued processing and dispatch, adds skipped-run cleanup and queue lifecycle hooks, updates shared typing keepalive behavior, tests, docs, API baseline, and changelog.

## Best Possible Solution

Keep the carried feedback owner, but align accepted-preflight typing with the existing source-reply delivery and typing policy so default tool-only guild/channel turns prestart while explicit typing modes still win.

## Reproduction Assessment

Yes, at source level. Current main has a DM-only accepted cue before queueing and a later dispatcher typing owner after reply-plan resolution, and the related report gives Discord channel delay steps after upgrading to 2026.4.29.

## Solution Assessment

No, not yet. Carrying one feedback owner is the right narrow fix, but the preflight gate must follow the documented tool-only typing policy before this fully fixes delayed Discord channel typing.
