## Summary

Keep open: the PR addresses a real Discord typing lifecycle bug and has sufficient terminal proof, but the current head still changes the default unmentioned guild typing behavior before merge.

## What This Changes

The PR carries Discord reply typing feedback from accepted preflight through queued processing and dispatch, adds skipped-run cleanup and keepalive refresh behavior, and updates tests, changelog, SDK docs, and API hashes.

## Best Possible Solution

Land the carried Discord typing-feedback owner after the accepted-preflight start is gated to the effective instant cases while preserving message, thinking, never, and default unmentioned-group semantics.

## Reproduction Assessment

Yes at source level: current main has a one-shot accepted Discord cue and a later separate dispatcher typing owner. I did not run a live Discord bot in this read-only pass, but the PR body provides real host proof for the observed improvement.

## Solution Assessment

No as written: carrying one owner is the right narrow design, but the current preflight gate treats default unmentioned guild messages as instant. The safer fix is to prestart only when the effective typing mode is instant, or when an unmentioned group was explicitly configured to instant.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve default group typing timing:** `extensions/discord/src/monitor/message-handler.ts:77`
  - body: This treats an unset typing mode as accepted-preflight typing for every accepted Discord message, including unmentioned guild allowlist messages. The typing contract keeps default group chats without a mention in message-on-text mode, so silent or delayed group turns will now show typing immediately. Gate the preflight start on the effective instant cases instead, leaving unmentioned groups to prestart only when `typingMode: "instant"` is explicitly configured.
