## Summary

This PR should stay open because the latest head still skips accepted-preflight typing for default no-mention Discord guild/channel turns, which is the delayed channel path the fix is meant to cover.

## What This Changes

The PR carries Discord reply typing feedback from accepted preflight through queued processing and dispatch, adds skipped-queue cleanup plus shared typing keepalive changes, and updates focused tests, docs, changelog, and the SDK API hash.

## Best Possible Solution

Keep the single carried feedback owner, but derive accepted-preflight typing from the same source-reply delivery and typing policy used by dispatch so default message_tool_only guild/channel replies prestart while explicit typing modes still win.

## Reproduction Assessment

Yes, at source level: current main has a DM-only accepted cue before queueing and a separate dispatcher typing owner after Discord reply-plan resolution. The related report at https://github.com/openclaw/openclaw/issues/76396 also describes delayed Discord channel typing after upgrading to 2026.4.29.

## Solution Assessment

No, not yet: carrying one feedback owner is the narrow maintainable fix, but the accepted-preflight gate must align with the existing tool-only group/channel typing policy before this broadly fixes delayed Discord channel typing.
