## Summary

Keep open: current main still has split Discord typing owners, and this PR addresses that by carrying one feedback owner through the accepted-message, queued-run, and reply-dispatch lifecycle. I did not find a blocking correctness or security finding in the diff, but the PR is still draft and needs normal readiness, CI, and maintainer review before merge.

## What This Changes

The PR threads Discord reply typing feedback from accepted preflight through queued processing and reply dispatch, adds skipped-run cleanup hooks to the channel run queue, refreshes repeated typing starts, and updates tests, docs, changelog, and the SDK API baseline.

## Best Possible Solution

Land a ready version that preserves one carried Discord typing owner, retargets it after reply-plan resolution, and keeps the additive queue skip cleanup API covered by focused tests and SDK API validation.
