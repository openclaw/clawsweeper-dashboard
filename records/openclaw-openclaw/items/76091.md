## Summary

Keep this PR open: the carried typing-owner design is the right direction, but the current branch still skips accepted preflight typing for default Discord guild/channel allowlist turns without a mention, so the delayed first typing signal can remain for the reported channel setup.

## What This Changes

The PR carries Discord reply typing feedback from accepted preflight into queued processing and dispatch, adds skipped-queue cleanup plus shared typing keepalive changes, and updates tests, docs, changelog, and the SDK API hash.

## Best Possible Solution

Keep the carried-owner design, but make accepted-preflight typing use the same source-reply delivery and typing policy as dispatch for Discord guild/channel messages, especially default message_tool_only allowlist no-mention turns.

## Reproduction Assessment

Yes at source level: current main has a DM-only accepted cue before queueing and a separate dispatcher typing owner after Discord reply-plan resolution. The current PR still skips the default unmentioned guild/channel path even though the later runtime treats tool-only group replies as instant typing.

## Solution Assessment

No, not yet: carrying one feedback owner is the narrow maintainable fix, but the accepted-preflight gate must align with the existing tool-only group typing policy before this solves the Discord channel delay broadly.
