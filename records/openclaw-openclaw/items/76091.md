## Summary

Keep open: current main does not contain this carried Discord typing lifecycle change, and the latest PR head still leaves default no-mention Discord guild/channel messages without accepted-preflight typing despite the existing tool-only instant typing contract.

## What This Changes

The PR carries Discord reply typing feedback across preflight, queue, processing, and dispatch; adds skipped-queue cleanup; updates shared typing/run-queue contracts, tests, docs, and changelog; and includes a small shell-inline lint cleanup.

## Best Possible Solution

Keep the carried feedback owner, but derive accepted-preflight typing from the same source-reply delivery and typing policy used by dispatch so default tool-only guild/channel replies prestart while explicit typing modes still win.

## Reproduction Assessment

Yes, at source level: current main has a DM-only accepted cue before queueing and a separate dispatcher typing owner after Discord reply-plan resolution. The related user report at https://github.com/openclaw/openclaw/issues/76396 gives a Discord channel delay reproduction after upgrading to 2026.4.29.

## Solution Assessment

No, not yet: carrying one feedback owner is the narrow maintainable fix, but the accepted-preflight gate must align with the existing tool-only group/channel typing policy before this broadly fixes delayed Discord channel typing.
