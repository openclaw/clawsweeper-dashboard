## Summary

Keep this PR open: it is the active scoped fix path for the Discord accepted typing regression, but the current patch disables periodic Discord typing refresh on the carried feedback owner, so long queued or tool-heavy replies can still go visually idle before cleanup. The remaining changelog/docs issues are mechanical, while the public `ChannelRunQueue` `onSkip` contract still deserves maintainer attention before merge.

## What This Changes

The PR carries Discord accepted reply typing feedback from preflight through queued processing and dispatch, adds skipped-run cleanup support to the shared channel run queue, updates SDK docs/changelog, and adds focused typing/queue tests.

## Best Possible Solution

Keep the scoped Discord accepted-typing owner, restore periodic Discord typing refresh until idle/cleanup, remove release-owned changelog churn, collapse the duplicated SDK docs row, and have maintainers explicitly accept or avoid the public `onSkip` SDK contract.

## Reproduction Assessment

Yes by source/proof, not by a fresh live run in this review. Current main has the Discord guild accepted-typing gap described by the related reports, and the PR-introduced keepalive regression is clear from `keepaliveIntervalMs: 0` plus Discord's documented 10-second typing expiry.

## Solution Assessment

No, not yet. Carrying one Discord typing owner from preflight through dispatch is the right shape, but disabling periodic refresh undercuts the fix for long queued or tool-heavy runs; restoring keepalive is the narrower maintainable repair.
