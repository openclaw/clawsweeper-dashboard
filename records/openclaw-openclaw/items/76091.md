## Summary

Keep open: the Discord typing lifecycle fix is valid and has real before/after proof, but the current branch still contains an unrelated cron-store behavior change that can retain malformed jobs and send them back into scheduler code that expects a valid schedule.

## What This Changes

This PR carries Discord reply typing feedback from accepted preflight through queued processing and reply dispatch, adds skipped-queue cleanup and focused tests, and includes several unrelated conflict-repair touches outside Discord.

## Best Possible Solution

Land this PR or a clean equivalent after removing the unrelated cron-store retention change, preserving the single carried Discord typing-feedback owner and leaving the broader persistent-indicator race to https://github.com/openclaw/openclaw/pull/75403.

## Reproduction Assessment

Yes. Current main source shows the DM-only accepted typing gate plus a later separate dispatcher typing owner, and the PR recording plus user corroboration demonstrate the Discord guild allowlist case with typingMode set to instant.

## Solution Assessment

Mostly yes. Carrying one Discord-owned typing feedback object from accepted preflight through queue and dispatch is the narrow maintainable fix, but the unrelated cron-store behavior must be removed or split before merge.
