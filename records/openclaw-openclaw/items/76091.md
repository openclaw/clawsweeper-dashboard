## Summary

Keep this PR open: the Discord typing-lifecycle fix is well-supported, but the branch still introduces unrelated cron-store behavior that can retain malformed schedule records and feed them into runtime paths that expect a valid `job.schedule.kind`.

## What This Changes

The PR carries one Discord reply typing feedback owner from accepted preflight through queued processing and dispatch, updates shared queue/typing hooks, and includes conflict-repair test/docs churn.

## Best Possible Solution

Rebase the branch, remove or split the unrelated cron invalid-job retention, then land this Discord typing fix or a clean equivalent while leaving the broader persistent-indicator race to https://github.com/openclaw/openclaw/pull/75403.

Yes, at source level. Current main still gates accepted Discord typing to DMs, and the supplied real Discord recording plus https://github.com/openclaw/openclaw/issues/79104 cover the guild allowlist `typingMode: instant` case.
