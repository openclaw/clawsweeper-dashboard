## Summary

Keep open: the PR still skips carried preflight typing for default no-mention Discord guild/channel turns, even though those turns resolve to tool-only visible replies that should get instant typing. Real terminal proof is sufficient and the security pass is clear, but the P2 correctness blocker needs repair before merge.

## What This Changes

## Best Possible Solution

Keep the carried feedback owner, but align the accepted-preflight gate and tests with default tool-only group/channel typing so accepted no-mention allowlist messages start feedback immediately while explicit typing modes still win.
