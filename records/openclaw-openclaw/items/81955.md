## Summary

Current main addresses the reported Ollama persona/context loss through the native Ollama `num_ctx` compatibility fix that followed v2026.5.12; the issue matches the already-fixed regression where OpenClaw showed large injected context while Ollama fell back to a small runtime context and truncated the prompt.

## What This Changes

The issue reports that after updating to v2026.5.12, agents using Ollama stop following injected workspace/persona files such as `IDENTITY.md`, `SOUL.md`, and `BOOTSTRAP.md`.

## Best Possible Solution

Keep the current main fix that migrates explicit native Ollama context budgets into `params.num_ctx`, propagates those params to resolved models, and applies the migration through doctor/update; request a new focused report only if the behavior persists after that fix is installed.

## Reproduction Assessment

Yes. The high-confidence path is a native Ollama config with `contextWindow` or `maxTokens` but no `params.num_ctx` on v2026.5.12, where Ollama receives a small runtime context and truncates the system prompt despite OpenClaw listing injected files.

## Solution Assessment

Yes. The current fix is the narrow compatibility path: preserve explicit-only native Ollama runtime behavior, migrate older explicit budgets into `params.num_ctx`, and make provider params reach auto-discovered models.
