Full review comments:

- **[P2] Remove Claude Code from provider examples:** `docs/install/xcloud.mdx:15`
  - body: This lists `Claude Code` alongside Anthropic, OpenAI, and Gemini as an AI provider, and repeats the same wording later in the credential step. Current docs treat Claude CLI as an Anthropic auth/runtime path, not a separate provider, so this can send users down the wrong credential path.
  - confidence: 0.95
- **[P2] Document the hosted Control UI auth path:** `docs/install/xcloud.mdx:101-103`
  - body: The guide tells users to collect a hosted Control UI URL and Basic Auth password, but does not state the generated Gateway auth mode, `gateway.controlUi.allowedOrigins`, device pairing, trusted proxy, or secure ingress setup. Current Gateway config requires auth and explicit browser origins for non-loopback Control UI deployments, so the official guide needs the exact xCloud setup before endorsement.
  - confidence: 0.88
