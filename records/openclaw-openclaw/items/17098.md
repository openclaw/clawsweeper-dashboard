## Summary

Keep open: current main still has no xCloud docs, and the PR needs maintainer/provider validation plus docs fixes before OpenClaw should publish it as official hosted deployment guidance.

## What This Changes

The PR adds an xCloud managed-VPS hosting guide, docs navigation and redirects, and an xCloud card on the VPS provider page.

## Best Possible Solution

Publish the xCloud guide only after the hosted deployment is verified against current Gateway, Control UI, provider-auth, Telegram, and remote-access contracts, then revise the page to document the exact generated configuration and security tradeoffs.

## Reproduction Assessment

Not applicable because this is a docs feature PR rather than a runtime bug. The high-confidence verification path is source and docs inspection: current main lacks xCloud coverage, while the PR head exposes provider wording and hosted-Control-UI contract gaps.

## Solution Assessment

No, the PR is not the best solution as-is. A provider-specific xCloud page may be appropriate, but official docs should not endorse the hosted flow until auth, allowed-origin, device-pairing, credential, and allowlist behavior is verified and documented.
