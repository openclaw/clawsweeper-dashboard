## Summary

Keep open: this is useful docs-only work and the author addressed the earlier wording nits, but the hosted xCloud Control UI and Telegram setup claims still need provider or owner validation plus real deployment proof before becoming official docs.

## What This Changes

The PR adds an xCloud hosting guide, an xCloud card on the VPS picker, and docs navigation plus redirects for `/xcloud` and `/platforms/xcloud`.

## Best Possible Solution

Publish the guide only after a maintainer or xCloud/provider owner verifies a real deployment and the page documents the exact Gateway auth, allowed origins, device identity, proxy/TLS, recovery, and Telegram allowlist contracts.

## Reproduction Assessment

Not applicable: this is a docs/provider-guide PR, not a runtime bug. The meaningful validation path is PR-head docs inspection plus real xCloud deployment proof, and the current evidence only proves the docs files and navigation shape.

## Solution Assessment

No, not as-is. A provider-specific official guide is maintainable only if it either documents the exact hosted security contracts after validation or stays out of official docs until that proof exists.
