## What This Changes

The PR changes `models auth paste-token` to route interactive token entry through the masked `@clack/prompts.password()` path and updates the models auth tests to use the password mock.

## Best Possible Solution

Land the narrow masked prompt change after normal draft readiness and CI/security review, keeping storage and config behavior unchanged.

## Reproduction Assessment

Yes. Current main and `v2026.6.1` route `paste-token` through the unmasked text prompt, and the PR body includes a before screenshot showing the token echoed in Windows PowerShell.

## Solution Assessment

Yes. Flipping the existing `masked` flag at the `paste-token` call site is the narrowest maintainable fix; changing `readPastedSecret` defaults would risk other callers and docs-only guidance would not stop terminal echo.
