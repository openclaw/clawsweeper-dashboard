## Summary

Keep open: the PR fixes the reported configured-cap path only for Anthropic 1M Opus/Sonnet routes, while the same Claude CLI runtime-alias mismatch remains for non-1M Anthropic models.

## What This Changes

The PR changes context-token resolution and regression tests so Claude CLI Anthropic Opus sessions can prefer configured `contextTokens` over the 1M runtime window.

## Best Possible Solution

Make the explicit-provider config scan runtime-alias-aware for all configured model caps, then keep the Anthropic 1M fallback only when no explicit `contextTokens` cap is configured.

## Reproduction Assessment

Yes, by source inspection: current main persists session budgets from Claude CLI runtime metadata, while the documented cap lives under the canonical Anthropic provider. I did not run a live Claude CLI compaction session in this read-only review.

## Solution Assessment

No. The PR handles the 1M Opus/Sonnet override case, but alias-aware configured-cap lookup should also cover non-1M Anthropic models routed through `claude-cli`.
