## Summary

Keep open: the reported BlueBubbles SecretRef webhook-auth bug is valid on current main, but this PR does not actually resolve SecretRef objects and would fail closed for SecretRef-backed passwords.

## What This Changes

The PR imports the BlueBubbles secret-input helper and wraps the webhook auth password trim in `normalizeSecretInputString()`.

## Best Possible Solution

Resolve BlueBubbles webhook passwords through the runtime SecretRef resolver before constant-time auth comparison, with focused webhook regression coverage and a changelog entry.

## Reproduction Assessment

Yes, by source inspection: current main accepts SecretRef passwords but the webhook auth callback calls `.trim()` on the raw password value. I did not execute Vitest in this read-only review.

## Solution Assessment

No. The direction is right, but this diff uses a string-normalization helper that does not resolve SecretRefs; the narrow maintainable fix should use the runtime secret-resolution path before comparing the token.
