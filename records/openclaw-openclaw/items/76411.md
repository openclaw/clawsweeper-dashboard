## Summary

Keep this PR open, but do not merge it as-is. The branch targets the right BlueBubbles webhook-auth bug, but `normalizeSecretInputString` does not resolve SecretRef objects, so the proposed line changes a crash into a failed auth match while inbound SecretRef-backed webhooks are still dropped.

## What This Changes

This PR imports `normalizeSecretInputString` in the BlueBubbles monitor and wraps the webhook password before trimming it during auth.

## Best Possible Solution

Resolve the BlueBubbles password to the runtime string before webhook target registration and catchup use, keep constant-time comparison, add SecretRef-shaped regression coverage, and add the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a BlueBubbles account whose `password` remains a SecretRef object reaches the webhook auth matcher, and current main calls `.trim()` on that object before request handling continues.

## Solution Assessment

No. The one-line wrapper is not the best fix because `normalizeSecretInputString` only accepts already-string values; the maintainable fix should use the runtime-resolved secret value before synchronous webhook auth runs.

## Review Findings
