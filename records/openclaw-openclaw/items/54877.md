## Summary

Keep open. Current main still has the reported CLI mismatch: doctor detects missing recent transcript files, but the preview command it prints omits the flag that the cleanup preview requires before it counts missing transcript pruning.

## What This Changes

The issue asks for the missing-transcript doctor cleanup preview to include `--fix-missing` or otherwise make `sessions cleanup --dry-run` match the doctor warning.

## Best Possible Solution

Make the doctor missing-transcript preview hint include `--dry-run --fix-missing`, update the focused state-integrity test to assert that exact preview command, and keep the existing cleanup command semantics flag-gated.

## Reproduction Assessment

Yes. The issue body gives concrete CLI steps, and current main has a source-level reproduction path: a missing recent transcript makes doctor print a dry-run preview without `--fix-missing`, while cleanup only counts missing entries when that flag is true.

## Solution Assessment

Yes. Correcting the doctor hint is the narrowest maintainable fix because `--fix-missing` is already the documented explicit opt-in for previewing missing-transcript pruning; changing global dry-run semantics would be broader.
