## What This Changes

`openclaw doctor` should suggest a missing-transcript cleanup preview that includes `--fix-missing` so the dry run matches the warning it just printed.

## Best Possible Solution

Update the doctor missing-transcript preview hint to include `--dry-run --fix-missing`, add focused regression coverage, and preserve cleanup’s explicit flag-gated pruning behavior.

## Reproduction Assessment

Yes. The issue body gives concrete CLI steps, and current main has a source-level reproduction path: doctor emits a dry-run preview without `--fix-missing`, while cleanup only counts missing transcript pruning when that flag is true.

## Solution Assessment

Yes. Correcting the doctor hint is the narrowest maintainable fix because cleanup already documents `--dry-run --fix-missing` as the explicit missing-transcript preview path, and changing dry-run semantics globally would be broader.
