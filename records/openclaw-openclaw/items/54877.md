## Summary

Keep open. Current main still has the reported mismatch: doctor detects missing recent transcript files, but its preview command omits the `--fix-missing` flag that `sessions cleanup --dry-run` requires before it counts missing transcript pruning.

## What This Changes

The issue asks for the missing-transcript doctor cleanup preview to include `--fix-missing` or otherwise make the dry-run preview match the doctor warning.

## Best Possible Solution

Make the doctor missing-transcript preview hint include `--dry-run --fix-missing`, and update the focused doctor state-integrity test to assert that exact preview command while leaving cleanup behavior flag-gated.

## Reproduction Assessment

Yes. The issue body gives a concrete CLI reproduction, and current main has a source-level reproduction path: a missing recent transcript causes doctor to print a dry-run preview without `--fix-missing`, while cleanup returns a missing count only when that flag is true.

## Solution Assessment

Yes. The narrowest maintainable fix is to correct the doctor hint, because `--fix-missing` is already documented as the explicit cleanup-preview opt-in and changing dry-run semantics globally would be broader than the reported bug.
