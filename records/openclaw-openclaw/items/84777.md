## Summary

Keep open: this is a recent, detailed production report with a second cross-platform confirmation, and current main still has compaction liveness/wait paths that do not prove the reported Pi post-summary freeze is fixed.

## What This Changes

Fix Pi compaction so successful summary generation cannot leave an agent session deadlocked and silent across all channels until a fresh session is created.

## Best Possible Solution

Add a bounded compaction recovery path that reconciles success, failure, timeout, abort, and lost completion so session liveness, wait promises, and transcript locks always converge without requiring `/new`.

## Reproduction Assessment

No high-confidence runnable current-main reproduction was established in this read-only review. The source path is clear enough to treat as source-reproducible, and two production reports describe the same post-compaction silent-agent behavior.

## Solution Assessment

No patch is proposed yet, and the reporter's workaround is not the final fix. The maintainable solution should reconcile compaction completion and cleanup state rather than relying on `truncateAfterCompaction` or lower compaction frequency.
