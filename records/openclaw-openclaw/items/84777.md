## What This Changes

The issue reports an agent-scoped freeze after compaction summary generation, with no further transcript writes and recovery only by starting a fresh session.

## Best Possible Solution

Add a bounded recovery/reconciliation path so compaction success, failure, timeout, or lost completion cannot leave the agent runtime paused, compacting, or pinned to unusable session state.

## Reproduction Assessment

No high-confidence runnable current-main repro is available from this review. Source inspection does show a plausible stuck path where compaction liveness/wait state can keep the session lane blocked if completion is lost or never reconciled.

## Solution Assessment

Unclear as a final fix: enabling truncateAfterCompaction is a useful workaround but does not solve the liveness cleanup problem. The maintainable fix should make compaction completion and timeout cleanup converge session state even when summary generation or retry handling finishes late.
