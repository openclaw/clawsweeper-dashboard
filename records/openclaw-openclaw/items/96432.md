## What This Changes

The PR changes Gemini media-understanding output extraction to search backward for a parseable JSON object and adds a regression test for nested JSON metadata.

## Best Possible Solution

Land this narrow parser fix after normal CI, keeping the behavior in `packages/media-understanding-common` with regression coverage instead of duplicating Gemini-specific parsing in the runner.

## Reproduction Assessment

Yes. Current main's source uses `lastIndexOf("{")`, and the read-only shell check showed that nested Gemini JSON returns `null` under the old algorithm while the proposed scan finds the top-level response object.

## Solution Assessment

Yes. The shared output extractor is the narrow owner boundary because `runCliEntry` already delegates Gemini CLI stdout parsing there; moving the fix into the runner would duplicate parser policy.
