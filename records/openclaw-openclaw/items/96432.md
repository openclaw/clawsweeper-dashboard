## Summary

Keep open: current main and the latest release still use the `lastIndexOf("{")` extractor that drops nested Gemini JSON responses, while this PR is a narrow parser/test fix with no blocking review finding.

## What This Changes

This PR changes the media-understanding Gemini output extractor to recover complete JSON objects from noisy stdout and adds regression tests for nested JSON, quoted noise, and malformed trailing records.

## Best Possible Solution

Land the narrow shared extractor fix after maintainer review, keeping Gemini JSON recovery in `packages/media-understanding-common` rather than duplicating it in the runner.

## Reproduction Assessment

Yes, at source level. Current main chooses `trimmed.lastIndexOf("{")`, and a read-only shell check of that exact algorithm returns null for noisy stdout followed by a JSON response containing nested `usage`.

## Solution Assessment

Yes. The shared media-understanding extractor is the narrow owner boundary because `resolveCliOutput` already delegates Gemini stdout parsing there; reusing the agent CLI parser directly would cross into a broader agent-owned parser path.
