## Summary

Keep open. Current main still has the direct-thread silent-reply classification gap, but this PR is not ready to merge because its new thread-suffix branch treats the shared parser result as a trust signal, while current main preserves caller-shaped explicit session keys and the parser does not prove an internal producer.

## What This Changes

The PR imports `parseThreadSessionSuffix` into silent-reply classification, returns `internal` for parsed `:thread:` suffixes before direct/dm handling, adds direct-thread and cron-thread regression tests, and adds a changelog entry.

## Best Possible Solution

Land a refined narrow fix that keeps trusted generated internal thread sessions silent without letting arbitrary direct session keys override direct-chat silent-reply policy, with coverage for internal direct-thread, cron/internal-thread, user-facing group/channel-thread, and malformed/caller-shaped suffix cases.

## Reproduction Assessment

Yes. Source inspection on current main shows `agent:main:telegram:direct:123:thread:99` reaches the direct branch, and the outbound planner rewrites `NO_REPLY` for direct conversations under the default disallow/rewrite settings.

## Solution Assessment

No. The PR is close in scope, but using `parseThreadSessionSuffix` as the trust boundary is not the best fix; a trusted internal-thread predicate or explicit metadata would be safer and more maintainable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Validate thread provenance before returning internal:** `src/shared/silent-reply-policy.ts:81-83`
  - body: The new branch relies on `parseThreadSessionSuffix()` as a trust signal, but that parser accepts any non-empty final `:thread:` tail and `sessions.create` preserves caller-provided keys. A caller-shaped direct key such as `agent:main:telegram:direct:123:thread:x` would therefore be classified as internal and skip the direct rewrite defaults; gate this branch on trusted generated thread metadata or a stricter internal-session predicate before returning `internal`.
  - confidence: 0.86
