## Summary

Keep this PR open: it fixes the right timeout mismatch for the main Codex dynamic-tool path, but the current patch is still too broad for non-lifecycle `skill_workshop` actions and too narrow for the sibling side-thread Codex path.

## Best Possible Solution

Use an action-aware timeout resolver shared by main and side Codex dynamic-tool paths, extending only `apply`, `reject`, and `quarantine` above the approval ceiling while preserving the generic 90s watchdog for non-lifecycle actions.

## Solution Assessment

No, not yet: raising the lifecycle timeout is the right direction, but the current implementation is tool-wide and misses the sibling side-thread dynamic-tool resolver. The maintainable fix is action-aware shared timeout logic with tests proving lifecycle and non-lifecycle behavior.
