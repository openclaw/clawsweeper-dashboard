## Best Possible Solution

Make the main and /btw timeout decision cover the final Skill Workshop lifecycle approval path after hook-adjusted params, while preserving 90s behavior for truly non-lifecycle calls that cannot require that approval.

Yes. Current source shows a 90s Codex dynamic-tool default racing a 120s plugin approval wait plus gateway grace, and the existing before-tool-call test shows hooks can rewrite Skill Workshop params before final approval.

## Solution Assessment

No. The direct lifecycle timeout branch is a good part of the fix, but using only raw Codex arguments is not the best complete solution because final approval can be decided after hook-adjusted params.
