## Summary

Keep open: current main still has the source-level false-negative path where an immediate `claude-cli` transcript probe can invalidate a stored session as `missing-transcript`. Adjacent fixes now improve embedded transcript persistence and raw OpenClaw history reseeding, but they do not make the Claude transcript probe tolerate flush or path-lag races; an open PR is already targeting that specific gap.

## What This Changes

Prevent chat sessions using `claude-cli` from losing context when OpenClaw invalidates a resumable session as `missing-transcript` before Claude’s JSONL transcript is visible or populated.

## Best Possible Solution

Keep this as the canonical tracker and land a narrow fix that preserves phantom-binding protection while making the Claude CLI transcript probe deterministic or freshness-aware, with regression tests and real Claude CLI chat proof.

## Reproduction Assessment

Yes, source-reproducible with high confidence: on current main, a stored `claude-cli` binding plus a transcript file that is absent or lacks an assistant record makes the helper return false, and both resume paths map that result to `missing-transcript`. I did not execute the live Telegram/Claude flush race locally.

## Solution Assessment

Yes in direction: a narrow probe fix with focused tests is the maintainable path, and the open PR is the right work item to review rather than creating a duplicate repair branch. The separate LCM maintenance gap in https://github.com/openclaw/openclaw/issues/81558 should stay separate because it does not replace the probe-race fix.
