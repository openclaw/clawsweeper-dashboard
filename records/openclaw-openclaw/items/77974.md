## Summary

Keep open: current main still has the source-level false-negative path for the reported claude-cli `missing-transcript` race, and the remaining fix is being worked in a narrower open PR rather than being implemented on main yet.

## What This Changes

The issue requests that claude-cli resume validation tolerate transcript flush latency before dropping a saved session and starting without history.

## Best Possible Solution

Land a focused fix that preserves the stale phantom-session guard but makes the Claude transcript probe deterministic or freshness-aware, then handle the LCM maintenance gap in its separate tracker.

## Reproduction Assessment

Yes, source-reproducible: a matching Claude CLI transcript that is absent or lacks an assistant record at probe time returns false, and both current resume paths treat that as a missing transcript. I did not run a live Telegram/Claude CLI timing repro in this review.

## Solution Assessment

Yes in direction: the narrow fix should be in the shared Claude CLI transcript probe and its call sites, preserving the phantom-binding protection while avoiding false negatives from flush timing. The current open PR is the right repair lane, but it still needs its review finding resolved before landing.
