## Summary

Keep open: current main still has the one-shot Claude CLI transcript-content probe that can turn a transient flush miss into `missing-transcript` invalidation, and the active fix PRs are not merged yet.

## What This Changes

The issue requests a fix for claude-cli chat sessions losing context when OpenClaw invalidates a reusable session before the Claude transcript has flushed.

## Best Possible Solution

Land a focused Claude CLI resume fix that avoids false transcript-missing invalidation, does not persist unflushed or ephemeral session ids, and preserves stale-session protection with regression tests plus redacted live proof.

## Reproduction Assessment

Yes, at source level: current main maps a false one-shot Claude transcript probe to `missing-transcript` invalidation in both CLI preparation and direct command paths. I did not run a live Telegram/Claude CLI scenario in this read-only review.

## Solution Assessment

Yes, the narrow fix direction is to make the Claude transcript/resume validation freshness-aware and gate bad session-id persistence while keeping the stale-binding guard. Closing this issue before the active PRs merge would be premature.
