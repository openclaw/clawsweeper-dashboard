## Summary

Keep open: current main still invalidates saved `claude-cli` resume state from an immediate transcript-content miss, and the later discussion confirms this is the canonical tracker for the remaining resume-side context-loss path rather than a solved Telegram-only report.

## What This Changes

The issue reports context loss in continuing `claude-cli` sessions when OpenClaw marks a saved resume binding as `missing-transcript` before the Claude transcript is visible or usable.

## Best Possible Solution

Keep the stale-transcript guard, but make it freshness-aware and non-destructive until a bounded retry or equivalent liveness check proves the Claude CLI resume target is truly unusable.

## Reproduction Assessment

Yes, source-reproducible: current main still routes a saved `claude-cli` binding through a one-shot transcript probe and immediately disables resume when that probe returns false. I did not run a live Claude CLI/channel scenario in this read-only review.

## Solution Assessment

Yes. A narrow bounded retry, freshness guard, or equivalent liveness check around the Claude CLI transcript probe is the best repair, with the existing phantom-binding invalidation retained after the wait expires.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/77974. Current main immediately invalidates a saved `claude-cli` resume binding when `claudeCliSessionTranscriptHasContent` returns false, which can race transcript flush or live binding visibility and produce `useResume=false session=none historyPrompt=none`. Add a bounded freshness-aware retry/backoff or equivalent liveness guard before returning false for the Claude CLI transcript probe, and apply it consistently to the `prepareCliRunContext` path and the direct `runAgentAttempt` binding-clear path. Preserve the stale/phantom transcript protection from https://github.com/openclaw/openclaw/pull/77030 after the bounded wait expires. Add regression coverage where the injected transcript check fails initially and then succeeds, and where it remains false and still invalidates.
