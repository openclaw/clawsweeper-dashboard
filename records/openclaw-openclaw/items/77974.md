## Summary

Keep open: current main still makes a one-shot Claude CLI transcript-content decision and invalidates/clears resume state when that immediate probe is false, so the reported context-loss path remains source-backed.

## What This Changes

Prevent continuing `claude-cli` channel sessions from losing context when `missing-transcript` invalidation races or misclassifies Claude transcript availability.

## Best Possible Solution

Keep the stale/phantom transcript guard, but make the Claude CLI transcript probe freshness-aware with bounded retry/backoff before invalidation, preserve or reseed context when reset is unavoidable, and cover both prepare and user-run paths with regression tests.

## Reproduction Assessment

Yes, source-reproducible: current main performs a single Claude CLI transcript scan and invalidates or clears saved resume state when that immediate check is false. I did not run a live Telegram or WebChat Claude CLI scenario in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is a bounded freshness-aware retry or equivalent liveness distinction around the existing Claude CLI transcript probe, without removing the stale-binding protection or adding a new config option.
