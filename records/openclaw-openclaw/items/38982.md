## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Seed runtime theme from current detection:** `src/tui/theme/theme.ts:191-192`
  - body: The branch initializes `palette` and `currentThemeName` to `default`, so users who currently start in light mode through `OPENCLAW_THEME=light` or a light `COLORFGBG` background will start in the dark default until they manually run `/theme`. Seed the runtime theme state from current main's detection result instead of hard-coding `default`.
  - confidence: 0.9
- **[P2] Refresh cached TUI chrome on theme changes:** `src/tui/tui-command-handlers.ts:253-265`
  - body: `applyTheme(...)` only requests a render, but the header, footer, and idle status text are already formatted with ANSI colors and are recomputed by update hooks rather than by rendering alone. Refresh those cached strings on preview, select, and cancel so live preview does not leave mixed old/new theme colors.
  - confidence: 0.88
- **[P2] Rebase off the old TUI renderer imports:** `src/tui/theme/theme.ts:7-9`
  - body: Current main moved the TUI to `@earendil-works/pi-tui` and removed direct `cli-highlight` usage from this renderer, but this branch imports the old Pi package and `cli-highlight`. Rebase onto the current imports and keep dependency changes separate from the picker feature.
  - confidence: 0.86
- **[P3] Document the theme command before shipping:** `src/tui/commands.ts:113-120`
  - body: This adds a user-facing `/theme` picker/list flow, but the branch does not update the public TUI documentation. Add docs for the command, picker behavior, and runtime-only scope before shipping the feature.
  - confidence: 0.78
