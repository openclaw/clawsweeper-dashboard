## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Seed runtime theme from current detection:** `src/tui/theme/theme.ts:191-192`
  - body: Current main selects the light palette at startup when `OPENCLAW_THEME=light` or `COLORFGBG` indicates a light background, but this branch initializes runtime state from hard-coded `default`. That would force light-background users back to the dark palette until they manually switch themes.
  - confidence: 0.91
- **[P2] Refresh cached chrome after theme changes:** `src/tui/tui-command-handlers.ts:253-265`
  - body: `applyTheme(...)` only requests a render, while the header, status, and footer are already formatted into `Text` nodes and recomputed by their update functions. Preview, select, cancel, and direct apply need to refresh those strings or live preview can leave mixed old/new colors until another event updates the chrome.
  - confidence: 0.88
- **[P2] Rebase onto current TUI renderer imports:** `src/tui/theme/theme.ts:1-9`
  - body: The branch still imports `@mariozechner/pi-tui` and `cli-highlight` in the TUI theme path, while current main uses `@earendil-works/pi-tui` and no longer imports `cli-highlight` there. Rebase before merging so the feature builds against the current package surface.
  - confidence: 0.86
