## Summary

Keep open. Current main still does not implement the interactive `/theme` picker, but this PR is not merge-ready because it overlaps #44382, regresses current startup theme detection, leaves cached TUI chrome stale during live preview, and includes unrelated secret-baseline churn.

## What This Changes

The PR adds a runtime TUI `/theme` slash command with a searchable live-preview picker, six mutable palettes, terminal OSC color helpers, syntax theme variants, and related tests while also changing `.secrets.baseline`.

## Best Possible Solution

Land one canonical TUI theming path that preserves current startup detection, adds the picker/runtime switching cleanly, recomputes cached chrome on preview/apply/revert, updates docs and changelog, and excludes unrelated baseline metadata.

## Reproduction Assessment

Yes. Source inspection of the PR branch shows the runtime state starts at `default` instead of current main's detected palette, and theme preview only requests render while TUI chrome is stored as preformatted `Text` strings.

## Solution Assessment

No. The feature direction is reasonable, but this patch is not the best mergeable solution until maintainers choose between it and #44382 and the chosen branch preserves current detection, refreshes cached chrome, documents the user-facing command, and removes unrelated security-baseline churn.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve startup theme detection:** `src/tui/theme/theme.ts:191-192`
  - body: The new mutable runtime state always starts from `themes.default`, so merging this over current main would drop the existing `OPENCLAW_THEME` and `COLORFGBG` light-background detection until users manually run `/theme`. Seed the runtime state from the existing detection result instead of hard-coding `default`.
  - confidence: 0.86
- **[P2] Recompute TUI chrome after theme changes:** `src/tui/tui-command-handlers.ts:253-254`
  - body: `applyTheme(..., tui)` mutates formatter functions and requests a render, but the header, footer, and idle status are stored as already formatted `Text` strings. Preview, commit, and cancel can leave those bars in the old ANSI colors until another state update recomputes them.
  - confidence: 0.88
- **[P3] Add docs and changelog for the new command:** `src/tui/commands.ts:110-117`
  - body: This adds a user-facing slash command and picker, but the branch does not update public TUI docs or `CHANGELOG.md`. Add the active-release changelog entry and document `/theme` usage before merge.
  - confidence: 0.82
