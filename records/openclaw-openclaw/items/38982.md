## Summary

Keep open: current main still lacks a runtime `/theme` picker, but this PR has blocking integration issues, missing real behavior proof, and overlaps a newer open TUI theme-system PR, so it is not a conservative cleanup close.

## What This Changes

The PR adds a TUI `/theme` command with a searchable live-preview picker, six runtime palettes, terminal color helpers, syntax theme changes, tests, and unrelated `.secrets.baseline` churn.

## Best Possible Solution

Land one canonical TUI theming design that preserves current light/dark detection, updates mounted and cached TUI chrome on runtime theme changes, documents the user-facing command, and excludes unrelated baseline churn.

## Reproduction Assessment

Yes for the blocking PR issues: source inspection shows the PR hard-codes default runtime theme state and only requests render on theme changes, while current main stores header/footer/status as preformatted `Text` strings. I did not run the TUI because this review was read-only.

## Solution Assessment

No. The feature direction is plausible, but this branch is not the best merge shape until it is rebased onto current theme detection and dependency cleanup, coordinated with the overlapping theme-system PR, documented, and stripped of unrelated baseline churn.
