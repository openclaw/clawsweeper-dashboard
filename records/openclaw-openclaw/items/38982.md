Full review comments:

- **[P2] Seed runtime theme from current detection:** `src/tui/theme/theme.ts:191-192`
  - body: Current main selects the light palette at startup when `OPENCLAW_THEME=light` or `COLORFGBG` indicates a light background, but this branch initializes `palette` and `currentThemeName` from the hard-coded `default` theme. That would force light-background users back to the dark palette until they manually switch themes.
  - confidence: 0.91
- **[P2] Refresh cached chrome after theme changes:** `src/tui/tui-command-handlers.ts:253-265`
  - body: `applyTheme(...)` only requests a render, while the header, status, and footer are already formatted into `Text` nodes and recomputed by their update functions. Preview, select, and cancel need to refresh those cached strings or the live preview can leave mixed old/new colors until another event updates the chrome.
  - confidence: 0.88
- **[P2] Rebase onto current TUI renderer imports:** `src/tui/theme/theme.ts:1-9`
  - body: The branch still imports `@mariozechner/pi-tui` and `cli-highlight` in the TUI theme path, while current main has moved this surface to `@earendil-works/pi-tui` and the renderer no longer uses direct `cli-highlight` here. Rebase before merging so the feature uses the current package surface and does not reintroduce obsolete dependencies.
  - confidence: 0.86
