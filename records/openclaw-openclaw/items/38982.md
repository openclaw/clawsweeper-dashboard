## Summary

Keep open: current main still lacks the runtime interactive theme picker, but this PR is not merge-ready because it conflicts with current theme detection, leaves cached TUI chrome stale, reintroduces stale dependency/baseline surface, lacks user docs, and overlaps https://github.com/openclaw/openclaw/pull/44382.

## What This Changes

Adds a TUI `/theme` command with a searchable live-preview picker, six runtime palettes, OSC terminal color helpers, syntax theme variants, and tests while also modifying `.secrets.baseline`.

## Best Possible Solution

Land one canonical TUI theming design that preserves current light/dark auto-detection, keeps mounted theme objects and cached chrome in sync, documents the user-facing command, and excludes unrelated secret baseline or dependency churn.

## Reproduction Assessment

Yes for the blocking patch issues. Source inspection shows the PR head seeds runtime theme state to `default` and only requests render on theme changes while current main stores header/footer/status as preformatted `Text` strings.

## Solution Assessment

No. The feature direction is reasonable, but the current patch is not the narrowest maintainable merge shape until it is rebased onto current theme detection, coordinated with https://github.com/openclaw/openclaw/pull/44382, documented, and stripped of unrelated baseline churn.

Full review comments:

- **[P2] Seed runtime theme from current detection:** `src/tui/theme/theme.ts:191-192`
  - body: The branch initializes `palette` and `currentThemeName` to `default`, so users with `OPENCLAW_THEME=light` or light `COLORFGBG` backgrounds would start in the default dark palette until they manually run `/theme`. Seed the runtime state from current main's detection result instead of hard-coding `default`.
  - confidence: 0.9
- **[P2] Refresh cached TUI chrome on theme changes:** `src/tui/tui-command-handlers.ts:253-254`
  - body: The picker preview/apply path mutates theme functions and requests a render, but the header, footer, and idle status are already formatted strings set by `updateHeader`, `updateFooter`, and `renderStatus`. Call those refresh hooks on preview/apply/cancel so live preview does not leave old ANSI colors in the TUI chrome.
  - confidence: 0.88
- **[P2] Remove the undeclared cli-highlight import:** `src/tui/theme/theme.ts:9`
  - body: Current main removed direct TUI `cli-highlight` usage and no longer declares it as a root dependency, but this file imports it directly. A clean rebased merge would rely on a transitive package or fail resolution; keep the lightweight current highlighter path or make a separately reviewed dependency change.
  - confidence: 0.84
- **[P3] Document the /theme command before shipping:** `src/tui/commands.ts:113-120`
  - body: This adds a user-facing `/theme` picker/list flow, but current public TUI docs only cover the existing `OPENCLAW_THEME` light/dark environment behavior. Add user docs for the new command and picker behavior before shipping the feature.
  - confidence: 0.82
