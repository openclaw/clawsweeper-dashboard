Keep open: current main still has the renderer/CSS mismatch that can let Control UI chat action buttons cover short assistant text, and an open linked fix PR is already the active implementation path.

## What This Changes

Control UI/WebChat should keep short assistant reply text visible when in-bubble action buttons appear.

## Best Possible Solution

Land or replace the linked fix PR with a Control UI renderer/CSS contract fix that keeps action buttons from covering text, with grouped-render and browser hover/touch coverage.

## Reproduction Assessment

## Solution Assessment
