## Reproduction Assessment

Yes at source level: current main slices `trimmed` with `.slice(0, 20)`, so 19 ASCII UTF-16 units plus an emoji can leave a lone high surrogate. I did not run tests because this review is read-only.

Yes: using the existing plugin SDK UTF-16-safe truncation helper in `generateJobName` is narrower and more maintainable than adding a QQBot-local truncator or changing cron naming semantics. The alternative of changing the budget or job-name format would be broader and unnecessary for this bug.
