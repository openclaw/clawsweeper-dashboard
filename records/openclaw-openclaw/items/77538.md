## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Bound auth payload bytes, not TypeBox graphemes:** `packages/gateway-protocol/src/schema/primitives.ts:32-43`
  - body: `Type.String({ maxLength })` in typebox@1.1.39 counts grapheme clusters, not UTF-8 bytes. A value such as `"a" + "\u0300".repeat(30000)` still satisfies `maxLength: 256` while producing a roughly 60KB auth string inside the pre-auth frame limit, so it can still reach `safeEqualSecret` and force the large buffer padding/scanning this PR is meant to prevent. Please add byte-length validation or restrict generated token fields to their ASCII token alphabet, and cover this with a regression case.
  - confidence: 0.93
- **[P1] Document or narrow the new connect-auth caps:** `packages/gateway-protocol/src/schema/frames.ts:74-78`
  - body: This changes connect auth from accepting any string within the pre-auth frame to rejecting bootstrap/device tokens over 256 characters and shared-secret/approval auth over 4096 characters. Current docs and the latest release do not establish those per-field limits, so existing long operator secrets or custom clients could fail schema validation before auth handling; please get gateway/security acceptance and document the contract, or narrow the cap to generated-token paths.
  - confidence: 0.86
