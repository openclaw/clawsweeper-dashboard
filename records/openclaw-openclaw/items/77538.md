## Summary

Keep open: current main still has the connect-frame auth schema gap, and this PR is a plausible targeted security hardening fix with sufficient after-fix terminal proof. The remaining work is maintainer/security approval for the public wire limits and a small protocol-docs update.

## What This Changes

The PR adds bounded connect-auth schema primitives, applies them to `auth.bootstrapToken`/`auth.deviceToken` and `auth.token`/`auth.password`, adds validator coverage, and records the change in the changelog.

## Best Possible Solution

Land the schema caps after Gateway/security owner approval and protocol docs are updated, letting https://github.com/openclaw/openclaw/issues/77977 close through the merge.

## Reproduction Assessment

Yes. Source inspection on current main shows sub-64 KiB `auth.*` strings pass `ConnectParamsSchema`, then can reach bootstrap/device/shared-secret auth paths before rejection.

## Solution Assessment

Mostly yes. Schema-level `maxLength` rejection is the narrow boundary before handshake auth work; the safer finish is to document the wire limits and get owner/security approval for the cap values.
