## Summary

Keep open: current main still has the connect-auth schema gap, and this PR is a plausible narrow fix with supplied real behavior proof and green CI, but the public protocol limits still need docs coverage and Gateway/security maintainer signoff before merge.

## What This Changes

The PR adds bounded Gateway connect-auth string primitives, applies them to `ConnectParamsSchema.auth`, adds regression tests, and records a changelog entry.

## Best Possible Solution

Land the schema-level caps after documenting the new auth field limits and getting Gateway/security owner approval for the chosen values.

## Reproduction Assessment

Yes, source-level: current main validates connect params before auth work while the four `auth.*` fields are unbounded, and oversized bootstrap/device tokens can reach the secret comparison paths. I did not live-drive a full WebSocket upgrade in this read-only review.

## Solution Assessment

Mostly yes: rejecting oversized values in the protocol schema is the narrow boundary fix before handshake secret comparison. The safer finish is to document the changed wire contract and get Gateway/security approval for the exact caps.
