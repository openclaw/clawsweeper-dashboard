## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Bound auth payload bytes, not graphemes:** `packages/gateway-protocol/src/schema/primitives.ts:32-43`
  - body: `Type.String({ maxLength })` in typebox@1.1.39 counts grapheme clusters, not UTF-8 bytes. A value like `a` plus many combining marks can stay within `maxLength: 256` while still fitting roughly 60KB in the pre-auth frame, so it can reach `safeEqualSecret` and force the byte padding/scanning this PR is meant to prevent. Please add byte-length validation or restrict generated token fields to their ASCII token alphabet, with a regression for that payload shape.
  - confidence: 0.93
- **[P1] Document or narrow the new connect-auth caps:** `packages/gateway-protocol/src/schema/frames.ts:74-78`
  - body: This changes connect auth from accepting any string inside the pre-auth frame to rejecting bootstrap/device tokens over 256 characters and shared-secret/approval auth over 4096 characters. Current docs and the latest release do not establish those per-field limits, so existing long operator secrets or custom clients could fail schema validation before auth handling; please get gateway/security acceptance and document the contract, or narrow the cap to generated-token paths.
  - confidence: 0.86
