## Summary

Keep open. Current main still does not implement the requested AI-guided in-app help flow, and the remaining work is a broad Control UI, Gateway, plugin API, and safety-design problem rather than conservative cleanup.

## What This Changes

The issue requests an in-app AI guide that reads the live UI, uses a stable helpable-element catalog, and executes validated typed UI actions to walk users through OpenClaw.

## Best Possible Solution

Keep this as an open product/design issue until maintainers define a narrow MVP for the helpable-element contract, guide action protocol, Control UI controller, Gateway session behavior, plugin contribution model, and safety limits.
