## Summary

Keep open. Current main does not implement the proposed AI-guided in-app help flow, and the request is broad enough to need maintainer product/API design rather than conservative cleanup. It should not be closed as ClawHub-only yet because the existing plugin UI descriptor surface is metadata-oriented and does not currently provide the DOM catalog, validated UI action protocol, or Control UI runtime needed for the requested behavior.

## What This Changes

Request an AI-guided Control UI help mode that reads live UI state, catalogs helpable elements, and drives validated highlight, click, reveal, and navigation actions through a dedicated guide agent.

## Best Possible Solution

Keep this as an open product/design issue until maintainers define a narrow MVP: the helpable-element contract, the Control UI controller, the guide session/action protocol, validation rules, safety limits, and whether plugins can contribute guide metadata through a new or expanded API.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is source/docs inspection, which shows the proposed guide agent, data-guide catalog, and typed UI action protocol are not present.

## Solution Assessment

Unclear. The issue’s architecture is coherent, but the best maintainable solution should be decided through product/API design because the current plugin descriptor surface is not enough for a DOM-driving assistant and a one-shot implementation would be too large.
