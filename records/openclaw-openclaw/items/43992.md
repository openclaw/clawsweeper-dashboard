## Summary

Keep open: this is a valid source-reproducible bug, but the proposed patch is incomplete against current main and lacks required real behavior proof from an actual after-fix subagent run.

## What This Changes

The branch adds gateway `agent.wait` result propagation through dedupe/responses, uses that result as subagent announce/registry fallback text, and adds gateway/subagent regression tests.

## Best Possible Solution

Port the fix onto current main so result payloads flow through dedupe, `agent.wait`, `waitForAgentRun`, and `createSubagentRunManager`, while decoding normal `{ payloads, meta }` results and keeping chat-history capture authoritative.

## Reproduction Assessment

Yes, at source level. Current main stores completed agent output in `payload.result`, then drops it from terminal snapshots, `agent.wait`, `waitForAgentRun`, and the active subagent run-manager completion path.

## Solution Assessment

No. The direction is useful, but this patch misses the current run-manager path, normal `{ payloads, meta }` decoding, and the lifecycle-before-dedupe race.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Parse agent.wait result payload envelopes:** `src/agents/subagent-announce.ts:345-347`
  - body: `extractAgentWaitResultText` passes `result` directly to the text extractor, but normal gateway agent runs return `{ payloads, meta }`. That leaves `payloads[*].text` invisible during the history catch-up window, so the fallback can still produce no completion reply.
  - confidence: 0.91
- **[P1] Port the fallback into the run manager:** `src/agents/subagent-registry.ts:1262-1264`
  - body: This writes fallback text into the older inline registry wait path, while current main completes registered subagent waits through `createSubagentRunManager` and `waitForAgentRun`. A current-main port would still complete and announce without the preserved wait result.
  - confidence: 0.9
- **[P2] Wait for the result-bearing dedupe snapshot:** `src/gateway/server-methods/agent.ts:772`
  - body: The live `agent.wait` path can return the lifecycle snapshot first and abort the dedupe waiter, but lifecycle snapshots do not carry `result`. Normal runs can emit lifecycle end before the gateway writes the result-bearing dedupe payload, so this can still return `undefined` in the target catch-up window.
  - confidence: 0.85
