## Summary

Keep open. Current main still drops the terminal `agent.wait` result before the active subagent completion path can use it, and this PR is directionally useful but not merge-ready against current main because it misses result-envelope parsing, the active run-manager path, and a lifecycle/dedupe race.

## What This Changes

The PR threads terminal `agent.wait` result payloads through gateway dedupe responses and tries to use that text as a subagent announce/registry fallback with regression tests.

## Best Possible Solution

Land a current-main port that carries result payloads through dedupe, `agent.wait`, and `waitForAgentRun`, parses `{ payloads, meta }` envelopes, freezes fallback text in the active run manager only when chat-history capture has not already won, and includes focused regression tests plus the required changelog entry.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main stores `payload.result`, then drops it from terminal snapshots, `agent.wait` responses, the shared run-wait helper, and the active subagent run-manager completion path.

## Solution Assessment

No. The PR is directionally useful, but it is not the best current fix because it targets an obsolete registry wait path, misses normal result-envelope parsing, and can still return a lifecycle snapshot before the result-bearing dedupe snapshot is available.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Parse result payload envelopes:** `src/agents/subagent-announce.ts:345-347`
  - body: `extractAgentWaitResultText` checks direct scalar fields and chat-style arrays, but normal gateway agent results are envelopes like `{ payloads: [{ text: "..." }], meta }`. In the history-not-caught-up case this still drops the completion text the fallback is meant to preserve.
  - confidence: 0.91
- **[P1] Port fallback into the run manager:** `src/agents/subagent-registry.ts:1259-1264`
  - body: This adds fallback freezing to the older inline registry wait path, while current main completes registered subagent waits through `createSubagentRunManager` and `waitForAgentRun`. After rebase, the active path still will not receive or freeze the fallback text.
  - confidence: 0.9
- **[P2] Wait for result-bearing snapshots:** `src/gateway/server-methods/agent.ts:772`
  - body: The live `agent.wait` path can return the first terminal lifecycle snapshot and abort the dedupe waiter. Because the result is written in the gateway dedupe payload, this can still return `undefined` for the final result during the exact catch-up window the PR is trying to fix.
  - confidence: 0.85
