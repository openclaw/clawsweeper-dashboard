## Summary

Keep open. Current main still drops terminal `agent.wait` result payloads before they can reach subagent completion delivery, and this PR is directionally useful but stale against the active run-manager path and still misses normal result-envelope handling.

## What This Changes

The PR threads `agent.wait` result payloads through gateway wait responses and reuses that text as a fallback for subagent completion announces when chat history has not caught up.

## Best Possible Solution

Land a current-main port that carries result payloads through dedupe, `agent.wait`, and `waitForAgentRun`, parses `{ payloads, meta }` envelopes, and freezes fallback text in the active run manager only when chat-history capture has not already won.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main writes `payload.result` but drops it from terminal snapshots, wait responses, the run-wait helper, and the active subagent completion path.

## Solution Assessment

No. The PR is directionally useful, but merging it unchanged is not the best fix because it targets an obsolete registry wait path, misses normal result-envelope parsing, and can still let lifecycle completion beat the result-bearing dedupe snapshot.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Parse result payload envelopes:** `src/agents/subagent-announce.ts:345-347`
  - body: `extractAgentWaitResultText` only checks direct scalar fields and chat-style arrays. Normal gateway agent results are envelopes like `{ payloads: [{ text: "..." }], meta }`, so the fallback still drops completion text when history has not caught up.
  - confidence: 0.91
- **[P1] Port fallback into the run manager:** `src/agents/subagent-registry.ts:1259-1264`
  - body: This adds the fallback to the older inline registry wait path, but current main completes subagent waits through `createSubagentRunManager` and `waitForAgentRun`. After rebase, the active completion path still will not receive or freeze fallback text.
  - confidence: 0.9
- **[P2] Wait for result-bearing snapshots:** `src/gateway/server-methods/agent.ts:772`
  - body: The live `agent.wait` path returns the first terminal snapshot and aborts the other waiter. Because lifecycle end can arrive before the gateway dedupe entry writes `payload.result`, this can still return `undefined` for the completion payload.
  - confidence: 0.85
