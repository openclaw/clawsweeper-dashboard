## Summary

Keep open. The PR is a plausible, test-backed fix for the custom/local provider private `baseUrl` problem, but it is still a draft that intentionally changes the SSRF trust boundary and plugin SDK policy surface, so it needs maintainer/security approval before merge.

## Best Possible Solution

Have a maintainer/security owner approve or adjust the exact-origin SSRF boundary and SDK exposure; if approved, land this PR with its focused regression coverage and track the remaining codex-provider or docs follow-ups separately.
