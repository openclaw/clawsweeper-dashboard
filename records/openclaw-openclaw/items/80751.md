## Summary

Keep open. The PR targets a real provider-baseUrl SSRF gap, but the current patch weakens an existing explicit private-network denial path and is still a draft security-boundary change that needs maintainer approval.

## What This Changes

The branch adds exact-origin `allowedOrigins` SSRF policy handling for custom/local model provider `baseUrl` requests, exports the helper through the SDK runtime, and adds focused regression tests.

## Best Possible Solution

Approve the SSRF trust boundary first, then land exact-origin provider trust only after preserving explicit-deny semantics with regression coverage and aligned docs.

## Reproduction Assessment

Yes. Source inspection shows current main does not grant non-loopback custom/local provider base URLs private-host trust without `request.allowPrivateNetwork`, and the PR finding is source-reproducible because the transport only receives a collapsed boolean.

## Solution Assessment

No, not yet. Exact-origin trust may be a reasonable direction, but this patch must preserve explicit `request.allowPrivateNetwork: false` and needs maintainer/security approval before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve explicit private-network denial:** `src/agents/provider-transport-fetch.ts:458-462`
