## Summary

Keep open. The PR targets a source-reproducible provider transport gap, but it is a draft SSRF trust-boundary change with missing live provider proof and a docs mismatch that should be resolved before merge.

## What This Changes

The PR adds exact-origin `allowedOrigins` SSRF policy handling for custom/local model provider `baseUrl` requests, threads it through fetch guard and SDK helpers, and adds regression tests.

## Best Possible Solution

Decide the SSRF trust boundary first; if exact-origin trust is approved, land it with docs/help updates, targeted tests, and redacted live proof against a local or self-hosted provider.

## Reproduction Assessment

Yes, by source inspection: on current main, a non-loopback private custom/local provider base URL without `request.allowPrivateNetwork` reaches the fetch guard with fake-IP host policy only, and private literal or resolved addresses remain blocked. I did not run a live LM Studio, vLLM, or Ollama server in this read-only review.

## Solution Assessment

Unclear. Exact-origin trust is narrower than host-only trust and the tests cover important redirect/proxy cases, but the boundary, SDK exposure, docs, and live proof still need maintainer review before this is the best mergeable solution.
