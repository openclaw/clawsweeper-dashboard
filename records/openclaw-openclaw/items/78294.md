## Summary

Keep open: this is a broad interoperability standards proposal, not conservative cleanup. Current main already has AgentSkills-compatible skills, ClawHub/plugin distribution, bundle imports, and ACP/MCP interop surfaces, while ASFS plus signed cross-agent handoff still needs maintainer product and security scoping.

## What This Changes

The issue proposes adopting an ASFS portable skill format and a signed Handoff Protocol for cross-agent interoperability.

## Best Possible Solution

Treat this as a maintainer design discussion: prototype skill conversion through ClawHub, bundles, or a plugin where possible, then propose narrowly scoped SDK/protocol changes only for a threat-modeled gap.

## Reproduction Assessment

Not applicable: this is a standards and feature proposal, not a report of broken current-main behavior. Source inspection only confirms that ASFS/IACP is not implemented today.

## Solution Assessment

Unclear: adopting the full ASFS and signed handoff proposal in core is not yet shown to be the narrowest maintainable solution. A ClawHub/bundle/plugin converter plus a separate threat-modeled handoff design would be safer first.
