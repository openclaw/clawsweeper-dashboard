## Summary

Keep open: this is a broad interoperability standards proposal, not conservative cleanup. Current main has AgentSkills-compatible skills, ClawHub/plugin skill distribution, bundle imports, ACP/MCP interop, and related A2A handoff discussions, but ASFS plus a signed cross-framework handoff protocol would need maintainer product and security design before any core change.

## What This Changes

The issue proposes adopting ASFS YAML/Markdown skill interchange plus a signed Ed25519 handoff protocol for OpenClaw and other agent frameworks.

## Best Possible Solution

Treat this as a maintainer design discussion: prototype ASFS conversion through ClawHub/bundle/plugin tooling where possible, and add core SDK or protocol seams only for a concrete, threat-modeled gap.

## Reproduction Assessment

Not applicable: this is a feature and standards proposal, not a report of broken current-main behavior. Source inspection confirms ASFS/IACP is not implemented today, while adjacent skill and handoff surfaces already exist.

## Solution Assessment

Unclear: adopting the full external standard and signed handoff protocol in core is not yet shown to be the narrowest maintainable solution. A ClawHub/bundle converter or focused plugin-SDK gap proposal is safer unless maintainers choose core support.
