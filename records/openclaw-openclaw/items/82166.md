## Summary

Keep this PR open: the bug is real and the fix direction is narrow, but the latest head still relies on WHATWG URL reserialization to preserve explicit default ports, which drops `:80` and `:443` and contradicts the new test contract.

## What This Changes

The PR changes browser CDP URL/profile resolution so explicit URL ports win, portless URLs preserve configured `cdpPort`, and config tests cover port precedence cases.

## Best Possible Solution

Keep the proposed precedence rule, but build port-preserving normalized URLs without relying on WHATWG URL serialization for explicit default ports, with focused regression coverage for the failing default-port cases.

Yes. Current main is source-reproducible through `resolveProfile` plus `parseBrowserHttpUrl`: a portless HTTP profile URL resolves to port 80 and overwrites the configured `cdpPort`.

No, not yet. The intended precedence change is the right narrow fix, but the current PR head needs a parser repair because URL reserialization still drops explicit default ports.
