## Summary

Keep open: the reported bug path is valid and this PR handles the host-only URL case, but the current head still treats explicit default URL ports as omitted because it checks WHATWG `URL.port` after normalization.

## What This Changes

The PR changes browser profile CDP URL resolution so a host-only `cdpUrl` preserves an explicit `cdpPort` and adds a focused regression test.

## Best Possible Solution

Adjust `resolveProfile` to distinguish omitted ports from explicit default ports before WHATWG URL normalization, add `:80` and `:443` regression coverage, and keep the host-only `cdpPort` preservation behavior.

## Reproduction Assessment

Yes. Source inspection on current main follows `cdpPort: 18800` plus `cdpUrl: "http://127.0.0.1"` through `resolveProfile` to parser default port 80, and the linked issue supplies matching Chrome stderr.

## Solution Assessment

No. The fix belongs in `resolveProfile`, but using `parsed.parsed.port` as the explicit-port detector misses explicit default ports because WHATWG URL normalization clears `:80` and `:443`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
