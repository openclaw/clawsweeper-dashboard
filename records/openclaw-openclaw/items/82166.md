## Summary

Keep open: the PR addresses the reported browser profile precedence bug, but the current patch misidentifies explicitly written default ports as omitted because it relies on WHATWG `URL.port`. That leaves a concrete correctness blocker before merge.

## What This Changes

The PR changes browser profile resolution so a portless `cdpUrl` preserves an explicit `cdpPort` and adds a focused config regression test.

## Best Possible Solution

Keep the narrow precedence fix, but record whether the raw `cdpUrl` syntactically included a port before URL normalization and cover omitted-port plus explicit default-port cases in tests.

## Reproduction Assessment

Yes from source inspection. Current main routes `cdpPort: 18800` plus `cdpUrl: "http://127.0.0.1"` through `resolveProfile` to parser default port 80, and the linked report shows the matching Chrome bind failure.

## Solution Assessment

No, not as-is. The intended fix direction is right, but `parsed.parsed.port` loses explicit default ports, so the implementation needs raw explicit-port detection before it can safely preserve URL-port precedence.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Detect explicit default ports before checking URL.port:** `extensions/browser/src/browser/config.ts:509`

That contradicts the PR's stated rule that explicit URL ports win; track explicit port syntax before normalization or return that metadata from the parser.

Repair openclaw/openclaw pull request 82166. The intended behavior is: an explicit `browser.profiles.<name>.cdpPort` should be preserved only when `cdpUrl` truly omits a port; any syntactically explicit URL port must keep URL precedence, including default ports such as `http://host:80`, `https://host:443`, `ws://host:80`, and `wss://host:443`. Add a small helper or parser metadata that detects explicit port syntax before normalization, update `resolveProfile`, and add regression tests in `extensions/browser/src/browser/config.test.ts` for the reported omitted-port case and at least one explicit default-port case with a conflicting `cdpPort`. Keep the change scoped to browser config resolution and tests; do not change docs, unrelated browser behavior, dependencies, or config schema. Changelog is maintainer-owned for landing.
