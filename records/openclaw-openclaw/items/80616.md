## Summary

Keep open: this protected maintainer-labeled PR is a plausible narrow bug fix, but the current diff should not merge until abort-marker detection is stricter because ordinary user text containing `<turn_aborted>` could prematurely release the turn.

## What This Changes

The PR makes Codex app-server treat current-turn raw `<turn_aborted>` response-item markers as terminal, marks the projector aborted, adds a regression test, and updates the changelog.

## Best Possible Solution

Keep the fix direction, but require an exact synthetic abort-marker shape and a negative regression test for ordinary user content mentioning the tag before merge.

## Reproduction Assessment

Yes. Source inspection shows current main resolves the attempt only on current-turn `turn/completed`; a current-turn raw abort marker without that notification would wait for the terminal watchdog, and the PR adds a focused harness scenario for that path.

## Solution Assessment

No. The direction is right, but substring matching is not the best fix; the maintainable path is a strict parser for the Codex synthetic abort envelope while preserving current thread/turn checks.
