## Summary

Keep open: current main still misses explicit cleanup of stale `.deleted.*` and `.reset.*` transcript archives, but this branch should not merge until archive-retention ownership is unified and full current-head CLI proof is supplied.

## What This Changes

The PR makes `openclaw sessions cleanup` prune stale `.deleted.*` archives by `pruneAfter` and `.reset.*` archives by `session.maintenance.resetArchiveRetention`, with regression tests for preview/apply behavior.

## Best Possible Solution

Share one archive cleanup eligibility/removal contract between normal store maintenance and explicit `openclaw sessions cleanup`, then land with full current-head CLI dry-run/apply proof for default, longer, shorter, and disabled reset retention.

## Reproduction Assessment

Yes. Current-main source shows explicit cleanup calls `pruneUnreferencedSessionArtifacts()` without archive-specific retention, while that sweep excludes archive names, so stale `.deleted.*` and `.reset.*` files are missed by this path.

## Solution Assessment

No, not in the current shape. Threading reset retention fixes the immediate behavior, but duplicating archive-expiration policy is weaker than reusing or extracting the existing archive cleanup contract.
