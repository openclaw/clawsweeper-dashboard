## Summary

This is still a feature/API policy decision, with partial overlap but not full duplication of the broader watchdog issue.

## What This Changes

Request to add a default-off `sessions_spawn` liveness-notification policy that periodically tells the parent/requester session a native subagent is still running.

## Best Possible Solution

Have maintainers choose a lean default-off liveness-notice contract, or fold this narrow request into the broader watchdog design; if accepted, reuse existing subagent run/task state and requester delivery with throttling and terminal cleanup.

## Reproduction Assessment

## Solution Assessment
