## Summary

Keep open: current main still has completion events, `sessions_yield`, active-subagent prompt context, task/status surfaces, and stale-run liveness cleanup, but it does not implement the requested opt-in periodic requester notice while a subagent is still running.

## What This Changes

The issue requests a default-off policy for long-running subagent runs to periodically notify the requester that work is still running.

## Best Possible Solution

Keep this open for a default-off deterministic subagent liveness notifier that reuses existing run/task state and requester delivery without invoking an LLM turn.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection confirms current main has status/completion surfaces but no periodic requester notice path.

## Solution Assessment

Unclear until maintainers choose the API shape. The narrow maintainable direction is a default-off deterministic notifier derived from `SubagentRunRecord` or task-ledger state, with per-run throttling and terminal cleanup tests.
