## Summary

Keep open: current main and the latest release still expose pre-prompt budget and compaction facts, but not a post-recovery retained-context ratio, degraded-context flag, or user-visible warning for successful low-retention `compact_only` recovery.

## What This Changes

Request a user-visible signal when `compact_only` overflow recovery sends a much smaller assembled context than the pre-recovery context.

## Best Possible Solution

Add a generic runner/session degradation signal for post-recovery retained context and render it in operator-visible status/session surfaces before considering agent-facing prompt text.

## Solution Assessment

Unclear until maintainers choose the product surface. A generic runner/session metadata signal with `/status` and dashboard rendering looks safer than a context-engine-specific workaround or a first-pass agent prompt change.
