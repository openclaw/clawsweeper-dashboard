## Summary

Keep open: current main still has the compact-only overflow recovery path and generic compaction/status indicators, but I did not find a retained-context ratio, degraded-context event, assistant-message flag, dashboard warning, or /status warning for the low-retention successful recovery described here.

## What This Changes

The issue asks OpenClaw to surface a user-visible or operator-visible warning when compact-only overflow recovery leaves the model with only a tiny recovered context.

## Best Possible Solution

Define a generic degraded-context recovery signal at the overflow recovery boundary, including a maintainer-approved retention metric, then render it in the selected operator/user surfaces without changing normal compaction semantics.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection does show the compact_only precheck and successful compaction retry path, and the issue provides detailed logs, but I did not replay the lossless-claw scenario on current main.

## Solution Assessment

Unclear until maintainers choose the product surface. The narrowest maintainable direction appears to be a generic degraded-context metadata/event signal from the runner, with dashboard and /status rendering layered on top rather than a context-engine-specific workaround.
