## Summary

Keep open: current main still has the compact-only overflow recovery path, but I did not find retained-context ratio metadata, a degraded-context flag, a dashboard/session warning, or an agent-visible signal when successful recovery leaves only a tiny context.

## What This Changes

The issue asks OpenClaw to surface a user-visible signal when compact-only context-overflow recovery trims most context before a model turn.

## Best Possible Solution

Define a generic degraded-context recovery record when compact-only post-recovery context falls below a chosen retention threshold, persist it with session/run metadata, and render it in /status plus session/dashboard surfaces with focused regression coverage.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. Source inspection plus the issue logs show the compact_only precheck and successful compaction-retry path, but I did not replay the external context-engine low-retention scenario on current main.

## Solution Assessment

Unclear until maintainers choose the product surface. A generic runner/session metadata signal with status and dashboard rendering looks safer than a context-engine-specific workaround or a first-pass agent prompt change.
