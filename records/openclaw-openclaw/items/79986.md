## Summary

Keep open: the raw subagent-output leak remains source-reproducible on current main, and this report is useful fresh Telegram evidence even though related root work is already tracked.

## What This Changes

The issue reports duplicate Telegram progress delivery and raw subagent tool-output leakage from `sessions_spawn(mode="run")` completion handling.

## Best Possible Solution

Subagent completion handoff should wait for a post-compaction assistant result or synthesize a bounded failure summary, and Telegram should receive at most one intentional user-facing completion/progress message.

## Reproduction Assessment

Yes for the raw-output half from source: current main still selects latest tool/toolResult text when assistant output is empty, and tests assert that fallback. The exact duplicate Telegram progress bubble was not live-reproduced in this read-only review.

## Solution Assessment

No; the current requester-agent handoff is safer than direct raw-send, but it can still feed raw child tool output into the parent completion event. The narrower fix is to delay/poll for assistant text after compaction or emit a controlled summary instead of embedding raw tool dumps.
