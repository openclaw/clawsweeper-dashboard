## Summary

Keep open: current main still has a source-reproducible path where raw child tool/toolResult text can become the subagent completion result handed to the requester, and the fresh Telegram duplicate-progress evidence is not covered by a merged fix.

## What This Changes

The issue reports duplicate Telegram progress delivery and raw subagent tool output leaking through subagent completion announce handling.

## Best Possible Solution

Land one consolidated agents fix that prevents raw tool/toolResult text from becoming a successful subagent completion result across selection, capture, frozen-result refresh, and requester delivery, with focused regression coverage and docs/changelog updates.

## Reproduction Assessment

Yes for the raw-output path: current main can select latestRawText from a child tool/toolResult and feed it into the completion event when assistant output is empty or delayed. The exact duplicate Telegram progress message still needs live reproduction or a narrower source trace.

## Solution Assessment

No; the current mediated requester handoff is safer than direct raw-send fallback, but it can still carry raw child output as the completion result. The narrower maintainable fix is to suppress or defer raw successful completions and use assistant-authored text or bounded failure/timeout summaries.
