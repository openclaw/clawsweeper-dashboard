## Summary

Keep open: the raw subagent-output leak remains source-reproducible on current main, and this issue adds useful 2026.5.7 Telegram DM evidence plus a still-unisolated duplicate-progress symptom.

## What This Changes

The issue reports that Telegram received duplicate parent progress and raw child tool output from `sessions_spawn(mode="run")` subagent completion handling.

## Best Possible Solution

Consolidate the subagent announce fix so successful or continuation-window completions wait for assistant-authored output or emit a bounded failure/partial summary, while keeping internal completion envelopes hidden from external chats.

## Reproduction Assessment

Yes for the raw-output path from source: current main can select latest `toolResult` text as subagent completion output, and current tests assert that fallback. The exact duplicate Telegram progress bubble still needs a live or transcript-level isolation pass.

## Solution Assessment

No; the current requester-agent handoff is the right delivery boundary, but it is not sufficient while the handoff payload can be populated from raw child tool output. The narrower fix is in output selection/capture and completion handoff, not a broad Telegram-specific workaround.
