## Summary

Keep open. Current main already projects Codex commentary as preamble/item progress and Discord renders those item events through the progress-draft tool path, but the requested independent `progress.commentary` contract and commentary-with-`toolProgress:false` behavior are not implemented.

## What This Changes

Request an opt-in Discord progress setting that renders Codex assistant commentary in temporary progress drafts independently from tool-progress rows.

## Best Possible Solution

Define and document a maintainer-approved commentary progress contract, then wire it through Codex projection, dispatch, Discord draft rendering, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new feature/config request. Source inspection does show the requested config split is absent and `toolProgress: false` currently suppresses the progress lines that carry commentary.

## Solution Assessment

Unclear until maintainers decide the public contract. The proposed direction is plausible, but adding a new config/API surface for commentary is product-facing work rather than a strict repair to an existing documented behavior.
