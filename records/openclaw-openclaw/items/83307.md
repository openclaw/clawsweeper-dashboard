## Summary

Keep open. Current main still exposes Codex commentary through the existing item/tool-progress draft path, with no `progress.commentary` or dedicated commentary callback, and `toolProgress: false` still suppresses the shared progress-line path, so the requested opt-in config split remains a product/API decision rather than implemented behavior.

## What This Changes

Add an opt-in Discord progress-draft setting that renders Codex assistant commentary temporarily and independently from tool-progress rows.

## Best Possible Solution

Add a documented, maintainer-approved `streaming.progress.commentary` contract or equivalent that separates Codex commentary from tool progress across config types, Codex projection/reply plumbing, Discord rendering, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection does show the requested config split is absent and `toolProgress: false` currently suppresses the shared progress line path.

## Solution Assessment

Unclear until maintainers decide the public contract. The proposed direction is plausible, but current main documents commentary as using the tool-progress path, so the maintainable solution needs an explicit config/API decision before implementation.
