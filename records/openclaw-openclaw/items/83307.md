## Summary

Keep open. Current main already routes Codex commentary-phase messages into the existing item/tool-progress preview path, but the requested opt-in `progress.commentary` contract and the ability to show commentary while `toolProgress` is disabled are not implemented.

## What This Changes

Request an opt-in Discord progress setting that renders Codex assistant commentary in temporary progress drafts while keeping final replies durable and allowing tool rows to be hidden.

## Best Possible Solution

Define and document a maintainer-approved `streaming.progress.commentary` contract, or an equivalent named status lane, then wire it through Codex projection, reply callbacks, Discord draft rendering, config schema, docs, and focused tests without changing final-answer delivery.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection does confirm the requested split is absent and that `toolProgress: false` currently leaves Discord progress drafts label-only.

## Solution Assessment

Unclear until maintainers approve the public contract. The proposed direction is plausible, but the safest implementation should separate assistant-authored status/commentary from generic tool progress while preserving the final-answer-only durable reply boundary.
