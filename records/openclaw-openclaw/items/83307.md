## Summary

Keep open. Current main already streams Codex commentary through the existing item/tool-progress draft path, but the requested opt-in `progress.commentary` control and the ability to show commentary while `toolProgress` is disabled are not implemented.

## What This Changes

Request to add opt-in Discord progress draft rendering for Codex assistant commentary events while keeping final answers unchanged and allowing tool progress to be hidden independently.

## Best Possible Solution

Add a documented, maintainer-approved `streaming.progress.commentary` contract or equivalent that separates Codex preamble/commentary lines from tool progress in config types, Discord rendering, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection does show the requested config split is absent and `toolProgress: false` currently makes Discord progress drafts label-only.

## Solution Assessment

Unclear until maintainers decide the public contract. The proposed direction is plausible, but current main treats commentary as item progress, so the safest implementation needs an explicit config/API decision before code changes.
