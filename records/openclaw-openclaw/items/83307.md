## Summary

Keep open. Current main already sends Codex commentary as keyed preamble/item progress into Discord's existing progress-draft path, but the requested opt-in `progress.commentary` contract and the ability to show commentary while `toolProgress` is disabled are still missing and need maintainer product/API approval.

## What This Changes

Add an opt-in Discord progress feature that renders assistant commentary in the temporary progress draft independently of tool-progress rows.

## Best Possible Solution

Define and document a public progress-commentary contract that decouples commentary/preamble display from tool rows, then update Codex projection, Discord draft rendering, and focused tests around that contract.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection shows current main lacks `progress.commentary`, and commentary/preamble item lines are still tied to the existing tool-progress gate.

## Solution Assessment

Unclear until maintainers decide the public contract. The proposed direction is plausible, but the current implementation deliberately treats commentary as item progress, so the maintainable fix needs an explicit config/API decision first.
