## Summary

Keep open. Current main already carries Codex commentary as keyed preamble/item progress, but Discord still routes it through the same tool-progress draft gate and the public config/schema has no separate opt-in commentary switch, so the requested behavior remains a product/config decision.

## What This Changes

Request an opt-in Discord progress-draft setting that displays Codex assistant commentary temporarily while allowing tool-progress rows to be hidden and keeping final replies unchanged.

## Best Possible Solution

Add a documented, maintainer-approved `streaming.progress.commentary` contract or equivalent that separates Codex preamble/commentary lines from tool progress while preserving final-only durable reply delivery.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection shows the requested config split is absent and current main couples commentary/preamble progress to the existing tool-progress gate.

## Solution Assessment

Unclear until maintainers choose the public contract. The proposed direction is plausible, but the safest implementation needs a narrow commentary-progress option with tests that preserve the final-answer leak protections.
