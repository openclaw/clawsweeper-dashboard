## Label Justifications

- `P2`: The report describes a bounded core agent delivery bug that can silently suppress a response in hook-enabled group-chat retry flows.
- `impact:message-loss`: The observable failure mode is a real assistant answer not being delivered to the channel.
- `impact:session-state`: The failure depends on transcript/session state being visible to the retry pass before the finalize-hook revision is complete.

## Summary

Keep open: current main still has the finalize-hook retry path where a visible first pass can be suppressed, retried from current transcript state, and then resolved as terminal silence when the retry pass collapses to `NO_REPLY`. The only direct fix candidate was closed unmerged after maintainer feedback rejected the restore-after-silent approach.

## What This Changes

Fix `before_agent_finalize` retry handling so a real first answer is not silently dropped when the revision pass returns `NO_REPLY` in group-chat silent-reply mode.

## Best Possible Solution

Repair retry/transcript ordering so the retry pass does not see the rejected first answer as finalized history, while preserving the revision pass as authoritative and adding silent group-chat regression coverage.

## Reproduction Assessment

Yes, source-reproducible: the current run loop can suppress a visible first pass, retry from current transcript state, and then return terminal silence when the retry pass collapses to `NO_REPLY`. No live current-main group-channel run was executed in this read-only review.

## Solution Assessment

Yes, keeping this open is the right path; the closed restore-based PR is not the best fix because it violates the finalization contract. The maintainable fix should prevent the retry context from treating the rejected first answer as finalized history.
