## Summary

Keep open: current main still has the source path where an honored `before_agent_finalize` revision suppresses first delivery, retries, and then treats a silent retry pass as the terminal payload. The linked candidate PR was closed unmerged because restoring the pre-revision answer violates the finalization contract, so the remaining fix should target retry/transcript ordering instead.

## What This Changes

The issue reports that `before_agent_finalize` revision retries can persist the first assistant answer before the retry, causing group-chat silent-token logic to drop the deliverable response.

## Best Possible Solution

Repair the core retry/transcript ordering so an honored finalize-hook revision does not make the retry pass see the rejected first answer as finalized history, while keeping the revision pass authoritative and adding a silent group-chat regression test.

## Reproduction Assessment

No, no executed high-confidence reproduction was run in this read-only review. The source path is clear enough for a focused harness: first attempt returns visible text and a finalize revision, the retry pass returns `NO_REPLY`, payloads strip to empty, and terminal delivery resolves to silence.

## Solution Assessment

No, the closed restore-after-silent PR is not the best fix because maintainer feedback identified finalization-contract and transcript-duplication problems. The safer direction is retry/transcript ordering that prevents the retry pass from seeing the rejected first answer as finalized context.

## Repair Work Prompt

Fix the core embedded-runner bug reported in https://github.com/openclaw/openclaw/issues/93166. When `before_agent_finalize` returns `revise`, the retry pass must not see the rejected first assistant answer as finalized session history in a way that makes group-chat silent-token logic return `NO_REPLY` and drop delivery. Keep the revision pass authoritative; do not restore the first answer after a silent retry, because https://github.com/openclaw/openclaw/pull/93480 was closed for violating the finalization contract and duplicating transcript content. Prefer a retry/transcript-ordering or retry-context fix in `src/agents/embedded-agent-runner/run.ts` and/or `src/agents/embedded-agent-runner/run/attempt.ts`, add a focused regression in `src/agents/embedded-agent-runner/run.before-agent-finalize.test.ts` for first answer -> finalize revision -> `NO_REPLY` retry under `allowEmptyAssistantReplyAsSilent`, and keep plugin API, config, docs, and changelog unchanged unless the implementation truly changes those contracts.
