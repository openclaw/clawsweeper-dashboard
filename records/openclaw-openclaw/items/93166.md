## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/93166 in one focused PR. The observable bug is that a `before_agent_finalize` revise request suppresses first terminal delivery, then the retry can see the first assistant answer in transcript state and resolve to `NO_REPLY`, dropping visible delivery. Do not restore or deliver the rejected pre-revision answer after a silent revision; maintainers rejected that approach in https://github.com/openclaw/openclaw/pull/93480 because the revision pass must remain authoritative and transcript duplication must be avoided. Repair the core retry/transcript ordering so the revision pass is replay-safe, likely by keeping the pre-revision assistant answer out of retry-visible history or rolling back/forking it before retry, while preserving side-effect guards, max revision limits, and silent-reply semantics. Add focused regression coverage for visible first answer -> revise -> second pass exact `NO_REPLY` or empty silent output, plus a real revised answer replacing the original. Treat the hot-reload hook lifecycle note as out of scope unless the code path naturally requires it. Do not add config, plugin API, docs changelog, or channel-specific fallback behavior.
