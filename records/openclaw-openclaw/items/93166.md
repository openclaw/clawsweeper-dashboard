## Summary

Keep open: current main and v2026.6.9 still contain the source path where an honored `before_agent_finalize` revision retries after suppressing terminal delivery, and a silent retry pass resolves to the silent token. The only linked fix PR was closed unmerged after maintainer review rejected restoring the pre-revision answer as a contract violation, so the remaining fix needs different retry/transcript ordering.

## What This Changes

The issue reports that `before_agent_finalize` revision retries can make the second pass return `NO_REPLY`, causing the real first response to remain in transcript state but not be delivered.

## Best Possible Solution

Repair the core retry/transcript ordering so a `before_agent_finalize` revision pass does not see a finalized copy of the rejected first answer, while preserving the revision pass as authoritative and adding silent group-chat regression coverage.

## Reproduction Assessment

No executed high-confidence reproduction was run in this read-only review. The source path is clear enough for a focused harness: first pass returns visible text and a finalize revision, the retry pass returns `NO_REPLY`, payload building strips it, and terminal delivery resolves to silence.

## Solution Assessment

Unclear for the exact implementation, but the report points at the right core boundary. The closed restore-after-silent PR is not the best fix; retry/transcript ordering or retry context should be repaired so the revision pass remains authoritative without seeing the rejected first answer as finalized history.
