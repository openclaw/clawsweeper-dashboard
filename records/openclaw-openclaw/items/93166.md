## Summary

Keep open: current main and the latest release still contain the source path where a `before_agent_finalize` revision suppresses first delivery, continues to a second attempt, and treats an empty/silent second pass as the terminal result. The only linked fix PR was closed unmerged after maintainer review rejected its restore-the-first-answer approach as violating the finalization contract.

## What This Changes

The issue asks OpenClaw to prevent `before_agent_finalize` revise/retry attempts from committing or reusing the first assistant answer in a way that can make a later silent revision drop delivery.

## Best Possible Solution

Repair retry/transcript ordering in the core embedded runner so an answer rejected for revision is not visible to the revision pass, while preserving `NO_REPLY` as the authoritative final outcome and tracking hot-reload hook lifecycle separately if confirmed.

## Reproduction Assessment

Yes at source level, but not as an executed live repro in this read-only pass. A focused runner test can model attempt one returning `beforeAgentFinalizeRevisionReason` and attempt two stripping `NO_REPLY` to an empty payload under `allowEmptyAssistantReplyAsSilent`.

## Solution Assessment

No current fix is present on main. The rejected PR shows that restoring the first answer after a silent revision is not the best fix; the safer direction is staging or rolling back the pre-revision assistant turn so the revision pass does not see rejected content.
