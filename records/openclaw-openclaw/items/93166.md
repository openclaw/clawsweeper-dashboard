## Summary

Keep open: current main and the latest release still have the source-reproducible silent-revision drop path, and the exact linked fix PR was closed unmerged.

## What This Changes

The issue reports that a `before_agent_finalize` retry can suppress the first deliverable response, retry into a `NO_REPLY` silent turn, and leave the user with no delivered reply.

## Best Possible Solution

Land a narrow core runner fix that preserves the suppressed deliverable reply only for a honored `before_agent_finalize` revision whose follow-up pass collapses to silent or empty output, with regression coverage for both silent and real revised replies.

## Reproduction Assessment

Yes, source-reproducible: a first attempt with `beforeAgentFinalizeRevisionReason` is suppressed, a second `NO_REPLY` attempt is stripped to no payload, and current main returns the silent token instead of the first payload. I did not run tests because this review is read-only.

## Solution Assessment

Yes: the narrowest maintainable fix is to retain the suppressed first payload at the revision honor point and restore it only when the revision pass is silent or empty. The closed unmerged PR demonstrates the right shape, but it needs to be reapplied and reviewed against current main.
