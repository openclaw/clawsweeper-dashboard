- `P2`: The issue describes a bounded but real core agent delivery bug that can silently suppress a response in finalize-hook retry flows.
- `impact:message-loss`: The reported failure mode drops the assistant response that should have been delivered.
- `impact:session-state`: The bug depends on transcript/session state being visible to a retry pass before finalize-hook acceptance is complete.

## Summary

Keep open: current main still honors a `before_agent_finalize` retry, continues into another model pass, and treats a silent retry pass as terminal silence; the two related fix attempts were closed unmerged after maintainer review rejected their safety-contract tradeoffs.

## What This Changes

The issue reports silent response loss when a `before_agent_finalize` revision retry leaves the first answer in transcript and the retry pass returns the silent token.

## Best Possible Solution

Repair the core retry/transcript ordering so a finalize-hook retry does not treat the rejected first answer as finalized context, while preserving the revision pass as the authoritative finalization decision.

## Reproduction Assessment

No, no live end-to-end reproduction was run. Source inspection shows the current path can honor a finalize-hook retry, continue, then treat a silent retry attempt as terminal silence; a focused harness should still be added.

## Solution Assessment

Yes for a retry/transcript-ordering repair; no for the closed restore-after-silent workaround. The safer solution is to prevent the retry pass from seeing the rejected first answer as finalized history while keeping the revision pass authoritative.
