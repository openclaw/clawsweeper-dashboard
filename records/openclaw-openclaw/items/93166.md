## Label Justifications

- `P2`: The report describes a bounded core agent delivery bug that can silently suppress a response in finalize-hook retry flows.
- `impact:message-loss`: The failure mode drops the assistant response that should have been delivered to the channel.
- `impact:session-state`: The bug depends on transcript/session state being updated before the finalize-hook retry has completed.

## Summary

Keep open: current main still honors a `before_agent_finalize` revision by continuing to another attempt and still maps an allowed silent retry pass to `NO_REPLY`; the only linked fix PR was closed unmerged because its restore-based approach violated the finalization contract.

## What This Changes

The issue asks OpenClaw core to prevent a suppressed first answer from being committed into the retry context where a later silent revision pass can drop the user-visible response.

## Best Possible Solution

Repair the core retry/transcript ordering so a finalize-hook revision pass does not see the rejected first answer as finalized conversation history, while keeping the revision pass authoritative and covering the silent group-chat case with a focused regression test.

## Reproduction Assessment

No. I did not run a live or deterministic repro in this read-only review, but the current source path is clear: a revise decision suppresses first delivery, retries, and an allowed silent retry pass returns terminal `NO_REPLY`.

## Solution Assessment

No. The closed restore-after-silent PR is not the best fix because maintainer feedback identified finalization-contract and transcript-duplication problems; the safer direction is retry/transcript ordering that prevents the rejected first answer from becoming retry-visible history.
