## Summary

Keep open: current main and the latest release still show the retry path where an honored `before_agent_finalize` revision continues with only user-message persistence suppressed, and a silent retry pass resolves to the silent token for delivery. The linked candidate fix at https://github.com/openclaw/openclaw/pull/93480 was closed unmerged after maintainer review rejected restoring the suppressed first answer as a contract violation.

## What This Changes

Fix a core agent-runner bug where `before_agent_finalize` revision retries can make a group-chat response disappear when the retry pass returns the silent token.

## Best Possible Solution

Repair the core retry/transcript ordering so an honored `before_agent_finalize` revision retries without showing the model a finalized copy of the rejected first answer, while keeping the revision pass authoritative.

## Reproduction Assessment

No executed high-confidence reproduction was run in this read-only review. The source path is clear enough for a focused harness: first attempt returns visible text and a finalize revision, the retry pass returns `NO_REPLY`, the payload builder strips it, and terminal delivery resolves to silence.

## Solution Assessment

No current fix is present, and the closed restore-after-silent candidate is not the best repair because maintainer review identified finalization-contract and transcript-duplication risk. The safer path is to fix retry/transcript ordering or retry context before the second model pass.

## Repair Work Prompt

Fix the core embedded-agent `before_agent_finalize` revision retry bug where a first visible answer can be present in the retry transcript/context, the retry pass returns `NO_REPLY` in automatic group-chat mode, and delivery resolves to silence. Do not restore the suppressed first answer after a silent retry and do not expose hook-rejected drafts; https://github.com/openclaw/openclaw/pull/93480 was closed because that violates the finalization contract and can duplicate transcript content. Instead, repair retry/transcript ordering or retry prompt/session context so the revision pass remains authoritative but does not see the rejected first answer as a finalized assistant turn. Add focused regression coverage for a visible first answer plus finalize revision plus silent retry under `allowEmptyAssistantReplyAsSilent`, and include assertions that the transcript/order does not become `First answer -> NO_REPLY -> First answer`. Keep scope to agents/runtime hook handling and tests; do not add config, provider routing, plugin API surface, or changelog changes.
