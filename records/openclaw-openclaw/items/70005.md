## Summary

Keep open. Current main partially denoises Dreaming, but REM still builds live themes from concept-tag recurrence and Dream Diary still lacks a semantic-importance material selector, so the central material-selection complaint remains unresolved.

## What This Changes

Request semantic-importance material selection for memory-core Dreaming so summaries prioritize meaningful decisions, preferences, and conclusions over frequent transcript tokens.

## Best Possible Solution

Add an internal material-importance scorer before REM reflections and Dream Diary input, coordinate it with the adjacent Dreaming hygiene/scoring issues, and avoid adding a public promptStyle knob before the ranking problem is fixed.

## Reproduction Assessment

Yes, at source level: current main still ranks REM themes by repeated concept-tag counts and feeds Dream Diary without a semantic-importance selector. A full live reproduction of the reporter's private corpus is not available in this read-only review.

## Solution Assessment

Partly: a semantic-density selector before REM and Dream Diary is the maintainable direction, but exposing a new promptStyle/config knob first would add product surface before fixing the internal ranking problem.
