## Summary

Keep open. Current main mitigates some transcript-noise and `assistant` tag symptoms, and it has a grounded historical backfill lane, but the live REM path still ranks reflection themes by concept-tag count/strength and feeds Dream Diary from candidate truths or the first recent snippets rather than a semantic-importance selector. The reporter’s follow-up specifically says a prompt tweak was insufficient because frequency-shaped REM material still reached the model, and I found no merged fix or canonical duplicate for that remaining behavior.

## What This Changes

Improve memory-core dreaming so REM material selection and Dream Diary inputs prefer semantically important decisions, preferences, conclusions, and unusual events over repeated high-frequency tokens or scaffold chatter.

## Best Possible Solution

Add an internal memory-core semantic-importance/material-density selection layer before REM reflections and REM Dream Diary input. It should prefer snippets with decisions, preferences, conclusions, and unusual events, downrank routine role/scaffold chatter and repeated generic tags, and avoid adding public selection config unless maintainers deliberately choose that surface.

## Reproduction Assessment

Yes, for the code-level failure mode: a focused unit case can feed `previewRemDreaming` repeated generic concept tags plus lower-frequency meaningful snippets and observe the current tag-count theme ranking. A full live reproduction of the reporter’s 10-day corpus is not available in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable solution is an internal semantic-density scorer in `memory-core` before REM reflection and narrative input selection. A prompt-only change is not enough because the material has already been selected before the narrative prompt runs, and a new public `dreaming.promptStyle` knob is not justified as the first fix.
