## Summary

Keep open. Current main has added useful transcript-noise and stopword filtering, but REM reflections still rank repeated concept tags and Dream Diary still consumes recurrence-shaped themes without a semantic-importance selector; the related open memory issues overlap but do not fully own this REM/Dream Diary material-selection request.

## What This Changes

The issue asks for Dreaming material selection to prioritize meaningful decisions, conclusions, preferences, and unusual events over high-frequency low-value transcript terms.

## Best Possible Solution

Define an internal material-importance scorer for REM and Dream Diary input, coordinate it with the broader Dreaming corpus-quality threads, and avoid exposing a public promptStyle knob until the ranking contract is settled.

## Reproduction Assessment

Yes, at source level: current main still builds REM themes from repeated concept-tag counts and feeds those themes into Dream Diary. The reporter's exact output is not live-reproducible without their private corpus.

## Solution Assessment

Partly. A semantic-density or material-importance selector before REM and narrative generation is the maintainable direction, but adding a public promptStyle setting first would expose product surface before the internal ranking problem is settled.
