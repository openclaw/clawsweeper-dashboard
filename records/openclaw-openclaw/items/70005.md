## Summary

Keep open. Current main filters some scaffolding tokens and transcript noise, but REM still ranks reflection themes by concept-tag counts and passes those themes plus candidate truths or recent snippets to Dream Diary without a semantic-importance selector.

## What This Changes

Improve memory-core Dreaming material selection so summaries prefer decisions, conclusions, preferences, and unusual events over high-frequency transcript tokens or routine chatter.

## Best Possible Solution

Add an internal memory-core material-importance scorer before REM reflection and Dream Diary input so decisions, preferences, conclusions, and unusual events outrank repeated generic tags and routine scaffold chatter.

## Reproduction Assessment

Yes for the code-level failure mode: a focused unit case can feed repeated generic concept tags plus lower-frequency meaningful snippets into REM preview and observe the current count-based theme ranking. A full live reproduction of the reporter's private corpus is not available here.

## Solution Assessment

Yes, an internal semantic-density scorer before REM selection is the narrowest maintainable direction. A prompt-only tweak is insufficient because the frequency-shaped material has already been selected, and a public `dreaming.promptStyle` knob should not be the first repair.

Fix https://github.com/openclaw/openclaw/issues/70005 by adding an internal memory-core material-importance scoring step before REM reflection and Dream Diary narrative input. The scorer should favor snippets that contain user decisions, preferences, conclusions, plans, unusual events, named entities, or durable technical outcomes, and downrank routine assistant acknowledgments, scaffold/role chatter, repeated generic concept tags, cron/heartbeat remnants, and low-content exchanges. Keep this inside memory-core; do not add a public `dreaming.promptStyle` config unless the implementation proves a config seam is necessary. Add focused tests showing lower-frequency meaningful snippets outrank repeated generic tags for REM preview/Dream Diary material. Preserve existing stopword, transcript-sanitization, and grounded REM behavior, and do not broaden this into legacy corpus deletion or unrelated #68449 cleanup unless a small shared helper is necessary.
