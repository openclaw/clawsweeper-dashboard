## Summary

Keep open. Current main filters some transcript scaffolding, but REM still ranks reflection themes by repeated concept-tag counts and Dream Diary still consumes those recurrence-shaped themes without a semantic-importance selector; the related open issues overlap but do not fully own this REM/Dream Diary material-selection request.

## What This Changes

The issue asks for memory-core Dreaming to choose semantically meaningful session material for REM and Dream Diary instead of high-frequency transcript terms.

## Best Possible Solution

Define an internal material-importance scorer for REM and Dream Diary input, align it with the broader Dreaming corpus-quality threads, and avoid exposing a new promptStyle knob until the ranking contract is clear.

## Reproduction Assessment

Yes, at source level: current main still builds REM themes from repeated concept-tag counts and feeds those themes into Dream Diary. The reporter's exact output is not live-reproducible without their private corpus.

## Solution Assessment

Partly. A semantic-density or material-importance selector before REM and narrative generation is the maintainable direction, but adding a public promptStyle setting first would expose product surface before the internal ranking problem is settled.
