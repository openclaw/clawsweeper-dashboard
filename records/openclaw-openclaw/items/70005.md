## Summary

Keep open. Current main has partial noise cleanup for role tokens and transcript scaffolding, but the automatic REM and Dream Diary path still selects material through fixed source scores, recurrence/count-based themes, and concept-density confidence rather than a semantic-importance selector; the related memory-core issues overlap but do not fully replace this user-facing Dream Diary material-selection request.

## What This Changes

The issue requests semantic-importance material selection for memory-core Dreaming so Dream Diary summaries favor decisions, preferences, conclusions, and unusual events over repeated transcript words.

## Best Possible Solution

Define an internal material-importance scorer for REM and Dream Diary input, coordinate it with the broader Dreaming corpus-quality and single-event promotion trackers, and avoid exposing a public promptStyle knob until the ranking contract is settled.

## Reproduction Assessment

Yes, at source level: current main still builds REM themes from repeated concept-tag counts, uses recurrence-weighted candidate confidence, and feeds those outputs into Dream Diary. The reporter's exact dream output is not live-reproducible without their private corpus.

## Solution Assessment

Partly. A semantic-density or material-importance selector before REM and narrative generation is the maintainable direction, but adding a public promptStyle setting first would expose product surface before the internal ranking policy is settled.
