## Summary

Keep open. Current main still builds live REM themes from concept-tag frequency and feeds Dream Diary from candidate truths or recent snippets without a semantic-importance selector, so the remaining material-selection complaint is not implemented by main or by the related open PRs yet.

## What This Changes

The issue requests that Dreaming choose meaningful session material by semantic importance instead of letting repeated generic terms dominate summaries.

## Best Possible Solution

Land a reviewed memory-core change that adds an internal material-importance scorer before REM reflections and Dream Diary input, while treating cron/noise filtering as related but separately tracked cleanup.

## Reproduction Assessment

Yes for the source-level failure mode: current main’s REM preview still ranks repeated concept tags by count and the narrative input path has no semantic-importance prefilter. A full live reproduction of the reporter’s private corpus is not available in this read-only review.

## Solution Assessment

Partly: a semantic-density selector before REM/Dream Diary is the maintainable direction, but adding a public promptStyle/config knob first would add product surface before fixing the internal ranking problem.
