## Summary

Keep open. Current main filters some scaffold noise, including `assistant`, but live REM still ranks recurring concept tags by count/strength and Dream Diary input is built from candidate truths or recent snippets without a semantic-importance selector, so the reported product-quality gap remains valid.

## What This Changes

Improve memory-core Dreaming material selection so REM/Dream Diary summaries prioritize meaningful decisions, preferences, conclusions, and unusual events over frequent generic words or routine transcript noise.

## Best Possible Solution

Add an internal memory-core material-importance scorer before REM reflection and Dream Diary input so decisions, preferences, conclusions, and unusual events outrank repeated generic tags and routine scaffold chatter.

## Reproduction Assessment

Yes for the code-level failure mode: a focused test can feed repeated generic concept tags plus lower-frequency meaningful snippets into `previewRemDreaming`/REM narrative input and observe count-based theme ranking. A full live reproduction of the reporter's private corpus is unavailable from this read-only review.

## Solution Assessment

Yes: an internal semantic-density scorer before REM selection is the narrowest maintainable direction. A prompt-only change is insufficient because frequency-shaped material has already been selected before narrative generation, and a public `dreaming.promptStyle` knob should not be the first repair.
