## Summary

Keep open. Current main mitigates some Dreaming noise, but the live REM path still ranks recurring concept tags by count/strength and feeds Dream Diary from candidate truths or recent snippets without a semantic-importance selector, so the reporter's remaining product-quality complaint is still valid.

## What This Changes

The issue requests a semantic-importance material selection layer for Dreaming so summaries prioritize decisions, preferences, conclusions, and unusual events over repeated generic words or scaffold chatter.

## Best Possible Solution

Add an internal memory-core material-importance scorer before REM reflection and Dream Diary input so decisions, preferences, conclusions, and unusual events outrank repeated generic tags and routine scaffold chatter.

## Reproduction Assessment

Yes for the code-level failure mode: a focused unit case can feed repeated generic concept tags plus lower-frequency meaningful snippets into the REM preview path and observe the current count-based theme ranking. A full live reproduction of the reporter's private corpus is not available from this read-only review.

## Solution Assessment

Yes, an internal semantic-density scorer before REM selection is the narrowest maintainable fix. A prompt-only change is not enough because the frequency-shaped material has already been selected before narrative generation, and a public promptStyle knob should not be the first repair.
