## What This Changes

The branch updates WebChat `chat.abort` handling to persist terminal killed session state, locally clear confirmed stopped runs, add stale run-generation guards, and cover the flow with Gateway/UI regressions.

## Best Possible Solution

Land this PR after maintainer session-state review and current-head checks, then let it close the linked WebChat Stop recovery bug.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main does not clear WebChat local run state on confirmed abort and does not reject older same-session terminal lifecycle events by run generation. I did not run a live repro in this read-only review.

## Solution Assessment

Yes, this appears to be the best fix shape: the server makes a confirmed abort terminal in stored session state, the canonical lifecycle writer rejects stale generations, and the client only clears the captured run after the response confirms that run. A client-only fix would leave persisted session rows wrong, while a server-only fix would not cover delayed or lost terminal UI events.
