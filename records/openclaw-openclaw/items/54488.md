## Summary

Keep this open: current main has useful mitigations, but it still does not fully solve the central follow-up drain fairness problem because active non-heartbeat work can still be appended to the same follow-up FIFO and drained through the same session lane without a foreground-user yield policy.

## What This Changes

The issue reports session-lane starvation where follow-up or background work can monopolize a single session lane and delay inbound or outbound user dispatch for long periods.

## Best Possible Solution

Add scheduler-level fairness so fresh user/control turns can overtake or interrupt accumulated background/follow-up work on the same session key while preserving deterministic ordering within each priority class and transcript safety.

## Reproduction Assessment

Yes at source level: current main still enqueues active non-heartbeat follow-up work into a FIFO follow-up queue and drains it through the same session lane. I did not run the live multi-channel timing reproduction in this read-only pass.

## Solution Assessment

No, current main is not yet the complete fix. The existing foreground/background command-lane priority and open admission PR are useful pieces, but the best solution still needs an explicit foreground fairness policy for the follow-up drain itself.
