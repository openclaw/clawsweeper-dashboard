## Summary

Keep open. Current main still has the central followup-drain fairness problem: active non-heartbeat work is appended to a FIFO followup queue, the drain runs until the queue is empty, and downstream session-lane priority does not create a fresh-inbound preemption point.

## What This Changes

The issue requests scheduler fairness so fresh inbound and control work is not starved behind accumulated followup turns on the same session lane.

## Best Possible Solution

Define a scheduler-level fairness contract that lets fresh user/control work win over accumulated background/followup backlog while preserving per-session serialization and queue metadata semantics.

## Reproduction Assessment

Yes, at source level: active non-heartbeat work can still enqueue into the same FIFO followup queue, and the drain runs that queue until empty. I did not run the full long-lived multi-channel reproduction in this read-only pass.

## Solution Assessment

Unclear on the exact implementation. The best direction is scheduler-level fairness; session-lane priority or concurrency alone does not solve the followup queue's selection and preemption problem.
