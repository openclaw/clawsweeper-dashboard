## Summary

Keep open. Current main has useful partial fixes, including clearing the reply run before scheduling followup drain and a lane task timeout, but the followup queue is still FIFO and drains until empty, so fresh inbound work can still be appended behind accumulated followup turns.

## What This Changes

Request scheduler fairness so followup drains cannot monopolize a session lane and delay inbound or outbound message dispatch for long periods.

## Best Possible Solution

Add scheduler-level fairness to the auto-reply followup path: bound consecutive followup work or drain time, let fresh user/control inbound work run ahead of accumulated system/followup backlog on the same session key, and preserve FIFO ordering within each class with regression coverage.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: accumulate followup queue work, keep a run active, then send fresh inbound work and it is appended behind the same FIFO followup drain. I did not run a live 20-30 minute gateway reproduction in this read-only pass.

## Solution Assessment

Yes, the requested scheduler-fairness direction is the narrow maintainable fix. Current main's cleanup-before-drain and lane timeout reduce adjacent failure windows, but they do not bound or prioritize the followup drain itself.
