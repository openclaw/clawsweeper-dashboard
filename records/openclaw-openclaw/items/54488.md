## Summary

Keep open. Current main still has the central fairness gap: active non-heartbeat inbound work is appended to the same followup FIFO, the drain keeps running until that queue is empty, and each followup turn re-enters the per-session FIFO lane. The related open reports are adjacent responsiveness and notification-delivery work, not a merged or shipped replacement for this scheduler bug.

## What This Changes

The issue reports that auto-reply followup draining can monopolize a per-session lane and delay later inbound Discord or WhatsApp messages for 20-30 minutes.

## Best Possible Solution

Add scheduler-level fairness for auto-reply followups: bound consecutive followup turns or drain elapsed time, let fresh user/control inbound work run ahead of accumulated system/followup backlog on the same session key, preserve FIFO ordering within each class, and cover the behavior with focused queue and run-reply regressions.
