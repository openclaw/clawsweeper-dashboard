## Summary

Keep this PR open: it fixes a real unbounded ChatGPT Responses error-body read, but the current patch applies the 400-character display cap before JSON parsing, which can regress existing friendly usage-limit errors for valid bounded JSON bodies over 400 characters.

## What This Changes

The PR replaces two ChatGPT Responses provider `response.text()` error-body reads with the shared bounded snippet reader and adds focused provider regression tests.

## Best Possible Solution

Keep the bounded byte and idle-timeout protection, but parse JSON from a bounded untruncated body and apply the 400-character cap only to the fallback display diagnostic.
