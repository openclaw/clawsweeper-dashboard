## Summary

Keep this PR open as a focused, useful security/availability fix. Current main still has the two unbounded ChatGPT Responses error-body reads, while the PR moves both through the existing capped snippet reader, mirrors the merged Anthropic hardening pattern, and includes enough after-fix runtime proof for this defensive malformed-response path.

## What This Changes

Bounds ChatGPT Responses non-OK error body reads with the shared snippet reader and adds regression coverage for oversized stream cancellation plus normal JSON friendly errors.

## Best Possible Solution

Merge this focused bounded-reader fix after required CI stays green, keeping ChatGPT Responses error diagnostics aligned with the shared media-core and Anthropic bounded-read pattern.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? ## Solution Assessment

Is this the best way to solve the issue? Yes; reusing `readResponseTextSnippet` at both current body-read sites is the narrowest maintainable fix, and a bespoke reader or retry rewrite would duplicate the existing media-core and Anthropic pattern.
