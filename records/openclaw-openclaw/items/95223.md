## Summary

Keep this PR open: the OpenAI/ChatGPT Responses error-body hardening is still useful because current main still has unbounded non-OK body reads, but the current patch has blocking regressions in browser-safe provider loading and structured error parsing.

## What This Changes

The PR replaces ChatGPT Responses non-OK error-body reads with a shared bounded snippet reader and adds provider regression tests for oversized streamed errors and normal usage-limit JSON.

## Best Possible Solution

Land a bounded, browser-safe ChatGPT Responses error-body reader that parses structured JSON from the bounded raw body before truncating only fallback diagnostics.

## Reproduction Assessment

Yes. Source inspection shows current main has two unbounded response.text() reads, and the PR diff shows the new helper truncates before JSON parsing and relies on a Buffer-backed helper in the browser-sensitive provider path.

## Solution Assessment

No as written. Reusing the bounded-read idea is the right mitigation, but this provider needs a browser-safe bounded raw-body path and should apply the 400-character cap only after structured error parsing.
