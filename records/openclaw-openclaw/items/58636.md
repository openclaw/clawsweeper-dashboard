## Summary

Keep open. Current main still does not implement TUI `/upload`, and the PR is a useful feature direction, but the proposed patch still has merge-blocking file-read and queued-context lifecycle bugs plus missing user-facing docs/changelog coverage.

## What This Changes

The PR adds a TUI `/upload` slash command, a file-browser select-list overlay, theme wiring, and tests so selected local files are prepended to the next TUI message.

## Best Possible Solution

Repair the PR branch or a narrow replacement branch with bounded regular-file reads, correct pending-context lifecycle, command-handler tests, TUI docs, and a changelog entry while preserving the stated no-CLI/no-protocol scope.

## Reproduction Assessment

Yes. The remaining bugs have high-confidence code-path reproductions: selecting a huge or special file reaches `readFileSync` before validation, failed `sendChat` drops queued attachments, and a replacement upload where every path is skipped leaves older context queued.

## Solution Assessment

No. The TUI-only direction is reasonable, but this implementation is not the best merge path until file reads are bounded, queued context is cleared only at the right lifecycle points, and user-facing documentation/release notes are added.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Validate files before reading them:** `src/tui/tui-command-handlers.ts:257`
  - body: `readFileSync(filePath)` runs before a regular-file check or size cap. Because the browser treats every non-directory entry as selectable, choosing a FIFO/device node or very large file can block or exhaust the TUI before binary detection runs.
  - confidence: 0.9
- **[P2] Clear queued uploads only after send succeeds:** `src/tui/tui-command-handlers.ts:570-572`
  - body: `pendingFileContext` is nulled before `client.sendChat` resolves. If the gateway/provider send rejects, retrying the same message loses the selected file context and sends only the typed text.
  - confidence: 0.86
- **[P2] Clear stale context when upload attaches no files:** `src/tui/tui-command-handlers.ts:278-282`
  - body: When a replacement upload is confirmed but all selected paths are skipped or fail to read, this branch leaves the previous `pendingFileContext` intact. The next message can silently send stale file contents.
  - confidence: 0.84
- **[P3] Add TUI upload docs and changelog:** `src/tui/commands.ts:117`
  - body: This adds a user-facing slash command, but the PR does not update `docs/web/tui.md` or `CHANGELOG.md`, so the public command list and release notes would omit the feature.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: Security review needs attention because the patch introduces an unbounded local file-read path in the TUI upload handler.

Concerns:

- **[medium] Unbounded local file read:** `src/tui/tui-command-handlers.ts:257`
  - body: The upload handler synchronously reads each selected path before checking that it is a regular file or enforcing a size limit. A selected special file or very large file can block or exhaust the TUI process, so the handler should stat and bound reads before loading file contents.
  - confidence: 0.88
