## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Keep pending upload context until send succeeds:** `src/tui/tui-command-handlers.ts:572`
  - body: `pendingFileContext` is cleared before `client.sendChat` is awaited. If the gateway or provider send fails, the user's selected files are gone and a retry sends only the typed message, so clear the context only after a successful send or restore it in the catch path.
  - confidence: 0.93
- **[P1] Clear stale upload context when no files attach:** `src/tui/tui-command-handlers.ts:278`
  - body: When a user confirms a replacement upload but every selected path is skipped or unreadable, this branch leaves the previous `pendingFileContext` queued. The next message can silently include older files, so reset the pending context before processing or explicitly clear it when `blocks` is empty.
  - confidence: 0.91
- **[P1] Validate selected files before synchronous reads:** `src/tui/tui-command-handlers.ts:257`
  - body: The handler reads each selected path with `readFileSync` before checking that it is a regular, bounded-size file. Because the browser exposes every non-directory entry as selectable, paths like FIFOs, devices, or very large files can block or exhaust the TUI; filter to regular files and enforce a size cap before reading.
  - confidence: 0.88
