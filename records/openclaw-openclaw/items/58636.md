## Summary

Keep open: current main still lacks the `/upload` TUI command, and this PR contains useful new feature work with screenshot proof, but the branch has blocking package, lifecycle, and file-read validation defects before maintainers can make the product decision to ship it.

## What This Changes

The PR adds a TUI `/upload` slash command, a file-browser selector, theme wiring, pending file-context injection into the next message, and file-browser component tests.

## Best Possible Solution

Repair the PR branch against the current TUI package and command-handler contract, add upload lifecycle and regular-file validation coverage, then let maintainers decide whether core TUI should ship `/upload`.

## Reproduction Assessment

Yes for the PR blockers: PR-head source shows the stale package import, premature pending-context clear, stale replacement path, and non-regular file predicate. The broader `/upload` behavior is a new feature, so there is no current-main bug reproduction path.

## Solution Assessment

No. The feature direction may be useful, but this implementation should first rebase to the current TUI package, preserve pending uploads until `sendChat` succeeds, clear stale context on empty replacement uploads, and restrict reads to regular files.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Import pi-tui from the current package:** `src/tui/components/file-browser-select-list.ts:10`
  - body: Current main depends on `@earendil-works/pi-tui`, but this new component imports `@mariozechner/pi-tui`. A rebased branch will fail to resolve the old package unless the PR updates its TUI imports to the current package.
  - confidence: 0.93
- **[P1] Preserve pending uploads until send succeeds:** `src/tui/tui-command-handlers.ts:570-572`
  - body: `pendingFileContext` is cleared before `client.sendChat` resolves. If the gateway or provider rejects the send, the selected files are lost and retrying sends only the typed text.
  - confidence: 0.9
- **[P1] Clear stale context when replacement upload attaches nothing:** `src/tui/tui-command-handlers.ts:278-282`
  - body: A confirmed upload that produces no `blocks` leaves the previous `pendingFileContext` intact. If every newly selected path is skipped as binary or fails to read, the next message silently includes older files instead of no attachment.
  - confidence: 0.88
- **[P1] Filter selectable entries to regular files:** `src/tui/components/file-browser-select-list.ts:77-81`
  - body: Every non-directory entry is shown and selectable as a file, but the handler later calls `readFileSync` on the selected path. Device nodes, FIFOs, and sockets are non-directories too, so selecting one can block or consume memory before binary detection runs.
  - confidence: 0.87
