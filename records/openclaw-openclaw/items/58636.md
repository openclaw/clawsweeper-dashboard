  - confidence: 0.9
- **[P1] Keep pending file context until sendChat succeeds:** `src/tui/tui-command-handlers.ts:572`
  - body: `pendingFileContext` is cleared while composing the outbound message, before `client.sendChat` accepts the turn; a transient send failure drops the selected files so retrying sends only the typed prompt.
  - confidence: 0.92
- **[P1] Clear stale context when an upload attaches no files:** `src/tui/tui-command-handlers.ts:278`
  - body: If every selected path is skipped or fails to read, `blocks.length` stays zero and the old pending file context remains queued, so a later normal message can silently include previous files.
  - confidence: 0.9
- **[P1] Filter selectable entries to regular files:** `src/tui/components/file-browser-select-list.ts:77-81`
  - body: The browser treats every non-directory entry as selectable, including devices, sockets, and FIFOs; those paths later go to `readFileSync`, which can block or consume unbounded data.
  - confidence: 0.91
- **[P1] Bound file reads before loading content:** `src/tui/tui-command-handlers.ts:257`
  - body: The handler reads the entire selected path into memory before any size or prompt-budget check, so a large text file can freeze the TUI or create an enormous prompt.
  - confidence: 0.9
- **[P1] Wrap attached file text as untrusted content:** `src/tui/tui-command-handlers.ts:273`
  - body: Raw file text is concatenated with spoofable delimiters and a `[User message]` marker, while current Gateway file inputs wrap decoded file text as untrusted external content before adding it to model context.
  - confidence: 0.88
