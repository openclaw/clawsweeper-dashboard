## Summary

Keep open: current main still has no TUI `/upload` command and this PR is the active implementation attempt, but the patch is not ready because PR-head source still has blocking TUI dependency, upload lifecycle, and filesystem-read issues.

## What This Changes

The PR adds a TUI `/upload` slash command, a file-browser select-list overlay, theme wiring, upload handling that prepends selected local file contents to the next message, and component tests.

## Best Possible Solution

Keep the PR open, repair the TUI implementation against current main, add focused handler/file-browser coverage for the edge cases, then let maintainers make the product decision on shipping the feature.

## Reproduction Assessment

Yes for the PR blockers: PR-head source shows the stale package import, pending context cleared before awaited send, empty replacement uploads leaving prior context queued, and non-regular entries selectable. The underlying `/upload` behavior is a new feature, not a current-main bug reproduction path.

## Solution Assessment

No. The feature direction is plausible, but this implementation should first be rebased to current TUI contracts, preserve queued uploads until send success, clear stale uploads on failed replacement attempts, and restrict reads to regular bounded files.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the current pi-tui package:** `src/tui/components/file-browser-select-list.ts:10`
  - body: Current main imports TUI APIs from `@earendil-works/pi-tui` and only pins that package, but the new file browser still imports `@mariozechner/pi-tui`. A rebased branch will fail to resolve or mix incompatible component types until these imports are updated.
  - confidence: 0.93
- **[P1] Keep uploaded context until sendChat succeeds:** `src/tui/tui-command-handlers.ts:570-572`
  - body: `pendingFileContext` is cleared before `client.sendChat` resolves. If the gateway or provider rejects the send, retrying loses the selected file context and sends only the typed message.
  - confidence: 0.9
- **[P1] Clear old uploads when a replacement attaches nothing:** `src/tui/tui-command-handlers.ts:278-282`
  - body: When a later `/upload` confirms but every selected path is skipped or unreadable, this branch leaves the previous `pendingFileContext` queued. The next message can silently include old files after the user attempted to replace them with no attachment.
  - confidence: 0.88
- **[P1] Limit upload selection to regular bounded files:** `src/tui/components/file-browser-select-list.ts:77-81`
  - body: Every non-directory entry is selectable, but the handler later calls `readFileSync` on selected paths. Device nodes, FIFOs, sockets, and very large files can block the TUI or consume unbounded memory before binary detection runs.
  - confidence: 0.87
