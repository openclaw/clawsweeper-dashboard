Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Update the file browser to the current Pi TUI package:** `src/tui/components/file-browser-select-list.ts:10`
  - body: Current main imports and depends on `@earendil-works/pi-tui`, but the new component imports `@mariozechner/pi-tui`. Rebase and update the new/touched imports so the branch builds against current main instead of reintroducing the retired package name.
  - confidence: 0.95
- **[P1] Do not reserve `/upload` ahead of dynamic commands:** `src/tui/commands.ts:117`
  - body: `getSlashCommands` seeds `seen` from built-ins before appending gateway and dynamic commands, so adding `upload` here suppresses any plugin or gateway command with that alias. This needs maintainer-approved command ownership or a collision-safe routing plan.
  - confidence: 0.9
- **[P1] Keep upload context queued until `sendChat` accepts the turn:** `src/tui/tui-command-handlers.ts:572`
  - body: `pendingFileContext` is cleared before `client.sendChat` resolves, so a transient gateway/provider failure drops the selected files and a retry sends only the typed prompt. Clear the context only after the send is accepted, with regression coverage for failure and retry.
  - confidence: 0.93
- **[P1] Clear stale upload context when replacement upload attaches nothing:** `src/tui/tui-command-handlers.ts:278-282`
  - body: If every selected path is skipped or fails to read, `blocks.length` stays zero and the old `pendingFileContext` remains queued. A later normal message can silently include previous files, so clear pending context before processing a replacement upload or explicitly clear it on an empty result.
  - confidence: 0.91
- **[P1] Only surface regular files for upload:** `src/tui/components/file-browser-select-list.ts:77-81`
  - body: The browser treats every non-directory entry as a selectable file, including FIFOs, devices, and sockets. Passing those paths to `readFileSync` can block or consume unbounded data, so only surface regular files and revalidate before reading.
  - confidence: 0.92
- **[P1] Bound file reads before loading selected content:** `src/tui/tui-command-handlers.ts:257`
  - body: The handler reads the entire selected path into memory before any size or prompt-budget check. A large text file can freeze the TUI or create an enormous prompt, so stat and cap size before reading through a bounded helper.
  - confidence: 0.9
- **[P1] Wrap attached file text as untrusted external content:** `src/tui/tui-command-handlers.ts:273`
  - body: Raw file text is concatenated into the user prompt with spoofable delimiters, bypassing the existing wrapper that marks decoded file text as untrusted external content. Use the existing external-content boundary so file bytes stay data, not trusted prompt structure.
  - confidence: 0.88
