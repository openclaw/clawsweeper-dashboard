## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep pending upload context until send succeeds:** `src/tui/tui-command-handlers.ts:572`
  - body: `pendingFileContext` is cleared before `client.sendChat` resolves, so a transient send failure drops the selected file context and a retry sends only the typed message. Clear it only after the send succeeds, or restore it in the catch path.
  - confidence: 0.93
- **[P1] Clear stale upload context when no files attach:** `src/tui/tui-command-handlers.ts:278-282`
  - body: When a user confirms a replacement upload but every selected path is skipped or unreadable, `blocks.length` stays zero and the previous `pendingFileContext` remains queued. That can silently send older file contents with the next message; clear pending context before processing or in the empty result path.
  - confidence: 0.91
- **[P1] Reject non-regular files before reading:** `src/tui/components/file-browser-select-list.ts:80-81`
  - body: The browser treats every non-directory as a selectable file, including FIFOs and device nodes, and the handler later calls `readFileSync` on the selected path. Selecting something like a FIFO or `/dev/zero` can block or consume unbounded memory, so only regular files should be selectable and the handler should validate again before reading.
  - confidence: 0.9
