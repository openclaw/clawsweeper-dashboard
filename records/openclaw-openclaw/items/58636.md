## Summary

Keep this PR open, but it is not mergeable as-is: current main still lacks the requested TUI `/upload` feature, while the PR head still has build-compatibility, pending-context lifecycle, and filesystem-read blockers.

## What This Changes

The PR adds a TUI `/upload` slash command with a file-browser overlay that multi-selects local files and prepends their contents to the next chat message.

## Best Possible Solution

Repair the branch against current TUI contracts, keep uploads queued until a send succeeds, clear stale uploads on empty replacement attempts, restrict uploads to regular files, then let maintainers decide whether the core TUI should carry the feature.

## Reproduction Assessment

Yes for the PR blockers: PR-head source shows the stale package import, premature pending-context clear, stale replacement-upload path, and non-regular file selection. The underlying `/upload` behavior is a new feature, so there is no current-main bug reproduction path.

## Solution Assessment

No. The feature direction may be useful, but this implementation should first rebase to the current package imports and fix pending-context and regular-file validation before maintainers judge the product fit.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Update pi-tui imports to the current package:** `src/tui/components/file-browser-select-list.ts:10`
  - body: Current main imports TUI APIs from `@earendil-works/pi-tui`, but the added file browser imports `@mariozechner/pi-tui`. A rebased branch will fail to resolve or split TUI types across stale package names until this is updated.
  - confidence: 0.93
- **[P1] Keep pending uploads until sendChat succeeds:** `src/tui/tui-command-handlers.ts:570-572`
  - body: `pendingFileContext` is cleared before `client.sendChat` resolves. If the gateway or provider rejects the send, the user loses the selected file context and retrying sends only the typed message.
  - confidence: 0.9
- **[P1] Clear stale context when uploads attach nothing:** `src/tui/tui-command-handlers.ts:278-282`
  - body: When a replacement `/upload` confirms but every selected path is skipped or unreadable, this branch leaves the previous `pendingFileContext` queued. The next message can silently include old files instead of no attachment.
  - confidence: 0.88
- **[P1] Limit selectable entries to regular files:** `src/tui/components/file-browser-select-list.ts:77-81`
  - body: Every non-directory entry is shown as a selectable file, but the handler later calls `readFileSync` on selected paths. Device nodes, FIFOs, sockets, and similar entries can block the TUI or consume unbounded memory before binary detection runs.
  - confidence: 0.87
