Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the current Pi TUI package:** `src/tui/components/file-browser-select-list.ts:10`
  - body: Current main depends on `@earendil-works/pi-tui` and current TUI imports use that package, but this new component imports `@mariozechner/pi-tui`. Rebase and update the new file so a fresh install/build does not depend on an undeclared retired package path.
  - confidence: 0.96
- **[P1] Avoid reserving /upload ahead of dynamic commands:** `src/tui/commands.ts:117`
  - body: `getSlashCommands` seeds `seen` from built-ins before appending gateway and dynamic commands, so adding a built-in `upload` entry can suppress a plugin or gateway command with the same alias. This needs an explicit namespace or collision plan instead of silently taking over `/upload`.
  - confidence: 0.9
- **[P1] Preserve queued files until sendChat succeeds:** `src/tui/tui-command-handlers.ts:572`
  - body: `pendingFileContext` is cleared before `client.sendChat` accepts the turn. If the send fails, the user loses the selected files and a retry sends only the typed prompt, so clear the context only after successful acceptance.
  - confidence: 0.93
- **[P1] Clear stale queued files when upload attaches nothing:** `src/tui/tui-command-handlers.ts:278-282`
  - body: When every selected path is skipped or fails to read, `blocks.length` stays zero and the old `pendingFileContext` remains queued. A later normal message can silently include previous files, so clear pending context before processing a replacement upload or on an empty result.
  - confidence: 0.91
- **[P1] Only allow regular files through the picker:** `src/tui/components/file-browser-select-list.ts:77-81`
  - body: The browser treats every non-directory entry as a selectable file, including FIFOs, sockets, and device nodes. Passing those paths to the synchronous reader can block or consume unbounded data, so surface only regular files and revalidate before reading.
  - confidence: 0.92
- **[P1] Check file size before reading selected paths:** `src/tui/tui-command-handlers.ts:257`
  - body: The handler reads the entire selected path into memory before any size or prompt-budget check. A large text file can freeze the TUI or create an enormous prompt, so stat and cap selected files before loading their contents.
  - confidence: 0.9
- **[P1] Wrap selected file text as untrusted content:** `src/tui/tui-command-handlers.ts:273`
  - body: Raw file text is concatenated with spoofable delimiters and a `[User message]` marker, bypassing the existing external-content boundary used for decoded file text. Reuse that wrapper so selected files are treated as data rather than trusted prompt structure.
  - confidence: 0.89
