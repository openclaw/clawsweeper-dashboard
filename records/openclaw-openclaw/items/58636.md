## Summary

Keep open. Current main still does not implement a TUI `/upload` command, and the PR contains useful feature work, but the branch is not merge-ready because PR-head source still has stale TUI package imports plus pending-context lifecycle and filesystem-read blockers.

## What This Changes

The PR adds a TUI `/upload` slash command, file-browser selector, theme wiring, pending file-context injection into the next chat send, and selector unit tests.

## Best Possible Solution

Rebase and repair the branch against current TUI contracts, add focused lifecycle and filesystem validation coverage, then let maintainers decide whether the feature belongs in core TUI.

## Reproduction Assessment

Yes for the PR blockers: PR-head source shows the stale package import, premature pending-context clear, stale replacement path, and non-regular file predicate. The broader /upload behavior is a new feature, so there is no current-main bug reproduction path.

## Solution Assessment

No. The feature direction may be useful, but this implementation should first rebase to the current TUI package, preserve pending uploads until sendChat succeeds, clear stale context on empty replacement uploads, and restrict reads to regular files.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Import pi-tui from the current package:** `src/tui/components/file-browser-select-list.ts:10`
  - body: Current main depends on `@earendil-works/pi-tui`, but this added component imports the old `@mariozechner/pi-tui` package. A rebased branch will fail to resolve or will keep the TUI split across stale package names until these imports are updated.
  - confidence: 0.93
- **[P1] Keep pending uploads until send succeeds:** `src/tui/tui-command-handlers.ts:570-572`
  - body: `pendingFileContext` is cleared before `client.sendChat` resolves. If the gateway or provider rejects the send, the selected file context is lost and retrying sends only the typed message.
  - confidence: 0.9
- **[P1] Clear stale context when an upload yields nothing:** `src/tui/tui-command-handlers.ts:278-282`
  - body: When a replacement upload is confirmed but every selected path is skipped or fails to read, this branch leaves the prior `pendingFileContext` queued. The next message can silently include older files instead of no attachment.
  - confidence: 0.88
- **[P1] Restrict selectable entries to regular files:** `src/tui/components/file-browser-select-list.ts:77-81`
  - body: Every non-directory entry is shown and selectable as a file, but the handler later reads selected paths synchronously. Device nodes, FIFOs, sockets, and similar entries can block or exhaust memory before binary detection runs.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff adds local file reads and currently lets non-regular filesystem entries reach synchronous reads.

Concerns:

- **[medium] Unbounded reads from special files:** `src/tui/components/file-browser-select-list.ts:77`
  - body: The browser classifies every non-directory as selectable while the handler calls `readFileSync` on selected paths; character devices, FIFOs, sockets, and very large files can block the TUI or consume unbounded memory before binary detection can help.
  - confidence: 0.87
