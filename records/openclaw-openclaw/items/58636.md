## Best Possible Solution

Merge only after a rebased `/upload` implementation uses current TUI dependencies, treats queued file context transactionally, and accepts only bounded regular text files with focused regression tests.

## Solution Assessment

No. The feature direction is plausible, but the current implementation should be rebased and hardened so file context is transactional and file reads are bounded before maintainers decide whether to accept the new TUI command.

## Review Findings

Overall confidence: 0.89

Full review comments:

- **[P1] Use the current pi-tui package name:** `src/tui/components/file-browser-select-list.ts:10`
  - body: Current main no longer depends on `@mariozechner/pi-tui`; this new component would import a missing package after rebase/merge. Update the added imports to `@earendil-works/pi-tui` and refresh the branch against current main.
  - confidence: 0.9
- **[P1] Keep pending uploads until send succeeds:** `src/tui/tui-command-handlers.ts:572`
  - body: `pendingFileContext` is cleared before `client.sendChat` resolves, so a transient send failure drops the selected files and a retry sends only the typed text. Clear it only after `sendChat` succeeds, and keep it available on failure.
  - confidence: 0.93
- **[P1] Clear stale context when an upload attaches nothing:** `src/tui/tui-command-handlers.ts:278`
  - body: If a user already has files queued and then confirms a replacement upload where every selected file is skipped or unreadable, this branch leaves the old context queued. Clear pending context at the start of a confirmed upload or explicitly when `blocks.length === 0`.
  - confidence: 0.9
- **[P1] Filter selectable entries to regular files:** `src/tui/components/file-browser-select-list.ts:80-81`
  - body: Every non-directory entry is pushed into the selectable file list, which includes FIFOs and device nodes. Because the handler later reads selected paths synchronously, entries such as device files can freeze the TUI; only regular files should be surfaced or accepted.
  - confidence: 0.88
- **[P2] Bound file reads before loading content:** `src/tui/tui-command-handlers.ts:257`
  - body: The handler reads the entire file before sampling for binary content, so selecting a huge regular file can allocate unbounded memory and stall the TUI. Check file size first and reject or truncate before `readFileSync`.
  - confidence: 0.86

## Security Review

Status: needs_attention
