## Summary

Keep open. Current main still does not include a TUI `/upload` command, and this PR is a real feature proposal, but the branch still has correctness and security blockers that should be repaired before maintainers decide whether to ship the new core TUI surface.

## What This Changes

The PR adds a TUI `/upload` slash command with a file-browser overlay, file selection tests, theme wiring, and pending file-context injection into the next sent message.

## Best Possible Solution

Repair the branch against the current TUI package and command-handler contract, add upload lifecycle plus regular-file and size validation coverage, then make the product call on shipping `/upload` in core TUI.

## Reproduction Assessment

Yes for the PR blockers: PR-head source shows the stale package import, premature pending-context clear, stale replacement path, and non-regular file predicate. The broader `/upload` behavior is a new feature, so there is no current-main bug reproduction path.

## Solution Assessment

No. The feature direction may be useful, but this implementation should first rebase to the current TUI package, preserve pending uploads until `sendChat` succeeds, clear stale context on empty replacement uploads, and restrict/bound file reads.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Import pi-tui from the current package:** `src/tui/components/file-browser-select-list.ts:10`
  - body: Current main uses `@earendil-works/pi-tui`, but the added file-browser component imports the older `@mariozechner/pi-tui` package. A rebased branch will fail or stay conflicted unless the new component follows the current package name.
  - confidence: 0.93
- **[P1] Preserve pending uploads until send succeeds:** `src/tui/tui-command-handlers.ts:570-572`
  - body: `pendingFileContext` is cleared before `client.sendChat` resolves. If the gateway or provider rejects the send, the selected files are lost and retrying sends only the typed text.
  - confidence: 0.9
- **[P1] Clear stale context when replacement upload has no files:** `src/tui/tui-command-handlers.ts:278-282`
  - body: A confirmed upload that produces no `blocks` leaves the previous `pendingFileContext` intact. If every newly selected path is skipped or fails to read, the next message silently includes older files instead of no attachment.
  - confidence: 0.88
- **[P1] Filter selectable entries to regular files:** `src/tui/components/file-browser-select-list.ts:77-81`
  - body: Every non-directory entry is shown and selectable as a file, but the handler later calls `readFileSync` on selected paths. Device nodes, FIFOs, and sockets are non-directories too, so selecting one can block or consume memory before binary detection runs.
  - confidence: 0.87
- **[P2] Bound file reads before loading attachment content:** `src/tui/tui-command-handlers.ts:257`
  - body: The handler reads the entire selected path synchronously before sampling the first 8KB or checking size. A large regular file can freeze the TUI or exhaust memory before the binary heuristic can help.
  - confidence: 0.84
