## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the current pi-tui package:** `src/tui/components/file-browser-select-list.ts:10`
  - body: Current main depends on `@earendil-works/pi-tui`, but this added component imports `@mariozechner/pi-tui`. As written, the new file does not match the current dependency surface and will fail to resolve after rebasing onto main.
  - confidence: 0.93
- **[P1] Keep pending uploads until send succeeds:** `src/tui/tui-command-handlers.ts:572`
  - body: `pendingFileContext` is cleared before `client.sendChat` resolves. If the send fails, retrying drops the selected file context and sends only the typed message, causing user-visible data loss.
  - confidence: 0.9
- **[P1] Clear stale uploads when replacement attaches nothing:** `src/tui/tui-command-handlers.ts:278-282`
  - body: When a later `/upload` confirms but every selected path is skipped or unreadable, the previous `pendingFileContext` remains queued. The next message can silently include old files after a failed replacement upload.
  - confidence: 0.88
- **[P1] Reject non-regular or oversized files before reading:** `src/tui/tui-command-handlers.ts:257`
  - body: The handler reads selected paths synchronously before checking file type or size. Device nodes, FIFOs, sockets, and very large files can block the TUI or consume unbounded memory before binary detection runs.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff adds local file reads without regular-file and size bounds.
