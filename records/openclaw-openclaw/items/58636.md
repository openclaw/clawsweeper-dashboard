## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Update the file browser to the current Pi TUI package:** `src/tui/components/file-browser-select-list.ts:10`
  - body: Current main depends on `@earendil-works/pi-tui`, but this new component imports the old `@mariozechner/pi-tui` package. Rebase the branch and update the imports so the new file browser builds against the current dependency surface.
  - confidence: 0.95
- **[P1] Do not reserve /upload ahead of dynamic commands:** `src/tui/commands.ts:117`
  - body: `getSlashCommands` seeds `seen` from built-ins before appending gateway and dynamic commands, so adding `upload` here suppresses any plugin or gateway command with that alias. This needs an explicit namespace or collision plan instead of silently taking over `/upload`.
  - confidence: 0.9
- **[P1] Keep upload context queued until sendChat accepts the turn:** `src/tui/tui-command-handlers.ts:572`
  - body: `pendingFileContext` is cleared while composing `messageToSend`, before `client.sendChat` has accepted the turn. A transient send failure drops the selected files, so retrying sends only the typed prompt instead of the attached context.
  - confidence: 0.93
- **[P1] Clear stale upload context when a replacement attaches nothing:** `src/tui/tui-command-handlers.ts:278-282`
  - body: If every selected path is skipped or fails to read, `blocks.length` stays zero and the old `pendingFileContext` remains queued. Clear pending context before processing a replacement upload or explicitly clear it on an empty result.
  - confidence: 0.91
- **[P1] Only allow regular files to be selected:** `src/tui/components/file-browser-select-list.ts:77-81`
  - body: The browser treats every non-directory entry as a selectable file, including FIFOs, devices, and sockets. Passing those paths to `readFileSync` can block or consume unbounded data, so only surface regular files and revalidate before reading.
  - confidence: 0.92
- **[P1] Bound selected file reads before reading:** `src/tui/tui-command-handlers.ts:257`
  - body: The handler reads each selected path fully into memory before any size or prompt-budget check. A large text file can freeze the TUI or create an enormous prompt, so stat and cap size before reading through a bounded helper.
  - confidence: 0.9
- **[P1] Wrap attached file text as untrusted content:** `src/tui/tui-command-handlers.ts:273`
  - body: Raw file text is concatenated with spoofable delimiters and a `[User message]` marker, bypassing the existing file-input wrapper that marks decoded file text as untrusted external content. Use the existing external-content boundary so file bytes stay data, not trusted prompt structure.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff introduces local-file-to-prompt ingestion without regular-file, size-limit, and untrusted-content guardrails.
