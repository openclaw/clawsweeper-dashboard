## Summary

Keep open: current main still has no TUI `/upload` command, and the PR remains useful feature work, but PR head has concrete blockers around stale TUI imports, pending upload lifecycle, and unbounded local file reads before it can be considered for maintainer product review.

## What This Changes

The PR adds a TUI `/upload` slash command, file-browser select-list, file-content prepending in command handling, theme wiring, and component tests.

## Best Possible Solution

Repair the branch against current TUI contracts, add regular-file and size-bounded upload guards, make pending file context retry-safe, and then have maintainers decide whether this local-file upload UX belongs in core.

## Reproduction Assessment

Yes for the PR blockers: PR-head source shows stale imports, retry-unsafe pending state, stale context after an empty replacement upload, and unbounded local reads. There is no current-main bug reproduction path because `/upload` is a new feature not present on main.

## Solution Assessment

No. The feature direction may be useful, but the proposed implementation needs current-main rebasing, bounded regular-file validation, and safer pending-context lifecycle before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the current pi-tui package:** `src/tui/components/file-browser-select-list.ts:10`
  - body: Current main depends on `@earendil-works/pi-tui`, but this new component imports `@mariozechner/pi-tui`. As written, the branch does not resolve against the current TUI dependency surface.
  - confidence: 0.93
- **[P1] Preserve pending uploads until send succeeds:** `src/tui/tui-command-handlers.ts:572`
  - body: `pendingFileContext` is cleared before `client.sendChat` resolves. If the send fails, retrying drops the selected file context and sends only the typed message, causing user-visible data loss.
  - confidence: 0.9
- **[P1] Clear stale uploads when nothing attaches:** `src/tui/tui-command-handlers.ts:278-282`
  - body: When a later `/upload` confirms but every selected path is skipped or unreadable, this branch leaves the previous `pendingFileContext` queued. The next message can silently include old files after a failed replacement upload.
  - confidence: 0.88
- **[P1] Restrict uploads to regular bounded files:** `src/tui/components/file-browser-select-list.ts:77-81`
  - body: Every non-directory entry is selectable, but the handler later calls `readFileSync` on selected paths. Device nodes, FIFOs, sockets, and very large files can block the TUI or consume unbounded memory before binary detection runs.
  - confidence: 0.87
