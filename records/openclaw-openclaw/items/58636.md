The branch adds a TUI `/upload` slash command, a file-browser overlay with multi-select/search, themed selector plumbing, and tests that prepend selected local file contents to the next chat message.

## Best Possible Solution

Forward-port the feature onto current main, use the current TUI package/backend contracts, preserve queued context until a send is accepted, clear stale context when no files attach, restrict reads to bounded regular files, and settle the local-cwd versus session-workspace boundary before merge.

- **[P1] Forward-port to the current Pi TUI package:** `src/tui/components/file-browser-select-list.ts:10`
  - body: This new component imports `@mariozechner/pi-tui`, but current main declares and imports `@earendil-works/pi-tui`. After resolving the dirty branch against current main, the added file needs the current package and backend contracts or the TUI build will fail.
  - confidence: 0.91
- **[P1] Keep queued upload context until send succeeds:** `src/tui/tui-command-handlers.ts:570-572`
  - body: `pendingFileContext` is nulled before `client.sendChat` runs. If the gateway call throws, the user's selected file context is dropped, so retrying sends only the typed message; clear it only after an accepted send or restore it on failure.
  - confidence: 0.9
- **[P1] Clear stale context when upload attaches nothing:** `src/tui/tui-command-handlers.ts:278-282`
  - body: When a replacement `/upload` confirms paths but every path is skipped or unreadable, this branch leaves the previous `pendingFileContext` queued. The next message can include older file contents after the user tried to replace them, so clear pending context before processing or when `blocks.length === 0`.
  - confidence: 0.88
- **[P1] Validate regular bounded files before reading:** `src/tui/components/file-browser-select-list.ts:77-81`
  - body: The browser treats every non-directory stat result as a selectable file, and the handler later calls `readFileSync` on selected paths. FIFOs, devices, sockets, or very large files can block or consume unbounded memory; only accept regular files and enforce a size limit before reading.
  - confidence: 0.92
