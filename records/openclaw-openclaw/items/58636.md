## Summary

Keep open: current main does not implement the TUI `/upload` feature, and the PR still has source-confirmed correctness and security blockers that should be fixed before maintainer product review.

## What This Changes

The PR adds a TUI `/upload` slash command, a file-browser overlay, selector/theme wiring, and component tests that attach selected local file contents to the next chat message.

## Best Possible Solution

Repair the PR branch, keep the feature TUI-only, add handler regression coverage, then have maintainers decide whether `/upload` is the right product surface.

## Reproduction Assessment

Yes for the review blockers: PR-head source shows the stale package import, premature context clearing, stale replacement context, and non-regular file selection path. The broader `/upload` behavior is a new feature, so there is no current-main bug reproduction.

## Solution Assessment

No. The feature direction may be useful, but this implementation should not merge until it imports the current TUI package, preserves queued files across failed sends, clears stale context on empty replacement uploads, and restricts reads to regular files.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Import pi-tui from the current package:** `src/tui/components/file-browser-select-list.ts:10`
  - body: Current main imports and depends on `@earendil-works/pi-tui`, but this new component imports `@mariozechner/pi-tui`. Since the old package is no longer present in `package.json` or the lockfile, the rebased PR will fail to resolve this module.
  - confidence: 0.93
- **[P1] Preserve pending uploads until send succeeds:** `src/tui/tui-command-handlers.ts:570-572`
  - body: `pendingFileContext` is cleared before `client.sendChat` resolves. If the gateway or provider rejects the send, the catch path reports the failure but the selected files are already gone, so retrying sends only the typed text.
  - confidence: 0.9
- **[P1] Clear stale upload context when replacement attaches nothing:** `src/tui/tui-command-handlers.ts:278-279`
  - body: A confirmed upload that produces no `blocks` leaves the previous `pendingFileContext` intact. If every newly selected path is skipped or fails to read, the next message silently includes older files instead of nothing.
  - confidence: 0.88
- **[P1] Filter the picker to regular files:** `src/tui/components/file-browser-select-list.ts:77-81`
  - body: Every non-directory entry is shown and selectable as a file, but the handler later calls `readFileSync` on the selected path. Device nodes, FIFOs, and sockets can block or consume memory before binary detection runs.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff introduces local file reads and currently allows non-regular filesystem entries to reach synchronous reads.
