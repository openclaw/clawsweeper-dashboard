## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Keep pending upload context until send succeeds:** `src/tui/tui-command-handlers.ts:572`
  - body: `pendingFileContext` is cleared before `client.sendChat` resolves, so a transient gateway or provider failure drops the selected file context and a retry sends only the typed message. Clear it only after a successful send, or restore it in the catch path.
  - confidence: 0.94
- **[P1] Clear stale upload context when no files attach:** `src/tui/tui-command-handlers.ts:278-282`
  - body: When a replacement upload is confirmed but every selected path is skipped or unreadable, `blocks.length` stays zero and the previous `pendingFileContext` remains queued. Clear pending context before processing or in the empty-result path so older file contents are not silently sent later.
  - confidence: 0.92
- **[P1] Validate file type and size before reading:** `src/tui/tui-command-handlers.ts:257`
  - body: The handler calls `readFileSync` before checking that the selected path is a regular bounded-size file, while the browser exposes every non-directory entry as selectable. A FIFO, device node, or huge file can hang or exhaust the TUI; validate with stat/lstat and a size cap before reading.
  - confidence: 0.91
- **[P1] Wrap attached file text as untrusted content:** `src/tui/tui-command-handlers.ts:273`
  - body: Current main wraps extracted file text with the external-content sentinel before model consumption, but this path concatenates raw local file text into the user message. Use the existing untrusted-content wrapper or an equivalent file-context helper so document instructions are treated as data, not first-party user instructions.
  - confidence: 0.88
