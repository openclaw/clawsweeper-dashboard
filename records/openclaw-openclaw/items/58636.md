## Summary

Keep open: current main still lacks the requested TUI upload feature, and the PR remains a useful feature proposal, but it is not merge-ready because it is stale against current TUI dependencies and still has pending-context and local-file-read safety blockers.

## What This Changes

This PR adds a TUI `/upload` slash command, file-browser select-list, theme/selectors wiring, file-context prepending in TUI command handling, and component tests.

## Best Possible Solution

Repair the branch against current TUI contracts with regular-file and size-bounded upload guards plus retry-safe pending context, then have maintainers decide whether `/upload` belongs in core TUI.

## Reproduction Assessment

Yes for the PR blockers: PR-head source shows the stale import, pending-context lifecycle bugs, and unbounded local reads. There is no current-main bug reproduction path because `/upload` is a new feature absent from main.

## Solution Assessment

No. The feature direction may be useful, but this implementation needs current-main rebasing, bounded regular-file validation, and safer pending-context lifecycle before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the current pi-tui package:** `src/tui/components/file-browser-select-list.ts:10`
  - body: Current main depends on `@earendil-works/pi-tui`, but this new component imports `@mariozechner/pi-tui`. After rebasing or merging into main, this import will not resolve against the current dependency surface.
  - confidence: 0.93
- **[P1] Keep pending uploads until send succeeds:** `src/tui/tui-command-handlers.ts:572`
  - body: `pendingFileContext` is cleared before `client.sendChat` resolves. If the send fails, retrying drops the selected file context and sends only the typed message, causing user-visible data loss.
  - confidence: 0.9
- **[P1] Clear stale uploads when replacement attaches nothing:** `src/tui/tui-command-handlers.ts:278-282`
  - body: When a later `/upload` confirms but every selected path is skipped or unreadable, the previous `pendingFileContext` remains queued. The next message can silently include old files after a failed replacement upload.
  - confidence: 0.88
- **[P1] Reject non-regular or oversized files before reading:** `src/tui/components/file-browser-select-list.ts:77-81`
  - body: Every non-directory entry is selectable, but the handler later calls `readFileSync` on selected paths. Device nodes, FIFOs, sockets, and very large files can block the TUI or consume unbounded memory before binary detection runs.
  - confidence: 0.87
