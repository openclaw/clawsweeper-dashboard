  - confidence: 0.95

  - body: `getSlashCommands` seeds `seen` from built-ins before appending gateway and dynamic command aliases, so adding `upload` here silently suppresses any plugin or gateway command with that alias. This needs an explicit collision or namespace plan instead of claiming the name by insertion order.
  - confidence: 0.92
- **[P1] Keep upload context queued until sendChat accepts the turn:** `src/tui/tui-command-handlers.ts:572`
  - body: `pendingFileContext` is cleared before `client.sendChat` returns. If the send fails or times out, retrying sends only the typed message and drops the files the user selected.
  - confidence: 0.93
- **[P1] Clear stale upload context when replacement attaches nothing:** `src/tui/tui-command-handlers.ts:278-282`
  - body: When every selected path is skipped or fails to read, `blocks.length` stays zero and the old queued context remains. Clear the pending upload before processing a replacement or explicitly clear it on an empty result.
  - confidence: 0.91
- **[P1] Validate regular bounded files before reading:** `src/tui/tui-command-handlers.ts:257`
  - body: The handler reads each selected path fully into memory before validating that it is a regular file or below a size limit. FIFOs, devices, sockets, or very large files can block or exhaust memory, so stat and cap paths before reading.
  - confidence: 0.91
- **[P1] Wrap attached file text as untrusted content:** `src/tui/tui-command-handlers.ts:273`
  - body: Raw file text is concatenated into the user message with spoofable delimiters, bypassing the existing external-content wrapper used for decoded file text. Use that boundary so local file bytes are treated as data, not trusted prompt structure.
  - confidence: 0.88

## Security Review

Status: needs_attention
