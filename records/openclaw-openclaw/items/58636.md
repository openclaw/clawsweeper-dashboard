## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Update the file browser to the current Pi TUI package:** `src/tui/components/file-browser-select-list.ts:10`
  - body: Current main depends on `@earendil-works/pi-tui`, but this new component imports `@mariozechner/pi-tui`. Rebase and update the import so the branch builds against the current dependency surface.
  - confidence: 0.96
- **[P1] Do not reserve /upload ahead of dynamic commands:** `src/tui/commands.ts:117`
  - body: `getSlashCommands` seeds `seen` from built-ins before appending gateway and dynamic commands, so adding `upload` here suppresses any plugin or gateway command with that alias. This needs an explicit namespace or collision plan instead of silently taking over `/upload`.
  - confidence: 0.91
- **[P1] Keep upload context queued until sendChat accepts the turn:** `src/tui/tui-command-handlers.ts:572`
  - body: `pendingFileContext` is cleared before `client.sendChat` has accepted the turn. A transient send failure drops the selected files, so retrying sends only the typed prompt instead of the attached context.
  - confidence: 0.94
- **[P1] Clear pending context when replacement upload attaches nothing:** `src/tui/tui-command-handlers.ts:278-282`
  - body: If every selected path is skipped or fails to read, `blocks.length` stays zero and the old `pendingFileContext` remains queued. Clear pending context before processing a replacement upload or explicitly clear it on an empty result.
  - confidence: 0.92
- **[P1] Only surface regular files for upload:** `src/tui/components/file-browser-select-list.ts:77-81`
  - body: The browser treats every non-directory entry as selectable, including FIFOs, devices, and sockets. Passing those paths to `readFileSync` can block or consume unbounded data, so only regular files should be selectable and the handler should revalidate before reading.
  - confidence: 0.92
- **[P1] Bound file reads before loading selected content:** `src/tui/tui-command-handlers.ts:257`
  - body: The handler reads the entire selected path into memory before any size or prompt-budget check. A large text file can freeze the TUI or create an enormous prompt, so stat and cap size before reading through a bounded helper.
  - confidence: 0.9
- **[P1] Wrap attached file text as untrusted external content:** `src/tui/tui-command-handlers.ts:273`
  - body: Raw file text is concatenated with spoofable delimiters and a `[User message]` marker, bypassing the existing file-input wrapper that marks decoded file text as untrusted external content. Use that wrapper so file bytes stay data, not trusted prompt structure.
  - confidence: 0.89
