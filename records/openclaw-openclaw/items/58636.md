  - confidence: 0.92

  - body: `pendingFileContext` is cleared before `client.sendChat` resolves. If the send fails, retrying drops the selected file context and sends only the typed message.
  - confidence: 0.91
- **[P1] Clear stale context when upload attaches no files:** `src/tui/tui-command-handlers.ts:278-282`
  - body: When a later `/upload` confirms but every selected path is skipped or unreadable, the previous `pendingFileContext` remains queued. The next message can silently include old files after a failed replacement upload.
  - confidence: 0.89
- **[P1] Wrap uploaded file text as untrusted external content:** `src/tui/tui-command-handlers.ts:271-273`
  - body: Current main wraps decoded file inputs with the external untrusted-content boundary, but this PR inserts raw selected file text directly into the user prompt. Use the existing wrapper/rendering path so attachments cannot masquerade as trusted instructions.
