Adding a built-in `upload` here hides any gateway/plugin `/upload` command and `handleCommand` will consume it locally, so this needs an explicit core-namespace decision or a non-shadowing design.
  - confidence: 0.9

  - body: `pendingFileContext` is cleared before `client.sendChat` returns. If the gateway or provider rejects the send, the user's selected file context is lost and a retry sends only the typed message.
  - confidence: 0.94
- **[P1] Clear stale context when replacement upload attaches nothing:** `src/tui/tui-command-handlers.ts:278`
  - body: When every selected path is skipped or unreadable, this branch leaves any previous `pendingFileContext` in place. A failed replacement upload can therefore send older file contents with the next message.
  - confidence: 0.92
- **[P1] Bound and validate file reads before loading contents:** `src/tui/tui-command-handlers.ts:257`
  - body: The browser treats every non-directory entry as a selectable file, and the handler then calls `readFileSync` with no regular-file or size guard. Selecting a FIFO, device, or huge file can block or exhaust the TUI process.
  - confidence: 0.92
- **[P1] Wrap attached file text as untrusted external content:** `src/tui/tui-command-handlers.ts:273`
  - body: OpenClaw already wraps decoded file and attachment text with external-content boundaries before model dispatch. This code appends selected local file text directly into the user prompt, bypassing those prompt-injection and special-token guardrails.
  - confidence: 0.91
