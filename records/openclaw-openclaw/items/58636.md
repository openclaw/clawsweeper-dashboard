## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Update the file browser to the current Pi TUI package:** `src/tui/components/file-browser-select-list.ts:10`
  - body: Current main depends on and imports `@earendil-works/pi-tui`, but this new component imports `@mariozechner/pi-tui`. After rebasing onto current dependencies, the new file-browser path will fail to resolve or typecheck unless it uses the current package.
  - confidence: 0.96
- **[P1] Do not reserve /upload ahead of dynamic commands:** `src/tui/commands.ts:117`
  - body: `getSlashCommands()` seeds `seen` from built-ins before appending gateway and dynamic commands, so adding built-in `upload` suppresses any plugin/gateway `/upload` alias. This needs an explicit maintainer namespace decision or a collision-safe design before core claims the command.
  - confidence: 0.9
- **[P1] Keep upload context queued until sendChat accepts the turn:** `src/tui/tui-command-handlers.ts:572`
  - body: `pendingFileContext` is cleared before `client.sendChat` resolves. A timeout or provider/gateway failure drops the selected files, so retrying the message sends only the typed text instead of the attached context.
  - confidence: 0.95
- **[P1] Clear stale upload context when replacement attaches nothing:** `src/tui/tui-command-handlers.ts:278-282`
  - body: If a user confirms a new upload but every selected path is skipped or unreadable, this branch leaves the previous `pendingFileContext` queued. The next message can silently include stale file contents after the user attempted to replace them.
  - confidence: 0.92
- **[P1] Only surface regular files for upload:** `src/tui/components/file-browser-select-list.ts:81`
  - body: The browser treats every non-directory entry as a selectable file, which includes FIFOs, sockets, device nodes, and other special paths. Those paths are later read synchronously, so the picker should surface only regular files or the handler must reject non-regular entries before reading.
  - confidence: 0.9
- **[P1] Bound file reads before loading selected content:** `src/tui/tui-command-handlers.ts:257`
  - body: `readFileSync(filePath)` loads the entire selected path before any size cap or safety check. Selecting a very large file or blocking special path can freeze the TUI or exhaust memory; the upload path needs a regular-file check and byte limit before reading.
  - confidence: 0.94
- **[P1] Wrap attached file text as untrusted external content:** `src/tui/tui-command-handlers.ts:273`
  - body: Selected file text is interpolated with spoofable `--- path ---` delimiters instead of OpenClaw's untrusted external-content wrapper. Local file bytes are still untrusted prompt data and can impersonate instructions or boundaries unless wrapped with the established marker/metadata helper.
  - confidence: 0.9
