## What This Changes

The PR adds a TUI `/upload` slash command, file-browser overlay, pending file-context prepending, theme wiring, and component tests.

## Best Possible Solution

Repair the branch against current TUI contracts, bound uploads to regular size-limited files, make pending file context retry-safe, and then let maintainers decide whether the local-file upload UX belongs in core.

Yes for the PR blockers: PR-head source shows stale imports, retry-unsafe pending state, stale context after an empty replacement upload, and unbounded file reads. There is no current-main bug reproduction path because `/upload` is a new feature.

## Solution Assessment

No. The feature direction may be useful, but the proposed implementation needs current-main rebasing, bounded regular-file validation, and safer pending-context lifecycle before it is the right core solution.

  - body: Every non-directory entry is selectable, but the handler later calls `readFileSync` on selected paths. Device nodes, FIFOs, sockets, and very large files can block the TUI or consume unbounded memory before binary detection runs.
  - confidence: 0.87
