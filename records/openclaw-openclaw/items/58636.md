## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep upload context until chat send succeeds:** `src/tui/tui-command-handlers.ts:572`
  - body: `pendingFileContext` is cleared while composing `messageToSend`, before `client.sendChat` has succeeded. A transient send failure then drops the user's selected files, so retrying sends only the typed prompt instead of the attached context; clear this state only after the send is accepted or keep it in the pending draft path.
  - confidence: 0.93
- **[P1] Clear stale upload context when no files attach:** `src/tui/tui-command-handlers.ts:278-282`
  - body: If a user already has pending file context and then confirms an upload where every selected path is skipped or fails to read, `blocks.length` stays zero and the old context remains queued. The next normal message can silently include the previous files, so clear pending context before processing a replacement upload or explicitly clear it when no blocks are attached.
  - confidence: 0.91
- **[P1] Validate regular files before reading selected paths:** `src/tui/components/file-browser-select-list.ts:77-81`
  - body: The browser classifies every non-directory entry as a selectable file, but FIFOs, sockets, devices, and other special entries are not safe to pass to `readFileSync`. Selecting one can hang or consume unbounded data in the TUI, so only surface `stat.isFile()` entries and revalidate that in the upload handler before reading.
  - confidence: 0.92
- **[P1] Enforce a size cap before reading file content:** `src/tui/tui-command-handlers.ts:257`
  - body: The handler reads the entire selected file into memory before any size or prompt-budget check. A large text file or special proc-style path can freeze the process or create an enormous prompt, so stat/limit first and read through a bounded helper aligned with the existing input-file limits.
  - confidence: 0.9
- **[P1] Wrap attached file text as untrusted content:** `src/tui/tui-command-handlers.ts:273`
  - body: Raw file contents are concatenated with spoofable `--- end ... ---` and `[User message]` markers, so file text can fake boundaries or inject model instructions as if they were user-authored structure. Reuse the existing sanitized file-context and external-content wrapper pattern so selected files stay clearly untrusted model input.
  - confidence: 0.88
