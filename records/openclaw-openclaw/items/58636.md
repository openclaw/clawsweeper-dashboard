  - body: When every selected path is skipped or fails to read, `blocks.length` stays zero and the old queued context remains. Clear the pending upload before processing a replacement or explicitly clear it on an empty result.
  - confidence: 0.91
- **[P1] Only expose regular files for selection:** `src/tui/components/file-browser-select-list.ts:77-81`
  - body: The browser treats every non-directory entry as a selectable file, including FIFOs, sockets, and devices. Since the handler later reads selected paths synchronously, filter to regular files and revalidate before reading.
  - confidence: 0.9
- **[P1] Bound selected file reads before loading them:** `src/tui/tui-command-handlers.ts:257`

  - confidence: 0.9
- **[P1] Wrap attached file text as untrusted content:** `src/tui/tui-command-handlers.ts:273`
  - body: Raw file text is concatenated into the user message with spoofable delimiters, bypassing the existing external-content wrapper used for decoded file text. Use that boundary so local file bytes are treated as data, not trusted prompt structure.
  - confidence: 0.88
- **[P1] Validate regular bounded files before reading:** `src/tui/components/file-browser-select-list.ts:77-81`
  - body: The browser treats every non-directory stat result as a selectable file, and the handler later calls `readFileSync` on selected paths. FIFOs, devices, sockets, or very large files can block or consume unbounded memory; only accept regular files and enforce a size limit before reading.
  - confidence: 0.92
