## Summary

Keep this PR open: current main still does not implement a TUI `/upload` command and no canonical duplicate was found, but the branch has blocking correctness and security issues plus a current merge conflict before it can be considered for merge.

## What This Changes

The PR adds a TUI `/upload` slash command, file-browser overlay, file-content prepending path, theme wiring, and component tests for selecting local files as prompt context.

## Best Possible Solution

Repair the branch against current TUI contracts, add bounded regular-file upload guards, make pending context retry-safe, and then have maintainers decide whether the core TUI should ship this local-file upload UX.

## Reproduction Assessment

Yes for the PR blockers: PR-head source shows the stale pi-tui import, retry-unsafe pending state, stale context after an empty replacement upload, and selectable non-regular file entries. There is no current-main bug reproduction path because `/upload` is a new feature absent from main.

## Solution Assessment

No. The feature direction may be useful, but this diff is not the narrow maintainable solution until it is rebased to the current TUI package, bounds local reads, and fixes pending-context lifecycle behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Import the current pi-tui package:** `src/tui/components/file-browser-select-list.ts:10`
  - body: Current main depends on `@earendil-works/pi-tui`, but this new component imports `@mariozechner/pi-tui`. As written, the branch does not resolve against the current TUI dependency surface.
  - confidence: 0.93
- **[P1] Keep pending uploads until send succeeds:** `src/tui/tui-command-handlers.ts:572`
  - body: `pendingFileContext` is cleared before the awaited `client.sendChat` call. If sending fails, retrying drops the selected file context and sends only the typed message.
  - confidence: 0.9
- **[P1] Clear stale uploads after empty confirmation:** `src/tui/tui-command-handlers.ts:278-282`
  - body: When a later `/upload` confirms but every selected path is skipped or unreadable, this branch leaves the previous `pendingFileContext` queued. The next message can silently include old files after a failed replacement upload.
  - confidence: 0.88
- **[P1] Reject special and oversized files before reading:** `src/tui/components/file-browser-select-list.ts:77-81`
  - body: Every non-directory entry is selectable, but the handler later calls `readFileSync` on selected paths. Device nodes, FIFOs, sockets, and very large files can block the TUI or consume unbounded memory before binary detection runs.
  - confidence: 0.87
