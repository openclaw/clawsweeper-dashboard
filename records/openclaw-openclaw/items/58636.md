## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Keep queued upload context until send succeeds:** `src/tui/tui-command-handlers.ts:572`
  - body: `pendingFileContext` is nulled before `client.sendChat` runs. If the gateway call throws or returns a terminal failure, the user's selected file context is dropped, so retrying the message silently sends only the typed text; clear the context only after an accepted send or restore it on failure.
  - confidence: 0.9
- **[P1] Clear stale context when nothing attaches:** `src/tui/tui-command-handlers.ts:278-282`
  - body: When a replacement `/upload` confirms paths but every path is skipped or unreadable, this branch leaves the previous `pendingFileContext` queued. The next message can then include older file contents after the user tried to replace them, so clear the pending context before processing or explicitly when `blocks.length === 0`.
  - confidence: 0.88
- **[P1] Validate regular files before upload reads:** `src/tui/components/file-browser-select-list.ts:77-81`
  - body: The browser treats every non-directory stat result as a selectable file, and the handler later calls `readFileSync` on the selected path. FIFOs, devices, sockets, or very large files can block or consume unbounded memory; only expose or accept regular files and enforce a size limit before reading.
  - confidence: 0.92
