Full review comments:

- **[P2] Treat all non-separator filename chars as path content:** `packages/terminal-core/src/display-string.ts:77`
  - body: The new boundary check only protects letters/digits plus `._~$-`, but POSIX filenames can contain characters such as `+`, `@`, and `:`. With an effective home like `/tmp/al`, a sibling path such as `/tmp/al+logs/project` would still be rendered as `$OPENCLAW_HOME+logs/project`, leaving the same misleading home-prefix rewrite this PR is meant to prevent. Please base the after-home check on path separators/end or a stricter path-token parser, and add regression cases for these valid sibling names.
