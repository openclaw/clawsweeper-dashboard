## Summary

Keep open. This is a current-release isolated cron startup failure: the exact error is the cron setup watchdog, and current main has diagnostics/tests for that boundary but no evidence that the memory-core runner-start stall itself is fixed or owned by a canonical open replacement issue.

## What This Changes

The issue reports that the memory-core managed isolated agentTurn cron job times out before runner startup on OpenClaw 2026.5.12 despite light context and healthy interactive gateway sessions.

## Best Possible Solution

Fix or instrument the pre-runner path so the managed memory-core isolated cron either reliably enters the embedded runner or reports the precise blocking phase before fallback exhaustion.

No high-confidence live reproduction was established in this read-only review. Source and regression tests do show the exact current-main path: an isolated agentTurn that never calls `onExecutionStarted` trips the 60s setup watchdog with the reported error.

## Solution Assessment
