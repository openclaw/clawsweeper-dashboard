## What This Changes

The PR adds a Discord-only `message_tool_only` delivery guard in `runReplyAgent` plus e2e coverage for `NO_REPLY`, empty-final, and skipped `message.send` paths.

## Best Possible Solution

Fix the guard so it selects the actual assistant final after operational and blocked-payload metadata is known, then provide real Discord/runtime proof and get maintainer acceptance of the fallback delivery contract.

## Reproduction Assessment

Yes at source level: dispatch suppresses unmarked `message_tool_only` replies, and the PR helper can stop on pre-marked notices or run before blocked metadata exists. I did not run tests because this review is read-only.

## Solution Assessment

No: the current patch is close in shape but marks at the wrong abstraction point for existing operational and hook-blocked payload ordering. The safer solution is to classify/select the assistant final after those metadata decisions are known.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Do not let notices consume the fallback marker:** `src/auto-reply/reply/agent-runner.ts:215-217`
  - body: `markMessageToolOnlyGuardPayloadsForSourceDelivery` treats any existing `deliverDespiteSourceReplySuppression` payload as if the substantive final has already been handled. Real fallback transition notices are already marked and are prepended before the model payloads, so a fallback run can deliver only the notice while leaving the actual assistant answer suppressed. Skip operational notices when deciding whether the assistant final has been marked, and add a regression for a fallback notice followed by a real final.
  - confidence: 0.9
- **[P2] Apply the guard after hook-blocked metadata is known:** `src/auto-reply/reply/agent-runner.ts:2227-2234`
  - body: The guard runs before `markBeforeAgentRunBlockedPayloads`, so the new `beforeAgentRunBlocked` check cannot catch real hook-blocked runs. That can add `deliverDespiteSourceReplySuppression` first and then add the blocked metadata, making dispatch deliver a payload that existing `message_tool_only` coverage expects to suppress. Move the guard after the hook-blocked marking or pass the hook-blocked state into the helper directly.
  - confidence: 0.86
