The supplied proof also does not show the latest source change working in a live Discord turn with quiet negative cases.

## What This Changes

The PR adds a Discord-specific `runReplyAgent` guard and e2e coverage that marks or synthesizes final reply payloads when a Discord turn lacks committed message-tool delivery evidence.

## Best Possible Solution

Keep strict `message_tool_only` privacy as the default until maintainers approve a narrower fallback; if approved, implement it with explicit policy, docs, focused tests, live Discord proof, and marking only the intended payload.
