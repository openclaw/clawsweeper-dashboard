## Summary

Keep open: the PR targets a real Discord silent-reply failure class, but the current patch breaks the existing message_tool_only privacy contract by publishing ordinary private finals and can still skip recovery after an unverified/cross-target message send.

## What This Changes

Adds a Discord runtime guard in runReplyAgent that marks substantive final payloads or synthesizes a guard/error payload when Discord turns finish without visible delivery, plus e2e tests.

## Best Possible Solution

Keep the Discord recovery guard, but make it emit only generic visible guard/error payloads and skip recovery only after verified current-source message-tool delivery.

## Reproduction Assessment

Yes, source-reproducible: current main defines message_tool_only finals as private and the PR tests exercise Discord turns that would otherwise end without visible delivery, although I did not run a live current-main reproduction.

## Solution Assessment

No; the guard belongs near the result/suppression boundary, but publishing private finals and accepting unverified target sends are not the best or safest shape.
