## Summary

Keep open: current main still has generic non-array assistant-content crash paths, and the two linked repair PRs for this report are closed unmerged.

## What This Changes

The issue asks OpenClaw to stop failing embedded agent runs with `500 v.content is not iterable` when a third-party provider returns malformed assistant `content`.

## Best Possible Solution

Normalize or guard malformed assistant content at the Responses conversion and replay/sanitization boundaries while preserving well-formed arrays and intentional silent empty assistant turns.

## Reproduction Assessment

Yes, from source inspection. Object or null assistant `content` can pass through replay normalization unchanged and later reach direct iteration, while Responses conversion can call `.some(...)` on non-array content.

## Solution Assessment

Yes. The narrow maintainable fix is provider-agnostic assistant-content normalization and `Array.isArray` guards, not a GptGod-specific workaround or a broad transcript migration.
