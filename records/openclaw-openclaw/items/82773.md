## Summary

Keep open: current main still matches the reported blocking path, and an open linked fix PR now owns the narrow implementation. The issue should close only after that PR, or an equivalent fix, lands.

## What This Changes

The issue asks for `usage.cost` and `sessions.usage` gateway request-path cache loads to refresh in the background instead of blocking with `sync-when-empty` on cold or stale caches.

## Best Possible Solution

Land the narrow refresh-mode fix after runtime proof, preserve the existing infra cache contract, and then close this issue as implemented by the merged PR.

## Reproduction Assessment

Yes at source level, but not through a live large-profile run here. Current main still passes `sync-when-empty` at both request-path callsites, and the cache helper awaits rebuild or direct scan work in that mode.

## Solution Assessment

Yes: switching only the two gateway request-path callers to `background` uses the existing helper contract and avoids broad cache churn. The open linked PR follows that shape, but should land with real runtime proof.
