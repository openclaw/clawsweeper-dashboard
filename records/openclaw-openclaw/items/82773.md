## What This Changes

The issue asks for `usage.cost` and `sessions.usage` gateway request paths to refresh usage caches in the background instead of blocking on `sync-when-empty`.

## Best Possible Solution

Review and land the linked focused PR after sufficient real behavior proof, then close this issue from the merged fix.

## Reproduction Assessment

Yes, source-reproducible: current main still passes `sync-when-empty` at both reported request-path call sites, and the helper awaits refresh or direct scans for that mode. I did not run a live multi-GB profile reproduction in this read-only review.

## Solution Assessment

Yes, the narrow fix is to switch only these gateway request-path callers to `background` and update the direct tests. The already-open linked PR follows that shape, but should wait for real runtime proof before merge.
