## Summary

Keep open: current main still routes both `usage.cost` and `sessions.usage` through `refreshMode: "sync-when-empty"`, and the infra loader still performs synchronous rebuild/load work for that mode. The reporter’s 2026.5.12 logs and workaround align with the current source path, so this looks like a real incomplete follow-up to the durable cache work.

## What This Changes

The issue asks OpenClaw to stop `usage.cost` and `sessions.usage` from synchronously rebuilding usage cache data on the gateway request path when the cache is empty or stale.

## Best Possible Solution

Change only the gateway request-path refresh mode for these usage endpoints to background refresh, keep the durable cache worker ownership in infra, and add focused regression coverage that proves cold or stale cache requests return promptly with cache status instead of awaiting a rebuild.

## Reproduction Assessment

We have a high-confidence source reproduction, but not a live large-profile run in this read-only pass. Current main still uses `sync-when-empty` at both reported gateway callsites, and the infra loader synchronously rebuilds or directly scans under that mode.

## Solution Assessment

Yes, the reporter’s proposed direction is the narrowest maintainable fix: move these request-path calls to background refresh and update the tests that currently lock in synchronous behavior. A broader cache architecture change is not needed for this specific regression.
