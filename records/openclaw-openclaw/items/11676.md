## Summary

Keep open: current main and the latest shipped release still describe a Node/OS-hosted OpenClaw runtime, while the existing plugin and agent-harness seams run inside that host rather than defining a portable agent-core protocol for MimiClaw-style embedded/no-Node ports.

## What This Changes

The issue requests runtime-agnostic support for running OpenClaw-style agents on devices without Linux or Node.js, including embedded and microcontroller-class targets.

## Best Possible Solution

Define a maintainer-approved runtime-agnostic agent-core protocol with schemas for tool calls, memory/session state, streaming/events, auth/model selection, adapter boundaries, security guidance, conformance tests, and a small non-Node reference path.

## Reproduction Assessment

Not applicable as a feature/API portability request. Static inspection is high-confidence because package metadata, launcher code, runtime guard/tests, install/platform docs, shipped release files, and harness APIs still describe a Node/OS-hosted OpenClaw runtime.

## Solution Assessment

Unclear as a concrete implementation. The requested runtime-neutral protocol is plausible, but a plugin-only or ClawHub-only route is not sufficient today because current plugin and harness APIs still execute inside the Node-hosted OpenClaw runtime.
