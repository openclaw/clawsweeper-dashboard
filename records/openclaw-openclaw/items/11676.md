## Summary

Keep open: current main and v2026.6.8 still require a Node-hosted OS runtime, and the available agent-core plus harness surfaces are in-process OpenClaw/plugin APIs rather than the requested runtime-neutral embedded/no-Node protocol. The linked browser/WASM item is already closed as a duplicate pointing back here, so this remains the canonical design discussion.

## What This Changes

The issue asks OpenClaw to support bare-metal, embedded, C/Rust/WASM, or other no-Linux/no-Node runtimes using MimiClaw as prior art.

## Best Possible Solution

Keep this as a maintainer design discussion for a runtime-neutral agent-core contract with schemas, adapter boundaries, conformance proof, and possibly a small reference port.
