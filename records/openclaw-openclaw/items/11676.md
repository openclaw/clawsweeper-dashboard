## Summary

Current main has useful TypeScript agent-core extraction work, but OpenClaw still requires Node.js and does not provide the stable cross-language protocol or conformance boundary needed for bare-metal, C, Rust, WASM, or ESP32-class implementations. The closed browser/WASM request is narrower, and the existing discussion explicitly preserves this broader product question for maintainer judgment.

## What This Changes

Request a runtime-agnostic OpenClaw agent core or protocol that supports bare-metal, microcontroller, C/Rust, or WASM implementations without Node.js or a conventional OS.

## Best Possible Solution

Keep this as the product-level portability tracker, treat `packages/agent-core` as groundwork, and decide whether to define a minimal versioned cross-language protocol with conformance tests before attempting any embedded reference implementation.

## Reproduction Assessment

Not applicable; this proposes a new architecture capability rather than reporting broken existing behavior. Source inspection does confirm that current main still lacks the requested non-Node runtime contract.

## Solution Assessment

The protocol-first direction is the narrowest maintainable approach, but the permanent API, supported semantics, versioning, and target runtimes require explicit maintainer product judgment.
