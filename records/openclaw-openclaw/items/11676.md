## Summary

Keep open: current main and v2026.6.10 still ship a Node-hosted OpenClaw runtime, while the current agent-core and harness seams are useful partial groundwork rather than the requested runtime-neutral, no-Node embedded protocol. MimiClaw is relevant prior art, but the remaining decision is core runtime/API direction, not conservative cleanup.

## What This Changes

The issue requests support for running OpenClaw-style agent loops on no-Linux/no-Node embedded or low-resource devices, with MimiClaw as a C/ESP32 reference implementation.

## Best Possible Solution

Keep this as a maintainer design discussion for a runtime-neutral agent-core contract with schemas, platform-adapter boundaries, conformance tests, and possibly a small reference port before deciding what belongs in core, plugin APIs, docs, ClawHub, or an external project.

## Solution Assessment
