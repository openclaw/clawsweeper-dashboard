## Summary

The request remains necessary: current main has useful internal agent-core modularization, but OpenClaw still ships as a Node application and does not define a supported language-neutral contract for embedded implementations.

## What This Changes

Add a runtime-agnostic OpenClaw agent-core protocol that third-party C, Rust, WASM, and bare-metal implementations can support.

## Best Possible Solution

If maintainers approve this direction, define a small versioned serialized agent-core protocol with platform-neutral schemas and conformance tests, while leaving embedded implementations such as MimiClaw in their own repositories.

## Reproduction Assessment

Not applicable. This requests a new supported runtime and protocol capability; current-source inspection shows that a public non-Node compatibility contract is not presently defined.

## Solution Assessment

No, directly rewriting or decoupling the complete core for several runtimes is not yet a bounded best solution; a deliberately versioned protocol and conformance boundary would be narrower and more maintainable if approved.
