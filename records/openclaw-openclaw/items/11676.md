## Summary

Keep open: current main and latest release still require a Node-hosted OS runtime, while the agent-core and harness work remains TypeScript/OpenClaw-process groundwork rather than the requested runtime-neutral embedded/no-Node protocol. MimiClaw is useful external prior art, and the narrower browser/WASM issue is already closed into this broader discussion.

## What This Changes

The issue requests runtime-agnostic support for running OpenClaw on no-Linux/no-Node embedded or ultra-low-resource devices, using MimiClaw as prior art.

## Best Possible Solution

Keep this as a maintainer design discussion for a runtime-neutral agent-core contract: schemas for tool calls, memory/session state, streaming/events, auth/model selection, platform adapters, conformance tests, and possibly a small reference port that proves an OpenClaw-compatible loop can run outside the Node-hosted Gateway.

Not applicable. This is a runtime/platform feature request, and source/docs inspection confirms there is no supported no-Node runtime path on current main to reproduce as broken existing behavior.

## Solution Assessment

Unclear. The current TypeScript agent-core extraction is useful groundwork, but maintainers still need to decide whether OpenClaw should own a stable runtime-neutral protocol, extend plugin APIs, publish reference docs, or decline the core scope.
