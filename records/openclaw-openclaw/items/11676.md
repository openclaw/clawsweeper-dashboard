## Summary

Keep open: current main still ships a Node-hosted CLI/Gateway with in-process TypeScript plugin and agent-harness seams, so the no-Linux/no-Node embedded proposal remains a broad runtime/API product decision rather than implemented, duplicate, or ClawHub-only.

## What This Changes

The issue requests runtime-agnostic OpenClaw support for embedded or ultra-low-resource devices without Linux or Node.js, using MimiClaw as prior art.

## Best Possible Solution

Define a maintainer-approved runtime-agnostic agent-core protocol with schemas, adapter boundaries, security guidance, conformance tests, and a small non-Node reference path before deciding what belongs in core, SDK docs, or external ports.

## Reproduction Assessment

## Solution Assessment
