## Summary

Keep open. Current main remains a Node-hosted TypeScript/Gateway runtime, the available agent-harness surface is still an in-process experimental plugin seam, and the existing discussion treats MimiClaw as useful prior art rather than a reason to close.

## What This Changes

Request support for a runtime-agnostic OpenClaw agent core/protocol that can run on no-Linux/no-Node embedded or microcontroller environments, with MimiClaw as prior art.

## Best Possible Solution

Define a stable runtime-agnostic agent-core protocol and conformance boundary before deciding whether embedded ports belong in core docs, SDK contracts, or external implementations.

## Reproduction Assessment

Not applicable as a feature/API request. Static current-main verification is high-confidence: package metadata, launcher, runtime guard/tests, install docs, and harness docs still require or mirror the Node-hosted Gateway runtime.

## Solution Assessment

Unclear. A runtime-agnostic protocol is a plausible direction, but the issue is too broad for an automatic fix PR and the current plugin harness is not sufficient because it remains experimental and in-process.
