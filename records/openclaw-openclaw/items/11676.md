## Summary

Keep open. Current main is still a Node-hosted CLI/Gateway with experimental in-process harness/plugin seams, so the no-Linux/no-Node embedded runtime request remains a broad runtime/API product design item rather than implemented, duplicate, or ClawHub-only work.

## What This Changes

Request support for running an OpenClaw-style agent loop on bare-metal, embedded, or ultra-low-resource devices without Linux or Node.js, using MimiClaw as prior art.

## Best Possible Solution

Define a stable runtime-agnostic agent-core protocol with schemas, conformance tests, and clear platform-adapter boundaries before deciding whether embedded reference ports live in core docs/SDK contracts or outside implementations.

## Reproduction Assessment

Not applicable as a feature/API request. Static current-main verification is high-confidence: package metadata, launcher, runtime guard/tests, install docs, and harness docs still require or mirror a Node-hosted OpenClaw runtime.

## Solution Assessment

Unclear. A runtime-agnostic agent-core protocol is a plausible direction, but the narrowest maintainable solution needs maintainer-owned product/API/security design rather than a small code repair or a ClawHub-only plugin.
