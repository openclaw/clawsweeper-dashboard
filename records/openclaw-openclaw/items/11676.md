## Summary

Keep open. Current main still requires a Node-hosted OpenClaw runtime and the agent-harness surface is an in-process experimental plugin seam, so the embedded/no-Node request remains a broad product/API design item rather than implemented, duplicate, or ClawHub-only work.

## What This Changes

Request runtime-agnostic OpenClaw support for bare-metal, embedded, WASM/C/Rust, or other no-Node devices, with MimiClaw linked as prior art.

## Best Possible Solution

Define a stable runtime-agnostic agent-core protocol and conformance boundary before deciding whether embedded ports belong in core docs, SDK contracts, or external implementations.

## Reproduction Assessment

Not applicable as a feature/API request. Static current-main verification is high-confidence: package metadata, launcher, runtime guard/tests, install docs, and harness docs still require or mirror the Node-hosted Gateway runtime.

## Solution Assessment

Unclear. A runtime-agnostic protocol is a plausible direction, but the best maintainable solution needs an explicit design and conformance boundary; the current plugin harness is not enough because it remains experimental and in-process.
