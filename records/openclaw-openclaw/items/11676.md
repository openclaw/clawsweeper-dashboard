## Summary

Keep open. Current main is still a Node-hosted CLI/Gateway with in-process plugin and harness seams, so the embedded/no-Node request remains a broad runtime/API design request rather than implemented, duplicated, or ClawHub-only work.

## What This Changes

The issue requests a runtime-agnostic OpenClaw agent/core interface that could run on embedded or ultra-low-resource devices without Linux or Node.js.

## Best Possible Solution

Define a stable runtime-agnostic agent-core protocol with schemas, platform-adapter boundaries, security guidance, conformance tests, and a small reference path before deciding whether embedded ports live in core docs/SDK contracts or external projects.

## Reproduction Assessment

Not applicable as a feature/API request. Static current-main verification is high-confidence: package metadata, launcher, runtime guard/tests, install docs, protocol docs, and harness/plugin APIs still describe a Node-hosted OpenClaw runtime.

## Solution Assessment

Unclear as a specific implementation. A runtime-agnostic agent-core protocol is a plausible direction, but the narrowest maintainable path needs maintainer-owned product, API, and security design rather than a small repair or ClawHub-only plugin.
