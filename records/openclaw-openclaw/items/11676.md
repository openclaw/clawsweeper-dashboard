## Summary

Keep open. Current main still requires a Node-hosted OpenClaw CLI/Gateway and only exposes in-process or local-process agent runtime seams, so the embedded/no-Node request remains a broad runtime/API design item; the existing maintainer discussion also keeps MimiClaw as prior art and says closed #45496 is narrower browser/WASM work.

## What This Changes

Request support for running OpenClaw-style agent loops on devices without Linux or Node.js, including embedded, microcontroller, C/Rust/WASM, or bare-metal targets.

## Best Possible Solution

Define a stable runtime-agnostic agent-core protocol with conformance tests before deciding whether embedded ports belong in core docs, SDK contracts, or external implementations.

## Reproduction Assessment

Not applicable as a feature/API request. Static current-main verification is high-confidence: package metadata, launcher, runtime guard/tests, install docs, and harness/runtime docs still require or mirror the Node-hosted runtime.

## Solution Assessment

Unclear. A runtime-agnostic protocol is a plausible direction, but the narrowest maintainable solution needs explicit maintainer design and conformance boundaries; the current in-process plugin harness is not sufficient.
