## Summary

Keep open: current main and v2026.6.8 still require a Node-hosted OpenClaw runtime, and the new private TypeScript `agent-core` package is useful partial refactoring but not the requested stable no-Node/bare-metal agent-core protocol.

## What This Changes

The issue requests runtime-agnostic OpenClaw support for devices that cannot run Linux or Node.js, using MimiClaw as embedded prior art.

## Best Possible Solution

Keep this as a maintainer design discussion for a runtime-neutral agent-core contract with schemas, adapter boundaries, conformance proof, and possibly a small reference port.

## Reproduction Assessment

Not applicable. This is a feature and architecture request, and source/docs inspection confirms there is no supported no-Node runtime path to reproduce as broken current behavior.

## Solution Assessment

Unclear. The direction is plausible, but maintainers need to decide whether OpenClaw should own a runtime-neutral protocol, extend plugin APIs, publish reference docs, or decline core work.
