## Summary

Keep open: current main and the latest shipped release still require a Node/OS-hosted OpenClaw runtime, and the existing plugin plus agent-harness seams are in-process extension points rather than a portable agent-core protocol for MimiClaw-style embedded or no-Node runtimes.

## What This Changes

The issue requests runtime-agnostic OpenClaw support for no-Linux/no-Node targets such as embedded devices and microcontrollers, with MimiClaw linked as prior art.

## Best Possible Solution

Define a maintainer-approved runtime-agnostic agent-core protocol with schemas for tool calls, memory/session state, streaming/events, auth/model selection, adapter boundaries, conformance tests, and a small non-Node reference path.

## Reproduction Assessment

Not applicable as a feature/API portability request. Static inspection is high-confidence because current main and v2026.5.19 still describe a Node/OS-hosted OpenClaw runtime rather than a no-Node embedded agent-core protocol.

## Solution Assessment

Unclear as a concrete implementation because no patch is proposed. A ClawHub/plugin-only route is not sufficient today because current plugin and harness APIs execute inside the Node-hosted OpenClaw runtime, so this needs maintainer-approved core protocol/API direction.
