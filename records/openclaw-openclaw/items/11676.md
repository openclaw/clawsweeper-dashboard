## Summary

Keep open. Current main is still a Node-hosted CLI/Gateway with in-process or local-process agent runtime seams, so the no-Linux/no-Node embedded runtime request remains a broad product/API design item rather than implemented, duplicate, or ClawHub-only work.

## What This Changes

Request support for running OpenClaw-style agent loops on no-Linux/no-Node embedded or lightweight runtimes, using MimiClaw as prior art.

## Best Possible Solution

Define a stable runtime-agnostic agent-core protocol with schemas and conformance tests, then decide whether embedded reference ports belong in core docs/SDK contracts or external implementations.

## Reproduction Assessment

Not applicable as a feature/API request. Static current-main verification is high-confidence: package metadata, launcher, runtime guard/tests, install docs, and harness docs still require or mirror a Node-hosted OpenClaw runtime.

## Solution Assessment

Unclear. A runtime-agnostic agent-core protocol is a plausible direction, but the narrowest maintainable solution needs maintainer-owned design and conformance boundaries; the current plugin/harness API is not sufficient by itself.
