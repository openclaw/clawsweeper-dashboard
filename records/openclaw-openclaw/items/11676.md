## Summary

Keep open: current main and the latest release are still Node-hosted, OS-runtime OpenClaw builds, and the existing agent-core/harness surfaces are in-process TypeScript/plugin contracts rather than the requested no-Node, runtime-neutral embedded protocol. The closed browser/WASM issue is a duplicate pointing back here, not a canonical replacement.

## What This Changes

The issue requests support for running OpenClaw-style agent logic on no-Linux/no-Node embedded targets, including a runtime-agnostic protocol, adapter boundaries, and possible reference implementations inspired by MimiClaw.

## Best Possible Solution

Keep this as a maintainer design discussion for a runtime-neutral agent-core contract with schemas, platform-adapter boundaries, conformance tests, and possibly a small reference port before deciding whether embedded/no-Node support belongs in core, plugin APIs, docs, or outside scope.

## Reproduction Assessment

Not applicable. This is a feature and architecture request, and source/docs inspection confirms there is no supported no-Node runtime path to reproduce as broken current behavior.

## Solution Assessment

Unclear. The current agent-core package and harness APIs are useful groundwork, but maintainers still need to decide whether OpenClaw should own a stable runtime-neutral protocol, extend plugin APIs, publish reference docs, or decline the core scope.
