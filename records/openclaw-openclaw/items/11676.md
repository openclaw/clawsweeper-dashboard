## Summary

Keep open: current main still requires a Node/OS-hosted OpenClaw runtime, and the existing plugin/harness seams run inside that host rather than defining a portable agent-core protocol for MimiClaw-style embedded/no-Node ports.

## What This Changes

The issue requests runtime-agnostic OpenClaw support for bare-metal, embedded, C/C++/Rust/WASM, or other non-Node environments.

## Best Possible Solution

Define a maintainer-approved runtime-agnostic agent-core protocol with schemas for tool calls, memory/session state, streaming/events, auth/model selection, adapter boundaries, conformance tests, and a small non-Node reference path.

## Reproduction Assessment

Not applicable as a feature/API portability request. Static inspection is high-confidence because package metadata, launcher code, runtime guard/tests, install/platform docs, and harness APIs still describe a Node/OS-hosted OpenClaw runtime.

## Solution Assessment

Unclear as a concrete implementation. A ClawHub/plugin-only route is not sufficient today because current plugin and harness APIs execute inside the Node-hosted OpenClaw runtime; this needs maintainer-approved core protocol/API direction.
