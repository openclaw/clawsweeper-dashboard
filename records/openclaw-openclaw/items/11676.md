## Summary

Keep open: current main still requires a Node/OS-hosted OpenClaw gateway, and the existing plugin plus agent-harness seams are in-process extension points rather than a portable agent-core protocol for MimiClaw-style embedded or no-Node runtimes.

## What This Changes

The issue requests support for running OpenClaw on non-Linux, non-Node runtimes such as embedded devices, microcontrollers, C/Rust/WASM ports, and bare-metal-style environments.

## Best Possible Solution

Define a maintainer-approved runtime-agnostic agent-core protocol with schemas for tool calls, memory/session state, streaming/events, auth/model selection, adapter boundaries, conformance tests, and a small non-Node reference path.

## Reproduction Assessment

Not applicable as a feature/API portability request. Static inspection is high-confidence because package metadata, launcher code, runtime guard/tests, install/platform docs, gateway docs, and harness APIs still describe a Node/OS-hosted OpenClaw runtime.

## Solution Assessment

Unclear as a concrete implementation. A ClawHub/plugin-only route is not sufficient today because current plugin and harness APIs execute inside the Node-hosted OpenClaw runtime, so this needs maintainer-approved core protocol/API direction.
