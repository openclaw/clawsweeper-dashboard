## Summary

Keep this issue open. Current main and the latest release still require a Node-hosted OpenClaw runtime, while the requested no-Node/bare-metal portability work is a broad product and protocol decision rather than a cleanup item that can be closed.

## What This Changes

The issue requests runtime-agnostic OpenClaw support for embedded, bare-metal, C/Rust/WASM, or other no-Node/no-Linux devices, using MimiClaw as prior art.

## Best Possible Solution

Keep this as the canonical embedded/no-Node portability discussion; if maintainers accept the direction, design a small versioned serialized protocol with conformance fixtures and an external reference adapter before attempting a broad core decoupling.

## Reproduction Assessment

Not applicable. This is a new portability and interoperability capability request, and source/docs checks show current OpenClaw intentionally runs on Node-hosted platforms.

## Solution Assessment

No, not as a wholesale decoupling of the existing core from Node. The maintainable path is a maintainer-approved protocol boundary with security semantics and conformance tests before any embedded or no-Node reference runtime.
