## Summary

Keep open: current main and the latest shipped release still require an OS-hosted Node CLI/Gateway, and the existing plugin/harness seams are in-process TypeScript contracts rather than a runtime-agnostic agent-core protocol for MimiClaw-style embedded/no-Node ports. Prior review comments also kept this open, while the narrower browser/WASM request at https://github.com/openclaw/openclaw/issues/45496 was closed as a duplicate of this broader item.

## What This Changes

Request to support OpenClaw-style agent loops on devices without Linux or Node.js by defining runtime-agnostic core interfaces, adapter boundaries, and possible lightweight reference implementations.

## Best Possible Solution

Define a maintainer-approved runtime-agnostic agent-core protocol with schemas, adapter boundaries, security guidance, conformance tests, and a small non-Node reference path before deciding what belongs in core, SDK docs, or external ports.

## Reproduction Assessment

Not applicable as a feature/API request. Static inspection is high-confidence because package metadata, launcher code, runtime guard/tests, install docs, shipped release files, and harness APIs still describe a Node/OS-hosted OpenClaw runtime.

## Solution Assessment

Unclear as a concrete implementation. The requested runtime-neutral protocol is plausible, but the maintainable path needs maintainer product, API, and security design before a focused code change or plugin-only route.
