## Summary

Keep open. Current main still ships and documents a Node-hosted OpenClaw CLI/Gateway with in-process runtime and plugin seams, while this issue asks for a broader no-Node embedded/runtime-agnostic agent protocol; prior review comments also kept this exact scope open and treated MimiClaw as useful prior art.

## What This Changes

The issue requests runtime-agnostic or embedded support so OpenClaw can run without Linux or Node.js on constrained devices.

## Best Possible Solution

Define a maintainer-approved runtime-agnostic agent-core protocol with schemas, adapter boundaries, security guidance, conformance tests, and a small reference path before deciding whether embedded ports live in core docs/SDK contracts or external projects.

## Reproduction Assessment

Not applicable as a feature/API request. Static current-main verification is high-confidence: package metadata, launcher, runtime guard/tests, install docs, and harness/plugin APIs still describe a Node-hosted OpenClaw runtime.

## Solution Assessment

Unclear as a specific implementation. A runtime-agnostic agent-core protocol is a plausible direction, but the narrowest maintainable path needs maintainer-owned product, API, and security design rather than a small repair or ClawHub-only plugin.
