## Summary

Keep open. Current main still ships a Node-hosted OpenClaw CLI/Gateway with Node runtime guards, OS-hosted install docs, and in-process plugin/agent-harness seams; the requested no-Node embedded/runtime-agnostic agent protocol remains a broad maintainer product/API/security design question, and prior review comments already kept this exact MimiClaw scope open.

## What This Changes

Request support for a runtime-agnostic OpenClaw agent loop that can run on embedded or ultra-low-resource devices without Linux or Node.js, using MimiClaw as prior art.

## Best Possible Solution

Define a maintainer-approved runtime-agnostic agent-core protocol with schemas, adapter boundaries, security guidance, conformance tests, and at least one small reference path before deciding which embedded ports belong in core docs, SDK contracts, or external projects.

## Solution Assessment
