## Summary

Keep open: current main and the latest release still require a Node-hosted OS runtime, and the existing TypeScript agent-core and harness surfaces are useful groundwork but not the requested runtime-neutral embedded/no-Node protocol.

## What This Changes

Requested support for OpenClaw-style agent execution on devices without Linux or Node.js, including embedded, microcontroller, C/Rust/WASM, or bare-metal runtimes.

## Best Possible Solution

Keep this as a maintainer design discussion for a runtime-neutral agent-core contract with schemas, adapter boundaries, conformance proof, and possibly a small reference port.

## Reproduction Assessment

Not applicable. This is a feature and architecture request, and source/docs inspection confirms there is no supported no-Node runtime path to reproduce as broken current behavior.

## Solution Assessment

Unclear. The existing agent-core package is partial groundwork, but maintainers still need to decide whether OpenClaw should own a stable runtime-neutral protocol, extend plugin APIs, publish reference docs, or decline core scope.
