## Summary

Keep this issue open as the canonical product/API discussion: current main still requires Node.js, its plugin and agent-harness seams remain host-process APIs rather than a portable serialized protocol, and the narrower browser/WASM request was closed in favor of this broader item.

## What This Changes

Define a runtime-agnostic OpenClaw agent interface and platform adapters so third-party C, Rust, WASM, and embedded implementations can run without Node.js or a conventional OS.

## Best Possible Solution

Retain one canonical design discussion and, if maintainers prioritize it, begin with a narrow versioned serialized protocol, explicit capability negotiation, and conformance tests plus one independently maintained reference port rather than attempting to decouple or duplicate the full Node runtime at once.

## Reproduction Assessment

Not applicable; this issue proposes a new supported runtime architecture rather than reporting broken documented behavior. Current source gives high-confidence evidence that no supported non-Node execution path exists today.

## Solution Assessment

No, wholesale decoupling of the current agent core is not yet a sufficiently bounded solution. A deliberately versioned portable protocol with capability negotiation and conformance coverage is the narrower maintainable direction if the product decision is affirmative.
