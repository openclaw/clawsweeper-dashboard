## Summary

Keep open. Current main remains an OS-hosted Node/TypeScript Gateway runtime, the agent-harness surface is still an in-process experimental plugin seam, and the existing maintainer/Codex discussion already treats MimiClaw as useful prior art rather than a reason to close.

## What This Changes

Request support for OpenClaw-style agent loops on no-Linux/no-Node embedded or ultra-low-resource devices through runtime-agnostic agent-core/protocol seams.

## Best Possible Solution

Define a stable runtime-agnostic agent-core protocol and conformance boundary before deciding whether embedded ports belong in core docs, SDK contracts, or external implementations.

## Reproduction Assessment

Not applicable as a feature/API request. Static current-main verification is high-confidence: package metadata, launcher, runtime guard/tests, install docs, and agent-harness types still require or mirror the Node-hosted Gateway runtime.

## Solution Assessment

Unclear. A runtime-agnostic protocol is a plausible direction, but the issue is too broad for an automatic fix PR and the current plugin harness is not sufficient because it remains experimental and in-process.
