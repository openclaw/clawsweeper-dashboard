## Summary

Keep this open: current main and the latest release still require a Node-hosted OpenClaw runtime, and the existing browser, Gateway, plugin, and agent-harness surfaces do not provide the requested runtime-agnostic agent-core protocol for bare-metal, C/Rust/WASM, or ESP32-class ports.

## What This Changes

The issue requests runtime-agnostic OpenClaw support for devices without Linux or Node.js, using MimiClaw as embedded prior art.

## Best Possible Solution

Keep this as a maintainer product/API design discussion for a runtime-neutral agent-core contract covering tool calls, memory/session state, streaming/events, auth/model selection, platform adapters, and conformance proof outside the Node-hosted Gateway.
