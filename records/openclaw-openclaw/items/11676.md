## Summary

Keep open. Current main still describes and enforces a Node-hosted CLI/Gateway with in-process plugin and harness seams, while this issue asks for a broader no-Node embedded/runtime-agnostic agent protocol; the linked browser/WASM thread was closed into this issue, not the other way around.

## What This Changes

The issue requests a runtime-agnostic OpenClaw core or protocol so embedded/no-Node devices can run an OpenClaw-style agent loop.

## Best Possible Solution

Define a stable runtime-agnostic agent-core protocol with schemas, adapter boundaries, security guidance, conformance tests, and a small reference path before deciding whether embedded ports live in core docs/SDK contracts or external projects.

## Reproduction Assessment

Not applicable as a feature/API request. Static current-main verification is high-confidence: package metadata, launcher, runtime guard/tests, install docs, and harness/plugin APIs still describe a Node-hosted OpenClaw runtime.

## Solution Assessment

Unclear as a specific implementation. A runtime-agnostic agent-core protocol is a plausible direction, but the narrowest maintainable path needs maintainer-owned product, API, and security design rather than a small repair or ClawHub-only plugin.
