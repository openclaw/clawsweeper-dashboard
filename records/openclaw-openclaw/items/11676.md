## Summary

Keep open. Current main still requires a Node-hosted OpenClaw CLI/Gateway and the existing plugin/agent-harness seams are in-process TypeScript contracts, so the embedded/no-OS/no-Node request remains a broad runtime/API/security design question rather than something already implemented, ClawHub-only, or superseded.

## What This Changes

Request runtime-agnostic OpenClaw support for embedded or no-Node devices, using MimiClaw as prior art.

## Best Possible Solution

Define a maintainer-approved runtime-agnostic agent-core protocol with schemas, adapter boundaries, security guidance, conformance tests, and a small non-Node reference path before deciding what belongs in core, SDK docs, or external ports.

## Reproduction Assessment

## Solution Assessment
