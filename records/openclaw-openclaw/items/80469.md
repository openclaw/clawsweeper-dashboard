## Summary

Keep open: current main still has a shared `/btw` path that resolves model/auth and then streams directly, while normal OpenAI model runs are expected to use the Codex harness/transport.

## What This Changes

The issue asks for `/btw` to use the active Codex/OpenAI transport for Codex OAuth sessions rather than a plain OpenAI Responses call.

## Best Possible Solution

Update the shared `/btw` runner to honor the active session’s selected harness/auth transport for Codex-backed OpenAI models while preserving the tool-less, non-persistent side-question behavior.
