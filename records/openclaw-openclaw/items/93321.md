## What This Changes

The issue reports compaction retaining assistant tool_use/toolCall blocks without matching tool_result/toolResult after a timeout, causing strict-provider replay failures on later turns.

## Best Possible Solution

Land one narrow compaction repair that drops or repairs aborted orphaned tool calls before summarization/replay, with regression coverage for no-drop and dropped-chunk paths, then close the duplicate candidate PR.

## Reproduction Assessment

Yes, as a source-level reproduction: current main can preserve an aborted assistant toolCall without a matching toolResult through the compaction-safeguard no-drop path. I did not run a live Bedrock replay in this read-only review.

## Solution Assessment

Yes, the active PR direction is the best narrow fix: reuse the shared transcript repair contract at the compaction planning/safeguard boundary and set the aborted-turn drop policy where invalid retained history is prepared. Provider-adapter stripping would be later and less maintainable because the invalid transcript should not reach summarization or replay.
