## Summary

The reported Feishu behavior matches OpenClaw's documented group/channel visible-reply policy: the group turn is processed, but normal final replies are suppressed by default unless the model uses the message tool or the deployment opts into automatic group replies.

## What This Changes

Request that Feishu group messages automatically deliver the agent's final response when group policy and mention settings allow the turn.

## Best Possible Solution

Keep the safe group/channel default and resolve this deployment by setting messages.groupChat.visibleReplies to "automatic" or using message-tool sends for visible group output.

## Reproduction Assessment

Yes for the observed replies=0 path: source inspection shows group turns without messages.groupChat.visibleReplies="automatic" use message_tool_only delivery, so the agent can complete while normal final replies remain unqueued. This is expected configuration behavior rather than a Feishu send-path failure.

## Solution Assessment

Yes: the narrowest solution is an existing config change, messages.groupChat.visibleReplies="automatic", or a tool-calling model/runtime that sends via message(action=send). Changing Feishu delivery code would bypass the current cross-channel group safety policy.
