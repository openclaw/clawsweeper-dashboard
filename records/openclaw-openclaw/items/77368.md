## Summary

Close as configuration/support rather than a Feishu send-path bug: the reported `queuedFinal=false, replies=0` path matches the documented group/channel visible-reply policy when `messages.groupChat.visibleReplies` is unset and the model did not call the message tool.

## What This Changes

The issue asks OpenClaw to deliver normal assistant replies automatically in Feishu group chats after a successful group turn.

## Best Possible Solution

Keep the cross-channel group safety default and resolve this deployment by using `messages.groupChat.visibleReplies: "automatic"` or by making visible group output go through `message(action=send)`.

## Reproduction Assessment

Yes for the observed `replies=0` path: source and tests show group turns without `messages.groupChat.visibleReplies: "automatic"` suppress normal final delivery while still allowing the agent run to complete. This is expected configuration behavior, not a failing Feishu delivery call.

## Solution Assessment

Yes: the narrowest maintainable solution is the existing visible-reply config or a tool-calling model/runtime that sends through the message tool. Changing Feishu to auto-post final text would bypass the current cross-channel group/channel policy.
