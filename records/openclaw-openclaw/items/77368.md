## Summary

Current source and docs show the reported Feishu `replies=0` group behavior is the expected group/channel visible-reply policy, not a Feishu transport failure; the fix for this deployment is existing configuration or explicit message-tool output.

## What This Changes

The issue reports that Feishu group-chat turns complete successfully but do not deliver visible replies, while direct messages do.

## Best Possible Solution

Keep the shared safe group/channel default and resolve this deployment with `messages.groupChat.visibleReplies: "automatic"` or explicit `message(action=send)` output for visible group replies.

## Reproduction Assessment

Yes: source inspection gives a high-confidence path for the observed `replies=0` state. A Feishu group turn without `messages.groupChat.visibleReplies: "automatic"` resolves to message-tool-only delivery, runs the agent, and suppresses normal final reply delivery.

## Solution Assessment

Yes: the existing config switch or message-tool send path is the narrowest maintainable solution. Changing Feishu delivery code would bypass the current cross-channel group safety policy.
