## What This Changes

The PR adds positive-value validation to Telegram reply-to message ID normalization and expands outbound-params tests for numeric, string, non-finite, missing, UUID, zero, and negative cases while leaving thread ID parsing intact.

## Best Possible Solution

## Solution Assessment

Yes for the functional patch. Centralizing the `> 0` guard in `normalizeTelegramReplyToMessageId` is the narrowest maintainable fix because the relevant Telegram send surfaces already route through that helper; the safer remaining change is only the required changelog entry.
