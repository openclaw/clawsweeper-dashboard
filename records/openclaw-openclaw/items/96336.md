## Summary

Keep open: the private-permission direction is valid, but the patch is not merge-ready because it hardens caller-owned custom store dirs and still leaves existing permissive managed dirs unrepaired.

## What This Changes

This PR adds explicit `0o700` directory modes to session, gateway transcript, tool, plugin skill, settings, and atomic text write directory creation paths.

## Best Possible Solution

Land a managed-private-directory helper that applies mkdir plus chmod/doctor repair only for OpenClaw-owned state dirs, preserves caller-owned custom paths, and includes fresh-install plus upgrade proof under a permissive umask.

## Reproduction Assessment

Yes, at source level. Current main still has managed session and transcript directory creation that can inherit umask-derived modes, and the PR body includes before/after mode output for a few paths, but I did not run a live filesystem repro in this read-only review.

## Solution Assessment

No. The hardening direction is right, but the best fix should distinguish managed from custom paths and repair existing permissive managed dirs instead of relying on mode-only mkdir changes.
