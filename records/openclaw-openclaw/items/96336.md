## Summary

Keep open: the private-directory hardening is useful, but the current head still applies 0o700/chmod policy to caller-owned project settings, export destinations, and custom session.store transcript parents, so it is not merge-ready.

## What This Changes

The PR adds explicit 0o700 directory modes and best-effort chmod repairs across session, settings, tool, gateway transcript, and plugin-skill directory creation paths.

## Best Possible Solution

Narrow private-mode hardening to OpenClaw-managed state directories, preserve caller-owned/custom paths unless maintainers explicitly choose otherwise, and add focused tests plus final-head umask proof.

## Reproduction Assessment

Yes at source level: current main has recursive mkdir paths without explicit private mode, and Node's mkdir mode default is 0o777 before platform/umask handling. I did not run a live umask reproduction in this read-only review.

## Solution Assessment

No. The managed-state hardening direction is appropriate, but the current implementation is too broad; the better fix separates managed OpenClaw state from caller-owned directories and proves both policies.
