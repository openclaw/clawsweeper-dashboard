Full review comments:

- **[P1] Preserve project settings directory permissions:** `src/agents/sessions/settings-manager.ts:217`
  - body: `withLock()` handles both global settings and project settings under `cwd/.openclaw/settings.json`. Adding `mode: 0o700` and chmod here also owner-locks shared workspace project settings, which is broader than managed private state.
  - confidence: 0.9
- **[P1] Preserve explicit export destination permissions:** `src/agents/sessions/agent-session.ts:3197`
  - body: `exportToJsonl(outputPath)` accepts an arbitrary caller-provided output path. Creating that parent as `0700` changes user export directory permissions outside OpenClaw-managed state, despite the PR saying export targets are preserved.
  - confidence: 0.9
- **[P1] Preserve custom session.store transcript roots:** `src/config/sessions/session-accessor.ts:1026`
  - body: This transcript parent is derived from `path.dirname(storePath)`, and `session.store` is documented for custom disk layout and mounted-volume storage. A fresh custom root would now be created owner-only, so carry managed/custom ownership before applying private mode.
  - confidence: 0.86
