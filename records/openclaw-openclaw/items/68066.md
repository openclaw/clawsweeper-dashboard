## Summary

Keep open. Current main still finalizes OpenRouter chat-completion stream usage from the shared OpenAI-compatible chunk parser and local cost estimation, with no OpenRouter `/generation` reconciliation before the assistant message is emitted or later transcript cost summaries are read.

## What This Changes

The issue requests authoritative OpenRouter billed-cost accounting for streamed tier-priced calls instead of persisting a base-tier or locally estimated `usage.cost.total`.

## Best Possible Solution

Add OpenRouter-owned billed-cost reconciliation for completed streaming chat completions on verified OpenRouter routes, using the response id and credential to fetch `/generation`, validate numeric `total_cost`, and preserve existing estimates on lookup failure.

## Reproduction Assessment

Yes, at source level. Feed the OpenAI-compatible stream path an OpenRouter-style final usage chunk with an id and current main records the id and local parsed usage, then emits `done` without any mocked `/generation` lookup.

## Solution Assessment

Yes. The narrowest maintainable fix is provider-owned post-stream OpenRouter generation reconciliation with guarded route detection, credential checks, bounded lookup, and fallback to the existing estimate.
