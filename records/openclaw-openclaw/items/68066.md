## Summary

Keep this issue open. Current main still persists OpenRouter streaming usage through the shared OpenAI-compatible stream path and local/model-catalog cost estimation, without reconciling completed OpenRouter generations against the authoritative `/generation` `total_cost`.

## What This Changes

The issue asks OpenClaw to stop under-recording OpenRouter tier-priced streaming calls by using OpenRouter's authoritative billed generation cost instead of the streamed/base-tier estimate.

## Best Possible Solution

Add OpenRouter-owned post-stream, pre-finalization billed-cost reconciliation for verified OpenRouter chat-completions calls, using `/generation` `total_cost` when valid and preserving the existing estimate on lookup failure or non-OpenRouter endpoints.

## Reproduction Assessment

Yes from source inspection, without a live billed replay. Current main records the streamed response id and local usage estimate, then finalizes the assistant message without any OpenRouter generation metadata lookup.

## Solution Assessment

Yes. A bounded OpenRouter-specific reconciliation before the assistant message is finalized is the narrowest maintainable fix; broader dashboard, `agent.wait`, or cross-agent usage changes should stay out of scope.
