## Summary

Keep open. Current main still finalizes OpenRouter chat-completion streams with the shared OpenAI-compatible usage parser and local pricing calculation, and there is no OpenRouter-owned `/generation` reconciliation before the assistant transcript message is emitted.

## What This Changes

The issue requests correcting OpenRouter streamed cost persistence so assistant JSONL records use the authoritative billed generation cost for tier-priced models instead of the streamed/local estimated `usage.cost.total`.

## Best Possible Solution

Implement OpenRouter-owned billed-cost reconciliation for completed streaming chat completions. When the stream finishes with a response id and a real OpenRouter credential on the verified OpenRouter endpoint, fetch generation metadata, validate numeric `data.total_cost`, and persist that billed value without changing unrelated providers or dashboard aggregation semantics.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to drive the OpenAI-compatible completions stream with an OpenRouter-style final chunk containing a response id and streamed usage, then assert current main finalizes `usage.cost.total` from local `calculateCost` without any mocked `/generation` lookup. The issue also provides a live OpenRouter comparison path for long-context tiered calls.

## Solution Assessment

Yes. The best direction is the issue's post-hoc OpenRouter generation lookup with conservative guardrails: only verified OpenRouter streams, only after successful completion, bounded lookup behavior, and fallback to the existing estimate on lookup failure or abort.
