## Summary

Keep open. Current `main` still finalizes OpenRouter chat-completion stream usage from the shared OpenAI-compatible chunk parser and local `calculateCost`, with no OpenRouter `/generation` reconciliation before the assistant transcript message is emitted.

## What This Changes

The issue requests correcting OpenRouter transcript cost accounting by reconciling streamed usage with OpenRouter's authoritative per-generation billed `total_cost`.

## Best Possible Solution

Add OpenRouter-owned billed-cost reconciliation for completed streaming chat completions, guarded to verified OpenRouter routes with a real response id and credential, while leaving unrelated providers and usage dashboard aggregation unchanged.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed the OpenAI-compatible stream path an OpenRouter-style final usage chunk with a response id and assert current `main` emits locally calculated `usage.cost.total` without any mocked `/generation` lookup.

## Solution Assessment

Yes. The post-hoc OpenRouter generation lookup is the narrowest maintainable fix if implemented in the OpenRouter provider wrapper or a minimal generic post-stream seam, with fallback to the existing estimate on failure.
