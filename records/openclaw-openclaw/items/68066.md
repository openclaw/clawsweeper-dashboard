## Summary

Keep open. Current main still records OpenRouter streamed usage through the shared OpenAI-compatible parser and local pricing estimate, while the provider-specific authoritative generation-cost lookup is absent.

## What This Changes

The issue asks OpenClaw to stop under-recording OpenRouter costs for tier-priced streamed calls by reconciling transcript `usage.cost.total` against OpenRouter's authoritative billed generation cost.

## Best Possible Solution

Add OpenRouter-owned billed-cost reconciliation for completed streaming chat completions, using `/generation` only on verified OpenRouter routes and falling back non-destructively to the existing estimate when reconciliation is unavailable.

## Reproduction Assessment

No live billed OpenRouter replay was run, but source inspection gives a high-confidence reproduction path: a streamed OpenRouter response with final `usage` is parsed and finalized without any `/generation` lookup. The reporter supplied real billing comparisons, and current main still lacks the code path needed to correct them.

## Solution Assessment

Yes. The narrow maintainable fix is to reconcile only verified OpenRouter streaming completions after successful completion, validate numeric `data.total_cost`, and leave existing usage untouched for aborts, custom proxies, missing ids, or lookup failures.

Current main persists OpenRouter streamed `usage.cost.total` from the shared OpenAI-compatible parser/local estimate, which can undercount tier-priced OpenRouter generations.
