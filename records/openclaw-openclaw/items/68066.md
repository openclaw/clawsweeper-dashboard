## Summary

Keep open. Current main still emits OpenRouter chat-completion stream messages with locally calculated OpenAI-compatible usage cost and has no OpenRouter `/generation` reconciliation before the assistant transcript message is finalized. The related open PRs in the provided context address usage exposure or aggregation, not this per-generation billed-cost correction.

## What This Changes

The issue requests correcting OpenRouter streaming usage accounting so transcript `usage.cost.total` reflects OpenRouter's authoritative billed `total_cost` for tier-priced generations rather than the streamed lower-bound estimate.

## Best Possible Solution

Add OpenRouter-owned billed-cost reconciliation for completed streaming chat completions: when a verified OpenRouter request finishes with a response id and credential, fetch generation metadata, validate a finite non-negative `total_cost`, and persist that billed value while leaving existing estimates intact for failures, aborts, missing ids, and custom non-OpenRouter endpoints.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to drive the OpenAI-compatible completion stream with an OpenRouter-style final usage chunk and response id, then assert current main finalizes `usage.cost.total` from `calculateCost` without any mocked `/generation` lookup; the issue also provides live OpenRouter billed-vs-recorded examples.

## Solution Assessment

Yes. The issue's post-hoc OpenRouter generation lookup is the narrowest maintainable direction if guarded to verified OpenRouter endpoints and successful completed streams; unrelated providers, dashboard aggregation, and `agent.wait` metadata should not change as part of this fix.
