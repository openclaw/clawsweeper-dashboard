## Summary

Keep this open. Current main still records OpenRouter streamed usage through the shared OpenAI-compatible parser and local pricing path, while OpenRouter exposes an authenticated generation metadata endpoint with authoritative `total_cost`; the adjacent tiered-pricing and dashboard work does not update the persisted assistant message cost from that endpoint.

## What This Changes

The issue reports that OpenRouter streamed `message.usage.cost.total` persisted to assistant JSONL can undercount the billed amount on tier-priced long-context generations.

## Best Possible Solution

Add OpenRouter-owned billed-cost reconciliation for completed streamed chat completions, updating the final assistant usage cost from verified `/generation` metadata before persistence while preserving the existing estimate on lookup failure.

## Reproduction Assessment

Yes. Source inspection shows a high-confidence path: OpenRouter stream usage is parsed locally, `responseId` is retained, and the final assistant message is emitted without any `/generation` lookup; the reporter also provided live billed-vs-recorded examples.

## Solution Assessment

Yes. A bounded OpenRouter-specific reconciliation before final `done` emission is the narrowest maintainable fix; broader dashboard, `agent.wait`, or public runtime metadata changes should stay out of scope.
