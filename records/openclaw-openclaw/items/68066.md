## Summary

Keep this open. Current main still finalizes OpenRouter streaming completions with the streamed/OpenAI-compatible usage parsed in the generic transport and locally calculated cost, and I found no OpenRouter `/generation` reconciliation path that would replace the assistant JSONL `usage.cost.total` with authoritative billed `total_cost` for tier-priced calls.

## What This Changes

The issue asks OpenClaw to stop persisting undercounted OpenRouter streamed cost totals for tier-priced long-context calls and use authoritative billed cost instead.

## Best Possible Solution

Add OpenRouter-owned billed-cost reconciliation for successful streaming chat completions on verified OpenRouter routes, updating the persisted assistant message from `/generation` only when a response id, real credential, and numeric billed total are available.

## Reproduction Assessment

Yes, at source level. A streaming OpenRouter completion with a final usage chunk and response id will be finalized from `parseTransportChunkUsage` and emitted without any `/generation` lookup, so any billed `total_cost` divergence remains uncorrected.

## Solution Assessment

Yes. The narrow maintainable fix is provider-scoped reconciliation before final assistant persistence, with no unrelated changes to the usage dashboard, agent.wait metadata, or local tiered-pricing estimates.
