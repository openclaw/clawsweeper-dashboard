## Summary

Keep open. Current main and the latest checked release still finalize OpenRouter streaming usage through the shared OpenAI-compatible parser and local cost estimation, with no OpenRouter `/generation` reconciliation for authoritative billed `total_cost`.

## What This Changes

The issue asks OpenClaw to record accurate OpenRouter billed costs for tier-priced streaming calls instead of undercounting long-context requests.

## Best Possible Solution

Add OpenRouter-owned post-stream billed-cost reconciliation using the response id and `/generation`, update the assistant usage total only on a validated numeric billed cost, and preserve existing estimates on lookup failure.

## Reproduction Assessment

Yes at source level: feed the OpenAI-compatible stream path a final OpenRouter-style usage chunk with a response id and current main will emit locally calculated `usage.cost.total` without any `/generation` lookup. I did not run a live OpenRouter billing request in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is an OpenRouter-scoped post-stream generation lookup with validation and fallback, not a dashboard aggregation change or an `agent.wait` metadata-only change.
