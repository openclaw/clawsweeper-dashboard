## Summary

Keep open. Current main still records OpenRouter streaming usage through the shared OpenAI-compatible parser and local cost calculation, with no OpenRouter `/generation` reconciliation for authoritative billed `total_cost`; the related tiered-pricing and usage-aggregation work is adjacent but does not fix this transcript write path.

## What This Changes

The issue requests accurate OpenRouter billed cost recording for streamed calls whose final streamed `usage.cost.total` can undercount tier-priced model billing.

## Best Possible Solution

Add OpenRouter-scoped post-stream billed-cost reconciliation using the response id and `/generation`, validate numeric `total_cost`, and preserve the current estimate when lookup is unavailable or out of scope.

## Reproduction Assessment

Yes, at source level: an OpenRouter-style final stream chunk with a response id is parsed into local `usage.cost.total`, and current main has no `/generation` lookup before the assistant record is finalized. I did not run a live billed OpenRouter request in this read-only review.

## Solution Assessment

Yes, with one maintainer decision outstanding. The narrow maintainable fix is provider-owned post-stream reconciliation in the OpenRouter path with validation and fallback, not a dashboard aggregation change or unrelated `agent.wait` metadata work.
