## Summary

Keep open. Current main still records OpenRouter streamed usage through the shared OpenAI-compatible stream parser/local cost path, and I found no OpenRouter `/generation` reconciliation that would replace the assistant record with authoritative billed `total_cost` for tier-priced models.

## What This Changes

The issue asks OpenClaw to correct OpenRouter streamed cost persistence so tier-priced calls use OpenRouter's authoritative billed generation cost instead of the lower streamed estimate.

## Best Possible Solution

Add an OpenRouter-scoped post-stream reconciliation step that fetches generation metadata by response id, validates numeric `total_cost`, records the authoritative billed value before final persistence when available, and leaves the existing estimate intact when reconciliation is not possible.

## Reproduction Assessment

Yes, at source level: current main parses an OpenRouter-style final stream chunk into local usage/cost and finalizes the assistant message without any `/generation` lookup. I did not run a live billed OpenRouter request in this read-only review.

## Solution Assessment

Yes with one policy decision remaining: the narrow maintainable fix belongs in the OpenRouter/provider transport path, but maintainers should choose whether `usage.cost.total` becomes authoritative billed cost or whether a separate billed field is added.
