## Summary

Keep this issue open. Current main still finalizes OpenRouter chat-completion stream usage through the shared OpenAI-compatible path and local pricing calculation, with no OpenRouter `/generation` lookup to replace the persisted transcript cost with authoritative billed `total_cost`.

## What This Changes

The issue reports that OpenRouter streaming usage costs persisted by the gateway can undercount billed cost for tier-priced models because they are not reconciled against OpenRouter generation metadata.

## Best Possible Solution

Add OpenRouter-owned billed-cost reconciliation for completed streaming chat completions: when a verified OpenRouter route, response id, and credential are available, fetch `/generation`, validate numeric `total_cost`, persist it before finalizing the assistant message, and leave the existing estimate untouched on lookup failure or non-OpenRouter routes.

## Reproduction Assessment

Yes, source-reproducible without a live billed replay. Current main records the OpenRouter response id and local parsed usage, then finalizes the assistant message without any `/generation` metadata lookup.

## Solution Assessment

Yes. A bounded OpenRouter-specific reconciliation before assistant-message finalization is the narrowest maintainable fix; broader dashboard, `agent.wait`, or public runtime metadata changes should stay out of scope for this issue.
