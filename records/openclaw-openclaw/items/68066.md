## Summary

Keep this open. Current main still persists OpenRouter streamed usage through the shared OpenAI-compatible parser/local pricing path and emits the final assistant message without an OpenRouter `/generation` lookup, while the reporter and OpenRouter's own metadata contract point to `total_cost` as the billed source of truth.

## What This Changes

Reconcile persisted OpenRouter streamed `usage.cost.total` with authoritative billed generation metadata for tier-priced models.

## Best Possible Solution

Add OpenRouter-owned billed-cost reconciliation for completed streaming chat completions, using `/generation` `total_cost` when safely available and preserving the existing estimate on lookup failure.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: OpenRouter stream chunks can set `responseId` and usage, local cost is calculated, and the final message is emitted without a `/generation` reconciliation; the reporter also supplied live billed-vs-recorded examples.

## Solution Assessment

Yes. A bounded OpenRouter-specific post-stream reconciliation before final persistence is the narrow maintainable fix; dashboard, `agent.wait`, and broader runtime metadata changes should stay out of scope.
