## Summary

Keep this issue open because current main still has the redundant plugin-discovery call pattern, and the issue is explicitly paired with an open implementation PR that closes it.

## What This Changes

The issue requests function-scoped reuse of a `PluginDiscoveryResult` across plugin startup helpers to avoid repeated filesystem discovery walks.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/75451 as the implementation path, preserving stateless discovery while allowing explicit current-flow reuse.

## Reproduction Assessment

Yes, at source level: current main shows the named helpers still call `discoverOpenClawPlugins` independently, and the contract retry path can call the bundled runtime registry twice. I did not run a runtime benchmark in this read-only review.

## Solution Assessment

Yes, the requested explicit discovery threading is the narrow maintainable direction because it avoids hidden caches and matches the plugin boundary guidance. The linked PR should be reviewed for correctness and proof before this issue closes.
