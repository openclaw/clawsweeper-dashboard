## Summary

Keep this open because current main still has the source-level duplicate discovery pattern, and the issue is paired with an open implementation PR that uses closing syntax. The right cleanup path is to review, land, or reject https://github.com/openclaw/openclaw/pull/75451 rather than closing the issue first.

## What This Changes

The issue asks to avoid repeated plugin discovery filesystem walks by threading an explicit caller-owned `PluginDiscoveryResult` through startup plugin helper flows while keeping discovery stateless.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/75451 as the implementation candidate, preserving stateless discovery while allowing same-flow callers to reuse an explicit discovery result.

## Reproduction Assessment

Yes. Source inspection on current main shows the named helpers still call `discoverOpenClawPlugins` independently, and the scoped registry retry loop can call the bundled runtime registry twice.

## Solution Assessment

Yes. The requested direction is the narrow maintainable solution because it threads an explicit discovery object through the affected call chain without adding a hidden discovery cache.
