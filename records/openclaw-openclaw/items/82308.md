## Summary

Keep this issue open because current main still has the repeated plugin discovery pattern and the issue is paired with an open implementation PR that closes it; closing before that PR lands would break the normal issue/PR workflow.

## What This Changes

The issue requests function-scoped reuse of plugin discovery results during startup to avoid repeated filesystem walks while keeping discovery stateless.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/75451, preserving stateless discovery while allowing explicit discovery reuse within one startup flow.

## Reproduction Assessment

Yes, at source level: current main still calls `discoverOpenClawPlugins` inside the named helpers, and the scoped contracts retry loop can invoke the bundled runtime registry load twice. I did not run a runtime benchmark in this read-only review.

## Solution Assessment

Yes, the proposed direction is the narrow maintainable fix: thread an explicit discovery result through the affected call chain without adding a hidden cache. The issue should close only after the linked implementation merges or maintainers choose a different concrete fix.
