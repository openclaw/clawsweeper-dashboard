## Summary

Keep open: current main still has the duplicated discovery pattern, and the live implementation path is the linked open PR with closing syntax. Closing this issue before that PR lands or is rejected would break the normal issue-to-fix flow.

## What This Changes

Request reuse of an explicit `PluginDiscoveryResult` across plugin startup helpers to avoid duplicate filesystem walks without adding a hidden discovery cache.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/75451 as the implementation candidate, preserving stateless discovery while allowing same-flow callers to pass an explicit discovery result.

## Reproduction Assessment

Yes, at source level: current main shows the named helpers each call `discoverOpenClawPlugins` internally, and the scoped registry retry loop can call the bundled runtime registry twice. I did not run a benchmark because this review is read-only.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix: thread an explicit caller-owned discovery result through the affected functions while leaving `discoverOpenClawPlugins` stateless. The linked PR should be reviewed for proof and correctness before this issue closes.
