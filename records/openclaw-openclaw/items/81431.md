## Summary

Keep open: this is a useful Feishu Bitable feature, but the branch is not mergeable because it fails TypeScript, forwards `search.filter` with the wrong SDK shape, and lacks external real behavior proof.

## What This Changes

The branch extends `feishu_bitable_list_records` with optional `filter`, `sort`, `field_names`, and `view_id` parameters and switches to `appTableRecord.search` when `filter` or `sort` is present.

## Best Possible Solution

Land one reconciled Feishu plugin update that exposes the supported Bitable list/search parameters with SDK-correct schemas, focused coverage, and redacted real Feishu proof.

## Reproduction Assessment

Yes for the review findings: the PR CI `check-prod-types` failure and pinned Lark SDK types show the filter shape and tool param type errors. No real Feishu setup proof exists for the feature behavior itself.

## Solution Assessment

No. The feature direction is narrow, but this patch should either keep the list endpoint string-parameter contract or expose/translate the search endpoint's structured filter contract, then add tests and live proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Use the search filter object shape:** `extensions/feishu/src/bitable.ts:215`
  - body: The PR switches to `appTableRecord.search` when `filter` is set, but the new schema still accepts a Feishu expression string and forwards it as `data.filter`. The pinned SDK expects a structured `{ conjunction, conditions }` object for search, and CI is already failing TS2322 here; either translate/expose that structured filter or keep the list endpoint's string filter contract.
  - confidence: 0.96
- **[P2] Include new params in the registered tool type:** `extensions/feishu/src/bitable.ts:716-719`
  - body: The `registerBitableTool` generic still declares only pagination fields, so the added `params.filter`, `params.sort`, `params.field_names`, and `params.view_id` reads fail `tsgo` with TS2339. Add the optional fields to this parameter type after settling their SDK-compatible shapes.
  - confidence: 0.98
