## Summary

Keep open. Current main still only has partial Telegram reaction prompt support: Telegram exposes `reactionNotifications` and `reactionLevel`, but not the requested public `reactionGuidance` config/schema/docs surface, and the prompt text lacks the requested inbound reaction soft-CTA guidance.

## What This Changes

Requested configurable bidirectional Telegram reaction guidance in the agent prompt, including inbound reaction interpretation and a new `reactionGuidance: "off" | "minimal" | "full"` option.

## Best Possible Solution

Define the public Telegram reaction-guidance contract, then implement it consistently across Telegram config/schema/generated metadata/docs, prompt composition, notification gating, and focused tests for `off`, `minimal`, `full`, and inbound reaction interpretation.

## Reproduction Assessment

Not applicable as a bug reproduction path. Current source, docs, generated metadata search, focused tests, and related item inspection show the requested public config key, notification-gated guidance behavior, and inbound soft-CTA prompt text are absent.

## Solution Assessment

Unclear until maintainers choose the public config shape. The narrow maintainable path is to decide how `reactionGuidance` composes with `reactionLevel`, `reactionNotifications`, and open reaction wake/semantics work, then update the Telegram plugin and shared prompt composition together.
