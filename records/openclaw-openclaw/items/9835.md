## Summary

Keep this issue open. Current main has Telegram reaction send/receive support and outbound reaction prompt guidance, but it still lacks the requested public `reactionGuidance` config contract, schema/docs metadata, notification-gated prompt policy, and incoming reaction soft-CTA guidance.

## What This Changes

The issue requests bidirectional Telegram reaction prompt guidance plus a new `reactionGuidance` config option with `off`, `minimal`, and `full` modes.

## Best Possible Solution

Define the public Telegram reaction-guidance contract first, then implement it in the bundled Telegram plugin and shared prompt builder with schema, generated metadata, docs, and focused tests for `off`, `minimal`, `full`, and notification gating.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main still lacks the requested `reactionGuidance` config and incoming soft-CTA prompt behavior.

## Solution Assessment

Unclear until maintainers choose the public contract. The requested direction is plausible, but it needs to be reconciled with existing `reactionLevel`, `reactionNotifications`, and the related prompt-default issue at https://github.com/openclaw/openclaw/issues/70043.
