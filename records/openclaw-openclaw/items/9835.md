## Summary

Keep open: current main and `v2026.5.20` still have Telegram reaction prompt support driven by `reactionLevel`, but they do not expose the requested public `reactionGuidance` config/schema/docs contract or the incoming-reaction soft-CTA prompt guidance. Related open reaction work covers wake policy and default prompt stability, not this full prompt/config contract.

## What This Changes

The issue requests Telegram reaction prompt guidance plus a public `reactionGuidance: "off" | "minimal" | "full"` config option gated by `reactionNotifications`.

## Best Possible Solution

Define the public Telegram reaction guidance contract, then implement it across the bundled Telegram plugin prompt adapter, shared prompt builder, strict schema, generated metadata, docs, and focused tests while coordinating with https://github.com/openclaw/openclaw/issues/70043 and the open wake-policy PRs.

## Solution Assessment

Unclear until maintainers choose the public contract. The requested direction is plausible, but it must be reconciled with existing `reactionLevel`, `reactionNotifications`, and the open prompt-stability issue before implementation.
