## Summary

Keep open: current main still only has partial Telegram reaction support. The requested `reactionGuidance: "off" | "minimal" | "full"` config, generated metadata/docs coverage, notification-gated prompt guidance, and incoming-reaction soft-CTA wording are not implemented.

## What This Changes

Request configurable Telegram reaction prompt guidance for outbound reactions and inbound reaction interpretation, gated by `reactionNotifications`.

## Best Possible Solution

Define one Telegram prompt-control contract, update schema/generated metadata/docs, gate incoming soft-CTA guidance on enabled reaction notifications, and add focused off/minimal/full plus notification-gating tests.

## Reproduction Assessment

Not applicable; this is a feature/config request rather than a broken existing contract. Source inspection gives a high-confidence current-main gap: partial reaction support exists, but the requested config and incoming prompt guidance do not.

## Solution Assessment

Unclear until maintainers choose the config model. Adding `reactionGuidance` may be right, but it overlaps existing `reactionLevel` and the open #70043 defaulting concern, so the narrow maintainable solution should be decided before implementation.
