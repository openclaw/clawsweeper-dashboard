## Summary

Keep open. Current main still has partial Telegram reaction support, but it does not implement the requested public `reactionGuidance` config/schema/docs surface, notification-gated prompt behavior, or inbound soft-CTA reaction guidance; related open Telegram reaction work covers semantics, wake behavior, or custom emoji rather than this prompt contract.

## What This Changes

Request to add bidirectional Telegram reaction prompt guidance, gated by `reactionNotifications`, with a new `reactionGuidance: "off" | "minimal" | "full"` Telegram config option.

## Best Possible Solution

Define the public Telegram reaction-guidance contract, then implement it consistently across Telegram config types, strict schema, generated metadata, docs, prompt composition, notification gating, and focused tests for `off`, `minimal`, `full`, and inbound reaction interpretation.

## Reproduction Assessment

Not applicable as a bug reproduction path. Source, docs, generated metadata, live issue context, and related PR search show the requested config key and incoming soft-CTA prompt guidance are absent on current main.

## Solution Assessment

Unclear until maintainers choose the public config shape. The maintainable path is to decide how `reactionGuidance` composes with existing `reactionLevel`, `reactionNotifications`, and reaction wake/semantics work, then update Telegram and shared prompt composition together.
