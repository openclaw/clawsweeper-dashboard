## Summary

Keep open: current main and the latest release still lack the requested public `reactionGuidance` config/schema/docs surface and the incoming-reaction soft-CTA prompt guidance. Existing Telegram reaction support is partial, and the related open default-stability issue and wake-on-reaction PR do not supersede this prompt-contract request.

## What This Changes

The issue requests configurable Telegram reaction prompt guidance with outbound reaction etiquette, incoming reaction interpretation, and a `reactionGuidance` `off|minimal|full` config option.

## Best Possible Solution

Define the public Telegram reaction guidance contract, then implement it in the bundled Telegram plugin and shared prompt builder with aligned config schema, generated metadata, docs, and focused tests.
