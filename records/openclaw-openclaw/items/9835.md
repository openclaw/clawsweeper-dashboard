## Summary

Keep open. Current main still has partial Telegram reaction support, but the requested `reactionGuidance` config/schema/docs surface and incoming reaction soft-CTA prompt guidance are still missing; the related open Telegram reaction PRs address semantics or wake behavior, not this public prompt contract.

## What This Changes

The issue requests bidirectional Telegram reaction prompt guidance plus a new `channels.telegram.reactionGuidance` config option with `off|minimal|full` behavior.

## Best Possible Solution

Define the public Telegram reaction-guidance contract, then implement it consistently across Telegram config/schema/generated metadata/docs, prompt composition, notification gating, and focused tests for `off`, `minimal`, `full`, and inbound reaction interpretation.

## Reproduction Assessment

Not applicable as a bug reproduction path. Source, docs, generated metadata search, related issue/PR context, and focused test inspection show the requested public config key and incoming soft-CTA prompt guidance are absent on current main.

## Solution Assessment

Unclear until maintainers choose the public config shape. The narrow maintainable path is to decide how `reactionGuidance` composes with existing `reactionLevel`, `reactionNotifications`, and adjacent reaction semantics/wake work, then update Telegram and shared prompt composition together.
