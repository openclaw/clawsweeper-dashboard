## Summary

Keep open. Current main still only has Telegram `reactionLevel`-driven outbound prompt guidance; the requested public `reactionGuidance` config/schema/docs surface, notification-gated prompt behavior, and inbound soft-CTA reaction guidance are still absent.

## What This Changes

The issue requests configurable Telegram reaction prompt guidance with inbound reaction interpretation tied to existing reaction notification support.

## Best Possible Solution

Define the public Telegram reaction-guidance contract first, then implement it consistently across Telegram config types, strict schema, generated metadata, docs, prompt composition, notification gating, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction path. Source, docs, generated metadata, live issue context, and related PR search show the requested config key and incoming soft-CTA prompt guidance are absent on current main.

## Solution Assessment

Unclear until maintainers choose the public config shape. The maintainable path is to decide how `reactionGuidance` composes with existing `reactionLevel`, `reactionNotifications`, and reaction wake/semantics work, then update Telegram and shared prompt composition together.
