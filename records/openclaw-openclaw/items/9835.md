## Summary

Keep open. Current main still has partial Telegram reaction support, but the requested public `reactionGuidance` option, notification-gated prompt guidance, and incoming reaction soft-CTA prompt semantics remain absent.

## What This Changes

This issue requests bidirectional Telegram reaction prompt guidance, gated by `reactionNotifications`, plus a new `reactionGuidance: "off" | "minimal" | "full"` config option.

## Best Possible Solution

Define the public Telegram reaction-guidance contract first, then implement it through the bundled Telegram plugin prompt resolver, shared system prompt, strict schema, generated config metadata, docs/changelog, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction path. Source and docs inspection on current main give high confidence that the requested public config key, notification-gated prompt behavior, and incoming reaction guidance are missing.

## Solution Assessment

Unclear until maintainers choose the public config shape. The narrow maintainable path is to settle how `reactionGuidance` composes with `reactionLevel`, `reactionNotifications`, and reaction wake behavior, then update Telegram and prompt composition consistently.
