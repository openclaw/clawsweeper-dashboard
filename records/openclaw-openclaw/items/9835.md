## Summary

Keep open. Current main still has partial Telegram reaction support, but the requested public `reactionGuidance` config/schema/docs surface, notification-gated prompt behavior, and incoming-reaction soft-CTA guidance are still missing; the related open reaction-wake and semantics items are adjacent rather than replacements.

## What This Changes

The issue requests Telegram prompt guidance for sending and interpreting reactions, controlled by a new `reactionGuidance` config option that composes with existing reaction notification behavior.

## Best Possible Solution

Define the public Telegram reaction-guidance contract, then update Telegram config types, strict schema, generated metadata, docs, prompt composition, notification gating, and focused tests for `off`, `minimal`, `full`, and inbound reaction interpretation.

## Reproduction Assessment

Not applicable as a bug reproduction. Source, docs, schema, and related-item context show the requested config key and inbound soft-CTA prompt guidance are absent on current main.

## Solution Assessment

Unclear until maintainers choose the public config shape. The maintainable path is to decide how `reactionGuidance` composes with existing `reactionLevel`, `reactionNotifications`, https://github.com/openclaw/openclaw/issues/70043, and https://github.com/openclaw/openclaw/pull/47687 before updating Telegram and shared prompt composition together.
