## Summary

Current main partially satisfies the request by providing outbound Telegram reaction guidance through the existing `reactionLevel` policy, but it still lacks inbound soft-CTA interpretation and notification-gated semantics. The item remains useful, but maintainers should decide one canonical policy rather than add a parallel `reactionGuidance` control, especially while the default/session ambiguity remains tracked in https://github.com/openclaw/openclaw/issues/70043.

## What This Changes

Add configurable Telegram prompt guidance for sending and interpreting reactions, gated by reaction notification settings.

## Best Possible Solution

Keep one Telegram reaction policy, settle its upgrade-safe default and session semantics with the related defaulting issue, then extend that policy with notification-gated inbound interpretation and aligned schema, metadata, docs, and focused tests.

## Reproduction Assessment

Not applicable: this issue requests new inbound reaction interpretation and configuration semantics rather than reporting a failure of an existing defined contract.

## Solution Assessment

No: a separate `reactionGuidance` option is not currently the best solution because `reactionLevel` already owns outbound prompt guidance; maintainers should define one canonical control and its composition with notifications.
