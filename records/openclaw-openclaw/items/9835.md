## Summary

The requested incoming-reaction interpretation remains absent on current main, while outbound sparse/liberal reaction guidance is already implemented through the existing `reactionLevel` policy; this should remain open for a maintainer decision on canonical configuration and notification-gating semantics.

## What This Changes

Add Telegram prompt guidance for sparse outbound reactions and interpreting inbound reactions as soft conversational signals, with configurable guidance levels.

## Best Possible Solution

Keep one canonical Telegram reaction policy: add inbound soft-signal guidance only when inbound notifications are enabled, retain outbound guidance according to `reactionLevel`, and add a separate setting only if maintainers explicitly require independent control.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing contract, and the remaining behavior gap is directly established from current prompt, configuration, tests, and docs.

## Solution Assessment

No, not exactly as proposed: a separate `reactionGuidance` option duplicates the existing `reactionLevel` control; extending the canonical policy with independently gated inbound guidance is narrower and more maintainable.
