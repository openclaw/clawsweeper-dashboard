## Summary

Keep open: current main and the latest release still only provide partial Telegram reaction prompt support. The requested `reactionGuidance` config/schema/metadata/docs surface is absent, prompt guidance is still driven by `reactionLevel`, and the current `## Reactions` prompt lacks incoming soft-CTA guidance and `reactionNotifications` gating.

## What This Changes

Requests bidirectional Telegram reaction prompt guidance plus a configurable `reactionGuidance: "off" | "minimal" | "full"` option gated by Telegram reaction notifications.

## Best Possible Solution

Define the Telegram reaction prompt API contract first, then implement it in the bundled Telegram plugin and shared prompt composition with aligned config schema, generated metadata, docs, and focused tests for guidance levels plus notification gating.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection gives high confidence that the requested config option, prompt text, and gating remain missing on current main.

## Solution Assessment

Unclear until maintainers choose the product contract. The requested direction is plausible, but the maintainable solution must define how `reactionGuidance` composes with existing `reactionLevel`, `reactionNotifications`, and related reaction wake/default work.
