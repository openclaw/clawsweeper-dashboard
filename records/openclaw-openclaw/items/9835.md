## Summary

Keep open. Current main has partial Telegram reaction support, but the requested configurable two-way prompt guidance is still incomplete: there is no `reactionGuidance` config/schema/docs metadata, the prompt adapter is driven by `reactionLevel`, and the `## Reactions` text lacks incoming reaction soft-CTA guidance.

## What This Changes

Request to add configurable Telegram reaction prompt guidance tied to reaction notifications, outbound reaction restraint, and incoming reaction interpretation.

## Best Possible Solution

Keep this open until maintainers settle the config model; if accepted, implement it in the Telegram plugin plus shared prompt composition with schema, generated metadata, docs, changelog, and focused prompt/config tests.

## Reproduction Assessment

Not applicable as a feature request. Static inspection confirms the requested config field, schema/metadata/docs coverage, notification gating, and incoming reaction prompt guidance are not present on current main.

## Solution Assessment

Unclear. The requested direction is plausible, but adding `reactionGuidance` beside `reactionLevel` is not obviously the best API until maintainers decide how it composes with reaction notifications and the related default-prompt concern.
