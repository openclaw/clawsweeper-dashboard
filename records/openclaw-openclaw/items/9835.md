## Summary

Keep open: prior review context still matches current main. Telegram has outbound reaction prompt guidance through `reactionLevel`, but the requested `reactionGuidance` config/schema/docs contract and incoming-reaction soft-CTA guidance are still missing, so this needs maintainer product/API direction rather than cleanup closure.

## What This Changes

Request adds configurable two-way Telegram reaction prompt guidance with a `reactionGuidance: "off" | "minimal" | "full"` option gated by Telegram reaction notification behavior.

## Best Possible Solution

Define one public Telegram reaction-guidance contract that composes `reactionLevel`, `reactionNotifications`, and wake policy, then update the bundled Telegram plugin prompt adapter, shared prompt text, config schema/metadata/docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature/config request. Source inspection gives high confidence that current main and `v2026.5.20` lack the requested `reactionGuidance` field and incoming soft-CTA prompt guidance.

## Solution Assessment

Unclear until maintainers choose the contract. The requested direction is plausible, but it must be reconciled with existing `reactionLevel`, `reactionNotifications`, https://github.com/openclaw/openclaw/issues/70043, and https://github.com/openclaw/openclaw/pull/47687 before implementation.
