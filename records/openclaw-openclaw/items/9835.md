## Summary

Keep open: current main and latest release still have partial Telegram reaction support, but the requested separate `reactionGuidance` config, notification-gated prompt semantics, and inbound soft-CTA reaction guidance are still missing and need a maintainer product decision because `reactionLevel` already controls reaction prompt behavior.

## What This Changes

The issue requests two-way Telegram reaction prompt guidance, including inbound reaction interpretation and a new `reactionGuidance: off|minimal|full` config option.

## Best Possible Solution

Define one canonical Telegram reaction policy, then update the bundled Telegram plugin, shared prompt text, schema, generated metadata, docs, and focused tests together.

## Reproduction Assessment

## Solution Assessment

No. A separate `reactionGuidance` option is not clearly the best fix while `reactionLevel` already owns reaction prompt behavior; maintainers should first choose how it composes with `reactionNotifications`.
