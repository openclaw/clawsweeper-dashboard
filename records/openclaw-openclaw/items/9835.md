## Summary

Keep open: current main and v2026.6.10 still have Telegram reaction events and outbound reaction prompting through `reactionLevel`, but they do not implement the requested `reactionGuidance` config surface, `reactionNotifications`-gated prompt policy, or incoming soft-CTA reaction guidance.

## What This Changes

The issue requests configurable two-way Telegram reaction prompt guidance, including outbound reaction etiquette and incoming reaction interpretation.

## Best Possible Solution

Define one canonical Telegram reaction prompt policy, then update the bundled Telegram plugin, shared prompt composition, config schema/metadata, docs, and focused tests so notification gating, outgoing etiquette, and incoming soft-CTA guidance compose predictably.

## Reproduction Assessment

## Solution Assessment

No. The requested direction is plausible, but a separate `reactionGuidance` config is not clearly the best fix while `reactionLevel` already controls agent reaction prompting; maintainers should choose the canonical policy and precedence first.
