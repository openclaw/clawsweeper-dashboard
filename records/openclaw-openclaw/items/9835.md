## Summary

Keep open: current main still only has partial Telegram reaction support, while the requested public guidance config, prompt gating, and incoming-reaction interpretation are absent and overlap active reaction-default/semantics work.

## What This Changes

The issue requests bidirectional Telegram reaction prompt guidance plus a new `reactionGuidance: "off" | "minimal" | "full"` Telegram config option.

## Best Possible Solution

Define the public Telegram reaction-guidance contract, then update Telegram config types, strict schema, generated metadata, docs, prompt composition, notification gating, and focused tests for `off`, `minimal`, `full`, and inbound reaction interpretation.

## Reproduction Assessment

Not applicable as a bug reproduction. Source, docs, schema, and related-item context show the requested config key and inbound soft-CTA prompt guidance are absent on current main.

## Solution Assessment

Unclear until maintainers choose the public config shape. The maintainable path is to decide how `reactionGuidance` composes with existing `reactionLevel`, `reactionNotifications`, and adjacent reaction wake/semantics work before changing Telegram and shared prompt composition together.
