## Summary

Request to add two-way Telegram reaction prompt guidance, gated by reaction notifications and controlled by a new `reactionGuidance` config option.

## Best Possible Solution

Define one canonical Telegram reaction prompt policy, then update the bundled Telegram plugin, shared prompt composition, config schema/metadata, docs, and focused tests so notification gating, outgoing etiquette, and incoming soft-CTA guidance compose predictably.

## Solution Assessment

No. The requested direction is plausible, but a separate `reactionGuidance` config is not clearly the best fix while `reactionLevel` already controls Telegram reaction prompting; maintainers should choose the canonical policy and precedence first.
