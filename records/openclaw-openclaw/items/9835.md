## Summary

Keep open: current main and latest release still provide Telegram reaction events plus outbound prompt guidance from `reactionLevel`, but the requested `reactionGuidance` config surface, notification-gated prompt policy, and incoming-reaction soft-CTA guidance are still missing. This remains a product/config decision because it overlaps the existing `reactionLevel` policy and the related Telegram default-stability issue.

## What This Changes

Requested Telegram two-way reaction prompt guidance, including a new `reactionGuidance` config option and incoming-reaction soft-CTA guidance when Telegram reaction notifications are enabled.

## Best Possible Solution

Define one canonical Telegram reaction policy, then update the bundled Telegram plugin, shared prompt text, schema, generated metadata, docs, and focused tests together.

## Reproduction Assessment

Not applicable. This is a feature/configuration request, and source inspection confirms the requested public config and inbound prompt guidance surfaces are not present.

## Solution Assessment

No. The requested behavior is plausible, but a separate `reactionGuidance` option is not clearly the best fix while `reactionLevel` already controls reaction prompt behavior; maintainers should choose the canonical policy first.
