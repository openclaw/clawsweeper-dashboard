## Summary

Keep open: current main still lacks the requested public `reactionGuidance` config/schema/docs surface and incoming-reaction soft-CTA prompt guidance. Existing Telegram reaction support covers outbound guidance and incoming system events, while the related open items address adjacent defaulting, wake, and hook/semantics behavior rather than superseding this prompt-contract request.

## What This Changes

Request to add configurable Telegram reaction prompt guidance for both sending sparse reactions and interpreting incoming user reactions when Telegram reaction notifications are enabled.

## Best Possible Solution

Define the public Telegram reaction guidance contract first, then wire it through the bundled Telegram plugin, shared prompt builder, strict schema, generated metadata, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main still lacks the requested `reactionGuidance` config and incoming soft-CTA prompt behavior.

## Solution Assessment

Unclear until maintainers choose the public contract. The requested direction is plausible, but it must be reconciled with existing `reactionLevel`, `reactionNotifications`, and https://github.com/openclaw/openclaw/issues/70043.
