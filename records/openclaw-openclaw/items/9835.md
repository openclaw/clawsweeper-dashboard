## Summary

Keep open: current main still has partial Telegram reaction support, but the requested separate `reactionGuidance` config, notification-gated incoming soft-CTA guidance, schema metadata, and docs coverage are not implemented.

## What This Changes

Add configurable Telegram reaction prompt guidance for sending and interpreting reactions, gated by Telegram reaction notification settings.

## Best Possible Solution

Define how `reactionGuidance` composes with existing `reactionLevel` and `reactionNotifications`, then implement that contract in the bundled Telegram plugin, shared prompt composition, strict schema/generated metadata, docs, and focused off/minimal/full plus notification-gating tests.

## Reproduction Assessment

Yes, by source inspection rather than a runtime bug repro. Current main has `reactionLevel`-driven prompt guidance but no requested `reactionGuidance` config, schema entry, docs entry, or incoming soft-CTA prompt text.

## Solution Assessment

Unclear until maintainers decide the product contract. A new `reactionGuidance` knob may be right, but it must be reconciled with the already documented and tested `reactionLevel` default and the related unset-default concern in https://github.com/openclaw/openclaw/issues/70043.
