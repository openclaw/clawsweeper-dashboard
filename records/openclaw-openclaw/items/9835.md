## Summary

Keep open. Current main still has partial Telegram reaction support, but it does not implement the requested `reactionGuidance` config, notification-gated prompt behavior, or incoming reaction soft-CTA guidance.

## What This Changes

Request adds Telegram reaction prompt guidance for sending and receiving reactions plus a configurable `reactionGuidance` option gated by `reactionNotifications`.

## Best Possible Solution

Define the product semantics first, then implement the chosen behavior through the bundled Telegram plugin prompt resolver, shared system-prompt text, strict config schema, generated metadata, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction path. Source inspection gives high confidence that current main lacks the requested config key, notification-gated prompt behavior, and incoming reaction guidance.

## Solution Assessment

Unclear until maintainers choose the public config shape. The narrow maintainable path is to settle how `reactionGuidance` composes with existing `reactionLevel` and `reactionNotifications`, then update the Telegram plugin and shared prompt contract consistently.
