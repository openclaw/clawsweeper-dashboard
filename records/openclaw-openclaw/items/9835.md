## Summary

Keep open. Current main still has partial Telegram reaction support, but the requested public `reactionGuidance` config/schema/docs surface, notification-gated prompt behavior, and incoming-reaction soft-CTA guidance are still missing; the related open Telegram reaction items are adjacent rather than canonical replacements.

## What This Changes

Add Telegram reaction prompt guidance gated by `reactionNotifications`, including inbound reaction interpretation, plus a configurable `reactionGuidance: "off" | "minimal" | "full"` option.

## Best Possible Solution

Define the public Telegram reaction-guidance contract, then update Telegram config types, strict schema, generated metadata, docs, prompt composition, notification gating, and focused tests for `off`, `minimal`, `full`, and inbound reaction interpretation.

## Reproduction Assessment

Not applicable as a bug reproduction. Source, docs, generated metadata search, and related-item context show the requested config key and inbound soft-CTA prompt guidance are absent on current main.

## Solution Assessment

Unclear until maintainers choose the public config shape. The maintainable path is to decide how `reactionGuidance` composes with existing `reactionLevel`, `reactionNotifications`, https://github.com/openclaw/openclaw/issues/70043, and https://github.com/openclaw/openclaw/pull/47687 before updating Telegram and shared prompt composition together.
