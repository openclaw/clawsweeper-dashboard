## Summary

Keep open. Current main still has only partial Telegram reaction prompt support: the requested public `reactionGuidance` option, notification-gated prompt behavior, and incoming reaction soft-CTA guidance are still absent, while the related open items cover adjacent wake/default/semantics work rather than superseding this feature.

## What This Changes

Add configurable two-way Telegram reaction prompt guidance, including a public `reactionGuidance` option and incoming reaction interpretation when reaction notifications are enabled.

## Best Possible Solution

Define the public Telegram reaction-guidance contract, then implement it through Telegram config/schema/metadata/docs, prompt composition, notification gating, and focused tests for `off`, `minimal`, `full`, and incoming reaction interpretation.

## Reproduction Assessment

Not applicable as a bug reproduction path. Source, docs, generated metadata, and focused test inspection on current main show the requested public config key, notification-gated behavior, and incoming reaction guidance are absent.

## Solution Assessment

Unclear until maintainers choose the public config shape. The narrow maintainable path is to decide how `reactionGuidance` composes with `reactionLevel`, `reactionNotifications`, and reaction wake semantics, then update Telegram and shared prompt composition consistently.
