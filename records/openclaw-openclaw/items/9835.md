## Summary

Current main already provides outbound Telegram reaction guidance through the existing `reactionLevel` policy, but the requested inbound soft-CTA interpretation and notification-gated semantics remain absent. This should stay open for a maintainer product decision because adding a separate `reactionGuidance` option would overlap the current canonical control and could compound the defaulting ambiguity tracked in https://github.com/openclaw/openclaw/issues/70043.

## What This Changes

Add configurable Telegram prompt guidance for sending and interpreting reactions when reaction notifications are enabled.

## Best Possible Solution

Keep one canonical Telegram reaction policy, add inbound interpretation only when reaction notifications provide sufficiently reliable context, and document and test the chosen defaults and gating semantics without introducing a parallel setting unless distinct behavior cannot be expressed through the existing policy.
