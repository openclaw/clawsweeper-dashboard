## Summary

Keep open: current main and latest release still have Telegram reaction events plus outbound prompt guidance driven by `reactionLevel`, but the requested public `reactionGuidance` config, notification-gated prompt policy, and incoming-reaction soft-CTA guidance are still missing. This remains a maintainer product/config decision because it overlaps the existing Telegram reaction-level policy and the related default-stability issue.

## What This Changes

The issue requests bidirectional Telegram reaction prompt guidance, including incoming reaction interpretation, plus a new `reactionGuidance: off|minimal|full` config option.

## Best Possible Solution

Define one canonical Telegram reaction prompt policy, then update the bundled Telegram plugin, shared prompt text, config type/schema, generated metadata, docs, and focused tests together.

## Reproduction Assessment

Not applicable as a runtime bug repro. Source inspection confirms the requested public config and incoming soft-CTA prompt guidance are absent from current main and the latest release.

## Solution Assessment

No, not as written. The behavior is plausible, but a separate `reactionGuidance` option is not clearly the best shape while `reactionLevel` already controls reaction prompt behavior; maintainers should choose the canonical policy first.
