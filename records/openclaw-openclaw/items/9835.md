## Summary

Keep open. Current main has partial Telegram reaction support, but the requested configurable two-way prompt guidance is still incomplete: there is no `reactionGuidance` config/schema/docs metadata, prompt guidance is driven by `reactionLevel` rather than `reactionNotifications`, and the `## Reactions` text still lacks incoming reaction soft-CTA guidance.

## What This Changes

Request configurable Telegram reaction prompt guidance that covers both outbound reactions and interpreting incoming reactions, gated by `reactionNotifications` and controlled by `reactionGuidance: "off" | "minimal" | "full"`.

## Best Possible Solution

Keep this open until maintainers decide the config model. If accepted, implement the behavior in the Telegram plugin plus shared prompt composition: add or explicitly map `reactionGuidance`, define how it composes with `reactionNotifications` and `reactionLevel`, add incoming reaction soft-CTA prompt text, update strict schema/generated metadata/docs/changelog, and cover off/minimal/full plus notification gating in focused tests.

## Reproduction Assessment

Not applicable as a feature request, but the missing behavior is reproducible by static inspection: current config/schema/docs contain only `reactionNotifications` and `reactionLevel`, and the prompt text contains outbound-only reaction guidance.

## Solution Assessment

Unclear. The requested direction is plausible, but adding a second prompt-affecting knob beside `reactionLevel` is not obviously the best API until maintainers settle the composition and default behavior with the related default-prompt surprise report.
