The issue requests bidirectional Telegram reaction prompt guidance plus a new configurable `reactionGuidance` option for Telegram.

## Best Possible Solution

Define one public Telegram reaction-guidance contract that composes `reactionLevel`, `reactionNotifications`, and notification wake semantics, then update the Telegram plugin prompt adapter, shared prompt text, config schema/metadata/docs, and focused tests.
