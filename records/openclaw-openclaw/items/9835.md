## Summary

Current main and v2026.6.5 ship bidirectional Telegram reactions and outbound minimal/extensive prompt guidance, but notification-aware inbound soft-CTA guidance remains absent; the proposed separate config overlaps existing `reactionLevel` and requires a maintainer product and compatibility decision.

## What This Changes

Add configurable Telegram system-prompt guidance for sparse outbound reactions and interpretation of inbound reactions as soft calls to action.

## Best Possible Solution

Define one canonical Telegram reaction policy in the plugin that separately models outbound capability and inbound notification interpretation, preserves explicit upgrade behavior, and does not promise reaction-triggered turns unless that separate contract is enabled.

## Reproduction Assessment

Not applicable: this issue requests new configurable prompt semantics rather than reporting failure of an already-defined behavior. Source inspection clearly shows the requested inbound guidance is absent.

## Solution Assessment

No: the proposed parallel `reactionGuidance` option is not yet the best maintainable solution because `reactionLevel` already owns outbound guidance and related default and wake semantics remain unresolved.
