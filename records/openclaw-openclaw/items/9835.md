## Summary

Keep open. Current main still has only partial Telegram reaction prompt support: there is no public `reactionGuidance` config/schema/docs surface, Telegram prompt guidance is still derived from `reactionLevel`, and the prompt text does not include the requested incoming reaction soft-CTA guidance.

## What This Changes

Request bidirectional Telegram reaction prompt guidance with a new `reactionGuidance` config option and notification-gated prompt injection.

## Best Possible Solution

Define the public Telegram reaction-guidance contract, then implement it consistently across Telegram config/schema/generated metadata/docs, prompt composition, notification gating, and focused tests for `off`, `minimal`, `full`, and inbound reaction interpretation.

## Reproduction Assessment

Not applicable as a bug reproduction path. Source, docs, generated metadata search, and focused test inspection on current main show the requested public config key, notification-gated behavior, and incoming reaction guidance are absent.

## Solution Assessment

Unclear until maintainers choose the public config shape. The narrow maintainable path is to decide how `reactionGuidance` composes with `reactionLevel`, `reactionNotifications`, and reaction wake/semantics work, then update Telegram and shared prompt composition consistently.
