## Summary

Keep open: current main and the latest release still only have partial Telegram reaction prompt support. The requested `reactionGuidance` config/schema/metadata/docs surface is absent, prompt guidance is still driven by `reactionLevel`, and the current `## Reactions` section lacks incoming soft-CTA guidance and `reactionNotifications` gating.

## What This Changes

The issue requests configurable Telegram reaction prompt guidance with outbound reaction etiquette, incoming reaction interpretation, and a new `reactionGuidance` config option.

## Best Possible Solution

Define the Telegram reaction prompt API contract first, then implement it in the bundled Telegram plugin and shared prompt composition with aligned config schema, generated metadata, docs, and focused tests for `off|minimal|full` plus notification gating.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection gives high confidence that the requested config, prompt text, and gating remain missing on current main.

## Solution Assessment

Unclear until maintainers choose the product contract. The requested direction is plausible, but the safest solution must first define how `reactionGuidance` composes with existing `reactionLevel`, `reactionNotifications`, and the related unset-default report.
