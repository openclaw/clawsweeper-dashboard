## Summary

Keep open. Current main still has Telegram reaction support, but the requested public `reactionGuidance` config/schema/docs surface, notification-gated prompt behavior, and inbound soft-CTA prompt guidance are still absent; related open Telegram reaction work covers semantics/wake behavior rather than this prompt contract.

## What This Changes

Request adding Telegram reaction prompt guidance gated by `reactionNotifications`, plus a public `reactionGuidance` config option for `off|minimal|full` behavior.

## Best Possible Solution

Define the public Telegram reaction-guidance contract, then implement it consistently across Telegram config types, strict schema, generated metadata, docs, prompt composition, notification gating, and focused tests for `off`, `minimal`, `full`, and inbound reaction interpretation.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection, docs, generated metadata search, and live related-item context show the requested config key and inbound soft-CTA prompt guidance are absent on current main.

## Solution Assessment

Unclear until maintainers choose the public config shape. The maintainable path is to decide how `reactionGuidance` composes with existing `reactionLevel`, `reactionNotifications`, https://github.com/openclaw/openclaw/issues/70043, and https://github.com/openclaw/openclaw/pull/47687 before updating Telegram and shared prompt composition together.
