## Summary

Keep this issue open. Current main has partial Telegram reaction support, but it still lacks the requested public `reactionGuidance` config contract, notification-gated prompt behavior, and incoming-reaction soft-CTA guidance; the related default-prompt behavior report at https://github.com/openclaw/openclaw/issues/70043 is also still open.

## What This Changes

The issue requests bidirectional Telegram reaction prompt guidance with a new `reactionGuidance` config option tied to Telegram reaction notifications.

## Best Possible Solution

Define the public config contract first, then implement it in the bundled Telegram plugin and shared prompt composition: add/document `reactionGuidance`, compose it with `reactionNotifications` and `reactionLevel`, include incoming soft-CTA guidance only when notifications are enabled, and update schema, generated metadata, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms the requested config and prompt behavior are not present on current main, while partial reaction notification and outbound guidance support already exists.

## Solution Assessment

Unclear. The requested direction is plausible, but the safest solution needs maintainer agreement on how `reactionGuidance` should compose with existing `reactionLevel` defaults and `reactionNotifications`, especially with https://github.com/openclaw/openclaw/issues/70043 still open.
