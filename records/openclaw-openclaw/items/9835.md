## Summary

Keep this open. Current main has partial Telegram reaction support, but the requested separate `reactionGuidance` config, notification-gated prompt guidance, and incoming reaction soft-CTA guidance are still missing.

## What This Changes

The issue requests configurable Telegram reaction prompt guidance tied to Telegram reaction notifications, covering both outbound reaction etiquette and incoming reaction interpretation.

## Best Possible Solution

If maintainers want the feature, implement it in the bundled Telegram plugin plus shared prompt composition: add documented `reactionGuidance` semantics, gate incoming soft-CTA guidance on enabled reaction notifications, update strict schema/generated metadata/docs, and add focused tests for off/minimal/full plus notification gating.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that the requested config and incoming reaction prompt guidance are not present on current main.

## Solution Assessment

Unclear until maintainers choose the product semantics. The narrow maintainable path is a Telegram-owned config/schema/docs addition feeding the existing shared prompt section, without broadening unrelated channel reaction behavior.
