## Summary

Keep open: current main and v2026.5.20 still have partial Telegram reaction support, but no `reactionGuidance` config/schema/docs contract or incoming-reaction soft-CTA prompt guidance. Because that contract overlaps existing `reactionLevel`, `reactionNotifications`, and notification wake behavior, this needs maintainer product/API direction rather than cleanup closure.

## What This Changes

The issue requests configurable Telegram reaction prompt guidance and incoming reaction interpretation gated by Telegram reaction notifications.

## Best Possible Solution

Define one public Telegram reaction-guidance contract that composes `reactionLevel`, `reactionNotifications`, and notification wake semantics, then update the Telegram plugin prompt adapter, shared prompt text, config schema/metadata/docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature/config request. Source inspection gives high confidence that current main and v2026.5.20 lack the requested `reactionGuidance` field and incoming soft-CTA prompt guidance.

## Solution Assessment

Unclear until maintainers choose the contract. The requested direction is plausible, but it must be reconciled with existing `reactionLevel`, `reactionNotifications`, https://github.com/openclaw/openclaw/issues/70043, and https://github.com/openclaw/openclaw/pull/47687 before implementation.
