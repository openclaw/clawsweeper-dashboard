## Summary

Keep open: current main still resolves local macOS gateway auth as plain strings only, and this PR is the active focused fix, but it still needs a bounded keychain helper and real after-fix behavior proof before merge.

## What This Changes

The PR adds a macOS GatewayEndpointStore SecretInputResolver to resolve local gateway.auth token/password SecretRef dictionaries from the keychain and adds Swift tests for helper and localConfig paths.

## Best Possible Solution

Land a bounded, tested app-side SecretRef resolver for local macOS gateway auth, with remote auth semantics kept string-only unless separately designed.

## Reproduction Assessment

Yes, by source inspection. Current main casts local gateway.auth.token/password as strings while the browser proxy consumes GatewayEndpointStore.localConfig(), so a SecretRef dictionary resolves to nil on that path.

## Solution Assessment

No, not yet. The resolver seam and tests match the requested narrow fix shape, but the keychain subprocess needs a timeout/termination path and the contributor still needs after-fix real behavior proof.

- **[P2] Bound the keychain subprocess wait:** `apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift:146`
