## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Bound the keychain subprocess before browser proxy auth:** `apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift:146`
  - body: The sync localConfig() path used by MacNodeBrowserProxy.defaultEndpoint() can execute /usr/bin/security and then wait forever here. If keychain access hangs or prompts in the background, every browser proxy request can stall before the normal URLRequest timeout is reached; add a timeout/termination path or avoid sync keychain resolution on this request path.
  - confidence: 0.88
- **[P1] Stop launchd fallback after unresolved SecretRefs:** `apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift:314-318`
  - body: When a parsed keychain SecretRef returns nil, this branch still returns launchd credentials; the token path has the same fallback below. That can silently use stale auth even though the user explicitly configured gateway.auth.* as a SecretRef; track parsed-unresolved separately and fail closed or make the fallback a deliberate documented/tested compatibility mode.
  - confidence: 0.8
