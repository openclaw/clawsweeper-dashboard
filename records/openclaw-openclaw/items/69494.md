## Summary

This PR should stay open: the root bug is real and the branch now covers the intended local token/password SecretRef paths, but the current patch still has an unbounded keychain subprocess wait and lacks required after-fix real behavior proof.

## What This Changes

The PR changes macOS `GatewayEndpointStore` to resolve local gateway auth token/password SecretRef dictionaries through a keychain-backed resolver and adds Swift tests for helper and `localConfig` paths.

## Best Possible Solution

Land a bounded, tested macOS app SecretRef resolver for local gateway auth, keeping remote auth semantics unchanged unless a separate design expands that surface.

## Reproduction Assessment

Yes, by source inspection. Current main casts local `gateway.auth.token` and `gateway.auth.password` to strings while `MacNodeBrowserProxy.defaultEndpoint()` consumes `GatewayEndpointStore.localConfig()`, so structured SecretRefs resolve to nil before auth headers are built.

## Solution Assessment

No, not yet. The resolver seam and tests match the narrow fix direction, but the keychain subprocess needs a timeout/termination path and the contributor still needs redacted after-fix runtime proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Bound the keychain subprocess wait:** `apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift:146`
  - body: `securityFindGenericPassword` starts `/usr/bin/security` and then waits with no timeout. If keychain access locks up or the helper stalls, both the async resolver and sync `localConfig()` browser-proxy path can hang instead of falling back to nil, so add a bounded timeout that terminates the process and returns nil.
  - confidence: 0.84
