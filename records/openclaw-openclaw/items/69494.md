## Summary

Keep open: current main still resolves local macOS gateway auth as plain strings only, and this PR addresses a real browser-proxy auth bug, but it is not ready because the new keychain helper can wait indefinitely and the PR lacks inspectable after-fix real behavior proof.

## What This Changes

The PR adds macOS `GatewayEndpointStore` SecretRef/keychain resolution for local gateway auth token/password and regression tests for helper and `localConfig` paths.

## Best Possible Solution

Land a bounded, tested app-side SecretRef resolver for local macOS gateway auth, keeping remote auth semantics unchanged unless a separate design expands that surface.

## Reproduction Assessment

Yes, by source inspection. Current main casts local `gateway.auth.token` and `gateway.auth.password` as strings while the browser proxy consumes `GatewayEndpointStore.localConfig()`, so a SecretRef dictionary resolves to nil before auth headers are built.

## Solution Assessment

No, not yet. The resolver seam and tests match the narrow fix direction, but the keychain subprocess needs a timeout/termination path and the contributor still needs redacted after-fix runtime proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Bound the keychain subprocess wait:** `apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift:146`
  - body: `securityFindGenericPassword` waits indefinitely for `/usr/bin/security`. If the keychain is locked or the helper stalls, both the async resolver and sync `localConfig()` browser-proxy path can hang instead of falling back to nil, so add a timeout that terminates the process and returns nil.
