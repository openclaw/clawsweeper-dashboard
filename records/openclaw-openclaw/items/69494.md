## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Fail closed when keychain SecretRefs are unresolved:** `apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift:355-356`
  - body: When a configured local keychain SecretRef resolves to `nil`, this new sync resolver falls through to the launchd token/password fallback. Current gateway auth docs and tests require unresolved local `gateway.auth.*` SecretRefs to fail closed instead of being masked by another credential source, so this can silently authenticate with stale credentials and hide a broken keychain setup. Please distinguish an absent config value from a configured-but-unresolved SecretRef and block fallback in that case.
  - confidence: 0.89
- **[P1] Bound synchronous keychain resolution:** `apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift:146`
  - body: `localConfig()` is the path used by `MacNodeBrowserProxy.defaultEndpoint()`, and it now reaches `resolveSync()`, which runs `/usr/bin/security` and waits without any timeout. If keychain access hangs or prompts in a non-interactive context, the browser proxy actor can stall indefinitely instead of returning nil/failing cleanly; add a bounded timeout or otherwise avoid an unbounded synchronous wait on this request path.
  - confidence: 0.84
