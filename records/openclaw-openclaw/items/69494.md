## Label Justifications

- `P2`: This is a normal-priority macOS app browser-proxy auth bug with limited blast radius, not a core-runtime outage.
- `merge-risk: 🚨 compatibility`: The PR changes credential precedence for existing local gateway auth setups and can preserve fallback behavior after a configured SecretRef fails.
- `merge-risk: 🚨 auth-provider`: The diff changes how gateway auth token/password SecretRefs are resolved and selected for browser proxy requests.
- `merge-risk: 🚨 availability`: The new keychain subprocess can block endpoint resolution without a bounded timeout, potentially stalling browser proxy calls.

## Summary

Keep open: this targets a real macOS browser-proxy auth gap that still exists on current main and the latest release, but the branch is not merge-ready because it can mask unresolved SecretRefs with fallback credentials, adds an unbounded keychain subprocess on the sync proxy path, and still lacks inspectable real macOS behavior proof.

## What This Changes

This PR changes the macOS GatewayEndpointStore to resolve local gateway.auth token/password SecretRef dictionaries through a keychain resolver and adds focused tests for that path.

## Best Possible Solution

Keep the macOS app fix in GatewayEndpointStore, but make parsed SecretRefs report configured/unresolved separately, fail closed instead of launchd fallback, bound keychain subprocesses, keep the focused regression tests, and require redacted real macOS proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main and v2026.6.6 call GatewayEndpointStore.localConfig from MacNodeBrowserProxy, while the local auth resolver only accepts String token/password values. I did not run a live macOS keychain/browser-proxy reproduction in this read-only review.

## Solution Assessment

No. The PR is in the right owner path and covers the intended local token/password surfaces, but it is not the best merge shape until unresolved SecretRefs fail closed and keychain lookup has a bounded failure mode.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Fail closed when a parsed SecretRef is unresolved:** `apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift:347-356`
  - body: After `resolveConfigTokenSync` or `resolveConfigPasswordSync` parses a local SecretRef and the keychain lookup returns nil, this falls through to launchd token/password fallback. That can silently authenticate with stale supervisor credentials instead of honoring the explicit configured SecretRef; return a configured-but-unresolved state and stop fallback for parsed refs.
  - confidence: 0.86
- **[P1] Bound the keychain lookup on the proxy path:** `apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift:145-147`
  - body: `securityFindGenericPassword` waits for `/usr/bin/security` with no timeout, and the browser proxy reaches it through synchronous `localConfig(..., secretResolver: .blocking)`. A locked or stalled keychain process can hang endpoint resolution before the request timeout applies; add a timeout/termination path or avoid blocking the sync proxy path.
  - confidence: 0.84
