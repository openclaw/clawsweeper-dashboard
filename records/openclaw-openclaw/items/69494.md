## Summary

Keep open. Current main still has the macOS browser-proxy auth bug for structured local `gateway.auth.token` and `gateway.auth.password` SecretRefs, and this PR is the right area to fix it, but the latest PR head still adds an unbounded keychain subprocess wait and lacks the required user-facing changelog entry.

## What This Changes

The PR changes the macOS app `GatewayEndpointStore` to resolve local gateway token/password SecretRefs through a keychain-backed `SecretInputResolver`, threads that resolver through `localConfig()`, makes actor auth lookup async, and adds focused Swift tests.

## Best Possible Solution

Land this PR or an equivalent macOS-app patch after bounding and terminating the keychain lookup on timeout, preserving env/config/launchd precedence, keeping remote auth semantics string-only unless separately designed, and documenting the user-facing fix in the changelog.

## Reproduction Assessment

Yes. From current main source, configure local `gateway.auth.token` or `gateway.auth.password` as a structured SecretRef dictionary; the Swift `as? String` casts return nil, so the browser proxy endpoint is built without the expected auth secret.

## Solution Assessment

No, not yet. The resolver shape and test coverage are close, but the current PR should add a timeout/fail-closed path for the keychain subprocess and the required changelog entry before merge.
