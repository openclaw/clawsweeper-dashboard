## Summary

Keep open. Current main still has the reported macOS browser-proxy auth gap, and the PR is a valid direction that now covers both token/password SecretRefs and the sync `localConfig()` path, but the latest head still introduces an unbounded `/usr/bin/security` subprocess and lacks the required user-facing changelog entry.

## What This Changes

The PR changes the macOS desktop `GatewayEndpointStore` to resolve local `gateway.auth.token` and `gateway.auth.password` SecretRefs through a keychain-backed resolver, threads that resolver through `localConfig()`, makes actor auth lookup async, and adds focused Swift tests.

## Best Possible Solution

Land this PR or an equivalent macOS app patch after adding a bounded timeout with termination/fail-closed behavior for the keychain subprocess and documenting the fix in the Unreleased changelog, while preserving env/config/launchd precedence and keeping remote auth semantics unchanged unless separately designed.

## Reproduction Assessment

Yes. On current main, configure local `gateway.auth.token` or `gateway.auth.password` as a structured SecretRef dictionary; the Swift `as? String` casts return nil, so `MacNodeBrowserProxy.defaultEndpoint()` builds requests without the expected auth secret.

## Solution Assessment

No, not yet. The resolver seam and tests now cover the intended local token/password and `localConfig()` paths, but the subprocess timeout and changelog blockers should be fixed before merge.
