## Summary

Keep open: the PR fixes a real current-main macOS browser-proxy auth bug and now covers token, password, and the sync localConfig path, but the latest head still has an unbounded keychain subprocess and no required changelog entry.

## What This Changes

The PR changes the macOS app GatewayEndpointStore to resolve local gateway token/password SecretRef dictionaries through a keychain-backed resolver, threads it into localConfig, makes endpoint auth lookup async, and adds Swift regression tests.

## Best Possible Solution

Land this PR or an equivalent macOS-app patch after adding a timeout/terminate/fail-closed keychain runner and a changelog entry, while preserving local-only SecretRef handling and existing env/config/launchd precedence.

## Reproduction Assessment

Yes. Configure local gateway.auth.token or gateway.auth.password as a structured SecretRef on current main; the localConfig auth resolver casts those values to String, produces nil, and MacNodeBrowserProxy then builds requests without the expected auth header.

## Solution Assessment

No, not yet. The resolver shape and test coverage are close, but the patch should bound the keychain subprocess and document the user-facing fix before merge.
