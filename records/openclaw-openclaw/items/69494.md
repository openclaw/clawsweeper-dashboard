## Summary

Keep this PR open. Current main still has the macOS browser-proxy auth gap for local `gateway.auth.token` and `gateway.auth.password` SecretRefs, and the PR is relevant, but the latest diff still adds an unbounded `/usr/bin/security` subprocess path that can hang endpoint resolution instead of degrading to nil.

## What This Changes

The PR updates macOS `GatewayEndpointStore` to resolve local gateway auth token/password SecretRefs via a keychain resolver, makes actor token/password resolution async, threads a resolver into `localConfig`, and adds `GatewayEndpointStore` regression tests.

## Best Possible Solution

Land this PR or an equivalent macOS-app patch after bounding the keychain subprocess, preserving env and launchd precedence, keeping remote auth string-only unless separately designed, and covering both helper and `localConfig()` browser-proxy paths.

## Reproduction Assessment

Yes. Current main can be reproduced from source by configuring local `gateway.auth.token` or `gateway.auth.password` as a structured SecretRef: the Swift string casts return nil, and the browser proxy then builds requests without an auth header.

## Solution Assessment

No, not yet. The PR’s resolver shape and tests are now close to the right fix, but the unbounded `security` process needs a timeout/termination path before this is the safest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Bound the keychain lookup subprocess:** `apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift:144`
  - body: `securityFindGenericPassword` waits indefinitely for `/usr/bin/security`. Both the async resolver and the sync `localConfig()` browser-proxy path call this runner, so a locked or hung keychain can stall endpoint resolution instead of falling back to nil. Add a timeout and terminate/fail closed on expiry.
  - confidence: 0.88
