## Summary

Keep open. The PR targets a real macOS browser-proxy auth bug in current main and the latest force-push addresses the earlier password/source/localConfig coverage comments, but the new keychain subprocess still has no bounded timeout and the user-facing fix is missing the required changelog entry.

## What This Changes

The PR adds macOS GatewayEndpointStore resolution for local keychain SecretRefs in `gateway.auth.token` and `gateway.auth.password`, wires it into the browser-proxy local config path, and adds focused Swift tests.

## Best Possible Solution

Land the macOS-owned fix with one bounded, injectable keychain resolver shared by token/password and the browser-proxy localConfig path, plus regression tests and a changelog entry.

## Reproduction Assessment

Yes, at source level. Current main casts `gateway.auth.token` and `gateway.auth.password` to `String`, while the browser proxy gets auth from `GatewayEndpointStore.localConfig()`, so a SecretRef dictionary produces nil auth and no request header.

## Solution Assessment

No, not as-is. The latest PR has the right shape for SecretRef parsing and coverage, but the keychain process needs a timeout/termination path and the user-facing fix needs a changelog entry before merge.
