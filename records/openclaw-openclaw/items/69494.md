## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Fail closed when a parsed SecretRef is unresolved:** `apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift:347-356`
  - body: After `resolveConfigTokenSync` or `resolveConfigPasswordSync` parses a local SecretRef and the keychain resolver returns nil, this path still falls through to launchd token/password fallback. Current Gateway auth docs require explicit unresolved local SecretRefs to fail closed instead of being masked by fallback credentials, so return a configured-but-unresolved state and stop the fallback for parsed refs.
  - confidence: 0.86
- **[P1] Bound the keychain lookup on the proxy path:** `apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift:145-147`
  - body: `securityFindGenericPassword` calls `process.waitUntilExit()` with no timeout, and the browser proxy reaches it through synchronous `localConfig(..., secretResolver: .blocking)`. A locked or stalled keychain process can hang endpoint resolution before the request timeout applies; add a timeout/termination path or avoid blocking the sync proxy path.
  - confidence: 0.84
