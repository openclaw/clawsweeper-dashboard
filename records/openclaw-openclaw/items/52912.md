## Summary

Keep open: this is a plausible bug-fix PR, but the current patch broadens OAuth credential propagation across sibling agents, lacks contributor real behavior proof, and leaves parts of the linked Anthropic token flow unresolved.

## What This Changes

The PR routes plugin-provider auth profile writes through a shared sibling-sync helper, syncs Anthropic token profiles to sibling agent directories, adds auth-choice coverage, and adds a DeepSeek contract-registry import.

## Best Possible Solution

Land a narrower maintainer-approved Anthropic token persistence fix that covers the intended setup-token and paste-token surfaces, preserves explicit OAuth copy policy, and leaves unrelated registry changes out of the auth PR.

## Reproduction Assessment

Yes for the PR-introduced regression: source inspection shows the PR predicate makes every OAuth credential sibling-syncable and the generic plugin-auth path uses it. No live Anthropic setup-token flow was run, and current-main read-through makes the original gateway failure configuration-dependent.

## Solution Assessment

No. A shared persistence helper may be useful, but this implementation broadens OAuth credential copying and only partially covers the linked issue; a token-scoped fix with explicit policy boundaries is safer.
