## Summary

Keep open: the linked Anthropic token propagation bug is still present on current main, but this PR is not safe to merge because it broadens OAuth credential copying and only fixes one of the reported token persistence paths.

## What This Changes

The PR routes plugin-provider auth profile writes through a new sibling-sync helper, syncs Anthropic token and OAuth credentials to sibling agent directories, adds Anthropic token sibling-sync coverage, and updates an old bundled-provider contract registry.

## Best Possible Solution

Land a narrow, token-scoped auth-profile writer used by the Anthropic setup-token, paste-token, and non-interactive token flows; keep OAuth sibling copying explicit opt-in and split unrelated registry cleanup.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main writes plugin auth, models auth setup-token, paste-token, and Anthropic non-interactive setup-token profiles only to the selected agentDir, so sibling agent stores can remain missing.

## Solution Assessment

No. The PR moves one plugin-provider path toward sibling sync but broadens OAuth credential propagation and misses the other reported token entrypoints; a token-scoped shared persistence helper is safer.

## Review Findings

Overall correctness: patch is incorrect

  - body: `shouldSyncSiblingAgentsForCredential` returns true for every OAuth credential, so any plugin OAuth flow will copy refresh material into all sibling agent stores.   - confidence: 0.84
