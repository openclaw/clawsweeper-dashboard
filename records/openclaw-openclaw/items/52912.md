## Summary

Keep open. The PR addresses a real Anthropic auth propagation gap, but it should not merge as-is because it broadens OAuth credential fanout across agents, only partially covers the linked flow, is conflicting with current main, and lacks contributor real behavior proof.

## What This Changes

The PR routes plugin-provider auth profile writes through a shared helper, syncs Anthropic token credentials and all OAuth credentials to sibling agent directories, adds an Anthropic token sync contract test, and adds DeepSeek to the contract registry.

## Best Possible Solution

Revise or replace this with a token-scoped Anthropic setup-token/paste-token persistence fix that preserves OAuth opt-in portability, covers the intended entrypoints, and keeps registry cleanup separate.

## Reproduction Assessment

Yes for the PR-introduced regression: source inspection shows every OAuth credential becomes sibling-syncable and the generic plugin auth path applies that predicate. I did not run a live Anthropic setup-token flow, and the original gateway failure remains configuration-dependent because current main has default/main auth read-through.

## Solution Assessment

No. A shared persistence helper may be useful, but this implementation broadens OAuth copying; a narrower Anthropic token-scoped fix with explicit policy boundaries is safer.
