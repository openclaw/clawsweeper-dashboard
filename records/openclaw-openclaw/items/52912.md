## Summary

Keep open. The PR targets a real linked auth bug, but the current branch is not safe to merge because it broadens OAuth credential copying across sibling agent stores, lacks after-fix real behavior proof, and is stale/conflicting against current main.

## What This Changes

The PR routes plugin-provider auth-profile writes through a sibling-sync helper, syncs Anthropic token and OAuth credentials to sibling agent dirs, adds Anthropic token sync coverage, updates related mocks, and adds DeepSeek to the provider contract registry.

## Best Possible Solution

Rework or replace the branch with a narrow Anthropic token-focused persistence fix that preserves OAuth isolation, covers the intended setup-token and paste-token surfaces, and splits unrelated registry cleanup out of the auth PR.

## Reproduction Assessment

Yes for the PR-introduced regression: source inspection shows every OAuth credential becomes sibling-syncable and the generic plugin auth path applies that predicate. The original linked gateway failure is still configuration-dependent on current main because runtime read-through can mask missing physical sibling copies.

## Solution Assessment

No. A shared writer may be useful, but this implementation broadens OAuth copying and does not fully cover the linked paste-token, non-interactive setup-token, guidance, and validation surfaces.
