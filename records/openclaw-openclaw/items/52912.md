## Summary

Keep open: the patch still has a blocking credential-isolation regression and lacks real after-fix proof, while the linked bug remains an active implementation target rather than a cleanup close.

## What This Changes

The PR routes plugin-provider auth profile writes through a shared helper, syncs Anthropic token and OAuth credentials to sibling agent dirs, adds auth-choice coverage, and adds a DeepSeek contract-registry import.

## Best Possible Solution

Rework this as a narrow token-focused Anthropic setup-token/paste-token persistence fix that preserves explicit OAuth copy policy, removes unrelated registry churn, and includes real setup-token proof.

## Reproduction Assessment

Yes for the PR-introduced regression from source inspection: the predicate makes every OAuth credential sibling-syncable and the generic plugin-auth write path uses it. No live Anthropic setup-token flow was run, and current main read-through can make the original gateway failure configuration-dependent.

## Solution Assessment

No. A shared persistence helper may be useful, but this implementation broadens OAuth credential copying and only partially covers the linked issue; a token-scoped fix with explicit policy boundaries is safer.
