## Summary

Keep open: the PR still has a blocking credential-propagation issue. Current main now documents and implements read-through auth inheritance plus explicit OAuth portability rules, while this branch copies every OAuth credential to sibling agent stores and also includes an unrelated contract-registry change.

## What This Changes

The PR routes plugin auth profile writes through a new sibling-sync helper, adds Anthropic setup-token sibling-sync coverage, and imports DeepSeek into the contract registry.

## Best Possible Solution

Use the current read-through inheritance and explicit portability contract, then add only a token-scoped persistence fix if an Anthropic setup-token gap remains; do not copy OAuth refresh credentials by default.

## Reproduction Assessment

Yes. Source inspection of the PR diff shows the new predicate syncs every OAuth credential to sibling agents, and current main’s code/docs define OAuth refresh credentials as non-portable unless explicitly opted in.

## Solution Assessment

No. The PR is not the best fix because it broadens credential propagation beyond the Anthropic token case; a token-scoped helper or existing read-through inheritance is safer and matches current policy.
