## Summary

Keep open: the PR targets a real Anthropic token propagation bug, but the branch is dirty against current main and its helper would broaden OAuth refresh-token copying across sibling agent stores, conflicting with current credential portability policy.

## What This Changes

The PR routes plugin-provider auth profile writes through a shared helper, syncs Anthropic token and all OAuth credentials to sibling agent directories, adds Anthropic token sibling-sync coverage, and adds DeepSeek to the old contract registry list.

## Best Possible Solution

Land a narrower current-main auth persistence change that covers the intended Anthropic setup-token and paste-token contexts, preserves OAuth copying as explicit opt-in, and keeps unrelated registry cleanup separate.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current plugin auth, models auth setup-token, paste-token, and Anthropic non-interactive setup-token writes all persist to a single selected agent store. I did not run tests because this review is read-only.

## Solution Assessment

No. The PR moves one plugin auth path toward sibling sync but also copies all OAuth credentials and leaves reported token entrypoints uncovered; a token-scoped current-main helper is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restrict OAuth auth-profile sync to explicit opt-in:** `src/plugins/provider-auth-helpers.ts:222-223`
  - body: `shouldSyncSiblingAgentsForCredential` returns true for every OAuth credential, so any plugin OAuth flow would copy refresh material into all sibling agent stores. That conflicts with the documented portability policy that OAuth copying must be provider/context opt-in because refresh tokens can be rotation-sensitive.
  - confidence: 0.92
- **[P2] Persist token sync on all reported token entrypoints:** `src/plugins/provider-auth-choice.ts:133-135`
  - body: The linked bug includes `models auth setup-token`, `paste-token`, and Anthropic non-interactive setup-token surfaces, but this patch only changes `runProviderPluginAuthMethod`. The remaining direct upsert paths can still report success while leaving other configured agent stores without the intended token profile.
  - confidence: 0.84
