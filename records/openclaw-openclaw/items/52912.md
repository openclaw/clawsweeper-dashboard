## Summary

Keep open. This PR still targets a real Anthropic setup-token auth gap, but the current patch broadens sibling sync to every OAuth credential, conflicts with the documented portability boundary, and lacks real after-fix behavior proof.

## What This Changes

The branch routes plugin-provider auth profile writes through a shared sibling-sync helper, syncs Anthropic token credentials to sibling agent dirs, adds auth-choice coverage, and adds a DeepSeek registry import.

## Best Possible Solution

Rework or replace this with a token-scoped, maintainer-approved persistence and guidance fix that preserves OAuth isolation and covers the intended Anthropic setup-token and paste-token surfaces.

## Reproduction Assessment

Yes for the blocking PR regression from source inspection: the PR predicate makes every OAuth credential sibling-syncable, and the write path fans that into all discovered agent dirs. I did not run a live Anthropic setup-token flow in this read-only review.

## Solution Assessment

No. A shared persistence helper could be useful, but this implementation makes OAuth fanout generic; a narrower token-scoped sync or explicit provider/context opt-in is safer.
