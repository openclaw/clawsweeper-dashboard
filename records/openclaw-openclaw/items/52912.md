## Summary

Keep open: current main does not implement the PR's intended sibling sync, and the branch has blocking credential-propagation and incomplete command-surface coverage concerns before merge.

## What This Changes

The PR routes plugin provider auth profile writes through a new sibling-sync helper, syncs Anthropic token and all OAuth credentials to sibling agent directories, adds Anthropic token sync coverage, and adds a DeepSeek contract-registry import.

## Best Possible Solution

Land a narrower auth persistence change that covers the intended Anthropic token setup paths, keeps OAuth sibling copying explicitly opt-in, and leaves unrelated registry cleanup out of the auth PR.

## Reproduction Assessment

Yes. The PR diff itself reproduces the main regression risk because OAuth credentials become sibling-syncable, while current docs and portability code keep OAuth non-portable by default; the linked command surfaces remain traceable in current models auth and Anthropic setup-token code.

## Solution Assessment

No. The safer approach is a token-scoped persistence helper used by plugin auth, models setup-token/paste-token, and Anthropic non-interactive setup where needed, with OAuth copying gated by explicit provider/context opt-in.
