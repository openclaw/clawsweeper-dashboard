## Summary

Keep open. The PR targets a real linked auth bug, but it should not merge as-is because it makes every OAuth credential sibling-syncable, conflicts with the documented OAuth portability boundary, is merge-conflicting, and still lacks contributor real behavior proof.

## What This Changes

The PR routes plugin-provider auth-profile writes through a sibling-sync helper, syncs Anthropic token and OAuth credentials to sibling agent dirs, adds Anthropic token sync coverage, updates related unit mocks, and adds DeepSeek to the provider contract registry.

## Best Possible Solution

Rework or replace the branch with a token-scoped Anthropic setup-token and paste-token persistence fix that preserves OAuth isolation, covers the intended entrypoints, and leaves unrelated registry cleanup separate.

## Reproduction Assessment

Yes for the PR-introduced regression: source inspection shows every OAuth credential becomes sibling-syncable and the generic plugin auth path applies that predicate. The original gateway failure remains configuration-dependent on current main because runtime read-through can mask missing physical sibling copies.

## Solution Assessment

No. A shared persistence helper may be useful, but this implementation broadens OAuth copying; a narrower Anthropic token-scoped fix with explicit policy boundaries is safer.
