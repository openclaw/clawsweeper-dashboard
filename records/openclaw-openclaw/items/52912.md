## Summary

Keep open: this is a plausible implementation path for the linked Anthropic setup-token bug, but the patch currently broadens OAuth credential propagation, lacks real behavior proof, and leaves the linked issue’s paste-token/non-interactive/guidance surfaces unresolved.

## What This Changes

The PR routes plugin-provider auth writes through a new helper, sibling-syncs Anthropic token and OAuth credentials, adds token-sync regression coverage, and adds DeepSeek to the provider contract registry.

## Best Possible Solution

Land a narrower maintainer-approved auth fix that covers intended Anthropic setup-token and manual-token persistence, preserves OAuth opt-in copying, and updates validation or guidance for the supported flow.

## Reproduction Assessment

Yes for the PR-introduced defect: source inspection of the PR head shows every OAuth credential is treated as sibling-syncable and the generic plugin-provider auth path uses that predicate. The original gateway failure remains tied to the open linked issue and is configuration-dependent on current main.

## Solution Assessment

No. A shared writer may be useful, but this implementation expands OAuth refresh-token copying; the safer fix is token-focused and explicit about which auth surfaces sync or inherit credentials.
