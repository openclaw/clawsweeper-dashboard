## Summary

Keep open: this PR targets a real linked Anthropic auth gap, but the current patch broadens OAuth credential copying across sibling agents, lacks real behavior proof, and leaves parts of the linked setup-token/paste-token problem unresolved.

## What This Changes

The PR routes plugin auth-profile writes through a sibling-sync helper, syncs Anthropic token and all OAuth credentials to sibling agent directories, adds Anthropic token sync coverage, updates plugin-provider auth mocks, and adds DeepSeek to the provider contract registry.

## Best Possible Solution

Land a narrower maintainer-approved auth persistence fix that covers the intended Anthropic setup-token and paste-token paths, preserves OAuth opt-in portability, adds focused regression coverage, and updates guidance or validation where needed.

## Reproduction Assessment

Yes for the PR-introduced regression: source inspection of the PR head shows every OAuth credential becomes sibling-syncable and the generic plugin auth path applies that predicate. The original gateway failure remains configuration-dependent on current main because read-through inheritance can mask missing physical sibling copies.

## Solution Assessment

No. The shared writer concept may be useful, but this patch applies sibling sync too broadly to OAuth credentials and misses linked paste-token, non-interactive setup-token, guidance, and validation surfaces.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restrict OAuth auth-profile sync to intended contexts:** `src/plugins/provider-auth-helpers.ts:224`
