## Summary

Keep open, but do not merge as-is. The PR addresses a real auth-sync area, but the diff broadens OAuth credential fanout contrary to current portability policy, lacks real behavior proof, and should be revised or replaced with a narrower maintainer-approved fix.

## What This Changes

The PR routes plugin-provider auth profile writes through a new sibling-sync helper, syncs Anthropic token and all OAuth credentials to sibling agent dirs, adds Anthropic token contract coverage, and adds DeepSeek to the contract registry.

## Best Possible Solution

Revise or replace this with a narrow token-scoped auth persistence and guidance fix that preserves OAuth opt-in portability, respects current read-through inheritance, and keeps unrelated registry cleanup separate.

## Reproduction Assessment

Yes for the blocking PR regression from source inspection: the PR predicate makes every OAuth credential trigger sibling sync, while current docs and portability code keep OAuth non-portable by default. I did not run a live Anthropic setup-token flow because this review is read-only.

## Solution Assessment

No. A shared persistence helper could be useful, but this implementation makes OAuth fanout generic; the safer solution is token-scoped sync or explicit provider/context opt-in after maintainer security policy review.
