The PR routes plugin auth-profile writes through a shared sibling-sync helper, syncs Anthropic token and all OAuth plugin-auth credentials to sibling agent directories, updates auth-choice tests, and adds DeepSeek to the provider contract registry.

## Best Possible Solution

Land or replace this with a narrower maintainer-approved token-focused auth fix that preserves OAuth opt-in portability, covers the intended setup-token and paste-token paths, and includes redacted real behavior proof.

## Reproduction Assessment

Yes for the PR-introduced defect: PR-head source marks every OAuth credential sibling-syncable and generic plugin auth uses that result. The original linked gateway failure is less direct on current main because runtime read-through can mask missing physical sibling copies.

## Solution Assessment

No. A shared writer may be useful, but this implementation expands OAuth credential copying and does not deliberately cover all linked setup-token surfaces; a narrower token-focused fix is safer.
