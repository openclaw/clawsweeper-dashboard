## Summary

Keep open; the PR is not safe to merge because its sync predicate copies every OAuth credential across sibling agents, conflicting with the current OAuth portability policy, and it does not cover the models auth setup-token/paste-token paths described in #51911.

## What This Changes

The PR routes plugin provider auth profile writes through a shared writer, syncs Anthropic token and OAuth credentials to sibling agent directories, adds Anthropic token sync coverage, and adds a DeepSeek contract-registry import.

## Best Possible Solution

Land a narrower maintainer-approved auth persistence change that covers Anthropic setup-token/paste-token sibling behavior, preserves OAuth isolation/read-through semantics, and removes unrelated registry work.

## Reproduction Assessment

Yes. The PR diff itself reproduces the main regression risk through the every-OAuth sync predicate, and current-main source shows the linked models auth setup-token/paste-token paths still write one agent store.

## Solution Assessment

No. The safer solution is a token-scoped helper or explicit provider/context opt-in for the intended Anthropic token flows, with OAuth copying gated by provider policy and tests covering the CLI entrypoints named in #51911.
