## Summary

Keep open. The PR still has a blocking credential-propagation bug, lacks after-fix real behavior proof, and is paired with the open linked issue that covers broader setup-token and paste-token follow-up.

## What This Changes

The PR routes plugin-provider auth-profile writes through a sibling-sync helper, syncs Anthropic token and all OAuth credentials to sibling agent directories, adds Anthropic token sync coverage, updates related mocks, and adds DeepSeek to the provider contract registry.

## Best Possible Solution

Revise or replace this with a narrow maintainer-approved token persistence and guidance fix that preserves OAuth isolation, covers the intended setup-token and paste-token surfaces, and splits unrelated registry cleanup out of the auth change.

## Reproduction Assessment

Yes for the PR-introduced regression: source inspection of the PR head shows every OAuth credential becomes sibling-syncable and the generic plugin auth path applies that predicate. The original linked gateway failure remains configuration-dependent on current main because read-through inheritance can mask missing physical sibling copies.

## Solution Assessment

No. A shared writer may be useful, but this implementation broadens OAuth copying and does not fully cover the linked paste-token, non-interactive setup-token, guidance, and validation surfaces.
