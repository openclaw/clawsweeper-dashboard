## Summary

Keep open. Current main still acknowledges ordinary Telegram callback queries inside the downstream callback handler after the grammY sequentializer, and the merged approval-lane work only covers approval payloads, not general inline buttons.

## What This Changes

Telegram callback query acknowledgements should happen before the per-chat/per-topic sequentializer so inline button presses do not time out behind an active agent turn.

## Best Possible Solution

Add a Telegram plugin middleware before botRuntime.sequentialize(getTelegramSequentialKey) that acknowledges callback queries immediately, logs acknowledgement failures without blocking the pipeline, and leaves the actual callback action processing sequentialized.

## Reproduction Assessment

Yes. The source-level reproduction is to hold a same-chat or same-topic Telegram turn in the sequentializer and submit an ordinary inline-button callback; current keying maps that callback to the same lane while the acknowledgement is inside the delayed handler.

## Solution Assessment

Yes. The narrow maintainable fix is to acknowledge callback queries before sequentialize while preserving ordered downstream callback behavior; the approval-only lane from #64979 is useful but not sufficient for ordinary callbacks.

## Repair Work Prompt

Fix openclaw/openclaw#42156 in the Telegram plugin. Current main acknowledges `callback_query` updates inside `extensions/telegram/src/bot-handlers.runtime.ts` after `botRuntime.sequentialize(getTelegramSequentialKey)` is registered in `extensions/telegram/src/bot-core.ts`, so ordinary inline-button callbacks can wait behind an active same-chat/topic turn and miss Telegram's answer window. Add middleware in `createTelegramBotCore` before the sequentializer that detects callback queries and acknowledges them immediately using `ctx.answerCallbackQuery()` when available or `bot.api.answerCallbackQuery(id)` as a fallback. Catch/log acknowledgement failures without blocking `next()`. Keep actual callback action processing, authorization, dedupe, approval handling, model callbacks, native command callbacks, and callback routing in the downstream ordered handler. Remove or avoid redundant handler-level acknowledgement so duplicate API errors are not introduced. Do not add config, toast text, or broader UX changes; #76622 is separate. Add/update regression coverage proving an ordinary same-chat callback is acknowledged before its action waits behind a busy sequentialized turn, and keep the existing approval-lane behavior from #64979 intact. Add a user-facing changelog fix entry, crediting the human contributor if repo policy allows.
