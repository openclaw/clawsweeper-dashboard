## Summary

Keep open. Current main and the latest release still acknowledge ordinary Telegram callback queries only inside the queued callback handler, while the related closed PRs were never merged and the open callback-ack-text PR does not move the acknowledgement before sequentialize.

## What This Changes

Telegram inline button callbacks should be acknowledged before per-chat/topic sequentialization so button spinners do not time out during an active agent turn.

## Best Possible Solution

Add an accepted-update, pre-sequentialize Telegram callback acknowledgement path for ordinary callbacks, keep callback action routing ordered in the existing handler, and cover the timing with focused regression tests.

## Reproduction Assessment

Yes, at source level. Current main puts ordinary callback-query handling behind the same sequential key as an active turn, and the only acknowledgement call is inside that queued handler; I did not run a live Telegram client reproduction.

## Solution Assessment

Yes. The narrow maintainable fix is to move only the callback acknowledgement before sequentialize after update acceptance/dedupe, while leaving the actual callback routing in the existing ordered handler and preserving sanitized logging and throttling behavior.
