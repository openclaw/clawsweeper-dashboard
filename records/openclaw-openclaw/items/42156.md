## Summary

Keep open. Current main and the latest release still acknowledge ordinary Telegram callback queries downstream of the per-chat/topic sequentializer, while the shipped approval-lane fix only covers approval payloads and the open toast-feedback PR tracks a different UX request.

## What This Changes

The issue requests that Telegram inline-button callback queries be acknowledged before same-chat/topic agent turns can queue them behind `sequentialize`.

## Best Possible Solution

Add an accepted-update, pre-sequentialize callback acknowledgement path while keeping callback action routing in the ordered handler, with focused regression coverage and redacted live Telegram proof.

## Reproduction Assessment

Yes, at source level. A normal callback query shares the chat/topic sequential key, `@grammyjs/runner` waits on same-constraint updates, and current `answerCallbackQuery` runs inside the downstream handler.

## Solution Assessment

Yes. The narrow maintainable fix is to move only the acknowledgement timing before `sequentialize` after accepted-update gating, without changing callback action routing or adding config.
