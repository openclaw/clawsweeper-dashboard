## Summary

Keep open. Current main and the latest release still acknowledge ordinary Telegram callback queries inside the downstream callback handler after the per-chat/topic sequentializer; the shipped approval-lane fix only covers approval callbacks, and the open ack-text PR targets separate toast feedback.

## What This Changes

The issue reports that Telegram inline-button callback acknowledgements can time out when queued behind the per-chat/topic `sequentialize` middleware during an active agent turn.

## Best Possible Solution

Add an accepted-update, pre-sequentialize callback acknowledgement path while keeping callback action routing ordered in the existing handler, then prove it with focused tests and redacted live Telegram proof.

## Reproduction Assessment

Yes, at source level. A normal callback query shares the chat/topic sequential key, `@grammyjs/runner` waits before `next()`, and current `answerCallbackQuery` runs inside the downstream handler.

## Solution Assessment

Yes. The narrow maintainable fix is to move only the acknowledgement timing before `sequentialize` after accepted-update gating, without changing callback action routing or adding config.
