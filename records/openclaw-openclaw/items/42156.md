## Summary

Keep open: current main still acknowledges Telegram callback queries only inside the queued callback handler, so inline button presses in an already-busy chat/topic can wait behind `sequentialize` before Telegram's progress bar is cleared.

## What This Changes

Acknowledge Telegram callback queries before the per-chat/topic grammY `sequentialize` queue while preserving ordered callback action processing.

## Best Possible Solution

Move only callback acknowledgement into a small post-dedupe, pre-sequentialize middleware, keep callback action routing ordered in the existing handler, and add focused regression coverage plus a user-facing changelog entry.

## Reproduction Assessment

Yes, from source inspection. A callback in the same chat/topic receives the same sequential key as the active turn, while the only acknowledgement call remains inside the queued callback handler; I did not run a live Telegram bot session.

## Solution Assessment

Yes. The narrow maintainable fix is to acknowledge callbacks after accepted-update dedupe but before `sequentialize`, without changing callback data parsing, action routing, or the separate UX request in #76622.
