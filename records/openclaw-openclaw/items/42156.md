## Summary

Keep open. Current main still acknowledges ordinary Telegram callback queries only inside the downstream callback handler, so same-chat/topic callbacks can sit behind `sequentialize`; the related pre-sequentialize fix attempts are closed unmerged, and the open callback-ack-text PR does not fix this timing path.

## What This Changes

Requests acknowledging Telegram inline-button callback queries before grammY `sequentialize` can queue them behind active same-chat/topic agent turns.

## Best Possible Solution

Add an accepted-update, pre-sequentialize callback acknowledgement path for ordinary Telegram callbacks, keep callback action routing ordered in the existing handler, and cover the timing with focused regression and live Telegram proof.

## Reproduction Assessment

Yes, at source level. A same-chat/topic callback reaches `answerCallbackQuery` only after `sequentialize` calls the downstream callback handler, and upstream `sequentialize` waits on the previous same-key turn before `next()`.

## Solution Assessment

Yes. The narrow maintainable fix is to acknowledge accepted callback updates before `sequentialize` while leaving callback routing downstream and preserving dedupe, sanitized logging, and throttling behavior.

Fix https://github.com/openclaw/openclaw/issues/42156 with a narrow Telegram plugin patch. Current main calls `answerCallbackQuery` only inside `extensions/telegram/src/bot-handlers.runtime.ts` after `sequentialize`, so ordinary same-chat/topic callback queries can time out while an agent turn is active. Add an accepted-update pre-sequentialize acknowledgement path in `extensions/telegram/src/bot-core.ts` after `updateTracker.beginUpdate(ctx)` accepts the update and before `botRuntime.sequentialize(getTelegramSequentialKey)` can queue it. Remove or adjust redundant handler-level ACK only if tests prove duplicate ACKs are avoided without changing callback routing. Add focused regression coverage showing ACK happens before a blocked same-lane callback handler, plus non-callback pass-through and ACK-error resilience. Add a single-line user-facing changelog fix entry; contributor credit should follow repo policy and not thank bots.
