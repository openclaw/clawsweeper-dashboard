## Summary

Keep open. Current main still acknowledges ordinary Telegram callback queries only after the update has passed through the per-chat/topic sequentializer, while the prior timing-fix PRs were closed unmerged and the open callback-ack text PR targets a different UX gap.

## What This Changes

Requests acknowledging Telegram inline-button callback queries before `sequentialize` so button spinners clear while an agent turn is busy in the same chat/topic.

## Best Possible Solution

Add an accepted-update, pre-sequentialize callback acknowledgement path, keep callback action routing ordered in the existing handler, and prove it with focused tests plus redacted live Telegram proof.

## Reproduction Assessment

Yes, at source level. A normal callback query shares the chat/topic sequential key, upstream `sequentialize` waits before `next()`, and current `answerCallbackQuery` is inside the downstream handler.

## Solution Assessment

Yes. The narrow maintainable fix is to move only the acknowledgement timing before `sequentialize` after accepted-update gating, without changing callback routing or adding a config surface.
