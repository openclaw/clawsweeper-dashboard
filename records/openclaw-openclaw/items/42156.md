- `impact:message-loss`: Queued callback acknowledgements can time out before OpenClaw responds, making Telegram inline-button interactions appear lost.

## Summary

Keep open. Current main and the latest release still answer ordinary Telegram callback queries inside the downstream callback handler after the per-chat/topic sequentializer; the shipped approval-lane fix only covers approval payloads, and the open acknowledgement-text PR tracks separate toast feedback.

## What This Changes

Telegram should acknowledge inline-button callback queries before per-chat/topic sequentialization so button spinners clear while an agent turn is already busy.

## Best Possible Solution

Add an accepted-update, pre-sequentialize callback acknowledgement path while keeping callback action routing in the existing ordered handler, then prove it with focused tests and redacted live Telegram proof.

## Reproduction Assessment

Yes, at source level. A normal callback query shares the chat/topic sequential key, `@grammyjs/runner` waits before `next()`, and current `answerCallbackQuery` runs inside the downstream handler.

## Solution Assessment

Yes. The narrow maintainable fix is to move only the acknowledgement timing before `sequentialize` after accepted-update gating, without changing callback action routing or adding config.
