## Summary

Keep open: current main still installs Telegram `sequentialize` before the callback handler, and `answerCallbackQuery` still runs inside that downstream handler. The related fix PRs that directly targeted this issue were closed unmerged; newer open PRs touch adjacent callback UX but do not provide a clean canonical fix for this timeout.

## What This Changes

The issue reports that Telegram inline-button callback acknowledgements are delayed behind per-chat sequentialization, causing callback queries to time out while another agent turn is active.

## Best Possible Solution

Move only the empty callback acknowledgement into middleware before `sequentialize`, skip any duplicate downstream acknowledgement after early success, keep callback routing ordered, and add focused tests plus live Telegram proof.

## Reproduction Assessment

Yes, at source level: current main installs `sequentialize` before the callback handler, and ordinary callback queries use the same chat/topic key before `answerCallbackQuery` runs. I did not run a live Telegram client reproduction in this read-only review.

## Solution Assessment

Yes. The narrow best fix is to acknowledge the callback before `sequentialize` while preserving ordered callback handling downstream; broader final-button special cases or callback-ack text config solve different UX problems.
