## Summary

Keep open: current main and the latest release still acknowledge Telegram callback queries downstream of the same-key sequentializer, so the reported timeout path remains. The directly related fix PRs are closed unmerged, and the remaining open related PR addresses callback acknowledgement text rather than the queued-timeout bug.

## What This Changes

Telegram inline-button callback acknowledgements can time out because answerCallbackQuery is delayed behind grammY sequentialize when a same-chat/topic turn is already running.

## Best Possible Solution

Move only the empty callback acknowledgement into pre-sequentialize middleware, mark successful early acknowledgements to avoid duplicate Bot API calls, keep callback routing ordered, and add focused tests plus redacted live Telegram proof.

## Reproduction Assessment

Yes at source level: current main installs sequentialize before the callback handler, ordinary callback queries use the same chat/topic key, and runner waits before invoking downstream middleware. I did not run a live Telegram client reproduction in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to acknowledge the callback before sequentialize while preserving ordered callback routing downstream; callback toast text config solves a different UX problem.
