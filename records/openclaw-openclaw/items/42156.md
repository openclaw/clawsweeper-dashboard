## Summary

Keep open. Current main and the latest release still answer ordinary Telegram callback queries inside the downstream callback handler after `sequentialize`, so same-chat/topic callbacks can miss Telegram's acknowledgement window; the related timing-fix PRs at https://github.com/openclaw/openclaw/pull/42181 and https://github.com/openclaw/openclaw/pull/42516 were closed unmerged, while https://github.com/openclaw/openclaw/pull/76975 addresses acknowledgement text, not this queueing path.

## What This Changes

Telegram inline-button callback acknowledgements can time out when their updates are queued behind an active same-chat/topic agent turn.

## Best Possible Solution

Add an accepted-update, pre-sequentialize acknowledgement path for callback queries, keep callback action routing ordered in the existing handler, and prove the fix with focused tests plus live Telegram proof.

## Reproduction Assessment

Yes, at source level. A same-chat/topic callback reaches `answerCallbackQuery` only after `sequentialize` calls the downstream callback handler, and `@grammyjs/runner` waits for the previous same-key task before calling `next()`.

## Solution Assessment

Yes. The narrow maintainable fix is to acknowledge accepted callback updates in the existing pre-sequentialize update middleware while leaving routing, authorization, dedupe, sanitized logging, and throttling behavior intact.
