- `impact:message-loss`: The issue is about Telegram inline-button callback acknowledgement being delayed until Telegram times out, making the user-visible callback interaction fail.

## Summary

Keep open: current main and v2026.6.6 still acknowledge Telegram callback queries inside the downstream callback handler after `sequentialize`, and the related fix PRs were closed unmerged rather than landing a canonical fix.

## What This Changes

The issue requests moving Telegram inline-button callback acknowledgement ahead of per-chat/topic sequentialization so button presses do not time out behind a running agent turn.

## Best Possible Solution

Move only the empty callback acknowledgement into middleware before `sequentialize`, skip any duplicate downstream acknowledgement after early success, keep callback routing ordered, and add focused unit plus live Telegram proof.

## Reproduction Assessment

Yes, at source level: current main installs the same-chat/topic sequentializer before the callback handler, and ordinary callback queries use that same sequential key before `answerCallbackQuery` runs. I did not run a live Telegram client reproduction in this read-only review.

## Solution Assessment

Yes, the narrow best fix is to acknowledge the callback before `sequentialize` while preserving ordered callback handling downstream. Bypassing the sequentializer for all callbacks or adding callback-ack text config would solve different problems and risk broader behavior drift.
