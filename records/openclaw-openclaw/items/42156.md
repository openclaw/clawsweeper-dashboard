- `impact:message-loss`: Queued callback queries can time out before OpenClaw acknowledges the button press, making inline-button interactions appear lost to the Telegram user.

## Summary

Keep open. Current main still answers ordinary Telegram callback queries inside the downstream callback handler after the per-chat/topic sequentializer, while the merged approval-lane fix only covers approval callbacks and the open ack-text PR targets a separate toast-copy gap.

## What This Changes

The issue asks OpenClaw to acknowledge Telegram inline-button callback queries before `sequentialize` so button spinners clear even while a same-chat/topic agent turn is running.

## Best Possible Solution

Add an accepted-update, pre-sequentialize callback acknowledgement path while keeping callback action routing ordered in the existing handler, then prove it with focused tests and redacted live Telegram proof.

## Reproduction Assessment

Yes, at source level. A normal callback query shares the chat/topic sequential key, upstream `sequentialize` waits before `next()`, and current `answerCallbackQuery` runs inside the downstream handler.

## Solution Assessment

Yes. The narrow maintainable fix is to move only the acknowledgement timing before `sequentialize` after accepted-update gating, without changing callback routing or adding a config surface.
