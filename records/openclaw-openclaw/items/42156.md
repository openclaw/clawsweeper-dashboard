- `impact:message-loss`: Queued callback acknowledgements can time out before OpenClaw responds, making inline-button interactions appear lost or stuck to Telegram users.

## Summary

Keep open. Current main and the latest release still answer ordinary Telegram callback queries inside the downstream `callback_query` handler after the per-chat/topic sequentializer, and the related closed PRs were never merged.

## What This Changes

The issue reports Telegram inline-button callback acknowledgements timing out when callback updates wait behind the same chat/topic sequential queue as an active agent turn.

## Best Possible Solution

Add an accepted-update, pre-sequentialize callback acknowledgement path while preserving ordered callback action routing, sanitized Telegram API error logging, focused regression coverage, changelog, and redacted live Telegram proof.

## Reproduction Assessment

Yes, at source level. A normal callback query shares the chat/topic sequential key, the pinned `@grammyjs/runner` sequentializer waits on same-constraint updates before `next()`, and current `answerCallbackQuery` runs inside the downstream handler.

## Solution Assessment

Yes. The narrow maintainable fix is to move only callback acknowledgement timing before `sequentialize` after accepted-update gating, without changing callback action routing or adding a config surface.
