## Summary

Keep open: the patch is a plausible narrow bug fix, but the PR cannot pass merge review yet because the only supplied proof is mocked unit-test output and the behavior change leaves nearby docs wording stale.

## What This Changes

The PR changes subagent output selection to suppress latest raw tool/toolResult text for successful `ok` runs and adds regressions that preserve raw output for failed runs.

## Best Possible Solution

Merge a narrow selector-level guard with regression coverage, aligned docs, and real runtime proof that successful child runs without assistant final text no longer report raw tool output.

## Reproduction Assessment

Yes at source level: current main returns `snapshot.latestRawText` for an `ok` child history with no final assistant text, and the announce path passes that `ok` outcome into the selector. I did not establish a live runtime reproduction in this read-only review.

## Solution Assessment

Yes, the selector-level guard is the narrowest maintainable fix because it preserves assistant output and non-`ok` diagnostics while blocking only successful raw-tool fallback. The remaining work is proof and documentation alignment, not a different code shape.
