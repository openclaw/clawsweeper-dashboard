## Summary

Keep open: the PR addresses a source-reproducible session-pin fallback gap, but the current diff broadens global empty override semantics that current docs and tests define as exact.

## What This Changes

The PR changes model fallback candidate construction so an empty fallback override appends the configured primary, and updates agent fallback tests for the session-pin case.

## Best Possible Solution

If maintainers want session pins to retain a safety net, implement a session-pin-scoped candidate expansion and update docs, changelog, and regression coverage while preserving exact empty overrides elsewhere.

## Reproduction Assessment

Yes. By source inspection, a user session model override returns [] in src/agents/agent-scope.ts, and current fallback candidate construction then leaves only the pinned candidate; the PR body also provides local gateway debug output after the proposed fix.

## Solution Assessment

No. The global empty-override change is broader than the reported session-pin case and conflicts with the documented contract; a safer fix would scope configured-primary retry to the accepted session-pin policy path.
