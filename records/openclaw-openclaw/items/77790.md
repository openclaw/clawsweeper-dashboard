## Summary

Keep open: the linked fallback collapse is source-reproducible, but this PR currently has a test defect, broadens the global empty-override contract that current docs/tests rely on for strict selections, and still lacks real runtime proof.

## What This Changes

The PR changes fallback candidate construction so an explicitly empty `fallbacksOverride` also appends the configured primary, and updates agent fallback tests for session-pinned models.

## Best Possible Solution

Decide the session-pin fallback policy first; if maintainers want configured-primary recovery, implement it in the session-pin path while preserving exact empty-override semantics for other callers, then update docs, changelog, regression coverage, and real runtime proof.

## Reproduction Assessment

Yes, by source inspection: user session overrides return `[]` in `resolveEffectiveModelFallbacks`, and current candidate construction leaves only the requested model. I did not run tests or live sessions because this review is read-only.

## Solution Assessment

No. The PR fixes the visible pin-collapse path by changing all explicit empty overrides; a narrower maintainable fix would distinguish session-pin recovery from callers that intentionally pass `[]` to disable hidden fallback.
