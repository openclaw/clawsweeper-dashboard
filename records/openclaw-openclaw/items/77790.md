## Summary

Keep open: the PR is a focused attempt at the linked fallback report, but the current patch still has a test compile/runtime defect, broadens documented empty-override semantics, and supplies only targeted Vitest output rather than real OpenClaw behavior proof.

## What This Changes

The PR changes model fallback candidate construction to append the configured primary when `fallbacksOverride` is undefined or empty, and updates agent fallback and command tests for session-pinned models.

## Best Possible Solution

If maintainers want configured-primary recovery for pinned fallback models, scope it to that session-pin path, keep other explicit empty overrides strict, and update docs, changelog, regression tests, and runtime proof together.

## Reproduction Assessment

Yes, by source inspection: user session model overrides return `[]`, and current candidate construction treats any defined override as exact, leaving only the pinned model. I did not run tests or live sessions because this review is read-only.

## Solution Assessment

No. The PR fixes the visible pin-collapse path by changing all explicit empty overrides, but the safer maintainable path is a session-pin-specific policy change if maintainers approve it.
