## Summary

Keep open: the reported collapse is source-reproducible on current main, but this patch broadens the global empty-override contract and conflicts with current docs/tests that treat empty overrides and user session model selections as exact; it also still needs real behavior proof before merge.

## What This Changes

The PR changes model fallback candidate construction so an empty fallback override appends the configured primary, and updates agent fallback tests for the session-pin case.

## Best Possible Solution

Decide the session-pin policy explicitly; if accepted, add a scoped fallback candidate only for that path while preserving exact empty overrides for other callers and updating docs/regression tests.

## Reproduction Assessment

Yes. By source inspection, a user session model override returns [] at src/agents/agent-scope.ts:217, and resolveFallbackCandidates then skips configured-primary append at src/agents/model-fallback.ts:596, leaving one candidate and fallbackConfigured:false.

## Solution Assessment

No. The global empty-override change is not the narrowest maintainable fix; a safer path would preserve fallbacksOverride: [] as exact and scope any configured-primary retry to the session-pin case after maintainer policy approval.
