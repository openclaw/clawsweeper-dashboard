## Summary

Keep open: the session-pin collapse is source-reproducible, but this PR changes global empty `fallbacksOverride` semantics contrary to current docs, tests, and exact-override callers, so it needs maintainer policy review before merge.

## What This Changes

The branch widens `resolveFallbackCandidates` so an empty `fallbacksOverride` appends the configured primary, and updates agent fallback tests for session-pinned models.

## Best Possible Solution

Decide the session-pin fallback policy first; if maintainers want a safety net, implement a session-pin-scoped expansion that preserves exact empty overrides elsewhere, with docs, changelog, regression coverage, and real runtime proof.

## Reproduction Assessment

Yes, by source inspection: user session model overrides resolve to `fallbacksOverride: []`, and current candidate construction then leaves only the pinned model. I did not run tests because this review is read-only.

## Solution Assessment

No. The PR solves the specific session-pin case by changing the global empty-override contract; a safer maintainable solution would scope any configured-primary retry to the accepted session-pin policy path.
