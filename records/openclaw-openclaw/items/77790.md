## Summary

Keep open: the reported fallback collapse is source-reproducible, but this PR broadens the meaning of every empty `fallbacksOverride` and would make exact override paths fall through to the configured primary. It also needs after-fix real behavior proof before merge.

## What This Changes

The PR changes model fallback candidate construction so an explicitly empty fallback override also appends the configured primary, and updates model fallback tests for the session-pin collapse case.

## Best Possible Solution

Preserve `fallbacksOverride: []` as an exact disable and add the configured primary only for the user session-pin collapse path, with regression coverage for both the pinned-session case and an exact override caller such as memory flush.

## Reproduction Assessment

Yes. By source inspection, a user session model override returns `fallbacksOverride: []`, and current `resolveFallbackCandidates` then keeps only the effective primary, so pinning to the configured fallback can collapse the chain to one candidate.

## Solution Assessment

No. The proposed fix addresses the session-pin collapse, but it does so by changing the global contract for empty fallback overrides; a narrower fix should pass or inject the configured primary only for that session-pin case.
