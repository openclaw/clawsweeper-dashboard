## Summary

Keep open. The PR targets a valid Slack preview-streaming bug, but the patch is not merge-ready because it calls Slack draft `clear()` for mid-turn tool-bullet transitions even though `clear()` permanently stops that draft stream, so later narrative preview updates on the same stream are dropped.

## What This Changes

The PR deletes Slack tool-progress bullet draft previews at boundaries/finalization, preserves narrative drafts as permanent messages, enables preview streaming for top-level DMs, and updates Slack streaming eligibility tests.

## Best Possible Solution

Keep the Slack tool-progress preview ephemeral, but delete bullet messages through a non-terminal draft operation or a fresh draft-stream instance so narrative preview streaming can continue after tool bursts.

## Reproduction Assessment

Yes, from source inspection. Current main can preserve a `Working…` bullet draft via `forceNewMessage()` at a boundary, and the PR's fix path can be traced to a stopped draft stream after `clear()` without needing live Slack credentials.

## Solution Assessment

No. The intended direction is good, but reusing a Slack draft stream after `clear()` is not the right implementation because `clear()` is terminal; the safer fix is a non-terminal delete/reset operation or recreating the draft stream before later narrative updates.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep the draft stream usable after clearing bullets:** `extensions/slack/src/monitor/message-handler/dispatch.ts:996`
  - body: `draftStream.clear()` is terminal for Slack drafts: it calls `discardPending()`, which calls `stop()`, and the draft loop ignores every later `update()` while stopped. After a tool-progress bullet preview transitions back to narrative, this line clears the bullet message and then the same stopped `draftStream` is reused for the narrative update, so the next agent text segment no longer streams or gets preserved by the draft path. Use a non-terminal delete/reset operation or create a fresh draft stream before continuing.
  - confidence: 0.96
