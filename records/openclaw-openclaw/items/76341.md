## Summary

Keep open. The latest commit addressed the earlier stopped-stream issue and Slack DM docs conflict, but the new delete helper still has a source-level race around in-flight draft sends that can leave the `Working…` preview behind; the user-facing Slack fix also still needs a changelog entry.

## What This Changes

The PR changes Slack preview streaming so non-off modes include top-level DMs and tool-progress `Working…` bullet drafts are deleted at transitions while narrative text remains permanent, with docs and eligibility tests adjusted.

## Best Possible Solution

Keep the ephemeral Slack tool-progress direction, but delete/reset bullet previews through a path that drains in-flight draft sends without stopping the reusable stream, then cover tool-to-final and tool-to-text races and add the required changelog entry.

## Reproduction Assessment

Yes, by source inspection. A `Working…` draft send can be in flight when the PR helper captures empty ids, resets the draft, clears `currentDraftKind`, and then waits only after the final/cleanup delete opportunity has passed.

## Solution Assessment

No. The requested behavior is reasonable, but the helper needs to wait for or flush the draft loop before resetting/deleting; otherwise the branch still has an orphan-preview race and lacks the required changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Drain in-flight bullet sends before resetting the draft:** `extensions/slack/src/monitor/message-handler/dispatch.ts:962-965`
  - body: `deleteBulletPreview()` reads the current ids and immediately calls `forceNewMessage()` without waiting for the draft loop's in-flight send/edit. If a `Working…` send is still in flight when finalization or cleanup reaches this helper, no `chat.delete` is issued, `currentDraftKind` is cleared, and the send can complete afterward as the same orphan preview this PR is trying to remove. Flush or otherwise wait for the in-flight draft operation before resetting/deleting while keeping the stream reusable.
  - confidence: 0.88
