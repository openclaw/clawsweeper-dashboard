## Summary

Keep open. The latest PR head addresses the prior bullet-delete race and changelog gap, but the new narrative-to-bullets transition can still lose or overwrite an agent text segment before it becomes permanent.

## What This Changes

The PR changes Slack preview streaming to enable top-level DM draft previews, delete tool-progress bullet drafts on transitions/final cleanup, update Slack docs/tests/changelog, and adjust raw channel-fetch allowlist line numbers.

## Best Possible Solution

Keep the ephemeral Slack tool-progress direction, but make every draft-kind transition drain or otherwise settle the previous draft before rotating ids, with focused coverage for text-to-tool-to-text and text-to-tool-to-final turns.

## Reproduction Assessment

Yes, by source inspection. A narrative partial can be queued or in flight when a tool event calls `pushPreviewToolProgress()`, and the PR rotates the draft with `forceNewMessage()` before that narrative draft is flushed.

## Solution Assessment

No. The requested behavior is reasonable, but the patch needs to drain the narrative draft before the narrative-to-bullets rotation so permanent agent text is not dropped, overwritten, or later deleted with the bullet preview.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Flush narrative drafts before rotating to bullets:** `extensions/slack/src/monitor/message-handler/dispatch.ts:1004`
  - body: `pushPreviewToolProgress()` now rotates a narrative draft with `forceNewMessage()` before queuing the `Working…` bullets. Because draft sends are throttled/asynchronous, the narrative text can still be pending or in flight; `forceNewMessage()` clears pending text and resets ids, and an in-flight narrative send can then become the message edited/deleted as the bullet preview. Drain the narrative draft before starting the bullet preview so agent text segments are actually preserved.
