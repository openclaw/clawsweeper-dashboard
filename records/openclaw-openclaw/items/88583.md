## Summary

Keep this PR open: it targets a real session-state corruption bug, but the current patch can also reject legitimate queued-followup lifecycle events after preflight compaction rotates the session id.

## What This Changes

The PR adds optional sessionId ownership to in-memory agent lifecycle events/run contexts and skips session lifecycle persistence when the event owner sessionId differs from the current session row.

## Best Possible Solution

Keep the sessionId stale-event guard, but refresh or delay AgentRunContext registration so legitimate post-preflight rotations stamp the current sessionId before lifecycle persistence runs.

## Reproduction Assessment

Yes from source inspection: queued followups register the stamped sessionId before preflight compaction, while preflight can later mutate the same run to a rotated sessionId. I did not run tests because this was a read-only review.

## Solution Assessment

No. The session identity guard is the right direction, but the implementation must keep the stamped run context synchronized with legitimate session rotations before it is safe to merge.
