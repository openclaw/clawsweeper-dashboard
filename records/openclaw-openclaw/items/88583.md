## Summary

Keep open: this is a focused member-authored fix for a real session-state race, and the updated head addresses the prior stale-session stamping concern without an obvious blocking diff bug.

## What This Changes

The branch stamps a run's sessionId onto lifecycle events, skips lifecycle persistence when it targets a rotated session row, refreshes embedded run context on legitimate compaction rotations, and adds focused unit coverage.

## Best Possible Solution

Land the bounded identity guard after maintainers accept the session-state risk and the relevant checks pass, keeping unknown-owner lifecycle events on the legacy path.

## Reproduction Assessment

Yes from source inspection: current main persists lifecycle status by sessionKey without a sessionId identity check, and the linked report provides a concrete reset-then-late-terminal-event sequence. I did not run the live Slack reset race.

## Solution Assessment

Yes, the PR's event-boundary sessionId stamp plus persistence identity guard is a narrow maintainable fix because it avoids persisted schema, config, plugin SDK, and protocol changes while preserving unknown-owner legacy behavior.
