## Summary

Keep open: current main and v2026.6.6 still only guard the missing-user-service path against system-scope Linux gateway units, not a live gateway running as another Unix user's user service/home, and the safe detector needs maintainer/security policy.

## What This Changes

Request a high-signal doctor warning before `openclaw doctor --fix` repairs the current Unix user's home when the live gateway appears to belong to another user/home.

## Best Possible Solution

Add a conservative Linux Doctor preflight that warns only on high-confidence different-user/home gateway evidence, redacts sensitive paths, and requires explicit operator confirmation before creating or repairing a separate user service.

## Reproduction Assessment

Yes from source, but not from a live multi-user host: current main checks the caller's user service and system units, not another user's user-scoped gateway service/home, before it can proceed toward current-user service repair.

## Solution Assessment

Unclear until maintainers choose the detection and redaction boundary; the warning is directionally right, but the durable fix should not use broad `/home` enumeration as the main signal.
