## Summary

Keep open: current main guards only system-level Linux gateway units and current-user service discovery, so it still does not warn before mutable doctor repairs when the live gateway is a root or other-user systemd user service.

## What This Changes

Request a `doctor --fix` warning or confirmation when the current Unix user/home differs from an already-running gateway's user or config home.

## Best Possible Solution

Add a privacy-preserving Doctor preflight before mutable repairs and service installation that detects safely discoverable live-gateway ownership or config-home mismatches, prompts clearly, and documents detection limits.

## Reproduction Assessment

No. There is not a high-confidence live reproduction from this review, but source inspection gives a medium-confidence path: run Doctor from one Unix home while the live gateway is represented only by another user's systemd user service.

## Solution Assessment

Yes directionally. A warning or confirmation is narrower than implicitly switching state directories, but maintainers need to choose the safe detection scope, redaction policy, and whether the prompt gates all mutable repairs or only service install/repair steps.
