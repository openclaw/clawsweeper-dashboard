## Summary

Keep open: current main still only blocks detectable system-level Linux gateway services before installing a user service, and it does not warn when the live gateway is a root or other-user user-scoped service/home before doctor repairs the caller's environment.

## What This Changes

Request a clearer `openclaw doctor --fix` warning when the command is run from one Unix home while an already-running gateway appears to belong to another user or home.

## Best Possible Solution

Add a pre-mutation Doctor preflight that safely detects known live-gateway ownership or config-home mismatches, warns or asks for confirmation before repairing the caller's home, and documents privacy-preserving detection limits.

## Reproduction Assessment

Yes, at source level: with the caller's user service unloaded and only a root or sibling-user user-scoped gateway present, current main checks system units and then can proceed to install/repair from the caller's `process.env`. I did not run a destructive live root/non-root systemd reproduction.

## Solution Assessment

Yes directionally: warning before mutable Doctor repairs is narrower than silently switching state directories or changing ownership. The exact implementation should avoid broad cross-home scans and raw cross-user path disclosure.
