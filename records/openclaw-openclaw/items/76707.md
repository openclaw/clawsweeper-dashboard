## Summary

Keep open: current main has partial duplicate-service protection, but it does not cover the reported root user-systemd/user-home mismatch before doctor repair work can target the current operator home.

## What This Changes

Add a clear doctor --fix warning when the active local gateway appears to belong to another Unix user/home/config path so repairs do not silently target a separate OpenClaw environment.

## Best Possible Solution

Add a targeted doctor preflight that warns and asks before repair mode writes local state or installs/repairs current-user services when an active local OpenClaw gateway appears owned by another Unix user/home/config.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run doctor as a non-root user while a root-owned OpenClaw gateway is reachable on the configured port and no matching current-user service/config exists; current scans do not cover root's user home and the healthy gateway path skips daemon repair.

## Solution Assessment

Yes. The requested warning is the narrow maintainable direction, ideally implemented through existing doctor gateway-service, port-listener, and status/config diagnostics rather than a startup migration or new policy setting.
