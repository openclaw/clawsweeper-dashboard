## Summary

Keep open: current main has safeguards for system-level Linux gateway units, but it still scopes doctor repairs and user-service installs to the current environment and does not warn about a root or other-user user service before repairing the caller's home.

## What This Changes

Request a `doctor --fix` warning or confirmation when the active Unix user/home differs from a discoverable live gateway environment.

## Best Possible Solution

Add a doctor preflight guard for supported live-gateway home/user mismatches, ask before mutable repairs, and document detection limits.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection gives a medium-confidence path: run doctor from a different Unix home while the live gateway is only represented by another user's user service, and current main still targets the caller's state dir/user service unless a system unit is found.

## Solution Assessment

Unclear until maintainer policy is set. A warning/confirmation is directionally narrow, but the exact detection scope and prompt timing need product/ops judgment.
