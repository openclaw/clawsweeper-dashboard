## Summary

Keep open: current main and v2026.6.8 still block only system-scope Linux gateway services before installing the current user's service, so a root-owned user-scope gateway under another Unix home can still be missed; the remaining fix needs explicit security/product policy for cross-user detection and redaction.

## What This Changes

Request a clear `doctor --fix` warning/confirmation when the current Unix user/home differs from an already-running gateway's home/config so doctor does not silently repair a separate environment.

## Best Possible Solution

Add a conservative Linux doctor preflight that warns only on high-confidence different-user/home gateway evidence, redacts sensitive details, and requires explicit confirmation before creating or repairing a separate current-user service.

## Reproduction Assessment

Yes, from source: the missing-service repair path only checks system-scope units and current-HOME user units before it can install the current user's service. I did not run a live multi-user Linux/systemd reproduction.

## Solution Assessment

Unclear until maintainers choose the detection and redaction boundary. The warning is directionally right, but the durable fix should avoid broad home scans, raw path disclosure, and false positives for legitimate multi-gateway setups.
