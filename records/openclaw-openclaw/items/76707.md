## Summary

Keep open: current main and latest release still scope Doctor's missing-service repair to the current Unix home plus system-scope units, so another user's live user-scope gateway can still be missed before `doctor --fix` installs or repairs the current user's service.

## What This Changes

The issue asks for `openclaw doctor --fix` to warn before repairing the current Unix user's OpenClaw home when a live gateway appears to be running from another Unix user or home.

## Best Possible Solution

Add a conservative Linux Doctor preflight that warns only on high-confidence different-user/home gateway evidence, redacts sensitive details, and requires explicit confirmation before creating or repairing a separate current-user service.

## Reproduction Assessment

Yes, source-level. Current main derives the user service path from the current HOME and Doctor's install branch only checks system-scope gateway services before installing a current-user service; I did not run a live multi-user systemd reproduction.

## Solution Assessment

Unclear until maintainers choose the cross-user detection and redaction boundary. The requested warning is directionally right, but the durable fix should avoid broad home scans, raw path disclosure, and false positives for legitimate multi-gateway setups.
