## Summary

Keep open: current main and v2026.6.10 still only check the current-HOME user systemd unit plus system-scope units before Doctor can install or repair the current user's service, so another Unix user's live user-scope gateway can still be missed. The related open PRs address adjacent state-scan and same-name user/system conflicts, not this cross-user live-gateway warning.

## What This Changes

The issue requests a Linux Doctor warning before `doctor --fix` repairs the current Unix user's OpenClaw home when a live gateway appears to belong to another Unix user or home.

## Best Possible Solution

Add a conservative Linux Doctor preflight that warns only on high-confidence different-user/home gateway evidence, redacts sensitive details, and requires explicit confirmation before creating or repairing a separate current-user service.

## Reproduction Assessment

Yes, source-level. Current main derives the user service path from the current HOME and the Doctor install branch only checks system-scope gateway services before installing a current-user service; I did not run a live multi-user systemd reproduction.

## Solution Assessment

Unclear until maintainers choose the cross-user detection and redaction boundary. The requested warning is directionally right, but the durable fix should avoid broad home scans, raw path disclosure, and false positives for legitimate multi-gateway setups.
