## Summary

Keep open: current main and v2026.6.6 still block only system-level Linux gateway service duplication before doctor installs the caller's user service; they do not warn when the live gateway is another user's user-scoped service or home.

## What This Changes

The issue requests a clear `openclaw doctor --fix` warning before repairing the caller's Unix home when a running gateway appears to use another user's OpenClaw home.

## Best Possible Solution

Add a conservative Linux doctor preflight that warns only on high-confidence different-user/home gateway evidence, redacts sensitive paths, and requires explicit confirmation before creating or repairing a separate user service.

## Reproduction Assessment

Yes from source inspection, but not from a live multi-user host in this review. Current main still has a missing-current-user-service install path guarded by system-level service detection, not another-user user-service/home detection.

## Solution Assessment

Unclear until maintainers choose the detection and redaction boundary. The suggested warning is directionally right, but the durable fix should avoid broad `/home` leaks and false positives.
