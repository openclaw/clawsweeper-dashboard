## Summary

Keep open: current main has useful safeguards for system-level Linux units and current-home state-dir drift, but it still does not warn before repairing the caller's home when the live gateway is represented only by another Unix user's user service.

## What This Changes

Warn before `openclaw doctor --fix` repairs the current Unix user's home when a live gateway appears to be running from another Unix user or config path.

## Best Possible Solution

Add a doctor preflight or repair guard that detects discoverable live-gateway ownership/path mismatches, warns or confirms before mutable current-home repairs, and documents the limits of cross-user detection.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. Source inspection gives a medium-confidence path: run doctor as another Linux user while the live gateway is only represented by root or another user's systemd user service, and current main remains scoped to the caller's HOME plus system units.

## Solution Assessment

Yes directionally: a high-signal warning or confirmation is narrower than silently changing state-dir selection. The unresolved part is the exact privileged/cross-user discovery scope and when the warning should fire.
