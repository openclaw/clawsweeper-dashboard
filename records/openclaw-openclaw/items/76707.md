## Summary

Keep open: current main protects against system-level Linux gateway services, but source inspection does not show a cross-user/root user-service or live config/home mismatch warning before doctor repairs the caller's current OpenClaw home.

## What This Changes

The issue requests a high-signal `doctor --fix` warning before repairing the current Unix user's OpenClaw home when a live gateway appears to run from another Unix user's home.

## Best Possible Solution

Add a doctor preflight or repair guard that detects discoverable live-gateway ownership/path mismatches before mutable current-home repairs or service installation, asks for explicit confirmation, and documents detection limits.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection gives a medium-confidence path: run `openclaw doctor --fix` from a different Unix home while the live gateway is only discoverable as another user's systemd user service, and current main still targets the caller's state/config.

## Solution Assessment

Unclear pending maintainer policy: a warning/confirmation before mutable repairs is the narrowest visible UX fix, but the detection scope across Unix users/root services needs an owner decision.
