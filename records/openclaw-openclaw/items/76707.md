## Summary

Keep open: current main still warns for system-scope Linux gateway units but does not detect a live gateway running from another Unix user's user-scoped service/home, and the remaining fix needs maintainer security/product policy.

## What This Changes

The issue requests a high-signal `doctor --fix` warning before repairing the current Unix user's OpenClaw home when a likely live gateway belongs to another user/home.

## Best Possible Solution

Add a conservative Linux doctor preflight that warns only on high-confidence different-user/home gateway evidence, redacts sensitive details, and requires explicit confirmation before creating or repairing a separate current-user service.

## Reproduction Assessment

Yes, from source: the current missing-service repair path only checks system-scope units and current-home user units before it can repair the current user's service. I did not run a live multi-user Linux/systemd host reproduction.

## Solution Assessment

Unclear until maintainers choose the detection and redaction boundary. The suggested warning is directionally right, but the durable fix should avoid broad `/home` leaks, root-home assumptions, and false positives in legitimate multi-gateway setups.
