## Summary

Keep open: current main still has only the system-level Linux service guard and broad state-dir warnings, not the requested root/other-user user-service mismatch warning before doctor repairs the caller's home.

## What This Changes

Request `doctor --fix` to warn or confirm when a running gateway appears to belong to another Unix user/home before repairing the current user's environment.

## Best Possible Solution

Add a doctor preflight or repair guard for discoverable live-gateway ownership/path mismatches, then document the best-effort detection limits.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection gives a medium-confidence path: run doctor from a different Unix home while the live gateway is only represented by another user's user service, and current main still resolves repairs to the caller's state/config/service paths unless a system unit is found.

## Solution Assessment

Yes directionally, but the exact implementation needs maintainer policy. A warning/confirmation guard is narrower than changing state-dir selection, but the detection scope and prompt timing should be chosen deliberately.
