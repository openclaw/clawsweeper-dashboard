## Summary

Keep open: current main has partial duplicate-environment safeguards, but it still does not clearly detect a root or other-user systemd user gateway before mutable doctor repairs in the caller's home.

## What This Changes

The issue requests a clearer `openclaw doctor --fix` warning when the caller's Unix home differs from a discoverable already-running gateway owned by another user or home.

## Best Possible Solution

Add a doctor preflight or repair guard that surfaces discoverable live-gateway owner/state-dir mismatches before mutable repairs, with documented detection limits.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection gives a medium-confidence path: run doctor from a different Unix home while the live gateway is only represented by another user's systemd user service or root state, and current main still resolves repairs to the caller's state dir.

## Solution Assessment

Yes directionally: a high-signal warning or confirmation is narrower than changing the selected state dir automatically. The unresolved decision is the detection scope and whether the prompt should block all mutable repairs or only service repair/install paths.
