## Summary

Keep open: current main and the latest release still protect only against system-level Linux gateway services before doctor installs or repairs the caller's user service; they do not warn when the live gateway is another user's user-scoped service/home.

## What This Changes

Request a high-signal `openclaw doctor --fix` warning before repairing the caller's Unix home when a live gateway appears to belong to another Unix user or home.

## Best Possible Solution

Add a conservative Linux doctor preflight that only warns on high-confidence different-user/home gateway evidence, redacts sensitive paths, and requires explicit confirmation before creating or repairing a separate user service.

## Reproduction Assessment

Yes from source, but not from a live multi-user host in this review. The source path is clear: with no current-user service and no system unit, doctor has no other-user user-service/home check before the install/repair path.

## Solution Assessment

Unclear until maintainers choose the detection and redaction boundary. The suggested warning is directionally right, but the durable fix should avoid broad `/home` leaks and false positives.
