## Summary

Keep open: current main has useful safeguards for system-level Linux units and noninteractive service repairs, but source inspection does not show the requested warning for a root or other-user user service before repairing the current user's home.

## What This Changes

Request that `openclaw doctor --fix` warn before mutating the current user's OpenClaw home when a running gateway appears to belong to another Unix user/home.

## Best Possible Solution

Add a doctor preflight or repair guard that detects discoverable live-gateway ownership/path mismatches and asks for explicit confirmation before current-home repairs or service installation, with docs stating the detection limits.

## Reproduction Assessment

Yes, source inspection gives a medium-confidence reproduction path: run doctor from a different Unix home while the live gateway is represented only by another user's service, and current main still resolves repairs to the caller's state dir. I did not run a live root/user-service reproduction because this review is read-only.

## Solution Assessment

Yes directionally: a clear warning/confirmation is narrower and safer than silently changing the selected state dir. The maintainer decision is the exact detection scope and whether the prompt should occur before all mutable repairs or only service repairs.
