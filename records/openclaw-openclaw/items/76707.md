## Summary

Keep open: current main has partial duplicate-service protection, but source inspection still leaves the reported cross-Unix-user/root user-service mismatch without a targeted warning before current-user doctor repairs can create separate state or service metadata.

## What This Changes

Warn or guard when `openclaw doctor --fix` is run from a Unix user/home different from an already-running local OpenClaw gateway.

## Best Possible Solution

Add a doctor gateway-service safety guard that detects a live OpenClaw gateway owned by another Unix user/home, clearly states the current repair target versus the live gateway, and skips or prompts before creating a second current-user service.

## Reproduction Assessment

Yes, source-level: when the current user's service is missing, current main can report a busy OpenClaw port, only blocks system-level units, and then reaches the current-user service install prompt. I did not run a live root/non-root systemd setup because this review is read-only.

## Solution Assessment

No, current main is only a partial solution. The existing system-level duplicate guard should be extended with a narrow cross-user live-gateway warning/guard using existing port or service metadata.

## Repair Work Prompt

Implement a focused doctor safety improvement for https://github.com/openclaw/openclaw/issues/76707. When `openclaw doctor --fix` / `--repair` on Unix is about to apply gateway service repairs from a user/home that differs from an already-running local OpenClaw gateway on the configured port, print a high-signal warning that this run targets the current OpenClaw state/config path and not the live gateway. Do not silently install/restart a second current-user gateway service in that mismatch case; preserve the existing system-level duplicate guard and `OPENCLAW_SERVICE_REPAIR_POLICY=external` behavior. Prefer existing port listener and service metadata; do not read arbitrary other users' config files and do not add a config option. Add focused tests for a root/other-user OpenClaw listener with no current-user service, plus a non-regression test for intentional current-user/service and system-level guard behavior. Add a user-facing changelog entry if the repository's active changelog section requires it.
