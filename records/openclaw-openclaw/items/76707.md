## Summary

Keep open: current main has partial duplicate-service safeguards, but the inspected Linux doctor path does not cover a root-owned user service/home mismatch before the service-install flow remains reachable.

## What This Changes

The issue requests a clearer `doctor --fix` warning when the CLI is repairing the current Unix home while the live gateway appears to run from another Unix user or home.

## Best Possible Solution

Add a maintainer-approved doctor preflight that detects likely live-gateway ownership/home mismatch before service or state repairs, then warn or require explicit confirmation while preserving intentional isolated multi-gateway setups.

## Reproduction Assessment

Yes, from source inspection rather than a live host run. A Linux doctor run with no current-user service, a busy default gateway port, and no `/etc` system OpenClaw unit can proceed to the current-user install prompt while another user's user service is outside the scanned paths.

## Solution Assessment

Unclear. The requested warning is the right direction, but maintainers should choose the exact detection and confirmation policy because port ownership, process details, and intentional multi-gateway layouts can overlap.
