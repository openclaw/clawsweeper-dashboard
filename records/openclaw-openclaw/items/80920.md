## Summary

Keep open: current main and v2026.5.12 still have the unbounded IPv6 loopback bind probe, and the linked timeout fix PR is open and unmerged.

## What This Changes

The issue asks for a timeout around `canBindToHost("::1")` so gateway startup can fall back to IPv4-only binding on macOS Big Sur when IPv6 loopback never resolves.

## Best Possible Solution

Review and land the linked narrow timeout fix, or an equivalent maintainer patch, with focused coverage for a never-settling probe and no gateway bind-policy change.

## Reproduction Assessment

No, there is no high-confidence live Big Sur reproduction from this review. Source inspection shows current gateway startup awaits an unbounded `canBindToHost("::1")`, so the failure mode is clear if macOS 11 never emits either socket event.

## Solution Assessment

Yes. A bounded timeout around the existing temporary bind probe is the narrowest maintainable fix, and this issue should stay open until the linked PR or an equivalent patch lands.
