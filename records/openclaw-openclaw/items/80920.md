## Summary

Keep open: current main still has an unbounded IPv6 loopback bind probe in gateway startup, and the linked closing pull request remains open and unmerged.

## What This Changes

The issue reports that gateway startup can hang on macOS Big Sur when `canBindToHost("::1")` never emits `listening` or `error` and has no timeout.

## Best Possible Solution

Land a narrow timeout or abort around the IPv6 loopback bind probe in `src/gateway/net.ts`, with regression coverage for a never-settling probe, then close this issue through the linked PR.

## Reproduction Assessment

No for a high-confidence live platform repro: I did not reproduce the macOS Big Sur socket behavior. Source inspection does show that a never-settling `canBindToHost("::1")` promise would block gateway startup because current main awaits it without a timeout.

## Solution Assessment

Yes. A bounded timeout or equivalent abort around the existing temporary bind probe is the narrowest maintainable fix, and the linked PR is the right implementation path once it has proof and normal review.
