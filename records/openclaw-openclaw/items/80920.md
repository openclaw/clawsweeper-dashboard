## Summary

Keep open: current main and v2026.6.6 still have the unbounded IPv6 loopback bind probe, and the linked fix at https://github.com/openclaw/openclaw/pull/80921 remains open and unmerged.

## What This Changes

The issue reports that gateway startup can hang on macOS Big Sur because the IPv6 loopback bind probe in `src/gateway/net.ts` has no timeout.

## Best Possible Solution

Review, rebase or replace, and land the linked timeout fix with focused coverage and redacted macOS 11 after-fix startup proof.

## Reproduction Assessment

Yes, source-reproducible: if the IPv6 bind probe never emits `listening` or `error`, the helper promise never settles and startup remains blocked while resolving listen hosts. I did not live-reproduce the macOS Big Sur kernel behavior here.

## Solution Assessment

Yes for the fix direction: a bounded timeout on the bind probe is the narrow maintainable solution because it preserves working IPv6 loopback binding while avoiding an indefinite await on stalled probes.
