## Summary

Keep open: current main and the latest release still have an unbounded IPv6 loopback bind probe on the gateway startup path, and the linked fix pull request remains open and unmerged.

## What This Changes

The issue requests bounding the gateway `::1` bind probe so macOS Big Sur startup does not hang when the probe never settles.

## Best Possible Solution

Land the linked timeout fix or an equivalent bounded bind probe with regression coverage and macOS proof, then close this issue after the fix is merged.

## Reproduction Assessment

No high-confidence live reproduction is available here. Source inspection shows the startup path awaits `canBindToHost("::1")` with no timeout if the socket never emits `error` or `listening`, but the macOS Big Sur kernel/event behavior was not reproduced live.

## Solution Assessment

Yes. A bounded timeout or equivalent abort around the temporary IPv6 bind probe is the narrowest maintainable fix, and the linked PR is pointed at the right helper but still needs proof/rebase handling.
