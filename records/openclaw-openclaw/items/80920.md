## Summary

Keep open: current main and the latest release still have an unbounded IPv6 loopback bind probe in gateway startup, and the linked fix pull request remains open with a real-behavior-proof blocker.

## What This Changes

Report that gateway startup can hang on macOS Big Sur because `resolveGatewayListenHosts("127.0.0.1")` awaits an unbounded `canBindToHost("::1")` probe.

## Best Possible Solution

Review and land a narrow timeout or equivalent abort around the gateway bind probe, with macOS 11 after-fix proof and focused regression coverage, then close this issue.

## Reproduction Assessment

No for live reproduction. Source inspection gives a clear current-main failure path if the `::1` bind probe never emits `error` or `listening`, but I did not reproduce the macOS 11 socket behavior directly.

## Solution Assessment

Yes. Bounding the existing temporary bind probe is the narrowest maintainable fix because it preserves successful IPv6 loopback binding while preventing gateway startup from waiting forever.
