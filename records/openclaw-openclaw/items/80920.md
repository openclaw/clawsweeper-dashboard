## Summary

Keep open: current main and the latest release still have an unbounded IPv6 loopback bind probe in gateway startup, and the linked closing pull request remains open rather than merged.

## What This Changes

The issue asks OpenClaw to prevent gateway startup from hanging by adding a timeout to the IPv6 loopback bind probe used during listen-host resolution on macOS Big Sur.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80921, or an equivalent replacement, with a bounded timeout around the temporary bind probe and focused coverage for a never-settling probe.

## Reproduction Assessment

No for live macOS reproduction: I did not run a macOS Big Sur host. Source inspection on current main shows startup awaits `resolveGatewayListenHosts()`, which awaits `canBindToHost("::1")` with no timeout if neither socket event fires.

## Solution Assessment

Yes. A bounded timeout or equivalent abort around the temporary bind probe is the narrowest maintainable fix; the linked pull request is the active implementation path but needs normal merge readiness work.
