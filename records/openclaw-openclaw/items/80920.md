## Summary

Keep open: current `main` and the latest release still have the unbounded IPv6 loopback bind probe, and the same-author PR that closes this issue remains open and unmerged.

## What This Changes

The issue reports a macOS Big Sur gateway startup hang because the IPv6 loopback bind probe in `src/gateway/net.ts` has no timeout.

## Best Possible Solution

Land or replace the linked timeout fix with focused stalled-probe coverage and redacted macOS 11 startup proof.

## Reproduction Assessment

Yes, source-reproducible: if the IPv6 bind probe never emits `listening` or `error`, the current helper promise never settles and gateway startup waits while resolving listen hosts. I did not live-reproduce the macOS Big Sur kernel behavior here.

## Solution Assessment

Yes for the fix direction: a bounded timeout on the bind probe is the narrow maintainable solution because it preserves working IPv6 loopback binding while avoiding an indefinite await on stalled probes.
