## Summary

Keep open: current main and v2026.5.12 still have the unbounded IPv6 loopback bind probe, and the linked closing PR is open, conflicting, and still needs real macOS 11 behavior proof before it can resolve this issue.

## What This Changes

The issue requests that the gateway avoid hanging indefinitely on macOS Big Sur when the IPv6 loopback bind probe for `::1` never emits `listening` or `error`.

## Best Possible Solution

Review and land the linked timeout PR, or an equivalent narrow patch, with focused coverage for a never-settling probe and redacted macOS 11 startup proof.

## Reproduction Assessment

No, not with high confidence in this review because I did not run macOS 11. Source inspection does show current gateway startup awaits an unbounded `canBindToHost("::1")`, so the reported hang is source-reproducible if Big Sur never emits either socket event.

## Solution Assessment

Yes. A bounded timeout around the existing temporary bind probe is the narrowest maintainable fix, and this issue should stay open until the linked PR or an equivalent patch lands.
