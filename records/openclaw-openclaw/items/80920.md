## Summary

Keep open: current main still has an unbounded IPv6 loopback bind probe in gateway startup, and the linked fix pull request is open but unmerged.

## What This Changes

The issue requests adding a timeout around the gateway `canBindToHost("::1")` probe so macOS Big Sur startup can fall back to IPv4 instead of hanging.

## Best Possible Solution

Review and land a narrow fix that bounds the IPv6 loopback bind probe and adds regression coverage for a never-settling probe without changing gateway bind policy.

## Reproduction Assessment

No. I did not reproduce the macOS Big Sur hang live, but source inspection shows the reported startup path awaits `canBindToHost("::1")` with no timeout if the socket never emits `error` or `listening`.

## Solution Assessment

Yes. A bounded timeout or equivalent abort around the existing bind probe is the narrowest maintainable fix; the linked pull request is in the right area but still needs normal review and validation.
