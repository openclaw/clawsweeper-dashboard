## What This Changes

The issue requests a timeout around the gateway IPv6 loopback bind probe so macOS Big Sur startup cannot hang indefinitely.

## Best Possible Solution

Land or replace the linked timeout fix with focused stalled-probe coverage and redacted macOS 11 startup proof.

## Reproduction Assessment

Yes, source-level: if the IPv6 bind probe never emits `listening` or `error`, the current helper promise never settles and gateway startup waits in `resolveGatewayListenHosts`. I did not live-reproduce the macOS Big Sur kernel behavior here.

## Solution Assessment

Yes for the fix direction: a bounded bind-probe timeout is the narrow maintainable solution because it preserves working IPv6 loopback binding while avoiding an indefinite await on stalled probes.
