## What This Changes

Bound the gateway IPv6 loopback bind probe on macOS Big Sur so startup falls back to IPv4 instead of hanging after `starting HTTP server...`.

## Best Possible Solution

Review and land a narrow fix such as https://github.com/openclaw/openclaw/pull/80921 that bounds the IPv6 loopback bind probe and adds focused coverage for a never-settling probe without changing gateway bind policy.
