## Summary

Keep open: current main still has the unbounded IPv6 loopback bind probe, gateway startup still awaits it on non-Windows loopback binds, and the direct closing PR remains open and unmerged.

## What This Changes

The issue reports that gateway startup can hang on macOS Big Sur because the IPv6 loopback bind probe in `src/gateway/net.ts` has no timeout.

## Best Possible Solution

Review and land the linked narrow timeout fix, or an equivalent maintainer patch, with focused coverage for a never-settling probe and no gateway bind-policy change.

## Reproduction Assessment

No, not as a live reproduction: I did not reproduce the macOS 11 kernel/event behavior. Source inspection does show a clear path where current main awaits an unbounded `canBindToHost("::1")` during gateway startup on non-Windows loopback binds.

## Solution Assessment

Yes. A bounded timeout around the existing temporary bind probe is the narrowest maintainable fix, and this issue should stay open until the linked PR or an equivalent patch lands.
