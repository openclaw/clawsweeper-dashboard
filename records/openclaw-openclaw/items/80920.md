## Summary

Keep open: current main still has the unbounded IPv6 loopback bind probe, v2026.5.12 shipped the same behavior, and the linked closing PR remains open and unmerged.

## What This Changes

The issue requests a timeout for `canBindToHost("::1")` so gateway startup does not hang indefinitely on macOS Big Sur when the IPv6 loopback bind probe never settles.

## Best Possible Solution

Review and land the linked narrow timeout fix, or an equivalent maintainer patch, with focused coverage for a never-settling bind probe and no gateway bind-policy change.

## Reproduction Assessment

No high-confidence live reproduction is available from this review. Source inspection shows current gateway startup awaits an unbounded `canBindToHost("::1")`, so the failure mode is clear if macOS 11 never emits either socket event.

## Solution Assessment

Yes. A bounded timeout around the existing temporary bind probe is the narrowest maintainable fix, and this issue should stay open until the linked PR or an equivalent patch lands.
