## Summary

Keep open: current main and v2026.5.12 still have the unbounded IPv6 loopback bind probe, and https://github.com/openclaw/openclaw/pull/80921 is open, unmerged, and linked with closing syntax.

## What This Changes

The issue reports that gateway startup can hang on macOS Big Sur because `canBindToHost("::1")` in `src/gateway/net.ts` has no timeout.

## Best Possible Solution

Review and land the linked narrow timeout fix, or an equivalent maintainer patch, with focused coverage for a never-settling probe and no gateway bind-policy change.

## Reproduction Assessment

No high-confidence live reproduction was established here because this review did not run on macOS 11. Source inspection shows the failure path clearly if that platform never emits either socket event: gateway startup awaits the unbounded `canBindToHost("::1")` probe.

## Solution Assessment

Yes. A bounded timeout around the temporary bind probe is the narrowest maintainable fix, and the issue should stay open until the linked PR or an equivalent patch lands.
