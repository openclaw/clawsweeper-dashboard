## Summary

Keep open: current main and the latest release still have the unbounded IPv6 loopback bind probe, and the linked timeout fix PR is open, unmerged, conflicting, and waiting on real macOS 11 proof.

## What This Changes

The issue reports that gateway startup can hang on macOS Big Sur because the IPv6 loopback bind probe in `src/gateway/net.ts` has no timeout.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80921, or an equivalent narrow timeout patch, with focused coverage and macOS 11 after-fix proof without changing gateway bind policy.

## Reproduction Assessment

No, there is no high-confidence live Big Sur reproduction from this review. Source inspection does show current gateway startup awaits the unbounded `canBindToHost("::1")` path if that probe never emits `error` or `listening`.

## Solution Assessment

Yes. A bounded timeout around the existing temporary bind probe is the narrowest maintainable fix, and this issue should remain open until the linked PR or an equivalent patch lands.
