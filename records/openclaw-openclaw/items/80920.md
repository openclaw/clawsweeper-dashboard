## Summary

Keep open: current main and v2026.6.8 still have the unbounded IPv6 loopback bind probe, and https://github.com/openclaw/openclaw/pull/80921 is still open as the linked candidate fix rather than merged implementation.

## What This Changes

The issue reports that Gateway startup can hang indefinitely on macOS Big Sur because `canBindToHost("::1")` has no timeout.

## Best Possible Solution

Keep this issue open until the linked timeout fix is rebased, proven with macOS 11 startup evidence or maintainer override, and merged or replaced by an equivalent narrow PR.

## Reproduction Assessment

Yes, source-reproducible: if the IPv6 bind probe never emits `listening` or `error`, the current helper promise never settles and Gateway startup waits while resolving listen hosts. I did not live-reproduce the macOS Big Sur kernel behavior here.

## Solution Assessment

Yes for the fix direction: a bounded timeout on the bind probe is the narrow maintainable solution because it preserves working IPv6 loopback binding while avoiding an indefinite await on stalled probes.
