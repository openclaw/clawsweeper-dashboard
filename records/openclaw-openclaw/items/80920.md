## Summary

Keep open: current main and the latest release still have the unbounded IPv6 loopback bind probe, and the same-author fix PR that uses closing syntax for this issue is still open and unmerged.

## What This Changes

Issue requests a timeout around the gateway `::1` bind probe so macOS Big Sur startup can fall back to IPv4 instead of hanging.

## Best Possible Solution

Review and land the linked narrow timeout fix, or an equivalent maintainer patch, with focused coverage for a never-settling probe and no gateway bind-policy change.

## Reproduction Assessment

No, there is no live high-confidence Big Sur reproduction from this review. Source inspection shows current gateway startup awaits an unbounded `canBindToHost("::1")`, so the failure mode is clear if macOS 11 never emits either socket event.

## Solution Assessment

Yes. A bounded timeout or equivalent abort around the existing temporary bind probe is the narrowest maintainable fix, and this issue should stay open until the linked pull request or an equivalent patch lands.
