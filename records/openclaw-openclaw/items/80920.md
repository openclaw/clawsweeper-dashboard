## Summary

Keep open: current main and the latest release still have an unbounded IPv6 loopback bind probe in gateway startup, and the linked fix pull request is open but unmerged.

## What This Changes

The issue requests bounding the gateway IPv6 loopback bind probe so macOS Big Sur startup falls back to IPv4 instead of hanging.

## Best Possible Solution

Review and land the linked narrow timeout fix, or a maintainer-applied equivalent, with focused coverage for a never-settling probe and no gateway bind-policy change.

No live high-confidence reproduction is available from this review. Source inspection shows current gateway startup awaits an unbounded `canBindToHost("::1")`, so the failure mode is clear if macOS Big Sur never emits either socket event.

## Solution Assessment

Yes. A bounded timeout or equivalent abort around the existing temporary bind probe is the narrowest maintainable fix, and the issue should stay open until the linked pull request or an equivalent patch lands.
