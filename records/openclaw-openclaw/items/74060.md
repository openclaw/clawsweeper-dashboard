## Summary

Keep open. Current main still has the Feishu setup-wizard status probe using top-level `channels.feishu` credentials, while this PR applies a narrow account-resolver fix with focused regression coverage and I found no blocking review finding.

## What This Changes

The PR changes Feishu setup-wizard status probing to resolve the selected/default Feishu account before calling `probeFeishu` and adds a multi-account regression test.

## Best Possible Solution

Land this targeted Feishu plugin fix or an equivalent patch after checks are green, keep the account-scoped regression test, and only close the linked health report once the affected status surface is confirmed.

## Reproduction Assessment

Yes. A config with `channels.feishu.defaultAccount` and credentials only under `channels.feishu.accounts.<id>` exercises the current-main gap because setup status sees Feishu as configured but cannot pass account-scoped credentials to `probeFeishu`.

## Solution Assessment

Yes for the setup-wizard status surface. Reusing `resolveFeishuAccount({ cfg, accountId })` is the narrowest maintainable fix because it follows the existing Feishu account contract and requires no new core or plugin API.
