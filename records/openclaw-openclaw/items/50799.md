## Summary

Keep open: current main still restricts the macOS paperclip picker and Swift attachment loader to images, while the existing web UI and Gateway paths already support non-video file attachments.

## What This Changes

The issue asks the macOS GUI paperclip picker to allow common documents/files such as PDF, TXT, and DOCX instead of selecting only images.

## Best Possible Solution

Broaden the shared Swift ChatUI attachment intake to match the existing Control UI and Gateway contract: accept supported non-video documents, keep image previews for images, show generic file rows for documents, and preserve size/type validation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: the macOS picker restricts `NSOpenPanel` to `.image`, and the view model rejects non-image UTTypes even if a document URL reaches it.

## Solution Assessment

Yes. Fixing the shared Swift picker/loading/preview path is the narrowest maintainable solution because Gateway `chat.send` and the Control UI already support non-video document attachments.
