## Summary

Keep open. Current main still enforces image-only selection and validation in the shared Swift macOS attachment flow, while the Control UI and Gateway already support non-image chat attachments; the provided context and a bounded public PR search did not show a merged fix, paired implementation PR, or canonical duplicate.

## What This Changes

The issue requests that the macOS GUI paperclip attachment picker allow common document uploads such as PDF, TXT, and DOCX instead of only selectable image files.

## Best Possible Solution

Broaden the macOS shared Swift attachment path to accept the same supported non-video file classes as the Control UI and chat.send contract, keep image previews where available, show a generic file chip for documents, and add focused Swift coverage for PDF/TXT/DOCX acceptance plus image parity and video rejection.
