## Summary

Keep open. Current main still starts diagnostic liveness sampling without a startup-grace guard, and open PR #76401 already uses closing syntax for this issue, so the maintainer path is to review and land or replace that PR before closing.

## What This Changes

The issue asks for diagnostic liveness warnings to honor the gateway startup-grace window so cold-start `models.list` and `sessions.list` delays do not produce WARN-level noise.

## Best Possible Solution

Review and land #76401, or an equivalent focused diagnostics change, so startup-grace suppresses false liveness reports while preserving sampler reset semantics and regression coverage.

## Reproduction Assessment

Yes from source inspection: current main starts diagnostics at gateway startup, samples liveness on the first heartbeat, and has no grace guard while diagnostic work can be queued. I did not run a live cold-start replay in this read-only review.

## Solution Assessment

Yes, the requested direction is the right narrow fix: wire a diagnostics startup-grace gate to the existing gateway startup behavior. The issue should remain open until #76401 or an equivalent implementation merges.
