## Summary

Keep open. Current main still lacks diagnostic startup-grace wiring, and #76401 is an open unmerged closing PR for the same behavior, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Apply the gateway startup-grace window to diagnostic liveness sampling so cold-start event-loop delay does not emit WARN-level warnings.

## Best Possible Solution

Land or replace #76401 with a narrow diagnostic heartbeat grace-window fix that preserves post-grace liveness warnings and includes regression coverage plus a changelog entry.

## Reproduction Assessment

Yes, source-level. Current main starts diagnostic heartbeat with no startup-grace option and samples liveness each interval, while the channel health monitor separately skips checks inside `monitorStartupGraceMs`; I did not run a live cold-start.

## Solution Assessment
