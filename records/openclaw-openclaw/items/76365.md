## Summary

Keep open. Current main still runs diagnostic liveness sampling without a startup-grace guard, and an open closing PR is already attached, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for diagnostic liveness warnings to honor the 60s startup-grace window during gateway cold start.

## Best Possible Solution

Land or replace #76401 with a narrow diagnostic heartbeat grace-window fix that preserves normal post-startup liveness warnings and includes regression coverage plus changelog entry.

## Reproduction Assessment

Yes, source-level: current main has a diagnostic heartbeat path that samples liveness without startup grace while the channel health monitor separately skips checks during `monitorStartupGraceMs`. I did not run a live cold-start, so this is source-reproducible rather than runtime-reproduced.

## Solution Assessment

Yes. Applying the existing 60s startup-grace policy to diagnostic liveness emission is the narrow maintainable fix; current main has not done that yet, and #76401 is the active implementation path.
