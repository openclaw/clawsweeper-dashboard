## Summary

Keep open. Current main still has no startup-grace guard for diagnostic liveness emission, and the exact startup-grace PR was closed unmerged, so there is still narrow repo work left despite the current idle-sample downgrade.

## What This Changes

Diagnostic liveness warnings should respect the gateway startup grace window during cold-start operations.

## Best Possible Solution

Add a diagnostic heartbeat startup-grace option, wire the existing 60s gateway startup grace into it, keep sampler resets running during grace, and cover suppression-before/allow-after behavior in diagnostic tests.

## Reproduction Assessment

Yes, source-level. On current main, `startDiagnosticHeartbeat` samples every 30s without startup age checks, and existing tests show an `event_loop_delay` sample with queued diagnostic work logs a liveness warning before the health monitor's 60s grace would expire.

## Solution Assessment

Yes. The narrow maintainable fix is the emission-only grace approach from the closed PR: continue sampling to reset metrics, but suppress liveness event/WARN emission until the startup window expires.
