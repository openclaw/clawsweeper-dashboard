## Repair Work Prompt

Fix openclaw/openclaw#76365. Current main starts diagnostic liveness sampling without honoring the 60s gateway startup grace, so cold-start event-loop delay can emit WARN-level `liveness warning` logs during the grace window. Implement a narrow fix in `src/logging/diagnostic.ts` and `src/gateway/server.impl.ts`: add a startup-grace option or equivalent to diagnostic heartbeat startup, pass the same 60_000ms policy used by gateway health monitoring, suppress diagnostic liveness warning/event emission during that window, and preserve normal memory heartbeat behavior and post-grace liveness warnings. Add/update focused regression coverage in `src/logging/diagnostic.test.ts` proving a liveness sample with queued work is suppressed inside grace and emitted after grace expires. Use closed PR #76401 only as source context; do not broaden this into the related cold-start latency/prewarming issue #76360. Add a user-facing `CHANGELOG.md` fix entry if policy requires one for this WARN-log behavior change.
