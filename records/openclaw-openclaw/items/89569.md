## Summary

The silent access-request workflow remains useful, but the current patch is not mergeable: its grouped allowlists duplicate a shipped canonical access-group system, while the pre-auth hook requires an explicit maintainer decision about exposing blocked-message content to plugins.

## What This Changes

The PR adds pre-auth observation hooks for blocked Telegram and WhatsApp DMs, grouped DM allowlist entries with sender-group metadata, grouped `/allowlist` edits, automatic trusted-hook loading, tests, and documentation.

## Best Possible Solution

Retain one canonical `accessGroups` model, project a deterministic matched group into trusted inbound metadata, and review the sessionless pre-auth observer separately as an explicit, narrowly scoped plugin API with opt-in and upgrade proof.
