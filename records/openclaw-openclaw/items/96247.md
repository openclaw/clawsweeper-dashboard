## Summary

Keep open: the PR is a focused attempt at a real duplicate-delivery bug, but the patch does not yet close the failure mode it describes because `unknown_after_send` entries are still replayed when reconciliation returns `not_sent`.

## What This Changes

The PR hoists `platformSendStarted`, marks queued mid-batch outbound failures with prior send evidence as `unknown_after_send`, and adds mocked plus real SQLite queue tests.

## Best Possible Solution

Keep the send-evidence classification, but pair it with explicit partial-send recovery semantics and regression coverage that prevent replaying already-sent payloads when reconciliation is wrong or incomplete.

## Reproduction Assessment

## Solution Assessment

No. Marking the queue entry `unknown_after_send` is a useful part of the fix, but current drain semantics still replay that state after `not_sent`, so the patch is not yet the narrow complete solution for the reported duplicate path.
