## Summary

Keep open: current main and the latest release still return raw summary-model text into the Auto-TTS audio path, and this issue is already paired with open closing PRs that need review or consolidation rather than a new cleanup close.

## What This Changes

The issue reports that TTS auto-summarization can speak reasoning tags, prompt scaffolding, and echoed source text instead of only the summarized audible text.

## Best Possible Solution

Land one canonical TTS summary-output sanitizer at `summarizeText`, with regression coverage for reasoning tags, assistant scaffolding, prompt echoes, and source-text echoes before `summary.summary` reaches speech.

## Reproduction Assessment

Yes. Source inspection shows `summarizeText` can return raw `TextContent` containing reasoning or prompt scaffolding, and `maybeApplyTtsToPayload` assigns that returned summary to `textForAudio` and `spokenText`.

## Solution Assessment

Yes. Sanitizing summary-model output at the `summarizeText` boundary is the narrowest maintainable fix; switching summary models or expanding config-write allowlists would be workaround/config work, not the core leak fix.
