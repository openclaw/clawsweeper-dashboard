## Summary

Keep open: current main still routes pure `claude-cli` sessions through the CLI live-session path without trajectory recorder plumbing, and the open linked PR is the implementation candidate rather than a reason to close this issue before merge.

## What This Changes

Ensure pure `claude-cli` sessions write runtime trajectory events and a `<session>.trajectory.jsonl` sidecar, or explicitly document an intentional CLI exemption.

## Best Possible Solution

Land a narrow CLI executor trajectory-recorder integration with focused regression coverage, then close this issue via the merged fix PR; use a docs-only clarification only if maintainers intentionally exempt CLI runs.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction: current main's CLI executor and Claude live-session path have no trajectory recorder event sink, while docs and the embedded provider path define default runtime trajectory events. I did not run a live Claude CLI session in this read-only review.

## Solution Assessment

Yes, the best direction is to reuse the existing runtime trajectory recorder in the CLI executor path without changing Claude session reuse, auth, or fallback policy. The existing linked PR should be reviewed and corrected rather than replacing this issue with a separate cleanup action.
