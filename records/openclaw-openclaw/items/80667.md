## What This Changes

The issue reports that pure `claude-cli` sessions write the trajectory pointer but never emit runtime trajectory events or create the `.trajectory.jsonl` sidecar.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81039, or a narrow equivalent, so the existing trajectory recorder covers the CLI executor lifecycle without changing Claude CLI auth, fallback, or session reuse semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows the `claude-cli` stdio path is selected by backend config and has no recorder calls, while docs and the embedded path define default runtime trajectory events; I did not run a live claude-cli session.

## Solution Assessment
