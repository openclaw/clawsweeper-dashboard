## Summary

Keep open: current main still routes pure claude-cli runs through the CLI live-session path without a trajectory recorder, while trajectory docs say runtime capture is default-on for OpenClaw agent runs. Source inspection matches the reporter's v2026.5.7 audit and shows this is a real parity gap, not an already-fixed cleanup item.

## What This Changes

Fix pure claude-cli sessions so they write `<session>.trajectory.jsonl` runtime events instead of only creating the trajectory pointer when no model fallback occurs.

## Best Possible Solution

Wire the existing trajectory recorder into the CLI/claude live-session run path so it emits equivalent session, prompt, model, artifact, and end events, with focused regression coverage; use a docs-only clarification only if maintainers intentionally exempt CLI runs.

## Reproduction Assessment

Yes, from current-main source inspection: claude-cli runs call `runCliAgent` and `runClaudeLiveSessionTurn` without any trajectory recorder, while the fallback recorder only emits `model.fallback_step`. I did not run a live Claude CLI session in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing trajectory recorder contract in the CLI path without changing Claude CLI session reuse, fallback behavior, or auth plumbing.
