## Summary

Keep open: current main and v2026.5.7 still route healthy pure claude-cli turns through the live-session runner without any trajectory event sink, while the trajectory contract says runtime capture is default-on for OpenClaw agent runs.

## What This Changes

Ensure pure claude-cli sessions create and flush the same trajectory runtime sidecar events as non-CLI provider sessions.

## Best Possible Solution

Wire the existing trajectory recorder contract into the claude-cli live-session run path and add focused regression coverage; use a docs-only clarification only if maintainers intentionally redefine CLI runs as trajectory-silent.

## Reproduction Assessment

Yes, from current-main source inspection: pure claude-cli runs enter `runCliAgent` and `runClaudeLiveSessionTurn` without a recorder, while only fallback steps record to the trajectory file. I did not run a live Claude CLI session in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to pass/reuse the existing trajectory recorder through the CLI path and emit equivalent lifecycle, prompt, completion, artifact, and end events without changing auth, fallback, or live-session reuse behavior.
