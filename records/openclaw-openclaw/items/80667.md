- `impact:data-loss`: Pure claude-cli sessions silently miss the trajectory JSONL sidecar events used as recorded session data.
- `impact:session-state`: Missing prompt/model/end trajectory events create session trace gaps for post-mortem and regression tooling.

## Summary

Keep open: current main still lacks trajectory recorder plumbing in the pure claude-cli executor path, and the linked closing PR is still open rather than merged. This issue should close only after that fix lands, or after maintainers explicitly choose and document a CLI trajectory exemption.

## What This Changes

Pure claude-cli sessions should write the same trajectory lifecycle JSONL sidecar as other agent provider paths.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81039, or an equivalent CLI-runner trajectory integration, so pure claude-cli runs emit the existing lifecycle events.

## Reproduction Assessment

Yes. Current-main source inspection shows pure claude-cli runs enter `executePreparedCliRun` and `runClaudeLiveSessionTurn` without any trajectory recorder, while the fallback recorder only emits `model.fallback_step`.

## Solution Assessment

Yes. Reusing the existing trajectory recorder contract in the CLI executor path is the narrow maintainable fix; a docs-only exemption would require an explicit maintainer decision that conflicts with current docs.
