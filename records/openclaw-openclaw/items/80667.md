## Summary

## What This Changes

The issue reports that pure `claude-cli` sessions write a trajectory pointer but never write the corresponding trajectory event JSONL sidecar.

## Best Possible Solution

Review and land `https://github.com/openclaw/openclaw/pull/81039` or an equivalent CLI-runner trajectory integration, then close this issue with the merged commit and release provenance.

## Reproduction Assessment

Yes. Source inspection on current `main` shows pure `claude-cli` runs flow through `executePreparedCliRun` and `runClaudeLiveSessionTurn` without a trajectory recorder, while the docs and embedded runner establish default lifecycle trajectory events.

## Solution Assessment

Yes. The narrow maintainable direction is to reuse the existing trajectory recorder in the CLI executor path without changing Claude CLI session reuse, fallback classification, or auth behavior.
