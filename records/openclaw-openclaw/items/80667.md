## Summary

Keep open: current main still has the source-level trajectory gap for pure `claude-cli` runs, and an open linked PR explicitly targets this issue but has not merged.

## What This Changes

The issue reports that pure `claude-cli` sessions write a trajectory pointer but never materialize `<session>.trajectory.jsonl` runtime events.

## Best Possible Solution

Review and land a narrow CLI-runner recorder integration that emits the existing runtime trajectory lifecycle events for pure `claude-cli` runs, with focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible with high confidence: current main creates trajectory pointers and documents default runtime events, but the pure CLI executor path has no recorder calls. I did not run a live Claude CLI session in this read-only review.

## Solution Assessment

Yes, the narrow maintainable direction is to reuse the existing trajectory recorder in `executePreparedCliRun` without changing Claude CLI auth, reuse, fallback, or live-session semantics. The issue should remain open until the linked fix PR merges or maintainers choose an explicit docs-only exemption.
