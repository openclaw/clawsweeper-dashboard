## Summary

Keep open: current main still routes pure `claude-cli` live sessions through the generic CLI runner without any trajectory recorder hook, while the documented trajectory contract says runtime capture is on by default for agent runs. The adjacent diagnostic issue at https://github.com/openclaw/openclaw/issues/79366 is related, but it does not cover the missing `prompt.submitted`/completion trajectory parity reported here.

## What This Changes

Make pure `claude-cli` sessions write standard trajectory runtime events to `<session>.trajectory.jsonl`, or document intentional trajectory silence if maintainers choose that policy.

## Best Possible Solution

Add recorder support in the generic CLI runner or Claude live-session path so CLI-backed turns emit the documented runtime events, with focused regression coverage and no change to fallback-step recording.

## Reproduction Assessment

Yes from source inspection: a pure `claude-cli` live turn enters `shouldUseClaudeLiveSession` and no trajectory recorder is created or passed on that path. I did not run a live Claude CLI session during this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to wire the existing trajectory recorder contract into the CLI runner/live-session path and test the emitted events. A docs-only clarification would conflict with the current docs unless maintainers intentionally decide that CLI runs should remain trajectory-silent.
