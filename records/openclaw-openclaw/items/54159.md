## Summary

Keep open. Current main still has the shared overflow path that can await a non-settling stream cancellation, so the PR addresses a real remaining bug, but the branch is dirty against current main and includes stale Matrix and agent import hunks that should not land as-is.

## What This Changes

The PR updates the shared bounded-response reader to cancel oversized streams without awaiting `reader.cancel()`, adds a non-settling-cancel regression test, applies the same pattern to the then-current Matrix helper, and removes two agent imports.

## Best Possible Solution

Land a narrow current-main patch in `src/media/read-response-with-limit.ts` that fire-and-forgets overflow cancellation, add the focused non-settling-cancel unit regression in `src/media/read-response-with-limit.test.ts`, let Matrix inherit the behavior through its SDK wrapper, and leave unrelated agent cleanup out of the change.

## Reproduction Assessment

Yes. A mocked streamed `Response` whose first chunk exceeds `maxBytes` and whose `reader.cancel()` promise never settles directly reproduces the current main hang in the shared reader.

## Solution Assessment

No, not in this PR shape. The shared reader change and focused test are the right direction, but the stale Matrix copy edit and obsolete agent import cleanup should be dropped through a rebase or narrow replacement.
