## Summary

## What This Changes

The PR changes the shared bounded-response reader to cancel overflowed streams without awaiting `reader.cancel()`, adds a never-resolving-cancel unit test, applies the same pattern to a Matrix helper copy, and removes two agent imports.

## Best Possible Solution

Land a current-main patch limited to the shared response reader, its focused regression test, and one changelog entry; Matrix should inherit the behavior through the SDK wrapper.

## Reproduction Assessment

Yes, by source inspection. A streamed response whose next chunk exceeds `maxBytes` and whose `reader.cancel()` promise never settles will keep current-main `readResponseWithLimit()` waiting before it can throw the overflow error.

## Solution Assessment

No, not as currently shaped. The shared-reader change and regression test are the right fix, but the stale Matrix hunk and missing changelog entry should be corrected in a current-main repair or replacement.
