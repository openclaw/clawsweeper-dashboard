## Summary

Keep open: the patch addresses the reported OpenRouter Anthropic prefix path, but the broad normalization would also rewrite the canonical OpenRouter default model from `openrouter/auto` to `auto`, which is a likely provider-routing regression.

## What This Changes

The branch imports OpenRouter model-id normalization into `normalizeResolvedModel`, rewrites resolved `model.id` values with an `openrouter/` prefix, and adds unit assertions for Hunter Alpha and Anthropic refs.

## Best Possible Solution

Land a narrowed OpenRouter-owned normalization that preserves API slugs such as `openrouter/auto`, strips only prefixed provider catalog refs before transport, and backs it with regression tests plus redacted live probe or completion proof.

## Reproduction Assessment

Yes, at source level. Current main sends the resolved OpenRouter `model.id` through the OpenAI-compatible payload unchanged, and the PR-introduced regression is source-reproducible because its helper strips `openrouter/auto` to `auto`.

## Solution Assessment

No. Provider-owned normalization is the right layer, but this implementation is too broad; the best fix must preserve OpenRouter API slugs such as `openrouter/auto` while normalizing nested provider refs like `openrouter/anthropic/...`.
