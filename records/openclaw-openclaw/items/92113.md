## Summary

This PR should stay open for a narrow scoping repair before merge.

## What This Changes

## Best Possible Solution

Keep the shared marker-gated helper, use it in both availability and final resolution, and preserve plugin-side fallback only under the existing managed-marker/source-snapshot contract; add a mismatched-config regression test.

## Solution Assessment

No, not in its current breadth. Sharing the marker-gated lookup is the narrow maintainable fix, but the unconditional global-snapshot fallback should be removed or guarded by the active source-snapshot contract.
