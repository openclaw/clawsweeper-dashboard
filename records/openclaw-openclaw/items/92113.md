## Summary

This PR fixes the reported availability gap and has credible file-backed runtime proof, but the latest head still permits a caller whose source config does not match the active snapshot to inherit that snapshot’s credential whenever it carries the generic managed marker. That credential-isolation exception remains merge-blocking, and maintainers must also select one canonical implementation among this PR, https://github.com/openclaw/openclaw/pull/92208, and the proof-positive https://github.com/openclaw/openclaw/pull/92235.

## What This Changes

The PR adds shared runtime-snapshot resolution for managed SecretRef custom-provider API keys in availability and credential selection, with expanded regression tests and a file-backed reproduction script.

## Best Possible Solution

Use one canonical resolver that requires active source-snapshot identity for credential reuse, preserves explicit managed SecretRef precedence over ambient or synthetic auth, and proves the reporter’s persisted marker-only upgrade scenario without allowing credentials to cross independent config contexts.
