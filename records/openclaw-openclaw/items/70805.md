## Summary

Keep open: the latest head still misses the staged OpenClaw SDK alias package and wrapper outputs that caused the reported Telegram import failure, so the PR does not fully fix the watch-mode runtime-postbuild gap despite green CI.

## What This Changes

The PR adds watch-mode build/runtime-postbuild freshness checks in `scripts/run-node.mjs`, expands `src/infra/run-node.test.ts`, and records a changelog entry.

## Best Possible Solution

Extend the runtime-postbuild preflight to require the staged OpenClaw SDK alias package and wrappers whenever runtime staging would create them, with a regression that removes one under clean stamps.

## Reproduction Assessment

Yes, source-reproducible. With matching build/runtime-postbuild stamps and clean git status, deleting `dist/extensions/node_modules/openclaw` is not included in the PR's required-output list, so `resolveRuntimePostBuildRequirement` can still return `clean`.

## Solution Assessment

No, not as currently patched. The run-node/runtime-postbuild seam is the right place to fix this, but the implementation must include the alias package and `plugin-sdk` wrappers created by `stageBundledPluginRuntime`.
