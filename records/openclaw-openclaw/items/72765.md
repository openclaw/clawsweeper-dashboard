## Summary

Keep open. The underlying data-loss bug is still present on current main and the production fix is the right narrow direction, but this PR is not merge-ready because its regression is still aimed at a stale gateway sessions test file and the user-facing gateway fix needs a changelog entry.

## What This Changes

The PR adds an active-run interruption guard to `sessions.compact` when `maxLines` is used, makes the tracked-run check tolerate missing `chatAbortControllers`, and adds a regression test for a stuck embedded run.

## Best Possible Solution

Land a rebased narrow gateway fix that calls `interruptSessionRunIfActive` before the `maxLines` transcript archive/rewrite, keeps the active-run regression in the current compaction test suite, and records the fix under Unreleased fixes in `CHANGELOG.md`.

Yes. A high-confidence source-level reproduction is to mark an embedded run active for a session, make `waitForEmbeddedPiRunEnd` return false, then call `sessions.compact` with `maxLines`; current main can reach the archive/rewrite path without first interrupting the active run.

## Solution Assessment

Yes in direction, not in the current branch form. Reusing `interruptSessionRunIfActive` is the narrowest maintainable production fix because other session mutation paths already use it, but the regression and changelog need to be adapted to current main before merge.
