## Summary

Keep this PR open. It targets a real transcript-redaction gap, but the current branch still has merge-blocking security/API issues: delivery mirrors would ignore active redaction config, blocked writes can report success, and internal guard plumbing is exposed through a public SDK barrel.

## What This Changes

The branch wraps raw session transcript append paths in `guardSessionManager`, threads config to some call sites, changes Codex transcript mirroring/update behavior, exports the guard through the Plugin SDK runtime barrel, updates generated SDK checksums, and adds transcript redaction/idempotency tests.

## Best Possible Solution

Land a corrected, maintainer-reviewed security fix that applies the shared transcript guard to every raw transcript append path, preserves transcript update semantics, threads the active `OpenClawConfig` through production callers, handles guard-blocked writes explicitly, and keeps any new SDK surface narrow or deliberately documented/versioned.

## Reproduction Assessment

Yes. Current main shows the raw append paths, and the branch-level blockers can be reproduced with an outbound delivery mirror using custom `logging.redactPatterns` or `redactSensitive: "off"`, plus a `before_message_write` hook that blocks an assistant write.

## Solution Assessment

No. The direction is right, but the current PR is not the best maintainable fix until it threads config through all production mirror callers, treats blocked guarded appends as failed/no-op writes, and resolves the public SDK exposure through a narrow internal seam or explicit versioned API documentation.

  - body: Adding a `config` option to the transcript helper is not enough while production delivery mirror callers still omit their active `cfg`.   - confidence: 0.87

  - body: Exporting `guardSessionManager` from `openclaw/plugin-sdk/agent-harness-runtime` promotes mutable SessionManager redaction/update plumbing as a third-party Plugin SDK contract. Use a narrow bundled/internal seam, or document and version this intentionally before syncing the public baseline.
  - confidence: 0.8
