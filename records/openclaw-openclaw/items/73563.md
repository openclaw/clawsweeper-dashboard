## Summary

Keep open: the PR targets a real remaining transcript-redaction gap, but it is a security-sensitive SDK/runtime change with unresolved correctness and public API boundary blockers before merge.

## What This Changes

The PR wraps several session transcript write paths with `guardSessionManager`, threads config/update-mode options, updates Plugin SDK API metadata, and adds transcript redaction/update tests.

## Best Possible Solution

Land a maintainer-reviewed security fix that applies the same redaction and hook-blocking semantics to every transcript persistence route while keeping any new SDK seam narrow, documented, and versioned.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: write a secret-bearing assistant message through the delivery mirror, gateway injection, or Codex mirror path on current main and inspect the JSONL transcript, where the current helper writes the message payload directly without the shared redaction guard.

## Solution Assessment

No. Reusing `guardSessionManager` is the right direction, but the proposed patch still needs blocked-write handling and a narrower SDK/API boundary before it is the safest maintainable fix.
