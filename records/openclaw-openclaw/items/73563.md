## Summary

Keep this PR open. Current main still has unguarded transcript append paths, so the security fix is relevant, but the latest PR head still has blocking issues around config propagation, blocked-write success reporting, and SDK surface exposure.

## What This Changes

The PR wraps several session transcript append paths with the shared transcript guard, adds update-mode and config plumbing, updates the Codex transcript mirror and gateway injection paths, syncs the Plugin SDK baseline, and adds transcript redaction/idempotency tests.

## Best Possible Solution

Land a corrected security fix that guards every raw transcript append path, passes active OpenClawConfig through all production mirror callers, reports blocked writes as non-success, preserves update semantics, and keeps any SDK seam narrow, documented, and versioned.

## Reproduction Assessment

Yes. A focused reproduction is to install a before_message_write hook that blocks an assistant write and call the guarded transcript helper; PR head can return ok: true without a persisted message id. A second reproduction is to set logging.redactSensitive or custom redactPatterns and exercise outbound mirror delivery paths whose callers still omit cfg.

## Solution Assessment

No. The direction is right, but this PR head is not yet the best maintainable fix because it leaves production config propagation gaps, reports blocked appends as successful, and exposes internal guard plumbing through a broad public runtime barrel.
