## Summary

Keep this PR open. The security fix is aimed at a real current-main transcript-redaction gap, but the latest head still has concrete correctness and security/API-boundary blockers that should be amended before merge.

## What This Changes

The PR wraps several session transcript append paths with the shared transcript guard, adds config/update-mode plumbing, adjusts the Codex transcript mirror, updates Plugin SDK API output, and adds transcript redaction regression tests.

## Best Possible Solution

Land a corrected, maintainer-reviewed security patch that applies the shared guard to every raw transcript append path, propagates the active OpenClawConfig through production callers, treats hook-blocked writes as non-success, and keeps any SDK seam narrow, documented, and versioned.

## Reproduction Assessment

Yes. Current main shows raw transcript append paths, and the PR-head blockers can be reproduced with a before_message_write hook that blocks an assistant write or with custom logging.redactPatterns/redactSensitive="off" on outbound mirror paths whose callers still omit the active config.

## Solution Assessment

No. The implementation direction is right, but the current PR is not yet the best maintainable fix because it leaves active config out of outbound mirrors, can report success for blocked writes, and exports internal transcript guard plumbing through a broad SDK barrel.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Thread config through outbound transcript mirrors:** `src/infra/outbound/deliver.ts:1297-1302`
  - body: The new transcript helper only honors `logging.redactPatterns` and `redactSensitive` when a config object is supplied, but this production mirror call still omits `params.cfg`; the plugin-handled send mirror in `outbound-send-service` likewise omits `params.ctx.cfg`. Those delivered-message transcript writes fall back to default redaction and can ignore operator redaction settings.
  - confidence: 0.9
- **[P2] Handle blocked guard appends as failed writes:** `src/config/sessions/transcript.ts:290-291`
  - body: `guardSessionManager` can return `undefined` from `appendMessage` when a `before_message_write` hook blocks the write. Returning `ok: true` with that value reports a successful transcript append even though no entry was persisted and no valid message id exists.
  - confidence: 0.87
- **[P2] Keep transcript guard out of the runtime barrel:** `src/plugin-sdk/agent-harness-runtime.ts:129`
  - body: Exporting `guardSessionManager` from `openclaw/plugin-sdk/agent-harness-runtime` makes mutable transcript redaction, hook blocking, idempotency, and update emission plumbing part of the broad Plugin SDK surface. Please use a narrow internal/bundled seam or explicitly document and version the public contract before syncing the baseline.
  - confidence: 0.83
