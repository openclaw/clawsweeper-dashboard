## Summary

Keep this PR open. It targets a real security gap in current main and later commits appear to address the earlier sessionKey, idempotency, and updateMode review comments, but the latest head still has blocking issues around active config passthrough, blocked append handling, and public SDK surface exposure.

## What This Changes

The PR wraps several session transcript append paths in the shared transcript guard, adds update-mode/config plumbing, updates the Codex transcript mirror, exports the guard through the plugin SDK runtime barrel, and adds redaction/update regression tests.

## Best Possible Solution

Land a corrected, maintainer-reviewed security fix that applies the shared transcript guard to every raw transcript append path, preserves transcript update semantics, passes the active OpenClawConfig through production callers, treats hook-blocked writes as non-success, and keeps any SDK seam narrow, documented, and versioned.

## Reproduction Assessment

Yes. Current main shows raw SessionManager append paths, and the PR-head blockers can be reproduced with a before_message_write hook that blocks an assistant write or with custom logging.redactPatterns/redactSensitive="off" on outbound mirror paths whose callers still omit the active config.

## Solution Assessment

No. The implementation direction is right, but the current PR is not yet the best maintainable fix because it leaves active config out of production delivery mirrors, can report success for blocked writes, and promotes internal transcript guard plumbing through a broad public SDK barrel.
