## Summary

Keep open: this PR targets a real persisted-transcript redaction gap, but the current patch still needs after-fix real behavior proof and has blocking transcript append integrity plus SDK-boundary issues before it is safe to merge.

## What This Changes

The PR wraps transcript append call sites with `guardSessionManager`, threads config/update-mode options, exports the guard through the agent harness runtime barrel, and adds transcript redaction/update tests.

## Best Possible Solution

Land a maintainer-reviewed security fix that applies redaction before every transcript persistence sink while preserving OpenClaw's locked append semantics, keeping the guard on a narrow documented/internal seam, adding a changelog entry, and including redacted real-run proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: append a secret-bearing assistant message through the current transcript helper and inspect the JSONL, because current main serializes `params.message` directly while docs promise persisted transcript redaction. I did not run a live repro because this review was read-only.

## Solution Assessment

No. Reusing the existing guard is the right direction, but the current patch should preserve the locked append helper semantics and avoid promoting the guard through the broad public SDK barrel; a guarded locked append wrapper or narrow internal runtime seam is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep guarded appends on the locked append path:** `src/config/sessions/transcript.ts:284`
  - body: This replaces `appendSessionTranscriptMessage`, which queues writes and acquires `acquireSessionWriteLock`, with a fresh `SessionManager.open(...).appendMessage`. The upstream `SessionManager` appends from its preloaded leaf without OpenClaw's cross-process lock, so concurrent transcript writes can append sibling leaves and hide one message from the default leaf path. Please preserve the locked append semantics while applying redaction.
  - confidence: 0.87
- **[P2] Keep transcript guards off the broad runtime barrel:** `src/plugin-sdk/agent-harness-runtime.ts:129`
  - body: Exporting `guardSessionManager` from `openclaw/plugin-sdk/agent-harness-runtime` promotes redaction, hook-blocking, idempotency, and update-emission internals to a public SDK surface without docs or a versioned contract. Use a narrow internal/bundled seam or document the supported contract before exposing it.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/config/sessions/transcript.ts:284`
  - body: This is a user-facing security fix for persisted session transcript redaction, but the PR files do not include `CHANGELOG.md`. Repo policy requires a changelog entry for fix/security behavior changes.
  - confidence: 0.78
