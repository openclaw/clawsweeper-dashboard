## Summary

Keep open: current main and the latest release still reject OAuth profiles for OpenAI batch audio transcription, and the issue is paired with open fix PRs that need maintainer auth-contract review before the issue can close.

## What This Changes

The issue asks OpenClaw to restore OAuth-backed OpenAI batch audio transcription or explicitly validate and document that `tools.media.audio` OpenAI STT is API-key-only.

## Best Possible Solution

Decide and implement the OpenAI batch STT auth contract: land a proven OAuth-backed path if it is supported, or keep API-key-only behavior and add targeted validation, docs, and doctor guidance for OAuth-profile audio configs.

Yes. Source inspection shows an explicit OpenAI OAuth `tools.media.audio` profile reaches `resolveApiKeyForProvider` with `modelApi: "openai-audio-transcriptions"` and is rejected before transcription; the reporter also supplied direct OAuth runtime probes outside OpenClaw.
