## Summary

## What This Changes

The PR changes unsupported Gateway service resolution to return a read-only unsupported adapter, keeps daemon status readable, and adds regression coverage for unsupported start, stop, and restart lifecycle paths.

## Best Possible Solution

Land the narrow adapter-source fix after maintainer compatibility acceptance, keeping read-only status readable while mutating lifecycle commands fail at the service-check boundary.

## Reproduction Assessment

Yes from source inspection: current main throws from `resolveGatewayService()` on unsupported platforms before status can gather service details, matching the linked OpenBSD stack. I did not run tests because this was a read-only review.

## Solution Assessment

Yes, with maintainer compatibility acceptance: the source-boundary unsupported adapter is cleaner than a status-only catch because status and lifecycle callers already share the GatewayService contract. The PR also avoids the earlier wrong shape where unsupported `isLoaded()` looked like an ordinary not-loaded service.
