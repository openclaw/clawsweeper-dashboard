## Summary

Keep this PR open. Current main still short-circuits fresh persisted token totals before token-threshold preflight compaction when transcript-byte compaction is inactive, so the intended bug fix is not implemented; the PR is focused, but it needs maintainer consolidation or rebase because overlapping compaction work is still open.

## What This Changes

This PR changes `runPreflightCompactionIfNeeded` so fresh persisted token totals only take the fast-path return when below the preflight threshold, adds regression coverage, and adds a changelog entry.

## Best Possible Solution

Land one consolidated fix in `runPreflightCompactionIfNeeded` that lets fresh persisted totals at or above `contextWindowTokens - reserveTokensFloor - softThresholdTokens` reach the existing threshold helper, keeps below-threshold fresh totals on the cheap fast path, preserves transcript-byte compaction, and includes focused regression tests for both sides of the threshold.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a `SessionEntry` with `totalTokensFresh: true`, positive `totalTokens` above the computed threshold, and no active transcript-byte trigger; current main returns before `shouldRunPreflightCompaction` is called.

## Solution Assessment

Yes in direction, but not merge-ready as-is. The threshold-gated fast path is the narrow maintainable fix, provided it is rebased over current main and keeps the transcript-byte trigger independent.
