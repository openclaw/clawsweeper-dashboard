## Summary

Keep open: current main still has the fresh-token preflight compaction bug, and this PR is pointed at the right fix, but it needs maintainer consolidation with overlapping work and a rebase that preserves current transcript-byte compaction behavior.

## What This Changes

This PR gates `runPreflightCompactionIfNeeded`'s fresh-token fast path on the computed preflight token threshold, adds regression tests for fresh totals above and below the threshold, and adds a changelog entry.

## Best Possible Solution

Land one consolidated `runPreflightCompactionIfNeeded` fix that lets fresh persisted totals at or above `contextWindowTokens - reserveTokensFloor - softThresholdTokens` reach the existing threshold helper, keeps below-threshold fresh totals cheap, preserves transcript-byte compaction, and includes focused regression tests for both paths.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a `SessionEntry` with `totalTokensFresh: true`, a positive `totalTokens` above the computed threshold, and no active transcript-byte trigger; current main returns before `shouldRunPreflightCompaction` is called.

## Solution Assessment

Yes in direction, but not as-is over current main. The threshold-gated fast path is the narrow maintainable fix if it is rebased with the transcript-byte trigger preserved and consolidated with the overlapping open implementation.
