## Summary

Keep open. Current main still lets auto memory embeddings select OpenAI credentials independently of a DeepSeek chat model; the tool failure path is mitigated, but the reported credential-selection behavior remains unresolved.

## What This Changes

The issue asks memory-core to stop resolving OpenAI embedding credentials from OPENAI_API_KEY for DeepSeek-based agents after the 2026.4.25 regression.

## Best Possible Solution

Keep the non-fatal memory_search behavior, then make embedding selection deterministic through clearer provider pinning/diagnostics or a DeepSeek-owned embedding adapter if maintainers choose that contract.

## Reproduction Assessment

Yes for the source-level credential path: with memorySearch.provider unset or auto and a resolvable OpenAI key, current main can select OpenAI memory embeddings without consulting the DeepSeek chat model. The exact 401 needs the reporter's external key/proxy setup.

## Solution Assessment

Unclear. Automatically reusing DeepSeek chat credentials for embeddings may cross provider credential boundaries; explicit provider pinning diagnostics or a DeepSeek-owned embedding adapter is safer to evaluate first.
