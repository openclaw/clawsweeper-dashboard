## What This Changes

This PR updates the Nextcloud Talk webhook monitor to parse signed JSON separately, ignore valid Activity Streams payloads that are not supported chat-message Notes, and adds a signed non-message replay fixture.

## Best Possible Solution

Land the plugin-local parser change after redacted real webhook proof shows a signed non-message Nextcloud Talk event returns 200 without dispatch and a malformed supported Note still returns 400.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, source inspection shows current main and v2026.6.1 return 400 for any signed payload that misses the strict Note schema, and the linked issue provides concrete Nextcloud Talk file-share steps; I did not run a live Nextcloud server in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? Yes for the code boundary: the Nextcloud Talk plugin owns webhook decoding, and the PR keeps signature/header/body-limit checks in place while distinguishing unsupported signed Activity Streams events from malformed supported Note messages.
