## Summary

This PR addresses a real Nextcloud Talk contract mismatch and the implementation is narrowly scoped: current `main` rejects every signed schema miss, while upstream Talk sends several signed Activity Streams event classes beyond chat-message Notes. The staged JSON/schema parsing and message-shaped rejection preserve the existing malformed-Note behavior, and I found no blocking correctness or security defect; however, the contributor’s evidence remains replay-harness-only, so real after-fix runtime proof is still required before merge. Likely related people: Vincent Koc (high confidence, recent webhook/replay work), Peter Steinberger (high confidence, webhook pipeline and security refactors), and Jacob Tomlinson (medium confidence, recent webhook ingress hardening).

## What This Changes

The PR separates signed webhook JSON decoding from chat-message schema validation, acknowledges non-message Activity Streams payloads without dispatch, and adds reusable signed fixtures plus regression coverage.

## Best Possible Solution

Keep the two-stage JSON/message-schema boundary, acknowledge signed non-message Activity Streams events without dispatch, retain strict rejection for malformed message-shaped Notes, and verify both outcomes through a real Nextcloud Talk or standalone production-server run.

## Reproduction Assessment

Yes, source-reproducible with high confidence: current `main` sends every signed payload that misses the strict Note schema to HTTP 400, while upstream Talk documents signed non-Note event envelopes. This review did not run a live Nextcloud server.

## Solution Assessment

Yes, the proposed split between malformed JSON, malformed supported message payloads, and valid non-message Activity Streams events is the narrowest maintainable fix; explicit real-runtime proof is still needed to validate the production envelope.
