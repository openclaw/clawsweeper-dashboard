## Summary

The implementation matches Nextcloud Talk’s upstream contract: the signed bot endpoint receives non-chat Activity Streams events as well as Note messages, so acknowledging authenticated non-message events avoids false bot failures while retaining strict validation for message-shaped payloads. The PR should remain open because it is paired with the still-open issue, is currently draft, and its external-contributor proof is fixture-based rather than a real Nextcloud or standalone production-server run.

## What This Changes

The PR separates webhook JSON decoding from strict chat-Note validation, acknowledges signed non-message Activity Streams payloads without dispatch, and adds a reusable signing fixture plus a Document-event regression test.

## Best Possible Solution

Keep the two-stage authenticated JSON and message-schema boundary, acknowledge contract-valid signed non-message Talk activities, retain HTTP 400 for malformed message-shaped Notes, and verify both outcomes through the production webhook server before merge.

## Reproduction Assessment

Yes. Current main deterministically routes a signed non-Note Activity Streams payload through the strict Note-schema failure to HTTP 400, matching the linked Nextcloud file-share report; this review did not run a live Nextcloud instance.

## Solution Assessment

Yes, structurally. Separating valid JSON and signed Activity-envelope classification from strict supported-Note validation is the narrowest maintainable fix, provided the production path confirms malformed message-shaped events remain errors.
