## Summary

Keep open: the PR targets a real Nextcloud Talk bug, but it is not merge-ready because the ignore heuristic can turn malformed signed message-like payloads into HTTP 200, and the supplied proof remains fixture-based Vitest output rather than live or standalone runtime proof.

## What This Changes

The PR changes Nextcloud Talk webhook decoding to parse signed JSON before the strict Note schema, ignore non-message Activity Streams payloads with HTTP 200/no dispatch, and add signed fixture replay coverage.

## Best Possible Solution

Keep the JSON-before-schema split, but tighten it so partial message-shaped signed payloads still return 400, add replay coverage for ignored file/reaction/lifecycle variants and partial-message invalid variants, and require redacted real webhook-server proof before merge.

## Reproduction Assessment

Yes. Source inspection shows current main routes signed non-Note Activity Streams payloads through the strict Note schema and returns HTTP 400 before the ignore branch; I did not run a live Nextcloud server in this read-only review.

## Solution Assessment

No, not yet. The JSON-before-schema boundary is the right direction, but this implementation is too loose for partial message-shaped payloads and still needs real runtime proof.
