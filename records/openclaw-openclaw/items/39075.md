Full review comments:

- **[P1] Gate preflight transcription on sender authorization:** `src/telegram/bot-message-context.ts:363-369`
  - body: This moves audio preflight before `commandGate` is computed and the new predicate no longer checks whether the sender is authorized. Current main includes the merged authorization gate from https://github.com/openclaw/openclaw/pull/57566 so unauthorized group voice messages do not trigger STT; this branch would reintroduce that security and cost boundary regression.
  - confidence: 0.96
- **[P2] Preserve running state during in-flight typing ticks:** `src/channels/typing-lifecycle.ts:72`
  - body: `scheduleNext` clears `timer` before awaiting `tick()`, but `isRunning()` still returns only `timer !== undefined`. `createTypingController` uses `isRunning()` before TTL cleanup, so if the TTL fires while `onTick` is awaiting I/O, cleanup can be skipped and typing lifecycle state can remain active longer than intended.
  - confidence: 0.9
- **[P2] Keep markdown chunking markdown-aware without fences:** `src/auto-reply/chunk.ts:321-323`
  - body: The no-fence fast path now sends all long markdown through `chunkText()`, whose shared splitter trims chunk ends and starts. That can strip indentation or newline context from no-fence markdown such as indented code blocks or nested lists, changing rendered replies compared with the previous markdown splitter.
  - confidence: 0.84
- **[P2] Trim map-backed option values before lookup:** `src/auto-reply/thinking.ts:162`
  - body: These normalizers used to trim before lowercasing; the new map lookups use raw `toLowerCase()` for verbose, usage, elevated, and reasoning values. Existing config or directive inputs with harmless leading or trailing spaces will stop normalizing, so keep the trim step before each lookup.
  - confidence: 0.82
