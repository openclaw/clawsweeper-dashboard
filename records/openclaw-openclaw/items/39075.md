- **[P1] Gate audio transcription before starting STT:** `src/telegram/bot-message-context.ts:363-369`
  - body: Current main computes Telegram command authorization before audio preflight and gates STT with `senderAllowedForAudioPreflight`. This branch starts `transcribeFirstAudio` before that authorization result is available, so an unauthorized group voice message can be sent to STT before the later access block drops it.
  - confidence: 0.93
- **[P2] Keep typing loops running during in-flight ticks:** `src/channels/typing-lifecycle.ts:72`
  - body: `scheduleNext` clears `timer` before awaiting `tick()`, but `isRunning()` still only checks `timer`. `createTypingController` uses `isRunning()` for TTL cleanup, so a TTL callback that fires while `onTick` is awaiting I/O can treat the loop as stopped and skip cleanup.
  - confidence: 0.87
- **[P2] Preserve markdown spacing in the no-fence path:** `src/auto-reply/chunk.ts:321-323`
  - body: The fast path sends all markdown without fences through `chunkText()`, whose shared splitter trims chunk ends and starts. That can strip indentation or newline context from long markdown replies such as indented code blocks or nested lists compared with the current markdown splitter.
  - confidence: 0.78
- **[P2] Preserve max thinking as its own level:** `src/auto-reply/thinking.ts:72`
  - body: Current main treats `max` as a distinct `ThinkLevel` and has regression coverage for `normalizeThinkLevel("max")`. This map downgrades `max` to `high`, so rebasing this optimization without preserving the current contract would silently change user/profile thinking settings.
  - confidence: 0.76
