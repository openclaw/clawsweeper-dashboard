## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve audio preflight authorization before transcription:** `src/telegram/bot-message-context.ts:363-369`
  - body: Current main gates Telegram audio preflight through command/access authorization and also transcribes DM voice notes. This PR starts transcription from the group-only preflight conditions before the command block, so unauthorized group audio can reach STT and the DM voice path is lost; keep the sender authorization and DM contract when parallelizing.
  - confidence: 0.9
- **[P2] Keep typing active while ticks are in flight:** `src/channels/typing-lifecycle.ts:42-43`
  - body: The timeout callback clears timer before awaiting tick(), but isRunning() still checks only timer. During a slow sendChatAction call the loop reports stopped, so TTL cleanup can be skipped or lost; track loop activity separately or count tickInFlight as running.
  - confidence: 0.86
- **[P2] Preserve markdown whitespace in no-fence chunks:** `src/auto-reply/chunk.ts:322-323`
  - body: The new no-fence fast path routes markdown through chunkText(), which trims chunk boundaries. Indented code blocks and nested-list whitespace can render differently from the existing markdown splitter, so keep a whitespace-preserving markdown path or cover this fast path with regression tests.
  - confidence: 0.82
