Full review comments:

- **[P1] Gate audio transcription before starting STT:** `src/telegram/bot-message-context.ts:363-369`
  - body: Current main gates Telegram audio preflight on command authorization before calling STT. This branch starts `transcribeFirstAudio` before `commandGate` is computed, so an unauthorized group voice message can be sent to STT before the later access block drops it.
  - confidence: 0.94
- **[P2] Keep typing loops running during in-flight ticks:** `src/channels/typing-lifecycle.ts:42-43`
  - body: The timeout callback clears `timer` before awaiting `tick()`, but `isRunning()` still only checks `timer`. `createTypingController` uses that state for TTL cleanup, so cleanup can be skipped while `onTick` is awaiting I/O.
  - confidence: 0.87
- **[P2] Preserve markdown spacing in the no-fence path:** `src/auto-reply/chunk.ts:321-323`
  - body: The new no-fence fast path sends long markdown through `chunkText()`, whose shared splitter trims chunk boundaries. That can strip indentation or newline context from long markdown replies such as nested lists or indented code blocks.
  - confidence: 0.79
- **[P2] Preserve max thinking as its own level:** `src/auto-reply/thinking.ts:72`
  - body: Current main treats `max` as a distinct thinking level with regression coverage. This branch maps `max` to `high`, so rebasing the optimization without preserving the current contract would silently change provider/profile thinking settings.
  - confidence: 0.76
