## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve DM audio preflight transcription:** `src/telegram/bot-message-context.ts:363-369`
  - body: The new `needsPreflightTranscription` gate requires `isGroup`, so direct voice notes without text never call `transcribeFirstAudio` and fall back to `<media:audio>`. Current Telegram coverage expects DM voice notes to be preflight-transcribed, so keep the DM path while parallelizing the group path.
  - confidence: 0.88
- **[P2] Preserve running state during awaited typing ticks:** `src/channels/typing-lifecycle.ts:42-43`
  - body: The timeout callback clears `timer` before awaiting `tick()`, but `isRunning()` still only checks `timer !== undefined`. If the typing TTL fires while `onTick` awaits I/O, `refreshTypingTtl` can return early and skip cleanup for that run.
  - confidence: 0.84
- **[P2] Preserve markdown whitespace on no-fence chunks:** `src/auto-reply/chunk.ts:323`
  - body: The no-fence fast path delegates to `chunkText()`, whose shared splitter trims chunk boundaries. Long markdown without fenced blocks can rely on indentation or leading newlines, so this can change rendered lists or indented code at split boundaries.
  - confidence: 0.81
