## Summary

Keep open. The PR targets a valid gateway startup regression, but the latest diff still has merge-blocking defects in the new async skill scan and plugin tool factory cache key behavior.

## What This Changes

This PR adds an async workspace skill snapshot path for gateway startup, caches hydrated skill snapshots, switches plugin tool factory config keys toward object identity, updates cron skill-snapshot mocks, and adds a changelog entry for #75944.

## Best Possible Solution

Repair this PR or a replacement branch so startup avoids blocking while preserving skill discovery and value-based plugin tool cache semantics, then continue review against #75944.

## Reproduction Assessment

Yes for the PR defects: a Node 22 probe reproduces the fs.Dir double-close failure, and current tests define the equivalent-context cache contract. The original Windows/MiniMax freeze has logs and steps, but this read-only pass did not live-reproduce that environment.

## Solution Assessment

No. The performance direction is plausible, but the implementation should use async directory iteration without a second close and memoize expensive config cache keys while preserving equivalent config and runtime-revision semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Let async directory iteration own the handle:** `src/agents/skills/workspace.ts:267`
  - body: `fs.Dir` async iteration closes the handle when the loop exits. This extra `await handle.close()` throws `ERR_DIR_CLOSED`, is caught by the outer catch, and makes child-directory scans return `[]`, so the new startup path can silently lose bundled, workspace, managed, and plugin skills.
  - confidence: 0.96
- **[P2] Preserve value-based cache keys:** `src/plugins/tool-factory-cache.ts:44-46`
  - body: `getPluginToolFactoryCacheObjectId()` always returns an id for any object, so the `resolveRuntimeConfigCacheKey()` fallback below is unreachable. Newly constructed but equivalent configs no longer share factory cache entries, breaking the existing equivalent-request-context contract.
  - confidence: 0.9
