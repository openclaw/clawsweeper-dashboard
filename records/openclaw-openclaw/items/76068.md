## Summary

Keep open. The PR targets a real gateway regression report, but the proposed async skills scan is not mergeable as written because it double-closes the async directory handle and causes ordinary child-directory skills to disappear from the new agent startup path.

## What This Changes

The PR changes gateway agent startup to build skill snapshots asynchronously, adds skill snapshot hydration caching, changes plugin tool-factory config cache keys to object identity, and documents the gateway performance fix.

## Best Possible Solution

Keep the performance direction, but first make async skill discovery behaviorally identical to the current synchronous scanner and cover it with async parity tests before re-evaluating the gateway/tool-cache changes.

## Reproduction Assessment

Yes. A focused Node v24.14.1 probe reproduces the double-close ERR_DIR_CLOSED behavior, and the PR routes gateway startup through the affected async scanner.

## Solution Assessment

No. Async startup scanning may be the right direction, but this implementation should use an async traversal pattern that does not close an already auto-closed fs.Dir and should preserve the existing skill discovery contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Stop closing the async directory handle twice:** `src/agents/skills/workspace.ts:267`
  - body: `for await...of` over an `fs.Dir` closes the handle when iteration exits. This added `finally` closes it again, which throws `ERR_DIR_CLOSED`; the surrounding catch then returns an empty directory scan. Since agent startup now calls `buildWorkspaceSkillSnapshotAsync`, normal `skills/<name>/SKILL.md` directories will be omitted from startup snapshots.
  - confidence: 0.96
