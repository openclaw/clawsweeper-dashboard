## Summary

Keep open. The linked gateway freeze remains actionable and the PR targets relevant hot paths, but the current patch is not mergeable because it would drop discovered skills on the async startup path and weakens the existing plugin tool factory cache contract.

## What This Changes

The PR switches agent startup to a new async skill snapshot builder, adds hydration caching, changes plugin tool factory cache keys, and documents the gateway startup regression fix in the changelog.

## Best Possible Solution

Keep the PR open and repair the async skill loader and cache-key strategy before review for #75944 continues.

## Reproduction Assessment

Yes for the PR defects: a Node 22 probe reproduces the fs.Dir double-close behavior, and current tests define the equivalent-context cache contract. The linked user bug has logs and steps, but this read-only review did not live-reproduce the Windows/MiniMax freeze.

## Solution Assessment

No. The performance direction is plausible, but the implementation should preserve skill discovery and equivalent-context caching, for example by using async directory iteration without a second close and by memoizing config hash/revision keys instead of replacing them with object identity only.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Let async directory iteration own the handle:** `src/agents/skills/workspace.ts:267`
  - body: `fs.Dir` async iteration closes the handle when the loop exits. This extra `await handle.close()` throws `ERR_DIR_CLOSED`, is caught by the outer catch, and makes every async child-directory scan return `[]`, so the new agent startup path silently loses bundled/workspace/managed skills.
  - confidence: 0.96
- **[P2] Preserve value-based cache keys:** `src/plugins/tool-factory-cache.ts:44-46`
  - body: `getPluginToolFactoryCacheObjectId()` always returns an id for any object, so the `resolveRuntimeConfigCacheKey()` fallback below is unreachable. Newly constructed but equivalent configs no longer share the factory cache, which breaks the existing equivalent-request-context contract and undercuts the perf fix.
  - confidence: 0.9
