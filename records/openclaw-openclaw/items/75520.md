## Summary

Keep open. The issue is paired with open PR #75521, which explicitly closes this issue, so the issue should stay open until that PR is reviewed, merged, or closed. Current main has also added a partial hot-path mitigation that disables bundled runtime-dependency repair during plugin-tool fallback loads, but I did not find the scoped-active-to-unscoped-request registry reuse described by the issue implemented on main.

## What This Changes

The issue requests fixing a plugin loader cache-key mismatch where a scoped gateway startup registry is not reused by unscoped embedded tool resolution, causing repeated plugin reload and runtime-dependency staging latency.

## Best Possible Solution

Review PR #75521 against current main and either land a narrow compatibility-preserving scoped-registry reuse with regression coverage, or close it with evidence that the 29ed5266 hot-path mitigation fully addresses the user-visible latency without weakening loader compatibility.

## Reproduction Assessment

Yes. Current main changes the runtime-deps repair behavior, so the exact 20-40s latency should be remeasured before final severity judgment.
