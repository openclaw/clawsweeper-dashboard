## Summary

Keep this open: current main and latest release still requeue unclassified Telegram spooled update failures, while the open candidate PR is not yet a safe merged fix path because it has review blockers and missing real Telegram proof.

## What This Changes

The issue asks for Telegram spooled updates with permanent module-missing or missing-harness failures, plus repeated poison failures, to be dead-lettered instead of retried indefinitely.

## Best Possible Solution

Keep this issue as the canonical report until the Telegram-owned polling-session failure policy dead-letters ERR_MODULE_NOT_FOUND and scoped repeated poison failures with focused regression coverage plus maintainer-accepted real Telegram proof or proof override.

Partly. The owner path is the Telegram polling-session failure classifier/dead-letter path, but a configurable poison retry limit adds config surface and the open PR's counter scoping needs maintainer review.
