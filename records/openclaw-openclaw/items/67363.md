## Summary

Keep open. Current main still writes valid promotion candidate snippets verbatim into `MEMORY.md`; the related contamination fixes and open PRs cover narrower artifact leaks, not the broader durable-memory write contract raised here.

## What This Changes

Request that memory-core deep dreaming distill or otherwise curate short-term promotion snippets before appending them to `MEMORY.md`.

## Best Possible Solution

Codify the durable-promotion contract, then implement it consistently in memory-core schema, CLI/docs, and tests: either document provenance-preserving raw append with a cleanup/review workflow, or add a reviewed/distilled output path that preserves source provenance and deterministic fallbacks.

## Reproduction Assessment

Yes. Source inspection on current main shows the promotion path records, ranks, rehydrates, and appends `candidate.snippet` verbatim; I did not run tests because this review was required to stay read-only.

## Solution Assessment

Unclear. A distillation pass would address the quality complaint, but maintainers should first choose the product contract because a review queue or deterministic pre-promotion filters may be safer than adding model latency and cost directly to the deep phase.
