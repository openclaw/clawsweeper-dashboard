## Summary

Keep open. Current main still carries eligible daily-memory snippets through ranking and rehydration and appends the snippet text directly to `MEMORY.md`; merged contamination fixes only cover narrower artifact leaks, while the broader distillation/review policy remains unresolved.

## What This Changes

The issue asks memory-core Dreaming deep promotion to stop appending raw daily-memory snippets to `MEMORY.md` without distillation or an explicit write policy.

## Best Possible Solution

Codify the durable promotion contract in memory-core, then align source, schema/SDK, CLI, docs, and tests so promoted output is either explicitly provenance-preserving raw text or goes through a documented reviewed/distilled path with deterministic fallback and source provenance.

## Reproduction Assessment

Yes, source-reproducible. Current main stores normalized snippets, ranks and rehydrates them, and appends `candidate.snippet` directly into `MEMORY.md`; I did not execute a live promotion because this review is read-only.

## Solution Assessment

Unclear. A distillation pass would address the quality complaint, but maintainers should first choose the durable-promotion contract because review queues, deterministic filters, raw provenance, and LLM rewriting solve different tradeoffs.
