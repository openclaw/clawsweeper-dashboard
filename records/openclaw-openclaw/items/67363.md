## Summary

Keep open. Current main still ranks eligible daily-memory snippets and appends the rehydrated snippet text directly to `MEMORY.md`; the current docs and schema describe thresholds and direct append, not a distilled or review-mode promotion contract. Related reports and PRs cover narrower dreaming-artifact leakage, but they do not resolve the broader raw valid-snippet behavior raised here.

## What This Changes

Requests clarification or a behavior change so memory-core dreaming deep promotion writes distilled long-term memory instead of raw daily-log snippets, or documents/configures raw promotion if intentional.

## Best Possible Solution

Define the durable-promotion contract for memory-core. If raw append is intentional, document that `MEMORY.md` receives provenance-preserving candidates and give operators a review/cleanup path; if it is not intentional, add a distilled or reviewed promotion path that preserves source provenance, has deterministic fallback behavior, and updates manifest schema, docs, and focused memory-core tests.

## Reproduction Assessment

Yes. The source path deterministically stores, ranks, rehydrates, and writes `candidate.snippet`, and existing tests demonstrate direct promotion into `MEMORY.md`; this review did not execute tests because the requested pass is read-only.

## Solution Assessment

Unclear. A model distillation call would address the quality problem, but it may not be the narrowest maintainable first step until maintainers choose the policy; deterministic filters or a review queue may be safer pieces depending on the intended product contract.
