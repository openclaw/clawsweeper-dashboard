## Summary

Keep open. Current main still promotes valid short-term candidates by carrying the stored snippet through ranking/rehydration and appending it directly to `MEMORY.md`; the related contamination fixes and open PR cover narrower artifact leaks, not the broader durable-memory write contract and distillation/review policy raised here.

## What This Changes

The issue asks whether memory-core deep dreaming should stop appending raw daily-memory snippets directly into `MEMORY.md` and instead distill, review, or otherwise curate promoted content.

## Best Possible Solution

Codify the final promotion contract, then update memory-core source, schema, CLI/docs, and tests so `MEMORY.md` either remains explicitly provenance-preserving raw output or receives a reviewed/distilled path with deterministic fallbacks and source provenance.

## Reproduction Assessment

Yes, source-reproducible. Current main stores normalized snippets, ranks them into promotion candidates, rehydrates the source range, and writes `candidate.snippet` into `MEMORY.md`; I did not run a live promotion because this review is read-only.

## Solution Assessment

Unclear. A distillation pass would address the quality complaint, but maintainers should first choose the durable-promotion contract because review queues, deterministic filters, and LLM rewriting solve different parts of the problem.
