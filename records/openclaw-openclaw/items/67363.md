## Summary

Keep open. Current main still preserves daily-memory snippets through recall, ranking, rehydration, and direct `MEMORY.md` append; the related merged fixes cover narrower contamination and size-pressure paths, while the broader distillation/review contract remains unresolved.

## What This Changes

Request that memory-core Deep Dreaming stop promoting raw daily-log snippets verbatim into `MEMORY.md`, or add a distilled/reviewed promotion path so long-term memory stays curated.

## Best Possible Solution

Define the durable promotion contract first, then align memory-core source, schema/SDK types, CLI, docs, and regression tests around either documented raw provenance with review tooling or a distilled/reviewed write path that preserves source provenance.

## Reproduction Assessment

Yes, source-reproducible. Current main stores normalized snippets, ranks and rehydrates them, then appends `candidate.snippet` directly into `MEMORY.md`; no live promotion run was executed because this review is read-only.

## Solution Assessment

Unclear. A distillation or review layer would address the quality complaint, but maintainers should first choose the public write policy and default behavior before automation implements one path.
