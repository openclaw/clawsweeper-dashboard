## Summary

Keep open. Current main still carries normalized daily-memory snippets through recall, ranking, rehydration, and direct `MEMORY.md` append; recent fixes cover narrower dream-artifact contamination but do not settle the broader durable-memory distillation/review contract.

## What This Changes

The issue asks memory-core Dreaming deep promotion to stop writing raw daily-log snippets directly into `MEMORY.md` without distillation or review policy.

## Best Possible Solution

Define the durable promotion contract first, then align memory-core source, schema/SDK, CLI, docs, and tests around either documented raw provenance with review tooling or a distilled/reviewed write path with source provenance and safe defaults.

## Reproduction Assessment

Yes, source-reproducible. Current main stores normalized snippets, ranks and rehydrates them, then appends `candidate.snippet` directly into `MEMORY.md`; I did not execute a live promotion because this review is read-only.

## Solution Assessment

Unclear. A distillation layer would address the quality complaint, but maintainers should first choose between raw provenance, deterministic filters, review queues, and LLM rewriting because those are different product and privacy tradeoffs.
