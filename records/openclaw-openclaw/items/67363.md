## Summary

Keep open. Current main and v2026.5.18 still carry raw daily-memory snippets through ranking, rehydration, and direct `MEMORY.md` append; related review-queue, CJK, and contamination threads do not settle the raw-vs-distilled durable-memory policy, and comments add credible distillation design plus production audit evidence.

## What This Changes

Request to stop memory-core Dreaming deep promotion from writing raw daily-memory snippets into `MEMORY.md` without distillation or review.

## Best Possible Solution

Define the durable promotion contract first, then align memory-core source, schema, CLI, docs, and regression tests around either documented raw provenance with review tooling or a distilled/reviewed write path that preserves source provenance.

## Reproduction Assessment

Yes, source-reproducible. Current main stores normalized snippets, ranks and rehydrates them, then appends `candidate.snippet` directly into `MEMORY.md`; no live promotion run was executed because this review is read-only.

## Solution Assessment

Unclear. Distillation or a review layer would address the quality complaint, but maintainers should choose the public write policy, default behavior, and privacy/provenance tradeoff before automation implements one path.
