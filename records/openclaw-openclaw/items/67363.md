## Summary

Keep open. Current main still promotes eligible daily-memory text by carrying raw snippets through ranking, rehydration, and direct `MEMORY.md` append; recent merged fixes address narrower dream-artifact contamination but do not settle the broader distillation/review contract raised by this issue and later audit comments.

## What This Changes

The issue asks whether memory-core deep dreaming should distill or review promoted daily-log snippets instead of appending raw verbatim snippets to `MEMORY.md`.

## Best Possible Solution

Codify the durable promotion contract in memory-core, then align source, schema/SDK, CLI, docs, and tests so promoted output is either explicitly provenance-preserving raw text or goes through a documented reviewed/distilled path with source provenance and deterministic fallback.

## Reproduction Assessment

Yes, source-reproducible. Current main stores normalized snippets, ranks and rehydrates them, then appends `candidate.snippet` directly into `MEMORY.md`; I did not execute a live promotion in this read-only review.

## Solution Assessment

Unclear. A distillation pass would address the quality complaint, but maintainers should first choose the durable-promotion contract because review queues, deterministic filters, raw provenance, and LLM rewriting solve different tradeoffs.
