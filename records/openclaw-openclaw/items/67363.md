## Summary

Keep open. Current main still promotes the rehydrated candidate snippet text directly into `MEMORY.md`; the related contamination fixes and open PRs address narrower artifact leakage, not the broader raw valid-snippet promotion contract raised here.

## What This Changes

The issue asks for clarification or a configurable/distilled promotion path so deep dreaming does not write raw daily-memory snippets directly into `MEMORY.md`.

## Best Possible Solution

Codify the durable-promotion contract, then implement it consistently in memory-core schema, CLI, docs, and focused tests: either document raw provenance-preserving append with a cleanup/review workflow, or add a reviewed/distilled output path that preserves source provenance and deterministic fallbacks.

## Reproduction Assessment

Yes. The current source path deterministically records, ranks, rehydrates, and appends `candidate.snippet`, and existing tests assert direct promotion into `MEMORY.md`; I did not run tests because this review was required to stay read-only.

## Solution Assessment

Unclear. A model distillation pass would address the quality complaint, but maintainers should first choose the product contract because a review queue or deterministic pre-promotion filters may be safer pieces than adding LLM latency and cost directly to the deep phase.
