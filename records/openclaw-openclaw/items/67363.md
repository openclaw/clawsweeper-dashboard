## Summary

Keep open. Current main still carries normalized daily-memory snippets through ranking, rehydration, and direct `MEMORY.md` append; related fixes address narrower contamination, size, and placeholder paths, but not the raw-vs-distilled durable-promotion contract.

## What This Changes

Request to change memory-core Dreaming deep promotion so raw daily-log snippets are not appended verbatim to `MEMORY.md` without distillation or review.

## Best Possible Solution

Define the durable promotion contract first, then align memory-core source, schema/SDK types, CLI, docs, and regression tests around either documented raw provenance with review tooling or a distilled/reviewed write path that preserves source provenance.

## Reproduction Assessment

Yes, source-reproducible. Current main stores normalized snippets, ranks and rehydrates them, then appends `candidate.snippet` directly into `MEMORY.md`; no live promotion run was executed because this review is read-only.

## Solution Assessment

Unclear. Distillation or a review layer would address the quality complaint, but maintainers should choose the public write policy, default behavior, and privacy/provenance tradeoff before automation implements one path.
