## Summary

Keep open. Current main partially addresses child-process survival with a QA parent watchdog, but the requested acceptance is still not met: temp-root cleanup still depends on normal suite teardown, restart handling still includes a fixed sleep, and qa-channel config is still emitted before the run-specific readiness check; the related implementation PR at https://github.com/openclaw/openclaw/pull/65511 was closed unmerged.

## What This Changes

Request reliable QA suite shutdown/temp-root cleanup, readiness-based restart handling, and qa-channel config gating so orphan QA runtimes do not pollute live gateway diagnostics.

## Best Possible Solution

Land a focused QA-lab repair that preserves the parent watchdog, adds suite-owned interrupt cleanup for gateway/provider/lab/temp roots, replaces restart settle sleeps with concrete readiness checks, and prevents qa-channel activation when that run's QA bus is unhealthy.
