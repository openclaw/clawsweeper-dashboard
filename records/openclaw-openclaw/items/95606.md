## Best Possible Solution

Review and land or close the active memory-lancedb fix PRs, keeping automatic conflict replacement/deprecation tracked as broader memory product work rather than part of this narrow deletion-handoff bug.

## Solution Assessment

No single current implementation is complete yet. Exposing deletable UUIDs in recall is the narrowest maintainable fix for exact deletion, while semantic replacement/deprecation belongs to the broader memory product flow tracked separately.
