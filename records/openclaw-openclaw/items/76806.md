## Summary

The branch addresses a real current-main availability bug, but it is not ready because the irreducible overflow fail-fast path still reports a resettable `context_overflow` kind and its guidance omits oversized current input. Keep the PR open for a narrow branch repair.

## What This Changes

The PR adds an `irreducible_overflow` preemptive-compaction route, fail-fast runner handling, exhausted-overflow metadata, auto-reply reset suppression for exhausted overflows, tests, docs, and a changelog entry.

## Best Possible Solution

Carry a distinct non-resettable irreducible overflow classification through embedded-runner metadata and auto-reply handling, update the copy to cover system prompt plus current input, resolve conflicts, and validate the focused runner and auto-reply tests.

## Reproduction Assessment

Yes. Source inspection on current main shows a zero-history system prompt plus current prompt overflow routes to `compact_only`, then the runner can spend the overflow compaction attempts even though compaction cannot reduce that baseline.

## Solution Assessment

No, not yet. The baseline detector is the right narrow direction, but the PR still emits resettable `context_overflow` for irreducible overflow; a distinct irreducible or otherwise non-resettable kind is the safer fix.
