## Summary

This PR should stay open: the latest head fixes the earlier route mismatch and now includes real behavior proof, but it still has two blocking correctness issues in recovery behavior and user guidance.

## What This Changes

The PR adds an irreducible context-overflow route for the embedded agent runner, fail-fast handling, reset-skip metadata, docs, changelog, and regression tests.

## Best Possible Solution

Keep the irreducible baseline fail-fast path, but limit non-resettable handling to truly irreducible overflows and make all returned guidance name both oversized system prompt and oversized current input.

## Reproduction Assessment

Yes. Source inspection shows current main routes baseline overflow into normal compaction and resettable `context_overflow` handling; the PR source and tests show the intended new route, though I did not run the live gateway reproduction in this read-only pass.

## Solution Assessment

No, not yet. The irreducible route is the right narrow shape, but the PR should not make all exhausted history overflows non-resettable and the user-facing copy must match the baseline condition.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Keep exhausted overflows resettable:** `src/auto-reply/reply/agent-runner-execution.ts:1973-1974`
  - body: `context_overflow_exhausted` is emitted after three failed compaction attempts, including history-only `compact_only` overflows where `systemPrompt + prompt` still fits. Current reset handling starts a new session and replays only recent user/assistant turns, so skipping reset for every exhausted overflow can turn a recoverable transcript/tool-context overflow into a terminal error; keep the no-reset guard limited to `irreducible_overflow` or split out a separately proven non-resettable exhausted kind.
  - confidence: 0.86

A huge current message will hit the same path, so users need guidance to shorten the current input too; align this payload and `IRREDUCIBLE_OVERFLOW_ERROR_TEXT` with the baseline condition.
  - confidence: 0.93
