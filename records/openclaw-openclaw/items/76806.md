## Summary

Keep open: this is a plausible availability bug fix, but the current PR head still makes recoverable exhausted history overflows terminal, the runtime error copy misidentifies oversized current input as only a system-prompt problem, and the external real-behavior proof gate is still unsatisfied.

## What This Changes

The PR adds irreducible context-overflow routing and new terminal overflow kinds in the embedded Pi runner, updates auto-reply reset handling, docs/changelog, and regression tests.

## Best Possible Solution

Keep the baseline irreducible-overflow fail-fast path, but bypass reset only for truly irreducible baseline overflow, keep exhausted history overflow recoverable by reset, align user-facing copy with system-plus-current-message overflow, and require current-head real behavior proof before merge.

## Reproduction Assessment

Yes, by source inspection. Current main routes baseline-sized overflows into `compact_only` and generic resettable context-overflow handling; I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

No, not yet. The core irreducible-overflow route is the right shape, but the PR still broadens non-resettable handling to `context_overflow_exhausted` and the runtime copy does not match the baseline condition.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Keep exhausted history overflows resettable:** `src/auto-reply/reply/agent-runner-execution.ts:2041-2042`
  - body: `context_overflow_exhausted` is produced whenever the retry budget is consumed, including history-only overflows where the zero-history baseline still fits. The existing reset path creates a new session and replays only recent user/assistant turns, so suppressing reset here can turn a recoverable bloated-transcript overflow into a terminal error; keep the non-resettable guard to `irreducible_overflow` unless a separate non-recoverable exhausted condition is proven.
  - confidence: 0.87
- **[P2] Mention oversized current input in overflow errors:** `src/agents/pi-embedded-runner/run.ts:1484-1486`
  - body: `irreducible_overflow` fires when `systemPrompt + prompt` exceeds the budget, but this payload blames only the system prompt and points only to skills/plugins/model size. A large current message will hit the same branch and users need the direct mitigation to shorten the current input; align this text, the exported constant, and tests with the baseline condition.
  - confidence: 0.93
