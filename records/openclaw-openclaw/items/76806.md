## Summary

Keep this PR open: the overflow circuit-breaker goal is valid, but the current patch would also skip the existing fresh-session reset for exhausted history overflows, which can leave the same bloated session bound and keep later messages failing; real behavior proof is still only test/simulation text.

## What This Changes

The PR adds terminal context-overflow kinds and preflight routing for irreducible agent prompt overflow, updates auto-reply reset handling, and changes related agents docs, changelog, and tests.

## Best Possible Solution

Land a smaller rebased patch that only makes truly irreducible baseline overflow non-resettable, keeps reset recovery for exhausted history overflow, uses accurate user guidance, restores unrelated auth coverage, and includes redacted runtime proof.

## Reproduction Assessment

Yes, by source inspection: current main routes embedded context-overflow meta errors through a reset callback that rotates to a new session, while the PR explicitly exempts context_overflow_exhausted from that branch. I did not run live gateway proof in this read-only review.

## Solution Assessment

No. The irreducible baseline detector is the right direction, but treating exhausted history overflow as non-resettable is not the safest fix because the existing reset path can recover by rotating away from oversized history.
