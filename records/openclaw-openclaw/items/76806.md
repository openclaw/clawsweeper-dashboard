## Summary

Keep open. The irreducible baseline-overflow circuit breaker is useful, but the current head still makes all exhausted overflows non-resettable, leaves system-prompt-only guidance for a system-plus-current-message condition, and lacks current-head real behavior proof.

## What This Changes

The PR adds irreducible context-overflow detection and terminal overflow kinds for the embedded Pi agent runner, updates auto-reply reset handling, and adds docs, changelog, and regression tests.

## Best Possible Solution

Keep the baseline irreducible-overflow fail-fast path, but bypass session reset only for truly irreducible overflows, keep exhausted history overflows recoverable, align runtime/docs/tests with the system-plus-current-message condition, and require current-head real behavior proof before merge.

## Reproduction Assessment

Yes, by source inspection. Current main routes baseline-sized overflows into `compact_only` and resettable `context_overflow` handling; I did not run the live gateway path in this read-only pass.

## Solution Assessment

No, not yet. The core route shape is maintainable, but the PR still broadens non-resettable handling to `context_overflow_exhausted` and the user-facing copy does not match the baseline condition.
