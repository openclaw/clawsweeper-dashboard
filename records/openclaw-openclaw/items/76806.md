## Summary

Keep this PR open; the fix direction is useful, but the current head still turns recoverable history-only overflow into a terminal irreducible overflow and leaves user guidance incomplete.

## What This Changes

The PR adds an `irreducible_overflow` precheck route, runner fail-fast handling, non-resettable overflow metadata, auto-reply reset skips, tests, docs, and a changelog entry for context-overflow compaction loops.

## Best Possible Solution

Keep the fail-fast path only for a zero-history baseline that exceeds budget, preserve `compact_only` recovery for history-driven overflow, and align user-facing guidance with both system prompt and current message causes.

## Reproduction Assessment

Yes. Source inspection shows current main routes over-budget baseline prompts into normal compaction and generic resettable overflow handling, while the PR branch creates a clear history-only overflow regression in the precheck route.

## Solution Assessment

No, not yet. The distinct non-resettable irreducible-overflow kind is the right shape, but the implementation must limit that route to baseline overflow and fix the guidance copy before merge.
