## Summary

Keep open: the availability fix is useful, but the current PR head still turns recoverable history-only overflow into terminal `irreducible_overflow`, and the user-facing runtime guidance still omits oversized current input.

## What This Changes

The PR adds `irreducible_overflow` and `context_overflow_exhausted` metadata, fail-fast embedded-runner handling, auto-reply reset skips, tests, docs, and a changelog entry for context-overflow compaction loops.

## Best Possible Solution

Keep the fail-fast, non-resettable path only for zero-history baseline overflow, preserve `compact_only` for history-only overflow, and align runtime/docs guidance with both oversized system prompts and oversized current messages.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main routes over-budget zero-history baselines through recoverable compaction and auto-reply can reset generic context-overflow results; the PR regression is also source-reproducible from the changed branch and its contradictory test.

## Solution Assessment

No, not yet. The baseline detector and non-resettable metadata are the right direction, but the PR must not classify recoverable history-only overflow as irreducible and should update runtime copy to mention current input.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Preserve compaction for history-only overflow:** `src/agents/pi-embedded-runner/run/preemptive-compaction.ts:119-120`
  - body: When the zero-history baseline fits but session history pushes the prompt over budget, `toolResultReducibleChars` is commonly `0` because there are no tool-result messages. This branch now returns `irreducible_overflow`, so normal long-history sessions skip compaction and surface a terminal error even though compaction is the recovery path; keep this case on `compact_only` unless the baseline itself exceeds budget.
  - confidence: 0.95

  - body: The route is triggered by the baseline `systemPrompt + prompt` with no history, but this user-facing message blames only the system prompt and skills/plugins. A very large current message can trigger the same path, so users need guidance to shorten the current message as well as reduce skills/plugins or choose a larger-context model.
  - confidence: 0.87
