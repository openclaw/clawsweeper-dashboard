## Summary

Keep open: the PR is a plausible availability fix for context-overflow loops, but current head still fails `check-test-types`, lacks current-head real behavior proof, and has runtime review issues around reset handling and user-facing overflow guidance.

## What This Changes

The PR adds irreducible context-overflow routing and overflow-exhaustion error kinds in the embedded Pi runner, updates auto-reply reset handling, and adds compaction docs/changelog/tests.

## Best Possible Solution

Keep the zero-history irreducible-overflow fail-fast path, but guard payload access in the new test, align error copy with the system-plus-current-message condition, keep or prove the exhausted-overflow reset policy, add auto-reply reset boundary coverage, and require current-head real proof.

## Solution Assessment

No, not yet. The central baseline route is maintainable, but current head fails typecheck and still has correctness concerns around exhausted-overflow reset behavior and system-prompt-only guidance for a system-plus-current-input condition.
