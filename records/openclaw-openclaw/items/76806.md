## Summary

## What This Changes

This PR adds irreducible context-overflow routing and terminal overflow kinds for embedded Pi compaction, changes auto-reply reset handling, and updates tests, docs, and changelog.

## Best Possible Solution

Keep the irreducible-baseline fail-fast direction, but make only true irreducible overflows non-resettable, fix the user-facing guidance and typecheck failure, and require redacted current-head runtime proof before merge.

## Reproduction Assessment

Yes, by source inspection. Current main routes baseline-sized overflows into compact_only and then into resettable context_overflow handling, so an oversized system-prompt-plus-current-message baseline can repeat after reset; I did not run a live gateway reproduction.

## Solution Assessment

No, not yet. The route shape is maintainable, but the patch still makes exhausted history overflows terminal, gives inaccurate guidance for large current inputs, and fails check-test-types.
