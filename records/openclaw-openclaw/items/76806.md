## Summary

Keep this PR open: the core irreducible-overflow direction is useful, but the current head still has blocking correctness issues, a failing test-type CI check, and only test/simulation text instead of current-head real behavior proof.

## What This Changes

The branch adds irreducible overflow routing/error kinds for embedded Pi compaction, changes auto-reply reset handling, and adds docs, changelog, and regression tests.

## Best Possible Solution

Keep the baseline irreducible-overflow fail-fast path, but limit non-resettable handling to truly irreducible overflows, keep recoverable history overflows resettable unless proven otherwise, fix the test type error, align user copy with system prompt plus current input, and add current-head real behavior proof.

## Reproduction Assessment

## Solution Assessment

No, not yet. The PR is aimed at the right runner boundary, but it still suppresses reset for exhausted history overflows, ships inaccurate error guidance, fails test typechecking, and lacks current-head real behavior proof.
