## Summary

Keep open: current main still has the compaction-loop source path, but the latest PR head leaves the new irreducible overflow result classified as resettable `context_overflow`, so the auto-reply layer can still reset the session and hide the intended actionable error.

## What This Changes

The branch adds an `irreducible_overflow` preemptive-compaction route, fail-fast runner handling, exhausted overflow metadata, an auto-reply reset skip for exhausted errors, tests, docs, and a changelog entry.

## Best Possible Solution

Carry a distinct non-resettable irreducible overflow classification through embedded run metadata and auto-reply handling, align the user-facing copy with both system prompt and current input causes, then resolve branch conflicts and validate the targeted runner and auto-reply tests.

## Reproduction Assessment

Yes. Source inspection shows current main routes a zero-history system/user baseline overflow to `compact_only`, then the runner can spend three overflow compaction attempts and the auto-reply layer can reset context-overflow results.

## Solution Assessment

No, not yet. The baseline route is the right direction, but the PR still emits resettable `context_overflow` for irreducible overflow; the narrow fix is a distinct irreducible or otherwise non-resettable kind plus matching copy/tests.
