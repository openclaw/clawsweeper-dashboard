## Summary

Keep this PR open: the central irreducible-overflow direction is useful and source-reproducible, but current head still fails test type checking, lacks current-head real behavior proof, and has two behavior/copy blockers around reset handling and user guidance.

## What This Changes

The PR adds irreducible context-overflow routing and new embedded Pi runner error kinds, changes auto-reply reset handling, and updates compaction docs, changelog, and regression tests.

## Best Possible Solution

Keep the baseline irreducible-overflow fail-fast path, but only make truly irreducible overflows non-resettable, keep history-exhaustion recovery available, align user-facing guidance with system-plus-current-message overflow, fix the test type error, and add current-head real behavior proof.

## Solution Assessment

No, not yet. The route split is the right shape, but the current patch still over-blocks reset for exhausted history overflows, misstates the current-input case, fails test type checking, and needs current-head real behavior proof.
