## Summary

Keep this PR open: the current-main bug is source-reproducible and the PR is on the right path, but the latest diff still leaves the new irreducible overflow as a resettable `context_overflow` and gives incomplete user guidance.

## What This Changes

The PR adds an `irreducible_overflow` preemptive-compaction route, fail-fast embedded-runner handling, exhausted-overflow metadata, auto-reply reset suppression for exhausted overflows, tests, docs, and a changelog entry.

## Best Possible Solution

Carry an explicit non-resettable irreducible-overflow classification through embedded run metadata and auto-reply handling, update user/docs/test copy to mention oversized current input, then rebase and run the targeted agent and auto-reply checks.

## Reproduction Assessment

Yes. Source inspection shows current main routes a zero-history system+prompt baseline overflow to `compact_only`, then the runner can spend bounded overflow compaction attempts and surface resettable context-overflow metadata.

## Solution Assessment

No, not yet. The baseline route is the right fix direction, but this PR still emits resettable `context_overflow` for irreducible overflow and the user-facing guidance omits the oversized current-input cause.
