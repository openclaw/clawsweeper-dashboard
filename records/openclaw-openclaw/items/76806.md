## Summary

Keep open. The PR targets a real source-reproducible availability bug, but the current head is not mergeable because the type-check lane fails, current-head real behavior proof is still test-only, and the diff still makes exhausted history overflows non-resettable while showing misleading irreducible-overflow guidance.

## What This Changes

This PR adds an irreducible-overflow compaction route and error kinds in the embedded Pi runner, bypasses some auto-reply resets, and updates compaction docs, changelog, and overflow tests.

## Best Possible Solution

Keep the irreducible baseline fail-fast path, but make only proven irreducible overflows non-resettable, align runtime/docs/tests with the system-plus-current-message condition, fix the type-check failure, and require current-head real behavior proof before merge.

## Reproduction Assessment

Yes, by source inspection. Current main routes baseline-sized overflows into `compact_only` and resettable `context_overflow` handling; I did not run a live gateway reproduction in this read-only pass.

## Solution Assessment

No, not yet. The baseline route is the right direction, but the PR still treats generic exhausted overflows as non-resettable and its user-facing guidance does not match the actual baseline condition.
