## Summary

Keep open: current main still has a plausible Control UI hydration gap where an oversized placeholder can be treated as a renderable final message, and no merged or shipped fix covers the specific placeholder-until-refresh symptom.

## What This Changes

The issue reports that Control UI audio attachments initially show an oversized-message placeholder and only render the inline player after manual refresh.

## Best Possible Solution

Keep the no-flicker final-message optimization, but add a narrow placeholder/media exception that reloads or hydrates oversized final/session rows so persisted audio metadata appears without manual refresh.

## Reproduction Assessment

No high-confidence live repro was run. Source inspection shows a plausible current-main path: gateway oversized placeholders are non-empty assistant text and the final-event reload predicate skips history reload for non-empty assistant text.

## Solution Assessment

Yes as a direction, but not proven by live behavior: the narrow repair should preserve the renderable-final no-reload optimization while recognizing oversized-placeholder rows as needing reload or hydration.
