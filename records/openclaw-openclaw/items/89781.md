## Summary

Keep open: current main and v2026.6.8 still expose successful WebChat tool-result rows by default, and the linked PRs address adjacent replay/error cases rather than the generic `Tool output` visibility contract.

## What This Changes

The issue asks WebChat to hide or collapse a visible `Tool output` block after a normal assistant reply completes.

## Best Possible Solution

Define the normal WebChat transcript visibility contract, then hide or collapse raw successful tool-result rows in normal chat while preserving explicit debug/tool visibility access.

## Reproduction Assessment

Yes, at source level: default `chatShowToolCalls` is true, `buildChatItems` appends tool messages under that default, and the renderer labels successful results as `Tool output`. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

Unclear. Hiding or collapsing these rows is plausible, but current settings and tests intentionally expose tool output cards, so maintainers should choose the normal direct-chat/default contract before implementation.
