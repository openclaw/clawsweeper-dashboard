## Summary

Keep open: current main and v2026.6.6 still default WebChat to showing tool rows and still render successful tool results as visible `Tool output`; the related open PRs address failed-tool styling or replay status, not this generic post-final visibility contract.

## What This Changes

The issue requests hiding or collapsing WebChat `Tool output` rows after a normal final assistant reply.

## Best Possible Solution

Decide the WebChat visibility contract, then align defaults/rendering/tests/docs so normal direct chat hides or collapses raw tool-result rows after a final answer while explicit debug/tool visibility remains available.

## Reproduction Assessment

Yes, at source level: with the default `chatShowToolCalls: true`, WebChat appends tool messages and renders successful tool results with visible `Tool output` labels. I did not run a live browser session.

## Solution Assessment

Unclear. Hiding or collapsing raw tool rows is plausible, but current docs/settings intentionally expose tool output cards, so maintainers should choose the default/direct-chat contract before implementation.
