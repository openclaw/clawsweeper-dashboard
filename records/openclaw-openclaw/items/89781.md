## Summary

Keep open: current main and the latest release still show successful WebChat tool-result rows by default, including visible `Tool output` labels after assistant text; the remaining work is a maintainer UX/default decision, not something already solved by the related PRs.

## What This Changes

Hide or collapse WebChat successful tool-output rows after a normal final assistant reply in direct chat.

## Best Possible Solution

Define the normal WebChat transcript visibility contract, then keep explicit debug/tool visibility available while hiding or collapsing raw successful tool-result rows after a final answer when that contract says they are not user-facing.

## Reproduction Assessment

Yes, at source level: default `chatShowToolCalls` is true, `buildChatItems` appends tool messages under that default, and the renderers label successful results as `Tool output`. I did not run a live browser reproduction.

## Solution Assessment

Unclear. Hiding or collapsing these rows is plausible, but current docs, tests, and settings intentionally expose tool output cards, so maintainers should choose the normal direct-chat/default contract before implementation.
