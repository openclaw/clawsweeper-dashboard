## Summary

Keep open: current main still defaults WebChat to showing tool calls/tool results and renders successful tool results with a visible `Tool output` row. The related open replay PR at https://github.com/openclaw/openclaw/pull/87471 addresses a different missing-visible-status problem and is not a safe supersession for this UI visibility request.

## What This Changes

Request hiding or collapsing persistent WebChat `Tool output` rows after a final assistant reply in normal direct-chat transcripts.

## Best Possible Solution

Decide the WebChat visibility contract, then align defaults/rendering/tests/docs so normal direct chat hides or collapses raw tool-result rows after a final answer while explicit tool-debug visibility remains available.

## Reproduction Assessment

Yes, by source inspection: with the default `chatShowToolCalls: true`, WebChat includes tool messages and renders successful tool results with `Tool output` labels. I did not run a live browser session.

## Solution Assessment

Unclear. Hiding or collapsing raw tool results by default is plausible, but current docs/settings intentionally expose tool output cards, so maintainers should choose the default/direct-chat contract before implementation.
