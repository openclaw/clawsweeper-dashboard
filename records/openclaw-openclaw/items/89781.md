## Summary

Keep open: current main and the latest release still show successful WebChat tool-result rows under the normal default, and the linked PRs cover adjacent failed-error or replay behavior rather than this generic visibility contract.

## What This Changes

Request to hide or collapse persistent WebChat `Tool output` cards after a normal assistant reply in direct chat.

## Best Possible Solution

Define the normal WebChat transcript visibility contract, then hide or collapse raw successful tool-result rows in normal chat while preserving an explicit debug/tool-visibility path.

## Reproduction Assessment

Yes, at source level: the default setting keeps tool calls visible, `buildChatItems` renders toolResult rows under that default, and grouped rendering labels successful tool rows as `Tool output`. I did not run a live browser reproduction.

## Solution Assessment

Unclear. Hiding or collapsing these rows is plausible, but current docs, settings, and tests intentionally expose tool output cards, so maintainers should choose the normal direct-chat/default contract before implementation.
