## Summary

Keep open: current main and the latest release still show successful WebChat tool-result rows by default, and the related open PRs cover adjacent failed-tool or replay cases rather than the generic completed-turn `Tool output` visibility contract.

## What This Changes

The issue requests hiding or collapsing the visible WebChat `Tool output` row after a normal assistant reply completes.

## Best Possible Solution

Define the normal WebChat transcript visibility contract, then hide or collapse raw successful tool-result rows in normal chat while preserving an explicit debug/tool visibility path.

## Reproduction Assessment

Yes, at source level: fresh settings keep `chatShowToolCalls` enabled, `buildChatItems` appends tool messages under that default, and `grouped-render` labels successful generic tool results as `Tool output`. I did not run a live browser session in this read-only review.

## Solution Assessment

Unclear. Hiding or collapsing the row is plausible, but current docs, settings, and tests intentionally expose tool output cards, so maintainers should choose the normal WebChat/default debug visibility contract before implementation.
