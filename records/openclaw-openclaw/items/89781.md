## Summary

Keep open: current main and the latest release still expose successful WebChat tool-result rows by default, and the remaining question is the normal WebChat UX/debug visibility contract rather than an already-implemented fix.

## What This Changes

Request to hide, collapse, or move successful WebChat `Tool output` rows out of the normal end-user transcript after a final assistant reply.

## Best Possible Solution

Define the normal WebChat transcript visibility contract, then keep explicit debug/tool visibility available while hiding or collapsing raw successful tool-result rows after final answers only if maintainers choose that default.

## Reproduction Assessment

Yes, at source level: default `chatShowToolCalls` is true, `buildChatItems` appends tool messages under that default, and the renderers label successful results as `Tool output`. I did not run a live browser reproduction.

## Solution Assessment

Unclear. Hiding or collapsing these rows is plausible, but current docs, tests, and settings intentionally expose tool output cards, so maintainers should choose the normal direct-chat/default contract before implementation.
