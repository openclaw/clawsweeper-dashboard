Full review comments:

- **[P1] Preserve bash first-install profile fallback:** `src/cli/completion-runtime.ts:239`
  - body: Current main and v2026.6.8 fall back to creating `.bash_profile` when `.bashrc` is absent during bash install. Returning `candidates[0]` here changes that first-install path to `.bashrc`, which can make completion inactive for login-shell bash users after a reported-successful install.
  - confidence: 0.9
- **[P2] Use the installed bash profile in setup hints:** `src/wizard/setup.completion.ts:31`
  - body: When a bash home has only `.bash_profile`, `installCompletion()` selects that existing file, but setup recomputes the canonical path and tells the user to source `.bashrc`. The immediate reload command can point at a file OpenClaw did not update.
  - confidence: 0.88
- **[P2] Use the installed bash profile in doctor hints:** `src/commands/doctor-completion.ts:30`
  - body: Doctor runs `installCompletion()` and then formats the reload note from `resolveCompletionProfilePath()`, which cannot see that bash selected an existing `.bash_profile`. The repair can therefore succeed while the follow-up command targets `.bashrc`.
  - confidence: 0.82
