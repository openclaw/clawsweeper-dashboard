## Summary

Keep open. The PR is the right direction, but it is not merge-ready because the shared resolver can still write zsh/fish completion to a HOME-backed profile when an env-scoped profile should be created, and it regresses bash no-profile installs from the prior .bash_profile fallback to .bashrc.

## What This Changes

The PR centralizes shell completion profile path resolution in `src/cli/completion-runtime.ts` and adds regression tests for ZDOTDIR, XDG_CONFIG_HOME, and bash profile lookup.

## Best Possible Solution

Land the shared runtime resolver after fixing fallback semantics and adding regression tests for env-target-missing/HOME-existing zsh and fish cases plus bash no-profile install behavior.

## Reproduction Assessment

Yes. Current main is source-reproducible: ZDOTDIR/XDG_CONFIG_HOME are ignored by the check/install paths, and the PR branch source shows the remaining fallback cases without needing broad runtime setup.

## Solution Assessment

No, not yet. A shared resolver is the narrow maintainable fix, but it should keep env-scoped zsh/fish targets canonical when those env vars are set and preserve the existing bash .bash_profile creation fallback when .bashrc is absent.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep env-scoped profile targets canonical:** `src/cli/completion-runtime.ts:199-201`
  - body: When `ZDOTDIR` or `XDG_CONFIG_HOME` is set but that target profile has not been created yet, this loop skips it and can return an existing HOME fallback. `installCompletion` then writes to a file zsh or fish will not load under those env-var configurations, so the wrong-profile bug can persist.
  - confidence: 0.9
- **[P2] Preserve the bash_profile install fallback:** `src/cli/completion-runtime.ts:204`
  - body: When neither bash profile exists, `getShellProfilePath()` returns the first candidate, `~/.bashrc`. Current install behavior creates `~/.bash_profile` whenever `~/.bashrc` is absent, which matters for login bash shells; creating only `~/.bashrc` can report success while completion remains inactive.
  - confidence: 0.88
