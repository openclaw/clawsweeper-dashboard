- **[P1] Preserve the bash profile fallback:** `src/cli/completion-runtime.ts:239`
  - body: Current main falls back to `.bash_profile` when `.bashrc` is absent during bash install. This helper now returns `candidates[0]` when neither candidate exists, so fresh bash installs create `.bashrc` instead, silently changing current CLI setup behavior.
  - confidence: 0.93
- **[P2] Use the installed bash profile in doctor hints:** `src/commands/doctor-completion.ts:30`
  - body: For a bash home that has only `.bash_profile`, `installCompletion()` selects that existing file, but this reload hint uses `resolveCompletionProfilePath()` and still prints the canonical `.bashrc` path. Doctor can therefore tell users to source a file OpenClaw did not update.
  - confidence: 0.91
- **[P2] Use the installed bash profile in setup hints:** `src/wizard/setup.completion.ts:31`
  - body: Quickstart setup installs completion and then recomputes the hint with `resolveCompletionProfilePath()`. With only `.bash_profile` present, install writes `.bash_profile` while the note still points at `.bashrc`, leaving the just-installed completion inactive in the current shell.
  - confidence: 0.9
