## Summary

Keep this PR open. It is a focused fix for a valid current-main completion bug and includes useful terminal proof, but the current diff introduces a blocking bash fallback regression when neither bash profile file exists yet.

## What This Changes

The PR centralizes shell completion profile-path resolution to honor ZDOTDIR and XDG_CONFIG_HOME, align bash check/install behavior, and add profile-path regression tests.

## Best Possible Solution

Land the shared resolver, but preserve the previous bash .bash_profile creation behavior when .bashrc is absent and add a regression test for that empty-profile install path.

## Reproduction Assessment

Yes. Current main source shows zsh/fish paths hardcoded under HOME and a bash check/install mismatch, and the linked issue gives concrete ZDOTDIR, XDG_CONFIG_HOME, and bash fallback scenarios.

## Solution Assessment

No, not yet. The shared resolver is the right maintainable direction, but returning the first bash candidate when no profile exists changes existing install behavior and can leave macOS login-bash users with inactive completion.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve bash_profile fallback when no profile exists:** `src/cli/completion-runtime.ts:204`
  - body: When no bash profile file exists yet, this returns the first bash candidate, ~/.bashrc. Current install behavior falls back to ~/.bash_profile when ~/.bashrc is absent and then creates it, which matters for login bash shells such as macOS; creating only ~/.bashrc can report a successful install while the shell never sources it.
