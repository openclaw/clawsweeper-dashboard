## Summary

Keep open: the ZDOTDIR/XDG direction is useful, but this PR is not merge-ready because it changes the shipped bash no-profile install fallback and still lets setup/doctor reload hints point at a different bash profile than install selected. The head is also currently conflicting, and the real behavior proof is from an older head.

## What This Changes

The PR adds shell profile candidate resolution for zsh, fish, bash, and PowerShell, routes completion install/status checks and reload hints through it, and adds regression tests.

## Best Possible Solution

Preserve the current bash `.bash_profile` fallback, make install/status/doctor/setup share the actual selected profile path, add bash hint coverage, rebase cleanly, and update current-head proof.

## Reproduction Assessment

Yes, source reproducible. On PR head, bash candidates fall back to `.bashrc` when no profile exists, while current main and v2026.6.8 fall back to `.bash_profile` after `.bashrc` access fails.

## Solution Assessment

No. A shared resolver is the right shape, but this implementation must preserve shipped bash fallback behavior and reuse the selected profile for user-facing reload hints.

- **[P1] Preserve bash's shipped fallback profile:** `src/cli/completion-runtime.ts:239`
  - body: When no bash profile file exists, this helper falls back to `candidates[0]`, which is `.bashrc`.   - confidence: 0.95
- **[P2] Use the selected bash profile in setup hints:** `src/wizard/setup.completion.ts:31`
  - body: `resolveProfileHint()` asks the canonical resolver, not the existence-aware path used by install. In a `.bash_profile`-only bash setup, install selects `.bash_profile` but the wizard note can say `source ~/.bashrc`; current main avoided this by checking `.bashrc` before choosing the hint. Reuse the same selected profile path that install/status use.
  - confidence: 0.93
- **[P2] Use the selected bash profile in doctor hints:** `src/commands/doctor-completion.ts:30`
  - body: Doctor reload notes now route through `resolveCompletionProfilePath()`, which returns the first bash candidate rather than the profile that exists or was installed.
