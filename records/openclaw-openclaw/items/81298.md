## Best Possible Solution

Keep the shared candidate resolver, but make install/status/doctor/setup share the same selected profile path, preserve the shipped bash fresh-install fallback, and refresh current-head proof before merge.

## Reproduction Assessment

Yes. Current main is source-reproducible: zsh/fish profile resolution ignores `ZDOTDIR`/`XDG_CONFIG_HOME`, and bash install/status/hint paths can diverge around `.bashrc` and `.bash_profile`.

## Solution Assessment

No. The shared resolver is the right direction, but this is not the best complete fix until selected-profile behavior is reused by reload hints and the shipped bash first-install fallback is preserved or explicitly changed.
