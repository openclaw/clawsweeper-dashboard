## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep env-owned profiles as the install target:** `src/cli/completion-runtime.ts:199-204`
  - body: When `ZDOTDIR` or `XDG_CONFIG_HOME` is set but the env-owned profile does not exist yet, this first-existing lookup can fall through to an existing HOME/default profile. `installCompletion()` will then write to a file zsh/fish will not read, so the original bug remains for users who need OpenClaw to create the moved profile.
  - confidence: 0.9
- **[P2] Preserve bash_profile creation for fresh bash installs:** `src/cli/completion-runtime.ts:204`
  - body: If neither `.bashrc` nor `.bash_profile` exists, this fallback returns the first bash candidate, `.bashrc`; current main's install path creates `.bash_profile` when `.bashrc` is absent. That changes the fresh-install target for login-shell bash users and can make the install report success while the shell never reads the new file.
  - confidence: 0.82
- **[P2] Preserve current PowerShell completion support:** `src/cli/completion-runtime.ts:258-260`
  - body: This branch still rejects `powershell` installs, but current main now resolves concrete PowerShell profile paths and installs into them. Rebase the resolver without reintroducing the unsupported path, or `openclaw completion --install --shell powershell` and setup will regress.
  - confidence: 0.86
