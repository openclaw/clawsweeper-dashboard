## Best Possible Solution

Keep the shared candidate resolver, make doctor and setup reload hints use the same selected/installed profile path as installCompletion, add bash .bash_profile-only hint coverage, and refresh current-head proof before merge.

## Solution Assessment

No. The shared resolver is the right direction, but the PR is not the best complete fix until reload hints use the selected first-existing profile path and current-head proof is refreshed.
