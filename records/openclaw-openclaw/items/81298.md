## Summary

Keep open. The PR addresses a real current-main CLI completion bug, but the live head still has fallback cases that can select a profile the shell will not read, so it needs a narrow repair before merge.

## What This Changes

The PR adds a shared shell completion profile resolver in `src/cli/completion-runtime.ts` for ZDOTDIR/XDG_CONFIG_HOME/bash fallback handling and adds colocated regression tests.

## Best Possible Solution

Land the shared runtime resolver after it always targets the shell's active env-selected zsh/fish profile and preserves the existing bash `.bash_profile` fallback contract, with focused regression tests.

## Reproduction Assessment

Yes. Current main source still shows the HOME-backed zsh/fish resolver and bash check/install mismatch, and the PR head's remaining fallback bugs are source-reproducible from the resolver loop and missing tests.

## Solution Assessment

No, not yet. Moving the resolver into `completion-runtime.ts` is the right direction, but the path-selection semantics need to keep env-selected profiles canonical and preserve the bash no-profile fallback before this is the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep ZDOTDIR/XDG targets canonical:** `src/cli/completion-runtime.ts:199-201`
  - body: When `ZDOTDIR` or `XDG_CONFIG_HOME` is set, this loop can fall through to an existing HOME profile if the env-specific file is absent. That recreates the wrong-profile behavior for users whose shell reads the env-selected path, so install/check should target the active env path instead of an unrelated HOME fallback.
  - confidence: 0.88
- **[P2] Preserve bash_profile fallback when no profile exists:** `src/cli/completion-runtime.ts:204`
  - body: When neither bash profile exists, this returns `~/.bashrc`. Current install behavior creates `~/.bash_profile` when `.bashrc` is absent, which matters for login bash shells such as macOS; creating only `.bashrc` can report success while completion is not loaded.
  - confidence: 0.87
