## Summary

Keep open. The PR targets a valid current-main completion bug, but the branch still has path-selection gaps: env-moved zsh/fish profiles can fall back to default HOME paths when the env-owned file is absent, and fresh bash installs lose the existing `.bash_profile` creation fallback.

## What This Changes

The PR centralizes shell completion profile path resolution in `src/cli/completion-runtime.ts` to honor `ZDOTDIR` and `XDG_CONFIG_HOME`, with new profile-path regression tests.

## Best Possible Solution

Land the shared resolver after separating actual install-target selection from fallback detection, preserving bash `.bash_profile` creation behavior, and adding focused regression tests for those empty-profile cases.

## Reproduction Assessment

Yes. Current-main source shows zsh/fish paths ignore `ZDOTDIR` and `XDG_CONFIG_HOME`, and bash status/install paths disagree; the linked issue gives concrete zsh, fish, and bash setups.

## Solution Assessment

No, not yet. A shared resolver is the right direction, but the proposed resolver conflates installed-profile detection with install-target choice and changes the existing bash empty-profile fallback.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep env-owned profiles as the install target:** `src/cli/completion-runtime.ts:199-204`
  - body: When `ZDOTDIR` or `XDG_CONFIG_HOME` is set but that profile file does not exist yet, this loop falls through to an existing HOME/default candidate. `installCompletion()` can then write to a file zsh/fish will not read, and status can treat an old orphan profile as installed, leaving the reported bug in place for users who need OpenClaw to create the moved profile.
  - confidence: 0.86
- **[P2] Preserve bash_profile fallback when no profile exists:** `src/cli/completion-runtime.ts:204`
  - body: When no bash profile file exists yet, `getShellProfilePath()` falls through to the first bash candidate, `~/.bashrc`. Current install behavior chooses `~/.bash_profile` when `~/.bashrc` is absent, which matters for login bash shells such as macOS; creating only `.bashrc` can report a successful install while the shell never sources it.
  - confidence: 0.88
