## Summary

Keep open. The PR addresses a valid completion-profile bug, but current head still regresses bash installs when neither bash profile exists by creating `.bashrc` instead of preserving the previous `.bash_profile` fallback.

## What This Changes

The PR adds a shared completion profile resolver honoring `ZDOTDIR` and `XDG_CONFIG_HOME`, routes completion status/install checks through it, and adds profile-path regression tests.

## Best Possible Solution

Keep the shared resolver direction, but preserve the bash empty-profile install target as `.bash_profile` and add a regression test that `installCompletion()` creates that file when no bash profile exists.

## Reproduction Assessment

Yes. Current main and the linked issue give a high-confidence source reproduction for the original bug, and the PR regression is visible from the new resolver returning the first bash candidate when no profile exists.

## Solution Assessment

No, not yet. A shared resolver is the right maintainable direction, but the resolver or install path needs a bash-specific no-profile fallback before this is safe to merge.
