## Summary

Keep open: the current-main bug is real by source inspection, but the proposed fix misses the `OPENCLAW_HOME` path that `saveExecApprovals` prefers over `HOME`, and the external real-behavior proof gate is still missing.

## What This Changes

This PR sandboxes `src/security/audit-exec-surface.test.ts` by redirecting `HOME` and `USERPROFILE` to per-case temp directories and adds a changelog note.

## Best Possible Solution

Keep the tempdir isolation, but redirect and restore `OPENCLAW_HOME` alongside `HOME`/`USERPROFILE`, then provide redacted before/after proof that the live approvals file hash is unchanged.

## Reproduction Assessment

Yes, source-level: current main calls `saveExecApprovals` from this test without sandboxing the effective OpenClaw home, and `saveExecApprovals` writes the expanded approvals path. I did not execute the test in this read-only pass.

## Solution Assessment

No, not yet: redirecting `HOME` and `USERPROFILE` is the right shape but is not the complete OpenClaw home contract because `OPENCLAW_HOME` wins first. The narrow maintainable fix is to sandbox and restore all three environment inputs used by the approvals path.
