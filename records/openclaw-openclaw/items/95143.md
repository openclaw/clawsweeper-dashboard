## Summary

Keep open, but the patch is not merge-ready: it repeats the unconditional app-chat clearing approach, which can erase newer composer text or attachments during delayed sends, while the linked stale-text bug still belongs at the chat-view input replay boundary.

The branch removes the submitted-draft and attachment equality guard in `clearSubmittedComposerState` and always clears `host.chatMessage` and `host.chatAttachments` after send.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

No. Unconditionally clearing app-level composer state is too broad; the narrow maintainable fix is a stale-replay discriminator in `ui/src/ui/views/chat.ts` that preserves delayed-send draft and attachment edits.
