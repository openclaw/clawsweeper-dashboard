## Summary

Keep open: current main still runs the ClawDock dashboard lookup through `docker compose run --rm openclaw-cli` without `--no-deps`, while the CLI service depends on the gateway service. The exact custom `GATEWAY_IMAGE` setup is not part of the current compose contract, but the helper can still start or recreate the gateway when it should only fetch a dashboard URL.

## What This Changes

The issue reports that `clawdock-dashboard --no-open` can switch a running Docker Compose gateway from a custom tools image to the default OpenClaw image.

## Best Possible Solution

Make `clawdock-dashboard` fetch the dashboard URL without starting Compose dependencies, update any dashboard command examples that share the same hazard, and add focused regression coverage around the helper command.

## Reproduction Assessment

No, not for the exact reported image switch as a live current-main reproduction. Source inspection gives a clear current-main dependency-start path: the helper runs `openclaw-cli` without `--no-deps`, and that service depends on `openclaw-gateway`.

## Solution Assessment

Yes. The narrow maintainable fix is to run the dashboard helper's one-off CLI command with `--no-deps` or an equivalent existing helper path, without adding a new image variable or changing gateway image policy.
