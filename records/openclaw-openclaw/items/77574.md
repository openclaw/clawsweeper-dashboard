## Repair Work Prompt

Fix the ClawDock dashboard dependency-start bug reported in https://github.com/openclaw/openclaw/issues/77574. On current main, `scripts/clawdock/clawdock-helpers.sh` runs `_clawdock_compose run --rm openclaw-cli dashboard --no-open`; because `openclaw-cli` has `network_mode: "service:openclaw-gateway"` and `depends_on: openclaw-gateway`, fetching a URL can start/recreate the gateway. Add `--no-deps` to the `clawdock-dashboard` URL lookup, update Docker/GCP/install docs that tell users to run the same dashboard URL command when appropriate, and add a focused regression in `test/scripts/clawdock-helpers.test.ts` that stubs `docker`/browser openers, sources the real helper, and asserts the command includes `run --rm --no-deps openclaw-cli dashboard --no-open` while preserving compose override ordering. Do not edit `CHANGELOG.md`. Treat https://github.com/openclaw/openclaw/pull/80494 as useful source work/proof, and coordinate with https://github.com/openclaw/openclaw/pull/75754 so host-port remapping remains separate unless a maintainer explicitly chooses to combine the same helper edit.
