## Summary

Keep open: current main still lets `clawdock-dashboard` run the Docker Compose CLI service without `--no-deps`, while that service depends on `openclaw-gateway`, so the helper can start or recreate the gateway when it should only fetch the Control UI URL.

## What This Changes

Prevent `clawdock-dashboard` from changing the running Docker Compose gateway image while fetching the Control UI URL.

## Best Possible Solution

Make `clawdock-dashboard` fetch the dashboard URL through a dependency-safe Compose invocation and add focused coverage proving the gateway is not started or recreated.

## Reproduction Assessment

No, not as a high-confidence live reproduction of the image swap; this read-only review did not run Docker. Source inspection on current main does show the dashboard helper can start linked Compose dependencies because it omits `--no-deps`.

## Solution Assessment

Yes, the narrowest maintainable solution is to add `--no-deps` or an equivalent dependency-safe path to the dashboard helper. A broader image-variable migration would be separate product behavior.
