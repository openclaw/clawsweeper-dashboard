## Summary

Keep open: current main still runs `clawdock-dashboard` through the dependency-aware `openclaw-cli` service without `--no-deps`, and https://github.com/openclaw/openclaw/pull/80494 is the open implementation candidate that closes this issue.

## What This Changes

Prevent `clawdock-dashboard` from starting, recreating, or switching the running Docker Compose gateway image while fetching the Control UI URL.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80494 or an equivalent focused main-branch change, then let GitHub close this issue after merge.

## Reproduction Assessment

No, not for the exact before/after image switch in this read-only pass. Source inspection gives a clear current-main root path: the helper calls the dependency-aware Compose CLI service, and Docker Compose documents `--no-deps` as the way to avoid starting linked services.

## Solution Assessment

Yes. The linked `--no-deps` fix is the narrowest maintainable solution because it preserves the existing dashboard URL lookup while preventing Compose dependency startup for that lookup.
