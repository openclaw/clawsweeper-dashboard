## Summary

Keep open: current main still has the dependency-aware ClawDock dashboard command, and an open linked PR already proposes the focused `--no-deps` fix; this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that running `clawdock-dashboard` can cause Docker Compose to start or recreate `openclaw-gateway` with a different image.

## Best Possible Solution

Land the focused `--no-deps` helper/docs/test fix from https://github.com/openclaw/openclaw/pull/80494, or an equivalent main-branch change, then let GitHub close the issue after merge.

## Reproduction Assessment

No, not for the exact before/after image switch in this read-only pass. Source inspection on current main shows the root dependency-start path: the helper runs the dependency-aware CLI service without `--no-deps`, and that service depends on `openclaw-gateway`.

## Solution Assessment

Yes, the linked PR's `--no-deps` change is the narrowest maintainable fix because it preserves the existing CLI dashboard path while preventing Compose from starting linked services for a URL lookup. A broader image variable migration is not needed for the observed helper behavior.
