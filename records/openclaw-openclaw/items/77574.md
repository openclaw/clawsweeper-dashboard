Keep open: current main still lets the ClawDock dashboard helper start Compose dependencies while only fetching a dashboard URL, and dry-run confirms gateway start planning remains.

## What This Changes

The issue asks that `clawdock-dashboard --no-open` not replace or recreate an already-running Docker Compose gateway image while fetching the dashboard URL.

## Best Possible Solution

Update the ClawDock dashboard and matching Docker examples so dashboard/device CLI lookups use the existing gateway without starting Compose dependencies, then add focused regression coverage for the exact command path.

## Reproduction Assessment

Yes for the unsafe Compose behavior: current-main source and Docker Compose dry-run show the dashboard helper command plans to start `openclaw-gateway`. I did not live-reproduce an already-running container changing images.

## Solution Assessment

Yes: the narrowest maintainable solution is to add `--no-deps` or an equivalent gateway-safe execution path for the dashboard URL lookup and related docs/examples, without broad image-variable migration.
