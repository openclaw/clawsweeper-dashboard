## Summary

Keep open: current main still has a plausible dependency-start regression in the ClawDock dashboard helper, and Docker Compose dry-run confirms the current helper command plans to start `openclaw-gateway` while only fetching a dashboard URL.

## What This Changes

The issue asks OpenClaw to prevent `clawdock-dashboard` from changing or recreating the running Docker gateway image.

## Best Possible Solution

Update the ClawDock dashboard URL lookup and matching Docker examples so fetching the URL cannot start or recreate `openclaw-gateway`, then add focused regression coverage for that command path.

## Reproduction Assessment

No, not as a high-confidence live reproduction of the exact image switch. Source inspection and Docker Compose dry-run on current main do show the helper command can start the gateway dependency unless `--no-deps` is used.

## Solution Assessment

Yes, the narrowest maintainable fix is to stop the dashboard helper from starting Compose dependencies, likely by using `--no-deps` or an equivalent gateway-safe execution path. A broader image-variable migration would be riskier and is not needed for the observed helper behavior.
