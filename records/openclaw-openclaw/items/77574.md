## Summary

Keep open: current main and the latest release still run the ClawDock dashboard lookup through the dependency-aware Compose CLI service, and the open closing PR is the right path to resolve this issue after it lands.

## What This Changes

The issue reports that `clawdock-dashboard` can start or recreate the Docker Compose gateway and switch the running gateway image while only fetching the dashboard URL.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80494 or an equivalent focused main-branch change, then let the issue close after the fix merges.

## Reproduction Assessment

No, not for the exact before/after image switch in this read-only review. Source inspection gives a clear current-main reproduction path for the root cause: the helper calls the dependency-aware Compose CLI service, and Docker Compose exposes `--no-deps` specifically to avoid starting linked services.

## Solution Assessment

Yes. Adding `--no-deps` to the dashboard URL lookup is the narrowest maintainable fix because it preserves the existing CLI path while preventing Compose from starting or recreating linked services for a URL-only command.
