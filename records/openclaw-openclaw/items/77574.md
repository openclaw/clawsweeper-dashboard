The issue reports that `clawdock-dashboard --no-open` may switch or recreate the running Docker gateway image.

## Best Possible Solution

Update the ClawDock dashboard URL lookup and matching Docker examples so fetching the URL cannot start or recreate `openclaw-gateway`, then add focused regression coverage for that helper path.

## Repair Work Prompt

Fix the ClawDock dashboard helper regression tracked by https://github.com/openclaw/openclaw/issues/77574. On current main, `clawdock-dashboard` runs `docker compose run --rm openclaw-cli dashboard --no-open`, and `openclaw-cli` depends on and shares the network namespace of `openclaw-gateway`, so Compose can start/recreate the gateway while only fetching the dashboard URL. Keep the fix scoped to preventing the dashboard URL lookup from starting linked services, likely by adding `--no-deps` or routing through an existing runtime CLI helper with `no-deps`; preserve current `OPENCLAW_IMAGE` semantics and do not add a `GATEWAY_IMAGE` migration. Add or update focused regression coverage for the helper or Docker flow, and update Docker/ClawDock docs only if the user-facing command examples change. Add a changelog entry because this is a user-facing Docker helper fix.
