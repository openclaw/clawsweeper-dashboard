## Summary

Keep open: current main still lets `clawdock-dashboard` invoke Compose without `--no-deps`, so Docker Compose is allowed to pull/create/start `openclaw-gateway` while the helper is only fetching the dashboard URL; dry-run confirmed the dependency-start plan.

## What This Changes

The issue asks that `clawdock-dashboard` avoid replacing or recreating the running Docker Compose gateway image when opening the dashboard URL.

## Best Possible Solution

Add `--no-deps` or an equivalent safe path to the ClawDock dashboard URL lookup and align the matching docs/tests so fetching the URL cannot start or recreate the gateway.

## Reproduction Assessment

No, not for the exact already-running container image switch, because I did not perform a live Docker run against an existing stack. Source inspection and `docker compose --dry-run` on current main do reproduce the gateway dependency-start plan behind the reported behavior.

## Solution Assessment

Yes, the narrowest maintainable solution is to prevent dependency startup in the dashboard helper, likely with `--no-deps`. Changing image-variable semantics would be broader and is not needed for the observed helper path.
