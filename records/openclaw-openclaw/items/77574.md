## Summary

Keep open: current main still lets the ClawDock dashboard helper run `openclaw-cli` with Compose dependencies enabled, and a current dry-run shows Compose planning gateway pull/create/start work while only fetching the dashboard URL.

## What This Changes

Prevent `clawdock-dashboard` from starting or recreating the Docker Compose gateway container with a different image while fetching the Control UI URL.

## Best Possible Solution

Update the ClawDock dashboard URL lookup and matching Docker examples so fetching the URL cannot start or recreate `openclaw-gateway`, then add focused regression coverage for that command path.

## Reproduction Assessment

No, not for the exact live image switch. Source inspection and Docker Compose dry-run on current main do reproduce the dependency-start path that can pull/create/start `openclaw-gateway` during `clawdock-dashboard`.

## Solution Assessment

Yes, the narrowest maintainable solution is to prevent the dashboard helper from starting Compose dependencies, likely by adding `--no-deps` or using an equivalent gateway-safe execution path. A broader `GATEWAY_IMAGE` migration is not needed for the observed helper behavior.
