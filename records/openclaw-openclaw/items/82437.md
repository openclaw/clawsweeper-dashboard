The issue reports that `v2026.5.12` passes readiness but `/v1/chat/completions` returns a generic HTTP 500 when the selected `codex` agent harness is not registered.

## Best Possible Solution

## Reproduction Assessment

Yes, source-reproducible: an `openai/*` default model selects runtime `codex`, a missing registered `codex` harness throws the reported error, readiness does not check that harness, and the HTTP endpoint can map it to the observed generic 500. I did not run the live Docker image.

## Solution Assessment

Unclear: the report names several plausible repairs, and the best one depends on whether the release image should include/repair Codex by default, force PI for this deployment shape, fail readiness/startup, or return a clearer OpenAI-compatible configuration error.
