## Summary

Keep open. Current main does not implement the PR's requested explicit gateway workspace forwarding or missing-bootstrap omission, and the PR still has concrete merge blockers around gateway trust boundaries, stale bootstrap coverage, and generated protocol artifacts.

## What This Changes

The PR adds `workspaceDir` to gateway agent params, forwards explicit workspace roots ahead of spawned-session inheritance, omits missing bootstrap files from prompt context, and updates selected gateway/agent tests.

## Best Possible Solution

Revise this PR before merge: keep explicit workspace overrides restricted to trusted internal/admin callers or validated spawned-session metadata, update all affected bootstrap expectations, include generated protocol artifacts if the wire schema changes, and align the missing-bootstrap behavior with #73401.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: the PR diff forwards `request.workspaceDir` from a write-scoped method, current tests still expect missing bootstrap entries, and protocol generation rules require Swift model drift to be addressed.

## Solution Assessment

No. Omitting missing bootstrap noise may be a reasonable direction, but accepting arbitrary `workspaceDir` through the public `agent` request is not the narrowest maintainable fix; the safer path is a trusted/internal gate or continued use of stored spawned-session metadata.
