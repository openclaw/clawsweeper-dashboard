## Summary

Keep this PR open. Current main and the latest release still do not implement the explicit gateway workspace override or the missing-bootstrap omission, but the PR is not safe to merge as-is because it introduces an unguarded workspace path override, misses existing stability coverage, and leaves generated protocol clients stale.

## What This Changes

The PR adds an explicit gateway `workspaceDir` agent parameter, prefers it over inherited spawned-session workspace metadata, and changes bootstrap loading/prompt injection so missing bootstrap files are omitted.

## Best Possible Solution

Revise this PR before merge: only allow explicit gateway workspaces through a trusted internal/admin path or continue deriving them from validated spawned-session metadata, update all affected bootstrap tests/docs, regenerate protocol clients if `workspaceDir` remains in the wire schema, and coordinate the bootstrap semantics with #73401.

## Reproduction Assessment

Yes. Static current-main checks reproduce the absence of the intended behavior, and the PR diff plus existing `system-prompt-stability.test.ts` gives a high-confidence failing-test path for the missing-bootstrap change.

## Solution Assessment

No. The bootstrap omission may be a reasonable direction, but forwarding arbitrary request `workspaceDir` on a write-scoped method is not the narrowest maintainable fix; a trusted internal/admin gate or validated session metadata is safer.

  - body: This forwards `request.workspaceDir` directly into ingress options for every gateway `agent` request.   - confidence: 0.93

  - confidence: 0.88

  - body: Adding `workspaceDir` to `AgentParamsSchema` changes the gateway wire schema, but the generated Swift `AgentParams` models are not updated. `pnpm protocol:check` regenerates these files and diffs them, so this schema change should include the generated protocol artifacts.
  - confidence: 0.83
