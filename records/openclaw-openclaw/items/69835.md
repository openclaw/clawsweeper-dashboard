## Summary

This PR should stay open but is not mergeable as-is: it reintroduces a caller-supplied gateway workspace override across a write-scoped RPC, leaves affected bootstrap stability coverage stale, changes the protocol schema without generated client updates, and still lacks real behavior proof.

## What This Changes

The branch adds an optional `AgentParams.workspaceDir`, forwards it ahead of spawned-session workspace inheritance, omits missing bootstrap files from agent context, and updates selected tests/type exports.

## Best Possible Solution

Keep public `agent` requests on stored spawned-session metadata unless maintainers approve a trusted internal override, and address bootstrap noise through the narrower setup-aware loader path tracked in https://github.com/openclaw/openclaw/pull/42542.

## Reproduction Assessment

Yes. Source inspection is enough for the review: the diff accepts and forwards `request.workspaceDir` from a write-scoped method, while current tests and generated Swift models show the missing-bootstrap and protocol drift points; I did not run tests because this review is read-only.

## Solution Assessment

No. The safer solution is to preserve the existing internal spawned-session workspace boundary or add an explicitly trusted/admin-only path, then split the missing-bootstrap fix into the setup-aware behavior already covered by the related PR.
