## Summary

Keep this PR open for human review: it contains useful bootstrap-noise context, but the current diff reopens a public gateway workspace override that a prior security-hardening PR deliberately removed, and the contributor has not supplied inspectable after-fix real behavior proof.

## What This Changes

The PR adds request-level `workspaceDir` to gateway `agent` protocol/dispatch, suppresses missing bootstrap entries from agent context, and updates related tests/type exports.

## Best Possible Solution

Keep public `agent` requests on stored session metadata unless maintainers explicitly approve a trusted internal override, and solve bootstrap noise through a narrow setup-aware `BOOTSTRAP.md` loader fix like https://github.com/openclaw/openclaw/pull/42542.

## Reproduction Assessment

Yes. Source inspection is enough for this PR review: the diff adds and forwards `request.workspaceDir`, while current main excludes it from the public schema and derives workspace inheritance from stored session metadata.

## Solution Assessment

No. The PR mixes a new public protocol override with broad missing-file suppression; the safer direction is a trusted/internal workspace path only if approved, plus a narrower setup-aware BOOTSTRAP-only loader fix.
