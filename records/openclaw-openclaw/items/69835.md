## Summary

Keep open: the branch still has blocking security, protocol-generation, and test-coverage issues, and the contributor has not supplied real behavior proof. The prior Greptile and ClawSweeper comments point to the same core problems, and current main intentionally keeps spawned workspace overrides internal.

## What This Changes

The branch adds `AgentParams.workspaceDir` forwarding in gateway agent runs, stops converting missing bootstrap files into context entries, and updates selected gateway and agent tests.

## Best Possible Solution

Keep public `agent` requests on stored spawned-session metadata unless maintainers approve a trusted internal or admin-only override, and handle bootstrap noise through a narrow setup-aware loader path such as https://github.com/openclaw/openclaw/pull/42542.

## Reproduction Assessment

Yes. Source inspection is enough for this PR review: the diff accepts and forwards `request.workspaceDir` from a write-scoped method, while current tests and generated Swift models show the missing-bootstrap and protocol drift points.

## Solution Assessment

No. The safer path is to preserve the internal spawned-session workspace boundary or add an explicit trusted/admin-only path, then update all affected tests and generated protocol artifacts if the wire schema changes.
