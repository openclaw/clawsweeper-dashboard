## Summary

Keep this PR open, but it is not merge-ready: it reintroduces a caller-controlled gateway workspace root that current main and the merged security-hardening history keep internal, broadly suppresses missing bootstrap diagnostics, lacks generated client updates, and has no contributor-supplied real behavior proof.

## What This Changes

The PR adds request-level `workspaceDir` to gateway agent protocol/dispatch, omits missing bootstrap files from agent context, and updates related tests/type exports.

## Best Possible Solution

Keep public `agent` requests on stored session metadata unless maintainers approve a trusted/internal override, and handle bootstrap noise through a narrow setup-aware loader fix like the related BOOTSTRAP PR.

## Reproduction Assessment

Yes. Source inspection is enough for this PR review: the diff adds and forwards `request.workspaceDir`, while current main excludes it from the public schema and derives workspace inheritance from stored session metadata.

## Solution Assessment

No. The PR mixes a new public protocol override with broad missing-file suppression; the safer direction is a trusted/internal workspace path, if approved, plus a narrower BOOTSTRAP-only loader fix.
