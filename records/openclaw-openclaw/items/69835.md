## Summary

Keep this PR open: the branch still introduces a gateway workspace trust-boundary regression, leaves protocol clients/tests out of sync, and lacks required real behavior proof before merge.

## What This Changes

The branch adds a public `AgentParams.workspaceDir` field, forwards it into gateway agent dispatch, removes missing-bootstrap marker injection, and updates selected tests.

## Best Possible Solution

Keep public `agent` requests on stored spawned-session metadata unless maintainers approve a trusted internal override, and handle missing-bootstrap noise through a narrow setup-aware loader path such as https://github.com/openclaw/openclaw/pull/42542.

## Reproduction Assessment

Yes. Source inspection is enough for the review: the PR diff accepts and forwards `request.workspaceDir` from a write-scoped method, while current tests and generated Swift models show the missing-bootstrap and protocol drift points.

## Solution Assessment

No. The safer solution is to preserve the existing internal spawned-session workspace boundary or add an explicit trusted/admin-only path, then update all affected tests and generated protocol artifacts if the wire schema changes.
