## Summary

Keep open: the PR has blocking security, protocol, and test-coverage issues, and the external contributor has not supplied real behavior proof. Current main intentionally keeps gateway workspace overrides internal after the merged security-hardening work in https://github.com/openclaw/openclaw/pull/43801.

## What This Changes

The PR adds `workspaceDir` to gateway `AgentParams`, forwards it into agent ingress ahead of spawned-session inheritance, omits missing bootstrap files from agent context, and updates selected tests/exports.

## Best Possible Solution

Keep public `agent` requests on stored spawned-session metadata unless maintainers approve a trusted/internal override, and handle bootstrap noise through a setup-aware loader path like the narrower work in https://github.com/openclaw/openclaw/pull/42542.

## Reproduction Assessment

Yes. Source inspection is enough for this PR review: the diff adds and forwards `request.workspaceDir`, while current main keeps that value out of the public schema and derives workspace inheritance from stored session metadata; I did not run tests because this review is read-only.

## Solution Assessment

No. The safer solution is to preserve the internal spawned-session workspace boundary or add a clearly trusted/admin-only path, then split the missing-bootstrap behavior into the setup-aware loader fix already represented by the related PR.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Gate workspaceDir overrides behind trusted callers:** `src/gateway/server-methods/agent.ts:954-960`
  - body: This forwards `request.workspaceDir` directly into agent ingress for every gateway `agent` call.   - confidence: 0.88

  - body: Adding `workspaceDir` to the gateway wire schema changes `AgentParams`, but the generated Swift `AgentParams` model still lacks that field. Keep schema and generated protocol clients in sync before merging this protocol change.
  - confidence: 0.83
