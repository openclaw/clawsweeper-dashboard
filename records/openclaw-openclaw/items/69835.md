## Summary

Close as superseded: the gateway workspace part reopens a public `workspaceDir` boundary that merged security work intentionally kept internal, and the missing-bootstrap noise is already tracked by a narrower setup-aware loader PR.

## What This Changes

The branch adds a public `AgentParams.workspaceDir`, forwards it ahead of stored spawned workspace metadata, removes missing bootstrap entries, and updates selected gateway and agent tests.

## Best Possible Solution

Close this branch, keep gateway workspace roots internal, and let https://github.com/openclaw/openclaw/pull/42542 and https://github.com/openclaw/openclaw/pull/79821 carry the two remaining focused paths.

## Reproduction Assessment

Yes. Source inspection of the PR diff and current main shows the public `workspaceDir` schema/forwarding change, the write-scoped `agent` method, the stale missing-bootstrap test expectation, and the generated Swift model drift.

## Solution Assessment

No. The maintainable path is to preserve the internal spawned-session workspace boundary and use the narrower setup-aware bootstrap loader work rather than adding an ungated public workspace-root override.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Gate workspaceDir overrides behind trusted callers:** `src/gateway/server-methods/agent.ts:954-960`
  - body: This forwards `request.workspaceDir` into agent ingress for every gateway `agent` request. Because `agent` is write-scoped rather than admin-only, this would let write-scoped callers redirect the run workspace root unless the field is kept internal or separately authorized.
  - confidence: 0.93
- **[P2] Update stale missing-bootstrap stability coverage:** `src/agents/workspace.ts:555`
  - body: Removing missing bootstrap entries means callers that only created `AGENTS.md` no longer receive a `TOOLS.md` entry. The existing system prompt stability coverage still expects that entry to be marked missing, so the broader affected test must be updated or the behavior narrowed.
  - confidence: 0.88
- **[P2] Regenerate protocol clients for AgentParams:** `src/gateway/protocol/schema/agent.ts:159`
  - body: Adding `workspaceDir` to `AgentParamsSchema` changes the gateway wire schema, but the generated Swift `AgentParams` model is not updated. Include regenerated client artifacts if this schema change is intentionally kept.
  - confidence: 0.83

## Security Review

Status: needs_attention
