## Summary

## What This Changes

The PR changes Tavily API-key normalization so unresolved configured SecretRefs return `undefined` and the existing `TAVILY_API_KEY` environment fallback can run, with one Tavily tools regression test.

## Best Possible Solution

Use the guarded inspect-mode resolver shape: preserve unavailable ref metadata, allow fallback only for an allowed env-backed `TAVILY_API_KEY` ref, block other refs, add negative regression tests, and require redacted real command proof before merge.
