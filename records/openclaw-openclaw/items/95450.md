## Summary

Keep open: the Tavily bug is still present on current main and in v2026.6.9, but this PR is not merge-ready because it uses an invalid SecretRef resolver mode, broadens credential fallback behavior, and lacks real runtime proof.

## What This Changes

The PR changes Tavily API-key normalization so unresolved configured SecretRefs return `undefined` and the existing `TAVILY_API_KEY` environment fallback can run, with one Tavily tools regression test.

## Best Possible Solution

Use the guarded inspect-mode resolver shape: preserve unavailable ref metadata, allow env fallback only for an allowed env-backed `TAVILY_API_KEY` ref, block other refs, add negative regression tests, and require redacted real command proof before merge.

## Solution Assessment
