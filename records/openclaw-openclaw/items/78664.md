## Summary

Keep this PR open for maintainer review: current main does not already have the provider tool-schema cache or cache-key hook, and the PR is a coherent hook-gated implementation with focused tests and runtime proof. The remaining question is compatibility acceptance for the new public provider hook, default-on bundled-provider cache behavior, and env kill switch.

## What This Changes

The PR adds a provider-owned `resolveToolSchemaCacheKey` hook, a process-local cache for provider-normalized tool `parameters`, docs/API baseline updates, and focused runtime/provider tests.

## Best Possible Solution

Land the hook-gated, parameters-only cache if maintainers accept the new provider API and env kill switch; otherwise defer the public SDK expansion and keep provider normalization uncached until a narrower contract is chosen.
