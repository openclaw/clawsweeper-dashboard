## Summary

Keep this PR open for maintainer review: the runtime cache is conservative and proof is supplied, but it adds a public provider SDK hook plus an environment-variable kill switch, and the two public docs hook tables still have numbering gaps.

## What This Changes

The PR caches provider-normalized tool schema parameters behind a provider-owned cache key, adds the `resolveToolSchemaCacheKey` provider hook, documents `OPENCLAW_TOOL_SCHEMA_CACHE=0`, and expands related runtime/provider/SDK tests.

## Best Possible Solution

Land only after maintainers accept the public hook and env kill-switch contract, fix the docs numbering gaps, and keep the cache opt-in with cloned parameters only; otherwise keep the optimization internal to bundled provider-family helpers.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a performance/API PR, not a reported current-main failure. The remaining docs issues are source-reproducible from the PR tree, where the hook tables skip numbers.

## Solution Assessment

Unclear until maintainer approval: the cache design is plausible and conservative, but adding a public SDK hook and env kill switch is not purely mechanical. A safer alternative is to internalize the cache key for bundled provider-family helpers if maintainers do not want the public contract yet.
