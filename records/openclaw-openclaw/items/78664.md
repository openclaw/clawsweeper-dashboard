## Summary

Keep open. The hook-owned cache direction is plausible and the supplied terminal proof is enough, but PR head still resolves the cache key without the prepared provider runtime handle while normalization uses that handle, leaving a cross-hook cache mismatch path.

## What This Changes

Adds a provider-owned normalized tool-schema parameter cache for embedded agent turns, a plugin SDK `resolveToolSchemaCacheKey` hook, tests/docs/API baseline updates, a rollback env var, and a changelog entry.

## Best Possible Solution

Keep the provider-owned schema-parameter cache design, but resolve cache keys through the same prepared provider runtime handle used for normalization and add mismatch regression coverage.

## Reproduction Assessment

Yes for the review finding: PR-head source shows cache-key resolution omits the prepared runtime handle while normalization uses it. This is a PR-introduced correctness path, not a current-main end-user bug reproduction.

## Solution Assessment

No, not as currently written. The cache can remain hook-owned, but key resolution must share the prepared provider handle with the normalizer before this is the narrow maintainable fix.
