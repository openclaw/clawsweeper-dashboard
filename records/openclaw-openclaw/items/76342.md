## Summary

Keep open: the PR targets a real memory-core Dream Diary timeout path, but the current branch still accepts partial env override values, lacks stable operator docs for the new env knob, and is missing required after-fix real behavior proof.

## What This Changes

The PR raises the memory-core Dream Diary narrative timeout from 60s to 240s, adds an `OPENCLAW_DREAMING_NARRATIVE_TIMEOUT_MS` override, updates the focused timeout assertion, and adds a changelog fix entry.

## Best Possible Solution

Land a bounded 240s default with strict whole-string positive-integer env parsing, focused override coverage, stable Dreaming/memory docs, changelog coverage, and contributor-provided live Dream Diary proof.

## Reproduction Assessment

Yes at source level, but not from a live run in this review: current main hardcodes 60s and passes it to `waitForRun`, while #76333 and #77094 provide field logs showing Dream Diary narratives timing out and leaving `DREAMS.md` stale.

## Solution Assessment

No, not yet: raising the default and adding an operator override is the right narrow shape, but the parser should reject partial values, the env knob needs stable docs, and external real behavior proof is still required.
