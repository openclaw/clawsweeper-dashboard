## Review Findings

Overall confidence: 0.87

Full review comments:

- **[P2] Reject partial timeout env values:** `extensions/memory-core/src/dreaming-narrative.ts:108`
  - body: `Number.parseInt` accepts numeric prefixes, so values operators are likely to try, such as `240_000`, `240s`, `1e5`, or `240.5`, become much smaller millisecond values instead of falling back to the 240s default. That can reintroduce the timeout failure this PR is meant to fix; parse the whole trimmed string as a positive safe integer and cover partial/invalid values.
  - confidence: 0.88
- **[P3] Document the timeout override in stable docs:** `extensions/memory-core/src/dreaming-narrative.ts:99-100`
  - body: This adds `OPENCLAW_DREAMING_NARRATIVE_TIMEOUT_MS` as an operator-facing setting, but the Dreaming and memory config docs still omit the env var name, millisecond unit, 240s default, and invalid-value fallback. A changelog entry helps upgrades, but later debugging should not require source or PR history.
  - confidence: 0.9
