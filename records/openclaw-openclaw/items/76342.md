## Summary

Keep open: current main still has the 60s Dream Diary narrative timeout, and this PR is the active fix for the source-reproducible timeout reports. The runtime direction is sound, but the new env parser accepts partial numeric strings that can collapse the timeout to milliseconds, and stable docs still do not describe the operator override.

## What This Changes

The PR raises the memory-core Dream Diary narrative timeout default to 240s, adds an `OPENCLAW_DREAMING_NARRATIVE_TIMEOUT_MS` override, updates the focused timeout assertion, and adds a changelog entry.

## Best Possible Solution

Land the timeout/default fix after strict positive-integer env parsing, regression coverage for invalid overrides, and stable docs for the env var's unit, default, and fallback behavior; leave schema-level config and late-output recovery for separate maintainer decisions.

## Reproduction Assessment

Yes, at source level: current main waits only 60s and returns without appending diary text on timeout, while #76333 provides repeated field logs from a multi-workspace host. I did not live-run that host-scale setup in this read-only review.

## Solution Assessment

Not yet: raising the bounded default and adding an env override is the right narrow shape, but the override should parse strict positive millisecond integers and be documented in stable operator docs before merge.

The changelog helps during upgrade, but later timeout debugging would still require source or PR history.
