## Summary

Keep open: the source-level timeout bug is real, but this PR still accepts malformed timeout env values as tiny positive budgets and does not document the new operator-facing override in stable Dreaming/config docs.

## What This Changes

The PR raises the memory-core Dream Diary narrative timeout default to 240s, adds `OPENCLAW_DREAMING_NARRATIVE_TIMEOUT_MS`, updates the focused timeout assertion, and adds a changelog entry.

## Best Possible Solution

Land the timeout-budget fix after strict whole-string positive-integer env parsing, focused invalid-override tests, and stable docs that state the env var name, millisecond unit, 240s default, and fallback behavior.

## Reproduction Assessment

Yes, at source level: current `main` waits only 60s and returns before appending diary text on timeout, while #76333 provides repeated field logs from a multi-workspace host. I did not live-run that host-scale setup.

## Solution Assessment

Not yet: the higher bounded default and env override are the right narrow shape, but the override should parse strict positive millisecond integers and be documented outside the changelog before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Reject partial timeout env values:** `extensions/memory-core/src/dreaming-narrative.ts:109`
  - body: `Number.parseInt` accepts numeric prefixes, so values operators are likely to try, such as `240_000`, `240s`, or `1e5`, become 240ms, 240ms, or 1ms instead of falling back to the 240s default. That can reintroduce the timeout failure this PR is meant to fix; parse the whole trimmed string as a strict positive integer and cover partial/invalid values.
  - confidence: 0.86
- **[P3] Document the timeout override:** `extensions/memory-core/src/dreaming-narrative.ts:100-105`
  - body: This adds `OPENCLAW_DREAMING_NARRATIVE_TIMEOUT_MS` as an operator-facing setting, but the stable Dreaming/config docs still list only the existing dreaming keys and do not state the millisecond unit, 240s default, or invalid-value fallback. A changelog entry helps upgrades, but later debugging still requires source or PR history.
  - confidence: 0.9
