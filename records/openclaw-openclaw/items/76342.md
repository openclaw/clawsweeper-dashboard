## Summary

Keep open: the PR addresses a real source-reproducible Dream Diary timeout problem, but the new environment override currently accepts malformed partial values and the operator-facing setting still needs stable docs.

## What This Changes

The PR raises the memory-core Dream Diary narrative timeout default from 60s to 240s, adds `OPENCLAW_DREAMING_NARRATIVE_TIMEOUT_MS`, updates focused timeout coverage, and adds a changelog entry.

## Best Possible Solution

Land the timeout/default fix after strict positive-integer env parsing, regression coverage for invalid and partial overrides, and stable docs that state the env var unit, default, and fallback behavior.

## Reproduction Assessment

Yes, at source level: current main hardcodes 60s and passes that value into `waitForRun`, while #76333 provides repeated timeout logs from a multi-workspace host. I did not live-run that host-scale setup in this read-only review.

## Solution Assessment

Not yet: raising the bounded default and adding an env override is the right narrow shape, but the parser should reject malformed whole-string values and the new operator setting should be documented before merge.

## Review Findings

Overall correctness: patch is incorrect

The changelog helps for this release, but later timeout debugging would still require source or PR history.
