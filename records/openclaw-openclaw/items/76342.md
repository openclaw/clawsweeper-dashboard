## Summary

Keep this PR open: current main still has the 60s Dream Diary narrative timeout, and the PR is a narrow source-backed fix for the timeout path, but the new operator env override still needs durable docs beyond the changelog before merge.

## What This Changes

The PR raises the memory-core Dream Diary narrative timeout default to 240s, adds `OPENCLAW_DREAMING_NARRATIVE_TIMEOUT_MS`, updates the focused test assertion, and adds a changelog entry.

## Best Possible Solution

Land the timeout increase and env override after adding stable operator docs for the override, leaving any schema-level config knob for a separate maintainer decision.

## Reproduction Assessment

Yes, at source level: current main hardcodes a 60s wait budget and returns without appending diary text on timeout, while #76333 provides repeated field logs from a 15-workspace host. I did not live-run that host-scale scenario.

## Solution Assessment

Mostly yes: raising the bounded default directly addresses the exhausted wait budget, and the env override is a narrow operator escape hatch. The safer merge path is to document the new knob in durable Dreaming and memory config docs before landing.
