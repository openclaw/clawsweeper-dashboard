## Summary

Keep this PR open: it targets a real current-main memory-core timeout bug, but the latest patch still accepts partial timeout env values and leaves the new operator setting out of stable Dreaming/config docs.

## What This Changes

The PR raises the memory-core Dream Diary narrative timeout from 60s to 240s, adds `OPENCLAW_DREAMING_NARRATIVE_TIMEOUT_MS`, updates one focused timeout assertion, and adds a changelog entry.

## Best Possible Solution

Land the timeout/default fix after strict whole-string positive-integer env parsing, regression coverage for invalid overrides, stable docs for the env var, and normal PR rebase/validation.

## Reproduction Assessment

Yes, at source level: current main waits only 60s and returns on timeout before appending Dream Diary text, while the linked issue provides repeated field logs from a multi-workspace host. I did not live-run that host-scale setup.

## Solution Assessment

Not yet: raising the bounded default and adding an env override is the narrow maintainable shape, but the override must reject partial numeric strings and the operator-facing setting needs stable docs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reject partial timeout env values:** `extensions/memory-core/src/dreaming-narrative.ts:108`
  - body: `Number.parseInt` accepts numeric prefixes, so values like `240_000`, `240s`, `1e5`, or `1.5` become small positive timeouts instead of falling back to 240s. That can reintroduce the Dream Diary timeout failure this PR is trying to fix; parse the whole trimmed value as a positive integer and cover malformed cases.
  - confidence: 0.88
- **[P3] Document the timeout override:** `extensions/memory-core/src/dreaming-narrative.ts:99`
  - body: This introduces `OPENCLAW_DREAMING_NARRATIVE_TIMEOUT_MS` as an operator-facing setting, but the stable Dreaming and memory config docs still list only the existing dreaming keys. Please document the millisecond unit, 240s default, and invalid-value fallback outside the changelog.
  - confidence: 0.9
