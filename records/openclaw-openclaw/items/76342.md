## What This Changes

The branch raises the memory-core Dream Diary narrative wait default from 60s to 240s, adds an `OPENCLAW_DREAMING_NARRATIVE_TIMEOUT_MS` override, updates the focused test assertion, and adds a changelog entry.

## Best Possible Solution

Land or replace this with one memory-core fix that uses a strict positive-integer timeout contract, updates tests and release notes, and includes redacted real Dream Diary runtime proof.

## Reproduction Assessment

Yes, source-backed rather than live-run: current main and v2026.5.12 still pass a fixed 60s timeout into the Dream Diary subagent wait, and the linked issue provides repeated real timeout logs from multi-workspace setups.

## Solution Assessment

Unclear. The PR touches the right wait path, but maintainers still need to choose the supported timeout contract and the parser should reject mixed numeric strings first.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.82

Full review comments:

- **[P3] Reject partially numeric timeout overrides:** `extensions/memory-core/src/dreaming-narrative.ts:109`
  - body: `Number.parseInt` accepts mixed strings such as `1m` as `1`, so a malformed `OPENCLAW_DREAMING_NARRATIVE_TIMEOUT_MS=1m` would turn the intended bounded wait into 1 ms and make narratives time out immediately. Parse the full trimmed value as a positive integer and cover a mixed-string case.
  - confidence: 0.84
