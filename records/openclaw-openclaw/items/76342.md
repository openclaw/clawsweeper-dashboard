## Summary

Keep open: current main still has the hardcoded 60s Dream Diary wait and the linked reports show a real timeout problem, but this PR head still has a partial env-parser bug, a stable-docs gap, and insufficient real Dream Diary proof.

## What This Changes

The PR raises the memory-core Dream Diary narrative timeout default to 240s, adds an `OPENCLAW_DREAMING_NARRATIVE_TIMEOUT_MS` env override, updates the focused timeout assertion, and adds a changelog entry.

## Best Possible Solution

Land a bounded 240s default with strict whole-string positive-integer env parsing, stable docs for the override, focused regression coverage, changelog coverage, green checks, and real Dream Diary run proof.

## Reproduction Assessment

Yes at source and field-log level, but not from a live run in this review. Current main hardcodes 60s and passes it to `waitForRun`, while #76333 and #77094 provide logs showing Dream Diary narratives timing out and leaving `DREAMS.md` stale.

## Solution Assessment

No, not as currently written. Raising the default and adding an override is the right narrow shape, but the parser should reject partial values, the env knob needs stable docs, and the proof should show actual Dream Diary behavior after the fix.
