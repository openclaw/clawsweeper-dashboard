## Summary

Keep open. The PR is a narrow functional fix for a real memory-core Dream Diary timeout path on current main, but it adds an operator-facing timeout override without the required changelog and docs coverage.

## What This Changes

This PR raises the memory-core Dream Diary narrative timeout from 60s to 240s, adds `OPENCLAW_DREAMING_NARRATIVE_TIMEOUT_MS` parsing, and updates the focused timeout assertion.

## Best Possible Solution

Land the timeout increase with explicit release notes and operator-facing documentation, then evaluate a schema-backed config knob separately if maintainers want non-env tuning.

## Reproduction Assessment

Yes, at source level. Current main hardcodes a 60s `waitForRun` budget and the linked issue supplies repeated timeout logs for that path; this review did not live-reproduce the 15-workspace latency environment.

## Solution Assessment

Mostly yes. Raising the hardcoded budget directly addresses the observed timeout, and the env override is a narrow operator escape hatch, but it needs docs and changelog coverage before merge.
