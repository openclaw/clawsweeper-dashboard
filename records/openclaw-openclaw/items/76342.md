## Summary

Keep open: current main still hardcodes the 60s Dream Diary narrative wait, and the linked field reports plus source path support the bug, but this PR still needs strict env parsing and stable docs before merge.

## What This Changes

The PR raises the memory-core Dream Diary narrative timeout default to 240 seconds, adds an `OPENCLAW_DREAMING_NARRATIVE_TIMEOUT_MS` override, updates the focused timeout assertion, and adds a changelog entry.

## Best Possible Solution

Land the timeout fix after strict positive-integer env parsing, focused override regression tests, and stable docs for the env var's unit, default, and fallback behavior; leave schema-level config and late-output recovery to separate maintainer decisions.

## Reproduction Assessment

Yes, at source level: current main waits only 60s for the narrative subagent and returns before appending diary text on timeout, while #76333 and #77094 provide repeated field logs from affected hosts. I did not live-run that host-scale workload in this read-only review.

## Solution Assessment

Not yet: raising the bounded default and adding an env override is the right narrow shape, but the override must reject partial numeric strings and the operator-facing setting needs stable docs before merge.
