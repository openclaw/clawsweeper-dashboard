## Summary

Keep this PR open: it targets a valid memory-core Dream Diary timeout bug, but the current patch still needs a strict env-var parser and stable docs for the new operator-facing timeout knob.

## What This Changes

The PR raises the memory-core Dream Diary narrative wait default from 60s to 240s, adds `OPENCLAW_DREAMING_NARRATIVE_TIMEOUT_MS`, updates one focused timeout assertion, and adds a changelog entry.

## Best Possible Solution

Land a bounded 240s default with strict whole-string positive-integer env parsing, focused override regression tests, and stable docs for the env knob.

## Reproduction Assessment

Yes, at source level: current main hardcodes 60s into `waitForRun` and returns before appending to `DREAMS.md` on timeout, while #76333 and #77094 provide matching field logs. I did not live-run a many-workspace host in this read-only pass.

## Solution Assessment

No, not yet: raising the bounded default and adding an env override is the right narrow shape, but the parser accepts partial values and the new operator knob is not in stable docs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

- **[P2] Reject partial timeout override values:** `extensions/memory-core/src/dreaming-narrative.ts:108`
  - body: `Number.parseInt` accepts numeric prefixes, so likely operator inputs such as `240_000`, `240s`, or `1e5` become 240ms, 240ms, or 1ms instead of falling back to the 240s default. That can recreate the narrative timeout loss this PR is meant to fix; parse the trimmed whole string as a positive integer and cover valid and invalid override cases.
  - confidence: 0.9
- **[P3] Document the timeout override:** `extensions/memory-core/src/dreaming-narrative.ts:99-101`
  - body: The diff introduces `OPENCLAW_DREAMING_NARRATIVE_TIMEOUT_MS` as an operator-facing knob, but the Dreaming and memory config docs still only list `enabled`, `frequency`, and `model`. The changelog helps upgrades, but stable docs should state the env var, millisecond unit, 240s default, and invalid-value fallback.
  - confidence: 0.86
