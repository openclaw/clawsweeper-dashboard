## Summary

Keep open: the timeout failure remains present on current main, and this PR is a valid fix direction, but the override parser still accepts partial values that can collapse the intended 240s budget to milliseconds.

## What This Changes

The PR raises the memory-core Dream Diary narrative timeout from 60s to 240s, adds `OPENCLAW_DREAMING_NARRATIVE_TIMEOUT_MS`, updates one focused assertion, and adds a changelog entry.

## Best Possible Solution

Land the timeout-budget fix after strict whole-string positive-integer env parsing, focused override tests, stable docs, and branch rebase/conflict cleanup.

## Reproduction Assessment

Yes, at source level: current main waits only 60s and returns before appending diary text on timeout, while #76333 provides repeated multi-workspace timeout logs. I did not live-run that host-scale setup.

## Solution Assessment

No, not yet: raising the bounded default and adding an env override is the right narrow shape, but the current parser accepts partial values and the new operator knob is not documented in stable docs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reject partial timeout env values:** `extensions/memory-core/src/dreaming-narrative.ts:108`
  - body: `Number.parseInt` accepts numeric prefixes, so values operators are likely to try, such as `240_000`, `240s`, or `1e5`, become 240ms, 240ms, or 1ms instead of falling back to the 240s default. That can reintroduce the timeout failure this PR is meant to fix; parse the whole trimmed string as a strict positive integer and cover partial/invalid values.
  - confidence: 0.89
- **[P3] Document the timeout override:** `extensions/memory-core/src/dreaming-narrative.ts:99`
  - body: This adds `OPENCLAW_DREAMING_NARRATIVE_TIMEOUT_MS` as an operator-facing setting, but the stable Dreaming/config docs still list only the existing dreaming keys and do not state the millisecond unit, 240s default, or invalid-value fallback. A changelog entry helps upgrades, but later debugging still requires source or PR history.
  - confidence: 0.86
