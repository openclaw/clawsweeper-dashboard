## Summary

Keep this PR open: the bounded-read direction is the right layer for the provider usage DoS class, but the current patch changes JSON decoding semantics and the external-contributor proof is only test/mocked evidence.

## What This Changes

The PR routes provider usage JSON parsing through `readResponseWithLimit` with a 16 MiB cap and adds shared-helper tests for normal, malformed, and oversized responses.

## Best Possible Solution

Keep the shared bounded-read approach, decode bytes with `TextDecoder` to preserve `response.json()` semantics, cover the BOM case, and require redacted real behavior proof before merge.

## Reproduction Assessment

Yes for the review finding: Node shows `Response.json()` and `TextDecoder` accept a UTF-8 BOM while `JSON.parse(buffer.toString("utf8"))` rejects it. Current main uses `response.json()`, and the PR diff switches that exact parse path to `Buffer.toString()`.

## Solution Assessment

No, not yet: bounding the shared helper is the best fix layer, but the implementation should preserve the old JSON decoding contract and the PR needs real behavior proof before merge.
