## Summary

This PR is a focused and useful fix for the Feishu WebSocket ambient-proxy bug, with sufficient after-fix transport proof and no blocking code-review findings. It should stay open for maintainer approval because it intentionally changes the default for existing proxy-dependent Feishu deployments.

## What This Changes

The PR makes Feishu WebSocket proxy routing opt-in via `OPENCLAW_FEISHU_WS_USE_PROXY=1`, adds Feishu client tests, and documents the upgrade behavior.

## Best Possible Solution

Land the Feishu-owned direct-by-default proxy policy only after maintainer approval, keeping the explicit opt-in docs and regression tests.

## Reproduction Assessment

## Solution Assessment

Mostly yes. The implementation is narrow and Feishu-owned, but the direct-by-default behavior is an upgrade policy choice maintainers should explicitly accept.
