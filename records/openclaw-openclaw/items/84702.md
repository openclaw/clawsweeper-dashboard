## Summary

Keep this PR open: the patch looks focused and evidence-backed, but it intentionally flips the Feishu WebSocket proxy default for existing deployments, so merge should wait for maintainer acceptance of that upgrade tradeoff.

## What This Changes

The PR makes Feishu WebSocket proxy routing direct by default, adds `OPENCLAW_FEISHU_WS_USE_PROXY=1` as the ambient proxy opt-in, and updates Feishu tests, docs, and changelog.

## Best Possible Solution

Land the Feishu-owned direct-by-default proxy policy only with the explicit opt-in docs/tests and maintainer acceptance of the upgrade tradeoff.

## Solution Assessment
