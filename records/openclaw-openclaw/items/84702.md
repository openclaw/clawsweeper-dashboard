## Label Justifications

- `P1`: The linked bug affects real Feishu inbound message delivery and can make the channel stop receiving messages while appearing otherwise healthy.
- `merge-risk: 🚨 compatibility`: The PR intentionally flips the default for existing Feishu deployments that relied on ambient proxy env.

## Summary

This PR is a focused, plausible fix for a real Feishu inbound-delivery failure, and I found no discrete code defect in the patch. It should stay open for maintainer approval because the default changes for proxy-only Feishu deployments: existing setups that relied on ambient proxy env must now set the documented opt-in before upgrade.

## What This Changes

The PR changes Feishu WebSocket creation to ignore ambient proxy env by default, adds `OPENCLAW_FEISHU_WS_USE_PROXY=1` as the opt-in, and updates Feishu tests, docs, and changelog.

## Best Possible Solution

Land the Feishu-owned direct-by-default proxy policy only after maintainer approval, preserving the explicit opt-in docs and regression tests.

## Reproduction Assessment

Yes. Current main unconditionally resolves an ambient Feishu WS proxy agent, the linked issue describes a concrete broken proxy setup, and the PR supplies after-fix local WSS/proxy output.

## Solution Assessment

Mostly yes. The implementation is narrow and owner-local, but the direct-by-default behavior is an upgrade policy choice that maintainers should explicitly accept.
