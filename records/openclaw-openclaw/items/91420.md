## What This Changes

The issue reports that a failed sessions_send/delivery path can duplicate delivery and route through active persistent sessions despite no-deliver settings, corrupting session state and causing later provider schema errors.

## Best Possible Solution

Keep sessions_send delivery internal to the intended active/run-scoped target: if the active-run queue rejects a message_tool_only/no-deliver send, fail or surface a delivery error instead of falling back through a persistent main session, with regression coverage for duplicate delivery and transcript safety.
