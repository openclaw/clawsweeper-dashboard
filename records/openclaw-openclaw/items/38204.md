## Summary

Keep this open: current main supports standard Grok 4.20 reasoning/non-reasoning models, but still explicitly excludes Grok 4.20 multi-agent IDs and documents that the multi-agent endpoint is unsupported.

## What This Changes

Request native support for xAI Grok 4.20 multi-agent model IDs instead of the standard Grok 4.20 workaround.

## Best Possible Solution

If maintainers want bundled support, add a provider-owned xAI multi-agent path in extensions/xai with explicit model resolution, endpoint/payload handling, docs, regression tests, and live xAI proof; otherwise document that this belongs in an external plugin/ClawHub path.

## Reproduction Assessment

Yes. Source inspection on current main shows multi-agent IDs resolve to undefined and are rejected by the provider model hook, with tests asserting that behavior; no live xAI call was needed to confirm OpenClaw currently treats the ID as unsupported.

## Solution Assessment

Not applicable because there is no proposed patch. A catalog-only model addition is not the best fix unless the upstream multi-agent API matches the existing Responses transport, which current docs and tests say it does not.
