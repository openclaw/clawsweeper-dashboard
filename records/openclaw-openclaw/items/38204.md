## Summary

This is a useful xAI-specific integration idea, but the remaining native multi-agent endpoint support is optional provider/tool work that can live as a ClawHub plugin rather than OpenClaw core; current main already documents the supported Grok 4.20 chat-compatible variants and the unsupported multi-agent path.

## What This Changes

Request support for xAI Grok 4.20 multi-agent model/endpoint, with discussion noting chat-compatible 4.20 reasoning variants already work.

## Best Possible Solution

Keep OpenClaw core focused on the shared xAI Chat/Responses provider and publish any xAI multi-agent endpoint integration as a ClawHub provider or tool plugin unless a concrete missing plugin API is identified.

## Reproduction Assessment

Not applicable. This is a feature/integration request, and source inspection confirms current main intentionally supports chat-compatible Grok 4.20 variants while excluding multi-agent ids from the normal xAI provider path.

## Solution Assessment

Yes. A separate ClawHub provider/tool plugin is the narrowest maintainable direction because xAI's multi-agent variant has different endpoint and tool semantics, while OpenClaw already exposes provider and tool plugin seams for custom endpoints.
