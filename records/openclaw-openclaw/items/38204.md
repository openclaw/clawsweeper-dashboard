## Summary

Close as ClawHub scope: the remaining ask is an xAI-specific multi-agent endpoint integration, while current main already supports and documents the chat-compatible Grok 4.20 variants and intentionally keeps unsupported multi-agent ids out of the bundled xAI provider path.

## What This Changes

Request support for xAI's Grok 4.20 multi-agent model/endpoint in OpenClaw instead of using the chat-compatible Grok 4.20 variants.

## Best Possible Solution

Keep the bundled xAI provider on shared Responses/tool contracts and publish any Grok 4.20 multi-agent endpoint integration as a ClawHub provider/tool plugin unless a concrete missing plugin API is identified.

## Reproduction Assessment

Not applicable. This is a feature/integration request, and source inspection confirms current main intentionally supports chat-compatible Grok 4.20 variants while excluding multi-agent ids from the normal xAI provider path.

## Solution Assessment

Yes. ClawHub is the narrower maintainable path because the multi-agent variant has different endpoint/tool semantics, while OpenClaw already exposes provider, tool, and agent-harness plugin seams for custom integrations.
