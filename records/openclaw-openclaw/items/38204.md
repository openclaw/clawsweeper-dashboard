## Summary

Current main still intentionally excludes Grok 4.20 multi-agent IDs from the bundled xAI provider. Standard Grok 4.20 reasoning/non-reasoning support and the xAI Responses migration have shipped, but the requested native multi-agent surface remains unsupported in code, docs, and regression tests.

## What This Changes

Request support for xAI Grok 4.20 multi-agent models, instead of only the standard Grok 4.20 reasoning/non-reasoning variants that work through the normal xAI provider path.

## Best Possible Solution

Keep this open until maintainers decide the product boundary. If bundled support is approved, implement it inside `extensions/xai` with explicit multi-agent model resolution, payload shaping, unsupported-parameter filtering, docs, focused regression coverage, and live xAI proof. If maintainers decide it belongs outside the bundled provider, document that path and close only with explicit plugin/ClawHub guidance.
