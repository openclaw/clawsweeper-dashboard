## Summary

Keep open: current main supports xAI Grok 4.20 beta reasoning and non-reasoning models on the Responses path, but it still intentionally excludes multi-agent IDs and documents the multi-agent variant as unsupported. Native multi-agent support remains a feature/product decision, not conservative cleanup.

## What This Changes

The issue asks for native support for xAI Grok 4.20 multi-agent variants instead of only the standard Grok 4.20 reasoning-compatible models.

## Best Possible Solution

Maintainers should decide whether Grok multi-agent belongs in the bundled xAI plugin, a new provider/agent-harness surface, or an external ClawHub plugin; if accepted, keep the implementation extension-owned with generic core seams only if multiple providers need them.

## Reproduction Assessment

Yes, from source inspection: current main maps supported Grok 4.20 beta single-agent IDs but returns `undefined` for IDs containing `multi-agent`, and the docs list the multi-agent variant as unsupported. I did not run a live xAI request.

## Solution Assessment

No proposed patch is under review. The documented workaround is already present, but native multi-agent support is not a narrow bug fix and needs a maintainer decision on the provider/plugin contract first.
