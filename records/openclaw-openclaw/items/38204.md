## Summary

This can close as ClawHub scope: current main supports and documents chat-compatible Grok 4.20 variants, deliberately excludes multi-agent model ids from the normal xAI provider path, and already has provider/tool/harness plugin seams for this optional xAI-specific integration.

## What This Changes

The issue asks for OpenClaw support or guidance for xAI Grok 4.20 multi-agent models, which do not fit the normal xAI model provider path.

## Best Possible Solution

Keep OpenClaw core focused on the shared xAI provider path and move Grok 4.20 multi-agent support into a ClawHub provider, tool, or trusted harness plugin unless a concrete missing SDK seam is identified.

## Reproduction Assessment

Not applicable. This is a feature/integration request, and source/docs inspection confirms current main intentionally supports chat-compatible Grok 4.20 ids while excluding multi-agent ids from the normal xAI provider path.

## Solution Assessment

Yes. A ClawHub provider/tool or trusted harness plugin is the narrowest maintainable direction because the upstream multi-agent variant has server-side agent/tool semantics that do not fit OpenClaw's normal client-side tool-calling provider loop.
