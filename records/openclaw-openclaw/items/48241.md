## Summary

Keep open. Current main still lacks an aborted-run signal in embedded payload assembly and still has source-backed paths that can forward nonblank abort buffers, while the live replacement PR at https://github.com/openclaw/openclaw/pull/81646 is open, maintainer-labeled, and lists this issue as a closing reference.

## What This Changes

Discard or suppress user-facing assistant output from aborted agent runs when partial buffers may be stale or untrusted, especially in Telegram/embedded PI delivery.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81646 or an equivalent patch that passes aborted state into embedded payload assembly and suppresses aborted assistant text, reasoning, directives, and stale fallback replies without accidentally removing the intentional gateway/WebChat partial-retention contract.

No, there is not a high-confidence live Telegram/sub2api reproduction from this review. Source inspection on current main does show nonblank abort buffers and embedded assistant texts can still flow into user-facing assistant payload paths, matching the reported failure class.
