## Summary

Keep open. Current main still treats a missing `node:sqlite` as memory-manager unavailability and returns an empty unavailable payload; the existing fallback code only works after a SQLite-backed index is already open, and the related #69199/#59457 work is explicitly the narrower diagnostic-message path.

## What This Changes

The issue asks for `memory_search` to fall back to a clearly marked keyword scan of `MEMORY.md` and `memory/*.md` when `node:sqlite` is unavailable.

## Best Possible Solution

Add a bounded memory-core fallback that detects the missing-`node:sqlite` manager-init failure, scans only the configured workspace memory files, ranks keyword snippets deterministically, and labels the response as fallback keyword search.

## Reproduction Assessment

Yes. A source-level reproduction is to make `require("node:sqlite")` throw, create an agent workspace with `MEMORY.md` or `memory/*.md`, and call `memory_search`; current main reaches the unavailable-result path before any file scan can run.

## Solution Assessment

Yes. The requested fallback is the narrowest maintainable direction as long as it is limited to the SQLite-runtime-unavailable case and reuses existing memory path/read constraints instead of changing provider fallback policy.

## Repair Work Prompt

Implement the fallback requested by https://github.com/openclaw/openclaw/issues/59451. When `memory_search` cannot initialize the builtin memory manager specifically because `node:sqlite` is unavailable, perform a bounded keyword scan over the configured agent workspace memory files (`MEMORY.md` and `memory/*.md`) and return ranked snippets with explicit fallback/mode/debug metadata so callers can tell these are fallback keyword results, not vector search. Keep the trigger narrow: do not change quota/provider failures, supplement-only `corpus=wiki`, QMD fallback policy, or unrelated startup warning behavior. Reuse existing memory path/config helpers where possible, respect session/corpus filtering semantics, avoid reading outside allowed memory files, and coordinate with #69199/#59457 as diagnostic-message scope only. Add focused tests that mock the sqlite-missing manager-init path and verify file hits plus metadata, no hits for missing files, and existing unavailable responses for non-sqlite failures. Add a single-line `CHANGELOG.md` user-facing entry crediting the human reporter if policy allows.
