## Summary

Keep open. Current main still returns an unavailable empty result when the memory manager cannot initialize, and the related diagnostic work in https://github.com/openclaw/openclaw/pull/69199 is explicitly narrower than the requested file-based fallback.

## What This Changes

Request `memory_search` to fall back to bounded keyword scanning of `MEMORY.md` and `memory/*.md` when `node:sqlite` cannot initialize.

## Best Possible Solution

Add a bounded memory-core fallback that detects missing-`node:sqlite` manager initialization, scans only canonical workspace memory files, ranks snippets deterministically, and labels the response as fallback keyword search.

## Reproduction Assessment

Yes. A source-level reproduction is to make `require("node:sqlite")` throw, create an agent workspace with `MEMORY.md` or `memory/*.md`, and call `memory_search`; current main reaches the unavailable-result path before any file scan can run.

## Solution Assessment

Yes. The requested fallback is the narrow maintainable direction if it is limited to the SQLite-runtime-unavailable case and reuses existing memory path/read constraints rather than changing provider fallback policy.
