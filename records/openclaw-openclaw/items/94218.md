Full review comments:

- **[P1] Preserve fallback send context:** `extensions/telegram/src/send.ts:877-879`
  - body: The rich send uses `acceptedParams` for topic, reply, and final-chunk button context, but the fallback plain send calls `sendTelegramTextChunk` with only the plain text. If rich sending fails in a topic, reply, or interactive final chunk, the fallback can land in the wrong context or lose markup.
  - confidence: 0.93
- **[P2] Cover the durable rich edit fallback:** `extensions/telegram/src/send.ts:1613-1618`
  - body: This PR wraps sends and draft edits, but `performTextEdit` still calls `richRawApi.editMessageText` directly when rich messages are enabled. Opt-in rich edits can still fail on the same unsupported-rich condition instead of retrying through the regular HTML/plain edit path.
  - confidence: 0.88
- **[P3] Tighten the unsupported-client matcher:** `extensions/telegram/src/send.ts:251`
  - body: The `not supported on .* telegram` branch can match future or unrelated Telegram errors and convert them into plain-text retries. Keep this scoped to known Telegram Web or version-unsupported wording so unrelated API failures still propagate.
  - confidence: 0.78
