## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve fallback send context:** `extensions/telegram/src/send.ts:877-879`
  - body: The rich send uses `acceptedParams` for topic, reply, silent, and final-chunk button context, but the fallback plain send does not pass those params into `sendTelegramTextChunk`. If rich sending fails in a topic, reply, or interactive final chunk, the fallback can land in the wrong context or lose markup.
  - confidence: 0.93
- **[P2] Cover the durable rich edit path:** `extensions/telegram/src/send.ts:1611-1618`
  - body: The PR wraps sends and draft edits, but `performTextEdit` still calls `richRawApi.editMessageText` directly when rich messages are enabled. Editing an opt-in rich message on an unsupported client can still surface the same failure instead of retrying through the plain edit path.
  - confidence: 0.88
- **[P3] Tighten the unsupported-client matcher:** `extensions/telegram/src/send.ts:251`
  - body: The `not supported on .* telegram` branch can match future or unrelated Telegram errors and convert them into plain-text retries. Keep this scoped to the known Telegram Web or version-unsupported wording so unrelated API failures still propagate.
  - confidence: 0.78
