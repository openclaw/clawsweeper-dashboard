## Summary

This is an exact repeat of earlier closed task-maintenance reports, and current main still shows the supported `--apply` path pruning expired terminal task rows through the SQLite delete path.

## What This Changes

The issue reports that `openclaw tasks maintenance --apply` does not delete expired terminal task records from the SQLite store after `cleanupAfter` passes.

## Best Possible Solution

Keep `openclaw tasks maintenance --apply` and the automatic Gateway sweeper as the single cleanup contract, and use the prior canonical reports for any future evidence on this behavior.

## Reproduction Assessment

No. I did not establish a failing current-main path; source, docs, and regression coverage show `--apply` pruning expired terminal records, and the report duplicates earlier closed cleanup reports.

## Solution Assessment
