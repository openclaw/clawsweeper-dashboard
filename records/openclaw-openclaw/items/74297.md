## Summary

Keep open. Current main mitigates the original Anthropic wrong-header path for API-key-shaped values, but it still accepts, resolves, and can prioritize static Anthropic `token` profiles without a doctor/status warning or explicit Enterprise API-key auth-profile guidance.

## What This Changes

Request Anthropic auth-profile handling to warn and document that Enterprise/API keys should use `type: "api_key"` rather than static `type: "token"` entries.

## Best Possible Solution

Preserve Anthropic setup-token support, add Anthropic-specific doctor/status guidance for API-key-shaped or static token profiles, and document the canonical Enterprise `api_key` profile shape.
