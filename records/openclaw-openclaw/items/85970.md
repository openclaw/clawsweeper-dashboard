The PR updates agent runtime policy matching and CLI alias resolution so providerless session models can match provider-prefixed `agentRuntime` entries, with regression tests and a changelog entry.

## Best Possible Solution

Land the PR after maintainers accept the fail-closed ambiguity behavior and upgrade impact, keeping the matched-provider handoff as the canonical runtime-policy source instead of adding another fallback lookup.
