## Summary

Keep open. Current main still rejects the requested durable `session.contextSafety` and top-level `contextSafety` paths, while current source and docs expose the supported compaction controls under `agents.defaults.compaction`, so this needs a maintainer config-contract decision rather than cleanup closure.

## What This Changes

The issue requests a durable public config schema/type/docs surface for context safety settings, preferably under `session.contextSafety`, with possible top-level legacy alias support.

## Best Possible Solution

Maintainers should either define and document an end-to-end `contextSafety` public config contract, including schema/types/runtime/docs metadata, or explicitly document `agents.defaults.compaction` as the supported replacement and remove any artifact-only expectation.

## Reproduction Assessment

Yes for the schema rejection: static inspection of the strict `SessionSchema`, strict root `OpenClawSchema`, and normal validation path shows both requested durable keys are rejected. The runtime-support half is unclear because the reported runtime symbols and slash commands are absent from current source.

## Solution Assessment

Unclear. A schema-only addition is not clearly the best fix until maintainers decide whether `contextSafety` should become a public contract or whether users should configure the existing `agents.defaults.compaction` surface.
