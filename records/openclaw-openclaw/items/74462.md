## Summary

Keep open. Current main still rejects the reported durable config paths, and the claimed `contextSafety` runtime/command surface is not present in current source, so this needs maintainer config-contract follow-up rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to add a supported durable config schema/types surface for `session.contextSafety` or top-level `contextSafety`.

## Best Possible Solution

Pick one public contract and align schema, types, metadata, docs, runtime behavior, and tests end to end: either implement session-level context rollover/handoff or explicitly document `contextSafety` as unsupported and route operators to `agents.defaults.compaction`.

## Reproduction Assessment

Yes for the schema rejection: the strict session/root schemas and normal validation path make both requested paths unknown on current main. The runtime-support claim is unclear because current source lacks the cited `contextSafety` symbols and rollover/handoff commands.

## Solution Assessment

Unclear. A schema-only patch is not the best fix until maintainers confirm the intended public config surface; the safer path is either a complete rollover contract tracked with #75699 or a docs/support correction to the existing compaction surface.
