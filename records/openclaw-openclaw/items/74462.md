## Summary

Keep open. Current main still rejects the requested durable `contextSafety` paths, while current source does not show the cited runtime/command support, so this needs maintainer config-contract follow-up rather than cleanup closure.

## What This Changes

The issue requests a durable schema/types/docs/runtime config surface for context safety, preferably under `session.contextSafety`, because installed artifacts appeared to read that setting while config validation rejected it.

## Best Possible Solution

Choose one public contract and align schema, types, generated config metadata, docs, runtime behavior, and tests end to end: either implement a session-level rollover/handoff policy with #75699 or explicitly document `contextSafety` as unsupported and route operators to `agents.defaults.compaction`.

## Reproduction Assessment

Yes for the schema rejection: the strict session and root schemas plus the normal validation path make both `session.contextSafety` and top-level `contextSafety` unknown. The runtime-support half is unclear because current source lacks the cited symbols and commands.

## Solution Assessment

Unclear. Accepting the field in schema alone is not the best fix until maintainers confirm the intended public config surface; the safer solution is an end-to-end rollover contract or an explicit support/docs correction to the existing compaction surface.
