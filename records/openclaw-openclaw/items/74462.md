## Summary

Keep open. Current main still rejects both requested `contextSafety` config paths, and current source still lacks the `contextSafety` runtime surface and handoff/rollover commands cited from the installed artifact, so this needs maintainer product/API direction rather than cleanup closure.

## What This Changes

The issue requests durable config schema, TypeScript types, metadata, and docs for `contextSafety`, preferably under `session.contextSafety`, with possible top-level alias support.

## Best Possible Solution

Maintainers should decide the durable public contract: either implement `contextSafety`/rollover end to end with schema, runtime, commands, docs, and tests, or document `agents.defaults.compaction` as the supported surface and close or consolidate after that decision.

## Reproduction Assessment

Yes for the schema rejection: source inspection shows both `session.contextSafety` and top-level `contextSafety` are unknown to strict current-main schemas. The runtime-support half is unclear against current main because the cited symbols and commands are absent from source/docs.

## Solution Assessment

Unclear. Adding only schema/types is not the best fix until maintainers confirm whether `contextSafety` is intended public API; the safer path is an end-to-end lifecycle/config design or a docs/support correction.
