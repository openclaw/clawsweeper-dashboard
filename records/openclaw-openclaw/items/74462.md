## Summary

Keep open. Current main still rejects both requested durable config paths, and the current source lacks the reported `contextSafety` runtime reader and slash commands, so this remains a config-contract decision rather than cleanup closure.

## What This Changes

The issue requests durable schema, type, metadata, and docs support for configuring context safety under `session.contextSafety` or a compatible top-level path.

## Best Possible Solution

Align the public config contract with the intended runtime by either adding typed, schema-validated, documented context-safety support end to end, or explicitly directing users to the existing `agents.defaults.compaction` surface.

## Reproduction Assessment

Yes for the schema rejection: current strict schemas omit both requested paths and normal validation calls the strict root schema. The runtime-support half is unclear from current source because the cited reader and slash command names are absent.

## Solution Assessment

Unclear. Adding only a schema field is not the best fix until maintainers decide whether `contextSafety` should be a public contract or whether the supported answer is the existing compaction configuration surface.
