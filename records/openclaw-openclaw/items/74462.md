## Summary

Keep open. Current main still rejects both requested config paths, but it also lacks the reported `contextSafety` runtime and command surface, and the same author’s broader open rollover request now owns the product/API decision.

## What This Changes

The issue asks OpenClaw to accept durable `session.contextSafety` or top-level `contextSafety` config for automatic context rollover/handoff behavior.

## Best Possible Solution

Choose one public session context-rollover contract in https://github.com/openclaw/openclaw/issues/75699, then align schema, types, metadata, docs, runtime behavior, commands, Telegram/direct delivery safety, and tests together.

## Reproduction Assessment

Yes for the schema rejection path: the strict session/root schemas and raw validation path make both requested config paths unknown on current main. The runtime-support premise is not reproduced because the searched `contextSafety` runtime and cited commands are absent from current source.

## Solution Assessment

No, not as a schema-only fix. The maintainable solution requires a product/API decision and end-to-end rollover implementation, or explicit docs that `contextSafety` is unsupported while operators use existing compaction settings.
