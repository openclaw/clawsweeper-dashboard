## Summary

Keep open. Current main still rejects both requested durable config paths, and the reported runtime/command surface is not present in source; this needs a maintainer decision on whether `contextSafety` should become public config or be redirected to the existing compaction contract.

## What This Changes

Request to add a durable `contextSafety` config contract, likely under `session.contextSafety`, because reported runtime artifacts read that path while schema/types reject it.

## Best Possible Solution

Define one supported end state: either implement `contextSafety` end to end with schema/types/help/docs/runtime wiring, or document/doctor the artifact-only alias away in favor of `agents.defaults.compaction`.

## Reproduction Assessment

Yes for the schema rejection: current strict schemas make both `session.contextSafety` and top-level `contextSafety` unknown keys on normal config validation. The runtime-support half is unclear because the cited runtime symbols and slash commands are absent from current source.

## Solution Assessment

Unclear. A schema-only patch is not the best fix until maintainers confirm whether `contextSafety` is intended public config; the safer path is an end-to-end contract or an explicit docs/doctor correction to the existing compaction surface.
