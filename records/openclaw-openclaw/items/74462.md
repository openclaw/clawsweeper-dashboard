## Summary

Keep open. Current main still rejects both requested config paths, while the cited `contextSafety` runtime and command surface are absent from source, so this needs the broader session rollover product/config decision tracked in #75699 rather than a schema-only cleanup.

## What This Changes

Request to add a durable `session.contextSafety` or legacy top-level `contextSafety` config schema/type/docs surface for context safety settings.

## Best Possible Solution

Choose and implement one public session context rollover contract end to end, or explicitly document `contextSafety` as unsupported and point operators to existing compaction/reset settings.

Yes for the schema rejection from source inspection: both `session.contextSafety` and top-level `contextSafety` are unknown to strict config validation. No for the runtime-support premise: the current source tree lacks the cited `contextSafety` runtime and commands.

## Solution Assessment

Unclear. Adding only a schema field would expose durable knobs without the corresponding current runtime/command behavior; the safer solution is the end-to-end session rollover contract tracked by #75699.
