## Summary

## What This Changes

Make `contextSafety` a durable supported OpenClaw config path so schema/types/config tooling accept explicit context rollover and handoff settings.

## Best Possible Solution

Keep the schema-only request consolidated into https://github.com/openclaw/openclaw/issues/75699 and choose one end-to-end session rollover contract before adding config keys.

## Reproduction Assessment

Yes for the schema rejection path: current strict config schemas and raw validation make both requested config paths unknown. The runtime-support premise is not reproducible on current main because the reported `contextSafety` runtime and commands are absent from source.

## Solution Assessment

No, a schema-only fix is not the best solution. Accepting `contextSafety` without the runtime rollover, durable handoff, command authorization, docs, and Telegram/direct delivery contract would create a durable no-op or misleading config surface.
