## Summary

Keep open. Current main still rejects both requested durable config paths, and the runtime/command surface cited from installed artifacts is not present in current source, so this needs maintainer product/config-contract follow-up alongside #75699 rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to accept and type a durable `contextSafety` configuration path for context rollover and handoff behavior.

## Best Possible Solution

Pick one public contract and align schema, types, metadata, docs, runtime behavior, and tests end to end: either implement session-level context rollover/handoff from #75699 or document `contextSafety` as unsupported and route operators to `agents.defaults.compaction`.

## Reproduction Assessment

Yes for the schema rejection: the current strict schemas and validation path make both requested paths unknown. The runtime-support half is unclear because current source lacks the `contextSafety` symbol and the cited rollover/handoff commands.

## Solution Assessment

Unclear. Adding schema fields alone is not the best fix until maintainers confirm the intended public config surface; the safer path is an end-to-end rollover contract or an explicit docs/support correction to the existing compaction surface.
