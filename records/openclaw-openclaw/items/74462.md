## Summary

Keep open. Current main still rejects both requested durable config paths, and the current source does not contain the `contextSafety` runtime surface named in the report; this needs a maintainer decision on whether to bless that contract or point users to the existing `agents.defaults.compaction` surface.

## What This Changes

The issue requests schema, type, metadata, and docs support for a durable `contextSafety` config path because an installed artifact allegedly reads `cfg.session.contextSafety` or `cfg.contextSafety`.

## Best Possible Solution

Maintainers should either implement and document `contextSafety` end to end as a public config contract, or explicitly confirm `agents.defaults.compaction` as the supported durable surface and treat the artifact-only path as unsupported.

## Reproduction Assessment

Yes for the schema rejection: current strict schemas make both `session.contextSafety` and top-level `contextSafety` unknown. The runtime-support part is unclear because the cited runtime symbol and slash commands are absent from current main source.

## Solution Assessment

Unclear. Adding only schema fields is not the best fix until maintainers decide whether `contextSafety` is intended public config or whether users should use the existing compaction settings instead.
