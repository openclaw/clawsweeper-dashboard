## Summary

Keep open. Current main still rejects the requested durable `contextSafety` config paths, and the current source/docs do not contain the runtime symbols or slash commands cited from installed artifacts; the remaining work needs maintainer config-contract judgment alongside the broader open rollover request.

## What This Changes

Request to add a durable config schema/types/docs surface for context safety so `openclaw.json` can configure session rollover and handoff without being clobbered.

## Best Possible Solution

Pick one public contract and align schema, TypeScript types, generated metadata, docs, runtime behavior, commands, and tests end to end; otherwise document `contextSafety` as unsupported and route operators to the existing compaction surface.

## Reproduction Assessment

Yes for the schema rejection: the strict current schemas omit both requested paths and raw validation calls `OpenClawSchema.safeParse`. The runtime-support half is not reproducible from current source because the cited symbols and commands are absent.

## Solution Assessment

Unclear. Adding schema fields alone is not the best fix until maintainers decide whether the public surface should be `session.contextSafety`, the broader `session.contextRollover` direction from #75699, or the existing `agents.defaults.compaction` knobs.
