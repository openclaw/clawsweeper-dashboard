## Summary

Keep open: current main covers the generic SGLang/OpenAI-compatible path, but the bundled Mistral provider still owns its replay policy and does not opt into the existing same-role turn validators, so the Mistral-family strict-alternation part remains unresolved.

## What This Changes

The issue requests built-in strict role-alternation handling for Mistral/SGLang-style backends so consecutive user or assistant turns are merged before provider replay.

## Best Possible Solution

Extend the Mistral provider replay policy to reuse the existing strict OpenAI-compatible turn validators while preserving strict9 tool-call IDs, then add focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes, from source inspection: Mistral's provider-owned replay policy bypasses the generic strict OpenAI-compatible fallback and does not enable the validators that merge consecutive user/assistant turns. I did not run a live Mistral/SGLang HTTP 400 reproduction.

## Solution Assessment

Yes: the narrow maintainable fix is to reuse the existing replay-policy flags and validators in the Mistral provider instead of adding a new config knob or side-car proxy path.
