## What This Changes

The issue requests built-in strict role-alternation handling for Mistral-family OpenAI-compatible backends, especially Mistral models served through SGLang.

## Best Possible Solution

Extend the bundled Mistral replay policy to reuse the existing strict-turn validators while preserving strict9 tool-call IDs, then add focused regression coverage.

## Solution Assessment

Yes: the narrow maintainable fix is to reuse the existing replay-policy flags and turn validators in the Mistral provider, not add a new config knob or side-car proxy.
