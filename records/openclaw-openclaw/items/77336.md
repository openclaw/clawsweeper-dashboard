## Summary

Keep open: current main has strict-turn replay helpers for OpenAI-compatible backends, but the bundled Mistral provider still owns a replay policy that only enables strict9 tool-call IDs, so the Mistral-family strict alternation request remains unresolved.

## What This Changes

Request built-in handling for Mistral/SGLang strict role alternation by merging consecutive same-role messages before provider sends.

## Best Possible Solution

Extend the Mistral provider replay policy to opt into the existing strict turn validators while preserving strict9 tool-call IDs, then add regression coverage and a changelog entry.

## Reproduction Assessment

Yes, source-reproducible: resolveTranscriptPolicy uses the provider policy when present, and Mistral's policy omits the turn-validation flags that merge consecutive same-role turns. I did not run a live SGLang/Mistral server.

## Solution Assessment

Yes: the narrow maintainable path is to reuse the existing provider replay-policy flags and validators for Mistral rather than adding a new config knob or side-car proxy behavior.
