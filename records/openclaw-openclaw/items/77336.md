## Summary

Keep open: current main already has strict turn validators for unowned OpenAI-compatible routes such as SGLang, but the bundled Mistral provider owns its replay policy and still only opts into strict9 tool-call IDs. The reporter's side-car workaround and follow-up comment both point to the same narrow repair: reuse the existing turn validators in the Mistral provider policy.

## What This Changes

Request built-in handling for strict Mistral/SGLang role alternation so consecutive same-role transcript turns are merged before provider replay.

## Best Possible Solution

Have the bundled Mistral provider opt into the existing strict turn validators while preserving strict9 tool-call IDs, then document the updated transcript hygiene behavior.

## Reproduction Assessment

Yes, source-reproducible: current main's Mistral provider policy bypasses the strict OpenAI-compatible fallback and does not enable the validators that merge consecutive user or assistant turns. I did not run a live Mistral/SGLang HTTP 400 reproduction.

## Solution Assessment

Yes: the narrow maintainable fix is to reuse the existing provider replay-policy flags and validators in the Mistral plugin, not add a new config knob or external proxy path.
