## Summary

Keep open: current main has the strict-turn merge machinery and applies it to unowned/custom OpenAI-compatible routes such as SGLang, but the bundled Mistral provider still overrides replay policy with only strict9 tool-call ID sanitization, leaving the Mistral-family alternation failure unresolved.

## What This Changes

Request built-in replay handling that merges consecutive same-role turns for strict Mistral/SGLang chat-template backends instead of requiring an external proxy.

## Best Possible Solution

Have the Mistral provider compose the existing strict OpenAI-compatible turn validators while retaining strict9 tool-call ID sanitization; leave the SGLang/custom path on the generic fallback.

## Reproduction Assessment

Yes, from source inspection: the Mistral provider policy bypasses the generic strict OpenAI-compatible fallback and does not enable the validators that merge consecutive user/assistant turns. I did not run a live Mistral/SGLang server, so this is source-reproducible rather than locally reproduced.

## Solution Assessment

Yes: the narrow maintainable solution is to reuse the existing provider replay-policy flags and validators in the Mistral plugin rather than adding a new config knob or requiring a side-car proxy.
