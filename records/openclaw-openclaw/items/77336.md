## Summary

Keep open: current main still leaves the bundled Mistral provider on a strict9-only replay policy, so it does not opt into the existing same-role turn validators; SGLang/custom OpenAI-compatible routes already use fallback validators, but the Mistral-family path remains unresolved.

## What This Changes

Request built-in handling that merges consecutive same-role transcript messages for Mistral/SGLang backends that reject non-alternating chat roles.

## Best Possible Solution

Extend the Mistral plugin replay policy to enable the existing assistant/user turn validators while preserving strict9 tool-call IDs, then add focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes, source-reproducible: Mistral's template requires alternating user/assistant roles, and current main's Mistral replay policy does not enable the validators that would merge consecutive same-role messages. I did not run a live Mistral/SGLang server.

## Solution Assessment

Yes: reusing the existing provider replay-policy flags is the narrowest maintainable fix and avoids adding a new config knob or requiring a side-car proxy.
