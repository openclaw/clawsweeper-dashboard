## Summary

Keep open: current main still leaves the bundled Mistral provider on a replay policy that only preserves strict9 tool-call IDs, so it bypasses the existing strict OpenAI-compatible turn validators that would merge consecutive user/assistant turns. The prior ClawSweeper comment and the reporter's follow-up both point to the same narrow fix direction.

## What This Changes

Request built-in handling for Mistral/SGLang strict role alternation by merging consecutive same-role replay messages before provider send.

## Best Possible Solution

Extend the Mistral provider replay policy to enable the existing strict turn validators while preserving strict9 tool-call IDs, then add focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes. The issue is source-reproducible on current main: Mistral's provider-owned replay policy omits the validators that merge consecutive same-role turns, while upstream Mistral templates reject those sequences; I did not run a live SGLang server.

## Solution Assessment

Yes. Reusing the existing replay-policy flags and validators in the Mistral plugin is narrower and more maintainable than adding a new config knob or requiring a side-car proxy.
