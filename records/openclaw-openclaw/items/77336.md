## Summary

Keep open: current main already covers the SGLang/custom strict OpenAI-compatible path, but the bundled Mistral provider owns its replay policy and still only opts into strict9 tool-call IDs, so the Mistral-family strict-alternation part of the request remains unresolved.

## What This Changes

Request built-in transcript handling that merges consecutive same-role turns before sending to Mistral/SGLang strict-alternation backends.

## Best Possible Solution

Extend the Mistral provider replay policy to use the existing strict turn validators while preserving strict9 tool-call IDs, then add regression coverage and a user-facing changelog entry.

## Reproduction Assessment

Yes, from source inspection: the Mistral provider policy bypasses the generic strict OpenAI-compatible fallback and does not enable the validators that merge consecutive user/assistant turns. I did not run a live Mistral/SGLang server, so this is source-reproducible rather than locally reproduced.

## Solution Assessment

Yes: the narrow maintainable fix is to reuse the existing provider replay-policy flags and validators rather than adding a new config knob or side-car behavior.
