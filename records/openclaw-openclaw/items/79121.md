## Summary

Keep open: the report matches a current voice-call code path where Twilio streaming activity can proceed without `answeredAt`, while the stale reaper still ends any non-terminal call older than the configured threshold.

## What This Changes

The issue requests that active Twilio streaming conversation calls not be terminated by the voice-call stale-call reaper after about 120 seconds.

## Best Possible Solution

Fix the voice-call plugin so accepted or active Twilio streaming calls are treated as non-stale while preserving cleanup for truly pre-answer or stuck calls, with tests and docs/defaults aligned to the final behavior.

## Reproduction Assessment

Yes from source inspection, though not from a live phone call in this review. A Twilio inbound streaming call can receive media and transcripts while `answeredAt` remains unset, and the 120-second default reaper will end it if it is still non-terminal.

## Solution Assessment

Yes: the narrow maintainable fix belongs in the voice-call owner path, either by marking accepted media-stream calls as answered/active for reaper purposes or by teaching the reaper to ignore genuinely active streaming calls.
