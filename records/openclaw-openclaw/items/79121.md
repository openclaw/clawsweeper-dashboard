Keep open: current main still has the source-provable voice-call stale reaper bug, the focused fix PR was closed unmerged, and no open replacement PR was found.

## What This Changes

Fix Twilio streaming voice-call conversations so active `speaking` or `listening` calls are not ended by the stale-call reaper after about 120 seconds.

## Best Possible Solution

Land or recreate a focused voice-call plugin fix that treats active streaming conversation states as non-stale while preserving cleanup for truly stuck pre-answer calls, with regression coverage and docs/config alignment if behavior text changes.

## Reproduction Assessment

Yes, from source inspection: construct a call older than `staleCallReaperSeconds` in `speaking` or `listening` with no `answeredAt`, and current main will pass the reaper guard and call `endCall`. The reporter's redacted Twilio logs show the same state and age in a real call.

## Solution Assessment

Yes: the narrow maintainable fix belongs in `extensions/voice-call/src/webhook/stale-call-reaper.ts`, either by skipping active conversation states or equivalently marking accepted streaming calls active for stale-reaper purposes. The prior closed PR's direction is suitable, but it still needs to be landed or recreated on current main.
