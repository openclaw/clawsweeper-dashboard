## Repair Work Prompt

Fix the open voice-call stale-reaper bug at https://github.com/openclaw/openclaw/issues/79121. Current main reaps any non-terminal call older than `staleCallReaperSeconds` unless `answeredAt` is set, but Twilio streaming conversations can enter `speaking`/`listening` and process transcripts/responses before `call.answered` sets `answeredAt`. Implement a narrow owner-local fix in the voice-call plugin so active conversation states are not treated as stale, while preserving cleanup for genuinely stuck pre-answer/setup calls. Use the closed unmerged PR at https://github.com/openclaw/openclaw/pull/79124 as useful source work, but re-check the current code and avoid broad disabling or timeout-only changes. Add focused regression coverage for old `speaking` and `listening` calls with unset `answeredAt`; update docs/default wording if the stale-reaper contract is touched, and add a changelog entry if this is release-facing. Do not change unrelated voice approval, Telnyx auto-response, or generic channel behavior.
