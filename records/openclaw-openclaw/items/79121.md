## Summary

Keep open: the report matches a current owner-local voice-call code path where Twilio streaming activity can proceed without `answeredAt`, while the default stale reaper still ends any non-terminal call older than 120 seconds.

## What This Changes

The issue asks OpenClaw to stop the voice-call stale reaper from ending active Twilio inbound streaming conversations after about 120 seconds.

## Best Possible Solution

Fix the voice-call plugin so accepted/active Twilio streaming calls are treated as non-stale while preserving cleanup for truly pre-answer or stuck calls, with docs/defaults aligned to the final behavior.

## Reproduction Assessment

Yes from source inspection, without a live phone call: a Twilio inbound streaming call can receive media and transcripts while `answeredAt` remains unset, and the 120-second default reaper will end it if it is still non-terminal.

## Solution Assessment

Yes: the narrow maintainable fix belongs in the voice-call owner path, either by marking accepted media-stream calls as answered/active for reaper purposes or by teaching the reaper to ignore active streaming calls.
