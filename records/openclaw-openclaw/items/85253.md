## What This Changes

The PR refactors GatewayClient connect-challenge handling so local connect assembly failures are marked, reported through onConnectError, and surfaced by callGateway without waiting for the wrapper timeout.

## Best Possible Solution

Land the focused Gateway client/call fix after normal CI, preserving existing server rejection, socket close, and challenge-timeout behavior while making deterministic local assembly failures terminal.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to send connect.challenge to callGateway with malformed device identity PEM; current main throws during local signing under handleMessage's broad catch and the wrapper has no connect-error settlement path.

## Solution Assessment

Yes. The patch keeps the repair inside the Gateway client/call boundary, marks only internally generated assembly failures, and avoids changing public SDK callbacks or normal server rejection paths.
