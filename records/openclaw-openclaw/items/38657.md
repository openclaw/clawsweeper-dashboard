## Summary

Keep open. Current main still preserves existing generated provider credentials/endpoints in merge mode, while the provider onboarding helper has no distinct signal for a deliberate same-provider edit, so the broader Onboard persistence behavior remains unresolved.

## What This Changes

The issue asks for Onboard provider edits to overwrite existing provider configuration or clearly report that an update path is needed.

## Best Possible Solution

Define an explicit user-initiated provider update path that updates source config and generated per-agent state consistently while preserving documented merge-mode and SecretRef invariants.

## Reproduction Assessment

Yes, by source inspection: start with merge mode, a same-provider config entry, and an existing generated provider entry with non-empty `apiKey`/`baseUrl`; current planning reaches the preservation helper and tests assert the existing values survive. I did not run the live UI because this review was read-only.

## Solution Assessment

Unclear for a concrete patch because none is proposed here. The maintainable direction is an explicit deliberate-update path for Onboard/configure, not a global reversal of merge-mode precedence.
