## Summary

Keep open. The narrow stale `baseUrl` precedence fix from https://github.com/openclaw/openclaw/pull/39103 does not settle the broader Onboard overwrite path, and current main still has merge-mode preservation behavior that can conflict with a deliberate edit to an existing provider.

## What This Changes

The issue requests that Onboard edits to an existing provider persist across canonical config, generated `models.json`, and auth profile state instead of silently retaining old provider values.

## Best Possible Solution

Define an explicit user-initiated Onboard update contract that updates canonical config, generated `models.json`, and auth profiles consistently while preserving normal merge-mode and SecretRef-safe persistence behavior.

Yes. This is source-reproducible: current main routes Onboard provider saves into merge-mode generation where existing same-provider generated state can still be preserved, and tests assert that preservation behavior.

## Solution Assessment

Unclear because no patch is proposed here. The safer direction is an explicit Onboard update path or intent marker, not a global reversal of documented merge-mode precedence.
