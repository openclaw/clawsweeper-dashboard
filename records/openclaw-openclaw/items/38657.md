## Summary

Keep open. Current main still preserves existing generated provider credentials and same-surface endpoints in merge mode, while the shared Onboard provider path has no explicit same-provider overwrite intent; the narrower stale baseUrl bug was handled separately, but the broader Onboard persistence path remains unresolved.

## What This Changes

The issue asks for the Onboard UI to persist deliberate edits to existing provider API keys and base URLs instead of silently keeping older configuration values.

## Best Possible Solution

Add an explicit deliberate provider-update path from Onboard/configure that updates canonical config and generated per-agent state consistently while preserving documented merge-mode and SecretRef invariants.

## Reproduction Assessment

Yes, by source inspection. A same-provider Onboard save in merge mode with existing non-empty generated `apiKey` or same-surface `baseUrl` reaches the preservation helper, and current tests assert those existing values survive.

## Solution Assessment

Unclear because there is no active patch here. The maintainable direction is an explicit user-initiated update path or clear UI refusal, not a blanket reversal of documented merge-mode precedence.
