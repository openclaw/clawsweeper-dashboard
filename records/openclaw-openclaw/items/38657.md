## Summary

Keep open. Current main still preserves existing same-provider generated values in merge mode, and the discussion explicitly left the broader Onboard overwrite/persistence problem open after the narrower stale baseUrl fix.

## What This Changes

The issue asks Onboard to overwrite existing provider configuration values, or clearly expose an update/error path, when a user edits an already configured provider.

## Best Possible Solution

Define explicit user-initiated Onboard update semantics that update canonical config and generated agent state consistently while preserving merge-mode and SecretRef behavior during normal regeneration.

## Reproduction Assessment

## Solution Assessment

Unclear because no accepted patch is proposed here. The safer direction is an explicit Onboard update path, not a global reversal of merge-mode precedence.
