## Summary

Keep open. Current main still has a source-reproducible same-provider persistence gap: generated provider state can keep winning in merge mode, and the shared Onboard provider helper has no explicit overwrite/update intent. The narrower stale `baseUrl` fix in https://github.com/openclaw/openclaw/pull/39103 does not close this broader Onboard persistence issue.

## What This Changes

The issue asks for Onboard provider edits to overwrite or clearly reject existing provider configuration state instead of silently leaving old values in config files.

## Best Possible Solution

Define an explicit Onboard/configure update path that marks deliberate same-provider edits, updates canonical config and generated agent state consistently, and preserves merge-mode plus SecretRef invariants by default.

## Reproduction Assessment

Yes, from source inspection rather than a live UI run. A same-provider Onboard save can still flow through merge-mode generation where existing generated apiKey or same-surface baseUrl values are preserved, and current tests assert that behavior.

## Solution Assessment

Unclear because no patch is proposed here. The safest direction is an explicit user-initiated overwrite/update path, not a blanket reversal of documented merge-mode precedence.
