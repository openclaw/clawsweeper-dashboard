## Summary

Keep open. Current main still preserves existing generated provider credentials/endpoints in merge mode, and the shared Onboard provider path does not carry explicit same-provider overwrite intent, so the broader Onboard persistence bug remains unresolved.

## What This Changes

The issue requests that Onboard provider edits overwrite or clearly handle existing provider credentials/endpoints across OpenClaw config layers.

## Best Possible Solution

Add an explicit deliberate provider-update path from Onboard/configure that updates canonical config and generated per-agent state consistently while preserving merge-mode and SecretRef invariants.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a same-provider Onboard save in merge mode reaches `mergeWithExistingProviderSecrets`, where current code and tests preserve existing non-empty generated `apiKey` and same-surface `baseUrl`.

## Solution Assessment

Unclear because no current patch is proposed. A blanket reversal of merge precedence would conflict with documented behavior; the safer solution is an explicit user-initiated overwrite/update intent from the Onboard path.
