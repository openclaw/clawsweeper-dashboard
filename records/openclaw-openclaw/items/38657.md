## Summary

Keep this open. Current main still has the same unresolved contract gap: ordinary merge-mode regeneration intentionally preserves existing generated provider `apiKey` and `baseUrl`, while the Onboard/configure path does not expose a distinct deliberate-overwrite signal for same-provider edits. The related discussion and PR history show that narrower stale `baseUrl` cases were handled separately, but this broader persistence/update behavior remains unresolved.

## What This Changes

Onboard/configure should persist deliberate updates to an existing provider configuration across source config, generated per-agent `models.json`, and auth-profile state, or clearly report that no update occurred.

## Best Possible Solution

Define a small explicit provider-update contract for user-initiated Onboard/configure edits. That path should update source config and generated state consistently, distinguish deliberate edits from normal merge-mode regeneration, preserve SecretRef markers, and include regression coverage for Moonshot `.ai` to `.cn` base URL changes plus plaintext and SecretRef key replacement.
