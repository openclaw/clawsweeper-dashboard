## Summary

Keep open. The narrow stale `baseUrl` merge-mode bug was handled separately, but current main still lacks an explicit Onboard overwrite/update path and still preserves same-provider generated values under the documented merge contract.

## What This Changes

The issue asks for Onboard edits to an existing provider to persist updated provider values instead of silently retaining older config or generated state.

## Best Possible Solution

Define an explicit Onboard update path that marks deliberate same-provider edits, updates canonical config and generated agent state consistently, and preserves merge-mode plus SecretRef defaults during normal regeneration.

## Reproduction Assessment

Yes, from source inspection rather than a live UI run. Current main still routes shared Onboard provider saves through merge-mode generation where existing generated apiKey and same-surface baseUrl values can be preserved, and current tests assert that behavior.

## Solution Assessment

Unclear because no accepted patch is proposed here. The maintainable fix should add explicit user-initiated overwrite/update semantics rather than reversing merge-mode precedence globally.
