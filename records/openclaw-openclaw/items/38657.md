## Summary

Keep open. Current main still has a source-level path where onboarding/config writes can be followed by merge-mode regeneration that preserves existing provider `apiKey`/`baseUrl` from agent `models.json`, and the issue discussion explicitly kept this broader Onboard persistence contract open after the narrower stale `baseUrl` fix.

## What This Changes

Requests that Onboard update or clearly reject edits to an existing provider configuration instead of silently leaving old provider state in config/generated agent files.

## Best Possible Solution

Define explicit user-initiated Onboard update semantics that update canonical config and generated agent state consistently while preserving normal merge-mode and SecretRef behavior.

## Reproduction Assessment

Yes, source-reproducible rather than live UI-reproduced. Current main routes Onboard provider saves into merge-mode generation where existing same-provider generated `apiKey`/`baseUrl` values can be preserved, and current tests assert that behavior.

## Solution Assessment

Unclear because no accepted patch is proposed here. The safer direction is an explicit Onboard update path, not a global reversal of documented merge-mode precedence.
