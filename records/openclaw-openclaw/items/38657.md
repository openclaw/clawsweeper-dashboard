## Summary

Keep open. The narrower stale `baseUrl` merge-mode bug was handled in the related closed thread, but current main and the latest release still document and implement same-provider merge preservation that can conflict with a user-initiated Onboard overwrite path.

## What This Changes

The issue asks for Onboard provider edits to update existing provider configuration consistently across canonical config, generated agent `models.json`, and auth profile state, or to clearly tell users when an update is not applied.

## Best Possible Solution

Define an explicit user-initiated Onboard update contract that updates canonical config and generated agent state consistently while preserving normal merge-mode and SecretRef-safe persistence behavior.

Unclear because no accepted patch is proposed here. The safer direction is an explicit Onboard update path or intent marker, not a global reversal of documented merge-mode precedence.
