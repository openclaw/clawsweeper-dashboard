## Summary

Keep open. Current main still preserves existing generated provider credentials/endpoints in merge mode, while the shared Onboard path has no explicit same-provider update intent, so the broader persistence bug remains unresolved.

## What This Changes

The issue asks Onboard provider setup to persist same-provider API key and base URL edits instead of silently retaining older generated provider state.

## Best Possible Solution

Define an explicit user-initiated provider update path that updates source config and generated per-agent state consistently while preserving merge-mode and SecretRef invariants.

## Reproduction Assessment

Yes, by source inspection: start with merge mode, a same-provider config entry, and an existing generated provider entry with non-empty `apiKey` or same-surface `baseUrl`; current planning reaches the preservation helper and tests assert the existing values survive.

## Solution Assessment

Unclear for a concrete patch because none is proposed here. The maintainable direction is an explicit deliberate-update path for Onboard/configure, not a blanket reversal of documented merge-mode precedence.
