## Summary

Keep open. Current main still preserves existing generated provider credentials/endpoints in merge mode, and the shared Onboard provider path has no explicit signal that a same-provider save should overwrite generated state.

## What This Changes

The issue asks for Onboard provider edits to persist over existing generated provider credentials/endpoints instead of silently keeping stale values.

## Best Possible Solution

Add an explicit deliberate provider-update path from Onboard/configure that updates canonical config and generated per-agent state consistently while preserving merge-mode and SecretRef invariants.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: same-provider Onboard save in merge mode reaches mergeWithExistingProviderSecrets, and current tests assert existing generated apiKey/baseUrl values survive.

## Solution Assessment

Unclear because no patch is proposed here. The maintainable direction is an explicit user-initiated update path, not a blanket reversal of documented merge-mode precedence.
