## Summary

Keep open. The reported fake-IP failure path is source-reproducible on current main, but this PR is not ready to merge because it changes a default SSRF trust policy without updated tests, changelog, or explicit maintainer/security acceptance.

## What This Changes

The PR changes trusted and self-hosted web-tool SSRF policies to allow RFC2544 and IPv6 ULA fake-IP DNS ranges by default.

## Best Possible Solution

Land a tested policy update only after maintainers accept the trusted-helper SSRF exception, with `web-guarded-fetch` regression coverage and a single-line Unreleased changelog entry.

## Reproduction Assessment

Yes, from source inspection. Current main sends trusted web-search provider calls through `withTrustedWebToolsEndpoint` with `policy: {}`, while the SSRF guard only exempts RFC2544 or IPv6 ULA DNS answers when the active policy has the corresponding flag.

## Solution Assessment

No, not yet. The direction may be correct, but the PR is not the best mergeable form until tests and changelog are updated and maintainers explicitly accept the default SSRF policy widening.
