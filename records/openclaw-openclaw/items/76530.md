## Summary

Keep open. Current main still has the reported trusted web-tool fake-IP gap, and the PR direction is plausible, but the patch changes a default SSRF policy without maintainer/security acceptance and leaves required test and changelog work incomplete.

## What This Changes

The PR changes web-tool guarded fetch policies so trusted endpoints allow RFC2544 and IPv6 ULA fake-IP DNS ranges, and self-hosted endpoints also include the IPv6 ULA allowance.

## Best Possible Solution

Land a tested policy update only after maintainers accept the trusted-helper SSRF exception; if that default is too broad, use a narrower hostname or configuration-based policy path instead.

Yes, source inspection gives a high-confidence reproduction path: current main passes `{}` for trusted web-tool endpoints, and the SSRF resolver only exempts RFC2544 or IPv6 ULA DNS answers when the active policy has the matching flag.

## Solution Assessment

No, not as submitted. The policy direction may be right, but this patch still needs maintainer/security acceptance, updated policy expectations, regression coverage, and a changelog entry before it is the best mergeable fix.
