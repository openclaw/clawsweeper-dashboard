## Summary

Keep this PR open for maintainer/security review: it contains useful QQBot SQLite cleanup, but the current patch still has a blocking credential-boundary issue and lacks real doctor/setup migration proof.

## What This Changes

The PR removes QQBot runtime imports of legacy JSON/JSONL cache files, adds a setup/doctor migration for legacy QQBot credential backup JSON into plugin SQLite KV, and updates startup preflight, docs, and tests around that migration.

## Best Possible Solution

Keep the runtime SQLite-only cleanup, but constrain credential-backup migration and preflight detection to the active state root unless maintainers explicitly approve a separate cross-root recovery mode with tests and docs.

## Reproduction Assessment

Yes, source-reproducible: the PR detector adds HOME/USERPROFILE as a credential-backup root even when the active stateDir is different, and QQBot startup restores imported backup values into config.

## Solution Assessment

No. Moving legacy import out of runtime is the right direction, but automatic OS-home import into arbitrary active state roots is not the safest fix; it should stay active-state scoped or become an explicit maintainer-approved recovery mode.
