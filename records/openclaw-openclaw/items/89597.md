## Summary

Keep open. The PR is still useful but not merge-ready because it automatically imports HOME/USERPROFILE QQBot credential backups into the active plugin KV state, which can re-cross the state-root credential isolation boundary, and the PR still lacks real doctor/setup migration proof.

## What This Changes

The PR removes QQBot runtime JSON/JSONL cache imports, adds a setup/doctor migration for QQBot credential backup JSON files into plugin SQLite KV, and updates startup preflight, docs, and tests.

## Best Possible Solution

Keep the runtime SQLite-only cleanup, but migrate QQBot credential backups only from the active state root unless maintainers explicitly approve a documented cross-root recovery mode with security-focused tests.

## Reproduction Assessment

Yes, source-reproducible: set a custom OPENCLAW_STATE_DIR, place a valid QQBot credential-backup file under HOME/.openclaw/qqbot/data, and the PR's detector can import it into the active plugin KV state. I did not execute the migration because this review is read-only.

## Solution Assessment

No. Moving legacy JSON import out of runtime is the right direction, but automatic OS-home import into arbitrary active state roots is not the safest fix; it should stay scoped to the active root or become an explicit recovery mode.
