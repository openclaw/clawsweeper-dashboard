## Summary

Keep open: this PR has the right runtime direction, but it is not merge-ready because the new doctor migration can import home-global QQBot credential backups into a different active state root, weakening the isolation fixed by the prior merged security change.

## What This Changes

This PR moves QQBot credential backup JSON imports from runtime reads into setup/doctor plugin-state migration, removes legacy cache imports, and adds startup preflight detection plus docs and tests.

## Best Possible Solution

Keep the runtime SQLite-only cleanup, but constrain credential-backup migration and preflight detection to the active state root unless maintainers explicitly approve a separate cross-root recovery mode.

## Reproduction Assessment

Yes, source-reproducible: the PR detector adds HOME/USERPROFILE as a QQBot credential-backup root even when active stateDir is different, and QQBot startup restores appId/clientSecret from the imported plugin KV backup. I did not execute the migration because this review is read-only.

## Solution Assessment

No. Moving legacy import out of runtime is the right direction, but automatic OS-home import into arbitrary active state roots is not the safest fix; the migration should stay scoped to the active state root or become an explicit recovery mode.
