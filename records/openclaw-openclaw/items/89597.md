## Summary

## What This Changes

This PR removes QQBot runtime legacy JSON/JSONL imports, adds setup/doctor migration for QQBot credential-backup JSON files into plugin SQLite KV, and updates preflight/docs/tests around that migration.

## Best Possible Solution

Keep the runtime SQLite-only cleanup, migrate only active-state QQBot credential backups through doctor/setup, and require explicit maintainer-approved recovery if HOME-global credential recovery should exist.

## Reproduction Assessment

Yes, source-reproducible: on PR head, a legacy HOME credential backup is detected while a different OPENCLAW_STATE_DIR is active; migration then writes that secret into the active plugin KV and startup can restore it into config.

## Solution Assessment

No. Moving legacy import out of runtime is the right direction, but automatic HOME-to-active-state import reopens the isolation boundary; the narrower fix is active-state migration only unless recovery is explicitly approved.
