## Summary

Keep open: this is a focused bug-fix PR whose central change is still useful on current main. Source inspection confirms current main creates manual cron task runs without marking the job active, while task maintenance treats cron tasks as backed only when `isCronJobActive(jobId)` is true after the grace window; the PR mirrors the existing scheduled-run contract and supplies real before/after proof.

## What This Changes

The PR marks prepared manual cron runs active, clears that marker in a `finally` block after completion, and adds regression tests for success and throwing manual runs.

## Best Possible Solution

Land this PR, or an equivalent patch, so manual cron runs share the existing scheduled-run active-job ownership contract; keep the linked issue open until the fix is merged.
