## Best Possible Solution

## Reproduction Assessment

Yes, at source level: a Telegram mirror update calls `publishSessionTranscriptUpdateByIdentity` with scoped fields, but the wrapper passes only `params.update` to a central publisher that emits only the update payload plus `sessionFile`. I did not run live Telegram or Discord proof.

## Solution Assessment

No. The refactor direction is plausible, but the best fix is to make the scoped publisher itself carry `agentId` and `sessionKey` into emitted transcript updates and cover that final event payload, not only the wrapper call.
