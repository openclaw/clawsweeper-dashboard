## Summary

Keep open: the live PR head is now a focused two-file patch, but it is incomplete because the Codex app-server still emits a no-op PreToolUse command for inactive selected PreToolUse, and Codex still runs configured command hooks before tools. The predicate change is plausible, but the claimed default relay subprocess reduction is not achieved yet and the PR needs after-fix real behavior proof.

## What This Changes

The PR changes native hook relay loop-detection gating so Codex PreToolUse local work is active only when `tools.loopDetection.enabled` is explicitly true, and updates relay registry tests for unset, disabled, and enabled cases.

## Best Possible Solution

Omit the default Codex PreToolUse command hook entirely when there is no `before_tool_call` policy and no explicit loop detection, while preserving explicit loop detection, policy relays, and stale/unknown relay fail-closed behavior.

## Reproduction Assessment

Yes, source-reproducible: current main treats unset loop detection as active relay work, and the unchanged Codex app-server builder installs a no-op PreToolUse command when that work is inactive. I did not run a live harness probe in this read-only review.

## Solution Assessment

No. The core predicate is the right invariant, but the Codex app-server config builder still converts inactive selected PreToolUse into a no-op command hook, so the fix must update that higher-level hot path too.
