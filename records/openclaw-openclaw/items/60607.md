## Summary

Keep open. ## What This Changes

The issue asks OpenClaw to notify the user when an agent turn completes with no replyable content instead of ending with `queuedFinal=false` and zero replies.

## Best Possible Solution

Keep this issue as the broad tracker until maintainers land or supersede https://github.com/openclaw/openclaw/pull/63025 with a scoped dispatch fallback that respects delivery suppression, message-tool mode, silentReply policy, routed-origin delivery, and already-delivered side effects.

## Reproduction Assessment

Yes, at source level: current main can return `undefined` from the runner after empty or post-filtered payloads, and dispatch can complete with zero queued final replies. I did not run a live Feishu websocket reproduction.

## Solution Assessment

No, not as a blanket fallback or universal `dispatch.requireReply` option. The maintainable fix is a scoped dispatch-level fallback that preserves intentional silence and explicit delivery-suppression policy.
