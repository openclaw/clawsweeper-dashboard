## Summary

Keep this PR open. The Windows EBUSY update problem is still present on current main, and this PR is a credible continuation of the earlier detached-updater work, but it has one small failure-reason regression plus an unmergeable/dirty branch state before it can land.

## What This Changes

The PR changes Windows npm-global Gateway updates to verify the staged package before exit, delegate the final locked package-tree swap and Scheduled Task restart to a detached helper, reconcile detached results through update status, and update docs, changelog, and tests.

## Best Possible Solution

Fix the detached swap failure-reason mapping, rebase or otherwise resolve the dirty branch, then land the staged-verify-plus-detached-swap design after focused Windows/update validation confirms the restart and status reconciliation semantics.

## Reproduction Assessment

Yes. #40540 gives concrete Windows npm-global EBUSY rename logs, and current main still performs the active package-root rename from the live Gateway update flow.

## Solution Assessment

Yes, mostly. Preserving staged package verification in the live process while delegating only the locked Windows swap and restart is narrower than moving the whole install into the detached helper; the failure-reason mapping should be fixed before merge.
