## Summary

Keep this PR open: current main still lacks the detached Windows `update.run` handoff, and the PR usefully carries forward #44614, but it is currently dirty/unmergeable and still over-claims #40540 as fixed even though the CLI `openclaw update` package path remains on the in-process swap flow.

## What This Changes

The PR adds a Windows detached helper for verified staged npm global update swaps from the Gateway `update.run` path, records detached results through restart sentinels, and updates docs, tests, and changelog text.

## Best Possible Solution

Land the detached verified-swap design for the Gateway `update.run` path after rebase and validation, and either extend the same handoff to the CLI package-update path or keep #40540 referenced rather than fixed until the CLI reproduction is covered.

## Reproduction Assessment

Yes. #40540 provides a concrete Windows npm global self-update EBUSY reproduction, and current main still reaches the live staged package-root swap; the PR's tests also give a focused way to exercise the new detached `update.run` response path.

## Solution Assessment

Partly. The PR is the narrow maintainable shape for Control UI updates because it preserves staged verification and delegates only the locked Windows swap, but it is not yet the best complete fix for the linked CLI report unless the CLI path is covered or the issue reference is narrowed.
