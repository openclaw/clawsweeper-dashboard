## What This Changes

The PR changes Windows npm-global Gateway updates to verify the staged package before exit, then delegate the final locked package-tree swap and Scheduled Task restart to a detached helper, with docs, changelog, sentinel, and tests updated.

## Best Possible Solution

Address the small failure-reason mapping gap, rebase or otherwise resolve the dirty branch state, then land this staged-verify-plus-detached-swap design if focused Windows/update CI and maintainer review agree with the restart semantics.

## Reproduction Assessment

Yes. #40540 provides a concrete Windows npm-global self-update EBUSY reproduction with logs, and current main still performs the staged package-root rename from the live update flow.

## Solution Assessment

Yes, mostly. Preserving staged verification in the live process while delegating only the locked Windows swap is narrower and safer than moving the whole npm install into the detached helper; the remaining cleanup is to keep failure reasons consistent.
