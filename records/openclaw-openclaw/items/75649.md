## Summary

Keep this PR open. The detached Windows handoff looks like useful work for the Control UI `update.run` path, but the current diff still claims to fix the separate CLI `openclaw update` EBUSY report even though the CLI package-update path does not pass the new deferred-swap hook.

## What This Changes

The PR adds a Windows detached helper for verified staged npm global update swaps from the Gateway `update.run` path, reconciles detached results through restart sentinels, and updates tests, docs, and changelog text.

## Best Possible Solution

Land the detached verified-swap design for Gateway `update.run` after rebase and validation, and either extend the same Windows handoff to the CLI package-update path or change the changelog from a closing `Fixes #40540` claim to a non-closing reference.

## Reproduction Assessment

Yes. #40540 gives a concrete Windows `openclaw update` EBUSY reproduction, and source inspection shows the CLI path still reaches the live staged package-root swap while this PR detaches only the Gateway `update.run` path.

## Solution Assessment

Partly. The proposed design is a maintainable fix for Control UI updates because it keeps staged verification before delegating the locked swap, but it is not the best complete fix for the claimed CLI issue unless the CLI path is also covered or the claim is narrowed.
