## Summary

Keep open: current main still loses `data.details` for numeric ACP/JSON-RPC `RequestError` values, and the linked open PR should be reviewed or landed before this issue closes.

## What This Changes

The issue reports that ACP numeric JSON-RPC errors with `data.details` are normalized to a generic `Internal error` message, hiding the actionable diagnostic.

## Best Possible Solution

Land or close the linked fix PR after review, preserving redacted `data.details` from numeric ACP/JSON-RPC errors with focused runtime and formatter regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is to pass a `RequestError`-shaped `Error` with `code: -32603`, `message: "Internal error"`, and `data.details` into `toAcpRuntimeError`, then render it with `formatAcpRuntimeErrorText`; current main prints only the generic message.

## Solution Assessment

Yes. The narrow maintainable fix is in ACP runtime error normalization and formatting, using the existing redaction path rather than adding Claude-specific or option-handling behavior.
