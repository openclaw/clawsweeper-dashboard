## Summary

Keep open: current main has useful updater, auto-update, backup, health, and plugin-convergence primitives, but it still lacks the requested opt-in safe-update workflow with update-coupled full backups, retention policy, real channel/agent smoke checks, and controlled rollback plus state restore. The discussion adds real rollout evidence and an external operator workaround, and the related open issues/PRs are partials rather than a single superseding implementation.

## What This Changes

The issue requests an optional safe-update workflow that snapshots OpenClaw state before update, runs post-update health and smoke checks, and rolls back or restores state if checks fail.

## Best Possible Solution

Make a maintainer decision on whether safe update belongs in core; if accepted, design one documented opt-in updater mode that composes backup/verify, status/deep health, channel readiness, real-agent smoke, rollback/restore, retention, and secret-handling policy.
