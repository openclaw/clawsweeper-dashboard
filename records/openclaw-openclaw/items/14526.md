## Summary

Current main and v2026.6.5 provide substantial adjacent safeguards, but the requested update-coupled full-state backup, retention, channel or real-agent smoke verification, compatible package rollback, and state restoration remain missing; the narrower related issues do not supersede this broader recovery contract.

## What This Changes

Add an opt-in safer update workflow with pre-update state backup, post-update runtime checks, and rollback or restoration on failure.

## Best Possible Solution

Design a version-aware update transaction around the existing updater, backup, doctor, status, and restart primitives, with explicit snapshot scope and retention, side-effect-safe smoke levels, downgrade compatibility checks, and an operator-authorized restoration boundary.
