## Summary

Keep open: current main and latest release have useful updater, backup, doctor, plugin convergence, and gateway health primitives, but not the requested opt-in safe-update transaction with update-coupled full-state backup, retention, real channel/agent smoke checks, and automatic package-plus-state rollback/restore.

## What This Changes

The issue asks for an opt-in safe self-update workflow that snapshots OpenClaw state before updating, runs post-update health and smoke checks, and can roll back or restore state on failure.

## Best Possible Solution

Define a maintainer-approved core updater transaction, or explicitly split/reject that scope; if accepted, it should orchestrate existing backup, doctor, health, plugin, restart, and package rollback primitives under one security-reviewed recovery policy.

## Reproduction Assessment

Not applicable: this is a new safe-update workflow proposal rather than a deterministic current-main bug. The remaining capability gap is directly verifiable from the current CLI, updater source, backup command, docs, and latest release tag.

## Solution Assessment

Yes as a product direction, but not as an implementation-ready one-shot patch. The maintainable path is core updater orchestration over existing primitives after maintainers define rollback, retention, secrets, and smoke-check policy.
