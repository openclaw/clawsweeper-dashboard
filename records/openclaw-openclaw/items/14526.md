## Summary

Keep open: current main has useful updater, backup, doctor, gateway health, plugin convergence, and manual rollback pieces, but it still does not implement the requested opt-in safe-update transaction with update-coupled full-state backup, retention, real channel/agent smoke checks, and automatic package-plus-state rollback/restore.

## What This Changes

The issue requests an optional safer self-update workflow with pre-update snapshots, post-update health checks, retention policy, and rollback/restore if verification fails.

## Best Possible Solution

Define one core updater transaction that creates and verifies a permission-hardened snapshot, records the prior package and schema boundary, runs non-destructive gateway/channel/agent checks, and restores package plus compatible state only under an explicit recovery policy.

## Security Review

Status: needs_attention

Summary: The issue is security-sensitive because a safe-update workflow would package, retain, and potentially restore private OpenClaw state.

Concerns:

- **[high] Define secure snapshot storage and retention:**
  - body: Update-integrated archives can include credentials, auth profiles, channel identifiers, sessions, logs, and workspace files, so the design needs owner-only permissions, safe destination rules, retention/deletion policy, and clear operator visibility.
  - confidence: 0.96
- **[high] Prevent unsafe automatic state restoration:**
  - body: Rollback must avoid replacing newer valid state or restoring a schema that is incompatible with the downgraded binary; package and data recovery need one version-aware policy.
  - confidence: 0.94
- **[medium] Keep smoke checks non-destructive:**
  - body: Channel and agent verification can send messages, consume tools or model quota, leak private context, or mutate sessions unless the checks are isolated and explicitly authorized.
  - confidence: 0.9
