## Summary

Keep open. Current main has updater, manual backup, health/status, and manual rollback primitives, plus related narrower items, but it still lacks the requested opt-in safe-update workflow with update-tied backup, retention, channel/agent smoke checks, and automatic rollback plus state restore.

## What This Changes

Request an opt-in safe-update workflow that snapshots OpenClaw state before updating, verifies gateway/channel/agent health afterward, and rolls back or restores state when checks fail.

## Best Possible Solution

Make a maintainer scope decision for a first-class safe-update mode; if accepted, implement an opt-in orchestrator around existing update, backup, health/status, channel/agent smoke, retention, rollback, and restore primitives with explicit secret and downgrade guarantees.

## Reproduction Assessment

Not applicable: this is a feature proposal rather than a current-main bug. Source and docs inspection on current main show adjacent primitives exist, but no mode ties backup, retention, smoke checks, rollback, and restore into the updater.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. The narrow maintainable direction is either a designed core updater mode that reuses existing primitives or an explicit split across backup-restore, rollback, smoke-check, retention, and Control UI follow-ups.
