## Summary

Keep open: current main and v2026.6.5 still provide adjacent updater, auto-update, backup, health, restart verification, plugin convergence, and manual rollback primitives, but not the requested integrated opt-in safe-update workflow with update-coupled backups, retention, real channel/agent smoke checks, automatic rollback, and state restore.

## What This Changes

The issue requests an opt-in safe-update workflow that snapshots state before update, runs post-update health and smoke checks, and can roll back or restore state on failure.

## Best Possible Solution

Make a maintainer product/security decision on whether safe update belongs in core; if accepted, design one documented opt-in updater mode that composes backup/verify, deep status, channel readiness, real-agent smoke, rollback/restore, retention, and secret-handling policy.

## Risks / Open Questions

- The optional update-break reporting idea overlaps telemetry and privacy policy and should not be folded into an automated fix without maintainer approval.
