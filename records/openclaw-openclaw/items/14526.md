## Summary

Keep open: current main and v2026.6.5 have adjacent updater, auto-update, backup, status/health, restart verification, and plugin-smoke primitives, but they still do not provide the requested opt-in safe-update workflow with update-coupled snapshots, retention, real channel/agent smoke checks, automatic rollback, and state restore.

## What This Changes

The issue proposes an optional safe-update workflow that creates a pre-update backup, applies the update, runs deeper post-update health checks, and rolls back or restores state if verification fails.

## Best Possible Solution

Make a maintainer product/security decision on whether safe update belongs in core; if accepted, design one documented opt-in updater mode that composes backup/verify, deep status, channel readiness, real-agent smoke, rollback/restore, retention, and secret-handling policy.

## Risks / Open Questions

- The optional update-break reporting idea overlaps telemetry and privacy policy and should not be folded into an automated fix without maintainer approval.
