## Summary

Keep open. Current main has updater, auto-update config, manual backup/verify, health/status, and manual rollback guidance, but it still lacks the requested opt-in safe-update workflow tying pre-update snapshots, retention, channel/agent smoke checks, automatic rollback, and state restore into one supported updater mode.

## What This Changes

Request an optional safe-update workflow that backs up OpenClaw state before updates, runs post-update health and smoke checks, and rolls back or restores state when checks fail.

## Best Possible Solution

Make a maintainer scope decision for a first-class safe-update mode; if accepted, implement an opt-in orchestrator around existing update, backup, health/status, channel/agent smoke, retention, rollback, and restore primitives with explicit secret and downgrade guarantees.

## Reproduction Assessment

Not applicable: this is a feature proposal rather than a current-main bug. Source and docs inspection show adjacent primitives exist, but no mode ties backup, retention, smoke checks, rollback, and restore into the updater.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. The maintainable direction is either a designed core updater mode that reuses existing primitives or an explicit split across backup-restore, rollback, smoke-check, retention, and Control UI follow-ups.
