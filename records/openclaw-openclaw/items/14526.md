## Summary

This should stay open: current main has useful updater, backup, restart-health, channel-readiness, and manual recovery primitives, but it still does not provide the integrated opt-in safe-update workflow with update-tied state backups, retention policy, real-agent smoke checks, and controlled package rollback plus state restore. The discussion and related open items support maintainer follow-up rather than cleanup closure.

## What This Changes

The issue requests an opt-in safe-update workflow that snapshots OpenClaw state before updating, runs post-update health and smoke checks, and rolls back or restores state on failure.

## Best Possible Solution

Design an explicit core safe-update mode, or reject that scope after maintainer review; if accepted, orchestrate update, backup/restore, doctor/status, channel readiness, real-agent smoke, package rollback, and retention/secret policy through one documented path.

## Reproduction Assessment

Not applicable. This is a feature/design proposal rather than a report of one broken existing contract, although the discussion includes real upgrade incidents showing why the workflow would be useful.

## Solution Assessment

Unclear until maintainers choose the product and security policy. The direction is reasonable, but core scope, automatic versus prompted rollback, retention limits, smoke criteria, and restore guarantees need review before implementation.
