## Summary

Keep open. Current main has useful updater, backup, health/status, package-swap, and manual rollback pieces, but it still does not implement the requested opt-in safe-update workflow tying pre-update snapshots, retention, gateway/channel/real-agent checks, and rollback/restore together.

## What This Changes

The issue requests an optional safer self-update workflow with pre-update backup, post-update health checks, and automatic rollback or state restore.

## Best Possible Solution

Make a maintainer scope decision for safe update as a first-class updater mode; if accepted, implement an explicit opt-in orchestrator around existing update, backup, doctor, status/health, channel-status, Control UI update, and package rollback primitives with secret-safe retention and compatibility semantics.

## Reproduction Assessment

Not applicable as a feature request rather than a bug report. The high-confidence verification path is source and docs inspection, and current main still lacks a safe-update workflow tying backups, retention, smoke checks, rollback, and restore together.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. The current adjacent primitives are not the complete solution; the narrowest maintainable direction is a designed core updater mode or a maintainer-approved split across backup-restore, rollback, smoke-check, retention, and Control UI follow-ups.

## Security Review

Status: needs_attention

Summary: This issue is security-sensitive because it proposes automated local-state archiving plus automated downgrade and restore behavior.

Concerns:

  - confidence: 0.88
