## Summary

This should stay open because current main has useful update, backup, status, and manual rollback primitives, but it still lacks the integrated safe-update workflow with update-tied snapshots, retention, channel or real-agent smoke checks, and controlled rollback plus state restore. Recent discussion includes production-rollout evidence and a prior keep-open review, and the related open items are partial rather than superseding the broader product decision.

## What This Changes

The issue requests an optional safer self-update workflow with pre-update backup, post-update health checks, retention policy, and rollback or restore on failure.

## Best Possible Solution

Design an explicit core safe-update mode that orchestrates update, backup, restore, doctor/status, channel health, real-agent smoke, and package rollback primitives with documented retention, permission, and secret-preservation policies.
