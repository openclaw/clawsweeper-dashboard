## Summary

Keep open: current main has meaningful updater, auto-update, backup, health, and plugin-convergence primitives, but the integrated opt-in safe-update workflow with update-coupled backups, retention, real-agent/channel smoke checks, and controlled rollback plus state restore is still missing.

## What This Changes

The issue requests an optional safe-update workflow that backs up OpenClaw state before updating, runs post-update health/smoke checks, and can roll back or restore safely on failure.

## Best Possible Solution

Make a maintainer decision on whether safe update belongs in core; if accepted, design one documented opt-in updater mode that composes backup/verify, status/deep health, channel readiness, real-agent smoke, rollback/restore, retention, and secret-handling policy.

## Risks / Open Questions

- The newer optional failure-reporting idea overlaps telemetry and privacy policy and should not be folded into a one-shot automated fix without maintainer approval.
