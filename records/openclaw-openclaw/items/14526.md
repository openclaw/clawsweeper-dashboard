## Summary

Keep this issue open: current main has useful updater, manual backup, config-snapshot, doctor, plugin-convergence, and gateway restart checks, but not the requested integrated state backup, retention, real agent/channel smoke verification, and controlled package-plus-state rollback workflow. The related open issues cover scheduling and compatibility scanning only, while the narrower rollback PR closed unmerged, so no canonical replacement owns the complete request.

## What This Changes

Add an optional safe-update workflow that snapshots critical state, validates the upgraded system, and rolls back the package and state when validation fails.

## Best Possible Solution

Define one core updater transaction that creates and verifies a permission-hardened snapshot, records the prior package and schema boundary, runs non-destructive gateway/channel/agent checks, and restores both package and compatible state only under an explicit recovery policy.

## Reproduction Assessment

Not applicable: this issue requests a new integrated safety mode rather than reporting one deterministic current-main defect. The discussion nevertheless supplies concrete production incidents showing why gateway-only verification is insufficient.

## Solution Assessment

Yes in product direction, but not yet in implementation detail: orchestration belongs in the canonical core updater rather than a second updater, while rollback boundaries, retention, secrets handling, and safe smoke semantics require maintainer design first.

## Risks / Open Questions

- Automatic state restore can overwrite valid post-update writes or reintroduce state that is incompatible with the restored package unless the updater defines an atomic, version-aware recovery boundary.
- Backup archives include credentials, sessions, and potentially workspaces, so destination permissions, encryption expectations, retention, deletion, and operator visibility need explicit security design.
- A generic smoke turn could consume provider quota, send channel-visible messages, mutate sessions, or exercise external tools unless health criteria and isolation are deliberately scoped.
