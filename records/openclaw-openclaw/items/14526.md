## Summary

Keep open: current main and the latest release have useful update, backup, health, and manual rollback primitives, but not the integrated opt-in safe-update workflow with update-tied backup retention, real agent/channel smoke checks, and automatic state restore.

## What This Changes

Request an opt-in safe update workflow that snapshots critical state before updating, verifies health after updating, and rolls back or restores state when checks fail.

## Best Possible Solution

Treat this as a product design task for an explicit opt-in core updater mode that composes backup create/verify, retention, doctor/status/gateway/channel/agent checks, package rollback, and controlled state restore, or document that these remain manual operator steps.
