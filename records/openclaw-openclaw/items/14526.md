## Summary

Keep open: current main has adjacent updater, auto-update, manual backup, doctor, plugin convergence, and gateway health/version checks, but it still lacks the requested opt-in safe-update workflow with update-tied backups, retention, real channel/agent smoke checks, and automatic rollback plus state restore.

## What This Changes

The issue proposes an opt-in safe update workflow that creates pre-update backups, runs post-update health and smoke checks, and rolls back or restores state on failure.

## Best Possible Solution

Make a maintainer product and security decision on core scope; if accepted, design one documented opt-in updater mode that composes backup/verify, deep status, channel readiness, real-agent smoke, rollback/restore, retention, and secret-handling policy.

## Solution Assessment

Unclear until maintainers choose the product and security policy. The direction is reasonable, but automatic rollback, restore guarantees, retention limits, smoke criteria, and optional failure reporting need explicit design before implementation.
