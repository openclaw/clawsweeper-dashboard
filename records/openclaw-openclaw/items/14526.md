## Summary

This should stay open: current main has update, manual backup/verify, restart health, channel status, auto-update, and limited recovery guidance, but it still does not implement the requested integrated safe-update workflow with update-coupled backups, retention, real-agent smoke checks, and controlled rollback plus state restore. The recent issue discussion adds production evidence and a reporting idea, both of which reinforce maintainer product/security review rather than cleanup closure.

## What This Changes

The issue proposes an opt-in safe update workflow that snapshots OpenClaw state before updating, runs post-update health and smoke checks, and rolls back or restores state when checks fail.

## Best Possible Solution

Make a maintainer decision on whether safe update belongs in core; if accepted, design one documented opt-in updater mode that orchestrates backup/verify, doctor/status, channel readiness, real-agent smoke, rollback/restore, retention, and secret-handling policy.
