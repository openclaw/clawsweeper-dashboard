## Summary

Keep open. Current main has useful adjacent updater, backup, auto-update, gateway health, plugin convergence, and manual rollback pieces, but it still does not provide the proposed opt-in safe-update workflow with full pre-update state backup, retention, channel/agent smoke checks, automatic rollback/downgrade, and state restore.

## What This Changes

Request an optional safe self-update mode that snapshots OpenClaw state before updating, runs post-update health and smoke checks, and rolls back or restores state when checks fail.

## Best Possible Solution

Make a maintainer scope decision for a first-class safe-update mode; if accepted, implement an opt-in orchestrator around existing update, backup, health/status, channel/agent smoke, retention, rollback, and restore primitives with explicit secret and downgrade guarantees.

## Reproduction Assessment

Not applicable: this is a feature proposal rather than a current-main bug. Source, docs, tests, and live issue discussion show adjacent primitives exist, but no supported mode ties full backup, retention, smoke checks, rollback, and state restore into the updater.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. The maintainable direction is either a designed core updater mode that reuses existing primitives or an explicit split across backup-restore, rollback, smoke-check, retention, and Control UI follow-ups.
