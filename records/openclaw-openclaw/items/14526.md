## Summary

Keep open: current main has useful updater, auto-update, backup-create/verify, restart-health, and manual recovery primitives, but it still lacks the integrated opt-in safe-update workflow with update-tied state backups, retention policy, real-agent smoke checks, and controlled rollback plus state restore.

## What This Changes

Add an opt-in safe self-update workflow that snapshots OpenClaw state before updating, runs post-update health and smoke checks, and offers or performs rollback/restore on failure.

## Best Possible Solution

Design this as an explicit core safe-update mode, or reject that scope after maintainer review; if accepted, it should orchestrate existing update, backup/restore, doctor/status, channel readiness, real-agent smoke, rollback, retention, and secret-policy surfaces through one documented path.

## Solution Assessment

Unclear until maintainers choose the product and security policy. The direction is reasonable, but core scope, prompted versus automatic rollback, retention limits, smoke criteria, failure-reporting, and restore guarantees need review before implementation.
