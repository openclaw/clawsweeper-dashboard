## Summary

Keep open. The stale-running task maintenance gap is still source-reproducible on current main, and https://github.com/openclaw/openclaw/pull/79288 is an open implementation PR with closing syntax for this issue.

## What This Changes

The issue asks for `openclaw tasks maintenance --apply` to detect multi-day stale `running` task rows and mark them terminal without manual cancellation or `doctor --fix`.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/79288, with focused regression coverage proving very stale `running` tasks without fresh active-work evidence become `lost` while genuinely live work is preserved.

## Reproduction Assessment

Yes, from source inspection. A stale active subagent task with a persisted non-wedged child session row can pass the current backing-session check and remain `running`, while audit still reports `stale_running` once the threshold is exceeded.

## Solution Assessment

Yes, task registry maintenance is the right fix surface. The narrowest maintainable solution is to improve stale active-task liveness reconciliation and add regression tests, while leaving optional CLI flags or doctor orchestration for later if maintainers still want them.
