## Summary

Keep open. Current main has narrowed one CLI stale-row path, but task maintenance can still leave stale `running` subagent rows active when an old non-wedged child-session row is treated as backing state; the related restart PR is open and restart-scoped, not a merged maintenance fix.

## What This Changes

Task maintenance should reconcile multi-day `running` task rows into a terminal state without requiring `doctor --fix` or manual cancellation.

## Best Possible Solution

Teach task maintenance to require authoritative live runtime or run evidence for very stale subagent-backed active tasks, with focused regression coverage, while keeping CLI run-context, cron authority, and recoverable subagent paths intact.

## Reproduction Assessment

Yes, from source inspection. A stale active `runtime: "subagent"` task older than the grace period can keep returning `hasBackingSession: true` when its child-session row still exists, so maintenance leaves it `running` and audit continues to report `stale_running`.

## Solution Assessment

Yes for the maintenance surface. The narrowest maintainable fix is runtime-backed stale subagent reconciliation with tests, not a broad age-only config knob; https://github.com/openclaw/openclaw/pull/78575 is useful but only covers restart-interrupted tasks.
