## Summary

Keep open. Current main still has a source-backed task-maintenance gap for stale `running` rows, and an open unmerged PR specifically references and implements the focused fix for this issue.

## What This Changes

Task maintenance should reconcile multi-day `running` task records as lost or cancelled when no fresh runtime backing remains, instead of requiring manual cancel or doctor repair.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79288 if maintainer review and CI stay acceptable; otherwise carry the same focused stale-running reconciliation fix with docs and regression tests.

## Reproduction Assessment

Yes, source-reproducible. On current main, a running subagent task with an old but present child-session row can keep `hasBackingSession()` true, so maintenance leaves it running while audit still reports `stale_running`.

## Solution Assessment

Yes. The focused maintenance-predicate direction in the open PR is the narrowest maintainable fix because it requires fresh active-work evidence for very stale running tasks instead of terminalizing by age alone.
