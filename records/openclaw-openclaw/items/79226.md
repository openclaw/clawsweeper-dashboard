## Summary

Keep open. Current main still has a source-backed stale-running maintenance gap, and an open closing PR specifically targets this issue, so this should remain open until that PR lands or is superseded.

## What This Changes

Request that task maintenance automatically reconciles multi-day `running` task records to `lost` or another terminal state when no fresh live-work evidence remains.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/79288 with a focused maintenance fix that marks stale running tasks lost only when fresh active-work evidence is absent, preserving active run contexts, cron ownership, and fresh child sessions.

## Reproduction Assessment

Yes, from source inspection. A multi-day `running` subagent task with a stale persisted child-session row can still pass the current backing-session check and remain `running`, while task audit reports `stale_running`.

## Solution Assessment

Yes. Task maintenance is the right surface, and the linked PR targets the reconciliation predicate with freshness checks rather than adding a broad age-only cancellation path first.
