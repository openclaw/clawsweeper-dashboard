## Summary

Keep open. Current main still has a source-backed maintenance gap where stale `running` task rows can remain active when an old persisted child/session row is treated as backing state, while related open work covers only parts of the stale-task family.

## What This Changes

Request that `openclaw tasks maintenance --apply` reconcile multi-day stale `running` task records without requiring manual cancel or doctor cleanup.

## Best Possible Solution

Teach task maintenance to require real live runtime/run evidence for very stale active tasks, add focused regression coverage for stale subagent/session-backed rows, and leave optional CLI flags or doctor orchestration for a separate maintainer decision if still needed.

## Reproduction Assessment

Yes, from source inspection. A stale active subagent task with a non-wedged persisted child session row can pass the current backing-session check, remain `running`, and continue to trigger the existing `stale_running` audit error after maintenance.

## Solution Assessment

Yes, `tasks maintenance` is the right surface for the core bug. The narrowest maintainable fix is to improve runtime-backed liveness reconciliation and regression tests, not to add a broad age-only config knob first.
