## Summary

The physical-device screenshots convincingly show the intended branding, but the implementation is not mergeable because it adds an App Store runtime dependency that explicitly reads undocumented UIKit state. The normal release project also remains unverified with the Watch target enabled.

## What This Changes

The PR adds a TopNotch-backed OpenClaw logo overlay to the iPhone notch, presents it when the app scene becomes active, and hides it from task-switcher capture.

## Best Possible Solution

Remove TopNotch and either implement a branding treatment entirely through documented public APIs or defer the notch effect, then validate the unmodified release archive and supported device classes.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new visual feature, and physical-device screenshots demonstrate the intended active-app and app-switcher results.

## Solution Assessment

No. The visual behavior works, but shipping an App Store app with an explicitly undocumented UIKit lookup is not the narrowest maintainable solution; use supported APIs or omit the effect.
