## Summary

Keep open. Current main still lacks the requested Mission Control/Control UI File Gallery left-nav entry and workspace docs/images gallery; existing file surfaces are chat attachments and a restricted Agents Core Files editor, so this is a real feature gap that needs maintainer product and security/API design rather than cleanup closure.

## What This Changes

The issue requests a permanent Mission Control left-nav File Gallery for workspace documents/images with refresh, open-in-browser, and empty/error states.

## Best Possible Solution

Design a scoped workspace File Gallery surface with authenticated Gateway index/open/download APIs constrained to the configured workspace, a Control UI tab with refresh and browser-open actions, docs/images filtering, clear empty/error states, tests for unsafe paths, and user-facing docs/changelog if shipped.

## Reproduction Assessment

Yes. Source inspection reproduces the absence on current main: the Control UI tab map/sidebar has no File Gallery route, and existing file RPCs are allowlisted to core agent files rather than workspace docs/images.

## Solution Assessment

Unclear as an automated repair. The requested left-nav/gallery direction is plausible, but the safest maintainable solution needs maintainer approval on the core-vs-plugin boundary and the Gateway file exposure contract before implementation.
