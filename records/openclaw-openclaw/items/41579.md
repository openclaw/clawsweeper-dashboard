## Summary

Keep open. Current main still lacks the requested Mission Control/Control UI File Gallery left-nav entry and workspace docs/images gallery, while the existing file surfaces are chat attachments and an agent Core Files editor scoped to bootstrap files.

## What This Changes

Request adds a permanent Mission Control File Gallery left-nav view for workspace documents/images with refresh, browser-open, and empty/error states.

## Best Possible Solution

Design a scoped workspace File Gallery surface with authenticated Gateway APIs constrained to the configured workspace, a Control UI tab/view, refresh and browser-open actions, docs/images filtering, unsafe-path tests, and user-facing docs/changelog if shipped.

## Reproduction Assessment

Yes. Source inspection on current main shows the static tab map/sidebar has no File Gallery route, and the only Gateway file RPCs are the existing agent bootstrap file helpers rather than a workspace docs/images index.

## Solution Assessment

Unclear as an automated repair. The requested left-nav gallery direction is plausible, but the maintainable solution needs maintainer approval on the core-vs-plugin boundary and the security contract for exposing workspace files through the Gateway.
