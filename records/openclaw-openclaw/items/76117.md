## Summary

Keep open: current main still has the reported 120-second default model idle watchdog, and this is a same-day local Ollama regression report with an open related PR proposing a broader timeout change. The source confirms the suspected timeout path and documented workaround, but the discussion does not yet prove whether the degraded tool use is caused by the idle watchdog, Ollama thinking defaults, model context pressure, or local hardware latency.

## What This Changes

The issue reports degraded local Ollama/qwen3.6:35B webchat behavior after upgrading to v2026.4.29, suspected to come from the 120-second LLM idle watchdog cutting off slow local reasoning.

## Best Possible Solution

Confirm the failure with a live Ollama/qwen3.6 gateway webchat or diagnostic trace, then choose a narrow Ollama/local-provider timeout or thinking-policy fix with tests rather than changing the global watchdog without root-cause proof.

## Reproduction Assessment

Unclear. The 120-second no-chunk watchdog path is reproducible from source and unit tests, but the full reported degraded tool-use behavior still needs a live ollama/qwen3.6:35B:a3b webchat run or logs tying the symptom to that watchdog.

## Solution Assessment
