## Summary

Keep open: current main and the latest release still lack the requested Discord `self-profile` action/gate for bot nickname/avatar updates. Existing code covers adjacent status/activity paths through `set-presence`, static presence config, and auto-presence, but the broader identity mutation surface needs maintainer product and security direction.

## What This Changes

The issue requests a Discord-only `self-profile` message action for bot self-only nickname, avatar, status, and activity updates behind a default-off config gate.

## Best Possible Solution

Have maintainers sponsor a Discord-owned, default-off self-profile contract that covers only bot self fields, requires explicit account selection when ambiguous, and documents/tests precedence with existing presence paths.
