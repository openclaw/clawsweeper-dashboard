## Summary

Keep open: current main still has Discord presence and auto-presence support, but not the requested bot self-profile action, `selfProfile` gate, nickname/avatar REST mutation path, or self-only target guardrails. The closed related PR contains useful source context but was not merged and had review comments on account routing and best-effort behavior.

## What This Changes

Request to add a default-off Discord `self-profile` action for bot-only nickname, avatar, status, and activity updates with strict scope and multi-account guards.

## Best Possible Solution

If accepted, implement this inside the bundled Discord plugin with a separate default-off `channels.discord.actions.selfProfile` gate, bot-self REST helpers, strict accountId and target guards, avatar validation, and focused docs/tests/changelog.

## Reproduction Assessment

Not applicable: this is a feature request, not broken current behavior. Source inspection on current main gives a high-confidence absence check: Discord exposes `set-presence` only, with no `selfProfile` gate, nickname path, avatar path, or self-profile handler.

## Solution Assessment

Unclear until maintainer approval: the requested direction is plausible and narrowly scoped to Discord, but it adds a new user-facing mutation capability and config gate. The maintainable path is plugin-local implementation with strict self-only REST helpers, not a generic cross-provider profile API.
