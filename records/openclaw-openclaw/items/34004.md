## Summary

Keep open: current main and the latest release provide Discord static presence, auto-presence, and a gated `set-presence` action, but they still do not provide the requested `self-profile` action, `selfProfile` gate, bot nickname/avatar mutation path, or self-only profile guardrails. The older umbrella issue was already closed in favor of this issue, and the remaining work is a security-sensitive Discord product/API decision rather than a safe autonomous fix.

## What This Changes

The issue requests a Discord-only, default-off `self-profile` message action for bot self-only nickname, avatar, status, and activity updates with strict scope guards, account targeting, validation, and partial-result behavior.

## Best Possible Solution

Keep this issue open as the canonical self-profile product/security discussion; maintainers should either approve a Discord-owned default-off action/API with explicit trust, account, validation, and precedence rules, or de-scope nickname/avatar mutation while leaving https://github.com/openclaw/openclaw/issues/29725 to track the narrower plugin-runtime presence API.
