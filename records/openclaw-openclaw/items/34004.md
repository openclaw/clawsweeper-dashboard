## Summary

Keep open: current `main` and `v2026.6.6` cover Discord presence through `set-presence`/`setPresence` and `autoPresence`, but the requested self-profile action, `selfProfile` gate, nickname mutation, avatar mutation, self-only target checks, and partial-result behavior are still absent. The merged related work at https://github.com/openclaw/openclaw/pull/33277 explicitly scoped itself to auto-presence and left self-profile out, while https://github.com/openclaw/openclaw/pull/34010 was a closed unmerged implementation attempt rather than a safe canonical fix.

## What This Changes

The issue requests a default-off Discord `self-profile` message action for bot-only nickname, avatar, status, and activity updates with strict self-scope and multi-account guards.

## Best Possible Solution

Keep this as the canonical self-profile decision item until maintainers approve the Discord plugin action/API shape, then implement it as a default-off, self-only bundled Discord action with focused validation, tests, and docs.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source and release inspection give a high-confidence gap: `selfProfile`/`self-profile` is absent while only presence actions are implemented.

## Solution Assessment

Unclear as proposed: the self-only/default-off direction is plausible, but maintainers still need to choose the public action contract, account scoping, audit model, and partial-failure semantics before a fix PR is appropriate.
