## Summary

Keep open: current main has a gated Discord `set-presence` action, but it does not implement the requested bot self-profile action for nickname/avatar updates or the separate `selfProfile` gate; the related contributor PR was closed unmerged.

## What This Changes

The issue requests a Discord-only, default-off `self-profile` message action for bot self nickname, avatar, status, and activity updates with strict self-only guards.

## Best Possible Solution

If accepted, implement this inside the bundled Discord plugin with a separate default-off `channels.discord.actions.selfProfile` gate, bot-self REST helpers, strict accountId and target guards, avatar validation, and focused docs/tests/changelog.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection on current main gives a high-confidence absence check: Discord exposes `set-presence` only for this area, with no self-profile action, gate, nickname path, or avatar path.
