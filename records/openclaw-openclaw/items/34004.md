## Summary

Keep open: the canonical Discord self-profile request remains unimplemented on current main. AutoPresence is shipped, but the Discord plugin still exposes only the separate `set-presence` action and has no `selfProfile` gate, nickname/avatar mutation path, or self-only profile guardrails.

## What This Changes

Request adds a Discord-only, opt-in self-profile message action for bot-only nickname, avatar, status, and activity updates with strict self-scope controls.

## Best Possible Solution

Make an owner-reviewed Discord plugin change that adds the default-off self-profile action, self-only and multi-account safeguards, avatar validation, docs, and focused tests, or explicitly de-scope profile mutation from the bundled Discord plugin.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection of current main shows the requested Discord self-profile action is absent rather than a broken existing contract.

## Solution Assessment

Unclear. A Discord-plugin-local, default-off action appears to be the narrowest maintainable direction if maintainers want this capability, but the new profile mutation surface needs product/security owner approval first.
