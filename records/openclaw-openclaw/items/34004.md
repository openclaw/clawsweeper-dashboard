## Summary

Keep open: current main still lacks the requested Discord self-profile action and gate. The shipped surface covers `set-presence` and autoPresence, while #33270 was explicitly closed as superseded by this narrower tracker and both related self-profile PRs are closed unmerged.

## What This Changes

The issue requests a gated Discord self-profile action for bot-only nickname, avatar, status, and activity updates with strict self-scope and validation safeguards.

## Best Possible Solution

Implement the feature, if approved, inside the bundled Discord plugin: add `channels.discord.actions.selfProfile` defaulting off, expose `self-profile` only through Discord action discovery, use Discord self endpoints only, reject all non-self selectors, validate avatar inputs before network calls, require explicit `accountId` when account selection is ambiguous, and cover docs/config/tests/changelog together.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to enable the proposed action mentally against current main: `selfProfile`/`self-profile` has no config gate, discovery entry, handler branch, or runtime dispatch path, while `set-presence` is the only profile-adjacent action currently wired.

## Solution Assessment

Yes, with maintainer review of the privileged mutation boundary. Keeping the capability Discord-plugin-owned, self-only, and default-off is the narrowest maintainable direction; the exact `bestEffort` and avatar-source contract should be settled before coding.

## Security Review

Status: needs_attention

Summary: The issue asks for a new privileged Discord bot identity mutation surface, so implementation should get explicit security review even though there is no patch diff here.

Concerns:

- **[medium] Gate and scope profile mutation tightly:**
  - body: A self-profile action must not expose arbitrary user/member edits; it should be default-off, Discord-only, and reject `userId`, `memberId`, or `target` values that do not resolve to the bot itself.
  - confidence: 0.9
- **[medium] Validate avatar inputs before outbound mutation:**
  - body: Avatar support can accept user-controlled media/path/buffer data, so MIME, strict base64, size, and local media-root checks should happen before calling Discord APIs.
  - confidence: 0.86
