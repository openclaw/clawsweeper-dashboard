## Summary

Keep open: current main still lacks the requested Discord `self-profile` action and `selfProfile` gate, while the related implementation PRs are closed and unmerged. The request is concrete, but it adds a new Discord bot profile mutation surface and config knob, so it needs maintainer product/security approval before implementation.

## What This Changes

Add a Discord-only `self-profile` message action for bot-owned nickname, avatar, status, and activity updates behind a default-off `channels.discord.actions.selfProfile` gate.

## Best Possible Solution

If accepted, implement this in the official Discord plugin with a default-off gate, strict self-only checks, avatar validation, explicit multi-account routing, docs, focused tests, and live Discord proof before merge.

## Reproduction Assessment

No; this is a feature request rather than a broken existing behavior report. Source inspection gives a high-confidence current-main gap: Discord exposes `setPresence`, but not the requested `self-profile` action or `selfProfile` gate.
