## Summary

Keep open: current main still lacks bot nickname/avatar mutation, and this issue remains the canonical request after the broader issue was closed in its favor. Existing presence support covers part of the proposal, but the remaining identity-mutation authority and overlap with the separate runtime-presence API need explicit product and security decisions.

## What This Changes

Add a default-off Discord self-profile action for bot-only nickname, avatar, status, and activity updates with strict target, input, and multi-account safeguards.

## Best Possible Solution

Define one least-privilege, Discord-plugin-owned contract with explicit account and authorization handling, separating or clearly composing REST profile mutation and gateway presence so there is no duplicate presence source of truth.
