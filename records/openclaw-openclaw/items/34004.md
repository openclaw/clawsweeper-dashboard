## Summary

Keep open: current main and v2026.6.6 have static Discord presence config, auto-presence, and a gated `set-presence` action, but they still do not implement the requested Discord `self-profile` action, `selfProfile` gate, bot nickname/avatar mutation path, or self-only profile guardrails. The closest open related item covers presence only, while the companion implementation PR is closed unmerged and not a safe canonical fix path.

## What This Changes

The issue requests a Discord-only self-profile action for bot self nickname, avatar, presence, and activity updates behind an explicit default-off gate with strict self-only and multi-account safeguards.

## Best Possible Solution

Keep the existing `set-presence` path for presence; if maintainers sponsor self-profile, implement it inside the Discord bundled plugin with a default-off gate, explicit multi-account targeting, strict self-only authorization, avatar validation, focused docs/tests, and live Discord proof.
