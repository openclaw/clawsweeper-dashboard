## Summary

Keep open: current main still only supports Discord presence/activity updates, not the requested default-off bot self-profile action for nickname/avatar/status with self-only and multi-account guardrails. The related implementation PR was closed unmerged, so it is useful context but not shipped behavior.

## What This Changes

Request to add a Discord `self-profile` message action for bot-only nickname, avatar, and status/activity updates behind a default-off `channels.discord.actions.selfProfile` gate.

## Best Possible Solution

If accepted, implement this inside the bundled Discord plugin with a separate default-off `selfProfile` gate, explicit account routing, self-only target rejection, avatar validation, and focused docs/tests/changelog.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection gives a high-confidence absence check: current main exposes `set-presence` only, with no `selfProfile` gate, nickname/avatar REST path, or self-profile handler.
