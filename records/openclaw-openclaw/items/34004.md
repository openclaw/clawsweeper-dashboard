## Summary

Current main still lacks bot self-avatar and guild-nickname mutation, and the earlier umbrella report was deliberately narrowed into this canonical issue. Keep it open for an explicit product/security decision, but separate the missing identity mutations from presence because Discord already has a gated `set-presence` action and a narrower open runtime-presence API discussion.

## What This Changes

Add an opt-in Discord action for bot-only nickname, avatar, and presence updates with strict self-scope, account routing, validation, and optional partial-result handling.

## Best Possible Solution

Keep `set-presence` as the sole presence mutation path, then approve or reject a separate Discord-plugin capability limited to bot self-avatar and guild nickname updates, with explicit account and trusted-requester scope, strict media validation, focused tests, documentation, and live Discord proof.

## Reproduction Assessment

Not applicable: this is a new-capability request, not a broken-behavior report; current-main source inspection clearly shows supported presence mutation and absent avatar/nickname mutation.

## Solution Assessment

No, the combined `self-profile` shape is not the best boundary because it duplicates `set-presence`; the maintainable direction is a separate, narrowly scoped self-identity capability if maintainers approve that security surface.
