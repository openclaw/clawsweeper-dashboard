## Summary

Keep open: current `main` still has Discord presence and auto-presence support, but not the requested `self-profile` action, `selfProfile` gate, bot nickname/avatar mutation path, or self-only profile guardrails. The request is well scoped but adds a new security-sensitive Discord bot mutation surface, so it needs maintainer product/security approval before implementation.

## What This Changes

The issue requests a Discord-only `self-profile` message action, gated by `channels.discord.actions.selfProfile`, for bot-only nickname, avatar, status, and activity updates.

## Best Possible Solution

If maintainers want this capability, implement it inside the official Discord plugin behind a default-off `selfProfile` gate with self-only checks, avatar validation, docs, config schema updates, and focused regression tests.

## Reproduction Assessment

No; this is a feature request rather than a broken existing behavior report. Source inspection gives a high-confidence current-main gap: the requested action and gate are absent while only `setPresence` is implemented.

## Solution Assessment

Unclear pending maintainer approval. The requested plugin-local, default-off Discord action is the right technical shape if maintainers want it, but the new profile mutation surface should be explicitly accepted first.
