## Summary

Keep open: current main still exposes Discord presence/status updates only, not the requested default-off self-profile action for bot nickname/avatar/status with self-only and multi-account guardrails. The related implementation PR at https://github.com/openclaw/openclaw/pull/34010 was closed unmerged, so it is useful context rather than shipped behavior.

## What This Changes

Request a Discord `self-profile` action for bot self-only nickname, avatar, status, and activity updates behind a default-off `channels.discord.actions.selfProfile` gate.

## Best Possible Solution

If accepted, implement this inside the bundled Discord plugin with a separate default-off `selfProfile` gate, explicit account routing, self-only target rejection, strict avatar validation, docs, tests, and changelog coverage.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection on origin/main gives a high-confidence absence check: only `set-presence` is registered and routed, with no `selfProfile` config gate or nickname/avatar mutation path.

## Solution Assessment

Unclear until maintainer approval: the requested Discord-only, default-off action is narrower than a generic cross-provider profile API, but it adds a security-sensitive identity mutation capability.
