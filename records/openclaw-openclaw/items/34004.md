## Summary

Keep open: current main still lacks the requested Discord `self-profile` action and `channels.discord.actions.selfProfile` gate, while the related auto-presence work shipped only presence health signaling and the self-profile PR was closed unmerged.

## What This Changes

Request adds a Discord `self-profile` message action, gated off by default, for bot self-only nickname, avatar, status, and activity updates with strict scope and validation.

## Best Possible Solution

If approved, implement this inside the bundled Discord plugin with a default-off `selfProfile` gate, action discovery, runtime dispatch, Discord self endpoints only, non-self selector rejection, avatar validation, multi-account ambiguity handling, docs, schema, tests, and changelog together.

## Reproduction Assessment

Yes for source-level verification of the missing feature: current main has no `self-profile` discovery, handler, runtime action, schema key, or docs entry. This is not a broken existing contract; it is an absent requested capability.

## Solution Assessment

Yes directionally, with maintainer review first. Keeping the work Discord-plugin-owned, self-only, and default-off is the narrow maintainable path, but the privileged mutation boundary and avatar/best-effort semantics should be approved before coding.

## Security Review

Status: needs_attention

Summary: No patch was reviewed, but the requested feature is security-sensitive because it would let agents mutate bot identity/profile fields and should receive explicit security review before implementation.

Concerns:

- **[medium] Gate and scope profile mutation tightly:**
  - body: A self-profile action must stay Discord-only, default-off, and self-only; it should reject `userId`, `memberId`, or `target` values that do not resolve to the bot itself and must not expose arbitrary member edits.
  - confidence: 0.9
- **[medium] Validate avatar inputs before outbound mutation:**
  - body: Avatar support would accept user-controlled media, path, file, or buffer data, so MIME, strict base64, size, and local media-root checks should happen before any Discord API call.
  - confidence: 0.86
