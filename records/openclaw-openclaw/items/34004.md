## Summary

Keep open: current main and v2026.6.6 support static Discord presence, auto-presence, and a gated `set-presence` action, but they still do not provide the requested Discord `self-profile` action, `selfProfile` gate, nickname/avatar mutation path, or self-only profile guardrails. ## What This Changes

Request adding a Discord-only, bot-self-scoped `self-profile` message action for nickname, avatar, status, and activity updates behind a default-off gate.

## Best Possible Solution

Keep this issue as the canonical Discord self-profile design thread, then implement a Discord-owned action only after maintainers choose the API shape, default-off gate, account-selection rules, self-only validation, avatar validation, docs, and tests.
