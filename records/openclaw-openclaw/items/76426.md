## Summary

Keep open. The report matches a real post-update failure path, and the latest release already contains the missing-payload retry before emitting this error, so this needs root-cause triage rather than cleanup close.

## What This Changes

The issue reports that upgrading to v2026.5.2 fails to update/install the bundled `@openclaw/diffs` plugin because the expected package directory is missing after the update.

## Best Possible Solution

Identify why the `@openclaw/diffs` install record survives while the npm package directory is absent, then fix the install/update or package-release path and add a regression covering the post-update sync failure.

## Reproduction Assessment

Yes from source for the reported terminal error: a tracked `diffs` install record whose `installPath` is missing reaches `collectMissingPluginInstallPayloads` and emits the exact message. No live upgrade was run in this read-only review, and the underlying reason the npm package directory disappears is still unclear.

## Solution Assessment

No current implemented solution is proven sufficient. The existing v2026.5.2 and main retry/detect path still allows this failure, so the better fix is to find why install/update records become inconsistent instead of closing against the detector.
