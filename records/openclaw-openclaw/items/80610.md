## Summary

Keep open: the gateway enrichment itself is plausible, but the PR does not propagate the new field through the Swift event payloads that the named macOS clients actually decode, and the external-PR real behavior proof gate is still unmet.

## What This Changes

The PR adds an optional `isHeartbeat` field to gateway `agent` event broadcasts, the TypeBox schema, generated Swift protocol model, and gateway unit coverage.

## Best Possible Solution

Keep the additive gateway field, but carry it through the actual Swift/macOS event payload types and tests so external consumers can use it without parsing event text.

## Reproduction Assessment

For the feature request itself, reproduction is not applicable; it is an additive protocol surface. The blocking PR issue is source-reproducible because current macOS/Swift decoder structs omit `isHeartbeat` while the PR only updates the generated `AgentEvent` model.

## Solution Assessment

No. Injecting the field at the gateway boundary is likely the right direction, but the current patch is incomplete until the Swift payloads actually decoded by Control UI/WebChat also expose it and real output proof is attached.
