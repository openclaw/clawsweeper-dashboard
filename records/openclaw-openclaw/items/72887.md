## Summary

Keep open. Current main still has no configurable review queue for deep Dreaming promotions: managed deep dreaming ranks candidates, immediately applies them through the MEMORY.md append path, and only writes the deep report afterward. The linked pollution/storage reports are adjacent but do not provide the requested final-promotion destination or approval contract.

## What This Changes

Request an additive memory-core workflow/config option that stages deep Dreaming promotion candidates for review before they are written to MEMORY.md.

## Best Possible Solution

Add an explicit, additive final-promotion destination contract in memory-core that preserves direct MEMORY.md writes as the compatible default while allowing promoted candidates to be staged in a structured review artifact before canonical long-term memory is mutated; update the manifest, SDK resolver, docs, CLI/UI expectations, and focused tests around the chosen contract.

## Reproduction Assessment

Not applicable as a feature/design request. The current behavior has a high-confidence source verification path: managed deep dreaming calls applyShortTermPromotions, and that helper appends selected candidates to MEMORY.md before the deep report is written.

## Solution Assessment

Unclear until maintainers choose the workflow contract. Directionally, a separate final-promotion destination or approval setting is cleaner than overloading dreaming.storage.mode, which currently describes phase/report artifact placement rather than the durable write target.
