## Summary

Keep open. Current main still lacks a final-promotion review queue: managed deep dreaming ranks candidates, calls the direct MEMORY.md append helper, and only writes the deep report afterward. The linked memory-pollution reports and storage-mode fix are adjacent, but they do not provide the requested destination or approval contract.

## What This Changes

Request support for routing memory-core deep dreaming promotions to a review queue before they are written to MEMORY.md.

## Best Possible Solution

Add an explicit, additive final-promotion destination contract in memory-core that preserves direct MEMORY.md writes as the compatible default while allowing selected candidates to be staged in a structured review artifact before canonical long-term memory is mutated; update the manifest, resolver, docs, CLI/UI expectations, and focused tests around the chosen contract.

## Reproduction Assessment

Not applicable as a feature/design request. The current behavior has a high-confidence source verification path: managed deep dreaming calls applyShortTermPromotions, and that helper appends selected candidates to MEMORY.md before the deep report is written.

## Solution Assessment

Unclear until maintainers choose the workflow contract. Directionally, a separate final-promotion destination or approval setting is cleaner than overloading dreaming.storage.mode, which currently describes phase/report artifact placement rather than the durable write target.
