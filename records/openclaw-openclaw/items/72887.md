## Summary

Keep open. Current main still lacks a final-promotion review queue: managed deep dreaming ranks candidates and calls the direct promotion apply helper, which writes selected entries to `MEMORY.md`; the existing `dreaming.storage.mode` and deep report paths only cover phase/report artifacts. The related storage-mode fix and memory-pollution reports are relevant context, but they do not provide the requested destination or approval contract.

## What This Changes

The issue asks memory-core Dreaming to support a configurable review destination for deep-phase promoted candidates before automatically mutating `MEMORY.md`.

## Best Possible Solution

Add an additive final-promotion destination contract in memory-core that preserves direct `MEMORY.md` writes as the compatible default while allowing selected candidates to be staged in a structured review artifact before canonical long-term memory is mutated; update the manifest/schema resolver, docs, CLI/UI expectations, and focused tests around the chosen contract.

## Reproduction Assessment

Not applicable as a feature/design request. The current behavior has a high-confidence source verification path: managed deep dreaming calls `applyShortTermPromotions`, and that helper appends selected candidates to `MEMORY.md`.

## Solution Assessment

Unclear until maintainers choose the workflow contract. Directionally, a separate final-promotion destination or approval setting is cleaner than overloading `dreaming.storage.mode`, which currently describes phase/report artifact placement rather than the durable write target.
