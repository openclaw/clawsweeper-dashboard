## Summary

Keep open. Current main still has no final-promotion review queue: managed deep Dreaming and manual `memory promote --apply` both use the direct `MEMORY.md` append helper, while existing storage configuration, docs, and tests only cover phase/report artifacts.

## What This Changes

Request a configurable deep Dreaming promotion destination so candidates can be staged for review before being appended to `MEMORY.md`.

## Best Possible Solution

Add an additive final-promotion destination contract in memory-core that defaults to the current append behavior while supporting structured review artifacts and later approval into canonical memory.

## Reproduction Assessment

Not applicable as a feature/design request. Current-main verification is high-confidence: eligible managed deep Dreaming candidates and manual `memory promote --apply` both flow through the direct `MEMORY.md` append path.

## Solution Assessment

Unclear until maintainers choose the workflow contract. A separate final-promotion destination or approval flow looks safer than reusing `dreaming.storage.mode`, which currently controls phase/report storage rather than canonical memory mutation.
