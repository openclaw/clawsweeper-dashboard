## Summary

Keep open. Current main still lacks a final-promotion review queue: managed deep dreaming ranks candidates, sends selected candidates through the direct MEMORY.md append helper, and writes the deep report only afterward. The linked raw-snippet, metadata-contamination, storage-mode, and promote-then-rotate items are adjacent but do not provide the requested destination or approval contract.

## What This Changes

Request support for staging memory-core deep dreaming promotions in a review queue before they are written into MEMORY.md.

## Best Possible Solution

Add an explicit, additive final-promotion destination contract in memory-core that preserves direct MEMORY.md writes as a compatible mode while allowing promoted candidates to be staged in a structured review artifact before canonical long-term memory is mutated; update the manifest, resolver, docs, and focused tests around that chosen contract.

## Reproduction Assessment

Not applicable as a feature/design request. The current behavior has a high-confidence source verification path: managed deep dreaming calls applyShortTermPromotions, and that helper appends selected candidates to MEMORY.md.

## Solution Assessment

Unclear until maintainers choose the workflow contract. Directionally, a separate final-promotion destination or approval setting is cleaner than overloading dreaming.storage.mode, which currently describes phase/report artifact placement.
