## Summary

Keep open. Current main still has no final-promotion review queue: managed deep Dreaming and manual apply both route eligible candidates through the direct MEMORY.md append path, while existing storage settings only cover phase/report artifacts.

## What This Changes

Request configurable deep Dreaming promotion output so candidates can be staged for review before automatic MEMORY.md writes.

## Best Possible Solution

Add an additive final-promotion destination contract in memory-core, defaulting to current MEMORY.md append behavior while allowing deep candidates to be staged in a structured review artifact and later approved into canonical memory.

## Reproduction Assessment

Not applicable as a feature/design request. Current-main verification is high-confidence: an eligible managed deep Dreaming candidate calls applyShortTermPromotions and the focused tests assert the promoted snippet appears in MEMORY.md.

## Solution Assessment

Unclear until maintainers choose the workflow contract. A separate final-promotion destination setting or approval flow looks safer than reusing dreaming.storage.mode, because that setting currently means phase/report storage rather than canonical memory mutation.
