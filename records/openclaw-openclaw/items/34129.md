## Summary

Keep open: current main still preserves the blank-name plus `call_auto_1` path, and #56391 is an open PR that explicitly closes this issue with the narrow normalization fix.

## What This Changes

The issue reports NVIDIA Kimi K2.5 emitting blank tool-call names through the OpenAI-completions path, causing repeated `call_auto_*` `Tool not found` loops and chat-visible token leakage.

## Best Possible Solution

Review and land #56391 or an equivalent narrow runner-normalization fix that gives blank or whitespace-only tool calls a deterministic non-empty dispatch name, or drops them before dispatch, with regression coverage for the `call_auto_1` case.

## Reproduction Assessment

Yes, at source level: current main keeps a blank tool-call name blank when the id is also blank and assigns `call_auto_1`. I did not rerun the live NVIDIA Kimi endpoint.

## Solution Assessment

Yes, the best direction is the narrow pre-dispatch blank-name guard already proposed in #56391 or an equivalent drop policy. Current main is not the best solution because it still allows blank names to reach dispatch.
