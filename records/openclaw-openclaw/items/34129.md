## Summary

Keep open. Current main still preserves the reported blank-name plus `call_auto_*` failure shape, and the issue is already paired with open closing PR #56391 rather than being fixed on main.

## What This Changes

The issue reports NVIDIA/Kimi emitting blank tool-call names that normalize into `call_auto_*` fallback IDs and repeated unavailable-tool loops.

## Best Possible Solution

Land PR #56391 or an equivalent narrow fix in the tool-call normalization layer that replaces blank or whitespace-only non-inferable tool names with a deterministic non-blank failure sentinel before dispatch, with regression coverage replacing the current blank-preserving expectation. Keep the separate `parallel_tool_calls` compatibility work tracked outside this issue.

## Reproduction Assessment

Yes. The current unit-level path reproduces the core failure shape: a blank name and blank id survive as `name: ""` plus `id: "call_auto_1"`, and the live runner uses this normalization wrapper before dispatch.

## Solution Assessment

Yes. The narrow maintainable fix is to handle blank/non-inferable tool names before dispatch in the existing normalization wrapper rather than adding NVIDIA-specific branching or relying on the unknown-tool loop guard, which currently ignores incomplete blank-name calls.
