## Summary

Keep open. Current main still lets blank, non-inferable tool names survive normalization, assigns `call_auto_*` fallback ids, and the existing unknown-tool loop guard does not count blank names. #56391 remains an open closing PR for the same bug, so this should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that NVIDIA Kimi K2.5 can emit blank tool-call names that become `call_auto_*` fallback calls and repeat as `Tool not found` loops.

## Best Possible Solution

Keep this issue open and review/land #56391, or merge an equivalent maintainer-approved fix that handles blank or whitespace-only tool names without an inferable id before dispatch and adds regression coverage proving the blank-name plus `call_auto_*` loop cannot recur.
