## What This Changes

The issue asks OpenClaw to stop NVIDIA Kimi K2.5 blank tool-call names from becoming `call_auto_*` / `Tool not found` retry loops.

## Best Possible Solution

Land PR #56391 or an equivalent narrow fix in the tool-call normalization wrapper that replaces non-inferable blank or whitespace-only tool names with a deterministic non-blank failure sentinel before dispatch.

## Reproduction Assessment

Yes. Source inspection shows the current wrapper preserves a blank non-inferable tool name, assigns `call_auto_1`, and has tests that lock in that behavior without needing a live NVIDIA run.

## Solution Assessment

Yes. The narrow maintainable fix is in the existing normalization layer before dispatch, not in NVIDIA-specific branching or the separate `parallel_tool_calls` compatibility path.
