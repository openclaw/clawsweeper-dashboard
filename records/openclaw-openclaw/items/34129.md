## Summary

Keep open: current main still allows the blank-name plus `call_auto_1` path, and #56391 is an open closing PR for the narrow normalization fix.

## What This Changes

The issue requests preventing malformed blank or whitespace-only Kimi tool-call names from becoming `call_auto_*` unknown-tool loops and leaking internal call IDs to chat.

## Best Possible Solution

Review and land #56391 or an equivalent narrow runner-normalization fix that prevents blank tool names from reaching dispatch, with regression coverage for the blank-name plus `call_auto_1` case.

## Reproduction Assessment

Yes, at source level: a blank name and blank id still pass through normalization as blank, then id normalization assigns `call_auto_1`; the existing focused test asserts that behavior. I did not run the live NVIDIA endpoint.

## Solution Assessment

Yes, the narrow pre-dispatch blank-name guard proposed by #56391, or an equivalent drop policy, is the best fix boundary. A NVIDIA-only provider workaround would miss the generic malformed tool-call path.
