## What This Changes

The issue reports that NVIDIA Kimi K2.5 can emit blank tool-call names that normalize into `call_auto_*` / `unknown` tool results and repeated `Tool not found` loops.

## Best Possible Solution

Land a revised generic embedded-runner fix that treats blank emitted tool names as invalid and non-dispatchable while preserving id-based recovery and replay pruning, then close the issue after the fix merges.
