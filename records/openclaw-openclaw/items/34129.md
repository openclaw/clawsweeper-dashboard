## Summary

Keep open: current `main` and latest release still preserve the blank tool-name plus `call_auto_1` path, and the open closing PR at https://github.com/openclaw/openclaw/pull/56391 has not landed.

## What This Changes

The issue reports that NVIDIA Kimi K2.5 can emit blank tool-call names that become fallback `call_auto_*` IDs and repeat as tool-not-found loops.

## Best Possible Solution

Land a reviewed generic embedded-runner fix that treats blank emitted tool names as invalid and non-dispatchable while preserving id-based recovery, then close the issue after the fix merges.

## Reproduction Assessment

Yes, at source level: current main keeps a blank tool name blank, assigns `call_auto_1` for a blank id, and has a focused test asserting that behavior. I did not run the live NVIDIA endpoint.

## Solution Assessment

Yes: the maintainable fix boundary is generic runner-side malformed tool-call handling, not an NVIDIA-only provider workaround. The open linked PR should be reviewed, revised if needed, and landed or closed before creating another repair lane.
