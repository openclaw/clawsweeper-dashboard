## Summary

This should stay open because #56391 explicitly closes it and current main still preserves the blank-name plus `call_auto_1` path instead of converting it to a non-empty dispatch name.

## What This Changes

The issue reports NVIDIA Kimi K2.5 producing blank tool-call names that become `call_auto_*` tool results and trigger repeated `Tool not found` loops with user-visible token leakage.

## Best Possible Solution

Land #56391 or an equivalent narrow runner-normalization fix that gives blank and whitespace-only tool calls a deterministic non-empty dispatch name or drops them before dispatch, with regression coverage for the `call_auto_1` case.

## Reproduction Assessment

Yes at source level: a tool call with blank name and blank id still keeps `name: ""` and receives `call_auto_1` on current main. The full live NVIDIA Kimi loop was not rerun.

## Solution Assessment

No, current main is not a complete fix because blank names remain blank and are classified as incomplete rather than guarded unknown tools. The linked sentinel-style PR or an equivalent non-empty-name/drop policy is the narrow maintainable direction.
