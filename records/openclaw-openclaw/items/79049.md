## Summary

Keep open: the PR is a plausible narrow localization fix with after-fix proof, but it still leaves the tool details side-panel content built by `buildToolCardSidebarContent` in hard-coded English, which is one of the affected chat/tool-card metadata surfaces.

## What This Changes

## Best Possible Solution

Yes. Source inspection gives a high-confidence path: set a non-English Control UI locale, render a tool card, open its side panel, and the side-panel markdown content still contains hard-coded English labels from `buildToolCardSidebarContent`.

## Solution Assessment
