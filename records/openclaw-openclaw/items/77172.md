Full review comments:

- **[P1] Use the existing wake flag name:** `src/infra/heartbeat-runner.ts:1190`
  - body: The PR checks `preflight.isWakeReason`, but `HeartbeatPreflight` only carries `isWakePayload`, `isExecEventWake`, and `isCronWake`. Strict TypeScript should fail here; use the existing wake-payload flag instead.
  - confidence: 0.94
- **[P2] Keep unauthenticated remote binds fail-closed:** `src/gateway/server-runtime-config.ts:149-153`
  - body: Changing the unauthenticated non-loopback guard from a thrown error to a loopback fallback contradicts the documented startup behavior and the current tests that reject this configuration. Security-sensitive misconfiguration should stay explicit instead of silently changing bind host.
  - confidence: 0.91
- **[P2] Check channel readiness before heartbeat delivery:** `src/infra/heartbeat-runner.ts:1420-1429`
  - body: The refactor removes the `heartbeatPlugin.heartbeat.checkReady` gate before normal heartbeat delivery, so alert heartbeats can send through a channel that reports it is not ready. Keep the readiness check on the main delivery path, not only the OK fallback.
  - confidence: 0.84
- **[P2] Keep ACP client progress quiet when verbose is false:** `src/acp/client.ts:122`
  - body: Assigning `log.isEnabled = () => false` does not silence `log.info()`, because subsystem logger methods call `emitLog()` directly. This makes the ACP client emit spawn, initialize, and session progress even when `opts.verbose` is false.
  - confidence: 0.86
- **[P3] Remove local scratch and test output files:** `scratch/debug_regex.js:1`
  - body: The PR adds an ad hoc scratch script plus root-level test-output artifacts containing local failed-test output. These should be removed or converted into proper regression tests before merge.
  - confidence: 0.96
