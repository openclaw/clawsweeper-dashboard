## Summary

Keep open: the hardening direction is useful, but this PR is not merge-ready because several schema caps conflict with current runtime/API contracts and the external contributor has not provided real after-fix behavior proof.

## What This Changes

The branch adds TypeBox `maximum`, `minimum`, `maxItems`, and description hints across agent tool schemas for browser, message, nodes, sessions, gateway, memory, canvas, subagents, and web fetch tools.

## Best Possible Solution

Rebase onto current main, keep schema hardening only where bounds match runtime and dependency contracts, add focused schema/contract coverage, and include redacted real gateway/model proof for representative bounded calls.

## Reproduction Assessment

Yes. The review findings are source-reproducible from the PR diff against current-main contracts for node recordings, session listing, long subagent runs, Discord GuildMedia, and browser snapshots; I did not run tests because this is a read-only review.

## Solution Assessment

No. Schema hardening is the right direction, but this implementation sets several maxima below existing runtime/API limits and should be revised rather than merged as-is.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Allow node media timeouts to cover recordings:** `src/agents/tools/nodes-tool.ts:114`
  - body: The new `timeoutMs` maximum is 120000, but the same schema allows `durationMs` up to 300000 for `camera_clip` and `screen_record`. Since current execution forwards `timeoutMs` to `node.invoke`, valid 5-minute recordings cannot request a sufficient gateway timeout.
  - confidence: 0.92
- **[P2] Keep sessions_list compatible with 200-row lookups:** `src/agents/tools/sessions-list-tool.ts:34`
  - body: Capping the tool schema at `maximum: 50` puts existing `sessions_list({ limit: 200 })` usage outside the advertised contract. Current session announce resolution still requests 200 rows, so larger installs can lose the rows needed to find a target.
  - confidence: 0.9
- **[P2] Do not cap spawned sessions at one hour:** `src/agents/tools/sessions-spawn-tool.ts:41`
  - body: The runtime and docs treat `runTimeoutSeconds` as any non-negative value, with `0` meaning no timeout, and pass explicit values through to child runs. Adding `maximum: 3600` makes legitimate longer background jobs invalid at the tool-schema level.
  - confidence: 0.9
- **[P2] Permit Discord GuildMedia channel type:** `src/agents/tools/message-tool.ts:329`
  - body: `discord-api-types` 0.38.47 defines `GuildMedia` immediately after `GuildForum`, making it channel type 16, and current Discord runtime handles it as forum-like. A `maximum: 15` blocks valid media-channel create requests.
  - confidence: 0.88
- **[P2] Keep browser snapshot limit at the backend cap:** `src/agents/tools/browser-tool.schema.ts:98`
  - body: Both current ARIA snapshot backends clamp limits at 2000 nodes, but this schema lowers the maximum to 1000. That removes supported large-page snapshot requests without a matching runtime contract change.
  - confidence: 0.86
