## Summary

Keep open: the hardening direction is useful, but this PR still adds schema caps that conflict with current tool/runtime contracts and would reject valid calls; it also lacks contributor real-behavior proof.

## What This Changes

The PR adds TypeBox descriptions plus maxItems, minimum, and maximum constraints across agent tool schemas for browser, canvas, gateway, memory, message, nodes, sessions, subagents, and web-fetch.

## Best Possible Solution

Rebase the hardening onto current main, keep only contract-backed schema bounds, update the active plugin-owned browser schema, and include redacted live gateway/model proof.

## Reproduction Assessment

Yes. Source inspection of the PR diff against current main shows specific schema caps that would reject valid node media, sessions_list, sessions_spawn, Discord GuildMedia, and browser snapshot calls.

## Solution Assessment

No. Schema hardening is reasonable, but this implementation hard-codes caps that conflict with current contracts; the safer solution is a rebased, contract-backed patch with real gateway/model proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Allow node timeouts to cover media durations:** `src/agents/tools/nodes-tool.ts:110-116`
  - body: The schema caps timeoutMs at 120000 ms while durationMs remains valid up to 300000 ms for node media. Valid five-minute camera_clip or screen_record calls cannot set a gateway timeout long enough to finish.
  - confidence: 0.92
- **[P2] Keep sessions_list compatible with 200-row callers:** `src/agents/tools/sessions-list-tool.ts:33-38`
  - body: Current code still calls sessions.list with limit: 200, and the gateway protocol only requires a positive integer. Adding maximum: 50 makes existing valid session enumeration calls fail schema validation.
  - confidence: 0.9
- **[P2] Preserve long-running session spawns:** `src/agents/tools/sessions-spawn-tool.ts:38-44`
  - body: The runtime accepts non-negative runTimeoutSeconds and timeoutSeconds, preserves 0 as no timeout, and forwards the value into ACP and native subagent execution. Capping the schema at one hour rejects legitimate long-running jobs.
  - confidence: 0.9
- **[P2] Permit Discord media channel type 16:** `src/agents/tools/message-tool.ts:329`
  - body: discord-api-types 0.38.47 includes GuildMedia after GuildForum, and the Discord runtime treats GuildMedia as forum-like. The new maximum: 15 rejects valid channel-create requests for that supported type.
  - confidence: 0.88
- **[P2] Restore browser snapshot limit to backend capacity:** `src/agents/tools/browser-tool.schema.ts:98`
  - body: The active browser schema is now under extensions/browser, and the snapshot backends clamp ARIA snapshots at 2000 nodes. Lowering the old schema path to 1000 both misses the active path after rebase and blocks supported larger snapshots.
  - confidence: 0.86
