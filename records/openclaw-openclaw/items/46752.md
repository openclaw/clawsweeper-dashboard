## Summary

Keep open: the schema-hardening direction is useful, but this PR is not merge-ready because several proposed caps conflict with current runtime and upstream contracts, the branch is conflicting, and the contributor has not provided real behavior proof.

## What This Changes

The PR adds TypeBox descriptions plus maxItems, minimum, and maximum constraints across agent tool schemas for browser, message, nodes, sessions, gateway, canvas, memory, subagents, and web-fetch tools.

## Best Possible Solution

Rebase the hardening onto current main, apply bounds only where they match current runtime or upstream contracts, move browser changes to the active browser plugin schema, and include redacted live gateway/model proof.

## Reproduction Assessment

Yes. Source inspection against current main and the PR diff is enough to show the proposed schema caps would reject valid node media, sessions_list, sessions_spawn, Discord GuildMedia, and browser snapshot calls.

## Solution Assessment

No. Schema hardening is a reasonable direction, but this implementation hard-codes caps that conflict with current contracts; the safer path is a rebased patch whose bounds are derived from runtime and upstream limits.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Allow node timeouts to cover media durations:** `src/agents/tools/nodes-tool.ts:110-116`
  - body: `durationMs` remains valid up to 300000 ms for node media actions, but this caps gateway `timeoutMs` at 120000 ms. Valid five-minute `camera_clip` or `screen_record` calls cannot set a timeout long enough to finish.
  - confidence: 0.92
- **[P2] Keep sessions_list compatible with 200-row callers:** `src/agents/tools/sessions-list-tool.ts:33-38`
  - body: Current code still calls `sessions.list` with `limit: 200`, and the gateway protocol only requires a positive integer. Adding `maximum: 50` makes existing valid session enumeration calls fail schema validation.
  - confidence: 0.9
- **[P2] Preserve long-running session spawns:** `src/agents/tools/sessions-spawn-tool.ts:38-44`
  - body: The runtime accepts non-negative `runTimeoutSeconds` and `timeoutSeconds` and forwards the resolved value into subagent and ACP spawning. Capping the schema at one hour rejects legitimate long-running jobs that currently pass through.
  - confidence: 0.9
- **[P2] Permit Discord media channel type 16:** `src/agents/tools/message-tool.ts:329`
  - body: Discord media channels are represented as `ChannelType.GuildMedia`, and the Discord runtime treats that type as forum-like. The new `maximum: 15` rejects valid channel-create requests for that type.
  - confidence: 0.88
- **[P2] Restore browser snapshot limit to backend capacity:** `src/agents/tools/browser-tool.schema.ts:98`
  - body: The active browser snapshot path forwards `limit` to backends that clamp at 2000 nodes, while current main serves the schema from the browser plugin. Lowering the old schema maximum to 1000 both misses the active path after rebase and blocks supported larger snapshots.
  - confidence: 0.86
