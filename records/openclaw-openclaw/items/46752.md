## Summary

Keep this PR open, but it is not merge-ready: the schema-hardening direction is useful, yet the current diff adds several caps below current runtime or dependency contracts and the contributor has not provided real after-fix gateway/model proof.

## What This Changes

The PR adds TypeBox `maxItems`, numeric maxima, minima, and richer descriptions across multiple agent tool schemas without changing execution logic.

## Best Possible Solution

Keep the hardening goal, rebase onto current main, move browser changes to the active plugin schema, derive each bound from current contracts, and add redacted live gateway/model proof.

## Reproduction Assessment

Yes. Source inspection of the PR diff against current main shows the proposed schemas would reject currently valid calls such as `sessions_list({ limit: 200 })`, `sessions_spawn({ runTimeoutSeconds: 7200 })`, Discord channel type `16`, and browser snapshots above 1000 nodes.

## Solution Assessment

No. Schema hardening is a reasonable direction, but this implementation is not the best fix because several caps conflict with existing contracts and the branch needs to be rebased onto the active browser plugin surface.

Full review comments:

- **[P1] Allow node timeouts to cover media durations:** `src/agents/tools/nodes-tool.ts:110-116`
  - body: `durationMs` remains valid up to 300000 ms for node media actions, but this caps the gateway `timeoutMs` at 120000 ms. Valid five-minute `camera_clip` or `screen_record` calls can no longer set a timeout long enough to finish.
  - confidence: 0.92
- **[P2] Keep sessions_list compatible with 200-row callers:** `src/agents/tools/sessions-list-tool.ts:33-38`
  - body: Current code still calls `sessions.list` with `limit: 200`, and the gateway protocol only requires a positive integer. Adding `maximum: 50` makes existing valid session enumeration calls fail schema validation.
  - confidence: 0.9
- **[P2] Preserve long-running session spawns:** `src/agents/tools/sessions-spawn-tool.ts:38-44`
  - body: The runtime accepts non-negative `runTimeoutSeconds`/`timeoutSeconds` and forwards the value to subagent and ACP spawning. Capping the schema at one hour rejects legitimate long-running jobs that currently pass through.
  - confidence: 0.9
- **[P2] Permit Discord media channel type 16:** `src/agents/tools/message-tool.ts:329`
  - body: The pinned Discord API types include `GuildMedia` after `GuildForum`, and the Discord runtime treats it as forum-like. The new `maximum: 15` rejects valid channel-create requests for that type.
  - confidence: 0.88
- **[P2] Restore browser snapshot limit to backend capacity:** `src/agents/tools/browser-tool.schema.ts:98`
  - body: Browser snapshot execution forwards `limit` to backends that clamp ARIA snapshots at 2000 nodes. Lowering the schema maximum to 1000 prevents agents from requesting the full supported snapshot size on large pages.
  - confidence: 0.86
