## Summary

Keep this PR open, but the current branch should not merge as-is. The security-hardening direction is valid and current main still has many unbounded tool-schema parameters, but static review against current main shows several proposed maxima are below supported runtime/docs behavior, and the browser changes target a stale core path instead of the active browser plugin schema. The linked #62 is already merged but does not supersede this schema-hardening work.

## What This Changes

The PR adds TypeBox descriptions plus maxItems and numeric min/max bounds across built-in agent tool schemas for browser, canvas, gateway, memory, message, nodes, sessions, subagents, and web_fetch.

## Best Possible Solution

Land a rebased hardening patch that encodes only contract-backed limits, applies browser bounds to the active browser plugin schema, preserves existing long-running/session/media workflows, and adds focused schema contract tests for each bounded tool surface.

## Reproduction Assessment

Yes. Static review reproduces the regressions by comparing the PR's proposed schema maxima with current-main runtime code, protocol schemas, docs, and regression tests for node recordings, sessions.list, sessions_spawn, browser snapshots, and Discord media threads.

## Solution Assessment

No. Schema-only hardening is a maintainable direction, but this branch is not the best implementation because several maxima are below supported behavior and one browser change targets a stale schema path instead of the active plugin-owned schema.
