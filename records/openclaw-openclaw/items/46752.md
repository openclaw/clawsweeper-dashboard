## Summary

Keep open, but not merge-ready: source inspection against current main confirms several proposed bounds conflict with existing runtime/API contracts, the browser schema moved to the browser plugin, and real behavior proof is missing.

## What This Changes

The PR adds TypeBox descriptions, maxItems, and numeric minimum/maximum bounds across agent tool schemas for browser, message, node, session, gateway, canvas, memory, and web-fetch tools.

## Best Possible Solution

Rebase the hardening onto current main, apply bounds only where they match current runtime or upstream contracts, move browser changes to the active browser plugin schema, and include redacted live gateway/model proof.

## Reproduction Assessment

Yes. Source inspection against current main is enough to show the PR schema caps would reject valid calls such as five-minute node media actions, sessions_list limit 200, long-running sessions_spawn jobs, Discord GUILD_MEDIA type 16, and 2000-node browser snapshots.

## Solution Assessment

No. Schema hardening is a reasonable direction, but this implementation hard-codes caps that conflict with current contracts; the safer solution is a rebased patch whose bounds are derived from runtime and upstream limits.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Allow node timeouts to cover media durations:** `src/agents/tools/nodes-tool.ts:110-116`
  - body: `durationMs` remains valid up to 300000 ms for node media actions, but this caps gateway `timeoutMs` at 120000 ms. Valid five-minute `camera_clip` or `screen_record` calls can no longer set a timeout long enough to finish.
  - confidence: 0.92
- **[P2] Keep sessions_list compatible with 200-row callers:** `src/agents/tools/sessions-list-tool.ts:33-38`
  - body: Current code still calls `sessions.list` with `limit: 200`, and the gateway protocol only requires a positive integer. Adding `maximum: 50` makes existing valid session enumeration calls fail schema validation.
  - confidence: 0.9
- **[P2] Preserve long-running session spawns:** `src/agents/tools/sessions-spawn-tool.ts:38-44`
  - body: The runtime accepts non-negative `runTimeoutSeconds` and `timeoutSeconds` and forwards the resolved value into subagent and ACP spawning. Capping the schema at one hour rejects legitimate long-running jobs that currently pass through.
  - confidence: 0.9
- **[P2] Permit Discord media channel type 16:** `src/agents/tools/message-tool.ts:329`
  - body: Discord documents `GUILD_MEDIA` as channel type 16, and the Discord runtime treats it as forum-like. The new `maximum: 15` rejects valid channel-create requests for that type. ([docs.discord.com](https://docs.discord.com/developers/resources/channel?utm_source=openai))
  - confidence: 0.88
- **[P2] Restore browser snapshot limit to backend capacity:** `src/agents/tools/browser-tool.schema.ts:98`
  - body: The active browser snapshot path forwards `limit` to backends that clamp at 2000 nodes, while current main serves the schema from the browser plugin. Lowering the old schema maximum to 1000 both misses the active path after rebase and blocks the supported larger snapshots.
  - confidence: 0.86
