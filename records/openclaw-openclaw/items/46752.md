## Summary

Keep open: the hardening goal is valid, but the current PR is not merge-ready because several proposed schema caps reject values that current main still accepts and uses, and the branch also lacks after-fix real behavior proof.

## What This Changes

The PR adds TypeBox descriptions, maxItems, and numeric minimum/maximum bounds across multiple agent tool schemas.

## Best Possible Solution

Rebase the hardening work onto current main, move browser changes to the active plugin schema, derive each bound from current contracts, and add redacted live gateway/model proof.

## Reproduction Assessment

Yes. Source inspection of the PR diff against current main shows the proposed schemas would reject currently valid calls such as five-minute node media actions, sessions_list with limit 200, long-running sessions_spawn, Discord channel type 16, and browser snapshots above 1000 nodes.

## Solution Assessment

No. Schema hardening is a reasonable direction, but this implementation is not the narrow maintainable fix because multiple caps conflict with existing contracts and the branch targets a stale browser path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Allow node timeouts to cover media durations:** `src/agents/tools/nodes-tool.ts:110-116`
  - body: durationMs remains valid up to 300000 ms for node media actions, but this caps the gateway timeoutMs at 120000 ms. Valid five-minute camera_clip or screen_record calls can no longer set a timeout long enough to finish.
  - confidence: 0.92
- **[P2] Keep sessions_list compatible with 200-row callers:** `src/agents/tools/sessions-list-tool.ts:33-38`
  - body: Current code still calls sessions.list with limit: 200, and the gateway protocol only requires a positive integer. Adding maximum: 50 makes existing valid session enumeration calls fail schema validation.
  - confidence: 0.9
- **[P2] Preserve long-running session spawns:** `src/agents/tools/sessions-spawn-tool.ts:38-44`
  - body: The runtime accepts non-negative runTimeoutSeconds and timeoutSeconds and forwards the resolved value into subagent and ACP spawning. Capping the schema at one hour rejects legitimate long-running jobs that currently pass through.
  - confidence: 0.9
- **[P2] Permit Discord media channel type 16:** `src/agents/tools/message-tool.ts:329`
  - body: The pinned Discord API types define GuildMedia as channel type 16, and the Discord runtime treats it as forum-like. The new maximum: 15 rejects valid channel-create requests for that type.
  - confidence: 0.88
- **[P2] Restore browser snapshot limit to backend capacity:** `src/agents/tools/browser-tool.schema.ts:98`
  - body: The active browser snapshot backends clamp ARIA snapshots at 2000 nodes. Lowering the schema maximum to 1000 prevents agents from requesting the full supported snapshot size on large pages.
  - confidence: 0.86
