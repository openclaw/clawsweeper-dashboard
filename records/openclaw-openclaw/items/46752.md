## Summary

Keep this PR open but do not merge it as-is: the schema-hardening direction is useful, but the current diff adds several caps below current runtime, docs, or dependency contracts and the external PR still lacks real behavior proof.

## What This Changes

The PR adds TypeBox `maxItems`, numeric maxima, and richer descriptions across several agent tool schemas for browser, message, node, sessions, memory, gateway, canvas, subagents, and web fetch tools.

## Best Possible Solution

Keep the schema-hardening goal, rebase onto current main, derive each bound from the live runtime/docs/provider contract, update the active browser plugin schema, and add redacted real gateway/model proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: the proposed schemas would reject currently valid calls such as `sessions_list({ limit: 200 })`, `sessions_spawn({ runTimeoutSeconds: 7200 })`, Discord channel type `16`, and browser snapshots above 1000 nodes.

## Solution Assessment

No. Schema hardening is a reasonable solution direction, but this implementation is not the narrowest maintainable fix because several caps are below current contracts and the branch needs to be rebased onto the active browser plugin surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Allow node timeouts to cover media durations:** `src/agents/tools/nodes-tool.ts:110-116`
  - body: `durationMs` is still valid up to 300000ms for node media actions, but this caps the gateway `timeoutMs` at 120000ms. Valid five-minute `camera_clip` or `screen_record` calls can no longer set a timeout long enough to finish.
  - confidence: 0.92
- **[P2] Keep sessions_list compatible with 200-row callers:** `src/agents/tools/sessions-list-tool.ts:33-38`
  - body: Current code still calls `sessions.list` with `limit: 200`, and the gateway protocol only requires a positive integer. Adding `maximum: 50` makes existing valid session enumeration calls fail schema validation.
  - confidence: 0.9
- **[P2] Preserve long-running session spawns:** `src/agents/tools/sessions-spawn-tool.ts:38-44`
  - body: The docs and runtime allow any non-negative `runTimeoutSeconds`, with `0` meaning no timeout. Capping the schema at one hour rejects legitimate long-running subagent and ACP jobs that currently pass through.
  - confidence: 0.9
- **[P2] Permit Discord media channel type 16:** `src/agents/tools/message-tool.ts:329`
  - body: Discord `GuildMedia` is channel type 16 in the pinned API types, and the Discord runtime treats it as a supported forum-like channel. The new `maximum: 15` rejects valid channel-create requests for that type.
  - confidence: 0.88
- **[P2] Restore browser snapshot limit to backend capacity:** `src/agents/tools/browser-tool.schema.ts:98`
  - body: Browser snapshot execution forwards `limit` to backends that clamp ARIA snapshots at 2000 nodes. Lowering the schema maximum to 1000 prevents agents from requesting the full supported snapshot size on large pages.
  - confidence: 0.86
