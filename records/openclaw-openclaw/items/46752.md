## Summary

Keep this PR open. The hardening goal is valid, but the branch is not merge-ready: several proposed schema maxima reject supported current-main workflows, and the browser change targets a schema path that current main no longer registers as the active browser plugin tool.

## What This Changes

The PR adds TypeBox descriptions plus maxItems and numeric bounds across built-in agent tool schemas for browser, canvas, gateway, memory, message, nodes, sessions, subagents, and web_fetch.

## Best Possible Solution

Land a rebased schema-hardening patch that encodes only contract-backed limits, updates the active browser plugin schema, preserves existing long-running/session/media workflows, and adds focused schema contract tests for every bounded tool surface.

## Reproduction Assessment

Yes. Static review reproduces the regressions by comparing the PR head schema maxima with current-main runtime code, docs, and tests for node recordings, sessions.list, sessions_spawn, browser snapshots, and Discord media threads.

## Solution Assessment

No. Schema-only hardening is a maintainable direction, but this branch is not the best implementation because several limits are below supported behavior and one browser change targets a stale path instead of the active plugin-owned schema.

  - body: The new timeoutMs maximum is below valid media durations: current main allows durationMs up to 300000ms and screen_record clamps to that value, so recordings over 120s cannot set a gateway timeout long enough to complete.
  - confidence: 0.93

  - body: Capping sessions_list.limit at 50 rejects an existing lookup path: announce-target resolution still asks sessions.list for 200 rows, and the gateway protocol accepts larger positive limits.
  - confidence: 0.9
- **[P2] Preserve long-running subagent timeouts:** `src/agents/tools/sessions-spawn-tool.ts:44`
  - body: The runtime and docs allow any finite non-negative runTimeoutSeconds, with 0 meaning no timeout. Adding maximum: 3600 rejects legitimate subagent or ACP jobs intended to run past an hour.
  - confidence: 0.91
- **[P2] Harden the active browser schema:** `src/agents/tools/browser-tool.schema.ts:98`
  - body: Current main registers the browser tool from extensions/browser, so editing the old core browser schema leaves the active surface unchanged. The proposed 1000 snapshot cap is also below the backend cap of 2000.
  - confidence: 0.84
- **[P2] Permit Discord media channel types:** `src/agents/tools/message-tool.ts:329`
  - body: The type maximum blocks Discord GuildMedia, which the current Discord runtime treats as forum-like and tests cover for thread creation. Valid media-channel thread behavior would fail schema validation.
  - confidence: 0.88
