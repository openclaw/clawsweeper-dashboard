  - confidence: 0.9

  - body: The runtime accepts non-negative `runTimeoutSeconds` and `timeoutSeconds`, with `0` preserving no-timeout behavior. Capping the schema at one hour would reject legitimate longer subagent jobs.
  - confidence: 0.9

  - body: Discord runtime handling already treats `GuildMedia` as forum-like, but the new schema maximum of 15 would reject supported channel-create calls for that channel type.
  - confidence: 0.87

  - confidence: 0.86
