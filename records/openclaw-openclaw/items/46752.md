## Summary

Keep this PR open, but it is not merge-ready: the schema-hardening goal is valid, yet the branch is conflicted and several proposed bounds reject tool calls that current main still supports.

## What This Changes

Adds TypeBox descriptions plus maxItems and numeric maximum/minimum bounds across agent tool schemas for browser, node, message, session, memory, canvas, gateway, and web fetch tools.

## Best Possible Solution

Rebase onto current main, keep only bounds that are backed by current runtime or upstream contracts, preserve existing valid ranges, add focused regression coverage, and include redacted real gateway/model-call proof.

## Reproduction Assessment

Yes. Source inspection of the PR diff against current main shows schema limits below active runtime contracts for node media, sessions_list, sessions_spawn, Discord channel types, and browser snapshots; no live run was performed because this was a read-only review.

## Solution Assessment

No. Schema hardening is a reasonable direction, but this implementation hard-codes incompatible limits and is stale against the active browser plugin path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Allow node timeouts to cover media durations:** `src/agents/tools/nodes-tool.ts:110-116`
  - body: The schema caps `timeoutMs` at 120000 ms while the same tool still permits `durationMs` up to 300000 ms. Valid five-minute `camera_clip` or `screen_record` calls would be unable to set a gateway timeout long enough to finish.
  - confidence: 0.92
- **[P2] Keep sessions_list compatible with 200-row lookups:** `src/agents/tools/sessions-list-tool.ts:33-38`
  - body: Current code still calls `sessions.list` with `limit: 200`, and the gateway contract only requires a positive integer. Adding `maximum: 50` would reject an existing valid session lookup path.
  - confidence: 0.9
- **[P2] Preserve long-running session spawns:** `src/agents/tools/sessions-spawn-tool.ts:38-45`
  - body: The runtime accepts non-negative `runTimeoutSeconds` and `timeoutSeconds`, with `0` preserving no-timeout behavior. Capping the schema at one hour would reject legitimate longer subagent jobs.
  - confidence: 0.9
- **[P2] Permit Discord media channel type 16:** `src/agents/tools/message-tool.ts:329`
  - body: Discord upstream types and the plugin runtime recognize `GuildMedia`, but the new schema maximum of 15 would reject supported channel-management calls using that channel type.
  - confidence: 0.84
- **[P2] Restore browser snapshot limit to backend capacity:** `src/agents/tools/browser-tool.schema.ts:99`
  - body: Current browser snapshot backends still accept larger snapshots than the proposed 1000-node schema maximum, and the active schema now lives under the browser plugin. Lowering the stale path would reject supported large snapshots and miss the current source of truth after rebase.
  - confidence: 0.86
