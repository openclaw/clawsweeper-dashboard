## Summary

Keep open: the hardening direction is useful, but this branch is not merge-ready because several proposed schema caps are below current runtime or upstream contracts and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds TypeBox descriptions plus numeric and array bounds across browser, canvas, gateway, memory, message, nodes, sessions, subagents, and web fetch tool schemas.

## Best Possible Solution

Rebase the hardening onto current main, keep only bounds backed by active runtime or upstream contracts, update active plugin paths, and include redacted live gateway/model proof.

## Reproduction Assessment

Yes. Source inspection of the PR diff against current main shows the proposed caps would reject valid node media, sessions_list, sessions_spawn, Discord GuildMedia, and browser snapshot calls.

## Solution Assessment

No. Schema hardening is the right direction, but this implementation hard-codes several limits below current contracts; the safer path is a rebased, contract-backed set of bounds with real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Allow node timeouts to cover media durations:** `src/agents/tools/nodes-tool.ts:110-116`
  - body: The schema caps timeoutMs at 120000 ms while durationMs remains valid up to 300000 ms for node media. Valid five-minute camera_clip or screen_record calls cannot set a gateway timeout long enough to finish.
  - confidence: 0.92
- **[P2] Keep sessions_list compatible with 200-row callers:** `src/agents/tools/sessions-list-tool.ts:33-38`
  - body: Current code still calls sessions.list with limit: 200, and the gateway protocol only requires a positive integer. Adding maximum: 50 makes existing valid session enumeration calls fail schema validation.
  - confidence: 0.9
- **[P2] Preserve long-running session spawns:** `src/agents/tools/sessions-spawn-tool.ts:38-44`
  - body: The runtime accepts non-negative runTimeoutSeconds and timeoutSeconds, with 0 meaning no timeout. Capping the schema at one hour rejects legitimate longer subagent jobs.
  - confidence: 0.9
- **[P2] Permit Discord media channel type 16:** `src/agents/tools/message-tool.ts:329`
  - body: discord-api-types defines GuildMedia after GuildForum, and the Discord runtime already treats it as forum-like. The new maximum: 15 rejects valid channel-create requests for that supported type.
  - confidence: 0.88
- **[P2] Restore browser snapshot limit to backend capacity:** `src/agents/tools/browser-tool.schema.ts:98`
  - body: The active browser schema now lives under extensions/browser, and snapshot backends still clamp ARIA snapshots at 2000 nodes. Lowering the old schema path to 1000 misses current main and blocks supported larger snapshots.
  - confidence: 0.86
