## Summary

Keep open: the hardening goal is still useful and current main has not implemented it, but this PR is not merge-ready because several new schema caps are below current runtime or documented contracts and the browser tool path is stale after the plugin move.

## What This Changes

The PR adds TypeBox bounds and descriptions to multiple agent tool schemas for browser, canvas, gateway, memory, message, nodes, sessions, subagents, and web fetch parameters.

## Best Possible Solution

Rebase the hardening work onto current main and land only schema bounds that match active runtime, dependency, and documented limits.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: the PR schema would reject currently valid values such as browser snapshot limit 2000, sessions_list limit 200, GuildMedia channel type 16, and long-running sessions_spawn timeouts. I did not need a live gateway run to establish those schema-contract mismatches.

## Solution Assessment

No, not as-is. Schema hardening is the right direction, but this PR needs a rebase and bound fixes so it preserves current runtime contracts instead of narrowing them accidentally.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Allow node timeout to cover max recording durations:** `src/agents/tools/nodes-tool.ts:114`
  - body: This cap makes valid long node media actions fail before completion: the same schema allows recording durationMs up to 300000 ms, but timeoutMs is capped at 120000 ms, so callers cannot set a sufficient RPC timeout for five-minute camera_clip or screen_record requests.
  - confidence: 0.91
- **[P2] Restore browser snapshot limit to the backend cap:** `src/agents/tools/browser-tool.schema.ts:98`
  - body: The browser snapshot implementations still clamp to 2000 nodes, but the schema now rejects limit values above 1000. Large-page snapshot requests that the backend can serve would fail schema validation before reaching the browser tool.
  - confidence: 0.9
- **[P2] Keep sessions_list limit aligned with 200-row callers:** `src/agents/tools/sessions-list-tool.ts:34`
  - body: The new maximum of 50 rejects existing sessions_list calls with limit 200, including current internal session-resolution code. Larger installs can no longer enumerate enough sessions to find older targets.
  - confidence: 0.89
- **[P2] Do not cap sessions_spawn runs at one hour:** `src/agents/tools/sessions-spawn-tool.ts:41`
  - body: The runtime accepts any non-negative runTimeoutSeconds and treats 0 as no timeout. Adding maximum 3600 turns legitimate long-running spawned sessions into schema-validation failures.
  - confidence: 0.88
- **[P2] Permit Discord media channel types:** `src/agents/tools/message-tool.ts:329`
  - body: Discord runtime code already handles GuildMedia as a supported forum-like channel, but this maximum excludes that enum value. Channel-management calls for supported media channels would be rejected at schema validation.
  - confidence: 0.86
