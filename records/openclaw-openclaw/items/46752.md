## Summary

Keep open: the hardening direction is useful, but the current branch rejects valid existing tool calls and lacks the required after-fix real behavior proof.

## What This Changes

The PR adds TypeBox schema descriptions and upper bounds to agent tool parameters across browser, message, node, session, gateway, memory, canvas, subagent, and web-fetch tools.

## Best Possible Solution

Keep the schema hardening, but rebase it onto current main and derive every bound from current runtime constants, gateway contracts, or upstream API types while preserving existing valid long-run and large-result workflows.

## Reproduction Assessment

Yes. The blocking regressions are reproducible at source level because the proposed schema maximums are lower than current main callers, backends, and dependency types; I did not run a live gateway in this read-only review.

## Solution Assessment

No. Schema hardening is the right direction, but this patch should not merge until the branch is rebased and the caps are aligned with existing runtime/API contracts instead of lower ad hoc values.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Allow node timeouts to cover valid recordings:** `src/agents/tools/nodes-tool.ts:110-114`
  - body: The schema caps `timeoutMs` at 120000 ms while `durationMs` in the same tool can be 300000 ms. Valid `screen_record` or `camera_clip` calls longer than about two minutes cannot request enough gateway timeout and can fail before the allowed recording completes.
  - confidence: 0.94
- **[P2] Keep sessions_list limit aligned with runtime callers:** `src/agents/tools/sessions-list-tool.ts:34-38`
  - body: Capping `sessions_list.limit` at 50 rejects existing 200-row discovery calls used for announce-target resolution. Larger installs can no longer enumerate enough visible sessions to find older targets.
  - confidence: 0.92
- **[P2] Do not cap spawned session runs at one hour:** `src/agents/tools/sessions-spawn-tool.ts:41-45`
  - body: `runTimeoutSeconds` and its `timeoutSeconds` alias currently accept any non-negative value, with 0 preserving no-timeout behavior. Adding `maximum: 3600` turns legitimate long-running spawned sessions into schema-invalid calls.
  - confidence: 0.9
- **[P2] Keep browser snapshot limits at the backend cap:** `src/agents/tools/browser-tool.schema.ts:98-99`
  - body: The active browser snapshot backends still accept up to 2000 aria nodes, but the PR lowers the old agent schema maximum to 1000. That blocks large-page snapshots that current backends can serve, and the branch also needs rebasing onto the moved `extensions/browser` schema.
  - confidence: 0.88
- **[P2] Permit all supported Discord channel types:** `src/agents/tools/message-tool.ts:329`
  - body: The proposed `type` maximum of 15 excludes `ChannelType.GuildMedia` while the Discord runtime already treats GuildMedia as supported forum-like behavior. Channel-create requests for that valid type would be rejected at the schema layer.
  - confidence: 0.87
