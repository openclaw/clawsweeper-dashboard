## Summary

Keep this PR open, but it is not merge-ready. The schema-hardening goal remains useful, while several proposed bounds are below current runtime or dependency contracts and would reject valid node recording, session listing, subagent, browser snapshot, and Discord media-channel workflows.

## What This Changes

The PR adds TypeBox `maxItems`, numeric bounds, and schema descriptions across built-in agent tools for browser, canvas, gateway, memory, message, nodes, sessions, subagents, and web fetch.

## Best Possible Solution

Land a rebased or replacement schema-hardening patch that encodes only contract-backed limits in the active core and plugin schemas, preserves existing workflows, and adds focused schema contract tests.

## Reproduction Assessment

Yes. Static review against current main reproduces the regressions by comparing the PR's schema maxima with current runtime code, dependency types, docs, and tests for node recordings, sessions, browser snapshots, subagent timeouts, and Discord media channels.

## Solution Assessment

No. Schema-only hardening is a maintainable direction, but this branch is not the best implementation because several limits are too tight and one browser change targets a stale path instead of the active plugin-owned schema.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Raise node timeouts to cover recordings:** `src/agents/tools/nodes-tool.ts:113-117`
  - body: The new `timeoutMs` maximum is below valid media durations: current main allows `durationMs` up to 300000ms and `screen_record` clamps to that value, so recordings over 120s cannot set a gateway timeout long enough to complete.
  - confidence: 0.93
- **[P2] Keep sessions_list compatible with 200-row lookups:** `src/agents/tools/sessions-list-tool.ts:36-40`
  - body: Capping `sessions_list.limit` at 50 rejects an existing lookup path: announce-target resolution still asks `sessions.list` for 200 rows, and the gateway protocol accepts larger positive limits.
  - confidence: 0.9
- **[P2] Preserve long-running subagent timeouts:** `src/agents/tools/sessions-spawn-tool.ts:44-48`
  - body: The runtime and docs allow finite non-negative `runTimeoutSeconds`, with 0 meaning no timeout. Adding `maximum: 3600` rejects legitimate subagent or ACP jobs intended to run past an hour.
  - confidence: 0.91
- **[P2] Harden the active browser schema:** `src/agents/tools/browser-tool.schema.ts:98-101`
  - body: Current main registers the browser tool from `extensions/browser`, so editing the old core browser schema leaves the active surface unchanged. The proposed 1000 snapshot cap is also below the backend cap of 2000.
  - confidence: 0.84
- **[P2] Permit Discord media channel types:** `src/agents/tools/message-tool.ts:329`
  - body: The `type` maximum blocks Discord `GuildMedia` channels, which `discord-api-types@0.38.47` defines as 16 and the current Discord runtime treats as forum-like for thread creation.
  - confidence: 0.88
