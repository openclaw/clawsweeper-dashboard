## Summary

Keep this PR open, but it is not merge-ready. The schema-hardening direction is useful, yet the current branch conflicts with main and several proposed bounds reject currently supported tool calls or miss the active browser tool surface.

## What This Changes

The PR adds TypeBox `maxItems`, numeric `minimum`/`maximum` bounds, and tool-parameter descriptions across built-in agent tool schemas.

## Best Possible Solution

Land a rebased schema-hardening patch that encodes only contract-backed limits, updates the active plugin-owned browser schema, preserves existing long-running/session/media workflows, and adds focused schema contract coverage.

## Reproduction Assessment

Yes. Static source review against current main reproduces the PR regressions by comparing the proposed schema maxima with runtime code, dependency types, docs, and tests for node recordings, session lookup, subagent timeouts, browser snapshots, and Discord media channels.

## Solution Assessment

No. Schema-only hardening is a good direction, but this branch is not the best implementation because several maxima are below current contracts and one browser change targets a stale path instead of the active plugin schema.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Raise node timeouts to cover recordings:** `src/agents/tools/nodes-tool.ts:113-117`
  - body: The new `timeoutMs` maximum is below valid media durations: current main allows `durationMs` up to 300000ms and tests assert `screen_record` clamps to that value, so recordings over 120s cannot set a gateway timeout long enough to complete.
  - confidence: 0.93
- **[P2] Keep sessions_list compatible with 200-row lookups:** `src/agents/tools/sessions-list-tool.ts:36-40`
  - body: Capping `sessions_list.limit` at 50 rejects an existing lookup path: announce-target resolution still asks `sessions.list` for 200 rows, and the gateway protocol accepts larger positive limits.
  - confidence: 0.9
- **[P2] Preserve long-running subagent timeouts:** `src/agents/tools/sessions-spawn-tool.ts:44-48`
  - body: The runtime and docs allow finite non-negative `runTimeoutSeconds`, with 0 meaning no timeout. Adding `maximum: 3600` rejects legitimate subagent or ACP jobs intended to run past an hour.
  - confidence: 0.91
- **[P2] Harden the active browser schema:** `src/agents/tools/browser-tool.schema.ts:98-101`
  - body: Current main registers the browser tool from `extensions/browser`, so editing the old core browser schema leaves the active surface unchanged. The proposed 1000 snapshot cap is also below the backend cap of 2000.
  - confidence: 0.84
- **[P2] Permit Discord media channel types:** `src/agents/tools/message-tool.ts:329`
  - body: The `type` maximum blocks Discord `GuildMedia`, which the current Discord runtime treats as forum-like and tests cover for thread creation. Valid media-channel thread behavior would fail schema validation.
  - confidence: 0.88
