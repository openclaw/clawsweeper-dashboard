## Summary

Keep this PR open, but it is not merge-ready: the schema-hardening direction is useful, while the current branch is conflicted, lacks real behavior proof, and proposes bounds that would reject valid current tool calls.

## What This Changes

The PR adds TypeBox descriptions plus maxItems and numeric minimum/maximum constraints across agent tool schemas for browser, node, message, session, memory, canvas, gateway, and web fetch tools.

## Best Possible Solution

Rebase onto current main, keep only schema bounds backed by current runtime or upstream contracts, preserve existing valid ranges, add focused regression coverage, and include redacted real gateway/model-call proof.

## Reproduction Assessment

Yes. Source inspection of the PR head against current main shows schema limits below active runtime, docs, and upstream contracts for node media, sessions_list, sessions_spawn, Discord channel types, and browser snapshots.

## Solution Assessment

No. Schema hardening is a reasonable direction, but this implementation hard-codes incompatible limits and targets a stale browser schema path; the safer fix is contract-aligned bounds after rebase.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Allow node timeouts to cover media durations:** `src/agents/tools/nodes-tool.ts:110-116`
  - body: The schema caps `timeoutMs` at 120000 ms while the same tool permits `durationMs` up to 300000 ms. Valid five-minute `camera_clip` or `screen_record` calls would be unable to set a gateway timeout long enough to finish.
  - confidence: 0.92
- **[P2] Keep sessions_list compatible with 200-row lookups:** `src/agents/tools/sessions-list-tool.ts:33-39`
  - body: Current code still calls `sessions.list` with `limit: 200`, and the Gateway contract only requires a positive integer. Adding `maximum: 50` would reject an existing valid session lookup path.
  - confidence: 0.9
- **[P2] Preserve long-running session spawns:** `src/agents/tools/sessions-spawn-tool.ts:41-55`
  - body: The runtime accepts non-negative `runTimeoutSeconds` and `timeoutSeconds`, with `0` preserving no-timeout behavior. Capping the schema at one hour would reject legitimate longer subagent jobs.
  - confidence: 0.9
- **[P2] Permit Discord media channel type 16:** `src/agents/tools/message-tool.ts:329`
  - body: The Discord API types and the plugin runtime recognize `GuildMedia`, but the new schema maximum of 15 would reject supported channel-management calls using that channel type.
  - confidence: 0.84
- **[P2] Restore browser snapshot limit to backend capacity:** `src/agents/tools/browser-tool.schema.ts:98`
  - body: Current browser snapshot backends still accept up to 2000 nodes, and the active schema now lives under `extensions/browser`. Lowering the stale path to 1000 would both miss the current source of truth and reject supported large snapshots after rebase.
  - confidence: 0.86
