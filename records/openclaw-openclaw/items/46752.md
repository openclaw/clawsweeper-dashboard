## Summary

Keep this PR open, but it is not merge-ready: the schema-hardening direction is useful, yet the branch is dirty against current main and several proposed maxima conflict with current runtime contracts.

## What This Changes

The PR adds TypeBox bounds and descriptions to built-in agent tool schemas for browser, canvas, gateway, memory, message, nodes, sessions, subagents, and web fetch parameters.

## Best Possible Solution

Land a rebased schema-hardening patch that encodes only contract-backed limits, updates the active plugin-owned browser schema, preserves existing long-running/session/media workflows, and adds focused schema contract coverage.

## Reproduction Assessment

Yes. Static review against current main reproduces the proposed regressions by comparing the PR's schema maxima with runtime code, dependency types, and existing tests for nodes, sessions, browser snapshots, and Discord media channels.

## Solution Assessment

No. Schema-only hardening is a maintainable direction, but this branch is not the best implementation because several bounds are too tight and one browser change targets a stale core path instead of the active plugin schema.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Raise node timeouts to cover recordings:** `src/agents/tools/nodes-tool.ts:113-117`
  - body: The new timeoutMs maximum is below valid media durations: current main allows durationMs up to 300000ms and clamps screen_record to that value, so five-minute recordings cannot set a gateway timeout long enough to finish.
  - confidence: 0.93
- **[P2] Keep sessions_list compatible with 200-row lookups:** `src/agents/tools/sessions-list-tool.ts:36-40`
  - body: Capping sessions_list.limit at 50 rejects an existing lookup path: announce-target resolution still asks sessions.list for 200 rows, and the gateway protocol accepts larger positive limits.
  - confidence: 0.9
- **[P2] Preserve long-running subagent timeouts:** `src/agents/tools/sessions-spawn-tool.ts:44-48`
  - body: The runtime still accepts finite non-negative runTimeoutSeconds/timeoutSeconds values and passes them through to child runs. Adding maximum: 3600 turns legitimate longer subagent or ACP jobs into schema-validation failures.
  - confidence: 0.9
- **[P2] Move browser hardening to the active plugin schema:** `src/agents/tools/browser-tool.schema.ts:98-101`
  - body: Current main registers the browser tool from extensions/browser, while this PR edits the old core browser schema path. The proposed snapshot cap is also below the active backends' 2000-node clamp.
  - confidence: 0.87
- **[P2] Allow Discord media channel types:** `src/agents/tools/message-tool.ts:329`
  - body: The type maximum blocks Discord GuildMedia channels, whose enum value is 16 and which the current Discord runtime/tests treat as forum-like for thread creation.
  - confidence: 0.88
