The PR adds TypeBox descriptions plus array and numeric bounds across agent tool schemas for browser, nodes, message, sessions, memory, canvas, gateway, and web-fetch.

## Best Possible Solution

Rebase onto current main, move browser schema changes to the active plugin path, keep only contract-backed bounds, add focused regression coverage, and include redacted real gateway/model-call proof.

## Reproduction Assessment

Yes. Source inspection of the PR diff against current main shows schema limits below active runtime, docs, and plugin contracts for node media, sessions_list, sessions_spawn, Discord channel types, and browser snapshots.

## Solution Assessment

No. Schema hardening is a reasonable direction, but this implementation hard-codes incompatible limits and targets stale browser schema placement; the maintainable fix is a rebased, contract-aligned patch with real behavior proof.

  - body: The PR caps `timeoutMs` at 120000 ms while `durationMs` remains valid up to 300000 ms, so five-minute media actions cannot set a sufficient gateway timeout and can fail before completion.
  - confidence: 0.92
- **[P2] Keep sessions_list compatible with 200-row lookups:** `src/agents/tools/sessions-list-tool.ts:33-39`
  - body: Current code still issues `sessions.list` with `limit: 200`, and the gateway contract only requires a positive integer. Adding `maximum: 50` would reject an existing valid lookup path.
  - confidence: 0.9
- **[P2] Preserve long-running session spawns:** `src/agents/tools/sessions-spawn-tool.ts:41-55`
  - body: The runtime accepts non-negative `runTimeoutSeconds` and `timeoutSeconds`, with `0` preserving no-timeout behavior. Capping the schema at one hour rejects legitimate longer subagent jobs.
  - confidence: 0.9
- **[P2] Permit Discord media channel type 16:** `src/agents/tools/message-tool.ts:329`
  - body: The Discord runtime recognizes `GuildMedia`, but the new schema maximum of 15 would reject supported channel-management calls using that channel type.
  - confidence: 0.84
- **[P2] Move browser bounds to the active schema:** `src/agents/tools/browser-tool.schema.ts:98`
  - body: Current main's browser schema lives under `extensions/browser`, and the PR lowers the stale core-path snapshot limit to 1000 even though the active backend still accepts larger snapshots.
  - confidence: 0.86
