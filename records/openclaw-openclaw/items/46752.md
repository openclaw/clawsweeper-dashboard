## Summary

Keep open: the schema-hardening goal is useful and current main has not generally implemented it, but this branch is not merge-ready because several new bounds are narrower than current runtime contracts and the external PR lacks after-fix real behavior proof.

## What This Changes

The PR adds TypeBox `minimum`, `maximum`, `maxItems`, and descriptive metadata to agent tool schemas across browser, message, nodes, sessions, gateway, memory, canvas, subagent, and web-fetch tools.

## Best Possible Solution

Rebase onto current main and keep the hardening, but derive every new schema bound from existing runtime/API contracts while preserving 2000-node browser snapshots, 200-row session discovery, no-timeout spawned runs, Discord GuildMedia, and node recording timeout coverage.

## Reproduction Assessment

Yes, for the blocking PR regressions at source level: the proposed schema caps are lower than current runtime contracts and internal callers. I did not run a live gateway because this is a read-only review, but the affected source paths are direct.

## Solution Assessment

No. Schema hardening is the right direction, but this patch should not merge until the bounds are rebased and aligned with runtime/API limits instead of lower ad hoc caps.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Allow node timeouts to cover valid recordings:** `src/agents/tools/nodes-tool.ts:110-114`
  - body: The schema now caps `timeoutMs` at 120000 ms while `durationMs` in the same tool can be 300000 ms. Valid `screen_record` or `camera_clip` calls longer than about two minutes cannot request enough gateway timeout and will fail before the allowed recording duration completes.
  - confidence: 0.94
- **[P2] Keep sessions_list limit aligned with runtime callers:** `src/agents/tools/sessions-list-tool.ts:34-38`
  - body: Capping `sessions_list.limit` at 50 rejects existing 200-row discovery calls; current main still uses `sessions.list` with `limit: 200` for announce-target resolution. Larger installs can no longer enumerate enough visible sessions to find older targets.
  - confidence: 0.92
- **[P2] Do not cap spawned session runs at one hour:** `src/agents/tools/sessions-spawn-tool.ts:41-45`
  - body: `runTimeoutSeconds` and its `timeoutSeconds` alias currently accept any non-negative value, with 0 preserving no-timeout behavior. Adding `maximum: 3600` turns legitimate long-running spawned sessions into schema-invalid calls.
  - confidence: 0.9
- **[P2] Keep browser snapshot limit at the backend cap:** `src/agents/tools/browser-tool.schema.ts:98-99`
  - body: The browser snapshot implementation still clamps aria snapshot limits to 2000 nodes, but the PR lowers the agent schema maximum to 1000. That blocks legitimate large-page snapshots that the current backends can serve.
  - confidence: 0.88
- **[P2] Permit all supported Discord channel types:** `src/agents/tools/message-tool.ts:329`
  - body: The proposed `type` maximum of 15 excludes newer Discord channel types such as `GuildMedia`, while the Discord runtime already treats GuildMedia as supported forum-like behavior. Channel-create requests for that valid type would be rejected at the schema layer.
  - confidence: 0.87
