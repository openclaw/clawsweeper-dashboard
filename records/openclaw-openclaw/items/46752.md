## Summary

Keep open: the hardening direction is useful, but this PR should not merge because several proposed bounds reject valid current-main tool calls, and the external PR only has build/test/format proof rather than after-fix real behavior proof.

## What This Changes

The PR adds TypeBox descriptions plus `minimum`, `maximum`, and `maxItems` schema bounds across browser, canvas, gateway, memory, message, nodes, sessions, subagents, and web-fetch tools.

## Best Possible Solution

Rebase onto current main and keep the schema-hardening goal, but derive each bound from existing runtime/API contracts and preserve browser 2000-node snapshots, 200-row session discovery, long-running spawned sessions, Discord GuildMedia, and node recording timeout coverage.

## Reproduction Assessment

Yes. The blocking regressions are source-reproducible by comparing the PR's schema caps with current-main runtime contracts and internal callers for nodes, sessions_list, sessions_spawn, browser snapshots, and Discord GuildMedia handling.

## Solution Assessment

No. Schema hardening is the right direction, but this patch is not the best implementation until the caps are rebased and aligned with current runtime/API limits rather than lower ad hoc maxima.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Allow node timeouts to cover valid recordings:** `src/agents/tools/nodes-tool.ts:110-114`
  - body: The schema caps `timeoutMs` at 120000 ms while the same tool allows recording `durationMs` up to 300000 ms, so valid `screen_record` or `camera_clip` calls cannot request enough gateway timeout to complete.
  - confidence: 0.94
- **[P2] Keep sessions_list limit aligned with 200-row callers:** `src/agents/tools/sessions-list-tool.ts:34-38`
  - body: Capping `sessions_list.limit` at 50 rejects existing 200-row session discovery calls used for announce-target resolution, so larger installs can fail to find older visible sessions.
  - confidence: 0.92
- **[P2] Do not cap spawned session runs at one hour:** `src/agents/tools/sessions-spawn-tool.ts:41-45`
  - body: `runTimeoutSeconds` and its `timeoutSeconds` alias currently accept any non-negative value, with 0 preserving no-timeout behavior; adding `maximum: 3600` makes legitimate longer spawned sessions schema-invalid.
  - confidence: 0.9
- **[P2] Keep browser snapshot limit at the backend cap:** `src/agents/tools/browser-tool.schema.ts:98-99`
  - body: The browser snapshot implementation still accepts up to 2000 nodes, but the PR lowers the tool schema maximum to 1000, blocking large-page snapshots the backend can serve.
  - confidence: 0.88
- **[P2] Permit all supported Discord channel types:** `src/agents/tools/message-tool.ts:329`
  - body: The proposed `type` maximum of 15 excludes supported newer Discord channel types such as GuildMedia, which the runtime already treats as forum-like behavior.
  - confidence: 0.87
