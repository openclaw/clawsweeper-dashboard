## Summary

Keep open: the schema-hardening direction is useful, but this PR is not merge-ready because several proposed bounds are lower than current runtime contracts and the branch is conflicted against current main.

## What This Changes

The PR adds TypeBox `maxItems`, `maximum`, and descriptive schema metadata across agent tool schemas for browser, canvas, gateway, memory, message, nodes, sessions, subagents, and web fetch parameters.

## Best Possible Solution

Rebase or replace the PR against current main, then land only schema bounds that match active runtime, dependency, and documented limits.

## Reproduction Assessment

Yes. Source inspection against current main plus the PR diff gives a high-confidence path: the PR schema would reject valid values such as node media timeouts over 120s, browser snapshot limit 2000, sessions_list limit 200, long sessions_spawn timeouts, and Discord GuildMedia type 16.

## Solution Assessment

No, not as-is. Schema hardening is the right direction, but the fix needs to preserve current contracts and apply to the current browser plugin schema path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Allow node timeouts to cover max recording durations:** `src/agents/tools/nodes-tool.ts:114`
  - body: The schema allows `durationMs` up to 300000 ms for media actions, but this new `timeoutMs` cap is only 120000 ms. Callers requesting valid five-minute `camera_clip` or `screen_record` work cannot set a sufficient gateway timeout and would fail before completion.
  - confidence: 0.91
- **[P2] Restore browser snapshot limit to the backend cap:** `src/agents/tools/browser-tool.schema.ts:98`
  - body: The active browser snapshot backends still clamp and serve up to 2000 nodes, but the PR caps the tool schema at 1000. Large-page snapshot requests that the implementation can handle would be rejected by schema validation; after rebase this also needs to target `extensions/browser/src/browser-tool.schema.ts`.
  - confidence: 0.9
- **[P2] Keep sessions_list limit aligned with 200-row callers:** `src/agents/tools/sessions-list-tool.ts:34`
  - body: The new maximum of 50 rejects existing `sessions_list` calls with `limit: 200`, including current internal session-resolution code. Larger installs could no longer enumerate enough sessions to find older targets.
  - confidence: 0.89
- **[P2] Do not cap sessions_spawn runs at one hour:** `src/agents/tools/sessions-spawn-tool.ts:41`
  - body: Current runtime accepts any non-negative `runTimeoutSeconds` and forwards it to the child run, with `0` preserving no-timeout behavior. Adding `maximum: 3600` would make legitimate long-running spawned sessions fail schema validation.
  - confidence: 0.88
- **[P2] Permit Discord media channel types:** `src/agents/tools/message-tool.ts:329`
  - body: Discord runtime handling already treats `GuildMedia` as a supported forum-like channel, but this maximum excludes that enum value. Channel-management calls for supported media channels would be rejected before reaching the provider.
  - confidence: 0.86
