## Summary

Keep open: the hardening direction is useful, but the current branch is not merge-ready because several new schema bounds reject values that current runtime contracts still accept, the branch is conflicted, and the external PR only provides build/test proof rather than real behavior proof.

## What This Changes

The branch adds TypeBox maxItems, minimum, maximum, and description metadata to agent tool schemas across browser, message, nodes, sessions, gateway, canvas, memory, subagents, and web fetch tools.

## Best Possible Solution

Keep the hardening, but rebase it onto current main and only add schema bounds that match existing runtime clamps or upstream API limits, with regression coverage for the previously valid edge values.

## Reproduction Assessment

Yes, for the review findings there is a high-confidence source reproduction: the PR adds schema maximums that reject values current main still accepts, such as browser snapshot limit 2000, sessions_list limit 200, and runTimeoutSeconds values above 3600. I did not run the PR branch because this review was read-only.

## Solution Assessment

No. Schema hardening is the right direction, but this implementation needs to align each bound with the runtime or upstream contract before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Allow node timeout to cover recording duration:** `src/agents/tools/nodes-tool.ts:114`
  - body: The schema allows media recordings up to 300000 ms via durationMs, but caps timeoutMs at 120000 ms. A valid 5-minute camera_clip or screen_record request can no longer set a long enough gateway timeout and will time out before the accepted action can complete.
  - confidence: 0.93
- **[P2] Permit Discord media channel type:** `src/agents/tools/message-tool.ts:329`
  - body: Capping channel type at 15 rejects Discord GuildMedia channels, whose enum value is 16 in discord-api-types and which the Discord runtime already treats as forum-like. This blocks legitimate channel-create calls that previously passed through to Discord.
  - confidence: 0.91
- **[P2] Keep browser snapshot limit at the backend cap:** `src/agents/tools/browser-tool.schema.ts:99`
  - body: The browser snapshot action forwards limit to the snapshot backends, and current backends clamp ARIA snapshots at 2000 nodes. Lowering the tool schema maximum to 1000 rejects valid large-page snapshot requests before the backend can serve them.
  - confidence: 0.9
- **[P2] Keep sessions_list compatible with 200-row callers:** `src/agents/tools/sessions-list-tool.ts:38`
  - body: The new maximum of 50 rejects existing sessions_list({ limit: 200 }) use, including internal announce-target lookup. Larger installs can fail to find older or less-active sessions that the current gateway contract still allows.
  - confidence: 0.92
- **[P2] Do not cap sessions_spawn runs at one hour:** `src/agents/tools/sessions-spawn-tool.ts:45`
  - body: The runtime treats runTimeoutSeconds and timeoutSeconds as any non-negative value, with 0 preserving no-timeout behavior. Adding maximum: 3600 makes legitimate long-running spawned sessions fail schema validation before execution.
  - confidence: 0.9
