## Summary

This PR should stay open because the proposed schema caps still conflict with current `main` contracts for node media, session listing/spawn, Discord channel types, and browser snapshots, and the external PR still lacks real behavior proof for the changed tool-calling path.

## What This Changes

The PR adds TypeBox descriptions plus `maxItems`, `maximum`, and related bounds across browser, canvas, gateway, memory, message, nodes, sessions, subagents, and web fetch tool schemas.

## Best Possible Solution

Rebase the hardening onto current `main`, keep only bounds backed by active runtime or upstream contracts, update the active plugin/schema paths, and include redacted real gateway or model-call proof.

## Reproduction Assessment

Yes. Source inspection of the PR diff against current `main` shows proposed maximums below currently valid node media, `sessions_list`, `sessions_spawn`, Discord GuildMedia, and browser snapshot calls.

## Solution Assessment

No. Schema hardening is a reasonable security direction, but this implementation hard-codes bounds that do not match current contracts; the safer path is a rebased, contract-backed bounds set with real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Allow node timeouts to cover media durations:** `src/agents/tools/nodes-tool.ts:110-116`
  - body: The schema caps `timeoutMs` at 120000 ms while node media `durationMs` remains valid up to 300000 ms. Valid five-minute `camera_clip` or `screen_record` calls cannot set a gateway timeout long enough to finish.
  - confidence: 0.92
- **[P2] Keep sessions_list compatible with 200-row callers:** `src/agents/tools/sessions-list-tool.ts:33-38`
  - body: Current code still calls `sessions.list` with `limit: 200`, and the gateway/session contract only requires a positive integer. Adding `maximum: 50` makes existing valid session enumeration calls fail schema validation.
  - confidence: 0.9
- **[P2] Preserve long-running session spawns:** `src/agents/tools/sessions-spawn-tool.ts:38-44`
  - body: The runtime accepts non-negative `runTimeoutSeconds` and `timeoutSeconds`, with `0` meaning no timeout. Capping the schema at one hour rejects legitimate longer subagent jobs.
  - confidence: 0.9
- **[P2] Permit Discord media channel type 16:** `src/agents/tools/message-tool.ts:329`
  - body: The Discord runtime treats `GuildMedia` as forum-like, but the new schema maximum of 15 rejects that supported channel type. This would block valid channel-create requests that currently pass through.
  - confidence: 0.87
- **[P2] Restore browser snapshot limit to backend capacity:** `src/agents/tools/browser-tool.schema.ts:98`
  - body: Browser snapshot backends still clamp snapshots at 2000 nodes, while this schema lowers the tool maximum to 1000. That rejects supported larger snapshots for complex pages and also targets the pre-plugin browser schema path.
  - confidence: 0.86
