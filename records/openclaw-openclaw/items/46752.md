## Summary

Keep open. The hardening direction is useful, but this PR is not merge-ready because several proposed schema maxima are stricter than current runtime/docs/provider contracts, and the PR only provides build/test proof rather than real after-fix behavior proof.

## What This Changes

The PR adds TypeBox `maxItems`/`maximum` bounds and richer descriptions across agent tool schemas for browser, message, node, sessions, memory, gateway, canvas, subagents, and web fetch tools.

## Best Possible Solution

Keep the schema-hardening goal, but rebase onto current main, derive each bound from the live runtime/docs/provider contract, update the current browser plugin schema surface, and add real gateway/model proof showing the bounded schemas behave correctly.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: the proposed schemas would reject currently valid calls such as `sessions_list({ limit: 200 })`, `sessions_spawn({ runTimeoutSeconds: 7200 })`, Discord channel type `16`, and browser snapshots above 1000 nodes.

## Solution Assessment

No. Schema hardening is a reasonable direction, but this diff is not the best implementation because several caps are invented below current runtime or dependency limits instead of being sourced from the existing contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Allow node timeouts to cover media durations:** `src/agents/tools/nodes-tool.ts:110-116`
  - body: `durationMs` is still allowed up to 300000ms for node media actions, but the new `timeoutMs` schema cap is 120000ms. Valid 5-minute `camera_clip` or `screen_record` calls can no longer set an RPC timeout long enough to finish.
  - confidence: 0.92
- **[P2] Keep sessions_list compatible with 200-row callers:** `src/agents/tools/sessions-list-tool.ts:33-38`
  - body: Current code still calls `sessions.list` with `limit: 200`, and the Gateway protocol only requires a positive integer. Adding `maximum: 50` makes existing valid session enumeration calls fail schema validation.
  - confidence: 0.9
- **[P2] Preserve long-running session spawns:** `src/agents/tools/sessions-spawn-tool.ts:38-44`
  - body: The docs and runtime allow any non-negative `runTimeoutSeconds`, with `0` meaning no timeout. Capping the schema at one hour rejects legitimate long-running subagent and ACP jobs that currently pass through.
  - confidence: 0.9
- **[P2] Permit Discord media channel type:** `src/agents/tools/message-tool.ts:329`
  - body: Discord `GuildMedia` is channel type 16 in the pinned API types, and the Discord runtime already treats it as a supported forum-like channel. The new `maximum: 15` rejects valid channel-create requests for that type.
  - confidence: 0.88
- **[P2] Restore browser snapshot limit to backend capacity:** `src/agents/tools/browser-tool.schema.ts:98`
  - body: The browser snapshot action forwards `limit` to backends that clamp ARIA snapshots at 2000 nodes. Lowering the schema maximum to 1000 prevents agents from requesting the full supported snapshot size on large pages.
  - confidence: 0.86
