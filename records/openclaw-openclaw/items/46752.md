## Summary

Keep this PR open, but it is not merge-ready. The schema-hardening direction is useful, and prior review comments already identified concrete blockers; current main still shows several proposed maxima would reject valid runtime-supported calls, and the branch is reported dirty against main.

## What This Changes

The PR adds TypeBox descriptions plus maxItems, maximum, minimum, and related schema hints across built-in agent tool schemas.

## Best Possible Solution

Land a rebased schema-hardening patch that encodes only contract-backed limits in active schemas, preserves existing long-running/session/media workflows, and adds focused schema contract coverage.

## Reproduction Assessment

Yes. Static review against current main reproduces the blockers by comparing the PR's proposed schema maxima with runtime code, protocol schemas, and existing tests for nodes, sessions, browser snapshots, and Discord media channels.

## Solution Assessment

No. Schema-only hardening is a maintainable direction, but this branch is not the best implementation because several bounds are too tight and one browser change targets a stale core path instead of the active plugin schema.

  - body: The new timeoutMs maximum is below valid media durations: current main allows durationMs up to 300000ms, so five-minute screen recordings or camera clips cannot set a gateway timeout long enough to finish.
  - confidence: 0.93

  - confidence: 0.9
- **[P2] Preserve long-running subagent timeouts:** `src/agents/tools/sessions-spawn-tool.ts:44-48`
  - body: The runtime still accepts finite non-negative runTimeoutSeconds and timeoutSeconds values and passes runTimeoutSeconds through to child runs. A maximum of 3600 rejects legitimate longer subagent or ACP jobs.
  - confidence: 0.9
- **[P2] Move browser hardening to the active plugin schema:** `src/agents/tools/browser-tool.schema.ts:98-101`
  - body: Current main registers the browser tool from extensions/browser, while this core browser schema path is stale. The proposed snapshot cap is also below the active backends' 2000-node clamp.
  - confidence: 0.87
- **[P2] Allow Discord media channel types:** `src/agents/tools/message-tool.ts:329`
  - body: The type maximum blocks Discord GuildMedia channels, whose enum value is 16 and which the current Discord runtime/tests treat as forum-like for thread creation.
