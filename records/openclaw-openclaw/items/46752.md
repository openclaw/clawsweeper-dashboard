## Summary

Keep open: the hardening direction is useful, but this PR is stale/conflicted and several new schema maximums reject values that current main still documents or supports at runtime.

## What This Changes

The PR adds TypeBox schema bounds and descriptions for agent tool parameters across browser, node, message, session, gateway, canvas, memory, and web tools.

## Best Possible Solution

Rebase or replace the PR on current main and keep only schema bounds that are at least as permissive as current runtime, gateway, dependency, and docs contracts.

## Reproduction Assessment

Yes. Source inspection reproduces the failure class: the PR adds schema maximums below existing valid runtime values for node recordings, browser snapshots, session listing, long-running spawned sessions, and Discord media channel types.

## Solution Assessment

No. Schema hardening is the right direction, but the current patch is not the best fix because several bounds narrow existing contracts instead of reflecting them; a current-main rebase or replacement with contract-aligned limits is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Raise node timeout cap to cover recording durations:** `src/agents/tools/nodes-tool.ts:113`
  - body: The schema now caps `timeoutMs` at 120s while the same tool still allows `durationMs` up to 300000 ms for `camera_clip` and `screen_record`. Valid 5-minute recordings cannot set a matching RPC timeout and can fail before the requested action finishes.
  - confidence: 0.9
- **[P2] Restore browser snapshot limit to the backend cap:** `src/agents/tools/browser-tool.schema.ts:99`
  - body: The PR lowers the browser snapshot `limit` maximum to 1000, but the current CDP, Playwright, and Chrome MCP snapshot backends all accept up to 2000 nodes. Large-page snapshots that previously worked would be rejected at schema validation.
  - confidence: 0.88
- **[P2] Keep sessions_list compatible with 200-row callers:** `src/agents/tools/sessions-list-tool.ts:36`
  - body: Capping `sessions_list.limit` at 50 rejects existing internal and documented usage that requests up to 200 rows, including announce-target lookup on current main. Larger installs can no longer enumerate enough sessions to resolve older targets.
  - confidence: 0.91
- **[P2] Do not cap sessions_spawn below the timeout contract:** `src/agents/tools/sessions-spawn-tool.ts:40`
  - body: The runtime treats `runTimeoutSeconds` and the back-compat `timeoutSeconds` alias as any non-negative value, with `0` preserving no-timeout behavior. A 3600-second schema maximum turns valid long-running spawned jobs into validation failures.
  - confidence: 0.9
- **[P2] Allow the Discord GuildMedia channel type:** `src/agents/tools/message-tool.ts:329`
  - body: The new `type` maximum of 15 excludes Discord `GuildMedia` channels, whose API value is 16 and which current Discord handling already treats as forum-like. That blocks legitimate channel-management calls that should still pass through.
  - confidence: 0.86
