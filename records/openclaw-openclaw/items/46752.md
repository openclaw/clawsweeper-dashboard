## Summary

Keep this PR open, but it is not merge-ready. The schema-hardening direction is useful and the latest discussion already calls out the same stale/conflicted state, but the current diff adds several bounds that are tighter than existing runtime, docs, and plugin contracts.

## What This Changes

The PR adds TypeBox maxItems, minimum/maximum bounds, and descriptions to built-in agent tool schemas across browser, canvas, gateway, memory, message, nodes, sessions, subagents, and web fetch tools.

## Best Possible Solution

Land a rebased schema-only patch that applies bounds to the active core/plugin schemas using runtime- and API-backed limits, preserves existing long-running/session/media workflows, and adds focused schema contract coverage.

## Reproduction Assessment

Yes for the PR regressions: static review reproduces the blockers by comparing the proposed schema maxima with current main runtime code, protocol schemas, docs, and existing tests. I did not run the test suite because this review was read-only.

## Solution Assessment

No. Schema-only hardening is the right direction, but this branch is not the best implementation because several limits are below existing contracts and the browser change targets a stale core path instead of the active plugin schema.

Full review comments:

- **[P1] Raise node timeouts to cover max recordings:** `src/agents/tools/nodes-tool.ts:113-117`
  - body: The new timeoutMs maximum is below valid media durations: current main allows durationMs up to 300000ms, so five-minute camera_clip or screen_record requests cannot set a gateway timeout long enough to complete.
  - confidence: 0.93
- **[P2] Keep sessions_list compatible with 200-row lookups:** `src/agents/tools/sessions-list-tool.ts:36-40`
  - body: Capping sessions_list.limit at 50 rejects an existing lookup path: announce-target resolution still asks sessions.list for 200 rows, and the gateway sessions schema only requires a positive integer.
  - confidence: 0.91
- **[P2] Preserve long-running subagent timeouts:** `src/agents/tools/sessions-spawn-tool.ts:44-48`
  - body: The runtime and docs still allow non-negative runTimeoutSeconds/timeoutSeconds values, with 0 meaning no timeout. Adding maximum: 3600 turns legitimate longer subagent or ACP jobs into schema-validation failures.
  - confidence: 0.9
- **[P2] Move browser hardening to the active plugin schema:** `src/agents/tools/browser-tool.schema.ts:98-101`
  - body: Current main registers the browser tool from extensions/browser, while this PR edits the old core browser schema path. The proposed snapshot cap is also below the active backends' 2000-node clamp.
  - confidence: 0.88
- **[P2] Allow Discord media channel types:** `src/agents/tools/message-tool.ts:329`
  - body: The type maximum blocks Discord GuildMedia channels, whose enum value is 16 and which the current Discord runtime/tests treat as forum-like for thread creation.
  - confidence: 0.88
