## What This Changes

The issue asks for Chromium/Vivaldi Control UI chat to render after pairing instead of leaving `<openclaw-app>` unregistered while backend RPCs succeed.

## Best Possible Solution

Keep the shipped explicit custom-element registration guard; if a blank Control UI still reproduces on v2026.5.3 or newer, open a fresh report with current browser console errors and network/module-load details.

## Solution Assessment
