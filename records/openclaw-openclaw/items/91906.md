## Summary

Keep open: the optimization is useful, but the current lane-only predicate is too broad because current main still has delivered native and ACP subagent session paths that use the subagent lane and live callbacks.

## What This Changes

The PR threads a `suppressLiveStreamOutput` option through embedded subagent runs so subagent `message_update` chunks are buffered but not parsed into live preview output before final `message_end` delivery.

## Best Possible Solution

Gate the optimization on a concrete no-live-consumer condition rather than the subagent lane alone, then prove both detached subagent final readback and directly delivered subagent streaming behavior.

## Reproduction Assessment

Yes. Source inspection shows native and ACP session spawns can set `lane: AGENT_LANE_SUBAGENT` while `deliver` is true, and delivered embedded runs wire live partial/reasoning/block callbacks that depend on `message_update` parsing.

## Solution Assessment

No. The optimization is plausible for detached subagents, but the lane-only predicate is not the best fix because delivered subagent session paths still have live consumers; the safer boundary is an explicit no-live-consumer condition.
