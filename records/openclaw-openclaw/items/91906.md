## Summary

Keep this PR open, but not merge-ready: the performance idea is useful, yet the branch suppresses live parsing for every subagent lane even though current main has directly delivered native and ACP subagent sessions that still use that lane.

## What This Changes

The branch threads a `suppressLiveStreamOutput` flag from subagent-lane attempts into embedded subscription handling so `message_update` live parsing is skipped while `message_end` final output remains enabled.

## Best Possible Solution

Gate suppression on a real detached/no-live-consumer condition, cover both detached and directly delivered subagent sessions, and add redacted real-run or benchmark proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: native and ACP session spawns can deliver directly or inline while still using `AGENT_LANE_SUBAGENT`, and this PR suppresses live parsing based only on that lane.

## Solution Assessment

No. The optimization is plausible for detached subagents, but the current predicate is too broad; the safer fix is to key suppression off the absence of a live delivery consumer.
