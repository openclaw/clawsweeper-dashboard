## Summary

Keep open: the optimization is useful, but the current lane-only suppression is still too broad and the contributor proof remains test-only. Current main has native and ACP subagent session paths that use the subagent lane while still wiring live delivery callbacks, so merging this as-is can suppress visible partial/block/reasoning updates.

## What This Changes

This PR threads a new `suppressLiveStreamOutput` boolean through embedded agent runs and enables it for all subagent-lane attempts to skip per-chunk live stream parsing while preserving final `message_end` delivery.

## Best Possible Solution

Keep the optimization only after suppression is gated on an actual no-live-consumer condition, with coverage for detached and directly delivered subagents plus redacted real-run or benchmark proof.

## Reproduction Assessment

Yes. Source inspection shows native and ACP session spawns can use `AGENT_LANE_SUBAGENT` while direct or inline delivery gives the embedded subscription live callbacks.

## Solution Assessment

No. The optimization is plausible for detached subagents, but the current lane-only predicate is too broad; the safer fix is to key suppression off the absence of a live delivery consumer.
