## Summary

The PR also still has mock-only proof and GitHub reports the head as dirty against the base.

## What This Changes

The PR threads a `suppressLiveStreamOutput` flag through embedded agent run and subscribe paths, then enables it for all subagent-lane attempts to skip `message_update` live parsing while keeping `message_end` final delivery.

## Best Possible Solution

Keep the optimization only after suppression is gated on an actual no-live-consumer condition, add focused coverage for detached and directly delivered subagents, rebase cleanly, and attach redacted real-run or benchmark proof.

## Reproduction Assessment

Yes. Source inspection shows native and ACP session spawns can use `AGENT_LANE_SUBAGENT` while direct or inline delivery gives the embedded subscription live callbacks.

## Solution Assessment

No. The optimization is plausible for detached subagents, but the current lane-only predicate is too broad; the safer fix is to key suppression off the absence of a live delivery consumer.
