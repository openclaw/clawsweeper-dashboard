## Summary

Keep open: this PR targets a real source-reproducible ACP timeout bug, but the current head has blocking correctness and validation issues because it forwards all late events into the visible reply projector, the new test file fails test typechecking, and no real after-fix ACP proof has been provided.

## What This Changes

The PR modifies ACP turn-stream consumption to forward post-timeout events through `onEvent`, surface late error events, throw saved stream errors after the loop, and adds a new helper test file.

## Best Possible Solution

Keep the timeout boundary for visible output, surface late error or terminal diagnostics through a filtered non-visible path, and cover that contract with compiling helper plus dispatch/projector tests and real ACP proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: close `eventGate.open` during an ACP timeout, then have `runtime.runTurn()` yield an error or output-like event; current main skips it before delivery or error surfacing, and CI reproduced the new test type errors.

## Solution Assessment

No. Throwing captured late errors is aligned with the bug, but forwarding every late event through `onEvent` is not the narrowest maintainable fix because ACP dispatch wires `onEvent` to visible reply projection.
