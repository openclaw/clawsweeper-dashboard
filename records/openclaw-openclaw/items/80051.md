## Summary

Keep open: the linked ACP timeout bug is valid on current main, but this PR has blocking correctness issues and is missing required real behavior proof before merge.

## What This Changes

The PR changes ACP turn-stream handling to forward late post-timeout events to `onEvent`, surface late errors, add focused tests, and add a changelog entry.

## Best Possible Solution

Define the ACP post-timeout stream contract, then keep the fix in `consumeAcpTurnStream` with tests proving late errors remain observable and late output is not delivered after the timeout boundary.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: close the gate during a timed-out ACP turn, then have the runtime iterator yield an event or error afterward; current main skips it before delivery or error conversion. I did not run a live ACP timeout scenario in this read-only review.

## Solution Assessment

No, not as written. The helper is the right narrow place to fix this, but the closed-gate branch must filter output events and the new test fixture must satisfy the runtime turn type before this is mergeable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Filter late output before calling onEvent:** `src/acp/control-plane/manager.turn-stream.ts:32`
  - body: The closed-gate branch calls `onEvent` before checking the event type. Because `onEvent` feeds projectors that render `text_delta` and tool events, late output can still reach users after the timeout gate closes, contrary to the non-output-only contract in the title/body/changelog. Skip `text_delta` and `tool_call` here while still observing late errors/terminal/status events.
  - confidence: 0.92
- **[P2] Provide a complete turn fixture:** `src/acp/control-plane/manager.turn-stream.test.ts:18`
  - body: The new `turn` fixture only has `handle` and `text`, but `consumeAcpTurnStream` requires an `AcpRuntimeTurnInput`, whose required fields also include `mode` and `requestId`. Add those fields so the new test file passes the repo's test typecheck lane.
  - confidence: 0.88
