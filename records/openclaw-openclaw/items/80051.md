## Summary

Keep open: the ACP timeout bug is source-reproducible, but the latest PR head still forwards late output through `onEvent`, has type-invalid test fixtures, and lacks required real behavior proof.

## What This Changes

The PR changes ACP turn-stream handling to pass closed-gate events to `onEvent`, preserve late `error` events, always throw captured stream errors, and adds focused turn-stream tests.

## Best Possible Solution

Keep the fix in `consumeAcpTurnStream`, but deliver only late non-output/error/terminal information after timeout, fix the type-correct tests, and require real ACP timeout proof before merge.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: close the ACP turn gate during a timed-out run, then let `runtime.runTurn()` yield an event or error afterward; current main skips it before delivery and error conversion. I did not run a live ACP timeout scenario in this read-only review.

## Solution Assessment

No, not as written. `consumeAcpTurnStream` is the right narrow fix point, but the PR must filter late output before `onEvent`, repair the type-invalid fixtures, and add real timeout proof before merge.
