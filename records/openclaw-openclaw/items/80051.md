## What This Changes

The PR updates `consumeAcpTurnStream` to forward closed-gate events to `onEvent`, preserve late stream errors, remove the final gate guard, and add focused turn-stream tests.

## Best Possible Solution

Keep the fix in `consumeAcpTurnStream`, but filter late output before `onEvent`, use type-correct test fixtures, and require real ACP timeout proof before merge.

I did not run a live ACP timeout scenario in this read-only review.
