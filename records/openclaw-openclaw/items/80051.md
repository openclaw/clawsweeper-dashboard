## Summary

Keep open: the underlying ACP timeout bug is source-reproducible, but this PR is not merge-ready because it forwards every late event through the visible `onEvent` path, its new test fixtures fail type checking, and the submitted proof is mock-only.

## What This Changes

The PR updates `consumeAcpTurnStream` to call `onEvent` and capture errors after the timeout gate closes, and adds a new helper test file.

## Best Possible Solution

Define the ACP post-timeout stream contract, then preserve late error or terminal diagnostics through a non-visible or filtered path, keep late output suppressed, fix the test fixture types, and add targeted manager plus dispatch/projector coverage.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: close the shared gate during a timed-out turn, then have the runtime iterator yield an `error` or visible event; current main skips it, while this PR forwards visible late events through `onEvent`.

## Solution Assessment

No. Capturing late errors is the right direction, but this implementation forwards every closed-gate event through the visible projector path and the new tests currently fail type checking.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Filter late events before invoking onEvent:** `src/acp/control-plane/manager.turn-stream.ts:32`
  - body: After timeout closes the gate, this calls `onEvent` for every ACP event before checking the type. ACP dispatch passes `onEvent` to the reply projector, which renders `text_delta`, `status`, and `tool_call`, so a late stream can emit visible output after the turn has already timed out; filter to late error/terminal diagnostics or use a non-visible path.
  - confidence: 0.91
- **[P2] Use valid stream values in test events:** `src/acp/control-plane/manager.turn-stream.test.ts:31`
  - body: The new test uses `stream: "reply"`, but `AcpRuntimeEvent.stream` only accepts `"output"`, `"thought"`, or `undefined`, so `check-test-types` fails at the PR head. Use a valid stream value here and in the later fixture before this can pass CI.
  - confidence: 0.97
