## Summary

Keep open: current main still has the source-level ACP timeout bug, but this PR head is not mergeable because it forwards output-capable late events through `onEvent`, adds type-invalid test fixtures, and still lacks real behavior proof.

## What This Changes

The PR changes `consumeAcpTurnStream` to forward closed-gate events to `onEvent`, capture late error events, throw captured errors after the gate closes, and add focused turn-stream tests.

## Best Possible Solution

Keep the fix in `consumeAcpTurnStream`, but filter closed-gate output events before `onEvent`, use type-correct fixtures and valid stream values, and require real ACP timeout proof before merge.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: close the ACP turn gate during a timed-out run, then let `runtime.runTurn()` yield an event or error afterward; current main skips it before delivery and keeps late errors from throwing.

## Solution Assessment

No, not as written. `consumeAcpTurnStream` is the right narrow fix point, but the PR must avoid forwarding late output through `onEvent`, repair the invalid test fixtures, and add real ACP timeout proof before merge.
