## Summary

Keep open: current main still has the late ACP event/error drop bug, but this PR remains incorrect because it forwards every post-timeout event through `onEvent`, which ACP dispatch can project into visible replies, and the supplied proof is still mock-only.

## What This Changes

The PR changes `consumeAcpTurnStream` to call `onEvent` and capture errors after the timeout gate closes, and adds focused unit tests for gate-open, gate-closed, error, and terminal cases.

## Best Possible Solution

Define the ACP post-timeout stream contract, preserve late error or terminal diagnostics without rendering visible output after timeout, and cover both the helper and ACP dispatch/projector boundary.

## Reproduction Assessment

Yes. Source inspection on current main shows `awaitTurnWithTimeout` sets `eventGate.open = false` while `consumeAcpTurnStream` keeps draining; any post-gate error or event hits `continue` before conversion or delivery.

## Solution Assessment

No. Throwing saved late errors is aligned with the bug, but forwarding every closed-gate event through `onEvent` is not the narrowest maintainable fix because that path can render visible ACP reply output.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Filter late events before invoking onEvent:** `src/acp/control-plane/manager.turn-stream.ts:32`
  - body: After the timeout gate closes, this line forwards the raw ACP event to `onEvent` before checking whether it is terminal or an error. ACP dispatch passes `onEvent` into the reply projector, which renders `text_delta`, `status`, and `tool_call` events, so this can emit visible output after the turn already timed out; route only late error/terminal diagnostics or use a non-visible channel.
  - confidence: 0.9
