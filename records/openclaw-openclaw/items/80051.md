## Summary

Keep open: the current main bug is source-reproducible, but the latest PR still forwards every late event through `onEvent`, which can leak visible ACP output after the timeout boundary, and the supplied proof is mock/unit-test-only.

## What This Changes

The PR changes ACP turn-stream consumption so events after timeout-gate closure still reach `onEvent`, late error events become `AcpRuntimeError`s, and focused unit tests cover the helper.

## Best Possible Solution

Define the ACP post-timeout stream contract, surface late errors or terminal diagnostics without replaying visible output after timeout, and cover the helper plus dispatch/projector boundary.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: close `eventGate.open` during `awaitTurnWithTimeout`, then have `runtime.runTurn()` yield an `error` or visible event afterward; current main skips it before delivery or error conversion.

## Solution Assessment

No. Throwing saved late errors is aligned with the bug, but forwarding every closed-gate event through `onEvent` is not the narrowest maintainable fix because ACP dispatch maps `onEvent` to visible reply projection.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
