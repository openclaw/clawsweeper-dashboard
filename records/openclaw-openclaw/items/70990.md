## Summary

Keep this PR open: the terminal hook and access gate look reasonable, but the latest head still has a blocking public contract issue where `model_failover` promises current/selected target metadata while the prompt failover path emits the same failed provider/model as both source and target. The PR also currently has failing CI shards, so it is not ready to merge.

## What This Changes

The PR adds typed plugin hooks for embedded model failover decisions and terminal model-failure outcomes, plus documentation, config help, changelog, and targeted tests.

## Best Possible Solution

Settle the public `model_failover` contract before merge: either emit selected fallback target facts from the outer fallback seam, or explicitly narrow docs/types so the hook is failed-attempt telemetry; keep the terminal hook behind the existing conversation-access gate.

## Reproduction Assessment

Not applicable as a current-main bug reproduction because this is a feature PR. The review finding is source-reproducible on PR head: the prompt failover call passes the same failed provider/model into both target and source fields while current main has selected `nextCandidate` facts in the outer fallback layer.

## Solution Assessment

No, not yet. The terminal hook gate and fire-and-forget plumbing are reasonable, but the public `model_failover` payload/timing needs a maintainer API decision before it is safe to publish.
