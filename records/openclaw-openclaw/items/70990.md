Full review comments:

- **[P1] Pass the next candidate through auto-reply:** `src/auto-reply/reply/agent-runner-execution.ts:1975`
  - body: The Gateway auto-reply `runWithModelFallback` call never sets `exposeNextCandidateToRun`, so `runOptions?.nextCandidate` is not available when this path calls `runEmbeddedAgent`. On a normal channel fallback, `model_failover` can therefore report the failed source as the current target instead of the selected fallback model, which breaks the main hook payload contract.
  - confidence: 0.91
- **[P1] Emit the terminal hook after final copy resolution:** `src/auto-reply/reply/agent-runner-execution.ts:2908`
  - body: `model_failure_terminal` is emitted with `finalMessage: message` before this function computes `fallbackText` and `userVisibleFallbackText`, which is the reply actually returned to the channel. Rate-limit, billing, context-overflow, Control UI, and generic failure branches can all rewrite the delivered text after this point, so plugins can observe a different final message than users receive.
  - confidence: 0.93
