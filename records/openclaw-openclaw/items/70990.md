## What This Changes

The PR adds typed `model_failover` and `model_failure_terminal` plugin hooks, wires them into failover and terminal failure paths, and updates docs, config help, and focused tests.

## Best Possible Solution

Ship the hook only after the failover payload has precise source/target semantics, while keeping the terminal failure hook behind the existing conversation-access gate.

## Reproduction Assessment

Not applicable as a current-main bug reproduction because this is a feature PR. The contract issue is source-reproducible on PR head: prompt fallback passes the failed attempt as both source and target while docs promise the current target.

## Solution Assessment

No, not yet. The terminal failure gate and fail-open hook plumbing look reasonable, but the failover event target semantics need to be fixed or narrowed before shipping the public contract.
