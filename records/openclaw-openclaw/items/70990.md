## Summary

Keep open. Current main still does not implement these hooks, and this PR is not merge-ready because the terminal failure hook exposes provider-error-bearing text through an ungated public plugin surface and the public docs/changelog contract is missing.

## What This Changes

The PR adds typed `model_failover` and `model_failure_terminal` plugin hooks, emits them from embedded failover and terminal before-reply failure paths, and adds focused hook plumbing tests.

## Best Possible Solution

Keep the narrow hook API only if maintainers accept the product direction, then align terminal payload access with the existing plugin data-access contract and document the public surface before merge.

## Reproduction Assessment

Yes. The issue is reproducible by inspecting the PR head emission site, current-main `CONVERSATION_HOOK_NAMES` and registry gate, and current-main fallback summary construction from per-attempt provider errors.

## Solution Assessment

No, not as currently written. The hook direction is maintainable, but the terminal hook should be gated or sanitized like other conversation/error-observation hooks and shipped with docs plus a changelog entry.
