## Summary

Keep open. Current main does not implement the requested hook surface, but this PR is not merge-ready because the terminal failure hook exposes final error text without the existing conversation-access gate and the new public plugin API lacks docs and changelog coverage.

## What This Changes

The PR adds typed `model_failover` and `model_failure_terminal` plugin hooks, wires failover and terminal failure emission, and adds focused hook plumbing tests.

## Best Possible Solution

Keep the narrow hook direction only if maintainers accept the API shape, but make terminal failure events follow the plugin data-access policy and document the payload/access contract before merge.

## Reproduction Assessment

Yes. Static review reproduces the issue by inspecting the PR diff that sends `finalMessage: message`, current-main fallback summary construction from per-attempt errors, and current-main conversation hook gating.

## Solution Assessment

No, not as currently written. The failover hook is a plausible narrow extension, but the terminal hook should be sanitized or gated like other conversation/error-observation hooks before becoming public.
