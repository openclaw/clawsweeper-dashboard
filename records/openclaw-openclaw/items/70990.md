## Summary

Keep open: the hook plumbing is useful and the PR includes real terminal proof, but the public `model_failover` contract still does not reliably publish the selected fallback target that the docs and operator feedback describe.

## What This Changes

The PR adds typed `model_failover` and `model_failure_terminal` plugin hooks, wires them into embedded failover and terminal before-reply failure paths, and updates docs, config help, and tests.

## Best Possible Solution

Finalize the `model_failover` payload so selected fallback target semantics are either actually emitted from the outer fallback step or explicitly removed from the docs/types before merge.

## Reproduction Assessment

Not applicable as a current-main bug reproduction because this is a feature PR. The blocking contract issue is source-reproducible on PR head: prompt fallback passes the failed attempt as both source and target while docs promise the current target.

## Solution Assessment

No, not yet. The terminal failure gate and fail-open hook plumbing look reasonable, but the failover event target semantics need to be fixed or narrowed before shipping the public contract.
