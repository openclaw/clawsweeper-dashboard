## Summary

Keep open. Current main does not implement these hooks, but the PR is not merge-ready because the terminal failure hook exposes a provider-error-bearing message through an ungated public plugin surface and the public docs/changelog contract is missing.

## What This Changes

The PR adds typed `model_failover` and `model_failure_terminal` plugin hooks, emits them from embedded failover and terminal before-reply failure paths, and adds focused hook plumbing tests.

## Best Possible Solution

Keep the narrow hook API only if maintainers accept the product direction, but make the terminal failure payload match the existing plugin data-access contract, document both hooks, add the required changelog entry, and validate the focused plugin/failover tests before merge.

## Reproduction Assessment

Yes. The review can reproduce the issue by inspecting PR head `agent-runner-execution.ts:1546`, current-main `CONVERSATION_HOOK_NAMES` and registry gating, and current-main fallback summary construction from per-attempt provider errors.

## Solution Assessment

No, not as currently written. The hook direction is maintainable, but the terminal hook should be sanitized or gated like other conversation/error-observation hooks before becoming a public plugin contract.
