## Summary

Keep open. The hook idea and terminal access gate are useful and the PR includes sufficient terminal proof, but current head still has incorrect or unresolved `model_failover` target semantics and duplicates shipped changelog entries.

## What This Changes

This PR adds typed `model_failover` and `model_failure_terminal` plugin hooks, wires them into embedded failover and terminal before-reply failure paths, and updates plugin docs, config help, tests, and changelog.

## Best Possible Solution

Finalize the public `model_failover` contract so `provider`/`model` either carry the selected fallback target or the docs/types explicitly define failed-attempt telemetry, then keep only the plugin hook changelog entry.

## Reproduction Assessment

Not applicable as a current-main bug reproduction because this is a feature PR. The blocking PR issue is source-reproducible: prompt failover emits the failed provider/model as both source and target while current main has selected `nextCandidate` facts elsewhere.

## Solution Assessment

No, not yet. The terminal hook gate and fail-open plumbing look reasonable, but the public `model_failover` payload semantics need a maintainer API decision before merge.
