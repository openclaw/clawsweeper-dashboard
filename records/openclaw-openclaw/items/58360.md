## Summary

Keep this PR open but not merge-ready: the raw payload hook API still needs access-policy/product review, source inspection shows lifecycle and correlation gaps, and the contributor proof is only targeted tests.

## What This Changes

The branch adds `before_model_call` and `after_model_call` plugin hook types, hook-runner methods, embedded-runner `streamFn` wiring, replay-normalization helper extraction, and targeted tests.

## Best Possible Solution

Keep sanitized telemetry on main as the default; accept raw payload hooks only after maintainers approve the trust model and the branch is rebased, gated, lifecycle-complete, documented/tested, and proven in a real plugin or embedded run.

## Reproduction Assessment

Yes for the review findings by source inspection. PR head exposes raw `requestMessages`/`responseMessage`, emits `after_model_call` only from `.result()`, and builds call IDs from identifiers that the retry loop reuses.

## Solution Assessment

No. The useful observability goal should be solved either by the existing sanitized `model_call_started`/`model_call_ended` hooks or by a raw hook API that is explicitly gated, lifecycle-complete, provider-bound, documented, and proven in a real run.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate raw model-call hooks behind conversation access:** `src/plugins/types.ts:2023-2049`
  - body: `requestMessages` exposes provider-facing prompt/history, and `responseMessage` can expose the full assistant response, but the new hook names are not tied to the conversation-content access policy. Add these hooks to that gate, with registry coverage, before exposing this data to non-bundled plugins.
  - confidence: 0.9
- **[P2] Emit after hooks when streams are abandoned:** `src/agents/pi-embedded-runner/run/attempt.ts:1261-1270`
  - body: `after_model_call` is only fired from the patched `.result()` method. If a caller consumes or abandons the async iterator without calling `.result()`, tracing plugins get an unmatched `before_model_call`; wrap iteration/finally like the diagnostic model-call wrapper or document and test a one-sided contract.
  - confidence: 0.84
- **[P2] Make model-call IDs unique across retry attempts:** `src/agents/pi-embedded-runner/run/attempt.ts:271-272`
  - body: `buildModelCallId` combines `runId`, `sessionId`, and a wrapper-local sequence, but retry attempts can reuse the same run/session while the wrapper counter resets. Include an attempt-unique token or keep the counter outside the retry boundary so downstream correlation cannot merge distinct provider calls.
  - confidence: 0.82
- **[P2] Report provider-bound model metadata:** `src/agents/pi-embedded-runner/run/attempt.ts:1180-1192`
  - body: The hook captures provider/model/api before delegating to inner stream wrappers. Wrappers such as MiniMax fast mode can rewrite the model before the actual provider call, so telemetry can be attributed to the wrong model; emit closer to the final provider boundary or propagate final metadata back.
  - confidence: 0.76
