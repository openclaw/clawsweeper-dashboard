## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Reuse the current model-call hook contract:** `src/plugins/types.ts:1799-1831`
  - body: Current `main` already exposes `model_call_started` / `model_call_ended` as the documented model-call telemetry hooks. Adding `before_model_call` / `after_model_call` as a parallel public API would give plugin authors two competing sources of truth for the same lifecycle, so this needs to rebase onto and extend the current contract or document an intentional replacement path.
  - confidence: 0.9
- **[P1] Gate raw model payload access:** `src/plugins/types.ts:2003-2038`
  - body: The new event types expose `systemPrompt`, raw `requestMessages`, and `responseMessage` to hook consumers, but current main only allows raw conversation hooks for non-bundled plugins when `allowConversationAccess` is enabled. Without adding these new hooks to that policy or reducing them to sanitized metadata, external plugins can get conversation content through a new bypass path.
  - confidence: 0.92
- **[P2] Emit terminal hooks when streams are abandoned:** `src/agents/pi-embedded-runner/run/attempt.ts:1208-1228`
  - body: `after_model_call` still depends on intercepting `.result()`, so callers that consume or abort the async stream without calling `.result()` can leave a `before_model_call` with no terminal pair. Current main's diagnostic wrapper observes the async iterator and emits from `finally`; this hook needs the same lifecycle coverage or a deliberately documented weaker guarantee before it becomes public API.
  - confidence: 0.83
