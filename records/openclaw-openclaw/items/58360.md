## Summary

Keep open, but do not merge as-is: this branch is still the canonical raw model-call hook discussion from the earlier closed feature request, while current main already has sanitized per-provider-call `model_call_started`/`model_call_ended` telemetry and this PR adds a competing raw payload API without a settled access-control or lifecycle contract.

## What This Changes

The branch adds `before_model_call` and `after_model_call` plugin hook types and runner methods, wraps embedded-runner `streamFn` calls to emit raw request/response events, extracts replay tool-call normalization, and adds targeted tests.

## Best Possible Solution

Settle one coherent model-call hook API that reuses or explicitly extends the current sanitized hook contract, gates any raw content with the existing conversation-access policy, and covers retries, blocked access, and stream-finalization behavior with tests and real plugin proof.

## Reproduction Assessment

Yes for the PR review defects by source inspection: the branch adds new raw hook names and payload fields, derives call IDs from reused run/session IDs, and ties after-hook delivery to `.result()`. This is not a current-main bug reproduction because the requested raw model-call hook is a new public plugin capability.

## Solution Assessment

No; the branch is not the best merge shape as-is because current main already has a sanitized per-call hook contract and this PR adds a competing raw payload contract. The safer path is to reuse or explicitly extend the current contract behind the existing conversation-access policy.
