## Summary

Keep this PR open for maintainer API/security review: current main now has sanitized per-model-call telemetry, while this branch still proposes a separate raw prompt/response hook surface without a settled access policy or real behavior proof.

## What This Changes

The PR adds `before_model_call` and `after_model_call` plugin hooks around embedded-runner provider calls, exposes raw request/response payload fields, and adds targeted hook/payload tests.

## Best Possible Solution

Land one coherent model-call hook API that preserves the shipped sanitized telemetry, gates any raw payload access with the conversation-access policy, defines terminal event semantics for retries and abandoned streams, and includes real plugin proof.

## Reproduction Assessment

Yes for the PR review defects by source inspection: the branch adds raw payload fields and its after hook is tied to `.result()`. This is not a current-main bug reproduction because the requested raw model-call hook is a new public plugin capability.

## Solution Assessment

No as submitted. The maintainable path is to reconcile with the existing sanitized model-call hooks, apply conversation-access gating for raw content, and prove terminal semantics in real plugin use.
