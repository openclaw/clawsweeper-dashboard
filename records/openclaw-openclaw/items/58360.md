## Summary

Keep open. Current main has the safer sanitized model-call telemetry hooks, but this PR still proposes a raw prompt/response hook API and has unresolved data-access and lifecycle blockers before it can merge.

## What This Changes

The PR adds `before_model_call`/`after_model_call` plugin hook types, hook-runner methods, embedded-runner `streamFn` instrumentation, replay normalization extraction, and focused tests.

## Best Possible Solution

Decide the plugin data-access policy first, then either keep the shipped sanitized telemetry hooks or rework this into an explicitly conversation-gated, documented raw-payload API with terminal-safe lifecycle coverage.

## Reproduction Assessment

Yes. Code inspection reproduces the blockers: the PR exposes raw `requestMessages`/`responseMessage` while the current conversation-access gate does not include the new hook names, and the terminal hook is tied to `.result()` consumption.

## Solution Assessment

No. The proposed raw hook shape is not the best solution as-is; the safer maintainable path is the current sanitized hook pair or a gated raw-payload extension with docs, loader tests, and stream-finalization coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Gate raw model-call hooks behind conversation access:** `src/plugins/types.ts:2023`
  - body: `before_model_call` exposes provider-facing `requestMessages`, and `after_model_call` can expose the assistant response, but the PR adds them as normal typed hooks. Non-bundled plugins could receive prompt/history/response content without `plugins.entries.<id>.hooks.allowConversationAccess=true`; add both raw hooks to the conversation hook set with loader coverage, or keep these events sanitized.
  - confidence: 0.86
- **[P2] Emit after hooks when streams are abandoned:** `src/agents/pi-embedded-runner/run/attempt.ts:1248-1258`
  - body: `after_model_call` is emitted from the wrapped stream's `.result()` path, so callers that consume the async iterator and return early can skip the terminal event. That leaves tracing plugins with unmatched call IDs; wrap async iteration/finally like the current diagnostic model-call wrapper or document and test a one-sided contract before exposing it.
  - confidence: 0.78

## Security Review
