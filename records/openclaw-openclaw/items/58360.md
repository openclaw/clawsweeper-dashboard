## Label Justifications

- `P2`: This is a normal-priority plugin API improvement with real observability value but no current core runtime outage.

## Summary

Keep open for maintainer API/security review: the branch is a real attempt at per-call observability, but current main already has sanitized `model_call_started` / `model_call_ended` hooks and this PR adds a competing raw payload surface without settled access policy or real behavior proof.

## What This Changes

The PR adds `before_model_call` and `after_model_call` plugin hook names, runner methods, embedded-runner stream wrapping, replay-payload normalization, and targeted hook tests.

## Best Possible Solution

Settle one documented model-call observability contract: keep sanitized metadata hooks as the default, gate any raw payload access through the existing conversation-access policy, and prove terminal behavior across normal, retry, error, and abandoned-stream paths.

## Reproduction Assessment

Yes for the PR review defects by source inspection: the branch adds raw payload hook fields, a second model-call hook family, and `.result()`-dependent terminal events. This is not a current-main bug reproduction because the requested raw model-call hook is a new public plugin capability.

## Solution Assessment

No as submitted. The maintainable path is to reconcile with the current sanitized `model_call_*` hooks, make any raw payload access an explicit gated contract, and prove lifecycle behavior before exposing a new public API.
