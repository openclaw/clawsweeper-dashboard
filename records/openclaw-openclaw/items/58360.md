## Summary

Keep open. Current main has sanitized call-level model hooks, but this PR still raises a raw payload data-access policy blocker and a terminal-event lifecycle bug before it can merge.

## What This Changes

The PR adds raw `before_model_call` and `after_model_call` plugin hooks, hook-runner methods, embedded-runner `streamFn` wiring, replay-payload normalization extraction, and focused tests.

## Best Possible Solution

Decide the public plugin data-access policy first, then either keep the shipped sanitized telemetry hooks or rework this PR into an explicitly conversation-gated, documented raw-payload API with terminal-safe lifecycle coverage.

## Reproduction Assessment

Yes. Code inspection reproduces both blockers: the PR exposes raw `requestMessages` while the current conversation-access gate excludes the new hook names, and the proposed terminal hook path can be skipped when a stream is consumed by async iteration and abandoned before `.result()`.

## Solution Assessment

No. The proposed raw hook shape is not the best solution until the data-access policy is settled; the safer maintainable path is the current sanitized hook pair or a gated raw-payload extension built on that contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Gate raw model-call hooks behind conversation access:** `src/plugins/types.ts:2023`
  - body: `before_model_call` exposes provider-facing `requestMessages`, and `after_model_call` can expose the response message, but the PR adds them as normal typed hooks rather than conversation-content hooks. A non-bundled plugin could receive prompt/history/response content without `plugins.entries.<id>.hooks.allowConversationAccess=true`; add both raw hooks to the conversation hook set with loader tests, or keep these events sanitized.
  - confidence: 0.86
- **[P2] Emit after hooks when streams are abandoned:** `src/agents/pi-embedded-runner/run/attempt.ts:1258-1267`
  - body: `after_model_call` is emitted from the wrapped stream's `.result()` path, so callers that consume the async iterator and return early can skip the terminal event. That leaves tracing plugins with unmatched call IDs; wrap async iteration/finally like the current diagnostic model-call wrapper or document and test a one-sided contract before exposing it.
  - confidence: 0.78

That broadens third-party plugin access to conversation content without the existing explicit opt-in.
