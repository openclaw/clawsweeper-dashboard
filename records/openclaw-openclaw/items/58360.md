## Summary

Keep open: current main intentionally exposes sanitized `model_call_started` / `model_call_ended` telemetry, while this PR still proposes a raw prompt/response plugin surface that needs real behavior proof and maintainer/security API review before it can merge.

## What This Changes

The PR adds raw `before_model_call` and `after_model_call` plugin hooks around embedded-runner provider calls, refactors replay tool-call normalization, and adds focused hook tests.

## Best Possible Solution

Keep sanitized model-call telemetry as the default; if raw payload hooks are accepted, add them behind explicit conversation-access policy with terminal-safe emission, provider-bound metadata, current-file rebase, docs, and loader/runtime tests.

## Reproduction Assessment

Yes. Source inspection of the PR head reproduces the review blockers: raw request/response payload fields are exposed through new hooks, terminal emission depends on `.result()`, and model metadata is captured before deeper provider wrappers can rewrite it.

## Solution Assessment

No. The raw hook shape is not the best mergeable solution until maintainers decide the plugin data-access policy; the safer path is the current sanitized telemetry or a gated raw-payload API with lifecycle and metadata fixes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Gate raw model-call hooks behind conversation access:** `src/plugins/types.ts:2023-2049`
  - body: `before_model_call` carries provider-facing `requestMessages`, and `after_model_call` can carry the full response message. The PR adds those hook names without showing they are covered by the existing conversation-content access gate, so a non-bundled plugin could receive raw conversation content without the explicit opt-in required for `llm_input` / `llm_output`.
  - confidence: 0.88
- **[P2] Emit after hooks when streams are abandoned:** `src/agents/pi-embedded-runner/run/attempt.ts:1258-1264`
  - body: `after_model_call` is fired by replacing the stream's `.result()` method. A caller that consumes the async iterator and returns early without calling `.result()` never emits the terminal event, leaving tracing plugins with unmatched call IDs; wrap async iteration/finally like the current diagnostic model-call wrapper or document and test a one-sided contract.
  - confidence: 0.81
- **[P2] Report provider-bound model metadata:** `src/agents/pi-embedded-runner/run/attempt.ts:1180-1192`
  - body: The hook reads provider/model/api from the wrapper input before delegating to the inner stream chain. Existing wrappers can rewrite the model before the real provider call, so per-call telemetry can attribute latency and errors to the wrong provider-bound model; move the observation closer to the final provider boundary or pass final metadata out of the wrapper chain.
  - confidence: 0.74
- **[P2] Use attempt-unique model-call IDs:** `src/agents/pi-embedded-runner/run/attempt.ts:271-272`
  - body: `buildModelCallId` combines `runId`, `sessionId`, and a per-wrapper sequence. The prior review thread notes retry attempts can reuse the run/session identifiers, so the first provider call in a retried attempt can still reuse a call ID; include an attempt-unique token if this raw hook remains per-provider-call telemetry.
  - confidence: 0.72
