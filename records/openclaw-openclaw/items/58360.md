## Summary

Keep open. The request still matters, but current main now has a sanitized model-call hook path and this PR’s raw payload API still bypasses the existing conversation-access policy and can leave unmatched terminal events when streams are abandoned.

## What This Changes

Adds raw `before_model_call` and `after_model_call` plugin hooks with event types, hook-runner methods, embedded-runner `streamFn` wiring, replay payload normalization extraction, and focused tests.

## Best Possible Solution

Decide the public plugin data-access policy first. If sanitized telemetry is sufficient, keep the current `model_call_started` / `model_call_ended` API; if raw model-call payload hooks are still needed, implement them as an explicitly conversation-gated, documented API with terminal-safe lifecycle coverage and changelog/docs updates.

## Reproduction Assessment

Yes. Code inspection reproduces the blockers: the PR exposes raw `requestMessages` while the current conversation-access gate excludes the new hook names, and the proposed terminal hook path can be skipped when a stream is consumed by async iteration and abandoned before `.result()`.

## Solution Assessment

No. The proposed shape is not the narrowest maintainable solution until the raw-payload policy is settled; the safer path is either the current sanitized hook pair or a gated raw-payload extension with stream-finalization coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Gate raw model-call hooks behind conversation access:** `src/plugins/types.ts:2019`
  - body: `before_model_call` exposes provider-facing `requestMessages`, but the new hook is added as a normal typed hook rather than being covered by the existing `allowConversationAccess` gate used for raw conversation hooks. A non-bundled plugin could register for prompt/history payloads without the explicit conversation-data opt-in; add both raw model-call hooks to the conversation hook set with loader tests, or keep the event payload sanitized.
  - confidence: 0.86
- **[P2] Emit after hooks when streams are abandoned:** `src/agents/pi-embedded-runner/run/attempt.ts:1170-1196`
  - body: `after_model_call` is emitted from the wrapped stream’s `.result()` path, so callers that consume the async iterator and return early can skip the terminal event. That leaves tracing plugins with unmatched call IDs; wrap async iteration/finally like the current diagnostic model-call wrapper or document and test a one-sided contract before exposing it.
  - confidence: 0.78
