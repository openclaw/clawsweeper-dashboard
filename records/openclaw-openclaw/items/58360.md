## Summary

Keep open, but not merge-ready: current main already has sanitized `model_call_started` / `model_call_ended` hooks and an open canonical raw per-call observability issue, while this branch adds a conflicting raw payload API without settled access policy, terminal-event semantics, or real behavior proof.

## What This Changes

The branch adds `before_model_call` / `after_model_call` plugin hooks, wraps embedded-runner `streamFn` calls, exposes raw request/response event fields, normalizes replay payloads, and adds focused hook tests.

## Best Possible Solution

Land one maintainer-approved per-call observability contract that keeps sanitized `model_call_*` telemetry as the default, preserves aggregate `llm_output` compatibility, gates any raw payload access through conversation-access policy, and guarantees retry-safe correlation plus terminal events.

## Reproduction Assessment

Yes for the PR review defects: source inspection of the PR head shows the raw payload fields, `.result()` terminal hook path, and retry-id inputs. This is not a current-main bug reproduction because the raw per-call hook is a new public capability.

## Solution Assessment

No as submitted. The useful direction is a single coordinated hook contract built on current `model_call_*` semantics, access gating, retry-safe IDs, and real proof rather than a parallel raw API on an outdated branch.
