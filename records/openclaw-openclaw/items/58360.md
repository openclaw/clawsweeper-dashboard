## Summary

Keep open. The current PR still needs maintainer/security review because it exposes raw prompt/history and response payloads through new public plugin hooks without the existing conversation-access gate, and its lifecycle/correlation semantics are not yet reliable enough for tracing.

## What This Changes

The PR adds before_model_call and after_model_call plugin hooks, hook-runner methods, embedded-runner streamFn wiring for raw request/response payload observation, replay-tool normalization extraction, and focused tests.

## Best Possible Solution

Keep the shipped sanitized model_call_started/model_call_ended hooks as the default telemetry path; if raw payload observation is accepted, add it as an explicitly documented allowConversationAccess-gated API with terminal-safe stream lifecycle coverage and retry-unique call IDs.

## Reproduction Assessment

Yes. Static inspection reproduces the blockers: the PR exposes requestMessages/responseMessage outside the conversation hook set, emits after_model_call only through the stream result path, and derives call IDs from identifiers that can repeat across retry attempts.

## Solution Assessment

No. The proposed raw hook surface is not the best merge path until the data-access policy is settled and the stream lifecycle/correlation bugs are fixed; the safer baseline is the existing sanitized telemetry hook pair or a gated raw-payload extension of that contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Gate raw model-call hooks behind conversation access:** `src/plugins/types.ts:2023`
  - body: `before_model_call` exposes provider-facing `requestMessages` and `systemPrompt`, and `after_model_call` can expose `responseMessage`, but the PR adds these as normal typed hooks. Non-bundled plugins could receive raw conversation content without `plugins.entries.<id>.hooks.allowConversationAccess=true`; either classify both names as conversation hooks with loader coverage or keep these payloads sanitized.
  - confidence: 0.87
- **[P2] Emit after hooks when streams are abandoned:** `src/agents/pi-embedded-runner/run/attempt.ts:1261-1267`
  - body: `after_model_call` is emitted only from the patched `.result()` method, so an async-iterator consumer that breaks early can skip the terminal event and leave tracing plugins with unmatched call IDs. Wrap async iteration/finally like the existing diagnostic model-call wrapper, or document and test a one-sided contract before exposing this hook.
  - confidence: 0.8
- **[P2] Make model-call IDs unique across retry attempts:** `src/agents/pi-embedded-runner/run/attempt.ts:272`
  - body: `buildModelCallId` uses `runId`, `sessionId`, and a wrapper-local sequence, but retry attempts can reuse the same run and session while reinstalling the wrapper and resetting the counter. The first provider call in separate retries can collide; include an attempt-unique token or move the counter outside the per-attempt wrapper.
  - confidence: 0.78
