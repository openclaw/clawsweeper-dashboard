## Summary

Keep open. Current main intentionally provides sanitized `model_call_started` / `model_call_ended` telemetry, while this PR still proposes a raw prompt/response hook API with unresolved access-policy, lifecycle, and proof blockers.

## What This Changes

The PR adds additive `before_model_call` and `after_model_call` plugin hooks around embedded-runner provider invocations, plus hook-runner and embedded-runner tests.

## Best Possible Solution

Keep sanitized model-call telemetry as the default API; add raw payload hooks only if maintainers approve the trust model and the implementation is gated, documented, lifecycle-complete, and proven in a real run.

## Reproduction Assessment

Yes. Source inspection of PR head shows raw `requestMessages` / `responseMessage` hook payloads, result-only terminal emission, and retry-collision-prone call ID construction; no runtime tests were run because this review is read-only.

## Solution Assessment

No. The raw hook API may be useful, but this implementation is not the safest merge path until access gating, stream terminal behavior, attempt-unique IDs, provider-bound metadata, docs/tests, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate raw model-call hooks behind conversation access:** `src/plugins/types.ts:2023`
  - body: `requestMessages` exposes provider-facing prompts/history, and `responseMessage` can expose the full assistant response, but the new hook names are not tied to the conversation-content access policy used for raw conversation hooks. Add these hooks to that gate, with registry tests, before exposing this data to non-bundled plugins.
  - confidence: 0.9
- **[P2] Emit after hooks when streams are abandoned:** `src/agents/pi-embedded-runner/run/attempt.ts:1261-1270`
  - body: `after_model_call` is only fired from the patched `.result()` method. If a caller consumes or abandons the async iterator without calling `.result()`, tracing plugins get an unmatched `before_model_call`; wrap iteration/finally like the diagnostic model-call wrapper or document and test a one-sided contract.
  - confidence: 0.84
- **[P2] Make model-call IDs unique across retry attempts:** `src/agents/pi-embedded-runner/run/attempt.ts:271-272`
  - body: `buildModelCallId` combines `runId`, `sessionId`, and a per-wrapper sequence, but retry attempts can reuse the same run/session while the wrapper counter resets. Include an attempt-unique token or keep the counter outside the retry boundary so downstream correlation cannot merge distinct provider calls.
  - confidence: 0.82
- **[P2] Report provider-bound model metadata:** `src/agents/pi-embedded-runner/run/attempt.ts:1180-1192`
  - body: The hook captures provider/model/api before delegating to inner stream wrappers. Wrappers such as MiniMax fast mode can rewrite the model before the actual provider call, so telemetry can be attributed to the wrong model; emit closer to the final provider boundary or propagate final metadata back.
  - confidence: 0.76
