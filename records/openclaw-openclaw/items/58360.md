## Summary

Keep open, but this PR is not merge-ready. It remains the canonical raw model-call payload hook discussion after the related feature issue was closed in its favor, while current main intentionally provides sanitized `model_call_started` / `model_call_ended` telemetry; the PR still needs security/product judgment, code fixes, and contributor real behavior proof.

## What This Changes

The PR adds raw `before_model_call` and `after_model_call` plugin hooks, wires them around embedded-runner `streamFn` calls, normalizes replay payloads, and adds focused hook-runner tests.

## Best Possible Solution

Keep sanitized `model_call_started` / `model_call_ended` as the default telemetry API, and only add raw before/after payload hooks if maintainers approve the trust policy and the implementation matches lifecycle, metadata, docs, tests, and proof expectations.

## Reproduction Assessment

Yes. Source inspection of PR head shows the raw payload event types, result-only terminal emission, retry-collision-prone call ID construction, and metadata capture before provider-bound rewrites; I did not run tests because this review is read-only.

## Solution Assessment

No. A raw hook surface may be useful, but this implementation is not the safest merge path until access gating, terminal lifecycle behavior, attempt-unique IDs, provider-bound metadata, docs, tests, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate raw model-call hooks behind conversation access:** `src/plugins/types.ts:2023`
  - body: `before_model_call` exposes provider-facing `requestMessages`, and `after_model_call` can expose the complete response message. Add these raw hooks to the same conversation-content access policy used for `llm_input` / `llm_output`, otherwise non-bundled plugins can receive prompt/history/response content without explicit `allowConversationAccess=true`.
  - confidence: 0.9
- **[P2] Emit after hooks when streams are abandoned:** `src/agents/pi-embedded-runner/run/attempt.ts:1261-1270`
  - body: `after_model_call` only fires from the patched `.result()` method. If a caller consumes or abandons the async iterator without calling `.result()`, tracing plugins get an unmatched `before_model_call`; wrap iteration/finally like the current diagnostic model-call wrapper or document and test a one-sided contract.
  - confidence: 0.84
- **[P2] Make model-call IDs attempt-unique:** `src/agents/pi-embedded-runner/run/attempt.ts:271-272`
  - body: `buildModelCallId` combines `runId`, `sessionId`, and a counter that resets when this wrapper is installed. Retry attempts can reuse the same run/session identity, so the first model call in each attempt can collide; include an attempt-unique token or keep the counter outside the retry boundary.
  - confidence: 0.82
- **[P2] Report provider-bound model metadata:** `src/agents/pi-embedded-runner/run/attempt.ts:1180-1192`
  - body: The hook records provider/model/api from the wrapper input before delegating to inner stream wrappers. Wrappers such as MiniMax fast mode can rewrite the model before the real provider call, so telemetry can be attributed to the wrong model; emit closer to the final provider boundary or propagate final metadata back.
  - confidence: 0.76
