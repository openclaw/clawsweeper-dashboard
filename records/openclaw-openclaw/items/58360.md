## Summary

Keep this PR open, but it is not ready to merge: the linked feature request was closed in favor of this PR as the canonical raw-payload hook discussion, and the branch still needs security/product review, lifecycle fixes, unique per-attempt call IDs, and real behavior proof.

## What This Changes

The PR adds raw `before_model_call` and `after_model_call` plugin hooks around embedded-runner provider calls, refactors replay tool-call normalization, and adds focused hook-runner and payload tests.

## Best Possible Solution

Keep `model_call_started` / `model_call_ended` as the default sanitized telemetry API; if raw payload hooks are accepted, gate them under explicit conversation-access policy and add terminal-safe emission, unique attempt-scoped call IDs, provider-bound metadata, docs, and runtime proof.

## Reproduction Assessment

Yes. Source inspection of the PR head reproduces the review blockers: raw payload fields are exposed through new hooks, `after_model_call` depends on `.result()`, and call IDs use identifiers that can repeat across retry attempts.

## Solution Assessment

No. The proposed raw hook surface is not the best mergeable solution until access policy and lifecycle semantics are fixed; the safer path is the existing sanitized telemetry or a gated raw-payload API with the corrections above.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Gate raw model-call hooks behind conversation access:** `src/plugins/types.ts:2023`
  - body: `before_model_call` carries provider-facing `requestMessages`, and `after_model_call` can carry the full response message. Current registry policy only gates names classified as conversation hooks, so these new raw hook names would let a non-bundled plugin receive conversation content without `allowConversationAccess=true`. Add both raw hook names to the conversation-access set with loader coverage, or keep these events sanitized.
  - confidence: 0.87
- **[P2] Emit after hooks when streams are abandoned:** `src/agents/pi-embedded-runner/run/attempt.ts:1261-1270`
  - body: `after_model_call` is fired by replacing the stream's `.result()` method. A caller that consumes the async iterator and exits early without calling `.result()` never emits the terminal event, leaving tracing plugins with unmatched call IDs. Wrap async iteration/finally like the current diagnostic model-call wrapper or document and test a one-sided contract before exposing it.
  - confidence: 0.82
- **[P2] Make model-call IDs unique across retries:** `src/agents/pi-embedded-runner/run/attempt.ts:271-272`
  - body: `buildModelCallId` combines `runId`, `sessionId`, and a counter that resets when `runEmbeddedAttempt` is re-entered. The retry loop passes the same active session ID back into each attempt, so the first provider call in separate retries can reuse the same call ID. Include an attempt-unique token or carry the counter outside the retry boundary.
  - confidence: 0.82
- **[P2] Report provider-bound model metadata:** `src/agents/pi-embedded-runner/run/attempt.ts:1180-1192`
  - body: The hook captures `provider`/`model`/`api` before delegating to the inner stream chain. Existing wrappers can rewrite the model before the real provider call, so the new per-call telemetry can attribute latency/errors to the wrong model. Move this observation closer to the final provider boundary or pass final metadata back from the wrapper chain.
  - confidence: 0.74
