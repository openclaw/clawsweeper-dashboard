## Summary

Keep open: the model-call observability gap is real, but this branch is not merge-ready because it conflicts with current main, overlaps the existing sanitized `model_call_*` hook contract, exposes raw conversation payloads without a settled access boundary, and has only unit-test proof.

## What This Changes

Adds `before_model_call` and `after_model_call` plugin hooks, embedded-runner stream wrapping, raw model request/response event fields, replay normalization, and targeted hook tests.

## Best Possible Solution

Ship one documented per-call observability contract that keeps sanitized `model_call_*` telemetry as the default, gates any raw payload access through the existing conversation-access policy, and covers retry plus abandoned-stream terminal behavior.

## Reproduction Assessment

## Solution Assessment

No as submitted. The useful direction is an additive, documented per-call observability contract, but it should be reconciled with current `model_call_*` hooks and the plugin conversation-access policy before raw payloads become public API.

  - confidence: 0.87

  - body: The after hook is fired by monkey-patching `.result()`, so a stream that is iterated or abandoned without calling `.result()` leaves plugins with an unmatched before event. Current main's diagnostic wrapper observes async iteration and early return; use that style of terminal handling for the public hook too.
