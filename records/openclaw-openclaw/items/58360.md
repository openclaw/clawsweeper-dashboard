## Best Possible Solution

Land one documented per-call observability contract that either extends the current sanitized model_call_* telemetry or deliberately adds a gated raw-payload API with explicit access policy, terminal-event guarantees, tests, docs, and real plugin proof.

## Solution Assessment

No as submitted. The maintainable path is a rebase and maintainer-approved API/security design that reuses or deliberately extends current model_call_* telemetry instead of adding an ungated parallel raw payload contract.

- **[P1] Gate raw model payload access:** `src/plugins/types.ts:2006-2049`
  - body: The new hook events expose `systemPrompt`, `requestMessages`, and `responseMessage` to plugins, but the branch does not wire these hooks into the current conversation-access policy.   - confidence: 0.92

  - body: Current main already exposes sanitized per-provider-call `model_call_started` and `model_call_ended` hooks, but this branch adds a parallel `before_model_call`/`after_model_call` API with overlapping lifecycle semantics. Rebase and extend the current contract deliberately, or document a clear deprecation/upgrade relationship, to avoid two competing plugin APIs for the same event family.
  - confidence: 0.88
- **[P2] Emit terminal events without relying on result():** `src/agents/pi-embedded-runner/run/attempt.ts:1251-1261`
  - body: `after_model_call` is emitted by replacing the stream's `.result()` method, so a caller that consumes or abandons the async iterator without calling `.result()` leaves plugins with an unmatched before event. Use the current iterator/finally-style terminal handling pattern or narrow the contract before exposing this as a public tracing API.
  - confidence: 0.84
