## Summary

Keep open for maintainer judgment: the observability goal is real, but this branch is not mergeable as-is because it exposes raw model payloads without an access gate, has lifecycle/correlation gaps, lacks real behavior proof, and overlaps the newer gated preflight direction in https://github.com/openclaw/openclaw/pull/77256.

## What This Changes

The PR adds `before_model_call` and `after_model_call` plugin hook contracts, hook-runner methods, embedded-runner `streamFn` wiring, replay-normalization extraction, and targeted tests.

## Best Possible Solution

Keep sanitized `model_call_*` telemetry as the default; accept raw payload hooks only as a gated, documented, lifecycle-complete API with maintainer security approval and real plugin proof.

## Reproduction Assessment

Yes for the review findings by source inspection. PR head exposes raw `requestMessages` and `responseMessage`, emits `after_model_call` from `.result()` only, and derives call IDs from retry-reused identifiers.

## Solution Assessment

No. The useful observability direction should be solved by the existing sanitized hooks or by a raw hook design that is explicitly gated, provider-bound, lifecycle-complete, documented, tested, and proven in a real run.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate raw model-call hooks behind conversation access:** `src/plugins/types.ts:2023-2049`
  - body: `requestMessages` exposes provider-facing prompt/history, and `responseMessage` can expose the full assistant response, but the new hook names are not tied to the conversation-content access policy. Add these hooks to that gate, with registry coverage, before exposing this data to non-bundled plugins.
  - confidence: 0.9
- **[P2] Emit after hooks when streams are abandoned:** `src/agents/pi-embedded-runner/run/attempt.ts:1261-1270`
  - body: `after_model_call` is only fired from the patched `.result()` method. If a caller consumes or abandons the async iterator without calling `.result()`, tracing plugins get an unmatched `before_model_call`; wrap iteration/finally like the diagnostic model-call wrapper or document and test a one-sided contract.
  - confidence: 0.84
- **[P2] Make model-call IDs unique across retry attempts:** `src/agents/pi-embedded-runner/run/attempt.ts:271-272`
  - body: `buildModelCallId` combines `runId`, `sessionId`, and a wrapper-local sequence, but retry attempts can reuse the same run and session while the wrapper counter resets. Include an attempt-unique token or keep the counter outside the retry boundary so downstream correlation cannot merge distinct provider calls.
  - confidence: 0.82
- **[P2] Report provider-bound model metadata:** `src/agents/pi-embedded-runner/run/attempt.ts:1180-1192`
  - body: The hook captures provider/model/api before delegating to inner stream wrappers. Wrappers can rewrite the model before the actual provider call, so telemetry can be attributed to the wrong model; emit closer to the final provider boundary or propagate final metadata back.
  - confidence: 0.76
