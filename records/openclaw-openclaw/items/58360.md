## Summary

Keep this PR open for maintainer/security review, not cleanup-close: current main has sanitized per-call telemetry, but this PR proposes a new raw provider-payload hook surface with unresolved access-policy, lifecycle, metadata, and proof blockers.

## What This Changes

The PR adds raw `before_model_call` and `after_model_call` plugin hooks around embedded-runner provider calls, plus hook-runner types, embedded-run wiring, replay payload normalization, and focused tests.

## Best Possible Solution

Keep `model_call_started` / `model_call_ended` as the default sanitized telemetry API; if raw payload hooks are accepted, add them with explicit conversation-access gating, terminal-safe emission, attempt-unique call IDs, provider-bound metadata, docs, tests, and redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection of PR head shows the concrete review blockers: raw payload fields are exposed through new hooks, terminal emission depends on `.result()`, call IDs are retry-collision prone, and model metadata is captured before later provider-bound rewrites.

## Solution Assessment

No. A raw hook surface is not the narrowest mergeable solution until the access policy and lifecycle semantics match the current sanitized telemetry contract; a gated raw-payload API or continued sanitized telemetry is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Gate raw model-call hooks behind conversation access:** `src/plugins/types.ts:2023`
  - body: `before_model_call` carries provider-facing `requestMessages`, and `after_model_call` can carry the full response message. The PR adds new raw hook names without classifying them under the conversation-content access policy, so a non-bundled plugin could receive prompt/history/response content without `allowConversationAccess=true`.
  - confidence: 0.87
- **[P2] Emit after hooks when streams are abandoned:** `src/agents/pi-embedded-runner/run/attempt.ts:1261-1270`
  - body: `after_model_call` is emitted only from the patched `.result()` method. If a caller consumes or exits the async iterator without calling `.result()`, tracing plugins get an unmatched `before_model_call`; wrap iteration/finally like the current diagnostic model-call wrapper or document and test a one-sided contract.
  - confidence: 0.82
- **[P2] Make model-call IDs unique across retries:** `src/agents/pi-embedded-runner/run/attempt.ts:271-272`
  - body: `buildModelCallId` uses `runId`, `sessionId`, and a counter that resets each time `runEmbeddedAttempt` installs the wrapper. Retry loops reuse the same run and session IDs, so separate attempts can emit the same first call ID; include an attempt-unique token or carry the counter outside the retry boundary.
  - confidence: 0.82
- **[P2] Report provider-bound model metadata:** `src/agents/pi-embedded-runner/run/attempt.ts:1180-1192`
  - body: The hook captures provider/model/api from the wrapper input before delegating to inner stream wrappers. Existing wrappers can still rewrite the model before the actual provider call, so per-call telemetry can be attributed to the wrong model; observe closer to the final provider boundary or propagate final metadata back.
  - confidence: 0.74
