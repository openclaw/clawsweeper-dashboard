The PR adds session-store normalization and doctor repair to strip transient lifecycle, reply-turn, runtime model, fallback, and automatic auth fields from normal channel sessions while preserving internal execution sessions.

## Best Possible Solution

Keep the fallback/auth cleanup, but scope it so durable lifecycle recovery state is preserved or replaced with an explicit recovery marker and covered by interrupted normal-session regression tests.
