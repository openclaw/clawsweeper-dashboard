Fix the OpenClaw issue where `agent.send` auto-rotates a `sessionKey` to a new `sessionId` without emitting plugin session lifecycle hooks. In `src/gateway/server-methods/agent.ts`, preserve the existing `sessions.changed` create/send emissions while reusing the existing helpers from `src/gateway/session-reset-service.ts` so a new rotated session emits `session_start` exactly once. Add focused regression coverage in the agent handler tests; keep reset/create behavior unchanged. Add a changelog entry if maintainers consider the plugin lifecycle fix user-visible.
