- `impact:session-state`: The affected behavior is session ID rotation and lifecycle notification for plugins that track session state.

## Summary

Keep open: current main and v2026.5.20 still show `agent.send` rotating session IDs and emitting `sessions.changed` without calling the existing `session_start` lifecycle helper, while reset/create paths do call that helper.

## What This Changes

The issue requests `agent.send` session auto-rotation to emit the existing `session_start` plugin hook when it creates or replaces a session ID.

## Best Possible Solution

Route the `agent.send` `isNewSession` transition through the existing gateway session lifecycle helper, preserve the current `sessions.changed` events, and cover stale daily rotation plus explicit sessionId replacement.

## Reproduction Assessment

Yes, at source level. The current `agent.send` path can rotate a stale or explicitly replaced `sessionKey` to a new session ID and emits only `sessions.changed`, while reset/create paths call the lifecycle hook helper.

## Solution Assessment

Yes. Reusing the existing lifecycle helper for the existing `isNewSession` transition is the narrow maintainable fix, with tests proving the hook and existing `sessions.changed` events fire once.
