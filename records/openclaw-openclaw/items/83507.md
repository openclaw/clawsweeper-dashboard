## What This Changes

The issue requests that `agent.send` auto-rotation emit the same `session_start` plugin hook as other session-creation paths.

## Best Possible Solution

Land a corrected gateway fix that reuses the existing session lifecycle hook helper for `agent.send` auto-rotation, preserves `sessions.changed`, and adds regression coverage for stale daily rotation and explicit replacement session IDs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: run `agent.send` against a stale or explicitly replaced `sessionKey` and current `main` will emit `sessions.changed` without invoking the `session_start` hook helper.

## Solution Assessment

Yes conceptually. The maintainable fix is to route the existing `isNewSession` transition through the established lifecycle helper, but the open PR must be corrected and reviewed before this issue can close.
