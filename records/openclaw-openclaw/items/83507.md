## What This Changes

The issue asks for `agent.send` session auto-rotation to emit the same plugin session lifecycle hook used by other session creation/reset paths.

## Reproduction Assessment

Yes. Source inspection on current main shows `agent.send` can rotate/create a session entry and emit `sessions.changed`, while `agent.ts` has no lifecycle hook call and reset/create paths use `emitGatewaySessionStartPluginHook`.

## Solution Assessment

Yes. The best fix is the narrow existing-helper path represented by the linked PR: emit the established lifecycle hooks from the `agent.send` new-session transition while preserving existing `sessions.changed` behavior and adding regression coverage.
