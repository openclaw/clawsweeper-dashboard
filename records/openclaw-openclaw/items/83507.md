## Summary

This should stay open: current main and the latest release still show the reported session lifecycle asymmetry, and the closed unmerged fix attempt did not land. The issue is a narrow gateway/plugin lifecycle bug with clear source evidence and no open competing PR.

## What This Changes

`agent.send` should emit the documented `session_start` plugin hook when it auto-rotates a `sessionKey` to a new `sessionId`.

## Best Possible Solution

Reuse the existing `emitGatewaySessionStartPluginHook` helper in the `agent.send` auto-rotation path after the new session is persisted, preserving `sessions.changed` and adding regression coverage for stale daily rotation plus explicit replacement session IDs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run `agent.send` against a stale or explicitly replaced `sessionKey` and observe that the rotation path emits only `sessions.changed`, while reset/create paths call the `session_start` helper.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing lifecycle hook helper for the already-detected `isNewSession` transition, not add a new plugin API or require plugins to maintain a dual-source workaround.
