## Label Justifications

- `P2`: This is a valid plugin lifecycle bug with limited blast radius: plugins can miss session bookkeeping, but core runtime availability is not broken.
- `impact:session-state`: The affected behavior is session ID rotation and lifecycle notification for plugin-maintained session state.

## Summary

Keep open: current main still rotates `agent.send` sessions through the direct store-update path and only emits `sessions.changed`, while reset/create lifecycle paths emit `session_start`. The linked fixing PR is still open, so this issue should remain open until that PR or an equivalent corrected fix lands.

## What This Changes

`agent.send` should emit the `session_start` plugin lifecycle hook whenever it auto-rotates a session key to a new session ID.

## Best Possible Solution

Land a corrected, tested version of https://github.com/openclaw/openclaw/pull/83523, or close that PR and merge an equivalent narrow fix that reuses the existing session lifecycle hook helper from the `agent.send` rotation path.

## Reproduction Assessment

Yes. Source inspection shows a high-confidence path: `agent.send` can set `isNewSession`, persist the rotated session, and emit only `sessions.changed`, while reset/create paths call the session lifecycle hook helper.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing gateway `session_start` lifecycle helper for the `isNewSession` auto-rotation path and add focused coverage that the hook and existing `sessions.changed` event both fire once.
