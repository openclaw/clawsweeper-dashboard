## What This Changes

The issue requests that `agent.send` session ID auto-rotation emit the same plugin session lifecycle hooks as reset/create session paths.

## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection on current main shows the `agent.send` `isNewSession` path persists a rotated session and emits `sessions.changed`, while reset/create paths call the plugin lifecycle helper.

## Solution Assessment

Yes. Reusing the existing gateway session lifecycle helper in the `agent.send` rotation path, while preserving `sessions.changed` and adding regression coverage, is the narrow maintainable fix; the open PR is the current review vehicle.
