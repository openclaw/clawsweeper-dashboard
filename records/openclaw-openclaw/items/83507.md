## Summary

Keep open: current main still rotates `agent.send` sessions and broadcasts `sessions.changed` without invoking the `session_start` lifecycle hook, while the reset and `sessions.create` opt-in paths do emit it. I found related lifecycle work, but no exact duplicate or open closing PR for this `agent.send` path.

## What This Changes

The issue asks for `agent.send` session rotations to emit the `session_start` plugin hook when a `sessionKey` receives a new `sessionId`.

## Best Possible Solution

Add a narrow `agent.send` lifecycle emission for `isNewSession` rotations using the existing session lifecycle helper, with regression coverage for stale daily rollover, requested-session-id rotation, and fresh-session reuse.

## Reproduction Assessment

Yes, at source level: current `agent.send` rotates stale or changed sessions, persists the new session id, and only emits `sessions.changed`. No live plugin run was performed in this read-only review.

## Solution Assessment

Yes, the best fix is narrow: reuse the existing gateway session lifecycle hook helper in the `agent.send` new-session branch and add focused regression tests. The implementation should preserve existing broadcasts and avoid changing protocol or configuration.
