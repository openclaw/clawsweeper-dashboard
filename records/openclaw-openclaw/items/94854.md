## Summary

Keep open: the updated PR fixed the earlier error-only and reasoning-only predicate problem, but the new gate is still text-only, so valid media-only or rich payload replies can still be dropped when post-turn compaction fails. The external proof is also a focused mocked Vitest shard rather than real after-fix runtime evidence.

## What This Changes

The branch wraps post-turn CLI compaction in `agentCommandInternal` so text-bearing CLI results can continue to delivery after compaction throws, and adds focused regression tests for text, empty, error-only, and reasoning-only payloads.

## Best Possible Solution

Land one canonical agent-command fix that gates soft-failing compaction on actual outbound deliverability, keeps empty/error/reasoning-only results fail-closed, and includes real setup proof.

Yes. Current main persists the CLI transcript, awaits post-turn compaction, and only then calls final delivery, so forcing `runCliTurnCompactionLifecycle` to reject reproduces skipped delivery from source.

## Solution Assessment

No. The owner boundary is plausible, but the best fix should use canonical outbound deliverability rather than a text-only predicate, or maintainers should choose the sibling https://github.com/openclaw/openclaw/pull/94928 shape.
