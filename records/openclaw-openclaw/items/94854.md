## Summary

Keep open: the branch is a useful lifecycle-owner fix, but it is not merge-ready because the recovery predicate is narrower than the shared delivery predicate and the external proof is still a mocked focused Vitest shard.

## What This Changes

The PR wraps post-turn CLI compaction in `agentCommandInternal` so text-bearing CLI results can continue to delivery after compaction throws, and adds focused regression tests for text, empty, error-only, and reasoning-only payloads.

## Best Possible Solution

Land one canonical lifecycle-owner fix that gates compaction soft-fail on the shared outbound payload projection, keeps empty/error/reasoning-only turns fail-closed, includes real or production-like proof, and then supersedes the duplicate candidate PRs.

## Reproduction Assessment

Yes. Source inspection shows current main persists the CLI transcript, awaits post-turn compaction, and only then calls final delivery; forcing `runCliTurnCompactionLifecycle` to reject reproduces skipped delivery before `deliverAgentCommandResult`.

## Solution Assessment

No. The lifecycle-owner shape is plausible, but this PR's text-only predicate is narrower than the shared outbound payload plan, so the best fix should use `normalizeOutboundPayloads` or the equivalent delivery projection.
