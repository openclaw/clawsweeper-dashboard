## Best Possible Solution

Resolve trusted policies through the same live registry composition and ownership rules as the global hook runner, then add a loader-style regression for pinned gateway policies after a later default activation.

## Reproduction Assessment

Yes. Source inspection shows the existing loader test state where `getGlobalPluginRegistry()` points at the later default registry while the global runner still sees pinned gateway hooks; this PR would inspect only that later registry for trusted policies.

## Solution Assessment

No. The patch is a plausible partial fix, but the best fix has to compose trusted-policy lookup with the same live registry view used by the hook runner, not the single last-initialized registry.
