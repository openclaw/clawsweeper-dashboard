## Summary

Close as ClawHub/plugin work: the requested hard outbound-send policy is useful, but current plugin lifecycle hooks already expose enough route metadata to cancel configured top-level sends without adding a new built-in Slack/Discord config key.

## What This Changes

Requested per-channel configuration to reject outbound Slack/Discord sends that target a channel without a valid thread id.

## Best Possible Solution

Publish this as a ClawHub policy plugin that uses outbound message hooks to cancel configured top-level sends, and reopen core only if that plugin needs a missing SDK hook or route metadata field.

Not applicable as a feature request; current source shows top-level sends are allowed unless a plugin or caller supplies a stricter policy.

## Solution Assessment

No for core config: the narrower maintainable solution is an external policy plugin using message_sending or reply_payload_sending hooks, with core changes only if the plugin API lacks a needed field.
