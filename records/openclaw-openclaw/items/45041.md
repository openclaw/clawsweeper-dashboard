## Summary

Keep open. Current main still lacks the requested `outputFilters`/`outputFilter` config surface; the existing sanitizer and `message_sending` hook are related mechanisms, but they do not settle the per-agent configurable filtering API, and open PR #60551 is still only an overlapping implementation candidate.

## What This Changes

The issue requests optional per-agent output regex/pattern filters that strip leaked reasoning or planning text from assistant messages before channel delivery.

## Best Possible Solution

Keep this as the product/API tracker until maintainers choose one explicit path: a conservative shared cleaner, a documented opt-in config surface, or a ClawHub plugin backed by stable hook context.

## Reproduction Assessment

Yes for the requested surface gap: current main has no `outputFilters`/`outputFilter` key in config types, strict schema, docs, or reply normalization. A live multi-agent Discord run is not required to verify that absence.

## Solution Assessment

Unclear. Regex filtering would address the examples, but heuristic cleanup in #60551 and a `message_sending` plugin are plausible alternatives, so the API boundary should be chosen before implementation.
