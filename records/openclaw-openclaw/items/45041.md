## Summary

Keep open: current main and the latest release still lack the requested global or per-agent output filter contract. Existing sanitizers and the `message_sending` hook can remove or rewrite some outbound text, but they do not settle the requested configurable, agent-scoped, security-sensitive behavior; the related sanitizer PR is narrower than this feature request.

## What This Changes

Requests optional global and per-agent output-level regex or pattern filters to strip leaked reasoning and meta-planning text before assistant messages are delivered to channels.

## Best Possible Solution

Pick one explicit boundary: a guarded opt-in core config with schema/docs/preservation tests/live channel proof, or a blessed plugin path with the minimal agent-scoped hook context needed for safe per-agent filtering.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this requests a new configurable output-filter capability. Source inspection on current main and `v2026.5.22` confirms the requested config surface is absent.

## Solution Assessment

Unclear. A raw regex filter is flexible but security-sensitive and over-stripping-prone, so maintainers should first choose whether the durable solution belongs in core config or a plugin/API path.
