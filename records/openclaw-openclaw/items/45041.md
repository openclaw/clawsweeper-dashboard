## Summary

Keep open for maintainer product/API review. Current main still lacks the requested `outputFilters` config surface, while the existing sanitizer only removes fixed scaffolding classes and the plugin `message_sending` hook plus open PR #60551 create plausible alternate paths rather than a settled implementation.

## What This Changes

Request an optional output-level regex/pattern filter, configurable per agent, to strip leaked reasoning or meta-planning text from assistant messages before channel delivery.

## Best Possible Solution

Maintain this as the tracker until maintainers choose one explicit path: land a conservative shared cleaner, add a documented opt-in filter/config or plugin hook with agent-scoped context, or steer this to a ClawHub plugin built on `message_sending` with clear docs.

## Reproduction Assessment

Yes for the requested surface gap: current main has no `outputFilters`/`outputFilter` key in config types, strict schema, docs, or delivery normalization. A live multi-agent Discord run is not required to verify that absence.

## Solution Assessment

Unclear. Regex filtering could solve the examples, but #60551's heuristic sanitizer and a plugin based on `message_sending` are plausible safer alternatives, so maintainers should choose the API boundary before implementation.
