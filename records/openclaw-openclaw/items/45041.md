## Summary

Keep open: current main has sanitizer paths for tagged/tool/memory scaffolding, but it does not implement the requested opt-in regex/pattern output filter, and the related open PR has not merged.

## What This Changes

Request an optional per-agent output filter that strips reasoning or meta-planning text from assistant replies before channel delivery.

## Best Possible Solution

Decide the product boundary first, then land either a conservative shared last-mile sanitizer or a documented opt-in per-agent filter with schema/help/docs and focused sanitizer plus reply-delivery tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a new configurable filtering feature request. The missing capability is source-verifiable because current config types/schema/docs do not expose `outputFilters`.

## Solution Assessment

Unclear: an arbitrary regex output filter is not obviously the safest default design. The narrower related PR may be a better first implementation, but maintainers should choose between that sanitizer approach and an explicit config surface.
