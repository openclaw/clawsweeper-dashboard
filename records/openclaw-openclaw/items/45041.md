## Summary

Keep open: current main and the latest release still lack the requested configurable global or per-agent output filter, and the related open sanitizer work is narrower hardening rather than a settled config/privacy contract.

## What This Changes

The issue requests an opt-in output-level regex or pattern filter for assistant messages before they are sent to channels.

## Best Possible Solution

Maintainers should first settle the privacy/product contract, then either land conservative sanitizer hardening or add a guarded opt-in config surface with aligned types, schema, docs, and false-positive regression tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a feature/config request. Source inspection does confirm that the requested `outputFilters` surface is absent on current main and v2026.5.12.

## Solution Assessment

Unclear. A raw regex filter is flexible but privacy-sensitive and over-stripping-prone; the safer path is a maintainer-approved sanitizer/config contract with explicit false-positive coverage.
