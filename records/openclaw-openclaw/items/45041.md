## Summary

Current main still has only code-owned sanitizers and plugin message hooks, not the requested configurable per-agent/global output filter, and the open related sanitizer PR is narrower than the requested product surface.

## What This Changes

The issue requests an opt-in global or per-agent output filter that strips configured reasoning or meta-planning patterns from assistant messages before channel delivery.

## Best Possible Solution

Choose an explicit, tested boundary: either a guarded opt-in core config with schema/docs/preservation coverage, or a plugin-backed filter path with the minimal agent-scoped hook context needed to implement it safely.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this requests a new configurable output-filter capability. Source and release inspection confirm the requested config surface is absent on current main and v2026.5.22.

## Solution Assessment

Unclear. A raw regex filter is flexible but over-stripping-prone, so maintainers should first choose whether the durable solution belongs in core config or in a plugin/API path with agent-scoped context.
