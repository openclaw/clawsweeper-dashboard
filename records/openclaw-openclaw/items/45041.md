## Summary

Keep open: current main has static assistant-visible sanitizers and a documented `message_sending` hook that can rewrite outbound content, plus an active narrower sanitizer PR at https://github.com/openclaw/openclaw/pull/60551, but it still lacks the requested global or per-agent `outputFilters` config contract. The remaining decision is privacy/security-sensitive product scope: guarded core config versus a plugin/ClawHub hook path, possibly with additional hook context such as https://github.com/openclaw/openclaw/issues/66235.

## What This Changes

The issue requests an opt-in global or per-agent outbound output filter that can strip reasoning or meta-planning text from assistant messages before channel delivery.

## Best Possible Solution

Choose one explicit boundary: either add a guarded opt-in core config with schema, docs, false-positive tests, and live channel proof, or bless a ClawHub/plugin `message_sending` filter path and add any missing hook context needed for reliable filtering.

## Solution Assessment

Unclear. A raw regex filter is flexible but security-sensitive and over-stripping-prone, so maintainers should first choose a guarded core contract or plugin/API path with false-positive coverage.
