## Summary

Keep open. Current main still lacks the requested `outputFilters`/`outputFilter` config surface, and the existing sanitizer plus `message_sending` hook are related mechanisms rather than a settled product/API decision.

## What This Changes

Request an optional per-agent output filter config that strips regex or pattern matches from assistant messages before channel delivery.

## Best Possible Solution

Keep this as the tracker until maintainers choose one explicit path: a conservative shared cleaner, a documented opt-in config/API, or a ClawHub plugin built on stable hook context.

## Solution Assessment

Unclear. Regex filtering would address the examples, but heuristic cleanup in #60551 and a `message_sending` plugin are plausible safer alternatives, so the API boundary should be chosen before implementation.
