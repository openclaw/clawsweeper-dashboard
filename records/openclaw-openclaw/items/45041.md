## Summary

Keep open: current main has structured reasoning/scaffolding sanitizers, but it does not provide the requested configurable output filter or a merged last-mile cleaner for the plain planning and CJK examples. The open related PR is the active implementation path, so this should not be closed ahead of maintainer review.

## What This Changes

The issue requests an optional per-agent output-level pattern filter that strips leaked reasoning or meta-planning text before assistant messages are sent to channels.

## Best Possible Solution

Review and refine the shared outbound sanitizer in the open related PR, or make an explicit product decision to add a documented per-agent filter config with conservative defaults and regression coverage.

## Reproduction Assessment

No, not as a live current-main multi-agent Discord reproduction. Source inspection does show the requested output-filter config is absent and that current shared sanitizers do not strip the issue’s plain planning-preamble examples.

## Solution Assessment

Unclear. A raw regex filter is not obviously the safest long-term contract; a conservative shared sanitizer plus a maintainer decision on optional per-agent filters is the narrower maintainable path.
