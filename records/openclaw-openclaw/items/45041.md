## Summary

Keep open: current main and the latest release still lack the requested configurable global or per-agent output filter, and the existing static sanitizer plus `message_sending` hook path does not settle the product/security boundary. Add optional global or per-agent output-level regex/pattern filters that strip leaked reasoning or meta-planning text from assistant messages before channel delivery.

## Best Possible Solution

Choose one explicit boundary: either a guarded opt-in core config with schema, docs, false-positive tests, and live channel proof, or a plugin/API path with the minimal hook context needed for per-agent filtering.
