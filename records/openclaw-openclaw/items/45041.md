## Summary

Keep open. Current main still lacks the requested `outputFilters`/`outputFilter` config surface; the existing fixed sanitizer and `message_sending` hook are adjacent mechanisms, and https://github.com/openclaw/openclaw/pull/60551 is still only an overlapping implementation candidate rather than a merged API decision.

## What This Changes

Requests an opt-in, configurable output filter that strips reasoning or meta-planning patterns from assistant messages before channel delivery.

## Best Possible Solution

Keep this as the product/API tracker until maintainers choose a bounded path: documented opt-in config, a conservative shared cleaner, or enough stable hook context for a ClawHub plugin.

## Reproduction Assessment

Yes for the requested surface gap: current main has no `outputFilters`/`outputFilter` key in config types, strict schema, docs, or reply normalization. A live multi-agent Discord run is not required to verify that the requested API is absent.

## Solution Assessment

Unclear. Regex filtering would address the examples, but it can over-strip legitimate replies, and the open heuristic-cleaner PR plus plugin-hook approaches are plausible alternatives that need maintainer API judgment.
