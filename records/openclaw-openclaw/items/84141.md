## What This Changes

The issue asks for isolated cron agent turns with payload.toolsAllow requesting exec or local file tools to either receive those tools or fail loudly before model dispatch.

## Best Possible Solution

Land a focused fix, or a replacement for the active PRs, that either hydrates explicitly requested local tools safely or fails cron preflight with a structured missing-tool error before model dispatch, then verify it on a real upgraded gateway cron run.

Yes. Current source shows the finite Codex allowlist path filters out exec/read/write before allowlist application and disables native Code Mode for finite allowlists, and the issue has independent live reports through v2026.5.20.

## Solution Assessment

Yes on the requested end state, but the exact implementation needs maintainer security review. The safest direction is to keep native Code Mode fail-closed for unrelated narrow allowlists while explicitly satisfying or preflighting required local tools for cron.
