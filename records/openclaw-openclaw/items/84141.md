## Summary

Keep open: current main still forwards cron `toolsAllow`, but the Codex app-server path removes OpenClaw `exec`/`process` before applying the allowlist and disables Codex native shell for restricted allowlists; the discussion now also has an independent macOS live repro on v2026.5.18.

## What This Changes

The issue reports that isolated Codex-backed cron `agentTurn` jobs requesting `exec` via `payload.toolsAllow` can run without any command-execution tool and sometimes still be marked successful.

## Best Possible Solution

Keep the issue open until maintainers add a Codex-specific path that either hydrates an explicitly requested OpenClaw executable tool for cron or fails cron preflight loudly before model invocation, while preserving restricted-allowlist shell boundaries.

## Reproduction Assessment

Yes, at source level: current main forwards cron `toolsAllow` but removes `exec` before the Codex allowlist filter and disables native shell for restricted allowlists. Reporter comments add two independent live v2026.5.18 reproductions, but this review did not run a live cron job.

## Solution Assessment

Unclear until maintainers choose the security boundary. The narrow maintainable fix is Codex-specific dynamic `exec` hydration for explicit cron requests or a loud preflight failure, not globally enabling native shell for all restricted allowlists.
