## Summary

Keep open: current main still has a source-reproducible isolated cron tool-surface gap, and the active repair work is unmerged. The cron path forwards `toolsAllow`, but the Codex app-server still removes local command/file tools before finite allowlist filtering and does not fail the cron run preflight when requested tools are absent.

## What This Changes

Ensure isolated cron agent turns that declare `toolsAllow` receive required `exec` and local tools, or fail before model dispatch instead of appearing successful without running commands.

## Best Possible Solution

Land or rework one canonical fix that preserves narrow-allowlist security boundaries, explicitly hydrates requested OpenClaw dynamic tools or fails cron preflight, and verifies a real isolated cron run after upgrade.

## Reproduction Assessment

Yes at source level: current main forwards `toolsAllow` into the Codex run, but finite allowlists disable native Code Mode while the dynamic filter removes `exec` and file tools before allowlist filtering. The issue comments add live shipped-version repros on Linux and macOS, including 2026.5.20.

## Solution Assessment

Yes, the requested direction is the right shape if it preserves the security boundary: required tools should be present before dispatch or the cron run should fail with a typed preflight error. Current main is not sufficient, and the open PRs need maintainer review to choose the safest canonical repair.
