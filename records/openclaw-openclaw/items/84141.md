## Summary

## What This Changes

Ensure isolated cron `agentTurn` jobs that declare `toolsAllow` with `exec` or other local tools either receive those tools or fail before model dispatch.

## Best Possible Solution

Keep this issue as the canonical tracker until a focused fix lands that either hydrates explicitly requested local tools for isolated Codex cron runs under the approved security boundary or fails preflight before any model turn.

## Reproduction Assessment

Yes at source level: current main forwards cron `toolsAllow` into embedded runs, but the Codex path filters out `exec`/file tools before final allowlist filtering and disables native shell/file fallback for finite allowlists. I did not run a live cron job in this read-only review.

## Solution Assessment

Yes in direction, but the exact implementation needs maintainer security review: required tools should either be hydrated under existing deny/sandbox/runtime gates or fail preflight before model invocation. The open draft PR is the right review target rather than closing this issue early.
