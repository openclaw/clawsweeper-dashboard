## Summary

Keep open: current main still has the source-level cron/Codex tool-surface gap, and later comments report the same missing local tools on the shipped 2026.5.20 release. Two open PRs now target the issue, so the next step is maintainer review of the security-sensitive exec exposure path rather than cleanup closure.

## What This Changes

The issue reports that isolated cron `agentTurn` jobs with `toolsAllow` including `exec` can start without an executable tool surface and still complete with success-shaped output.

## Best Possible Solution

Review and land a narrowed fix that keeps restricted native Codex surfaces fail-closed, restores only explicitly requested OpenClaw dynamic local tools when policy permits, and fails cron preflight before model dispatch when required tools are unavailable.

## Reproduction Assessment

Yes at source level: current main forwards cron `toolsAllow`, then the Codex path filters `exec` and other local tools before the allowlist while disabling native shell/file access for narrow allowlists. The issue also has independent live repro comments on 2026.5.18 and 2026.5.20.

## Solution Assessment

No for current main: it preserves a valid fail-closed native-surface rule but leaves explicit `exec` cron requests without an executable fallback or preflight failure. The maintainable fix should either hydrate the allowed dynamic local tools or fail before model invocation with a clear required-tool diagnostic.
