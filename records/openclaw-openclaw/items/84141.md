## What This Changes

Isolated cron `agentTurn` jobs that request `exec` through `payload.toolsAllow` should receive command-execution tooling or fail before model dispatch with a clear missing-tool error.

## Best Possible Solution

Keep the issue open and land a single maintainer-approved fix that either restores explicitly requested Codex dynamic `exec`/file fallbacks under restricted allowlists or fails isolated cron before model dispatch, with tests and redacted real cron proof.

## Reproduction Assessment

Yes, at source level: current main forwards `toolsAllow` into the embedded run, then the Codex path excludes `exec`/file tools before final allowlist filtering while disabling native shell/file mode for finite allowlists. I did not run a live cron in this read-only review, but reporters added independent 2026.5.18 and 2026.5.20 live evidence.

## Solution Assessment

No, current main is not the best solution because it preserves a valid fail-closed native-shell rule but leaves explicit cron `exec` requests without a usable tool or preflight failure. The best fix is a narrow dynamic-tool restoration or loud required-tool preflight that keeps unrelated restricted allowlists fail-closed.
