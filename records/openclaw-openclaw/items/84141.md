## Summary

Keep open: current main still has a source-reproducible Codex isolated-cron tool-surface gap, and the latest release context includes a 2026.5.20 live report that `toolsAllow` still omitted requested local tools.

## What This Changes

The issue reports that isolated cron `agentTurn` jobs with `toolsAllow` including `exec` can start without an executable tool and can still appear successful.

## Best Possible Solution

Land one maintainer-approved fix path that preserves fail-closed native Codex behavior for unrelated narrow allowlists while either hydrating explicitly requested OpenClaw `exec`/file dynamic tools or failing cron preflight before model dispatch.

## Reproduction Assessment

Yes, at source level: current main forwards `toolsAllow`, then the Codex app-server path removes `exec` before the final allowlist and disables the native shell surface for finite allowlists. I did not execute a live cron run in this read-only review, but issue comments include independent live reports on 2026.5.18 and 2026.5.20.

## Solution Assessment

Unclear until maintainer review: restoring explicit dynamic fallbacks is the narrowest direction, but the exact fix must avoid widening native shell/file access and must account for sandbox, node, CLI, and fail-loud cron behavior.
