## What This Changes

The issue reports isolated Codex cron agent turns on 2026.5.18 missing the requested `exec` tool even when `payload.toolsAllow` explicitly includes `exec`.

## Best Possible Solution

Keep Codex native shell fail-closed for restrictive allowlists, but add a narrow path that either hydrates policy-permitted OpenClaw `exec`/`process` for explicit cron requests or fails cron preflight before model invocation.

## Solution Assessment

No current-main fix exists yet. The maintainable solution is a narrow Codex cron preflight or dynamic-tool hydration path that preserves existing command-execution security gates.
