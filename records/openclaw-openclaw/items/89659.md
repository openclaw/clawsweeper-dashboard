Full review comments:

- **[P1] Do not return fulfilled rate-limit bodies on the final retry:** `extensions/feishu/src/comment-shared.ts:166`
  - body: When the SDK resolves `{ code: 11232 }` or `{ code: 230020 }` on the last attempt, `fulfilledRateLimit` is still set but `attempt < FEISHU_SEND_MAX_RETRIES` is false, so this line returns the rate-limit body as if the request succeeded. That bypasses the synthetic wrapped-error path below, and the new exhaustion test for fulfilled 11232 would fail for the exact case it is meant to cover.
  - confidence: 0.97
- **[P2] Honor reset headers before retrying HTTP 429 sends:** `extensions/feishu/src/comment-shared.ts:154`
  - body: The patch classifies HTTP 429 as retryable and even models `x-ogw-ratelimit-reset` in the test helper, but the retry loop always sleeps only 500 ms then 1000 ms. For gateway quota 429s, both retries can fire before the reset window and still drop the send, so the 429 path needs to parse the reset or retry-after header and cover that timing in tests.
  - confidence: 0.87
