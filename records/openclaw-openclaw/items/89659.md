## What This Changes

This PR adds two bounded linear-backoff retries for Feishu 230020 send rate-limit Axios errors, routes text reply and streaming-card initial sends through the helper, and adds retry/concurrency tests.

## Best Possible Solution

Land this only after the Feishu owner accepts bounded shared-helper retries as the desired delivery mitigation, with any durable queue/throttle behavior tracked separately.

## Reproduction Assessment

## Solution Assessment

Unclear as final product policy, but this is an acceptable bounded mitigation if the Feishu owner accepts it. A durable queue/throttle would be the stronger solution for bursts that exceed two retries, but that is broader than this PR.
