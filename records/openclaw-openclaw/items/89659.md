- Merging changes Feishu outbound delivery semantics: an explicit `230020` send failure can now remain in flight for about 1.5 seconds and issue up to three POST attempts before surfacing failure.

- The Feishu SDK types expose optional create/reply `uuid` fields, but the PR intentionally retries explicit `230020` rejections without adding idempotency keys; this is probably acceptable for rejected rate-limit responses, but it should be accepted by a Feishu owner before merge.
