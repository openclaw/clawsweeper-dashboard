## Summary

## What This Changes

The PR adds bounded retry/backoff for Feishu send rate-limit code 230020, routes text reply and streaming-card send calls through the shared API wrapper, and adds retry/concurrency tests.

## Best Possible Solution

Keep the shared retry shape, but make every retried Feishu message send carry a stable per-logical-send uuid, cover retry uuid reuse and distinct-send uniqueness in tests, then land with redacted live direct-send and reply-send proof.

## Solution Assessment
