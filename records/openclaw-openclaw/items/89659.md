## Risks / Open Questions

- Merging changes Feishu outbound delivery semantics: a 230020 send failure can stay in flight for about 1.5 seconds and issue up to three create/reply API attempts before surfacing failure.
- The Feishu SDK exposes uuid fields for create/reply sends, but this PR retries without setting a stable uuid; explicit 230020 rejections probably make duplicate delivery unlikely, but that boundary should be accepted by a Feishu owner.
- The helper-level retry also affects any current or future requestFeishuApi caller that throws the same 230020 body code, so maintainers should be comfortable that 230020 is message-send-specific in practice.
