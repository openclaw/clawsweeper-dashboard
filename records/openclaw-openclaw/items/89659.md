Full review comments:

- **[P1] Make retried sends idempotent:** `extensions/feishu/src/comment-shared.ts:135-136`
  - body: This helper now retries Feishu message POSTs, but the retried create/reply payloads do not include a stable `uuid` even though Feishu and the SDK expose it for send de-duplication. If an attempt is accepted server-side but the client retries after an ambiguous failure, users can see duplicate messages; generate one uuid per logical send and reuse it for every retry.
  - confidence: 0.9

A resolved `{ code: 230020 }` body therefore still fails without retry, leaving part of the reported Feishu rate-limit loss unfixed.
