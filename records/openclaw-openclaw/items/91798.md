## What This Changes

## Best Possible Solution

Retain the intrinsic-row CSS rule, add browser-level coverage that constrains column height and verifies nonzero title boxes plus vertical overflow, and supply a populated real OpenClaw Workboard screenshot or recording.

## Reproduction Assessment

Yes, from source and the linked issue: a constrained Chromium Workboard with enough cards produces compressed grid rows and zero-height title boxes. This review did not independently run that browser scenario.

## Solution Assessment

Yes for the implementation direction: setting intrinsic implicit-row sizing on the existing scroll container is narrower than changing card internals or adding another layout path. The proposed DOM-only test is not an adequate regression check.
