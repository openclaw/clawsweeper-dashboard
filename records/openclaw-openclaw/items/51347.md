## Summary

Keep open. Current main still lacks a supported Feishu local-root config key and still converts Feishu media upload/access failures into a fallback text send that the CLI formats as a successful delivery.

## What This Changes

The issue requests Feishu `openclaw message send --media` to honor or document local media roots and stop printing success when media upload fails.

## Best Possible Solution

Define the Feishu media failure contract, then implement fail-fast or explicit partial-success reporting and only add a documented local-root config surface if that policy is approved.

## Reproduction Assessment

Yes, at source level. A `sendMediaFeishu` rejection still reaches Feishu outbound's catch block, returns fallback text, and the CLI formats the returned direct result as a successful send.

## Solution Assessment

Unclear. Fail-fast handling for `LocalMediaAccessError` is the narrow fix for misleading success output, but a configurable Feishu root list changes local filesystem read scope and needs owner approval.
