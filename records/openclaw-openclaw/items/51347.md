## Summary

Keep open. Current main has improved generic media-root handling, but Feishu still has no supported `channels.feishu.mediaLocalRoots` config surface and its media-send adapter still turns upload/access failures into a text fallback that the CLI reports as a successful send.

## What This Changes

The issue asks Feishu CLI media sends to honor a configured local media root or fail/document clearly, and to stop reporting success when media upload or access fails.

## Best Possible Solution

Keep this issue open until maintainers choose a Feishu outbound media contract: either add a supported local-root config surface with schema/docs/tests, or document managed-root-only behavior and make required media access failures fail clearly instead of returning an unconditional success.

## Reproduction Assessment

Yes at source level. A live Feishu E2E was not run, but current main still has a clear path where `sendMediaFeishu` rejection is caught by `feishuOutbound.sendMedia`, converted to `sendOutboundText`, and then formatted as a successful direct send.

## Solution Assessment

Unclear. Fail-fast handling for local access errors is the narrow fix for misleading success output, but honoring `channels.feishu.mediaLocalRoots` is a file-read policy/config expansion that needs maintainer product and security approval first.
