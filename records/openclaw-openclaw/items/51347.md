## Summary

Keep open. Current main and the latest release still have a source-reproducible Feishu media status problem: media upload errors are caught and converted into fallback text success, while the Feishu config schema still has no `mediaLocalRoots` contract.

## What This Changes

The issue requests Feishu CLI media sends to honor a configured local-media root or fail/document clearly, and to stop reporting success when media upload failed.

## Best Possible Solution

Define Feishu media failure semantics at the outbound result layer, then implement fail-fast or explicit partial-success output; add any local-root config only with schema, docs, and security-reviewed tests.

## Reproduction Assessment

Yes, at source level. A `sendMediaFeishu` rejection still reaches Feishu outbound's catch block, returns fallback text, and the CLI formatter prints a successful send summary; I did not run a live Feishu send in this read-only review.

## Solution Assessment

Unclear. Fail-fast or explicit partial-success reporting is the narrow fix for misleading success output, but supporting a Feishu local-root config key changes filesystem read/upload scope and needs maintainer policy first.
