## Summary

Keep open. Current main still has the two core gaps: Feishu does not expose or read a supported `channels.feishu.mediaLocalRoots` config key, and Feishu media upload/access failures are still caught and converted into a text fallback that can be formatted as a successful send.

## What This Changes

The issue asks OpenClaw to make Feishu CLI media sends honor configured media roots or clearly document that the config is unsupported, and to stop reporting media upload failures as successful sends.

## Best Possible Solution

Keep this issue open and choose a clear Feishu outbound media contract: either document that Feishu CLI sends only use core-managed/runtime media roots and fail fast on local access violations, or add a supported Feishu `mediaLocalRoots` config surface with schema, generated metadata, docs, merge semantics, and regression tests. In both paths, required media upload/access failures should not be presented as an unconditional successful media send.

## Reproduction Assessment

Yes. The report has concrete CLI steps, and the current source path still supports a high-confidence code-level reproduction: Feishu media errors are caught in `extensions/feishu/src/outbound.ts` and returned as a successful text fallback; live Feishu credentials are only needed for an end-to-end send.

## Solution Assessment

Unclear. The closed unmerged PR #51374 is a useful narrow starting point for fail-fast local allowlist errors, but maintainers still need to decide whether to support channel-level media roots or explicitly document them as unsupported before a final fix should land.
