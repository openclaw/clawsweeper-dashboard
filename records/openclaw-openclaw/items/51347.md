## Summary

Keep open. Current main still has the Feishu media fallback and lacks a supported Feishu `mediaLocalRoots` config surface, while the related fail-fast PR was closed unmerged.

## What This Changes

This issue asks Feishu CLI media sends to honor or document local media roots and to stop reporting success when media upload/access fails.

## Best Possible Solution

Define Feishu outbound local-media policy, then implement either fail-fast `LocalMediaAccessError` handling with clear staging guidance or a documented Feishu `mediaLocalRoots` schema with safe merge semantics and regression coverage.

## Reproduction Assessment

Yes, at source level. Current main still routes `sendMediaFeishu` errors into Feishu text fallback, and the CLI formats that returned direct delivery as a successful send while Feishu schema/docs lack `mediaLocalRoots`.

## Solution Assessment

Unclear. Fail-fast handling is the narrowest fix for misleading success output, but honoring `channels.feishu.mediaLocalRoots` is a new local file-read policy surface that should be approved and documented first.
