## Summary

Keep open: current main and v2026.6.10 still lack a Feishu `mediaLocalRoots` config contract, and Feishu media upload errors still fall back to a successful text-send result. The remaining work needs maintainer product/security direction before changing local-root authorization or delivery-failure semantics.

## What This Changes

The issue reports that Feishu CLI media sends ignore `channels.feishu.mediaLocalRoots` and still report success when media upload fails.

## Best Possible Solution

Choose a security-reviewed local-media root contract shared with outbound media policy, then update Feishu to report media upload failure as either a real failure or explicit partial success with focused regression coverage.

## Reproduction Assessment

Yes, source-level: the strict Feishu config schema still lacks `mediaLocalRoots`, and the Feishu media adapter still catches `sendMediaFeishu` errors into a fallback text send. I did not run a live Feishu credential send in this read-only review.

## Solution Assessment

No, not as an automatic Feishu-only repair. The maintainable fix needs a shared, security-reviewed local-root policy plus explicit Feishu media failure semantics rather than silently adding a parallel config path.
