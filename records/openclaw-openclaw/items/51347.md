## Summary

Keep open. Current main still lacks a supported Feishu `mediaLocalRoots` config surface and still turns `sendMediaFeishu` upload/access failures into fallback text sends that the CLI formats as successful delivery, so the reported behavior remains valid.

## What This Changes

The report asks Feishu CLI media sends to honor configured media roots or fail clearly instead of reporting success when media upload/access fails.

## Best Possible Solution

Define the Feishu outbound media contract, then implement schema/docs/tests for either managed-root fail-fast behavior or an explicit local-root configuration surface with clear security boundaries.

## Reproduction Assessment

Yes, at source level. Current main still catches Feishu media upload/access failures, returns a text fallback result, and formats that direct result as a successful send while the Feishu config/docs lack `mediaLocalRoots`.

## Solution Assessment

Unclear. Fail-fast handling for `LocalMediaAccessError` is the narrow fix for misleading success output, but honoring `channels.feishu.mediaLocalRoots` would add a local file-read/upload policy surface that needs maintainer approval.
