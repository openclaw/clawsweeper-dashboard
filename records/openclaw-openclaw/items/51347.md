## Summary

Keep open. Current main still has a source-reproducible Feishu CLI/plugin send path where media upload failure becomes a text fallback and the plugin action returns success, and the remaining config/status behavior is not owned by any merged or viable related PR.

## What This Changes

The issue requests that Feishu CLI media sends honor the intended media root policy or clearly fail instead of reporting success when media upload fails.

## Best Possible Solution

Define the canonical Feishu outbound media policy: either document/configure supported roots and fail non-delivered required media with non-zero CLI status, or intentionally expose a separate best-effort fallback status that does not claim the media was sent.

## Reproduction Assessment

Yes from source: current Feishu `send` action calls `feishuOutbound.sendMedia`; that function catches media upload errors, sends fallback text containing the local path/URL, and returns a success-shaped result that the CLI formats as sent.

## Solution Assessment

No current implemented solution is sufficient; the narrowest maintainable path is a Feishu/core media outcome contract that distinguishes delivered media from text fallback and a maintainer-approved decision on whether `channels.feishu.mediaLocalRoots` is supported.
