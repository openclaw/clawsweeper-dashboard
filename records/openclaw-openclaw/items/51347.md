## Summary

Keep open. Current main has broader generic local-media root handling than the original report, but Feishu still has no supported `channels.feishu.mediaLocalRoots` config surface and still converts media upload/access failures into a successful text fallback that the CLI reports as sent.

## What This Changes

The issue asks for Feishu CLI media sends to honor local media roots or fail clearly instead of reporting success after media access/upload failure.

## Best Possible Solution

Define the Feishu outbound media contract explicitly: either add a documented and tested local-root config surface with safe merge semantics, or document managed-root-only behavior, and make local media access failures fail clearly instead of returning a successful fallback send.

## Reproduction Assessment

Yes, at source level. Current main still has a clear path where `sendMediaFeishu` rejects, `feishuOutbound.sendMedia` catches the error and returns `sendOutboundText`, and the CLI formats that returned delivery as success; no live Feishu send was run.

## Solution Assessment

Unclear. Fail-fast handling for local access errors is the narrow fix for misleading success output, but honoring `channels.feishu.mediaLocalRoots` is a local file-read policy expansion, so maintainers should choose between supported config and documented managed-root-only behavior first.
