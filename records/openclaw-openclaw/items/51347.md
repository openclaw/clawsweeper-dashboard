## Summary

Keep open. Current main and the latest release still do not define `channels.feishu.mediaLocalRoots`, and Feishu explicit media sends still catch upload errors and fall back to a text/link send that the CLI formats as success.

## What This Changes

The issue reports that Feishu CLI media sends ignore a configured media-root allowlist and report success after media upload failure.

## Best Possible Solution

Define the supported Feishu media-root contract, then make allowlist failures produce either a clear failed send or an explicit degraded-media result instead of an unconditional success summary.

## Reproduction Assessment

Yes. Source inspection shows the Feishu adapter still catches media upload errors and returns a successful fallback text send, and the current schema/manifest do not expose `channels.feishu.mediaLocalRoots`.

## Solution Assessment

Unclear. The fail-fast repair is narrow, but maintainers need to choose whether Feishu should support a channel media-root config, rely on the existing `tools.fs` media policy, or preserve link fallback with explicit degraded status.
