## Summary

Keep open. Current main still lacks a supported Feishu `mediaLocalRoots` config surface and still converts Feishu media upload/access failures into text fallback results that the CLI formats as successful sends, so the reported behavior remains valid.

## What This Changes

The issue asks Feishu CLI media sends to honor configured local media roots or fail/document unsupported paths clearly, and to stop reporting success when media upload/access fails.

## Best Possible Solution

Define the Feishu outbound media contract first, then implement schema/docs/tests for either managed-root fail-fast behavior or an explicit `mediaLocalRoots` config surface with clear security boundaries.

## Reproduction Assessment

Yes, at source level. Current main still catches `sendMediaFeishu` failures, returns a Feishu text fallback, and formats the returned direct delivery as success; Feishu schema/docs still lack `mediaLocalRoots`.

## Solution Assessment

Unclear. Fail-fast handling for `LocalMediaAccessError` is the narrow fix for misleading success output, but honoring `channels.feishu.mediaLocalRoots` would add a Feishu local file-read/upload policy surface that needs maintainer approval.
