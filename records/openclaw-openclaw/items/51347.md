## Summary

Keep open: current main still has a source-reproducible Feishu media failure path where upload errors are caught and converted into fallback text, which lets the higher-level send action report success even when the requested attachment was not delivered.

## What This Changes

The issue asks for Feishu `openclaw message send --media` to honor local media root configuration or fail clearly instead of reporting success after a media upload failure.

## Best Possible Solution

Make Feishu local-media allowlist violations fail fast with clear allowed-root/staging guidance, and either document that Feishu has no channel-specific `mediaLocalRoots` key or add that key as a real schema-backed contract.

## Reproduction Assessment

Yes, source-reproducible: a Feishu media send whose `sendMediaFeishu` call throws a local media access error still enters the catch block, sends fallback text, and returns a successful result to the action/send layer. I did not execute a live Feishu send in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to special-case local media access failures and propagate a clear error instead of using the generic upload-failure fallback. Adding `channels.feishu.mediaLocalRoots` should only happen if schema, docs, and config metadata are updated together.

## Repair Work Prompt

Fix openclaw/openclaw#51347. In Feishu outbound media sends, do not report success when a local media path is rejected by the local media allowlist. Detect `LocalMediaAccessError` / `path-not-allowed` from `sendMediaFeishu` (including wrapped `cause` chains if the existing media loader wraps it) and propagate a clear error with the rejected path, allowed roots or staging guidance, and the macOS tmpdir note, instead of sending fallback text and returning ok. Preserve existing fallback behavior for non-allowlist upload failures only if the current tests/contracts expect that. Treat `channels.feishu.mediaLocalRoots` carefully: current Feishu schema does not define it, so either leave it unsupported and document the supported staging/agent-scoped roots, or add a schema-backed Feishu config option with docs and metadata in the same PR. Add focused regression coverage in `extensions/feishu/src/outbound.test.ts` and, if the plugin action path is affected, `extensions/feishu/src/channel.test.ts` or `src/infra/outbound/outbound-send-service.test.ts`. Add a user-facing changelog entry. Do not change unrelated channel media policy or broaden filesystem read access.
