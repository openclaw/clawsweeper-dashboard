- `impact:message-loss`: The reported path can fail to deliver the intended media while returning a success-shaped Feishu send result.

## Summary

Keep open. Current main still lacks a supported Feishu `mediaLocalRoots` config key and still converts Feishu media upload/access failures into fallback text that the CLI formats as a successful send; the newer fs-root expansion path helps some host-local media sends but does not settle the failure-status contract.

## What This Changes

The issue asks OpenClaw to honor or clearly reject Feishu local-root media configuration and stop reporting CLI success when media upload/access fails.

## Best Possible Solution

Define and document Feishu media failure semantics, implement fail-fast or explicit partial-success reporting, and only add a Feishu-specific local-root config surface after security-reviewed schema/docs/tests approval.

## Reproduction Assessment

Yes, at source level. A `sendMediaFeishu` rejection still reaches Feishu outbound's catch block, returns fallback text, and the CLI formats the plugin-handled send as successful.

## Solution Assessment

Unclear. Fail-fast or explicit partial-success reporting is the narrow fix for misleading success output, but supporting a Feishu local-root config key changes filesystem read/upload scope and needs maintainer/security policy first.
