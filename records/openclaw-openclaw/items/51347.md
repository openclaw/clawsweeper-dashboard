## Summary

Keep open. Current main has generic media-root widening for some local sources, but Feishu still has no supported `channels.feishu.mediaLocalRoots` config surface and its media adapter still converts upload/access failures into a text fallback that the CLI reports as a successful direct send.

## What This Changes

The issue asks Feishu CLI media sends to honor configured local media roots and report media upload/access failures clearly instead of printing success.

## Best Possible Solution

Define the Feishu outbound media contract, then implement schema/docs/tests for either managed-root fail-fast behavior or an explicit `mediaLocalRoots` config surface with clear security boundaries.

## Reproduction Assessment

Yes, at source level. Current main still catches `sendMediaFeishu` failures, returns a text fallback, and formats the returned direct delivery as success; I did not run a live Feishu E2E.

## Solution Assessment

Unclear. Fail-fast handling for `LocalMediaAccessError` is the narrow fix for misleading success output, but honoring `channels.feishu.mediaLocalRoots` would add a local file-read/upload policy surface that needs maintainer approval.
