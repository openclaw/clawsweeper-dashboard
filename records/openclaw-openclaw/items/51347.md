## Summary

Keep open. Current main still has a source-reproducible Feishu media status problem: Feishu media upload failures are caught and converted to fallback text that the CLI reports as a successful send, while the reported Feishu-specific local-root config remains unsupported and security-sensitive.

## What This Changes

The issue asks Feishu CLI media sends to honor configured local media roots or clearly reject unsupported roots, and to avoid success output when media upload fails.

## Best Possible Solution

Define Feishu media failure semantics, implement fail-fast or explicit partial-success reporting, and only add a Feishu-specific local-root config surface with schema, docs, and security-reviewed tests if maintainers approve that scope.

## Reproduction Assessment

Yes, at source level. A sendMediaFeishu rejection still reaches Feishu outbound's catch block, returns fallback text, and the CLI formats the plugin-handled result as a successful send.

## Solution Assessment

Unclear. Fail-fast or explicit partial-success reporting is the narrow fix for misleading success output, but supporting a Feishu local-root config key changes filesystem read/upload scope and needs maintainer/security policy first.
