Keep open: current main and v2026.6.10 still lack a Feishu `mediaLocalRoots` config contract, and Feishu media upload failures still fall back to a success-shaped text send. ## What This Changes

The issue reports that Feishu CLI media sends ignore configured local media roots and can print success after media upload failure.

## Best Possible Solution

Define a security-reviewed outbound local-media root contract, then update Feishu to report media upload failure as either a real failure or explicit partial success with schema/docs/tests aligned.

## Reproduction Assessment

Yes, source-level: current main has a strict Feishu schema without `mediaLocalRoots`, and the Feishu media adapter still catches upload errors into fallback text send. I did not run a live Feishu credential send in this read-only review.

No, not as an automatic Feishu-only patch. The maintainable fix needs a shared, security-reviewed local-root policy plus explicit Feishu media failure semantics rather than silently adding a parallel channel config path.
