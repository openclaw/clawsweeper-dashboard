## Summary

Keep open. Current main still does not define a supported Feishu `mediaLocalRoots` config surface, and Feishu outbound media upload failures are still caught and converted into a text fallback that the CLI formats as a successful direct send.

## What This Changes

Feishu CLI media sends should honor a documented local media-root policy or clearly fail/report when media upload fails instead of presenting a successful send.

## Best Possible Solution

Define the Feishu outbound media contract, then add schema/docs/tests for the chosen local-root policy and adjust `LocalMediaAccessError` handling so denied media is not reported as delivered.

## Reproduction Assessment

Yes, at source level. Current main still catches Feishu media upload errors, sends a fallback text containing the media URL/path, and routes the returned direct delivery through the CLI success formatter; I did not perform a live Feishu send.

## Solution Assessment

Unclear. Fail-fast handling for `LocalMediaAccessError` is the narrow fix for misleading success output, but honoring `channels.feishu.mediaLocalRoots` would add a Feishu local file-read policy surface that needs maintainer approval.
