## Summary

Keep open. Current main still preserves the Feishu media-upload fallback path and does not expose a supported Feishu `mediaLocalRoots` config key, so the reported misleading success behavior remains a valid maintainer decision point.

## What This Changes

The issue asks Feishu CLI media sends to honor configured local media roots or clearly fail/report partial failure when media upload/access fails.

## Best Possible Solution

Define the Feishu outbound media contract, then implement fail-fast or explicit partial-success reporting and only add a documented local-root config surface if that policy is approved.

## Reproduction Assessment

Yes, at source level. A `sendMediaFeishu` upload/access rejection still reaches Feishu outbound's catch block, returns a fallback text send, and the CLI formats returned direct results as successful delivery.

## Solution Assessment

Unclear. Fail-fast handling for `LocalMediaAccessError` is the narrow fix for misleading success output, but honoring `channels.feishu.mediaLocalRoots` would add a security-sensitive local file-read policy surface that needs owner approval.
