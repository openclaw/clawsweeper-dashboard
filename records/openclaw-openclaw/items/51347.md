## Label Justifications

- `P2`: This is a normal-priority Feishu CLI/media bug with real user impact but limited to one channel and media-send paths.
- `impact:security`: The issue concerns local file allowlists and whether Feishu media upload may read host paths.
- `impact:message-loss`: The CLI can report success when the requested media attachment was not delivered as media.

## Summary

Keep open. Current `main` and latest release still leave the core Feishu problem unresolved: the Feishu config contract has no `mediaLocalRoots` field, and Feishu media send errors are still converted into fallback text success rather than a failed media send result.

## What This Changes

The issue asks for Feishu CLI media sends to honor or clearly reject the requested local-root configuration and to stop reporting success when media upload fails.

## Best Possible Solution

Define one Feishu media failure contract: either document and use the generic host media policy, or add a reviewed Feishu root config, then fail fast for local access violations while preserving intentional fallback only for non-access upload failures.

## Reproduction Assessment

Yes, source-reproducible: if `sendMediaFeishu` throws a local media access error, current Feishu outbound catches it and returns a text fallback. The exact `/tmp` default-owner case may be partly mitigated by generic media-root expansion, but restricted policies still exercise the failing path.

## Solution Assessment

Unclear. A narrow code fix exists for fail-fast LocalMediaAccessError handling, but maintainers first need to choose whether Feishu should gain a channel-local root config or rely on the generic host media policy.
