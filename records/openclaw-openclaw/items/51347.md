## Summary

Keep open: current main has generic source-based media-root widening, but it still does not accept a Feishu `mediaLocalRoots` config key and Feishu media upload failures still fall back to a successful text-send result. The discussion correctly split this into config trust and failure-propagation concerns, and both need maintainer security/product direction before a fix should land.

## What This Changes

The issue asks Feishu CLI media sends to honor configured media roots, or otherwise fail/report clearly instead of returning success when media upload fails.

## Best Possible Solution

Choose a shared, security-reviewed local-media root contract and explicit Feishu media failure semantics, then update schema/docs and focused Feishu/outbound tests together.

## Reproduction Assessment

Yes, source-level: the current strict Feishu schema omits `mediaLocalRoots`, and the Feishu media adapter still catches upload errors into a text-send fallback. I did not run a live Feishu credential send in this read-only review.

## Solution Assessment

No, not as an automatic Feishu-only config patch. The maintainable fix needs a shared local-root authorization model plus an explicit Feishu media failure contract rather than silently adding a parallel channel-specific allowlist.
