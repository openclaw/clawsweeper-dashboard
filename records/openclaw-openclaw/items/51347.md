## Summary

Keep open. Current main and the latest shipped tag still have a source-reproducible Feishu media status problem: `sendMediaFeishu` failures are caught and converted into a fallback text send, while the reported Feishu-specific media-root config key is still not a supported schema field.

## What This Changes

The issue asks for Feishu CLI media sends to either honor a configured local media root or fail/report clearly instead of returning a success-shaped result when media upload is denied.

## Best Possible Solution

Define Feishu media failure semantics first, then either fail fast or report explicit partial success, and only introduce a Feishu local-root config key with schema, docs, and security-reviewed tests if maintainers approve that filesystem scope.

## Reproduction Assessment

Yes, at source level. A `sendMediaFeishu` rejection still reaches Feishu outbound's catch block, returns fallback text, and the CLI formats the plugin-handled send as successful; I did not run a live Feishu send in this read-only review.

## Solution Assessment

Unclear. Fail-fast or explicit partial-success reporting is the narrow fix for misleading success output, but supporting a Feishu local-root config key changes filesystem read/upload scope and needs maintainer policy first.
