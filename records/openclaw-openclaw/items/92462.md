## Summary

Current main already supports corpus-wide recall across the active memory plugin and Memory Wiki, but it does not provide an external knowledge-base provider contract or a renderable plugin page/sidebar surface in Control UI. The issue remains necessary as a maintainer product/API design item, although the proposal needs decomposition into narrower work.

## What This Changes

Requests a built-in Control UI knowledge-base/wiki experience or a standardized plugin contract for external knowledge-base providers.

## Best Possible Solution

Keep the existing Memory Wiki corpus supplement as the canonical local recall path, then separately design a narrow external retrieval contract and a safe renderable Control UI plugin surface; leave vendor connectors, document conversion, and service-specific management flows to external plugins.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection does establish that local `corpus=all` recall exists while external knowledge-base federation and renderable plugin pages do not.

## Solution Assessment

No, not as one combined implementation. The maintainable direction is to reuse the existing corpus-supplement path and split the missing generic retrieval and Control UI extension contracts from optional file-management and vendor integrations.
