## Summary

Keep this open for maintainer product/API design. Current main has partial Memory Wiki visibility in the Dreaming UI, but it does not provide the requested full Knowledge Base page, file-management/import workflow, external knowledge-base provider contract, or transparent external-KB integration with memory search.

## What This Changes

The issue requests a built-in Control UI knowledge-base/wiki experience or a standardized plugin contract for external knowledge-base providers.

## Best Possible Solution

Define the product boundary first, likely as a narrow plugin/API design for knowledge-base providers plus selective Control UI integration that reuses existing Memory Wiki and Dreaming UI primitives.
