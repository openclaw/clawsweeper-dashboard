## Summary

Keep open: current main and the latest release still do not collect context-window, max-token, or full modality metadata during the custom OpenAI-compatible onboarding path. The closest implementation is the open PR at https://github.com/openclaw/openclaw/pull/43493, but that PR still needs repair and a maintainer scope decision before this issue can close.

## What This Changes

The issue asks for optional onboarding prompts that collect OpenAI-compatible custom model metadata such as context window, max output tokens, and modalities.

## Best Possible Solution

Land a scoped, repaired custom-provider metadata prompt that writes existing ModelDefinitionConfig fields, preserves current defaults when skipped, and has explicit maintainer scope on configure versus onboarding/non-interactive parity.

## Reproduction Assessment

Yes. Source inspection shows the current custom-provider path collects endpoint, model, alias, and image-input support, then saves default context/max-token metadata without prompting for the requested fields.

## Solution Assessment

Unclear as-is. A custom-provider metadata wizard is a reasonable direction, but maintainers need to settle the configure/onboard/non-interactive scope and land a repaired version of the linked implementation.
