## Summary

Keep open. Current main still does not collect context-window, max-output-token, or general modality metadata during custom/OpenAI-compatible onboarding, and the related implementation candidate at https://github.com/openclaw/openclaw/pull/43493 remains open with review blockers.

## What This Changes

Request to add optional context window, max output token, and modality prompts to the OpenAI-compatible custom-provider onboarding flow.

## Best Possible Solution

Extend the shared custom-provider setup/config path to collect optional model metadata, preserve defaults when left blank, map modality choices to ModelDefinitionConfig.input, and review or replace https://github.com/openclaw/openclaw/pull/43493 as the implementation path.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to follow the custom-api-key onboarding path after verification: current main asks for endpoint ID, alias, and image support, then writes fixed contextWindow and maxTokens defaults without the requested metadata prompts.

## Solution Assessment

Yes as a feature direction, but the implementation should be coordinated with the open PR and scoped deliberately across onboard versus configure and text/image versus audio/video modalities.
