Keep open: current main and the latest release still do not prompt for context window or max-token metadata during custom/OpenAI-compatible provider onboarding, and the related open PRs are candidates rather than merged fixes.

## What This Changes

Request interactive custom/OpenAI-compatible provider onboarding prompts for model metadata such as context window, max output tokens, and input modalities.

## Best Possible Solution

Keep this as the canonical product tracker and review or replace the linked metadata prompt implementation with a focused onboarding/configure flow that preserves existing model metadata and needs no new config schema.

## Reproduction Assessment

Yes for a source-level feature gap: current main prompts for custom endpoint details and image input only, then writes default or preserved metadata without prompting for context window or max output tokens. This is not a failing bug reproduction because the requested behavior is new setup UX.

## Solution Assessment

Unclear: collecting metadata during onboarding is plausible and aligned with setup ergonomics, but the best final shape needs maintainer scope across `openclaw onboard`, `openclaw configure --section model`, and non-interactive flags.
