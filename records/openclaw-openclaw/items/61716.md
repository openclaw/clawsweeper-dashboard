## Summary

Keep open. Current main still lacks prompts for context window, max output tokens, or general modality metadata in the custom/OpenAI-compatible onboarding path, and the related implementation PR remains open with review blockers.

## What This Changes

The issue requests optional model metadata prompts for context window, max output tokens, and modalities during OpenAI-compatible/custom provider onboarding.

## Best Possible Solution

Extend the shared custom-provider setup path to collect optional model metadata, preserve existing defaults when fields are skipped, map modality choices to ModelDefinitionConfig.input, and review or replace https://github.com/openclaw/openclaw/pull/43493 as the implementation path.

## Solution Assessment

Yes as a feature direction, but not through a duplicate repair job while the related PR is open. The implementation should be coordinated with that PR and scoped deliberately across onboard versus configure and text/image versus audio/video modalities.
