## Summary

Keep open: current main and the latest release still lack the durable completion-route fallback, and every issue-specific fix PR found in the linked/search context is closed unmerged. The remaining work is real, but the storage/key/retirement shape needs maintainer design review before another fix PR is queued.

## What This Changes

Ensure an isolated cron job's explicit `delivery.channel` survives through the final completion announcer path back to the controller/requester session.

## Best Possible Solution

Land a fresh, maintainer-approved fix that stores the resolved completion route in the shared SQLite-backed state at task prep, resolves it only as a fallback after session/requester context, retires it after delivery settles, and proves the isolated cron webchat return path end to end.

## Reproduction Assessment

Yes at source level, but not as a live repro in this read-only pass. Current main resolves the explicit cron delivery target, yet final subagent announcement origin resolution has no durable route fallback when the session entry/requester origin lacks delivery context.

## Solution Assessment

Unclear until maintainer design review completes. The durable SQLite `task_completion_routes` direction matches repository state-storage policy better than the older session-entry or in-memory lease attempts, but the key, scope, retirement, and cleanup semantics need explicit acceptance.
