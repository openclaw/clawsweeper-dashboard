## Summary

This remains a plausible, distinct current-main bug. Normal cron final-text delivery uses the resolved outbound target, but A2A-style completion follows the separate subagent announcer, where external delivery depends on the persisted requester origin; existing tests do not prove that an isolated cron job’s explicit webchat channel reaches that handoff.

## What This Changes

Fix isolated cron A2A completion announcements so the cron job’s explicit delivery channel is retained through the subagent completion-announcer handoff.

## Best Possible Solution

Carry the canonical resolved cron delivery context into the registered completion requester origin, then add a focused regression test proving an isolated A2A completion uses explicit webchat delivery without relying on session history.

## Reproduction Assessment

Yes at source level with medium confidence: the report isolates a separate completion-announcer route whose external delivery depends on requester-origin propagation, but a current-main live replay is still missing.

## Solution Assessment

Yes in direction: propagating the already-resolved cron delivery context is narrower and safer than adding another channel-inference fallback; implementation should first confirm the exact registry or spawn handoff that drops it.
