## Summary

Keep open: current main and v2026.6.9 still only support global/group/request-scoped message-tool-only source-reply policy, not the requested per-agent or per-account opt-out for automatic assistant final-text replies.

## What This Changes

The issue requests an opt-in per-agent or per-channel-account delivery policy so plain assistant text stays private and only explicit `message` tool sends reach the channel.

## Best Possible Solution

Define one generic, default-preserving source-reply delivery policy that reuses the existing `message_tool_only` machinery, then align config schema, docs, doctor expectations if needed, and gateway/cron regression coverage.

## Reproduction Assessment

Yes, at source level: current main has global/group/request-scoped source-reply controls but no per-agent or channel-account delivery opt-out. I did not run a live Feishu/DARGO cron reproduction.

## Solution Assessment

Unclear until maintainers choose the product surface. Reusing the existing `message_tool_only` source-reply primitive looks maintainable, but the exact config location should be generic and default-preserving rather than Feishu-only.
