## Summary

Keep open: current main has global/group and request-scoped message-tool-only source delivery, but I did not find a supported per-agent or per-account opt-out for the DARGO/Feishu shape, and the remaining config/API surface needs maintainer choice.

Request a per-agent or per-account setting that keeps ordinary assistant final text private while allowing explicit message tool sends to produce channel output.

## Best Possible Solution

Reuse the existing `sourceReplyDeliveryMode: "message_tool_only"` machinery behind one canonical opt-in surface, keep automatic delivery as the default, and add schema, docs, and focused gateway/channel tests for the chosen scope.

## Reproduction Assessment

## Solution Assessment

Unclear: the requested behavior should reuse the existing message-tool-only delivery machinery, but adding a Feishu `replyToCurrent` account flag may be the wrong owner surface. Maintainers should choose between a generic agent/run delivery contract and channel account configuration before implementation.
