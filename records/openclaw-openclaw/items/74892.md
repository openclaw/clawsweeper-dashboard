## Best Possible Solution

Land the small flag-semantics fix with explicit no-override coverage while preserving current model persistence, and record the user-visible response change in the active changelog.

## Solution Assessment

Yes, with the missing test and changelog added. `Boolean(modelOverride?.trim())` is the narrowest maintainable flag source if `modelApplied` means the caller's explicit override was acknowledged.
