## Summary

Keep this PR open. Current main has not implemented the PR's flag-semantics change, and the proposed patch is not merge-ready because it can still close the linked runtime model-override bug without proving that a spawned child actually runs with the requested model.

## What This Changes

The PR changes native sub-agent model planning and direct spawn response handling so `modelApplied` is true only when the caller passes a non-empty model override.

## Best Possible Solution

Clarify whether `modelApplied` means an explicit caller override or any selected child model, then land a tested fix matching that API contract. If the linked runtime bug is the target, the fix should prove that an explicit `sessions_spawn.model` is persisted and used by the child run, ideally with explicit-model, configured-default, and no-override regression cases.
