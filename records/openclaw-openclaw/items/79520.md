## Summary

Keep this open: current main still has the source-backed sandbox new-file mode gap and still does not convert raw local-media permission errors into an actionable media error. The only linked fix PR was closed unmerged, so there is no implemented or superseding main-branch fix to close against.

## What This Changes

The issue requests fixing outbound media uploads from owner-only files by aligning OpenClaw-managed media write modes, surfacing typed permission errors instead of a generic 500, and documenting the local media permission contract.

## Best Possible Solution

Keep the LaunchAgent umask hardening, make supported OpenClaw-managed outbound media artifacts readable under a scoped media/workspace contract, map local media EACCES/EPERM to an actionable typed error, and defer any operator config knob to a separate product decision.

## Reproduction Assessment

No live high-confidence reproduction was established for the exact Discord 500. The issue is source-reproducible for the central write-side mode drift and raw permission-error escape path on current main.

## Solution Assessment

Unclear whether the issue's broad chmod proposal is the final best boundary. The maintainable direction is a narrowly approved media/workspace permission contract plus typed outbound error mapping, not a blanket loosening of daemon-private file modes or an immediate config knob.
