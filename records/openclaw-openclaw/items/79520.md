## Summary

Keep open: current main still has the source-backed sandbox write-mode and media error-mapping gaps, and the linked fix PR was closed without merging. The remaining work is real but should get maintainer direction because the repair touches file-permission policy.

## What This Changes

The issue requests fixing outbound media uploads for owner-only files by aligning OpenClaw-managed media write modes and surfacing actionable local-media permission errors.

## Best Possible Solution

Preserve the LaunchAgent umask hardening, make supported OpenClaw-managed outbound media artifacts readable under the established media contract, and map local media permission failures to an actionable typed error before revisiting any optional config knob.

Yes, from source inspection: current main still creates first-time sandbox fs-bridge writes at 0600 and still rethrows non-FsSafe local media read errors. I did not live-run the Discord upload path in this read-only review.

## Solution Assessment

Unclear as a complete direction: typed local-media error mapping is narrow, but changing all generic sandbox first-time writes to 0644 needs maintainer agreement on the media-versus-workspace permission boundary.
