## Summary

This should stay open: current main still has the first-time sandbox write mode drift, the local media reader path can still rethrow non-typed permission errors, and the focused fix PR was closed without merging.

## What This Changes

The issue asks OpenClaw to make outbound channel media uploads work or fail actionably when source files are created with restrictive 0600 permissions.

## Best Possible Solution

Preserve the LaunchAgent umask hardening, make supported OpenClaw-managed outbound media artifacts readable under the established media contract, and map local media permission failures to an actionable typed error.

## Reproduction Assessment

Yes, at source level: first-time sandbox helper writes still keep 0600 while upload-file/path remains a documented outbound media surface. I did not live-run the Discord gateway upload in this read-only pass.

## Solution Assessment

Unclear for the full write-side boundary: the typed EACCES/EPERM mapping is narrow, but changing all generic sandbox new-file writes to 0644 needs maintainer confirmation or a media-scoped approach.
