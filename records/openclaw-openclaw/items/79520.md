## Summary

Keep open: current main still has the fresh-file mode gap in the sandbox mutation helper, and the documented/channel action surfaces still support local outbound media paths. Source inspection supports the write-side bug and a likely weak error path, but I did not establish a live macOS/Discord 500 reproduction in this read-only review.

## What This Changes

The issue asks OpenClaw to make outbound media writes/read failures consistent by avoiding fresh `0o600` media files and returning actionable upload errors.

## Best Possible Solution

Patch the fresh-write sandbox helper to publish new outbound-capable files with the managed media mode, add targeted regression coverage, and improve local media read/upload errors without weakening the LaunchAgent umask or safe-FS checks.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not live in this review; source inspection gives a medium-confidence reproduction for the write-side leak because fresh sandbox writes still create `0o600` files and only preserve mode on overwrites.

## Solution Assessment

Is this the best way to solve the issue? Yes for the narrow bug: set the intended new-file mode in the sandbox write helper and return typed outbound media errors; leave the proposed config knob for separate maintainer direction.
