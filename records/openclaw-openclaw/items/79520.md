## Summary

Keep open: current main still has the source-backed write-side 0600 drift and read-side permission-error mapping gap, and the linked fix PR was closed without merging.

## What This Changes

The issue asks OpenClaw to make supported outbound media artifacts readable despite restrictive umask paths and to surface actionable local-media permission errors instead of a bare 500.

## Best Possible Solution

Preserve the LaunchAgent umask hardening, make supported OpenClaw-managed outbound media artifacts meet the 0644 media contract, and map local media EACCES/EPERM failures to an actionable typed error while deferring any operator opt-out knob.

## Reproduction Assessment

Yes for the source-level path: current main creates first-time sandbox fs-bridge files at 0600, upload-file/filePath is supported, and the local media loader can rethrow raw EACCES/EPERM. The live Discord 500 was not rerun in this read-only pass.

## Solution Assessment

Unclear as a final implementation boundary: fchmoding before atomic replace and adding typed EACCES/EPERM mapping are narrow repairs, but maintainers should choose whether the chmod applies only to media/artifacts or to all new sandbox fs-bridge writes.
