## Summary

This should stay open: current main still has the first-time sandbox write mode drift and still lacks typed handling for raw local-media EACCES/EPERM, while the only linked fix PR was closed unmerged.

## What This Changes

The issue asks OpenClaw to make outbound channel media uploads work or fail actionably when sandbox/browser-created files land at owner-only permissions.

## Best Possible Solution

Land a focused fix that preserves existing file modes, gives new sandbox-created outbound artifacts a readable media/workspace mode where intended, and maps local-media permission failures to an actionable typed error while deferring any config knob.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: first-time sandbox helper writes keep 0o600, upload-file/path remains a documented outbound surface, and local-media EACCES/EPERM can still escape typed handling. I did not live-run the macOS Discord upload flow in this read-only pass.

## Solution Assessment

Yes for the narrow core direction: fchmoding the new temp fd before atomic replace and mapping EACCES/EPERM at the media boundary are the maintainable repair. The optional operator config knob should be deferred unless maintainers explicitly want a policy surface.
