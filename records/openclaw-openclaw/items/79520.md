## Summary

Keep open: current main and the latest release still show the reported source-level write-side mode drift, and the related fix PR was closed unmerged rather than shipped.

## What This Changes

Issue reports outbound channel media uploads returning an unhelpful 500 when source files are mode 0600, with a suspected sandbox write-mode leak and missing typed read-side error.

## Best Possible Solution

Preserve the LaunchAgent umask hardening, make supported OpenClaw-managed outbound media artifacts readable under the established media contract, and map local permission failures to an actionable typed media error.

## Reproduction Assessment

Yes from source inspection for the source-level bug: current main’s first-time sandbox fs-bridge writes keep 0600 while Discord upload-file/path remains a supported outbound surface. The live macOS Discord 500 was not run in this read-only review.

## Solution Assessment

Unclear whether the issue’s broad fchmod proposal is the final boundary. The typed error mapping is narrow, but the write-side change should be approved or scoped so non-media sandbox writes are not unintentionally loosened.
