## Summary

Keep this open: current main still has the source-backed write-side mode drift and still lacks a typed local-media permission error path, while the linked fix PR closed unmerged.

## What This Changes

The issue reports outbound channel media uploads failing with a bare 500 when source files are mode 0600, and asks for OpenClaw-managed media writes to use the media-mode contract plus actionable permission errors.

## Best Possible Solution

Preserve daemon-private umask hardening, make supported outbound-media artifacts honor the existing 0644 media contract, and translate local media permission failures into an actionable typed error while deferring any config knob.

## Reproduction Assessment

Yes, source-reproducible: current main opens first-time sandbox bridge temp files at 0600 and only fchmods overwrites while filePath/upload-file remains a documented outbound surface. I did not run the live macOS Discord upload in this read-only pass.

## Solution Assessment

Unclear whether broad 0644 for every generic sandbox new-file write is the best boundary. The maintainable direction is to scope the permission contract to supported outbound media/artifacts and add typed EACCES/EPERM mapping at the media boundary.
