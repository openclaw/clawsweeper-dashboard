## Summary

This should stay open: current main still has the reported write-side mode drift, and the outbound upload path can still lose the actionable lower-level media-read reason behind generic gateway error mapping.

## What This Changes

The issue requests fixing outbound media uploads from restrictive local files by correcting OpenClaw-managed media write modes and surfacing a typed permission error instead of a bare 500.

## Best Possible Solution

Preserve the LaunchAgent umask hardening, make supported OpenClaw-managed outbound media artifacts readable under the established media contract, and map local media permission failures to an actionable typed error.

## Reproduction Assessment

Yes from source inspection for the write-side bug: first-time sandbox helper writes keep 0600 while upload-file/path remains a documented outbound surface. The exact live Discord 500 was not reproduced in this read-only pass.

## Solution Assessment

Unclear whether the issue's broad fchmod proposal is the best final boundary. The safer maintainable direction is a media-scoped permission contract plus typed outbound error mapping, with any config knob deferred.
