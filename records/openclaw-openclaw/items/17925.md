## Summary

Keep open. Current main covers the Gemini side, but ZAI/GLM web_search is still absent; the remaining decision is whether this provider-specific search integration should be bundled officially or shipped through ClawHub/community plugin distribution.

## What This Changes

Request native/provider-owned web_search support for ZAI/GLM and Google/Gemini without requiring a separate third-party search API.

## Best Possible Solution

Decide official bundled promotion versus ClawHub/npm plugin; if approved, implement it in extensions/zai via webSearchProviders with endpoint-aware runtime, docs, and focused contract tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: Gemini declares/registers web_search and sends google_search, while ZAI declares only mediaUnderstandingProviders and registers no web-search provider.

## Solution Assessment

Unclear. A ZAI-owned webSearchProviders implementation is the narrow technical path, but official bundling versus ClawHub needs maintainer scope approval because plugin APIs already support external web-search providers.
