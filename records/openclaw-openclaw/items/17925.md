## Summary

Keep open. Current main covers the Gemini side, including the later freshness follow-up, but ZAI/GLM still has no bundled `webSearchProviders` contract entry, runtime registration, provider runtime, or docs entry; the remaining decision is whether OpenClaw should ship official bundled ZAI search support or route it to ClawHub/npm plugin distribution.

## What This Changes

The issue requests provider-native `web_search` support for ZAI/GLM and Gemini so users can use built-in provider search without a separate third-party search key.

## Best Possible Solution

Maintainers should decide the distribution boundary first; if bundled support is approved, add a ZAI-owned `WebSearchProviderPlugin` with manifest metadata, guarded lazy runtime execution, credential/config handling, docs, and focused registration/runtime tests while leaving Gemini intact.

## Reproduction Assessment

Yes. A high-confidence source-level check on current main shows Google/Gemini is declared and registered as a web-search provider, while ZAI declares only media understanding and has no web-search registration or runtime path.

## Solution Assessment

Unclear. The existing `webSearchProviders` plugin contract is the narrow technical seam, but choosing official bundled support versus external ClawHub/npm plugin distribution is a maintainer product decision.
