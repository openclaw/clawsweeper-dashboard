## Summary

Keep this open: current main and the latest release implement the Gemini side, but ZAI/GLM still has no bundled web_search provider contract, registration path, runtime provider, or docs entry. The remaining blocker is a maintainer product decision about official bundled ZAI support and fallback policy versus an external ClawHub/plugin path.

## What This Changes

The issue requests provider-native web_search support for ZAI/GLM and Google Gemini so users can use built-in model-provider search without a separate third-party search key.

## Best Possible Solution

Choose the ZAI ownership/default boundary; if bundled support is desired, implement it as a ZAI-owned webSearchProviders provider with manifest metadata, lazy runtime execution, credential/config handling, docs, and registration/runtime tests rather than ad hoc core passthrough.

## Reproduction Assessment

Yes. The remaining gap is source-reproducible: current main registers and documents Gemini web_search, while ZAI has no webSearchProviders manifest contract, no registerWebSearchProvider call, no docs entry, and no shared contract-test expectation.

## Solution Assessment

Unclear until maintainers choose the product boundary. The maintainable implementation path is a provider-owned webSearchProviders integration or external ClawHub distribution, not request-time core passthrough.
