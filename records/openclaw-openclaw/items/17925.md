## Summary

Keep open. Current main and v2026.6.5 cover the Gemini side through the bundled Google/Gemini web-search provider, but ZAI/GLM still has no web-search manifest contract, runtime registration, provider implementation, tests, or docs; the remaining question is maintainer product ownership, not a stale or already-implemented issue.

## What This Changes

The issue requests native web-search support for ZAI/GLM and Google/Gemini providers without requiring a separate third-party search API key.

## Best Possible Solution

Keep Gemini as implemented and make an explicit maintainer decision for ZAI: either add a ZAI-owned webSearchProviders implementation with manifest metadata, runtime, credential/config handling, docs, tests, and live API proof, or direct the capability to an external ClawHub/npm plugin.

## Reproduction Assessment

Yes, at the source level: current main has a complete Gemini web-search provider but no ZAI web-search manifest contract, registration, runtime, docs, or tests. I did not run a live ZAI API call in this read-only review.

## Solution Assessment

No for the issue's unconditional passthrough shape: the maintainable path is the existing provider-owned webSearchProviders contract, not ad hoc model-request injection. The unresolved part is the ownership/product decision for bundled ZAI versus an external ClawHub plugin and any fallback policy.
