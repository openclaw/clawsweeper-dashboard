## Summary

Keep this open: current main implements the Gemini half through a bundled web_search provider, but the ZAI/GLM web_search provider is still missing from the manifest, registration path, docs, and shared contract expectations. Existing maintainer and ClawSweeper comments also route the remaining work through a product/API ownership decision rather than immediate cleanup closure.

## What This Changes

Request to add provider-native web_search support for ZAI/GLM and Gemini so users can use built-in provider search without configuring a separate third-party search API key.

## Best Possible Solution

Choose the ZAI ownership boundary, then add a ZAI-owned WebSearchProvider through the existing plugin contract with manifest metadata, runtime execution, credential handling, docs, and registration tests if maintainers want it bundled.

## Reproduction Assessment

Yes for the remaining gap: source inspection on current main shows Gemini is registered and documented as a web-search provider while ZAI has no web-search manifest contract, registration call, docs entry, or contract-test expectation.

## Solution Assessment

Unclear until maintainers choose the product boundary; the maintainable path is provider-owned webSearchProviders integration, not ad hoc core request passthrough.
