## Summary

Keep open. Gemini/Google web search is implemented on current main, but the ZAI/GLM side is still absent and now needs maintainer product judgment because the same ZAI bundled-provider direction was also closed as ClawHub/plugin-scoped in a related PR.

## What This Changes

Request provider-native or bundled `web_search` support for ZAI/GLM and Google/Gemini using existing provider credentials.

## Best Possible Solution

Decide whether ZAI search should be promoted as an official bundled provider; if approved, implement it inside the ZAI plugin through the existing `webSearchProviders` contract while leaving Gemini as already shipped.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: Gemini has manifest, registration, runtime, docs, and tests for web search, while ZAI lacks the matching web-search contract, registration, runtime, and docs.

## Solution Assessment

Unclear. The narrow technical solution is a ZAI-owned `webSearchProviders` implementation, but the best product path needs maintainer approval because current plugin surfaces can host it externally and a related bundled ZAI PR was closed as ClawHub-scoped.
