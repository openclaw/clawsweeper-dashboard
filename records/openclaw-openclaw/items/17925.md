## Summary

Keep open. Current main implements the Gemini side through the bundled Google/Gemini web_search provider, but the ZAI/GLM side is still absent from the ZAI manifest, runtime registration path, registration contract expectations, tests, and docs.

## What This Changes

The issue requests provider-native web_search support for ZAI/GLM and Google/Gemini without requiring a separate third-party search API key.

## Best Possible Solution

Add a bundled ZAI-owned web_search provider through the existing webSearchProviders plugin contract, with manifest metadata, lazy runtime execution, ZAI credential reuse, docs, changelog, and registration/runtime tests; keep the shipped Gemini provider intact.

## Reproduction Assessment

Yes. The high-confidence reproduction is structural on current main: ZAI has no webSearchProviders manifest entry, no registerWebSearchProvider call, no ZAI search runtime, and no docs entry, while Gemini has those pieces.

## Solution Assessment

Yes. The narrow maintainable solution is a ZAI provider-plugin implementation behind the existing webSearchProviders contract, not a core special case in generic model transport.
