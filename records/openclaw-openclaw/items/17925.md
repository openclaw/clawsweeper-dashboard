## Summary

Keep open. Current main implements the Gemini half through the bundled Google/Gemini web_search provider, but the ZAI/GLM half is still missing from the ZAI plugin manifest, runtime registration, shared registration expectations, tests, and docs; prior maintainer comments also intentionally kept the issue open for that remaining scope.

## What This Changes

Request to support provider-native web_search for ZAI/GLM and Gemini so users can use built-in provider search without a separate third-party search API key.

## Best Possible Solution

Add a bundled ZAI-owned web_search provider through the existing webSearchProviders plugin contract, reusing ZAI auth/base-url semantics and existing guarded web-search runtime patterns while leaving the shipped Gemini provider intact.

## Reproduction Assessment

Yes. A high-confidence structural reproduction exists on current main: ZAI has no webSearchProviders manifest entry, no registerWebSearchProvider call, no ZAI search runtime, no contract expectation, and no docs entry, while Gemini has those pieces.

## Solution Assessment

Yes. The narrow maintainable solution is a ZAI provider-plugin implementation behind the existing webSearchProviders contract, not a core special case in generic model transport.
