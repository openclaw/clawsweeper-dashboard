## Summary

Gemini search is already implemented in the bundled Google plugin; the only remaining request is optional ZAI web search, and current main exposes the manifest, SDK, registration, credential, and guarded-content seams needed to ship it externally. A focused bundled ZAI attempt was subsequently closed with explicit direction to ClawHub/community-plugin distribution, so this core umbrella issue no longer represents planned OpenClaw work.

## What This Changes

Add web search for ZAI/GLM and Google/Gemini using provider-native capabilities and existing model-provider credentials, optionally as automatic fallback.

## Best Possible Solution

Keep Gemini’s bundled provider as-is and publish ZAI web search as a focused ClawHub/npm plugin using the existing provider-web-search SDK, with explicit regional endpoint selection, enforced timeout handling, canonical ZAI credential reuse, and untrusted-content wrapping.

## Reproduction Assessment

Yes at source level: Gemini is present and documented, while current main has no ZAI web-search registration or provider implementation. This is a missing optional capability, not broken established behavior.

## Solution Assessment

No, direct generic core passthrough or an implicit fallback is not the best path. The existing provider-owned web-search plugin contract supports a narrower external ZAI implementation without adding one-off core policy.
