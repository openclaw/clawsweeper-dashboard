## Label Justifications

- `P2`: The missing ZAI capability blocks provider-native search for affected users but is a bounded enhancement rather than a core outage.
- `impact:auth-provider`: The issue concerns provider-specific tool routing and whether existing ZAI model credentials can drive web search.

## Summary

Current main and release v2026.6.5 implement Gemini web search but still lack any ZAI web-search contract, registration, runtime, tests, or docs. The issue remains necessary, with the remaining decision being whether official ZAI search belongs in the bundled provider or an external ClawHub plugin.

## What This Changes

Requests provider-native web search for ZAI/GLM and Google Gemini using existing provider credentials instead of requiring a separate search service.

## Best Possible Solution

Keep Gemini as implemented and add ZAI through the existing provider-owned webSearchProviders contract only if maintainers approve official bundling; otherwise publish a ZAI search plugin through ClawHub without adding core fallback policy.

## Reproduction Assessment

Yes at the source level: current main has a complete Gemini web-search provider but no corresponding ZAI manifest contract, registration, runtime, test, or docs path. This review did not perform a live ZAI API call.

## Solution Assessment

No, unconditional model-request passthrough is not the best repository shape. The existing plugin web-search provider contract preserves provider ownership, external-content safeguards, credential resolution, caching, and consistent tool behavior; maintainers still need to choose bundled versus ClawHub ownership and fallback policy.

## Risks / Open Questions
