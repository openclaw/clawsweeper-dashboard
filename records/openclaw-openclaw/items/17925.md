## Summary

Keep open. Current main implements the Gemini side, but ZAI/GLM still has no bundled web_search provider or native passthrough, and prior maintainer-side reviews leave the remaining official-bundled-versus-ClawHub scope decision unresolved.

## What This Changes

Request to add native provider-backed web_search for ZAI/GLM and Gemini, preferably using existing provider credentials or fallback behavior when no separate search provider is configured.

## Best Possible Solution

Maintainers should decide the scope first; the end state should be either a maintainer-approved ZAI webSearchProviders implementation in extensions/zai or an explicitly external ClawHub/npm plugin, with Gemini left intact.

## Reproduction Assessment

Yes. A high-confidence source-level check on current main shows Gemini is declared and registered as a web_search provider while ZAI declares only mediaUnderstandingProviders and registers no ZAI web_search provider.

## Solution Assessment

Unclear. The narrow technical solution is a ZAI-owned webSearchProviders implementation, but the repository path needs maintainer approval because current plugin surfaces can host it externally and prior ZAI search attempts were not merged into core.
