## Summary

Keep this open: current main has the Gemini/Google web_search provider implemented, but the ZAI/GLM half of the requested native passthrough is still absent from the ZAI plugin manifest, registration code, docs, and plugin contract inventory.

## What This Changes

The issue requests native web_search passthrough support for both ZAI/GLM and Google/Gemini provider surfaces.

## Best Possible Solution

Choose the intended ZAI ownership model, then add a single provider-contract implementation with docs and tests if it belongs in the bundled ZAI plugin.

## Reproduction Assessment

Yes for the remaining gap: source inspection on current main shows Gemini is registered as a web-search provider while ZAI has no web-search manifest contract, registration call, docs, or contract-test expectation.

## Solution Assessment

Unclear until maintainers choose the product boundary; implementing ZAI search in the bundled plugin may be right, but an external ClawHub/npm plugin could also satisfy the use case without expanding core-bundled provider scope.
