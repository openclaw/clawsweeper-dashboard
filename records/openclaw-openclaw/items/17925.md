## Summary

Keep open. Current main covers the Gemini half through the bundled `gemini` web_search provider, but ZAI/GLM still has no `webSearchProviders` contract entry, runtime registration, provider runtime, or docs entry; prior maintainer-side comments also leave the official-bundled versus external-plugin scope decision unresolved.

## What This Changes

Support native or bundled web_search integration for ZAI/GLM, while the Gemini half of the original request is already covered.

## Best Possible Solution

Maintainers should decide the distribution boundary first; if bundled support is approved, implement a ZAI-owned `WebSearchProviderPlugin` with manifest metadata, guarded lazy runtime execution, credential/config handling, docs, and registration/runtime tests while leaving Gemini intact.

## Reproduction Assessment

Yes. A high-confidence source-level check on current main shows Google/Gemini is declared and registered as a web_search provider, while ZAI declares only media understanding and has no web_search registration or runtime path.

## Solution Assessment

Unclear. The narrow maintainable technical path is the existing `webSearchProviders` plugin contract, but choosing bundled official support versus external ClawHub/npm distribution is a maintainer product boundary decision.
