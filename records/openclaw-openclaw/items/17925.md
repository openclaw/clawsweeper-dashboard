## Summary

Keep open. Current main covers the Gemini side, but ZAI/GLM web_search remains absent, and the remaining question is whether OpenClaw should bundle it officially or route it to ClawHub/community plugin distribution.

## What This Changes

Request native/provider-owned web_search support for ZAI/GLM and Google/Gemini so users can search without a separate third-party search key.

## Best Possible Solution

Maintainers should decide whether ZAI search deserves official bundled-provider promotion; if approved, implement it in extensions/zai through webSearchProviders with docs and focused tests, otherwise route contributors to a ClawHub/npm plugin.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main declares/registers Gemini web_search and sends google_search, while ZAI declares only mediaUnderstandingProviders and registers no web_search provider.

## Solution Assessment

Unclear. The narrow technical solution is a ZAI-owned webSearchProviders implementation, but the best product path needs maintainer approval because the existing plugin surface can host it externally and a prior bundled ZAI search PR was closed as ClawHub/community work.
