## Summary

Keep open. Current main implements the Gemini side through the bundled Google/Gemini web_search provider, but the remaining ZAI/GLM side is still absent from the ZAI manifest, runtime registration path, registration contract expectations, tests, and docs.

## What This Changes

Request provider-native web_search support for ZAI/GLM and Gemini so existing model API credentials can power search without a separate search provider key.

## Best Possible Solution

Add a bundled ZAI-owned web_search provider through the existing webSearchProviders plugin contract, with manifest metadata, lazy runtime execution, ZAI credential/config reuse, docs, changelog, and registration/runtime tests; keep the shipped Gemini provider intact.

## Reproduction Assessment

Yes. The high-confidence reproduction is structural on current main: ZAI has no webSearchProviders manifest entry, no registerWebSearchProvider call, and no ZAI search runtime or docs entry, while Gemini has those pieces.

## Solution Assessment

Yes. The narrow maintainable path is a provider-plugin implementation behind the existing webSearchProviders contract, not a core special case in the generic model transport.
