## Summary

Keep this open: current main has the Google/Gemini web_search provider implemented, but the ZAI/GLM half of the requested native search support is still absent from the bundled ZAI plugin surface. Prior maintainer and ClawSweeper comments correctly identify the remaining ZAI provider/product-boundary question, and current main still matches that assessment.

## What This Changes

The issue requests native web_search support for ZAI/GLM and Google/Gemini providers without requiring a separate third-party search API key.

## Best Possible Solution

Choose the ZAI ownership model, then implement a ZAI-owned webSearchProviders contract with manifest metadata, runtime execution, credential/config handling, docs, and provider registration tests if maintainers want it bundled.

## Reproduction Assessment

Yes for the remaining gap: source inspection on current main shows Gemini is registered and documented as a web-search provider while ZAI has no web-search manifest contract, registration call, docs entry, or contract-test expectation.

## Solution Assessment

Unclear until maintainers choose the product boundary; the maintainable path is provider-owned webSearchProviders integration, not ad hoc core request passthrough.
