## Summary

Keep open. Current main implements the Gemini half, but ZAI/GLM still lacks a bundled web_search provider contract entry, runtime registration, provider runtime, and docs entry; existing maintainer-side discussion also leaves the bundled-official versus external-plugin boundary unresolved.

## What This Changes

Request native provider-backed web_search support for ZAI/GLM and Google/Gemini so users can reuse model-provider search without a separate third-party search API key.

## Best Possible Solution

Maintainers should decide the distribution boundary first; if bundled support is approved, implement a ZAI-owned WebSearchProviderPlugin with manifest metadata, guarded lazy runtime execution, credential/config handling, docs, and registration/runtime tests while leaving Gemini intact.

## Reproduction Assessment

Yes. Source inspection on current main shows Google/Gemini is declared and registered as a web_search provider, while ZAI declares only media understanding and has no web_search registration or runtime path.

## Solution Assessment

Unclear. The existing webSearchProviders plugin contract is the narrow technical path, but choosing official bundled support versus external ClawHub/npm distribution is a maintainer product boundary decision.
