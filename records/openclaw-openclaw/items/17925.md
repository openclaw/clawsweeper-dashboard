## Summary

Keep open. Current main implements the Gemini/Google half, but the ZAI/GLM web_search provider remains absent and now needs maintainer scope judgment because multiple related bundled ZAI search PRs were closed as ClawHub/plugin-scoped rather than merged into core.

## What This Changes

The issue requests provider-native web_search support for ZAI/GLM and Google/Gemini without requiring separate third-party search API keys.

## Best Possible Solution

Maintainers should decide whether ZAI search should be promoted as an official bundled provider; if approved, implement it in the ZAI plugin through webSearchProviders with docs and tests, otherwise route the request to a ClawHub plugin.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: on current main, Gemini declares/registers a web_search provider and sends google_search, while ZAI only declares mediaUnderstandingProviders and registers no web_search provider.

## Solution Assessment

Unclear. The narrow technical solution is a ZAI-owned webSearchProviders implementation, but the best product path needs maintainer approval because current plugin surfaces can host it externally and related bundled ZAI search PRs were closed as ClawHub-scoped.
