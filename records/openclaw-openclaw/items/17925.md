## Summary

Keep open. Current main implements the Gemini side through the bundled Google/Gemini web_search provider, but the ZAI/GLM side is still missing from manifest discovery, runtime registration, contract expectations, tests, and docs; the issue discussion also intentionally kept the remaining ZAI scope open.

## What This Changes

Request native/provider-owned web search support for ZAI/GLM and Google/Gemini so users can search through configured model-provider credentials instead of separate third-party search keys.

## Best Possible Solution

Add a bundled ZAI-owned web_search provider through the existing webSearchProviders plugin contract, reusing ZAI auth/base-url semantics and guarded web-search runtime patterns while leaving the shipped Gemini provider intact.

## Reproduction Assessment

Yes. A high-confidence structural reproduction exists on current main: Gemini has manifest, registration, runtime, tests, and docs support, while ZAI lacks the corresponding webSearchProviders manifest entry, registerWebSearchProvider call, contract expectation, and docs entry.

## Solution Assessment

Yes. The narrow maintainable solution is a ZAI provider-plugin implementation behind the existing webSearchProviders contract, not a core model-transport special case.
