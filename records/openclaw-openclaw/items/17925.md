## Summary

Keep this open: Gemini web_search is implemented on current main and in the latest release, but ZAI/GLM still has no bundled webSearchProviders contract, registration path, runtime provider, or docs entry. The remaining decision is whether official bundled ZAI search and fallback behavior belongs in OpenClaw or should stay on the external plugin/ClawHub path.

## What This Changes

The issue requests native web_search passthrough support for ZAI/GLM and Google/Gemini providers without requiring a separate search API key.

## Best Possible Solution

Choose the ZAI ownership/default boundary; if bundled support is desired, implement it as a ZAI-owned webSearchProviders provider with manifest metadata, lazy runtime execution, credential/config handling, docs, and registration/runtime tests rather than request-time core passthrough.

## Reproduction Assessment

Yes. The remaining gap is source-reproducible: current main and v2026.5.22 register and document Gemini web_search, while ZAI has no webSearchProviders manifest contract, no registerWebSearchProvider call, no docs entry, and no shared contract-test expectation.

## Solution Assessment

Unclear until maintainers choose the product boundary. The maintainable implementation path is provider-owned webSearchProviders support or external ClawHub distribution, not ad hoc request-time core passthrough.
