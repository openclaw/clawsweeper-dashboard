## Summary

Keep open. Current main implements the Gemini/Google half, but the ZAI/GLM web_search provider remains absent, and the remaining bundled-vs-ClawHub scope needs maintainer judgment before an implementation PR should be queued.

## What This Changes

Request native provider web-search passthrough for ZAI/GLM and Google/Gemini so users can use provider-hosted search without a separate third-party search key.

## Best Possible Solution

Maintainers should decide whether ZAI search deserves official bundled-provider promotion; if approved, implement it in the ZAI plugin through webSearchProviders with docs and focused tests, otherwise route it to a ClawHub plugin.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main declares/registers Gemini web_search and sends google_search, while ZAI declares only mediaUnderstandingProviders and registers no web_search provider.

## Solution Assessment

Unclear. The narrow technical solution is a ZAI-owned webSearchProviders implementation, but the best product path needs maintainer approval because the existing plugin surface can host it externally and related ZAI search PRs were closed without merging into core.
