## Summary

Keep open. Current main implements the Gemini/Google side through the bundled Gemini web_search provider, but the requested ZAI/GLM web search support is still absent from manifest discovery, plugin registration, contract expectations, docs, and ZAI runtime files.

## What This Changes

The issue requests native or provider-owned web_search support for ZAI/GLM and Google/Gemini so users can search without a separate third-party search API key.

## Best Possible Solution

Add a ZAI-owned web_search provider through the existing webSearchProviders plugin contract while keeping the shipped Gemini provider intact.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: Gemini has manifest, registration, runtime, tests, and docs support, while ZAI lacks the corresponding webSearchProviders manifest entry, registerWebSearchProvider call, contract expectation, and docs entry.

## Solution Assessment

Yes. The narrow maintainable solution is a bundled ZAI provider-plugin implementation behind the existing webSearchProviders contract, not a core model-transport special case.
