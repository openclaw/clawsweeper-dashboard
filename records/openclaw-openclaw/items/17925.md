## Summary

Keep open. Current main implements the Gemini side, but ZAI/GLM still has no bundled web_search provider or native passthrough, and the remaining question is product scope: official bundled ZAI plugin support versus ClawHub/community plugin distribution.

## What This Changes

Request native or fallback web_search support for ZAI/GLM and Gemini so provider-owned search can work without a separate third-party search API key.

## Best Possible Solution

Maintainers should decide the scope first; if official support is approved, implement ZAI in extensions/zai through webSearchProviders with manifest metadata, runtime execution, docs, and focused provider-registration tests, otherwise route it to ClawHub/npm plugin distribution.

## Reproduction Assessment

Yes. A high-confidence source-level check shows Gemini is declared and registered as a web_search provider while ZAI declares only mediaUnderstandingProviders and registers no ZAI web_search provider on current main.

## Solution Assessment

Unclear. The narrow technical solution is a ZAI-owned webSearchProviders implementation, but the best repository path needs maintainer approval because current plugin surfaces can host it externally and prior ZAI search PRs were closed as community/plugin-scope work.
