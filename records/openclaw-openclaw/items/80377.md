Current main preserves the reported session fields, but the production reset path and tests now describe that as intentional reset policy, and the same reporter has an open linked cleanup PR; this should stay open for maintainer product/cleanup coordination rather than be auto-closed or auto-fixed.

## What This Changes

The issue asks for `/reset` to clear session-level thinking, verbose, reasoning, TTS, model, provider, and auth-profile overrides so configured agent defaults are used again.

## Best Possible Solution

Keep the tested reset-preservation policy unless maintainers choose a user-facing semantics change, and treat the linked PR as a cleanup-only fast-path refactor rather than a fix for this issue.

## Solution Assessment
