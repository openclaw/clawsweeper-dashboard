## Summary

Keep open for maintainer judgment: current main intentionally preserves behavior and user-selected overrides across reset, while only auto-sourced model/auth selections are cleared, so the requested semantic change is not a narrow bug-fix cleanup candidate.

## What This Changes

The issue asks for `/reset` to clear session-level thinking, verbose, reasoning, TTS, model, provider, and auth overrides so configured agent defaults apply again.

## Best Possible Solution

Define the reset and override contract explicitly; if maintainers want reset to clear more than auto-sourced selections, update the full session path, fast bootstrap helper, docs, and regression tests together.

## Reproduction Assessment

Yes for the observed preservation behavior: seed a session with `thinkingLevel` or other overrides and run `initSessionState` with `/reset`; current source and tests show the values are carried forward. I did not run a live gateway session because the source and regression tests directly cover the path.

## Solution Assessment

No as an automatic bug fix: clearing all listed fields would contradict current comments, tests, docs, and the `/think default` clear mechanism. The no-op ternary cleanup is narrower and is already represented by the linked cleanup-only PR.
