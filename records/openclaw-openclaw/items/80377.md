## Summary

Keep this open for maintainer product judgment: current main intentionally preserves user behavior overrides across `/reset` and clears only auto-sourced model/auth overrides, while the same-author companion PR at https://github.com/openclaw/openclaw/pull/80414 is still open and now scoped as cleanup-only rather than a fix.

## What This Changes

Requests that `/reset` clear session-level thinking, verbose, reasoning, TTS, model, provider, and auth overrides so configured agent defaults are consulted again.

## Best Possible Solution

Define and document the intended reset contract; if maintainers want a true clean-slate mode, add it deliberately with focused tests rather than treating the fast-path ternaries as the production bug.

## Reproduction Assessment

Yes, from source and tests: current main preserves explicitly set behavior overrides across `/reset` and clears only auto-sourced model/auth overrides. I did not run the gateway live, but the tested code path is direct.

## Solution Assessment

Unclear as a product direction: clearing all user overrides on `/reset` would contradict current tests and source comments. A safer path is to first decide and document whether `/reset` should preserve user preferences or add a separate clean-slate behavior.
