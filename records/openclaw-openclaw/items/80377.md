## Summary

Keep this open for maintainer handling, not an automated cleanup close. Current `main` and the latest shipped release treat `/reset` preserving user-set thinking/verbose/reasoning/TTS overrides as intentional behavior, but the same author still has an open paired cleanup PR that references this issue, so this sweep should not close only the issue side of the pair.

## What This Changes

The issue asks for `/reset` to clear session-level thinking, verbose, reasoning, TTS, model, provider, and auth-profile overrides so configured agent defaults are consulted again.

## Best Possible Solution

Keep the shipped reset semantics: preserve explicit user behavior selections, clear only auto-sourced model/auth selections, and resolve the paired cleanup PR before closing this issue by design if maintainers agree.

## Reproduction Assessment

Yes for the observable behavior: current source and tests show `/reset` preserves user-set thinking/verbose/reasoning/TTS overrides. No for the defect claim: that preservation is currently a tested, shipped reset policy rather than a proven regression.

## Solution Assessment

No, clearing all session-level overrides during `/reset` is not the best automated fix because it would reverse the current shipped behavior and requires maintainer product judgment. The safer path is to keep explicit clear commands such as `/think default` and the existing user-vs-auto reset distinction.
