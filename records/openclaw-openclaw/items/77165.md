## Summary

Keep open. Current main still only derives session titles from existing metadata or the first user message, and the issue is already paired with two open implementation PRs that reference it with closing syntax.

## What This Changes

Request AI-generated concise session titles after a few turns, replacing first-message truncation, with config controls and manual override preservation.

## Best Possible Solution

Converge on one reviewed implementation that stores generated titles as gateway-owned session metadata with explicit precedence, async generation policy, config/schema/docs coverage, and Control UI rendering tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks AI title generation and only has first-message-derived title plumbing.

## Solution Assessment

Unclear until maintainers choose the product contract. The requested UX is plausible, but the maintainable path should define gateway metadata, persistence, precedence, config semantics, and async generation policy before merging an implementation.
