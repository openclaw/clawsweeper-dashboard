## Summary

Keep open. Current main still lacks AI-generated session titles and the requested config surface, and two open implementation PRs reference this issue with closing syntax, so the issue should remain open pending maintainer review.

## What This Changes

Request AI-generated session titles after a configurable number of turns, replacing first-message truncation while preserving manual names.

## Best Possible Solution

Consolidate around one reviewed implementation that stores generated titles as gateway-owned session metadata with clear precedence, async generation policy, config/schema/docs coverage, and Control UI tests.
