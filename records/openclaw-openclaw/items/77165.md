## Summary

Keep open. Current main has manual/session-metadata labels and optional first-user-message title derivation, but no AI-generated, cached, configurable session-title flow; this needs product/API design rather than cleanup closure.

## What This Changes

Request AI-generated, configurable, cached session titles after a few conversation turns instead of first-message truncation.

## Best Possible Solution

Design this as a gateway-owned session metadata feature with persisted generated titles, clear precedence over existing label/displayName/subject fields, async generation after a chosen turn threshold, config/schema/docs coverage, and Control UI rendering tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks AI title generation and only has first-message-derived title plumbing.

## Solution Assessment

Unclear until maintainers decide the product contract. The requested direction is plausible, but the maintainable path should be a gateway/session metadata design rather than a UI-only summarizer.
