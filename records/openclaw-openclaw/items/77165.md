## Summary

Keep open. Current main still derives session titles from displayName, subject, the first user message, or the session id, and the requested AI-generated title flow would add new model calls, persisted metadata semantics, and configuration.

## What This Changes

Request automatic AI-generated session titles after a few turns, with caching, manual override behavior, and configurable enablement/threshold/length settings.

## Best Possible Solution

Design this as a gateway-owned session metadata feature with persisted generated titles, clear precedence rules, async generation after a maintainer-chosen threshold, config/schema/docs coverage, and Control UI rendering tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks AI title generation and only has first-message-derived title plumbing.

## Solution Assessment

Unclear until maintainers decide the product contract. The requested direction is plausible, but the maintainable path should be gateway/session metadata design rather than a UI-only summarizer.
