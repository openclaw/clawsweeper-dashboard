## Summary

Keep open. Current main has deterministic session-title derivation from displayName, subject, or a truncated first user message, and the Control UI session list does not request or render generated titles; this is a valid enhancement that needs product/config policy rather than conservative cleanup.

## What This Changes

The issue requests AI-generated, cached session titles after a configurable number of turns, replacing first-message truncation in the Control UI while preserving manual overrides.

## Best Possible Solution

Keep this open for maintainer product design; if accepted, add a Gateway-owned generated-title field/workflow that runs asynchronously after the chosen turn threshold, persists the result, preserves manual label/displayName overrides, and updates Control UI display/search plus config schema/docs.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior report. Source inspection confirms current main only has deterministic displayName/subject/first-user-message title derivation when requested.

## Solution Assessment

Unclear. AI summarization is a plausible direction and has an adjacent Discord thread-title precedent, but the exact default, config shape, persistence semantics, and model-selection policy need maintainer review before this is the best implementation.
