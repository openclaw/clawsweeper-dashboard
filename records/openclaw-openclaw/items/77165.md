## Summary

Keep open. Current main still only derives optional titles from displayName, subject, or the first user message, and two open unmerged PRs (#77216 and #77225) already reference this issue with closing syntax, so the issue should stay open while maintainers review the implementation path.

## What This Changes

Request automatic AI-generated, cached, configurable session titles after a few conversation turns instead of first-message truncation.

## Best Possible Solution

Design gateway-owned session title metadata with clear manual-title precedence, async generation policy, config/schema/docs coverage, and Control UI rendering tests, then land one canonical implementation PR.
