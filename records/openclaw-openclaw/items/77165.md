Keep open. Current main still derives session titles from existing metadata, first-message truncation, or session id fallback, and open PRs #77216 and #77225 already target this feature with closing syntax, so this should wait for maintainer product/API review rather than cleanup closure.

The issue requests AI-generated session titles after a few conversation turns, with configurable enablement, turn threshold, and title length.
