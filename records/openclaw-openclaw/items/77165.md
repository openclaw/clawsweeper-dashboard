## Summary

Keep open. Current main still only derives session titles from existing session metadata or the first user message, and open PRs #77216 and #77225 are active implementation candidates that reference this work with closing syntax.

## What This Changes

## Best Possible Solution

Consolidate around one reviewed implementation that makes generated titles durable gateway session metadata with clear manual-override precedence, async generation policy, config/schema/docs coverage, and Control UI rendering tests.
