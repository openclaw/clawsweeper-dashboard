## Summary

Keep open. Current main still derives session titles from existing session metadata or the first user message, and an open implementation PR already references this issue with closing syntax, so cleanup should not close it before that PR is reviewed and either landed or closed.

## What This Changes

Request to auto-generate concise session titles after a few message exchanges using AI summarization instead of first-message truncation.

## Best Possible Solution

Design this as a gateway-owned persisted session metadata feature with clear title precedence, async generation after a chosen threshold, opt-in/config defaults, docs/schema coverage, and Control UI rendering tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks AI title generation and only has metadata plus first-message-derived title plumbing.

## Solution Assessment

Unclear until maintainers decide the product contract. The requested direction is plausible, but the maintainable path should be gateway/session metadata design reviewed through the linked PR rather than a UI-only summarizer.
