## Summary

Keep open. Current main still exposes only manual and heuristic session-title plumbing, while #77216 and #77225 are open implementation PRs with closing syntax; this needs maintainer product/API review rather than cleanup closure.

## What This Changes

Request AI-generated session titles after a few conversation turns, with configurable enablement, threshold, and max title length, replacing first-message truncation in the Control UI.

## Best Possible Solution

Consolidate around one reviewed implementation that stores generated titles as gateway session metadata, preserves manual naming precedence, runs asynchronously under clear config policy, and updates schema/docs/UI/tests together.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main still lacks AI-generated session titles and only has existing first-message-derived title plumbing.

## Solution Assessment

Unclear until maintainers choose the product contract. The requested direction is plausible, but the maintainable path should define gateway metadata, persistence, precedence, config semantics, and model-call policy before merging an implementation PR.
