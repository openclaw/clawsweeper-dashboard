## Summary

Keep this PR open, but it is not ready to merge: it changes a security-sensitive exec default while leaving current default-reporting and docs surfaces aligned to `auto`, and it still lacks after-fix real behavior proof.

## What This Changes

The PR changes omitted exec host resolution from `auto` to `sandbox` and updates exec runtime/path/agent config tests while preserving explicit `host: "auto"` fallback.

## Best Possible Solution

Decide the exec default policy first; if fail-closed is intended, centralize the omitted-host default and update docs, `/exec` status/default reporting, and related tests together.

## Reproduction Assessment

Yes. Source inspection and current tests show current main resolves an omitted exec host through `auto` to `gateway` when no sandbox runtime is available; I did not run tests because this review is read-only.

## Solution Assessment

No. Changing only `resolveExecTarget` is too narrow for a documented default-policy change; the maintainable fix needs the default resolver, command/status surfaces, docs, and proof aligned together.
