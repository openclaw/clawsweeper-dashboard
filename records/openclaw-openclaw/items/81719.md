## Summary

Close as superseded: current main and the latest release intentionally keep omitted exec host config on the `auto` path, resolving to gateway when no sandbox runtime exists; this PR reverses that maintainer-set contract and would break existing no-sandbox setups while docs and audit still advertise `auto`.

## What This Changes

The PR changes omitted exec host defaults from `auto` to `sandbox` and retargets exec runtime tests to expect fail-closed behavior without a sandbox runtime.

## Best Possible Solution

Keep the shipped `auto` default and rely on explicit `tools.exec.host=sandbox` for fail-closed installs; any default change should be maintainer-owned with docs, audit, doctor, fresh-install, and upgrade proof.

## Reproduction Assessment

Yes. Source inspection shows current main defaults omitted exec host to `auto` and routes no-sandbox sessions to gateway, while the PR changes that omitted path to `sandbox`, which reaches the existing no-sandbox runtime error.

## Solution Assessment

No. This is not the best fix because current main already resolved the original phantom-sandbox problem with the documented `auto` default; changing omitted config to fail closed needs an explicit maintainer product and upgrade decision.
