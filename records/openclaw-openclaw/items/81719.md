## Summary

Keep open. The PR changes a security-sensitive exec default, but current main still defines and documents omitted exec host as `auto`, and this branch leaves other default-reporting surfaces out of sync while providing only test-command validation instead of real behavior proof.

## What This Changes

The PR changes omitted exec host routing to sandbox/fail-closed while preserving explicit `host: "auto"` fallback and adjusts exec runtime/config tests.

## Best Possible Solution

Decide the exec default policy first; if fail-closed is intended, centralize the omitted-host default and update docs/status/skill surfaces while keeping explicit `host:auto` as the opt-in fallback.

## Reproduction Assessment

Yes, source inspection and current tests show the current-main behavior: omitted exec host resolves through `auto` to `gateway` when no sandbox runtime is available. I did not run tests because this review is read-only.

## Solution Assessment

No. Changing only `resolveExecTarget` is too narrow for a documented default-policy change; the maintainable fix needs the default resolver, command/status surfaces, docs, and proof aligned together.
