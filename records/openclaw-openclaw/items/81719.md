## What This Changes

The PR changes omitted exec host config to select `sandbox` instead of `auto` and updates exec runtime/config tests so explicit `host: "auto"` remains the gateway fallback path.

## Best Possible Solution

Preserve the shipped `auto` default unless maintainers explicitly approve a breaking fail-closed default; if approved, change the resolver, docs, tests, and upgrade guidance together and prove fresh-install plus upgrade/no-sandbox behavior.

## Reproduction Assessment

Yes from source, not from an applied runtime run. Current main tests and docs show omitted exec host resolves through `auto`; applying this one-line fallback change would make no-sandbox omitted-host exec throw as `sandbox` instead.

## Solution Assessment

No. The one-line low-level fallback is not the best fix because default semantics are owned by the canonical resolver, docs, config types, and upgrade policy, not only `resolveExecTarget`.
