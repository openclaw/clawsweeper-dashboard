## Summary

Keep open: the patch is a plausible exec security hardening, but it changes a documented default policy incompletely and still lacks real behavior proof from a real setup.

## What This Changes

This PR changes omitted exec host resolution to prefer sandbox/fail-closed, keeps explicit `host: "auto"` as the gateway/node fallback path, and updates exec runtime/config tests.

## Best Possible Solution

Decide the exec default policy first; if fail-closed is intended, update the resolver, default/status reporting, docs, and skill/node availability surfaces together while preserving explicit `host:auto` fallback.

## Reproduction Assessment

Yes. Source inspection and current tests show current main defaults omitted exec host to `auto` and expects gateway execution when no sandbox runtime is available; I did not run tests because this review is read-only.

## Solution Assessment

No. Changing only `resolveExecTarget` is too narrow for a default-policy/security change because the default resolver, `/exec` status text, docs, and related advertising surfaces would still describe the old behavior.
