## Summary

Close as superseded: current main and the latest release already define the safer supported contract here, where omitted exec host stays `auto`, no-sandbox `auto` routes to `gateway`, and only explicit `host=sandbox` fails closed without a sandbox runtime. ## What This Changes

The PR changes omitted exec host configuration from `auto` to `sandbox` and updates exec runtime/config tests to expect fail-closed behavior when no sandbox runtime is available, while preserving explicit `host=auto` as the gateway fallback path.

## Best Possible Solution

Keep the shipped `auto` default, keep explicit `host=sandbox` as the fail-closed path, and handle any remaining docs nuance through https://github.com/openclaw/openclaw/issues/61009 rather than changing runtime defaults.

## Reproduction Assessment

Yes. Source and release-tag tests show the current no-sandbox `auto` path runs on gateway, while the PR's changed default would make omitted config select `sandbox` and hit the sandbox-runtime-required error.

## Solution Assessment

No. The proposed runtime default is not the best fix because current source, docs, tests, and prior maintainer discussion define `auto` as the compatibility-preserving default; explicit `host=sandbox` is already the fail-closed path.
