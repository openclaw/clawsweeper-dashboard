## Summary

Keep open: this PR is a security-sensitive exec default-policy change, but it is not mergeable as-is because it changes only the runtime resolver while current-main default/status/docs surfaces still define omitted exec host as `auto`, and the branch has no real after-fix behavior proof.

## What This Changes

The PR changes omitted exec host routing to sandbox/fail-closed, preserves explicit `host: "auto"` fallback coverage, and updates exec runtime/config tests.

## Best Possible Solution

If fail-closed is intended, centralize the omitted-host default across exec config/default reporting, `/exec` status, skill eligibility, docs, and regression tests; otherwise keep `auto` as the documented default.

## Reproduction Assessment

Yes. Source inspection and current tests show omitted exec host resolves through `auto` to `gateway` when no sandbox runtime is available; I did not run tests because this review is read-only.

## Solution Assessment

No. Changing only `resolveExecTarget` is too narrow for a documented default-policy change; the maintainable fix needs the runtime, default-reporting, status/skill surfaces, docs, and proof aligned together.
