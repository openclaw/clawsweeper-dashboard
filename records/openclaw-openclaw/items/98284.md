## Summary

Keep open: this PR targets a reproduced managed-browser cookie/session loss bug, but the new plaintext cookie sidecar still has a concrete permission-enforcement defect and needs maintainer approval for the sensitive-storage model.

## What This Changes

The PR adds managed Chrome cookie snapshot/restore through browser-level CDP, a `cookies.json` sidecar with periodic and stop-time flushing, reset cleanup, focused tests, and CLI docs.

## Best Possible Solution

Enforce 0600 permissions on every sidecar save, keep the current CDP lifecycle wiring only if maintainer security/product review accepts the plaintext cookie sidecar, and otherwise pause for an opt-in or encrypted design.

## Reproduction Assessment

Yes. The linked issue includes a current-main live repro where a browser-level CDP cookie exists before restart and disappears after relaunch, and current main has no cookie snapshot/restore path.

## Solution Assessment

No, not yet. Browser-level `Storage.getCookies`/`Storage.setCookies` is the right owner-boundary mechanism, but the sidecar writer must enforce restrictive permissions on overwrite and maintainers still need to accept the sensitive-storage model.
