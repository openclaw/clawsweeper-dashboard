## Summary

Keep open. Current main implements a useful subset of the field report: selector targeting for many managed browser act actions, batched act execution, click delay handling, refs="aria" guidance, wait-by-selector, and tests for normalized batch dispatch. The issue still asks for meaningful browser-tool surfaces that are not present or not aligned in the agent schema/docs: CDP inspectElement/event-listener inspection, snapshot captchaDetected signaling, refsStale or auto-refresh signaling, post-fill value verification, wait-by-ref semantics, and schema/docs alignment for the already-shipped batch/selector behavior.

## Best Possible Solution

Keep this open and split it into focused browser-plugin follow-ups. Treat selector targeting, managed-route batch actions, click delay, refs="aria" guidance, and wait-by-selector as partially shipped; separately track inspectElement/CDP event-listener inspection, snapshot CAPTCHA detection, stale-ref signaling or auto-refresh behavior, post-fill verification or stronger slowly=true guidance, wait-by-ref semantics, and schema/docs alignment for supported act actions.
