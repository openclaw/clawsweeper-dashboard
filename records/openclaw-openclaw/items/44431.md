Keep open. Current main and v2026.6.6 already cover important parts of the field report, especially selector-backed managed actions, click delay, selector waits, ARIA refs, and server-side batch execution, but the issue still has meaningful missing browser action/snapshot contracts around stale-ref/readiness signaling, CAPTCHA signaling, CDP event-listener inspection, post-fill verification, action-result diffs, and model-facing batch/schema/docs clarity.

## What This Changes

The issue requests seven browser-tool ergonomics and reliability improvements spanning selectors, stale refs, CDP inspection, batched actions, form-fill verification, CAPTCHA/readiness signaling, and element waits.

## Best Possible Solution

Keep the shipped selector, click-delay, ARIA-ref, selector-wait, and server-side batch support, then split the remaining browser-plugin work into focused maintainer-approved contracts with live proof for readiness/stale refs, CAPTCHA signaling, CDP inspection, post-fill verification, action-result diffs, and schema/docs alignment.
