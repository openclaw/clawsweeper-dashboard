## Summary

Keep open: current main covers important parts of this field report through selector targeting, batch actions, click delay, selector waits, and ARIA-ref guidance, but several requested capabilities remain unimplemented and are not superseded by a canonical open follow-up. The related #45457 merge addressed only part of the umbrella request from https://github.com/openclaw/openclaw/issues/44431.

## What This Changes

The issue requests seven browser tool ergonomics improvements: selector-based actions, ref staleness handling, CDP element inspection, batch actions, fill verification, CAPTCHA signaling, and wait-for-element/ref support.

## Best Possible Solution

Keep the existing selector/batch/ARIA-ref work, then split the remaining browser ergonomics into narrower issues or PRs with explicit API contracts for CDP inspection, CAPTCHA signaling, fill verification, and any ref-staleness or wait-for-ref behavior.
