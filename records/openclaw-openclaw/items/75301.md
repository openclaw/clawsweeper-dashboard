## Summary

Keep open. Current main has closed much of the legacy `plugin-runtime-deps` growth path, but it still lacks the requested cross-cache inventory/prune surface, and maintainer discussion explicitly says the broader browser/tools/memory/orphan-session cleanup feature should remain open.

## What This Changes

The issue requests a first-class `openclaw caches` or doctor-integrated command to inventory and safely prune cache-class state under `~/.openclaw/`.

## Best Possible Solution

Design a conservative core local-state inventory and prune surface with dry-run output, size accounting, allowlisted cache classes, protected data paths, docs, and legacy plugin-runtime-deps treated as diagnostic cleanup rather than an active cache class.
