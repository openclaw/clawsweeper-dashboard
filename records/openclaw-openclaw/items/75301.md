## Summary

Keep open. Current main has made the `plugin-runtime-deps` growth example mostly historical, but it still lacks the requested cross-cache inventory and prune surface for browser/tools/memory/session artifacts; the remaining work needs maintainer design around command shape and deletion boundaries.

## What This Changes

Request an `openclaw caches` command or doctor integration that lists cache-class directories under OpenClaw state with sizes and safely prunes stale plugin deps, orphan transcripts, browser/tools downloads, and memory caches.

## Best Possible Solution

Keep the issue open and scope it as a core local-state inventory feature: a dry-run-first cache table with explicit cache-vs-data classification, allowlisted prune targets, current/active/locked artifact protection, and opt-in handling for expensive regenerable browser, tools, memory, and legacy plugin-dependency state.

## Reproduction Assessment

Not applicable as a feature request. The verification path is source/docs inspection on current main: there is no cross-cache command, while existing doctor, plugin-dependency cleanup, browser, memory, and sessions surfaces cover only partial slices.

## Solution Assessment

Unclear. The requested class-based inventory plus conservative dry-run pruning is the right direction, but the best public entry point and deletion defaults need maintainer design before a safe implementation PR.
