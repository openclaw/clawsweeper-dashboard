## Summary

Keep open. Current main has addressed much of the old `plugin-runtime-deps` growth and now has retention-based session artifact pruning, but it still lacks the requested cross-cache inventory/prune surface for browser, tools, memory, and local-state cache classes; the maintainer discussion also explicitly says the broader feature should remain open.

## What This Changes

Add an `openclaw caches` or doctor-integrated command that lists cache-class directories under `~/.openclaw/` and safely prunes stale/regenerable entries.

## Best Possible Solution

Design a conservative local-state inventory and prune surface with dry-run output, size accounting, allowlisted cache/data classes, protected paths, docs, and plugin-runtime-deps treated as legacy diagnostics; keep transcript-specific audit controls aligned with https://github.com/openclaw/openclaw/issues/77941.

## Reproduction Assessment

Not applicable as a feature request. Source and docs provide a high-confidence verification path: current main lacks `openclaw caches` or equivalent cross-cache inventory/prune behavior, while existing cleanup paths cover only legacy plugin deps and session artifacts.

## Solution Assessment

Unclear. The proposed inventory/prune direction is reasonable, but the command shape, destructive defaults, and browser/tools/memory/session cache policy need maintainer product decisions before code work.
