## Summary

Keep open. Current main still lacks the requested cross-cache inventory/prune surface, existing cleanup commands cover only partial slices, and a maintainer comment explicitly keeps the broader local-state feature open after making plugin-runtime-deps mostly legacy.

## What This Changes

Add an `openclaw caches` command or doctor integration to inspect cache-class directories under `~/.openclaw/` and safely prune stale runtime deps, orphan transcripts, browser/tools downloads, and related regenerable state.

## Best Possible Solution

Design a core-owned cache inventory and prune surface with dry-run-first output, allowlisted classes, active/fresh safeguards, docs, and `plugin-runtime-deps` treated as legacy diagnostic cleanup rather than an actively growing cache.

## Reproduction Assessment

Not applicable as a feature request. Current-main source and docs verification gives a high-confidence way to confirm the requested cross-cache command is still missing while only partial cleanup paths exist.

## Solution Assessment

Unclear. The class-based inventory and conservative prune defaults are a good direction, but the public entry point and destructive deletion boundaries need maintainer design before implementation.
