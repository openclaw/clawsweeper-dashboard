## Summary

Keep open. Current main still has only narrow cleanup paths for bundled runtime deps and sessions, and it does not provide the requested cross-cache `openclaw caches` inventory/prune surface for `~/.openclaw`; the related runtime-deps and session-cleanup reports remain relevant rather than superseding this feature request.

## What This Changes

Add a first-class cache inspection and pruning command, or equivalent doctor integration, for cache-class directories under `~/.openclaw` such as plugin-runtime-deps, browser/tools downloads, memory caches, and orphan transcripts.

## Best Possible Solution

Design a core-owned, dry-run-first cache inventory and prune surface with explicit cache-vs-data classes, deletion allowlists, current-version and lock protection for runtime deps, fresh-session safeguards, opt-in pruning for expensive caches, and matching docs/changelog coverage.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection provide high-confidence verification that current main lacks the requested cross-cache command while only partial cleanup surfaces exist.

## Solution Assessment

Unclear whether the final UI should be a top-level command or doctor integration. The requested class-based inventory and explicit prune rules are the right direction, but deletion policy needs maintainer guardrails before implementation.
