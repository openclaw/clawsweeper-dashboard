## Summary

Keep open. Current main has targeted plugin runtime-deps and session cleanup paths, including newer pruning after bundled dependency repair, but it still does not provide the requested cross-cache `openclaw caches list/prune` inventory for `~/.openclaw` or a maintainer-approved deletion model across browser, tools, memory, runtime-deps, transcripts, and user-owned data.

## What This Changes

Request to add a first-class command, or doctor integration, that lists cache-class directories under the OpenClaw state directory and safely prunes stale/regenerable cache entries.

## Best Possible Solution

Add a core-owned cache inventory abstraction that classifies state-dir entries as active cache, stale cache, expensive regenerable cache, or user data, then expose a conservative dry-run command and optional doctor prompt with explicit allowlists, lock/current-version protection, transcript age/live guards, and docs/changelog coverage.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection gives high-confidence verification that current main lacks the requested cross-cache command while only partial cleanup surfaces exist.

## Solution Assessment

Unclear whether a top-level command or doctor-integrated flow is the final product shape. The maintainable direction is a shared cache-class registry with dry-run-first pruning and opt-in handling for expensive or user-adjacent caches.
