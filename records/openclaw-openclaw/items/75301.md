## Summary

Keep open. Current main makes the plugin-runtime-deps growth path mostly legacy cleanup, but it still lacks the requested cross-cache inventory/prune command; existing doctor and sessions commands cover only partial slices, and maintainer context explicitly keeps the broader feature open.

## What This Changes

Request a core CLI surface to inventory cache-class directories under the OpenClaw state directory, report size/class/owner information, and safely prune stale or regenerable cache data.

## Best Possible Solution

Design a core-owned cache inventory and prune surface with dry-run-first output, allowlisted cache classes, active/locked/fresh safeguards, and separate opt-ins for expensive regenerable data such as browser, tools, and memory artifacts.

## Reproduction Assessment

Not applicable as a feature request. Current-main source and docs inspection gives high-confidence verification that the requested cross-cache command is still missing while only partial cleanup paths exist.

## Solution Assessment

Unclear. A class-based inventory plus conservative prune defaults looks like the right direction, but the public entry point and destructive deletion boundaries need maintainer design before implementation.
