Keep open. Current main only covers pieces of the request: legacy plugin dependency debris and scoped session cleanup, while the requested cross-cache inventory/prune command is still absent, and a maintainer comment explicitly keeps the broader feature open.

## What This Changes

The issue requests a CLI surface to inspect and safely prune OpenClaw-managed cache classes across plugin dependency debris, browser/tools downloads, memory artifacts, and orphan session files.

## Best Possible Solution

Design a core-owned cache inventory and prune surface with dry-run-first output, explicit cache/data classes, allowlisted stale targets, active-state safeguards, and separate opt-ins for expensive regenerable artifacts.
