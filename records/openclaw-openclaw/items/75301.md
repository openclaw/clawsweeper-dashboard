## Summary

Keep open. Current main has partial runtime-deps and session cleanup paths, but it still lacks the requested cross-cache inventory/prune command for `~/.openclaw` and the safe deletion boundaries need maintainer design.

## What This Changes

Request an `openclaw caches` or doctor-integrated command that inventories cache-class state directories, reports size/class/source, and safely prunes stale runtime deps, orphan transcripts, and regenerable browser/tools/memory caches.

## Best Possible Solution

Design a core-owned cache inventory with dry-run-first pruning, explicit cache-vs-data classes, an allowlisted deletion model, current-version and lock protection for runtime deps, fresh/live transcript safeguards, and opt-in pruning for expensive or user-adjacent caches such as memory, browser, and tools.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection gives high-confidence verification that current main lacks the requested cross-cache command while only partial cleanup surfaces exist.

## Solution Assessment

Unclear whether the best public surface is a top-level command or doctor integration. The safest direction is the requested class-based inventory plus conservative dry-run pruning rules, with automatic deletion left for later maintainer approval.
