## Summary

Keep open. Current main still lacks the requested cross-cache inventory/prune surface; existing cleanup covers only legacy plugin dependency debris and session-store artifacts, and a maintainer comment explicitly keeps the broader local-state feature open.

## What This Changes

Add an `openclaw caches` or doctor-integrated command to list and safely prune cache-like state under `~/.openclaw`, including legacy plugin-runtime-deps, browser/tools/memory caches, and orphan transcripts.

## Best Possible Solution

Define a core-owned cache inventory and conservative prune flow with dry-run output, explicit cache/data classes, allowlisted deletion targets, active/fresh safeguards, and docs; treat `plugin-runtime-deps` as legacy diagnostic cleanup on current main.

## Reproduction Assessment

Not applicable as a feature request. Current-main source and docs inspection gives a high-confidence verification path: the requested cross-cache command is absent while only scoped legacy plugin and session cleanup paths exist.

## Solution Assessment

Unclear. The proposed class-based, dry-run-first command is a good direction, but the public entry point and destructive deletion policy need maintainer design before implementation is the best fix.
