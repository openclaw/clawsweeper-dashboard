## Label Justifications

- `P1`: A permanently non-durable Codex native subagent completion can wake and re-run the parent agent every five minutes indefinitely in real channel workflows.
- `impact:message-loss`: The child completion never reaches a durable parent delivery state when the parent handoff remains non-durable.
- `impact:session-state`: The monitor leaves the completion pending forever and repeatedly re-enters the parent session instead of settling the task delivery state.

## Summary

Keep open: current main and v2026.6.10 still leave Codex native subagent completion delivery retries unbounded when the parent handoff remains non-durable, and no open related item or PR specifically owns this monitor retry cap.

## What This Changes

The issue requests a bounded retry/deadline and terminal give-up state for Codex native subagent completion delivery when parent completion handoff is permanently non-durable.

## Best Possible Solution

Add a bounded retry policy in the Codex native subagent monitor, record a terminal failed or blocked delivery state on give-up, and cover permanent non-durable plus success-after-retry cases without adding new config surface.

## Reproduction Assessment

Yes, at source level: current main keeps pending completion state and re-arms `deliverPendingCompletion` on every non-durable result with no cap or deadline. I did not run a live external-channel failure in this read-only review.

## Solution Assessment

Yes: a bounded retry/deadline with terminal failed or blocked delivery in the Codex native monitor is the narrowest maintainable fix. Adding config knobs is not necessary for this bug and would raise the surface area.
