## Summary

Keep open: the branch has source-level correctness blockers, lacks after-fix runtime proof, and introduces a security-sensitive config surface needing maintainer review.

## What This Changes

The PR adds agent-level and cron-payload provider request override config, threads request override parameters into model resolution, adds resolver tests, and changes a Canvas A2UI bundle hash.

## Best Possible Solution

Thread provider request overrides as prepared runtime facts through agent and cron model execution, cover default/no-list cron and update paths, add redaction/docs/security coverage, and remove unrelated generated output.

## Reproduction Assessment

Yes for the review defects: source inspection of the PR head shows the late agent merge, agents.list-dependent cron merge, and missing cron payload update handling. There is no current-main user bug reproduction because this PR is adding a new feature surface.

## Solution Assessment

No. Mutating cron agent config and merging after providerConfig resolution is not the narrow maintainable path; the safer direction is to pass resolved request overrides into model resolution directly and update the public config/redaction contract consistently.
