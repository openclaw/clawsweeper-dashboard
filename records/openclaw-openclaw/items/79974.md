## Summary

This PR should stay open: the feature is useful, but the current patch misses common runtime paths for agent/job overrides, changes an unrelated generated hash, and lacks required real behavior proof for an external contribution.

## What This Changes

The PR adds agent-scoped and cron-payload provider request override fields, threads them into model request transport resolution, adds resolver tests, and changes a Canvas generated bundle hash.

## Best Possible Solution

Thread request overrides as prepared runtime facts through agent and cron model resolution, cover built-in provider/no agents.list and cron update/normalization cases, document and redaction-test the new secret-bearing surface, and remove unrelated generated output.

## Reproduction Assessment

Yes for the review defects: source inspection shows the new merge runs after the no-providerConfig early return and the cron payload update path never consumes providers. This PR is otherwise a feature request, not a user bug report with a current-main failing reproduction.

## Solution Assessment

No. Mutating cron agent config and merging after the providerConfig branch misses common runtime paths; the maintainable direction is to pass resolved request overrides into model resolution directly and update the public config contract consistently.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Apply agent overrides without requiring global provider config:** `src/agents/pi-embedded-runner/model.ts:556-561`
  - body: The new agent request merge sits after the existing `if (!providerConfig)` return, so a built-in provider with only `agents.list[].providers.<id>.request` never reaches this code. That makes the advertised per-agent headers a no-op unless users also define a global `models.providers.<id>` block.
  - confidence: 0.9
- **[P2] Pass job overrides when no agent list entry exists:** `src/cron/isolated-agent/run.ts:496-499`
  - body: This only applies `payload.providers` by cloning `cfgWithAgentDefaults.agents.list`; default-agent cron jobs without an `agents.list` row, or jobs whose agent id is not present there, silently drop the job-level request override. The job override should be threaded to model resolution instead of depending on a config list entry.
  - confidence: 0.88
- **[P2] Handle providers in cron payload patching:** `src/cron/types.ts:176-178`
  - body: The type now advertises `payload.providers` and null clearing for cron updates, but `mergeCronPayload` and `buildPayloadFromPatch` never copy, merge, or delete that field. API/CLI update paths will accept the shape at the type level while dropping the new override at runtime.
  - confidence: 0.84
- **[P3] Remove the unrelated generated bundle hash change:** `extensions/canvas/src/host/a2ui/.bundle.hash:1`
  - body: This hash-only Canvas generated file change is unrelated to provider request headers and has no matching bundle/source change in the PR. It should be removed or moved to the generated bundle workflow that produced it.
  - confidence: 0.86
