## Best Possible Solution

Land the plugin-local bounded-read fix after required CI is green and maintainers accept the 16-read cap; tune the constant later only if FileProvider proof shows it remains too high.

## Reproduction Assessment

Yes for the source-level trigger: current main fans out page-summary `fs.readFile` calls through unbounded `Promise.all`, and compile can hit that path multiple times. I did not live-reproduce the exact macOS FileProvider `EDEADLK` symptom in this read-only review.

## Solution Assessment

Yes: using the existing documented `openclaw/plugin-sdk/concurrency-runtime` helper inside the Memory Wiki compile path is the narrowest maintainable fix. A config knob is not needed unless follow-up provider-specific proof shows maintainers need operator tuning.
