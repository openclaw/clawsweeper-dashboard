## Repair Work Prompt

Fix the narrow core subprocess timeout-result problem from https://github.com/openclaw/openclaw/issues/73743. In current main, `runCommandWithTimeout` can return `termination: "timeout"` or `"no-output-timeout"` with `code: null` after SIGKILL, and plugin callers that only treat numeric exit codes as failures can misclassify partial stdout as success. Normalize timeout and no-output-timeout results to a stable numeric nonzero code, preferably `124`, when the resolved process code is `null` or `0`, while preserving explicit nonzero child exit codes. Add focused tests that assert timeout/no-output-timeout results have numeric nonzero codes, not just `not.toBe(0)`, and keep the plugin runtime forwarding contract covered. Do not attempt to modify external recipes/Kitchen code or solve the macOS launchd fan-out/startup performance work in this repair PR. Add a user-facing `CHANGELOG.md` fix entry credited to `@rjdjohnston`.
