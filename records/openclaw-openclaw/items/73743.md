## Summary

Keep open. Current main includes several startup and plugin-runtime mitigations, but it still does not prove the reported macOS recipes/Kitchen fan-out regression is fixed, and the core subprocess timeout result path can still surface a null code to plugin callers.

## What This Changes

The issue reports slow `openclaw` CLI subprocess startup on macOS, launchd-driven recipes workflow fan-out causing multi-minute worker ticks, and ClawKitchen caching partial timed-out subprocess output as a successful `recipes list` result.

## Best Possible Solution

Keep this as an active reliability issue: first harden core subprocess timeout results so timeout kills always look like numeric failures to plugin callers, then separately profile and lighten the recipes/Kitchen command and worker startup path, including descriptor coverage, lightweight worker execution, and launchd jitter guidance if needed.
