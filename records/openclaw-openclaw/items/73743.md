## Repair Work Prompt

Fix the timeout-result part of https://github.com/openclaw/openclaw/issues/73743. In `src/process/exec.ts`, make `runCommandWithTimeout` return an unambiguous failure for `termination: "timeout"` and `termination: "no-output-timeout"` even when Node reports no child exit code, preserving Windows shim success behavior for normal exits. Prefer a stable numeric nonzero code such as `124` for timeout/no-output-timeout null-code cases, and keep the existing `termination`/`noOutputTimedOut` fields intact for callers. Add focused regression coverage for global timeout and no-output-timeout cases where both close/exit code values are null, and update plugin runtime tests only if the public runtime contract needs an assertion. Do not attempt launchd jitter, recipes worker RPC redesign, plugin-specific cache rewrites, or new config flags in this PR. Add a single-line `CHANGELOG.md` fix entry because this affects user-visible plugin subprocess behavior.
