## Repair Work Prompt

Implement a narrow core fix for https://github.com/openclaw/openclaw/issues/73743: `runCommandWithTimeout` must return a numeric non-zero code for `timeout` and `no-output-timeout` results when the child reports no code, so plugin runtime callers cannot treat timeout-killed partial output as success. Preserve the Windows shim behavior that infers success only for non-timeout exits with missing codes. Update focused tests to assert timeout and no-output-timeout results are numeric and non-zero, and keep `runtime.system.runCommandWithTimeout` forwarding behavior covered. Add a single-line `CHANGELOG.md` fix entry because this is user-facing subprocess behavior. Do not change plugin startup policy, launchd scheduling, recipes/Kitchen plugin behavior, dependency patches, or unrelated process spawning semantics.
