## Summary

Keep open. Current main still allows timeout-killed subprocesses to reach plugin runtime with `code: null`, so the cache-poisoning failure mode is not fixed; current lazy plugin CLI activation is only a partial startup mitigation, and the recipes/Kitchen fan-out path cannot be proven resolved from this tree.

## What This Changes

The issue reports slow OpenClaw CLI subprocess startup on macOS, launchd worker fan-out causing multi-minute recipes ticks, and ClawKitchen caching partial timeout output as successful JSON.

## Best Possible Solution

Keep this active until timeout-killed subprocesses always report a numeric non-zero failure to plugin callers, then profile the recipes/Kitchen macOS command path to choose a startup or scheduling fix with evidence.
