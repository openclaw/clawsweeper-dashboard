## Summary

Keep open: current main and latest release still expose memory-core-owned dreaming plus a memory-core sidecar path for selected memory plugins, not a public dreaming provider protocol. The posted ClawHub plugin helps LanceDB users today, but it does not settle the OpenClaw core/plugin-SDK product decision.

## What This Changes

Requests a memory plugin dreaming protocol so non-memory-core memory plugins can participate in light, REM, and deep consolidation under OpenClaw orchestration.

## Best Possible Solution

Have maintainers decide a public plugin-SDK dreaming contract, likely in coordination with memory role-slot work, or explicitly document that dreaming orchestration remains plugin-owned through ClawHub rather than OpenClaw-managed.
