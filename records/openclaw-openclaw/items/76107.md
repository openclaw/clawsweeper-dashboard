## Summary

Keep open. Current main has unreleased cache work in the reported hot paths, but the code still performs per-attempt auth, tool preparation, and prompt assembly, and there is no current-main MiniMax/Feishu benchmark proving the reported 6.2s auth and 16s fixed overhead are gone.

## What This Changes

The issue asks to reduce per-message auth, core plugin tool, and system-prompt overhead by caching credentials, prompt work, and plugin initialization across warm sessions.

## Best Possible Solution
