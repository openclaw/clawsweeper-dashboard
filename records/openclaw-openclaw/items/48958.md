## Summary

Keep open: this PR targets a real prompt/media contamination bug, but the patch is incomplete because it sanitizes only the embedded `<relevant-memories>` path while CLI prompt-image loading and active-memory untrusted-context blocks can still turn historical media markers into current image input.

## What This Changes

The PR strips `<relevant-memories>` and `<relevant_memories>` blocks before embedded-runner prompt image detection and adds focused regression tests in `images.test.ts`.

## Best Possible Solution

Land a shared prompt-image sanitization boundary used by both embedded and CLI image scanners, stripping injected relevant-memory and active-memory/untrusted-context blocks before image detection while leaving broader recent-history pruning to the linked canonical bug if it is out of this PR's scope.

## Reproduction Assessment

Yes, source-level: current main prepends `before_prompt_build` context into the prompt and then scans that prompt for `[media attached: ...]` image refs in both embedded and CLI paths. I did not run a live model session because this was a read-only review.

## Solution Assessment

No. The PR's direction is right for memory-lancedb `<relevant-memories>` blocks, but the maintainable fix should sanitize the shared prompt-image boundary, cover active-memory untrusted-context blocks, and fix the case-sensitive precheck plus the vacuous test assertion.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Sanitize the CLI image scan path too:** `src/agents/pi-embedded-runner/run/images.ts:329`
  - body: The new sanitizer only runs inside `detectAndLoadPromptImages()`. CLI runs also prepend hook memory context before `loadPromptRefImages()` scans the prepared prompt, so stale `[media attached: ...]` lines from recalled memories can still become active CLI image inputs after this patch.
  - confidence: 0.9
- **[P2] Strip active-memory untrusted-context blocks:** `src/agents/pi-embedded-runner/run/images.ts:25-30`
  - body: The helper strips only `<relevant-memories>` variants, but active-memory injects `Untrusted context` plus `<active_memory_plugin>` blocks through the same hook path. Media markers inside those summaries are still scanned as current prompt images.
  - confidence: 0.86
- **[P2] Make the memory-block precheck case-insensitive:** `src/agents/pi-embedded-runner/run/images.ts:27-29`
  - body: `stripInjectedMemoryBlocks()` returns early on `text.includes("relevant")`, while the replacement regex is case-insensitive. Mixed-case tags such as `<RELEVANT-MEMORIES>` would skip stripping and preserve the stale image refs.
  - confidence: 0.82
- **[P3] Replace the vacuous memory-block assertion:** `src/agents/pi-embedded-runner/run/images.test.ts:316`
  - body: `expect(refs.length).toBeGreaterThanOrEqual(0)` passes for every array, so this test does not prove or document the intended behavior. Assert that `detectImageReferences()` still finds the raw ref, or remove this unit test and keep the integration coverage.
  - confidence: 0.95
