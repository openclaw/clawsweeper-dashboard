## Summary

Keep open: this is a plausible focused bug fix for prompt-image contamination, but current main does not have this exact image-loader guard and the external PR still lacks after-fix real behavior proof. The branch also needs a small cleanup for the case-sensitive precheck and a vacuous test assertion before it is merge-ready.

## What This Changes

The PR strips `<relevant-memories>` and `<relevant_memories>` blocks before prompt image detection and adds focused image-loading tests.

## Best Possible Solution

Land a narrow image-loading boundary fix that ignores injected memory/runtime context before resolving prompt image refs, covers the relevant memory variants in regression tests, and preserves current-turn attachments only.

## Reproduction Assessment

Yes, at source level. Current main scans the prompt text passed to `detectAndLoadPromptImages()`, and memory hooks can inject `<relevant-memories>` text containing `[media attached]` markers; I did not run tests because this review is read-only.

## Solution Assessment

Mostly yes. Filtering injected memory context before image-reference detection is a narrow defensive fix, but the patch should align the case guard with the regex and clean up the ineffective test assertion.
