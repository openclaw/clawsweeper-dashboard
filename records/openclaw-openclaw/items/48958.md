## Summary

Keep open: the PR is aimed at a real current-main bug path, but the patch is incomplete against current memory injection surfaces and has small correctness/test issues that should be fixed before merge.

## What This Changes

The PR strips `<relevant-memories>`/`<relevant_memories>` blocks before prompt image detection and adds focused `images.test.ts` coverage.

## Best Possible Solution

Land a narrow prompt-image sanitizer at the image-detection boundary, or pass only current-turn text into detection, covering memory-lancedb and active-memory injected wrappers with regression tests and a changelog entry.

## Reproduction Assessment

Yes, by source inspection: a vision-capable prompt containing hook-prepended memory context with `[media attached: ...]` reaches `detectAndLoadPromptImages()` and is scanned as current prompt text. I did not execute the Vitest reproduction because `pnpm` is unavailable in this checkout.

## Solution Assessment

No. Stripping injected context before image detection is the right direction, but this PR only covers relevant-memories blocks and should also cover active-memory/untrusted-context wrappers or avoid scanning injected context altogether.

  - confidence: 0.93
