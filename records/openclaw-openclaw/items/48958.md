## Summary

Keep open: the PR is a plausible narrow bug fix for prompt image hydration, but it still lacks external after-fix behavior proof and has two small branch issues that should be fixed before normal maintainer review.

## What This Changes

The PR strips `<relevant-memories>` and `<relevant_memories>` blocks before prompt image-reference detection and adds focused tests around stale media markers in those blocks.

## Best Possible Solution

Land a rebased narrow image-loading boundary fix that prevents injected memory/runtime context from being scanned for current-turn image refs, while leaving broader recent-history image policy to the canonical follow-up.

## Reproduction Assessment

Yes, at source level: a prompt string containing `<relevant-memories>` with `[media attached: ...]` markers can be scanned by `detectImageReferences()` when passed to `detectAndLoadPromptImages()`. I did not run a live memory or Telegram setup in this read-only review.

## Solution Assessment

Mostly yes: scanner-level stripping is a narrow defensive fix for this relevant-memory path. The branch should fix the guard/test issues, provide real after-fix proof, and be validated against current main's prompt/runtime-context split.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.82

Full review comments:

- **[P3] Make the memory-block precheck case-insensitive:** `src/agents/pi-embedded-runner/run/images.ts:26-30`
  - body: `stripInjectedMemoryBlocks` uses a case-sensitive `text.includes("relevant")` guard before a case-insensitive regex. Mixed-case relevant-memory tags would skip stripping even though the regex is written to match them, so lowercase the guard input or remove the guard.
  - confidence: 0.86
- **[P3] Replace the vacuous image-ref assertion:** `src/agents/pi-embedded-runner/run/images.test.ts:316`
  - body: `expect(refs.length).toBeGreaterThanOrEqual(0)` passes for every array length, so this test does not verify parser behavior or caller-level stripping. Assert the raw parser behavior or remove the test and rely on the integration cases.
  - confidence: 0.91
