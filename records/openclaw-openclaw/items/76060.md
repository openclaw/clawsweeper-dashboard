## Summary

Keep open. The PR addresses real Discord proxy gaps on current main, but the latest head still forces Discord gateway WebSocket connections through an env proxy even when NO_PROXY excludes the gateway host, so it needs a narrow follow-up before merge.

## What This Changes

The branch adds Discord undici-based proxy transport wiring for REST, webhooks, monitor REST, and gateway WebSocket paths, removes loopback-only proxy validation, adds proxy tests, and updates the changelog.

## Best Possible Solution

Land the Discord-owned transport fix after the WebSocket managed-proxy resolver preserves NO_PROXY semantics and focused gateway tests cover that bypass case.

## Reproduction Assessment

Yes. Current main has a source-level reproduction for the original bug: a remote channels.discord.proxy URL is rejected by validateDiscordProxyUrl, and the PR review finding reproduces on head with HTTPS_PROXY plus NO_PROXY=gateway.discord.gg because the WebSocket resolver returns the env proxy URL without a no-proxy check.

## Solution Assessment

No, not quite. The Discord-local undici transport approach is the right shape, but gateway WebSocket env-proxy selection should use shouldUseEnvHttpProxyForUrl("https://gateway.discord.gg", env) or equivalent before constructing HttpsProxyAgent.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Respect NO_PROXY for gateway WebSocket proxies:** `extensions/discord/src/fetch.ts:75-78`
  - body: The new WebSocket resolver returns an env proxy URL whenever HTTPS_PROXY/HTTP_PROXY/ALL_PROXY is set, and gateway-plugin passes that URL straight into HttpsProxyAgent. That bypasses the existing NO_PROXY/no_proxy matcher used by EnvHttpProxyAgent, so HTTPS_PROXY plus NO_PROXY=gateway.discord.gg still forces the Discord gateway socket through the excluded proxy. Gate the env-proxy branch with shouldUseEnvHttpProxyForUrl("https://gateway.discord.gg", env), and do not fall back to OPENCLAW_PROXY_URL when env proxy is configured but bypassed for this host.
  - confidence: 0.88
