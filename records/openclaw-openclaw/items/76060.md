## Summary

Keep open: the PR addresses a real Discord proxy gap, but the new gateway WebSocket env-proxy path should honor NO_PROXY/no_proxy before merge.

## What This Changes

The PR adds a Discord undici-based transport, wires Discord REST/webhook/monitor/gateway proxy paths into explicit and managed proxy config, allows remote Discord proxy URLs, and expands proxy tests plus the changelog.

## Best Possible Solution

Land a Discord-owned transport fix after adding NO_PROXY-aware gateway WebSocket env-proxy resolution while keeping the remote-proxy behavior covered by focused tests.

## Reproduction Assessment

Yes. Current main has a source-level reproduction for the original bug: a remote channels.discord.proxy URL is rejected by validateDiscordProxyUrl, and managed proxy env is not used when no Discord-specific proxy is configured. The PR review finding is also source-reproducible with HTTPS_PROXY plus NO_PROXY=gateway.discord.gg, because the WebSocket resolver returns the env proxy URL without consulting matchesNoProxy.

## Solution Assessment

No, not quite. The Discord-local transport approach is the right shape, but the gateway WebSocket env-proxy branch needs to preserve standard NO_PROXY semantics before this is the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Respect NO_PROXY for gateway WebSocket proxies:** `extensions/discord/src/fetch.ts:76-77`
  - body: The new WebSocket resolver returns an env proxy URL whenever HTTPS_PROXY/HTTP_PROXY is set, but that path constructs HttpsProxyAgent directly, so NO_PROXY/no_proxy is never consulted. With HTTPS_PROXY plus NO_PROXY=gateway.discord.gg, Discord REST calls can bypass through undici's EnvHttpProxyAgent while the gateway socket is still forced through a proxy the operator excluded. Gate the env-proxy branch with shouldUseEnvHttpProxyForUrl("https://gateway.discord.gg", env) or an equivalent no-proxy check before falling back to managed OPENCLAW proxy handling.
  - confidence: 0.88
