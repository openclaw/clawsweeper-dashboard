## Summary

Keep open. Current main and the latest release still expose raw provider strings in Control UI Usage/Sessions display, query, CSV, and aggregate paths, while the current provider-alias contract is not settled enough to queue another blind fix for the requested bailian-to-zai display semantics.

## What This Changes

The issue asks for Control UI provider labels to display canonical normalized provider ids instead of raw configured aliases such as bailian, z-ai, or aws-bedrock.

## Best Possible Solution

Define the supported provider display canonicalization contract, then apply it to derived Control UI Usage/Sessions display, query, CSV, and aggregate surfaces while preserving raw stored config/session fields.

## Reproduction Assessment

Yes. Current-main source inspection shows Usage and Sessions display/query/aggregate paths still derive provider values from raw modelProvider, providerOverride, or entry.provider fields, and the latest release shows the same raw display pattern.

## Solution Assessment

Unclear. Canonicalizing derived display/query/aggregate surfaces is likely the right repair shape, but current main lacks a single supported display-alias normalizer and the bailian alias needs maintainer approval.
