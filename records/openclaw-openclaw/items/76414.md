Keep open: current main still rewrites local Codex app-server shells to an isolated HOME without also pinning the active OpenClaw config path, so CLI commands run from that sandbox can load defaults and send Signal traffic to port 8080.

## Best Possible Solution

Propagate the resolved active OpenClaw config/state identity into local Codex app-server child environments before HOME is rewritten, while preserving explicit overrides and avoiding symlink-based config writes.
