## Summary

Keep open: current main still rewrites the Codex app-server `HOME` without guaranteeing the OpenClaw CLI sees the gateway's active config path, so sandboxed CLI commands can fall back to defaults and target Signal port `8080`. The reporter logs plus current source make this a narrow repair candidate.

## What This Changes

Ensure CLI commands run from a Codex/OpenClaw agent sandbox use the host OpenClaw config instead of sandbox-home defaults, so Signal uses the configured port.

## Best Possible Solution

Propagate the resolved host OpenClaw config/state identity into local Codex app-server child environments before `HOME` is rewritten, with explicit user overrides taking precedence.

## Reproduction Assessment

Yes, source-reproducible with high confidence. A local Codex app-server stdio launch that lacks explicit `OPENCLAW_CONFIG_PATH` rewrites `HOME` to the agent Codex home, after which config resolution derives a missing sandbox config and Signal falls back to `8080`.

## Solution Assessment

Yes. Fixing the Codex app-server environment bridge is narrower than changing global config fallback behavior, changing Signal defaults, or relying on symlinks in the isolated home.
