## Summary

Keep open: current main still isolates the Codex app-server `HOME`/`CODEX_HOME` without carrying the active OpenClaw config path, so CLI commands inside that shell can resolve an empty sandbox config and Signal then defaults to port 8080.

## What This Changes

The issue asks for OpenClaw CLI commands run inside an agent sandbox to use the host/canonical OpenClaw config instead of falling back to sandbox-local defaults.

## Best Possible Solution

Propagate the active host OpenClaw config path into Codex app-server/agent-shell environments when OpenClaw isolates `HOME`, while keeping Codex native state isolation and Signal account defaults intact.

## Reproduction Assessment

Yes from source inspection. A Codex app-server shell gets sandbox `HOME`, config resolution then looks under that home when `OPENCLAW_CONFIG_PATH` is absent, missing config loads as `{}`, and Signal falls back to `127.0.0.1:8080`; the issue's live steps on v2026.5.2 match that path.

## Solution Assessment

Yes. The narrowest maintainable fix is to preserve or inject the active `OPENCLAW_CONFIG_PATH` at the Codex app-server env bridge; changing Signal defaults or global config lookup would be broader and would not solve other sandboxed CLI config drift.
