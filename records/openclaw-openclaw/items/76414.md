## Summary

Keep open: current main still rewrites the local Codex app-server child `HOME`/`CODEX_HOME` without also pinning the active OpenClaw config path, so CLI commands inside that sandbox can resolve a missing sandbox config and fall back to Signal's default `8080` port.

## What This Changes

Request that CLI commands run from a Codex/OpenClaw agent sandbox use the host OpenClaw config instead of sandbox-home defaults, preventing Signal from falling back to port 8080.

## Best Possible Solution

Propagate the resolved active OpenClaw config/state identity into local Codex app-server spawned environments while keeping sandbox `HOME` isolation and explicit overrides intact.

## Reproduction Assessment

Yes, source-backed. In a stdio Codex app-server launch without `OPENCLAW_CONFIG_PATH`, the bridge rewrites `HOME` to the isolated Codex home; config resolution then targets that missing sandbox config and `loadConfig()` returns defaults.

## Solution Assessment

Yes. Setting or inheriting the active OpenClaw config/state env at the Codex app-server spawn bridge is narrower than changing global config fallback behavior, changing Signal defaults, or relying on per-agent symlinks.
