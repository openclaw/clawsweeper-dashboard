## Summary

Keep open: current main still isolates the Codex app-server `HOME` under the agent directory without also pinning the OpenClaw config path, while the CLI config resolver falls back to the isolated home and Signal defaults to `127.0.0.1:8080` when the configured `httpPort` is not loaded.

## What This Changes

The issue asks for OpenClaw CLI commands run inside an agent/Codex sandbox to resolve the host OpenClaw config instead of falling back to sandbox-local defaults.

## Best Possible Solution

Propagate the active OpenClaw config identity into the Codex/app-server agent environment, preferably by setting `OPENCLAW_CONFIG_PATH` to the resolved host config path while preserving Codex auth isolation.

## Reproduction Assessment

Yes from source, not from a live command: an isolated Codex app-server `HOME` without `OPENCLAW_CONFIG_PATH` makes the CLI resolver use the sandbox default config path, and Signal then resolves its default `8080` port.

## Solution Assessment

Yes: the narrow maintainable fix is to propagate the already-supported `OPENCLAW_CONFIG_PATH` into the agent/Codex environment rather than changing Signal defaults or adding another config knob.
