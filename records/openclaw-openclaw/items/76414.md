## Summary

Keep open: current main still lets the local Codex app-server sandbox rewrite `HOME` without carrying the active OpenClaw config path, so an in-sandbox `openclaw` CLI can load an empty sandbox-local config and then use Signal's documented `127.0.0.1:8080` default. The affected path is narrow enough for a focused env-bridging fix with regression coverage.

## What This Changes

The issue asks OpenClaw CLI commands run inside a Codex/OpenClaw agent sandbox to use the same canonical config as the gateway instead of HOME-local defaults.

## Best Possible Solution

Codex local app-server launches should carry the active OpenClaw config file path into the isolated environment so in-sandbox OpenClaw CLI commands read the gateway's config while keeping personal Codex state isolated.

## Reproduction Assessment

Yes, from source inspection: with `HOME` set to the agent `codex-home/home` and no `OPENCLAW_CONFIG_PATH`, current main resolves the config under that sandbox home, returns `{}` when it is missing, and then Signal uses `127.0.0.1:8080`. I did not run a live Codex/Signal send in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to bridge the active `OPENCLAW_CONFIG_PATH` into the local Codex app-server sandbox env while preserving explicit overrides. Changing Signal defaults or the global home resolver would be broader and would not address the Codex-specific split cleanly.

## Repair Work Prompt

Fix openclaw/openclaw#76414. Current main sets local Codex app-server `CODEX_HOME` and `HOME` to per-agent `codex-home` directories, but does not ensure `OPENCLAW_CONFIG_PATH` points at the active OpenClaw config. As a result, `openclaw message send --channel signal` run inside the agent shell can load missing sandbox-local config and fall back to Signal port 8080. Implement a narrow env bridge for local Codex app-server launches: when no explicit `OPENCLAW_CONFIG_PATH` is supplied, set it to the active config file path resolved before `HOME` is overwritten. Preserve explicit `CODEX_HOME`, `HOME`, `OPENCLAW_CONFIG_PATH`, and `clearEnv` semantics. Do not change Signal's default port, global config path resolution, or personal Codex state isolation. Prefer existing plugin SDK/public seams; if a tiny generic seam is needed, keep it additive. Add regression coverage around `bridgeCodexAppServerStartOptions`, including preservation of explicit env overrides. Add a single-line `CHANGELOG.md` fix entry because this is user-facing.
