## Summary

Keep open. The PR addresses a real Codex plugin failover gap on current main, but the latest head still has a small fallback bug where a whitespace-only upstream message can become an empty prompt error instead of the non-empty default.

## What This Changes

The PR adds a Codex app-server error projection helper, stores account rate-limit snapshots, preserves structured Codex prompt errors through timeout finalization, expands tests, and adds a changelog entry.

## Best Possible Solution

Land the Codex plugin-owned fix after ignoring empty trimmed fallback messages, adding a focused regression test, and waiting for exact-head checks to finish green.

## Reproduction Assessment

Yes. A source-level reproduction is to send `account/rateLimits/updated`, then a terminal Codex `error` with `usageLimitExceeded`, and then let the watchdog fire; current main filters the snapshot and overwrites the prompt error with a timeout. The remaining PR defect reproduces with a whitespace-only Codex error message and no projected detail.

## Solution Assessment

Yes, with one small correction. Keeping the projection in the Codex plugin and reusing the existing failover/rate-limit formatter is the narrowest maintainable path; the fallback expression should require a non-empty trimmed message.
