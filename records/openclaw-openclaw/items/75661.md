## Summary

Keep open. The PR addresses a real Codex plugin gap on current main, but the current head still has one narrow fallback bug where a whitespace-only Codex error message can become an empty prompt error instead of using the non-empty fallback.

## What This Changes

The PR adds Codex app-server error projection, captures account rate-limit snapshots, preserves structured prompt errors through timeout handling, adds focused tests, and updates the changelog.

## Best Possible Solution

Land the Codex plugin-owned fix after treating empty trimmed messages as absent and letting the exact-head Codex app-server checks finish green.

## Reproduction Assessment

Yes. A source-level reproduction is to send `account/rateLimits/updated`, then a terminal Codex `error` with `usageLimitExceeded`, and then let the watchdog fire; current main ignores the snapshot and overwrites the prompt error with a timeout.

## Solution Assessment

Yes, with a minor correction. Keeping the projection in the Codex plugin and reusing the existing failover/rate-limit formatter is the narrowest maintainable path; the fallback expression should reject empty trimmed messages.
