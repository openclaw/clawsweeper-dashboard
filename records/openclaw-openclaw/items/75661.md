## Summary

Keep open. The source-level bug is still present on current main and the PR is the right plugin-owned fix, but the latest head still lets whitespace-only Codex error messages become empty prompt errors, so it needs one narrow fix before merge.

## What This Changes

The branch adds Codex app-server structured error projection, stores rate-limit snapshots, preserves structured prompt errors across timeout finalization, and adds tests plus a changelog entry.

## Best Possible Solution

Treat blank trimmed messages as absent, keep the fix inside the Codex plugin boundary, and land after the exact-head Codex app-server checks are green.

## Reproduction Assessment

Yes. A source-level reproduction is to send `account/rateLimits/updated`, then a terminal Codex `error` with `usageLimitExceeded`, and then let timeout finalization run; current main ignores the snapshot and can relabel the prompt error as a timeout.

## Solution Assessment

Yes, with a small correction. Projecting Codex app-server error metadata in the Codex plugin and reusing the existing failover/rate-limit formatter is the narrowest maintainable approach; the fallback expression should reject empty trimmed messages.
