## Summary

## What This Changes

The PR adds Control UI styling, extraction/controller logic, and tests so live and persisted web chat model failures render as inline assistant error bubbles.

## Best Possible Solution

Land a rebased focused Control UI fix, or a canonical replacement, that renders live and persisted assistant error failures inline while preserving shared assistant extraction, `formatConnectError`, current session/attachment behavior, and adding after-fix real behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: live `state: "error"` events only update `lastError`, and persisted assistant error-only messages with empty content currently have no assistant text through shared extraction.

## Solution Assessment

No. The direction is right, but this branch needs a rebase and two focused fixes before it is the best implementation: route the assistant error fallback through the active assistant extraction path and preserve structured `formatConnectError` formatting for send failures.
