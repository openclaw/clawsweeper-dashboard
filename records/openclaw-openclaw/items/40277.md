## Summary

Keep open: current main still lacks inline Control UI chat error bubbles, and this PR is useful but currently incorrect because assistant error-only history still bypasses the new fallback and chat send failures lose structured connect-error formatting.

## What This Changes

The PR updates Control UI chat extraction, controller error handling, grouped rendering/CSS, and tests so model errors appear as inline highlighted assistant bubbles.

## Best Possible Solution

Land a focused Control UI fix that wires assistant error fallback into assistant extraction, appends one structured live error bubble, preserves `formatConnectError`, renders empty-detail error labels, adds regression tests, and records the fix in the changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a live `state: "error"` chat event only updates `lastError`, and an assistant history entry with `stopReason: "error"`, `errorMessage`, and `content: []` produces no visible assistant text before grouped rendering suppresses it.

## Solution Assessment

No. The PR direction is appropriate, but the current patch is not the best implementation because the fallback is attached to `extractRawText`, which assistant messages bypass, and the send-failure rewrite drops structured `formatConnectError` handling.
