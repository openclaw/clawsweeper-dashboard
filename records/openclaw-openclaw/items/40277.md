## Summary

Keep open: current main still drops or hides the relevant model-error surfaces, and this PR is directionally useful, but the branch has blocking defects that would leave refresh visibility broken and would regress structured Control UI connection-error copy.

## What This Changes

The PR changes Control UI chat extraction, live error handling, grouped chat rendering/CSS, and UI tests so model failures appear as inline highlighted assistant bubbles.

## Best Possible Solution

Land a focused Control UI fix that routes assistant error fallback through assistant extraction, preserves `formatConnectError`, keeps assistant error-only history out of heartbeat filtering, renders empty-detail error labels, adds regression tests, and records the fix in the changelog.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: live `state: "error"` events only update `lastError`, and persisted assistant error entries with `content: []` have no visible text and are also caught by empty-content history filtering.

## Solution Assessment

No. The PR direction is the right narrow product fix, but the current patch is not the best implementation because it places the fallback on a path assistant messages bypass, misses the history filter exception, and replaces structured connection-error formatting.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Route assistant error fallback through assistant extraction:** `ui/src/ui/chat/message-extract.ts:123`
  - body: The new fallback is returned from `extractRawText`, but `extractText` routes assistant messages through `extractSharedAssistantVisibleText(message)` instead. An error-only assistant history entry with `content: []` still extracts as `null`, so the refresh-visible error bubble path remains broken until the fallback is applied in the assistant branch.
  - confidence: 0.95
- **[P2] Exclude assistant errors from heartbeat filtering:** `ui/src/ui/controllers/chat.ts:225-231`
  - body: The added history case uses an assistant message with `content: []`, but `loadChatHistory` still runs `shouldHideHistoryMessage`, and empty assistant content is classified as a heartbeat ack. Add an explicit assistant-error exception before the heartbeat filter so persisted error-only runs are not dropped before rendering.
  - confidence: 0.92
- **[P2] Preserve structured connect-error formatting:** `ui/src/ui/controllers/chat.ts:263`
  - body: Replacing `formatConnectError(err)` with generic normalization drops the detail-code mapping for `GatewayRequestError`. For cases like `CONTROL_UI_ORIGIN_NOT_ALLOWED`, current behavior and tests expect actionable origin guidance in both `lastError` and the assistant error text, not generic `Fetch failed` copy.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `CHANGELOG.md:30`
  - body: This is a user-facing Control UI fix, but the branch does not update `CHANGELOG.md`. Add a single-line Fixes entry under the active unreleased section so release notes capture the visible chat behavior change.
  - confidence: 0.86
