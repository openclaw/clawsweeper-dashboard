## Summary

Keep this PR open: current main still has the WebChat inline-error visibility gap, but the submitted patch is not merge-ready because two functional regressions and the required changelog entry remain.

## What This Changes

The PR updates Control UI chat extraction, controller error handling, grouped rendering/CSS, and tests so model errors appear as highlighted inline assistant bubbles from history and live error events.

## Best Possible Solution

Land a focused Control UI fix that routes assistant error fallback through assistant extraction, appends one structured live error bubble while preserving `formatConnectError`, renders empty-detail error labels, adds regression tests, and records the user-facing fix in `CHANGELOG.md`.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: live `state: "error"` events only update `lastError`, and assistant error history with `content: []` has no visible text before the grouped renderer suppresses the empty bubble.

## Solution Assessment

No. The PR direction is right, but the current patch is not the best implementation because the fallback is attached to the raw extractor that assistant messages bypass, and the send-failure rewrite drops structured `formatConnectError` behavior.
