## Summary

Keep open. Current main still lacks inline Control UI model-error bubbles and persisted error-only assistant history rendering, so this PR is not obsolete; however, the proposed patch still has blocking correctness issues around assistant extraction and structured connect-error formatting.

## What This Changes

The PR converts Control UI chat send/live failures into assistant error messages, extracts assistant `errorMessage` text for rendering, adds error-bubble styling, and covers the behavior with UI tests.

## Best Possible Solution

Land a focused Control UI fix that preserves `formatConnectError`, appends structured assistant error messages for send/live failures, routes persisted assistant error-only history through the assistant extraction path, renders grouped error bubbles for detailed and empty-detail cases, and adds the required changelog entry.

## Reproduction Assessment

Yes. Current-main source inspection gives a high-confidence reproduction path: a live `state: "error"` chat event only updates `lastError`, and an assistant history entry with `stopReason: "error"`, `errorMessage`, and empty `content` still has no visible assistant text through the current extraction path.

## Solution Assessment

No. The PR's direction is appropriate, but this patch is not yet the best implementation because the fallback is wired below the assistant extraction branch and the send-failure change drops existing structured Gateway error formatting.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

  - body: The fallback is added at the end of `extractRawText`, but assistant messages are routed through `extractSharedAssistantVisibleText` in `extractText`.   - confidence: 0.94

  - confidence: 0.9
