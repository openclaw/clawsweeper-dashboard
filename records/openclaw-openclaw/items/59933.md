## Summary

Keep open. Current `main` still filters `sessions_list` through `tree` visibility before kind filtering, so separate same-agent group/channel sessions remain hidden unless `tools.sessions.visibility` is widened.

## What This Changes

The issue asks `sessions_list` to show active same-agent Telegram group and Discord channel sessions, or provide a filter/mode that can reveal them without manual transcript inspection.

## Best Possible Solution

Define the intended session-tool diagnostic contract, then add focused regression coverage and either expose same-agent group/channel sessions through a safe explicit mode/filter or make hidden-by-visibility results discoverable without widening defaults.

## Reproduction Assessment

Yes, at source level. With default `tree` visibility, a requester session and a separate same-agent Telegram group or Discord channel session will be enumerated by the gateway but rejected by `createSessionVisibilityGuard` unless the target is the requester or a spawned child.

## Solution Assessment

Unclear. The opt-in `agent` visibility is a valid workaround, but widening defaults or adding a special group diagnostic path changes privacy semantics and needs maintainer approval first.
