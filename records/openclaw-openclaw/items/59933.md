## Summary

Keep open. Current main still hides separate same-agent group/channel sessions under the default `tree` visibility before `kinds` filtering runs, and changing that behavior needs an explicit session-visibility product/security decision rather than a cleanup close.

## What This Changes

Expose same-agent Telegram/Discord group or channel sessions through `sessions_list`, or provide a filter or mode that makes those hidden sessions discoverable.

## Best Possible Solution

Define the session-tool diagnostic contract, then add focused coverage and either expose same-agent group/channel sessions through a safe explicit mode/filter or return hidden-by-visibility metadata without weakening default privacy.

## Reproduction Assessment

Yes, at source level. With default `tree` visibility, a requester session and a separate same-agent Telegram group or Discord channel session are enumerated by the gateway but rejected unless the target is the requester or a spawned/owned child.

## Solution Assessment

Unclear. The existing `tools.sessions.visibility="agent"` setting is a workaround, but changing defaults or making `kinds` bypass visibility would alter privacy semantics and needs maintainer approval first.
