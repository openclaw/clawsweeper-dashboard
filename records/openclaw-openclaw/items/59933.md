## Summary

Keep open. Current main still filters `sessions_list` rows through the session-tool visibility guard before group-kind filtering, so separate same-agent group/channel rows remain hidden under the default `tree` scope; changing that default or adding a diagnostic escape hatch needs maintainer product and security judgment.

## What This Changes

The issue asks for `sessions_list` to surface same-agent Telegram group or Discord channel sessions that exist in the session store and CLI enumeration but are hidden from tool output.

## Best Possible Solution

Define the session-tool diagnostic visibility contract first, then add focused coverage and either expose same-agent group/channel sessions through an explicit safe mode or return hidden-by-visibility diagnostics without weakening default privacy unexpectedly.

## Reproduction Assessment

Yes, at source level. With default `tree` visibility, `sessions_list` asks the gateway for rows but rejects a separate same-agent group/channel row before the `group` kind filter can include it.

## Solution Assessment

Unclear. The documented `tools.sessions.visibility="agent"` setting is an explicit broader scope, but changing the default or adding diagnostic behavior needs maintainer approval because it changes privacy semantics.
