## Summary

Keep open. Current main and the latest release still filter `sessions_list` rows through the session-tool visibility guard before `kinds` filtering, so same-agent group/channel rows outside the current tree remain hidden unless the operator broadens session visibility; changing the default or adding a diagnostic escape hatch needs maintainer product and security direction.

## What This Changes

The issue asks for `sessions_list` to surface same-agent Telegram group sessions, or at least provide a filter/diagnostic path so active group sessions are not mistaken for missing.

## Best Possible Solution

Define the session-tool diagnostic visibility contract first, then add focused coverage and either expose same-agent group/channel sessions through an explicit safe mode or return hidden-by-visibility diagnostics without weakening default privacy.

## Reproduction Assessment

Yes, at source level. With default `tree` visibility, `sessions_list` receives the gateway rows but rejects a separate same-agent group/channel row before the `group` kind filter can include it.

## Solution Assessment

Unclear. The existing `tools.sessions.visibility="agent"` setting is a documented broader scope, but changing the default or adding diagnostic behavior needs maintainer approval because it changes privacy semantics.
