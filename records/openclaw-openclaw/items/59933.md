## Summary

Keep open. Current main still hides separate same-agent group/channel rows under the default `tree` session-tool visibility, and deciding whether to change the default, add diagnostics, or rely on `tools.sessions.visibility="agent"` is a privacy-sensitive product decision.

## What This Changes

The issue requests that `sessions_list` surface active same-agent group/channel sessions, or provide a filter or diagnostic path that makes them discoverable.

## Best Possible Solution

Define the session-tool visibility and diagnostic contract, preserve default privacy, then add either an explicit safe discovery mode or hidden-by-visibility diagnostics with focused coverage.

## Reproduction Assessment

Yes, at source level. With default `tree` visibility, a requester session and a separate same-agent Telegram group or Discord channel row can be listed by the gateway but rejected by the tool-layer visibility checker unless it is the requester or owner/spawned/parent-owned.

## Solution Assessment

Unclear. The documented `tools.sessions.visibility="agent"` setting is a workaround, but changing the default or making `kinds:["group"]` bypass visibility would alter privacy semantics and needs maintainer approval.
