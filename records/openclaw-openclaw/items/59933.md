## Summary

Keep open. Current main still hides separate same-agent group/channel sessions under the default `tree` session-tool visibility before `kinds` filtering runs, and the remaining fix is a product/privacy decision rather than a cleanup close.

## What This Changes

The issue requests that `sessions_list` surface same-agent Telegram group or Discord channel sessions that exist in the session store/transcripts, or provide an explicit group diagnostic/filter path.

## Best Possible Solution

Define the session-tool diagnostic contract explicitly, then add focused tests and either expose same-agent group/channel sessions through an intentional mode or make hidden-by-visibility results discoverable without silently expanding access.

## Reproduction Assessment

Yes, at source level. With default `tree` visibility, a requester session and a separate same-agent Telegram group or Discord channel session will be enumerated from the store but rejected by `createSessionVisibilityRowChecker` before the `kinds` filter can surface it.

## Solution Assessment

Unclear. The existing `tools.sessions.visibility="agent"` knob is a workaround, but changing defaults or adding a special group diagnostic path needs maintainer product/privacy approval first.
