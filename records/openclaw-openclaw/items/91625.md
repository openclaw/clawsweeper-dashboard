## Summary

Keep this PR open: the patch is still necessary because current main supports `cron.update` with `payload.model: null`, but `openclaw cron edit` still cannot emit that patch shape from the CLI. The proposed flag is narrow, additive, documented, and backed by real live-gateway output plus focused tests.

## What This Changes

The PR adds `openclaw cron edit --clear-model`, maps it to `payload.model = null`, rejects it with `--model`, and documents/tests the new CLI path.

## Best Possible Solution

Land the additive CLI flag, docs, and regression coverage so users can clear cron model overrides through the existing `model:null` update contract.

## Reproduction Assessment

Yes. Source inspection shows current main can clear `payload.model` through `cron.update`, but the CLI edit command lacks a `--clear-model` option and only serializes non-empty `--model` strings; I did not execute a live repro in this read-only review.

## Solution Assessment

Yes. Mapping an additive CLI flag to the already-landed `model:null` patch contract is the narrowest maintainable fix; a new backend clear shape or manual state editing would duplicate the existing contract.
