## What This Changes

The PR adds a `cron edit --clear-model` CLI flag, regression coverage, and cron documentation for clearing a stored per-job model override.

## Best Possible Solution

Land the additive CLI flag after the public wording says it removes the per-job override and returns the job to normal cron model precedence.

## Reproduction Assessment

Yes, at source level. Current main has no CLI path to produce `payload.model = null`, while current main gateway schema and service code already accept that patch shape and delete the stored model override.

## Solution Assessment

Yes, with one wording cleanup. Mirroring the existing `--clear-tools` CLI pattern is the narrow owner-boundary fix; the public text should describe returning to normal cron model precedence rather than always agent/default selection.
