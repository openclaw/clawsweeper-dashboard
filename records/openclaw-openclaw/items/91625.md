## Summary

Keep open: this is a focused and useful cron CLI bug fix with sufficient live behavior proof, but the new public `cron edit` flag should be documented before merge.

## What This Changes

The PR adds `openclaw cron edit --clear-model`, rejects it with `--model`, sends `payload.model = null`, and adds cron edit unit/help coverage.

## Best Possible Solution

Land the CLI flag with a small docs update that documents `openclaw cron edit <jobId> --clear-model` next to the existing model override and API null-clear behavior.

## Reproduction Assessment

Yes. Current main has no `--clear-model` option and only sends `payload.model` when `--model` normalizes to a non-empty string, while the gateway update path already accepts `model: null`.

## Solution Assessment

Yes, with one docs follow-up. The PR fixes the narrow CLI owner boundary by mirroring `--clear-tools` and relying on the existing gateway null-clear contract rather than changing cron storage or model resolution.
