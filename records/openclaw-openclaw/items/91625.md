## Summary

This PR remains necessary and is ready for normal maintainer landing: current `main` supports the nullable cron update needed to delete a job model override but still lacks a CLI route to send it, while the final branch adds the flag, conflict guard, focused coverage, accurate docs, and convincing live gateway proof without introducing a competing behavior path.

## What This Changes

Adds `cron edit --clear-model` to emit the existing `payload.model: null` update signal, reject conflicting model flags, and document the resulting cron model-selection precedence.

## Best Possible Solution

Land the additive CLI flag with its focused tests and revised precedence wording, leaving nullable clearing for other cron payload fields to separate API work such as https://github.com/openclaw/openclaw/pull/67429.

## Reproduction Assessment

Yes. On current `main`, the gateway accepts `payload.model: null` but `cron edit --clear-model` is absent, so the CLI rejects the intended user path; the PR also supplies live before/after gateway output.

## Solution Assessment

Yes. An additive clear flag mirrors the established cron edit clear-option pattern and uses the canonical nullable update contract, which is narrower and more maintainable than direct state editing or a second clearing API.
