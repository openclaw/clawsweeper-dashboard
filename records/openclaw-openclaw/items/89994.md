## Summary

Keep this open: current main and the latest release still contain the reported whole-file fuzzy-normalization write/diff path, and the linked closing fix PR is open rather than merged.

## What This Changes

The issue reports that fuzzy edit fallback can normalize an entire file and silently rewrite unrelated lines while showing a diff that only includes the intended edit.

## Best Possible Solution

Land or replace the linked PR’s scoped fuzzy-offset fix with regression coverage so writes and diffs stay based on the original LF-normalized file while fuzzy normalization is used only for match discovery.

## Reproduction Assessment

Yes. Source inspection on current main shows that any fuzzy match can switch baseContent to normalizeForFuzzyMatch(normalizedContent), and edit.ts writes and diffs from that base; I did not run the supplied Vitest because this review is read-only.

## Solution Assessment

Yes. The maintainable fix is to keep the original LF-normalized content as the write/diff base and scope fuzzy normalization to match discovery and offset mapping, which the linked PR already proposes.
