## Summary

Keep this open: current main still contains the whole-file fuzzy-normalization write/diff path, and the linked closing PR is still open rather than merged.

## What This Changes

The issue reports that fuzzy edit matching normalizes the whole file and silently rewrites unrelated lines while hiding those changes from the model-facing diff.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/90060, or replace it with an equivalent fix that uses fuzzy normalization only for match discovery while preserving untouched file bytes and honest diffs.

## Reproduction Assessment

Yes from source inspection: a fuzzy match makes `baseContent` the whole-file `normalizeForFuzzyMatch` output, and the edit tool writes and diffs from that returned base. I did not run the supplied Vitest because this review is read-only.

## Solution Assessment

Yes, the right fix shape is to keep the original LF-normalized content as the write/diff base and use fuzzy normalization only for locating match offsets. The open linked PR already owns that path, so the issue should stay open until that PR or an equivalent fix lands.
