## Summary

Keep open. Current main still rotates expired sessions and only runs automatic memory flush as pre-compaction housekeeping, so the requested session-end continuity nudge or sticky-session behavior remains unresolved and needs maintainer product direction.

## What This Changes

Request a session lifecycle continuity mechanism that prompts or stitches agents so significant in-flight work is durably remembered before sessionId rotation severs context.

## Best Possible Solution

Keep this as a maintainer design item for a narrow continuity mechanism, likely reusing memory-flush machinery and recorded tool-effect metadata before considering sticky sessionId semantics.

## Reproduction Assessment

Not applicable as a strict bug reproduction. Source inspection confirms reset rotation and compaction-only memory flush, but current docs/tests do not promise short-idle session stitching or a session-end nudge.

## Solution Assessment

Unclear pending product decision. A session-end nudge looks narrower than sticky sessionId reuse, but maintainers need to define significant-work triggers, cost limits, and whether the hook belongs before session.ended or on the next session start.
