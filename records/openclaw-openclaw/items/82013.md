## Summary

Current main still implements documented idle/daily sessionId rotation and only runs the automatic memory-flush turn before compaction pressure, so the requested session-end continuity nudge or sticky-session behavior remains new core lifecycle work that needs maintainer product judgment.

## What This Changes

Add a runtime continuity mechanism that nudges or stitches sessions so substantial work is written to durable memory around idle sessionId rotation.

## Best Possible Solution

Keep this as a core lifecycle design item for a narrow continuity mechanism, likely reusing memory-flush machinery and recorded tool metadata without changing default sticky-session semantics until the UX contract is chosen.

## Solution Assessment
