## Summary

Keep open. Current main still rotates expired sessions and only runs the automatic memory-flush turn around compaction pressure, while the existing plugin hooks are adjacent extension points rather than an implemented default session-continuity policy.

## What This Changes

The issue requests a session-end or next-session continuity nudge, plus an optional sticky-session mode, so significant work is written to durable memory before short idle session rotation loses context.

## Best Possible Solution

Keep this as a design item for a narrow continuity mechanism that reuses existing tool/finalize hooks or memory-flush machinery, with explicit significant-work criteria, bounded extra-turn cost, and clear core-versus-plugin ownership.

## Reproduction Assessment

Not applicable as a strict bug reproduction. Source and docs confirm current main does not promise short-idle session stitching or a significant-work memory nudge; the reporter's observed behavior is a requested lifecycle feature rather than a broken documented contract.

## Solution Assessment

Unclear pending maintainer decision. A ClawHub plugin could likely use `before_tool_call`/`after_tool_call` plus `before_agent_finalize`, while a default core nudge or sticky-session mode would change session lifecycle UX and needs explicit policy choices.
