## Summary

Keep open. Current main still rotates expired sessions by reset policy and only runs the automatic memory flush as pre-compaction maintenance, so the requested session-end continuity nudge, startup continuity prelude, or optional stitching mode is not implemented and needs a maintainer product choice.

## What This Changes

Request a core session-continuity mechanism that reminds agents to persist durable memory before or after sessionId rotation when significant work occurred.

## Best Possible Solution

Design a core-owned continuity guard around session rollover, preferably a conservative session-end or next-start memory nudge tied to existing memory-flush plumbing before considering a new sticky-session configuration mode.

## Reproduction Assessment

No live high-confidence repro was run, but the source path is clear: idle or daily freshness can rotate the sessionId, while memory flush only runs near compaction thresholds and session.ended has no continuity nudge.

## Solution Assessment

Unclear. The session-end nudge is likely the narrowest first solution, while sticky sessionId reuse would add a compatibility-sensitive config surface and should wait for maintainer direction.
