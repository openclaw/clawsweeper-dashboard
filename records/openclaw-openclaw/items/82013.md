## Summary

Keep open. Current main still rotates stale idle/daily sessions and only runs the automatic memory-flush turn as pre-compaction housekeeping, so the requested session-end continuity nudge or optional stitching behavior is unresolved and needs maintainer product direction.

Request to add a session-continuity nudge or optional stitching behavior so agents flush durable memory before or around short-idle sessionId rotation after significant work.

## Best Possible Solution

Keep this as a design item for a narrow continuity mechanism, likely reusing memory-flush machinery plus recorded tool-effect metadata while avoiding sticky sessionId semantics until maintainers define the UX and config contract.

## Solution Assessment
