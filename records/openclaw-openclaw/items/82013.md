## Summary

Keep open. Current main still rotates stale sessions to a new sessionId and the automatic memory flush remains compaction-gated, so the requested session-end continuity nudge, startup prelude, or sticky-session behavior is not implemented and still needs maintainer product direction.

## What This Changes

Request a runtime continuity guard so agents are nudged to persist memory when significant work happened before a sessionId rollover.

## Best Possible Solution

Design a core-owned continuity guard around session rollover, likely starting with a conservative session-end or next-start memory nudge before considering sticky-session configuration.

## Reproduction Assessment

No live high-confidence reproduction was run. The source path is reproducible from current main: idle or daily reset can rotate the sessionId, while memory flush only runs for compaction pressure and session.ended has no continuity nudge.

## Solution Assessment

Unclear. A session-end nudge is the narrowest plausible direction, while startup preludes and sticky sessionId reuse need explicit maintainer choices about trigger policy, compatibility, and config surface.
