## Summary

Keep open: current main still handles WebChat code-block Copy with a direct Clipboard API call and an empty rejection path, so the reported Edge failure remains a no-op. The related fallback PRs were all closed unmerged, leaving a narrow Control UI bug to fix.

## What This Changes

The issue reports that WebChat code-block Copy buttons do not copy fenced code content in Microsoft Edge on Windows 11.

## Best Possible Solution

Add a small shared Control UI clipboard helper with Clipboard API first and textarea/`execCommand('copy')` fallback, then wire both WebChat code-block Copy and adjacent chat copy buttons through it with regression coverage.

## Reproduction Assessment

Yes, source-level. On current main, a rejected `navigator.clipboard.writeText` call in the `.code-block-copy` handler is swallowed by an empty rejection callback, matching the reported Edge no-op failure mode.

## Solution Assessment

Yes. A narrow shared clipboard helper is the maintainable fix because the issue is browser copy behavior in the Control UI, not markdown parsing, gateway protocol, or a product setting.
