## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/51664. WebChat code-block Copy can silently fail in Edge because `ui/src/ui/views/chat.ts` calls `navigator.clipboard.writeText(code)` directly and ignores rejection. Add or reuse a small Control UI clipboard helper, likely under `ui/src/ui/chat/clipboard.ts` or adjacent chat copy code, that tries `navigator.clipboard.writeText` first and falls back to a hidden readonly textarea plus `document.execCommand('copy')` with cleanup. Wire `.code-block-copy` through the helper and only show copied feedback on success. Add focused regression coverage for rejected Clipboard API fallback while preserving existing markdown rendering and i18n behavior. Do not change gateway protocol, markdown parsing semantics, or add a config option. Add a user-facing changelog fix entry if the repair branch owns release notes. Validate locally with focused UI tests and formatter checks, then include a manual Microsoft Edge on Windows smoke if available: receive a fenced code block, click Copy, paste into a text field, and verify exact copied content.
