## What This Changes

The PR removes Discord early accepted typing cue's DM-only gate so instant-mode guild/channel messages call sendTyping before queued processing, and updates the queue test to expect that ordering.

## Best Possible Solution

Land a narrow Discord fix that sends the accepted early cue for actionable user_request turns, preserves room_event quiet typing suppression with coverage, and includes redacted live Discord proof.

## Reproduction Assessment

Yes for source-level reproduction: current main's accepted typing cue rejects guild messages and the existing queue test asserts no guild early typing. I did not run a live Discord gateway reproduction in this read-only review.

## Solution Assessment

No as submitted: removing the DM-only gate fixes the core guild/channel delay but does not preserve room_event quiet suppression. The safer solution is to send early typing for actionable user_request contexts and add coverage for quiet room_event contexts.
