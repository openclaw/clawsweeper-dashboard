## Summary

Keep open. Current main still does not protect active subagent child sessions from session-store maintenance, and this PR is a valid narrow fix direction, but it still misses persisted-only active runs and the required changelog entry.

## What This Changes

The PR threads protected session keys through pruning, count capping, disk-budget eviction, and adds helper/tests/logging for active subagent sessions.

## Best Possible Solution

Land a rebased fix that uses the current preserveKeys-style maintenance seam, derives active subagent child session keys from the persisted-plus-memory run snapshot, protects prune/cap/disk-budget eviction, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. The product bug has a clear reproduction with session maintenance in enforce mode, tight maxEntries or disk budget, and an active subagent child session older than retained entries; the PR-specific blocker is reproduced by an active run visible only through the persisted registry snapshot.

## Solution Assessment

No, not as written. The direction is maintainable, but the active-key helper must read through getSubagentRunsSnapshotForRead(subagentRuns), and the patch should be rebased onto current main's preserveKeys API with changelog coverage.
