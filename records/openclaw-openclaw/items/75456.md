## Summary

Keep open. Current main still has the reported audit-boundary behavior in source: routine security audit can leave plugin security collector discovery enabled, collector fallback can still reach metadata snapshot loading without a bundled runtime-deps opt-out, and installed-plugin audit helpers still accept hidden extension directories. Open PR #75613 is a relevant implementation candidate for the plain-audit runtime-loading half, but it has not merged and the hidden backup-directory scan boundary is not fully covered by current main.

## What This Changes

Request that `openclaw security audit --json` avoid staging plugin runtime dependencies during collector discovery and ignore hidden/backup extension directories during installed-plugin security scans.

## Best Possible Solution

Keep this issue open until routine JSON audit stays on a read-only metadata/filesystem path by default and installed-plugin audit scanners reject hidden, backup, and non-plugin directories. The clean end state is either #75613 expanded to cover both boundaries or #75613 merged for the runtime-loading half with a narrow follow-up for the audit directory filters.

## Reproduction Assessment

Yes for a high-confidence source-level reproduction: current main traces from the CLI into collector loading by default, then into metadata snapshot loading without a runtime-deps opt-out, and both installed-plugin directory helpers still accept hidden names. The live timeout/orphan-process symptom was not reproduced because this review was read-only.

## Solution Assessment

Yes, the issueâ€™s narrow direction is the best maintainable fix: keep plain audit off plugin runtimes, reserve collector imports for `--deep` or explicit internal callers, and filter hidden/backup/non-plugin directories in the audit scan helpers or a shared discovery boundary.
