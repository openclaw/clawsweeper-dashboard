## Summary

Keep open. Current main resolves the plain audit plugin-runtime collector/staging half, but both audit-side installed-plugin directory enumerators still accept every direct directory under `stateDir/extensions`, including hidden backup directories.

## What This Changes

The issue reports that `openclaw security audit --json` can hang because routine audit collector discovery stages plugin runtime dependencies and audit scanners treat hidden extension backup directories as installed plugins.

## Best Possible Solution

Keep the runtime collector changes, then add a shared audit-side installed-plugin directory filter with regression coverage for dotdirs, `.openclaw-install-backups`, `node_modules`, backup, and disabled directory names.

## Reproduction Assessment

Yes. A `stateDir/extensions/.openclaw-install-backups` directory would still be returned by both audit `listInstalledPluginDirs` helpers and then scanned or treated as an installed plugin candidate on current main.

## Solution Assessment

Yes for the remaining work. Filtering at the audit installed-plugin directory boundary is narrower and safer than reworking plugin loading or changing the already-fixed routine audit collector path.

Fix the remaining scanner-boundary bug described at https://github.com/openclaw/openclaw/issues/75456. Current main already keeps plain `security audit` off plugin runtime collector discovery; do not rework that path. Add a shared audit-side installed-plugin directory filter, or equivalent small helper, used by `src/security/audit-extra.async.ts` and `src/security/audit-plugins-trust.ts` so audit enumeration ignores hidden direct children, `.openclaw-install-backups`, `node_modules`, `.bak`, `.backup-*`, and `.disabled*` style backup/disabled names while preserving real installed plugin directories. Match the existing discovery precedent in `src/plugins/discovery.ts` where appropriate. Add regression tests in `src/security/audit-extra.async.test.ts` and `src/security/audit-plugins-trust.test.ts` proving ignored dirs are skipped and a normal plugin dir is still scanned/audited. Add a single-line `CHANGELOG.md` fix entry if the change is user-facing.
