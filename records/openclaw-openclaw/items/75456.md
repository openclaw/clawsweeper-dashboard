## Summary

Keep open. Current main resolves the plain `security audit` plugin-runtime collector path, but both audit installed-plugin scanners still enumerate hidden backup directories as plugin candidates.

## What This Changes

The issue asks for `openclaw security audit --json` to avoid heavy plugin runtime work and to ignore hidden or backup directories under `stateDir/extensions` during audit scans.

## Best Possible Solution

Keep the shipped runtime collector behavior, then add a shared audit-side installed-plugin directory filter with regression coverage for dotdirs, `.openclaw-install-backups`, `node_modules`, and backup/disabled entries.

## Reproduction Assessment

Yes. On current main, a `stateDir/extensions/.openclaw-install-backups` directory would be returned by both audit `listInstalledPluginDirs` helpers and then scanned or treated as an installed plugin candidate.

## Solution Assessment

Yes. Filtering at the audit installed-plugin directory boundary is the narrowest maintainable fix and avoids reopening the already-fixed runtime collector path.
