## Summary

Keep open. Current main has the merged fix for the plain `security audit --json` plugin-runtime collector path, but the reported hidden backup-directory boundary still remains in the audit scanners that enumerate installed plugin directories.

## What This Changes

The issue requests that `openclaw security audit --json` avoid plugin runtime dependency staging during routine audits and skip hidden or backup extension directories during plugin audit scans.

## Best Possible Solution

Keep the merged plain-audit runtime collector behavior, then add a narrow audit-side installed-plugin directory filter with regression coverage so hidden, backup, disabled, and `node_modules` entries under `stateDir/extensions` are not treated as installed plugins.

## Reproduction Assessment

Yes for a high-confidence source-level reproduction. Current main still enumerates all direct directories in both audit plugin-dir helpers, so a `.openclaw-install-backups` directory under `stateDir/extensions` remains in scope without needing to run a mutating live audit.

## Solution Assessment

Yes, the narrow remaining fix is the best path: filter audit scanner candidates at the installed-plugin directory boundary and leave the global plugin loader/runtime behavior unchanged.
