## Summary

Keep open. Current main still has the reported audit-boundary behavior in source: the plain security audit CLI does not disable plugin security collector loading, collector discovery can still reach metadata snapshot loading without a bundled runtime-deps opt-out, and the installed-plugin audit scans still accept hidden extension directories. The open #75613 PR is a relevant implementation candidate for the plain-audit runtime-loading part, but it has not merged and does not appear to cover the hidden backup-directory scan boundary.

## What This Changes

The report asks for `openclaw security audit --json` to stay bounded and read-only by avoiding plugin runtime dependency staging during collector discovery and by ignoring hidden or backup directories under the extensions state directory.

## Best Possible Solution

Keep this issue open until the audit path is made read-only for routine JSON audits and the installed-plugin audit scanners reject hidden, backup, and non-plugin directories. The clean end state is either #75613 expanded to cover both boundaries or #75613 merged for the runtime-loading half plus a narrow follow-up for the backup-directory scan filters.

## Reproduction Assessment

Yes for a high-confidence source-level reproduction. Current main can be traced from the CLI into default-enabled plugin collector discovery and from both audit scan helpers into unfiltered directory listings; the live timeout/orphan-process symptom was not reproduced during this read-only review.

## Solution Assessment

Yes, the narrow audit-scoped direction is the best maintainable fix: keep plain audit off plugin runtimes, reserve collector imports for deep or explicit internal calls, and filter hidden/backup/non-plugin directories in the audit scan helpers instead of changing global plugin loader behavior.
