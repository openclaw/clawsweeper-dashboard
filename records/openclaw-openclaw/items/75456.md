## Summary

Keep open. Current main still shows the reported audit-boundary regression: the CLI waits for the full audit before emitting JSON, plugin security collector discovery still reaches the metadata snapshot loader without a manifest-only/no-runtime-deps guard, and installed plugin directory enumeration still includes hidden backup directories.

## What This Changes

The issue reports that `openclaw security audit --json` can hang because audit collector discovery stages bundled plugin runtime dependencies and hidden extension backup directories are scanned as installed plugins.

## Best Possible Solution

Keep this open and make the audit boundary read-only and bounded: collector discovery should avoid staging bundled runtime dependencies, extension root enumeration should ignore hidden and `node_modules` directories, and focused regression tests plus a user-facing changelog entry should cover both paths.

## Reproduction Assessment

Yes. The source path is enough to reproduce at high confidence: `security audit --json` waits on `runSecurityAudit`, the collector fallback reaches the plugin loader with default module/runtime-deps behavior, and both installed-plugin enumerators include hidden directories.

## Solution Assessment

Yes. The narrowest maintainable fix is to stop audit-time runtime dependency staging at the collector discovery boundary and filter backup/hidden dirs at the plugin directory enumeration boundary; broad plugin loading or installer behavior should not be changed.
