## Summary

Keep open. Current main resolves the plain `security audit` runtime collector/staging half, but both installed-plugin audit scanners still enumerate every direct directory under `stateDir/extensions`, so hidden backup directories remain in scope.

## What This Changes

The issue asks for `security audit --json` to avoid plugin runtime-deps staging and to ignore hidden or backup extension directories during plugin audit scans.

## Best Possible Solution

Keep the shipped runtime-collector behavior, then add a shared audit-side installed-plugin directory filter with regression coverage for dotdirs, `.openclaw-install-backups`, `node_modules`, and backup/disabled entries.

## Reproduction Assessment

Yes. On current main, a `stateDir/extensions/.openclaw-install-backups` directory would be returned by both audit `listInstalledPluginDirs` helpers and then scanned or treated as an installed plugin candidate.

## Solution Assessment

Yes for the remaining work. Filtering at the audit installed-plugin directory boundary is narrower and safer than changing global plugin loading or reworking security audit runtime collection again.
