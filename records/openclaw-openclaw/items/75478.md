## Summary

Keep open: this PR targets a real Telegram CLI gateway-dispatch bug, but the branch is not merge-ready because the new CLI fast path bypasses existing presentation parsing and local-media access handling, and it is missing the required user-facing changelog entry.

## What This Changes

The PR forwards Telegram message-action execution mode through the bundled plugin wrapper, switches message-action gateway calls to the normal gateway helper, adds a Telegram CLI send gateway fast path, and adds focused regression tests.

## Best Possible Solution

Keep the wrapper `resolveExecutionMode` fix and the gateway-helper cleanup, but make the CLI gateway path faithful to the existing message-action contract: preserve JSON parsing for `presentation` and `delivery`, support presentation-only sends, and avoid breaking local media root expansion. If full parity would broaden the gateway API too much for this PR, constrain the fast path to simple text sends and leave richer sends on the normalized path until the gateway can safely carry the required media-access context.

## Reproduction Assessment

Yes. The linked issue gives a concrete `openclaw message send --channel telegram ... --json` path, and current-main source shows why it can reproduce: the CLI loads configured channel plugins before dispatch while the Telegram wrapper drops the gateway execution-mode declaration.

## Solution Assessment

No, not as-is. Delegating Telegram sends to the live gateway is the right direction, but the current fast path is too narrow and bypasses normalization/media contracts that existing CLI sends rely on.

  - body: The fast path catches every non-dry-run Telegram send with a target, but then rejects calls unless `--message` or `--media` is present and forwards `presentation`/`delivery` as raw strings. The gateway `message.action` handler dispatches directly to Telegram without `mediaLocalRoots`, while the existing runner expands local media parent roots before dispatch, so explicit files outside default OpenClaw roots can regress to local media access failures.
  - confidence: 0.86
