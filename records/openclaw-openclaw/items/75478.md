## Summary

Keep open: the PR addresses a real Telegram CLI hang path, but the proposed fast path bypasses existing send normalization and would regress supported presentation-only and local-media sends before it is safe to merge.

## What This Changes

The PR forwards Telegram action execution mode through the bundled wrapper, switches message-action gateway calls to `callGateway`, adds a Telegram CLI gateway fast path, and adds focused regression tests.

## Best Possible Solution

Keep the wrapper execution-mode and gateway-helper direction, but make Telegram CLI gateway sends reuse the existing action normalization/parsing/media-access contract or limit the fast path to cases it can faithfully preserve, then add focused tests and a changelog entry.

## Reproduction Assessment

Yes. The linked #75477 gives a concrete `openclaw message send --channel telegram ... --json` path, and current-main source shows both the eager CLI plugin load and the wrapper gap that can prevent gateway-owned Telegram sends from being delegated early.

## Solution Assessment

No, not as proposed. Delegating Telegram sends to the live gateway is the right direction, but this implementation bypasses the normal presentation JSON parsing and local-media access derivation that existing CLI sends rely on.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve presentation-only Telegram sends:** `src/cli/program/message/helpers.ts:74-76`
  - body: The fast path intercepts every non-dry-run Telegram `send` with a target, then throws unless `--message` or `--media` is present. The documented CLI contract allows `--presentation` alone, and this path also forwards presentation JSON as a string that the Telegram action runtime's `normalizeMessagePresentation` will ignore. Reuse the normal action parsing/normalization before the gateway call or keep this fast path out of presentation sends.
  - confidence: 0.9
- **[P2] Keep local media sends on the normalized path:** `src/cli/program/message/helpers.ts:86-88`
  - body: This path forwards `--media /some/local/file` as only `mediaUrl` to gateway `message.action`. The gateway handler dispatches directly to the Telegram plugin without the runner's derived `mediaLocalRoots`/read capability, so explicit local files outside default trusted roots can regress to local-media access failures.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/cli/program/message/helpers.ts:157`
  - body: This is a user-facing CLI/Telegram bug fix, but the PR does not update `CHANGELOG.md`. Repo policy requires user-visible fixes to be listed under the active release `### Fixes` section so the Telegram CLI hang fix is visible in release notes.
  - confidence: 0.97
