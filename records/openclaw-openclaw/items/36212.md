## Summary

Current main still drops the queued platform message identity when collect-mode and overflow-summary drains synthesize a new follow-up run. The issue should remain open because https://github.com/openclaw/openclaw/pull/91272 explicitly owns the fix and is still open; its focused queue-boundary approach is sound, but the PR has only test evidence and explicitly lacks a live Feishu native-reply check.

## What This Changes

Preserve queued platform message and reply-target identifiers when collect-mode drains create a synthesized deferred agent turn.

## Best Possible Solution

Land the focused queue-layer metadata propagation after redacted live Feishu proof confirms that a collected queued turn can successfully use the preserved platform message ID for native reply targeting.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: enqueue same-route collect-mode runs with platform message IDs, drain them, and observe that the synthesized run omits those IDs.

## Solution Assessment

Yes. Preserving the already-typed metadata where the queue drain synthesizes the follow-up is narrower and more maintainable than Feishu-specific policy or embedding IDs into model-visible prompt text.
