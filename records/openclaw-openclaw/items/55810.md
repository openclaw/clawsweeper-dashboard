## Security Review

Status: needs_attention

Summary: Needs attention because the PR exposes `rslave`/`rshared` workspace mount propagation for sandbox Docker binds and the docs currently misstate `rslave` as read-only.

Concerns:

- **[medium] Review rshared workspace propagation policy:** `src/config/types.sandbox.ts:68`
  - body: Docker documents `rshared` as recursive bidirectional propagation between original and replica mount points, so exposing it on sandbox workspace binds should be explicitly approved, documented with warnings, or narrowed to safer modes.
