Keep this PR open: current main still lacks the requested Docker workspace propagation config, but the submitted patch is not ready because it only wires the main sandbox container, misdocuments propagation semantics, lacks public docs and real Docker proof, and exposes security-sensitive sandbox mount behavior.

## What This Changes

The PR adds a `sandbox.docker.workspaceMountPropagation` enum, threads it through Docker sandbox config resolution, formats workspace bind propagation on the main sandbox container, and updates schema/generated config plus focused tests.

## Best Possible Solution

Land a narrow, security-reviewed implementation that applies typed workspace propagation to both agent and browser containers, documents Linux/Docker limitations and security semantics, and includes redacted real Docker proof.

## Security Review

Status: needs_attention
