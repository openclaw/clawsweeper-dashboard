## Summary

Keep open: current main still lacks this Docker sandbox config surface, and the PR remains incomplete because browser sandbox containers would not receive the propagation setting and the security-sensitive mode lacks proof/approval.

## What This Changes

This PR adds a `sandbox.docker.workspaceMountPropagation` enum through config/schema/generated schema, formats Docker workspace binds with propagation options, and updates focused sandbox mount/config-hash tests.

## Best Possible Solution

Land a focused update that adds the typed config plus help/docs metadata, applies propagation consistently to agent and browser workspace mounts, and scopes `rshared` according to sandbox security policy.

## Reproduction Assessment

Yes for the PR defect by source inspection: the branch only passes `cfg.workspaceMountPropagation` in `createSandboxContainer`, while `ensureSandboxBrowser` still calls `appendWorkspaceMountArgs` without a propagation value. I did not run live Docker in this read-only review.

## Solution Assessment

No for this exact patch. The direction is maintainable only if browser-container parity, public help/docs metadata, real Docker proof, and sandbox security approval for `rshared` are handled together.
