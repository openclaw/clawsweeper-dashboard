## Summary

Keep open: current main does not already provide this VirtualBox/Vagrant installer surface, and the PR head still has source-reproducible install, runtime, and security blockers plus missing real behavior proof.

## What This Changes

This PR adds a VirtualBox/Vagrant OpenClaw VM guide plus host installer, Vagrantfile, CLI wrapper, and guest provisioning scripts under `scripts/virtualbox`.

## Best Possible Solution

Either rework this into official VirtualBox support after owner/security signoff with working install paths, safe env parsing, failure propagation, docs navigation, and correct gateway access guidance, or keep it as an external/community tool.

## Reproduction Assessment

Yes. Source inspection plus small bash probes give high-confidence reproduction paths for the installer failure, unset-variable exits, raw env sourcing, masked provisioning failures, dropped SSH arguments, and incorrect gateway access guidance.

## Solution Assessment

No. The direction may be useful, but this is not the best mergeable form until OpenClaw decides whether to own the installer/security model and the implementation matches current gateway and shell-safety contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Make the documented curl installer work:** `scripts/virtualbox/install.sh:149`
  - body: The guide tells users to pipe this script into `bash`, but in that mode `BASH_SOURCE[0]` is empty and `LOCAL_DIR` never points at the checked-in VM files. The installer therefore fails here after installing dependencies, so the advertised install path cannot complete.
  - confidence: 0.96
- **[P2] Enter the cloned checkout before local install:** `docs/install/vm.md:60`
  - body: The fallback instructions clone `openclaw/openclaw` and then run `cd scripts/virtualbox` from the parent directory. That path only exists after first entering the newly cloned `openclaw` directory, so the documented fallback fails.
  - confidence: 0.94
- **[P1] Initialize vagrant_status before testing it:** `scripts/virtualbox/openclaw-vm:323`
  - body: On the first-create path, a successful `vagrant up | track_provision_progress` leaves `vagrant_status` unset. With `set -u`, the later expansion exits with an unbound-variable error after provisioning succeeds.
  - confidence: 0.96
- **[P2] Initialize halt_status before testing it:** `scripts/virtualbox/openclaw-vm:357`
  - body: A successful `vagrant halt` leaves `halt_status` unset, then this check expands it under `set -u`. A normal successful stop can therefore fail before printing success.
  - confidence: 0.91
- **[P1] Parse VM config without sourcing password text:** `scripts/virtualbox/openclaw-vm:396`
  - body: `write_vm_env` stores `VM_PASSWORD` verbatim and `status` later sources the file. A password containing shell syntax or unmatched quotes can execute commands or break parsing on the host.
  - confidence: 0.97
- **[P2] Forward ssh subcommand arguments:** `scripts/virtualbox/openclaw-vm:570`
  - body: `cmd_ssh` accepts arguments, but the dispatcher drops them here. The documented `openclaw-vm ssh -c "openclaw gateway"` example opens an interactive SSH session instead of running the command.
  - confidence: 0.97
- **[P2] Propagate reprovision failures:** `scripts/virtualbox/openclaw-vm:442`
  - body: `provision` masks `vagrant provision` failures with `|| true` and always prints success. Failed package setup, firewall changes, or SSH provisioning can be hidden from users and automation.
  - confidence: 0.88
- **[P2] Configure gateway access before promising WebChat:** `docs/install/vm.md:134`
  - body: The guide starts the default gateway inside the VM and then tells users to open host `localhost:18789`. Current gateway defaults bind to loopback, so Vagrant forwarding will not reach it without an explicit bind change or SSH tunnel guidance.
  - confidence: 0.85
- **[P3] Add the page to install navigation:** `docs/install/vm.md:1`
  - body: The PR adds a new install page but does not register it in `docs/docs.json`. The page would be published without an Install navigation entry, so readers would need to guess the URL.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The diff adds executable host/guest setup, LAN SSH exposure, and raw env sourcing, so it needs security review before merge.

Concerns:

- **[high] Raw .env sourcing executes password text:** `scripts/virtualbox/openclaw-vm:396`
  - body: `VM_PASSWORD` is written verbatim to `.env`, then `openclaw-vm status` sources that file. A password containing shell syntax can execute on the host, so status should parse known keys without shell evaluation.
  - confidence: 0.97
- **[medium] SSH is forwarded on all host interfaces:** `scripts/virtualbox/Vagrantfile:50`
  - body: The Vagrantfile exposes guest SSH on host port 2222 with `host_ip: "0.0.0.0"`, and provisioning can enable password auth for the `vagrant` user. Official docs/scripts need explicit owner approval and hardening for this LAN-exposed surface.
  - confidence: 0.87
