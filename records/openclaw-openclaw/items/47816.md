## Summary

Keep open: current main does not have this VirtualBox support, but the PR head is not mergeable because source inspection shows broken install/start/status/headless paths and security-sensitive SSH/env handling that need maintainer review.

## What This Changes

The PR adds a VirtualBox/Vagrant install guide plus `scripts/virtualbox` installer, Vagrantfile, CLI wrapper, and guest provisioning scripts.

## Best Possible Solution

Either land official VirtualBox support after maintainer product/security signoff with working install paths, safe env parsing, failure propagation, docs navigation/changelog, and correct gateway access guidance, or keep it as an unofficial community guide outside core.

## Reproduction Assessment

Yes. Source inspection of PR head gives high-confidence reproduction paths for the blocking defects: `curl | bash` has no local script directory, first-create success reads an unset variable, `status` sources password text, and the documented headless SSH command drops its arguments.

## Solution Assessment

No. The direction may be useful, but this is not the best mergeable form until OpenClaw owns the installer/security model and the scripts/docs match current gateway bind and auth behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Make the documented curl installer work:** `scripts/virtualbox/install.sh:149`
  - body: The guide tells users to pipe this script into `bash`, but in that mode `BASH_SOURCE[0]` is not a local `scripts/virtualbox` file. `LOCAL_DIR` remains empty and this branch aborts, so the advertised install path cannot complete.
  - confidence: 0.96
- **[P2] Enter the cloned repository before local install:** `docs/install/vm.md:60`
  - body: The fallback instructions clone `openclaw/openclaw` and then run `cd scripts/virtualbox` from the parent directory. That path does not exist until the user first enters the `openclaw` checkout.
  - confidence: 0.94
- **[P1] Initialize vagrant_status before testing it:** `scripts/virtualbox/openclaw-vm:320-323`
  - body: On the first-create path, a successful `vagrant up | track_provision_progress` never assigns `vagrant_status`. With `set -u`, the later `[[ -n "$vagrant_status" ]]` exits with an unbound variable after provisioning succeeds.
  - confidence: 0.95
- **[P2] Initialize halt_status before testing it:** `scripts/virtualbox/openclaw-vm:355-357`
  - body: A successful `vagrant halt` leaves `halt_status` unset, then the next `[[ -n "$halt_status" ]]` expansion trips `set -u`. That makes a normal successful stop fail before printing success.
  - confidence: 0.9
- **[P1] Stop sourcing the raw password file:** `scripts/virtualbox/openclaw-vm:396`
  - body: `write_vm_env` stores `VM_PASSWORD` verbatim and `status` later `source`s that file. A password containing shell syntax, backticks, or unmatched quotes can execute commands or break parsing on the host.
  - confidence: 0.96
- **[P2] Forward ssh subcommand arguments:** `scripts/virtualbox/openclaw-vm:570`
  - body: `cmd_ssh` accepts arguments, but the dispatcher drops them here. The documented `openclaw-vm ssh -c "openclaw gateway"` example opens an interactive SSH session instead of running the command.
  - confidence: 0.97
- **[P2] Propagate reprovision failures:** `scripts/virtualbox/openclaw-vm:442-445`
  - body: `provision` masks `vagrant provision` failures with `|| true` and always prints success. Failed package setup, firewall changes, or SSH provisioning can be hidden from users and automation.
  - confidence: 0.86
- **[P2] Configure gateway access before promising host WebChat:** `docs/install/vm.md:134`
  - body: The guide starts the default gateway inside the VM and then tells users to open host `localhost:18789`. Current gateway defaults bind to loopback on VM hosts, so Vagrant forwarding will not reach the service without an explicit LAN/custom bind plus auth/origin guidance or an SSH tunnel.
  - confidence: 0.84
- **[P3] Register the guide in install navigation:** `docs/install/vm.md:1`
  - body: Adding the page without updating `docs/docs.json` leaves it out of the Install navigation, so readers would need to guess the URL even after the page is published.
  - confidence: 0.9
- **[P3] Add a changelog entry for the VM installer:** `docs/install/vm.md:1`
  - body: This adds a user-facing `openclaw-vm` installer and lifecycle command surface, but there is no `CHANGELOG.md` entry. Repo policy expects user-facing feature changes to be recorded before merge.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The PR adds executable host/guest setup plus LAN SSH exposure and raw env sourcing, so it needs security attention before merge.

Concerns:

- **[high] Raw .env sourcing executes password text:** `scripts/virtualbox/openclaw-vm:396`
  - body: `VM_PASSWORD` is written verbatim to `.env`, then `openclaw-vm status` sources that file. A password containing shell syntax can execute on the host, so status should parse known keys without shell evaluation.
  - confidence: 0.96
- **[medium] SSH is forwarded on all host interfaces:** `scripts/virtualbox/Vagrantfile:50`
  - body: The Vagrantfile exposes guest SSH on host port 2222 with `host_ip: "0.0.0.0"`, and provisioning can enable password auth for the `vagrant` user. Official docs/scripts should require explicit maintainer approval and clear hardening for this LAN-exposed surface.
  - confidence: 0.86
