## What This Changes

The issue requests exposing Deepgram's `utterances=true` transcription query option through OpenClaw's Deepgram media provider configuration.

## Best Possible Solution

Keep the canonical `providerOptions.deepgram.utterances` path and avoid expanding the deprecated Deepgram compat object; open a separate issue only if structured Deepgram utterance arrays need downstream storage or APIs.
