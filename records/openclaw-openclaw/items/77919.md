## Summary

Keep open. Current main can pass `providerOptions.deepgram.utterances` through to Deepgram, but OpenClaw still has no native/documented utterance contract and the Deepgram plugin only returns transcript text, so this needs a maintainer scope decision rather than cleanup closure.

## What This Changes

Request to add first-class Deepgram `utterances` support for audio transcription, following the existing Deepgram `smartFormat`/`smart_format` option pattern.

## Best Possible Solution

Define the intended native utterances contract: document and test the existing `providerOptions.deepgram.utterances` pass-through if query support is enough, or add a versioned media-understanding/SDK result shape and Deepgram parser support if timestamp/confidence metadata should be consumable.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config/API request. Source inspection gives high confidence that the query flag can already be passed, while structured utterance data is not exposed.

## Solution Assessment

Unclear; the proposed two-line compat addition is not the complete solution because `providerOptions` already forwards `utterances`, while native timestamp/confidence support would require an output-contract change.
