## Summary

Keep open: current main already appears to pass `providerOptions.deepgram.utterances` through to Deepgram, but OpenClaw still does not document the option or preserve Deepgram's returned utterance and word metadata in the transcription result/output contract. The remaining scope needs a maintainer API/product decision, not conservative cleanup closure.

## What This Changes

Request native Deepgram `utterances` support alongside `smartFormat`, including the query option and useful downstream utterance/word metadata.

## Best Possible Solution

Document existing pass-through if parameter-only support is intended; otherwise add an additive transcription metadata contract and preserve Deepgram `results.utterances` through the provider/SDK output.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that the query flag can already be sent, while the missing structured metadata surface is visible in the current parser and output types.

## Solution Assessment

No, the proposed two-line compat change is not the best complete solution because generic `providerOptions` already preserves `utterances`; native support needs docs and likely an additive media-output metadata design.
