## Summary

The linked Agent Teams bug is real, but this PR is not merge-ready: it infers a successful subagent completion from `running` plus `endedAt`, which is not an explicit successful terminal state, and its supplied proof still shows the new case failing.

## What This Changes

The PR changes session-store reconciliation to treat `status: "running"` entries with `endedAt` as successful subagent completions and adds focused Vitest coverage for that helper.

## Best Possible Solution

Use only an explicit successful terminal child state from the current run's private status/transcript boundary for lost-context recovery, or keep the existing lost-context fallback until that boundary exists.

## Reproduction Assessment

Yes, source-level only: the caller and fallback path are clear on current main, but this review did not run a live Agent Teams orchestration repro.

## Solution Assessment

No. The proposed predicate is too broad because `running` plus `endedAt` is not a safe success signal for lifecycle recovery.

- **[P1] Do not infer success from a running session row:** `src/agents/subagent-session-reconciliation.ts:156-169`
