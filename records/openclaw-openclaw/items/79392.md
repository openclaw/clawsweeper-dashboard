## Summary

Keep open. Current main still matches the reported timeout-policy mismatch: `agents.defaults.timeoutSeconds=300` is capped at the 120s model idle watchdog, `agents.defaults.llm` is intentionally removed, and the supported replacement is provider-scoped timeout configuration that needs maintainer scoping for bundled Gemini ergonomics and diagnostics.

## What This Changes

The issue requests that slow Gemini model calls honor the configured timeout, or provide an accepted replacement for the removed `agents.defaults.llm.idleTimeoutSeconds` key.

## Best Possible Solution

Define the intended long-model-silence contract clearly: either make provider-scoped timeout configuration ergonomic and well-diagnosed for bundled Google/Gemini, or add a documented global model-idle timeout policy with regression coverage.

## Reproduction Assessment

Yes for the config/timeout mismatch: current source and tests show `agents.defaults.timeoutSeconds=300` still resolves to the 120s idle watchdog, and the legacy `agents.defaults.llm` key is removed. No live Docker/Gemini request was run.

## Solution Assessment

Unclear. A mechanical cap removal or legacy-key restore would conflict with the current provider-scoped timeout direction, so maintainers should choose the public config contract before implementation.
