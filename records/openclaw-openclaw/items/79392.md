## What This Changes

The issue reports that configured agent timeout settings are not preventing `LLM request timed out` failures around 125 seconds for Gemini requests in Docker.

## Best Possible Solution

Keep the remaining timeout-config work centralized in https://github.com/openclaw/openclaw/issues/77744; add Gemini-specific logs there if provider timeout still fails after using the documented key.

## Reproduction Assessment

Yes for cleanup: closing as duplicate is the best repository action because the same remaining timeout-config gap is already tracked in an open canonical issue.
