## Summary

Current main covers this Gemini 3.1 timeout pattern with the merged Google transport first-response retry, and that fix shipped after the reporter's v2026.5.7 build in v2026.5.9-beta.1.

## What This Changes

The issue reports Gemini LLM calls timing out around 125s on OpenClaw 2026.5.7 despite `agents.defaults.timeoutSeconds=300`.

## Best Possible Solution

Keep the provider-owned Gemini 3 first-response retry and ask for a fresh report only if the timeout still reproduces on v2026.5.9-beta.1 or newer.

## Reproduction Assessment

No current-main failure reproduction was run; the historical reproduction is high-confidence from this issue's logs plus the live before/after proof in https://github.com/openclaw/openclaw/pull/79668. Current source and tests show the failing Gemini 3 no-first-response path now has a retry before the idle watchdog.

## Solution Assessment

Yes. The merged fix is the narrow provider-owned solution: it handles the direct official Google Gemini 3 first-response stall without changing global agent timeout policy or restoring the removed `agents.defaults.llm` config path.
