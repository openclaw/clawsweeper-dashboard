The issue reports a Windows Feishu DM performance regression after 2026.5.2 where dispatch-to-agent latency grows with a large long-lived session and also notes `Runtime: unknown` in gateway status.

## Solution Assessment

Unclear. Existing opt-in transcript byte guards and CLI token-budget compaction cover some long-session cases, but changing defaults or adding Feishu-only policy without profiling could mask the real shared bottleneck.
