Investigate a Windows Feishu DM regression after upgrading to OpenClaw 2026.5.2 where long-lived session context appears to make agent dispatch very slow.

## Best Possible Solution

Profile a current-main Windows Feishu DM turn with per-phase timings for Feishu dispatch, session loading, context assembly, model/harness execution, compaction, and `schtasks` status, then fix the smallest shared session/compaction or Windows parser gap that profiling proves.
