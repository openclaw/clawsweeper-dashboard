## What This Changes

The issue asks OpenClaw to make OpenAI-compatible streaming classify `content` and `reasoning_content` consistently so reasoning-only final answers do not appear as empty replies and reasoning text does not leak as visible text.

## Best Possible Solution

Keep reasoning-field final-answer policy in https://github.com/openclaw/openclaw/issues/74021 and visible meta-reasoning cleanup in https://github.com/openclaw/openclaw/issues/45041; avoid a generic `reasoning_content` to visible-text fallback.

## Reproduction Assessment

Yes for the source-level parser behavior: current main deterministically treats non-empty `delta.content` as visible text and bare `reasoning_content` as thinking. The live Bailian/Z.AI nondeterminism was not rerun, but the remaining behavior is already covered by canonical tracking.

## Solution Assessment

No, the suggested global fallback is not the best fix because it can expose provider reasoning as user-visible text. The safer direction is opt-in provider/model policy plus the separate visible-output leakage tracker.
