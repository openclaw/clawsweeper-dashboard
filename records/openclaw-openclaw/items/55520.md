## Summary

Keep open. Current main still has the reported parser-level mismatch: non-empty `delta.content` becomes visible text before reasoning fields are considered, while bare `reasoning_content`/`reasoning`/`reasoning_text` still becomes `thinking`; the newer #74021 is related but broader and does not replace the Bailian/Z.AI-specific evidence here.

## What This Changes

The issue asks OpenClaw to make streaming `content` and `reasoning_content` classification stable so final answers are not hidden as thinking or internal reasoning exposed as normal text.

## Best Possible Solution

Add provider-scoped stream normalization or explicit model metadata for providers/models proven to return final answers in reasoning fields, with fixtures for the mirrored samples and guards that keep real reasoning private by default.

## Reproduction Assessment

Yes for the parser-level behavior: source inspection shows `delta.content` deterministically becomes text and bare `reasoning_content` deterministically becomes thinking. The live provider nondeterminism still needs redacted Bailian/Z.AI fixtures or live confirmation.

## Solution Assessment

No, the suggested global fallback is not the safest fix because it can leak real chain-of-thought. The maintainable solution is provider/model-scoped compatibility with tests and documented reasoning-exposure policy.
