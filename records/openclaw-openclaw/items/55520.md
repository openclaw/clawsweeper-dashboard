## Summary

Keep open. Current main still has the parser-level mismatch: non-empty `delta.content` is accepted as visible text before reasoning fields are considered, while bare `reasoning_content`/`reasoning`/`reasoning_text` is still stored as thinking; the related #74021 is broader and does not preserve this issue's Bailian/Z.AI-specific samples by itself.

## What This Changes

The issue asks OpenClaw to classify streaming `content` and `reasoning_content` consistently for GLM-style providers so replies do not become blank visible messages or expose reasoning as normal text.

## Best Possible Solution

Add provider/model-scoped stream normalization or explicit compatibility metadata for providers proven to return final answers in reasoning fields, with regression fixtures and guards that keep real reasoning private by default.

## Reproduction Assessment

Yes for the parser-level behavior: source inspection shows `delta.content` becomes visible text and bare reasoning fields become thinking on current main. The live Bailian/Z.AI nondeterminism still needs redacted fixtures or a live provider replay.

## Solution Assessment

No, the suggested blanket fallback is not the safest solution because it can leak real chain-of-thought. The narrower maintainable fix is provider/model-scoped compatibility with tests for the reported samples.
