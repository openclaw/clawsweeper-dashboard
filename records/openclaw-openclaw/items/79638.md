## Summary

Keep open: current main and the latest release still keep heartbeat isolation opt-in, while the active-reply and pending-delivery guards only cover part of the reported cross-session/stale-context behavior. ## What This Changes

The issue reports a regression where heartbeat runs in multi-agent setups can mix sessions and reply with stale conversation content instead of only the heartbeat acknowledgement.

## Best Possible Solution

Keep this issue open until maintainers choose a compatibility-safe heartbeat session isolation policy and land a fresh current-main repair with docs, focused tests, and live multi-agent or Telegram proof.

## Reproduction Assessment

Partly. Source inspection shows current main and v2026.6.6 still default heartbeat runs into the shared/main session unless isolatedSession is enabled, matching the stale-context risk; no live WSL2 multi-agent reproduction was run here.

## Solution Assessment

No, closing is not the best solution. The active-run and pending-final guards are useful partial fixes, but the remaining default isolation/session-boundary behavior needs maintainer decision and live proof before this can be resolved.
