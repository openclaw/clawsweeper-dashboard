## Summary

Keep open: this is a recent external regression report with plausible current-main overlap in heartbeat routing and transcript handling, and related heartbeat routing work is still open rather than shipped as a terminal fix.

## What This Changes

Heartbeat runs in multi-agent setups are reported to leak replies across agent sessions and deliver stale prior-conversation content followed by HEARTBEAT_OK after upgrades from 2026.4.15 to 2026.4.29-5.4.

## Best Possible Solution

Capture a minimal redacted multi-agent heartbeat transcript and then land a focused fix that prevents heartbeat delivery/session state contamination and suppresses HEARTBEAT_OK-tailed stale content without changing unrelated heartbeat routing semantics.

## Reproduction Assessment

Partially. Source inspection shows current main can still deliver long text that ends with HEARTBEAT_OK, but there is no high-confidence live reproduction for the cross-agent DM contamination without the reporter's config and transcript.

## Solution Assessment

Unclear. The best fix boundary depends on whether the failing path is heartbeat reply normalization, persisted delivery context poisoning, provider replay/transcript repair, or a combination already partly addressed by open routing PRs.
