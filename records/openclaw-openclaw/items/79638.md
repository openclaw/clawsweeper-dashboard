## Summary

Keep open: current main still leaves credible heartbeat/session-contamination paths unresolved, including shared heartbeat history by default, no active reply-run guard in the heartbeat runner, and related stale replay/context-pollution reports that remain open.

## What This Changes

The issue reports a regression where heartbeat turns can reuse stale conversation content or deliver one agent's reply into another agent's session.

## Best Possible Solution

Keep this open until maintainers land or replace the active reply-run guard, decide the heartbeat isolation default, and route the duplicate-context/replay branches through the narrower open Telegram and stale-replay reports.

## Reproduction Assessment

No for the exact Zai multi-agent incident: there is no redacted transcript or live reproduction. Yes at source level for plausible branches: current main still defaults to shared heartbeat history and lacks an active reply-run guard in `runHeartbeatOnce`.

## Solution Assessment

Unclear. `isolatedSession: true` plus `lightContext: true` is a good mitigation, but the best repository fix depends on maintainer approval for default isolation and on landing a narrow active-reply guard without masking the separate stale replay/context-duplication branches.
