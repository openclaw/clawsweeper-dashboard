## Summary

Keep open: the report matches a current source path where explicit MiniMax thinking bypasses the MiniMax disable guard, while the Anthropic stream parser only handles OpenAI-style `reasoning_content`/`content` deltas for Xiaomi endpoints, so MiniMax can drop both streamed reasoning and final text instead of delivering a reply.

## What This Changes

The issue reports that MiniMax M2.7 with high thinking and `/reasoning stream` returns no visible reasoning or final answer on beta.5/beta.6.

## Best Possible Solution

Normalize MiniMax Anthropic-compatible `reasoning_content`/`content` stream deltas into internal reasoning and final text, then keep that reasoning out of replay history so explicit thinking works without silent replies or feedback loops.

## Reproduction Assessment

Yes at source level, but not live: the reported config follows the documented explicit MiniMax thinking path, and current main preserves that payload while excluding MiniMax from the only parser branch that consumes OpenAI-style `reasoning_content` plus `content` deltas.

## Solution Assessment

Yes: the narrowest maintainable fix is in the MiniMax/provider stream and replay boundary, not in channel delivery or Gateway finalization; avoid solving it by silently disabling explicit MiniMax thinking unless the docs and product contract change.
