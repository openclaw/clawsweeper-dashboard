## Label Justifications

- `P1`: The report describes a regression that can make an agent/channel workflow return no user-visible reply.
- `impact:message-loss`: The user-visible failure is a silently missing final answer after a successful model call.
- `impact:session-state`: The report also states the reasoning and final-answer blocks are absent from the session transcript.

## Summary

Current main contains the merged reply-delivery fix that matches the reported regression: preview/reasoning stream text is no longer treated as proof that the final answer was delivered, and regression coverage now expects final payloads after a preview.

## What This Changes

The issue reports that MiniMax M2.7 with thinking high and `/reasoning stream` stopped returning both streamed reasoning and the final answer on beta.5/beta.6.

## Best Possible Solution

Keep the merged final-delivery behavior and ship it in the next beta/release so preview or reasoning stream text cannot replace the completed answer.

## Reproduction Assessment

Yes for the pre-fix regression: the removed `previewStreamedText` path treated partial preview text as delivery evidence and suppressed matching final payloads, and the current e2e test covers that shape. I did not run a live MiniMax call in this read-only review.

## Solution Assessment

Yes. The merged fix is the narrow owner-boundary change: keep block/direct/message-tool dedupe, but stop using preview text alone to suppress final delivery.
