## Summary

Keep open: current main still has a plausible source-level gap for MiniMax Anthropic-compatible streams when the user explicitly enables thinking, because MiniMax `reasoning_content`/`content` deltas are only parsed on the Xiaomi-compatible branch and otherwise fall through to native Anthropic delta handling.

## What This Changes

Issue reports that MiniMax M2.7 with thinking level high and reasoning streaming enabled returns no visible reasoning or final answer on 2026.5.16 beta.5/beta.6.

## Best Possible Solution

Split response parsing from replay policy: parse MiniMax Anthropic-compatible explicit-thinking streams into reasoning and final text blocks while preserving the existing policy that drops historical MiniMax reasoning unless replay support is separately proven.

## Reproduction Assessment

Yes, at source level: if MiniMax emits the documented OpenAI-style `reasoning_content` plus `content` deltas while explicit thinking is enabled, current main does not enter the parser branch that preserves those deltas for MiniMax. I did not run a live MiniMax CN credential repro.

## Solution Assessment

Yes, a narrow transport-parser fix is the best path: handle MiniMax response-side `reasoning_content`/`content` deltas without changing unrelated provider replay semantics. Closing as a duplicate of https://github.com/openclaw/openclaw/issues/81607 would be premature because that report is scoped to non-streaming infer CLI behavior.
