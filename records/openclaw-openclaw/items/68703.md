## Summary

Keep open. Current main still has the reported Discord guild-admin requester-authorization gap, and the issue is paired with open closing PR #68716, so this should remain open until that PR or a narrow replacement lands.

## What This Changes

The issue reports that Discord guild-admin mutation actions can execute with bot credentials without checking the requesting guild member's Discord permissions.

## Best Possible Solution

Land a narrow Discord plugin fix that marks privileged guild-admin message actions as trusted-requester-only, forwards the trusted Discord requester id into guild-admin runtime calls, checks the corresponding Discord permission before bot-credentialed mutations, and adds focused regression tests for missing sender identity, unauthorized senders, disabled action gates, and direct runtime permission writes.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists on current main: dispatch a Discord guild-admin tool action such as channel-delete without requesterSenderId, and the adapter/runtime path reaches deleteChannelDiscord because no Discord trusted-requester hook or guild-admin permission check runs.

## Solution Assessment

Yes. The maintainable fix belongs in the Discord plugin adapter and guild-admin runtime, using the existing core trusted-requester seam and the moderation permission-check pattern rather than adding Discord-specific policy to core.
