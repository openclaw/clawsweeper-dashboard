## Summary

Keep open. Current main still exposes the Discord guild-admin requester authorization gap, and open PR #68716 is the right implementation candidate to review before closing this security-sensitive issue.

## What This Changes

Discord guild-admin message actions should require a trusted requester identity and matching Discord guild permissions before bot-credentialed mutations run.

## Best Possible Solution

Land a narrow Discord plugin fix that marks privileged guild-admin tool actions as trusted-requester-only, forwards the trusted Discord requester id, checks the matching Discord permission before mutation, and adds focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source reproduction remains on current main: dispatch a Discord guild-admin action such as channel-delete without requesterSenderId, and the path reaches the bot-credentialed guild runtime because no trusted-requester hook or guild-admin permission check runs.

## Solution Assessment

Yes. The maintainable fix belongs in the Discord plugin adapter/runtime because core already provides the generic trusted-requester seam and moderation already demonstrates the Discord-owned permission-check pattern.
