## Summary

Keep open. Current main still has the reported Discord guild-admin requester-authorization gap, and the issue is paired with open closing PR #68716, so it should remain open until that PR or a narrow replacement lands.

## What This Changes

The issue reports that Discord guild-admin message actions can run bot-credentialed guild mutations without checking the requesting Discord member's guild permissions.

## Best Possible Solution

Land a narrow Discord plugin authorization fix that marks guild-admin tool actions as trusted-requester-only, forwards the trusted Discord requester id, checks the matching Discord guild permission before bot-credentialed mutations, and covers disabled-action ordering plus trusted local CLI/manual operator behavior in focused tests.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists on current main: dispatch a Discord guild-admin tool action such as channel-delete without requesterSenderId, and the current adapter/runtime path reaches bot-credentialed mutation code because no Discord trusted-requester hook or guild-admin permission check runs.

## Solution Assessment

Yes, the requested direction is the maintainable fix boundary. The fix should stay in the Discord plugin action adapter/runtime rather than core, because the core dispatcher already provides the generic trusted-requester seam and moderation already shows the Discord-owned permission-check pattern.
