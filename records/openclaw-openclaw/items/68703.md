## Summary

Keep open. Current main still has the Discord guild-admin requester-authorization gap, and the linked closing PR is open, unmerged, and conflicting, so this issue should remain the canonical tracker until a corrected fix lands.

## What This Changes

The issue reports that Discord guild-admin message actions can execute bot-privileged guild mutations without checking the requesting guild member's Discord permissions.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/68716 with a narrow Discord plugin fix that marks privileged guild-admin actions as trusted-requester-only, forwards trusted requester identity, checks matching Discord permissions before mutation, and preserves trusted owner/manual execution.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: a Discord guild-admin action such as channel-delete can reach the bot-credentialed guild runtime without requester authorization and with only feature-gate checks.

## Solution Assessment

Yes for the issue direction. The maintainable fix belongs in the Discord plugin adapter and guild runtime because core already has the trusted-requester seam and Discord moderation already demonstrates the permission-check pattern.
