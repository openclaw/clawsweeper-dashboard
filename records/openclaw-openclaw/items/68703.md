## Summary

Keep open. Current main still has the Discord guild-admin requester-authorization gap, and the linked closing PR is open, unmerged, and has review concerns, so this issue remains the canonical tracker.

## What This Changes

The issue reports that Discord guild-admin message actions can execute bot-credentialed guild mutations without checking the requesting Discord user's permissions.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/68716 with a narrow Discord plugin fix that gates requester-driven guild-admin actions, forwards trusted requester identity, checks matching Discord permissions, and preserves trusted owner/manual flows.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: channel-delete can reach the Discord guild runtime without requester authorization and with only feature-gate checks.

## Solution Assessment

Yes for the issue direction, but the exact patch needs maintainer/security review. The maintainable fix belongs in the Discord plugin adapter and guild runtime because core already has the trusted-requester seam and Discord moderation already demonstrates the permission-check pattern.
