## Summary

Keep open. Current main and the latest release still show the Discord guild-admin requester-authorization gap, and the open closing PR has not landed.

## What This Changes

The issue reports that Discord guild-admin message actions can perform bot-credentialed guild mutations without verifying the requester's Discord permissions.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/68716 with a narrow Discord plugin/runtime fix that requires trusted requester identity for Discord-origin guild-admin actions, checks matching Discord permissions before mutation, and preserves trusted owner/manual execution paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path where a Discord guild-admin action such as channel-delete can flow from the message action dispatcher into bot-credentialed mutation code without requester authorization.

## Solution Assessment

Yes for the issue direction, but the exact fix needs maintainer/security judgment on the owner-vs-requester trust boundary. The maintainable solution belongs in the Discord plugin adapter and guild runtime authorization layer, not in a broad core policy change.
