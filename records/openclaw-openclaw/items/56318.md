## Summary

Keep open. This PR still contains useful hardening ideas, but it is broad, stale against current main, partly superseded, and introduces concrete regressions in media compatibility, log streaming, session usage lookup, TUI shell behavior, and SecretRef exec environment handling.

## What This Changes

The PR applies security, memory-safety, and performance hardening across media loading, SecretRef exec resolution, SQL schema construction, gateway/agent caches, channel bridge event handling, stream writing, PDF extraction, TUI local shell execution, and related tests.

## Best Possible Solution

Refresh this as smaller owner-scoped PRs: keep the focused fixes that are already on main, preserve existing media, TUI shell, log streaming, SecretRef exec, and session archive contracts, and port any still-valid cache or hardening work with targeted compatibility and security tests.

## Reproduction Assessment

Yes. The regressions are reproducible from source-level checks against current main and the PR diff: active callers still pass `localRoots: "any"`, `openclaw logs` treats writer false as stop, session archive fallback depends on fresh readdir results, and TUI `!` commands currently rely on shell semantics.

## Solution Assessment

No. This broad stale patch is not the best way to solve the hardening work because it changes established API and trust-boundary contracts at once; smaller current-main patches with explicit tests and security review are safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve localRoots any compatibility:** `src/media/web-media.ts:35`
  - body: This removes `"any"` from the `loadWebMedia` option type, but current main still has production callers, including WhatsApp outbound media, that pass `localRoots: "any"` with a `readFile` override. Preserve the compatibility type or migrate every caller in the same PR, otherwise affected lanes will fail typecheck or lose the intended bypass behavior.
  - confidence: 0.9
- **[P1] Keep backpressure separate from writer failure:** `src/terminal/stream-writer.ts:49-50`
  - body: `stream.write()` returning false is normal backpressure, not a closed stream. Since `openclaw logs --follow` exits its output loop when the safe writer returns false, this change can truncate healthy high-volume log streams instead of waiting for drain or preserving the old success semantics.
  - confidence: 0.93
- **[P2] Read session directories fresh for archive fallback:** `src/infra/session-cost-usage.ts:344`
  - body: `resolveExistingUsageSessionFile` uses this listing to fall back from a missing active JSONL to reset/deleted archives. Caching the directory for five seconds can return a stale active filename after rotation or deletion, causing usage detail and log lookups to miss until the TTL expires.
  - confidence: 0.86
- **[P2] Preserve local shell command semantics:** `src/tui/tui-local-shell.ts:116-117`
  - body: Local `!` commands are documented and implemented as fresh shell commands after explicit in-session approval. Parsing into argv with `shell: false` treats pipes, redirects, globs, and `$VAR` expansion as literal arguments, so existing operator workflows silently misbehave.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff changes the SecretRef exec resolver child-process trust boundary by adding unconditional baseline environment passthrough.

Concerns:

- **[medium] Exec resolver environment is broadened:** `src/secrets/resolve.ts:700`

Adding HOME, USER, LANG, TERM, TMPDIR, TMP, and TEMP unconditionally gives every exec-backed SecretRef resolver more host context than configured and should be opt-in, documented, and tested if kept.
