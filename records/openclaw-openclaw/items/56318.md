## Summary

Keep open: the hardening work is useful, but this PR is not mergeable as-is because it mixes broad security/performance changes with source-reproducible regressions and one concrete SecretRef exec security concern. A narrower merged follow-up, #68842, already handled the `costUsageCache` portion, but the remaining work needs maintainer split/replace judgment.

## What This Changes

The PR bundles security, memory-safety, and performance hardening across media loading, SecretRef exec resolution, SQL schema creation, gateway and agent caches, channel bridge handling, stream writing, PDF extraction, and TUI local shell execution.

## Best Possible Solution

Refresh this as smaller owner-scoped PRs that preserve existing media, TUI shell, log streaming, SecretRef exec, and session archive contracts while adding focused hardening tests for each accepted change.

## Reproduction Assessment

Yes. The regressions are source-reproducible from current main plus the PR diff: active `localRoots: "any"` callers remain, `openclaw logs` exits on false writer results, session archive lookup depends on fresh directory reads, and TUI `!` commands are documented shell commands.

## Solution Assessment

No. The patch changes existing compatibility and security contracts in one large bundle; smaller current-main patches that either preserve contracts or migrate every caller with tests are the narrowest maintainable path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve localRoots any compatibility:** `src/media/web-media.ts:35-36`
  - body: This removes `"any"` from the `loadWebMedia` option type, but current main still has production callers, including WhatsApp outbound media, that pass `localRoots: "any"` with a `readFile` override. Preserve the compatibility type or migrate every caller in the same PR, otherwise affected plugin lanes will fail typecheck or lose the intended bypass behavior.
  - confidence: 0.9
- **[P1] Keep backpressure separate from writer failure:** `src/terminal/stream-writer.ts:49-50`
  - body: `stream.write()` returning false is normal backpressure, not a closed stream. Since `openclaw logs --follow` exits its output loop when the safe writer returns false, this change can truncate healthy high-volume log streams instead of waiting for drain or preserving the old success semantics.
  - confidence: 0.93
- **[P2] Read session directories fresh for archive fallback:** `src/infra/session-cost-usage.ts:344`
  - body: `resolveExistingUsageSessionFile` uses this listing to fall back from a missing active JSONL to reset/deleted archives. Caching the directory for five seconds can return a stale active filename after rotation or deletion, causing usage detail and log lookups to miss until the TTL expires.
  - confidence: 0.86
- **[P2] Preserve local shell command semantics:** `src/tui/tui-local-shell.ts:116-117`
  - body: Local `!` commands are documented and implemented as shell commands after explicit in-session approval. Parsing into argv with `shell: false` treats pipes, redirects, globs, and `$VAR` expansion as literal arguments, so existing operator workflows silently misbehave.
  - confidence: 0.84
