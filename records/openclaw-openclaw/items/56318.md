## What This Changes

The PR bundles security, memory-safety, and performance hardening across media loading, SecretRef exec environments, gateway/agent caches, channel bridge event queues, memory schema SQL construction, terminal writing, TUI local shell commands, and tests.

## Best Possible Solution

## Reproduction Assessment

Yes. The blocking regressions are source-reproducible from the PR diff against current main: local media callers still rely on localRoots: "any", logs treats writer false as stop-output, session fallback needs fresh directory state, and TUI local commands intentionally use shell semantics.

## Solution Assessment

No. This is not the best way to solve the issue because it bundles contract-changing security, memory, and performance changes across several ownership areas; smaller compatibility-preserving hardening PRs are safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve localRoots any compatibility:** `src/media/web-media.ts:35-36`
  - body: Current main still exposes and tests localRoots: "any" for host-read callers with a readFile override, and WhatsApp outbound media still uses it. Removing the type and bypass leaves existing callers unmigrated.
  - confidence: 0.91
- **[P1] Do not treat backpressure as write failure:** `src/terminal/stream-writer.ts:49-50`
  - body: stream.write() returning false is normal backpressure, but openclaw logs exits output loops when the writer returns false. This can truncate healthy high-volume log output instead of waiting for drain or preserving the old success semantics.
  - confidence: 0.93
- **[P2] Bypass stale dir-entry cache for session fallback:** `src/infra/session-cost-usage.ts:344`
  - body: resolveExistingUsageSessionFile uses this listing to fall back from a missing active JSONL file to reset/deleted archives. A five-second cached listing can select stale entries after rotation or deletion.
  - confidence: 0.86
- **[P2] Preserve approved local shell semantics:** `src/tui/tui-local-shell.ts:116-117`
  - body: Local ! commands are currently approved operator shell commands. Parsing to argv with shell: false treats pipes, redirects, globs, and variable expansion as literal arguments, silently changing existing workflows.
  - confidence: 0.82
- **[P2] Keep exec resolver environment opt-in:** `src/secrets/resolve.ts:700`
  - body: Current exec SecretRef resolution passes only configured passEnv plus provider env. Adding HOME, USER, LANG, TERM, TMPDIR, TMP, and TEMP for every provider broadens subprocess context without an explicit contract or test.
  - confidence: 0.78
