## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve host-read local media compatibility:** `src/media/web-media.ts:57-58`
  - body: Current WhatsApp outbound media still calls `loadWebMedia` with `localRoots: "any"`, `readFile`, and `hostReadCapability`. Removing the `"any"` bypass sends that existing value through the root guard instead of honoring the trusted host-read path, so existing outbound media can fail; keep the compatibility path or migrate all callers with tests.
  - confidence: 0.93
- **[P1] Do not treat backpressure as output closure:** `src/terminal/stream-writer.ts:49-51`
  - body: `stream.write()` returning `false` is normal backpressure, not a closed pipe. `openclaw logs` treats this writer return value as a stop signal, so this change can truncate healthy high-volume logs instead of waiting for drain or preserving the old success semantics.
  - confidence: 0.94
- **[P1] Keep exec resolver environment explicit:** `src/secrets/resolve.ts:701-710`
  - body: Exec SecretRef resolvers currently receive only `passEnv` and configured `env` entries. Adding `PATH`, `HOME`, `USER`, `TERM`, and temp variables to every resolver broadens the credential process boundary and can change resolver behavior without an opt-in contract or tests.
  - confidence: 0.88
- **[P1] Preserve approved local shell command semantics:** `src/tui/tui-local-shell.ts:109-118`
  - body: The current TUI `!` path intentionally uses `shell: true` after an in-session approval prompt. Replacing it with argv spawning silently turns pipes, redirects, globs, and variable expansion into literal arguments, breaking existing operator commands unless maintainers approve and document that migration.
  - confidence: 0.9
- **[P2] Bypass stale directory caches for session file resolution:** `src/infra/session-cost-usage.ts:305-321`
  - body: `resolveExistingUsageSessionFile` is a correctness path for reset/deleted transcript archives. Caching `readdirSync` results can return a now-deleted primary file or miss a new archive until the TTL expires, making usage/log lookups stale after transcript churn.
  - confidence: 0.84
- **[P1] Drop the already-shipped cost cache rewrite:** `src/gateway/server-methods/usage.ts:60-61`
  - body: Current main already bounds `costUsageCache` with a focused `COST_USAGE_CACHE_MAX = 256` helper that preserves in-flight coalescing. Replacing it with a new `LruMap(100)` duplicate would regress that merged behavior and create a second cache policy for the same problem.
  - confidence: 0.87
