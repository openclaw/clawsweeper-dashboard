Overall confidence: 0.9

Full review comments:

- **[P1] Preserve host-read media compatibility:** `src/media/web-media.ts:37-38`
  - body: Current main still accepts `localRoots: "any"`, and WhatsApp outbound media uses that path with `readFile` plus `hostReadCapability`. Removing that union and guard means existing host-read outbound media callers will fail instead of taking the supported compatibility path.
  - confidence: 0.9
- **[P1] Do not treat backpressure as write failure:** `src/terminal/stream-writer.ts:49-51`
  - body: `stream.write()` returning `false` is normal backpressure, not a broken pipe. `logs-cli` returns from the tail loop when the writer returns false, so this change can truncate healthy high-volume log output.
  - confidence: 0.86
- **[P1] Keep exec resolver environment opt-in:** `src/secrets/resolve.ts:701-710`
  - body: Current exec SecretRef resolution passes only `passEnv` and explicit provider `env` entries to the child. Adding baseline PATH/HOME/USER/LANG/TERM/TMPDIR/TMP/TEMP to every resolver broadens the credential execution context without a documented opt-in contract.
  - confidence: 0.88
- **[P2] Avoid stale directory entries in usage fallback:** `src/infra/session-cost-usage.ts:344`
  - body: `resolveExistingUsageSessionFile` is the fallback after the primary transcript file is missing. Caching that directory listing can keep returning a deleted primary entry and hide a reset/deleted archive until TTL expiry, regressing usage/log lookups after transcript churn.
  - confidence: 0.78
- **[P2] Do not silently remove TUI shell operators:** `src/tui/tui-local-shell.ts:109-118`
  - body: The existing local TUI command path intentionally runs through a shell after operator approval, so commands using pipes, redirection, globbing, or env expansion currently work. Switching to argv spawning needs either a compatibility path or an explicit tested policy change instead of silently changing operator commands.
  - confidence: 0.8
