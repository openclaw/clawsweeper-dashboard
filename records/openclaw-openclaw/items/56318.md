## Summary

Keep open: the branch contains useful hardening ideas, but it is not mergeable as-is because it source-reproducibly changes active runtime contracts, has a SecretRef exec environment concern, and lacks real behavior proof.

## What This Changes

The branch bundles security, memory-safety, and performance hardening across media loading, SecretRef exec resolution, gateway caches, session usage lookup, channel bridge handling, TUI shell execution, PDF/media fetch, stream writing, and tests.

## Best Possible Solution

Split this into smaller owner-scoped patches that preserve existing contracts or migrate all callers, with focused regression tests and redacted runtime proof before merge.

## Reproduction Assessment

Yes. The blocking regressions are source-reproducible by comparing current main with the PR diff: active media callers still accept `localRoots: "any"`, log tailing stops on false writer results, session archive lookup expects fresh directory reads, and TUI `!` commands intentionally use shell semantics.

## Solution Assessment

No. A broad bundled patch is not the best maintainable path because it changes several established contracts at once; the safer path is smaller hardening PRs that preserve or deliberately migrate each contract with tests and proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve localRoots any compatibility:** `src/media/web-media.ts:35-36`
  - body: This narrows `loadWebMedia`'s `localRoots` option and removes the `localRoots === "any"` branch, but current main still has public load options and outbound media paths that can pass `"any"` with a `readFile` override. Preserve that compatibility or migrate every caller in the same PR, otherwise affected media sends can fail typecheck or lose the intended host-read bypass.
  - confidence: 0.91
- **[P1] Keep backpressure separate from writer failure:** `src/terminal/stream-writer.ts:49-50`
  - body: `stream.write()` returning false is normal backpressure, not a closed stream. Since `openclaw logs` exits its output loop when the safe writer returns false, this change can truncate healthy high-volume log streams instead of waiting for drain or preserving the old success semantics.
  - confidence: 0.93
- **[P2] Read session directories fresh for archive fallback:** `src/infra/session-cost-usage.ts:344`
  - body: `resolveExistingUsageSessionFile` uses this listing to fall back from a missing active JSONL file to reset/deleted archives. Caching the directory for five seconds can return a stale active filename after rotation or deletion, causing usage detail and log lookups to miss until the TTL expires.
  - confidence: 0.86
- **[P2] Preserve local shell command semantics:** `src/tui/tui-local-shell.ts:116-117`
  - body: Local `!` commands are documented in current code as shell commands after explicit in-session approval. Parsing into argv with `shell: false` treats pipes, redirects, globs, and variable expansion as literal arguments, so existing operator workflows silently misbehave.
  - confidence: 0.82
