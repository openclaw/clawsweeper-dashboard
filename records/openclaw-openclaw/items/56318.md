## Summary

Keep open: the PR still has source-reproducible regressions on current main, lacks after-fix real behavior proof, and needs security/owner review before any subset should land.

## What This Changes

The branch bundles security, memory-safety, and performance hardening across media loading, SecretRef exec resolution, gateway caches, session usage lookup, channel bridge handling, TUI shell execution, PDF/media fetch, stream writing, and tests.

## Best Possible Solution

Refresh this as smaller owner-scoped PRs that preserve current media, TUI shell, log streaming, SecretRef exec, and session archive contracts while porting still-valid hardening with focused tests and real behavior proof.

## Reproduction Assessment

Yes. The blocking regressions are source-reproducible by comparing current main with the PR diff: active callers still use `localRoots: "any"`, log tailing stops when a writer returns false, session archive lookup expects fresh directory reads, and TUI `!` commands rely on shell semantics.

## Solution Assessment

No. A broad bundled patch is not the best way to solve this hardening work because it changes several established runtime/security contracts together; smaller current-main patches with compatibility tests, owner review, and proof are safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve localRoots any compatibility:** `src/media/web-media.ts:35-36`
  - body: This removes `"any"` from the `loadWebMedia` option type, but current main still has production callers, including WhatsApp outbound media, that pass `localRoots: "any"` with a `readFile` override. Preserve the compatibility type or migrate every caller in the same PR, otherwise affected plugin lanes can fail typecheck or lose the intended bypass behavior.
  - confidence: 0.91
- **[P1] Keep backpressure separate from writer failure:** `src/terminal/stream-writer.ts:49-50`
  - body: `stream.write()` returning false is normal backpressure, not a closed stream. Since `openclaw logs` exits its output loop when the safe writer returns false, this change can truncate healthy high-volume log streams instead of waiting for drain or preserving the old success semantics.
  - confidence: 0.93
- **[P2] Read session directories fresh for archive fallback:** `src/infra/session-cost-usage.ts:344`
  - body: `resolveExistingUsageSessionFile` uses this listing to fall back from a missing active JSONL file to reset/deleted archives. Caching the directory for five seconds can return a stale active filename after rotation or deletion, causing usage detail and log lookups to miss until the TTL expires.
  - confidence: 0.86
- **[P2] Preserve local shell command semantics:** `src/tui/tui-local-shell.ts:116-117`
  - body: Local `!` commands are documented in current code as shell commands after explicit in-session approval. Parsing into argv with `shell: false` treats pipes, redirects, globs, and `$VAR` expansion as literal arguments, so existing operator workflows silently misbehave.
  - confidence: 0.82
