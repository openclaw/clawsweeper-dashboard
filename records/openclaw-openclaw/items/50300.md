## Summary

Keep this PR open because current main still has the reported UTC transcript timestamp behavior and the PR is a relevant implementation, but it is not merge-ready: the wrapper still skips recovered headers after upstream repair rewrites, malformed persisted timestamps can make rewrite paths throw, and the user-facing fix lacks the required changelog entry.

## What This Changes

The PR adds a local-offset session timestamp formatter, wraps selected SessionManager transcript write/rewrite paths, updates manual session header creation paths, and adds regression tests for local-offset JSONL timestamps.

## Best Possible Solution

Land a rebased patch that centralizes local-offset formatting for OpenClaw-owned session transcript headers and best-effort normalizes SessionManager-created entries without changing transcript filenames or batch-migrating old files. The fix should cover current main's session/gateway write paths, preserve malformed persisted timestamps instead of throwing, and include targeted regression tests plus a changelog entry.

## Reproduction Assessment

Yes. Current main can be checked by creating or appending a session transcript and inspecting the JSONL: manual headers still call new Date().toISOString(), and append paths still use the dependency's UTC timestamp writer. The PR's recovery-path bug is reproducible from upstream SessionManager.open() repair behavior, which writes a fresh UTC header and sets flushed=true before the wrapper can normalize it.

## Solution Assessment

No. The direction is narrow and maintainable, but this PR is not the best version until it normalizes recovered headers despite flushed=true, treats invalid persisted timestamps as best-effort/no-op, rebases onto current main's session surfaces, and adds the required changelog entry.

## Review Findings

Full review comments:

- **[P2] Normalize recovered headers after repair rewrites:** `src/sessions/local-session-timestamps.ts:65-66`
  - body: When SessionManager.open() repairs an empty or corrupted transcript, upstream writes a fresh header and then sets flushed=true before this wrapper runs. This guard returns in that state, so the recovered header remains a UTC ...Z timestamp even though it is a newly written transcript header.
  - confidence: 0.9

  - body: normalizeAllEntries() and the eager header normalization path can pass existing JSONL timestamp strings into formatLocalSessionTimestamp(), and toDate() throws on invalid strings. A user-edited or partially corrupted timestamp would make branch/rewrite operations fail instead of leaving the old value unchanged.
  - confidence: 0.8

  - body: This changes user-visible session transcript JSONL timestamps, but the PR does not add a CHANGELOG.md entry. OpenClaw policy requires a changelog line for user-facing fixes, with appropriate contributor credit.
  - confidence: 0.92
