## Summary

Keep this PR open, but it should not merge as-is. Current main still has the reported UTC `...Z` session transcript behavior, so the PR is not obsolete; however the proposed wrapper still misses the upstream empty/corrupted-file recovery path and can throw on malformed persisted timestamps during normalization.

## What This Changes

The PR adds a local-offset session timestamp formatter, wraps selected `SessionManager` write paths, updates manual transcript header creation sites, and adds regression tests for local-offset session JSONL timestamps.

## Best Possible Solution

Update the PR so all new and recovered transcript headers use the same local-offset formatter, normalization of persisted entries is best-effort for malformed data, and the rebase preserves current main’s session import and append semantics.

## Reproduction Assessment

Yes. Current main can be verified statically from the header creation and `SessionManager.open(...).appendMessage(...)` paths, and the PR’s remaining recovery-path bug follows directly from upstream `setSessionFile()` setting `flushed = true` after rewriting an invalid file before the wrapper runs.

## Solution Assessment

No. The direction is narrow and appropriate, but the current implementation is not the best version until it handles recovered headers after `SessionManager.open()` repair and avoids throwing on invalid persisted timestamps.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Normalize headers after recovery rewrites:** `src/sessions/local-session-timestamps.ts:65`
  - body: When `SessionManager.open()` repairs an empty or corrupted transcript, upstream writes a fresh header and sets `flushed = true` before this wrapper runs. Because this guard returns on `flushed`, the eager normalization call skips that recovered header, leaving a UTC `...Z` timestamp in exactly the kind of newly written transcript this PR is trying to fix.
  - confidence: 0.89
- **[P3] Leave malformed timestamps unchanged:** `src/sessions/local-session-timestamps.ts:61`
  - body: `normalizeAllEntries()` can pass existing transcript timestamps into `formatLocalSessionTimestamp()`, which throws on invalid strings. A user-edited or partially corrupted JSONL timestamp can therefore break branch/fork rewrites; the wrapper should skip or preserve invalid persisted values instead of failing the session operation.
  - confidence: 0.78
