## Summary

Keep open. Current main still has UTC `...Z` transcript timestamp write paths, and this PR is useful source work, but the branch is stale against the current transcript append helper and still has correctness/security review issues before it should merge.

## What This Changes

The PR adds a local-offset timestamp formatter and SessionManager wrapper, updates manual session header creation paths, and adds regression tests for local-offset JSONL transcript timestamps.

## Best Possible Solution

Land a rebased or replacement patch that centralizes local-offset formatting in the current transcript append/header paths, preserves parentId/DAG behavior, treats malformed persisted timestamps as best-effort/no-op, and adds targeted tests plus a changelog entry.

## Reproduction Assessment

Yes, from source inspection. Current main still creates session transcript headers and appended JSONL entries with `new Date().toISOString()`, which produces UTC `...Z` strings rather than local-offset ISO strings.

## Solution Assessment

No. The PR's goal is right, but the best current fix should rebase onto the shared transcript append helper and avoid making malformed existing JSONL fatal; this branch also needs the required changelog entry.
