## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Resolve described tool ids for bridged calls:** `src/agents/tool-display-common.ts:401`
  - body: For the common bridge shape `const tool = await openclaw.tools.describe('openclaw:core:exec'); await openclaw.tools.call(tool.id, ...)`, this strips `.id` and returns the local variable name `tool`. `server-chat.ts` then projects the channel payload to `name: "tool"`, so Telegram still shows a generic Tool label instead of the native inner tool; track `describe()` bindings or skip projection when the target is only a variable.
  - confidence: 0.86
- **[P2] Skip aborted and error turns during file repair:** `src/agents/session-file-repair.ts:203-207`
  - body: `isCodeModeToolCallRepairCandidate` matches every `openai-codex-responses` assistant entry, so `insertMissingCodeModeToolResults` can synthesize `aborted` tool results for assistant turns whose own `stopReason` is `error` or `aborted`. The live guard already avoids synthesizing for those turns because incomplete tool-use blocks can make replay worse; disk repair should use the same gate and cover it with a regression test.
  - confidence: 0.75
