## Summary

Keep open: current main still has native Codex reasoning/plan/progress signals, but the Control UI only renders thinking that is already embedded in assistant message content and ignores non-tool Codex plan/thinking event streams. The related Gateway reasoning-stream PR at https://github.com/openclaw/openclaw/pull/79687 is producer-side only and explicitly leaves Control UI rendering as follow-up work.

## What This Changes

The issue requests safe visible native Codex thinking/progress state in the Control UI instead of only the running dots.

## Best Possible Solution

Add a Control UI-native Codex activity surface that consumes approved safe plan/tool/lifecycle/progress events, or explicitly states when visible reasoning is unavailable for the selected runtime.

## Reproduction Assessment

Yes at source level, but not from a live browser run. Current main emits native Codex reasoning/plan signals while the Control UI only renders thinking from assistant message content and ignores non-tool plan/thinking streams.

## Solution Assessment

Unclear without maintainer UX/privacy judgment. The safest direction is not to map raw reasoning deltas blindly into .chat-thinking, but to render approved Codex progress/plan summaries or a clear unsupported-state notice.
