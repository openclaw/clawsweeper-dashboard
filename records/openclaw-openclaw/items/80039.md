## Summary

Keep open: current main still has the Control UI gap described in the issue. Native Codex app-server code projects reasoning and plan/progress signals, but the browser chat surface only renders thinking when it is embedded in assistant message content or legacy tags, and the live agent-event handler still ignores Codex plan streams. A fresh follow-up asks whether this is planned, so this needs maintainer UX/privacy direction rather than cleanup closure.

## What This Changes

The issue requests a safe visible thinking/progress surface in Control UI for native Codex runtime sessions when reasoning is enabled.

## Best Possible Solution

Add a Control UI-native Codex activity surface that consumes approved app-server plan/tool/lifecycle progress events and either renders safe user-visible summaries or explicitly states that visible reasoning is unavailable for this runtime.

## Reproduction Assessment

Yes, at source level: current main produces native Codex reasoning/plan signals while the Control UI only renders thinking from assistant message content and filters out non-tool plan events. I did not run a live browser session, so this is not marked as a full live reproduction.

## Solution Assessment

Unclear without maintainer UX/privacy guidance. The narrow maintainable direction is to render safe Codex plan/tool/lifecycle activity, not raw private reasoning, and to show an explicit unavailable state when the runtime cannot provide approved visible summaries.
