## Summary

Keep open. Current main still has native Codex app-server plan/reasoning signals, but the Control UI only renders thinking from assistant message content and drops non-tool plan/progress agent streams; the related tool-card fix does not cover this Codex reasoning/progress UX.

## What This Changes

Request safe visible Codex reasoning/progress, or an explicit unavailable status, in Control UI when the native Codex runtime is active.

## Best Possible Solution

Define a small approved Codex activity surface for Control UI that consumes safe plan/tool/lifecycle signals and clearly avoids raw private chain-of-thought.

## Solution Assessment
