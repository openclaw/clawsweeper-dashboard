## Summary

Keep open: current main and v2026.6.8 still emit native Codex plan/item/reasoning signals, but the Control UI only routes compaction, fallback, lifecycle, and tool streams into visible status/activity surfaces, so the requested safe native Codex thinking/progress UX remains unresolved.

## What This Changes

The issue asks OpenClaw to show safe native Codex thinking or progress in the Control UI instead of only the active-work indicator during native Codex runs.

## Best Possible Solution

Ship a reviewed current-main implementation that maps only safe native Codex lifecycle, plan/item status, and tool progress into a compact Control UI status or Activity surface while excluding raw reasoning text, tool arguments, and sensitive results.

## Reproduction Assessment

Yes, at source level: current main emits native Codex plan/item/reasoning signals, but the Control UI drops non-tool streams before Activity/status rendering. I did not run a live native Codex Control UI session in this read-only review.

## Solution Assessment

No current implementation is sufficient. The best path is a maintainer-reviewed safe-progress mapping rather than reviving the stale closed branch or relying on the narrower final-answer-candidate PR.
