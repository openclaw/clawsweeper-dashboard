## Summary

Keep open: current main still has a real gap between native Codex plan/reasoning events and the Control UI surfaces that can render them.

## What This Changes

Request the Control UI to show safe native Codex thinking/progress activity, or clearly state that visible reasoning is unavailable for that runtime.

## Best Possible Solution

Add a Control UI-native Codex activity surface that consumes approved safe plan/tool/lifecycle signals and falls back to an explicit unavailable state when visible reasoning cannot be shown.

## Reproduction Assessment

Yes at source level: native Codex emits reasoning and plan signals, while the Control UI only renders thinking embedded in assistant message content and ignores non-tool plan streams. I did not run a live browser session in this read-only review.

## Solution Assessment

Unclear until maintainers choose the safe UX/privacy boundary; the narrow maintainable direction is to render approved Codex activity/progress summaries, not raw private chain-of-thought.
