## Summary

Keep open: current main still has the reported native Codex Control UI gap. Codex app-server reasoning and plan signals exist, but the browser chat path only renders reasoning already embedded in assistant message content and ignores Codex plan/progress agent events.

## What This Changes

Request that native Codex runs expose safe reasoning/progress state in Control UI or clearly state that visible reasoning is unavailable.

## Best Possible Solution

## Solution Assessment
