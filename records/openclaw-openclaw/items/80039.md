## Summary

Keep open: current main still drops native Codex plan/progress streams before they can become a Control UI thinking/progress surface, and choosing which Codex signals to display safely needs maintainer UX/privacy direction.

## What This Changes

The issue asks the native Codex runtime to surface safe visible reasoning or progress in the Control UI instead of only the running dots.

## Best Possible Solution

Add a Control UI-native Codex activity surface that consumes approved plan/tool/lifecycle/reasoning-summary signals, or explicitly tells users that visible reasoning is unavailable for the selected runtime.

## Reproduction Assessment

Yes, at source level: native Codex projects reasoning and plan signals, while the Control UI only renders thinking from assistant message content and ignores non-tool plan streams. I did not run a live browser native-Codex session in this read-only review.

## Solution Assessment

Unclear until maintainers choose the safe UX contract; directly rendering raw reasoning would not be the right default. The maintainable path is an approved activity/progress surface or an explicit unsupported-status message.
