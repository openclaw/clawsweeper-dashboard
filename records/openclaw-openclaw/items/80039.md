## Summary

Keep open: current main still emits native Codex plan/reasoning activity, but the Control UI only renders thinking that is already embedded in assistant message content and drops non-tool agent-event streams such as Codex plan updates. The issue is valid, but the fix needs maintainer UX/privacy direction on which Codex app-server events are safe and useful to surface.

## What This Changes

The issue requests a Control UI surface for safe native Codex runtime thinking/progress activity, or an explicit indication that visible reasoning is unavailable for that runtime.

## Best Possible Solution

Add a Control UI-native Codex activity surface that consumes approved plan/tool/lifecycle progress events, or clearly says visible reasoning is unavailable for the selected runtime while keeping raw private reasoning out of the UI.

## Reproduction Assessment

Yes at source level: native Codex plan/reasoning signals are produced, while the Control UI handler ignores non-tool streams and the chat renderer only shows thinking embedded in assistant message content. I did not run a live browser session, so this is not marked as a live reproduced bug.

## Solution Assessment

Unclear until maintainers choose the safe UX boundary. The requested direction is reasonable, but the maintainable fix should surface approved activity/progress events or an explicit unsupported state rather than piping raw reasoning text into the existing thinking box.
