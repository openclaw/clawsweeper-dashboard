## Summary

Keep open: current main still has the reported gap, because native Codex emits reasoning/plan signals while Control UI only renders thinking from assistant message content and drops non-tool progress streams.

## What This Changes

Request Control UI to surface safe native Codex reasoning/progress state, or clearly say visible reasoning is unavailable, instead of only showing the running indicator.

## Best Possible Solution

Add a Control UI-native Codex activity surface that consumes approved safe plan/tool/lifecycle signals and explicitly explains unsupported visible reasoning instead of exposing raw reasoning text.

## Solution Assessment

Unclear until maintainers choose the UX/privacy policy for Codex progress. The narrowest maintainable path is to render approved safe activity summaries or clearly state that visible reasoning is unavailable for the selected runtime.
