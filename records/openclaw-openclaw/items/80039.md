## Summary

Keep open: current main has native Codex reasoning/plan/progress signals, but the Control UI path still drops the non-tool Codex plan/progress streams and only renders thinking when it is already embedded in assistant message content.

## What This Changes

Request the Control UI to show safe native Codex runtime thinking/progress state, or clearly explain when visible reasoning is unavailable.

## Best Possible Solution

Add a Control UI-native Codex activity surface that consumes safe plan/tool/lifecycle events and either renders approved visible summaries or explicitly states that visible reasoning is unavailable for the selected runtime.

## Solution Assessment

Unclear without maintainer UX/privacy judgment; the narrowest safe path is likely to show Codex plan/tool/lifecycle activity and avoid raw reasoning text unless the app-server contract marks it safe for users.
