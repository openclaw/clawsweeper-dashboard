## Summary

Keep open: current main still has the reported Control UI gap for native Codex progress visibility. Codex app-server plan/reasoning signals exist, but the Control UI thinking box only renders assistant-message thinking blocks and the browser agent-event handler drops non-tool plan/thinking streams.

## What This Changes

The issue requests that native Codex runtime sessions show safe visible thinking/progress in the Control UI, or clearly state when that output is unavailable.

## Best Possible Solution

Define a Control UI-native Codex activity surface that consumes approved plan/tool/lifecycle/thinking events, or clearly marks visible reasoning as unavailable for native Codex when no safe stream exists.

## Reproduction Assessment

Yes at source level: current main emits Codex plan/reasoning signals, but the Control UI renders thinking only from assistant message content and ignores non-tool agent streams. I did not run a live browser session in this read-only review.

## Solution Assessment

Unclear until maintainers choose the privacy/UX boundary; the narrowest maintainable path is likely an approved activity panel using safe Codex plan/tool/lifecycle summaries, with an explicit unavailable state when reasoning cannot be shown safely.
