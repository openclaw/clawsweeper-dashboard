## Summary

Keep open: current main still has native Codex progress/reasoning signals that do not reach the Control UI thinking surface, and the safe UX needs maintainer product/privacy direction before implementation.

## What This Changes

Request that native Codex app-server reasoning/progress signals be surfaced in the Control UI, or that the UI/status clearly says visible reasoning is unavailable for that runtime.

## Best Possible Solution

Add a Control UI activity/progress surface for native Codex that consumes approved safe plan/lifecycle/tool signals and explicitly states when visible reasoning is unavailable.

## Reproduction Assessment

Yes at source level: native Codex emits plan/reasoning events, but the Control UI handler ignores plan streams and the thinking renderer only reads embedded assistant thinking blocks. A live browser run was not performed in this read-only sweep.

## Solution Assessment

Unclear until maintainers choose the safe UX: mapping raw reasoning directly is not the best default, while an approved activity/progress panel or explicit unavailable state would be narrower and safer.
