## Summary

Keep open: current main still lacks the requested server-side WebChat dictation path, and open PR #76021 explicitly fixes this issue but has not merged, so the issue should wait for that implementation review and landing.

## What This Changes

Request server-side STT/dictation for WebChat microphone input using browser audio capture and configured `tools.media.audio` instead of browser Web Speech APIs.

## Best Possible Solution

Review and land #76021, or an equivalent authenticated and size-bounded batch dictation implementation, then let GitHub close this issue after merge.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is source inspection showing current main has Talk and audio attachment flows but no MediaRecorder-style dictation control or Gateway transcription RPC.

## Solution Assessment

Yes, the active PR direction is the narrow maintainable solution: a dedicated draft-transcription RPC avoids overloading `chat.send`, keeps provider credentials server-side, and can enforce upload limits before returning text to the composer.
