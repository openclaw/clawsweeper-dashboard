## Summary

Keep open: current main still lacks the server-side WebChat dictation RPC/UI path, and open PR #76021 explicitly fixes this issue but has not merged.

## What This Changes

Add a WebChat voice-input path that records browser audio, sends it to Gateway, transcribes through configured `tools.media.audio` backends, and inserts the transcript into the draft.

## Best Possible Solution

Review and land #76021, or an equivalent authenticated and size-bounded `chat.transcribeAudio` plus MediaRecorder composer path, then let the merge close this issue.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is source inspection on current main: the composer exposes attachment and Talk controls only, and Gateway advertises no draft-transcription RPC.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable fix: a dedicated authenticated transcription RPC can enforce upload limits and return draft text without overloading `chat.send` or starting an agent run.
