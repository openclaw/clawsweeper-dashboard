## Summary

Keep open. Current main still lacks the requested one-shot server-side STT/dictation path for WebChat: users can attach audio files to an agent turn and use realtime Talk, but there is no MediaRecorder capture flow or Gateway method that transcribes audio through `tools.media.audio` and returns text to the composer draft.

## What This Changes

Add a Control UI/WebChat server-side voice dictation option that records browser microphone audio and transcribes it through the configured `tools.media.audio` backend.

## Best Possible Solution

Implement a focused batch dictation path for Control UI/WebChat: capture completed microphone audio with `getUserMedia` plus `MediaRecorder`, send it through an authenticated and size-bounded Gateway RPC, run the existing media-understanding audio transcription path, and append the returned transcript to the composer draft without starting an agent turn or conflating it with realtime Talk.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is source inspection on current main showing no MediaRecorder/server-dictation UI or Gateway transcription RPC, while audio attachments and realtime Talk cover different workflows.

## Solution Assessment

Yes, the requested direction is the narrow maintainable solution. A dedicated batch transcription method is safer than reusing `chat.send` because it can return draft text while keeping upload limits, auth checks, and provider credentials on the Gateway.
