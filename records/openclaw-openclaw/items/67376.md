  - body: The scoped token only carries `sessionKey`, `accountId`, `messageProvider`, and `senderIsOwner`, but current main also feeds session id, channel/thread/message ids, inbound audio, event kind, delivery mode, and explicit-target policy into loopback tool resolution.   - confidence: 0.96
- **[P1] Guard runtime clearing by the owning server:** `src/gateway/mcp-http.loopback-runtime.ts:30-32`
  - body: `clearActiveMcpLoopbackRuntime()` now clears the active runtime and token table unconditionally. If an old server close races with a newly started loopback server, the old close can wipe the new server's registered tokens and cause fresh clients to get 401s.
  - confidence: 0.94
- **[P2] Revoke tokens even when backend cleanup throws:** `src/agents/cli-runner.ts:124-126`
  - body: The token is revoked after `preparedBackend.cleanup`, so any cleanup exception skips `unregisterMcpLoopbackToken` and leaves a scoped bearer registered after the run teardown path.
  - confidence: 0.9
- **[P2] Revoke registered tokens on later prepare failures:** `src/agents/cli-runner/prepare.ts:173-177`
  - body: The catch only unregisters when `prepareCliBundleMcpConfig` fails. Later awaits, such as docs/reference resolution or skill prompt preparation, can throw before the cleanup handle is returned and leave the token in the process registry.
