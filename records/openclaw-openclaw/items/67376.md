## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Guard runtime clearing to the closing server:** `src/gateway/mcp-http.ts:117`
  - body: `clearActiveMcpLoopbackRuntime()` clears the global runtime and token table for any closing server. Because `closeMcpLoopbackServer()` drops `activeMcpLoopbackServer` before awaiting close, a concurrent restart can install a new runtime that the old close callback then wipes, causing fresh backend tokens to 401.
  - confidence: 0.9
- **[P2] Revoke tokens even when backend cleanup fails:** `src/agents/cli-runner.ts:124-126`
  - body: Token revocation runs after `await context.preparedBackend.cleanup?.()`, so a cleanup exception skips `unregisterMcpLoopbackToken` and leaves a bearer registered after run teardown. Put revocation in its own finally path so cleanup failures cannot preserve authorization.
  - confidence: 0.86
- **[P2] Revoke tokens on later prepare failures:** `src/agents/cli-runner/prepare.ts:173-177`
  - body: The token is registered before later preparation work, but only the bundle-MCP config failure path unregisters it. If docs path, skills, or system prompt preparation throws before the prepared context is returned, the scoped token remains registered without a backend owner.
  - confidence: 0.82
