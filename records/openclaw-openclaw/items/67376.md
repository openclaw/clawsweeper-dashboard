  - body: `clearActiveMcpLoopbackRuntime()` clears the global runtime and token table without proving the closing server still owns them. Because `closeMcpLoopbackServer()` clears the active server before awaiting close, a concurrent new server can be started and then have its fresh tokens wiped by the old close callback, producing 401s for valid clients.
  - confidence: 0.9

  - body: The token is unregistered only after `preparedBackend.cleanup` resolves. If backend cleanup throws during run teardown, the bearer remains registered past the backend lifetime, keeping stale authorization alive and growing the token scan set.
  - confidence: 0.9
- **[P2] Revoke issued tokens on all prepare failures:** `src/agents/cli-runner/prepare.ts:173-177`
  - body: This catch only covers `prepareCliBundleMcpConfig`. Later preparation failures before the returned cleanup is installed still leave the newly registered token in the process registry, so the failure path needs one cleanup guard around all post-registration work.
  - confidence: 0.87
