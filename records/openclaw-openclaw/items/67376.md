  - confidence: 0.91
- **[P1] Guard runtime clearing to the closing server:** `src/gateway/mcp-http.ts:117`
  - body: closeMcpLoopbackServer clears activeMcpLoopbackServer before awaiting close(), and this callback then clears the global runtime/token table unconditionally. A concurrent ensureMcpLoopbackServer can start a fresh server in that window, only for the old close callback to wipe the new tokens and make valid fresh backends get 401s.
  - confidence: 0.9

  - body: Token revocation runs after awaiting preparedBackend.cleanup. If cleanup throws, unregisterMcpLoopbackToken is skipped, leaving the bearer token registered past run teardown and keeping stale authorization alive until the loopback server closes.
  - confidence: 0.88
- **[P2] Revoke tokens on every prepare failure:** `src/agents/cli-runner/prepare.ts:173-176`
