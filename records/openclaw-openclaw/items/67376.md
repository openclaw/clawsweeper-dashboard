## Review Findings

Overall confidence: 0.9

Full review comments:

- **[P1] Guard runtime clearing by server identity:** `src/gateway/mcp-http.loopback-runtime.ts:30-32`
  - body: `clearActiveMcpLoopbackRuntime()` clears the global runtime and token table unconditionally. Because `ensureMcpLoopbackServer()` can start a new server while an old server is still closing, the old close callback can wipe the new server's registered tokens and make fresh backends receive 401s.
  - confidence: 0.9
- **[P2] Revoke tokens even when cleanup fails:** `src/agents/cli-runner.ts:125-126`
  - body: The new revocation runs only after `preparedBackend.cleanup` resolves. If cleanup throws, the scoped bearer token remains registered after run teardown, preserving stale authorization and growing the token scan set.
  - confidence: 0.86
- **[P2] Clean up tokens on every prepare failure:** `src/agents/cli-runner/prepare.ts:173-178`
  - body: The registration is only guarded by a `.catch()` on `prepareCliBundleMcpConfig`; later preparation failures can leave the token registered until server shutdown. Wrap all post-registration preparation work so every throw unregisters the token.
  - confidence: 0.84
