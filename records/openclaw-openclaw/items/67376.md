Land a rebased, security-reviewed implementation that binds the full current MCP loopback context tuple to server-side token identity, removes mutable scope headers only with explicit upgrade proof, and preserves safe token cleanup.

  - body: This branch registers only sessionKey/accountId/messageProvider/senderIsOwner. Current main also carries current channel/thread/message/audio, inbound event, source reply delivery mode, and explicit-target context through the same loopback path, so rebasing this as-is would leave spoofable scope fields in place.
  - confidence: 0.9
- **[P1] Guard runtime clearing to the server being closed:** `src/gateway/mcp-http.loopback-runtime.ts:30-31`
  - body: clearActiveMcpLoopbackRuntime() clears the global runtime and token table without checking which server is closing. A concurrent restart can register a new server, then the old close callback wipes the fresh tokens and valid requests start returning 401.
  - confidence: 0.87
- **[P2] Revoke the loopback token if backend cleanup throws:** `src/agents/cli-runner.ts:122-124`
  - body: runPreparedCliAgent awaits preparedBackend.cleanup before unregistering the token. If cleanup throws, the unregister call is skipped and the bearer stays registered past run teardown.
  - confidence: 0.86
- **[P2] Clean up registered tokens on all prepare failures:** `src/agents/cli-runner/prepare.ts:171-177`
  - body: The catch only wraps prepareCliBundleMcpConfig. If later preparation work throws after the token is registered, the token remains live even though no backend should own it.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The patch changes the MCP loopback authorization boundary but is stale, incomplete for current main’s scope tuple, and has token lifecycle defects.

Concerns:

- **[high] Incomplete scope binding leaves current fields mutable:** `src/gateway/mcp-http.loopback-runtime.ts:8`
  - body: The PR scopes only four fields to the token while current main routes additional channel, message, inbound-event, reply-delivery, and explicit-target context from headers into tool resolution.
  - confidence: 0.9
- **[medium] Unconditional registry clearing can revoke fresh tokens:** `src/gateway/mcp-http.loopback-runtime.ts:30`
