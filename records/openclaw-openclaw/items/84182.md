## Summary

Keep open. Current main has reusable Discord component registry support, but the portable presentation button contract, message tool schema, normalizer, and Discord presentation mapper do not carry per-button reusable intent, so the reported gap is still present.

## What This Changes

Add a per-button reusable flag to portable message presentations and preserve it through Discord component generation.

## Best Possible Solution

Extend the generic presentation button contract with `reusable?: boolean`, preserve it through normalization and channel-limit adaptation, map it to Discord component entries, and keep absent flags one-shot by default.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a presentation button cannot declare `reusable`, normalization would not preserve it, and the Discord mapper does not copy it into component entries. I did not run tests because this review was read-only.

## Solution Assessment

Yes, the narrow maintainable fix is to add a generic button-level flag and map it into the existing Discord entry-level reusable support. Using message-level reusable or Discord-specific `components` payloads would keep the portable contract gap open.
