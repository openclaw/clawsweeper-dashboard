## Summary

Keep open. Current main still lacks a portable per-button `reusable` field and the Discord presentation mapper cannot pass that intent into component entries; the issue should stay open while the linked implementation PR is reviewed.

## What This Changes

Add an opt-in reusable flag to portable presentation buttons and preserve it through Discord component registration.

## Best Possible Solution

Review and land the linked PR if maintainers accept the new public field, keeping today’s one-shot default unless `reusable: true` is explicitly set.

## Reproduction Assessment

Yes. Source inspection against current main shows no `reusable` field in the presentation type/schema, no normalizer preservation, and no Discord mapper propagation into button specs.

## Solution Assessment

Yes. The narrow maintainable path is an opt-in generic button field mapped into the existing Discord reusable-entry support; the open linked PR is the implementation path to review.
