## Summary

Keep open. Current main still lacks per-button `reusable` in the portable presentation contract and Discord mapper, and the linked open implementation PR is the right path to resolve it.

## What This Changes

The issue requests a portable `reusable` flag on message presentation buttons and Discord mapping so repeated button callbacks remain available after first use.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/84187; once it merges, close this issue as implemented by that PR.

## Reproduction Assessment

Yes. Source inspection shows the button type, tool schema, and Discord presentation mapper cannot carry a per-button `reusable` flag while the Discord registry already honors reusable entries.

## Solution Assessment

Yes, the narrow maintainable path is to add an opt-in generic button field and propagate it into the existing Discord component entry mechanism; Discord-specific payloads or message-level defaults would leave the portable contract gap open.
