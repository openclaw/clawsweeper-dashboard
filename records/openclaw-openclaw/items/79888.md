## Summary

Close as implemented: current main already stores the configured subagent model on the child session before the first turn, so the parent session's stored model override does not win for `sessions_spawn` spawns.

## What This Changes

The issue asks for `sessions_spawn` without a `model` parameter to use `agents.defaults.subagents.model.primary` before inheriting the parent session model.

## Best Possible Solution

Keep the current child-session model override persistence; any new report should include a redacted session-store entry and exact installed package/dist hash if an official release still behaves differently.

## Reproduction Assessment

No current-main failing reproduction was established. Source and published package inspection show `sessions_spawn` now writes the configured subagent model onto the child session before the first turn, so the reported parent-model inheritance path should not reproduce.

## Solution Assessment

Yes. Persisting the configured subagent model as a direct child-session override is the narrowest maintainable fix because `resolveStoredModelOverride()` already prefers direct child overrides before parent fallback.
