## Summary

Keep open. Current main still has the Slack draft-preview identity gap: normal Slack delivery supports custom agent identity, but the draft stream constructor and its initial preview send do not accept or forward identity. The PR contains the right central fix idea, but it is stale against current `extensions/slack` paths and includes unrelated changes, so it should be ported rather than merged as-is.

## What This Changes

The PR adds Slack identity plumbing to the draft stream initial send path, adds a draft-stream identity regression test, and also changes daemon cleanup hints plus Slack draft duplicate-race fallback handling.

## Best Possible Solution

Land a focused current-main Slack plugin fix: add optional `SlackSendIdentity` support to `extensions/slack/src/draft-stream.ts`, pass the resolved `slackIdentity` from `dispatchPreparedSlackMessage`, forward identity only on the initial draft `sendMessageSlack` call, and cover that path with a focused regression test.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists: with Slack draft preview enabled and an agent identity configured, dispatch resolves `slackIdentity`, but the draft stream has no identity parameter and its first send cannot include the custom username/icon options.

## Solution Assessment

No, not as currently shaped. The identity plumbing itself is the right narrow fix, but the best maintainable solution is a current-main port under `extensions/slack` that drops the unrelated daemon cleanup and already-separate duplicate-race changes.
