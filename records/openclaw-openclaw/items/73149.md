Keep open because the narrow no-op sentinel fix is still represented by an open conflicting closing PR, while current main still lacks the broader cron/job-defined no-op delivery contract requested here.

## What This Changes

Request runtime-level suppression for cron and heartbeat no-op outputs so sentinel acknowledgements and short no-op summaries do not reach user-visible channels.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/73785 or an equivalent narrow sentinel fix, then separately scope a structured cron noReply/suppressDelivery contract.

## Reproduction Assessment

## Solution Assessment

Unclear for the full request. Exact sentinel suppression is the right narrow fix, but adding general job-defined no-op delivery semantics should be a deliberate cron/API contract decision.
