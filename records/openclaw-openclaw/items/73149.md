## Summary

Keep open. Current main covers structured heartbeat quiet outcomes and standard `HEARTBEAT_OK`/`NO_REPLY` suppression, but it still lacks the requested general cron/job-defined no-op contract and #73785 remains the open closing PR for the focused sentinel work.

## What This Changes

Request runtime-level suppression for cron/heartbeat no-op outputs, including structured no-reply delivery and job-defined sentinel codes.

## Best Possible Solution

Land #73785 or an equivalent narrow sentinel fix, then make a maintainer decision on an explicit cron `noReply`/`suppressDelivery` API for job-defined no-op results.

## Reproduction Assessment

Yes by source inspection for the remaining custom-sentinel/no structured contract path: `NO_NEW_AUDIO` or `SESSION_WATCHDOG_OK` are not recognized by current filters. The duplicate-output example is not reproducible without run identifiers and timestamps.

## Solution Assessment

Unclear for the final shape. The focused sentinel PR is the narrow repair for visible sentinel leaks, but a first-class cron no-reply contract is a public API/config decision rather than a purely mechanical bug fix.
