## Summary

Keep open. Current main has heartbeat-specific quiet handling and `NO_REPLY` cron suppression, but it still does not provide the broader cron/job-defined no-op contract for custom sentinels requested here; #73785 remains the open closing PR and should be resolved first.

## What This Changes

The issue requests first-class suppression for cron/heartbeat no-op outputs so sentinel or no-op results are not delivered to user-visible channels.

## Best Possible Solution

Keep this as the canonical tracker until #73785 or an equivalent focused fix lands, then split any remaining structured cron no-op contract into a narrower follow-up if needed.
