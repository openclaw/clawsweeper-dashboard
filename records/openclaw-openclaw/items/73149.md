## Summary

Keep open. Current main has heartbeat-specific ack suppression, but the broader job-defined no-op sentinel and structured silent-delivery contract requested here is still not implemented, and open PR #73785 is explicitly paired with this issue.

## What This Changes

Cron and heartbeat no-op runs should suppress sentinel-only or successful no-op output instead of posting visible channel messages.

## Best Possible Solution

A centralized runtime delivery gate should consume exact job-defined no-op results before channel dispatch, with a structured cron/job silent-result contract for future non-heartbeat sentinels while preserving real alerts and media.

## Reproduction Assessment

Yes for the primary suppression-gap class: the issue discussion gives a concrete delivery-path trace, and current main still shows token-specific rather than job-defined no-op handling. The duplicate-message example is unclear without gateway timestamps and clientRunId or sourceRunId.
