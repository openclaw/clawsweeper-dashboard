## Summary

Keep open. Current main has heartbeat-specific ack suppression, but the broader job-defined no-op sentinel or structured no-reply contract requested here is still not implemented, and PR #73785 remains open and explicitly paired with this issue.

## What This Changes

The issue asks OpenClaw to suppress user-visible delivery for cron and heartbeat no-op responses, including exact sentinel tokens and possibly a structured no-reply flag.

## Best Possible Solution

Keep this issue as the canonical tracker until PR #73785 or an equivalent narrow fix lands. The maintainable end state is centralized delivery-boundary suppression for known no-op sentinels, with a structured no-reply job-result contract handled only if maintainers choose the larger API direction.
