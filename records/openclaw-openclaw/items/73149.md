## Summary

Keep open. Current main covers the narrow heartbeat ack and shared `NO_REPLY` paths, but it still does not provide the broader job-defined no-op sentinel or structured suppress-delivery contract requested here, and PR #73785 is open, unmerged, and explicitly paired with this issue.

## What This Changes

Request a runtime-level silent/no-op delivery mode for cron, heartbeat, and system-event runs so no-op outputs such as `HEARTBEAT_OK`, `NO_NEW_AUDIO`, and `SESSION_WATCHDOG_OK` do not become visible channel messages.

## Best Possible Solution

Keep this issue as the canonical tracker while PR #73785 is reviewed. The desired end state is centralized delivery-boundary suppression for known exact no-op sentinels, with a separate structured no-reply job result contract only if maintainers choose the broader cron/system-event API direction.
