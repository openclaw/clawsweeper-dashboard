## What This Changes

The issue reports that Telegram `/status` rich-message output collapses newline-separated status rows into one run-on line when `channels.telegram.richMessages` is enabled.

## Best Possible Solution

Keep the shared Telegram rich HTML newline materialization path, ship it in the next release, and have a maintainer close this issue once satisfied that `/status` is covered.

## Reproduction Assessment

Yes at source level: `/status` builds newline-delimited rows, and the affected rich-message path previously rendered raw newlines through Telegram rich HTML where they collapse. I did not run an independent live Telegram reproduction in this read-only pass.

## Solution Assessment

Yes: the current shared `prepareTelegramRichHtml` materialization layer is the narrow maintainable fix because it covers markdown and explicit HTML rich-message sends while preserving code/pre/math content and leaving rich-disabled sends alone.
