## Summary

Keep open for maintainer merge review: current main still has the stuck-running CLI task behavior, and this PR appears to fix it narrowly with regression coverage, docs, changelog, and no blocking review findings.

## What This Changes

The branch makes CLI task liveness depend on active run context, removes dead session chat-type caching, updates audit/docs/changelog expectations, and adds regression coverage for stale exec-approval-followup tasks.

## Best Possible Solution

Land this PR after maintainer approval so the linked bug closes with the run-context contract, regression test, docs, and changelog aligned.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: a stale CLI task with inactive `sourceId`/`runId`, `childSessionKey: "agent:main:main"`, and an existing persistent session row remains backed instead of becoming `lost`.

## Solution Assessment

Yes. Returning `hasActiveCliRun(task)` for all CLI tasks is the narrow maintainable fix because current CLI task producers register run contexts, while session rows represent conversation state rather than run liveness.
