## What This Changes

The PR makes CLI task backing depend on the live agent run context, updates task audit expectations/docs/changelog, and adds regression coverage for the `exec-approval-followup` persistent-session case.

## Best Possible Solution

Land this PR after maintainer review, preserving CLI task reconciliation as live-run-context based with the added regression coverage and user-facing docs/changelog alignment.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a stale `runtime: "cli"` task with inactive `sourceId`/`runId`, `childSessionKey: "agent:main:main"`, and an existing persistent session row remains backed on current main.

## Solution Assessment

Yes. Returning `hasActiveCliRun(task)` for all CLI tasks is the narrow maintainable fix because current CLI task producers track run IDs, while session rows are conversation state rather than task-run liveness.
