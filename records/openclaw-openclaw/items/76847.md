## Summary

Keep open for maintainer merge review: current main still has the CLI task liveness fallthrough, and the PR is a narrow fix with regression/docs/changelog coverage and no blocking review findings found.

## What This Changes

The PR makes all CLI task liveness depend on the owning active agent run context, then updates task maintenance tests, the task audit expectation, docs, and changelog for the exec-approval-followup stuck-running case.

## Best Possible Solution

Land this PR after maintainer approval, preserving the run-context liveness contract and the new regression coverage for persistent-session CLI tasks.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an inactive stale CLI task with `sourceId`/`runId` set to an ended follow-up run and `childSessionKey: "agent:main:main"` can still be treated as backed by the persistent session row on current main.

## Solution Assessment

Yes. Returning `hasActiveCliRun(task)` for all CLI tasks is the narrow maintainable fix because the registered run context is the authoritative live-run signal, while session rows are conversation state and may persist after the task run ends.
