## Summary

Keep open: this PR targets a real documented Windows completion/time-format bug, but the completion portion is now patched in the stale CLI helper while current main routes installs through completion-runtime, and the external PR still lacks native Windows after-fix proof.

## What This Changes

The PR changes Windows time-format detection to use the PowerShell resolver and adds PowerShell completion install/profile handling plus focused tests in the CLI completion helper.

## Best Possible Solution

Retarget the PowerShell install/source-line/reload handling and tests to src/cli/completion-runtime.ts, keep the date-time resolver change, and coordinate profile-path messaging with https://github.com/openclaw/openclaw/issues/44296.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: docs and CLI accept powershell, but the live runtime install helper falls through to the unsupported branch while date-time detection still invokes hard-coded powershell.

## Solution Assessment

No, not as currently patched. The date-time resolver change is the right direction, but completion install/source/reload behavior must move into completion-runtime.ts because CLI, setup, doctor, and update flows use that helper.
