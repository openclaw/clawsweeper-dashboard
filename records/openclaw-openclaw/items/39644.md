## Summary

Keep open: the Windows bugs still exist on current main, but this stale PR no longer patches the active completion install path after the runtime split and still lacks real Windows CLI/profile proof.

## What This Changes

The PR changes Windows time-format PowerShell selection, adds PowerShell completion install/profile dot-source handling, and adds PowerShell completion tests.

## Best Possible Solution

Port the installer/profile changes into `src/cli/completion-runtime.ts`, keep `src/agents/date-time.ts` using `resolvePowerShellPath()`, add focused runtime tests, and require redacted Windows PowerShell install proof before merge.

## Reproduction Assessment

Yes, by source inspection: docs advertise PowerShell install, current runtime accepts and caches `.ps1` completions but falls through to the unsupported installer branch, and date-time detection invokes literal `powershell`. I did not run a live Windows install in this read-only review.

## Solution Assessment

No. The bug boundary is right, but the branch edits the old completion helper layout; the maintainable fix is to port the behavior to `completion-runtime.ts` and keep PowerShell-specific dot-source/reload syntax with coverage.
