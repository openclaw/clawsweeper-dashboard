## Summary

Keep open. The Windows bug is real and still present on current main, but this PR cannot merge as-is because it modifies the older completion helper instead of the active runtime module and lacks after-fix Windows CLI/profile-install proof.

## What This Changes

The branch updates Windows time-format PowerShell selection, adds PowerShell completion install/reload syntax in the completion CLI helper, and adds PowerShell completion tests.

## Best Possible Solution

Port the intended install/profile/reload behavior into `src/cli/completion-runtime.ts`, keep the PS7-preferring `resolvePowerShellPath()` change in `src/agents/date-time.ts`, add focused runtime regression coverage, and require redacted Windows CLI proof before merge.

## Reproduction Assessment

Yes, by source inspection. Current main accepts and caches `powershell` completions but the active `installCompletion()` still falls through to the unsupported branch, and Windows time-format detection still invokes literal `powershell`.

## Solution Assessment

No, not as currently proposed. The requested fix is the right narrow direction, but this branch targets the older helper layout and its tests import moved exports instead of exercising the active runtime module.
