## What This Changes

The PR changes Windows time-format detection to use the PowerShell resolver and adds PowerShell completion install/profile-source handling with focused tests.

## Best Possible Solution

Retarget the completion install/source-line/reload fix and tests to completion-runtime.ts, keep the date-time resolver change, and share profile-path helpers with the onboarding path where practical.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: docs list powershell as supported, but current completion-runtime accepts it and then falls through to the unsupported install branch, while date-time detection still execs hard-coded powershell. I did not run a live native Windows reproduction.

## Solution Assessment

Mostly yes: dot-source syntax and resolvePowerShellPath are the narrow maintainable direction. The completion fix must move to completion-runtime.ts and needs redacted native Windows proof before merge.
