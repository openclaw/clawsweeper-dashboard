## Summary

Keep open. The Windows bug is valid and still present on current main, but this PR is stale/incorrect because the completion install fix targets the old CLI helper instead of the current runtime helper; it also still needs native Windows after-fix proof.

## What This Changes

The PR changes Windows time-format detection to use the PowerShell resolver and adds PowerShell completion install/profile handling plus tests against the CLI completion helper.

## Best Possible Solution

Retarget completion install/source-line/reload formatting and tests to src/cli/completion-runtime.ts, keep the date-time resolver change, and align setup/doctor/onboarding profile hints with https://github.com/openclaw/openclaw/issues/44296.

## Reproduction Assessment

Yes, from source inspection on current main: docs and CLI accept powershell, --write-state creates a .ps1 cache, but completion-runtime.ts still falls through to unsupported install and date-time.ts invokes powershell directly. I did not run a native Windows repro in this read-only review.

## Solution Assessment

No, not as currently patched. The date-time resolver change is in the right place, but the completion install/source/reload behavior must move to completion-runtime.ts because CLI, setup, doctor, and update flows use that helper.
