## Summary

Keep open. The underlying Windows completion/time-format bugs are still valid on current main, but this PR is not merge-ready because it patches the older `completion-cli.ts` install surface while current main routes installs through `completion-runtime.ts`, and the external PR still only provides test output rather than native Windows after-fix proof.

## What This Changes

The PR branch updates Windows time-format detection to use `resolvePowerShellPath()` and adds PowerShell completion install/profile handling plus tests in the older `completion-cli.ts` surface.

## Best Possible Solution

Retarget completion install/source-line/reload formatting and tests to `src/cli/completion-runtime.ts`, keep the date-time resolver change, and coordinate the profile/reload guidance with https://github.com/openclaw/openclaw/issues/44296.

## Reproduction Assessment

Yes, from source inspection on current main: docs and CLI accept PowerShell, `--write-state` creates a `.ps1` cache, but `completion-runtime.ts` still rejects PowerShell installation and `date-time.ts` still calls `powershell` directly. I did not run a native Windows reproduction in this read-only review.

## Solution Assessment

No, not as currently patched. The date-time resolver change is in the right direction, but the completion install/source/reload behavior needs to move to `completion-runtime.ts`, which is shared by CLI, setup, doctor, and update flows.
