## Summary

Keep this PR open. The reported Windows bugs are still present on current main and in the latest shipped release, but the branch targets the older `completion-cli.ts` install layout, so the useful fix needs to be rebased or ported to the current `completion-runtime.ts` split before it can land.

## What This Changes

The PR adds PowerShell completion profile installation with dot-source syntax and a PowerShell reload hint, updates Windows time-format detection to use the PS7-preferring resolver, and adds PowerShell completion regression tests.

## Best Possible Solution

Port the same Windows behavior to the current runtime boundary: handle PowerShell in `src/cli/completion-runtime.ts`, use `. "..."` for the profile line and reload hint, update `src/agents/date-time.ts` to call `resolvePowerShellPath()` with non-interactive flags, and keep the tests focused on those paths without changing completion generation or non-Windows behavior.

Yes. ## Solution Assessment

No. The intended fix is the right narrow product direction, but this PR is no longer the best merge shape because it edits the old install helper location; the safer fix is to port the logic and tests to `completion-runtime.ts` on current main.
