## Summary

Keep open. The reported Windows bugs still exist on current main, but this PR is not mergeable as-is because the completion install fix targets the older CLI helper instead of the current runtime helper, and the external PR still lacks required native Windows behavior proof.

## What This Changes

The PR changes Windows time-format detection to use the PowerShell resolver and adds PowerShell completion install/profile handling plus focused tests in the CLI completion helper.

## Best Possible Solution

Retarget the PowerShell install/source-line/reload handling and tests to src/cli/completion-runtime.ts, keep the date-time resolver change, and share profile/reload formatting with setup and doctor paths.

## Reproduction Assessment

Yes. Source inspection on current main shows docs and CLI accept powershell, the runtime writes a .ps1 cache, but installCompletion still falls through to the unsupported branch and date-time still hard-codes powershell.

## Solution Assessment

No, not as currently patched. The date-time change is the right direction, but the completion install/source/reload behavior must move to completion-runtime.ts because the CLI, setup, doctor, and update flows use that helper.
