## Summary

Keep open: the reported Windows behavior is still source-reproducible on current main, but this branch is stale and now patches the old completion helper surface instead of the active runtime module. It also still lacks after-fix real Windows behavior proof, so it needs contributor or maintainer follow-up before merge.

## What This Changes

The branch changes Windows time-format detection to use the PowerShell resolver, adds PowerShell completion install/reload syntax, and adds focused completion tests.

## Best Possible Solution

Port the intended fix into `src/cli/completion-runtime.ts`, keep PowerShell dot-source and reload syntax there, reuse `resolvePowerShellPath()` in `src/agents/date-time.ts`, and add runtime-targeted regression coverage plus redacted Windows proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current main accepts and caches `powershell` completions but the active runtime installer still falls through to unsupported, and Windows time-format detection still invokes literal `powershell`.

## Solution Assessment

No. The requested behavior is the right narrow fix, but this branch edits the older `completion-cli.ts` helper layout and imports tests from that old surface instead of the current runtime module.
