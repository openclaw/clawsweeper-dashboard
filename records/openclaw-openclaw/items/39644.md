## Summary

Keep open. Current main still has the reported Windows PowerShell completion install and time-format selection bugs, but this stale PR now targets the old completion helper layout, is merge-conflicting, and lacks after-fix Windows CLI/profile install proof.

## What This Changes

The PR branch changes Windows time-format PowerShell selection, adds PowerShell completion tests, and edits the older completion CLI install helper for PowerShell profile dot-sourcing.

## Best Possible Solution

Port the PowerShell install/source-line/reload behavior into `src/cli/completion-runtime.ts`, use `resolvePowerShellPath()` in `src/agents/date-time.ts`, add runtime-targeted regression coverage, and require redacted Windows PowerShell proof before merge.

## Solution Assessment

No. The requested fix is the right narrow behavior, but this branch edits stale completion files and tests stale exports; the maintainable fix is to port it to `completion-runtime.ts` and keep the date-time resolver change.
