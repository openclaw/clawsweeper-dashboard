## Summary

Keep open. The underlying Windows bug is still present on current main, but this stale PR now patches an older completion helper location and lacks after-fix real Windows CLI/profile-install proof.

## What This Changes

The PR adds PowerShell completion installation and reload syntax, switches Windows time-format detection to the PS7-preferring resolver, and adds PowerShell completion tests.

## Best Possible Solution

Port the intended behavior into `src/cli/completion-runtime.ts`, reuse `resolvePowerShellPath()` in `src/agents/date-time.ts`, add focused regression coverage against the current runtime exports, and preserve contributor credit at landing.

## Reproduction Assessment

Yes. Source inspection on current main shows `powershell` is accepted and cached, but runtime install still falls into the unsupported branch and Windows time-format detection still invokes literal `powershell`; I did not run a live Windows install in this read-only review.

## Solution Assessment

No. The requested behavior is the right narrow fix, but this PR patches the older helper layout and its tests import moved exports; the maintainable fix is to port it to `completion-runtime.ts` and add real Windows CLI proof.
