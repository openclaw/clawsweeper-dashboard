## Summary

Keep open. The Windows PowerShell bug still exists on current main, but this branch is stale against the completion-runtime split and lacks required after-fix Windows behavior proof, so it needs a port/rebase plus contributor proof before merge.

## What This Changes

This PR changes Windows PowerShell completion installation and reload syntax, switches Windows time-format detection to the existing PowerShell resolver, and adds PowerShell completion tests.

## Best Possible Solution

Apply the behavior in `src/cli/completion-runtime.ts` and `src/agents/date-time.ts`, keep profile/cache guards intact, add focused runtime tests, then require redacted Windows PowerShell CLI/profile proof before merge.

## Reproduction Assessment

Yes. Source inspection shows current main accepts `powershell` as a completion shell and writes `.ps1` cache files, but `installCompletion()` still falls through to the unsupported branch; `detectSystemTimeFormat()` also invokes literal `powershell`.

## Solution Assessment

No. The requested behavior is the right narrow fix, but this branch modifies the older `completion-cli.ts` helper path; current main runs install/profile logic from `completion-runtime.ts`, so the patch must be ported there.
