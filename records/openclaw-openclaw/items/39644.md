## Summary

Keep open. The PR addresses a real supported Windows completion/time-format bug, but it is not correct against current main because the live install path now uses `completion-runtime.ts` while the PR edits the older `completion-cli.ts` helper, and the external PR still lacks required native Windows real behavior proof.

## What This Changes

The PR adds PowerShell completion profile installation/dot-source handling, a PowerShell reload hint, Windows time-format `pwsh` resolution, and focused completion tests.

## Best Possible Solution

Retarget completion install/source-line/reload formatting and tests to `src/cli/completion-runtime.ts`, keep the date-time resolver change, and coordinate concrete profile/reload guidance with https://github.com/openclaw/openclaw/issues/44296.

## Reproduction Assessment

Yes, from source inspection on current main: docs expose PowerShell as a completion target, `--write-state` emits a `.ps1` cache, but the live runtime installer still rejects PowerShell and date-time detection still calls `powershell` directly. I did not run a native Windows reproduction in this read-only review.

## Solution Assessment

No, not as currently patched. The date-time resolver change is appropriate, but completion install/source/reload behavior needs to move to `completion-runtime.ts`, which is shared by CLI, setup, doctor, and update flows.
