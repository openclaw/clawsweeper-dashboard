## Summary

Keep open. Current main still records unmentioned Feishu group messages into pending history under `requireMention`, and the requested mention-only context recording policy is not available in Feishu config or docs.

## What This Changes

The issue requests a Feishu group option so each bot records context only for messages that explicitly mention that bot.

## Best Possible Solution

Add a backwards-compatible Feishu context-recording policy, available globally and per group/account, that defaults to current pending-history behavior but can skip recording unmentioned group messages, with schema, config metadata, tests, docs, and changelog aligned.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: in an allowed Feishu group with `requireMention: true` and `historyLimit > 0`, send an unmentioned message, then a mentioned message; the first message is buffered and prepended to the handled turn.

## Solution Assessment

Unclear until a maintainer chooses the public config semantics. A separate backwards-compatible context-recording policy is likely safer than silently changing documented `requireMention` pending-history behavior.
