## Summary

Keep open. Current main and the latest release still record unmentioned Feishu group messages into pending history and inject that history into later handled turns, so the requested mention-only per-bot context policy is still missing and needs a maintainer-approved config/default decision.

## What This Changes

Request to add Feishu per-bot context isolation so multi-bot groups only record context for messages that explicitly mention that bot.

## Best Possible Solution

Add an opt-in, maintainer-approved Feishu context-recording policy at account/group scope that preserves documented pending-history behavior by default and covers source tests plus docs.

## Reproduction Assessment

Yes, at source level. With `requireMention=true`, unmentioned Feishu group messages are recorded into pending history and later injected into a handled group turn; no live Feishu tenant run was performed.

## Solution Assessment

Unclear until maintainers choose the public config contract. A separate opt-in context-recording policy is safer than silently changing the documented `requireMention` pending-history behavior.
