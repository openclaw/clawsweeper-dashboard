## Summary

Keep open. Current main still buffers unmentioned Feishu group messages under mention gating, and the requested mention-only context recording policy is not present in the Feishu schema or docs.

## What This Changes

Request to add Feishu per-bot context isolation so group messages only enter a bot's context when that bot is @mentioned.

## Best Possible Solution

Add a backward-compatible Feishu context-recording policy, available globally and per group, that defaults to current pending-history behavior but can skip recording unmentioned group messages, with schema, docs, and regression tests updated together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: in an allowed Feishu group with `requireMention: true` and `historyLimit > 0`, send an unmentioned message and then a mentioned message in the same group; the first is recorded as pending history and prepended to the handled turn.

## Solution Assessment

Unclear until a maintainer chooses the public config semantics. A separate explicit context-recording policy is likely safer than silently changing the documented `requireMention` pending-history behavior.
