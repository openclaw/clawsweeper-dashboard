## Summary

Keep open. Current main still records unmentioned Feishu group messages into pending history when `requireMention` gates the reply, and there is no Feishu config surface for mention-only context recording, so this remains a valid feature/config decision rather than a closeable cleanup item.

## What This Changes

Request a Feishu option or policy so each bot in a shared group only records context for messages explicitly mentioning that bot.

## Best Possible Solution

Add a backward-compatible Feishu context-recording policy, available globally and per group, that defaults to current pending-history behavior but can skip recording unmentioned group messages, with schema, docs, metadata, and regression tests updated together.

## Reproduction Assessment

Yes. Source inspection on current main shows an allowed Feishu group with `requireMention: true` and `historyLimit > 0` records an unmentioned message to pending history and prepends it to the next mentioned turn.

## Solution Assessment

Unclear until a maintainer chooses the public config shape. A separate backward-compatible context-recording policy is likely safer than silently changing documented `requireMention` pending-history behavior.
