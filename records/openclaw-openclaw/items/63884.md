## Summary

Keep open. Current main still records unmentioned Feishu group messages as pending history when mention gating skips them, then injects that history into the next handled group turn; the requested mention-only context recording policy is not present in schema or docs.

## What This Changes

Request a Feishu configuration option so each bot in a shared group only records context for turns where that bot is explicitly mentioned.

## Best Possible Solution

Add a backwards-compatible Feishu context-recording policy, available globally and per group, that defaults to existing pending history but can skip unmentioned group messages, with schema, docs, and regression tests updated together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: in an allowed Feishu group with `requireMention: true` and `historyLimit > 0`, send an unmentioned message, then a mentioned message in the same group; the first message is recorded as pending history and prepended to the handled turn.

## Solution Assessment

Unclear until maintainers choose the public config shape. A separate opt-in context-recording policy is safer than silently changing documented `requireMention` pending-history behavior.
