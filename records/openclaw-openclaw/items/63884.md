## Summary

Keep open: current main and v2026.6.10 still store unmentioned Feishu group messages as pending context and later inject that history into handled turns, while the requested mention-only Feishu context policy is not present. This is a valid feature/product decision, not a cleanup close.

## What This Changes

The issue requests Feishu group context isolation so each bot only records conversation history when that bot is explicitly mentioned.

## Best Possible Solution

Add a maintainer-approved Feishu group-history context policy with schema, docs, focused regression tests, and live multi-bot Feishu proof before changing the default recording behavior.

## Reproduction Assessment

Yes, at source level: the Feishu skipped-message path records unmentioned group messages, and later handled turns inject that history into context. I did not run a live multi-bot Feishu tenant in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public contract. A Feishu context policy aligned with existing group-history and context-visibility patterns is safer than silently redefining requireMention or adding one-off config names without an upgrade contract.
