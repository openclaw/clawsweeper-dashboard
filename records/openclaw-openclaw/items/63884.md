## Summary

Keep open: current main and v2026.6.10 still buffer unmentioned Feishu group messages as pending history and inject them into later handled turns, while no Feishu mention-only context-recording policy exists.

## What This Changes

Request adds per-bot Feishu group context isolation so messages are recorded into a bot's context only when that bot is explicitly mentioned.

## Best Possible Solution

Add a maintainer-approved opt-in Feishu context-recording policy at account/group scope, preserving documented pending-history behavior by default and covering schema, docs, regression tests, and live Feishu proof.

## Reproduction Assessment

Yes, at source level. With requireMention enabled, current Feishu records unmentioned group messages into pending history and later injects that history into handled group turns; no live Feishu tenant run was performed.

## Solution Assessment

Unclear until maintainers choose the public config contract. A separate opt-in context-recording policy is safer than silently changing documented requireMention pending-history behavior.
