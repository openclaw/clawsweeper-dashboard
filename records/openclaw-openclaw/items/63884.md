## Summary

Keep open. Current main still records unmentioned Feishu group messages as pending history under `requireMention`, and the Feishu config/docs do not expose a mention-only context recording policy.

## What This Changes

Request to add Feishu group context isolation so each bot only records group messages where it was explicitly mentioned.

## Best Possible Solution

Add a backwards-compatible Feishu context-recording policy, available globally and per group, that defaults to current pending-history behavior but can skip recording unmentioned group messages, with schema, tests, docs, and config metadata updated together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: in an allowed Feishu group with `requireMention: true` and `historyLimit > 0`, send an unmentioned message, then a mentioned message in the same group; the first message is buffered and prepended to the handled turn.

## Solution Assessment

Unclear until a maintainer chooses the public config shape. A separate backwards-compatible context-recording policy is likely safer than changing the documented meaning of `requireMention`.
