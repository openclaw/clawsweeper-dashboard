## Summary

Keep open: current main and v2026.6.10 still record unmentioned Feishu group messages into pending history and later inject that history into handled group turns, while the requested Feishu mention-only context policy/config is absent. This remains a valid feature/product decision, not a closeable implemented item.

## What This Changes

The issue requests a Feishu group context policy so each bot records context only for messages explicitly addressed to that bot in multi-bot Feishu groups.

## Best Possible Solution

Add a maintainer-approved Feishu group-history context policy, preferably aligned with the existing Telegram mention-only/recent/none pattern, with schema/docs/tests and live multi-bot Feishu proof.

## Reproduction Assessment

Yes, at source level. The Feishu skipped-message path records unmentioned group messages into history, and later handled group turns inject that history into context; no live Feishu tenant run was performed.

## Solution Assessment

Unclear until maintainers choose the public contract. A Feishu policy aligned with the existing Telegram group-history mode looks safer than silently redefining requireMention or adding one-off config names.
