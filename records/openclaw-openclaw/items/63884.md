## Label Justifications

- `P2`: The request affects a real Feishu multi-bot group workflow but is a bounded feature/config decision rather than an urgent runtime outage.
- `impact:session-state`: The reported problem is unwanted Feishu group messages entering another bot's conversation context.

## Summary

Keep open. Current main and the latest shipped tag still record unmentioned Feishu group messages into pending group history and later inject that history into handled turns, while the requested mention-only recording policy would require a new Feishu config/product contract.

## What This Changes

Request per-bot Feishu group context isolation so a bot only records Feishu group messages into its context when that bot is explicitly mentioned.

## Best Possible Solution

Add a backward-compatible Feishu context-recording policy, available at account and group scope, that defaults to current pending-history behavior but can skip recording unmentioned group messages for that bot.

## Reproduction Assessment

Yes, at source level. A Feishu group with `requireMention=true` and nonzero history still records unmentioned messages into pending history and injects them into the next handled group turn.

## Solution Assessment

Unclear until maintainers choose the public config contract. A separate opt-in context-recording policy is safer than silently changing documented `requireMention` pending-history behavior.
