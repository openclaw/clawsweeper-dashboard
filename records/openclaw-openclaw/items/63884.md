## Summary

Keep open. Current main still records unmentioned Feishu group messages as pending history and later injects them into a handled turn, while the Feishu config/docs do not offer the requested mention-only context recording policy.

## What This Changes

Request a Feishu group context-isolation option so each bot records context only for messages that explicitly mention that bot in multi-bot groups.

## Best Possible Solution

Add a backward-compatible Feishu context-recording policy, available globally and per group, that defaults to current pending-history behavior but can skip unmentioned group messages when enabled, with schema, docs, metadata, and regression tests updated together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: in an allowed Feishu group with `requireMention: true` and `historyLimit > 0`, an unmentioned message is recorded under the group history key and prepended to the next mentioned turn.

## Solution Assessment

Unclear until a maintainer chooses the public config semantics. A separate backwards-compatible context-recording policy is safer than silently changing the documented `requireMention` pending-history behavior.
