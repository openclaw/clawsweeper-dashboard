## Summary

Keep open: current main and v2026.6.10 still buffer unmentioned Feishu group messages as pending history and replay that history into later handled turns, while the requested per-bot mention-only context policy is not present. The related Feishu union-id PR is adjacent mention-detection work, not a fix for context retention.

## What This Changes

The issue requests Feishu group context isolation so a bot configured with mention gating only records context from messages explicitly addressed to that bot.

## Best Possible Solution

Add a maintainer-approved Feishu plugin context-retention policy at account/group scope, preserving documented pending-history behavior by default unless maintainers explicitly choose a new default.

## Reproduction Assessment

Yes, at source level. With requireMention enabled, current Feishu records unmentioned group messages into pending history and later injects that history into handled group turns; no live Feishu tenant run was performed.

## Solution Assessment

Unclear until maintainers choose the public config contract. A separate opt-in context-retention policy is safer than silently changing documented requireMention pending-history behavior.
