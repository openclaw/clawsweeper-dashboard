## Summary

Keep open. Current main and v2026.5.18 still buffer unmentioned Feishu group messages into pending group history and inject that buffer into later handled turns; solving this needs a maintainer-approved Feishu context-recording/config contract.

## What This Changes

Request a Feishu group/account mode that only records messages into a bot's context when that bot was explicitly mentioned.

## Best Possible Solution

Add a backward-compatible Feishu context-recording policy at account and group scope that defaults to current pending-history behavior but can skip unmentioned group messages for that bot.

## Reproduction Assessment

Yes, at source level. A Feishu group with requireMention=true and nonzero history still records unmentioned messages into pending history and injects them into the next handled group turn.

## Solution Assessment

Unclear until maintainers choose the public config contract. A separate opt-in context-recording policy is safer than silently changing the documented requireMention pending-history behavior.
