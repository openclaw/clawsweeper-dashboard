## Summary

Keep open: current main and v2026.6.10 still record unmentioned Feishu group messages into pending history and later inject that history into handled group turns, while the requested per-bot mention-only context-recording policy is not implemented or owned by a merged canonical item.

## What This Changes

The issue requests a Feishu group policy so each bot records context only for messages that explicitly mention that bot, avoiding cross-bot context pollution in shared groups.

## Best Possible Solution

Add a maintainer-approved Feishu context-recording policy at account/group scope, preserving documented pending-history behavior by default unless maintainers intentionally change the default.

## Reproduction Assessment

Yes, at source level. With `requireMention` enabled, current Feishu records unmentioned group messages into pending history and later injects that history into handled group turns; no live Feishu tenant run was performed.

## Solution Assessment

Unclear until maintainers choose the public config contract. A separate opt-in context-recording policy is safer than silently changing documented `requireMention` pending-history behavior.
