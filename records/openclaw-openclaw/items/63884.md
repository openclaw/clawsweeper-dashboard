## Summary

Keep open. Current main and v2026.6.6 still intentionally record unmentioned Feishu group messages into pending history and later inject that history, so the requested per-bot mention-only context policy is still missing and needs a maintainer product/config decision.

## What This Changes

The issue requests Feishu group context isolation so a bot records group context only when that specific bot is mentioned.

## Best Possible Solution

Add a maintainer-approved, backward-compatible Feishu context-recording policy at account/group scope that can skip unmentioned group messages for that bot while preserving the documented pending-history default unless maintainers intentionally change it.

## Reproduction Assessment

Yes, at source level. A Feishu group with requireMention=true and nonzero history still records unmentioned messages into pending history and injects them into the next handled group turn.

## Solution Assessment

Unclear until maintainers choose the public config contract. A separate opt-in context-recording policy is safer than silently changing the documented requireMention pending-history behavior.
