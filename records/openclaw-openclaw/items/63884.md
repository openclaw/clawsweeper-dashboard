## Summary

Keep open. Current main still records unmentioned Feishu group messages into pending history and later prepends that history to a handled group turn, while the Feishu config/schema/docs expose no mention-only context-recording policy. The related open PR #40782 is useful adjacent Feishu multi-bot mention-identity work, but it does not implement context isolation.

## What This Changes

Request a Feishu configuration option that prevents a bot in a shared group from recording messages into its context unless that bot was explicitly mentioned.

## Best Possible Solution

Add a backwards-compatible Feishu context-recording policy, available at least globally and per group, that defaults to the current pending-history behavior but can skip recording unmentioned group messages. Cover it with Feishu handler tests and update the Feishu/group docs plus generated config metadata if the schema changes.

## Reproduction Assessment

Yes. In an allowed Feishu group with `requireMention: true` and `historyLimit > 0`, send an unmentioned group message and then a mentioned message in the same group; current source records the first message as pending history and prepends it to the second turn.

## Solution Assessment

Unclear until a maintainer chooses the public config contract. A backwards-compatible Feishu context-recording policy is likely the narrowest maintainable solution; silently changing `requireMention` would conflict with documented pending-history behavior.
