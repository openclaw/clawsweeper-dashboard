## Summary

Keep open. Current main still records unmentioned Feishu group messages into pending history when `requireMention` is true and later prepends that history to a handled group turn, while the Feishu schema and docs expose no mention-only context-recording policy. Related open PR #40782 is useful adjacent multi-bot mention-identity work, but it does not implement context isolation.

## What This Changes

Request a Feishu group context policy so a bot only records shared-group messages when that bot was explicitly mentioned.

## Best Possible Solution

Add a backwards-compatible Feishu context-recording policy, available globally and per group, that defaults to current pending-history behavior but can skip recording unmentioned group messages, with Feishu tests, docs, and config metadata updated together.

## Reproduction Assessment

Yes. In an allowed Feishu group with `requireMention: true` and `historyLimit > 0`, send an unmentioned message and then a mentioned message in the same group; current source records the first as pending history and prepends it to the second turn.

## Solution Assessment

Unclear until a maintainer chooses the public config contract. A backwards-compatible Feishu context-recording policy is likely the narrowest maintainable direction; silently changing `requireMention` would conflict with documented pending-history behavior.
