## Summary

Keep open. Current main and v2026.5.18 still record unmentioned Feishu group messages into shared pending group history and replay that history on later handled turns; solving this needs a new Feishu context-recording/config policy rather than a cleanup close.

## What This Changes

Request a Feishu group option or policy so each bot records context only for messages where that bot is explicitly mentioned.

## Best Possible Solution

Add an opt-in, backward-compatible Feishu context-recording policy at account and group scope that defaults to current pending-history behavior but can skip unmentioned group messages for that bot.

## Reproduction Assessment

Yes, at source level. A Feishu group with requireMention enabled and nonzero history still records unmentioned messages into pending history and injects that history into the next handled group turn.

## Solution Assessment

Unclear until maintainers choose the public config contract. A separate opt-in context-recording policy is safer than silently changing documented requireMention pending-history behavior.
