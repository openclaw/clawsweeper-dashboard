## Summary

Keep open. Current main still buffers unmentioned Feishu group messages when `requireMention` is true and later includes that pending history in a handled group turn, while the Feishu config schema and docs do not expose a mention-only context recording policy.

## What This Changes

Request a Feishu group context-isolation option so each bot records only messages that explicitly mention it in shared multi-bot groups.

## Best Possible Solution

Add a backwards-compatible Feishu context-recording policy, available globally and per group, that defaults to current pending-history behavior but can skip recording unmentioned group messages, with schema, tests, docs, and config metadata updated together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: in an allowed Feishu group with `requireMention: true` and `historyLimit > 0`, send an unmentioned message and then a mentioned message in the same group; the first is recorded as pending history and prepended to the second turn.

## Solution Assessment

Unclear until a maintainer chooses the public config shape. A separate backwards-compatible context-recording policy is likely safer than silently changing documented `requireMention` pending-history behavior.
