## Summary

Keep open: current main and the latest release still record unmentioned Feishu group messages into pending group history and inject that history into later handled turns, while the requested mention-only per-bot context policy is not present. The related open PR only addresses Feishu mention detection, not this context-recording/default policy.

## What This Changes

Request a Feishu account/group context-isolation policy so multi-bot groups only record messages addressed to the current bot.

## Best Possible Solution

Add a maintainer-approved opt-in Feishu context-recording policy at account/group scope, preserving documented pending-history behavior by default and covering source tests, docs, and live Feishu proof.

## Reproduction Assessment

Yes, at source level. With requireMention enabled, current Feishu records unmentioned group messages into pending history and later injects that history into handled group turns; no live Feishu tenant run was performed.

## Solution Assessment

Unclear until maintainers choose the public config contract. A separate opt-in context-recording policy is safer than silently changing documented requireMention pending-history behavior.
