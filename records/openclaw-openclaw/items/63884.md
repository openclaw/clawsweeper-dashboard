## Summary

Keep open: current main and v2026.6.10 still record unmentioned Feishu group messages into shared group history and replay that history into later mentioned turns, while the requested mention-only/per-bot context policy is not implemented.

## What This Changes

Request a Feishu group context-isolation option so a bot only records context for messages that explicitly mention that bot in multi-bot groups.

## Best Possible Solution

Add a maintainer-approved Feishu group context policy with schema/docs, focused regression tests, and live multi-bot Feishu proof before changing default context recording.

## Reproduction Assessment

Yes, at source level: current Feishu code records unmentioned requireMention group messages and later injects that group history into handled turns. I did not establish a live multi-bot Feishu reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public contract. A Feishu-owned context policy with explicit defaults is safer than silently redefining requireMention or adding ad hoc config keys.
