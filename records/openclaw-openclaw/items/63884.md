## Summary

Keep open. Current main still records unmentioned Feishu group messages into per-account pending history and later injects that history into handled turns, while the requested mention-only recording policy is a new Feishu config/product decision rather than an implemented fix.

## What This Changes

Request a Feishu group configuration policy so each bot only records context for messages that explicitly mention that bot in multi-bot groups.

## Best Possible Solution

Add a backward-compatible Feishu context-recording policy, available globally and per group/account, that defaults to current pending-history behavior but can skip recording unmentioned group messages, with schema, docs, metadata, and regression tests updated together.

## Reproduction Assessment

Yes, at source level. An allowed Feishu group with requireMention=true and historyLimit>0 records unmentioned messages into pending history and injects that history into the next handled group turn.

## Solution Assessment

Unclear until a maintainer chooses the public config shape. A separate backward-compatible context-recording policy is safer than silently changing documented requireMention pending-history behavior.
