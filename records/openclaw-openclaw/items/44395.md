## Summary

## What This Changes

Request heading-aware memory chunking plus entity extraction or graph-style synthesis so memory search keeps distinct markdown sections and entities from blending together.

## Best Possible Solution

Land a maintainer-approved section-aware chunking strategy through the memory-host SDK and memory-core indexing/reindex path, then split entity extraction, wiki synthesis, reranking, and consolidation into opt-in follow-ups with explicit privacy, cost, provenance, and reindex controls.

## Reproduction Assessment

## Solution Assessment

Unclear for the full issue. Heading-aware chunking is a plausible narrow direction and https://github.com/openclaw/openclaw/pull/68019 is the current candidate, but entity extraction and automated wiki synthesis need product, privacy, cost, and API decisions first.
