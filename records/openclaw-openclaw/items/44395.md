## What This Changes

The issue requests heading-aware Markdown memory chunking plus broader entity extraction, graph/wiki memory, reranking, and consolidation improvements for memory search.

## Best Possible Solution

Land a maintainer-approved section-aware chunking strategy through the shared memory-host SDK/indexing path, then split entity extraction, graph/wiki memory, reranking, and consolidation into opt-in follow-ups with explicit privacy, cost, provenance, and reindex controls.

Unclear for the full issue. Heading-aware chunking is a plausible narrow fix, but https://github.com/openclaw/openclaw/pull/68019 still needs maintainer review, and the broader entity/wiki automation scope needs product and privacy decisions before implementation.
