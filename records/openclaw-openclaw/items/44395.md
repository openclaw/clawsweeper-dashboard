## What This Changes

Request heading-aware memory chunking for structured markdown plus longer-term entity extraction, graph/wiki synthesis, reranking, and consolidation for memory search.

## Best Possible Solution

Land a maintainer-approved opt-in section-aware chunking strategy through the memory-host SDK and memory-core indexing/reindex path, then track automated entity/wiki synthesis and ingestion helpers through https://github.com/openclaw/openclaw/issues/79004 and https://github.com/openclaw/openclaw/issues/42651 with explicit privacy, cost, provenance, and operator controls.

## Solution Assessment

Unclear for the full issue. Heading-aware chunking is a plausible narrow direction and https://github.com/openclaw/openclaw/pull/68019 is the current candidate, but entity extraction and automated wiki synthesis need product, privacy, cost, and API decisions first.
