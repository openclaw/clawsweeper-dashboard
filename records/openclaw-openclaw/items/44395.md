## What This Changes

The issue requests markdown heading-aware memory chunking plus later entity extraction and graph-style memory improvements for better memory search precision.

## Best Possible Solution

Land a maintainer-approved section-aware chunking strategy through the memory-host SDK, config schema, and memory-core reindex path, while keeping entity extraction and Memory Wiki synthesis as opt-in follow-ups with clear privacy, cost, provenance, and scheduling controls.

Unclear for the full issue. Heading-aware chunking is a plausible narrow direction and https://github.com/openclaw/openclaw/pull/68019 is the current candidate, but entity extraction and automated wiki synthesis need product, privacy, cost, and API decisions first.
