## What This Changes

The issue asks for heading-aware Markdown chunking for memory search and explores entity extraction or graph-memory improvements for recall precision.

## Best Possible Solution

Keep this as the canonical tracker, split the narrow heading-aware chunking design from entity/graph-memory work, and have maintainers choose the chunking API/default and index identity contract before implementation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: index a Markdown memory file with several short headed sections under the current fixed token/overlap chunker, and unrelated headed sections can share one embedding chunk until the size budget flushes.

## Solution Assessment

Unclear. Heading-aware chunking is plausible, but the best fix still needs maintainer choice on default versus opt-in strategy, index revalidation, and separation from the larger entity-extraction work.
