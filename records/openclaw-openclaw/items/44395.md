## Summary

Keep open. Current main still uses fixed token/overlap chunking for memory indexing, while the discussion adds real production use cases and the likely chunking implementation is already represented by open PR #68019; the entity extraction, graph memory, reranking, and consolidation pieces need separate maintainer scoping.

## What This Changes

The issue requests heading-aware memory chunking for Markdown memory files, with broader follow-up ideas for entity extraction, graph memory, reranking, and consolidation.

## Best Possible Solution

Ship a documented heading-aware or section-aware chunking strategy in the shared memory-host indexing path with strategy-aware reindex metadata, then split entity extraction, graph memory, reranking, and consolidation into narrower memory-core/QMD/memory-wiki follow-ups.
