## What This Changes

The issue requests heading-aware memory-search chunking plus later entity extraction, graph memory, and consolidation improvements.

## Best Possible Solution

Keep this as the canonical tracker, split the narrow heading-aware chunking design from entity/graph-memory work, and have maintainers choose the config/default and reindex contract before implementation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: index a Markdown memory file with several short headed sections under the current fixed token/overlap chunker, and unrelated headed sections can share one embedding chunk until the size budget flushes.

## Solution Assessment

No current implementation solves it yet. Heading-aware chunking is plausible, but the best fix needs maintainer choice on default versus opt-in strategy, index revalidation, and separation from the larger entity-extraction work.
