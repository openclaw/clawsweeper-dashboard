## Summary

Keep open: current main and the latest release still use fixed token/overlap memory chunking, while heading-aware chunking has an open implementation candidate and the entity/wiki automation portion remains unshipped product work.

## What This Changes

Add heading-aware memory chunking and broader entity extraction or graph/wiki memory support to improve memory search precision.

## Best Possible Solution

Land a maintainer-approved section-aware chunking strategy through the shared memory-host indexing path, then split entity extraction, graph/wiki memory, reranking, and consolidation into opt-in follow-ups with explicit privacy, cost, and provenance controls.

The entity-extraction portion is a requested new capability rather than a broken existing contract.
