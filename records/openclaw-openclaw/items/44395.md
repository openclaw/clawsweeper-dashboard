Keep open. Current main still lacks heading/section-aware memory chunking and built-in entity extraction; the open multi-strategy chunking PR is only an implementation candidate and still needs maintainer API/security review, so this remains valid product work rather than cleanup closure.

## What This Changes

The issue requests heading-aware memory chunking, with broader follow-up ideas for entity extraction, graph memory, reranking, and consolidation.

## Best Possible Solution

Land a maintainer-approved heading/section-aware chunking strategy through the package-owned memory-host SDK and memory-core indexing path with strategy-aware reindex metadata, then track entity extraction, graph/wiki memory, reranking, and consolidation as narrower follow-ups or ClawHub/plugin work where appropriate.

Yes. Source inspection gives a high-confidence reproduction path: create several short adjacent `##` sections whose combined estimated size stays under the fixed chunk budget, and current `chunkMarkdown` will keep them in one chunk because it only flushes on size.
