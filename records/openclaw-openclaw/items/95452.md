## Summary

Keep this PR open: the narrow code change appears correct and still necessary because current main and the latest release keep the memory-status session denominator on a strict `.jsonl` filter while the session indexer uses the usage-counted transcript predicate. There are no blocking code findings in the one-file diff, but exact-head CI is not green and maintainers should decide whether this PR should close the broader linked issue scope.

## What This Changes

The PR changes `extensions/memory-core/src/cli.runtime.ts` so the session status scanner uses `isUsageCountedSessionTranscriptFileName()` instead of `.endsWith(".jsonl")` for the session file denominator.

## Best Possible Solution

Land the narrow predicate-alignment fix once exact-head CI is green or explicitly dispositioned, and split any remaining dirty-state or archive-search policy work if maintainers still want it.

## Reproduction Assessment

Yes. Source inspection shows current main renders the session status denominator from a strict `.jsonl` scan while session listing/indexing uses the usage-counted predicate; the PR body supplies after-fix terminal proof from an isolated state directory.

## Solution Assessment

Yes for the denominator mismatch. Reusing the existing usage-counted filename predicate is the narrowest maintainable fix; changing archive indexing or dirty-state policy would be separate work and would conflict with the current reset/deleted archive contract unless maintainers change that policy explicitly.
