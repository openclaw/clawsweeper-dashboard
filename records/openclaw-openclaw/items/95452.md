## What This Changes

The PR changes `scanSessionFiles()` to count session-status denominator files with `isUsageCountedSessionTranscriptFileName()` instead of a strict `.jsonl` suffix.

## Best Possible Solution

Land this narrow denominator alignment, then keep any remaining dirty-state or archive-policy work on a separate issue if needed.

## Reproduction Assessment

Yes. Source inspection on current main shows `scanSessionFiles()` uses `.endsWith(".jsonl")` while the session listing/indexing path uses `isUsageCountedSessionTranscriptFileName()`, and the PR body includes an after-fix CLI run showing the corrected session denominator.

## Solution Assessment

Yes. Reusing the canonical usage-counted filename helper in the status scanner is the narrowest maintainable fix; dropping reset/deleted archives would conflict with the current artifact contract and tests.
