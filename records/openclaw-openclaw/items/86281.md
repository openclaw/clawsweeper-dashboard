## Summary

Keep this PR open. Current main still warms provider auth on the gateway process, while the PR is a focused root-cause fix that moves that warm path into a worker and covers the startup/reload/logout call sites; I found no discrete blocking code finding, but auth-provider and availability risk still need maintainer review before merge.

## What This Changes

The PR moves provider-auth warm work into a worker thread, adds worker packaging/release checks, passes presence-only auth-store snapshots to the worker, and rewires gateway startup/reload/logout rewarm call sites.

## Best Possible Solution

Land the worker-backed warmer only after maintainer auth/gateway review confirms worker failure fallback, presence-only auth semantics, packaging, and CI are acceptable for upgrades.

## Solution Assessment
