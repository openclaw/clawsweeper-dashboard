## Summary

Keep this PR open: the off-main-thread warm shape is the right direction and the supplied Testbox proof strongly supports the event-loop starvation fix, but the current head still has a blocking auth-provider regression at the worker boundary.

## What This Changes

This PR adds a worker-thread provider-auth warm path, serializes warm snapshots, carries sanitized runtime auth-store presence into the worker, rewires gateway startup/reload/logout rewarm calls, and adds packaging/release coverage for the worker entry.

## Best Possible Solution

Preserve active synthetic-auth provider refs across the worker boundary, or avoid publishing negative synthetic-auth answers when the worker cannot see a complete active registry, then rerun focused auth/gateway/release proof.

## Solution Assessment
