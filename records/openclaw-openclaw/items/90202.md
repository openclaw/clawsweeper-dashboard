Full review comments:

- **[P2] Check the source-sync cap before writing local pages:** `extensions/memory-wiki/src/local-import.ts:210-213`
  - body: Bridge and unsafe-local imports call `assertMemoryWikiSourceSyncStateCapacity` after collecting artifacts and before any page writes, because the source-sync store rejects more than 20,000 entries. This new path starts `writeLocalImportSourcePage` for every artifact first and only fails when `writeMemoryWikiSourceSyncState` runs, so a large configured directory can leave thousands of unmanaged `sources/local-import-*` pages behind with no saved state to skip or prune them. Reuse the existing capacity assertion for `group: "local-import"` before the `Promise.all` and add the oversized-import regression test.
  - confidence: 0.9
- **[P2] Document the new local-import config and command:** `extensions/memory-wiki/src/cli.ts:1120-1125`
  - body: This PR adds both `plugins.entries.memory-wiki.config.localImports` and `openclaw wiki local-import import`, but the public Memory Wiki and wiki CLI docs still only describe `wiki ingest`, `wiki unsafe-local import`, and the existing config keys. Because this is a new user-facing local-file import API, the docs need to explain how to configure it, how it differs from one-shot ingest and unsafe-local, and what privacy/path-selection boundary operators are accepting.
  - confidence: 0.84
