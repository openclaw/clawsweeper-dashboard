## What This Changes

The issue requests that `openclaw doctor` treat platform-incompatible skills as skipped or incompatible instead of actionable missing requirements or disable targets.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89260, or an equivalent fix, so doctor rolls up OS-only skill mismatches separately while preserving warnings for genuinely missing bins, env, config, or mixed requirements.

Yes. Source inspection shows unsupported OS requirements populate `missing.os`, make the skill ineligible, and current doctor code counts/repairs those entries as ordinary unavailable skills; I did not run the Windows bundle scenario live.

## Solution Assessment

Yes. The best solution is the linked shared-evaluator-based doctor classification fix, because it covers both status output and doctor --fix without duplicating local-only platform logic.
