## Summary

Keep open: the PR addresses a real background-exec completion delivery bug, but the immediate-wake implementation reopens the interval-gate protection documented by current tests and by a prior maintainer closure for feedback-loop risk.

## What This Changes

The PR changes background exec completion wake requests to `intent: "immediate"`, adds an exec-completion transcript marker, exposes exec details in delivery-disabled prompts, and updates transcript/chat filtering for the new marker.

## Best Possible Solution

Preserve the interval-gated exec-event scheduler contract and solve completion visibility through a maintainer-approved coalesced wake, next-heartbeat delivery, or non-model notification path, while keeping the scoped exec transcript marker work if maintainers accept it.

## Reproduction Assessment

Yes. Source inspection shows current exec completion producers use event intent, and current cooldown code can return non-retryable `not-due` for later exec-event wakes inside a future `nextDueMs`.

## Solution Assessment

No, not as submitted. The exec transcript marker direction is plausible, but changing exec completion producers to `immediate` bypasses the scheduler guard that current tests and prior maintainer review identify as production feedback-loop protection.
