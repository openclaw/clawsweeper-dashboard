## Summary

Keep this PR open. Current main still lacks a same-group Telegram multi-agent walkthrough, and the PR fills a real docs gap, but the current sample is not merge-ready because it frames the setup as one intended Telegram group while using open group policy without account-level group allowlists.

## What This Changes

The branch adds a `docs/concepts/multi-agent.md` section showing multiple Telegram bot accounts bound to separate agents in one group, with BotFather privacy-mode, `groupAllowFrom`, and `groupPolicy` setup notes.

## Best Possible Solution

Keep the docs contribution, but revise the sample so each Telegram account explicitly lists the shared group under `channels.telegram.accounts.<id>.groups`, while preserving the Privacy Mode remove/re-add note, sender-ID wording for `groupAllowFrom`, and per-account `groupPolicy` precedence guidance. This should remain a docs-only change with no changelog entry.

## Reproduction Assessment

Yes. The issue is reproducible by reading the PR sample against current `resolveChannelGroupPolicy` and Telegram docs: `groupPolicy: "open"` without `groups` permits any joined group, and multi-account Telegram configs do not inherit channel-level `groups`.
