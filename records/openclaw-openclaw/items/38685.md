## Summary

Keep this PR open: the Telegram multi-agent group recipe fills a real docs gap, but the current sample still teaches unscoped group access for a single intended group.

## What This Changes

The PR adds a `docs/concepts/multi-agent.md` section showing two Telegram bot accounts bound to separate agents in one group, with setup notes for privacy mode, allowlists, and per-account group policy.

## Best Possible Solution

Keep the dedicated recipe, but revise it to scope each Telegram account to the intended group with account-level `groups` or an equivalent allowlisted pattern and align the setup notes with current Telegram group-ID docs.

## Reproduction Assessment

Yes. Source inspection reproduces the review issue by comparing the PR-head sample against current Telegram docs, `resolveChannelGroupPolicy`, and multi-account group inheritance tests.

## Solution Assessment

No. The docs direction is useful, but the current patch is not the best final form until the sample scopes the intended Telegram group and avoids teaching open group access without `groups`.
