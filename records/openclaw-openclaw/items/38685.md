## Summary

Keep open: the docs gap is real and current main still lacks this dedicated Telegram multi-agent group recipe, but the PR is not mergeable as-is because the sample opens group access without scoping the intended chat and lacks required real Telegram proof.

This PR adds a `docs/concepts/multi-agent.md` recipe for routing two Telegram bot accounts to separate agents in one shared group.

## Best Possible Solution

Keep the recipe, but revise it so each Telegram account scopes the intended group with account-level `groups` or equivalent per-group allowlisting, align it with current Telegram group-ID docs, and add redacted live proof.
