## Summary

Keep open: the Telegram multi-agent docs gap is still real, but this PR currently teaches unscoped open Telegram group access for a one-group setup, which conflicts with the current Telegram group allowlist contract.

## What This Changes

The PR adds a new `docs/concepts/multi-agent.md` section showing how to configure two Telegram bot accounts for separate agents in one shared group.

## Best Possible Solution

Keep the Telegram multi-agent recipe, but scope each bot account to the intended group chat with account-level `groups`, preserve the sender allowlist guidance, and align the prose with the current Telegram docs.

## Reproduction Assessment

Yes for the docs defect. Source inspection of the PR diff shows open group policy without `groups`, while current docs and runtime show `groups` is the Telegram chat allowlist and open policy is unscoped after chat checks.

## Solution Assessment

No. The docs direction is useful, but the sample should use the current scoped Telegram contract instead of recommending broad open group access for a one-group setup.
