## Summary

Keep open: the Telegram group docs recipe is useful and not present on current main, but the PR still teaches an unscoped open-group authorization pattern and has no after-fix real Telegram proof.

## What This Changes

This PR adds a `docs/concepts/multi-agent.md` section showing two Telegram bot accounts routed to two agents in one shared group, with BotFather privacy mode, sender allowlist, and groupPolicy notes.

## Best Possible Solution

Revise the sample so each Telegram account scopes the intended group with account-level `groups` or an equivalent explicit chat allowlist pattern, then add redacted live Telegram proof.

## Reproduction Assessment

Yes. Source inspection reproduces the docs defect by comparing the PR sample to current Telegram docs, group-policy runtime, group-access runtime, and multi-account group inheritance tests.

## Solution Assessment

No. The documentation direction is useful, but the current example is not the safest final form until it scopes the intended Telegram group instead of relying on unscoped open group policy.
