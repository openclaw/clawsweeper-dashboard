## Summary

This PR should stay open with changes requested: the recipe is useful and current main still lacks a dedicated multiple-Telegram-agents-in-one-group example, but the added sample scopes senders only and leaves group chat access open to any group the bots join.

## What This Changes

The PR adds a `docs/concepts/multi-agent.md` section with a JSON5 example and setup notes for running two Telegram bot accounts, each bound to a different agent, in one shared group.

## Best Possible Solution

Keep the dedicated recipe, but revise it so each Telegram account is scoped to the intended group with account-level `groups` and wording that matches the current Telegram group-ID docs.

## Reproduction Assessment

Yes. Source inspection reproduces the issue by comparing the PR-head sample against current Telegram docs, `resolveChannelGroupPolicy`, and multi-account group inheritance tests.

## Solution Assessment

No. The docs direction is useful, but the current patch is not ready until the example scopes the intended Telegram group instead of teaching open group access without `groups`.
