## Summary

Keep open. Current main still lacks the dedicated Telegram same-group walkthrough, and the PR fills a useful docs gap, but the proposed sample still needs a safety fix: it uses open Telegram group policy without scoping the intended group, while the runtime/docs make that accept any group the bots join.

## What This Changes

This PR adds a dedicated `docs/concepts/multi-agent.md` section for configuring multiple Telegram bot accounts in one group, including bindings, BotFather privacy-mode notes, and sender allowlist guidance.

## Best Possible Solution

Keep the docs contribution, but revise the added example so each Telegram account explicitly scopes the shared group under `channels.telegram.accounts.<id>.groups` or uses an equivalent allowlist-scoped pattern.

## Reproduction Assessment

Yes. Source inspection reproduces the issue by comparing the PR sample to `resolveChannelGroupPolicy`, account config merging, Telegram docs, and existing tests for multi-account group inheritance.

## Solution Assessment

No. The docs direction is useful, but the current patch is not the best final form until the sample includes account-level group scoping for the intended Telegram group.
