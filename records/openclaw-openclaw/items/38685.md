## Summary

Keep open: the contribution fills a real Telegram multi-agent docs gap, and the earlier placeholder/groupAllowFrom feedback appears addressed, but the current sample still uses unscoped group access for a one-group recipe.

## What This Changes

This PR adds a dedicated multi-agent docs section for running multiple Telegram bot accounts in one group, including config, BotFather privacy notes, sender allowlists, and routing bindings.

## Best Possible Solution

Keep the docs contribution, but revise the Telegram example so each participating account explicitly scopes the shared group under account-level groups or uses an equivalent scoped allowlist pattern.

## Reproduction Assessment

Yes. Source inspection reproduces the defect by comparing the PR sample to resolveChannelGroupPolicy and Telegram account merging: groupPolicy open with no groups leaves chat allowlisting disabled, and multi-account Telegram configs do not inherit channel-level groups.

## Solution Assessment

No. The docs direction is useful, but the patch is not the narrowest safe solution until the sample includes account-level groups entries for the intended Telegram group or an equivalent scoped allowlist pattern.
