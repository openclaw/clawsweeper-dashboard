## Summary

Keep this PR open: the Telegram multi-agent recipe fills a real docs gap and is not implemented on current main, but the proposed sample still recommends unscoped open group access for a setup described as one intended group.

## What This Changes

The PR adds a multi-agent docs section showing two Telegram bot accounts bound to two agents in one shared group, with BotFather privacy setup and group sender allowlist notes.

## Best Possible Solution

Keep the Telegram recipe, but scope each bot account to the intended group chat with account-level groups and keep sender IDs under the documented sender allowlist fields.

## Reproduction Assessment

Yes for the docs defect. Source inspection shows the PR sample uses open group policy without groups, while current docs/runtime make that unscoped for group chat access.

## Solution Assessment

No. The docs direction is useful, but the sample should follow the current Telegram groups contract instead of recommending open policy alone for a one-group setup.
