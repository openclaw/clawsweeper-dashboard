## What This Changes

The PR adds a `docs/concepts/multi-agent.md` section with a JSON5 config and setup notes for routing two Telegram bot accounts to separate agents in one shared group.

## Best Possible Solution

Keep the Telegram multi-agent recipe, but revise the example to use the current Telegram contract: explicit intended group chat IDs under account-level `groups`, sender filtering with `allowFrom` or `groupAllowFrom`, and prose aligned with the merged group-ID docs.

## Solution Assessment

No. The docs direction is useful, but the current sample should be scoped to the intended Telegram group, preferably with account-level `groups` entries for this multi-account setup.
