## Summary

Keep this PR open because the Telegram multi-agent recipe is still useful and not already on current main, but the current patch should not merge while its one-group setup recommends unscoped Telegram group access.

## What This Changes

The PR adds a `docs/concepts/multi-agent.md` example for running two Telegram bot accounts bound to two agents in one shared group.

## Best Possible Solution

Keep the Telegram multi-agent recipe, but make the sample copy-paste safe by scoping each account to the intended group chat and using the documented sender allowlist fields.

## Reproduction Assessment

Yes for the docs defect: the PR-head sample uses `groupPolicy: "open"` on both accounts and no `groups` entries. Current docs and runtime show that this allows any group where the bot is added to pass group-ID checks.

## Solution Assessment

No, not as proposed. The docs direction is useful, but the sample should follow the current Telegram group contract with per-account group scoping or an equivalent documented allowlist pattern before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93
