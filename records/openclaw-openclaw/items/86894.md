## What This Changes

The issue requests an explicit opt-in path for Control UI/WebChat to auto-play assistant audio attachments delivered through MEDIA.

## Best Possible Solution

Add a default-off Control UI/WebChat audio-autoplay design that covers the intended MEDIA and TTS cases, includes an explicit user/operator enable step, and documents the chosen setting or directive without changing existing users' playback behavior.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection confirms the current and shipped renderer use a manual audio control and the WebChat schema/docs expose no autoplay option.

## Solution Assessment

Unclear. A default-off opt-in with a permission gesture is the right safety shape, but maintainers still need to choose whether the durable surface is WebChat config, a UI preference, a per-message directive, or a shared design with the TTS autoplay issue.
