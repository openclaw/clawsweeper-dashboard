## Summary

Keep open. Current main and v2026.6.8 still render assistant MEDIA audio as manual native controls only, and the requested opt-in autoplay behavior needs maintainer UX/product direction because WebChat has no persisted config section and the permission/setting/directive surface is undecided.

## What This Changes

Request opt-in Control UI autoplay for assistant audio attachments delivered via MEDIA, covering TTS and non-TTS audio.

## Best Possible Solution

Add a default-off Control UI assistant-audio autoplay design that covers MEDIA and TTS attachments, uses explicit user consent or a clearly scoped directive, avoids reviving channels.webchat config, and documents the chosen behavior with browser proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection confirms current main and v2026.6.8 render playable but manual assistant audio controls and expose no autoplay opt-in surface.

## Solution Assessment

Unclear. A default-off opt-in with a user gesture is the right safety shape, but maintainers still need to choose the durable browser-local setting, directive, or other permission surface before implementation.
