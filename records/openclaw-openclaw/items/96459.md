## Summary

Keep open for maintainer review: source inspection supports the bug and the patch appears correct with sufficient real QMD proof, but it changes plugin-facing memory lifecycle semantics that should be accepted explicitly before merge.

## What This Changes

The PR adds a scoped `index-managers` mode to the memory search close facade/runtime and uses it from active-memory timeout cleanup so memory-core releases disposable index managers without closing the shared QMD manager.

## Best Possible Solution

Land this PR after maintainer acceptance of the scoped lifecycle contract, keeping the legacy fallback and focused regression tests.

## Reproduction Assessment

Yes for source-level reproduction: current main routes active-memory timeout cleanup into full manager close, and memory-core then closes the cached QMD manager that later chat `memory_search` calls can reuse. I did not run a live gateway timeout reproduction in this read-only review.

## Solution Assessment

Yes, conditionally: the patch is the narrow owner-boundary fix if maintainers accept the optional lifecycle contract. A separately named release facade remains the safer alternative if maintainers reject overloading the close facade with scoped cleanup.
