Full review comments:

- **[P2] Fall back when the release hook is absent:** `src/plugins/memory-runtime.ts:102`
  - body: `releaseActiveMemoryIndexManagers` only calls the new optional release hook. Since active-memory now calls this path instead of the shipped close facade, any loaded memory runtime that only implements `closeMemorySearchManager` stops receiving timeout cleanup and can leak the resources this path is meant to release. Fall back to the existing scoped close hook when the release hook is absent, or make the strict no-op behavior an explicit maintainer decision.
  - confidence: 0.88
- **[P2] Update timeout tests to the release facade:** `extensions/active-memory/index.ts:966`
  - body: Changing the timeout cleanup caller to `releaseActiveMemoryIndexManagers` leaves `extensions/active-memory/index.test.ts` mocking only `closeActiveMemorySearchManager` and asserting that old spy in the timeout cases. That test module will either fail on the missing mock export or stop proving the new cleanup path; update the mock and expectations to the release facade.
  - confidence: 0.94
- **[P2] Refresh the SDK surface budget for the new export:** `src/plugin-sdk/memory-host-search.ts:97-102`
  - body: Adding `releaseActiveMemoryIndexManagers` increases the public SDK surface, and CI now fails `test/scripts/plugin-sdk-surface-report.test.ts` because the accepted public export/callable export counts were not updated.
