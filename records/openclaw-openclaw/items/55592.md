## Summary

Keep open. The underlying bug is valid because current main still hides deleted chat groups only in browser storage, and this PR moves in the right direction by adding a backend truncate RPC. The latest PR head is still not merge-ready because its destructive truncate path maps the UI sequence number by raw JSONL order, while current chat history sequence numbers come from the active SessionManager branch for tree transcripts.

## What This Changes

This PR adds a `sessions.truncate` Gateway RPC with protocol and Swift model wiring, operator scope registration, server-side transcript rewrite logic, and Control UI delete handling that calls the backend instead of only persisting a local deleted-message marker.

## Best Possible Solution

Keep the PR open and require the truncate implementation to resolve the requested UI sequence through the active SessionManager branch entry or id before performing any destructive rewrite. The mergeable shape should include regression coverage for linear transcripts, branched transcripts with abandoned entries, compaction entries, stale seq/no-op behavior, webchat rejection, active-run cleanup, file-mode preservation, rollback behavior, Control UI rollback behavior, and a user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a tree transcript with an abandoned raw entry before the active branch: `readSessionMessages()` assigns UI seqs from `SessionManager.getBranch()`, while the PR counts raw JSONL message and compaction lines, so the same seq targets a different raw line.

## Solution Assessment

No. The direction is right, but the current implementation is not the safest maintainable fix until it maps the UI target through the same active-branch semantics as chat history and only then performs the destructive transcript mutation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Map truncate seqs through the active branch:** `src/gateway/server-methods/sessions.ts:1387-1390`
  - body: The UI seq values come from `readSessionMessages()`, which assigns them while iterating `SessionManager.open(filePath).getBranch()` for tree transcripts. This scan counts every raw JSONL message/compaction line instead, including abandoned branch entries, so deleting seq N in a branched session can cut the wrong file line and remove unrelated history. Resolve the seq to an active-branch entry/id before choosing the raw cut point.
  - confidence: 0.9
