## Summary

Keep open: the PR addresses a real Control UI delete gap on current main, but the patch still maps a rendered visible sequence number to raw JSONL order and lacks contributor real-behavior proof, so it should not merge yet.

## What This Changes

Adds a `sessions.truncate` Gateway RPC, protocol/Swift model wiring, and Control UI delete integration to truncate backend transcripts, plus minor style and formatting changes.

## Best Possible Solution

Keep the backend-synced delete direction, but target a stable active-branch transcript record or record id before mutating JSONL, then add regression coverage and redacted real Control UI proof.

## Reproduction Assessment

Yes. Source inspection shows current main delete is localStorage-only, and PR head is source-reproducible against current active-branch transcript semantics where rendered seq order can differ from raw JSONL order.

## Solution Assessment

No. Backend-synced delete is the right direction, but raw ordinal truncation is not the best implementation for current parent-chain transcript semantics; the backend should resolve a selected active record or stable record id before slicing.
