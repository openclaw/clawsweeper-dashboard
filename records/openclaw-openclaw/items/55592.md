## Summary

Keep open: this PR addresses a real Control UI transcript-delete bug, but the latest head still has a blocking data-integrity issue because the new backend truncate handler interprets UI-visible sequence numbers as raw JSONL ordinals. The PR also still lacks contributor real-behavior proof for the destructive transcript mutation path.

## What This Changes

Adds a `sessions.truncate` Gateway RPC, protocol/Swift model wiring, and Control UI delete integration so deleting a chat message group truncates the backend transcript, plus small style/formatting changes.

## Best Possible Solution

Keep the backend-synced delete direction, but resolve the delete target through selected active transcript records or a stable transcript record id before slicing, then add focused regression coverage and real behavior proof.

## Reproduction Assessment

Yes, by source inspection. Current main is localStorage-only for delete, and PR head is source-reproducible against the existing active-branch fixture where raw JSONL order and rendered `__openclaw.seq` order diverge.

## Solution Assessment

No. A backend-synced delete is the right direction, but raw ordinal truncation is not the best implementation for current parent-chain transcript semantics; the backend should map the visible active record or stable record id before mutating the file.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Map truncate seqs through the active branch:** `src/gateway/server-methods/sessions.ts:1387-1390`
  - body: The UI sends the rendered `__openclaw.seq` after active-branch selection, but this loop counts every raw `message` or `compaction` record. In a branched transcript with an abandoned raw entry before the active branch, visible seq 1 points at the clean branch while this handler cuts at the abandoned record and removes history the user did not select. Resolve the target through selected transcript records or a stable record id before slicing.
  - confidence: 0.92
