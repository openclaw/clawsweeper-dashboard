## What This Changes

Adds backend-synced Control UI message deletion by introducing a `sessions.truncate` Gateway RPC and wiring protocol, Swift models, and UI delete handling.

## Best Possible Solution

Revise the PR to target a selected active-branch transcript record or stable transcript id before truncating, add focused Gateway regression coverage, and then provide redacted real Control UI proof.

## Reproduction Assessment

Yes, source-reproducible. Current main delete only writes local `DeletedMessages` storage, and PR head maps rendered active-branch seqs onto raw JSONL ordinals while the active-branch fixture demonstrates visible seqs can diverge from raw transcript order.

## Solution Assessment

No. Backend-synced delete is the right direction, but raw ordinal truncation is not the safest maintainable implementation for parent-chain transcripts.
