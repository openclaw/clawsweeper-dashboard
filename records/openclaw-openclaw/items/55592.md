## Summary

Keep open. Current main still has the reported Control UI local-only delete behavior and does not expose `sessions.truncate`, but the PR is not merge-ready because its destructive backend truncate maps UI sequence numbers by raw JSONL order instead of the active `SessionManager` branch used to produce those UI sequence numbers.

## What This Changes

This PR adds a `sessions.truncate` Gateway RPC with protocol and Swift model wiring, operator scope registration, server-side transcript rewriting, and Control UI delete handling that calls the backend instead of only hiding messages locally.

## Best Possible Solution

Keep the PR open, but require the backend truncate implementation to resolve the browser-provided seq through the same active `SessionManager` branch used by `readSessionMessages()` before choosing the raw file cut point. The mergeable version should include focused regression coverage for branched transcripts, linear transcripts, compaction markers, stale seq/no-op behavior, active-run cleanup, webchat rejection, file-mode preservation, rollback, and Control UI optimistic rollback.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a tree transcript with an abandoned branch entry before the active branch: current main assigns UI seqs from `SessionManager.open(filePath).getBranch()`, while the PR handler counts raw JSONL message and compaction lines, so the same seq can target a different raw line.

## Solution Assessment

No. The direction is right, but this is not the safest implementation until the destructive rewrite maps seqs through active branch entry IDs instead of raw line order; that is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Map truncate seqs through the active branch:** `src/gateway/server-methods/sessions.ts:1387-1390`
  - body: The browser seq comes from `readSessionMessages()`, which assigns it while iterating `SessionManager.open(filePath).getBranch()` for tree transcripts. This loop counts every raw JSONL message/compaction line instead, including abandoned branch entries, so deleting seq N in a branched session can cut from the wrong file line and remove unrelated history. Resolve the seq to an active-branch entry/id before truncating.
  - confidence: 0.9
