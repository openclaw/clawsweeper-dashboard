Full review comments:

- **[P1] Register truncate in the active protocol package:** `src/gateway/protocol/schema/sessions.ts:166-172`
  - body: Current main no longer uses `src/gateway/protocol`; server methods import validators from `packages/gateway-protocol/src/index.js`. Adding `SessionsTruncateParamsSchema` and the validator only under the retired path leaves `sessions.truncate` unavailable or stale after merge. Port the schema, types, `ProtocolSchemas`, validators, generated models, and imports to the active package.
  - confidence: 0.91
- **[P1] Carry agentId through truncate requests:** `ui/src/ui/app-render.ts:1510`
  - body: Current reset/delete/compact session mutations carry optional `agentId` and resolve selected global sessions before mutating transcripts. This new request sends only `{ key, seq }`, and the schema does not allow `agentId`, so deleting from a selected non-default global session can resolve and rewrite the wrong transcript. Add `agentId` to the protocol and pass the same scoped params used by existing session mutations.
  - confidence: 0.88
- **[P1] Truncate by transcript index, not raw JSONL count:** `src/gateway/server-methods/sessions.ts:1386-1401`
  - body: The UI seq is assigned after current main selects the active transcript parent chain, but this handler counts raw physical JSONL lines and slices `allLines`. Branch/leaf updates, side appends, or compaction can make raw order differ from visible seq, causing the delete to remove the wrong range or leave hidden sensitive entries. Use the session transcript index/accessor boundary so the mutation targets the active record identity.
  - confidence: 0.86
