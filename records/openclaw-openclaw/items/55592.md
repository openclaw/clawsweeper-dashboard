## Summary

Keep open: the PR addresses a real Control UI/backend transcript divergence, but the current head is not merge-ready because the truncate handler maps a UI active-history sequence number onto raw JSONL records. Branched or compacted transcripts can therefore truncate from the wrong record or resurrect abandoned history, and the PR still lacks redacted real behavior proof plus has a conflicting head against current main.

## What This Changes

The branch adds a new `sessions.truncate` Gateway RPC, protocol/Swift model registration, and Control UI delete wiring so deleting a message group is meant to truncate backend JSONL history.

## Best Possible Solution

Make truncation branch-safe by resolving the selected UI message through the active transcript graph and rewriting only the intended active-history prefix, then rebase onto the current Gateway method registry and provide redacted real Control UI plus backend transcript proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main assigns `__openclaw.seq` after active transcript branch selection, while the PR re-counts raw JSONL records before truncating.

## Solution Assessment

No. A raw `seq` scan is not the best fix because it does not share the active transcript identity contract; the safer path is to use the transcript index/id/offset semantics and avoid resurrecting stale branches.
