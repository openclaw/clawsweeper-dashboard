Full review comments:

- **[P1] Carry selected agent scope into truncate:** `ui/src/ui/app-render.ts:1508`
  - body: The new UI request sends only `{ key, seq }`. Current main routes `sessions.reset`, `sessions.get`, and `sessions.compact` through selected-agent scope, so deleting from `global` while a non-default agent is selected can truncate or look up the wrong session unless `agentId` is included in the schema, UI request, handler target resolution, events, and tests.
  - confidence: 0.9
- **[P1] Truncate through the active transcript index:** `src/gateway/server-methods/sessions.ts:1368-1370`
  - body: The handler maps `seq` by walking every physical JSONL line and then writes `allLines.slice(0, freshCutLineIndex)`. Current transcript reads derive visible `seq` from the active parent/leaf path, so this can delete side-branch or metadata records incorrectly, or preserve records that are no longer in the active conversation; use the current transcript accessor/index byte ranges instead.
  - confidence: 0.88
- **[P1] Abort before taking the current non-reentrant scan lock:** `src/gateway/server-methods/sessions.ts:1288`
  - body: The Phase 1 scan relies on `acquireSessionWriteLock({ sessionFile })` being reentrant, but current main defaults `allowReentrant` to `false`. Active runs hold that lock for the attempt, so deleting during a response can time out before Phase 2 gets a chance to abort and release the run; rework the rebase so cleanup/scan ordering matches current lock semantics.
  - confidence: 0.82
