## Summary

Keep open. Current main still lacks the requested portable message-tool multi-select contract: the shared presentation schema, public SDK export, docs, and channel renderers support buttons and a single select, while provider-native Discord/Slack capabilities and Telegram poll/button paths do not provide one cross-platform min/max multi-select submission with selected values returned in one callback. Related PR #18286 is useful design history but was closed unmerged and no dialog/multiselect API is present on main.

## What This Changes

Add a portable multi-select capability to the message tool so agents can show visual options, let users batch-select them, and receive all selected values in one interaction.

## Best Possible Solution

Add an additive portable batch-selection contract to MessagePresentation/message tool, export it through the plugin SDK, document it on the message presentation page, and implement channel-specific rendering plus callback normalization for Slack, Discord, Telegram, and a conservative text fallback.

## Reproduction Assessment

Not applicable as a feature request, but we do have a high-confidence verification path: current-main source and docs show only single-select presentation and no portable multi-select callback contract.

## Solution Assessment

Yes, keeping this open is the best current outcome. The related dialog PR is closed unmerged, and the maintainable path is a deliberate additive public contract rather than treating provider-native select handling as completion.
