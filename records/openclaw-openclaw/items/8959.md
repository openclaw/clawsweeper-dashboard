## Summary

Keep open. Current main and v2026.5.20 still lack the requested portable message-tool multi-select contract; existing single-select presentation, Telegram managed multiselect/polls, Discord component primitives, Slack Block Kit primitives, and the merged plugin interception hook are adjacent but do not satisfy the cross-channel selected-values callback surface.

## What This Changes

The issue requests a cross-platform multi-select option for the message tool so users can batch-select items and submit them to the agent in one interaction.

## Best Possible Solution

Decide the API boundary first, then add an additive portable batch-selection contract to message presentation/message-tool or explicitly expose enough plugin APIs for a ClawHub implementation to provide the same cross-channel behavior.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source, docs, release, and related-PR inspection give high confidence that the requested portable multi-select contract is absent today.

## Solution Assessment

Unclear pending maintainer product/API review: the requested direction is plausible, but the permanent public block shape, selected-values payload, channel fallback behavior, and plugin-vs-core boundary need a decision first.
