## Summary

Keep open: current main and the latest release still expose a single-select message presentation contract only. The merged plugin hook and Telegram callback fixes are useful adjacent work, but they do not provide the requested portable message-tool batch-select UI, min/max selection semantics, or structured selected-values callback across channels.

## What This Changes

The issue requests a portable message-tool multi-select UI and callback contract so users can select multiple items in one interaction.

## Best Possible Solution

Design one additive portable batch-select presentation/message-tool contract with typed selected-value aggregation, explicit per-channel limits, deterministic fallback text, docs, SDK exports, and channel adapter coverage.

## Solution Assessment
