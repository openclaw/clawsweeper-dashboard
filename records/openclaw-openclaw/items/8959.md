## Summary

Keep open. Current main and the latest tagged release still lack the requested portable message-tool multi-select contract; existing Slack/Discord/Telegram capabilities and the merged plugin hook cover adjacent workflows but do not define a shared batch-selection API or one selected-values callback across channels.

## What This Changes

The issue requests a cross-platform multi-select component for the shared `message` tool so users can pick several options and submit them in one interaction.

## Best Possible Solution

Add an additive portable batch-selection contract to message presentation/message-tool, export and document it through the plugin SDK, then implement Slack, Discord, Telegram, and fallback callback behavior after maintainer API review.
