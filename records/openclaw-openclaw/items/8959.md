Keep open. Current main and the latest shipped release still expose only buttons plus single-select presentation in the message tool; the linked dialog PR was closed unmerged, and the merged before_agent_reply hook enables adjacent plugin-side dialogs but does not add the requested portable message-tool API.

Request a portable message-tool multi-select control so agents can present options, receive batch selections in one callback, and avoid one agent turn per toggle.

## Best Possible Solution

Add an additive portable batch-selection contract to message presentation/message-tool, export and document it through the plugin SDK, then implement Slack, Discord, Telegram, and fallback callback behavior after maintainer API review.
