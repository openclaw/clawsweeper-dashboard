## Summary

Keep this open; current main still lacks the requested portable message-tool multi-select contract, and the closed dialog PR did not land an equivalent implementation.

## What This Changes

Request to add a cross-platform multi-select option to the message tool so users can choose multiple items visually and return one callback payload.

## Best Possible Solution

Add an additive portable multi-select block and selected-values callback contract to message presentation/message tool, export it through the SDK, document it, and implement channel render/parsing plus fallback coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence gap check because the current schema, SDK export, docs, and renderers only model single-select presentation behavior.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is a shared message-presentation contract first, followed by channel render/callback implementations or an explicit plugin API decision if core should stay lean.
