## Summary

Keep open: current main and v2026.6.8 still only expose single-select message-presentation/message-tool semantics. The closed dialog PR, merged plugin hook, and Discord-only component support are useful adjacent history, but they do not provide the requested portable batch-select contract with min/max values and one selected-values callback across channels.

## What This Changes

The issue requests a cross-platform multi-select capability for the message tool so users can select multiple options visually and submit them in one agent turn.

## Best Possible Solution

Design an additive portable batch-select block in the message-presentation/message-tool contract with typed selected-values callbacks, explicit channel limits, deterministic fallback text, SDK/docs coverage, and channel adapter support.

## Reproduction Assessment

Not applicable: this is a new capability request, not broken established behavior. Source, docs, channel renderer, related-PR, and release inspection give high confidence that the requested portable batch-select contract is absent.

## Solution Assessment

No, not as a direct channel-only or hook-only implementation. The maintainable path is an additive public message-presentation/message-tool contract with selected-values semantics and per-channel fallback behavior chosen before implementation.
