## Summary

Keep open: current main still lacks the requested portable message-tool multi-select contract. The existing shared presentation surface, SDK export, docs, and channel renderers cover single-select only; the merged before_agent_reply hook makes plugin-hosted dialog workflows more feasible, but it does not add a cross-channel message-tool multi-select API or callback contract.

## What This Changes

The issue requests a portable message-tool multi-select UI so users can choose several options and submit one structured selection callback across channels.

## Best Possible Solution

Keep this open for maintainer product/API design; if accepted, add an additive portable batch-selection contract in message presentation, SDK exports, docs, channel renderers, callback handling, fallback text, and focused compatibility tests.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Current-main source inspection shows the requested contract is absent from the message tool, shared presentation types, SDK export, docs, and renderer paths.

## Solution Assessment

No autonomous fix is the best next step because this changes a public cross-channel interaction contract. A plugin-hosted dialog tool is now more feasible through before_agent_reply, but that is not the same as implementing multi-select in the built-in message tool.
