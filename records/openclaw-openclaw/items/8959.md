## Summary

Keep open: current main and v2026.6.8 still lack the requested portable message-tool batch multi-select contract. The existing shared presentation surface supports single-select blocks, while adjacent Slack/Discord/native primitives and the merged plugin hook do not provide a cross-channel min/max multi-select UI with one selected-values callback.

## What This Changes

Request to add a cross-platform multi-select control to the message tool so users can choose multiple options visually and submit them in one interaction.

## Best Possible Solution

Design one additive portable batch-select presentation/message-tool contract with typed selected-value aggregation, explicit channel limits, deterministic fallback text, SDK/docs updates, and channel adapter coverage.

## Reproduction Assessment

Not applicable: this is a new-capability request, not a broken established behavior. Source, docs, channel-renderer, related-PR, and v2026.6.8 inspection give high confidence that the requested portable batch-select contract is absent.

## Solution Assessment

Unclear on the exact API shape, but the best owner boundary is the shared message presentation/message tool contract rather than a channel-only renderer or hook-only workaround.
