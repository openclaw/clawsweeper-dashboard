## Summary

Keep this open. Current main still lacks the requested portable message-tool multi-select contract; the schema, SDK export, docs, and channel renderers remain single-select shaped, and the related dialog PR was closed unmerged while the later plugin hook only provides adjacent interception infrastructure.

## What This Changes

The issue requests a portable message-tool multi-select/selectMenu block with min/max selections and one selected-values callback across Slack, Discord, Telegram, and fallback paths.

## Best Possible Solution

Add an additive portable multi-select block and selected-values callback contract to message presentation/message tool, export it through the SDK, document it, and implement channel-specific render/parsing plus fallback text coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence gap check because the current schema, SDK export, docs, and renderers only model single-select presentation behavior.

## Solution Assessment

Unclear until maintainers choose the additive API shape. The narrow maintainable direction is a shared message-presentation contract first, then channel render/callback implementations, rather than a Slack-, Discord-, or Telegram-only escape hatch.
