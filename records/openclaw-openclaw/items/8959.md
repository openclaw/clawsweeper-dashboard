## Summary

Keep open; current main still lacks the requested portable message-tool multi-select contract across the shared schema, SDK export, docs, and renderers.

## What This Changes

Add a cross-platform multi-select capability to the message tool so users can choose multiple visual options and submit them in one agent callback.

## Best Possible Solution

Define an additive batch-selection contract at the shared message presentation/message-tool boundary, or explicitly document a plugin path if maintainers decide this UX belongs outside core.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence gap check because current main still models only single-select presentation in the tool schema, SDK export, docs, and portable renderers.

## Solution Assessment

Unclear until maintainers choose the public API and channel callback semantics. The narrow maintainable direction is contract-first shared presentation work, with channel render/callback implementations and docs following that contract.
