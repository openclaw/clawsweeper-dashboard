## Summary

Keep open; current main still only exposes single-select message presentation, and the merged plugin hook plus the closed dialog PR do not implement the requested portable message-tool multi-select contract.

## What This Changes

Add a cross-platform message-tool multi-select capability so agents can send visual option lists and receive all selected values in one callback.

## Best Possible Solution

Define an additive shared message presentation/message-tool selected-values contract, then implement native render/parsing plus fallback behavior in the supported channel plugins.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence gap check because current main only models single-select presentation behavior in the message tool, SDK export, docs, and channel renderers.

## Solution Assessment

Unclear until maintainers choose the public API shape and plugin/core boundary. If core owns this UX, the narrow maintainable path is a shared presentation contract first, followed by channel render and callback implementations.
