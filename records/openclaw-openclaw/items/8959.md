## Summary

Keep open because current main still exposes only a single-select message presentation/message-tool contract, while the requested portable batch multi-select API and callback semantics remain a product/API design gap.

## What This Changes

Request an additive cross-platform multi-select component for the message tool so users can choose multiple options and submit one selected-values callback.

## Best Possible Solution

Add an additive portable batch-selection contract to message presentation/message tool, export it through the public SDK, document it, and implement channel render plus callback handling with text fallback.

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence current-main gap because the public message-tool and presentation contracts still only define single-select semantics.

## Solution Assessment
