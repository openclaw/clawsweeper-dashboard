## Summary

Keep open; current main still lacks the requested portable message-tool multi-select contract, and the related dialog-tool PR is closed unmerged rather than superseding this feature request.

## What This Changes

The issue requests a cross-platform message-tool multi-select UI that batches selected option values into one callback/agent turn.

## Best Possible Solution

Define an additive batch-selection contract at the shared message presentation/message-tool boundary, export and document it through the plugin SDK, then implement channel-specific rendering and callback handling behind that contract.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence gap check because current main still models only single-select presentation in the tool schema, SDK export, docs, and portable renderers.

## Solution Assessment

Unclear until maintainers choose the public API and callback semantics. The narrow maintainable direction is contract-first shared presentation work; channel-only native support would not solve the cross-platform request.
