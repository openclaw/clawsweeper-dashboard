## Summary

Keep open: current main and the latest release still expose only a single-select portable message presentation/message-tool contract. The closed dialog PR and the merged `before_agent_reply` hook are useful adjacent context, but they do not add the requested cross-channel multi-select API or one structured selected-values callback.

## What This Changes

Requests a cross-platform multi-select/selectMenu message-tool surface so users can pick several options visually and submit them in one callback.

## Best Possible Solution

Add an additive shared batch-selection contract to message presentation/message tool, export and document it through the public SDK, then implement channel render/callback handling for Slack, Discord, Telegram, and text fallback.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection gives high confidence that current main still lacks the requested portable multi-select contract.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable path is an explicit shared presentation/message-tool contract rather than treating the closed dialog PR, the plugin hook, or one channel's native component support as completion.
