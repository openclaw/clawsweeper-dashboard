## Summary

Keep open. Current main still has a portable message presentation/message tool contract for buttons and single-select only; adjacent Telegram managed-select callbacks, Slack selected_options parsing, Discord select min/max internals, and the merged before_agent_reply plugin hook are useful building blocks but do not provide the requested cross-platform message-tool multi-select API or selected-values callback.

## What This Changes

Request adds a cross-platform multi-select/selectMenu capability to the message tool so users can batch-select options and submit them in one interaction.

## Best Possible Solution

Add an additive portable batch-selection contract in message presentation/message tool, document it, export it through the SDK, and implement channel render/callback handling with text fallback.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence current-main gap: the public message-tool/presentation contract still has only single-select semantics.

## Solution Assessment

No current implementation or close path is sufficient. The merged plugin hook is useful prior work, but the narrow maintainable solution requires an explicit shared API decision rather than treating channel-specific callbacks or a plugin-only workaround as completion.
