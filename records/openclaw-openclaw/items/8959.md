## Summary

Keep open. Current main and the latest release still lack the requested portable message-tool multi-select contract; adjacent native select, poll, wizard, and plugin-hook paths do not provide the shared batch-selection API or one selected-values callback across channels.

## What This Changes

The issue requests a cross-platform multi-select capability in the `message` tool so users can pick multiple options and submit them in one interaction.

## Best Possible Solution

Add an additive portable batch-selection contract to message presentation/message-tool, export and document it through the plugin SDK, then implement Slack, Discord, Telegram, and fallback callback behavior after maintainer API review.
