## Summary

Keep open: current main and v2026.6.6 still expose only single-select presentation/message-tool semantics, while the related dialog PR closed unmerged and the merged plugin hook only covers plugin-owned workflow interception.

## What This Changes

The issue requests a cross-platform batch multi-select control for the message tool so agents can receive multiple selected values in one user interaction.

## Best Possible Solution

Design one additive batch-select presentation/message-tool contract with typed selected-value aggregation, explicit per-channel limits, deterministic text fallback, and channel adapters for Slack, Discord, Telegram, and plain transports.

## Reproduction Assessment

Not applicable: this is a new-capability request, not a broken established behavior. Source and docs inspection give high confidence that the requested portable batch-select contract is absent on current main and v2026.6.6.

## Solution Assessment

Unclear as proposed: the one-callback goal is sound, but the exact selectMenu/multiSelect shape needs maintainer API direction. The maintainable path is a minimal shared presentation contract plus channel-specific adapters and fallback semantics, with workflow state left to plugin hooks where appropriate.
