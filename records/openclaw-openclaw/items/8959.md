## Summary

Keep open; current main still lacks the requested portable message-tool multi-select contract. The code supports shared buttons and single-select presentation, while the related dialog PR is closed unmerged and the later Telegram/plugin work is adjacent infrastructure rather than the requested cross-channel batch-selection UI.

## What This Changes

Add a cross-platform multi-select capability to the message tool so users can visually choose multiple options and submit them in one agent turn.

## Best Possible Solution

Define an additive shared batch-selection contract in the message presentation/message-tool boundary, export and document it through the plugin SDK, and implement compatible render plus callback behavior for supported channels and text fallback.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence gap check because current main still models the shared message presentation surface as buttons plus single select.

## Solution Assessment

Unclear until maintainers choose the public API and callback semantics. The maintainable path is contract-first shared presentation work, or an explicit plugin-first design with any missing SDK seams, rather than a one-channel native component patch.
