## Summary

Keep open; current main still lacks the requested portable message-tool multi-select contract. The shared message presentation surface remains single-select plus buttons, and the related dialog PR is closed unmerged while the merged plugin hook only enables interception, not a cross-channel batch-selection UI/callback contract.

## What This Changes

The issue requests a cross-platform batch-selection capability in the message tool so agents can collect multiple selected values in one user interaction.

## Best Possible Solution

Define an additive shared batch-selection block or message-tool parameter, export and document it through the plugin SDK, then implement compatible render and callback handling for Slack, Discord, Telegram, and fallback text paths.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence gap check because current main still models shared presentation as buttons plus single-select.

## Solution Assessment

Unclear until maintainers choose the public API and callback semantics. The narrow maintainable direction is contract-first shared presentation work, not a channel-only native component patch.
