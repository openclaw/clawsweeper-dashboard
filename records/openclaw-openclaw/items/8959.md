Keep open. Current main and the latest release still lack a portable message-tool multi-select contract; nearby Telegram, Slack, Discord, and plugin-hook primitives are useful context but do not provide the requested cross-channel selected-values callback surface.

## What This Changes

Request an additive cross-platform multi-select component for the message tool so users can choose multiple options and submit them in one callback instead of one agent turn per toggle.

## Best Possible Solution

Add an additive portable batch-selection contract to message presentation/message-tool, export and document it through the plugin SDK, then implement consistent channel render/callback and fallback behavior after maintainer API review.
