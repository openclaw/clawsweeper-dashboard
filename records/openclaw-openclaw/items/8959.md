## Summary

Keep open: current main and latest release still expose only a portable single-select message presentation/message-tool contract. The closed dialog-tool PR and merged before-agent-reply hook are useful adjacent design history, but neither provides the requested cross-platform batch-select UI, min/max semantics, or one selected-values callback.

## What This Changes

Request to add a portable message-tool multi-select/select-menu surface so users can choose multiple options visually and submit them in one interaction.

## Best Possible Solution

Design one additive portable batch-select presentation/message-tool contract with typed selected-value aggregation, explicit channel limits, deterministic fallback text, docs, SDK exports, and channel adapter coverage.

## Reproduction Assessment

Not applicable: this is a new-capability request, not a broken established behavior. Source, docs, channel-renderer, related-PR, and v2026.6.8 inspection give high confidence that the requested portable batch-select contract is absent.

## Solution Assessment

No, not as a channel-only or hook-only implementation. The maintainable solution is an additive public contract with selected-values semantics and per-channel fallback behavior chosen before implementation.
