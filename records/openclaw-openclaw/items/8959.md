## Summary

Keep this open; current main still lacks the requested portable message-tool multi-select contract, and the related dialog PR plus the later plugin hook do not provide the same cross-channel message presentation feature.

## What This Changes

The issue requests a portable `message` tool multi-select component that lets users batch-select options and return all selected values in one callback.

## Best Possible Solution

Define an additive portable selected-values contract in message presentation/SDK, then implement channel render/parsing and text fallback behavior for the supported chat plugins.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence gap check because current main only models single-select presentation behavior in the message tool, SDK export, docs, and channel renderers.

## Solution Assessment

Unclear until maintainers choose the public API shape and core-versus-plugin boundary. If core owns this UX, the narrow maintainable path is a shared message-presentation contract first, followed by channel render/callback implementations.
