## Summary

Keep open. Current main and the latest release still lack the requested portable message-tool multi-select contract; existing single-select, Telegram managed-select, Slack/Discord native primitives, and the closed dialog PR are adjacent but do not satisfy the cross-channel selected-values callback surface.

## What This Changes

The issue requests a cross-platform message-tool multi-select component that lets users choose multiple options and submit one selected-values callback across channels.

## Best Possible Solution

Add an additive portable batch-selection contract to message presentation/message-tool, export and document it through the plugin SDK, then implement consistent Slack, Discord, Telegram, and fallback callback behavior after maintainer API review.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source and latest-release inspection give high confidence that the requested portable multi-select contract is absent because the shared message presentation surface supports buttons and single select only.

## Solution Assessment

Unclear pending maintainer product/API review: the requested direction is plausible, but the exact public SDK shape, selected-values payload, and channel fallback semantics should be decided before implementation.
