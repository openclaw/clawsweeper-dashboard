## Summary

Keep open. Current main and the latest release still expose portable buttons and single-select presentation blocks, plus adjacent poll or channel-specific controls, but not the requested cross-platform message-tool multi-select contract with one selected-values callback.

## What This Changes

Request a portable message-tool multi-select/selectMenu contract so agents can collect multiple chosen values in one user interaction across channels.

## Best Possible Solution

Add an additive portable multi-select block to the MessagePresentation/message-tool contract, export and document it through the plugin SDK, and implement channel render/callback handling plus fallback text behavior.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. The source check is high-confidence that the requested capability is absent because current message presentation supports buttons and single-select only.

## Solution Assessment

No current implementation solves it. Existing polls and channel-specific controls are adjacent, but the maintainable solution is an additive portable SDK/message presentation contract with channel-specific rendering and callback semantics.
