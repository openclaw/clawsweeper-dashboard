## Summary

Keep this open. Current main still lacks the requested portable message-tool multi-select contract: the message presentation schema, SDK export, docs, and Slack/Discord/Telegram renderers remain single-select shaped, and related PR #18286 was closed unmerged.

## What This Changes

Add a portable message-tool multi-select/select-menu capability so users can choose multiple options visually and submit all selected values in one callback across supported channels.

## Best Possible Solution

Add an additive portable multi-select block and selected-values callback contract to message presentation/message tool, export it through the SDK, document it, and implement Slack, Discord, Telegram, and fallback render/parsing coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence gap check because the current schema, SDK export, docs, and renderers only model single-select presentation behavior.

## Solution Assessment

Unclear until maintainers choose the additive API shape. The narrow maintainable direction is a shared message-presentation contract first, then channel render/callback implementations, rather than a Discord-, Slack-, or Telegram-only escape hatch.
