## Summary

Keep open; current main still does not provide the requested portable message-tool multi-select contract, and the merged plugin hook only solves part of the pre-LLM interception story rather than the shared outbound UI and callback API.

## What This Changes

Request to add a cross-platform multi-select capability to the message tool so users can choose multiple options and submit them in one interaction.

## Best Possible Solution

Define an additive batch-selection contract at the shared message presentation/message-tool boundary, or explicitly choose and document the plugin/ClawHub path if maintainers decide this UX should stay outside core.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence gap check because current main still models only single-select presentation in the tool schema, SDK export, docs, and portable renderers.

## Solution Assessment

Unclear until maintainers choose the public API and plugin boundary. The maintainable path is contract-first shared presentation work, or a documented plugin surface that can actually render and collect cross-channel multi-selects without changing core again.
