## Summary

Keep open: current main and latest release still expose a portable single-select message presentation/message-tool contract, while the requested batch multi-select payload and one selected-values callback remain missing. The merged plugin hook work helps possible plugin-based dialogs, but it does not supersede the requested cross-platform message tool surface, and the related dialog PR at https://github.com/openclaw/openclaw/pull/18286 was closed unmerged.

## What This Changes

Request a cross-platform multi-select capability in the message tool so users can select multiple options visually and submit them in one interaction.

## Best Possible Solution

Add an additive portable batch-selection contract with documented selected-values callback semantics, or explicitly choose and document a plugin/ClawHub alternative that can deliver the same workflow without changing the message tool.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new capability request. Source and docs inspection on current main verifies that the requested multi-select message-tool contract is absent.

## Solution Assessment

Unclear until maintainers choose the API direction. The narrow maintainable path is either an additive shared presentation/message-tool block with channel render and callback handling, or a documented plugin-owned alternative if the existing hook surface is deemed sufficient.
