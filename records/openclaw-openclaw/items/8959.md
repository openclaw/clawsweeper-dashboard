## Summary

Keep open; current main still lacks the requested portable message-tool multi-select contract across the shared schema, SDK export, docs, and channel renderers.

## What This Changes

Add a cross-platform message-tool multi-select component that lets users choose multiple options and submit them in one interaction.

## Best Possible Solution

Define an additive batch-selection contract at the shared message presentation/message-tool boundary, then implement channel render/callback handling and docs against that contract.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence gap check because current main still models only single-select presentation in the tool schema, SDK export, docs, and portable renderers.

## Solution Assessment

Unclear until maintainers choose the public API and callback semantics. The narrow maintainable direction is contract-first shared presentation work, with Slack, Discord, Telegram, and fallback behavior following that contract.
