## Summary

Keep open: current main still exposes a portable single-select message presentation surface, not the requested multi-select/min/max/array-callback contract, and related PR #18286 is closed unmerged.

## What This Changes

The issue requests a cross-platform message-tool multi-select/select menu contract that lets users batch-select options and submit them in one interaction.

## Best Possible Solution

Add an additive shared multi-select presentation/message-tool contract, export and document it through the plugin SDK, and implement channel-specific rendering plus one batched callback/fallback path.

## Reproduction Assessment

Not applicable as a feature request. Source inspection of current main verifies that the requested multi-select contract is absent from the schema, shared types, SDK export, docs, and channel renderers.

## Solution Assessment

Yes, keeping this open for a designed additive MessagePresentation/message-tool contract is the best path. A single-channel native select change would not deliver the requested cross-platform one-callback behavior.
