## Summary

Keep this open. Current main still lacks the requested portable message-tool multi-select contract: the shared schema, SDK type, docs, and channel renderers remain single-select shaped, and the related dialog PR was closed unmerged rather than landing this feature.

## What This Changes

Add a cross-platform message-tool multi-select control that lets users choose several options visually and submit the selected values in one interaction.

## Best Possible Solution

Add an additive portable multi-select block and callback contract to message presentation/message tool, export it through the SDK, document it, and implement Slack, Discord, Telegram, and fallback render/parsing coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence gap check because the current message presentation schema, docs, SDK export, and renderers only model single-select behavior.

## Solution Assessment

Unclear until maintainers choose the additive API shape. The narrow maintainable direction is a shared message-presentation contract first, then channel render/callback implementations, rather than adding a Discord-, Slack-, or Telegram-only escape hatch.
