## Summary

Keep open. Current main has a portable single-select presentation surface, but it still lacks the requested cross-platform multi-select/min-max/array-callback contract, and related PR #18286 was closed unmerged rather than superseding this work.

## What This Changes

The issue requests a portable message-tool multi-select/selectMenu surface that lets users choose multiple options visually and submit them as one batched callback.

## Best Possible Solution

Add an additive MessagePresentation/message-tool multi-select contract, export and document it through the plugin SDK, and implement normalized Slack, Discord, Telegram, and text-fallback behavior with focused compatibility tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection of current main verifies the requested multi-select contract is absent from the schema, shared types, SDK export, docs, and channel renderers.

## Solution Assessment

Yes, keeping this open for a designed additive public contract is the best current path. A single channel renderer tweak would not deliver the requested cross-platform one-callback selected-values behavior.
