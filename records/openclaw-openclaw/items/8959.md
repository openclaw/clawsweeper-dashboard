## Summary

Current main and release v2026.6.5 still lack the requested portable batch-selection contract. The merged plugin interception hook enables plugin-owned workflow state, but the shared presentation schema, SDK, documentation, and channel adapters remain single-select, so this issue still needs maintainer product and API direction.

## What This Changes

Add a portable message-tool multi-select interaction that batches selected values into one callback across supported channels.

## Best Possible Solution

Design a minimal additive batch-select presentation block with typed selected-value aggregation, explicit channel capability limits, and deterministic numbered-text fallback, while keeping dialog and session workflow state plugin-owned through the shipped before_agent_reply hook.

## Reproduction Assessment

Not applicable: this is a new-capability request, and source inspection provides high-confidence evidence that the portable batch-selection contract is absent rather than reproducing broken established behavior.

## Solution Assessment

The one-callback goal is sound, but the exact proposal is not yet the complete best solution; the maintainable boundary is plugin-owned workflow state plus one additive shared presentation contract with channel-specific adapters and explicit fallback semantics.
