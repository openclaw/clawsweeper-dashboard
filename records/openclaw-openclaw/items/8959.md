## Summary

Keep open: current main and v2026.6.6 still expose only single-select message presentation semantics, while the related dialog proposal closed unmerged and the merged plugin hook is only adjacent infrastructure, not the requested portable message-tool multi-select contract.

## What This Changes

Requested an additive cross-platform multi-select capability for the message tool so users can batch-select options and submit them in one callback.

## Best Possible Solution

Design one additive batch-select presentation/message-tool contract with typed selected-value aggregation, explicit per-channel limits, deterministic text fallback, and channel adapters for Slack, Discord, Telegram, and plain transports.

## Reproduction Assessment

Not applicable: this is a new-capability request, not a broken established behavior. Source and docs inspection give high confidence that the requested portable batch-select contract is absent on current main and v2026.6.6.

## Solution Assessment

No; closing or implementing this automatically would skip the required public API decision. A one-channel native selector is too narrow because the requested solution is a shared message-tool/presentation contract with callbacks and fallbacks.
