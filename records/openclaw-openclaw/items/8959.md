## Summary

Keep open. Current main still lacks the requested portable message-tool multi-select contract: the shared message presentation surface remains buttons plus single select, the SDK/docs expose no selected-values array callback, and the related dialog PR was closed unmerged in favor of a plugin-hook direction that does not implement this message-tool API.

## What This Changes

The issue requests a cross-platform message-tool multi-select capability so users can batch-select options visually and submit one callback.

## Best Possible Solution

Add an additive portable batch-selection contract to message presentation/message-tool, export and document it through the plugin SDK, then implement Slack, Discord, Telegram, and fallback callback behavior after maintainer API review.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source, docs, and latest-release inspection give high confidence that the requested portable multi-select contract is absent because message presentation supports buttons and single select only.

## Solution Assessment

Unclear pending maintainer product/API review: the direction is plausible, but the public SDK shape, min/max semantics, selected-values payload, and plugin-versus-core boundary need to be decided before implementation.
