## Summary

Keep open: current main and v2026.6.8 still expose only a portable single-select message presentation/message-tool contract. Adjacent Discord component, Slack Block Kit, Telegram managed callback/poll, and plugin hook primitives do not provide the requested cross-platform batch-select UI with min/max semantics and one selected-values callback.

## What This Changes

Request to add a portable message-tool multi-select interaction that lets users choose multiple options visually and submit all selected values in one callback across supported channels.

## Best Possible Solution

Design one additive portable batch-select presentation/message-tool contract with typed selected-value aggregation, explicit channel limits, deterministic fallback text, docs, SDK exports, and channel adapter coverage.

## Reproduction Assessment

Not applicable: this is a new-capability request, not a broken established behavior. Source, docs, channel-renderer, related-PR, and v2026.6.8 inspection give high confidence that the requested portable batch-select contract is absent.

## Solution Assessment

Unclear on the exact API shape, but the best owner boundary is the shared message presentation/message tool contract rather than a channel-only renderer or hook-only workaround.
