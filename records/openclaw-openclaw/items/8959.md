## Summary

Keep open; current main still lacks the requested portable message-tool multi-select contract, and the related dialog PR remains closed unmerged rather than providing a shipped replacement.

## What This Changes

The issue requests a cross-platform message-tool multi-select component that lets users batch-select options and submit one structured callback.

## Best Possible Solution

Define an additive shared batch-selection contract in the message presentation/message-tool surface, export and document it, then implement channel render, callback parsing, and text fallback together.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence gap check because current main still models only single-select presentation in the tool schema, SDK export, docs, and portable renderers.

## Solution Assessment

Unclear until maintainers choose the public API shape and core/plugin boundary. If OpenClaw owns this UX in the message tool, the maintainable path is contract-first shared presentation work rather than a channel-specific workaround.
