## Summary

Keep open: current main still exposes a portable single-select presentation surface, not the requested multi-select/min-max/array-callback contract, and related PR #18286 was closed unmerged.

## What This Changes

The issue requests a portable message-tool multi-select component with min/max selection limits and one callback containing all selected values.

## Best Possible Solution

Add an additive shared multi-select block/parameter, export and document it through the plugin SDK, and implement normalized render plus callback behavior for Slack, Discord, Telegram, and text fallback.

## Reproduction Assessment

Not applicable as a feature request. Source inspection of current main verifies the requested multi-select contract is absent from the schema, shared types, SDK export, docs, and channel renderers.

## Solution Assessment

Yes, keeping this open for a designed additive MessagePresentation/message-tool contract is the right direction. A single-channel native select change would not deliver the requested cross-platform one-callback behavior.
