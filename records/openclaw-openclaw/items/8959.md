## Summary

Current main and v2026.6.5 still provide only a portable single-select presentation contract; the requested one-submit batch selection remains meaningful work, but its public SDK and cross-channel semantics require maintainer product judgment.

## What This Changes

Add a cross-platform message-tool multi-select presentation that returns all selected values in one interaction and one agent turn.

## Best Possible Solution

Approve an additive typed batch-selection contract in the shared presentation/message-tool surface, with capability-aware channel renderers, one structured selected-values callback, documented fallback behavior, and focused contract plus channel tests.

Not applicable: this requests a new capability rather than reporting broken established behavior, and source inspection confirms the current contract is single-select.
