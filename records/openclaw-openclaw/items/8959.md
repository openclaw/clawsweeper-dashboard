## Summary

Keep open; current main still lacks the requested portable message-tool multi-select contract, and the related dialog PR was closed unmerged while the merged hook work only enables plugin interception, not native message-tool batch selection.

## What This Changes

Add a portable message-tool multi-select/selectMenu surface so agents can show visual options and receive multiple selected values in one callback.

## Best Possible Solution

Define an additive shared message presentation/message-tool selected-values contract, export and document it, then implement native render/parsing plus text fallback in the supported channel plugins.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence gap check because current main still models only single-select presentation behavior in the message tool, SDK export, docs, and portable channel renderers.

## Solution Assessment

Unclear until maintainers choose the public API shape and core/plugin boundary. If OpenClaw owns this UX in the message tool, the narrow maintainable path is a shared contract first, followed by channel render and callback implementations rather than a channel-only workaround.
