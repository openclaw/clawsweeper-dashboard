## Summary

Keep open: current main and v2026.6.8 still expose a portable single-select message presentation contract only. The closed dialog PR and the merged Telegram callback safety work are useful adjacent context, but neither supplies the requested cross-platform message-tool multi-select API or one selected-values callback across channels.

## What This Changes

Add a cross-platform multi-select capability to the message tool so users can choose multiple options visually and submit them in one interaction.

## Best Possible Solution

Design one additive portable batch-selection presentation/message-tool contract with typed selected-value aggregation, explicit per-channel limits, deterministic fallback text, SDK/docs coverage, and channel adapters for Slack, Discord, Telegram, and plain transports.
