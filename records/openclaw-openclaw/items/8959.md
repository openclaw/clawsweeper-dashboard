Keep open. Current main and the latest inspected release still lack the requested portable message-tool multi-select contract; adjacent platform-specific select, poll, and Telegram managed-button paths do not provide the cross-channel min/max multi-select API and single selected-values callback requested here.

## Best Possible Solution
