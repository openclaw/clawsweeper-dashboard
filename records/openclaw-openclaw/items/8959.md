## Summary

Keep open. Current main still has portable buttons and single-select presentation only; the merged plugin hook can help a plugin build dialog-style flows, but it does not add the requested cross-platform message-tool multi-select contract or selected-values callback.

## What This Changes

Add a portable multi-select capability to the message tool so users can batch-select multiple options and submit them in one interaction across supported channels.

## Best Possible Solution

Add an additive portable multi-select presentation block to the message-tool/MessagePresentation contract, export and document it through the plugin SDK, and implement channel render/callback handling plus fallback text behavior.
