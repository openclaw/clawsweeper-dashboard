## Summary

Keep this open. Current main still lacks the requested portable message-tool multi-select contract; the existing schema, SDK export, docs, and renderers remain single-select shaped, while the merged plugin hook only provides adjacent inbound interception infrastructure.

## What This Changes

Add a portable message-tool multi-select/selectMenu component so users can choose multiple options and submit one selected-values callback across channels.

## Best Possible Solution

Add an additive portable multi-select block and selected-values callback contract to message presentation/message tool, export it through the SDK, document it, and implement channel-specific render/parsing plus fallback text coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence gap check because the current schema, SDK export, docs, and renderers only model single-select presentation behavior.

## Solution Assessment

Unclear until maintainers choose the additive API shape. The maintainable direction is a shared message-presentation contract first, followed by channel render/callback implementations or an explicit plugin API if core should stay lean.
