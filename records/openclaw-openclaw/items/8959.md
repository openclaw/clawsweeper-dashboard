## Summary

Keep open. Current main still lacks the requested portable message-tool multi-select contract: the shared schema, SDK export, docs, and Slack/Discord renderers remain single-select, while adjacent Telegram/poll/plugin-hook paths do not provide the cross-platform selected-values callback requested here.

## What This Changes

Request a cross-platform message-tool multi-select UI that lets users submit several selected values in one callback.

## Best Possible Solution

Add an additive portable multi-select presentation block to the message-tool/MessagePresentation contract, export and document it through the plugin SDK, and implement channel render/callback handling plus fallback text behavior.

Not applicable: this is a feature request, not a bug report. Source inspection gives high confidence that current main lacks the requested portable message-tool multi-select contract because the shared presentation contract supports buttons and single select only.

## Solution Assessment
