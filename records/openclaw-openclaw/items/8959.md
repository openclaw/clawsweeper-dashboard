## Summary

Keep open; current main still lacks the requested portable message-tool multi-select contract, and the related dialog PR was closed unmerged rather than shipping a replacement.

## What This Changes

The issue requests an additive cross-platform multi-select surface for the message tool so users can select multiple options visually and submit them in one callback.

## Best Possible Solution

Define an additive shared message presentation/message-tool batch-selection contract, export and document it, then implement native Discord, Slack, Telegram, and fallback behavior together.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence gap check because current main still models only single-select message presentation in the tool schema, SDK export, docs, and portable renderers.

## Solution Assessment

Unclear until maintainers choose the public API shape and core/plugin boundary. If OpenClaw owns this UX in the message tool, the maintainable path is contract-first shared presentation work rather than a one-channel workaround.
