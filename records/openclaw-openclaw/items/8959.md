Keep open. Current main still lacks the requested portable message-tool multi-select contract, and the related dialog/multiselect PR was closed unmerged rather than landing a replacement API.

## What This Changes

The issue requests a cross-platform message-tool multi-select/selectMenu capability with min/max selection bounds and one callback containing all selected values.

## Best Possible Solution

Add an additive portable batch-selection contract to MessagePresentation/message tool, export and document it through the plugin SDK, and implement consistent Slack, Discord, Telegram, and text-fallback rendering plus callback normalization.
