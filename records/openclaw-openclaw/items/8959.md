## Summary

Keep open. Current main still exposes buttons and single-select through the shared message presentation/message tool contract, while the adjacent Telegram, Slack, Discord, wizard, and plugin-hook capabilities do not provide the requested portable multi-select API with min/max values and one selected-values callback across channels.

## What This Changes

Request a cross-platform multi-select capability in the message tool so agents can collect several visual selections in one user interaction and one callback.

## Best Possible Solution

Add an additive portable batch-selection contract in message presentation/message tool, export and document it through the SDK, and implement channel render/callback handling plus text fallback.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence current-main gap because the public message-tool/presentation contract still has only single-select semantics.

## Solution Assessment

No current close path is sufficient. The maintainable direction is a shared additive API plus channel implementations, not treating the merged plugin hook or channel-specific callbacks as the requested cross-platform message-tool feature.
