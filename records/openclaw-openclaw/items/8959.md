## Summary

Keep open. Current main and the latest release still have only a portable single-select message presentation contract; adjacent poll, wizard, Discord, Slack, and Telegram primitives do not provide the requested cross-channel multi-select message-tool callback surface.

## What This Changes

Request to add a cross-platform multi-select component to the message tool so users can select several options visually and submit them in one interaction.

## Best Possible Solution

Add an additive portable batch-selection contract to message presentation/message-tool, export and document it through the plugin SDK, then implement consistent Slack, Discord, Telegram, and fallback callback behavior after API review.

Not applicable: this is a feature request, not a bug report. Source and latest-release inspection give high confidence that the requested portable multi-select contract is absent because the shared message presentation surface supports buttons and single select only.

## Solution Assessment
