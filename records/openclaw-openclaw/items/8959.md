## Summary

Keep open; current main still lacks the requested portable message-tool multi-select contract. The existing shared presentation/message-tool surface remains single-select plus buttons, while the related dialog PR is closed unmerged and the later plugin hook only provides interception infrastructure, not a cross-channel batch-selection UI contract.

## What This Changes

Add a cross-platform message-tool multi-select capability so users can visually choose multiple options and submit one structured callback.

## Best Possible Solution

Define an additive shared batch-selection contract in the message presentation/message-tool boundary, export and document it through the plugin SDK, and implement compatible render plus callback behavior for the supported channels and text fallback.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence gap check because current main still models the shared message presentation surface as single-select plus buttons.

## Solution Assessment

Unclear until maintainers choose the public API and callback semantics. The maintainable direction is contract-first shared presentation work, or an explicit plugin-first design with any missing SDK seams, rather than a channel-only native component patch.
