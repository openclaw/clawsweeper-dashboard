## Summary

Keep open. This is a recent feature PR, current main does not already provide the new prompt mode or skill tools, and source review still finds a prompt/tool mismatch plus missing public docs before merge.

## What This Changes

The branch adds global `skills.promptMode`, compact progressive skill prompt formatting, `skill_view` and `skill_search` tools, schema updates, tests, and telemetry.

## Best Possible Solution

Align progressive prompt text with the actual tool surface on every runtime path, document `skills.promptMode`, and require a real agent-turn proof before maintainer rollout approval.

## Reproduction Assessment

Yes for the review findings: source inspection shows `skills.promptMode` changes the shared skills snapshot while CLI and command prompt paths can consume that prompt without registering `skill_view` or `skill_search`. I did not run an end-to-end model turn because this review is read-only.

## Solution Assessment

No, this is not merge-ready. The direction is plausible, but the implementation needs a runtime-capability guard or full snapshot plumbing for all prompt consumers, plus public docs and real behavior proof.
