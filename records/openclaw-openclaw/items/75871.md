## Summary

Keep open: current main does not include this opt-in progressive skills mode, and the PR is not merge-ready because the new public config lacks docs/changelog coverage and existing session snapshots can keep the old prompt shape after `skills.promptMode` changes.

## What This Changes

The PR adds a global `skills.promptMode` config, compact skill prompt formatting, optional `skill_view`/`skill_search` tools, schema updates, telemetry, and focused tests.

## Best Possible Solution

Add release/docs coverage, make skill snapshot reuse account for prompt disclosure mode, then let maintainers decide whether the scoped global pilot should land.

## Reproduction Assessment

Yes for the cache bug by source inspection: create or reuse a session with a legacy `skillsSnapshot`, change config to `skills.promptMode: "search"`, and the PR head reuses the old prompt while registering skill tools from the new config. For the docs/changelog blocker, the reproduction is the current `rg` search returning no public docs or changelog entries.

## Solution Assessment

Mostly yes, but not merge-ready. The implementation belongs in skills prompt assembly and OpenClaw tool construction, but it needs snapshot invalidation plus docs/changelog before the approach is maintainable.
