## Summary

Keep open: the branch is useful but not merge-ready because it can advertise progressive skill tools on paths that do not register them, and the new public config lacks docs and changelog coverage.

## What This Changes

The PR adds a global `skills.promptMode` config with legacy/compact/view/search modes, compact skill prompt formatting, optional `skill_view` and `skill_search` tools, schema updates, tests, and prompt/tool telemetry.

## Best Possible Solution

Keep the PR open, align progressive prompt text with the actual tool surface on every runtime path, add public docs and changelog coverage, then let maintainers decide whether the global pilot scope is acceptable.

## Reproduction Assessment

Yes for the review findings: source inspection shows `skills.promptMode` changes the shared skills snapshot while native CLI prompt preparation consumes that snapshot without registering `skill_view` or `skill_search`. I did not run an end-to-end model turn.

## Solution Assessment

No, not merge-ready. The implementation belongs near skills prompt assembly and OpenClaw tool construction, but it needs a runtime-capability guard or downgrade for paths without the new tools, plus docs and changelog updates.
