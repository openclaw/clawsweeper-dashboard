## Summary

Keep open. Current main has meaningful pending/suspended delivery state and partial recovery, but it still has retry-limit, expiry, discard, and orphan-prune paths that can end durable completion delivery before the survivable spool/orphan-retention design requested here is complete.

## What This Changes

The issue requests durable subagent completion delivery so successful child work is not lost during direct-announce failures, drain/restart gaps, or orphan cleanup.

## Best Possible Solution

Keep this as the durable-delivery planning thread until maintainers choose and land one proof-positive completion spool or queue design with restart/orphan preservation, duplicate-delivery protection, task-ledger alignment, and updated docs.

## Solution Assessment
