## Summary

Keep open. Current main still has partial subagent announce hardening rather than the requested durable completion spool: the completion path is direct-first with conditional queue fallback, failed completion cleanup can still become terminal, and restore/resume orphan reconciliation still prunes missing child-session records before a durable completion inbox can replay them.

## What This Changes

The issue requests a durable subagent completion inbox/spool so completed child work cannot be lost across direct announce timeout, drain/restart, or orphan pruning.

## Best Possible Solution

Keep this as the planning thread for a core subagent lifecycle change: persist a self-contained pending completion record before cleanup can give up, have direct and queued delivery consume/update that state, keep task delivery pending while the spool is pending, and make resume/orphan reconciliation retry or deliberately expire pending completions instead of pruning them silently. The eventual user-facing reliability fix should include focused registry/dispatch tests, docs, and a changelog entry.

## Reproduction Assessment

Yes. Current source and tests provide a high-confidence reproduction path: make completion-mode direct delivery fail while queue fallback returns none, or remove the child session before steer-restart cleanup; current main records failure/prunes instead of preserving a pending completion spool.

## Solution Assessment

Not applicable as a patch review because this issue has no PR attached. The requested durable spool direction is the right target, but the exact persisted data model and orphan-reconciliation behavior should be confirmed by maintainers before implementation.
