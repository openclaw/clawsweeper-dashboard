## Summary

Keep open. Current main still has direct-first subagent completion delivery with conditional, process-local fallback and terminal cleanup/orphan-prune paths; the issue discussion also explicitly says the April partial fixes did not implement the durable registry-level completion spool requested here.

## What This Changes

The issue requests a durable subagent completion inbox/spool so completion delivery survives direct announce failures, drain/restart gaps, and orphan pruning.

## Best Possible Solution

Keep this as the planning thread for a core subagent lifecycle change: persist a self-contained pending completion record before cleanup can give up, have direct and queued delivery consume/update that state, keep task delivery pending while the spool is pending, and make resume/orphan reconciliation retry or deliberately expire pending completions instead of pruning them silently.

## Reproduction Assessment

Yes. Current source and tests provide a high-confidence reproduction path: make completion-mode direct delivery fail while queue fallback returns none, or remove the child session before steer-restart cleanup; current main records failure or prunes instead of preserving a pending completion spool.

## Solution Assessment

Not applicable as a patch review because this issue has no PR attached. The requested durable spool direction appears to be the right target, but the persisted data model, retry expiry, and orphan-reconciliation semantics should be approved by maintainers before implementation.
