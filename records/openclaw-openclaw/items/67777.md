## Summary

Keep open. Current main has partial pending final-delivery state, but the reported durable completion-spool gap remains source-proven: completion-mode dispatch can still fail direct delivery plus fallback steering, cleanup can terminally clear pending delivery on give-up, and orphan reconciliation still removes the run record instead of preserving pending completion state.

## What This Changes

The issue asks for durable subagent completion delivery across direct-announce timeout, drain/restart, and orphan-prune loss windows.

## Best Possible Solution

Keep this as the focused core planning thread for a persisted subagent completion spool: store a self-contained pending completion before cleanup can give up, replay it on resume/restart, and make orphan pruning preserve or explicitly expire that pending delivery.

## Reproduction Assessment

Yes, at source level. Current main still exposes the direct-failure plus fallback-none path, terminal give-up cleanup, and orphan-prune behavior that match the reported loss windows.

## Solution Assessment

Not applicable as a patch review because this is an issue. The durable spool direction is the right target, but the persisted status contract and overlap with open completion-delivery PRs need maintainer settlement before implementation.
