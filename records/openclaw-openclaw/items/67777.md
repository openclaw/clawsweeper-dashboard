## Summary

Keep open. Current main has partial durable pending-final-delivery state, but the issue’s remaining restart/orphan-prune durability gap is still present: orphan reconciliation can remove ended runs before a pending completion is replayed, and give-up still clears pending state and marks delivery failed.

## What This Changes

The issue requests a durable subagent completion inbox/spool so child completion delivery survives direct announce failure, drain/restart, and orphan pruning.

## Best Possible Solution

Keep this as the focused core planning thread for a persisted subagent completion spool: store a self-contained pending completion before cleanup can give up, replay it on resume/restart, and preserve or intentionally expire it during orphan reconciliation.

## Reproduction Assessment

Yes, at source level rather than by a live run. Current main still has the direct-failure plus queue-none path, terminal give-up, and orphan-prune behavior that match the reported loss windows.

## Solution Assessment

Not applicable as a patch review because this is an issue. The durable spool direction is still the right target, but current main only partially implements pending final delivery and does not settle the orphan-preservation or spool status contract.
