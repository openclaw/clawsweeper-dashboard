## Summary

Keep open. Current main and the latest release still reject non-off thinking for the reported Bailian/Qwen Coding Plan models because the bundled catalog marks `qwen3.5-plus`, `glm-5`, and `kimi-k2.5` as non-reasoning, while the provider already has the request adapter needed once the model capability metadata is corrected.

## What This Changes

The issue requests official Ali Bailian/Qwen Coding Plan reasoning support for subscription models such as `qwen3.5-plus`, `glm-5`, and `kimi-k2.5`.

## Best Possible Solution

Update the Qwen provider catalog and static metadata for only Alibaba-confirmed Coding Plan reasoning models, reuse the existing `enable_thinking` adapter, and add focused catalog, stream, docs, and live Coding Plan proof for on/off thinking.

## Reproduction Assessment

Yes, at source level with high confidence: choose a bundled Coding Plan model such as `qwen3.5-plus`, request `/thinking low`, and the catalog-driven thinking profile rejects it because the model row says `reasoning: false`. The same failure was reported by a user on v2026.6.5, but no live authenticated Alibaba request was run in this review.

## Solution Assessment

Yes in architecture, but not as a broad toggle. The maintainable fix is provider-owned model capability metadata feeding the existing Qwen request adapter, with maintainer approval of the exact model list and live endpoint proof.
