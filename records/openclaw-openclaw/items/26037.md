## Summary

Keep open: current main now supports Qwen-style thinking for explicitly configured OpenAI-compatible models, but the bundled Qwen Coding Plan catalog still marks the reported models as non-reasoning, so the official bundled path will not send `enable_thinking` for them.

## What This Changes

Support Ali Bailian/Qwen Coding Plan models with OpenClaw thinking/reasoning enabled, or document an official way to turn it on.

## Best Possible Solution

Verify the current Alibaba Coding Plan contract, enable only confirmed bundled Qwen/Bailian catalog entries for reasoning, keep payload mapping provider-owned, and update Qwen docs plus focused regression tests.

## Reproduction Assessment

Yes for the support gap by source inspection: bundled Coding Plan models named in the report are still cataloged as non-reasoning, and the Qwen wrapper patches only reasoning models. No for the original gateway restart failure, because the report lacks a current restart log and credentialed Coding Plan repro.

## Solution Assessment

Yes, the focused provider-owned direction remains the narrowest maintainable solution. The generic custom-provider path is already on main, so the remaining decision is which bundled Qwen/Bailian Coding Plan models should be marked reasoning-capable and documented.
