## Summary

Current main and v2026.6.5 still only partially satisfy the request: OpenClaw can emit Qwen thinking payloads and supports Qwen thinking-format compatibility, but the bundled Coding Plan entries named in the report remain non-reasoning, so they bypass that runtime path. The issue remains necessary, with the final catalog/default decision gated on verified Alibaba model and endpoint support.

## What This Changes

Add official reasoning support for Ali Bailian Coding Plan models that support the vendor’s thinking API.

## Best Possible Solution

Retain the existing Qwen stream wrapper, verify the current Alibaba Coding Plan support matrix with real API calls, mark only supported bundled models reasoning-capable, and document and test both supported and intentionally unsupported entries.

## Reproduction Assessment

Yes at source level: select any reported bundled model with `reasoning: false` and the Qwen wrapper’s `shouldPatch` gate skips `enable_thinking`; a live Alibaba subscriber-account reproduction was not performed.

## Solution Assessment

Yes for the provider-owned seam, but a blanket reasoning enablement is not the best fix; the narrow maintainable solution is to reuse the existing wrapper and gate catalog capabilities from a verified vendor support matrix.
