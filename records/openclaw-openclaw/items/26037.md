## Summary

Keep this issue open. Current main still only has partial Qwen/Bailian Coding Plan thinking support: the Qwen stream wrapper can emit top-level `enable_thinking`, but several named Coding Plan models remain cataloged as non-reasoning and the public custom-provider compat schema still has no accepted Qwen thinking-format path.

## What This Changes

Add official OpenClaw support or a documented configuration path for Alibaba Bailian/Qwen Cloud Coding Plan models with thinking/reasoning enabled.

## Best Possible Solution

Align the bundled Qwen provider and the documented custom-provider path with the current Coding Plan contract: only documented thinking-capable models should advertise reasoning and send top-level `enable_thinking`, custom configs should validate through a supported Qwen thinking surface, and vLLM’s provider-owned Qwen thinking behavior should remain unchanged.
