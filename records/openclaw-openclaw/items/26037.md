## Summary

Keep open: current main has generic Qwen custom-provider thinkingFormat support, but official bundled Ali Bailian/Coding Plan reasoning support is still incomplete and current Alibaba guidance is model-specific and not fully consistent with OpenClaw's bundled catalog.

## What This Changes

Request official OpenClaw support for Ali Bailian/Qwen Coding Plan models with thinking/reasoning enabled.

## Best Possible Solution

Verify the current Alibaba/DashScope Coding Plan contract per model and endpoint, then update the Qwen provider catalog, payload mapping, docs, and regression tests only for confirmed reasoning-capable models.

## Reproduction Assessment

Yes for the official-support gap by source inspection: the bundled Coding Plan models named in the report are still non-reasoning, and the Qwen wrapper only patches reasoning models. No for the original gateway restart failure, because no current restart log or credentialed reproduction was provided.

## Solution Assessment

Yes, the provider-owned and vendor-verified direction is the right shape. The generic custom-provider support from https://github.com/openclaw/openclaw/pull/79777 is useful but does not decide which bundled Qwen Coding Plan models should officially advertise reasoning.
