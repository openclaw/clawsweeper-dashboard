Keep open. Current main and the latest stable release still mark the reported Qwen/Bailian Coding Plan models as non-reasoning, so non-off `/thinking` remains rejected; the remaining work is model-specific provider/catalog approval plus live Alibaba Coding Plan proof, not cleanup closure.

Request official Ali Bailian/Qwen Coding Plan support for enabling thinking/reasoning on subscription models such as qwen3.5-plus, GLM-5, and Kimi K2.5.

## Best Possible Solution

Update the bundled Qwen provider metadata only for Alibaba-confirmed Coding Plan reasoning models, reuse the existing `enable_thinking`/Qwen thinkingFormat paths, and land docs plus focused regression and live endpoint proof.
