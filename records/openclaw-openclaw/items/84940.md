## Summary

No blocking findings. Current main still raw-concatenates normalized provider/model pairs in the two allowlist gates, while the PR switches those sites to the existing canonical helper that already dedupes provider-prefixed model ids and adds focused regression coverage.

## What This Changes

The PR replaces five ad-hoc provider/model key concatenations in plugin runtime LLM and gateway fallback subagent allowlist policy paths with `modelKey()` and adds one regression test per path.

## Best Possible Solution

Land the focused helper substitution with its regression tests after normal CI, keeping broader deduplication of the duplicated `normalizeAllowedModelRef` helpers as a separate refactor if maintainers want it later.

## Reproduction Assessment

Yes, by source inspection: current main normalizes both sides and then raw-concatenates provider/model strings, while OpenRouter can produce a resolved model id that already begins with `openrouter/`. I did not run a local repro because this review is read-only.

## Solution Assessment

Yes. Reusing the existing canonical `modelKey()` helper is the narrow maintainable fix, and it avoids inventing a second runtime-only dedupe rule or changing legacy lookup behavior.
