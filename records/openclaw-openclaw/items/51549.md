## Summary

Keep open: current main and the latest shipped tag restore WebChat transcript history from the gateway, but queued messages and draft text are still browser-memory state and the open closing pull request has not merged.

## What This Changes

The issue asks WebChat to preserve pending queued messages, restored conversation history, and unsent draft text across browser refresh or tab reopen.

## Best Possible Solution

Keep the gateway-backed history reload, then review and rework https://github.com/openclaw/openclaw/pull/73154 or a focused replacement to define durable queue persistence, safe active-run replay, attachment behavior, and draft retention before closing this issue.

## Reproduction Assessment

Yes, at source level. Current main restores visible history from chat.history, but chatMessage and chatQueue are initialized and mutated in component memory with no current browser-storage or gateway restore path for a full page refresh.

## Solution Assessment

Unclear. Gateway-backed history reload is the right contract, but queue and draft persistence still need reviewed retention, attachment, and active-run replay semantics before the linked pull request or a replacement is the best fix.
