## Summary

Keep open. Current main restores conversation history from the gateway, but queued messages and draft text are still rebuilt from in-memory UI state on refresh, and the issue is paired with an open unmerged PR that uses closing syntax.

## What This Changes

The issue asks WebChat to restore queued messages, conversation history, and draft input after a browser refresh.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/73154 or a replacement lands for queue persistence, while either implementing draft persistence with reviewed retention semantics or splitting that UX gap into a narrower follow-up.

## Reproduction Assessment

Yes, source-level. Refresh rebuilds `chatMessage` and `chatQueue` from empty component state, while targeted searches found no queue or draft storage restoration path on current main.

## Solution Assessment

Unclear. Gateway-backed history reload is already the right contract, but queue persistence must coordinate with active-run/session lifecycle and draft retention needs a maintainer scope decision.
