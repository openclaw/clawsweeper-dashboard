## Summary

Keep open. Current main restores WebChat history from the gateway, but queued messages and draft text still rebuild from in-memory UI state on refresh, and https://github.com/openclaw/openclaw/pull/73154 is still an open unmerged PR that references this issue with closing syntax.

## What This Changes

The issue requests preserving WebChat queued messages, restored conversation history, and unsent draft text across browser refresh or reconnect.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/73154 or a replacement lands queue persistence through the shared chat/session lifecycle, while deciding whether draft persistence belongs in the same fix or a narrower follow-up.

## Reproduction Assessment

Yes, at source level. Current main reloads history from `chat.history`, but `chatMessage` and `chatQueue` are initialized and mutated in component memory with no browser-storage restore path for a full page refresh.

## Solution Assessment

Unclear. Gateway-backed history reload is the right contract, but the remaining queue and draft work needs reviewed lifecycle, attachment, and retention semantics before the linked PR or a replacement can be considered the best fix.
