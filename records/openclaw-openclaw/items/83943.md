## What This Changes

The issue reports that embedded-runner `session-resource-loader` latency grows across warm turns on Windows with Feishu/Lark and MiniMax OAuth, causing event-loop stalls and timeouts.

## Best Possible Solution

Add finer substage timing or a CPU profile for one slow warm turn, then remove the confirmed per-turn whole-state work while preserving session correctness, compaction guards, and the Pi resource-loader contract.

## Reproduction Assessment

No high-confidence live reproduction was run. The uploaded logs plus current source give a medium-confidence source-level reproduction shape: each warm turn still reopens the session JSONL and reloads Pi resources before the `session-resource-loader` mark.

## Solution Assessment
