## Summary

Keep open. Current main still has the main-session yield persistence gap this PR targets, but the proposed patch is not merge-ready because it derives `pauseReason: "sessions_yield"` from the raw `yielded` flag even when the lifecycle snapshot status is `failed`, and current lifecycle emission can carry `yielded: true` on error events.

## What This Changes

The PR adds a persisted `paused`/`sessions_yield` main-session lifecycle state, propagates it through gateway session surfaces and sessions tooling, skips restart recovery for paused entries, and adds focused tests plus a changelog entry.

## Best Possible Solution

Keep the narrow gateway persistence approach, but gate `pauseReason` on the derived paused status or end-phase yield path, add an error-plus-yield regression test, then land only after CI and the manual main-session yield/restart smoke checks pass.

## Reproduction Assessment

Yes. For the original bug, a lifecycle `end` event with `yielded: true` and `stopReason: "end_turn"` maps to `done` on current main; for the PR regression, a lifecycle `error` event with `yielded: true` would map to `failed` while still persisting `pauseReason: "sessions_yield"`.

## Solution Assessment

No, not as proposed. The overall seam is the right narrow place to fix main-session yield persistence, but `pauseReason` should be derived from `terminalStatus === "paused"` or `phase === "end" && yielded` so failed runs do not look paused.
