## Summary

Keep open: the source-level bug is real and the PR targets the right gateway lifecycle seam, but the latest head still exposes a new `paused` session status without Control UI/shared consumer handling and still lacks required after-fix real behavior proof.

## What This Changes

The PR persists main-session `sessions_yield` lifecycle endings as paused, propagates `pauseReason`, skips restart recovery for paused-yield entries, and adds focused tests plus changelog text.

## Best Possible Solution

Land a complete fix that treats `paused` as a non-terminal sessions-yield state across gateway persistence, session tools, shared row contracts, and Control UI rendering/reconciliation, then merge after redacted real runtime proof.

## Reproduction Assessment

Yes, at source level: current main emits `yielded: true` from the runner while gateway lifecycle persistence ignores that flag and maps a clean terminal event to `done`. I did not run a live smoke in this read-only review.

## Solution Assessment

No, not yet. The gateway lifecycle seam is the right place to classify yielded main-session runs, but the new `paused` status must be wired through Control UI/shared session handling before the fix safely covers dashboards and follow-up turns.
