## Summary

Keep open: current main still drops yielded main-session lifecycle events into a terminal done state, and this PR is a focused fix with no blocking review findings found.

## What This Changes

Adds persisted paused/sessions_yield state for main-session lifecycle rows, propagates it through gateway/session-list surfaces, skips restart recovery for yielded entries, and adds tests plus a changelog entry.

## Best Possible Solution

Land this PR after maintainer approval so main sessions use the same persisted paused-yield state model already used for yielded subagent runs.

## Reproduction Assessment

Yes. Current main can be reproduced at the lifecycle helper level with a phase end event carrying yielded: true; the helper ignores yielded and persists done while the runner already emits that metadata.

## Solution Assessment

Yes. Persisting paused/sessions_yield at the gateway lifecycle seam is the narrow maintainable fix because runner, agent.wait, and job snapshot propagation already carry the yielded signal.
