## Summary

This PR fixes a real source-reproducible main-session lifecycle gap, but it should stay open because the new persisted `paused` status is not yet handled by the Control UI session types/formatting and the supplied proof is diff output rather than a real gateway run on the PR head.

## What This Changes

The PR persists `sessions_yield` main-session lifecycle ends as `status: "paused"` with `pauseReason: "sessions_yield"`, propagates that metadata through gateway session rows/broadcasts, skips restart recovery for yielded entries, and adds focused tests plus a changelog entry.

## Best Possible Solution

Keep the narrow lifecycle fix, but update all public session consumers for `paused` and require redacted real runtime proof of yield persistence, follow-up reactivation, and restart-recovery skip before merge.

## Reproduction Assessment

Yes, source-reproducible: current main propagates `yielded` through runner/wait metadata but `deriveGatewaySessionLifecycleSnapshot` ignores it and maps clean yielded lifecycle ends to `done`. I did not run a live gateway smoke in this read-only review.

## Solution Assessment

No, not yet: the lifecycle/restart-recovery fix is the right narrow shape, but adding a new session status without updating Control UI consumers leaves the stated dashboard behavior incomplete and proof is not real runtime proof.
