## Summary

Keep open. Current main still has the yielded main-session persistence gap, and this PR appears to address it with a narrow lifecycle/recovery fix, but the external PR still needs after-fix real behavior proof before merge.

## What This Changes

The branch persists main-session `sessions_yield` terminal lifecycle events as `paused`, propagates the pause marker to session surfaces, skips restart recovery for paused entries, and adds focused tests plus changelog text.

## Best Possible Solution

Land this PR after contributor-supplied real behavior proof and maintainer approval, while keeping the broader UI progress/streaming work in the related open issues separate.

## Reproduction Assessment

Yes, at source level. Current main emits `yielded: true` from the runner but the gateway lifecycle helper ignores it and persists a clean yielded end as `done`; I did not run the live smoke in this read-only review.

## Solution Assessment

Yes. Mapping yielded terminal lifecycle events to a paused persisted state at the gateway lifecycle seam, clearing it on the next start, and skipping restart recovery for that marker is the narrow maintainable fix.
