## Summary

Keep open: this is a real session-state fix with useful live proof, but the current patch is not merge-ready because the new generic persisted `paused` status can still be interpreted as successful subagent completion by an existing reconciliation path, and the new stored/Gateway/UI status value needs maintainer compatibility acceptance.

## What This Changes

The branch persists yielded main-session lifecycle ends as `paused` with `pauseReason`, propagates that status through Gateway/session tools/Control UI, skips restart recovery, and forwards native/Codex yield terminal metadata earlier.

## Best Possible Solution

Keep the paused-session model only after `paused`/`sessions_yield` is treated as nonterminal across subagent reconciliation and maintainers explicitly accept the new stored/Gateway/UI status contract.

## Reproduction Assessment

Yes. The original race has contributor live proof, and the blocking review finding is source-reproducible: a persisted row with `status: "paused"` plus `endedAt` reaches `resolveCompletionFromSessionEntry` and is returned as an `ok` completion today.

## Solution Assessment

No, not yet. Persisting a paused main-session state is a plausible owner-boundary fix, but the generic persisted status must be reconciled with sibling subagent completion logic and accepted as a compatibility-sensitive status contract before merge.
