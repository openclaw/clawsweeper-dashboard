## Summary

Keep this PR open: the backend direction is plausible and the root cause is source-reproducible, but the branch still leaves Control UI consumers unable to model the new paused row state and it still lacks after-fix real behavior proof.

## What This Changes

The PR persists main-session `sessions_yield` lifecycle endings as paused, propagates `pauseReason`, skips restart recovery for paused-yield entries, and adds focused tests plus changelog text.

## Best Possible Solution

Land a complete fix that updates gateway persistence and every shared/UI session consumer to treat `paused` as a non-terminal sessions-yield state, then merge only after redacted real runtime proof.

## Reproduction Assessment

Yes, at source level: current main emits `yielded: true` from the runner while gateway lifecycle persistence ignores it and maps a clean terminal event to `done`. I did not run a live smoke in this read-only review.

## Solution Assessment

No, not yet. The gateway lifecycle seam is the right place to classify yielded main-session runs, but the proposed status must be wired through Control UI/shared session handling before it solves the dashboard and follow-up-turn behavior safely.

## Review Findings
