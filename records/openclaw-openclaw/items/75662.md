## Summary

Keep open. Current main still persists yielded top-level session lifecycle ends as completed, so the PR addresses a real gap, but the proposed patch should not persist `pauseReason: "sessions_yield"` on lifecycle error events and the exact PR head currently has failing CI checks.

## What This Changes

The PR adds a persisted `paused`/`sessions_yield` state for top-level session lifecycle events, propagates the new status through gateway/session list surfaces, skips restart recovery for yielded sessions, and adds focused tests plus a changelog entry.

## Best Possible Solution

Keep the PR open, gate `pauseReason` so only genuinely paused clean-yield terminal snapshots carry it, add the error-plus-yielded regression test, then land after CI and the manual yield/restart smokes pass.

## Reproduction Assessment

Yes. A focused current-main reproduction is a lifecycle `end` event with `yielded: true` and `stopReason: "end_turn"`, which current main maps to `done`; the PR adds unit coverage for that path. The review finding reproduces with `phase: "error"` plus `yielded: true`, which the lifecycle emitter can produce because it spreads terminal yielded metadata into error events.

## Solution Assessment

No, not as currently proposed. The persistence seam is the right place to fix the main-session yielded state, but `pauseReason` should be derived from `terminalStatus === "paused"` or a clean yielded end, not from the raw yielded flag for every terminal phase.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate pauseReason on the paused terminal status:** `src/gateway/session-lifecycle-state.ts:148`
  - body: The lifecycle handler carries `yielded` in terminal metadata for both `end` and `error` events. Since `resolveTerminalStatus` returns `failed` for `phase: "error"`, this line can persist and broadcast a failed run with `pauseReason: "sessions_yield"`, which contradicts the new marker's meaning that the session is waiting on a queued continuation. Gate `pauseReason` on `terminalStatus === "paused"` and add the error-plus-yielded regression case.
  - confidence: 0.89
