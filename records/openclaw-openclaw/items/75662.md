## Summary

Keep open. Current main still maps yielded main-session lifecycle ends to `done`; the PR head addresses the earlier `pauseReason` review finding, but exact-head CI/security checks are failing and the manual yield/restart smokes remain unverified.

## What This Changes

The PR persists clean `sessions_yield` main-session ends as `paused` with `pauseReason`, propagates the state through gateway/session-list surfaces, skips restart recovery for yielded sessions, and adds tests plus a changelog entry.

## Best Possible Solution

Land this PR after the exact-head CI/security failures are triaged or rerun green and the planned main-session yield/restart smokes pass.

Yes. A focused current-main reproduction is a lifecycle `phase: "end"` event with `yielded: true` and `stopReason: "end_turn"`; current main ignores `yielded` and persists `done`, while the PR adds direct unit coverage for that path.

## Solution Assessment
