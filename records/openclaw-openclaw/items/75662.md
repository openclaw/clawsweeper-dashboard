## Summary

Keep open: the gateway-side fix is directionally right and source-reproducible, and the Copilot error/yield concern was addressed by the follow-up commit, but the latest PR still introduces a `paused` session status without Control UI handling and has no after-fix real behavior proof.

## What This Changes

The PR persists yielded main-session lifecycle events as `paused`, carries `pauseReason`, skips restart recovery for those entries, and adds focused tests plus changelog text.

## Best Possible Solution

Land a complete fix that treats `paused` as a non-terminal sessions-yield state across gateway persistence, session tools, session broadcasts, and Control UI rendering/reconciliation, then merge after redacted real runtime proof.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current main emits `yielded: true` from the runner while gateway lifecycle persistence ignores that flag and maps a clean terminal event to `done`. I did not run a live smoke in this read-only review.

## Solution Assessment

No, not yet. The gateway lifecycle seam is the right place to classify yielded main-session runs, but the new `paused` status needs Control UI/shared row handling before dashboards and follow-up-turn reconciliation are safe.

## Review Findings
