## Summary

Keep open: current main still persists clean yielded main-session lifecycle endings as completed, and this PR is a focused fix with the earlier review concern addressed.

## What This Changes

The PR persists yielded main-session lifecycle endings as paused with `pauseReason`, propagates that state to session surfaces, skips restart recovery for paused entries, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land this PR after maintainer approval and final exact-head validation so main sessions share the paused-yield persistence model already used for yielded subagent runs.

## Reproduction Assessment

Yes, from source inspection. Current main’s lifecycle resolver ignores `yielded` and resolves clean yielded ends as `done`, while the runner already emits `yielded: true` for `sessions_yield`.

## Solution Assessment

Yes. Classifying yielded terminal events at the gateway lifecycle seam and skipping restart recovery by the specific pause reason is the narrow maintainable fix.
