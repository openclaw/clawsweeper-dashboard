## Summary

Keep open. Current main still has the reported lifecycle gap for yielded top-level sessions, but this PR is not merge-ready because the proposed pause marker can also be attached to failed lifecycle snapshots when an error event carries `yielded: true`.

## What This Changes

The branch adds a persisted `paused`/`sessions_yield` state for top-level session lifecycle events, propagates `pauseReason` through session rows and broadcasts, skips restart recovery for yielded sessions, and adds focused tests plus a changelog entry.

## Best Possible Solution

Keep the persistence seam fix, but make `pauseReason` represent only genuinely paused yielded continuations and preserve failed lifecycle events as failed without a paused marker. After that, land with the focused lifecycle/recovery tests and the normal changed gate.

## Reproduction Assessment

Yes. On current main, a lifecycle `end` event with `stopReason: "end_turn"` and `yielded: true` reproduces the reported `done` mapping; on the PR branch, a lifecycle `error` event with `yielded: true` reproduces the failed-plus-`sessions_yield` inconsistency.

## Solution Assessment

No, not as currently proposed. The session lifecycle persistence layer is the right place to fix the top-level yielded state, but `pauseReason` should be derived from `terminalStatus === "paused"` or a clean yielded `end` event, not from raw `yielded` metadata on every terminal phase.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate pauseReason on the paused terminal status:** `src/gateway/session-lifecycle-state.ts:148`
  - body: The lifecycle handler can carry `yielded: true` on `phase: "error"` events. Since `resolveTerminalStatus` still returns `failed` for that phase, deriving `pauseReason` from raw `yielded` persists a failed run as if it were waiting on a queued continuation, which can make recovery and consumers skip a real failure. Gate this marker on `terminalStatus === "paused"` and add the error-plus-yielded regression case.
