## Summary

Keep open: the central main-session `sessions_yield` fix is still needed, but the latest head is incomplete because `pauseReason` is only propagated on the lifecycle broadcast path, the paused transcript-rotation guard lacks a regression test in the public diff, and the new persisted/Gateway/UI-visible status contract still needs maintainer acceptance.

## What This Changes

The PR treats top-level `sessions_yield` lifecycle endings as a persisted nonterminal `paused` session state with `pauseReason`, propagates that status to session tools and Control UI, and skips restart recovery/transcript terminal handling for yielded sessions.

## Best Possible Solution

Land a complete owner-boundary fix where yielded main sessions are nonterminal across every session row, broadcast, recovery, transcript, and UI path, with focused regression tests and explicit maintainer acceptance of the new status contract.
