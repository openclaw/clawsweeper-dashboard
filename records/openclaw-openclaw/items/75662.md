## What This Changes

The PR persists yielded main-session runs as a nonterminal `paused` session state with `pauseReason: "sessions_yield"`, skips restart recovery for that state, and teaches session/UI consumers about it.

## Best Possible Solution

Land the gateway/session-state fix after syncing the Control UI generated locale artifacts and confirming the targeted lifecycle, restart-recovery, UI, and i18n checks on the refreshed branch.

## Reproduction Assessment

Yes. Current main source shows the runner emits `yielded: true`/`livenessState: "paused"`, while current gateway lifecycle derivation ignores `yielded` and maps non-error end events through the terminal outcome path as `done`.

## Solution Assessment

Yes, aside from the i18n artifact drift. Persisting the paused state at `deriveGatewaySessionLifecycleSnapshot` is the cleanest owner boundary because it uses the existing runner signal before restart recovery, session rows, broadcasts, and UI consumers observe the row.
