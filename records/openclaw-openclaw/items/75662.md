## Summary

Keep open. Current main still has the yielded main-session persistence gap, and this PR targets the right lifecycle/recovery seam, but the patch is not merge-ready because it can persist an inconsistent failed session with `pauseReason: "sessions_yield"` when a lifecycle error event also carries the yielded flag.

## What This Changes

The branch adds a persisted `paused`/`sessions_yield` state for top-level sessions, propagates it through gateway session rows and broadcasts, skips restart recovery for yielded entries, and adds focused lifecycle/recovery tests plus a changelog entry.

## Best Possible Solution

Keep the narrow lifecycle-persistence approach, but gate `pauseReason` on the derived terminal status being `paused`, add an error-plus-yielded regression test, and update the Control UI row contract if `pauseReason` is meant to be observable there before landing with CI and the manual smokes.

## Reproduction Assessment

Yes. A focused reproduction is a lifecycle `end` event with `yielded: true`, which current main maps to `done`; an additional PR-specific reproduction is a lifecycle `error` event with `yielded: true`, which the branch would persist as `failed` plus `pauseReason: "sessions_yield"`.

## Solution Assessment

Mostly yes, but not as written. The gateway persistence/recovery seam is the narrow maintainable fix, while the safer implementation derives `pauseReason` only when the resolved terminal status is `paused` and keeps failed lifecycle events unpaused.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Gate pauseReason on the paused terminal status:** `src/gateway/session-lifecycle-state.ts:148`
  - body: `handleAgentEnd` includes the shared terminal metadata on both `end` and `error` lifecycle events, so an error event can still arrive with `yielded: true`. This line then persists `status: "failed"` together with `pauseReason: "sessions_yield"`, leaving consumers with a failed session that also looks intentionally paused. Set `pauseReason` only when `terminalStatus === "paused"` and add an error/yielded test.
  - confidence: 0.89
- **[P3] Propagate pauseReason through the UI row contract:** `src/gateway/server-chat.ts:281`
  - body: The server now sends `pauseReason` in `sessions.changed`, but the Control UI normalizes those events through a fixed row-field allowlist and its mirrored `GatewaySessionRow` type does not include the new field. As a result, UI session state drops this field on broadcasts; add it to the UI type, field allowlist, and controller coverage if it is part of the new row contract.
  - confidence: 0.78
