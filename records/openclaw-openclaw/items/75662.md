## Best Possible Solution

Land a version that either keeps the public session status contract backward-compatible or updates every shipped `SessionRunStatus` consumer to treat `paused` as nonterminal, with focused UI/tool/SDK coverage and redacted runtime proof of the main-session yield and restart path.

No: the runtime direction is plausible, but the PR is not the best mergeable shape until all session-row consumers treat `paused` as nonterminal or maintainers choose a backward-compatible representation.
