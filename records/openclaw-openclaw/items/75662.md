## Summary

Keep open: current main still has the yielded-main-session persistence gap, and this PR is a focused implementation with no discrete defect found in the reviewed diff.

## What This Changes

The PR persists main-session `sessions_yield` lifecycle ends as `paused` with `pauseReason: "sessions_yield"`, propagates that state to session surfaces, skips restart recovery for paused entries, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land this PR after maintainer approval and final exact-head CI so main sessions share the paused-yield persistence model already used for yielded subagent runs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main emits `yielded: true` for `sessions_yield`, while the gateway lifecycle helper ignores it and resolves the terminal session status as `done`.

## Solution Assessment

Yes. Handling the yielded marker at the gateway lifecycle persistence seam and teaching restart recovery to skip that explicit pause reason is the narrow maintainable fix.
