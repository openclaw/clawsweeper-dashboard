## Summary

Keep open. The PR addresses a real gap on current main: yielded lifecycle ends already carry `yielded: true`, but the persisted gateway session snapshot currently ignores that flag and records a clean end as `done`; the proposed patch maps that state to `paused`, clears it on the next start, and prevents restart recovery from racing the queued continuation. I did not find a blocking correctness or security issue in the diff.

## What This Changes

The PR adds a persisted `paused`/`sessions_yield` lifecycle state for main sessions, propagates it through gateway session rows and sessions tooling, skips restart recovery for yielded sessions, adds focused tests, and records the fix in the changelog.

## Best Possible Solution

Land this after CI and the manual yield/restart smoke checks confirm the persisted session remains paused until the next legitimate run starts, with no broader runner or plugin API changes needed.

## Reproduction Assessment

Yes. A focused reproduction is a lifecycle `end` event with `yielded: true` and `stopReason: "end_turn"`; current main maps that to `done`, while the PR's new unit coverage expects `paused` plus `pauseReason: "sessions_yield"`.

## Solution Assessment

Yes. The patch fixes the narrow persistence/recovery seam that was left after the subagent yield fix, while reusing the existing lifecycle metadata instead of changing runner behavior or adding a separate main-session registry.
