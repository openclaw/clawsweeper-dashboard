## Summary

## What This Changes

The PR persists top-level `sessions_yield` lifecycle endings as paused, carries `pauseReason` through gateway/session rows, skips restart recovery for yielded entries, and adds focused tests plus changelog text.

## Best Possible Solution

Land the gateway lifecycle fix together with Control UI/shared row handling for `paused` and `pauseReason`, plus redacted runtime proof that a yielded main session stays paused and reactivates cleanly on follow-up.

Yes, at source level: current main emits `yielded: true` from the runner while gateway lifecycle persistence ignores that flag and maps clean terminal events to `done`. I did not run a live smoke in this read-only review.

## Solution Assessment
