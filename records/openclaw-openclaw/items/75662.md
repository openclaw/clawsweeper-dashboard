## Summary

Keep open: current main still misclassifies yielded main-session lifecycle ends, and this PR appears to fix the bug at the right gateway persistence and restart-recovery seam with focused coverage. I found no blocking review findings; it should stay open for maintainer approval and final exact-head CI.

## What This Changes

The PR persists main-session `sessions_yield` lifecycle endings as `paused` with `pauseReason: "sessions_yield"`, propagates that state to session surfaces, skips restart recovery for paused entries, and adds regression coverage plus a changelog entry.

## Best Possible Solution

Land this PR after maintainer approval and final exact-head CI so main sessions share the persisted paused-yield state model already used for yielded subagent runs.

## Reproduction Assessment

Yes. From source inspection, current main’s lifecycle helper ignores `yielded` and resolves a clean yielded end as `done`, while the runner already emits `yielded: true` for `sessions_yield`.

## Solution Assessment

Yes. Mapping yielded end events to `paused` with a specific pause reason at the gateway lifecycle seam, then teaching restart recovery to skip that marker, is the narrow maintainable fix.
