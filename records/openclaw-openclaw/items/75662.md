## Summary

The core projection is internally coherent, but a supported Codex-backed main-session runtime detects `sessions_yield` only inside its attempt result and does not propagate that fact into the terminal lifecycle metadata consumed by this patch. The PR therefore creates a persisted/API/UI state that one supported runtime cannot reach, while also expanding a compatibility-sensitive status union; maintainers should keep it open until the runtime boundary and exact-head proof are complete.

## What This Changes

This PR adds a persisted `paused` state and `sessions_yield` pause reason for yielded main-session runs across lifecycle persistence, restart recovery, session APIs, transcript handling, and Control UI presentation.

## Best Possible Solution

Propagate `yieldDetected` from the Codex attempt result into the common terminal lifecycle metadata, audit status/`endedAt` consumers for paused nonterminal semantics, and then land one canonical persisted state with exact-head fresh-run and restart proof.

## Reproduction Assessment

Yes. The exact-head live run and source path show that a Codex-backed main session can call `sessions_yield` while its terminal lifecycle event lacks the `yielded` field required by the new projection.

## Solution Assessment

No, not yet. Persisting `paused` is a reasonable state model, but the narrow durable fix must also update the supported Codex runtime producer and prove uniform behavior before expanding the external contract.
