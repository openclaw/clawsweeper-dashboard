## Summary

Keep open: the PR targets a real current-main lifecycle gap, but it is not merge-ready because the new paused session state is not wired through Control UI/session consumers and the PR still lacks after-fix real behavior proof.

## What This Changes

The PR persists main-session `sessions_yield` lifecycle ends as `status: "paused"`, propagates `pauseReason`, skips restart recovery for paused-yield sessions, and adds focused tests plus changelog text.

## Best Possible Solution

Update the branch so `paused` and `pauseReason` are part of the shared/UI session contract and are rendered/reconciled as a paused non-terminal state, then land after redacted real behavior proof and maintainer approval.

## Reproduction Assessment

Yes, at source level: current main emits `yielded: true` from the runner while gateway lifecycle persistence ignores it and maps a clean yielded end to `done`. I did not run a live smoke in this read-only review.

## Solution Assessment

Not yet. The gateway persistence seam is the right place to classify yielded terminal events, but the patch also needs Control UI/shared session handling for the new paused state and after-fix runtime proof.

## Review Findings

Overall correctness: patch is incorrect
