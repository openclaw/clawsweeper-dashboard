## Summary

This PR addresses a real current-main lifecycle gap, but it is not ready to merge because the new canonical paused state remains unreachable on Codex-backed main sessions and the exact-head proof did not demonstrate the changed persisted state or restart behavior.

## What This Changes

This PR adds a persisted paused state for yielded main-session runs across gateway lifecycle state, restart recovery, session APIs, transcript handling, and Control UI presentation.

## Best Possible Solution

Define yielded pause as a shared main-session lifecycle contract, wire every supported runner to emit it, and then expose the paused state consistently through persistence, gateway APIs, restart recovery, transcript handling, and UI.

## Reproduction Assessment

Yes from source: current main ignores yielded when projecting main-session lifecycle state, while native embedded runs already emit yielded and paused terminal metadata. The exact-head live evidence confirms real yield continuation, but not the new persisted state on the Codex-backed lane.

## Solution Assessment

No, not yet. The downstream projection is coherent, but making paused canonical while one supported main-session runner cannot emit its source metadata is a one-sided fix; the durable solution completes the shared producer contract first.
