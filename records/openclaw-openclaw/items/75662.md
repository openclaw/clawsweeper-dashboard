## Best Possible Solution

## Solution Assessment

Mostly yes: persisting `paused` at the gateway lifecycle seam is the maintainable owner-boundary fix because that is where main-session rows are written and broadcast. The PR should not merge as-is until the generated Control UI i18n metadata is synced.
