## Summary

This PR is still useful because current main still maps clean yielded main-session lifecycle ends to `done`, but it is not merge-ready: the backend adds a new persisted/broadcast `paused` status without updating Control UI consumers, and the supplied proof does not show the real runtime path on the PR head.

## What This Changes

The PR persists top-level `sessions_yield` lifecycle ends as `paused`, carries `pauseReason: "sessions_yield"` through session rows, skips restart recovery for paused entries, and adds focused gateway/recovery tests plus a changelog entry.

## Best Possible Solution

Land the gateway pause-state contract together with matching Control UI/status-consumer handling and redacted runtime proof for yield persistence, restart skip, and next-run reactivation.

## Reproduction Assessment

Yes, source-reproducible: current main emits `yielded: true` and `livenessState: "paused"` from the runner, but gateway lifecycle persistence ignores `yielded` and maps clean terminal events to `done`. I did not run a live gateway smoke in this read-only review.

## Solution Assessment

No, not yet: the gateway lifecycle seam is the right place to persist a yielded pause, but the PR needs matching Control UI/status-consumer support and real runtime proof before it is the best mergeable solution.
