## Summary

Keep open: the core pause/recovery direction is valid and source-reproducible, but the PR is not merge-ready because the new `paused` status is not handled by Control UI consumers and the supplied proof is diff inspection rather than an actual patched run.

## What This Changes

The PR persists main-session `sessions_yield` lifecycle ends as `status: "paused"` with `pauseReason: "sessions_yield"`, propagates that through gateway/session rows, skips restart recovery for paused entries, and adds focused tests plus a changelog entry.

## Best Possible Solution

Update the PR so `paused` is a full session-row contract across gateway, sessions_list, SDK/UI types, Control UI update/reconcile/render paths, and add focused tests plus real patched-run proof before merge.

## Reproduction Assessment

Yes, source inspection on current main gives a high-confidence reproduction path: a lifecycle end event with `yielded: true` still resolves to `done`, and restart recovery has no paused-main-session skip. I did not run a live current-main reproduction.

## Solution Assessment

No, not as-is. The persisted gateway state fix is the right core direction, but the best solution must also update session-row consumers so `paused` is non-terminal and visible rather than being treated as an interrupted run.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Teach Control UI that paused is non-terminal:** `src/gateway/session-lifecycle-state.ts:55`
  - body: This introduces `status: "paused"`, but the Control UI still treats every defined non-`running` status as terminal: `applySessionsChangedEvent` copies `status`, drops `pauseReason`, forces `hasActiveRun=false`, and `reconcileChatRunFromCurrentSessionRow` then clears the chat run as interrupted. A yielded main-session broadcast can therefore still make the dashboard/chat look terminal instead of awaiting continuation. Please update the UI row type, event field whitelist, status rendering, and run reconciliation tests for `paused`.
