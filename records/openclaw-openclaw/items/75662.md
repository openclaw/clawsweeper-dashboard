## Summary

Keep open: the backend lifecycle seam is real, but the PR is not merge-ready because the new paused row state is not wired through Control UI and the supplied proof shows diff inspection rather than a real patched runtime flow.

## What This Changes

The PR persists main-session `sessions_yield` endings as `paused`, propagates `pauseReason`, skips restart recovery for yielded entries, and adds focused tests plus a changelog entry.

## Best Possible Solution

Land the gateway lifecycle fix together with Control UI/shared row handling for `paused` and `pauseReason`, plus redacted runtime proof that a yielded main session stays paused and reactivates cleanly on follow-up.

## Reproduction Assessment

Yes, at source level: current main emits `yielded: true` and `livenessState: "paused"` from the runner, while gateway lifecycle persistence ignores `yielded` and maps the clean terminal event to `done`. I did not run a live smoke in this read-only review.

## Solution Assessment

No, not yet. The gateway lifecycle seam is the right fix point, but the new paused state must be wired through Control UI/shared session handling and proven in a real patched runtime before merge.
