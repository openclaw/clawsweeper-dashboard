## What This Changes

The PR persists main-session `sessions_yield` terminal lifecycle events as `paused`, propagates `pauseReason`, skips restart recovery for paused-yield sessions, and adds focused tests plus changelog text.

## Best Possible Solution

Land this PR after redacted real behavior proof and maintainer approval, while keeping broader sessions_yield progress or streaming work in the related open items.

## Reproduction Assessment

Yes, at source level: current main emits `yielded: true` from the runner but gateway lifecycle persistence ignores it and maps a clean end to `done`. I did not run the live smoke in this read-only review.

## Solution Assessment

Yes. Classifying yielded terminal lifecycle events at the gateway persistence seam, clearing the marker on the next start, and skipping restart recovery for that marker is the narrow maintainable fix.
