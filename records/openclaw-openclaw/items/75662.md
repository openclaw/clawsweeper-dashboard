## Summary

Keep open. The PR addresses a real gap on current main: yielded main-session lifecycle ends still derive a finished session status, while runner metadata already carries yielded/paused semantics. The proposed direction is narrow and well-scoped, but one small consistency issue remains around `pauseReason` on lifecycle error events, and the current head has failing CI checks that need follow-up before merge.

## What This Changes

The PR persists yielded top-level agent sessions as `paused`, adds a `sessions_yield` pause reason through session rows and tooling, skips restart recovery for yielded sessions, and adds focused tests plus a changelog entry.

## Best Possible Solution

Land the PR after the `pauseReason` consistency cleanup, green CI, and the two manual smoke checks confirm that yielded main sessions remain paused until the next legitimate run starts and are not recovered as interrupted turns.

## Reproduction Assessment

Yes. A focused reproduction is a lifecycle `end` event with `yielded: true` and `stopReason: "end_turn"`; current main maps it to `done`, while the PR adds unit coverage expecting `paused` with `pauseReason: "sessions_yield"`.

## Solution Assessment

Mostly yes. Reusing the existing yielded lifecycle metadata in gateway session persistence is the narrowest maintainable fix, but `pauseReason` should be gated on the derived paused status to avoid inconsistent failed+paused snapshots.
