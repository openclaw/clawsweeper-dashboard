## Summary

Keep this PR open for fixes rather than cleanup-close: the reported cron preflight auth gap is plausible from current source, but the branch is not mergeable because it has compile-time errors, does not use the target cron agent's auth scope, can send synthetic no-auth markers as bearer tokens, and lacks after-fix real behavior proof.

## What This Changes

The PR adds API-key resolution and an Authorization bearer header to the cron local-provider preflight request in `src/cron/isolated-agent/model-preflight.runtime.ts`.

## Best Possible Solution

Pass the resolved cron `agentDir` into `preflightCronModelProvider`, call `resolveApiKeyForProvider` with its supported contract, suppress non-secret no-auth markers, and prove the behavior with focused tests plus redacted live cron/LiteLLM output.

## Reproduction Assessment

No high-confidence live reproduction was run. Current main source shows the preflight GET is sent without headers, and the PR body includes before LiteLLM 401 logs, so this is source-reproducible but not live-reproduced here.

## Solution Assessment

No. Reusing `resolveApiKeyForProvider` is the right direction, but this implementation must compile, use the resolved cron `agentDir`, and filter synthetic no-auth markers before it is the best fix.

- **[P1] Import an existing agent-dir resolver:** `src/cron/isolated-agent/model-preflight.runtime.ts:1`
  - body: This import points at `../../agents/agent-paths.js`, but current main has no `src/agents/agent-paths.ts` or matching barrel. The branch will fail module resolution before the cron preflight can run; use an existing resolver or pass the caller's resolved `agentDir` instead.
  - confidence: 0.98
- **[P1] Call the auth resolver with supported parameters:** `src/cron/isolated-agent/model-preflight.runtime.ts:172`
  - body: `resolveApiKeyForProvider` does not accept an `env` property; env lookup is already internal to the resolver. This object literal will fail type-checking, so remove `env` and keep the call inside the exported resolver contract.
  - confidence: 0.96
- **[P1] Pass the cron agentDir into auth resolution:** `src/cron/isolated-agent/model-preflight.runtime.ts:167`
  - body: `runCronIsolatedAgentTurn` already resolves the selected agent's `agentDir` before preflight, and auth profiles are scoped per agent. Resolving an agent dir inside the preflight helper can miss non-default agent credentials or use the main agent's credentials for another cron agent; pass the caller's `agentDir` into `resolveApiKeyForProvider`.
  - confidence: 0.9
- **[P2] Do not send synthetic no-auth markers as bearer tokens:** `src/cron/isolated-agent/model-preflight.runtime.ts:188-190`
  - body: For custom local providers without a real key, `resolveApiKeyForProvider` can return the non-secret `custom-local` marker that normal inference strips before sending headers. This helper turns any non-empty string into `Authorization: Bearer ...`, so authless local servers can start seeing a fake bearer token instead of the current no-header probe.
  - confidence: 0.88
- **[P2] Remove the unused providerCfg parameter:** `src/cron/isolated-agent/model-preflight.runtime.ts:159`
  - body: `providerCfg` is added to `resolveProbeApiKey` but never read, and `tsconfig.core.json` enables `noUnusedParameters`. After the missing import is fixed, this still leaves the branch failing typecheck unless the argument is removed or used.
  - confidence: 0.95
