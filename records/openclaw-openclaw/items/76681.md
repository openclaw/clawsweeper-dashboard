## Summary

Keep open: current main intentionally warns on unreferenced primary session JSONL files and archives them only after interactive confirmation, but the report says current releases keep producing many orphaned sessions after gateway restarts and does not yet include enough state/log evidence to distinguish expected maintenance from a persistence bug.

## What This Changes

The issue reports that `openclaw doctor` repeatedly finds orphaned session transcripts and has archived 599 session files with `.deleted` suffixes after gateway restarts.

## Best Possible Solution

Keep the intentional doctor safety behavior, but diagnose one concrete transcript-to-`sessions.json` mismatch from the reporter and patch the session persistence or restart path if current sessions are becoming unreferenced.

## Reproduction Assessment

No high-confidence reproduction for the suspected restart data-loss root cause. Source and tests show the doctor warning/archive path for unreferenced primary `.jsonl` files, but the report lacks the state sample needed to prove why those files became unreferenced on current main.

## Solution Assessment

Unclear because there is no proposed patch and the root cause is not isolated. The narrowest maintainable path is to collect a `sessions.json` plus orphan filename example and fix the writer/restart path, rather than weakening doctor’s orphan detection.
