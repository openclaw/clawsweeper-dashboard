## Summary

Keep open: current main has intentional doctor handling for orphan transcript files and the obvious session-file persistence paths preserve `sessionFile`, but the report is recent and there is not enough evidence to prove that post-restart active orphan creation is fixed or non-reproducible.

## What This Changes

The issue asks for OpenClaw to stop losing session transcript references so `openclaw doctor` does not repeatedly find orphaned sessions after gateway restarts.

## Best Possible Solution

Capture one current post-restart transcript-to-`sessions.json` mismatch, then patch the specific session writer or restart-recovery path that leaves an active transcript unreferenced while preserving doctor’s orphan detection.

## Reproduction Assessment

No. Source inspection shows doctor can intentionally report and archive unreferenced `.jsonl` files, but there is no high-confidence current-main sequence showing gateway restart creates new active orphan transcripts.

## Solution Assessment

Unclear. The maintainable fix is to identify the exact store or restart path that loses `sessionFile`; weakening doctor’s orphan detection or applying the closed PR without a failing path is not supported by the evidence.
