## Summary

Keep open: current main intentionally detects and can archive unreferenced primary session transcripts, and archived `.deleted` files are excluded from repeat orphan scans, but the report may still indicate a fresh restart/session-persistence bug producing new unreferenced transcripts.

## What This Changes

Investigate why `openclaw doctor` repeatedly finds and archives orphaned session transcripts after gateway restarts on recent npm releases.

## Best Possible Solution

Keep doctor’s orphan warning/archive behavior, collect one fresh transcript-to-`sessions.json` mismatch before and after gateway restart, and patch the session writer or restart handoff if current sessions are becoming unreferenced.
