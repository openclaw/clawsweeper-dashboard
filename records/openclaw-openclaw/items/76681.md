## Summary

Keep open: current main documents and implements intentional doctor archival for unreferenced transcript files, and the obvious session update paths preserve or backfill `sessionFile`, but there is still no current-main proof that active transcripts cannot become orphaned after a gateway restart.

## What This Changes

The issue reports repeated `openclaw doctor` orphaned session findings after gateway restarts, with hundreds of archived `.deleted` transcript files.

## Best Possible Solution

Capture a current post-restart `sessions.json` entry and matching transcript file that become mismatched, then patch the exact writer or restart-recovery path while keeping doctor’s orphan detection intact.

No high-confidence end-to-end reproduction yet. Source shows doctor will report an unreferenced primary `.jsonl` file, but the reported gateway-restart creation path lacks a current-main sequence or a concrete store/transcript sample.

## Solution Assessment

Unclear. The maintainable fix is to identify the store or restart path that loses `sessionFile`; weakening doctor’s orphan detection or reviving the closed PR without a failing path is not supported by the current evidence.
