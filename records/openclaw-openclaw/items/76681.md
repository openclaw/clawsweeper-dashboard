## Summary

Keep open: current main explains the doctor warning and has safeguards that preserve visible `sessionFile` metadata, but there is still no current-main proof for the reported restart-time path that creates newly unreferenced active transcripts.

## What This Changes

The issue requests that OpenClaw stop losing session transcript references across gateway restarts so `openclaw doctor` does not repeatedly archive active sessions as orphaned `.deleted` files.

## Best Possible Solution

Keep doctor’s orphan detection and archive safeguards intact, capture one current post-restart `sessions.json` to transcript mismatch, then patch the specific session writer or restart-recovery path and add focused regression coverage.

## Reproduction Assessment

No. Source confirms the doctor warning for unreferenced primary `.jsonl` files, but the report does not include a current `sessions.json` entry plus matching transcript before and after restart that would reproduce new active transcript orphaning.

## Solution Assessment

Unclear. The narrow maintainable fix is to identify the exact path that drops or fails to persist `sessionFile`; weakening doctor’s orphan detection or relying on the closed unmerged PR would not be justified by the current evidence.
