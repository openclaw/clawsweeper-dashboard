## Summary

## What This Changes

The issue asks OpenClaw to stop losing session transcript references so `openclaw doctor` does not repeatedly archive active session files as orphaned `.deleted` transcripts.

## Best Possible Solution

Capture one current post-restart `sessions.json` entry and transcript filename mismatch, then patch the specific writer or restart-recovery path that drops the reference while keeping doctor’s orphan detection intact.

## Reproduction Assessment

## Solution Assessment
