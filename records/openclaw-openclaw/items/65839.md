## What This Changes

The issue asks for context-overflow recovery guidance to stop recommending a fixed `reserveTokensFloor` of `20000` when that value can be too low for large-context models.

## Best Possible Solution

Have the generic overflow recovery hint compute context-window-aware, numeric reserve-floor guidance from resolved model or session metadata while preserving heartbeat-bleed diagnosis and small-window floor capping.

## Reproduction Assessment

Yes, at source level. Calling `buildContextOverflowRecoveryText()` without heartbeat-bleed evidence still falls through to the static `CONTEXT_OVERFLOW_RESET_HINT` containing the reported `20000 or higher` advice.

## Solution Assessment

Yes. A narrow context-window-aware helper with regression tests, docs, and changelog is more maintainable than only raising the hard-coded number, and it preserves the existing small-window cap behavior.
