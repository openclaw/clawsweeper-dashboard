## Summary

Keep this open. Current main still exposes the reported generic context-overflow hint with the hard-coded `20000 or higher` `reserveTokensFloor` recommendation, and the provided GitHub context shows open PR #73561 as the active unmerged implementation candidate using closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to replace or expand the context-overflow recovery hint so the suggested `reserveTokensFloor` is high enough for larger model context windows.

## Best Possible Solution

Review and land #73561, or an equivalent focused patch, so overflow recovery guidance is derived from the active model/session context window, the heartbeat-specific diagnostic stays intact, and the focused recovery-text tests plus compaction/config docs cover the new behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is calling `buildContextOverflowRecoveryText()` without heartbeat-model evidence; current main returns the generic fallback containing `reserveTokensFloor` and the hard-coded `20000 or higher` guidance, and the focused tests assert that fallback remains present.

## Solution Assessment

Yes for the context-aware direction, but not for a blind constant bump. The runtime already caps reserve floors for small context windows, so the maintainable fix is to compute user-facing guidance from the active context budget and preserve the existing special-case heartbeat diagnostic.
