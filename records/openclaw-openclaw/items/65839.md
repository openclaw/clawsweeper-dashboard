## Summary

Keep open. Current main and the latest release still emit the reported hard-coded `20000 or higher` recovery hint on the generic overflow path, and both directly related implementation PRs were closed without merging.

## What This Changes

The issue asks for context-overflow recovery guidance to stop recommending a hard-coded `reserveTokensFloor` value that is too low for larger context windows.

## Best Possible Solution

Update the overflow recovery helper to emit context-window-aware, copy-pasteable numeric reserve-floor guidance from resolved model/session metadata, while preserving heartbeat-bleed diagnosis and small-window floor capping.

## Reproduction Assessment

Yes, at source level. Calling `buildContextOverflowRecoveryText()` without heartbeat-bleed evidence still appends the static `CONTEXT_OVERFLOW_RESET_HINT`, which contains the reported `20000 or higher` advice on current main.

## Solution Assessment

Yes. A narrow helper/test/docs update is the maintainable path; simply raising the hard-coded number would conflict with current small-context floor capping and still lack model-window guidance.
