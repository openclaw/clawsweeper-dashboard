## Summary

Keep open. Current main and the latest release still emit the reported hard-coded `20000 or higher` recovery hint on the generic overflow path, while the two direct implementation PRs were closed unmerged.

## What This Changes

The issue asks OpenClaw to stop recommending a hard-coded `agents.defaults.compaction.reserveTokensFloor` value of `20000` when that value is too low for larger model context windows.

## Best Possible Solution

Update the overflow recovery helper to emit context-window-aware, copy-pasteable numeric reserve-floor guidance from resolved model/session metadata while preserving the heartbeat-bleed diagnosis and small-window floor capping.

## Reproduction Assessment

Yes, at source level. Calling `buildContextOverflowRecoveryText()` without heartbeat-bleed evidence still appends the static `CONTEXT_OVERFLOW_RESET_HINT`, which contains the reported `20000 or higher` advice on current main.

## Solution Assessment

Yes. A narrow helper, test, and docs update is the maintainable path; only raising the hard-coded value would conflict with small-context floor capping and still lack model-window guidance.
