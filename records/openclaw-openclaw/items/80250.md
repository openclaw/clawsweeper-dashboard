## Summary

Keep open; the PR is useful and has sufficient real CLI proof, but latest head still misses default-agent warnings when a configured channel has only scoped bindings because the collector suppresses the whole channel once any binding exists.

## What This Changes

This PR adds tool-policy prompt guidance, a doctor preview warning for channel-routed/default agents missing the `message` tool, regression tests, and a changelog note.

## Best Possible Solution

Keep the prompt guidance and warning, but make doctor target collection mirror routing fallback for configured channels with only scoped bindings, then land with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a configured channel, a default agent without `message`, and only an account/peer/guild-scoped binding on that same channel will route unmatched traffic to the default agent while this PR skips the warning.

## Solution Assessment

No. The direction is maintainable, but the implementation should only treat a channel as fully covered when a route actually prevents default fallback for that channel class; scoped bindings still need default-agent warning coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Include default warnings for scoped channel bindings:** `src/commands/doctor/shared/preview-warnings.ts:249-250`
  - body: `explicitlyBoundChannels` marks a channel as covered even when the binding is scoped by account, peer, guild, team, or roles. Routing still falls back to the default agent for unmatched traffic on that same configured channel, so a Discord config with only an account-specific route to a messaging agent and a default agent without `message` will emit no warning for default-routed Discord requests. Please include default fallback targets for partially scoped channel bindings and add a regression test for that case.
  - confidence: 0.9
