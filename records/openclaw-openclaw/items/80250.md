## Summary

Keep open: the latest PR head addresses the earlier missing-accountId suppression, but the doctor target collection is still too channel-level and can warn the default agent for channels whose configured accounts are already covered by explicit account routes.

## What This Changes

The PR adds system-prompt guidance, doctor preview warnings for routed/default channel agents missing the `message` tool, regression tests, and a changelog entry.

## Best Possible Solution

Make doctor target collection account-aware: warn the default agent only for uncovered configured account/default traffic, while keeping the prompt guidance and route-bound warning.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a channel with explicit account routes that cover all configured accounts and a default agent without `message`; the PR still adds the default agent because there is no wildcard account route.

## Solution Assessment

No. The prompt guidance and core warning direction are right, but the doctor fallback target collection should mirror route account semantics instead of treating all non-wildcard channels as default-agent traffic.
