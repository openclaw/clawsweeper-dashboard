## Summary

Keep open: the branch still has a source-level correctness blocker because doctor route coverage canonicalizes route channel aliases while runtime route matching only lowercases channel ids, so an alias route can suppress the default-agent missing-message warning even though runtime falls back to that default agent.

## What This Changes

The PR adds system-prompt guidance for policy-filtered tools, a doctor preview warning for routed/default channel agents missing the `message` tool, regression tests, and a changelog entry.

## Best Possible Solution

Keep the prompt guidance and message-tool doctor warning, but make doctor route coverage use the same channel/account matching semantics as `resolveAgentRoute` and add an alias-vs-canonical regression.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `channels.imessage`, add a route with `match.channel: "imsg"` to a messaging agent, and keep the default agent without `message`; runtime falls back to default while the PR suppresses that default warning.

## Solution Assessment

No. The direction is maintainable, but the collector must mirror runtime route matching, or runtime must intentionally gain alias matching with paired tests.
