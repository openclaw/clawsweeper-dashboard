## Summary

Keep open: the PR is the active fix path and the supplied CLI proof is sufficient, but current head still misses default-agent warnings when a configured channel has only scoped route bindings and unmatched traffic falls back to the default agent.

## What This Changes

Adds prompt guidance, doctor preview warnings for channel-routed/default agents missing the `message` tool, regression tests, and a changelog entry.

## Best Possible Solution

Keep the prompt guidance and warning, but make doctor target collection include default fallback targets for configured channels that are only partially covered by scoped bindings.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a configured channel, a default agent without `message`, and only an account/peer/guild/team/role-scoped binding on that channel leaves unmatched traffic routed to the default agent while the PR suppresses the warning.

## Solution Assessment

No. The direction is maintainable, but the implementation should only treat a channel as fully covered when a route actually prevents default fallback for that channel class.

## Review Findings
