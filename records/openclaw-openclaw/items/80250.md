## Summary

Keep open: the PR fixes the right doctor/prompt surface and has sufficient CLI proof, but the latest head still has false-positive doctor warning paths because its route coverage collector does not fully mirror runtime routing semantics.

## What This Changes

The PR adds prompt guidance, doctor preview warnings for routed/default channel agents missing the `message` tool, regression tests, and a changelog entry.

## Best Possible Solution

Make the doctor collector reuse the runtime route key and default-account coverage semantics, preserving the new warning while avoiding false positives.

## Reproduction Assessment

Yes. Source inspection gives high-confidence paths: a `channels.discord` config with `match.channel: "discord"` and no `accountId` covers the default account at runtime, and `match.channel: "Discord"` is normalized by runtime routing but not by the PR collector.

## Solution Assessment

No. The prompt guidance and doctor warning are the right direction, but the collector should normalize route channel ids and count default-account bindings as coverage before warning the default agent.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Honor default-account route coverage:** `src/commands/doctor/shared/preview-warnings.ts:307-310`
  - body: A route with no `match.accountId` matches the runtime default account, and a channel with no configured `accounts` only has that default account path. This branch treats `configuredAccountIds.length === 0` as uncovered and adds the default agent anyway, so `doctor` can warn `main` for Discord traffic that runtime will route to the explicit agent. Include `DEFAULT_ACCOUNT_ID` in the coverage check when no accounts are configured, or otherwise treat a covered default account as full coverage for that case.
  - confidence: 0.9
- **[P2] Normalize route channel keys before coverage checks:** `src/commands/doctor/shared/preview-warnings.ts:282-288`
  - body: The collector stores `binding.match.channel.trim()` in the target and coverage maps, but runtime routing lowercases channel tokens before indexing bindings. With `channels.discord` and a route using `match.channel: "Discord"`, runtime selects the routed agent while `doctor` misses the coverage and can warn the default agent. Normalize the channel key before `add`, `fullyCoveredChannels`, and `coveredAccountsByChannel`, and add a regression for the mixed-case route.
  - confidence: 0.86
