## Summary

Keep open: the latest head still has a source-verifiable false positive where doctor warns the default agent for a configured channel whose default-account traffic is already covered by a channel-only route.

## What This Changes

The PR adds prompt guidance, doctor preview warnings for channel-routed/default channel agents missing the `message` tool, regression tests, and a changelog entry.

## Best Possible Solution

Make the new doctor target collection account-aware so it warns only selected agents for uncovered default/configured account traffic, while keeping the prompt guidance and route-bound warning.

## Reproduction Assessment

Yes. Source inspection is enough: current routing maps a missing route `accountId` to the default account, and the PR's wildcard-only coverage still warns the default agent for default-account traffic already routed to another agent.

## Solution Assessment

No. The prompt guidance and warning direction are right, but the doctor fallback target collection should mirror route account semantics instead of treating only wildcard account routes as covering channel traffic.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

- **[P2] Make channel-only routes cover default-account traffic:** `src/commands/doctor/shared/preview-warnings.ts:222`

This helper only marks `accountId: "*"` as covered, so doctor also adds the default agent and emits a spurious missing-`message` warning for `main`. Track covered default/configured accounts instead of using wildcard-only channel coverage.
