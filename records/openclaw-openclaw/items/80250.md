## Summary

Keep open: the PR takes the right prompt-plus-doctor direction, but its warning target collection still misses configured channels that fall back to the default agent when any explicit route binding exists.

## What This Changes

The PR adds agent prompt guidance, doctor preview warnings for routed or default-channel agents missing the `message` tool, focused tests, and a changelog entry for the linked bug.

## Best Possible Solution

Keep the prompt guidance and doctor warning, but make target collection mirror routing fallback by assigning unclaimed configured channels to the default agent and covering mixed explicit/default route configs.

## Reproduction Assessment

Yes. The bug and the PR blocker are source-reproducible from tool-policy filtering, current main’s missing route-bound warning, and routing’s default fallback path; I did not run a live Discord/Ollama flow.

## Solution Assessment

No. The direction is maintainable, but this implementation is incomplete for mixed binding configurations; the safer fix is to include default-agent fallback channel targets when configured channels are not covered by explicit bindings.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Include default-route channels when bindings are partial:** `src/commands/doctor/shared/preview-warnings.ts:242-247`
  - body: This only adds the default agent's configured channels when there are zero route bindings. Routing still falls back to the default agent when no binding matches, so a config with Discord explicitly routed to another agent and Telegram configured without a binding will route Telegram to the default agent but emit no warning if that default agent lacks `message`. Please include fallback-default channel targets for mixed binding configs and cover that case in the new tests.
