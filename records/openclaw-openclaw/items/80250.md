## Summary

Keep open: the PR is aimed at the right bug, but the latest head still suppresses the default-agent warning for multi-account channels when a route uses `match.channel` without `accountId`. Current routing treats that binding as default-account-only, not any-account.

## What This Changes

The PR adds agent prompt guidance, doctor preview warnings, regression tests, and a changelog entry for channel-routed agents that cannot call the `message` tool.

## Best Possible Solution

Mirror route account semantics in doctor target collection: only wildcard account bindings should fully cover a channel, and default-agent warnings should remain for uncovered configured accounts.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a channel with another account, bind only `match.channel` to a messaging-capable agent, and keep the default agent without `message`; routing sends the other account to default while this PR suppresses the warning.

## Solution Assessment

No. The PR is aimed at the right surfaces, but target collection must distinguish default-account bindings from wildcard account bindings before this is the narrowest correct fix.
