## Summary

Keep open: this PR is the right direction and the supplied CLI proof is sufficient, but the updated doctor collector still has a default-account false-positive path for channels with no configured accounts.

## What This Changes

The PR adds prompt guidance for policy-filtered tools, doctor preview warnings for channel-bound agents missing the message tool, regression tests, and a changelog entry.

## Best Possible Solution

Make the doctor collector mirror runtime route account coverage, treating a no-account configured channel as the default account before adding a default-agent warning.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `channels.discord` with no `accounts`, a route `match.channel: "discord"` without `accountId`, and a default agent without `message` still produces the PR collector's fallback warning even though runtime routes default-account traffic to the explicit agent.

## Solution Assessment

No. The prompt guidance and new doctor warning are maintainable, but the route collector should apply the same default-account coverage semantics as runtime before warning the default agent.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Honor default-account route coverage:** `src/commands/doctor/shared/preview-warnings.ts:317-319`
  - body: When a configured channel has no `accounts`, runtime still resolves omitted account ids as the `default` account. Treat the empty configured-account set as `DEFAULT_ACCOUNT_ID`, or otherwise check covered default-account routes before adding the fallback.
