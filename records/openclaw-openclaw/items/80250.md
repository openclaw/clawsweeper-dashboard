## Summary

Keep open: the PR is a good direction and has sufficient proof for the main doctor warning, but the latest head still lets doctor route coverage diverge from runtime route matching for channel aliases, so it can suppress a real default-agent missing-`message` warning.

## What This Changes

The PR adds policy-filtered tool guidance to the agent prompt, adds doctor preview warnings for channel-routed/default-channel agents without the `message` tool, adds regressions, and updates the changelog.

## Best Possible Solution

Keep the prompt guidance and message-tool doctor warning, but make doctor route coverage use the same channel/account matching semantics as `resolveAgentRoute`, or intentionally change runtime alias matching with paired route tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `channels.imessage`, add a route with `match.channel: "imsg"` to a messaging agent, and keep the default agent without `message`; runtime falls back to default while the PR suppresses that default warning.

## Solution Assessment
