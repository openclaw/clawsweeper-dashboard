## Summary

Keep open: the PR is a plausible fix direction and has real `doctor` output proof, but the current head still misses default-agent fallback channels whenever any explicit route binding exists.

## What This Changes

The PR adds agent prompt guidance, a doctor preview warning for channel-bound/default-channel agents lacking the `message` tool, regression tests, and a changelog entry.

## Best Possible Solution

Keep the prompt guidance and doctor warning, but make target collection mirror routing fallback for mixed explicit-route/default-channel configs before merging.

## Reproduction Assessment

Yes. Current main source shows `message` can be filtered out by a non-empty allowlist and lacks the new doctor warning; the PR-specific miss is source-reproducible from routing's default fallback plus the `routeBindings.length === 0` guard.

## Solution Assessment

No. The direction is maintainable, but this implementation is incomplete for partial route configurations; the safer fix is to include fallback-default channel targets for enabled configured channels not matched by explicit bindings.
