## Summary

Keep open: the PR fixes the earlier mixed-channel fallback gap and addresses a real bug, but its target collection still treats any binding for a channel as covering the whole channel. Routing falls back to the default agent when account/peer/guild-scoped bindings do not match, so doctor can still miss a default-routed channel agent without the `message` tool.

## What This Changes

This PR adds policy-filtered missing-tool guidance to agent prompts, a doctor warning for channel-routed or default-routed agents without the `message` tool, regression tests, and a changelog entry.

## Best Possible Solution

Keep the prompt guidance and doctor warning, but make target collection mirror route fallback for scoped bindings and add focused regression coverage before merge.

## Reproduction Assessment

Yes. Source inspection shows route fallback returns the default agent when scoped bindings do not match, and the PR target collection suppresses that default warning by channel id; the PR body also supplies real `openclaw doctor` output for the main fixed path.

## Solution Assessment

No. The direction is right, but the current implementation should only treat truly channel-wide bindings as covering the default fallback, or otherwise include the default agent for scoped binding gaps.
