## Summary

Keep open: the latest head fixes the earlier account-coverage problem, but the new doctor target collector still compares raw route channel strings with configured channel keys, so it can warn the default agent even when routing would use a covered channel binding.

## What This Changes

The PR adds system-prompt guidance, doctor preview warnings for routed/default channel agents missing the `message` tool`, regression tests, and a changelog entry.

## Best Possible Solution

Normalize channel ids in the doctor collector with the same route key used by runtime routing, while preserving the account-specific and wildcard coverage semantics added by the latest head.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `channels.discord`, a covered route using `match.channel: "Discord"`, and a default agent without `message`; runtime routing normalizes the route to `discord`, but the PR's doctor coverage keeps `Discord` raw and can warn the default agent.

## Solution Assessment

No. The prompt guidance and routed-agent warning direction are right, but the doctor target collector should normalize route channel ids before computing covered channels and covered accounts.
