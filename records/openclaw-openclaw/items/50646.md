## Summary

Keep open: current main and the latest shipped release still filter `sessions_list` results by effective session visibility, then return a complete-looking payload without result-level scope disclosure. I found adjacent visibility work, but no merged or open canonical item that owns this exact disclosure fix.

## What This Changes

Request additive disclosure when session list/status tools return visibility-scoped results so users and agents do not mistake filtered output for the full session store.

## Best Possible Solution

Add additive `sessions_list` result metadata or a concise warning when effective visibility is not `all`, preserve current filtering, avoid leaking hidden counts, and update focused tests/docs/changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with restricted session visibility and additional off-scope sessions in the store, `sessions_list` filters rows and returns only `count` plus visible `sessions` without scope metadata.

## Solution Assessment

Yes. The narrowest maintainable solution is additive disclosure after the existing visibility decision; changing the default scope or exposing hidden counts would be broader and riskier.
