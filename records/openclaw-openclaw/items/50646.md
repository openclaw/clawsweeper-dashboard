## Summary

Keep open: current main still filters agent-facing `sessions_list` by effective `tools.sessions.visibility` and returns a complete-looking `count`/`sessions` payload without scope metadata. The KeepMyClaw comment is useful adjacent session-durability context, but it does not replace the missing OpenClaw tool-output disclosure.

## What This Changes

Request that `sessions.list` and related session tools disclose when results are scoped by `tools.sessions.visibility` instead of returning restricted results that look complete.

## Best Possible Solution

Add an additive scope disclosure to agent-facing session tool results when effective visibility is not `all`, while preserving current filtering, defaults, and privacy boundaries.

## Reproduction Assessment

Yes. Source inspection shows current main resolves restricted `tools.sessions.visibility`, filters rows through the list visibility checker, then returns only `count` and `sessions`, so hidden rows can produce a complete-looking result.

## Solution Assessment

Yes. The narrow maintainable fix is an additive result-level disclosure; changing the default visibility or adding a new config knob would be a larger and riskier policy change.
