## Summary

Keep open: current main and the latest shipped release still apply restricted session visibility before returning `sessions_list`, but the final tool payload only exposes `count` and `sessions`, so scoped results can still look complete. I did not find an open or merged canonical item that already owns this exact result-level disclosure work.

## What This Changes

The issue requests an additive warning or metadata when session-list results are scoped by restricted session visibility.

## Best Possible Solution

Add additive result-level scope metadata or a concise warning to `sessions_list` when effective visibility is not `all`, while preserving current filtering and avoiding hidden-session counts that would leak unauthorized state.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure restricted session visibility, have more stored sessions than the caller can list, and `sessions_list` returns a filtered `count`/`sessions` payload without scope metadata.

## Solution Assessment

Yes. The narrowest maintainable fix is additive disclosure after the existing visibility decision; changing default visibility or exposing unauthorized hidden counts would be broader and riskier.
