## Summary

Keep open: current main still filters agent-facing `sessions_list` by effective session-tool visibility and returns a complete-looking `count`/`sessions` payload without result-level scope metadata. The closed Discord scoping report and the KeepMyClaw/session-state comments are useful context, but they do not replace the missing OpenClaw tool-output disclosure requested here.

## What This Changes

The issue requests an additive disclosure when session tools return results scoped by `tools.sessions.visibility` instead of appearing globally complete.

## Best Possible Solution

Add an additive scope disclosure to `sessions_list` results when effective visibility is not `all`, preserving current filtering, defaults, and privacy boundaries.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: configure restricted session visibility, have more rows in the gateway list than the caller can see, and `sessions_list` will return only `count` plus visible `sessions` without scope metadata. I did not run a live tool invocation in this read-only review.

## Solution Assessment

Yes, the best fit is a narrow additive result-level disclosure; changing the default visibility or adding a new config knob would be a larger and riskier policy change.
