## Summary

Keep open: current main and the latest shipped tag still filter the agent `sessions_list` result by effective session visibility, then return a complete-looking `count`/`sessions` payload with no result-level scope disclosure.

## What This Changes

Request that session listing/status tools disclose when their results are scoped by `tools.sessions.visibility` instead of presenting filtered results as complete.

## Best Possible Solution

Add additive scope metadata or a short warning to `sessions_list` when effective visibility is not `all`, while preserving current filtering and avoiding unauthorized hidden-session counts.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure restricted session visibility, have more stored sessions than the caller can list, and `sessions_list` returns a filtered `count`/`sessions` payload without scope metadata.

## Solution Assessment

Yes. The narrowest maintainable fix is additive result-level disclosure after the existing visibility decision; changing default visibility or exposing unauthorized hidden counts would be broader and riskier.
