## Summary

Keep open. Current main still applies restricted session visibility to `sessions_list` without surfacing the effective scope in the successful tool output, and the related #50621/KeepMyClaw discussion is useful adjacent context rather than a merged fix or superseding tracker.

## What This Changes

The issue requests explicit runtime disclosure when session tools, especially `sessions_list`, return visibility-scoped results.

## Best Possible Solution

Add leak-safe additive visibility disclosure to restricted session-tool outputs, especially `sessions_list`, without broadening access or changing the default policy.

## Reproduction Assessment

Yes. A high-confidence source-level path exists on current main: default visibility resolves to `tree`, `sessions_list` filters through `createSessionVisibilityGuard`, then returns `count` and `sessions` without any scope disclosure.

## Solution Assessment

Yes, the additive disclosure direction is the narrowest maintainable solution; changing default visibility or returning hidden session identities would be broader and riskier. The filtered-count part should remain a maintainer decision because it can leak or mislead under other filters.
