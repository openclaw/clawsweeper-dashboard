## Summary

Keep open: current main still filters agent-facing `sessions_list` by effective `tools.sessions.visibility` and returns a complete-looking `count`/`sessions` payload without scope metadata. This is a narrow UX enhancement, not a backend correctness bug.

## What This Changes

Request session tools, especially `sessions_list`, to disclose when results are scoped by restricted `tools.sessions.visibility` such as the default `tree` mode.

## Best Possible Solution

Add an additive visibility/scope note or metadata field to agent-facing `sessions_list` results when effective visibility is not `all`, preserving current filtering and defaults.

## Reproduction Assessment

Yes. Source-reproducible: current main resolves effective visibility, filters Gateway rows through `createSessionVisibilityRowChecker`, then returns only `count` and `sessions`, so hidden rows can produce a complete-looking result.

## Solution Assessment

Yes. The narrow maintainable fix is an additive tool-output signal; changing the default visibility or adding a new config option would be a larger product decision.
