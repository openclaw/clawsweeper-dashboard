## Summary

Keep open: current main and the latest shipped tag still filter the agent `sessions_list` result by effective session visibility, then return a complete-looking `count`/`sessions` payload with no result-level scope disclosure. Related searches found broader session-visibility work, but this issue remains the canonical tracker for the exact metadata/warning gap.

## What This Changes

The issue asks for session-list tooling to disclose when results are scoped by restricted `tools.sessions.visibility` rather than appearing complete.

## Best Possible Solution

Add additive scope metadata or a warning to `sessions_list` results when effective visibility is not `all`, while preserving current filtering and privacy boundaries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure restricted session visibility, have more Gateway rows than the caller can see, and `sessions_list` returns only filtered `count` plus `sessions` without scope metadata.

## Solution Assessment

Yes. A narrow additive result-level disclosure is the best fit; changing default visibility or returning hidden-session counts would be broader and riskier.
