## Summary

Current main and the v2026.6.6 release already include the bounded memory-core retry that fixes the reported same-run narrative message visibility race, with regression coverage for delayed assistant text before fallback.

## What This Changes

The issue reports misleading memory-core dreaming narrative warnings when narrative subagent sessions contain extractable assistant text.

## Best Possible Solution

Keep the shipped bounded-retry implementation and use the open related Dreaming issues for any distinct diary or session-switcher behavior that still reproduces.

## Reproduction Assessment

Yes for the original affected behavior: the report included concrete run counts and the pre-fix path performed a single immediate post-run message read before fallback. On current main the failing path is covered by regression tests rather than reproduced live.

## Solution Assessment

Yes. The merged bounded retry is the narrowest maintainable fix for this issue because it preserves the existing subagent API and fallback path while waiting briefly for persisted assistant text.
