## Summary

Current main and the shipped v2026.6.6 release already include the bounded memory-core retry that fixes the reported same-run narrative message visibility race, with regression coverage proving delayed assistant text is used before the no-text fallback warning fires.

## What This Changes

The issue reports misleading memory-core dreaming narrative warnings when assistant narrative text exists but is not visible to the extractor at warning time.

## Best Possible Solution

Keep the shipped bounded-settle implementation and close this report; broader Dreaming narrative or session-visibility concerns should remain tracked in their narrower open issues.

## Reproduction Assessment

Yes by source and merged-fix evidence: the reported path is a successful narrative run followed by an immediate `getSessionMessages` read before assistant text is visible. The shipped regression test models delayed persistence and verifies no `produced no text` warning when text appears on a later read.

## Solution Assessment

Yes. The shipped fix is the narrowest maintainable solution for this symptom because it stays inside memory-core, preserves the fallback after bounded attempts, and does not alter the plugin SDK or gateway session contract.
