## Summary

Current main and the shipped 2026.6.6 release train now fix the reported memory-core race: successful narrative runs retry the session message read briefly before warning and falling back, with regression coverage for delayed assistant text.

## What This Changes

The issue reports misleading memory-core dreaming narrative no-text warnings when the narrative subagent session contains extractable assistant text.

## Best Possible Solution

Keep the shipped bounded retry implementation and use newer, distinct Dreaming fallback reports for any remaining variants that reproduce after the release.

## Reproduction Assessment

Yes, at source level. The pre-fix code performed one immediate post-run session read before warning/fallback, and the merged regression test models delayed assistant text becoming visible on a later read.

## Solution Assessment

Yes. The merged fix is the narrow memory-core owner-boundary solution: retry the existing session reader briefly after a successful run, preserve the fallback path, and avoid SDK or persistence contract changes.
