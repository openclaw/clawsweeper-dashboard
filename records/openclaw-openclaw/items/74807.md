## Summary

Keep open. Current main still lets all-error isolated cron payloads override recovered `finalAssistantVisibleText`, and PR #74840 is already the open closing implementation candidate.

## What This Changes

The issue reports that isolated cron agent-turn announcements can summarize an intermediate tool error instead of the final assistant reply after self-correction.

## Best Possible Solution

Review and land PR #74840, or a narrow equivalent change in `resolveCronPayloadOutcome` with regression coverage that preserves explicit run-level and structured fatal errors.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence helper-level reproduction by calling `resolveCronPayloadOutcome` with only error payloads, `finalAssistantVisibleText: "Recovered final answer"`, and `preferFinalAssistantVisibleText: true`; the current test asserts the error wins.

## Solution Assessment

Yes on the direction. The narrow maintainable fix belongs in `resolveCronPayloadOutcome` with focused regression coverage, while preserving run-level failures and structured fatal-error signals.
