## Summary

Keep open. Current main still preserves the reported cron summary behavior: an all-error structured payload set blocks `finalAssistantVisibleText` from becoming the selected output, and the focused helper test still asserts that a recovered assistant reply loses to the last error payload. The issue also has an open closing PR, so the conservative maintainer path is to review and land or replace that PR rather than close this issue now.

## What This Changes

The issue asks isolated cron `agentTurn` summaries to report the agent's final self-corrected assistant reply instead of an intermediate tool error after recovery.

## Best Possible Solution

Review and land PR #74840, or a narrow equivalent, so isolated cron summaries can surface recovered final assistant output while preserving fatal reporting for true run-level failures and denial or unresolved-error cases.

## Reproduction Assessment

Yes. A source-level reproduction is clear: call `resolveCronPayloadOutcome` with `preferFinalAssistantVisibleText: true`, `finalAssistantVisibleText: "Recovered final answer"`, and only structured error payloads; the existing helper test asserts that `last error` is still selected.

## Solution Assessment

Unclear as written in the issue, because always trusting the last assistant message could hide real fatal runs. The best fix should narrowly prefer final assistant text only for recovered or presentation-only errors while preserving existing fatal behavior for unresolved failures.
