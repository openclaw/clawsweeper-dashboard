## Summary

Keep open. Current main still has a source-level path and regression test where all-error payloads override recovered final assistant text, and open PR #74840 is already the active closing implementation path.

## What This Changes

Cron `agentTurn` summaries should prefer a recovered final assistant reply over intermediate tool error payloads when an isolated run self-corrects.

## Best Possible Solution

Review and land PR #74840, or a narrow equivalent helper/test change that prefers recovered final assistant text while preserving fatal error contracts.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `resolveCronPayloadOutcome` with only error payloads, `finalAssistantVisibleText: "Recovered final answer"`, and `preferFinalAssistantVisibleText: true`; the current test asserts the last error still wins.

## Solution Assessment

Yes on the direction, with a narrow boundary. The maintainable fix belongs in `resolveCronPayloadOutcome` and should preserve run-level failures, denial-token handling, failure signals, and structured delivery behavior.
