## Summary

Keep open. Current main still lets all-error cron delivery payloads become fatal before the recovered `finalAssistantVisibleText` can be selected, and PR #74840 is already the open closing implementation candidate.

## What This Changes

The issue asks isolated cron `agentTurn` summaries to prefer the final recovered assistant reply over an intermediate tool-error payload.

## Best Possible Solution

Review and land PR #74840, or a narrow equivalent change in `resolveCronPayloadOutcome` with regression coverage that preserves explicit run-level and fatal failure-signal errors.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path by calling `resolveCronPayloadOutcome` with only error payloads, `finalAssistantVisibleText: "Recovered final answer"`, and `preferFinalAssistantVisibleText: true`; the current helper test asserts the error wins.

## Solution Assessment

Yes on the direction. The narrow maintainable fix belongs in `resolveCronPayloadOutcome` with focused regression coverage, while preserving true run-level failures and structured fatal-error contracts.
