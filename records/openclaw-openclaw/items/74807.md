## Summary

Keep open. Current main still lets fatal structured error payloads override `finalAssistantVisibleText`, and the existing helper test still expects a recovered final assistant reply to lose to the last error payload. The issue is also paired with open closing PR #74840, so it should stay open until that PR or a narrow equivalent lands.

## What This Changes

The issue asks for isolated cron `agentTurn` summaries to prefer the final self-corrected assistant reply over an earlier tool error when the run later succeeds.

## Best Possible Solution

Review and land PR #74840, or a narrow equivalent, so recovered final assistant output can drive isolated cron summaries while preserving fatal reporting for run-level errors, denial markers, and unresolved tool failures.

## Reproduction Assessment

Yes. The current helper test directly exercises the boundary by passing recovered final assistant text alongside only error payloads and expecting the last error to win; source inspection shows the runner uses that helper for the recorded cron result.

## Solution Assessment

No, not as the issue's blanket rule. The maintainable fix should prefer final assistant text only when the error was recovered or presentation-only, while leaving explicit run failures and denial signals fatal.
