## Summary

Keep open. Current main still lets an all-error isolated cron payload set override recovered `finalAssistantVisibleText`, and PR #74840 is already the open closing implementation candidate.

## What This Changes

The issue asks cron isolated agent-turn announcements to summarize the final self-corrected assistant reply instead of an earlier tool-error payload.

## Best Possible Solution

Land PR #74840 or a narrow equivalent in `resolveCronPayloadOutcome` with regression coverage, while preserving explicit run-level errors, failure signals, and denial markers as fatal.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence helper-level reproduction: call `resolveCronPayloadOutcome` with only error payloads, `finalAssistantVisibleText: "Recovered final answer"`, and `preferFinalAssistantVisibleText: true`; the current test expects the last error to win.

## Solution Assessment

Yes on the direction. The narrow maintainable fix belongs in `resolveCronPayloadOutcome` with focused regression coverage and guardrails for real run-level/failure-signal errors.
