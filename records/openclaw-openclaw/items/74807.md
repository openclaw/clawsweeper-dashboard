## Summary

Keep open. Current main still blocks recovered `finalAssistantVisibleText` when the collected cron payloads are all errors, and PR #74840 is already the open closing implementation candidate.

## What This Changes

The issue asks isolated cron `agentTurn` summaries to prefer the agent's final assistant reply after self-correction instead of announcing an intermediate tool error.

## Best Possible Solution

Review and land PR #74840, or a narrow equivalent change in `resolveCronPayloadOutcome` with regression coverage that preserves explicit run-level failures.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence helper-level reproduction by calling `resolveCronPayloadOutcome` with only error payloads, `finalAssistantVisibleText: "Recovered final answer"`, and `preferFinalAssistantVisibleText: true`; the current test asserts the error wins.

## Solution Assessment

Yes on the direction. The narrow maintainable fix belongs in `resolveCronPayloadOutcome` with focused regression coverage, while preserving run-level failures and structured fatal-error signals.
