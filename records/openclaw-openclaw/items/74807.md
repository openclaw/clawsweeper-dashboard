## Summary

Keep open. Current main still marks all-error delivery payloads as fatal before allowing recovered `finalAssistantVisibleText`, and #74840 is already the open closing implementation candidate.

## What This Changes

Cron isolated `agentTurn` summaries should prefer a recovered final assistant reply over an intermediate tool-error payload.

## Best Possible Solution

Land #74840 or an equivalent narrow helper/test change that lets recovered final assistant text override transient tool-error payloads while preserving explicit run-level failures.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence helper-level reproduction by calling `resolveCronPayloadOutcome` with only error payloads, `finalAssistantVisibleText: "Recovered final answer"`, and `preferFinalAssistantVisibleText: true`; the current test asserts the error wins.

## Solution Assessment

Yes on the direction. The narrow maintainable fix belongs in `resolveCronPayloadOutcome` with focused regression coverage, while preserving run-level failures and structured fatal-error signals.
