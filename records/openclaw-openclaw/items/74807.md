## Summary

Keep open. Current main still marks an all-error isolated cron payload set as fatal before `finalAssistantVisibleText` can be selected, and PR #74840 remains the open closing implementation candidate.

## What This Changes

Cron isolated `agentTurn` summaries should prefer a recovered final assistant reply over intermediate tool-error payloads when no run-level failure remains.

## Best Possible Solution

Review and land PR #74840, or an equivalent narrow change in `resolveCronPayloadOutcome` with regression coverage that preserves explicit run-level and structured fatal failures.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence helper-level reproduction by calling `resolveCronPayloadOutcome` with only error payloads, `finalAssistantVisibleText: "Recovered final answer"`, and `preferFinalAssistantVisibleText: true`; current tests assert the error wins.

## Solution Assessment

Yes on the direction. The narrow maintainable fix belongs in `resolveCronPayloadOutcome` with focused regression coverage, while preserving run-level failures and structured fatal-error signals.
