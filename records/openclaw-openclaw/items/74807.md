## Summary

Keep open. Current main still promotes all-error isolated cron payloads to a fatal result before `finalAssistantVisibleText` can win, and PR #74840 is already the open implementation candidate.

## What This Changes

The issue asks isolated cron `agentTurn` summaries to prefer the final assistant-visible reply over intermediate tool-error payloads after the agent self-corrects.

## Best Possible Solution

Adjust `resolveCronPayloadOutcome` and its regression coverage so recovered final assistant text can supersede intermediate tool-error payloads, while true run-level failures remain fatal.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence helper reproduction by calling `resolveCronPayloadOutcome` with only error text payloads, `finalAssistantVisibleText: 'Recovered final answer'`, and `preferFinalAssistantVisibleText: true`; the current test expects `last error`.

## Solution Assessment

Yes on the requested direction. The narrow maintainable fix belongs in `resolveCronPayloadOutcome`, with focused regression coverage and explicit preservation of run-level failure semantics.
