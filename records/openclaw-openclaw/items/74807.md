## Summary

Keep open. Current `main` still lets an all-error delivery payload set make `resolveCronPayloadOutcome` ignore `finalAssistantVisibleText`, and the existing helper test preserves that behavior. The issue is also paired with open, unmerged PR #74840, so the issue should remain open until that PR or a narrow equivalent lands.

## What This Changes

The issue reports that isolated cron `agentTurn` summaries can announce an intermediate tool error instead of the agent’s final self-corrected assistant reply.

## Best Possible Solution

Review and land PR #74840, or a narrow equivalent, so recovered final assistant output can drive isolated cron summaries while preserving fatal reporting for run-level errors, explicit denial signals, and truly unresolved tool failures.

## Reproduction Assessment

Yes. The current helper test directly models recovered final assistant text alongside only error payloads and expects the last error to win; source inspection shows the isolated cron runner uses that helper for recorded run status and fallback delivery.

## Solution Assessment

No, not as the blanket rule stated in the issue. The best maintainable fix should prefer final assistant text only for recovered or presentation-only cases, while preserving fatal semantics for run-level failures, denial markers, and unresolved error payloads.
