## Summary

Keep open. Current main and the latest release still contain the reported resolver behavior, and the open related PRs are implementation candidates with unresolved review rather than merged fixes.

## What This Changes

Cron `agentTurn` summaries should report a self-corrected final assistant reply instead of surfacing an intermediate recovered tool error as the run result.

## Best Possible Solution

Define and land one consolidated resolver contract that marks recovered intermediate tool errors nonfatal only with trusted recovery evidence, while preserving fatal run-level errors, denials, provider failures, and unrecovered trailing tool errors with regression coverage for both sides.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `resolveCronPayloadOutcome` with only error payloads plus `finalAssistantVisibleText` and `preferFinalAssistantVisibleText: true`; the current helper test already asserts that the last error still wins.

## Solution Assessment

No, not as a blanket final-assistant-wins rule. The fix should distinguish recovered intermediate tool failures from genuine fatal run-level, denial, provider, structured delivery, and unrecovered trailing tool errors before the fatal delivery override runs.
