## Summary

Keep open. Current main still lets an all-error isolated cron payload set override recovered `finalAssistantVisibleText`, and the open PR #74840 already tracks the narrow closing fix.

## What This Changes

Cron isolated `agentTurn` summaries should prefer the final self-corrected assistant reply over intermediate tool-error payloads.

## Best Possible Solution

Review and land PR #74840, or an equivalent narrow `resolveCronPayloadOutcome` change with regression coverage that preserves explicit fatal run-level errors.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence helper-level path: call `resolveCronPayloadOutcome` with only error payloads, recovered `finalAssistantVisibleText`, and `preferFinalAssistantVisibleText: true`; current main still selects the error.

## Solution Assessment

Yes on direction. The helper is the narrow maintainable boundary because both isolated cron execution paths call it, but the issue should stay open until PR #74840 or an equivalent fix merges.
