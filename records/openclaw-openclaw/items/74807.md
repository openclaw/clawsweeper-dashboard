## Summary

Keep open. Current main still classifies remaining structured error payloads as fatal before it can use `finalAssistantVisibleText`, and the issue is paired with open unmerged PR #74840 using closing syntax, so this should remain open until that PR or a narrow equivalent lands.

## What This Changes

The issue reports that isolated cron `agentTurn` summaries can announce an intermediate tool error instead of the agent's final self-corrected assistant reply.

## Best Possible Solution

Review and land PR #74840 or a narrow equivalent that lets recovered assistant output drive cron summary and output selection while preserving fatal reporting for run-level errors, denial signals, and genuinely unresolved mutating-tool failures.

## Reproduction Assessment

Yes. The helper boundary reproduces the classifier choosing an error payload over `finalAssistantVisibleText`, and the payload tests show how a success-looking assistant reply can be followed by a mutating tool warning.

## Solution Assessment

Unclear as implemented on main: the current classifier is too broad for the reported recovery case. The maintainable fix should prefer final assistant text for recovered or presentation-only errors while keeping structured fatal errors and unresolved mutating-tool failures fatal.
