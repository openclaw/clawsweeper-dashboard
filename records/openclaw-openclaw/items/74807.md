## Summary

Keep open. Current main still treats all-error isolated cron delivery payloads as fatal even when recovered `finalAssistantVisibleText` is available and preferred, and PR #74840 remains the open closing implementation candidate.

## What This Changes

Cron `agentTurn` summaries should prefer the final assistant-visible self-corrected reply over intermediate tool-error payloads.

## Best Possible Solution

Review and land PR #74840, or a narrow equivalent helper change with regression coverage for recovered final assistant text while preserving fatal-error contracts.

## Reproduction Assessment

Yes. A source-level reproduction is clear by calling `resolveCronPayloadOutcome` with only error payloads, `finalAssistantVisibleText: "Recovered final answer"`, and `preferFinalAssistantVisibleText: true`; current main still asserts the error wins.

## Solution Assessment

Yes on the direction. The narrow maintainable fix belongs in `resolveCronPayloadOutcome`, with focused regression coverage and no broad masking of run-level or structured fatal failures.
