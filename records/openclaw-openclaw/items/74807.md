## Summary

Keep open. Current main and v2026.6.1 still preserve the generic all-error-payload path that can choose the last tool error over terminal assistant-visible text, and the open closing PR is the implementation path rather than a reason to close this issue early.

## What This Changes

Cron isolated agent runs can report a recovered task as failed when transcript payloads end in tool-error payloads even though the final assistant-visible text indicates success.

## Best Possible Solution

Refresh, review, and land https://github.com/openclaw/openclaw/pull/84156 or an equivalent classifier-level fix that marks recovered tool-error payloads as non-fatal while keeping run-level errors and typed fatal signals fatal.

## Reproduction Assessment

Yes, at source level: current main has a focused helper test that passes all-error payloads plus terminal assistant-visible text and expects the last error to win. I did not run tests because this review is required to keep the checkout read-only.

## Solution Assessment

Yes, the durable fix belongs in `resolveCronPayloadOutcome` because both interim retry handling and final cron persistence consume that classifier. A summary-only change would leave `hasFatalErrorPayload` and persisted `status` wrong.
