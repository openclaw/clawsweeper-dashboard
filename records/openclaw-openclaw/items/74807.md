## Summary

Keep open. Current main still has the reported outcome-selection shape: the cron classifier blocks `finalAssistantVisibleText` when only fatal error payloads remain, and the issue is already paired with open PR #74840 using closing syntax, so it should not be closed before that fix or an equivalent lands.

## What This Changes

The issue asks isolated cron `agentTurn` summaries to report the final self-corrected assistant reply instead of an intermediate tool-error payload when the agent recovered successfully.

## Best Possible Solution

Land PR #74840 or a narrow equivalent that lets genuinely recovered final assistant output drive cron summaries while preserving fatal reporting for run-level errors, denial signals, unresolved mutating-tool failures, and structured/media delivery payloads.

## Reproduction Assessment

Yes. The current helper test reproduces the boundary by passing `finalAssistantVisibleText` with only error payloads and asserting the last error wins; the run path then maps that classifier result to a cron error status.

## Solution Assessment

No, not as a blanket rule. The maintainable fix should prefer final assistant text only when the error payload is known to be recovered or presentation-only, while keeping real execution failures and denial markers fatal.
