## Summary

Keep open. Current main still contains the reported behavior: isolated cron result selection refuses to use `finalAssistantVisibleText` when fatal structured error payloads remain, and the helper test still expects a recovered final assistant answer to lose to the last error text. Open PR #74840 is already the linked implementation candidate, so this issue should stay open until that PR or an equivalent focused fix lands.

## What This Changes

The issue requests that isolated cron `agentTurn` summaries prefer the agent's final self-corrected assistant reply over intermediate tool error payloads when the run recovered successfully.

## Best Possible Solution

Land PR #74840 or a narrow equivalent that lets recovered final assistant output drive isolated cron summaries while preserving fatal reporting for true run-level errors, denial markers, and unresolved tool failures.

## Reproduction Assessment

Yes. Source inspection and the existing helper test provide a high-confidence reproduction path: pass recovered `finalAssistantVisibleText` with only fatal error payloads and current main still selects the last error text.

## Solution Assessment

No, the issue's broad rule to always trust the last assistant reply is too blunt. The maintainable fix should prefer final assistant text for recovered or presentation-only failures while preserving explicit run-level failure and denial signals.
