## Summary

Keep open: current main still has cron outcome paths where terminal assistant text can lose to recovered error payloads, and the linked focused fix remains open and unmerged.

## What This Changes

The issue reports that isolated cron `agentTurn` summaries and run status can prefer an intermediate tool error over the agent's final self-corrected assistant reply.

## Best Possible Solution

Consolidate on one narrow cron outcome classifier fix that preserves true fatal run-level signals while treating proven recovered tool errors as diagnostics instead of final run failures.

## Reproduction Assessment

Yes from source, but not from a live cron run in this read-only review. The current helper and tests still show terminal assistant text can be ignored for some all-error payload shapes, and cron finalization persists that helper result as the run status.

## Solution Assessment

Partly. The cron outcome resolver is the right layer, but the best fix should use final assistant text as recovery evidence while preserving fatal run-level errors, failure signals, structured delivery payloads, and genuine unrecovered tool failures.
