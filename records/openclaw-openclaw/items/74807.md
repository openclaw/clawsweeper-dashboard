## Summary

Keep open. Current main still classifies all-error isolated cron payloads as fatal before `finalAssistantVisibleText` can win, and PR #74840 is already the open closing implementation candidate.

## What This Changes

The issue requests isolated cron `agentTurn` summaries to prefer the agent's final assistant reply over intermediate tool-error payloads after self-correction.

## Best Possible Solution

Review and land PR #74840, or an equivalent narrow change in `resolveCronPayloadOutcome` with regression coverage for recovered final assistant text and preserved true fatal failures.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence helper-level reproduction: call `resolveCronPayloadOutcome` with only error payloads, `finalAssistantVisibleText: "Recovered final answer"`, and `preferFinalAssistantVisibleText: true`; the current test asserts the last error wins.

## Solution Assessment

Yes on the direction. The narrow maintainable fix belongs in `resolveCronPayloadOutcome` with focused regression coverage, while preserving run-level failures and structured fatal-error signals.
