## Summary

Current main still has the reported cron outcome-selection failure shape: fatal structured error payloads prevent `finalAssistantVisibleText` from driving the isolated cron result, and a current helper test explicitly expects the recovered assistant text to lose to the last error payload. The issue also has open PR #74840 linked with closing syntax, so it should stay open until that PR or a narrow equivalent lands.

## What This Changes

The issue asks isolated cron `agentTurn` summaries to prefer the agent's final self-corrected assistant reply over an earlier tool-error payload when the run actually recovered.

## Best Possible Solution

Review and land PR #74840 or a narrow equivalent that lets genuinely recovered final assistant output drive cron summaries while preserving fatal reporting for run-level errors, denial signals, and unresolved mutating-tool failures.

## Reproduction Assessment

Yes. The current helper boundary reproduces the classifier choosing the last error payload over `finalAssistantVisibleText`, and existing payload tests show how a success-looking assistant reply can be followed by a mutating tool warning.

## Solution Assessment

No, not as a blanket rule. The safest fix should prefer final assistant text only for recovered/presentation-only cases, while keeping run-level errors, denial markers, and unresolved mutating-tool failures fatal.
