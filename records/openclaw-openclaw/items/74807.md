## Summary

Keep open. Current main still treats all-error isolated cron payloads as fatal before it can select recovered final assistant text, and the linked open closing PR remains the implementation path.

## What This Changes

The issue requests that isolated cron `agentTurn` announcements prefer the final assistant reply over an intermediate tool error after self-correction.

## Best Possible Solution

Update `resolveCronPayloadOutcome` and its focused helper coverage so recovered final assistant text wins for non-fatal all-error tool payloads without masking true cron failures.

## Reproduction Assessment

Yes. A high-confidence source reproduction is `resolveCronPayloadOutcome` with only error payloads, `finalAssistantVisibleText: "Recovered final answer"`, and `preferFinalAssistantVisibleText: true`; the current test asserts the last error wins.

## Solution Assessment

Yes on the direction. The narrow maintainable fix belongs in the outcome helper with regression coverage, while keeping run-level and explicit fatal failure contracts intact.
