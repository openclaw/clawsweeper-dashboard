## Summary

Keep open. Current main still has a source-level reproduction for the reported cron outcome bug, and the issue is paired with open unmerged PR #74840 using closing syntax, so it should stay open until that PR or a narrow equivalent lands.

## What This Changes

Cron `agentTurn` summaries should report the final recovered assistant outcome instead of an intermediate tool-error payload after self-correction.

## Best Possible Solution

Review and land PR #74840 or a narrow equivalent that lets recovered assistant output drive cron summary/output selection while preserving fatal handling for true run-level errors, denial signals, structured media failures, and genuinely unresolved mutating-tool failures.

## Reproduction Assessment

Yes. The helper boundary gives a high-confidence source reproduction: all-error payloads plus `finalAssistantVisibleText` still produce `last error` as output and delivery on current main.

## Solution Assessment

No for current main. The maintainable direction is not a blanket “always trust the last assistant turn”; it should prefer final assistant text for recovered cases while preserving intentional fatal paths.
