## Summary

Keep open. Current main still lets an all-error structured payload set block `finalAssistantVisibleText` from being selected, and the focused helper test still asserts that a recovered final assistant reply loses to the last error payload. The open closing PR #74840 is the right implementation candidate to review rather than closing this issue before merge.

## What This Changes

The issue asks isolated cron `agentTurn` announcements to prefer the final recovered assistant-visible reply over intermediate tool-error payloads when the run completes successfully.

## Best Possible Solution

Land PR #74840 or a narrow equivalent that lets recovered final assistant text win over intermediate tool errors while preserving fatal run-level errors, explicit denial/failure signals, and unresolved error payload reporting.

## Reproduction Assessment

Yes. A source-level reproduction is clear: call `resolveCronPayloadOutcome` with `preferFinalAssistantVisibleText: true`, `finalAssistantVisibleText: "Recovered final answer"`, and only error payloads; the existing helper test asserts the last error is selected.

## Solution Assessment

Unclear as broadly phrased in the issue, because always trusting the final assistant message could hide true fatal runs. The maintainable fix should narrowly recognize recovered tool-error cases while keeping the documented fatal-run behavior intact.
