## Summary

Keep open. Current main still routes an all-error cron payload set to the last error text even when `finalAssistantVisibleText` contains a recovered final answer, and an open closing PR already targets this issue.

## What This Changes

The issue asks isolated cron `agentTurn` summaries to prefer a recovered final assistant reply over intermediate tool-error payloads.

## Best Possible Solution

Review and land PR #74840, or a narrow equivalent, so recovered final assistant output wins for this cron summary case while preserving fatal reporting for run-level failures, denials, and unresolved errors.

## Reproduction Assessment

Yes. A source-level reproduction is clear: call `resolveCronPayloadOutcome` with `preferFinalAssistantVisibleText: true`, `finalAssistantVisibleText: "Recovered final answer"`, and only structured error payloads; the current helper test asserts the last error still wins.

## Solution Assessment

Unclear as phrased in the issue because always trusting the last assistant reply could hide real fatal runs. The maintainable fix is a narrow classifier change that prefers final assistant text only for recovered tool-error summaries while keeping explicit fatal signals authoritative.
