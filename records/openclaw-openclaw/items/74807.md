## Summary

Keep open. Current main still treats an all-error isolated cron payload set as fatal before it can prefer recovered `finalAssistantVisibleText`, and PR #74840 is already the open closing implementation candidate.

## What This Changes

The issue asks isolated cron `agentTurn` announcements to prefer the final self-corrected assistant reply instead of an earlier tool-error payload.

## Best Possible Solution

Review and land PR #74840, or a narrow equivalent change in `resolveCronPayloadOutcome` with regression coverage that preserves explicit fatal run-level errors.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence helper-level reproduction by calling `resolveCronPayloadOutcome` with only error payloads, `finalAssistantVisibleText: "Recovered final answer"`, and `preferFinalAssistantVisibleText: true`; the current helper test asserts the error wins.

## Solution Assessment

Yes on the direction. The narrow maintainable fix belongs in `resolveCronPayloadOutcome` with focused regression coverage, while preserving run-level failures and structured fatal-error signals.
