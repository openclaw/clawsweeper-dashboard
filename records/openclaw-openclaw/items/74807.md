## Summary

Keep open. Current main still lets all-error cron delivery payloads become fatal before recovered `finalAssistantVisibleText` can be selected, and PR #74840 is already the open closing implementation candidate.

## What This Changes

The issue asks isolated cron `agentTurn` summaries to prefer the recovered final assistant reply over intermediate tool-error output.

## Best Possible Solution

Review and land PR #74840, or a narrow equivalent change in `resolveCronPayloadOutcome` with regression coverage that preserves explicit fatal-error contracts.

## Solution Assessment

Yes on the direction. The narrow maintainable fix belongs in `resolveCronPayloadOutcome` with focused regression coverage, while preserving run-level failures and structured fatal-error signals.
