## Summary

Keep open. Current main still lets an all-error isolated cron payload set override recovered `finalAssistantVisibleText`, and the open closing PR #74840 remains the right implementation candidate to review before this issue closes.

## What This Changes

Cron isolated `agentTurn` summaries should prefer the final assistant reply over an intermediate recovered tool-error payload.

## Best Possible Solution

Review and land PR #74840, or a narrow equivalent change in `resolveCronPayloadOutcome` with regression coverage that still preserves explicit fatal run-level errors.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence helper-level reproduction by calling `resolveCronPayloadOutcome` with only error payloads, `finalAssistantVisibleText: "Recovered final answer"`, and `preferFinalAssistantVisibleText: true`; the current test asserts the error wins.

## Solution Assessment

Yes on the direction. The narrow maintainable fix belongs in `resolveCronPayloadOutcome` with focused regression coverage, while preserving run-level failures and structured fatal-error signals.
