## Summary

Keep open. Current main still treats an all-error isolated cron payload set as fatal before recovered `finalAssistantVisibleText` can be selected, and PR #74840 is already the open closing implementation candidate.

## What This Changes

Cron isolated `agentTurn` summaries should prefer the final assistant-visible reply over self-corrected intermediate tool-error payloads.

## Best Possible Solution

Review and land PR #74840, or an equivalent narrow change in `resolveCronPayloadOutcome` with regression coverage that keeps explicit run-level failures fatal.

## Reproduction Assessment

Yes. A helper-level call to `resolveCronPayloadOutcome` with only error payloads, `finalAssistantVisibleText: "Recovered final answer"`, and `preferFinalAssistantVisibleText: true` reproduces from source; the current test asserts the error wins.

## Solution Assessment

Yes on the direction. The narrow maintainable fix belongs in `resolveCronPayloadOutcome` with focused regression coverage, but this issue should stay open until the linked closing PR merges.
