## What This Changes

This PR adds a bounded `gateway.chatHistoryTextMaxChars` config option, wires WebChat/Control UI history projection to read it, and updates schema/help/docs/tests.

## Best Possible Solution

Keep bounded `chat.history` plus `chat.message.get` as the supported WebChat contract unless maintainers explicitly sponsor a new gateway-level history-cap config key with upgrade and browser reload proof.

## Reproduction Assessment

Yes, source-level. Current main ignores `cfg` in `resolveEffectiveChatHistoryMaxChars`, the HTTP history endpoint uses the default cap, and Control UI history requests do not send an RPC `maxChars` override.

## Solution Assessment

No as a merge decision today. The implementation is a plausible bounded config solution, but the current product contract is bounded history plus `chat.message.get`, so a new persisted gateway setting needs maintainer sponsorship first.
