## What This Changes

The PR adds `gateway.chatHistoryTextMaxChars` to Gateway config, schema/help/labels/docs/tests, and wires WebSocket plus HTTP session history projection to use it when no RPC `maxChars` override is supplied.

## Best Possible Solution

If maintainers sponsor this config knob, land it as a bounded Gateway config contract aligned with the existing history cap, update all WebChat/config docs, and require redacted WebChat reload proof; otherwise keep the current bounded-history plus full-message-reader design.
