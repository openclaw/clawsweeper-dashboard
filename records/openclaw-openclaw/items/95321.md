## Summary

Keep open: the patch is focused and the generated config baseline is now present, but it adds a new public Gateway config key and permits larger WebChat history payloads, so maintainer acceptance of that product contract is still required before merge.

## What This Changes

The branch adds `gateway.chatHistoryTextMaxChars` to Gateway config schema/types/help/docs/tests and uses it for chat history projection when no RPC `maxChars` override is supplied.

## Best Possible Solution

A maintainer should either sponsor `gateway.chatHistoryTextMaxChars` and its bound as the supported contract, or decline the knob in favor of bounded history plus the existing full-message reader.

## Reproduction Assessment

Yes, at source level. Current main hardcodes the 8,000-character resolver/default and documents no persisted WebChat config key; I did not run a live browser reload repro in this read-only review.

## Solution Assessment

Unclear as a product decision. The resolver-level implementation is a reasonable place if maintainers accept a global Gateway config key, but the existing `chat.message.get` full-message reader is the lower-payload alternative.
