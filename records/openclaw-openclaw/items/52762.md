## Summary

Keep open: the current main branch and latest release still have the eager prune path, but this PR is not merge-ready because it leaves the saturated-cache scan and changes shared map-limit semantics.

## What This Changes

The PR changes `createDedupeCache().check()` to prune only after the cache exceeds `maxSize` and rewrites `pruneMapToMaxSize()` to delete old entries with a counted forward pass.

## Best Possible Solution

Land or replace this with a focused shared-helper fix that removes per-message full scans in both normal and saturated cache states, preserves invalid numeric limit handling, keeps public SDK helper semantics stable, and proves the helper plus Discord replay path with targeted tests and runtime evidence.

Yes for the PR defects: source inspection of the proposed guard and counted loop shows both the saturated-cache full-prune path and the invalid-limit semantic change. I did not establish a live Discord replay of the original 33-122 second delay.

## Solution Assessment

No. The direction is useful, but this patch is not the best fix because it leaves the expensive saturated-cache path and changes shared helper behavior for invalid numeric limits.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Avoid full-cache pruning once saturated:** `src/infra/dedupe.ts:57-59`
  - body: `check()` still calls the full `prune(now)` path whenever a new key pushes the map over `maxSize`. With Discord's five-minute, 5000-entry replay guard, a busy server can keep the cache saturated, so each new unique delivery still scans roughly the whole map and the targeted delay can continue.
  - confidence: 0.91
- **[P2] Preserve invalid-limit pruning behavior:** `src/infra/map-size.ts:10-14`
  - body: When `maxSize` is `NaN`, `deleteCount` becomes `NaN` and `i++ >= deleteCount` never becomes true, so the loop deletes every key. Current behavior is a no-op for `NaN`, and webhook guard callers can normalize malformed numeric config to `NaN` before pruning.
  - confidence: 0.88
