## Summary

Keep open: the PR targets a real Discord inbound dedupe hot path, but the branch is not merge-ready because it still performs full-cache scans after saturation and changes shared map-pruning behavior for invalid numeric limits.

## What This Changes

The branch changes `createDedupeCache().check()` to prune only when the cache exceeds `maxSize` and rewrites `pruneMapToMaxSize()` to delete old entries with a counted forward pass.

## Best Possible Solution

Revise or replace this with a bounded pruning strategy that avoids saturated-cache full scans, preserves or explicitly validates invalid numeric limit handling, adds focused helper and Discord replay tests, and includes the required changelog entry.

## Reproduction Assessment

Yes. Source inspection plus read-only Node simulations reproduce the PR defects: the proposed size-gated prune still scans millions of entries after a 5000-entry cache saturates, and the map-size rewrite deletes all entries for `NaN`.

## Solution Assessment

No. The direction is useful, but this patch is not the best fix because it delays the O(n) scan until saturation and changes shared SDK helper semantics instead of making pruning bounded and stable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Avoid full-cache prune after saturation:** `src/infra/dedupe.ts:57-59`
  - body: `check()` still calls the full `prune(now)` path whenever `cache.size > maxSize`. With Discord's five-minute, 5000-entry replay guard, a saturated unique-message stream still scans about the whole cache on every new key, so the targeted delay can continue on busy servers.
  - confidence: 0.91
- **[P2] Preserve invalid-limit behavior in map pruning:** `src/infra/map-size.ts:10-14`
  - body: When `maxSize` is `NaN`, `deleteCount` becomes `NaN` and this loop never breaks, deleting every entry. Current behavior is effectively a no-op for `NaN`, and SDK guard callers can pass `NaN` after numeric normalization, so active guard state can be cleared.
  - confidence: 0.88
