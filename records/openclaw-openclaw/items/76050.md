## Summary

Keep open because current main still sends the Control UI Sessions tab's initial `sessions.list` request without an `activeMinutes` bound, and the linked fix PR #76051 is open and unmerged with closing syntax for this issue.

## What This Changes

Request to bound the Control UI Sessions tab default query to recent sessions and lower the default limit while keeping the filters editable.

## Best Possible Solution

Review and land #76051 or an equivalent small UI-only default change, leaving the existing Gateway API and editable filters intact.

## Reproduction Assessment

Yes for the query shape: current main initializes an empty active-window filter and constructs `sessions.list` without `activeMinutes`. A full latency reproduction requires a long-running session store, but the source path and reporter measurements support the reported precondition.

## Solution Assessment

Yes, the linked PR direction is the narrowest maintainable fix because the Gateway already supports `activeMinutes` and `limit`; changing the Control UI defaults avoids broader API or storage changes.
