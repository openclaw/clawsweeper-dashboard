## Summary

Keep open: current main still shows only scoped cleanup for OpenClaw-owned MCP runtime handles, while this report describes downstream `claude mcp serve` orphans plus a broader cross-scope Hermes reuse request that needs maintainer design.

## What This Changes

The issue requests cleanup of orphaned Claude MCP child processes after session end and reuse of an existing local Hermes MCP daemon instead of spawning a full child fleet per invocation.

## Best Possible Solution

Fix owned subprocess cleanup through a generic ownership marker and conservative orphan-reconciliation design, and handle Hermes reuse through a generic opt-in MCP daemon or HTTP reuse contract rather than hard-coding Hermes probing in core.
