## Summary

Keep open: current main has stronger owned bundle-MCP and Claude live-session cleanup than the originally reported release, but it still does not prove cleanup of downstream `claude mcp serve` grandchildren or implement a generic Hermes daemon reuse/isolation contract.

## What This Changes

The issue asks OpenClaw to clean up orphaned CLI-launched MCP server processes after sessions end and avoid respawning a full Hermes MCP child fleet for every Hermes-as-tool invocation.

## Best Possible Solution

Split the work: add conservative generic cleanup for OpenClaw-owned subprocess trees and design Hermes reuse through a plugin/provider-owned MCP daemon or HTTP reuse contract rather than hard-coded Hermes process probing in core.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection gives a plausible path: cleanup owns known MCP transports/runtimes, but current main has no startup orphan sweep and the Claude config-file handoff only owns a temp config file.

## Solution Assessment

Unclear as a single fix. Owned process cleanup is a narrow lifecycle repair candidate, while Hermes cross-scope daemon sharing needs a generic opt-in contract with isolation semantics.
