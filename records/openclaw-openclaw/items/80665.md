## What This Changes

The issue requests cleanup for orphaned `claude mcp serve` descendants after session end and reduced duplicate Hermes MCP fleet startup when OpenClaw and Hermes expose each other as MCP tools.

## Best Possible Solution

Add validated process-group/grandchild cleanup for OpenClaw-owned stdio MCP transports, and handle Hermes daemon reuse through explicit HTTP/SSE MCP configuration or a generic opt-in reuse design.

## Reproduction Assessment

No live WSL2/Hermes reproduction was established. Source inspection gives a credible current-main path for the orphan class: an MCP wrapper can exit cleanly while leaving grandchildren, and OpenClaw then skips process-tree cleanup.

## Solution Assessment

Unclear as a single fix. A focused stdio process-group cleanup is likely the narrow bug fix, while automatic Hermes daemon reuse needs maintainer product judgment and should avoid hard-coded service probing.
