## What This Changes

The PR adds a replace-mode guard around implicit model-provider discovery and a unit test file covering replace, merge, unset, and planning behavior.

## Best Possible Solution

Land the callsite guard with the default resolver result safely coalesced, preserve merge/default behavior, add focused regression coverage and a single-line Unreleased/Fixes changelog entry, and leave pricing bootstrap behavior to #74020.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by calling `resolveProvidersForModelsJsonWithDeps` or `planOpenClawModelsJsonWithDeps` on current main with `models.mode: "replace"` and an injected implicit resolver; the resolver is still reached before mode handling.

## Solution Assessment

Yes in direction, but not as submitted. Guarding before `resolveImplicitProviders()` is the narrow maintainable fix, but the patch must normalize the optional default resolver result and add the required changelog entry before merge.
