## Summary

Keep this PR open. Current main still runs implicit provider discovery before mode handling despite the documented `models.mode="replace"` contract, and the PR is the right narrow direction, but the exact head is not merge-ready because the new assignment can receive an optional default resolver result and the user-facing fix lacks the required changelog entry.

## What This Changes

The PR computes `models.mode` before provider discovery, skips implicit provider discovery in replace mode, and adds unit coverage for replace, merge, default, and planning behavior.

## Best Possible Solution

Land a corrected callsite guard in `resolveProvidersForModelsJsonWithDeps` that skips implicit discovery before it starts when `models.mode` is `replace`, preserves merge/default behavior and provider normalization, adds the required changelog credit, and leaves pricing bootstrap behavior to #74020.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure `models.mode: "replace"` with an injected or observable implicit resolver on current main, and the resolver is still called before mode handling; #66957 also provides concrete startup-delay logs.

## Solution Assessment

Yes in direction, but not as submitted. The resolver-callsite guard is the narrowest maintainable fix, but the implementation should normalize the optional default resolver result and add the missing changelog entry before merge.
