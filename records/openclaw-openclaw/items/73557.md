## Summary

Keep this PR open. The linked bug is still present on current main and the PR takes the right narrow approach, but the submitted head is not merge-ready because the new assignment can receive an optional provider map and the required user-facing changelog entry is missing.

## What This Changes

The PR computes `models.mode` before provider discovery, skips `resolveImplicitProviders` for replace mode, and adds unit coverage for replace, merge, default, and planning behavior.

## Best Possible Solution

Land a corrected callsite guard in `resolveProvidersForModelsJsonWithDeps` that skips implicit discovery before it starts when `models.mode` is `replace`, preserves merge/default behavior and provider normalization, adds the required changelog credit, and leaves pricing bootstrap behavior to #74020.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: call `resolveProvidersForModelsJsonWithDeps` or `planOpenClawModelsJsonWithDeps` with `models.mode: "replace"` and an injected implicit resolver on current main; the resolver is still reached before mode handling.

## Solution Assessment

Yes in direction, but no as submitted. The resolver-callsite guard is the narrowest maintainable fix, but the implementation should coalesce the optional implicit-provider result and include the required changelog entry before merge.
