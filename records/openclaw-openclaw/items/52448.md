## Summary

Keep open, but do not merge as-is: the requested sandbox authority hydration is not implemented on current main, and the PR still changes a filesystem trust boundary with unresolved security blockers around resolved symlink targets and hardlinked files.

## What This Changes

The branch adds sandbox workspace hydration for allowlisted top-level authority symlinks such as `README.md`, `CLAUDE.md`, `docs`, and `voro-docs`, plus unit tests for materialization and nested symlink skipping.

## Best Possible Solution

Define a documented authority-link model before merging: supported names, resolved-target root policy, symlink and hardlink rejection throughout copied trees, refresh behavior for reused sandboxes, Windows-safe tests, and docs for hydrated authority files.

## Reproduction Assessment

Yes. At PR head, point an allowlisted `README.md` or `docs` symlink at an outside path, or put a same-filesystem hardlink inside an allowed authority tree, then call `ensureSandboxWorkspace`; the copied sandbox will contain content current main's boundary seed path rejects.

## Solution Assessment

No. Copying resolved symlink targets with a custom recursive copier is not the safest fix for this sandbox boundary; the safer direction is to reuse boundary validation or add an explicit maintainer-approved authority-root policy with hardlink checks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Validate authority links before copying:** `src/agents/sandbox/workspace.ts:86-91`
  - body: `fs.realpath(src)` resolves an allowlisted name and then copies that resolved path without checking it is inside the seed workspace or another trusted authority root. A workspace can point `README.md` or `docs` at private host content and have it materialized into a non-rw sandbox.
  - confidence: 0.93
- **[P1] Reject hardlinks while copying authority trees:** `src/agents/sandbox/workspace.ts:56-57`
  - body: `lstat()` plus `copyFile()` treats hardlinked files as ordinary files. That bypasses the current bootstrap seed protection, which rejects hardlink aliases through the boundary-file safe-open path.
  - confidence: 0.9
- **[P2] Refresh existing authority copies:** `src/agents/sandbox/workspace.ts:40-42`
  - body: The copy helper returns as soon as the destination exists. Agent-scoped sandboxes are reused, so later changes to authority docs or edits left inside the sandbox are not refreshed for later sessions.
  - confidence: 0.86
- **[P2] Guard symlink tests on Windows:** `src/agents/sandbox.workspace.test.ts:25-34`
  - body: The new tests create file and directory symlinks unconditionally, while existing sandbox symlink and hardlink tests are skipped on Windows because those runners may lack symlink privileges.
  - confidence: 0.88
