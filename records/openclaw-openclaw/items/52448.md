## Summary

Keep open: current main has not implemented authority-link hydration, and this PR still needs sandbox trust-boundary/security review plus contributor real behavior proof before merge.

## What This Changes

The PR adds sandbox workspace hydration for allowlisted top-level authority symlinks and unit tests for materialization, allowlisting, and nested symlink skipping.

## Best Possible Solution

Define the authority-link model first, then implement it with explicit supported names, target-root validation, symlink and hardlink rejection throughout copied trees, deliberate refresh semantics, focused tests, and matching docs.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the review findings: source inspection of the PR head and merge ref shows the realpath copy path, hardlink copy path, and existing-destination early return; the original request is a new capability rather than a current-main bug reproduction.

## Solution Assessment

Is this the best way to solve the issue? No. Hydrating authority links may be useful, but the patch copies resolved targets before the sandbox trust root, alias rejection, and refresh contract are defined and enforced.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Validate authority targets before copying:** `src/agents/sandbox/workspace.ts:86-91`
  - body: `fs.realpath(src)` resolves an allowlisted name and then copies that resolved target without checking it is inside the seed workspace or another explicitly trusted root. A workspace can point `README.md` or `docs` at private host content and have it materialized into a non-rw sandbox, bypassing the current seed contract.
  - confidence: 0.93
- **[P1] Reject hardlinks while copying authority trees:** `src/agents/sandbox/workspace.ts:56-57`
  - body: `lstat()` plus `copyFile()` treats hardlinked files as ordinary files. Current bootstrap seeding rejects hardlink aliases through the boundary-file path, but this new copier would copy a same-filesystem hardlink inside an allowed authority tree into the sandbox as readable content.
  - confidence: 0.9
- **[P2] Refresh existing authority copies:** `src/agents/sandbox/workspace.ts:40-42`
  - body: `copyAuthorityTreeIfMissing` returns as soon as the destination exists. Agent-scoped sandboxes are reused, so later changes to authority docs or previous-session edits in copied files can leave later sessions reading stale authority content.
  - confidence: 0.86
