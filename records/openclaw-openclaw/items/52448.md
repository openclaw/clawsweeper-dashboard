## Summary

Keep open: the PR addresses a real sandbox authority-context gap, but the latest head still changes the sandbox trust boundary with unresolved source-level security findings and lacks real behavior proof from a real sandbox run.

## What This Changes

The PR adds sandbox workspace hydration for allowlisted top-level authority symlinks such as `README.md`, `CLAUDE.md`, `docs`, and `voro-docs`, plus unit coverage for materialization and nested symlink skipping.

## Best Possible Solution

Define a documented authority-link model first, then implement it with resolved-target validation, symlink and hardlink rejection throughout copied trees, deliberate refresh semantics, focused tests, and matching sandbox docs.

## Reproduction Assessment

Yes for the review findings from source inspection: a seed workspace with an allowlisted symlink resolving outside the seed, or an allowed authority tree containing a hardlinked file, would be copied by the PR-head helper. I did not run a local repro because this review is required to keep the checkout read-only.

## Solution Assessment

No. The current PR direction is useful, but copying allowlisted symlink targets without a documented trust root and hardlink validation is not the narrowest maintainable sandbox-boundary solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Validate authority links before copying:** `src/agents/sandbox/workspace.ts:86-91`
  - body: `fs.realpath(src)` resolves an allowlisted name and then copies the resolved path without checking that it stays inside the seed workspace or another documented trusted root. A workspace can point `README.md` or `docs` at private host content and have it materialized into a `workspaceAccess: "none"`/`"ro"` sandbox.
  - confidence: 0.93
- **[P1] Reject hardlinks while copying authority trees:** `src/agents/sandbox/workspace.ts:56-57`
  - body: `lstat()` plus `copyFile()` treats hardlinked files as ordinary files. Current bootstrap seeding rejects hardlink aliases, but this new path would copy a same-filesystem hardlink inside an allowed authority tree into the sandbox as readable content.
  - confidence: 0.9
- **[P2] Refresh existing authority copies:** `src/agents/sandbox/workspace.ts:40-42`
  - body: `copyAuthorityTreeIfMissing` returns as soon as the destination exists. Agent-scoped sandboxes are reused, so later source authority edits or copied-file modifications can leave subsequent sessions reading stale authority content.
  - confidence: 0.84
