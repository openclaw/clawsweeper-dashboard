## Summary

Keep open: the feature is useful, but the current PR still changes a sandbox filesystem boundary with unresolved security issues, stale current-main integration, and no real behavior proof beyond tests/CI.

## What This Changes

The PR adds sandbox hydration for allowlisted top-level symlinked authority files/directories and unit tests around materialization, allowlisting, and nested symlink skipping.

## Best Possible Solution

Define a documented authority-link trust model first, then implement hydration through the current fs-safe/root-bounded path with hardlink rejection, clear refresh semantics for reused sandboxes, preserved current bootstrap options, and focused docs/tests.

## Reproduction Assessment

Yes, source-reproducible for the review findings. At PR head, an allowlisted `README.md` or `docs` symlink can resolve outside the seed workspace, and a hardlinked file inside an allowed authority tree is copied as readable sandbox content.

## Solution Assessment

No. The PR is not the best way to solve this yet because it adds a custom realpath-copy path around the existing sandbox boundary checks; the safer path is a maintainer-approved authority-root policy implemented with the current fs-safe helpers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Validate authority targets before copying:** `src/agents/sandbox/workspace.ts:86-91`
  - body: `fs.realpath(src)` resolves an allowlisted workspace name and then copies that target without checking it stays inside the seed workspace or another explicitly trusted root. In non-rw sandbox modes, a workspace entry like `README.md` or `docs` can point at a private host path and have that content materialized under `/workspace`, bypassing the current seed contract.
  - confidence: 0.93
- **[P1] Reject hardlinked authority files:** `src/agents/sandbox/workspace.ts:56-57`
  - body: The new tree copier skips symlinks, but `lstat()` plus `copyFile()` still treats hardlinks as ordinary files. That reintroduces the alias escape the existing bootstrap seeding path rejects, so a hardlinked file inside an allowed authority tree can be copied into the sandbox as readable content.
  - confidence: 0.9
- **[P2] Refresh copied authority content:** `src/agents/sandbox/workspace.ts:40-42`
  - body: Returning as soon as the destination exists leaves reused agent-scoped sandboxes with stale or self-modified authority docs after the first run. Later edits to the source `README.md` or `docs` target will not be reflected in subsequent sandbox sessions.
  - confidence: 0.86
