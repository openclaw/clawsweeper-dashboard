## Summary

Keep open: current main still does not implement authority-link hydration, and this PR still changes a sandbox trust boundary with unresolved P1 security findings plus missing real behavior proof.

## What This Changes

The PR adds sandbox hydration of allowlisted top-level authority symlinks such as README.md, CLAUDE.md, docs, and voro-docs into managed sandbox workspaces, with unit tests for materialization and nested symlink skipping.

## Best Possible Solution

Define a documented authority-link model first, then implement it with resolved-target validation, hardlink rejection, deliberate refresh semantics for reused sandboxes, focused tests, and matching sandbox docs.

## Reproduction Assessment

Yes for the review findings: a seed workspace with README.md pointing outside the seed, or an allowed authority tree containing a hardlinked file, follows the PR-head copy path from source inspection. I did not execute a local repro because this review is read-only.

## Solution Assessment

No. Hydrating explicit authority links may be useful, but copying resolved targets before defining and enforcing the sandbox trust boundary is not the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Validate authority targets before copying:** `src/agents/sandbox/workspace.ts:86-91`
  - body: `fs.realpath(src)` resolves an allowlisted symlink and then copies that resolved path without checking it stays under the seed workspace or another explicitly trusted root. A workspace can point `README.md` or `docs` at private host content and expose it in a non-rw sandbox.
  - confidence: 0.93
- **[P1] Reject hardlinks while copying authority trees:** `src/agents/sandbox/workspace.ts:56-57`
  - body: `lstat()` plus `copyFile()` treats hardlinked files as ordinary files. A hardlink inside an allowed authority tree can alias host content and be materialized into `/workspace`, unlike the existing bootstrap path that rejects hardlink aliases.
  - confidence: 0.9
- **[P2] Refresh reused authority copies:** `src/agents/sandbox/workspace.ts:40-42`
  - body: Returning when `dest` already exists means reused agent-scoped sandboxes can keep stale or previous-session-modified authority files after the source docs change. Either refresh deliberately or document/recreate the seed semantics for these authority copies.
  - confidence: 0.84
