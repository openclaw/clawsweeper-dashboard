The PR adds allowlisted top-level symlink hydration for `README.md`, `CLAUDE.md`, `docs`, and `voro-docs` into sandbox workspaces, plus unit tests for materialization and exclusions.

## Best Possible Solution

Define the sandbox authority-link contract first, then implement target-root validation, hardlink and symlink rejection through copied trees, deliberate refresh semantics for reused sandboxes, docs, focused tests, and redacted real sandbox proof; if the need is only fixed bootstrap files, defer to the narrower scoped bootstrap path.

Yes for the review findings: source inspection of the PR head shows the unvalidated realpath copy, hardlink-copy path, and existing-destination early return. The original request is a new sandbox capability rather than a current-main bug reproduction.

## Solution Assessment

No, not as-is. Hydrating authority links may be useful, but copying external host content into a sandbox needs a documented trust model and enforcement before merge.
