## Summary

Keep open. This is a plausible sandbox feature PR, but current head still needs maintainer security/product review, real behavior proof, and fixes for sandbox boundary regressions before merge; current main has not implemented authority-link hydration.

## What This Changes

The PR adds sandbox workspace hydration for allowlisted top-level authority symlinks such as `README.md`, `CLAUDE.md`, `docs`, and `voro-docs`, plus unit tests for materialization, allowlisting, and nested symlink skipping.

## Best Possible Solution

Define and document a sandbox authority-link model before merging: supported names, trusted target policy, symlink and hardlink rejection, refresh behavior for reused sandboxes, and regression coverage against current sandbox config.

## Reproduction Assessment

Not applicable as a current-main bug; the requested behavior is a new sandbox feature. The blocking review issues are source-proven from the PR head and current main sandbox seed contract.

## Solution Assessment

No. Materializing authority links may be useful, but this implementation is not the best solution until maintainers decide the target trust model and the patch preserves the current sandbox boundary/config contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the current boundary-file reader API:** `src/agents/sandbox/workspace.ts:4`
  - body: Current main no longer exports or uses `openBoundaryFile`; this stale head imports it from `boundary-file-read.ts`, whose current public export is `openRootFile`, so the sandbox workspace module will not typecheck after rebasing or merging against current main.
  - confidence: 0.9
- **[P1] Validate authority links before copying:** `src/agents/sandbox/workspace.ts:86-91`
  - body: `fs.realpath(src)` resolves an allowlisted name and then copies that resolved path without checking it is inside the seed workspace or another explicit trusted root. A workspace can point `README.md` or `docs` at a credential directory or sibling checkout and have that content materialized into a non-rw sandbox.
  - confidence: 0.93
- **[P1] Reject hardlinks while copying authority trees:** `src/agents/sandbox/workspace.ts:56-57`
  - body: `lstat()` plus `copyFile()` treats hardlinked files as ordinary files. Current bootstrap seeding rejects hardlink aliases through the boundary-file reader, but this path would copy a same-filesystem hardlink inside an allowed authority tree into the sandbox as readable content.
  - confidence: 0.9
- **[P2] Refresh authority copies for reused sandboxes:** `src/agents/sandbox/workspace.ts:40-42`
  - body: The copy helper returns as soon as the destination exists. Agent-scoped sandboxes are reused by default, so later changes to `README.md` or `docs` symlink targets, or edits left in copied sandbox files, will not be refreshed for later sessions.
  - confidence: 0.86
- **[P2] Preserve optional bootstrap skip config:** `src/agents/sandbox/workspace.ts:133-136`
  - body: Current main forwards `agents.defaults.skipOptionalBootstrapFiles` through sandbox workspace setup, but this head drops the parameter and calls `ensureAgentWorkspace` without it. Sandboxed sessions would recreate optional bootstrap files that users explicitly configured OpenClaw to skip.
  - confidence: 0.88
