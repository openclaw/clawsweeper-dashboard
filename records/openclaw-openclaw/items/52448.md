## Summary

Keep open: this is a plausible sandbox feature, but the current PR head still changes host-content exposure without a settled trust model, leaves concrete security findings unresolved, and lacks after-fix real behavior proof.

## What This Changes

The PR adds allowlisted materialization of top-level authority symlinks such as README.md, CLAUDE.md, docs, and voro-docs into sandbox workspaces, plus unit tests for the new copy behavior.

## Best Possible Solution

Define and document the authority-link contract first, then implement target-root validation, symlink and hardlink rejection throughout copied trees, deliberate refresh semantics, focused tests, and real sandbox proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the review findings: source inspection of the PR merge result shows the realpath copy path, hardlink copy path, and existing-destination early return; the original request is a new capability rather than a current-main bug reproduction.

## Solution Assessment

Is this the best way to solve the issue? No. Hydrating authority links may be useful, but this patch copies resolved targets before the sandbox authority trust root, alias rejection policy, and refresh contract are defined and enforced.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Validate authority targets before copying:** `src/agents/sandbox/workspace.ts:87-92`
  - body: `fs.realpath(src)` resolves an allowlisted workspace symlink and then copies that resolved path without checking it is inside the seed workspace or another documented trusted root. A workspace can point `README.md` or `docs` at private host content and have it materialized into a non-`rw` sandbox.
  - confidence: 0.93
- **[P1] Reject hardlinks while copying authority trees:** `src/agents/sandbox/workspace.ts:57-58`
  - body: `lstat()` plus `copyFile()` treats hardlinked files as ordinary files. Current bootstrap seeding rejects hardlink aliases through the boundary-file path, but this path would copy a same-filesystem hardlink inside an allowed authority tree into the sandbox as readable content.
  - confidence: 0.9
- **[P2] Refresh existing authority copies:** `src/agents/sandbox/workspace.ts:41-43`
  - body: `copyAuthorityTreeIfMissing` returns as soon as the destination exists. Agent-scoped sandboxes are reused, so later source changes or previous-session edits in copied authority files can leave later sessions reading stale authority content.
  - confidence: 0.86
