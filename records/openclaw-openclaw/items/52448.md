## Summary

Keep open. The PR addresses a real sandbox context gap and the author already fixed the broad top-level symlink and nested-symlink issues, but the current head still has sandbox boundary bugs, drops current-main optional bootstrap behavior, and lacks after-fix real behavior proof.

## What This Changes

The PR adds sandbox workspace hydration for allowlisted top-level authority symlinks such as README.md, CLAUDE.md, docs, and voro-docs, plus unit tests for materialization, allowlisting, and nested symlink skipping.

## Best Possible Solution

Define and document the sandbox authority-link contract, then update this PR or a replacement to validate resolved targets with the existing fs-safe boundary model, reject hardlink aliases, preserve current bootstrap config behavior, choose deliberate refresh semantics, and provide real sandbox proof.

## Reproduction Assessment

Yes for the review findings by source inspection: the PR head resolves allowlisted symlinks and copies the resolved target without root validation, and its file-copy path still accepts hardlinks. I did not run tests because this was a read-only review.

## Solution Assessment

No. The current PR shape is not the safest maintainable solution until it reuses or matches the existing boundary-file validation contract, handles hardlinks and current config propagation, and documents the authority-link policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Validate authority links before copying:** `src/agents/sandbox/workspace.ts:86-91`
  - body: `fs.realpath(src)` resolves an allowlisted name and then copies that resolved path without checking it stays inside the seed workspace or another explicitly trusted root. In `workspaceAccess: "none"` or `"ro"`, `README.md -> ~/.ssh/config` or `docs -> ../private-repo` can still be materialized into `/workspace`, bypassing the current seed contract.
  - confidence: 0.94
- **[P1] Reject hardlinks while copying authority trees:** `src/agents/sandbox/workspace.ts:56-57`
  - body: `lstat()` skips symlinks, but `stat.isFile()` plus `copyFile()` still accepts hardlinked files as ordinary files. That reintroduces the alias escape the bootstrap path avoids with boundary-file validation, so an allowed authority tree can copy a same-filesystem hardlink to sensitive host content into the sandbox.
  - confidence: 0.91
- **[P2] Preserve optional bootstrap skip settings:** `src/agents/sandbox/workspace.ts:133-136`
  - body: Current main passes `skipOptionalBootstrapFiles` through `ensureSandboxWorkspace` into `ensureAgentWorkspace`, but the PR head's signature and final call omit it. With non-rw sandbox workspaces, configs that intentionally skip optional files such as `USER.md` or `IDENTITY.md` would have those files recreated inside the sandbox.
  - confidence: 0.88
- **[P2] Refresh or define authority copy staleness:** `src/agents/sandbox/workspace.ts:40-42`
  - body: `copyAuthorityTreeIfMissing` returns as soon as the destination exists, while agent-scoped sandboxes are reused. After the first run, changes to the linked README/docs targets, or edits left in the copied sandbox files, are not refreshed, so later sessions can read stale authority content unless this one-shot behavior is explicitly part of the contract.
  - confidence: 0.82
