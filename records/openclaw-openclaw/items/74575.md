## Summary

Keep open. Current main has classic onboarding, section-based configure, doctor/health checks, and Crestodian setup/repair, but it still does not provide the requested goal-driven, host-aware configuration recommender; the remaining work needs maintainer product and security design.

## What This Changes

The issue asks for a built-in assistant or wizard that probes the host, interviews users about goals, and recommends or updates OpenClaw configuration over time.

## Best Possible Solution

Design a scoped setup recommendation flow that builds on onboarding, configure, plugin metadata, and Crestodian typed operations while making host discovery opt-in and every persistent write explicit, audited, and reversible.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection gives a high-confidence current-main gap: existing setup, configure, doctor/status, and Crestodian flows do not provide a goal-driven host-capability configuration recommender.

## Solution Assessment

No, not as an unscoped automation attached to the normal main agent. A safer direction is a maintainer-scoped design using typed setup/config operations, explicit approvals, audit logs, opt-in host inventory, and rollback paths.
