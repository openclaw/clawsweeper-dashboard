## Summary

Keep open. Current main has guided onboarding, section-based configure, plugin setup prompts, doctor/status checks, and Crestodian typed setup/repair, but it still does not provide the requested assistant-style recommender that interviews the user, probes host capability, and maintains best-practice configuration over time.

## What This Changes

Request a built-in automation linked to the main agent that asks setup questions, inspects local capabilities such as CPU/memory/GPU, and tunes OpenClaw configuration as features change.

## Best Possible Solution

Design a scoped setup/recommendation UX that extends onboarding, configure, plugin metadata, and Crestodian typed operations while keeping host discovery opt-in and all persistent writes explicit, audited, reversible, and documented.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection show existing setup/configuration flows but no goal-driven host-capability configuration recommender on current main.

## Solution Assessment

No, not as an unscoped automation attached to the normal main agent. A safer direction is a maintainer-scoped design using typed setup/config operations, recommendation metadata, explicit approvals, audit logs, and rollback paths.
