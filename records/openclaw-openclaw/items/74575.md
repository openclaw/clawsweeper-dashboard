## Summary

Keep open. Current main has fixed-section onboarding/configure flows plus bounded Crestodian setup/repair, but it does not provide a goal-driven, host-aware configuration recommender; the related open PRs address narrower setup prompts and first-run routing rather than this broader feature.

## What This Changes

The issue requests a built-in, agent-guided configuration wizard that probes the host, interviews the user about goals, and updates OpenClaw configuration as best practices and features evolve.

## Best Possible Solution

Define a scoped setup recommendation flow that builds on onboarding, configure, plugin metadata, and Crestodian typed operations while making host discovery opt-in and every persistent write explicit, audited, and reversible.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection gives a high-confidence current-main gap: existing onboarding, configure, doctor/status, and Crestodian flows do not provide a goal-driven host-capability configuration recommender.

## Solution Assessment

No, not as an unscoped automation attached to the normal main agent. A safer direction is a maintainer-scoped design using typed setup/config operations, explicit approvals, audit logs, opt-in host inventory, and rollback paths.
