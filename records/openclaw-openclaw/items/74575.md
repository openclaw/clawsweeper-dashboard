## Summary

Keep open. Current main has fixed-step onboarding, section-based configure, health/doctor checks, and Crestodian typed setup/repair, but it still does not provide the requested assistant-style recommender that interviews the user, probes host resources, and proposes best-practice configuration updates over time.

## What This Changes

Request a built-in wizard-like automation that asks the user questions, inspects host capability, and tunes OpenClaw configuration based on goals and new features.

## Best Possible Solution

Design a scoped setup/recommendation flow that builds on onboarding, configure, plugin metadata, and Crestodian typed operations while keeping host discovery opt-in and every persistent write explicit, audited, and reversible.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection gives a high-confidence current-main gap: existing setup, configure, doctor/status, and Crestodian flows do not provide a goal-driven host-capability configuration recommender.

## Solution Assessment

No, not as an unscoped automation attached to the normal main agent. A safer direction is a maintainer-scoped design using typed setup/config operations, explicit approvals, audit logs, and rollback paths.
