## Summary

Keep open. Current main has guided onboarding, section-based reconfiguration, plugin field prompts, and Crestodian typed setup/repair, but it still does not provide the requested assistant that interviews users, probes host resources, recommends best-practice settings, and revisits configuration as features change. The request is broad and security-sensitive enough to need maintainer product/security scoping rather than cleanup-close or an automated fix PR.

## What This Changes

Requests a built-in guided configuration assistant that asks users about goals, probes host capabilities such as CPU, memory, and GPU, recommends best-practice OpenClaw settings, and updates recommendations as new features ship.

## Best Possible Solution

Define a scoped next-generation setup UX that builds on onboarding, `openclaw configure`, plugin `configUiHints`, and Crestodian typed operations: gather user goals, expose consented host/resource discovery, let plugins contribute schema-backed recommendation metadata where appropriate, and keep persistent config writes explicit, audited, reversible, and documented.

## Reproduction Assessment

Not applicable as a feature request. The current-main gap is high confidence from docs and source inspection: onboarding, configure, plugin field prompts, and Crestodian do not provide the requested goal-driven host-capability recommender.

## Solution Assessment

No, not as a blind built-in config writer linked to the normal agent. The maintainable path is a maintainer-scoped design that extends typed setup/config surfaces and plugin metadata while preserving explicit approval, audit, and rollback boundaries.
