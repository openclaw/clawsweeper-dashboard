## Summary

Keep open. Current main has onboarding, section-based configure, doctor/health checks, and Crestodian setup/repair, but it still lacks the requested goal-driven, host-aware configuration recommender; related open PRs are narrower onboarding UX changes, so this needs maintainer product and security design.

## What This Changes

Request to add a built-in agent-guided configuration wizard that probes host capabilities, asks about user goals, and tunes OpenClaw settings over time.

## Best Possible Solution

Design a scoped, opt-in setup recommendation flow that builds on onboarding, configure, plugin metadata, and Crestodian typed operations while keeping host discovery explicit and every persistent write approved, audited, and reversible.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection gives a high-confidence current-main gap: existing onboarding, configure, doctor/status, and Crestodian flows do not provide a goal-driven host-capability configuration recommender.

## Solution Assessment

No, not as an unscoped automation attached to the normal main agent. A safer direction is a maintainer-scoped design using typed setup/config operations, explicit approvals, audit logs, opt-in host inventory, and rollback paths.
