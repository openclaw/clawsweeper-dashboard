## Summary

Keep open. Current main has guided onboarding, section-based configure, doctor checks, plugin setup prompts, and Crestodian typed setup/repair, but it still does not provide the requested assistant-style configuration recommender that interviews the user, probes host resources, and revisits best-practice settings over time.

## What This Changes

Request to add a built-in agent-guided configuration wizard that asks questions, checks host resources such as CPU/memory/GPU, recommends best-practice settings, and updates configuration as OpenClaw changes.

## Best Possible Solution

Design a scoped setup recommendation UX that extends onboarding/configure/Crestodian with explicit host-discovery consent, typed proposed config changes, approvals, audit logs, rollback, tests, and docs.

## Reproduction Assessment

Not applicable as a feature request. The current-main gap is high-confidence from source and docs inspection: existing setup/configure/Crestodian flows do not provide a goal-driven host-capability configuration recommender.

## Solution Assessment

No, not as an unscoped automation linked to the normal main agent. A safer path is a maintainer-designed recommendation flow built on typed setup/config operations with explicit approval, audit, and rollback boundaries.
