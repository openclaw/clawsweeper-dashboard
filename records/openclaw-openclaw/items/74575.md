## Summary

Keep open. Current main has guided onboarding, section-based configuration, plugin field prompts, and Crestodian typed setup/repair, but it still does not provide the requested goal-driven configuration advisor; the remaining work is broad product and security design, not conservative cleanup.

## What This Changes

The issue requests an assistant-style OpenClaw configuration wizard that asks goal questions, probes host resources, recommends best-practice settings, and revisits configuration as features change.

## Best Possible Solution

Define a scoped next-generation setup UX on top of onboarding, configure, Crestodian, and plugin setup metadata: gather user goals, use consented read-only host discovery, let plugins provide schema-backed recommendation hints where appropriate, present proposed changes for approval, and keep writes audited, reversible, and documented.

## Reproduction Assessment

Not applicable as a feature request. The current-main gap is high-confidence from source and docs inspection: existing setup/configuration paths do not interview the user, inspect CPU/memory/GPU, or generate a best-practice configuration plan.

## Solution Assessment

No, not as a broad built-in config writer linked to the normal agent. The safer direction is a maintainer-scoped design that extends typed setup/config surfaces and plugin metadata while preserving explicit approval, audit, and rollback boundaries.
