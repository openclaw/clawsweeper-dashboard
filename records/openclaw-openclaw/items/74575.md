## Summary

Keep open. Current main has guided onboarding, section-based configure, schema-driven plugin prompts, and Crestodian typed setup/repair, but it still does not provide the requested assistant-style configuration recommender that interviews the user, probes host resources, recommends best-practice settings, and revisits recommendations as features change.

## What This Changes

Request for a built-in guided configuration automation that asks users goal-oriented questions, checks host capabilities, and tunes OpenClaw settings over time.

## Best Possible Solution

Design a scoped next-generation setup UX that builds on onboarding, configure, plugin metadata, and Crestodian typed operations while keeping host/resource discovery consented and all persistent writes explicit, audited, reversible, and documented.

## Reproduction Assessment

Not applicable as a feature request. The current-main gap is high-confidence from source and docs inspection: existing setup/configure/Crestodian flows do not provide a goal-driven host-capability configuration recommender.

## Solution Assessment

No, not as an unscoped built-in automation linked to the normal agent. A safer solution is a maintainer-scoped design that extends typed setup/config surfaces and plugin recommendation metadata while preserving approval, audit, and rollback boundaries.
