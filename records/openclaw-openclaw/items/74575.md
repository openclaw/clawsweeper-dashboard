## Summary

Keep open. Current main has guided onboarding, section-based `openclaw configure`, plugin config prompts, schema-driven config editing, and Crestodian typed setup/repair, but it still lacks the requested assistant-style configuration recommender that interviews the user, probes host resources such as CPU/memory/GPU, proposes best-practice settings, and revisits recommendations as features change. This is broad product and security design work rather than a cleanup-close or safe automated repair.

## What This Changes

The issue requests a built-in wizard-guided automation that interviews the user, inspects host capabilities, recommends best-practice OpenClaw settings, and updates configuration as features evolve.

## Best Possible Solution

Define a scoped next-generation setup UX that builds on existing onboarding, `openclaw configure`, plugin schema/config metadata, and Crestodian typed operations. It should gather user goals, use consented host/resource discovery, let plugins contribute schema-backed recommendation metadata where appropriate, and keep persistent writes explicit, audited, reversible, and documented.

## Reproduction Assessment

Not applicable as a feature request. The current-main gap is high-confidence from source and docs inspection: onboarding, configure, plugin prompts, config schema editing, and Crestodian do not provide a goal-driven host-capability configuration recommender.

## Solution Assessment

No, not as an unscoped built-in config writer linked to the normal agent. The safer solution is a maintainer-scoped design that extends typed setup/config surfaces and plugin metadata while preserving explicit approval, audit, and rollback boundaries.
