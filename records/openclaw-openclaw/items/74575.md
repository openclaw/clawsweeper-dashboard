## Summary

Keep open. Current main covers guided onboarding, section-based reconfiguration, plugin-provided setup/config prompts, and Crestodian typed setup/repair, but it still lacks the requested assistant that interviews users, probes host resources, recommends best-practice settings, and revisits configuration as features change; that remaining gap is broad product work with security-sensitive config-write authority.

## What This Changes

The issue requests a built-in wizard or agent-guided automation that asks users questions, inspects host capability such as CPU, memory, and GPU, and tunes OpenClaw configuration over time.

## Best Possible Solution

Define a scoped next-generation setup UX that builds on onboarding, openclaw configure, plugin configUiHints/setup descriptors, and Crestodian typed operations: gather user goals, expose consented host/resource discovery, let plugins contribute schema-backed recommendations where appropriate, and keep persistent config changes explicit, audited, reversible, and documented.

## Reproduction Assessment

Not applicable as a feature request. The current-main gap is high confidence from docs and source inspection: onboarding, configure, plugin prompts, and Crestodian do not provide the requested goal-driven host-capability recommender.

## Solution Assessment

No, not as an unrestricted built-in config writer linked to the normal agent. The maintainable path is a maintainer-scoped design that extends typed setup/config surfaces and plugin metadata while preserving explicit approval, audit, and rollback boundaries.
