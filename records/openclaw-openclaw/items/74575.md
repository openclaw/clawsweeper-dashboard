## Summary

Keep open. Current main has guided onboarding, fixed-section configuration, plugin field prompts, first-run identity bootstrapping, and Crestodian typed setup/repair, but it still does not provide the requested assistant-style recommender that interviews the user, probes host resources, and updates best-practice configuration as features change.

## What This Changes

Request a built-in guided automation that asks setup questions, checks host resources, and tunes OpenClaw configuration over time.

## Best Possible Solution

Design a scoped setup/recommendation flow that extends onboarding, configure, plugin metadata, and Crestodian typed operations while keeping host discovery consented and all persistent writes explicit, audited, reversible, and documented.

## Reproduction Assessment

Not applicable as a feature request. The current-main gap is high-confidence from docs and source inspection: existing setup/configure/Crestodian flows do not provide a goal-driven host-capability configuration recommender.

## Solution Assessment

No, not as an unscoped built-in automation attached to the normal agent. A safer direction is a maintainer-scoped design using typed setup/config operations, recommendation metadata, explicit approvals, audit logs, and rollback paths.
