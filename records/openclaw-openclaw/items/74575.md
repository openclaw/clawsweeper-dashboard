## Summary

Keep open. Current main has guided onboarding, section-based configure, plugin field prompts, and Crestodian typed setup/repair, but it still lacks the requested assistant-style configuration recommender that interviews the user, probes host resources, and revisits best-practice recommendations as features change.

## What This Changes

Add a built-in assistant-style configuration wizard that probes host capabilities, interviews the user, recommends best-practice settings, and can rerun as OpenClaw features evolve.

## Best Possible Solution

Design a scoped setup/recommendation UX that extends onboarding, configure, plugin metadata, and Crestodian typed operations while keeping host discovery consented and all persistent writes explicit, audited, reversible, and documented.

## Reproduction Assessment

Not applicable as a feature request. The current-main gap is high-confidence from source and docs inspection: existing setup/configure/Crestodian flows do not provide a goal-driven host-capability configuration recommender.

## Solution Assessment

No, not as an unscoped built-in automation attached to the normal agent. A safer direction is a maintainer-scoped design that uses typed setup/config operations, recommendation metadata, explicit approvals, audit logs, and rollback paths.
