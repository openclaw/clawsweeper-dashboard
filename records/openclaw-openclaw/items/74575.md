## Summary

Keep open. Current main has guided onboarding, section-based configure, plugin setup prompts, status/doctor checks, and Crestodian typed setup/repair, but it still lacks the requested assistant-style configuration recommender that interviews the user, probes host resources such as CPU/memory/GPU, and revisits best-practice settings as OpenClaw changes.

## What This Changes

Add a built-in automation that asks goal-oriented questions, checks the host environment, and applies recommended OpenClaw configuration updates over time.

## Best Possible Solution

Design a scoped setup/recommendation UX that extends onboarding, configure, plugin metadata, and Crestodian typed operations while keeping host discovery consented and all persistent writes explicit, audited, reversible, and documented.

## Reproduction Assessment

Not applicable as a feature request. The current-main gap is high-confidence from docs/source inspection: existing setup/configure/Crestodian flows do not provide a goal-driven host-capability configuration recommender.

## Solution Assessment

No, not as an unscoped automation attached to the normal main agent. A safer direction is a maintainer-scoped design using typed setup/config operations, recommendation metadata, explicit approvals, audit logs, and rollback paths.

## Security Review

Status: needs_attention

Summary: The requested feature is security-sensitive because it combines local host probing, assistant guidance, and persistent configuration writes.
