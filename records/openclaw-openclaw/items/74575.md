## Summary

Keep open. Current main still has fixed-step onboarding, section-based configure, metadata-driven plugin prompts, status/doctor checks, and Crestodian typed setup/repair, but it does not provide the requested assistant-style configuration recommender that interviews the user, probes host capabilities, and revisits best-practice settings as OpenClaw changes.

## What This Changes

The issue requests a built-in agent or wizard that asks goal questions, inspects host capability, and recommends or applies OpenClaw configuration updates over time.

## Best Possible Solution

Design a scoped recommendation UX on top of onboarding, configure, plugin metadata, and Crestodian typed operations, with opt-in host probes and explicit approval/audit for every persistent write.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection on current main gives a high-confidence gap: existing setup, configure, plugin setup, doctor/status, and Crestodian flows do not provide a goal-driven host-capability configuration recommender.

## Solution Assessment

No, not as an unscoped automation attached to the normal main agent. A safer direction is a maintainer-scoped setup/recommendation design using typed operations, explicit approvals, audit logs, and rollback paths.

## Security Review

Status: needs_attention

Summary: The requested feature is security-sensitive because it combines local host probing, assistant guidance, and persistent configuration writes.

- **[medium] Keep host probes opt-in and minimized:**
  - body: The request asks to inspect CPU, memory, GPU, and similar local traits. That inventory should be explicit, minimized, and not placed into normal agent context unless the user approves sharing it.
  - confidence: 0.78

  - body: Crestodian currently requires approval for persistent operations and logs applied writes; any future recommender should keep that boundary instead of letting planner output directly mutate configuration.
  - confidence: 0.88
