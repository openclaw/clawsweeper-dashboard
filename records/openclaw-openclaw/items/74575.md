## Summary

Keep open. Current main has guided onboarding, section-based configure, plugin setup metadata, and Crestodian typed setup/repair, but it still lacks the requested assistant-style recommender that interviews the user, probes host resources, and revisits best-practice configuration as OpenClaw changes.

## What This Changes

The issue requests a built-in wizard-like automation that asks goal questions, inspects machine capabilities, and tunes OpenClaw configuration over time.

## Best Possible Solution

Design a scoped setup recommendation flow that extends onboarding, configure, plugin metadata, and Crestodian typed operations while keeping discovery consented and every persistent change explicit, audited, reversible, and documented.

## Reproduction Assessment

Not applicable as a feature request. Current-main inspection gives a high-confidence gap: existing onboarding/configure/Crestodian flows do not provide a goal-driven host-capability configuration recommender.

## Solution Assessment

No, not as an unscoped built-in automation attached to the normal agent. A safer direction is a maintainer-designed recommender using typed setup operations, declarative recommendation metadata, explicit approvals, audit logs, and rollback paths.
