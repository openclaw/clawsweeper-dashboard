## What This Changes

The issue reports that interrupted Anthropic-family thinking streams can persist unsigned thinking blocks to session JSONL and poison later session replay.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/84448, then keep a separate write-side persistence follow-up open only if replay-side recovery is not considered sufficient.

## Reproduction Assessment

Yes at source level, but not live: current main can preserve an unsigned thinking block when replay policy has `preserveSignatures: false`, and Anthropic’s streaming contract explains why an interrupt before `signature_delta` can produce that state. I did not run the live Anthropic interruption path.

## Solution Assessment

Yes for replay recovery: provider-scoped stripping before Anthropic/Bedrock/Vertex sends is the narrowest maintainable mitigation. It may not fully solve write-side prevention, so that part should be explicitly accepted as follow-up or required before closing.
