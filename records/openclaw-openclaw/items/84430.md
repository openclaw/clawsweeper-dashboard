## Summary

Keep this issue open: current main and the latest release still gate invalid thinking-signature stripping on `policy.preserveSignatures`, and the linked fixing PR is open rather than merged.

## What This Changes

The issue reports that interrupted Anthropic signed-thinking streams can persist unsigned thinking blocks in session JSONL, poisoning future session replay with `Invalid signature in thinking block`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84448 or a replacement replay-hardening fix, then keep a separate follow-up only if write-side prevention remains required.

## Reproduction Assessment

No live high-confidence reproduction was run. The source path is reproducible by inspection: current main can persist assistant messages without a thinking-signature repair pass, and replay cleanup is skipped whenever `policy.preserveSignatures` is false.

## Solution Assessment

Yes for the main replay failure: using the existing invalid-signature stripper on signed-thinking provider paths is the narrow maintainable fix. The full suggested prevention of writing incomplete thinking blocks is a separate hardening scope if maintainers want that guarantee too.
