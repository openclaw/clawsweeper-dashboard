## Summary

Keep this issue open: current source has partial replay protection, but the reported persistent-session poisoning path is still plausible because unsigned thinking blocks can be written to JSONL and the replay sanitizer is still gated by transcript policy.

## What This Changes

The issue asks OpenClaw to prevent incomplete Anthropic/Bedrock thinking blocks without replay signatures from being persisted or replayed into the next provider request.

## Best Possible Solution

Make unsigned thinking-block cleanup provider-API aware before Anthropic/Bedrock sends, and add persistence or load-time repair so incomplete thinking-only blocks cannot poison future session replay.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection gives a credible repro target: persist an assistant thinking block without a nonblank signature, then replay it through an Anthropic-family path where signature stripping is not forced independently of `preserveSignatures`.

## Solution Assessment

The suggested direction is sound, but the safest implementation should key the defensive strip to provider/model API requirements and also repair persistence; a blanket all-provider strip could break providers where reasoning blocks are owned by a different contract.
