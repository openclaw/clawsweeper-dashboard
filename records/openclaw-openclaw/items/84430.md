## Summary

Keep open: current main has normal Anthropic/Bedrock replay cleanup, but source still leaves a credible gap where an interrupted stream can leave an unsigned thinking block in persisted session history and there is no write-time/load-time repair for already poisoned JSONL.

## What This Changes

The issue requests hardening Anthropic-family thinking-block persistence and replay so unsigned or incomplete thinking blocks cannot make a session fail on the next turn.

## Best Possible Solution

Add provider-aware unsigned-thinking cleanup before Anthropic/Bedrock sends and add write-time or load-time repair so incomplete thinking-only blocks cannot persistently poison session replay.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection gives a credible repro target: persist an assistant thinking block without a nonblank signature, then replay it through an Anthropic-family path that does not repair it before the provider call.

## Solution Assessment

Yes, the suggested direction is mostly right: defensive provider-aware replay stripping plus persistence/load repair is the narrow maintainable fix. A blanket all-provider strip would be too broad because other providers own different reasoning-block contracts.
