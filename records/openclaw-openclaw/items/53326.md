## Summary

Keep open: current main and the latest release still use the nested `.openclaw/workspace-state.json` path, so the PR is not obsolete, but the PR head still has blocking fallback, migration, and local data-integrity issues before it can merge.

## What This Changes

The PR changes `src/agents/workspace.ts` to write workspace setup state at `<workspace>/workspace-state.json`, fallback-read the legacy dot path, and stop creating the dot-prefixed state directory.

## Best Possible Solution

Revise this PR to keep setup state out of dot-prefixed workspace entries while using a collision-resistant non-dot filename or foreign-file guard, best-effort legacy probing, explicit legacy-state promotion, and focused workspace tests.

## Reproduction Assessment

Yes. Source inspection of the PR diff gives high-confidence paths for the legacy dot-path `EPERM`/`EINVAL` failure, foreign root `workspace-state.json` overwrite, and legacy-only state not being promoted.

## Solution Assessment

No. Moving setup state out of the dot directory is the right direction, but this PR is not the narrowest safe implementation until it handles legacy probe errors, root filename collisions, and migration persistence.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Make legacy fallback reads best-effort:** `src/agents/workspace.ts:269`
  - body: The fallback probes `.openclaw/workspace-state.json` whenever the new root state has no markers, but `readWorkspaceSetupState()` rethrows non-`ENOENT` errors. On TigerFS-style dot-restricted mounts, first-run setup and status checks can still abort before the root state is written; catch and ignore legacy-probe I/O errors.
  - confidence: 0.91
- **[P2] Avoid overwriting foreign root state files:** `src/agents/workspace.ts:207`
  - body: Writing to `path.join(dir, WORKSPACE_STATE_FILENAME)` uses the generic root name `workspace-state.json`. An existing non-OpenClaw JSON file there is parsed as empty state and later overwritten, so namespace the filename or detect and refuse foreign content before writing.
  - confidence: 0.88
- **[P2] Promote legacy state after fallback reads:** `src/agents/workspace.ts:401`
  - body: When `ensureAgentWorkspace()` loads state from `resolveLegacyWorkspaceStatePath(dir)`, `stateDirty` remains false, so completed legacy state is never written to the new root location. Persist fallback-loaded state to finish the migration and stop depending on the dot-prefixed path.
  - confidence: 0.86

## Security Review
