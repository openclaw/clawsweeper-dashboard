## Summary

Keep this PR open: the nested workspace-state bug remains on current main and in the latest release, but the current PR head still has blocking correctness issues, lacks real behavior proof, and is conflicting with main.

## What This Changes

The PR moves workspace setup state from `<workspace>/.openclaw/workspace-state.json` to `<workspace>/workspace-state.json`, adds legacy fallback reads, and stops creating the dot-prefixed state subdirectory.

## Best Possible Solution

Revise this PR to keep setup state out of dot-prefixed workspace entries with a collision-safe non-dot state file, best-effort legacy probes, legacy-state promotion, non-dot atomic temp writes, and focused workspace tests.

## Reproduction Assessment

Yes. Current main and `v2026.5.7` still resolve and write workspace setup state under `<workspace>/.openclaw/workspace-state.json`; this review verified the failure path from source and PR discussion, not by live TigerFS writes.

## Solution Assessment

No. The direction is appropriate, but the current PR can still fail on legacy dot-path probes, can overwrite a generic root `workspace-state.json`, and does not promote fallback-loaded legacy state to the new location.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Make legacy fallback reads best-effort:** `src/agents/workspace.ts:269`
  - body: `readWorkspaceSetupStateForDir` now probes `.openclaw/workspace-state.json` whenever the new root file has no markers, but `readWorkspaceSetupState()` rethrows non-`ENOENT` I/O errors. On the dot-restricted filesystems this PR targets, that legacy probe can abort setup or status checks before the new root state is created, so fallback probes need to catch and ignore unavailable legacy paths.
  - confidence: 0.91
- **[P2] Avoid overwriting foreign workspace-state files:** `src/agents/workspace.ts:207`
  - body: Writing OpenClaw state to `<workspace>/workspace-state.json` can clobber an existing project file with the same generic name. If that JSON parses without OpenClaw markers, setup treats it as empty state and later overwrites it, so the new root location needs namespacing or foreign-file detection before writing.
  - confidence: 0.88
- **[P2] Promote fallback state to the new location:** `src/agents/workspace.ts:400-401`
  - body: When `ensureAgentWorkspace` loads state from the legacy path, `stateDirty` remains false, so the later write never creates `<workspace>/workspace-state.json`. Existing workspaces keep depending on the dot-prefixed directory and can lose completion markers if that directory is unavailable; persist fallback-loaded state to the new path.
  - confidence: 0.86
