## Summary

Keep open. Current main still uses the nested `.openclaw/workspace-state.json` path, and the PR’s replacement path has blocking compatibility and data-integrity issues before merge.

## What This Changes

The PR changes `src/agents/workspace.ts` to write workspace setup state at `<workspace>/workspace-state.json`, fallback-read legacy state, and stop creating the dot-prefixed state directory.

## Best Possible Solution

Revise the PR around the same compatibility goal with a collision-resistant non-dot state location or foreign-file guard, best-effort legacy probing, legacy-state promotion, and focused workspace tests.

## Reproduction Assessment

Yes. The PR diff supports targeted reproductions for a dot-path I/O error during legacy fallback, a foreign root `workspace-state.json` collision, and a legacy-only state file that is read without being migrated.

## Solution Assessment

No. Moving state out of the dot directory is reasonable, but this implementation is not the safest maintainable fix until it handles fallback I/O errors, avoids root-file clobbering, and persists migrated legacy state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Make legacy state probes best-effort:** `src/agents/workspace.ts:269`
  - body: `readWorkspaceSetupStateForDir` probes `.openclaw/workspace-state.json` whenever the new root file has no markers, but `readWorkspaceSetupState()` rethrows non-`ENOENT` errors. On the dot-restricted filesystems this targets, first-run setup and status checks can still fail before the new root state is written; catch and ignore legacy-probe I/O errors in the fallback path.
  - confidence: 0.91
- **[P2] Avoid overwriting foreign workspace-state files:** `src/agents/workspace.ts:207`
  - body: Writing OpenClaw state to `<workspace>/workspace-state.json` can clobber an existing workspace file with the same generic name. If that file parses without OpenClaw markers, setup treats it as empty state and later overwrites it, so use a safer non-dot name or detect foreign files before writing.
  - confidence: 0.88
- **[P2] Promote fallback state to the new location:** `src/agents/workspace.ts:401`
  - body: When `ensureAgentWorkspace` loads state from the legacy path, `stateDirty` remains false, so the later write never creates the new root state file. Existing workspaces keep depending on `.openclaw/workspace-state.json`; mark fallback-loaded state dirty or explicitly write it to complete migration.
  - confidence: 0.86
