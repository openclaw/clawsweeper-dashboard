## What This Changes

The PR changes `src/agents/workspace.ts` to write workspace setup state at `<workspace>/workspace-state.json`, read that root path first, and fallback-read legacy `<workspace>/.openclaw/workspace-state.json`.

## Best Possible Solution

Revise this PR around the same goal with a collision-safe non-dot state file policy, best-effort legacy reads, legacy-state promotion to the new location, rebase-aware temp-file behavior, and focused workspace migration tests.

## Solution Assessment

No. Moving state out of dot-prefixed entries is the right direction, but this PR still needs safer filename policy, best-effort legacy probes, migration promotion, and rebase-safe behavior before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Make legacy state probes best-effort:** `src/agents/workspace.ts:269`
  - body: `readWorkspaceSetupStateForDir` falls back to `.openclaw/workspace-state.json` whenever the new root state has no markers, but `readWorkspaceSetupState()` rethrows non-`ENOENT` I/O errors. On the dot-restricted filesystems this PR is meant to support, that legacy probe can fail before new root state is created, so catch and ignore legacy-probe errors in fallback paths.
  - confidence: 0.91
- **[P2] Avoid overwriting foreign workspace-state files:** `src/agents/workspace.ts:207`
  - body: Writing OpenClaw state to `<workspace>/workspace-state.json` can clobber a project file with the same generic name. If that file parses without OpenClaw markers, setup treats it as empty state and later overwrites it, so use a safer non-dot name or detect foreign files before writing.
  - confidence: 0.88
- **[P2] Promote fallback state to the new location:** `src/agents/workspace.ts:401`
  - body: When `ensureAgentWorkspace` loads state from the legacy path, `stateDirty` remains false, so the later write never creates the new root state file. Existing workspaces keep depending on `.openclaw/workspace-state.json`; mark fallback-loaded state dirty or explicitly persist it to the new path.
  - confidence: 0.86
