## Summary

Keep open: current main still stores workspace setup state under the dot-prefixed `.openclaw` workspace subdirectory, but this PR has blocking correctness and data-integrity issues plus missing after-fix real behavior proof.

## What This Changes

The PR moves workspace setup state from `<workspace>/.openclaw/workspace-state.json` to `<workspace>/workspace-state.json`, adds legacy fallback reads, and stops creating the dot-prefixed state directory.

## Best Possible Solution

Revise this PR to keep state out of dot-prefixed workspace entries while using a collision-resistant non-dot filename or foreign-file guard, best-effort legacy probes, explicit legacy-state promotion, focused workspace tests, and redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection shows current main writes the dot-prefixed `.openclaw/workspace-state.json` path, and the PR diff gives high-confidence paths for non-`ENOENT` legacy probe failures, legacy-state non-promotion, and root filename clobbering.

## Solution Assessment

No. Moving state out of the dot directory is the right direction, but this PR is not the safest implementation until it handles legacy probe errors, data collision policy, and migration persistence.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Make legacy state probes best-effort:** `src/agents/workspace.ts:269`
  - body: The new fallback probes `.openclaw/workspace-state.json` when the root file has no markers, but `readWorkspaceSetupState()` rethrows non-`ENOENT` I/O errors. On the dot-restricted filesystems this targets, that can still abort first-run setup or status checks before root state is written; catch and ignore legacy-probe errors.
  - confidence: 0.91
- **[P2] Avoid overwriting foreign workspace-state files:** `src/agents/workspace.ts:207`
  - body: Writing OpenClaw state to `<workspace>/workspace-state.json` can clobber an existing project file with the same generic name. If that JSON lacks OpenClaw markers it is treated as empty state and later overwritten, so namespace the filename or refuse foreign content before writing.
  - confidence: 0.88
- **[P2] Promote fallback state to the new location:** `src/agents/workspace.ts:401`
  - body: When `ensureAgentWorkspace()` loads state from the legacy path, `stateDirty` remains false, so no root state file is written. Existing workspaces keep depending on the dot-prefixed path and can lose completion markers when that path is unavailable; mark fallback-loaded state dirty or explicitly write it to the new path.
  - confidence: 0.86
