## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Make legacy state probes best-effort:** `src/agents/workspace.ts:269`
  - body: `readWorkspaceSetupStateForDir` falls back to `.openclaw/workspace-state.json` when the new root state has no markers, but `readWorkspaceSetupState()` only swallows `ENOENT`. On dot-restricted backends this probe can throw `EPERM` or `EINVAL` and abort setup/status before the new root state is created, so legacy probes need to catch and ignore unavailable legacy paths.
  - confidence: 0.91
- **[P2] Avoid overwriting foreign workspace-state files:** `src/agents/workspace.ts:207`
  - body: Writing OpenClaw state to `<workspace>/workspace-state.json` can clobber an existing project file with the same generic name. If that JSON parses without OpenClaw markers, setup treats it as empty state and later overwrites it, so the new root location needs namespacing or foreign-file detection before writing.
  - confidence: 0.88
- **[P2] Persist legacy state at the new location:** `src/agents/workspace.ts:400-401`
  - body: When `ensureAgentWorkspace` loads state from the legacy path, `stateDirty` remains false, so the later write never creates the new root state file. Existing workspaces keep depending on the dot-prefixed directory and can lose completion markers if that directory is unavailable; promote successfully loaded legacy state to the new path.
  - confidence: 0.86
