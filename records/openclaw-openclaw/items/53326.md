## Summary

Keep this PR open: current main still uses the nested workspace state path, so the work is not obsolete, but the branch has blocking correctness gaps and lacks required real behavior proof before merge.

## What This Changes

The PR moves workspace setup state writes from `<workspace>/.openclaw/workspace-state.json` to `<workspace>/workspace-state.json` and adds legacy fallback reads in `src/agents/workspace.ts`.

## Best Possible Solution

Revise the same direction with a collision-resistant non-dot state filename or foreign-file guard, best-effort legacy probes, legacy-state promotion, non-dot temp-file behavior after rebase, and focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible. Current main resolves and tests workspace setup state under `<workspace>/.openclaw/workspace-state.json`; I did not run a live TigerFS mount.

## Solution Assessment

No. Moving setup state out of dot-prefixed workspace entries is the right direction, but this branch is not complete until it handles legacy probe errors, filename collision/data-loss risk, legacy-state promotion, and current-main rebase fallout.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Make legacy fallback probes best-effort:** `src/agents/workspace.ts:269`
  - body: `readWorkspaceSetupStateForDir` now probes `.openclaw/workspace-state.json` whenever the new root file has no markers, but `readWorkspaceSetupState()` rethrows non-ENOENT errors. On the dot-restricted filesystems this PR targets, that legacy probe can abort setup or status before the new root state is created.
  - confidence: 0.91
- **[P2] Avoid overwriting foreign workspace-state files:** `src/agents/workspace.ts:207`
  - body: Writing OpenClaw state to `<workspace>/workspace-state.json` can clobber a project file with the same generic name. If that file parses without OpenClaw markers, setup treats it as empty state and can later overwrite it, so use a safer non-dot name or detect foreign files before writing.
  - confidence: 0.88
- **[P2] Persist fallback state at the new location:** `src/agents/workspace.ts:401`
  - body: When `ensureAgentWorkspace` loads state from the legacy path, `stateDirty` remains false, so the later write never creates the new root state file. Existing workspaces keep depending on the dot-prefixed directory and can lose completion markers when that directory is unavailable.
  - confidence: 0.86
