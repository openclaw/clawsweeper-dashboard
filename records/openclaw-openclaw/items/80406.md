## Summary

Keep open. Current main supports zh-TW locale loading, but the settings Form view still resolves Config Schema labels and help from English schema/uiHints metadata rather than the i18n layer, so this is a real localization gap that needs maintainer direction.

## What This Changes

Request localizing settings Config Schema title/description labels for zh-TW instead of leaving generated form labels in English.

## Best Possible Solution

Define a first-class config metadata localization path for the Control UI, with English as the source, generated locale output, and fallback to English for missing schema labels/help.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with the Control UI in zh-TW, Settings > Config/Form labels still come from English `uiHints`/schema metadata rather than locale translations; I did not run a browser in this read-only review.

## Solution Assessment

Unclear. The requested direction is valid, but maintainers should choose the durable architecture before implementation; hand-editing `zh-TW.ts` is not the right fix because non-English locale bundles are generated.
