## Summary

Keep open: current main still renders settings Config Schema labels and help from English schema/UI hint metadata, and the linked broad i18n issue is closed not-planned rather than an active canonical fix for this narrower schema-label gap.

## What This Changes

The issue requests zh-TW localization coverage for Control UI settings form labels and descriptions that currently come from Config Schema title/description metadata.

## Best Possible Solution

Define one supported schema-label localization contract for Config Schema metadata, then update the form renderer, schema/UI hint generation, locale sync checks, and focused UI/schema tests around that contract.

## Reproduction Assessment

Yes, source-level: current main builds Config Schema labels/help from English FIELD_LABELS/FIELD_HELP and the form renderer displays hint.label or schema.title directly. A browser run was not needed to see why zh-TW locale files cannot affect those labels today.

## Solution Assessment

Unclear as an implementation choice: the report identifies the right boundary, but maintainers should first choose whether schema localization belongs in the gateway schema response, a UI-side path-key overlay, plugin metadata, or a combined contract.
