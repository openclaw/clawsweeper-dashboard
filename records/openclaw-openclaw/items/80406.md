## Summary

This should stay open because current main still renders Config form labels from English schema/uiHint metadata, while zh-TW locale bundles only cover keyed Control UI strings; the remaining fix needs schema/i18n design rather than stale cleanup.

## What This Changes

The issue asks for settings-page Config Schema title and description labels to be localizable for zh-TW instead of remaining English.

## Best Possible Solution

Define a locale-aware schema metadata contract that covers root config, plugin/channel uiHints, generated locale bundles, and docs without hand-maintaining non-English generated files.

## Reproduction Assessment

Yes, from source. The Config form consumes schema/uiHint labels directly, and the zh-TW locale bundle is a separate generated UI-string bundle, so changing zh-TW translation keys cannot localize those schema labels.

## Solution Assessment

No, a direct zh-TW locale edit is not the best fix. The maintainable solution is a schema/i18n contract that can translate the same metadata for all supported locales, including plugin and channel schema fields.
