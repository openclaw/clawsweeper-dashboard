## Summary

Keep open: current main still has a source-backed localization gap for settings schema labels/descriptions, and the related i18n issues are closed broad tracking/product-decision threads rather than an active fix for schema metadata.

## What This Changes

Config Schema title/description labels in the Control UI settings form should be localizable for zh-TW instead of remaining English.

## Best Possible Solution

Define a locale-aware schema metadata contract covering core labels/help plus plugin/channel hints, then wire the settings form to that contract and regenerate the Control UI locale outputs.

## Reproduction Assessment

Yes. Source inspection of current main shows the settings form reads `hint?.label ?? schema.title` and schema metadata is populated from English FIELD_LABELS/FIELD_HELP, so zh-TW locale bundle edits cannot affect those labels.

## Solution Assessment

Partly. Bringing schema title/description metadata into i18n or an override layer is the right direction, but maintainers should choose the schema/plugin/channel localization contract before implementation.
