## Summary

Keep open. Current main still renders Settings Config Schema labels/help from English schema and uiHints metadata rather than the Control UI i18n layer, so the reported zh-TW gap remains and needs maintainer direction on the schema metadata localization contract.

## What This Changes

Request to localize Settings Config Schema form labels and help text in zh-TW by bringing schema title/description metadata into i18n or adding an override layer.

## Best Possible Solution

Define a first-class config metadata localization path for the Control UI with English source strings, generated locale output, plugin/core hint coverage, and English fallback for missing translations.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set the Control UI locale to zh-TW and open Settings Form mode; labels/help still come from English uiHints/schema metadata with no locale lookup.

## Solution Assessment

Unclear. The requested direction is valid, but maintainers should choose the durable contract before implementation because schema metadata is shared between core, UI rendering, docs, and plugin-provided hints.
