## Summary

Keep open: current main still renders Config form labels and help from English schema metadata, and no active canonical item appears to own this narrower schema-i18n gap.

## What This Changes

Localize zh-TW Settings Config Schema form labels and descriptions instead of rendering English `title`/`description` metadata.

## Best Possible Solution

Define a schema-metadata localization design that keeps canonical English schema metadata stable for tooling while letting Control UI render generated locale-specific labels and descriptions.

## Reproduction Assessment

Yes, at source level. Current main resolves Config form labels from English `uiHints`/JSON Schema metadata, and the zh-TW locale tree is not consulted on that path.

## Solution Assessment

Unclear. The requested direction is valid, but maintainers should choose the contract boundary before implementation because both gateway-side localized schema and browser-side overrides have different compatibility tradeoffs.
