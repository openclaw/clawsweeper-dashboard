## Summary

Keep open. Current main still has zh-TW locale support, but the Settings form labels/help are resolved from English schema and uiHints metadata rather than the locale system, and the related broad localization issues are closed rather than an active canonical fix.

## What This Changes

Request to make Config Schema form `title` and `description` labels localizable for zh-TW in the Settings page.

## Best Possible Solution

Define a first-class config metadata localization path for the Control UI, with English as source, generated locale output, and English fallback for missing schema labels/help.

## Solution Assessment
