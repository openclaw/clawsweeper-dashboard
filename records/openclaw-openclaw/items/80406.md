## Summary

Keep open: current main still renders the reported settings labels from English config-schema metadata outside the Control UI i18n catalog. This is a fresh, source-reproducible localization gap, but the fix needs maintainer design for schema metadata localization rather than cleanup closure.

## What This Changes

Request to localize zh-TW settings form labels and descriptions that are currently sourced from Config Schema title/description metadata.

## Best Possible Solution

Define a locale-aware config-schema metadata layer that translates first-party labels/help through the Control UI i18n catalog while preserving schema validation semantics and plugin-owned metadata contracts.

## Reproduction Assessment

Yes. Source inspection on current main shows zh-TW is supported, while settings labels are resolved from uiHints/schema title/description populated by English FIELD_LABELS/FIELD_HELP rather than locale keys.

## Solution Assessment

Unclear. The requested direction is valid, but maintainers should choose between client-side path-key localization, locale-aware schema responses, and plugin schema behavior before implementation.
