## Summary

Keep open. Current main has zh-TW locale loading, but Settings Config/Form labels still resolve from English schema/uiHints metadata, and the closed broad i18n threads do not own this narrower schema-metadata localization gap.

## What This Changes

The issue requests that Settings Config Schema form field titles and descriptions be localizable for zh-TW instead of remaining English.

## Best Possible Solution

Define a first-class config metadata localization contract with English fallback, then wire Settings form labels/help through the generated Control UI i18n pipeline or a maintainer-approved schema-specific layer.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `config.schema` stores English `uiHints`, and `resolveFieldMeta` renders those English labels/help without a locale lookup.

## Solution Assessment

Unclear. The requested direction is valid, but maintainers should choose the durable contract because config schema metadata is also consumed by CLI/API tooling and plugin/channel schemas, not only the Settings UI.
