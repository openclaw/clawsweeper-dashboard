## Summary

Keep open: the PR targets a real Windows command-wrapper hardening gap, but the proposed resolver still accepts unsafe Windows root forms and the user-visible security fix needs a changelog entry.

## What This Changes

The PR changes Windows `.cmd`/`.bat` wrapper launches to derive `cmd.exe` from SystemRoot/SYSTEMROOT/windir/WINDIR instead of ComSpec and adds mocked Windows process-wrapper coverage.

## Best Possible Solution

Resolve the wrapper through the shared Windows install-root resolver, add regression coverage for poisoned ComSpec and unsafe root values, and record the security hardening in the changelog.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main uses `process.env.ComSpec` for Windows batch wrappers, and the PR’s replacement accepts unsafe absolute Windows root forms. I did not run tests because this review was read-only.

## Solution Assessment

No. Ignoring ComSpec is the right direction, but the proposed helper is weaker than the existing shared Windows install-root resolver and should not merge as the trust boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reject unsafe Windows root values:** `src/process/exec.ts:52-53`
  - body: The new helper treats any `path.win32.isAbsolute()` root as trusted, which still accepts UNC, root-relative, and semicolon-delimited values and then launches `System32\cmd.exe` from that location. Current main already has `getWindowsInstallRoots()`/`normalizeWindowsInstallRoot()` for this boundary; use that resolver instead of a new looser check.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `src/process/exec.ts:120`
  - body: This is a user-visible Windows command-execution security hardening, but the patch only changes the implementation and tests. OpenClaw policy requires user-facing fixes to be recorded in `CHANGELOG.md`, so the release notes would omit this hardening.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff narrows ComSpec use but introduces a weaker trust-root resolver that can still select a non-system command interpreter.

Concerns:

- **[medium] Unsafe Windows root values can select cmd.exe:** `src/process/exec.ts:52`
