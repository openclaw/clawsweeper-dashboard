## Summary

Keep open. The DeepSeek provider hook from the earlier fix is present and documented, but current `sessions.list` lightweight rows strip per-session `thinkingLevels`, and both Control UI `/think` paths prefer default-level metadata before falling back to the active session model, which can leave a DeepSeek V4 session showing only the base/default options.

## What This Changes

DeepSeek V4 Pro `/think` UI should expose `off`, `minimal`, `low`, `medium`, `high`, `xhigh`, and `max` instead of only the five base levels.

## Best Possible Solution

Restore active-session provider thinking metadata to the UI without undoing the lightweight `sessions.list` performance work, and add regression coverage for a DeepSeek V4 active session whose global defaults are not DeepSeek.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `sessions.list` returns lightweight rows without `thinkingLevels`, then the Control UI `/think` selector/executor falls back to global defaults instead of the active DeepSeek V4 row profile.

## Solution Assessment

No. The DeepSeek plugin-side fix is already present; the narrower maintainable fix is in the session-list/UI metadata path so active-session model options are preserved while keeping bulk session listing lightweight.
