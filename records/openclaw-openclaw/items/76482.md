## Summary

Keep open. Current main still has a source-reproducible UI/session-row path that can hide DeepSeek V4 `xhigh` and `max`, and open draft PR #76548 references this issue with closing syntax, so the issue should wait for that PR to be reviewed, landed, or closed.

## What This Changes

Expose all DeepSeek V4 Pro `/think` levels, including `xhigh` and `max`, in TUI and Control UI selectors.

## Best Possible Solution

Preserve active-session provider thinking metadata or add an active-session-specific fallback for UI/TUI `/think`, while keeping bulk `sessions.list` lightweight and adding a DeepSeek V4 regression where global defaults are not DeepSeek.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: `sessions.list` emits lightweight rows without active-session `thinkingLevels`, then Control UI/TUI `/think` paths can fall back to default/base options instead of the active DeepSeek V4 profile.

## Solution Assessment

No current-main fix is present. The provider hook is already correct; the narrow maintainable fix belongs in the session-row/UI metadata path and is being attempted by #76548.
