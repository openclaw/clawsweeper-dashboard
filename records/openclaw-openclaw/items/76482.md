## Summary

Keep open. Current main still has a source-reproducible path where lightweight `sessions.list` rows drop active-session thinking levels and Control UI `/think` paths can fall back to default-model levels; open draft PR #76548 is already the active fix path.

## What This Changes

Expose all DeepSeek V4 Pro `xhigh` and `max` thinking levels in TUI and Control UI `/think` options instead of collapsing to base/default levels.

## Best Possible Solution

Keep the DeepSeek plugin profile as-is and fix the gateway/UI session metadata path so active-session provider thinking levels reach `/think`, preferably by reviewing and landing #76548 or an equivalent narrow patch.

## Reproduction Assessment

Yes. Source inspection shows `sessions.list` builds lightweight rows with `thinkingLevels: []`, and both Control UI `/think` paths prefer `defaults.thinkingLevels` when session levels are missing, so an active DeepSeek V4 session can show only the default model's level set.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve or derive active-session thinking metadata in session rows and adjust fallback ordering, rather than changing the already-correct DeepSeek provider profile.
