## Summary

Keep open. Current main has the DeepSeek V4 provider profile and transport mapping, but lightweight `sessions.list` rows still remove active-session thinking options and both Control UI `/think` paths can fall back to global defaults before using the active DeepSeek model.

## What This Changes

The issue asks for DeepSeek V4 Pro `/think` UI/options to expose `off|minimal|low|medium|high|xhigh|max` instead of only the five base levels.

## Best Possible Solution

Restore active-session provider thinking metadata to Control UI and local `/think` while preserving the lightweight `sessions.list` performance work, with regression coverage for a DeepSeek V4 active session whose global defaults are not DeepSeek.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes: source inspection shows a DeepSeek V4 active row from `sessions.list` has no `thinkingLevels`, after which both Control UI option paths can use global default levels instead of the row's provider/model profile.

## Solution Assessment

Is this the best way to solve the issue? Yes: the DeepSeek plugin-side fix is already present, so the narrow maintainable repair is in the session-list/UI metadata fallback path rather than another provider-specific core change.
