## Summary

Keep open. Current main still has a source-clear path where an oversized transcript JSONL record is converted into a whole-message `chat.history` placeholder before display projection can preserve adjacent text, and the open linked PR is still the active fix path.

## What This Changes

Preserve normal text blocks in `chat.history` when a mixed text plus oversized inline image transcript entry is too large to keep verbatim.

## Best Possible Solution

Review and land a corrected version of https://github.com/openclaw/openclaw/pull/78730, or a narrow replacement, that preserves recoverable text while omitting only oversized media payloads.

## Reproduction Assessment

Yes. Source inspection shows a transcript line over 256 KiB with normal text plus a large inline image will hit `parseTailTranscriptRecord`'s oversized-line branch and become a generic placeholder before `chat.history` can display the text.

## Solution Assessment

Yes. The narrow maintainable fix is in the transcript tail/history projection boundary: preserve bounded text blocks and replace only oversized media, rather than increasing history budgets or keeping raw base64.
