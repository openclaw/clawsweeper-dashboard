## Summary

Keep open. Current main still has a source-clear path where an oversized transcript JSONL record is replaced with a whole-message placeholder before `chat.history` display projection can preserve recoverable text, and the paired open fix PR at https://github.com/openclaw/openclaw/pull/78730 is the active resolution path.

## What This Changes

Preserve normal text blocks in `chat.history` for oversized multimodal transcript entries while omitting the large inline image payload.

## Best Possible Solution

Land or close the linked PR after review, with a focused gateway history regression test that preserves text and replaces only the oversized media payload.

## Reproduction Assessment

Yes, from source inspection: a transcript line over 256 KiB with text plus a large inline image block will hit `parseTailTranscriptRecord`'s oversized-line path and become a generic placeholder. I did not execute a live repro because this review was constrained to read-only inspection.

## Solution Assessment

Yes. Preserving recoverable text while replacing only large media is narrower than changing history budgets and matches the existing parsed-message sanitizer behavior.
