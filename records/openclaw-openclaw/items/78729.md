## Summary

Keep open. Current main still replaces an oversized transcript JSONL record with a whole-message placeholder before `chat.history` projection can preserve adjacent text, and the same-author closing PR at https://github.com/openclaw/openclaw/pull/78730 remains open and unmerged.

## What This Changes

The issue requests preserving normal text blocks from oversized multimodal transcript history entries while omitting only large inline image payloads.

## Best Possible Solution

Review and land a corrected version of the linked PR with a focused regression test that preserves recoverable text and replaces only omitted oversized media.

## Reproduction Assessment

Yes. A transcript JSONL line over 256 KiB containing both a text block and large inline image payload will hit `parseTailTranscriptRecord`'s oversized-line branch and become a whole-message placeholder before `chat.history` display projection runs.

## Solution Assessment

Yes. The maintainable fix is a narrow gateway transcript-tail parsing/sanitization change with regression coverage, not a larger history budget or product-policy change.
