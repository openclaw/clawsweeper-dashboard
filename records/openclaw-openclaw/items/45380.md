## What This Changes

Adds `scripts/with-env.mjs` and updates selected `package.json` scripts to pass inline env assignments through that helper for Windows-compatible npm script execution.

## Best Possible Solution

Land a rebased cross-platform env wrapper for the current package scripts while preserving `scripts/run-vitest.mjs` and `scripts/test-live.mjs`, then attach redacted native-Windows proof.

## Reproduction Assessment

Yes by source inspection: current `package.json` still has POSIX-style env prefixes on Windows-relevant gateway and TUI scripts, and native Windows shells do not parse `KEY=value` as an environment prefix. I did not run a native Windows host in this read-only pass.

## Solution Assessment

No for the PR as-is. A repo-local helper is a reasonable fix direction, but this branch needs a rebase that preserves current test wrappers plus inspectable native-Windows proof before merge.
