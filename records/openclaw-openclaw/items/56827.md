## Summary

Keep open. Current main still materializes the monolithic session store on the doctor path, and the explicit cleanup/disk-budget paths still do not classify `sessions.json.*.tmp` sidecars as removable stale session artifacts.

## What This Changes

`openclaw doctor` should complete on large long-lived session stores and should not be destabilized by accumulated stale `sessions.json.*.tmp` sidecars.

## Best Possible Solution

Add conservative stale session-store temp sidecar detection and cleanup to explicit doctor/session cleanup paths, while leaving broader session-store redesign to https://github.com/openclaw/openclaw/issues/55334 and corrupt-store recovery to https://github.com/openclaw/openclaw/pull/79910.

No high-confidence runtime reproduction of the exact V8 failure was established. The source path is reproducible by inspection: doctor still loads and enumerates the whole store, and cleanup/disk-budget paths still exclude stale `sessions.json.*.tmp` sidecars.

## Solution Assessment

Yes for a focused first repair. The maintainable path is explicit stale sidecar cleanup in doctor/session maintenance, not a SQLite migration or a duplicate corrupt-store recovery implementation.
