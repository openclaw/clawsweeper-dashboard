## Summary

Keep open. Current main has reduced some large-store risk through JSON-shaped session-store cloning and load-time entry maintenance, but it still reads/parses the monolithic store before doctor state checks and still has no source-backed path that detects or removes stale `sessions.json.<uuid>.tmp` files. The stale-temp accumulation remains actionable and bounded.

## What This Changes

`openclaw doctor` should remain reliable on large long-lived session stores and should detect or safely clean stale `sessions.json.*.tmp` atomic-write leftovers.

## Best Possible Solution

Add a narrow session-maintenance fix: make doctor/session cleanup report and safely remove stale same-directory session-store temp files, keep disk-budget cleanup from evicting real sessions while ignored temp files dominate usage, and only add a lightweight large-store guard or warning unless maintainers separately choose a SQLite redesign.

## Reproduction Assessment

Unclear. The report gives concrete store sizes and current source confirms the heavy full-store read plus missing temp cleanup path, but this read-only review did not build a 1.2 GB fixture or reproduce the exact V8 stack.

## Solution Assessment

Yes for a focused first repair, no for making SQLite the immediate fix. The maintainable path is to harden existing doctor/session cleanup and disk-budget behavior around stale atomic-write temp files before considering a broader metadata-store migration.

## Repair Work Prompt

Fix the stale session-store temp-file maintenance gap for https://github.com/openclaw/openclaw/issues/56827. Current main still reads/parses `sessions.json` for doctor state checks and does not detect or remove stale same-directory `sessions.json.<uuid>.tmp` files. Keep the fix narrow: add a helper/predicate for session-store atomic temp files, teach doctor and/or `openclaw sessions cleanup` to report and safely clean only old `sessions.json.<uuid>.tmp` files, and make disk-budget cleanup account for those temp files before evicting real session entries. Use a conservative age threshold so active writes are not raced. Do not implement a SQLite metadata migration in this PR. Add regression tests for stale temp detection/removal, fresh temp preservation, and disk-budget behavior when temp files dominate the sessions directory. Update docs and `CHANGELOG.md` with user-facing wording and reporter credit as policy requires.
