## Summary

Keep open. Current main still reads and parses the full `sessions.json` on the doctor path, and current cleanup/disk-budget artifact classifiers still do not recognize stale `sessions.json.*.tmp` sidecars as removable session-store artifacts.

## What This Changes

`openclaw doctor` should complete on large long-lived session stores and avoid being destabilized by accumulated stale `sessions.json.*.tmp` sidecars.

## Best Possible Solution

Add conservative stale `sessions.json.*.tmp` detection and age-gated cleanup to explicit doctor/session maintenance paths, while leaving broader SQLite/session-store redesign to https://github.com/openclaw/openclaw/issues/55334 and corrupt-store recovery to https://github.com/openclaw/openclaw/pull/79910.

## Solution Assessment

Yes for a focused first repair. The maintainable path is explicit stale sidecar cleanup in doctor/session maintenance, not a SQLite migration or a duplicate corrupt-store recovery implementation.
