## Summary

Keep open: current main still contains source-level home-path shortening for user-facing display paths, and the only directly linked fix PR was closed unmerged. The requested all-agent-output path policy is useful ergonomics work, but it needs maintainer direction because current prompts and renderers distinguish Markdown link targets, workspace-relative paths, and compact display paths.

## What This Changes

The issue requests that agent replies use full absolute file paths instead of `~` shorthand for copyable file references.

## Best Possible Solution

Maintainers should choose one canonical user-facing path policy for agent replies, then implement it at the prompt/display boundary without adding a broad environment variable unless that new config surface is explicitly desired.

## Reproduction Assessment

No high-confidence live repro was established. Source inspection does show current display helpers shortening home-directory paths and a user-facing tool-display caller using that helper, so the reported class of behavior is source-reproducible.

## Solution Assessment

Unclear. Prompt guidance may be enough for model-authored Markdown file links, but post-processing every agent response or adding an env-var mode could create policy drift unless maintainers first choose the desired path contract.
