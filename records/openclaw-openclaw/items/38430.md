Keep open: current main still does not implement a default rule expanding or forbidding `~/` in user-visible agent replies, and the related fix PR was closed unmerged. The remaining question is a global assistant-output policy choice, not a proven regression against an existing contract.

## What This Changes

## Best Possible Solution

Make an explicit maintainer decision on user-visible path style, then implement the chosen policy in the central prompt and snapshot coverage rather than rewriting arbitrary assistant text.

## Solution Assessment

Unclear. A scoped central prompt directive is likely safer than post-processing model output, but the repo first needs a product decision because existing prompt, tests, and docs intentionally support relative and home-relative paths.
