## Summary

Keep open: current main still does not define a global full-absolute-path policy for agent replies, and existing prompt/docs/UI behavior deliberately permits relative or home-relative paths in some contexts, so this needs maintainer product direction rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to stop emitting `~/...` in agent replies and use full absolute paths for file references.

## Best Possible Solution

Make an explicit product decision for user-visible path style, then implement the chosen rule in the central prompt and focused tests, avoiding broad output rewriting unless the safe boundaries are specified.

## Reproduction Assessment

No. Source inspection shows no central output rule and confirms `MEDIA:~/...` is preserved, but there is no high-confidence transcript or test proving ordinary assistant prose emits tilde paths on current main.

## Solution Assessment

Unclear. A scoped prompt directive would be safer than blanket post-processing if maintainers want this behavior, because docs and UI tests deliberately preserve home-relative media paths.
