## Summary

Keep open: current main and the latest release still intentionally shorten home-directory paths to `~` across shared helpers, tool summaries, skill prompts, media docs, approval display, and Control UI display. The linked env-var implementation was closed unmerged, so the remaining work is a maintainer path-presentation policy decision rather than conservative cleanup.

## What This Changes

The issue asks OpenClaw to show full absolute file paths in agent-visible responses instead of `~` home-directory shorthand.

## Best Possible Solution

Define and document a per-surface path-presentation contract first, then change only the agent prose, tool summaries, skill prompts, media directives, approval UI, or clients that should show absolute paths.

## Reproduction Assessment

No, not for the full end-to-end agent-prose behavior from the report. Source inspection does show deterministic current-main `~/` compaction or preservation across the relevant display, prompt, media, approval, and client paths.

## Solution Assessment

Unclear. The suggested blanket prompt/post-processing or env-var approach is not obviously safest because current docs and tests rely on `~/`; a maintainer-defined surface policy should come first.
