## Summary

Keep open: current main still does not implement a default rule or post-processor expanding `~/` in agent replies, and related PR #38495 was closed unmerged. This is better treated as an assistant-output policy request than a proven regression.

## What This Changes

The issue asks for agent reply file paths to use full absolute paths instead of `~/...` shorthand.

## Best Possible Solution

Make an explicit maintainer decision on user-visible path style, then implement the chosen policy in the central agent prompt and prompt coverage rather than blanket rewriting assistant text by default.

## Reproduction Assessment

No. Source inspection shows OpenClaw does not enforce the requested absolute-path style and preserves assistant text through final payload assembly, but there is no high-confidence live transcript or failing current-main test showing a model response emitting `~/...`.

## Solution Assessment

Unclear. A scoped central prompt directive with prompt snapshot coverage is likely safer than post-processing model output, but maintainers first need to decide whether this should override documented relative and home-relative path behavior.
