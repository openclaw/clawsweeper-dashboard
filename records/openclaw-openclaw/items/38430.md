## Summary

Keep open: current main still has no central rule expanding or forbidding `~/` in user-visible agent replies, but the existing docs/tests intentionally allow relative and home-relative paths in several contexts, so this needs a maintainer policy decision rather than cleanup closure.

## What This Changes

The issue requests that user-visible agent file paths use full absolute paths instead of `~/` home-relative shorthand.

## Best Possible Solution

Make an explicit product decision for final-answer path style, then encode the chosen policy in the central system prompt with focused prompt/test coverage; avoid broad text rewriting unless maintainers define exact safe boundaries.

## Reproduction Assessment

No. Source inspection confirms current main does not enforce full absolute path output, but there is no deterministic transcript, live reproduction, or failing test showing an assistant reply emitting `~/...` for ordinary file references.

## Solution Assessment

Unclear. A scoped central prompt directive would be safer than blanket post-processing if maintainers want this behavior, but current docs/tests deliberately preserve `~/` in media and workspace-related contexts.
