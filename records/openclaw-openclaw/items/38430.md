## Summary

Keep open: current main still intentionally compacts or preserves home-relative `~/` paths across shared path helpers, tool summaries, UI summaries, skill prompt locations, and media directives, so this remains a product-level path presentation decision rather than a resolved bug.

## What This Changes

The issue asks OpenClaw to stop showing `~` shorthand in agent-visible file paths and show full absolute paths instead.

## Best Possible Solution

Define a surface-specific path presentation contract, then update only the agent/user-facing surfaces that should show absolute paths while preserving documented `~/` inputs where intended.

## Reproduction Assessment

No, not for the full end-to-end agent-prose behavior from the report. Source inspection does show deterministic current-main `~/` compaction or preservation across shared display, prompt, media, approval, and UI paths.

## Solution Assessment

Unclear. A blanket prompt rule, post-processor, or env var is not obviously the safest fix because current docs and tests rely on `~/`; maintainers should define the per-surface contract first.
