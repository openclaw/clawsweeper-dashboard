## Summary

Keep open: current main and the latest release still intentionally compact home-directory paths to `~` across shared helpers, tool summaries, skill prompts, media docs/tests, approval display, and client display. The related env-var PR was closed unmerged, so the remaining work is a maintainer path-presentation policy decision rather than cleanup that can be safely closed.

## What This Changes

The issue requests that agent-facing file paths be shown as full absolute paths instead of home-relative `~/...` paths.

## Best Possible Solution

Define and document a per-surface path-presentation contract first, then update only the agent prose, tool summaries, skill prompts, media directives, approval UI, and clients that should show absolute paths.

## Reproduction Assessment

No, not for the full end-to-end agent-prose behavior from the report. Source inspection does show deterministic current-main `~/` compaction or preservation across the relevant display, prompt, media, approval, and client paths.

## Solution Assessment

Unclear. The suggested blanket prompt/post-processing or env-var approach is not obviously the best fix because current docs and tests rely on `~/`; a maintainer-defined policy is safer.
