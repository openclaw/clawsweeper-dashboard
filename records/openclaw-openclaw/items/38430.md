## Summary

Keep open: current main still intentionally preserves or emits home-relative paths in several assistant/output display surfaces and lacks a central all-absolute-path contract, so this needs product direction rather than cleanup closure.

## What This Changes

Issue asks that agent responses use full absolute file paths instead of home-relative `~` shorthand.

## Best Possible Solution

Define the intended user-visible path style centrally, then update the prompt/display helpers and focused tests without breaking documented `MEDIA:~/...` and sandbox-relative path contracts.

## Reproduction Assessment

No high-confidence end-to-end reproduction is available for ordinary prose replies. Source inspection does show current main still preserves or emits `~/` in shared display helpers, assistant media normalization, and approval/tool display contexts.

## Solution Assessment

Unclear. A scoped central prompt or display-policy change is likely safer than blanket output rewriting, but maintainers first need to decide which surfaces should use full absolute paths, repo-root paths, relative paths, or documented `~/` media paths.
