## Summary

Keep open: current main still has no central rule requiring full absolute paths in assistant replies, and changing that would be a product policy decision because existing path display, sandbox guidance, and media contracts still allow or emit home-relative paths.

## What This Changes

Request that agent replies show full absolute file paths instead of home-relative `~/` shorthand.

## Best Possible Solution

Define the final-answer path-style policy explicitly, then implement it centrally in prompt guidance with focused coverage while preserving intentional `MEDIA:` and tool-path contracts.

## Reproduction Assessment

No high-confidence live reproduction exists. Source inspection confirms current main still shortens home paths and lacks a full-absolute reply rule, but there is no deterministic transcript or failing test for the reported assistant reply behavior.

## Solution Assessment

Unclear. A scoped central prompt directive would be safer than blanket text rewriting if maintainers want the policy, but the repo currently documents and tests some home-relative path behavior as valid.
