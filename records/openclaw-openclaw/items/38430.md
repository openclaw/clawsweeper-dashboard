## Summary

Keep open. Current main still preserves or produces `~/...` paths on adjacent user-visible surfaces and has no final-reply expansion contract; the related #38495 implementation was closed unmerged, so this still needs maintainer scope judgment.

## What This Changes

The issue requests a user-visible path policy that expands home-relative `~/...` paths to full absolute paths in assistant replies.

## Best Possible Solution

Define the user-visible path-display contract first, then implement a narrow final-reply solution with regression coverage that preserves internal prompt compaction and accepted tilde tool inputs.

## Reproduction Assessment

Yes, at source level. Current main has no final-reply tilde expansion step, and the host edit recovery regression suite still expects a tilde-form success message; tests were not run because this review is read-only.

## Solution Assessment

Unclear. A final-reply-specific formatter or explicit final-answer prompt policy looks safer than the unmerged #38495 environment-variable approach, but maintainers need to decide the exact path-display boundary.
