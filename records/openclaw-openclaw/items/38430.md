## Summary

Keep open: current main and the latest release still intentionally shorten home paths across shared utilities, tool display, browser UI, and native display, while media docs/tests deliberately allow home-relative paths. The linked env-var PR was closed unmerged and did not settle the broader path-presentation policy.

## What This Changes

Request that agent-facing file path output use full absolute paths instead of tilde shorthand, either through prompt guidance or output rewriting.

## Best Possible Solution

Define the intended path-presentation policy per surface, then update prompt, display, media, client helpers, and focused tests to match without breaking documented media or sandbox contracts.

## Reproduction Assessment

No high-confidence end-to-end transcript was established for ordinary assistant prose. Source inspection does show current main and v2026.5.12 still shorten or preserve home-relative paths on shared display, tool display, native client, and media surfaces.

## Solution Assessment

Unclear. A maintainer-defined policy with targeted prompt/display/media changes is safer than a blanket output rewrite or reviving the closed env-var PR before deciding which surfaces should remain home-relative.
