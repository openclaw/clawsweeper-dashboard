## Summary

Keep open: current main still deliberately shortens or preserves home-relative paths across shared display helpers, tool display, browser/native clients, and media handling, so this needs a maintainer path-presentation policy decision rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to stop emitting `~/...` in agent-visible file paths and use full absolute paths instead.

## Best Possible Solution

Define the intended path-presentation policy for prose, tool summaries, media directives, and native/browser clients, then update shared helpers/prompts and tests without breaking documented home-relative media or sandbox path behavior.

## Reproduction Assessment

No. I did not establish a high-confidence end-to-end assistant transcript for ordinary prose, but source inspection shows current-main tool display and media/client surfaces still shorten or preserve home-relative paths.

## Solution Assessment

Unclear. A maintainer-defined policy per surface is safer than blanket post-processing or the closed env-var bypass because current docs/tests intentionally support `MEDIA:~/...` and sandbox prompt guidance prefers relative paths.
