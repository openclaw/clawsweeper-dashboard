Keep open: current main and the latest release still intentionally shorten or preserve home-relative paths across shared display helpers, tool summaries, skill prompts, media directives, approval display, and UI surfaces. ## What This Changes

Requests that agent responses show full absolute filesystem paths instead of `~/` shorthand.

## Best Possible Solution

Define and document a per-surface path-presentation contract first, then update only the agent prose, tool summaries, skill prompts, media directives, approval UI, or clients that should show absolute paths.

## Reproduction Assessment

No, not for the full end-to-end agent-prose behavior from the report. Source inspection does show deterministic current-main `~/` compaction or preservation across the relevant display, prompt, media, approval, and client paths.

## Solution Assessment

Unclear. A blanket prompt rule, post-processor, or env var is not obviously the safest fix because current docs and tests rely on `~/`; a maintainer-defined surface policy should come first.
