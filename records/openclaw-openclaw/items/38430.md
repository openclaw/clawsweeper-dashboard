Keep open: current main and the latest release still intentionally shorten or preserve home paths with `~/` across shared display, tool summary, skill prompt, approval, and media surfaces. The related fix PR was closed unmerged, so this remains a maintainer path-presentation policy decision rather than a conservative cleanup close.

## What This Changes

Request agent responses to emit full absolute filesystem paths instead of `~/` shorthand.

## Best Possible Solution

Define and document a surface-specific path presentation contract, then update only the assistant/user-facing surfaces that should show copy-paste absolute paths while preserving documented `~/` inputs where intended.

## Reproduction Assessment

No, not for the full end-to-end agent-prose behavior in the report. Source inspection does show deterministic current-main `~/` compaction or preservation across shared display, prompt, media, approval, and UI paths.

## Solution Assessment

Unclear. A blanket prompt rule, post-processor, or env var is not obviously the safest fix because current docs and tests rely on `~/`; maintainers should define the per-surface contract first.
