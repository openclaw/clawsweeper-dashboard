## Summary

Keep open: current main does not implement a default rule expanding or forbidding `~/` in user-visible agent replies, and the related PR was closed unmerged. This is better treated as an assistant-output policy request than a proven regression.

## What This Changes

The issue requests that agent responses use full absolute file paths instead of `~` shorthand.

## Best Possible Solution

Make an explicit maintainer decision on user-visible path style, then implement the chosen policy in the central agent prompt and prompt snapshot coverage rather than blanket rewriting assistant text.

## Reproduction Assessment

No. Source inspection shows OpenClaw does not currently enforce the requested path style, but there is no high-confidence live reproduction or failing current-main test showing a model response emitting `~/...`.

## Solution Assessment

Unclear. A scoped central prompt directive would likely be safer than post-processing model output, but the repository first needs a product decision because current prompt/docs intentionally support relative and home-relative paths in some contexts.
