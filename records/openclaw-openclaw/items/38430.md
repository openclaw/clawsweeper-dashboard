## Summary

Keep open: current main still intentionally shortens or preserves home-relative paths across shared display helpers, skill prompt locations, tool display, native clients, and media contracts, so this needs maintainer path-presentation policy rather than cleanup closure.

## What This Changes

The issue requests that agent-visible file paths be emitted as full absolute paths instead of `~/` home-relative shorthand.

## Best Possible Solution

Define a per-surface path-presentation contract for assistant prose, tool/client summaries, skill prompt locations, media directives, and sandbox paths, then update only the surfaces that should stop using home-relative display.

## Reproduction Assessment

No, not for the full agent-prose behavior; I did not establish a deterministic current-main transcript. Source inspection does show deterministic tilde compaction or preservation in display helpers, skill prompts, media handling, and native/web clients.

## Solution Assessment

Unclear. A targeted maintainer-defined output policy is safer than a blanket post-processor or reviving the closed environment-variable PR before deciding which surfaces should remain home-relative.
