The issue asks OpenClaw to stop presenting home-directory path shorthand in agent outputs and instead use expanded absolute paths.

## Best Possible Solution

Unclear. A blanket prompt rule or output post-processor is not obviously the best fix because current tests, prompt construction, UI rendering, and SDK helpers intentionally support compact home-relative display.
