## What This Changes

The issue asks OpenClaw to present agent-referenced file paths as full absolute paths instead of `~/` shorthand.

## Best Possible Solution

Define a surface-specific path presentation contract, then update only the agent/user-facing surfaces that should show absolute paths while preserving documented home-relative inputs and intentional prompt compaction.

No, not for the full end-to-end ordinary agent-prose behavior from the report. Source inspection does reproduce deterministic current-main `~/` generation or preservation in tool summaries, UI summaries, skill prompt locations, and media directive handling.

## Solution Assessment

Unclear. A blanket prompt rule, env var, or output post-processor is not clearly the best fix because current docs and tests rely on home-relative paths; maintainers should choose the per-surface contract first.
