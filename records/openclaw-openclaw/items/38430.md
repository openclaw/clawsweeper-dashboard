## Summary

Keep open: current main and the latest release still intentionally compact or preserve home-directory paths as `~/` across shared display helpers, tool summaries, skill prompts, media rendering, and UI display. ## What This Changes

The issue requests that agent responses show full absolute file paths instead of `~` home-directory shorthand.

## Best Possible Solution

Define a per-surface path-presentation contract, then update only the agent prose, tool summaries, skill prompts, media directives, and clients that should show absolute paths.

No high-confidence live reproduction path for the full agent-prose behavior was established. Source inspection does show deterministic current-main `~/` compaction or preservation across shared display, skill prompt, media, and UI paths.

## Solution Assessment

Unclear. A blanket prompt rule, output post-processor, or env var is not obviously the best fix because current docs and tests intentionally allow `~/`; a maintainer-defined per-surface policy is safer.
