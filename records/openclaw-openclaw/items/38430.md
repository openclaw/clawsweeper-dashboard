## Summary

Keep open: current main still deliberately shortens home-prefixed paths to `~/` in shared display helpers, tool summaries, UI/mobile display code, and skill prompt locations. The linked implementation PR was closed unmerged, so this remains a maintainer product decision rather than a resolved bug.

## What This Changes

The issue asks for agent-visible file path references to use full absolute paths instead of `~/` shorthand.

## Best Possible Solution

Define a surface-specific path presentation contract, then update only the agent/user-facing surfaces that should show copy-paste absolute paths while preserving documented `~/` input support where intended.

## Reproduction Assessment

No high-confidence end-to-end agent-prose reproduction was run. Source inspection does show deterministic current-main `~/` compaction or preservation across shared display, prompt, media, tool-summary, and UI paths.

## Solution Assessment

Unclear. A blanket prompt rule, post-processor, or env var is not obviously the best fix because current docs and tests rely on `~/`; maintainers should define the per-surface contract first.
