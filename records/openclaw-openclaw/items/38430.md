## Summary

Keep open: current main still has no central product rule requiring agent replies to use full absolute paths, and existing docs/tests intentionally preserve `~/` in some assistant-output contexts, so this is not a cleanup-close candidate.

## What This Changes

The issue requests a global agent-output policy that avoids `~/...` home shorthand and uses full absolute paths in user-visible file references.

## Best Possible Solution

Make an explicit maintainer decision for final-answer path style, then encode the chosen rule in the central prompt and focused tests while preserving documented media and sandbox path contracts.

## Reproduction Assessment

No. Source inspection shows the absence of a central all-absolute-path output rule and confirms `~/` remains accepted in assistant media paths, but there is no deterministic transcript or live repro for ordinary prose file references on current main.

## Solution Assessment

Unclear. A scoped central prompt directive is likely safer than blanket output rewriting if maintainers want this behavior, but the policy has to account for existing home-relative media and sandbox-relative path guidance.
