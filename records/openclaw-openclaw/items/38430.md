## Summary

Keep this issue open. Current main still has no final-reply policy or delivery-time formatter that expands `~/...` before user-visible agent replies, while several adjacent surfaces intentionally compact or preserve home-relative paths; the related #38495 implementation was closed unmerged.

## What This Changes

Request that user-visible agent replies expand tilde home-path shorthand to full host paths instead of emitting `~/...`.

## Best Possible Solution

Define the desired user-visible reply contract, then implement a narrow final-reply solution with regression coverage that expands only assistant-delivered path references while preserving internal prompt compaction and accepted tilde tool inputs.

## Reproduction Assessment

Yes. A source-level reproduction is clear: final reply normalization has no tilde expansion step, and the host edit recovery test currently expects a tilde-form success message; I did not run tests because this review was required to stay read-only.

## Solution Assessment

Unclear. The unmerged #38495 env-var approach is useful prior art, but a broad `src/utils.ts` switch is likely too wide; a final-reply-specific formatter or explicit final-answer prompt policy would be safer after maintainer scope confirmation.
