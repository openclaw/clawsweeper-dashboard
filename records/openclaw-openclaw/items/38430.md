## Summary

Keep open: current `main` and the latest release still intentionally shorten or preserve home-relative paths across shared helpers, tool summaries, skill prompts, media directives, and clients. This is not closeable as implemented; choosing what should change needs maintainer path-presentation policy rather than a blanket formatter change.

## What This Changes

Request that agent-visible file paths be shown as full absolute paths instead of home-relative `~/` shorthand.

## Best Possible Solution

Define a per-surface path-presentation contract for assistant prose, tool/client summaries, skill prompt locations, media directives, approval UI, and sandbox paths, then update only the surfaces that should stop using home-relative display.

## Reproduction Assessment

No, not for the full end-to-end agent-prose behavior. Source inspection does show deterministic `~` compaction or preservation in the shared helpers, tool summaries, skill prompts, media handling, and native/web clients.

## Solution Assessment

Unclear. A targeted maintainer-defined path-presentation policy is safer than reviving the closed env-var PR or applying a blanket post-processor across surfaces that currently rely on `~`.
