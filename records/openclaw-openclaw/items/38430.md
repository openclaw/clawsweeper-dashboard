## Summary

Keep open: current `main` and the latest release still intentionally compact or preserve home-relative paths in shared display helpers, tool summaries, skill prompt locations, native clients, and media contracts. The linked env-var fix at https://github.com/openclaw/openclaw/pull/38495 was closed unmerged, and this now needs a maintainer path-presentation policy rather than cleanup closure.

## What This Changes

Request that agent-facing file path output use full absolute paths instead of home-relative `~` shorthand.

## Best Possible Solution

Define a per-surface path-presentation contract for assistant prose, tool/client summaries, skill prompt locations, media directives, approval UI, and sandbox paths, then update only the surfaces that should stop using home-relative display.

## Reproduction Assessment

No, not for the full end-to-end agent-prose behavior; I did not establish a deterministic current-main transcript. Source inspection does show deterministic tilde compaction or preservation in shared display helpers, tool summaries, skill prompts, media handling, and native/web clients.

## Solution Assessment

Unclear. A targeted maintainer-defined path-presentation policy is safer than reviving the closed env-var PR or applying a blanket post-processor across surfaces that currently rely on `~`.
