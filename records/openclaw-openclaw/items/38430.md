## Summary

Keep open because current main and the latest release still shorten home-directory paths in OpenClaw-generated tool/progress displays, while the default agent prompt only says to show file paths clearly and does not require full absolute paths. The linked implementation PR closed unmerged, so the remaining work is a maintainer product decision about path-display policy rather than conservative cleanup.

## What This Changes

The issue requests that user-visible agent path references use full absolute paths instead of `~/...` home-directory shorthand.

## Best Possible Solution

Decide a single user-visible path display policy per surface, then implement it narrowly in the prompt and display helpers with focused tests; avoid a broad new env var unless maintainers explicitly want an opt-in mode.

## Reproduction Assessment

Medium confidence from source: OpenClaw-generated tool and progress metadata under the effective home is shortened through `shortenHomePath` and `shortenHomeInString`. The model-authored final-prose behavior is not live-reproduced because the issue has no current transcript, command, or screenshot.

## Solution Assessment

No. The prior env-var PR was a plausible opt-in, but the best solution needs a maintainer-approved display policy across prompts, tool summaries, channel progress, and final replies before implementation.
