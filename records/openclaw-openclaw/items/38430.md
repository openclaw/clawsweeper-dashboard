## Summary

Keep this open: current main still intentionally shortens home paths in OpenClaw-generated progress/status text, and the request now needs a maintainer product decision because repo guidance prefers repo-root references while the issue asks for full absolute paths.

## What This Changes

The issue asks OpenClaw agent replies to avoid `~` home shorthand and provide copy-pasteable absolute file paths instead.

## Best Possible Solution

Decide the product convention first, then make a narrow boundary change: keep compact `~` where it is intentional diagnostic/progress display, and adjust only agent-facing final-reply guidance or formatter paths if maintainers want copy-paste paths to avoid `~`.

## Reproduction Assessment

Yes at source level for OpenClaw-generated progress/status text: current tests and formatters intentionally shorten home paths to `~/...`. No live reproduction proves that model-authored final prose always defaults to tilde shorthand.

## Solution Assessment
