## Summary

Keep this PR open for normal maintainer review: the SDK direction is plausible, but the patch is not merge-ready because contributor real behavior proof is missing and `sendToSession` currently defaults non-main sessions to the `main` agent identity.

## What This Changes

The PR adds Plugin SDK runtime methods for gateway-backed session cancellation and direct outbound text delivery to a session's stored channel route.

## Best Possible Solution

Keep the PR open, fix the session identity default, document the new public runtime methods, and add real gateway-backed behavior proof for cancel plus send-to-session.

## Reproduction Assessment

Not applicable: this PR adds new Plugin SDK runtime capabilities rather than reporting broken current behavior. Source inspection confirms current main does not already expose these two methods.

## Solution Assessment

No, not as-is: the proposed seam is plausible, but `sendToSession` should preserve the target session's agent identity and the public API needs docs and real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Use the target session's agent identity:** `src/plugins/runtime/index.ts:257`
  - body: `sendToSession` falls back to `resolveAgentOutboundIdentity(cfg, "main")` whenever the caller omits an identity. For any `agent:<id>:...` session that is not `main`, this delivers the preemption/status text with the wrong agent name/avatar in channels that support identity metadata; derive the agent id from `sessionKey` or require the identity explicitly.
  - confidence: 0.86
- **[P3] Document the new public runtime methods:** `src/plugins/runtime/types-core.ts:210-223`

Please document `runtime.session.cancel()` and `runtime.channel.outbound.sendToSession()`, including `not-wired` and `no-route-found`, so plugin authors do not have to infer the contract from types.
  - confidence: 0.78
