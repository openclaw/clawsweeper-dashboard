## Summary

Keep open: the previous identity and docs findings were addressed, but the current diff still makes `sendToSession` read the default main session store before resolving the target agent, so non-main agent session routes can incorrectly return `no-route-found`; the PR also still lacks sufficient after-fix real behavior proof.

## What This Changes

This PR adds `runtime.session.cancel`, `runtime.channel.outbound.sendToSession`, gateway cancellation wiring, docs, mocks, tests, and Plugin SDK API baseline updates.

## Best Possible Solution

Keep the narrow host-mediated runtime seam, but derive both route store and default identity from the target `sessionKey`, document the contract, and require real gateway-backed proof before merge.

## Reproduction Assessment

Not applicable: this PR adds new Plugin SDK capabilities rather than reporting broken existing behavior. The blocking review finding is source-reproducible from the current per-agent session-store contract and the PR diff.

## Solution Assessment

No, not as-is: the proposed seam is plausible, but `sendToSession` must load the target agent's session store and the PR needs real behavior proof plus maintainer API review before merge.
