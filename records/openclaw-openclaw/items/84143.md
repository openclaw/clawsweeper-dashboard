## Summary

Close: the local/self-hosted realtime STT use case is valid, but current main already exposes realtime transcription and realtime voice provider-plugin seams and Talk consumes registered provider-owned config, so this belongs as a ClawHub provider plugin rather than a new bundled core OpenAI baseUrl path.

## What This Changes

Request support for local/self-hosted STT in Talk Mode by adding configurable custom endpoints for realtime and Gateway relay providers.

## Best Possible Solution

Publish a ClawHub realtime transcription or realtime voice provider plugin for the local ASR server, and use the existing native ASR tracking issue only if the plugin path exposes a concrete missing core SDK hook.

## Reproduction Assessment

Not applicable; this is a feature request, not a report of broken documented behavior. Source and docs inspection show the realtime route is already extensible through provider-plugin registration, while native ASR endpoint configuration is tracked separately.

## Solution Assessment

No for the proposed core/baseUrl direction. The maintainable solution is a ClawHub provider plugin using the current realtime transcription or realtime voice contracts, with a separate native ASR provider seam only if maintainers accept that product scope.
