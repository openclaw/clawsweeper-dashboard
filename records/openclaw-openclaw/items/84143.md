## What This Changes

The issue requests local or self-hosted STT support for Talk Mode realtime voice, including configurable OpenAI-compatible realtime base URLs and gateway relay endpoints.

## Best Possible Solution

Publish a local/self-hosted realtime STT or realtime voice provider plugin through ClawHub, and open a narrow SDK/API issue only if that implementation proves a missing Talk provider contract.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection does confirm the requested core `baseUrl` behavior is not part of the bundled OpenAI realtime provider, while the plugin SDK already has provider seams for this kind of capability.

## Solution Assessment

Yes for closing the core item: a provider plugin is the maintainable boundary for self-hosted realtime STT/voice services. Adding an arbitrary local endpoint knob to the bundled OpenAI provider would blur provider ownership and security review boundaries.
