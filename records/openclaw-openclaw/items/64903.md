Keep open. Current main and the latest release still match the reported Android 15/16 dataSync foreground-service quota crash path: the service starts/promotes with dataSync, does not handle startForeground denial, and has no Service.onTimeout handling.

## What This Changes

The issue requests hardening the Android NodeForegroundService startup path so dataSync foreground-service quota exhaustion does not crash the Android app.

## Best Possible Solution

Harden the Android foreground-service lifecycle so dataSync quota exhaustion stops or demotes cleanly, handles Android 15+ timeout callbacks, and avoids restart loops without changing unrelated gateway behavior.

## Solution Assessment
