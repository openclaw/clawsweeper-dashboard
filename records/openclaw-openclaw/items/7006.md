## Summary

Current main still does not expose the OpenRouter auto-routed model or provider-reported cost in the response footer/runtime metadata. The existing code keeps the selected `openrouter/auto` identity and local estimated-cost path, while related cost/model-transparency items remain open rather than superseding this request.

## What This Changes

The issue requests exposing the concrete OpenRouter model selected by `openrouter/auto` and its provider-reported cost so response footers can show the resolved model and spend.

## Best Possible Solution

Add a versioned runtime metadata path that captures OpenRouter response `model` and provider cost, preserves selected-versus-resolved identity, and threads the data through footers/status/transcripts with targeted `openrouter/auto` regression coverage while coordinating the related cost and actual-model issues.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes by source inspection: current main initializes and propagates the selected model/cost estimate only, while the OpenRouter response `model` and `usage.cost`/`cost_details` fields have no parsed or displayed path.

## Solution Assessment

Is this the best way to solve the issue? The requested direction is valid, but the best implementation needs maintainer agreement on the runtime metadata contract and cost authority before a narrow patch should pick field names and footer behavior.
