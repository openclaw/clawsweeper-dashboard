## Summary

Keep open: current main still keeps `openrouter/auto` as a catalog/selected-model placeholder, ignores returned OpenRouter model and raw provider cost fields in the shared stream parser, and formats response usage footers from local estimates only. Related open items cover parts of the work, but no merged or canonical item fully replaces this OpenRouter auto resolved-model plus cost request.

## What This Changes

Expose the actual OpenRouter routed model and provider-reported cost for `openrouter/auto` so agent/runtime footers can display the resolved model and cost.

## Best Possible Solution

Add additive selected-versus-resolved model metadata and provider-reported cost fields, capture them at the OpenAI-compatible/OpenRouter response boundary, and thread them through transcripts, session/status/runtime metadata, response footers, docs, and targeted regression tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main does not expose returned OpenRouter model metadata or provider-reported cost on the requested runtime/footer surfaces.

## Solution Assessment

Unclear until maintainers choose the public selected-versus-resolved model and provider-cost contract. The narrow maintainable direction is additive metadata plus a distinct provider-reported cost object, not a footer-only patch or overloading the existing estimated-cost fields.
