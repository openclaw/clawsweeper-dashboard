## Summary

Current main still leaves the requested OpenRouter auto transparency gap open: OpenClaw keeps the selected router model, computes local estimated cost, and does not carry OpenRouter's returned routed model or provider-reported cost into agent/session/footer metadata.

## What This Changes

Expose the actual model selected by OpenRouter for `openrouter/auto` plus its provider-reported usage cost so agents and response footers can show accurate per-turn model and cost metadata.

## Best Possible Solution

Carry selected model, resolved OpenRouter response model, and provider-reported usage cost as distinct metadata from the OpenAI-compatible transport through assistant/session records, footers, transcripts, and status/usage surfaces, with focused `openrouter/auto` regression coverage.

## Reproduction Assessment

Yes, from source inspection: `openrouter/auto` still flows through as the selected `model.id`, parsed usage ignores provider cost fields, and the response footer has no resolved-model/provider-cost inputs. I did not run live OpenRouter traffic in this read-only review.

## Solution Assessment

Unclear until maintainers choose the product/API contract. The maintainable direction is to preserve selected model, resolved model, local estimate, and provider-reported cost as separate fields rather than overwriting one with another or adding only a footer special case.
