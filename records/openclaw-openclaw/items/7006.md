## Summary

Keep open: current main still keeps `openrouter/auto` as the selected placeholder and has no resolved OpenRouter model or provider-reported cost metadata path for agent/runtime footers.

## What This Changes

Expose the actual OpenRouter-routed model and usage cost when `openrouter/auto` is selected so replies can report the resolved model and cost.

## Best Possible Solution

Add an additive runtime metadata contract that preserves selected versus resolved OpenRouter model identity and provider-reported cost with explicit source and units, then thread it through transport, transcripts, session/status surfaces, response footers, and targeted `openrouter/auto` tests.

## Reproduction Assessment

Yes for the observable feature gap by source inspection: current main keeps `openrouter/auto` as the stored model, parses local estimated cost from token counters, and formats the footer without resolved-model or provider-cost inputs. This is not a strict bug reproduction because the issue requests a new metadata/display capability.

## Solution Assessment

Unclear until maintainers choose the public metadata and cost-authority contract. The narrow maintainable direction is additive selected/resolved model metadata plus a distinct provider-reported cost object, not overloading existing estimated-cost fields.
