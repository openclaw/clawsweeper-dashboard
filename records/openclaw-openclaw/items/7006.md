## Summary

Keep open: current main still keeps `openrouter/auto` as the selected placeholder and does not carry OpenRouter's returned model or provider-reported cost into agent, session, or response-footer surfaces.

## What This Changes

The issue requests exposing the concrete model and cost selected by OpenRouter Auto so replies can show a resolved model label and accurate cost instead of only `openrouter/auto`.

## Best Possible Solution

Define additive selected-versus-resolved model metadata and provider-reported cost fields, then capture OpenRouter `response.model`/`chunk.model` and `usage.cost`/`cost_details` at the OpenAI-compatible transport boundary and thread them through runtime, transcripts, sessions, status, and footers.

## Reproduction Assessment

Yes for the observable feature gap by source inspection: current main stores `openrouter/auto`, parses local estimated cost from token counters, and formats the footer without resolved-model or provider-cost inputs. This is not a strict bug reproduction because the item requests a new metadata/display capability.

## Solution Assessment

Unclear until maintainers choose the public selected-versus-resolved model and provider-cost contract. The narrow maintainable direction is additive metadata plus a distinct provider-reported cost object, not overloading existing estimated-cost fields or landing a footer-only patch.
