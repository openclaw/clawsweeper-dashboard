## Summary

Current main still does not expose OpenRouter’s routed model or provider-reported cost through agent or response surfaces, and the focused prior implementation closed unmerged while broader related issues remain unresolved. Keep this issue open for a consolidated maintainer decision.

## What This Changes

Expose the concrete model selected by `openrouter/auto` and its provider-reported usage cost to agents and response footers.

## Best Possible Solution

Define a provider-agnostic per-turn metadata contract for requested model, resolved model, and explicit cost provenance; capture it at the OpenAI-compatible transport boundary and reuse it consistently across agent, session, status, transcript, and optional response-footer surfaces.

## Reproduction Assessment

Yes, at source level: current main clearly retains the configured model identity and computes locally estimated cost without propagating OpenRouter's resolved model or reported cost. This read-only review did not execute a credential-backed live OpenRouter request.

## Solution Assessment

No, a footer-specific OpenRouter special case is not the best solution. A shared per-turn routing and cost-provenance contract is narrower architecturally and avoids multiple conflicting sources of truth.
