## Summary

Keep this open: current main still treats `openrouter/auto` as the selected model, computes local estimated cost from token counters, and has no resolved OpenRouter model or provider-reported cost path for the agent/runtime footer.

## What This Changes

Expose the OpenRouter auto-routed model name and provider-reported usage cost to agents and per-response footers.

## Best Possible Solution

Add versioned runtime metadata that preserves selected versus resolved OpenRouter model identity and provider cost, then thread it through transcripts, session/status surfaces, and response footers with targeted `openrouter/auto` coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes by source inspection: current main keeps `openrouter/auto` as the stored model, parses normalized token usage/local estimated cost, and formats the footer without resolved-model or provider-cost inputs.

## Solution Assessment

Is this the best way to solve the issue? The requested direction is valid, but the best fix needs maintainer agreement on the metadata and cost-authority contract before choosing field names and display behavior.
