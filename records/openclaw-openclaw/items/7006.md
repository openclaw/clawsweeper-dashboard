## Summary

Keep open: current main still preserves the selected `openrouter/auto` model and local estimated-cost path, but it does not capture OpenRouter's routed model or provider-reported cost for agent/session/status or response-footer use. Related open work covers overlapping model and cost slices, but it does not replace this specific OpenRouter auto transparency request.

## What This Changes

The issue requests exposing the actual model and cost returned when OpenRouter routes `openrouter/auto`, including response-footer display.

## Best Possible Solution

Define additive selected-versus-resolved model and provider-reported-cost metadata at the OpenAI-compatible/OpenRouter boundary, preserve local estimates separately, and surface the data consistently through assistant/session records, response footers, status/usage tools, transcripts, and focused OpenRouter auto tests.

## Reproduction Assessment

Yes. Source inspection on current main shows `openrouter/auto` is stored as `model.id`, streamed `chunk.model` is not captured, OpenRouter cost fields are ignored, and the response footer has no resolved-model or provider-cost inputs.

## Solution Assessment

Unclear until maintainers choose the public metadata and UX contract. The safest direction is additive metadata that preserves selected model, resolved model, local estimate, and provider-reported cost separately instead of overwriting existing fields or adding only a footer special case.
