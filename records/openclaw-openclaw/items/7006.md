## Summary

Keep open: current main still preserves the selected `openrouter/auto` model and local estimated-cost path, but it does not capture OpenRouter's returned routed model or provider-reported cost for agent, session, status, or response-footer use.

## What This Changes

Expose the actual model selected by OpenRouter `openrouter/auto` and its provider-reported usage cost to agents and response footers.

## Best Possible Solution

Define additive selected-versus-resolved model and provider-reported-cost metadata at the OpenAI-compatible/OpenRouter response boundary, then surface it consistently through assistant records, session persistence, response footers, status/usage tools, transcripts, and focused OpenRouter auto tests.

## Reproduction Assessment

Yes. Source inspection on current main shows `openrouter/auto` is stored as `model.id`, streamed `chunk.model` is not captured, OpenRouter cost fields are ignored, and the response footer has no resolved-model or provider-cost inputs.

## Solution Assessment

Unclear until maintainers choose the public metadata and UX contract. The safest direction is additive metadata that preserves selected model, resolved model, local estimate, and provider-reported cost separately instead of overwriting existing fields or adding only a footer special case.
