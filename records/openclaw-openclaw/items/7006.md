## Summary

Keep open: current main still does not expose the OpenRouter auto-routed model or provider-reported cost through the agent/runtime footer path, and the related open cost/model-transparency issues are complementary rather than replacements.

## What This Changes

Expose the concrete model selected by `openrouter/auto` and its usage cost to the agent/response footer instead of showing only the selected auto model and local estimates.

## Best Possible Solution

Add a versioned runtime metadata path that captures OpenRouter response `model` and provider cost, preserves selected-versus-resolved identity, and threads it through footers, status/session/transcript surfaces with targeted `openrouter/auto` coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes by source inspection: current main keeps `openrouter/auto` as the selected model, parses only normalized token usage/local estimated cost, and formats the footer without resolved-model or provider-cost inputs.

## Solution Assessment

Is this the best way to solve the issue? The requested direction is valid, but the best implementation needs maintainer agreement on the runtime metadata and cost-authority contract before a narrow repair chooses field names and display behavior.
