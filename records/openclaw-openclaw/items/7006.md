## Summary

Keep open. Current main still keeps `openrouter/auto` as the selected model through the OpenAI-compatible transport and response footer, and it recomputes local estimated cost instead of preserving OpenRouter's returned resolved model and provider cost.

## What This Changes

Expose the actual model selected by OpenRouter Auto and its provider-reported usage cost to agents and response usage footers.

## Best Possible Solution

Capture OpenRouter's returned model and provider cost at the OpenAI-compatible transport boundary, preserve selected-versus-resolved model metadata, and render the resolved Auto label plus provider-reported cost in usage/session/status surfaces with focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed an OpenAI-compatible OpenRouter final stream chunk with top-level `model` and numeric `usage.cost`; current main ignores those fields and the footer can only render local token estimates.

## Solution Assessment

Yes. Keeping this open and fixing the transport-to-footer metadata path is the narrowest maintainable solution; ClawRouter and the closed #43026 are useful prior art, while #9016, #68066, and #22021 cover adjacent surfaces rather than a shipped replacement.
