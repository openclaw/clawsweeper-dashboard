## Summary

Keep open: current main still treats `openrouter/auto` as the selected placeholder in the OpenAI-compatible transport/footer path and does not expose OpenRouter provider-reported cost; the related open items cover pieces but do not replace this combined model-and-cost transparency request.

## What This Changes

Request to expose the OpenRouter-routed model and actual/provider-reported usage cost for `openrouter/auto` so agents can show them in response footers.

## Best Possible Solution

Define an additive selected-vs-resolved model and provider-cost metadata contract, capture OpenRouter response model and raw usage at the shared transport boundary, then thread it through agent/session/footer surfaces with targeted OpenRouter auto coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction. Source inspection gives high-confidence proof of the feature gap: current main keeps the selected `openrouter/auto` model and local estimated-cost path without resolved-model or provider-cost metadata.

## Solution Assessment

Unclear until maintainers choose the public metadata and cost-authority contract. The narrow maintainable direction is additive resolved-model metadata plus a distinct provider-reported cost object, not overloading existing estimated-cost fields.
