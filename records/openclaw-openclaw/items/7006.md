## Summary

Keep open: current main still preserves `openrouter/auto` as the selected placeholder, ignores OpenRouter returned-model/provider-cost fields in the shared transport, and renders response usage from local estimates. The related open issues cover adjacent cost/model visibility work, while the prior footer PR was closed unmerged and does not supersede this combined resolved-model-plus-cost request.

## What This Changes

Expose the actual OpenRouter model selected for `openrouter/auto` and its provider-reported usage cost to agents and response footers.

## Best Possible Solution

Define an additive resolved-model and provider-reported cost contract, capture OpenRouter response `model` plus raw usage cost fields at the OpenAI-compatible/OpenRouter boundary, and thread them through agent metadata, transcripts, session/status surfaces, response footers, docs, and targeted tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main still lacks the requested OpenRouter returned-model and provider-cost metadata surfaces.

## Solution Assessment

Unclear until maintainers choose the public metadata and cost-authority contract. The maintainable direction is additive selected-versus-resolved model and provider-cost metadata, not a footer-only patch or overloading existing estimate fields.
