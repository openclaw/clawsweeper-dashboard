## Summary

Keep open: current main and v2026.6.5 only capture OpenRouter's routed model as low-level `responseModel`, while resolved model and provider-reported cost still do not reach agent metadata, session/status state, or response usage footers.

## What This Changes

Expose the actual OpenRouter-routed model and provider-reported cost for `openrouter/auto` to agents and response usage footers.

## Best Possible Solution

Define additive resolved-model and provider-reported-cost metadata at the OpenAI-compatible/OpenRouter boundary, then surface it consistently through agent metadata, session/status records, transcripts, usage tools, and reply footers.

## Reproduction Assessment

Yes, source-level. Current main captures `chunk.model` into `responseModel`, but agent metadata, session/status types, and reply footers still lack resolved-model and provider-reported-cost fields.

## Solution Assessment

No, current main is only a partial foundation. The best fix is an additive cross-surface metadata contract, not a footer-only special case or silently replacing the selected model.
