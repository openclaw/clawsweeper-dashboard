## Summary

Keep open: current main still keeps `openrouter/auto` as the selected catalog model, ignores OpenRouter's returned routed model and raw provider cost fields in the shared OpenAI-compatible stream path, and renders response usage footers from local estimates only. The related open issues and PRs cover parts of the same area, but none fully replaces this combined resolved-model plus provider-cost transparency request.

## What This Changes

Expose the actual model selected by OpenRouter for `openrouter/auto` and its provider-reported cost to agents and response footers.

## Best Possible Solution

Define an additive selected-versus-resolved model and provider-reported cost contract, capture OpenRouter response `model` and raw usage cost fields at the OpenAI-compatible/OpenRouter boundary, and thread them through agent metadata, transcripts, session/status surfaces, response footers, docs, and targeted tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main does not expose OpenRouter returned model metadata or provider-reported cost on the requested runtime and footer surfaces.

## Solution Assessment

Unclear until maintainers choose the public metadata and cost-authority contract. The safer direction is additive resolved-model and provider-cost metadata, not overloading existing estimated USD fields or applying a footer-only patch.
