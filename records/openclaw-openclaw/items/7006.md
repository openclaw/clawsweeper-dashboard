## Summary

Current main still has the OpenRouter auto transparency gap: OpenClaw preserves the selected `openrouter/auto` identity and local estimated-cost path, but it does not carry OpenRouter's returned routed model or provider-reported cost into agent/session/footer metadata.

## What This Changes

The issue requests exposing the actual model and cost returned by OpenRouter when `openrouter/auto` is selected, so agents and response footers can report the routed model and cost.

## Best Possible Solution

Define an additive selected-versus-resolved model and provider-reported-cost contract, capture OpenRouter response `model` and usage cost fields in the OpenAI-compatible transport, and thread them through assistant/session records, response footers, transcripts, status/usage surfaces, and focused `openrouter/auto` regression tests.

## Reproduction Assessment

Yes, from source inspection: `openrouter/auto` still flows through as the selected `model.id`, parsed usage ignores provider cost fields, and the response footer has no resolved-model/provider-cost inputs. I did not run live OpenRouter traffic in this read-only review.

## Solution Assessment

Unclear until maintainers choose the API and UX contract. The maintainable direction is additive metadata that preserves selected model, resolved model, local estimate, and provider-reported cost separately instead of overwriting existing fields or adding only a footer special case.
