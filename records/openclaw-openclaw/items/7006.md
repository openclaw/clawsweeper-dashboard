## Summary

Keep open: current main and v2026.5.22 still keep `openrouter/auto` as the selected model and local estimated-cost path, without carrying OpenRouter's returned response model or provider-reported cost into agent/session/status/footer surfaces. Related open issues cover cost accounting and broader resolved-model metadata, but no merged fix or single canonical item fully replaces this specific OpenRouter auto transparency request.

## What This Changes

Request to expose the actual model and usage cost selected by OpenRouter when `openrouter/auto` is used, especially for response footers and agent-visible reporting.

## Best Possible Solution

Define additive selected-versus-resolved model and provider-reported-cost metadata at the OpenAI-compatible/OpenRouter boundary, preserve local estimates separately, and surface it consistently through assistant/session records, response footers, status/usage tools, transcripts, and focused OpenRouter auto regression tests.

## Reproduction Assessment

Yes, at source level. Current main stores `openrouter/auto` as `model.id`, does not capture provider response `model`, ignores OpenRouter cost fields, and formats the response footer from local token/cost inputs only.

## Solution Assessment

Unclear until maintainers choose the public metadata and UX contract. The safest direction is additive metadata that keeps selected model, resolved model, local estimated cost, and provider-reported cost distinct instead of overwriting existing fields or adding only a footer special case.
