## Summary

Current main now captures the routed model and reconciles OpenRouter billed cost at the assistant-message layer, but it still does not expose those values through the requested agent, session, or response-footer surfaces; the issue remains necessary and overlaps broader open work without being fully superseded.

## What This Changes

Expose the concrete model selected by `openrouter/auto` and its actual cost to the agent and response usage footer.

## Best Possible Solution

Add a provider-generic per-turn metadata path that preserves selected and resolved model identities plus cost source and authority, persists it for agent and session introspection, and lets the optional usage footer render the resolved model and authoritative cost when available.

## Reproduction Assessment

Yes. On current main, `responseModel` and billed OpenRouter cost can be traced at the assistant-message layer, while the response footer and session/agent metadata paths still omit those fields.

## Solution Assessment

No, a narrow OpenRouter-only footer patch is not the best complete solution; the maintainable path is a provider-generic selected-versus-resolved model and cost-authority contract with the footer as one consumer.
