## Summary

Keep open: current main has adjacent OpenRouter billed-cost reconciliation, but it still does not carry OpenRouter Auto's resolved response model or provider-reported cost details through agent/session metadata or the default reply footer. The remaining work is a provider/runtime metadata design decision, not a conservative cleanup close.

## What This Changes

Request to expose the actual OpenRouter-routed model and accurate usage cost when a user selects `openrouter/auto`.

## Best Possible Solution

Define provider-neutral resolved-model and provider-cost metadata, populate it at the OpenAI-compatible/OpenRouter boundary, persist it through agent/session runtime state, and render footer/session_status from that metadata while keeping local estimates distinct from provider-billed cost.

## Reproduction Assessment

Yes. Source inspection shows current main can reconcile an OpenRouter billed cost total, but it still does not propagate OpenRouter's response model or provider cost details into agent/runtime/session metadata or the default usage footer.

## Solution Assessment

Unclear until maintainers choose the metadata contract; the best fix should be provider-neutral runtime metadata rather than a hardcoded OpenRouter Auto footer string.
