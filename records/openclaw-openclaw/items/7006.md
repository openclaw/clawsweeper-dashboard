## Summary

Keep open: current main has a built-in usage footer and OpenRouter billed-total reconciliation, but it still does not expose the routed `openrouter/auto` model or provider-reported cost/source metadata through agent/session metadata or response usage display.

## What This Changes

Issue #7006 requests that OpenRouter `auto` responses expose the concrete routed model and its cost so replies can show the resolved model instead of only `openrouter/auto`.

## Best Possible Solution

Define a provider-neutral selected-versus-resolved model and provider-cost metadata contract, capture OpenRouter response model and usage at the transport/provider boundary, persist it through assistant/session/runtime metadata, and render it in usage/status surfaces.

## Reproduction Assessment

Yes, source-level: current main has local usage footer and OpenRouter billed-total reconciliation, but the transport, usage state, and footer paths still do not expose the routed `openrouter/auto` model or provider cost/source metadata.

## Solution Assessment

No for a footer-only patch; the best fix needs a provider-neutral metadata contract first, with OpenRouter-specific footer rendering as one consumer.
