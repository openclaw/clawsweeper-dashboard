## Summary

Keep open: current main still preserves the selected `openrouter/auto` placeholder, drops OpenRouter returned-model and provider-cost fields in the shared OpenAI-compatible transport, and renders response footers from local estimates only. The related issues and closed PR are useful context, but they do not supersede this combined resolved-model-plus-cost request.

## What This Changes

Request to expose the concrete model OpenRouter routes to for `openrouter/auto` and its provider-reported usage cost in agent-visible and footer surfaces.

## Best Possible Solution

Define an additive resolved-model and provider-reported cost contract, capture OpenRouter response `model` plus raw usage cost fields at the OpenAI-compatible/OpenRouter boundary, and thread them through agent metadata, transcripts, session/status surfaces, response footers, docs, and targeted tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a strict bug reproduction because this is a feature request; source inspection on current main gives high-confidence proof that the requested OpenRouter returned model and provider-cost metadata are not exposed.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the public metadata and cost-authority contract; the maintainable direction is additive selected-versus-resolved model and provider-cost metadata, not a footer-only patch or overloading existing estimate fields.
