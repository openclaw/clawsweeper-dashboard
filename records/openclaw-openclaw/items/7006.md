## Summary

Keep open: current main still records the selected OpenRouter router model and local estimated-cost path, but does not preserve OpenRouter's returned routed model or provider-reported cost for agent/session/footer use.

## What This Changes

Expose the resolved OpenRouter auto model and provider-reported usage cost to agents and response footers instead of showing only the selected `openrouter/auto` identity and estimated cost.

## Best Possible Solution

Define additive selected-versus-resolved model and provider-reported-cost metadata, capture it at the OpenAI-compatible transport boundary, and surface it consistently through assistant/session records, response footers, transcripts, status/usage tools, and focused OpenRouter auto tests.

## Reproduction Assessment

Yes. Source inspection on current main shows `openrouter/auto` is stored as `model.id`, streamed `chunk.model` is not captured, OpenRouter cost fields are ignored, and the response footer has no resolved-model or provider-cost inputs.

## Solution Assessment

Unclear until maintainers choose the API and UX contract. The safest direction is additive metadata that preserves selected model, resolved model, local estimate, and provider-reported cost separately instead of overwriting existing fields or adding only a footer special case.
