## Summary

Keep open. Current main has partial plumbing for OpenRouter response metadata and billed cost, but the requested agent/footer-visible resolved `openrouter/auto` model label plus provider-reported cost is still not exposed as a coherent user-facing contract.

## What This Changes

The issue requests making the concrete model and cost selected behind `openrouter/auto` available to agents and response footers.

## Best Possible Solution

Define one provider-aware resolved-model and provider-reported-cost contract, then thread it through OpenRouter/OpenAI-compatible transport output, agent metadata, session/status surfaces, transcripts, and response usage footers with focused OpenRouter-auto tests.

## Reproduction Assessment

Yes, at source level: current main captures `responseModel` and reconciles OpenRouter `usage.cost.total`, but those values are not exposed through `EmbeddedAgentMeta`, session rows/status, or response footers. I did not run a live OpenRouter request in this read-only review.

## Solution Assessment

Unclear pending maintainer direction. A footer-only OpenRouter special case is too narrow; the maintainable path is a generic selected-versus-resolved model and provider-reported-cost contract consumed by footer, status, transcript, and agent-runtime surfaces.
