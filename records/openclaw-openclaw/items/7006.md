## Summary

Keep open: current main partially reconciles OpenRouter billed cost, but it still does not expose the resolved `openrouter/auto` model plus provider-reported cost through agent/runtime, session/status, or response-footer surfaces.

## What This Changes

Expose the actual model selected by `openrouter/auto` and its cost so replies can show the resolved model and cost instead of only `OpenRouter Auto`.

## Best Possible Solution

Define provider-neutral selected-vs-resolved model and provider-reported cost metadata, capture it at the OpenAI-compatible/OpenRouter response boundary, and thread it through agent metadata, session/status APIs, and usage footers.

## Reproduction Assessment

Yes, at source level: current main still initializes OpenAI-compatible assistant metadata from the requested model, keeps agent usage metadata token-only, and formats costs from local estimates. No live OpenRouter request was run in this read-only review.

## Solution Assessment

No, a footer-only OpenRouter special case is not the best complete fix; the safer path is one metadata contract shared by footer, status, transcript, and agent-runtime surfaces.
