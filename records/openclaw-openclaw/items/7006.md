## Summary

Keep open: current main and the latest release only partially address OpenRouter cost reconciliation; the resolved `openrouter/auto` model and provider-reported cost metadata still are not consistently exposed through agent/session/status/footer surfaces.

## What This Changes

The issue requests exposing the concrete model and provider-reported cost selected by OpenRouter when `openrouter/auto` is used, so agents and response footers can report the actual routed model and cost.

## Best Possible Solution

Define additive resolved-model and provider-reported-cost metadata at the OpenAI-compatible/OpenRouter boundary, then thread it through agent metadata, transcripts, session/status records, usage tools, and reply footers with targeted `openrouter/auto` regression coverage.

## Reproduction Assessment

## Solution Assessment

No, current main is only a partial foundation. The maintainable fix is an additive cross-surface metadata contract, not a footer-only special case or silently replacing the selected model.
