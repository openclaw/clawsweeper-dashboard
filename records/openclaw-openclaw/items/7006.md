## Summary

Keep open: current main still keeps `openrouter/auto` as the selected model, computes local estimated cost from configured pricing, and does not surface OpenRouter's returned routed model or provider-reported cost through agent, session, or footer metadata. The related open issues cover adjacent cost/model-attribution surfaces, and the prior direct footer PR was closed unmerged.

## What This Changes

Expose the actual OpenRouter-routed model and provider-reported cost to the agent and response footer when `openrouter/auto` is selected.

## Best Possible Solution

Define one additive selected-versus-resolved model and provider-cost metadata contract at the OpenAI-compatible/OpenRouter boundary, keep exact provider cost separate from local estimates, and thread it through agent/session/footer surfaces with focused tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main does not carry OpenRouter returned model or provider-reported cost into agent, session, or footer metadata.

## Solution Assessment

Yes on the requested direction, but not as a footer-only patch. The maintainable solution is an additive provider metadata contract that preserves both selected and resolved model identities and separates provider-reported cost from local estimated USD.
