## Summary

Keep open: current main and v2026.5.12 still retain `openrouter/auto` as the selected model, compute local estimated cost, and do not carry OpenRouter's returned model or provider-reported cost through agent/session/footer metadata. The related open model and cost issues are adjacent, while the closed footer PR was not merged and the open footer PR does not yet solve the OpenRouter cost contract.

## What This Changes

The issue requests exposing the concrete model and provider-reported cost chosen behind `openrouter/auto` so agents and response footers can report them accurately.

## Best Possible Solution

Define an additive selected-versus-resolved OpenRouter metadata and provider-reported cost contract at the OpenAI-compatible transport boundary, then persist and expose it through agent metadata, transcripts/session status, and response footers with focused `openrouter/auto` coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction, but yes for source-level proof: current main and v2026.5.12 do not capture response `model`, `usage.cost`, or `cost_details` into agent/session/footer metadata. The upstream fields exist in OpenRouter's documented response shape, so the gap is observable without live credentials.

## Solution Assessment

Yes on the requested direction, but not as a footer-only patch. The maintainable solution is a transport-level metadata contract that keeps selected model, resolved model, local estimates, and provider-reported charges separate before rendering them in user-facing surfaces.
