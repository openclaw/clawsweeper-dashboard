## Summary

Keep open: current main has partial OpenRouter cost-total reconciliation, but it still does not expose the routed `openrouter/auto` model or provider-reported cost/source metadata through agent/session metadata or the built-in response usage footer.

## What This Changes

The issue requests exposing the actual model selected by OpenRouter auto routing and its cost so agents and response footers can report the resolved model rather than only `openrouter/auto`.

## Best Possible Solution

Define a provider-neutral selected-versus-resolved model and provider-cost metadata contract, then carry it through OpenAI-compatible transport, assistant/session metadata, `session_status`, transcripts, and response usage footers.

## Reproduction Assessment

Yes, source-level: current main can reconcile an OpenRouter billed cost total, but the active transport, metadata, session, and footer paths still do not expose the routed model or provider-cost/source details. I did not run a live OpenRouter auto request in this read-only pass.

## Solution Assessment

No, a footer-only patch is too narrow; the maintainable fix is a provider-neutral metadata contract first, with OpenRouter-specific rendering as a consumer of that contract.
