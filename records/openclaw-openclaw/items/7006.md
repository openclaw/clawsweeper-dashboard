## Summary

Keep open: current main still keeps `openrouter/auto` as the selected placeholder and does not carry a resolved OpenRouter model or provider-reported cost into agent/runtime footers.

## What This Changes

Expose the actual OpenRouter-routed model and provider-reported cost for `openrouter/auto` responses so agents can show them in reply footers.

## Best Possible Solution

Add additive selected/resolved model metadata and provider-reported cost fields, then thread them through transport, transcripts, session/status surfaces, response footers, and targeted OpenRouter auto tests.

## Reproduction Assessment

Yes for the observable feature gap by source inspection: current main stores `openrouter/auto`, parses local estimated cost from token counters, and formats the footer without resolved-model or provider-cost inputs. This is not a strict bug reproduction because the item requests a new metadata/display capability.

## Solution Assessment

Unclear until maintainers choose the public selected-versus-resolved model and provider-cost contract. The narrow maintainable direction is additive metadata plus a distinct provider-reported cost object, not overloading existing estimated-cost fields.
