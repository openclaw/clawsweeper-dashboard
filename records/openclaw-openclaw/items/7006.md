## Summary

Keep open: current main partially fixed OpenRouter cost reconciliation, but it still does not expose the resolved `openrouter/auto` model or provider-reported cost through the agent/runtime footer surface requested here.

## What This Changes

Request to expose the actual OpenRouter auto-routed model and cost to agents and response footers instead of only showing `openrouter/auto`.

## Best Possible Solution

Capture OpenRouter's routed `model` and provider cost in the OpenAI-compatible/OpenRouter transport, preserve selected-versus-resolved metadata in agent/session records, and thread it through footers, `session_status`/status, transcripts, and focused `openrouter/auto` tests.

## Reproduction Assessment

Yes, at source level: current main still formats response footers from selected model/local cost config and does not carry resolved OpenRouter model or provider cost through agent metadata. I did not run a live OpenRouter call for this read-only review.

## Solution Assessment

Yes, the best direction is a transport-to-agent metadata path, not a ClawRouter workaround or a footer-only string patch. The implementation should explicitly preserve both selected and resolved model identities and define cost units before exposing them.
