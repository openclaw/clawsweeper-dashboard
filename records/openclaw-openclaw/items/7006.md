## Summary

Keep open. Current main still preserves the selected `openrouter/auto` placeholder through the OpenAI-compatible transport and response usage footer, while ignoring OpenRouter's returned resolved model and provider-reported cost fields.

## What This Changes

The issue requests exposing the resolved model and usage cost returned by OpenRouter when `openrouter/auto` is selected.

## Best Possible Solution

Capture OpenRouter's returned model and provider cost at the OpenAI-compatible transport boundary, preserve selected-versus-resolved metadata, and render that data in usage/session/status surfaces with focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed an OpenAI-compatible OpenRouter final stream chunk containing top-level `model` and numeric `usage.cost`; current main ignores both for the assistant metadata and response footer.

## Solution Assessment

Yes. Keeping this open and fixing the transport-to-footer metadata path is the narrowest maintainable direction; ClawRouter and closed #43026 are useful prior art, while #9016, #68066, and #22021 cover adjacent cost/model-visibility surfaces rather than a shipped replacement.
