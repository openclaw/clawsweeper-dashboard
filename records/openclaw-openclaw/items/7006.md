## Summary

Keep open: current main and v2026.5.12 still keep `openrouter/auto` as the selected model, compute local estimated cost, and do not capture OpenRouter's returned routed model or provider-reported cost for agent/session/footer use.

## What This Changes

Expose OpenRouter's resolved model and provider-reported usage cost for `openrouter/auto` to agent, session, and response-footer surfaces.

## Best Possible Solution

Define one additive OpenRouter/OpenAI-compatible metadata contract that preserves selected and resolved model identities, keeps provider-reported cost separate from local estimates, and threads those fields through agent/session/footer surfaces with focused tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main and v2026.5.12 do not carry OpenRouter's returned model or provider-reported cost into agent, session, or footer metadata.

## Solution Assessment

Yes on the requested direction, but not as a footer-only patch. The maintainable solution is a provider metadata contract at the OpenAI-compatible/OpenRouter boundary, followed by consistent display in response usage surfaces.
