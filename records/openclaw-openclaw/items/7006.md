## Summary

Keep open: current main has adjacent OpenRouter billed-cost reconciliation, but it still does not expose the OpenRouter Auto routed model or provider cost details through agent/session metadata or the default usage footer. The related open issues cover overlapping cost and resolved-model contracts, but no single canonical item or merged PR fully replaces this OpenRouter Auto request.

## What This Changes

The issue requests exposing the concrete model and cost returned when OpenRouter routes `openrouter/auto`, so agent-visible metadata and response footers can show the actual backend choice instead of only the selected alias.

## Best Possible Solution

Define provider-neutral resolved-model and provider-cost metadata, populate it at the OpenAI-compatible/OpenRouter boundary, and render footer/session_status from that metadata while keeping selected aliases and local estimates distinct.

## Reproduction Assessment

Yes. Source inspection shows current main can reconcile an OpenRouter billed cost total, but it still does not propagate OpenRouter's response model or provider cost details into agent/runtime/session metadata or the default usage footer.

## Solution Assessment

Unclear pending maintainer direction. The best fix should be a provider-neutral runtime metadata contract rather than a hardcoded OpenRouter Auto footer string.
