## Summary

Keep open: current main and v2026.6.6 partially reconcile OpenRouter billed `total_cost`, but the central request remains because the routed `openrouter/auto` model and source-aware provider cost metadata are still not exposed through agent/session metadata or response footers.

## What This Changes

Expose the actual OpenRouter-routed model and provider-reported cost when `openrouter/auto` is selected, including agent-accessible metadata and response footer display.

## Best Possible Solution

Define a provider-neutral selected-versus-resolved model and source-aware cost metadata contract, then capture OpenRouter response model/cost details at the transport/plugin boundary and thread them through agent metadata, sessions, status, transcripts, and response footers.

## Reproduction Assessment

Yes, source-level. Current main can reconcile an OpenRouter billed cost total, but the active transport, metadata, session, and footer paths still do not expose the routed model or source-aware provider cost details.

## Solution Assessment

No, not as a footer-only patch. The maintainable solution is to define the metadata contract first, then make the footer one consumer of selected-versus-resolved model and provider-cost metadata.
