## Summary

Current main still does not expose OpenRouter’s routed model or provider-reported cost through the agent/session/footer path, and the directly proposed footer PR closed unmerged. The related open reports cover cost-only, Gateway HTTP, or broader routing-proxy metadata, so none safely replaces this distinct OpenRouter auto transparency request.

## What This Changes

Expose the concrete backend model selected by `openrouter/auto` and its provider-reported cost to the agent and response usage footer.

## Best Possible Solution

Define one provider-agnostic per-turn execution metadata shape that preserves requested and resolved provider/model identities plus cost value and provenance, populate it from OpenRouter’s final response, then reuse it in agent context, session/status, transcripts, and configurable usage footers.

## Reproduction Assessment

Yes at source level: current main visibly ignores the response model and provider cost fields in the OpenAI-compatible stream path. This read-only review did not perform a billed live OpenRouter turn.

## Solution Assessment

No, a footer-only helper is not the best solution because it assumes concrete runtime metadata already exists and would leave agent/session consumers inconsistent. Capture canonical execution metadata at the transport boundary first, then render it through shared consumers.
