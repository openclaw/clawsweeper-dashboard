## Summary

Keep open: current main still keeps `openrouter/auto` as the selected model, normalizes only token usage plus local estimated cost, and does not surface OpenRouter's returned model or provider-reported cost through agent, session, or footer metadata. Related issues and the open footer PR are adjacent or partial work, not a merged replacement for this request.

## What This Changes

Expose OpenRouter Auto's resolved model and provider-reported usage cost to the agent and response footer.

## Best Possible Solution

Define an additive OpenRouter/OpenAI-compatible metadata contract that captures returned `model`, `usage.cost`, `cost_details`, and cache-write/read details, preserves selected and resolved model identities separately, and threads them through agent/session metadata plus response footer/status surfaces with focused regression coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main does not carry OpenRouter `response.model`, `usage.cost`, or `cost_details` into agent, session, or footer metadata.

## Solution Assessment

Yes on the direction, but not as a footer-only patch. The maintainable solution is an additive provider metadata contract that keeps local estimated USD separate from OpenRouter-reported cost and preserves both selected and resolved model identities.
