## Summary

Keep open: current main has partial OpenRouter billed-cost reconciliation, but it still does not expose the routed `openrouter/auto` model or provider-reported cost/source metadata through agent/session metadata or the response usage footer.

## What This Changes

The issue requests exposing the concrete model selected by `openrouter/auto` and its cost so agents and reply footers can report the actual model used.

## Best Possible Solution

Define a provider-neutral selected-versus-resolved model and provider-cost metadata contract, then carry it through OpenAI-compatible transport, assistant/session metadata, `session_status`, transcripts, and response usage footers.

## Reproduction Assessment

Yes, source-level: current main can reconcile an OpenRouter billed cost total, but the active transport, metadata, session, and footer paths still do not expose the routed model or provider-cost/source details.

## Solution Assessment

No, a footer-only patch is too narrow; the maintainable fix is a provider-neutral metadata contract first, with OpenRouter-specific rendering as a consumer of that contract.
