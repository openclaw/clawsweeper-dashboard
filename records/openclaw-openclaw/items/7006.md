## Summary

Keep open: current main still records the selected `openrouter/auto` identity and locally estimated costs, but it does not preserve OpenRouter's routed model or provider-reported cost through runtime/session/footer metadata.

## What This Changes

Expose the actual OpenRouter model and cost used by `openrouter/auto` so agents and response footers can report it accurately.

## Best Possible Solution

Define additive selected-model, resolved-model, local-estimate, and provider-reported-cost metadata at the OpenAI-compatible transport boundary and thread it through assistant/session records, response footers, transcripts, status/usage tools, and focused OpenRouter auto tests.

## Reproduction Assessment

Yes. Source inspection shows `openrouter/auto` still flows through as the selected `model.id`, parsed usage ignores provider cost fields, and the response footer has no resolved-model or provider-cost inputs.

## Solution Assessment

Unclear until maintainers choose the API and UX contract. The maintainable direction is additive metadata that preserves selected model, resolved model, local estimate, and provider-reported cost separately instead of overwriting existing fields or adding only a footer special case.
