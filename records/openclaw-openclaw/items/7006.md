## Summary

Current main still does not satisfy the request: OpenClaw keeps the selected `openrouter/auto` model, computes local estimated cost, and does not expose OpenRouter's routed model or provider-reported cost to the agent/footer path.

## What This Changes

The issue requests surfacing the actual model and cost returned by OpenRouter when a user selects `openrouter/auto`, including in response footers and agent-accessible usage metadata.

## Best Possible Solution

Capture OpenRouter response `model` plus provider-reported `usage.cost` and `cost_details` at the OpenAI-compatible transport boundary, store selected-versus-resolved model metadata distinctly, and thread that through usage footers, session/status surfaces, transcripts, and targeted `openrouter/auto` regression tests.

## Reproduction Assessment

Yes, from source inspection: `openrouter/auto` still flows through as the selected `model.id`, parsed usage ignores OpenRouter provider cost fields, and the response footer has no resolved-model/provider-cost inputs. I did not run live OpenRouter traffic in this read-only review.

## Solution Assessment

Unclear until maintainers choose the product contract. The narrow maintainable direction is to carry selected model, resolved model, and provider-reported cost as distinct metadata rather than replacing the configured model id or adding only a footer special case.
