## Summary

Keep open: current main still has the same gap. OpenRouter documents both the selected Auto Router model and provider-reported usage cost fields, but OpenClaw keeps the selected `openrouter/auto` model, parses only normalized token counters, computes local estimates, and formats the reply footer without a resolved model label or provider-reported cost.

## What This Changes

Expose the concrete model selected by OpenRouter for `openrouter/auto` and the provider-reported usage cost so response footers can show the actual routed model and cost.

## Best Possible Solution

Add an explicit OpenRouter metadata path that captures response `model`, `usage.cost`, `cost_details`, and cache-write/read details at the OpenAI-compatible boundary, preserves selected-versus-resolved model state, and threads it through assistant/session metadata and response footer/status surfaces.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main does not expose OpenRouter's resolved Auto Router model or provider-reported cost through the transport, agent metadata, session state, or footer path.

## Solution Assessment

Yes on the direction, but not as a footer-only patch. The maintainable solution is an additive provider metadata contract that keeps local estimated USD separate from provider-reported OpenRouter cost and preserves both selected and resolved model identities.
