## Summary

Keep open: current main and the latest release still do not expose OpenRouter Auto's returned model or provider-reported cost through the OpenAI-compatible transport, agent/session metadata, or response usage footer. Related open issues and PRs cover overlapping model-attribution and cost surfaces, but none is a merged replacement for this OpenRouter Auto transparency request.

## What This Changes

Expose the actual model selected by OpenRouter Auto and its provider-reported cost so agents and response footers can show the routed model and cost instead of only `openrouter/auto`.

## Best Possible Solution

Define an additive OpenRouter/OpenAI-compatible metadata contract that captures returned `model`, `usage.cost`, `cost_details`, and cache-write/read details, preserves selected and resolved model identities separately, and threads them through agent/session metadata and response footer/status surfaces with focused regression coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main does not expose OpenRouter Auto's returned model or provider-reported cost through the relevant transport, metadata, session, or footer paths.

## Solution Assessment

Yes on the direction, but not as a footer-only change. The maintainable solution is an additive provider metadata contract that keeps local estimated USD separate from OpenRouter-reported cost and preserves both selected and resolved model identities.
