## Summary

Keep open: current main has partial building blocks for OpenRouter response IDs, responseModel, billed-total reconciliation, and the new /usage full footer, but it still does not expose the resolved OpenRouter auto model plus provider-reported cost through the agent/footer contract.

## What This Changes

Request to expose the concrete model selected by openrouter/auto and its per-turn cost to agents so replies can show the resolved model and cost instead of only the auto alias.

## Best Possible Solution

Define a provider-neutral selected-vs-resolved model and provider-reported-cost contract, then thread it from OpenAI-compatible/OpenRouter responses through agent metadata, session usage, usage footers, docs, and focused regression tests.

## Reproduction Assessment

Yes, at source level: current main captures responseModel and some OpenRouter billed cost data, but the embedded agent and footer contract still expose reported model plus estimated turnUsd rather than the routed model and provider-reported cost.

## Solution Assessment

No final implementation path is selected yet; a footer-only OpenRouter special case would be too narrow because the durable fix crosses provider transport, agent/session metadata, and the usage-footer API.
