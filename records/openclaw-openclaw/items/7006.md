## Summary

Keep open: current main still records `openrouter/auto` as the selected placeholder, parses only normalized token usage plus local estimated cost, and has no resolved OpenRouter model or provider-reported cost path for the agent/runtime footer. The closed related PR at https://github.com/openclaw/openclaw/pull/43026 was not merged, and the open cost/model metadata issues are adjacent rather than replacements.

## What This Changes

Expose the concrete OpenRouter model and provider-reported usage cost selected behind `openrouter/auto` so response footers and agent/runtime surfaces can show accurate usage.

## Best Possible Solution

Add an additive runtime metadata contract that preserves selected versus resolved OpenRouter model identity and provider-reported cost with explicit source/units, then thread it through transport, transcripts, session/status surfaces, and response footers with targeted `openrouter/auto` coverage.

## Reproduction Assessment

Yes for the observable feature gap by source inspection: current main keeps `openrouter/auto` as the stored model, parses local estimated cost from token counters, and formats the footer without resolved-model or provider-cost inputs. This is not a strict bug reproduction because the issue requests a new metadata/display capability.

## Solution Assessment

Unclear until maintainers choose the public metadata and cost-authority contract. The narrow maintainable direction is additive selected/resolved model metadata plus a distinct provider-reported cost object, not overloading existing estimated-cost fields.
