## Summary

Keep open. Current main still carries the selected `openrouter/auto` model and locally estimated token cost through the transport, session, diagnostics, and response footer paths instead of preserving OpenRouter's resolved model and provider-reported cost.

## What This Changes

Expose the concrete model and cost returned by OpenRouter when `openrouter/auto` is selected, especially in response usage footers.

## Best Possible Solution

Capture OpenRouter's returned model and provider cost at the OpenAI-compatible transport boundary, preserve selected-versus-resolved model metadata, and render the resolved Auto label plus provider-reported cost in usage surfaces with focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed an OpenAI-compatible OpenRouter final stream chunk with top-level `model` and numeric `usage.cost`; current main ignores those fields and the footer can only render local token estimates.

## Solution Assessment

Yes. Keeping this open and fixing the transport-to-footer metadata path is the narrowest maintainable solution; #43026 is useful prior art, while #9016, #68066, and #22021 cover adjacent surfaces rather than a shipped replacement.
