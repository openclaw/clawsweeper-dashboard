## Summary

Keep open: current main still preserves the selected `openrouter/auto` identity and local estimated-cost path, but does not carry OpenRouter's returned routed model or provider-reported cost into agent, session, or footer metadata. Related open issues cover broader resolved-model and cost-accounting pieces, but they are not a merged replacement for the combined OpenRouter auto transparency request.

## What This Changes

Expose the actual model selected by OpenRouter auto and the associated provider-reported cost so agents and response footers can report them transparently.

## Best Possible Solution

Define additive selected-versus-resolved model and provider-reported-cost metadata, capture it at the OpenAI-compatible transport boundary, and surface it consistently through assistant/session records, response footers, transcripts, status/usage tools, and focused OpenRouter auto tests.

## Reproduction Assessment

Yes. Source inspection shows `openrouter/auto` still flows through as the selected `model.id`, parsed usage ignores provider cost fields, and the response footer has no resolved-model or provider-cost inputs.

## Solution Assessment

Unclear until maintainers choose the API and UX contract. The maintainable direction is additive metadata that preserves selected model, resolved model, local estimate, and provider-reported cost separately instead of overwriting existing fields or adding only a footer special case.
