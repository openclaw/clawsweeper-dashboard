## Summary

Keep this open: current main partially addresses OpenRouter billed cost lookup, but it still does not expose the resolved `openrouter/auto` model through the agent/OpenAI-compatible transport path or the default response usage footer.

## What This Changes

Expose the concrete model selected by OpenRouter Auto and its cost to the agent so replies can show a resolved-model usage footer instead of only `openrouter/auto`.

## Best Possible Solution

Add a provider-neutral selected-versus-resolved model and provider-cost metadata path, populate it from OpenRouter Auto responses, and thread it through agent/session records, response footers, usage/status surfaces, and focused OpenRouter Auto regression tests.

## Reproduction Assessment

Yes, source-level. Current main shows the OpenAI-compatible agent path keeps selected model metadata and local cost formatting, while the OpenRouter wrapper only partially addresses billed cost.

## Solution Assessment

Unclear until maintainers choose the metadata contract. The narrowest durable direction is a provider-neutral resolved-model/provider-cost path, not a one-off `openrouter/auto` footer string.
