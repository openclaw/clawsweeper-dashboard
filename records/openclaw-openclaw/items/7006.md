## Summary

Keep open. Current main still does not expose the concrete OpenRouter-routed model or provider-reported cost for `openrouter/auto`; the transport, session usage, and response footer paths still carry the selected model and local estimated cost only.

## What This Changes

Expose the actual model resolved by OpenRouter auto routing and its usage cost to the agent and response footer.

## Best Possible Solution

Capture OpenRouter's returned model and provider usage cost at the OpenAI-compatible transport boundary, preserve selected-versus-resolved model metadata through assistant/session records, and render the resolved auto label plus provider-reported cost in response usage surfaces with focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed an OpenAI-compatible OpenRouter final stream chunk with top-level `model` and numeric `usage.cost`; current main ignores those fields and the footer can only render local token estimates.

## Solution Assessment

Yes. Keeping this open and fixing the transport-to-footer metadata path is the narrowest maintainable solution; closed PR #43026 is useful prior art, while #9016, #68066, and #22021 cover adjacent cost/model visibility surfaces rather than a shipped fix.
