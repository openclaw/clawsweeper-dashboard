## Summary

Keep open: current main has partial OpenRouter cost plumbing, but it still does not expose the resolved `openrouter/auto` model plus provider-reported cost through agent metadata, session/status state, or the response usage footer.

## What This Changes

The issue requests that OpenClaw expose the concrete model and provider-reported cost selected by OpenRouter when users configure `openrouter/auto`, so agents and response footers can report the actual routed model and cost.

## Best Possible Solution

Define additive resolved-model and provider-reported-cost metadata at the OpenAI-compatible/OpenRouter boundary, then thread it consistently through agent metadata, session/status records, transcripts, usage tools, and reply footers while coordinating with the related resolved-model and cost issues.

## Reproduction Assessment

Yes, source-level. Current main can be traced to show `openrouter/auto` remains the selected model in agent/footer surfaces and provider-reported cost is not carried into `EmbeddedAgentMeta` or the response usage formatter.

## Solution Assessment

No, current main is only a partial foundation. The best fix is an additive cross-surface metadata contract, not a footer-only special case or overwriting the selected model.
