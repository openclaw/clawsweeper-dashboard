- **[P1] Align protocol schema with null clears:** `src/cron/types.ts:271-276`
  - body: The public `cron.update` path normalizes the patch and then validates it before calling the service. This PR preserves `payload.model = null` and the sibling clear markers, but `CronPayloadPatchSchema` still only allows `toolsAllow` to be null, so the linked API call becomes `INVALID_REQUEST` instead of clearing. Update the gateway protocol schema/generated validators and add validator/server regression coverage for all nullable clear fields.
  - confidence: 0.94
- **[P2] Omit null fields on payload replacement:** `src/cron/types.ts:271-276`
  - body: After these fields become nullable patch fields, `buildPayloadFromPatch` still constructs a fresh `agentTurn` payload by copying `patch.model`, `patch.fallbacks`, and the other optional fields directly when the payload kind changes. A replacement patch with `model: null` would either fail typecheck or carry `null` into a `CronPayload`; build the replacement payload with only non-null optional fields.
  - confidence: 0.86
