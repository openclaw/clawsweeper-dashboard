## Summary

Close as duplicate/superseded. The suffix-stripping request is a narrow, unsafe variant of the open canonical model-profile/key decoupling work tracked by https://github.com/openclaw/openclaw/issues/59168, with an open implementation candidate at https://github.com/openclaw/openclaw/pull/60506.

## What This Changes

Request to let `agents.defaults.models` expose multiple selectable variants of the same provider model by stripping a custom key suffix before provider API calls.

## Best Possible Solution

Keep the model-profile/API-id decoupling work consolidated in https://github.com/openclaw/openclaw/issues/59168 and review a routed-profile contract such as `routeTo` instead of delimiter stripping.

## Reproduction Assessment

Not applicable as a feature/config-contract request. Source inspection still confirms the reported current behavior path: aliases preserve configured keys and runtime construction uses the selected model id literally.

## Solution Assessment

No. Stripping after `:` is not the best solution because `:` is already valid in provider model ids; a documented routed-profile or internal-key/API-id contract is the safer maintainable path.
