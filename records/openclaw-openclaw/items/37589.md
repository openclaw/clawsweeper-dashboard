## Summary

Keep open: current main still rebuilds every array-backed message content item and returns a new message object even when no inline directive text changes, and the latest release has the same behavior. The related implementation PRs are closed unmerged, and no open PR currently owns the remaining fix.

## What This Changes

The issue requests preserving message and text-part object identity when inline directive tag stripping makes no content changes.

## Best Possible Solution

Update the helper and tests so unchanged text parts keep their references, changed text parts are rebuilt, and the original message object is returned when no directive tag was stripped.

## Reproduction Assessment

Yes, at source level. An array-backed message with plain text follows the unconditional map/spread/new-message path even though the lower-level stripper would return `changed: false`; I did not run tests because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to consume the existing `changed` result, reuse unchanged text-part objects, return the original message when nothing changed, and add identity assertions without altering stripped text output.
