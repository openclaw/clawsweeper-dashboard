## Summary

Keep open. Current main still rebuilds array-backed message content and returns a fresh message object on the no-op display-stripping path, while the directly related implementation PRs are closed unmerged.

## What This Changes

Request preservation of message and text-part object identity when stripping inline directive tags makes no text changes.

## Best Possible Solution

Land a narrow helper and regression-test update that reuses unchanged text parts, returns the original array-backed message when no text changed, and preserves existing stripping semantics.

## Reproduction Assessment

Yes. From source inspection, an array-backed message with plain text and no directive tags will still flow through `map`, spread its text part, and return a new message object, so a focused identity assertion would fail on current main.

## Solution Assessment

Yes. The narrow maintainable fix is to use the existing `{ changed }` result, allocate only changed text parts, track whether any part changed, and return the original message when none did.
