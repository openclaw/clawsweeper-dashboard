## Summary

Keep open: current `main` and the latest release still rebuild array-backed message content and return a new message object even when no inline directive text changes, while the related implementation PRs are closed unmerged.

## What This Changes

The issue requests preserving message and text-part object identity when inline directive stripping makes no text changes.

## Best Possible Solution

Land a narrow helper and regression-test update that reuses unchanged text parts, returns the original message when no part changed, and keeps directive stripping semantics stable.

## Reproduction Assessment

Yes, at source level. An array-backed message with plain text follows the unconditional map/spread/new-message path even though `stripInlineDirectiveTagsForDisplay` would report `changed: false`.

## Solution Assessment

Yes. The narrow maintainable fix is to consume the existing `changed` result, rebuild only changed text parts, and return the original message when nothing changed.
