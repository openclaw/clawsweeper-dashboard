## Summary

Keep open: current main and the latest release still rebuild unchanged array-backed message content, and the related implementation PRs are closed unmerged.

## What This Changes

The issue requests preserving object identity in `stripInlineDirectiveTagsFromMessageForDisplay` by rebuilding only text parts whose directive-stripped text changes and returning the original message when nothing is stripped.

## Best Possible Solution

Land a narrow helper and regression-test update that reuses unchanged text-part references, tracks whether any part changed, and returns the original message when no directives were stripped.

## Reproduction Assessment

Yes, at source level. Passing an array-backed message with plain text through the current helper follows the unconditional map/spread/new-message path even though the lower-level stripper would report `changed: false`.

## Solution Assessment

Yes. The narrow maintainable fix is to use `stripInlineDirectiveTagsForDisplay(...).changed`, rebuild only changed text parts, and return the original message when no part changed.
