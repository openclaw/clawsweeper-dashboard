## Summary

Keep open: current `main` still maps every array-backed message content item, spreads every text part, and returns a new message object even when directive stripping reports no text change. The latest release has the same behavior, the focused identity assertions are still absent, and the related implementation PRs are closed unmerged.

## What This Changes

The issue asks `stripInlineDirectiveTagsFromMessageForDisplay` to rebuild only changed text parts and return the original message object when no inline directive tags are stripped.

## Best Possible Solution

Update the helper and tests so unchanged text parts keep their references, changed text parts are rebuilt, and the original message object is returned when no directive tag was stripped.

## Reproduction Assessment

Yes, at source level. An array-backed message with plain text follows the unconditional map/spread/new-message path even though the lower-level stripper would return `changed: false`.

## Solution Assessment

Yes. The narrow maintainable fix is to consume the existing `changed` result, reuse unchanged text-part objects, return the original message when nothing changed, and add identity assertions without altering stripped text output.
