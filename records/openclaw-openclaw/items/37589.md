## Summary

Keep open: current main still unconditionally rebuilds array-backed message content and returns a new message object even when the inline directive stripper reports no text change. The related implementation PRs are closed unmerged, and the fix remains narrow.

## What This Changes

The issue asks `stripInlineDirectiveTagsFromMessageForDisplay` to preserve the original message and unchanged text-part objects when no inline directive tags are stripped.

## Best Possible Solution

Update `src/utils/directive-tags.ts` and its colocated tests so changed text parts are rebuilt, unchanged parts keep their references, and the original message is returned when no directives are stripped.

## Reproduction Assessment

Yes, at source level. An array-backed message with plain text follows the unconditional map/spread/new-message path even though the lower-level stripper would report `changed: false`.

## Solution Assessment

Yes. The narrow maintainable fix is to consume the existing `changed` result, rebuild only changed text parts, return the original message when nothing changed, and add identity assertions.
