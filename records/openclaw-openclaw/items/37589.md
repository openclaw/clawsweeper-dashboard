## Summary

Keep open: current main still rebuilds every array-backed text part and returns a new message object even when no directive text changes; the latest release has the same behavior, identity-focused tests are absent, and the directly related implementation PRs are closed unmerged.

## What This Changes

Request preserving object identity in `stripInlineDirectiveTagsFromMessageForDisplay` by only rebuilding parts and messages when directive text is actually stripped.

## Best Possible Solution

Update the helper and colocated tests so unchanged text parts keep their references, the original message is returned when no part changes, and existing directive-stripping semantics remain stable.

## Reproduction Assessment

Yes, at source level. An array-backed message containing plain text follows the unconditional map/spread/new-message path even though `stripInlineDirectiveTagsForDisplay` would report `changed: false`.

## Solution Assessment

Yes. The narrow maintainable solution is to consume the existing `changed` signal, rebuild only changed text parts, return the original message when nothing changed, and add identity assertions.
