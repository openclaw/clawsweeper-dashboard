## Summary

Keep open: current main and v2026.5.19 still rebuild array-backed message content and return a new message object when no directive text changes, and the focused implementation PRs are closed unmerged.

## What This Changes

Request that `stripInlineDirectiveTagsFromMessageForDisplay` preserve original message and text-part references when no inline directive tags are stripped.

## Best Possible Solution

Update the helper and colocated tests so unchanged text parts keep their references, the original message is returned when no part changes, and all directive-stripping output remains stable.

## Reproduction Assessment

Yes, at source level. An array-backed message containing plain text follows the unconditional map/spread/new-message path even though the lower-level stripper would report `changed: false`.

## Solution Assessment

Yes. The narrow maintainable fix is to consume the existing `changed` signal, rebuild only changed text parts, return the original message when nothing changed, and add identity assertions.
