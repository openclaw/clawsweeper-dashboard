## Summary

Keep open: current main still unconditionally rebuilds array-backed message content and returns a new message object even when no inline directive text changes, and the related implementation PRs are closed unmerged.

## What This Changes

The issue asks for `stripInlineDirectiveTagsFromMessageForDisplay` to preserve message and text-part object identity when no inline directive tags are stripped.

## Best Possible Solution

Update the helper and focused tests so unchanged text parts keep their references, the original message is returned when no directives are stripped, and all existing stripping semantics remain stable.

## Reproduction Assessment

Yes from source inspection: an array-backed message with plain text still goes through `map`, spreads its text part, and returns a new message object. The current tests do not assert the requested reference-preservation behavior.

## Solution Assessment

Yes, the narrow maintainable solution is to use the existing `stripInlineDirectiveTagsForDisplay(...).changed` result, rebuild only changed text parts, and return the original message when no part changed.

Fix the current issue by updating `stripInlineDirectiveTagsFromMessageForDisplay` to preserve object identity when no inline directive tags are stripped. In `src/utils/directive-tags.ts`, keep existing stripping semantics, reuse unchanged non-text and unchanged text part references, track changes via `stripInlineDirectiveTagsForDisplay(...).changed`, and return the original message object when no array-backed text content changed. Add focused tests in `src/utils/directive-tags.test.ts` that assert `toBe` identity for an unchanged array-backed message and unchanged parts in a mixed message, while keeping existing stripping cases stable. Do not change SDK exports, directive parsing semantics, or add config/product behavior. Consider `CHANGELOG.md` only if maintainers classify this as user-facing performance behavior.
