## Summary

Keep open. Current main still rebuilds array-backed message content and text-part objects on the no-op display-stripping path, the focused identity regression tests are still absent, and the direct implementation PRs are closed unmerged.

## What This Changes

Request to preserve message and text-part object identity in `stripInlineDirectiveTagsFromMessageForDisplay` when no inline directive text is stripped.

## Best Possible Solution

Land a narrow helper and regression-test update that reuses unchanged text-part objects, returns the original array-backed message when no text changes, and leaves all directive-stripping output semantics intact.

## Reproduction Assessment

Yes. A focused unit test can pass an array-backed message with plain text to `stripInlineDirectiveTagsFromMessageForDisplay` and assert `result === input` plus unchanged part identity; current source necessarily fails because it always maps the array, spreads text parts, and returns a new message object.

## Solution Assessment

Yes. The narrow maintainable fix is to use the existing `{ changed }` result, allocate only changed text parts, track whether any part changed, and return the original message when none did.
