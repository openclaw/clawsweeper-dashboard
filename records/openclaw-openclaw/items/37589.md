## Summary

Keep open: current main still rebuilds array-backed message content and returns a new message object even when no inline directive text changes, and the related implementation PRs are closed unmerged.

## What This Changes

The issue asks to preserve unchanged message and text-part object references when display-time inline directive stripping makes no content change.

## Best Possible Solution

Land a narrow helper and regression-test update that reuses unchanged text parts, returns the original message when no part changed, and keeps directive stripping semantics stable.

## Reproduction Assessment

Yes, at source level. An array-backed message containing plain text follows the unconditional map/spread/new-message path even though `stripInlineDirectiveTagsForDisplay` would report `changed: false`.

## Solution Assessment

Yes. The narrow maintainable fix is to consume the existing `changed` result, rebuild only changed text parts, return the original message when nothing changed, and add identity assertions.
