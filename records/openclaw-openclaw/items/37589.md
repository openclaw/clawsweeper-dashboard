## Summary

Keep open. Current main and v2026.4.29 still rebuild array-backed message content and text-part objects on the no-op display-stripping path, and the directly related implementation PRs are closed unmerged.

## What This Changes

Preserve message and text-part object identity when display-stripping inline directive tags does not change array-backed message content.

## Best Possible Solution

Land a narrow helper and regression-test update that reuses unchanged text parts, returns the original array-backed message when no text changes, and preserves all existing directive-stripping output semantics.

## Reproduction Assessment

Yes. A focused unit test can pass an array-backed message with plain text to `stripInlineDirectiveTagsFromMessageForDisplay` and assert `result === input` plus unchanged part identity; current source necessarily fails that identity check.

## Solution Assessment

Yes. The narrow maintainable fix is to use the existing `{ changed }` result, allocate only changed text parts, track whether any part changed, and return the original message when none did.
