## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/37589 on current main. `stripInlineDirectiveTagsFromMessageForDisplay` in `src/utils/directive-tags.ts` currently maps/spreads array-backed text parts and returns a new message object even when `stripInlineDirectiveTagsForDisplay(part.text).changed` is false. Update the helper to reuse unchanged text-part objects, track whether any text part changed, return the original message when no directives were stripped, and keep undefined/non-array early returns plus stripped text output exactly stable. Add focused Vitest coverage in `src/utils/directive-tags.test.ts` using `toBe` for an unchanged array-backed message and for unchanged parts in a mixed changed/unchanged content array. Do not alter directive regexes, whitespace semantics, SDK export names, or unrelated gateway behavior. Add a concise `CHANGELOG.md` Fixes entry for the public plugin SDK text helper identity behavior unless a maintainer says this is release-note-internal only.
