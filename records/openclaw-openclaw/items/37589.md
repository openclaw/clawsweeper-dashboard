## Summary

Keep open: current main and the latest release still rebuild array-backed message content and return a new message object even when no directive text changes, while all focused implementation PRs linked from the discussion are closed unmerged.

## What This Changes

The issue asks for `stripInlineDirectiveTagsFromMessageForDisplay` to preserve object identity for unchanged message content while still rebuilding changed text parts.

## Best Possible Solution

Land a narrow helper and test update that uses the existing `changed` result, reuses unchanged text-part references, returns the original message when nothing changed, and preserves stripped text output and the exported helper signature.

## Reproduction Assessment

Yes, at source level. Passing an array-backed message with plain text through the current helper follows the unconditional map/spread/new-message path even though the lower-level stripper would report `changed: false`.

## Solution Assessment

Yes. The narrow maintainable fix is to use `stripInlineDirectiveTagsForDisplay(...).changed`, rebuild only changed text parts, and return the original message when no part changed.
