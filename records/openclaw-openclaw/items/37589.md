## Best Possible Solution

Update the helper and colocated tests so unchanged array-backed text parts keep their references, the original message is returned when no part changes, and existing directive-stripping behavior remains unchanged.

## Solution Assessment

Yes. The narrow maintainable fix is to consume the existing `changed` result, rebuild only changed text parts, return the original message when nothing changed, and add identity assertions.
