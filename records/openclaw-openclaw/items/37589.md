## Summary

Keep open: current main still rebuilds array-backed message content and returns a new message object even when no inline directive text changes, and the directly related implementation PRs remain closed unmerged.

## What This Changes

The issue requests preserving message/text-part identity when `stripInlineDirectiveTagsFromMessageForDisplay` strips no inline directive tags.

## Best Possible Solution

Land a focused helper and test update that reuses unchanged text parts, returns the original message when no part changes, and keeps existing directive stripping semantics stable.

## Reproduction Assessment

Yes. Source inspection is enough: pass an array-backed message with plain text and the helper still maps the content, spreads the text part, and returns a new message even though the lower-level stripper would report `changed: false`.

## Solution Assessment

Yes. The narrow maintainable fix is to use the existing `changed` result, rebuild only changed text parts, and return the original message when no parts changed.
