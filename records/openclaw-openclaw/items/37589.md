## Summary

Keep open: current main and the latest release still rebuild unchanged array-backed message content, and the directly related implementation PRs are closed unmerged.

## What This Changes

The issue requests preserving message and text-part object identity when `stripInlineDirectiveTagsFromMessageForDisplay` removes no inline directive tags.

## Best Possible Solution

Preserve object identity only when no directive text changes, while keeping the helper signature and existing stripped-text semantics stable.

## Reproduction Assessment

Yes, at source level. Passing an array-backed message with plain text follows the unconditional map/spread/new-message path even though `stripInlineDirectiveTagsForDisplay` would report `changed: false`.

## Solution Assessment

Yes. The narrow maintainable fix is to use the existing `changed` result, rebuild only changed text parts, and return the original message when no part changed.
