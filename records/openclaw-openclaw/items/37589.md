## Summary

Keep open: current main still rebuilds array-backed message content and returns a new message object even when no directive text changes, and the focused implementation PRs linked from the discussion are closed unmerged.

## What This Changes

The issue requests preserving message and text-part object identity when inline directive stripping makes no content changes.

## Best Possible Solution

Update the helper and focused tests so unchanged text parts keep their references, the original message is returned when no directives are stripped, and existing directive stripping semantics remain stable.

## Reproduction Assessment

Yes, at source level. Passing an array-backed message with plain text through the current helper follows the unconditional map/spread/new-message path even though the lower-level stripper would report `changed: false`.

## Solution Assessment

Yes. The narrow maintainable fix is to use `stripInlineDirectiveTagsForDisplay(...).changed`, rebuild only changed text parts, and return the original message when no part changed.
