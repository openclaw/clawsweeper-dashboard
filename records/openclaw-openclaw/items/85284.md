## Summary

Keep open: this PR targets a real Windows schtasks fallback bug, but the patch currently turns every `/Create` exit code 1 into a successful Startup-folder fallback and still lacks after-fix proof from a real non-English Windows run.

## What This Changes

The PR changes the Windows schtasks Startup-folder fallback predicate to accept exit code 1 and more localized access-denied strings, plus adds mocked fallback tests for Chinese, Japanese, and Russian messages.

## Best Possible Solution

Keep the denied Scheduled Task fallback locale-independent, but either prove that all `/Create` code-1 cases should intentionally use Startup-folder fallback or narrow the predicate to denied/blocked create failures with real Windows proof.

## Reproduction Assessment

Yes, source-reproducible: current main reaches the `/Create` failure branch with the reported zh-CN denial text, and the existing predicate does not match that localized message. I did not run a live Chinese Windows install in this read-only review.

## Solution Assessment

No, not as written: the localized denial coverage is on the right path, but treating bare exit code 1 as access denied is broader than the documented denied/blocked fallback path without proof.
