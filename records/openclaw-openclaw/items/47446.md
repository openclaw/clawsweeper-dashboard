## Summary

Keep this PR open, but do not merge it as-is: the proxy fix is plausibly useful, current main only covers part of the requested behavior, and the branch still has blocking plugin-boundary, NO_PROXY, Undici proxy transport, and real-behavior-proof gaps.

## What This Changes

The PR bootstraps env-proxy dispatching for gateway startup, derives Discord gateway proxy settings from env vars, disables proxy keep-alive on proxy agents, and updates focused proxy tests plus small gateway command cleanup.

## Best Possible Solution

Land a current-main Discord proxy fix that stays behind SDK/plugin boundaries, uses target-aware NO_PROXY checks for Discord hosts, configures proxy tunnel policy through proxyTls, preserves Discord proxy guardrails, and includes focused tests plus redacted real local-proxy proof.

## Reproduction Assessment

Yes for the patch defects: source inspection of the PR diff, plugin boundary rules, current proxy helpers, and Undici 8.3.0 source show the boundary, NO_PROXY, and proxyTls issues. The original Discord local-proxy ECONNRESET path was not live-reproduced because the PR still lacks manual proof.

## Solution Assessment

No. The fix direction is useful, but this implementation is not the best way because it imports core source from plugin code, ignores target-aware NO_PROXY handling, and configures proxy tunnel policy through connect instead of proxyTls.
