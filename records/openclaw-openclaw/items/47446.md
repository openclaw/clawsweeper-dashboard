## Summary

Keep open: the proxy direction is useful, but the current PR still has blocking correctness/security issues and lacks real Discord local-proxy proof before merge.

## What This Changes

The PR bootstraps gateway Undici env-proxy dispatching, adds Discord gateway/REST env-proxy fallback, disables selected proxy keep-alive settings, updates tests, and includes small node-command cleanup.

## Best Possible Solution

Rework this as a current-main Discord proxy fix using public SDK proxy helpers, target-aware NO_PROXY checks for Discord hosts, proxyTls tunnel policy, focused tests, and redacted real local-proxy proof.

## Reproduction Assessment

Yes for the patch defects: the PR diff, plugin boundary rules, current proxy helpers, and Undici 8.2.0 source/types reproduce the blockers by inspection. The original live Discord ECONNRESET report is not reproduced because the manual local-proxy check is still unchecked.

## Solution Assessment

No. The direction is valid, but this implementation is not the best fix because it crosses the plugin boundary, ignores NO_PROXY for explicit Discord env agents, and configures proxy tunnel policy through connect instead of proxyTls.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use the SDK proxy helper instead of core src imports:** `extensions/discord/src/monitor/gateway-plugin.ts:7`
  - body: This relative import reaches from the Discord plugin into core src/**, which violates the plugin boundary and can break packaged plugin loading. Use the public openclaw/plugin-sdk/fetch-runtime subpath for these helpers.
  - confidence: 0.95
- **[P1] Honor NO_PROXY before creating Discord env proxy agents:** `extensions/discord/src/monitor/gateway-plugin.ts:271`
  - body: resolveEnvHttpProxyUrl("https") only selects a proxy URL; it does not check whether discord.com or gateway.discord.gg is excluded by NO_PROXY. With HTTPS_PROXY set and Discord bypassed, auth-bearing traffic is still forced through the proxy.
  - confidence: 0.92
- **[P2] Set env proxy tunnel policy through proxyTls:** `src/infra/net/undici-global-dispatcher.ts:96`
  - body: Undici applies proxy-connection transport options through proxyTls; connect.keepAlive does not configure the proxy tunnel for EnvHttpProxyAgent. Local proxies that close reused tunnels can still surface ECONNRESET.
  - confidence: 0.88
- **[P2] Set gateway metadata proxy policy through proxyTls:** `extensions/discord/src/monitor/gateway-plugin.ts:288`
  - body: The Discord gateway metadata ProxyAgent also puts keepAlive: false under connect, which Undici does not use for the proxy connection. This leaves metadata fetches vulnerable to stale proxy tunnel reuse.
  - confidence: 0.86
- **[P2] Set Discord REST proxy policy through proxyTls:** `extensions/discord/src/monitor/rest-fetch.ts:12`
  - body: The REST proxy agent uses the same unsupported connect.keepAlive shape, so explicit Discord REST proxy calls can still reuse closed CONNECT tunnels. Put the proxy transport policy on proxyTls.
  - confidence: 0.86
