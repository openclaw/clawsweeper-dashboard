## Summary

Keep open: the Discord proxy fix is still plausible and not fully implemented on current main, but this branch is not mergeable as-is because it crosses the plugin boundary, bypasses NO_PROXY for auth-bearing Discord traffic, applies the Undici tunnel policy in the wrong option, and lacks real local-proxy proof.

## What This Changes

The PR adds gateway and Discord proxy handling so env proxy variables are used for global fetch, Discord WebSocket, and REST paths, and tries to disable proxy tunnel keep-alive.

## Best Possible Solution

Refresh this as a current-main Discord proxy fix using SDK fetch-runtime helpers, target-aware NO_PROXY handling, proxyTls/shared dispatcher tunnel policy, preserved direct metadata fetch behavior, focused tests, and redacted local-proxy proof.

## Reproduction Assessment

Yes for the patch defects: source inspection of the PR diff, extension boundary policy, current NO_PROXY helpers, and Undici 8.3.0 source shows the failures. No live reproduction exists for the original local-proxy ECONNRESET improvement because manual proxy verification remains unchecked.

## Solution Assessment

No. The direction is useful, but this branch is not the best fix because current main already centralizes much of the dispatcher work and the remaining Discord WebSocket/env fallback needs a boundary-safe, NO_PROXY-aware implementation.
