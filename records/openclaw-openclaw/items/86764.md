## Best Possible Solution

Keep the early-persistence design, but route direct CLI user persistence through the existing hook-approved recorder seam so blocked prompts remain redacted while ACP/internal transcript coverage stays intact.

## Solution Assessment

No, not as submitted. Early persistence is the right fix shape, but the CLI path must preserve the existing hook-approved persistence boundary instead of writing the raw turn before policy hooks run.
