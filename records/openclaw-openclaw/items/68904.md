## Summary

Keep open. Current main still derives the Daily Log Review summary and staged section from `groundedCount`, while a paired open PR by the reporter is the active fix path and should be reviewed or superseded before this issue closes.

## What This Changes

The issue asks for Dreaming Advanced's Daily Log Review surfaces to consistently use `dailyCount` for daily-log-backed entries, or to rename the UI if the intended concept is grounded replay.

## Best Possible Solution

Land or supersede the focused UI fix so Daily Log Review summary, staged entries, origin labels, and regression coverage align with `dailyCount`; if grounded replay is intended instead, rename the page/section/copy consistently.

## Reproduction Assessment

Yes. Source inspection shows a short-term entry with `dailyCount > 0` and `groundedCount === 0` is excluded from the Daily Log Review summary and staged section because current main filters those surfaces with `groundedCount > 0`.

## Solution Assessment

Yes, for the current UI wording. The narrow maintainable fix is to use `dailyCount > 0` for Daily Log Review surfaces with focused regression coverage; a full copy rename is the alternative only if maintainers intend the page to mean grounded replay.
