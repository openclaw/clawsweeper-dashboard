## Summary

Keep open. Current main still uses `groundedCount` for Daily Log Review summary/section/badge behavior, and the same-author open PR https://github.com/openclaw/openclaw/pull/72552 is the active fix path rather than a reason to close this issue early.

## What This Changes

The issue requests aligning the Dreaming Advanced Daily Log Review summary, staged section, and origin badge with `dailyCount > 0` rather than `groundedCount > 0`.

## Best Possible Solution

Land or supersede the focused UI fix so Daily Log Review consistently uses `dailyCount > 0`, with regression coverage for daily-only and grounded-only entries; if grounded replay is intended, rename the title, summary, section, empty state, and badge copy consistently instead.

## Reproduction Assessment

Yes. A short-term entry with `dailyCount > 0` and `groundedCount === 0` is excluded by the current `groundedEntries` filter, so current main can show zero Daily Log entries while daily-log-backed entries exist.

## Solution Assessment

Yes, for the current UI wording. The narrow maintainable fix is to use `dailyCount > 0` for the Daily Log Review surfaces, while a complete copy rename is the safer alternative only if maintainers intend this page to mean grounded replay.
