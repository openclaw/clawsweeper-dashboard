## Summary

Keep this open. Current main still uses `groundedCount` for the Daily Log Review summary, staged section, and origin badge while the memory store keeps `dailyCount` and `groundedCount` as separate signals; open PR #72552 is the active paired fix path, so the issue should not be closed before that work lands or is superseded.

## What This Changes

The issue asks for the Dreaming Advanced Daily Log Review summary, staged section, and origin badge to align with `dailyCount` instead of `groundedCount`, or for the UI labels to be renamed if grounded replay is the intended concept.

## Best Possible Solution

Land or supersede the focused UI fix so the Daily Log Review summary, staged section, and origin badges consistently use `dailyCount > 0`, with regression coverage for daily-only and grounded-only entries. If grounded replay is the intended product meaning, rename the title, summary, section, empty state, and badge copy consistently instead.

## Reproduction Assessment

Yes. A short-term entry with `dailyCount > 0` and `groundedCount === 0` is excluded by the current `groundedEntries` filter, so the Daily Log summary and staged section can render zero even when daily-log-backed entries exist.

## Solution Assessment

Yes, the narrow implementation path in #72552 is the best fit for the current wording. The safer alternative is a complete copy/semantics rename if maintainers decide the page should represent grounded replay rather than daily-log-backed entries.
