## Summary

Keep open. Current `main` and latest release still derive Dreaming Advanced “From Daily Log” summary/section data from `groundedCount`, and the active paired fix path is https://github.com/openclaw/openclaw/pull/72552.

## What This Changes

The issue requests aligning Dreaming Advanced Daily Log Review summary, section, and origin badges with `dailyCount` rather than `groundedCount`.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/72552 with a focused fix that makes the Daily Log Review summary, staged entries, origin labels, and regression coverage consistently match `dailyCount`; if grounded replay is intended, rename the visible copy instead.

## Reproduction Assessment

Yes. Source inspection on current `main` shows a `dailyCount > 0` and `groundedCount === 0` entry is excluded from the Daily Log Review summary and staged section because those surfaces filter on `groundedCount > 0`.

## Solution Assessment

Yes for keeping the issue open. The narrow maintainable solution is to align the Daily Log Review UI with the existing `dailyCount` contract or deliberately rename the UI if maintainers choose grounded replay semantics.
