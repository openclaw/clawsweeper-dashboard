## Summary

Keep open. Current main and the latest release still derive the Dreaming Advanced Daily Log Review summary and staged section from `groundedCount`, while the memory store keeps `dailyCount` and `groundedCount` as separate counters; the same reporter has an open paired fix path at https://github.com/openclaw/openclaw/pull/72552.

## What This Changes

The issue requests that Dreaming Advanced Daily Log Review use a consistent daily-log definition instead of mapping Daily Log labels and counts to grounded replay counters.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/72552 with a narrow patch that makes Daily Log Review consistently `dailyCount`-backed, including summary, staged entries, origin labels, section action behavior, and regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows a `dailyCount > 0` and `groundedCount === 0` entry is excluded from the Daily Log Review summary and staged section because those surfaces filter on `groundedCount > 0`.

## Solution Assessment

Yes for keeping this issue open. The maintainable path is to land or supersede the linked fix PR with one consistent daily-log definition rather than close while current main and the latest release still reproduce the mismatch.
