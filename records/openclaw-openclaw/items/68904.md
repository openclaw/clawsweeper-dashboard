## Summary

Keep open. Current main still derives the Daily Log summary and staged section from `groundedCount`, while the backing store keeps `dailyCount` and `groundedCount` as separate signals; #68942 and #72552 are open implementation PRs that should be reviewed or superseded before this issue closes.

## What This Changes

The issue asks for the Dreaming Advanced Daily Log summary, staged section, and origin labels to consistently use daily-log-backed entries or be renamed to grounded replay semantics.

## Best Possible Solution

Keep the issue open until a maintainer lands or supersedes the UI fix work. The end state should either make the Daily Log Review summary, staged section, and origin badges consistently use `dailyCount > 0`, or rename the surface to grounded replay throughout, with regression coverage for daily-only and grounded-only entries.

## Reproduction Assessment

Yes. A daily-only entry with `dailyCount > 0` and `groundedCount === 0` is excluded by the current `groundedEntries` filter, so the Daily Log summary and staged section can show zero even when daily-log-backed entries exist.

## Solution Assessment

Yes, the narrow implementation path is the best fit for the current wording: update the Dreaming Advanced UI mapping and add focused UI tests. If product intent is actually grounded replay, the safer alternative is a complete wording change across the title, summary, section, empty state, and badge semantics.
