## Summary

Keep open. ## What This Changes

Make the Dreaming Advanced Daily Log Review summary, staged section, and origin labels consistent with daily-log-backed entries instead of grounded-replay entries.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/72552 with a narrow patch that makes Daily Log Review consistently `dailyCount`-backed, including summary, staged entries, origin labels, section action, and regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows a `dailyCount > 0` and `groundedCount === 0` entry is excluded from the Daily Log Review summary and staged section because those surfaces filter on `groundedCount > 0`.

## Solution Assessment

Yes for keeping this issue open. The maintainable path is to land or supersede the linked fix PR with one consistent daily-log definition rather than close while current main and the latest release still reproduce the mismatch.
