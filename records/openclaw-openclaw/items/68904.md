Keep open. Current main and the latest release still use `groundedCount` for the Dreaming Advanced Daily Log Review surfaces, and the reporter’s paired PR is still open as the active fix path.

## What This Changes

The issue asks for the Dreaming Advanced Daily Log Review summary, staged section, and origin badges to align with `dailyCount` instead of `groundedCount`.

## Best Possible Solution

Land or supersede the focused UI fix so Daily Log Review summary, staged entries, origin labels, and regression coverage consistently use `dailyCount`; if grounded replay is intended instead, rename the visible copy consistently.

Yes. Source inspection on current main shows a short-term entry with `dailyCount > 0` and `groundedCount === 0` is excluded from the Daily Log Review summary and staged section because those surfaces filter on `groundedCount > 0`.

## Solution Assessment

Yes. Keeping this issue open is the right path while the paired PR is open; the narrow maintainable fix is to switch the Daily Log Review surfaces to `dailyCount > 0` with focused tests, or explicitly rename the UI if maintainers want grounded replay semantics.
