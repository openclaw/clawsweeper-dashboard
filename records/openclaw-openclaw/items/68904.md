## Summary

Keep open. Current main still renders the Dreaming Advanced Daily Log Review from `groundedCount` in the summary, staged section, and origin badge logic while the store and status payload keep `dailyCount` and `groundedCount` as separate signals; open PR #72552 is the active paired fix path, so this issue should remain open until that PR lands or is superseded.

## What This Changes

The issue asks for the Dreaming Advanced Daily Log Review summary, staged section, and origin badges to use `dailyCount` consistently instead of `groundedCount`, or to rename the UI if grounded replay is intended.

## Best Possible Solution

Land or supersede the focused UI fix so Daily Log Review uses `dailyCount > 0` for the summary, staged section, and origin badges, with regression coverage for daily-only and grounded-only entries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a short-term entry with `dailyCount > 0` and `groundedCount === 0` remains in `waitingEntries` but is excluded from `groundedEntries`, so the Daily Log summary and staged section can show zero despite daily-log-backed entries.

## Solution Assessment

Yes, for the current UI wording. The narrow maintainable fix is to key Daily Log Review behavior off `dailyCount > 0`; if maintainers intend grounded replay instead, the safer alternative is a complete copy and badge rename.
