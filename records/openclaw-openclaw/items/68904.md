## Summary

Keep open. Current main still uses `groundedCount` for the Daily Log Review summary, staged section, and origin badge while the store keeps `dailyCount` and `groundedCount` as separate signals; open PR #72552 is the active paired fix path, so this issue should remain open until that work lands or is superseded.

## What This Changes

The issue asks for the Dreaming Advanced Daily Log Review summary, staged section, and origin badge to align with `dailyCount` instead of `groundedCount`, or for the UI labels to be renamed if grounded replay is the intended concept.

## Best Possible Solution

Land or supersede the focused UI fix so the Daily Log Review summary, staged section, and origin badges consistently use `dailyCount > 0`, with regression coverage for daily-only and grounded-only entries; if grounded replay is intended, rename the title, summary, section, empty state, and badge copy consistently instead.

## Reproduction Assessment

Yes. A short-term entry with `dailyCount > 0` and `groundedCount === 0` is excluded by the current `groundedEntries` filter, so the Daily Log summary and staged section can render zero even when daily-log-backed entries exist.

## Solution Assessment

Yes, for the current UI wording. The narrow maintainable fix is to use `dailyCount > 0`; the safer alternative is a complete copy/semantics rename if maintainers decide the page should represent grounded replay rather than daily-log-backed entries.
