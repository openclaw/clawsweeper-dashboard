## Summary

Keep this open. Current main still uses `groundedCount` for Daily Log Review summary, staged section, and origin badge behavior, while an open same-author PR (#72552) is already the active fix path.

## What This Changes

The issue asks for the Dreaming Advanced Daily Log summary, section, and origin labels to use daily-log-backed `dailyCount` semantics instead of grounded replay `groundedCount` semantics.

## Best Possible Solution

Land or supersede the focused UI fix so the Daily Log Review summary, staged section, and badges consistently use `dailyCount > 0`, with regression coverage for daily-only and grounded-only entries.

## Reproduction Assessment

Yes. Rendering the Advanced tab with a short-term entry where `dailyCount > 0` and `groundedCount === 0` will exclude it from the current Daily Log summary and staged section because the code filters on `groundedCount`.

## Solution Assessment

Yes, the narrowest maintainable fix is to align the current Daily Log wording with `dailyCount`; the safer alternative is a complete copy rename if maintainers decide the page should represent grounded replay instead.
