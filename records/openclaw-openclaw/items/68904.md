## Summary

Keep open. Current `main` and `v2026.5.12` still use grounded replay entries for the Daily Log Review count/list, and the active paired fix path remains open at https://github.com/openclaw/openclaw/pull/72552.

## What This Changes

The issue reports that Dreaming Advanced Daily Log Review surfaces use `groundedCount` where the visible UI says daily log, causing daily-log-backed entries to be counted and labeled incorrectly.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/72552 with a focused fix that makes the Daily Log Review summary, staged entries, origin labels, and regression coverage consistently use the intended daily-log semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows a `dailyCount > 0` and `groundedCount === 0` entry is excluded from the Daily Log Review summary and staged section because both surfaces filter on `groundedCount > 0`.

## Solution Assessment

Yes for issue routing. Keeping this issue open while https://github.com/openclaw/openclaw/pull/72552 is reviewed or superseded is the narrow maintainable path; if maintainers intended grounded replay semantics instead, the visible copy should be renamed deliberately.
