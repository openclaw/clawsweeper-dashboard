## Summary

Keep open: the hardening idea is still valid on current main, but this PR patches retired `src/browser/**` files while the active noVNC route now lives under `extensions/browser/**`, and the contributor has not provided inspectable real behavior proof.

## What This Changes

Adds a nonce-based CSP, `X-Content-Type-Options: nosniff`, and `X-Frame-Options: DENY` to the noVNC bootstrap response with matching nonce/header assertions in the old `src/browser` bridge files.

## Best Possible Solution

Port the nonce-based CSP, `nosniff`, and frame-deny hardening to `extensions/browser/src/browser/bridge-server.ts` and the matching active auth test while preserving one-time token redemption and fragment-only password handling.

## Reproduction Assessment

Yes, at source level. Current main serves `/sandbox/novnc` from `extensions/browser/src/browser/bridge-server.ts` with cache/referrer headers and an un-nonced inline script, but without the requested CSP, `nosniff`, or frame-deny headers.

## Solution Assessment

No, not as written. The nonce-based CSP approach is narrow and maintainable, but the patch must be applied to the active `extensions/browser` files and backed by inspectable real response proof.
