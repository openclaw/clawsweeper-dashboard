## Summary

Keep open: the hardening is still useful, but this PR is incorrect as written because it patches retired `src/browser` files while current main serves `/sandbox/novnc` from the browser plugin path, which still lacks the proposed headers.

## What This Changes

The PR changes the old browser bridge files to add a nonce-based CSP, `X-Content-Type-Options: nosniff`, `X-Frame-Options: DENY`, and matching auth-test assertions for the noVNC bootstrap response.

## Best Possible Solution

Port the nonce-based CSP, `nosniff`, and frame-deny hardening to `extensions/browser/src/browser/bridge-server.ts` and the active auth test while preserving bridge auth, one-time token redemption, and fragment-only password handling.

## Reproduction Assessment

## Solution Assessment

No, not as written. The nonce-based header approach is narrow and maintainable, but the patch must be applied to the active `extensions/browser` files and backed by inspectable real response proof.
