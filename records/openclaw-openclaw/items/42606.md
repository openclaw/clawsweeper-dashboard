## Summary

The hardening remains useful, but this PR is not mergeable as written because it patches retired `src/browser` files while current `main` serves the live noVNC bootstrap from `extensions/browser`. It also lacks inspectable after-fix real behavior proof for the response headers.

## What This Changes

The PR adds nonce-based CSP, `nosniff`, and frame-deny headers to the noVNC bootstrap route and expands the auth test assertions.

## Best Possible Solution

Port the nonce-based CSP, `X-Content-Type-Options: nosniff`, and frame-deny hardening to the active `extensions/browser` route and test while preserving one-time token redemption and fragment-only password behavior.

## Reproduction Assessment

## Solution Assessment

No, not as written. The nonce-based CSP approach is narrow and maintainable, but it must be applied to the active `extensions/browser` files and backed by real response proof before merge.
