## Summary

Keep open: the hardening remains valid, but this branch is not mergeable because it patches the retired `src/browser` path while current `main` serves `/sandbox/novnc` from the browser plugin and still lacks the requested CSP, `nosniff`, and frame-deny headers.

## What This Changes

The PR adds a nonce-based CSP, `X-Content-Type-Options: nosniff`, `X-Frame-Options: DENY`, and nonce/header assertions for the noVNC bootstrap handler under `src/browser`.

## Best Possible Solution

Port the nonce-based CSP, `nosniff`, and frame-deny headers to the active browser plugin bridge and its auth test while preserving auth gating, one-time token redemption, and fragment-only password handoff.

## Reproduction Assessment

Yes, at source level. Current `main` serves `/sandbox/novnc` from `extensions/browser/src/browser/bridge-server.ts` with only cache/referrer headers and an un-nonced inline redirect script, while the active test lacks the proposed header assertions.

## Solution Assessment

No, not as currently implemented. The nonce-based CSP approach is narrow and maintainable, but the patch must target the active `extensions/browser` files and include inspectable real response proof before merge.
