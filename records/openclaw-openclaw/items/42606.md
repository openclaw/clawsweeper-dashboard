## Summary

Keep open: the noVNC header hardening is still useful, but the PR branch edits the old `src/browser` bridge files while current main serves the live route from `extensions/browser`, so this patch would not protect the active bootstrap page.

## What This Changes

The PR adds nonce-based CSP, `nosniff`, and frame-deny headers plus tests for `/sandbox/novnc` under the old `src/browser` bridge files.

## Best Possible Solution

Port the nonce-based CSP, `X-Content-Type-Options: nosniff`, and frame-deny hardening to the active browser plugin route with focused regression coverage while preserving auth, one-time token redemption, and fragment-only password behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to request the active `/sandbox/novnc` route with bridge auth and a valid observer token, then inspect that current main only sets cache/referrer headers and emits an un-nonced inline script.

## Solution Assessment

No, not as currently proposed. The nonce-based CSP shape is the right narrow solution, but it must land in `extensions/browser/src/browser/bridge-server.ts` and the matching active auth test rather than stale `src/browser` files.
