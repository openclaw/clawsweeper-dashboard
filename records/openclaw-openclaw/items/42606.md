## Summary

Keep open: the hardening is still valid, but this PR patches the retired `src/browser` bridge path while current `main` serves `/sandbox/novnc` from the browser plugin path without the requested CSP, `nosniff`, or frame-deny headers.

## What This Changes

The PR adds a per-response CSP nonce, `X-Content-Type-Options: nosniff`, and `X-Frame-Options: DENY` to the noVNC bootstrap response and expands the auth test assertions, but only under the old `src/browser` paths.

## Best Possible Solution

Port the nonce-based CSP, `nosniff`, and frame-deny hardening to `extensions/browser/src/browser/bridge-server.ts` and its active auth test while preserving one-time token redemption and fragment-only password behavior.

## Reproduction Assessment

Yes, at source level. Current `main` serves `/sandbox/novnc` from the active browser plugin route with only cache/referrer headers and an un-nonced inline redirect script.

## Solution Assessment

No, not as written. The nonce-based CSP approach is narrow and reasonable, but it must be applied to the active `extensions/browser` files and accompanied by inspectable real response proof.
