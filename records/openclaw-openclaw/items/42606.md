## Summary

Keep open: the hardening is still valid, but this branch patches retired `src/browser` files while current `main` serves the noVNC bootstrap from `extensions/browser`; it also still lacks inspectable real behavior proof for an external security PR.

## What This Changes

The PR adds nonce-based CSP, `X-Content-Type-Options: nosniff`, `X-Frame-Options: DENY`, and nonce/header assertions for the noVNC bootstrap handler and auth test under the retired `src/browser` path.

## Best Possible Solution

Port the nonce-based CSP, nosniff, and frame-deny hardening to `extensions/browser/src/browser/bridge-server.ts` and its auth test while preserving bridge auth, one-time token redemption, and fragment-only password handoff.

## Reproduction Assessment

Yes, at source level. Current `main` serves `/sandbox/novnc` from `extensions/browser/src/browser/bridge-server.ts` with only cache/referrer hardening and an un-nonced inline redirect script, while the active test lacks the requested header assertions.

## Solution Assessment

No, not as written. The nonce-based CSP approach is narrow and maintainable, but the patch must target the active `extensions/browser` files and include inspectable real response proof before merge.
