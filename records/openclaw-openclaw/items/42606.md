## Summary

Keep open: the hardening remains useful, but this branch still patches retired `src/browser/**` files while current `main` serves the noVNC bootstrap from the browser plugin path, so the shipped route would remain unhardened if merged as-is.

## What This Changes

The PR adds a per-response CSP nonce, `X-Content-Type-Options: nosniff`, and `X-Frame-Options: DENY` to the noVNC bootstrap response and updates the matching auth test under `src/browser/**`.

## Best Possible Solution

Port the nonce-based CSP, `nosniff`, and frame-deny hardening to `extensions/browser/src/browser/bridge-server.ts` and the matching auth test while preserving one-time token redemption and fragment-only password handling.

## Reproduction Assessment

## Solution Assessment

No, not as written. The nonce-based CSP approach is narrow and maintainable, but it must be applied to the active `extensions/browser` files and backed by inspectable real response proof.
