Keep open: the hardening remains useful, but this branch is not mergeable because it patches the retired `src/browser` path while current `main` serves `/sandbox/novnc` from the browser plugin path; the external PR also still lacks inspectable after-fix real behavior proof.

## What This Changes

The PR adds nonce-based CSP, `nosniff`, and frame-deny headers plus nonce assertions for the noVNC bootstrap response under `src/browser`.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

No, not as currently proposed. The nonce-based CSP shape is the right narrow solution, but it must be applied to `extensions/browser/src/browser/bridge-server.ts` and the matching active test, not the stale `src/browser` files.
