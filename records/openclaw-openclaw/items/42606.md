## Summary

Keep open. The hardening request is still valid on current main, but this PR changes stale `src/browser` files while the live noVNC bootstrap route now lives under the browser plugin.

## What This Changes

The PR branch adds a nonce-based CSP, `X-Content-Type-Options: nosniff`, `X-Frame-Options: DENY`, and matching assertions to the old noVNC bootstrap handler path.

## Best Possible Solution

Apply the nonce-based CSP, `nosniff`, and frame-deny headers to the active browser plugin route with focused regression coverage, preserving one-time token redemption and fragment-only password behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to request the active `/sandbox/novnc` route with valid bridge auth and a valid observer token, then inspect the response headers/body; current main lacks the requested CSP, `nosniff`, frame-deny, and nonce-bearing script.

## Solution Assessment

No, not as currently proposed. The nonce-based CSP approach is the right narrow shape, but this PR must be ported from stale `src/browser` files to `extensions/browser/src/browser/bridge-server.ts` and its active test.
