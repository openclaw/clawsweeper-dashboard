## Summary

Keep open. The hardening idea is valid, but this PR still patches the retired `src/browser` bridge files while current `main` and the latest shipped release serve `/sandbox/novnc` from `extensions/browser/src/browser/bridge-server.ts` without the proposed CSP, `nosniff`, or frame-deny headers; it also lacks inspectable real behavior proof.

## What This Changes

The PR adds nonce-based CSP, `X-Content-Type-Options: nosniff`, `X-Frame-Options: DENY`, and matching assertions for the noVNC bootstrap response under the old `src/browser` bridge files.

## Best Possible Solution

Port the nonce-based CSP, `nosniff`, and frame-deny hardening to the active browser plugin route and auth test while preserving auth, one-time token redemption, and fragment-only password handoff.

## Reproduction Assessment

Yes, at source level. Current `main` serves `/sandbox/novnc` from `extensions/browser/src/browser/bridge-server.ts` with cache/referrer headers and an un-nonced inline script, while the active test lacks assertions for the requested hardening.

## Solution Assessment

No, not as written. The nonce-based header approach is narrow and maintainable, but the patch must be applied to the active `extensions/browser` files and backed by inspectable real response proof.
