## Summary

Keep open: the hardening is still valid, but this branch patches stale `src/browser` bridge files while current main serves the noVNC bootstrap from `extensions/browser`, so merging it would not protect the active route.

## What This Changes

The branch modifies `src/browser/bridge-server.ts` and its auth test to generate a CSP nonce for the noVNC bootstrap page and assert CSP, `nosniff`, and frame-deny headers.

## Best Possible Solution

Port the nonce-based CSP, `nosniff`, and frame-deny hardening to the active browser plugin route with focused regression coverage while preserving auth, one-time token redemption, and fragment-only password behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to request the active `/sandbox/novnc` route with valid bridge auth and a valid observer token, then inspect that current main only sets cache/referrer headers and emits an un-nonced inline script.

## Solution Assessment

No, not as currently proposed. The nonce-based CSP approach is the right narrow shape, but it must land in `extensions/browser/src/browser/bridge-server.ts` and the matching active auth test rather than stale `src/browser` files.
