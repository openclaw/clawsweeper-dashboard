## Summary

## What This Changes

The PR propagates terminal `chat.send` acknowledgement statuses through Web, Android, Swift Talk, TUI, ACP, gateway Talk, slash-command, and Skill Workshop forwarding paths so terminal ACKs clear or restore pending state instead of being treated as active runs.

## Best Possible Solution

Land the terminal ACK propagation after representative real Gateway/client proof and CI confirm that active `started`/`in_flight` behavior remains intact while `ok`, `timeout`, and `error` clear or restore state consistently.

## Reproduction Assessment

Yes, at source level: current main collapses terminal Web ACK statuses and TUI pending state still follows returned run ids without terminal-status branching. I did not run a live current-main repro because this review is read-only.

## Solution Assessment

Yes, this is a plausible best fix direction because the gateway already emits terminal ACK statuses and each consumer needs to preserve that contract at its state boundary. A shared TypeScript ACK helper could reduce duplication later, but I did not find a concrete regression that makes this implementation the wrong layer.
