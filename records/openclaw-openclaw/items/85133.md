## Summary

Keep open: the report describes a credible macOS gateway availability outage, and current main does not have a newer fix beyond the affected release for the launchd/update paths. ## What This Changes

Reports that a macOS self-update can leave the OpenClaw Gateway LaunchAgent plist on disk but no longer bootstrapped in the user launchd domain.

## Best Possible Solution

Prove the failing macOS update path, then make the update flow verify launchd registration and gateway health before success, auto-rebootstrap when the plist exists but launchd is missing, and persist a clear failure sentinel/log if recovery fails.
