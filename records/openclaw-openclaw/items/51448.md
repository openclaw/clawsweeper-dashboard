## Summary

Keep open: the underlying bug is still source-reproducible on current main, but this PR is not merge-ready because the patch uses a stream option type that does not declare `toolChoice`, loses alias precedence, targets a stale test file layout, and lacks real behavior proof.

## What This Changes

The PR forwards configured `tool_choice`/`toolChoice` agent model params into embedded runner stream options and adds a regression assertion that captures stream options.

## Best Possible Solution

Port the fix to current main by adding a local OpenAI-completions stream-option extension, canonicalizing `tool_choice` aliases across ordered sources, and placing regression coverage in the maintained extra-param suite.

## Reproduction Assessment

Yes. Source inspection is enough: config accepts provider params, current extra-param stream wrapping does not set `options.toolChoice`, and the OpenAI transport only emits request `tool_choice` when that option is present.

## Solution Assessment

No. Forwarding configured tool choice is the right fix direction, but this patch should be ported with the correct option type, ordered alias handling, and current-main regression coverage.
