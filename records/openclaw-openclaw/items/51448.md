## Summary

Keep open: the reported forwarding bug is valid and still present on current main, but this PR cannot merge as-is because the branch needs current-main porting, type-safe stream option handling, alias-precedence fixes, and real behavior proof.

## What This Changes

The PR forwards configured `tool_choice`/`toolChoice` params into embedded runner stream options and adds regression capture for that path.

## Best Possible Solution

Port the narrow forwarding fix to current main with an explicit local/provider stream option type for `toolChoice`, ordered alias precedence, and coverage in the current extra-param tests.

## Reproduction Assessment

Yes. Source inspection is enough: current config can carry `tool_choice`, the embedded runner extra-param bridge does not forward it, and the OpenAI transport only serializes request `tool_choice` when `options.toolChoice` is present.

## Solution Assessment

No. The fix direction is right, but this PR branch needs a current-main port that fixes the option type, preserves alias precedence across sources, and moves regression coverage into the current layout.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Declare a stream option type with toolChoice:** `src/agents/pi-embedded-runner/extra-params.ts:100`
  - body: `CacheRetentionStreamOptions` is based on `Partial<SimpleStreamOptions>`, and pi-ai 0.74.0's `SimpleStreamOptions` does not declare `toolChoice`. This assignment and indexed access will fail typechecking; add an explicit local/provider stream option extension before forwarding it.
  - confidence: 0.93
- **[P2] Preserve alias precedence before forwarding:** `src/agents/pi-embedded-runner/extra-params.ts:93-97`
  - body: Resolving `tool_choice` and `toolChoice` from the already-merged `extraParams` object makes the snake_case key win when both aliases survive from different precedence sources. That can let inherited config override a higher-precedence runtime or per-agent override; canonicalize from ordered sources before building stream params.
  - confidence: 0.9
- **[P2] Move the regression into the current test layout:** `src/agents/pi-embedded-runner/extra-params.openai.test.ts:3`
  - body: Current main no longer has this OpenAI extra-params test file, and the shared helper already captures stream options. Port the regression into the current extra-params test files instead of reviving the deleted layout.
  - confidence: 0.87
