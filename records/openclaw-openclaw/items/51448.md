## Summary

Keep open. The reported bug is valid on current main, but this PR still needs a current-main port, fixes for the typed stream option and alias precedence issues, and after-fix real behavior proof before merge.

## What This Changes

The PR forwards configured `tool_choice`/`toolChoice` agent model params into embedded runner stream options and adds regression capture coverage.

## Best Possible Solution

Port the fix to current main with a local OpenAI-completions stream option type, ordered alias canonicalization, and regression coverage in `src/agents/pi-embedded-runner-extraparams.test.ts`.

## Reproduction Assessment

Yes. Source inspection is enough: config accepts provider params, current extra-param stream wrapping does not set `options.toolChoice`, and the OpenAI transport only emits request `tool_choice` when that option is present.

## Solution Assessment

No. The fix direction is right, but this patch uses a stream option type that does not declare `toolChoice`, loses alias precedence after merge, and adds coverage to a stale test file.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Declare a stream option type with toolChoice:** `src/agents/pi-embedded-runner/extra-params.ts:100`
  - body: `CacheRetentionStreamOptions` is based on `Partial<SimpleStreamOptions>`, and `@earendil-works/pi-ai@0.74.0` does not declare `toolChoice` on `SimpleStreamOptions`. This assignment and indexed access will fail typechecking on current deps; add an explicit local/provider stream option extension before forwarding it.
  - confidence: 0.93
- **[P2] Preserve alias precedence before forwarding:** `src/agents/pi-embedded-runner/extra-params.ts:93-97`
  - body: Resolving `tool_choice` and `toolChoice` from the already-merged `extraParams` object makes the snake_case key win when both aliases survive from different precedence sources. That can let inherited config override a higher-precedence runtime or per-agent override; canonicalize from ordered sources before building stream params.
  - confidence: 0.9

  - body: Current main no longer tracks `src/agents/pi-embedded-runner/extra-params.openai.test.ts`; the maintained coverage lives in the shared extra-param suite and helper. Port this regression there instead of reviving the stale OpenAI-specific test file.
