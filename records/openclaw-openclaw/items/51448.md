## Summary

Keep open. The requested bug fix is valid on current main, but this PR is not merge-ready: current main still lacks configured `tool_choice` forwarding, while the proposed patch uses a stream option type that does not declare `toolChoice`, resolves aliases after precedence has already been flattened, and adds the regression to a stale test path.

## What This Changes

This PR forwards configured `tool_choice`/`toolChoice` agent model params into embedded runner stream options and adds a regression capture for that path.

## Best Possible Solution

Port the narrow forwarding fix to current main by canonicalizing `tool_choice`/`toolChoice` with ordered source precedence, forwarding through a local/provider stream option type that explicitly declares `toolChoice`, preserving call-time option override behavior, and adding focused regression coverage in the current extra-params test files.

## Reproduction Assessment

Yes. Configure `agents.defaults.models["openai/gpt-5.4"].params.tool_choice = "required"` and follow `applyExtraParamsToAgent()` into `createStreamFnWithExtraParams()`; current main never adds `toolChoice` to stream options even though the OpenAI transport consumes it when present.

## Solution Assessment

No. The intended fix is the right narrow direction, but this PR's implementation should be updated for current main's type contract, alias precedence, and test layout before merge.
