## Summary

Keep this PR open. The reported tool-choice forwarding gap still exists on current main, but the submitted patch is dirty against the current test layout and has blocking type and alias-precedence defects, so it should be repaired rather than merged as-is.

## What This Changes

The PR forwards configured `tool_choice`/`toolChoice` agent model params into embedded runner stream options and adds capture-based regression coverage for that path.

## Best Possible Solution

Land a current-main repair that forwards configured tool choice through a typed extra-param bridge, preserves source and alias precedence across defaults, model params, agent params, and runtime overrides, and adds regression coverage in the current extra-param test suites.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure `agents.defaults.models["openai/gpt-5.4"].params.tool_choice = "required"`, call `applyExtraParamsToAgent()`, and observe that current `createStreamFnWithExtraParams()` never adds `toolChoice` to the stream options even though the OpenAI transport serializes it when present.

## Solution Assessment

No. The intent is the right narrow fix, but this patch is not the best implementation because it uses a stream option type that lacks `toolChoice`, resolves aliases after source precedence has been lost, and adds the regression to an obsolete test file path.
