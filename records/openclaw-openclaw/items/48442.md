## Summary

Keep open. Current main still lacks the proposed partial prompt customization surface, and the PR is not merge-ready because it adds a typed config key without strict schema/runtime/docs wiring and introduces a security-sensitive way to remove the Safety section.

## What This Changes

The PR adds optional `agents.defaults.systemPrompt.identityLine` and `disabledSections` fields, threads them into direct `buildAgentSystemPrompt` calls, adds direct builder tests, and adds a test setup OAuth mock.

## Best Possible Solution

Keep this as an implementation candidate only after maintainers settle the prompt customization API. A mergeable version should update typed config, strict Zod schemas, generated schema/help, public docs, runtime prompt assembly paths, default/per-agent precedence, changelog, and end-to-end tests, with explicit guardrails for any ability to remove safety guidance.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: a config using `agents.defaults.systemPrompt` would be outside the current strict schema, and the PR diff does not wire those values into the real embedded, compact, CLI, or auto-reply prompt builders.

## Solution Assessment

No. The current PR is a partial direct-builder change; the safer maintainable solution is an end-to-end config/schema/runtime/docs implementation plus a maintainer-reviewed decision on disabling core prompt sections such as Safety.
