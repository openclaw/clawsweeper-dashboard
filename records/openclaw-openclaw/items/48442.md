## Summary

Keep open: this PR is a partial feature implementation, and the prior review comments are still supported by current-main source inspection. The proposed config key is not accepted by the strict schema, production prompt paths do not consume it, and disabling built-in Safety guidance needs maintainer/security product judgment before merge.

## What This Changes

The PR adds `agents.defaults.systemPrompt.identityLine` and `disabledSections`, direct `buildAgentSystemPrompt` support/tests, and a test setup OAuth mock.

## Best Possible Solution

Define a maintainer-approved prompt customization contract, then implement schema, generated config docs/help, runtime prompt assembly, per-agent/default precedence, tests, docs, and changelog end to end with explicit guardrails for Safety.

## Reproduction Assessment

Yes. Source inspection is enough: a config using `agents.defaults.systemPrompt` would be outside the current strict schema, and the PR diff only adds direct-builder plumbing rather than real runtime config wiring.

## Solution Assessment

No. The PR is not the best mergeable shape yet; the safer solution is an end-to-end config/schema/runtime/docs implementation after maintainers decide whether section disabling, especially Safety, is allowed.
