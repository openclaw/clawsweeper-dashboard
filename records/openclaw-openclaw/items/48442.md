## Summary

Keep this PR open. Current main still does not implement the proposed nested prompt config, and the PR branch is not merge-ready because the new config is absent from strict runtime schema/generated docs and is not wired into the production prompt assembly paths.

## What This Changes

The PR adds `agents.defaults.systemPrompt.identityLine` and `disabledSections` types, direct `buildAgentSystemPrompt` parameter handling and section gating tests, plus a test setup OAuth mock.

## Best Possible Solution

Keep the PR as an implementation candidate only after maintainers settle the prompt customization API. The final version should update the typed config, strict Zod schemas, generated schema/help, public docs, runtime prompt assembly paths, per-agent/default precedence, and end-to-end tests, with clear guardrails for any ability to remove safety guidance.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: a config using `agents.defaults.systemPrompt` would be outside the current strict schema, and the PR diff does not wire those values into the real embedded/CLI/auto-reply prompt builders.

## Solution Assessment

No. The current patch is a partial direct-builder change, not the narrowest maintainable end-to-end solution; the safer path is to finish config/schema/docs/runtime wiring and make a maintainer-reviewed decision about disabling core sections such as Safety.
