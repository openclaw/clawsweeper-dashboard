## Summary

Keep open. Current main still lacks the requested session-entry slot projection, so this PR is an active implementation slice, but the latest head still has a security-sensitive trust-boundary bug: it adds full config to the shared tool hook context used by ordinary `before_tool_call` plugins while implementing a trusted-policy-only read helper.

## What This Changes

The PR adds Plugin SDK session-entry slot metadata, mirrors plugin session-extension projections into guarded top-level `SessionEntry` slots with cleanup, adds trusted-policy session-extension reads, and updates tests, changelog, and the SDK API baseline.

## Best Possible Solution

Keep the additive session-entry projection seam, but feed `getSessionExtension` through a private trusted-policy-only context or explicit `runTrustedToolPolicies` parameter instead of widening public tool-hook context. Add regression coverage that ordinary `before_tool_call` hooks cannot observe `ctx.config`, and seed the slot-key guard from `Object.prototype` names or reject those names explicitly.

## Reproduction Assessment

Yes. Static tracing reproduces the main issue: the PR adds `config` to shared `toolContext`, then passes that same object into `hookRunner.runBeforeToolCall`, so any ordinary `before_tool_call` plugin can inspect `ctx.config`.

## Solution Assessment

No. The projection and cleanup direction is reasonable, but using the shared public hook context to support trusted-policy reads is broader than necessary; a trusted-policy-only context is the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep config out of normal tool hooks:** `src/agents/pi-tools.before-tool-call.ts:482`
  - body: The added `config` property is placed on the shared `toolContext`, but that same object is later passed to `hookRunner.runBeforeToolCall`. This exposes the full `OpenClawConfig` to ordinary external `before_tool_call` plugins, even though only trusted policies need config for scoped session-extension reads. Pass config through a trusted-policy-only context or parameter and leave the normal hook context unchanged.
  - confidence: 0.89
- **[P3] Reserve inherited object slot names:** `src/plugins/session-entry-slot-keys.ts:1-3`
  - body: The new guard blocks `__proto__`, `constructor`, and `prototype`, but still allows `Object.prototype` names such as `toString` and `hasOwnProperty`. Because promoted slots are written as top-level own `SessionEntry` properties, those names can create confusing collisions. Seed the guard from `Object.getOwnPropertyNames(Object.prototype)` or reject those names explicitly.
  - confidence: 0.74
