## Summary

Needs changes before merge: the latest PR head still leaves stale `pluginExtensionSlotKeys` ownership metadata behind when restart cleanup clears an obsolete promoted slot, which can later delete another plugin's mirror for the same freed slot.

## What This Changes

The PR adds plugin-owned `SessionEntry` slot projection for session extensions, scoped trusted-policy session extension reads, cleanup and validation hardening, tests, SDK API baseline updates, and a changelog entry.

## Best Possible Solution

Keep the additive SDK seam, but make restart cleanup remove or refresh obsolete `pluginExtensionSlotKeys` entries when it clears stale promoted mirrors, with a regression proving later slot owners are preserved.

## Reproduction Assessment

Yes for the review finding. Source inspection plus the current regression test show a restart that drops `sessionEntrySlotKey` clears the top-level mirror while preserving stale ownership metadata.

## Solution Assessment

No, not yet. The PR direction is narrow, but cleanup needs to treat persisted slot ownership as current ownership metadata and update it when stale mirrors are removed.
