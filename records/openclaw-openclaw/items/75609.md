## Summary

Keep open: current main lacks the requested SDK surface, and the PR head still has a restart cleanup bug that can leave stale slot ownership and delete preserved promoted mirrors.

## What This Changes

The PR adds plugin-owned SessionEntry slot projection, scoped trusted-policy session extension reads, cleanup hardening, tests, SDK API baseline updates, and a changelog entry.

## Best Possible Solution

Keep the additive SDK seam, but make restart promoted-slot cleanup prune only obsolete ownership entries and preserve still-declared mirrors, with regression coverage for multi-slot restarts and later new owners.

## Reproduction Assessment

Yes for the review finding. Source inspection of PR head plus the current regression expectation shows a restart that drops sessionEntrySlotKey clears the top-level mirror while preserving stale pluginExtensionSlotKeys ownership metadata.

## Solution Assessment

No, not yet. The proposed API direction is narrow, but the cleanup path needs to treat persisted slot ownership as mutable current metadata and update it when stale mirrors are removed.
