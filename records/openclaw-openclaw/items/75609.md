## Summary

Keep open. Current main still lacks the requested Plugin SDK session-entry slot projection seam, and this PR is the active implementation slice, but the latest head still has a security-sensitive config exposure through Codex tool-result plugin contexts and exact-head CI is not fully green.

## What This Changes

The PR adds plugin-owned SessionEntry slot projection for session extensions, scoped trusted-policy session-extension reads across tool surfaces, contract tests, Plugin SDK API baseline updates, and a changelog entry.

## Best Possible Solution

Keep the additive SDK seam, but split the runtime config plumbing so config is available only to trusted-policy session-extension lookup and never reaches Codex tool-result middleware or legacy extension contexts; then rerun exact-head validation before merge.

## Reproduction Assessment

Not applicable as a user bug reproduction. The review path is source-level: current main lacks the new API, and on the PR head the added Codex hookContext.config is reused by tool-result runners that pass context objects to plugin handlers.

## Solution Assessment

No, not as submitted. The slot projection and trusted-policy read helper are a reasonable narrow direction, but the context split must be tightened so full host config is not exposed outside the intended trusted-policy lookup path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep config out of Codex tool-result contexts:** `extensions/codex/src/app-server/dynamic-tools.ts:51`
  - body: Adding `config` to `hookContext` also sends it through the existing `...params.hookContext` spread into Codex tool-result middleware and the legacy app-server extension runner. Those runners pass their context object to plugin handlers, whose public context types do not include config, so this broadens full host config visibility beyond the trusted-policy session-read helper. Build a sanitized tool-result context without `config`, and pass config only to before-tool-call policy lookup.
  - confidence: 0.87
