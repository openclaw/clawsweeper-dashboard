## Summary

Keep open. Current main still lacks the session-entry slot projection API and trusted-policy session-extension reader, while this PR is the active implementation and appears to have addressed the earlier collision, cleanup, and config-exposure review concerns; the remaining actionable gap is public SDK documentation for the new fields.

## What This Changes

The PR adds optional plugin session-entry slot projection fields, mirrors projected session-extension state into guarded top-level SessionEntry slots, and lets trusted tool policies read plugin-owned session extension state across tool surfaces.

## Best Possible Solution

Land the narrow slot-projection seam after adding focused public SDK docs, keeping top-level SessionEntry slots mirror-only and trusted-policy session reads scoped away from normal hook contexts.

## Reproduction Assessment

Not applicable as a user bug reproduction. The source-level check is high confidence: current main lacks the new fields/helper, and the PR head adds contract tests for projection, cleanup, guards, and trusted-policy reads.

## Solution Assessment

Yes for the implementation approach: central validation, mirror-only writes through patchPluginSessionExtension, and policy-only session reads are the narrow maintainable shape; the safer follow-up is documentation rather than a code redesign.
