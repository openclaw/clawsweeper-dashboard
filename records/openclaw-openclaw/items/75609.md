## Summary

Keep open. Current main still lacks the requested Plugin SDK session-entry slot projection surface, and this PR is the active implementation slice. The latest head addresses the earlier duplicate-slot and host-config exposure feedback, but the slot-key guard still allows inherited Object names and exact-head CI is not yet clean.

## What This Changes

The PR adds `sessionEntrySlotKey` / `sessionEntrySlotSchema` metadata, mirrors projected plugin session-extension state into guarded top-level `SessionEntry` slots, adds scoped trusted-policy session-extension reads, and updates tests, changelog, and SDK API baseline hashes.

## Best Possible Solution

Keep the additive session-slot projection and scoped trusted-policy read design, but tighten the slot-key validator to reject all inherited Object names before merge and then require exact-head CI to pass.

## Reproduction Assessment

Yes. Static inspection is enough: `normalizeSessionEntrySlotKey("toString")` passes the PR head regex and reserved-set check, while the resulting mirror can be written as an own top-level `SessionEntry` property.

## Solution Assessment

Mostly yes. Passing config only through the trusted-policy options path is the narrowest fix for the earlier trust-boundary issue, but the reserved-key implementation should derive from or explicitly include Object prototype property names.

## Review Findings

Overall correctness: patch is correct
