## Summary

Keep open. Current main does not contain this session-entry slot projection SDK surface, and the PR is a fresh split from the closed predecessor stack rather than obsolete cleanup. The patch has a concrete reserved-key bug that lets a plugin collide with the current `subagentRecovery` session field, plus the required changelog entry is missing.

## What This Changes

This draft PR adds `sessionEntrySlotKey` and `sessionEntrySlotSchema` to plugin session extension registrations, mirrors projected extension values into guarded top-level `SessionEntry` slots, adds reserved-key validation and contract tests, and refreshes the Plugin SDK API baseline hash.

## Best Possible Solution

Keep the PR open as the current session-entry projection slice, fix the reserved-key guard so every current `SessionEntry` field is protected, add regression coverage for that collision, add the required Unreleased changelog bullet, and then continue normal maintainer review on the exact head.

## Reproduction Assessment

Yes. Register a session extension with `sessionEntrySlotKey: "subagentRecovery"` on the PR head and patch it; the guard accepts the key and `patchPluginSessionExtension` writes the projected value into the host-owned `SessionEntry.subagentRecovery` slot.

## Solution Assessment

No, not yet. The generic projection seam fits the host-hook direction, but the implementation is not safe until the reserved-key guard covers all current host-owned session fields and the public SDK change is reflected in release notes.
