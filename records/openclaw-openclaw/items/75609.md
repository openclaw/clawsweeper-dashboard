## Summary

Keep open. Current main does not have the requested SessionEntry slot projection or scoped policy read helper, and this PR is the active implementation slice; however, the latest head still appears to leak the full runtime config into Codex tool-result extension contexts, which should be fixed before merge.

## What This Changes

The PR adds plugin-owned SessionEntry projection slots, reserved-key validation, cleanup behavior, scoped trusted-policy session extension reads, config plumbing for tool-call surfaces, contract tests, API baseline updates, and a changelog entry.

## Best Possible Solution

Keep the additive SDK seam, but route runtime config only into trusted-policy session-extension lookup and keep Codex tool-result middleware/extension contexts limited to their documented agent/session/run identifiers.

## Reproduction Assessment

Not applicable as a user bug reproduction. The review path is source-level: current main lacks the new SDK fields/helper, while the PR head adds them and the remaining finding follows directly from the PR head's hookContext config addition plus existing context spreads into plugin-visible runners.

## Solution Assessment

No, not as submitted. The slot projection and scoped policy helper are a reasonable narrow implementation, but the config plumbing should be split so the full OpenClawConfig is not attached to unrelated Codex tool-result extension contexts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep config out of Codex tool-result contexts:** `extensions/codex/src/app-server/dynamic-tools.ts:47`
  - body: Adding `config` to `hookContext` also sends it through the existing `...params.hookContext` spread into the Codex tool-result middleware and legacy extension runners.
