## Summary

Keep open. Current main does not yet implement the requested Plugin SDK session-entry slot projection, and this draft PR is the active fresh-main slice replacing part of the closed consolidated stack. The latest head fixed the earlier `subagentRecovery` and changelog gaps, but the patch should not merge as-is because it adds `config` to the shared tool hook context that is also passed to ordinary `before_tool_call` plugin hooks, not only bundled trusted policies.

## What This Changes

The PR adds Plugin SDK session extension slot metadata, mirrors projected plugin session state into guarded top-level `SessionEntry` fields, exposes scoped session-extension reads to trusted tool policies, updates cleanup/contract tests, and refreshes the changelog plus SDK API baseline.

## Best Possible Solution

Keep the additive session-slot seam, but keep full config out of the public/shared tool hook context. Trusted policy session reads should be implemented through a private trusted-policy context or dedicated `runTrustedToolPolicies` parameter, with regression coverage proving ordinary `before_tool_call` hooks do not receive `ctx.config`.

## Reproduction Assessment

Yes. Static source tracing reproduces the bug: the PR adds `config` to the shared `toolContext`, and the existing code passes that same object into `hookRunner.runBeforeToolCall`, so any normal `before_tool_call` plugin can observe `ctx.config`.

## Solution Assessment

No. The slot projection direction is reasonable, but the current implementation solves trusted-policy reads by widening the ordinary hook context; the safer narrow fix is to pass config only inside the trusted-policy path and leave normal tool hooks on the existing context shape.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep config out of normal tool hooks:** `src/agents/pi-tools.before-tool-call.ts:482`
  - body: The added `config` property is placed on the shared `toolContext`, but that same object is later passed to `hookRunner.runBeforeToolCall`. This gives every ordinary `before_tool_call` plugin access to the full `OpenClawConfig`, even though the new read helper only needs config inside bundled trusted policies. Pass config through a private trusted-policy context or strip it before running normal hooks.
  - confidence: 0.89
- **[P3] Reserve inherited object slot names:** `src/plugins/session-entry-slot-keys.ts:1-3`
  - body: The new reserved-key set only blocks `__proto__`, `constructor`, and `prototype` from the object prototype family. Identifier keys such as `toString` and `hasOwnProperty` still pass validation and can be persisted as top-level `SessionEntry` fields, despite the guard's collision goal. Seed the guard from `Object.prototype` names or add those names explicitly with regression coverage.
  - confidence: 0.74
