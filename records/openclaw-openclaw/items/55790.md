## Summary

Keep open. Current main still has the reported native subagent thinking-resolution gap: omitted `sessions_spawn.thinking` can leave the child without the documented inherited thinking level, so the child falls back independently.

## What This Changes

The issue asks OpenClaw to make `sessions_spawn(runtime="subagent")` honor explicit, configured, and inherited thinking defaults instead of starting child sessions at a lower fallback level.

## Best Possible Solution

Fix the native subagent spawn planner so it computes, persists, and forwards the first child run's thinking level according to the documented precedence before the child `agent` call.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: omit explicit `thinking` while relying on inherited caller thinking, and current main can pass `undefined` into the child run. The issue also provides transcript-level proof from installed releases.

## Solution Assessment

Yes. The narrow maintainable fix belongs in native subagent thinking resolution/planning; downstream child fallback cannot recover the requester's effective thinking once the first child run starts without it.

Native `sessions_spawn(runtime="subagent")` currently resolves thinking only from explicit `sessions_spawn.thinking` plus subagent defaults and can pass `undefined` into the child `agent` call, so children fall back independently instead of inheriting the caller. Update the native subagent spawn thinking resolver/planner so omitted explicit thinking follows the documented precedence: explicit spawn override, configured subagent thinking default where applicable, caller effective thinking, then normal fallback only when none applies. Preserve ACP behavior and do not add a new config key. Add regression coverage for the reporter shape with a parent/requester session at high thinking and a child spawn without explicit thinking, plus coverage for explicit override still winning and existing subagent defaults still working.
