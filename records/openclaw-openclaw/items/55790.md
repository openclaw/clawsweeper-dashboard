## Summary

Keep open. Current main still matches the reported failure mode: native subagent spawn resolves only explicit `thinking` and `agents.*.subagents.thinking`, does not receive the caller's effective thinking level, and lets the child `agent` path fall back independently when no subagent-specific default is set.

## What This Changes

The issue requests that `sessions_spawn(runtime="subagent")` apply documented thinking precedence instead of initializing child sessions at a fallback level when explicit thinking is omitted.

## Best Possible Solution

Fix the native subagent spawn planning path so it computes the child thinking level before the first child `agent` call: explicit override first, target/global subagent thinking defaults next, caller effective thinking when no subagent default exists, then the child agent's normal thinking default/fallback. Persist the resolved child `thinkingLevel`, pass it to the gateway call, and add focused precedence tests.

## Reproduction Assessment

Yes. The issue and follow-up comment provide transcript-level reproductions, and current-main source inspection gives a deterministic unit-level reproduction: omit explicit `thinking` and subagent thinking defaults, and the resolver returns no thinking override or initial session patch.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the native subagent spawn planner/resolver; relying on the downstream child `agent` fallback is not sufficient because that path cannot see the caller's current thinking level and does not implement the documented subagent inheritance rule.
