## Summary

Keep open. Current main still has the reported gap: native subagent spawn can omit a child thinking override when `sessions_spawn.thinking` is absent, so the child falls back independently instead of inheriting the caller as documented.

## What This Changes

The issue asks `sessions_spawn(runtime="subagent")` to resolve child thinking from explicit spawn input, configured subagent defaults, caller inheritance, and only then normal fallback behavior.

## Best Possible Solution

Resolve native subagent thinking before the first child `agent` call using the documented precedence, persist the child `thinkingLevel`, and add regression coverage for caller inheritance plus requester-scoped subagent defaults.

## Reproduction Assessment

Yes. The issue provides transcript-level reproductions, and current-main source inspection gives a deterministic path: omit explicit `thinking` and the native subagent planner can return no child thinking override despite the documented inheritance rule.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the native subagent spawn resolver/planner and initial child session patch; relying on the downstream child agent fallback cannot inherit the requester session's effective thinking level.
