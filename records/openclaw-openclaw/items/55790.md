## Summary

Keep open. Current main and the latest shipped tag still resolve native subagent thinking only from explicit input or configured `subagents.thinking`, so the documented caller-inheritance path can still fall through to the child agent's normal default.

## What This Changes

The issue asks for `sessions_spawn(runtime="subagent")` to initialize child thinking from explicit spawn input, configured subagent defaults, or caller-inherited thinking instead of falling back to low/default behavior.

## Best Possible Solution

Resolve the effective child thinking level before the initial child session patch and first `agent` call, following the documented explicit/config/caller order and adding regression coverage for caller inheritance.

## Reproduction Assessment

Yes, at source level. Omitting explicit/configured child thinking lets the current resolver return no override despite the documented caller-inheritance rule, and the issue includes transcript-level examples of low child initialization.

## Solution Assessment

Yes. The narrow maintainable fix is in the native subagent thinking resolver/planner before the first child session patch and `agent` call; generic downstream agent defaults cannot recover the caller's effective thinking after spawn omits it.
