## Summary

Keep open. Current main still has the same source-level gap: native subagent spawn can omit a child thinking level entirely, so the first child run falls through to normal agent defaults instead of the documented caller inheritance path.

## What This Changes

The issue requests that `sessions_spawn(runtime="subagent")` honor explicit, per-agent, global, and caller-inherited subagent thinking defaults instead of falling back to low.

## Best Possible Solution

Resolve the effective child thinking level before creating the initial child session and first `agent` call, using the documented explicit/config/caller order, then add focused regression coverage for caller inheritance and configured subagent thinking defaults.

## Reproduction Assessment

Yes. The issue has transcript-level reproductions and current source still gives a deterministic path: omit explicit/configured child thinking and the native subagent planner can return no thinking override despite the documented caller-inheritance rule.

## Solution Assessment

Yes. The narrow maintainable fix is in the native subagent thinking resolver/planner and initial child session patch; downstream agent defaults cannot reconstruct the caller's effective thinking level after spawn omitted it.
