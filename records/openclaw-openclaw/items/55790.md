## Summary

Keep open: current main and the latest release still lack the documented caller-inheritance fallback for native subagent thinking, and https://github.com/openclaw/openclaw/pull/84007 is the open closing PR that should be reviewed before this issue closes.

## What This Changes

The issue requests that `sessions_spawn(runtime="subagent")` inherit configured or caller thinking defaults instead of initializing omitted-thinking children at `low`.

## Best Possible Solution

Review and land, or explicitly close, https://github.com/openclaw/openclaw/pull/84007; once it merges, close this issue with the merged commit and release proof.

## Reproduction Assessment

Yes, at source level. The docs promise caller inheritance, while current main returns no thinking override when explicit and configured subagent thinking are absent, so the child session patch and gateway call carry no inherited thinking.

## Solution Assessment

Yes. The best path is the linked focused PR or an equivalent narrow fix that resolves effective child thinking before the child session patch and first gateway agent call; a separate new fix PR would duplicate that work.
