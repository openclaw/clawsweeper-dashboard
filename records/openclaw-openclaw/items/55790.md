## Summary

Keep open: current main and v2026.5.20 still lack the documented caller-inheritance path for subagent thinking, and https://github.com/openclaw/openclaw/pull/84007 is the open closing PR that should be reviewed before this issue closes.

## What This Changes

The issue reports that `sessions_spawn(runtime="subagent")` initializes child sessions at low thinking instead of honoring explicit, configured, or caller-inherited subagent thinking defaults.

## Best Possible Solution

Review and land, or explicitly close, the linked fix PR; once it merges, close this issue with the merged commit and release proof.

## Reproduction Assessment

Yes, at source level. The docs promise caller inheritance, while current main returns no thinking override when explicit and configured subagent thinking are absent, so the child session patch and gateway call carry no inherited thinking.

## Solution Assessment

Yes, the best path is the linked focused PR or an equivalent narrow fix that resolves effective child thinking before the child session patch and first gateway agent call. A separate ClawSweeper fix PR would duplicate the open closing PR.
