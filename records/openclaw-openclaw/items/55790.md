## Label Justifications

- `impact:session-state`: The child session can persist and display a thinking level that does not match the caller or configured subagent policy.

## Summary

Keep open: current main and the latest shipped tag still omit the requester/caller thinking inheritance path for native `sessions_spawn`, so an omitted `thinking` can still fall through instead of honoring the documented caller policy.

## What This Changes

The issue asks OpenClaw to make native `sessions_spawn` resolve child thinking from explicit input, requester subagent defaults, global subagent defaults, and then the caller's effective thinking before falling back.

## Best Possible Solution

Resolve the effective child thinking level before the initial child session patch and first gateway `agent` call, using explicit input, requester subagent policy, global subagent defaults, and caller effective thinking in the documented order.

## Reproduction Assessment

Yes, at source level. The docs promise caller inheritance, while current main and `v2026.5.18` return no thinking override when explicit/configured child thinking is absent, matching the supplied transcripts.

## Solution Assessment

Yes. The narrow maintainable fix is to pass requester/caller thinking context into the native subagent planning path before the child session is patched or the gateway child run starts.
