## Summary

Keep open. Current main still has the reported source-level gap: native subagent spawn only resolves explicit or configured subagent thinking and can omit caller-inherited thinking, so the child falls through to normal agent defaults.

## What This Changes

The issue asks for `sessions_spawn(runtime="subagent")` to honor explicit, configured subagent, and inherited caller thinking defaults instead of starting children at the fallback low level.

## Best Possible Solution

Resolve the effective child thinking level before creating the initial child session and first `agent` call, using the documented explicit/config/caller order, then add regression coverage for caller inheritance and configured subagent thinking defaults.

## Reproduction Assessment

Yes, at source level. Omitting explicit/configured child thinking lets the current resolver return no override despite the documented caller-inheritance rule, and the issue provides transcript-level examples of the resulting low-level child initialization.

## Solution Assessment

Yes. The narrow maintainable fix is in the native subagent thinking resolver/planner before the initial child session patch and first child `agent` call; downstream generic agent defaults cannot recover the caller's effective thinking after spawn omits it.
