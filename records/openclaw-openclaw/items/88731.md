## Summary

Keep open: the doctor-owned migration direction matches OpenClaw policy, but the PR can drop retryable legacy voice-call records when the SQLite event store is already at capacity, and the PR body only provides test/lint proof.

## What This Changes

The PR removes voice-call runtime JSONL call-log reads and writes, adds a doctor migration from `calls.jsonl` into plugin-state SQLite, and updates voice-call store, restore, and webhook tests.

## Best Possible Solution

Before merge, doctor should preserve the combined newest voice-call records across existing SQLite and legacy JSONL, leave unimported sources retryable, and include upgrade proof for the migration path.

Yes, source-reproducible: seed 1000 existing SQLite call events plus a newer `calls.jsonl` entry, and the PR’s `eventRoom` calculation prunes the legacy entry before archiving the source. I did not execute tests because this review is read-only.

## Solution Assessment

No, not yet: moving legacy import into doctor is the right owner boundary, but the migration must preserve the combined newest records before runtime fallback removal is safe.
