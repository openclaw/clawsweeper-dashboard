## Summary

Keep open for maintainer review: the patch is focused and I found no blocking correctness or security defect, but it deliberately changes default message delivery for existing non-Discord parent channels already using progress mode and the PR has the protected maintainer label.

## What This Changes

The branch changes the ACP parent stream relay so progress-mode parent channels default to relaying assistant commentary/status progress, then updates ACP docs and relay regression coverage for generic and Telegram parent channels.

## Best Possible Solution

Use one documented parent-progress rule across channels: require progress mode, honor explicit `commentary=false`, and land the default flip only if maintainers accept the upgrade-visible extra progress messages.

## Reproduction Assessment

Yes from source: current main's relay passes a commentary default only for Discord, and the existing generic progress-mode regression expects no parent output. I did not execute tests because this was a read-only review.

## Solution Assessment

Yes, if maintainers accept the default change: switching the parent relay fallback to true is the narrowest runtime change because the shared helper still requires progress mode and honors explicit `commentary=false`. If the default flip is not intended, preserving the current opt-in behavior outside Discord is the safer alternative.
