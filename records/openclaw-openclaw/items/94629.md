The PR adds an Apple/iCloud context gate around app-specific-password-shaped redaction in generic structured fields and adds tests for preserved kebab-case identifiers, Apple-context masking, and dedicated Apple field keys.

## Best Possible Solution

Land one canonical redaction fix that masks dedicated Apple/iCloud fields unconditionally and evaluates Apple/iCloud/app-password context around each candidate token in generic fields, with transcript/log regression coverage.
