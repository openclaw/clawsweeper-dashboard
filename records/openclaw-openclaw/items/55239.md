## Summary

Keep open. Current main and the latest release include useful Telegram runtime support for plugin-provided `descriptionLocalizations`, from https://github.com/openclaw/openclaw/pull/81351, but the requested config-backed Chinese/native descriptions for built-in commands like `/help` and `/tts` are still not exposed. The broader open slash-command i18n tracker at https://github.com/openclaw/openclaw/issues/79458 overlaps the built-in metadata problem, but it does not fully replace this focused Telegram config request.

## What This Changes

Request OpenClaw config support for localized Telegram bot command descriptions, especially Chinese descriptions for built-in commands such as `/help` and `/tts`.

## Best Possible Solution

Define an additive Telegram localization contract that maps stable command names to per-locale descriptions, reuses the existing `descriptionLocalizations` and `setMyCommands(language_code)` path, and coordinates built-in command metadata with https://github.com/openclaw/openclaw/issues/79458.

## Reproduction Assessment

Yes for the source-level feature gap: current main still has no locale-keyed Telegram custom-command config and built-in `/help` and `/tts` remain English-only. A live Telegram run is not needed to prove the missing config/native localization surface.

## Solution Assessment

No complete implementation exists yet. The merged runtime work is a good foundation, but the requested user-configurable/native Chinese command descriptions still need an additive config and command-registry design.
