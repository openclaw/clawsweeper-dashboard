## Summary

Keep this issue open. Current main still exposes only one Telegram command-menu description per command and registers only the default command list, so the requested user-configurable Chinese/localized Telegram menu descriptions remain missing. The broad i18n tracker is useful context, but this can be scoped as a narrow Telegram plugin config/runtime feature rather than a repo-maintained translation catalog.

## What This Changes

The issue asks for Telegram Bot command-menu descriptions to be configurable per language, especially Simplified Chinese descriptions for commands such as /tts and /help.

## Best Possible Solution

Add a backwards-compatible Telegram plugin config surface for language-keyed command-menu description overrides, keep command tokens stable, register the default menu as today plus per-language menus through Telegram language_code, and update config metadata, docs, changelog, and focused Telegram tests.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is available: current config/schema/tests accept only one description per command, and the runtime calls setMyCommands without language_code, so no live Telegram bot is needed to confirm the missing surface.

## Solution Assessment

Yes for the proposed repair direction. The narrow maintainable fix is additive Telegram config plus per-language command registration; translating command names or adding a global repo translation catalog would be broader and riskier than the reporter’s concrete menu-description request.
