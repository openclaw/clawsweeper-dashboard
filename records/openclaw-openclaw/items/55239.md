## Summary

Keep open. Current main and v2026.6.10 have Telegram transport plumbing for `descriptionLocalizations`, but the requested configurable Chinese/built-in Telegram command-menu descriptions are still missing and need a maintainer API/product decision.

## What This Changes

The issue requests Chinese/localized Telegram bot command-menu descriptions, preferably configurable for commands such as `/tts` and `/help`.

## Best Possible Solution

Adopt one additive `descriptionLocalizations` locale-map contract for built-in commands and Telegram custom config, with schema/docs/tests, or explicitly consolidate this under the broader slash-command i18n tracker.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.10 shows localized Telegram registration only works when commands already carry `descriptionLocalizations`, while built-in commands and Telegram custom config still expose one English description string.

## Solution Assessment

Unclear. A Telegram config-only Chinese override would solve the immediate report, but the maintainable path is likely to reuse the existing `descriptionLocalizations` map and decide built-in translation ownership first.
