## Summary

Keep open. The merged Telegram work added the low-level `setMyCommands(language_code)` plumbing, but current main still lacks the requested user/config path for Chinese command-menu descriptions: Telegram custom commands only accept `command` and `description`, and built-in commands such as `/help` remain English-only.

## What This Changes

The issue requests configurable localized Telegram Bot command-menu descriptions, especially Chinese descriptions for commands such as `/tts` and `/help`.

## Best Possible Solution

Define a maintainer-approved Telegram localization surface that keeps command names stable, supports locale-keyed descriptions for the intended built-in/custom/plugin/skill command sources, and documents/tests the Chinese menu path end to end.

## Reproduction Assessment

Yes. Current main can register localized variants only when a command already has `descriptionLocalizations`, but the requested config-based Chinese path is still source-reproducibly missing because `TelegramCustomCommand` and built-in command definitions have no locale-keyed descriptions.

## Solution Assessment

No. The merged related PR is a useful partial plumbing fix, but it is not the full solution for this issue because it does not expose a user configuration surface or localized built-in command descriptions for normal Telegram bot menus.
