## Summary

Keep open. Current main includes the lower-level Telegram `descriptionLocalizations` and `language_code` registration path from the merged related PR, but it still does not provide the config-backed Chinese/native command-menu localization requested for commands such as `/help` and `/tts`.

## What This Changes

Support localized Telegram Bot command-menu descriptions, especially Chinese descriptions for commands such as `/tts` and `/help`, through OpenClaw configuration.

## Best Possible Solution

Define an additive Telegram config/native-command localization contract that maps stable command names to locale-specific descriptions and reuses the existing per-language `setMyCommands` runtime path.

## Reproduction Assessment

Yes for the source-level feature gap: current main still rejects locale-keyed Telegram custom-command config and built-in `/help` and `/tts` remain English-only. A live Telegram run is not needed to prove the requested config/native localization surface is still missing.

## Solution Assessment

No, not as a completed solution. The merged runtime work is a good foundation, but the requested user-configurable/native Chinese command descriptions still need an additive config and command-registry design.
