## Summary

Keep open. Current main and v2026.6.8 have Telegram `descriptionLocalizations` plumbing for plugin/skill command specs and per-language `setMyCommands` calls, but the requested configurable Chinese/built-in Telegram command-menu description path is still missing and needs a maintainer product/API decision.

## What This Changes

Request to support Chinese or other localized Telegram Bot command-menu descriptions, preferably configurable through OpenClaw config.

## Best Possible Solution

Use one additive locale-map contract, likely the existing `descriptionLocalizations` shape, then decide whether Telegram custom config and built-in commands should own Chinese translations before updating schema, docs, and focused menu tests.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main and v2026.6.8 expose only one Telegram custom-command description and one built-in command description, while localized Telegram registration only works when a command already carries `descriptionLocalizations`. No live Telegram run was needed to prove the config/API gap.

## Solution Assessment

Unclear as requested. A config-only Chinese override would address the reporter's immediate Telegram use case, but the safer permanent solution is to extend the existing `descriptionLocalizations` map consistently and decide bundled translation ownership first.
