## Summary

Keep open. Current main and v2026.6.10 have partial Telegram `descriptionLocalizations` transport support, but the requested configurable or built-in Chinese command-menu descriptions are still missing and need a maintainer API/product decision.

## What This Changes

The issue requests localized Telegram Bot command-menu descriptions, especially Chinese descriptions for commands such as `/tts` and `/help`, preferably configurable per language.

## Best Possible Solution

Use one additive locale-map contract, likely the existing `descriptionLocalizations` shape, for built-in commands and Telegram custom config, with schema/docs/tests; alternatively explicitly consolidate the remaining work under https://github.com/openclaw/openclaw/issues/79458.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.10 shows localized Telegram registration only works when commands already carry `descriptionLocalizations`, while built-in commands and Telegram custom config still expose one English/default description string.

## Solution Assessment

Unclear. A Telegram config-only Chinese override would solve the immediate report, but the maintainable path is likely to extend the existing `descriptionLocalizations` map consistently after maintainers choose translation ownership and API scope.
