## Summary

Keep open. Current main still lacks locale-keyed Telegram command-menu descriptions: the config accepts one description string, menu assembly flattens native/plugin/custom commands into one command list, and sync calls `setMyCommands` without `language_code`.

## What This Changes

The issue asks OpenClaw to support Chinese and other localized descriptions for Telegram Bot command-menu entries such as `/tts` and `/help`.

## Best Possible Solution

Add a Telegram plugin config/runtime path for per-language command descriptions, register fallback plus language-specific command lists with Telegram `language_code`, and update docs, metadata, and focused tests.

## Reproduction Assessment

Yes for the missing feature: current source shows one description string per Telegram command and no `language_code` passed to `setMyCommands`. A live Telegram run is not needed to prove that locale-keyed menu descriptions cannot currently be configured.

## Solution Assessment

Yes. The narrow maintainable direction is a Telegram plugin config/runtime/docs/test change for localized descriptions while keeping slash command tokens stable, not a broad repo-wide i18n catalog.
