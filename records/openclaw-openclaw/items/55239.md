## Summary

Keep open. Current main and the latest release have useful Telegram runtime plumbing for plugin-provided `descriptionLocalizations`, but the requested config-backed Chinese/native descriptions for built-in commands such as `/help` and `/tts` are still not exposed; the broader open slash-command i18n issue overlaps but does not replace this focused Telegram config request.

## What This Changes

The issue requests configurable Chinese/localized descriptions for Telegram bot command-menu entries such as `/tts` and `/help`.

## Best Possible Solution

Define an additive Telegram localization contract that maps stable command names to per-locale descriptions, reuses the existing `descriptionLocalizations` and `setMyCommands(language_code)` path, and coordinates built-in translation metadata with https://github.com/openclaw/openclaw/issues/79458.

## Reproduction Assessment

Yes for the source-level feature gap: current main still rejects locale-keyed Telegram custom-command config and built-in `/help` and `/tts` remain English-only. A live Telegram run is not needed to prove the missing config/native localization surface.

## Solution Assessment

No complete implementation exists yet. The merged runtime work is a good foundation, but the requested user-configurable/native Chinese command descriptions still need an additive config and command-registry design.
