## Summary

Keep open. Current main and v2026.6.8 have Telegram `descriptionLocalizations` plumbing for plugin/skill command specs and per-language `setMyCommands` calls, but the requested config/built-in Chinese command-menu description path is still absent; the broader open i18n issue overlaps but does not replace this Telegram-specific config request.

## What This Changes

The issue requests configurable localized Telegram bot command-menu descriptions, especially Chinese descriptions for commands such as `/tts` and `/help`.

## Best Possible Solution

Adopt one additive locale-map contract, preferably `descriptionLocalizations`, and wire it through Telegram custom commands, built-in command definitions, docs, and focused Telegram menu tests after maintainer product approval.

## Reproduction Assessment

Yes, source-level. Current main shows Telegram can register localized variants only when commands already carry `descriptionLocalizations`, while Telegram `customCommands` and `defineChatCommand` still expose only one description string.

## Solution Assessment

No, not as a standalone Chinese-only config knob. The maintainable solution is a single maintainer-approved localization contract that uses the existing locale-map shape across config, built-in commands, plugin/skill commands, docs, and tests.
