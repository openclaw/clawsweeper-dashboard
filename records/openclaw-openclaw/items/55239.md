## Summary

Keep open. Current main and v2026.6.8 have the Telegram `language_code` plumbing for commands that already carry `descriptionLocalizations`, but the user-requested config-based Chinese command-menu description path is still absent from Telegram custom-command config and built-in command definitions.

## What This Changes

Requests configurable localized Chinese descriptions for Telegram bot command-menu entries such as `/tts` and `/help`.

## Best Possible Solution

Choose one additive locale-map contract, preferably the existing `descriptionLocalizations` shape, then wire it through Telegram built-in, custom, plugin, and skill command sources with docs and focused tests.

## Reproduction Assessment

Yes. Current main shows Telegram registers localized variants only when commands already carry `descriptionLocalizations`, while `customCommands` and `defineChatCommand` still expose only one description string.

## Solution Assessment

No. The merged Telegram plumbing is useful but incomplete; the maintainable fix is a single maintainer-approved localization contract across config, built-in commands, plugins, skills, docs, and tests.
