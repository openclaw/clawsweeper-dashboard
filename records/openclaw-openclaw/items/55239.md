## Summary

Keep open. Current main and the latest release have partial Telegram runtime support for plugin/skill-provided localized command descriptions, but the requested config-backed Chinese descriptions for Telegram custom and built-in command menus are still missing and overlap an unresolved slash-command i18n design decision.

## What This Changes

The issue requests configurable localized, specifically Chinese, descriptions for Telegram bot command menus.

## Best Possible Solution

Define one additive locale-map contract that reuses descriptionLocalizations, keeps command tokens stable, feeds Telegram setMyCommands language_code, and coordinates Telegram config with the broader slash-command description i18n design.

## Reproduction Assessment

Yes for the source-level feature gap: current main still lacks a locale-keyed Telegram custom-command description config path, and built-in command declarations remain English-only. A live Telegram run is not needed to prove the missing config surface.

## Solution Assessment

No complete solution exists yet. The merged Telegram runtime work is the right foundation, but the requested config-backed Chinese/native descriptions still need a maintainer-approved config and command-registry design.
