## Summary

Keep open. Current main has partial Telegram runtime support for per-locale command-menu descriptions from the merged localized menu work, but the issue specifically asks for config-backed Chinese descriptions for Telegram bot commands such as `/help` and `/tts`, and that public config surface is still missing.

## What This Changes

The issue requests OpenClaw configuration support for localized Telegram bot command-menu descriptions, especially Chinese descriptions for built-in commands.

## Best Possible Solution

Define an additive localization contract that lets Telegram register per-locale descriptions while keeping command names stable, reusing the existing runtime `descriptionLocalizations` path and coordinating built-in command metadata with https://github.com/openclaw/openclaw/issues/79458.

## Reproduction Assessment

Yes for the source-level feature gap: current main still lacks a locale-keyed Telegram command-description config path, and built-in command descriptions remain English-only. A live Telegram run is not needed to prove the missing config surface.

## Solution Assessment

No complete implementation exists yet. The existing Telegram runtime work is the right foundation, but the requested config-backed Chinese/native descriptions still need an additive config and command-registry design.
