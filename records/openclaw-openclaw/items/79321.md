## Summary

Keep open: current main has adjacent text completion, media-understanding, and JSON-only tool surfaces, but none provides the requested plugin SDK provider for bounded schema-validated text+image extraction. This is valid additive platform API work that needs maintainer design.

## What This Changes

Add a generic OAuth-backed structured extraction SDK/runtime hook so plugins can request bounded text/image JSON extraction without user model API keys or product-specific core routes.

## Best Possible Solution

Define an additive, vendor-agnostic SDK/runtime contract for structured extraction, with host-owned auth/model resolution, text and image inputs, JSON/schema validation, bounded execution, and explicit no-tools/no-approval guarantees.

## Reproduction Assessment

Not applicable; this is an additive SDK capability request, not a reported runtime failure. Source inspection does show the gap: current LLM completion is text-only and current media understanding returns text descriptions/transcripts rather than schema-validated JSON extraction.

## Solution Assessment

Yes at the goal level: a generic host-owned SDK capability is more maintainable than a GBrain- or Codex-specific route. The exact API needs maintainer product/security design before implementation.
