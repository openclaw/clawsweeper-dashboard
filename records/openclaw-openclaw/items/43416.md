## Summary

Keep open. The assistant-message copy affordance is useful and is not already present on current main, but this PR still copies raw assistant payload text while the renderer can hide thinking segments, so it can put non-visible trace content on the clipboard; it also needs the required changelog entry.

## What This Changes

The PR adds a native OpenClawChatUI copy button for assistant messages with macOS hover visibility, iOS clipboard support, and copied-state feedback.

## Best Possible Solution

Ship the native copy affordance after deriving clipboard text from visible AssistantTextParser response segments, adding focused Swift coverage for hidden-trace copying, and adding the required changelog entry.

## Reproduction Assessment

Yes for the PR patch. A source-level reproduction is an assistant message like `<think>secret</think><final>answer</final>` with `showsAssistantTrace == false`: rendering shows only the response segment, while the PR passes the raw primary text to the copy button.

## Solution Assessment

No. The copy feature direction is reasonable, but the implementation should copy the same visible text transformation used by ChatAssistantTextBody instead of raw primaryText.
