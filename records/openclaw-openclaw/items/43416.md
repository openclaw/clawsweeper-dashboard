## Summary

Keep open: current main still lacks this native assistant-message copy affordance, but the PR should not merge as-is because it copies the raw assistant payload instead of the visible rendered assistant text, and the submitted proof is not enough for an external UI/clipboard change.

## What This Changes

The PR adds a SwiftUI copy button to native OpenClawKit assistant bubbles with macOS/iOS pasteboard writes, macOS hover visibility, and temporary copied-state feedback.

## Best Possible Solution

Derive clipboard text from the same visible assistant segments used for the current trace setting, add focused parser/copy coverage, and require redacted native proof before merge.

## Reproduction Assessment

Yes for the blocking PR finding by source inspection: with traces off, a payload like `<think>internal</think><final>answer</final>` renders only response text while the PR copies raw `primaryText`. I did not run a native app reproduction in this read-only review.

## Solution Assessment

No. The copy affordance is reasonable, but the implementation should copy the same visible rendered assistant text rather than the raw payload string.
