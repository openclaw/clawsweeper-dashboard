## Summary

Keep open: current main does not already provide this native assistant-message copy control, and the PR's clipboard payload is inconsistent with the trace-aware rendering path, creating a privacy-sensitive mismatch.

## What This Changes

The PR adds a native OpenClawChatUI assistant-message copy button with macOS/iOS pasteboard support and copied-state feedback.

## Best Possible Solution

Ship the native copy affordance on the current OpenClawChatUI code path, but derive clipboard text from the same visible `AssistantTextParser` response segments used for rendering, keep touch and pointer behavior usable, and add focused Swift coverage plus the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is an assistant message like `<think>secret</think><final>answer</final>` with `showsAssistantTrace == false`: current rendering shows only the response segment, while this PR copies the raw primary text.

## Solution Assessment

No. The feature direction is useful, but the implementation should compute a visible clipboard string from `AssistantTextParser.segments(from:includeThinking:)` instead of writing raw payload text.
