## Summary

Keep open: the PR is focused and useful, but it copies raw assistant message text even though current rendering can hide assistant thinking, so it needs a visible-text clipboard fix and stronger behavior proof before merge.

## What This Changes

The PR adds a copy button to assistant messages in the shared Swift chat UI, with macOS/iOS pasteboard handling and temporary copied-state feedback.

## Best Possible Solution

Land the feature only after the copy payload is derived from visible assistant segments under the current trace setting, with proof that hidden thinking is not copied when traces are off.

## Reproduction Assessment

Yes from source inspection: with an assistant message like `<think>hidden</think><final>answer</final>` and `showsAssistantTrace` false, current rendering filters thinking while the PR copies raw `primaryText`. I did not run the Swift app because this review is read-only.

## Solution Assessment

No. The UI affordance is reasonable, but the maintainable implementation should copy the same text currently visible to the user rather than the raw assistant payload.
