## Summary

Keep open: the PR still copies raw assistant payload text, so hidden thinking tags can reach the platform clipboard, and the attached screenshot is not enough real behavior proof for the fixed behavior.

## What This Changes

The PR adds a native SwiftUI copy button to assistant message bubbles using platform pasteboards and temporary checkmark feedback.

## Best Possible Solution

Keep the native copy affordance only if it derives clipboard text from the same visible assistant segments the UI renders, with focused coverage and redacted native proof.

## Reproduction Assessment

Yes for the blocking PR finding by source inspection: with traces off, `<think>internal</think><final>answer</final>` renders only response text, while the added button copies raw `primaryText`. I did not run a native app reproduction in this read-only review.

## Solution Assessment

No. The copy affordance is a reasonable direction, but the implementation should copy rendered visible assistant text rather than the raw payload string.
