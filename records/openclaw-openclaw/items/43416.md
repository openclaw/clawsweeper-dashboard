## Summary

Keep open: the feature is useful, but the patch is not merge-ready because the copy action writes raw assistant payload text to the clipboard while the rendered UI can hide thinking segments.

## What This Changes

The PR adds a SwiftUI copy button to assistant chat bubbles in OpenClawKit, with hover visibility on macOS, pasteboard writes on macOS/iOS, and copied-state icon feedback.

## Best Possible Solution

Derive clipboard text from the same visible assistant segments used for the active trace setting, add focused parser/copy coverage, then collect redacted native proof of hover, copy, paste, and reset behavior.

## Reproduction Assessment

Yes, from source inspection: with an assistant message like `<think>hidden</think><final>answer</final>` and `showsAssistantTrace` false, current rendering filters thinking while the PR copies raw `primaryText`. I did not run the Swift app in this read-only review.

## Solution Assessment

No. The UI affordance is reasonable, but the maintainable implementation should copy the same visible text currently shown to the user rather than the raw assistant payload.
