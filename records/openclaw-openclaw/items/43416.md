## Summary

Keep open. The native Swift copy-button feature is not on current main, but this PR is not merge-ready because it copies the raw assistant payload while the renderer can hide `<think>` segments from the visible chat transcript.

## What This Changes

The PR adds a hover/touch copy button to assistant messages in the shared OpenClawKit Swift chat UI, writing the message text to the platform clipboard and showing temporary copied feedback.

## Best Possible Solution

Copy from the same visible assistant response segments used by rendering, add focused coverage for hidden tagged output, and provide redacted native proof of hover/tap, copy feedback, and pasted clipboard text.

## Reproduction Assessment

Yes for the blocking review finding. Source inspection shows a message like `<think>internal</think><final>answer</final>` renders only the response text with traces off, while this PR would copy the raw payload.

## Solution Assessment

No. The copy affordance is a reasonable feature direction, but the implementation should derive clipboard text from the rendered visible assistant segments instead of passing raw `primaryText` to the pasteboard.

## Review Findings

Overall correctness: patch is incorrect
