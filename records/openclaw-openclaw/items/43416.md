## Summary

Keep open: current main does not already have this native Swift copy affordance, and the PR still has a blocking clipboard mismatch with hidden assistant trace rendering plus a missing changelog entry.

## What This Changes

The PR adds a SwiftUI copy button to assistant message bubbles with platform pasteboard writes, macOS hover visibility, and copied-state feedback.

## Best Possible Solution

Ship the copy affordance after deriving clipboard text from visible assistant parser segments, adding focused hidden-thinking coverage, and adding the changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is an assistant message like `<think>secret</think><final>answer</final>` with `showsAssistantTrace == false`: current rendering hides the thinking segment, while the PR copies raw `primaryText`.

## Solution Assessment

No. The feature direction is useful, but the clipboard payload should be computed from `AssistantTextParser.segments(from:includeThinking:)` using the current trace visibility instead of writing the raw payload string.
