## Summary

Keep open: this is a focused native UI feature, but the branch is not merge-ready because it copies raw assistant payload text instead of the rendered visible text and still lacks convincing click/paste proof. Current main does not already provide this native OpenClawKit copy affordance, and the related Control UI copy/print issues do not supersede this PR.

## What This Changes

The PR adds a hover-revealed copy button to native OpenClawKit assistant message bubbles, using NSPasteboard/UIPasteboard and transient checkmark feedback.

## Best Possible Solution

Update the native copy action to copy the same visible text transformation the renderer shows, add focused coverage for hidden-thinking payloads, and require real native click/paste proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence PR-branch reproduction path: an assistant payload like `<think>internal</think><final>answer</final>` renders only the visible response when traces are off, while the new button copies raw `primaryText`.

## Solution Assessment

No. The copy affordance is useful, but the implementation should copy the same visible text transformation used by the renderer and then prove the native click/paste path.
