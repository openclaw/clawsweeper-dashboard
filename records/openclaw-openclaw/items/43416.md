## Summary

## What This Changes

The branch adds a SwiftUI hover-revealed copy button to native OpenClawChatUI assistant bubbles and writes copied text through NSPasteboard/UIPasteboard with temporary checkmark feedback.

## Best Possible Solution

Compute clipboard text from the same visible AssistantTextParser segments used for the active trace setting, cover the hidden-thinking case, then collect redacted native proof of hover, copy, paste, and reset behavior.

## Reproduction Assessment

Yes, from source inspection: with an assistant message like `<think>hidden</think><final>answer</final>` and `showsAssistantTrace` false, rendering filters thinking while the PR copies raw `primaryText`. I did not run the Swift app in this read-only review.

## Solution Assessment

No. The UI affordance is reasonable, but the maintainable implementation should copy the same visible text currently shown to the user rather than the raw assistant payload.
