## Summary

Keep open. The feature is not on current main, and the branch still copies raw assistant payload text while the renderer can hide `<think>` segments, so it needs a code fix and stronger native proof before merge.

## What This Changes

The PR adds a native Swift chat copy button for assistant messages, writing message text to NSPasteboard/UIPasteboard with temporary checkmark feedback.

## Best Possible Solution

Keep the native copy affordance only if it copies the same rendered visible assistant segments, with focused coverage and native proof for hidden-thinking payloads.

## Reproduction Assessment

Yes for the blocking review finding by source inspection: a `<think>internal</think><final>answer</final>` payload renders only the response text with traces off, while the PR copies the raw string. I did not run a native app reproduction in this read-only review.

## Solution Assessment

No. The copy affordance is a reasonable direction, but the implementation should derive clipboard text from the same parsed segments the UI renders instead of raw `primaryText`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92
