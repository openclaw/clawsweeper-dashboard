## Summary

Keep open: the feature is not implemented on current main and the PR is focused, but the latest diff still copies raw assistant payload text while current rendering can hide or strip thinking/final tags before display.

## What This Changes

The PR adds a SwiftUI copy button to native OpenClawKit assistant message bubbles with macOS hover visibility, iOS pasteboard support, and temporary copied-state feedback.

## Best Possible Solution

Derive clipboard text from the same visible assistant segments used by the renderer for the active trace setting, add focused parser/copy coverage, then provide redacted native proof of hover, click, paste, and copied-state feedback.

## Reproduction Assessment

Yes for the blocking PR finding by source inspection: with traces off, a payload like `<think>internal</think><final>answer</final>` renders only response text while the PR copies raw `primaryText`. I did not run a native app reproduction in this read-only review.

## Solution Assessment

No. The copy affordance is reasonable, but the implementation should copy the same visible rendered assistant text rather than the raw payload string.
