## Summary

Keep open: the native copy affordance is useful, but the current patch still has a merge-blocking clipboard-source bug that can copy hidden assistant thinking text, and the attached screenshot does not prove the fixed click/paste path.

## What This Changes

The PR adds a hover-revealed SwiftUI copy button for native assistant messages, with macOS/iOS pasteboard writes and temporary copied-state feedback.

## Best Possible Solution

Copy from the same visible AssistantTextParser segments used by the active trace setting, cover the hidden-thinking case, and then land with redacted native proof.

## Reproduction Assessment

Yes, from source inspection: with an assistant payload like `<think>hidden</think><final>answer</final>` and traces disabled, the renderer filters thinking while the PR copies raw `primaryText`. I did not run the native app in this read-only review.

## Solution Assessment

No. The affordance is reasonable, but the implementation should copy the same visible text the renderer shows instead of the raw assistant payload.
