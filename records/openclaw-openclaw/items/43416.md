## Summary

Keep open. The copy affordance is useful, but the PR still copies the raw assistant payload to the platform clipboard while current rendering can hide thinking segments, and the submitted screenshot is not enough real behavior proof for the changed copy path.

## What This Changes

The PR adds a SwiftUI copy button to native assistant chat bubbles in OpenClawKit, including hover state, macOS/iOS pasteboard writes, and temporary copied-state feedback.

## Best Possible Solution

Derive clipboard text from the same visible assistant segments rendered for the current trace setting, add focused parser/copy coverage, and require redacted native proof before merge.

## Reproduction Assessment

Yes for the blocking PR finding by source inspection: with traces off, a tagged assistant payload such as `<think>internal</think><final>answer</final>` renders only the response text, while the added button copies raw `primaryText`. I did not run a native app reproduction in this read-only review.

## Solution Assessment

No. The copy affordance is a reasonable direction, but the implementation should copy the same rendered visible assistant text rather than the raw payload string.
