## Summary

Keep open: current main does not already provide this native Swift assistant-message copy affordance, but the PR is not merge-ready because it copies raw assistant payload text that can include hidden thinking content and it lacks the required changelog entry.

## What This Changes

The PR adds a native Swift copy button for assistant message bubbles, with platform pasteboard writes and copied-state feedback.

## Best Possible Solution

Ship the native copy affordance after deriving clipboard text from the same visible assistant segments used for rendering, adding focused Swift coverage for hidden thinking, and adding a single-line changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is an assistant message such as `<think>secret</think><final>answer</final>` with `showsAssistantTrace == false`: current rendering shows only the response segment, while this PR copies raw `primaryText`.

## Solution Assessment

No. The feature direction is useful, but the implementation should compute copy text from `AssistantTextParser.segments(from:includeThinking:)` with the current trace visibility instead of writing the raw payload string.
