## Summary

Keep open: the native copy affordance is useful, but this branch is not merge-ready because it copies raw assistant payload text to the system clipboard while current rendering can hide thinking segments, and the screenshot only proves the icon appears.

## What This Changes

The PR adds a hover-triggered native OpenClawKit assistant-message copy button that writes message text to NSPasteboard/UIPasteboard and shows a temporary checkmark state.

## Best Possible Solution

Keep the native affordance concept only if it copies the same visible assistant text segments the renderer shows and includes redacted after-fix click/paste proof for the native app.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: an assistant payload such as `<think>internal</think><final>answer</final>` renders only the visible response when traces are off, while this PR copies raw `primaryText`. I did not run the native app in this read-only review.

## Solution Assessment

No. The copy affordance is reasonable, but the implementation should copy the same visible text transformation as the renderer and then prove the native click/paste path.
