## Summary

Keep open: current main still does not have this native assistant-message copy affordance, but the PR has a blocking clipboard bug because it copies raw assistant payload text while current rendering can hide thinking segments. The contributor proof is also insufficient for the changed copy behavior.

## What This Changes

The PR adds a SwiftUI copy button to native assistant message bubbles, using NSPasteboard/UIPasteboard with temporary checkmark feedback.

## Best Possible Solution

Keep the copy affordance only if it derives clipboard text from the same visible assistant segments that the UI renders, with focused parser coverage and native proof for hidden-thinking payloads.

## Reproduction Assessment

Yes for the blocking review finding by source inspection: a tagged assistant payload such as `<think>internal</think><final>answer</final>` renders only the response text when traces are off, while this PR copies raw `primaryText`. I did not run a native app reproduction.

## Solution Assessment

No. The copy affordance is a reasonable direction, but the implementation should derive the clipboard string from `AssistantTextParser` visible/rendered segments instead of raw payload text.
