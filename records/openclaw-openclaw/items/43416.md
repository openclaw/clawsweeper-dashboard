## Summary

Keep open because the feature is still useful, but the current PR can copy raw hidden assistant payload text to the clipboard and the attached screenshot does not prove the click, paste, copied-state, or hidden-thinking behavior.

## What This Changes

The PR adds a SwiftUI copy button to native OpenClawKit assistant messages with platform pasteboard writes, hover visibility on macOS, and temporary copied-state feedback.

## Best Possible Solution

Derive clipboard text from the same visible assistant segments used by the renderer for the active trace setting, add focused coverage for hidden-thinking payloads, and require redacted native behavior proof before merge.

## Reproduction Assessment

Yes for the blocking PR finding by source inspection: with traces off, a payload like `<think>internal</think><final>answer</final>` renders only response text while the PR copies raw `primaryText`. I did not run a native app reproduction in this read-only review.

## Solution Assessment

No. The copy affordance is reasonable, but the implementation should copy the same visible rendered assistant text rather than the raw payload string.
