## Summary

Keep this PR open, but do not merge the current patch. The native copy-button feature is not already implemented on current main, and the PR is still relevant, but it copies raw assistant payload text that the renderer may intentionally hide from the visible message.

## What This Changes

The PR adds a native OpenClawChatUI copy control for assistant message bubbles, using macOS/iOS pasteboard APIs and a temporary copied-state icon.

## Best Possible Solution

Ship the native copy affordance only after the clipboard string is derived from the same visible assistant segments as rendering, with focused Swift coverage and a single-line changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is an assistant message like `<think>secret</think><final>answer</final>` with `showsAssistantTrace == false`: current rendering shows only response segments, while this PR copies the raw payload text.

## Solution Assessment

No. The feature direction is good, but the patch should compute clipboard text from `AssistantTextParser.segments(from:includeThinking:)` using the same visibility setting as the UI before it writes to the pasteboard.
