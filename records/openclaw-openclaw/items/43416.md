## Summary

Keep open: current main does not have this native Swift assistant-message copy button, and the PR still copies raw assistant payload text, which can put hidden thinking content or tags on the clipboard instead of only the visible rendered message.

## What This Changes

The PR adds a hover/tap copy button to native Swift assistant message bubbles, using platform pasteboards and temporary checkmark feedback.

## Best Possible Solution

Keep the native copy affordance only if it derives clipboard text from the same visible assistant segments the UI renders, with focused coverage and redacted native proof.

## Reproduction Assessment

Yes for the blocking PR finding by source inspection: a `<think>internal</think><final>answer</final>` payload renders only response text with traces off, while the PR copies the raw string. I did not run a native app reproduction in this read-only review.

## Solution Assessment

No. The copy affordance is a reasonable direction, but the implementation should copy the rendered visible assistant text rather than raw `primaryText`.
