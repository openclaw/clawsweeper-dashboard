## Summary

Keep open: the copy affordance is useful, but the current diff copies the raw assistant payload to the system clipboard, including hidden reasoning tags that the UI intentionally filters out.

## What This Changes

The PR adds a SwiftUI copy button to assistant chat messages with hover visibility, platform pasteboard writes, and copied-state icon feedback.

## Best Possible Solution

Ship the copy button after deriving clipboard text from the same visible assistant segments used for rendering, with focused parser/copy coverage and real after-click proof.

## Reproduction Assessment

Yes, at source level. A message like `<think>secret</think><final>answer</final>` with `showsAssistantTrace == false` renders only the visible response, while the PR copies raw `primaryText`.

## Solution Assessment

No. The feature direction is reasonable, but the pasteboard payload should be computed from `AssistantTextParser.segments(from:includeThinking:)` using the current trace visibility instead of copying raw payload text.
