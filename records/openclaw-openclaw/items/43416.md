## Summary

Keep open: the native copy affordance is useful, but the PR still copies raw assistant payload text to the system clipboard, which can expose hidden thinking content that the UI suppresses.

## What This Changes

Adds a SwiftUI copy button to native assistant message bubbles with macOS/iOS pasteboard writes and temporary copied-state feedback.

## Best Possible Solution

Keep the native copy affordance only if it derives clipboard text from the same visible assistant segments the UI renders, with focused parser coverage and redacted native proof.

## Reproduction Assessment

Yes for the blocking PR finding by source inspection: with traces off, `<think>internal</think><final>answer</final>` renders only response text, while the added button copies raw `primaryText`. I did not run a native app reproduction in this read-only review.

## Solution Assessment

No. The copy affordance is a reasonable direction, but the implementation should copy rendered visible assistant text rather than the raw payload string.
