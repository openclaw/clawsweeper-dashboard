## Summary

Keep open: the native copy-button feature is useful, but this PR is not merge-ready because it copies the raw assistant payload and can put hidden thinking text or parser tags on the clipboard when traces are disabled; the screenshot proof only shows the icon.

## What This Changes

The PR adds a hover-revealed SwiftUI copy button to native OpenClawChatUI assistant messages that writes message text to NSPasteboard/UIPasteboard and shows temporary copied feedback.

## Best Possible Solution

Copy from the same visible assistant segments used by the active trace setting, add focused parser/copy coverage, then collect redacted native proof of hover, copy, paste, and reset behavior.

## Reproduction Assessment

Yes, from source inspection: with an assistant message like `<think>hidden</think><final>answer</final>` and `showsAssistantTrace` false, current rendering filters thinking while the PR copies raw `primaryText`. I did not run the Swift app in this read-only review.

## Solution Assessment

No. The UI affordance is reasonable, but the maintainable implementation should copy the same visible text currently shown to the user rather than the raw assistant payload.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92
