## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Wire external CLI overlays to async readers:** `src/agents/cli-credentials.ts:838-854`
  - body: The new async cached readers are not used by the runtime external CLI overlay path, which still calls the synchronous Codex and Claude cached readers. Gateway auth bootstrap/fallback can therefore still hit sync keychain or file I/O after this patch.
  - confidence: 0.9
- **[P2] Use async readers on the default auth-epoch path:** `src/agents/cli-auth-epoch.ts:184-195`
  - body: With production deps, `hasAsyncDepOverride` is false, so this branch still calls the synchronous cached reader. The normal runtime path should await the async reader and reserve any sync fallback for sync-only test overrides.
  - confidence: 0.92
- **[P2] Remove sync file reads from async credential readers:** `src/agents/cli-credentials.ts:617`
  - body: `readClaudeCliCredentialsAsync` still falls back to `loadJsonFile`, and the Codex async reader similarly uses sync JSON/stat helpers. File-backed credentials can still block the event loop when keychain is unavailable or disabled.
  - confidence: 0.86
- **[P2] Route model config commands through async helpers:** `src/agents/sessions/resolve-config-value.ts:193-220`
  - body: The PR adds async config helpers but leaves `ModelRegistry.getApiKeyAndHeaders` on the synchronous `resolveConfigValueOrThrow` and `resolveHeadersOrThrow` calls. Custom provider `!command` API keys or headers can still block during model resolution.
  - confidence: 0.84
- **[P2] Make Windows configured-shell execution async:** `src/agents/sessions/resolve-config-value.ts:151-154`
  - body: The async config resolver still calls `executeWithConfiguredShell` on Windows, and that helper uses `spawnSync`. A `!command` API key or header can still block on the new async path.
  - confidence: 0.82
