  - body: The PR adds async cached readers here, but the external CLI auth provider table is unchanged and still calls the sync Codex and Claude cached readers. That reachable auth-provider path can still hit the sync keychain/file helpers, so the linked Gateway auth stall can remain after merge.

  - body: With default deps, `hasAsyncDepOverride` is false, so production calls the sync cached reader and only uses the async reader when tests override the async dependency. Invert the selection so the default runtime path uses the async reader and only sync-only test overrides fall back.
  - confidence: 0.92
- **[P2] Remove sync file fallbacks from async readers:** `src/agents/cli-credentials.ts:617-618`
  - body: `readClaudeCliCredentialsAsync` still calls `loadJsonFile`, and the Codex async reader has the same sync file read plus `fs.statSync` fallback. File-backed credentials still block the event loop when keychain is unavailable or disabled.
  - confidence: 0.84

  - body: The async config resolver still calls `executeWithConfiguredShell` on Windows, but that helper uses `spawnSync`. A `!command` API key or header on Windows can therefore still block the event loop on the new async path.
