Full review comments:

- **[P1] Wire external CLI auth through async readers:** `src/agents/cli-credentials.ts:837-843`
  - body: The new async cached readers are not used by `external-cli-sync.ts`, which still feeds runtime auth overlays through the synchronous Codex and Claude cached readers. Gateway auth discovery can therefore still hit sync keychain/file I/O after this patch.
  - confidence: 0.9
- **[P2] Use async readers by default in auth epoch:** `src/agents/cli-auth-epoch.ts:187-195`
  - body: With production deps, `hasAsyncDepOverride` is false, so this branch still calls the synchronous cached reader. The normal runtime path should await the async reader and reserve the sync fallback for sync-only test overrides.
  - confidence: 0.92
- **[P2] Use async file reads in async credential readers:** `src/agents/cli-credentials.ts:617`
  - body: `readClaudeCliCredentialsAsync` still falls back to `loadJsonFile`, and the Codex async reader similarly uses sync JSON/stat helpers. File-backed credentials can still block the event loop when keychain is unavailable or disabled.
  - confidence: 0.85
- **[P2] Make Windows configured-shell execution async:** `src/agents/sessions/resolve-config-value.ts:151-154`
  - body: The async config resolver still calls `executeWithConfiguredShell`, which uses `spawnSync`. A `!command` API key or header on Windows can still block on the new async path.
  - confidence: 0.8
