## Summary

Keep open. The requester-routing bug is real on current main, but this PR is not mergeable as-is because the new requester `agent` call can use strict webchat/internal delivery without `bestEffortDeliver`, which the gateway rejects before dispatching the requester run.

## What This Changes

The PR changes `sessions_send` A2A announce delivery to send announce replies to the requester session before falling back to target-channel delivery when no requester session exists.

## Best Possible Solution

Route A2A announce replies through the requester session using the existing gateway best-effort/session-only delivery contract, add coverage that exercises real webchat/internal requester behavior, and include the required changelog entry.

## Reproduction Assessment

Yes. Source inspection shows current main still posts the final A2A announce via target-channel `send`, and the PR’s new requester path conflicts with existing gateway tests for strict webchat delivery.

## Solution Assessment

No. The direction is right, but the proposed `agent` call is too strict for webchat/internal or missing-target requester sessions; it should use best-effort/session-only downgrade or the established announce-delivery helper pattern.
