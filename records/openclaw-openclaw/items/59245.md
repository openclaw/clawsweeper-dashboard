## Summary

Keep open: current `main` and the latest release still expose Voice Call call-control primitives, realtime audio, DTMF, and consult hooks, but not the requested autonomous outbound task-call workflow with IVR/hold handling, user escalation, allowed-information policy, timeout/reschedule behavior, or callback continuity. The related private-objective PR is useful groundwork, but it explicitly does not replace this umbrella feature.

## What This Changes

Request an autonomous outbound Voice Call task mode where an agent can place calls for reservations/inquiries, navigate IVR/hold states, escalate to the user, manage callbacks, and use natural speech.

## Best Possible Solution

Design a Voice Call task-call layer on top of the existing initiate/continue/speak/DTMF/status/realtime primitives, with a task objective schema, call state machine, IVR/hold strategy, trusted user escalation, allowed-information policy, cost/duration caps, transcript reporting, and callback/reschedule semantics.

## Reproduction Assessment

Not applicable as a bug reproduction, but the feature gap is source-confirmed. Current docs, schemas, lifecycle states, and manager/tool surfaces show call-control primitives rather than autonomous outbound task-call orchestration.

## Solution Assessment

Unclear because no full implementation is proposed. The maintainable direction is to layer task orchestration inside the Voice Call plugin, review https://github.com/openclaw/openclaw/pull/83942 as one smaller primitive, and settle security/product policy before implementation.
