## Summary

Keep this open. Current main has outbound voice-call primitives and DTMF support, but not the requested autonomous task-call workflow; a maintainer already made the same current-main assessment and explicitly kept the issue open. The VoIPBin/media-offloading comment is useful design input, but it does not remove the missing OpenClaw task-call product contract.

## What This Changes

The issue requests an autonomous outbound voice-call agent that can place calls for reservations, inquiries, and flight changes, handle IVR/hold/callback states, escalate to the user, and share only authorized information.

## Best Possible Solution

Define a voice-call plugin task-call layer on top of the existing initiate/continue/speak/DTMF/status/realtime primitives. The maintainable end state should include a task objective schema, call state machine, IVR/hold strategy, trusted user escalation with timeout behavior, allowed-information policy, duration/cost limits, transcript/reporting, callback/reschedule semantics, and provider adapter guidance for Twilio/Telnyx/Plivo or media-offloading services.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. There is a high-confidence gap check: current source, docs, targeted searches, and maintainer discussion all show primitives only and no autonomous outbound task-call workflow.

## Solution Assessment

Unclear because no patch is proposed. The narrowest maintainable direction is to layer orchestration into the voice-call plugin and evaluate media-offloading providers as adapters, rather than adding a new core-wide telephony abstraction first.
