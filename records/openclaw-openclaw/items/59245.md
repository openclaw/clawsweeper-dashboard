## Summary

Keep open: current main and the latest release still expose Voice Call primitives, DTMF, realtime voice, and consult hooks, but not the requested autonomous outbound task-call workflow with IVR/hold handling, user escalation, allowed-information policy, timeout/reschedule behavior, or callback continuity. The related private-objective PR is a useful narrow slice, not a replacement for this umbrella feature.

## What This Changes

Request an outbound Voice Call task mode where an agent can place real-world phone calls on a user's behalf, navigate waits or IVRs, escalate to the user for decisions, and resume or report call outcomes safely.

## Best Possible Solution

Design a Voice Call task-call layer on top of existing initiate/continue/speak/DTMF/status/realtime primitives, with a task objective schema, call state machine, IVR/hold strategy, trusted user escalation, allowed-information policy, cost/duration caps, transcript reporting, and callback/reschedule semantics.

## Reproduction Assessment

Not applicable as a bug reproduction, but the feature gap is source-confirmed. Current docs, schemas, lifecycle states, and manager/tool surfaces show call-control primitives rather than autonomous task-call orchestration.

## Solution Assessment

Unclear because no full implementation is proposed. The maintainable direction is to layer task orchestration inside the Voice Call plugin, review the open private-objective PR as one smaller primitive, and settle the security/product policy before implementation.
