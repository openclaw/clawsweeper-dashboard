## Summary

Keep open: current main and v2026.6.8 still provide Voice Call call-control, realtime voice, DTMF, session-scope, and consult primitives, but not the autonomous outbound task-call workflow with IVR/hold handling, user escalation, authorization policy, timeout/reschedule behavior, or callback continuity.

## What This Changes

The issue requests an autonomous outbound task-call capability for Voice Call so an agent can place calls, navigate waits/IVR, escalate to the user, resume callbacks, and speak naturally while sharing only authorized information.

## Best Possible Solution

Keep this as the umbrella product/security tracker until maintainers approve a Voice Call plugin task layer or split the remaining IVR, hold, callback, escalation, and authorization-policy work into narrower canonical issues; treat https://github.com/openclaw/openclaw/pull/83942 as a partial primitive only.
