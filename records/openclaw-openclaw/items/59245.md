## Summary

Keep open: current main and the latest release still provide outbound call-control primitives, DTMF, realtime voice, and consult hooks, but not the requested autonomous task-call workflow. The linked private-objective PR is a useful narrow slice and explicitly does not cover IVR/hold handling, callback continuity, escalation, recording, or authorization policy.

## What This Changes

The issue requests an autonomous outbound task-call capability where Voice Call can place calls for reservations/inquiries, handle IVR and holds, escalate to the user, manage callback continuity, and use natural speech.

## Best Possible Solution

Design a Voice Call task-call layer on top of existing initiate/continue/speak/DTMF/status/realtime primitives, with a task objective schema, call state machine, IVR/hold strategy, trusted user escalation, allowed-information policy, cost/duration caps, transcript reporting, and callback/reschedule semantics.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a bug reproduction, but the feature gap is source-confirmed: current docs, schemas, lifecycle states, and manager/tool surfaces show call-control primitives rather than autonomous task-call orchestration.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers settle the product and security policy; the maintainable direction is to layer task orchestration inside the Voice Call plugin and review the open private-objective PR as one smaller primitive.
