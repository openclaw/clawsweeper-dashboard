## Summary

Keep open: current main has outbound Voice Call primitives, DTMF, realtime voice, and consult hooks, but it still lacks the requested autonomous task-call workflow with IVR/hold handling, user escalation, allowed-information policy, timeout/reschedule behavior, and callback continuity. The open private-objective PR is a useful narrow slice, not a replacement for the umbrella feature.

## What This Changes

Requests an autonomous outbound voice-call workflow where an agent can place task calls, navigate waits/IVRs, escalate to the user, resume callbacks, and speak naturally on the user's behalf.

## Best Possible Solution

Design a Voice Call task-call layer on top of existing initiate/continue/speak/DTMF/status/realtime primitives, with a task objective schema, call state machine, IVR/hold strategy, trusted user escalation, allowed-information policy, cost/duration caps, transcript reporting, and callback/reschedule semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The gap is source-confirmed: current docs, schemas, lifecycle states, and manager/tool surfaces show call-control primitives rather than autonomous task-call orchestration.

## Solution Assessment

Unclear because no full implementation is proposed. The maintainable direction is to layer task orchestration inside the Voice Call plugin and evaluate media-offloading providers as adapters, while reviewing the open private-objective PR as one smaller primitive.
