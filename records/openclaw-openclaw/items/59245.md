## Summary

Keep open: current main still provides outbound voice-call primitives, DTMF control, realtime audio, and delegated conversation hooks, but not the requested autonomous outbound task-call workflow with IVR/hold handling, user escalation, authorization policy, timeout/reschedule behavior, or callback continuity.

## What This Changes

Request to add an outbound voice-call task mode where an agent can place real-world phone calls on the user's behalf for reservations, inquiries, and changes that may require escalation back to the user.

## Best Possible Solution

Define a voice-call plugin task-call layer on top of the existing initiate/continue/speak/DTMF/status/realtime primitives, with a task objective schema, call state machine, IVR/hold strategy, trusted user escalation, allowed-information policy, cost/duration caps, transcript reporting, and callback/reschedule semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The gap is source-reproducible: current docs, config, types, manager code, and live discussion show primitives only, not autonomous outbound task-call workflow.

## Solution Assessment

Unclear because no patch is proposed. The narrowest maintainable direction is to layer task orchestration inside the voice-call plugin and evaluate media-offloading providers as adapters, rather than adding a new core-wide telephony abstraction first.
