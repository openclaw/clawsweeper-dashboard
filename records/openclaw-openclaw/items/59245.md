## Summary

Keep open: current main still has outbound voice-call primitives and DTMF controls, but not the requested autonomous task-call workflow; the maintainer discussion and canonical search did not show a merged replacement that owns the remaining work.

## What This Changes

Request an autonomous outbound voice-call task mode that can place calls, navigate IVRs and holds, escalate to the user, share only authorized information, and preserve callback continuity.

## Best Possible Solution

Define a voice-call plugin task-call layer on top of the existing initiate/continue/speak/DTMF/status/realtime primitives, with a task objective schema, call state machine, IVR/hold strategy, trusted user escalation, allowed-information policy, cost/duration caps, transcript reporting, and callback/reschedule semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The gap is source-reproducible: current docs, config, types, manager code, and discussion show primitives only, not autonomous outbound task-call workflow.

## Solution Assessment

Unclear because no patch is proposed. The narrowest maintainable direction is to layer task orchestration into the voice-call plugin and evaluate media-offloading providers as adapters, rather than adding a new core-wide telephony abstraction first.
