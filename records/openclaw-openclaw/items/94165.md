## Summary

Keep open: this is a small, coherent Control UI bug-fix PR with no blocking findings. Current main still drops these legacy tagged reasoning blocks from the visible answer without extracting them for the thinking panel, and the PR aligns the fallback extractor with the known reasoning-tag family already used by shared visible-text stripping.

## What This Changes

The PR broadens Control UI `extractThinking()` fallback parsing and regression tests to extract MiniMax/Anthropic namespaced and related reasoning tags from legacy text blocks.

## Best Possible Solution

Land this narrow parser alignment after normal maintainer and check gates; broader consolidation of duplicated reasoning-tag patterns can stay a separate follow-up if maintainers want to reduce drift.

## Reproduction Assessment

Yes. Source inspection shows current main strips these tags from assistant-visible text but `extractThinking()` only extracts bare legacy tags, and the PR body includes a negative-control function-call proof for the missed cases.

## Solution Assessment

Yes. The Control UI fallback extractor is the owner of legacy tagged text-block thinking extraction, and extending that local matcher to the existing known tag names is the narrowest maintainable fix for this surface.
