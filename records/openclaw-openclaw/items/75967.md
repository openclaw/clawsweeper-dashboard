## Summary

Keep this PR open: the latest head still removes dependency-backed ClawHub CLI dry-run validation from the release preview, while current main and the pinned ClawHub source show that dry-run is supported and builds the publish plan before upload.

## What This Changes

The PR changes `scripts/plugin-clawhub-publish.sh --dry-run` to print a local command preview without invoking `clawhub`, and adds a regression test for that behavior.

## Best Possible Solution

Preserve the ClawHub CLI dry-run preflight in the preview job; if a real unsupported CLI regression is reproduced, update the pinned ClawHub commit or add a targeted compatibility fix without dropping publish-plan validation.

## Reproduction Assessment

Yes for the review finding: current main invokes `clawhub package publish --dry-run`, while the PR head exits before invoking the CLI. Source inspection of the pinned ClawHub commit shows `--dry-run` is registered and returns only after building the publish plan.

## Solution Assessment

No. Replacing the CLI dry-run with an echo is not the narrowest maintainable fix because it removes the dependency-backed validation the preview job is meant to provide.
