## Summary

Keep open: this maintainer-labeled, member-authored PR needs explicit maintainer handling, and the patch bypasses a working pinned ClawHub CLI dry-run preflight in the plugin release path.

## What This Changes

This PR changes `scripts/plugin-clawhub-publish.sh --dry-run` to print the resolved ClawHub publish command without invoking the ClawHub CLI, with a regression test for that behavior.

## Best Possible Solution

Keep the preview step exercising the pinned ClawHub CLI dry-run, or update the ClawHub pin/CLI contract if a concrete unsupported-version regression is found.

## Reproduction Assessment

Yes for the review regression: current main and the PR diff show the dry-run call being replaced by an echo, and the pinned ClawHub source implements `package publish --dry-run`. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No. The proposed local-only echo is not the best fix because it avoids dependency-backed package validation instead of preserving the release preflight; the safer path is to keep invoking the CLI dry-run or update the pinned CLI if needed.
