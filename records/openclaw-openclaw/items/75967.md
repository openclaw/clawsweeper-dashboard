## Summary

Keep open: this PR is maintainer-protected, and source/dependency checks show the proposed local-only dry-run would remove the release package-plan preflight even though current main pins a ClawHub CLI commit that supports `package publish --dry-run`.

## What This Changes

The PR changes ClawHub plugin publish dry-run mode to print a local preview instead of invoking the ClawHub CLI, with a regression test for that behavior.

## Best Possible Solution

Preserve a dependency-backed preview by keeping the ClawHub CLI dry-run preflight, or update the pinned ClawHub CLI only if a concrete unsupported-version regression is reproduced.

## Reproduction Assessment

Yes. The review regression is reproducible by comparing the PR diff at `scripts/plugin-clawhub-publish.sh:84` with current main and the pinned ClawHub source, which registers and returns safely on `package publish --dry-run`.

## Solution Assessment

No. The local-only echo avoids the dependency-backed validation that the preview job is meant to provide; keeping the CLI dry-run is the narrower maintainable solution.
