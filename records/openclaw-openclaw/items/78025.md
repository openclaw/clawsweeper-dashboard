## What This Changes

The issue reports intermittent message-tool media sends to the external openclaw-weixin channel failing with `Unknown channel` or `Channel is required` despite active inbound WeChat traffic.

## Best Possible Solution

Keep the shipped generic pinned-registry and context-fallback implementation; new reports should target current OpenClaw plus the installed WeChat plugin version if the behavior reappears.

## Reproduction Assessment

No high-confidence current-main failing reproduction is needed for closure: the report gives a 2026.4.5 path, and source/history show that exact old third-party channel visibility failure is fixed. Current main retains registered external plugin ids as deliverable channels and falls back to the active tool context before raising the reported errors.

## Solution Assessment

Yes. The generic pinned channel-registry lookup plus message-action context fallback is the narrow maintainable fix because it preserves external plugin support without adding WeChat-specific core behavior.
