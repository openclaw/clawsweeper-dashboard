## Summary

Keep open. Current main still has the legacy non-tool exec-completion prompt that asks for `HEARTBEAT_OK only` without final-tag or `NO_REPLY` guidance, while the strict final-tag and empty-response paths can still turn an unwrapped silent reply into a zero-payload incomplete turn.

## What This Changes

Align the internal async exec-completion prompt with OpenClaw's silent-reply/final-tag contract so `deliverToUser=false` does not produce empty-response retries or incomplete-turn errors.

## Best Possible Solution

Update the legacy non-tool exec/no-output prompt to an intentional silent terminal contract, preferably explicit `<final>NO_REPLY</final>` wording, with focused regression coverage and no change to the structured `heartbeat_respond` path.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes by source inspection: use `buildExecEventPrompt` with `deliverToUser=false` and `useHeartbeatResponseTool=false`, then a strict-final run where the model emits a bare silent token so the runner sees zero payloads and enters the empty-response retry path.

## Solution Assessment

Is this the best way to solve the issue? Yes. The narrowest maintainable fix is to align the legacy prompt with the existing silent-reply/final-tag contract; #73785's output filtering and #76338's response-tool prompt changes are adjacent but do not remove this legacy branch.
