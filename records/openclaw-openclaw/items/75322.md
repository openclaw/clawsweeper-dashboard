## Summary

Keep open. Current main still has the legacy/non-tool internal exec-completion prompt asking for `HEARTBEAT_OK only` without `<final>` or `NO_REPLY` guidance, so the reported strict-final zero-payload path remains source-reproducible on surfaces that do not allow empty assistant turns as silent.

## What This Changes

The issue asks to align disabled-delivery async exec completion prompts with the runtime silent-reply contract so they do not surface incomplete-turn errors.

## Best Possible Solution

Update the legacy exec-completion prompt contract to produce an intentional silent terminal result, preferably explicit `<final>NO_REPLY</final>` guidance with focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source path is `deliverToUser=false`, non-response-tool exec completion, strict final-tag enforcement, and a bare `NO_REPLY`, which current main can strip to empty visible text before assistant text finalization.

## Solution Assessment

Yes. The narrowest maintainable fix is to align the internal/no-output exec-completion prompt with the existing silent-reply contract or add a tightly scoped raw silent-token guard; PR #73785 is adjacent output filtering, not enough by itself.
