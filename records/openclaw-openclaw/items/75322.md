## Summary

Keep open. Current `main` still has the prompt-layer conflict reported here, and the adjacent #73785 work filters emitted sentinel payloads rather than fixing the strict-final zero-payload path.

## What This Changes

Request fixes the no-delivery async exec-completion prompt so suppressed completions produce an intentional silent terminal reply instead of triggering empty-response retries.

## Best Possible Solution

Make the no-delivery exec-completion path produce or accept an intentional silent terminal result without invoking visible-answer retries, while preserving normal relay behavior for output-bearing and failed exec completions.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists by resolving an exec completion with `canRelayToUser` false, following the current `HEARTBEAT_OK` prompt into strict `<final>` extraction, and observing that an unwrapped `NO_REPLY` or empty final becomes `payloadCount=0` and enters the retry/error path.

## Solution Assessment

Unclear for the reporter's exact diff, but a narrow prompt plus regression-test fix is the best direction. The safest implementation should explicitly align the internal exec-completion branch with `<final>NO_REPLY</final>` or mark only that internal path as intentional silence, without broadening silent behavior elsewhere.

## Repair Work Prompt

Fix the heartbeat exec-completion no-delivery bug reported at https://github.com/openclaw/openclaw/issues/75322. In `src/infra/heartbeat-events-filter.ts`, remove the ambiguity where no-output or `deliverToUser: false` exec-completion prompts ask for `HEARTBEAT_OK only` without a final wrapper. Prefer explicit `Reply with exactly <final>NO_REPLY</final> and nothing else` for the internal no-delivery path, or otherwise mark only this internal exec-completion path as intentionally silent before empty-response retries. Preserve user-relay behavior for output-bearing exec completions and failed metadata-only exec completions that should report status when delivery is enabled. Update `src/infra/heartbeat-events-filter.test.ts` for the prompt contract and add focused regression coverage in `src/agents/pi-embedded-runner/run.incomplete-turn.test.ts` and/or heartbeat runner tests for strict-final zero-payload no-delivery behavior. Do not change generic direct-chat silent-reply policy, provider final-tag extraction, or broad outbound sentinel suppression. Add a user-facing `CHANGELOG.md` fix entry if the behavior change affects released heartbeat/channel noise; credit `Thanks @mbrya859-byte`.
