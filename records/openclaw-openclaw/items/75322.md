## Summary

Keep open. Current main still has the no-delivery exec-completion prompt conflict, and the related #73785 work is adjacent runtime sentinel filtering rather than a fix for this zero-payload strict-final path.

## What This Changes

Fix async exec-completion prompts so disabled user delivery cannot trigger bare silent replies that strict final-tag extraction treats as empty turns.

## Best Possible Solution

Align the internal exec-completion branch with the silent-reply contract, or otherwise treat that exact no-delivery completion as intentional silence, while preserving normal relay behavior for output-bearing exec completions.

Yes. A high-confidence static reproduction exists: disabled delivery builds the ambiguous `HEARTBEAT_OK` prompt, strict final-tag extraction discards a bare `NO_REPLY`, and the runner retries then surfaces the zero-payload incomplete-turn error.

## Solution Assessment

Yes in direction. The narrowest maintainable fix is prompt/runtime handling for this specific no-delivery exec-completion path with regression tests, not a broad delivery-layer suppression change.
