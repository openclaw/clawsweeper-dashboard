## Summary

Keep open. Current main still has the reported prompt/runtime mismatch: internal-only exec-completion prompts ask for `HEARTBEAT_OK` without final-tag or `NO_REPLY` guidance, while strict final-tag handling can erase a bare silent token before incomplete-turn suppression sees it.

## What This Changes

The issue requests fixing internal async exec-completion handling so `deliverToUser=false` runs do not surface incomplete-turn errors when the model produces an intentional silent reply.

## Best Possible Solution

Update the internal/no-output exec-completion contract to produce an intentional silent terminal result, preferably by instructing exactly `<final>NO_REPLY</final>` and covering the strict-final zero-payload path with focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source reproduction is an exec completion with `canRelayToUser=false`, strict final-tag enforcement, and a bare `NO_REPLY`: current code can strip it to empty visible text before the incomplete-turn guard can classify it as silent.

## Solution Assessment

Yes for a narrow prompt-contract fix. The safest repair is to align the exec-event prompt with the runtime silent sentinel and final-tag requirement, rather than broadening final-tag extraction globally.
