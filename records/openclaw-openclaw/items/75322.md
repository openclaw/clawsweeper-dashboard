## Repair Work Prompt

Fix the async exec-completion no-delivery bug described in https://github.com/openclaw/openclaw/issues/75322. Current `buildExecEventPrompt(..., { deliverToUser: false })` asks for `HEARTBEAT_OK only` without `<final>` wrapping; on strict final-tag providers a model can emit bare `NO_REPLY`, leaving zero payloads and causing the embedded runner to retry and surface an incomplete-turn error. Keep the fix narrow: update the no-delivery exec-completion behavior so it becomes an intentional silent/no-op terminal result accepted by the runner, while preserving normal relay behavior for output-bearing exec completions. Be careful with the no-output branch when a real delivery target exists, because blindly switching it to `NO_REPLY` could leak or rewrite the silent token through outbound delivery. Add/update tests in `src/infra/heartbeat-events-filter.test.ts` and the closest embedded-runner/heartbeat-runner tests to prove no retry/error is emitted for the no-delivery path and relayable exec output still delivers. Add a single-line `CHANGELOG.md` fix entry crediting `@mbrya859-byte` and `@martingarramon`. Do not change generic direct/group silent-reply policy or unrelated heartbeat token semantics.
