## Summary

Keep open. Current main still has the reported prompt conflict: internal-only exec completion prompts request bare `HEARTBEAT_OK`, while strict final extraction can turn an unwrapped silent reply into zero visible payloads and the empty-response retry path can surface an incomplete-turn error.

## What This Changes

The issue requests that internal-only async exec completion prompts stop racing silent-reply handling by using an unambiguous wrapped silent response or equivalent runtime path.

## Best Possible Solution

Make internal-only and metadata-only successful exec completions produce an intentional silent terminal result, while preserving relayable exec output and failed no-output command reporting.

## Reproduction Assessment

Yes. A high-confidence path is an exec completion with `canRelayToUser=false`, strict final-tag enforcement, and a bare silent reply; current source shows that can become zero visible payloads and enter the empty-response retry/incomplete-turn path.

## Solution Assessment

Yes for a narrow prompt-layer fix with regression coverage. Prefer explicit `<final>NO_REPLY</final>` guidance or an equivalent scoped heartbeat outcome over broad relaxation of strict final-tag extraction.
