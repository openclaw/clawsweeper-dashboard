- **[P2] Reset saved paths when retargeting the gateway:** `src/node-host/runner.ts:251`
  - body: `runNodeHost` now falls back to the saved `contextPath` whenever the caller omits `--context-path`. That differs from the existing host/port retarget behavior, where explicit endpoint changes avoid inheriting stale TLS state. After a user connects once through `/openclaw-gw`, a later `openclaw node run --host 127.0.0.1 --port 18789` still connects to `ws://127.0.0.1:18789/openclaw-gw`, and `node install` has the same fallback through `resolveNodeDefaults`. Preserve the saved path only when the gateway was not explicitly retargeted, or provide an explicit clear/override path.
  - confidence: 0.86
