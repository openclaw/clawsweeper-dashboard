## What This Changes

Adds a `--context-path` option to `openclaw node run` and `openclaw node install`, persists `contextPath` in `node.json`, forwards it into daemon service arguments, documents it, and adds node-host URL tests.

## Best Possible Solution

Settle the node endpoint API shape first, then either land this path-only implementation or adjust it to the approved full-URL/existing-Gateway-contract shape with focused run, install, and persisted-config coverage.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.11` expose node Gateway targeting as host/port/TLS only and build `ws(s)://host:port`, while the PR wires a path through node run/install.

## Solution Assessment

Unclear as a product API decision. The implementation is a clean plausible fix, but maintainers should confirm whether a path-only flag is the best permanent shape versus a full Gateway URL or existing `gateway.remote.url`-style contract.
