## Summary

Keep open: this PR targets a real Google Chat add-on parser bug that still exists on current main, but it is not merge-ready because the diff omits a Google-documented add-on payload variant and the external contributor has not attached real behavior proof.

## What This Changes

The PR broadens Google Chat Workspace Add-on payload parsing in `extensions/googlechat/src/monitor-webhook.ts` and adds monitor webhook tests for derived add-on event types.

## Best Possible Solution

Accept add-on envelopes through the full Google-documented Chat payload union, return successful acknowledgements for non-message events, keep message dispatch unchanged, and consolidate the result with the open Google Chat parser cluster.

## Reproduction Assessment

Yes from source inspection: a current-main Workspace Add-on request with `commonEventObject.hostApp: "CHAT"` and `chat.addedToSpacePayload` is not transformed because only `messagePayload` is handled, then it lacks a root `type` and returns HTTP 400. I did not run tests because this was a read-only review.

## Solution Assessment

No: the direction is right, but the patch should include the documented `widgetUpdatedPayload` path and provide real behavior proof before merge. Consolidating with the open canonical Google Chat parser issue is the cleaner maintainer path.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Handle widgetUpdatedPayload too:** `extensions/googlechat/src/monitor-webhook.ts:100-103`
