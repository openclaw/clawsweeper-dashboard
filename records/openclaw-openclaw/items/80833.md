Keep open. The current-main bug is source-reproducible and the PR is pointed at the right Google Chat parser, but the branch still misses a documented Workspace Add-on Chat payload variant and has no inspectable real behavior proof after both the bot review and follow-up triage asked for it.

## What This Changes

The PR broadens Google Chat Workspace Add-on payload parsing in `extensions/googlechat/src/monitor-webhook.ts` and adds monitor webhook tests for derived add-on event types.

## Best Possible Solution

Update the Google Chat parser to cover the full documented Workspace Add-on Chat payload union, add focused regression coverage for `widgetUpdatedPayload`, and merge only after the contributor adds redacted real setup proof.

## Reproduction Assessment

Yes, from source inspection: a current-main Workspace Add-on request with `commonEventObject.hostApp: "CHAT"` and a lifecycle-only or widget-updated `chat.*Payload` is not normalized unless it is `messagePayload`, then fails the event-shape guard with HTTP 400. I did not run local tests because this review is read-only.

No: the parser direction is the narrow owner-module fix, but this branch should also handle `chat.widgetUpdatedPayload` and provide inspectable real behavior proof before merge.
