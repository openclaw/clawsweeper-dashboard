## Summary

Keep open. Current main and the latest v2026.5.3 release still grant `internalDiagnostics` only to bundled diagnostics services while official metadata still advertises the Prometheus exporter as npm-published, and ClawHub still reports no installable Prometheus version.

## What This Changes

The issue asks OpenClaw to align the `diagnostics-prometheus` install path with its bundled-only `internalDiagnostics` capability so users do not get an enabled plugin that serves empty metrics and logs errors.

## Best Possible Solution

Pick one supported contract and align source, packaging, install metadata, and docs: publish a trusted bundled/ClawHub Prometheus exporter that receives the capability, or reject/document npm installs as unsupported until a safe trust mechanism exists.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: npm-installed plugins are discovered as `global`, `internalDiagnostics` is granted only to bundled diagnostics services, and the Prometheus service returns before subscribing when the capability is absent.

## Solution Assessment

Unclear. The report names valid directions, but the maintainable fix depends on a maintainer decision about trusted diagnostics-exporter distribution versus install-time rejection or documentation.
