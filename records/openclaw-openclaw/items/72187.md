## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Expose the harness through a focused test subpath:** `src/plugin-sdk/testing.ts:35-39`
  - body: Current docs and package guardrails mark `openclaw/plugin-sdk/testing` as deprecated compatibility and reject new real imports. Adding this new helper only from that barrel puts the documented API on the wrong SDK surface; export it from `plugin-sdk/plugin-test-runtime` or a new focused test subpath and update docs, tests, and baselines.
  - confidence: 0.9
- **[P2] Mirror the production gateway_start context port:** `src/plugins/service-lifecycle-test-harness.ts:184`
  - body: Production `gateway_start` invokes hooks with the listening port in both `event.port` and `ctx.port`, but the harness default context only includes `config` and `workspaceDir`. Hooks that read `ctx.port` can pass under the real Gateway and fail under this helper unless every test manually duplicates the value.
  - confidence: 0.86
