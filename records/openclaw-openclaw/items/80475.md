## Summary

Keep open: the branch targets the right ACPx test failure linked from https://github.com/openclaw/openclaw/issues/80431, but the changed helper computes the wrong dist path and the copied proof does not show a run with dist entries present.

## What This Changes

The PR renames the ACPx config test helper and tries to prefer built dist MCP server args when those files exist, falling back to source-mode tsx args.

## Best Possible Solution

Mirror the resolver's absolute dist/mcp path construction in the test helper while keeping the bridge assertions scoped to the two built-in MCP server configs.

## Reproduction Assessment

Yes. Source inspection shows the helper is called with relative src/mcp strings, so replace("/src/", "/dist/") never matches and a checkout with dist/mcp entries would still fall back to source-mode expected args.

## Solution Assessment

No. Accepting dist-first resolution is the right narrow direction, but this implementation should compute the same absolute dist path as the resolver before checking fs.existsSync.
