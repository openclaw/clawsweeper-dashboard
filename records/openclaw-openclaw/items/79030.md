## Summary

Keep open: current main still has a source-provable path where model-provider fetches without an explicit dispatcher policy opt into trusted env-proxy mode, while the local NO_PROXY matcher only supports Undici-style exact host, suffix, optional port, and global wildcard matching.

## What This Changes

The issue asks OpenClaw to prevent private model-provider requests with `allowPrivateNetwork` from using env proxy routing when NO_PROXY only contains CIDR or IP-wildcard bypass entries.

## Best Possible Solution

Add target-aware CIDR and narrowly defined IP-wildcard handling to the OpenClaw env-proxy decision for provider fetches, with regression coverage for CGNAT/Tailscale private endpoints and no change to managed proxy bypass policy.

## Reproduction Assessment

Yes for source-level reproduction: current main selects trusted env-proxy mode for provider requests when `shouldUseEnvHttpProxyForUrl` returns true, and the matcher cannot match `100.64.*` or `100.64.0.0/10` to `100.64.0.3`. I did not run a live proxy capture in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable fix is to extend OpenClaw's target-aware NO_PROXY decision for CIDR and carefully scoped IP wildcards before selecting trusted env-proxy mode. Defaulting all `allowPrivateNetwork` providers to direct transport would be a broader security/product change and should not be the first repair path.

Fix https://github.com/openclaw/openclaw/issues/79030 as a narrow model-provider env-proxy routing bug. Current main lets `buildGuardedModelFetch` choose trusted env-proxy mode when `shouldUseEnvHttpProxyForUrl(url)` is true, but `matchesNoProxy` treats CIDR entries such as `100.64.0.0/10` and IP wildcard entries such as `100.64.*` as literal hostnames, so private/Tailscale provider requests with `request.allowPrivateNetwork: true` can still route through `HTTP_PROXY`. Implement target-aware NO_PROXY matching for CIDR and tightly scoped IPv4 octet wildcards before provider fetches opt into trusted env-proxy mode. Prefer reusing existing IP/CIDR helpers from `src/shared/net/ip.ts`; do not change managed proxy mode behavior that clears NO_PROXY, do not default every `allowPrivateNetwork` provider to direct transport unless a maintainer explicitly asks, and do not add a config option. Add regression tests for `matchesNoProxy` and `shouldUseEnvHttpProxyForUrl` covering `100.64.0.3` against `100.64.0.0/10`, `100.64.*`, exact host, port mismatch, nonmatching CIDR, and invalid CIDR/wildcard inputs. Add or adjust provider-transport coverage if needed to prove private provider requests do not enter trusted env-proxy mode when NO_PROXY matches by CIDR. Add a single-line user-facing changelog fix entry; do not touch unrelated proxy, managed-proxy, or media-transcription surfaces. Related context: https://github.com/openclaw/openclaw/issues/52058 and https://github.com/openclaw/openclaw/pull/38687.
