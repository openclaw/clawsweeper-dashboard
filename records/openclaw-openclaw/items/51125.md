## Summary

Keep this PR open. Current main still lacks Cyrillic/Greek confusable-letter folding, so the contribution targets a real sanitizer gap, but the PR head leaves a concrete lowercase Cyrillic bypass and is dirty against the current index-map sanitizer path.

## What This Changes

The PR adds confusable-letter folding for Cyrillic, Greek, and a small Latin-extended set in external-content marker sanitization, plus Cyrillic and Greek spoof marker tests.

## Best Possible Solution

Rebase the confusable-letter normalization onto the current foldMarkerTextWithIndexMap pipeline, keep the regex derived from the folding maps, add U+0442 coverage, and expand tests to cover uppercase and lowercase spoof markers while preserving surrounding text.

## Reproduction Assessment

Yes. A focused source-level reproduction is wrapWebContent('Before <<<ex\u0442ernal_un\u0442rusted_con\u0442ent>>> after', 'web_search'); the PR head does not fold U+0442, and current main does not fold Cyrillic/Greek letters at all.

## Solution Assessment

No. The direction is narrow and relevant, but the proposed patch is incomplete and stale against current main; the safer fix is a rebase that preserves the index-map sanitizer and closes the lowercase Cyrillic gap.
