## Summary

Keep open. Current main still lacks the Discord announce-target session lookup opt-in, and this PR is a narrow, correct fix for the linked multi-account Discord A2A announce routing bug. The remaining blocker is mergeability: GitHub reports the branch as dirty against current main, so it needs maintainer rebase/conflict handling or an equivalent replacement before landing.

## What This Changes

The PR adds Discord `preferSessionLookupForAnnounceTarget` metadata in runtime and package descriptors, updates Discord/A2A regression coverage for `deliveryContext.accountId` and `lastAccountId`, and records the user-facing fix in `CHANGELOG.md`.

## Best Possible Solution

Land this PR after resolving the dirty merge state, or land an equivalent narrow Discord-owned patch that adds the metadata opt-in in both runtime and manifest/package metadata with the same account-routing regression coverage and changelog entry.

## Reproduction Assessment

Yes. Current main gives a high-confidence source-level reproduction: a Discord session key resolves to a fallback target without accountId, Discord metadata does not request session lookup, and the A2A announce send forwards that missing accountId to gateway `send`.

## Solution Assessment

Yes. The proposed direction is the narrowest maintainable fix: opt Discord into the existing session-backed announce lookup seam already used by Slack and WhatsApp, without changing core defaults for unrelated channel plugins.
