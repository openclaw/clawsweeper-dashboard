## Summary

Keep open. Current main still lacks the core remediation this PR is trying to port, and the PR head remains incorrect because health can call an undefined helper and the protected Telegram delivery guards miss supported account-scoped direct-session keys.

## What This Changes

The PR ports broad installed-distribution stability changes into source across session routing, session-store cleanup, write locks, Telegram polling and delivery protection, overflow compaction recovery, health/status output, SDK exports, and regression tests.

## Best Possible Solution

Keep the PR open as a rework candidate: rebase it onto current main, split the broad remediation where practical, define or import the health issue resolver, make Telegram protected-DM guards cover both accountless and account-scoped direct keys, review the SDK additions deliberately, and preserve focused regression coverage for routing, degraded health, locks, Telegram polling liveness, protected delivery, and overflow compaction provenance.

## Reproduction Assessment

Yes for the PR blockers: the raw PR diff shows the undefined `resolveOperationalHealthIssues` call and accountless-only Telegram direct-session regexes. Current main source and docs also provide a high-confidence supported key shape for `per-account-channel-peer` sessions.

## Solution Assessment

No. The remediation direction may be valuable, but this PR is not the best merge shape until the health crash, Telegram account-scoped protection gaps, broad SDK contract changes, and cross-cutting risk are addressed.
