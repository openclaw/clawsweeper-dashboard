## Summary

Keep open. Current main still treats `sandbox.docker.env` as a plain string map: schema validation, public config types, generated schema, Docker config merge, secret collectors, and the target registry do not accept or resolve SecretRef values for this surface. The related implementation PR #38310 was closed unmerged, while #15138 only shipped plain string Docker env forwarding.

## What This Changes

The issue requests allowing SecretRef objects as `sandbox.docker.env` values for defaults and per-agent Docker sandbox configuration while preserving resolved Docker env values as strings.

## Best Possible Solution

Implement an end-to-end SecretInput addition for Docker sandbox env: accept `SecretInput` in config and generated schemas, keep unresolved settings typed separately from resolved Docker runtime config, collect defaults and per-agent env entries through the secrets runtime and target registry, resolve before Docker create/backend handoff, document the supported surface, and cover schema, collector, registry, merge, and create-args behavior.
