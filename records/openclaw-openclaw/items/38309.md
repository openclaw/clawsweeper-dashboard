Keep open: current main and the latest release still treat `sandbox.docker.env` values as strings and do not collect Docker env SecretRefs, so the requested security/config feature remains missing and needs maintainer security/product judgment.

## What This Changes

Request SecretRef object support for `agents.*.sandbox.docker.env` while keeping the resolved Docker sandbox env as plain string values at container launch.

## Best Possible Solution

Make an explicit maintainer security decision: either support `SecretInput` for Docker sandbox env with schema, runtime, registry, docs, and metadata-exposure coverage, or document Docker env as string-only and recommend a mounted-file/custom-image secret path.
