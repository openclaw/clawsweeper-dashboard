## Summary

Current main already implements the LM Studio JIT opt-out with the documented `models.providers.lmstudio.params.preload: false` setting, regression coverage, docs, and changelog; this PR’s `skipModelLoad` field is now redundant and would create a second config surface.

## What This Changes

The PR adds a top-level `models.providers.lmstudio.skipModelLoad` flag to bypass LM Studio manual load calls in inference and embedding warmup, with schema/help metadata and an inference regression test.

## Best Possible Solution

Keep the current main `params.preload: false` contract; if embeddings later need the same opt-out, extend that existing provider params flag rather than adding a second top-level setting.

## Reproduction Assessment

Yes. The linked bug gives concrete LM Studio JIT/TTL steps, and current main has a focused stream test proving the supported opt-out avoids `ensureLmstudioModelLoaded`.

## Solution Assessment

Yes for current main, no for the PR as proposed. The main fix is narrower because it uses the existing provider `params` contract and updates docs/changelog, while the PR adds a redundant top-level config key.
