## Summary

Keep open. Current main still applies the Fireworks Kimi wrapper to K2.6 model IDs and removes reasoning payload fields, while the open closing PR remains unmerged; the gateway-status side has improved degraded-status classification but does not resolve the agent incomplete-terminal-response path.

## What This Changes

The report asks OpenClaw to stop Kimi K2.6/kimi-k2p6 agent turns from ending with incomplete terminal responses and to make `openclaw gateway status` complete its local WebSocket probe while the gateway is running.

## Best Possible Solution

Keep this issue open until maintainers review and land #74289 or a replacement provider-level fix. After that merge, re-check whether the self-hosted Kimi-K2.6 and gateway-status timeout evidence still reproduces and split any remaining provider-neutral behavior into a narrower follow-up.
