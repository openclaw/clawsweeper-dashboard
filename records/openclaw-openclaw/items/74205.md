## Summary

Keep open. Current main still routes Fireworks Kimi K2.6 through the wrapper that disables thinking and deletes reasoning payload fields, and the linked closing PR #74289 is still open and unmerged. The gateway-status side now treats post-connect read timeouts as degraded reachability, but that does not resolve the incomplete terminal response path or the self-hosted Kimi-K2.6 evidence added in the discussion.

## What This Changes

The issue reports Kimi K2.6 turns ending with incomplete terminal responses and `openclaw gateway status` timing out against a running systemd gateway.

## Best Possible Solution

Keep the issue open until maintainers review and land #74289 or replace it with one canonical provider-level fix; after that, retest the custom/self-hosted Kimi-K2.6 path and split any remaining gateway-status or model-tag behavior into a narrower follow-up.
