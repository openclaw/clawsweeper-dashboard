## Summary

Keep open; the PR is useful directionally but is not merge-ready because it regresses existing Discord DM voice-note preflight, adds a new config key without strict schema support, and hardcodes live Discord channel IDs into bundled plugin behavior.

## What This Changes

The PR adds Discord channel-level `sttEnabled` plumbing and changes audio preflight so selected guild voice-message channels can be transcribed without mention gating, with transcripts carried into message content/context.

## Best Possible Solution

Keep the feature config-driven: add a schema-backed Discord channel STT opt-in, preserve existing DM and mention-gated audio paths, remove hardcoded channel IDs, and cover the new path with focused tests/docs/changelog.

## Reproduction Assessment

Yes, source-reproducible. Current main has an explicit DM audio preflight path and test, while the PR changes the helper to exclude DMs; the new config key is also visibly absent from the strict zod schema.

## Solution Assessment

No. The maintainable solution is a schema-backed `sttEnabled` channel option that preserves existing paths and removes the hardcoded channel-ID bypass.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve DM voice-note preflight:** `extensions/discord/src/monitor/preflight-audio.ts:53`
  - body: The new condition starts with `!params.isDirectMessage`, which prevents audio-only DMs from calling `transcribeFirstAudio`. Current main explicitly allows DMs here and has a preflight test asserting DM voice notes are transcribed, so this regresses existing Discord DM behavior.
  - confidence: 0.94
- **[P2] Wire sttEnabled into the config schema:** `src/config/types.discord.ts:72`
  - body: Adding the TypeScript config type is not enough because `DiscordGuildChannelSchema` is strict and does not include `sttEnabled`. A user config using the new key from the PR body will fail validation before the allow-list mapping can use it.
  - confidence: 0.92
- **[P2] Remove the hardcoded STT channel IDs:** `extensions/discord/src/monitor/preflight-audio.ts:24-27`
  - body: Bundled Discord behavior should not special-case concrete channel IDs. This bypasses the config contract and can send voice attachments from those channels to the configured STT provider without an explicit `sttEnabled` opt-in, so keep the feature driven by config only.
  - confidence: 0.88
