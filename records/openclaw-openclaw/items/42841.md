Keep this PR open because current main still lacks the requested node WebSocket upgrade header path, but the PR head is not mergeable as-is. The core direction is valid, and prior bot/user follow-up resolved several early review points, but the latest patch still has credential-storage and install-time env precedence problems plus a dirty branch state.

## What This Changes

The PR adds repeatable `--header` support for `openclaw node run` and `openclaw node install`, threads headers through node config/service environment into `GatewayClient`, redacts service env in node status, and updates node docs, changelog, and targeted tests.

## Best Possible Solution

Land this PR or an equivalent core change after making the header source contract explicit: CLI headers should have predictable precedence, env-only secret headers should remain runtime-only unless the user explicitly opts into persistence, node status/logs/process args must not expose credentials, docs must match the storage model, and targeted tests should cover install-time rotation and no-leak behavior.
