## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep ambient CF_* from overriding install --header:** `src/commands/node-daemon-install-helpers.ts:67-73`
  - body: When `params.headers` comes from explicit `openclaw node install --header`, this still persists ambient `CF_ACCESS_CLIENT_ID`/`CF_ACCESS_CLIENT_SECRET` into the service environment. The daemon later applies `CF_*` after `OPENCLAW_NODE_HEADERS`, so stale exported credentials override the values the user just supplied; avoid copying `CF_*` when explicit headers are set, or make the daemon precedence preserve explicit headers.
  - confidence: 0.9
