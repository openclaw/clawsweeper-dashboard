- **[P2] Avoid overriding installed headers with ambient CF env:** `src/commands/node-daemon-install-helpers.ts:67-73`
  - body: When `params.headers` comes from explicit `node install --header` flags, this still copies `CF_ACCESS_CLIENT_ID` and `CF_ACCESS_CLIENT_SECRET` from the install shell into the service environment.   - confidence: 0.93

  - body: `runNodeHost` saves the fully resolved header set into `config.gateway.headers`, including values sourced only from `OPENCLAW_NODE_HEADERS` or `CF_ACCESS_CLIENT_SECRET`. That contradicts the docs' promise that env vars keep secrets out of config and creates unexpected credential-at-rest behavior for operators choosing env-based storage.
  - confidence: 0.9
