## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep env headers out of node config:** `src/node-host/runner.ts:209-213`
  - body: The docs say service env vars can keep header secrets out of config, but this saves the fully resolved env-derived headers into `node.json` on startup. A daemon using `OPENCLAW_NODE_HEADERS` or `CF_ACCESS_CLIENT_SECRET` to avoid secret-at-rest in config will write those values back to config anyway, so either do not persist env-derived secrets or make the storage policy explicit and tested.
  - confidence: 0.9
- **[P2] Do not let ambient CF env override install headers:** `src/commands/node-daemon-install-helpers.ts:63-70`
  - body: `node install --header ...` serializes the explicit headers into `OPENCLAW_NODE_HEADERS`, but this also copies any ambient `CF_ACCESS_CLIENT_ID`/`CF_ACCESS_CLIENT_SECRET` into the installed service. At runtime those `CF_*` values are applied after `OPENCLAW_NODE_HEADERS`, so stale shell credentials can replace the values the user explicitly installed.
  - confidence: 0.88
