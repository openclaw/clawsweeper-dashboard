- **[P1] Forward headers in the gateway-client package:** `src/gateway/client.ts:88-90`
  - body: This hunk adds `headers` to the old `src/gateway/client.ts` implementation, but current main now creates the WebSocket in `packages/gateway-client/src/client.ts`.   - confidence: 0.86
- **[P1] Keep env-only headers out of node config:** `src/node-host/runner.ts:206-211`
  - body: `resolvedHeaders` includes `OPENCLAW_NODE_HEADERS` and `CF_*` values, then this block writes those resolved values into `config.gateway.headers` before saving. That means operators who choose service env vars to avoid storing secrets in config still get their Cloudflare secret copied into `node.json`, contradicting the new docs/security contract and widening credential exposure.
  - confidence: 0.9
- **[P2] Do not copy ambient CF vars beside explicit headers:** `src/commands/node-daemon-install-helpers.ts:65-72`
  - body: When `node install --header ...` is used from a shell that also has old `CF_ACCESS_CLIENT_*` values exported, this code still copies those ambient vars into the service environment alongside `OPENCLAW_NODE_HEADERS`. The runtime precedence then lets `CF_*` override the explicit install headers, so a credential rotation can silently install stale proxy credentials.
  - confidence: 0.84
- **[P3] Remove the release-owned changelog entry:** `CHANGELOG.md:17`
  - body: Normal PRs should not edit `CHANGELOG.md`; release generation owns that file. Keep the release-note context in the PR body or squash message and drop this line to avoid release-train conflicts.
