## Summary

Keep open: current main still lacks node WebSocket header plumbing, and this PR is the active implementation path for the closed node-specific request, but the PR head still has a concrete install-time credential precedence bug that should block merge.

## What This Changes

The PR adds repeatable `--header` support to `openclaw node run/install`, persists and resolves node gateway headers from config/env, forwards them to `GatewayClient`, redacts service status env, and updates tests, docs, and changelog.

## Best Possible Solution

Land a current-main node-only implementation where explicit `--header` values win over env/config for foreground and installed daemons, secret-bearing values stay out of status output, and broader non-node CLI header support remains tracked by https://github.com/openclaw/openclaw/issues/49599.

## Reproduction Assessment

Yes. Source inspection shows current main lacks the requested node header path, and the PR-head blocker follows from `buildNodeInstallPlan()` copying `CF_*` env while runtime applies those values after `OPENCLAW_NODE_HEADERS`; I did not run a live Cloudflare or installed-daemon repro in this read-only review.

## Solution Assessment

No. The node-scoped implementation direction is maintainable, but this PR should not merge until explicit install headers cannot be overridden by ambient `CF_*` env and the branch is refreshed against current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve explicit install headers over ambient CF env:** `src/commands/node-daemon-install-helpers.ts:67-74`
  - body: When users run `openclaw node install --header` with stale `CF_ACCESS_CLIENT_ID` or `CF_ACCESS_CLIENT_SECRET` exported, this serializes the explicit headers into `OPENCLAW_NODE_HEADERS` but also copies those ambient `CF_*` values into the installed service. The daemon later merges `OPENCLAW_NODE_HEADERS` before `CF_*` and has no opts headers, so the installed service can use stale ambient credentials instead of the requested flags and hit the same auth/302 failure. Skip copying `CF_*` when `params.headers` is set, or make the explicit serialized headers win at runtime.
  - confidence: 0.91
