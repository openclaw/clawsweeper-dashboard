## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid overriding installed headers with ambient CF env:** `src/commands/node-daemon-install-helpers.ts:67-73`
  - body: Even when `params.headers` came from explicit `openclaw node install --header` flags, this copies any exported `CF_ACCESS_CLIENT_ID`/`CF_ACCESS_CLIENT_SECRET` into the installed service. At runtime the PR applies those CF vars after `OPENCLAW_NODE_HEADERS`, so stale shell credentials can silently replace the headers the user explicitly installed and reintroduce 302/auth failures.
  - confidence: 0.9
- **[P2] Do not persist env-only header secrets unexpectedly:** `src/node-host/runner.ts:209-217`
  - body: The docs say env vars keep secrets out of config and command lines, but `runNodeHost` saves the fully resolved header map back to node config on startup. A daemon configured only with `OPENCLAW_NODE_HEADERS` or `CF_ACCESS_CLIENT_SECRET` will write those secret header values to `node.json`, which violates the new documented secret-handling contract.
  - confidence: 0.88
