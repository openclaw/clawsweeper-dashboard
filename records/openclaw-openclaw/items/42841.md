## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Avoid overriding installed headers with ambient CF env:** `src/commands/node-daemon-install-helpers.ts:67-73`
  - body: When `params.headers` is set from explicit `node install --header` flags, this still copies `CF_ACCESS_CLIENT_ID` and `CF_ACCESS_CLIENT_SECRET` into the service environment. At daemon startup those CF env vars are applied after `OPENCLAW_NODE_HEADERS`, so stale exported credentials can replace the values the user just installed and recreate the Zero Trust auth failure.
  - confidence: 0.93
- **[P2] Align env header docs with config persistence:** `docs/cli/node.md:72`
  - body: The docs tell operators that env vars keep header secrets out of config, but `runNodeHost` saves the resolved env-derived headers into `config.gateway.headers` before writing `node.json`. Either stop persisting env-only header secrets or document that they are written to node config.
  - confidence: 0.9
