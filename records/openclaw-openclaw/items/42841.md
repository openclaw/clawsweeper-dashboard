Full review comments:

  - body: When `node install --header` runs from a shell with stale `CF_ACCESS_*` vars, this writes both `OPENCLAW_NODE_HEADERS` and the ambient CF env into the service. `runNodeHost` then applies CF env after `OPENCLAW_NODE_HEADERS`, so the installed daemon can use stale credentials instead of the explicit flags and hit the same 302/auth failure.
  - confidence: 0.9

This sentence tells operators env vars avoid config storage, which is wrong for the current patch and can cause unexpected credential-at-rest exposure.
  - confidence: 0.88
