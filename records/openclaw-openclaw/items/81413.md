## Summary

Current main and the v2026.5.7 release source both route Google Chat non-DM spaces through a `group` peer into the shared session-key builder, so the reported collapse to `agent:<agentId>:main` does not match the checked code path.

## What This Changes

The issue reports that Google Chat group-space messages create the agent main session key and send replies to WhatsApp instead of the originating Google Chat space.

## Best Possible Solution

Keep the existing group-peer routing path; if this still happens in a deployment, handle it as a new runtime/package mismatch report with raw routing logs and exact installed artifact details.

## Reproduction Assessment

No. The current and v2026.5.7 source path for a Google Chat non-DM space constructs a group peer and the shared session-key builder cannot turn that group peer into `agent:<agentId>:main`.

## Solution Assessment

Yes. Closing this specific report as not reproducible is the narrowest maintainable outcome; changing Google Chat session-key construction would be speculative without logs showing the current code receiving a group space and producing a main key.
