Full review comments:

- **[P1] Pin self-hosted discovery to the configured host:** `src/plugins/provider-self-hosted-setup.ts:70-71`
  - body: Current main now builds `hostnameAllowlist: [parsed.hostname]` for this self-hosted provider path, but the PR branch sends only `{ allowPrivateNetwork: true }`. That broadens API-key-bearing discovery requests from the configured host to any private-network or redirect target accepted by the guard, so the rebase needs to preserve `buildSelfHostedBaseUrlSsrFPolicy(trimmedBaseUrl)` instead of dropping the pin.
  - confidence: 0.94
- **[P1] Do not silently clamp external hook priorities:** `src/plugins/registry.ts:960-961`
  - body: Plugin docs currently define `priority` as descending handler ordering without a public cap, and terminal hooks can stop lower-priority handlers. Clamping external plugins to 100 changes existing plugins that intentionally used higher priorities, so this needs an owner-approved contract, migration/docs/tests, or compatibility-preserving opt-in behavior before merge.
  - confidence: 0.87
- **[P2] Match aliases after earlier import specifiers:** `src/security/skill-scanner.ts:203`
  - body: The new alias rule only matches when `exec` or `spawn` is the first name after `{`, so `import { readFile, exec as run } from "child_process"` or `{ readFile, exec: run }` still bypasses the added scanner rule. The hardening should scan all import/destructuring specifiers or use a parser-backed check and include positive and negative tests.
  - confidence: 0.86
