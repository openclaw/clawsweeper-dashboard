## Summary

Keep this PR open, but not merge-ready: the lease-aware retirement direction is useful, yet the lease is acquired only after startup completes, so a concurrent shared-client startup can still be closed underneath a sibling before it is protected. The supplied proof is also a helper-level terminal probe, not a real app-server startup-close or turn-timeout recovery run.

## What This Changes

The PR adds Codex app-server shared-client lease/deferred-retirement helpers, wires them into startup retry and turn-timeout cleanup paths, and expands focused app-server tests.

## Best Possible Solution

Acquire the shared-client lease as soon as a shared startup client is obtained, release it on all startup and run exits, add a concurrent-startup regression test, and require redacted live runtime proof before merge.

## Reproduction Assessment

Not with a live run in the supplied evidence. Source inspection shows the startup race clearly: the PR defers close only when a lease exists, but run-attempt does not retain the shared client until after startup completes.

## Solution Assessment

Not yet. Lease-aware retirement is the right boundary, but the lease needs to cover startup work itself and the final patch needs real app-server recovery proof.
