## Summary

Keep open: the patch is focused, but source inspection shows it still leaves shared Codex app-server clients unleased during startup, and the supplied proof only exercises the helper rather than the real startup-close or turn-timeout recovery path.

## What This Changes

The PR modifies Codex app-server run-attempt and shared-client lifecycle code to add lease-aware deferred retirement, startup retry isolation, timeout retirement events, and focused tests.

## Best Possible Solution

Acquire the shared-client lease immediately after a shared startup client is obtained, release it on every startup and run exit path, add concurrent-startup regression coverage, and require redacted runtime proof for startup-close and turn-timeout recovery.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection is enough to see the remaining PR race: shared clients are not retained until after startup completes, while retirement only defers when a lease is already present.

## Solution Assessment

Not yet. Lease-aware retirement is the right boundary, but the lease needs to cover startup work itself and the final patch needs real app-server recovery proof.
