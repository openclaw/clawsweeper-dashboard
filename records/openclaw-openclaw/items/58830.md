## Summary

Keep open: current main has useful partial coverage for visibility, checkpoints, restore/branch recovery, hooks, and provider-backed compaction, but it does not implement the requested preview/approve/reject flow or the proposed notify-only/require-confirmation compaction modes.

The issue requests transparent, user-visible context compaction with preview, confirmation, rollback/re-expansion, configuration modes, and developer hooks.

## Best Possible Solution

Define a maintainer-approved compaction UX contract that layers preview and opt-in confirmation on top of the existing notices, checkpoints, hooks, and provider registry without blocking emergency overflow recovery by default.
