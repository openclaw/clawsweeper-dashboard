## Summary

Keep open: current main and the latest release have partial compaction visibility, hooks, and post-apply checkpoint recovery, but they still do not provide the requested pre-apply summary preview, user approval/rejection/modification flow, or confirmation-mode config contract. ## What This Changes

Request to add transparent context compaction controls: user notices, summary preview/edit/approval, rollback buffers, configuration modes, and developer hooks for context trimming or summarization.

## Best Possible Solution

Define a privacy-safe, opt-in compaction lifecycle contract first, then implement preview/confirmation modes with redacted metadata, timeout behavior, checkpoint recovery semantics, UI/API coverage, docs, and focused tests.
