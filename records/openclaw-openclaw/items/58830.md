## Summary

Keep open: current main now covers partial visibility and recovery for compaction, but the central request for preview plus approve/reject confirmation modes is still not implemented and needs maintainer UX/product direction.

## What This Changes

Request adding visible notifications, generated-summary preview, approve/reject controls, rollback, config, and developer hooks for automatic context compaction.

## Best Possible Solution

Design this as a core compaction UX/policy layer: keep automatic behavior as the default, add opt-in preview/confirmation only where an operator can respond, reuse existing checkpoints for recovery, and document non-interactive fallback behavior.

## Reproduction Assessment

Not applicable as a feature request rather than a bug report. Source, schema, docs, tests, and RPC inspection give high-confidence current-main evidence that notices/checkpoints exist but preview and approve/reject modes do not.

## Solution Assessment

Unclear as one immediate patch: the requested direction is useful, but the best solution needs maintainer decisions about interactive versus unattended sessions. A phased core design built on notifyUser, checkpoint, and hook surfaces is safer than forcing confirmation into every automatic compaction path.
