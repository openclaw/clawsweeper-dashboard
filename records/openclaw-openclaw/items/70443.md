## Summary

Keep this open. Current main and the latest shipped tag still build `openclaw:local` or pull `OPENCLAW_IMAGE` before the rest of Docker setup, and there is no supported `--offline` flag or documented offline rerun path. The report is narrow, current, and has a linked operator workaround, so it is a good focused repair candidate rather than a close candidate.

## What This Changes

Add an explicit `--offline` mode to `scripts/docker/setup.sh` so airgapped operators can rerun Docker setup against preloaded images without build or pull network steps.

## Best Possible Solution

Implement a small, explicit `--offline` mode in `scripts/docker/setup.sh`: parse the flag at startup, preserve default build/pull behavior when absent, use `docker image inspect "$OPENCLAW_IMAGE"` to require the main preloaded image, skip sandbox image builds with a warning when offline, and update Docker docs, changelog, and focused e2e coverage.

## Reproduction Assessment

Yes. The high-confidence reproduction path is static and direct: pass or imagine passing `--offline` to current `scripts/docker/setup.sh`; there is no top-level arg parsing, and the script still reaches the unconditional build/pull branch before the supported setup steps.

## Solution Assessment

Yes. An explicit opt-in flag is the narrowest maintainable fix because it leaves existing online behavior unchanged while allowing airgapped reruns to reuse the supported setup path; the safer implementation should be smaller and stricter than the linked workaround patch.
