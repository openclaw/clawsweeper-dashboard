## What This Changes

Request a new opt-in `--offline` mode for `scripts/docker/setup.sh` that skips network image build or pull and continues the supported Docker setup flow against preloaded images.

## Best Possible Solution

Add a Bash 3.2-compatible `--offline` flag that preserves default online behavior, requires the main image via `docker image inspect`, warns for a missing optional sandbox image, and updates Docker setup docs and e2e coverage.

## Reproduction Assessment

## Solution Assessment

Yes. An explicit opt-in flag is the narrow maintainable direction; the linked external workaround is useful evidence but should be refined to preserve default build/pull and compose-startup semantics.
