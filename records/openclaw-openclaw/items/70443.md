## Summary

Keep this open: current main still has no supported Docker setup offline rerun mode, and the script still reaches build or pull before the setup steps the reporter needs after loading an image out of band.

## What This Changes

The issue requests a `--offline` flag for `scripts/docker/setup.sh` that skips build/pull, verifies preloaded images, and continues the normal setup rerun flow.

## Best Possible Solution

Add a Bash 3.2-compatible `--offline` mode to Docker setup, keep default online behavior unchanged, require the main preloaded image with `docker image inspect`, warn for a missing optional sandbox image, and cover the flow in docs and focused e2e tests.

## Reproduction Assessment

Yes. Static source inspection on current main is enough: there is no `--offline` argument path, and the setup script still builds or pulls the image before the supported rerun steps.

## Solution Assessment

Yes. An explicit opt-in flag is the narrowest maintainable direction because it leaves default Docker setup unchanged while allowing airgapped hosts to reuse the supported setup routine after `docker load`.
