## Summary

Keep open: current main still lacks a supported Docker setup offline rerun mode, and the script still builds or pulls the gateway image before the setup steps the reporter needs after loading an image out of band.

## What This Changes

Add an explicit `--offline` flag to `scripts/docker/setup.sh` so airgapped operators can rerun setup with preloaded Docker images without build or pull attempts.

## Best Possible Solution

Add a Bash 3.2-compatible `--offline` mode to Docker setup, keep default online behavior unchanged, require the main preloaded image with `docker image inspect`, warn for missing optional sandbox image, and document/test the flow.

## Reproduction Assessment

Yes. Static source inspection on current main is enough: the setup script ignores `--offline`, has no offline mode variable, and still reaches build or pull before the supported rerun steps.

## Solution Assessment

Yes. An explicit opt-in flag is the narrowest maintainable solution because it preserves default setup behavior while letting airgapped hosts reuse the supported setup routine after `docker load`; the linked workaround should be reworked before adoption.
