## Repair Work Prompt

Implement the offline Docker setup rerun feature requested in https://github.com/openclaw/openclaw/issues/70443. Add a Bash 3.2-compatible `--offline` flag to `scripts/docker/setup.sh` that defaults off and leaves existing online behavior unchanged. In offline mode, skip the main `docker build`/`docker pull`, require `docker image inspect "$IMAGE_NAME"` to pass with an error that points users to `docker load` when missing, and keep the rest of setup running unchanged: env validation/upsert, permission fix-up, onboarding unless skipped, gateway config sync, compose startup, and sandbox overlay/config handling. For `OPENCLAW_SANDBOX`, do not build the sandbox image in offline mode; inspect `openclaw-sandbox:bookworm-slim` and warn/skip sandbox setup if it is missing. Do not copy the reporter's external patch as-is because it changes default local-build behavior and adds `--force-recreate`. Update `src/docker-setup.e2e.test.ts` so the helper can pass script args and covers offline success, missing main image failure, default online behavior, and sandbox warn/skip behavior. Update `docs/install/docker.md` with the public offline rerun flow and add a maintainer-owned changelog entry if policy requires one for this user-facing setup flag.
