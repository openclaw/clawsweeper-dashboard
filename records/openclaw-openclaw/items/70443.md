## Summary

Keep this open: current main still has no supported Docker setup offline rerun mode, and the request remains actionable. The reporter's linked external patch is useful workflow evidence, but a core fix should be narrower because that patch also changes compose startup semantics.

## What This Changes

Add a `--offline` flag to `scripts/docker/setup.sh` so airgapped operators can rerun setup with a preloaded Docker image without rebuilding or pulling.

## Best Possible Solution

Add a Bash 3.2-compatible `--offline` flag that preserves default online behavior, requires the main image via `docker image inspect`, warns for a missing optional sandbox image, and documents/tests the rerun flow.

## Reproduction Assessment

Yes. Static source inspection on current main is enough to reproduce the support gap: there is no `--offline` argument path, and setup still reaches `docker build` or `docker pull` before the later rerun steps.

## Solution Assessment

Yes. An explicit opt-in flag is the narrow maintainable direction; the linked external workaround should be treated as workflow evidence rather than applied directly because it changes default behavior and compose startup semantics.

Implement a narrow `--offline` mode for `scripts/docker/setup.sh` for https://github.com/openclaw/openclaw/issues/70443. Parse the flag in Bash 3.2-compatible shell without changing default behavior. When offline, skip the main image build/pull, require `docker image inspect "$OPENCLAW_IMAGE"` to pass with an error that points operators to `docker load` if missing, and continue through env validation, `.env` upsert, permission fix-up, onboarding, gateway config sync, compose startup, and sandbox overlay handling. For `OPENCLAW_SANDBOX`, skip rebuilding `openclaw-sandbox:bookworm-slim`; inspect it and warn/disable sandbox safely if the optional image is missing. Do not copy the reporter's external patch as-is, and do not add `--force-recreate` or change default local-build semantics. Add focused coverage in `src/docker-setup.e2e.test.ts`, update `docs/install/docker.md`, and add a `CHANGELOG.md` Changes entry.
