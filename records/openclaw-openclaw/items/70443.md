## Summary

Keep this open. Current main still has no supported `--offline` flag for Docker setup, and the setup script still builds `openclaw:local` or pulls `OPENCLAW_IMAGE` before the rest of the rerun steps that airgapped operators need.

## What This Changes

Add an explicit `--offline` mode to `scripts/docker/setup.sh` so airgapped operators can rerun Docker setup against preloaded images without build or pull network steps.

## Best Possible Solution

Implement a small Bash 3.2-compatible `--offline` flag in `scripts/docker/setup.sh`, preserve default behavior when absent, require the main preloaded image with `docker image inspect`, keep later setup steps intact, warn for a missing optional sandbox image, and document/test the rerun flow.

## Reproduction Assessment

Yes. The high-confidence reproduction path is static: current `scripts/docker/setup.sh` has no top-level `--offline` parsing and still reaches the build/pull branch before the supported setup rerun steps.

## Solution Assessment

Yes. An explicit opt-in flag is the narrowest maintainable solution because it leaves online setup unchanged while letting airgapped hosts reuse the supported setup routine after `docker load`.
