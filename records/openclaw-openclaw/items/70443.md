## Summary

Keep this open: current main still has no supported Docker setup offline rerun mode, and the issue remains actionable. The reporter's linked external patch is useful context, but it is a local workaround rather than an upstream implementation and should not be copied directly because it changes default local-build behavior.

## What This Changes

Add an explicit `--offline` mode to `scripts/docker/setup.sh` so airgapped operators can rerun the supported Docker setup flow with an image loaded out of band.

## Best Possible Solution

Add a Bash 3.2-compatible `--offline` flag that preserves default online behavior, requires the main image via `docker image inspect`, warns for a missing optional sandbox image, and documents/tests the flow.

## Reproduction Assessment

Yes. Static source inspection on current main is enough: there is no top-level `--offline` argument path, and the script still builds or pulls before the supported rerun steps.

## Solution Assessment

Yes. An explicit opt-in flag is the narrow maintainable direction; copying the external workaround directly would be less safe because it changes default local-build behavior instead of guarding the new mode behind `--offline`.
