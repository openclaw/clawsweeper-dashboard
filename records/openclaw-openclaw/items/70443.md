## Summary

Keep this open: current main still has no supported Docker setup offline rerun mode, and the requested opt-in flag remains a clear, narrow feature request.

## What This Changes

Add a `--offline` flag to `scripts/docker/setup.sh` so preloaded Docker images can skip build/pull while still running the rest of setup.

## Best Possible Solution

Add a Bash 3.2-compatible `--offline` flag that preserves default online behavior, requires the main image via `docker image inspect`, warns for a missing optional sandbox image, and documents/tests the rerun flow.

## Reproduction Assessment

Yes. Static source inspection on current main is enough to show there is no `--offline` argument path and setup still reaches `docker build` or `docker pull` before later rerun steps.

## Solution Assessment

Yes. An explicit opt-in flag is the narrow maintainable direction; the linked external workaround should be treated as workflow evidence rather than copied directly because it changes default local-build and compose-startup semantics.

## Repair Work Prompt

Implement https://github.com/openclaw/openclaw/issues/70443 as a narrow Docker setup feature. Add a Bash 3.2-compatible `--offline` flag to `scripts/docker/setup.sh` that is parsed from script arguments near the top. When unset, preserve the existing build/pull behavior. When set, skip main image build/pull, require `docker image inspect "$IMAGE_NAME"` to succeed, fail with a clear message pointing operators to `docker load` if the main image is missing, and keep all later setup work unchanged: env validation/upsert, data-dir permission fix, onboarding unless skipped, gateway config sync, compose startup, and sandbox overlay handling. For `OPENCLAW_SANDBOX=1`, skip the sandbox image build in offline mode and warn, not fail, if `openclaw-sandbox:bookworm-slim` is missing, while avoiding messages that claim sandbox is usable when prerequisites are absent. Do not add an env-var alias, do not change default compose startup semantics, and do not copy the external workaround's unconditional inspect-only or `--force-recreate` changes. Update `src/docker-setup.e2e.test.ts` to pass script args and cover successful preloaded-image use, missing main image failure, and missing optional sandbox warning. Update `docs/install/docker.md` with the offline rerun flow and add a concise `CHANGELOG.md` entry because this is user-facing setup behavior.
