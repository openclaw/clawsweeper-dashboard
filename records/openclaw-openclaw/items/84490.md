## Summary

Keep open: current main and v2026.6.8 mitigate the visible unknown context display with an estimated fallback, but they still do not make a generic hosted USTC OpenAI-compatible provider persist exact fresh `totalTokens` unless streaming-usage compatibility is explicitly declared.

## What This Changes

The issue asks OpenClaw to persist and display real direct-session token usage instead of `unknown/1049k (?%)` for custom USTC OpenAI-compatible sessions that stored `totalTokens` as null on v2026.5.18.

## Best Possible Solution

Verify whether the USTC endpoint supports streamed usage, then either declare that capability through provider/plugin metadata or document the existing model-level compat setting while keeping the generic hosted-provider default conservative.

## Reproduction Assessment

Yes at source level for generic hosted custom OpenAI-compatible providers without `compat.supportsUsageInStreaming`: current code will not send `stream_options.include_usage`, so exact usage cannot reach session persistence. The USTC-specific endpoint still needs live proof.

## Solution Assessment

No as a complete fix: current main's estimated context fallback is useful but does not persist exact provider totals. The maintainable fix depends on verified USTC behavior and a maintainer choice between provider metadata, docs/config guidance, or changing generic defaults.
