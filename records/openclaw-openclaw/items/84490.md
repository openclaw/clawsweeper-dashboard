## Summary

Keep open: current main and v2026.6.10 still keep hosted custom OpenAI-compatible providers conservative unless `compat.supportsUsageInStreaming` or provider metadata says streamed usage is supported. ## What This Changes

The issue asks for direct sessions using a hosted OpenAI-compatible USTC provider to persist and display exact token usage instead of unknown or estimated totals.

## Best Possible Solution

Verify USTC streamed-usage behavior, then either add provider/plugin metadata for that endpoint or document the existing model-level compat setting while keeping the generic hosted-provider default conservative.

## Reproduction Assessment

No full current-main live repro is available. Source inspection shows a hosted custom OpenAI-compatible provider without `compat.supportsUsageInStreaming` still will not request streamed usage, but the USTC endpoint itself needs live proof.

## Solution Assessment

No, current main is only an acceptable mitigation for the visible status display. The best fix depends on verified provider behavior and a maintainer choice between provider metadata, documentation, or changing the conservative generic default.
