## Summary

## What This Changes

The PR relaxes cron persisted-store validation to accept any non-empty string `payload.kind` while keeping known-kind structural checks, and adds unit coverage for legacy and unknown payload kinds.

## Best Possible Solution

Land a lossless persistence fix with store-reload regression coverage and real proof, while keeping execution limited to canonical payload kinds unless a separate doctor migration defines legacy-kind conversion.

## Solution Assessment

Partially yes: relaxing persisted-shape validation is a narrow preservation fix, but the PR still needs full store-cycle proof and maintainer confirmation that unknown kinds should be retained but not made runnable.
