## What This Changes

The PR relaxes cron persisted payload validation so unknown non-empty string `payload.kind` values are preserved while known kinds keep their existing structural checks.

## Best Possible Solution

Land a narrow preservation fix with focused store-reload regression coverage and real reload proof, then add or track doctor migration for specific legacy kinds if those jobs are expected to run again automatically.
