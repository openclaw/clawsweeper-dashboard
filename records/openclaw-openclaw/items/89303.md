Full review comments:

- **[P1] Keep `/cancel` out of shared abort detection:** `src/auto-reply/reply/abort-primitives.ts:82-84`
  - body: This makes `/cancel` an abort request for every surface that uses `tryFastAbortFromMessage`, so Slack/Discord/text messages can now abort the current session even though this PR scopes `/cancel` to Telegram replies.   - confidence: 0.92
- **[P1] Check activity before clearing reply-scoped queues:** `src/auto-reply/reply/commands-session-abort.ts:214-218`
  - body: For a stored replied-to target with no active run, `clearSessionQueues` has already cleared follow-ups/lanes before `requireActive` can return false, so the user can get a no-active-work response after queued work was dropped. Move queue cleanup behind a successful active abort for reply-scoped targets.
  - confidence: 0.87
- **[P2] Align the new SDK export with the plugin contract:** `src/plugin-sdk/session-store-runtime.ts:23`
  - body: `session-store-runtime` is a public plugin SDK subpath, but the PR adds a mutating helper without updating the SDK subpath docs or making the maintainer API decision visible. Either keep this internal or update the SDK contract/docs so external plugin authors know it is supported.
  - confidence: 0.78
