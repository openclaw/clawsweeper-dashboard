## Summary

Keep open. Current main still lacks the requested OpenClaw-to-OpenClaw cross-platform archive export/import flow; the remaining work is a new, security-sensitive product surface and is still tied to an open fork implementation plus a previously closed upstream PR that was explicitly closed for owner-backed product direction.

## What This Changes

Add cross-platform `openclaw migrate export` and `openclaw migrate import` commands for moving selected OpenClaw state between Linux, Windows, and macOS machines.

## Best Possible Solution

Choose the recovery safety model first, then either evolve `openclaw backup` with platform-aware export/convert and clearer manual restore docs or integrate a guarded archive import into the existing provider-based migrate namespace with security-focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction. The feature-gap check is high-confidence from source/docs: current `migrate` is provider-based, `backup` is create/verify only, and machine-to-machine migration remains documented as manual copy/extract/doctor steps.

## Solution Assessment

Unclear. The user problem is valid, but the exact `migrate import` design is not yet proven as the best path because maintainer discussion raised safer `backup` platform-targeting or conversion alternatives.

## Security Review

Status: needs_attention

Summary: The requested native import/restore flow is security-sensitive because it would apply user-provided archives to credential, session, plugin, and workspace paths.

Concerns:

- **[medium] Define archive trust and target containment before import:** `docs/cli/backup.md:54`
  - body: Any first-party archive import must settle path traversal, symlink/hardlink, target-root, overwrite, rollback, and partial-failure behavior before destructive writes are allowed.
  - confidence: 0.9
- **[medium] Define secret and runtime-state handling:** `docs/concepts/agent-workspace.md:105`
  - body: The export/import flow must explicitly include, exclude, or redact auth profiles, credentials, sessions, managed skills, executable plugin content, and machine-local paths so users do not create unsafe portable archives.
  - confidence: 0.88
