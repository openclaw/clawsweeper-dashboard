## Summary

Keep open. Current main has provider-based imports and backup create/verify, but it still lacks same-product cross-platform export/import or restore semantics; the matching implementation work is unmerged and prior maintainer discussion flagged restore/import as a product and security decision.

## What This Changes

Request `openclaw migrate export` and `openclaw migrate import` commands for cross-platform OpenClaw workspace and state migration.

## Best Possible Solution

Keep this as the narrow OpenClaw-to-OpenClaw migration tracker until maintainers decide whether the end state is hardened backup restore/convert or a new preview-first migration import surface with explicit secret and state policy.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken existing behavior. Current source and docs give high-confidence evidence that backup create/verify and provider-based migrate exist, but not same-product export/import.

## Solution Assessment

Unclear. The requested direction is useful, but the safest maintainable solution may be backup convert/restore or a redesigned import flow rather than adding archive import directly to the existing provider migration namespace.
