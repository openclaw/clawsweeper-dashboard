## Summary

Keep open. Current main has provider-based migration imports, backup create/verify, and manual machine-migration docs, but it still lacks the requested first-party same-product cross-platform export/import or restore workflow; the linked implementation attempts are unmerged and the discussion needs product/security direction.

## What This Changes

The issue requests new `openclaw migrate export` and `openclaw migrate import` commands for selective, cross-platform OpenClaw state migration.

## Best Possible Solution

Choose one first-party portable state migration model, then implement it as a hardened preview-first flow with archive validation, path remapping, secret policy, conflict handling, rollback behavior, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, tests, and related GitHub checks give high-confidence proof that current main lacks same-product `migrate export/import` or native cross-platform restore.

## Solution Assessment

Unclear as an immediate implementation path. The requested capability is useful, but maintainers should choose the command namespace and safety model before landing any archive-writing import or restore flow.
