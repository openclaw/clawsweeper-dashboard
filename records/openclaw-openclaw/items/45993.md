## Summary

Keep open. Current main still lacks the requested OpenClaw-to-OpenClaw cross-platform export/import flow, the canonical duplicate issues point back here, and the linked open implementation remains a product/security decision rather than a stale cleanup candidate.

## What This Changes

Request `openclaw migrate export` and `openclaw migrate import` commands for selective, cross-platform OpenClaw state/workspace migration.

## Best Possible Solution

Choose the recovery safety model first, then either evolve `openclaw backup` with platform-aware export/convert and clearer manual restore docs or add a guarded import path with threat-modelled archive validation and rollback behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. The feature-gap check is high-confidence from current source/docs: `migrate` is provider-based, `backup` is create/verify only, and machine-to-machine migration remains manual.

## Solution Assessment

Unclear. The user problem is valid, but the exact `migrate import` design is not yet proven as the best path because backup conversion or manual restore guidance may be safer than native destructive archive import.

## Security Review

Status: needs_attention

Summary: The requested native import/restore flow is security-sensitive because it would apply user-provided archives to credential, session, plugin, and workspace paths.

Concerns:

- **[medium] Define archive trust and target containment before import:** `docs/cli/backup.md:54`
  - body: Any first-party archive import must settle path traversal, symlink/hardlink, target-root, overwrite, rollback, and partial-failure behavior before destructive writes are allowed.
  - confidence: 0.9
- **[medium] Define secret and runtime-state handling:** `docs/concepts/agent-workspace.md:105`
  - body: The export/import flow must explicitly include, exclude, or redact auth profiles, credentials, sessions, managed skills, executable plugin content, and machine-local paths so users do not create unsafe portable archives.
  - confidence: 0.88
