## Summary

Keep open. Current main has `openclaw backup create/verify` and provider-based imports from other agent systems, but it still lacks the requested same-product `migrate export/import` or native restore/import workflow; the closed target PR and open cross-repository PR show useful source work, not a merged implementation.

## What This Changes

Request to add cross-platform `openclaw migrate export` and `openclaw migrate import` commands for moving OpenClaw state between machines.

## Best Possible Solution

Keep this as the cross-platform OpenClaw-to-OpenClaw migration tracker, align it with https://github.com/openclaw/openclaw/issues/13616, and have maintainers choose the backup-vs-migrate namespace plus restore/archive safety model before splitting implementation into smaller PRs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, GitHub search, and related PR checks give high-confidence proof that current main has create/verify and provider import paths, but not the requested same-product export/import or restore workflow.

## Solution Assessment

No, not as a single automated fix. The maintainable path is to settle archive trust, cross-platform path remapping, secret handling, overwrite/merge/rollback behavior, and CLI namespace first, then implement reviewable slices.
