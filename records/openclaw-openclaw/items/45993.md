## Summary

Keep open. Current main still has provider-based imports from other agent systems plus backup create/verify, but it does not implement same-product cross-platform migration export/import, archive import, or backup restore; an open external PR and the prior closed PR show useful implementation work still needs maintainer product and security review.

## What This Changes

Add same-product cross-platform OpenClaw migration export/import commands with selective archive contents, path adaptation, environment-variable handling, memory merge/replace options, and dependency checks.

## Best Possible Solution

Keep this issue as the canonical tracker for same-product OpenClaw migration. Maintainers should choose between `migrate export/import`, `backup restore`, or a safer backup conversion/manual-restore workflow, then review or replace the open implementation with archive validation, secret handling, path remapping, docs, and targeted tests.
