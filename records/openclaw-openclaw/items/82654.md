## Summary

Keep open: the bug is source-proven and the production change is close, but this PR is not mergeable because the new test imports non-exported helpers, fractional positive limits can still floor to zero and hit `slice(-0)`, and external real behavior proof is missing.

## What This Changes

The PR changes `getRecentDiagnosticPhases` to reject zero, negative, and non-finite limits and adds colocated regression tests.

## Best Possible Solution

Normalize the limit before slicing, require the normalized value to be finite and positive, seed tests through existing exports such as `recordDiagnosticPhase` and `resetDiagnosticPhasesForTest`, and add redacted after-fix proof before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows `getRecentDiagnosticPhases(0)` reaches `slice(-0)`, which returns the retained recent-phase buffer instead of an empty list.

## Solution Assessment

No. The intended helper-level guard is the right direction, but this patch leaves a normalized-zero path for fractional limits and its regression test imports helpers that are not exported.
