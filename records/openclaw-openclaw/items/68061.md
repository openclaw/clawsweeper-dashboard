## Summary

Keep this PR open, but it is not merge-ready. Current main still lacks WebChat Mermaid rendering, so the feature request is real, but this branch introduces build/runtime blockers and removes existing markdown security and rendering contracts on a user-controlled Control UI surface.

## What This Changes

The PR adds Mermaid diagram rendering to the WebChat/Control UI markdown path by adding `mermaid`, changing the markdown renderer and lifecycle code, updating the lockfile, and broadly modifying Control UI component CSS.

## Best Possible Solution

Keep the feature active but require a narrower WebChat/Control UI implementation: preserve the current `markdown-it` contracts, task-list behavior, URL-scheme hardening, localized English-source UI strings, and existing layout styling; add a browser-safe lazy Mermaid rendering path plus focused regression tests for Mermaid fences, sanitizer URLs, task lists, code blocks, and non-browser imports.

## Reproduction Assessment

Yes. Static reproduction is high confidence: the PR imports a non-exported `attachThemeListener`, touches `window` at module load, drops sanitizer allowlist entries that current tests cover, and current main renders Mermaid fences as ordinary code blocks because no UI Mermaid implementation exists.

## Solution Assessment

No. The shared Control UI/WebChat markdown surface is the right place for the feature, but this branch is not the best solution because it rewrites parser behavior, removes hardening, adds eager browser globals, and changes unrelated CSS instead of adding a focused Mermaid rendering seam.

## Security Review

Status: needs_attention

Summary: Security review needs attention because the PR adds a Mermaid SVG insertion path and removes existing markdown URL-scheme hardening.

Concerns:

- **[high] URL scheme allowlist removed:** `ui/src/ui/markdown.ts:187`
  - body: Current main explicitly strips anchor hrefs unless their protocol is `http:`, `https:`, or `mailto:`. The PR removes that hook logic from user-controlled rendered markdown, reducing defense-in-depth around dangerous URL schemes.
  - confidence: 0.9
- **[medium] Mermaid SVG bypasses DOMPurify output sanitization:** `ui/src/ui/markdown.ts:361`
  - body: `renderMermaidInContainer` assigns Mermaid's returned SVG to `innerHTML` after the normal markdown HTML has already been sanitized. This new trusted-SVG boundary should be isolated and reviewed before merge.
  - confidence: 0.82
- **[low] Large new transitive dependency surface:** `pnpm-lock.yaml:1329`
  - body: Adding `mermaid` brings a substantial diagram-rendering dependency graph into the Control UI lockfile. That may be acceptable for the feature, but it should be intentional and kept out of non-browser import paths.
