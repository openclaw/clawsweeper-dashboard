## Summary

Keep open. This is a real WebChat/Control UI feature request, but current main still renders Mermaid fences as normal code blocks and this PR is not mergeable because it introduces build/runtime blockers, removes markdown sanitizer hardening, and mixes a security-sensitive SVG dependency with broad unrelated CSS churn.

## What This Changes

The PR adds Mermaid rendering to the WebChat/Control UI markdown path by adding mermaid, changing markdown and lifecycle code, updating the lockfile, and altering broad component CSS.

## Best Possible Solution

Land a narrower WebChat/Control UI Mermaid implementation that preserves the current markdown-it contracts, DOMPurify URL-scheme hardening, task-list and code-block behavior, English/i18n-ready UI copy, and existing layout styling except for focused diagram presentation.

## Reproduction Assessment

Yes. Static reproduction is high confidence: current main has no UI Mermaid renderer, while the PR patch imports a non-exported theme listener, touches window at module load, removes sanitizer hardening, and deletes CSS still referenced by UI components.

## Solution Assessment

No. Adding Mermaid support to the shared WebChat/Control UI renderer is the right surface, but this branch is not the best implementation because it rewrites core markdown behavior and introduces security-sensitive SVG/dependency changes with unrelated CSS churn.

## Security Review

Status: needs_attention

Summary: Security review needs attention because the PR adds a new Mermaid SVG rendering dependency/path and removes existing markdown sanitizer hardening.

Concerns:

- **[high] URL scheme allowlist removed:** `ui/src/ui/markdown.ts:183`
  - body: Current main explicitly removes anchor hrefs unless their protocol is http:, https:, or mailto:. The PR removes that hook logic from a user-controlled markdown surface, reducing defense-in-depth around dangerous URL schemes.
  - confidence: 0.9
- **[medium] Mermaid SVG bypasses the normal sanitizer pass:** `ui/src/ui/markdown.ts:361`
  - body: The PR assigns Mermaid's returned SVG into innerHTML after the normal markdown HTML has already been sanitized. Mermaid strict mode helps, but this is a new trusted-SVG boundary that should be reviewed and isolated behind a browser-only render path.
  - confidence: 0.82
- **[low] Large new transitive dependency surface:** `pnpm-lock.yaml:1329`
  - body: Adding mermaid brings a large graph of rendering and parser dependencies into the Control UI bundle and lockfile. That may be acceptable for the feature, but it should be intentional and kept out of non-browser import paths.
  - confidence: 0.75
