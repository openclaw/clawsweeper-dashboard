## Summary

Keep this PR open. It usefully changes Active Memory cache and zero-hit terminal behavior, but it does not implement the claimed `status=no-results` path: normal `NONE`/empty-summary recall still resolves to `status=empty`, and the supplied proof does not match the actual diff.

## What This Changes

The PR changes Active Memory to stop caching empty recall results, stop treating zero-hit `memory_search` transcript entries as terminal empty results, and report unavailable terminal searches as `status=unavailable`, with updated tests.

## Best Possible Solution

Land one covered Active Memory fix that adds a chosen distinct normal no-relevant-memory status, reserves empty/unavailable/timeout for their real states, and covers cache, terminal watcher, and status-line tests together.

## Reproduction Assessment

Yes, source-reproducible: `NONE` normalizes to no summary and the current empty-summary branch persists `status="empty"`; adjacent tests assert the same status and cache behavior. I did not run tests in this read-only review.

## Solution Assessment

No. The current PR is not the best standalone fix because it handles cache and terminal zero-hit behavior but leaves the normal no-summary branch at `status="empty"`; the safer path is a single status-classification change with aligned tests and proof.
