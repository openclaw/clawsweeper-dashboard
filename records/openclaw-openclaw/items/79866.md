## Summary

Keep open: this targets a real source-reproducible Active Memory bug, but the patch does not implement the claimed `status=no-results` path; normal `NONE`/empty-summary recall still returns `status=empty`, so the linked bug would remain unresolved.

## What This Changes

The branch changes Active Memory so empty recall results are not cached, zero-hit terminal `memory_search` results no longer end the recall race, and unavailable terminal search results surface `status=unavailable`, with matching test updates.

## Best Possible Solution

Land one covered Active Memory fix that maps successful no-relevant-memory recall to a chosen non-error status, avoids sticky empty caching, preserves retry behavior, and updates the focused tests plus any user-facing docs/changelog needed at landing.

## Reproduction Assessment

Yes. Source inspection shows `normalizeActiveSummary("NONE")` returns null and `maybeResolveActiveRecall` then creates an `ActiveRecallResult` with `status: "empty"`; adjacent tests assert `status=empty` and cached empty behavior.

## Solution Assessment

No. The patch addresses cache stickiness and the terminal zero-hit fast-fail, but the narrow maintainable fix also needs to change the normal `NONE`/empty-summary result classification and align the chosen public status vocabulary and tests.
