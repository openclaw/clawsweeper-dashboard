## Summary

Keep open: this is a focused bug-fix PR, but the current diff does not implement the advertised `no-results`/distinct no-memory status. It stops caching `empty` results and renames unavailable terminal exits, while the normal `NONE`/no-useful-memory path still builds `status="empty"`, so the linked user-visible bug remains.

## What This Changes

The PR changes Active Memory caching and terminal memory-search handling so empty recall results are not cached and unavailable search exits report `unavailable` instead of `empty`.

## Best Possible Solution

Update the Active Memory result contract so normal `NONE`/no-useful-memory recalls produce a distinct non-failure status, keep unavailable/error and timeout states separate, and cover the status line plus cache behavior with focused tests.
