## Summary

Keep open: the branch changes useful adjacent Active Memory behavior, but it does not implement the promised no-results/no-useful-memory status on the normal `NONE` path, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR changes Active Memory cache eligibility, terminal `memory_search` early-exit classification, and tests so empty results are not cached and unavailable searches use `unavailable`.

## Best Possible Solution

Keep the no-injection behavior, add a distinct successful no-result status for normal `NONE` or empty-summary recall, reserve `empty` and `unavailable` for genuinely empty/unavailable states, and update tests and docs to pin that contract.

## Reproduction Assessment

## Solution Assessment

No, this PR is not yet the best way to solve the issue. Its cache and terminal watcher changes may be useful, but the normal no-summary branch still returns `empty`; the narrow fix is to add and test a distinct no-result status there.
