## Summary

Keep open: current main still has the internal hook-runner timeout map seam and plugin-author `timeoutMs`, but no schema-backed user config key or gateway wiring that can populate those maps.

## What This Changes

Request to add user configuration for per-hook plugin timeout maps and pass it into the gateway hook runner.

## Best Possible Solution

If maintainers accept the policy, add a bounded documented config contract, normalize it, wire it into global hook-runner initialization, and preserve plugin registration `timeoutMs` precedence.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that the internal runner option exists but no current user config path reaches it.

## Solution Assessment

Unclear pending maintainer product choice: the runner seam makes this mechanically feasible, but the public config shape, bounds, and global-versus-entry scope should be decided before implementation.
