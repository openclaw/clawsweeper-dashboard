## Summary

Keep open: current main has internal hook-runner timeout maps and plugin-author `timeoutMs`, but no schema/type/loader/global-runner path that lets user config populate per-hook timeout maps.

## What This Changes

Request a user-configurable per-hook plugin timeout map so operators can raise hook budgets without editing installed plugin files.

## Best Possible Solution

Define a bounded schema-backed timeout config contract, include it in the normalized plugin config/cache path, pass it into global hook-runner initialization, and document that plugin-owned `timeoutMs` remains the highest-precedence override.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection confirms the requested user config surface is absent while the internal runner override seam and plugin-author timeout option exist.

## Solution Assessment

Unclear pending maintainer product choice: the runner seam makes the implementation mechanically feasible, but the public path, bounds, and global-versus-entry scope need a deliberate config-contract decision.
