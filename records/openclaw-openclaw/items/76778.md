## Summary

Keep open: current main has the hook-runner override seam and plugin-author `timeoutMs`, but no schema/type/global-runner path that lets user config populate per-hook timeout maps.

## What This Changes

Expose a user-configured per-hook timeout map and wire it into plugin hook runner defaults for modifying and observation hooks.

## Best Possible Solution

If maintainers accept the policy, add a bounded schema-backed hook-timeout config contract, wire it into global hook runner initialization, and document precedence against plugin-owned `timeoutMs`.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection confirms the requested user config surface is absent while the internal runner override seam and plugin-author timeout option exist.

## Solution Assessment

Unclear pending maintainer product choice: the runner seam makes the implementation mechanically feasible, but the public path, max bounds, and whether this belongs globally or under plugin entries need a deliberate config-contract decision.
