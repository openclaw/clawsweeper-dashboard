## Summary

Keep open: current main still has only the internal hook-runner timeout maps and plugin-author `timeoutMs`; exposing user config would add a new public config option and needs maintainer policy on shape and bounds.

## What This Changes

Request to expose per-hook plugin hook timeout overrides in user config and feed them into the hook runner timeout maps.

## Best Possible Solution

If maintainers accept the policy, add a bounded schema-backed hook-timeout config contract, wire it into global runner initialization, and document precedence against plugin-owned `timeoutMs`.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection on current main confirms the internal timeout maps exist, but no schema/type/global-runner path exposes them to user config.

## Solution Assessment

Unclear until maintainer product decision. The implementation is mechanically feasible through `HookRunnerOptions`, but the narrow maintainable fix needs an agreed config path, validation bounds, and docs contract.
