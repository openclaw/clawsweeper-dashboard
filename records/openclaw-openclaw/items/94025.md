## Summary

Keep open. Current main and v2026.6.8 still build Docker sandbox bind sources from Gateway-visible materialized skill paths, and this issue is already paired with two open closing PRs that need maintainer/security review rather than a new cleanup close.

## What This Changes

The issue requests host-visible path handling for Docker-outside-of-Docker sandbox binds so materialized Gateway state resources mount correctly into sandbox containers.

## Best Possible Solution

Land one reviewed, security-validated Docker Gateway/DooD host-path mapping contract for managed, custom, and browser sandbox binds, with schema/docs/tests and live DooD proof for materialized skill readability.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: current main builds materialized skill paths under Gateway state and passes them directly as Docker bind sources, while the docs say DooD bind sources are resolved in the host namespace.

## Solution Assessment

Is this the best way to solve the issue? Unclear: host-path translation is plausible, but the best fix needs a maintainer-approved operator/config contract and sandbox validation before either linked PR lands.
