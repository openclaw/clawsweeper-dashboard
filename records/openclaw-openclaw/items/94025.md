The issue asks OpenClaw to make Docker-outside-of-Docker sandbox materialized skill mounts use host-visible paths instead of Gateway-container paths.

## Best Possible Solution

Land one security-reviewed Docker/DooD bind-source contract that covers managed workspace and materialized skill mounts, with schema/docs/tests and redacted live proof that sandboxed skill reads work.
