## Summary

Keep open. ## What This Changes

The issue requests Docker-outside-of-Docker support for translating OpenClaw Gateway container state paths to host-visible bind-mount paths for sandbox resources such as materialized skills.

## Best Possible Solution

Review and consolidate the linked fix PRs into one supported Docker sandbox path-mapping design, or explicitly keep the documented host-path parity requirement if maintainers do not want a new config surface.

## Reproduction Assessment

Yes at source level, but not from a live Docker DooD run in this review. Current main formats sandbox mount sources directly into Docker `-v` arguments without a host-path translation step, and Docker bind mounts are resolved by the daemon host.

## Solution Assessment

Unclear. A path-translation config could solve this deployment, but OpenClaw already documents a host-path parity requirement for DooD and adding a new env/config surface needs maintainer approval.
