## Summary

This should stay open: current main still does not pass Claude Code `--effort` from the resolved OpenClaw thinking level, and #77410 is an open closing PR for the same bug rather than a merged fix.

## What This Changes

The issue asks OpenClaw to either map stored `/think` levels to Claude CLI `--effort` or make the unsupported CLI-backend status controls clear.

## Best Possible Solution

Review and land #77410, or an equivalent plugin-owned argv seam, so non-off `/think` levels map to Claude CLI `--effort` with tests and docs; keep this issue open until the fix merges.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: select `claude-cli`, set a non-off `/think` level, and the run still builds argv from backend args/resumeArgs with no effort input; I did not run a live Claude CLI repro because no `claude` binary is present.

## Solution Assessment

Yes. The narrow maintainable direction is to consume the existing resolved thinking level in the CLI backend execution path, preferably through a plugin-owned request-args seam rather than hard-coding Claude behavior in generic runner code.
