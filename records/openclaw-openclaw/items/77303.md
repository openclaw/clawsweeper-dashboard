## Summary

Current main still leaves the selected OpenClaw thinking level disconnected from the Anthropic `claude-cli` argv path, while the linked docs PR is open and not yet a merged resolution.

## What This Changes

Make Claude CLI users either get an effective `/think` mapping to Claude Code effort or clear docs/status behavior that the runtime does not support it.

## Best Possible Solution

Decide whether #77370 is sufficient as the supported contract; otherwise add a typed backend-owned per-request argv seam and let the Anthropic plugin map supported non-off thinking levels to `--effort`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: select `claude-cli`, set a non-off thinking level, and the run still builds argv from backend args that never receive `thinkLevel` or `--effort`.

## Solution Assessment

Unclear. A docs-only mitigation is already proposed in #77370, while real runtime support likely needs an additive CLI backend contract change rather than hardcoding Claude-specific policy in core.
