## Summary

This should stay open: current main still carries OpenClaw thinking state to the CLI run but never maps it into the Anthropic Claude CLI argv, while status continues to display Think and Reasoning controls for the session.

## What This Changes

The issue asks for the `claude-cli` backend to honor `/think`/`thinkingLevel` by passing Claude Code `--effort`, or to stop presenting unsupported controls as active.

## Best Possible Solution

Wire the existing resolved thinking level into the Claude CLI backend as `--effort` for supported non-off levels across fresh, resumed, and live sessions, keep `/reasoning` as visibility-only, and add focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set a non-off `/think` level for `claude-cli`, then the run still builds argv from backend args and `buildCliArgs` without any `--effort` mapping.

## Solution Assessment

Yes. The narrowest maintainable fix is to consume the existing `thinkLevel` in the Claude CLI backend/arg path instead of adding another user config knob; reasoning visibility should remain separate.
