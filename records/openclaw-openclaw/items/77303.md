## Summary

Current main still carries the selected thinking level into the CLI run path, but neither the bundled Claude CLI backend args nor the generic CLI arg builder map it to Claude Code's `--effort` flag, while `/status` still renders `Think` and `Reasoning`. The related docs PR is open and unmerged, so this needs maintainer choice rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to either make the `claude-cli` backend honor `/think` through Claude Code `--effort` or clearly mark the Think/Reasoning controls as unsupported for CLI runtime mode.

## Best Possible Solution

Pick one explicit contract: either add tested plugin-owned per-request CLI arg support that maps Claude non-off thinking levels to `--effort`, including live-session restart behavior, or merge docs/status copy that labels these controls unsupported for CLI backends.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: use `claude-cli` with a non-off `/think` level, then the runner still builds argv from backend args that contain no `--effort` while `/status` displays the selected Think/Reasoning state.

## Solution Assessment

Unclear. Both proposed directions are technically plausible, but selecting docs-only unsupported labelling versus adding a Plugin SDK/runtime argv mapping is a maintainer product-boundary decision; the open docs PR is only a mitigation until merged or replaced.
