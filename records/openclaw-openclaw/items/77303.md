## Summary

Keep open: current main still carries an OpenClaw thinking level into CLI execution, but the Claude CLI argv path never consumes it, so the backend cannot honor the existing `/think` setting.

## What This Changes

The issue requests wiring OpenClaw's existing thinking level into the bundled Anthropic `claude-cli` backend, or clearly marking that status control unsupported for that runtime.

## Best Possible Solution

Add backend-owned effort mapping for `claude-cli` runs that respects the existing thinking profile, covers fresh, resume, and live sessions, and keeps `/reasoning` as the separate visibility control.

## Reproduction Assessment

Yes, by source inspection: choose `claude-cli`, set a non-off `/think` level, and the CLI run still builds argv from backend args that contain no `--effort`. I did not perform a live Claude CLI run because no `claude` binary is present in this environment.

## Solution Assessment

Yes, the narrow maintainable fix is to consume the existing `thinkLevel` in the Claude CLI backend path rather than adding a new user-facing config knob. Any `minimal` or `adaptive` mapping should follow the existing Anthropic thinking profile and supported Claude effort levels.
