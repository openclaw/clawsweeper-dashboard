## Summary

Keep open: current main still carries the selected thinking level into CLI run params, but the Anthropic `claude-cli` argv path never maps it to `--effort`. The open implementation PR #77410 uses closing syntax for this issue, so the issue should remain open until that PR lands or is replaced.

## What This Changes

The issue asks OpenClaw to either pass the selected `/think` level to Anthropic's `claude` CLI as `--effort` or stop presenting the Think/Reasoning status controls as effective in `claude-cli` mode.

## Best Possible Solution

Review and land #77410 or an equivalent plugin-owned argv rewrite seam that maps non-off Claude CLI thinking levels to `--effort` for fresh and resumed CLI runs, with docs clarifying `/think` controls budget and `/reasoning` controls visibility.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: use `claude-cli` with a non-off `/think` level, and current main still builds the spawned argv from backend args that contain no `--effort`.

## Solution Assessment

Yes. A plugin-owned per-request argv rewrite is the narrow maintainable fix; the docs-only #77370 path would reduce confusion but would leave the existing `/think` control ineffective for Claude CLI users.
