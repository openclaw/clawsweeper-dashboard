## Summary

Keep open: current main still has the unbounded gateway bind probe, and this PR targets the right path for the linked Big Sur startup hang, but an external contributor still needs to provide after-fix real behavior proof before merge.

## What This Changes

The PR adds a 3-second timeout around the temporary `canBindToHost` TCP bind probe so unresolved IPv6 loopback probes fall back to IPv4-only gateway listeners.

## Best Possible Solution

Land a rebased narrow gateway-net timeout fix after acceptable macOS 11 after-fix proof, preferably with focused timeout coverage and the settled/cleanup shape from the prior closed implementation considered.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not live: I did not reproduce macOS 11 kernel/socket behavior, but source inspection shows gateway startup awaits `canBindToHost("::1")` with no timeout if neither socket event fires.

## Solution Assessment

Is this the best way to solve the issue? Yes, directionally: bounding the temporary bind probe is the narrowest maintainable fallback for a never-settling probe, while https://github.com/openclaw/openclaw/pull/27983 is useful context for adding a settled guard and test coverage.
