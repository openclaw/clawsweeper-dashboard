## What This Changes

The PR modifies `src/gateway/net.ts` to add a default 3000 ms timeout to `canBindToHost` so a stalled bind probe resolves `false` and gateway startup can continue IPv4-only.

## Best Possible Solution

Reapply or rebase the timeout onto current main, add focused stalled-probe coverage, and land it after redacted macOS 11 startup proof or an explicit maintainer proof override.

## Reproduction Assessment

Yes, source-level: if the IPv6 bind probe never emits `listening` or `error`, current main's helper promise never settles and gateway startup waits in `resolveGatewayListenHosts`. I did not live-reproduce the macOS Big Sur kernel behavior here.

## Solution Assessment

Yes for the fix direction, but not for the current landing state. A bounded generic bind-probe timeout is narrower than a macOS-only skip because it preserves working IPv6 loopback binding, but the branch still needs rebase, proof, and focused coverage.
