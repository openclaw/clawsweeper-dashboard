## Summary

Keep open: current main and the latest release still lack the bind-probe timeout, and this external PR is a plausible active fix for the linked macOS Big Sur gateway startup hang, but it is not merge-ready because real behavior proof is missing and GitHub reports conflicts.

## What This Changes

The branch modifies `src/gateway/net.ts` to add a 3000 ms fallback timeout to the bind probe used by gateway listen-host resolution.

## Best Possible Solution

Land a rebased timeout fix in `src/gateway/net.ts` with a focused stalled-probe regression test and redacted macOS 11 startup proof, or replace this branch with an equivalent narrow PR that credits the contributor.

## Reproduction Assessment

Yes, source-reproducible: if the IPv6 `listen(0, "::1")` probe never emits `listening` or `error`, current `canBindToHost` never settles and startup waits at `resolveGatewayListenHosts`. I did not live-reproduce the macOS Big Sur kernel behavior in this review environment.

## Solution Assessment

Yes for the direction, but not for the current landing state. A bounded generic bind-probe timeout is narrower than a platform-specific skip because it preserves IPv6 loopback when the probe succeeds, but this branch still needs rebase, proof, and focused coverage.
