## Summary

Keep open: current main and the latest release still lack a timeout for the gateway bind probe, and this PR is the active candidate for the linked startup-hang issue, but it is not merge-ready without real macOS 11 proof and a rebase.

## What This Changes

The PR adds a 3000ms timeout to gateway `canBindToHost` so stalled bind probes resolve `false` and loopback startup can continue IPv4-only.

## Best Possible Solution

Land a rebased timeout fix in `src/gateway/net.ts` with focused stalled-probe coverage and redacted macOS 11 startup proof, or replace this branch with an equivalent narrow PR that credits the source.

## Reproduction Assessment

Yes at source level: if Node's `listen(0, "::1")` never emits `listening` or `error`, current `canBindToHost` never settles and startup waits at `resolveGatewayListenHosts`. I did not live-reproduce macOS 11 in this review.

## Solution Assessment

Yes for the direction, but this branch is not the best landing state until rebased and proved. A bounded generic bind-probe timeout is narrower than a platform-specific macOS skip because it preserves IPv6 loopback when the probe succeeds.
