## Summary

Keep open: the gateway bind-probe timeout is still missing from current main and v2026.6.8, and the linked issue remains valid, but this external PR is not merge-ready while it conflicts with main and lacks real macOS 11 after-fix proof.

## What This Changes

The branch updates `src/gateway/net.ts`, most importantly adding a 3000 ms timeout to `canBindToHost` so a stalled IPv6 loopback probe resolves false instead of blocking startup.

## Best Possible Solution

Land a rebased timeout fix in `src/gateway/net.ts` with focused stalled-probe coverage and redacted macOS 11 startup proof, or replace this branch with an equivalent narrow PR that credits the contributor.

## Reproduction Assessment

Yes, source-reproducible: if the IPv6 `listen(0, "::1")` probe never emits `listening` or `error`, current `canBindToHost` never settles and startup waits at `resolveGatewayListenHosts`. I did not live-reproduce the macOS Big Sur kernel behavior here.

## Solution Assessment

Yes for the direction, but this branch is not the best landing state until rebased, proved, and covered. A bounded generic bind-probe timeout is narrower than a platform-specific skip because it preserves IPv6 loopback when the probe succeeds.
