## What This Changes

The PR adds a 3-second timeout to the gateway `canBindToHost` probe so a never-settling `::1` bind falls back to IPv4-only listen hosts.

## Best Possible Solution

Land a rebased narrow timeout fix in the gateway net helper after redacted macOS 11 proof is added, ideally with focused coverage for a never-settling bind probe.

## Solution Assessment

Yes, with proof still needed: a bounded timeout on the temporary bind probe is the narrowest maintainable fallback for a never-settling probe, and it preserves existing IPv6 behavior when the probe succeeds.
