## Summary

Keep open: current main still has an unbounded non-Windows IPv6 loopback bind probe, and this PR is a plausible narrow fix but is still missing real macOS 11 after-fix proof and is currently merge-conflicting.

## What This Changes

The PR branch adds a 3-second timeout to `canBindToHost` in `src/gateway/net.ts` so an unresolved `::1` probe returns `false` and gateway startup can continue IPv4-only.

## Best Possible Solution

Land a rebased narrow timeout fix in the gateway net helper, ideally with focused timeout coverage, after the contributor supplies redacted macOS 11 after-fix proof or a maintainer explicitly overrides that proof gate.

## Reproduction Assessment

Unclear for live reproduction: source inspection of current main shows gateway startup awaits `resolveGatewayListenHosts(...)`, which awaits `canBindToHost("::1")` without a timeout on non-Windows. I did not reproduce the macOS 11 kernel/event behavior.

## Solution Assessment

Yes, with proof and rebase still needed: a bounded timeout around the temporary bind probe is a narrow maintainable fallback that preserves normal IPv6 loopback behavior when the probe succeeds. A focused timeout test or equivalent regression guard would make the patch easier to land confidently.
