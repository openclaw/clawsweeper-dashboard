## What This Changes

## Best Possible Solution

Land a narrow, rebased gateway bind-probe timeout with focused stalled-probe coverage and redacted macOS 11 startup proof.

## Reproduction Assessment

Yes, source-reproducible: if the IPv6 `listen(0, "::1")` probe never emits `listening` or `error`, current `canBindToHost` never settles and startup waits at `resolveGatewayListenHosts`. I did not live-reproduce the macOS Big Sur kernel behavior in this review environment.

## Solution Assessment

Yes for the direction, but not for the current landing state. A bounded generic bind-probe timeout is narrower than a platform-specific skip because it preserves IPv6 loopback when the probe succeeds, but this branch still needs rebase, proof, and focused coverage.
