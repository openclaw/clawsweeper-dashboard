## Summary

Keep open: current main and v2026.6.9 still have the unbounded IPv6 loopback bind probe, but this PR is not merge-ready because it lacks after-fix macOS 11 startup proof and is currently conflicting against main.

## What This Changes

The PR changes `src/gateway/net.ts` to add a 3000 ms false fallback when `canBindToHost` neither reaches `listening` nor emits `error`, while carrying older gateway bind-helper edits from its stale base.

## Best Possible Solution

Land a narrow, rebased Gateway bind-probe timeout with stalled-probe regression coverage and redacted macOS 11 startup proof.

## Reproduction Assessment

Yes, source-reproducible: if the IPv6 `listen(0, "::1")` probe never emits `listening` or `error`, current `canBindToHost` never settles and startup waits in `resolveGatewayListenHosts`. I did not live-reproduce the macOS Big Sur kernel behavior in this review environment.

## Solution Assessment

Yes for the fix direction, but not for the current landing state. A bounded generic bind-probe timeout is narrower than a platform-specific skip because it preserves IPv6 loopback when the probe succeeds, but this branch still needs rebase, proof, and focused coverage.
