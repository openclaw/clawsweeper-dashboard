## What This Changes

The PR adds a 3000 ms fallback timeout to the gateway bind probe in `src/gateway/net.ts` so stalled `listen(0, "::1")` probes resolve false instead of blocking startup.

## Best Possible Solution

Land a rebased timeout fix in `src/gateway/net.ts` with focused stalled-probe coverage and redacted macOS 11 startup proof, or replace this branch with an equivalent narrow PR that credits the source.

## Reproduction Assessment

Yes, source-reproducible: if Node's `listen(0, "::1")` never emits `listening` or `error`, current `canBindToHost` never settles and startup waits at `resolveGatewayListenHosts`. I did not live-reproduce the macOS Big Sur kernel behavior in this review.

## Solution Assessment

Yes for the fix direction, but not for the current landing state. A bounded generic bind-probe timeout is narrower than a platform-specific skip because it preserves IPv6 loopback when the probe succeeds, but the branch still needs rebase/proof and preferably focused timeout coverage.
