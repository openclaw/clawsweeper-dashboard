## What This Changes

The PR adds a fixed 3-second timeout to `canBindToHost` so stalled bind probes resolve false and loopback listen-host selection can continue IPv4-only.

## Best Possible Solution

Land a rebased narrow timeout fix with contributor-supplied macOS 11 startup proof and focused stalled-probe coverage, or replace it with an equivalent robust helper if maintainers prefer the prior cleanup-and-settled-guard shape.

## Reproduction Assessment

## Solution Assessment

Yes for the direction, but this branch is not merge-ready. A bounded timeout in `canBindToHost` is a narrow mitigation that preserves successful IPv6 probes, but it needs real macOS proof, a clean rebase, and preferably focused stalled-probe coverage.
