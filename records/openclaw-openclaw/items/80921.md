## What This Changes

The branch adds a 3000 ms timeout to gateway `canBindToHost` in `src/gateway/net.ts` so a stalled bind probe resolves `false` instead of hanging startup.

## Best Possible Solution

Land a rebased timeout fix in `src/gateway/net.ts` with focused stalled-probe coverage and redacted macOS 11 startup proof, or replace this branch with an equivalent narrow PR that credits the source.

## Reproduction Assessment

Yes at source level: if the IPv6 bind probe never emits `listening` or `error`, current main's helper promise never settles and gateway startup awaits `resolveGatewayListenHosts`. I did not live-reproduce the macOS 11 kernel behavior in this review.

## Solution Assessment

Yes for the direction, but this branch is not the best landing state until rebased and proved. A bounded generic bind-probe timeout is narrower than a platform-specific macOS skip because it preserves IPv6 loopback when the probe succeeds.
