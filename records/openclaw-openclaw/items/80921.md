## What This Changes

The branch changes `src/gateway/net.ts` so the temporary `canBindToHost` TCP server resolves `false` after 3 seconds when neither `listening` nor `error` fires.

## Best Possible Solution

Refresh the branch onto current main, keep the timeout fix narrow in `canBindToHost`, and require redacted macOS 11 gateway startup proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main awaits a bind-probe promise that only settles on `listening` or `error`, so a socket path that emits neither event can hang startup. I did not live-reproduce the macOS Big Sur kernel behavior in this environment.

## Solution Assessment

Yes for the fix direction: a bounded timeout in the existing bind probe is the narrowest maintainable mitigation because it preserves working IPv6 loopback binding while preventing an indefinite await. The submitted branch still needs conflict resolution and real behavior proof before merge.
