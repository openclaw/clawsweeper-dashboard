## Summary

Keep open: the follow-up widens the shared-client option type, but the default startup factory still cannot carry scoped auth stores, so the new shared refresh handler can preempt the run-specific scoped handler and refresh with missing or wrong credentials.

## What This Changes

The branch adds a shared Codex app-server ChatGPT OAuth refresh handler, forwards run-attempt scoped stores to direct refresh, widens shared-client options, and adds regression tests.

## Best Possible Solution

Land the shared refresh handler only after scoped `authProfileStore` is carried through shared and leased startup paths with focused regression coverage, while leaving broader OAuth timeout policy to the existing auth-refresh issues.

## Reproduction Assessment

Yes from source inspection, though I did not run a live Telegram repro in this read-only pass. Current main shared clients lack a handled ChatGPT refresh response for the bound-conversation path, and upstream Codex requires a shaped `accessToken` response.

## Solution Assessment

No, not yet. The shared refresh handler is the right layer for bound Telegram refresh, but the fix is incomplete until the shared startup factory carries scoped auth stores or the handler cannot preempt the scoped run-attempt handler.
