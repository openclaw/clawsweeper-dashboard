## Summary

Keep open for normal PR review/merge. The PR addresses a real Codex shared-client auth gap: current main only registers the ChatGPT OAuth refresh handler on isolated app-server clients, while bound Telegram Codex turns reuse shared clients that can receive the same upstream refresh request.

## What This Changes

## Best Possible Solution

Keep the repair in the Codex plugin shared-client/auth bridge layer and land it once maintainers accept the auth-provider risk and proof level.

## Reproduction Assessment

Yes. Source inspection shows current main shared clients lack the refresh request handler while upstream Codex can emit account/chatgptAuthTokens/refresh on 401, and the PR discussion reports the same failure in a live Telegram /codex bind flow.

## Solution Assessment

Yes. Mirroring the isolated-client refresh handler onto shared clients, forwarding scoped auth stores, and keying shared clients by scoped store is the narrow owner-layer fix; patching Telegram transport would leave the shared Codex client contract broken.
