## Summary

Keep this PR open: current main still has the nested-skills-root bug, and the PR is pointed at the right skills loader boundary, but the current diff should not merge as-is because the new root probe misses documented grouped skills and bypasses the existing directory scan cap.

## What This Changes

The PR adds a skills-root validity probe in workspace skill discovery, passes the skill file size limit into that probe, and adds a regression test for a configured skills root containing a direct skill alongside a nested `skills/` decoy.

## Best Possible Solution

Revise this PR so nested-root detection first proves the configured directory already has at least one loadable direct or one-level grouped skill using the same containment, size, and scan-limit rules as normal discovery, then preserve repo-style `repo/skills/*/SKILL.md` compatibility and document the user-visible fix in the changelog.

## Reproduction Assessment

Yes. Current main can be reproduced with a configured skills root containing `my-skill/SKILL.md` plus `skills/decoy/SKILL.md`, which redirects to `dir/skills` before root discovery; the PR still leaves a grouped variant such as `group/my-skill/SKILL.md` plus `skills/decoy/SKILL.md` uncovered.

## Solution Assessment

No. The direction is right, but the current patch is not the best final fix because the root probe does not mirror documented grouped discovery and performs an uncapped directory scan before the nested-root check.
