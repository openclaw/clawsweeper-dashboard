The PR adds a pre-redirect root-skill probe in skills discovery and a regression test for a configured skills root that also contains a nested `skills/` directory.

## Best Possible Solution

Land a rebased patch that validates a configured root as loadable before repo-style nested redirect, using current direct and grouped discovery semantics while preserving containment checks, size caps, and regression coverage.

## Reproduction Assessment

Yes. Current main is source-reproducible: a configured root containing both `my-skill/SKILL.md` and `skills/decoy/SKILL.md` can redirect to `dir/skills` before root-level skills are enumerated.

## Solution Assessment

No, not yet. The PR is in the right narrow area, but it must be rebased onto current scan helper semantics and should recognize current main's grouped root-skill layout before redirecting.
