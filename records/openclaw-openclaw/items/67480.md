## Summary

Keep open. Current main still has the false nested-root redirect path, and this external PR is in the right area, but the branch is not merge-ready against current skills discovery semantics and still lacks after-fix real behavior proof.

## What This Changes

The PR adds a pre-redirect root-skill probe in skills discovery and a regression test for a configured skills root that also contains a nested `skills/` directory.

## Best Possible Solution

Land a reviewed version that validates the configured directory as a loadable direct or grouped skills root before applying the repo-style nested redirect, while preserving containment checks, size caps, scan limits, and regression coverage.

## Reproduction Assessment

Yes. Source inspection shows a configured root containing both `my-skill/SKILL.md` and `skills/decoy/SKILL.md` can redirect to `dir/skills` before root-level skills are enumerated.

## Solution Assessment

No, not yet. The PR targets the right bug, but the root probe must be rebased to current scan helper semantics and should recognize grouped root skills before suppressing the nested redirect.

## Review Findings
