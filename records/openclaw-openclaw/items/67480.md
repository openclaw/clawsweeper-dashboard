## Summary

Keep open. The underlying skills-root redirect bug is still source-reproducible on current main, and this PR is aimed at the right code path, but the branch needs current-main revision and contributor real behavior proof before merge.

## What This Changes

The PR adds a pre-redirect root-skill probe in skills discovery and a snapshot regression test for a configured skills root that also contains a nested `skills/` subdirectory.

## Best Possible Solution

Land a rebased patch that recognizes a configured directory as a loadable root before repo-style nested redirect, including direct and one-level grouped skills, with focused regression coverage.

## Reproduction Assessment

Yes. Current main is source-reproducible: `resolveNestedSkillsRoot()` can redirect a configured root to `dir/skills` before `loadSkillEntries()` enumerates root-level or grouped sibling skills.

## Solution Assessment

No, not yet. The fix belongs in this narrow skills discovery path, but this branch must be adapted to current scan-helper semantics and grouped-root behavior before it is the best merge candidate.
