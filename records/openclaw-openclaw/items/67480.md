## Summary

Keep open. This is a valid narrow skills-loader bug and the PR is useful, but the proposed branch still needs review changes and external after-fix behavior proof before merge.

## What This Changes

The PR adds an early root-validity probe to skills discovery so a configured skills root is not redirected to a nested `skills` directory when root-level skills are present, plus one regression test.

## Best Possible Solution

Land a reviewed version that proves a configured root has a loadable direct or one-level grouped skill before applying the repo-style nested-root heuristic, using the same containment, size, and scan-cap rules as normal discovery.

## Reproduction Assessment

Yes, source-reproducible with high confidence. Current main redirects a configured root to `dir/skills` before checking root-level skills, and the same path can be reasoned from `resolveNestedSkillsRoot()` plus `loadSkillEntries()`.

## Solution Assessment

No. Validating the configured root before redirecting is the right direction, but the current PR probe is direct-only and bypasses the existing directory scan cap.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Apply the scan cap before probing root skills:** `src/agents/skills/workspace.ts:315`
  - body: The new root probe calls `listChildDirectories(dir)` without `maxCandidateDirs`, so the helper uses an unbounded raw scan before the later slice. Large configured roots now get fully walked during every nested-root check; pass the existing scan cap into this probe or share the capped discovery helper.
  - confidence: 0.86
- **[P2] Probe grouped root skills before redirecting:** `src/agents/skills/workspace.ts:320-324`
  - body: Configured roots support one grouping level, but this early-exit only recognizes direct `dir/<child>/SKILL.md` skills. A root with `group/my-skill/SKILL.md` and `skills/decoy/SKILL.md` still redirects to `dir/skills` and drops the grouped skill, leaving the same bug class for a documented layout.
  - confidence: 0.84
