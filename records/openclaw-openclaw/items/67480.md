## Summary

Keep open for maintainer review: the reported skills-root bug is still source-reproducible on current main, and this is a focused external fix, but the patch needs current-main adaptation and contributor real behavior proof before merge.

## What This Changes

The PR adds an early root-validity probe to skills discovery so a configured skills root with direct `dir/*/SKILL.md` entries is not redirected to `dir/skills`, plus a regression test.

## Best Possible Solution

Land a reviewed version that validates a configured root as loadable before nested redirect while reusing the normal capped direct/grouped discovery, containment, and `SKILL.md` size rules.

## Reproduction Assessment

Yes. Current main is source-reproducible: `resolveNestedSkillsRoot()` can redirect a configured root to `dir/skills` before `loadSkillEntries()` enumerates root-level skills.

## Solution Assessment

No, not yet. The direction is right, but the patch should share current main's capped discovery behavior and recognize grouped root skills before applying the repo-style nested-root heuristic.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Cap the root probe before scanning children:** `src/agents/skills/workspace.ts:314`
  - body: The new probe calls `listChildDirectories(dir)` without passing the scan cap, so current main's raw directory limit is bypassed before the later slice. Large configured roots can be walked during every nested-root check; pass the existing candidate cap into this probe or share the capped discovery helper.
  - confidence: 0.86
- **[P2] Recognize grouped root skills before redirecting:** `src/agents/skills/workspace.ts:319-327`
  - body: Current main supports `root/<group>/<skill>/SKILL.md`, but this early-exit only checks direct `root/<child>/SKILL.md` entries. A configured root with only grouped skills plus a `skills/*/SKILL.md` child still redirects and hides the grouped skill, leaving the same bug class for a supported layout.
  - confidence: 0.82
