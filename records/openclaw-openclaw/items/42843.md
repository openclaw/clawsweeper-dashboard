## Summary

Keep open: current main still has the WhatsApp reconnect monitor stop after finite max attempts, so the user problem is not implemented. The PR is useful source work, but it is stale against the current plugin layout and needs a port plus config-help/logging cleanup before it can merge.

## What This Changes

The PR changes WhatsApp Web reconnect defaults to unlimited retries and replaces max-attempt shutdown with a degraded slow-retry loop, with reconnect tests updated for that behavior.

## Best Possible Solution

Port the fix into the current WhatsApp plugin reconnect/controller path, preserve terminal logout/conflict stops, align config help/docs with the retry semantics, and add focused regression coverage plus a changelog entry.
