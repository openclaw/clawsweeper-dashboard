## Summary

Keep open: the WhatsApp reliability problem is still source-reproducible on current main, but this branch is not mergeable as submitted because it edits obsolete `src/web` paths instead of the current `extensions/whatsapp` plugin path and leaves config/logging contract gaps.

## What This Changes

The PR changes WhatsApp Web reconnect behavior from finite max-attempt shutdown to unlimited/default degraded retries and updates reconnect tests.

## Best Possible Solution

Port the useful reconnect policy into `extensions/whatsapp` controller/monitor code, preserve terminal logout/conflict stops, align config help/docs with `0 = unlimited`, and add focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes from source inspection: current main defaults to 12 reconnect attempts, returns a stop decision when that positive cap is reached, and the monitor then shuts down and breaks. I did not execute tests because this review is read-only.

## Solution Assessment

No as submitted: the reliability direction is reasonable, but the patch targets obsolete files and conflicts with current config help/logging. The maintainable fix is a current-plugin port that treats https://github.com/openclaw/openclaw/pull/47433 as the separate active-listener root-cause fix.
