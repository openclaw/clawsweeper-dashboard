## Summary

Keep this PR open, but it still needs changes before merge. The reconnect reliability problem remains visible on current main, but the branch edits obsolete `src/web` files while the live WhatsApp implementation is now under `extensions/whatsapp/src`, so the proposed patch does not update the current runtime path.

## What This Changes

The PR branch changes the old WhatsApp Web reconnect helper and monitor tests under `src/web` to default `maxAttempts` to `0` and use degraded cooldown retries instead of stopping after the retry cap.

## Best Possible Solution

Ship the reconnect reliability change in the WhatsApp plugin: make the default retry policy long-lived, enter an explicit degraded retry cooldown after bounded caps, preserve terminal auth/conflict exits, and align plugin tests plus config help around the final semantics.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: with default or positive `web.reconnect.maxAttempts`, retryable closes eventually make `consumeReconnectAttempt()` return `stop`, and the monitor then shuts down and breaks.

## Solution Assessment

No. The behavioral direction is useful, but the branch edits obsolete `src/web` files; the maintainable fix is to refresh the logic into `extensions/whatsapp/src` and update the plugin tests/config docs there.
