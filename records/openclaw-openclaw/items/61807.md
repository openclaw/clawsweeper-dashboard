## Summary

Keep this PR open: the Feishu thumbnail gap is still present on current main, but this patch should not merge as-is because it adds a bare `ffmpeg` child-process path without trusted resolution or a timeout, and the contributor has not provided real after-fix Feishu proof.

## What This Changes

The branch modifies `extensions/feishu/src/media.ts` to extract a first-frame JPEG for Feishu video sends, upload it as an image, and include the resulting `image_key` in `msg_type: "media"` payloads.

## Best Possible Solution

Rework the Feishu thumbnail extraction through a trusted, timeout-bounded ffmpeg path, preserve best-effort fallback to sending the video, add focused thumbnail tests, and require redacted real Feishu proof before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: send an MP4 through `sendMediaFeishu`, observe video routing to `msg_type: "media"`, and then `sendFileFeishu` builds content with only `file_key`. I did not run a live Feishu account flow in this read-only review.

## Solution Assessment

No as-is. The owner-local thumbnail direction is narrow, but the implementation should use or extend the shared trusted, timeout-bounded ffmpeg runtime instead of adding a new bare child-process path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86
