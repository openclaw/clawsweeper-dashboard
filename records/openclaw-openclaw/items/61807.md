## Summary

Keep open: the PR targets a real Feishu video media gap, but the current head still has blocking implementation issues and no after-fix live Feishu proof.

## What This Changes

The PR changes Feishu video sending to extract a JPEG thumbnail with ffmpeg, upload it as an image, and include the returned `image_key` in native media payloads.

## Best Possible Solution

Implement thumbnail extraction through the plugin SDK media-runtime surface or a focused SDK helper, bound ffmpeg output, add success/fallback coverage, then merge only after redacted live Feishu proof shows a native video cover.

## Reproduction Assessment

Yes from source inspection: current main routes MP4/video to `msg_type: "media"` but serializes only `file_key` in the message content. I did not run a live Feishu tenant send in this review.

## Solution Assessment

No as-is. Owner-local thumbnail generation is the right direction, but this implementation should stay inside the plugin SDK boundary and use bounded ffmpeg output before live proof is accepted.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use the plugin SDK ffmpeg surface:** `extensions/feishu/src/media.ts:8-9`
  - body: `extensions/feishu/src/media.ts` is production plugin code, but these imports reach into `../../../src/...`. The extension boundary checks require bundled plugins to use `openclaw/plugin-sdk/*` instead of core internals, so this will fail the boundary contract and weakens the package/runtime split; route the helper through `openclaw/plugin-sdk/media-runtime` or add a focused SDK export.
  - confidence: 0.94
- **[P2] Bound ffmpeg thumbnail stdout:** `extensions/feishu/src/media.ts:57-58`
  - body: `extractVideoCover` appends every stdout chunk until the child closes. A large or malformed video can make this best-effort thumbnail step buffer an oversized frame and bypass the existing ffmpeg `maxBuffer` limit used by the shared helper, so cap the output or write through a bounded/temp-file helper before uploading.
  - confidence: 0.83
