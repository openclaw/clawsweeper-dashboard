## Summary

Keep this PR open: the requested Feishu thumbnail behavior is still not on current main, but the current patch needs real Feishu proof and a safer ffmpeg execution path before merge.

## What This Changes

The PR adds Feishu video thumbnail extraction with ffmpeg and sends the resulting image key in native media messages.

## Best Possible Solution

Rework thumbnail extraction through the shared trusted, bounded ffmpeg runtime, add Feishu media tests for `image_key` and fallback behavior, then require redacted real Feishu proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send an MP4 through `sendMediaFeishu`, observe routing to `msg_type: "media"`, and then `sendFileFeishu` builds content with only `file_key`; I did not run a live Feishu account flow.

## Solution Assessment

No as-is. The owner-local thumbnail direction is reasonable, but the implementation should use or extend `runFfmpeg` and trusted binary resolution before it is mergeable.
