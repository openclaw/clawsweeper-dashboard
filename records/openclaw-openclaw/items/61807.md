## Summary

Keep open. Current main still sends Feishu video media with only `file_key`, so the thumbnail change is not obsolete, but this patch should not merge until the ffmpeg path is trusted/bounded and the contributor provides real Feishu proof.

## What This Changes

The PR changes Feishu media sending to extract a first-frame JPEG with ffmpeg, upload it as an image, and include the returned `image_key` in native video media payloads.

## Best Possible Solution

Implement the thumbnail path inside the Feishu plugin using a trusted, timeout-bounded ffmpeg helper, add focused success/fallback tests, and require redacted live Feishu proof before merge.

## Reproduction Assessment

Yes from source inspection, though not from a live Feishu account in this review. Current main routes MP4/video through `sendMediaFeishu` to `msg_type: "media"`, then `sendFileFeishu` serializes only `{ file_key }` with no thumbnail `image_key` path.

## Solution Assessment

No as-is. Owner-local thumbnail generation is the right boundary, but the implementation should reuse or extend the shared trusted/bounded ffmpeg runtime and add focused tests before live proof is accepted.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86
