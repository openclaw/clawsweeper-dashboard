## Summary

Keep open. Current main still lacks Feishu video cover `image_key` handling, and this PR is a useful Feishu fix, but the branch is dirty/unmergeable and the patch introduces an unbounded PATH-resolved `ffmpeg` process without the shared trusted-bin safeguards.

## What This Changes

The PR modifies `extensions/feishu/src/media.ts` to extract a JPEG cover from outbound videos with `ffmpeg`, upload it as a Feishu image, and include the resulting `image_key` in `media` message content.

## Best Possible Solution

Land a rebased Feishu-owned patch that preserves the useful thumbnail behavior, routes extraction through trusted timeout-bounded ffmpeg handling, adds focused Feishu media tests for success and fallbacks, and includes the required changelog entry.

## Reproduction Assessment

Yes. A static current-main reproduction is enough: send an mp4 through `sendMediaFeishu`, observe routing to `msgType: "media"`, then `sendFileFeishu` builds content with only `file_key` while the PR discussion and Feishu media contract identify `image_key` as the cover field.

## Solution Assessment

No as-is. The owner-local direction is right, but the implementation should use or extend the shared media runtime for trusted, timeout-bounded ffmpeg execution before merge.

## Security Review

Status: needs_attention
