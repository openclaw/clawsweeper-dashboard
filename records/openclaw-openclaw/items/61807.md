## Summary

Keep open. Current main still lacks outbound Feishu video `image_key` support, and this PR targets the right Feishu-owned surface, but the current head is dirty and the patch introduces untrusted, unbounded `ffmpeg` execution plus missing test and changelog coverage.

## What This Changes

The PR adds Feishu outbound video thumbnail extraction with `ffmpeg`, uploads the JPEG cover as a Feishu image, and includes the resulting `image_key` in `media` message content.

## Best Possible Solution

Land a rebased Feishu-owned patch that preserves thumbnail behavior, routes extraction through trusted timeout-bounded ffmpeg handling, adds focused Feishu media tests, and includes the required changelog entry.

## Reproduction Assessment

Yes. A static current-main reproduction is enough: send an mp4 through `sendMediaFeishu`, observe routing to `msgType: "media"`, then `sendFileFeishu` builds content with only `file_key` while the PR discussion identifies Feishu's `image_key` cover field.

## Solution Assessment

No as-is. The owner-local direction is right, but the implementation should use or extend the shared media runtime for trusted, timeout-bounded ffmpeg execution before merge.

## Security Review

Status: needs_attention

Summary: The diff introduces a new child-process execution path for `ffmpeg` that resolves through PATH and has no timeout.
