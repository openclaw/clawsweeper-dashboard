## Summary

Keep open. Current main still has the Windows duplicate-output path, but this PR should not merge as-is because it can make completed Windows restarts silent, is stale against the fallback-aware scheduled-task launcher on main, and mixes browser CDP SSRF policy work into a gateway output fix.

## What This Changes

The PR removes Windows schtasks/startup-fallback restart stdout writes, factors scheduled-task run helpers/lowercase normalization, and adds browser profile CDP SSRF validation behavior and tests.

## Best Possible Solution

Centralize Windows gateway restart success messaging so schtasks and startup-fallback completed restarts emit exactly one human success line after the final retry/check path, while keeping browser CDP SSRF policy work separate.

## Reproduction Assessment

Yes. Source inspection shows schtasks/startup-fallback restart paths write to stdout, and the stale-PID retry path invokes the restart implementation again; lifecycle tests also cover that retry shape.

## Solution Assessment

No. Deleting the service-layer writes alone is not the narrowest maintainable fix because completed restarts are not centrally printed today, and the browser SSRF changes should not ride along with the gateway output fix.
