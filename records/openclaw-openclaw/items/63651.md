## What This Changes

The PR removes Windows schtasks/startup-fallback restart stdout success writes and also changes browser CDP SSRF profile validation and tests.

## Best Possible Solution

Land a narrow gateway-only fix that emits exactly one completed Windows restart success line after retry/health handling, with browser CDP policy work split into its own reviewed PR.

## Reproduction Assessment

Yes, by source inspection: current main writes Windows restart success inside restartScheduledTask/restartStartupEntry, and stale-PID recovery can call service.restart a second time. I did not run a live Windows schtasks command in this read-only review.

## Solution Assessment

No. Removing both service-layer writes without adding one central non-JSON completed-restart message would make normal successful Windows restarts silent, and the browser CDP SSRF changes should be split for separate review.
