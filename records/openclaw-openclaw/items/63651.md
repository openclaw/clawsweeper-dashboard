## Summary

Keep open: the duplicate Windows restart output is source-reproducible on current main, but this PR cannot merge as-is because it removes the only normal completed-restart success line, lacks attached real Windows after-fix proof, is currently conflicting, and bundles unrelated browser CDP SSRF behavior.

## What This Changes

The PR removes Windows schtasks/startup-fallback restart stdout success writes, refactors schtasks helpers, normalizes string handling, and bundles browser CDP SSRF validation changes/tests.

## Best Possible Solution

Land a narrow gateway-only fix that emits exactly one completed Windows restart success line after the final retry/health path, and split browser CDP SSRF policy work into its own reviewed PR.

## Reproduction Assessment

Yes. Source inspection shows current main writes Windows restart success inside the service implementation, and stale-PID recovery can call that implementation a second time; I did not run a live Windows schtasks reproduction in this read-only review.

## Solution Assessment

No. Removing both service-layer writes without adding one central completed-restart message makes ordinary successful Windows restarts silent, and the unrelated browser CDP SSRF changes should be split from the gateway fix.
