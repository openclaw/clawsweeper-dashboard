## Summary

Keep open. Current main and v2026.5.19 still have no supported tools-level umask, file-mode, or directory-mode policy for agent-created user-facing output, and the related Write/Edit PR is open, conflicting, and only a partial hard-coded-mode proposal.

## What This Changes

Request a configurable tools-level umask or explicit file/directory modes for agent-created user-facing files such as documents, scripts, exports, and workspace tool output.

## Best Possible Solution

Define a scoped user-output-only permission policy with validated octal config, thread it through host Write/Edit, sandbox bridge/mutation, and relevant tool-output surfaces, and preserve private internal-state defaults unless maintainers intentionally make a separate shared-state policy decision.

## Reproduction Assessment

Not applicable as a feature/config request. Source inspection gives high confidence that current main and v2026.5.19 have no accepted config fields or cross-surface runtime plumbing for the requested user-output permission policy.

## Solution Assessment

Yes directionally, but not as a broad process-wide umask. The safer maintainable solution is a scoped user-output policy that cannot affect OpenClaw-managed secrets or internal state and is tested across host and sandbox write paths.
