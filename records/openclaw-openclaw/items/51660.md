## Summary

Keep open. Current main still has no supported tools-level umask, file-mode, or directory-mode policy for user-facing agent output; the related open Write/Edit PR is only a partial fixed-mode proposal and does not settle the broader configurable permission policy.

## What This Changes

Request a configurable umask or explicit file/directory mode policy for agent-created user-facing files such as Write/Edit, exec output, and tool-generated workspace files.

## Best Possible Solution

Define a maintainer-approved, user-output-only permission policy with validated octal config, schema/help/docs coverage, and explicit host plus sandbox write plumbing while preserving restrictive internal-state writes.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main has no accepted config fields or cross-surface runtime plumbing for the requested user-output permission policy.

## Solution Assessment

Yes directionally, but not as a broad process-wide umask. The safer maintainable solution is a scoped user-output policy that cannot affect OpenClaw-managed secrets or internal state.
