## Summary

Keep open. Current main still lacks a supported tools-level umask, file-mode, or directory-mode policy for agent-created user-facing output, and the related Write/Edit PR is open, partial, and does not settle sandbox, exec-created files, or the security boundary.

## What This Changes

Request a configurable tools permission policy for agent-created user-facing output files and directories, including Write/Edit, exec-created files, and sandbox/tool output.

## Best Possible Solution

Define an opt-in, user-output-only permission policy with validated octal modes, thread it through host Write/Edit, sandbox bridge writes, and exec runtime setup, and keep internal OpenClaw state on its private defaults.

## Reproduction Assessment

Not applicable as a feature/config request. Source inspection gives high confidence that current main has no accepted config fields or cross-surface runtime plumbing for the requested user-output permission policy.

## Solution Assessment

Yes directionally, but not as a broad process-wide umask. The maintainable path is a scoped, opt-in user-output policy that cannot affect OpenClaw-managed secrets or internal state and is tested across host and sandbox write paths.
