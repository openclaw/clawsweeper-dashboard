## Summary

Keep open. Current main still has no supported tools-level umask, file-mode, dir-mode, or filePermissions policy for agent-created user-facing files, and the related open PR only proposes a partial hard-coded Write/Edit default rather than the requested configurable cross-surface behavior.

## What This Changes

The issue requests configurable umask or explicit file/directory modes for user-facing files created by agent tools such as exec, write, edit, and apply_patch.

## Best Possible Solution

Define a scoped user-output permission policy with validated octal inputs, integrate it through config schema and docs, and thread it only through approved user-output write paths while preserving restrictive internal state permissions.

## Reproduction Assessment

Not applicable as a feature request. Static verification is high confidence: current main has no accepted config fields or call-site plumbing for the requested user-output file-permission policy.

## Solution Assessment

Yes directionally, but not as a broad process.umask toggle. The safer maintainable path is an explicit, secops-reviewed user-output permission policy with per-surface integration and internal-file exclusions.
