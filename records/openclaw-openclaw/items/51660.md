## Summary

Keep open. Current main still lacks a supported tools-level umask, file mode, dir mode, or filePermissions policy for agent-created user-facing files, and the requested behavior needs maintainer/secops design before implementation.

## What This Changes

Add configurable file-permission behavior for user-facing files created by agent tools such as exec, write/edit, apply_patch, and sandbox filesystem output.

## Best Possible Solution

Design a scoped, documented user-output file creation policy with validated octal inputs and integrate it through config schema, docs, host write/edit, apply_patch, sandbox bridge writes, and feasible exec output handling while preserving restrictive internal file modes.

## Reproduction Assessment

Not applicable as a feature request. Static verification is high confidence: current main has no accepted config fields or call-site plumbing for the requested file-permission policy.

## Solution Assessment

Yes directionally, but not as a broad process.umask toggle. The safer maintainable solution is an explicit user-output permission policy with secops-reviewed boundaries and per-call-site integration.
