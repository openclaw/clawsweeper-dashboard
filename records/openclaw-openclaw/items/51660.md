## Summary

Keep open. Current main still has no supported tools-level umask, file-mode, or directory-mode policy for agent-created user-facing output, and the related open Write/Edit PR is only a partial hard-coded-mode proposal rather than the requested configurable policy.

## What This Changes

Request a configurable umask or explicit file/directory mode policy for user-facing files created by agent tools while excluding sensitive OpenClaw internal state.

## Best Possible Solution

Define a scoped user-output-only permission policy with validated octal config, then thread it through current host root.write, sandbox bridge/mutation, and relevant tool-output surfaces while preserving private internal-state defaults.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main has no accepted config fields or cross-surface runtime plumbing for the requested user-output permission policy.

## Solution Assessment

Yes directionally, but not as a broad process-wide umask. The safer maintainable solution is a scoped user-output policy that cannot affect OpenClaw-managed secrets or internal state.
