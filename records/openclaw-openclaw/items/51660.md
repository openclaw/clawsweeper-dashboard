## Summary

Keep open. Current main and the latest release still do not provide a supported tools-level umask, file-mode, or directory-mode policy for agent-created user-facing output, and the related open Write/Edit PR is a partial hard-coded-mode proposal rather than the requested configurable cross-surface policy.

## What This Changes

The issue requests configurable filesystem permissions for agent-created user-facing files such as exec, write, edit, and generated tool outputs.

## Best Possible Solution

Define a scoped user-output-only permission policy with validated octal config, then thread it through current host root.write, sandbox bridge/mutation, exec/output write paths, docs, and tests while preserving private defaults for OpenClaw internal state.

## Reproduction Assessment

Not applicable as a feature request. Source and release inspection give high confidence that OpenClaw currently lacks accepted config fields or runtime plumbing for the requested user-output permission policy.

## Solution Assessment

Yes directionally, but not as a broad process-wide umask. The maintainable solution is a scoped user-output policy that cannot affect OpenClaw-managed secrets, credentials, transcripts, or internal state.
