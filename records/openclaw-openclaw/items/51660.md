## Summary

Keep open. Current main still has no supported tools-level umask, file-mode, dir-mode, or filePermissions policy for agent-created user-facing files; the adjacent #56263 remains broader sensitive-state permission policy work, not a replacement for this user-output surface.

## What This Changes

Add a configurable tools-level umask or explicit file/directory modes for agent-created user-facing output files in shared Docker/NAS deployments.

## Best Possible Solution

Design and document a scoped user-output permission policy with validated octal inputs, category boundaries, and per-write-surface integration while preserving restrictive modes for internal state.

## Reproduction Assessment

Not applicable as a feature request. Static source and docs checks show current main has no accepted config fields or write-path plumbing for the requested user-output permission policy.

## Solution Assessment

Yes directionally, but not as a process-wide umask toggle. The safer solution is a scoped user-output policy wired through the relevant tool write paths with explicit exclusions for internal files.
