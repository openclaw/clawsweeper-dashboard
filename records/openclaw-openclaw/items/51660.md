## Summary

Keep open. Current main still has no supported tools-level user-output umask/file-mode policy, the open Write/Edit PR is only a partial hard-coded default, and the broader state-permissions issue covers a different sensitive-file boundary.

## What This Changes

Add configurable umask or explicit file and directory modes for agent-created user-facing files such as Write/Edit output, exec-created files, exports, and scripts.

## Best Possible Solution

Define a scoped user-output permission policy with validated octal inputs, wire it through config/schema/docs and approved write surfaces, and preserve restrictive internal-state permissions.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main has no accepted config fields or cross-surface plumbing for the requested user-output permission policy.

## Solution Assessment

Yes directionally, but not as a broad process-wide umask. The safer maintainable solution is a category-scoped, secops-reviewed user-output policy that avoids internal sensitive state.
