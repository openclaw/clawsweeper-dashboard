## Summary

Keep open: current main and v2026.6.10 still lack an operator-controlled permission policy for agent-created user-facing files, and the remaining work is a security-sensitive config/product decision rather than conservative cleanup.

## What This Changes

Request to add configurable umask or explicit file/directory modes for agent-created user-facing files while keeping private OpenClaw state restrictive.

## Best Possible Solution

Define an opt-in user-output filesystem permission policy at the tools/runtime boundary, preserve restrictive defaults for private OpenClaw state, and document and test host, sandbox, exec, and apply_patch behavior together.

## Reproduction Assessment

Yes, at source level: current main has no tools.fs permission key and does not carry mode options through host, sandbox, apply_patch, or agent-core write paths. I did not run a live Docker/Unraid reproduction in this read-only review.
