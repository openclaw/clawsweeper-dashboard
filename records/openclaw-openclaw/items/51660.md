## Summary

Keep open: current main and the latest tagged release still lack an operator-controlled permission policy for agent-created user-facing files, and implementing one would change a security-sensitive filesystem/config boundary that needs maintainer product review.

## What This Changes

Request a configurable umask or explicit file/directory modes for agent-created user-facing output files across write/edit/exec and related filesystem tools.

## Best Possible Solution

Define an opt-in tools filesystem output permission policy that applies consistently to user-facing outputs while preserving restrictive private-state defaults and documenting the security contract.

## Reproduction Assessment

Yes, at source level: current main has no tools.fs permission key and the host, sandbox, and apply_patch write paths do not carry a configured output mode. I did not run a live Docker/Unraid reproduction in this read-only review.

## Solution Assessment

Unclear as a final implementation: the direction is plausible, but maintainers need to choose the exact config shape and draw a hard boundary between user-facing output and private OpenClaw state.
