## Summary

Keep open. Current main and the latest shipped tag still lack a supported tools-level umask, file-mode, directory-mode, or ownership policy for agent-created user-facing output, and the related Write/Edit PR was closed unmerged as superseded by this broader policy request.

## What This Changes

The issue requests configurable permissions, such as a tools-level umask or file and directory modes, for user-facing files created by agents in shared Docker/NAS workspaces.

## Best Possible Solution

Define a maintainer-approved, opt-in user-output permission policy with validated octal modes, apply it consistently to host and sandbox write surfaces plus exec-created output, and keep internal OpenClaw state on private defaults.

## Reproduction Assessment

Not applicable as a bug reproduction, but source inspection gives high confidence that current main has no accepted config fields or cross-surface runtime plumbing for the requested user-output permission policy.

## Solution Assessment

Yes directionally, but not as a broad process-wide umask. The maintainable path is a scoped, opt-in user-output policy that cannot affect OpenClaw-managed secrets or internal state and is tested across host, sandbox, apply_patch, and exec paths.
