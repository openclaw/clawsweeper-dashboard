## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reset sink state after each reasoning block:** `extensions/telegram/src/reasoning-stream-sink.ts:126`
  - body: Telegram can end one reasoning block and later start another in the same turn. `onEnd()` clears `started` but leaves `lastSnapshot`, so a new block that shares the previous prefix is emitted as only the suffix instead of a fresh block; reset the snapshot when the block ends or start a new per-block lifecycle.
  - confidence: 0.9
- **[P2] Make HTTPS URL validation non-throwing:** `src/config/zod-schema.providers-core.ts:356`
  - body: This refinement calls `new URL(v)` after `.url()`. Zod 4 URL-format issues are non-aborting by default and custom refinement functions are not caught, so a bad value like `not-a-url` can escape config validation as a raw `TypeError`; wrap the parse or use a non-throwing protocol check.
  - confidence: 0.86
- **[P2] Regenerate the Plugin SDK API baseline:** `src/plugin-sdk/config-runtime.ts:119`
  - body: Adding `ReasoningStreamSinkConfig` to the public Plugin SDK config export changes the generated API surface, but the PR leaves `docs/.generated/plugin-sdk-api-baseline.sha256` unchanged. `pnpm plugin-sdk:api:check` will report drift until the baseline is regenerated and committed.
  - confidence: 0.9
