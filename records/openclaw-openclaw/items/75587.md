The PR adds Telegram `channels.telegram.reasoningStreamSink` config, shared config types, docs/changelog, generated config baseline updates, HTTPS/HMAC guarded POST delivery for reasoning lifecycle events, and a small Discord test fixture adjustment.

## Reproduction Assessment

Yes. A direct sink-handle sequence of `onToken('Reasoning:\nPlan')`, `onEnd()`, then `onToken('Reasoning:\nPlan after tool')` reproduces the suffix-only second block; invalid sink URL validation exercises the raw `new URL` exception path.

## Solution Assessment

No. The HTTPS-only guarded egress design is plausible, but the current patch is not the narrow maintainable solution until lifecycle state, validation behavior, and generated SDK API proof are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reset sink state after each reasoning block:** `extensions/telegram/src/reasoning-stream-sink.ts:126`
  - body: Telegram can end one reasoning block and later start another in the same turn. `onEnd()` clears `started` but leaves `lastSnapshot`, so a new block that shares the previous prefix is emitted as only the suffix instead of a fresh block; reset the snapshot when the block ends or start a new per-block lifecycle.
  - confidence: 0.9
- **[P2] Make the HTTPS URL refinement non-throwing:** `src/config/zod-schema.providers-core.ts:347`
  - body: This refinement calls `new URL(v)` after `.url()`. Zod refinements must not throw, so a bad value like `not-a-url` can escape config validation as a raw `TypeError` instead of a normal Zod issue; wrap the parse or use a non-throwing protocol helper.
  - confidence: 0.84
- **[P2] Regenerate the plugin SDK API baseline:** `src/plugin-sdk/config-runtime.ts:119`
  - body: Adding `ReasoningStreamSinkConfig` to the plugin SDK config exports changes the generated SDK API surface, but the PR leaves `docs/.generated/plugin-sdk-api-baseline.sha256` unchanged. `pnpm plugin-sdk:api:check` will report drift until the baseline hash is regenerated and committed.
  - confidence: 0.9
