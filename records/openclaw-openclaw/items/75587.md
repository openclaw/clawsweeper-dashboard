## Summary

Keep open: current main does not implement this feature, and the PR is not merge-ready because it drops replacement assistant updates, accepts an unhandled deprecated Telegram key, and needs maintainer/security approval for the new egress contract.

## What This Changes

The PR adds a top-level `agentEventSink` config and gateway-level event sink that POSTs thinking/reply events to a signed HTTPS webhook, updates SDK/config/docs/changelog surfaces, and removes Telegram-specific sink wiring.

## Best Possible Solution

Land the core sink only after maintainer/security approval, with replacement-aware payload semantics, a clear migration story for the deprecated Telegram config, focused unit tests, and aligned docs/changelog.

## Reproduction Assessment

Yes, for the review blockers: source inspection shows the PR drops empty-delta replacement assistant events and accepts `channels.telegram.reasoningStreamSink` while startup reads only top-level `agentEventSink`. No live webhook test was run under the read-only constraint.

## Solution Assessment

No. The core event-bus direction is plausible, but this implementation loses replacement updates, leaves a deprecated accepted config as a no-op, and still needs explicit approval for raw thinking/reply egress.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Honor assistant replacement events:** `src/infra/agent-event-sink.ts:145-151`
  - body: When embedded Pi finalization emits a replacement update, current main sets `replace: true`, `delta: ""`, and `text` to the corrected full reply. This handler treats the empty delta as no payload and returns, so the webhook keeps the stale streamed text and never receives the correction; include replacement/full-text semantics or handle `replace` before dropping empty deltas.
  - confidence: 0.89
- **[P2] Wire or remove the Telegram sink key:** `src/config/zod-schema.providers-core.ts:351-362`
  - body: The PR still accepts/deprecates `channels.telegram.reasoningStreamSink`, but the gateway only starts the sink from top-level `agentEventSink` and Telegram dispatch no longer reads the account-level key. That makes an accepted config silently do nothing; either bridge it with a deprecation warning or remove the schema/docs/changelog surface.
  - confidence: 0.87
