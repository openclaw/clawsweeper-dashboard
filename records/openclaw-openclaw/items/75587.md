## Summary

Keep this PR open: current main does not implement the requested core event sink, and the latest PR head fixed the earlier stale changelog issue, but the new egress sink is not ready because it logs the configured webhook URL verbatim. The broader top-level thinking/reply egress capability also needs maintainer product/security approval before merge.

## What This Changes

The PR adds a top-level `agentEventSink` config and core AgentEventStream webhook sink for thinking/reply events, wires gateway startup/shutdown, updates config/SDK baselines and Telegram docs, and removes the stale Telegram-specific changelog entry.

## Best Possible Solution

Redact or reject sensitive URL parts in sink logs, add focused coverage for the sink’s security/logging behavior, then have maintainers decide whether a root external thinking/reply egress knob belongs in core.

## Reproduction Assessment

Yes for the review finding: the leak path is source-reproducible from PR head because `agentEventSink.url` accepts HTTPS URLs and the sink logs `config.url` directly. Reproduction is not applicable to the feature request itself because this is a new user-facing runtime/config capability.

## Solution Assessment

No, the PR is not ready as-is: the event-bus sink is a plausible implementation direction, but the raw URL logging should be fixed before merge. The safer narrow repair is to use the existing URL redaction helper, or reject credentials and sensitive query params before logging.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Redact the sink URL before logging it:** `src/infra/agent-event-sink.ts:95-96`
  - body: `agentEventSink.url` accepts normal HTTPS URLs, including userinfo and query parameters, but this warning logs `config.url` verbatim. A configured endpoint like `https://user:pass@example.com/events?token=...` would leak secrets into gateway logs on non-2xx responses and the startup log has the same issue. Please redact the URL before any log output, using the existing sensitive URL redaction helper or by rejecting sensitive URL forms.
  - confidence: 0.89
