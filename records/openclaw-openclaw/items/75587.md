## Summary

Keep open: the PR is not obsolete on current main, but the patch has source-reproducible blockers around thinking-stream coverage, replacement reply semantics, deprecated Telegram config behavior, and the security/privacy review for raw stream egress.

## What This Changes

The branch adds a core `agentEventSink` webhook, top-level config/SDK/docs/changelog surfaces, and gateway lifecycle wiring for streaming thinking and reply events outside channel-specific code.

## Best Possible Solution

Land this only after explicit maintainer/security approval, with source-level thinking emission that is truly channel-agnostic, replacement-aware reply payloads, a clear Telegram config migration story, and focused regression tests/docs.

## Reproduction Assessment

Yes, for the review blockers: source inspection shows the PR subscribes to thinking events that current main only emits when a channel reasoning-stream callback exists, drops replacement assistant events with empty deltas, and accepts a deprecated Telegram key that startup never reads. No live webhook test was run under the read-only constraint.

## Solution Assessment

No. The event-bus direction is plausible, but this implementation does not yet satisfy the claimed channel-agnostic thinking stream, loses replacement corrections, leaves a deprecated accepted config as a no-op, and needs security/product approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Decouple thinking events from channel callbacks:** `src/infra/agent-event-sink.ts:198-202`
  - body: This listener assumes `stream: "thinking"` is a channel-agnostic source, but current `emitReasoningStream` only emits that bus event when reasoning mode is `stream` and a channel `onReasoningStream` callback exists. With only top-level `agentEventSink` configured, Telegram sessions without reasoning drafts and other channels will not send the promised thinking webhook events; move the bus emission out from behind the channel callback gate or install a generic sink callback.
  - confidence: 0.91
- **[P2] Honor assistant replacement events:** `src/infra/agent-event-sink.ts:144-151`
  - body: When finalization emits a replacement update, current main sends `replace: true`, full `text`, and empty `delta`. This handler treats the empty delta as no payload and returns, so the webhook keeps stale streamed text and never receives the correction; handle `replace` before dropping empty deltas or include full-text replacement semantics in the payload.
  - confidence: 0.9
- **[P2] Wire or reject the Telegram sink key:** `src/config/zod-schema.providers-core.ts:351-362`
  - body: The PR still validates deprecated `channels.telegram.reasoningStreamSink`, but gateway startup only reads top-level `agentEventSink` and Telegram dispatch no longer reads the account-level key. That makes accepted config silently do nothing; either bridge it with a deprecation warning or remove the schema/type surface.
  - confidence: 0.88
