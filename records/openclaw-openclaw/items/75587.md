## Summary

Keep open. The latest revision addresses the earlier plaintext-header and snapshot/delta concerns, but the patch still has blocking source-backed issues: it violates the plugin SDK config-runtime boundary and fails open when configured sink credentials cannot be resolved.

## What This Changes

The PR adds a Telegram `reasoningStreamSink` config, sink helper, shared config type/schema/export, and changelog entry to POST reasoning stream lifecycle and delta events to an external HTTP endpoint with optional HMAC signing and secret-backed headers.

## Best Possible Solution

If maintainers want this capability, ship it as an explicit opt-in egress surface that uses the narrow secret-input SDK facade, fails closed on unresolved configured credentials, documents payloads and trust boundaries, and adds focused tests for SecretRef failure, HMAC/header resolution, start/stream/end events, and Telegram streaming mode interactions.

## Reproduction Assessment

Yes. Source inspection is enough: the architecture guard regex matches the added broad `config-runtime` import, and the SecretInput resolver contract returns `unresolvedRefReason` without throwing while the PR discards that status and still creates the sink.

## Solution Assessment

No. The PR is directionally useful, but the current approach is not ready because it bypasses an established SDK boundary and can send unsigned or unauthenticated reasoning events after configured credentials fail to resolve.

  - body: This production Telegram plugin imports from the broad `openclaw/plugin-sdk/config-runtime` barrel.   - body: `resolveConfiguredSecretInputString` reports unresolved SecretRefs by returning `unresolvedRefReason` with no value, but this code ignores that and still creates the sink. A configured signing secret or Authorization header can fail to resolve and the code will POST reasoning events without the expected signature/header; skip the sink or surface a non-secret error before any POST when a configured credential is unresolved.
  - confidence: 0.88
- **[P3] Document the reasoning egress boundary:** `extensions/telegram/src/reasoning-stream-sink.ts:71-75`
  - body: This new sink posts reasoning text plus Telegram chat/thread/account and session identifiers to an arbitrary configured URL, but the PR only adds code and a changelog entry. Add public Telegram or thinking docs covering payloads, signing, SecretRef-backed headers, failure behavior, and how this differs from Telegram preview streaming before exposing the config.
  - confidence: 0.83
