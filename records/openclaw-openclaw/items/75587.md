## Summary

Keep this PR open, but do not merge as-is: the feature is not implemented on current main, and the proposed patch still has a blocking behavior gap because its channel-agnostic thinking sink depends on thinking events that current source only emits when per-channel reasoning preview callbacks are already enabled.

## What This Changes

The PR adds a top-level `agentEventSink` HTTPS webhook config, a core agent-event subscriber for thinking/reply deltas, gateway startup/shutdown wiring, SDK/config baseline updates, Telegram docs, and a changelog entry.

## Best Possible Solution

Decide whether this should be a core config surface or a plugin-backed event subscriber, then ensure thinking events are emitted through a channel-independent contract before documenting all-channel behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new feature/config PR. Source inspection does give a high-confidence behavior check for the proposed feature gap: current main emits `thinking` events only behind existing reasoning-preview callback gates.

## Solution Assessment

No, not yet. The guarded event-bus sink is plausible, but the current patch subscribes to a thinking stream that is not emitted channel-agnostically and still needs a maintainer decision on core versus plugin placement.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Decouple thinking events from preview callbacks:** `src/infra/agent-event-sink.ts:203-212`
  - body: The new sink subscribes to `thinking` events as the source for webhook thinking_start/stream/end, but current main only emits that stream inside `emitReasoningStream`, which returns unless `state.streamReasoning` and `params.onReasoningStream` are already set. Because this top-level sink does not provide that callback or enable reasoning preview, channels/backends that do not already opt into per-channel reasoning previews will never produce the documented thinking events, so the advertised channel-agnostic thinking stream is broken. Please emit AgentEvent thinking deltas independently of channel preview callbacks or narrow the feature/docs accordingly.
  - confidence: 0.88
