## Summary

This PR is not implemented on current main and still needs maintainer review: the guarded webhook path is plausible, but the proposed channel-agnostic thinking sink currently depends on thinking events that are only emitted when the existing channel reasoning-stream callback is active.

## What This Changes

The PR adds a top-level `agentEventSink` config, a core HTTPS webhook sink for thinking/reply events, gateway startup/shutdown wiring, SDK/config baselines, and Telegram docs/changelog updates.

## Best Possible Solution

Have maintainers decide the core-vs-plugin/API boundary first; if this belongs in core, wire thinking AgentEventStream emission independently of channel preview callbacks and add focused tests for event ordering, HMAC, disabled config, unresolved secrets, and shutdown cleanup.

## Reproduction Assessment

Not applicable as a user bug report, but the review finding has a high-confidence source path: `agentEventSink` subscribes to thinking events while current thinking emission still returns unless a channel `onReasoningStream` callback is active.

## Solution Assessment

No. The HTTPS guarded event-bus sink is a plausible implementation direction, but it is not yet the narrowest maintainable solution because thinking emission is still channel-callback-gated and the root egress surface needs maintainer product/security approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Emit thinking events independent of channel callbacks:** `src/infra/agent-event-sink.ts:205-207`
  - body: The new sink only listens for `thinking` bus events, but the current reasoning emitter returns before `emitAgentEvent({ stream: "thinking" })` unless `state.streamReasoning` and a channel `onReasoningStream` callback are already active. Configuring only `agentEventSink` will therefore produce reply webhooks but no `thinking_start`/`thinking_stream` events for normal runs, contrary to the channel-agnostic behavior documented here. Please decouple thinking bus emission from the channel preview callback or explicitly wire this sink into that emission path.
  - confidence: 0.86
