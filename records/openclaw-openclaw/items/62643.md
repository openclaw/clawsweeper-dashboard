## Best Possible Solution

Keep the shipped resolver behavior from `f5a48efa...`. If maintainers want the explicit plugin-author contract, land a rebased docs-only patch on current main that documents `default`, `status`, and `cli` purpose semantics in the SDK-facing type/docs; otherwise resolve this after the canonical #62599 direction is decided.
