## What This Changes

The PR changes Discord, Slack, and Telegram ack-reaction logic, tests, channel docs, and changelog text so `messages.ackReactionScope: "all"` can emit initial ack reactions for ambient `room_event` messages while lifecycle status reactions are intended to remain suppressed.

## Best Possible Solution

Carry a plain-ack versus lifecycle-status decision through Slack dispatch, then land this PR or a replacement only after maintainers accept the ambient-room `all` semantics and live Discord/Slack/Telegram proof shows only the intended ack appears for room events.

No. Honoring `all` across sibling chat plugins is plausible, but this patch is not the best fix until Slack dispatch preserves the plain-ack-only room-event path and live proof covers the visible transport behavior.
