## Summary

Keep this open: current main still has a source-level Slack draft lifecycle path matching the report, and the open focused fix PR already references this issue with closing syntax, so the issue should close only after that PR merges or is replaced.

## What This Changes

The issue reports that Slack partial streaming can replace or clear an in-flight assistant draft with a later tool-warning failure payload even when `streaming.preview.toolProgress=false`.

## Best Possible Solution

Review and land the focused Slack draft lifecycle fix, or replace it with an equivalent narrow change that detaches finalized assistant drafts from later non-finalizable tool-warning handling.

## Reproduction Assessment

Yes, source-level: current main can finalize a Slack draft preview for the assistant answer, then a later `isError` tool-warning final is non-finalizable and follows the fallback path that clears the same draft stream. I did not run a live Slack workspace reproduction.

## Solution Assessment

Yes, the best direction is the narrow Slack lifecycle fix in the linked PR: after successful preview finalization, later same-turn payloads should not reuse that finalized draft as a deletable preview surface.
