## What This Changes

Slack streaming should preserve the assistant draft/final answer when a later tool failure status is emitted with preview tool progress disabled.

## Best Possible Solution

Review and land the focused Slack draft-lifecycle fix in https://github.com/openclaw/openclaw/pull/81979 after acceptable real Slack behavior proof, then close this issue as implemented.

## Reproduction Assessment

Yes, source-level: current main can finalize a Slack draft preview for assistant text, then route a later `isError` tool-warning final through the non-finalizable fallback path that clears the same draft stream. I did not run a live Slack workspace repro in this read-only review.

## Solution Assessment

Yes, the linked PR's direction is the narrow maintainable fix: stop reusing an already-finalized Slack draft as a deletable preview surface for later same-turn warning payloads. Closing the issue before that PR merges would be premature.
