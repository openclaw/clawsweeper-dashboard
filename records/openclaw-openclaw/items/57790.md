## Summary

Keep open. Current main still treats gateway shutdown/restart as gateway lifecycle only; the signal close path does not enumerate active sessions or call `runSessionEnd`, and the related umbrella issue was already closed while leaving this narrower shutdown hook gap as the remaining canonical item.

## What This Changes

Emit `session_end` hooks for active sessions when the Gateway stops or restarts.

## Best Possible Solution

Add a gateway-owned, bounded active-session finalization path that emits typed per-session `session_end` on stop/restart while keeping `gateway_stop` as the service lifecycle hook.

## Reproduction Assessment

Yes. A high-confidence source reproduction is clear: start a gateway with an active session and a `session_end` plugin hook, send SIGTERM/SIGINT/SIGUSR1 or request restart, and the current run-loop/close path reaches gateway shutdown hooks without `runSessionEnd`.

## Solution Assessment

Yes. The narrow maintainable fix belongs in gateway shutdown/session lifecycle code with bounded hook execution and regression coverage, not in plugins inferring per-session endings from `gateway_stop`.

Fix https://github.com/openclaw/openclaw/issues/57790. Current main emits `session_end` for reset/delete/new-session/compaction paths, but Gateway stop/restart only emits gateway lifecycle hooks and closes services. Add a bounded gateway-owned finalization path that snapshots active sessions and emits plugin `session_end` for each active session during SIGTERM/SIGINT/SIGUSR1/restart shutdown, with backward-compatible typed reasons such as `shutdown` and `restart`.
