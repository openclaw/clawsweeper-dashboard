## Summary

Keep open. Current main still has graceful gateway stop/restart hooks and session_end coverage for reset/delete/new-session paths, but the shutdown path does not enumerate active sessions or fire plugin session_end, so the reported lifecycle gap remains valid.

## What This Changes

Request that gateway stop/restart finalizes active sessions by firing plugin session_end hooks during SIGTERM, SIGINT, SIGUSR1 restart, or managed restart shutdown.

## Best Possible Solution

Add a gateway-owned, bounded active-session finalization path that emits typed session_end for shutdown/restart while keeping gateway_stop as the plugin service lifecycle hook.

## Reproduction Assessment

Yes. A source-level reproduction is clear: start a gateway with an active session and a session_end hook, send SIGTERM or SIGUSR1, and the current run-loop/close path never reaches runSessionEnd for that session.

## Solution Assessment

Yes. The narrow maintainable fix belongs in gateway shutdown/session lifecycle code, with explicit shutdown/restart reasons and tests, rather than asking plugins to infer per-session endings from gateway_stop.
