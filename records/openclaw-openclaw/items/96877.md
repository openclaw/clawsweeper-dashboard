## Summary

## What This Changes

## Best Possible Solution

Land the narrow core helper cap after maintainer acceptance of the 16 MiB auth-provider boundary, and keep any remaining plugin-side Copilot response-read sweep separate.

## Solution Assessment

Yes. Applying the existing bounded reader at the shared `fetchJson` helper is the narrowest maintainable fix for this core path; broader Copilot plugin hardening is adjacent follow-up, not a blocker for this patch.
