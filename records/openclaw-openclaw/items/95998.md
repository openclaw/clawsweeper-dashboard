Keep open: current main and latest release still use the raw global EnvHttpProxyAgent path for proxy-configured gateway traffic, the reported qqbot COS media failure has credible logs, and several open PRs are competing candidate fixes rather than merged closure evidence.

## What This Changes

The issue reports that gateway-level undici env proxy setup can route qqbot COS chunked uploads through a proxy despite NO_PROXY exclusions, causing image delivery failures.

## Best Possible Solution

Land one generic core fix that applies OpenClaw's enhanced proxy-bypass behavior at the global dispatcher boundary, preserves existing proxy and timeout semantics, proves the qqbot/COS or equivalent runtime path, and then close or supersede the competing PRs.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. The source path is credible because gateway startup installs a process-wide EnvHttpProxyAgent under proxy env and qqbot chunked media upload performs COS PUTs through the gateway runtime.

## Solution Assessment

Unclear until the PR review path chooses the exact fix. The safest solution is a generic global-dispatcher bypass using existing OpenClaw matcher semantics, not plugin-specific domain hard-coding or no-op NO_PROXY forwarding.
