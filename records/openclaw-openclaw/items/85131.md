## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.8

Full review comments:

- **[P1] Keep auth profile discovery nonprompting:** `src/plugin-sdk/provider-auth.ts:306-307`
  - body: `listUsableProviderAuthProfileIds` feeds `isProviderAuthProfileConfigured`, which provider `isConfigured` checks call during capability discovery. This fallback now reopens the store with `allowKeychainPrompt: true`, so a configured check can synchronously read/prompt for legacy OAuth sidecar secrets even when no session is being started. Please keep discovery nonprompting and move any prompt-capable fallback behind an explicit runtime/session-start path or maintainer-approved policy.
  - confidence: 0.78
- **[P2] Close the relay when microphone startup fails:** `apps/ios/Sources/Voice/RealtimeTalkRelaySession.swift:98`
  - body: `start()` creates the gateway realtime relay session before `startMicrophonePump()` runs. If the input format is invalid or `audioEngine.start()` throws, the manager catches the error and drops `realtimeRelaySession` without calling `talk.session.close`, leaving the provider/gateway relay alive until timeout. Wrap the post-create startup work so partial startup failure closes the relay before rethrowing.
  - confidence: 0.84
