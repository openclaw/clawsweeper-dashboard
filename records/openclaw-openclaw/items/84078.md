## Summary

Keep open: current main still contains the source-proven Slack required durable-send failure path, and https://github.com/openclaw/openclaw/pull/84232 is an open closing PR for this issue. The issue should close only after that PR lands or maintainers choose a different Slack reconciliation path.

## What This Changes

Slack `message_tool_only` source replies can fail when `message(action="send")` is forced into required durable delivery on Slack, because Slack does not expose `reconcileUnknownSend`.

## Best Possible Solution

Review or land https://github.com/openclaw/openclaw/pull/84232 if maintainers accept the current-source best-effort contract; otherwise implement Slack `reconcileUnknownSend` while preserving required durable fail-closed behavior for explicit external sends.

## Reproduction Assessment

Yes at source level: current main maps `bestEffort:false` to required delivery, required delivery demands `reconcileUnknownSend`, and Slack does not declare that capability. I did not run a live Slack current-main reproduction in this read-only pass.

## Solution Assessment

Unclear until maintainer review of the open PR: the proposed current-source best-effort downgrade is narrow and has real Slack proof, but maintainers may instead prefer Slack reconciliation as the durable-contract fix.
