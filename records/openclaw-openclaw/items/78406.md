## Summary

## What This Changes

Report that Discord native slash-command deployment registers commands for the default bot but can leave secondary configured Discord accounts with no commands.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/77359 and https://github.com/openclaw/openclaw/pull/77367 as the single fix path, app-scope the Discord command deploy cache, and land it with live multi-bot proof.

## Reproduction Assessment

Yes. Source inspection on current main shows two Discord applications sharing `commandDeployHashStorePath` and identical command specs can collide on `global:reconcile`, causing the later account to return before REST reconciliation; the canonical issue also has live v2026.5.7 confirmation.

## Solution Assessment

Yes for cleanup routing. The best solution is not a separate issue thread here, but to keep review and validation on the canonical issue and open PR that scope the deploy cache by application/account while preserving same-application cache skips.
