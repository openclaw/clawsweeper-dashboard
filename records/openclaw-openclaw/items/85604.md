## Summary

Keep open: current main still lacks Feishu `unsend`, and this is a focused bug-fix PR, but it is not merge-ready because the new handler ignores the Feishu API response and the supplied proof does not show a live Feishu recall. The work also overlaps the broader open Feishu message-management PR at https://github.com/openclaw/openclaw/pull/83517, so maintainers should choose whether to land this narrow fix or fold it into that path.

## What This Changes

The PR adds Feishu `unsend` to the advertised message actions, dispatches it through `client.im.message.delete`, and adds a unit test plus action-list assertions.

## Best Possible Solution

Keep the plugin-local `unsend` handler using the SDK delete/recall endpoint, but validate `{ code, msg }`, add a negative regression test, and require live Feishu proof or an explicit maintainer proof override before merge.
