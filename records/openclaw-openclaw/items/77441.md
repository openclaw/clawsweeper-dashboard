## Summary

Keep open. Current main and the latest release still prefer a normalized channel alias over the installed manifest claim, and an open linked PR (#77452) explicitly targets this issue, so the issue should remain open until that fix or an equivalent lands.

## What This Changes

Configured WeCom/Yuanbao channels should auto-enable their owning plugin IDs in `plugins.allow` instead of channel aliases.

## Best Possible Solution

Review and land #77452 or an equivalent narrow fix that prefers manifest plugin IDs when channel claims exist, with regression coverage for external channel plugins without `preferOver`.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: configured `wecom`/`yuanbao` channels produce a non-empty claim set plus a `builtInId`, and the current fallback returns the alias that later gets written to `plugins.allow`.

## Solution Assessment

Yes. The narrow maintainable fix is to keep the existing no-claim bundled fallback and `preferOver` handling, but prefer the claiming manifest plugin ID when claims exist; the linked PR follows that direction but has not landed.
