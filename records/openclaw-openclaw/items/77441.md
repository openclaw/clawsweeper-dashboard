## Summary

Keep open. Current main and v2026.5.3 still prefer the normalized built-in channel alias over an installed manifest claim, so WeCom/Yuanbao can be written to `plugins.allow` as non-plugin IDs.

## What This Changes

The issue requests that channel auto-enable add the installed WeCom/Yuanbao plugin IDs to `plugins.allow` instead of the channel aliases.

## Best Possible Solution

Update channel auto-enable resolution so installed manifest plugin claims win over built-in channel aliases when plugin IDs differ, then add regression coverage and a user-facing changelog entry.

## Reproduction Assessment

Yes, source-level: configured channel IDs flow through `resolveConfiguredPluginAutoEnableCandidates` into `collectPluginIdsForConfiguredChannel`, which returns the built-in alias before the owning plugin ID. I did not run a live gateway restart in this read-only pass.

## Solution Assessment

Yes. The narrow maintainable fix is in `collectPluginIdsForConfiguredChannel`, with regression tests for built-in channel IDs whose installed plugin ID differs from the channel ID; doctor cleanup alone would not stop restart-time regeneration.

## Repair Work Prompt

Fix issue https://github.com/openclaw/openclaw/issues/77441. The bug is that `collectPluginIdsForConfiguredChannel` in `src/config/plugin-auto-enable.shared.ts` returns a built-in channel alias such as `wecom` or `yuanbao` before the installed manifest plugin ID, causing `plugins.allow` to persist non-plugin IDs. Change only the channel auto-enable resolution so installed manifest plugin claims win when the plugin ID differs from the channel ID, while preserving bundled-channel fallback and existing `preferOver` behavior. Add regression coverage in `src/config/plugin-auto-enable.channels.test.ts` for WeCom/Yuanbao-style built-in channel IDs claimed by external plugin IDs, including restrictive `plugins.allow` behavior. Add a user-facing `CHANGELOG.md` fix entry. Do not change plugin catalog IDs, install specs, or doctor cleanup policy except as required by the regression.
