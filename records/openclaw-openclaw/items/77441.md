## Summary

Keep open. Current main still has the same source-level bug, and the issue is already paired with open closing PR #77452 rather than something to close before merge.

## What This Changes

The issue asks for plugin auto-enable to add the owning WeCom and Yuanbao plugin IDs instead of persisting their channel aliases in `plugins.allow`.

## Best Possible Solution

Review and land or replace #77452 so the channel auto-enable resolver prefers an installed manifest plugin claim over a built-in channel alias when no `preferOver` candidate applies.

## Reproduction Assessment

Yes, source-level. A configured `channels.wecom` or `channels.yuanbao` can resolve to a built-in-style channel alias, and the current resolver returns that alias before the manifest plugin id, which is then allowlisted.

## Solution Assessment

Yes. The narrow maintainable fix is in `collectPluginIdsForConfiguredChannel`, with regression coverage for a known channel id whose owning manifest plugin id differs from the channel alias.
