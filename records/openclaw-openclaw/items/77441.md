## Summary

Keep open: current main still has the alias-before-plugin fallback, and an open closing PR already targets the same bug, so the issue should remain open until that fix or an equivalent lands.

## What This Changes

Configured WeCom or Yuanbao channels should auto-enable their external plugin IDs in `plugins.allow` instead of the channel aliases.

## Best Possible Solution

Review and land #77487, or an equivalent narrow change that makes manifest plugin IDs win for built-in channel claims while preserving no-claim built-in fallback and `preferOver` behavior.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: configured `wecom` or `yuanbao` channels flow through `collectPluginIdsForConfiguredChannel`, which currently returns the built-in channel alias before the claiming plugin ID. I did not run the gateway live.

## Solution Assessment

Yes. The narrow maintainable solution is in `collectPluginIdsForConfiguredChannel`, with regression coverage for built-in channel IDs whose owning plugin ID differs from the channel alias; #77487 already proposes that direction.
