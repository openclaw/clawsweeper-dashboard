## Summary

Keep open: current main and v2026.6.9 provide a durable `--wrapper` escape hatch and still block raw `NODE_OPTIONS`, but they do not implement the requested heap default, `--heap-mb` option, env override, or status warning, and the new crash-loop evidence overlaps active Gateway memory trackers.

## What This Changes

The issue asks for managed Gateway installs to set a higher Node heap by default, expose heap tuning, or warn when no heap cap is configured to avoid launchd crash loops under large state.

## Best Possible Solution

Choose a maintainer-approved Gateway heap mitigation path: a narrow heap flag/status diagnostic, an explicit decision to rely on wrappers and docs, or folding this evidence into the canonical memory tracker before closure.

No high-confidence current-main OOM replay was run. Source inspection does verify that current and shipped installers lack the requested heap-specific option/default/warning, while the issue provides strong older-release live crash-loop evidence.

## Solution Assessment
