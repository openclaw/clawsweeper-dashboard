## Label Justifications

- `P0`: The report describes a Gateway restart loop every 10-12 seconds that made RPC unavailable and blocked workflow processing.
- `impact:crash-loop`: The central failure mode is a managed Gateway service repeatedly exiting from V8 heap exhaustion and being restarted by launchd.

## Summary

Keep this issue open: current main and the latest release still generate managed Gateway service environments without a heap-setting `NODE_OPTIONS`, while https://github.com/openclaw/openclaw/pull/96250 is an open closing PR for the requested daemon heap default. The earlier wrapper discussion is useful related context, but it is not the same as the requested default or supported heap knob.

## What This Changes

Request a durable managed Gateway heap limit during daemon install, or a supported heap-size override, to prevent V8 OOM crash-loops under larger workspaces.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/96250; close this issue only after that candidate or an equivalent maintainer-approved daemon heap path reaches main.

## Reproduction Assessment

No high-confidence local OOM reproduction was run. The issue provides strong live crash-loop evidence, and source inspection confirms current main does not install a managed-service heap setting and filters ambient `NODE_OPTIONS`.

## Solution Assessment

Unclear until the linked PR is reviewed. Setting the heap at the daemon install boundary matches the reported failure mode, but the exact default size and any override must preserve the existing `NODE_OPTIONS` security boundary and upgrade behavior.
