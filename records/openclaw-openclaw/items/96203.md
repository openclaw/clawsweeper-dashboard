## Summary

Keep this issue open: current main and v2026.6.10 still generate managed Gateway service environments without a heap-setting `NODE_OPTIONS`, and the linked closing pull request is open rather than merged.

## What This Changes

The issue asks OpenClaw to prevent managed Gateway V8 heap crash loops by setting daemon install `NODE_OPTIONS` by default or adding a supported heap-size knob or warning.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/96250; if landing, keep the daemon heap behavior narrow, preserve the `NODE_OPTIONS` security boundary, and cover fresh install plus reinstall behavior.

## Reproduction Assessment

No, not for the full OOM crash loop on current main. The issue includes strong live crash-loop evidence, and source inspection confirms the relevant install behavior: current managed services do not set heap `NODE_OPTIONS` and ambient `NODE_OPTIONS` is filtered.

## Solution Assessment

Unclear until the linked PR is reviewed. Setting heap headroom at the daemon install boundary matches the reported failure mode, but the exact default and override path must preserve upgrade behavior and the intentional `NODE_OPTIONS` security boundary.
