## Summary

Current main and v2026.5.6 already provide the durable managed-service wrapper path needed to run the Gateway with custom Node startup flags, so the crash-loop workaround no longer requires hand-editing generated launchd files.

## What This Changes

The issue requests a managed Gateway heap-size default, flag, env override, or status warning after a launchd Gateway crash-looped at the default Node heap limit.

## Best Possible Solution

Keep the shipped wrapper-based service customization path; if discoverability remains a problem, add a heap-specific wrapper example or diagnostic hint rather than persisting raw NODE_OPTIONS by default.

## Reproduction Assessment

No high-confidence current-main OOM reproduction was run; the report provides strong live symptoms from an older release, and source inspection verifies the current supported startup-flag path.

## Solution Assessment

Yes for cleanup: the existing wrapper path is the narrower maintainable solution because raw NODE_OPTIONS is intentionally blocked and a default heap bump would need product review.
