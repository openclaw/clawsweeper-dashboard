## What This Changes

Adds PathGuard path ACL config/enforcement for file tools, media loads, apply_patch, and spawned-subagent filesystem policy ceilings, plus tests/schema/help updates and a minimatch dependency.

## Best Possible Solution

Land a narrower owner-reviewed ACL implementation only after one generated-schema-backed policy contract is enforced consistently across prompt media, file tools, apply_patch, and spawned subagents, with green CI and redacted real behavior proof.

## Reproduction Assessment

Yes. The PR-head blockers are source-reproducible from code inspection: prompt image ingestion receives only workspaceOnly, relative deny globs return false for outside-workspace symlink targets, and the generated public schema lacks the runtime-added ACL fields.

## Solution Assessment

No. PathGuard is a plausible direction, but this branch is not the best mergeable solution until the bypasses, generated schema drift, failed checks, and real behavior proof gap are resolved under security/design review.

- **[P1] Enforce full fsPolicy for prompt image loads:** `src/agents/pi-embedded-runner/run/attempt.ts:466`
  - body: Adding the spawned `fsPolicy` only to `createOpenClawCodingTools` leaves prompt-inline image loading below on `workspaceOnly` alone. A vision prompt can still load a local image path that should be blocked by `allowedPaths`, `denyPaths`, or a spawned-session ceiling.
  - confidence: 0.91

  - confidence: 0.88

  - confidence: 0.86
