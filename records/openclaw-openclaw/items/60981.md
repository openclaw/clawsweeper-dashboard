Full review comments:

- **[P1] Preserve deny globs for symlinked relative entries:** `src/security/path-guard.ts:271-275`
  - body: For a config like `workspaceOnly: false` with `denyPaths: ["vendor/**"]`, if `vendor` is a symlink to an external directory, the requested file is canonicalized outside the workspace and this branch returns `false` before matching the deny glob. That lets the symlinked target bypass the deny rule, weakening the filesystem boundary the PR is trying to add.
  - confidence: 0.87
- **[P1] Propagate fsPolicy to prompt-inline image loading:** `src/agents/pi-embedded-runner/run/attempt.ts:466`
  - body: This only sends `toolFsPolicy` to `createOpenClawCodingTools`, but `detectAndLoadPromptImages` later receives only `workspaceOnly`. In vision-capable runs, a local image path written directly in the prompt can still be loaded without the new `allowedPaths`/`denyPaths` or spawned-session ceiling checks.
  - confidence: 0.9
- **[P2] Avoid host PathGuard checks on container-only media paths:** `src/agents/tools/image-tool.ts:536-540`
  - body: In sandbox mode, `resolveSandboxedBridgeMediaPath` may return a container path when there is no host mapping, but this call still compares that path against a host policy root. That can reject valid sandbox-local image/PDF inputs or give a false sense that host-path policy was enforced; translate to a host path first, or skip host PathGuard for container-only paths and rely on the sandbox bridge.
  - confidence: 0.78
