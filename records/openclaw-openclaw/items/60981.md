## Summary

Keep open. Current main still has only the existing workspaceOnly filesystem policy, and this PR remains an active but not mergeable security feature because the latest head has source-visible policy bypasses, generated-schema drift, failing CI, and no real behavior proof.

## What This Changes

The PR adds a PathGuard filesystem ACL layer with allowedPaths/denyPaths config, media and apply_patch enforcement, spawned-subagent filesystem ceilings, tests, schema/help updates, and a minimatch dependency.

## Best Possible Solution

Land a narrower owner-reviewed filesystem ACL contract only after one generated-schema-backed policy is enforced consistently across file tools, prompt media, local media tools, apply_patch, and spawned subagents, with green CI and redacted real setup proof.

## Reproduction Assessment

Yes. The PR-head blockers are source-reproducible: prompt-inline images receive only workspaceOnly, relative deny globs still miss outside-workspace symlink targets, and the generated schema omits the new FS policy keys.

## Solution Assessment

No. PathGuard is a plausible direction, but this branch is not the best mergeable solution until the policy bypasses, schema drift, CI failures, and real behavior proof gap are resolved under owner/security review.
