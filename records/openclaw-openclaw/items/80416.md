Keep open. ## Best Possible Solution

Keep this issue as the narrow isolated-cron performance tracker until maintainers can live-profile the reported path and choose whether existing no-allowlist cron jobs should be migrated/narrowed or whether broad warm-run tool construction should be made cheap enough by design.
