Keep open. The report is credible and current main still has an isolated-cron-specific path that creates a fresh run session, constructs OpenClaw/plugin tools, materializes bundle tools, and retires bundle MCP resources per isolated run; overlapping open perf issues do not fully supersede this narrower cron path.

The issue asks OpenClaw to eliminate recurring core-plugin-tools, bundle-tools, and stream-setup overhead on isolated cron embedded agent runs after earlier registry reuse fixes.

## Best Possible Solution

Profile current main on the isolated cron path, then keep the existing plugin registry compatibility contract while caching or reusing only safe static tool/bundle descriptors and preserving isolated-session cleanup boundaries.
