## Summary

Keep open. The reporter narrowed the symptom to isolated cron `agentTurn` sessions, and current `main` has a distinct cron path that force-creates run sessions and retires bundled MCP runtimes, so the prior general plugin-tool prep fixes do not prove this specific path is fixed.

## What This Changes

The issue reports repeated `core-plugin-tools`, `bundle-tools`, and `stream-setup` overhead on isolated cron `agentTurn` sessions after the fixes associated with https://github.com/openclaw/openclaw/issues/75520 and https://github.com/openclaw/openclaw/issues/77029.

## Best Possible Solution

Preserve isolated cron transcript and authorization isolation while moving reusable plugin/tool and bundled-runtime preparation to a cache boundary that is safe across repeated runs of the same cron job.

## Reproduction Assessment

No live repro was run. Source inspection gives a medium-confidence reproduction path for the isolated-cron distinction: repeated `sessionTarget: isolated` jobs force fresh run sessions and session-id-keyed bundled runtime setup, matching the reporter's update.

## Solution Assessment

Unclear. The right fix should first profile isolated cron substage timings on current `main`, then cache only data and runtimes that do not carry per-run transcript, auth, delivery, or tool-policy state.
