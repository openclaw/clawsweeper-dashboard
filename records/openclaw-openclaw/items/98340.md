## What This Changes

The PR changes extension session exec cleanup to spawn detached processes and call the shared process-tree killer on timeout, abort, or output-limit termination.

## Best Possible Solution

Land this narrow process-tree cleanup fix after the contributor adds redacted real behavior proof for api.exec descendant cleanup and normal merge checks are green; keep https://github.com/openclaw/openclaw/issues/98335 open until the fix merges.

## Reproduction Assessment

Yes at source level: current main and v2026.6.11 show api.exec cleanup using direct child kills, while sibling shell execution uses process-tree cleanup. I did not run a live orphan-process reproduction in this read-only review.

## Solution Assessment

Yes, this appears to be the best narrow fix: it moves api.exec onto the existing process-tree helper and detached Unix spawn boundary without adding config or changing the result shape. The safer alternative of a new custom termination path would duplicate existing process helper behavior.
