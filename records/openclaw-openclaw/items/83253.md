## Summary

Keep open: current main still counts retained terminal task failures in `openclaw status`, and the overlapping open PR does not yet settle the requested current/retained/soft-retained health model.

## What This Changes

The issue requests task status semantics that separate live task failures from retained historical and soft-retained task records.

## Best Possible Solution

Define a maintainer-approved task-health contract, then land a revised implementation that aligns registry summaries, CLI status/tasks/flows output, audit and maintenance semantics, docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection shows a retained `failed`, `timed_out`, or `lost` task passed through `getInspectableTaskRegistrySummary` is counted in `failures` and rendered by `openclaw status` as task issues.

## Solution Assessment

Unclear. The requested split is directionally sound, but the exact current/retained/soft-retained semantics and the open implementation path need maintainer product direction before a new fix should land.
