## Summary

Keep open: current main and v2026.6.8 only quiet the retained-lost subset, while the broader current/retained/soft-retained task-health model for failed, timed-out, delivery-failed, and reviewed historical task records is still missing.

## What This Changes

The issue requests native task status/audit semantics that separate current operational failures from retained historical task records and provide a supported cleanup or acknowledgement path.

## Best Possible Solution

Define one maintainer-approved task health model across summaries, audit output, maintenance, docs, and tests so default status alerts on live or unreviewed failures while retained records remain inspectable.

## Reproduction Assessment

Yes, at source level: retained failed, timed_out, and lost records still feed aggregate failures, while status only discounts retained lost findings. I did not run a live state-database reproduction because this review is read-only.

## Solution Assessment

Unclear: the requested split is directionally maintainable, but the exact current, retained, soft-retained, acknowledgement, and JSON compatibility semantics need maintainer approval before implementation.
