## Summary

Keep open: current main and v2026.6.8 only implement a retained-lost discount, while the broader current/retained/soft-retained task-health model for retained failed, timed-out, delivery-failed, and reviewed historical rows is still missing.

## What This Changes

The issue asks OpenClaw to distinguish live task failures from retained historical task records in task/status health output and provide a supported cleanup path.

## Best Possible Solution

Define one maintainer-approved task health model across summaries, audit output, maintenance, docs, and tests so default status alerts on live or unreviewed operational failures while retained records remain inspectable.

## Reproduction Assessment

Yes, at source level: retained failed, timed_out, and lost task records still increment aggregate failures, and status only subtracts retained lost findings. I did not run a live state-database reproduction because this review is read-only.

## Solution Assessment

Unclear: the requested split is directionally maintainable, but the exact current, retained, soft-retained, acknowledgement, and JSON compatibility semantics need maintainer approval before implementation.
