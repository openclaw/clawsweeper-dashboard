## Label Justifications

- `P1`: The reported bug can repeatedly spawn isolated agent sessions and duplicate scheduled channel output after a one-shot job should have finished.
- `impact:message-loss`: The issue reports duplicate Discord posts from a scheduled job that should have delivered once.

## Summary

Keep open: current main and v2026.6.8 still leave the normal timer path without the terminal one-shot guard, and ordinary due-job outcomes are buffered until the worker batch drains. A same-repo candidate fix PR is open, so cleanup should not close this issue before that path lands or is replaced.

## What This Changes

Reported successful one-shot `kind: "at"` cron jobs with `deleteAfterRun: true` can remain due and re-trigger instead of being deleted after the first run.

## Best Possible Solution

Keep this issue open and land or replace the focused scheduler fix that finalizes completed due-job outcomes promptly and prevents terminal one-shot at jobs left in the store from being selected again by the normal timer path.

## Reproduction Assessment

Yes. Source inspection shows the normal timer path still omits the terminal one-shot guard and ordinary due-job outcomes are still batch-buffered before deletion is applied; no tests were run because this review is read-only.

## Solution Assessment

Yes, a scheduler-layer fix is the right direction. The best path is to review and land https://github.com/openclaw/openclaw/pull/81572 or a narrow replacement, while keeping failed-run deleteAfterRun policy in the separate linked issue.
