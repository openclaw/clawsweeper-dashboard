## Summary

This PR is a focused Discord bugfix, but it should remain open because the patch uses the broad suppressed-source progress gate and can also cut off explicit Discord status-reaction lifecycle callbacks; it also still lacks after-fix real Discord proof.

## What This Changes

The PR disables Discord draft-progress creation and suppressed-source progress callbacks for message-tool-only replies, and updates the Discord process regression test to expect no private draft updates.

## Best Possible Solution

Keep message-tool-only Discord draft progress private while preserving explicit status-reaction lifecycle callbacks through a narrower condition or split callback path, then require redacted real Discord proof for the no-draft and final-delivery behavior.

## Reproduction Assessment

Yes, at source level. Current main creates a Discord progress draft for message-tool-only progress mode, and the dispatcher contract shows that removing allowProgressCallbacksWhenSourceDeliverySuppressed also suppresses explicit status-reaction progress callbacks under source delivery suppression.

## Solution Assessment

No, not as written. Disabling the draft stream is the right boundary for the leak, but disabling all suppressed-source progress callbacks is too broad because Discord also uses that callback path for explicit status-reaction lifecycle updates.
