  - body: Current main and v2026.6.10 expose `loadSessionStore`, `saveSessionStore`, `updateSessionStore`, and `resolveSessionFilePath` on `api.runtime.agent.session`, and the SDK subpath also shipped compatible helpers. That open path can mkdir/chmod, run schema DDL, and register DBs, so read-only doctor modes may mutate state before an operator chooses import or fix; use non-creating probes or gate these opens to mutating modes.
  - confidence: 0.88
- **[P3] Align SDK docs with the retained file helper:** `docs/plugins/sdk-runtime.md:169`
  - body: The PR head still exports `resolveAndPersistSessionFile` from `openclaw/plugin-sdk/session-store-runtime`, but this docs line says active transcript-file helpers are no longer exported.
