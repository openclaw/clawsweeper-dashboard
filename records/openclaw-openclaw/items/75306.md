## Summary

Keep open: the Claude ACP timeout failure is real, but the latest diff does not fix current main's manager-side unsupported-control path and also drops timeout controls for any ACPX backend that advertises them.

## What This Changes

This PR no-ops `timeout` and `timeout_seconds` in the ACPX runtime wrapper for all backends and changes the Claude ACP unit test to expect no forwarding.

## Best Possible Solution

Filter unsupported timeout controls in the manager/runtime-option application path before advertised-key validation, while preserving timeout forwarding for ACP agents that explicitly advertise it.

## Reproduction Assessment

Yes, with high confidence from source and live report evidence: current main builds a timeout runtime config option, Claude ACP 0.33.1 does not advertise or accept that key, and related reports show the resulting Claude ACP spawn failure.

## Solution Assessment

No. Skipping unsupported timeout controls is the right direction, but this PR patches a downstream wrapper after current main's manager validation can already fail and it globally suppresses timeout for backends that support it.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
