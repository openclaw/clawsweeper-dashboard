## Summary

Keep open: this PR targets a real Claude ACP regression, but the current two-file acpx wrapper change is downstream of the current `main` failure path and can still fail before the new guard is reached.

## What This Changes

The PR changes the acpx runtime wrapper to drop `timeout` and `timeout_seconds` config-option writes for all ACP backends and updates the Claude runtime unit test to expect no forwarding.

## Best Possible Solution

Move the fix to the manager runtime-control boundary by skipping or fail-softing unsupported auto-injected timeout controls before `runtime.setConfigOption`, while preserving strict rejection for explicit user config-option writes.

## Reproduction Assessment

Yes. Current `main` still builds a `timeout` runtime config option and rejects non-advertised keys before the acpx wrapper call, and the linked reports reproduce the same Claude ACP failure on recent releases.

## Solution Assessment

No. The PR handles direct acpx `setConfigOption` forwarding, but the reported `sessions_spawn` path can fail earlier in `applyManagerRuntimeControls`; the fix should live at the manager runtime-control construction/application boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91
