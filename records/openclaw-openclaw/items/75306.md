## Summary

Keep open for normal maintainer review and merge gating. Current main still forwards ACPX timeout config to Claude ACP, and the proposed patch is a narrow fix that matches the bundled Claude ACP adapter behavior while preserving OpenClaw's manager-level turn timeout.

## What This Changes

The PR hoists ACPX `timeout`/`timeout_seconds` config-option filtering so all ACPX backends skip those keys, updates the Claude ACP runtime unit test, and adds a changelog entry.

## Best Possible Solution

Land this focused ACPX wrapper fix after the required checks pass, keeping timeout persistence and enforcement in the OpenClaw manager while avoiding unsupported session-level timeout replay into Claude ACP.

## Reproduction Assessment

Yes. The source path is high confidence: current main emits `timeout`, ACPX exposes generic config-option forwarding without advertised keys, and the bundled Claude ACP adapter rejects unknown config IDs; the PR body also includes a live Dash `sessions_spawn` reproduction.

## Solution Assessment

Yes. Filtering the unsupported timeout keys in the ACPX wrapper is the narrowest maintainable fix for this regression because OpenClaw already stores and enforces the turn timeout outside the child ACP session.
