## Summary

Keep open. Current main still forwards `timeout` for Claude ACPX sessions, while the proposed PR is a narrow and appropriate fix with no discrete correctness finding from this review; merge should wait on normal maintainer review and required checks.

## What This Changes

The PR hoists ACPX `timeout` and `timeout_seconds` config-option filtering so it applies to all backends, updates the Claude ACP runtime unit test, and adds a changelog entry.

## Best Possible Solution

Land this narrow ACPX runtime fix after required CI completes, keeping OpenClaw's manager-level turn timeout as the canonical enforcement point and avoiding a broader capability-negotiation redesign for this focused Claude ACP regression.

## Reproduction Assessment

Yes. The source-level path is high confidence: current main builds a `timeout` config pair, ACPX exposes generic config-option control without key metadata, and `claude-agent-acp@0.31.4` throws on any config ID not in its `mode`/`model`/optional `effort` list; the PR body also includes a live Dash `sessions_spawn` reproduction.

## Solution Assessment

Yes. Filtering unsupported timeout keys in the ACPX wrapper is the narrowest maintainable fix because OpenClaw already persists and enforces the turn timeout in the manager, while Claude ACP has no session-level handler for that key.
