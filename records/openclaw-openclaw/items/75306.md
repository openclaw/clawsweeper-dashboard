## Summary

Keep open. Current main still forwards `timeout` to Claude ACP through the ACPX wrapper, and this PR is a narrow, well-scoped fix with matching test and changelog coverage; merge should wait for maintainer review and exact-head CI triage.

## What This Changes

The PR hoists ACPX `timeout` and `timeout_seconds` config-option filtering out of the Codex-only branch so those keys are ignored for all ACPX backends, updates the Claude ACP runtime test, and adds a changelog entry.

## Best Possible Solution

Land this focused ACPX timeout-filtering fix after exact-head CI is understood, while keeping broader backend capability negotiation separate from this targeted Claude ACP regression.

## Solution Assessment
