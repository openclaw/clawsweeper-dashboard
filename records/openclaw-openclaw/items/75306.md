## Summary

Keep open. Current main still forwards `timeout` to Claude ACP through ACPX, while the PR’s small diff removes that unsupported session config replay without touching OpenClaw’s separate manager-level turn watchdog. I found no discrete correctness or security finding in the proposed patch; it should remain open for normal maintainer review and exact-head CI.

## What This Changes

The PR hoists ACPX `timeout`/`timeout_seconds` config-option filtering so it applies to all ACP backends, updates the Claude ACP runtime unit test, and adds a changelog entry.

## Best Possible Solution

Land this focused ACPX runtime fix after required checks pass, keeping OpenClaw’s manager-level timeout as the canonical timeout enforcement path and avoiding a broader ACP capability negotiation redesign for this Claude-specific regression.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: `sessions_spawn` with ACP runtime options can persist `timeoutSeconds`, current main converts that to `timeout`, and Claude ACP 0.31.4 rejects unsupported `session/set_config_option` IDs; the PR body also includes a live Dash `sessions_spawn` failure and success after the patch.

## Solution Assessment

Yes. Filtering unsupported timeout keys in the OpenClaw ACPX wrapper is the narrowest maintainable fix because OpenClaw already owns turn-timeout enforcement in the ACP manager, and the external Claude ACP adapter does not expose a session-level `timeout` option.
