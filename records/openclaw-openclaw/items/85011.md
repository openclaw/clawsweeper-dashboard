## Summary

Keep open: current main still drops route facts before calling `plugin.approval.request`, while gateway approval routing depends on those fields to avoid no-route expiry or stale session fallback.

## What This Changes

The issue asks for `before_tool_call` approval prompts to carry source routing so WebChat can receive approvals in offline deployments.

## Best Possible Solution

Carry prepared turn-source provider, target, account, and thread facts into `before_tool_call` approval requests, then add focused coverage for stale session channel routing, missing session-origin subagents, and WebChat/internal approval delivery.

## Reproduction Assessment

We have a high-confidence source reproduction, but not an executed one: the before-tool-call caller omits turn-source fields while the gateway no-route and stale-session paths branch on those fields. A focused test can mock `callGatewayTool` and assert the missing route payload, then cover gateway behavior with no approval clients.

## Solution Assessment

Yes, the maintainable fix is to propagate canonical route facts already prepared by the agent run into `plugin.approval.request`; a broad unconditional WebChat fallback would be less safe than preserving explicit source routing and existing approval auth checks.
