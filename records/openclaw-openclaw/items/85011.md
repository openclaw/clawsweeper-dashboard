## What This Changes

The issue reports that before_tool_call plugin approval prompts cannot reach WebChat-only offline deployments because the agent does not forward approval routing context to plugin.approval.request.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/74003 as the canonical tracker and land or close https://github.com/openclaw/openclaw/pull/84205 after trusted route forwarding plus real behavior proof, carrying the WebChat/offline case there if extra coverage is needed.

## Solution Assessment

Yes. Consolidating this duplicate under the canonical issue is the best tracking outcome; the code fix should forward trusted runtime turn-source metadata, not model/tool-call params, and prove the routed approval path.
