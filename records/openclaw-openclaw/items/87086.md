- **[P1] Preserve object-form context-engine owners before doctor rewrites:** `src/commands/doctor/shared/codex-route-warnings.ts:1947-1948`
  - body: This PR lets `plugins.slots.contextEngine` be an object, but the migration guard above this changed branch still only recognizes string values before calling `canAutoMigrateLegacyLosslessCompaction()`. With `{ owner: "other-engine" }`, the guard treats the slot as unset and this branch can rewrite it to `lossless-claw` during `doctor --fix`, silently replacing the user's selected owner. Resolve the owner before the safety check and cover a non-Lossless object-form owner.
  - confidence: 0.88
- **[P2] Normalize slot owners in configured-plugin install repair:** `src/config/zod-schema.ts:1210-1211`
  - body: The schema now accepts object-form slot values, but `collectSlotPluginIds()` still passes raw `slots?.[key]` to `normalizeId()`. For `{ owner: "memory-lancedb" }` that contributes no id, so configured-plugin install repair can skip a slot-selected official plugin even though the rest of the PR treats the owner as selected. Route this through `resolvePluginSlotOwner()` and add focused coverage.
  - confidence: 0.89
- **[P2] Use normalized memory owner for active-memory defaults:** `src/config/types.plugins.ts:51`
  - body: After this type change, `plugins.slots.memory` can be `{ owner: "memory-lancedb" }`, but `extensions/active-memory/index.ts` still checks the raw value for equality with `"memory-lancedb"`. Object-form LanceDB selection will fall back to the default `memory_search`/`memory_get` tool set instead of the LanceDB `memory_recall` default, so the two slot forms are not equivalent.
