## Summary

Keep open: the runtime/test patch is narrow and matches the current-main regression, but the PR still needs the docs and changelog cleanup required for a user-facing TTS fix before merge.

## What This Changes

The PR scopes speech-core short-text guards to untagged auto-TTS, adds an empty post-markdown explicit-text guard, and covers tagged short, untagged short, and markdown-only tagged payloads.

## Best Possible Solution

Merge after adding an Unreleased Fixes changelog bullet and updating the TTS docs for the explicit-tag short-text exception; leave the separate message-tool send-path parity work to #74955.

## Reproduction Assessment

Yes. On current main, `[[tts:text]]hello[[/tts:text]]` is parsed as explicit TTS text, but the unconditional under-10-character guard returns before synthesis.

## Solution Assessment

Yes, with small text-surface cleanup. Separating explicit tagged speech from untagged auto-TTS is the narrow runtime fix, and the added empty post-markdown guard covers the reviewed edge case.
