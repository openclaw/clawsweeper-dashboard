## Summary

Keep open: this is a focused and still-useful Volta path canonicalization PR, but current main has not implemented the Volta resolver behavior and the submitted branch still has blocking resolver gaps plus missing real behavior proof.

## What This Changes

The PR adds Volta tools/image matching to `resolveStableNodePath` and unit tests for default Unix shim resolution plus missing-shim fallback.

## Best Possible Solution

Update the resolver to derive Volta home generically, handle Unix and native Windows image/shim layouts, preserve existing fallback and supported-system-Node preference, add focused coverage, and attach redacted real Volta proof.

## Reproduction Assessment

Yes, source-level. Current main returns Volta tools/image Node paths unchanged, and the PR branch still leaves custom `VOLTA_HOME` and native Windows Volta layouts version-pinned despite accessible stable shims.

## Solution Assessment

No, not as submitted. Resolver-level normalization is the right boundary, but the matcher should parse Volta home generically and follow Volta's Unix and Windows layouts before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Match custom Volta homes:** `src/infra/stable-node-path.ts:19`
  - body: The matcher only recognizes paths containing `/.volta/tools/image/...`, but Volta supports `VOLTA_HOME` and OpenClaw already accepts `${VOLTA_HOME}/bin` in adjacent service PATH handling. With a custom home such as `/opt/volta`, this still returns the version-pinned image binary and the service can go stale after a pin or upgrade.
  - confidence: 0.94
- **[P2] Handle native Windows Volta layout:** `src/infra/stable-node-path.ts:19-25`
  - body: The Windows path handling still assumes a Unix-shaped `.volta/.../bin/node.exe` image and a `node.exe` shim. Upstream Volta defaults to `%LOCALAPPDATA%\Volta`, places Node image executables at the version root on Windows, and exposes shims as `.cmd`, so native Windows Volta paths fall through unchanged.
  - confidence: 0.88
