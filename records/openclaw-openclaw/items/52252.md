## Summary

Keep open: current main still lacks Volta tools/image-to-shim normalization, and this PR is the active implementation for https://github.com/openclaw/openclaw/issues/52184, but it is not merge-ready because the matcher only covers the default Unix layout and the PR body only shows unit-test proof.

## What This Changes

The PR adds `resolveStableNodePath` handling for default Unix Volta `~/.volta/tools/image/node/<version>/bin/node` paths and tests accessible and missing shim cases.

## Best Possible Solution

Land a rebased resolver change that derives Volta home from `<VOLTA_HOME>/tools/image/node/<version>`, covers Unix and Windows layouts, preserves missing-shim fallback, and includes focused tests plus redacted real Volta gateway/install proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main returns Volta tools/image paths unchanged, and the PR still misses custom `VOLTA_HOME` and Windows Volta layouts despite accessible stable shims.

## Solution Assessment

No, not as submitted. Rewriting Volta image paths to stable shims is the right direction, but the matcher should parse the Volta home generically and account for upstream Windows layout before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Match custom Volta homes:** `src/infra/stable-node-path.ts:19`
  - body: The matcher only accepts paths containing `/.volta/tools/image/...`, but Volta supports custom `VOLTA_HOME` and OpenClaw already treats `${VOLTA_HOME}/bin` as a service PATH input. A path such as `/opt/volta/tools/image/node/24.14.0/bin/node` still falls through unchanged, so the installed service remains version-pinned after this fix.
  - confidence: 0.94
- **[P2] Handle Windows Volta image paths:** `src/infra/stable-node-path.ts:19-25`
  - body: The Windows branch still requires a `.volta` home and `bin/node.exe`, but upstream Volta defaults to `%LOCALAPPDATA%\Volta`, puts the Node image binary directly under the version directory on Windows, and uses `.cmd` shims. Those installs will not be canonicalized by this PR.
  - confidence: 0.88
