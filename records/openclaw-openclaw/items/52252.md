Full review comments:

- **[P1] Match custom Volta homes:** `src/infra/stable-node-path.ts:19`
  - body: The matcher only recognizes paths containing `/.volta/tools/image/...`, but OpenClaw already accepts `${VOLTA_HOME}/bin` in daemon service PATH handling.   - confidence: 0.94
- **[P2] Handle native Windows Volta layout:** `src/infra/stable-node-path.ts:19-25`
  - body: The Windows handling assumes a Unix-shaped `.volta/.../bin/node.exe` image and a `node.exe` shim. Volta defaults to a local-data `Volta` home on Windows, places Node image executables at the version root, and exposes command shims as `.cmd`, so native Windows Volta paths fall through unchanged.
  - confidence: 0.88
