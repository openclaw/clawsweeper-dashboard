## Summary

Keep open. The Volta daemon-path problem is valid and current main still only canonicalizes Homebrew Cellar Node paths, but this PR is incomplete because it only matches the default Unix `.volta` layout and misses custom `VOLTA_HOME`, Windows Volta image paths, and the required changelog entry.

## What This Changes

The branch adds Volta tools/image path rewriting to `resolveStableNodePath` plus default Unix shim and missing-shim fallback tests.

## Best Possible Solution

Update this PR or land a narrow replacement that derives the Volta home generically, covers Unix default/custom and Windows default/custom image layouts, preserves missing-shim fallback, adds focused tests, and includes a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a focused path: call `resolveStableNodePath` with `/opt/volta/tools/image/node/24.14.0/bin/node` or a Windows `...\Volta\tools\image\node\24.14.0\node.exe` path plus an accessible shim; the proposed regex does not match those layouts and returns the original version-pinned path.

## Solution Assessment

No, not as submitted. Rewriting Volta image paths to stable shims is the right direction, but the implementation needs to parse `<voltaHome>/tools/image/node/<version>/...` generically instead of hard-coding `.volta` and the Unix `bin/node` image shape.
