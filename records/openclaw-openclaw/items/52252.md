## Summary

Keep open: the PR targets the right daemon runtime-selection seam, but its Volta rewrite only matches default `~/.volta` layouts and still leaves supported custom Volta roots pinned to version-specific image paths.

## What This Changes

The PR adds Volta tools/image-to-shim handling in `resolveStableNodePath` plus resolver tests for the default `~/.volta` layout.

## Best Possible Solution

Generalize the Volta resolver to detect `<VOLTA_HOME>/tools/image/node/<version>/bin/node[.exe]`, return `<VOLTA_HOME>/bin/node[.exe]` when accessible, and cover default plus custom-root cases in tests.

## Reproduction Assessment

Yes, by source inspection. With an exec path like `/opt/volta/tools/image/node/24.14.0/bin/node` and an accessible `/opt/volta/bin/node`, the PR regex does not match, so the resolver still returns the version-pinned path.

## Solution Assessment

No, not as written. The resolver seam is the right place to fix this, but the pattern should be root-relative to Volta rather than hard-coded to `/.volta/`.
