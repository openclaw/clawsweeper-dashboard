## What This Changes

The PR adds Volta `tools/image` Node path canonicalization in `resolveStableNodePath`, resolving matching `~/.volta/tools/image/node/<version>/bin/node[.exe]` paths to `~/.volta/bin/node[.exe]` when the shim is accessible, plus two unit tests.

## Best Possible Solution

Canonicalize Volta image paths for default and custom Volta homes across Unix and native Windows layouts, keep the existing supported-system-Node preference and fallback behavior, add focused coverage, and attach redacted real Volta proof.

## Reproduction Assessment

Yes, source-level. Current main returns Volta image paths unchanged, and the PR branch still misses supported custom `VOLTA_HOME` and native Windows Volta layouts.

## Solution Assessment

No, not as submitted. Resolver-level canonicalization is the right boundary, but the implementation needs broader Volta layout handling and real behavior proof before merge.
