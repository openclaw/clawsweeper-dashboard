## Summary

Keep open. Current main still only canonicalizes Homebrew Cellar Node paths, while this PR’s Volta implementation is still underfixed for supported custom `VOLTA_HOME` roots and Windows Volta image layouts, matching the unresolved review discussion.

## What This Changes

The PR adds Volta `tools/image/node/<version>` handling to `resolveStableNodePath` plus tests for resolving an accessible default Unix Volta shim and falling back when the shim is missing.

## Best Possible Solution

Update this PR, or land a narrow replacement, so Volta image-path rewriting is layout-aware across default Unix, custom `VOLTA_HOME`, Windows default/custom homes, and missing-shim fallback while keeping the existing daemon runtime selection order intact.

## Reproduction Assessment

Yes. A focused unit check can call `resolveStableNodePath` with `/opt/volta/tools/image/node/24.14.0/bin/node` or a Windows `...\Volta\tools\image\node\24.14.0\node.exe` path plus an accessible shim; the submitted regex still returns the original path for those cases.

## Solution Assessment

No, not as submitted. Rewriting Volta image paths to stable shims is the right direction, but the patch needs to derive the Volta home generically and cover Windows layout instead of matching only `~/.volta/.../bin/node`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Match Volta home layouts instead of only ~/.volta:** `src/infra/stable-node-path.ts:19`
  - body: The new regex only accepts paths containing `/.volta/tools/image/.../bin/node`, so an install with `VOLTA_HOME=/opt/volta` still returns `/opt/volta/tools/image/node/<ver>/bin/node`. OpenClaw already preserves `VOLTA_HOME/bin` in service PATH, so this supported setup remains pinned to a stale image path after Volta changes versions.
  - confidence: 0.94
- **[P2] Handle Windows Volta image and shim layout explicitly:** `src/infra/stable-node-path.ts:19-25`
  - body: The added Windows branch looks for `.../bin/node.exe` under a `.volta` home and returns `.volta/bin/node.exe`, but Windows Volta homes default to `%LOCALAPPDATA%\Volta`, and Node image binaries are reported directly under the version directory. As written, Windows Volta installs will not be rewritten to the stable shim.
  - confidence: 0.89
